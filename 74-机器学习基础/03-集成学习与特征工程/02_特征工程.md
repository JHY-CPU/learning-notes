## 1. 特征工程概述


特征工程是从原始数据中提取、变换、选择特征的过程，直接影响模型的性能上限。Andrew Ng 曾说："应用机器学习基本上就是特征工程。"


### 1.1 特征工程的流程


1. **数据理解**
   ：理解业务和数据含义
2. **特征构造**
   ：从原始数据提取有意义的特征
3. **特征清洗**
   ：处理缺失值、异常值
4. **特征变换**
   ：缩放、编码、非线性变换
5. **特征选择**
   ：选择最相关的特征子集
6. **特征降维**
   ：PCA 等方法减少维度


### 1.2 为什么特征工程重要


- 好的特征可以弥补模型的不足
- 差的特征即使最好的模型也无能为力
- "数据和特征决定了上限，模型和算法只是逼近这个上限"


## 2. 特征缩放


特征缩放使不同量纲的特征具有可比性，对基于距离和梯度的算法至关重要。


### 2.1 标准化 (Z-score Normalization)


$$
x' = (x - μ) / σ
$$


- 使特征均值为 0，标准差为 1
- 不改变分布形状
- 对异常值有一定鲁棒性
- 适用：SVM、逻辑回归、神经网络


### 2.2 归一化 (Min-Max Scaling)


$$
x' = (x - xₘᵢₙ) / (xₘₐₓ - xₘᵢₙ)
$$


- 缩放到 [0, 1] 区间
- 对异常值敏感（最大最小值受极端值影响）
- 适用：图像像素值、需要有界输入的场景


### 2.3 鲁棒缩放 (Robust Scaling)


$$
x' = (x - median) / IQR
$$


- 使用中位数和四分位距（IQR = Q3 - Q1）
- 对异常值非常鲁棒
- 适用：数据中存在较多异常值


### 2.4 哪些算法需要缩放


| 需要缩放 | 不需要缩放 |
| --- | --- |
| SVM、KNN、神经网络 | 决策树、随机森林 |
| 线性/逻辑回归（梯度下降） | Naive Bayes |
| PCA、LDA | 基于树的 Boosting 模型 |


> **Note:** **关键原则：**
> 涉及距离计算、梯度优化的算法需要缩放；基于决策树分裂的算法不需要。注意：应在训练集上 fit 缩放器，然后 transform 训练集和测试集，避免数据泄露。


## 3. 特征编码


将非数值型特征（类别、文本）转换为数值表示。


### 3.1 标签编码 (Label Encoding)


$$
红 → 0, 绿 → 1, 蓝 → 2
$$


- 为每个类别分配一个整数
- 引入了不存在的序关系（0 < 1 < 2）
- 适用：有序类别特征（如低/中/高）
- 树模型中可以接受


### 3.2 独热编码 (One-Hot Encoding)


$$
红 → [1, 0, 0], 绿 → [0, 1, 0], 蓝 → [0, 0, 1]
$$


- 为每个类别创建一个二值特征
- 不引入序关系
- 类别数多时导致维度爆炸（稀疏矩阵）
- 适用：类别数较少（<15）的无序特征


### 3.3 目标编码 (Target Encoding)


$$
类别 c 的编码 = 目标变量在类别 c 上的均值
$$


- 用目标变量的统计量（均值）来编码类别
- 不会增加维度
- 容易导致过拟合和目标泄露
- 需要正则化（如加先验平滑、使用交叉验证）
- 适用：高基数类别特征（类别数很多）


### 3.4 其他编码方法


| 方法 | 原理 | 适用 |
| --- | --- | --- |
| 频率编码 | 用类别出现频率代替 | 高基数特征 |
| 二进制编码 | 整数转二进制位 | 中等基数 |
| Base-N 编码 | 整数转 N 进制 | 中等基数 |
| Hash 编码 | 哈希到固定维度 | 极高基数 |


## 4. 缺失值处理


### 4.1 缺失值的类型


- **MCAR（完全随机缺失）**
   ：缺失与任何变量无关
- **MAR（随机缺失）**
   ：缺失与其他变量有关
- **MNAR（非随机缺失）**
   ：缺失与自身值有关（最难处理）


### 4.2 处理方法


| 方法 | 做法 | 优缺点 |
| --- | --- | --- |
| 删除行 | 去掉含缺失值的样本 | 简单，但丢失数据 |
| 删除列 | 去掉缺失率高的特征 | 适用于缺失率>50% |
| 均值/中位数填充 | 用统计量填充 | 简单，但可能引入偏差 |
| 众数填充 | 用最频繁的值填充 | 适用于类别特征 |
| KNN 填充 | 用最近邻的值填充 | 利用样本间关系 |
| 模型预测 | 用其他特征预测缺失值 | 效果好但复杂 |
| 添加缺失指示器 | 添加是否缺失的标记特征 | 保留缺失信息 |


> **Tip:** **实践建议：**
> 树模型（XGBoost、LightGBM）能自动处理缺失值，无需手动填充。其他模型需要预处理。添加"是否缺失"的标记特征通常有益。


## 5. 特征选择


特征选择去除无关和冗余特征，提高模型性能、减少过拟合、加速训练。


### 5.1 过滤法 (Filter)


独立于模型，根据统计指标筛选特征：


- **方差阈值**
   ：去除方差过低（近似常数）的特征
- **相关系数**
   ：选择与目标变量相关性高的特征
- **互信息**
   ：衡量特征与目标的非线性依赖关系
- **卡方检验**
   ：适用于分类任务的离散特征
- **ANOVA F 检验**
   ：适用于分类任务的连续特征


优点：速度快。缺点：忽略了特征之间的交互。


### 5.2 包装法 (Wrapper)


使用模型性能作为评价标准搜索特征子集：


- **前向选择**
   ：从空集开始，每次添加最有贡献的特征
- **后向消除**
   ：从全集开始，每次删除最不重要的特征
- **递归特征消除（RFE）**
   ：反复训练模型，每次删掉最不重要的特征


优点：考虑特征交互。缺点：计算量大。


### 5.3 嵌入法 (Embedded)


模型训练过程中自动进行特征选择：


- **L1 正则化（Lasso）**
   ：将不重要特征的系数压缩为 0
- **树模型特征重要性**
   ：基于不纯度减少量或排列重要性
- **SelectFromModel**
   ：基于模型权重自动选择


优点：兼顾效率和效果。缺点：依赖于特定模型。


## 6. 特征交叉与构造


### 6.1 特征交叉 (Feature Crossing)


将两个或多个特征组合，捕捉非线性关系和交互效应：


- **乘法交叉**
   ：x₁ × x₂（如：面积 = 长 × 宽）
- **除法交叉**
   ：x₁ / x₂（如：密度 = 质量 / 体积）
- **多项式交叉**
   ：x₁², x₁x₂, x₂² 等


### 6.2 时间特征构造


- 年、月、日、星期几、小时
- 是否周末、是否节假日
- 距离某个日期的天数
- 周期性编码：sin(2π·hour/24), cos(2π·hour/24)


### 6.3 聚合特征


- 分组统计：按用户/商品分组的均值、计数、最大/最小值
- 滑动窗口：最近 N 天的统计量
- 排名特征：在组内的排名


### 6.4 多项式特征


sklearn 的 PolynomialFeatures 自动生成多项式交叉特征：


```
from sklearn.preprocessing import PolynomialFeatures
poly = PolynomialFeatures(degree=2, interaction_only=False)
X_poly = poly.fit_transform(X)
# [1, a, b] → [1, a, b, a², ab, b²]
```


## 7. 文本特征工程


### 7.1 词袋模型 (Bag of Words)


统计每个词在文档中出现的次数，忽略顺序。


### 7.2 TF-IDF


$$
TF(t,d) = 词 t 在文档 d 中出现的次数 / 文档 d 的总词数
                IDF(t) = log(文档总数 / 包含词 t 的文档数)
                TF-IDF = TF × IDF
$$


核心思想：一个词在当前文档中频繁出现但在其他文档中少见，则它对当前文档很重要。


### 7.3 词嵌入 (Word Embeddings)


- **Word2Vec**
   ：CBOW / Skip-gram，学习词的分布式表示
- **GloVe**
   ：基于全局共现矩阵的词向量
- **FastText**
   ：考虑子词信息，能处理未登录词


## 8. Python 代码实现


### 8.1 完整的特征工程 Pipeline


```
import pandas as pd
from sklearn.pipeline import Pipeline
from sklearn.compose import ColumnTransformer
from sklearn.preprocessing import StandardScaler, OneHotEncoder
from sklearn.impute import SimpleImputer

# 定义数值和类别特征的处理
numeric_features = ['age', 'income']
categorical_features = ['city', 'gender']

numeric_transformer = Pipeline([
    ('imputer', SimpleImputer(strategy='median')),
    ('scaler', StandardScaler())
])

categorical_transformer = Pipeline([
    ('imputer', SimpleImputer(strategy='most_frequent')),
    ('encoder', OneHotEncoder(handle_unknown='ignore'))
])

preprocessor = ColumnTransformer([
    ('num', numeric_transformer, numeric_features),
    ('cat', categorical_transformer, categorical_features)
])
```

## 9. 总结


1. **特征缩放**
   ：Z-score 适用于大多数场景，树模型不需要
2. **编码**
   ：低基数用 One-Hot，高基数用 Target Encoding，有序用 Label
3. **缺失值**
   ：树模型自动处理，其他用中位数/KNN 填充 + 缺失指示器
4. **特征选择**
   ：过滤法快但粗糙，嵌入法效果好，包装法最精确
5. **特征构造**
   ：时间/聚合/交叉特征往往带来最大提升
6. **避免数据泄露**
   ：所有预处理都应在训练集上 fit，然后 transform 测试集
