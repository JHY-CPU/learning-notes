## 1. 决策树基础


决策树是一种基于树结构进行决策的监督学习算法。通过对特征空间进行递归划分，构建一棵从根节点到叶节点的树，每个内部节点表示一个特征判断，每个叶节点表示一个类别或回归值。


### 1.1 核心思想


- 选择最优特征对数据集进行划分
- 递归地对子数据集进行划分
- 直到满足停止条件（纯度足够高、达到最大深度等）


### 1.2 关键问题


1. **特征选择**
   ：用什么标准选择最优划分特征？
2. **划分方式**
   ：连续特征如何划分？
3. **停止条件**
   ：何时停止分裂？
4. **剪枝策略**
   ：如何防止过拟合？


## 2. ID3 算法 — 信息增益


ID3（Iterative Dichotomiser 3）由 Quinlan 于 1986 年提出，使用**信息增益**作为特征选择标准。


### 2.1 信息熵 (Entropy)


信息熵衡量数据集的纯度（不确定性）：


$$
H(D) = -Σₖ₌₁ᴷ pₖ log₂(pₖ)
$$


其中 pₖ 是第 k 类样本在数据集 D 中的比例。熵越小，数据越纯。


- 当数据集完全纯（只有一个类）时，H(D) = 0
- 当各类均匀分布时，H(D) 最大 = log₂(K)


### 2.2 条件熵


$$
H(D|A) = Σᵥ (|Dᵥ|/|D|) H(Dᵥ)
$$


其中 Dᵥ 是特征 A 取值为 v 的子数据集。


### 2.3 信息增益


$$
g(D, A) = H(D) - H(D|A)
$$


信息增益 = 划分前的熵 - 划分后的加权熵。信息增益越大，说明该特征的分类能力越强。


### 2.4 ID3 的局限


- 偏向取值多的特征（取值越多，条件熵可能越小）
- 只能处理离散特征
- 没有剪枝策略
- 不能处理缺失值


## 3. C4.5 算法 — 信息增益率


C4.5 是 ID3 的改进版，由同一作者 Quinlan 于 1993 年提出。


### 3.1 信息增益率


$$
gᴿ(D, A) = g(D, A) / Hₐ(D)
$$


其中 Hₐ(D) = -Σᵥ (|Dᵥ|/|D|) log₂(|Dᵥ|/|D|) 是特征 A 的固有值（Intrinsic Value），取值越多的特征，固有值越大。


### 3.2 C4.5 的改进


| 改进 | 说明 |
| --- | --- |
| 使用增益率 | 克服 ID3 偏向多值特征的问题 |
| 连续特征处理 | 通过二分法将连续值离散化 |
| 缺失值处理 | 使用概率权重分配缺失样本 |
| 悲观剪枝 | 使用统计方法进行后剪枝 |


## 4. CART 算法 — 基尼指数


CART（Classification and Regression Tree）由 Breiman 等人于 1984 年提出，是 scikit-learn 中决策树的默认实现。


### 4.1 基尼指数 (Gini Impurity)


$$
Gini(D) = 1 - Σₖ₌₁ᴷ pₖ²
$$


基尼指数表示从数据集中随机抽取两个样本，它们类别不一致的概率。值越小越纯。


- 完全纯时 Gini = 0
- K 类均匀分布时 Gini = 1 - 1/K


### 4.2 划分标准


对于特征 A 的某个划分点，划分后的基尼指数为：


$$
Gini(D, A) = (|D₁|/|D|) Gini(D₁) + (|D₂|/|D|) Gini(D₂)
$$


选择使 Gini(D, A) 最小的特征和划分点。


### 4.3 CART 的特点


- **二叉树结构**
   ：每次只将数据分为两组（非多叉树）
- **同时支持分类和回归**
   ：分类用基尼指数，回归用 MSE
- **连续特征**
   ：在所有可能的二分点中选择最优


### 4.4 三种算法对比


| 算法 | 划分标准 | 树结构 | 特征类型 |
| --- | --- | --- | --- |
| ID3 | 信息增益 | 多叉树 | 仅离散 |
| C4.5 | 信息增益率 | 多叉树 | 离散+连续 |
| CART | 基尼指数/MSE | 二叉树 | 离散+连续 |


## 5. 剪枝策略


决策树容易过拟合，剪枝是防止过拟合的主要手段。


### 5.1 预剪枝 (Pre-pruning)


在树的构建过程中提前停止分裂：


- **最大深度限制**
   ：max_depth 参数
- **最小样本数**
   ：节点样本数小于阈值时停止
- **最小信息增益**
   ：增益小于阈值时停止
- **最大叶子数**
   ：max_leaf_nodes 参数


优点：简单高效，降低过拟合风险。缺点：可能"贪心地"停止过早，导致欠拟合。


### 5.2 后剪枝 (Post-pruning)


先生成完整的树，再自底向上修剪：


- **代价复杂度剪枝（CCP）**
   ：最小化 T(树) = Σ_leaf Nₜ·Gini(t) + α·|T_leaf|
- **悲观剪枝**
   ：C4.5 使用，基于统计置信区间


sklearn 中通过 `ccp_alpha` 参数控制后剪枝强度。


> **Tip:** **实践建议：**
> 一般先用预剪枝（设置 max_depth 和 min_samples_leaf），再用交叉验证选择最优超参数。后剪枝（ccp_alpha）通常效果更好但计算更慢。


## 6. 随机森林


随机森林（Random Forest）是基于 Bagging（Bootstrap Aggregating）的集成学习方法，通过构建多棵决策树并投票/平均来提高泛化能力。


### 6.1 Bagging 原理


1. 从原始训练集有放回地抽取 n 个 Bootstrap 样本
2. 每个样本训练一棵决策树
3. 分类任务：投票法；回归任务：平均法


### 6.2 随机森林的"随机"


随机森林有两层随机性：


1. **样本随机**
   ：每棵树使用不同的 Bootstrap 样本（约 63.2% 的原始样本）
2. **特征随机**
   ：每个节点只从随机选取的 √p（分类）或 p/3（回归）个特征中选最优


特征随机使得各棵树之间的差异更大，降低了集成的相关性，提高了泛化能力。


### 6.3 袋外误差 (OOB Error)


每棵 Bootstrap 样本约有 36.8% 的数据未被选中（袋外数据），可用于无偏估计泛化误差，相当于内置了交叉验证。


### 6.4 随机森林 vs 单棵决策树


| 特性 | 决策树 | 随机森林 |
| --- | --- | --- |
| 方差 | 高（容易过拟合） | 低（集成降低方差） |
| 偏差 | 低 | 略高（受 Bootstrap 影响） |
| 可解释性 | 高 | 较低（多棵树的集成） |
| 训练速度 | 快 | 慢（但可并行） |
| 抗噪能力 | 弱 | 强 |


## 7. 特征重要性


决策树和随机森林天然支持特征重要性评估。


### 7.1 基于不纯度的重要性


计算每个特征在所有树的所有节点上带来的不纯度减少的加权平均：


$$
Importance(j) = Σₜ Σ_node (Nₜ_node / N) · ΔGini(node, j)
$$


其中 Nₜ_node 是节点的样本数，ΔGini 是使用特征 j 划分后的基尼减少量。


### 7.2 基于排列的重要性


随机打乱某个特征的值，观察模型性能下降程度。下降越多，特征越重要。这种方法更可靠，不受特征取值数量偏差影响。


```
from sklearn.inspection import permutation_importance
result = permutation_importance(model, X_test, y_test, n_repeats=10)
```


## 8. Python 代码实现


### 8.1 决策树


```
from sklearn.tree import DecisionTreeClassifier, DecisionTreeRegressor, export_text, plot_tree
from sklearn.model_selection import cross_val_score

# 分类树
clf = DecisionTreeClassifier(
    criterion='gini',        # 'gini' 或 'entropy'
    max_depth=5,             # 最大深度（预剪枝）
    min_samples_split=10,    # 节点最小样本数才分裂
    min_samples_leaf=5,      # 叶子节点最小样本数
    ccp_alpha=0.01,          # 代价复杂度剪枝参数
    random_state=42
)
clf.fit(X_train, y_train)

# 可视化树结构
print(export_text(clf, feature_names=feature_names))
# 或用 matplotlib
plot_tree(clf, feature_names=feature_names, class_names=class_names, filled=True)
```


### 8.2 随机森林


```
from sklearn.ensemble import RandomForestClassifier

rf = RandomForestClassifier(
    n_estimators=100,        # 树的数量
    max_depth=10,            # 每棵树的最大深度
    max_features='sqrt',     # 每次分裂考虑的特征数
    min_samples_leaf=5,
    oob_score=True,          # 计算袋外分数
    n_jobs=-1,               # 并行计算
    random_state=42
)
rf.fit(X_train, y_train)

print(f"OOB 分数: {rf.oob_score_:.4f}")
print(f"测试准确率: {rf.score(X_test, y_test):.4f}")

# 特征重要性
import pandas as pd
importance = pd.Series(rf.feature_importances_, index=feature_names)
print(importance.sort_values(ascending=False))
```


## 9. 总结


1. **ID3**
   用信息增益，偏向多值特征；
   **C4.5**
   用增益率修正，支持连续值和缺失值
2. **CART**
   用基尼指数，生成二叉树，是 sklearn 默认实现
3. 决策树需
   **剪枝**
   防过拟合：预剪枝限制深度/样本数，后剪枝用 CCP
4. **随机森林**
   通过 Bagging + 特征随机显著降低方差，是最实用的通用模型之一
5. 随机森林天然支持
   **特征重要性**
   评估和
   **OOB 误差**
   估计


> **Tip:** **实践要点：**
> 随机森林通常作为首选的基线模型之一。对缺失值和异常值鲁棒，无需特征缩放。主要缺点是模型较大、预测速度较慢。
