第一节：线性回归

## 1. 线性回归 (Linear Regression)


线性回归是监督学习中最基础、最经典的回归算法，用于建立自变量（特征）与因变量（目标）之间的线性关系模型。


### 1.1 基本模型


线性回归假设目标变量 y 与特征 x 之间存在线性关系：


$$
h(x) = w₀ + w₁x₁ + w₂x₂ + ... + wₙxₙ = wᵀx
$$


其中：


- **w₀**
   ：偏置项（截距），也记作 b 或 θ₀
- **w₁, w₂, ..., wₙ**
   ：各特征的权重系数
- **x₁, x₂, ..., xₙ**
   ：输入特征


### 1.2 损失函数 — 均方误差 (MSE)


线性回归使用均方误差作为损失函数，衡量预测值与真实值之间的差距：


$$
J(w) = (1/2m) Σᵢ₌₁ᵐ (h(x⁽ⁱ⁾) - y⁽ⁱ⁾)²
$$


其中 m 为样本数量。系数 1/2 是为了求导时消去平方的系数，方便计算。


### 1.3 线性回归的基本假设


1. **线性关系**
   ：自变量与因变量之间存在线性关系
2. **独立性**
   ：样本之间相互独立
3. **同方差性**
   ：误差项的方差恒定（Homoscedasticity）
4. **正态性**
   ：误差项服从正态分布
5. **无多重共线性**
   ：自变量之间不存在高度相关

第二节：最小二乘法

## 2. 最小二乘法 (Ordinary Least Squares, OLS)


最小二乘法是求解线性回归参数的解析方法（闭式解），通过最小化残差平方和来求解最优参数。


### 2.1 数学推导


目标是最小化损失函数 J(w)，对 w 求偏导并令其为零：


$$
∂J/∂w = (1/m) Xᵀ(Xw - y) = 0
$$


解得正规方程（Normal Equation）：


$$
w* = (XᵀX)⁻¹Xᵀy
$$


### 2.2 OLS 的特点


| 优点 | 缺点 |
| --- | --- |
| 有闭式解，不需要迭代 | 当 XᵀX 不可逆时无法求解 |
| 计算结果精确 | 时间复杂度 O(n³)，大规模数据慢 |
| 不需要调节学习率 | 特征维度高时（n > 10000）不适用 |
| 全局最优解 | 对异常值敏感 |


> **Note:** **注意：**
> 当特征数量 n 很大时，(XᵀX) 的求逆运算非常耗时。此时应使用梯度下降法迭代求解。一般经验：n < 10000 时可用 OLS，否则用梯度下降。


### 2.3 多重共线性问题


当特征之间高度相关时，XᵀX 接近奇异矩阵，求逆不稳定，导致参数估计值方差很大。解决方案包括：


- 使用正则化（Ridge 回归添加 L2 正则项）
- 使用主成分分析（PCA）降维
- 删除高度相关的特征

第三节：梯度下降

## 3. 梯度下降法 (Gradient Descent)


梯度下降是一种迭代优化算法，通过沿损失函数梯度的反方向更新参数，逐步逼近最优解。


### 3.1 参数更新规则


$$
wⱼ := wⱼ - α · (∂J/∂wⱼ)
$$


其中 α 是学习率（learning rate），控制每次更新的步长。在线性回归中，偏导数为：


$$
∂J/∂wⱼ = (1/m) Σᵢ₌₁ᵐ (h(x⁽ⁱ⁾) - y⁽ⁱ⁾) · xⱼ⁽ⁱ⁾
$$


### 3.2 梯度下降的三种变体


| 变体 | 每步使用数据量 | 特点 |
| --- | --- | --- |
| **批量梯度下降 (BGD)** | 全部 m 个样本 | 收敛稳定，但每步计算量大 |
| **随机梯度下降 (SGD)** | 1 个样本 | 更新频繁，可能震荡，但速度快 |
| **小批量梯度下降 (Mini-batch)** | b 个样本（通常 32/64/128） | 兼顾效率与稳定性，最常用 |


### 3.3 学习率的选择


- **太大**
   ：损失函数震荡甚至发散，无法收敛
- **太小**
   ：收敛速度极慢，可能陷入局部最优
- **合适范围**
   ：通常从 0.001、0.01、0.1 开始尝试


> **Tip:** **实用技巧：**
> 使用学习率衰减策略（如指数衰减、余弦退火），或使用自适应学习率优化器（Adam、AdaGrad、RMSProp）。


### 3.4 收敛判断


- 损失函数变化量小于阈值 ε
- 参数变化量小于阈值
- 达到最大迭代次数

第四节：逻辑回归

## 4. 逻辑回归 (Logistic Regression)


逻辑回归虽然名字中带有"回归"，但实际上是一种**分类算法**。它通过 Sigmoid 函数将线性回归的输出映射到 [0,1] 区间，表示样本属于正类的概率。


### 4.1 Sigmoid 函数


$$
σ(z) = 1 / (1 + e⁻ᶻ)
$$


Sigmoid 函数的特性：


- 输出范围 (0, 1)，适合表示概率
- 关于 (0, 0.5) 中心对称
- 导数有简洁形式：σ'(z) = σ(z)(1 - σ(z))
- 当 z → +∞ 时 σ(z) → 1，z → -∞ 时 σ(z) → 0


### 4.2 模型定义


$$
h(x) = σ(wᵀx) = 1 / (1 + e⁻ʷᵀˣ)
$$


决策边界：当 h(x) ≥ 0.5 时预测为正类（y=1），否则预测为负类（y=0）。即当 wᵀx ≥ 0 时预测为正类。


### 4.3 损失函数 — 交叉熵 (Cross-Entropy)


逻辑回归使用对数损失（交叉熵损失）：


$$
J(w) = -(1/m) Σᵢ₌₁ᵐ [y⁽ⁱ⁾log(h(x⁽ⁱ⁾)) + (1-y⁽ⁱ⁾)log(1-h(x⁽ⁱ⁾))]
$$


这个损失函数的特点：


- 当 y=1 且 h(x)→1 时，损失→0（正确预测无惩罚）
- 当 y=1 且 h(x)→0 时，损失→∞（严重错误被大幅惩罚）
- 损失函数是凸函数，保证全局最优解


> **Warning:** **切勿使用 MSE 作为逻辑回归的损失函数！**
> 因为 Sigmoid + MSE 组合会导致损失函数非凸，存在多个局部最优解，梯度下降难以收敛到全局最优。


### 4.4 多分类扩展


逻辑回归本身是二分类算法，多分类可通过以下方式实现：


- **One-vs-Rest (OvR)**
   ：训练 K 个二分类器，每个区分一类与其余所有类
- **One-vs-One (OvO)**
   ：训练 C(K,2) 个二分类器，每对类别一个
- **Softmax 回归**
   ：直接推广为多分类，使用 Softmax 函数代替 Sigmoid


$$
Softmax: P(y=k|x) = e^(wₖᵀx) / Σⱼ e^(wⱼᵀx)
$$

第五节：正则化

## 5. 正则化 (Regularization)


正则化通过在损失函数中添加惩罚项来限制模型复杂度，防止过拟合。


### 5.1 L2 正则化 (Ridge)


$$
J(w) = 原始损失 + (λ/2m) Σⱼ wⱼ²
$$


- 使权重趋向于较小的值，但不会精确为零
- 有效处理多重共线性问题
- λ 越大，正则化越强，模型越简单


### 5.2 L1 正则化 (Lasso)


$$
J(w) = 原始损失 + (λ/m) Σⱼ |wⱼ|
$$


- 能产生稀疏解，部分权重精确为零
- 具有特征选择功能，自动去除不重要特征
- 在特征数量远大于样本数量时特别有用


### 5.3 Elastic Net


$$
J(w) = 原始损失 + (λ₁/m) Σⱼ |wⱼ| + (λ₂/2m) Σⱼ wⱼ²
$$


结合 L1 和 L2 的优势，既能做特征选择，又能处理共线性。


### 5.4 L1 vs L2 对比


| 特性 | L1 (Lasso) | L2 (Ridge) |
| --- | --- | --- |
| 惩罚项 | Σ\|wⱼ\| | Σwⱼ² |
| 解的特性 | 稀疏解（部分权重为0） | 权重趋小但非零 |
| 特征选择 | 是 | 否 |
| 适用场景 | 高维稀疏数据 | 特征间存在共线性 |
| 求解 | 不可导点，需特殊处理 | 处处可导，易优化 |
| 几何形状 | 菱形约束区域 | 圆形约束区域 |

第六节：对比

## 6. 线性回归 vs 逻辑回归


#### 线性回归


- **任务类型：**
   回归（连续值预测）
- **输出范围：**
   (-∞, +∞)
- **损失函数：**
   均方误差 MSE
- **激活函数：**
   无（恒等映射）
- **应用场景：**
   房价预测、销量预测、温度预测


#### 逻辑回归


- **任务类型：**
   分类（离散类别预测）
- **输出范围：**
   (0, 1) 概率值
- **损失函数：**
   交叉熵 Cross-Entropy
- **激活函数：**
   Sigmoid（二分类）/ Softmax（多分类）
- **应用场景：**
   垃圾邮件检测、疾病诊断、信用评分


### 共同点


- 都假设特征与目标之间存在某种线性关系
- 都可以使用梯度下降法优化
- 都可以添加正则化项
- 模型可解释性强，训练速度快
- 对特征的尺度敏感，通常需要特征标准化

第七节：代码实现

## 7. Python 代码实现


### 7.1 从零实现线性回归


```
import numpy as np

class LinearRegression:
    def __init__(self, lr=0.01, n_iters=1000):
        self.lr = lr
        self.n_iters = n_iters
        self.weights = None
        self.bias = None
    def fit(self, X, y):
        n_samples, n_features = X.shape
        self.weights = np.zeros(n_features)
        self.bias = 0
        for _ in range(self.n_iters):
            y_pred = np.dot(X, self.weights) + self.bias
            # 计算梯度
            dw = (1 / n_samples) * np.dot(X.T, (y_pred - y))
            db = (1 / n_samples) * np.sum(y_pred - y)
            # 更新参数
            self.weights -= self.lr * dw
            self.bias -= self.lr * db

    def predict(self, X):
        return np.dot(X, self.weights) + self.bias
```


### 7.2 从零实现逻辑回归


```
class LogisticRegression:
    def __init__(self, lr=0.01, n_iters=1000):
        self.lr = lr
        self.n_iters = n_iters

    def _sigmoid(self, z):
        return 1 / (1 + np.exp(-z))

    def fit(self, X, y):
        n_samples, n_features = X.shape
        self.weights = np.zeros(n_features)
        self.bias = 0
        for _ in range(self.n_iters):
            linear = np.dot(X, self.weights) + self.bias
            y_pred = self._sigmoid(linear)
            # 计算梯度（交叉熵的导数）
            dw = (1 / n_samples) * np.dot(X.T, (y_pred - y))
            db = (1 / n_samples) * np.sum(y_pred - y)
            self.weights -= self.lr * dw
            self.bias -= self.lr * db

    def predict(self, X, threshold=0.5):
        linear = np.dot(X, self.weights) + self.bias
        y_pred = self._sigmoid(linear)
        return [1 if i > threshold else 0 for i in y_pred]
```


### 7.3 使用 scikit-learn


```
from sklearn.linear_model import LinearRegression, LogisticRegression, Ridge, Lasso
from sklearn.preprocessing import StandardScaler
from sklearn.model_selection import train_test_split
from sklearn.metrics import mean_squared_error, accuracy_score

# 数据准备
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

# 特征标准化（重要！）
scaler = StandardScaler()
X_train = scaler.fit_transform(X_train)
X_test = scaler.transform(X_test)

# 线性回归
model = LinearRegression()
model.fit(X_train, y_train)
print(f"权重: {model.coef_}, 截距: {model.intercept_}")

# Ridge 回归 (L2)
ridge = Ridge(alpha=1.0)
ridge.fit(X_train, y_train)

# Lasso 回归 (L1)
lasso = Lasso(alpha=0.1)
lasso.fit(X_train, y_train)

# 逻辑回归
clf = LogisticRegression(penalty='l2', C=1.0, max_iter=1000)
clf.fit(X_train, y_train)
y_pred = clf.predict(X_test)
print(f"准确率: {accuracy_score(y_test, y_pred)}")
```

第八节：总结

## 8. 总结与实践建议


### 核心要点


1. **线性回归**
   用于回归任务，最小化均方误差，可用 OLS（解析解）或梯度下降（迭代解）求解
2. **逻辑回归**
   用于分类任务，通过 Sigmoid 将线性输出映射为概率，使用交叉熵损失
3. **正则化**
   防止过拟合：L1 产生稀疏解（特征选择），L2 使权重平滑
4. 两种模型都对特征尺度敏感，训练前应进行
   **特征标准化**


### 实践建议


> **Tip:** **选型指南：**
>
> - 预测连续值 → 线性回归 / Ridge / Lasso
> - 二分类问题 → 逻辑回归（作为基线模型首选）
> - 特征数 > 样本数 → Lasso（L1 正则化）
> - 特征间有共线性 → Ridge（L2 正则化）
> - 不确定选哪个 → Elastic Net


### 常见面试问题


1. 为什么逻辑回归不用 MSE 而用交叉熵作为损失函数？（非凸性）
2. L1 正则化为什么能产生稀疏解？（几何角度：菱形约束区域更容易在坐标轴上取到交点）
3. 逻辑回归的假设是什么？（对数几率与特征线性相关，无多重共线性）
4. 什么时候用 OLS，什么时候用梯度下降？（数据量和特征数的权衡）
