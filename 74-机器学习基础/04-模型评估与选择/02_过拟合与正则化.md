## 1. 偏差-方差权衡


偏差-方差分解是理解模型泛化误差的核心框架。


### 1.1 误差分解


对于回归问题，泛化误差可以分解为：


$$
E[(y - f̂(x))²] = Bias²(f̂(x)) + Var(f̂(x)) + σ²
$$


- **偏差 (Bias)**
   ：模型预测的期望值与真实值的差距，衡量模型的系统性错误
- **方差 (Variance)**
   ：模型在不同训练集上预测的波动程度，衡量模型的稳定性
- **不可约误差 (σ²)**
   ：数据本身的噪声，无法通过模型改进消除


### 1.2 偏差与方差的权衡


#### 高偏差


模型太简单


欠拟合


- 训练误差高
- 测试误差高
- 例：线性模型拟合非线性数据


#### 理想状态


偏差方差平衡


最佳泛化


- 训练误差适中
- 测试误差最低
- 模型复杂度最优


#### 高方差


模型太复杂


过拟合


- 训练误差很低
- 测试误差高
- 例：高阶多项式拟合少量数据


### 1.3 不同模型的偏差-方差特性


| 模型 | 偏差 | 方差 |
| --- | --- | --- |
| 线性回归 | 高 | 低 |
| 决策树（不剪枝） | 低 | 高 |
| 随机森林 | 略高 | 低 |
| KNN（K 小） | 低 | 高 |
| KNN（K 大） | 高 | 低 |
| 神经网络（深层） | 低 | 高 |


## 2. 过拟合与欠拟合


### 2.1 过拟合的识别


- 训练集表现优秀，测试集表现差
- 训练误差与验证误差差距大
- 学习曲线中验证分数远低于训练分数
- 模型过于复杂（参数多、层数深、树过深）


### 2.2 欠拟合的识别


- 训练集和测试集表现都不好
- 训练误差本身就高
- 模型过于简单，无法捕捉数据模式


### 2.3 解决策略总览


| 策略 | 过拟合 | 欠拟合 |
| --- | --- | --- |
| 模型复杂度 | 降低（简化模型） | 增加（更复杂模型） |
| 数据量 | 增加训练数据 | 收集更多相关数据 |
| 特征 | 减少/选择特征 | 增加/构造特征 |
| 正则化 | 加入正则化 | 减小正则化强度 |
| 训练 | 早停 | 训练更久 |


## 3. L1/L2 正则化


### 3.1 L2 正则化 (Ridge / Weight Decay)


$$
J(w) = L(y, f(x)) + λ Σⱼ wⱼ²
$$


- 惩罚权重的平方，使权重趋向较小值
- 权重平滑分布，不会精确为零
- λ 越大，正则化越强
- 在神经网络中也叫"权重衰减"（Weight Decay）


### 3.2 L1 正则化 (Lasso)


$$
J(w) = L(y, f(x)) + λ Σⱼ |wⱼ|
$$


- 惩罚权重的绝对值，产生稀疏解
- 自动进行特征选择（不重要特征权重为 0）
- 适合高维稀疏数据


### 3.3 L1 vs L2 几何直觉


- L1 约束区域是菱形，更容易在坐标轴上与损失等高线相切（产生零值）
- L2 约束区域是圆形，切点通常不在坐标轴上（权重趋小但非零）


### 3.4 神经网络中的权重衰减


在参数更新时直接缩小权重：


$$
w := w - lr · ∇L - lr · λ · w = (1 - lr·λ) · w - lr · ∇L
$$


每步将权重缩小 (1 - lr·λ) 倍，等价于 L2 正则化。


## 4. Dropout 正则化


Dropout 是神经网络特有的正则化方法，训练时随机"丢弃"一部分神经元。


### 4.1 工作原理


- **训练时**
   ：每个神经元以概率 p 被置零（通常 p = 0.2~0.5）
- **测试时**
   ：使用所有神经元，输出乘以 (1-p)（或训练时除以 1-p，即 Inverted Dropout）


### 4.2 为什么有效


- 防止神经元之间的共适应（co-adaptation）
- 等价于训练指数级多个子网络的集成
- 迫使网络学习更鲁棒的特征


### 4.3 使用建议


- p = 0.5 用于全连接层（最常用）
- p = 0.2 用于卷积层
- 不要在输出层使用
- 与 Batch Normalization 同时使用时需谨慎


## 5. 早停法 (Early Stopping)


在训练过程中监控验证集性能，当性能不再提升时提前终止训练。


### 5.1 算法


1. 将数据分为训练集和验证集
2. 每个 epoch 在验证集上评估性能
3. 记录验证集最优性能和对应模型参数
4. 如果连续 patience 个 epoch 验证性能没有改善，停止训练
5. 返回最优性能对应的模型参数


### 5.2 优点


- 简单有效，计算开销小
- 隐式地控制模型复杂度（训练时间越长，模型越复杂）
- 可以与其他正则化方法同时使用


> **Note:** **注意：**
> 早停法使得超参数的选择依赖于验证集，因此最好使用单独的测试集进行最终评估。


## 6. 数据增强 (Data Augmentation)


通过对训练数据进行变换来增加数据量和多样性，是防止过拟合的有力手段。


### 6.1 图像数据增强


- 随机翻转（水平/垂直）
- 随机旋转（±15°）
- 随机裁剪和缩放
- 颜色抖动（亮度、对比度、饱和度）
- 随机擦除（Random Erasing）
- Mixup / CutMix：混合两张图片


### 6.2 文本数据增强


- 同义词替换
- 随机插入、删除、交换
- 回译（翻译到另一语言再翻译回来）
- EDA（Easy Data Augmentation）


### 6.3 表格数据增强


- SMOTE：对少数类过采样
- 添加高斯噪声
- 特征混合


## 7. 超参数调优


### 7.1 超参数 vs 模型参数


- **模型参数**
   ：训练过程中学习的（如权重 w）
- **超参数**
   ：训练前设定的（如学习率、正则化系数、树深度）


### 7.2 网格搜索 (Grid Search)


遍历所有超参数组合，选择交叉验证分数最高的。


- 优点：全面、简单
- 缺点：组合爆炸，计算量大（超参数多时不现实）


### 7.3 随机搜索 (Random Search)


在超参数空间中随机采样一定数量的组合。


- Bergstra & Bengio (2012) 证明随机搜索比网格搜索更高效
- 原因：部分超参数对性能影响更大，随机搜索能更充分地探索这些维度
- 建议：60 次随机搜索通常覆盖足够的搜索空间


### 7.4 贝叶斯优化 (Bayesian Optimization)


使用概率模型（如高斯过程、TPE）建模目标函数，智能选择下一个评估点。


- 利用历史评估结果指导搜索方向
- 比随机搜索更高效（更少评估找到更好结果）
- 工具：Optuna、Hyperopt、scikit-optimize


### 7.5 三种方法对比


| 方法 | 效率 | 全面性 | 实现难度 | 推荐场景 |
| --- | --- | --- | --- | --- |
| 网格搜索 | 低 | 全面 | 简单 | 1-2 个超参数 |
| 随机搜索 | 中 | 概率覆盖 | 简单 | 3-5 个超参数 |
| 贝叶斯优化 | 高 | 智能探索 | 中等 | 计算预算有限 |


### 7.6 常见超参数搜索范围


| 模型 | 关键超参数 | 典型范围 |
| --- | --- | --- |
| 随机森林 | n_estimators, max_depth | 100-1000, 3-20 |
| XGBoost | learning_rate, max_depth, subsample | 0.01-0.3, 3-10, 0.5-1.0 |
| SVM | C, gamma | 0.1-100, 0.001-1 |
| 神经网络 | lr, dropout, hidden_size | 1e-4~1e-2, 0.1-0.5, 64-512 |

### 8.2 早停法（PyTorch）


```
class EarlyStopping:
    def __init__(self, patience=10, delta=0):
        self.patience = patience
        self.delta = delta
        self.counter = 0
        self.best_score = None
        self.early_stop = False
def __call__(self, val_loss, model):
        score = -val_loss
        if self.best_score is None:
            self.best_score = score
        elif score < self.best_score + self.delta:
            self.counter += 1
if self.counter >= self.patience:
                self.early_stop = True
else:
            self.best_score = score
            self.counter = 0

early_stopping = EarlyStopping(patience=10)
```

## 9. 总结


1. **偏差-方差权衡**
   是机器学习的核心困境：简单模型高偏差，复杂模型高方差
2. **L1**
   正则化产生稀疏解（特征选择），
   **L2**
   正则化使权重平滑
3. **Dropout**
   是神经网络特有的正则化，防止共适应
4. **早停法**
   简单有效，是最常用的正则化方法之一
5. **数据增强**
   从数据层面增加多样性，是图像任务的关键
6. 超参数调优：优先用
   **随机搜索**
   或
   **贝叶斯优化**
   （Optuna）


> **Tip:** **正则化方法优先级：**
> 1) 先收集更多数据；2) 使用合适的模型复杂度；3) 加入 L1/L2/Dropout；4) 早停；5) 数据增强。多种方法可组合使用。
