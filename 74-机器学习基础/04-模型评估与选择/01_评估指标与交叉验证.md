## 1. 混淆矩阵


混淆矩阵是分类任务最基本的评估工具，直观展示模型的预测结果与真实标签的对应关系。


### 1.1 二分类混淆矩阵


|  | 预测正类 | 预测负类 |
| --- | --- | --- |
| **实际正类** | TP (真阳性) | FN (假阴性) |
| **实际负类** | FP (假阳性) | TN (真阴性) |


- **TP**
   ：实际为正，预测为正（正确）
- **FP**
   ：实际为负，预测为正（误报，Type I Error）
- **FN**
   ：实际为正，预测为负（漏报，Type II Error）
- **TN**
   ：实际为负，预测为负（正确）


## 2. 分类评估指标


### 2.1 准确率 (Accuracy)


$$
Accuracy = (TP + TN) / (TP + TN + FP + FN)
$$


- 最直观的指标，但在类别不平衡时具有误导性
- 例：99% 负类的二分类中，全预测负类也有 99% 准确率


### 2.2 精确率 (Precision)


$$
Precision = TP / (TP + FP)
$$


"预测为正的样本中，有多少是真正的正类？"关注误报（FP）的代价。


### 2.3 召回率 (Recall / Sensitivity / TPR)


$$
Recall = TP / (TP + FN)
$$


"所有正类样本中，有多少被正确找出来了？"关注漏报（FN）的代价。


### 2.4 F1 分数


$$
F1 = 2 × Precision × Recall / (Precision + Recall)
$$


Precision 和 Recall 的调和平均数。F1 对极端值更敏感，只有两者都高时 F1 才高。


### 2.5 F-beta 分数


$$
Fβ = (1 + β²) × Precision × Recall / (β² × Precision + Recall)
$$


- β = 1：标准 F1
- β = 2：更看重 Recall（如医疗诊断，漏诊代价大）
- β = 0.5：更看重 Precision（如垃圾邮件检测，误报代价大）


### 2.6 何时关注哪个指标


| 场景 | 优先指标 | 原因 |
| --- | --- | --- |
| 疾病筛查 | Recall（高） | 漏诊代价极高 |
| 垃圾邮件过滤 | Precision（高） | 误删正常邮件代价大 |
| 类别不平衡 | F1 / AUC | Accuracy 不可靠 |
| 均衡分类 | Accuracy | 简单直观 |


### 2.7 多分类指标


- **宏平均 (Macro)**
   ：对每个类别计算指标后取平均（等权重）
- **微平均 (Micro)**
   ：汇总所有类别的 TP/FP/FN 后计算（大类主导）
- **加权平均 (Weighted)**
   ：按各类样本数加权平均


## 3. AUC-ROC 曲线


ROC 曲线（Receiver Operating Characteristic）和 AUC 值是评估二分类模型在不同阈值下性能的重要工具。


### 3.1 ROC 曲线


- **横轴**
   ：FPR = FP / (FP + TN) = 1 - Specificity（假阳率）
- **纵轴**
   ：TPR = TP / (TP + FN) = Recall = Sensitivity（真阳率）
- 通过改变分类阈值，得到一系列 (FPR, TPR) 点，连成曲线


### 3.2 AUC 值


AUC（Area Under Curve）是 ROC 曲线下的面积，取值 [0, 1]：


| AUC | 含义 |
| --- | --- |
| 1.0 | 完美分类器 |
| 0.9 - 1.0 | 优秀 |
| 0.8 - 0.9 | 良好 |
| 0.7 - 0.8 | 一般 |
| 0.5 - 0.7 | 较差 |
| 0.5 | 随机猜测 |
| < 0.5 | 比随机还差（应反转预测） |


### 3.3 AUC 的概率解释


AUC 等于随机选取一个正样本和一个负样本时，模型给正样本打分更高的概率：


$$
AUC = P(score(x⁺) > score(x⁻))
$$


### 3.4 PR 曲线


Precision-Recall 曲线：横轴 Recall，纵轴 Precision。在类别严重不平衡时，PR 曲线比 ROC 曲线更有信息量。


## 4. 回归评估指标


### 4.1 均方误差 (MSE)


$$
MSE = (1/n) Σ(yᵢ - ŷᵢ)²
$$


对大误差惩罚重（平方效应）。单位是目标值的平方。


### 4.2 均方根误差 (RMSE)


$$
RMSE = √MSE
$$


与目标变量同单位，更直观。


### 4.3 平均绝对误差 (MAE)


$$
MAE = (1/n) Σ|yᵢ - ŷᵢ|
$$


对异常值不如 MSE 敏感。


### 4.4 决定系数 R²


$$
R² = 1 - SS_res / SS_tot = 1 - Σ(yᵢ-ŷᵢ)² / Σ(yᵢ-ȳ)²
$$


- R² = 1：完美拟合
- R² = 0：等价于预测均值
- R² < 0：比预测均值还差


### 4.5 调整 R²


$$
R²_adj = 1 - (1-R²)(n-1) / (n-p-1)
$$


考虑了特征数量 p 的影响，增加无用特征时调整 R² 会下降。


### 4.6 MAPE


$$
MAPE = (100/n) Σ|yᵢ - ŷᵢ| / |yᵢ|
$$


百分比误差，不依赖于量纲，但 y=0 时有问题。


## 5. 数据集划分


### 5.1 简单划分


- **训练集**
   （60-80%）：用于模型训练
- **验证集**
   （10-20%）：用于超参数调优
- **测试集**
   （10-20%）：最终评估，只用一次


### 5.2 划分注意事项


- 分类任务使用
   **分层采样**
   （stratify），保持各类比例
- 时间序列数据不能随机划分，必须按时间顺序
- 同一组数据（如同一患者的多次测量）应在同一集合中
- 确保测试集与训练集同分布


## 6. 交叉验证


交叉验证（Cross-Validation）通过多次划分和评估，更可靠地估计模型的泛化性能。


### 6.1 K 折交叉验证 (K-Fold CV)


1. 将数据集分为 K 个大小相等的子集（Folds）
2. 轮流将每个子集作为验证集，其余 K-1 个作为训练集
3. 训练 K 次，取 K 次评估指标的平均值


- 常用 K = 5 或 10
- K 越大，评估越可靠，但计算量越大
- 极端情况 K = n 即留一法（LOO-CV）


### 6.2 分层 K 折 (Stratified K-Fold)


保证每个 Fold 中各类别的比例与原始数据一致。在类别不平衡时特别重要，是分类任务的推荐方法。


### 6.3 其他交叉验证方法


| 方法 | 特点 | 适用场景 |
| --- | --- | --- |
| 重复 K 折 | 多次 K 折取平均 | 需要更稳定的评估 |
| 分组 K 折 | 同组数据在同一 Fold | 数据有分组结构 |
| 时间序列 CV | 训练集在验证集之前 | 时序数据 |
| 留一法 LOO | K = n | 小数据集 |


> **Tip:** **最佳实践：**
> 使用 Stratified K-Fold（分类）或 K-Fold（回归），K=5 或 10。将交叉验证用于模型选择和超参数调优，最终在独立测试集上评估。


## 7. 学习曲线


学习曲线绘制训练集大小（或训练轮数）与模型性能的关系，帮助诊断过拟合/欠拟合。


### 7.1 典型模式


- **高偏差（欠拟合）**
   ：训练和验证分数都很低且接近，增加数据量帮助不大
- **高方差（过拟合）**
   ：训练分数高，验证分数低，两者差距大，增加数据可能有帮助
- **理想情况**
   ：两条曲线收敛到较高水平且差距小


### 7.2 诊断与应对


| 症状 | 诊断 | 解决方案 |
| --- | --- | --- |
| 训练分数低 | 欠拟合 | 增加模型复杂度、增加特征 |
| 训练高、验证低，差距大 | 过拟合 | 正则化、更多数据、减少特征 |
| 两曲线差距逐渐缩小 | 正常，增加数据有帮助 | 继续收集数据 |


## 8. Python 代码实现


### 8.1 分类评估指标


```
from sklearn.metrics import (
    accuracy_score, precision_score, recall_score, f1_score,
    classification_report, confusion_matrix, roc_auc_score, roc_curve
)
import matplotlib.pyplot as plt

# 基本指标
print(f"Accuracy: {accuracy_score(y_test, y_pred):.4f}")
print(f"Precision: {precision_score(y_test, y_pred):.4f}")
print(f"Recall: {recall_score(y_test, y_pred):.4f}")
print(f"F1: {f1_score(y_test, y_pred):.4f}")

# 完整报告
print(classification_report(y_test, y_pred, target_names=['负类', '正类']))

# AUC-ROC
y_prob = model.predict_proba(X_test)[:, 1]
auc = roc_auc_score(y_test, y_prob)
fpr, tpr, thresholds = roc_curve(y_test, y_prob)
plt.plot(fpr, tpr, label=f'AUC = {auc:.3f}')
plt.plot([0,1], [0,1], 'k--')
plt.xlabel('FPR'); plt.ylabel('TPR')
plt.title('ROC Curve'); plt.legend()
```


### 8.2 交叉验证


```
from sklearn.model_selection import (
    cross_val_score, StratifiedKFold, GridSearchCV,
    learning_curve, train_test_split
)

# K 折交叉验证
cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
scores = cross_val_score(model, X, y, cv=cv, scoring='f1')
print(f"F1: {scores.mean():.4f} ± {scores.std():.4f}")

# 网格搜索 + 交叉验证
param_grid = {'C': [0.1, 1, 10], 'gamma': ['scale', 'auto']}
grid = GridSearchCV(model, param_grid, cv=5, scoring='f1', n_jobs=-1)
grid.fit(X_train, y_train)
print(f"最优参数: {grid.best_params_}, 最优得分: {grid.best_score_:.4f}")

# 学习曲线
train_sizes, train_scores, val_scores = learning_curve(
    model, X, y, cv=5, train_sizes=np.linspace(0.1, 1.0, 10),
    scoring='accuracy'
)
```


### 8.3 回归评估


```
from sklearn.metrics import mean_squared_error, mean_absolute_error, r2_score

mse = mean_squared_error(y_test, y_pred)
rmse = np.sqrt(mse)
mae = mean_absolute_error(y_test, y_pred)
r2 = r2_score(y_test, y_pred)
print(f"RMSE: {rmse:.4f}, MAE: {mae:.4f}, R²: {r2:.4f}")
```


## 9. 总结


1. 混淆矩阵是分类评估的基础，从中可推导出 Precision、Recall、F1 等指标
2. **类别不平衡**
   时不要用 Accuracy，改用 F1 或 AUC
3. **AUC-ROC**
   不受阈值和类别比例影响，是比较模型的好指标
4. 回归任务用 RMSE、MAE、R² 评估
5. **交叉验证**
   提供更可靠的泛化性能估计，推荐 Stratified 5-Fold
6. **学习曲线**
   帮助诊断过拟合/欠拟合
