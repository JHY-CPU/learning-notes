# 插件与模块开发

## 核心概念

Unreal的功能扩展通过模块(Module)和插件(Plugin)实现。模块是编译单元，插件是功能集合（可包含多个模块）。掌握模块和插件开发是构建可复用功能的基础。

## 模块结构

```
YourModule/
├── Public/                  # 头文件（对外暴露）
│   ├── YourModule.h         # 模块接口
│   ├── YourModuleAPI.h      # 导出宏
│   └── *.h                  # 公共头文件
├── Private/                 # 实现文件（内部使用）
│   ├── YourModule.cpp       # 模块实现
│   └── *.cpp               # 私有实现
├── YourModule.Build.cs      # 构建规则
└── YourModule.uplugin       # 插件描述（如果是插件模块）
```

## Build.cs构建规则

```csharp
// YourModule.Build.cs
using UnrealBuildTool;

public class YourModule : ModuleRules
{
    public YourModule(ReadOnlyTargetRules Target) : base(Target)
    {
        PCHUsage = PCHUsageMode.UseExplicitOrSharedPCHs;

        // 公共依赖（使用此模块的项目也会继承这些依赖）
        PublicDependencyModuleNames.AddRange(new string[]
        {
            "Core",
            "CoreUObject",
            "Engine",
        });

        // 私有依赖（仅此模块使用）
        PrivateDependencyModuleNames.AddRange(new string[]
        {
            "Slate",
            "SlateCore",
            "InputCore",
            "UMG",
        });

        // Editor-only依赖
        if (Target.bBuildEditor)
        {
            PrivateDependencyModuleNames.Add("UnrealEd");
        }

        // 第三方库
        PublicAdditionalLibraries.Add(Path.Combine(ModuleDirectory, "ThirdParty", "lib", "mylib.lib"));
        PublicIncludePaths.Add(Path.Combine(ModuleDirectory, "ThirdParty", "include"));

        // 定义宏
        PublicDefinitions.Add("MY_FEATURE_ENABLED=1");
    }
}
```

## 模块实现

```cpp
// YourModule.h - 模块接口
#pragma once

#include "Modules/ModuleManager.h"

class FYourModule : public IModuleInterface
{
public:
    virtual void StartupModule() override;
    virtual void ShutdownModule() override;

    // 模块单例访问
    static FYourModule& Get()
    {
        return FModuleManager::LoadModuleChecked<FYourModule>("YourModule");
    }

    static bool IsAvailable()
    {
        return FModuleManager::Get().IsModuleLoaded("YourModule");
    }

    // 自定义功能
    void RegisterCustomFeature();
    void UnregisterCustomFeature();

private:
    // 模块内部状态
    TSharedPtr<FSlateStyleSet> StyleSet;
    TArray<TSharedRef<IAssetTypeActions>> RegisteredAssetActions;
};
```

```cpp
// YourModule.cpp - 模块实现
#include "YourModule.h"
#include "ISettingsModule.h"
#include "AssetToolsModule.h"

#define LOCTEXT_NAMESPACE "FYourModule"

void FYourModule::StartupModule()
{
    UE_LOG(LogTemp, Log, TEXT("YourModule started"));

    // 注册项目设置
    if (ISettingsModule* SettingsModule = FModuleManager::GetModulePtr<ISettingsModule>("Settings"))
    {
        SettingsModule->RegisterSettings(
            "Project", "Game", "YourModule Settings",
            LOCTEXT("SettingsName", "Your Module"),
            LOCTEXT("SettingsDesc", "Configure your module settings"),
            GetMutableDefault<UYourModuleSettings>()
        );
    }

    // 注册Slate样式
    StyleSet = MakeShareable(new FSlateStyleSet("YourModuleStyle"));
    StyleSet->SetContentRoot(FPaths::ProjectContentDir());

    // 注册资产类型操作
    if (FModuleManager::Get().IsModuleLoaded("AssetTools"))
    {
        IAssetTools& AssetTools = FModuleManager::LoadModuleChecked<FAssetToolsModule>("AssetTools").Get();
        TSharedRef<IAssetTypeActions> Actions = MakeShareable(new FAssetTypeActions_YourAsset());
        AssetTools.RegisterAssetTypeActions(Actions);
        RegisteredAssetActions.Add(Actions);
    }

    // 注册控制台命令
    IConsoleManager::Get().RegisterConsoleCommand(
        TEXT("my.YourCommand"),
        TEXT("Description of your command"),
        FConsoleCommandWithArgsDelegate::CreateLambda([](const TArray<FString>& Args)
        {
            UE_LOG(LogTemp, Log, TEXT("Command executed with %d args"), Args.Num());
        })
    );
}

void FYourModule::ShutdownModule()
{
    // 注销设置
    if (ISettingsModule* SettingsModule = FModuleManager::GetModulePtr<ISettingsModule>("Settings"))
    {
        SettingsModule->UnregisterSettings("Project", "Game", "YourModule Settings");
    }

    // 注销资产类型操作
    if (FModuleManager::Get().IsModuleLoaded("AssetTools"))
    {
        IAssetTools& AssetTools = FModuleManager::LoadModuleChecked<FAssetToolsModule>("AssetTools").Get();
        for (auto& Action : RegisteredAssetActions)
        {
            AssetTools.UnregisterAssetTypeActions(Action);
        }
    }

    // 清理样式
    if (StyleSet.IsValid())
    {
        FSlateStyleRegistry::UnRegisterSlateStyle(*StyleSet);
    }
}

#undef LOCTEXT_NAMESPACE

// 注册模块
IMPLEMENT_MODULE(FYourModule, YourModule);
```

## Slate UI开发

Slate是Unreal的底层UI框架，UMG基于Slate构建：

```cpp
// 自定义Slate控件
class SMyCustomWidget : public SCompoundWidget
{
    SLATE_BEGIN_ARGS(SMyCustomWidget)
        : _DefaultText(FText::FromString("Default"))
    {}
        SLATE_ARGUMENT(FText, DefaultText)
        SLATE_EVENT(FOnClicked, OnButtonClicked)
    SLATE_END_ARGS()

    void Construct(const FArguments& InArgs)
    {
        DefaultText = InArgs._DefaultText;

        ChildSlot
        [
            SNew(SVerticalBox)
            + SVerticalBox::Slot()
            .AutoHeight()
            .Padding(5)
            [
                SNew(STextBlock)
                .Text(DefaultText)
                .Font(FCoreStyle::GetDefaultFontStyle("Bold", 16))
            ]
            + SVerticalBox::Slot()
            .AutoHeight()
            .Padding(5)
            [
                SNew(SButton)
                .Text(FText::FromString("Click Me"))
                .OnClicked(InArgs._OnButtonClicked)
            ]
        ];
    }

private:
    FText DefaultText;
};

// 使用自定义Slate控件
SNew(SMyCustomWidget)
.DefaultText(FText::FromString("Hello Slate"))
.OnClicked_Raw(this, &FYourModule::OnMyButtonClicked)
```

## 加载阶段 (Loading Phase)

| 阶段 | 时机 | 典型用途 |
|------|------|----------|
| EarliestPossible | 最早加载 | 核心基础模块 |
| PostConfigInit | 配置初始化后 | 设置系统 |
| PostSplashScreen | 启动画面后 | 早期UI |
| PreEarlyLoadingScreen | 加载画面前 | 加载器模块 |
| PreLoadingScreen | 加载画面 | 进度条 |
| PreDefault | 默认前 | 一般模块 |
| Default | 默认阶段 | 大多数模块 |
| PostDefault | 默认后 | 扩展模块 |
| PostEngineInit | 引擎初始化后 | 编辑器工具 |

## 常见陷阱与最佳实践

1. **循环依赖**: 模块间不能有循环引用，需合理设计依赖层级
2. **DLL导出宏**: 跨模块使用的类需要MODULE_API宏导出
3. **加载阶段选择**: 依赖的模块必须在之前阶段加载
4. **热重载**: 编辑器中修改模块代码支持热重载，但修改Build.cs需重启
5. **插件分发**: 确保插件不依赖项目私有代码，保持独立性

## 与其他系统的关联

- **GamePlay框架**: 插件可提供自定义GameMode、Component等
- **蓝图系统**: 插件可暴露自定义蓝图节点
- **编辑器扩展**: Editor模块可添加菜单、工具栏、自定义窗口
