# 网络同步(UE)

## 核心概念

Unreal采用权威服务器模型(Server-Authoritative)，服务器是游戏状态的唯一权威。客户端通过属性复制(Replication)和远程过程调用(RPC)与服务器同步。理解网络模型是多人游戏开发的基础。

## 网络模型

```
客户端1 ←→ 服务器(权威) ←→ 客户端2
            ↑
         客户端3

- 服务器拥有所有游戏状态的最终决定权
- 客户端发送意图（输入、请求），服务器验证并执行
- 服务器将结果复制到所有相关客户端
```

## 属性复制 (Replication)

```cpp
UCLASS()
class AMyReplicatedActor : public AActor
{
    GENERATED_BODY()

public:
    // 声明需要复制的属性
    UPROPERTY(Replicated)
    float Health;

    UPROPERTY(Replicated)
    int32 TeamID;

    // RepNotify: 属性变化时触发回调
    UPROPERTY(ReplicatedUsing = OnRep_CurrentState)
    EAIState CurrentState;

    // 属性变化回调（客户端调用）
    UFUNCTION()
    void OnRep_CurrentState()
    {
        // 状态变化时更新动画、特效等
        UpdateVisuals();
    }

    // 注册复制属性（必须实现）
    virtual void GetLifetimeReplicatedProps(TArray<FLifetimeProperty>& OutLifetimeProps) const override
    {
        Super::GetLifetimeReplicatedProps(OutLifetimeProps);

        DOREPLIFETIME(AMyReplicatedActor, Health);
        DOREPLIFETIME(AMyReplicatedActor, TeamID);
        DOREPLIFETIME(AMyReplicatedActor, CurrentState);

        // 条件复制：只在特定条件下复制
        DOREPLIFETIME_CONDITION(AMyReplicatedActor, SomeValue, COND_OwnerOnly);
    }
};
```

### 复制条件

| 条件 | 说明 |
|------|------|
| COND_None | 始终复制到所有客户端 |
| COND_OwnerOnly | 只复制到Owner客户端 |
| COND_SkipOwner | 复制到除Owner外的所有客户端 |
| COND_InitialOnly | 只在首次同步时复制 |
| COND_SimulatedOnly | 只复制到模拟代理 |

## RPC（远程过程调用）

```cpp
UCLASS()
class AMyCharacter : public ACharacter
{
    GENERATED_BODY()

public:
    // === Server RPC: 客户端 → 服务器 ===
    // Reliable: 保证送达（类似TCP）
    // Unreliable: 不保证送达（类似UDP，用于高频更新）
    // WithValidation: 带验证函数

    UFUNCTION(Server, Reliable, WithValidation)
    void ServerFire(FVector Direction);
    bool ServerFire_Validate(FVector Direction) { return Direction.Size() > 0; } // 验证
    void ServerFire_Implementation(FVector Direction) // 实现
    {
        // 服务器端执行开火逻辑
        SpawnProjectile(Direction);
    }

    // 客户端调用方式
    void FirePressed()
    {
        // 客户端预测
        PlayFireEffect();

        // 发送到服务器
        ServerFire(GetAimDirection());
    }

    // === Client RPC: 服务器 → 特定客户端 ===
    UFUNCTION(Client, Reliable)
    void ClientShowDamage(float Damage, FVector HitLocation);
    void ClientShowDamage_Implementation(float Damage, FVector HitLocation)
    {
        // 在拥有此Actor的客户端上显示伤害数字
        SpawnDamageNumber(Damage, HitLocation);
    }

    // === Multicast RPC: 服务器 → 所有客户端 ===
    UFUNCTION(NetMulticast, Unreliable)
    void MulticastPlayEffect(UParticleSystem* Effect, FVector Location);
    void MulticastPlayEffect_Implementation(UParticleSystem* Effect, FVector Location)
    {
        // 在所有客户端播放效果
        UGameplayStatics::SpawnEmitterAtLocation(GetWorld(), Effect, Location);
    }

    // === 调用示例 ===
    void TakeDamage(float Damage)
    {
        if (HasAuthority()) // 服务器端
        {
            Health -= Damage;

            // 通知拥有者客户端
            ClientShowDamage(Damage, GetActorLocation());

            // 广播到所有客户端
            MulticastPlayEffect(DamageEffect, GetActorLocation());
        }
    }
};
```

## Ownership与Relevancy

```cpp
// Ownership: 决定RPC的目标和属性复制范围
void SetOwnership()
{
    // 设置Owner（通常是PlayerController）
    Projectile->SetOwner(this);

    // 检查是否为Owner
    if (IsLocallyControlled())
    {
        // 本地控制的客户端（非模拟代理）
    }

    if (GetLocalRole() == ROLE_Authority)
    {
        // 服务器端
    }
}

// Relevancy: 控制Actor对客户端是否相关
// 不相关的Actor不会同步，节省带宽
virtual bool IsNetRelevantFor(const AActor* RealViewer, const AActor* ViewTarget, const FVector& SrcLocation) const override
{
    float Distance = FVector::Dist(GetActorLocation(), ViewTarget->GetActorLocation());
    return Distance < NetCullDistance; // 距离裁剪
}
```

// 移动预测由CharacterMovementComponent自动处理

void AMyCharacter::MoveForward(float Value)
{
    if (Controller && Value != 0.f)
    {
        // 客户端立即执行（预测）
        AddMovementInput(GetActorForwardVector(), Value);

        // 服务器通过内置的移动复制自动处理
        // ServerMove → 客户端预测 → ServerMove响应 → 客户端纠正
    }
}

## 网络角色检查

if (GetLocalRole() > ROLE_SimulatedProxy)
{
    // 本地控制的客户端或服务器
}

switch (GetLocalRole())
{
case ROLE_Authority:
    // 服务器端逻辑
    break;
case ROLE_AutonomousProxy:
    // 客户端的本地控制Pawn
    break;
case ROLE_SimulatedProxy:
    // 其他客户端的Pawn（预测/插值）
    break;
}

// 检查是否为服务器
if (HasAuthority())
{
    // 服务器逻辑
}

## 常见陷阱与最佳实践

1. **服务器验证**: 所有关键逻辑必须在服务器执行，客户端只做表现
2. **RPC频率**: Unreliable RPC适合高频数据（位置、旋转），Reliable用于关键事件
3. **RepNotify vs Tick**: 用RepNotify响应状态变化，不要在Tick中轮询
4. **带宽优化**: 合理使用条件复制和距离裁减，减少同步数据量
5. **客户端预测延迟**: 感知到的延迟来自网络往返，使用客户端预测改善体验

## 与其他系统的关联

- **GamePlay框架**: PlayerController是RPC的主要发起者
- **物理系统**: 物理模拟在服务器端运行，客户端通过复制显示
- **动画系统**: 动画状态通过属性复制同步
