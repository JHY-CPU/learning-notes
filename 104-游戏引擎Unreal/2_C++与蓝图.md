# C++与蓝图

## 核心概念

Unreal采用C++和Blueprint双语言开发模式。C++负责底层逻辑和性能关键代码，Blueprint负责美术/设计师可编辑的逻辑和快速原型。两者通过反射系统无缝互操作。理解何时使用C++、何时使用Blueprint，以及如何高效地在两者之间通信，是Unreal开发的核心技能。

## Blueprint可视化脚本详解

Blueprint通过节点连线实现逻辑，底层是基于字节码的虚拟机解释执行：

```
Blueprint执行原理:
Blueprint源文件 (.uasset)
    ↓ [编译]
字节码 (Bytecode)
    ↓ [运行时]
Blueprint VM (虚拟机解释执行)
    ↓
调用C++实现的节点函数

性能对比:
纯C++函数调用: ~1-10ns
Blueprint函数调用: ~100-1000ns (解释执行)
Blueprint Nativized: ~10-50ns (打包时编译为C++)
```

### 核心节点类型

| 节点类型 | 功能 | C++对应 | 使用场景 |
|----------|------|---------|---------|
| Event BeginPlay | 游戏开始事件 | BeginPlay() | 初始化逻辑 |
| Event Tick | 每帧事件 | Tick() | 持续更新 |
| Custom Event | 自定义事件 | UFUNCTION(BlueprintCallable) | 自定义回调 |
| Function Call | 调用函数 | 普通C++函数 | 调用已有功能 |
| Variable | 变量读写 | UPROPERTY() | 数据存储 |
| Branch | 条件分支 | if/else | 逻辑判断 |
| ForLoop | 循环 | for循环 | 批量处理 |
| Sequence | 顺序执行 | 顺序语句 | 分步操作 |
| Delay | 延时执行 | 协程/Timer | 延时逻辑 |
| Timeline | 时间线 | 动画曲线 | 平滑过渡 |
| Event Dispatcher | 事件分发器 | 多播委托 | 事件广播 |
| Cast To | 类型转换 | Cast<T>() | 类型判断 |
| Get All Actors | 获取所有Actor | TActorIterator | 批量查找 |

## C++暴露到蓝图完整指南

```cpp
// MyCharacter.h - 完整的C++到Blueprint暴露示例
UCLASS(Blueprintable, BlueprintType, ClassGroup=(Combat))
class YOURGAME_API AMyCharacter : public ACharacter
{
    GENERATED_BODY()

public:
    AMyCharacter();

    // ================================================================
    // UPROPERTY - 属性暴露
    // ================================================================

    // 编辑器和蓝图可读写
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category="Combat",
        meta=(ToolTip="角色攻击力"))
    float AttackDamage = 20.f;

    // 仅蓝图可读，编辑器中可编辑默认值
    UPROPERTY(EditDefaultsOnly, BlueprintReadOnly, Category="State")
    bool bIsAlive = true;

    // 运行时可编辑（实例上修改）
    UPROPERTY(EditInstanceOnly, BlueprintReadWrite, Category="Config")
    int32 Level = 1;

    // 分组显示（嵌套类别用|分隔）
    UPROPERTY(EditDefaultsOnly, Category="Config|Movement")
    float WalkSpeed = 300.f;

    // 滑条范围（编辑器中显示为滑条）
    UPROPERTY(EditAnywhere, meta=(ClampMin="0.0", ClampMax="100.0"),
        Category="Combat")
    float HealthPercent = 100.f;

    // 固定数组
    UPROPERTY(EditAnywhere, Category="Config")
    TArray<FVector> PatrolPoints;

    // 场景中可见的组件引用
    UPROPERTY(VisibleAnywhere, BlueprintReadOnly, Category="Components",
        meta=(AllowPrivateAccess="true"))
    class USpringArmComponent* SpringArm;

    UPROPERTY(VisibleAnywhere, BlueprintReadOnly, Category="Components")
    class UCameraComponent* Camera;

    // ================================================================
    // UFUNCTION - 函数暴露
    // ================================================================

    // 蓝图可调用，显示为执行节点
    UFUNCTION(BlueprintCallable, Category="Combat")
    void Attack(AActor* Target);

    // 蓝图可调用，有返回值
    UFUNCTION(BlueprintCallable, Category="Combat")
    float CalculateDamage(float BaseDamage, float CriticalMultiplier) const;

    // 纯函数（无执行引脚），显示为数据节点
    UFUNCTION(BlueprintPure, Category="State")
    float GetHealthPercent() const;

    // 纯函数：获取当前位置
    UFUNCTION(BlueprintPure, Category="State")
    FVector GetAimLocation() const;

    // 蓝图可重写事件（蓝图中实现，C++中不能有函数体）
    UFUNCTION(BlueprintImplementableEvent, Category="Events")
    void OnDamageReceived(float Damage, AActor* DamageCauser);

    // C++默认实现，蓝图可覆盖（最灵活的扩展点）
    UFUNCTION(BlueprintNativeEvent, Category="Events")
    void Die();
    virtual void Die_Implementation();

    // 蓝图可设置的事件分发器
    DECLARE_DYNAMIC_MULTICAST_DELEGATE_TwoParams(FOnHealthChanged,
        float, OldHealth, float, NewHealth);
    UPROPERTY(BlueprintAssignable, Category="Events")
    FOnHealthChanged OnHealthChanged;

    // 可绑定的委托
    DECLARE_DYNAMIC_DELEGATE_OneParam(FOnDeath, AActor*, DeadActor);
    UPROPERTY(BlueprintReadWrite, Category="Events")
    FOnDeath OnDeathDelegate;

    // ================================================================
    // 蓝图暴露枚举和结构体
    // ================================================================

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category="State")
    ECharacterState CurrentState;

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category="Stats")
    FCharacterStats Stats;
};

// MyCharacter.cpp
AMyCharacter::AMyCharacter()
{
    // 在构造函数中创建组件
    SpringArm = CreateDefaultSubobject<USpringArmComponent>(TEXT("SpringArm"));
    SpringArm->SetupAttachment(RootComponent);
    SpringArm->TargetArmLength = 300.f;

    Camera = CreateDefaultSubobject<UCameraComponent>(TEXT("Camera"));
    Camera->SetupAttachment(SpringArm, USpringArmComponent::SocketName);

    // 设置默认值
    PrimaryActorTick.bCanEverTick = true;
}

void AMyCharacter::Attack(AActor* Target)
{
    if (!Target || !bIsAlive) return;

    // C++逻辑
    IDamageable* Damageable = Cast<IDamageable>(Target);
    if (Damageable)
    {
        float FinalDamage = CalculateDamage(AttackDamage, 1.5f);
        Damageable->Execute_ApplyDamage(Target, FinalDamage, this);
    }

    // 触发蓝图可实现事件（蓝图中重写时调用蓝图版本）
    OnDamageReceived(AttackDamage, this);

    // 广播委托
    float OldHealth = HealthPercent;
    HealthPercent -= 5f;
    OnHealthChanged.Broadcast(OldHealth, HealthPercent);
}

float AMyCharacter::CalculateDamage(float BaseDamage, float CriticalMultiplier) const
{
    // 随机暴击
    if (FMath::RandRange(0.f, 1.f) < 0.2f)
        return BaseDamage * CriticalMultiplier;
    return BaseDamage;
}

float AMyCharacter::GetHealthPercent() const
{
    return HealthPercent;
}

FVector AMyCharacter::GetAimLocation() const
{
    // 从相机射线获取瞄准位置
    APlayerController* PC = Cast<APlayerController>(GetController());
    if (PC)
    {
        FVector WorldLoc, WorldDir;
        PC->DeprojectMousePositionToWorld(WorldLoc, WorldDir);

        FHitResult Hit;
        if (GetWorld()->LineTraceSingleByChannel(Hit, WorldLoc,
            WorldLoc + WorldDir * 10000.f, ECC_Visibility))
        {
            return Hit.ImpactPoint;
        }
    }
    return GetActorLocation() + GetActorForwardVector() * 1000.f;
}

void AMyCharacter::Die_Implementation()
{
    bIsAlive = false;
    CurrentState = ECharacterState::Dead;

    // C++默认死亡逻辑
    GetMesh()->SetSimulatePhysics(true);
    GetMesh()->SetCollisionProfileName("Ragdoll");

    // 广播死亡事件
    OnDeathDelegate.ExecuteIfBound(this);
}
```

## 蓝图调用C++的通信模式

```cpp
// 模式1: BlueprintCallable - 蓝图直接调用C++函数
UFUNCTION(BlueprintCallable, Category="Gameplay")
static void SpawnEffectAtLocation(UWorld* World, UParticleSystem* Effect,
    FVector Location);

// 模式2: BlueprintImplementableEvent - 蓝图实现
UFUNCTION(BlueprintImplementableEvent, Category="Events")
void OnPlayerEnteredZone(FVector ZoneCenter, float ZoneRadius);
// 注意: C++中不提供函数体，蓝图中实现

// 模式3: BlueprintNativeEvent - C++默认+蓝图可覆盖
UFUNCTION(BlueprintNativeEvent, Category="Combat")
float GetAttackRange();
virtual float GetAttackRange_Implementation() { return 500.f; }

// 模式4: 委托/事件分发器 - 蓝图绑定回调
DECLARE_DYNAMIC_MULTICAST_DELEGATE(FOnGameStarted);
UPROPERTY(BlueprintAssignable)
FOnGameStarted OnGameStarted;

// 模式5: 接口 - 多态调用
UFUNCTION(BlueprintNativeEvent, BlueprintCallable)
void Interact(AActor* Interactor);

// 蓝图中使用接口:
// [Interface Message] "Interact" → 调用实现了IInteractable的Actor
```

## 委托系统详解

Unreal的委托(Delegate)系统是C++和Blueprint通信的核心机制：

```cpp
// ===== 单播委托 =====
DECLARE_DELEGATE(FOnSimpleDelegate);
DECLARE_DELEGATE_OneParam(FOnValueDelegate, float);
DECLARE_DELEGATE_RetVal(bool, FOnCheckDelegate);

FOnSimpleDelegate OnReady;

// 绑定
OnReady.BindUObject(this, &AMyActor::OnReadyCallback);
OnReady.BindLambda([]() { UE_LOG(LogTemp, Log, TEXT("Ready")); });

// 执行
OnReady.ExecuteIfBound(); // 安全执行（未绑定时不崩溃）

// ===== 动态委托（可用于Blueprint） =====
DECLARE_DYNAMIC_DELEGATE_OneParam(FOnDynamicValue, float, Value);
UPROPERTY(BlueprintReadWrite)
FOnDynamicValue OnDynamicCallback;

// 绑定
OnDynamicCallback.BindUFunction(TargetActor, FName("OnValueChanged"));

// ===== 多播委托 =====
DECLARE_DYNAMIC_MULTICAST_DELEGATE_TwoParams(FOnHealthUpdate,
    float, OldHealth, float, NewHealth);
UPROPERTY(BlueprintAssignable)
FOnHealthUpdate OnHealthUpdate;

// 绑定（可在蓝图中绑定多个回调）
OnHealthUpdate.AddDynamic(this, &AMyActor::OnHealthChangedHandler);

// 广播
OnHealthUpdate.Broadcast(OldHealth, NewHealth);

// ===== 完整使用示例 =====
class YOURGAME_API AGameMode : public AGameModeBase
{
    GENERATED_BODY()

public:
    // C++和蓝图都能绑定的多播委托
    DECLARE_DYNAMIC_MULTICAST_DELEGATE_OneParam(FOnPlayerDied,
        AActor*, DeadPlayer);
    UPROPERTY(BlueprintAssignable, Category="Game Events")
    FOnPlayerDied OnPlayerDied;

    DECLARE_DYNAMIC_MULTICAST_DELEGATE(FOnGameOver);
    UPROPERTY(BlueprintAssignable, Category="Game Events")
    FOnGameOver OnGameOver;

    void NotifyPlayerDied(AActor* DeadPlayer)
    {
        OnPlayerDied.Broadcast(DeadPlayer);

        alivePlayers--;
        if (alivePlayers <= 0)
            OnGameOver.Broadcast();
    }

private:
    int32 alivePlayers = 0;
};
```

## Blueprint Nativization

UE4.24之前支持Blueprint Nativization（将Blueprint编译为C++），UE5中此功能已被移除，替代方案是：

```
UE4 Nativization (已废弃):
Blueprint → Bytecode → 打包时转为C++代码 → 编译为原生机器码
性能提升: 5-10倍

UE5替代方案:
1. 关键逻辑用C++编写
2. 使用Blueprint的"Inline"编译选项（部分优化）
3. 将热点Blueprint函数迁移到C++
4. 使用Verse脚本语言（UE5.1+实验性功能）
```

```cpp
// 热点函数迁移示例

// Before (Blueprint): 每帧执行的移动逻辑
// → 性能开销: ~1000ns/frame

// After (C++): 同样的逻辑
void AMyCharacter::Tick(float DeltaTime)
{
    Super::Tick(DeltaTime);

    FVector Velocity = GetVelocity();
    float Speed = Velocity.Size();

    // 更新动画参数
    GetCharacterMovement()->MaxWalkSpeed = Speed > 10.f ? RunSpeed : WalkSpeed;

    // 旋转朝向移动方向
    if (Speed > 10.f)
    {
        FRotator TargetRot = Velocity.Rotation();
        FRotator NewRot = FMath::RInterpTo(GetActorRotation(),
            TargetRot, DeltaTime, RotationSpeed);
        SetActorRotation(NewRot);
    }
}
// → 性能开销: ~10ns/frame
```

## 何时使用C++ vs Blueprint

| 场景 | 推荐 | 原因 |
|------|------|------|
| 核心游戏循环 | C++ | 性能关键路径 |
| AI行为树Task | C++ | 高频调用 |
| UI流程 | Blueprint | 迭代快，设计师可参与 |
| 数值计算公式 | C++ | 每帧执行 |
| 网络同步 | C++ | 需要精确控制 |
| 动画事件 | Blueprint | 与美术紧密耦合 |
| 关卡逻辑 | Blueprint | 非频繁执行，需可视化编辑 |
| 配置数据 | DataAsset/DataTable | 数据驱动设计 |
| 特效触发 | Blueprint | 迭代频繁 |
| 战斗系统 | C++为主+Blueprint扩展 | 性能+灵活性 |
| 任务系统 | Blueprint为主 | 逻辑复杂且需频繁调整 |
| 对话系统 | Blueprint/DataTable | 设计师主导 |

## 线程安全与Blueprint

```cpp
// Blueprint只能在游戏线程执行
// 异步操作需要通过以下方式回到游戏线程

// AsyncTask回到游戏线程
AsyncTask(ENamedThreads::GameThread, [this]()
{
    // 在游戏线程执行
    // 可以安全地调用Blueprint事件
    OnAsyncTaskCompleted.Broadcast(Result);
});

// 使用FFunctionGraphTask创建异步任务
FFunctionGraphTask::CreateAndDispatchWhenReady(
    [this]()
    {
        // 在后台线程执行耗时计算
        float Result = ExpensiveCalculation();

        // 回到游戏线程更新UI
        AsyncTask(ENamedThreads::GameThread, [this, Result]()
        {
            UpdateUI(Result);
        });
    },
    TStatId(), nullptr, ENamedThreads::AnyBackgroundThreadNormalTask
);
```

## 实际游戏案例

### 案例: C++基础类 + Blueprint变体

```
C++层:
AMyCharacter (基类)
├── AHeroCharacter (英雄角色基类)
│   ├── AWarriorCharacter (战士C++特化)
│   └── AMageCharacter (法师C++特化)
└── AEnemyCharacter (敌人基类)

Blueprint层:
BP_Warrior (蓝图继承AWarriorCharacter)
├── 修改外观 (Mesh, Material)
├── 修改属性 (Damage, Health)
├── 添加蓝图逻辑 (特效触发、音效)
└── 动画事件处理

BP_Mage (蓝图继承AMageCharacter)
├── 不同的外观和属性
├── 特有的法术特效触发
└── UI交互逻辑
```

```cpp
// C++基类提供核心功能
UCLASS(Blueprintable)
class AHeroCharacter : public ACharacter
{
    GENERATED_BODY()

public:
    // C++实现核心战斗逻辑
    UFUNCTION(BlueprintCallable, Category="Combat")
    void PerformAttack();

    // Blueprint可实现视觉反馈
    UFUNCTION(BlueprintImplementableEvent, Category="Combat")
    void PlayAttackEffect(FVector ImpactPoint);

    // Blueprint可配置的参数
    UPROPERTY(EditDefaultsOnly, BlueprintReadOnly, Category="Combat")
    float AttackRange = 200.f;
};

void AHeroCharacter::PerformAttack()
{
    // C++: 性能关键的射线检测
    FHitResult Hit;
    FVector Start = GetActorLocation() + FVector(0,0,50);
    FVector End = Start + GetActorForwardVector() * AttackRange;

    if (GetWorld()->LineTraceSingleByChannel(Hit, Start, End, ECC_Pawn))
    {
        // 造成伤害
        if (IDamageable* Target = Cast<IDamageable>(Hit.GetActor()))
        {
            Target->Execute_ApplyDamage(Hit.GetActor(), AttackDamage, this);
        }

        // 调用Blueprint实现的特效
        PlayAttackEffect(Hit.ImpactPoint);
    }
}
```

## 常见陷阱与最佳实践

1. **GENERATED_BODY()位置**: 必须放在类体第一行（public:之后），决定UHT代码生成位置
2. **UPROPERTY必要性**: 只有UPROPERTY标记的变量才能在蓝图中访问和编辑，也能被序列化
3. **BlueprintImplementableEvent不能有C++实现**: 如果需要C++默认实现+蓝图覆盖，使用BlueprintNativeEvent
4. **头文件修改触发重编译**: 尽量将复杂逻辑放cpp中，头文件保持简洁。使用前向声明减少依赖
5. **蓝图可读性**: 蓝图节点过多时应拆分为多个函数或子蓝图，保持单个蓝图不超过50个节点
6. **委托生命周期**: 确保委托绑定的对象在委托触发时仍然有效，使用TWeakObjectPtr防止野指针
7. **蓝图变量访问**: UPROPERTY的访问修饰符（EditAnywhere等）决定蓝图中的读写权限
8. **避免蓝图中做密集计算**: 每帧执行的复杂逻辑应迁移到C++

## 性能分析

| 操作 | 开销 | 说明 |
|------|------|------|
| C++函数调用 | ~1-10ns | 直接调用 |
| Blueprint函数调用 | ~100-1000ns | 解释执行 |
| Blueprint Nativized | ~10-50ns | UE4已废弃 |
| 委托广播 | ~50-200ns | 取决于绑定数量 |
| Cast<T> | ~10-50ns | 运行时类型检查 |
| UFUNCTION调用 | ~1-10ns | 编译后为普通函数 |

## 与其他系统的关联

- **反射系统**: C++暴露依赖UHT(Unreal Header Tool)解析UCLASS/UFUNCTION宏生成反射代码
- **GamePlay框架**: GameMode、PlayerController、GameState等框架类大量使用蓝图扩展
- **行为树**: AI系统的Task节点可通过C++或蓝图实现
- **UMG/UI**: UMG控件可以绑定C++属性和函数
- **网络复制**: UPROPERTY(Replicated)标记的属性自动进行网络同步
