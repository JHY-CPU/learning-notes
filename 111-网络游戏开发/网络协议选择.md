# 网络协议选择

## 核心概念

网络协议选择直接影响游戏的延迟、可靠性和带宽效率。TCP 提供可靠有序传输但有队头阻塞问题；UDP 延迟低但需自行处理可靠性和排序；可靠 UDP（ENet/KCP）结合两者优点。选择合适的协议是网络游戏架构的基础决策。

## 数学基础

### TCP 延迟分析

TCP 在丢包时的延迟：

$$L_{TCP} = L_{base} + T_{RTO} \cdot N_{retransmit}$$

其中 $T_{RTO}$ 是重传超时（通常 200-1000ms），$N_{retransmit}$ 是重传次数。队头阻塞导致后续所有数据包都被延迟。

### UDP 延迟

UDP 的延迟：

$$L_{UDP} = L_{base}$$

无重传、无排序、无拥塞控制（除非自行实现）。

### KCP 延迟

KCP 的延迟介于 TCP 和纯 UDP 之间：

$$L_{KCP} = L_{base} + T_{resend}$$

其中 $T_{resend}$ 是快速重传时间（KCP 可配置为 10-30ms，远低于 TCP 的 200ms+）。

### 带宽效率

不同协议的头部开销：

| 协议 | 头部大小 | 有效数据比例（100字节负载） |
|------|---------|-------------------------|
| TCP | 20+20=40字节 | 60% |
| UDP | 8+20=28字节 | 72% |
| ENet | 28+8=36字节 | 64% |
| KCP | 28+24=52字节 | 48%（可靠模式） |

## 完整 C++ 可靠 UDP 实现

```cpp
#include <vector>
#include <unordered_map>
#include <queue>
#include <cstdint>
#include <cstring>
#include <chrono>

// ===== 序列号处理 =====
// 处理序列号回绕（16位序列号，0-65535循环）
inline bool SequenceGreaterThan(uint16_t s1, uint16_t s2) {
    return ((s1 > s2) && (s1 - s2 <= 32768)) ||
           ((s1 < s2) && (s2 - s1 > 32768));
}

inline int SequenceDifference(uint16_t s1, uint16_t s2) {
    if (s1 >= s2) return s1 - s2;
    return s1 + 65536 - s2;
}

// ===== 数据包头 =====
struct PacketHeader {
    uint16_t sequence;        // 序列号
    uint16_t ack;             // 确认号（最近收到的对方序列号）
    uint32_t ackBitfield;     // 确认位域（确认之前的32个包）
    uint8_t channel;          // 通道号
    uint8_t flags;            // 标志位
    // flags: bit0=可靠, bit1=有序, bit2=ACK, bit3=连接请求
};

// ===== 发送端缓冲 =====
struct SentPacket {
    uint16_t sequence;
    std::vector<uint8_t> data;
    float sendTime;
    int retryCount;
    bool acked;
};

// ===== 可靠 UDP 实现 =====
class ReliableUDP {
public:
    // 配置
    struct Config {
        float rttSmoothFactor = 0.1f;     // RTT 平滑因子
        float resendTimeout = 0.1f;       // 重传超时（秒）
        int maxRetries = 10;              // 最大重传次数
        int sendWindowSize = 256;         // 发送窗口大小
        int recvWindowSize = 256;         // 接收窗口大小
        bool nodelay = true;              // nodelay 模式
    };

    Config config;

    // 发送端状态
    uint16_t localSequence = 0;
    std::unordered_map<uint16_t, SentPacket> sentPackets;
    std::queue<std::vector<uint8_t>> sendQueue;

    // 接收端状态
    uint16_t remoteSequence = 0;
    uint32_t ackBitfield = 0;  // 记录最近32个包的接收情况
    std::unordered_map<uint16_t, std::vector<uint8_t>> recvBuffer;

    // RTT 统计
    float smoothedRTT = 0.1f;
    float rttVariance = 0.05f;

    // ===== 发送数据 =====
    void Send(const std::vector<uint8_t>& data, bool reliable = true,
              bool ordered = true, int channel = 0) {
        PacketHeader header;
        header.sequence = localSequence++;
        header.ack = remoteSequence;
        header.ackBitfield = ackBitfield;
        header.channel = (uint8_t)channel;
        header.flags = 0;
        if (reliable) header.flags |= 0x01;
        if (ordered) header.flags |= 0x02;

        // 打包数据
        std::vector<uint8_t> packet(sizeof(PacketHeader) + data.size());
        memcpy(packet.data(), &header, sizeof(PacketHeader));
        memcpy(packet.data() + sizeof(PacketHeader), data.data(), data.size());

        if (reliable) {
            // 保存到待确认队列
            SentPacket sent;
            sent.sequence = header.sequence;
            sent.data = packet;
            sent.sendTime = GetTime();
            sent.retryCount = 0;
            sent.acked = false;
            sentPackets[header.sequence] = sent;
        }

        // 发送到网络
        SendToNetwork(packet);
    }

    // ===== 接收数据 =====
    bool Receive(std::vector<uint8_t>& outData, int& outChannel) {
        // 检查有序接收缓冲
        auto it = recvBuffer.find((uint16_t)(remoteSequence + 1));
        if (it != recvBuffer.end()) {
            outData = std::move(it->second);
            recvBuffer.erase(it);
            remoteSequence++;
            return true;
        }
        return false;
    }

    // ===== 处理收到的原始数据 =====
    void OnPacketReceived(const uint8_t* rawData, int length) {
        if (length < (int)sizeof(PacketHeader)) return;

        PacketHeader header;
        memcpy(&header, rawData, sizeof(PacketHeader));

        // 1. 处理 ACK（对方确认了我们的包）
        ProcessAck(header.ack, header.ackBitfield);

        // 2. 更新远程序列号
        if (SequenceGreaterThan(header.sequence, remoteSequence)) {
            // 更新 ACK 位域
            int diff = SequenceDifference(header.sequence, remoteSequence);
            if (diff <= 32) {
                ackBitfield = (ackBitfield << diff) | 1;
            } else {
                ackBitfield = 0;
            }
            remoteSequence = header.sequence;
        }

        // 3. 存储数据
        if (header.flags & 0x01) {  // 可靠数据
            int payloadLen = length - sizeof(PacketHeader);
            std::vector<uint8_t> payload(rawData + sizeof(PacketHeader),
                                          rawData + sizeof(PacketHeader) + payloadLen);
            recvBuffer[header.sequence] = std::move(payload);
        }

        // 4. 发送 ACK
        SendAck(header.sequence);
    }

    // ===== 每帧更新 =====
    void Update(float currentTime) {
        // 检查重传
        for (auto& [seq, packet] : sentPackets) {
            if (packet.acked) continue;

            float elapsed = currentTime - packet.sendTime;
            if (elapsed > config.resendTimeout * (1 << packet.retryCount)) {
                if (packet.retryCount < config.maxRetries) {
                    // 重传
                    SendToNetwork(packet.data);
                    packet.retryCount++;
                    packet.sendTime = currentTime;
                } else {
                    // 超过最大重试次数，断开连接
                    OnConnectionLost();
                }
            }
        }

        // 清理已确认的包
        for (auto it = sentPackets.begin(); it != sentPackets.end(); ) {
            if (it->second.acked && (currentTime - it->second.sendTime) > smoothedRTT * 4) {
                it = sentPackets.erase(it);
            } else {
                ++it;
            }
        }
    }

    // ===== 获取 RTT =====
    float GetRTT() const { return smoothedRTT; }

private:
    void ProcessAck(uint16_t ackSeq, uint32_t ackBits) {
        auto it = sentPackets.find(ackSeq);
        if (it != sentPackets.end() && !it->second.acked) {
            it->second.acked = true;
            UpdateRTT(GetTime() - it->second.sendTime);
        }

        // 处理位域中确认的历史包
        for (int i = 0; i < 32; i++) {
            if (ackBits & (1 << i)) {
                uint16_t histSeq = ackSeq - 1 - i;
                auto histIt = sentPackets.find(histSeq);
                if (histIt != sentPackets.end() && !histIt->second.acked) {
                    histIt->second.acked = true;
                }
            }
        }
    }

    void UpdateRTT(float rtt) {
        smoothedRTT = smoothedRTT * (1 - config.rttSmoothFactor) + rtt * config.rttSmoothFactor;
        rttVariance = rttVariance * (1 - config.rttSmoothFactor) +
                      std::abs(rtt - smoothedRTT) * config.rttSmoothFactor;
    }

    void SendAck(uint16_t ackSeq) {
        PacketHeader ackHeader;
        ackHeader.sequence = 0;
        ackHeader.ack = ackSeq;
        ackHeader.ackBitfield = ackBitfield;
        ackHeader.channel = 0;
        ackHeader.flags = 0x04;  // ACK 标志

        std::vector<uint8_t> packet(sizeof(PacketHeader));
        memcpy(packet.data(), &ackHeader, sizeof(PacketHeader));
        SendToNetwork(packet);
    }

    void SendToNetwork(const std::vector<uint8_t>& data) {
        // 实际的网络发送（socket sendto）
    }

    float GetTime() const {
        using namespace std::chrono;
        return duration<float>(steady_clock::now().time_since_epoch()).count();
    }

    void OnConnectionLost() {
        // 处理连接丢失
    }
};
```

## KCP 配置示例

```cpp
// KCP 协议配置
void ConfigureKCPForGame(void* kcp) {
    // KCP 核心配置
    // nodelay=1: 开启 nodelay 模式
    // interval=10: 内部更新间隔 10ms
    // resend=2: 快速重传阈值（2次ACK丢失即重传）
    // nc=1: 关闭拥塞控制（游戏场景通常不需要）
    ikcp_nodelay((ikcpcb*)kcp, 1, 10, 2, 1);

    // 窗口大小
    ikcp_wndsize((ikcpcb*)kcp, 256, 256);  // 发送/接收窗口各 256

    // MTU 设置
    ikcp_setmtu((ikcpcb*)kcp, 1400);  // 避免 IP 分片
}

// KCP 的典型游戏配置总结
// - MOBA/RTS: nodelay=1, interval=10, resend=2, nc=1
// - FPS: nodelay=1, interval=5, resend=2, nc=1 (更低间隔)
// - MMO: nodelay=0, interval=40, resend=3, nc=0 (更省带宽)
```

## 常见陷阱与解决方案

| 陷阱 | 现象 | 解决方案 |
|------|------|---------|
| TCP 队头阻塞 | 一个丢包阻塞后续所有数据 | 使用 UDP 或可靠 UDP |
| UDP 被运营商限制 | 连接不稳定 | 使用端口伪装（如 443 端口） |
| MTU 超限 | IP 分片增加丢包率 | 设置 MTU 1400 以下 |
| 可靠 UDP 参数不当 | 带宽浪费或延迟过高 | 根据游戏类型调整 |

## 实际游戏案例

- **英雄联盟**：ENet（可靠 UDP），通道分离（输入/状态/聊天各自独立通道）
- **CS2**：Steam Datagram Relay (SDR)，基于 UDP 的中继服务
- **王者荣耀**：KCP + TCP 混合（实时战斗用 KCP，大厅/聊天用 TCP）
- **原神**：自研可靠 UDP + TCP 混合方案
- **堡垒之夜**：自研可靠 UDP，支持跨平台同步
