# 客户端预测

## 核心概念

客户端预测（Client-Side Prediction）允许客户端在玩家输入后立即响应，无需等待服务端确认，消除输入延迟感。当服务端返回权威结果后，客户端进行校验，若预测错误则回滚纠正。这是 FPS/TPS 类型游戏中消除延迟感的核心技术。

## 数学基础

### 预测误差模型

预测的准确性取决于客户端和服务端逻辑的一致性。误差随时间累积：

$$error(t) = \sum_{i=0}^{t/\Delta t} \delta_i$$

其中 $\delta_i$ 是第 $i$ 帧的微小误差（浮点精度、碰撞检测差异等）。典型的误差增长：

$$|error(t)| \approx \sigma \sqrt{N}$$

其中 $\sigma$ 是单帧标准差（约 0.001-0.01），$N$ 是未确认帧数。

### 纠正平滑公式

直接跳到正确位置会导致视觉抖动。使用指数衰减平滑：

$$pos_{display}(t+1) = pos_{display}(t) + (pos_{authoritative} - pos_{display}(t)) \cdot (1 - e^{-\lambda \cdot \Delta t})$$

其中 $\lambda$ 是纠正速度因子。$\lambda$ 越大纠正越快，但可能导致视觉抖动。

### 输入缓冲延迟

客户端预测的总延迟：

$$L_{perceived} = \max(0, L_{input} - T_{predict})$$

其中 $L_{input}$ 是网络传输延迟，$T_{predict}$ 是本地预测时间。理想情况下 $T_{predict} \ge L_{input}$，感知延迟为零。

## 完整 C++ 客户端预测系统

```cpp
#include <vector>
#include <deque>
#include <cstring>
#include <algorithm>
#include <glm/glm.hpp>

using Vec3 = glm::vec3;

// ===== 输入记录 =====
struct InputRecord {
    uint32_t sequenceId;      // 输入序列号
    uint32_t frameId;         // 对应的游戏帧
    uint8_t buttons;          // 按钮位掩码
    int16_t moveX;            // 移动方向 X
    int16_t moveY;            // 移动方向 Y
    int16_t viewYaw;          // 朝向角度
    int16_t viewPitch;        // 俯仰角度

    // 预测结果
    Vec3 predictedPosition;
    Vec3 predictedVelocity;
    float predictedHealth;
    bool predictionVerified;   // 是否已由服务端验证
};

// ===== 实体状态 =====
struct EntityState {
    Vec3 position;
    Vec3 velocity;
    float health;
    uint32_t frameId;
};

// ===== 客户端预测系统 =====
class ClientPrediction {
public:
    // 配置
    static constexpr int MAX_PENDING_INPUTS = 128;
    static constexpr float CORRECTION_SPEED = 10.0f;    // 纠正速度
    static constexpr float MAX_SNAP_DISTANCE = 3.0f;     // 最大瞬移距离（超过直接跳转）
    static constexpr float PREDICTION_ERROR_THRESHOLD = 0.1f;  // 预测误差阈值

    // 当前状态
    EntityState serverState;         // 最近一次服务端确认的状态
    EntityState displayState;        // 当前显示状态（可能包含纠正插值）
    uint32_t currentSequenceId = 0;
    uint32_t lastAckedSequenceId = 0;

    // 待确认的输入队列
    std::deque<InputRecord> pendingInputs;

    // 预测误差统计
    float totalError = 0;
    int errorCount = 0;
    int correctionCount = 0;

    // ===== 本地输入处理 =====
    void OnLocalInput(uint8_t buttons, int16_t moveX, int16_t moveY,
                      int16_t viewYaw, int16_t viewPitch) {
        // 1. 创建输入记录
        InputRecord input;
        input.sequenceId = currentSequenceId++;
        input.buttons = buttons;
        input.moveX = moveX;
        input.moveY = moveY;
        input.viewYaw = viewYaw;
        input.viewPitch = viewPitch;

        // 2. 立即在本地执行预测
        EntityState predictedState = ApplyInput(displayState, input);
        input.predictedPosition = predictedState.position;
        input.predictedVelocity = predictedState.velocity;
        input.predictedHealth = predictedState.health;
        input.predictionVerified = false;

        // 3. 更新显示状态（玩家立即看到效果）
        displayState = predictedState;

        // 4. 保存并发送
        pendingInputs.push_back(input);
        if (pendingInputs.size() > MAX_PENDING_INPUTS) {
            pendingInputs.pop_front();  // 丢弃最老的输入
        }

        // 5. 发送到服务端
        SendInputToServer(input);
    }

    // ===== 服务端确认回调 =====
    void OnServerAck(uint32_t ackedSequenceId, const EntityState& authoritativeState) {
        // 1. 找到对应的输入记录
        auto it = std::find_if(pendingInputs.begin(), pendingInputs.end(),
            [ackedSequenceId](const InputRecord& r) {
                return r.sequenceId == ackedSequenceId;
            });

        if (it == pendingInputs.end()) {
            // 已经被确认过了
            return;
        }

        // 2. 检查预测是否准确
        float positionError = glm::length(it->predictedPosition - authoritativeState.position);
        float healthError = std::abs(it->predictedHealth - authoritativeState.health);

        totalError += positionError;
        errorCount++;

        if (positionError > PREDICTION_ERROR_THRESHOLD || healthError > 0.01f) {
            // 预测错误，需要纠正
            correctionCount++;
            ApplyCorrection(authoritativeState, ackedSequenceId);
        }

        // 3. 清除已确认的输入
        pendingInputs.erase(pendingInputs.begin(), it + 1);
        lastAckedSequenceId = ackedSequenceId;
        serverState = authoritativeState;
    }

    // ===== 应用纠正 =====
    void ApplyCorrection(const EntityState& authoritative, uint32_t ackedSeq) {
        // 1. 如果误差过大，直接跳转
        float snapDist = glm::length(authoritative.position - displayState.position);
        if (snapDist > MAX_SNAP_DISTANCE) {
            displayState.position = authoritative.position;
            displayState.velocity = authoritative.velocity;
            displayState.health = authoritative.health;

            // 重新执行后续未确认的输入
            ReSimulateFrom(authoritative, ackedSeq);
            return;
        }

        // 2. 小误差：平滑纠正
        // 设置纠正目标，每帧在 UpdateCorrection 中渐进移动
        correctionTarget = authoritative;
        isCorrecting = true;

        // 3. 重新模拟后续未确认的输入
        ReSimulateFrom(authoritative, ackedSeq);
    }

    // ===== 每帧更新纠正插值 =====
    void UpdateCorrection(float dt) {
        if (!isCorrecting) return;

        // 指数衰减平滑纠正
        float factor = 1.0f - std::exp(-CORRECTION_SPEED * dt);

        displayState.position = glm::mix(displayState.position,
                                          correctionTarget.position, factor);
        displayState.velocity = glm::mix(displayState.velocity,
                                          correctionTarget.velocity, factor);

        // 检查是否已纠正到位
        if (glm::length(displayState.position - correctionTarget.position) < 0.001f) {
            isCorrecting = false;
        }
    }

    // ===== 获取当前显示位置 =====
    Vec3 GetDisplayPosition() const {
        return displayState.position;
    }

    // ===== 统计信息 =====
    struct PredictionStats {
        float averageError;
        int totalCorrections;
        int pendingInputCount;
        float correctionRate;
    };

    PredictionStats GetStats() const {
        return {
            (errorCount > 0) ? totalError / errorCount : 0.0f,
            correctionCount,
            (int)pendingInputs.size(),
            (errorCount > 0) ? (float)correctionCount / errorCount : 0.0f
        };
    }

private:
    EntityState correctionTarget;
    bool isCorrecting = false;

    // 应用输入到状态（确定性逻辑）
    EntityState ApplyInput(const EntityState& current, const InputRecord& input) {
        EntityState next = current;

        // 解析移动输入
        float moveScale = 1.0f / 1000.0f;
        Vec3 moveDir((float)input.moveX * moveScale, 0, (float)input.moveY * moveScale);
        float moveLen = glm::length(moveDir);
        if (moveLen > 1.0f) moveDir /= moveLen;  // 归一化

        float speed = 5.0f;  // 移动速度
        float dt = 1.0f / 30.0f;  // 固定时间步

        next.velocity = moveDir * speed;
        next.position = current.position + next.velocity * dt;

        // 简化：假设服务端有碰撞检测
        return next;
    }

    // 重新模拟后续输入
    void ReSimulateFrom(const EntityState& authoritative, uint32_t ackedSeq) {
        EntityState state = authoritative;

        for (auto& input : pendingInputs) {
            if (input.sequenceId > ackedSeq) {
                state = ApplyInput(state, input);
                input.predictedPosition = state.position;
                input.predictedVelocity = state.velocity;
                input.predictedHealth = state.health;
            }
        }

        // 更新显示状态为最新模拟结果
        displayState.position = state.position;
        displayState.velocity = state.velocity;
        displayState.health = state.health;
    }

    void SendInputToServer(const InputRecord& input) {
        // 网络层发送输入到服务端
    }
};
```

## 带插值缓冲的实体渲染

```cpp
// ===== 远端实体的插值系统 =====
class InterpolationBuffer {
public:
    struct StateEntry {
        Vec3 position;
        Quat rotation;
        float time;  // 服务端时间戳
    };

    std::deque<StateEntry> buffer;
    float interpolationDelay = 0.1f;  // 100ms 插值延迟

    void AddState(const Vec3& pos, const Quat& rot, float serverTime) {
        buffer.push_back({pos, rot, serverTime});

        // 清理过旧的状态
        float currentTime = GetClientTime();
        while (!buffer.empty() && buffer.front().time < currentTime - interpolationDelay - 1.0f) {
            buffer.pop_front();
        }
    }

    // 获取插值后的渲染位置
    bool GetInterpolatedState(float renderTime, Vec3& outPos, Quat& outRot) const {
        float interpTime = renderTime - interpolationDelay;

        if (buffer.size() < 2) return false;

        // 找到包围 interpTime 的两个状态
        const StateEntry* before = nullptr;
        const StateEntry* after = nullptr;

        for (size_t i = 0; i < buffer.size() - 1; i++) {
            if (buffer[i].time <= interpTime && buffer[i + 1].time >= interpTime) {
                before = &buffer[i];
                after = &buffer[i + 1];
                break;
            }
        }

        if (!before || !after) {
            // 超出缓冲范围，使用最新的
            outPos = buffer.back().position;
            outRot = buffer.back().rotation;
            return true;
        }

        // 线性插值
        float t = (interpTime - before->time) / (after->time - before->time);
        t = std::clamp(t, 0.0f, 1.0f);

        outPos = glm::mix(before->position, after->position, t);
        outRot = glm::slerp(before->rotation, after->rotation, t);

        return true;
    }

private:
    float GetClientTime() const {
        return 0;  // 实际实现使用系统时钟
    }
};
```

## 常见陷阱与解决方案

1. **预测逻辑与服务端不一致**：导致频繁回滚和视觉抖动。解决方案：客户端和服务端共享同一份物理/逻辑代码，确保确定性。

2. **纠正插值速度不当**：太快导致抖动，太慢导致角色"漂移"。解决方案：根据误差大小动态调整纠正速度，大误差直接跳转。

3. **累积误差**：浮点精度误差随帧数累积。解决方案：定期从服务端同步完整的权威状态。

4. **高延迟下预测回滚过多**：RTT > 200ms 时大量回滚导致画面不稳定。解决方案：增加输入延迟帧数以减少回滚频率。

## 实际游戏案例

- **CS2**：移动预测 + 射击延迟补偿的黄金标准，客户端预测 + 服务端权威校验
- **守望先锋**：20Hz 服务端 + 60Hz 客户端预测 + 插值缓冲
- **堡垒之夜**：建造系统的客户端预测（需极低延迟响应）
- **Rocket League**：物理预测 + 服务端回溯修正，球体物理的精确同步
- **Valorant**：128 tick 服务器 + 客户端预测 + 延迟补偿
