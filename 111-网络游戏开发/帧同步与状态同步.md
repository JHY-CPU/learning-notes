# 帧同步与状态同步

## 核心概念

帧同步（Lockstep）和状态同步（State Sync）是网络游戏的两大核心同步架构。帧同步只同步玩家输入，所有客户端按相同逻辑执行，保证确定性结果；状态同步由服务端计算世界状态，再同步给客户端。两种架构在带宽、延迟感、安全性、适用游戏类型等方面有根本性的差异。

## 数学基础

### 确定性要求

帧同步要求所有客户端的计算结果完全一致。浮点运算的不确定性是最大挑战：

不同 CPU 架构的浮点实现可能有微小差异。解决方案是使用定点数：

$$x_{fixed} = \lfloor x_{float} \times 2^{16} \rfloor$$

$$x_{float} = x_{fixed} / 2^{16}$$

其中 16 位小数位提供约 $1/65536 \approx 0.000015$ 的精度，足够游戏逻辑使用。

### 帧同步延迟模型

帧同步的输入延迟为：

$$L_{input} = \frac{1}{fps_{game}} + RTT_{max} + \frac{1}{fps_{server}}$$

其中 $RTT_{max}$ 是所有玩家中最大的往返延迟。典型值：游戏 30fps + 服务器收集 1帧 + RTT 100ms = 约 133ms 输入延迟。

### Rollback 回滚帧数

使用回滚机制时，回滚帧数：

$$N_{rollback} = \lceil \frac{RTT}{T_{frame}} \rceil$$

其中 $T_{frame}$ 是帧间隔时间。

```cpp
#include <vector>
#include <cstring>
#include <cstdint>

// ===== 定点数 =====
class Fixed {
public:
    int64_t value;
    static constexpr int FRACTIONAL_BITS = 16;
    static constexpr int64_t ONE = 1LL << FRACTIONAL_BITS;

    Fixed() : value(0) {}
    Fixed(int v) : value((int64_t)v << FRACTIONAL_BITS) {}
    Fixed(float v) : value((int64_t)(v * ONE)) {}

    float ToFloat() const { return (float)value / ONE; }

    Fixed operator+(Fixed o) const { Fixed r; r.value = value + o.value; return r; }
    Fixed operator-(Fixed o) const { Fixed r; r.value = value - o.value; return r; }
    Fixed operator*(Fixed o) const {
        Fixed r; r.value = (value * o.value) >> FRACTIONAL_BITS; return r;
    }
    Fixed operator/(Fixed o) const {
        Fixed r; r.value = (value << FRACTIONAL_BITS) / o.value; return r;
    }
    bool operator>(Fixed o) const { return value > o.value; }
    bool operator<(Fixed o) const { return value < o.value; }
    bool operator>=(Fixed o) const { return value >= o.value; }
    bool operator==(Fixed o) const { return value == o.value; }
};

// ===== 2D定点向量 =====
struct FixedVec2 {
    Fixed x, y;
    FixedVec2() {}
    FixedVec2(Fixed x, Fixed y) : x(x), y(y) {}

    Fixed LengthSq() const { return x * x + y * y; }

    FixedVec2 operator+(FixedVec2 o) const { return {x + o.x, y + o.y}; }
    FixedVec2 operator-(FixedVec2 o) const { return {x - o.x, y - o.y}; }
    FixedVec2 operator*(Fixed s) const { return {x * s, y * s}; }
};

// ===== 确定性随机数生成器 =====
class DeterministicRNG {
public:
    uint32_t seed;

    DeterministicRNG(uint32_t s = 12345) : seed(s) {}

    uint32_t Next() {
        // Xorshift32 算法（确定性、快速）
        seed ^= seed << 13;
        seed ^= seed >> 17;
        seed ^= seed << 5;
        return seed;
    }

    int Range(int min, int max) {
        return min + (int)(Next() % (uint32_t)(max - min + 1));
    }

    Fixed FixedRange(Fixed min, Fixed max) {
        uint32_t r = Next();
        Fixed t = Fixed((float)r / 0xFFFFFFFF);
        return min + (max - min) * t;
    }
};

// ===== 玩家输入 =====
struct PlayerInput {
    uint8_t playerId;
    uint32_t frameId;
    int16_t moveX;      // -1000 ~ 1000（定点化移动方向）
    int16_t moveY;
    uint8_t buttons;     // 位掩码：攻击、跳跃、技能1、技能2等
    uint8_t skillDir;    // 技能方向（0-255，映射到360度）

    static constexpr int MAX_PLAYERS = 8;
};

// ===== 游戏实体 =====
struct GameEntity {
    uint32_t entityId;
    FixedVec2 position;
    FixedVec2 velocity;
    Fixed health;
    uint8_t state;       // 状态枚举
    uint32_t animFrame;
};

// ===== 世界状态 =====
struct WorldState {
    uint32_t frameId;
    std::vector<GameEntity> entities;
    DeterministicRNG rng;
};

// ===== 帧同步核心系统 =====
class LockstepSystem {
public:
    static constexpr int FRAME_RATE = 30;
    static constexpr float FIXED_DT = 1.0f / FRAME_RATE;
    static constexpr int INPUT_DELAY_FRAMES = 2;  // 输入延迟帧数

    uint32_t currentFrame = 0;
    WorldState world;
    DeterministicRNG rng;

    // 输入缓冲：每帧每个玩家的输入
    PlayerInput inputBuffer[1024][PlayerInput::MAX_PLAYERS];  // 帧 -> 玩家
    bool inputReceived[1024][PlayerInput::MAX_PLAYERS];

    // 游戏逻辑回调
    std::function<void(WorldState&, const PlayerInput*, int)> gameLogicCallback;

    // 服务端：收集并广播输入
    void ServerUpdate() {
        // 1. 检查当前帧所有玩家的输入是否到齐
        bool allReceived = true;
        for (int p = 0; p < MAX_PLAYERS; p++) {
            if (!inputReceived[currentFrame % 1024][p]) {
                allReceived = false;
                break;
            }
        }

        if (!allReceived) {
            // 等待最慢的玩家（或使用超时机制）
            return;
        }

        // 2. 广播所有玩家的输入
        BroadcastFrameInputs(currentFrame);

        // 3. 推进帧
        currentFrame++;
    }

    // 客户端：执行逻辑
    void ClientUpdate(float dt) {
        // 收到服务器广播后执行
        const PlayerInput* inputs = inputBuffer[currentFrame % 1024];

        // 执行确定性游戏逻辑
        if (gameLogicCallback) {
            gameLogicCallback(world, inputs, PlayerInput::MAX_PLAYERS);
        }

        // 确定性更新
        SimulateWorld(FIXED_DT);

        currentFrame++;
    }

    // 客户端预测：输入后立即执行，服务端确认后校验
    void ClientPredictLocalInput(const PlayerInput& localInput) {
        // 保存输入
        inputBuffer[currentFrame % 1024][localInput.playerId] = localInput;
        inputReceived[currentFrame % 1024][localInput.playerId] = true;

        // 立即执行（客户端预测）
        ClientUpdate(FIXED_DT);

        // 发送输入到服务器
        SendInputToServer(localInput);
    }

    // 服务端确认后回滚校验
    void OnServerConfirmed(uint32_t frameId, const PlayerInput* authoritativeInputs) {
        // 比较预测的输入和权威输入
        bool needsResync = false;
        for (int p = 0; p < PlayerInput::MAX_PLAYERS; p++) {
            if (memcmp(&inputBuffer[frameId % 1024][p], &authoritativeInputs[p],
                       sizeof(PlayerInput)) != 0) {
                needsResync = true;
                break;
            }
        }

        if (needsResync) {
            // 回滚到该帧，使用权威输入重新模拟
            RollbackToFrame(frameId, authoritativeInputs);
        }
    }

    // 回滚到指定帧并重新模拟
    void RollbackToFrame(uint32_t frame, const PlayerInput* authoritativeInputs) {
        // 1. 加载该帧的快照
        WorldState savedState = LoadSnapshot(frame);

        // 2. 使用权威输入重新模拟到当前帧
        for (uint32_t f = frame; f <= currentFrame; f++) {
            // 获取该帧的输入
            const PlayerInput* inputs;
            if (f == frame) {
                inputs = authoritativeInputs;  // 使用权威输入
            } else {
                inputs = inputBuffer[f % 1024];  // 后续帧使用本地输入
            }

            if (gameLogicCallback) {
                gameLogicCallback(savedState, inputs, PlayerInput::MAX_PLAYERS);
            }
            SimulateWorldState(savedState, FIXED_DT);
        }

        // 3. 用重新模拟的结果替换当前状态
        world = savedState;
    }

private:
    void SimulateWorld(float dt) {
        SimulateWorldState(world, dt);
    }

    void SimulateWorldState(WorldState& state, float dt) {
        for (auto& entity : state.entities) {
            // 固定时间步物理更新（确定性）
            entity.position = entity.position + entity.velocity * Fixed(dt);
        }
    }

    void BroadcastFrameInputs(uint32_t frame) {
        // 网络层广播
    }

    void SendInputToServer(const PlayerInput& input) {
        // 网络层发送
    }

    WorldState LoadSnapshot(uint32_t frame) {
        // 从快照存储加载
        return world;  // 简化
    }

    int MAX_PLAYERS = PlayerInput::MAX_PLAYERS;
};
```

## 状态同步完整实现

```cpp
// ===== 状态同步系统 =====
class StateSyncSystem {
public:
    struct EntityState {
        uint32_t entityId;
        Vec3 position;
        Quat rotation;
        Vec3 velocity;
        float health;
        uint32_t animState;
        uint32_t sequenceNumber;  // 序列号，用于增量同步
    };

    struct WorldSnapshot {
        uint32_t frameId;
        uint64_t timestamp;
        std::vector<EntityState> entities;
    };

    // 服务端：生成状态快照
    WorldSnapshot GenerateFullSnapshot() const {
        WorldSnapshot snap;
        snap.frameId = currentFrame;
        snap.timestamp = GetTimestamp();
        snap.entities = currentEntities;
        return snap;
    }

    // 增量快照：只包含变化的实体
    WorldSnapshot GenerateDeltaSnapshot(uint32_t lastAckFrame,
                                         const std::unordered_set<uint32_t>& dirtyEntities) const {
        WorldSnapshot snap;
        snap.frameId = currentFrame;
        snap.timestamp = GetTimestamp();

        for (const auto& entity : currentEntities) {
            if (dirtyEntities.count(entity.entityId)) {
                snap.entities.push_back(entity);
            }
        }

        return snap;
    }

    // 客户端：接收并应用快照
    void ApplySnapshot(const WorldSnapshot& snapshot) {
        for (const auto& entityState : snapshot.entities) {
            auto it = localEntities.find(entityState.entityId);
            if (it != localEntities.end()) {
                // 平滑插值到新状态
                it->second.targetState = entityState;
                it->second.interpStartTime = GetTime();
                it->second.interpDuration = 0.1f;  // 100ms 插值
            } else {
                // 新实体
                localEntities[entityState.entityId] = {entityState, entityState, GetTime(), 0.1f};
            }
        }
    }

    // 客户端插值渲染
    void InterpolateForRender(float renderTime) {
        for (auto& [id, data] : localEntities) {
            float t = (renderTime - data.interpStartTime) / data.interpDuration;
            t = std::clamp(t, 0.0f, 1.0f);

            data.renderPosition = glm::mix(data.currentState.position,
                                           data.targetState.position, t);
            data.renderRotation = glm::slerp(data.currentState.rotation,
                                             data.targetState.rotation, t);
        }
    }

private:
    uint32_t currentFrame = 0;
    std::vector<EntityState> currentEntities;  // 服务端权威状态

    struct InterpData {
        EntityState currentState;
        EntityState targetState;
        float interpStartTime;
        float interpDuration;
        Vec3 renderPosition;
        Quat renderRotation;
    };
    std::unordered_map<uint32_t, InterpData> localEntities;
};
```

```cpp
class HybridSyncSystem {
public:
    // 输入同步用于低延迟的核心逻辑（移动、攻击）
    LockstepSystem inputSync;

    // 状态同步用于非关键逻辑（特效、动画状态、道具刷新）
    StateSyncSystem stateSync;

    void Update(float dt) {
        // 1. 核心逻辑使用帧同步（低延迟）
        inputSync.ClientUpdate(dt);

        // 2. 非关键逻辑使用状态同步（带宽友好）
        stateSync.InterpolateForRender(GetTime());

        // 3. 定期校验关键状态
        frameCounter++;
        if (frameCounter % 30 == 0) {  // 每秒校验一次
            SyncKeyStates();

    void OnLocalInput(const PlayerInput& input) {
        // 本地输入立即执行（帧同步）
        inputSync.ClientPredictLocalInput(input);

    int frameCounter = 0;
```

## 适用场景深度对比

| 维度 | 帧同步 | 状态同步 | 混合方案 |
|------|--------|---------|---------|
| 带宽消耗 | 极低（~1KB/s/玩家） | 较高（~10-50KB/s/玩家） | 中等 |
| 输入延迟 | 固定 ~100ms | 可低至 ~50ms | 低延迟+稳定性 |
| 反作弊 | 客户端有完整逻辑，易破解 | 服务端权威，安全性高 | 核心逻辑安全 |
| 断线重连 | 需回放所有帧（耗时） | 直接同步最新快照（快速） | 快速重连 |
| 服务器负载 | 极低（只转发输入） | 高（需计算所有状态） | 中等 |
| 确定性要求 | 极高（跨平台定点数） | 无要求 | 部分要求 |
| 适用类型 | RTS、格斗、MOBA | FPS、MMO、RPG | 混合类型 |
| 逻辑更新频率 | 固定（通常20-30Hz） | 灵活 | 灵活 |

## 常见陷阱与解决方案

### 帧同步陷阱

1. **浮点运算不一致**：不同编译器/CPU 的 `sin`/`cos` 结果可能有微小差异。解决方案：使用定点数数学库，或用查表法实现三角函数。

2. **随机数不一致**：不同平台的 `rand()` 实现不同。解决方案：使用确定性随机数生成器（如 Xorshift），并确保种子一致。

3. **实体更新顺序不一致**：`std::unordered_map` 遍历顺序不确定。解决方案：按实体 ID 排序后遍历。

4. **本地时间/系统调用**：使用 `time()` 或 `clock()` 会导致不一致。解决方案：只使用帧计数器作为时间源。

### 状态同步陷阱

1. **带宽爆炸**：1000 个实体每帧同步位置导致带宽过高。解决方案：使用增量同步+优先级+量化压缩。

2. **插值延迟**：100ms 的插值缓冲导致操作不跟手。解决方案：本地玩家使用客户端预测，远端实体使用插值。

3. **快照丢失**：网络丢包导致客户端缺少某些帧的状态。解决方案：定期发送完整快照，或使用冗余发送。

## 实际游戏案例

- **英雄联盟**：经典帧同步架构，输入延迟约 100ms，使用确定性物理和定点数
- **星际争霸2**：帧同步 + 确定性锁步，回放系统基于输入记录
- **CS2**：状态同步 + 客户端预测 + 延迟补偿，高 Tick Rate 服务器（64/128 tick）
- **守望先锋**：状态同步 + 高 Tick Rate（60Hz 客户端，60Hz 服务器）
- **王者荣耀**：帧同步（20Hz）+ 延迟补偿，移动端优化
- **Dota 2**：帧同步架构，使用 Source 2 引擎的确定性系统
- **Apex Legends**：状态同步 + 客户端预测，使用改良的 Source 引擎
