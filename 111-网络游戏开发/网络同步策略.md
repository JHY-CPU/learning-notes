# 网络同步策略

## 核心概念

网络同步策略决定游戏状态如何从服务端传输到客户端，核心目标是以最小带宽和延迟保证各客户端的游戏状态一致。主要包括全量同步、增量同步、属性同步和兴趣管理等策略。

## 数学基础

### 带宽预算

每个客户端的带宽预算 $B$ 每秒，帧率为 $f$，则每帧可传输：

$$budget\_per\_frame = \frac{B}{f} \times 8 \text{ bits}$$

假设预算 50KB/s，30fps：

$$budget\_per\_frame = \frac{50 \times 1024 \times 8}{30} \approx 13653 \text{ bits} \approx 1.7 \text{ KB/frame}$$

### 增量压缩增益

增量同步的压缩率取决于属性变化频率：

$$compression\_ratio = 1 - \frac{changed\_properties}{total\_properties}$$

如果每帧只有 10% 的属性变化，理论压缩率为 90%。

### 优先级衰减函数

实体的重要性随距离衰减：

$$priority(d) = \frac{P_0}{1 + (d/d_0)^n}$$

其中 $P_0$ 是基础优先级，$d_0$ 是特征距离，$n$ 是衰减指数（通常 1-2）。

## 完整 C++ 同步策略系统

```cpp
#include <vector>
#include <unordered_map>
#include <unordered_set>
#include <queue>
#include <algorithm>
#include <cstring>
#include <glm/glm.hpp>

using Vec3 = glm::vec3;

// ===== 脏标记系统 =====
class DirtyFlagSystem {
public:
    // 属性脏标记
    enum DirtyFlags : uint32_t {
        DIRTY_POSITION   = 1 << 0,
        DIRTY_ROTATION   = 1 << 1,
        DIRTY_VELOCITY   = 1 << 2,
        DIRTY_HEALTH     = 1 << 3,
        DIRTY_ANIM_STATE = 1 << 4,
        DIRTY_INVENTORY  = 1 << 5,
        DIRTY_BUFFS      = 1 << 6,
        DIRTY_ALL        = 0xFFFFFFFF
    };

    struct SyncedEntity {
        uint32_t entityId;
        uint32_t dirtyFlags = DIRTY_ALL;
        uint32_t lastSyncFrame = 0;

        // 当前状态
        Vec3 position;
        Quat rotation;
        Vec3 velocity;
        float health;
        uint8_t animState;

        // 上次同步的状态（用于增量比较）
        Vec3 lastSyncedPosition;
        Quat lastSyncedRotation;
        float lastSyncedHealth;
    };

    std::unordered_map<uint32_t, SyncedEntity> entities;

    // 设置属性并标记脏位
    void SetPosition(uint32_t entityId, const Vec3& pos) {
        auto it = entities.find(entityId);
        if (it != entities.end()) {
            it->second.position = pos;
            it->second.dirtyFlags |= DIRTY_POSITION;
        }
    }

    void SetHealth(uint32_t entityId, float hp) {
        auto it = entities.find(entityId);
        if (it != entities.end()) {
            it->second.health = hp;
            it->second.dirtyFlags |= DIRTY_HEALTH;
        }
    }

    // 获取指定实体的脏标记
    uint32_t GetDirtyFlags(uint32_t entityId) const {
        auto it = entities.find(entityId);
        return (it != entities.end()) ? it->second.dirtyFlags : 0;
    }

    // 清除已同步的脏标记
    void ClearDirtyFlags(uint32_t entityId) {
        auto it = entities.find(entityId);
        if (it != entities.end()) {
            it->second.dirtyFlags = 0;
            it->second.lastSyncedPosition = it->second.position;
            it->second.lastSyncedRotation = it->second.rotation;
            it->second.lastSyncedHealth = it->second.health;
            it->second.lastSyncFrame = currentFrame;
        }
    }

private:
    uint32_t currentFrame = 0;
};

// ===== 同步频率 =====
enum SyncFrequency {
    EVERY_TICK,      // 位置、旋转（每帧同步）
    HALF_TICK,       // 速度、动画状态（隔帧同步）
    ON_CHANGE,       // 生命值、装备（变化时同步）
    ON_DEMAND,       // 详细信息（客户端请求时同步）
    NEVER            // 不同步（仅供服务端使用）
};

// ===== 优先级同步 =====
class PrioritySync {
public:
    struct EntityPriority {
        uint32_t entityId;
        float priority;
    };

    // 计算实体对特定玩家的优先级
    float CalculatePriority(uint32_t entityId, const Vec3& viewerPos,
                            const Vec3& viewerForward, float fovAngle) {
        auto it = dirtySystem->entities.find(entityId);
        if (it == dirtySystem->entities.end()) return 0;

        const auto& entity = it->second;
        float priority = 0;

        // 1. 距离因子（越近越重要）
        Vec3 toEntity = entity.position - viewerPos;
        float dist = glm::length(toEntity);
        priority += 10.0f / (dist + 1.0f);

        // 2. 视锥因子（在视野内更优先）
        if (dist > 0.001f) {
            Vec3 dir = toEntity / dist;
            float dot = glm::dot(dir, viewerForward);
            if (dot > std::cos(fovAngle * 0.5f)) {
                priority += 5.0f * dot;  // 视野中心优先级更高
            }
        }

        // 3. 脏标记因子（有变化的更优先）
        if (entity.dirtyFlags != 0) {
            priority += 3.0f;

            // 关键属性变化优先级更高
            if (entity.dirtyFlags & DirtyFlagSystem::DIRTY_HEALTH) priority += 2.0f;
            if (entity.dirtyFlags & DirtyFlagSystem::DIRTY_POSITION) priority += 1.0f;
        }

        // 4. 类型因子
        // 玩家 > 怪物 > NPC > 物品
        priority += GetTypePriority(entityId);

        return priority;
    }

    // 获取本帧需要同步的实体（按优先级排序）
    std::vector<uint32_t> GetTopEntities(const Vec3& viewerPos,
                                          const Vec3& viewerForward,
                                          int maxPerFrame) {
        std::vector<EntityPriority> allPriorities;

        for (const auto& [id, entity] : dirtySystem->entities) {
            float p = CalculatePriority(id, viewerPos, viewerForward, 90.0f);
            if (p > 0) {
                allPriorities.push_back({id, p});
            }
        }

        // 按优先级降序排序
        std::sort(allPriorities.begin(), allPriorities.end(),
            [](const EntityPriority& a, const EntityPriority& b) {
                return a.priority > b.priority;
            });

        std::vector<uint32_t> result;
        for (int i = 0; i < std::min(maxPerFrame, (int)allPriorities.size()); i++) {
            result.push_back(allPriorities[i].entityId);
        }

        return result;
    }

    DirtyFlagSystem* dirtySystem = nullptr;

private:
    float GetTypePriority(uint32_t entityId) const {
        // 根据实体类型返回优先级
        return 1.0f;  // 简化
    }
};

// ===== 增量同步打包器 =====
class DeltaSyncPacker {
public:
    // 打包实体的增量数据
    std::vector<uint8_t> PackEntityDelta(const DirtyFlagSystem::SyncedEntity& entity,
                                          uint32_t dirtyFlags) {
        std::vector<uint8_t> packet;

        // 写入实体 ID
        WriteUint32(packet, entity.entityId);
        // 写入脏标记
        WriteUint32(packet, dirtyFlags);

        if (dirtyFlags & DirtyFlagSystem::DIRTY_POSITION) {
            // 量化位置：精度 0.01m, 范围 -1000~1000
            WriteUint16(packet, QuantizeFloat(entity.position.x, -1000, 1000, 0.01f));
            WriteUint16(packet, QuantizeFloat(entity.position.y, -1000, 1000, 0.01f));
            WriteUint16(packet, QuantizeFloat(entity.position.z, -1000, 1000, 0.01f));
        }

        if (dirtyFlags & DirtyFlagSystem::DIRTY_ROTATION) {
            // 量化四元数：每个分量 10 bit
            WriteUint32(packet, PackQuaternion(entity.rotation));
        }

        if (dirtyFlags & DirtyFlagSystem::DIRTY_VELOCITY) {
            WriteUint16(packet, QuantizeFloat(entity.velocity.x, -50, 50, 0.01f));
            WriteUint16(packet, QuantizeFloat(entity.velocity.y, -50, 50, 0.01f));
            WriteUint16(packet, QuantizeFloat(entity.velocity.z, -50, 50, 0.01f));
        }

        if (dirtyFlags & DirtyFlagSystem::DIRTY_HEALTH) {
            WriteUint8(packet, QuantizeFloat(entity.health, 0, 100, 0.4f));
        }

        if (dirtyFlags & DirtyFlagSystem::DIRTY_ANIM_STATE) {
            WriteUint8(packet, entity.animState);
        }

        return packet;
    }

private:
    uint16_t QuantizeFloat(float value, float min, float max, float precision) {
        float normalized = (value - min) / (max - min);
        return (uint16_t)(normalized / precision);
    }

    uint32_t PackQuaternion(const Quat& q) {
        // 找出绝对值最大的分量，只编码其他3个
        int largest = 0;
        float largestVal = std::abs(q.x);
        if (std::abs(q.y) > largestVal) { largest = 1; largestVal = std::abs(q.y); }
        if (std::abs(q.z) > largestVal) { largest = 2; largestVal = std::abs(q.z); }
        if (std::abs(q.w) > largestVal) { largest = 3; largestVal = std::abs(q.w); }

        float values[4] = {q.x, q.y, q.z, q.w};
        float scale = 1.0f / (largVal * 2.0f);  // 归一化到 -0.5~0.5

        uint32_t packed = (uint32_t)largest;
        int idx = 0;
        for (int i = 0; i < 4; i++) {
            if (i == largest) continue;
            uint32_t quantized = (uint32_t)((values[i] * scale + 0.5f) * 1023);
            packed |= (quantized & 0x3FF) << (2 + idx * 10);
            idx++;
        }

        return packed;
    }

    void WriteUint8(std::vector<uint8_t>& buf, uint8_t val) { buf.push_back(val); }
    void WriteUint16(std::vector<uint8_t>& buf, uint16_t val) {
        buf.push_back(val & 0xFF);
        buf.push_back((val >> 8) & 0xFF);
    }
    void WriteUint32(std::vector<uint8_t>& buf, uint32_t val) {
        buf.push_back(val & 0xFF);
        buf.push_back((val >> 8) & 0xFF);
        buf.push_back((val >> 16) & 0xFF);
        buf.push_back((val >> 24) & 0xFF);
    }
};
```

## 常见陷阱与解决方案

1. **增量丢失基础状态**：客户端丢失某帧的完整快照后，后续增量无法应用。解决方案：定期（如每秒）发送完整快照作为基准。

2. **脏标记未清除**：导致同一属性被重复同步。解决方案：在同步数据发送后立即清除对应脏标记。

3. **优先级计算过频**：每帧对所有实体计算优先级消耗 CPU。解决方案：降低优先级计算频率（如每 0.1 秒），或使用空间哈希加速。

4. **丢包导致增量链断裂**：网络丢包导致客户端缺少中间帧的状态。解决方案：冗余发送（包含前几帧的增量），或检测到断裂后请求完整快照。

## 实际游戏案例

- **魔兽世界**：AOI + 属性分级同步，按实体类型和距离调整同步频率
- **堡垒之夜**：100 人战场使用优先级 + 增量同步，带宽控制在 50KB/s/客户端
- **Rust**：实体优先级同步 + 增量压缩，支持 300+ 玩家同服
- **Unreal Engine**：Replication 系统支持属性同步 + 条件复制 + 优先级
- **Apex Legends**：60 人战场使用高效的增量同步和属性分级
