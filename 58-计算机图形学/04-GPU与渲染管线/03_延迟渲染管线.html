延迟渲染管线

一、前向渲染 vs 延迟渲染

传统前向渲染（Forward Rendering）对每个物体执行完整的光照计算，当场景中光源数量增多时性能线性下降。延迟渲染（Deferred Rendering）将几何信息和光照计算分离，光照计算的复杂度与场景几何复杂度无关。

1.1 对比

维度前向渲染延迟渲染
渲染Pass物体 × 光源次几何Pass + 光源Pass
光照复杂度O(物体数 × 光源数)O(像素数 × 光源数)
多光源支持较差优秀
带宽消耗低高（G-buffer读写）
MSAA支持原生支持需要额外处理
透明物体容易处理需要额外前向Pass

二、G-Buffer（几何缓冲区）

G-Buffer是延迟渲染的核心数据结构，存储了场景中每个像素的几何和材质信息。它通常由多个渲染目标（MRT）组成。

2.1 G-Buffer内容

RT通道内容格式
RT0RGBA漫反射颜色 (RGB) + 环境光遮蔽 (A)RGBA8
RT1RGBA世界空间法线 (RGB) + 粗糙度 (A)RGBA16F
RT2RGBA镜面反射颜色 (RGB) + 金属度 (A)RGBA8
RT3RG运动向量 (RG)RG16F
DepthD深度值D24S8

2.2 几何Pass（Geometry Pass）

// 几何Pass - 片段着色器
#version 330 core

layout (location = 0) out vec4 gAlbedoAO;
layout (location = 1) out vec4 gNormalRoughness;
layout (location = 2) out vec4 gSpecMetallic;

in vec3 FragPos;
in vec3 Normal;
in vec2 TexCoord;

uniform sampler2D albedoMap;
uniform sampler2D normalMap;
uniform sampler2D metallicRoughnessMap;

void main() {
    // RT0: 漫反射颜色 + AO
    gAlbedoAO.rgb = texture(albedoMap, TexCoord).rgb;
    gAlbedoAO.a = 1.0; // AO

    // RT1: 世界空间法线 + 粗糙度
    vec3 normal = normalize(Normal); // 实际应从法线贴图读取
    gNormalRoughness.rgb = normal * 0.5 + 0.5;
    gNormalRoughness.a = texture(metallicRoughnessMap, TexCoord).g; // roughness

    // RT2: 镜面反射 + 金属度
    gSpecMetallic.rgb = vec3(0.04); // 非金属的F0
    gSpecMetallic.a = texture(metallicRoughnessMap, TexCoord).b; // metallic
}

三、光照Pass（Lighting Pass）

光照Pass是一个全屏后处理操作，读取G-Buffer中的信息，对每个像素计算光照。

3.1 光照Pass实现

// 光照Pass - 片段着色器
#version 330 core

uniform sampler2D gAlbedoAO;
uniform sampler2D gNormalRoughness;
uniform sampler2D gSpecMetallic;
uniform sampler2D gDepth;

uniform vec3 viewPos;
uniform mat4 invViewProj;

// 光源数组
struct PointLight {
    vec3 position;
    vec3 color;
    float radius;
};
uniform PointLight lights[64];
uniform int numLights;

out vec4 FragColor;

void main() {
    vec2 uv = gl_FragCoord.xy / screenSize;

    // 从G-Buffer读取数据
    vec3 albedo = texture(gAlbedoAO, uv).rgb;
    vec3 normal = texture(gNormalRoughness, uv).rgb * 2.0 - 1.0;
    float roughness = texture(gNormalRoughness, uv).a;
    float metallic = texture(gSpecMetallic, uv).a;
    float depth = texture(gDepth, uv).r;

    // 重建世界坐标
    vec4 clipPos = vec4(uv * 2.0 - 1.0, depth * 2.0 - 1.0, 1.0);
    vec4 worldPos = invViewProj * clipPos;
    vec3 fragPos = worldPos.xyz / worldPos.w;

    vec3 viewDir = normalize(viewPos - fragPos);
    vec3 result = vec3(0.0);

    // 对每个光源计算光照
    for (int i = 0; i < numLights; i++) {
        vec3 lightDir = normalize(lights[i].position - fragPos);
        vec3 halfDir = normalize(lightDir + viewDir);

        float dist = length(lights[i].position - fragPos);
        float attenuation = clamp(1.0 - dist / lights[i].radius, 0.0, 1.0);
        attenuation *= attenuation; // 平方衰减

        // 简化光照
        float NdotL = max(dot(normal, lightDir), 0.0);
        float NdotH = max(dot(normal, halfDir), 0.0);
        float spec = pow(NdotH, mix(1.0, 128.0, 1.0 - roughness));

        vec3 diffuse = albedo * NdotL;
        vec3 specular = mix(vec3(0.04), albedo, metallic) * spec;

        result += (diffuse + specular) * lights[i].color * attenuation;
    }

    FragColor = vec4(result, 1.0);
}

四、光源体积优化

全屏光照Pass会处理所有像素，即使某些像素不受特定光源影响。使用光源体积可以只处理受光源影响的像素。

4.1 点光源体积

  • 为每个点光源渲染一个球体（包围球）
  • 只对球体覆盖的像素执行光照计算
  • 球体大小等于光源的衰减半径
  • 使用模板测试或深度测试剔除无关像素
优化技巧：
- 使用Icosphere近似球体，减少顶点数
- 用背面剔除渲染球体，相机在光源内部时切换为正面剔除
- 在球体片段着色器中读取G-Buffer并计算光照

五、Tiled Deferred Rendering

Tiled延迟渲染将屏幕划分为小的瓦片（tile），为每个瓦片计算影响它的光源列表，避免对不受光源影响的像素执行光照计算。

5.1 算法流程

  1. 几何Pass：生成G-Buffer（同普通延迟渲染）
  2. 光源剔除Pass：将屏幕划分为16×16像素的瓦片，用计算着色器判断每个光源与哪些瓦片相交
  3. 光照Pass：每个瓦片只计算其光源列表中的光照

5.2 Clustered Deferred Rendering

Clustered延迟渲染在Tiled的基础上增加深度维度，将视锥体划分为3D簇（cluster），更精确地剔除光源。

方案划分方式优势劣势
Tiled2D屏幕瓦片简单高效深度不连续时过度包含
Clustered3D视锥体簇更精确剔除实现复杂

六、Forward+ Rendering

Forward+结合了前向渲染和延迟渲染的优点：使用与Tiled/Clustered相同的光源剔除策略，但在前向渲染Pass中直接计算光照，避免了G-Buffer的带宽开销。

6.1 Forward+流程

  1. 深度Pre-Pass：渲染场景深度（可选，但推荐）
  2. 光源剔除Pass：用计算着色器构建瓦片/簇的光源列表
  3. 前向渲染Pass：渲染物体，片段着色器从光源列表计算光照

6.2 延迟 vs Forward+ 对比

维度DeferredForward+
G-Buffer需要（高带宽）不需要
MSAA困难原生支持
透明物体需要额外Pass同Pass处理

多光源效率优秀优秀
显存消耗高较低

- Unreal Engine 5：Clustered Deferred + 前向渲染混合
- Unity：Forward+（URP）/ Deferred（内置管线）
- Frostbite（战地）：Tiled Deferred
- 寒霜引擎：Clustered Forward
