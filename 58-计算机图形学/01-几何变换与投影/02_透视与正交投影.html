透视与正交投影

一、投影概述

投影是将3D场景映射到2D平面的过程，是图形渲染管线中观察变换后的关键步骤。

投影类型特点应用场景
正交投影平行线保持平行，无近大远小工程制图、CAD、2D游戏
透视投影平行线汇聚于灭点，近大远小3D游戏、虚拟现实、电影

二、正交投影（Orthographic Projection）

正交投影将观察空间中的长方体映射到标准化设备坐标（NDC）的立方体中。

2.1 观察体定义

正交投影的观察体是一个轴对齐长方体，由六个面定义：

参数含义
left, right左右平面（x范围）
bottom, top上下平面（y范围）
near, far近远平面（z范围）

2.2 正交投影矩阵

| 2/(r-l) 0 0 -(r+l)/(r-l) | | 0 2/(t-b) 0 -(t+b)/(t-b) | | 0 0 -2/(f-n) -(f+n)/(f-n) | | 0 0 0 1 |

将[left,right]×[bottom,top]×[near,far]映射到[-1,1]³（OpenGL NDC）。

三、透视投影（Perspective Projection）

透视投影模拟人眼视觉，产生"近大远小"的效果。观察体是一个视锥体（Frustum）。

3.1 视锥体（View Frustum）

  • 定义参数：垂直视场角（fovY）、宽高比（aspect）、近平面（near）、远平面（far）
  • 形状：截断的金字塔形，近平面小、远平面大
  • 近裁剪面：太近会产生视觉伪影，通常设为较小正值
  • 远裁剪面：太远会导致z-fighting精度问题

3.2 透视投影矩阵（OpenGL风格）

| 1/(aspect·tan(fov/2)) 0 0 0 | | 0 1/tan(fov/2) 0 0 | | 0 0 -(f+n)/(f-n) -2fn/(f-n) | | 0 0 -1 0 |

3.3 透视除法（Perspective Division）

透视投影矩阵将w分量设置为-z，之后需要执行透视除法：(x,y,z,w) → (x/w, y/w, z/w)。

效果：距离相机越远的点，w值越大，除以w后坐标越靠近中心，产生"近大远小"效果。

四、裁剪（Clipping）

裁剪移除视锥体外的图元，只保留在可见区域内的部分。

4.1 裁剪空间

  • 齐次裁剪：在w分量空间中判断：-w ≤ x,y,z ≤ w
  • 线段裁剪：Cohen-Sutherland、Liang-Barsky算法
  • 多边形裁剪：Sutherland-Hodgman算法

4.2 裁剪与视口变换流程

  1. 应用投影矩阵（齐次裁剪空间）
  2. 执行裁剪（-w ≤ x,y,z ≤ w）
  3. 透视除法（NDC空间，[-1,1]³）
  4. 视口变换（屏幕像素坐标）

五、正交 vs 透视投影对比

维度正交投影透视投影
平行线保持平行汇聚于灭点
大小关系物体大小不随距离变化近大远小
观察体长方体视锥体（平截头体）
深度值线性分布非线性分布（近处精度高）
矩阵复杂度简单需要透视除法
视觉效果工程感、不真实符合人眼视觉
深度精度问题：透视投影的深度值非线性分布，靠近近平面精度高，靠近远平面精度低。应尽量减小far/near比值来减少z-fighting。
