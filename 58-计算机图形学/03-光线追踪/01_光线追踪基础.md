# 光线追踪基础

一、光线追踪概述

## 一、光线追踪概述


光线追踪是一种基于物理光学的渲染技术，通过追踪光线在场景中的传播路径来生成逼真的图像。


### 1.1 基本原理


- 从相机发射光线穿过每个像素
- 计算光线与场景中物体的最近交点
- 在交点处计算光照和反射/折射
- 递归追踪反射和折射光线


### 1.2 光栅化 vs 光线追踪


| 维度 | 光栅化 | 光线追踪 |
| --- | --- | --- |
| 核心操作 | 图元→像素 | 像素→图元 |
| 速度 | 极快（实时） | 较慢（离线/实时） |
| 全局效果 | 难以实现 | 自然支持 |
| 硬件支持 | GPU高度优化 | RT核心加速 |

二、光线表示

## 二、光线的数学表示

**光线参数方程：**


`R(t) = O + t × D`


O = 光线起点（Origin）


D = 光线方向（Direction，单位向量）


t = 参数（t ≥ 0 表示光线的正向）
三、光线-物体求交

## 三、光线-物体求交


### 3.1 光线-球体求交


球心C、半径r，求解方程：

`|O + tD - C|² = r²`


展开为：at² + bt + c = 0


其中 a = D·D, b = 2D·(O-C), c = (O-C)·(O-C) - r²


判别式 Δ = b² - 4ac：


Δ < 0：无交点


Δ = 0：相切（一个交点）


Δ > 0：两个交点，取最小正t值

### 3.2 光线-平面求交

平面方程：
`N·P + d = 0`
（N为法线，d为偏移）


交点参数：
`t = -(N·O + d) / (N·D)`


t > 0 且 N·D ≠ 0 时有交点

### 3.3 光线-三角形求交


1. 先求光线与三角形所在平面的交点
2. 用重心坐标判断交点是否在三角形内
3. α, β, γ > 0 且 α+β+γ = 1 时在三角形内

四、反射与折射

## 四、反射与折射


### 4.1 反射（Reflection）

**反射方向：**


`R = D - 2(D·N)N`


D = 入射方向


N = 表面法线


R = 反射方向

### 4.2 折射（Refraction）

**Snell定律：**


`η₁ sin(θ₁) = η₂ sin(θ₂)`


η₁, η₂ = 两种介质的折射率


θ₁ = 入射角，θ₂ = 折射角

### 4.3 全内反射（Total Internal Reflection）


当光从光密介质射向光疏介质，入射角大于临界角时，发生全反射。


> **Note:** **临界角：**
> `θ_c = arcsin(η₂/η₁)`
> ，当 η₁ > η₂ 且 θ₁ > θ_c 时，无折射，只有反射。

五、阴影

## 五、阴影光线（Shadow Rays）


在交点处向光源发射阴影光线，检查是否被其他物体遮挡。


- 从交点向光源发射光线
- 如果在到达光源前有交点，则该点在阴影中
- 只使用环境光分量
- 软阴影需要面光源和多条阴影光线

六、加速结构

## 六、加速结构


光线追踪需要对所有图元进行求交测试，加速结构可以大幅减少不必要的测试。


| 结构 | 原理 | 特点 |
| --- | --- | --- |
| BVH | 包围体层次结构 | 最常用，动态场景友好 |
| KD-Tree | 空间二分树 | 静态场景效率高 |
| Octree | 八叉树 | 简单实现 |
| Uniform Grid | 均匀网格 | 均匀分布场景高效 |


> **Note:** **BVH构建：**
> 递归地将物体集合分为两个子集，用包围盒（AABB）包裹每个子集。遍历时如果光线不与包围盒相交，则跳过该子树的所有物体。

七、光线追踪方程

## 七、渲染方程

**渲染方程（Rendering Equation）：**


`L_o(p, ω_o) = L_e(p, ω_o) + ∫_Ω f_r(p, ω_i, ω_o) × L_i(p, ω_i) × (n·ω_i) dω_i`


L_o = 出射光


L_e = 自发光


f_r = BRDF（双向反射分布函数）


L_i = 入射光


n·ω_i = Lambert余弦项
