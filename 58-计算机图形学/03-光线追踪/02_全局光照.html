全局光照

一、全局光照概述

全局光照（Global Illumination, GI）是指模拟光线在场景中多次反弹的渲染技术，包括间接光照、颜色渗色、柔和阴影、焦散等效果。它是实现照片级真实感渲染的核心。

1.1 直接光照 vs 全局光照

类型包含内容效果
直接光照光源→表面→相机基本明暗
全局光照光源→多次反弹→相机渗色、间接照明、焦散、柔和阴影

二、路径追踪（Path Tracing）

路径追踪是基于蒙特卡洛方法对渲染方程进行数值求解的技术。它从相机发射光线，每次反弹随机选择方向，通过大量采样收敛到精确解。

2.1 渲染方程

渲染方程（Kajiya 1986）：
L_o(p, ω_o) = L_e(p, ω_o) + ∫_Ω f_r(p, ω_i, ω_o) × L_i(p, ω_i) × cos(θ_i) dω_i

L_o = 出射辐射亮度
L_e = 自发光
f_r = BRDF（双向反射分布函数）
L_i = 入射辐射亮度（来自所有方向的光线）
cos(θ_i) = Lambert余弦项
Ω = 半球积分

2.2 路径追踪算法

// 路径追踪伪代码
Color trace(Ray ray, int depth) {
    if (depth > MAX_DEPTH) return BLACK;

    Hit hit = scene.intersect(ray);
    if (!hit) return environmentColor(ray.direction);

    // 自发光
    Color color = hit.emission;

    // 随机选择反射方向（半球采样）
    vec3 wi = randomHemisphere(hit.normal);

    // BRDF权重
    float pdf = 1.0 / (2.0 * PI);  // 均匀采样PDF
    float cosTheta = dot(hit.normal, wi);
    Color brdf = hit.brdf / PI;     // Lambert BRDF

    // 递归追踪
    Ray newRay(hit.point, wi);
    Color incoming = trace(newRay, depth + 1);

    // 蒙特卡洛估计
    color += brdf * incoming * cosTheta / pdf;

    return color;
}

// 对每个像素多次采样取平均
for each pixel:
    color = BLACK
    for i = 1 to N_SAMPLES:
        ray = camera.getRay(pixel, randomJitter())
        color += trace(ray, 0)
    pixel.color = color / N_SAMPLES

三、蒙特卡洛积分

蒙特卡洛积分通过随机采样来估计积分值。在路径追踪中，它用于估计渲染方程中的半球积分。

3.1 基本原理

蒙特卡洛估计：
积分 I = ∫ f(x) dx 的蒙特卡洛估计：
I ≈ (1/N) × Σ f(x_i) / p(x_i)

其中 x_i 是从概率密度函数 p(x) 中采样的随机点
N越大估计越精确，收敛速度为 O(1/√N)

3.2 重要性采样（Importance Sampling）

选择与被积函数形状相似的PDF可以大幅降低方差，加速收敛。

场景推荐PDF原因
Lambert漫反射cos加权半球采样BRDF包含cosθ，PDF也包含cosθ可抵消
光滑镜面BRDF加权采样采样集中在镜面反射方向
面光源照明光源面积采样直接采样光源方向贡献最大
环境光照环境贴图加权亮度高的方向贡献大
Cos加权半球采样：
PDF = cos(θ) / π，此时BRDF/PDF = (ρ/π) / (cosθ/π) = ρ/π × π/cosθ × cosθ = ρ/π
cosθ项被抵消，方差大大降低。这是Lambert材质最有效的采样方式。

四、双向路径追踪（BDPT）

BDPT同时从光源和相机各追踪一条路径，然后连接两条路径上的顶点来估计光照。对于某些困难场景（如小光源+间接照明），BDPT比单向路径追踪收敛更快。

4.1 算法流程

  1. 相机路径：从相机追踪s个顶点到达场景
  2. 光源路径：从光源追踪t个顶点到达场景
  3. 连接：将两条路径的端点连接（需可见性测试）
  4. 多策略：对所有可能的(s,t)组合取加权平均

4.2 BDPT优势

  • 对小面积光源+间接光照效果显著好于单向追踪
  • 自动适应各种光路（漫反射-光泽-漫反射等）
  • 是更高级算法（VCM、MLT）的基础

五、MLT（Metropolis Light Transport）

MLT使用马尔可夫链蒙特卡洛（MCMC）方法，从已有的高贡献样本出发，通过小的随机扰动生成新样本。它能自适应地聚焦于贡献大的光路。

5.1 核心思想

MCMC采样：
1. 从一个初始样本x0出发
2. 提议一个新样本x'（通过扰动x得到）
3. 计算接受概率 a = min(1, f(x')/f(x))
4. 以概率a接受x'，否则保留x
5. 重复步骤2-4

长期来看，样本分布会趋近于目标分布f(x)，不需要知道归一化常数。

5.2 MLT优缺点

优点缺点
自适应聚焦困难光路样本之间相关，不适用于并行
理论上能渲染任意复杂场景难以估计收敛进度
对焦散、间接照明效率高可能存在"萤火虫"伪影
不需要显式采样策略每个像素需要独立链（效率低）

六、全局光照算法对比

算法收敛速度并行性实现难度适用场景
路径追踪O(1/√N)完美并行简单通用渲染
BDPT更快（间接光）中等中等小光源场景
MLT自适应差较难复杂光路
VCM/UPS综合最优中等较难通用高端渲染
现代渲染器选择：
- 实时渲染：通常使用屏幕空间GI（SSGI）、光追降噪
- 电影级渲染：路径追踪+BDPT+MLT混合使用
- 游戏引擎（Unreal 5 Lumen）：软件光追+硬件光追混合
- 学术研究：VCM（Vertex Connection and Merging）结合BDPT和光子映射
