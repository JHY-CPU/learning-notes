# 03 - 光线追踪

> 光线追踪通过模拟光线在场景中的传播来生成逼真的图像，是离线渲染和实时光追的核心技术。

---

## 1. 光线追踪基本原理

### 1.1 光线投射（Ray Casting）

从相机出发，对每个像素发射一条光线：
1. 光线穿过像素对应的屏幕位置
2. 与场景中最近的物体求交
3. 根据交点处的材质和光照计算颜色

### 1.2 递归光线追踪

Whitted 风格光线追踪在交点处递归生成：
- **反射光线**：模拟镜面反射
- **折射光线**：模拟透明物体
- **阴影光线**：检测是否被光源直接照射

递归终止条件：达到最大深度、光线未命中任何物体、贡献值低于阈值。

### 1.3 与光栅化的对比

| 特性 | 光栅化 | 光线追踪 |
|------|--------|----------|
| 方向 | 物体→像素 | 像素→场景 |
| 全局光照 | 困难 | 天然支持 |
| 性能 | 实时 | 较慢（传统） |
| 阴影/反射 | 需额外处理 | 自然产生 |

## 2. 光线-物体求交

### 2.1 光线参数方程

```
P(t) = O + t·D,  t > 0
```

O 为光线起点，D 为方向向量，t 为参数。

### 2.2 光线-平面求交

平面方程 **n** · P + d = 0：

```
t = -(n · O + d) / (n · D)
```

需检查 t > 0 且分母不为零。

### 2.3 光线-球面求交

将光线方程代入球面方程 |P - C|² = r²，得到关于 t 的二次方程：

```
a·t² + b·t + c = 0
```

其中 a = D·D，b = 2(O-C)·D，c = (O-C)·(O-C) - r²

根据判别式 Δ = b² - 4ac 判断：Δ < 0 无交点，Δ = 0 相切，Δ > 0 两个交点（取较小的 t）。

### 2.4 光线-三角形求交

**Möller-Trumbore 算法**（无需预计算平面方程）：

```
P = O + tD = (1-u-v)V₀ + uV₁ + vV₂
```

通过 Cramer 法则求解 t, u, v，检查 t > 0, u ≥ 0, v ≥ 0, u+v ≤ 1。

### 2.5 光线-AABB 包围盒求交

AABB（轴对齐包围盒）由 [tmin, tmax] 定义：
1. 对每个轴分别计算光线进入和离开的时间
2. 取三个进入时间的最大值和三个离开时间的最小值
3. 若 tmin ≤ tmax 且 tmax > 0，则相交

这是光线追踪中最频繁的求交测试。

## 3. 加速结构

### 3.1 均匀空间划分（Uniform Grid）

将场景空间均匀划分为网格单元，每个单元存储与之相交的物体列表：
- 光线沿 DDA 方式遍历经过的网格单元
- 仅与当前单元内的物体求交

**适用场景**：物体均匀分布的场景。对聚集或极度分散的场景效率低。

### 3.2 BVH（Bounding Volume Hierarchy）

递归地将物体集合划分为两组，用包围盒包裹：
- 构建：按面积最大轴划分，选择 SAH（表面面积启发式）代价最优的划分
- 遍历：若光线不命中父节点包围盒，则跳过整个子树

BVH 是现代光线追踪的主流加速结构，构建快、遍历高效。

### 3.3 KD-Tree

沿某一轴将空间二分，递归划分：
- 划分轴轮流选择 x, y, z 或选面积最大的面
- 每个内部节点存储划分位置和轴向
- 叶节点存储物体列表

**特点**：空间划分而非物体划分，遍历效率高但构建较慢。

### 3.4 八叉树（Octree）

每次将空间划分为 8 个子立方体：
- 适合三维空间的层次划分
- 构建简单但可能产生不均衡的树

## 4. 反射与折射

### 4.1 反射

反射方向公式：

```
r = d - 2(d · n)n
```

d 为入射方向，n 为法线。完美镜面反射。

### 4.2 折射（Snell 定律）

```
η₁ sinθ₁ = η₂ sinθ₂
```

η 为折射率。折射方向的计算需考虑全内反射（当光线从高折射率介质射向低折射率介质且入射角超过临界角时）。

### 4.3 Fresnel 方程

描述反射和折射能量的分配：
- 垂直入射：反射比例小
- 掠射入射：反射比例趋近 1

**Schlick 近似**：

```
R(θ) = R₀ + (1 - R₀)(1 - cosθ)⁵
```

R₀ = ((η₁ - η₂)/(η₁ + η₂))²

Fresnel 效应解释了为什么几乎所有物体在掠射角都显得更亮。

## 5. 阴影光线

从交点向光源发射阴影光线：
- 若阴影光线命中任何物体，则该点处于阴影中
- 硬阴影：只有全亮和全暗
- 软阴影：对面积光源采样多条阴影光线

## 6. 全局光照基础

### 6.1 渲染方程（Rendering Equation）

```
Lo(p, ωo) = Le(p, ωo) + ∫ fr(p, ωi, ωo) · Li(p, ωi) · (n · ωi) dωi
```

- Lo：出射辐射亮度
- Le：自发光
- fr：BRDF（双向反射分布函数）
- Li：入射辐射亮度

渲染方程是图形学中光照计算的统一理论框架。

### 6.2 路径追踪（Path Tracking）

用 Monte Carlo 方法求解渲染方程：
1. 从相机发射光线
2. 在交点处按 BRDF 采样一个随机方向
3. 沿该方向递归追踪
4. 用蒙特卡洛估计累加辐射度

路径追踪能收敛到精确解，是物理正确的渲染算法。

### 6.3 辐射度方法（Radiosity）

基于热传导的全局光照方法：
- 将场景表面划分为面片
- 计算面片之间的形状因子（Form Factor）
- 迭代求解能量平衡方程

适合漫反射为主的场景（如室内），不能处理镜面反射。

## 7. 蒙特卡洛光线追踪

### 7.1 Monte Carlo 积分

用随机采样估计积分值：

```
∫f(x)dx ≈ (1/N) Σ f(xi)/p(xi)
```

p(x) 为采样概率密度函数。

### 7.2 方差缩减

- **分层采样**：将积分域分层，在每层内采样
- **准蒙特卡洛**：使用低差异序列（Halton, Sobol）
- **多重重要性采样**：混合多个采样策略

---

## 8. 重要性采样

### 8.1 原理

选择与被积函数形状相似的概率密度函数采样，可大幅降低方差。

### 8.2 常用策略

- **BRDF 采样**：按材质的 BRDF 分布采样
- **光源采样**：直接对光源区域采样（对小光源特别有效）
- **余弦加权半球采样**：适合漫反射表面

### 8.3 多重重要性采样（MIS）

将多种采样策略的结果加权组合，自动选择最优策略：

```
wᵢ(xi) = (pᵢ(xi) · nᵢ)² / Σⱼ (pⱼ(xj) · nⱼ)²
```

## 9. 降噪技术

### 9.1 空域滤波

- **双边滤波**：保边平滑，考虑像素值相似性
- **联合双边滤波**：利用辅助信息（深度、法线、albedo）引导滤波
- **非局部均值**：利用图像中的相似块进行去噪

### 9.2 时域滤波

- **时间累积**：多帧之间累积采样结果
- **运动补偿**：利用运动向量对齐不同帧的数据
- **指数移动平均**：根据场景变化自适应调整累积权重

### 9.3 AI 降噪

- **基于 CNN 的降噪器**：训练神经网络从少量采样中重建高质量图像
- **NVIDIA OptiX AI Denoiser**：实时光追标配降噪方案
- **OIDN（Open Image Denoise）**：Intel 开源的 AI 降噪库

## 10. 实时光线追踪

### 10.1 DirectX Raytracing (DXR)

微软在 DirectX 12 中引入的光追 API：
- 光线追踪管线（Ray Tracing Pipeline）
- 加速结构（Acceleration Structure）
- Shader 类型：Ray Generation、Closest Hit、Any Hit、Miss

### 10.2 NVIDIA RTX

RTX 系列 GPU 的专用硬件：
- **RT Core**：硬件加速 BVH 遍历和三角形求交
- **Tensor Core**：用于 AI 降噪
- **混合渲染**：光栅化为主 + 光追用于反射/阴影/GI

### 10.3 Vulkan Ray Tracing

Khronos 的跨平台光追扩展：
- VK_KHR_ray_tracing_pipeline
- VK_KHR_acceleration_structure
- 与 DXR 类似的管线模型

---

*光线追踪是通向照片级真实感渲染的关键路径。从经典 Whitted 光追到现代实时光追，这一领域持续推动着图形学的发展。*
