三角形光栅化

一、光栅化概述

光栅化是将几何图元（主要是三角形）转换为离散像素片段（Fragment）的过程，是实时渲染的核心步骤。

为什么使用三角形：三角形是最简单的多边形，总是平面的、凸的，任何多边形都可以剖分为三角形，且三点确定一个平面便于插值。

二、扫描线算法（Scanline）

扫描线算法从上到下逐行扫描，计算每行与三角形边的交点，填充交点之间的像素。

2.1 算法步骤

  1. 按y坐标对三角形顶点排序
  2. 从最高顶点开始，逐行向下扫描
  3. 对每条扫描线，计算与三角形边的交点
  4. 填充交点之间的所有像素

2.2 优化：增量计算

边的斜率倒数 m = dx/dy 在扫描过程中不变，因此可以用增量计算避免每行重复求交：
x_new = x_old + m

三、边方程法（Edge Equation）

边方程法通过计算像素中心到每条边的位置关系来判断是否在三角形内部。

3.1 边方程定义

对于边从点A到点B，边方程为：

E(x, y) = (x - xA)(yB - yA) - (y - yA)(xB - xA)

E > 0：点在边的左侧
E = 0：点在边上
E < 0：点在边的右侧

3.2 三角形内外判断

  • 如果三角形顶点是逆时针排列，则像素中心对三条边的E值都 ≥ 0时在三角形内
  • 可以通过统一顶点顺序来处理任意排列
  • 需要处理边界情况（像素在边上的归属规则）

四、重心坐标（Barycentric Coordinates）

重心坐标是一种描述点在三角形内部位置的参数化方法，用于顶点属性的插值。

4.1 定义

三角形内任意点P可以表示为三个顶点的加权和：

P = α·A + β·B + γ·C
其中 α + β + γ = 1

α = P对面三角形面积 / ABC面积
β = P对面三角形面积 / ABC面积
γ = P对面三角形面积 / ABC面积

4.2 计算方法

利用叉积计算面积比：

α = E_BC(P) / E_BC(A)
β = E_CA(P) / E_CA(B)
γ = 1 - α - β

4.3 属性插值

重心坐标可用于插值任何顶点属性（颜色、纹理坐标、法线等）：

V(P) = α·V(A) + β·V(B) + γ·V(C)

注意：透视投影下，需要进行透视校正插值，不能直接使用屏幕空间重心坐标。

4.4 透视校正插值

问题：屏幕空间的重心坐标与世界空间不同，直接插值会导致纹理扭曲。
解决方案：对属性除以w进行插值，再乘以w恢复：
V(P) = (α·V_A/w_A + β·V_B/w_B + γ·V_C/w_C) / (α/w_A + β/w_B + γ/w_C)

五、Z-Buffer（深度缓冲）

Z-Buffer算法通过比较像素的深度值来解决可见性问题，是最常用的隐藏面消除算法。

5.1 算法原理

  • 维护一个与屏幕同尺寸的深度缓冲区，初始值为最大深度
  • 对每个像素，计算当前片段的深度值
  • 如果深度 < Z-Buffer中的值，更新颜色和深度
  • 否则丢弃该片段

5.2 优缺点

优点缺点
简单高效，易于硬件实现需要额外的深度缓冲内存
支持任意几何体透明物体处理困难
逐像素处理深度精度有限（z-fighting）
Z-Fighting：当两个表面非常接近时，浮点精度不足导致深度值交替胜出，产生闪烁。
解决方案：增大深度精度、减小far/near比、多边形偏移（polygon offset）。
