阴影算法

一、阴影概述

阴影是增强3D场景真实感的关键视觉线索，它提供了物体之间的空间关系和光源位置信息。实时渲染中主要有两种阴影技术：阴影映射（Shadow Mapping）和阴影体（Shadow Volumes）。

1.1 阴影类型

类型特点形成原因
本影（Umbra）完全被遮挡，最暗区域点光源被完全遮挡
半影（Penumbra）部分被遮挡，渐变区域面光源被部分遮挡

二、阴影映射（Shadow Mapping）

阴影映射是实时渲染中最广泛使用的阴影技术。其核心思想是从光源视角渲染深度图，然后在相机渲染时比较深度来判断是否在阴影中。

2.1 算法步骤

  1. Pass 1 - 生成阴影贴图：从光源视角渲染场景，只记录深度到Shadow Map
  2. Pass 2 - 场景渲染：从相机视角渲染，将每个片段变换到光源空间
  3. 深度比较：比较片段的深度与Shadow Map中的深度
    - 片段深度 > Shadow Map深度 → 在阴影中
    - 片段深度 ≤ Shadow Map深度 → 被照亮

2.2 阴影贴图实现

// Pass 1: 从光源视角生成深度图
mat4 lightView = lookAt(lightPos, target, up);
mat4 lightProj = ortho(-10, 10, -10, 10, 0.1, 100);
// 使用光源的VP矩阵渲染场景到深度纹理

// Pass 2: 片段着色器中判断阴影
uniform sampler2D shadowMap;
uniform mat4 lightSpaceMatrix;

float calculateShadow(vec3 fragPos, vec3 normal, vec3 lightDir) {
    // 将片段位置变换到光源空间
    vec4 fragPosLS = lightSpaceMatrix * vec4(fragPos, 1.0);
    vec3 projCoords = fragPosLS.xyz / fragPosLS.w;
    projCoords = projCoords * 0.5 + 0.5; // 变换到[0,1]

    // 从Shadow Map读取深度
    float closestDepth = texture(shadowMap, projCoords.xy).r;
    float currentDepth = projCoords.z;

    // 深度偏移防止阴影痤疮
    float bias = max(0.05 * (1.0 - dot(normal, lightDir)), 0.005);
    float shadow = currentDepth - bias > closestDepth ? 1.0 : 0.0;

    return shadow;
}

2.3 阴影痤疮与彼得潘现象

阴影痤疮（Shadow Acne）：
由于Shadow Map分辨率有限和浮点精度问题，同一表面在Shadow Map中的深度和当前片段深度可能不一致，导致表面出现条纹状伪影。

解决方案：
- 深度偏移（Bias）：比较时减去一个小偏移量
- 仅对正面生成Shadow Map（glCullFace(GL_FRONT)）
- 使用斜率相关的偏移量

彼得潘现象（Peter Panning）：
Bias过大导致阴影与物体根部分离，看起来物体悬浮在地面之上。解决方案是使用尽可能小的bias。

三、PCF（Percentage Closer Filtering）

PCF通过在多个相邻纹素上进行深度比较，然后取平均值来实现柔和的阴影边缘，解决阴影边缘锯齿问题。

3.1 PCF原理

PCF vs 普通过滤的区别：
- 普通过滤：先对Shadow Map做模糊，再比较深度 → 不正确，会混合不同物体的深度
- PCF：先对每个纹素比较深度得到0/1，再对比较结果做平均 → 正确的软阴影效果

3.2 PCF实现

// 3x3 PCF采样
float PCFShadow(vec2 uv, float currentDepth, float bias) {
    float shadow = 0.0;
    vec2 texelSize = 1.0 / textureSize(shadowMap, 0);

    for (int x = -1; x <= 1; ++x) {
        for (int y = -1; y <= 1; ++y) {
            float pcfDepth = texture(shadowMap,
                uv + vec2(x, y) * texelSize).r;
            shadow += currentDepth - bias > pcfDepth ? 1.0 : 0.0;
        }
    }
    return shadow / 9.0; // 9个样本取平均
}

四、方差阴影映射（VSM）

VSM存储深度的一阶矩（均值）和二阶矩（均方），利用切比雪夫不等式估计遮挡概率，可以用普通的纹理过滤实现软阴影。

4.1 VSM原理

VSM数据存储：
Shadow Map存储两个值：
- R通道：深度均值 E[X]
- G通道：深度平方均值 E[X²]

切比雪夫不等式：
P(x ≥ t) ≤ σ² / (σ² + (t - μ)²)
其中 μ = E[X], σ² = E[X²] - E[X]²（方差）
t = 当前片段深度
P表示"当前片段在阴影中的概率"

4.2 VSM优缺点

优点缺点
可用硬件mipmap和双线性过滤光渗（Light Bleeding）问题
柔和的阴影边缘需要浮点纹理格式
不需要多次采样两个物体在同一深度时出错
光渗（Light Bleeding）：当遮挡物和被遮挡物的深度分布重叠时，VSM可能错误地让一些光线"渗漏"过来。可以通过钳制最小概率或使用指数方差阴影映射（EVSM）缓解。

五、级联阴影映射（CSM）

CSM将视锥体划分为多个层级，每层使用独立的Shadow Map，近处使用高分辨率，远处使用低分辨率，解决大场景阴影精度不均的问题。

5.1 CSM原理

  1. 将相机视锥体沿深度分为2~4个层级（Cascade）
  2. 为每个层级计算合适的光源投影矩阵
  3. 每个层级使用独立的Shadow Map
  4. 渲染时根据片段深度选择对应的Cascade

Cascade边界处理：
为了避免相邻Cascade之间的可见接缝，通常在过渡区域混合两个Cascade的结果。混合区间一般为Cascade深度范围的10~20%。

六、阴影体（Shadow Volumes）

阴影体通过几何方法精确计算阴影区域。从光源视角，将物体轮廓边沿光线方向拉伸形成"阴影体"，然后用模板缓冲判断像素是否在阴影体内。

6.1 模板缓冲计数

Z-pass算法：
1. 初始化模板缓冲为0
2. 渲染阴影体的正面：深度测试通过时模板值+1
3. 渲染阴影体的背面：深度测试通过时模板值-1
4. 模板值 > 0 的像素在阴影中

原理：从相机到像素的射线，每穿过一次阴影体正面就进入阴影，穿过背面就离开阴影。

6.2 阴影体 vs 阴影映射
