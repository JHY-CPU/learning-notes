# 02 - 光栅化与着色

> 光栅化将连续的几何图元转化为离散的像素，着色则为每个像素赋予最终颜色。

---

## 1. 直线光栅化

### 1.1 DDA 算法（Digital Differential Analyzer）

DDA 算法通过数值微分逐步计算直线上的像素点：

**基本思想**：
- 以 x 或 y 中变化较大的方向为步进方向
- 每步计算另一坐标的值
- 取整得到像素位置

**时间复杂度**：O(max(|Δx|, |Δy|))

**缺点**：每步涉及浮点乘法和取整，效率较低。

### 1.2 Bresenham 算法

Bresenham 算法仅使用整数运算，是实际应用中的标准方法：

**核心思想**：
- 维护决策参数 d（误差项）
- 每步根据 d 的符号决定下一个像素的位置
- d 的更新仅涉及整数加减法

**算法流程**（以斜率 0 < m < 1 为例）：
1. 初始化 d = 2Δy - Δx
2. 若 d < 0：选择右方像素，d ← d + 2Δy
3. 若 d ≥ 0：选择右上方像素，d ← d + 2Δy - 2Δx

**优势**：纯整数运算，无浮点误差，速度快。

## 2. 圆的光栅化

### 2.1 Bresenham 圆算法

利用圆的八对称性，只需计算八分之一圆弧上的点：

**决策参数**：
- 若选择水平方向像素（E 点），d ← d + 2x + 3
- 若选择对角方向像素（SE 点），d ← d + 2(x - y) + 5

初始 d = 3 - 2r（r 为半径）。

### 2.2 中点圆算法

在中点处判断下一个像素应选在圆内还是圆外：
- 计算中点到圆心的距离与半径的差
- 根据正负选择更接近圆的像素

与 Bresenham 圆算法本质相同，只是决策函数的推导方式不同。

## 3. 多边形填充

### 3.1 扫描线算法（Scan-Line）

**基本流程**：
1. 建立边表（Edge Table），按 y 排序
2. 从下到上逐行扫描
3. 维护活跃边表（AET），计算每行的交点
4. 在交点对之间填充像素

**处理特殊情况**：
- 水平边：直接忽略
- 局部极值顶点：记录为一个交点
- 非局部极值顶点：记录为两个交点

### 3.2 种子填充算法（Seed Fill）

从多边形内部一个种子点出发，向四周扩散填充：

- **四连通**：检查上下左右四个方向
- **八连通**：检查八个方向（可填充带对角缝隙的区域）

**缺点**：递归深度可能很大，需要大量栈空间；适合交互式填充。

## 4. 深度缓冲（Z-Buffer）

### 4.1 算法原理

Z-Buffer 是最常用的隐藏面消除算法：

1. 初始化深度缓冲区为最大深度值
2. 对每个图元的每个像素，计算其深度值 z
3. 若 z < ZBuffer[x][y]，更新颜色和深度

**优点**：简单通用，无需预排序图元。

**缺点**：需要额外的深度缓冲内存；对每个像素进行深度测试。

### 4.2 浮点精度问题

- 深度值非线性分布（透视投影后近处精度高，远处精度低）
- 可能导致 z-fighting（深度冲突）
- 解决方案：对数深度缓冲、反转深度（reversed-Z）

---

## 5. 画家算法（Painter's Algorithm）

### 5.1 原理

按照深度从远到近的顺序绘制图元，后绘制的覆盖先绘制的。

### 5.2 问题

- 深度排序可能不存在（循环重叠的情况）
- 无法处理图元互相穿插的场景
- 效率低于 Z-Buffer（可能需要多次排序）

画家算法更多用于概念理解，实际渲染中已被 Z-Buffer 取代。

## 6. 光照模型

### 6.1 环境光（Ambient Light）

模拟场景中的间接散射光：

```
I_ambient = ka · Ia
```

ka 为材质的环境反射系数，Ia 为环境光强度。

### 6.2 漫反射（Lambertian Reflection）

光线向各方向均匀散射，亮度与入射角余弦成正比：

```
I_diffuse = kd · Il · max(0, n · l)
```

- **n** 为表面法线，**l** 为光源方向
- 光线与法线夹角越大，亮度越低

### 6.3 镜面反射（Phong Specular）

模拟光滑表面的高光：

```
I_specular = ks · Il · max(0, r · v)^n
```

- **r** 为反射方向，**v** 为视线方向
- n_shininess 控制高光的集中程度（越大越锐利）

### 6.4 Blinn-Phong 模型

用半程向量 **h** 替代反射向量 **r**，避免计算反射：

```
I_specular = ks · Il · max(0, n · h)^n
```

**h** = normalize(**l** + **v**)。Blinn-Phong 是 OpenGL 早期固定管线的标准光照模型。

## 7. 着色模型

### 7.1 Flat Shading

每个面片使用统一的法线和颜色，面片内部无明暗变化。

- 计算速度快
- 面片边界明显，适合低多边形风格

### 7.2 Gouraud Shading

在顶点处计算光照，面片内部通过插值得到颜色：

1. 计算每个顶点的法线（相邻面法线的平均）
2. 在顶点处计算光照颜色
3. 通过重心坐标插值像素颜色

**缺点**：高光可能被"削平"或出现在错误位置。

### 7.3 Phong Shading

在像素级别计算光照（与 Phong 光照模型不同！）：

1. 插值顶点法线到每个像素
2. 在每个像素处重新计算光照

**优势**：高光效果更真实，是现代 GPU 的标准做法。
**注意**：Phong 着色（插值法线）≠ Phong 光照模型（高光公式）。

## 8. 纹理映射（Texture Mapping）

### 8.1 纹理坐标与映射

纹理坐标 (u, v) ∈ [0, 1]² 将 2D 图像映射到 3D 表面：
- 每个顶点关联一个纹理坐标
- 像素的纹理坐标通过重心插值获得

### 8.2 Mipmap 与三线性插值

**Mipmap**：预生成纹理的多级缩小版本（金字塔结构）

- 解决远处纹理的走样问题
- 每级尺寸减半，共 log₂(N) 级

**三线性插值**：
1. 计算像素对应的纹理级别 D
2. 在 D 和 D+1 级分别做双线性插值
3. 在两级结果之间做线性插值

### 8.3 凹凸贴图与法线贴图

**凹凸贴图（Bump Mapping）**：
- 用高度图扰动表面法线
- 不改变几何，仅改变光照计算
- 轮廓线不变，近看可能失真

**法线贴图（Normal Mapping）**：
- 直接存储扰动后的法线方向
- 在切线空间中定义法线
- 比凹凸贴图更精确，效果更好

## 9. 阴影生成（Shadow Mapping）

### 9.1 算法流程

Shadow Mapping 是实时渲染中最常用的阴影技术：

1. 从光源视角渲染场景，生成深度图（Shadow Map）
2. 从相机视角渲染时，将像素变换到光源空间
3. 比较像素深度与 Shadow Map 中的深度
4. 若像素深度 > Shadow Map 深度，说明被遮挡，处于阴影中

### 9.2 常见问题与解决

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 阴影锯齿 | Shadow Map 分辨率不足 | 级联阴影（CSM）、光源空间 PCF |
| 自阴影瑕疵（Shadow Acne） | 深度精度误差 | 深度偏移（Bias） |
| 悬浮阴影（Peter Panning） | Bias 过大 | 法线相关 Bias |
| 阴影边缘锯齿 | 硬阴影判定 | PCF 软阴影、PCSS |
