用例规约与活动图

一、用例规约（Use Case Specification）

用例规约是用例图的详细文字描述，将图形化的用例转化为可执行的需求规格。它是需求到开发的关键桥梁。

1.1 用例规约模板

字段说明示例
用例名称动词短语，简洁描述功能创建订单
用例编号唯一标识符UC-001
参与者执行用例的角色注册用户
前置条件用例开始前系统必须满足的状态用户已登录；购物车非空
后置条件用例成功结束后系统的状态订单创建成功，库存扣减
触发器启动用例的事件用户点击"提交订单"按钮
基本流程正常执行的步骤序列见1.2示例
替代流程正常流程的分支变体使用优惠券、选择不同支付方式
异常流程出错时的处理步骤支付失败、库存不足
业务规则约束和计算规则满100减10、每人限购3件
非功能需求性能、安全等约束下单接口响应时间<500ms

1.2 用例规约示例：创建订单

UC-001 创建订单

参与者：注册用户
前置条件：用户已登录，购物车中有至少一件商品
触发器：用户点击"提交订单"

基本流程 (Basic Flow)：
1. 系统显示订单确认页面（商品列表、收货地址、支付方式）
2. 用户确认/修改收货地址和支付方式
3. 用户点击"确认支付"
4. 系统验证库存可用性
5. 系统计算订单金额（商品金额 + 运费 - 优惠）
6. 系统创建订单（状态：待支付）
7. 系统调用支付网关发起支付
8. 支付网关返回支付结果
9. 系统更新订单状态为"已支付"
10. 系统扣减库存、清空购物车
11. 系统显示订单成功页面

替代流程：
2a. 用户使用优惠券：系统验证优惠券有效性并计算折扣
7a. 用户取消支付：系统保留订单（状态：待支付），30分钟后自动取消

异常流程：
E1. 库存不足（步骤4）：系统提示"库存不足"，引导用户修改数量
E2. 支付失败（步骤8）：系统提示支付失败原因，允许重新支付
E3. 支付超时（步骤8）：系统查询支付状态，未支付则保留订单等待

1.3 编写要点

  • 使用第三人称和祈使句（"系统显示"而非"你应该看到"）
  • 每步只描述一个动作
  • 区分基本流程（成功路径）和异常流程
  • 替代流程用"Xa"编号，异常流程用"E"编号
  • 业务规则独立列出，不混入流程步骤

二、活动图（Activity Diagram）

活动图是UML的行为图，用于描述业务流程或算法的工作流程，是用例规约的可视化补充。

2.1 活动图元素

元素图形说明
初始节点实心圆流程起始点
终止节点圆圈+实心圆流程结束点
活动/动作圆角矩形一个处理步骤
决策节点菱形条件分支
合并节点菱形分支汇合
分叉粗横线并行开始
汇合粗横线并行结束（同步）
泳道竖向分区区分不同职责/参与者
对象节点矩形流程中传递的数据

2.2 创建订单活动图描述

创建订单流程（泳道图描述）：

[用户泳道] 提交订单 → 确认收货地址 → 选择支付方式 → 确认支付
[系统泳道] 验证库存 → 是否充足? → 是 → 计算金额 → 创建订单 → 调用支付接口
[系统泳道] 是否充足? → 否 → 提示库存不足 → [结束]
[支付网关泳道] 处理支付 → 返回结果
[系统泳道] 收到支付结果 → 支付成功? → 是 → 扣减库存 → 清空购物车 → 显示成功
[系统泳道] 支付成功? → 否 → 提示失败 → 允许重试

并行处理（分叉）：
[分叉] → 扣减库存（并行）→ 更新订单状态（并行）→ 发送通知（并行）→ [汇合]

2.3 活动图与流程图的区别

特性活动图流程图
并行处理支持（分叉/汇合）不支持
泳道支持（职责划分）不支持
对象流支持（数据传递）不支持
信号支持（事件触发）不支持
适用场景复杂业务流程、并发简单算法逻辑

三、序列图在需求中的应用

序列图侧重于对象之间的交互时序，适合描述系统响应事件的协作过程。

3.1 需求级序列图要点

  • 参与者：外部用户或系统，放在图两侧
  • 生命线：代表参与交互的对象/组件
  • 消息：同步调用（实线箭头）、异步（开放箭头）、返回（虚线箭头）
  • 组合片段：alt(条件)、loop(循环)、par(并行)、opt(可选)

3.2 创建订单序列图描述

序列图描述（文字版）：

用户 → 前端：点击"提交订单"
前端 → API网关：POST /api/orders
API网关 → 订单服务：createOrder(cartId, addressId, paymentMethod)
订单服务 → 库存服务：checkStock(items)
库存服务 → 订单服务：stockAvailable=true
订单服务 → 优惠服务：calculateDiscount(order, couponCode)
优惠服务 → 订单服务：discountAmount=20.00
订单服务 → 订单服务：createOrderRecord()
订单服务 → 支付服务：initiatePayment(orderId, amount)
支付服务 → 外部网关：发起支付请求
外部网关 → 支付服务：支付成功回调
支付服务 → 订单服务：paymentSuccess(orderId)
订单服务 → 库存服务：deductStock(items)
订单服务 → 通知服务：sendOrderConfirmation(userId, orderId)
订单服务 → API网关：orderCreated(orderId)
API网关 → 前端：200 OK
前端 → 用户：显示订单成功页面
文档工具选择：用例规约用文字模板编写（清晰准确），活动图用PlantUML或draw.io绘制（展示流程分支），序列图用PlantUML绘制（展示交互时序）。三者互补，共同构成完整的需求规格文档。
