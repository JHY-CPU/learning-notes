# 51-需求工程

需求工程是软件工程的核心过程之一，关注于确定系统应该做什么以及如何约束系统行为。

## 目录

### [01-需求获取与分析](01-需求获取与分析/README.md)
需求工程概述、功能需求与非功能需求、需求获取技术（访谈、问卷、观察、原型法、头脑风暴、JAD）、利益相关者分析、用例与用例图、用户故事与INVEST原则、需求分析方法、DFD、ER图、需求优先级排序（MoSCoW）。

### [02-需求规格说明](02-需求规格说明/README.md)
SRS结构与内容、IEEE 830标准、形式化vs非形式化描述、原型的作用、场景描述、需求跟踪矩阵（RTM）。

### [03-需求验证与管理](03-需求验证与管理/README.md)
需求验证方法（评审、原型、测试用例设计）、确认vs验证、需求变更管理、版本控制、需求跟踪、需求工具（DOORS、Confluence、Jira）。
