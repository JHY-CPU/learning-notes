02 - 图的遍历

离散数学 | 图论

1. 图的遍历概述

什么是图的遍历？

图的遍历（Graph Traversal）是从图中的某个顶点出发，按照某种策略访问图中所有顶点恰好一次的过程。

两种基本策略：

  • BFS（广度优先搜索）：Breadth-First Search — 按层遍历，先近后远
  • DFS（深度优先搜索）：Depth-First Search — 一路深入，回溯探索

G = (V, E)，V = {A, B, C, D, E, F, G}，从 A 出发遍历

3. 广度优先搜索（BFS）

BFS 算法思想

从起始顶点出发，先访问所有距离为1的顶点，再访问距离为2的顶点，以此类推 — "逐层扩展"。

数据结构：队列（Queue）— 先进先出

特点：

  • 找到的路径是最短路径（无权图中）
  • 适合求最短路径、层序遍历
  • 空间复杂度较高（需存储整层顶点）

BFS 伪代码

BFS(G, start):
    创建队列 Q
    将 start 加入 Q
    标记 start 为已访问

    while Q 不为空:
        v = Q.dequeue()        // 取出队首
        访问 v
        for v 的每个邻居 w:
            if w 未访问:
                标记 w 为已访问
                Q.enqueue(w)   // 加入队尾

3.1 BFS 执行过程（从 A 出发）

步骤当前访问队列状态已访问集合操作说明
初始化-[A]{}A 入队
1A[B, D]{A}访问A，邻居B、D入队
2B[D, E, C]{A, B}访问B，邻居E、C入队
3D[E, C, G]{A, B, D}访问D，邻居G入队
4E[C, G]{A, B, D, E}访问E，无新邻居
5C[G, F]{A, B, D, E, C}访问C，邻居F入队
6G[F]{A, B, D, E, C, G}访问G，无新邻居
7F[]（空）{A, B, D, E, C, G, F}访问F，队列空，结束

BFS 序列（层序）：A → B → D → E → C → G → F

BFS 树（按层）：第1层: {A}，第2层: {B, D}，第3层: {E, C, G}，第4层: {F}

4. 深度优先搜索（DFS）

DFS 算法思想

从起始顶点出发，沿着一条路径尽可能深入，直到无法继续时回溯到上一个分叉点，继续探索其他路径。

数据结构：栈（Stack）或递归调用栈 — 后进先出

特点：

  • 适合判断连通性、检测环、拓扑排序
  • 空间复杂度较低（只需存储当前路径）
  • 找到的路径不一定是最短的

DFS 伪代码

DFS(G, v):
    标记 v 为已访问
    访问 v

    for v 的每个邻居 w:
        if w 未访问:
            DFS(G, w)          // 递归深入

4.1 DFS 执行过程（从 A 出发）

步骤当前访问递归栈（底→顶）已访问集合操作说明
1A[A]{A}从A出发，探索邻居B
2B[A, B]{A, B}从B出发，探索邻居E
3E[A, B, E]{A, B, E}E无未访问邻居，回溯到B
4C[A, B, C]{A, B, E, C}从B继续，探索C
5F[A, B, C, F]{A, B, E, C, F}从C出发，探索F
6回溯[A, B, C] → [A, B] → [A]{A, B, E, C, F}F/C/B无未访问邻居，回溯到A
7D[A, D]{A, B, E, C, F, D}从A继续，探索D
8G[A, D, G]{A, B, E, C, F, D, G}从D出发，探索G

DFS 序列：A → B → E → C → F → D → G

注意：DFS序列不唯一，取决于邻居的访问顺序。上面按字母序探索。

5. BFS vs DFS 对比

特性BFS（广度优先）DFS（深度优先）
数据结构队列（Queue）栈（Stack）/ 递归
搜索方式逐层扩展一路深入再回溯
最短路径保证（无权图）不保证
空间复杂度O(bd) — 较大O(b·m) — 较小
时间复杂度O(V + E)O(V + E)
适用场景最短路径、层序连通性、环检测、拓扑排序
完备性完备（能找到解）可能陷入深路径
