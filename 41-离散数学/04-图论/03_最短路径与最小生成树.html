2. Dijkstra 最短路径算法

算法思想

Dijkstra 算法用于求解单源最短路径：从一个源点到图中所有其他顶点的最短路径。

贪心策略：每次从未确定最短路径的顶点中，选择距离最小的顶点，更新其邻居的距离。

要求：所有边的权重必须非负。

时间复杂度：O(V²)（朴素实现）或 O((V+E)logV)（优先队列实现）

Dijkstra 算法步骤

Dijkstra(G, source):
    初始化 dist[source] = 0，其余 dist[v] = ∞
    初始化 visited = {}（未确定最短路径的集合）

    重复 |V| 次：
        1. 从未访问顶点中选 dist 最小的顶点 u
        2. 将 u 标记为已访问
        3. 对 u 的每个邻居 v：
             如果 dist[u] + w(u,v) < dist[v]：
                 dist[v] = dist[u] + w(u,v)  // 松弛操作
                 prev[v] = u                   // 记录前驱

3. 生成树（Spanning Tree）

定义

连通图 G 的生成树是 G 的一个子图，它包含 G 的所有顶点，且是一棵树（连通无环）。

  • n 个顶点的生成树恰好有 n-1 条边
  • 添加任意一条非树边会形成一个环
  • 删除任意一条树边会使图不连通

4. 最小生成树（Minimum Spanning Tree, MST）

定义

加权连通图的最小生成树是所有生成树中边权之和最小的那个。

MST 的边权之和 = 所有生成树中最小的总权重

应用：通信网络设计、管道铺设、电路布线等需要最小代价连接所有节点的场景。

4.1 Prim 算法

Prim 算法思想

从任意顶点开始，每次选择连接树内和树外的最小权边，将新顶点加入树中。

类似 Dijkstra：贪心地扩展，但 Prim 关注的是"连接树"而非"路径长度"。

时间复杂度：O(V²) 或 O(E log V)

步骤已选顶点候选边（最小的标*）添加的边累计权重

MST 边：A-B, B-E, E-D, E-G, D-F, A-C

4.2 Kruskal 算法

Kruskal 算法思想

将所有边按权重从小到大排序，依次选择最小的边加入 MST，只要不会形成环。

数据结构：并查集（Union-Find）用于高效检测环

时间复杂度：O(E log E)

执行步骤

排序后边权重是否加入MST原因
1D-E1是不形成环
2B-E2是不形成环
3C-F3是不形成环
4A-B4是不形成环
5E-G5是不形成环
6D-F6否D-E-B-A...与C-F不连通 → 是（纠正：实际不形成环）

更正：排序后边为 E-D(1), B-E(2), C-F(3), A-B(4), E-G(5), D-F(6), B-D(7), D-G(8)... 依次加入前6条边恰好连接所有7个顶点。

MST 权重 = 1+2+3+4+5+6 = 21（与 Prim 结果一致）

5. Prim vs Kruskal 对比

特性Prim 算法Kruskal 算法
策略从顶点出发扩展全局最小边优先
数据结构优先队列/数组并查集（Union-Find）
时间复杂度O(V²) 或 O(E log V)O(E log E)
适合场景稠密图（边多）稀疏图（边少）
结果MST 权重相同（可能选不同边）

6. 相关概念补充

关键术语

  • 路径（Path）：顶点序列 v₁, v₂, ..., vₖ，相邻顶点之间有边连接
  • 简单路径：路径上的顶点互不相同
  • 环（Cycle）：起点和终点相同的路径
  • 连通图：任意两个顶点之间都有路径
  • 树：连通无环的无向图 | n个顶点，n-1条边
  • 最短路径：两顶点间所有路径中权值之和最小的路径

算法适用总结

问题算法说明
单源最短路径（非负权）Dijkstra贪心，O(V²)
所有顶点对最短路径Floyd-Warshall动态规划，O(V³)

有负权边的最短路径Bellman-Ford可检测负权环
