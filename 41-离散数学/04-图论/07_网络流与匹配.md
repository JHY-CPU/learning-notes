==================== 1. 网络流基础 ====================

## 1. 网络流基础


### 1.1 基本定义


> **Note:** **流网络 (Flow Network)：**
> 有向图 G=(V,E)，其中：
>
> - 每条边 (u,v) 有非负
>    **容量**
>    c(u,v) ≥ 0
> - 有且仅有一个
>    **源点**
>    s 和一个
>    **汇点**
>    t
> - 不存在反向边（若 (u,v) ∈ E 则 (v,u) ∉ E）


> **Note:** **可行流 (Flow)：**
> 函数 f: V×V → ℝ 满足：
>
> - **容量限制：**
>    0 ≤ f(u,v) ≤ c(u,v) 对所有边
> - **流量守恒：**
>    对所有非 s,t 的顶点 v：∑f(u,v) = ∑f(v,w)
>
> 流的值：
> `|f| = ∑f(s,v) - ∑f(v,s)`
> （流出源点的净流量）

==================== 2. 最大流最小割定理 ====================

## 2. 最大流最小割定理 (Max-Flow Min-Cut)


### 2.1 割的定义


> **Note:** **s-t 割 (Cut)：**
> 将顶点集 V 划分为 S 和 T=V\S，使得 s ∈ S，t ∈ T。
>
>
> **割的容量：**
> `c(S,T) = ∑ c(u,v)`
> ，其中 u ∈ S, v ∈ T。
>
>
> **最小割：**
> 容量最小的 s-t 割。


### 2.2 核心定理


> **Theorem:** **最大流最小割定理 (Ford & Fulkerson, 1956)：**
>
>
> 网络中最大流的值 = 最小割的容量：
>
> $$
> max |f| = min c(S,T)
> $$
>
> 这是图论中最重要的对偶性结果之一。

==================== 3. Ford-Fulkerson 算法 ====================

## 3. Ford-Fulkerson 算法


### 3.1 增广路径


> **Note:** **残余网络 (Residual Network)：**
> 对当前流 f，定义残余容量：
>
>
> c_f(u,v) = c(u,v) - f(u,v) （正向边）
>
>
> c_f(v,u) = f(u,v) （反向边，允许"退流"）
>
>
>
>
> **增广路径 (Augmenting Path)：**
> 残余网络中从 s 到 t 的路径。沿增广路径增加流量可增大总流。


### 3.2 算法步骤


> **Note:** 1. 初始化所有边流量 f = 0
> 2. **循环：**
>    在残余网络中寻找 s→t 的增广路径
> 3. 若找到：沿路径增加 min{c_f(e)} 的流量
> 4. 若找不到：当前流即为最大流，算法结束
>
> **BFS 找增广路径**
> 时称为
> **Edmonds-Karp 算法**
> ，时间复杂度 O(VE²)。


> **Example:** 上图的最大流计算：
>
>
> 路径1: s→A→C→t, 流量 = min(10,10,10) = 10
>
>
> 路径2: s→B→D→t, 流量 = min(8,10,8) = 8
>
>
> 路径3: s→A→B→D→t, 流量 = min(0,2,...) 无残余
>
>
> 检查残余网络，发现 s→A 无剩余容量，s→B→D→t 也饱和。
>
>
> 最大流 = 10 + 8 = 18，最小割 = {s} 与其余的割 = 18。


### 3.3 整数流定理


> **Theorem:** 若所有容量为整数，则存在一个所有流量为整数的最大流。
>
>
> Ford-Fulkerson 算法在整数容量下一定在有限步内终止。

==================== 4. 二分图匹配 ====================

## 4. 二分图匹配 (Bipartite Matching)


### 4.1 基本定义


> **Note:** **匹配 (Matching)：**
> 图中没有公共顶点的边的集合。
>
>
> **最大匹配：**
> 边数最多的匹配。
>
>
> **完美匹配 (Perfect Matching)：**
> 覆盖所有顶点的匹配。
>
>
> **二分图：**
> 顶点可分为两个不相交集合 X, Y，所有边连接 X 和 Y 中的顶点。

### 4.2 用网络流求最大匹配


> **Note:** 将二分图匹配转化为最大流问题：
>
> 1. 添加超级源点 s，连向 X 中所有顶点（容量1）
> 2. 添加超级汇点 t，Y 中所有顶点连向 t（容量1）
> 3. 原图中 X-Y 的边容量设为 1（或无穷）
> 4. 求 s-t 最大流 = 最大匹配


### 4.3 匈牙利算法


> **Note:** 通过寻找
> **增广路径**
> （交替经过非匹配边和匹配边的路径）来增大匹配。
>
>
> 时间复杂度：O(VE)。等价于 Ford-Fulkerson 在二分图上的应用。

==================== 5. Hall 定理 ====================

## 5. Hall 婚姻定理


> **Theorem:** **Hall 定理 (1935)：**
> 二分图 G=(X,Y,E) 中存在覆盖 X 的匹配（即 |M|=|X|）
> **当且仅当**
>
>
> 对 X 的任意子集 S，其邻居集合 N(S) 满足：
>
> $$
> |N(S)| ≥ |S|    对所有 S ⊆ X
> $$
>
> 这个条件称为
> **Hall 条件**
> 。


> **Example:** **应用：**
> 有 n 个男生和 n 个女生，每个男生认识若干女生。
>
>
> 问：能否给每个男生配对一个他认识的女生？
>
>
> 定理回答：当且仅当任意 k 个男生认识的女生总数 ≥ k。


### 5.1 推论


| 推论 | 内容 |
| --- | --- |
| k-正则二分图 | k-正则二分图一定有完美匹配（每个顶点度数=k） |
| 婚姻定理 | \|X\|=\|Y\| 时，存在完美匹配 ⟺ Hall 条件成立 |
| 覆盖定理 | 最大匹配大小 = 最小顶点覆盖大小 (König定理，仅限二分图) |


### 5.2 König-Egerváry 定理


> **Theorem:** 在二分图中：
>
> $$
> 最大匹配大小 = 最小顶点覆盖大小
> $$
>
> 这是最大流最小割定理在二分图上的特例。

==================== 6. 最大匹配与最小覆盖 ====================

## 6. 相关概念对比


| 概念 | 定义 | 对偶概念 |
| --- | --- | --- |
| 最大流 | s-t 流量最大 | 最小割 |
| 最大匹配 | 边数最多的匹配 | 最小顶点覆盖 (二分图) |
| 最大独立集 | 不相邻顶点最多的集合 | 最小顶点覆盖 |


> **Warning:** **注意：**
> 一般图中最大匹配 ≠ 最小顶点覆盖。König 定理仅适用于二分图。
>
>
> 一般图的最大匹配可用 Edmonds 的"花算法"求解。

==================== 总结 ====================

> **Summary:** ## 本节总结
>
>
> - **流网络：**
>    有向图 + 容量 + 源汇，可行流满足容量限制和流量守恒
> - **最大流最小割定理：**
>    max|f| = min c(S,T)，网络流的对偶性
> - **Ford-Fulkerson：**
>    通过增广路径迭代增大流；BFS 版本为 Edmonds-Karp O(VE²)
> - **残余网络：**
>    允许退流（反向边），是寻找增广路径的关键
> - **二分图匹配：**
>    可转化为网络流；匈牙利算法 O(VE)
> - **Hall 定理：**
>    |N(S)|≥|S| 对所有 S⊆X ⟺ 存在覆盖X的匹配
> - **König 定理：**
>    二分图中最大匹配 = 最小顶点覆盖

## Python实现

### 最大流 (Edmonds-Karp 算法)

```python
from collections import deque, defaultdict

def max_flow(graph, source, sink):
    """Edmonds-Karp 最大流算法 (BFS)"""
    # 构建残余容量
    capacity = defaultdict(lambda: defaultdict(int))
    for u in graph:
        for v, c in graph[u]:
            capacity[u][v] += c

    def bfs_path():
        """BFS 找增广路径"""
        visited = {source}
        queue = deque([(source, [source])])
        while queue:
            node, path = queue.popleft()
            for neighbor in capacity[node]:
                if neighbor not in visited and capacity[node][neighbor] > 0:
                    if neighbor == sink:
                        return path + [neighbor]
                    visited.add(neighbor)
                    queue.append((neighbor, path + [neighbor]))
        return None

    total_flow = 0
    while True:
        path = bfs_path()
        if path is None:
            break
        # 找路径上的最小残余容量
        flow = min(capacity[path[i]][path[i + 1]] for i in range(len(path) - 1))
        # 更新残余网络
        for i in range(len(path) - 1):
            capacity[path[i]][path[i + 1]] -= flow
            capacity[path[i + 1]][path[i]] += flow
        total_flow += flow

    return total_flow

# 示例网络
graph_flow = {
    's': [('A', 10), ('B', 8)],
    'A': [('C', 10), ('B', 2)],
    'B': [('D', 10)],
    'C': [('t', 10)],
    'D': [('C', 6), ('t', 8)]
}

print("=== 最大流 ===")
max_f = max_flow(graph_flow, 's', 't')
print(f"最大流值: {max_f}")
print(f"最大流最小割定理: max|f| = min c(S,T) = {max_f}")
```

### 二分图匹配 (匈牙利算法)

```python
def bipartite_match(graph, left, right):
    """二分图最大匹配 (匈牙利算法)"""
    match_R = {}  # right -> left
    result = 0

    def dfs(u, visited):
        for v in graph[u]:
            if v not in visited:
                visited.add(v)
                if v not in match_R or dfs(match_R[v], visited):
                    match_R[v] = u
                    return True
        return False

    for u in left:
        visited = set()
        if dfs(u, visited):
            result += 1

    return result, match_R

# 示例: 工人-任务匹配
graph_bi = {
    'w1': ['t1', 't2'],
    'w2': ['t2', 't3'],
    'w3': ['t1', 't3'],
    'w4': ['t2']
}
left = ['w1', 'w2', 'w3', 'w4']
right = ['t1', 't2', 't3']

print("=== 二分图匹配 ===")
size, matching = bipartite_match(graph_bi, left, right)
print(f"最大匹配: {size}")
print(f"匹配方案: {dict((v, k) for k, v in matching.items())}")

# 验证 Hall 条件
print("\n=== Hall 条件验证 ===")
for S_size in range(1, len(left) + 1):
    from itertools import combinations
    for S in combinations(left, S_size):
        neighbors = set()
        for u in S:
            neighbors.update(graph_bi[u])
        if len(neighbors) < len(S):
            print(f"Hall 条件不满足: S={S}, |N(S)|={len(neighbors)} < |S|={len(S)}")
            break
    else:
        continue
    break
else:
    print(f"Hall 条件满足: 所有子集 S 都有 |N(S)| ≥ |S|")
    print(f"存在完美匹配条件: 最大匹配={size}, |X|={len(left)}")
```
