欧拉路径与哈密顿路径

离散数学 · 图论 · 第5节

1. 欧拉路径与欧拉回路

1.1 基本定义

欧拉路径 (Eulerian Path)：经过图中每条边恰好一次的路径。
欧拉回路 (Eulerian Circuit)：起点和终点相同的欧拉路径。
欧拉图：含有欧拉回路的图。

历史背景：源于 1736 年欧拉解决的"哥尼斯堡七桥问题"。

1.2 欧拉定理 (度数条件)

对于无向连通图 G：
  • G 有欧拉回路 ⟺ 所有顶点度数为偶数
  • G 有欧拉路径（非回路）⟺ 恰好 2 个顶点度数为奇数，其余为偶数
对于有向连通图 G：
  • G 有有向欧拉回路 ⟺ 每个顶点入度 = 出度
  • G 有有向欧拉路径 ⟺ 恰好一个顶点出度-入度=1，一个入度-出度=1，其余相等

1.3 Fleury 算法 (构造欧拉回路)

算法步骤：
  1. 从任意奇度顶点出发（若无奇度顶点则任意出发）
  2. 沿未访问的边行走，除非没有其他选择，不走桥（割边）
  3. 直到所有边都被访问
时间复杂度：O(E²)。Hierholzer 算法可优化到 O(E)。

2. 哈密顿路径与哈密顿回路

2.1 基本定义

哈密顿路径 (Hamiltonian Path)：经过图中每个顶点恰好一次的路径。
哈密顿回路 (Hamiltonian Circuit)：起点和终点相同的哈密顿路径。
哈密顿图：含有哈密顿回路的图。
核心区别：
  • 欧拉路径：遍历所有边（度数条件可判定）
  • 哈密顿路径：遍历所有顶点（无简洁充要条件，是 NP-完全问题）

2.2 充分条件

Dirac 定理：若简单图 G 有 n≥3 个顶点，且每个顶点度数 ≥ n/2，则 G 有哈密顿回路。
Ore 定理：若简单图 G 有 n≥3 个顶点，且对每对不相邻顶点 u,v 有 deg(u)+deg(v) ≥ n，则 G 有哈密顿回路。

3. 旅行商问题 (TSP)

3.1 问题描述

给定 n 个城市和城市间的距离（或费用），找到访问每个城市恰好一次并回到起点的最短回路。

TSP 的解空间大小 = (n-1)! / 2 （对称情况）

3.2 复杂度与求解方法

方法类型时间复杂度近似比
暴力枚举精确O(n!)最优
动态规划 (Held-Karp)精确O(n²·2ⁿ)最优
分支限界精确指数级（实践较快）最优
最近邻法贪心近似O(n²)无保证
Christofides近似O(n³)≤1.5倍最优
模拟退火/遗传算法启发式视参数通常较好
4 个城市的 TSP：
距离矩阵 D = [[0,10,15,20],[10,0,35,25],[15,35,0,30],[20,25,30,0]]
枚举 (4-1)!/2 = 3 条不同回路：
A→B→C→D→A: 10+35+30+20 = 95
A→B→D→C→A: 10+25+30+15 = 80 ✓ 最优
A→C→B→D→A: 15+35+25+20 = 95

3.3 TSP 与哈密顿回路的关系

哈密顿回路问题 = TSP 中所有边权为 1 的特殊情况。
由于哈密顿回路是 NP-完全的，TSP 至少也是 NP-难的。
TSP 的优化版本不在 NP 中（验证最优解需要更长时间），但判定版本（"是否存在费用 ≤ k 的回路"）是 NP-完全的。

本节总结

  • 欧拉回路存在条件：无向连通图所有顶点度数为偶数
  • 欧拉路径存在条件：恰好 2 个奇度顶点
  • 有向欧拉回路：每个顶点入度 = 出度
  • Fleury 算法：避免走桥，O(E²)；Hierholzer 算法 O(E)
  • 哈密顿路径：遍历每个顶点恰好一次，NP-完全
  • Dirac 定理：δ(G) ≥ n/2 → 哈密顿
  • Ore 定理：不相邻顶点度数和 ≥ n → 哈密顿
  • TSP：NP-难，精确算法 O(n²·2ⁿ) (Held-Karp)，Christofides 近似比 1.5
