# 视频理解

*3D CNN、双流网络、SlowFast与Video Transformer*


**视频理解**是对视频数据进行分析和理解的任务，核心挑战是如何有效建模视频中的**空间信息（每帧的内容）**和**时间信息（帧间的变化）**。


| 任务 | 说明 | 典型数据集 |
| --- | --- | --- |
| 动作识别 | 分类视频中的动作类别 | Kinetics, UCF-101, Something-Something |
| 时序动作定位 | 检测动作的起止时间 | ActivityNet, THUMOS |
| 时空动作检测 | 在视频中检测并定位动作 | AVA, JHMDB |
| 视频目标分割 | 跟踪并分割视频中的目标 | DAVIS, YouTube-VOS |
| 视频描述 | 生成视频内容的文字描述 | MSVD, MSR-VTT |


### 2.1 C3D


C3D（2015）直接将2D卷积扩展为3D卷积，同时捕获空间和时间特征：


```
# 2D Conv: 卷积核在空间维度上滑动
# 3D Conv: 卷积核在空间+时间维度上滑动

# 2D Conv:  k_h × k_w × C_in → 2D特征图
# 3D Conv:  k_t × k_h × k_w × C_in → 3D特征图（含时间维度）

# C3D架构: 所有卷积核为3×3×3, 池化核为2×2×2（第一层1×2×2）
# 8层3D卷积 + 2层FC
# 输入: 16帧 × 112×112
```


### 2.2 I3D（Inflated 3D）


核心思想：**膨胀（Inflate）**预训练的2D CNN为3D CNN——将所有2D卷积核沿时间维度复制N倍。


- 将ImageNet预训练的Inception/ResNet膨胀为3D版本
- 保留了预训练的空间特征提取能力
- 时间维度的权重初始化为2D权重的均值/N
- 在Kinetics数据集上显著优于C3D


### 3.1 Two-Stream ConvNet（2014）


Simonyan等人提出用两个独立的CNN分支分别处理空间和时间信息：

RGB帧
(空间流)
→
空间CNN
(2D CNN)
→
融合
←
时间CNN
(2D CNN)
←
光流帧
(时间流)

### 3.2 光流（Optical Flow）


光流描述了相邻帧之间像素的运动信息：


- **密集光流**
   ：每个像素都有运动向量（dx, dy）
- 堆叠连续多帧的光流作为时间流CNN的输入
- 常用TV-L1或FlowNet计算光流


```
# 光流输入格式
# 堆叠10帧的水平和垂直方向光流
# 输入: H × W × 20 (10帧 × 2通道)

# 融合方式
# 1. 平均融合: p = (p_spatial + p_temporal) / 2
# 2. SVM融合: 训练SVM融合两流的softmax分数
# 3. 加权平均: p = α·p_spatial + (1-α)·p_temporal
```


### 3.3 TSN（Temporal Segment Networks）


为解决长视频的时间建模问题，TSN将视频均匀分为K段，每段随机采样一帧：


- 对每段的snippet分别提取特征
- 段共识函数（Segmental Consensus）聚合所有段的预测
- 训练时使用跨模态预训练和数据增强
- 大幅提升了双流网络在长视频上的表现


### 4.1 核心思想


SlowFast（2019，何恺明）受人类视觉系统启发，用两个分支分别处理**空间语义**和**时间运动**：


| 分支 | 帧率 | 通道数 | 功能 |
| --- | --- | --- | --- |
| Slow Pathway | 低帧率（4fps） | 多（β倍） | 捕获空间语义和外观 |
| Fast Pathway | 高帧率（32fps） | 少（1/β，通常1/8） | 捕获快速运动变化 |


### 4.2 架构细节


```
# Slow路径: 采样T帧（如T=4），通道数C
# Fast路径: 采样 αT 帧（如α=8, 4×8=32帧），通道数 C/β

# 侧连接（Lateral Connection）: 将Fast特征融合到Slow
# 通过 1×1×5 卷积将Fast的T×C/β特征转换为T×2C
# 然后拼接到Slow路径中

# Fast路径:
# - 通道数少（高时间分辨率、低空间-语义信息）
# - 不使用时间下采样（保持高帧率）
# - 轻量级，仅占总计算量的~20%
```


> **Note:** **设计哲学：**
> 视频中大部分区域是静止的（背景），只有少量区域有运动。Slow路径用大量通道学习"什么"（what），Fast路径用少量通道学习"如何变化"（how），两者互补。


### 4.3 性能


- Kinetics-400: 79.8% top-1 accuracy（SlowFast R101+NL）
- 比同计算量的I3D高出约4-5%
- 加入Non-local模块后进一步提升


### 5.1 ViViT


将Vision Transformer扩展到视频领域，将视频分解为时空patch token：


```
# 视频 patch 嵌入
# 输入: T × H × W × 3
# 将每帧分为 P×P 的patch
# 每个patch线性嵌入为 D 维token
# 总共 (T×H/P×W/P) 个token

# 因式分解注意力（Factorized Attention）:
# 1. Temporal Attention: 同一空间位置的token之间
# 2. Spatial Attention: 同一时间帧的token之间
# 计算量从 O((THW/P²)²) 降到 O(T²HW/P² + TH²W²/P⁴)
```


### 5.2 TimeSformer


提出**分治注意力（Divided Space-Time Attention）**：


- 将自注意力分解为空间注意力和时间注意力
- 每个token关注同一帧的其他token（空间）+ 同一位置的其他帧（时间）
- 计算效率远高于全时空注意力


### 5.3 Video Swin Transformer


将Swin Transformer扩展到3D，使用3D窗口注意力和移位：


- 3D窗口：时间×高度×宽度的局部窗口
- 移位机制：沿时间维度也进行移位，实现跨窗口连接
- 层次化结构：逐层合并时空token


### 5.4 Video MAE（Masked Autoencoder）


自监督视频预训练方法：


- 随机遮挡视频中大部分patch（如90%~95%）
- 编码器只处理可见patch
- 解码器重建被遮挡的patch
- 视频的时间冗余使遮挡率可以比图像MAE更高
- 预训练后在下游任务上微调，效果显著


### 6.1 任务定义


视频目标跟踪（Visual Object Tracking, VOT）在给定第一帧目标位置的情况下，持续追踪该目标在后续帧中的位置。


| 类型 | 说明 | 代表方法 |
| --- | --- | --- |
| 单目标跟踪（SOT） | 跟踪一个指定目标 | SiamFC, SiamR-CNN, TransT |
| 多目标跟踪（MOT） | 同时跟踪多个目标 | SORT, DeepSORT, ByteTrack |
| 视频目标分割（VOS） | 像素级跟踪目标mask | STM, AOT, SAM 2 |


### 6.2 经典方法


- **相关滤波（KCF, MOSSE）**
   ：在频域高效计算目标与搜索区域的相关性
- **Siamese网络（SiamFC）**
   ：孪生网络学习目标模板与搜索区域的相似度
- **Transformer跟踪（TransT）**
   ：用交叉注意力替代相关性计算


### 6.3 多目标跟踪流程


```
# 多目标跟踪基本流程（以DeepSORT为例）
1. 目标检测: 每帧用检测器（如YOLO）检测所有目标
2. 特征提取: 对每个检测框提取外观特征（ReID网络）
3. 数据关联: 用卡尔曼滤波预测轨迹 + 匈牙利算法匹配检测与轨迹
4. 轨迹管理: 处理新出现、消失、遮挡的目标

# ByteTrack改进: 不直接丢弃低置信度检测，分两阶段匹配
# 第一阶段: 高置信度检测与轨迹匹配
# 第二阶段: 低置信度检测与未匹配轨迹匹配
```
