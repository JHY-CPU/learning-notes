# 图像预处理与增强

*归一化、数据增强策略、迁移学习与微调方法全面解析*


### 1.1 为什么需要预处理


原始图像数据存在像素值范围不一致、光照变化、尺度差异等问题。预处理的目的是**消除数据分布差异**，使模型更容易学习到有效的特征表示。


- 统一数据分布，加速梯度下降收敛
- 避免某些特征因数值范围大而主导训练
- 提高模型的数值稳定性和泛化能力


### 1.2 像素值归一化方法


| 方法 | 公式 | 适用场景 |
| --- | --- | --- |
| Min-Max归一化 | x' = (x - min) / (max - min) | 像素值缩放到[0,1] |
| 均值-标准差标准化 | x' = (x - μ) / σ | 零均值化，最常用 |
| 逐通道标准化 | x'_c = (x_c - μ_c) / σ_c | ImageNet预训练模型 |
| Per-image标准化 | 每张图独立计算μ, σ | 风格迁移等场景 |


### 1.3 ImageNet标准均值和标准差


```
# ImageNet数据集的均值和标准差（RGB三通道）
mean = [0.485, 0.456, 0.406]   # R, G, B
std  = [0.229, 0.224, 0.225]   # R, G, B

# PyTorch中的标准化
transform = transforms.Compose([
    transforms.ToTensor(),                    # [0,255] → [0,1]
    transforms.Normalize(mean, std),          # 标准化
])

# 反归一化（可视化时使用）
def denormalize(tensor, mean, std):
    for t, m, s in zip(tensor, mean, std):
        t.mul_(s).add_(m)
    return tensor.clamp(0, 1)
```


> **Note:** **实践建议：**
> 如果使用预训练模型，务必使用该模型训练时的均值和标准差。对于自定义数据集，可以计算数据集自身的统计量，或直接沿用ImageNet的值。

### 2.1 数据增强的意义


数据增强通过对训练图像施加各种变换来**人工扩大训练集**，提高模型的泛化能力，减少过拟合。好的增强策略相当于对数据分布的先验知识编码。


- 增加训练样本的多样性
- 使模型对光照、角度、遮挡等变化具有鲁棒性
- 隐式地起到正则化作用
- 尤其在数据量有限时效果显著


### 2.2 基础几何变换


#### 水平翻转（Horizontal Flip）


沿垂直轴翻转图像。适用于大多数场景（自然图像、航拍等），不适用于文字、医学影像等方向敏感的数据。


```
torchvision: transforms.RandomHorizontalFlip(p=0.5)
```


#### 垂直翻转（Vertical Flip）


沿水平轴翻转。适用于卫星图像、显微镜图像等上下对称性不敏感的场景。


```
torchvision: transforms.RandomVerticalFlip(p=0.5)
```


#### 随机旋转（Random Rotation）


在指定角度范围内随机旋转。常用范围：-15°~15°或-30°~30°。旋转后的空白区域通常用0或边缘像素填充。


```
transforms.RandomRotation(degrees=15, fill=0)
```


#### 随机裁剪（Random Crop）


从原图中随机裁剪子区域，迫使模型关注局部特征。训练时使用RandomCrop，推理时使用CenterCrop。


```
transforms.RandomCrop(224, padding=4)
```


#### 随机缩放（Random Resize）


先将图像缩放到不同大小，再裁剪到目标尺寸。模拟不同距离拍摄的图像。


```
transforms.RandomResizedCrop(224, scale=(0.08, 1.0))
```


#### 仿射变换（Affine）


包括旋转、平移、缩放、剪切的组合变换。提供更丰富的几何变化。


```
transforms.RandomAffine(degrees=15, translate=(0.1,0.1),
    scale=(0.9, 1.1), shear=10)
```

### 2.4 高级增强策略


### Mixup


将两张图及其标签按比例混合，生成"软标签"样本。有效防止过拟合，提高模型校准度。


$$
x̃ = λxᵢ + (1-λ)xⱼ,　ỹ = λyᵢ + (1-λ)yⱼ,　其中 λ ~ Beta(α, α)
$$


```
# PyTorch实现
def mixup(data, targets, alpha=0.2):
    indices = torch.randperm(data.size(0))
    lam = np.random.beta(alpha, alpha)
    mixed_data = lam * data + (1 - lam) * data[indices]
    return mixed_data, targets, targets[indices], lam
```


### CutMix


将一张图的矩形区域替换为另一张图的对应区域，标签按面积比例混合。比Mixup保留更多局部结构信息。


$$
x̃ = M⊙xᵢ + (1-M)⊙xⱼ,　ỹ = (Area(M)/Total)·yᵢ + (1-Area(M)/Total)·yⱼ
$$


### CutOut


在图像上随机遮挡一个矩形区域（填充为0或均值），迫使模型关注多个区域而非单一判别特征。


### AutoAugment / RandAugment


- **AutoAugment**
   ：用强化学习搜索最优增强策略组合
- **RandAugment**
   ：从预定义的增强操作集中随机选择N个操作，每个操作强度为M。超参数简单（N, M），效果接近AutoAugment


```
# RandAugment使用
transforms.RandAugment(num_ops=2, magnitude=9)
```


### 2.5 增强策略对比总结


| 策略 | 原理 | 优势 | 注意事项 |
| --- | --- | --- | --- |
| Mixup | 像素级线性混合 | 平滑决策边界 | 混合后的图视觉上不自然 |
| CutMix | 区域替换+标签混合 | 保留局部结构 | 遮挡区域大小需调参 |
| CutOut | 随机遮挡 | 简单有效 | 标签未更新，可能引入噪声 |
| Mosaic | 4张图拼接 | YOLO系列标配 | 需配合对应检测框架 |
| RandAugment | 随机策略搜索 | 超参数少 | 操作集需精心设计 |


### 3.1 什么是迁移学习


迁移学习将在**大规模数据集（如ImageNet）**上预训练的模型知识，迁移到**目标任务**上。这在目标数据集较小时尤其有效——预训练模型已经学会了通用的视觉特征（边缘、纹理、形状等）。


### 3.2 迁移学习的策略


| 策略 | 方法 | 适用场景 |
| --- | --- | --- |
| 特征提取 | 冻结预训练模型，只训练新的分类头 | 数据量很小（几百张） |
| 微调（Fine-tuning） | 解冻部分层，在小学习率下训练 | 数据量中等（几千张） |
| 全量微调 | 解冻所有层，整体微调 | 数据量大且与源域相似 |
| 从头训练 | 不使用预训练权重 | 数据量极大或域差异大 |


### 3.3 微调策略详解


### 逐层解冻策略（Discriminative Learning Rates）


不同层学习不同层次的特征，微调时应给予不同的学习率：


- **浅层（底层）**
   ：学习通用特征（边缘、纹理），使用
   **较小学习率**
- **深层（顶层）**
   ：学习任务特定特征，使用
   **较大学习率**


$$
lr_layer_k = lr_base × decay^(N-k)　　（通常 decay=0.1~0.5）
$$


```
# PyTorch微调示例
import torchvision.models as models

model = models.resnet50(pretrained=True)

# 1. 冻结所有层
for param in model.parameters():
    param.requires_grad = False

# 2. 替换最后的全连接层
model.fc = nn.Linear(2048, num_classes)

# 3. 设置不同层的学习率
optimizer = torch.optim.Adam([
    {'params': model.layer4.parameters(), 'lr': 1e-4},   # 浅层，小学习率
    {'params': model.fc.parameters(), 'lr': 1e-3},        # 新层，大学习率
], lr=1e-3)
```


> **Note:** **微调最佳实践：**
>
>
> 1. 先训练新加入的层若干epoch（其他层冻结）
>
>
> 2. 再以较小学习率解冻训练所有层
>
>
> 3. 使用较小的batch size（如16-32）通常比大batch效果好
>
>
> 4. 使用学习率warmup防止预训练权重被破坏
>
>
> 5. 源域和目标域差异大时，需要解冻更多层


### 3.4 域适应与跨域迁移


当源域和目标域差异较大时（如从ImageNet自然图像迁移到医学影像），需要特殊处理：


- **域自适应（Domain Adaptation）**
   ：通过对抗训练对齐源域和目标域的特征分布
- **域随机化（Domain Randomization）**
   ：在训练时随机化渲染参数，提高泛化能力
- **对比学习预训练**
   ：在目标域上使用自监督对比学习（SimCLR、MoCo等）获取更好的初始表示


```
# 完整的数据预处理和增强Pipeline
from torchvision import transforms

# 训练集：丰富的数据增强
train_transform = transforms.Compose([
    transforms.RandomResizedCrop(224, scale=(0.08, 1.0)),
    transforms.RandomHorizontalFlip(p=0.5),
    transforms.ColorJitter(brightness=0.2, contrast=0.2, saturation=0.2, hue=0.05),
    transforms.RandomRotation(15),
    transforms.RandomAffine(degrees=0, translate=(0.1, 0.1)),
    transforms.ToTensor(),
    transforms.Normalize([0.485, 0.456, 0.406], [0.229, 0.224, 0.225]),
    transforms.RandomErasing(p=0.5, scale=(0.02, 0.33)),
])

# 验证/测试集：仅基础预处理（无随机增强）
val_transform = transforms.Compose([
    transforms.Resize(256),
    transforms.CenterCrop(224),
    transforms.ToTensor(),
    transforms.Normalize([0.485, 0.456, 0.406], [0.229, 0.224, 0.225]),
])

# Albumentations（更高效的增强库）
import albumentations as A
from albumentations.pytorch import ToTensorV2

train_transform_albu = A.Compose([
    A.RandomResizedCrop(224, 224, scale=(0.08, 1.0)),
    A.HorizontalFlip(p=0.5),
    A.OneOf([
        A.ColorJitter(brightness=0.2, contrast=0.2, saturation=0.2, hue=0.05),
        A.RandomBrightnessContrast(brightness_limit=0.2, contrast_limit=0.2),
    ], p=0.8),
    A.OneOf([
        A.GaussNoise(var_limit=(10, 50)),
        A.GaussianBlur(blur_limit=(3, 7)),
    ], p=0.3),
    A.CoarseDropout(max_holes=1, max_height=56, max_width=56,
                    min_height=16, min_width=16, p=0.5),
    A.Normalize([0.485, 0.456, 0.406], [0.229, 0.224, 0.225]),
    ToTensorV2(),
])
```


| 问题 | 解答 |
| --- | --- |
| 数据增强会降低训练速度吗？ | 是的，但通常收益大于代价。可在CPU上做增强，GPU上做训练（num_workers > 0）。 |
| 增强越多越好吗？ | 过度增强可能导致训练困难，需根据数据量和任务调整。 |
| 验证集需要增强吗？ | 不需要随机增强，只需与训练集相同的确定性预处理（Resize、Normalize）。 |
| TTA是什么？ | Test-Time Augmentation，推理时对同一样本做多种增强取平均，提升泛化。 |
| 如何选择预训练模型？ | 数据少选小模型（ResNet-18），数据多选大模型（EfficientNet-B4）。源域与目标域越接近越好。 |
