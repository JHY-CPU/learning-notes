# 两阶段目标检测器

*R-CNN系列演进：从Region Proposal到端到端训练*


**目标检测（Object Detection）**不仅要识别图像中物体的类别，还要定位其位置（通常用边界框Bounding Box表示）。两阶段检测器的核心思想是**先生成候选区域，再对每个区域分类和回归**。


### 检测任务的基本要素


- **分类（Classification）**
   ：判断候选区域属于哪个类别
- **定位（Localization）**
   ：预测边界框坐标 (x, y, w, h) 或 (x1, y1, x2, y2)
- **置信度（Confidence）**
   ：评估检测结果的可信度


### 评估指标


| 指标 | 说明 |
| --- | --- |
| IoU（Intersection over Union） | 预测框与真实框的交并比，IoU ≥ 阈值（通常0.5）视为正确检测 |
| Precision（精确率） | TP / (TP + FP)，检测出的目标中正确的比例 |
| Recall（召回率） | TP / (TP + FN)，所有真实目标中被检测到的比例 |
| AP（Average Precision） | P-R曲线下面积，单类别的综合指标 |
| mAP（mean AP） | 所有类别AP的平均值，目标检测的核心指标 |


### 2.1 核心思想


R-CNN（Regions with CNN features）首次将CNN特征引入目标检测，开启了深度学习检测时代。

输入图像
→
Selective Search
生成~2000个候选框
→
每个候选框
warp到227×227
→
AlexNet
提取特征
→
SVM分类
+ 边框回归

### 2.2 训练流程


1. **预训练**
   ：在ImageNet上预训练AlexNet
2. **微调**
   ：将AlexNet的fc层替换为(N+1)类的分类层，在候选框上微调
3. **SVM训练**
   ：用CNN特征训练每个类别的线性SVM分类器
4. **边框回归训练**
   ：训练线性回归器精修候选框位置


### 2.3 缺陷


- 每个候选框独立过CNN前向传播（~2000次），速度极慢（47秒/张）
- 训练分多步，流程复杂（CNN微调、SVM训练、回归训练分开进行）
- Selective Search耗时且不可学习
- 特征存储占用大量磁盘空间


### 3.1 核心改进：共享卷积计算


Fast R-CNN的关键创新是**对整张图像只做一次卷积**，然后在特征图上提取每个候选框对应的特征。

输入图像
→
CNN骨干网络
（整图一次前向）
→
ROI Pooling
（特征图上提取）
→
FC层
→
分类 + 回归
（多任务联合训练）

### 3.2 ROI Pooling


ROI Pooling将不同大小的候选区域特征统一到固定尺寸（如7×7），使后续全连接层可以处理：


1. 将候选框映射到特征图上的对应区域
2. 将该区域划分为固定大小的网格（如7×7）
3. 对每个网格单元执行Max Pooling


```
# ROI Pooling示意
输入特征图上ROI区域: [x1, y1, x2, y2]
输出大小: 7×7

# 每个bin的大小:
bin_h = (y2 - y1) / 7
bin_w = (x2 - x1) / 7

# PyTorch实现
from torchvision.ops import roi_pool
output = roi_pool(feature_maps, rois, output_size=(7, 7), spatial_scale=1/16)
```


### 3.3 多任务损失


$$
L = L_cls(p, u) + λ·[u≥1]·L_loc(t, v)
$$


- **L_cls**
   ：交叉熵分类损失
- **L_loc**
   ：Smooth L1损失（对异常值不敏感，比L2更稳定）
- **λ**
   ：平衡系数（通常设为1）
- **[u≥1]**
   ：背景类不做边框回归


### 3.4 改进点总结


| 改进 | R-CNN | Fast R-CNN |
| --- | --- | --- |
| 特征提取 | 每个候选框独立过CNN | 整图一次CNN，ROI Pooling提取 |
| 分类器 | SVM | Softmax（端到端） |
| 边框回归 | 独立训练 | 与分类联合训练 |
| 训练步骤 | 3步（CNN→SVM→回归） | 1步端到端 |
| 速度 | 47秒/张 | 2秒/张（不含SS） |


### 4.1 核心创新：Region Proposal Network（RPN）


Faster R-CNN用**区域提议网络（RPN）**替代Selective Search，实现端到端可学习的候选框生成。这是整个检测流程速度从秒级提升到毫秒级的关键。

输入图像
→
CNN骨干
（ResNet/VGG）
→
RPN
生成候选框
→
ROI Pooling/
ROI Align
→
分类 + 回归

### 4.2 RPN详解


### Anchor机制


在特征图的每个位置预设多个**锚框（Anchor）**，覆盖不同尺度和宽高比：


- **尺度**
   ：通常使用 32², 64², 128², 256², 512² 像素（5种）
- **宽高比**
   ：1:1, 1:2, 2:1（3种）
- 每个位置共 5×3 = 15 个锚框
- 对于 60×40 的特征图，共产生 60×40×15 = 36000 个锚框


### RPN网络结构


```
特征图 → 3×3 Conv(256) → ┬─ 1×1 Conv(2×k) → 分类（前景/背景）
                          └─ 1×1 Conv(4×k) → 边框回归

k = anchor数量（每位置），2k个分类分数，4k个回归参数
```


### RPN损失函数


$$
L_rpn = (1/N_cls) Σ L_cls(pᵢ, pᵢ*) + λ·(1/N_reg) Σ pᵢ*·L_reg(tᵢ, tᵢ*)
$$


- 正样本：IoU ≥ 0.7 的锚框，或与任意GT框IoU最高的锚框
- 负样本：IoU ≤ 0.3 的锚框
- 既非正也非负的锚框不参与训练


### 4.3 ROI Align — 解决量化误差


ROI Pooling在量化过程中引入了两次取整误差（ROI边界量化和bin划分量化），导致特征与原始位置不精确对齐。**ROI Align**使用双线性插值避免量化：


1. 不做任何取整，保留浮点坐标
2. 在每个bin内均匀采样4个点（或更多）
3. 通过双线性插值计算采样点的值
4. 对采样点取Max或Average


```
# ROI Align vs ROI Pooling
# ROI Pooling: 坐标取整 → 网格划分 → Max Pooling
# ROI Align:   浮点坐标 → 均匀采样 → 双线性插值 → Average/Max

from torchvision.ops import roi_align
output = roi_align(feature_maps, rois, output_size=(7, 7),
                   spatial_scale=1/16, sampling_ratio=2)
# sampling_ratio: 每个bin内采样点数，-1为自适应
```


> **Note:** **ROI Align的效果：**
> 在Mask R-CNN中，使用ROI Align替代ROI Pooling使mask精度提升了约10-20%，尤其是在需要精确定位的实例分割任务中效果显著。


### 4.4 FPN（Feature Pyramid Network）


不同尺度的物体需要不同层次的特征：小物体需要高分辨率的浅层特征，大物体需要语义丰富的深层特征。FPN通过**自顶向下路径+横向连接**构建特征金字塔。


```
自底向上路径（骨干网络）:
C1(1/2) → C2(1/4) → C3(1/8) → C4(1/16) → C5(1/32)

自顶向下路径 + 横向连接:
P5 = 1×1 Conv(C5)
P4 = 1×1 Conv(C4) + Upsample(P5)
P3 = 1×1 Conv(C3) + Upsample(P4)
P2 = 1×1 Conv(C2) + Upsample(P3)

# 每个P层都送入独立的RPN和检测头
```


- **横向连接**
   ：1×1卷积统一通道数（通常256维）
- **上采样**
   ：2×最近邻上采样
- **3×3卷积**
   ：每个P层后接3×3卷积减少混叠效应
- 所有层共享检测头的参数，减少计算量


### 5.1 Cascade R-CNN


使用多个检测头级联训练，每个阶段使用更高的IoU阈值，逐步精修检测框：


- Stage 1: IoU阈值=0.5（粗检测）
- Stage 2: IoU阈值=0.6（中等精度）
- Stage 3: IoU阈值=0.7（高精度检测）


### 5.2 其他重要改进


| 模型 | 核心改进 | 效果 |
| --- | --- | --- |
| Mask R-CNN | 增加mask分支，ROI Align | 同时做检测+分割 |
| Libra R-CNN | 均衡采样、特征融合、损失函数 | 解决样本和特征不平衡 |
| Grid R-CNN | 网格点预测替代边框回归 | 更好的空间定位 |
| HTC | 级联+交错的检测和分割 | COCO榜首方法之一 |
