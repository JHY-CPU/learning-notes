# 单阶段目标检测器

*YOLO系列、SSD与Anchor-Free方法全解析*


与两阶段检测器（先生成候选框再分类）不同，**单阶段检测器（One-Stage Detector）**直接在特征图上同时预测类别和边界框，省去了区域提议步骤，因此**速度更快**，适合实时应用。


| 对比维度 | 两阶段检测器 | 单阶段检测器 |
| --- | --- | --- |
| 速度 | 较慢（5-20 FPS） | 更快（30-150+ FPS） |
| 精度 | 通常更高 | 略低（差距在缩小） |
| 小目标检测 | 更优 | 相对困难 |
| 密集目标 | 较弱 | 更强 |
| 代表模型 | Faster R-CNN, Cascade R-CNN | YOLO, SSD, RetinaNet |


### 2.1 YOLO v1（2016）


YOLO（You Only Look Once）将目标检测重新定义为**回归问题**：将图像划分为S×S网格，每个网格预测B个边界框和C个类别概率。


```
# YOLO v1 输出: S×S×(B×5+C)
# S=7, B=2, C=20 (VOC)
# 每个网格预测: 2个框×(x,y,w,h,confidence) + 20个类别概率
# 输出张量: 7×7×30
```


- **优点**
   ：全局推理，速度快（45 FPS），背景误检少
- **缺点**
   ：每个网格只预测一个类别，密集小物体检测差，定位精度低


### 2.2 YOLO v2 / YOLO9000（2017）


多项改进使YOLO在保持速度的同时大幅提升精度：


| 改进 | 说明 |
| --- | --- |
| Batch Normalization | 所有卷积层后加BN，去掉Dropout |
| 高分辨率分类器 | 先在448×448上预训练分类，再微调检测 |
| Convolutional with Anchor Boxes | 借鉴Faster R-CNN使用anchor |
| Dimension Clusters | 用K-means在训练集上聚类anchor尺寸 |
| Direct Location Prediction | 直接预测相对网格偏移，而非绝对坐标 |
| Passthrough层 | 将26×26特征拼接到13×13，增强细粒度信息 |
| Multi-Scale Training | 训练时每隔10个batch随机切换输入尺寸 |
| Darknet-19 | 新的骨干网络，19层卷积+5层MaxPooling |


### 2.3 YOLO v3（2018）


引入多尺度预测，使用Darknet-53骨干网络，在三个不同尺度上检测目标。


```
# 多尺度检测
# 大尺度 (13×13): 检测大物体
# 中尺度 (26×26): 检测中等物体
# 小尺度 (52×52): 检测小物体

# 每个尺度预测: 3个anchor × (4坐标 + 1置信度 + 类别数)
# 使用多标签分类（Sigmoid替代Softmax），支持一个物体属于多个类别
```


- **Darknet-53**
   ：残差结构，比Darknet-19更深
- **FPN类似结构**
   ：上采样+拼接实现多尺度特征融合
- 在COCO上mAP达33.0%（与SSD相当但速度快3倍）


### 2.4 YOLO v4（2020）


Bochkovskiy等人提出的"最佳实践集合"，系统性地融合了大量tricks：


| 类别 | 技术 |
| --- | --- |
| 免费包（Bag of Freebies） | CutMix, Mosaic增强, DropBlock, CIoU Loss, Label Smoothing, 自对抗训练 |
| 特价包（Bag of Specials） | Mish激活, CSPDarknet53, SPP, SAM注意力, PANet路径聚合 |


- **CSPNet**
   ：将特征图分为两部分，一部分直接拼接，一部分经过密集块，减少计算量
- **SPP（Spatial Pyramid Pooling）**
   ：多尺度池化后拼接，增大感受野
- **PANet**
   ：在FPN基础上增加自底向上的路径聚合


### 2.5 YOLO v5（2020）


Ultralytics发布，虽非学术论文但工程实现极其出色：


- **自适应anchor**
   ：训练时自动计算最佳anchor
- **Focus结构**
   ：将空间信息切片到通道维度（后续版本改为Conv替代）
- **自适应图片缩放**
   ：减少灰边填充
- **模型缩放**
   ：n/s/m/l/x五种规模，灵活适配不同场景
- PyTorch原生实现，部署简单，社区生态丰富


### 2.6 YOLO v6/v7/v8


| 版本 | 发布方 | 核心改进 |
| --- | --- | --- |
| YOLOv6 | 美团 | 高效骨干网络（EfficientRep）+ RepVGG风格重参数化 |
| YOLOv7 | WongKinYiu | E-ELAN结构、复合模型缩放、辅助检测头 |
| YOLOv8 | Ultralytics | Anchor-free、解耦头、C2f模块、统一检测/分割/关键点/分类 |


YOLOv8是目前最流行的YOLO版本，采用**Anchor-Free**设计，使用**解耦头（Decoupled Head）**分别做分类和回归，并引入**C2f模块**（跨阶段部分融合）替代C3模块。


### 3.1 核心思想


SSD在多个不同尺度的特征图上同时检测目标，利用浅层特征检测小物体，深层特征检测大物体。


```
# SSD多尺度检测
conv4_3 (38×38): 检测小物体
conv7   (19×19): 检测中小物体
conv8_2 (10×10): 检测中物体
conv9_2 (5×5):   检测中大物体
conv10_2(3×3):   检测大物体
conv11_2(1×1):   检测超大物体

# 每个位置设置不同尺寸和比例的default box
# conv4_3: scales=[0.1], ratios=[1, 2, 1/2] (+额外的ratio=1的box)
# 后续层: scales逐渐增大
```


### 3.2 关键技术


- **多尺度特征图**
   ：在不同分辨率的特征图上独立检测
- **Default Box（默认框）**
   ：类似anchor，每个位置预设不同尺度和比例的框
- **Hard Negative Mining**
   ：正负样本比例1:3，只取loss最大的负样本
- **Atrous Convolution**
   ：空洞卷积替代部分下采样，扩大感受野不降分辨率


### 4.1 正负样本不平衡问题


单阶段检测器面临严重的**正负样本类别不平衡**：一张图像可能有数万个锚框，但只有少数包含目标（正样本）。大量简单的负样本主导了梯度更新，导致模型性能下降。


### 4.2 Focal Loss


$$
FL(p_t) = -α_t · (1 - p_t)^γ · log(p_t)
$$


- **p_t**
   ：模型对正确类别的预测概率
- **α_t**
   ：类别平衡因子（正样本α通常0.25，负样本0.75）
- **γ（gamma）**
   ：聚焦参数，通常设为2
- 当p_t很大（容易分类）时，(1-p_t)^γ趋近于0，loss被大幅降低
- 当p_t很小（难分类）时，(1-p_t)^γ趋近于1，loss保持不变


```
# Focal Loss实现
def focal_loss(pred, target, alpha=0.25, gamma=2.0):
    ce_loss = F.cross_entropy(pred, target, reduction='none')
    pt = torch.exp(-ce_loss)
    focal_weight = alpha * (1 - pt) ** gamma
    return (focal_weight * ce_loss).mean()
```


### 4.3 RetinaNet


专为验证Focal Loss效果而设计的单阶段检测器：


- **骨干网络**
   ：ResNet + FPN
- **分类子网络**
   ：4个3×3卷积 + Sigmoid输出
- **回归子网络**
   ：4个3×3卷积 + 线性输出
- 两个子网络
   **不共享参数**
   （解耦头）
- COCO mAP达39.1%，超越同时期的两阶段检测器


### 5.1 为什么需要Anchor-Free


基于Anchor的方法存在几个问题：


- Anchor的尺寸、比例、数量是超参数，需要针对数据集精心设计
- 正负样本分配依赖IoU阈值，不够灵活
- 大量Anchor导致正负样本极度不平衡
- 不同数据集之间Anchor难以迁移


### 5.2 FCOS（Fully Convolutional One-Stage）


将目标检测转化为**逐像素预测**：特征图上的每个点预测到边界框四条边的距离。


```
# FCOS预测目标
每个位置预测:
  - 分类: C维向量（类别概率）
  - 回归: 4维向量 (l, t, r, b) — 到左、上、右、下边的距离
  - Centerness: 1维标量（中心度，抑制远离中心的低质量检测）

# Centerness计算
centerness = sqrt(min(l,r)/max(l,r) × min(t,b)/max(t,b))
# 值越接近1表示越靠近目标中心
```


- 无需Anchor设计，超参数更少
- 每位置只需预测一个框（与GT框IoU最大的特征层负责预测）
- Centerness分支过滤低质量检测


### 5.3 CenterNet


将目标检测建模为**关键点检测**：检测目标中心点，然后回归其尺寸。


```
# CenterNet输出
热力图 (H×W×C): 每个类别的中心点概率
尺寸回归 (H×W×2): 目标的宽和高
偏移回归 (H×W×2): 中心点的量化误差补偿

# 训练时用高斯核生成GT热力图
# 推理时取热力图的局部极大值作为检测结果
```


- 无需NMS后处理（热力图峰值天然分离）
- 结构简洁，可扩展到3D检测、姿态估计等任务
- 骨干网络可选Hourglass、DLA、ResNet


### 5.4 Anchor-Free方法对比


| 方法 | 预测形式 | 正负样本分配 | 是否需要NMS |
| --- | --- | --- | --- |
| FCOS | 逐像素预测到四边距离 | GT框内的多层分配 | 需要 |
| CenterNet | 中心点热力图+尺寸 | 高斯核生成GT | 不需要 |
| CornerNet | 左上角+右下角热力图 | 角点嵌入匹配 | 需要 |
| YOLOv8 | 关键点+距离预测 | TaskAlignedAssigner | 需要 |


### 6.1 标准NMS


NMS是目标检测的标准后处理步骤，用于去除重叠的冗余检测框：


1. 按置信度降序排列所有检测框
2. 选择置信度最高的框，加入最终结果
3. 移除与该框IoU大于阈值（通常0.5）的所有框
4. 重复步骤2-3直到所有框处理完毕


### 6.2 NMS变体


| 方法 | 说明 | 适用场景 |
| --- | --- | --- |
| Soft-NMS | 不直接删除重叠框，而是降低其置信度 | 密集目标场景 |
| DIoU-NMS | 使用DIoU替代IoU作为抑制准则 | 考虑中心点距离 |
| Weighted NMS | 用加权平均融合重叠框 | 提高定位精度 |
| Matrix NMS | 并行化NMS，加速推理 | 实时检测场景 |


| 损失函数 | 公式/特点 | 优势 |
| --- | --- | --- |
| L1 / Smooth L1 | 直接回归坐标差值 | 简单，但不反映框的重叠度 |
| IoU Loss | 1 - IoU | 直接优化IoU，尺度不变 |
| GIoU Loss | 1 - (IoU - \|C\(A∪B)\|/\|C\|) | 解决不相交框梯度为0的问题 |
| DIoU Loss | 1 - IoU + ρ²(b,b_gt)/c² | 考虑中心点距离，收敛更快 |
| CIoU Loss | 1 - IoU + ρ²/c² + αv | 同时考虑重叠、距离、宽高比 |


$$
CIoU = IoU - ρ²(b, b_gt)/c² - α·v,　其中 v = (4/π²)(arctan(w_gt/h_gt) - arctan(w/h))²
$$
