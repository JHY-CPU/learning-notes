01 - 自顶向下语法分析

一、语法分析概述

语法分析（Parsing）是编译器的第二阶段，其任务是根据语言的上下文无关文法（CFG），检查记号流是否构成合法的语法结构，并构建语法树。

1.1 两种分析策略

策略方向代表方法能力
自顶向下（Top-Down）从根到叶递归下降、LL(1)LL文法（较受限）
自底向上（Bottom-Up）从叶到根LR、SLR、LALRLR文法（更强大）

1.2 上下文无关文法（CFG）四元组

G = (VT, VN, P, S)

  • VT — 终结符集合（Token）
  • VN — 非终结符集合（语法变量）
  • P — 产生式集合：A → α
  • S — 开始符号

二、消除左递归

自顶向下分析无法处理直接左递归，必须先消除。

2.1 直接左递归的消除

对产生式 A → Aα | β（β 不以 A 开头），转换为：

A → βA'

A' → αA' | ε

2.2 示例

原文法：E → E+T | T

消除后：

E → TE'

E' → +TE' | ε

2.3 间接左递归的消除

  1. 对非终结符按任意顺序排列：A1, A2, ..., An
  2. 对每个 i，将 Ai → Ajγ（j<i）替换为 Ai 的展开式
  3. 消除直接左递归

三、提取左因子

当一个非终结符的多个产生式有公共前缀时，自顶向下分析需要额外的超前搜索符号。通过提取左因子消除公共前缀。

原文法：A → αβ1 | αβ2 | γ

提取后：

A → αA' | γ

A' → β1 | β2

四、FIRST 集与 FOLLOW 集

4.1 FIRST 集

FIRST(α) = 从 α 推导出的所有串的首终结符集合。若 α ⇒* ε，则 ε ∈ FIRST(α)。

计算规则：

  1. 若 X 是终结符，FIRST(X) = {X}
  2. 若 X → ε 是产生式，将 ε 加入 FIRST(X)
  3. 若 X → Y1Y2...Yk：
    • 将 FIRST(Y1) - {ε} 加入 FIRST(X)
    • 若 ε ∈ FIRST(Y1)，继续加入 FIRST(Y2)...
    • 若所有 Yi 都可推出 ε，将 ε 加入 FIRST(X)

4.2 FOLLOW 集

FOLLOW(A) = 在某些句型中紧跟在非终结符 A 右边的终结符集合。若 A 可以是某句型的最右符号，则 $（输入结束符）∈ FOLLOW(A)。

计算规则：

  1. 将 $ 加入 FOLLOW(S)（S为开始符号）
  2. 若有产生式 A → αBβ，将 FIRST(β) - {ε} 加入 FOLLOW(B)
  3. 若有产生式 A → αB 或 A → αBβ 且 ε ∈ FIRST(β)，将 FOLLOW(A) 加入 FOLLOW(B)

4.3 计算示例

文法：

E → TE'

E' → +TE' | ε

T → FT'

T' → *FT' | ε

F → (E) | id


FIRST集：FIRST(E) = FIRST(T) = FIRST(F) = {(, id}

FOLLOW集：FOLLOW(E) = FOLLOW(E') = {), $}，FOLLOW(T) = FOLLOW(T') = {+, ), $}

五、LL(1) 分析

5.1 LL(1) 文法的条件

文法 G 是 LL(1) 当且仅当对任意两个产生式 A → α | β：

  1. FIRST(α) ∩ FIRST(β) = ∅
  2. 若 ε ∈ FIRST(β)，则 FIRST(α) ∩ FOLLOW(A) = ∅

5.2 LL(1) 分析表构造

对每个产生式 A → α：

  1. 对每个 a ∈ FIRST(α)，在 M[A, a] 中填入 A → α
  2. 若 ε ∈ FIRST(α)，对每个 b ∈ FOLLOW(A)，在 M[A, b] 中填入 A → α

5.3 LL(1) 分析表示例

非终结符id+*()$

E'E'→+TE'E'→εE'→ε
TT→FT'T→FT'
T'T'→εT'→*FT'T'→εT'→ε

六、递归下降分析（Recursive Descent）

6.1 原理

为每个非终结符编写一个递归函数，函数体按照产生式右部的顺序调用对应函数或匹配终结符。

6.2 递归下降分析器示例

void E() { T(); E_prime(); } void E_prime() { if (lookahead == '+') { match('+'); T(); E_prime(); } // else: ε 产生式，什么都不做 } void T() { F(); T_prime(); } void T_prime() { if (lookahead == '*') { match('*'); F(); T_prime(); } } void F() { if (lookahead == 'id') { match('id'); } else if (lookahead == '(') { match('('); E(); match(')'); } else { error("语法错误"); } }

6.3 预测分析器（Predictive Parser）

使用显式栈代替递归调用的 LL(1) 分析器：

输入: id + id * id $ ┌──────────────────────────────────────┐ │ 分析栈 输入 动作 │ ├──────────────────────────────────────┤ │ $E id+id*id$ E→TE' │ │ $E'T id+id*id$ T→FT' │ │ $E'T'F id+id*id$ F→id │ │ $E'T'id id+id*id$ 匹配id│ │ $E'T' +id*id$ T'→ε │ │ $E' +id*id$ E'→+TE│ │ $E'T+ +id*id$ 匹配+ │ │ $E'T id*id$ ... │ └──────────────────────────────────────┘
