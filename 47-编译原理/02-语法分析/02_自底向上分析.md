# 自底向上分析 - 编译原理笔记


自底向上分析从输入串出发，通过**归约**（Reduction）逐步将终结符序列替换为非终结符，最终归约到**开始符号**S。


### 1.1 核心操作


- **移进（Shift）**
   ：将下一个输入符号移入分析栈
- **归约（Reduce）**
   ：将栈顶的某个符号串替换为对应的非终结符（产生式左部）
- **接受（Accept）**
   ：输入完毕且栈中只剩开始符号
- **报错（Error）**
   ：既不能移进也不能归约


### 1.2 句柄（Handle）


句柄是当前句型中对应某个产生式右部的子串，对其进行归约后仍是合法句型。

**关键思想**
：自底向上分析本质上就是在每一步找到当前句型的
**句柄**
并进行归约。

### 2.1 LR(0) 项目


LR(0) 项目是在产生式右部某位置插入一个点（·）的产生式。例如产生式 `A → XYZ` 对应的项目为：


- `A → ·XYZ`
   — 即将开始归约
- `A → X·YZ`
   — 已匹配X
- `A → XY·Z`
   — 已匹配XY
- `A → XYZ·`
   — 可进行归约


### 2.2 增广文法


在原文法基础上添加新开始符号 `S'` 和产生式 `S' → S`，确保接受状态唯一。


### 2.3 闭包（Closure）与转移（GOTO）


> **Example:** **CLOSURE(I)**：
>
>
> 1. 将 I 中所有项目加入闭包
> 2. 若
>    `A → α·Bβ`
>    在闭包中，对每个
>    `B → γ`
>    ，将
>    `B → ·γ`
>    加入
> 3. 重复直到无新项目加入
>
>
> **GOTO(I, X)** = CLOSURE({ `A → αX·β` | `A → α·Xβ` ∈ I })


### 2.4 LR(0) 自动机构造步骤


1. 初始状态 I
   ~0~
   = CLOSURE({
   `S' → ·S`
   })
2. 对每个状态 I 和每个文法符号 X，计算 GOTO(I, X) 作为新状态
3. 重复直到无新状态
4. 若项目
   `A → α·`
   在状态 I 中，则对所有终结符，I 的动作是"归约 A→α"


SLR（Simple LR）在 LR(0) 基础上引入**FOLLOW集**来解决归约冲突。


### 3.1 SLR 动作规则


- 若
   `A → α·aβ`
   ∈ I
   ~i~
   ，且 GOTO(I
   ~i~
   , a) = I
   ~j~
   ，则 ACTION[i, a] =
   **Shift j**
- 若
   `A → α·`
   ∈ I
   ~i~
   ，对每个 a ∈ FOLLOW(A)，则 ACTION[i, a] =
   **Reduce A→α**
- 若
   `S' → S·`
   ∈ I
   ~i~
   ，则 ACTION[i, $] =
   **Accept**


### 3.2 SLR 的局限


SLR 使用 FOLLOW 集过于保守，某些文法即使 SLR 表有冲突，实际分析中也不存在歧义。


### 4.1 LR(1) 项目


LR(1) 项目是一个二元组 `[A → α·β, a]`，其中 a 是**向前搜索符号**（lookahead）。


- 仅当输入符号为 a 时才对
   `A → α·`
   进行归约
- 比 LR(0) 和 SLR 更精确，解决了更多的冲突
- 缺点：状态数可能非常多


### 4.2 LALR(1) 分析


LALR（Look-Ahead LR）是 LR(1) 的**合并优化**版本：


1. 构造 LR(1) 自动机
2. 合并核心（core）相同的状态（即忽略向前搜索符号后相同的项目集）
3. 状态数与 LR(0) 相同，但能力接近 LR(1)

**LALR(1) 的实用性**
：Yacc/Bison 默认生成 LALR(1) 分析器，是实际应用中最常用的 LR 分析方法。

### 5.1 冲突类型


| 冲突类型 | 产生原因 | 解决方法 |
| --- | --- | --- |
| 移进-归约冲突 | 同一状态下既可移进又可归约 | 优先移进 / 优先级规则 |
| 归约-归约冲突 | 同一状态下可按多个产生式归约 | 选择列表中靠前的产生式 |

**示例：悬空else问题**


`S → if E then S | if E then S else S | other`


当输入为
`if E~1~ then if E~2~ then S~1~ else S~2~`
时，else 可以与第一个 if 匹配（移进），也可以与第二个 if 匹配（归约）。


**解决**
：采用"最近匹配"策略 — 优先移进。

### 5.2 运算符优先级和结合性


```
%left '+' '-'     /* 左结合，相同优先级 */
%left '*' '/'     /* 左结合，高于 + - */
%right '^'        /* 右结合，最高优先级 */
%nonassoc '<' '>' /* 不结合 */
```


### 6.1 LR 分析表结构


LR 分析表由**ACTION表**和**GOTO表**组成：


- ACTION[state, terminal] = Shift/Reduce/Accept/Error
- GOTO[state, nonterminal] = next state


### 6.2 分析过程


```
输入串: id + id * id

步骤  │ 栈              │ 输入       │ 动作
──────┼─────────────────┼────────────┼──────────
  1   │ 0               │ id+id*id$  │ 移进
  2   │ 0 id            │ +id*id$    │ 归约 F→id
  3   │ 0 F             │ +id*id$    │ goto(0,F)=3
  4   │ 0 F 3           │ +id*id$    │ 归约 T→F
  5   │ 0 T             │ +id*id$    │ goto(0,T)=2
  6   │ 0 T 2           │ +id*id$    │ 归约 E→T
  7   │ 0 E             │ +id*id$    │ goto(0,E)=1
  8   │ 0 E 1           │ +id*id$    │ 移进
  9   │ 0 E 1 +         │ id*id$     │ 移进
  ... │ ...             │ ...        │ ...
      │ 0 E 1           │ $          │ 接受 ✓
```


| 方法 | 状态数 | 能力 | 构造复杂度 | 实用工具 |
| --- | --- | --- | --- | --- |
| LR(0) | 最少 | 最弱 | O(n²) | 教学用 |
| SLR(1) | 与LR(0)相同 | 中等 | O(n²) | 教学用 |
| LALR(1) | 与LR(0)相同 | 接近LR(1) | O(n³) | Yacc/Bison |
| LR(1) | 可能指数增长 | 最强 | 指数级 | 极少使用 |

**实际选择**
：绝大多数编程语言使用 LALR(1) 分析器（Bison），少数使用手写的递归下降分析器（如GCC的早期版本）。
