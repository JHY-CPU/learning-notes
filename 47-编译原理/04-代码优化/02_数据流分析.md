# 数据流分析 - 编译原理笔记


数据流分析（Data Flow Analysis）是一组用于收集程序中**数据如何在基本块之间流动**的信息的技术。它是许多全局优化的基础。


### 1.1 基本框架


```
控制流图(CFG)
  ┌───────────┐
  │  基本块 B1 │
  │  in[B1]    │──▶ 传递函数 f_B ──▶ out[B1]
  └───────────┘
       │
       ▼
  满足数据流方程:
  out[B] = f_B(in[B])
  in[B]  = ∩/∪ out[P]  (P为B的所有前驱)
```


### 1.2 数据流分析的方向


| 分析方向 | 方程 | 典型问题 |
| --- | --- | --- |
| 前向分析 | `in[B] = ∪ out[P]` | 到达定义、可用表达式 |
| 后向分析 | `out[B] = ∪ in[S]` | 活跃变量、非常忙表达式 |


其中 P 为 B 的前驱，S 为 B 的后继。


**问题**：对于程序中的每个点，哪些变量赋值（定义）可能"到达"该点而不被重新定义？


### 2.1 形式化


- **全集**
   ：程序中所有定义（赋值语句）的集合
- **传递函数**
   ：
   `f~B~(x) = gen~B~ ∪ (x - kill~B~)`
- **gen~B~**
   ：在 B 中产生的定义（B 中的赋值）
- **kill~B~**
   ：在 B 中被杀死的定义（同一变量在其他块的赋值）
- **合并运算**
   ：并集 ∪
- **边界条件**
   ：in[entry] = ∅


### 2.2 示例


```
d1: x = a + b     B1: gen={d1}, kill={d4}
      │
  ┌───┴───┐
  ▼       ▼
B2        B3
d2: y=1   d3: x=c+1
gen={d2}  gen={d3}
kill=∅    kill={d1,d4}
  │       │
  └───┬───┘
      ▼
B4: z=x+y     ← in[B4] = {d1,d2} ∪ {d2,d3} = {d1,d2,d3}
d4: x=z+1     （d1或d3可能到达，取决于分支）
```


**问题**：对于程序中的每个点和每个变量 x，是否存在一条从该点出发的路径，在该路径上 x 在被重新定义之前被使用？


### 3.1 形式化


- **分析方向**
   ：后向分析
- **传递函数**
   ：
   `f~B~(x) = use~B~ ∪ (x - def~B~)`
- **use~B~**
   ：在 B 中被使用且在 B 中该使用之前未被定义的变量
- **def~B~**
   ：在 B 中被定义（赋值）的变量
- **合并运算**
   ：并集 ∪
- **边界条件**
   ：out[exit] = ∅


### 3.2 应用


- **死代码消除**
   ：若某定义的结果不活跃，可删除该定义
- **寄存器分配**
   ：活跃变量必须分配到寄存器，不活跃的可溢出到内存


**问题**：对于程序中的每个点，哪些表达式在到达该点的每条路径上都已被计算过，且其操作数未被修改？


### 4.1 形式化


- **分析方向**
   ：前向分析
- **传递函数**
   ：
   `f~B~(x) = gen~B~ ∪ (x - kill~B~)`
- **合并运算**
   ：交集 ∩（与到达定义不同！）
- **边界条件**
   ：in[entry] = U（全集），因为入口处没有任何信息

**为什么用交集？**
"可用"要求表达式在
**所有**
到达路径上都被计算过。若路径A计算了，路径B没计算，该表达式在汇合点
**不可用**
。

### 4.2 应用


驱动**全局公共子表达式消除**：若表达式 e 在某点可用，且该点重新计算了 e，可直接使用之前的结果。


| 属性 | 到达定义 | 活跃变量 | 可用表达式 |
| --- | --- | --- | --- |
| 分析方向 | 前向 | 后向 | 前向 |
| 传递函数 | gen ∪ (x - kill) | use ∪ (x - def) | gen ∪ (x - kill) |
| 合并运算 | 并集 ∪ | 并集 ∪ | 交集 ∩ |
| 边界值（起点） | in[entry] = ∅ | — | in[entry] = U |
| 边界值（终点） | — | out[exit] = ∅ | — |
| 初始值（迭代） | in[B] = ∅ | out[B] = ∅ | in[B] = U |
| 格结构 | 并集半格 | 并集半格 | 交集半格 |
| 典型应用 | 常量传播 | 死代码消除/寄存器分配 | CSE |


### 6.1 Worklist 算法（前向分析）


```
void dataFlowAnalysis() {
    // 初始化
    for (each block B) {
        in[B] = (mergeOp == UNION) ? ∅ : U;
    }
    worklist = {all blocks};

    while (!worklist.isEmpty()) {
        B = worklist.remove();
        // 计算输入
        in[B] = merge(all predecessors P of B, out[P]);

        // 保存旧输出
        oldOut = out[B];

        // 计算传递函数
        out[B] = gen[B] ∪ (in[B] - kill[B]);

        // 若输出变化，将后继加入工作列表
        if (out[B] ≠ oldOut) {
            worklist.add(all successors of B);
        }
    }
}
```


### 6.2 收敛性保证


- 数据流值构成
   **有限高度的格（Lattice）**
- 传递函数是
   **单调的**
   （monotone）
- 根据
   **Kleene不动点定理**
   ，迭代一定收敛


### 7.1 格（Lattice）的定义


格是一个偏序集 (L, ≤)，其中任意两个元素都有唯一的**最小上界**（join, ⊔）和**最大下界**（meet, ⊓）。


### 7.2 常见格结构


```
常量格 (ConstLattice):       高度2的交集格:

      ⊤ (任意值)                  U (全集)
     / | \                      / | \  ...
   c1  c2  c3 ...             e1  e2  e3 ...
     \ | /                      \ | /
      ⊥ (矛盾/不可达)            ∅ (空集)

  到达定义: 用并集格(高度为1, 集合大小有限)
  可用表达式: 用交集格
  常量传播: 用常量格(高度为2, 结合并集和交集)
```


### 7.3 不动点定理


若 F 是格上的单调函数，则 F 有**最小不动点**（Least Fixed Point），迭代算法通过单调上升/下降逼近该不动点。
