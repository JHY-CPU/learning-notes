01 - 局部与全局代码优化

一、代码优化概述

代码优化是在保持程序语义不变的前提下，对中间代码进行等价变换，以提高目标代码的执行效率或减小代码体积。

1.1 优化的分类

分类维度类别说明
范围局部优化在单个基本块内进行
全局优化在函数/过程范围内进行
过程间优化跨越函数边界进行
依赖机器无关优化不依赖目标机器特性
机器相关优化依赖目标机器的体系结构

1.2 安全优化 vs 不安全优化

  • 安全优化：保证不改变程序语义（如常量折叠）
  • 不安全优化：可能改变浮点精度、异常行为等（如代数简化）

二、局部优化（基本块内）

2.1 常量折叠（Constant Folding）

在编译时计算常量表达式的值：

优化前：
t1 = 3 + 5
t2 = t1 * 2
优化后：
t1 = 8
t2 = 16

2.2 常量传播（Constant Propagation）

用已知的常量值替换变量引用：

优化前：
x = 5
y = x + 3
z = x * y
优化后：
x = 5
y = 8
z = 40

2.3 代数简化（Algebraic Simplification）

原始表达式简化后规则
x + 0x恒等元
x * 1x恒等元
x * 00零元
x * 2x + x乘法→加法
x * 8x << 3乘法→移位
x - x0自消去
x / x1自消去（x≠0）
!!xx双重否定

2.4 公共子表达式消除（CSE）

在基本块内，若表达式已计算过且变量未被修改，可复用之前的结果：

优化前：
a = b + c
d = b + c // 重复计算
优化后：
a = b + c
d = a

2.5 死代码消除（Dead Code Elimination）

删除计算结果从未被使用的语句：

优化前：
x = 10
y = x + 5 // y未被使用
z = x * 2
return z
优化后：
x = 10
z = x * 2
return z

三、循环优化

3.1 循环不变量外提（Loop-Invariant Code Motion, LICM）

将循环内计算结果不变的表达式提到循环外部：

优化前：
for (i=0; i<n; i++) {
t = a * b; // 不依赖i
c[i] = t + i;
}
优化后：
t = a * b;
for (i=0; i<n; i++) {
c[i] = t + i;
}

3.2 归纳变量强度削弱（Induction Variable Strength Reduction）

用开销小的运算替换开销大的运算：

优化前：
for (i=0; i<n; i++) {
a[i] = i * 4; // 每次乘法
}
优化后：
t = 0;
for (i=0; i<n; i++) {
a[t] = t; // 加法替代乘法
t = t + 4;
}

3.3 循环展开（Loop Unrolling）

减少循环控制开销，增加指令级并行：

优化前：
for (i=0; i<4; i++)
sum += a[i];
完全展开：
sum += a[0];
sum += a[1];
sum += a[2];
sum += a[3];

3.4 循环交换与循环分块

  • 循环交换：交换嵌套循环的顺序，改善缓存局部性
  • 循环分块（Tiling）：将大循环拆分为小块，适应缓存大小

四、全局优化

4.1 全局公共子表达式消除

跨基本块识别相同的表达式计算：

B1: a = b + c │ ┌───┴───┐ ▼ ▼ B2: ... B3: ... │ │ └───┬───┘ ▼ B4: d = b + c ← 在B1中已计算，可在B4中复用

4.2 复制传播（Copy Propagation）

用赋值的右值替换对左值的后续引用：

x = y
z = x + 1
x = y
z = y + 1

4.3 代码移动（Code Motion）

将不变表达式移到支配其所有使用点的最早位置。

支配关系（Dominance）：基本块 d 支配基本块 n，当且仅当从入口到 n 的每条路径都经过 d。记作 d dom n。

五、优化效果的度量

度量说明典型优化
执行时间程序运行总时间所有优化
指令数执行的指令总数死代码消除、CSE
缓存命中率数据/指令缓存的命中比例循环分块、数据布局
寄存器压力同时活跃的变量数寄存器分配优化
代码体积生成代码的总字节数内联决策
注意：不同优化之间可能存在冲突（如循环展开增加代码体积），编译器需要根据优化级别（O1/O2/O3）平衡各种指标。
