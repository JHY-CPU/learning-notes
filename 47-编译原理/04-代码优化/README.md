# 04-代码优化（Code Optimization）

> 代码优化是编译器在不改变程序语义的前提下，对中间代码进行等价变换以提高目标代码执行效率的过程。优化可以减少运行时间、降低内存占用或减少代码体积。

---

## 1. 优化的分类

### 1.1 按优化范围分类

| 类别 | 说明 | 举例 |
|------|------|------|
| **局部优化** | 在单个基本块内部进行 | 常量折叠、公共子表达式消除 |
| **循环优化** | 针对循环结构 | 循环不变量外提、归纳变量消除 |
| **全局优化** | 在整个函数/过程范围内进行 | 全局公共子表达式消除、全局寄存器分配 |
| **过程间优化** | 跨越函数边界 | 内联展开、过程间常量传播 |

### 1.2 按与机器的关系分类

- **机器无关优化（Machine-Independent）**：在中间代码层面进行，不依赖目标机器特性。如常量折叠、公共子表达式消除等。
- **机器相关优化（Machine-Dependent）**：依赖目标机器的指令集、寄存器数目、流水线特性等。如指令调度、寄存器分配。

### 1.3 优化的基本原则

- **安全性**：优化不能改变程序的可观察行为
- **收益性**：优化应使程序在某方面有所改善
- **编译时间权衡**：过度优化可能导致编译时间过长

## 2. 基本块与流图

### 2.1 基本块（Basic Block）

基本块是满足以下条件的最大指令序列：
1. 控制流只能从**第一条指令**进入
2. 只能从**最后一条指令**退出
3. 中间没有跳转入口，也没有跳转出口

### 2.2 划分基本块的算法

```
1. 确定入口指令（leader）：
   a) 第一条指令是入口指令
   b) 跳转指令的目标指令是入口指令
   c) 紧跟在跳转指令之后的指令是入口指令

2. 每个入口指令到下一个入口指令（不含）之间的指令构成一个基本块
```

### 2.3 流图（Control Flow Graph, CFG）

- **节点**：基本块
- **有向边**：表示基本块之间的控制转移关系
- **入口节点**：包含程序第一条指令的基本块

```
流图示例：
    B1 (入口)
   /  \
  B2   B3
   \  /
    B4
    |
    B5 (出口)
```

### 2.4 基本块的DAG表示

每个基本块可以表示为一个**有向无环图（DAG）**：
- 叶节点：标识变量的初始值
- 内部节点：表示运算
- 每个节点可附加一个变量名列表，表示在该点计算的值

DAG可以直观地暴露基本块内的公共子表达式和冗余计算。

## 3. 基本块内的优化

### 3.1 常量折叠（Constant Folding）

编译时计算常量表达式的值。

```
优化前:              优化后:
t1 = 3 + 5          t1 = 8
t2 = t1 * 2         t2 = 16
```

### 3.2 常量传播（Constant Propagation）

将已知为常量的变量值替换到其使用处。

```
优化前:              优化后:
x = 5               x = 5
y = x + 3           y = 8
z = x * y           z = 40
```

常量传播可能触发更多的常量折叠。

### 3.3 公共子表达式消除（Common Subexpression Elimination）

如果表达式 `x op y` 在之前已被计算且其操作数未被重新赋值，则可以复用之前的结果。

```
优化前:              优化后:
a = b + c           a = b + c
d = b + c           d = a    // 复用 a
```

### 3.4 复写传播（Copy Propagation）

将形如 `x = y` 的赋值传播到 `x` 的使用处，用 `y` 替换 `x`。

```
优化前:              优化后:
x = y               x = y
z = x + 3           z = y + 3
```

复写传播使得后续优化（如公共子表达式消除）更容易发现优化机会。

### 3.5 死代码消除（Dead Code Elimination）

删除对程序最终输出没有任何影响的代码。

```
优化前:              优化后:
x = 1               // x = 1 被删除
y = x + 2           // y = 3 被删除（如果y未被使用）
z = 5               z = 5
output(z)
```

判断变量"活跃"的方法：从输出回溯，标记所有被引用的变量，未被标记的即为死代码。

### 3.6 代数化简（Algebraic Simplification）

利用代数恒等式进行简化：

| 原表达式 | 化简结果 |
|----------|----------|
| `x + 0` | `x` |
| `x * 1` | `x` |
| `x * 0` | `0` |
| `x - x` | `0` |
| `x / x` | `1` |
| `x ** 2` | `x * x` |
| `x << 0` | `x` |

## 4. 数据流分析

数据流分析是进行全局优化的基础，通过在流图上传播信息来分析程序行为。

### 4.1 活跃变量分析（Liveness Analysis）

**问题**：对于变量 `x` 和程序点 `p`，`x` 在 `p` 是否活跃（将来是否会被使用）？

**用途**：寄存器分配（不活跃的变量无需占用寄存器）、死代码消除。

**数据流方程（向后分析）**：

```
LiveIn(B) = Use(B) ∪ (LiveOut(B) - Def(B))
LiveOut(B) = ∪{LiveIn(S) | S 是 B 的后继}
```

- `Use(B)`：在B中被使用且在使用前未被定值的变量集合
- `Def(B)`：在B中被定值的变量集合

**迭代算法**：
```
1. 初始化所有 LiveIn(B) 和 LiveOut(B) 为空集
2. 重复计算数据流方程直到不动点
```

### 4.2 可用表达式分析（Available Expression Analysis）

**问题**：对于表达式 `e` 和程序点 `p`，`e` 在 `p` 是否可用（从入口到 `p` 的所有路径上 `e` 都已被计算且其操作数未被重新定值）？

**用途**：全局公共子表达式消除。

**数据流方程（向前分析）**：

```
AvailIn(B) = ∩{AvailOut(P) | P 是 B 的前驱}    (B不是入口块时)
AvailOut(B) = (AvailIn(B) - Kill(B)) ∪ Gen(B)
```

- `Gen(B)`：B中生成的可用表达式（B中计算且操作数未在B中被重新定值的表达式）
- `Kill(B)`：B中杀死的可用表达式（B中对某个操作数进行了定值）

### 4.3 到达-定值分析（Reaching Definition Analysis）

**问题**：对于定值 `d`（即赋值语句）和程序点 `p`，`d` 是否能到达 `p`（即从 `d` 到 `p` 存在一条路径，且沿该路径 `d` 的目标变量未被重新定值）？

**用途**：常量传播、复写传播。

**数据流方程（向前分析）**：

```
ReachIn(B) = ∪{ReachOut(P) | P 是 B 的前驱}    (B不是入口块时)
ReachOut(B) = (ReachIn(B) - Kill(B)) ∪ Gen(B)
```

### 4.4 数据流分析的方向性

| 分析类型 | 传播方向 | 交汇运算 | 典型应用 |
|----------|---------|---------|---------|
| 活跃变量 | 向后 | ∪（并集） | 寄存器分配 |
| 可用表达式 | 向前 | ∩（交集） | 公共子表达式消除 |
| 到达-定值 | 向前 | ∪（并集） | 常量传播 |

## 5. 循环优化

循环是程序中最耗时的部分，循环优化能带来最大的性能提升。

### 5.1 循环的识别

**自然循环**满足：
1. 有唯一的入口节点（循环头）
2. 从循环外到循环内任意节点必须经过循环头

**支配节点（Dominator）**：节点 `d` 支配节点 `n`，当且仅当从入口到 `n` 的每条路径都经过 `d`。

**回边（Back Edge）**：`n → d` 是一条回边，当且仅当 `d` 支配 `n`。

**循环识别算法**：
1. 找出流图中的回边 `n → d`
2. 循环 = {d} ∪ {能不经过d到达n的节点}

### 5.2 循环不变量外提（Loop-Invariant Code Motion）

将循环内值不变的表达式提到循环外。

```
优化前:                    优化后:
while (i < n) {            t = x * y;
    t = x * y;             while (i < n) {
    a[i] = t + i;              a[i] = t + i;
    i++;                       i++;
}                          }
```

**条件**：
- 表达式的所有操作数在循环内未被重新定值
- 或者操作数的定值也是循环不变的
- 提到循环外是安全的（不会改变异常行为）

### 5.3 归纳变量消除（Induction Variable Elimination）

**归纳变量**：在循环中其值是循环控制变量的线性函数的变量。

```
优化前:                    优化后:
i = 0;                     i = 0;
while (i < 10) {           while (i < 10) {
    j = 3 * i;                 a[i] = i;  // 用i替代j
    a[j] = j;                  i++;
    i++;                   }
}
```

**强度削弱（Strength Reduction）**：将强度大的运算替换为强度小的运算。
- `j = 3 * i` → `j = j + 3`（用加法替代乘法）
- `x = i ** 2` 的相关优化

### 5.4 循环展开（Loop Unrolling）

将循环体复制多次，减少循环控制开销。

```
优化前:                    优化后（展开2次）:
for (i = 0; i < 4; i++) {  for (i = 0; i < 4; i += 2) {
    sum += a[i];               sum += a[i];
}                              sum += a[i+1];
                           }
```

**优点**：减少分支指令，增加指令级并行。
**缺点**：增大代码体积。

### 5.5 强度削弱（Strength Reduction）

将代价高的运算替换为代价低的运算：

| 原运算 | 替换为 | 条件 |
|--------|--------|------|
| `x * 2` | `x << 1` | 2的幂次 |
| `x * 9` | `(x << 3) + x` | 特殊常数 |
| `x / 2` | `x >> 1` | 无符号整数 |
| `x % 8` | `x & 7` | 2的幂次 |

## 6. 全局公共子表达式消除

### 6.1 基本思想

利用可用表达式分析的结果，检测不同基本块中的公共子表达式。

### 6.2 算法

```
对于每个基本块 B：
  对 B 中的每条赋值 x = y op z：
    如果 y op z 在 AvailIn(B) 中可用：
      则用已有的结果替换该计算
    否则：
      将 y op z 加入 Gen(B)
```

### 6.3 与基本块内消除的区别

- 基本块内：只需扫描块内指令
- 全局消除：需要数据流分析确定表达式是否在所有到达路径上都可用

## 7. 过程间优化简介

### 7.1 内联展开（Inlining）

将函数调用替换为函数体本身，消除调用开销。

```
优化前:                    优化后:
int add(int a, int b) {    // main中
    return a + b;          int x = a + b;  // 内联
}
int main() {
    int x = add(1, 2);
}
```

**优点**：消除调用开销，暴露更多优化机会。
**缺点**：增大代码体积，可能影响指令缓存命中率。

### 7.2 过程间常量传播

在调用图上传播常量值：
- 如果调用点的实参是常量，且被调用函数内该参数未被修改，则可将常量传播到被调用函数内部。

### 7.3 尾调用优化

如果函数的最后一条指令是调用另一个函数，可以复用当前的栈帧，将调用转为跳转。

### 7.4 过程间分析的挑战

- **别名分析**：两个指针可能指向同一内存位置
- **指针分析**：确定指针可能指向的内存位置集合
- **调用图构建**：处理函数指针和多态调用

## 8. 优化的顺序与迭代

### 8.1 优化遍的顺序

编译器通常执行多遍优化，常见顺序：

```
1. 构建流图，划分基本块
2. 基本块内优化（常量折叠、代数化简）
3. 数据流分析（到达-定值、活跃变量、可用表达式）
4. 全局优化（全局公共子表达式消除、复写传播）
5. 循环优化（循环不变量外提、归纳变量消除）
6. 死代码消除
7. 迭代上述过程直到不再有改进
```

### 8.2 优化之间的协同

- 常量折叠 → 常量传播 → 更多常量折叠
- 复写传播 → 公共子表达式消除 → 更多复写传播
- 死代码消除 → 简化流图 → 发现新的优化机会

---

## 参考与总结

代码优化是编译器提高目标代码性能的关键阶段。基本块内的局部优化相对简单，而全局优化和循环优化需要精确的数据流分析。数据流分析（活跃变量、可用表达式、到达-定值）是408考试的重中之重，必须掌握数据流方程的建立和迭代求解过程。循环优化（循环不变量外提、归纳变量消除）和强度削弱也是常考内容。
