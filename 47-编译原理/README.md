## 编译器总体架构

```
源程序 → [词法分析] → [语法分析] → [语义分析] → [中间代码生成]
    → [代码优化] → [目标代码生成] → 目标代码
```

### 各阶段简述

1. **词法分析（Lexical Analysis）**：将字符流转换为Token流，识别关键字、标识符、常量等。
2. **语法分析（Syntax Analysis / Parsing）**：根据文法规则将Token流组织成语法树（AST），检测语法错误。
3. **语义分析（Semantic Analysis）**：进行类型检查、作用域分析，收集语义信息。
4. **中间代码生成（Intermediate Code Generation）**：生成与机器无关的中间表示（IR），如三地址码。
5. **代码优化（Code Optimization）**：在中间代码层面进行等价变换以提高程序效率。
6. **目标代码生成（Code Generation）**：将中间代码转换为目标机器的汇编或机器码。

---

## 408考试要点提示

- **词法分析**：重点掌握NFA↔DFA转换、DFA最小化，常考构造题。
- **语法分析**：LL(1)和LR分析是必考内容，FIRST/FOLLOW/SELECT集的计算务必熟练。
- **语义分析**：属性文法与语法制导翻译是难点，回填技术需理解。
- **代码优化**：数据流分析（活跃变量、可用表达式）和循环优化是重点。
- **目标代码生成**：寄存器分配策略、窥孔优化为常考点。
