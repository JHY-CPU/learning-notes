# 03-语义分析与中间代码生成

> 语义分析阶段检查程序的语义正确性（如类型检查），并收集语义信息用于后续的中间代码生成。中间代码生成将源程序转换为与目标机器无关的中间表示（IR），为代码优化和目标代码生成提供基础。

---

## 1. 属性文法（Attribute Grammar）

### 1.1 基本概念

属性文法是在上下文无关文法的基础上，为文法符号附加**属性**，为产生式附加**语义规则**。

- **综合属性（Synthesized Attribute）**：从子节点向父节点传递信息。节点的综合属性值由其子节点的属性值计算得到。
- **继承属性（Inherited Attribute）**：从父节点或兄弟节点向子节点传递信息。

### 1.2 S-属性文法（S-attributed Grammar）

- **只包含综合属性**
- 可以在自底向上分析的同时自底向上计算属性值
- 实现简单，与LR分析兼容

### 1.3 L-属性文法（L-attributed Grammar）

- 允许综合属性和继承属性
- 继承属性的计算限制：产生式 `A → X1X2...Xn` 中，`Xj` 的继承属性只能依赖于：
  - `A` 的继承属性
  - `X1, X2, ..., Xj-1` 的属性（左侧兄弟）
- 可以在深度优先的自顶向下遍历中计算属性值
- 与LL(1)分析兼容

### 1.4 属性计算的例子

```
产生式                    语义规则
L → E n                  L.val = E.val    （综合属性）
E → E1 + T               E.val = E1.val + T.val
E → T                    E.val = T.val
T → T1 * F               T.val = T1.val * F.val
T → F                    T.val = F.val
F → ( E )                F.val = E.val
F → digit                F.val = digit.lexval
```

## 2. 语法制导翻译（Syntax-DDriven Translation）

### 2.1 基本思想

将语义动作（翻译方案）嵌入到文法产生式中，在语法分析的同时执行语义动作，完成翻译工作。

### 2.2 两种实现方式

**语法制导定义（SDD）**：声明式，给出属性和语义规则，不规定计算顺序。
**语法制导翻译方案（SDT）**：过程式，将语义动作嵌入产生式，明确动作执行位置。

### 2.3 动作的执行时机

- **自底向上（S-属性）**：在归约时执行动作
- **自顶向下（L-属性）**：在展开非终结符时执行动作

### 2.4 示例：表达式求值

```
输入: 3 + 5 * 2

产生式推导及属性计算:
E → E1 + T    { E.val = E1.val + T.val = 3 + 10 = 13 }
  → T + T     { T.val = 3 }
  → F + T     { F.val = 3 }
  → 3 + T     { 3 的 lexval = 3 }
  → 3 + T1 * F  { T.val = T1.val * F.val = 5 * 2 = 10 }
  → 3 + F * F   { T1.val = 5, F.val = 2 }
  → 3 + 5 * 2
```

## 3. 类型检查

### 3.1 类型检查的作用

- 验证运算符的操作数类型是否匹配
- 验证赋值语句两侧的类型是否兼容
- 验证函数调用的参数类型是否正确
- 检查数组下标是否为整型

### 3.2 类型等价

**结构等价（Structural Equivalence）**：
- 两个类型结构相同则等价
- `int[3]` 和 `int[3]` 等价，`int[3]` 和 `int[4]` 不等价

**名等价（Name Equivalence）**：
- 两个类型名字相同则等价
- `typedef int Length;` 中 Length 和 int 可能不等价（取决于语言定义）

### 3.3 类型转换（Coercion）

当操作数类型不匹配时，编译器可能自动进行类型转换：
- **隐式转换**：`int → float`，`char → int`
- **显式转换（强制类型转换）**：`(float) i`

类型转换规则通常按类型层次组织：
```
double ← float ← long ← int ← short ← char
```

### 3.4 类型表达式

类型可以用**类型表达式**来描述：
- 基本类型：`int`, `float`, `char`, `bool`
- 数组类型：`array(index_type, element_type)`
- 指针类型：`pointer(base_type)`
- 函数类型：`function(return_type, parameter_types)`
- 记录类型：`record(field_name: field_type, ...)`
- 乘积类型：`type1 × type2`

## 4. 声明语句的处理

### 4.1 变量声明

```
int x;
float y, z;
```

**处理过程**：
1. 遇到类型说明符（`int`, `float`），记录当前声明类型
2. 对每个变量名，在符号表中创建条目
3. 记录变量的类型、作用域、存储大小、偏移量等信息

### 4.2 数组声明

```
int a[10][20];
```

**处理过程**：
1. 计算数组元素类型
2. 计算数组总大小：`10 × 20 × sizeof(int)`
3. 计算各维的上下界
4. 在符号表中记录数组类型信息

### 4.3 函数/过程声明

```
int func(int a, float b) { ... }
```

**处理过程**：
1. 在符号表中创建函数条目
2. 记录返回类型、参数列表
3. 创建新的作用域，处理参数声明

## 5. 表达式的翻译

### 5.1 简单算术表达式

对于表达式 `E → E1 + T`：

```
E.place = newtemp()        // 生成新临时变量
emit(E.place, '=', E1.place, '+', T.place)  // 生成三地址码
```

示例：`a + b * c` 的翻译结果
```
t1 = b * c
t2 = a + t1
```

### 5.2 布尔表达式

布尔表达式可以翻译为跳转指令或布尔值：
- **数值编码**：用 0/1 表示 false/true
- **跳转编码**：用跳转目标地址表示（用于控制流语句）

## 6. 控制流语句的翻译

### 6.1 if-else 语句

```
if (E) S1 else S2
```

**翻译方案**：
```
E.true  = newlabel()    // E为真时跳转的标号
E.false = newlabel()    // E为假时跳转的标号
S1.next = S.next        // S1执行完后跳转的标号
S2.next = S.next

生成代码：
  [E的代码]
  if E.place goto E.true
  goto E.false
E.true:
  [S1的代码]
  goto S.next
E.false:
  [S2的代码]
S.next:
```

### 6.2 while 语句

```
while (E) S
```

**翻译方案**：
```
S.begin = newlabel()    // 循环开始标号
E.true  = newlabel()    // E为真时跳转到循环体
E.false = S.next        // E为假时跳出循环

生成代码：
S.begin:
  [E的代码]
  if E.place goto E.true
  goto S.next
E.true:
  [S的代码]
  goto S.begin
S.next:
```

### 6.3 for 语句

```
for (E1; E2; E3) S
```

等价变换为：
```
E1;
while (E2) {
    S;
    E3;
}
```

### 6.4 switch 语句

```
switch (E) {
    case C1: S1;
    case C2: S2;
    default: Sd;
}
```

**翻译方法**：
1. **嵌套if-else**：简单但效率低
2. **跳转表**：当case值连续时效率高
3. **二分查找+跳转表**：折中方案

## 7. 过程调用的翻译

### 7.1 调用序列与返回序列

```
调用序列（Call Sequence）：
  1. 计算实参值
  2. 保存调用者的状态（返回地址、寄存器）
  3. 将参数传递给被调用者
  4. 将控制转移到被调用者

返回序列（Return Sequence）：
  1. 将返回值放到约定位置
  2. 恢复调用者的状态
  3. 释放被调用者的栈帧
  4. 将控制返回给调用者
```

### 7.2 参数传递方式

| 方式 | 说明 | 示例 |
|------|------|------|
| **传值（Call by Value）** | 传递参数的副本 | C语言默认 |
| **传引用（Call by Reference）** | 传递参数的地址 | C++的 `&` |
| **传名（Call by Name）** | 将参数表达式原样替换到函数体中 | 宏展开类似 |
| **传结果（Call by Result）** | 返回时将结果写回实参 | Ada的 `out` 参数 |

### 7.3 活动记录（Activation Record）

函数调用时在栈上分配的内存块，包含：
- 参数区
- 返回地址
- 旧的帧指针（动态链）
- 局部变量区
- 临时变量区

## 8. 中间代码形式

### 8.1 后缀表达式（逆波兰表示）

将运算符放在操作数之后。

```
中缀: a + b * c
后缀: a b c * +
```

**优点**：无需括号，易于用栈求值。

### 8.2 三地址码（Three-Address Code）

每条指令最多包含一个运算符和三个地址（两个操作数，一个结果）。

#### 四元式（Quadruple）

```
(op, arg1, arg2, result)
```

示例：`x = a + b * c`
```
(*, b, c, t1)
(+, a, t1, t2)
(=, t2, _, x)
```

#### 三元式（Triple）

用位置（序号）代替临时变量名：
```
序号  (op, arg1, arg2)
0     (*,  b,  c)
1     (+,  a,  (0))
2     (=,  (1), x)   // 赋值给x
```

#### 间接三元式

用指针表引用三元式，便于代码优化时调整顺序：
```
间接表: [0, 1, 2]
三元式:
  0: (*, b, c)
  1: (+, a, (0))
  2: (=, (1), x)
```

### 8.3 静态单赋值形式（SSA）

- **每个变量只被赋值一次**
- 在控制流汇合点引入 **φ函数（Phi Function）**：

```
原始代码:              SSA形式:
x = 1;                 x1 = 1;
if (cond)              if (cond)
  x = 2;                 x2 = 2;
                       x3 = φ(x1, x2);
```

**优点**：简化数据流分析，便于优化。

## 9. 回填技术（Backpatching）

### 9.1 问题背景

翻译布尔表达式和控制流语句时，需要生成跳转指令，但跳转目标标号在翻译时尚未确定（例如 `if` 语句中跳过 `S1` 的跳转目标在 `S1` 之后才确定）。

### 9.2 回填的基本思想

1. 当生成跳转指令但目标未知时，将该指令的标号位置记录在一个**链表**中（称为"真链"或"假链"）
2. 当目标标号确定后，回过头来填补这些指令中的跳转目标

### 9.3 布尔表达式的回填

对于 `E → E1 or E2`：
```
1. E1 为真时，直接跳到 E 的真出口
2. E1 为假时，需要回填到 E2 的开始位置
3. E2 的真链合并到 E 的真链
4. E2 的假链成为 E 的假链
```

对于 `E → E1 and E2`：
```
1. E1 为真时，需要回填到 E2 的开始位置
2. E1 为假时，直接跳到 E 的假出口
3. E2 的真链成为 E 的真链
4. E2 的假链合并到 E 的假链
```

### 9.4 回填的数据结构

使用链表存储待回填的指令地址：
- `makelist(i)`：创建只含指令 `i` 的链表
- `merge(p1, p2)`：合并两个链表
- `backpatch(p, i)`：将链表 `p` 中所有指令的跳转目标设为标号 `i`

### 9.5 示例

翻译 `if (a < b || c < d) x = 1;`：

```
(1) if a < b goto (4)     // E1.true: 回填到 (4)
    goto (2)              // E1.false: 指向 E2 开始
(2) if c < d goto (4)     // E2.true
    goto (next)           // E2.false
(4) x = 1                 // 回填目标
(next):
```

## 参考与总结

语义分析与中间代码生成是连接语法分析和代码优化的桥梁。属性文法提供了形式化的语义描述框架，语法制导翻译是核心翻译方法。中间代码的形式（后缀表达式、三地址码、SSA）直接影响后续优化的效率。408考试中，属性计算、语法制导翻译方案、回填技术是重点内容，需要掌握手工翻译过程。
