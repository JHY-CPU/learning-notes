01 - 语义分析与符号表

一、语义分析概述

语义分析（Semantic Analysis）是编译器在语法分析之后的阶段，负责检查程序的语义正确性，并收集类型信息用于后续的代码生成。

1.1 语义分析的主要任务

  • 类型检查（Type Checking）：验证操作数类型是否匹配运算符要求
  • 作用域分析（Scope Analysis）：确定标识符的声明与引用关系
  • 控制流检查：break/continue/return 是否出现在合法位置
  • 唯一性检查：同一作用域内标号和枚举值是否唯一
  • 类型推导（Type Inference）：自动推断表达式的类型

1.2 属性文法（Attribute Grammar）

属性文法在CFG基础上为每个文法符号关联属性，用语义规则计算属性值：

  • 综合属性（Synthesized）：由子节点属性计算父节点属性（自底向上传递）
  • 继承属性（Inherited）：由父节点或兄弟节点属性计算子节点属性（自顶向下/侧向传递）

二、类型检查

2.1 类型表达式

类型检查涉及将类型表示为类型表达式：

类型表示方式示例
基本类型integer, float, boolean, charint x;
数组类型array(indexType, elementType)int[10] a;
记录类型record(field1:type1, ...)struct { int x; float y; }
指针类型pointer(type)int* p;
函数类型type1 × ... × typen → returnTypeint(int,int)

2.2 类型等价

  • 结构等价（Structural Equivalence）：类型表达式结构完全相同则等价
  • 名字等价（Name Equivalence）：同一类型声明的名字相同才等价

2.3 类型推导规则

算术运算 e1 + e2：若 e1:T, e2:T, T ∈ {integer, float}，则结果类型为 T

赋值 x = e：需 T(x) 与 T(e) 类型兼容（或可隐式转换）

数组访问 a[i]：需 a: array(Tindex, Telem), i: Tindex，结果类型 Telem

函数调用 f(a1,...,an)：需参数类型与函数声明的参数类型逐个匹配

四、符号表（Symbol Table）

符号表是编译器中用于存储标识符及其属性信息的数据结构，是语义分析和代码生成的基础。

4.1 符号表存储的信息

属性说明示例
名称（Name）标识符字符串count
类别（Kind）变量/函数/类型/常量variable
类型（Type）类型表达式int, float[10]
作用域（Scope）声明所在作用域global, local-2
存储类别static/extern/registerauto
内存地址/偏移运行时存储位置offset=12
维度信息数组的维度和大小[3][4]
参数信息函数参数类型列表(int, float)

4.2 符号表的数据结构

4.2.1 哈希表（Hash Table）

最常用的方式，支持 O(1) 平均查找。

struct SymbolEntry { string name; SymbolKind kind; Type* type; int scope; int offset; SymbolEntry* next; // 解决冲突的链表 }; // 哈希表实现 SymbolEntry* hashTable[TABLE_SIZE]; int hash(string name) { int h = 0; for (char c : name) h = (h * 31 + c) % TABLE_SIZE; return h; }

4.2.2 树结构（Scope Tree）

用树表示嵌套作用域，每个节点对应一个作用域。

[全局作用域] / | \ [foo] [bar] [main] / \ [if] [for]

4.2.3 作用域栈 + 哈希表（实际常用方案）

维护一个作用域栈，每个作用域对应一个哈希表。查找时从栈顶（当前作用域）向下搜索。

// 作用域栈实现 vector> scopeStack; void enterScope() { scopeStack.push_back({}); } void exitScope() { scopeStack.pop_back(); } SymbolEntry* lookup(string name) { // 从内层向外层查找 for (int i = scopeStack.size()-1; i >= 0; i--) { if (scopeStack[i].count(name)) return scopeStack[i][name]; } return nullptr; // 未找到 } void insert(string name, SymbolEntry* entry) { scopeStack.back()[name] = entry; // 插入当前作用域 }

五、隐式类型转换与类型强制

5.1 类型转换层次

常见语言中的隐式类型转换规则（widening coercion）：

char → short → int → long → float → double （窄类型自动向宽类型转换，不会丢失信息）

5.2 类型强制（Coercion）

编译器在表达式求值时自动插入类型转换指令：

int i; float f;

f + i → 编译器生成 f + (float)i

注意：C语言中的隐式转换规则宽松容易出错，现代语言如Go、Rust要求显式转换。
