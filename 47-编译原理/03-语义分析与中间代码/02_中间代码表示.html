02 - 中间代码表示（IR）

一、中间代码（IR）的作用

中间代码是介于源代码和目标机器代码之间的独立于具体硬件的表示形式。

1.1 使用中间代码的好处

  • 可移植性：只需修改后端即可支持新目标平台
  • 优化便利：在IR上做机器无关优化更容易
  • 结构清晰：比源码更接近机器指令，便于分析
  • 多语言支持：不同前端可生成相同的IR
源代码(C/Java/...) │ 前端(Frontend) ▼ ┌──────────────┐ │ 中间代码(IR) │ ← 机器无关优化在此进行 └──────┬───────┘ │ 后端(Backend) ▼ 目标机器代码(x86/ARM/...)

二、三地址码（Three-Address Code, TAC）

三地址码是最常见的IR形式，每条指令最多有一个运算符和三个地址（两个操作数，一个结果）。

2.1 基本形式

x = y op z 或 x = op y

2.2 常见的三地址指令

指令类型格式示例
赋值x = y op zt1 = a + b
一元运算x = op yt2 = -t1
复制x = yx = t2
无条件跳转goto Lgoto L1
条件跳转if x goto Lif t1 goto L2
条件跳转（比较）if x relop y goto Lif a < b goto L3
索引赋值x = y[i] / x[i] = yt3 = a[t1]
取地址/解引用x = &y / x = *yp = &a
函数调用param x; call f, ncall foo, 2

2.3 源代码到三地址码的翻译示例

源代码：

result = (a + b) * (c - d) / e;


三地址码：

t1 = a + b t2 = c - d t3 = t1 * t2 t4 = t3 / e result = t4

三、四元组与三元组

3.1 四元组（Quadruple）

用四个字段表示：(op, arg1, arg2, result)

源代码：x = a + b * c

#oparg1arg2result
0*bct1
1+at1t2
2=t2x

3.2 三元组（Triple）

省略结果字段，用指令的序号作为结果的引用：(op, arg1, arg2)

源代码：x = a + b * c

#oparg1arg2
0*bc
1+a(0)
2=(1)x

其中 (0) 表示引用第0条三元组的结果

3.3 间接三元组（Indirect Triple）

增加一个指令指针数组，通过重排指针实现指令调度而不移动三元组本身。

四、静态单赋值形式（SSA）

SSA（Static Single Assignment）要求每个变量只被赋值一次，每次赋值创建一个新版本。

4.1 SSA 的特点

普通三地址码：

x = 1 x = x + 2 y = x * 3

SSA 形式：

x1 = 1 x2 = x1 + 2 y1 = x2 * 3

4.2 φ 函数（Phi Function）

当控制流汇合时，不同路径可能为同一变量赋予不同值，用 φ 函数选择正确的版本：

if (cond) x1 = a + 1 ──────┐ else │ x2 = b + 2 ──────┤ ▼ x3 = φ(x1, x2) y1 = x3 * 2
φ 函数的语义：如果控制流从 true 分支到达，x3 = x1；如果从 false 分支到达，x3 = x2。

4.3 SSA 的优势

  • 简化数据流分析：每个定义唯一标识，无需复杂的到达定义分析
  • 便于常量传播和死代码消除
  • 利于寄存器分配优化
  • LLVM IR 就是基于SSA形式的

五、从 AST 翻译到 IR

5.1 表达式的翻译

// 递归翻译表达式 string translateExpr(ASTNode* node) { if (node->isLeaf()) return node->name; string left = translateExpr(node->left); string right = translateExpr(node->right); string temp = newTemp(); // 生成临时变量 t1, t2, ... emit(temp + " = " + left + " " + node->op + " " + right); return temp; }

5.2 控制流语句的翻译

使用回填（Backpatching）技术处理跳转目标：

// if语句翻译 void translateIf(ASTNode* node) { string trueLabel = newLabel(); string falseLabel = newLabel(); string endLabel = newLabel(); translateCond(node->cond, trueLabel, falseLabel); emitLabel(trueLabel); translateStmt(node->thenBranch); emit("goto " + endLabel); emitLabel(falseLabel); translateStmt(node->elseBranch); emitLabel(endLabel); }

5.3 控制流图（CFG）

三地址码可以组织成基本块（Basic Block）的有向图：

  • 基本块：只有一个入口和一个出口的最长指令序列
  • 基本块之间通过跳转/分支连接
  • 控制流图是后续优化和寄存器分配的基础结构
