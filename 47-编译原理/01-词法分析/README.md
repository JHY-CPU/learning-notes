# 01-词法分析（Lexical Analysis）

> 词法分析是编译过程的第一个阶段，其任务是将源程序的字符流转换为有意义的Token序列（记号流），为后续语法分析提供输入。

---

## 1. 词法分析器的作用与接口

### 1.1 主要功能

- **读入字符流**：从源文件中逐个读取字符。
- **识别单词**：根据构词规则将字符组合成有意义的单词（Token）。
- **过滤空白与注释**：跳过空格、换行符、制表符和注释。
- **构建符号表**：将标识符和常量的相关信息填入符号表。
- **错误处理**：报告词法错误（如非法字符、不合法的字符串常量等）。

### 1.2 与其他阶段的接口

```
源程序(字符流) → 词法分析器 → Token流 → 语法分析器
                     ↓
                  符号表
```

- **输入**：源程序字符流
- **输出**：Token序列，每个Token通常用二元组表示：`<Token类别, 属性值>`
- **副作用**：填写/查询符号表

### 1.3 词法分析器的独立性

词法分析可以作为独立的一遍（pass），也可以与语法分析合并。独立的优点：
- 简化编译器设计
- 提高编译效率（使用专门的缓冲技术）
- 可移植性增强

## 2. Token的分类

### 2.1 Token的基本类别

| 类别 | 说明 | 示例 |
|------|------|------|
| **关键字（Keyword）** | 语言保留的具有特殊含义的单词 | `if`, `while`, `int`, `return` |
| **标识符（Identifier）** | 用户定义的变量名、函数名等 | `count`, `main`, `temp_var` |
| **常量（Constant）** | 整型、浮点型、字符型、字符串常量 | `123`, `3.14`, `'A'`, `"hello"` |
| **运算符（Operator）** | 算术、关系、逻辑、赋值运算符等 | `+`, `==`, `&&`, `=` |
| **界符（Delimiter）** | 分隔符、括号等 | `;`, `,`, `(`, `)`, `{`, `}` |

### 2.2 Token的表示

每个Token用一个二元式表示：

```
<token_type, attribute_value>
```

- **token_type**：Token的类别编码（如 KEYWORD、ID、NUMBER 等）
- **attribute_value**：Token的属性值。对于标识符，是指向符号表条目的指针；对于常量，是常量的值或指向常量表的指针。

### 2.3 关键字与标识符的区分

关键字是标识符的特殊情况。处理策略：
1. **保留字表法**：维护一张关键字表，识别出标识符后查表判断是否为关键字。
2. **单独识别法**：为每个关键字写单独的识别规则（正则表达式），优先匹配关键字。

---

## 3. 正则表达式描述词法规则

### 3.1 基本概念

正则表达式（Regular Expression, RE）是描述词法规则的最常用形式化工具。它能精确地定义一类字符串的集合，即正则语言（Regular Language）。

### 3.2 正则表达式的代数定律

| 定律 | 表达式 |
|------|--------|
| 交换律 | `r \| s = s \| r` |
| 结合律 | `r\|(s\|t) = (r\|s)\|t`，`(rs)t = r(st)` |
| 分配律 | `r(s\|t) = rs \| rt`，`(r\|t)s = rs \| ts` |
| 幂等律 | `r* = (r*)*` |
| 恒等律 | `εr = r`，`rε = r`，`r\|ε = r`（部分教材将ε视为空） |
| 零律 | `∅r = ∅`，`r∅ = ∅`，`r\|∅ = r` |
| 幂等闭包 | `r* = r \| rr* = ε \| r \| rr \| ...` |
| 正闭包 | `r+ = rr* = r*r` |

### 3.3 示例：常见Token的正则定义

```
letter   → [a-zA-Z_]
digit    → [0-9]
id       → letter (letter | digit)*
number   → digit+ (. digit+)? (E [+-]? digit+)?
keyword  → if | else | while | for | return | int | float | ...
operator → + | - | * | / | = | == | != | <= | >= | && | ||
```

## 4. 正则表达式 → NFA（Thompson构造法）

### 4.1 基本思想

Thompson构造法是一种从正则表达式构造NFA的系统方法。其核心思想是递归地将正则表达式分解为子表达式，为每个子表达式构造NFA片段，再将片段组合。

### 4.2 基本构造规则

**基础情形：**
- 对于单个字符 `a`：构造NFA，包含两个状态，一条标记为 `a` 的转移边。
- 对于 `ε`：构造两个状态，一条标记为 `ε` 的转移边。

**归纳情形：**
- **并运算 `r|s`**：新增初始状态和接受状态，通过ε转移分别连接到 `r` 和 `s` 的NFA。
- **连接运算 `rs`**：将 `r` 的接受状态与 `s` 的初始状态合并（或通过ε转移连接）。
- **闭包运算 `r*`**：新增初始状态和接受状态，添加ε转移实现零次或多次匹配。

### 4.3 Thompson构造法的特点

- 产生的NFA状态数 = 正则表达式中符号和运算符的总数
- 每个状态最多有两条ε出边或一条非ε出边
- 构造过程保证NFA是确定性的结构

## 5. NFA → DFA（子集构造法）

### 5.1 动机

NFA在运行时存在不确定性（一个状态对同一输入符号可有多条转移边），需要回溯或并行模拟，效率低。将其转换为等价的DFA可以消除不确定性。

### 5.2 子集构造法（Subset Construction）

核心思想：DFA的每个状态对应NFA的一个**状态集合**。

**算法步骤：**

```
1. DFA的初始状态 = ε-closure(NFA的初始状态)
2. 对于DFA中每个未标记的状态S（NFA状态集合）：
   对于每个输入符号a：
     T = ε-closure(move(S, a))
     如果T不在DFA中，将T作为新的DFA状态
     添加DFA转移边 S --a--> T
3. 包含NFA接受状态的DFA状态为DFA的接受状态
```

### 5.3 ε-closure的计算

- `ε-closure(s)` = 从状态 `s` 出发，仅通过ε转移能到达的所有状态的集合（含 `s` 自身）
- 对集合 `S`：`ε-closure(S) = ∪{ε-closure(s) | s ∈ S}`
- 使用DFS或BFS实现

### 5.4 状态爆炸问题

子集构造法在最坏情况下，DFA的状态数可达 $2^n$（n为NFA状态数）。但实践中，由于正则表达式的结构特性，实际产生的状态数通常远小于理论上限。

## 6. DFA的最小化（Hopcroft算法）

### 6.1 最小DFA的唯一性

对于任意正则语言，存在唯一的（同构意义下）状态数最少的DFA，称为**最小DFA**。

### 6.2 Hopcroft最小化算法

**基本思想**：对DFA的状态集合进行不断划分（partition refinement），使得同一等价类中的状态行为完全相同。

**算法步骤：**

```
1. 初始化划分 P = {F, S-F}（接受状态集、非接受状态集）
2. 重复以下过程，直到P不再变化：
   对P中的每个组G：
     对每个输入符号a：
       对G中的状态进行细分：
         如果两个状态p,q在输入a下转移到不同组，则将G分裂
3. 每个最终的等价类选一个代表状态
4. 构建最小DFA
```

### 6.3 不可区分状态

两个状态 `s` 和 `t` 等价（不可区分），当且仅当：
- 对于任意输入串 `w`，从 `s` 出发读入 `w` 到达接受状态，当且仅当从 `t` 出发读入 `w` 到达接受状态。

### 6.4 408考试注意点

- 手工构造最小DFA时，通常先画出状态转换表，然后逐步合并等价状态。
- 注意死状态（对所有输入都转移到自身、且为非接受状态）的处理。

## 7. 词法分析器生成器（Lex/Flex原理）

### 7.1 Lex的工作流程

```
源程序(.l) → Lex/Flex → C源程序 → C编译器 → 词法分析器（可执行文件）
```

### 7.2 Lex源文件的结构

```lex
%{
  /* C声明部分：头文件、全局变量定义 */
%}

/* 正则定义部分 */
letter   [a-zA-Z_]
digit    [0-9]
id       {letter}({letter}|{digit})*

%%
  /* 规则部分：模式 { 动作 } */
{digit}+       { return NUMBER; }
{letter}+      { return IDENTIFIER; }
"if"           { return IF; }
"+"            { return PLUS; }
[ \t\n]        { /* 跳过空白 */ }
.              { printf("非法字符: %c\n", yytext[0]); }

%%
  /* C辅助函数部分 */
int yywrap() { return 1; }
```

### 7.3 内部实现原理

1. 根据正则表达式构造NFA
2. 将NFA转换为DFA（子集构造法）
3. 最小化DFA
4. 生成DFA驱动表（转换表）
5. 运行时通过查表实现状态转换
6. 最长匹配原则 + 优先级规则解决歧义

## 8. 符号表的初步构建

### 8.1 符号表的作用

- 记录标识符的名称、类型、作用域、存储位置等信息
- 在词法分析阶段进行初步填写
- 在语义分析阶段进行完善

### 8.2 符号表的数据结构

常用实现方式：
- **线性表**：简单但查找效率低，O(n)
- **哈希表**：查找效率高，O(1)平均，408考试中最常考
- **树结构**：平衡BST，O(log n)
- **栈式符号表**：支持嵌套作用域

### 8.3 词法分析阶段的符号表操作

1. 识别标识符后，查符号表
2. 若已存在，返回已有条目的指针
3. 若不存在，创建新条目，返回新条目的指针

## 9. 超前扫描（Lookahead）

### 9.1 为什么需要超前扫描

某些情况下，仅凭当前字符无法确定Token类型，需要向前查看若干字符。

### 9.2 典型场景

- **关键字 vs 标识符**：`if` 是关键字还是变量名 `iffy` 的前缀？
- **`=` vs `==`**：读到 `=` 时需要看下一个字符
- **`>` vs `>>` vs `>=`**
- **注释 `/*`**：读到 `/` 后需要判断是否为注释开始

### 9.3 超前扫描策略

- 最多超前扫描一个字符（1个字符的Lookahead）通常足够
- 使用指针回退机制：匹配完成后将多余字符放回输入流

## 10. 例题：构造识别特定模式的DFA

### 例题1：构造识别二进制串（以0结尾）的DFA

**题目**：设计一个DFA，识别所有以0结尾的二进制串。

**状态定义**：
- `q0`：初始状态，尚未读入字符或最近读入的字符是1
- `q1`：最近读入的字符是0（接受状态）

**转换函数**：
| 状态 | 输入0 | 输入1 |
|------|-------|-------|
| q0   | q1    | q0    |
| q1   | q1    | q0    |

接受状态集：`{q1}`

### 例题2：NFA转DFA

**题目**：给定正则表达式 `(a|b)*abb` 对应的NFA，用子集构造法转换为DFA。

**步骤**：
1. 计算NFA初始状态的ε-closure，得到DFA初始状态A
2. 对A计算 `move(A, a)` 和 `move(A, b)` 的ε-closure
3. 对新产生的状态重复上述过程
4. 包含NFA接受状态的DFA状态标记为接受状态

### 例题3：DFA最小化

**题目**：给定一个DFA，用Hopcroft算法求其最小DFA。

**步骤**：
1. 初始划分 P = {接受状态组, 非接受状态组}
2. 检查每组内的状态是否行为一致
3. 如不一致，进行分裂
4. 重复直到不再分裂
5. 每组取一个代表，构造最小DFA

---

## 参考与总结

词法分析是编译器的第一个阶段，核心工具是**有限自动机**。从正则表达式出发，经过Thompson构造法得到NFA，经子集构造法转为DFA，再通过Hopcroft算法最小化DFA，形成高效的词法分析引擎。408考试中，NFA与DFA的互转、DFA最小化是高频考点，务必熟练掌握手工构造过程。
