# 词法分析器实现 - 编译原理笔记


词法分析器（Lexer / Scanner）是编译器的**第一个阶段**，其任务是将源程序的字符流转换为**记号流（Token Stream）**。


```
源代码字符流
  │
  ▼
┌──────────────┐    ┌──────────────┐
│  词法分析器   │───▶│  记号流      │
│  (Lexer)     │    │  (Tokens)    │
└──────────────┘    └──────┬───────┘
                           │
                           ▼
                    ┌──────────────┐
                    │  语法分析器   │
                    │  (Parser)    │
                    └──────────────┘
```


### 1.1 词法分析器的主要职责


- 识别并产生各类
   **记号**
   （Token）
- 过滤
   **空白符**
   和
   **注释**
- 跟踪
   **行号**
   和
   **列号**
   ，用于错误报告
- 将标识符填入
   **符号表**
- 发现
   **词法错误**
   （如非法字符、未闭合的字符串）


### 2.1 记号的分类


| 类别 | 示例 | 正则表达式 |
| --- | --- | --- |
| 关键字 | `if, else, while, return` | 预定义列表匹配 |
| 标识符 | `count, myVar, x1` | `letter(letter\|digit)*` |
| 常量 | `42, 3.14, 'A'` | `digit+('.'digit+)?` |
| 运算符 | `+, -, *, ==, <=, &&` | 逐个枚举 |
| 分隔符 | `(, ), {, }, ;, ,` | 逐个枚举 |
| 空白符 | 空格、制表符、换行 | `( \|\t\|\n)+` |
| 注释 | `// ...`, `/* ... */` | 特定模式 |


### 2.2 记号的数据结构


```
struct Token {
    TokenType type;    // 记号类型：KEYWORD, ID, NUMBER, OP, ...
    string lexeme;     // 实际的字符串（源码中的原始文本）
    int line;          // 所在行号
    int column;        // 所在列号
    union {
        int intVal;    // 整型常量的值
        double realVal; // 浮点常量的值
        string* strVal; // 字符串常量的值
    };
};
```


### 3.1 双缓冲区（Two-Buffer Scheme）


当词法分析器需要**超前搜索**（lookahead）多个字符来确定记号边界时，使用双缓冲区避免频繁的I/O操作。


```
┌─────────────────────────┬─────────────────────────┐
│      缓冲区1 (half1)    │      缓冲区2 (half2)    │
│                         │                         │
│  当 lexeme 指针跨越     │  EOF 标记位于缓冲区     │
│  到另一半时，读取新数据  │  末尾 sentinel          │
└─────────────────────────┴─────────────────────────┘
         ▲
     lexemeBegin ──────── forward
     (记号起点)        (当前扫描位置)
```


### 3.2 超前搜索策略


某些语言的记号无法仅凭当前字符确定，需要"向前看"若干字符：


- **关键字 vs 标识符**
   ：
   `whilex`
   是标识符而非关键字
   `while`
- **浮点数 vs 范围运算符**
   ：
   `3.14`
   vs
   `3..10`
- **注释开始**
   ：
   `/`
   是除法还是
   `/*`
   注释的开始


### 3.3 超前搜索的实现


```
// 超前搜索伪代码
while (true) {
    c = getNextChar();
    if (c matches current token pattern) {
        forward++;
    } else {
        // 回退一个字符
        retract(forward);
        // 输出当前记号
        emitToken(type, lexemeBegin..forward);
        break;
    }
}
```


### 4.1 两种实现方式


| 方式 | 描述 | 优点 | 缺点 |
| --- | --- | --- | --- |
| 单独的DFA状态 | 每个关键字有独立的自动机路径 | 识别速度快 | 状态表膨胀 |
| 符号表预填入 | 先按标识符规则识别，再查符号表判断 | 简洁，易于添加新关键字 | 多一次查表操作 |

**推荐方式**
：在初始化阶段将所有关键字填入
**符号表/哈希表**
，词法分析器先按标识符规则扫描，然后查表判断是否为关键字。这是大多数实际编译器（如GCC、LLVM）采用的策略。

### 5.1 工具对比


| 工具 | 输入 | 输出 | 典型应用 |
| --- | --- | --- | --- |
| **Lex / Flex** | 正则表达式规则文件 (.l) | C代码的词法分析器 | Unix工具链 |
| **ANTLR** | 文法规则文件 (.g4) | Java/C++/Python词法器 | 多种语言项目 |
| **JFlex** | 正则规则 (.jflex) | Java词法分析器 | Java编译器 |


### 5.2 Lex/Flex 规则文件结构


```
%{
/* C声明区：头文件、全局变量等 */
#include "y.tab.h"
int line_num = 1;
%}

/* Flex定义区 */
DIGIT   [0-9]
LETTER  [a-zA-Z]
ID      {LETTER}({LETTER}|{DIGIT})*

%%  /* 规则分隔符 */

"if"        { return IF; }
"else"      { return ELSE; }
"while"     { return WHILE; }
{ID}        { yylval.str = strdup(yytext); return IDENTIFIER; }
{DIGIT}+    { yylval.num = atoi(yytext); return NUMBER; }
[ \t]+      { /* 跳过空白 */ }
\n          { line_num++; }
"//".*      { /* 单行注释，跳过 */ }
.           { fprintf(stderr, "非法字符: %c\n", *yytext); }

%%  /* 用户代码区 */
int yywrap() { return 1; }
```


### 6.1 直接编码法（Hand-Coded Scanner）


不依赖自动生成工具，直接用程序语言编写状态机。


```
Token nextToken() {
    // 1. 跳过空白和注释
    while (current == ' ' || current == '\t' || current == '\n')
        advance();
    if (current == '/' && peek() == '/') {
        while (current != '\n') advance();
        return nextToken(); // 递归继续
    }

    // 2. 识别标识符或关键字
    if (isLetter(current)) {
        StringBuilder sb = new StringBuilder();
        while (isLetterOrDigit(current)) {
            sb.append(current); advance();
        }
        String word = sb.toString();
        if (keywords.containsKey(word))
            return new Token(KEYWORD, word);
        else
            return new Token(IDENTIFIER, word);
    }

    // 3. 识别数字
    if (isDigit(current)) {
        StringBuilder sb = new StringBuilder();
        while (isDigit(current)) { sb.append(current); advance(); }
        if (current == '.') { sb.append(current); advance();
            while (isDigit(current)) { sb.append(current); advance(); }
            return new Token(FLOAT, sb.toString());
        }
        return new Token(INTEGER, sb.toString());
    }

    // 4. 识别运算符和分隔符
    // ...（逐个匹配）
}
```
