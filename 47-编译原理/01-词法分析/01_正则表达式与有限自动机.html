01 - 正则表达式与有限自动机

一、正则表达式基础

正则表达式（Regular Expression）是描述正则语言的代数表示方法，是词法分析器用来匹配记号（Token）模式的核心工具。

1.1 正则表达式的定义

设字母表为 Σ，正则表达式递归定义如下：

  1. 基础情形：空串 ε 和空集 ∅ 是正则表达式
  2. 单字符：对任意 a ∈ Σ，a 是正则表达式
  3. 并运算：若 r、s 是正则表达式，则 r|s 是正则表达式
  4. 连接运算：若 r、s 是正则表达式，则 rs 是正则表达式
  5. 闭包运算：若 r 是正则表达式，则 r* 是正则表达式（Kleene闭包）

1.2 运算优先级

优先级运算符含义结合性
1（最高）*Kleene闭包（0次或多次）左结合
2连接（隐式）拼接两个正则表达式左结合
3（最低）|选择（或）左结合

1.3 常见编程语言Token的正则表达式

标识符 letter (letter | digit)*

无符号整数 digit digit* 或 digit+

关键字 if | else | while | for | return

运算符 + | - | * | / | == | != | <= | >=

空白符 (blank | tab | newline)+

三、有限自动机（Finite Automata）

有限自动机是识别正则语言的计算模型，分为确定性有限自动机（DFA）和非确定性有限自动机（NFA）两种。

3.1 DFA 的形式化定义

DFA 是一个五元组 M = (Q, Σ, δ, q₀, F)：

  • Q — 有限状态集合
  • Σ — 有限输入字母表
  • δ — 转移函数：Q × Σ → Q（唯一确定）
  • q₀ — 初始状态，q₀ ∈ Q
  • F — 接受状态集合，F ⊆ Q

3.2 NFA 的形式化定义

NFA 与 DFA 类似，但转移函数为 δ: Q × (Σ ∪ {ε}) → P(Q)

  • 同一输入符号可以转移到多个状态
  • 允许ε-转移（不消耗输入的转移）

3.3 DFA 与 NFA 的对比

特性DFANFA
转移函数Q × Σ → QQ × (Σ ∪ {ε}) → P(Q)
ε-转移不允许允许
执行方式每个输入只有一条路径多条路径并行（概念上）
识别效率O(n)，每个字符处理一次模拟需 O(n·m²)
状态数可能指数级增长通常较少
表达能力等价 — 均能识别正则语言

五、子集构造法（NFA → DFA）

子集构造法（Subset Construction / Powerset Construction）是将 NFA 转换为等价 DFA 的经典算法。

5.1 ε-闭包

ε-closure(S) = 从集合 S 中任一状态出发，通过任意条 ε-转移所能到达的所有状态的集合。

5.2 算法步骤

  1. DFA 的初始状态为 ε-closure(NFA的q₀)
  2. 对 DFA 中每个未处理的状态集 S，对每个输入符号 a：
    • 计算 move(S, a) = 从 S 中状态经 a 转移可达的状态集
    • 计算 ε-closure(move(S, a)) 作为 DFA 的新状态
  3. 包含 NFA 任意接受状态的状态集为 DFA 的接受状态
  4. 重复直到没有新状态产生
最坏情况：NFA 有 n 个状态时，DFA 最多有 2n 个状态（指数级增长）。

六、DFA 最小化

6.1 Hopcroft 最小化算法

目标：找到状态数最少的等价 DFA。核心思想是合并不可区分的状态。

6.2 等价状态与可区分状态

  • 可区分：存在某个字符串 w，使得从 p 和 q 出发，一个到达接受状态，另一个到达非接受状态
  • 等价：对所有字符串 w，从 p 和 q 出发同时接受或同时拒绝

6.3 填表法（Table-Filling Algorithm）

  1. 标记所有"接受状态-非接受状态"对为可区分
  2. 对每对未标记的状态 (p, q)，检查所有输入 a：
    • 若 δ(p,a) 和 δ(q,a) 已被标记为可区分，则标记 (p,q) 为可区分
  3. 重复直到无新标记产生
  4. 未被标记的所有状态对合并为等价类
