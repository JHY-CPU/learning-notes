02 - 词法分析器实现

一、词法分析器在编译器中的角色

词法分析器（Lexer / Scanner）是编译器的第一个阶段，其任务是将源程序的字符流转换为记号流（Token Stream）。

源代码字符流 │ ▼ ┌──────────────┐ ┌──────────────┐ │ 词法分析器 │───▶│ 记号流 │ │ (Lexer) │ │ (Tokens) │ └──────────────┘ └──────┬───────┘ │ ▼ ┌──────────────┐ │ 语法分析器 │ │ (Parser) │ └──────────────┘

1.1 词法分析器的主要职责

  • 识别并产生各类记号（Token）
  • 过滤空白符和注释
  • 跟踪行号和列号，用于错误报告
  • 将标识符填入符号表
  • 发现词法错误（如非法字符、未闭合的字符串）

二、记号（Token）的类型与结构

2.1 记号的分类

类别示例正则表达式
关键字if, else, while, return预定义列表匹配
标识符count, myVar, x1letter(letter|digit)*

2.2 记号的数据结构

struct Token { TokenType type; // 记号类型：KEYWORD, ID, NUMBER, OP, ... string lexeme; // 实际的字符串（源码中的原始文本） int line; // 所在行号 int column; // 所在列号 union { int intVal; // 整型常量的值 double realVal; // 浮点常量的值 string* strVal; // 字符串常量的值 }; };

三、输入缓冲与超前搜索

3.1 双缓冲区（Two-Buffer Scheme）

当词法分析器需要超前搜索（lookahead）多个字符来确定记号边界时，使用双缓冲区避免频繁的I/O操作。

┌─────────────────────────┬─────────────────────────┐ │ 缓冲区1 (half1) │ 缓冲区2 (half2) │ │ │ │ │ 当 lexeme 指针跨越 │ EOF 标记位于缓冲区 │ │ 到另一半时，读取新数据 │ 末尾 sentinel │ └─────────────────────────┴─────────────────────────┘ ▲ lexemeBegin ──────── forward (记号起点) (当前扫描位置)

3.2 超前搜索策略

某些语言的记号无法仅凭当前字符确定，需要"向前看"若干字符：

  • 关键字 vs 标识符：whilex 是标识符而非关键字 while
  • 浮点数 vs 范围运算符：3.14 vs 3..10
  • 注释开始：/ 是除法还是 /* 注释的开始

3.3 超前搜索的实现

// 超前搜索伪代码 while (true) { c = getNextChar(); if (c matches current token pattern) { forward++; } else { // 回退一个字符 retract(forward); // 输出当前记号 emitToken(type, lexemeBegin..forward); break; } }

四、关键字（保留字）的处理

4.1 两种实现方式

方式描述优点缺点
单独的DFA状态 每个关键字有独立的自动机路径 识别速度快 状态表膨胀
符号表预填入 先按标识符规则识别，再查符号表判断 简洁，易于添加新关键字 多一次查表操作
推荐方式：在初始化阶段将所有关键字填入符号表/哈希表，词法分析器先按标识符规则扫描，然后查表判断是否为关键字。这是大多数实际编译器（如GCC、LLVM）采用的策略。

5.2 Lex/Flex 规则文件结构

%{ /* C声明区：头文件、全局变量等 */ #include "y.tab.h" int line_num = 1; %} /* Flex定义区 */ DIGIT [0-9] LETTER [a-zA-Z] ID {LETTER}({LETTER}|{DIGIT})* %% /* 规则分隔符 */ "if" { return IF; } "else" { return ELSE; } "while" { return WHILE; } {ID} { yylval.str = strdup(yytext); return IDENTIFIER; } {DIGIT}+ { yylval.num = atoi(yytext); return NUMBER; } [ \t]+ { /* 跳过空白 */ } \n { line_num++; } "//".* { /* 单行注释，跳过 */ } . { fprintf(stderr, "非法字符: %c\n", *yytext); } %% /* 用户代码区 */ int yywrap() { return 1; }

六、手工实现词法分析器

6.1 直接编码法（Hand-Coded Scanner）

不依赖自动生成工具，直接用程序语言编写状态机。

Token nextToken() { // 1. 跳过空白和注释 while (current == ' ' || current == '\t' || current == '\n') advance(); if (current == '/' && peek() == '/') { while (current != '\n') advance(); return nextToken(); // 递归继续 } // 2. 识别标识符或关键字 if (isLetter(current)) { StringBuilder sb = new StringBuilder(); while (isLetterOrDigit(current)) { sb.append(current); advance(); } String word = sb.toString(); if (keywords.containsKey(word)) return new Token(KEYWORD, word); else return new Token(IDENTIFIER, word); } // 3. 识别数字 if (isDigit(current)) { StringBuilder sb = new StringBuilder(); while (isDigit(current)) { sb.append(current); advance(); } if (current == '.') { sb.append(current); advance(); while (isDigit(current)) { sb.append(current); advance(); } return new Token(FLOAT, sb.toString()); } return new Token(INTEGER, sb.toString()); } // 4. 识别运算符和分隔符 // ...（逐个匹配） }

6.2 表驱动法（Table-Driven Scanner）

使用预计算的状态转移表驱动扫描过程，更通用、更易维护。

  1. 构建状态转移表（DFA转换表）
  2. 维护当前状态、接受状态集合、回退规则
  3. 查表驱动状态转移
