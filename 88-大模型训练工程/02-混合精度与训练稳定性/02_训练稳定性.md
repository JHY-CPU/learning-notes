## 1. Loss Spike 原因与处理


Loss Spike 是大模型训练中常见的现象，表现为损失值突然大幅上升（通常数十到数百倍），随后可能恢复或导致训练崩溃。


### 1.1 常见原因


| 原因 | 机制 | 出现阶段 |
| --- | --- | --- |
| 数据异常 | 脏数据、异常长序列、编码错误 | 整个训练过程 |
| 学习率过大 | 参数更新步长过大跳出局部最优 | warmup 结束后 |
| 梯度累积溢出 | FP16 精度不够导致梯度溢出 | 混合精度训练中 |
| 数值不稳定 | Softmax/Log 中的极端值 | 注意力层、输出层 |
| 分布式通信 | 梯度同步失败或延迟 | 多卡训练 |


### 1.2 处理策略


#### 策略一：跳过异常 Batch


```
python
# 检测并跳过导致 loss spike 的 batch
loss_threshold = 10.0 # 根据经验设定阈值
for batch in dataloader:
    loss = model(batch)

    if loss.item() > loss_threshold:
        logger.warning(f"Loss spike detected: {loss.item():.2f}, skipping batch")
        optimizer.zero_grad()
        continue # 跳过该 batch

    loss.backward()
    optimizer.step()
```


#### 策略二：梯度异常检测与恢复


```
python
def check_and_handle_spike(model, optimizer, scaler, threshold=1000.0):
    # 检查梯度是否包含 NaN/Inf
    has_bad_grad = False
    total_norm = 0.0
for p in model.parameters():
        if p.grad is not None:
            if torch.any(torch.isnan(p.grad)) or torch.any(torch.isinf(p.grad)):
                has_bad_grad = True
break
            total_norm += p.grad.data.norm(2).item() ** 2
    total_norm = total_norm ** 0.5
if has_bad_grad or total_norm > threshold:
        # 回滚到上一个 checkpoint
        logger.warning(f"Gradient anomaly: norm={total_norm:.2f}, reloading checkpoint")
        optimizer.zero_grad()
        return False
return True
```


#### 策略三：学习率回退 (LR Decay on Spike)


> **Note:** **LLaMA 论文中的方法：**
> 当检测到 loss spike 时，将学习率临时降低到当前值的 1/10 或 1/100，等待 loss 恢复后缓慢回升到目标学习率。这种方法可以让训练在 spike 后继续，而非从 checkpoint 重新开始。


## 2. 梯度裁剪 (Grad Norm)


梯度裁剪是防止梯度爆炸、稳定训练的核心技术。最常用的是全局梯度范数裁剪（Global Gradient Norm Clipping）。


### 2.1 全局梯度范数裁剪


$$
总梯度范数: ||g||_2 = sqrt(Σ_i ||g_i||_2^2)
                裁剪后: g'_i = g_i × (max_norm / max(||g||_2, max_norm))
$$


```
梯度裁剪示意:

裁剪前:
  Layer 1 梯度: [0.5, 1.2, ...]  范数 = 200 (过大!)
  Layer 2 梯度: [0.1, 0.3, ...]  范数 = 5
  总范数 = sqrt(200² + 5² + ...) ≈ 200

裁剪 (max_norm = 1.0):
  缩放因子 = 1.0 / 200 = 0.005

裁剪后:
  Layer 1 梯度: [0.5×0.005, 1.2×0.005, ...]  范数 ≈ 1.0
  Layer 2 梯度: [0.1×0.005, 0.3×0.005, ...]  范数 ≈ 0.025
  总范数 = 1.0 (所有层按相同比例缩放)
```


### 2.2 PyTorch 实现


```
python
# 方法一: torch.nn.utils.clip_grad_norm_ (推荐)
total_norm = torch.nn.utils.clip_grad_norm_(
    model.parameters(),
    max_norm=1.0,        # 裁剪阈值 (常用: 0.5 ~ 2.0)
    norm_type=2.0,        # L2 范数
    error_if_nonfinite=False # 遇到 inf/nan 不报错
)

# 方法二: clip_grad_value_ (逐元素裁剪)
torch.nn.utils.clip_grad_value_(model.parameters(), clip_value=0.5)

# 训练循环中的使用
loss.backward()
grad_norm = torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
optimizer.step()

# 记录梯度范数用于监控
wandb.log({"grad_norm": grad_norm.item()})
```


### 2.3 max_norm 选择建议


| 模型规模 | 推荐 max_norm | 说明 |
| --- | --- | --- |
| < 1B | 1.0 | 标准值 |
| 1B ~ 10B | 1.0 ~ 2.0 | LLaMA 使用 1.0 |
| 10B ~ 100B | 0.5 ~ 1.0 | GPT-3 使用 1.0 |
| > 100B | 1.0 | PaLM 使用 1.0 |


> **Warning:** **混合精度训练中：**
> 必须先
> `scaler.unscale_(optimizer)`
> 再裁剪梯度。否则裁剪的是缩放后的梯度，实际梯度并未被裁剪。PyTorch 的
> `GradScaler`
> 文档明确要求这个顺序。


## 3. 学习率调度策略


### 3.1 Cosine Annealing（余弦退火）


大模型训练中最常用的学习率调度策略。学习率从最大值按余弦曲线平滑下降到最小值。


$$
lr(t) = lr_min + 0.5 × (lr_max - lr_min) × (1 + cos(π × t / T))
$$


```
python
# Cosine Annealing with Warmup
import math

def cosine_schedule_with_warmup(
    current_step,
    warmup_steps,
    total_steps,
    lr_max,
    lr_min
):
    if current_step < warmup_steps:
        # 线性 warmup
return lr_max * current_step / warmup_steps
    else:
        # Cosine decay
        progress = (current_step - warmup_steps) / (total_steps - warmup_steps)
        return lr_min + 0.5 * (lr_max - lr_min) * (1 + math.cos(math.pi * progress))

# PyTorch LambdaLR 实现
scheduler = torch.optim.lr_scheduler.LambdaLR(
    optimizer,
    lr_lambda=lambda step: cosine_schedule_with_warmup(
        step, warmup_steps=2000, total_steps=500000,
        lr_max=3e-4, lr_min=3e-5
    )
)
```


### 3.2 其他调度策略


| 策略 | 公式 | 特点 | 适用场景 |
| --- | --- | --- | --- |
| Cosine | 余弦曲线衰减 | 平滑、效果好 | LLM 预训练首选 |
| Linear | 线性衰减 | 简单、衰减快 | BERT 预训练 |
| WSD | Warmup-Stable-Decay | 长稳定期 | 超长训练 |
| Constant | 恒定学习率 | 无衰减 | 短训练/微调 |


### 3.3 WSD (Warmup-Stable-Decay) 调度


近年来在大模型训练中流行的新策略，将训练分为三个阶段：


```
WSD 学习率调度:

lr
↑
│   ╱╲
│  ╱  ╲________________________________
│ ╱    Warmup  Stable (恒定 LR)  ╲
│╱                                  ╲
├─────┬────────────────────────┬───────► step
   warmup      stable           decay

优势: 稳定期可以让模型充分学习
     适合 checkpoint 后继续训练 (lr restart)
```


## 4. Warmup 策略


Warmup 是训练初期将学习率从零（或很小值）线性增加到目标值的过程，对训练稳定性至关重要。


### 4.1 为什么需要 Warmup


- **随机初始化：**
   模型初始化时参数随机，大的学习率会导致参数剧烈变化
- **Adam 状态：**
   Adam 优化器的动量和方差需要时间积累，初期状态不可靠
- **Layer Norm：**
   深层网络中，初始梯度可能被放大，需要逐步适应


### 4.2 Warmup 步数选择


| 模型规模 | 推荐 warmup 比例 | 典型值 |
| --- | --- | --- |
| < 1B | 1% ~ 2% | 2000 ~ 5000 步 |
| 1B ~ 10B | 1% ~ 2% | 2000 ~ 10000 步 |
| 10B ~ 100B | 0.5% ~ 2% | 2000 ~ 20000 步 |
| > 100B | 0.1% ~ 1% | 根据数据量调整 |


## 5. 权重衰减


权重衰减（Weight Decay）通过在损失函数中加入 L2 正则化项，防止模型参数过大，有助于泛化和训练稳定性。


### 5.1 AdamW 实现


AdamW 将权重衰减与梯度更新解耦，是大模型训练的标准选择：


$$
Adam 更新: m_t = β₁·m_{t-1} + (1-β₁)·g_t
                v_t = β₂·v_{t-1} + (1-β₂)·g_t²
                W_t = W_{t-1} - lr · (m̂_t / (√v̂_t + ε) + λ·W_{t-1})  ← Adam

                AdamW 更新: W_t = W_{t-1} - lr · (m̂_t / (√v̂_t + ε)) - lr·λ·W_{t-1}  ← AdamW
$$


```
python
# 分层权重衰减 (常见于 LLM 训练)
def get_optimizer_params(model, weight_decay=0.1):
    # 不对 bias 和 LayerNorm 参数施加权重衰减
    decay_params = []
    no_decay_params = []

    for name, param in model.named_parameters():
        if not param.requires_grad:
            continue
if "bias" in name or "LayerNorm" in name or "layernorm" in name:
            no_decay_params.append(param)
        else:
            decay_params.append(param)

    return [
        {"params": decay_params, "weight_decay": weight_decay},
        {"params": no_decay_params, "weight_decay": 0.0}
    ]

optimizer = torch.optim.AdamW(
    get_optimizer_params(model, weight_decay=0.1),
    lr=3e-4,
    betas=(0.9, 0.95),
    eps=1e-8
)
```


### 5.2 常见超参数


| 模型 | Weight Decay | Adam β₁ | Adam β₂ | ε |
| --- | --- | --- | --- | --- |
| GPT-3 | 0.1 | 0.9 | 0.95 | 1e-8 |
| LLaMA | 0.1 | 0.9 | 0.95 | 1e-5 |
| PaLM | 0.1 | 0.9 | 0.99 | 1e-8 |
| BERT | 0.01 | 0.9 | 0.999 | 1e-8 |


## 6. NaN 检测与恢复


### 6.1 NaN 的常见来源


- **除以零：**
   log(0)、1/0 等操作产生 inf/nan
- **梯度爆炸：**
   深层网络中的梯度累积导致数值溢出
- **混合精度：**
   FP16 精度不足导致梯度下溢或溢出
- **数据问题：**
   包含 inf/nan 的输入数据
- **数值不稳定：**
   Softmax 在极端值下可能产生 nan


### 6.2 NaN 检测工具


```
python
# 方法一: PyTorch Anomaly Detection
torch.autograd.set_detect_anomaly(True)

# 方法二: 自定义检测函数
def detect_anomaly(model, batch_idx):
    # 检查模型参数
for name, param in model.named_parameters():
        if torch.any(torch.isnan(param)):
            print(f"NaN in parameter {name}")
        if param.grad is not None:
            if torch.any(torch.isnan(param.grad)):
                print(f"NaN in gradient of {name}")

# 方法三: 在 forward 中插入检测
class SafeSoftmax(nn.Module):
    def forward(self, x):
        # 检查输入
if torch.any(torch.isnan(x)):
            print("NaN detected before softmax")
            x = torch.nan_to_num(x, nan=0.0)
        return F.softmax(x, dim=-1)
```


### 6.3 NaN 恢复策略

**处理优先级：**

1. 跳过当前 batch（最快的恢复方式）
2. 回滚到上一个 good checkpoint
3. 降低学习率后继续
4. 检查数据和模型代码（根本解决）


## 7. 训练稳定性清单


> **Tip:** ### 训练前检查
>
>
> 1. 数据清洗：移除异常长度、乱码、特殊字符
> 2. 学习率 warmup：至少 1% 的总步数
> 3. 梯度裁剪：max_norm = 1.0（或根据经验调整）
> 4. 权重衰减：AdamW + 分层衰减
> 5. 混合精度：BF16 优先于 FP16
>
>
> ### 训练中监控
>
>
> 1. Loss 曲线：正常应平滑下降
> 2. 梯度范数：应在合理范围内波动
> 3. 学习率：确认调度器正常工作
> 4. 显存使用：检查是否有异常增长
> 5. NaN/Inf 检测：及时发现数值问题
>
>
> ### 容错机制
>
>
> 1. 定期保存 checkpoint（每 N 步或每 N 分钟）
> 2. 异常 batch 跳过机制
> 3. 梯度异常检测与回滚
> 4. 学习率自适应调整
> 5. 分布式训练中的弹性重启
