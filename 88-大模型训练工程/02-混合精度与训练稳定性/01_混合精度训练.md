## 1. 为什么需要混合精度


大模型训练中，混合精度训练是同时使用不同精度的浮点数进行计算的技术，旨在在保持模型精度的同时，减少显存占用并加速训练。


### 1.1 核心优势


- **显存减半：**
   FP16/BF16 占用 2 bytes，相比 FP32 的 4 bytes 减半
- **速度翻倍：**
   现代 GPU（A100/H100）的 FP16/BF16 Tensor Core 吞吐量是 FP32 的 2 倍以上
- **带宽减半：**
   低精度数据传输量减小，缓解内存带宽瓶颈


### 1.2 精度损失的来源


直接使用 FP16 训练会遇到两个主要问题：


1. **梯度下溢（Underflow）：**
   小梯度值被舍入为零。例如梯度值为 1e-8 时，FP16 的最小正数约为 6e-5，梯度直接变为 0
2. **舍入误差（Rounding Error）：**
   大数加小数时，小数被忽略。如 2048 + 0.1 = 2048（FP16）


## 2. 浮点格式对比


| 格式 | 位宽 | 指数位 | 尾数位 | 动态范围 | 精度 |
| --- | --- | --- | --- | --- | --- |
| FP32 | 32 bit | 8 bit | 23 bit | ±3.4e38 | ~7 位有效数字 |
| TF32 | 19 bit (内部) | 8 bit | 10 bit | 与 FP32 相同 | ~10 位有效数字 |
| FP16 | 16 bit | 5 bit | 10 bit | ±6.5e4 | ~3 位有效数字 |
| BF16 | 16 bit | 8 bit | 7 bit | ±3.4e38 | ~2 位有效数字 |
| FP8 E4M3 | 8 bit | 4 bit | 3 bit | ±448 | ~1 位有效数字 |
| FP8 E5M2 | 8 bit | 5 bit | 2 bit | ±57344 | ~1 位有效数字 |


```
浮点数格式位布局:

FP32: [1 bit 符号][8 bit 指数][23 bit 尾数]  = 32 bit
      动态范围: 2^{-126} ~ 2^{127}
      精度: ~7 位十进制有效数字

FP16: [1 bit 符号][5 bit 指数][10 bit 尾数]  = 16 bit
      动态范围: 2^{-14} ~ 2^{15}
      问题: 范围太小, 梯度容易下溢!

BF16: [1 bit 符号][8 bit 指数][ 7 bit 尾数]  = 16 bit
      动态范围: 2^{-126} ~ 2^{127}
      优势: 范围与 FP32 相同, 不易溢出
      劣势: 精度较低

TF32: [1 bit 符号][8 bit 指数][10 bit 尾数]  = 19 bit (NVIDIA 内部格式)
      在 Tensor Core 中使用, 无需额外编码
      精度介于 FP32 和 BF16 之间
```


> **Note:** **FP16 的动态范围问题：**
> FP16 的最大正数约 65504，最小正数约 6e-5。对于深度学习中常见的梯度值（通常在 1e-4 到 1e-8 之间），FP16 无法表示，需要 Loss Scaling 来解决。


## 3. Loss Scaling 机制


Loss Scaling 通过在反向传播前将损失值乘以一个缩放因子 S，使梯度值进入 FP16 可表示的范围，反向传播后再除以 S 恢复真实梯度值。


### 3.1 基本原理


$$
缩放后梯度: g_scaled = S · ∇L(W)
                参数更新: W_{t+1} = W_t - lr · g_scaled / S = W_t - lr · ∇L(W)
$$


```
Loss Scaling 工作流程:

前向传播:
  loss = criterion(model(x), y)        ← 计算原始损失

缩放:
  loss_scaled = loss × scale_factor    ← 放大损失

反向传播:
  loss_scaled.backward()               ← 梯度被放大 scale_factor 倍
  model.grad = scale_factor × ∇L       ← 梯度值变大, 进入 FP16 范围

梯度还原:
  model.grad /= scale_factor           ← 恢复真实梯度

梯度裁剪:
  clip_grad_norm_(model.parameters(), max_norm=1.0)

参数更新:
  optimizer.step()
```


### 3.2 Static vs Dynamic Scaling


#### Static Loss Scaling


- 使用固定的缩放因子（如 2^16 = 65536）
- 简单但不灵活，可能过大（溢出）或过小（下溢）
- 适用于训练稳定的场景


#### Dynamic Loss Scaling（推荐）


```
python
# Dynamic Loss Scaling 算法
class DynamicLossScaler:
    def __init__(self,
                 init_scale=2.**16,       # 初始缩放因子 65536
                 scale_factor=2.,         # 每次调整的倍数
                 scale_window=2000,      # 检查间隔 (步数)
                 min_scale=1.,           # 最小缩放因子
                 max_scale=2.**24):      # 最大缩放因子
        self.scale = init_scale
        self.iter = 0
        self.last_overflow_iter = -1
def update(self, has_nan_or_inf):
        # 检测到 NaN/Inf → 缩放因子减半, 跳过本次更新
if has_nan_or_inf:
            self.scale /= self.scale_factor
            self.last_overflow_iter = self.iter
            return False # 跳过更新
# 连续 scale_window 步无溢出 → 缩放因子翻倍
        self.iter += 1
if self.iter - self.last_overflow_iter >= self.scale_window:
            self.scale *= self.scale_factor
            self.last_overflow_iter = self.iter

        return True # 正常更新
```


### 3.3 Overflow 检测


```
python
# 检测梯度中是否存在 NaN 或 Inf
def has_nan_or_inf(model):
    for param in model.parameters():
        if param.grad is not None:
            if torch.any(torch.isnan(param.grad)) or \
               torch.any(torch.isinf(param.grad)):
                return True
return False
# 训练循环中的使用
optimizer.zero_grad()
loss = model(batch)
scaler.scale(loss).backward()       # 缩放损失
# 检查溢出
if has_nan_or_inf(model):
    scaler.update(has_nan_or_inf=True)
    optimizer.zero_grad()
    continue # 跳过本次更新

scaler.unscale_(optimizer)          # 还原梯度
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
scaler.step(optimizer)             # 更新参数
scaler.update()                    # 更新缩放因子
```


## 4. PyTorch AMP 自动混合精度


PyTorch 的 `torch.amp` 模块提供了自动混合精度训练的支持，自动决定哪些操作使用低精度、哪些使用高精度。


### 4.1 autocast 上下文管理器


```
python
import torch
from torch.amp import autocast, GradScaler

# 创建 GradScaler
scaler = GradScaler("cuda")

for batch in dataloader:
    optimizer.zero_grad()

    # autocast 自动选择精度
with autocast(device_type="cuda", dtype=torch.float16):
        output = model(batch)
        loss = criterion(output, targets)

    # 缩放损失并反向传播
    scaler.scale(loss).backward()

    # 梯度裁剪 (需要先 unscale)
    scaler.unscale_(optimizer)
    torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)

    # 更新参数
    scaler.step(optimizer)
    scaler.update()
```


### 4.2 autocast 的精度选择规则


| 操作类型 | 自动选择精度 | 原因 |
| --- | --- | --- |
| 矩阵乘法 (matmul, linear) | FP16/BF16 | Tensor Core 加速 |
| 卷积 (conv1d/2d/3d) | FP16/BF16 | Tensor Core 加速 |
| LayerNorm, BatchNorm | FP32 | 数值稳定性 |
| Softmax, LogSoftmax | FP32 | 数值稳定性 |
| 损失函数 (CrossEntropy) | FP32 | 精度要求高 |
| Embedding | FP32 | 查找表需要精确 |
| Residual Add | FP32 | 累积误差控制 |


### 4.3 使用 BF16 的 AMP


```
python
# BF16 训练 (推荐用于 Ampere 及以上 GPU)
# BF16 不需要 GradScaler! 因为动态范围与 FP32 相同
with autocast(device_type="cuda", dtype=torch.bfloat16):
    output = model(batch)
    loss = criterion(output, targets)

loss.backward()  # 无需 scaler
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
optimizer.step()
```


## 5. Master Weight 策略


Master Weight（主权重）是在低精度训练中维护一份 FP32 精度的参数副本，用于累积参数更新，避免精度损失。


### 5.1 工作原理


```
Master Weight 策略:

┌──────────────────────────────────────────┐
│                                          │
│   Master Weight (FP32)                   │
│   ┌────────────────────────────┐         │
│   │ W_master (32 bit)          │         │
│   └────────────┬───────────────┘         │
│                │                         │
│                │ 转换为 FP16              │
│                ▼                         │
│   ┌────────────────────────────┐         │
│   │ W_compute (16 bit)         │ ──► 前向传播
│   └────────────┬───────────────┘         │
│                │                         │
│                │ 反向传播                │
│                ▼                         │
│   ┌────────────────────────────┐         │
│   │ ∇W (16 bit)                │         │
│   └────────────┬───────────────┘         │
│                │                         │
│                │ 梯度同步 + 转换为 FP32   │
│                ▼                         │
│   ┌────────────────────────────┐         │
│   │ W_master -= lr × ∇W (FP32) │        │
│   └────────────────────────────┘         │
│   (参数更新在 FP32 精度下完成)             │
└──────────────────────────────────────────┘
```


### 5.2 PyTorch 中的实现


PyTorch 的 `GradScaler` 自动处理 Master Weight。当使用 `scaler.step(optimizer)` 时，内部流程为：


1. 检查梯度是否有 NaN/Inf（
   `unscale_`
   ）
2. 如果没有溢出，将 FP16 梯度应用于 FP32 Master Weight
3. 用 FP32 精度执行参数更新
4. 将更新后的 FP32 Master Weight 转回 FP16 用于前向计算


> **Warning:** **显存开销：**
> 使用 Master Weight 意味着每个参数需要 4 bytes (FP32 master) + 2 bytes (FP16 copy) = 6 bytes，而非单纯的 2 bytes。这也是为什么混合精度训练的实际显存节省约为 30-40%，而非理论的 50%。


## 6. BF16 vs FP16 选择


| 特性 | FP16 | BF16 |
| --- | --- | --- |
| 动态范围 | ±6.5e4 (小) | ±3.4e38 (与 FP32 相同) |
| 精度 | ~3 位有效数字 | ~2 位有效数字 |
| 需要 Loss Scaling | 必须 | 不需要 |
| 需要 GradScaler | 必须 | 不需要 |
| 数值稳定性 | 需额外处理溢出 | 天然稳定 |
| 硬件支持 | Volta+ (V100+) | Ampere+ (A100+) |
| 推荐场景 | V100 GPU | A100/H100 GPU |


> **Tip:** **推荐：**
> 如果有 A100 或更新的 GPU，强烈推荐使用 BF16。它省去了 Loss Scaling 的复杂性，训练更稳定，且不会因为溢出导致训练中断。大多数现代大模型（LLaMA、GPT-4、PaLM）都使用 BF16 训练。


## 7. FP8 训练


FP8 是最新的低精度格式，由 H100 GPU 原生支持，可进一步提升训练效率和降低显存。


### 7.1 FP8 两种格式


- **E4M3：**
   4 bit 指数 + 3 bit 尾数，精度较高，适合前向传播
- **E5M2：**
   5 bit 指数 + 2 bit 尾数，范围较大，适合反向传播（梯度范围更广）


### 7.2 Transformer Engine 实现


```
python
# 使用 NVIDIA Transformer Engine 进行 FP8 训练
import transformer_engine.pytorch as te

# FP8 Linear 层
fp8_linear = te.Linear(
    in_features=4096,
    out_features=4096,
    bias=True,
    # FP8 配置
    params_dtype=torch.bfloat16,
    # 量化模式
    fp8_recipe=te.recipe.DelayedScaling(
        margin=0,
        fp8_format=te.recipe.Format.HYBRID,  # E4M3 前向, E5M2 反向
        amax_history_len=16,
        amax_compute_algo="max"
    )
)

# 使用方式与普通 Linear 相同
with te.fp8_autocast(enabled=True):
    output = fp8_linear(input_tensor)
```


> **Note:** **FP8 的挑战：**
>
> - 极低的精度可能影响模型最终收敛质量
> - 需要精细的量化策略（per-tensor scaling）
> - 目前主要用于推理加速，训练中仅用于矩阵乘法
> - LayerNorm、Softmax 等仍需更高精度


## 8. 最佳实践


### 8.1 配置推荐


> **Tip:** 1. **A100/H100：**
>    使用 BF16 + torch.bfloat16 autocast，无需 GradScaler
> 2. **V100：**
>    使用 FP16 + GradScaler + Dynamic Loss Scaling
> 3. **H100 + Transformer Engine：**
>    FP8 用于矩阵乘法加速
> 4. **梯度累积：**
>    在 unscale 之前累积，或在 no_sync 块中累积


### 8.2 常见问题排查


| 问题 | 原因 | 解决方案 |
| --- | --- | --- |
| Loss 变 NaN | FP16 梯度溢出 | 增大初始 scale，或切换 BF16 |
| Loss 不下降 | scale 过小导致梯度为零 | 检查 scale 是否持续缩小 |
| 训练突然中断 | Dynamic scaling 太频繁 | 增大 scale_window |
| 精度下降 | 过多操作用低精度 | 手动指定某些层用 FP32 |
