# 光照与氛围设计

## 核心概念

光照是塑造关卡氛围的核心手段，直接影响玩家的情绪感受和空间认知。同样的空间在不同光照下可以传达完全不同的感受：暖色调低角度光营造温馨感，冷色调高对比光营造紧张感，全向无阴影光营造不安感。

### 实时光照 vs 烘焙光照深度对比

| 特性 | 实时光照 | 烘焙光照 |
|------|---------|---------|
| 运行时开销 | 高（每帧计算阴影+光照） | 无（预计算为Lightmap） |
| 动态物体 | 完全支持光照变化 | 需要Light Probe采样间接光 |
| 间接光照 | 需要RTGI/Lumen（昂贵） | 自动包含在Lightmap中 |
| 内存占用 | 低 | 高（Lightmap纹理） |
| 适用场景 | 动态光源、昼夜变化 | 静态室内、移动平台 |
| 实时阴影 | 有 | 无（需要混合方案） |

### 光照探针详解

**Light Probe（光照探针）**：
- 采样烘焙的间接光照信息（颜色和强度）
- 动态物体经过时自动应用附近的光照数据
- 放置在光照变化明显的区域（明暗交界处、窗口附近、走廊转角）
- 使用球谐函数（Spherical Harmonics）存储低频光照信息

**Reflection Probe（反射探针）**：
- 烘焙或实时捕获环境反射
- 使金属/光滑表面反射周围环境
- 室内场景中每个房间通常需要独立的Reflection Probe
- Box Blend控制多个Probe之间的混合

### 后处理与氛围

后处理效果是快速改变场景氛围的强大工具：
- **Color Grading**：调整色温、对比度、饱和度，是最高效的氛围手段
- **Bloom**：发光效果，增加光源的"呼吸感"
- **Vignette**：暗角效果，引导视线到中心
- **Depth of Field**：景深效果，突出焦点
- **Chromatic Aberration**：色差效果，增加镜头感

## 具体实现方法

### Unity烘焙光照完整设置

```
步骤1: 准备场景
1. 将静态物体标记为Lightmap Static
2. 确保场景中有Lightmap Settings Asset

步骤2: 配置参数 (Window > Rendering > Lighting)
- Lighting Mode: Subtractive (移动端) 或 Shadowmask (PC)
- Directional Mode:
  - Non-Directional: 节省50%Lightmap内存，适合简单光照
  - Directional: 保留法线方向信息，效果更好但内存x2
- Lightmap Resolution: 2-10 texel/unit
  - 移动端: 2-4 texel/unit
  - PC端: 6-10 texel/unit
- Lightmap Padding: 2-4 texel（避免UV接缝处的漏光）
- Ambient Occlusion: 开启增强暗部细节
- Final Gather: 提高质量但增加烘焙时间（仅高质量需求时开启）

步骤3: 烘焙
- 点击Generate Lighting
- 使用Progressive Lightmapper（GPU版本更快）
- 可以边编辑边预览实时烘焙结果
```

### 体积雾效果系统

```csharp
using UnityEngine;
using UnityEngine.Rendering;
using UnityEngine.Rendering.Universal;

/// <summary>
/// 体积雾控制器
/// 支持动态调整雾密度和过渡
/// </summary>
public class VolumetricFogController : MonoBehaviour
{
    [SerializeField] private Volume volume;
    private Fog fog;

    [Header("预设")]
    [SerializeField] private FogPreset clearPreset;
    [SerializeField] private FogPreset lightFogPreset;
    [SerializeField] private FogPreset heavyFogPreset;

    [System.Serializable]
    public class FogPreset
    {
        public float meanFreePath = 100f; // 能见度（越大越清晰）
        public Color fogColor = Color.gray;
        public float baseHeight = 0f;
        public float maximumHeight = 50f;
    }

    void Start()
    {
        volume.profile.TryGet(out fog);
    }

    public void TransitionTo(FogPreset preset, float duration = 3f)
    {
        StartCoroutine(FadeFog(preset, duration));
    }

    System.Collections.IEnumerator FadeFog(FogPreset target, float duration)
    {
        if (fog == null) yield break;

        float startPath = fog.meanFreePath.value;
        Color startColor = fog.albedo.value;
        float elapsed = 0;

        while (elapsed < duration)
        {
            elapsed += Time.deltaTime;
            float t = elapsed / duration;

            fog.meanFreePath.value = Mathf.Lerp(startPath, target.meanFreePath, t);
            fog.albedo.value = Color.Lerp(startColor, target.fogColor, t);
            fog.baseHeight.value = Mathf.Lerp(fog.baseHeight.value, target.baseHeight, t);

            yield return null;
        }
    }

    /// <summary>
    /// 根据天气系统动态调整
    /// </summary>
    public void UpdateFromWeather(float rainIntensity, float timeOfDay)
    {
        if (fog == null) return;

        // 下雨时增加雾密度
        float fogDensity = Mathf.Lerp(100f, 20f, rainIntensity);
        fog.meanFreePath.value = fogDensity;

        // 黄昏时雾偏暖色
        Color fogColor = Color.Lerp(
            new Color(0.8f, 0.85f, 0.9f), // 白天
            new Color(1f, 0.7f, 0.5f),     // 黄昏
            Mathf.Sin(timeOfDay * Mathf.PI));
        fog.albedo.value = fogColor;
    }
}
```

### Light Probe自动放置工具

```csharp
#if UNITY_EDITOR
using UnityEditor;
using UnityEngine;
using System.Collections.Generic;

/// <summary>
/// Light Probe自动放置工具
/// 在场景中均匀放置Light Probe Group
/// </summary>
public class LightProbePlacer : EditorWindow
{
    [MenuItem("Tools/Light Probe Placer")]
    static void Open() => GetWindow<LightProbePlacer>();

    private float spacing = 3f;
    private float height = 1.5f;
    private bool placeOnGround = true;
    private Vector2 areaMin = new Vector2(-50, -50);
    private Vector2 areaMax = new Vector2(50, 50);

    void OnGUI()
    {
        spacing = EditorGUILayout.FloatField("Spacing", spacing);
        height = EditorGUILayout.FloatField("Height Above Ground", height);
        placeOnGround = EditorGUILayout.Toggle("Place on Ground", placeOnGround);
        areaMin = EditorGUILayout.Vector2Field("Area Min", areaMin);
        areaMax = EditorGUILayout.Vector2Field("Area Max", areaMax);

        if (GUILayout.Button("Auto Place Probes"))
            AutoPlace();
    }

    void AutoPlace()
    {
        LightProbeGroup group
            = FindObjectOfType<LightProbeGroup>();
        if (group == null)
        {
            GameObject go = new GameObject("LightProbeGroup");
            group = go.AddComponent<LightProbeGroup>();
        }

        var positions = new List<Vector3>();

        for (float x = areaMin.x; x < areaMax.x; x += spacing)
        for (float z = areaMin.y; z < areaMax.y; z += spacing)
        {
            if (placeOnGround)
            {
                // 射线检测地面
                if (Physics.Raycast(new Vector3(x, 100, z),
                    Vector3.down, out RaycastHit hit, 200f))
                {
                    positions.Add(hit.point + Vector3.up * height);
                }
            }
            else
            {
                positions.Add(new Vector3(x, height, z));
            }
        }

        group.probePositions = positions.ToArray();
        Debug.Log($"Placed {positions.Count} light probes");
    }
}
#endif
```

### 昼夜光照循环

```csharp
using UnityEngine;

/// <summary>
/// 完整的昼夜光照循环系统
/// 控制太阳角度、光照强度、颜色、环境光
/// </summary>
public class DayNightCycle : MonoBehaviour
{
    public static DayNightCycle Instance { get; private set; }

    [Header("光源")]
    [SerializeField] private Light sunLight;
    [SerializeField] private Light moonLight;

    [Header("时间设置")]
    [SerializeField] private float dayDurationSeconds = 600f; // 游戏中一天10分钟
    [Range(0, 1)] public float timeOfDay = 0.25f; // 0=午夜, 0.25=日出, 0.5=正午, 0.75=日落

    [Header("光照曲线")]
    [SerializeField] private AnimationCurve sunIntensityCurve;
    [SerializeField] private Gradient sunColorGradient;
    [SerializeField] private Gradient ambientColorGradient;
    [SerializeField] private AnimationCurve fogDensityCurve;

    [Header("时间标记")]
    [SerializeField] private float sunriseTime = 0.25f;
    [SerializeField] private float noonTime = 0.5f;
    [SerializeField] private float sunsetTime = 0.75f;

    public System.Action<float> OnTimeChanged; // 0-1

    void Awake()
    {
        Instance = this;
    }

    void Update()
    {
        timeOfDay += Time.deltaTime / dayDurationSeconds;
        if (timeOfDay >= 1f) timeOfDay -= 1f;

        UpdateLighting();
        OnTimeChanged?.Invoke(timeOfDay);
    }

    void UpdateLighting()
    {
        // 太阳角度
        float sunAngle = timeOfDay * 360f - 90f;
        sunLight.transform.rotation = Quaternion.Euler(sunAngle, 170f, 0f);

        // 太阳强度和颜色
        sunLight.intensity = sunIntensityCurve.Evaluate(timeOfDay);
        sunLight.color = sunColorGradient.Evaluate(timeOfDay);

        // 月亮（太阳的反方向）
        if (moonLight != null)
        {
            moonLight.transform.rotation = Quaternion.Euler(sunAngle + 180f, 170f, 0f);
            moonLight.intensity = Mathf.Max(0, 0.3f - sunLight.intensity);
        }

        // 环境光
        RenderSettings.ambientLight = ambientColorGradient.Evaluate(timeOfDay);

        // 雾效果
        RenderSettings.fogDensity = fogDensityCurve.Evaluate(timeOfDay);
    }

    public bool IsDaytime => timeOfDay > sunriseTime && timeOfDay < sunsetTime;
    public bool IsSunrise => Mathf.Abs(timeOfDay - sunriseTime) < 0.02f;
    public bool IsSunset => Mathf.Abs(timeOfDay - sunsetTime) < 0.02f;
}
```

## 最佳实践

- 移动平台优先使用烘焙光照，实时光源控制在1-2个
- Light Probe放在光照变化的过渡区域（窗口、门口、明暗交界），而非均匀铺满
- Reflection Probe的Box Size设置为刚好包裹对应空间，不要过大
- 使用后处理Volume统一调整场景色调和氛围，这比调整每盏灯更高效
- 为不同区域设置不同的光照风格以增强空间识别度（冷色走廊 vs 暖色房间）
- 使用HDR值（大于1）配合Bloom创造发光效果
- 阴影使用柔和的边缘（软阴影），硬阴影看起来不自然

## 常见陷阱与修复

**陷阱1：烘焙分辨率过高导致Lightmap内存爆炸**
- 症状：移动端Lightmap占用200MB+内存
- 修复：移动端使用2-4 texel/unit，PC端使用6-8

**陷阱2：动态物体未被Light Probe覆盖**
- 症状：动态角色在暗处发亮或在亮处发暗
- 修复：确保角色活动区域都有Light Probe覆盖

**陷阱3：实时阴影距离过大**
- 症状：远处物体的阴影计算消耗大量GPU
- 修复：减小Shadow Distance到30-50m

**陷阱4：Reflection Probe之间未设置混合区域**
- 症状：从一个房间到另一个房间时反射突然跳变
- 修复：设置Probe的Importance和Box Size确保有重叠混合区域

**陷阱5：未考虑不同时段的光照需求**
- 症状：关卡在白天看起来很好但夜晚不可见
- 修复：使用DayNightCycle确保各时段都有合理的光照
