# 地图编辑器使用

## 核心概念

地图编辑器是关卡设计师的核心工具，用于快速搭建和迭代游戏世界。高效的编辑器使用能将关卡搭建时间从几天缩短到几小时。不同的游戏类型需要不同的地图编辑方案。

### Unity Tilemap系统详解

Tilemap是Unity 2D的官方地图工具，基于网格的瓦片系统：

- **Tile Palette（瓦片调色板）**：管理和绘制瓦片的面板，支持拖拽和批量操作
- **多种笔刷**：普通笔刷（单个绘制）、填充笔刷（区域填充）、选取笔刷（区域选择）、Prefab笔刷（批量放置预制体）
- **多层Tilemap**：地面层（地面纹理）、碰撞层（碰撞体）、装饰层（植被/碎片）、交互层（可破坏物体）分离管理
- **Rule Tile（规则瓦片）**：根据8方向邻居关系自动选择正确的贴图变体，如草地边缘、墙壁转角

### UE Landscape地形系统

UE的大世界地形工具，用于大规模室外场景：

- **Sculpt模式**：推拉平滑地形高度，支持多种笔刷形状
- **Paint模式**：在地形上绘制多个材质层，使用权重混合
- **Foliage模式**：批量放置植被，支持密度控制和随机化
- **World Partition**：大规模世界流式管理，按网格分区加载

### 地图编辑器性能注意事项

| 操作 | Tilemap | Landscape | Terrain(旧) |
|------|---------|-----------|-------------|
| 最大尺寸 | 理论无限 | 8129x8129 | 4096x4096 |
| 内存效率 | 高（仅存储非空瓦片） | 中等 | 低 |
| 绘制速度 | 极快 | 快 | 慢 |
| LOD支持 | 需手动实现 | 内置 | 内置 |
| 适合场景 | 2D/等距 | 大型室外 | 中型室外 |

## 具体实现方法

### Unity Tilemap完整设置

```
步骤1: 创建Tilemap层级结构
1. GameObject > 2D Object > Tilemap > Rectangular
2. 会自动创建Grid父节点和Tilemap子节点
3. 复制Tilemap创建多层:
   - Tilemap_Ground (地面层)
   - Tilemap_Collision (碰撞层)
   - Tilemap_Decoration (装饰层)
   - Tilemap_Interactable (交互层)

步骤2: 创建Tile Palette
1. Window > 2D > Tile Palette
2. Create New Palette → 命名如"Level_01_Palette"
3. 将贴图素材拖入Palette自动生成Tile
4. 使用笔刷在Scene视图中绘制

步骤3: 配置碰撞
1. 为Tilemap_Collision添加Tilemap Collider 2D组件
2. 设置Composite Collider 2D（合并碰撞体提升性能）
3. Tilemap Collider 2D的Used By Composite = true
```

### Rule Tile高级配置

```csharp
/// <summary>
/// 自定义自动瓦片
/// 根据8方向邻居自动选择合适的贴图变体
/// 适用于草地边缘、水面边缘、墙壁转角等
/// </summary>
#if UNITY_EDITOR
using UnityEngine;
using UnityEngine.Tilemaps;

[CreateAssetMenu(fileName = "AutoTile", menuName = "Tiles/Auto Tile")]
public class AutoTile : RuleTile<AutoTile.Neighbor>
{
    // 定义邻居类型
    public class Neighbor : RuleTile.TilingRule.Neighbor
    {
        public const int Same = 3;
        public const int Different = 4;
    }

    // 自定义邻居匹配逻辑
    public override bool RuleMatch(int neighbor, TileBase tile)
    {
        switch (neighbor)
        {
            case Neighbor.Same:
                return tile is AutoTile; // 相同类型
            case Neighbor.Different:
                return !(tile is AutoTile); // 不同类型
        }
        return base.RuleMatch(neighbor, tile);
    }

    // 也可以定义扩展的邻居类型
    // 如水平连接、垂直连接、对角连接等
}
#endif
```

### UE Landscape创建步骤

```
步骤1: 创建地形
1. Mode > Landscape > Manage
2. 设置地形尺寸:
   - Section Size: 1x1 或 2x2
   - Sections Per Component: 1 或 2
   - Number of Components: 根据地图大小
   - 总尺寸 = Section Size * Sections Per Component * Number of Components
3. 点击Create创建

步骤2: 雕刻地形
1. 切换到Sculpt模式
2. 常用工具:
   - Sculpt: 推高/拉低（最常用）
   - Smooth: 平滑过渡
   - Flatten: 平整化到指定高度
   - Erosion: 模拟自然侵蚀（制作自然地形）
   - Noise: 添加随机起伏
3. 笔刷设置:
   - Size: 笔刷大小
   - Strength: 笔刷强度
   - Falloff: 笔刷衰减曲线

步骤3: 绘制材质
1. 创建地形材质（使用Layer Blend节点混合多个层）
2. 切换到Paint模式
3. 选择材质层绘制
4. 使用Weight-Blended Layer实现自然过渡

步骤4: 植被
1. 切换到Foliage模式
2. 添加植被类型（Static Mesh）
3. 设置密度、缩放范围、随机旋转
4. 使用笔刷绘制植被
```

### 程序化Tilemap生成

```csharp
using UnityEngine;
using UnityEngine.Tilemaps;

/// <summary>
/// 程序化地图生成器
/// 结合Tilemap和算法自动生成关卡
/// </summary>
public class ProceduralMapGenerator : MonoBehaviour
{
    [Header("Tilemap引用")]
    [SerializeField] private Tilemap groundTilemap;
    [SerializeField] private Tilemap wallTilemap;
    [SerializeField] private Tilemap decorationTilemap;

    [Header("Tile资产")]
    [SerializeField] private TileBase floorTile;
    [SerializeField] private TileBase wallTile;
    [SerializeField] private RuleTile waterRuleTile;
    [SerializeField] private TileBase[] decorationTiles;

    [Header("生成参数")]
    [SerializeField] private int mapWidth = 50;
    [SerializeField] private int mapHeight = 50;
    [SerializeField] private int roomCount = 8;
    [SerializeField] private int minRoomSize = 4;
    [SerializeField] private int maxRoomSize = 10;
    [SerializeField] private int seed = -1; // -1为随机

    public void Generate()
    {
        // 清除已有Tilemap
        groundTilemap.ClearAllTiles();
        wallTilemap.ClearAllTiles();
        decorationTilemap.ClearAllTiles();

        // 设置随机种子
        if (seed >= 0) Random.InitState(seed);

        // 生成房间
        bool[,] map = new bool[mapWidth, mapHeight];
        Rect[] rooms = GenerateRooms();

        // 绘制房间
        foreach (var room in rooms)
            DrawRoom(map, room);

        // 连接房间
        for (int i = 1; i < rooms.Length; i++)
            DrawCorridor(map, rooms[i - 1], rooms[i]);

        // 应用到Tilemap
        ApplyToTilemap(map);

        // 添加装饰
        AddDecorations(map, rooms);
    }

    Rect[] GenerateRooms()
    {
        var rooms = new List<Rect>();
        int attempts = 0;

        while (rooms.Count < roomCount && attempts < roomCount * 10)
        {
            attempts++;
            int w = Random.Range(minRoomSize, maxRoomSize);
            int h = Random.Range(minRoomSize, maxRoomSize);
            int x = Random.Range(1, mapWidth - w - 1);
            int y = Random.Range(1, mapHeight - h - 1);

            Rect newRoom = new Rect(x, y, w, h);

            // 检查是否与已有房间重叠
            bool overlaps = false;
            foreach (var r in rooms)
            {
                if (newRoom.Overlaps(r)) { overlaps = true; break; }
            }

            if (!overlaps)
                rooms.Add(newRoom);
        }

        return rooms.ToArray();
    }

    void DrawRoom(bool[,] map, Rect room)
    {
        for (int x = (int)room.x; x < room.x + room.width; x++)
        for (int y = (int)room.y; y < room.y + room.height; y++)
            map[x, y] = true;
    }

    void DrawCorridor(bool[,] map, Rect from, Rect to)
    {
        Vector2Int start = new Vector2Int(
            (int)(from.x + from.width / 2),
            (int)(from.y + from.height / 2));
        Vector2Int end = new Vector2Int(
            (int)(to.x + to.width / 2),
            (int)(to.y + to.height / 2));

        // L形走廊
        int x = start.x;
        while (x != end.x)
        {
            map[x, start.y] = true;
            x += (end.x > start.x) ? 1 : -1;
        }
        int y = start.y;
        while (y != end.y)
        {
            map[end.x, y] = true;
            y += (end.y > start.y) ? 1 : -1;
        }
    }

    void ApplyToTilemap(bool[,] map)
    {
        for (int x = 0; x < mapWidth; x++)
        for (int y = 0; y < mapHeight; y++)
        {
            Vector3Int pos = new Vector3Int(x, y, 0);
            if (map[x, y])
            {
                groundTilemap.SetTile(pos, floorTile);
            }
            else
            {
                wallTilemap.SetTile(pos, wallTile);
            }
        }
    }

    void AddDecorations(bool[,] map, Rect[] rooms)
    {
        if (decorationTiles.Length == 0) return;

        foreach (var room in rooms)
        {
            int decorCount = Random.Range(1, 4);
            for (int i = 0; i < decorCount; i++)
            {
                int x = Random.Range((int)room.x + 1, (int)(room.x + room.width - 1));
                int y = Random.Range((int)room.y + 1, (int)(room.y + room.height - 1));
                Vector3Int pos = new Vector3Int(x, y, 0);

                TileBase tile = decorationTiles[Random.Range(0, decorationTiles.Length)];
                decorationTilemap.SetTile(pos, tile);
            }
        }
    }
}
```

## 最佳实践

- 地面、碰撞、装饰使用不同Tilemap层管理，便于单独编辑和性能优化
- Rule Tile减少手动选择贴图的工作量，一次配置自动适配所有邻接
- 使用Prefab笔刷批量放置场景物件（路灯、箱子、NPC）
- UE中合理设置Landscape Component密度平衡性能与精度
- 定期保存编辑器工作，使用版本控制管理地图数据
- 为关卡设计师创建自定义编辑器工具提升效率
- 使用Tilemap Collider 2D的Composite模式合并碰撞体

## 常见陷阱与修复

**陷阱1：Tilemap Collider 2D的Collider Type设置不当**
- 症状：大量瓦片导致物理性能问题
- 修复：使用Composite Collider 2D合并碰撞体，Collider Type设为Grid

**陷阱2：Rule Tile规则过多导致编辑器卡顿**
- 症状：编辑Tilemap时卡顿
- 修复：减少Rule Tile的规则数量，或使用脚本生成

**陷阱3：UE Landscape尺寸设置过大**
- 症状：编辑和运行时性能问题
- 修复：使用World Partition分区域管理

**陷阱4：忘记设置Tilemap的Order in Layer**
- 症状：瓦片渲染顺序错误，地面覆盖了角色
- 修复：为每层Tilemap设置正确的Order in Layer

**陷阱5：多个Tilemap层的Z轴位置冲突**
- 症状：不同层的瓦片显示重叠
- 修复：保持所有Tilemap在同一Z平面，使用Order in Layer控制排序
