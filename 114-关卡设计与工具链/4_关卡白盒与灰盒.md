# 关卡白盒与灰盒

## 核心概念

白盒（Whitebox）/灰盒（Graybox）是关卡开发的早期阶段，使用简单几何体搭建关卡结构，专注于游戏性验证而非美术表现。这是关卡开发中最重要却最容易被跳过的阶段。几乎所有3A游戏的关卡都经过白盒阶段。

### 为什么白盒阶段至关重要

- **快速验证设计想法**：2小时的白盒搭建可以验证一个设计想法，避免投入数天美术资源后发现设计不可行
- **避免返工**：美术替换后的关卡如果需要大改，浪费的是美术师的时间
- **专注于空间体验**：去掉视觉干扰，专注于空间大小、通道宽度、视线引导
- **方便团队协作讨论**：任何人都能从白盒理解空间布局，无需美术想象力
- **作为迭代基准**：白盒截图作为对比基准，记录迭代过程

### Blockout完整流程

1. **概念阶段**：纸上或绘图工具中规划空间布局和动线。画出平面图标注关键路径
2. **Blockout阶段**：用方块/圆柱搭建3D空间，仅关注尺寸和比例
3. **游戏性测试**：Playtest验证流程、节奏、视线。记录问题
4. **迭代优化**：根据测试反馈调整空间大小、通道宽度、敌人位置
5. **通过评审**：团队确认白盒阶段达标
6. **美术替换**：用最终资产替换白盒几何体
7. **最终调整**：替换后的微调

### 白盒比例参考

| 元素 | 推荐尺寸 | 说明 |
|------|---------|------|
| 角色身高 | 1.8m (2单位) | 参考基准 |
| 门高 | 2.4m | 角色身高+0.6m |
| 通道宽度 | 2-4m | 单人通道2m，双人4m |
| 房间高度 | 3-5m | 室内3m，大厅5m |
| 楼梯踏步 | 高0.2m × 深0.3m | 标准楼梯比例 |
| 掩体高度 | 1-1.2m | 半身掩体 |

## 具体实现方法

### Unity白盒搭建工具（完整版）

```csharp
using UnityEngine;

/// <summary>
/// 白盒生成工具
/// 快速生成各种白盒几何体
/// </summary>
public class WhiteboxBuilder : MonoBehaviour
{
    // 统一的白盒材质
    private static Material _wallMat;
    private static Material _floorMat;
    private static Material _ceilingMat;

    static Material WallMat
    {
        get
        {
            if (_wallMat == null)
            {
                _wallMat = new Material(Shader.Find("Standard"));
                _wallMat.color = new Color(0.7f, 0.7f, 0.7f);
                _wallMat.SetFloat("_Metallic", 0f);
                _wallMat.SetFloat("_Glossiness", 0.1f);
            }
            return _wallMat;
        }
    }

    static Material FloorMat
    {
        get
        {
            if (_floorMat == null)
            {
                _floorMat = new Material(Shader.Find("Standard"));
                _floorMat.color = new Color(0.5f, 0.5f, 0.5f);
                _floorMat.SetFloat("_Metallic", 0f);
                _floorMat.SetFloat("_Glossiness", 0.05f);
            }
            return _floorMat;
        }
    }

    /// <summary>
    /// 创建基础方块
    /// </summary>
    public static GameObject CreateBox(Vector3 pos, Vector3 size,
        Material mat = null, Transform parent = null)
    {
        GameObject go = GameObject.CreatePrimitive(PrimitiveType.Cube);
        go.transform.position = pos;
        go.transform.localScale = size;
        if (parent != null) go.transform.SetParent(parent);
        go.GetComponent<Renderer>().material = mat ?? WallMat;
        go.name = $"WB_Box_{pos}";
        return go;
    }

    /// <summary>
    /// 创建完整房间（地板+四面墙+天花板）
    /// </summary>
    public static GameObject[] CreateRoom(Vector3 center,
        float width, float height, float depth, bool hasCeiling = true)
    {
        var parent = new GameObject($"WB_Room_{center}");
        parent.transform.position = center;

        float wallThickness = 0.1f;

        // 地板
        CreateBox(center + Vector3.down * height / 2,
            new Vector3(width, wallThickness, depth),
            FloorMat, parent.transform);

        // 天花板
        if (hasCeiling)
            CreateBox(center + Vector3.up * height / 2,
                new Vector3(width, wallThickness, depth),
                FloorMat, parent.transform);

        // 前墙
        CreateBox(center + Vector3.forward * depth / 2,
            new Vector3(width, height, wallThickness),
            WallMat, parent.transform);

        // 后墙
        CreateBox(center + Vector3.back * depth / 2,
            new Vector3(width, height, wallThickness),
            WallMat, parent.transform);

        // 左墙
        CreateBox(center + Vector3.left * width / 2,
            new Vector3(wallThickness, height, depth),
            WallMat, parent.transform);

        // 右墙
        CreateBox(center + Vector3.right * width / 2,
            new Vector3(wallThickness, height, depth),
            WallMat, parent.transform);

        return new[] { parent };
    }

    /// <summary>
    /// 创建带门洞的墙
    /// </summary>
    public static void CreateWallWithDoor(Vector3 wallCenter,
        float wallWidth, float wallHeight, float doorWidth, float doorHeight)
    {
        float wallThickness = 0.1f;
        float sideWidth = (wallWidth - doorWidth) / 2;

        // 左侧墙体
        CreateBox(wallCenter + Vector3.left * (doorWidth / 2 + sideWidth / 2),
            new Vector3(sideWidth, wallHeight, wallThickness));

        // 右侧墙体
        CreateBox(wallCenter + Vector3.right * (doorWidth / 2 + sideWidth / 2),
            new Vector3(sideWidth, wallHeight, wallThickness));

        // 门上方墙体
        float topHeight = wallHeight - doorHeight;
        CreateBox(wallCenter + Vector3.up * (doorHeight / 2 + topHeight / 2),
            new Vector3(doorWidth, topHeight, wallThickness));
    }

    /// <summary>
    /// 创建走廊
    /// </summary>
    public static void CreateCorridor(Vector3 start, Vector3 end,
        float width, float height)
    {
        Vector3 center = (start + end) / 2;
        float length = Vector3.Distance(start, end);
        Vector3 direction = (end - start).normalized;

        // 地板
        CreateBox(center + Vector3.down * height / 2,
            new Vector3(width, 0.1f, length));

        // 根据方向旋转
    }
}
```

### UE白盒工具指南

```
UE内置Geometry Brush (BSP):
步骤:
1. Mode面板选择Geometry
2. 选择形状类型: Box, Sphere, Cylinder, Cone等
3. 设置为Additive（添加空间）或Subtractive（挖空空间）
4. 在场景中拖拽创建
5. 使用Clip工具切割几何体

更现代的方式（UE5推荐）:
1. 使用自带的Blockout材质球（灰色线框材质）
2. 快速放置基础Mesh组合
3. 使用Mesh体素化工具将BSP转为Mesh
4. 使用Modeling Tools插件进行高级白盒操作

关键提示:
- Subtractive brush用于在Additive空间中挖洞（如门、窗）
- BSP适合快速迭代，但性能不如Static Mesh
- 评审通过后应将BSP转换为Static Mesh
```

### 白盒工具编辑器窗口

```csharp
#if UNITY_EDITOR
using UnityEditor;
using UnityEngine;

/// <summary>
/// 白盒工具EditorWindow
/// 关卡设计师的快速搭建工具
/// </summary>
public class WhiteboxTool : EditorWindow
{
    [MenuItem("Tools/Whitebox Builder")]
    static void Open() => GetWindow<WhiteboxTool>("Whitebox Tool");

    private float roomWidth = 10f;
    private float roomHeight = 3f;
    private float roomDepth = 10f;
    private float corridorWidth = 3f;
    private float doorWidth = 2f;
    private float doorHeight = 2.4f;
    private float wallThickness = 0.2f;

    void OnGUI()
    {
        EditorGUILayout.LabelField("白盒工具", EditorStyles.boldLabel);

        EditorGUILayout.Space();
        EditorGUILayout.LabelField("房间设置", EditorStyles.boldLabel);
        roomWidth = EditorGUILayout.FloatField("宽度", roomWidth);
        roomHeight = EditorGUILayout.FloatField("高度", roomHeight);
        roomDepth = EditorGUILayout.FloatField("深度", roomDepth);

        EditorGUILayout.Space();
        EditorGUILayout.LabelField("走廊/门", EditorStyles.boldLabel);
        corridorWidth = EditorGUILayout.FloatField("走廊宽度", corridorWidth);
        doorWidth = EditorGUILayout.FloatField("门宽度", doorWidth);
        doorHeight = EditorGUILayout.FloatField("门高度", doorHeight);

        EditorGUILayout.Space();
        if (GUILayout.Button("创建房间 (Scene中心)"))
        {
            Vector3 center = SceneView.lastActiveSceneView.pivot
                + Vector3.up * roomHeight / 2;
            WhiteboxBuilder.CreateRoom(center, roomWidth, roomHeight, roomDepth);
        }

        if (GUILayout.Button("创建地板"))
        {
            Vector3 center = SceneView.lastActiveSceneView.pivot;
            WhiteboxBuilder.CreateBox(center,
                new Vector3(roomWidth, 0.1f, roomDepth));
        }

        if (GUILayout.Button("创建墙壁(带门洞)"))
        {
            Vector3 center = SceneView.lastActiveSceneView.pivot
                + Vector3.up * roomHeight / 2;
            WhiteboxBuilder.CreateWallWithDoor(center,
                roomWidth, roomHeight, doorWidth, doorHeight);
        }

        if (GUILayout.Button("创建楼梯"))
        {
            // 生成楼梯白盒
            CreateStairs();
        }

        if (GUILayout.Button("创建掩体"))
        {
            Vector3 center = SceneView.lastActiveSceneView.pivot;
            WhiteboxBuilder.CreateBox(center, new Vector3(2f, 1.1f, 0.5f));
        }
    }

    void CreateStairs()
    {
        int steps = 10;
        float stepHeight = roomHeight / steps;
        float stepDepth = 0.3f;

        Vector3 startPos = SceneView.lastActiveSceneView.pivot;

        for (int i = 0; i < steps; i++)
        {
            Vector3 pos = startPos + new Vector3(0, stepHeight * (i + 0.5f), stepDepth * i);
            WhiteboxBuilder.CreateBox(pos, new Vector3(corridorWidth, stepHeight, stepDepth));
        }
    }
}
#endif
```

## 最佳实践

- 白盒阶段使用统一的灰色材质，避免视觉干扰分散注意力
- 保持白盒几何体命名规范（Room_01, Corridor_01, Wall_WithDoor_01），便于后续替换
- 白盒阶段就进行碰撞测试，确保可玩性（跳跃高度、通道宽度）
- 邀请非设计团队成员进行白盒阶段的Playtest，获取新鲜视角
- 拍摄白盒截图作为对比基准，记录迭代过程
- 白盒通过评审后再投入美术资源
- 使用真实的游戏角色控制器测试白盒（而非自由视角相机）
- 记录每个白盒版本的修改日志

## 常见陷阱与修复

**陷阱1：白盒阶段过度美化**
- 症状：花时间在白盒上添加贴图和细节，偏离了验证游戏性的初衷
- 修复：白盒阶段只使用纯色材质，禁止添加任何美术细节

**陷阱2：白盒碰撞体与最终美术资产尺寸不符**
- 症状：替换美术后通道变窄导致玩家卡住
- 修复：白盒的碰撞体尺寸应作为最终美术资产的约束

**陷阱3：忽略白盒阶段的Playtest**
- 症状：直接跳到美术阶段，发现问题后大量返工
- 修复：白盒阶段必须至少进行3次Playtest

**陷阱4：白盒阶段未考虑光照对空间感受的影响**
- 症状：白盒时感觉宽敞的空间，加光照后感觉压抑
- 修复：白盒阶段就添加基本光照，感受空间氛围

**陷阱5：团队成员无法从白盒想象最终效果**
- 症状：评审时团队不理解设计师的意图
- 修复：提供参考图片和概念图，帮助团队想象最终效果
