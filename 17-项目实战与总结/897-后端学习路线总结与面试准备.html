后端面试准备

// ========== 常见面试题分类 ========== // 1. 语言基础 (以 Go 为例) // - defer 执行顺序 (LIFO) // - slice vs array // - channel 有无缓冲区别 // - goroutine 调度模型 (GMP) // - interface 底层结构 // // 2. 数据库 // - 索引结构 (B-Tree vs Hash) // - 隔离级别与 MVCC // - 慢查询优化 (EXPLAIN ANALYZE) // - 分库分表策略 // - 索引最左前缀原则 // // 3. 网络 // - TCP 三次握手/四次挥手 // - HTTPS TLS 握手流程 // - HTTP/2 多路复用 // - DNS 解析过程 // - Cookie vs Token vs Session // // 4. 系统设计 // - SNAKE 框架 // - 短链接、聊天、限流器 // - CAP 权衡 // - 缓存策略 // - 分布式事务 // // 5. 综合 // - 从输入 URL 到页面展示全过程 // - 如何设计高可用系统 // - 如何排查性能问题 // - 项目难点与解决方案 // ========== 系统设计面试框架 (SNAKE) ========== // S — Scope: 明确需求边界 // N — Non-functional: 估算 QPS/存储/延迟 // A — Abstract: 画架构图 (LB → Service → DB → Cache) // K — Key: 深入核心组件 (数据模型/算法/API) // E — Evaluate: 瓶颈 + 扩展 + 权衡 // ========== 编码题准备方向 ========== // 1. 数据结构: 数组/链表/树/哈希表/堆 // 2. 算法: 排序/二分/DFS/BFS/DP // 3. 语言特性: 并发/错误处理/接口 // 4. 系统设计: 限流器/LRU Cache/线程池 // 推荐: LeetCode Medium 为主 // Easy 20% + Medium 60% + Hard 20%
