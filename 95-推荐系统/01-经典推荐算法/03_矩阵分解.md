# 矩阵分解


## 1. 评分矩阵的稀疏性问题


### 1.1 用户-物品评分矩阵


推荐系统的核心数据是用户对物品的评分/行为矩阵R，其中R~u,i~表示用户u对物品i的评分。矩阵的行对应用户，列对应物品。


> **Example:** **评分矩阵示例：**
> 假设5个用户对4个物品的评分（1-5分，?表示未评分）
>
> |  | 物品1 | 物品2 | 物品3 | 物品4 |
> | --- | --- | --- | --- | --- |
> | 用户A | 5 | 3 | ? | 1 |
> | 用户B | 4 | ? | ? | 1 |
> | 用户C | ? | 1 | ? | 5 |
> | 用户D | ? | ? | 4 | 4 |
> | 用户E | 1 | ? | 5 | ? |


### 1.2 稀疏性挑战


在实际系统中，用户-物品矩阵极其稀疏。例如Netflix有上亿用户和数万部电影，但每个用户平均只评价了约200部电影，稀疏度约 **99.8%**。


- 直接使用协同过滤难以找到共同评分，相似度计算不准确
- 矩阵分解的目标是将稀疏的高维矩阵分解为低维稠密的隐向量表示
- 通过低秩近似，可以预测缺失的评分


## 2. SVD矩阵分解：R ≈ P × Q^T^


### 2.1 基本原理


矩阵分解将M×N的评分矩阵R分解为两个低秩矩阵的乘积：


- **P**
   （M×K）：用户隐因子矩阵，每行是一个用户的K维隐向量
- **Q**
   （N×K）：物品隐因子矩阵，每行是一个物品的K维隐向量
- **K**
   ：隐因子维度，通常取 20~200


$$
ru,i ≈ pu · qi = Σk=1K puk · qik
                RM×N ≈ PM×K × QN×KT
$$


> **Note:** **直觉理解：**
> 每个用户和每个物品都可以用K个隐因子来表示。例如对于电影推荐，隐因子可能代表"动作片程度"、"浪漫片程度"、"喜剧片程度"等。用户隐向量表示用户对各维度的偏好，物品隐向量表示物品在各维度的特征，两者的点积即为预测评分。


### 2.2 带偏置的矩阵分解（Bias SVD）


实际中用户评分存在系统性偏差：有的用户整体评分偏高，有的物品整体评分偏高。引入偏置项可以更好地捕捉这些模式：


$$
ru,i ≈ μ + bu + bi + pu · qi
                其中：μ = 全局平均分，bu = 用户偏置，bi = 物品偏置
$$


- μ：所有已知评分的平均值，反映整体评分水平
- b
   ~u~
   ：用户u的评分习惯偏差（有的用户从严，有的从宽）
- b
   ~i~
   ：物品i的固有质量偏差（好电影/差电影）


## 3. 损失函数与优化


### 3.1 损失函数（MSE + L2正则化）


$$
L = Σ(u,i)∈D (ru,i - μ - bu - bi - pu · qi)2 + λ(||pu||2 + ||qi||2 + bu2 + bi2)
$$


- 第一项：均方误差（MSE），衡量预测评分与真实评分的差距
- 第二项：L2正则化，防止过拟合，λ是正则化系数
- D：已知评分的(u,i)对集合


### 3.2 SGD（随机梯度下降）优化


对每个已知评分(u, i, r)计算预测误差 e~ui~ = r - &hatr，然后更新参数：


$$
bu ← bu + α(eui - λbu)
                bi ← bi + α(eui - λbi)
                pu ← pu + α(eui · qi - λpu)
                qi ← qi + α(eui · pu - λqi)
$$


其中α为学习率，λ为正则化系数。


## 4. ALS（交替最小二乘法）vs SGD


### 4.1 ALS算法


ALS（Alternating Least Squares）的核心思想：固定用户矩阵P，求解物品矩阵Q；然后固定Q，求解P。交替进行直到收敛。


- **优势**
   ：每个子问题是凸优化问题，有解析解；易于并行化
- **劣势**
   ：收敛速度较慢，需要更多迭代次数
- **适用**
   ：大规模分布式计算（Spark MLlib常用ALS）


### 4.2 SGD算法


每次随机选取一个评分样本，计算梯度并更新参数。


- **优势**
   ：收敛速度快，实现简单
- **劣势**
   ：需要调学习率，不易并行化
- **适用**
   ：中小规模数据集，快速原型验证


### 4.3 对比总结


| 维度 | SGD | ALS |
| --- | --- | --- |
| 收敛速度 | 快（少轮次） | 慢（多轮次） |
| 调参难度 | 需要调学习率 | 无需学习率 |
| 并行化 | 较难 | 容易（天然并行） |
| 适用数据 | 显式评分 | 显式/隐式反馈 |
| 工程实现 | 简单 | 需要矩阵运算 |
| 典型框架 | PyTorch/TF | Spark MLlib |


## 5. 隐式反馈的矩阵分解（BPR）


### 5.1 隐式反馈 vs 显式反馈


| 维度 | 显式反馈 | 隐式反馈 |
| --- | --- | --- |
| 数据来源 | 评分、点赞、评价 | 点击、浏览、购买、播放 |
| 数据含义 | 明确的偏好程度 | 存在偏好（二值化） |
| 负样本 | 明确的低分 | 没有观测到不等于不喜欢 |
| 数据量 | 较少 | 大量 |


### 5.2 BPR（Bayesian Personalized Ranking）


BPR是一种基于隐式反馈的排序优化方法，核心思想是对每个用户，优化物品对的排序关系：已交互物品应该排在未交互物品之前。


$$
BPR损失函数：
                LBPR = -ΣuΣ(i,j) ln σ(&hatrui - &hatruj) + λ||Θ||2
                其中：i为用户u交互过的物品，j为用户u未交互的物品，σ为sigmoid函数
$$


### 5.3 BPR的采样策略


- 对每个用户，遍历其交互过的物品i
- 随机采样一个未交互的物品j作为负样本
- 构建正负样本对(i, j)
- 优化使得正样本得分高于负样本


> **Note:** **注意：**
> 隐式反馈中"未交互"不等于"不喜欢"，可能是用户没有看到。因此负采样策略非常重要，常用的有随机负采样、热度负采样（按物品热度加权采样）等。


## 6. Python代码：用SGD实现矩阵分解


```
import numpy as np

class MatrixFactorization:
    """基于SGD的矩阵分解实现（Bias SVD）"""

    def __init__(self, n_users, n_items, n_factors=50, lr=0.01,
                 reg=0.02, n_epochs=100):
        self.n_users = n_users
        self.n_items = n_items
        self.n_factors = n_factors
        self.lr = lr           # 学习率
        self.reg = reg         # 正则化系数
        self.n_epochs = n_epochs

        # 随机初始化参数
        self.P = np.random.normal(0, 0.1, (n_users, n_factors))
        self.Q = np.random.normal(0, 0.1, (n_items, n_factors))
        self.bu = np.zeros(n_users)   # 用户偏置
        self.bi = np.zeros(n_items)   # 物品偏置
        self.mu = 0                    # 全局均值

    def fit(self, ratings):
        """
        ratings: list of (user_id, item_id, rating) tuples
        """
        # 计算全局均值
        all_ratings = [r for _, _, r in ratings]
        self.mu = np.mean(all_ratings)

        # SGD训练
        for epoch in range(self.n_epochs):
            np.random.shuffle(ratings)
            total_loss = 0

            for u, i, r in ratings:
                # 预测评分
                pred = self.mu + self.bu[u] + self.bi[i] + \
                       np.dot(self.P[u], self.Q[i])
                error = r - pred
                total_loss += error ** 2

                # 更新偏置
                self.bu[u] += self.lr * (error - self.reg * self.bu[u])
                self.bi[i] += self.lr * (error - self.reg * self.bi[i])

                # 更新隐向量
                P_u_old = self.P[u].copy()
                self.P[u] += self.lr * (error * self.Q[i] - self.reg * self.P[u])
                self.Q[i] += self.lr * (error * P_u_old - self.reg * self.Q[i])

            # 正则化损失
            reg_loss = self.reg * (np.sum(self.P**2) + np.sum(self.Q**2) +
                                  np.sum(self.bu**2) + np.sum(self.bi**2))
            total_loss += reg_loss

            if (epoch + 1) % 20 == 0:
                rmse = np.sqrt(total_loss / len(ratings))
                print(f"Epoch {epoch+1}/{self.n_epochs}, RMSE: {rmse:.4f}")

    def predict(self, user_id, item_id):
        """预测用户对物品的评分"""
        pred = self.mu + self.bu[user_id] + self.bi[item_id] + \
               np.dot(self.P[user_id], self.Q[item_id])
        return np.clip(pred, 1, 5)  # 限制在评分范围内

    def recommend(self, user_id, n=10, exclude_known=None):
        """为用户推荐Top-N物品"""
        scores = self.mu + self.bu[user_id] + self.bi + \
                 self.P[user_id] @ self.Q.T
        if exclude_known is not None:
            scores[exclude_known] = -np.inf
        return np.argsort(scores)[::-1][:n]


# 使用示例
if __name__ == "__main__":
    # (user_id, item_id, rating)
    ratings_data = [
        (0, 0, 5), (0, 1, 3), (0, 3, 1),
        (1, 0, 4), (1, 3, 1),
        (2, 1, 1), (2, 3, 5),
        (3, 2, 4), (3, 3, 4),
        (4, 0, 1), (4, 2, 5),
    ]

    mf = MatrixFactorization(n_users=5, n_items=4,
                              n_factors=10, lr=0.01,
                              reg=0.02, n_epochs=200)
    mf.fit(ratings_data)

    # 预测用户0对物品2的评分
    print(f"用户0对物品2的预测评分: {mf.predict(0, 2):.2f}")

    # 为用户0推荐Top-3物品
    print(f"用户0的Top-3推荐: {mf.recommend(0, n=3)}")
```


## 7. 矩阵分解的扩展与变体


| 变体 | 特点 | 适用场景 |
| --- | --- | --- |
| SVD++ | 加入用户隐式反馈（交互过的物品集合） | 有大量隐式反馈数据 |
| TimeSVD++ | 引入时间因子，建模用户兴趣漂移 | 新闻、社交媒体 |
| NMF | 非负矩阵分解，隐向量非负 | 需要可解释性 |
| PMF | 概率矩阵分解，贝叶斯框架 | 不确定性估计 |
| BPR-MF | BPR损失 + 矩阵分解 | 隐式反馈排序 |
| ConvMF | CNN提取物品内容特征 + MF | 物品冷启动 |


> **Important:** **关键要点：**
> 矩阵分解是推荐系统的里程碑算法，将协同过滤从基于记忆（memory-based）推向基于模型（model-based）的方法。它不仅解决了数据稀疏问题，还为后续的深度学习推荐模型（如FM、DeepFM）奠定了基础。FM（Factorization Machines）可以看作是矩阵分解的一般化形式。
