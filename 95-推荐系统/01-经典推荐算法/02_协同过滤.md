# 协同过滤算法


## 1. UserCF：基于用户的协同过滤


### 1.1 核心思想


基于用户的协同过滤（User-based Collaborative Filtering）的核心假设是：**兴趣相似的用户会喜欢相似的物品**。给定目标用户，找到与其兴趣最相似的K个用户，将这些用户喜欢但目标用户未接触过的物品推荐给目标用户。


### 1.2 算法步骤


1. 构建用户-物品评分/行为矩阵
2. 计算目标用户与其他所有用户的相似度
3. 选取最相似的K个用户（Top-K邻居）
4. 根据K个邻居的行为，预测目标用户对未接触物品的评分
5. 按预测评分排序，取Top-N作为推荐结果


$$
用户u对物品i的预测评分：
                p(u, i) = &barru + Σv∈S(u,K) sim(u, v) × (rv,i - &barrv) / Σv∈S(u,K) |sim(u, v)|
$$


### 1.3 Python实现


```
import numpy as np
from collections import defaultdict

def user_cf_predict(ratings, target_user, target_item, K=20):
    """
    ratings: dict, {user: {item: rating}}
    返回: 对target_user在target_item上的预测评分
    """
    # 计算目标用户的平均评分
    target_items = ratings[target_user]
    target_mean = np.mean(list(target_items.values()))

    # 计算与其他用户的相似度
    similarities = {}
    for user, items in ratings.items():
        if user == target_user:
            continue
        # 找到共同评分的物品
        common_items = set(target_items.keys()) & set(items.keys())
        if len(common_items) < 2:
            continue
        # 余弦相似度
        vec_u = [target_items[i] for i in common_items]
        vec_v = [items[i] for i in common_items]
        sim = np.dot(vec_u, vec_v) / (np.linalg.norm(vec_u) * np.linalg.norm(vec_v) + 1e-8)
        similarities[user] = sim

    # 选取Top-K相似用户
    top_k_users = sorted(similarities.items(), key=lambda x: x[1], reverse=True)[:K]

    # 加权预测
    numerator, denominator = 0, 0
    for user, sim in top_k_users:
        if target_item in ratings[user]:
            v_mean = np.mean(list(ratings[user].values()))
            numerator += sim * (ratings[user][target_item] - v_mean)
            denominator += abs(sim)

    if denominator == 0:
        return target_mean
    return target_mean + numerator / denominator
```


## 2. ItemCF：基于物品的协同过滤


### 2.1 核心思想


基于物品的协同过滤（Item-based Collaborative Filtering）的核心假设是：**被相似用户群体喜欢的物品是相似的**。通过分析用户行为矩阵，计算物品之间的相似度，推荐与用户历史喜欢物品相似的其他物品。


### 2.2 算法步骤


1. 构建用户-物品评分/行为矩阵
2. 计算物品之间的相似度矩阵
3. 根据用户历史行为，找到用户喜欢的物品集合
4. 利用物品相似度，预测用户对未接触物品的评分
5. 按预测评分排序推荐


$$
物品i与物品j的相似度：
                wij = |N(i) ∩ N(j)| / √(|N(i)| × |N(j)|)
N(i)表示对物品i有行为的用户集合
$$


### 2.3 Python实现


```
import numpy as np
from collections import defaultdict

def item_cf_recommend(ratings, target_user, K=10, N=5):
    """
    ratings: dict, {user: {item: rating}}
    K: 每个物品的相似物品数
    N: 推荐物品数
    """
    # 构建物品-用户倒排索引
    item_users = defaultdict(set)
    for user, items in ratings.items():
        for item in items:
            item_users[item].add(user)

    # 计算物品相似度矩阵
    items = list(item_users.keys())
    sim_matrix = {}
    for i in items:
        for j in items:
            if i >= j:
                continue
            common_users = item_users[i] & item_users[j]
            if len(common_users) == 0:
                continue
            sim = len(common_users) / (np.sqrt(len(item_users[i]) * len(item_users[j])) + 1e-8)
            sim_matrix[(i, j)] = sim
            sim_matrix[(j, i)] = sim

    # 获取目标用户已交互物品
    user_items = set(ratings[target_user].keys())

    # 计算推荐分数
    scores = defaultdict(float)
    for item_i in user_items:
        # 找到与item_i最相似的K个物品
        item_sims = [(j, sim_matrix.get((item_i, j), 0)) for j in items if j not in user_items]
        item_sims.sort(key=lambda x: x[1], reverse=True)
        for item_j, sim in item_sims[:K]:
            scores[item_j] += ratings[target_user][item_i] * sim

    # 返回Top-N推荐
    recommended = sorted(scores.items(), key=lambda x: x[1], reverse=True)[:N]
    return recommended
```


## 3. 相似度计算方法


### 3.1 余弦相似度（Cosine Similarity）


$$
sim(x, y) = x · y / (||x|| × ||y||) = Σxiyi / √(Σxi2) √(Σyi2)
$$


衡量两个向量在方向上的相似程度，值域为[-1, 1]，值越大越相似。是最常用的相似度计算方法。


### 3.2 皮尔逊相关系数（Pearson Correlation）


$$
sim(x, y) = Σ(xi - &barx)(yi - &bary) / √(Σ(xi - &barx)2) √(Σ(yi - &bary)2)
$$


在余弦相似度的基础上减去了各自的均值，消除用户评分偏好的影响（有的用户评分整体偏高/偏低）。


### 3.3 其他相似度方法


| 方法 | 公式/描述 | 特点 |
| --- | --- | --- |
| Jaccard相似度 | \|A ∩ B\| / \|A ∪ B\| | 适用于隐式反馈（0/1），不考虑评分值 |
| 修正余弦 | 减去用户评分均值后计算余弦 | 消除用户评分偏差 |
| 欧氏距离 | d = √(Σ(x~i~ - y~i~)^2^) | 距离越小越相似，需归一化 |
| Tanimoto系数 | \|A ∩ B\| / (\|A\| + \|B\| - \|A ∩ B\|) | Jaccard的变体 |


## 4. UserCF vs ItemCF 对比


| 维度 | UserCF | ItemCF |
| --- | --- | --- |
| 核心思想 | 相似用户喜欢相似物品 | 相似物品被相似用户喜欢 |
| 适用场景 | 用户数较少、物品数多 | 物品数较少、用户数多 |
| 实时性 | 用户兴趣变化需重新计算 | 物品相似度相对稳定 |
| 可解释性 | 较弱（和你相似的人...） | 较强（你看过A，推荐类似的B） |
| 冷启动 | 新用户难以找到邻居 | 新物品无交互数据 |
| 计算复杂度 | O(M²) | O(N²) |
| 典型应用 | 新闻推荐（时效性强） | 电商、视频推荐 |


> **Note:** **工业实践经验：**
> 在大多数电商和视频平台中，ItemCF的应用更为广泛。因为物品数量相对稳定，物品相似度矩阵可以离线预计算，线上推荐时只需查找用户历史行为对应的相似物品即可，效率更高。Amazon早期的推荐系统就主要基于ItemCF。


## 5. 热门物品问题与改进策略


### 5.1 热门物品问题


热门物品被大多数用户交互过，导致它与几乎所有物品都有较高的相似度，容易主导推荐结果，降低推荐的多样性和新颖性。


### 5.2 惩罚策略


### IUF（Inverse User Frequency）惩罚


对活跃用户（交互大量物品的用户）在计算物品相似度时给予较低的权重。


$$
wij = Σu∈N(i)∩N(j) 1 / log(1 + |N(u)|) / √(|N(i)| × |N(j)|)
$$


### 物品热度惩罚


$$
wij = |N(i) ∩ N(j)| / (|N(i)| × |N(j)|)α，α ∈ [0.5, 1]
$$


当α=0.5时退化为标准公式，α越大对热门物品的惩罚越强。


### 5.3 其他改进方向


- **时间衰减**
   ：近期行为赋予更高权重，λ随时间衰减
- **行为类型加权**
   ：购买 > 加购 > 收藏 > 浏览，不同行为赋予不同权重
- **位置衰减**
   ：在用户行为序列中，靠后的行为权重更高


## 6. 优缺点分析与适用场景


### 6.1 优点


- 不需要领域知识和物品特征，完全依赖用户行为数据
- 能发现用户潜在的兴趣（Serendipity），推荐新颖物品
- 算法简单直观，易于实现和理解
- 随着时间推移和数据积累，推荐效果会持续改善


### 6.2 缺点


- **冷启动问题**
   ：新用户/新物品缺乏行为数据，无法有效推荐
- **数据稀疏**
   ：用户-物品矩阵极其稀疏，相似度计算不准确
- **可扩展性**
   ：用户/物品数量大时，相似度计算开销大（O(N²)）
- **热门物品偏差**
   ：热门物品容易被过度推荐
- **实时性差**
   ：用户兴趣变化后，系统响应不够及时


### 6.3 适用场景


| 场景 | 推荐方案 | 原因 |
| --- | --- | --- |
| 新闻/资讯推荐 | UserCF | 内容时效性强，物品生命周期短，相似用户更稳定 |
| 电商推荐 | ItemCF | 商品相对稳定，可解释性好（"买了A的人也买了B"） |
| 视频推荐 | ItemCF + 深度模型 | 视频库庞大但稳定，需结合深度特征 |
| 社交网络 | UserCF + 图方法 | 天然存在社交图结构，用户关系丰富 |


## 7. 工业实践与演进


### 7.1 从传统CF到矩阵分解


传统协同过滤直接在原始稀疏矩阵上计算相似度，效率低且效果受限。矩阵分解（SVD/ALS）通过将用户和物品映射到低维隐向量空间，缓解了数据稀疏问题，成为主流方案。


### 7.2 从矩阵分解到图神经网络


图神经网络（如LightGCN、NGCF）将用户-物品交互建模为二部图，通过多层消息传递捕获高阶协同信号，进一步提升了推荐效果。


> **Example:** **LightGCN核心思想：**
> 在用户-物品交互图上进行多层嵌入传播，将每一层的嵌入加权求和得到最终表示。相比NGCF去掉了特征变换和非线性激活，只保留了邻域聚合，效果反而更好。


### 7.3 协同过滤在工业系统中的位置


在现代工业推荐系统中，协同过滤通常作为**多路召回中的一路**存在，与向量召回、热门召回、内容召回等并行工作，而非作为唯一的推荐方案。
