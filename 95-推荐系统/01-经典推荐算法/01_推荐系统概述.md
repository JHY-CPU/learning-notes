# 推荐系统概述


## 1. 推荐系统定义与价值


### 1.1 什么是推荐系统


推荐系统（Recommendation System）是一种信息过滤系统，通过分析用户的历史行为、兴趣偏好和物品特征，主动向用户推荐可能感兴趣的物品或内容。它旨在解决**信息过载**（Information Overload）问题，帮助用户在海量信息中高效地发现感兴趣的内容。


### 1.2 推荐系统的核心价值


- **用户侧**
   ：降低信息搜索成本，提升用户体验，帮助发现长尾内容
- **平台侧**
   ：提高用户活跃度、留存率和转化率，增加平台收入
- **供给侧**
   ：帮助优质内容/商品获得更多曝光机会


> **Example:** **实际案例：**
>
> - Netflix：约80%的用户观看来自推荐系统
> - Amazon：约35%的销售额来自推荐系统
> - YouTube：推荐系统贡献了超过70%的视频观看时长
> - 抖音/TikTok：几乎100%的内容消费来自推荐流


### 1.3 推荐系统与搜索引擎的区别


| 维度 | 搜索引擎 | 推荐系统 |
| --- | --- | --- |
| 用户意图 | 明确（主动搜索） | 模糊/无（被动接收） |
| 触发方式 | 用户输入查询词 | 系统主动推送 |
| 个性化程度 | 中等 | 高度个性化 |
| 典型场景 | Google、百度搜索 | 抖音推荐、淘宝猜你喜欢 |
| 核心挑战 | 查询理解与文档排序 | 用户兴趣建模与冷启动 |


## 2. 推荐系统分类


### 2.1 基于内容的推荐（Content-Based Filtering）


根据用户历史喜欢的物品特征，推荐具有相似特征的其他物品。核心思想是"推荐与用户过去喜欢的物品相似的物品"。


- **流程**
   ：物品特征提取 → 用户画像构建 → 相似度计算 → 推荐
- **优点**
   ：不需要其他用户数据，不存在冷启动物品问题，可解释性强
- **缺点**
   ：特征工程依赖领域知识，无法发现用户潜在兴趣，新用户冷启动


### 2.2 协同过滤（Collaborative Filtering）


利用群体行为数据进行推荐，分为基于用户的协同过滤（UserCF）和基于物品的协同过滤（ItemCF）。


- **UserCF**
   ：找到与目标用户兴趣相似的用户，推荐相似用户喜欢的物品
- **ItemCF**
   ：找到与用户历史喜欢物品相似的物品进行推荐


### 2.3 混合推荐（Hybrid Recommendation）


将多种推荐策略进行组合，取长补短。工业界几乎都采用混合推荐方案。


> **Note:** **混合策略：**
> 加权混合、切换策略、级联策略、特征增强策略等。在实际工业系统中，通常是协同过滤+内容推荐+深度学习模型的多路召回+精排架构。


### 2.4 推荐算法分类总览


| 类别 | 代表算法 | 核心思想 | 数据需求 |
| --- | --- | --- | --- |
| 基于内容 | TF-IDF、余弦相似度 | 物品特征相似 | 物品属性特征 |
| 协同过滤 | UserCF、ItemCF | 群体行为相似 | 用户行为数据 |
| 矩阵分解 | SVD、ALS、BPR | 隐向量表示 | 评分/行为矩阵 |
| 深度学习 | Wide&Deep、DIN、DIEN | 端到端特征学习 | 大规模多源数据 |
| 知识图谱 | KGAT、RippleNet | 知识增强推荐 | 知识图谱+行为 |
| 强化学习 | DQN、Actor-Critic | 长期收益优化 | 用户实时反馈 |


## 3. 工业级推荐系统架构


工业级推荐系统通常采用**漏斗架构**，从海量候选集中逐层筛选出最终推荐给用户的少量结果。


### 3.1 召回层（Retrieval / Recall）


从百万甚至亿级候选集中快速筛选出数千个候选物品。


- **多路召回**
   ：协同过滤召回、热门召回、内容召回、向量召回（双塔模型）
- **特点**
   ：模型简单、速度快、覆盖度高，允许一定的噪声
- **规模**
   ：从N（亿级）缩减到数千


### 3.2 粗排层（Pre-ranking）


轻量级模型对召回结果进行初步排序，将候选集从数千缩减到数百。


- 使用浅层网络或简单交叉特征
- 延迟要求：单次请求 < 10ms


### 3.3 精排层（Ranking）


使用复杂的深度学习模型对候选集进行精确排序。


- 模型：DeepFM、DIN、DIEN、MMOE等
- 多目标融合：CTR、CVR、停留时长等综合打分
- 延迟要求：单次请求 < 100ms


### 3.4 重排层（Re-ranking）


考虑业务规则和用户体验对排序结果进行最终调整。


- 多样性保障（DPP、MMR）
- 去重与频控
- 业务规则插入（广告、置顶等）
- 实时调控（流量扶持、打压等）


```
推荐系统漏斗架构：

┌─────────────────────────────────┐
│         全量候选集（亿级）         │  ← 物品库
├─────────────────────────────────┤
│     召回层（多路召回 → 数千）       │  ← 快速粗筛
├─────────────────────────────────┤
│      粗排层（轻量模型 → 数百）      │  ← 初步排序
├─────────────────────────────────┤
│     精排层（深度模型 → 数十）       │  ← 精确排序
├─────────────────────────────────┤
│    重排层（业务规则 → 最终展示）     │  ← 最终调整
└─────────────────────────────────┘
```


## 4. 推荐系统核心问题


### 4.1 冷启动问题（Cold Start）


| 类型 | 描述 | 常见解决方案 |
| --- | --- | --- |
| 用户冷启动 | 新用户无历史行为数据 | 热门推荐、引导填写兴趣、利用注册信息、跨域迁移 |
| 物品冷启动 | 新物品无用户交互记录 | 基于内容推荐、利用物品属性、探索与利用策略 |
| 系统冷启动 | 新系统无任何数据 | 利用专家标注、迁移学习、基于规则推荐 |


### 4.2 数据稀疏问题（Data Sparsity）


用户-物品交互矩阵通常非常稀疏（稀疏度 > 99%），导致协同过滤效果受限。


- 矩阵分解（SVD/ALS）将稀疏矩阵映射到低维稠密空间
- 利用辅助信息（社交关系、物品属性）补充数据
- 图神经网络（GNN）挖掘高阶连接关系


### 4.3 实时性问题


用户兴趣实时变化，系统需要快速捕捉并响应。


- **实时特征**
   ：最近行为序列、实时统计特征
- **在线学习**
   ：增量更新模型参数
- **实时架构**
   ：Flink/Kafka实时特征计算流水线


> **Important:** **其他挑战：**
>
> - **可解释性**
>    ：用户需要理解"为什么推荐这个"
> - **公平性**
>    ：避免算法偏见，保证供给端公平曝光
> - **隐私保护**
>    ：联邦学习、差分隐私等隐私计算技术
> - **信息茧房**
>    ：过度个性化导致用户视野狭窄


## 5. 评估指标


### 5.1 预测评准确类指标


| 指标 | 公式/定义 | 适用场景 |
| --- | --- | --- |
| 准确率（Precision） | P = 推荐中相关的 / 推荐总数 | Top-N推荐 |
| 召回率（Recall） | R = 推荐中相关的 / 所有相关的 | Top-N推荐 |
| F1-Score | F1 = 2PR / (P + R) | 综合评估 |
| MAP | 平均精确率均值 | 排序任务 |
| NDCG | 归一化折损累积增益 | 排序质量评估 |


$$
NDCG = DCG / IDCG
                DCG = Σ (2reli - 1) / log2(i + 1)
$$


### 5.2 商业指标


| 指标 | 全称 | 含义 |
| --- | --- | --- |
| CTR | Click-Through Rate | 点击率 = 点击数 / 曝光数 |
| CVR | Conversion Rate | 转化率 = 转化数 / 点击数 |
| GMV | Gross Merchandise Volume | 成交总额 |
| 人均停留时长 | - | 用户平均使用时长 |
| DAU/MAU | - | 日/月活跃用户数 |


### 5.3 体验类指标


- **多样性（Diversity）**
   ：推荐列表中物品类型的丰富程度
- **新颖性（Novelty）**
   ：推荐用户此前未知物品的程度
- **惊喜度（Serendipity）**
   ：推荐出乎意料但用户满意的结果
- **覆盖率（Coverage）**
   ：推荐系统能推荐的物品占总物品的比例


> **Note:** **注意：**
> 离线指标好不等于线上效果好。需要通过A/B测试验证实际效果。NDCG常用于离线评估排序质量，CTR/CVR常用于线上A/B实验评估。


## 6. 主流应用场景


| 场景 | 平台 | 推荐特点 | 核心指标 |
| --- | --- | --- | --- |
| 电商推荐 | 淘宝、京东、Amazon | 商品关联、购买转化导向 | CVR、GMV |
| 短视频 | 抖音、快手、YouTube | 沉浸式信息流、注重时长 | 完播率、停留时长 |
| 新闻资讯 | 今日头条、Google News | 时效性强、内容新鲜度 | CTR、阅读时长 |
| 音乐推荐 | Spotify、网易云音乐 | 场景化、情绪化推荐 | 播放完成率、收藏 |
| 社交网络 | 微博、Facebook、Twitter | 关注推荐、好友发现 | 互动率、关注转化 |
| 广告系统 | Google Ads、巨量引擎 | eCPM排序、竞价排名 | CTR、CVR、eCPM |
| 搜索推荐 | 淘宝搜索、Amazon搜索 | 查询相关性+个性化 | NDCG、转化率 |


### 不同场景的推荐策略差异


> **Example:** **电商场景：**
> 用户购买行为稀疏，需要结合浏览、加购、收藏等多信号；商品生命周期短（季节性），物品冷启动频繁；推荐需兼顾多样性和转化效率。


> **Example:** **短视频场景：**
> 用户行为密集（每分钟可产生多个交互），需要快速捕捉兴趣变化；沉浸式推荐容易造成信息茧房，需引入探索机制；完播率是比点击率更有意义的信号。


## 7. 推荐系统发展趋势


- **大模型赋能推荐**
   ：利用LLM进行用户兴趣理解、冷启动、特征增强
- **图神经网络（GNN）**
   ：如GraphSAGE、LightGCN，利用用户-物品交互图进行高阶消息传递
- **因果推断**
   ：解决推荐中的偏差问题，如位置偏差、曝光偏差
- **强化学习**
   ：将推荐视为序列决策问题，优化长期收益
- **跨域推荐**
   ：利用其他领域的知识提升目标领域的推荐效果
- **可解释推荐**
   ：提高推荐系统的透明度和用户信任度
- **端云协同**
   ：在端侧部署轻量模型，实现超低延迟推荐
