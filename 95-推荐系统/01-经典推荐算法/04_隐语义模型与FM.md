# 隐语义模型与FM


## 1. 隐语义模型（LFM）


### 1.1 基本概念


隐语义模型（Latent Factor Model）的核心思想是：**通过隐含的语义维度来表示用户兴趣和物品特征**。它假设用户和物品之间存在一些不可直接观测的隐含因子（Latent Factor），这些因子可以解释用户的评分行为。


### 1.2 隐因子的直觉理解


> **Example:** **电影推荐的例子：**
>
>
> 假设用5个隐因子来描述电影：
>
> - 因子1：动作程度
> - 因子2：浪漫程度
> - 因子3：喜剧程度
> - 因子4：科幻程度
> - 因子5：文艺程度
>
> 用户隐向量表示用户对各维度的偏好程度，物品隐向量表示物品在各维度的特征强度。预测评分 = 用户偏好向量 · 物品特征向量。


### 1.3 LFM与协同过滤的关系


- UserCF和ItemCF直接在原始交互矩阵上计算相似度
- LFM通过学习隐向量，在低维稠密空间中计算相似度
- LFM缓解了数据稀疏问题，且具有更好的泛化能力
- 矩阵分解（SVD）是LFM的一个特例


> **Note:** **隐因子数K的选择：**
> K越大模型表达能力越强，但也越容易过拟合。实践中通常通过交叉验证选择最优K值，常见范围为20~200。Netflix Prize竞赛中获胜方案使用了约50~100个隐因子。


## 2. FM（Factorization Machines）


### 2.1 FM的动机


在CTR预估任务中，特征之间的交叉组合非常重要。例如"用户性别=男"和"物品类别=球鞋"的组合信号比单独的特征更强。传统方法（如逻辑回归）需要手动设计交叉特征，FM通过自动学习特征交叉来解决这一问题。


### 2.2 FM模型公式


$$
y(x) = w0 + Σi=1n wixi + Σi=1nΣj=i+1n <vi, vj> xixj
                其中：
                w0：全局偏置
                wi：第i个特征的权重
                vi ∈ Rk：第i个特征的k维隐向量
                <vi, vj>：特征i和j的隐向量内积
$$


### 2.3 FM的计算优化


直接计算二阶交叉项的复杂度为O(kn²)，通过数学变换可以优化到O(kn)：


$$
ΣiΣj>i <vi, vj>xixj = ½ Σf=1k [(Σi=1n vi,fxi)2 - Σi=1n vi,f2xi2]
$$


### 2.4 FM的核心优势


- **自动特征交叉**
   ：无需手动设计交叉特征，自动学到二阶特征交互
- **稀疏数据友好**
   ：即使特征对在训练集中从未共现，仍可通过隐向量内积估计其交互
- **线性时间复杂度**
   ：O(kn)的计算复杂度，可处理高维稀疏特征
- **通用性强**
   ：可用于回归、分类、排序等任务


## 3. FM vs 矩阵分解的关系


FM是矩阵分解的一般化形式。矩阵分解可以看作FM的一个特例：


### 3.1 从矩阵分解到FM


> **Example:** **矩阵分解的FM视角：**
>
>
> 将用户ID和物品ID分别做one-hot编码，得到特征向量x。
>
>
> 用户u的one-hot位置为1，物品i的one-hot位置为1，其余为0。
>
>
> FM公式退化为：y = w
> ~0~
> + w
> ~u~
> + w
> ~i~
> + <v
> ~u~
> , v
> ~i~
> >
>
>
> 这正好对应 Bias SVD：y = μ + b
> ~u~
> + b
> ~i~
> + p
> ~u~
> · q
> ~i~


### 3.2 FM的扩展性


| 能力 | 矩阵分解 | FM |
| --- | --- | --- |
| 输入特征 | 仅用户ID + 物品ID | 任意特征 |
| 特征交叉 | 仅用户×物品 | 任意二阶特征交叉 |
| 辅助信息 | 难以融入 | 自然融入（年龄、性别、品类等） |
| 冷启动 | 无法处理 | 可用新用户/物品的特征 |
| 任务类型 | 评分预测 | 回归/分类/排序 |


## 4. FFM（Field-aware Factorization Machines）


### 4.1 FFM的核心思想


在FM中，每个特征只有一个隐向量。FFM认为特征与其他不同Field（字段域）的特征交叉时，应该使用不同的隐向量。


$$
y(x) = w0 + Σi=1n wixi + Σi=1nΣj=i+1n <vi,fj, vj,fi> xixj
$$


### 4.2 Field的概念


> **Example:** **示例：**
> CTR预估中的特征分Field
>
>
>
> | Field | 特征 |
> | --- | --- |
> | 用户域 | 性别、年龄、城市 |
> | 广告域 | 广告ID、广告主、行业 |
> | 上下文域 | 时间、设备、位置 |
>
> "性别=男"这个特征在与"广告域"交叉时使用隐向量v
> ~性别,广告~
> ，在与"上下文域"交叉时使用v
> ~性别,上下文~
> 。


### 4.3 FM vs FFM 对比


| 维度 | FM | FFM |
| --- | --- | --- |
| 隐向量数 | 每特征1个（k维） | 每特征F个（F×k维） |
| 参数量 | O(nk) | O(nFk) |
| 表达能力 | 中等 | 更强 |
| 训练复杂度 | O(kn) | O(kn²) |
| 适用场景 | 通用 | 特征Field明确 |
| Criteo冠军 | - | 2016 Criteo CTR竞赛冠军 |


> **Important:** **FFM的代价：**
> 参数量是FM的F倍（F为Field数量），训练和存储成本大幅增加。在工业场景中，当特征Field划分合理时，FFM相比FM有明显提升，但需要注意过拟合风险。


## 5. Python代码：FM模型实现


```
import numpy as np

class FactorizationMachine:
    """二分类FM模型实现"""

    def __init__(self, n_features, k_factors=10, lr=0.01,
                 reg=0.01, n_epochs=100):
        self.n_features = n_features
        self.k = k_factors
        self.lr = lr
        self.reg = reg
        self.n_epochs = n_epochs

        # 初始化参数
        self.w0 = 0                           # 全局偏置
        self.w = np.zeros(n_features)         # 一阶权重
        self.V = np.random.normal(0, 0.1,     # 二阶隐向量矩阵
                (n_features, k_factors))

    def _sigmoid(self, x):
        return 1.0 / (1.0 + np.exp(-np.clip(x, -500, 500)))

    def predict(self, X):
        """X: (batch_size, n_features) 稀疏或稠密矩阵"""
        # 一阶项
        linear = np.dot(X, self.w) + self.w0

        # 二阶项（优化计算 O(kn)）
        # term1 = (XV)^2
        XV = np.dot(X, self.V)          # (batch, k)
        term1 = np.sum(XV ** 2, axis=1) # (batch,)
        # term2 = X^2 V^2
        X_sq_V_sq = np.dot(X ** 2, self.V ** 2) # (batch, k)
        term2 = np.sum(X_sq_V_sq, axis=1)
        # 二阶交叉 = 0.5 * (term1 - term2)
        interaction = 0.5 * (term1 - term2)

        y_hat = self._sigmoid(linear + interaction)
        return y_hat

    def fit(self, X, y):
        """训练FM模型，X为特征矩阵，y为标签(0/1)"""
        n_samples = X.shape[0]

        for epoch in range(self.n_epochs):
            # 随机打乱
            indices = np.random.permutation(n_samples)
            total_loss = 0

            for idx in indices:
                x_i = X[idx]
                y_true = y[idx]

                # 预测
                y_pred = self.predict(x_i.reshape(1, -1))[0]
                error = y_pred - y_true
                total_loss += -y_true * np.log(y_pred + 1e-8) - \
                              (1 - y_true) * np.log(1 - y_pred + 1e-8)

                # 梯度更新
                # w0
                self.w0 -= self.lr * error

                # w (一阶)
                self.w -= self.lr * (error * x_i + self.reg * self.w)

                # V (二阶隐向量)
                # 优化的梯度计算
                XV_sum = np.dot(x_i, self.V)  # (k,)
                for f in range(self.k):
                    for i in range(self.n_features):
                        if x_i[i] == 0:
                            continue
                        grad = error * x_i[i] * (XV_sum[f] - self.V[i, f] * x_i[i])
                        self.V[i, f] -= self.lr * (grad + self.reg * self.V[i, f])

            if (epoch + 1) % 20 == 0:
                avg_loss = total_loss / n_samples
                print(f"Epoch {epoch+1}/{self.n_epochs}, "
                      f"Log Loss: {avg_loss:.4f}")


# 使用示例
if __name__ == "__main__":
    # 生成模拟CTR数据
    np.random.seed(42)
    n_samples, n_features = 1000, 20
    X = np.random.randint(0, 2, (n_samples, n_features)).astype(float)
    y = (np.random.rand(n_samples) > 0.5).astype(float)

    fm = FactorizationMachine(n_features=n_features, k_factors=8,
                               lr=0.01, n_epochs=100)
    fm.fit(X, y)

    # 预测
    test_X = np.random.randint(0, 2, (5, n_features)).astype(float)
    predictions = fm.predict(test_X)
    print(f"预测概率: {predictions}")
```


## 6. FM的演进与现代变体


| 模型 | 改进点 | 来源 |
| --- | --- | --- |
| FM | 隐向量二阶特征交叉 | Rendle, 2010 |
| FFM | Field-aware隐向量 | Juan et al., 2016 |
| DeepFM | FM + DNN，端到端训练 | Guo et al., 2017 |
| xDeepFM | FM + CIN（压缩交互网络） | Lian et al., 2018 |
| AFM | 注意力机制加权特征交叉 | Xiao et al., 2017 |
| AutoInt | 自注意力自动学习特征交互 | Song et al., 2019 |
| FiBiNET | 双线性交互+SENet特征提取 | Huang et al., 2019 |


> **Note:** **FM的历史地位：**
> FM是推荐系统从传统机器学习向深度学习过渡的关键桥梁。它提出的"用隐向量内积代替独立权重"的思想，直接影响了后续所有深度推荐模型的Embedding层设计。DeepFM等模型本质上就是在FM的基础上加入了深度神经网络分支。


### 6.1 从FM到深度学习推荐模型


- FM的隐向量 = 深度推荐模型中的Embedding
- FM的二阶交叉 = DNN的低阶特征交互
- DeepFM = FM（低阶交叉）+ DNN（高阶交叉）
- 现代工业系统中FM常作为Baseline或与其他模型组合使用
