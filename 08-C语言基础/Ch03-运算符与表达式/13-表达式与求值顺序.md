# 13 - 表达式与求值顺序

## 一、表达式概述

### 1.1 什么是表达式

表达式（Expression）是由**运算符和操作数**组成的式子，用于计算一个值。每个表达式都有一个**类型**和一个**值**。

```c
// 基本表达式
42                // 常量表达式，值为42
x                 // 变量表达式，值为x的值
x + 3             // 算术表达式
a > b             // 关系表达式，值为0或1
x = 5             // 赋值表达式，值为5
x++               // 后缀自增表达式，值为x自增前的值
func(a, b)        // 函数调用表达式
(x > 0) ? x : -x  // 条件表达式
```

### 1.2 表达式的分类

| 类别 | 示例 | 说明 |
|------|------|------|
| 初等表达式 | `42`、`x`、`(a+b)` | 常量、变量、括号表达式 |
| 后缀表达式 | `a[i]`、`p->x`、`f()`、`x++` | 下标、成员、函数调用、后缀自增 |
| 一元表达式 | `++x`、`-a`、`!flag`、`*p`、`&x` | 前缀自增、单目运算、解引用等 |
| 二元表达式 | `a + b`、`x && y` | 双目运算 |
| 条件表达式 | `a ? b : c` | 三目运算 |
| 赋值表达式 | `a = b`、`a += b` | 赋值运算 |

## 二、完整表达式

**完整表达式（Full Expression）** 是指不是另一个表达式或声明的子表达式的表达式。在完整表达式结束处存在一个**序列点**。

```c
// 完整表达式示例
int x = a + b;           // a + b 是完整表达式
func(a++, b--);          // 整个函数调用是完整表达式
if (x > 0) { }           // x > 0 是完整表达式
return a + b;            // a + b 是完整表达式
```

## 三、求值顺序

### 3.1 优先级和结合性不等于求值顺序

**这是一个非常重要的概念**：运算符的优先级和结合性决定了表达式的**分组方式**，但**不一定**决定了操作数的求值顺序。

```c
int a = 1, b = 2, c = 3;
int result = a + b * c;

// 优先级决定了分组：a + (b * c)
// 但 a 和 (b * c) 谁先求值？C标准没有规定！
// 编译器可以先算 a，也可以先算 b * c
// 但最终结果保证是 7
```

### 3.2 未指定求值顺序的情况

C标准规定以下运算符的操作数求值顺序是**未指定的（Unspecified）**：

```c
// 1. 双目运算符的两个操作数
int a = f() + g();    // f() 和 g() 的调用顺序未指定

// 2. 函数参数的求值顺序
printf("%d %d\n", f(), g());   // f() 和 g() 的调用顺序未指定

// 3. 赋值表达式两边
arr[i] = i++;   // 未定义行为！arr[i]中的i和i++对i的修改冲突
```

### 3.3 保证求值顺序的情况

以下运算符**保证从左到右求值**，并且在左操作数完成后存在序列点：

```c
// 1. &&（逻辑与）—— 保证从左到右，且支持短路
if (ptr != NULL && ptr->value > 0) { }  // 先判断ptr

// 2. ||（逻辑或）—— 保证从左到右，且支持短路
if (index < 0 || index >= size) { }     // 先判断index

// 3. ?:（条件运算符）—— 条件先求值，然后只求值选中的分支
int x = (a > b) ? a : b;   // a > b 先求值

// 4. ,（逗号运算符）—— 保证从左到右
int x = (a = 1, b = 2, a + b);  // 顺序执行

// 5. && 和 || 的操作数之间
//    左操作数求值后存在序列点
```

## 四、序列点（Sequence Point）

### 4.1 定义

序列点是程序执行中的一个**位置**，保证在此之前的所有**副作用**（Side Effects）都已经完成，并且之后的操作还未开始。

### 4.2 C语言中的序列点位置

1. `&&`、`||`、`?:` 和 `,` 运算符的左操作数之后
2. 完整表达式结束处（分号 `;`）
3. 函数调用时，所有实参求值完成之后、函数体执行之前
4. 函数返回时，返回值计算完成之后
5. 初始化完成之后
6. `<stdlib.h>` 中的 `qsort` 比较函数调用之间

### 4.3 序列点的作用

```c
// 序列点保证副作用完成
int i = 0;
if (i++ == 0 || i++ == 1) { }
// || 的左操作数 i++ 求值后有序列点
// 保证 i 的自增副作用已经完成
// 然后才求值右操作数
// 第一次 i++ 使 i=1，结果为0（假）
// || 的右操作数求值：i++，此时i=1，表达式值为1（真），i变为2
// 结果：i = 2
```

## 五、求值顺序导致的问题

```c
// 问题1：同一表达式中修改同一变量多次
int i = 1;
int a = i++ + i++;     // 未定义行为
int b = f(i++, i++);   // 未定义行为

// 问题2：修改和读取同一变量
int i = 1;
int a = i + i++;       // 未定义行为（两个序列点之间修改并读取i）

// 问题3：数组初始化中的多次修改
int i = 0;
int arr[] = {i++, i++, i++};  // 未定义行为

// 安全写法
int i = 0;
int arr[3];
arr[0] = i++;   // 安全：每条语句是独立的完整表达式
arr[1] = i++;
arr[2] = i++;
```

## 六、关键要点

1. **优先级和结合性**决定表达式的语法分组，**不等于**求值顺序。
2. 除 `&&`、`||`、`?:`、`,` 外，双目运算符的操作数求值顺序是**未指定的**。
3. 序列点保证在此之前的所有副作用已完成。
4. **在两个序列点之间，一个对象最多只能被修改一次。**
5. 函数参数的求值顺序由编译器决定，不同编译器可能不同。
6. 编写可移植代码时，不要依赖未指定的求值顺序。
7. 将复杂表达式拆分成多条简单语句是最安全的做法。
