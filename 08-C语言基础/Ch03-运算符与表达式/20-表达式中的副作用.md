# 20 - 表达式中的副作用

## 一、什么是副作用

**副作用（Side Effect）** 是指表达式的求值过程中，除了计算结果值之外，还对程序状态产生了其他改变。

### 常见的副作用

```c
// 1. 赋值
int x = 5;    // 副作用：修改了x的值

// 2. 自增自减
i++;          // 副作用：i的值增加了1

// 3. 函数调用（可能有副作用）
printf("hello");  // 副作用：向标准输出写入了内容

// 4. volatile变量访问
volatile int reg = READ_REGISTER();  // 副作用：读取了硬件寄存器
```

### 对比：纯表达式

```c
// 没有副作用的表达式（纯表达式）
int a = 3 + 5;     // 只计算值，不改变程序状态
int b = x * y;     // 只读取x和y，不修改
int c = func();    // 如果func是纯函数，则没有副作用
```

## 二、序列点与副作用规则

### 2.1 序列点的作用

序列点（Sequence Point）保证：**在序列点之前，所有副作用都已经完成。**

```c
int i = 1;

// 函数调用前有序列点
// func(i++, i)  // 未定义行为（两个i++修改i，中间没有序列点）

// && 和 || 的左操作数后有序列点
if (i++ && i) { }  // 安全：i++ 的副作用在求值i之前完成

// 逗号运算符的左操作数后有序列点
int x = (i++, i + 1);  // 安全：i++ 在 i+1 之前完成
```

### 2.2 两个序列点之间的规则

C标准规定，在两个相邻的序列点之间：

1. 对一个对象的值**最多只能修改一次**。
2. 对一个对象值的**读取只能用于确定要存储的新值**。

```c
int i = 1;
int a = i++ + i++;    // 违反规则1：两次修改i
int b = i + i++;      // 违反规则2：读取i用于加法，但i又被修改

// 以上都是未定义行为（Undefined Behavior）
```

## 三、未定义行为示例

### 3.1 同一表达式中的多次修改

```c
int i = 1;

// ===== 未定义行为 =====
int a = i++ + i++;      // 两次修改i
int b = i++ * i++;      // 两次修改i
int c = ++i + ++i;      // 两次修改i
int d = (i++) + (++i);  // 两次修改i

// 不同编译器结果可能不同：
// GCC 可能输出一种结果
// Clang 可能输出另一种结果
// MSVC 可能又是一种结果
```

### 3.2 修改与读取冲突

```c
int i = 1;
int arr[5] = {0};

// 未定义行为
arr[i] = i++;      // 读取i作为下标，同时修改i
int x = i + i++;   // 读取i，同时修改i

// 在函数参数中
printf("%d %d\n", i, i++);  // 未定义行为
// 编译器可以先求值i，也可以先求值i++
// 两种顺序结果不同
```

### 3.3 赋值表达式中的自增

```c
int i = 0;

// ===== 未定义行为 =====
i = i++;           // 读取i，修改i（自增），修改i（赋值）
i = ++i;           // 同上
i += i++;          // 读取i两次，修改i两次

// ===== 安全写法 =====
int temp = i;
i = temp + 1;
// 或者
i++;
```

### 3.4 在数组初始化中

```c
int i = 0;
int arr[] = { i++, i++, i++ };  // 未定义行为！

// 安全写法
int arr[3];
arr[0] = i++;
arr[1] = i++;
arr[2] = i++;
```

## 四、安全的代码模式

```c
// 模式1：分开写
int i = 5;
int a = i;      // 读取i
i++;            // 修改i
int b = i;      // 再次读取

// 模式2：使用临时变量
int i = 5;
int old_i = i++;
int result = old_i + i;

// 模式3：在独立语句中使用自增
for (int i = 0; i < n; i++) { }  // i++在独立的完整表达式中，安全
```

## 五、C11/C23的改进

C11标准在附录J中列出了未定义行为的示例。C标准委员会不断澄清和完善副作用相关的规则。

```c
// C11明确的未定义行为
i = ++i + 1;      // 未定义
a[i] = i++;       // 未定义
func(i++, i);     // 未定义（如果两个参数都访问/修改i）

// C11明确的安全行为
i = i + 1;        // 安全（只有一个修改）
arr[i] = 0; i++;  // 安全（独立语句）
```

## 六、编译器警告

大多数现代编译器可以检测出潜在的副作用问题：

```bash
# GCC
gcc -Wall -Wextra -Wsequence-point test.c

# Clang
clang -Wall -Wextra test.c

# MSVC
cl /W4 test.c
```

```c
// 编译器警告示例
int i = 1;
int a = i++ + i++;  // warning: unsequenced modification and access to 'i'
```

## 七、volatile 与副作用

`volatile` 关键字告诉编译器该变量的值可能在程序之外被改变，不要优化对它的访问。

```c
volatile int *reg = (volatile int *)0x40021000;

// 每次读写都是真正的硬件访问，不会被优化掉
*reg = 1;      // 写入硬件寄存器
int val = *reg; // 读取硬件寄存器

// 多次读取 volatile 变量不能合并
int a = *reg;
int b = *reg;  // 必须重新读取，不能用a的值
```

## 八、关键要点

1. 副作用是指表达式求值过程中对程序状态的修改（赋值、自增、IO等）。
2. 序列点保证副作用在此之前已完成。
3. **两个序列点之间，同一对象最多只能修改一次。**
4. `i = i++`、`arr[i] = i++`、`func(i++, i)` 等都是**未定义行为**。
5. 将自增自减放在**独立语句**中是最安全的做法。
6. 开启编译器警告可以帮助发现副作用相关的问题。
7. `volatile` 变量的访问本身是一种副作用，编译器不会优化掉。
8. 函数调用的参数求值顺序是未指定的，不要依赖。
