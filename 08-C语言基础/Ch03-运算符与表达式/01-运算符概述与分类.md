# 01 - 运算符概述与分类

## 一、什么是运算符

运算符（Operator）是C语言中对操作数（Operand）进行特定运算的符号。运算符与操作数组合在一起就构成了**表达式（Expression）**。

```c
int a = 10 + 5;   // + 是运算符，10和5是操作数
int b = a * 2;    // * 是运算符，a和2是操作数
```

## 二、运算符分类

C语言的运算符可以按照**操作数的个数**和**功能**两种方式进行分类。

### 2.1 按操作数个数分类

| 类型 | 说明 | 示例 |
|------|------|------|
| 单目运算符（Unary） | 只需一个操作数 | `-a`、`!flag`、`++i`、`*p` |
| 双目运算符（Binary） | 需要两个操作数 | `a + b`、`x * y`、`a && b` |
| 三目运算符（Ternary） | 需要三个操作数 | `a ? b : c`（唯一一个） |

### 2.2 按功能分类

| 类别 | 运算符 | 说明 |
|------|--------|------|
| 算术运算符 | `+` `-` `*` `/` `%` | 基本数学运算 |
| 自增自减 | `++` `--` | 变量自增1或自减1 |
| 关系运算符 | `>` `<` `>=` `<=` `==` `!=` | 比较运算 |
| 逻辑运算符 | `&&` `\|\|` `!` | 逻辑与、或、非 |
| 位运算符 | `&` `\|` `^` `~` `<<` `>>` | 按位运算 |
| 赋值运算符 | `=` `+=` `-=` `*=` `/=` `%=` 等 | 赋值操作 |
| 条件运算符 | `?:` | 三目条件运算 |
| 逗号运算符 | `,` | 顺序求值 |
| 指针运算符 | `*` `&` | 解引用和取地址 |
| 成员运算符 | `.` `->` | 结构体/联合体成员访问 |
| 下标运算符 | `[]` | 数组元素访问 |
| 强制类型转换 | `(type)` | 类型转换 |
| sizeof | `sizeof` | 计算类型/变量大小 |

## 三、运算符优先级总览

C语言共有**15个优先级**，从高到低排列：

```
优先级1（最高）:  ()  []  ->  .  后缀++  后缀--
优先级2          :  前缀++  前缀--  +（正号） -（负号）  !  ~  (type)  *  &  sizeof
优先级3          :  *  /  %
优先级4          :  +  -
优先级5          :  <<  >>
优先级6          :  <  <=  >  >=
优先级7          :  ==  !=
优先级8          :  &
优先级9          :  ^
优先级10         :  |
优先级11         :  &&
优先级12         :  ||
优先级13         :  ?:
优先级14         :  =  +=  -=  *=  /=  %=  <<=  >>=  &=  ^=  |=
优先级15（最低）:  ,
```

## 四、结合性（Associativity）

当表达式中出现**相同优先级**的运算符时，需要根据结合性来决定求值顺序。

- **左结合（Left-to-right）**：从左向右依次运算。大多数双目运算符都是左结合。
- **右结合（Right-to-left）**：从右向左依次运算。赋值运算符、条件运算符、单目运算符是右结合。

```c
// 左结合示例
int result = 10 - 3 - 2;   // 等价于 (10 - 3) - 2 = 5

// 右结合示例
int a, b, c;
a = b = c = 100;   // 等价于 a = (b = (c = 100))
```

## 五、关键要点

1. C语言运算符种类丰富，掌握优先级和结合性是写出正确表达式的基础。
2. 当不确定优先级时，**使用括号明确优先级**是最佳实践。
3. 同一优先级的运算符可能具有不同的结合性，需要特别注意。
4. 逗号运算符是优先级最低的运算符。
5. 赋值运算符的优先级很低，仅高于逗号运算符。
6. `==` 的优先级低于关系运算符，这常常导致初学者犯错。
7. 理解运算符分类有助于后续深入学习表达式求值和类型转换。

## 六、常见误区

```c
// 误区：混淆赋值和相等判断
if (a = 5) { }   // 这是赋值！a被赋值为5，结果为真
if (a == 5) { }  // 这才是判断a是否等于5

// 误区：忽略优先级
int x = 3 + 4 * 5;    // 结果是23，不是35
int y = (3 + 4) * 5;  // 结果是35

// 误区：位运算与逻辑运算混淆
if (a & b) { }   // 按位与
if (a && b) { }  // 逻辑与
```
