# 18 - 常见运算符陷阱

## 一、优先级陷阱

### 1.1 `==` 与 `=` 混淆

```c
// 最经典的陷阱
if (x = 5) { }      // 赋值！x变为5，表达式为5（真）
if (x == 5) { }     // 比较x是否等于5

// 防御方法：把常量放在左边
if (5 == x) { }     // 如果误写成 5 = x，编译报错
```

### 1.2 位运算优先级低于关系运算

```c
// 错误写法
if (flags & 0x01 == 0) { }
// 实际等价于 flags & (0x01 == 0) = flags & 0 = 0
// 永远为假！

// 正确写法
if ((flags & 0x01) == 0) { }
```

### 1.3 移位与加减法

```c
int a = 1 << 2 + 3;    // 1 << 5 = 32（不是 4 + 3 = 7）
int b = (1 << 2) + 3;  // 4 + 3 = 7

int c = sizeof(int) + 2;    // sizeof(int) + 2 = 6（不是 sizeof(int + 2)）
```

### 1.4 逻辑运算优先级

```c
// && 高于 ||
int result = a && b || c && d;  // (a && b) || (c && d)

// 容易误解的写法
if (!ptr && ptr->value > 0) { }  // (!ptr) && (ptr->value > 0)
// 如果ptr为NULL，!ptr为真，但前面已短路，不会访问ptr->value
// 如果ptr非空，!ptr为假，短路不访问ptr->value
// 逻辑上不对！

if (ptr && ptr->value > 0) { }  // 正确的空指针检查
```

## 二、副作用陷阱

### 2.1 未定义行为：多次修改同一变量

```c
int i = 1;

// ===== 未定义行为 =====
int a = i++ + i++;      // 未定义
int b = i++ * i++;      // 未定义
int c = (i++) + (++i);  // 未定义
printf("%d %d\n", i, i++);  // 未定义

// ===== 安全写法 =====
int d = i;
d = d + i++;
int e = i;
i++;
e = e * i;

// 函数参数中
printf("%d %d\n", i, i++);  // 未定义行为！
// 安全：
int temp = i;
i++;
printf("%d %d\n", temp, i);
```

### 2.2 宏中的副作用

```c
#define MAX(a, b) ((a) > (b) ? (a) : (b))

int i = 5;
int m = MAX(i++, 3);
// 展开为：((i++) > (3) ? (i++) : (3))
// i++ > 3 为真，执行 i++，i变为7
// 结果是7，i变为8
// i被自增了两次！

// 解决方案：使用内联函数
static inline int max(int a, int b) {
    return a > b ? a : b;
}
int m = max(i++, 3);  // i只自增一次
```

### 2.3 函数参数求值顺序

```c
int i = 0;
func(i++, i, i++);  // 未定义行为
// 编译器可以按任意顺序求值参数
// 不同编译器结果可能不同

// 安全写法
int a = i++;
int b = i;
int c = i++;
func(a, b, c);
```

## 三、类型转换陷阱

### 3.1 有符号与无符号混合运算

```c
int a = -1;
unsigned int b = 1;

if (a < b) {
    printf("a < b\n");  // 不会执行！
} else {
    printf("a >= b\n");  // 执行这个！
}
// a被转为 unsigned int = 0xFFFFFFFF = 4294967295
// 4294967295 > 1 为真
```

### 3.2 整数截断

```c
int x = 300;
char c = x;    // 截断！300的低8位 = 44（','字符）
printf("%d\n", c);  // 44（有符号char可能显示44或-212）
```

### 3.3 浮点数精度

```c
float f = 16777216.0f;  // 2^24
float g = f + 1.0f;
printf("%f\n", g);      // 16777216.000000（1丢失了！）
// float只有24位精度，无法表示2^24 + 1

double d = 0.1 + 0.2;
printf("%.20f\n", d);   // 0.30000000000000004441
if (d == 0.3) { }       // 条件为假！
```

## 四、赋值与比较陷阱

### 4.1 链式比较

```c
int x = 5;

// C语言中不支持数学的链式写法
if (1 < x < 10) { }    // 永远为真！
// 计算过程：(1 < x) = 1，然后 1 < 10 = 1

// 正确写法
if (1 < x && x < 10) { }
```

### 4.2 赋值表达式的值

```c
// 赋值表达式返回被赋的值
if ((file = fopen("test.txt", "r")) != NULL) {
    // 使用file
}

// 容易漏掉括号
if (file = fopen("test.txt", "r")) { }  // 也可以工作但意图不清晰
```

## 五、移位陷阱

### 5.1 移位位数超出范围

```c
int x = 1;
int result = x << 32;  // 对32位int左移32位——未定义行为！

// 正确做法
unsigned int y = 1;
unsigned int result2 = y << 31;  // 安全：移位位数 < 32
```

### 5.2 负数移位

```c
int x = 1;
int shift = -1;
int result = x << shift;  // 移位量为负——未定义行为！

// 右移负数
int neg = -8;
int r = neg >> 1;  // 实现定义行为（大多数是算术右移，结果-4）
```

## 六、逗号运算符陷阱

```c
// 逗号运算符 vs 函数参数分隔符
printf("%d\n", (a, b));   // 输出b的值，(a,b)是逗号表达式
printf("%d %d\n", a, b);  // 输出a和b，逗号是参数分隔符

// 宏中的逗号
#define EXPR(a, b) a, b
printf("%d\n", EXPR(1, 2));  // 输出1，因为2是下一个参数
```

## 七、防御性编程建议

```c
// 1. 使用括号消除优先级歧义
#define BIT_SET(flags, bit) ((flags) |= (1u << (bit)))

// 2. 常量比较时把常量放左边
if (0 == ptr) { }    // 而不是 if (ptr == 0)

// 3. 避免在表达式中使用有副作用的函数/运算
int temp = i++;
process(temp, i);

// 4. 使用编译器警告
// gcc -Wall -Wextra -Werror

// 5. 有符号/无符号比较时显式转换
unsigned int u = 10;
int s = -1;
if (s < (int)u) { }  // 显式转换，避免隐式转换陷阱
```

## 八、关键要点

1. 运算符优先级是bug的常见来源，不确定时**加括号**。
2. **不要在同一表达式中多次修改同一变量**——这是未定义行为。
3. 宏中使用操作数时要注意**副作用**，考虑使用内联函数替代。
4. 有符号和无符号混合运算时，有符号数会被转为无符号数。
5. 浮点数不能直接用 `==` 比较。
6. 移位量不能为负，不能超过类型位数。
7. 启用编译器的所有警告（`-Wall -Wextra`）可以帮助发现很多陷阱。
