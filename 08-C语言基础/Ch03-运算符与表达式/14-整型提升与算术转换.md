# 14 - 整型提升与算术转换

## 一、概述

C语言在进行表达式求值时，会对操作数进行**隐式类型转换**，以确保运算在统一的类型上进行。这个过程包括两个阶段：

1. **整型提升（Integer Promotion）**
2. **普通算术转换（Usual Arithmetic Conversion）**

## 二、整型提升（Integer Promotion）

### 2.1 定义

在表达式中，**小于 `int` 的整数类型**会被自动提升为 `int` 类型（如果 `int` 能表示所有值）或 `unsigned int` 类型。

### 2.2 提升规则

| 原始类型 | 提升后 |
|----------|--------|
| `char`、`signed char` | `int` |
| `unsigned char` | `int`（如果 `int` 能表示所有值）或 `unsigned int` |
| `short`、`signed short` | `int` |
| `unsigned short` | `int`（如果 `int` 能表示所有值）或 `unsigned int` |
| 位域（Bit-field） | `int` 或 `unsigned int` |
| `_Bool` | `int` |
| `int` 及以上 | 不提升 |

### 2.3 示例

```c
#include <stdio.h>

int main(void) {
    char a = 100;
    char b = 200;

    // a 和 b 都被提升为 int 再进行运算
    int result = a + b;  // 100 + 200 = 300（不会溢出char的范围）
    printf("result = %d\n", result);  // 300

    // 注意：char是有符号时，值域是-128到127
    char c = 127;
    char d = 1;
    // c + d 中 c和d提升为int，127 + 1 = 128，不会溢出
    int sum = c + d;
    printf("sum = %d\n", sum);  // 128

    // 但赋值回char时会截断
    char e = c + d;  // 128截断为char，可能变为-128
    printf("e = %d\n", e);  // -128（有符号char溢出，实际值取决于实现）

    return 0;
}
```

### 2.4 整型提升的实际影响

```c
// 按位取反
unsigned char x = 0x0F;   // 00001111

// 不提升的话：~x = 11110000 = 240
// 实际提升后：
// 1. x 提升为 int: 0x0000000F
// 2. ~x: 0xFFFFFFF0
// 3. 如果赋值给unsigned char：截断为 0xF0 = 240
unsigned char result = ~x;
printf("~0x0F = 0x%X\n", result);  // 0xF0

// 移位运算中的整型提升
char c = 1;
int shifted = c << 30;  // c提升为int后左移30位
// 如果c不提升（char只有8位），左移30位就出问题了
```

## 三、普通算术转换（Usual Arithmetic Conversion）

### 3.1 适用范围

当双目运算符的两个操作数类型不同时，C语言会执行**普通算术转换**将它们转换为**公共类型**。

适用的运算符包括：`*` `/` `%` `+` `-` `<` `<=` `>` `>=` `==` `!=` `&` `^` `|` `?:`

### 3.2 转换规则

转换按以下顺序判断（从上到下）：

```
1. 如果任一操作数是 long double → 另一个转换为 long double
2. 否则，如果任一操作数是 double → 另一个转换为 double
3. 否则，如果任一操作数是 float → 另一个转换为 float
4. 否则（两个都是整数类型）：
   a. 先对两个操作数进行整型提升
   b. 如果类型相同，转换完成
   c. 如果类型不同：
      - 如果都是有符号或都是无符号 → 小类型转为大类型
      - 如果无符号类型 >= 有符号类型大小 → 有符号转为无符号
      - 如果有符号类型 > 无符号类型，且有符号类型能表示无符号类型所有值 → 无符号转为有符号
      - 否则 → 两个都转为有符号类型对应的无符号类型
```

### 3.3 示例

```c
#include <stdio.h>

int main(void) {
    // 示例1：int 和 double 运算
    int i = 3;
    double d = 2.5;
    double result1 = i + d;  // i被转换为double：3.0 + 2.5 = 5.5
    printf("3 + 2.5 = %f\n", result1);

    // 示例2：char 和 int 运算
    char c = 10;
    int n = 20;
    int result2 = c + n;  // c被提升为int
    printf("10 + 20 = %d\n", result2);

    // 示例3：unsigned 和 int 的陷阱
    unsigned int u = 1;
    int s = -1;
    // s 被转换为 unsigned int：4294967295
    // 结果：1 > 4294967295 为假
    if (u > s) {
        printf("u > s\n");  // 不会输出
    } else {
        printf("u <= s\n");  // 输出这个！
    }
    // 因为 -1 被转为 unsigned int 后变成很大的正数

    return 0;
}
```

### 3.4 有符号与无符号混合运算的陷阱

```c
// 经典陷阱
int a = -1;
unsigned int b = 1;

// a 被转为 unsigned int = 0xFFFFFFFF = 4294967295
printf("%u\n", a + b);  // 0（4294967295 + 1 溢出回到0）
printf("%d\n", a + b);  // 0（同一个值，只是解释不同）

// 解决方案：显式类型转换
if (a > (int)b) { }  // 将b转换为int后再比较
```

## 四、整型提升与算术转换的顺序

```c
char a = 100;
short b = 200;

// 步骤1：整型提升 —— a提升为int，b提升为int
// 步骤2：两者都是int，不需要进一步转换
int result = a + b;  // 300

char c = 100;
long long d = 200LL;

// 步骤1：整型提升 —— c提升为int
// 步骤2：普通算术转换 —— int 转为 long long
long long result2 = c + d;  // 300
```

## 五、显式类型转换（强制转换）

当隐式转换不符合预期时，使用强制类型转换显式指定：

```c
int a = 7, b = 3;

// 隐式转换：整数除法，结果为2
printf("7 / 3 = %d\n", a / b);

// 强制转换：浮点除法
printf("7.0 / 3 = %f\n", (double)a / b);  // 2.333333

// 截断浮点数
double pi = 3.14159;
int rounded = (int)pi;  // 3（截断，不四舍五入）
```

## 六、关键要点

1. **整型提升**：`char` 和 `short` 在运算前自动提升为 `int`。
2. **普通算术转换**：不同类型的操作数运算时，转换为公共类型。
3. 浮点类型和整数类型混合运算时，整数转为浮点类型。
4. **有符号和无符号混合运算**是最危险的陷阱——有符号数会被转为无符号数。
5. 比 `int` 小的类型在运算中不会溢出（因为提升为 `int`），但赋值回原类型时可能溢出。
6. 强制类型转换是显式的类型转换，建议在混合类型运算时使用以明确意图。
7. 整型提升和算术转换是**值保持**转换（值不变，类型改变），但有符号到无符号的转换可能改变值的含义。
