# 12 - 运算符结合性

## 一、什么是结合性

**结合性（Associativity）** 定义了当表达式中出现**多个相同优先级**的运算符时，求值的顺序。

C语言中存在两种结合性：
- **左结合（Left-to-right）**：从左向右依次求值
- **右结合（Right-to-left）**：从右向左依次求值

> **重要**：结合性只在相同优先级的运算符之间起作用。不同优先级的运算符总是按照优先级高低求值。

## 二、左结合的运算符

大多数**双目运算符**和**后缀运算符**是左结合的。

### 2.1 算术运算符

```c
// 减法是左结合
int a = 10 - 3 - 2;
// 等价于 (10 - 3) - 2 = 5
// 如果是右结合：10 - (3 - 2) = 9（错误）

// 除法是左结合
int b = 24 / 4 / 2;
// 等价于 (24 / 4) / 2 = 3
// 如果是右结合：24 / (4 / 2) = 12（错误）
```

### 2.2 关系和逻辑运算符

```c
// 关系运算符是左结合
int c = 5 > 3 > 1;
// 等价于 (5 > 3) > 1 = 1 > 1 = 0（不是你想的结果！）

// 逻辑运算符是左结合
int d = 1 && 1 && 0;
// 等价于 (1 && 1) && 0 = 1 && 0 = 0
```

### 2.3 后缀自增自减

```c
int arr[] = {1, 2, 3};
int *p = arr;

int x = *p++;
// 等价于 *(p++)：先取*p，然后p后移
// 不是 (*p)++ 这种歧义写法

int y = p[0]++;
// 等价于 (p[0])++：先取p[0]的值赋给y，然后p[0]自增
```

## 三、右结合的运算符

**赋值运算符**、**条件运算符**、**单目运算符**是右结合的。

### 3.1 赋值运算符

```c
// 连续赋值是右结合
int a, b, c;
a = b = c = 100;
// 等价于 a = (b = (c = 100))
// 执行顺序：c = 100 -> b = c -> a = b

// 复合赋值也是右结合
int x = 10;
x += x -= 3;
// 等价于 x += (x -= 3)
// x -= 3 得到 x = 7，然后 x += 7 得到 x = 14
// （注意：这种写法依赖求值顺序，实际中应避免）
```

### 3.2 条件运算符

```c
// 条件运算符是右结合
int a = 5, b = 10, c = 15;
int max = a > b ? a : b > c ? b : c;
// 等价于 a > b ? a : (b > c ? b : c)
// 如果a最大返回a，否则返回b和c中较大的
```

### 3.3 单目运算符

```c
int x = 5;

// 解引用和取地址是右结合
int *p = &x;
int **pp = &p;      // pp是指向指针的指针
int val = **pp;     // **pp = *(&p)解引用得到p，再解引用得到x

// sizeof 是右结合
int arr[3][4];
printf("%zu\n", sizeof arr);         // 48 = 3 * 4 * 4
printf("%zu\n", sizeof arr[0]);      // 16 = 4 * 4

// 前缀自增自减是右结合（虽然实际意义不大）
int i = 5;
int j = -++i;   // 等价于 -(++i)：i先加1变为6，然后取负得到-6
```

## 四、结合性与优先级的关系

```c
// 示例：* 和 / 优先级相同，左结合
int a = 12 / 3 * 2;
// 等价于 (12 / 3) * 2 = 8（不是 12 / (3 * 2) = 2）

// 示例：= 和 += 优先级相同，右结合
int x = 10;
x = x += 5;
// 等价于 x = (x += 5)：x先加5变为15，然后赋给x（结果还是15）

// 示例：*（解引用）和 ++（前缀）优先级相同，右结合
int arr[] = {10, 20, 30};
int *p = arr;
int val = *++p;   // 等价于 *(++p)：p先移动到arr[1]，再取值
printf("%d\n", val);  // 20
```

## 五、实际编程建议

### 5.1 用括号消除歧义

```c
// 不清晰
int result = a / b * c;

// 清晰
int result = (a / b) * c;   // 先除后乘
// 或
int result = a / (b * c);   // 先乘后除
```

### 5.2 避免依赖复杂结合性

```c
// 不推荐：依赖结合性和求值顺序
a = b++ + ++c * d--;

// 推荐：拆分成简单语句
int temp1 = b;
b++;
int temp2 = c + 1;
c = temp2;
int temp3 = d;
d--;
a = temp1 + temp2 * temp3;
```

## 六、关键要点

1. 结合性只对**相同优先级**的运算符起作用。
2. 双目运算符绝大多数是**左结合**。
3. 赋值、条件运算符、单目运算符是**右结合**。
4. `++` 和 `--` 前缀是右结合（与单目运算符同级），后缀也是右结合（优先级最高）。
5. 复杂表达式应使用**括号**明确意图，不要依赖记忆优先级和结合性。
6. 避免在同一表达式中混合使用多种运算符——拆分成多条语句更安全。
