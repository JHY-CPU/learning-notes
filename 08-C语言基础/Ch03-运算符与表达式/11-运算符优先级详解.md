# 11 - 运算符优先级详解

## 一、完整优先级表

下表列出了C语言所有运算符的优先级（从高到低，共15级）和结合性：

| 优先级 | 运算符 | 名称/描述 | 结合性 |
|--------|--------|-----------|--------|
| 1 | `()` `[]` `->` `.` | 括号、下标、指针成员、成员 | 左→右 |
| 1 | `++` `--`（后缀） | 后缀自增、自减 | 左→右 |
| 2 | `++` `--`（前缀） | 前缀自增、自减 | 右←左 |
| 2 | `+` `-`（单目） | 正号、负号 | 右←左 |
| 2 | `!` `~` | 逻辑非、按位取反 | 右←左 |
| 2 | `(type)` | 强制类型转换 | 右←左 |
| 2 | `*` | 解引用 | 右←左 |
| 2 | `&` | 取地址 | 右←左 |
| 2 | `sizeof` | 大小运算符 | 右←左 |
| 3 | `*` `/` `%` | 乘法、除法、取余 | 左→右 |
| 4 | `+` `-`（双目） | 加法、减法 | 左→右 |
| 5 | `<<` `>>` | 左移、右移 | 左→右 |
| 6 | `<` `<=` `>` `>=` | 关系运算符 | 左→右 |
| 7 | `==` `!=` | 等于、不等于 | 左→右 |
| 8 | `&` | 按位与 | 左→右 |
| 9 | `^` | 按位异或 | 左→右 |
| 10 | `\|` | 按位或 | 左→右 |
| 11 | `&&` | 逻辑与 | 左→右 |
| 12 | `\|\|` | 逻辑或 | 左→右 |
| 13 | `?:` | 条件运算符 | 右←左 |
| 14 | `=` `+=` `-=` `*=` `/=` `%=` `<<=` `>>=` `&=` `^=` `\|=` | 赋值运算符 | 右←左 |
| 15 | `,` | 逗号运算符 | 左→右 |

## 二、常见混淆优先级

### 2.1 `*` `/` `%` vs `+` `-`

```c
int result = 2 + 3 * 4;    // 14，不是20。* 高于 +
int result = 2 * 3 + 4;    // 10
```

### 2.2 移位 vs 关系 vs 相等

```c
int a = 1 << 2 + 3;   // 1 << (2 + 3) = 32，不是 (1 << 2) + 3 = 7
int b = 5 > 3 == 1;   // (5 > 3) == 1 = 1
int c = 5 & 3 == 3;   // 5 & (3 == 3) = 5 & 1 = 1，不是 (5 & 3) == 3
```

### 2.3 `&` `|` `^` vs `&&` `||` vs 关系运算符

```c
// 位运算符优先级低于关系运算符！
if (flags & 0x01 == 0) { }   // flags & (0x01 == 0) = flags & 0 —— 错误！
if ((flags & 0x01) == 0) { } // 正确写法

// 逻辑运算符优先级低于关系运算符
if (a > b && c > d) { }  // 等价于 (a > b) && (c > d)，正确
```

### 2.4 `==` vs `=`

```c
if (a = 5) { }    // 赋值！a被赋值为5，表达式值为5（真）
if (a == 5) { }   // 比较a是否等于5

// 防御技巧：将常量写在左边
if (5 == a) { }   // 如果误写成 5 = a，编译器会报错
```

### 2.5 自增自减的优先级

```c
int arr[] = {1, 2, 3, 4, 5};
int *p = arr;

int a = *p++;    // *(p++)：先取*p=1，然后p后移
int b = ++*p;    // ++(*p)：先取*p=2，加1后写回，b=3
int c = (*p)++;  // 先取*p=3，然后*p自增，c=3
int d = *++p;    // *(++p)：p先移动，再取值，d=4
```

## 三、经典陷阱示例

```c
// 陷阱1：赋值在条件中
while (ch = getchar() != EOF) { }
// 等价于：ch = (getchar() != EOF)
// ch 始终为0或1，不是字符值！
// 正确写法：while ((ch = getchar()) != EOF) { }

// 陷阱2：位运算优先级
int flags = 0x0A;
if (flags & 0x01 == 0) { }     // flags & 0 = 0，永远为假！
if ((flags & 0x01) == 0) { }   // 正确

// 陷阱3：移位优先级
int x = 1 << 1 + 1;    // 1 << 2 = 4，不是 (1 << 1) + 1 = 3
int y = (1 << 1) + 1;  // 3

// 陷阱4：sizeof的优先级
int arr[10];
int n = sizeof arr + 2;   // sizeof(arr) + 2 = 40 + 2 = 42，不是 sizeof(arr + 2)
```

## 四、括号使用建议

### 4.1 什么时候加括号

**建议在以下情况加括号以消除歧义：**

```c
// 1. 混合使用位运算和关系运算时
if ((flags & MASK) == 0) { }     // 推荐
if (flags & MASK == 0) { }       // 不推荐（有歧义）

// 2. 混合使用赋值和其他运算时
while ((ch = getchar()) != EOF) { }  // 推荐
while (ch = getchar() != EOF) { }    // 不推荐

// 3. 条件运算符嵌套时
int x = (a > b) ? (c > d ? c : d) : (e > f ? e : f);  // 推荐

// 4. 函数式宏中
#define SQUARE(x) ((x) * (x))   // 推荐
#define SQUARE(x) (x * x)       // 不推荐：SQUARE(a + b) 展开为 a + b * a + b
```

### 4.2 括号使用原则

- 不确定优先级时，**加括号**。
- 混合不同类别的运算符时，**加括号**。
- 写宏定义时，**每个参数和整个表达式都加括号**。
- 括号不影响运行效率，编译器会优化掉。

## 五、关键要点

1. C语言有15个优先级，记住常用的即可。
2. **单目 > 双目 > 三目 > 赋值 > 逗号** 是基本规律。
3. 在同一优先级中，注意区分单目和双目（如 `*` 既可解引用也可乘法）。
4. 位运算符优先级低于关系运算符，这是最容易踩的坑。
5. `==` 和 `=` 的优先级不同，容易混淆。
6. 括号不会降低效率，不确定时就加括号。
7. 建议养成习惯：**位运算和关系运算混合时，必须加括号**。
