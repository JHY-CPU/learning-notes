# 15 - 短路求值详解

## 一、概述

**短路求值（Short-circuit Evaluation）** 是指逻辑运算符 `&&` 和 `||` 在求值时，如果根据左操作数的值已经能确定最终结果，则**跳过（不执行）右操作数**的求值。

这是C语言标准明确规定的特性，不是编译器的优化行为。

## 二、短路求值的规则

### 2.1 `&&`（逻辑与）的短路规则

```
表达式1 && 表达式2
```

- 先求值 `表达式1`。
- 如果 `表达式1` 为**假（0）**，整个表达式结果为假，**`表达式2` 不会被求值**。
- 如果 `表达式1` 为**真（非0）**，继续求值 `表达式2`，并以其结果作为最终结果。

```c
#include <stdio.h>

int main(void) {
    int i = 0;

    // 左边为假，右边的 i++ 不执行
    if (0 && i++) {
        // 不进入
    }
    printf("i = %d\n", i);  // 0（i++未执行）

    int j = 1;
    // 左边为真，右边的 j++ 执行
    if (1 && j++) {
        // 进入
    }
    printf("j = %d\n", j);  // 2（j++执行了）

    return 0;
}
```

### 2.2 `||`（逻辑或）的短路规则

```
表达式1 || 表达式2
```

- 先求值 `表达式1`。
- 如果 `表达式1` 为**真（非0）**，整个表达式结果为真，**`表达式2` 不会被求值**。
- 如果 `表达式1` 为**假（0）**，继续求值 `表达式2`，并以其结果作为最终结果。

```c
int i = 1;

// 左边为真，右边的 i++ 不执行
if (1 || i++) {
    // 进入
}
printf("i = %d\n", i);  // 1（i++未执行）

int j = 0;
// 左边为假，右边的 j++ 执行
if (0 || j++) {
    // 进入
}
printf("j = %d\n", j);  // 1（j++执行了）
```

## 三、利用短路求值做安全检查

这是短路求值最重要的实际应用。

### 3.1 空指针检查

```c
// 安全：先检查指针是否为NULL，再访问成员
if (ptr != NULL && ptr->value > 0) {
    printf("value = %d\n", ptr->value);
}

// 如果ptr为NULL，ptr->value不会被访问，避免段错误
```

### 3.2 数组边界检查

```c
// 安全：先检查索引是否越界，再访问数组元素
if (index >= 0 && index < size && array[index] == target) {
    printf("找到了\n");
}

// 如果index越界，array[index]不会被访问
```

### 3.3 除零检查

```c
// 安全：先检查分母是否为零，再执行除法
if (denominator != 0 && numerator / denominator > threshold) {
    printf("比值大于阈值\n");
}

// 如果denominator为0，除法不会执行
```

### 3.4 字符串安全检查

```c
#include <string.h>

// 安全：先检查字符串指针，再检查长度
if (str != NULL && strlen(str) > 0) {
    printf("字符串非空: %s\n", str);
}

// 安全：先检查指针，再访问内容
if (str != NULL && str[0] == 'A') {
    printf("以A开头\n");
}
```

## 四、短路求值的序列点

`&&` 和 `||` 的左操作数和右操作数之间存在**序列点**，保证：

1. 左操作数的所有副作用已完成。
2. 右操作数（如果需要求值）在左操作数完全求值后才开始。

```c
int i = 0;

// 序列点保证 i++ 的副作用在右操作数求值前完成
if (i++ == 0 && i == 1) {
    printf("i = %d\n", i);  // i = 1
}
// 左操作数 i++ 求值：值为0（假），i变为1
// 序列点：i的自增副作用完成
// 右操作数 i == 1：i的值是1，表达式为真
// 结果：整个条件为真

// 对比：没有短路的情况
int j = 0;
if (j == 0 && j++ > 0) {
    // 左操作数为真，需要求值右操作数
    // j++ 的值是0，> 0 为假
    // j 变为1
}
printf("j = %d\n", j);  // j = 1
```

## 五、链式短路求值

多个 `&&` 或 `||` 连接时，每个连接处都有序列点。

```c
// 链式 &&
if (a && b && c && d) {
    // 从左到右依次求值，遇到假立即停止
    // 如果a为假，b、c、d都不会被求值
}

// 链式 ||
if (a || b || c || d) {
    // 从左到右依次求值，遇到真立即停止
    // 如果a为真，b、c、d都不会被求值
}

// 混合链式
if (a && b || c && d) {
    // 等价于 (a && b) || (c && d)
    // 先求值 (a && b)，如果为真，整个为真，(c && d)不求值
    // 如果 (a && b) 为假，再求值 (c && d)
}
```

## 六、短路求值在宏和函数中的应用

```c
// 安全的除法宏
#define SAFE_DIV(a, b) ((b) != 0 ? (a) / (b) : 0)

// 利用短路的替代方案
int safe_divide(int a, int b, int *result) {
    if (b != 0) {
        *result = a / b;
        return 1;  // 成功
    }
    return 0;  // 失败
}

// 短路用于条件赋值
int value = (ptr != NULL) ? ptr->x : default_value;
```

## 七、注意事项

```c
// 注意1：短路求值不适用于位运算符 & 和 |
// if (ptr & ptr->value) { }  —— 如果ptr为NULL会崩溃！

// 注意2：不要在右操作数中放置必须执行的副作用
int count = 0;
if (condition && count++) { }
// 如果condition为假，count不会自增！可能不符合预期

// 注意3：短路求值可能隐藏错误
if (ptr == NULL || ptr->x > 0) { }
// 虽然安全，但可能掩盖ptr为NULL的bug
// 调试时可能需要去掉短路来发现错误
```

## 八、关键要点

1. `&&` 在左操作数为假时短路，`||` 在左操作数为真时短路。
2. 短路求值是C语言标准规定的，不是编译器优化。
3. 短路求值可以用于**安全检查**（空指针、数组越界、除零）。
4. `&&` 和 `||` 的操作数之间存在**序列点**。
5. 位运算符 `&` 和 `|` **没有**短路特性，使用时要小心。
6. 不要依赖短路求值来执行副作用操作——这样代码意图不清晰。
7. 链式 `&&` 和 `||` 每个连接处都有序列点。
