# 常见内存错误总结

## 一、错误分类总览

```
C语言内存错误七大类:

  1. 内存泄漏         - 分配后未释放
  2. 悬空指针         - 释放后继续使用
  3. 野指针           - 未初始化就使用
  4. 缓冲区溢出       - 越界读写
  5. Double Free      - 重复释放
  6. 使用未初始化内存  - 读取垃圾值
  7. 类型混淆         - 错误的类型转换
```

## 二、各类错误详解与解决

### 2.1 内存泄漏 (Memory Leak)

```c
// 错误
void leak_example(void) {
    char *buf = malloc(1024);
    // 忘记 free(buf)
}

// 正确
void no_leak_example(void) {
    char *buf = malloc(1024);
    if (!buf) return;
    // 使用 buf...
    free(buf);
    buf = NULL;
}
```

### 2.2 悬空指针 (Dangling Pointer)

```c
// 错误
void dangling_example(void) {
    int *p = malloc(sizeof(int));
    *p = 42;
    free(p);
    printf("%d\n", *p);  // use-after-free
}

// 正确
void no_dangling_example(void) {
    int *p = malloc(sizeof(int));
    *p = 42;
    printf("%d\n", *p);
    free(p);
    p = NULL;
}
```

### 2.3 野指针 (Wild Pointer)

```c
// 错误
void wild_example(void) {
    int *p;        // 未初始化
    *p = 42;       // 写入随机地址
}

// 正确
void no_wild_example(void) {
    int *p = NULL;  // 初始化
    p = malloc(sizeof(int));
    if (p) *p = 42;
    free(p);
}
```

### 2.4 缓冲区溢出 (Buffer Overflow)

```c
#include <string.h>

// 错误
void overflow_example(void) {
    char buf[10];
    strcpy(buf, "Hello, World!");  // 13字节写入10字节缓冲区
}

// 正确
void no_overflow_example(void) {
    char buf[10];
    strncpy(buf, "Hello, World!", sizeof(buf) - 1);
    buf[sizeof(buf) - 1] = '\0';
}
```

### 2.5 Double Free

```c
// 错误
void double_free_example(void) {
    int *p = malloc(sizeof(int));
    free(p);
    free(p);  // 重复释放
}

// 正确
void no_double_free_example(void) {
    int *p = malloc(sizeof(int));
    free(p);
    p = NULL;   // 置空后再free安全
    free(p);    // free(NULL) 安全
}
```

### 2.6 使用未初始化内存

```c
// 错误
void uninitialized_example(void) {
    int arr[100];
    int sum = 0;
    for (int i = 0; i < 100; i++) {
        sum += arr[i];  // 垃圾值
    }
}

// 正确
void initialized_example(void) {
    int arr[100] = {0};  // 全部初始化为0
    int sum = 0;
    for (int i = 0; i < 100; i++) {
        sum += arr[i];
    }
}
```

### 2.7 类型混淆

```c
// 错误
void type_confusion(void) {
    void *p = malloc(sizeof(double));
    int *ip = (int *)p;     // 类型不匹配
    *ip = 42;               // 只写入4字节，double是8字节
    free(p);
}

// 正确
void no_type_confusion(void) {
    double *dp = malloc(sizeof(*dp));  // 类型匹配
    *dp = 42.0;
    free(dp);
}
```

## 三、错误检测方法速查

| 错误类型 | Valgrind | ASan | 编译警告 |
|---------|----------|------|---------|
| 内存泄漏 | --leak-check | -fsanitize=leak | - |
| 悬空指针 | 检测 | -fsanitize=address | - |
| 野指针 | 部分检测 | 部分检测 | -Wuninitialized |
| 缓冲区溢出 | 检测 | -fsanitize=address | -Wstringop-overflow |
| Double Free | 检测 | -fsanitize=address | - |
| 未初始化 | --track-origins | -fsanitize=memory | -Wuninitialized |
| 类型混淆 | 部分 | -fsanitize=undefined | -Wcast-align |

## 四、快速修复检查清单

```
内存错误快速修复:

□ 检查所有malloc/calloc/realloc返回值
□ 每个malloc/free/calloc/realloc配对
□ free后立即置NULL
□ 所有指针声明时初始化为NULL
□ 使用strncpy/snprintf替代strcpy/sprintf
□ 数组访问前检查索引范围
□ 使用sizeof(*ptr)模式避免类型错误
□ 结构体分配后初始化所有成员
□ 函数所有退出路径都有释放代码
□ 使用goto清理模式统一释放
□ 编译时加 -fsanitize=address
□ 定期用Valgrind全面检查
```

## 五、关键要点

> **内存错误总结**
> 1. 七类常见错误：泄漏、悬空、野指针、溢出、double free、未初始化、类型混淆
> 2. 预防比修复更重要：良好的编程习惯是第一道防线
> 3. 工具是第二道防线：ASan适合开发期，Valgrind适合全面检查
> 4. 代码审查是第三道防线：内存管理是审查重点
> 5. 安全的默认模式：SAFE_FREE、goto清理、封装安全接口
