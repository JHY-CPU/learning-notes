# 内存布局详解

## 一、程序内存布局总览

当一个C程序被加载到内存中运行时，操作系统会为其分配一个虚拟地址空间。这个空间被划分为多个区域，每个区域有特定的用途和特性。

```
高地址 0xFFFFFFFF (32位)
┌──────────────────────────────┐
│   内核空间 (Kernel Space)     │  用户程序不可访问
├──────────────────────────────┤
│   命令行参数 & 环境变量        │  argc, argv, envp
├──────────────────────────────┤
│   栈 (Stack)                 │  局部变量、函数调用
│   ↓ 向下增长                  │
├──────────────────────────────┤
│   内存映射段 (Memory Map)     │  mmap分配的区域
│   ↑ 向上增长                  │
├──────────────────────────────┤
│   堆 (Heap)                  │  malloc/free管理
│   ↑ 向上增长                  │
├──────────────────────────────┤
│   BSS段                      │  未初始化全局变量(清零)
├──────────────────────────────┤
│   数据段 (Data)               │  已初始化全局变量
├──────────────────────────────┤
│   代码段 (Text/RODATA)        │  程序代码、只读数据
└──────────────────────────────┘
低地址 0x00000000
```

## 二、代码段 (Text Segment)

### 2.1 特征

- 存放程序的**机器指令**（编译后的二进制代码）
- 通常是**只读**的，防止程序意外修改自身代码
- 多个运行同一程序的进程可以**共享**代码段

```c
#include <stdio.h>

// 代码段中的内容示例
void my_function(void) {
    // 这段函数的机器码存储在代码段
    printf("Hello, World!\n");
}

// 字符串字面量通常存储在只读数据段(RODATA)
const char *greeting = "Hello";  // "Hello" 在只读数据段

int main(void) {
    my_function();

    // 尝试修改字符串字面量是未定义行为
    // char *str = "test";
    // str[0] = 'T';  // 运行时会崩溃！段错误

    return 0;
}
```

### 2.2 只读数据段 (RODATA)

编译器通常将字符串常量和`const`修饰的全局变量放入只读数据段：

```c
// 这些数据通常在RODATA段
const int MAX_SIZE = 1024;         // const全局变量
const char *error_msg = "Error";   // 字符串常量

void demo(void) {
    // 局部const变量不一定在RODATA，可能在栈上
    const int local_const = 42;

    // 字符串字面量在RODATA
    char *msg = "Read Only Data";
    printf("%s\n", msg);
}
```

## 三、数据段 (Data Segment)

### 3.1 已初始化数据区

存放已初始化的全局变量和静态变量：

```c
#include <stdio.h>

// ===== 数据段(Data Segment)中的变量 =====

// 已初始化的全局变量
int global_var = 100;
double global_double = 3.14;

// 已初始化的静态全局变量
static int static_global = 200;

// 已初始化的静态局部变量
void counter(void) {
    static int count = 0;  // 数据段（只初始化一次）
    count++;
    printf("调用次数: %d\n", count);
}

int main(void) {
    printf("global_var = %d\n", global_var);
    printf("static_global = %d\n", static_global);

    counter();  // 调用次数: 1
    counter();  // 调用次数: 2
    counter();  // 调用次数: 3

    return 0;
}
```

## 四、BSS段 (Block Started by Symbol)

### 4.1 特征

- 存放**未初始化**（或初始化为0）的全局变量和静态变量
- 程序加载时由系统**自动清零**
- 不占用可执行文件的空间（只记录大小信息）

```c
#include <stdio.h>

// ===== BSS段中的变量 =====

// 未初始化的全局变量 → BSS段（自动清零）
int bss_global;
int bss_array[1000];

// 未初始化的静态全局变量 → BSS段
static int bss_static_global;

// 显式初始化为0也算BSS
int zero_init = 0;
static int static_zero = 0;

void show_bss(void) {
    // 未初始化的静态局部变量 → BSS段
    static int bss_local_static;

    printf("bss_global = %d (默认0)\n", bss_global);
    printf("bss_static_global = %d (默认0)\n", bss_static_global);
    printf("bss_local_static = %d (默认0)\n", bss_local_static);
    printf("zero_init = %d\n", zero_init);
}

int main(void) {
    show_bss();
    // BSS段的变量总是被初始化为0
    return 0;
}
```

### 4.2 Data段 vs BSS段

| 特性 | Data段 | BSS段 |
|------|--------|-------|
| 初始化 | 已初始化(非零值) | 未初始化或初始化为0 |
| 可执行文件大小 | 占用文件空间 | 不占用(只记录大小) |
| 运行时行为 | 直接从文件加载 | 系统自动清零 |
| 示例 | `int x = 5;` | `int y;` |

## 五、堆 (Heap)

### 5.1 特征

- 用于**动态内存分配**
- 大小不固定，由程序运行时决定
- 向**高地址**方向增长
- 由程序员手动管理（malloc/free）

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

void heap_demo(void) {
    // 在堆上分配内存
    int *arr = (int *)malloc(10 * sizeof(int));
    if (arr == NULL) {
        fprintf(stderr, "堆分配失败\n");
        return;
    }

    printf("堆分配地址: %p\n", (void *)arr);

    // 使用堆内存
    for (int i = 0; i < 10; i++) {
        arr[i] = i * i;
    }

    // 扩展堆内存
    int *new_arr = (int *)realloc(arr, 20 * sizeof(int));
    if (new_arr != NULL) {
        arr = new_arr;
        printf("堆扩展地址: %p\n", (void *)arr);
    }

    // 释放堆内存
    free(arr);
    arr = NULL;
}

int main(void) {
    heap_demo();
    return 0;
}
```

### 5.2 堆的生长方向

```c
#include <stdio.h>
#include <stdlib.h>

void heap_growth_demo(void) {
    int *ptr1 = malloc(sizeof(int));
    int *ptr2 = malloc(sizeof(int));
    int *ptr3 = malloc(sizeof(int));

    printf("第一次分配: %p\n", (void *)ptr1);
    printf("第二次分配: %p\n", (void *)ptr2);
    printf("第三次分配: %p\n", (void *)ptr3);

    // 堆整体向高地址增长，但连续malloc不一定地址严格递增
    // （取决于allocator的内存管理策略、对齐、碎片等）
    printf("本次分配地址递增: %s\n",
           (ptr2 > ptr1 && ptr3 > ptr2) ? "是" : "否");

    free(ptr1);
    free(ptr2);
    free(ptr3);
}

int main(void) {
    heap_growth_demo();
    return 0;
}
```

> **注：** `malloc`不保证连续分配的地址严格递增。虽然大多数allocator在无碎片时会按地址递增顺序分配，但地址对齐、内部元数据、arena策略等都可能导致乱序。堆"向高地址增长"指的是`brk`/`sbrk`扩展的方向，不是每次`malloc`返回地址的单调性。

## 六、栈 (Stack)

### 6.1 特征

- 存储**局部变量、函数参数、返回地址**
- 由编译器**自动管理**（不需要手动分配/释放）
- 向**低地址**方向增长
- 大小有限（通常1-8MB）

```c
#include <stdio.h>

void stack_demo(int param) {        // param在栈上
    int local_var = 42;             // 局部变量在栈上
    char buffer[256];               // 局部数组在栈上
    static int static_var = 100;    // static_var在数据段(不在栈上)

    printf("参数地址:  %p\n", (void *)&param);
    printf("局部变量:  %p\n", (void *)&local_var);
    printf("局部数组:  %p\n", (void *)buffer);
    printf("静态变量:  %p\n", (void *)&static_var);
}

int main(void) {
    int main_local = 10;
    printf("main局部变量: %p\n", (void *)&main_local);

    stack_demo(5);
    // 函数返回后，栈帧自动回收

    return 0;
}
```

### 6.2 栈帧结构

```
高地址
┌─────────────────────────┐
│   调用者的栈帧            │
├─────────────────────────┤
│   返回地址               │  ← 调用call时自动压入
├─────────────────────────┤
│   旧的帧指针(EBP)        │  ← 保存调用者的基址指针
├─────────────────────────┤
│   函数参数(部分)          │  ← 超过寄存器数量的参数
├─────────────────────────┤
│   局部变量               │  ← int, char, 数组等
├─────────────────────────┤
│   临时数据               │  ← 表达式计算中间结果
├─────────────────────────┤  ← ESP(栈顶指针)
│   (栈顶)                 │
└─────────────────────────┘
低地址
```

## 七、完整内存布局验证程序

```c
#include <stdio.h>
#include <stdlib.h>

// 全局已初始化变量 (Data段)
int g_initialized = 42;

// 全局未初始化变量 (BSS段)
int g_uninitialized;

// 静态已初始化变量 (Data段)
static int s_initialized = 100;

// 静态未初始化变量 (BSS段)
static int s_uninitialized;

void print_memory_layout(void) {
    // 局部变量 (栈)
    int stack_var = 10;

    // 动态分配 (堆)
    int *heap_var = (int *)malloc(sizeof(int));

    // 静态局部变量 (Data段)
    static int local_static_init = 50;

    // 静态局部未初始化变量 (BSS段)
    static int local_static_uninit;

    // 字符串常量 (只读数据段)
    const char *str_const = "Read Only";

    printf("===== 内存布局 (低地址→高地址) =====\n\n");

    printf("代码段/RODATA:\n");
    printf("  函数地址:     %p\n", (void *)print_memory_layout);
    printf("  字符串常量:   %p  \"%s\"\n", (void *)str_const, str_const);

    printf("\n数据段 (已初始化):\n");
    printf("  全局已初始化: %p  value=%d\n", (void *)&g_initialized, g_initialized);
    printf("  静态已初始化: %p  value=%d\n", (void *)&s_initialized, s_initialized);
    printf("  局部静态初始化: %p  value=%d\n", (void *)&local_static_init, local_static_init);

    printf("\nBSS段 (未初始化/清零):\n");
    printf("  全局未初始化: %p  value=%d\n", (void *)&g_uninitialized, g_uninitialized);
    printf("  静态未初始化: %p  value=%d\n", (void *)&s_uninitialized, s_uninitialized);
    printf("  局部静态未初始化: %p  value=%d\n", (void *)&local_static_uninit, local_static_uninit);

    printf("\n堆 (Heap):\n");
    printf("  堆变量地址:   %p  value=%d\n", (void *)heap_var, *heap_var);

    printf("\n栈 (Stack):\n");
    printf("  局部变量:     %p  value=%d\n", (void *)&stack_var, stack_var);

    printf("\n===== 地址区间分析 =====\n");
    printf("代码段 < BSS段 < 数据段 < 堆 < 栈\n");
    printf("%p < %p < %p < %p < %p\n",
           (void *)print_memory_layout,
           (void *)&g_uninitialized,
           (void *)&g_initialized,
           (void *)heap_var,
           (void *)&stack_var);

    free(heap_var);
}

int main(void) {
    print_memory_layout();
    return 0;
}
```

## 八、各内存区域对比总结

| 区域 | 存储内容 | 生命周期 | 管理方式 | 增长方向 | 大小 |
|------|---------|---------|---------|---------|------|
| 代码段 | 程序指令 | 进程生命周期 | 系统 | 固定 | 由程序决定 |
| RODATA | 只读数据 | 进程生命周期 | 系统 | 固定 | 由程序决定 |
| 数据段 | 已初始化全局变量 | 进程生命周期 | 系统 | 固定 | 由程序决定 |
| BSS段 | 未初始化全局变量 | 进程生命周期 | 系统 | 固定 | 由程序决定 |
| 堆 | 动态分配的内存 | 程序员控制 | 手动(malloc/free) | 高地址 | 受系统限制 |
| 栈 | 局部变量、参数 | 函数作用域 | 自动 | 低地址 | 通常1-8MB |

## 九、关键要点

> **重要概念**
> 1. 虚拟地址空间使每个进程都有独立的内存视图
> 2. 代码段和数据段的大小在编译时确定
> 3. 堆和栈的大小在运行时动态变化
> 4. 栈向低地址增长，堆向高地址增长
> 5. BSS段不占用可执行文件空间，由系统自动清零
> 6. 字符串字面量存储在只读区域，修改是未定义行为
