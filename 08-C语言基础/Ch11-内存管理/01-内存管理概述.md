# 内存管理概述

## 一、什么是内存管理

内存管理是指程序在运行过程中对计算机内存资源进行分配、使用和释放的过程。在C语言中，内存管理是**程序员的直接责任**，语言本身不提供自动垃圾回收机制。

```
┌──────────────────────────────────────────┐
│            C语言内存管理特点              │
├──────────────────────────────────────────┤
│  ✓ 手动分配内存 (malloc/calloc/realloc)  │
│  ✓ 手动释放内存 (free)                   │
│  ✓ 程序员完全控制内存生命周期             │
│  ✓ 无垃圾回收机制(GC)                    │
│  ✓ 高效但容易出错                        │
└──────────────────────────────────────────┘
```

## 二、为什么内存管理如此重要

### 2.1 C语言的特殊性

与Java、Python等高级语言不同，C语言将内存管理的权力和责任完全交给了程序员：

```c
// Java 中：自动垃圾回收
// String s = new String("Hello");  // 自动管理

// C 语言中：手动管理
char *s = (char *)malloc(6);    // 手动分配
strcpy(s, "Hello");             // 手动赋值
free(s);                        // 手动释放（忘记就泄漏！）
s = NULL;                       // 防止悬空指针
```

### 2.2 内存管理不当的后果

| 问题 | 描述 | 后果 |
|------|------|------|
| 内存泄漏 | 分配后未释放 | 程序内存持续增长，最终崩溃 |
| 悬空指针 | 释放后继续使用 | 未定义行为，数据损坏 |
| 野指针 | 未初始化就使用 | 随机崩溃，安全漏洞 |
| 缓冲区溢出 | 访问超出分配范围 | 栈溢出攻击，程序崩溃 |
| Double Free | 重复释放同一内存 | 堆结构损坏，程序崩溃 |

## 三、C语言内存区域划分

一个C程序运行时，其内存空间被划分为以下几个主要区域：

```
高地址
┌─────────────────────┐
│    命令行参数和      │
│    环境变量          │
├─────────────────────┤
│        栈(Stack)     │  ← 局部变量、函数参数、返回地址
│    ↓ 向下增长        │
├─────────────────────┤
│                     │
│    (未使用空间)      │
│                     │
├─────────────────────┤
│    ↑ 向上增长        │
│        堆(Heap)      │  ← malloc/calloc/realloc 分配
├─────────────────────┤
│   BSS段             │  ← 未初始化的全局/静态变量
├─────────────────────┤
│   数据段(Data)       │  ← 已初始化的全局/静态变量
├─────────────────────┤
│   代码段(Text)       │  ← 程序执行代码(只读)
└─────────────────────┘
低地址
```

### 3.1 各区域详解

```c
#include <stdio.h>
#include <stdlib.h>

// 全局已初始化变量 → 数据段(Data Segment)
int global_init = 42;

// 全局未初始化变量 → BSS段
int global_uninit;

// 静态已初始化变量 → 数据段
static int static_init = 100;

// 静态未初始化变量 → BSS段
static int static_uninit;

// 字符串常量 → 代码段(Text/RODATA)
const char *message = "Hello, Memory!";

void example_function() {
    // 局部变量 → 栈(Stack)
    int local_var = 10;
    char local_array[64];

    // 动态分配 → 堆(Heap)
    int *heap_var = (int *)malloc(sizeof(int) * 100);

    printf("栈变量地址:  %p\n", (void *)&local_var);
    printf("堆变量地址:  %p\n", (void *)heap_var);
    printf("全局变量地址: %p\n", (void *)&global_init);
    printf("BSS变量地址: %p\n", (void *)&global_uninit);
    printf("常量地址:    %p\n", (void *)message);

    free(heap_var);
}

int main() {
    example_function();
    return 0;
}
```

典型输出（地址从低到高）：
```
常量地址:    0x4007a4     ← 代码段(低地址)
BSS变量地址: 0x60104c     ← BSS段
全局变量地址: 0x601040     ← 数据段
堆变量地址:  0x1a23010    ← 堆
栈变量地址:  0x7ffd5c...  ← 栈(高地址)
```

## 四、手动内存管理的核心操作

### 4.1 分配与释放的基本模式

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

// 标准的内存管理模式
int memory_management_pattern(void) {
    // 1. 分配内存
    char *buffer = (char *)malloc(256);
    if (buffer == NULL) {
        fprintf(stderr, "内存分配失败\n");
        return -1;
    }

    // 2. 使用内存
    strcpy(buffer, "使用动态内存存储数据");
    printf("内容: %s\n", buffer);

    // 3. 释放内存
    free(buffer);

    // 4. 避免悬空指针
    buffer = NULL;

    return 0;
}
```

### 4.2 内存分配函数族

C标准库提供了以下核心内存管理函数：

```c
#include <stdlib.h>

void *malloc(size_t size);         // 分配指定字节数（不初始化）
void *calloc(size_t n, size_t sz); // 分配n个sz字节的元素（初始化为0）
void *realloc(void *ptr, size_t sz); // 调整已分配内存的大小
void  free(void *ptr);             // 释放内存
```

| 函数 | 功能 | 是否初始化 | 常用场景 |
|------|------|-----------|---------|
| `malloc` | 分配指定大小的内存块 | 否 | 通用内存分配 |
| `calloc` | 分配并清零数组内存 | 是（全零） | 数组分配 |
| `realloc` | 调整已分配内存大小 | 新增部分不确定 | 动态扩容 |
| `free` | 释放已分配的内存 | - | 清理资源 |

## 五、内存管理的常见策略

### 5.1 谁分配谁释放

```c
// 好的模式：函数内分配和释放
void process_data(const char *input) {
    char *buffer = malloc(strlen(input) + 1);
    if (!buffer) return;

    strcpy(buffer, input);
    // ... 处理数据 ...
    free(buffer);  // 分配者负责释放
}

// 有问题的模式：返回分配的内存给调用者
char *create_buffer(size_t size) {
    char *buf = malloc(size);
    // 调用者必须记得 free(buf) !
    return buf;
}
```

### 5.2 资源获取即初始化(RAII)思想

```c
// 模仿RAII的清理模式
#define CLEANUP __attribute__((cleanup(cleanup_free)))

void cleanup_free(void *p) {
    free(*(void **)p);
}

void raii_example(void) {
    // 使用cleanup属性，函数退出时自动释放
    CLEANUP char *data = malloc(256);
    if (!data) return;

    strcpy(data, "自动清理示例");
    printf("%s\n", data);
    // 函数结束时自动调用 free(data)
}
```

## 六、关键要点总结

> **核心原则**
> 1. **每次malloc必须对应一次free** - 分配和释放必须成对出现
> 2. **检查分配返回值** - malloc可能返回NULL
> 3. **释放后置NULL** - 防止悬空指针
> 4. **不要越界访问** - 只访问已分配范围内的内存
> 5. **不要重复释放** - 同一指针只能free一次
> 6. **不要使用已释放的内存** - 避免use-after-free

> **学习建议**
> - 内存管理是C语言中最容易出错的部分
> - 建议使用Valgrind等工具检测内存问题
> - 养成良好的分配/释放习惯
> - 在实际项目中使用智能指针思想简化管理
