# 堆管理原理

## 一、堆管理概述

堆管理器是malloc/free的底层实现，负责管理进程的堆内存区域。不同系统的堆管理器实现不同，但基本原理相似。

```
堆管理器在内存中的位置:

  程序调用: malloc() / free() / realloc()
       ↓
  ┌─────────────────────────┐
  │   堆管理器 (ptmalloc等)  │  ← glibc的malloc实现
  └─────────┬───────────────┘
            ↓
  ┌─────────────────────────┐
  │   brk/sbrk 或 mmap      │  ← 系统调用
  └─────────┬───────────────┘
            ↓
  ┌─────────────────────────┐
  │   操作系统内核           │
  └─────────────────────────┘
```

## 二、brk/sbrk系统调用

```c
#include <stdio.h>
#include <unistd.h>
#include <string.h>

void brk_sbrk_demo(void) {
    // sbrk调整program break的位置
    // sbrk(0) 返回当前break位置
    void *initial = sbrk(0);
    printf("初始break: %p\n", initial);

    // sbrk(4096) 向上移动break，分配4KB
    void *new_mem = sbrk(4096);
    if (new_mem == (void *)-1) {
        perror("sbrk");
        return;
    }
    printf("分配4KB: %p\n", new_mem);

    // 使用分配的内存
    strcpy((char *)new_mem, "通过sbrk分配的内存");
    printf("内容: %s\n", (char *)new_mem);

    void *current = sbrk(0);
    printf("当前break: %p\n", current);
    printf("差值: %ld 字节\n", (long)(current - initial));

    // 注意：sbrk不支持释放单个块
    // 现代malloc主要用mmap代替sbrk
}

int main(void) {
    brk_sbrk_demo();
    return 0;
}
```

## 三、ptmalloc (glibc malloc)

glibc使用ptmalloc2作为默认堆管理器，基于dlmalloc改进。

### 3.1 核心数据结构

```
ptmalloc的chunk结构:

  已分配chunk:                空闲chunk:
  ┌──────────────┐           ┌──────────────┐
  │ prev_size    │ (前一个    │ prev_size    │
  │ size         │  chunk大小)│ size         │
  ├──────────────┤           ├──────────────┤
  │ fd (用户数据) │           │ fd (前向指针) │  ← 空闲链表
  │ bk (用户数据) │           │ bk (后向指针) │
  │ ...          │           │ ... (空闲空间) │
  └──────────────┘           └──────────────┘

  chunk大小按16字节对齐(64位系统)
  size的最低3位用作标志位:
    bit 0: PREV_INUSE - 前一个chunk是否在使用
    bit 1: IS_MMAPPED - 是否由mmap分配
    bit 2: NON_MAIN_ARENA - 是否在非主arena
```

### 3.2 bins机制

```
ptmalloc的bins (空闲chunk分类):

  ┌──────────────────────────────────────────┐
  │  Fast Bins (11个)                        │
  │  大小: 16-112字节，单链表，不合并相邻空闲块  │
  ├──────────────────────────────────────────┤
  │  Small Bins (62个)                       │
  │  大小: 120-1024字节，双链表，FIFO         │
  ├──────────────────────────────────────────┤
  │  Large Bins (63个)                       │
  │  大小: >1024字节，双链表，排序存储         │
  ├──────────────────────────────────────────┤
  │  Unsorted Bin (1个)                      │
  │  刚释放的chunk先放这里，下次malloc时整理    │
  ├──────────────────────────────────────────┤
  │  TCache (Thread Cache, glibc 2.26+)     │
  │  每线程64个单链表，快速分配               │
  └──────────────────────────────────────────┘
```

### 3.3 分配策略

```c
#include <stdio.h>
#include <stdlib.h>

void allocation_strategy_demo(void) {
    // 小分配 → fast bin/tcache
    void *p1 = malloc(32);    // 可能走fast bin

    // 中等分配 → small bin
    void *p2 = malloc(512);   // 可能走small bin

    // 大分配 → large bin或mmap
    void *p3 = malloc(128 * 1024);  // 可能用mmap

    // 超大分配 → 直接mmap
    void *p4 = malloc(128 * 1024 * 1024);  // 用mmap

    printf("小分配:  %p\n", p1);
    printf("中分配:  %p\n", p2);
    printf("大分配:  %p\n", p3);
    printf("超大分配: %p\n", p4);

    free(p1); free(p2); free(p3); free(p4);
}

int main(void) {
    allocation_strategy_demo();
    return 0;
}
```

## 四、mmap大块分配

```c
#include <stdio.h>
#include <stdlib.h>
#include <sys/mman.h>

void mmap_allocation_demo(void) {
    // malloc对于大于一定阈值(通常128KB)的分配
    // 直接使用mmap而不是从堆中分配

    // 这些分配在free时直接munmap，不放回bins
    size_t threshold = 128 * 1024;

    void *small = malloc(threshold - 1);  // 从堆分配
    void *large = malloc(threshold + 1);  // 用mmap分配

    printf("堆分配: %p\n", small);
    printf("mmap分配: %p\n", large);

    // 验证：mmap分配的地址通常在不同的内存区域
    printf("地址差异: %ld 页\n",
           labs((long)large - (long)small) / 4096);

    free(small);
    free(large);
}

int main(void) {
    mmap_allocation_demo();
    return 0;
}
```

## 五、关键要点

> **堆管理原理要点**
> 1. ptmalloc是glibc默认堆管理器，基于dlmalloc改进
> 2. 小块用fast bins/tcache快速分配
> 3. 中等块用small/large bins管理
> 4. 大块(>128KB)直接用mmap分配
> 5. brk/sbrk扩展堆区域，但现代主要用mmap
> 6. chunk是堆管理的基本单位，包含元数据
> 7. 多线程使用多个arena减少锁竞争
