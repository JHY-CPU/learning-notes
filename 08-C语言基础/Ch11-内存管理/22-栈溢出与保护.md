# 栈溢出与保护

## 一、什么是栈溢出

栈溢出(Stack Overflow)是指程序使用的栈空间超过了系统分配的栈大小限制。常见原因包括递归过深和局部变量过大。

```
栈溢出过程:

  正常状态:               栈溢出:
  ┌─────────┐ 高地址      ┌─────────┐
  │ 栈帧1   │             │ 栈帧1   │
  ├─────────┤             ├─────────┤
  │ 栈帧2   │             │ 栈帧2   │
  ├─────────┤             ├─────────┤
  │ 栈帧3   │             │  ...    │ (继续增长)
  ├─────────┤             ├─────────┤
  │ 栈帧4   │             │ 栈帧N   │ ← 越过边界!
  ├─────────┤ 栈边界      ├─────────┤ 栈边界
  │ (空闲)  │             │  堆/其他 │ → 段错误!
  └─────────┘             └─────────┘
```

## 二、递归过深导致栈溢出

```c
#include <stdio.h>
#include <stdlib.h>

// 危险：无限递归
int factorial_bad(int n) {
    int local_data[100];  // 每次递归消耗约400字节
    local_data[0] = n;    // 防止优化掉

    if (n <= 1) return 1;
    return n * factorial_bad(n - 1);
    // 递归深度太大 → 栈溢出
}

// 安全：限制递归深度
int factorial_safe(int n, int max_depth) {
    static int depth = 0;
    if (depth >= max_depth) {
        fprintf(stderr, "递归深度超限\n");
        return -1;
    }
    depth++;
    int result = (n <= 1) ? 1 : n * factorial_safe(n - 1, max_depth);
    depth--;
    return result;
}

// 最安全：迭代实现
int factorial_iterative(int n) {
    int result = 1;
    for (int i = 2; i <= n; i++) {
        result *= i;
    }
    return result;
}

int main(void) {
    // factorial_bad(10000);  // 取消注释会导致栈溢出崩溃

    printf("安全递归: 5! = %d\n", factorial_safe(5, 1000));
    printf("迭代实现: 5! = %d\n", factorial_iterative(5));

    return 0;
}
```

## 三、大局部变量导致栈溢出

```c
#include <stdio.h>

// 危险：栈上分配过大数组
void large_stack_array(void) {
    // 在栈上分配1MB
    char big_buffer[1024 * 1024];
    big_buffer[0] = 'A';

    // 再分配1MB
    char another_buffer[1024 * 1024];
    another_buffer[0] = 'B';

    // 多次调用可能栈溢出
    printf("大缓冲区分配成功\n");
}

// 安全：使用堆分配
void safe_large_allocation(void) {
    char *big_buffer = malloc(1024 * 1024);
    if (!big_buffer) {
        fprintf(stderr, "堆分配失败\n");
        return;
    }

    big_buffer[0] = 'A';
    printf("堆上大缓冲区分配成功\n");

    free(big_buffer);
}

int main(void) {
    safe_large_allocation();
    return 0;
}
```

## 四、栈保护机制

### 4.1 Stack Canary (栈金丝雀)

```
栈保护原理:

  无保护的栈帧:            有Canary保护的栈帧:
  ┌──────────────┐         ┌──────────────┐
  │ 返回地址      │         │ 返回地址      │
  ├──────────────┤         ├──────────────┤
  │ 旧EBP        │         │ 旧EBP        │
  ├──────────────┤         ├──────────────┤
  │ 局部变量      │         │ 局部变量      │
  └──────────────┘         ├──────────────┤
  ↓ 如果缓冲区溢出          │ Canary值     │ ← 溢出会先破坏这里
  可以覆盖返回地址          └──────────────┘
                            函数返回前检查Canary
                            被修改则立即终止程序
```

```c
// GCC自动插入的栈保护代码概念
void protected_function(void) {
    unsigned long canary = __stack_chk_guard;  // 从全局变量读取
    char buffer[64];

    // 使用buffer...
    strcpy(buffer, "safe data");

    // 函数返回前检查
    if (canary != __stack_chk_guard) {
        __stack_chk_fail();  // 检测到栈溢出，终止程序
    }
}
```

### 4.2 编译选项

```bash
# 栈保护选项
gcc -fstack-protector       # 为有字符数组的函数加保护
gcc -fstack-protector-all   # 为所有函数加保护
gcc -fstack-protector-strong # 智能保护（推荐）
gcc -fstack-clash-protection # 栈冲突保护

# 禁用（不推荐）
gcc -fno-stack-protector
```

## 五、检测和预防栈溢出

```c
#include <stdio.h>
#include <stdlib.h>

// 方法1：检测栈指针位置
void check_stack_usage(void) {
    int local_var;
    static int *stack_bottom = NULL;

    if (stack_bottom == NULL) {
        // 首次调用，记录栈底
        stack_bottom = &local_var;
    }

    ptrdiff_t used = stack_bottom - &local_var;
    if (used < 0) used = -used;

    printf("栈已使用约 %td 字节\n", used * sizeof(int));

    // 如果使用过多，发出警告
    if (used > 100000) {
        fprintf(stderr, "警告：栈使用量过大!\n");
    }
}

// 方法2：用迭代替代递归
void safe_depth_first_search(void) {
    // 用显式栈（堆上）替代系统调用栈
    int *stack = malloc(sizeof(int) * 10000);
    int top = 0;

    stack[top++] = 0;  // 根节点

    while (top > 0) {
        int node = stack[--top];
        printf("访问节点 %d\n", node);

        // 将子节点压栈
        if (node < 10) {
            stack[top++] = node * 2 + 2;  // 右子节点
            stack[top++] = node * 2 + 1;  // 左子节点
        }
    }

    free(stack);
}

int main(void) {
    check_stack_usage();
    safe_depth_first_search();
    return 0;
}
```

## 六、关键要点

> **栈溢出与保护要点**
> 1. 栈大小通常1-8MB，深度递归和大局部变量可能导致溢出
> 2. 编译时加`-fstack-protector-strong`启用栈保护
> 3. 无限递归是最常见的栈溢出原因
> 4. 用迭代替代深度递归，或限制递归深度
> 5. 大数组/缓冲区使用堆分配而非栈分配
> 6. Stack Canary检测栈溢出并终止程序
> 7. `-Wstack-usage=N`编译警告可以发现潜在问题
