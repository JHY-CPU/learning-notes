# 项目经验总结

## 一、STAR法则

```c
/*
 * STAR法则描述项目经验：
 *
 * S (Situation) - 背景：项目是在什么情况下开展的
 * T (Task)      - 任务：你负责什么，承担什么角色
 * A (Action)    - 行动：你具体做了什么，技术方案是什么
 * R (Result)    - 结果：取得了什么成果，量化指标
 *
 * 示例：
 * "在XX项目中（S），我负责优化数据处理模块（T），
 *  采用了多线程+内存池的技术方案（A），
 *  最终将处理速度提升了3倍，内存使用降低40%（R）。"
 */
```

## 二、C语言项目经验示例

```c
/*
 * 示例1：嵌入式网关项目
 *
 * 背景：公司需要开发一款工业物联网网关设备
 * 角色：嵌入式软件工程师
 * 技术栈：C语言 + Linux + 多线程 + TCP/UDP
 *
 * 关键技术点：
 * 1. 设计了事件驱动的消息处理架构
 * 2. 实现了高效的协议解析引擎
 * 3. 使用epoll处理高并发连接
 * 4. 内存池管理减少碎片
 *
 * 成果：
 * - 支持1000+设备同时在线
 * - 99.9%的系统可用性
 * - 数据延迟 < 100ms
 */

// 展示代码能力的片段
typedef struct {
    uint16_t cmd_id;
    uint32_t device_id;
    uint8_t  payload[256];
    uint16_t crc;
} __attribute__((packed)) DevicePacket;

int parse_packet(const uint8_t *raw, int len, DevicePacket *pkt) {
    if (len < 8) return -1;  // 最小包长
    memcpy(pkt, raw, len);
    uint16_t calc_crc = crc16(raw, len - 2);
    if (calc_crc != pkt->crc) return -2;  // CRC错误
    return 0;
}
```

## 三、如何描述技术难点

```c
/*
 * 描述技术难点的模板：
 *
 * 1. 问题是什么（What）
 *    "在高并发场景下，频繁的小块内存分配导致严重的内存碎片"
 *
 * 2. 为什么难（Why）
 *    "系统运行24小时后，malloc/free耗时增加10倍，最终OOM"
 *
 * 3. 怎么解决（How）
 *    "设计了固定大小的内存池，预分配大块内存，内部用空闲链表管理"
 *
 * 4. 效果如何（Result）
 *    "分配耗时从平均50us降到5us，系统稳定运行30天无重启"
 */

// 内存池设计（简化版）
#define POOL_BLOCK_SIZE 256
#define POOL_BLOCK_COUNT 4096

typedef struct MemPool {
    char *memory;
    void *free_list;
    pthread_mutex_t lock;
} MemPool;

MemPool *pool_init() {
    MemPool *pool = malloc(sizeof(MemPool));
    pool->memory = malloc(POOL_BLOCK_SIZE * POOL_BLOCK_COUNT);
    pool->free_list = NULL;
    pthread_mutex_init(&pool->lock, NULL);

    // 构建空闲链表
    for (int i = 0; i < POOL_BLOCK_COUNT; i++) {
        void *block = pool->memory + i * POOL_BLOCK_SIZE;
        *(void **)block = pool->free_list;
        pool->free_list = block;
    }
    return pool;
}
```
