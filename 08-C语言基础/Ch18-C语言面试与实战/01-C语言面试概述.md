# C语言面试概述

## 一、面试考察范围

C语言面试通常涵盖以下几个核心领域：

### 1.1 语言基础

- 数据类型、变量、常量
- 运算符优先级与结合性
- 控制流语句（if/switch/for/while）
- 函数定义、声明、调用
- 作用域与生命周期

### 1.2 指针与内存

- 指针基础操作与指针运算
- 多级指针与指针数组
- 动态内存分配（malloc/free/calloc/realloc）
- 内存泄漏与野指针
- 内存对齐与结构体大小计算

### 1.3 数据结构与算法

- 链表（单链表、双链表、循环链表）
- 栈与队列
- 二叉树与二叉搜索树
- 排序算法（快排、归并、堆排序等）
- 查找算法（二分查找及其变体）

### 1.4 系统编程

- 进程与线程
- 进程间通信（IPC）
- 文件I/O操作
- 网络编程（socket）
- 并发与同步

### 1.5 预处理器与编译

- 宏定义与宏展开
- 条件编译
- 头文件包含机制
- 编译链接过程

## 三、常见题型分析

### 3.1 概念理解题

```c
// 经典问题：以下代码输出什么？为什么？
#include <stdio.h>

int main(void) {
    int a = 5;
    int b = a++ + ++a;  // 未定义行为！
    printf("b = %d\n", b);
    return 0;
}
```

> **关键点**：同一表达式中对同一变量多次修改属于未定义行为（Undefined Behavior），不同编译器结果不同。

### 3.2 代码阅读题

```c
// 阅读以下代码，指出问题并修复
#include <stdio.h>
#include <string.h>

void reverse_string(char *str) {
    int len = strlen(str);
    for (int i = 0; i < len / 2; i++) {
        char temp = str[i];
        str[i] = str[len - 1 - i];
        str[len - 1 - i] = temp;
    }
}

int main(void) {
    char *s = "hello";  // 问题：字符串字面量不可修改
    reverse_string(s);
    printf("%s\n", s);
    return 0;
}
```

### 3.3 手写代码题

```c
// 手写：实现strlen函数
#include <stddef.h>

size_t my_strlen(const char *str) {
    if (str == NULL) return 0;
    const char *p = str;
    while (*p != '\0') {
        p++;
    }
    return (size_t)(p - str);
}
```

### 3.4 算法实现题

```c
// 手写：快速排序
#include <stdio.h>

void swap(int *a, int *b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}

int partition(int arr[], int low, int high) {
    int pivot = arr[high];
    int i = low - 1;
    for (int j = low; j < high; j++) {
        if (arr[j] <= pivot) {
            i++;
            swap(&arr[i], &arr[j]);
        }
    }
    swap(&arr[i + 1], &arr[high]);
    return i + 1;
}

void quick_sort(int arr[], int low, int high) {
    if (low < high) {
        int pi = partition(arr, low, high);
        quick_sort(arr, low, pi - 1);
        quick_sort(arr, pi + 1, high);
    }
}
```

## 四、面试注意事项

1. **代码规范**：变量命名清晰，适当添加注释
2. **边界处理**：始终考虑空指针、空字符串、溢出等边界情况
3. **时间复杂度**：写完代码后主动分析时间/空间复杂度
4. **测试用例**：主动给出测试用例验证代码正确性
5. **沟通交流**：遇到不确定的问题，先说明思路再编码
