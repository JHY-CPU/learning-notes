# 22 - 函数宏与内联对比

## 一、函数宏（Function-Like Macro）

### 1.1 基本语法

```c
#define 宏名(参数) 替换文本
```

```c
#define SQUARE(x) ((x) * (x))
#define MAX(a, b) ((a) > (b) ? (a) : (b))
#define MIN(a, b) ((a) < (b) ? (a) : (b))
#define ABS(x) ((x) < 0 ? -(x) : (x))
```

### 1.2 使用示例

```c
#include <stdio.h>

#define SQUARE(x) ((x) * (x))
#define MAX(a, b) ((a) > (b) ? (a) : (b))

int main() {
    int a = 5;
    printf("SQUARE(5) = %d\n", SQUARE(5));       // 25
    printf("MAX(3, 7) = %d\n", MAX(3, 7));       // 7
    printf("SQUARE(a+1) = %d\n", SQUARE(a+1));   // 36（正确！因为有括号）
    return 0;
}
```

### 1.3 宏的陷阱

```c
// 陷阱一：多次求值
#define MAX(a, b) ((a) > (b) ? (a) : (b))

int x = 5, y = 3;
int z = MAX(x++, y++);  // 展开为 ((x++) > (y++) ? (x++) : (y++))
                        // x++ 执行了 2 次！结果不可预测

// 陷阱二：缺少括号导致优先级问题
#define BAD_ADD(a, b) a + b        // 错误
int r = BAD_ADD(1, 2) * 3;         // 展开为 1 + 2 * 3 = 7，不是 9！

#define GOOD_ADD(a, b) ((a) + (b)) // 正确
int r = GOOD_ADD(1, 2) * 3;        // 展开为 (1 + 2) * 3 = 9

// 陷阱三：分号问题
#define SWAP(a, b) { int t = a; a = b; b = t; }

if (condition)
    SWAP(x, y);    // 展开后 { ... }; 后面的分号导致 if 后面有空语句
else               // 语法错误！
    do_other();
```

### 1.4 使用 do-while(0) 包装多语句宏

```c
// 推荐的写法
#define SWAP(a, b) do { \
    int _t = (a);       \
    (a) = (b);          \
    (b) = _t;           \
} while (0)

// 使用时
if (condition)
    SWAP(x, y);   // 安全！展开后是一个语句
else
    do_other();
```

## 二、内联函数

### 2.1 基本语法

```c
inline int square(int x) { return x * x; }
inline int max(int a, int b) { return a > b ? a : b; }
```

### 2.2 安全性

```c
// 无副作用问题
inline int max(int a, int b) { return a > b ? a : b; }
int z = max(x++, y++);  // x++ 和 y++ 各执行一次，符合预期

// 类型安全
inline int square(int x) { return x * x; }
// square("hello");  // 编译错误！
```

## 三、全面对比

### 3.1 核心差异

| 特性           | 函数宏 (#define)          | 内联函数 (inline)          |
|---------------|--------------------------|--------------------------|
| 处理阶段       | **预处理**（文本替换）    | **编译**（语法分析）       |
| 类型检查       | **无**                   | **有**                    |
| 作用域         | 无作用域概念             | 遵循 C 作用域规则          |
| 调试           | 难以调试                 | 可以调试                   |
| 副作用         | 可能多次求值             | 参数只求值一次              |
| 返回值         | 无返回值概念             | 有明确返回类型              |
| 括号依赖       | 必须大量使用括号         | 无此问题                   |
| 递归           | 不支持                   | 支持（但可能不被内联）      |
| 指针           | 不能取地址               | 可以取函数地址              |
| 头文件守卫     | 不需要特殊处理           | 需要 `static inline`      |

### 3.2 性能对比

```c
// 宏——在预处理阶段展开
#define SQUARE(x) ((x) * (x))
int a = SQUARE(5);  // 预处理后: int a = ((5) * (5));
// 编译器看到的是已经展开的代码，没有函数调用

// 内联函数——在编译阶段展开（如果编译器选择内联）
inline int square(int x) { return x * x; }
int a = square(5);  // 编译器可能优化为: int a = 5 * 5;
// 编译器有选择权，可能不内联
```

### 3.3 实际性能

在 `-O2` 优化下，两者生成的汇编代码通常**完全相同**：

```c
// 以下两段代码在 -O2 下生成相同的汇编
#define MACRO_SQUARE(x) ((x) * (x))
inline int func_square(int x) { return x * x; }
```

## 四、何时使用宏 vs 内联

### 4.1 优先使用内联函数

```c
// 类型安全的通用操作
static inline int clamp(int val, int lo, int hi) {
    if (val < lo) return lo;
    if (val > hi) return hi;
    return val;
}

// 需要调试的操作
static inline void debug_print(const char *msg) {
    fprintf(stderr, "[DEBUG] %s\n", msg);
}
```

### 4.2 必须使用宏的场景

```c
// 1. 编译时常量
#define BUFFER_SIZE 1024
#define MAX_PATH 260

// 2. 条件编译
#ifdef DEBUG
    #define LOG(msg) fprintf(stderr, "%s\n", msg)
#else
    #define LOG(msg) ((void)0)
#endif

// 3. 字符串化和拼接
#define STRINGIFY(x) #x
#define CONCAT(a, b) a##b
#define PRINT_VAR(var) printf(#var " = %d\n", var)

// 4. 泛型操作（C11 _Generic 配合宏）
#define TYPE_NAME(x) _Generic((x), \
    int: "int", \
    double: "double", \
    char*: "string", \
    default: "unknown" \
)
```

### 4.3 内联替代宏的案例

```c
// 旧式（宏）
#define MAX(a, b) ((a) > (b) ? (a) : (b))
#define MIN(a, b) ((a) < (b) ? (a) : (b))
#define CLAMP(x, lo, hi) MAX(lo, MIN(x, hi))

// 新式（内联函数）—— 更安全
static inline int max_int(int a, int b) { return a > b ? a : b; }
static inline int min_int(int a, int b) { return a < b ? a : b; }
static inline int clamp_int(int x, int lo, int hi) {
    return max_int(lo, min_int(x, hi));
}

// 如果需要泛型，可以结合 _Generic（C11）
#define max(a, b) _Generic((a), \
    int: max_int, \
    double: max_double \
)(a, b)
```

## 五、要点总结

> **关键点：**
> 1. 宏是**文本替换**，没有类型检查，可能导致多次求值和优先级问题。
> 2. 内联函数是**真正的函数**，有类型检查，参数只求值一次。
> 3. 在 `-O2` 优化下，两者性能通常相同。
> 4. **优先使用内联函数**，宏只在必须使用预处理特性时使用。
> 5. 多语句宏必须用 `do { ... } while(0)` 包裹。
> 6. 宏适合：条件编译、字符串化、编译时常量、泛型（`_Generic`）。
> 7. 头文件中的内联函数使用 `static inline` 避免链接冲突。
