# 24 - 模块化编程

## 一、什么是模块化编程

**模块化编程**是将程序划分为多个独立的模块，每个模块负责一个明确的功能。在 C 语言中，一个模块通常由一个 `.c` 源文件和对应的 `.h` 头文件组成。

```
project/
  include/
    list.h           -- 链表模块的接口
    utils.h          -- 工具函数接口
    config.h         -- 配置接口
  src/
    list.c           -- 链表模块的实现
    utils.c          -- 工具函数实现
    config.c         -- 配置实现
    main.c           -- 主程序
  Makefile
```

## 二、源文件组织

### 2.1 单文件 vs 多文件

```c
// 单文件（bad）：所有代码都在 main.c 中
// 1000+ 行代码，难以维护

// 多文件（good）：按功能分模块
// main.c       -- 程序入口
// list.c/h     -- 链表
// sort.c/h     -- 排序
// utils.c/h    -- 工具函数
```

### 2.2 一个模块的典型结构

**module.h（接口）：**
```c
#ifndef MODULE_H
#define MODULE_H

// 类型定义
typedef struct Module Module;

// 公开函数声明
Module* module_create(void);
void module_destroy(Module *mod);
int module_process(Module *mod, const char *input);
int module_get_result(const Module *mod);

#endif
```

**module.c（实现）：**
```c
#include "module.h"
#include <stdlib.h>
#include <string.h>

// 内部结构定义
struct Module {
    char *buffer;
    int result;
    int state;
};

// static 函数：只在本文件可见
static int validate_input(const char *input) {
    return input != NULL && input[0] != '\0';
}

// 公开函数实现
Module* module_create(void) {
    Module *mod = malloc(sizeof(Module));
    if (mod) {
        mod->buffer = NULL;
        mod->result = 0;
        mod->state = 0;
    }
    return mod;
}

void module_destroy(Module *mod) {
    if (mod) {
        free(mod->buffer);
        free(mod);
    }
}
```

## 三、头文件设计

### 3.1 头文件应该包含

```c
// 1. 函数声明（不是定义）
void func(void);

// 2. 类型定义
typedef struct { } MyType;

// 3. 宏定义
#define MAX_SIZE 1024

// 4. 全局变量声明（extern）
extern int global_counter;

// 5. 常量定义
static const int VERSION = 1;
```

### 3.2 头文件不应该包含

```c
// 1. 函数定义（static inline 除外）
void func(void) { }  // 错误！会导致多重定义

// 2. 全局变量定义
int global_counter = 0;  // 错误！每个包含此头文件的 .c 文件都会有定义

// 3. static 变量的定义（会创建多个副本）
static int count = 0;  // 每个 .c 文件有自己的 count，可能不是意图
```

## 四、extern 的使用

### 4.1 声明全局变量

```c
// config.h
#ifndef CONFIG_H
#define CONFIG_H

extern int g_debug_level;      // 声明
extern const char *g_app_name; // 声明

#endif
```

```c
// config.c
#include "config.h"

int g_debug_level = 0;                // 定义
const char *g_app_name = "MyApp";     // 定义
```

```c
// main.c
#include "config.h"

int main() {
    g_debug_level = 1;  // 通过 extern 声明访问
    printf("%s\n", g_app_name);
    return 0;
}
```

### 4.2 extern 函数

```c
// 函数默认就是 extern 的，通常不需要显式写
extern int add(int a, int b);  // 显式 extern（可选）
int add(int a, int b);         // 等价（隐式 extern）
```

## 五、编译单元

### 5.1 什么是编译单元

每个 `.c` 文件（连同它包含的所有头文件）构成一个**编译单元**。编译器独立编译每个 `.c` 文件，然后链接器将它们合并。

```
main.c ──编译──> main.o ─┐
                          ├──链接──> 可执行文件
utils.c ──编译──> utils.o ─┤
                          │
list.c ──编译──> list.o ──┘
```

### 5.2 编译过程

```bash
# 分别编译
gcc -c -o main.o main.c
gcc -c -o utils.o utils.c
gcc -c -o list.o list.c

# 链接
gcc -o myprogram main.o utils.o list.o

# 一步完成
gcc -o myprogram main.c utils.c list.c
```

## 六、避免重复包含

### 6.1 头文件守卫

```c
// utils.h
#ifndef UTILS_H
#define UTILS_H

// 头文件内容...

#endif
```

### 6.2 #pragma once（非标准但广泛支持）

```c
// utils.h
#pragma once

// 头文件内容...
```

## 七、循环依赖问题

### 7.1 什么是循环依赖

```c
// a.h
#include "b.h"
void func_a(B *b);

// b.h
#include "a.h"
void func_b(A *a);  // 循环包含！
```

### 7.2 解决方法：前向声明

```c
// a.h
#ifndef A_H
#define A_H

// 前向声明（不包含 b.h）
typedef struct B B;
void func_a(B *b);

#endif
```

```c
// b.h
#ifndef B_H
#define B_H

typedef struct A A;
void func_b(A *a);

#endif
```

## 八、要点总结

> **关键点：**
> 1. 模块化将程序划分为独立的 `.c`/`.h` 文件对。
> 2. **头文件声明接口**，源文件提供实现。
> 3. `extern` 用于跨文件访问全局变量和函数。
> 4. 每个 `.c` 文件是一个独立的编译单元。
> 5. 使用头文件守卫（`#ifndef`）或 `#pragma once` 避免重复包含。
> 6. 避免循环依赖，使用**前向声明**解决。
> 7. 全局变量的定义只能在**一个** `.c` 文件中，其他文件用 `extern` 声明。
