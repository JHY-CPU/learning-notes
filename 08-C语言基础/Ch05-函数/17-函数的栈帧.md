# 17 - 函数的栈帧

## 一、什么是栈帧

**栈帧（Stack Frame）**，也叫活动记录（Activation Record），是函数调用时在**调用栈**上为该函数分配的一块内存区域。每次函数调用都会创建一个新的栈帧，函数返回时销毁。

```
高地址
  +------------------+
  |    main 栈帧     |   <- main 函数的局部变量等
  +------------------+
  |   func1 栈帧     |   <- func1 被调用时创建
  +------------------+
  |   func2 栈帧     |   <- func2 被 func1 调用时创建
  +------------------+
  |       ...        |
低地址（栈增长方向）
```

## 二、栈帧的组成

一个典型的栈帧包含以下部分：

```
+---------------------------+  高地址
| 调用者的栈帧               |
+---------------------------+
| 返回地址 (Return Address) |  <- 函数执行完后回到哪里
+---------------------------+
| 旧的帧指针 (Saved EBP)    |  <- 调用者的基址指针
+---------------------------+  <- 当前 EBP 指向这里
| 参数 1                    |  <- 函数的参数（或在调用者栈帧中）
| 参数 2                    |
+---------------------------+
| 局部变量 1                |  <- 函数的局部变量
| 局部变量 2                |
| ...                       |
+---------------------------+  <- 当前 ESP 指向这里
低地址
```

### 2.1 关键寄存器

| 寄存器    | 名称         | 作用                              |
|----------|-------------|----------------------------------|
| EBP/RBP  | 基址指针     | 指向当前栈帧的底部（帧指针）       |
| ESP/RSP  | 栈指针       | 指向当前栈顶                      |
| EIP/RIP  | 指令指针     | 指向下一条要执行的指令             |

## 三、函数调用的完整过程

### 3.1 调用前（Caller 端）

```
1. 将参数从右到左压栈
2. 执行 CALL 指令（将返回地址压栈，跳转到被调函数）
```

### 3.2 函数序言（Callee 端开头）

```
1. PUSH EBP           ; 保存调用者的帧指针
2. MOV EBP, ESP       ; 设置新的帧指针
3. SUB ESP, N         ; 为局部变量分配空间
```

### 3.3 函数尾声（Callee 端结尾）

```
1. MOV ESP, EBP       ; 释放局部变量空间
2. POP EBP            ; 恢复调用者的帧指针
3. RET                ; 弹出返回地址，跳转回去
```

## 四、实例分析

### 4.1 C 代码

```c
int add(int a, int b) {
    int result = a + b;
    return result;
}

int main() {
    int x = 3;
    int y = 5;
    int z = add(x, y);
    return 0;
}
```

### 4.2 对应的汇编（简化，x86）

```asm
add:
    push ebp              ; 保存旧的帧指针
    mov ebp, esp          ; 建立新帧
    sub esp, 4            ; 分配局部变量空间 (result)

    mov eax, [ebp+8]      ; 取参数 a
    add eax, [ebp+12]     ; 加上参数 b
    mov [ebp-4], eax      ; 存入 result
    mov eax, [ebp-4]      ; 取出 result 作为返回值

    mov esp, ebp          ; 释放局部变量
    pop ebp               ; 恢复旧帧指针
    ret                   ; 返回

main:
    push ebp
    mov ebp, esp
    sub esp, 12           ; 分配 x, y, z 的空间

    mov dword [ebp-4], 3  ; x = 3
    mov dword [ebp-8], 5  ; y = 5

    push dword [ebp-8]    ; 压入 y（参数从右到左）
    push dword [ebp-4]    ; 压入 x
    call add              ; 调用 add
    add esp, 8            ; 清理栈上的参数（cdecl 约定）
    mov [ebp-12], eax     ; z = add 的返回值

    mov eax, 0            ; return 0
    mov esp, ebp
    pop ebp
    ret
```

### 4.3 栈帧布局图

```
调用 add(x, y) 时的栈：

+--------------------+ 高地址
| main 的局部变量    |
|   x = 3 [ebp-4]   |
|   y = 5 [ebp-8]   |
|   z = ? [ebp-12]  |
+--------------------+
| 旧 EBP (main)     | <- main 的 ebp
+--------------------+
| 返回地址 (到 main) | <- call add 压入
+--------------------+
| 参数 a = 3         | <- ebp+8
| 参数 b = 5         | <- ebp+12
+--------------------+
| 旧 EBP (main)      | <- add 的 ebp
+--------------------+
| result [ebp-4]     | <- add 的局部变量
+--------------------+ <- add 的 esp
低地址
```

## 五、局部变量的布局

```c
void func(void) {
    int a = 1;
    int b = 2;
    char buf[16] = "hello";
    double d = 3.14;
}
```

```
栈帧布局（从高到低）：

+------------------+
| 旧 EBP           | <- EBP
+------------------+
| a = 1            | <- EBP - 4
+------------------+
| b = 2            | <- EBP - 8
+------------------+
| buf[15..0]       | <- EBP - 8 到 EBP - 24
| "hello\0..."     |   （16 字节）
+------------------+
| d = 3.14         | <- EBP - 32（8 字节对齐）
+------------------+
|                  | <- ESP
```

## 六、栈溢出

### 6.1 无限递归导致栈溢出

```c
void infinite(void) {
    int big[1000];  // 每次调用消耗 4000 字节
    infinite();     // 无限递归
}
// 很快就会栈溢出（通常栈大小 1-8 MB）
```

### 6.2 缓冲区溢出攻击

```c
// 危险代码——不要这样做！
void vulnerable(const char *input) {
    char buf[64];
    strcpy(buf, input);  // 如果 input > 64 字节，会覆盖栈帧
    // 攻击者可以覆盖返回地址，劫持程序控制流
}
```

### 6.3 防范措施

```c
// 使用安全函数
void safe_copy(const char *input) {
    char buf[64];
    strncpy(buf, input, sizeof(buf) - 1);  // 限制长度
    buf[sizeof(buf) - 1] = '\0';
}

// 或者使用动态分配（堆上的内存不受栈大小限制）
void safe_process(const char *input) {
    size_t len = strlen(input);
    char *buf = malloc(len + 1);
    if (buf) {
        strcpy(buf, input);
        // ...
        free(buf);
    }
}
```

## 七、要点总结

> **关键点：**
> 1. 每次函数调用都在调用栈上创建一个**栈帧**，包含参数、返回地址、局部变量等。
> 2. 栈帧由 **EBP（帧指针）** 和 **ESP（栈指针）** 共同管理。
> 3. 函数序言保存旧 EBP 并建立新帧，函数尾声恢复旧帧并返回。
> 4. 参数通常通过栈传递（或寄存器，取决于调用约定）。
> 5. 局部变量在栈帧中从高地址向低地址分配。
> 6. 递归深度过大或大数组局部变量会导致**栈溢出**。
> 7. 缓冲区溢出可以覆盖返回地址，是严重的安全漏洞。
