# 10 - 回调函数机制

## 一、回调函数的概念

**回调函数（Callback Function）** 是一种设计模式：将一个函数（回调）作为参数传递给另一个函数（框架/调用者），由后者在适当的时机"回调"执行这个函数。

```
调用者（Caller）          回调函数（Callback）
   +-----------+           +-----------+
   | 调用 callback(data) -->| 执行处理   |
   |           |           |           |
   | 等待回调  |<--返回-----| 处理完毕   |
   +-----------+           +-----------+
```

### 1.1 核心思想

```c
// 定义回调函数类型
typedef void (*Callback)(int);

// 框架函数：不关心具体做什么，只负责"调用"
void do_something(int value, Callback cb) {
    // 做一些处理...
    printf("处理数据: %d\n", value);

    // 调用回调函数，让调用者决定做什么
    if (cb) cb(value);
}

// 用户定义的回调函数
void my_callback(int val) {
    printf("回调收到值: %d\n", val * 2);
}

int main() {
    do_something(42, my_callback);
    return 0;
}
```

## 二、回调函数的实现方式

### 2.1 直接传递函数名

```c
void sort_int_array(int arr[], int n,
                    int (*cmp)(int, int)) {
    for (int i = 0; i < n - 1; i++)
        for (int j = 0; j < n - 1 - i; j++)
            if (cmp(arr[j], arr[j+1]) > 0) {
                int t = arr[j];
                arr[j] = arr[j+1];
                arr[j+1] = t;
            }
}

int ascending(int a, int b)  { return a - b; }
int descending(int a, int b) { return b - a; }

int main() {
    int arr[] = {5, 2, 8, 1, 9};
    sort_int_array(arr, 5, ascending);   // 升序
    sort_int_array(arr, 5, descending);  // 降序
    return 0;
}
```

### 2.2 使用 typedef 简化

```c
typedef int (*CompareFunc)(const void*, const void*);

void my_sort(void *base, size_t n, size_t size, CompareFunc cmp) {
    // 简化的排序实现
    char *arr = (char*)base;
    for (size_t i = 0; i < n - 1; i++)
        for (size_t j = 0; j < n - 1 - i; j++)
            if (cmp(arr + j * size, arr + (j+1) * size) > 0) {
                // 交换两个元素
                for (size_t k = 0; k < size; k++) {
                    char tmp = arr[j*size + k];
                    arr[j*size + k] = arr[(j+1)*size + k];
                    arr[(j+1)*size + k] = tmp;
                }
            }
}
```

### 2.3 带上下文的回调

```c
typedef void (*CallbackWithCtx)(int event, void *userdata);

void register_handler(CallbackWithCtx handler, void *userdata) {
    // 存储 handler 和 userdata

}

    const char *name = (const char*)userdata;
    printf("%s 收到事件: %d\n", name, event);
}

int main() {
    register_handler(my_handler, "我的处理器");
    return 0;
}
```

## 三、回调函数的使用场景

### 3.1 事件处理系统

```c
#include <stdio.h>
#include <string.h>

#define MAX_EVENTS 10
#define MAX_HANDLERS 5

typedef void (*EventHandler)(const char *event_data);

typedef struct {
    char event_name[32];
    EventHandler handlers[MAX_HANDLERS];
    int handler_count;
} EventBus;

EventBus bus;

void event_init() {
    memset(&bus, 0, sizeof(bus));
}

void event_on(const char *name, EventHandler handler) {
    strncpy(bus.event_name, name, 31);
    if (bus.handler_count < MAX_HANDLERS) {
        bus.handlers[bus.handler_count++] = handler;
    }
}

void event_emit(const char *data) {
    for (int i = 0; i < bus.handler_count; i++) {
        bus.handlers[i](data);
    }
}

// 用户定义的事件处理器
void on_click(const char *data) {
    printf("点击事件: %s\n", data);
}

void on_log(const char *data) {
    printf("[LOG] %s\n", data);
}

int main() {
    event_init();
    event_on("click", on_click);
    event_on("click", on_log);
    event_emit("按钮A");
    return 0;
}
```

### 3.2 排序中的比较回调

```c
typedef struct {

int cmp_by_age(const void *a, const void *b) {
```

## 五、回调函数的注意事项

### 5.1 检查回调是否为 NULL

```c
void safe_call(int value, void (*cb)(int)) {
    // 必须检查！
    if (cb != NULL) {
        cb(value);
    }
}
```

### 5.3 回调中的异常处理

```c
// 回调函数不应抛出异常（C 语言没有异常机制）
// 但应注意回调函数可能修改全局状态
typedef int (*SafeCallback)(int value, char *errbuf, int errsize);

int process(int value, SafeCallback cb) {
    char errbuf[256];
    int result = cb(value, errbuf, sizeof(errbuf));
    if (result < 0) {
        fprintf(stderr, "回调错误: %s\n", errbuf);
    }
    return result;
}
```

## 六、要点总结

> **关键点：**
> 1. 回调函数是通过函数指针传递的函数，由框架代码在适当时机调用。
> 2. 回调实现了**控制反转**——框架决定何时调用，用户决定做什么。
> 3. 常见场景：**事件处理**、**排序比较**、**迭代器**、**过滤器**。
> 4. 带 `void*` 上下文的回调更灵活，可以传递任意数据。
> 5. 始终检查回调指针是否为 `NULL`。
> 6. 回调函数的签名必须与框架期望的类型完全匹配。
