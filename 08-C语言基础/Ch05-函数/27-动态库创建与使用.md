# 27 - 动态库创建与使用

## 一、什么是动态库

**动态库（Dynamic/Shared Library）** 是在程序运行时才被加载的库。多个程序可以共享同一份库的内存副本，节省系统资源。

```
源文件 (.c) ──编译──> 目标文件 (.o) ──链接──> 动态库 (.so/.dll)
                                                     |
程序运行时 ───────────────────────────加载────────────┘
（库代码在运行时被加载，多个进程共享）
```

| 平台     | 扩展名    | 动态加载函数          |
|---------|----------|---------------------|
| Linux   | `.so`    | `dlopen/dlsym/dlclose` |
| macOS   | `.dylib` | `dlopen/dlsym/dlclose` |
| Windows | `.dll`   | `LoadLibrary/GetProcAddress/FreeLibrary` |

## 二、创建动态库

### 2.1 编译为位置无关代码（PIC）

```bash
# -fPIC 生成位置无关代码（Position Independent Code）
gcc -c -fPIC -I include -o math_utils.o src/math_utils.c
gcc -c -fPIC -I include -o string_utils.o src/string_utils.c
```

> **为什么需要 -fPIC：** 动态库在运行时被加载到不确定的内存地址，PIC 确保代码可以在任意地址正确执行。

### 2.2 链接为共享库

```bash
# -shared 表示创建共享库
gcc -shared -o libmyutils.so math_utils.o string_utils.o
```

### 2.3 一步完成

```bash
gcc -shared -fPIC -o libmyutils.so src/math_utils.c src/string_utils.c -I include
```

## 三、使用动态库

### 3.1 编译时链接

```bash
# 编译并链接动态库
gcc -o myapp main.c -I include -L lib -lmyutils
```

### 3.2 运行时查找

```bash
# 运行时需要找到 .so 文件
# 方式一：设置 LD_LIBRARY_PATH
export LD_LIBRARY_PATH=./lib:$LD_LIBRARY_PATH
./myapp

# 方式二：安装到系统目录
sudo cp libmyutils.so /usr/local/lib/
sudo ldconfig

# 方式三：在编译时指定运行时路径
gcc -o myapp main.c -L lib -lmyutils -Wl,-rpath,./lib
```

### 3.3 查看程序依赖的动态库

```bash
ldd myapp
# 输出示例：
#   linux-vdso.so.1 (0x00007ffd...)
#   libmyutils.so => ./lib/libmyutils.so (0x00007f...)
#   libc.so.6 => /lib/x86_64-linux-gnu/libc.so.6 (0x00007f...)
```

## 四、动态加载（dlopen/dlsym）

### 4.1 在运行时加载库

```c
#include <stdio.h>
#include <dlfcn.h>  // 动态加载函数

int main(void) {
    // 打开动态库
    void *handle = dlopen("./libmyutils.so", RTLD_LAZY);
    if (!handle) {
        fprintf(stderr, "加载失败: %s\n", dlerror());
        return 1;
    }

    // 查找函数符号
    // 注意：函数指针类型必须匹配
    typedef int (*AddFunc)(int, int);
    AddFunc add = (AddFunc)dlsym(handle, "add");

    const char *error = dlerror();
    if (error) {
        fprintf(stderr, "查找符号失败: %s\n", error);
        dlclose(handle);
        return 1;
    }

    // 调用函数
    int result = add(3, 5);
    printf("add(3, 5) = %d\n", result);

    // 关闭库
    dlclose(handle);
    return 0;
}
```

### 4.2 编译时链接 dl 库

```bash
gcc -o myapp main.c -ldl
```

### 4.3 dlopen 的标志

| 标志         | 说明                                      |
|-------------|------------------------------------------|
| `RTLD_LAZY` | 延迟绑定，符号在首次使用时解析（性能更好）   |
| `RTLD_NOW`  | 立即绑定，打开时解析所有符号（更安全）      |
| `RTLD_GLOBAL` | 库的符号对后续加载的库可见                |

## 五、Windows 上的动态库

### 5.1 创建 DLL

```c
// mylib.h
#ifdef _WIN32
    #ifdef BUILD_DLL
        #define DLL_API __declspec(dllexport)
    #else
        #define DLL_API __declspec(dllimport)
    #endif
#else
    #define DLL_API
#endif

DLL_API int add(int a, int b);
```

```c
// mylib.c
#define BUILD_DLL
#include "mylib.h"

DLL_API int add(int a, int b) {
    return a + b;
}
```

### 5.2 Windows 上的动态加载

```c
#include <windows.h>
#include <stdio.h>

int main(void) {
    HMODULE hDll = LoadLibrary("mylib.dll");
    if (!hDll) {
        fprintf(stderr, "加载失败\n");
        return 1;
    }

    typedef int (*AddFunc)(int, int);
    AddFunc add = (AddFunc)GetProcAddress(hDll, "add");

    if (add) {
        printf("add(3, 5) = %d\n", add(3, 5));
    }

    FreeLibrary(hDll);
    return 0;
}
```

## 六、符号可见性控制

### 6.1 默认导出所有符号

```bash
gcc -shared -fPIC -o libmyutils.so *.o  # 默认所有非 static 符号都导出
```

### 6.2 只导出指定符号

```c
// GCC 属性控制可见性
__attribute__((visibility("default"))) int public_func(void);
__attribute__((visibility("hidden")))  void internal_func(void);

// 或者用编译选项：-fvisibility=hidden
// 然后显式标记要导出的符号
```

```bash
# 编译时默认隐藏所有符号
gcc -shared -fPIC -fvisibility=hidden -o libmyutils.so *.c -I include
```

## 七、静态库 vs 动态库

| 特性           | 静态库 (.a)                | 动态库 (.so/.dll)           |
|---------------|--------------------------|---------------------------|
| 链接时机       | 编译时（静态链接）          | 运行时（动态链接）          |
| 文件体积       | 可执行文件较大              | 可执行文件较小              |
| 内存占用       | 每个进程独立副本            | 多进程共享                  |
| 更新           | 需要重新编译                | 替换 .so 即可              |
| 部署           | 简单（自包含）              | 需要附带 .so 文件          |
| 启动速度       | 更快                       | 略慢（需要加载）            |
| 版本管理       | 无版本问题                  | 可能有版本兼容问题          |

## 八、要点总结

> **关键点：**
> 1. 动态库在**运行时**被加载，多个进程共享同一份库代码。
> 2. 编译动态库需要 `-fPIC`（位置无关代码）和 `-shared` 选项。
> 3. 运行时通过 `LD_LIBRARY_PATH`、`ldconfig` 或 `-rpath` 找到库文件。
> 4. `dlopen/dlsym/dlclose` 实现运行时动态加载和符号查找。
> 5. Windows 上使用 `LoadLibrary/GetProcAddress/FreeLibrary`。
> 6. 使用符号可见性控制只导出必要的接口。
> 7. 动态库适合大型项目、需要热更新的场景；静态库适合简单部署。
