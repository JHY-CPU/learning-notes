# 28 - 函数调试技巧

## 一、函数调用栈追踪

### 1.1 使用 backtrace（Linux/GCC）

```c
#include <stdio.h>
#include <stdlib.h>
#include <execinfo.h>

#define MAX_FRAMES 64

void print_backtrace(void) {
    void *buffer[MAX_FRAMES];
    int nframes = backtrace(buffer, MAX_FRAMES);
    char **symbols = backtrace_symbols(buffer, nframes);

    printf("调用栈 (%d 帧):\n", nframes);
    for (int i = 0; i < nframes; i++) {
        printf("  [%d] %s\n", i, symbols[i]);
    }

    free(symbols);  // 必须释放
}

void func_c(void) { print_backtrace(); }
void func_b(void) { func_c(); }
void func_a(void) { func_b(); }

int main(void) {
    func_a();
    return 0;
}
```

### 1.2 使用 GCC 的 -rdynamic 选项

```bash
# 编译时加上 -rdynamic，以便 backtrace 显示函数名
gcc -rdynamic -o myapp main.c -lexecinfo
```

## 二、GDB 函数调试

### 2.1 编译时加入调试信息

```bash
gcc -g -Wall -o myapp main.c utils.c
# -g 生成调试信息，GDB 才能显示源码和变量
```

### 2.2 常用 GDB 命令

```bash
# 启动 GDB
gdb ./myapp

# 运行程序
(gdb) run
(gdb) run arg1 arg2        # 带参数运行

# 断点
(gdb) break main            # 在 main 函数设断点
(gdb) break utils.c:42      # 在文件第 42 行设断点
(gdb) break add             # 在 add 函数入口设断点
(gdb) break helper if n>10  # 条件断点

# 执行控制
(gdb) step (s)              # 单步执行（进入函数）
(gdb) next (n)              # 单步执行（跳过函数）
(gdb) finish                # 执行到当前函数返回
(gdb) continue (c)          # 继续运行到下一个断点

# 查看信息
(gdb) backtrace (bt)        # 显示调用栈
(gdb) info locals           # 显示局部变量
(gdb) info args             # 显示函数参数
(gdb) print var             # 打印变量值
(gdb) print *ptr            # 打印指针指向的值
(gdb) print arr[0]@10       # 打印数组前 10 个元素

# 帧操作
(gdb) frame 2               # 切换到第 2 帧
(gdb) up                    # 上移一帧
(gdb) down                  # 下移一帧

# 监视
(gdb) watch var             # 当 var 被修改时中断
(gdb) display var           # 每次暂停时自动显示 var
```

### 2.3 GDB 调试函数的典型流程

```bash
# 1. 启动并设置断点
(gdb) break compute_sum
(gdb) run

# 2. 程序在 compute_sum 入口停下
(gdb) info args             # 查看参数
(gdb) info locals           # 查看局部变量

# 3. 单步执行
(gdb) next                  # 观察每步的变量变化
(gdb) print result          # 打印中间结果

# 4. 继续到下一个断点
(gdb) continue

# 5. 查看调用栈
(gdb) backtrace
```

## 三、常见函数错误及调试

### 3.1 段错误（Segmentation Fault）

```c
// 常见原因
void func(void) {
    int *p = NULL;
    *p = 42;              // 解引用空指针 → 段错误

    char buf[10];
    buf[100] = 'A';       // 数组越界 → 未定义行为（可能段错误）

    int *q;
    *q = 10;              // 未初始化的野指针 → 段错误
}
```

**调试方法：**
```bash
# GDB 定位段错误
(gdb) run
Program received signal SIGSEGV, Segmentation fault.
(gdb) backtrace           # 查看出错位置
(gdb) print p             # 查看指针值是否为 NULL
```

### 3.2 栈溢出

```c
// 无限递归
int factorial(int n) {
    return n * factorial(n - 1);  // 没有基本情况！
}

// GDB 调试
(gdb) run
Program received signal SIGSEGV
(gdb) backtrace           # 会看到大量重复的帧
```

### 3.3 返回局部变量的指针

```c
// 错误：返回局部数组的指针
char* get_string(void) {
    char buf[] = "hello";  // 局部数组，函数返回后被销毁
    return buf;            // 危险！返回悬垂指针
}

// 正确的做法
char* get_string_safe(void) {
    static char buf[] = "hello";  // 静态变量
    return buf;
}
```

### 3.4 内存泄漏

```c
// 泄漏示例
void leaky_func(void) {
    char *p = malloc(100);
    // 忘记 free(p)！
}

// 使用 Valgrind 检测（Linux）
// valgrind --leak-check=full ./myapp
```

## 五、调试辅助技巧

### 5.1 断言宏

```c
#include <assert.h>

int divide(int a, int b) {
    assert(b != 0);       // 如果 b == 0，程序终止并显示位置
    return a / b;
}

// 编译时禁用断言
// gcc -DNDEBUG -o myapp main.c   （NDEBUG 定义后 assert 不起作用）
```

### 5.2 调试打印宏

```c
#include <stdio.h>

#ifdef DEBUG
    #define DBG(fmt, ...) \
        fprintf(stderr, "[DEBUG] %s:%d " fmt "\n", \
                __FILE__, __LINE__, ##__VA_ARGS__)
#else
    #define DBG(fmt, ...) ((void)0)
#endif

void process(int x) {
    DBG("进入 process, x = %d", x);
    // ...
    DBG("离开 process, result = %d", result);
}
```

```bash
# 编译时启用调试
gcc -DDEBUG -o myapp main.c
```

### 5.3 函数耗时测量

```c
#include <stdio.h>
#include <time.h>

#define TIME_CALL(func_call, label) do { \
    clock_t _start = clock(); \
    func_call; \
    clock_t _end = clock(); \
    double _elapsed = (double)(_end - _start) / CLOCKS_PER_SEC * 1000; \
    printf("%s 耗时: %.2f ms\n", label, _elapsed); \
} while (0)

// 使用
TIME_CALL(sort_array(arr, n), "排序");
```

### 5.4 GDB 的 pretty-print 结构体

```c
typedef struct {
    char name[32];
    int age;
    double score;
} Student;

// 在 GDB 中
(gdb) print student       # GDB 会显示结构体各字段的值
(gdb) print student.name  # 只打印 name 字段
```

## 六、调试检查清单

```
函数调试检查清单：

[ ] 参数是否为 NULL？
[ ] 数组索引是否越界？
[ ] 循环条件是否正确（< vs <=）？
[ ] 是否有未初始化的变量？
[ ] malloc 是否检查了返回值？
[ ] free 之后是否使用了指针？
[ ] 是否返回了局部变量的指针？
[ ] 递归是否有正确的基本情况？
[ ] 字符串是否以 '\0' 结尾？
[ ] 是否有整数溢出的风险？
```

## 七、要点总结

> **关键点：**
> 1. 使用 `gcc -g` 编译，生成调试信息供 GDB 使用。
> 2. GDB 的 `backtrace`、`break`、`step`、`next`、`print` 是核心调试命令。
> 3. 段错误通常由空指针、野指针、数组越界引起。
> 4. `valgrind` 可以检测内存泄漏和非法内存访问。
> 5. 使用 `assert` 守卫函数的前置条件。
> 6. 调试打印宏结合 `__FILE__` 和 `__LINE__` 快速定位问题。
> 7. 函数返回后检查返回值，及时发现错误。
