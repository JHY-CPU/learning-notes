# 18 - 函数调用约定

## 一、什么是调用约定

**调用约定（Calling Convention）** 定义了函数调用时，调用者（Caller）和被调用者（Callee）之间如何协作的规则，包括：

1. 参数如何传递（栈、寄存器）
2. 参数的压栈顺序
3. 谁负责清理栈（调用者还是被调用者）
4. 函数名的修饰方式
5. 返回值如何传递

## 二、常见调用约定

### 2.1 cdecl（C Declaration）

C 语言**默认**的调用约定。

```c
// 默认就是 cdecl
int add(int a, int b) {
    return a + b;
}

// 显式指定（GCC）
int __attribute__((cdecl)) add(int a, int b);
```

**特点：**

| 特性           | cdecl                          |
|---------------|--------------------------------|
| 参数传递       | 栈                             |
| 压栈顺序       | **从右到左**                    |
| 栈清理         | **调用者（Caller）清理**        |
| 名字修饰       | 函数名前加下划线 `_add`         |
| 可变参数       | **支持**                       |

```c
// 调用者清理栈
int main() {
    int r = add(1, 2);
    // 汇编：
    // push 2        ; 从右到左压栈
    // push 1
    // call _add
    // add esp, 8    ; 调用者清理栈（C 编译器自动生成）
    return 0;
}
```

**为什么 cdecl 支持可变参数：**

```c
// 因为调用者清理栈，调用者知道传了几个参数
printf("%d %d %d\n", 1, 2, 3);
// push 3
// push 2
// push 1
// push format_str
// call _printf
// add esp, 16    ; 调用者知道传了 4 个参数
```

### 2.2 stdcall（Standard Call）

Windows API 常用的调用约定。

```c
// Windows API 使用 stdcall
// __stdcall 是 MSVC 关键字
int __stdcall MyFunction(int a, int b);
```

**特点：**

| 特性           | stdcall                        |
|---------------|--------------------------------|
| 参数传递       | 栈                             |
| 压栈顺序       | **从右到左**                    |
| 栈清理         | **被调用者（Callee）清理**      |
| 名字修饰       | `_MyFunction@8`（8是参数字节数）|
| 可变参数       | **不支持**                     |

```asm
; stdcall 的函数自己清理栈
_MyFunction@8:
    push ebp
    mov ebp, esp
    ; ... 函数体 ...
    pop ebp
    ret 8          ; 返回时清理 8 字节参数
```

### 2.3 fastcall

部分参数通过寄存器传递，速度快。

```c
// GCC
int __attribute__((fastcall)) func(int a, int b, int c);
// MSVC
int __fastcall func(int a, int b, int c);
```

**特点：**

| 特性           | fastcall                       |
|---------------|--------------------------------|
| 参数传递       | 前两个整数参数用 **ECX、EDX**  |
| 其余参数       | 栈（从右到左）                  |
| 栈清理         | 被调用者清理                   |
| 速度           | 比 cdecl/stdcall 快            |

### 2.4 thiscall（C++ 用）

C++ 成员函数的调用约定：

```cpp
// C++ 中
class MyClass {
public:
    int method(int x);  // this 指针通过 ECX 传递
};
```

## 三、调用约定对比

| 特性           | cdecl        | stdcall      | fastcall      |
|---------------|-------------|-------------|---------------|
| 压栈顺序       | 从右到左     | 从右到左      | 从右到左        |
| 栈清理方       | 调用者       | 被调用者      | 被调用者        |
| 支持可变参数   | **是**       | 否           | 否             |
| 速度           | 慢           | 较快         | 最快           |
| 平台           | 所有         | Windows      | 所有           |
| 典型用途       | C 默认       | Win32 API    | 性能关键代码    |

## 四、在 C 代码中指定调用约定

### 4.1 GCC（Linux）

```c
// GCC 使用 __attribute__ 指定
void __attribute__((cdecl)) func1(int a);
void __attribute__((stdcall)) func2(int a);
void __attribute__((fastcall)) func3(int a);
```

### 4.2 MSVC（Windows）

```c
// MSVC 使用关键字
void __cdecl func1(int a);
void __stdcall func2(int a);
void __fastcall func3(int a);
```

### 4.3 跨平台宏

```c
#ifdef _MSC_VER
    #define CDECL __cdecl
    #define STDCALL __stdcall
#else
    #define CDECL __attribute__((cdecl))
    #define STDCALL __attribute__((stdcall))
#endif

int CDECL add(int a, int b);
```

## 五、调用约定不匹配的危险

```c
// file1.c —— 使用默认 cdecl
int add(int a, int b) { return a + b; }

// file2.c —— 声明为 stdcall
int __stdcall add(int a, int b);  // 调用约定不匹配！

int main() {
    int r = add(3, 5);  // 未定义行为！
    // file1 的 add 返回时不会清理栈（cdecl）
    // 但调用者假设是 stdcall，不会清理栈
    // 结果：栈不平衡，程序崩溃
    return 0;
}
```

## 六、返回值的传递

### 6.1 整数/指针返回值

通过 **EAX** 寄存器返回（32位）或 **RAX**（64位）。

### 6.2 浮点数返回值

通过 **ST(0)** 寄存器返回（x87 FPU）或 **XMM0**（SSE）。

### 6.3 大型结构体返回

```c
typedef struct {
    int data[100];  // 400 字节
} BigStruct;

BigStruct create(void) {
    BigStruct s;
    // ... 初始化
    return s;  // 调用者在栈上分配空间，传隐藏指针给函数
}
```

## 七、System V AMD64 ABI（64 位 Linux）

在 64 位 Linux 上，前 6 个整数参数通过寄存器传递：

| 参数   | 寄存器 |
|-------|--------|
| 第1个  | RDI    |
| 第2个  | RSI    |
| 第3个  | RDX    |
| 第4个  | RCX    |
| 第5个  | R8     |
| 第6个  | R9     |
| 更多   | 栈     |

前 8 个浮点参数通过 **XMM0-XMM7** 传递。

## 八、要点总结

> **关键点：**
> 1. C 语言默认使用 **cdecl** 约定：参数从右到左压栈，调用者清理栈。
> 2. **stdcall** 由被调用者清理栈，不支持可变参数，Windows API 常用。
> 3. **fastcall** 通过寄存器传递部分参数，速度最快。
> 4. 调用约定不匹配会导致栈不平衡，程序崩溃。
> 5. 返回值通过 EAX/RAX（整数）或 ST(0)/XMM0（浮点）传递。
> 6. 64 位 Linux 使用 System V ABI，前 6 个整数参数通过寄存器传递。
> 7. 跨平台开发时用宏统一处理调用约定差异。
