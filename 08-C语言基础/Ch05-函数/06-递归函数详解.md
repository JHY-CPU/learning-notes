# 06 - 递归函数详解

## 一、什么是递归

**递归（Recursion）** 是指函数直接或间接调用自身的一种编程技巧。递归是将复杂问题分解为与原问题结构相同但规模更小的子问题来求解的方法。

```c
// 直接递归：函数直接调用自己
void func() {
    // ...
    func();  // 调用自身
    // ...
}

// 间接递归：函数 A 调用函数 B，函数 B 再调用函数 A
void funcA() {
    // ...
    funcB();
}
void funcB() {
    // ...
    funcA();
}
```

## 二、递归三要素

任何正确的递归函数都必须满足以下三个条件：

### 2.1 要素一：基本情况（递归出口）

必须有一个或多个**不再递归**的分支，否则函数会无限调用自己。

```c
int factorial(int n) {
    if (n <= 1) return 1;  // 基本情况：不再递归
    // ...
}
```

### 2.2 要素二：递归情况

函数通过调用自身来解决规模更小的子问题。

```c
int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);  // 递归调用，规模减小
}
```

### 2.3 要素三：收敛性

每次递归调用必须使问题规模趋近于基本情况，否则会导致无限递归。

```c
// 错误示例：n 永远不会到达基本情况
int bad_factorial(int n) {
    if (n <= 1) return 1;
    return n * bad_factorial(n + 1);  // n 越来越大，永远到不了 n <= 1
}
```

## 三、递归的执行过程——调用栈

### 3.1 阶乘函数的递归展开

```c
int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);
}
```

调用 `factorial(5)` 时的**调用栈**：

```
factorial(5)
  = 5 * factorial(4)
             = 4 * factorial(3)
                        = 3 * factorial(2)
                                   = 2 * factorial(1)
                                              = 1 (基本情况，开始返回)
                                   = 2 * 1 = 2
                        = 3 * 2 = 6
             = 4 * 6 = 24
  = 5 * 24 = 120
```

### 3.2 调用栈的内存布局

```
函数调用时，每次调用在栈上创建一个新帧：

  栈帧4: factorial(1)  n=1  返回值=1     ← 最先入栈，最先返回
  栈帧3: factorial(2)  n=2  等待结果
  栈帧2: factorial(3)  n=3  等待结果
  栈帧1: factorial(4)  n=4  等待结果
  栈帧0: factorial(5)  n=5  等待结果     ← 最后入栈

返回过程：
  栈帧4 返回 1，被销毁
  栈帧3 计算 2*1=2，返回，被销毁
  栈帧2 计算 3*2=6，返回，被销毁
  栈帧1 计算 4*6=24，返回，被销毁
  栈帧0 计算 5*24=120，返回，被销毁
```

### 3.3 栈溢出（Stack Overflow）

当递归深度过大时，栈空间耗尽，程序崩溃：

```c
int main() {
    // 递归过深会导致栈溢出
    factorial(100000);  // 大概率 Segmentation Fault
    return 0;
}
```

## 四、递归的两种思路

### 4.1 自顶向下（Top-Down）

从大问题出发，逐步分解为小问题：

```c
// 自顶向下：先处理当前节点，再递归处理子问题
void print_reverse(int n) {
    if (n <= 0) return;
    printf("%d ", n);         // 先打印当前值
    print_reverse(n - 1);     // 再递归
}
// 输出: 5 4 3 2 1
```

### 4.2 自底向上（Bottom-Up）

先递归处理小问题，再组合结果：

```c
// 自底向上：先递归，再处理当前节点
void print_forward(int n) {
    if (n <= 0) return;
    print_forward(n - 1);     // 先递归
    printf("%d ", n);         // 再打印
}
// 输出: 1 2 3 4 5
```

## 五、尾递归

### 5.1 什么是尾递归

**尾递归（Tail Recursion）** 是递归调用是函数中**最后一个操作**的递归形式。尾递归可以被编译器优化为循环，避免栈溢出。

```c
// 尾递归版本的阶乘
int factorial_tail(int n, int acc) {
    if (n <= 1) return acc;
    return factorial_tail(n - 1, n * acc);  // 递归调用是最后的操作
}

int factorial(int n) {
    return factorial_tail(n, 1);
}
```

### 5.2 普通递归 vs 尾递归

```c
// 普通递归：递归返回后还要做乘法（不是尾递归）
int fib_normal(int n) {
    if (n <= 1) return n;
    return fib_normal(n-1) + fib_normal(n-2);  // 返回后还要做加法
}

// 尾递归版本
int fib_tail(int n, int a, int b) {
    if (n == 0) return a;
    if (n == 1) return b;
    return fib_tail(n - 1, b, a + b);  // 最后操作就是递归调用
}
```

### 5.3 尾递归优化原理

```c
// 尾递归函数
int sum_tail(int n, int acc) {
    if (n <= 0) return acc;
    return sum_tail(n - 1, acc + n);
}

// 编译器可以将其优化为等价的循环：
int sum_loop(int n) {
    int acc = 0;
    while (n > 0) {
        acc += n;
        n--;
    }
    return acc;
}
```

> **注意：** C 标准不要求编译器进行尾递归优化，但 GCC/Clang 在开启优化时通常会执行。使用 `-O2` 优化级别可以观察到尾递归被消除。

## 六、递归与循环的对比

| 特性       | 递归                          | 循环                          |
|-----------|------------------------------|------------------------------|
| 代码简洁性 | 对某些问题（树、分治）更简洁     | 对线性迭代更直接               |
| 空间复杂度 | O(n) 栈空间                   | O(1)                         |
| 栈溢出风险 | 深度过大时会栈溢出              | 无此风险                      |
| 性能       | 函数调用开销较大                | 通常更快                      |
| 适用场景   | 树结构、分治、回溯             | 简单迭代、大规模数据处理        |

## 七、何时使用递归

**适合使用递归的场景：**
- 数据结构本身就是递归定义的（树、链表、图）
- 问题可以自然地分解为相同类型的子问题（分治算法）
- 回溯搜索（全排列、迷宫问题）
- 代码清晰度远比性能重要时

**不适合使用递归的场景：**
- 深度很大，可能导致栈溢出
- 性能要求极高，递归开销不可接受
- 可以用简单的循环代替

## 八、要点总结

> **关键点：**
> 1. 递归必须有**基本情况**（出口），否则会导致无限递归。
> 2. 每次递归必须使问题规模**趋近**基本情况。
> 3. 递归使用**调用栈**存储每一层的局部变量和返回地址。
> 4. 递归深度过大时会导致**栈溢出**。
> 5. **尾递归**是递归调用作为最后操作的递归形式，可被优化为循环。
> 6. 递归适合处理树、分治、回溯等问题；循环适合简单的线性迭代。
> 7. 当递归深度不确定时，考虑用显式栈（数组模拟）替代递归。
