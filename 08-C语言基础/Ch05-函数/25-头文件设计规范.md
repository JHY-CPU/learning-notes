# 25 - 头文件设计规范

## 一、头文件守卫

### 1.1 传统方式：#ifndef

```c
// mymodule.h
#ifndef MYMODULE_H
#define MYMODULE_H

// 头文件内容...

#endif /* MYMODULE_H */
```

### 1.2 现代方式：#pragma once

```c
// mymodule.h
#pragma once

// 头文件内容...
```

| 特性           | #ifndef                     | #pragma once               |
|---------------|-----------------------------|---------------------------|
| 标准           | C 标准                      | 非标准，但广泛支持         |
| 可移植性       | 所有编译器                  | 绝大多数现代编译器         |
| 性能           | 每次都要检查宏              | 编译器可优化              |
| 命名冲突风险   | 宏名可能冲突                | 无此问题                  |

> **建议：** 优先使用 `#pragma once`，需要严格可移植性时用 `#ifndef`。

## 二、包含顺序

### 2.1 推荐的包含顺序

```c
// mymodule.c

// 1. 对应的头文件（确保头文件本身是自包含的）
#include "mymodule.h"

// 2. C 标准库
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

// 3. 系统/平台相关头文件
#include <unistd.h>
#include <sys/types.h>

// 4. 第三方库
#include <openssl/ssl.h>

// 5. 项目其他头文件
#include "utils.h"
#include "config.h"
```

### 2.2 为什么要将对应头文件放在最前

```c
// mymodule.c
#include "mymodule.h"  // 确保 mymodule.h 本身是自包含的
// 如果 mymodule.h 缺少必要的 #include，这里会立即报错
// 而不是在其他文件包含 mymodule.h 时才报错
```

## 三、头文件自包含

### 3.1 自包含的含义

头文件应该可以**独立编译**，即不需要在包含它之前先包含其他头文件：

```c
// 好的设计：自包含头文件
// utils.h
#ifndef UTILS_H
#define UTILS_H

#include <stddef.h>    // 自己包含所需的头文件

void* safe_malloc(size_t size);
void safe_free(void **ptr);

#endif
```

```c
// 差的设计：依赖包含顺序
// utils.h
#ifndef UTILS_H
#define UTILS_H

// 没有 #include <stddef.h>
void* safe_malloc(size_t size);  // size_t 未定义！

#endif
```

### 3.2 测试自包含性

```c
// test_utils.c —— 只包含一个头文件
#include "utils.h"

// 如果编译通过，说明 utils.h 是自包含的
```

```bash
gcc -c test_utils.c  # 测试头文件是否自包含
```

## 四、最小依赖原则

### 4.1 只包含必要的头文件

```c
// 不好：包含不需要的头文件
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <math.h>
#include <time.h>
// 实际只用到了 stdlib.h 中的 malloc/free
```

```c
// 好：只包含需要的
#include <stdlib.h>  // 只需要 malloc/free
```

### 4.2 使用前向声明减少依赖

```c
// 如果头文件只需要指针，不需要完整定义
// config.h
#ifndef CONFIG_H
#define CONFIG_H

// 前向声明——不需要 #include "database.h"
typedef struct Database Database;

typedef struct {
    int max_connections;
    Database *db;  // 只需要指针
} Config;

Config* config_load(const char *filename);
void config_free(Config *cfg);

#endif
```

## 五、循环包含的处理

### 5.1 问题示例

```c
// a.h
#include "b.h"      // 包含 b.h
void func_a(B *b);

// b.h
#include "a.h"      // 包含 a.h → 循环！
void func_b(A *a);
```

### 5.2 解决方案：前向声明

```c
// a.h
#ifndef A_H
#define A_H

// 不包含 b.h，用前向声明
typedef struct B B;

typedef struct A {
    int value;
} A;

void func_a(B *b);

#endif
```

```c
// b.h
#ifndef B_H
#define B_H

// 不包含 a.h，用前向声明
typedef struct A A;

typedef struct B {
    int value;
} B;

void func_b(A *a);

#endif
```

```c
// a.c
#include "a.h"
#include "b.h"   // 在 .c 文件中包含完整定义

void func_a(B *b) {
    // 可以访问 B 的完整定义
    printf("%d\n", b->value);
}
```

## 六、头文件中的内容规范

### 6.1 应该在头文件中的

```c
#ifndef MODULE_H
#define MODULE_H

// 1. 函数声明
int module_init(void);
void module_cleanup(void);

// 2. 类型定义
typedef struct Module Module;

// 3. 宏定义
#define MODULE_VERSION 1

// 4. extern 变量声明
extern int g_module_state;

// 5. enum 定义
typedef enum {
    STATE_IDLE,
    STATE_RUNNING,
    STATE_ERROR
} ModuleState;

// 6. static inline 函数
static inline int module_is_ready(void) {
    return g_module_state == STATE_RUNNING;
}

#endif
```

### 6.2 不应在头文件中的

```c
// 1. 函数定义（除非 static inline）
void func(void) { }  // 错误！多重定义

// 2. 全局变量定义
int global = 0;      // 错误！多重定义

// 3. static 变量
static int count;    // 每个 .c 文件都会有自己的副本

// 4. 使用 using namespace（C++，C 不适用）

// 5. #include 本模块的私有头文件
#include "internal.h"  // 不应暴露给使用者
```

## 七、头文件命名约定

```c
// 模块名作为前缀
math_utils.h      // 数学工具
string_utils.h    // 字符串工具
linked_list.h     // 链表
hash_map.h        // 哈希表

// 内部头文件用 _internal 后缀
linked_list_internal.h  // 不应该被外部包含
```

## 八、要点总结

> **关键点：**
> 1. 使用头文件守卫（`#ifndef` 或 `#pragma once`）防止重复包含。
> 2. 包含顺序：对应头文件、标准库、系统库、第三方库、项目头文件。
> 3. 头文件应该是**自包含的**，不依赖包含顺序。
> 4. 遵循**最小依赖原则**，只包含必要的头文件。
> 5. 循环依赖通过**前向声明**解决，不在头文件中互相包含。
> 6. 头文件只放声明，不放定义（`static inline` 除外）。
> 7. 测试头文件自包含性：单独创建一个 `.c` 文件只包含该头文件并编译。
