# 14 - 函数的作用域与链接

## 一、C 语言中的标识符作用域

C 语言中有四种作用域：

| 作用域类型     | 说明                                      | 示例                       |
|--------------|------------------------------------------|---------------------------|
| 文件作用域     | 在所有函数之外声明，整个文件可见             | 全局变量、全局函数          |
| 块作用域       | 在 `{}` 内声明，从声明处到块结束可见        | 局部变量、函数参数          |
| 函数原型作用域 | 函数声明中的参数名                          | `int add(int a, int b)`   |
| 函数作用域     | 仅 `goto` 标签使用                        | `label:`                  |

## 二、函数的链接属性

### 2.1 链接的三种类型

| 链接类型   | 说明                              | 关键字      |
|-----------|----------------------------------|------------|
| 外部链接   | 整个程序的所有文件都能访问         | 默认（无修饰）|
| 内部链接   | 只有定义它的源文件能访问           | `static`   |
| 无链接     | 只在当前作用域可见                | 局部变量     |

### 2.2 外部链接（External Linkage）

函数默认具有外部链接：

```c
// math.c
int add(int a, int b) {         // 外部链接
    return a + b;
}
```

```c
// main.c
extern int add(int, int);       // 声明外部函数

int main() {
    int r = add(3, 5);          // 调用 math.c 中定义的 add
    return 0;
}
```

编译：
```bash
gcc -o program main.c math.c
```

### 2.3 内部链接（Internal Linkage）

使用 `static` 限定：

```c
// utils.c
static void internal_func(void) {    // 内部链接
    printf("仅本文件可见\n");
}

void public_func(void) {             // 外部链接
    internal_func();                  // 可以调用
}
```

```c
// other.c
// 无法调用 internal_func，即使声明了也无法链接
```

## 三、extern 关键字

### 3.1 声明外部函数

```c
// main.c
// 声明其他文件中定义的函数
extern int add(int a, int b);
extern int multiply(int a, int b);

// 函数声明中的 extern 可以省略
int subtract(int a, int b);   // 等价于 extern int subtract(int, int);
```

### 3.2 声明外部变量

```c
// config.c
int global_value = 42;               // 定义

// main.c
extern int global_value;              // 声明（不是定义）
printf("%d\n", global_value);         // 42
```

### 3.3 extern 变量的常见错误

```c
// header.h
extern int count;         // 声明——OK，可以在多个文件中包含

// file1.c
#include "header.h"
int count = 0;            // 定义——只能有一个

// file2.c
#include "header.h"
count++;                  // 使用——通过 extern 声明访问
```

> **关键区分：** `extern int count;` 是**声明**，`int count = 0;` 是**定义**。声明可以有多份，定义只能有一份。

## 四、函数声明与定义的位置

### 4.1 声明在头文件中

```c
// utils.h
#ifndef UTILS_H
#define UTILS_H

void print_array(int arr[], int n);
int find_max(int arr[], int n);
int find_min(int arr[], int n);

#endif
```

```c
// utils.c
#include "utils.h"
#include <stdio.h>

void print_array(int arr[], int n) { /* ... */ }
int find_max(int arr[], int n) { /* ... */ }
int find_min(int arr[], int n) { /* ... */ }
```

```c
// main.c
#include "utils.h"  // 包含声明

int main() {
    int arr[] = {3, 1, 4, 1, 5};
    print_array(arr, 5);  // 调用 utils.c 中定义的函数
    return 0;
}
```

### 4.2 多文件项目的典型结构

```
project/
  include/
    utils.h        -- 函数声明
    types.h        -- 类型定义
  src/
    utils.c        -- 函数实现，包含 utils.h
    main.c         -- 程序入口，包含 utils.h
  Makefile
```

## 五、多重定义与 One Definition Rule

### 5.1 函数只能定义一次

```c
// file1.c
int add(int a, int b) { return a + b; }

// file2.c
int add(int a, int b) { return a + b; }  // 链接错误：重复定义！
```

### 5.2 inline 函数的特殊规则

```c
// 在所有使用该内联函数的文件中，定义必须完全相同
// header.h
inline int square(int x) { return x * x; }
```

### 5.3 static 函数不受限制

```c
// file1.c
static int helper(int x) { return x * x; }

// file2.c
static int helper(int x) { return x * x; }  // OK！不同文件的 static 函数独立
```

## 六、函数作用域的实际案例

### 6.1 事件驱动系统

```c
// event_system.c
#include <stdio.h>

// 文件级 static 变量
static int event_count = 0;
static void (*handlers[10])(int);

// 文件级 static 函数
static void log_event(int id) {
    printf("事件 #%d 被触发\n", id);
}

// 公开接口
void event_register(int id, void (*handler)(int)) {
    if (id >= 0 && id < 10)
        handlers[id] = handler;
}

void event_trigger(int id) {
    event_count++;
    log_event(id);
    if (id >= 0 && id < 10 && handlers[id])
        handlers[id](id);
}

int event_get_count(void) {
    return event_count;
}
```

## 七、static 与 extern 的对照

```c
// 在文件作用域中：

int x;                  // 定义，外部链接
static int y;           // 定义，内部链接
extern int z;           // 声明，外部链接

void f(void) { }        // 定义，外部链接
static void g(void) { } // 定义，内部链接
```

## 八、要点总结

> **关键点：**
> 1. 函数默认具有**外部链接**，整个程序都能访问。
> 2. `static` 使函数具有**内部链接**，仅本文件可见。
> 3. `extern` 用于声明其他文件中定义的函数或变量。
> 4. 函数声明放在头文件中，定义放在源文件中。
> 5. 同一个函数在程序中**只能定义一次**（`static` 和 `inline` 除外）。
> 6. `extern` 声明可以出现多次，但非 `static` 定义只能有一次。
> 7. 理解链接属性是模块化编程和避免链接错误的基础。
