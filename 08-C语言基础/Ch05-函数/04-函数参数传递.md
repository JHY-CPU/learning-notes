# 04 - 函数参数传递

## 一、值传递（Pass by Value）

### 1.1 值传递的概念

C 语言中**所有参数传递都是值传递**。调用函数时，实参的值被**复制**一份传给形参。函数内部对形参的修改**不会影响**实参。

```c
#include <stdio.h>

void try_to_change(int x) {
    x = 100;            // 修改的是 x 的副本
    printf("函数内: x = %d\n", x);
}

int main() {
    int a = 5;
    printf("调用前: a = %d\n", a);
    try_to_change(a);
    printf("调用后: a = %d\n", a);  // a 仍然是 5！
    return 0;
}
```

**输出：**
```
调用前: a = 5
函数内: x = 100
调用后: a = 5
```

### 1.2 内存模型理解值传递

```
调用 try_to_change(a) 时的内存布局：

  main 的栈帧            try_to_change 的栈帧
  +---------+            +---------+
  | a = 5   |  --复制-->  | x = 5   |   (x 是 a 的副本)
  +---------+            +---------+

函数内执行 x = 100 后：

  main 的栈帧            try_to_change 的栈帧
  +---------+            +----------+
  | a = 5   |  (不变)    | x = 100  |  (只改了副本)
  +---------+            +----------+
```

### 1.3 经典错误：试图用值传递交换两数

```c
// 这个函数不能交换 main 中的 a 和 b！
void swap_wrong(int x, int y) {
    int temp = x;
    x = y;
    y = temp;
    // 函数返回后，x 和 y 被销毁，main 中的 a 和 b 未被修改
}

int main() {
    int a = 10, b = 20;
    swap_wrong(a, b);
    printf("a = %d, b = %d\n", a, b);  // 仍然是 10, 20
    return 0;
}
```

## 二、传址——通过指针实现"引用"效果

### 2.1 传递指针（地址）

虽然 C 语言只有值传递，但我们可以传递**变量的地址**（指针），让函数通过指针修改原始数据：

```c
void swap_correct(int *px, int *py) {
    int temp = *px;
    *px = *py;
    *py = temp;
}

int main() {
    int a = 10, b = 20;
    swap_correct(&a, &b);       // 传递 a 和 b 的地址
    printf("a = %d, b = %d\n", a, b);  // 输出: 20, 10
    return 0;
}
```

### 2.2 内存模型理解传址

```
调用 swap_correct(&a, &b) 时的内存布局：

  main 的栈帧              swap_correct 的栈帧
  +---------+              +------------+
  | a = 10  | <--+         | px = 0x... | --指向 a 的地址
  +---------+    |         +------------+
  | b = 20  | <--|--+      | py = 0x... | --指向 b 的地址
  +---------+    |  |      +------------+
                 |  |      | temp = 10  |
                 +--|------| *px = 20   |  通过指针修改了 a
                    +------| *py = 10   |  通过指针修改了 b
                           +------------+
```

### 2.3 指针传递的本质

**指针传递仍然是值传递！** 只不过传递的"值"是地址：

```c
void modify_pointer(int *p) {
    // p 是指针的副本，但 *p 和原始指针指向同一块内存
    *p = 999;    // 通过指针修改数据——成功

    int other = 888;
    p = &other;  // 修改指针本身——不影响外部
}

int main() {
    int a = 1;
    int *ptr = &a;
    modify_pointer(ptr);
    printf("a = %d\n", a);       // a = 999（通过指针修改成功）
    printf("*ptr = %d\n", *ptr); // *ptr = 999
    return 0;
}
```

## 三、数组作为函数参数

### 3.1 数组参数退化为指针

当数组作为函数参数时，实际传递的是**指向首元素的指针**，不是整个数组的副本：

```c
// 以下三种写法等价
void process(int arr[10]) { }    // 形式上写数组大小
void process(int arr[]) { }      // 省略大小
void process(int *arr) { }       // 直接写指针（最准确）

// 三种声明完全等价，sizeof(arr) 都是 sizeof(int*)
```

### 3.2 必须额外传递数组长度

由于数组参数退化为指针，函数内部无法获知数组长度：

```c
void print_array(int arr[], int n) {  // 必须传长度
    for (int i = 0; i < n; i++) {
        printf("%d ", arr[i]);
    }
    printf("\n");
}

int main() {
    int data[] = {1, 2, 3, 4, 5};
    int len = sizeof(data) / sizeof(data[0]);  // 在定义处计算长度
    print_array(data, len);
    return 0;
}
```

### 3.3 数组参数可以修改原始数据

因为传的是指针，函数内对数组元素的修改会影响原始数组：

```c
void fill_zero(int arr[], int n) {
    for (int i = 0; i < n; i++)
        arr[i] = 0;    // 修改会影响 main 中的数组
}

void double_all(int arr[], int n) {
    for (int i = 0; i < n; i++)
        arr[i] *= 2;
}
```

### 3.4 防止函数修改数组——const

```c
void print_array(const int arr[], int n) {
    // arr[i] = 0;  // 编译错误！const 防止修改
    for (int i = 0; i < n; i++)
        printf("%d ", arr[i]);
}
```

## 四、结构体作为函数参数

### 4.1 值传递结构体

```c
typedef struct {
    int x, y;
} Point;

// 值传递：会复制整个结构体（大型结构体代价高）
Point move(Point p, int dx, int dy) {
    p.x += dx;
    p.y += dy;
    return p;
}
```

### 4.2 指针传递结构体

```c
// 指针传递：只传递一个地址（高效）
void move_ptr(Point *p, int dx, int dy) {
    p->x += dx;
    p->y += dy;
}

int main() {
    Point pt = {10, 20};
    move_ptr(&pt, 5, -3);        // 传地址，高效
    printf("(%d, %d)\n", pt.x, pt.y);  // (15, 17)
    return 0;
}
```

### 4.3 const 指针——只读访问

```c
void print_point(const Point *p) {
    printf("(%d, %d)\n", p->x, p->y);
    // p->x = 0;  // 编译错误！
}
```

## 五、C 语言只支持值传递的本质

```c
// C++ 中有真正的引用传递：
// void swap_cpp(int &x, int &y) { int t = x; x = y; y = t; }

// C 语言模拟"引用"效果的方式：传指针
void swap_c(int *x, int *y) { int t = *x; *x = *y; *y = t; }
```

| 特性           | 值传递                | 指针传递（C中模拟引用）     |
|---------------|----------------------|--------------------------|
| 传递内容       | 值的副本              | 地址的副本                |
| 函数内修改形参 | 不影响实参            | 可以通过解引用修改实参     |
| 开销           | 大结构体复制开销大     | 始终只传递一个地址         |
| 安全性         | 更安全（不会意外修改） | 可能意外修改外部数据       |

## 六、要点总结

> **关键点：**
> 1. C 语言**只有值传递**，实参的值被复制给形参。
> 2. 通过传递指针（地址），函数可以修改外部变量——这是"传址"的技巧。
> 3. 数组作为参数时会退化为指针，必须额外传递数组长度。
> 4. 使用 `const` 修饰指针参数，可以防止函数意外修改数据。
> 5. 大型结构体应该用指针传递，避免复制开销。
> 6. 指针传递的本质**仍然是值传递**——传递的是地址的副本。
> 7. "传值"与"传址"不是 C 语言的语法特性，而是编程技巧的区分。
