# 20 - 函数的副作用

## 一、什么是副作用

**副作用（Side Effect）** 是指函数除了返回值之外，还对外部状态产生了影响。包括：

1. 修改全局变量
2. 修改静态局部变量
3. 修改指针参数指向的数据
4. 进行 I/O 操作（读写文件、屏幕输出）
5. 修改硬件状态

```c
// 有副作用的函数
int global_count = 0;           // 全局变量

void increment_counter(void) {
    global_count++;             // 副作用：修改全局变量
    printf("计数: %d\n", global_count);  // 副作用：I/O 操作
}
```

## 二、纯函数 vs 有副作用函数

### 2.1 纯函数（Pure Function）

**纯函数**的特点：
- 相同的输入**总是**产生相同的输出
- 不修改任何外部状态
- 不依赖任何可变的外部状态

```c
// 纯函数示例
int add(int a, int b) {
    return a + b;  // 无副作用
}

double circle_area(double radius) {
    return 3.14159 * radius * radius;  // 无副作用
}

int is_even(int n) {
    return n % 2 == 0;  // 无副作用
}

int max(int a, int b) {
    return a > b ? a : b;  // 无副作用
}
```

### 2.2 有副作用的函数

```c
// 修改全局变量
int total = 0;
void add_to_total(int x) {
    total += x;  // 副作用
}

// 修改指针参数
void set_zero(int *arr, int n) {
    for (int i = 0; i < n; i++)
        arr[i] = 0;  // 副作用：修改了外部数据
}

// I/O 操作
void log_message(const char *msg) {
    fprintf(stderr, "%s\n", msg);  // 副作用：输出到屏幕
}

// 修改静态变量
int next_id(void) {
    static int id = 0;
    return ++id;  // 副作用：修改静态变量
}
```

## 三、全局变量的副作用

### 3.1 全局状态导致不可预测的行为

```c
int mode = 0;  // 全局模式

int compute(int x) {
    if (mode == 0) return x * 2;
    if (mode == 1) return x * x;
    return x + 1;
}

int main() {
    mode = 0;
    printf("%d\n", compute(5));  // 10

    mode = 1;
    printf("%d\n", compute(5));  // 25 —— 同样的输入，不同输出！

    // 函数的行为依赖于全局状态，难以预测和调试
    return 0;
}
```

### 3.2 隐藏的依赖关系

```c
// 看似简单，实际依赖全局配置
double calculate_tax(double amount) {
    return amount * g_tax_rate;  // g_tax_rate 是全局变量
    // 调用者不知道函数依赖了什么
}
```

### 3.3 线程安全问题

```c
// 全局变量在多线程环境下不安全
int counter = 0;

void increment(void) {
    counter++;  // 非原子操作！
    // 线程 A 和 B 可能同时读到旧值，导致最终结果不对
}
```

## 四、可重入函数

### 4.1 什么是可重入函数

**可重入函数（Reentrant Function）** 是可以在执行过程中被中断，然后再次安全调用的函数。可重入函数不使用全局或静态变量，不调用不可重入函数。

```c
// 可重入函数
int square(int x) {
    return x * x;  // 只使用参数和局部变量
}

// 不可重入函数
int next_id(void) {
    static int id = 0;  // 静态变量——不可重入
    return ++id;
}
```

### 4.2 信号处理中的可重入性

```c
// 信号处理函数中只能调用可重入函数
volatile sig_atomic_t flag = 0;

void handler(int sig) {
    flag = 1;  // OK：sig_atomic_t 的赋值是安全的
    // printf("收到信号\n");  // 危险！printf 不可重入
    // malloc(100);            // 危险！malloc 不可重入
}
```

### 4.3 可重入 vs 线程安全

| 特性       | 可重入                    | 线程安全                     |
|-----------|--------------------------|-----------------------------|
| 定义       | 可以被中断后重入          | 可以被多线程同时调用          |
| 全局变量    | 不使用                    | 可以使用（加锁保护）          |
| 静态变量    | 不使用                    | 可以使用（加锁保护）          |
| 锁         | 不使用                    | 可以使用                    |
| 关系       | 可重入一定是线程安全的     | 线程安全不一定是可重入的      |

## 五、减少副作用的方法

### 5.1 传参而非使用全局变量

```c
// 有副作用
double g_rate = 0.1;
double calculate(double amount) {
    return amount * (1 + g_rate);
}

// 无副作用
double calculate(double amount, double rate) {
    return amount * (1 + rate);
}
```

### 5.2 返回结果而非修改参数

```c
// 有副作用：修改传入的数组
void sort(int arr[], int n) {
    // 原地排序，修改了 arr
}

// 无副作用：返回新数组
int* sorted_copy(const int arr[], int n) {
    int *copy = malloc(n * sizeof(int));
    memcpy(copy, arr, n * sizeof(int));
    // 排序 copy，不修改原数组
    return copy;
}
```

### 5.3 使用 const 约束

```c
// const 参数明确表示不会修改数据
void print_array(const int arr[], int n) {
    for (int i = 0; i < n; i++)
        printf("%d ", arr[i]);
    // arr[i] = 0;  // 编译错误！
}
```

### 5.4 副作用隔离

```c
// 将 I/O 操作集中在特定函数中
// 纯计算函数不涉及 I/O
int compute_result(int a, int b) {
    return a * b + a - b;  // 纯函数
}

// I/O 集中在 main 或专门的输出函数中
int main() {
    int result = compute_result(3, 5);
    printf("结果: %d\n", result);  // I/O 操作集中在这里
    return 0;
}
```

## 六、要点总结

> **关键点：**
> 1. **纯函数**：相同输入总是产生相同输出，无副作用，易于测试和推理。
> 2. **副作用**：修改全局/静态变量、修改指针数据、I/O 操作等。
> 3. 全局变量使函数行为不可预测，增加调试难度。
> 4. **可重入函数**不使用全局或静态变量，信号处理函数必须可重入。
> 5. 使用 `const` 约束和参数传递来减少不必要的副作用。
> 6. 将计算逻辑（纯函数）和副作用（I/O）分离，使代码更清晰。
> 7. 多线程环境下，全局变量必须用同步机制保护。
