# 10 - 结构体位域

## 一、位域的基本概念

**位域（Bit Field）** 允许在结构体中以**位（bit）** 为单位指定成员的宽度，从而精确控制每个成员占用的比特数。

```c
struct Flags {
    unsigned int readable  : 1;  // 占 1 位
    unsigned int writable  : 1;  // 占 1 位
    unsigned int executable: 1;  // 占 1 位
    unsigned int reserved  : 5;  // 占 5 位
};
// 总共 8 位 = 1 字节（而不是普通 int 的 4 字节）
```

## 二、位域的定义语法

```c
struct 位域结构体名 {
    数据类型 成员名 : 位宽;
    数据类型 成员名 : 位宽;
    // ...
};
```

**规则：**
- 数据类型只能是 `int`、`unsigned int`、`signed int`（部分编译器支持 `_Bool`、`char` 等）
- 位宽必须是常量表达式
- 位宽不能超过类型本身的大小

```c
struct Packet {
    unsigned int type    : 3;   // 0~7
    unsigned int subtype : 4;   // 0~15
    unsigned int length  : 9;   // 0~511
    unsigned int flags   : 16;  // 0~65535
};
// 总共 32 位 = 4 字节
```

## 三、位域的存储方式

### 3.1 存储单元分配

位域成员存储在**存储单元（Storage Unit）** 中，通常是一个 `int` 大小（4 字节）。

```c
struct Example {
    unsigned int a : 3;   // 使用存储单元的第 0~2 位
    unsigned int b : 5;   // 使用第 3~7 位
    unsigned int c : 10;  // 使用第 8~17 位
    unsigned int d : 20;  // 超出剩余空间，分配新的存储单元
};
// sizeof 至少 8 字节（两个 int 存储单元）
```

### 3.2 零宽度位域

特殊用法：宽度为 0 的位域强制**对齐到下一个存储单元**。

```c
struct Align {
    unsigned int a : 4;
    unsigned int   : 0;  // 强制对齐，剩余位填充，下一个成员从新存储单元开始
    unsigned int b : 4;
};
```

### 3.3 未命名位域

不需要使用的位可以不命名，仅作填充。

```c
struct Padding {
    unsigned int low  : 4;
    unsigned int      : 4;  // 填充 4 位，不使用
    unsigned int high : 8;
};
```

## 四、位域的使用

### 4.1 读写位域

```c
#include <stdio.h>

struct Permissions {
    unsigned int read  : 1;
    unsigned int write : 1;
    unsigned int exec  : 1;
};

int main(void) {
    struct Permissions perm = {0};

    // 设置权限
    perm.read = 1;
    perm.write = 1;
    perm.exec = 0;

    printf("读: %d\n", perm.read);    // 1
    printf("写: %d\n", perm.write);   // 1
    printf("执行: %d\n", perm.exec);  // 0

    // 切换权限
    perm.exec = 1;
    perm.write = 0;

    return 0;
}
```

### 4.2 溢出截断

```c
struct Small {
    unsigned int value : 3;  // 只能存储 0~7
};

struct Small s;
s.value = 10;  // 10 = 0b1010，截断后为 0b010 = 2
printf("%u\n", s.value);  // 输出 2
```

## 五、位域的经典应用场景

### 5.1 硬件寄存器映射

嵌入式编程中最常见的用法，直接映射硬件寄存器的各个位。

```c
// UART 状态寄存器（假设 32 位）
struct UARTStatus {
    unsigned int tx_ready    : 1;  // bit 0: 发送就绪
    unsigned int rx_ready    : 1;  // bit 1: 接收就绪
    unsigned int tx_empty    : 1;  // bit 2: 发送缓冲区空
    unsigned int rx_full     : 1;  // bit 3: 接收缓冲区满
    unsigned int frame_err   : 1;  // bit 4: 帧错误
    unsigned int parity_err  : 1;  // bit 5: 校验错误
    unsigned int overrun_err : 1;  // bit 6: 溢出错误
    unsigned int reserved    : 25; // bit 7~31: 保留
};

// 使用
volatile struct UARTStatus *uart = (volatile struct UARTStatus *)0x40011000;
if (uart->rx_ready) {
    // 有数据可读
}
```

### 5.2 网络协议头部

```c
// IPv4 头部的部分字段
struct IPv4Flags {
    unsigned int version    : 4;
    unsigned int ihl        : 4;  // 头部长度
    unsigned int dscp       : 6;
    unsigned int ecn        : 2;
    unsigned int total_len  : 16;
};
```

### 5.3 标志位压缩

```c
// 用一个字节存储 8 个布尔标志
struct GameFlags {
    unsigned int is_paused      : 1;
    unsigned int is_sound_on    : 1;
    unsigned int is_fullscreen  : 1;
    unsigned int is_debug_mode  : 1;
    unsigned int difficulty     : 2;  // 0~3 四个难度级别
    unsigned int language       : 2;  // 0~3 四种语言
};
// 只用了 1 字节，如果用 8 个 int 需要 32 字节
```

## 六、位域与联合体结合

```c
// 同时支持按位访问和整体访问
union Register {
    struct {
        unsigned int enable    : 1;
        unsigned int interrupt : 1;
        unsigned int mode      : 2;
        unsigned int reserved  : 28;
    } bits;
    unsigned int value;
};

// 使用
union Register reg = {0};
reg.bits.enable = 1;
reg.bits.mode = 2;
printf("寄存器值: 0x%08X\n", reg.value);  // 一次性读取整个寄存器
```

## 七、位域的注意事项

### 7.1 不可取地址

```c
struct Flags {
    unsigned int a : 1;
    unsigned int b : 1;
};

struct Flags f;
// unsigned int *p = &f.a;  // 编译错误！位域不支持取地址
```

### 7.2 不可 sizeof

```c
// sizeof(f.a)  // 编译错误！
```

### 7.3 可移植性问题

```c
// 不同编译器的位域存储顺序可能不同（从低位到高位还是反过来）
// 依赖位域布局的代码是不可移植的

// 安全的做法：使用位运算代替位域
#define SET_BIT(reg, bit)   ((reg) |= (1U << (bit)))
#define CLR_BIT(reg, bit)   ((reg) &= ~(1U << (bit)))
#define GET_BIT(reg, bit)   (((reg) >> (bit)) & 1U)
```

## 八、位域 vs 位运算

| 特性 | 位域 | 位运算 |
|------|------|--------|
| 可读性 | 高（语义清晰） | 低（需要理解位操作） |
| 可移植性 | 低（存储顺序不确定） | 高（完全可控） |
| 性能 | 可能有额外开销 | 通常更快 |
| 地址操作 | 不支持 | 支持 |
| 建议场景 | 配置标志、协议定义 | 硬件寄存器、性能敏感代码 |

## 九、关键要点

> **重要提示：**
>
> 1. 位域可以精确控制成员占用的比特数，节省存储空间。
> 2. 位域成员**不能取地址**，不能用 `sizeof`。
> 3. 不同编译器对位域的存储顺序规定不同，**不要依赖位域的内存布局**。
> 4. 零宽度位域（`: 0`）用于强制对齐到下一个存储单元边界。
> 5. 硬件寄存器映射是位域最经典的应用场景。
> 6. 对于跨平台代码，位运算比位域更可靠。
