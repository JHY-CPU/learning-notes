# 21 - 不透明结构体

## 一、不透明指针模式

**不透明结构体（Opaque Struct）** 是一种**信息隐藏**技术：在头文件中声明结构体但不暴露其内部定义，用户只能通过**指针**和**接口函数**操作它。

```
用户视角：                实现者视角：
  - 知道类型存在             - 完整定义结构体
  - 只能用指针               - 实现所有接口函数
  - 不能访问内部成员          - 可以访问内部成员
```

## 二、实现步骤

### 2.1 头文件：公开接口

```c
// stack.h —— 用户包含此文件
#ifndef STACK_H
#define STACK_H

// 前向声明（不完整类型）
typedef struct Stack Stack;

// 接口函数
Stack  *stack_create(int capacity);
void    stack_destroy(Stack *s);
int     stack_push(Stack *s, int value);
int     stack_pop(Stack *s, int *out);
int     stack_top(const Stack *s, int *out);
int     stack_is_empty(const Stack *s);
int     stack_size(const Stack *s);

#endif
```

### 2.2 源文件：隐藏实现

```c
// stack.c —— 用户不直接看到此文件
#include "stack.h"
#include <stdlib.h>

// 完整定义（对外部不可见）
struct Stack {
    int *data;
    int  top;
    int  capacity;
};

Stack *stack_create(int capacity) {
    Stack *s = malloc(sizeof(Stack));
    if (s == NULL) return NULL;

    s->data = malloc(sizeof(int) * capacity);
    if (s->data == NULL) {
        free(s);
        return NULL;
    }
    s->top = 0;
    s->capacity = capacity;
    return s;
}

void stack_destroy(Stack *s) {
    if (s != NULL) {
        free(s->data);
        free(s);
    }
}

int stack_push(Stack *s, int value) {
    if (s->top >= s->capacity) return -1;
    s->data[s->top++] = value;
    return 0;
}

int stack_pop(Stack *s, int *out) {
    if (s->top <= 0) return -1;
    *out = s->data[--s->top];
    return 0;
}

int stack_top(const Stack *s, int *out) {
    if (s->top <= 0) return -1;
    *out = s->data[s->top - 1];
    return 0;
}

int stack_is_empty(const Stack *s) {
    return s->top == 0;
}

int stack_size(const Stack *s) {
    return s->top;
}
```

### 2.3 用户代码

```c
// main.c
#include <stdio.h>
#include "stack.h"

int main(void) {
    Stack *s = stack_create(10);

    stack_push(s, 10);
    stack_push(s, 20);
    stack_push(s, 30);

    int val;
    stack_top(s, &val);
    printf("栈顶: %d\n", val);       // 30

    printf("大小: %d\n", stack_size(s));  // 3

    while (!stack_is_empty(s)) {
        stack_pop(s, &val);
        printf("弹出: %d\n", val);
    }

    stack_destroy(s);
    return 0;
}

// 以下代码编译错误（在其他编译单元中）：
// s->top = 0;        // 错误：struct Stack 是不完整类型
// s->capacity = 100; // 错误：无法访问内部成员
```

## 三、不透明结构体的优势

### 3.1 信息隐藏

```c
// 用户不知道 Stack 的内部实现
// 实现者可以自由修改内部结构而不影响用户代码

// 版本 1：用数组实现
struct Stack { int *data; int top; int capacity; };

// 版本 2：改用链表实现（用户代码完全不用改）
struct Stack {
    struct Node *head;
    int size;
};
```

### 3.2 封装与安全性

- 用户不能直接修改内部状态
- 防止用户绕过接口函数破坏数据一致性
- 强制通过接口操作，增加安全性

### 3.3 ABI 兼容性

只要接口函数签名不变，可以更新实现而不需要重新编译用户代码。

## 四、FILE —— 标准库中的不透明结构体

`FILE` 是最常见的不透明结构体例子：

```c
// 标准库中 FILE 的使用
FILE *fp = fopen("data.txt", "r");  // 创建
// 用户不知道 FILE 的内部结构
// 只能通过 fread/fwrite/fclose 等函数操作
char buf[100];
fgets(buf, 100, fp);
fclose(fp);  // 销毁
```

## 五、关键要点

> **重要提示：**
>
> 1. 不透明结构体通过**前向声明 + 隐藏实现**实现信息隐藏。
> 2. 用户只能使用指针（`Stack *`），不能直接声明变量（`Stack s` 不行，因为大小未知）。
> 3. 所有操作必须通过接口函数进行，体现了**封装**思想。
> 4. `FILE`、`DIR` 等标准库类型就是不透明结构体的经典例子。
> 5. 实现者可以自由修改内部结构而不影响用户代码。
> 6. 这是 C 语言中实现**模块化设计**的核心技术。
