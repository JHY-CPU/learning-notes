# 02 - 结构体的定义方式

## 一、结构体定义的三种基本方式

C 语言提供了多种定义结构体的方式，各有适用场景。

### 方式一：先定义结构体类型，再声明变量

这是最常用、最规范的方式。

```c
// 第一步：定义结构体类型
struct Student {
    int id;
    char name[32];
    float score;
};

// 第二步：声明结构体变量
struct Student s1;
struct Student s2, s3;
```

**优点**：类型和变量分离，代码清晰，可复用性高。

### 方式二：定义结构体类型的同时声明变量

```c
struct Student {
    int id;
    char name[32];
    float score;
} s1, s2, s3;  // 定义类型的同时声明了三个变量

// 之后仍然可以继续声明
struct Student s4;
```

**注意**：`s1, s2, s3` 是全局变量（如果在函数外定义）或局部变量（如果在函数内定义）。

### 方式三：匿名结构体（不推荐）

```c
// 没有结构体标签名
struct {
    int x;
    int y;
} point1, point2;

// 无法再声明新的同类型变量，因为没有名字！
// struct ??? p3;  // 错误：无法引用此类型
```

**缺点**：没有类型名，无法复用，一般只在极特殊场景下使用。

## 二、使用 typedef 简化结构体

### 2.1 基本 typedef 用法

```c
// 方法 A：先定义结构体，再 typedef
struct Student {
    int id;
    char name[32];
    float score;
};
typedef struct Student Student;

// 之后可以直接用 Student 代替 struct Student
Student s1;
Student s2;
```

### 2.2 定义时同时 typedef（最简洁）

```c
typedef struct Student {
    int id;
    char name[32];
    float score;
} Student;  // Student 是 struct Student 的别名

// 使用
Student s1 = {1001, "张三", 95.0};
```

### 2.3 匿名结构体 + typedef

```c
typedef struct {
    int x;
    int y;
} Point;

// 使用
Point p1 = {10, 20};
Point p2 = {30, 40};
```

这种方式虽然没有结构体标签名，但通过 typedef 有了别名，因此可以正常使用。

### 2.4 typedef 的常见命名惯例

```c
// 风格 1：首字母大写（C++ 社区常用）
typedef struct {
    int x, y;
} Point;

// 风格 2：全大写（系统编程中常见）
typedef struct {
    int fd;
    int flags;
} FILE_HANDLE;

// 风格 3：带 _t 后缀（POSIX 惯例，但 _t 保留给标准库）
typedef struct {
    void *data;
    size_t len;
} buffer_t;

// 风格 4：保持 struct 前缀
typedef struct Node {
    int data;
    struct Node *next;
} ListNode;
```

## 三、结构体标签的作用域

```c
// 文件作用域 —— 全局可见
struct GlobalStruct {
    int value;
};

void func(void) {
    // 块作用域 —— 只在函数内可见
    struct LocalStruct {
        int data;
    };

    struct LocalStruct ls;  // OK
}

// struct LocalStruct ls2;  // 错误：此处不可见
```

## 四、结构体的不完整声明（前向声明）

当两个结构体相互引用时，需要前向声明：

```c
// 前向声明
struct B;

struct A {
    int value;
    struct B *b_ptr;  // 只能使用指针，不能直接嵌套
};

struct B {
    int data;
    struct A *a_ptr;
};
```

**关键点**：前向声明后只能使用**指针**类型，因为编译器还不知道 `struct B` 的完整大小。

## 五、定义结构体的常见错误

```c
// 错误 1：结构体内不能包含自身类型的实例（无限大小）
struct Node {
    int data;
    struct Node next;  // 错误！大小无法确定
};

// 正确：使用指针（指针大小固定）
struct Node {
    int data;
    struct Node *next;  // 正确
};

// 错误 2：结构体成员不能是不完整类型
struct Bad {
    int arr[];  // 错误（C99 柔性数组除外，且必须是最后一个成员）
};

// 错误 3：忘记分号
struct Point {
    int x;
    int y;
}  // ← 缺少分号！
```

## 六、完整示例

```c
#include <stdio.h>
#include <string.h>

// 使用 typedef 定义结构体
typedef struct {
    char street[64];
    char city[32];
    char province[32];
    int zipcode;
} Address;

typedef struct {
    int id;
    char name[32];
    int age;
    Address addr;  // 结构体嵌套
} Employee;

int main(void) {
    Employee emp;

    emp.id = 1001;
    strcpy(emp.name, "李明");
    emp.age = 28;
    strcpy(emp.addr.street, "中关村大街 1 号");
    strcpy(emp.addr.city, "北京");
    strcpy(emp.addr.province, "北京市");
    emp.addr.zipcode = 100080;

    printf("员工编号: %d\n", emp.id);
    printf("员工姓名: %s\n", emp.name);
    printf("员工年龄: %d\n", emp.age);
    printf("通讯地址: %s %s %s %d\n",
           emp.addr.province, emp.addr.city,
           emp.addr.street, emp.addr.zipcode);

    return 0;
}
```

## 七、关键要点

> **重要提示：**
>
> 1. 推荐使用 `typedef struct` 方式，省去每次写 `struct` 关键字。
> 2. 定义结构体时的变量声明容易被忽视，建议将类型定义和变量声明分开。
> 3. 结构体定义末尾**必须有分号**（这是高频错误）。
> 4. 匿名结构体无法二次声明变量，实用性有限。
> 5. 前向声明的结构体只能以**指针**形式使用。
> 6. 结构体不能包含自身类型的实例，但可以包含指向自身的指针。
> 7. 在 Linux/POSIX 编程中，`typedef struct { ... } name_t;` 非常常见。
