# 23 - 联合体与类型双关

## 一、什么是类型双关

**类型双关（Type Punning）** 是指通过一种数据类型来访问另一种数据类型在内存中的表示。C99 明确允许通过联合体进行类型双关。

```c
// 用联合体以不同方式解释同一块内存
union Pun {
    float    f;
    uint32_t i;
};

union Pun p;
p.f = 3.14f;
printf("0x%08X\n", p.i);  // 查看 float 的二进制表示
```

## 二、类型双关的常见用法

### 2.1 查看浮点数的位表示

```c
#include <stdio.h>
#include <stdint.h>
#include <string.h>

typedef union {
    float    f;
    uint32_t u;
    struct {
        uint32_t mantissa : 23;
        uint32_t exponent : 8;
        uint32_t sign     : 1;
    } parts;
} FloatPun;

void decompose_float(float f) {
    FloatPun p = {.f = f};
    printf("值: %f\n", p.f);
    printf("符号: %u\n", p.parts.sign);
    printf("指数: %u (实际: %d)\n", p.parts.exponent, (int)p.parts.exponent - 127);
    printf("尾数: 0x%06X\n", p.parts.mantissa);
    printf("原始: 0x%08X\n", p.u);
    printf("---\n");
}

int main(void) {
    decompose_float(3.14f);
    decompose_float(-1.0f);
    decompose_float(0.0f);
    decompose_float(1e10f);
    return 0;
}
```

### 2.2 双精度浮点数的位表示

```c
typedef union {
    double   d;
    uint64_t u;
    struct {
        uint64_t mantissa : 52;
        uint64_t exponent : 11;
        uint64_t sign     : 1;
    } parts;
} DoublePun;

// 将 double 的位模式重新解释为 uint64_t
uint64_t double_to_bits(double d) {
    DoublePun p = {.d = d};
    return p.u;
}

double bits_to_double(uint64_t bits) {
    DoublePun p = {.u = bits};
    return p.d;
}
```

## 三、端序（字节序）检测

```c
#include <stdio.h>
#include <stdint.h>

typedef union {
    uint32_t value;
    uint8_t  bytes[4];
} EndianTest;

void detect_endianness(void) {
    EndianTest test = {.value = 0x01020304};

    printf("字节序检测:\n");
    printf("值: 0x%08X\n", test.value);
    for (int i = 0; i < 4; i++) {
        printf("bytes[%d] = 0x%02X\n", i, test.bytes[i]);
    }

    if (test.bytes[0] == 0x04) {
        printf("结果: 小端序 (Little-Endian)\n");
    } else if (test.bytes[0] == 0x01) {
        printf("结果: 大端序 (Big-Endian)\n");
    } else {
        printf("结果: 混合序或其他\n");
    }
}

// 将小端序字节转为整数
uint32_t little_endian_to_u32(const uint8_t *buf) {
    return (uint32_t)buf[0] |
           ((uint32_t)buf[1] << 8) |
           ((uint32_t)buf[2] << 16) |
           ((uint32_t)buf[3] << 24);
}

// 将整数转为小端序字节
void u32_to_little_endian(uint32_t val, uint8_t *buf) {
    buf[0] = val & 0xFF;
    buf[1] = (val >> 8) & 0xFF;
    buf[2] = (val >> 16) & 0xFF;
    buf[3] = (val >> 24) & 0xFF;
}
```

## 四、整数与字节数组互转

```c
// 方式 1：联合体
typedef union {
    int32_t  value;
    uint8_t  bytes[4];
} IntBytes;

// 方式 2：指针类型双关（有 strict aliasing 风险）
int32_t bytes_to_int_unsafe(const uint8_t *buf) {
    return *(const int32_t *)buf;  // 未定义行为：违反 strict aliasing
}

// 方式 3：memcpy（安全，编译器会优化）
int32_t bytes_to_int_safe(const uint8_t *buf) {
    int32_t val;
    memcpy(&val, buf, sizeof(val));
    return val;
}
```

## 五、Strict Aliasing 规则

### 5.1 什么是 Strict Aliasing

C 标准规定，通过**不兼容类型的指针**访问同一内存是**未定义行为**：

```c
int32_t val = 0x12345678;

// 违反 strict aliasing —— 未定义行为
uint8_t *bytes = (uint8_t *)&val;
printf("%02X\n", bytes[0]);  // 在 -O2 优化下可能出错

// 安全的方式：使用联合体
union { int32_t i; uint8_t b[4]; } u;
u.i = 0x12345678;
printf("%02X\n", u.b[0]);  // OK：C99 明确允许
```

### 5.2 与指针类型双关的对比

```c
// 方式 1：指针类型双关 —— 有风险
float f = 3.14f;
uint32_t *bits = (uint32_t *)&f;     // 违反 strict aliasing
printf("0x%08X\n", *bits);           // 未定义行为！

// 方式 2：联合体 —— C99 允许
union { float f; uint32_t i; } u;
u.f = 3.14f;
printf("0x%08X\n", u.i);             // OK

// 方式 3：memcpy —— 总是安全
float f2 = 3.14f;
uint32_t bits2;
memcpy(&bits2, &f2, sizeof(bits2));  // OK
printf("0x%08X\n", bits2);
```

## 六、网络协议中的应用

```c
#include <stdint.h>
#include <string.h>

// 网络协议头：同时支持按字段访问和整体操作
#pragma pack(push, 1)
typedef union {
    struct {
        uint8_t  type;
        uint8_t  flags;
        uint16_t length;
        uint32_t sequence;
    } fields;
    uint8_t raw[8];
} PacketHeader;
#pragma pack(pop)

// 检查校验和
int verify_packet(const uint8_t *data, size_t len) {
    PacketHeader hdr;
    memcpy(&hdr, data, sizeof(hdr));

    // 可以用 fields 访问
    printf("类型: %u, 长度: %u\n",
           hdr.fields.type, hdr.fields.length);

    // 也可以用 raw 进行字节级操作
    uint8_t checksum = 0;
    for (size_t i = 0; i < sizeof(hdr.raw); i++) {
        checksum ^= hdr.raw[i];
    }
    return checksum == 0;
}
```

## 七、安全的类型双关实践总结

| 方法 | 安全性 | 标准 | 性能 |
|------|--------|------|------|
| 联合体类型双关 | 安全 | C99 明确允许 | 高 |
| `memcpy` | 安全 | 所有版本 | 高（编译器优化） |
| 指针类型双关 | 不安全 | 违反 strict aliasing | 可能被错误优化 |

**最佳实践**：优先使用联合体或 `memcpy`，避免指针类型双关。

## 八、关键要点

> **重要提示：**
>
> 1. C99 明确允许通过联合体进行类型双关。
> 2. 指针类型双关违反 strict aliasing 规则，在高优化级别下可能产生错误结果。
> 3. `memcpy` 是最安全的类型双关方式，现代编译器会将其优化掉。
> 4. 端序检测是类型双关的经典应用场景。
> 5. 浮点数的位级操作需要类型双关（如快速平方根倒数算法）。
> 6. 网络协议解析中经常需要以不同粒度访问同一块数据。
