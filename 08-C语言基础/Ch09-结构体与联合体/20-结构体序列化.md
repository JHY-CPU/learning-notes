# 20 - 结构体序列化

## 一、序列化的基本概念

**序列化（Serialization）** 是将结构体数据转换为可存储或传输的字节流的过程。**反序列化（Deserialization）** 是其逆过程。

```
内存中的结构体  --序列化-->  字节流（文件/网络）
字节流          --反序列化-->  内存中的结构体
```

## 二、直接写入文件

### 2.1 写入与读取（fwrite/fread）

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

typedef struct {
    int   id;
    char  name[32];
    float score;
} Student;

// 写入单个结构体
int write_student(const char *filename, const Student *stu) {
    FILE *fp = fopen(filename, "wb");
    if (fp == NULL) {
        perror("fopen");
        return -1;
    }
    fwrite(stu, sizeof(Student), 1, fp);
    fclose(fp);
    return 0;
}

// 读取单个结构体
int read_student(const char *filename, Student *stu) {
    FILE *fp = fopen(filename, "rb");
    if (fp == NULL) {
        perror("fopen");
        return -1;
    }
    fread(stu, sizeof(Student), 1, fp);
    fclose(fp);
    return 0;
}

// 写入数组
int write_students(const char *filename, const Student *arr, int count) {
    FILE *fp = fopen(filename, "wb");
    if (fp == NULL) return -1;
    fwrite(&count, sizeof(int), 1, fp);       // 先写入数量
    fwrite(arr, sizeof(Student), count, fp);   // 再写入数据
    fclose(fp);
    return 0;
}

// 读取数组
Student *read_students(const char *filename, int *out_count) {
    FILE *fp = fopen(filename, "rb");
    if (fp == NULL) return NULL;

    int count;
    fread(&count, sizeof(int), 1, fp);

    Student *arr = malloc(count * sizeof(Student));
    if (arr == NULL) {
        fclose(fp);
        return NULL;
    }
    fread(arr, sizeof(Student), count, fp);
    fclose(fp);

    *out_count = count;
    return arr;
}
```

## 三、直接序列化的问题

### 3.1 字节序问题

不同 CPU 架构使用不同的字节序：

```
大端序 (Big-Endian):    0x12345678 存储为 12 34 56 78
小端序 (Little-Endian): 0x12345678 存储为 78 56 34 12

x86/x64: 小端序
ARM: 可配置，通常小端
网络协议: 大端序（网络字节序）
```

直接写入的二进制文件在不同字节序的系统间**不可移植**。

### 3.2 对齐填充问题

不同编译器或不同平台的结构体对齐方式可能不同，导致二进制布局不一致。

### 3.3 指针问题

结构体中的指针成员序列化后毫无意义——另一个进程/机器上该地址肯定无效。

## 四、跨平台序列化方案

### 4.1 字节序转换

```c
#include <stdint.h>

// 主机字节序转网络字节序
uint32_t host_to_net32(uint32_t h) {
    uint8_t buf[4];
    buf[0] = (h >> 24) & 0xFF;
    buf[1] = (h >> 16) & 0xFF;
    buf[2] = (h >> 8) & 0xFF;
    buf[3] = h & 0xFF;
    return *(uint32_t *)buf;  // 或逐字节写入
}

// 网络字节序转主机字节序
uint32_t net_to_host32(const uint8_t *buf) {
    return ((uint32_t)buf[0] << 24) |
           ((uint32_t)buf[1] << 16) |
           ((uint32_t)buf[2] << 8) |
           (uint32_t)buf[3];
}
```

### 4.2 手动序列化/反序列化

```c
#pragma pack(push, 1)
typedef struct {
    int   id;
    char  name[32];
    float score;
} Student;
#pragma pack(pop)

// 序列化为字节流（大端序）
size_t student_serialize(const Student *stu, uint8_t *buf) {
    size_t pos = 0;

    // id: 4 字节大端
    buf[pos++] = (stu->id >> 24) & 0xFF;
    buf[pos++] = (stu->id >> 16) & 0xFF;
    buf[pos++] = (stu->id >> 8) & 0xFF;
    buf[pos++] = stu->id & 0xFF;

    // name: 32 字节直接拷贝
    memcpy(buf + pos, stu->name, 32);
    pos += 32;

    // score: float 转为 uint32_t 后大端写入
    uint32_t score_bits;
    memcpy(&score_bits, &stu->score, sizeof(float));
    buf[pos++] = (score_bits >> 24) & 0xFF;
    buf[pos++] = (score_bits >> 16) & 0xFF;
    buf[pos++] = (score_bits >> 8) & 0xFF;
    buf[pos++] = score_bits & 0xFF;

    return pos;  // 返回写入字节数
}

// 反序列化
void student_deserialize(Student *stu, const uint8_t *buf) {
    size_t pos = 0;

    stu->id = ((int)buf[0] << 24) | ((int)buf[1] << 16) |
              ((int)buf[2] << 8) | (int)buf[3];
    pos += 4;

    memcpy(stu->name, buf + pos, 32);
    pos += 32;

    uint32_t score_bits = ((uint32_t)buf[pos] << 24) |
                          ((uint32_t)buf[pos+1] << 16) |
                          ((uint32_t)buf[pos+2] << 8) |
                          (uint32_t)buf[pos+3];
    memcpy(&stu->score, &score_bits, sizeof(float));
}
```

### 4.3 文本格式序列化

文本格式天然跨平台，适合配置文件和调试。

```c
// 写入文本格式
int student_save_text(const char *filename, const Student *arr, int count) {
    FILE *fp = fopen(filename, "w");
    if (fp == NULL) return -1;

    fprintf(fp, "%d\n", count);
    for (int i = 0; i < count; i++) {
        fprintf(fp, "%d,%s,%.2f\n",
                arr[i].id, arr[i].name, arr[i].score);
    }
    fclose(fp);
    return 0;
}

// 从文本格式读取
int student_load_text(const char *filename, Student **out, int *out_count) {
    FILE *fp = fopen(filename, "r");
    if (fp == NULL) return -1;

    int count;
    fscanf(fp, "%d\n", &count);

    Student *arr = malloc(count * sizeof(Student));
    for (int i = 0; i < count; i++) {
        fscanf(fp, "%d,%31[^,],%f\n",
               &arr[i].id, arr[i].name, &arr[i].score);
    }
    fclose(fp);

    *out = arr;
    *out_count = count;
    return 0;
}
```

## 五、常见序列化格式对比

| 格式 | 可读性 | 体积 | 速度 | 跨平台 | 复杂度 |
|------|--------|------|------|--------|--------|
| 二进制直接写入 | 无 | 最小 | 最快 | 差 | 低 |
| 手动二进制序列化 | 无 | 小 | 快 | 好 | 高 |
| CSV/文本 | 好 | 大 | 慢 | 好 | 中 |
| JSON | 好 | 中 | 中 | 好 | 中 |
| MessagePack | 无 | 小 | 快 | 好 | 需库 |

## 六、关键要点

> **重要提示：**
>
> 1. 直接 `fwrite` 结构体的二进制数据不可跨平台。
> 2. 字节序是跨平台二进制序列化的最大障碍。
> 3. 结构体中的指针成员不能直接序列化。
> 4. `#pragma pack(1)` 消除对齐填充，使二进制布局确定。
> 5. 文本格式（CSV/JSON）天然跨平台，适合通用场景。
> 6. 手动序列化虽繁琐，但提供最好的跨平台兼容性和安全性。
