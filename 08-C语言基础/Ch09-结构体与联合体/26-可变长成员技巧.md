# 26 - 可变长成员技巧

## 一、问题背景

在 C 语言中，有时需要结构体包含**运行时才确定大小**的数据。传统做法是使用指针 + 两次 malloc，而柔性数组和零长度数组技巧提供了一次 malloc、内存连续的方案。

```c
// 问题：存储一条变长消息
// 消息内容长度在运行时才确定
```

## 二、C99 柔性数组（标准方式）

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <stdint.h>

#pragma pack(push, 1)
typedef struct {
    uint8_t  type;
    uint16_t length;
    uint8_t  data[];  // 柔性数组：长度在运行时确定
} Message;
#pragma pack(pop)

Message *msg_create(uint8_t type, const void *payload, uint16_t len) {
    Message *msg = malloc(sizeof(Message) + len);
    if (msg == NULL) return NULL;

    msg->type = type;
    msg->length = len;
    if (payload != NULL && len > 0) {
        memcpy(msg->data, payload, len);
    }
    return msg;
}

void msg_print(const Message *msg) {
    printf("类型: %u, 长度: %u, 数据: ", msg->type, msg->length);
    for (uint16_t i = 0; i < msg->length && i < 32; i++) {
        printf("%02X ", msg->data[i]);
    }
    printf("\n");
}

void msg_free(Message *msg) {
    free(msg);
}
```

## 三、零长度数组（GCC 扩展）

零长度数组 `data[0]` 是 GCC 的扩展，在 C99 柔性数组出现之前广泛使用。

```c
// GCC 扩展：零长度数组
typedef struct {
    int  length;
    char data[0];  // 零长度数组（非标准，但广泛支持）
} FlexBuf;

FlexBuf *flexbuf_create(int size) {
    FlexBuf *buf = malloc(sizeof(FlexBuf) + size);
    if (buf == NULL) return NULL;
    buf->length = size;
    return buf;
}
```

### 零长度数组 vs 柔性数组

| 特性 | `data[0]`（GCC扩展） | `data[]`（C99标准） |
|------|---------------------|-------------------|
| 标准 | 非标准 | C99 标准 |
| sizeof | 不包含 | 不包含 |
| 编译器支持 | GCC, Clang, MSVC | 所有现代编译器 |
| 编译器警告 | 可能有 | 无 |
| 推荐 | 旧代码维护 | 新代码使用 |

## 四、长度为 1 的数组（C89 兼容）

在 C99 之前，常用长度为 1 的数组来模拟柔性数组。

```c
// C89 兼容方式
typedef struct {
    int  length;
    char data[1];  // 实际大小由 malloc 控制
} FlexBuf89;

FlexBuf89 *flexbuf89_create(int size) {
    // 分配时减去 data[1] 已有的 1 字节
    FlexBuf89 *buf = malloc(sizeof(FlexBuf89) + size - 1);
    if (buf == NULL) return NULL;
    buf->length = size;
    return buf;
}
```

> **注意**：严格来说这在 C 标准中是未定义行为，但几乎所有编译器都支持。

## 五、动态结构体模式

### 5.1 带变长字符串的结构体

```c
typedef struct {
    size_t len;
    size_t capacity;
    char   str[];
} DynStr;

DynStr *dynstr_new(const char *init) {
    size_t init_len = init ? strlen(init) : 0;
    size_t cap = init_len + 1;
    if (cap < 32) cap = 32;  // 最小容量

    DynStr *ds = malloc(sizeof(DynStr) + cap);
    if (ds == NULL) return NULL;

    ds->capacity = cap;
    ds->len = init_len;
    if (init) {
        memcpy(ds->str, init, init_len + 1);
    } else {
        ds->str[0] = '\0';
    }
    return ds;
}

DynStr *dynstr_append(DynStr *ds, const char *suffix) {
    size_t suf_len = strlen(suffix);
    size_t new_len = ds->len + suf_len;

    if (new_len + 1 > ds->capacity) {
        size_t new_cap = (new_len + 1) * 2;
        DynStr *new_ds = realloc(ds, sizeof(DynStr) + new_cap);
        if (new_ds == NULL) return ds;
        new_ds->capacity = new_cap;
        ds = new_ds;
    }
    memcpy(ds->str + ds->len, suffix, suf_len + 1);
    ds->len = new_len;
    return ds;
}

void dynstr_free(DynStr *ds) {
    free(ds);
}
```

### 5.2 带变长数组的复杂结构体

```c
typedef struct {
    char     name[32];
    int      count;
    double   scores[];  // 柔性数组存储成绩
} StudentReport;

StudentReport *report_create(const char *name,
                              const double *scores,
                              int count) {
    StudentReport *r = malloc(sizeof(StudentReport) +
                              count * sizeof(double));
    if (r == NULL) return NULL;

    strncpy(r->name, name, sizeof(r->name) - 1);
    r->name[sizeof(r->name) - 1] = '\0';
    r->count = count;
    memcpy(r->scores, scores, count * sizeof(double));
    return r;
}

double report_average(const StudentReport *r) {
    double sum = 0;
    for (int i = 0; i < r->count; i++) {
        sum += r->scores[i];
    }
    return r->count > 0 ? sum / r->count : 0;
}
```

## 七、关键要点

> **重要提示：**
>
> 1. C99 柔性数组 `data[]` 是标准推荐的方式。
> 2. 零长度数组 `data[0]` 是 GCC 扩展，广泛支持但非标准。
> 3. `malloc(sizeof(Struct) + extra_size)` 一次性分配头部和变长数据。
> 4. `realloc` 可以对整个结构体进行扩容。
> 5. 柔性数组必须是结构体的最后一个成员，且结构体不能作为数组元素。
> 6. 包含柔性数组的结构体不能嵌套在其他结构体中。
