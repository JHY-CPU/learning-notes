# 31 - C17新特性

## 概述

C17（ISO/IEC 9899:2018）是C11的一个修正版本，主要修复了C11中的缺陷，没有引入重大新特性。它也被称为C18。

---

## 1. C17的定位

C17不是一个新标准，而是对C11的维护更新：

- **不添加新语言特性**
- **不删除现有特性**
- 修复C11规范中的缺陷和歧义
- 修正技术勘误（Technical Corrigenda）

---

## 2. 主要修正内容

### 2.1 __STDC_VERSION__ 宏更新

```c
#include <stdio.h>

void check_version(void) {
#if __STDC_VERSION__ == 201710L
    printf("编译器支持C17\n");
#elif __STDC_VERSION__ == 201112L
    printf("编译器支持C11\n");
#elif __STDC_VERSION__ == 199901L
    printf("编译器支持C99\n");
#else
    printf("未知版本: %ld\n", __STDC_VERSION__);
#endif
}
```

### 2.2 修复原子操作规范

C11中的`_Alignas`和`_Atomic`交互存在歧义，C17修复了相关规范。

```c
// C17中这些操作更明确
#include <stdatomic.h>

// C11/C17都支持
_Atomic int counter = ATOMIC_VAR_INIT(0);

// 原子操作
void increment(void) {
    atomic_fetch_add(&counter, 1);
}
```

### 2.3 修复函数指针转换

```c
// C17明确了函数指针与void*之间的转换规则
// 在支持的平台上，可以安全地进行转换
void callback_demo(void) {
    void (*func_ptr)(int) = (void (*)(int))printf;
    // C17明确了这种转换的行为
}
```

---

## 3. 与C11的兼容性

C17完全向后兼容C11。大多数代码无需修改即可在C17下编译。

### 编译器支持

```bash
# GCC
gcc -std=c17 app.c      # 明确指定C17
gcc -std=c1x app.c      # C11（可能也支持C17修复）

# Clang
clang -std=c17 app.c

# MSVC (部分支持)
cl /std:c17 app.c
```

### 检测编译器支持

```c
#if defined(__STDC_VERSION__) && __STDC_VERSION__ >= 201710L
    #define C17_AVAILABLE 1
#else
    #define C17_AVAILABLE 0
#endif
```

---

## 4. 从C11迁移到C17

迁移通常很简单：

1. 更新编译器到支持C17的版本
2. 修改编译选项从`-std=c11`到`-std=c17`
3. 编译并修复可能的新警告

```bash
# 检查可能的兼容性问题
gcc -std=c17 -Wall -Wextra -pedantic app.c
```

---

## 5. C17的重要性

虽然C17没有新特性，但它：

- 确保了C11规范的正确性
- 为C23奠定了基础
- 表明C语言仍在积极维护
- 解决了实际使用中发现的规范问题

---

|--------|-----------------|

| Clang | 6.0 |
| MSVC | 19.14 (VS 2017 15.7) |
| ICC | 19.0 |

---

## 要点总结

1. C17是C11的维护版本，没有新语言特性
2. 主要修复了C11规范中的歧义和错误
3. __STDC_VERSION__在C17中为201710L
4. 与C11完全兼容，大多数代码无需修改
5. 建议新项目使用`-std=c17`
6. C17为后续的C23标准做好了准备
