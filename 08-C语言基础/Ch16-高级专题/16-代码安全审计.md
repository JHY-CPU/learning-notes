# 16 - 代码安全审计

## 概述

代码安全审计是系统性地检查代码中潜在安全漏洞的过程。本节介绍C语言中常见的安全漏洞模式和审计方法。

---

## 1. 常见安全漏洞分类

| 类别 | 漏洞类型 | 严重程度 |
|------|----------|----------|
| 内存安全 | 缓冲区溢出、use-after-free、double-free | 严重 |
| 输入验证 | 注入攻击、路径遍历 | 严重 |
| 整数安全 | 整数溢出、符号错误 | 高 |
| 逻辑错误 | 竞态条件、死锁 | 中 |
| 信息泄露 | 未初始化内存、日志泄露 | 中 |

---

## 2. 缓冲区溢出

### 典型漏洞模式

```c
// 漏洞1: strcpy无边界检查
void vuln1(char *input) {
    char buf[64];
    strcpy(buf, input);  // 如果input超过63字节，溢出
}

// 漏洞2: 循环边界错误
void vuln2(int *arr, int count) {
    int buf[100];
    for (int i = 0; i <= count; i++) {  // <= 应为 <
        buf[i] = arr[i];  // off-by-one
    }
}

// 漏洞3: 指针运算错误
void vuln3(char *data, int len) {
    char buf[256];
    memcpy(buf, data, len);  // len未经验证
}

// 修复方案
void fixed1(const char *input) {
    char buf[64];
    strncpy(buf, input, sizeof(buf) - 1);
    buf[sizeof(buf) - 1] = '\0';
}

void fixed2(int *arr, int count) {
    if (count > 100) return;  // 边界检查
    int buf[100];
    for (int i = 0; i < count; i++) {
        buf[i] = arr[i];
    }
}
```

## 3. Use-After-Free

```c
#include <stdlib.h>

// 漏洞：使用已释放的内存
void uaf_vuln(void) {
    char *ptr = malloc(100);
    strcpy(ptr, "重要数据");
    free(ptr);        // 释放
    // ... 其他代码可能重新分配这块内存 ...
    printf("%s\n", ptr);  // 悬垂指针！
}

// 修复：释放后置空
void uaf_fixed(void) {
    char *ptr = malloc(100);
    strcpy(ptr, "重要数据");
    free(ptr);
    ptr = NULL;       // 置空防止使用
}
```

## 4. Double Free

```c
// 漏洞：重复释放
void df_vuln(int condition) {
    char *ptr = malloc(100);
    if (condition) {
        free(ptr);
    }
    // 忘记else分支也free了
    free(ptr);  // 如果condition为真，double free!
}

// 修复
void df_fixed(int condition) {
    char *ptr = malloc(100);
    if (condition) {
        free(ptr);
        ptr = NULL;  // 释放后置空
    }
    free(ptr);  // NULL是安全的
}
```

## 5. 竞态条件（TOCTOU）

```c
#include <unistd.h>
#include <fcntl.h>
#include <stdio.h>

// 漏洞：检查和使用之间的时间窗口
void race_vuln(const char *filename) {
    // 检查文件是否存在（Time of Check）
    if (access(filename, R_OK) == 0) {
        // 攻击者可能在此处替换文件
        // 打开文件（Time of Use）
        int fd = open(filename, O_RDONLY);
        // 读取的可能是攻击者控制的文件
        char buf[256];
        read(fd, buf, sizeof(buf));
        close(fd);
    }
}

// 修复：直接打开，用返回值判断
void race_fixed(const char *filename) {
    int fd = open(filename, O_RDONLY);
    if (fd < 0) {
        perror("open");
        return;
    }
    // 文件已被安全打开
    struct stat st;
    fstat(fd, &st);
    // 检查文件属性...
    close(fd);
}
```

## 6. 未初始化内存

```c
#include <stdlib.h>
#include <string.h>

// 漏洞：未初始化内存可能包含敏感信息
void leak_vuln(void) {
    char buf[256];
    // buf未初始化，可能包含之前栈上的敏感数据
    send(client_fd, buf, 256, 0);  // 信息泄露！
}

// 修复：初始化或清零
void leak_fixed(void) {
    char buf[256] = {0};  // 初始化为零
    // 或者使用memset
    // memset(buf, 0, sizeof(buf));
    send(client_fd, buf, 256, 0);
}

// 安全清理敏感数据
void secure_zero(void *ptr, size_t len) {
    volatile unsigned char *p = ptr;
    while (len--) *p++ = 0;
    // 使用volatile防止编译器优化掉memset
}
```

## 7. 静态分析工具

### Clang Static Analyzer

```bash
# 安装
sudo apt install clang clang-tools

# 分析
scan-build gcc -o app app.c
clang-tidy app.c -- -std=c11

# 生成报告
scan-build -o report_dir gcc app.c
```

### Cppcheck

```bash
# 安装
sudo apt install cppcheck

# 分析
cppcheck --enable=all --suppress=missingIncludeSystem app.c

# 生成HTML报告
cppcheck --enable=all --html app.c 2> report.html
```

### GCC警告选项

```bash
# 推荐的警告选项
gcc -Wall -Wextra -Wpedantic \
    -Wformat=2 -Wformat-security \
    -Wnull-dereference \
    -Wstack-protector \
    -Wtrampolines \
    -Wshift-overflow=2 \
    -Wcast-align \
    -Wconversion \
    -Wshadow \
    -o app app.c
```

### Valgrind（运行时检测）

```bash
# 内存错误检测
valgrind --leak-check=full ./app

# 线程错误检测
valgrind --tool=helgrind ./app

# 缓存分析
valgrind --tool=cachegrind ./app
```

## 要点总结

1. 内存安全是C语言最大的安全挑战
2. 使用静态分析工具可以在编译期发现大部分漏洞
3. Valgrind等运行时工具可以发现内存泄漏和越界访问
4. 安全审计应遵循系统的检查清单
5. 编译器安全选项（如-fstack-protector）提供额外保护层
6. 安全是持续的过程，不是一次性的检查
