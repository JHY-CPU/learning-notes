# 19 - C与C++互操作

## 概述

C和C++可以在同一项目中混合使用。理解`extern "C"`、名称修饰（name mangling）和ABI兼容性是实现互操作的关键。

---

## 1. extern "C" 的作用

### 名称修饰（Name Mangling）

C++编译器会对函数名进行修饰以支持函数重载：

```cpp
// C++编译后的符号名（示例）
void foo(int);    // _Z3fooi
void foo(double); // _Z3food
```

C编译器不做修饰：

```c
void foo(int);  // foo
```

### extern "C" 告诉C++编译器使用C的链接方式

```c
// mylib.h - 兼容C和C++的头文件
#ifndef MYLIB_H
#define MYLIB_H

#ifdef __cplusplus
extern "C" {
#endif

void my_function(int x);
int my_compute(int a, int b);

#ifdef __cplusplus
}
#endif

#endif
```

```cpp
// C++代码中使用
#include "mylib.h"  // extern "C"生效，使用C链接

void cpp_code() {
    my_function(42);  // 链接到C符号 "my_function"
}
```

## 2. C调用C++代码

### 方法1: 通过C包装层

```cpp
// myclass.hpp - C++类
#pragma once
#include <string>

class Calculator {
public:
    Calculator();
    ~Calculator();
    void add(double value);
    double getResult() const;
private:
    double result_;
};
```

```cpp
// myclass.cpp
#include "myclass.hpp"

Calculator::Calculator() : result_(0) {}
Calculator::~Calculator() {}
void Calculator::add(double value) { result_ += value; }
double Calculator::getResult() const { return result_; }
```

```cpp
// wrapper.cpp - C包装层
#include "myclass.hpp"

extern "C" {

// 创建对象
void* calculator_create(void) {
    return new Calculator();
}

// 销毁对象
void calculator_destroy(void* ptr) {
    delete static_cast<Calculator*>(ptr);
}

// 操作对象
void calculator_add(void* ptr, double value) {
    static_cast<Calculator*>(ptr)->add(value);
}

double calculator_get_result(void* ptr) {
    return static_cast<Calculator*>(ptr)->getResult();
}

}  // extern "C"
```

```c
// main.c - C代码调用C++类
#include <stdio.h>

// 声明C接口
void* calculator_create(void);
void calculator_destroy(void* ptr);
void calculator_add(void* ptr, double value);
double calculator_get_result(void* ptr);

int main(void) {
    void* calc = calculator_create();

    calculator_add(calc, 10.0);
    calculator_add(calc, 20.5);

    printf("结果: %.1f\n", calculator_get_result(calc));

    calculator_destroy(calc);
    return 0;
}
```

### 编译命令

```bash
# 编译C++部分
g++ -c myclass.cpp -o myclass.o
g++ -c wrapper.cpp -o wrapper.o

# 编译C部分
gcc -c main.c -o main.o

# 链接（必须用C++链接器）
g++ main.o myclass.o wrapper.o -o app
```

## 3. C++调用C代码

### 方法1: 头文件中使用extern "C"

```c
// math_utils.h
#ifndef MATH_UTILS_H
#define MATH_UTILS_H

int add(int a, int b);
int multiply(int a, int b);

#endif
```

```c
// math_utils.c
#include "math_utils.h"

int add(int a, int b) { return a + b; }
int multiply(int a, int b) { return a * b; }
```

```cpp
// main.cpp - C++调用C
extern "C" {
#include "math_utils.h"
}

#include <iostream>

int main() {
    std::cout << "add(2,3) = " << add(2, 3) << std::endl;
    std::cout << "mul(4,5) = " << multiply(4, 5) << std::endl;
    return 0;
}
```

### 方法2: 兼容性头文件

```c
// mylib.h - 通用头文件
#ifndef MYLIB_H
#define MYLIB_H

#ifdef __cplusplus
extern "C" {
#endif

typedef struct {
    int x, y;
} Point;

Point point_create(int x, int y);
int point_distance(Point a, Point b);

#ifdef __cplusplus
}
#endif

#endif
```

## 4. ABI兼容性

### 什么是ABI

ABI（Application Binary Interface）定义了：
- 函数调用约定（参数传递方式、栈帧布局）
- 数据类型的内存布局
- 名称修饰规则
- 异常处理机制

### 常见的ABI不兼容问题

```cpp
// 问题1: C++类的内存布局
// 不同编译器/版本可能有不同的vtable布局
class Base {
public:
    virtual void foo();
    virtual void bar();
};

// 问题2: 异常处理
// C++异常不能跨越C边界
// extern "C"函数不应抛出异常

// 问题3: 结构体对齐
// #pragma pack在不同编译器中行为可能不同
```

### 保持ABI兼容的实践

```c
// 使用不透明指针隐藏实现
// header.h
typedef struct MyStruct MyStruct;

MyStruct* mystruct_create(void);
void mystruct_destroy(MyStruct* obj);
void mystruct_do_something(MyStruct* obj);

// 实际定义在.c/.cpp文件中，外部不可见
```

## 5. 混合编译的注意事项

### 结构体对齐

```c
// 确保C和C++中结构体布局一致
#pragma pack(push, 4)
typedef struct {
    int id;
    char name[32];
    double value;
} DataRecord;
#pragma pack(pop)
```

### 全局对象

```cpp
// C++中全局对象的构造/析构
// 不要在extern "C"函数中依赖全局C++对象

// 正确做法：显式初始化
extern "C" void mylib_init(void) {
    // 手动初始化，不依赖全局对象构造
}
```

### 内存管理

```cpp
// 不要跨语言边界分配/释放内存
// C分配的内存用free，C++分配的用delete

// 统一使用malloc/free
extern "C" {
    void* my_alloc(size_t size) {
        return malloc(size);  // C风格
    }
    void my_free(void* ptr) {
        free(ptr);  // C风格
    }
}
```

## 6. 完整示例：C库被C++使用

```c
// db.h
#ifndef DB_H
#define DB_H

#ifdef __cplusplus
extern "C" {
#endif

typedef struct DB DB;

DB* db_open(const char *path);
void db_close(DB *db);
int db_put(DB *db, const char *key, const char *value);
const char* db_get(DB *db, const char *key);

#ifdef __cplusplus
}
#endif

#endif
```

```cpp
// main.cpp
#include "db.h"
#include <iostream>
#include <string>

int main() {
    DB* db = db_open("/tmp/test.db");
    if (!db) {
        std::cerr << "打开数据库失败" << std::endl;
        return 1;
    }

    db_put(db, "name", "张三");
    db_put(db, "age", "25");

    const char* name = db_get(db, "name");
    if (name) {
        std::cout << "name = " << name << std::endl;
    }

    db_close(db);
    return 0;
}
```

## 要点总结

1. `extern "C"`是C/C++互操作的核心，禁用C++名称修饰
2. 头文件中使用`#ifdef __cplusplus`保护`extern "C"`声明
3. C调用C++需要编写C包装层，使用void*传递对象
4. 不要跨语言边界传递C++异常
5. 不要跨语言边界分配/释放内存
6. 链接时必须使用C++链接器（g++）
