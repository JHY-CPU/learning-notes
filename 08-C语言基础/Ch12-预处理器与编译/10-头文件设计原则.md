# 10 - 头文件设计原则

## 头文件的核心作用

头文件在C语言中承担**接口声明**的职责——它告诉编译器某个函数、类型或变量的存在，而不提供实现。良好的头文件设计是构建可维护、可扩展C项目的基础。

## 原则一：自包含（Self-Contained）

每个头文件应该能够独立编译，即包含它所需要的所有依赖。

### 不好的示例

```c
// reader.h — 不自包含
#ifndef READER_H
#define READER_H

size_t read_data(Buffer *buf);  // Buffer 从哪里来？
int parse_token(Token *tok);    // Token 未定义！

#endif
```

### 好的示例

```c
// reader.h — 自包含
#ifndef READER_H
#define READER_H

#include <stddef.h>        // size_t
#include "buffer.h"        // Buffer 类型
#include "tokenizer.h"     // Token 类型

size_t read_data(Buffer *buf);
int parse_token(Token *tok);

#endif
```

### 验证自包含性

```bash
# 编译器直接验证头文件
gcc -fsyntax-only reader.h

# 或创建测试文件
echo '#include "reader.h"' > test_reader.c
gcc -c test_reader.c
```

## 原则二：最小依赖

只包含你实际需要的头文件。过多的 `#include` 会增加编译时间并引入不必要的耦合。

### 使用前向声明减少依赖

```c
// 不好：包含了完整的头文件
#ifndef PROCESSOR_H
#define PROCESSOR_H

#include "database.h"    // 只用到了 DB 类型的指针！
#include "logger.h"      // 只用到了 log 函数的声明

void process(DB *database, const char *input);
#endif

// 好：使用前向声明
#ifndef PROCESSOR_H
#define PROCESSOR_H

struct DB;  // 前向声明：只需要指针，不需要完整定义
typedef struct DB DB;

void process(DB *database, const char *input);
#endif
```

### 前向声明的限制

```c
// 前向声明可以用在以下场景：
struct Point;           // ✓ 声明指向 Point 的指针
void func(struct Point *p);  // ✓ 函数参数是指针
struct Point *create(void);  // ✓ 返回值是指针

// 前向声明不能用在以下场景：
// struct Point p;       // ✗ 需要知道 Point 的大小
// p.x = 10;            // ✗ 需要知道 Point 的成员
// sizeof(struct Point)  // ✗ 需要知道完整定义
```

## 原则三：避免循环依赖

### 什么是循环依赖

```
a.h 包含 b.h
b.h 包含 a.h
→ 循环依赖！
```

### 示例：循环依赖

```c
// a.h
#ifndef A_H
#define A_H
#include "b.h"    // a.h 需要 b.h

struct A {
    struct B *b;  // 使用 B 的指针
};
void use_b(struct A *a);
#endif

// b.h
#ifndef B_H
#define B_H
#include "a.h"    // b.h 需要 a.h → 循环！

struct B {
    struct A *a;  // 使用 A 的指针
};
void use_a(struct B *b);
#endif
```

### 解决方案：前向声明

```c
// a.h — 不再包含 b.h
#ifndef A_H
#define A_H

struct B;  // 前向声明

struct A {
    struct B *b;
};
void use_b(struct A *a);
#endif

// b.h — 不再包含 a.h
#ifndef B_H
#define B_H

struct A;  // 前向声明

struct B {
    struct A *a;
};
void use_a(struct B *b);
#endif

// a.c 中包含两个头文件获取完整定义
// a.c
#include "a.h"
#include "b.h"  // 这里才包含 b.h 获取完整定义
```

## 原则四：清晰的接口层次

### 头文件组织结构

```
project/
├── include/           # 公共头文件（对外接口）
│   └── mylib/
│       ├── mylib.h    # 总包含头文件
│       ├── config.h   # 配置
│       ├── types.h    # 类型定义
│       ├── api.h      # 公共 API
│       └── error.h    # 错误码
├── src/               # 源文件和私有头文件
│   ├── internal.h     # 内部头文件（不对外）
│   ├── main.c
│   ├── api.c
│   └── utils.c
└── tests/
    └── test_api.c
```

### 总包含头文件

```c
// include/mylib/mylib.h
#ifndef MYLIB_H
#define MYLIB_H

#include "mylib/config.h"
#include "mylib/types.h"
#include "mylib/api.h"
#include "mylib/error.h"

#define MYLIB_VERSION_MAJOR 1
#define MYLIB_VERSION_MINOR 0
#define MYLIB_VERSION_PATCH 0

#endif /* MYLIB_H */
```

## 原则五：头文件与源文件分离

```c
// math_utils.h — 接口
#ifndef MATH_UTILS_H
#define MATH_UTILS_H

int gcd(int a, int b);
int lcm(int a, int b);
double power(double base, int exp);

#endif

// math_utils.c — 实现
#include "math_utils.h"

int gcd(int a, int b) {
    while (b) { int t = b; b = a % b; a = t; }
    return a;
}

int lcm(int a, int b) {
    return a / gcd(a, b) * b;
}
```

## 原则六：C/C++ 兼容

```c
// myheader.h
#ifndef MYHEADER_H
#define MYHEADER_H

#ifdef __cplusplus
extern "C" {
#endif

void my_function(int x);
typedef struct { int data; } MyStruct;

#ifdef __cplusplus
}
#endif

#endif /* MYHEADER_H */
```

## 包含顺序规范

```c
// 在 .c 文件中：
#include "my_module.h"    // 1. 对应的头文件

#include <stdio.h>        // 2. C 标准库
#include <stdlib.h>
#include <string.h>

#include <openssl/ssl.h>  // 3. 第三方库

#include "internal.h"     // 4. 项目内其他头文件
#include "utils/log.h"
```

这样做的好处：如果 `my_module.h` 缺少某个依赖，编译 `.c` 文件时会立即报错。

## 文档注释

```c
/**
 * @file network.h
 * @brief 网络通信模块接口
 * @author Your Name
 * @version 1.0
 * 
 * 本模块提供 TCP/UDP 网络通信功能。
 * 使用前必须调用 net_init()，使用完毕后调用 net_cleanup()。
 */

/**
 * @brief 初始化网络子系统
 * @return 0 成功，-1 失败
 * @note 线程不安全，请在主线程中调用
 */
int net_init(void);

/**
 * @brief 连接到远程服务器
 * @param host 主机名或 IP 地址
 * @param port 端口号（1-65535）
 * @param timeout_ms 超时毫秒数，0 表示无限等待
 * @return 连接句柄，失败返回 -1
 */
int net_connect(const char *host, int port, int timeout_ms);
```

## 重要注意事项

> **关键点总结**：
> 1. 头文件应该**自包含**——包含它需要的一切依赖
> 2. 使用**前向声明**减少不必要的 `#include`
> 3. **循环依赖**通过前向声明打断，而非互相包含
> 4. 公共接口（`include/`）和私有实现（`src/`）要分离
> 5. C/C++ 混合项目使用 `extern "C"` 包裹头文件内容
> 6. 规范的包含顺序有助于发现依赖问题
