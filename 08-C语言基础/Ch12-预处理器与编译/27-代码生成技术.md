# 27 - 代码生成技术

## 代码生成概述

代码生成是通过工具或预处理器自动产生重复性代码的技术。它能减少手动维护的代码量，避免错误，保持代码一致性。

## 使用宏生成代码

### X-Macro 模式

```c
// 定义数据一次，生成多种代码
#define EVENT_LIST \
    X(ON_CLICK) \
    X(ON_HOVER) \
    X(ON_KEYPRESS)

// 生成枚举
#define X(name) EVENT_##name,
enum EventType {
    EVENT_LIST
    EVENT_COUNT
};
#undef X

// 生成字符串表
#define X(name) #name,
const char *event_names[] = {
    EVENT_LIST
};
#undef X
```

### 函数模板生成

```c
// 为多种类型生成排序函数
#define DEFINE_SORT_FUNC(type, cmp_fn) \
    void sort_##type(type *arr, size_t n) { \
        for (size_t i = 0; i < n - 1; i++) { \
            for (size_t j = i + 1; j < n; j++) { \
                if (cmp_fn(arr[j], arr[i])) { \
                    type tmp = arr[i]; \
                    arr[i] = arr[j]; \
                    arr[j] = tmp; \
                } \
            } \
        } \
    }

// 生成 int 版本
static inline int cmp_int(int a, int b) { return a < b; }
DEFINE_SORT_FUNC(int, cmp_int)

// 生成 double 版本
static inline int cmp_double(double a, double b) { return a < b; }
DEFINE_SORT_FUNC(double, cmp_double)

// 使用
int arr[] = {3, 1, 4, 1, 5};
sort_int(arr, 5);
```

### Getter/Setter 生成

```c
// 为结构体字段生成访问器
#define PROPERTY(struct_type, field_type, field_name) \
    field_type get_##field_name(struct_type *obj) { \
        return obj->field_name; \
    } \
    void set_##field_name(struct_type *obj, field_type value) { \
        obj->field_name = value; \
    }

// 定义结构体
typedef struct {
    int id;
    double score;
    char name[64];
} Student;

// 生成访问器
PROPERTY(Student, int, id)
PROPERTY(Student, double, score)

// 使用
Student s;
set_id(&s, 42);
int id = get_id(&s);
```

## 使用脚本生成代码

### Python 代码生成器

```python
# gen_protocol.py — 生成协议消息处理代码
import sys

messages = [
    ("MSG_LOGIN",    0x01, [("user", "char*"), ("pass", "char*")]),
    ("MSG_LOGOUT",   0x02, []),
    ("MSG_DATA",     0x03, [("data", "uint8_t*"), ("len", "size_t")]),
    ("MSG_ACK",      0x04, [("seq", "uint32_t")]),
]

def generate_enum():
    print("// 自动生成 — 请勿手动修改")
    print("typedef enum {")
    for name, code, _ in messages:
        print(f"    {name} = 0x{code:02X},")
    print("} MessageType;")
    print()

def generate_string_table():
    print("static const char *msg_names[] = {")
    for name, code, _ in messages:
        print(f"    [{name}] = \"{name}\",")
    print("};")
    print()

def generate_handler_stubs():
    for name, code, fields in messages:
        params = ", ".join(f"{ftype} {fname}" for fname, ftype in fields)
        print(f"void handle_{name.lower()}({params});")

if __name__ == "__main__":
    generate_enum()
    generate_string_table()
    generate_handler_stubs()
```

```bash
# 使用
python gen_protocol.py > protocol.h
```

### Shell 脚本生成

```bash
#!/bin/bash
# gen_config.sh — 生成配置定义头文件

cat << 'EOF'
// 自动生成的配置文件 — 请勿手动修改
#ifndef CONFIG_H
#define CONFIG_H
EOF

while IFS='=' read -r key value; do
    [[ -z "$key" || "$key" == \#* ]] && continue
    echo "#define CONFIG_${key^^} $value"
done < config.txt

echo "#endif /* CONFIG_H */"
```

```bash
# config.txt
#BUFFER_SIZE=4096
#MAX_CONNECTIONS=100
#TIMEOUT=30

# 运行
./gen_config.sh > config.h
```

## 代码生成工具

### 使用 AWK 生成枚举和字符串

```bash
# events.def（定义文件）
ON_CONNECT 连接事件
ON_DISCONNECT 断开事件
ON_DATA 数据事件
ON_ERROR 错误事件

# gen_events.awk
BEGIN {
    print "// 自动生成"
    print "typedef enum {"
}
{
    print "    " $1 ","
    names[NR] = $1
    desc[NR] = $2
}
END {
    print "    EVENT_COUNT"
    print "} EventType;"
    print ""
    print "static const char *event_desc[] = {"
    for (i in names)
        print "    [" names[i] "] = \"" desc[i] "\","
    print "};"
}
```

```bash
awk -f gen_events.awk events.def > events.h
```

## Makefile 中的代码生成

```makefile
# Makefile
.PHONY: generate

generate: protocol.h protocol.c

protocol.h protocol.c: protocol.def gen_protocol.py
	python gen_protocol.py protocol.def

# 编译时自动生成
main.o: protocol.h
	$(CC) $(CFLAGS) -c main.c -o main.o

clean:
	rm -f protocol.h protocol.c
```

## 代码生成的最佳实践

### 1. 标记生成的文件

```c
// ═══════════════════════════════════════
// ⚠ 此文件由 gen_protocol.py 自动生成
// ⚠ 请勿手动修改！
// ⚠ 生成时间: 2026-05-05 14:30:00
// ⚠ 源文件: protocol.def
// ═══════════════════════════════════════
```

### 2. 保持源文件可读

```c
// 协议定义文件（比生成的代码更易读和维护）
#define PROTOCOL_MESSAGES \
    X(LOGIN,    0x01, STR, STR) \
    X(LOGOUT,   0x02) \
    X(DATA,     0x03, BYTES, INT) \
    X(ACK,      0x04, INT)
```

### 3. 在版本控制中排除生成的文件

```gitignore
# .gitignore
# 如果生成的文件可以随时重新生成
protocol.h
protocol.c
```

或者保留生成的文件但标记清楚来源。

## 何时使用代码生成

```
场景                          推荐方案
──────────────────────────────────────────────
枚举与字符串同步             X-Macro
多种类型的类似函数            宏模板
协议/消息定义                 脚本生成
配置文件转头文件              构建脚本
复杂的数据结构生成            专用代码生成器
API 绑定/包装器              代码生成工具
```

## 重要注意事项

> **关键点总结**：
> 1. 代码生成的核心价值是**单一数据源**——修改一处，自动更新所有关联代码
> 2. **X-Macro** 适合简单的枚举-字符串同步，是最轻量的方案
> 3. 宏模板适合生成类型相关的函数，但可读性较差
> 4. 脚本生成适合复杂逻辑，且不依赖C预处理器的限制
> 5. 始终在生成的文件中**标注来源和警告**
> 6. 代码生成应在**构建阶段**自动完成，而非手动执行
