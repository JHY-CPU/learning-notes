# 28 - 构建系统对比

## 为什么需要构建系统

当项目从单个 `.c` 文件增长到数十甚至数百个文件时，手动编译变得不可行。构建系统自动化了编译、链接、依赖管理等过程。

```
单文件:           gcc main.c -o main
小项目:           gcc *.c -o project
中大型项目:       需要构建系统
```

## Make

### 概述

Make 是最古老、最广泛使用的构建工具。它通过 `Makefile` 定义规则，基于文件时间戳判断是否需要重新编译。

### Makefile 示例

```makefile
# 变量定义
CC = gcc
CFLAGS = -Wall -Wextra -O2
LDFLAGS = -lm
TARGET = myprogram

# 源文件和目标文件
SRCS = main.c utils.c math_ops.c
OBJS = $(SRCS:.c=.o)

# 默认目标
all: $(TARGET)

# 链接规则
$(TARGET): $(OBJS)
	$(CC) $(OBJS) -o $(TARGET) $(LDFLAGS)

# 通用编译规则
%.o: %.c
	$(CC) $(CFLAGS) -c $< -o $@

# 依赖关系（手动维护或用 -MM 自动生成）
main.o: main.c utils.h
utils.o: utils.c utils.h
math_ops.o: math_ops.c math_ops.h

# 清理
clean:
	rm -f $(OBJS) $(TARGET)

# 伪目标
.PHONY: all clean
```

### Make 的优缺点

**优点**：
- 几乎所有 Unix 系统都有
- 学习曲线适中
- 灵活性极高
- 与 shell 完美集成

**缺点**：
- 语法晦涩（Tab 缩进敏感）
- 跨平台支持差（Windows 需要特殊处理）
- 大型项目维护困难
- 需要手动管理依赖关系

## CMake

### 概述

CMake 是一个跨平台的构建系统生成器。它不直接构建项目，而是生成 Makefile、Ninja 文件、Visual Studio 项目等。

### CMakeLists.txt 示例

```cmake
cmake_minimum_required(VERSION 3.10)
project(MyProject VERSION 1.0 LANGUAGES C)

# 编译选项
set(CMAKE_C_STANDARD 11)
set(CMAKE_C_STANDARD_REQUIRED ON)
add_compile_options(-Wall -Wextra)

# 查找库
find_package(Threads REQUIRED)
find_library(MATH_LIB m)

# 可执行文件
add_executable(myprogram main.c utils.c math_ops.c)

# 链接库
target_link_libraries(myprogram PRIVATE Threads::Threads ${MATH_LIB})

# 安装规则
install(TARGETS myprogram DESTINATION bin)
```

### 多目录项目

```cmake
# CMakeLists.txt（根目录）
cmake_minimum_required(VERSION 3.10)
project(MyProject)

add_subdirectory(src)
add_subdirectory(lib)
add_subdirectory(tests)
```

```cmake
# src/CMakeLists.txt
add_executable(myprogram main.c)
target_link_libraries(myprogram PRIVATE mylib)
```

```cmake
# lib/CMakeLists.txt
add_library(mylib STATIC lib.c helper.c)
target_include_directories(mylib PUBLIC ${CMAKE_CURRENT_SOURCE_DIR}/include)
```

### CMake 的优缺点

**优点**：
- 跨平台（Linux、Windows、macOS）
- 强大的依赖查找机制
- 广泛的 IDE 支持
- 丰富的模块生态（CTest、CPack）

**缺点**：
- 学习曲线陡峭
- 语法不直观（DSL，非通用语言）
- 文档质量参差不齐
- 版本之间行为变化

## Meson

### 概述

Meson 是一个现代构建系统，设计目标是快速、用户友好。它使用 Python 风格的语法。

### meson.build 示例

```meson
project('myproject', 'c',
  version: '1.0.0',
  default_options: ['c_std=c11', 'warning_level=2', 'optimization=2'])

# 依赖
threads = dependency('threads')
m_dep = meson.get_compiler('c').find_library('m', required: false)

# 源文件
sources = files('main.c', 'utils.c', 'math_ops.c')

# 可执行文件
executable('myprogram', sources,
  dependencies: [threads, m_dep],
  install: true)
```

### Meson 的优缺点

**优点**：
- 语法简洁直观
- 构建速度极快
- 默认值合理
- 内置交叉编译支持
- 强制 out-of-source 构建

**缺点**：
- 生态不如 CMake 成熟
- 高级功能相对有限
- 必须配合 Ninja 使用

## Ninja

### 概述

Ninja 是一个专注于速度的小型构建系统。它不直接手写，而是由其他工具（CMake、Meson、GN）生成 Ninja 文件。

### build.ninja 示例

```ninja
# 通常由 CMake 或 Meson 自动生成
cc = gcc
cflags = -Wall -O2

rule compile
  command = $cc $cflags -c $in -o $out
  description = 编译 $in

rule link
  command = $cc $in -o $out
  description = 链接 $out

build main.o: compile main.c
build utils.o: compile utils.c
build myprogram: link main.o utils.o

default myprogram
```

### Ninja 的优缺点

**优点**：
- 构建速度**最快**
- 依赖关系自动计算
- 极简设计，专注于速度

**缺点**：
- 不适合手写构建文件
- 不直接支持复杂逻辑
- 通常作为其他构建系统的后端

## 对比总结

| 特性 | Make | CMake | Meson | Ninja |
|------|------|-------|-------|-------|
| 学习曲线 | 中等 | 陡峭 | 平缓 | N/A |
| 跨平台 | 差 | 好 | 好 | 好 |
| 构建速度 | 慢 | 中等 | 快 | 最快 |
| 语法 | 晦涩 | DSL | Python-like | 声明式 |
| IDE支持 | 有限 | 最好 | 良好 | 间接 |
| 适合规模 | 小-中 | 任意 | 中-大 | 后端 |
| 依赖管理 | 手动 | find_package | dependency() | 自动 |
| 交叉编译 | 困难 | 工具链文件 | 内置 | N/A |

## 选择建议

```
场景                           推荐
──────────────────────────────────────────────
小型项目/脚本                  Make
跨平台项目                     CMake
现代 Linux 项目                Meson + Ninja
大型项目                       CMake 或 Meson + Ninja
嵌入式开发                     CMake（工具链文件支持好）
学习构建系统                   从 Make 开始
CI/CD 环境                     Ninja（速度优势）
```

## 实际项目中的组合

```bash
# CMake + Make（传统方式）
cmake ..
make -j$(nproc)

# CMake + Ninja（推荐方式）
cmake -G Ninja ..
ninja

# Meson + Ninja（现代方式）
meson setup build
ninja -C build
```

## 重要注意事项

> **关键点总结**：
> 1. **Make** 是基础，理解它有助于理解其他构建系统
> 2. **CMake** 是目前最广泛使用的跨平台构建系统生成器
> 3. **Meson** 是现代化的替代品，语法更友好，速度更快
> 4. **Ninja** 是构建执行的后端，速度最快，但不直接手写
> 5. 小项目用 Make 足矣，中大型项目建议 CMake 或 Meson
> 6. CMake + Ninja 是当前的主流组合
