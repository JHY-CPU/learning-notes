# 26 - 编译时计算

## 编译时计算概述

C语言支持在编译阶段进行部分计算，减少运行时开销。虽然C没有C++的 `constexpr`，但通过各种技巧也能实现编译时求值。

## 常量表达式

### C标准支持的编译时常量表达式

```c
// 整数常量表达式（可用于数组大小、case标签等）
#define SIZE 100
int arr[SIZE];                    // ✓ 编译时确定数组大小

enum { BUFFER_SIZE = 1024 };
char buf[BUFFER_SIZE];            // ✓ enum 值是编译时常量

// 算术运算
int arr2[2 * 3 + 4];             // ✓ = 10
int arr3[sizeof(int) * 8];        // ✓ sizeof 是编译时常量

// 条件表达式
#define MAX_SIZE (DEBUG ? 64 : 4096)  // ✓ 编译时常量
```

### C99 变长数组（VLA）

```c
// C99 支持运行时确定数组大小（非编译时计算）
void func(int n) {
    int vla[n];  // VLA：运行时分配
}
```

## enum 技巧

### 基本编译时常量

```c
// enum 值是编译时常量
enum {
    MAX_CONNECTIONS = 100,
    BUFFER_SIZE = 4096,
    TIMEOUT_MS = 5000
};

int connections[MAX_CONNECTIONS];  // ✓ 编译时确定
```

### 枚举中的计算

```c
enum {
    BITS_PER_BYTE = 8,
    BYTES_PER_INT = sizeof(int),
    BITS_PER_INT = BITS_PER_BYTE * BYTES_PER_INT,  // 编译时计算
    MAX_INT_BITS = BITS_PER_INT - 1,
    HALF_INT_BITS = BITS_PER_INT / 2
};

// 运行时：
// BITS_PER_INT = 8 * 4 = 32（32位系统）
```

### 枚举用于数组大小

```c
enum {
    LOG_LEVELS = 4,
    MAX_MSG_LEN = 256,
    RING_BUFFER_SIZE = 1 << 10  // 1024
};

static const char *level_names[LOG_LEVELS];  // ✓
char msg_buffer[MAX_MSG_LEN];                // ✓
int ring_buf[RING_BUFFER_SIZE];              // ✓
```

## sizeof 编译时计算

```c
// sizeof 是编译时操作符
size_t int_size = sizeof(int);           // 编译时常量
size_t struct_size = sizeof(struct Point); // 编译时常量
size_t array_size = sizeof(arr) / sizeof(arr[0]); // 编译时常量

// 用于数组大小
char buffer[sizeof(int) * 100];  // ✓ 编译时确定
```

## 预处理器计算

### 宏计算

```c
// 预处理器进行算术运算
#define KB (1024)
#define MB (1024 * KB)
#define GB (1024 * MB)

char buffer[4 * KB];             // 编译时计算为 4096
char pool[16 * MB];              // 编译时计算为 16777216

// 位运算
#define FLAG_READ   (1 << 0)    // 1
#define FLAG_WRITE  (1 << 1)    // 2
#define FLAG_EXEC   (1 << 2)    // 4
#define FLAG_ALL    (FLAG_READ | FLAG_WRITE | FLAG_EXEC)  // 7
```

### 字符串长度（有限）

```c
// 字符串字面量长度是编译时可计算的
#define STR "Hello"
char buf[sizeof(STR)];  // sizeof 包含 null 终止符 = 6

// 但 strlen() 是运行时函数
size_t len = strlen(STR);  // 运行时计算
```

## static_assert（C11）

C11 的 `_Static_assert` 在编译时检查条件：

```c
#include <assert.h>

_Static_assert(sizeof(int) == 4, "int must be 4 bytes");
_Static_assert(sizeof(long) >= sizeof(int), "long must be at least as large as int");

// 用于结构体验证
struct Header {
    uint32_t magic;
    uint16_t version;
    uint8_t flags;
    uint8_t reserved;
};
_Static_assert(sizeof(struct Header) == 8, "Header must be 8 bytes");
```

## 编译时断言技巧（C89）

```c
// 方法1: 数组大小负数（不合法）
#define STATIC_ASSERT(cond, msg) \
    typedef char static_assert_##msg[(cond) ? 1 : -1]

STATIC_ASSERT(sizeof(int) == 4, int_size_must_be_4);

// 方法2: switch case
#define STATIC_ASSERT2(cond) \
    switch(0) { case 0: case cond: ; }

// 方法3: 位域大小
#define STATIC_ASSERT3(cond) \
    struct { unsigned int flag : (cond) ? 1 : -1; }
```

## 常见的编译时计算模式

### 1. 对齐计算

```c
// 编译时对齐到 2 的幂次
#define ALIGN(x, a) (((x) + (a) - 1) & ~((a) - 1))

enum { ALIGNED_SIZE = ALIGN(100, 16) };  // = 112
char buffer[ALIGNED_SIZE];

// 验证
_Static_assert(ALIGNED_SIZE == 112, "alignment error");
```

### 2. 位操作

```c
// 编译时计算位掩码
#define BIT(n) (1u << (n))
#define MASK(start, len) (((1u << (len)) - 1) << (start))

enum {
    FIELD_A_BITS = 4,
    FIELD_B_BITS = 8,
    FIELD_A_MASK = MASK(0, FIELD_A_BITS),                    // 0x0F
    FIELD_B_MASK = MASK(FIELD_A_BITS, FIELD_B_BITS),         // 0xFF0
    FIELD_A_SHIFT = 0,
    FIELD_B_SHIFT = FIELD_A_BITS
};
```

### 3. 日志级别

```c
enum {
    LOG_OFF   = 0,
    LOG_ERROR = 1,
    LOG_WARN  = 2,
    LOG_INFO  = 3,
    LOG_DEBUG = 4,
    LOG_TRACE = 5,
    LOG_LEVEL = 3  // 编译时设置
};

// 根据级别生成不同的代码
#if LOG_LEVEL >= LOG_DEBUG
    #define LOG_DEBUG_MSG(fmt, ...) printf("[DEBUG] " fmt "\n", ##__VA_ARGS__)
#else
    #define LOG_DEBUG_MSG(fmt, ...)  // 编译时消除
#endif
```

### 4. 查找表生成

```c
// 编译时生成 CRC 查找表
#define CRC_POLY 0xEDB88320

#define CRC_ENTRY(n) \
    ((n) & 1) ? (((n) >> 1) ^ CRC_POLY) : ((n) >> 1)

// 实际中通常用脚本生成完整的 256 项表
// 这里演示部分
static const uint32_t crc_table[256] = {
    [0] = 0,
    [1] = CRC_POLY,
    // ... 其余项由脚本或手动填写
};
```

## 与 C++ constexpr 的对比

```c
// C 语言：使用 enum 和宏
enum { MAX = 100 };
#define SQUARE(x) ((x) * (x))
int arr[SQUARE(10)];  // ✓ 编译时计算

// C++ constexpr（对比）
// constexpr int square(int x) { return x * x; }
// int arr[square(10)];  // ✓ 编译时计算
// C++ 更灵活，但 C 通过技巧也能实现大部分需求
```

## 重要注意事项

> **关键点总结**：
> 1. C语言中 `enum` 值、`sizeof`、常量表达式都是编译时可计算的
> 2. **enum 技巧**是C89中最实用的编译时计算手段
> 3. 宏只做文本替换和简单算术，不能做复杂逻辑计算
> 4. `_Static_assert`（C11）在编译时验证条件
> 5. 对齐、位操作、数组大小等是编译时计算的典型应用
> 6. 对于复杂的编译时计算，考虑用代码生成器预计算
