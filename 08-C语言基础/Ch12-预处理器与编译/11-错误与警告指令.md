# 11 - 错误与警告指令

## #error 指令

`#error` 指令在预处理阶段产生一条**编译错误**，终止编译过程。它常用于检测不满足编译条件的情况。

### 基本语法

```c
#error 错误消息文本
```

### 基本示例

```c
#error "此文件不应被直接编译"

#if !defined(CONFIG_H)
    #error "请先包含 config.h"
#endif

#if __STDC_VERSION__ < 201112L
    #error "本项目需要 C11 或更高版本的编译器"
#endif
```

### 实际应用场景

#### 1. 平台检查

```c
#if !defined(_WIN32) && !defined(__linux__) && !defined(__APPLE__)
    #error "不支持的平台。仅支持 Windows、Linux 和 macOS。"
#endif

#ifdef __i386__
    #error "本程序不支持 32 位系统，请使用 64 位编译器"
#endif
```

#### 2. 编译器版本检查

```c
#if defined(__GNUC__) && (__GNUC__ < 5)
    #error "需要 GCC 5.0 或更高版本"
#endif

#if defined(_MSC_VER) && (_MSC_VER < 1900)
    #error "需要 Visual Studio 2015 或更高版本"
#endif

#ifndef __STDC__
    #error "不支持非标准 C 编译器"
#endif
```

#### 3. 必需依赖检查

```c
#ifndef HAS_OPENSSL
    #error "编译本模块需要 OpenSSL 库。请安装 libssl-dev。"
#endif

#if !defined(HAVE_STDBOOL_H) && __STDC_VERSION__ < 199901L
    #error "编译器缺少 stdbool.h 且不支持 C99"
#endif
```

#### 4. 配置完整性检查

```c
#ifdef CONFIG_MAX_CONNECTIONS
    #if CONFIG_MAX_CONNECTIONS <= 0
        #error "CONFIG_MAX_CONNECTIONS 必须大于 0"
    #endif
    #if CONFIG_MAX_CONNECTIONS > 65535
        #error "CONFIG_MAX_CONNECTIONS 不能超过 65535"
    #endif
#else
    #error "未定义 CONFIG_MAX_CONNECTIONS"
#endif
```

#### 5. 编译时断言

```c
// 在预处理阶段检查类型的大小
#if sizeof(int) < 4  // 注意：这实际上不能工作！
    #error "int 类型至少需要 4 字节"
#endif

// 正确的方法：使用 enum 或 typedef 技巧
typedef char check_int_size[sizeof(int) >= 4 ? 1 : -1];
```

## #warning 指令（C99）

`#warning` 指令在预处理阶段产生一条**编译警告**，但**不会**终止编译。

### 基本语法

```c
#warning 警告消息文本
```

### 基本示例

```c
#warning "此代码尚未经过充分测试"

#if defined(ENABLE_EXPERIMENTAL)
    #warning "实验性功能已启用，可能不稳定"
#endif
```

### 实际应用场景

#### 1. 过期功能提醒

```c
#ifdef USE_OLD_API
    #warning "USE_OLD_API 已过期，请迁移到新 API"
    // 旧 API 实现...
#endif
```

#### 2. 配置建议

```c
#if !defined(NDEBUG) && defined(OPTIMIZE_SPEED)
    #warning "调试模式下启用速度优化可能影响调试效果"
#endif

#ifdef DEBUG
    #ifndef ENABLE_LOGGING
        #warning "调试模式下建议启用 ENABLE_LOGGING"
    #endif
#endif
```

#### 3. 已知问题提醒

```c
#if defined(__GNUC__) && (__GNUC__ == 12)
    #warning "GCC 12 在某些优化场景下存在已知问题，建议升级"
#endif

#if __STDC_VERSION__ == 201112L
    #warning "建议使用 C17 标准以获得更好的兼容性"
#endif
```

#### 4. 版本兼容警告

```c
#ifdef USE_DEPRECATED_FEATURE
    #warning "USE_DEPRECATED_FEATURE 将在 v3.0 中移除"
#endif

#if !defined(USE_NEW_PARSER)
    #warning "旧版解析器将在下一版本中移除，请使用 USE_NEW_PARSER"
#endif
```

## #error 和 #warning 的组合使用

```c
// 完整的编译时检查系统
#ifndef BUILD_CONFIG_H
#define BUILD_CONFIG_H

// 必需的检查（错误）
#if !defined(PROJECT_NAME)
    #error "必须定义 PROJECT_NAME"
#endif

#if __STDC_VERSION__ < 201112L
    #error "项目需要 C11 标准。请使用 -std=c11 编译选项。"
#endif

// 可选的检查（警告）
#ifndef BUFFER_SIZE
    #warning "未定义 BUFFER_SIZE，使用默认值 4096"
    #define BUFFER_SIZE 4096
#endif

#ifndef MAX_THREADS
    #warning "未定义 MAX_THREADS，使用默认值 4"
    #define MAX_THREADS 4
#endif

// 条件警告
#if defined(ENABLE_DEBUG) && defined(NDEBUG)
    #warning "同时定义了 ENABLE_DEBUG 和 NDEBUG，调试输出将被禁用"
#endif

#endif /* BUILD_CONFIG_H */
```

## 跨编译器兼容性

```c
// #warning 不是所有编译器都支持
#if defined(__GNUC__) || defined(__clang__)
    #define COMPILER_WARNING(msg) #warning msg
#elif defined(_MSC_VER)
    // MSVC 使用 #pragma message
    #define COMPILER_WARNING(msg) __pragma(message("WARNING: " msg))
#else
    #define COMPILER_WARNING(msg)
#endif

COMPILER_WARNING("此模块还未完成测试")
```

## 重要注意事项

> **关键点总结**：
> 1. `#error` 终止编译，用于**必须满足**的条件检查
> 2. `#warning` 只产生警告，编译继续，用于**建议性**的提醒
> 3. `#warning` 是 C99 标准引入的，不是所有编译器都支持
> 4. 错误/警告消息应清晰描述**问题是什么**以及**如何解决**
> 5. 善用这些指令进行编译时检查，及早发现问题
> 6. 消息文本可以用引号包裹，也可以不用
