# 25 - 交叉编译基础

## 交叉编译概念

交叉编译（Cross Compilation）是指在一种平台（**宿主机**，Host）上编译出在另一种平台（**目标机**，Target）上运行的可执行文件。

```
宿主机（Host）          目标机（Target）
x86_64 Linux     →    ARM64 Linux
Windows          →    Linux
macOS            →    ARM embedded
```

### 常见应用场景

- **嵌入式开发**：在 PC 上编译嵌入式设备代码
- **操作系统开发**：在 Linux 上编译 Windows 程序
- **移动开发**：在 x86 开发机上编译 ARM 应用
- **多平台发布**：一次编译多平台版本

## 工具链

### 工具链组成

交叉编译工具链包含目标平台的编译器、链接器和库：

```
arm-linux-gnueabihf-gcc      → C 编译器
arm-linux-gnueabihf-g++      → C++ 编译器
arm-linux-gnueabihf-as       → 汇编器
arm-linux-gnueabihf-ld       → 链接器
arm-linux-gnueabihf-ar       → 归档工具
arm-linux-gnueabihf-strip    → 符号剥离
arm-linux-gnueabihf-gdb      → 调试器
```

### 命名规则

工具链名称格式：`架构-厂商-系统-ABI`

```
arm-linux-gnueabihf-gcc
│    │      │      │
│    │      │      └── ABI: hard float
│    │      └── 系统: GNU + EABI
│    └── 厂商/OS: Linux
└── 目标架构: ARM

aarch64-linux-gnu-gcc        → ARM64 Linux
x86_64-w64-mingw32-gcc       → 64位 Windows
arm-none-eabi-gcc             → ARM 裸机（无 OS）
mips-linux-gnu-gcc           → MIPS Linux
riscv64-linux-gnu-gcc        → RISC-V 64位 Linux
```

## 安装交叉编译工具链

### Ubuntu/Debian

```bash
# ARM 32位
sudo apt install gcc-arm-linux-gnueabihf g++-arm-linux-gnueabihf

# ARM 64位
sudo apt install gcc-aarch64-linux-gnu g++-aarch64-linux-gnu

# Windows 交叉编译（MinGW）
sudo apt install gcc-mingw-w64

# MIPS
sudo apt install gcc-mips-linux-gnu

# RISC-V
sudo apt install gcc-riscv64-linux-gnu
```

### Arch Linux

```bash
sudo pacman -S aarch64-linux-gnu-gcc
# 或通过 AUR
yay -S arm-linux-gnueabihf-gcc
```

## 使用交叉编译工具链

### 基本编译

```c
// hello.c
#include <stdio.h>
int main(void) {
    printf("Hello, Cross Compilation!\n");
    return 0;
}
```

```bash
# 编译为 ARM64 Linux 可执行文件
aarch64-linux-gnu-gcc hello.c -o hello_arm64

# 编译为 ARM32 可执行文件
arm-linux-gnueabihf-gcc hello.c -o hello_arm32

# 编译为 Windows 64位可执行文件
x86_64-w64-mingw32-gcc hello.c -o hello.exe

# 验证文件类型
file hello_arm64
# 输出: ELF 64-bit LSB executable, ARM aarch64 ...

file hello.exe
# 输出: PE32+ executable (console) x86-64 ...
```

### 使用 QEMU 运行

```bash
# 安装 QEMU 用户态模拟
sudo apt install qemu-user

# 运行 ARM64 程序
qemu-aarch64 ./hello_arm64

# 运行 ARM32 程序
qemu-arm ./hello_arm32
```

## CMake 交叉编译

### 工具链文件

```cmake
# arm-toolchain.cmake
set(CMAKE_SYSTEM_NAME Linux)
set(CMAKE_SYSTEM_PROCESSOR arm)

set(CMAKE_C_COMPILER arm-linux-gnueabihf-gcc)
set(CMAKE_CXX_COMPILER arm-linux-gnueabihf-g++)

set(CMAKE_FIND_ROOT_PATH /usr/arm-linux-gnueabihf)
set(CMAKE_FIND_ROOT_PATH_MODE_PROGRAM NEVER)
set(CMAKE_FIND_ROOT_PATH_MODE_LIBRARY ONLY)
set(CMAKE_FIND_ROOT_PATH_MODE_INCLUDE ONLY)
```

```bash
# 使用工具链文件
cmake -DCMAKE_TOOLCHAIN_FILE=arm-toolchain.cmake ..
make
```

### 多平台构建

```bash
# 构建目录分离
mkdir build-arm && cd build-arm
cmake -DCMAKE_TOOLCHAIN_FILE=../arm-toolchain.cmake ..
make

mkdir build-aarch64 && cd build-aarch64
cmake -DCMAKE_TOOLCHAIN_FILE=../aarch64-toolchain.cmake ..
make
```

## Makefile 中使用

```makefile
# Makefile
CC_ARM = arm-linux-gnueabihf-gcc
CC_AARCH64 = aarch64-linux-gnu-gcc
CC_MINGW = x86_64-w64-mingw32-gcc

CFLAGS = -Wall -O2

all: hello_arm hello_aarch64 hello_win

hello_arm: hello.c
	$(CC_ARM) $(CFLAGS) $< -o $@

hello_aarch64: hello.c
	$(CC_AARCH64) $(CFLAGS) $< -o $@

hello_win: hello.c
	$(CC_MINGW) $(CFLAGS) $< -o $@.exe

clean:
	rm -f hello_arm hello_aarch64 hello_win.exe
```

## 链接目标平台的库

```bash
# 指定目标平台库路径
aarch64-linux-gnu-gcc main.c \
    -L/usr/aarch64-linux-gnu/lib \
    -I/usr/aarch64-linux-gnu/include \
    -lpthread -lm \
    -o main_arm64

# 使用 sysroot（目标平台的根文件系统）
aarch64-linux-gnu-gcc --sysroot=/path/to/arm-sysroot \
    main.c -o main_arm64
```

## 常见问题

### 头文件找不到

```bash
# 安装目标平台的开发库
sudo apt install linux-libc-dev-arm64-cross

# 或指定 sysroot
--sysroot=/path/to/target-sysroot
```

### 库找不到

```bash
# 查找库的搜索路径
aarch64-linux-gnu-gcc -print-search-dirs

# 添加库搜索路径
-L/usr/aarch64-linux-gnu/lib
```

## 重要注意事项

> **关键点总结**：
> 1. 交叉编译的核心是使用**目标平台的工具链**
> 2. 工具链命名格式：`架构-厂商-系统-ABI`
> 3. CMake 通过**工具链文件**配置交叉编译
> 4. 使用 `file` 命令验证生成的可执行文件类型
> 5. QEMU 可以在宿主机上模拟运行目标平台的程序
> 6. 交叉编译时注意头文件和库的搜索路径指向目标平台
