# 01 - 预处理器概述

## 什么是预处理器

C语言预处理器（Preprocessor）是编译过程中的第一个阶段，它在实际编译之前对源代码进行文本级别的处理。预处理器并不理解C语言的语法，它只进行简单的**文本替换**和**条件过滤**操作。

预处理器处理的指令以 `#` 符号开头，通常写在行首（前面可以有空格或制表符）。

## 编译过程概览

```
源代码 (.c) → [预处理] → [编译] → [汇编] → [链接] → 可执行文件
                 ↑
            本章重点
```

各阶段说明：

| 阶段 | 输入 | 输出 | 主要工作 |
|------|------|------|----------|
| 预处理 | .c 源文件 | 纯C代码（无#指令） | 宏展开、头文件包含、条件编译 |
| 编译 | 预处理后的代码 | 汇编代码 (.s) | 词法分析、语法分析、语义分析、优化 |
| 汇编 | 汇编代码 | 目标文件 (.o/.obj) | 将汇编指令翻译为机器码 |
| 链接 | 多个目标文件 + 库 | 可执行文件 | 符号解析、地址重定位 |

## 预处理指令分类

C语言预处理器提供以下几类指令：

### 1. 文件包含指令
```c
#include <stdio.h>    // 包含系统头文件
#include "myheader.h" // 包含用户头文件
```

### 2. 宏定义指令
```c
#define PI 3.14159           // 对象宏（宏常量）
#define MAX(a, b) ((a)>(b)?(a):(b))  // 函数宏（带参数的宏）
#undef PI                     // 取消宏定义
```

### 3. 条件编译指令
```c
#ifdef DEBUG
    printf("调试信息\n");
#endif

#if defined(WIN32) || defined(_WIN32)
    // Windows平台代码
#elif defined(__linux__)
    // Linux平台代码
#else
    // 其他平台
#endif
```

### 4. 其他指令
```c
#error "编译错误消息"       // 产生编译错误
#warning "编译警告消息"     // 产生编译警告（C99）
#line 100 "newfile.c"      // 修改行号和文件名
#pragma once               // 编译器特定指令
```

## 预处理器与编译器的区别

| 特性 | 预处理器 | 编译器 |
|------|----------|--------|
| 工作层次 | 文本级别 | 语法/语义级别 |
| 理解C语法 | 否 | 是 |
| 主要操作 | 文本替换、条件过滤 | 代码分析、优化、生成目标代码 |
| 错误检测 | 有限（如宏错误） | 完整的语法和语义检查 |
| 输出 | 纯文本 | 汇编代码或机器码 |

## 查看预处理结果

使用编译器的 `-E` 选项可以只运行预处理阶段，查看预处理后的结果：

```bash
# GCC
gcc -E source.c -o source.i

# 查看预处理结果（输出到终端）
gcc -E source.c
```

示例：观察宏展开

```c
// source.c
#define SQUARE(x) ((x) * (x))
int result = SQUARE(5);
```

预处理后的结果：
```c
int result = ((5) * (5));
```

## 预处理的执行顺序

预处理器按以下顺序处理指令：

1. **三字符组替换**（trigraphs，已废弃）：`??=` → `#` 等
2. **行拼接**：将以 `\` 结尾的行与下一行合并
3. **预处理记号化**：将源文件分解为预处理记号
4. **宏展开和include处理**：按遇到的顺序处理
5. **条件编译处理**：根据条件保留或删除代码段
6. **移除注释**：将所有注释替换为一个空格
7. **字符串拼接**：相邻字符串字面量的拼接

## 预处理的实际应用

```c
// 跨平台兼容
#ifdef _WIN32
    #include <windows.h>
    #define PATH_SEPARATOR '\\'
#else
    #include <unistd.h>
    #define PATH_SEPARATOR '/'
#endif

// 调试宏
#ifdef DEBUG
    #define LOG(msg) fprintf(stderr, "[%s:%d] %s\n", __FILE__, __LINE__, msg)
#else
    #define LOG(msg)  // 发布版本中不产生任何代码
#endif

// 版本控制
#if __STDC_VERSION__ >= 201112L
    // 使用C11特性
    #include <stdatomic.h>
#elif __STDC_VERSION__ >= 199901L
    // 使用C99特性
#else
    // C89/90兼容代码
#endif
```

## 重要注意事项

> **关键点**：预处理器是纯粹的文本处理工具，它不认识C语言的变量、类型或作用域。这意味着：
> - 宏展开可能导致意外的行为（如多次求值问题）
> - 宏没有类型检查
> - 宏定义的作用域从定义处开始到文件结束（或 `#undef` 处）
> - 预处理器指令末尾不需要分号（`;`），因为它们不是C语句

## 预处理与编译的根本区别

```c
// 这个例子说明预处理是文本替换
#define ADD_ONE(x) x + 1
int a = 10;
int b = ADD_ONE(a) * 2;  // 展开为: int b = a + 1 * 2;  // 结果是 12，不是 22！

// 正确的宏写法
#define ADD_ONE_CORRECT(x) ((x) + 1)
int c = ADD_ONE_CORRECT(a) * 2;  // 展开为: int c = ((a) + 1) * 2;  // 结果是 22
```

## 小结

预处理器是C语言编译流程中不可或缺的组成部分。它的核心功能包括：

1. **头文件包含** —— 复用已有代码
2. **宏定义** —— 创建常量和代码模板
3. **条件编译** —— 根据条件选择性编译代码
4. **其他控制** —— 错误报告、行号控制等

理解预处理器的工作原理对于编写可移植、可维护的C程序至关重要。后续章节将详细讲解每个预处理指令的用法和最佳实践。
