# 17 - 编译过程深入

## 编译四阶段模型

C语言的编译过程可以分为四个主要阶段：预处理（Preprocessing）、编译（Compilation）、汇编（Assembly）和链接（Linking）。理解每个阶段对于调试、优化和理解构建系统至关重要。

## 阶段一：预处理（Preprocessing）

### 输入与输出
- **输入**：`.c` 源文件 + 头文件
- **输出**：预处理后的 `.i` 文件（纯C代码）

### 处理内容

1. 展开所有 `#include` 指令
2. 展开所有宏定义
3. 处理条件编译指令
4. 删除注释（替换为空格）
5. 处理 `#line` 指令
6. 拼接相邻字符串字面量

### 查看预处理结果

```bash
# 只运行预处理
gcc -E source.c -o source.i

# 带搜索路径的预处理
gcc -E -I./include source.c -o source.i

# 查看宏展开
gcc -E -DDEBUG=1 source.c
```

### 预处理示例

```c
// source.c
#include <stdio.h>
#define SQUARE(x) ((x) * (x))

int main(void) {
    int x = SQUARE(5);
    printf("x = %d\n", x);
    return 0;
}
```

```c
// 预处理后的 source.i（简化）
// （stdio.h 的数千行内容被粘贴在这里）

int main(void) {
    int x = ((5) * (5));
    printf("x = %d\n", x);
    return 0;
}
```

## 阶段二：编译（Compilation）

### 输入与输出
- **输入**：预处理后的 `.i` 文件
- **输出**：汇编代码 `.s` 文件

### 编译子阶段

#### 1. 词法分析（Lexical Analysis）

将源代码分解为**记号（Token）**流：

```c
int x = 42 + y;
```

```
Token流: [INT] [IDENTIFIER:x] [ASSIGN] [NUMBER:42] [PLUS] [IDENTIFIER:y] [SEMICOLON]
```

#### 2. 语法分析（Parsing / Syntax Analysis）

将记号流组织为**抽象语法树（AST）**：

```
       Declaration
      /     |      \
    Type   Var     Init
    int     x    BinOp(+)
                 /     \
              Num(42)  Ident(y)
```

#### 3. 语义分析（Semantic Analysis）

检查类型匹配、变量声明、作用域规则等：

```c
int x = "hello";  // 语义错误：类型不匹配
```

#### 4. 中间代码生成（IR Generation）

生成中间表示（如三地址码、SSA形式）：

```c
// 源代码
int result = (a + b) * (c - d);

// 可能的中间代码（三地址码）
t1 = a + b
t2 = c - d
result = t1 * t2
```

#### 5. 优化

多遍优化，包括：
- 常量折叠：`3 + 4` → `7`
- 死代码消除
- 循环优化
- 函数内联
- 寄存器分配

#### 6. 代码生成

生成目标架构的汇编代码：

```bash
# 只运行到编译阶段（生成汇编）
gcc -S source.c -o source.s

# 带优化的汇编输出
gcc -S -O2 source.c -o source.s

# 使用 Intel 语法（GCC）
gcc -S -masm=intel source.c -o source.s
```

### 查看汇编输出

```c
// source.c
int add(int a, int b) {
    return a + b;
}
```

```asm
; x86-64 汇编（GCC -O2）
add:
    lea     eax, [rdi+rsi]
    ret
```

## 阶段三：汇编（Assembly）

### 输入与输出
- **输入**：汇编代码 `.s` 文件
- **输出**：目标文件 `.o` / `.obj`

### 处理内容

汇编器将汇编指令一一翻译为机器码：

```bash
# 只运行汇编阶段
gcc -c source.c -o source.o

# 或从汇编文件
gcc -c source.s -o source.o
```

### 目标文件格式

目标文件（ELF、COFF、Mach-O）包含：

| 段（Section） | 内容 |
|--------------|------|
| `.text` | 机器码（函数的二进制指令） |
| `.data` | 已初始化的全局/静态变量 |
| `.bss` | 未初始化的全局/静态变量 |
| `.rodata` | 只读数据（字符串常量等） |
| `.symtab` | 符号表（函数名、全局变量名） |
| `.rel.text` | 代码段的重定位信息 |
| `.rel.data` | 数据段的重定位信息 |
| `.strtab` | 字符串表 |
| `.debug` | 调试信息（使用 -g 时） |

### 查看目标文件信息

```bash
# 查看符号表
nm source.o

# 查看所有段
objdump -h source.o

# 反汇编
objdump -d source.o

# 查看详细信息
readelf -a source.o  # Linux
```

## 阶段四：链接（Linking）

### 输入与输出
- **输入**：多个 `.o` 文件 + 静态库 `.a` / 动态库 `.so`
- **输出**：可执行文件

### 处理内容

1. **符号解析**：将每个符号引用与符号定义关联
2. **重定位**：根据最终地址修改代码和数据中的引用
3. **库链接**：从库中提取需要的目标文件

```bash
# 链接多个目标文件
gcc main.o utils.o -o program

# 链接库
gcc main.o -lm -lpthread -o program

# 查看完整编译过程（显示每个阶段）
gcc -v main.c -o main
```

## 完整编译过程示例

```bash
# 项目结构：
# main.c
# math_ops.c → math_ops.h
# string_ops.c → string_ops.h

# 方法1: 分步编译
gcc -E main.c -o main.i          # 预处理
gcc -S main.i -o main.s          # 编译
gcc -c main.s -o main.o          # 汇编

gcc -E math_ops.c -o math_ops.i
gcc -S math_ops.i -o math_ops.s
gcc -c math_ops.s -o math_ops.o

gcc -E string_ops.c -o string_ops.i
gcc -S string_ops.i -o string_ops.s
gcc -c string_ops.s -o string_ops.o

gcc main.o math_ops.o string_ops.o -o program  # 链接

# 方法2: 一步完成
gcc main.c math_ops.c string_ops.c -o program

# 方法3: 分别编译再链接
gcc -c main.c -o main.o
gcc -c math_ops.c -o math_ops.o
gcc -c string_ops.c -o string_ops.o
gcc main.o math_ops.o string_ops.o -o program
```

## 编译依赖图

```
main.c           math_ops.c       string_ops.c
   |                |                  |
   v                v                  v
main.i          math_ops.i       string_ops.i     (预处理)
   |                |                  |
   v                v                  v
main.s          math_ops.s       string_ops.s     (编译)
   |                |                  |
   v                v                  v
main.o          math_ops.o       string_ops.o     (汇编)
   \                |                  /
    \               |                 /
     \              |                /
      v             v               v
           program (可执行文件)          (链接)
```

## 重要注意事项

> **关键点总结**：
> 1. 预处理是文本替换，编译是语法分析和代码生成
> 2. 每个 `.c` 文件独立编译为 `.o` 文件（翻译单元概念）
> 3. `gcc -E` 查看预处理，`gcc -S` 查看汇编，`gcc -c` 生成目标文件
> 4. 链接器负责**符号解析**和**地址重定位**
> 5. 目标文件包含代码段、数据段、符号表和重定位信息
> 6. 理解编译过程有助于高效调试和优化
