# 18 - 编译器优化选项

## 优化级别概述

编译器优化是在编译过程中对生成的机器码进行改进，以提高程序的运行速度或减小代码体积。GCC/Clang 提供了多个优化级别，从 `-O0`（不优化）到 `-O3`（高度优化）。

## 各优化级别详解

### -O0：不优化（默认）

```bash
gcc -O0 main.c -o main
```

- 编译速度最快
- 生成的代码与源代码结构最接近
- 调试体验最好
- 每个变量都实际分配内存
- 每个函数调用都实际执行

适用场景：**开发和调试阶段**

### -O1：基本优化

```bash
gcc -O1 main.c -o main
```

执行的优化包括：
- **常量折叠**：编译时计算常量表达式
- **死代码消除**：删除永远不会执行的代码
- **公共子表达式消除**：避免重复计算
- **简单的寄存器分配**

```c
// 常量折叠示例
int x = 3 + 4 * 5;  // -O0: 实际计算 | -O1: 直接替换为 23

// 死代码消除
int process(int x) {
    int unused = x * 2;  // -O1: 被删除
    return x + 1;
}
```

### -O2：推荐优化

```bash
gcc -O2 main.c -o main
```

在 `-O1` 基础上增加：
- **函数内联**：小函数被内联到调用处
- **循环展开**：减少循环控制开销
- **循环优化**：强度削减、归纳变量消除
- **指令调度**：重排指令以利用 CPU 流水线
- **分支预测优化**

```c
// 函数内联示例
static inline int square(int x) { return x * x; }
int result = square(5);  // -O2: 直接替换为 5 * 5

// 循环展开
for (int i = 0; i < 4; i++) {
    arr[i] = 0;
}
// -O2 可能展开为:
// arr[0] = 0; arr[1] = 0; arr[2] = 0; arr[3] = 0;
```

适用场景：**发布版本的默认选择**

### -O3：高度优化

```bash
gcc -O3 main.c -o main
```

在 `-O2` 基础上增加：
- **向量化**：使用 SIMD 指令（SSE、AVX）
- **更激进的循环优化**
- **函数克隆**：为不同调用场景生成专用版本
- **自动并行化**（某些编译器）

```c
// 向量化示例
void add_arrays(float *a, float *b, float *c, int n) {
    for (int i = 0; i < n; i++) {
        c[i] = a[i] + b[i];
    }
}
// -O3: 可能使用 SIMD 指令一次处理 4/8 个 float
```

**注意**：`-O3` 可能增大代码体积，某些情况下反而变慢。

### -Os：优化代码大小

```bash
gcc -Os main.c -o main
```

- 类似 `-O2`，但优先减小代码体积
- 避免增大代码的优化（如循环展开、函数内联）
- 适用于嵌入式系统或对代码大小敏感的场景

### -Og：调试友好的优化

```bash
gcc -Og main.c -o main
```

- 提供不影响调试体验的优化
- 比 `-O0` 生成更好的代码
- 比 `-O2` 更容易调试
- 推荐用于**开发阶段**

### -Ofast：最激进优化

```bash
gcc -Ofast main.c -o main
```

- 包含 `-O3` 的所有优化
- 可能违反严格的 IEEE 浮点标准
- 允许更激进的数学优化
- **可能改变浮点计算结果**

## 特定优化选项

```bash
# 单独启用特定优化
gcc -finline-functions main.c -o main
gcc -funroll-loops main.c -o main
gcc -ftree-vectorize main.c -o main

# 单独禁用特定优化
gcc -O2 -fno-inline main.c -o main       # 禁用内联
gcc -O2 -fno-unroll-loops main.c -o main # 禁用循环展开
```

## 查看优化效果

```bash
# 查看汇编代码对比
gcc -O0 -S main.c -o main_O0.s
gcc -O2 -S main.c -o main_O2.s
diff main_O0.s main_O2.s

# 查看编译器执行了哪些优化
gcc -O2 -fopt-info-optimized main.c -o main

# 查看哪些优化被跳过
gcc -O2 -fopt-info-missed main.c -o main
```

## 性能测试示例

```c
// test_optimization.c
#include <stdio.h>
#include <time.h>

#define N 100000000

double compute(double *arr, int n) {
    double sum = 0.0;
    for (int i = 0; i < n; i++) {
        sum += arr[i] * arr[i] + 1.0;
    }
    return sum;
}

int main(void) {
    double arr[N];
    for (int i = 0; i < N; i++) arr[i] = (double)i;
    
    clock_t start = clock();
    double result = compute(arr, N);
    clock_t end = clock();
    
    printf("结果: %.2f\n", result);
    printf("时间: %.3f 秒\n", (double)(end - start) / CLOCKS_PER_SEC);
    return 0;
}
```

```bash
# 不同优化级别的性能对比
gcc -O0 test_optimization.c -o test_O0 && time ./test_O0
gcc -O1 test_optimization.c -o test_O1 && time ./test_O1
gcc -O2 test_optimization.c -o test_O2 && time ./test_O2
gcc -O3 test_optimization.c -o test_O3 && time ./test_O3
```

## 函数级优化控制

```c
// GCC 属性控制单个函数的优化
__attribute__((optimize("O3")))
void hot_function(int *data, int n) {
    // 这个函数使用 -O3 优化
    for (int i = 0; i < n; i++)
        data[i] *= 2;
}

__attribute__((optimize("O0")))
void debug_function(void) {
    // 这个函数不优化
}

// 禁用特定优化
__attribute__((optimize("no-tree-vectorize")))
void precise_compute(double *arr, int n) {
    // 禁用向量化以保持浮点精度
}
```

## 优化的副作用

### 调试信息丢失

```bash
# 高度优化后，变量可能被优化掉
gcc -O2 -g main.c -o main
# 调试器可能显示 <optimized out>
```

### 浮点精度变化

```c
double sum = 0.0;
for (int i = 0; i < 1000000; i++)
    sum += 0.1;

// -O0: sum ≈ 99999.999999...
// -Ofast: 结果可能不同（重新关联运算顺序）
```

### 编译时间增加

```bash
# 编译时间对比
time gcc -O0 large_file.c -o large_O0   # 快
time gcc -O3 large_file.c -o large_O3   # 慢
```

## 重要注意事项

> **关键点总结**：
> 1. **开发阶段**用 `-O0` 或 `-Og`，**发布阶段**用 `-O2`
> 2. `-O3` 适合计算密集型程序，但要测试是否真的更快
> 3. `-Os` 适合嵌入式和对代码大小敏感的场景
> 4. `-Ofast` 可能改变浮点计算结果，谨慎使用
> 5. 优化可能影响调试——变量被优化掉、代码行号不准确
> 6. 使用 `-fopt-info` 查看编译器的优化决策
