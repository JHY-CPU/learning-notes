# 04 - 宏函数详解

## 带参数的宏（函数宏）

函数宏（Function-like Macro）是预处理器中功能最强大也最容易出错的部分。它看起来像函数调用，但本质是文本替换。

## 基本语法与定义

```c
#define 宏名(参数列表) 替换文本
```

```c
// 单参数宏
#define ABS(x) ((x) < 0 ? -(x) : (x))

// 多参数宏
#define MAX(a, b) ((a) > (b) ? (a) : (b))

// 无参数的"函数宏"
#define GET_VERSION() "1.0.0"
```

## 宏参数的求值问题

### 参数多次求值

这是宏函数最危险的陷阱。宏的参数在替换文本中可能出现多次，每次出现都会重新求值：

```c
#define SQUARE(x) ((x) * (x))

int i = 5;
int result = SQUARE(i++);
// 展开为: ((i++) * (i++))
// i 被自增了两次！结果未定义（undefined behavior）
```

```c
#define MAX(a, b) ((a) > (b) ? (a) : (b))

int x = 3, y = 5;
int result = MAX(x++, y++);
// 展开为: ((x++) > (y++) ? (x++) : (y++))
// x 和 y 都被多次自增，行为不可预测！
```

### 解决方案：使用内联函数或语句表达式

```c
// 方案1: 使用内联函数（C99）
static inline int square(int x) { return x * x; }
static inline int max(int a, int b) { return a > b ? a : b; }

// 方案2: GCC 语句表达式扩展
#define MAX_SAFE(a, b) ({ \
    typeof(a) _a = (a); \
    typeof(b) _b = (b); \
    _a > _b ? _a : _b; \
})
```

## 参数的类型无关性

宏不关心参数类型，这在某些场景下非常有用：

```c
#define MAX(a, b) ((a) > (b) ? (a) : (b))

int int_max = MAX(3, 5);           // int 类型
double dbl_max = MAX(3.14, 2.71);  // double 类型
char char_max = MAX('a', 'Z');     // char 类型
```

但这也意味着没有类型安全检查：

```c
#define ADD(a, b) ((a) + (b))

int result = ADD("hello", 5);  // 编译不报错！运行时崩溃
```

## do-while(0) 技巧

### 问题：宏作为语句块的陷阱

```c
// 不安全的宏定义
#define SWAP(a, b) { int tmp = (a); (a) = (b); (b) = tmp; }

if (condition)
    SWAP(x, y);  // 展开为: { int tmp = (x); (x) = (y); (y) = tmp; }
else
    do_other();  // 编译错误！else 前面有多余的分号
```

### 解决方案：do-while(0) 模式

```c
// 安全的宏定义
#define SWAP(a, b) do { \
    int _tmp = (a); \
    (a) = (b); \
    (b) = _tmp; \
} while(0)

if (condition)
    SWAP(x, y);   // 正确！展开为一个完整的语句
else
    do_other();   // 正确！
```

### do-while(0) 的工作原理

```c
// 宏展开后：
if (condition)
    do {
        int _tmp = (x);
        (x) = (y);
        (y) = _tmp;
    } while(0);   // 这是一个完整的语句，分号正确
else
    do_other();
```

`do { ... } while(0)` 在C语言中是一个**单条语句**，可以出现在任何需要语句的地方，而且后面必须有分号，这与普通语句的行为完全一致。

### 实际应用示例

```c
// 日志宏
#define LOG_ERROR(msg) do { \
    fprintf(stderr, "[ERROR] %s:%d: %s\n", __FILE__, __LINE__, msg); \
    fflush(stderr); \
} while(0)

// 内存释放宏
#define SAFE_FREE(ptr) do { \
    if (ptr) { \
        free(ptr); \
        (ptr) = NULL; \
    } \
} while(0)

// 使用示例
if (error_condition)
    LOG_ERROR("Something went wrong");
else
    process_data();

SAFE_FREE(buffer);
```

## 可变参数的函数宏

### 使用省略号和 `__VA_ARGS__`

```c
// C99 支持可变参数宏
#define PRINT(fmt, ...) printf(fmt, __VA_ARGS__)
#define DEBUG_LOG(...) fprintf(stderr, __VA_ARGS__)

PRINT("Hello %s, age %d\n", "Alice", 25);
DEBUG_LOG("Debug: x=%d, y=%d\n", x, y);
```

### 处理零个可变参数

```c
// ##__VA_ARGS__ 可以处理没有可变参数的情况（GCC扩展）
#define LOG(fmt, ...) printf(fmt, ##__VA_ARGS__)

LOG("Simple message\n");        // OK：没有额外参数
LOG("Value: %d\n", 42);         // OK：有一个额外参数
```

## 宏参数中的逗号问题

### 问题：逗号被当作参数分隔符

```c
#define CALL_FUNC(f, a, b) f(a, b)

CALL_FUNC(printf, "x=%d, y=%d\n", x, y);
// 错误！预处理器看到4个参数：
// 参数1: printf
// 参数2: "x=%d
// 参数3: y=%d\n"
// 参数4: x, y
```

### 解决方案：用括号包裹

```c
// 方法1: 用括号包裹含逗号的参数
CALL_FUNC(printf, ("x=%d, y=%d\n", x, y), 0);

// 方法2: 使用可变参数宏
#define PRINTF(fmt, ...) printf(fmt, __VA_ARGS__)
PRINTF("x=%d, y=%d\n", x, y);
```

## 宏中的局部变量命名

### 避免命名冲突

在宏中使用局部变量时，要避免与用户代码中的变量名冲突：

```c
// 不好：变量名可能冲突
#define SWAP(a, b) do { int tmp = a; a = b; b = tmp; } while(0)
// 如果用户代码中已经有 tmp 变量，会导致问题

// 好：使用特殊前缀
#define SWAP(a, b) do { \
    int _macro_tmp_##a##b = (a); \
    (a) = (b); \
    (b) = _macro_tmp_##a##b; \
} while(0)

// 更好：使用唯一前缀
#define SWAP(a, b) do { \
    int _sw_tmp_ = (a); \
    (a) = (b); \
    (b) = _sw_tmp_; \
} while(0)
```

## 宏与 inline 函数的选择

```c
// 使用宏的场景
#define ARRAY_SIZE(arr) (sizeof(arr) / sizeof((arr)[0]))
// 无法用 inline 函数实现，因为需要数组类型信息

#define TYPEOF(x) __typeof__(x)
// 需要类型推断，宏更合适

// 使用 inline 函数的场景
static inline int max_int(int a, int b) {
    return a > b ? a : b;  // 类型安全，不会多次求值
}
```

## 重要注意事项

> **关键点总结**：
> 1. 宏参数可能被**多次求值**，避免传入有副作用的表达式
> 2. 始终使用 **do-while(0)** 包裹多语句的宏
> 3. 宏中定义的局部变量使用**特殊前缀**避免命名冲突
> 4. 需要类型安全时，优先考虑 **inline 函数**
> 5. 宏的优势在于类型无关性和编译期计算
> 6. 调试困难时，使用 `gcc -E` 查看宏展开结果
