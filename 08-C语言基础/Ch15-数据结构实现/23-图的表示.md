# 图的表示

## 1. 概述

图（Graph）是由**顶点（Vertex）**和**边（Edge）**组成的非线性数据结构。图可以表示各种关系：社交网络、地图路线、依赖关系等。

### 图的分类

| 分类方式 | 类型 | 说明 |
|---------|------|------|
| 边是否有方向 | 无向图 / 有向图 | 无向图的边没有方向 |
| 边是否有权重 | 无权图 / 加权图 | 加权图的边有数值 |
| 是否有环 | 有环图 / 无环图 | DAG（有向无环图）很重要 |
| 连通性 | 连通图 / 非连通图 | 任意两点可达 |

## 2. 邻接矩阵

用二维数组表示顶点之间的关系。

```c
#include <stdio.h>
#include <stdlib.h>
#include <stdbool.h>

#define MAX_V 100
#define INF 999999  // 表示无边（用于加权图）

// 邻接矩阵表示
typedef struct {
    int vertexCount;            // 顶点数
    int adj[MAX_V][MAX_V];      // 邻接矩阵
} AdjMatrix;

void initAdjMatrix(AdjMatrix *g, int n) {
    g->vertexCount = n;
    for (int i = 0; i < n; i++)
        for (int j = 0; j < n; j++)
            g->adj[i][j] = 0;  // 0 表示无边
}

// 添加无向边
void addEdge(AdjMatrix *g, int u, int v) {
    g->adj[u][v] = 1;
    g->adj[v][u] = 1;
}

// 添加有向边
void addDirectedEdge(AdjMatrix *g, int u, int v) {
    g->adj[u][v] = 1;
}

// 添加带权重的边
void addWeightedEdge(AdjMatrix *g, int u, int v, int weight) {
    g->adj[u][v] = weight;
    g->adj[v][u] = weight;  // 无向图
}

// 打印邻接矩阵
void printMatrix(AdjMatrix *g) {
    printf("邻接矩阵:\n  ");
    for (int i = 0; i < g->vertexCount; i++) printf("%d ", i);
    printf("\n");
    for (int i = 0; i < g->vertexCount; i++) {
        printf("%d ", i);
        for (int j = 0; j < g->vertexCount; j++) {
            printf("%d ", g->adj[i][j]);
        }
        printf("\n");
    }
}
```

### 邻接矩阵的优缺点

| 优点 | 缺点 |
|------|------|
| 判断两顶点是否有边：O(1) | 空间复杂度 O(V^2) |
| 获取边权重：O(1) | 稀疏图浪费空间 |
| 实现简单 | 添加/删除顶点成本高 |

## 3. 邻接表

用数组+链表（或动态数组）表示图，每个顶点维护一个邻居列表。

```c
// 邻接表节点
typedef struct AdjNode {
    int vertex;              // 邻居顶点编号
    int weight;              // 边的权重（可选）
    struct AdjNode *next;
} AdjNode;

// 邻接表
typedef struct {
    int vertexCount;
    AdjNode *heads[MAX_V];   // 每个顶点的链表头
} AdjList;

void initAdjList(AdjList *g, int n) {
    g->vertexCount = n;
    for (int i = 0; i < n; i++)
        g->heads[i] = NULL;
}

// 添加无向边
void addListEdge(AdjList *g, int u, int v) {
    AdjNode *node = (AdjNode *)malloc(sizeof(AdjNode));
    node->vertex = v;
    node->weight = 1;
    node->next = g->heads[u];
    g->heads[u] = node;
}

// 添加带权重的边
void addListWeightedEdge(AdjList *g, int u, int v, int w) {
    AdjNode *node = (AdjNode *)malloc(sizeof(AdjNode));
    node->vertex = v;
    node->weight = w;
    node->next = g->heads[u];
    g->heads[u] = node;
}

// 打印邻接表
void printAdjList(AdjList *g) {
    printf("邻接表:\n");
    for (int i = 0; i < g->vertexCount; i++) {
        printf("  %d -> ", i);
        AdjNode *p = g->heads[i];
        while (p) {
            printf("%d(w=%d) ", p->vertex, p->weight);
            p = p->next;
        }
        printf("\n");
    }
}

// 检查两顶点是否有边
bool hasEdge(AdjList *g, int u, int v) {
    AdjNode *p = g->heads[u];
    while (p) {
        if (p->vertex == v) return true;
        p = p->next;
    }
    return false;
}

// 获取顶点的度
int getDegree(AdjList *g, int v) {
    int degree = 0;
    AdjNode *p = g->heads[v];
    while (p) {
        degree++;
        p = p->next;
    }
    return degree;
}
```

### 邻接表的优缺点

| 优点 | 缺点 |
|------|------|
| 空间：O(V+E)，适合稀疏图 | 判断是否有边：O(degree) |
| 添加边：O(1) | 获取边权重需要遍历 |
| 遍历邻居方便 | 实现稍复杂 |

## 4. 边列表

用边的列表来表示图，适合 Kruskal 算法等。

```c
typedef struct {
    int u, v;      // 两个顶点
    int weight;    // 权重
} Edge;

typedef struct {
    Edge edges[MAX_V * MAX_V];
    int edgeCount;
    int vertexCount;
} EdgeList;

void addEdgeList(EdgeList *g, int u, int v, int w) {
    g->edges[g->edgeCount].u = u;
    g->edges[g->edgeCount].v = v;
    g->edges[g->edgeCount].weight = w;
    g->edgeCount++;
}
```

## 5. 三种表示法对比

| 特性 | 邻接矩阵 | 邻接表 | 边列表 |
|------|---------|--------|--------|
| 空间 | O(V^2) | O(V+E) | O(E) |
| 判断是否有边 | O(1) | O(degree) | O(E) |
| 遍历所有邻居 | O(V) | O(degree) | O(E) |
| 适用场景 | 稠密图 | 稀疏图 | 边相关算法 |

## 6. 完整测试

```c
int main() {
    // 邻接矩阵测试
    AdjMatrix gm;
    initAdjMatrix(&gm, 5);
    addEdge(&gm, 0, 1);
    addEdge(&gm, 0, 4);
    addEdge(&gm, 1, 2);
    addEdge(&gm, 1, 3);
    addEdge(&gm, 1, 4);
    addEdge(&gm, 2, 3);
    addEdge(&gm, 3, 4);
    printMatrix(&gm);

    printf("\n");

    // 邻接表测试
    AdjList gl;
    initAdjList(&gl, 5);
    addListEdge(&gl, 0, 1); addListEdge(&gl, 1, 0);
    addListEdge(&gl, 0, 4); addListEdge(&gl, 4, 0);
    addListEdge(&gl, 1, 2); addListEdge(&gl, 2, 1);
    addListEdge(&gl, 1, 3); addListEdge(&gl, 3, 1);
    addListEdge(&gl, 1, 4); addListEdge(&gl, 4, 1);
    addListEdge(&gl, 2, 3); addListEdge(&gl, 3, 2);
    addListEdge(&gl, 3, 4); addListEdge(&gl, 4, 3);
    printAdjList(&gl);

    printf("0-3有边? %s\n", hasEdge(&gl, 0, 3) ? "是" : "否");
    printf("1的度: %d\n", getDegree(&gl, 1));

    return 0;
}
```

## 7. 重点总结

- 邻接矩阵适合**稠密图**，空间 O(V^2)，判断边 O(1)
- 邻接表适合**稀疏图**，空间 O(V+E)，遍历邻居高效
- 边列表适合**边相关算法**（如 Kruskal）
- 无向图的每条边在邻接矩阵中出现两次（对称）
- 有向图只需存储一个方向
- 加权图在邻接表中多存一个权重字段

> **选择建议**：大多数情况下使用邻接表。只有在图非常稠密或需要频繁判断边存在性时才使用邻接矩阵。
