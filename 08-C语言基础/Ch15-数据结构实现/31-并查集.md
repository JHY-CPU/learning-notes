# 31 - 并查集

## 概述

并查集（Union-Find / Disjoint Set Union）是一种用于处理不相交集合的合并与查询问题的数据结构，广泛应用于连通性判断、最小生成树等场景。

---

## 1. 基本概念

- **Find**：确定某个元素属于哪个集合（找到集合代表/根节点）
- **Union**：合并两个集合
- 使用树形结构，每个集合用一棵树表示

---

## 2. 基础实现

```c
#include <stdio.h>
#include <stdlib.h>

#define MAXN 10000

int parent[MAXN];  // 父节点数组
int rank_[MAXN];   // 秩（树的高度上界）
int count;         // 连通分量个数

// 初始化：每个元素自成一个集合
void uf_init(int n) {
    count = n;
    for (int i = 0; i < n; i++) {
        parent[i] = i;   // 自己是自己的父节点
        rank_[i] = 0;    // 初始秩为0
    }
}

// 查找根节点（基础版）
int uf_find(int x) {
    while (parent[x] != x) {
        x = parent[x];
    }
    return x;
}

// 合并两个集合（基础版）
void uf_union(int x, int y) {
    int root_x = uf_find(x);
    int root_y = uf_find(y);
    if (root_x != root_y) {
        parent[root_x] = root_y;
        count--;
    }
}

// 判断两个元素是否在同一集合
int uf_connected(int x, int y) {
    return uf_find(x) == uf_find(y);
}

// 返回连通分量个数
int uf_count(void) {
    return count;
}
```

---

## 3. 路径压缩（Path Compression）

### 原理

在Find操作时，将查找路径上的所有节点直接指向根节点，从而"压缩"树的高度。

### 效果

使后续Find操作接近O(1)。

```c
// 路径压缩版Find（递归）
int uf_find_compress(int x) {
    if (parent[x] != x) {
        parent[x] = uf_find_compress(parent[x]);  // 路径压缩
    }
    return parent[x];
}

// 路径压缩版Find（迭代）
int uf_find_compress_iter(int x) {
    int root = x;
    // 第一遍：找到根节点
    while (parent[root] != root) {
        root = parent[root];
    }
    // 第二遍：路径压缩
    while (parent[x] != x) {
        int next = parent[x];
        parent[x] = root;
        x = next;
    }
    return root;
}
```

## 4. 按秩合并（Union by Rank）

### 原理

合并时将较矮的树挂到较高的树下面，保持树的平衡。

```c
// 按秩合并
void uf_union_rank(int x, int y) {
    int root_x = uf_find_compress(x);
    int root_y = uf_find_compress(y);

    if (root_x == root_y) return;  // 已在同一集合

    // 将秩小的树合并到秩大的树
    if (rank_[root_x] < rank_[root_y]) {
        parent[root_x] = root_y;
    } else if (rank_[root_x] > rank_[root_y]) {
        parent[root_y] = root_x;
    } else {
        parent[root_y] = root_x;
        rank_[root_x]++;  // 秩相同，合并后秩加1
    }

    count--;
}
```

## 5. 按大小合并（Union by Size）

```c
int size_[MAXN];  // 每个集合的大小

void uf_init_size(int n) {
    count = n;
    for (int i = 0; i < n; i++) {
        parent[i] = i;
        size_[i] = 1;
    }
}

void uf_union_size(int x, int y) {
    int root_x = uf_find_compress(x);
    int root_y = uf_find_compress(y);

    if (root_x == root_y) return;

    // 小树合并到大树
    if (size_[root_x] < size_[root_y]) {
        parent[root_x] = root_y;
        size_[root_y] += size_[root_x];
    } else {
        parent[root_y] = root_x;
        size_[root_x] += size_[root_y];
    }

    count--;
}
```

## 6. 完整示例：连通性问题

```c
#include <stdio.h>
#include <stdlib.h>

#define MAXN 1000

int parent[MAXN], rank_[MAXN], cnt;

void init(int n) {
    cnt = n;
    for (int i = 0; i < n; i++) {
        parent[i] = i;
        rank_[i] = 0;
    }
}

int find(int x) {
    if (parent[x] != x)
        parent[x] = find(parent[x]);
    return parent[x];
}

void union_(int x, int y) {
    int rx = find(x), ry = find(y);
    if (rx == ry) return;
    if (rank_[rx] < rank_[ry]) parent[rx] = ry;
    else if (rank_[rx] > rank_[ry]) parent[ry] = rx;
    else { parent[ry] = rx; rank_[rx]++; }
    cnt--;
}

int main(void) {
    int n = 10;
    init(n);

    // 添加连接
    union_(0, 1);
    union_(2, 3);
    union_(4, 5);
    union_(1, 3);  // 合并 {0,1} 和 {2,3}

    printf("连通分量数: %d\n", cnt);  // 7
    printf("0和3连通: %s\n", find(0) == find(3) ? "是" : "否");  // 是
    printf("0和5连通: %s\n", find(0) == find(5) ? "是" : "否");  // 否

    union_(5, 6);
    union_(7, 8);
    union_(0, 5);  // 合并两大组

    printf("连通分量数: %d\n", cnt);  // 4

    return 0;
}
```

## 7. 应用场景

- **连通性判断**：判断图中两点是否连通
- **最小生成树**：Kruskal算法中的核心操作
- **网络连接**：社交网络中的好友关系
- **等价类划分**：将等价元素归类
- **图像处理**：连通区域标记

---

## 8. 复杂度分析

| 操作 | 仅按秩合并 | 仅路径压缩 | 两者结合 |
|------|-----------|-----------|----------|
| Find | O(log n) | 接近O(1) | 接近O(1) |
| Union | O(log n) | 接近O(1) | 接近O(1) |

使用路径压缩 + 按秩合并，m次操作的均摊时间复杂度为O(m * α(n))，其中α(n)是反阿克曼函数，增长极慢（实际可视为常数）。

---

## 要点总结

1. 并查集的核心操作是Find和Union
2. 路径压缩使树变平，大幅加速后续查找
3. 按秩合并保持树的平衡
4. 两者结合可实现近乎O(1)的均摊复杂度
5. 并查集是Kruskal算法等图算法的基础
