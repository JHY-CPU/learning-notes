# 红黑树思想

## 1. 概述

红黑树（Red-Black Tree）是一种**自平衡二叉搜索树**，通过给每个节点标记颜色（红或黑）并遵循特定规则来保持近似平衡。它在实际应用中比 AVL 树更常见（如 C++ STL 的 `map`/`set`、Java 的 `TreeMap`）。

### 为什么使用红黑树
- 保证最长路径不超过最短路径的两倍
- 插入和删除的旋转次数比 AVL 树少
- 综合性能优秀，适合频繁插入删除的场景

## 2. 红黑树的五条性质

1. **每个节点是红色或黑色**
2. **根节点是黑色**
3. **所有叶子节点（NIL/空节点）是黑色**
4. **红色节点的两个子节点必须是黑色**（不能有两个连续的红色节点）
5. **从任意节点到其所有叶子节点的路径包含相同数量的黑色节点**（黑高相同）

```
         13(B)
        /     \
      8(R)    17(R)
     /   \    /   \
   1(B) 11(B)15(B) 25(B)
         \         /
         6(R)    22(R)
```

这五条性质保证了：从根到叶子的最长路径不会超过最短路径的两倍。

## 3. 基本概念

### 3.1 黑高（Black Height）
从某节点到叶子节点路径上的黑色节点数量（不含该节点本身）。

### 3.2 节点定义

```c
#include <stdio.h>
#include <stdlib.h>

typedef enum { RED, BLACK } Color;

typedef struct RBNode {
    int data;
    Color color;
    struct RBNode *left;
    struct RBNode *right;
    struct RBNode *parent;  // 红黑树通常需要父指针
} RBNode;

RBNode *createRBNode(int value) {
    RBNode *node = (RBNode *)malloc(sizeof(RBNode));
    node->data = value;
    node->color = RED;  // 新节点默认红色（性质5更容易满足）
    node->left = node->right = node->parent = NULL;
    return node;
}
```

### 为什么新节点默认是红色？
- 插入红色节点不会改变黑高（性质5）
- 如果插入黑色节点，一定会破坏性质5
- 所以插入红色节点后可能需要调整的情况更少

## 4. 旋转操作

与 AVL 树相同，红黑树也需要左旋和右旋。

```c
// 左旋
void rotateLeft(RBNode **root, RBNode *x) {
    RBNode *y = x->right;
    x->right = y->left;
    if (y->left) y->left->parent = x;
    y->parent = x->parent;
    if (!x->parent) *root = y;
    else if (x == x->parent->left) x->parent->left = y;
    else x->parent->right = y;
    y->left = x;
    x->parent = y;
}

// 右旋
void rotateRight(RBNode **root, RBNode *y) {
    RBNode *x = y->left;
    y->left = x->right;
    if (x->right) x->right->parent = y;
    x->parent = y->parent;
    if (!y->parent) *root = x;
    else if (y == y->parent->left) y->parent->left = x;
    else y->parent->right = x;
    x->right = y;
    y->parent = x;
}
```

## 5. 插入调整

插入新节点后，如果父节点是红色（违反性质4），需要调整。

### 三种情况

**情况一：叔叔节点是红色**
```
        G(B)              G(R)    <- 祖父变色
       /   \             /   \
    P(R)   U(R)  ->   P(B)   U(B) <- 父叔变色
    /                /
  N(R)            N(R)
```
将父和叔变黑，祖父变红，然后以祖父为当前节点继续调整。

**情况二：叔叔是黑色，当前节点是父的右孩子（或左孩子，对应LR/RL）**
先对父节点进行旋转，转化为情况三。

**情况三：叔叔是黑色，当前节点是父的左孩子（或右孩子，对应LL/RR）**
```
        G(B)               P(B)
       /   \              /   \
    P(R)   U(B)  ->    N(R)   G(R)
    /                          /  \
  N(R)                      ...   U(B)
```
父节点变黑，祖父变红，对祖父进行旋转。

### 插入调整伪代码

```c
void fixInsert(RBNode **root, RBNode *node) {
    RBNode *parent, *grandparent, *uncle;

    while (node != *root && node->parent->color == RED) {
        parent = node->parent;
        grandparent = parent->parent;

        // 父节点是祖父的左孩子
        if (parent == grandparent->left) {
            uncle = grandparent->right;

            if (uncle && uncle->color == RED) {
                // 情况一：叔叔是红色
                parent->color = BLACK;
                uncle->color = BLACK;
                grandparent->color = RED;
                node = grandparent;  // 继续向上调整
            } else {
                // 情况二/三：叔叔是黑色
                if (node == parent->right) {
                    // 情况二：LR型，先左旋
                    node = parent;
                    rotateLeft(root, node);
                    parent = node->parent;
                    grandparent = parent->parent;
                }
                // 情况三：LL型，右旋
                parent->color = BLACK;
                grandparent->color = RED;
                rotateRight(root, grandparent);
            }
        } else {
            // 父节点是祖父的右孩子（镜像操作）
            uncle = grandparent->left;
            if (uncle && uncle->color == RED) {
                parent->color = BLACK;
                uncle->color = BLACK;
                grandparent->color = RED;
                node = grandparent;
            } else {
                if (node == parent->left) {
                    node = parent;
                    rotateRight(root, node);
                    parent = node->parent;
                    grandparent = parent->parent;
                }
                parent->color = BLACK;
                grandparent->color = RED;
                rotateLeft(root, grandparent);
            }
        }
    }
    (*root)->color = BLACK;  // 根节点始终为黑
}
```

## 6. 删除思想

红黑树的删除比插入更复杂。基本步骤：
1. 执行标准 BST 删除
2. 如果被删除节点是黑色，需要通过调整来维持性质5
3. 删除调整涉及多次颜色变换和旋转

> 完整实现代码量很大（200+行），在实际开发中通常使用成熟库。

## 7. 红黑树 vs AVL 树

| 特性 | 红黑树 | AVL 树 |
|------|--------|--------|
| 平衡标准 | 黑高相等 | 平衡因子 [-1, 0, 1] |
| 查找效率 | 略低于 AVL | 更高效（更严格平衡） |
| 插入旋转 | 最多 2 次 | 最多 1 次 |
| 删除旋转 | 最多 3 次 | 最多 O(log n) 次 |
| 适用场景 | 频繁插入删除 | 频繁查找 |
| 实际应用 | C++ STL, Java TreeMap | 数据库索引 |

## 8. 验证红黑树

```c
bool isValidRBTree(RBNode *root) {
    if (!root) return true;

    // 性质2：根节点是黑色
    if (root->color != BLACK) return false;

    // 性质4：红色节点的子节点必须是黑色
    if (root->color == RED) {
        if ((root->left && root->left->color == RED) ||
            (root->right && root->right->color == RED)) {
            return false;
        }
    }

    // 递归检查子树
    return isValidRBTree(root->left) && isValidRBTree(root->right);
}
```

## 9. 重点总结

- 红黑树通过**颜色约束 + 旋转**维持近似平衡
- 五条性质是红黑树的核心
- 新节点默认**红色**，减少需要调整的情况
- 插入调整最多涉及**3次变色和2次旋转**
- 红黑树的平衡条件比 AVL 树宽松，但旋转次数更少
- 实际工程中红黑树使用更广泛（综合性能更优）

> **学习建议**：红黑树的完整实现较复杂。建议先理解原理和性质，实际编码时优先使用标准库实现。面试中通常只考察红黑树的性质和旋转策略，而非完整代码。
