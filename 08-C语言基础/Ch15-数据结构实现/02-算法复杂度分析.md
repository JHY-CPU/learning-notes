# 算法复杂度分析

## 1. 为什么要分析复杂度

在编写程序时，同一个问题可能有多种不同的解法。如何衡量一个算法的好坏？我们需要一种**不依赖具体机器和编程语言**的分析方法——这就是算法复杂度分析。

复杂度分析关注两个维度：
- **时间复杂度**：算法执行需要多少时间
- **空间复杂度**：算法执行需要多少内存空间

## 2. 大O表示法（Big O Notation）

大O表示法描述的是算法执行时间（或空间占用）随输入规模增长的变化趋势。它忽略了常数项和低阶项，只关注**最高阶**的部分。

### 定义
如果存在正常数 `c` 和 `n0`，使得当 `n >= n0` 时，`f(n) <= c * g(n)`，则称 `f(n) = O(g(n))`。

简单理解：**大O表示的是算法运行时间的上界（最坏情况）**。

### 举例理解

```
f(n) = 3n² + 10n + 5
忽略系数和低阶项 → O(n²)
```

## 3. 常见时间复杂度

从快到慢排列：

```
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ) < O(n!)
```

| 复杂度 | 名称 | 说明 | 示例 |
|--------|------|------|------|
| O(1) | 常数时间 | 不随输入规模变化 | 数组按下标访问 |
| O(log n) | 对数时间 | 每步排除一半 | 二分查找 |
| O(n) | 线性时间 | 遍历一次 | 遍历数组 |
| O(n log n) | 线性对数 | 分治策略 | 归并排序、快速排序 |
| O(n²) | 平方时间 | 双层循环 | 冒泡排序 |
| O(n³) | 立方时间 | 三层循环 | 普通矩阵乘法 |
| O(2ⁿ) | 指数时间 | 每次规模翻倍 | 斐波那契递归 |
| O(n!) | 阶乘时间 | 全排列 | 旅行商问题暴力解 |

### 直观感受（n = 100 时）

```
O(1)       → 1 次操作
O(log n)   → 约 7 次操作
O(n)       → 100 次操作
O(n log n) → 约 700 次操作
O(n²)      → 10000 次操作
O(2ⁿ)      → 约 1.27 × 10³⁰ 次操作（不可行！）
```

## 4. 时间复杂度分析方法

### 4.1 只关注循环

最常影响时间复杂度的是循环结构：

```c
// O(1) - 常数时间
int sum = a + b;

// O(n) - 单层循环
for (int i = 0; i < n; i++) {
    printf("%d\n", i);
}

// O(n²) - 双层嵌套循环
for (int i = 0; i < n; i++) {
    for (int j = 0; j < n; j++) {
        arr[i][j] = 0;
    }
}

// O(log n) - 循环变量倍增
for (int i = 1; i < n; i *= 2) {
    printf("%d\n", i);
}

// O(log n) - 循环变量减半
while (n > 1) {
    n = n / 2;
}
```

### 4.2 加法法则（取最大值）

多个并列的代码块，总复杂度取最大的那个：

```c
// O(n²) + O(n) + O(1) = O(n²)
for (int i = 0; i < n; i++)      // O(n²)
    for (int j = 0; j < n; j++)
        x++;

for (int i = 0; i < n; i++)      // O(n)
    y++;

z++;                              // O(1)
```

### 4.3 乘法法则（嵌套相乘）

嵌套结构的复杂度是各层的乘积：

```c
// O(n) × O(n) = O(n²)
for (int i = 0; i < n; i++) {
    for (int j = 0; j < n; j++) {
        // O(1) 操作
    }
}
```

## 5. 空间复杂度

空间复杂度分析的是算法在运行过程中**额外占用的存储空间**（不包括输入数据本身）。

### 常见空间复杂度

```c
// O(1) - 常数空间
int sum = 0;
for (int i = 0; i < n; i++) {
    sum += arr[i];
}

// O(n) - 线性空间
int *temp = (int *)malloc(n * sizeof(int));
for (int i = 0; i < n; i++) {
    temp[i] = arr[i] * 2;
}

// O(n²) - 平方空间
int **matrix = (int **)malloc(n * sizeof(int *));
for (int i = 0; i < n; i++) {
    matrix[i] = (int *)malloc(n * sizeof(int));
}
```

### 递归的空间复杂度

递归调用会使用栈空间，空间复杂度等于递归深度：

```c
// O(n) 空间 - 递归深度为 n
int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);
}

// O(log n) 空间 - 递归深度为 log n
int binary_search(int arr[], int left, int right, int target) {
    if (left > right) return -1;
    int mid = (left + right) / 2;
    if (arr[mid] == target) return mid;
    if (arr[mid] < target)
        return binary_search(arr, mid + 1, right, target);
    else
        return binary_search(arr, left, mid - 1, target);
}
```

## 6. 最好、最坏、平均情况

| 情况 | 说明 | 使用场景 |
|------|------|---------|
| 最好情况 | 输入最有利时的复杂度 | 理论分析 |
| 最坏情况 | 输入最不利时的复杂度 | **最常用，保证性能下限** |
| 平均情况 | 所有输入的期望复杂度 | 综合评估 |

例如，在长度为 n 的数组中查找元素 x：
- 最好：O(1)，第一个元素就是
- 最坏：O(n)，遍历完才找到或不存在
- 平均：O(n/2) = O(n)

## 7. 均摊分析（Amortized Analysis）

某些操作偶尔很耗时，但平均下来仍然很高效。例如动态数组的扩容：
- 大多数 push 操作是 O(1)
- 偶尔需要扩容，花费 O(n)
- 均摊后每次 push 仍然是 O(1)

## 8. 重点总结

| 要点 | 说明 |
|------|------|
| 大O表示上界 | 关注增长趋势，忽略常数和低阶项 |
| 循环决定复杂度 | 关注循环层数和循环变量的变化方式 |
| 加法取最大 | 并列结构取复杂度最高的 |
| 乘法相乘 | 嵌套结构复杂度相乘 |
| 最坏情况最常用 | 保证算法性能的下界 |
| 时间空间可互换 | 往往可以用空间换时间，或用时间换空间 |

> **核心思想**：复杂度分析不是为了计算精确的操作次数，而是为了在算法设计时能够预判性能瓶颈，从而做出合理的选择。
