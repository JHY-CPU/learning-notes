# 数据结构概述

## 1. 什么是数据结构

数据结构（Data Structure）是计算机存储、组织数据的方式。它研究的是数据的**逻辑结构**、**物理结构**以及它们之间的相互关系，并定义了在数据上可以执行的**操作**。

简单来说，数据结构就是把数据按照特定的规则组织起来，使得数据的增删改查等操作更加高效。

## 2. 基本概念

### 2.1 数据（Data）
数据是信息的载体，是能够被计算机程序识别、存储和加工的符号总和。包括整数、浮点数、字符、图像、声音等。

### 2.2 数据元素（Data Element）
数据元素是数据的基本单位，在程序中通常作为一个整体来考虑和处理。一个数据元素可以由若干个**数据项**组成。

### 2.3 数据对象（Data Object）
数据对象是性质相同的数据元素的集合，是数据的一个子集。

## 3. 逻辑结构 vs 物理结构

### 3.1 逻辑结构（Logical Structure）
逻辑结构是指数据元素之间的逻辑关系，与数据在计算机中的存储方式无关。主要分为四类：

```
┌─────────────────────────────────────────────────┐
│                  逻辑结构                         │
├──────────┬──────────┬──────────┬────────────────┤
│  集合结构  │  线性结构  │  树形结构  │   图形结构     │
│ (Set)    │ (Linear) │  (Tree)  │  (Graph)     │
├──────────┼──────────┼──────────┼────────────────┤
│ 数据同属  │ 一对一    │  一对多   │   多对多       │
│ 一个集合  │ 关系      │  关系     │   关系         │
└──────────┴──────────┴──────────┴────────────────┘
```

- **集合结构**：数据元素之间没有特别的关系，仅同属一个集合
- **线性结构**：元素之间存在一对一的关系（如：线性表、栈、队列）
- **树形结构**：元素之间存在一对多的层次关系（如：二叉树、堆）
- **图形结构**：元素之间存在多对多的任意关系（如：社交网络、地图）

### 3.2 物理结构（Physical Structure）
物理结构（又称存储结构）是指数据在计算机内存中的实际存储方式。主要有两种：

| 存储方式 | 特点 | 适用场景 |
|---------|------|---------|
| **顺序存储** | 连续内存空间，随机访问 | 频繁按索引访问 |
| **链式存储** | 非连续内存，通过指针连接 | 频繁插入删除 |

## 4. 常见数据结构分类

### 4.1 线性结构
- **线性表**：顺序表、链表（单链表、双向链表、循环链表）
- **栈**（Stack）：后进先出（LIFO）
- **队列**（Queue）：先进先出（FIFO）
- **双端队列**（Deque）：两端都可插入删除

### 4.2 非线性结构
- **树**：二叉树、二叉搜索树、AVL树、红黑树、B树、B+树、堆
- **图**：有向图、无向图、加权图
- **哈希表**（Hash Table）：基于哈希函数的键值存储

### 4.3 特殊数据结构
- **并查集**（Union-Find）
- **字典树**（Trie）
- **跳表**（Skip List）
- **布隆过滤器**（Bloom Filter）

## 5. 数据结构与算法的关系

```
算法 + 数据结构 = 程序
```

- **数据结构**是算法处理的对象，决定了数据如何组织
- **算法**是对数据结构进行操作的步骤，解决问题的方法
- 好的数据结构可以大大简化算法的复杂度
- 同一个问题，不同的数据结构可能导致完全不同的性能

## 6. 抽象数据类型（ADT）

抽象数据类型是指一个数学模型以及定义在该模型上的一组操作。它只关心**做什么**，而不关心**怎么做**。

```
ADT 抽象数据类型名 {
    数据对象: D
    数据关系: R
    基本操作:
        - 初始化
        - 插入
        - 删除
        - 查找
        - 遍历
}
```

例如，线性表的 ADT 定义了插入、删除、查找等操作，但具体用数组还是链表来实现，由程序员决定。

## 7. 重点总结

| 要点 | 说明 |
|------|------|
| 数据结构是数据的组织方式 | 包含逻辑结构和物理结构两个层面 |
| 逻辑结构四大类 | 集合、线性、树形、图形 |
| 物理结构两大类 | 顺序存储、链式存储 |
| 数据结构与算法相辅相成 | 选择合适的数据结构是优化性能的关键 |
| ADT 关注"做什么" | 不关心底层实现细节 |

> **核心思想**：没有最好的数据结构，只有最适合特定场景的数据结构。理解每种结构的优缺点和适用场景，是学习数据结构的根本目标。
