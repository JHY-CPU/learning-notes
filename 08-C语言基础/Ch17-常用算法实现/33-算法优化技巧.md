# 算法优化技巧

## 1. 常数优化

在算法复杂度不变的情况下，通过减少实际操作次数来提升速度。

### 循环展开

```c
// 优化前
int sum = 0;
for (int i = 0; i < n; i++) sum += arr[i];

// 循环展开：每次处理4个元素
int sum = 0;
int i;
for (i = 0; i + 3 < n; i += 4) {
    sum += arr[i] + arr[i + 1] + arr[i + 2] + arr[i + 3];
}
for (; i < n; i++) sum += arr[i];  // 处理剩余元素
```

### 减少分支预测失败

```c
// 优化前：有分支
for (int i = 0; i < n; i++) {
    if (arr[i] > 0) sum += arr[i];
}

// 优化后：用位运算消除分支
for (int i = 0; i < n; i++) {
    int mask = (arr[i] >> 31);  // arr[i] >= 0 时为0，否则为-1
    sum += arr[i] & ~mask;      // 负数时加0
}

// 或者排序后处理（利用有序性减少分支）
```

### 内联函数与宏

```c
// 宏（无函数调用开销）
#define min(a, b) ((a) < (b) ? (a) : (b))
#define max(a, b) ((a) > (b) ? (a) : (b))

// 内联函数（编译器决定是否内联）
static inline int gcd(int a, int b) {
    while (b) { int t = b; b = a % b; a = t; }
    return a;
}
```

## 2. 预处理技巧

### 打表（预计算表）

```c
// 预计算所有素数
int primes[10000], primeCount = 0;
int isPrime[100001];
void precomputePrimes() {
    memset(isPrime, 1, sizeof(isPrime));
    isPrime[0] = isPrime[1] = 0;
    for (int i = 2; i <= 100000; i++) {
        if (isPrime[i]) {
            primes[primeCount++] = i;
            for (int j = i * 2; j <= 100000; j += i)
                isPrime[j] = 0;
        }
    }
}

// 预计算阶乘取模
long long fact[1000001], invFact[1000001];
void precomputeFactorial(int n) {
    fact[0] = 1;
    for (int i = 1; i <= n; i++) fact[i] = fact[i-1] * i % MOD;
    invFact[n] = fastPow(fact[n], MOD - 2, MOD);
    for (int i = n - 1; i >= 0; i--) invFact[i] = invFact[i+1] * (i+1) % MOD;
}
```

### 前缀信息

```c
// 预计算前缀和、前缀积、前缀异或等
int prefix[MAXN];
void buildPrefix(int arr[], int n) {
    prefix[0] = 0;
    for (int i = 1; i <= n; i++)
        prefix[i] = prefix[i-1] + arr[i-1];
}
int rangeSum(int l, int r) { return prefix[r+1] - prefix[l]; }
```

## 4. 数据结构优化

### 路径压缩并查集

```c
int find(int x) {
    return fa[x] == x ? x : (fa[x] = find(fa[x]));
}
```

### 懒惰删除

```c
// 堆中懒惰删除：不立即移除，标记为已删除
int deleted[MAXN];
int topValid() {
    while (heapSize > 0 && deleted[heap[0]])
        pop();
    return heap[0];
}
```

### 分块

```c
// 将数组分为sqrt(n)块，每块维护信息
int blockSize, blockCount;
int blockSum[500];  // 每块的和

void buildBlocks(int arr[], int n) {
    blockSize = (int)sqrt(n);
    blockCount = (n + blockSize - 1) / blockSize;
    for (int b = 0; b < blockCount; b++) {
        blockSum[b] = 0;
        int start = b * blockSize, end = (b + 1) * blockSize;
        if (end > n) end = n;
        for (int i = start; i < end; i++)
            blockSum[b] += arr[i];
    }
}
// O(1)单点修改, O(sqrt(n))区间查询
```

## 5. 算法优化策略

```c
// 1. 选择合适的算法
//    - 数据量小: O(n^2) 可能比 O(n log n) 的常数小
//    - 数据近似有序: 插入排序优于快速排序

// 2. 用空间换时间
//    - 哈希表: O(n^2) -> O(n)
//    - 记忆化: 指数 -> 多项式

// 3. 用时间换空间（内存受限时）
//    - 重复计算代替存储
//    - 位压缩

// 4. 优化数据结构选择
//    - 频繁查找: 哈希表
//    - 频繁插入删除: 链表
//    - 需要有序: 红黑树

// 5. I/O优化
//    - 使用getchar/putchar代替scanf/printf
//    - 使用fread批量读入
//    - 减少格式化输出
```

## 7. 总结

| 优化类型 | 方法 | 适用场景 |
|---------|------|---------|
| 常数优化 | 循环展开、消除分支 | 紧密循环 |
| 预处理 | 打表、前缀信息 | 多次查询 |
| 剪枝 | 最优性/可行性剪枝 | 搜索算法 |
| 数据结构 | 并查集路径压缩 | 动态连通性 |
| I/O优化 | 快速读入输出 | 竞赛编程 |
| 编译优化 | -O2选项、分支预测 | 所有场景 |
