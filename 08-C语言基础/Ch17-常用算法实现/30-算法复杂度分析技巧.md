# 算法复杂度分析技巧

## 1. 复杂度基本概念

### 时间复杂度

表示算法执行时间随输入规模增长的趋势。常见的复杂度从低到高：

```
O(1) < O(log n) < O(n) < O(n log n) < O(n^2) < O(2^n) < O(n!)
```

### 空间复杂度

算法运行过程中所需的额外存储空间。

## 2. 常见复杂度模式

```c
// O(1): 常数时间
int getFirst(int arr[], int n) { return arr[0]; }

// O(log n): 对数时间
int binarySearch(int arr[], int n, int target) {
    int l = 0, r = n - 1;
    while (l <= r) {
        int m = l + (r - l) / 2;
        if (arr[m] == target) return m;
        if (arr[m] < target) l = m + 1;
        else r = m - 1;
    }
    return -1;
}

// O(n): 线性时间
int linearSearch(int arr[], int n, int target) {
    for (int i = 0; i < n; i++)
        if (arr[i] == target) return i;
    return -1;
}

// O(n log n): 线性对数
// 归并排序、快速排序

// O(n^2): 平方
void bubbleSort(int arr[], int n) {
    for (int i = 0; i < n; i++)
        for (int j = 0; j < n - 1; j++)
            if (arr[j] > arr[j + 1])
                { int t = arr[j]; arr[j] = arr[j+1]; arr[j+1] = t; }
}

// O(2^n): 指数
int fib(int n) {
    if (n <= 1) return n;
    return fib(n - 1) + fib(n - 2);
}
```

## 3. 递推式求解方法

### 主定理（Master Theorem）

对于递推式 `T(n) = aT(n/b) + f(n)`：

```c
/*
 * 令 c = log_b(a)
 * 1. 若 f(n) = O(n^(c - ε))，则 T(n) = Θ(n^c)
 * 2. 若 f(n) = Θ(n^c)，则 T(n) = Θ(n^c * log n)
 * 3. 若 f(n) = Ω(n^(c + ε))，且 af(n/b) ≤ cf(n) (c<1)，则 T(n) = Θ(f(n))
 */

// 示例：
// 归并排序: T(n) = 2T(n/2) + O(n) → a=2, b=2, c=1 → O(n log n)
// 二分查找: T(n) = T(n/2) + O(1) → a=1, b=2, c=0 → O(log n)
// Karatsuba: T(n) = 3T(n/2) + O(n) → a=3, b=2, c=log2(3)≈1.585 → O(n^1.585)
```

### 代入法

```c
// 猜测解的形式，用数学归纳法证明
// 示例：T(n) = T(n-1) + n, T(1) = 1
// 猜测 T(n) = O(n^2)
// T(n) = 1 + 2 + ... + n = n(n+1)/2 = O(n^2)
```

### 递归树法

```c
// 将递推式展开为树结构，计算每层代价，求总和
// T(n) = 2T(n/2) + n
// 第0层: n
// 第1层: 2个 n/2 = n
// 第k层: 2^k个 n/2^k = n
// 共 log n 层 → 总计 O(n log n)
```

## 4. 均摊分析

对一系列操作的总时间进行分析，分摊到每个操作上。

### 聚合法

```c
// 动态数组扩容：每次扩容为2倍
// n次push操作的总成本: n + 2 + 4 + ... + n ≈ 3n
// 均摊成本: 3n / n = O(1)
```

### 记账法

```c
// 每次操作预付一定费用，余额用于支付昂贵操作
// push操作：支付3个单位（1个存元素，2个预存扩容费用）
// 当扩容时，之前预存的费用刚好够用
```

### 势能法

```c
// 定义势能函数 Φ(D)
// 均摊代价 = 实际代价 + Φ(D_i) - Φ(D_{i-1})
// 示例：栈的pop操作，若实际代价为O(1)，势能无变化
// 但连续n次push后的pop可能触发缩容
```

## 5. 分析技巧

```c
// 1. 循环嵌套：相乘
for (int i = 0; i < n; i++)        // n
    for (int j = 0; j < n; j++)    // n × n = O(n^2)

// 2. 循环变长：对数
for (int i = 1; i < n; i *= 2)    // O(log n)

// 3. 循环递减：除法
for (int i = n; i > 0; i /= 2)    // O(log n)

// 4. 递归：T(n) = T(n-1) + O(1) = O(n)
// 5. 递归：T(n) = 2T(n/2) + O(1) = O(n)

// 6. 嵌套递归
void foo(int n) {
    if (n <= 1) return;
    foo(n - 1);  // T(n-1)
    foo(n - 1);  // T(n-1)
    // T(n) = 2T(n-1) + O(1) = O(2^n)
}

// 7. 分析时忽略常数和低阶项
// 3n^2 + 5n + 100 = O(n^2)
```

## 6. 常见算法复杂度速查

| 操作/算法 | 时间复杂度 |
|----------|-----------|
| 数组访问 | O(1) |
| 链表查找 | O(n) |
| 哈希表查找 | O(1) 均摊 |
| 二叉搜索树 | O(log n) 均摊 |
| 二叉堆插入/删除 | O(log n) |
| 排序（比较型下界） | Ω(n log n) |
| BFS/DFS | O(V + E) |
| Dijkstra（堆优化） | O(E log V) |
| Floyd | O(V^3) |
| 矩阵乘法 | O(n^3)（朴素） |
| 快速幂 | O(log n) |
