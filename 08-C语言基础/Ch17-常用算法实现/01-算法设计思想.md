# 算法设计思想概述

## 1. 算法设计的核心目标

算法设计的核心目标是在有限的时间和空间资源内解决问题。好的算法应当具备**正确性**、**可读性**、**健壮性**和**高效性**。在学习具体算法之前，理解算法设计的基本思想至关重要。

## 2. 分治法（Divide and Conquer）

分治法将一个复杂问题分解为若干个规模较小的子问题，递归地求解这些子问题，然后合并结果。

### 核心步骤
1. **分解（Divide）**：将原问题分解为若干子问题
2. **解决（Conquer）**：递归求解子问题
3. **合并（Combine）**：将子问题的解合并为原问题的解

### 经典示例：归并排序

```c
#include <stdio.h>
#include <stdlib.h>

void merge(int arr[], int left, int mid, int right) {
    int n1 = mid - left + 1;
    int n2 = right - mid;
    int *L = (int *)malloc(n1 * sizeof(int));
    int *R = (int *)malloc(n2 * sizeof(int));

    for (int i = 0; i < n1; i++) L[i] = arr[left + i];
    for (int j = 0; j < n2; j++) R[j] = arr[mid + 1 + j];

    int i = 0, j = 0, k = left;
    while (i < n1 && j < n2) {
        if (L[i] <= R[j]) arr[k++] = L[i++];
        else arr[k++] = R[j++];
    }
    while (i < n1) arr[k++] = L[i++];
    while (j < n2) arr[k++] = R[j++];
    free(L); free(R);
}

void mergeSort(int arr[], int left, int right) {
    if (left < right) {
        int mid = left + (right - left) / 2;
        mergeSort(arr, left, mid);
        mergeSort(arr, mid + 1, right);
        merge(arr, left, mid, right);
    }
}
```

## 3. 贪心法（Greedy）

贪心法在每一步选择中都采取当前状态下最优的选择，期望通过局部最优达到全局最优。

### 适用条件
- **贪心选择性质**：局部最优能导致全局最优
- **最优子结构**：问题的最优解包含子问题的最优解

### 经典示例：活动选择问题

```c
#include <stdio.h>

// 活动结构体
typedef struct {
    int start;
    int end;
} Activity;

// 按结束时间排序的比较函数
int cmp(const void *a, const void *b) {
    return ((Activity *)a)->end - ((Activity *)b)->end;
}

int activitySelection(Activity activities[], int n) {
    qsort(activities, n, sizeof(Activity), cmp);
    int count = 1;
    int lastEnd = activities[0].end;
    printf("选择活动: [%d, %d]\n", activities[0].start, activities[0].end);

    for (int i = 1; i < n; i++) {
        if (activities[i].start >= lastEnd) {
            count++;
            lastEnd = activities[i].end;
            printf("选择活动: [%d, %d]\n", activities[i].start, activities[i].end);
        }
    }
    return count;
}
```

## 4. 动态规划（Dynamic Programming）

动态规划通过保存子问题的解避免重复计算，适用于具有**重叠子问题**和**最优子结构**的问题。

### 核心要素
- **状态定义**：确定子问题的表示方式
- **状态转移方程**：描述状态之间的关系
- **边界条件**：最小子问题的解
- **计算顺序**：确保计算某状态时其依赖的状态已求出

### 经典示例：斐波那契数列（DP版）

```c
#include <stdio.h>

long long fibonacci(int n) {
    if (n <= 1) return n;
    long long dp[n + 1];
    dp[0] = 0; dp[1] = 1;
    for (int i = 2; i <= n; i++) {
        dp[i] = dp[i - 1] + dp[i - 2];
    }
    return dp[n];
}
```

## 5. 回溯法（Backtracking）

回溯法通过探索所有可能的候选解来找出所有解或最优解。当发现当前候选解不可能是解时，就回退（回溯）到上一步继续搜索。

### 核心框架

```c
void backtrack(参数列表) {
    if (满足结束条件) {
        记录结果;
        return;
    }
    for (选择 in 选择列表) {
        做出选择;
        backtrack(新参数);
        撤销选择;
    }
}
```

## 6. 各种思想的对比

| 算法思想 | 适用场景 | 时间复杂度特点 | 代表算法 |
|---------|---------|--------------|---------|
| 分治法 | 子问题独立 | O(n log n) | 归并排序、快速排序 |
| 贪心法 | 局部最优即全局最优 | 通常较低 | 哈夫曼编码、Dijkstra |
| 动态规划 | 重叠子问题、最优子结构 | 通常多项式 | 背包、LCS、编辑距离 |
| 回溯法 | 组合、排列、子集问题 | 通常指数级 | N皇后、全排列 |

## 7. 总结

掌握这些算法设计思想是解决复杂问题的基础。在实际应用中，需要根据问题的特性选择合适的方法，有时还需要将多种思想结合起来使用。后续章节将对每种思想进行深入讲解。
