# 数组的常见错误

## 1. 数组越界访问

### 1.1 循环差1错误

```c
// 错误：i <= n 导致 arr[n] 越界
int arr[10];
for (int i = 0; i <= 10; i++) {  // 当i=10时越界！
    arr[i] = 0;
}

// 正确：i < n
for (int i = 0; i < 10; i++) {
    arr[i] = 0;
}
```

### 1.2 字符串结尾越界

```c
char buf[5];
strcpy(buf, "Hello");  // "Hello"需要6字节（含'\0'），buf只有5字节
// 后果：覆盖相邻内存

// 正确做法
strncpy(buf, "Hello", sizeof(buf) - 1);
buf[sizeof(buf) - 1] = '\0';
```

## 2. 使用未初始化的数组

```c
// 错误：局部数组的值不确定
void func(void) {
    int arr[10];
    int sum = 0;
    for (int i = 0; i < 10; i++) {
        sum += arr[i];  // arr[i]的值不确定！
    }
}

// 正确：初始化数组
int arr[10] = {0};  // 全部初始化为0
```

### 2.1 全局 vs 局部的区别

```c
int global[10];           // 全局：自动初始化为0

void func(void) {
    static int st[10];    // 静态：自动初始化为0
    int local[10];        // 局部：值不确定！
}
```

## 3. sizeof 的误用

### 3.1 在函数中使用 sizeof

```c
// 错误：函数参数中数组退化为指针
void print_array(int arr[], int n) {
    // sizeof(arr) 是指针大小，不是数组大小！
    int len = sizeof(arr) / sizeof(arr[0]);  // 错误！
    for (int i = 0; i < len; i++)
        printf("%d ", arr[i]);
}

// 正确：显式传递长度
void print_array(int arr[], int n) {
    for (int i = 0; i < n; i++)
        printf("%d ", arr[i]);
}
```

### 3.2 宏的副作用

```c
#define LEN(arr) (sizeof(arr) / sizeof(arr[0]))

// 当arr是指针时错误
void func(int *arr) {
    int n = LEN(arr);  // 错误！结果是指针大小/元素大小
}
```

## 4. 返回局部数组的地址

```c
// 错误：返回局部数组的地址
int* create_array(void) {
    int arr[5] = {1, 2, 3, 4, 5};
    return arr;  // 危险！函数返回后arr的内存已无效
}

// 正确：使用static或malloc
int* create_array_ok(void) {
    static int arr[5] = {1, 2, 3, 4, 5};
    return arr;
}

// 或者
int* create_array_malloc(void) {
    int *arr = malloc(5 * sizeof(int));
    // ...初始化...
    return arr;  // 调用者负责free
}
```

## 5. 数组退化陷阱

```c
int a[5] = {1, 2, 3, 4, 5};
int b[5] = {1, 2, 3, 4, 5};

// 错误：比较地址，不是内容
if (a == b) {  // 永远为false，比较的是地址
    printf("相等\n");
}

// 正确：逐元素比较
int equal = 1;
for (int i = 0; i < 5; i++) {
    if (a[i] != b[i]) { equal = 0; break; }
}
```

### 5.1 数组不能赋值

```c
int a[5] = {1, 2, 3, 4, 5};
int b[5];
// b = a;  // 编译错误！数组不能直接赋值

// 正确
memcpy(b, a, sizeof(a));
```

## 6. 整数溢出导致越界

```c
int n = 100000;
int arr[n];  // 如果n太大，可能栈溢出

// 计算索引时的溢出
int i = INT_MAX;
arr[i % 5];  // 结果可能为负数！
```

## 7. 数组大小计算错误

```c
// 错误：用 sizeof(ptr) 代替 sizeof(arr)
void func(int arr[100]) {
    size_t bytes = sizeof(arr);  // 8（指针大小），不是400！
}

// 正确：在定义处计算
int arr[100];
size_t bytes = sizeof(arr);  // 400
```

## 8. 多维数组的列数遗漏

```c
// 错误：函数参数必须指定列数
void func(int rows, int cols, int arr[][]) {  // 编译错误

// 正确
#define COLS 10
void func(int rows, int arr[][COLS]) {  // 必须指定列数

// 或使用VLA (C99)
void func(int rows, int cols, int arr[rows][cols]) {
```

## 9. 缓冲区溢出安全漏洞

```c
// 危险：永远不要信任输入长度
char name[20];
scanf("%s", name);  // 如果输入超过19个字符就溢出！

// 安全
scanf("%19s", name);  // 最多读取19个字符
// 或使用 fgets
fgets(name, sizeof(name), stdin);
```

## 10. 常见错误速查表

| 错误 | 后果 | 修正 |
|------|------|------|
| `arr[n]` 越界 | 未定义行为 | 改为 `i < n` |
| 局部数组未初始化 | 值不确定 | `= {0}` 或逐个赋值 |
| 函数中 sizeof(arr) | 指针大小 | 显式传递长度 |
| 返回局部数组地址 | 悬空指针 | static / malloc |
| 数组直接比较 | 比较地址 | 逐元素或 memcmp |
| 数组直接赋值 | 编译错误 | memcpy |

---

## 重点总结

- **越界**是最常见的错误，始终验证索引范围
- 局部数组**不自动初始化**，使用前必须赋值
- 函数中**sizeof 不能获取数组长度**，必须显式传递
- 不要返回局部数组的地址
- 数组不能直接比较或赋值，需要逐元素操作或使用标准库函数
