# 数组的边界检查

## 1. 越界访问的危险

C语言**不自动检查数组边界**，越界访问是**未定义行为（Undefined Behavior）**，后果不可预测。

```c
int arr[5] = {1, 2, 3, 4, 5};

arr[5] = 10;   // 越界！未定义行为
arr[-1] = 20;  // 越界！未定义行为
```

### 1.1 越界可能导致的后果

| 后果 | 说明 |
|------|------|
| 数据破坏 | 覆盖相邻变量的值 |
| 栈溢出 | 破坏栈帧，函数返回异常 |
| 程序崩溃 | 段错误（Segmentation Fault） |
| 安全漏洞 | 缓冲区溢出攻击 |
| 静默错误 | 数据悄悄被改，程序输出错误结果 |

```c
#include <stdio.h>

int main(void) {
    int x = 10;
    int arr[3] = {1, 2, 3};
    int y = 20;

    // 可能覆盖 x 或 y（取决于编译器布局）
    arr[3] = 999;
    arr[4] = 888;

    // x 或 y 的值可能已被破坏
    printf("x = %d, y = %d\n", x, y);

    return 0;
}
```

## 2. 缓冲区溢出攻击原理

```c
// 危险代码示例（不要在实际中使用）
void vulnerable(char *input) {
    char buffer[16];
    strcpy(buffer, input);  // 无长度检查！
}
```

如果 `input` 超过16字节，会覆盖栈上的返回地址，攻击者可以控制程序执行流程。

## 3. 防御措施

### 3.1 手动边界检查

```c
#include <stdio.h>

int safe_get(const int arr[], int n, int index) {
    if (index < 0 || index >= n) {
        fprintf(stderr, "下标越界: %d (有效范围 0-%d)\n", index, n - 1);
        return -1;  // 或使用其他错误处理方式
    }
    return arr[index];
}

void safe_set(int arr[], int n, int index, int value) {
    if (index < 0 || index >= n) {
        fprintf(stderr, "下标越界: %d\n", index);
        return;
    }
    arr[index] = value;
}
```

### 3.2 使用安全的字符串函数

```c
#include <string.h>

char dest[10];

// 危险
strcpy(dest, "这是一个很长的字符串");  // 越界！

// 安全替代
strncpy(dest, "这是一个很长的字符串", sizeof(dest) - 1);
dest[sizeof(dest) - 1] = '\0';  // 确保以null结尾
```

**安全函数对照表：**

| 危险函数 | 安全替代 |
|---------|---------|
| `strcpy` | `strncpy` / `strlcpy` |
| `strcat` | `strncat` / `strlcat` |
| `sprintf` | `snprintf` |
| `gets` | `fgets`（`gets`已在C11中移除） |
| `scanf("%s")` | `scanf("%9s", buf)` |

### 3.3 封装安全数组

```c
#include <stdio.h>
#include <stdlib.h>
#include <assert.h>

typedef struct {
    int *data;
    int size;
} SafeArray;

SafeArray *sa_create(int size) {
    SafeArray *sa = malloc(sizeof(SafeArray));
    sa->data = calloc(size, sizeof(int));
    sa->size = size;
    return sa;
}

int sa_get(const SafeArray *sa, int index) {
    if (index < 0 || index >= sa->size) {
        fprintf(stderr, "越界访问: index=%d, size=%d\n", index, sa->size);
        exit(EXIT_FAILURE);  // 严重错误，终止程序
    }
    return sa->data[index];
}

void sa_set(SafeArray *sa, int index, int value) {
    if (index < 0 || index >= sa->size) {
        fprintf(stderr, "越界写入: index=%d, size=%d\n", index, sa->size);
        exit(EXIT_FAILURE);
    }
    sa->data[index] = value;
}

void sa_destroy(SafeArray *sa) {
    free(sa->data);
    free(sa);
}
```

### 3.4 使用断言（Debug模式）

```c
#include <assert.h>

int get_element(const int arr[], int n, int index) {
    assert(index >= 0 && index < n);  // Debug模式下检查
    return arr[index];
}
```

> `assert` 在 Release 编译（定义了 `NDEBUG`）时会被消除。

## 4. 常见越界场景

```c
// 场景1：循环差1错误
int arr[10];
for (int i = 0; i <= 10; i++) {  // i=10时越界！
    arr[i] = 0;
}

// 场景2：忘记字符串结尾的'\0'
char buf[5];
strcpy(buf, "hello");  // 需要6字节（含'\0'），溢出！

// 场景3：指针算术越界
int arr[5];
int *p = arr;
p += 10;  // p已越界，解引用p是UB

// 场景4：负数下标
int arr[5];
arr[-1] = 0;  // 未定义行为
```

---

## 重点总结

- C语言**不检查数组边界**，越界是**未定义行为**
- 越界可能导致数据破坏、崩溃、安全漏洞
- 防御方法：手动边界检查、使用安全函数、封装安全数组
- 使用 `strncpy`、`snprintf`、`fgets` 等安全替代函数
- Debug模式下可使用 `assert` 捕获越界错误
- **永远不要假设输入长度是安全的**
