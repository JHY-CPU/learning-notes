# 数组性能优化

## 1. 缓存友好的访问模式

### 1.1 行优先 vs 列优先

C语言按**行优先**存储二维数组，行优先遍历的缓存命中率远高于列优先。

```c
#define N 10000
int matrix[N][N];

// 行优先遍历 —— 快（缓存友好）
for (int i = 0; i < N; i++)
    for (int j = 0; j < N; j++)
        sum += matrix[i][j];

// 列优先遍历 —— 慢（缓存不友好）
for (int j = 0; j < N; j++)
    for (int i = 0; i < N; i++)
        sum += matrix[i][j];
```

**性能差异可能达到 10 倍以上！**

```
行优先: 访问 matrix[0][0], matrix[0][1], matrix[0][2] ...
        连续内存 → 缓存命中率高

列优先: 访问 matrix[0][0], matrix[1][0], matrix[2][0] ...
        每次跳过一整行 → 缓存命中率低 → 频繁缓存未命中
```

### 1.2 证明缓存影响

```c
#include <stdio.h>
#include <time.h>

#define N 8000

int matrix[N][N];

int main(void) {
    // 初始化
    for (int i = 0; i < N; i++)
        for (int j = 0; j < N; j++)
            matrix[i][j] = i + j;

    volatile int sum = 0;

    // 行优先
    clock_t t1 = clock();
    for (int i = 0; i < N; i++)
        for (int j = 0; j < N; j++)
            sum += matrix[i][j];
    clock_t t2 = clock();

    // 列优先
    sum = 0;
    clock_t t3 = clock();
    for (int j = 0; j < N; j++)
        for (int i = 0; i < N; i++)
            sum += matrix[i][j];
    clock_t t4 = clock();

    printf("行优先: %.3f ms\n", (double)(t2-t1)/CLOCKS_PER_SEC*1000);
    printf("列优先: %.3f ms\n", (double)(t4-t3)/CLOCKS_PER_SEC*1000);

    return 0;
}
```

## 2. 循环展开

减少循环控制开销，提高指令级并行。

```c
// 普通循环
int sum = 0;
for (int i = 0; i < n; i++) {
    sum += arr[i];
}

// 展开4倍
int sum = 0;
int i;
for (i = 0; i + 3 < n; i += 4) {
    sum += arr[i] + arr[i+1] + arr[i+2] + arr[i+3];
}
for (; i < n; i++) {  // 处理剩余元素
    sum += arr[i];
}
```

## 3. 减少分支预测失败

```c
// 带条件分支 —— 分支预测可能失败
for (int i = 0; i < n; i++) {
    if (arr[i] > 0)
        sum += arr[i];
}

// 无分支写法（使用位运算避免分支）
for (int i = 0; i < n; i++) {
    int mask = arr[i] >> 31;  // 负数为-1，非负数为0
    sum += arr[i] & ~mask;    // 只加非负数
}
```

## 4. 数据局部性优化

### 4.1 结构体数组 vs 数组结构体

```c
// 方式A：结构体数组 (AoS)
struct Point { float x, y, z; };
struct Point points[N];

// 如果只需要x坐标，每次访问跳过y和z
// 缓存利用率低

// 方式B：数组结构体 (SoA)
struct Points {
    float x[N];
    float y[N];
    float z[N];
};

// 只访问x时，内存连续，缓存利用率高
```

当只处理部分数据时，**SoA** 通常更高效。

### 4.2 分块处理

```c
// 大矩阵乘法优化：分块
#define BLOCK 64
for (int ii = 0; ii < n; ii += BLOCK)
    for (int jj = 0; jj < n; jj += BLOCK)
        for (int kk = 0; kk < n; kk += BLOCK)
            for (int i = ii; i < ii+BLOCK && i < n; i++)
                for (int j = jj; j < jj+BLOCK && j < n; j++)
                    for (int k = kk; k < kk+BLOCK && k < n; k++)
                        C[i][j] += A[i][k] * B[k][j];
```

## 5. 编译器优化提示

```c
// restrict 关键字：告诉编译器指针不重叠
void add(int *restrict dst, const int *restrict a,
         const int *restrict b, int n) {
    for (int i = 0; i < n; i++)
        dst[i] = a[i] + b[i];  // 编译器可以向量化
}
```

## 6. 减少函数调用开销

```c
// 内联小函数
static inline int get_element(const int arr[], int i) {
    return arr[i];
}

// 或者直接写表达式
sum += arr[i];  // 比 sum += get_element(arr, i) 更快
```

## 7. 使用 const 优化

```c
// const 允许编译器做更多优化
int sum_array(const int arr[], int n) {
    int total = 0;
    for (int i = 0; i < n; i++) {
        total += arr[i];
    }
    return total;
    // 编译器知道 arr 不会被修改，可以安全优化
}
```

## 8. 性能优化清单

| 优化策略 | 效果 | 适用场景 |
|---------|------|---------|
| 行优先遍历 | 高 | 二维数组 |
| 循环展开 | 中 | 计算密集循环 |
| 避免分支 | 中 | 条件过滤 |
| 数据布局(SoA) | 高 | 部分字段访问 |
| 分块处理 | 高 | 矩阵运算 |
| restrict 指针 | 中 | 向量化优化 |
| 使用 const | 低 | 编译器优化 |
| 缓存对齐 | 中 | 大规模数据 |

---

## 重点总结

- **缓存友好**是最重要的优化维度，行优先遍历是基本原则
- 循环展开减少分支开销，但会增大代码体积
- SoA布局对SIMD向量化更友好
- `restrict` 帮助编译器生成更优代码
- 先写出正确代码，再用性能分析工具找到瓶颈后优化
