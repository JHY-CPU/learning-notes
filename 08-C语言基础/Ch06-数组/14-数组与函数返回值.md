# 数组与函数返回值

## 1. 核心规则

**C语言不能从函数返回数组。**

函数的返回类型不能是数组类型。以下写法都是非法的：

```c
// 非法：不能返回数组
int[5] func(void);          // 语法错误
int[] func(void);           // 语法错误
```

## 2. 不能返回局部数组

```c
// 错误写法：返回局部数组的地址
int* bad_func(void) {
    int arr[5] = {1, 2, 3, 4, 5};
    return arr;  // 危险！arr是局部变量，函数返回后内存无效
}
```

**为什么这是未定义行为：**
- `arr` 在栈上分配
- 函数返回后，栈帧被回收
- 返回的地址指向已释放的内存
- 调用者访问这块内存是未定义行为

```
函数执行时:  [栈帧: arr在有效内存中]
函数返回后:  [栈帧已释放，arr位置可能被覆盖]
调用者使用:  [访问无效内存 → 未定义行为！]
```

## 3. 合法的返回方式

### 3.1 返回指针 + 调用者提供数组（推荐）

```c
// 调用者提供目标数组
void square(const int src[], int dst[], int n) {
    for (int i = 0; i < n; i++) {
        dst[i] = src[i] * src[i];
    }
}

int main(void) {
    int a[] = {1, 2, 3, 4, 5};
    int b[5];
    square(a, b, 5);  // b = {1, 4, 9, 16, 25}
    return 0;
}
```

### 3.2 使用 static 局部数组

```c
// 使用static数组，生命周期延长到程序结束
int* get_static_array(void) {
    static int arr[5] = {1, 2, 3, 4, 5};
    return arr;
}
```

**注意事项：**
```c
int* func1(void) {
    static int arr[5] = {0};
    for (int i = 0; i < 5; i++) arr[i]++;
    return arr;
}

int* func2(void) {
    static int arr[5] = {0};
    for (int i = 0; i < 5; i++) arr[i] += 10;
    return arr;
}

int main(void) {
    int *p1 = func1();  // p1 = {1,1,1,1,1}
    int *p2 = func2();  // p2 = {11,11,11,11,11}
    // 但此时 p1 也指向 p2 的内容！static数组是唯一的
    // p1 == p2，因为两个函数的static数组共享同一块地址
    // （实际是不同的static变量，但容易造成混淆）
    return 0;
}
```

**最大问题：不是线程安全的，且所有调用共享同一数组。**

### 3.3 动态分配（malloc）

```c
#include <stdlib.h>

int* create_array(int n) {
    int *arr = malloc(n * sizeof(int));
    if (arr == NULL) return NULL;

    for (int i = 0; i < n; i++) {
        arr[i] = i * i;
    }
    return arr;
}

int main(void) {
    int *arr = create_array(5);
    if (arr == NULL) {
        fprintf(stderr, "内存分配失败\n");
        return 1;
    }

    for (int i = 0; i < 5; i++)
        printf("%d ", arr[i]);  // 0 1 4 9 16

    free(arr);  // 必须释放！
    arr = NULL;
    return 0;
}
```

**注意事项：**
- 调用者**必须记得free**，否则内存泄漏
- 返回 `NULL` 时调用者需检查

### 3.4 使用结构体包裹

```c
#include <stdio.h>

#define SIZE 5

typedef struct {
    int data[SIZE];
} IntArray;

IntArray create_array(void) {
    IntArray result = {{1, 2, 3, 4, 5}};
    return result;  // 结构体可以按值返回
}

int main(void) {
    IntArray a = create_array();
    for (int i = 0; i < SIZE; i++)
        printf("%d ", a.data[i]);
    return 0;
}
```

> 结构体可以按值返回（整体复制），但不适合大数组。

## 4. 各种方案对比

| 方案 | 优点 | 缺点 |
|------|------|------|
| 调用者提供数组 | 安全、无内存管理 | 调用者需预分配 |
| static数组 | 简单 | 非线程安全、共享状态 |
| malloc动态分配 | 大小灵活 | 需要free、可能失败 |
| 结构体包裹 | 按值返回、安全 | 大数组复制开销 |

## 5. 最佳实践

```c
// 推荐模式1：调用者提供输出缓冲区
int compute(const int input[], int output[], int n);

// 推荐模式2：返回动态分配的数组（配合长度输出参数）
int* generate(int n, int *out_len);

// 推荐模式3：小数组用结构体
typedef struct { int x, y, z; } Vec3;
Vec3 add(Vec3 a, Vec3 b);
```

---

## 重点总结

- C语言**不能直接返回数组**，函数返回类型不能是数组
- **不要返回局部数组的地址**，会导致未定义行为
- 推荐方案：调用者提供目标数组（最安全）
- 动态分配需要调用者负责 `free`
- static数组简单但非线程安全
- 小固定大小数组可考虑用结构体包裹
