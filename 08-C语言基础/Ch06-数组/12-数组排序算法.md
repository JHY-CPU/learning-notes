# 数组排序算法

## 1. 冒泡排序（Bubble Sort）

### 1.1 原理

重复遍历数组，比较相邻元素，如果顺序不对则交换。每轮遍历将最大元素"冒泡"到末尾。

```
第1轮: [5, 3, 8, 1] → [3, 5, 1, 8]   8到位
第2轮: [3, 5, 1, 8] → [3, 1, 5, 8]   5到位
第3轮: [3, 1, 5, 8] → [1, 3, 5, 8]   3到位
```

### 1.2 实现

```c
#include <stdio.h>

void bubble_sort(int arr[], int n) {
    for (int i = 0; i < n - 1; i++) {
        int swapped = 0;
        for (int j = 0; j < n - 1 - i; j++) {
            if (arr[j] > arr[j + 1]) {
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
                swapped = 1;
            }
        }
        if (!swapped) break;  // 优化：无交换则已有序
    }
}
```

### 1.3 复杂度分析

| 指标 | 值 |
|------|-----|
| 最好时间 | O(n)（已有序，加优化后） |
| 最坏时间 | O(n²) |
| 平均时间 | O(n²) |
| 空间 | O(1) |
| 稳定性 | **稳定** |

## 2. 选择排序（Selection Sort）

### 2.1 原理

每轮从未排序部分选出最小元素，放到已排序部分的末尾。

```
[5, 3, 8, 1, 2]
  ^--最小是1，与位置0交换
[1, 3, 8, 5, 2]
     ^--最小是2，与位置1交换
[1, 2, 8, 5, 3]
        ^--最小是3，与位置2交换
[1, 2, 3, 5, 8]
```

### 2.2 实现

```c
void selection_sort(int arr[], int n) {
    for (int i = 0; i < n - 1; i++) {
        int min_idx = i;
        for (int j = i + 1; j < n; j++) {
            if (arr[j] < arr[min_idx]) {
                min_idx = j;
            }
        }
        if (min_idx != i) {
            int temp = arr[i];
            arr[i] = arr[min_idx];
            arr[min_idx] = temp;
        }
    }
}
```

### 2.3 复杂度分析

| 指标 | 值 |
|------|-----|
| 最好时间 | O(n²) |
| 最坏时间 | O(n²) |
| 平均时间 | O(n²) |
| 空间 | O(1) |
| 稳定性 | **不稳定** |

## 3. 插入排序（Insertion Sort）

### 3.1 原理

将数组分为已排序和未排序两部分，每次从未排序部分取一个元素，插入到已排序部分的正确位置。

```
[5, 3, 8, 1]
 i=1: 3插入5前面  → [3, 5, 8, 1]
 i=2: 8已在正确位置 → [3, 5, 8, 1]
 i=3: 1插入最前面  → [1, 3, 5, 8]
```

### 3.2 实现

```c
void insertion_sort(int arr[], int n) {
    for (int i = 1; i < n; i++) {
        int key = arr[i];
        int j = i - 1;

        // 将大于key的元素后移
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];
            j--;
        }
        arr[j + 1] = key;
    }
}
```

### 3.3 复杂度分析

| 指标 | 值 |
|------|-----|
| 最好时间 | O(n)（已有序） |
| 最坏时间 | O(n²) |
| 平均时间 | O(n²) |
| 空间 | O(1) |
| 稳定性 | **稳定** |

> 插入排序对**近乎有序**的数组非常高效，实际中常用于小规模数组的排序。

## 4. 三种排序算法对比测试

```c
#include <stdio.h>
#include <string.h>
#include <time.h>

void bubble_sort(int arr[], int n);
void selection_sort(int arr[], int n);
void insertion_sort(int arr[], int n);

void test_sort(const char *name,
               void (*sort_fn)(int[], int),
               int arr[], int n) {
    int copy[n];
    memcpy(copy, arr, n * sizeof(int));

    clock_t start = clock();
    sort_fn(copy, n);
    clock_t end = clock();

    double time_ms = (double)(end - start) / CLOCKS_PER_SEC * 1000;
    printf("%s: %.3f ms\n", name, time_ms);
}

int main(void) {
    int arr[1000];
    srand(42);
    for (int i = 0; i < 1000; i++) {
        arr[i] = rand() % 10000;
    }

    test_sort("冒泡排序", bubble_sort, arr, 1000);
    test_sort("选择排序", selection_sort, arr, 1000);
    test_sort("插入排序", insertion_sort, arr, 1000);

    return 0;
}
```

## 5. 选择合适的排序算法

| 场景 | 推荐算法 | 原因 |
|------|---------|------|
| 小数组(<50) | 插入排序 | 实现简单，常数因子小 |
| 基本有序 | 插入排序 | 接近O(n) |
| 需要稳定性 | 冒泡/插入 | 保持相等元素的相对顺序 |
| 内存受限 | 三种均可 | 都是O(1)额外空间 |
| 大数组 | 快速排序/归并排序 | O(n log n)更高效 |

---

## 重点总结

- **冒泡排序**：简单但效率低，有提前终止优化
- **选择排序**：交换次数最少，但不稳定
- **插入排序**：对小数组和近乎有序的数组效率最高
- 三种算法都是 O(n²)，适合教学和小数据量
- 大数据量应使用标准库 `qsort()` 或更高级算法
