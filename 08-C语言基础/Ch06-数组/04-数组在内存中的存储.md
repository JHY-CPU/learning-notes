# 数组在内存中的存储

## 1. 数组的内存模型

数组是C语言中最高效的数据结构之一，其核心优势在于**连续内存布局**。理解数组的内存存储方式对编写高效程序至关重要。

### 1.1 一维数组的内存布局

```c
int arr[5] = {10, 20, 30, 40, 50};
```

假设 `arr` 的起始地址为 `0x1000`，`int` 类型占4字节：

```
内存地址:  0x1000  0x1004  0x1008  0x100C  0x1010
           ┌──────┬──────┬──────┬──────┬──────┐
    值:    │  10  │  20  │  30  │  40  │  50  │
           └──────┴──────┴──────┴──────┴──────┘
    下标:    [0]    [1]    [2]    [3]    [4]
```

### 1.2 验证内存连续性

```c
#include <stdio.h>

int main(void) {
    int arr[5] = {10, 20, 30, 40, 50};

    printf("各元素的地址:\n");
    for (int i = 0; i < 5; i++) {
        printf("arr[%d] 地址: %p, 值: %d\n",
               i, (void *)&arr[i], arr[i]);
    }

    printf("\n相邻元素地址差: %td 字节\n",
           (char *)&arr[1] - (char *)&arr[0]);

    return 0;
}
```

输出示例：
```
各元素的地址:
arr[0] 地址: 000000e8f3cff7a0, 值: 10
arr[1] 地址: 000000e8f3cff7a4, 值: 20
arr[2] 地址: 000000e8f3cff7a8, 值: 30
arr[3] 地址: 000000e8f3cff7ac, 值: 40
arr[4] 地址: 000000e8f3cff7b0, 值: 50

相邻元素地址差: 4 字节
```

## 2. 数组名与地址

### 2.1 数组名的值

**数组名在大多数表达式中会被隐式转换为数组首元素的地址**（即 `&arr[0]`）。

```c
int arr[5] = {10, 20, 30, 40, 50};

// 以下三个值相同
printf("%p\n", (void *)arr);       // 数组名 → 首元素地址
printf("%p\n", (void *)&arr[0]);   // 第一个元素的地址
printf("%p\n", (void *)&arr);      // 整个数组的地址
```

虽然这三个值相同（地址数值一样），但它们的**类型不同**：
- `arr` 的类型是 `int *`（指向 int 的指针）
- `&arr` 的类型是 `int (*)[5]`（指向长度为5的int数组的指针）

### 2.2 &arr 与 arr 的区别

```c
#include <stdio.h>

int main(void) {
    int arr[5] = {10, 20, 30, 40, 50};

    printf("arr       = %p\n", (void *)arr);
    printf("arr + 1   = %p\n", (void *)(arr + 1));
    printf("&arr      = %p\n", (void *)&arr);
    printf("&arr + 1  = %p\n", (void *)(&arr + 1));

    return 0;
}
```

输出：
```
arr       = 0x7ffd1234ab00
arr + 1   = 0x7ffd1234ab04    ← 前进4字节（一个int）
&arr      = 0x7ffd1234ab00
&arr + 1  = 0x7ffd1234ab14    ← 前进20字节（5个int）
```

**关键区别：**
- `arr + 1`：前进 `sizeof(int)` 字节（4字节），指向下一个元素
- `&arr + 1`：前进 `sizeof(int[5])` 字节（20字节），跳过整个数组

## 3. 数组元素的地址计算

### 3.1 地址计算公式

对于 `int arr[n]`，元素 `arr[i]` 的地址：
```
&arr[i] = 基地址 + i × sizeof(int)
```

对于通用情况：
```
&arr[i] = (char *)arr + i × sizeof(arr[0])
```

### 3.2 指针算术验证

```c
#include <stdio.h>

int main(void) {
    double arr[4] = {1.1, 2.2, 3.3, 4.4};

    printf("sizeof(double) = %zu\n", sizeof(double));

    for (int i = 0; i < 4; i++) {
        printf("arr[%d] 地址: %p\n", i, (void *)&arr[i]);
    }

    // 手动计算 arr[2] 的地址
    double *p = (double *)((char *)arr + 2 * sizeof(double));
    printf("手动计算arr[2]: %p, 值: %.1f\n", (void *)p, *p);

    return 0;
}
```

## 4. 数组在不同存储区的位置

```c
// 全局数组 —— 存储在数据段(.data)或BSS段
int global_arr[100];             // BSS段（未初始化）
int global_init[10] = {1,2,3};   // .data段（已初始化）

void func(void) {
    // 静态数组 —— 存储在数据段
    static int static_arr[50];    // BSS段

    // 局部数组 —— 存储在栈上
    int local_arr[20];            // 栈

    // 动态分配的"数组" —— 存储在堆上
    int *heap_arr = malloc(100 * sizeof(int));  // 堆
    free(heap_arr);
}
```

```
内存布局示意：

高地址  ┌──────────────────┐
        │      栈区        │  ← 局部数组 int a[10];
        │    （向下增长）    │
        ├──────────────────┤
        │                  │
        │      堆区        │  ← malloc 分配的数组
        │    （向上增长）    │
        │                  │
        ├──────────────────┤
        │   .data 段       │  ← 已初始化的全局/静态数组
        ├──────────────────┤
        │   .bss 段        │  ← 未初始化的全局/静态数组
        ├──────────────────┤
        │   .text 段       │  ← 程序代码
低地址  └──────────────────┘
```

## 5. 缓存与性能

数组的连续存储使得其对CPU缓存非常友好。

### 5.1 缓存行（Cache Line）

现代CPU缓存以**缓存行**（通常64字节）为单位加载数据。当访问 `arr[0]` 时，CPU会同时将 `arr[0]` 到 `arr[15]`（64/4=16个int）加载到缓存中。

```c
// 顺序访问 —— 缓存命中率高，快
for (int i = 0; i < n; i++) {
    sum += arr[i];
}

// 随机访问 —— 缓存命中率低，慢
for (int i = 0; i < n; i++) {
    sum += arr[rand() % n];
}
```

### 5.2 不同类型数组的内存占用

```c
#include <stdio.h>

int main(void) {
    char   a[5] = {0};
    int    b[5] = {0};
    double c[5] = {0};

    printf("char[5]  占 %zu 字节\n", sizeof(a));   // 5
    printf("int[5]   占 %zu 字节\n", sizeof(b));   // 20
    printf("double[5]占 %zu 字节\n", sizeof(c));   // 40

    printf("char相邻元素地址差:  %td\n",
           (char *)&a[1] - (char *)&a[0]);          // 1
    printf("int相邻元素地址差:   %td\n",
           (char *)&b[1] - (char *)&b[0]);          // 4
    printf("double相邻元素地址差: %td\n",
           (char *)&c[1] - (char *)&c[0]);          // 8

    return 0;
}
```

---

## 重点总结

- 数组元素在内存中**连续排列**，这是支持O(1)随机访问的基础
- `arr` 和 `&arr` 的值相同但**类型不同**，指针加减时步长不同
- 地址公式：`&arr[i] = 基地址 + i × sizeof(元素类型)`
- 全局数组在数据段，局部数组在栈上，动态数组在堆上
- 连续存储使数组对CPU缓存友好，顺序访问效率高
