# 数组概述

## 1. 什么是数组

数组（Array）是C语言中最基本的**构造数据类型**，它是由**相同类型**的元素按**一定顺序**排列组成的集合。数组中的每个元素通过一个统一的名称（数组名）和一个下标（索引）来唯一标识。

```c
int scores[5] = {90, 85, 78, 92, 88};
//  scores[0]=90  scores[1]=85  scores[2]=78  scores[3]=92  scores[4]=88
```

## 2. 数组的三大核心特点

### 2.1 连续内存

数组的元素在内存中是**连续存储**的，这是数组最本质的特征。

```
内存地址示意：
┌────────┬────────┬────────┬────────┬────────┐
│  1000  │  1004  │  1008  │  1012  │  1016  │
│ 90     │ 85     │ 78     │ 92     │ 88     │
│scores[0]│scores[1]│scores[2]│scores[3]│scores[4]│
└────────┴────────┴────────┴────────┴────────┘
   每个int占4字节，地址连续递增
```

**连续存储带来的优势：**
- 支持随机访问：通过下标O(1)时间访问任意元素
- 缓存友好：相邻元素在CPU缓存中命中率高
- 内存紧凑：无额外指针开销

### 2.2 元素类型相同

数组的所有元素必须是**同一数据类型**，不允许混合类型。

```c
int    a[5] = {1, 2, 3, 4, 5};       // 合法：全是int
double b[3] = {1.1, 2.2, 3.3};       // 合法：全是double
// int c[3] = {1, 2.5, "hello"};     // 非法：类型不一致
```

### 2.3 大小固定

数组在声明时必须确定其大小（VLA除外），一旦定义，**大小不可改变**。

```c
int arr[10];  // 固定10个元素，运行时不能变为15个或5个
```

## 3. 数组的分类

| 分类方式 | 类型 | 示例 |
|---------|------|------|
| 按维度 | 一维数组 | `int a[5];` |
| | 二维数组 | `int b[3][4];` |
| | 多维数组 | `int c[2][3][4];` |
| 按存储 | 静态数组 | `int a[100];` |
| | 动态数组 | `int *p = malloc(n * sizeof(int));` |
| 按元素类型 | 整型数组 | `int a[5];` |
| | 浮点数组 | `float f[5];` |
| | 字符数组 | `char s[20];` |
| | 指针数组 | `char *argv[10];` |
| | 结构体数组 | `struct Student stu[30];` |

## 4. 数组的内存布局

### 4.1 一维数组的内存模型

```c
int arr[4] = {10, 20, 30, 40};
```

假设 `arr` 起始地址为 `0x1000`，每个 `int` 占4字节：

```
地址：  0x1000  0x1004  0x1008  0x100C
值：     10      20      30      40
下标：   [0]     [1]     [2]     [3]
```

**元素地址计算公式：**
```
&arr[i] = 起始地址 + i × sizeof(元素类型)
```

### 4.2 地址与值的关系

```c
#include <stdio.h>

int main(void) {
    int arr[4] = {10, 20, 30, 40};

    printf("数组首地址:  %p\n", (void *)arr);
    printf("arr[0]地址:  %p\n", (void *)&arr[0]);
    printf("arr[1]地址:  %p\n", (void *)&arr[1]);
    printf("arr[2]地址:  %p\n", (void *)&arr[2]);

    printf("相邻元素地址差: %zu 字节\n",
           (char *)&arr[1] - (char *)&arr[0]);

    return 0;
}
```

输出示例：
```
数组首地址:  0000007f3a4ff8a0
arr[0]地址:  0000007f3a4ff8a0   ← 与arr相同
arr[1]地址:  0000007f3a4ff8a4   ← 相差4字节
arr[2]地址:  0000007f3a4ff8a8   ← 相差8字节
相邻元素地址差: 4 字节
```

## 5. 数组与其他数据结构的对比

| 特性 | 数组 | 链表 | 动态数组(vector) |
|------|------|------|-----------------|
| 内存布局 | 连续 | 分散 | 连续 |
| 随机访问 | O(1) | O(n) | O(1) |
| 插入/删除 | O(n) | O(1) | O(n) |
| 大小 | 固定 | 动态 | 动态 |
| 缓存友好 | 好 | 差 | 好 |
| 额外开销 | 无 | 每节点有指针 | 容量管理 |

## 6. 为什么要学习数组

1. **基础数据结构**：几乎所有程序都会用到数组
2. **底层理解**：理解数组有助于理解指针、内存管理
3. **高效存储**：连续内存带来优秀的缓存性能
4. **广泛应用**：矩阵运算、查找表、缓冲区、字符串等

---

## 重点总结

- 数组是**同类型**元素的**连续内存**集合，大小**固定**
- 数组下标从 **0** 开始，到 `n-1` 结束
- 数组的连续存储特性使其支持 **O(1) 随机访问**
- 数组名代表数组首地址，`arr` 等价于 `&arr[0]`
- C语言的数组**不自带边界检查**，越界是未定义行为
