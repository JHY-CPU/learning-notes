# 07 - 变量的作用域

## 一、什么是作用域

**作用域（Scope）** 指的是变量在程序中可以被访问的区域。C 语言中变量的作用域由其**定义的位置**决定。

```
作用域类型
├── 文件作用域（File Scope）
├── 函数作用域（Function Scope）
├── 块作用域（Block Scope）
└── 函数原型作用域（Function Prototype Scope）
```

---

## 二、局部变量（块作用域）

### 2.1 基本概念

在函数内部或代码块 `{}` 内定义的变量称为**局部变量**，只在该块内可见：

```c
#include <stdio.h>

void func(void) {
    int x = 10;       // x 的作用域：func 函数内部
    printf("x = %d\n", x);
}

int main(void) {
    // printf("%d\n", x);  // 错误！x 在 main 中不可见

    func();

    {
        int y = 20;     // y 的作用域：这个代码块内
        printf("y = %d\n", y);
    }
    // printf("%d\n", y);  // 错误！y 在代码块外不可见

    return 0;
}
```

### 2.2 嵌套块与变量遮蔽

```c
#include <stdio.h>

int main(void) {
    int x = 10;
    printf("外层 x = %d\n", x);    // 10

    {
        int x = 20;                  // 遮蔽（shadow）了外层的 x
        printf("内层 x = %d\n", x); // 20

        {
            int x = 30;              // 再次遮蔽
            printf("最内层 x = %d\n", x);  // 30
        }

        printf("内层 x = %d\n", x); // 20（内层 x 恢复）
    }

    printf("外层 x = %d\n", x);    // 10（外层 x 从未改变）
    return 0;
}
```

> **注意**：变量遮蔽虽然合法，但容易造成混淆，建议在实际编程中避免使用同名变量。

## 三、全局变量（文件作用域）

### 3.1 基本概念

在所有函数外部定义的变量称为**全局变量**，从定义处到文件末尾都可见：

```c
#include <stdio.h>

int global_count = 0;    // 全局变量，文件作用域

void increment(void) {
    global_count++;      // 函数内可以访问全局变量
}

void print_count(void) {
    printf("count = %d\n", global_count);
}

int main(void) {
    increment();
    increment();
    increment();
    print_count();       // count = 3
    return 0;
}
```

### 3.2 全局变量的初始化

- 全局变量**未显式初始化时自动初始化为 0**
- 全局变量的初始化值必须是**常量表达式**

```c
int a;              // 自动初始化为 0
double b;           // 自动初始化为 0.0
char c;             // 自动初始化为 '\0'
int *p;             // 自动初始化为 NULL

int x = 10;         // 正确
int y = x + 5;      // C99 之前错误！不是常量表达式
```

### 3.3 全局变量的优缺点

**优点**：
- 在多个函数之间共享数据方便
- 全局自动初始化为 0

**缺点**：
- 降低代码的可维护性和可读性
- 增加函数之间的耦合
- 多线程环境下不安全
- 不利于代码复用和测试

> **建议**：尽量少用全局变量，优先使用函数参数和返回值传递数据。

## 四、函数作用域

`goto` 语句的标签具有**函数作用域**，在整个函数内都可见：

```c
void func(void) {
    goto label;    // 跳转到 label

    {
        // label 在此也可见（虽然不建议这样做）
        int x = 10;
    }

label:
    printf("到达标签\n");
}
```

> 函数作用域只适用于标签，实际编程中应尽量避免使用 `goto`。

---

## 五、函数原型作用域

函数声明（原型）中的参数名具有**函数原型作用域**，只在声明的括号内有效：

```c
// 参数名可以省略，也可以随意命名
int add(int a, int b);       // 正确
int add(int, int);           // 也正确（省略参数名）
int add(int x, int y);       // 也正确（名字不同也没关系）
```

---

## 六、作用域的层次关系

```
文件作用域（全局变量）
  └── 函数作用域
        └── 块作用域（局部变量）
              └── 嵌套块作用域
```

内层作用域可以访问外层作用域的变量，反之不行：

```c
int g = 100;           // 文件作用域

void func(void) {
    int f = 200;       // 函数作用域（实际上是块作用域）

    {
        int b = 300;   // 块作用域
        // 可以访问 g, f, b
        printf("%d %d %d\n", g, f, b);
    }

    // 可以访问 g, f，不能访问 b
    printf("%d %d\n", g, f);
}

// 只能访问 g
```

---

## 七、要点总结

1. **局部变量**：在函数或代码块内定义，只在该块内可见
2. **全局变量**：在函数外定义，在整个文件中可见（及通过 `extern` 在其他文件中可见）
3. 内层作用域可以**遮蔽（shadow）** 外层同名变量，但应避免
4. 全局变量自动初始化为 0，局部变量不自动初始化
5. `goto` 标签具有函数作用域
6. 函数原型中的参数名具有原型作用域，可以省略
7. 尽量减少全局变量的使用，以降低耦合
8. 变量应在**最小必要作用域**内定义，遵循就近原则
