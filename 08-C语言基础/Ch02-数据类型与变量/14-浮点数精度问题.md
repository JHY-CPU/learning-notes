# 14 - 浮点数精度问题

## 一、浮点数的本质

计算机使用**有限的二进制位**来表示实数，而实数是**无限密集**的。这意味着绝大多数实数无法精确表示，只能近似存储。

```
实数轴：  无穷多个值
二进制：  有限个表示（2^32 个 float，2^64 个 double）
```

---

## 二、精度丢失的经典案例

### 2.1 0.1 + 0.2 不等于 0.3

```c
#include <stdio.h>

int main(void) {
    double a = 0.1;
    double b = 0.2;
    double c = a + b;

    printf("0.1 + 0.2 = %.20f\n", c);
    // 输出：0.1 + 0.2 = 0.30000000000000004441
    printf("c == 0.3 ? %s\n", c == 0.3 ? "true" : "false");
    // 输出：false

    return 0;
}
```

**原因**：0.1 在二进制中是无限循环小数：
```
0.1(十进制) = 0.000110011001100110011...(二进制，无限循环)
```

由于浮点数只能存储有限位，截断后就产生了误差。

### 2.2 大数吃小数

```c
#include <stdio.h>

int main(void) {
    double big = 1e16;
    double small = 1.0;

    double result = big + small;
    printf("1e16 + 1.0 = %.1f\n", result);
    // 输出：10000000000000000.0（1 被"吃掉"了）

    // 对比
    double a = 1.0 + 1e-16;
    printf("1.0 + 1e-16 = %.20f\n", a);
    // 1e-16 相对于 1.0 太小，可能丢失

    return 0;
}
```

## 三、浮点数的正确比较

### 3.1 绝对误差比较

```c
#include <math.h>

// 不可靠的比较
if (a == b) { /* ... */ }

// 绝对误差比较
double epsilon = 1e-9;
if (fabs(a - b) < epsilon) {
    printf("在误差范围内相等\n");
}
```

### 3.2 相对误差比较

```c
// 当数值可能很大或很小时，使用相对误差
bool nearly_equal(double a, double b, double rel_eps) {
    double diff = fabs(a - b);
    double max_val = fmax(fabs(a), fabs(b));
    return diff <= rel_eps * max_val;
}
```

### 3.3 使用 DBL_EPSILON

```c
#include <float.h>
#include <math.h>

bool equal_with_epsilon(double a, double b) {
    // 对于接近 1.0 的数
    return fabs(a - b) < DBL_EPSILON;
}

// 更通用的方式
bool roughly_equal(double a, double b) {
    double diff = fabs(a - b);
    double max_val = fmax(fabs(a), fabs(b));
    return diff <= DBL_EPSILON * fmax(1.0, max_val);
}
```

## 四、舍入误差的累积

```c
#include <stdio.h>

int main(void) {
    // 累加 0.1 一万次，期望结果为 1000.0
    double sum = 0.0;
    for (int i = 0; i < 10000; i++) {
        sum += 0.1;
    }
    printf("sum = %.15f (期望 1000.0)\n", sum);
    // 结果可能偏离期望值

    return 0;
}
```

### 4.1 Kahan 求和算法

```c
// Kahan 求和：减少浮点累加误差
double kahan_sum(const double *data, int n) {
    double sum = 0.0;
    double compensation = 0.0;

    for (int i = 0; i < n; i++) {
        double y = data[i] - compensation;
        double t = sum + y;
        compensation = (t - sum) - y;  // 补偿低位误差
        sum = t;
    }
    return sum;
}
```

## 五、特殊浮点值

### 5.1 特殊值表

| 值 | 含义 | 如何产生 |
|----|------|----------|
| `NaN` | 非数字 | `0.0/0.0`, `sqrt(-1.0)` |
| `+Inf` | 正无穷 | `1.0/0.0`, 溢出 |
| `-Inf` | 负无穷 | `-1.0/0.0` |
| `+0.0` | 正零 | 正常下溢 |
| `-0.0` | 负零 | 负数下溢 |

### 5.2 检测特殊值

```c
#include <stdio.h>
#include <math.h>

int main(void) {
    double nan_val = 0.0 / 0.0;
    double inf_val = 1.0 / 0.0;
    double neg_inf = -1.0 / 0.0;
    double neg_zero = -0.0;

    // NaN 的特性：不等于任何值，包括自身
    printf("NaN == NaN? %d\n", nan_val == nan_val);      // 0 (false)
    printf("isnan(NaN)? %d\n", isnan(nan_val));           // 非零 (true)

    // Inf 的检测
    printf("isinf(+Inf)? %d\n", isinf(inf_val));          // 正数 (true)
    printf("isinf(-Inf)? %d\n", isinf(neg_inf));          // 负数 (true)

    // 负零
    printf("neg_zero == 0.0? %d\n", neg_zero == 0.0);    // 1 (true)
    printf("signbit(-0.0)? %d\n", signbit(neg_zero));     // 非零 (true)

    return 0;
}
```

### 5.3 NaN 的传播

```c
double nan_val = 0.0 / 0.0;
double result = nan_val + 1.0;    // result 也是 NaN
double result2 = nan_val * 0.0;   // result2 也是 NaN
```

## 六、浮点数的有效位数

```c
#include <stdio.h>
#include <float.h>

int main(void) {
    printf("float 有效位数: %d\n", FLT_DIG);     // 6
    printf("double 有效位数: %d\n", DBL_DIG);    // 15
    printf("long double 有效位数: %d\n", LDBL_DIG);

    // 超过有效位数的数字会被截断
    float f = 123456789.0f;   // 只能保留约 7 位有效数字
    printf("float: %.0f\n", f);  // 可能不是精确值

    return 0;
}
```

## 七、避免精度问题的建议

1. **使用 `double` 而非 `float`**：除非有特殊的内存要求
2. **不要比较浮点数是否相等**：使用误差范围比较
3. **避免大数与小数混合运算**
4. **累加时对数据排序**：从小到大累加可以减少误差
5. **使用 Kahan 求和**：精确的浮点累加
6. **金融计算使用整数**：用"分"为单位而非"元"

```c
// 金融计算：用整数表示"分"
int price_cents = 999;    // 9.99 元
int tax_cents = 60;       // 0.60 元
int total_cents = price_cents + tax_cents;  // 精确！
printf("总价: %d.%02d 元\n", total_cents / 100, total_cents % 100);
```

## 八、要点总结

1. 浮点数使用**有限的二进制位**近似表示实数，存在固有精度限制
2. 0.1、0.2 等简单小数在二进制中**无法精确表示**
3. **永远不要用 `==` 比较浮点数**，应使用误差范围
4. 大数和小数混合运算会导致**精度丢失**
5. `NaN` 不等于任何值（包括自身），用 `isnan()` 检测
6. 金融计算使用**整数**表示最小货币单位
7. Kahan 求和算法可以有效减少累加误差
8. `double` 的精度优于 `float`，一般场景推荐使用
