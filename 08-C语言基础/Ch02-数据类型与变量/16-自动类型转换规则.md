# 16 - 自动类型转换规则

## 一、自动类型转换概述

C 语言中的**自动类型转换**（又称隐式类型转换）是编译器在不需要程序员显式指定的情况下，自动将一种类型转换为另一种类型。这是 C 语言灵活性的体现，但也可能带来隐患。

---

## 二、赋值转换

赋值运算符 `=` 右侧的值会被转换为左侧变量的类型：

```c
int a = 3.14;         // 截断：a = 3
double b = 42;        // 提升：b = 42.0
char c = 300;         // 溢出：c = 44（300 % 256）
short s = 100000;     // 溢出：s 的值不确定
```

### 2.1 整型间赋值

```c
// 大类型 → 小类型：截断高位
int big = 0x12345678;
short medium = big;     // medium = 0x5678
char small = big;       // small = 0x78

// 无符号 → 有符号：可能溢出
unsigned int u = UINT_MAX;
int i = u;              // i = -1（在补码系统中）
```

### 2.2 浮点与整型间赋值

```c
// 浮点 → 整型：截断小数部分
int x = 3.99;           // x = 3（不是四舍五入）
int y = -3.99;          // y = -3

// 整型 → 浮点：可能损失精度
float f = 16777217;     // float 尾数 23 位，可能不精确
double d = 16777217;    // double 可以精确表示
```

## 三、算术转换

### 3.1 二元运算符的转换

当二元运算符的两个操作数类型不同时，遵循**寻常算术转换（Usual Arithmetic Conversion）**：

```
转换阶梯（从低到高）：

char/short
    ↓
  int
    ↓
 unsigned int
    ↓
  long
    ↓
 unsigned long
    ↓
 long long
    ↓
 unsigned long long
    ↓
  float
    ↓
  double
    ↓
long double
```

### 3.2 具体规则

```c
// 规则：较小的类型转换为较大的类型
int + double → double + double
char + int → int + int
unsigned int + long → 依赖于 sizeof(int) 和 sizeof(long)
```

```c
#include <stdio.h>

int main(void) {
    int i = -5;
    unsigned int u = 5;

    // i 被转换为 unsigned int
    // -5 的补码表示作为无符号数是一个很大的正数
    if (i + u > 0) {
        printf("结果为正\n");   // 实际输出！
    }

    // 混合类型的 sizeof
    char c = 1;
    short s = 2;
    int a = 3;
    double d = 4.0;

    printf("c + s: %zu\n", sizeof(c + s));    // 4（int）
    printf("c + a: %zu\n", sizeof(c + a));    // 4（int）
    printf("a + d: %zu\n", sizeof(a + d));    // 8（double）

    return 0;
}
```

## 四、函数参数转换

### 4.1 默认参数提升

在调用函数时，某些类型会自动提升：

```c
// 有函数原型时：按原型指定的类型转换
void func(double x);    // 原型
func(42);               // 42 转换为 double

// 无函数原型时（旧式 C）：默认参数提升
// char, short → int
// float → double
```

### 4.2 可变参数函数

`printf`、`scanf` 等可变参数函数中，参数会进行默认提升：

```c
char c = 'A';
float f = 3.14;

printf("%c %f\n", c, f);
// c 提升为 int，f 提升为 double
// printf 根据格式串读取
```

## 五、表达式中的转换实例

### 5.1 混合运算

```c
char a = 10;
short b = 20;
int c = 30;
double d = 1.5;

// 表达式：a + b * c / d
// 步骤：
// 1. a → int, b → int（整型提升）
// 2. b * c → int
// 3. int / d → double（算术转换）
// 4. a + double → double
double result = a + b * c / d;  // 10.0 + 400.0 = 410.0
```

### 5.2 条件表达式

```c
// 三目运算符的类型转换
int a = 5;
double b = 3.0;
// 条件表达式的结果类型是两个分支类型的"公共类型"
double result = (a > 3) ? a : b;  // a 转换为 double
```

## 六、整型提升的特殊情况

### 6.1 位运算中的提升

```c
char c = 0xFF;   // 如果 char 是有符号的，c = -1

// ~c 的计算过程：
// 1. c 提升为 int：-1 的 int 表示为 0xFFFFFFFF
// 2. ~0xFFFFFFFF = 0x00000000 = 0
// 3. 结果是 int 类型的 0
int result = ~c;

printf("result = %d\n", result);   // 0（不是 0！）
```

> **注意**：位运算时 `char` 提升为 `int`，可能导致符号扩展。

### 6.2 sizeof 表达式

```c
char c = 1;
// sizeof(c + c) 的结果是 sizeof(int) = 4
// 因为 c 提升为 int 参与运算
printf("%zu\n", sizeof(c + c));  // 4

// 对比：sizeof(c) 返回 char 的大小
printf("%zu\n", sizeof(c));      // 1
```

## 七、转换的安全性

| 转换方向 | 安全性 | 说明 |
|----------|--------|------|
| 小整型 → 大整型 | 安全 | 值保持不变 |
| 有符号 → 无符号 | 注意 | 负数变大正数 |
| 大整型 → 小整型 | 危险 | 可能截断 |
| 浮点 → 整型 | 危险 | 截断小数 |
| 整型 → 浮点 | 通常安全 | 可能损失精度 |

---

## 八、要点总结

1. 赋值时右侧表达式自动转换为左侧变量类型
2. 混合类型运算遵循算术转换规则（小类型转大类型）
3. `char` 和 `short` 在运算时先提升为 `int`
4. 有符号与无符号混合运算时，有符号值转为无符号值
5. 函数参数会进行默认参数提升
6. 位运算时注意整型提升可能导致的符号扩展
7. 理解自动转换规则有助于避免隐蔽的 bug
8. 不确定时使用显式转换（cast）来表明意图
