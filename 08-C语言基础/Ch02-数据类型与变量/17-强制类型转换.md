# 17 - 强制类型转换

## 一、强制转换的基本语法

**强制类型转换**（Type Cast）是程序员通过显式语法将一种类型强制转换为另一种类型：

```c
(type_name) expression
```

```c
int a = 5, b = 2;
double result = (double)a / b;   // 将 a 转为 double

printf("%.2f\n", result);        // 2.50
```

---

## 二、整型之间的强制转换

### 2.1 大类型转小类型（截断）

```c
int big = 0x12345678;
short s = (short)big;    // s = 0x5678（取低 16 位）
char c = (char)big;      // c = 0x78（取低 8 位）

long long ll = 1234567890123456789LL;
int i = (int)ll;         // 截断，i 的值不确定
```

### 2.2 有符号与无符号转换

```c
int neg = -1;
unsigned int pos = (unsigned int)neg;  // pos = 4294967295

unsigned int big = 4294967295U;
int small = (int)big;                   // small = -1（补码解释）
```

---

## 三、浮点与整型之间的转换

### 3.1 浮点转整型

```c
double pi = 3.14159;
int truncated = (int)pi;     // truncated = 3（截断，不四舍五入）

double neg = -3.14;
int neg_truncated = (int)neg; // -3（向零截断）

// 注意：超出整型范围的浮点数转换是未定义行为
double huge = 1e20;
int bad = (int)huge;         // 未定义行为！
```

### 3.2 整型转浮点

```c
int x = 12345;
double d = (double)x;       // d = 12345.0

// 大整数转 float 可能损失精度
long long big = 100000001LL;
float f = (float)big;       // 可能不精确（float 只有 23 位尾数）
```

## 四、指针类型转换

### 4.1 void* 与其他指针

```c
// malloc 返回 void*，需要转换
int *arr = (int *)malloc(10 * sizeof(int));

// void* 可以赋给任意指针类型（C 中隐式转换，C++ 需要 cast）
void *ptr = arr;
int *p = (int *)ptr;   // 显式转换（良好习惯）
```

### 4.2 不同类型指针转换

```c
int x = 0x12345678;
char *cp = (char *)&x;

// 查看 x 在内存中的字节序
for (int i = 0; i < sizeof(int); i++) {
    printf("byte[%d] = 0x%02X\n", i, (unsigned char)cp[i]);
}
// 小端系统输出：byte[0]=0x78, byte[1]=0x56, byte[2]=0x34, byte[3]=0x12
```

### 4.3 整数与指针转换

```c
// 指针转整数（平台相关）
int x = 42;
uintptr_t addr = (uintptr_t)&x;   // 将地址转为整数

// 整数转指针
int *p = (int *)addr;             // 转回指针
printf("%d\n", *p);               // 42
```

> **注意**：指针与整数的转换高度依赖平台，应避免在跨平台代码中使用。

## 五、const 和 volatile 的转换

### 5.1 去除 const

```c
const int x = 10;
int *p = (int *)&x;     // 去除 const（危险！）
*p = 20;                 // 未定义行为！

// x 可能存储在只读内存中，修改会导致段错误
```

### 5.2 添加 const（安全）

```c
int x = 10;
const int *p = &x;      // 安全：添加 const 是允许的
```

## 六、函数指针的转换

```c
#include <stdio.h>

void greet(void) {
    printf("Hello!\n");
}

int main(void) {
    // 函数指针的转换
    void (*fp)(void) = greet;

    // 转为通用指针
    void *vp = (void *)fp;

    // 转回函数指针
    void (*fp2)(void) = (void (*)(void))vp;
    fp2();   // Hello!

    return 0;
}
```

> **注意**：函数指针之间的转换结果是实现定义的。

## 七、常见陷阱

### 7.1 隐式转换导致的 bug

```c
unsigned int len = 10;
for (int i = len - 1; i >= 0; i--) {
    // 当 i = -1 时，i >= 0 的比较：-1 转为 unsigned，变成很大的正数
    // 可能导致无限循环
}
```

### 7.2 精度丢失

```c
double value = 3.14159;
int result = (int)value;   // 丢失小数部分
// 有时这是期望的行为，有时是 bug
```

### 7.3 数据截断

```c
long long a = 0x0000000100000001LL;
int b = (int)a;            // b = 1（高位被截断）
```

## 八、强制转换的最佳实践

### 8.1 何时需要强制转换

```c
// 1. 整数除法需要浮点结果
int a = 7, b = 2;
double avg = (double)a / b;    // 必须转换

// 2. malloc 返回值（C 中可省略，C++ 中必须）
int *p = (int *)malloc(n * sizeof(int));

// 3. 回调函数的 void* 参数
void callback(void *data) {
    int *value = (int *)data;  // 必须转换
}
```

### 8.2 何时不需要强制转换

```c
// 1. 小类型到大类型的自动提升
char c = 'A';
int i = c;             // 不需要 cast

// 2. 整型到浮点的赋值
int n = 42;
double d = n;          // 不需要 cast

// 3. 任意指针到 void*（C 中）
int *p = malloc(100);  // C 中合法，C++ 中需要 cast
```

---

## 九、要点总结

1. 强制转换使用 `(type)` 语法，显式进行类型转换
2. 浮点转整数**截断**小数部分（不四舍五入）
3. 指针类型转换可以用于查看内存布局，但要注意对齐
4. 去除 `const` 后修改数据是**未定义行为**
5. 尽量减少强制转换，因为它可能隐藏真正的类型错误
6. `malloc` 在 C 中不需要强制转换（`void*` 自动转换），但 C++ 中需要
7. 整数与指针的转换是平台相关的，应避免跨平台代码使用
8. 强制转换是最后手段，优先考虑修改设计来避免类型不匹配
