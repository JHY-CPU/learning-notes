# 13 - 整数溢出与回绕

## 一、什么是整数溢出

当运算结果超出了类型的**表示范围**时，就会发生**整数溢出（Integer Overflow）**。整数溢出是 C 语言中最常见且最危险的 bug 之一，可能导致安全漏洞。

---

## 二、有符号整数溢出

### 2.1 未定义行为

在 C 标准中，**有符号整数溢出是未定义行为（Undefined Behavior, UB）**。这意味着编译器可以做任何处理：

```c
#include <stdio.h>
#include <limits.h>

int main(void) {
    int a = INT_MAX;     // 2147483647 (32位)
    int b = a + 1;       // 未定义行为！

    printf("a = %d\n", a);
    printf("a + 1 = %d\n", b);  // 可能是 -2147483648（回绕），也可能不

    int c = INT_MIN;     // -2147483648
    int d = c - 1;       // 未定义行为！
    printf("c - 1 = %d\n", d);

    return 0;
}
```

### 2.2 编译器优化的隐患

由于有符号溢出是 UB，编译器可能会基于"不会溢出"的假设进行优化：

```c
// 编译器可能优化掉溢出检查！
int add(int a, int b) {
    int result = a + b;
    if (result < a) {    // 编译器可能认为这永远不会发生
        // 处理溢出
    }
    return result;
}
```

## 三、无符号整数回绕

无符号整数溢出不会产生 UB，而是**回绕（Wrap Around）**—— 从 0 重新开始或从最大值回退：

```c
#include <stdio.h>
#include <limits.h>

int main(void) {
    unsigned int a = UINT_MAX;   // 4294967295
    unsigned int b = a + 1;      // 回绕到 0

    printf("UINT_MAX = %u\n", a);
    printf("UINT_MAX + 1 = %u\n", b);    // 0

    unsigned int c = 0;
    unsigned int d = c - 1;      // 回绕到 UINT_MAX
    printf("0 - 1 = %u\n", d);           // 4294967295

    return 0;
}
```

C 标准规定：无符号整数运算以 `2^N` 为模（N 为位数），结果总是可预测的。

## 四、常见溢出场景

### 4.1 加法溢出

```c
int a = INT_MAX, b = 1;
int sum = a + b;     // 有符号溢出（UB）
```

### 4.2 乘法溢出

```c
int a = 100000, b = 100000;
int product = a * b;  // 100亿，超过 int 范围
```

### 4.3 减法溢出

```c
int a = INT_MIN, b = 1;
int diff = a - b;     // 有符号溢出（UB）
```

### 4.4 取反溢出

```c
int a = INT_MIN;
int neg = -a;         // INT_MIN 的绝对值超过 INT_MAX
```

## 五、溢出检测方法

### 5.1 加法溢出检测

```c
#include <stdio.h>
#include <limits.h>
#include <stdbool.h>

bool safe_add(int a, int b, int *result) {
    if (b > 0 && a > INT_MAX - b) {
        return false;    // 正溢出
    }
    if (b < 0 && a < INT_MIN - b) {
        return false;    // 负溢出
    }
    *result = a + b;
    return true;
}

int main(void) {
    int result;
    if (safe_add(INT_MAX, 1, &result)) {
        printf("结果: %d\n", result);
    } else {
        printf("加法溢出！\n");
    }
    return 0;
}
```

### 5.2 乘法溢出检测

```c
#include <limits.h>
#include <stdbool.h>

bool safe_multiply(int a, int b, int *result) {
    if (a == 0 || b == 0) {
        *result = 0;
        return true;
    }
    if (a > 0 && b > 0 && a > INT_MAX / b) return false;
    if (a > 0 && b < 0 && b < INT_MIN / a) return false;
    if (a < 0 && b > 0 && a < INT_MIN / b) return false;
    if (a < 0 && b < 0 && a < INT_MAX / b) return false;
    *result = a * b;
    return true;
}
```

### 5.3 使用 GCC/Clang 内建函数

```c
// GCC 和 Clang 提供了带溢出检测的内建函数
int result;
if (__builtin_add_overflow(INT_MAX, 1, &result)) {
    printf("溢出！\n");
}
// __builtin_mul_overflow(a, b, &result)
// __builtin_sub_overflow(a, b, &result)
```

### 5.4 C23 标准函数（<stdckdint.h>）

```c
#include <stdckdint.h>   // C23

int result;
if (ckd_add(&result, INT_MAX, 1)) {
    printf("溢出！\n");
}
```

## 六、无符号整数的回绕陷阱

```c
// 经典错误：死循环
unsigned int i;
for (i = 10; i >= 0; i--) {   // 永远为真！
    printf("%u\n", i);        // 0 之后回绕到 UINT_MAX
}

// 正确写法
for (i = 10; i != (unsigned int)-1; i--) {
    printf("%u\n", i);
}
// 或者
for (i = 10; i <= 10; i--) {  // 当 i 回绕时会大于 10
    printf("%u\n", i);
    if (i == 0) break;         // 手动检查
}
```

```c
// 另一个常见陷阱
unsigned int a = 5, b = 10;
if (a - b < 0) {        // 永远为假！无符号数不小于 0
    printf("负数\n");    // 永远不会执行
}

if (a < b) {            // 正确的比较方式
    printf("a 小于 b\n");
}
```

## 七、安全编程建议

1. **使用更大的类型**：如果可能溢出，使用更大范围的类型
2. **输入验证**：运算前检查输入值是否合理
3. **使用安全函数**：实现带溢出检查的运算函数
4. **启用编译警告**：`-Wconversion -Wsign-conversion`
5. **避免混用 signed/unsigned**

```c
// 使用更大类型避免溢出
int a = INT_MAX, b = 1;
long long sum = (long long)a + b;   // 安全
```

---

## 八、要点总结

1. **有符号整数溢出**是未定义行为，结果不可预测
2. **无符号整数溢出**会回绕，行为有定义但可能非预期
3. 通过运算前的范围检查来检测溢出
4. GCC/Clang 提供 `__builtin_*_overflow` 内建函数
5. `INT_MIN` 的取反操作也会溢出
6. 无符号数永远不会小于 0，递减循环要特别注意
7. 编译器可能利用"有符号不溢出"假设进行优化
8. 安全关键代码必须实现溢出检测
