# 08 - 变量的生命周期

## 一、什么是生命周期

**生命周期（Lifetime）** 指的是变量从创建（分配内存）到销毁（释放内存）的时间段。作用域决定了变量在哪里可以访问，而生命周期决定了变量在何时存在。

---

## 二、存储类别与生命周期

C 语言有四种存储类别，决定了变量的生命周期和链接属性：

| 存储类别 | 关键字 | 存储位置 | 生命周期 | 默认初始化 |
|----------|--------|----------|----------|------------|
| 自动变量 | `auto` | 栈 | 进入块到退出块 | 不初始化（垃圾值） |
| 静态变量 | `static` | 数据段 | 程序运行全程 | 初始化为 0 |
| 寄存器变量 | `register` | CPU 寄存器 | 进入块到退出块 | 不初始化 |
| 外部变量 | `extern` | 数据段 | 程序运行全程 | 初始化为 0 |

---

## 三、自动变量（auto）

### 3.1 基本概念

函数内定义的变量默认就是**自动变量**，关键字 `auto` 可省略：

```c
void func(void) {
    int x = 10;           // 等价于 auto int x = 10;
    auto int y = 20;      // 显式声明为 auto（不常见）

    // x 和 y 在函数结束时被销毁
}
```

### 3.2 生命周期

- **创建时机**：程序执行到变量定义处时
- **销毁时机**：离开变量所在的代码块时
- **存储位置**：通常在**栈（stack）** 上

```c
#include <stdio.h>

void counter(void) {
    int count = 0;     // 每次调用都重新创建
    count++;
    printf("count = %d\n", count);
}

int main(void) {
    counter();  // count = 1
    counter();  // count = 1（不是 2！）
    counter();  // count = 1
    return 0;
}
```

---

## 四、静态变量（static）

### 4.1 局部静态变量

在函数内部使用 `static` 修饰的变量，**只初始化一次**，在程序运行期间一直存在：

```c
#include <stdio.h>

void counter(void) {
    static int count = 0;  // 只在第一次调用时初始化
    count++;
    printf("count = %d\n", count);
}

int main(void) {
    counter();  // count = 1
    counter();  // count = 2
    counter();  // count = 3
    return 0;
}
```

### 4.2 局部静态变量的特点

```c
void func(void) {
    static int x = 100;   // 只执行一次
    // 等价于：
    // 编译器在程序启动时分配 x 并初始化为 100
    // 此后每次进入 func，x 保留上次的值

    x += 10;
    printf("x = %d\n", x);
}
```

| 特性 | 自动变量 | 静态局部变量 |
|------|----------|--------------|
| 初始化 | 每次进入块时 | 仅一次（程序启动时） |
| 生命周期 | 进入块 ~ 退出块 | 程序运行全程 |
| 默认值 | 垃圾值 | 0 |
| 存储位置 | 栈 | 数据段 |

---

## 五、寄存器变量（register）

### 5.1 基本概念

`register` 建议编译器将变量存储在 CPU 寄存器中以提高访问速度：

```c
void func(void) {
    register int i;         // 建议存储在寄存器中

    for (i = 0; i < 1000000; i++) {
        // 高频访问的循环变量
    }
}
```

### 5.2 注意事项

- `register` 只是**建议**，编译器可以忽略
- 不能对寄存器变量取地址（`&i` 非法）
- 现代编译器的优化通常比手动指定 `register` 更好
- C++17 已废弃此关键字

```c
register int x = 10;
int *p = &x;    // 编译错误！不能对 register 变量取地址
```

> **建议**：现代 C 编程中很少需要使用 `register`，编译器会自动优化。

---

## 六、外部变量（extern）

外部变量具有**静态存储期**，在程序运行期间一直存在：

```c
// ===== globals.c =====
int global_value = 42;         // 定义（分配内存）
const char *app_name = "MyApp";

// ===== main.c =====
extern int global_value;       // 声明（不分配内存）
extern const char *app_name;

int main(void) {
    printf("%s: %d\n", app_name, global_value);
    return 0;
}
```

---

## 七、动态内存（malloc/free）

动态内存的生命周期由程序员**手动控制**：

```c
#include <stdio.h>
#include <stdlib.h>

int main(void) {
    // 生命周期开始：调用 malloc
    int *arr = (int *)malloc(10 * sizeof(int));

    if (arr == NULL) {
        fprintf(stderr, "内存分配失败\n");
        return 1;
    }

    // 使用 arr...
    for (int i = 0; i < 10; i++) {
        arr[i] = i * i;
    }

    // 生命周期结束：调用 free
    free(arr);
    arr = NULL;  // 避免悬空指针

    return 0;
}
```

| 特性 | 自动变量 | 静态变量 | 动态内存 |
|------|----------|----------|----------|
| 分配 | 自动 | 编译时/程序启动 | malloc/calloc |
| 释放 | 自动 | 程序结束 | free（手动） |
| 大小 | 编译时确定 | 编译时确定 | 运行时确定 |
| 存储位置 | 栈 | 数据段 | 堆 |

---

## 八、VLA（变长数组）—— C99

C99 引入了变长数组，其生命周期与自动变量相同，但大小在运行时确定：

```c
void func(int n) {
    int vla[n];          // 大小在运行时确定
    // vla 的生命周期：函数结束时释放
    // 注意：不能初始化（如 int vla[n] = {0}; 不合法）
}
```

> **注意**：VLA 在 C11 中变为可选特性，部分编译器可能不支持。

## 九、生命周期总结图

```
程序启动
  ├── 静态变量、全局变量 ──→ 分配内存
  │
main() 开始
  ├── 进入函数/块
  │     ├── 自动变量 ──→ 分配内存（栈）
  │     ├── register 变量 ──→ 分配（寄存器）
  │     └── malloc() ──→ 分配内存（堆）
  │
  ├── 退出函数/块
  │     ├── 自动变量 ──→ 销毁
  │     ├── register 变量 ──→ 销毁
  │     └── malloc 内存 ──→ 仍然存在（需手动 free）
  │
main() 结束
  └── 所有静态/全局变量 ──→ 销毁
```

---

## 十、要点总结

1. **自动变量**：进入块时创建，退出块时销毁，未初始化值不确定
2. **静态局部变量**：只初始化一次，程序运行期间一直存在
3. **全局/外部变量**：程序启动时创建，程序结束时销毁
4. **动态内存**：由 `malloc` 分配，`free` 释放，生命周期由程序员控制
5. `register` 是建议性的，现代编译器通常忽略
6. 静态变量未初始化时自动为 0，自动变量未初始化时为垃圾值
7. 使用 `static` 局部变量可以实现跨函数调用的状态保持
8. 动态分配的内存**必须手动释放**，否则造成内存泄漏
