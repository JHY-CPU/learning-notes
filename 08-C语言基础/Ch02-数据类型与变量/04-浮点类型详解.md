# 04 - 浮点类型详解

## 一、浮点类型概览

C 语言提供三种浮点类型用于表示带小数的数值：

| 类型 | 典型大小 | 精度（有效数字） | 范围（约） |
|------|----------|------------------|------------|
| `float` | 4 字节 | 6 ~ 7 位 | ±3.4×10^38 |
| `double` | 8 字节 | 15 ~ 16 位 | ±1.7×10^308 |
| `long double` | 8/12/16 字节 | 18+ 位 | 取决于实现 |

> 使用 `<float.h>` 中的宏可以获取各类型的精确参数。

---

## 二、IEEE 754 浮点表示

C 语言的浮点类型通常遵循 **IEEE 754 标准**：

### 2.1 float（32 位单精度）

```
符号位(1位) | 指数位(8位) | 尾数位(23位)
   S        |    EEEEEEEE | MMMMMMMMMMMMMMMMMMMMMMM
```

- **符号位 S**：0 为正，1 为负
- **指数 E**：8 位，偏移量 127（实际指数 = E - 127）
- **尾数 M**：23 位，隐含前导 1（1.M）

值的计算公式：`(-1)^S × 1.M × 2^(E-127)`

### 2.2 double（64 位双精度）

```
符号位(1位) | 指数位(11位) | 尾数位(52位)
```

- 指数偏移量为 1023
- 尾数 52 位，精度更高

---

## 三、浮点字面量与后缀

```c
float a = 3.14f;          // float 类型（带 f 后缀）
double b = 3.14;          // double 类型（默认）
long double c = 3.14L;    // long double 类型（带 L 后缀）

// 科学计数法
double avogadro = 6.022e23;      // 6.022 × 10^23
double small = 1.6e-19;          // 1.6 × 10^-19
float speed = 3.0e8f;            // 光速
double negative = -1.23e-4;      // -1.23 × 10^-4
```

> **注意**：没有后缀的浮点字面量默认为 `double` 类型。将 `double` 字面量赋给 `float` 变量会产生隐式转换警告（如 `-Wconversion`）。

## 四、float.h 中的宏常量

```c
#include <stdio.h>
#include <float.h>

int main(void) {
    printf("float 精度: %d 位有效数字\n", FLT_DIG);
    printf("float 范围: %e ~ %e\n", FLT_MIN, FLT_MAX);
    printf("float 最小正规数: %e\n", FLT_MIN);
    printf("float epsilon: %e\n", FLT_EPSILON);

    printf("double 精度: %d 位有效数字\n", DBL_DIG);
    printf("double 范围: %e ~ %e\n", DBL_MIN, DBL_MAX);
    printf("double epsilon: %e\n", DBL_EPSILON);

    printf("long double 精度: %d 位有效数字\n", LDBL_DIG);
    return 0;
}
```

| 宏 | 含义 |
|----|------|
| `FLT_DIG` / `DBL_DIG` | 十进制有效位数 |
| `FLT_EPSILON` | 1.0 与大于 1.0 的最小可表示值之差 |
| `FLT_MIN` / `FLT_MAX` | 最小/最大正规数 |
| `FLT_MANT_DIG` | 尾数的二进制位数 |

## 五、浮点数的精度问题

### 5.1 经典的精度丢失

```c
#include <stdio.h>

int main(void) {
    double a = 0.1;
    double b = 0.2;
    double c = a + b;

    printf("0.1 + 0.2 = %.20f\n", c);
    printf("c == 0.3 ? %s\n", c == 0.3 ? "true" : "false");

    return 0;
}
```

输出：
```
0.1 + 0.2 = 0.30000000000000004441
c == 0.3 ? false
```

**原因**：0.1 和 0.2 在二进制中无法精确表示，就像 1/3 在十进制中无法精确表示一样。

### 5.2 正确的浮点比较

```c
#include <math.h>

// 错误方式
if (a == b) { /* 不可靠 */ }

// 正确方式：使用误差范围
double epsilon = 1e-9;
if (fabs(a - b) < epsilon) {
    printf("a 和 b 在精度范围内相等\n");
}

// 或使用 FLT_EPSILON / DBL_EPSILON
if (fabs(a - b) < DBL_EPSILON * fabs(a)) {
    printf("相对误差在可接受范围内\n");
}
```

## 六、特殊浮点值

IEEE 754 定义了几个特殊值：

| 值 | 含义 | 产生方式 |
|----|------|----------|
| `NaN` | 非数字（Not a Number） | `0.0/0.0`, `sqrt(-1)` |
| `+Inf` | 正无穷 | `1.0/0.0`, 溢出 |
| `-Inf` | 负无穷 | `-1.0/0.0` |
| `+0` / `-0` | 正零和负零 | 特殊情况 |

```c
#include <stdio.h>
#include <math.h>

int main(void) {
    double nan_val = 0.0 / 0.0;
    double inf_val = 1.0 / 0.0;
    double neg_inf = -1.0 / 0.0;

    printf("NaN: %f\n", nan_val);
    printf("Inf: %f\n", inf_val);
    printf("-Inf: %f\n", neg_inf);

    // NaN 不等于任何值，包括自身
    printf("NaN == NaN? %d\n", nan_val == nan_val);  // 0 (false)

    // 使用 isnan() 和 isinf() 判断
    printf("isnan(NaN)? %d\n", isnan(nan_val));       // 非零 (true)
    printf("isinf(Inf)? %d\n", isinf(inf_val));       // 非零 (true)

    return 0;
}
```

## 七、格式化输出

```c
printf("%f\n", 3.14159);       // 3.141590（默认 6 位小数）
printf("%.2f\n", 3.14159);     // 3.14
printf("%e\n", 123456.789);    // 1.234568e+05（科学计数法）
printf("%g\n", 123.0);         // 123（自动选择 %f 或 %e）
printf("%g\n", 0.0000123);     // 1.23e-05
printf("%Lf\n", 3.14L);        // long double 用 %Lf
```

| 占位符 | 类型 |
|--------|------|
| `%f` | float / double |
| `%e` / `%E` | 科学计数法 |
| `%g` / `%G` | 自动选择最短形式 |
| `%Lf` | long double |

## 八、浮点运算的注意事项

### 8.1 累积误差

```c
// 累加小数时误差会累积
double sum = 0.0;
for (int i = 0; i < 1000000; i++) {
    sum += 0.000001;
}
printf("期望: 1.0, 实际: %.15f\n", sum);
// 实际值可能偏离 1.0
```

### 8.2 Kahan 求和算法

```c
// Kahan 求和：减少浮点累加误差
double kahan_sum(double *arr, int n) {
    double sum = 0.0;
    double compensation = 0.0;  // 补偿项

    for (int i = 0; i < n; i++) {
        double y = arr[i] - compensation;
        double t = sum + y;
        compensation = (t - sum) - y;  // 计算丢失的低位
        sum = t;
    }
    return sum;
}
```

## 九、要点总结

1. 浮点类型有 `float`（4字节）、`double`（8字节）、`long double` 三种
2. 浮点数遵循 IEEE 754 标准，使用符号位 + 指数位 + 尾数位编码
3. **不要用 `==` 比较浮点数**，应使用误差范围比较
4. 0.1 等简单小数在二进制中无法精确表示，存在精度问题
5. 注意 `NaN` 和 `Inf` 特殊值，使用 `isnan()` 和 `isinf()` 判断
6. 一般计算推荐使用 `double`，精度和性能较平衡
7. 浮点字面量默认为 `double`，`float` 需要加 `f` 后缀
8. 金融计算**不建议使用浮点数**，应使用定点数或整数模拟
