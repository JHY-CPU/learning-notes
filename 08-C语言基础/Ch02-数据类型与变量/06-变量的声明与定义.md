# 06 - 变量的声明与定义

## 一、什么是变量

变量是程序中用于存储数据的**命名内存单元**。每个变量都有一个类型，决定了它可以存储什么数据以及占用多少内存。

```c
int age = 20;       // 定义一个 int 类型变量，初始值为 20
double price = 9.9; // 定义一个 double 类型变量
char grade = 'A';   // 定义一个 char 类型变量
```

---

## 二、声明与定义的区别

### 2.1 变量定义（Definition）

变量定义会**分配内存空间**，可能同时进行初始化：

```c
int count;              // 定义，分配 4 字节内存
int value = 100;        // 定义并初始化
double pi = 3.14159;    // 定义并初始化
```

### 2.2 变量声明（Declaration）

变量声明告诉编译器"这个变量存在，但不在此处分配内存"：

```c
extern int global_count;   // 声明：global_count 在其他文件中定义
extern double pi;          // 声明：pi 在其他地方定义
```

### 2.3 声明与定义的关系

```c
// ===== file1.c =====
int global_var = 42;    // 这是定义（分配内存）

// ===== file2.c =====
extern int global_var;  // 这是声明（不分配内存）

void func(void) {
    printf("%d\n", global_var);  // 使用 file1 中定义的变量
}
```

| | 定义 | 声明 |
|---|------|------|
| 内存分配 | 是 | 否 |
| 可以多次 | 否（同一作用域） | 可以多次 |
| `extern` 关键字 | 可省略（不加 extern 即定义） | 必须使用 |
| 初始化 | 可以初始化 | 不能初始化 |

> **注意**：在函数内部，不加 `extern` 的变量声明同时也是定义。

## 三、变量的初始化

### 3.1 定义时初始化

```c
int a = 10;
double b = 3.14;
char c = 'X';
int *p = NULL;
```

### 3.2 C99 批量初始化

```c
int a = 1, b = 2, c = 3;    // 同时定义并初始化多个变量
```

### 3.3 C99 复合字面量与指定初始化

```c
// 数组指定初始化（C99）
int arr[5] = {[0] = 1, [2] = 3, [4] = 5};

// 结构体指定初始化（C99）
struct Point { int x, y; };
struct Point p = {.x = 10, .y = 20};
```

### 3.4 未初始化的局部变量

```c
void func(void) {
    int x;          // 未初始化
    printf("%d\n", x);  // 危险！x 的值是不确定的（垃圾值）
}
```

> **重要**：**局部变量**未初始化时包含**随机垃圾值**，使用未初始化的变量是 C 程序中最常见的 bug 之一。
>
> **全局变量和静态变量**会自动初始化为 **0**。

## 四、变量命名规则

### 4.1 基本规则

- 只能包含**字母、数字、下划线**（`_`）
- **不能以数字开头**
- **不能使用 C 关键字**（如 `int`、`if`、`return` 等）
- 区分大小写（`count` 和 `Count` 是不同的变量）

```c
// 合法命名
int count;
int _temp;
int maxValue;
int student_count;
int i;

// 非法命名
int 2ndPlace;     // 以数字开头
int my-var;       // 包含连字符
int int;          // 使用关键字
```

### 4.2 命名风格

| 风格 | 示例 | 使用场景 |
|------|------|----------|
| snake_case | `student_count` | Linux 内核风格、C 标准库 |
| camelCase | `studentCount` | 部分项目 |
| PascalCase | `StudentCount` | 宏或类型名 |
| UPPER_SNAKE | `MAX_SIZE` | 宏常量 |

### 4.3 命名建议

```c
// 好的命名
int line_count;         // 见名知意
FILE *input_file;       // 清晰描述用途
double temperature_celsius;

// 差的命名
int a;      // 含义不清
int tmp;    // 过于简短
int x1;     // 无意义
```

## 五、const 修饰的变量

```c
const int MAX = 100;     // 定义常量，不可修改
const double PI = 3.14159;

MAX = 200;  // 编译错误！不能修改 const 变量
```

`const` 变量必须在定义时初始化：

```c
const int x;     // 编译错误！未初始化的 const 变量
const int x = 5; // 正确
```

## 六、变量定义的位置

### 6.1 C99 之前

变量定义必须放在**块的开头**（在任何执行语句之前）：

```c
void func(void) {
    int a = 1;      // 变量定义在开头
    int b = 2;

    a = a + b;      // 执行语句在后面
    printf("%d\n", a);
}
```

### 6.2 C99 及以后

变量可以在代码的**任意位置**定义：

```c
void func(void) {
    printf("开始计算\n");

    int a = 10;          // C99 允许在使用前定义
    int b = 20;

    int sum = a + b;     // 随用随定义
    printf("sum = %d\n", sum);

    for (int i = 0; i < 5; i++) {  // C99 允许在 for 中定义
        printf("%d ", i);
    }
}
```

---

## 七、要点总结

1. **定义**分配内存，**声明**只告诉编译器变量的存在
2. 全局变量用 `extern` 声明时不会分配内存
3. **局部变量未初始化时值不确定**，全局/静态变量自动初始化为 0
4. 变量命名只能用字母、数字、下划线，不能以数字开头
5. `const` 变量必须在定义时初始化，之后不可修改
6. C99 允许在代码任意位置定义变量，建议就近定义
7. 避免使用无意义的变量名（如 `a`、`tmp`），应使用描述性名称
8. 使用未初始化的局部变量是常见的 C 语言 bug
