# 19 - 数据的存储与表示

## 一、数据在内存中的存储

计算机中所有数据最终都以**二进制**形式存储在内存中。理解数据的存储方式对于正确处理底层操作至关重要。

---

## 二、原码、反码、补码

### 2.1 原码（Sign-Magnitude）

最高位为符号位（0 正 1 负），其余位表示数值的绝对值：

```
+5 的原码：0000 0101
-5 的原码：1000 0101
+0 的原码：0000 0000
-0 的原码：1000 0000  （问题：有两个零）
```

### 2.2 反码（One's Complement）

- 正数的反码 = 原码
- 负数的反码 = 符号位不变，其余位取反

```
+5 的反码：0000 0101
-5 的反码：1111 1010
```

### 2.3 补码（Two's Complement）

- 正数的补码 = 原码
- 负数的补码 = 反码 + 1

```
+5 的补码：0000 0101
-5 的补码：1111 1011

验证：0000 0101 + 1111 1011 = 0000 0000（溢出位丢弃）= 0 ✓
```

### 2.4 为什么使用补码

```c
// 补码的优势：
// 1. 零只有一种表示：0000 0000
// 2. 加减法统一为加法运算

// 例子：5 - 3 = 5 + (-3)
// 5 的补码：  0000 0101
// -3 的补码： 1111 1101
// 相加：      0000 0010 = 2 ✓

// 3. 硬件实现简单，只需要加法器
```

### 2.5 补码的取值范围

以 8 位有符号整数为例：

| 表示 | 最小值 | 最大值 |
|------|--------|--------|
| 原码 | -127 | +127 |
| 反码 | -127 | +127 |
| 补码 | **-128** | +127 |

```
-128 的补码：1000 0000
+127 的补码：0111 1111
-1 的补码：  1111 1111
```

## 三、大端与小端

### 3.1 概念

多字节数据在内存中的存储有两种顺序：

- **大端（Big-Endian）**：高位字节存储在低地址
- **小端（Little-Endian）**：低位字节存储在低地址

以 `int x = 0x12345678` 为例：

```
地址：    低地址 ──────────────→ 高地址
大端：    [0x12] [0x34] [0x56] [0x78]
小端：    [0x78] [0x56] [0x34] [0x12]
```

### 3.2 检测当前平台的字节序

```c
#include <stdio.h>

int main(void) {
    unsigned int x = 0x12345678;
    unsigned char *p = (unsigned char *)&x;

    if (p[0] == 0x78) {
        printf("小端模式（Little-Endian）\n");
    } else if (p[0] == 0x12) {
        printf("大端模式（Big-Endian）\n");
    } else {
        printf("未知字节序\n");
    }

    return 0;
}
```

### 3.3 常见平台的字节序

| 平台 | 字节序 |
|------|--------|
| x86 / x86_64 | 小端 |
| ARM（通常） | 小端 |
| 网络协议 | 大端（网络字节序） |
| PowerPC（旧） | 大端 |

### 3.4 字节序转换

```c
#include <arpa/inet.h>   // Linux
// 或 <winsock2.h>       // Windows

uint32_t net_order = htonl(host_order);   // 主机序 → 网络序
uint32_t host_order = ntohl(net_order);   // 网络序 → 主机序
uint16_t s = htons(short_val);
uint16_t s = ntohs(short_val);

// 手动实现
uint32_t swap_endian(uint32_t val) {
    return ((val & 0xFF) << 24) |
           ((val & 0xFF00) << 8) |
           ((val & 0xFF0000) >> 8) |
           ((val & 0xFF000000) >> 24);
}
```

## 四、内存对齐

### 4.1 什么是内存对齐

数据在内存中的起始地址通常是某个值的倍数，称为**内存对齐**：

- `char`：任意地址（1 字节对齐）
- `short`：偶数地址（2 字节对齐）
- `int`：4 的倍数地址（4 字节对齐）
- `double`：8 的倍数地址（8 字节对齐）

### 4.2 对齐的原因

- **硬件要求**：某些 CPU 只能从对齐的地址读取数据
- **性能优化**：对齐的数据访问更快（减少总线事务）

### 4.3 结构体对齐规则

```c
struct Example {
    char a;     // 偏移 0，占 1 字节
                // 填充 3 字节
    int b;      // 偏移 4，占 4 字节
    char c;     // 偏移 8，占 1 字节
                // 填充 3 字节（对齐到 4 字节）
};
// sizeof = 12
```

对齐规则：
1. 每个成员的起始地址必须是该成员大小的倍数
2. 结构体的总大小必须是最大成员大小的倍数
3. 成员按声明顺序排列

### 4.4 对齐的计算

```c
#include <stddef.h>

struct Demo {
    char a;      // 偏移 0
    double b;    // 偏移 8（需要 8 字节对齐）
    int c;       // 偏移 16
    char d;      // 偏移 20
                 // 填充 4 字节（对齐到 8）
};  // sizeof = 24

// 验证偏移量
printf("a 偏移: %zu\n", offsetof(struct Demo, a));  // 0
printf("b 偏移: %zu\n", offsetof(struct Demo, b));  // 8
printf("c 偏移: %zu\n", offsetof(struct Demo, c));  // 16
printf("d 偏移: %zu\n", offsetof(struct Demo, d));  // 20
```

## 五、浮点数的 IEEE 754 表示

### 5.1 float（32 位）

```
符号位(1) + 指数位(8) + 尾数位(23)

例：-6.5
= -110.1(二进制)
= -1.101 × 2^2

符号位 = 1（负数）
指数 = 2 + 127 = 129 = 10000001
尾数 = 10100000000000000000000

存储：1 10000001 10100000000000000000000
```

### 5.2 特殊值

```
+0:  0 00000000 00000000000000000000000
-0:  1 00000000 00000000000000000000000
+Inf: 0 11111111 00000000000000000000000
NaN:  0 11111111 10000000000000000000000
```

## 六、查看内存的实际存储

```c
#include <stdio.h>

void print_bytes(void *ptr, size_t size) {
    unsigned char *p = (unsigned char *)ptr;
    for (size_t i = 0; i < size; i++) {
        printf("[%zu] = 0x%02X\n", i, p[i]);
    }
}

int main(void) {
    int x = 0x12345678;
    printf("int x = 0x12345678 的内存布局：\n");
    print_bytes(&x, sizeof(x));

    double d = 3.14;
    printf("\ndouble d = 3.14 的内存布局：\n");
    print_bytes(&d, sizeof(d));

    return 0;
}
```

## 七、要点总结

1. 计算机使用**补码**表示有符号整数，加减法统一为加法
2. **大端**：高位在低地址；**小端**：低位在低地址
3. x86/ARM 通常是小端，网络协议使用大端（网络字节序）
4. **内存对齐**影响结构体大小和访问性能
5. 结构体成员按声明顺序排列，遵循对齐规则
6. IEEE 754 标准定义了浮点数的编码方式
7. 使用 `offsetof` 宏查看结构体成员的偏移量
8. 理解数据存储方式有助于调试和底层开发
