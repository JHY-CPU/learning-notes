# 指针与动态内存

## 一、动态内存概述

C 语言中，变量的内存分配有三种方式：

| 方式 | 时机 | 生命周期 | 示例 |
|------|------|---------|------|
| 静态分配 | 编译时 | 整个程序运行期间 | 全局变量、static |
| 栈分配 | 运行时 | 函数调用期间 | 局部变量 |
| 堆分配 | 运行时 | 程序员控制 | malloc/free |

动态内存就是**堆分配**，由程序员手动管理。

## 二、malloc 函数

```c
#include <stdlib.h>

void* malloc(size_t size);
```

分配指定字节的内存块，返回指向该内存的 `void*` 指针。

```c
// 分配一个 int 大小的内存
int *p = (int*)malloc(sizeof(int));
if (p == NULL) {
    fprintf(stderr, "内存分配失败\n");
    return -1;
}
*p = 42;
printf("%d\n", *p);
free(p);
p = NULL;
```

### 分配数组

```c
int n = 10;
int *arr = (int*)malloc(n * sizeof(int));
if (arr == NULL) {
    fprintf(stderr, "内存分配失败\n");
    return -1;
}

// 使用数组
for (int i = 0; i < n; i++) {
    arr[i] = i * i;
}

free(arr);
arr = NULL;
```

## 三、calloc 函数

```c
void* calloc(size_t count, size_t size);
```

分配 `count` 个大小为 `size` 的内存块，并**初始化为 0**。

```c
// 分配 10 个 int，全部初始化为 0
int *arr = (int*)calloc(10, sizeof(int));

// 与 malloc 的区别
// malloc 分配的内存内容不确定
// calloc 分配的内存全部清零

// 等价写法
int *arr2 = (int*)malloc(10 * sizeof(int));
memset(arr2, 0, 10 * sizeof(int));
```

## 四、realloc 函数

```c
void* realloc(void* ptr, size_t new_size);
```

调整已分配内存块的大小。

```c
int *arr = (int*)malloc(5 * sizeof(int));
for (int i = 0; i < 5; i++) arr[i] = i + 1;

// 扩展到 10 个元素
int *new_arr = (int*)realloc(arr, 10 * sizeof(int));
if (new_arr == NULL) {
    free(arr);  // 原内存仍然有效
    return -1;
}
arr = new_arr;

// 新增的元素未初始化
for (int i = 5; i < 10; i++) arr[i] = 0;

// 也可以缩小
arr = (int*)realloc(arr, 3 * sizeof(int));  // 只保留前3个

free(arr);
```

### realloc 的行为

- `ptr == NULL`：等同于 malloc
- `new_size == 0`：等同于 free（实现定义）
- 可能原地扩展，也可能移动到新位置
- 返回 NULL 时，原内存块仍然有效

## 五、free 函数

```c
void free(void* ptr);
```

释放之前分配的动态内存。

```c
int *p = (int*)malloc(sizeof(int));
*p = 100;
// ... 使用 p ...
free(p);    // 释放内存
p = NULL;   // 置空，防止悬空指针
```

### free 的注意事项

```c
// 1. 只能释放 malloc/calloc/realloc 返回的指针
int x = 10;
// free(&x);  // 错误！不是动态分配的

// 2. 不能重复释放
int *p = malloc(sizeof(int));
free(p);
// free(p);  // 错误！重复释放

// 3. free(NULL) 是安全的（什么也不做）
int *q = NULL;
free(q);  // OK
```

## 六、动态内存的生命周期

```c
int main() {
    int *p = NULL;

    // 分配
    p = (int*)malloc(sizeof(int));
    if (p == NULL) return 1;

    // 使用
    *p = 42;

    // 释放
    free(p);
    p = NULL;

    // p 现在是 NULL，不是野指针
    return 0;
}
```

### 生命周期规则

```
malloc 成功 ──> 内存可用 ──> free ──> 内存归还系统
                                  │
                                  v
                            置 NULL ──> 安全
```

## 七、内存泄漏

### 什么是内存泄漏

分配了内存但没有释放，导致可用内存越来越少。

```c
void leak() {
    int *p = (int*)malloc(1000);
    // 忘记 free(p)
    // 函数返回后，p 被销毁，但 1000 字节仍然占用
}

// 多次调用会耗尽内存
for (int i = 0; i < 1000000; i++) {
    leak();
}
```

### 防止泄漏的策略

```c
// 1. 每个 malloc 对应一个 free
// 2. 释放后置 NULL
// 3. 在函数所有退出路径上释放
int process(int need_alloc) {
    int *p = NULL;
    if (need_alloc) {
        p = (int*)malloc(100);
        if (p == NULL) return -1;
    }

    // 处理逻辑...
    if (some_error) {
        free(p);  // 提前退出也要释放
        return -1;
    }

    free(p);
    return 0;
}
```

## 八、常见错误汇总

| 错误类型 | 描述 | 后果 |
|---------|------|------|
| 忘记检查 NULL | malloc 失败时 | 解引用空指针导致崩溃 |
| 内存泄漏 | 忘记 free | 内存耗尽 |
| 重复释放 | 对同一指针 free 两次 | 堆损坏 |
| 释放栈内存 | free(&local_var) | 未定义行为 |
| 越界访问 | 超出分配的大小 | 堆损坏 |

## 九、关键要点总结

> **四大函数**
> - `malloc(size)`：分配指定大小的内存
> - `calloc(count, size)`：分配并清零
> - `realloc(ptr, new_size)`：调整大小
> - `free(ptr)`：释放内存

> **核心原则**
> - 每次 malloc 都必须对应一次 free
> - 释放后立即置 NULL
> - 使用前检查返回值是否为 NULL
> - 避免内存泄漏和重复释放
