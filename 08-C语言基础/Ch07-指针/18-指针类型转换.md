# 指针类型转换

## 一、概述

指针类型转换（Type Casting）是指将一种指针类型转换为另一种指针类型。在 C 语言中，指针类型转换分为**隐式转换**和**显式（强制）转换**。

## 二、隐式转换

### void* 的自动转换

C 语言中，`void*` 可以与任何指针类型自动转换：

```c
int n = 42;
void *vp = &n;         // int* -> void*，自动
int *ip = vp;          // void* -> int*，自动
```

> 注意：C++ 中 `void*` 到其他类型的转换需要显式强制转换。

### 指针到 void* 的自动转换

```c
int *ip;
char *cp;
double *dp;

void *vp;
vp = ip;  // OK
vp = cp;  // OK
vp = dp;  // OK
```

## 三、显式（强制）转换

使用 `(目标类型*)` 语法进行强制转换：

```c
int n = 0x40490fdb;  // 3.14159 的 IEEE 754 表示
float *fp = (float*)&n;
printf("%f\n", *fp);  // 约 3.141593
```

### 常见场景

#### 1. malloc 返回值转换

```c
// C 中可以不转换，C++ 中必须转换
int *p = (int*)malloc(sizeof(int));
```

#### 2. 不同类型指针之间的转换

```c
double d = 3.14;
int *p = (int*)&d;  // 强制转换
// 但解引用 *p 是未定义行为
```

#### 3. 整数与指针之间的转换

```c
// 将整数转为指针（嵌入式常见）
volatile int *reg = (volatile int*)0x40021000;

// 将指针转为整数
int *p = &some_var;
uintptr_t addr = (uintptr_t)p;  // 使用 uintptr_t
```

## 四、对齐问题（Alignment）

### 什么是内存对齐

不同类型的变量有不同的对齐要求：

```c
// 典型对齐要求
// char: 1 字节对齐
// short: 2 字节对齐
// int: 4 字节对齐
// double: 8 字节对齐

printf("int 对齐: %zu\n", _Alignof(int));      // 4
printf("double 对齐: %zu\n", _Alignof(double)); // 8
```

### 对齐问题示例

```c
char buf[16];
// buf 的地址可能不满足 double 的对齐要求

// 危险：可能未对齐
double *dp = (double*)buf;
// *dp = 3.14;  // 可能崩溃或性能下降

// 安全：使用 memcpy
double value = 3.14;
memcpy(buf, &value, sizeof(double));
```

## 五、指针转换的类型安全

### 安全的转换

```c
// void* 与其他类型之间的转换
void *vp = malloc(100);
int *ip = (int*)vp;  // 安全（假设分配了足够空间）

// 向更宽类型的转换
int *ip2 = &some_int;
void *vp2 = ip2;  // 安全
```

### 不安全的转换

```c
// 大小不同的基本类型
double d = 3.14;
int *ip = (int*)&d;  // 转换本身合法，但解引用是 UB

// 指针与整数（可能丢失信息）
int *p = &some_var;
int addr = (int)p;  // 在 64 位系统上可能丢失高 32 位
// 应使用 uintptr_t
```

## 六、const 与 volatile 的转换

```c
// 去掉 const 是危险的
const int x = 10;
int *p = (int*)&x;     // 编译通过，但 *p = 20 是 UB

// 添加 const 是安全的
int y = 20;
const int *cp = &y;    // OK，隐式转换
```

## 七、函数指针的转换

函数指针与 `void*` 之间的转换在标准 C 中是**条件支持**的：

```c
void foo(void) { printf("Hello\n"); }

// 函数指针与 void* 之间的转换
void *vp = (void*)foo;             // 条件支持
typedef void (*FuncPtr)(void);
FuncPtr fp = (FuncPtr)vp;          // 条件支持

// 函数指针之间可以互相转换
typedef void (*FP1)(int);
typedef void (*FP2)(double);
FP1 f1 = some_func;
FP2 f2 = (FP2)f1;  // 转换合法，但调用结果是 UB
```

## 八、最佳实践

```c
// 1. 使用 void* 作为通用参数，接收时转换
void process(void *data, size_t type_size) {
    if (type_size == sizeof(int)) {
        int *ip = (int*)data;
        printf("%d\n", *ip);
    }
}

// 2. 使用 uintptr_t 进行指针-整数转换
uintptr_t addr = (uintptr_t)ptr;
int *p = (int*)addr;

// 3. 避免去掉 const
// int *p = (int*)const_ptr;  // 不推荐

// 4. 注意对齐要求
_Alignas(double) char buf[16];  // 保证对齐
double *dp = (double*)buf;      // 安全
```

## 九、关键要点总结

> **核心概念**
> - C 中 `void*` 可以与任何指针类型隐式转换
> - 强制转换使用 `(type*)` 语法
> - 类型转换不影响内存中的数据，只改变解释方式

> **注意事项**
> - 注意内存对齐要求
> - 不要随意去掉 const 限定
> - 指针与整数转换使用 `uintptr_t`
> - 转换合法不等于解引用合法
