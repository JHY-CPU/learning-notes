# 指针与数组的关系

## 一、数组名的退化

在大多数表达式中，**数组名会"退化"（decay）为指向其第一个元素的指针**。

```c
int arr[5] = {10, 20, 30, 40, 50};

// arr 退化为 &arr[0]，类型为 int*
int *p = arr;

printf("%d\n", *p);     // 10
printf("%d\n", *arr);   // 10
```

### 退化不发生的情况

```c
int arr[5] = {1, 2, 3, 4, 5};

// 情况一：sizeof
printf("%zu\n", sizeof(arr));      // 20（整个数组的大小）
printf("%zu\n", sizeof(&arr[0]));  // 8（指针大小）

// 情况二：取地址 &
int (*ptr)[5] = &arr;  // ptr 是指向整个数组的指针
printf("%zu\n", sizeof(*ptr));     // 20

// 情况三：字符串字面量初始化字符数组
char str[] = "hello";
// str 的类型仍然是 char[6]
```

## 二、`arr[i]` 与 `*(arr + i)` 的等价性

C 语言标准规定：`arr[i]` 严格等价于 `*(arr + i)`。

```c
int arr[5] = {10, 20, 30, 40, 50};

// 以下四种写法完全等价
printf("%d\n", arr[2]);
printf("%d\n", *(arr + 2));
printf("%d\n", 2[arr]);     // 合法但不推荐！
printf("%d\n", *(2 + arr));
```

### `[]` 的本质

```c
// 下标运算符 [] 的语义
a[b]  等价于  *(a + b)

// 所以 a[2] 和 2[a] 都是 *(a + 2)
// 因为加法满足交换律
```

## 三、指针遍历数组

### 方法一：下标法

```c
int arr[5] = {10, 20, 30, 40, 50};
int n = sizeof(arr) / sizeof(arr[0]);

for (int i = 0; i < n; i++) {
    printf("arr[%d] = %d\n", i, arr[i]);
}
```

### 方法二：指针移动法

```c
int arr[5] = {10, 20, 30, 40, 50};
int n = sizeof(arr) / sizeof(arr[0]);

for (int *p = arr; p < arr + n; p++) {
    printf("%d\n", *p);
}
```

### 方法三：指针偏移法

```c
int arr[5] = {10, 20, 30, 40, 50};
int n = sizeof(arr) / sizeof(arr[0]);
int *base = arr;

for (int i = 0; i < n; i++) {
    printf("%d\n", *(base + i));
}
```

## 四、指针与数组的区别

尽管数组名在很多情况下退化为指针，但它们有本质区别：

```c
int arr[5] = {1, 2, 3, 4, 5};
int *p = arr;

// 1. sizeof 行为不同
printf("sizeof(arr) = %zu\n", sizeof(arr));  // 20（5 * 4）
printf("sizeof(p)   = %zu\n", sizeof(p));     // 8（指针大小）

// 2. 可以对指针赋值，不能对数组名赋值
p = p + 1;     // 合法
// arr = arr + 1;  // 编译错误！

// 3. 取地址的含义
// &arr 的类型是 int(*)[5]（指向含5个int的数组的指针）
// &p   的类型是 int**
printf("&arr = %p\n", (void*)&arr);
printf("arr  = %p\n", (void*)arr);   // 值相同但类型不同
```

## 五、函数参数中的数组退化

数组作为函数参数时，**总是退化为指针**。

```c
// 以下三种声明完全等价
void func(int arr[10]);
void func(int arr[]);
void func(int *arr);

// 函数内 sizeof(arr) 得到的是指针大小，不是数组大小！
void func(int arr[10]) {
    printf("%zu\n", sizeof(arr));  // 输出 8（64位系统），不是 40
}
```

### 如何在函数中获取数组长度

```c
// 方法：额外传递长度参数
void print_array(int *arr, int n) {
    for (int i = 0; i < n; i++) {
        printf("%d ", arr[i]);
    }
    printf("\n");
}

int main() {
    int arr[5] = {1, 2, 3, 4, 5};
    print_array(arr, 5);
    return 0;
}
```

## 六、指针数组与数组指针的初步区分

```c
// 指针数组：数组中每个元素都是指针
int *ptr_arr[5];  // 5 个 int* 组成的数组

// 数组指针：指向一个数组的指针
int (*arr_ptr)[5];  // 指向含 5 个 int 的数组的指针
```

## 七、通过指针修改数组元素

```c
int arr[5] = {1, 2, 3, 4, 5};
int *p = arr;

// 通过指针修改
*p = 100;        // arr[0] = 100
*(p + 2) = 300;  // arr[2] = 300

// 验证
for (int i = 0; i < 5; i++) {
    printf("%d ", arr[i]);  // 100 2 300 4 5
}
```

## 八、关键要点总结

> **核心关系**
> - 数组名在表达式中退化为指向首元素的指针
> - `arr[i]` 严格等价于 `*(arr + i)`
> - 函数参数中的数组总是退化为指针

> **重要区别**
> - sizeof(arr) 不等于 sizeof(指针)
> - 数组名是常量，不能赋值
> - &arr 类型是数组指针，不是指针的指针

> **最佳实践**
> - 函数中传递数组时，额外传递长度
> - 遍历数组时，指针方式通常比下标方式稍快
