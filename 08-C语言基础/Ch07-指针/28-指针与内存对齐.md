# 指针与内存对齐

## 一、什么是内存对齐

内存对齐（Memory Alignment）是指数据在内存中的起始地址必须是某个值的整数倍。这是由**硬件架构**决定的要求。

```c
// 典型的对齐要求
char:     1 字节对齐（任意地址）
short:    2 字节对齐（地址为 2 的倍数）
int:      4 字节对齐（地址为 4 的倍数）
double:   8 字节对齐（地址为 8 的倍数）
```

## 二、为什么需要对齐

### 性能原因

```
未对齐访问：
地址: 0x01 0x02 0x03 0x04 0x05 0x06 0x07 0x08
数据: [xx] [A ] [B ] [C ] [D ] [xx] [xx] [xx]
      读取 int 需要跨越两个内存字，需要两次内存访问

对齐访问：
地址: 0x00 0x01 0x02 0x03 0x04 0x05 0x06 0x07
数据: [A ] [B ] [C ] [D ] [xx] [xx] [xx] [xx]
      读取 int 只需要一次内存访问
```

### 硬件原因

- 某些 CPU（如 ARM、SPARC）**严格要求**对齐，未对齐访问会触发硬件异常
- x86/x64 允许未对齐访问，但性能下降

## 三、查看类型的对齐要求

```c
#include <stdio.h>
#include <stdalign.h>  // C11

int main() {
    printf("char 对齐: %zu\n", alignof(char));       // 1
    printf("short 对齐: %zu\n", alignof(short));     // 2
    printf("int 对齐: %zu\n", alignof(int));         // 4
    printf("long 对齐: %zu\n", alignof(long));       // 8 (64位)
    printf("float 对齐: %zu\n", alignof(float));     // 4
    printf("double 对齐: %zu\n", alignof(double));   // 8
    printf("void* 对齐: %zu\n", alignof(void*));     // 8 (64位)

    // C11 之前的写法
    printf("int 对齐: %zu\n", _Alignof(int));

    return 0;
}
```

## 四、结构体的内存对齐

```c
struct Example {
    char  a;   // 1 字节 + 3 字节填充
    int   b;   // 4 字节
    char  c;   // 1 字节 + 7 字节填充
    double d;  // 8 字节
};

printf("sizeof(struct Example) = %zu\n", sizeof(struct Example));
// 可能输出 24（而不是 1+4+1+8=14）
```

```
内存布局（典型 64 位系统）：
[a][pad][pad][pad][b  b  b  b][c][pad][pad][pad][pad][pad][pad][pad][d  d  d  d  d  d  d  d]
|------ 4 ------|------ 4 ------|------ 8 ------|------ 8 ------|
```

### 优化结构体布局

```c
// 浪费空间的布局
struct Bad {
    char  a;   // 1 + 7 填充
    double d;  // 8
    char  b;   // 1 + 3 填充
    int   c;   // 4
};  // 总计 24 字节

// 优化布局（按大小降序排列）
struct Good {
    double d;  // 8
    int   c;   // 4
    char  a;   // 1
    char  b;   // 1 + 2 填充
};  // 总计 16 字节
```

## 五、指针类型转换的对齐问题

```c
char buf[16];

// 危险：buf 可能不满足 double 的对齐要求
double *dp = (double*)buf;
// *dp = 3.14;  // 可能在某些架构上崩溃

// 解决方案一：使用 memcpy
double value = 3.14;
memcpy(buf, &value, sizeof(double));

// 解决方案二：使用 _Alignas 保证对齐
_Alignas(double) char buf2[16];
double *dp2 = (double*)buf2;
*dp2 = 3.14;  // 安全
```

## 六、aligned_alloc（C11）

C11 引入了 `aligned_alloc`，可以分配指定对齐方式的内存。

```c
#include <stdlib.h>

// 分配 256 字节，按 64 字节对齐
void *p = aligned_alloc(64, 256);
if (p != NULL) {
    printf("对齐地址: %p\n", p);
    printf("是否 64 字节对齐: %s\n",
           ((uintptr_t)p % 64 == 0) ? "是" : "否");
    free(p);
}
```

### 注意事项

- 对齐值必须是 2 的幂
- 大小必须是对齐值的整数倍
- 必须用 `free` 释放

## 七、posix_memalign（POSIX）

```c
#include <stdlib.h>

void *p;
// 对齐值必须是 sizeof(void*) 的倍数，且是 2 的幂
int ret = posix_memalign(&p, 64, 256);
if (ret == 0) {
    printf("分配成功: %p\n", p);
    free(p);
}
```

## 八、编译器属性指定对齐

```c
// GCC / Clang
struct __attribute__((aligned(16))) AlignedStruct {
    int data[4];
};

// C11 标准写法
struct _Alignas(16) AlignedStruct2 {
    int data[4];
};

// 变量对齐
_Alignas(32) int aligned_var;
```

## 九、SIMD 与对齐

SIMD（单指令多数据）指令通常要求数据严格对齐：

```c
#include <immintrin.h>

// AVX 要求 32 字节对齐
_Alignas(32) float a[8];
_Alignas(32) float b[8];
_Alignas(32) float result[8];

__m256 va = _mm256_load_ps(a);   // 需要对齐
__m256 vb = _mm256_load_ps(b);
__m256 vr = _mm256_add_ps(va, vb);
_mm256_store_ps(result, vr);

// 如果不对齐，使用 loadu/storeu（未对齐版本）
__m256 vu = _mm256_loadu_ps(unaligned_ptr);  // 性能稍低
```

## 十、判断指针是否对齐

```c
int is_aligned(const void *ptr, size_t alignment) {
    return ((uintptr_t)ptr % alignment) == 0;
}

// 使用
double *p = (double*)some_address;
if (is_aligned(p, alignof(double))) {
    // 可以安全使用
} else {
    // 需要用 memcpy
}
```

## 十一、关键要点总结

> **核心概念**
> - 对齐是硬件要求，不遵守可能导致崩溃或性能下降
> - 不同类型的对齐要求不同
> - 结构体有填充字节来满足对齐

> **实践建议**
> - 指针类型转换时注意对齐要求
> - 按大小降序排列结构体成员
> - SIMD 操作必须保证对齐
> - 使用 `aligned_alloc` 或 `_Alignas` 保证对齐

> **跨平台注意**
> - ARM 严格要求对齐
> - x86 允许未对齐但性能差
> - 使用 `alignof` 查询当前平台的要求
