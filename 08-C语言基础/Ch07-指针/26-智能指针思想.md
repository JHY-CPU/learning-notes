# 智能指针思想

## 一、C 语言没有垃圾回收

与 Java、Python 等语言不同，C 语言**没有自动垃圾回收（GC）机制**。所有动态分配的内存都必须由程序员手动管理：

```c
int *p = malloc(sizeof(int));  // 分配
// 使用 p...
free(p);                        // 释放
p = NULL;                       // 置空
```

忘记释放会导致**内存泄漏**，重复释放会导致**未定义行为**。

## 二、RAII 思想模拟

RAII（Resource Acquisition Is Initialization）是 C++ 中的核心思想：**在构造函数中获取资源，在析构函数中释放资源**。

C 语言没有析构函数，但可以模拟类似的思想：

```c
// 使用作用域模拟 RAII
typedef struct {
    void *data;
    void (*cleanup)(void*);
} ScopedResource;

ScopedResource make_resource(void *data, void (*cleanup)(void*)) {
    return (ScopedResource){data, cleanup};
}

void release_resource(ScopedResource *res) {
    if (res->cleanup && res->data) {
        res->cleanup(res->data);
        res->data = NULL;
    }
}

// 使用示例
void process() {
    ScopedResource res = make_resource(malloc(1024), free);

    if (res.data == NULL) return;

    // 使用资源...

    release_resource(&res);  // 保证释放
}
```

## 三、引用计数手动实现

引用计数是智能指针的核心机制之一。每个对象维护一个计数器，记录有多少指针指向它。当计数为 0 时自动释放。

### 引用计数结构

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct {
    void *data;
    int ref_count;
} RefCounted;

RefCounted* rc_create(size_t size) {
    RefCounted *rc = (RefCounted*)malloc(sizeof(RefCounted));
    if (rc == NULL) return NULL;

    rc->data = malloc(size);
    if (rc->data == NULL) {
        free(rc);
        return NULL;
    }

    rc->ref_count = 1;
    return rc;
}

void rc_retain(RefCounted *rc) {
    if (rc != NULL) {
        rc->ref_count++;
        printf("引用计数增加: %d\n", rc->ref_count);
    }
}

void rc_release(RefCounted *rc) {
    if (rc == NULL) return;

    rc->ref_count--;
    printf("引用计数减少: %d\n", rc->ref_count);

    if (rc->ref_count <= 0) {
        printf("释放对象\n");
        free(rc->data);
        free(rc);
    }
}

void* rc_get(RefCounted *rc) {
    return rc ? rc->data : NULL;
}
```

### 使用示例

```c
typedef struct {
    char name[50];
    int age;
} Person;

int main() {
    RefCounted *rc = rc_create(sizeof(Person));
    Person *p = (Person*)rc_get(rc);

    snprintf(p->name, 50, "Alice");
    p->age = 25;

    // 模拟多个指针共享
    rc_retain(rc);  // ref = 2
    rc_retain(rc);  // ref = 3

    printf("Person: %s, %d\n", p->name, p->age);

    rc_release(rc);  // ref = 2
    rc_release(rc);  // ref = 1
    rc_release(rc);  // ref = 0，释放

    return 0;
}
```

## 四、智能指针包装

### UniquePtr 模拟（独占所有权）

```c
typedef struct {
    void *data;
    void (*deleter)(void*);
} UniquePtr;

UniquePtr unique_create(void *data, void (*deleter)(void*)) {
    return (UniquePtr){data, deleter ? deleter : free};
}

void unique_release(UniquePtr *ptr) {
    if (ptr && ptr->data) {
        ptr->deleter(ptr->data);
        ptr->data = NULL;
    }
}

// 移动语义：转移所有权
UniquePtr unique_move(UniquePtr *src) {
    UniquePtr moved = *src;
    src->data = NULL;
    src->deleter = NULL;
    return moved;
}
```

### 使用示例

```c
void process() {
    UniquePtr p1 = unique_create(malloc(100), free);

    // 转移所有权
    UniquePtr p2 = unique_move(&p1);

    // p1 现在为空，p2 拥有资源
    if (p1.data == NULL) {
        printf("p1 已转移\n");
    }

    unique_release(&p2);
}
```

## 五、Scope Guard 模式

确保在作用域退出时执行清理操作：

```c
typedef void (*CleanupFunc)(void*);

typedef struct {
    void *resource;
    CleanupFunc cleanup;
    int dismissed;
} ScopeGuard;

ScopeGuard guard_create(void *resource, CleanupFunc cleanup) {
    return (ScopeGuard){resource, cleanup, 0};
}

void guard_dismiss(ScopeGuard *guard) {
    guard->dismissed = 1;
}

void guard_release(ScopeGuard *guard) {
    if (!guard->dismissed && guard->resource) {
        guard->cleanup(guard->resource);
    }
}
```

```c
void safe_process() {
    FILE *fp = fopen("data.txt", "r");
    ScopeGuard file_guard = guard_create(fp, (CleanupFunc)fclose);

    int *buffer = malloc(1024);
    ScopeGuard buf_guard = guard_create(buffer, free);

    if (fp == NULL || buffer == NULL) {
        // guard_release 会自动清理已分配的资源
    }

    // 使用资源...

    // 正常退出时释放
    guard_dismiss(&file_guard);  // 如果需要保留文件
    guard_release(&buf_guard);
    guard_release(&file_guard);
}
```

## 六、实际项目中的策略

### 约定所有权

```c
// 约定：create 函数返回的指针由调用者负责释放
// 约定：以 _borrow 结尾的函数不转移所有权

Object* object_create(void);      // 调用者负责 free
Object* object_borrow(Object *o);  // 不增加引用，调用者不释放
void object_release(Object *o);    // 减少引用或释放
```

### 文档约定

```c
/**
 * 创建新对象
 * @return 新分配的对象，调用者负责调用 object_free 释放
 */
Object* object_create(void);

/**
 * 获取子对象
 * @return 指向内部数据的指针，不要释放，生命周期跟随父对象
 */
const Child* object_get_child(const Object *obj);
```

## 七、关键要点总结

> **核心问题**
> - C 没有 GC，所有动态内存必须手动管理
> - 内存泄漏和悬空指针是最常见的错误

> **解决方案**
> - 引用计数：自动管理共享资源
> - 所有权约定：明确谁负责释放
> - Scope Guard：确保作用域退出时清理

> **编程规范**
> - create/free 成对使用
> - 文档明确所有权归属
> - 尽量减少动态内存的传递和共享
