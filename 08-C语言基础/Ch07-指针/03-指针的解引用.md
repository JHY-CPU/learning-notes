# 指针的解引用

## 一、什么是解引用

解引用（Dereference）是指通过指针访问它所指向的内存地址中的值。使用 `*` 运算符来实现解引用操作。

```c
int x = 42;
int *p = &x;    // p 指向 x
printf("%d\n", *p);  // *p 解引用，输出 42
```

解引用就是"打开指针这扇门，看看门后有什么"。

## 二、通过解引用读取值

```c
#include <stdio.h>

int main() {
    int num = 255;
    int *ptr = &num;

    // 通过解引用读取值
    printf("num 的值: %d\n", num);
    printf("ptr 指向的值: %d\n", *ptr);
    // 两者输出相同

    return 0;
}
```

## 三、通过解引用修改值

解引用不仅可以读取值，还可以**直接修改**指针指向的内存中的值。

```c
#include <stdio.h>

int main() {
    int num = 100;
    int *ptr = &num;

    printf("修改前: num = %d\n", num);

    *ptr = 200;   // 通过解引用修改值

    printf("修改后: num = %d\n", num);  // 输出 200

    return 0;
}
```

### 解引用修改的等价关系

```c
int x = 0;
int *p = &x;

*p = 5;     // 等价于 x = 5;
*p += 3;    // 等价于 x += 3;
(*p)++;     // 等价于 x++;
```

## 四、解引用的安全注意事项

### 1. 不要解引用空指针

```c
int *p = NULL;
// *p = 10;  // 严重错误！会导致程序崩溃（段错误）
```

### 2. 不要解引用野指针

```c
int *p;       // 未初始化，是野指针
// *p = 10;   // 未定义行为，可能崩溃或写入未知内存
```

### 3. 不要解引用已释放的指针

```c
int *p = (int*)malloc(sizeof(int));
free(p);
// *p = 10;   // 悬空指针，未定义行为
```

### 安全解引用的模式

```c
// 模式：解引用前检查
void safe_deref(int *p) {
    if (p == NULL) {
        fprintf(stderr, "指针为空，无法解引用\n");
        return;
    }
    printf("值为: %d\n", *p);
}
```

## 五、多重解引用

指针可以指向指针，这时需要多次解引用。

```c
int x = 42;
int *p = &x;    // 一级指针
int **pp = &p;  // 二级指针

printf("x = %d\n", x);       // 42
printf("*p = %d\n", *p);     // 42
printf("**pp = %d\n", **pp); // 42

**pp = 100;
printf("x = %d\n", x);       // 100
```

## 六、解引用与指针类型

解引用时的类型由指针声明的类型决定：

```c
int n = 0x12345678;
int *ip = &n;
char *cp = (char*)&n;

printf("*ip = 0x%x\n", *ip);   // 读取 4 字节：0x12345678
printf("*cp = 0x%x\n", *cp);   // 读取 1 字节：0x78（小端序）
```

指针类型决定了：
- 每次解引用读取多少字节
- 如何解释这些字节的含义

## 七、解引用与运算符优先级

```c
int x = 10, y = 20;
int *p = &x;
int *q = &y;

// 后缀自增优先级高于解引用
(*p)++;     // 先解引用，再自增 x 的值。x 变为 11
// *p++;    // 等价于 *(p++)，先解引用当前 p，再移动 p。常用在遍历数组

// 赋值
*p = *q;    // 将 q 指向的值赋给 p 指向的内存
printf("x = %d\n", x);  // 20
```

## 八、关键要点总结

> **核心概念**
> - `*指针变量` 表示解引用，访问指针指向的内存
> - 解引用可以读取也可以写入
> - 解引用的类型由指针声明类型决定

> **安全原则**
> - 解引用前必须确保指针有效
> - NULL、野指针、悬空指针都不能解引用
> - 始终养成检查习惯
