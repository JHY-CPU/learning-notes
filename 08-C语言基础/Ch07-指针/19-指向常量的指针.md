# 指向常量的指针

## 一、基本概念

指向常量的指针（Pointer to Constant）是指**指针指向的数据被视为常量**，不能通过该指针修改数据。

```c
int x = 10;
const int *p = &x;  // p 指向 const int

// *p = 20;   // 错误！不能通过 p 修改
x = 20;        // 正确！通过原变量修改是可以的
printf("%d\n", *p);  // 20
```

### 重要理解

`const int *p` 的含义是："**承诺不通过 p 修改它指向的数据**"，而非"p 指向的数据是不可变的"。

## 二、等价写法

```c
const int *p1 = &x;
int const *p2 = &x;  // 完全等价
```

`const` 在 `*` 的左边，修饰的是**指向的数据类型**。

## 三、const 指针的赋值规则

### 安全的赋值

```c
int x = 10;
const int *p = &x;   // OK：普通变量赋给 const 指针

const int y = 20;
const int *q = &y;   // OK：const 赋给 const 指针
```

### 不安全的赋值（编译错误）

```c
const int y = 20;
int *p = &y;  // 错误！可能通过 p 修改 y
```

这条规则保证了 **const 的承诺不被绕过**。

## 四、在函数参数中的应用

### 保护输入数据

```c
// 这个函数不需要也不应该修改传入的数组
void print_array(const int *arr, int n) {
    for (int i = 0; i < n; i++) {
        printf("%d ", arr[i]);
        // arr[i] = 0;  // 编译错误
    }
    printf("\n");
}
```

### 字符串处理函数

```c
size_t my_strlen(const char *s) {
    const char *p = s;
    while (*p) p++;
    return p - s;
}

int my_strcmp(const char *s1, const char *s2) {
    while (*s1 && (*s1 == *s2)) {
        s1++;
        s2++;
    }
    return *(unsigned char*)s1 - *(unsigned char*)s2;
}
```

### const 的好处

1. **自文档化**：声明了函数不会修改数据的意图
2. **编译器检查**：防止意外修改
3. **更灵活**：可以接受 const 和非 const 两种参数
4. **优化**：编译器知道数据不变，可能生成更高效的代码

```c
void func(const int *p);

int a = 10;
const int b = 20;

func(&a);  // OK：普通变量可以传给 const 参数
func(&b);  // OK：const 变量也可以传给 const 参数
```

## 五、常量传播优化

编译器可能对 const 变量进行优化：

```c
const int x = 42;
// 编译器可能在所有使用 x 的地方直接替换为 42
// 类似于 #define，但有类型检查
```

```c
void example() {
    const int size = 100;
    int arr[size];  // 编译器可能用 100 替换 size
    // ...
}
```

### 只读内存

```c
const int x = 42;  // 可能存储在只读数据段
// 如果尝试通过强制转换修改，可能触发段错误
```

## 六、与常量指针的区别

```c
int x = 10, y = 20;

// 指向常量的指针：数据不可通过此指针修改
const int *p1 = &x;
// *p1 = 30;  // 错误
p1 = &y;      // OK

// 常量指针：指针本身不可修改
int *const p2 = &x;
*p2 = 30;     // OK
// p2 = &y;   // 错误
```

## 七、多级 const

```c
const int x = 10;
const int *p = &x;
const int **pp = &p;  // 指向 const int* 的指针

printf("%d\n", **pp);  // 10
// **pp = 20;          // 错误
```

### 赋值兼容性

```c
int x = 10;
int *p = &x;
const int *cp = p;  // OK：添加 const 是安全的

// 反过来不行
const int cx = 10;
const int *cpx = &cx;
// int *px = cpx;  // 错误：去掉 const 不安全
```

## 八、常见误区

### 误区一：const 意味着不可变

```c
int x = 10;
const int *p = &x;

x = 20;          // OK！通过原变量修改
printf("%d\n", *p);  // 20，数据确实变了
```

### 误区二：const 参数的值不会变

```c
void foo(const int *p, int *q) {
    // 虽然 p 是 const，但可能指向与 q 相同的内存
    *q = 100;  // 可能间接修改 *p
    printf("%d\n", *p);  // 可能输出 100
}
```

## 九、关键要点总结

> **核心概念**
> - `const int *p` 承诺不通过 p 修改数据
> - `const` 在 `*` 左边修饰数据
> - 数据本身的可变性取决于原始定义

> **编程规范**
> - 函数的输入参数使用 const 指针
> - 字符串参数使用 `const char*`
> - 遵循 const 正确性（const-correctness）

> **赋值规则**
> - 普通变量可赋给 const 指针
> - const 变量不可赋给非 const 指针
