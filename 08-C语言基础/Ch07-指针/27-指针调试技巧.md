# 指针调试技巧

## 一、常见指针问题的调试

调试指针问题是 C 语言编程中最具挑战性的任务之一。本节介绍常用的调试方法和工具。

## 二、使用 GDB 调试指针

### 启动调试

```bash
# 编译时加上调试信息
gcc -g -o program program.c

# 启动 GDB
gdb ./program
```

### 打印指针值

```gdb
(gdb) print p              # 打印指针的值（地址）
(gdb) print *p             # 打印指针指向的值
(gdb) print &x             # 打印变量 x 的地址
(gdb) print/x p            # 以十六进制打印指针
```

### 查看内存

```gdb
(gdb) x/10x p              # 从 p 开始查看 10 个十六进制字
(gdb) x/10d p              # 从 p 开始查看 10 个十进制整数
(gdb) x/20c p              # 从 p 开始查看 20 个字符
(gdb) x/s p                # 从 p 开始查看字符串
(gdb) x/10i p              # 从 p 开始查看 10 条指令
```

### 断点与监视

```gdb
(gdb) break main           # 在 main 函数设置断点
(gdb) watch *p             # 监视 *p 的值变化
(gdb) watch p              # 监视指针 p 的值变化
(gdb) rwatch *p            # 监视 *p 的读取
(gdb) awatch *p            # 监视 *p 的读写
```

### 检查指针链

```gdb
# 假设有 Node *head 链表
(gdb) print *head
(gdb) print *head->next
(gdb) print *head->next->next

# 打印整个链表
(gdb) set $n = head
(gdb) while $n
 > print *$n
 > set $n = $n->next
 > end
```

### 调试 segfault

```gdb
(gdb) run                    # 运行程序
# 程序崩溃后
(gdb) backtrace              # 显示调用栈
(gdb) frame 2                # 切换到第 2 帧
(gdb) print p                # 棇查指针
(gdb) info locals            # 显示所有局部变量
```

## 三、使用 Valgrind 检测内存问题

### 安装与使用

```bash
# 安装
sudo apt install valgrind

# 运行检测
valgrind --leak-check=full ./program
```

### 检测内容

Valgrind 可以检测：
- 使用未初始化的内存
- 读写已释放的内存
- 内存泄漏
- 越界访问
- 不匹配的 malloc/free

### 输出解读

```
==12345== Invalid read of size 4
==12345==    at 0x4005B6: main (test.c:10)
==12345==  Address 0x0 is not stack'd, malloc'd or free'd
```

含义：在 test.c 第 10 行读取了无效地址 0x0。

### 示例：检测内存泄漏

```c
// leak.c - 有内存泄漏的代码
#include <stdlib.h>

int main() {
    int *p = malloc(100);
    // 忘记 free(p)
    return 0;
}
```

```bash
$ valgrind --leak-check=full ./leak
==12345== 100 bytes in 1 blocks are definitely lost
==12345==    at 0x4C2AB80: malloc (in /usr/lib/valgrind/...)
==12345==    by 0x40054E: main (leak.c:5)
```

## 四、使用 AddressSanitizer（ASan）

### 编译启用

```bash
gcc -fsanitize=address -g -o program program.c
```

### ASan 检测能力

- 堆、栈、全局变量的越界访问
- use-after-free（释放后使用）
- use-after-return（返回后使用局部变量）
- double-free（重复释放）
- 内存泄漏

### 示例输出

```
==12345==ERROR: AddressSanitizer: heap-buffer-overflow
READ of size 4 at 0x60200000ef54
    #0 0x4005b6 in main test.c:8
```

## 五、在代码中添加调试断言

### assert 宏

```c
#include <assert.h>

void process(int *p) {
    assert(p != NULL);   // 断言指针不为空
    assert(*p >= 0);     // 断言值非负
    // ...
}
```

### 自定义断言

```c
#ifdef DEBUG
#define ASSERT_PTR(p) do { \
    if ((p) == NULL) { \
        fprintf(stderr, "NULL pointer at %s:%d\n", __FILE__, __LINE__); \
        abort(); \
    } \
} while(0)
#else
#define ASSERT_PTR(p) ((void)0)
#endif
```

## 六、调试技巧

### 1. 打印地址和值

```c
void debug_ptr(const char *name, void *p) {
    printf("[DEBUG] %s = %p\n", name, p);
}

void debug_int_ptr(const char *name, int *p) {
    if (p) {
        printf("[DEBUG] %s = %p, *%s = %d\n", name, (void*)p, name, *p);
    } else {
        printf("[DEBUG] %s = NULL\n", name);
    }
}
```

### 2. 内存标记模式

```c
#define FREED_MARKER 0xDEADBEEF

void debug_free(void **pp) {
    if (pp && *pp) {
        memset(*pp, 0xBE, 100);  // 填充特殊值
        free(*pp);
        *pp = NULL;
    }
}
```

### 3. 分配日志

```c
static int alloc_count = 0;

void* debug_malloc(size_t size, const char *file, int line) {
    void *p = malloc(size);
    printf("[ALLOC] %p size=%zu at %s:%d (total: %d)\n",
           p, size, file, line, ++alloc_count);
    return p;
}

void debug_free2(void *p, const char *file, int line) {
    printf("[FREE]  %p at %s:%d (remaining: %d)\n",
           p, file, line, --alloc_count);
    free(p);
}

#define MALLOC(s) debug_malloc(s, __FILE__, __LINE__)
#define FREE(p) debug_free2(p, __FILE__, __LINE__)
```

## 七、预防性编程

```c
// 释放后置空
#define SAFE_FREE(p) do { free(p); (p) = NULL; } while(0)

// 分配并检查
int *safe_malloc_int(size_t count) {
    int *p = malloc(count * sizeof(int));
    if (p == NULL) {
        fprintf(stderr, "内存分配失败: %zu 个 int\n", count);
        abort();
    }
    return p;
}
```

## 八、关键要点总结

> **调试工具**
> - GDB：断点、查看内存、监视变量
> - Valgrind：内存泄漏和非法访问检测
> - AddressSanitizer：编译时启用，实时检测

> **调试技巧**
> - 编译时加 `-g` 保留调试信息
> - 使用 `assert` 检查前置条件
> - 打印指针地址帮助定位问题
> - 释放后置空，便于检测悬空指针

> **最佳实践**
> - 开发阶段启用 ASan
> - 定期用 Valgrind 检测
> - 使用 SAFE_FREE 宏
