# 指针的常见错误

## 一、未初始化的指针

使用未初始化的指针是最常见也最危险的错误。

```c
// 错误示例
int *p;        // p 包含随机值（垃圾值）
*p = 42;       // 未定义行为：向随机地址写入

// 正确做法
int *p = NULL; // 初始化为空指针
if (p == NULL) {
    p = (int*)malloc(sizeof(int));
}
```

## 二、野指针

野指针是指**指向不确定地址**的指针。

### 来源一：未初始化

```c
int *p;       // 野指针
// *p = 10;   // 危险！
```

### 来源二：释放后未置空

```c
int *p = (int*)malloc(sizeof(int));
free(p);
// p 仍然是野指针，保存旧地址
// *p = 10;   // 未定义行为！

// 正确
free(p);
p = NULL;
```

### 来源三：超出作用域

```c
int* bad_func() {
    int local = 42;
    return &local;  // local 将被销毁
}

int main() {
    int *p = bad_func();
    printf("%d\n", *p);  // 未定义行为
    return 0;
}
```

## 三、悬空指针

悬空指针是指**曾经有效但已失效**的指针。

```c
// 场景一：free 后继续使用
int *p = malloc(sizeof(int));
*p = 100;
free(p);
// *p 现在是悬空指针

// 场景二：返回局部数组地址
char* get_string() {
    char buf[20] = "hello";
    return buf;  // buf 已销毁
}

// 场景三：realloc 失败时的旧指针
int *new_p = realloc(p, new_size);
if (new_p == NULL) {
    // p 仍然有效，但不要继续用 p 也能避免混乱
    free(p);
}
```

## 四、内存泄漏

分配了内存但忘记释放。

```c
// 泄漏示例
void process() {
    char *buf = (char*)malloc(1024);
    if (some_condition) {
        return;  // 泄漏！buf 没有被 free
    }
    free(buf);
}

// 修正
void process_fixed() {
    char *buf = (char*)malloc(1024);
    if (some_condition) {
        free(buf);
        return;
    }
    free(buf);
}
```

### 隐蔽的泄漏

```c
// 泄漏：重新赋值前没有释放旧指针
char *p = (char*)malloc(100);
p = (char*)malloc(200);  // 第一次分配的 100 字节泄漏了！

// 正确
char *p = (char*)malloc(100);
free(p);
p = (char*)malloc(200);
```

## 五、越界访问

```c
int arr[5] = {1, 2, 3, 4, 5};
int *p = arr;

// 越界读写
p[5] = 10;   // 越界！arr 只有 5 个元素 (0-4)
p[-1] = 20;  // 越界！负下标

// 动态数组越界
int *q = (int*)malloc(5 * sizeof(int));
q[5] = 100;  // 越界！
free(q);
```

## 六、重复释放

```c
int *p = (int*)malloc(sizeof(int));
free(p);
free(p);  // 重复释放！导致未定义行为

// 防止方法：释放后置空
free(p);
p = NULL;
free(p);  // free(NULL) 是安全的
```

## 七、解引用空指针

```c
int *p = NULL;
// *p = 10;   // 段错误（Segmentation Fault）

// 防止方法：使用前检查
if (p != NULL) {
    *p = 10;
}
```

## 八、类型不匹配的指针

```c
int n = 42;
float *fp = (float*)&n;  // 强制转换但类型不匹配
printf("%f\n", *fp);     // 未定义行为：按浮点数解释整数的位模式
```

## 九、指针运算越界

```c
int arr[5] = {1, 2, 3, 4, 5};
int *p = arr;

// 允许 p 指向 arr[0] 到 arr[4]
// 允许 p 等于 arr + 5（指向末尾之后，但不可解引用）
// 不允许 p 超过 arr + 5
// 不允许 p 小于 arr

p = p + 10;  // 越界！未定义行为
```

## 十、错误清单与预防

| 错误 | 症状 | 预防措施 |
|------|------|---------|
| 未初始化指针 | 崩溃、数据损坏 | 定义时初始化为 NULL |
| 野指针 | 随机崩溃 | 释放后置空 |
| 悬空指针 | 使用已释放内存 | 释放后置空 |
| 内存泄漏 | 内存耗尽 | 配对 malloc/free |
| 越界访问 | 堆栈损坏 | 检查边界 |
| 重复释放 | 堆损坏 | 释放后置空 |
| 空指针解引用 | 立即崩溃 | 使用前检查 NULL |
| 返回局部地址 | 随机值 | 使用 malloc 或 static |

## 十一、关键要点总结

> **黄金法则**
> 1. 定义时初始化：`int *p = NULL;`
> 2. 释放后置空：`free(p); p = NULL;`
> 3. 使用前检查：`if (p != NULL)`
> 4. 配对管理：每个 malloc 对应一个 free
> 5. 不返回局部地址

> **调试工具**
> - Valgrind：检测内存泄漏和非法访问
> - AddressSanitizer (ASan)：编译时启用 `-fsanitize=address`
> - 静态分析工具：cppcheck、clang-tidy
