# 指针与信号处理

## 一、信号概述

信号（Signal）是操作系统向进程发送的异步通知。C 语言通过 `<signal.h>` 提供信号处理机制，核心是**函数指针**。

```c
#include <signal.h>

// 常见信号
SIGINT   // 用户按下 Ctrl+C
SIGTERM  // 终止请求
SIGSEGV  // 段错误（空指针解引用等）
SIGFPE   // 浮点异常（除零等）
SIGABRT  // abort() 调用
SIGALRM  // alarm 定时器到期
```

## 二、signal 函数

```c
void (*signal(int sig, void (*handler)(int)))(int);
```

`signal` 函数接收：
- `sig`：信号编号
- `handler`：信号处理函数指针

返回：之前安装的处理函数指针

### 使用 typedef 简化

```c
typedef void (*SignalHandler)(int);

SignalHandler signal(int sig, SignalHandler handler);
```

## 三、基本信号处理

```c
#include <stdio.h>
#include <signal.h>
#include <stdlib.h>

void handle_sigint(int sig) {
    printf("\n收到信号 %d (SIGINT)，准备退出...\n", sig);
    exit(0);
}

int main() {
    // 安装信号处理函数
    signal(SIGINT, handle_sigint);

    printf("按 Ctrl+C 测试信号处理...\n");
    while (1) {
        // 主循环
    }

    return 0;
}
```

## 四、信号处理函数中的指针

### volatile 变量

在信号处理函数中访问的全局变量必须声明为 `volatile`：

```c
#include <stdio.h>
#include <signal.h>
#include <unistd.h>

// volatile 告诉编译器不要优化这个变量
volatile sig_atomic_t flag = 0;

void handle_signal(int sig) {
    flag = 1;  // 在信号处理函数中设置标志
}

int main() {
    signal(SIGINT, handle_signal);

    printf("运行中，按 Ctrl+C 退出...\n");
    while (!flag) {
        // 编译器不知道 flag 可能在信号处理中被修改
        // 没有 volatile，编译器可能将 while(!flag) 优化为死循环
    }

    printf("已退出循环\n");
    return 0;
}
```

### 为什么需要 volatile

```c
// 没有 volatile 时
int running = 1;
void handler(int sig) { running = 0; }

// 编译器可能将 while(running) 优化为 while(true)
// 因为编译器看不到 handler 修改了 running

// 使用 volatile
volatile int running = 1;
// 编译器每次都会从内存读取 running 的值
```

### sig_atomic_t 类型

```c
// sig_atomic_t 保证读写是原子的
volatile sig_atomic_t counter = 0;

void increment_on_signal(int sig) {
    // 即使在信号处理中，对 sig_atomic_t 的简单赋值也是安全的
    counter++;
}
```

## 五、信号处理中的限制

信号处理函数中**只能调用异步信号安全（async-signal-safe）的函数**。

```c
void handler(int sig) {
    // 安全
    write(STDOUT_FILENO, "收到信号\n", 12);

    // 不安全！printf、malloc、free 等都不是异步信号安全的
    // printf("收到信号\n");  // 可能导致死锁或数据损坏
}
```

### 安全的做法

```c
volatile sig_atomic_t signal_received = 0;

void handler(int sig) {
    signal_received = 1;  // 只设置标志
}

int main() {
    signal(SIGINT, handler);

    while (1) {
        if (signal_received) {
            // 在主循环中处理信号，可以调用任意函数
            printf("处理信号...\n");
            signal_received = 0;
        }
    }
}
```

## 六、signal 处理的存储与恢复

```c
#include <stdio.h>
#include <signal.h>

void handler(int sig) {
    printf("处理信号 %d\n", sig);
}

int main() {
    // 保存旧的处理函数
    void (*old_handler)(int) = signal(SIGINT, handler);

    printf("第一次按 Ctrl+C 会调用 handler\n");

    // 恢复旧的处理函数
    signal(SIGINT, old_handler);

    printf("第二次按 Ctrl+C 会调用原始处理函数\n");

    return 0;
}
```

## 七、指针相关的信号：SIGSEGV

空指针解引用会触发 SIGSEGV：

```c
#include <stdio.h>
#include <signal.h>
#include <stdlib.h>

void segfault_handler(int sig) {
    fprintf(stderr, "段错误！信号 %d\n", sig);
    // 注意：不要在信号处理中继续运行，程序状态可能已损坏
    exit(1);
}

int main() {
    signal(SIGSEGV, segfault_handler);

    int *p = NULL;
    *p = 42;  // 触发 SIGSEGV

    return 0;
}
```

## 八、使用指针传递上下文

信号处理函数只能接收一个 int 参数（信号编号），不能直接传递上下文：

```c
// 解决方案：使用全局指针（需配合 volatile）
static volatile void *g_context = NULL;

void handler(int sig) {
    if (g_context != NULL) {
        // 使用上下文
    }
}

int main() {
    MyContext ctx;
    g_context = &ctx;
    signal(SIGINT, handler);
    // ...
}
```

## 九、sigaction（更现代的信号接口）

`sigaction` 比 `signal` 更可移植和功能更丰富：

```c
#include <signal.h>

struct sigaction sa;
sa.sa_handler = handler;
sigemptyset(&sa.sa_mask);
sa.sa_flags = 0;

sigaction(SIGINT, &sa, NULL);
```

## 十、关键要点总结

> **核心概念**
> - signal 函数使用函数指针注册处理函数
> - 信号处理函数中访问的全局变量必须用 volatile
> - 使用 sig_atomic_t 保证原子性

> **安全原则**
> - 信号处理函数中只调用异步信号安全的函数
> - 推荐只在处理函数中设置标志，在主循环中处理
> - SIGSEGV 通常表示程序状态已损坏，应尽快退出

> **volatile 的作用**
> - 防止编译器优化掉对变量的读取
> - 确保信号处理中的修改对主程序可见
