# 09 - 缓冲机制详解

## 一、什么是缓冲

### 缓冲的动机

磁盘I/O比内存操作慢数个数量级（毫秒 vs 纳秒）。缓冲机制通过在内存中积累数据，减少实际的磁盘读写次数，从而大幅提高I/O性能。

```
无缓冲（直接I/O）：
  write("A") → 磁盘
  write("B") → 磁盘
  write("C") → 磁盘
  共 3 次磁盘写入

有缓冲：
  write("A") → 缓冲区 [A]
  write("B") → 缓冲区 [AB]
  write("C") → 缓冲区 [ABC]
  flush → 磁盘 [ABC]
  共 1 次磁盘写入
```

### 缓冲的三种模式

| 模式 | 宏常量 | 刷新条件 | 默认适用场景 |
|------|--------|---------|------------|
| **全缓冲** | `_IOFBF` | 缓冲区满 / 调用fflush / 程序正常结束 | 磁盘文件 |
| **行缓冲** | `_IOLBF` | 遇到'\n' / 缓冲区满 / 调用fflush | 终端设备 |
| **无缓冲** | `_IONBF` | 每次写入立即输出 | stderr |

## 二、全缓冲（Full Buffering）

### 工作方式

数据先写入缓冲区，当缓冲区满了（通常是4KB或8KB）才真正写入设备。

```c
#include <stdio.h>

int main() {
    FILE *fp = fopen("output.txt", "w");  // 磁盘文件 → 全缓冲

    // 这些数据可能还在缓冲区中，尚未写入磁盘
    for (int i = 0; i < 100; i++) {
        fprintf(fp, "第 %d 行数据\n", i);
    }

    // 缓冲区可能仍未满，数据仍在内存中
    // 如果此时程序崩溃，数据将丢失！

    fclose(fp);  // fclose 会自动刷新缓冲区
    return 0;
}
```

### 全缓冲的典型缓冲区大小

```c
#include <stdio.h>

int main() {
    FILE *fp = fopen("test.txt", "w");

    // Linux/glibc: 默认缓冲区大小通常为 4096 或 8192 字节
    // 可以通过以下方式查看
    char buf[1024];
    setvbuf(fp, buf, _IOFBF, sizeof(buf));

    // 现在缓冲区大小为 1024 字节
    // 写入超过 1024 字节时会自动刷新

    fclose(fp);
    return 0;
}
```

## 三、行缓冲（Line Buffering）

### 工作方式

数据写入缓冲区，遇到换行符 `\n` 时刷新缓冲区。

```c
#include <stdio.h>

int main() {
    // stdout 连接到终端 → 行缓冲
    printf("这是第一行");     // 暂存在缓冲区
    printf("（同一行继续）"); // 暂存在缓冲区
    printf("\n");             // 遇到 \n，刷新！用户看到输出

    printf("第二行\n");       // 立即显示

    // 如果 stdout 被重定向到文件，可能变为全缓冲！
    // ./program > output.txt  时，stdout 变为全缓冲

    return 0;
}
```

### 行缓冲的特殊情况

```c
#include <stdio.h>

int main() {
    // 1. 输出到终端时：行缓冲
    printf("没有换行");        // 可能看不到

    // 2. 缓冲区满时也会刷新
    for (int i = 0; i < 5000; i++) {
        printf("x");           // 最终会刷新（缓冲区满）
    }

    // 3. 读取 stdin 时会刷新 stdout
    printf("输入名字: ");      // 行缓冲，无\n
    char name[100];
    scanf("%s", name);         // 读取 stdin 前会先刷新 stdout

    // 4. 显式刷新
    printf("Loading...");
    fflush(stdout);            // 强制刷新

    return 0;
}
```

## 四、无缓冲（No Buffering）

### stderr 的无缓冲

```c
#include <stdio.h>

int main() {
    // stderr 无缓冲 → 每次写入立即输出
    fprintf(stderr, "这");      // 立即输出
    fprintf(stderr, "些");      // 立即输出
    fprintf(stderr, "字");      // 立即输出

    // 即使没有 \n，也立即显示
    fprintf(stderr, "Debug: x=%d\n", 42);

    return 0;
}
```

### 无缓冲 vs 全缓冲的性能对比

```
写入 10000 个字符 'A'：

无缓冲：
  for (i=0; i<10000; i++) fputc('A', fp);
  → 10000 次系统调用，10000 次磁盘写入
  → 极慢

全缓冲（默认）：
  for (i=0; i<10000; i++) fputc('A', fp);
  → 约 2-3 次系统调用（缓冲区满时刷新）
  → 快
```

## 五、setvbuf - 设置缓冲模式

### 函数原型

```c
#include <stdio.h>
int setvbuf(FILE *stream, char *buf, int mode, size_t size);
```

- `stream`：文件流指针
- `buf`：用户提供的缓冲区（NULL表示由系统自动分配）
- `mode`：`_IOFBF`（全缓冲）、`_IOLBF`（行缓冲）、`_IONBF`（无缓冲）
- `size`：缓冲区大小
- **返回值**：成功返回 `0`，失败返回非零值

> **必须在打开文件后、任何读写操作前调用 `setvbuf`！**

### 使用示例

```c
#include <stdio.h>

int main() {
    // 示例1：设置全缓冲，自定义缓冲区大小
    FILE *fp1 = fopen("large_output.txt", "w");
    char buf1[16384];  // 16KB缓冲区
    setvbuf(fp1, buf1, _IOFBF, sizeof(buf1));

    // 示例2：让系统自动分配缓冲区
    FILE *fp2 = fopen("data.txt", "w");
    setvbuf(fp2, NULL, _IOFBF, 8192);

    // 示例3：设置为无缓冲
    FILE *fp3 = fopen("debug.log", "w");
    setvbuf(fp3, NULL, _IONBF, 0);  // size 无意义，可设为0

    // 示例4：设置行缓冲
    FILE *fp4 = fopen("console.log", "w");
    setvbuf(fp4, NULL, _IOLBF, 4096);

    // 使用...
    fclose(fp1);
    fclose(fp2);
    fclose(fp3);
    fclose(fp4);
    return 0;
}
```

## 六、fflush - 强制刷新缓冲区

### 函数原型

```c
int fflush(FILE *stream);
```

- 将指定流的缓冲区内容写入目标设备
- 如果 `stream` 为 `NULL`，刷新所有打开的输出流
- **返回值**：成功返回 `0`，失败返回 `EOF`

### 使用场景

```c
#include <stdio.h>

int main() {
    // 场景1：输出进度信息（无换行）
    printf("Processing: 50%%");
    fflush(stdout);   // 用户立即看到进度
    // ... 继续处理 ...
    printf("\rProcessing: 100%%\n");

    // 场景2：日志文件需要立即持久化
    FILE *log = fopen("app.log", "a");
    fprintf(log, "[ERROR] 严重错误发生！\n");
    fflush(log);      // 确保错误日志立即写入磁盘
    // 如果不刷新，程序崩溃时日志可能丢失

    // 场景3：调试输出
    fprintf(stderr, "到达检查点 A\n");
    fflush(stderr);   // 虽然stderr无缓冲，但这是个好习惯

    // 场景4：刷新所有流
    fflush(NULL);     // 刷新所有打开的流

    fclose(log);
    return 0;
}
```

## 七、缓冲策略选择指南

### 何时使用哪种缓冲

| 应用场景 | 推荐缓冲 | 原因 |
|---------|---------|------|
| 大文件批量读写 | 全缓冲（大缓冲区） | 减少系统调用，提高吞吐 |
| 终端交互输出 | 行缓冲 | 用户按行查看输出 |
| 错误/调试日志 | 无缓冲 | 确保立即持久化 |
| 实时进度显示 | 行缓冲 + fflush | 按需刷新 |
| 关键数据保存 | 全缓冲 + 手动fflush | 平衡性能和可靠性 |

### 自定义缓冲区大小的影响

```
缓冲区过小（如 64 字节）：
  + 数据很快刷新，丢失风险低
  - 频繁系统调用，性能差

缓冲区过大（如 1MB）：
  + 系统调用少，性能好
  - 数据丢失风险高，内存占用大

适中大小（4KB - 64KB）：
  平衡性能和安全性，通常为最佳选择
```

## 八、重点与注意事项

> **要点总结：**
>
> 1. **全缓冲**适合文件，**行缓冲**适合终端，**无缓冲**适合错误输出
> 2. **缓冲区满、遇到换行符、调用 fflush、程序正常退出** 都会触发刷新
> 3. **程序异常退出（崩溃、abort）时缓冲区不会刷新**，导致数据丢失
> 4. `setvbuf` 必须在打开文件后、第一次读写前调用
> 5. **fflush(NULL)** 刷新所有输出流
> 6. 向终端输出提示后应 `fflush(stdout)` 确保用户看到
> 7. 对于关键数据（日志、金融记录），应在关键操作后调用 `fflush`
