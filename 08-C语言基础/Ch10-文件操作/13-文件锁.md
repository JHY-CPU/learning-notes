# 13 - 文件锁

## 一、为什么需要文件锁

### 并发访问的问题

```
进程A 和 进程B 同时写入 data.txt：

时间线：
  进程A: read(100) → write(200) → close
  进程B:    read(100) → write(300) → close

结果：进程A的写入被进程B覆盖，数据丢失！
```

### 文件锁的作用

文件锁用于协调多个进程对同一文件的访问，防止数据竞争：

- **共享锁（读锁）**：多个进程可同时持有，用于读取操作
- **排他锁（写锁）**：只允许一个进程持有，用于写入操作

```
共享锁（读锁）：  P1 ←读→ 文件 ←读→ P2  （可以同时读）
排他锁（写锁）：  P1 ←写→ 文件           （独占写入）
```

## 二、flock - 简单文件锁

### 函数原型

```c
#include <sys/file.h>
int flock(int fd, int operation);
```

| operation | 含义 |
|-----------|------|
| `LOCK_SH` | 加共享锁（读锁） |
| `LOCK_EX` | 加排他锁（写锁） |
| `LOCK_UN` | 解锁 |
| `LOCK_NB` | 非阻塞（与上述组合使用） |

### 使用示例

```c
#include <stdio.h>
#include <sys/file.h>
#include <unistd.h>

int main() {
    FILE *fp = fopen("data.txt", "a");
    if (!fp) { perror("打开失败"); return 1; }

    int fd = fileno(fp);

    // 尝试获取排他锁（非阻塞）
    if (flock(fd, LOCK_EX | LOCK_NB) == -1) {
        printf("文件已被其他进程锁定！\n");
        fclose(fp);
        return 1;
    }

    // 获取到锁，安全写入
    fprintf(fp, "进程 %d 写入的数据\n", getpid());
    fflush(fp);

    printf("写入完成，按回车释放锁...");
    getchar();

    flock(fd, LOCK_UN);  // 释放锁
    fclose(fp);
    return 0;
}
```

### flock 的特点

```
  + 简单易用，跨BSD系统
  + 锁与文件描述符关联，close 时自动释放
  - 只能锁定整个文件，不能锁定部分内容
  - 在NFS文件系统上可能不可靠
  - Linux 2.6.12 之前，flock 在NFS上不起作用
```

## 三、fcntl - 精细文件锁（POSIX）

### 函数原型

```c
#include <fcntl.h>
int fcntl(int fd, int cmd, ... /* struct flock *arg */);
```

### struct flock 结构体

```c
struct flock {
    short l_type;    // 锁类型: F_RDLCK, F_WRLCK, F_UNLCK
    short l_whence;  // 起始位置: SEEK_SET, SEEK_CUR, SEEK_END
    off_t l_start;   // 相对于 l_whence 的偏移
    off_t l_len;     // 锁定的长度（0表示到文件末尾）
    pid_t l_pid;     // 持有锁的进程ID（仅 F_GETLK 时有效）
};
```

### 锁定文件的指定区域

```c
#include <stdio.h>
#include <fcntl.h>
#include <unistd.h>
#include <sys/types.h>

// 加锁指定区域
int lock_region(int fd, int lock_type, off_t start, off_t len) {
    struct flock fl;
    fl.l_type   = lock_type;    // F_RDLCK 或 F_WRLCK
    fl.l_whence = SEEK_SET;
    fl.l_start  = start;
    fl.l_len    = len;
    fl.l_pid    = 0;

    return fcntl(fd, F_SETLKW, &fl);  // F_SETLKW = 阻塞等待
}

// 解锁指定区域
int unlock_region(int fd, off_t start, off_t len) {
    struct flock fl;
    fl.l_type   = F_UNLCK;
    fl.l_whence = SEEK_SET;
    fl.l_start  = start;
    fl.l_len    = len;

    return fcntl(fd, F_SETLK, &fl);  // F_SETLK = 非阻塞
}

// 检查锁状态
int check_lock(int fd, off_t start, off_t len) {
    struct flock fl;
    fl.l_type   = F_WRLCK;
    fl.l_whence = SEEK_SET;
    fl.l_start  = start;
    fl.l_len    = len;

    if (fcntl(fd, F_GETLK, &fl) == -1) {
        return -1;
    }

    if (fl.l_type == F_UNLCK) {
        printf("区域未被锁定\n");
        return 0;
    } else {
        printf("区域已被锁定: 类型=%s, PID=%d\n",
               fl.l_type == F_RDLCK ? "读锁" : "写锁",
               fl.l_pid);
        return 1;
    }
}

int main() {
    int fd = open("data.txt", O_RDWR | O_CREAT, 0644);
    if (fd == -1) { perror("打开失败"); return 1; }

    // 锁定第100-199字节区域
    if (lock_region(fd, F_WRLCK, 100, 100) == 0) {
        printf("区域锁定成功\n");

        // 在锁定区域安全写入
        lseek(fd, 100, SEEK_SET);
        write(fd, "locked data", 11);

        unlock_region(fd, 100, 100);
    }

    close(fd);
    return 0;
}
```

### fcntl 命令选项

| cmd | 含义 |
|-----|------|
| `F_SETLK` | 设置锁（非阻塞），失败立即返回 |
| `F_SETLKW` | 设置锁（阻塞），等待直到获取锁 |
| `F_GETLK` | 获取锁信息（不实际加锁） |

## 四、读写锁的典型应用

### 数据库记录锁

```c
#include <stdio.h>
#include <fcntl.h>
#include <unistd.h>
#include <string.h>

typedef struct {
    int id;
    char name[32];
    double balance;
} Account;

#define RECORD_SIZE sizeof(Account)
#define ACCOUNT_OFFSET(n) ((off_t)(n) * RECORD_SIZE)

int read_account(int fd, int index, Account *acc) {
    // 获取读锁
    struct flock fl = {
        .l_type   = F_RDLCK,
        .l_whence = SEEK_SET,
        .l_start  = ACCOUNT_OFFSET(index),
        .l_len    = RECORD_SIZE
    };

    if (fcntl(fd, F_SETLKW, &fl) == -1) return -1;

    // 安全读取
    lseek(fd, ACCOUNT_OFFSET(index), SEEK_SET);
    ssize_t n = read(fd, acc, RECORD_SIZE);

    // 释放锁
    fl.l_type = F_UNLCK;
    fcntl(fd, F_SETLK, &fl);

    return (n == RECORD_SIZE) ? 0 : -1;
}

int write_account(int fd, int index, const Account *acc) {
    // 获取写锁
    struct flock fl = {
        .l_type   = F_WRLCK,
        .l_whence = SEEK_SET,
        .l_start  = ACCOUNT_OFFSET(index),
        .l_len    = RECORD_SIZE
    };

    if (fcntl(fd, F_SETLKW, &fl) == -1) return -1;

    // 安全写入
    lseek(fd, ACCOUNT_OFFSET(index), SEEK_SET);
    ssize_t n = write(fd, acc, RECORD_SIZE);

    // 释放锁
    fl.l_type = F_UNLCK;
    fcntl(fd, F_SETLK, &fl);

    return (n == RECORD_SIZE) ? 0 : -1;
}
```

## 五、flock 与 fcntl 的比较

| 特性 | flock | fcntl |
|------|-------|-------|
| 锁定粒度 | 整个文件 | 文件的任意区域 |
| 锁类型 | 共享/排他 | 读锁/写锁 |
| 阻塞选项 | LOCK_NB | F_SETLK/F_SETLKW |
| 自动释放 | close 时 | 进程结束时 |
| NFS 兼容 | 较差 | 较好 |
| 标准 | BSD | POSIX |
| 复杂度 | 简单 | 复杂 |

## 六、文件锁的注意事项

### 锁是劝告性的（Advisory Locking）

```c
// 文件锁是"劝告性"的：
// 只有所有进程都遵守锁协议时才能正确工作
// 一个不检查锁的进程仍然可以读写被锁定的文件

// 进程A：遵守锁协议
flock(fd, LOCK_EX);
write(fd, data, len);  // 安全
flock(fd, LOCK_UN);

// 进程B：不遵守锁协议
write(fd, data, len);  // 仍然可以写入！（没有检查锁）
```

### 锁与文件描述符/文件的关系

```
flock 的锁与文件描述符关联：
  - 同一进程的 dup() 文件描述符共享锁
  - fork() 后子进程继承文件描述符，也共享锁
  - close() 释放锁

fcntl 的锁与进程关联：
  - 同一进程的不同文件描述符指向同一文件时共享锁
  - 进程结束时所有锁自动释放
```

## 七、重点与注意事项

> **要点总结：**
>
> 1. **文件锁默认是劝告性的**——不检查锁的进程可以无视锁
> 2. **flock 锁整个文件**，fcntl 可以锁文件的任意区域
> 3. **进程退出时所有锁自动释放**，但推荐显式解锁
> 4. **fork() 后子进程继承锁**，注意父子进程间避免死锁
> 5. **在NFS上使用文件锁需谨慎**，建议使用 fcntl 而非 flock
> 6. `F_SETLKW` 阻塞等待，`F_SETLK` 立即返回失败
> 7. 在多线程程序中，同一进程的所有线程共享文件锁
