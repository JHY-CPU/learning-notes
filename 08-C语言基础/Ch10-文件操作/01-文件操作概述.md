# 01 - 文件操作概述

## 一、什么是文件

文件是存储在外部介质（如硬盘、SSD、U盘）上的数据集合，是操作系统进行数据管理的基本单位。在C语言中，文件操作是程序与外部世界进行数据交换的重要手段。

### 文件的分类

从C语言的角度，文件可以分为两大类：

| 类型 | 说明 | 示例 |
|------|------|------|
| **文本文件** | 以ASCII码或Unicode字符存储，按行组织 | .txt, .c, .h, .csv |
| **二进制文件** | 以字节为单位存储原始数据 | .exe, .jpg, .dat |

### 文本文件与二进制文件的本质区别

```
整数 12345 在不同文件中的存储方式：

文本文件：'1' '2' '3' '4' '5'  →  5个字节（每个字符的ASCII码）
         0x31 0x32 0x33 0x34 0x35

二进制文件：0x30 0x39  →  2个字节（16位整数的原始表示）
          （大端模式：高字节0x30，低字节0x39）
```

## 二、流（Stream）的概念

### 什么是流

**流（Stream）** 是C标准库对文件操作的抽象模型。它将所有的I/O设备（磁盘文件、键盘、屏幕、网络等）统一抽象为"字节流"，程序员只需与流打交道，无需关心底层设备的具体差异。

```
┌──────────┐     ┌──────────┐     ┌──────────┐
│  程序     │ ←→  │   流      │ ←→  │  文件/设备 │
│ (Buffer) │     │ (Stream) │     │ (Device) │
└──────────┘     └──────────┘     └──────────┘
```

### 流的类型

- **文本流（Text Stream）**：由字符行组成，每行以换行符 `\n` 结尾。不同平台的换行符可能不同（Windows: `\r\n`，Linux: `\n`），但C库会自动转换。
- **二进制流（Binary Stream）**：字节序列，不做任何转换，精确地反映数据在内存中的表示。

### 流的缓冲

流通过**缓冲区（Buffer）** 提高I/O效率：

```
程序写入 → 缓冲区（积累数据）→ 一次性写入磁盘
程序读取 ← 缓冲区（预读数据）← 一次性从磁盘读取
```

## 三、文件指针 FILE*

### FILE 结构体

`FILE` 是C标准库定义的一个结构体类型，用于封装文件操作所需的所有信息：

```c
// FILE结构体的典型内容（不同实现细节不同）
typedef struct {
    int _file;          // 文件描述符
    char *_ptr;         // 当前读写位置指针
    char *_base;        // 缓冲区基地址
    int _bufsiz;        // 缓冲区大小
    int _flag;          // 文件状态标志
    int _cnt;           // 缓冲区中剩余字符数
    // ... 其他成员
} FILE;
```

### 文件指针的使用

文件指针是进行文件操作的核心：

```c
#include <stdio.h>

int main() {
    FILE *fp;  // 声明文件指针

    fp = fopen("data.txt", "r");  // 打开文件
    if (fp == NULL) {
        printf("文件打开失败！\n");
        return 1;
    }

    // 在此进行文件读写操作...

    fclose(fp);  // 关闭文件
    return 0;
}
```

**重要原则**：
- `FILE*` 不是文件本身，而是程序与文件之间的"连接句柄"
- 文件操作完成后必须调用 `fclose()` 关闭
- 文件指针可能为 `NULL`，表示打开失败，必须检查

## 四、标准流

C程序启动时自动打开三个标准流，无需手动打开或关闭：

| 标准流 | 类型 | 文件指针 | 说明 | 默认连接 |
|--------|------|----------|------|----------|
| 标准输入 | 输入流 | `stdin` | 程序的默认输入源 | 键盘 |
| 标准输出 | 输出流 | `stdout` | 程序的默认输出目标 | 终端屏幕 |
| 标准错误 | 输出流 | `stderr` | 程序的错误输出目标 | 终端屏幕 |

### 标准流的使用示例

```c
#include <stdio.h>

int main() {
    char name[50];

    // 等价于 scanf("%s", name)
    fprintf(stdout, "请输入您的姓名: ");
    fscanf(stdin, "%s", name);

    // 等价于 printf("你好, %s\n", name)
    fprintf(stdout, "你好, %s\n", name);

    // 错误信息输出到 stderr
    fprintf(stderr, "[调试] 用户名长度: %lu\n", strlen(name));

    return 0;
}
```

### 为什么需要区分 stdout 和 stderr

```bash
# 将正常输出重定向到文件，错误信息仍显示在屏幕
./program > output.txt

# stdout 写入 output.txt，stderr 仍输出到终端
# 这样错误信息不会丢失在输出文件中
```

## 五、文件操作的基本流程

文件操作遵循固定的模式：

```c
// 1. 打开文件 → 2. 操作文件 → 3. 关闭文件

FILE *fp = fopen("example.txt", "w");  // 步骤1: 打开
if (fp == NULL) {
    perror("打开文件失败");
    return 1;
}

fprintf(fp, "Hello, World!\n");         // 步骤2: 操作
// ... 更多读写操作

fclose(fp);                             // 步骤3: 关闭
```

### 常见的文件操作函数速查

| 功能 | 文本模式函数 | 二进制模式函数 |
|------|-------------|---------------|
| 打开/关闭 | `fopen` / `fclose` | 同左 |
| 读取单个字符 | `fgetc` | 同左 |
| 写入单个字符 | `fputc` | 同左 |
| 读取字符串 | `fgets` | 同左 |
| 写入字符串 | `fputs` | 同左 |
| 格式化读写 | `fscanf` / `fprintf` | 同左 |
| 二进制读写 | — | `fread` / `fwrite` |
| 文件定位 | `fseek`, `ftell`, `rewind` | 同左 |
| 判断结束 | `feof` | 同左 |
| 判断错误 | `ferror` | 同左 |

## 六、重点与注意事项

> **要点总结：**
>
> 1. **FILE\* 是指针**，指向 FILE 结构体，不是文件本身
> 2. **流是对I/O设备的抽象**，统一了不同设备的操作接口
> 3. **标准流自动打开**：stdin、stdout、stderr 不需要也不应该关闭
> 4. **文本流 vs 二进制流**：文本流会做换行符转换，二进制流不做任何转换
> 5. **文件操作三步曲**：打开 → 操作 → 关闭，缺一不可
> 6. **fopen 返回 NULL 表示失败**，必须检查返回值
> 7. 缓冲机制的存在意味着 `fprintf` 后数据可能还未真正写入磁盘，`fclose` 会自动刷新缓冲区
