# 08 - 标准IO流

## 一、三个标准流

### stdin、stdout、stderr

C程序启动时，系统自动打开三个标准I/O流：

```c
#include <stdio.h>

// 三个全局变量，定义在 <stdio.h> 中
extern FILE *stdin;   // 标准输入
extern FILE *stdout;  // 标准输出
extern FILE *stderr;  // 标准错误
```

| 流 | 默认连接 | 用途 | 缓冲模式 |
|----|---------|------|---------|
| `stdin` | 键盘 | 程序输入 | 行缓冲 |
| `stdout` | 终端屏幕 | 正常输出 | 行缓冲 |
| `stderr` | 终端屏幕 | 错误输出 | 无缓冲 |

### 标准流的使用

```c
#include <stdio.h>

int main() {
    char name[100];

    // 使用标准流显式调用
    fputs("请输入姓名: ", stdout);
    fflush(stdout);  // 确保提示信息立即显示

    fgets(name, sizeof(name), stdin);

    // 输出到 stdout
    fprintf(stdout, "你好, %s", name);

    // 错误信息输出到 stderr
    fprintf(stderr, "[INFO] 用户输入了 %zu 个字符\n", strlen(name));

    return 0;
}
```

## 二、重定向

### 输出重定向

```bash
# 将 stdout 重定向到文件
./program > output.txt

# 追加模式
./program >> output.txt

# 将 stderr 重定向到文件
./program 2> error.txt

# 将 stdout 和 stderr 都重定向到同一个文件
./program > all.txt 2>&1
# 或者（Bash 4+）
./program &> all.txt
```

### 输入重定向

```bash
# 将文件内容作为 stdin
./program < input.txt

# 同时输入输出重定向
./program < input.txt > output.txt
```

### 重定向示例程序

```c
#include <stdio.h>

int main() {
    int n, sum = 0;

    // 从 stdin 读取（可能是键盘或重定向的文件）
    while (scanf("%d", &n) == 1) {
        sum += n;
        fprintf(stderr, "读取: %d\n", n);  // 这个输出到终端
    }

    // 输出到 stdout（可能重定向到文件）
    printf("总和: %d\n", sum);

    return 0;
}
```

```bash
# 运行方式
./sum < numbers.txt > result.txt
# numbers.txt 内容: 10 20 30
# result.txt 内容: 总和: 60
# 终端显示（stderr）:
#   读取: 10
#   读取: 20
#   读取: 30
```

### 重定向的实际应用场景

```bash
# 1. 保存程序输出
./analysis data.csv > report.txt

# 2. 分离正常输出和错误信息
./build.sh > build.log 2> build.err

# 3. 用文件驱动程序输入
./sorter < unsorted.txt > sorted.txt

# 4. 在脚本中处理程序输出
./get_version | grep "Version"
```

## 三、管道（Pipe）

### 管道的概念

管道将一个程序的 stdout 连接到另一个程序的 stdin：

```
┌──────────┐    stdout     ┌──────────┐    stdin
│ 程序 A    │ ──────────→  │ 程序 B    │
└──────────┘   (pipe)      └──────────┘
```

### 管道使用示例

```bash
# 将 ls 的输出通过管道传给 grep
ls -la | grep "\.txt"

# 多级管道
cat data.txt | sort | uniq | wc -l

# 程序示例：统计文本中每个单词出现的次数
cat article.txt | tr ' ' '\n' | sort | uniq -c | sort -rn | head -10
```

### 编写可管道化的程序

```c
// wordcount.c - 统计输入的单词数
#include <stdio.h>
#include <ctype.h>

int main() {
    int ch, words = 0, in_word = 0;

    while ((ch = getchar()) != EOF) {
        if (isspace(ch)) {
            in_word = 0;
        } else if (!in_word) {
            in_word = 1;
            words++;
        }
    }

    printf("%d\n", words);
    return 0;
}
```

```bash
# 使用管道
echo "Hello World Foo Bar" | ./wordcount
# 输出: 4

cat essay.txt | ./wordcount
# 输出: 1523
```

## 四、在程序中操作标准流

### 重新打开标准流

```c
#include <stdio.h>

int main() {
    // 将 stdin 重新打开为文件
    freopen("input.txt", "r", stdin);
    if (stdin == NULL) {
        perror("freopen stdin 失败");
        return 1;
    }

    // 现在 scanf 从文件读取，而非键盘
    int n;
    scanf("%d", &n);
    printf("从文件读取: %d\n", n);

    // 恢复 stdin（Unix/Linux）
    // freopen("/dev/tty", "r", stdin);

    return 0;
}
```

### 使用 fflush 刷新标准输出

```c
#include <stdio.h>

int main() {
    printf("Loading...");
    fflush(stdout);  // 强制立即输出（无换行符时尤其重要）

    // 执行耗时操作
    sleep(2);

    printf(" Done!\n");
    return 0;
}
```

### 标准流也可以作为函数参数

```c
#include <stdio.h>

// 通用的打印函数，可以输出到任何流
void log_message(FILE *stream, const char *level, const char *msg) {
    fprintf(stream, "[%s] %s\n", level, msg);
}

int main() {
    log_message(stdout, "INFO",  "程序启动");
    log_message(stderr, "ERROR", "发生错误");
    log_message(stdout, "DEBUG", "进入循环");
    return 0;
}
```

## 五、标准流的关闭

```c
// 标准流不需要也不应该手动关闭！
// 程序退出时，系统会自动关闭它们

// ❌ 错误：手动关闭标准流
fclose(stdin);   // 可能导致后续输入失败
fclose(stdout);  // 可能导致后续输出失败

// ✅ 正确：让系统自动管理
// 不需要做任何事
```

## 六、重点与注意事项

> **要点总结：**
>
> 1. **三个标准流自动打开**，不需要 `fopen`，也不应该 `fclose`
> 2. **stdout 有缓冲，stderr 无缓冲**——调试信息应输出到 stderr 以确保立即显示
> 3. **重定向不影响程序代码**，`printf` 和 `fprintf(stdout, ...)` 完全等价
> 4. **管道连接 stdout 和 stdin**，stderr 不受管道影响，始终输出到终端
> 5. 编写命令行工具时，遵循 Unix 哲学：从 stdin 读取，输出到 stdout，错误到 stderr
> 6. `freopen` 可以重新绑定标准流到文件
> 7. 在输出提示信息后应 `fflush(stdout)` 确保用户能看到
