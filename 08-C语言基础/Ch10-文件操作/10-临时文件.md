# 10 - 临时文件

## 一、什么是临时文件

### 临时文件的用途

临时文件是在程序运行过程中创建的、用完即删的中间文件：

```
常见使用场景：
- 文本编辑器的自动保存
- 编译器的中间文件
- 解压缩软件的临时解压
- 程序间的中间数据交换
- 大数据量的外部排序
```

### 临时文件的要求

- **唯一性**：不能与其他文件重名
- **安全性**：不被其他进程读取或篡改
- **可清理**：程序结束后能自动或手动删除
- **跨平台**：在不同操作系统上都能正确创建

## 二、tmpfile - 创建临时文件

### 函数原型

```c
#include <stdio.h>
FILE *tmpfile(void);
```

- 创建一个临时的二进制文件（模式为 `"wb+"`）
- 文件在 `fclose` 或程序退出时自动删除
- **返回值**：成功返回 `FILE*`，失败返回 `NULL`

### 使用示例

```c
#include <stdio.h>

int main() {
    FILE *tmp = tmpfile();
    if (tmp == NULL) {
        perror("创建临时文件失败");
        return 1;
    }

    // 写入临时数据
    fprintf(tmp, "这是临时数据\n");
    fprintf(tmp, "第二行数据\n");

    // 回到开头读取
    rewind(tmp);

    char line[256];
    while (fgets(line, sizeof(line), tmp) != NULL) {
        printf("临时文件内容: %s", line);
    }

    fclose(tmp);  // 关闭时文件自动删除
    // 程序结束后文件一定不存在

    return 0;
}
```

### tmpfile 的特点

```
优点：
  + 文件名自动生成，无需关心
  + 文件自动删除，无需手动清理
  + 在大多数系统上，文件创建后立即被"取消链接"
    （文件描述符仍然有效，但目录中已看不到文件名）

缺点：
  - 无法指定文件名或路径
  - 文件在关闭后不可再次打开
  - 部分系统（Windows）上可能被其他进程看到
```

## 三、tmpnam - 生成唯一文件名

### 函数原型

```c
#include <stdio.h>
char *tmpnam(char *s);
```

- 生成一个唯一的临时文件名字符串
- 如果 `s` 不是 `NULL`，文件名写入 `s`（至少 `L_tmpnam` 字节）
- 如果 `s` 是 `NULL`，返回指向静态缓冲区的指针（非线程安全）
- **不创建文件**，只生成名称

### 使用示例

```c
#include <stdio.h>
#include <string.h>

int main() {
    // 方式1：传入缓冲区（线程安全）
    char name1[L_tmpnam];
    tmpnam(name1);
    printf("临时文件名1: %s\n", name1);

    // 方式2：使用静态缓冲区（非线程安全）
    char *name2 = tmpnam(NULL);
    printf("临时文件名2: %s\n", name2);

    // 手动创建和使用
    char filename[L_tmpnam];
    tmpnam(filename);

    FILE *fp = fopen(filename, "w+b");
    if (!fp) { perror("创建失败"); return 1; }

    fwrite("临时数据", 1, 8, fp);
    fclose(fp);

    // 手动删除
    remove(filename);

    return 0;
}
```

### tmpnam 的安全问题

```c
// ⚠️ 存在竞态条件（TOCTOU问题）！

char name[L_tmpnam];
tmpnam(name);             // 步骤1：生成文件名
                          // ⚠️ 此时另一个进程可能创建同名文件
FILE *fp = fopen(name, "w");  // 步骤2：打开文件
                              // 可能覆盖或打开错误的文件！

// ⚠️ tmpnam 在C11/C23中已被标记为废弃
// 建议使用 mkstemp 或平台特定的安全API
```

## 四、mkstemp - 安全创建临时文件（POSIX）

### 函数原型

```c
#include <stdlib.h>
int mkstemp(char *template);
```

- `template`：文件名模板，必须以 `XXXXXX` 结尾，会被替换为唯一字符串
- **原子操作**：生成名称和创建文件是一个不可分割的操作
- **返回值**：成功返回文件描述符（`int`），失败返回 `-1`

### 使用示例

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>

int main() {
    char template[] = "/tmp/myapp_XXXXXX";

    int fd = mkstemp(template);
    if (fd == -1) {
        perror("mkstemp 失败");
        return 1;
    }

    printf("临时文件: %s\n", template);
    // template 被修改为实际的文件名

    // 使用 fdopen 将文件描述符转为 FILE*
    FILE *fp = fdopen(fd, "w+");
    if (!fp) {
        perror("fdopen 失败");
        close(fd);
        unlink(template);
        return 1;
    }

    fprintf(fp, "安全的临时文件数据\n");
    rewind(fp);

    char buf[256];
    fgets(buf, sizeof(buf), fp);
    printf("读取: %s", buf);

    fclose(fp);  // 关闭文件
    unlink(template);  // 删除文件

    return 0;
}
```

### mkstemp vs tmpnam

| 特性 | tmpnam | mkstemp |
|------|--------|---------|
| 创建文件 | 否（只生成名） | 是（原子操作） |
| 竞态条件 | 有（TOCTOU） | 无（原子） |
| 返回值 | 字符串 | 文件描述符 |
| 标准 | ISO C | POSIX |
| 推荐程度 | 不推荐 | 推荐 |

## 五、跨平台临时文件处理

### 封装示例

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

#ifdef _WIN32
    #include <windows.h>
    #include <io.h>
    #define F_OK 0
#else
    #include <unistd.h>
#endif

// 跨平台获取临时目录
const char *get_temp_dir() {
#ifdef _WIN32
    static char temp_path[MAX_PATH];
    GetTempPath(MAX_PATH, temp_path);
    return temp_path;
#else
    const char *dir = getenv("TMPDIR");
    return (dir != NULL) ? dir : "/tmp";
#endif
}

// 创建安全的临时文件
FILE *create_temp_file(const char *prefix, char *out_name, size_t name_size) {
    const char *temp_dir = get_temp_dir();

#ifdef _WIN32
    char temp_name[MAX_PATH];
    GetTempFileName(temp_dir, prefix, 0, temp_name);
    if (out_name) {
        strncpy(out_name, temp_name, name_size - 1);
        out_name[name_size - 1] = '\0';
    }
    return fopen(temp_name, "w+b");
#else
    snprintf(out_name, name_size, "%s/%s_XXXXXX", temp_dir, prefix);
    int fd = mkstemp(out_name);
    if (fd == -1) return NULL;
    FILE *fp = fdopen(fd, "w+b");
    if (!fp) {
        close(fd);
        unlink(out_name);
    }
    return fp;
#endif
}

int main() {
    char filename[256];
    FILE *tmp = create_temp_file("myapp", filename, sizeof(filename));

    if (!tmp) {
        perror("创建临时文件失败");
        return 1;
    }

    printf("临时文件: %s\n", filename);

    fprintf(tmp, "跨平台临时文件\n");
    fclose(tmp);

    // 清理
    remove(filename);
    printf("临时文件已删除\n");

    return 0;
}
```

## 六、临时文件的最佳实践

```
1. 优先使用 tmpfile()（简单场景）
   - 自动清理，最安全
   - 适合只需临时存储数据的场景

2. 需要文件名时使用 mkstemp（POSIX系统）
   - 原子操作，无竞态条件
   - 需要手动调用 unlink 删除

3. 避免使用 tmpnam（已废弃）
   - 存在安全漏洞
   - C11/C23 已标记为不推荐

4. 始终在程序退出前清理临时文件
   - 使用 atexit 注册清理函数
   - 处理信号中断（SIGINT, SIGTERM）

5. 临时文件应放在标准临时目录
   - Linux: /tmp 或 $TMPDIR
   - Windows: GetTempPath()
```

## 七、重点与注意事项

> **要点总结：**
>
> 1. **`tmpfile()` 最安全**——创建和删除都是自动的
> 2. **`tmpnam` 已废弃**——存在竞态条件，应使用 `mkstemp`
> 3. **`mkstemp` 是原子操作**——生成名称和创建文件不可分割
> 4. 临时文件使用后必须删除，避免磁盘空间泄漏
> 5. 使用 `atexit` 注册清理函数，确保程序异常退出时也能清理临时文件
> 6. Windows 和 Linux 的临时文件路径不同，需要平台特定处理
