# 14 - 内存映射文件

## 一、什么是内存映射

### 基本概念

内存映射（Memory Mapping）将文件的内容直接映射到进程的虚拟地址空间，使得对文件的读写变成对内存的访问，无需使用传统的 `read`/`write` 系统调用。

```
传统IO方式：
  磁盘 → 内核缓冲区 → 用户缓冲区 → 程序
  （两次数据拷贝）

内存映射方式：
  磁盘 ←→ 物理内存页 ←→ 虚拟地址空间（程序直接访问）
  （零次或一次数据拷贝，按需加载）
```

### mmap 的优势

- **减少数据拷贝**：不需要内核缓冲区和用户缓冲区之间的复制
- **随机访问友好**：像访问数组一样访问文件内容
- **大文件高效**：按需加载页面，不需要一次性读入整个文件
- **进程间共享**：多个进程可以映射同一文件实现共享内存

## 二、mmap 函数

### 函数原型

```c
#include <sys/mman.h>
void *mmap(void *addr, size_t length, int prot, int flags, int fd, off_t offset);
```

| 参数 | 说明 |
|------|------|
| `addr` | 期望映射的起始地址（通常设为 `NULL`，由系统选择） |
| `length` | 映射区域的大小（字节） |
| `prot` | 内存保护标志 |
| `flags` | 映射标志 |
| `fd` | 文件描述符 |
| `offset` | 文件中的起始偏移（必须是页面大小的整数倍） |

**返回值**：成功返回映射区域的指针，失败返回 `MAP_FAILED`（即 `(void*)-1`）

### prot 参数（保护标志）

| 宏 | 含义 |
|----|------|
| `PROT_READ` | 页面可读 |
| `PROT_WRITE` | 页面可写 |
| `PROT_EXEC` | 页面可执行 |
| `PROT_NONE` | 页面不可访问 |

可以按位或组合，如 `PROT_READ | PROT_WRITE`

### flags 参数（映射类型）

| 宏 | 含义 |
|----|------|
| `MAP_SHARED` | 修改写回文件，多进程共享 |
| `MAP_PRIVATE` | 使用写时复制，修改不写回文件 |
| `MAP_FIXED` | 强制使用指定的 addr |
| `MAP_ANONYMOUS` | 不映射文件（匿名映射），fd设为-1 |

## 三、munmap - 解除映射

```c
#include <sys/mman.h>
int munmap(void *addr, size_t length);
```

- `addr`：`mmap` 返回的指针
- `length`：映射区域的大小
- **返回值**：成功返回 `0`，失败返回 `-1`

## 四、基本使用示例

### 示例1：读取文件内容

```c
#include <stdio.h>
#include <stdlib.h>
#include <fcntl.h>
#include <unistd.h>
#include <sys/mman.h>
#include <sys/stat.h>

int main() {
    int fd = open("data.txt", O_RDONLY);
    if (fd == -1) { perror("打开失败"); return 1; }

    // 获取文件大小
    struct stat st;
    fstat(fd, &st);
    off_t filesize = st.st_size;

    // 映射文件到内存
    char *data = mmap(NULL, filesize, PROT_READ, MAP_PRIVATE, fd, 0);
    if (data == MAP_FAILED) {
        perror("mmap 失败");
        close(fd);
        return 1;
    }

    // 像访问数组一样读取文件内容
    printf("文件大小: %ld 字节\n", (long)filesize);
    printf("文件内容:\n");
    for (off_t i = 0; i < filesize; i++) {
        putchar(data[i]);
    }

    // 清理
    munmap(data, filesize);
    close(fd);
    return 0;
}
```

### 示例2：修改文件内容

```c
#include <stdio.h>
#include <fcntl.h>
#include <unistd.h>
#include <sys/mman.h>
#include <sys/stat.h>
#include <string.h>

int main() {
    // 打开文件（读写模式）
    int fd = open("data.txt", O_RDWR);
    if (fd == -1) { perror("打开失败"); return 1; }

    struct stat st;
    fstat(fd, &st);
    off_t filesize = st.st_size;

    // 映射为可写
    char *data = mmap(NULL, filesize, PROT_READ | PROT_WRITE, MAP_SHARED, fd, 0);
    if (data == MAP_FAILED) {
        perror("mmap 失败");
        close(fd);
        return 1;
    }

    // 直接修改内存，修改会自动写回文件
    // 将文件中的 'a' 全部替换为 'A'
    for (off_t i = 0; i < filesize; i++) {
        if (data[i] == 'a') {
            data[i] = 'A';
        }
    }

    // 强制将修改写回磁盘（可选）
    msync(data, filesize, MS_SYNC);

    munmap(data, filesize);
    close(fd);
    printf("文件已修改\n");
    return 0;
}
```

## 五、大文件处理

### 分块映射大文件

```c
#include <stdio.h>
#include <fcntl.h>
#include <unistd.h>
#include <sys/mman.h>
#include <sys/stat.h>

#define CHUNK_SIZE (1024 * 1024)  // 每次映射1MB

int process_large_file(const char *filename) {
    int fd = open(filename, O_RDONLY);
    if (fd == -1) { perror("打开失败"); return -1; }

    struct stat st;
    fstat(fd, &st);
    off_t filesize = st.st_size;

    printf("文件大小: %.2f MB\n", filesize / (1024.0 * 1024.0));

    // 分块处理
    off_t offset = 0;
    while (offset < filesize) {
        size_t chunk = CHUNK_SIZE;
        if (offset + chunk > filesize) {
            chunk = filesize - offset;
        }

        char *data = mmap(NULL, chunk, PROT_READ, MAP_PRIVATE, fd, offset);
        if (data == MAP_FAILED) {
            perror("mmap 失败");
            close(fd);
            return -1;
        }

        // 处理当前块
        for (size_t i = 0; i < chunk; i++) {
            // 对每个字节进行处理
            // 例如：统计特定字符的出现次数
        }

        munmap(data, chunk);
        offset += chunk;
    }

    close(fd);
    return 0;
}
```

## 六、进程间共享内存

### 使用 mmap 实现进程间通信

```c
// writer.c - 写入共享数据
#include <stdio.h>
#include <fcntl.h>
#include <unistd.h>
#include <sys/mman.h>
#include <string.h>

typedef struct {
    int count;
    char message[256];
} SharedData;

int main() {
    int fd = open("/tmp/shared.dat", O_RDWR | O_CREAT, 0644);
    ftruncate(fd, sizeof(SharedData));  // 设置文件大小

    SharedData *shared = mmap(NULL, sizeof(SharedData),
                              PROT_READ | PROT_WRITE,
                              MAP_SHARED, fd, 0);

    shared->count = 0;
    strcpy(shared->message, "初始消息");

    for (int i = 0; i < 10; i++) {
        shared->count++;
        snprintf(shared->message, 256, "消息 #%d", i);
        printf("写入: %s\n", shared->message);
        sleep(1);
    }

    munmap(shared, sizeof(SharedData));
    close(fd);
    unlink("/tmp/shared.dat");
    return 0;
}
```

```c
// reader.c - 读取共享数据
#include <stdio.h>
#include <fcntl.h>
#include <unistd.h>
#include <sys/mman.h>

typedef struct {
    int count;
    char message[256];
} SharedData;

int main() {
    sleep(2);  // 等待 writer 先创建文件

    int fd = open("/tmp/shared.dat", O_RDONLY);
    if (fd == -1) { perror("打开失败"); return 1; }

    SharedData *shared = mmap(NULL, sizeof(SharedData),
                              PROT_READ, MAP_SHARED, fd, 0);

    int last_count = -1;
    while (1) {
        if (shared->count != last_count) {
            printf("读取: count=%d, message=%s\n",
                   shared->count, shared->message);
            last_count = shared->count;

            if (last_count >= 9) break;
        }
        usleep(100000);  // 100ms
    }

    munmap(shared, sizeof(SharedData));
    close(fd);
    return 0;
}
```

## 七、msync - 同步映射数据

```c
#include <sys/mman.h>
int msync(void *addr, size_t length, int flags);
```

| flags | 含义 |
|-------|------|
| `MS_SYNC` | 同步写回，阻塞直到完成 |
| `MS_ASYNC` | 异步写回，立即返回 |
| `MS_INVALIDATE` | 使其他映射的缓存失效 |

```c
// 确保修改写回磁盘
msync(data, length, MS_SYNC);
```

## 八、mmap vs 标准IO 对比

| 特性 | mmap | fread/fwrite |
|------|------|-------------|
| 数据拷贝 | 零拷贝或一次拷贝 | 两次拷贝 |
| 随机访问 | 直接指针运算 | 需要 fseek |
| 大文件 | 按需加载页面 | 需手动分块 |
| 进程共享 | 支持（MAP_SHARED） | 不直接支持 |
| 缓冲区管理 | 操作系统自动管理 | 用户自行管理 |
| 可移植性 | POSIX | ISO C 标准 |
| 适用场景 | 大文件随机访问、进程共享 | 简单顺序读写 |

## 九、重点与注意事项

> **要点总结：**
>
> 1. **offset 必须是页面大小的整数倍**（通常4KB），可用 `sysconf(_SC_PAGESIZE)` 获取
> 2. `MAP_SHARED` 的修改会写回文件，`MAP_PRIVATE` 使用写时复制不写回
> 3. 映射区域的大小不必与文件大小一致，但不能超过文件大小（读取时）
> 4. **映射后可以关闭文件描述符 fd**，映射仍然有效
> 5. `mmap` 失败返回 `MAP_FAILED`，不是 `NULL`
> 6. 使用 `msync` 确保数据持久化，否则可能因系统崩溃丢失数据
> 7. 匿名映射（`MAP_ANONYMOUS`）不涉及文件，fd 设为 -1，用于大块内存分配
