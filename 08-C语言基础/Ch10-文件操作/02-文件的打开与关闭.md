# 02 - 文件的打开与关闭

## 一、fopen 函数详解

### 函数原型

```c
#include <stdio.h>
FILE *fopen(const char *filename, const char *mode);
```

- `filename`：文件路径（可以是绝对路径或相对路径）
- `mode`：打开模式字符串
- **返回值**：成功返回 `FILE*` 指针，失败返回 `NULL`

### 打开模式一览

| 模式 | 含义 | 文件不存在时 | 文件存在时 | 读写位置 |
|------|------|------------|-----------|---------|
| `"r"` | 只读 | 报错 | 打开 | 文件开头 |
| `"w"` | 只写 | 创建 | **清空** | 文件开头 |
| `"a"` | 追加 | 创建 | 打开 | 文件末尾 |
| `"r+"` | 读写 | 报错 | 打开 | 文件开头 |
| `"w+"` | 读写 | 创建 | **清空** | 文件开头 |
| `"a+"` | 读写(追加) | 创建 | 打开 | 文件末尾 |

> 在模式后加 `"b"` 表示二进制模式，如 `"rb"`、`"wb"`、`"ab+"`

### 模式行为详解

```
"r"  - 只读模式
       文件不存在 → fopen 返回 NULL（报错）
       文件存在   → 正常打开，可读不可写

"w"  - 只写模式
       文件不存在 → 创建新文件
       文件存在   → 文件内容被清空！（长度截断为0）
       ⚠️ 危险：会丢失已有数据！

"a"  - 追加模式
       文件不存在 → 创建新文件
       文件存在   → 打开文件，写入位置强制在末尾
       特点：即使 fseek 也不能将写入位置移到开头

"r+" - 读写模式（文件必须已存在）
       既可读又可写
       写入会覆盖当前位置的数据，不会自动清空

"w+" - 读写模式（创建或清空）
       既可读又可写
       文件被清空，先写后读才有效

"a+" - 追加读写模式
       写操作始终在末尾追加
       读操作可以从任意位置开始
```

## 二、基本使用示例

### 示例1：只读打开

```c
#include <stdio.h>

int main() {
    FILE *fp = fopen("data.txt", "r");
    if (fp == NULL) {
        perror("打开文件失败");
        // perror 会输出: 打开文件失败: No such file or directory
        return 1;
    }

    char buffer[256];
    while (fgets(buffer, sizeof(buffer), fp) != NULL) {
        printf("%s", buffer);
    }

    fclose(fp);
    return 0;
}
```

### 示例2：写入模式（创建或覆盖）

```c
#include <stdio.h>

int main() {
    // "w" 模式：文件不存在则创建，存在则清空
    FILE *fp = fopen("output.txt", "w");
    if (fp == NULL) {
        perror("创建文件失败");
        return 1;
    }

    fprintf(fp, "第一行内容\n");
    fprintf(fp, "第二行内容\n");
    fprintf(fp, "数字: %d, 浮点: %.2f\n", 42, 3.14);

    fclose(fp);  // 关闭时自动刷新缓冲区
    printf("文件写入成功！\n");
    return 0;
}
```

### 示例3：追加模式

```c
#include <stdio.h>

int main() {
    // "a" 模式：在文件末尾追加内容
    FILE *fp = fopen("log.txt", "a");
    if (fp == NULL) {
        perror("打开文件失败");
        return 1;
    }

    fprintf(fp, "[%s] 新的日志条目\n", __DATE__);
    fclose(fp);

    printf("日志已追加！\n");
    return 0;
}
```

## 三、fclose 函数

### 函数原型

```c
int fclose(FILE *stream);
```

- **返回值**：成功返回 `0`，失败返回 `EOF`

### fclose 做了什么

```
1. 将缓冲区中残留的数据写入文件（刷新缓冲区）
2. 释放与文件关联的缓冲区内存
3. 释放 FILE 结构体占用的资源
4. 解除文件与操作系统的关联（释放文件锁等）
```

### 不关闭文件的后果

```c
// 错误示例：忘记关闭文件
void bad_function() {
    FILE *fp = fopen("data.txt", "w");
    fprintf(fp, "some data");
    // 忘记 fclose！
    // 后果：
    // 1. 缓冲区数据可能未写入磁盘（数据丢失）
    // 2. 文件描述符泄漏（占用系统资源）
    // 3. 其他程序可能无法访问该文件
    // 4. 进程可打开的文件数有限制（Linux默认1024）
}
```

### 正确的文件关闭模式

```c
// 方法1：简单直接
FILE *fp = fopen("data.txt", "r");
if (fp == NULL) return 1;
// ... 操作 ...
fclose(fp);

// 方法2：使用 goto 统一错误处理
FILE *fp = fopen("data.txt", "r");
if (fp == NULL) return 1;

// ... 一系列操作 ...
if (error_condition) goto cleanup;

// ... 更多操作 ...

cleanup:
    fclose(fp);
    return result;
```

## 四、文件打开失败的处理

### 常见失败原因

| errno 值 | 宏名 | 原因 |
|----------|------|------|
| 2 | ENOENT | 文件不存在（r模式） |
| 13 | EACCES | 权限不足 |
| 24 | EMFILE | 打开的文件数过多 |
| 28 | ENOSPC | 磁盘空间不足 |
| 21 | EISDIR | 路径是目录而非文件 |

### 使用 perror 输出错误信息

```c
#include <stdio.h>
#include <errno.h>
#include <string.h>

int main() {
    FILE *fp = fopen("/readonly_dir/test.txt", "w");
    if (fp == NULL) {
        // 方法1：使用 perror（最简单）
        perror("fopen 失败");
        // 输出: fopen 失败: Permission denied

        // 方法2：使用 strerror 获取错误描述
        printf("错误码: %d\n", errno);
        printf("错误信息: %s\n", strerror(errno));
        // 输出: 错误码: 13
        //       错误信息: Permission denied

        return 1;
    }
    fclose(fp);
    return 0;
}
```

### 使用 errno 需要包含的头文件

```c
#include <errno.h>    // errno 变量
#include <string.h>   // strerror() 函数
```

## 五、安全的文件打开模式

### 检查文件是否存在的安全写入

```c
#include <stdio.h>

int main() {
    // 先以 "r" 模式检查文件是否存在
    FILE *check = fopen("data.txt", "r");
    if (check != NULL) {
        fclose(check);
        printf("文件已存在，是否覆盖？(y/n): ");
        char c = getchar();
        if (c != 'y' && c != 'Y') {
            printf("操作取消。\n");
            return 0;
        }
    }

    // 安全地打开文件进行写入
    FILE *fp = fopen("data.txt", "w");
    if (fp == NULL) {
        perror("打开文件失败");
        return 1;
    }

    fprintf(fp, "新数据\n");
    fclose(fp);
    return 0;
}
```

## 六、文件路径的注意事项

```c
// 相对路径（相对于当前工作目录）
FILE *fp1 = fopen("data.txt", "r");          // 当前目录
FILE *fp2 = fopen("./data.txt", "r");        // 当前目录（显式）
FILE *fp3 = fopen("subdir/data.txt", "r");   // 子目录
FILE *fp4 = fopen("../data.txt", "r");       // 上级目录

// 绝对路径
// Windows:
FILE *fp5 = fopen("C:\\Users\\test\\data.txt", "r");
// Linux/macOS:
FILE *fp6 = fopen("/home/user/data.txt", "r");
```

> **注意**：Windows路径中反斜杠 `\` 需要转义为 `\\`，或者使用正斜杠 `/`：
> ```c
> fopen("C:/Users/test/data.txt", "r");  // 两种写法等价
> fopen("C:\\Users\\test\\data.txt", "r");
> ```

## 七、重点与注意事项

> **要点总结：**
>
> 1. **"w" 模式会清空文件**，使用前务必确认不会丢失重要数据
> 2. **fopen 返回 NULL 必须检查**，不检查会导致段错误（Segmentation Fault）
> 3. **fclose 可能失败**，在关键场景下应检查 fclose 的返回值
> 4. **文件描述符是有限资源**，打开文件后必须在用完后关闭
> 5. **"a" 模式的写入位置不可更改**，即使 fseek 也无法改变追加行为
> 6. 路径中的反斜杠在C语言字符串中需要转义
> 7. 文件操作函数通常设置 errno，配合 perror/strerror 可以获得详细错误信息
