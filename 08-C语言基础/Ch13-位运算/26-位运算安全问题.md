# 26 - 位运算安全问题

## 未定义行为概述

位运算中存在多种**未定义行为**（Undefined Behavior），会导致程序产生不可预测的结果。了解并避免这些陷阱至关重要。

## 1. 移位量越界

### 规则

- 移位量必须 >= 0
- 移位量必须 < 操作数类型的位宽（`sizeof(type) * 8`）
- 违反以上规则将导致未定义行为

```c
#include <stdio.h>

int main(void) {
    int a = 1;

    // 危险：未定义行为
    // a << 32;   // int 是32位，移32位是UB
    // a >> 32;   // 同样是UB
    // a << -1;   // 负移位量是UB
    // a >> 40;   // 更糟糕

    // 安全做法
    int shift = 31;  // 最大安全移位量（对于32位int）
    int result = a << shift;
    printf("1 << 31 = %d\n", result);

    return 0;
}
```

### 防护措施

```c
#include <stdio.h>
#include <stdint.h>

// 安全的左移
uint32_t safe_shift_left(uint32_t value, int shift) {
    if (shift < 0 || shift >= 32) return 0;
    return value << shift;
}

// 安全的右移
uint32_t safe_shift_right(uint32_t value, int shift) {
    if (shift < 0 || shift >= 32) return 0;
    return value >> shift;
}

int main(void) {
    printf("安全左移: %u\n", safe_shift_left(1, 31));
    printf("越界左移: %u\n", safe_shift_left(1, 32));  // 返回0
    printf("负移位: %u\n", safe_shift_left(1, -1));     // 返回0
    return 0;
}
```

## 2. 有符号整数溢出

### 问题

对有符号整数进行位操作可能导致有符号溢出，这是未定义行为。

```c
#include <stdio.h>
#include <limits.h>

int main(void) {
    // 危险：有符号左移溢出
    int x = 0x40000000;  // 1073741824
    // int y = x << 1;   // UB：结果 0x80000000 超出 int 范围

    // 安全做法：使用无符号类型
    unsigned int ux = 0x40000000U;
    unsigned int uy = ux << 1;  // 安全：0x80000000
    printf("安全: 0x%08X\n", uy);

    return 0;
}
```

## 3. 右移负数

### 问题

有符号数的右移行为是**实现定义的**：大多数平台使用算术右移（补符号位），但标准不保证。

```c
#include <stdio.h>

int main(void) {
    int x = -8;  // 补码: 11111111 11111111 11111111 11111000

    // 实现定义的行为
    int y = x >> 1;  // 大多数平台上是 -4（算术右移）
    printf("-8 >> 1 = %d\n", y);

    // 如果需要确定的行为，先转换为无符号
    unsigned int ux = (unsigned int)x;
    unsigned int uy = ux >> 1;  // 确定的行为（逻辑右移）
    printf("无符号右移: %u\n", uy);

    return 0;
}
```

## 4. 按位取反与有符号数

```c
#include <stdio.h>

int main(void) {
    // ~ 对有符号数的影响
    int a = 0;
    int b = ~a;  // -1（所有位为1的补码表示）

    printf("~0 = %d\n", b);  // -1

    // 注意：~x 不等于 !x
    int x = 5;
    printf("~5 = %d（位取反）\n", ~x);   // -6
    printf("!5 = %d（逻辑非）\n", !x);    // 0

    return 0;
}
```

## 5. 运算符优先级陷阱

位运算符的优先级经常被误解，导致逻辑错误。

```c
#include <stdio.h>

int main(void) {
    // 常见错误

    // 错误1：位运算 vs 比较运算
    int flags = 0x05;
    // if (flags & 0x01 == 1)  // 错误！== 优先级高于 &
    // 正确写法：
    if ((flags & 0x01) == 1)
        printf("第0位为1\n");

    // 错误2：位运算 vs 算术运算
    int a = 5;
    // int b = a + 1 << 2;  // 错误！可能不是期望的结果
    // 正确写法：
    int b = (a + 1) << 2;
    printf("(5+1)<<2 = %d\n", b);  // 24

    // 错误3：混合 & 和 |
    int c = 0x01 & 0x02 | 0x04;  // 等价于 (0x01 & 0x02) | 0x04 = 4
    // 如果意图是 0x01 & (0x02 | 0x04)，结果完全不同

    return 0;
}
```

### 运算符优先级参考

```
高优先级（先执行）:
  ~           按位取反
  <<  >>      移位
  &           按位与
  ^           按位异或
  |           按位或
低优先级（后执行）:
  ==  !=      比较
  &&          逻辑与
  ||          逻辑或
```

**建议**：对所有位运算表达式使用括号明确优先级。

## 6. 类型提升问题

```c
#include <stdio.h>

int main(void) {
    // 小整数类型会先提升为 int
    unsigned char a = 0xFF;
    unsigned char b = ~a;  // 危险！

    // ~a 实际上是 ~(int)0xFF = 0xFFFFFF00
    // 赋值给 unsigned char 时截断为 0x00
    printf("~0xFF (unsigned char) = 0x%02X\n", b);  // 0x00

    // 安全做法：显式转换
    unsigned char c = (unsigned char)(~a);
    printf("显式转换: 0x%02X\n", c);

    // 对于掩码操作，使用正确大小的类型
    unsigned int mask = ~0U;  // 全1
    printf("32位全1: 0x%08X\n", mask);

    return 0;
}
```

## 7. 移位赋值的类型陷阱

```c
#include <stdio.h>

int main(void) {
    // 1 << 31 在 int（有符号32位）上是未定义行为
    // int bad = 1 << 31;  // UB！

    // 使用 unsigned 类型
    unsigned int good = 1U << 31;
    printf("1U << 31 = 0x%08X\n", good);  // 0x80000000

    // 对于 64 位移位
    unsigned long long big = 1ULL << 40;
    printf("1ULL << 40 = 0x%llX\n", big);

    return 0;
}
```

## 安全编码检查清单

```c
// 位运算安全检查清单：

// 1. 永远使用 unsigned 类型进行位运算
unsigned int flags = 0;

// 2. 移位量使用 unsigned 字面量
unsigned int x = 1U << 5;

// 3. 检查移位量范围
int shift = 10;
if (shift >= 0 && shift < 32) {
    unsigned int result = x << shift;
}

// 4. 使用括号明确优先级
if ((flags & MASK) == VALUE) { }

// 5. 注意类型提升
unsigned char c = 0xFF;
unsigned int extended = c;  // 0x000000FF，不是 0xFFFFFFFF

// 6. 避免依赖实现定义的行为
// 有符号右移、有符号溢出等应避免
```

## 关键要点

- 移位量必须在 `[0, 类型位宽)` 范围内
- 使用 `unsigned` 类型避免有符号溢出的未定义行为
- 有符号数的右移是实现定义的
- 位运算优先级容易出错，使用括号明确优先级
- 小整数类型在位运算前会提升为 `int`
- 字面量加 `U` 后缀（如 `1U`、`0xFFU`）确保无符号运算
