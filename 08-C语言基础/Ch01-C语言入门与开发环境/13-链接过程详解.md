# 链接过程详解

## 一、链接概述

链接是编译的最后一个阶段，将多个目标文件和库文件合并为一个可执行文件（或共享库）。

```
目标文件A.o ──┐
目标文件B.o ──┼──→ 链接器（ld）──→ 可执行文件 / 共享库
库文件.a/.so ──┘
```

### 链接的核心任务

1. **符号解析**：将每个符号引用与符号定义关联
2. **重定位**：修正代码和数据的地址引用
3. **合并段**：将各目标文件的段合并到最终文件

## 二、符号解析

### 2.1 符号的定义与引用

```c
/* main.c */
#include <stdio.h>

extern int helper_value;      /* 符号引用：声明但未定义 */
void helper_function(void);   /* 符号引用：声明但未定义 */

int main_value = 10;          /* 符号定义：全局变量 */

int main(void) {              /* 符号定义：函数 */
    helper_function();        /* 符号引用：调用 */
    printf("%d\n", helper_value + main_value);
    return 0;
}
```

```c
/* helper.c */
int helper_value = 20;        /* 符号定义：全局变量 */

void helper_function(void) {  /* 符号定义：函数 */
    printf("Helper called\n");
}
```

### 2.2 符号解析过程

```
链接器维护：
- 全局符号表（E表）：记录所有已定义的全局符号
- 未解析符号表（U表）：记录所有引用但未定义的符号

处理 main.o：
  - U表添加: helper_value, helper_function
  - E表添加: main_value, main

处理 helper.o：
  - E表添加: helper_value, helper_function
  - 从U表移除: helper_value, helper_function

链接完成时U表应为空，否则报错。
```

### 2.3 符号解析规则

**规则1：强符号与弱符号**
- 函数和已初始化的全局变量是**强符号**
- 未初始化的全局变量是**弱符号**

```c
/* 强符号 */
int x = 10;
void func(void) { }

/* 弱符号 */
int y;
extern int z;
```

**规则2：解析规则**
1. 不允许有多个同名的强符号
2. 如果有一个强符号和多个弱符号同名，选择强符号
3. 如果有多个弱符号同名，任选一个

## 三、重定位

### 3.1 重定位的概念

编译器在生成目标文件时，不知道其他符号的最终地址，所以使用占位地址。链接器在知道所有符号的最终地址后，修正这些地址引用。

### 3.2 重定位过程

```
步骤1：合并段
  将所有目标文件的 .text 合并
  将所有目标文件的 .data 合并
  ...

步骤2：确定地址
  假设 .text 段起始地址: 0x401000
  main 的地址: 0x401000
  helper_function 的地址: 0x401050

步骤3：修正引用
  main.c 中调用 helper_function 的地方
  需要将占位地址修正为 0x401050
```

### 3.3 重定位类型

```bash
# 查看重定位信息
objdump -r main.o

# 输出示例：
# OFFSET   TYPE           VALUE
# 0000001a R_X86_64_PC32  helper_function-0x00000004
# 00000024 R_X86_64_PC32  helper_value-0x00000004
```

常见重定位类型（x86-64）：
- `R_X86_64_64`：64位绝对地址
- `R_X86_64_PC32`：32位PC相对地址
- `R_X86_64_PLT32`：通过PLT的32位PC相对地址
- `R_X86_64_GOT32`：GOT表项的32位偏移

## 四、静态链接

### 4.1 静态链接的概念

静态链接在编译时将库的代码完全复制到可执行文件中。

```bash
# 使用静态库
gcc main.c -L./lib -lmylib -o program

# 强制静态链接所有库
gcc -static main.c -o program
```

### 4.2 静态库的链接过程

```
main.o ──┐
         ├──→ 链接器 ──→ program (包含所有代码)
lib.a  ──┘
```

```
静态库 libmylib.a 内部:
┌─────────────────┐
│ helper.o        │  被引用，链接到程序
│ utils.o         │  未被引用，不链接
│ math_ops.o      │  被引用，链接到程序
└─────────────────┘

链接器只抽取被引用的目标文件
```

### 4.3 静态链接的特点

| 优点 | 缺点 |
|------|------|
| 可执行文件自包含 | 文件体积大 |
| 无运行时依赖 | 库更新需要重新编译 |
| 部署简单 | 多程序共享时浪费内存 |
| 运行效率高 | 符号冲突风险 |

## 五、动态链接

### 5.1 动态链接的概念

动态链接在运行时才加载共享库，多个程序可以共享同一份库代码。

```bash
# 编译共享库
gcc -fPIC -shared lib.c -o libmylib.so

# 使用共享库编译
gcc main.c -L./lib -lmylib -o program

# 运行时需要设置库搜索路径
export LD_LIBRARY_PATH=./lib:$LD_LIBRARY_PATH
./program
```

### 5.2 动态链接的机制

```
┌─────────────────────────────────────────────────────────┐
│                    可执行文件                             │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐  │
│  │    .text     │  │     GOT      │  │     PLT      │  │
│  │  程序代码     │  │ 全局偏移表    │  │ 过程链接表    │  │
│  │              │  │              │  │              │  │
│  │  call printf │→ │  printf@got  │→ │  printf@plt  │  │
│  │              │  │  (运行时填充) │  │  (跳板代码)   │  │
│  └──────────────┘  └──────────────┘  └──────────────┘  │
└─────────────────────────────────────────────────────────┘
         ↓ 运行时
┌─────────────────────────────────────────────────────────┐
│                  libc.so.6 (共享库)                      │
│  ┌──────────────┐                                       │
│  │   printf     │ ← GOT指向这里                         │
│  └──────────────┘                                       │
└─────────────────────────────────────────────────────────┘
```

### 5.3 GOT和PLT

**GOT（Global Offset Table，全局偏移表）**：
- 存储全局变量和函数的实际地址
- 在程序加载时由动态链接器填充

**PLT（Procedure Linkage Table，过程链接表）**：
- 为函数调用提供跳板
- 首次调用时触发动态链接（懒绑定）
- 后续调用直接跳转到GOT记录的地址

### 5.4 动态链接的特点

| 优点 | 缺点 |
|------|------|
| 文件体积小 | 运行时依赖库文件 |
| 多程序共享库代码 | 启动稍慢（需要链接） |
| 库更新无需重新编译 | 部署需确保库存在 |
| 内存共享节省资源 | 版本兼容性问题 |

## 六、链接命令详解

### 6.1 GCC链接选项

```bash
# 基本链接
gcc main.o utils.o -o program

# 指定库搜索路径
gcc main.c -L/usr/local/lib -o program

# 链接库
gcc main.c -lm -lpthread -o program     # 链接 libm 和 libpthread

# 静态链接特定库
gcc main.c -Wl,-Bstatic -lmylib -Wl,-Bdynamic -o program

# 静态链接所有库
gcc -static main.c -o program

# 指定共享库运行时搜索路径
gcc main.c -lmylib -Wl,-rpath,/opt/mylib -o program
```

### 6.2 链接器选项传递

```bash
# 使用 -Wl 传递选项给链接器
gcc main.o -Wl,--start-group lib1.a lib2.a -Wl,--end-group -o program

# 设置入口点
gcc main.o -Wl, -e my_entry -o program

# 生成map文件
gcc main.o -Wl,-Map=output.map -o program

# 指定链接器脚本
gcc main.o -Wl,-T linker.ld -o program
```

## 七、符号冲突与解决

### 7.1 强符号冲突

```c
/* file1.c */
int x = 10;  /* 强符号定义 */

/* file2.c */
int x = 20;  /* 错误：强符号重复定义 */
```

链接错误：
```
multiple definition of `x'
```

### 7.2 解决方案

**方案1：使用static限制作用域**
```c
/* file1.c */
static int x = 10;  /* 仅本文件可见 */

/* file2.c */
static int x = 20;  /* 不冲突 */
```

**方案2：使用extern声明**
```c
/* file1.c */
int x = 10;  /* 定义 */

/* file2.c */
extern int x;  /* 声明，不分配内存 */
```

**方案3：使用__attribute__((weak))**
```c
/* file1.c */
__attribute__((weak)) int x = 10;  /* 弱符号 */

/* file2.c */
int x = 20;  /* 强符号优先 */
```

### 7.3 未定义符号

```
undefined reference to `helper_function'
```

**原因**：
1. 函数未实现
2. 没有链接包含该函数的库
3. 链接顺序问题

**解决**：
```bash
# 确保实现了该函数
# 或者链接正确的库
gcc main.c -lmylib -o program

# 调整链接顺序（依赖的库放后面）
gcc main.o -ldep1 -ldep2 -o program
```

## 八、查看链接信息

```bash
# 查看可执行文件依赖的共享库
ldd program

# 输出示例：
# linux-vdso.so.1 (0x00007ffd...)
# libm.so.6 => /lib/x86_64-linux-gnu/libm.so.6 (0x00007f3a...)
# libc.so.6 => /lib/x86_64-linux-gnu/libc.so.6 (0x00007f3a...)
# /lib64/ld-linux-x86-64.so.2 (0x00007f3a...)

# 查看符号表
nm -C program

# 查看动态符号
nm -D program

# 查看ELF头
readelf -h program

# 查看程序头（段映射）
readelf -l program

# 查看动态段
readelf -d program

# 查看RPATH/RUNPATH
readelf -d program | grep PATH

# 查看链接器verbose输出
gcc -Wl,--verbose main.o -o program
```

## 九、链接脚本简介

### 9.1 链接脚本的作用

链接脚本控制如何将输入目标文件的段合并到输出文件中。

```ld
/* 简化的链接脚本 */
SECTIONS
{
    . = 0x400000;    /* 设置起始地址 */

    .text : {
        *(.text)     /* 合并所有目标文件的 .text 段 */
    }

    .rodata : {
        *(.rodata)
    }

    .data : {
        *(.data)
    }

    .bss : {
        *(.bss)
    }
}
```

### 9.2 使用链接脚本

```bash
# 使用自定义链接脚本
gcc main.o -T my_linker.ld -o program
```

## 十、关键要点

> **重要提示**：
> 1. 链接阶段完成符号解析和地址重定位
> 2. 强符号（函数、初始化全局变量）不能重复定义
> 3. 静态链接将库代码复制到可执行文件，动态链接在运行时加载
> 4. 使用 `ldd` 查看程序依赖的共享库
> 5. 使用 `nm` 查看符号表
> 6. `undefined reference` 错误通常是忘记链接某个库
> 7. 链接顺序很重要：被依赖的库应放在后面
> 8. `static` 关键字可以限制符号的作用域，避免冲突
> 9. GOT和PLT是动态链接的核心机制
