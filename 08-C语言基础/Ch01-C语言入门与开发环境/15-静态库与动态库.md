# 静态库与动态库

## 一、库的概念

库（Library）是预编译的代码集合，供其他程序复用。C语言中库分为两类：

| 类型 | Linux扩展名 | Windows扩展名 | 链接时机 |
|------|-----------|-------------|---------|
| 静态库 | .a（Archive） | .lib | 编译时链接 |
| 动态库 | .so（Shared Object） | .dll（Dynamic Link Library） | 运行时链接 |

## 二、静态库

### 2.1 创建静态库

```c
/* mymath.h */
#ifndef MYMATH_H
#define MYMATH_H

int add(int a, int b);
int subtract(int a, int b);
int multiply(int a, int b);

#endif
```

```c
/* mymath.c */
#include "mymath.h"

int add(int a, int b) {
    return a + b;
}

int subtract(int a, int b) {
    return a - b;
}

int multiply(int a, int b) {
    return a * b;
}
```

```bash
# 步骤1：编译为目标文件
gcc -c mymath.c -o mymath.o

# 步骤2：使用ar创建静态库
ar rcs libmymath.a mymath.o

# ar选项说明：
# r - 将文件插入归档（替换已有的）
# c - 创建归档文件
# s - 创建索引（相当于ranlib）
```

### 2.2 查看静态库内容

```bash
# 查看库中的目标文件
ar -t libmymath.a

# 查看符号表
nm libmymath.a

# 查看详细信息
ar -v -t libmymath.a

# 提取目标文件
ar -x libmymath.a
```

### 2.3 使用静态库

```c
/* main.c */
#include <stdio.h>
#include "mymath.h"

int main(void) {
    printf("3 + 4 = %d\n", add(3, 4));
    printf("10 - 3 = %d\n", subtract(10, 3));
    printf("5 * 6 = %d\n", multiply(5, 6));
    return 0;
}
```

```bash
# 方法1：直接链接
gcc main.c libmymath.a -o program

# 方法2：使用-L和-l选项
gcc main.c -L. -lmymath -o program

# 方法3：指定头文件路径
gcc -I./include main.c -L./lib -lmymath -o program

# -L 指定库文件搜索路径
# -l 指定库名（去掉lib前缀和.a后缀）
# -I 指定头文件搜索路径
```

### 2.4 静态链接过程

```
main.c → main.o ──┐
                   ├── 链接器 → program (包含所有代码)
libmymath.a ──────┘
       ↓
   mymath.o 被复制到 program 中
```

## 三、动态库

### 3.1 创建动态库

```bash
# 步骤1：编译为位置无关代码（-fPIC）
gcc -fPIC -c mymath.c -o mymath.o

# 步骤2：创建共享库
gcc -shared mymath.o -o libmymath.so

# 或一步完成
gcc -fPIC -shared mymath.c -o libmymath.so
```

**`-fPIC` 的含义**：
- PIC = Position Independent Code（位置无关代码）
- 使代码可以加载到任意内存地址
- 共享库必需的编译选项

### 3.2 使用动态库

```bash
# 编译链接
gcc main.c -L. -lmymath -o program

# 运行时需要找到共享库
# 方法1：设置LD_LIBRARY_PATH
export LD_LIBRARY_PATH=.:$LD_LIBRARY_PATH
./program

# 方法2：将库安装到系统路径
sudo cp libmymath.so /usr/local/lib/
sudo ldconfig
./program

# 方法3：使用rpath编译时指定运行时搜索路径
gcc main.c -L. -lmymath -Wl,-rpath,. -o program

# 方法4：修改/etc/ld.so.conf
echo "/path/to/lib" | sudo tee -a /etc/ld.so.conf
sudo ldconfig
```

### 3.3 查看动态库信息

```bash
# 查看动态库导出的符号
nm -D libmymath.so

# 查看动态库依赖
ldd libmymath.so

# 查看动态段
readelf -d libmymath.so

# 查看共享库版本信息
objdump -p libmymath.so | grep SONAME
```

### 3.4 动态链接过程

```
编译时：
main.c → main.o → program（含引用，不含库代码）
                    ↓
                  GOT/PLT 表项

运行时：
./program → 动态链接器(ld-linux.so) → 加载libmymath.so → 填充GOT → 执行
```

## 四、库版本管理

### 4.1 共享库命名

```
libname.so.X.Y.Z
  │      │  │ │
  │      │  │ └── 修订号（bug修复，接口不变）
  │      │  └──── 次版本号（新增接口，兼容）
  │      └─────── 主版本号（接口不兼容变更）
  └────────────── 库名

示例：
libmymath.so      → 链接名（soname的链接）
libmymath.so.1    → soname（运行时查找）
libmymath.so.1.2.3 → 真实文件名
```

### 4.2 创建带版本的共享库

```bash
# 编译带soname的共享库
gcc -fPIC -shared -Wl,-soname,libmymath.so.1 \
    mymath.c -o libmymath.so.1.2.3

# 创建符号链接
ln -s libmymath.so.1.2.3 libmymath.so.1
ln -s libmymath.so.1.2.3 libmymath.so

# 安装
cp libmymath.so.1.2.3 /usr/local/lib/
cd /usr/local/lib
ln -s libmymath.so.1.2.3 libmymath.so.1
ln -s libmymath.so.1.2.3 libmymath.so
ldconfig
```

## 五、Windows平台

### 5.1 创建静态库（.lib）

```cmd
:: 使用MSVC
cl /c mymath.c
lib mymath.obj /OUT:mymath.lib

:: 使用MinGW
gcc -c mymath.c -o mymath.o
ar rcs libmymath.a mymath.o
```

### 5.2 创建动态库（.dll）

```c
/* mymath.h（Windows版本）*/
#ifndef MYMATH_H
#define MYMATH_H

#ifdef _WIN32
    #ifdef BUILD_DLL
        #define DLL_API __declspec(dllexport)
    #else
        #define DLL_API __declspec(dllimport)
    #endif
#else
    #define DLL_API
#endif

DLL_API int add(int a, int b);
DLL_API int subtract(int a, int b);

#endif
```

```c
/* mymath.c */
#define BUILD_DLL
#include "mymath.h"

int add(int a, int b) {
    return a + b;
}

int subtract(int a, int b) {
    return a - b;
}
```

```cmd
:: 使用MinGW编译DLL
gcc -fPIC -shared -DBUILD_DLL mymath.c -o mymath.dll -Wl,--out-implib,libmymath.a

:: 使用DLL编译程序
gcc main.c -L. -lmymath -o program.exe

:: 运行时需要DLL在PATH中或当前目录
```

```cmd
:: 使用MSVC编译DLL
cl /LD mymath.c /Fe:mymath.dll

:: 使用LIB生成导入库
lib /DEF:mymath.def /OUT:mymath.lib

:: 使用DLL编译程序
cl main.c mymath.lib
```

### 5.3 查看Windows DLL信息

```cmd
:: 使用dumpbin（Visual Studio自带）
dumpbin /exports mymath.dll
dumpbin /imports program.exe
dumpbin /dependents program.exe
```

## 六、静态库 vs 动态库

### 6.1 对比总结

| 特性 | 静态库 | 动态库 |
|------|--------|--------|
| 文件扩展名 | .a/.lib | .so/.dll |
| 链接时机 | 编译时 | 运行时 |
| 文件大小 | 大（包含库代码） | 小（不含库代码） |
| 运行依赖 | 无 | 需要库文件 |
| 内存使用 | 每个进程独立拷贝 | 多进程共享 |
| 更新方式 | 需重新编译 | 替换库文件即可 |
| 加载速度 | 快 | 稍慢 |
| 版本管理 | 简单 | 需要版本控制 |

### 6.2 使用场景

**静态库适用于**：
- 独立分发的程序
- 对启动速度要求高
- 运行环境不确定
- 小型工具程序

**动态库适用于**：
- 多个程序共享的库
- 需要热更新
- 系统级库（如libc）
- 插件系统
- 减小可执行文件体积

## 七、CMake中使用库

```cmake
# 创建静态库
add_library(mymath STATIC mymath.c mymath.h)
target_include_directories(mymath PUBLIC ${CMAKE_SOURCE_DIR}/include)

# 创建动态库
add_library(mymath_shared SHARED mymath.c mymath.h)
target_include_directories(mymath_shared PUBLIC ${CMAKE_SOURCE_DIR}/include)

# 使用库
add_executable(main main.c)
target_link_libraries(main mymath)          # 链接静态库
# 或
target_link_libraries(main mymath_shared)   # 链接动态库
```

## 八、实用工具

### 8.1 Linux工具

```bash
# 查看程序依赖的共享库
ldd program

# 更新共享库缓存
sudo ldconfig

# 查看共享库缓存
ldconfig -p

# 查找共享库

```

### 8.2 nm工具高级用法

```bash
# 查看库中所有符号
nm -g libmymath.a    # 只看全局符号
nm -u libmymath.a    # 只看未定义符号
nm -C libmymath.a    # demangle符号名

# 比较两个库的符号差异
nm -g libv1.a | sort > v1.txt
nm -g libv2.a | sort > v2.txt
diff v1.txt v2.txt
```

## 九、常见问题

### 9.1 找不到共享库

```
./program: error while loading shared libraries: libmymath.so:
cannot open shared object file: No such file or directory
```

**解决方法**：
```bash
# 方法1：设置环境变量
export LD_LIBRARY_PATH=.:$LD_LIBRARY_PATH

# 方法2：使用rpath
gcc main.c -L. -lmymath -Wl,-rpath,. -o program

# 方法3：安装到系统路径
sudo cp libmymath.so /usr/local/lib/
sudo ldconfig
```

### 9.2 符号冲突

```
multiple definition of `add'
```

**解决方法**：
```bash
# 使用static限制作用域
static int add(int a, int b) { ... }

# 或者检查是否有重复的目标文件被链接
```

### 9.3 版本兼容

```bash
# 检查soname
readelf -d libmymath.so | grep SONAME

# 确保链接时的soname与运行时的库版本匹配
```

## 十、关键要点

> **重要提示**：
> 1. 静态库用 `ar` 创建，动态库用 `gcc -shared` 创建
> 2. 动态库编译需要 `-fPIC` 选项生成位置无关代码
> 3. `-L` 指定库搜索路径，`-l` 指定库名
> 4. 动态库运行时需要 `LD_LIBRARY_PATH` 或 `rpath` 找到库
> 5. 使用 `ldd` 查看程序依赖的共享库
> 6. 使用 `nm` 查看库中的符号
> 7. 静态库代码被复制到可执行文件，动态库运行时加载
> 8. 动态库版本管理使用soname和符号链接
> 9. Windows上动态库需要 `__declspec(dllexport/dllimport)` 声明
> 10. 使用 `ldconfig` 更新Linux共享库缓存
