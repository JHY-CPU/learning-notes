# 编译与汇编阶段

## 一、编译阶段概述

编译阶段将预处理后的C源代码转换为汇编代码，包括词法分析、语法分析、语义分析、中间代码生成和优化等步骤。

```
预处理后的代码 (.i)
    ↓
词法分析（Tokenization）
    ↓
语法分析（Parsing）→ 语法树（AST）
    ↓
语义分析（Semantic Analysis）
    ↓
中间代码生成（IR Generation）
    ↓
优化（Optimization）
    ↓
目标代码生成（Code Generation）
    ↓
汇编代码 (.s)
```

## 二、词法分析

### 2.1 概念

词法分析器（Lexer/Scanner）将源代码字符流分解为有意义的标记（Tokens）。

### 2.2 示例

```c
int main(void) {
    int x = 42 + 3;
    return x;
}
```

词法分析结果：

```
标记序列:
[KEYWORD: int]       [IDENTIFIER: main]
[LPAREN: (]          [KEYWORD: void]
[RPAREN: )]          [LBRACE: {]
[KEYWORD: int]       [IDENTIFIER: x]
[ASSIGN: =]          [INTEGER: 42]
[PLUS: +]            [INTEGER: 3]
[SEMICOLON: ;]       [KEYWORD: return]
[IDENTIFIER: x]      [SEMICOLON: ;]
[RBRACE: }]
[EOF]
```

### 2.3 标记类型

| 类型 | 示例 |
|------|------|
| 关键字 | `int`, `if`, `while`, `return` |
| 标识符 | `main`, `x`, `my_function` |
| 整数常量 | `42`, `0xFF`, `017` |
| 浮点常量 | `3.14`, `1.5e10` |
| 字符常量 | `'A'`, `'\n'` |
| 字符串常量 | `"hello"` |
| 运算符 | `+`, `-`, `*`, `==`, `&&` |
| 分隔符 | `(`, `)`, `{`, `}`, `;`, `,` |

### 2.4 关键字列表（C99，共37个）

```
auto        break       case        char        const
continue    default     do          double      else
enum        extern      float       for         goto
if          inline      int         long        register
restrict    return      short       signed      sizeof
static      struct      switch      typedef     union
unsigned    void        volatile    while
_Bool       _Complex    _Imaginary
```

## 三、语法分析

### 3.1 概念

语法分析器（Parser）根据语言的语法规则，将标记序列组织成语法树（Abstract Syntax Tree, AST）。

### 3.2 简化的语法规则

```
program         → declaration_list
declaration_list → declaration_list declaration | declaration
declaration     → var_declaration | func_declaration
var_declaration → type_specifier IDENTIFIER ';'
func_declaration→ type_specifier IDENTIFIER '(' params ')' compound_stmt
compound_stmt   → '{' local_declarations statement_list '}'
statement_list  → statement_list statement | statement
statement       → expression_stmt | compound_stmt | selection_stmt | iteration_stmt
expression_stmt → expression ';' | ';'
```

### 3.3 AST示例

```c
int main(void) {
    int x = 42;
    if (x > 0) {
        printf("positive\n");
    }
    return 0;
}
```

简化AST：
```
Program
├── Function: main (returns int)
│   ├── Parameters: void
│   └── Body:
│       ├── Declaration: int x = 42
│       ├── IfStatement
│       │   ├── Condition: BinaryOp(>)
│       │   │   ├── Identifier: x
│       │   │   └── Integer: 0
│       │   └── ThenBranch:
│       │       └── FunctionCall: printf
│       │           └── String: "positive\n"
│       └── ReturnStatement
│           └── Integer: 0
```

### 3.4 常见语法错误

```c
/* 缺少分号 */
int x = 10
int y = 20;

/* 缺少括号 */
if x > 0 {
    printf("positive");
}

/* 缺少花括号 */
for (int i = 0; i < 10; i++)
    printf("%d\n", i);
    printf("done");  /* 这行不在循环内！*/
```

## 四、语义分析

### 4.1 概念

语义分析检查程序的逻辑正确性，包括类型检查、作用域分析、声明检查等。

### 4.2 类型检查

```c
/* 类型不匹配 */
int x = 3.14;     /* double → int 转换，可能警告 */
int *p = x;       /* int → int* 错误 */

/* 类型兼容性 */
double d = 42;    /* int → double 隐式转换，允许 */
```

### 4.3 作用域分析

```c
int global = 100;  /* 文件作用域 */

void func(void) {
    int local = 50;  /* 函数作用域 */
    {
        int inner = 10;  /* 块作用域 */
    }
    // inner 在此处不可见
}
// local 在此处不可见
```

### 4.4 常见语义错误

```c
/* 未声明的变量 */
int main(void) {
    x = 10;  /* 错误：x 未声明 */
    return 0;
}

/* 类型不匹配的赋值 */
int *p;
p = 42;  /* 错误：int 赋给 int* */

/* 函数参数不匹配 */
int add(int a, int b);
int result = add(1);  /* 错误：参数不足 */
```

## 五、中间代码与优化

### 5.1 中间表示（IR）

编译器通常生成中间代码（如三地址码）来进行优化：

```c
/* 源代码 */
int x = a + b * c;

/* 三地址码 */
t1 = b * c
t2 = a + t1
x = t2
```

### 5.2 常见优化

```c
/* 常量折叠 */
int x = 3 + 4;     /* 优化为: int x = 7; */

/* 常量传播 */
int x = 10;
int y = x + 5;     /* 优化为: int y = 15; */

/* 死代码消除 */
int x = 10;
x = 20;            /* x = 10 被消除 */
return x;

/* 强度削弱 */
int y = x * 2;     /* 可能优化为: int y = x << 1; */

/* 循环优化 */
for (int i = 0; i < n * 2; i++) { }
/* n * 2 可能在循环外计算 */
```

## 六、目标代码生成

### 6.1 从IR到汇编

```c
/* C源码 */
int add(int a, int b) {
    return a + b;
}
```

```asm
# x86-64 汇编（AT&T语法）
add:
    pushq   %rbp
    movq    %rsp, %rbp
    movl    %edi, -4(%rbp)    # 参数a
    movl    %esi, -8(%rbp)    # 参数b
    movl    -4(%rbp), %eax
    addl    -8(%rbp), %eax
    popq    %rbp
    ret
```

### 6.2 寄存器分配

编译器需要决定哪些变量放在寄存器中，哪些放在内存中：

```c
/* 编译器可能的寄存器分配方案 */
int compute(int a, int b, int c) {
    int t1 = a + b;
    int t2 = t1 * c;
    return t2;
}

/*
 * 寄存器分配（x86-64）：
 * a → %edi
 * b → %esi
 * c → %edx
 * t1 → %eax (重用)
 * 返回值 → %eax
 */
```

### 6.3 指令选择

同一个操作可以用不同的指令实现：

```c
int y = x * 8;
```
```asm
# 方案1：乘法指令
imull $8, %eax

# 方案2：移位指令（更快）
sall $3, %eax
```

## 七、汇编阶段

### 7.1 汇编器的工作

汇编器（as）将汇编代码转换为机器码，生成目标文件（.o）。

```bash
# 从汇编代码生成目标文件
gcc -c source.s -o source.o

# 使用独立的汇编器
as source.s -o source.o
```

### 7.2 目标文件结构

```
目标文件 (.o) 结构：
┌──────────────────┐
│     ELF头/PE头    │  文件类型、架构等元信息
├──────────────────┤
│     .text         │  代码段：机器指令
├──────────────────┤
│     .data         │  数据段：已初始化的全局/静态变量
├──────────────────┤
│     .rodata       │  只读数据段：常量
├──────────────────┤
│     .bss          │  BSS段：未初始化的全局/静态变量
├──────────────────┤
│     .symtab       │  符号表：函数和变量名
├──────────────────┤
│     .strtab       │  字符串表：符号名字符串
├──────────────────┤
│     .rel.text     │  代码段重定位信息
├──────────────────┤
│     .rel.data     │  数据段重定位信息
├──────────────────┤
│     .debug        │  调试信息（如果有-g）
├──────────────────┤
│     .line         │  源码行号映射
└──────────────────┘
```

### 7.3 查看目标文件

```bash
# 查看段信息
objdump -h source.o

# 查看反汇编
objdump -d source.o

# 查看符号表
nm source.o

# 查看ELF头信息
readelf -h source.o

# 查看所有段
readelf -S source.o

# 查看重定位信息
objdump -r source.o
```

### 7.4 符号类型说明

```
nm命令输出的符号类型：
T/t - .text段中的符号（代码）
D/d - .data段中的符号（已初始化数据）
B/b - .bss段中的符号（未初始化数据）
R/r - .rodata段中的符号（只读数据）
U   - 未定义符号（需要在链接时解析）
W/w - 弱符号
C   - 公共符号

大写字母 = 全局符号
小写字母 = 局部符号
```

## 八、GCC编译选项

```bash
# 只预处理
gcc -E source.c -o source.i

# 只编译到汇编
gcc -S source.c -o source.s
gcc -S -O2 source.c -o source.s      # 带优化的汇编
gcc -S -masm=intel source.c -o source.s  # Intel语法

# 编译到目标文件（不链接）
gcc -c source.c -o source.o

# 查看详细的编译过程
gcc -v source.c -o source

# 保存中间文件
gcc -save-temps source.c -o source
# 生成: source.i (预处理), source.s (汇编), source.o (目标文件)
```

## 九、编译优化的层次

```bash
# 不同优化级别对生成代码的影响

gcc -S -O0 source.c -o O0.s   # 无优化，忠实于源码
gcc -S -O1 source.c -o O1.s   # 基本优化
gcc -S -O2 source.c -o O2.s   # 推荐优化
gcc -S -O3 source.c -o O3.s   # 最大优化（可能增大代码体积）
gcc -S -Os source.c -o Os.s   # 优化代码大小
```

### 优化示例

```c
/* 源代码 */
int sum(int n) {
    int s = 0;
    for (int i = 0; i < n; i++) {
        s += i;
    }
    return s;
}
```

```asm
# -O0 版本（未优化，较冗长）
sum:
    pushq   %rbp
    movq    %rsp, %rbp
    movl    %edi, -20(%rbp)
    movl    $0, -4(%rbp)
    movl    $0, -8(%rbp)
    jmp     .L2
.L3:
    movl    -8(%rbp), %eax
    addl    %eax, -4(%rbp)
    addl    $1, -8(%rbp)
.L2:
    movl    -8(%rbp), %eax
    cmpl    -20(%rbp), %eax
    jl      .L3
    movl    -4(%rbp), %eax
    popq    %rbp
    ret

# -O2 版本（优化后，编译器识别为等差数列求和）
sum:
    testl   %edi, %edi
    jle     .L4
    leal    -1(%rdi), %eax
    imull   %edi, %eax
    shrl    $31, %eax
    addl    %edi, %eax
    sarl    %eax
    ret
.L4:
    xorl    %eax, %eax
    ret
```

## 十、关键要点

> **重要提示**：
> 1. 编译过程包括：词法分析、语法分析、语义分析、代码优化、代码生成
> 2. 词法分析将源代码分解为标记（tokens）
> 3. 语法分析构建抽象语法树（AST）
> 4. 语义分析检查类型、作用域等逻辑正确性
> 5. 中间代码和优化是提升程序性能的关键
> 6. 汇编器将汇编代码转换为目标文件（.o）
> 7. 使用 `gcc -S` 可以查看汇编代码
> 8. 使用 `gcc -save-temps` 可以保留所有中间文件
> 9. 不同的优化级别会显著影响生成的汇编代码
