# 目标文件与可执行文件

## 一、文件格式概述

### 1.1 主要格式

| 格式 | 平台 | 扩展名 |
|------|------|--------|
| ELF（Executable and Linkable Format） | Linux/Unix | .o, .so, 无扩展名 |
| PE（Portable Executable） | Windows | .obj, .dll, .exe |
| Mach-O | macOS | .o, .dylib, 无扩展名 |

### 1.2 文件类型

```
目标文件类型：
┌──────────────────┬─────────────────────────────────────┐
│ 可重定位文件      │ .o/.obj，编译产生的中间文件           │
├──────────────────┼─────────────────────────────────────┤
│ 可执行文件       │ 无扩展名/.exe，可以直接运行           │
├──────────────────┼─────────────────────────────────────┤
│ 共享目标文件      │ .so/.dll，动态链接库                │
├──────────────────┼─────────────────────────────────────┤
│ 核心转储文件      │ core dump，程序崩溃时的内存转储       │
└──────────────────┴─────────────────────────────────────┘
```

## 二、ELF文件格式

### 2.1 ELF文件结构

```
┌──────────────────────────────────┐
│         ELF Header               │  文件类型、架构、入口地址等
├──────────────────────────────────┤
│     Program Header Table         │  描述段（Segment）信息
├──────────────────────────────────┤
│                                  │
│         Section 1 (.text)        │  代码段
│         Section 2 (.rodata)      │  只读数据段
│         Section 3 (.data)        │  已初始化数据段
│         Section 4 (.bss)         │  未初始化数据段
│         Section 5 (.symtab)      │  符号表
│         Section 6 (.strtab)      │  字符串表
│         Section 7 (.shstrtab)    │  段名字符串表
│         Section 8 (.rel.text)    │  代码段重定位信息
│         Section 9 (.debug)       │  调试信息
│         ...                      │
│                                  │
├──────────────────────────────────┤
│    Section Header Table          │  描述段（Section）信息
└──────────────────────────────────┘
```

### 2.2 查看ELF文件信息

```bash
# 查看ELF头
readelf -h program

# 查看段头（Section Headers）
readelf -S program

# 查看程序头（Program Headers，用于加载）
readelf -l program

# 查看符号表
readelf -s program

# 查看动态符号
readelf --dyn-syms program

# 查看重定位信息
readelf -r program.o

# 查看动态段
readelf -d program

# 查看所有信息
readelf -a program
```

## 三、重要段（Sections）

### 3.1 .text 段（代码段）

存放编译后的机器指令。

```bash
# 查看代码段
objdump -d program
readelf -x .text program
```

特点：
- 只读（通常标记为 RX：可读可执行）
- 所有函数的机器码都在这里
- 多个进程可以共享同一份代码段

### 3.2 .data 段（数据段）

存放已初始化的全局变量和静态变量。

```c
int global_init = 42;         /* 在 .data 段 */
static int static_init = 100; /* 在 .data 段 */

int main(void) {
    return 0;
}
```

```bash
# 查看数据段
objdump -s -j .data program
readelf -x .data program
```

特点：
- 可读可写（RW）
- 大小在编译时确定
- 初始值存储在文件中

### 3.3 .rodata 段（只读数据段）

存放常量和字符串字面量。

```c
const int MAX_SIZE = 1024;           /* 在 .rodata 段 */
const char *msg = "Hello, World!";   /* msg指针在 .data，字符串在 .rodata */

int main(void) {
    printf("This string is in .rodata\n");
    return 0;
}
```

```bash
# 查看只读数据段
objdump -s -j .rodata program
```

### 3.4 .bss 段（未初始化数据段）

存放未初始化或初始化为0的全局变量和静态变量。

```c
int global_uninit;            /* 在 .bss 段 */
int global_zero = 0;          /* 在 .bss 段（某些编译器）*/
static int static_uninit;     /* 在 .bss 段 */
```

特点：
- 不占用文件空间（只记录大小）
- 加载时由系统初始化为0
- 可读可写（RW）

### 3.5 各段对比

```
源代码对应关系：
┌────────────────────────────┬─────────────────┐
│ C代码                      │ 段               │
├────────────────────────────┼─────────────────┤
│ 函数定义                   │ .text            │
│ const常量                  │ .rodata          │
│ 字符串字面量               │ .rodata          │
│ 已初始化的全局/静态变量     │ .data            │
│ 未初始化的全局/静态变量     │ .bss             │
│ 局部变量                   │ 栈（运行时分配）  │
│ malloc分配的内存           │ 堆（运行时分配）  │
└────────────────────────────┴─────────────────┘
```

## 四、符号表

### 4.1 符号表内容

符号表记录了函数和变量的名称、地址、类型等信息。

```bash
# 查看符号表
nm program

# 查看动态符号
nm -D program

# demangle C++符号
nm -C program

# 查看目标文件的符号
nm -a program.o
```

### 4.2 nm命令输出格式

```
地址        类型  符号名
0000000000401106 T main
0000000000401130 T helper
0000000000404040 D global_init
0000000000404050 B global_uninit
                 U printf        (未定义，在动态库中)
                 U __libc_start_main (未定义)
```

符号类型：
- `T` / `t`：.text段（代码）
- `D` / `d`：.data段（已初始化数据）
- `B` / `b`：.bss段（未初始化数据）
- `R` / `r`：.rodata段（只读数据）
- `U`：未定义符号
- `W` / `w`：弱符号
- `C`：公共符号

### 4.3 符号可见性

```c
/* 全局符号（外部可见）*/
int global_var = 42;
void public_func(void) { }

/* 局部符号（仅本文件可见）*/
static int local_var = 10;
static void private_func(void) { }
```

```bash
# nm输出中：
0000000000404040 D global_var      # 大写D，全局
0000000000404044 d local_var       # 小写d，局部
0000000000401106 T public_func     # 大写T，全局
0000000000401118 t private_func    # 小写t，局部
```

## 五、PE格式（Windows）

### 5.1 PE文件结构

```
┌──────────────────────────┐
│      DOS Header (MZ)     │
├──────────────────────────┤
│      DOS Stub            │
├──────────────────────────┤
│      PE Signature        │
├──────────────────────────┤
│      COFF File Header    │
├──────────────────────────┤
│      Optional Header     │
├──────────────────────────┤
│      Section Headers     │
├──────────────────────────┤
│      .text (代码)        │
│      .rdata (只读数据)    │
│      .data (数据)        │
│      .bss (未初始化)     │
│      .rsrc (资源)        │
│      ...                 │
└──────────────────────────┘
```

### 5.2 查看PE文件（Windows工具）

```cmd
:: 使用dumpbin（Visual Studio自带）
dumpbin /headers program.exe
dumpbin /disasm program.exe
dumpbin /symbols program.obj
dumpbin /exports mylib.dll
dumpbin /imports program.exe

:: 使用 objdump（MinGW）
objdump -x program.exe
objdump -d program.exe
nm program.exe
```

## 六、段与Segment的区别

```
Section（节）：链接时的概念
  - .text, .data, .bss, .rodata 等
  - 链接器处理的单位

Segment（段）：加载时的概念
  - LOAD段：需要加载到内存
  - 多个Section可以合并到一个Segment
  - 加载器处理的单位

对应关系：
  ┌─────────────┐    ┌──────────────┐
  │   .text     │    │              │
  │   .rodata   │ ─→ │  LOAD (RX)   │  可读可执行
  └─────────────┘    │              │
                     └──────────────┘
  ┌─────────────┐    ┌──────────────┐
  │   .data     │    │              │
  │   .bss      │ ─→ │  LOAD (RW)   │  可读可写
  └─────────────┘    │              │
                     └──────────────┘
```

```bash
# 查看Section
readelf -S program

# 查看Segment
readelf -l program
```

## 七、实际操作

### 7.1 创建目标文件

```bash
# 编译生成目标文件
gcc -c main.c -o main.o
gcc -c utils.c -o utils.o

# 查看目标文件
file main.o        # 查看文件类型
readelf -h main.o  # 查看ELF头
readelf -S main.o  # 查看所有段
nm main.o          # 查看符号
objdump -d main.o  # 反汇编
```

### 7.2 对比目标文件和可执行文件

```bash
# 编译
gcc -c main.c -o main.o
gcc main.o -o program

# 对比
echo "=== 目标文件 ==="
readelf -S main.o
nm main.o

echo "=== 可执行文件 ==="
readelf -S program
nm program

# 注意：
# - 目标文件有未解析符号（U类型）
# - 可执行文件所有符号都已解析
# - 可执行文件有程序头（Program Header）
```

### 7.3 查看特定段的内容

```bash
# 查看.text段的十六进制内容
objdump -s -j .text program

# 查看.data段
objdump -s -j .data program

# 查看字符串
strings program

# 提取特定段
objcopy -O binary -j .text program text.bin
objcopy -O binary -j .data program data.bin
```

## 八、地址空间布局

### 8.1 Linux进程地址空间

```
高地址
┌────────────────────────────┐
│       内核空间              │  用户态不可访问
├────────────────────────────┤
│       栈（Stack）           │  局部变量、函数调用
│       ↓ 向下增长            │
│                            │
│       ↑ 向上增长            │
│       堆（Heap）            │  malloc分配
├────────────────────────────┤
│       .bss                 │  未初始化全局变量
├────────────────────────────┤
│       .data                │  已初始化全局变量
├────────────────────────────┤
│       .rodata              │  常量
├────────────────────────────┤
│       .text                │  程序代码
├────────────────────────────┤
│       保留区域              │
低地址
```

### 8.2 查看进程地址空间

```bash
# 运行程序后查看
cat /proc/<PID>/maps

# 或使用 pmap
pmap <PID>
```

## 九、关键要点

> **重要提示**：
> 1. ELF是Linux/Unix的主要目标文件格式，PE是Windows的格式
> 2. `.text`段存放代码，`.data`段存放已初始化全局变量
> 3. `.bss`段存放未初始化全局变量，不占用文件空间
> 4. `.rodata`段存放常量和字符串字面量
> 5. 使用 `readelf`（Linux）和 `dumpbin`（Windows）查看文件信息
> 6. 使用 `nm` 查看符号表，`objdump` 反汇编
> 7. 符号类型中大写表示全局，小写表示局部，`U`表示未定义
> 8. Section是链接时的概念，Segment是加载时的概念
> 9. `static`关键字使符号变为局部，仅本文件可见
