# 代码风格与规范

## 一、代码风格的重要性

良好的代码风格能提高代码的可读性、可维护性，并减少错误。团队开发中统一的代码风格尤为重要。

```
好的代码风格：
- 减少理解代码的时间
- 降低维护成本
- 减少bug产生
- 便于团队协作
```

## 二、命名规范

### 2.1 变量命名

```c
/* 推荐：小写字母 + 下划线（snake_case）*/
int student_count;
double average_score;
char file_name[256];
int is_valid;
int has_data;

/* 不推荐：匈牙利命名法（C语言中不常用）*/
int iStudentCount;
double dAverageScore;
char szFileName[256];

/* 不推荐：无分隔符 */
int studentcount;
double averagescore;
```

### 2.2 函数命名

```c
/* 推荐：动词_名词，snake_case */
int calculate_sum(int *arr, int n);
void print_error(const char *msg);
char *read_file(const char *path);
int is_prime(int n);
void init_database(void);
void cleanup_resources(void);

/* 不推荐 */
int Sum(int *arr, int n);       /* 首字母大写，不一致 */
int calculateSum();             /* camelCase，不推荐用于C */
```

### 2.3 宏和常量命名

```c
/* 宏：全大写 + 下划线 */
#define MAX_BUFFER_SIZE 1024
#define PI 3.14159265
#define DEBUG_MODE 1
#define ARRAY_SIZE(arr) (sizeof(arr) / sizeof((arr)[0]))

/* 枚举常量：全大写或前缀_全大写 */
typedef enum {
    COLOR_RED,
    COLOR_GREEN,
    COLOR_BLUE
} Color;

/* const常量：全大写或首字母大写 */
const int MAX_STUDENTS = 100;
const double SpeedOfLight = 299792458.0;
```

### 2.4 类型命名

```c
/* 结构体/联合体：首字母大写或全小写+下划线 */
typedef struct {
    int x;
    int y;
} Point;

typedef struct student_info {
    char name[50];
    int age;
    double gpa;
} StudentInfo;

/* 类型名后缀 */
typedef struct Config Config_t;      /* _t 后缀 */
typedef unsigned int uint32;         /* 简短类型 */
```

### 2.5 命名规则总结

| 元素 | 命名风格 | 示例 |
|------|---------|------|
| 变量 | snake_case | `student_count` |
| 函数 | snake_case | `calculate_sum()` |
| 宏 | UPPER_SNAKE | `MAX_BUFFER_SIZE` |
| 枚举值 | UPPER_SNAKE | `COLOR_RED` |
| 结构体 | 首字母大写 | `StudentInfo` |
| 类型 | _t 后缀 | `size_t` |
| 文件名 | snake_case | `utils.c`, `math_ops.h` |

## 三、缩进与格式

### 3.1 缩进风格

```c
/* 风格1：K&R风格（推荐）*/
int main(void) {
    if (condition) {
        do_something();
    } else {
        do_other();
    }
}

/* 风格2：Allman风格 */
int main(void)
{
    if (condition)
    {
        do_something();
    }
    else
    {
        do_other();
    }
}

/* 风格3：GNU风格 */
int
main (void)
{
  if (condition)
    {
      do_something();
    }
}
```

### 3.2 推荐的格式规则

```c
/* 1. 使用4个空格缩进（或1个Tab）*/
void function(void) {
    int x = 10;          /* 一级缩进 */
    if (x > 0) {
        printf("x > 0"); /* 二级缩进 */
    }
}

/* 2. 花括号风格 */
if (condition) {    /* 花括号在同一行 */
    /* code */
} else {            /* else 在同一行 */
    /* code */
}

/* 3. 运算符两侧加空格 */
int result = a + b;     /* 推荐 */
int result=a+b;         /* 不推荐 */

/* 4. 逗号后加空格 */
printf("%d, %d, %d", a, b, c);  /* 推荐 */
printf("%d,%d,%d",a,b,c);       /* 不推荐 */

/* 5. 指针声明的 * 靠近变量名 */
int *ptr;           /* 推荐 */
int* ptr;           /* 不推荐 */
int * ptr;          /* 也不推荐 */

/* 6. 函数参数 */
void function(int a, int b, int c) {
    /* code */
}

/* 7. 长行拆分（建议80-120列）*/
int result = very_long_function_name(
    argument1, argument2,
    argument3, argument4);

/* 8. 空行分隔逻辑块 */
void process_data(void) {
    /* 读取数据 */
    read_input();
    process_input();

    /* 处理数据 */
    validate_data();
    transform_data();

    /* 输出结果 */
    generate_output();
}
```

### 3.3 对齐

```c
/* 变量声明对齐 */
int         count   = 0;
double      average = 0.0;
char        name[50] = "unknown";

/* 结构体初始化对齐（Linux内核风格）*/
struct config cfg = {
    .name      = "default",
    .port      = 8080,
    .timeout   = 30,
    .max_conns = 100,
};

/* 枚举定义对齐 */
typedef enum {
    STATE_IDLE    = 0,
    STATE_RUNNING = 1,
    STATE_PAUSED  = 2,
    STATE_STOPPED = 3,
} State;
```

## 四、注释规范

### 4.1 文件头注释

```c
/*
 * file: utils.c
 * desc: 工具函数实现
 * auth: 张三
 * date: 2024-01-15
 *
 * 本文件包含常用的工具函数实现，包括字符串处理、
 * 数学计算、文件操作等。
 */
```

### 4.2 函数注释

```c
/**
 * @brief  计算数组元素的平均值
 * @param  arr    输入数组
 * @param  length 数组长度
 * @return 平均值；如果数组为空返回0.0
 *
 * @note   数组长度必须大于0
 * @see    calculate_sum()
 *
 * @example
 *   double data[] = {1.0, 2.0, 3.0};
 *   double avg = calculate_average(data, 3);  // avg = 2.0
 */
double calculate_average(const double arr[], int length) {
    if (length <= 0) return 0.0;
    double sum = 0.0;
    for (int i = 0; i < length; i++) {
        sum += arr[i];
    }
    return sum / length;
}
```

### 4.3 行内注释

```c
/* 推荐：注释在代码上方或右侧 */
int max_retries = 3;         /* 最大重试次数 */
int timeout_ms  = 5000;      /* 超时时间（毫秒）*/

/* 不推荐：过多或显而易见的注释 */
int x = 0;   /* 将x设置为0 */    /* 无意义的注释 */

/* 复杂逻辑的注释应该解释为什么，而不是做什么 */
/* 使用快速排序而非冒泡排序，因为数据量大时性能更好 */
qsort(arr, n, sizeof(int), compare);
```

### 4.4 TODO和FIXME

```c
/* TODO: 需要实现错误处理 */
/* FIXME: 在多线程环境下可能有竞态条件 */
/* HACK: 临时解决方案，需要重构 */
/* XXX: 这里有个已知问题 */
/* NOTE: 这个函数不是线程安全的 */
```

## 五、函数设计

### 5.1 函数长度

```c
/* 推荐：函数保持简短，一般不超过50行 */

/* 好的设计：职责单一 */
static int validate_input(const char *input) {
    if (input == NULL) return -1;
    if (strlen(input) == 0) return -1;
    return 0;
}

static int parse_input(const char *input, Data *data) {
    /* 解析逻辑 */
    return 0;
}

static int process_data(const Data *data, Result *result) {
    /* 处理逻辑 */
    return 0;
}

/* 避免：过长的函数 */
void do_everything(void) {
    /* 200行代码... 不推荐 */
}
```

### 5.2 参数数量

```c
/* 推荐：参数不超过4-5个 */

/* 参数过多时使用结构体 */
typedef struct {
    int x;
    int y;
    int width;
    int height;
    Color color;
    int border_width;
} DrawConfig;

void draw_rectangle(const DrawConfig *cfg) {
    /* 使用 cfg->x, cfg->y 等 */
}
```

### 5.3 返回值和错误处理

```c
/* 方式1：返回0成功，-1失败 */
int init_database(const char *path) {
    if (path == NULL) return -1;
    /* ... */
    return 0;  /* 成功 */
}

/* 方式2：返回指针，NULL表示失败 */
char *read_file(const char *path) {
    if (path == NULL) return NULL;
    /* ... */
    return buffer;  /* 成功 */
}

/* 方式3：错误码枚举 */
typedef enum {
    ERR_OK = 0,
    ERR_INVALID_PARAM,
    ERR_OUT_OF_MEMORY,
    ERR_FILE_NOT_FOUND,
    ERR_PERMISSION_DENIED,
} ErrorCode;

ErrorCode open_config(const char *path, Config **cfg) {
    if (path == NULL) return ERR_INVALID_PARAM;
    /* ... */
    return ERR_OK;
}
```

## 六、头文件设计

### 6.1 头文件保护

```c
/* 方式1：传统方式 */
#ifndef MYMODULE_H
#define MYMODULE_H

/* 头文件内容 */

#endif /* MYMODULE_H */

/* 方式2：#pragma once（非标准但广泛支持）*/
#pragma once

/* 头文件内容 */
```

### 6.2 头文件组织

```c
/* mymodule.h */
#ifndef MYMODULE_H
#define MYMODULE_H

/* 1. 标准库包含 */
#include <stddef.h>
#include <stdint.h>

/* 2. 第三方库包含 */
/* #include <sqlite3.h> */

/* 3. 项目其他头文件 */
/* #include "common.h" */

/* 4. 本模块声明 */

/* 前置声明 */
typedef struct MyStruct MyStruct;

/* 常量定义 */
#define MYMODULE_VERSION "1.0.0"

/* 类型定义 */
typedef enum {
    MYMODULE_OK,
    MYMODULE_ERROR,
} MyModuleStatus;

struct MyStruct {
    int value;
    char name[64];
};

/* 函数声明 */
MyModuleStatus mymodule_init(void);
void mymodule_cleanup(void);
int mymodule_process(MyStruct *obj);

#endif /* MYMODULE_H */
```

## 七、常用代码规范

### 7.1 Google C Style（摘录）

```c
/* 文件命名：小写 + 下划线 */
/* my_useful_class.c */

/* 头文件顺序 */
#include "myproject/public_header.h"   /* 本项目头文件 */
#include <stdio.h>                      /* C系统头文件 */
#include <string.h>
#include <vector>                        /* C++标准库 */
#include "myproject/private_header.h"   /* 本项目其他头文件 */

/* 命名空间（C++用）*/
/* 类名：首字母大写 MyClass */
/* 函数名：首字母小写 my_function() */
/* 变量名：小写 + 下划线 my_variable */
/* 常量：k + 首字母大写 kMaxEntries */
```

### 7.2 Linux内核编码风格

```c
/*
 * Linux内核编码风格要点：
 * - 缩进使用Tab（8字符宽）
 * - 花括号：函数另起一行，其他在同一行
 * - 行宽80列
 * - 函数长度尽量短
 */

/* 函数定义：左花括号另起一行 */
int
kernel_function(int arg)
{
	if (arg < 0)
		return -EINVAL;

	/* 代码块 */
	do_something();

	return 0;
}
```

### 7.3 MISRA C（嵌入式）

MISRA C是嵌入式系统中广泛使用的编码标准：
- 禁止隐式类型转换
- 限制指针的使用
- 禁止动态内存分配
- 所有函数必须有原型声明
- 禁止递归

## 八、代码格式化工具

### 8.1 clang-format

```bash
# 安装
sudo apt install clang-format

# 格式化文件
clang-format -i source.c

# 使用指定风格
clang-format -style=Google -i source.c
clang-format -style=LLVM -i source.c
clang-format -style=Mozilla -i source.c
```

**配置文件 `.clang-format`**：
```yaml
Language: C
BasedOnStyle: Google
IndentWidth: 4
TabWidth: 4
UseTab: Never
ColumnLimit: 100
AllowShortFunctionsOnASingleLine: None
AllowShortIfStatementsOnASingleLine: false
BreakBeforeBraces: Linux
IndentCaseLabels: true
SortIncludes: true
```

### 8.2 indent（GNU）

```bash
# GNU indent格式化
indent -kr -i4 -l120 source.c
```

### 8.3 astyle

```bash
# Artistic Style格式化
astyle --style=kr --indent=spaces=4 source.c
```

## 九、关键要点

> **重要提示**：
> 1. C语言推荐使用 `snake_case` 命名风格
> 2. 宏和常量使用 `UPPER_SNAKE_CASE`
> 3. 函数命名应体现动作（动词_名词）
> 4. 使用4空格缩进，保持行宽在80-120列
> 5. 指针声明的 `*` 靠近变量名：`int *ptr;`
> 6. 注释解释"为什么"而非"做什么"
> 7. 函数保持简短，参数不超过4-5个
> 8. 使用 `#ifndef` / `#define` / `#endif` 保护头文件
> 9. 使用 `clang-format` 自动格式化代码
> 10. 团队应统一代码规范并使用自动化工具检查
