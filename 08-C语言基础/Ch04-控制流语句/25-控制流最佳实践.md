# 控制流最佳实践

## 1. 概述

良好的控制流设计不仅让代码正确运行，更让代码易于阅读、调试和维护。本节总结控制流方面的最佳实践，帮助你写出更优雅的C代码。

## 2. 提高代码可读性

### 2.1 使用有意义的变量名

```c
// 不好
if (x > 18 && y == 1) { /* ... */ }

// 好
if (age >= 18 && is_male) { /* ... */ }
```

### 2.2 提取复杂条件为变量

```c
// 不好：条件表达式太长
if (user->is_logged_in && user->has_permission && !user->is_banned && time(NULL) < user->expire_time) {
    allow_access();
}

// 好：提取为有意义的变量
int is_authorized = user->is_logged_in
                 && user->has_permission
                 && !user->is_banned
                 && time(NULL) < user->expire_time;

if (is_authorized) {
    allow_access();
}
```

### 2.3 提取复杂逻辑为函数

```c
// 不好：循环体太长
for (int i = 0; i < n; i++) {
    // 50行的处理逻辑...
}

// 好：提取为函数
void process_item(Item *item) {
    // 50行的处理逻辑...
}

for (int i = 0; i < n; i++) {
    process_item(&items[i]);
}
```

## 3. 减少嵌套深度

深层嵌套是代码可读性的大敌。超过3层的嵌套就应该考虑重构。

### 3.1 使用 early return（早返回）

```c
// 不好：深层嵌套
int process_order(Order *order) {
    if (order != NULL) {
        if (order->is_valid) {
            if (order->has_items) {
                if (check_inventory(order)) {
                    if (charge_payment(order)) {
                        ship_order(order);
                        return SUCCESS;
                    } else {
                        return ERR_PAYMENT;
                    }
                } else {
                    return ERR_INVENTORY;
                }
            } else {
                return ERR_EMPTY;
            }
        } else {
            return ERR_INVALID;
        }
    } else {
        return ERR_NULL;
    }
}

// 好：guard clause（卫语句）模式
int process_order(Order *order) {
    if (order == NULL)
        return ERR_NULL;

    if (!order->is_valid)
        return ERR_INVALID;

    if (!order->has_items)
        return ERR_EMPTY;

    if (!check_inventory(order))
        return ERR_INVENTORY;

    if (!charge_payment(order))
        return ERR_PAYMENT;

    ship_order(order);
    return SUCCESS;
}
```

### 3.2 Guard Clause 模式

Guard clause 的核心思想：**尽早处理特殊情况并返回**，让主逻辑保持在最外层。

```c
// 不好
double calculate_discount(Customer *c, double price) {
    double discount = 0.0;
    if (c != NULL) {
        if (c->is_vip) {
            discount = price * 0.2;
        } else if (c->is_member) {
            discount = price * 0.1;
        } else {
            discount = 0.0;
        }
    }
    return discount;
}

// 好：guard clause
double calculate_discount(Customer *c, double price) {
    if (c == NULL)
        return 0.0;

    if (c->is_vip)
        return price * 0.2;

    if (c->is_member)
        return price * 0.1;

    return 0.0;
}
```

### 3.3 反转条件

```c
// 不好
for (int i = 0; i < n; i++) {
    if (is_valid(items[i])) {
        // 30行处理逻辑
    }
}

// 好：反转条件，减少嵌套
for (int i = 0; i < n; i++) {
    if (!is_valid(items[i]))
        continue;

    // 30行处理逻辑（少了一层缩进）
}
```

## 4. 选择合适的控制结构

### 4.1 if-else vs switch

```c
// 等值判断且选项多 → switch
switch (command) {
    case CMD_OPEN:  open();  break;
    case CMD_CLOSE: close(); break;
    case CMD_SAVE:  save();  break;
    case CMD_QUIT:  quit();  break;
    default:        error(); break;
}

// 范围判断或复杂条件 → if-else
if (score >= 90) {
    grade = 'A';
} else if (score >= 80) {
    grade = 'B';
} else {
    grade = 'C';
}
```

### 4.2 for vs while

```c
// 次数已知 → for
for (int i = 0; i < n; i++) {
    process(data[i]);
}

// 条件循环 → while
while (fgets(line, sizeof(line), fp) != NULL) {
    process(line);
}

// 至少执行一次 → do-while
do {
    printf("请输入(1-10): ");
    scanf("%d", &num);
} while (num < 1 || num > 10);
```

## 5. 保持一致性

### 5.1 统一花括号风格

```c
// 选择一种风格并始终遵守

// K&R 风格
if (condition) {
    do_something();
} else {
    do_other();
}

// 或 Allman 风格
if (condition)
{
    do_something();
}
else
{
    do_other();
}
```

### 5.2 统一循环模式

```c
// 数组遍历统一使用 [0, n) 模式
for (int i = 0; i < n; i++) { /* ... */ }

// 不要在不同地方混用 [0,n) 和 [1,n]
```

## 6. 避免常见反模式

### 6.1 不要用魔法数字

```c
// 不好
if (status == 3) { /* ... */ }

// 好
#define STATUS_ACTIVE 3
// 或使用枚举
enum { STATUS_ACTIVE = 3 };

if (status == STATUS_ACTIVE) { /* ... */ }
```

### 6.2 不要过度优化循环

```c
// 不要为了"性能"牺牲可读性
// 不好
for(i=0;i<n;i++)a[i]=b[i]+c[i];

// 好
for (int i = 0; i < n; i++) {
    result[i] = operand_a[i] + operand_b[i];
}
```

### 6.3 不要忽略错误

```c
// 不好
scanf("%d", &x);       // 忽略返回值

// 好
if (scanf("%d", &x) != 1) {
    fprintf(stderr, "输入错误\n");
    return -1;
}
```

## 7. 循环的设计原则

### 7.1 单一职责

一个循环只做一件事：

```c
// 不好：一个循环做两件事
int sum = 0, max = arr[0];
for (int i = 0; i < n; i++) {
    sum += arr[i];
    if (arr[i] > max) max = arr[i];
}

// 如果性能不是问题，拆分更清晰
int sum = 0;
for (int i = 0; i < n; i++) {
    sum += arr[i];
}

int max = arr[0];
for (int i = 1; i < n; i++) {
    if (arr[i] > max) max = arr[i];
}
```

### 7.2 明确的终止条件

```c
// 不好：隐含的终止条件
int i = 0;
while (1) {
    if (data[i] == 0) break;
    process(data[i]);
    i++;
}

// 好：明确的终止条件
for (int i = 0; data[i] != 0; i++) {
    process(data[i]);
}
```

## 8. 错误处理模式

### 8.1 集中错误处理

```c
int initialize_system(Config *cfg) {
    Database *db = NULL;
    Cache *cache = NULL;
    Server *server = NULL;

    db = open_database(cfg->db_path);
    if (!db) goto error;

    cache = create_cache(cfg->cache_size);
    if (!cache) goto error;

    server = start_server(cfg->port);
    if (!server) goto error;

    printf("系统初始化成功\n");
    return 0;

error:
    fprintf(stderr, "系统初始化失败\n");
    if (server) stop_server(server);
    if (cache) destroy_cache(cache);
    if (db) close_database(db);
    return -1;
}
```

### 8.2 返回错误码

```c
typedef enum {
    OK = 0,
    ERR_NULL_POINTER,
    ERR_OUT_OF_RANGE,
    ERR_ALLOCATION,
    ERR_IO
} ErrorCode;

ErrorCode safe_divide(int a, int b, int *result) {
    if (result == NULL)
        return ERR_NULL_POINTER;
    if (b == 0)
        return ERR_OUT_OF_RANGE;

    *result = a / b;
    return OK;
}
```

## 9. 代码审查清单

在审查控制流代码时，检查以下要点：

- [ ] 循环边界是否正确（`<` vs `<=`）
- [ ] 循环变量是否会被正确更新
- [ ] 是否有潜在的死循环
- [ ] 嵌套深度是否超过3层
- [ ] 是否使用了有意义的变量名
- [ ] 复杂条件是否提取为变量
- [ ] 是否有未处理的错误情况
- [ ] continue 在 while 中是否安全
- [ ] break 在嵌套循环中是否只跳出一层
- [ ] assert 是否有副作用

## 10. 要点总结

1. **减少嵌套**：使用 guard clause、early return、反转条件
2. **提高可读性**：有意义的命名、提取复杂条件为变量、提取复杂逻辑为函数
3. **选择合适的结构**：等值判断用 switch，范围判断用 if-else，次数已知用 for
4. **保持一致性**：统一花括号风格、循环边界模式
5. **避免反模式**：魔法数字、忽略错误、过度优化
6. **单一职责**：一个循环做一件事，一个函数做一件事

2. 为一个包含复杂条件的 if 语句提取有意义的变量名
3. 审查一段循环代码，找出所有潜在的bug并修复
