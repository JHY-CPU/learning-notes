# 递归基础

## 1. 什么是递归

**递归（Recursion）** 是指函数直接或间接调用自身的一种编程技术。递归是将大问题分解为相同类型的小问题来解决的策略。

简单来说：一个函数在执行过程中调用了自己，就称为递归。

## 2. 递归三要素

一个正确的递归函数必须包含三个要素：

### 2.1 基准情况（Base Case）

递归的终止条件，也叫"出口"。没有基准情况的递归会无限调用自身，导致栈溢出。

### 2.2 递归情况（Recursive Case）

函数调用自身来解决规模更小的同类问题。

### 2.3 向基准情况靠近

每次递归调用必须使问题规模向基准情况缩小，最终一定能到达基准情况。

## 3. 最简单的递归

```c
#include <stdio.h>

void countdown(int n) {
    if (n <= 0) {              // 基准情况
        printf("发射！\n");
        return;
    }
    printf("%d...\n", n);
    countdown(n - 1);          // 递归调用（规模减小）
}

int main(void) {
    countdown(5);
    return 0;
}
```

输出：
```
5...
4...
3...
2...
1...
发射！
```

## 4. 递归与循环的对比

### 4.1 计算 1 到 n 的和

**循环版本：**

```c
int sum_iterative(int n) {
    int sum = 0;
    for (int i = 1; i <= n; i++) {
        sum += i;
    }
    return sum;
}
```

**递归版本：**

```c
int sum_recursive(int n) {
    if (n == 1) {              // 基准情况
        return 1;
    }
    return n + sum_recursive(n - 1);  // 递归情况
}
```

### 4.2 执行过程对比

```
循环:  sum = 0 → 0+1 → 1+2 → 3+3 → 6+4 → 10

递归:  sum(4) = 4 + sum(3)
              = 4 + (3 + sum(2))
              = 4 + (3 + (2 + sum(1)))
              = 4 + (3 + (2 + 1))
              = 4 + (3 + 3)
              = 4 + 6
              = 10
```

递归是"展开"再"收拢"，循环是"逐步累积"。

## 5. 递归的执行机制

### 5.1 调用栈

每次函数调用都会在**调用栈**上创建一个新的**栈帧（Stack Frame）**，包含：
- 函数的参数
- 局部变量
- 返回地址

```c
int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);
}
```

调用 `factorial(4)` 的栈变化：

```
factorial(1) → 返回 1    [栈帧1: n=1]
  ↑ 返回
factorial(2) → 返回 2*1=2  [栈帧2: n=2]
  ↑ 返回
factorial(3) → 返回 3*2=6  [栈帧3: n=3]
  ↑ 返回
factorial(4) → 返回 4*6=24 [栈帧4: n=4]
```

### 5.2 递归展开

```c
#include <stdio.h>

int factorial(int n) {
    printf("进入 factorial(%d)\n", n);

    if (n <= 1) {
        printf("基准情况: factorial(%d) = 1\n", n);
        return 1;
    }

    int result = n * factorial(n - 1);
    printf("factorial(%d) = %d\n", n, result);
    return result;
}

int main(void) {
    int result = factorial(4);
    printf("最终结果: %d\n", result);
    return 0;
}
```

输出：
```
进入 factorial(4)
进入 factorial(3)
进入 factorial(2)
进入 factorial(1)
基准情况: factorial(1) = 1
factorial(2) = 2
factorial(3) = 6
factorial(4) = 24
最终结果: 24
```

## 6. 递归与循环的选择

| 特性 | 递归 | 循环 |
|------|------|------|
| 代码简洁性 | 通常更简洁 | 可能需要更多代码 |
| 性能 | 函数调用开销大 | 一般更快 |
| 内存使用 | 每次调用占栈空间 | 固定内存 |
| 栈溢出风险 | 有（深度过大时） | 无 |
| 适用问题 | 分治、树、图 | 简单迭代 |

### 何时使用递归

- 问题天然具有递归结构（树、图、分治）
- 递归版本比循环版本更清晰
- 问题规模可控（递归深度不会太大）

### 何时使用循环

- 简单的重复操作
- 对性能要求高
- 可能处理大规模数据
- 尾递归优化不可用

## 7. 常见错误

### 7.1 缺少基准情况

```c
// 错误：没有终止条件，无限递归
int bad_recursion(int n) {
    return n + bad_recursion(n - 1);  // 永远不会停止
}
```

### 7.2 基准情况不可达

```c
// 错误：n 永远到不了 0
int bad_recursion(int n) {
    if (n == 0) return 0;
    return n + bad_recursion(n - 2);  // 如果 n 是奇数，永远不会到 0
}
```

### 7.3 递归太深导致栈溢出

```c
// n=100000 会立即栈溢出
int sum = factorial(100000);
```

## 8. 要点总结

1. 递归 = 函数调用自身 + 基准情况 + 向基准情况靠近
2. 没有基准情况的递归会导致无限循环和栈溢出
3. 递归的核心思想是将大问题分解为小问题
4. 递归占用栈空间，深度过大会栈溢出
5. 递归和循环在计算能力上等价，但各有适用场景
6. 对于简单迭代，循环通常性能更好
