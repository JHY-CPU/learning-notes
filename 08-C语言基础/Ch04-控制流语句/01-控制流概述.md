# 控制流概述

## 1. 什么是控制流

程序的执行流程决定了代码按照什么顺序被执行。在最简单的情况下，程序从上到下逐行执行，但实际开发中，我们经常需要根据条件选择不同的执行路径，或者重复执行某些操作。**控制流（Control Flow）** 就是指程序中语句的执行顺序和方向。

控制流是编程的核心概念之一。掌握了控制流，就等于掌握了指挥程序"做什么"和"怎么做"的能力。C语言提供了一套完整的控制流机制，包括条件判断、循环迭代、跳转等。

## 2. 三大基本控制结构

1966年，Bohm和Jacopini证明了任何算法都可以由三种基本结构组成。这三种结构构成了结构化编程的基石。

### 2.1 顺序结构（Sequence）

顺序结构是最简单的控制结构，程序按照语句的书写顺序，从第一条到最后一条依次执行。

```
语句1
  ↓
语句2
  ↓
语句3
```

```c
#include <stdio.h>

int main(void) {
    int a = 10;          // 第1步执行
    int b = 20;          // 第2步执行
    int sum = a + b;     // 第3步执行
    printf("sum = %d\n", sum);  // 第4步执行
    return 0;            // 第5步执行
}
```

程序严格按照从上到下的顺序执行每条语句，不会跳过任何一条，也不会重复执行。这是所有程序的默认执行方式。

### 2.2 选择结构（Selection）

选择结构也叫**分支结构**，程序根据条件的真假，选择执行不同的代码路径。

```
      条件
      /  \
    真    假
    ↓      ↓
  语句A  语句B
```

```c
#include <stdio.h>

int main(void) {
    int score = 85;

    if (score >= 60) {
        printf("及格\n");
    } else {
        printf("不及格\n");
    }

    return 0;
}
```

选择结构的核心是**条件判断**。程序不再盲目地从上到下执行，而是根据条件表达式的值决定走哪条路。

### 2.3 循环结构（Iteration）

循环结构让程序能够重复执行某段代码，直到满足特定条件才停止。

```
      条件
       ↓
    ┌→ 真 → 执行循环体 ─┐
    │                    │
    └────────────────────┘
       ↓ 假
      继续
```

```c
#include <stdio.h>

int main(void) {
    int i = 1;
    int sum = 0;

    while (i <= 100) {
        sum += i;
        i++;
    }

    printf("1到100的和 = %d\n", sum);
    return 0;
}
```

循环结构解决了"重复"的问题。如果没有循环，计算1到100的和需要写100条加法语句。

## 3. C语言中的控制流语句

C语言提供了以下控制流语句：

### 分支语句
| 语句 | 用途 |
|------|------|
| `if` | 单分支条件判断 |
| `if-else` | 双分支条件判断 |
| `else-if` | 多分支条件判断 |
| `switch` | 多路分支（基于整数值） |

### 循环语句
| 语句 | 用途 |
|------|------|
| `while` | 先判断后执行的循环 |
| `do-while` | 先执行后判断的循环 |
| `for` | 计数循环，最常用的循环 |

### 跳转语句
| 语句 | 用途 |
|------|------|
| `break` | 跳出当前循环或switch |
| `continue` | 跳过本次循环剩余部分 |
| `goto` | 无条件跳转到标签处 |
| `return` | 从函数返回 |

## 4. 结构化编程思想

结构化编程（Structured Programming）强调：

- **自顶向下**：将复杂问题分解为小的子问题
- **模块化**：用函数组织代码
- **避免goto**：尽量使用三大基本结构
- **单入口单出口**：每个代码块只有一个入口和一个出口

### 流程图符号

```
┌─────────┐
│  圆角矩形  │  → 表示处理/运算
└─────────┘

◇
  菱形      → 表示判断/条件

┌─────────┐
│   矩形    │  → 表示输入/输出
└─────────┘

○
  圆形      → 表示流程线的连接点
```

### 三种结构可以自由组合

复杂的程序逻辑可以通过组合这三种基本结构来实现：

```c
#include <stdio.h>

int main(void) {
    // 顺序 + 循环 + 分支的组合
    int n, i;
    int is_prime = 1;

    printf("请输入一个正整数: ");
    scanf("%d", &n);

    if (n < 2) {                    // 分支
        is_prime = 0;
    } else {
        for (i = 2; i * i <= n; i++) {  // 循环
            if (n % i == 0) {           // 分支（嵌套在循环中）
                is_prime = 0;
                break;
            }
        }
    }

    if (is_prime) {                 // 分支
        printf("%d是素数\n", n);
    } else {
        printf("%d不是素数\n", n);
    }

    return 0;                       // 顺序
}
```

## 5. 程序流程的可视化理解

### 素数判断的流程图

```
开始
  │
  ▼
输入 n
  │
  ▼
n < 2 ? ──是──→ is_prime = 0
  │ 否
  ▼
i = 2
  │
  ▼
i*i <= n ? ──否──→ 输出结果 → 结束
  │ 是
  ▼
n % i == 0 ? ──是──→ is_prime = 0, break
  │ 否
  ▼
i++
  │
  └──→ (回到 i*i <= n 判断)
```

## 6. 要点总结

1. 顺序、选择、循环是所有程序的三大基本结构，任何复杂逻辑都可以用这三种结构组合实现
2. C语言提供了丰富的控制流语句：if/switch用于分支，while/do-while/for用于循环
3. 结构化编程强调清晰的控制流，避免随意跳转
4. 良好的控制流设计使代码逻辑清晰、易于调试和维护
5. 在实际编程中，通常需要组合使用多种控制结构来解决复杂问题

## 7. 常见问题

**Q: 为什么说goto应该避免使用？**
A: goto会打破程序的正常执行流程，使代码难以理解和维护。现代编程中，几乎所有的goto使用场景都可以用更清晰的结构化语句替代。

**Q: 三种循环有什么本质区别吗？**
A: 从计算理论上说，三种循环是等价的，可以互相转换。但在实际编程中，for适合已知次数的循环，while适合条件循环，do-while适合至少执行一次的场景。
