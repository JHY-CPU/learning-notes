# 循环性能优化

## 1. 概述

循环是程序中执行最频繁的部分，循环的性能直接影响整个程序的效率。本节介绍常见的循环优化技术，从算法层面和代码层面提升循环性能。

## 2. 减少循环内计算

### 2.1 循环不变量外提（Loop Invariant Code Motion）

将不依赖循环变量的计算移到循环外：

```c
// 未优化：每次循环都调用 strlen
for (int i = 0; i < strlen(str); i++) {
    process(str[i]);
}

// 优化：strlen 只调用一次
int len = strlen(str);
for (int i = 0; i < len; i++) {
    process(str[i]);
}
```

### 2.2 减少函数调用

```c
// 未优化：每次循环都调用函数
for (int i = 0; i < n; i++) {
    if (sqrt(arr[i]) > threshold) {
        result += arr[i];
    }
}

// 优化：减少函数调用次数
double threshold_sq = threshold * threshold;
for (int i = 0; i < n; i++) {
    if (arr[i] > threshold_sq) {   // 用乘法代替sqrt
        result += arr[i];
    }
}
```

### 2.3 预计算

```c
// 未优化：每次循环计算偏移
for (int i = 0; i < rows; i++) {
    for (int j = 0; j < cols; j++) {
        matrix[i][j] = i * cols + j;  // 每次都做乘法
    }
}

// 优化：用累加代替乘法
for (int i = 0; i < rows; i++) {
    int base = i * cols;
    for (int j = 0; j < cols; j++) {
        matrix[i][j] = base + j;
    }
}
```

## 3. 循环展开（Loop Unrolling）

### 3.1 基本概念

减少循环迭代次数，每次迭代处理多个元素：

```c
// 未展开
int sum = 0;
for (int i = 0; i < 1000; i++) {
    sum += arr[i];
}

// 展开2倍
int sum = 0;
for (int i = 0; i < 1000; i += 2) {
    sum += arr[i];
    sum += arr[i + 1];
}

// 展开4倍
int sum = 0;
for (int i = 0; i < 1000; i += 4) {
    sum += arr[i];
    sum += arr[i + 1];
    sum += arr[i + 2];
    sum += arr[i + 3];
}
```

### 3.2 处理余数

当总数不能被展开倍数整除时：

```c
int sum_array(int arr[], int n) {
    int sum = 0;
    int i;

    // 每次处理4个元素
    for (i = 0; i + 3 < n; i += 4) {
        sum += arr[i] + arr[i+1] + arr[i+2] + arr[i+3];
    }

    // 处理剩余的元素
    for (; i < n; i++) {
        sum += arr[i];
    }

    return sum;
}
```

### 3.3 展开的效果

```
原始循环:
  每次: 比较 → 索引计算 → 加法 → 自增 → 跳转 (5次操作)
  1000次: ~5000次操作

展开4倍:
  每次: 比较 → 4次加法 → 自增 → 跳转 (7次操作)
  250次: ~1750次操作
```

现代编译器通常会自动进行循环展开（使用 `-O2` 或 `-O3` 优化）。

## 4. 循环合并（Loop Fusion）

将两个独立但迭代次数相同的循环合并：

```c
// 未优化：两次遍历
for (int i = 0; i < n; i++) {
    a[i] = b[i] + c[i];
}
for (int i = 0; i < n; i++) {
    d[i] = a[i] * 2;
}

// 优化：一次遍历
for (int i = 0; i < n; i++) {
    a[i] = b[i] + c[i];
    d[i] = a[i] * 2;
}
```

但要注意数据依赖关系：

```c
// 不能合并！有数据依赖
for (int i = 1; i < n; i++) {
    a[i] = a[i-1] + 1;
}
for (int i = 0; i < n; i++) {
    b[i] = a[i] * 2;
}
```

## 5. 循环拆分（Loop Fission）

如果循环体太大导致缓存不友好，可以拆分：

```c
// 拆分前：一次遍历做太多事情
for (int i = 0; i < n; i++) {
    a[i] = expensive_func1(b[i]);
    c[i] = expensive_func2(d[i]);
    e[i] = expensive_func3(f[i]);
}

// 拍分后：更好的缓存局部性
for (int i = 0; i < n; i++) {
    a[i] = expensive_func1(b[i]);
}
for (int i = 0; i < n; i++) {
    c[i] = expensive_func2(d[i]);
}
for (int i = 0; i < n; i++) {
    e[i] = expensive_func3(f[i]);
}
```

## 6. 避免不必要的数组访问

```c
// 未优化：多次访问数组
for (int i = 0; i < n; i++) {
    arr[i] = arr[i] * 2 + arr[i] / 3;
}

// 优化：用局部变量缓存
for (int i = 0; i < n; i++) {
    int temp = arr[i];
    arr[i] = temp * 2 + temp / 3;
}
```

## 7. 提高缓存友好性

### 7.1 按行遍历 vs 按列遍历

```c
// 按行遍历（缓存友好）
for (int i = 0; i < rows; i++) {
    for (int j = 0; j < cols; j++) {
        matrix[i][j] = 0;   // 连续内存访问
    }
}

// 按列遍历（缓存不友好）
for (int j = 0; j < cols; j++) {
    for (int i = 0; i < rows; i++) {
        matrix[i][j] = 0;   // 跳跃式内存访问
    }
}
```

C语言的二维数组按行存储，按行遍历时访问的是连续内存，缓存命中率最高。

### 7.2 分块处理

```c
#define BLOCK_SIZE 64

// 矩阵乘法的分块优化
for (int ii = 0; ii < n; ii += BLOCK_SIZE) {
    for (int jj = 0; jj < n; jj += BLOCK_SIZE) {
        for (int kk = 0; kk < n; kk += BLOCK_SIZE) {
            // 处理一个块
            for (int i = ii; i < ii + BLOCK_SIZE && i < n; i++) {
                for (int j = jj; j < jj + BLOCK_SIZE && j < n; j++) {
                    for (int k = kk; k < kk + BLOCK_SIZE && k < n; k++) {
                        C[i][j] += A[i][k] * B[k][j];
                    }
                }
            }
        }
    }
}
```

## 8. 编译器优化

### 8.1 编译选项

```bash
gcc -O0 program.c    # 无优化（默认）
gcc -O1 program.c    # 基本优化
gcc -O2 program.c    # 推荐：大多数优化
gcc -O3 program.c    # 激进优化（包括循环展开、向量化等）
```

### 8.2 查看编译器优化效果

```bash
# 查看汇编代码
gcc -O2 -S program.c

# 查看编译器优化报告
gcc -O3 -fopt-info-loop program.c
```

### 8.3 restrict 关键字

告诉编译器指针没有重叠，帮助优化：

```c
// 编译器不能假设 arr1 和 arr2 不重叠
void add_arrays(int *arr1, int *arr2, int *result, int n) {
    for (int i = 0; i < n; i++) {
        result[i] = arr1[i] + arr2[i];
    }
}

// 告诉编译器它们不重叠，可以更激进地优化
void add_arrays_opt(int *restrict arr1, int *restrict arr2,
                    int *restrict result, int n) {
    for (int i = 0; i < n; i++) {
        result[i] = arr1[i] + arr2[i];
    }
}
```

## 9. 优化的优先级

1. **算法优化**：选择更好的算法（如 O(n log n) vs O(n^2)）
2. **数据结构**：选择合适的数据结构
3. **循环不变量外提**：避免重复计算
4. **减少循环内操作**：减少函数调用、数组访问
5. **循环展开**：减少循环开销
6. **缓存优化**：提高数据局部性
7. **编译器优化**：使用合适的编译选项

**黄金法则**：先保证正确性，再优化性能。没有性能瓶颈时不要过度优化。

## 10. 要点总结

1. **循环不变量外提**：将不依赖循环变量的计算移到循环外
2. **循环展开**：减少迭代次数，每次处理多个元素，减少分支开销
3. **缓存友好性**：按行遍历二维数组，利用空间局部性
4. 编译器（`-O2`/`-O3`）会自动进行很多循环优化
5. 优化前先用性能分析工具找到真正的瓶颈
6. 过度优化会降低代码可读性，得不偿失
