# 条件编译与控制流

## 1. 概述

C语言的预处理器提供了**条件编译**指令，可以在编译时根据条件决定哪些代码被包含在编译中。这与运行时的 `if/else` 控制流不同——条件编译在编译之前就决定了代码的命运。

## 2. 预处理器条件指令

### 2.1 #if / #elif / #else / #endif

```c
#if 条件表达式
    // 条件为真时编译的代码
#elif 其他条件
    // 其他条件为真时编译的代码
#else
    // 所有条件都为假时编译的代码
#endif
```

**示例：**

```c
#include <stdio.h>

#define DEBUG_LEVEL 2

int main(void) {
#if DEBUG_LEVEL == 0
    printf("发布模式\n");
#elif DEBUG_LEVEL == 1
    printf("基本信息模式\n");
#elif DEBUG_LEVEL == 2
    printf("详细调试模式\n");
#else
    printf("未知模式\n");
#endif

    return 0;
}
```

### 2.2 #ifdef / #ifndef

检查宏是否已定义：

```c
#ifdef 宏名
    // 如果宏已定义，编译这些代码
#endif

#ifndef 宏名
    // 如果宏未定义，编译这些代码
#endif
```

**示例：**

```c
#include <stdio.h>

#define USE_NEW_API

int main(void) {
#ifdef USE_NEW_API
    printf("使用新API\n");
#else
    printf("使用旧API\n");
#endif

#ifndef RELEASE
    printf("这是调试版本\n");
#endif

    return 0;
}
```

### 2.3 #if defined()

`defined` 运算符可以检查宏是否已定义，支持组合条件：

```c
#if defined(DEBUG) && defined(VERBOSE)
    printf("详细调试信息\n");
#elif defined(DEBUG)
    printf("基本调试信息\n");
#endif

// 等价于
#ifdef DEBUG
#ifdef VERBOSE
    printf("详细调试信息\n");
#else
    printf("基本调试信息\n");
#endif
#endif
```

### 2.4 #undef

取消宏定义：

```c
#define DEBUG
// ... 使用 DEBUG ...

#undef DEBUG
// DEBUG 不再被定义

#ifdef DEBUG
    // 这段代码不会被编译
#endif
```

## 3. 头文件保护

条件编译最常见的用途——防止头文件被多次包含：

```c
// myheader.h
#ifndef MYHEADER_H
#define MYHEADER_H

// 头文件内容
struct Student {
    char name[50];
    int age;
};

void print_student(struct Student *s);

#endif // MYHEADER_H
```

**工作原理：**
1. 第一次包含：`MYHEADER_H` 未定义 → 定义它 → 编译内容
2. 第二次包含：`MYHEADER_H` 已定义 → 跳过整个内容

**现代替代方案（C23或部分编译器支持）：**

```c
#pragma once    // 非标准但广泛支持
```

## 4. 平台相关代码

```c
#include <stdio.h>

#if defined(_WIN32) || defined(_WIN64)
    #define PLATFORM "Windows"
    #include <windows.h>
    void clear_screen() { system("cls"); }
#elif defined(__linux__)
    #define PLATFORM "Linux"
    #include <unistd.h>
    void clear_screen() { system("clear"); }
#elif defined(__APPLE__)
    #define PLATFORM "macOS"
    #include <unistd.h>
    void clear_screen() { system("clear"); }
#else
    #define PLATFORM "Unknown"
    void clear_screen() { /* 不支持 */ }
#endif

int main(void) {
    printf("当前平台: %s\n", PLATFORM);
    return 0;
}
```

## 5. 调试开关

```c
#include <stdio.h>

// 调试宏定义
#ifdef DEBUG
    #define LOG(fmt, ...) \
        fprintf(stderr, "[DEBUG] %s:%d: " fmt "\n", __FILE__, __LINE__, ##__VA_ARGS__)
#else
    #define LOG(fmt, ...)  // 空宏，不产生任何代码
#endif

int main(void) {
    int x = 42;

    LOG("程序开始");
    LOG("x = %d", x);

    int result = x * 2;
    LOG("result = %d", result);

    printf("结果: %d\n", result);
    return 0;
}

// 编译时加 -DDEBUG 启用调试
// gcc -DDEBUG program.c -o program
```

## 6. 编译时配置

```c
#include <stdio.h>

// 编译时配置缓冲区大小
#ifndef BUFFER_SIZE
    #define BUFFER_SIZE 1024   // 默认值
#endif

// 编译时配置功能开关
#ifndef ENABLE_FEATURE_X
    #define ENABLE_FEATURE_X 0
#endif

int main(void) {
    char buffer[BUFFER_SIZE];
    printf("缓冲区大小: %zu\n", sizeof(buffer));

#if ENABLE_FEATURE_X
    printf("功能X已启用\n");
#else
    printf("功能X已禁用\n");
#endif

    return 0;
}

// 编译时覆盖默认值
// gcc -DBUFFER_SIZE=2048 -DENABLE_FEATURE_X=1 program.c
```

## 7. 条件编译 vs 运行时控制流

| 特性 | 条件编译 | 运行时控制流 |
|------|----------|-------------|
| 决策时间 | 编译前（预处理阶段） | 运行时 |
| 条件 | 宏是否定义/宏的值 | 变量/表达式的值 |
| 不匹配的代码 | 不会被编译 | 会被编译但不执行 |
| 性能开销 | 零（代码不参与编译） | 有（条件判断开销） |
| 灵活性 | 编译时确定，不可更改 | 运行时可变 |
| 调试 | 较难（看不到被排除的代码） | 容易 |

### 对比示例

```c
// 条件编译：在编译时决定
#ifdef FAST_MODE
    #define CALC(x) ((x) * (x))     // 宏展开，无函数调用开销
#else
    int CALC(int x) { return x * x; } // 函数调用，便于调试
#endif

// 运行时控制流：在运行时决定
int calculate(int x, int fast_mode) {
    if (fast_mode) {
        return x * x;   // 简单运算
    } else {
        // 复杂但可调试的逻辑
        log("计算 %d 的平方", x);
        return x * x;
    }
}
```

## 8. 条件编译的注意事项

### 8.1 不要过度使用

```c
// 不推荐：过多的条件编译导致代码难以阅读
#if A
    #if B
        #if C
            // 核心逻辑深埋在条件编译中
        #endif
    #endif
#endif
```

### 8.2 注释掉代码的陷阱

```c
/*
  这是一个被注释掉的条件编译块
#if DEBUG
    printf("debug\n");
#endif
*/

// 如果 printf 中有 /*，可能意外终止注释
// printf("/* 这不是注释开始 */\n");  // 这样是安全的
```

### 8.3 条件编译中的语法

条件编译在预处理阶段工作，此时编译器尚未对代码进行语法分析：

```c
#if 1
    int x = ;    // 语法错误，但预处理器不管
#endif
// 只有这行被包含进编译后，编译器才会报错
```

## 9. 要点总结

1. `#if` / `#ifdef` / `#ifndef` 在编译前决定代码是否参与编译
2. 头文件保护（`#ifndef HEADER_H`）是条件编译最常见的用途
3. 条件编译可以实现平台适配、调试开关、编译时配置
4. 条件编译与运行时控制流的本质区别在于**决策时机**
5. 不要过度使用条件编译，以免降低代码可读性
6. `#pragma once` 是头文件保护的现代替代方案，但非标准
