# 常见循环错误

## 1. 概述

循环是程序中最容易出错的部分之一。本节总结常见的循环错误、它们的表现形式以及如何避免。

## 2. 差一错误（Off-by-One Error）

差一错误是最常见的循环错误：循环多执行一次或少执行一次。

### 2.1 `<` 与 `<=` 混淆

```c
// 意图：打印数组的前10个元素（索引0-9）
int arr[10];

// 错误：多执行一次，访问 arr[10] 越界！
for (int i = 0; i <= 10; i++) {
    printf("%d ", arr[i]);  // i=10 时越界
}

// 正确
for (int i = 0; i < 10; i++) {
    printf("%d ", arr[i]);
}
```

### 2.2 边界条件错误

```c
// 意图：处理从1到n的数字
int n = 10;

// 错误：少处理了 n
for (int i = 1; i < n; i++) {   // i从1到9，没有处理10
    process(i);
}

// 正确
for (int i = 1; i <= n; i++) {
    process(i);
}
```

### 2.3 计数错误

```c
// 意图：执行10次
int count = 0;
for (int i = 0; i < 10; i++) {
    count++;
}
// count = 10，正确

// 错误写法
int count = 0;
for (int i = 1; i <= 10; i++) {
    count++;
}
// count = 10，也是正确的，但容易与上面的混用导致错误

// 混用导致错误
int count = 0;
for (int i = 1; i < 10; i++) {  // i从1到9，只执行9次
    count++;
}
// count = 9，错误！
```

### 2.4 字符串处理中的差一

```c
char str[10] = "Hello";   // 5个字符 + '\0'

// 错误：多复制了一个字符或少复制了'\0'
for (int i = 0; i <= 10; i++) {  // 越界！
    dest[i] = src[i];
}

// 正确
for (int i = 0; src[i] != '\0'; i++) {
    dest[i] = src[i];
}
dest[i] = '\0';  // 别忘了终止符
```

## 3. 死循环

### 3.1 忘记更新循环变量

```c
// 错误：i 永远是 0
int i = 0;
while (i < 10) {
    printf("%d\n", i);
    // 忘记了 i++
}

// 正确
int i = 0;
while (i < 10) {
    printf("%d\n", i);
    i++;
}
```

### 3.2 更新方向错误

```c
// 错误：i 越来越小，永远不会 >= 10
int i = 0;
while (i < 10) {
    printf("%d\n", i);
    i--;    // 应该是 i++
}

// 类似错误
for (int i = 0; i < 10; i--) {  // 应该是 i++
    printf("%d\n", i);
}
```

### 3.3 条件永远为真

```c
// 错误：条件表达式的副作用
int x = 0;
while (x = 1) {    // 赋值，不是比较！x始终为1（真）
    printf("%d\n", x);
}

// 正确
while (x == 1) {
    printf("%d\n", x);
}
```

### 3.4 浮点数比较不精确

```c
// 错误：由于浮点误差，可能永远不等于1.0
double x = 0.0;
while (x != 1.0) {
    x += 0.1;
}

// 正确：使用误差范围
while (x < 1.0 - 1e-9) {
    x += 0.1;
}
```

## 4. 循环变量错误

### 4.1 在循环体内修改循环变量

```c
// 意外行为：i 被修改了
for (int i = 0; i < 10; i++) {
    if (some_condition) {
        i = i + 5;    // 跳过5次迭代！
    }
    printf("%d\n", i);
}
```

### 4.2 循环变量类型不匹配

```c
// 错误：unsigned 下溢后变成很大的数
unsigned int i;
for (i = 10; i >= 0; i--) {    // i=0 后再减1变成 UINT_MAX！
    printf("%u\n",i);
}

// 正确
for (int i = 10; i >= 0; i--) {
    printf("%d\n", i);
}

// 或
for (unsigned int i = 10; i > 0; i--) {  // 用 > 0 代替 >= 0
    printf("%u\n", i - 1);  // 需要调整打印值
}
```

### 4.3 循环变量作用域问题

```c
// C89 风格：i 在循环外仍然可用，可能被意外使用
int i;
for (i = 0; i < 10; i++) {
    // ...
}
// 有人可能在这里意外使用了 i

// C99 风格：i 的作用域限制在循环内
for (int i = 0; i < 10; i++) {
    // ...
}
// i 在此处不可用，更安全
```

## 5. continue 导致的问题

### 5.1 while 中的 continue 后跳过更新

```c
// 死循环：当 i==5 时，continue 跳过了 i++
int i = 0;
while (i < 10) {
    if (i == 5) {
        continue;    // i 永远是 5
    }
    printf("%d\n", i);
    i++;
}

// 修正1：在 continue 前更新
int i = 0;
while (i < 10) {
    if (i == 5) {
        i++;
        continue;
    }
    printf("%d\n", i);
    i++;
}

// 修正2：用 if 代替 continue
int i = 0;
while (i < 10) {
    if (i != 5) {
        printf("%d\n", i);
    }
    i++;
}
```

### 5.2 for 循环中 continue 安全

for 循环的更新表达式总是在循环体之后执行，所以 continue 不会导致死循环：

```c
for (int i = 0; i < 10; i++) {
    if (i == 5) continue;  // 安全：i++ 仍然会执行
    printf("%d\n", i);
}
```

## 6. 嵌套循环错误

### 6.1 内外层变量混淆

```c
// 错误：内外层用了同一个变量
for (int i = 0; i < 5; i++) {
    for (int i = 0; i < 3; i++) {  // 内层 i 遮蔽了外层 i
        printf("(%d,%d) ", i, i);  // 两个 i 都是内层的
    }
}

// 正确：使用不同的变量名
for (int i = 0; i < 5; i++) {
    for (int j = 0; j < 3; j++) {
        printf("(%d,%d) ", i, j);
    }
}
```

### 6.2 break 只跳出一层

```c
// 意图：找到目标后停止所有循环
int found = 0;
for (int i = 0; i < rows; i++) {
    for (int j = 0; j < cols; j++) {
        if (matrix[i][j] == target) {
            found = 1;
            break;   // 只跳出内层！外层还在继续
        }
    }
    // 应该在这里检查 found
}
```

## 7. 数组越界

### 7.1 循环条件越界

```c
int arr[5] = {1, 2, 3, 4, 5};

// 错误：i 可以等于 5，越界访问
for (int i = 0; i <= 5; i++) {
    printf("%d ", arr[i]);  // arr[5] 越界
}

// 正确
for (int i = 0; i < 5; i++) {
    printf("%d ", arr[i]);
}
```

### 7.2 字符串未考虑终止符

```c
char str[10];
strcpy(str, "Hello");   // str = "Hello\0"

// 错误：访问了超出字符串长度的位置
for (int i = 0; i < 10; i++) {
    printf("%c", str[i]);  // 后面是未初始化的垃圾值
}

// 正确
for (int i = 0; str[i] != '\0'; i++) {
    printf("%c", str[i]);
}
```

## 8. 常见错误清单

| 错误类型 | 表现 | 预防方法 |
|----------|------|----------|
| 差一错误 | 多/少执行一次 | 明确 `<` 和 `<=` 的选择 |
| 忘记更新 | 死循环 | 始终在循环体中包含更新语句 |
| 更新方向错 | 死循环 | 检查 `++` 还是 `--` |
| 赋值当比较 | 逻辑错误/死循环 | 编译器警告 + 代码审查 |
| 浮点比较 | 死循环 | 使用误差范围 |
| unsigned 下溢 | 逻辑错误 | 注意无符号数的边界 |
| continue 跳过更新 | 死循环 | 在 continue 前更新或用 for |
| 嵌套变量混淆 | 逻辑错误 | 使用不同的变量名 |
| 数组越界 | 未定义行为 | 仔细检查边界条件 |

## 9. 预防技巧

1. **使用编译器警告**：`gcc -Wall -Wextra`
2. **静态分析工具**：cppcheck, clang-tidy
3. **代码审查**：让他人检查循环逻辑
4. **单元测试**：测试边界条件（0、1、n-1、n、n+1）
5. **使用 for 而非 while**：减少忘记更新的风险

## 10. 要点总结

1. 差一错误是最常见的循环错误，注意 `<` 和 `<=` 的选择
2. 死循环的常见原因：忘记更新、更新方向错误、条件永远为真
3. 在 while 中使用 continue 要特别小心，确保循环变量被更新
4. unsigned 类型的下溢会导致意外的大数值
5. 嵌套循环中 break 只跳出一层，需要标志变量跳出多层
6. 使用编译器警告和测试来预防循环错误
