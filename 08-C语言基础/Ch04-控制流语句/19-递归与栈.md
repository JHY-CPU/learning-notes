# 递归与栈

## 1. 函数调用栈

### 1.1 什么是调用栈

每当程序调用一个函数时，系统会在**调用栈（Call Stack）**上为该函数创建一个**栈帧（Stack Frame）**。栈帧包含了：

- 函数的参数值
- 局部变量
- 返回地址（函数执行完后返回哪里）
- 保存的寄存器状态

函数返回时，对应的栈帧被弹出（销毁）。

### 1.2 栈帧结构

```
高地址
┌─────────────┐
│ main()帧     │  ← main的参数、局部变量、返回地址
├─────────────┤
│ func1()帧    │  ← func1的参数、局部变量、返回地址
├─────────────┤
│ func2()帧    │  ← func2的参数、局部变量、返回地址
├─────────────┤
│ func3()帧    │  ← 当前正在执行的函数
└─────────────┘
低地址（栈增长方向）
```

## 2. 递归的栈消耗

### 2.1 每次递归调用都消耗栈空间

```c
#include <stdio.h>

void recursive_print(int n) {
    char buffer[1024];   // 每次调用分配1KB局部变量
    if (n <= 0) return;
    printf("调用深度: %d\n", n);
    recursive_print(n - 1);
}

int main(void) {
    recursive_print(10000);  // 可能栈溢出！
    return 0;
}
```

每次递归调用消耗约 1KB + 函数开销，10000次调用约 10MB 栈空间，默认栈大小通常只有 1-8MB。

### 2.2 栈溢出（Stack Overflow）

当递归深度超过栈空间限制时，程序会崩溃并报告 **栈溢出**：

```
Segmentation fault (core dumped)
// 或
Stack overflow
```

**常见原因：**
- 缺少基准情况
- 递归深度过大
- 每次递归分配大量局部变量

### 2.3 测试栈大小

```c
#include <stdio.h>

int depth = 0;

void test_stack(int n) {
    int dummy;   // 占用栈空间的局部变量
    depth++;
    printf("深度: %d, 地址: %p\n", depth, (void *)&dummy);
    test_stack(n + 1);  // 无限递归
}

int main(void) {
    test_stack(0);
    return 0;
}
// 运行后观察在多少层崩溃
```

## 3. 递归深度

### 3.1 不同问题的递归深度

| 问题 | 递归深度 | 风险 |
|------|----------|------|
| 阶乘 n! | n | n > 10000 可能溢出 |
| 斐波那契 F(n) | n | n > 1000 可能溢出 |
| 二分查找 | log₂(n) | 非常安全 |
| 汉诺塔 | n | n > 20 有风险 |
| 快速排序（平均） | log₂(n) | 安全 |
| 快速排序（最差） | n | 有风险 |

### 3.2 递归深度估算

```c
// 二分查找：每次减半，深度 log₂(n)
int binary_search(int arr[], int l, int r, int x) {
    if (l > r) return -1;
    int mid = (l + r) / 2;
    if (arr[mid] == x) return mid;
    if (arr[mid] < x) return binary_search(arr, mid + 1, r, x);
    return binary_search(arr, l, mid - 1, x);
}
// 10亿个元素只需要约30层递归
```

## 4. 用循环替代递归

### 4.1 栈上模拟递归

任何递归都可以用循环+显式栈替代：

```c
#include <stdio.h>

#define MAX_SIZE 1000

// 手动栈模拟递归的阶乘
long long factorial_iterative(int n) {
    long long result = 1;
    for (int i = 2; i <= n; i++) {
        result *= i;
    }
    return result;
}

// 用显式栈模拟递归遍历二叉树（先序）
typedef struct { int data[MAX_SIZE]; int top; } Stack;

void push(Stack *s, int val) { s->data[++(s->top)] = val; }
int pop(Stack *s) { return s->data[(s->top)--]; }
int is_empty(Stack *s) { return s->top == -1; }
```

### 4.2 尾递归

**尾递归（Tail Recursion）** 是指递归调用是函数的最后一个操作，其返回值直接作为当前函数的返回值。

```c
// 非尾递归：递归调用后还有乘法操作
int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);  // 返回前还要乘 n
}

// 尾递归：递归调用是最后一步
int factorial_tail(int n, int acc) {
    if (n <= 1) return acc;
    return factorial_tail(n - 1, n * acc);  // 最后一步就是递归调用
}
// 调用: factorial_tail(5, 1)
```

尾递归的展开过程：
```
factorial_tail(5, 1)
factorial_tail(4, 5)
factorial_tail(3, 20)
factorial_tail(2, 60)
factorial_tail(1, 120)
→ 返回 120
```

不需要保存中间结果，理论上编译器可以将其优化为循环。

### 4.3 尾递归优化

支持尾递归优化的编译器可以将尾递归转换为循环，避免栈增长：

```c
// 尾递归版本的斐波那契
int fib_tail(int n, int a, int b) {
    if (n == 0) return a;
    if (n == 1) return b;
    return fib_tail(n - 1, b, a + b);
}
// 调用: fib_tail(10, 0, 1)
```

**注意**：C语言标准不要求编译器做尾递归优化。GCC 和 Clang 在 `-O2` 优化级别下会进行此优化，但不能保证。

## 5. 递归转循环的技巧

### 5.1 简单递归 → 循环

```c
// 递归版
int sum(int n) {
    if (n == 0) return 0;
    return n + sum(n - 1);
}

// 循环版
int sum_iter(int n) {
    int total = 0;
    for (int i = n; i > 0; i--) {
        total += i;
    }
    return total;
}
```

### 5.2 复杂递归 → 显式栈

```c
#include <stdio.h>

// 将递归的树遍历改为显式栈版本
void tree_traversal_iterative(Node *root) {
    Stack s;
    push(&s, root);

    while (!is_empty(&s)) {
        Node *current = pop(&s);
        process(current);

        if (current->right) push(&s, current->right);
        if (current->left) push(&s, current->left);
    }
}
```

## 6. 栈的大小调整

### 6.1 Linux 下调整栈大小

```bash
# 查看当前栈大小
ulimit -s
# 通常是 8192 (KB)

# 临时增大栈大小
ulimit -s 65536

# 运行程序
./myprogram
```

### 6.2 程序中调整（部分系统支持）

```c
#include <sys/resource.h>

int main(void) {
    const rlim_t stack_size = 64 * 1024 * 1024;  // 64MB
    struct rlimit rl;

    getrlimit(RLIMIT_STACK, &rl);
    if (rl.rlim_cur < stack_size) {
        rl.rlim_cur = stack_size;
        setrlimit(RLIMIT_STACK, &rl);
    }

    // 现在可以使用更深的递归
    return 0;
}
```

## 7. 要点总结

1. 每次递归调用都在调用栈上创建新的栈帧，消耗栈空间
2. 递归深度过大会导致**栈溢出**
3. 二分查找等 log 级别的递归非常安全
4. **尾递归**是指递归调用是函数最后一步操作，可以被编译器优化为循环
5. 所有递归都可以用循环+显式栈替代
6. 遇到栈溢出问题时，可以增大栈空间或将递归改为循环
