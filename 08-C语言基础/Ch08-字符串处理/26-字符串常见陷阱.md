# 26 - 字符串常见陷阱

## 一、忘记 \0 终止符

### 1.1 手动构造字符串忘记 \0

```c
// ❌ 错误：字符数组没有 \0
char str[] = {'H', 'e', 'l', 'l', 'o'};
printf("%s\n", str);  // 未定义行为！会继续读取后面的内存

// ✅ 正确：添加 \0
char str[] = {'H', 'e', 'l', 'l', 'o', '\0'};
```

### 1.2 strncpy 不自动添加 \0

```c
// ❌ 错误
char dest[5];
strncpy(dest, "Hello, World!", 5);
printf("%s\n", dest);  // 未定义行为！没有 \0

// ✅ 正确
char dest[6];
strncpy(dest, "Hello, World!", 5);
dest[5] = '\0';  // 手动添加
```

## 二、缓冲区溢出

### 2.1 strcpy 溢出

```c
// ❌ 危险：目标缓冲区不够大
char small[5];
strcpy(small, "Hello, World!");  // 溢出！写入了14字节到5字节的空间

// ✅ 安全：使用 strncpy 或 strcpy_s
char dest[50];
strcpy(dest, "Hello, World!");
```

### 2.2 gets 极度危险

```c
// ❌ 千万不要使用 gets！
// char buf[10];
// gets(buf);  // C11 已移除此函数

// ✅ 使用 fgets
char buf[10];
fgets(buf, sizeof(buf), stdin);
```

### 2.3 strcat 溢出

```c
// ❌ 错误
char str[10] = "Hello";
strcat(str, ", World!");  // 溢出！

// ✅ 正确
char str[50] = "Hello";
strcat(str, ", World!");
```

### 2.4 sprintf 溢出

```c
// ❌ 错误
char buf[10];
sprintf(buf, "Value: %d", 123456789);  // 溢出

// ✅ 使用 snprintf
char buf[20];
snprintf(buf, sizeof(buf), "Value: %d", 123456789);
```

## 三、指针未初始化

### 3.1 未初始化就使用

```c
// ❌ 错误：指针未指向有效内存
char *ptr;
strcpy(ptr, "Hello");  // 崩溃！

// ✅ 正确：指向数组
char buf[50];
char *ptr = buf;
strcpy(ptr, "Hello");

// ✅ 正确：动态分配
char *ptr = (char *)malloc(50);
if (ptr) {
    strcpy(ptr, "Hello");
    free(ptr);
}
```

### 3.2 返回局部数组指针

```c
// ❌ 错误
char *get_string() {
    char str[] = "Hello";  // 局部变量，在栈上
    return str;  // 返回后 str 无效
}

// ✅ 正确1：返回字面量（只读）
const char *get_string() {
    return "Hello";
}

// ✅ 正确2：返回动态分配的内存
char *get_string() {
    char *str = (char *)malloc(6);
    if (str) strcpy(str, "Hello");
    return str;  // 调用者负责 free
}

// ✅ 正确3：通过参数返回
void get_string(char *buf, size_t size) {
    strncpy(buf, "Hello", size - 1);
    buf[size - 1] = '\0';
}
```

## 四、修改字符串字面量

```c
// ❌ 未定义行为：修改只读内存
char *str = "Hello";
str[0] = 'h';  // 可能崩溃

// ✅ 正确：使用数组
char str[] = "Hello";
str[0] = 'h';  // 合法

// ✅ 正确：使用 const 明确标识
const char *str = "Hello";
// str[0] = 'h';  // 编译错误，编译器会捕获
```

## 五、sizeof 与 strlen 混淆

```c
// ❌ 错误
char buf[100] = "Hello";
for (int i = 0; i < sizeof(buf); i++) {  // 遍历了100个字符
    if (buf[i] == '\0') break;
}

// ✅ 正确
size_t len = strlen(buf);  // len = 5
for (int i = 0; i < len; i++) {
    // 只处理5个有效字符
}

// ❌ 错误：指针上用 sizeof
char *ptr = "Hello";
printf("%zu\n", sizeof(ptr));  // 8 (指针大小)，不是字符串长度
printf("%zu\n", strlen(ptr));  // 5 (字符串长度)
```

## 六、循环中重复调用 strlen

```c
// ❌ 低效：O(n²)
for (int i = 0; i < strlen(str); i++) {
    // 每次循环都调用 strlen
}

// ✅ 高效：O(n)
size_t len = strlen(str);
for (int i = 0; i < len; i++) {
    // 只调用一次
}
```

## 七、用 == 比较字符串

```c
// ❌ 错误：比较指针地址
char s1[20] = "Hello";
char s2[20] = "Hello";
if (s1 == s2) {  // 几乎总是假！
    printf("相等\n");
}

// ✅ 正确：使用 strcmp
if (strcmp(s1, s2) == 0) {
    printf("相等\n");
}
```

## 八、忽略返回值

```c
// ❌ 危险：忽略 malloc 返回值
char *ptr = (char *)malloc(100);
strcpy(ptr, "Hello");  // 如果 malloc 返回 NULL，崩溃！

// ✅ 正确：检查返回值
char *ptr = (char *)malloc(100);
if (ptr == NULL) {
    fprintf(stderr, "内存分配失败\n");
    return 1;
}
strcpy(ptr, "Hello");
free(ptr);
```

## 九、内存泄漏

```c
// ❌ 内存泄漏
char *str = (char *)malloc(100);
str = (char *)malloc(200);  // 第一次分配的100字节泄漏了！

// ✅ 正确
char *str = (char *)malloc(100);
// ... 使用 str ...
free(str);  // 释放
str = (char *)malloc(200);  // 重新分配
// ... 使用 str ...
free(str);
```

## 十、字符串拼接的 O(n²) 问题

```c
// ❌ 低效
char result[1000] = "";
for (int i = 0; i < 1000; i++) {
    strcat(result, "a");  // 每次 O(n)，总计 O(n²)
}

// ✅ 高效
char result[1001];
char *end = result;
for (int i = 0; i < 1000; i++) {
    *end++ = 'a';
}
*end = '\0';
```

## 十一、总结清单

> **常见陷阱速查：**
> 1. 字符数组构造字符串时忘记 `\0`
> 2. `strncpy` 不保证添加 `\0`
> 3. 缓冲区溢出：使用安全函数或手动检查大小
> 4. 指针未初始化就使用
> 5. 返回局部变量的地址
> 6. 修改字符串字面量（未定义行为）
> 7. `sizeof` 和 `strlen` 混淆
> 8. 用 `==` 比较字符串
> 9. 循环中重复调用 `strlen`
> 10. 忽略 `malloc` 返回值导致 NULL 解引用
