# 19 - 字符串匹配算法

## 一、暴力匹配算法（Brute Force）

### 1.1 基本思想

逐个字符比较模式串与文本串的每个可能位置。

```c
#include <stdio.h>
#include <string.h>

// 暴力匹配：返回模式串在文本中首次出现的位置
// 未找到返回 -1
int brute_force(const char *text, const char *pattern) {
    int n = (int)strlen(text);
    int m = (int)strlen(pattern);

    for (int i = 0; i <= n - m; i++) {
        int j;
        for (j = 0; j < m; j++) {
            if (text[i + j] != pattern[j]) {
                break;
            }
        }
        if (j == m) {
            return i;  // 找到匹配
        }
    }
    return -1;  // 未找到
}

int main() {
    const char *text = "ABABDABACDABABCABAB";
    const char *pattern = "ABABCABAB";

    int pos = brute_force(text, pattern);
    if (pos >= 0) {
        printf("找到模式串在位置: %d\n", pos);
        printf("%s\n", text);
        for (int i = 0; i < pos; i++) printf(" ");
        printf("%s\n", pattern);
    }

    return 0;
}
```

### 1.2 查找所有匹配

```c
#include <stdio.h>
#include <string.h>

void find_all(const char *text, const char *pattern) {
    int n = (int)strlen(text);
    int m = (int)strlen(pattern);

    printf("在文本中查找 '%s':\n", pattern);
    for (int i = 0; i <= n - m; i++) {
        int j;
        for (j = 0; j < m; j++) {
            if (text[i + j] != pattern[j]) break;
        }
        if (j == m) {
            printf("  匹配位置: %d\n", i);
        }
    }
}
```

### 1.3 时间复杂度

```
最好情况: O(n)   - 第一个字符就不匹配
最坏情况: O(n*m) - 每次都匹配到模式串末尾才失败
平均情况: O(n*m)
```

## 二、KMP 算法

### 2.1 核心思想

KMP（Knuth-Morris-Pratt）利用已匹配的信息，避免重复比较。当出现不匹配时，不回退文本指针，而是根据"部分匹配表"（next数组/失败函数）跳过不可能匹配的位置。

### 2.2 构建 next 数组

```c
#include <stdio.h>
#include <string.h>
#include <stdlib.h>

// 构建 next 数组（部分匹配表）
// next[i] 表示 pattern[0..i] 这个子串中，
// 最长的相等的"真前缀"和"真后缀"的长度
void build_next(const char *pattern, int *next, int m) {
    next[0] = -1;
    int j = -1;

    for (int i = 1; i < m; i++) {
        while (j >= 0 && pattern[j + 1] != pattern[i]) {
            j = next[j];
        }
        if (pattern[j + 1] == pattern[i]) {
            j++;
        }
        next[i] = j;
    }
}
```

**next 数组示例：**

```
pattern:  A  B  A  B  C  A  B  A  B
next:    -1 -1  0  1 -1  0  1  2  3

解读 next[8] = 3:
  pattern[0..8] = "ABABCABAB"
  最长相等前后缀: "ABA" (长度3)
  前缀: "ABA", 后缀: "ABA"
```

### 2.3 KMP 搜索实现

```c
// KMP 搜索
int kmp_search(const char *text, const char *pattern) {
    int n = (int)strlen(text);
    int m = (int)strlen(pattern);

    if (m == 0) return 0;

    // 构建 next 数组
    int *next = (int *)malloc(m * sizeof(int));
    build_next(pattern, next, m);

    int j = -1;  // 模式串的索引
    for (int i = 0; i < n; i++) {
        while (j >= 0 && pattern[j + 1] != text[i]) {
            j = next[j];  // 回退
        }
        if (pattern[j + 1] == text[i]) {
            j++;
        }
        if (j == m - 1) {
            free(next);
            return i - m + 1;  // 找到匹配
        }
    }

    free(next);
    return -1;  // 未找到
}
```

### 2.4 KMP 示例运行

```
text:    A B A B D A B A C D A B A B C A B A B
pattern: A B A B C A B A B

i=0: A==A, j=0
i=1: B==B, j=1
i=2: A==A, j=2
i=3: B==B, j=3
i=4: D!=C, j=next[3]=1, 仍D!=B, j=next[1]=-1
     B==B, j=0  (重新从头比较)
...继续直到找到匹配
```

### 2.5 KMP 的时间复杂度

```
构建 next 数组: O(m)
搜索: O(n)
总时间复杂度: O(n + m)

比暴力算法 O(n*m) 好很多（当 m 较大时）
```

## 三、其他匹配算法简介

### 3.1 Boyer-Moore 算法（思想）

```c
// Boyer-Moore 的核心思想：从右向左比较
// 利用两个启发规则跳过不可能的位置
// 实际应用中通常比 KMP 更快

// 坏字符规则：当发现不匹配时，根据文本中的"坏字符"
// 在模式串中的位置来决定跳过多少字符

// 好后缀规则：利用已匹配的后缀来决定跳过距离
```

### 3.2 Sunday 算法

```c
// Sunday 算法：比 Boyer-Moore 更简单
// 比较时从左向右，关注的是文本串中模式串末尾的下一个字符

#include <stdio.h>
#include <string.h>

#define CHAR_SIZE 256

int sunday_search(const char *text, const char *pattern) {
    int n = (int)strlen(text);
    int m = (int)strlen(pattern);

    // 构建偏移表
    int shift[CHAR_SIZE];
    for (int i = 0; i < CHAR_SIZE; i++) {
        shift[i] = m + 1;  // 默认跳过整个模式串+1
    }
    for (int i = 0; i < m; i++) {
        shift[(unsigned char)pattern[i]] = m - i;
    }

    int i = 0;
    while (i <= n - m) {
        int j;
        for (j = 0; j < m; j++) {
            if (text[i + j] != pattern[j]) break;
        }
        if (j == m) return i;  // 找到

        // 跳过：查看 text[i+m] 的偏移
        if (i + m >= n) break;
        i += shift[(unsigned char)text[i + m]];
    }
    return -1;
}
```

## 四、使用 strstr 的实际匹配

```c
#include <stdio.h>
#include <string.h>

// 查找所有匹配位置
void find_all_occurrences(const char *text, const char *pattern) {
    const char *pos = text;
    int count = 0;

    while ((pos = strstr(pos, pattern)) != NULL) {
        printf("匹配 #%d: 位置 %ld\n", ++count, pos - text);
        pos++;  // 移动一位继续查找（支持重叠匹配）
    }

    if (count == 0) {
        printf("未找到匹配\n");
    }
}
```

## 五、算法对比

| 算法 | 时间复杂度 | 空间复杂度 | 特点 |
|------|-----------|-----------|------|
| 暴力 | O(n*m) | O(1) | 简单，小规模数据够用 |
| KMP | O(n+m) | O(m) | 稳定，理论最优 |
| Boyer-Moore | O(n/m)最好 | O(m+σ) | 实际最快 |
| Sunday | O(n/m)最好 | O(σ) | 实现简单，接近BM |

## 六、重要注意事项

> **关键要点：**
> 1. **小规模数据用 `strstr` 足够**：标准库函数，简洁高效
> 2. **KMP 适合教学和竞赛**：理解其思想更重要
> 3. **实际项目中使用 `strstr` 或正则表达式**：除非有特殊性能需求
> 4. **注意边界条件**：空字符串、模式串比文本串长等情况
> 5. **重叠匹配与非重叠匹配**：匹配后指针移动位置不同
