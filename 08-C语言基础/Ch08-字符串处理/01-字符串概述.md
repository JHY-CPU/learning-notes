# 01 - 字符串概述

## 一、C语言中的字符串本质

C语言中**没有内置的字符串类型**。字符串在C语言中是通过**字符数组**加上一个特殊的**空终止符 `\0`** 来表示的。

### 1.1 字符串的内存模型

```
字符串 "Hello" 在内存中的存储：

地址:  0x1000 0x1001 0x1002 0x1003 0x1004 0x1005
内容:   'H'    'e'    'l'    'l'    'o'    '\0'
索引:    0      1      2      3      4      5
```

**关键概念：**
- 字符串以 `\0`（ASCII值为0）结尾
- `\0` 是字符串的终止标志，不是字符串内容的一部分
- 字符串的"长度"不包含 `\0`
- 占用的内存空间 = 字符个数 + 1（`\0` 占一个字节）

### 1.2 字符串与字符数组的区别

```c
// 这是一个字符数组，不是字符串（缺少 \0）
char arr1[] = {'H', 'e', 'l', 'l', 'o'};

// 这是一个字符串（自动添加 \0）
char arr2[] = "Hello";

// sizeof(arr1) == 5
// sizeof(arr2) == 6  （多了一个 \0）
```

## 二、字符串字面量

### 2.1 什么是字符串字面量

字符串字面量（String Literal）是用双引号括起来的字符序列。

```c
"Hello, World!"     // 字符串字面量
""                  // 空字符串（只包含 \0）
"A"                 // 包含一个字符和 \0 的字符串
"你好"               // 多字节字符串（UTF-8编码下占用更多字节）
```

### 2.2 字符串字面量的特殊字符

```c
char *s1 = "Hello\nWorld";      // 换行符
char *s2 = "Hello\tWorld";      // 制表符
char *s3 = "Hello\"World\"";    // 双引号
char *s4 = "Hello\\World";      // 反斜杠
char *s5 = "Hello\0World";      // 注意：\0会截断字符串
```

### 2.3 字符串字面量的拼接

相邻的字符串字面量会自动拼接：

```c
char *msg = "Hello, "
            "World!";           // 等价于 "Hello, World!"

// 常用于长字符串的格式化
printf("Error: file '%s' "
       "not found at line %d\n", filename, lineno);
```

## 三、字符串的存储方式

### 3.1 字符数组存储（栈区）

```c
void func() {
    char str[20] = "Hello";     // 在栈上分配20字节
    // str 可以被修改
    str[0] = 'h';               // 合法：变成 "hello"
}
```

### 3.2 字符指针指向字面量（只读区）

```c
void func() {
    char *str = "Hello";        // str指向只读内存区
    // str[0] = 'h';            // 未定义行为！可能崩溃
}
```

### 3.3 动态分配（堆区）

```c
void func() {
    char *str = (char *)malloc(20 * sizeof(char));
    strcpy(str, "Hello");       // 在堆上存储字符串
    str[0] = 'h';               // 合法：可以修改
    free(str);                  // 必须释放
}
```

## 四、字符串的三种表示方式对比

| 方式 | 存储位置 | 可修改 | 需要手动管理内存 |
|------|----------|--------|------------------|
| `char str[] = "Hello"` | 栈/全局区 | 可以 | 否 |
| `char *str = "Hello"` | 只读数据区 | 不可以 | 否 |
| `char *str = malloc(...)` | 堆 | 可以 | 是（需free） |

## 五、字符串的基本特征

### 5.1 字符串长度

```c
#include <string.h>

char str[] = "Hello";
size_t len = strlen(str);       // 返回 5（不含 \0）
size_t size = sizeof(str);      // 返回 6（含 \0）
```

### 5.2 空字符串

```c
char empty1[] = "";             // 长度为0，占用1字节（只有\0）
char *empty2 = "";              // 指向只读区的空字符串
char empty3[10] = "";           // 长度为0，占用10字节
```

## 六、重要注意事项

> **关键要点：**
> 1. **永远不要忘记 `\0`**：手动构造字符串时，必须确保有终止符
> 2. **区分字符数组和字符串**：字符数组不一定是字符串，必须以 `\0` 结尾
> 3. **不要修改字符串字面量**：`char *p = "hello"; p[0]='H';` 是未定义行为
> 4. **使用 sizeof 和 strlen 要分清**：sizeof 返回数组大小，strlen 返回字符串长度
> 5. **缓冲区大小要足够**：存储N个字符的字符串至少需要 N+1 字节的空间

## 七、常见面试问题

**Q: 为什么C语言用 `\0` 作为字符串结尾，而不是存储长度？**

A: 历史原因。使用终止符可以节省存储长度的空间（在早期计算机中内存非常宝贵），并且使得字符串处理函数的实现更简单统一。但代价是获取字符串长度需要 O(n) 时间遍历。

**Q: `char str[5] = "Hello"` 会怎样？**

A: 这是不安全的。"Hello" 需要6字节（5个字符 + \0），但数组只有5字节。`\0` 会被截断或写入数组之外的内存，导致未定义行为。编译器通常会报错。
