# 15 - 字符串字面量

## 一、什么是字符串字面量

### 1.1 定义

字符串字面量（String Literal）是用双引号括起来的零个或多个字符组成的常量。

```c
"Hello, World!"    // 字符串字面量
""                 // 空字符串字面量
"A"                // 单字符字符串字面量（注意：不是字符常量 'A'）
```

### 1.2 字符串字面量的类型

```c
// C中：字符串字面量的类型是 char[]
// 但不应该修改它
// C++中：字符串字面量的类型是 const char[]

// 推荐：始终用 const char* 指向字符串字面量
const char *str = "Hello";
```

## 二、字符串字面量的存储位置

### 2.1 内存布局

```
程序内存布局：

+------------------+
|    代码区 (.text)  |  ← 程序指令
+------------------+
|  只读数据区 (.rodata)|  ← 字符串字面量存放在这
|  "Hello"          |
|  "World"          |
+------------------+
|  已初始化数据区    |  ← 全局/静态初始化变量
+------------------+
|  未初始化数据区    |  ← 全局/静态未初始化变量
+------------------+
|      堆 (heap)    |  ← malloc/free
|        ↓          |
|        ↑          |
|      栈 (stack)   |  ← 局部变量
+------------------+
```

### 2.2 验证存储位置

```c
#include <stdio.h>

int main() {
    const char *literal = "Hello";
    char array[] = "Hello";
    int local_var;

    printf("字面量地址: %p\n", (void *)literal);
    printf("数组地址:   %p\n", (void *)array);
    printf("局部变量:   %p\n", (void *)&local_var);

    // 通常：字面量地址 < 数组地址（栈）
    // 字面量地址在只读数据区
    // 数组地址在栈区

    return 0;
}
```

## 三、修改字符串字面量是未定义行为

### 3.1 问题演示

```c
#include <stdio.h>

int main() {
    char *str = "Hello";

    // ⚠️ 未定义行为！修改只读内存
    // str[0] = 'h';
    // 可能的结果：
    // 1. 程序崩溃（段错误）
    // 2. 修改成功（某些编译器允许）
    // 3. 静默失败
    // 4. 程序行为异常

    return 0;
}
```

### 3.2 编译器警告

```c
// GCC/Clang 警告：
// warning: initialization discards 'const' qualifier
//          from pointer target type

// 正确做法
const char *str = "Hello";  // 明确标记为 const
// str[0] = 'h';  // 编译错误！
```

## 四、字符串字面量的拼接

### 4.1 相邻字面量自动拼接

```c
#include <stdio.h>

int main() {
    // 编译时自动拼接
    char *msg = "Hello, "
                "World!";
    // 等价于: "Hello, World!"

    // 用途1：长字符串换行
    char *sql = "SELECT id, name, age "
                "FROM users "
                "WHERE age > 18 "
                "ORDER BY name";

    // 用途2：格式化长文本
    printf("Error: file '%s' not found.\n"
           "Please check the path and try again.\n"
           "Current directory: %s\n",
           "config.txt", "/home/user");

    return 0;
}
```

### 4.2 与宏结合

```c
#include <stdio.h>

#define PREFIX "INFO: "
#define HELLO "Hello"
#define WORLD "World"

int main() {
    // 编译时拼接
    char *msg = PREFIX HELLO " " WORLD;
    printf("%s\n", msg);  // INFO: Hello World

    return 0;
}
```

## 五、字符串字面量的去重

### 5.1 编译器优化

```c
#include <stdio.h>

int main() {
    const char *s1 = "Hello";
    const char *s2 = "Hello";

    // 编译器可能将相同的字面量合并
    if (s1 == s2) {
        printf("指向同一地址（编译器优化）\n");
    } else {
        printf("指向不同地址\n");
    }
    // 这是实现定义的行为，不应依赖

    return 0;
}
```

### 5.2 不应依赖字面量去重

```c
// ❌ 错误：不应该比较指针来判断字符串相等
if (s1 == s2) {  // 不可靠！

// ✅ 正确：使用 strcmp
if (strcmp(s1, s2) == 0) {  // 可靠
```

## 六、特殊字符串字面量

### 6.1 宽字符串字面量

```c
#include <wchar.h>

// L 前缀：宽字符串（wchar_t）
const wchar_t *wide = L"Hello";

// u 前缀：UTF-16（char16_t，C11）
const char16_t *utf16 = u"Hello";

// U 前缀：UTF-32（char32_t，C11）
const char32_t *utf32 = U"Hello";

// u8 前缀：UTF-8（char，C11）
const char *utf8 = u8"Hello";
```

### 6.2 原始字符串（C23草案）

```c
// C23 可能支持 R"(...)" 语法
// 目前需要用转义
const char *path = "C:\\Users\\file.txt";  // 需要转义反斜杠
```

## 七、最佳实践

### 7.1 总是使用 const

```c
// ✅ 推荐：明确标记为 const
const char *msg = "Hello";

// ❌ 不推荐：丢失 const 限定符
char *msg = "Hello";

// 更好：使用数组代替指针（如果需要修改）
char msg[] = "Hello";
msg[0] = 'h';  // 合法
```

### 7.2 使用枚举或宏定义常量

```c
// ✅ 使用宏
#define ERROR_MSG "An error occurred"
#define MAX_PATH_LEN 260

// ✅ 使用 const 变量
static const char *DEFAULT_NAME = "Unknown";

// ✅ 使用枚举（数值常量）
enum { BUFFER_SIZE = 1024 };
```

## 八、重要注意事项

> **关键要点：**
> 1. **字符串字面量存储在只读区**：修改是未定义行为
> 2. **始终用 `const char*` 指向字面量**
> 3. **相邻字面量在编译时自动拼接**
> 4. **相同字面量可能被合并**（实现定义）
> 5. **字面量的生命周期是整个程序运行期**：可以安全返回其地址
> 6. **区分 `"A"`（字符串）和 `'A'`（字符）**：类型和值都不同
