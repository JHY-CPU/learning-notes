# 02 - 字符串的定义与初始化

## 一、字符数组方式定义字符串

### 1.1 最基本的初始化方式

```c
#include <stdio.h>

int main() {
    // 方式1：字符串字面量初始化（自动添加 \0）
    char str1[] = "Hello";

    // 方式2：逐个字符初始化（需手动添加 \0）
    char str2[] = {'H', 'e', 'l', 'l', 'o', '\0'};

    // 方式3：指定数组大小，字符串初始化（多余部分补 \0）
    char str3[10] = "Hello";
    // 内存: H e l l o \0 \0 \0 \0 \0

    // 方式4：指定大小，逐个字符初始化
    char str4[10] = {'H', 'e', 'l', 'l', 'o', '\0'};

    printf("str1: %s, len=%zu, size=%zu\n", str1, strlen(str1), sizeof(str1));
    printf("str2: %s, len=%zu, size=%zu\n", str2, strlen(str2), sizeof(str2));
    printf("str3: %s, len=%zu, size=%zu\n", str3, strlen(str3), sizeof(str3));

    return 0;
}
```

**输出：**
```
str1: Hello, len=5, size=6
str2: Hello, len=5, size=6
str3: Hello, len=5, size=10
```

### 1.2 不完全初始化

```c
// 部分初始化：剩余位置自动填充 \0
char str1[20] = "Hello";
// str1[0]='H', str1[1]='e', ..., str1[5]='\0', str1[6]='\0', ...

// 部分字符初始化（注意：这不是字符串！缺少 \0）
char str2[5] = {'H', 'e', 'l', 'l', 'o'};
// str2 不是以 \0 结尾，不能作为字符串使用

// 正确做法：预留空间给 \0
char str3[6] = {'H', 'e', 'l', 'l', 'o', '\0'};
```

### 1.3 未初始化的字符数组

```c
// 未初始化的局部数组：内容不确定（垃圾值）
char str1[20];
// str1 的内容是随机的，不能直接当字符串使用

// 全局或 static 数组：自动初始化为 \0
char str2[20];  // 全局变量，所有元素为 \0
```

## 二、字符指针方式定义字符串

### 2.1 指向字符串字面量

```c
char *str1 = "Hello, World!";

// 等价写法
char *str2;
str2 = "Hello, World!";

// 注意：指针可以重新指向
str1 = "New String";    // 合法：指针指向新的字面量
// str1[0] = 'h';       // 非法！不能修改字面量
```

### 2.2 指针与数组的区别

```c
// 数组方式
char arr[] = "Hello";
// sizeof(arr) == 6  （数组大小）
// arr 是一个常量地址，不能被重新赋值
// arr = "World";     // 编译错误

// 指针方式
char *ptr = "Hello";
// sizeof(ptr) == 8 (64位系统，指针大小)
// ptr 可以重新指向
ptr = "World";         // 合法
```

### 2.3 内存布局对比

```c
char arr[] = "Hello";   // 栈上分配6字节，内容可修改
char *ptr = "Hello";    // 指针在栈上(8字节)，
                        // "Hello"在只读数据区

/*
  内存布局：

  栈区：
  arr: [H][e][l][l][o][\0]    ← 6字节数组
  ptr: [地址] ────────────┐    ← 8字节指针
                          │
  只读数据区：              │
  [H][e][l][l][o][\0] ←──┘    ← 字符串字面量
*/
```

## 三、各种初始化方式详解

### 3.1 完整对比表

```c
#include <stdio.h>
#include <string.h>

int main() {
    // ① 自动推断大小
    char s1[] = "Hello";
    // sizeof(s1) = 6, 可修改

    // ② 指定大小（大于字符串需要的空间）
    char s2[20] = "Hello";
    // sizeof(s2) = 20, strlen(s2) = 5, 可修改

    // ③ 指定大小（刚好够用）
    char s3[6] = "Hello";
    // sizeof(s3) = 6, 可修改，但没有多余空间

    // ④ 指针指向字面量
    char *s4 = "Hello";
    // sizeof(s4) = 8(指针大小), 不可修改内容

    // ⑤ 动态分配
    char *s5 = (char *)malloc(20);
    strcpy(s5, "Hello");
    // 可修改，需手动 free

    // ⑥ calloc 初始化（全部为0）
    char *s6 = (char *)calloc(20, sizeof(char));
    // strlen(s6) = 0, 20字节全部为 \0

    free(s5);
    free(s6);
    return 0;
}
```

### 3.2 空字符串的不同写法

```c
char e1[10] = "";       // 10字节，全为\0，strlen=0
char e2[] = "";         // 1字节，只有\0
char *e3 = "";          // 指向只读区的空字符串
char e4[10];            // 未初始化，内容不确定
memset(e4, 0, 10);     // 手动清零
```

### 3.3 二维字符数组（字符串数组）

```c
// 方式1：二维数组
char names[5][20] = {
    "Alice",
    "Bob",
    "Charlie",
    "David",
    "Eve"
};

// 访问
printf("%s\n", names[0]);       // "Alice"
printf("%c\n", names[2][3]);    // 'r'

// 方式2：指针数组
char *names2[] = {
    "Alice",
    "Bob",
    "Charlie",
    "David",
    "Eve"
};
// sizeof(names)  = 5*20 = 100（固定每行20字节）
// sizeof(names2) = 5*8  = 40（64位，5个指针）
```

## 四、初始化的最佳实践

### 4.1 推荐写法

```c
// ✅ 推荐：数组方式，内容可修改
char name[64] = "Default";

// ✅ 推荐：const 指针，明确表示不可修改
const char *msg = "Hello, World!";

// ✅ 推荐：动态分配，灵活控制大小
char *buf = (char *)malloc(BUFFER_SIZE);
if (buf) {
    buf[0] = '\0';  // 初始化为空字符串
}

// ✅ 使用宏定义大小，便于管理
#define MAX_NAME_LEN 64
char username[MAX_NAME_LEN] = {0};
```

### 4.2 不推荐的写法

```c
// ❌ 不推荐：不加const的字面量指针
char *str = "Hello";    // 应该用 const char *

// ❌ 不推荐：数组大小刚好等于字符串
char name[5] = "John";  // 没有空间修改或扩展

// ❌ 不推荐：未初始化就使用
char buf[100];
printf("%s", buf);      // 未定义行为！

// ❌ 不推荐：指针未指向有效内存
char *ptr;
strcpy(ptr, "Hello");   // 崩溃！ptr未初始化
```

## 五、重要注意事项

> **关键要点：**
> 1. 数组初始化的字符串**可以修改**，指针指向的字面量**不可修改**
> 2. `char arr[] = "text"` 编译器自动计算大小并添加 `\0`
> 3. `char arr[n] = "text"` 若 n 不够大，编译器可能报错或截断
> 4. 指针可以重新赋值，数组名是常量不能重新赋值
> 5. 二维数组每行固定大小，指针数组每个字符串大小独立
> 6. 动态分配的字符串必须手动释放，避免内存泄漏
> 7. 养成初始化的习惯：`char buf[100] = {0};` 比 `char buf[100];` 安全
