命令式与声明式编程范式

一、编程范式概述

编程范式是一种编程的思维方式和方法论，决定了如何组织代码和表达计算。

范式核心问题代表语言
命令式"怎么做" (How)C, Java, Python
声明式"做什么" (What)SQL, HTML, Prolog
函数式"是什么" (What is)Haskell, Lisp, ML
逻辑式"什么是真的" (What is true)Prolog
面向对象"谁做什么" (Who does what)Java, C++, Python

二、命令式 vs 声明式

命令式 (Imperative)

  • 描述计算的步骤序列
  • 修改可变状态
  • 使用循环、赋值、副作用
  • 接近底层硬件执行模型

声明式 (Declarative)

  • 描述计算的目标/结果
  • 不描述执行步骤
  • 使用表达式、约束、关系
  • 更接近人类思维方式

对比示例：求列表中偶数的平方和

命令式 (Python)：

nums = [1, 2, 3, 4, 5, 6]
result = 0
for x in nums:
    if x % 2 == 0:
        result += x * x
print(result)  # 56

声明式 (Haskell)：

nums = [1, 2, 3, 4, 5, 6]
result = sum [x^2 | x <- nums, even x]
-- 或者：result = sum . map (^2) . filter even $ nums

声明式 (SQL)：

SELECT SUM(value * value)
FROM numbers
WHERE value % 2 = 0;

三、函数式编程范式

函数式编程是声明式范式的重要分支。

核心原则

原则含义好处
纯函数相同输入始终产生相同输出，无副作用可测试、可推理
不可变性数据不可修改，只创建新数据线程安全、无竞态
一等函数函数可作为参数和返回值高阶抽象
高阶函数操作其他函数的函数代码复用、组合

核心操作

-- map：对每个元素应用函数
map (*2) [1,2,3] = [2,4,6]

-- filter：筛选满足条件的元素
filter even [1,2,3,4,5] = [2,4]

-- fold/reduce：归约为单个值
foldl (+) 0 [1,2,3,4] = 10

-- 函数组合
(f . g) x = f(g(x))
-- 管道风格
result = nums |> filter even |> map square |> sum

四、逻辑编程范式 (Prolog)

逻辑编程是"告诉计算机什么是真的，让它自己推理"。

Prolog示例

% 事实 (Facts)
parent(tom, bob).
parent(tom, liz).
parent(bob, ann).
parent(bob, pat).

% 规则 (Rules)
grandparent(X, Z) :- parent(X, Y), parent(Y, Z).
sibling(X, Y) :- parent(Z, X), parent(Z, Y), X \= Y.

% 查询 (Queries)
?- grandparent(tom, ann).    % true (Tom是Ann的祖父)
?- grandparent(tom, Who).    % Who = ann ; Who = pat
?- sibling(bob, liz).        % true (Bob和Liz是兄妹)

程序员定义知识库（事实+规则），Prolog通过回溯搜索和合一自动求解。

五、范式对比总结

特性命令式函数式逻辑式
计算模型状态机函数归约逻辑推理
核心抽象变量、赋值函数、表达式事实、规则
控制流显式（循环、分支）递归、高阶函数回溯搜索
可变状态核心特性禁止/受限受限
并行化困难天然并行可并行搜索
可推理性中等高（等式推理）高（逻辑推理）
适用领域系统编程、通用数据处理、DSLAI、专家系统

六、现代语言中的范式融合

现代语言通常支持多范式：

// Python - 命令式 + 函数式 + 面向对象
# 命令式
result = []
for x in range(10):
    if x % 2 == 0:
        result.append(x ** 2)

# 函数式
result = [x**2 for x in range(10) if x % 2 == 0]
result = list(map(lambda x: x**2, filter(lambda x: x % 2 == 0, range(10))))

// Scala - OOP + FP
val result = (1 to 10).filter(_ % 2 == 0).map(x => x * x).sum

// Rust - 命令式 + 函数式
let result: i32 = (1..=10)
    .filter(|x| x % 2 == 0)
    .map(|x| x * x)
    .sum();

趋势：即使传统命令式语言也在吸收函数式特性（Java 8+ Stream API, C# LINQ, Python comprehensions, JavaScript Array methods）。
