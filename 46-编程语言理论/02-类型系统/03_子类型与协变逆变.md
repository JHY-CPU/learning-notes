# 子类型与协变逆变


Subtyping, Covariance & Contravariance — 子类型关系、变型注解与里氏替换

1. 子类型关系

## 1. 子类型 (Subtyping)


如果类型 `S` 是类型 `T` 的子类型，记为 `S <: T`，则任何期望 T 的上下文都可以安全地使用 S。


```
类型层次 (Animal继承体系):

         Animal (T)
        /        \
     Cat (S₁)   Dog (S₂)
      |
  Persian (S₃)

  Cat <: Animal  (Cat 是 Animal 的子类型)
  Persian <: Cat <: Animal  (传递性)
```


```
// Java子类型示例
class Animal { void speak() {} }
class Cat extends Animal { void purr() {} }

// Cat <: Animal, 所以Cat可以用在Animal的位置
Animal a = new Cat();   // ✓ 合法
// 反过来不行:
Cat c = new Animal();   // ✗ 编译错误
```

2. Liskov替换原则

## 2. 里氏替换原则 (Liskov Substitution Principle)


Barbara Liskov于1987年提出：子类型必须能够替换父类型而不破坏程序的正确性。


### 正式定义


若对类型 `T` 的每个对象 `o₁`，都存在类型 `S` 的对象 `o₂`，使得对所有程序 `P`，当 `P` 用 `o₁` 为参数时行为不变，则 `S` 是 `T` 的子类型。


### 前置条件与后置条件


- 前置条件不能强化
   — 子类型不能要求比父类型更多的前提
- 后置条件不能弱化
   — 子类型不能提供比父类型更少的保证
- 不变量必须保持
   — 父类型的不变量必须被子类型保持


```
// 违反LSP的反例
class Rectangle {
    protected width, height;
    void setWidth(w) { width = w; }
    void setHeight(h) { height = h; }
    int area() { return width * height; }
}

class Square extends Rectangle {
    // 正方形需要宽高相等
    void setWidth(w) { width = height = w; }  // 违反了父类契约!
}

// 客户端代码期望:
void test(Rectangle r) {
    r.setWidth(5);
    r.setHeight(2);
    assert(r.area() == 10);  // 传入Square时 area=4, 断言失败!
}
// Square不是Rectangle的合法子类型
```

3. 协变 (Covariance)

## 3. 协变 (Covariance)


协变意味着子类型关系与类型构造器同向保持：若 `S <: T`，则 `F<S> <: F<T>`。


```
协变 (Covariant):

  Cat <: Animal          数组/容器也保持同向
    ↓                       ↓
  F<Cat> <: F<Animal>    Cat[] <: Animal[]  ✓
```


```
// Java数组是协变的
Cat[] cats = {new Cat()};
Animal[] animals = cats;  // ✓ 编译通过 (协变)
// 但运行时可能出问题!
animals[0] = new Dog();  // 编译通过, 但运行时ArrayStoreException!

// Java泛型默认不变 (Invariant)
List<Cat> catList = new ArrayList<Cat>();
// List<Animal> animalList = catList;  // ✗ 编译错误!

// 用通配符实现协变 (Java)
List<? extends Animal> animals = catList;  // ✓ 上界通配符
Animal a = animals.get(0);   // ✓ 可以读取为Animal
// animals.add(new Cat());    // ✗ 不能写入!

// Kotlin协变声明
interface Source<out T> {   // out表示协变
    fun next(): T            // T只能出现在输出位置
}
val cats: Source<Cat> = ...
val animals: Source<Animal> = cats  // ✓ 协变允许
```

4. 逆变 (Contravariance)

## 4. 逆变 (Contravariance)


逆变意味着子类型关系与类型构造器反向保持：若 `S <: T`，则 `F<T> <: F<S>`。


```
逆变 (Contravariant):

  Cat <: Animal          函数参数反向!
    ↓                       ↓
  F<Animal> <: F<Cat>    (Animal → R) <: (Cat → R)  ✓
```


```
// 函数参数是逆变的
type Handler<T> = (event: T) => void;

// 能处理Animal的处理器, 一定能处理Cat
// 所以 Handler<Animal> <: Handler<Cat>
const animalHandler: Handler<Animal> = (a) => a.speak();
const catHandler: Handler<Cat> = animalHandler;  // ✓ 逆变允许

// TypeScript逆变 (函数参数)
type AnimalFunc = (a: Animal) => void;
type CatFunc = (c: Cat) => void;

let f: AnimalFunc = (a) => {};
let g: CatFunc = f;  // ✓ AnimalFunc <: CatFunc (逆变)

// Kotlin逆变声明
interface Comparable<in T> {  // in表示逆变
    fun compareTo(other: T): Int  // T只能出现在输入位置
}
val animalComparer: Comparable<Animal> = ...
val catComparer: Comparable<Cat> = animalComparer  // ✓ 逆变
```

5. 不变 (Invariance)

## 5. 不变 (Invariance)


不变意味着即使 `S <: T`，`F<S>` 与 `F<T>` 也没有子类型关系。


```
// Java泛型默认不变
List<Cat> cats = new ArrayList<>();
// List<Animal> animals = cats;  // ✗ 编译错误! 不变
// List<Object> objects = cats;  // ✗ 编译错误! 不变

// 不变是安全的默认值
// 既不能从List<Cat>读取为Animal,
// 也不能向List<Cat>写入Animal
```

6. PECS原则

## 6. PECS 原则 (Producer-Extends, Consumer-Super)


Java中选择协变还是逆变的实用规则：


| 角色 | 通配符 | 方向 | 原因 |
| --- | --- | --- | --- |
| 生产者 (只读) | `? extends T` | 协变 | 产出的值至少是T |
| 消费者 (只写) | `? super T` | 逆变 | 能接收至少T类型的值 |


```
// PECS 实践

// 生产者: 从list中读取 → 协变 (extends)
public static <T> T first(List<? extends T> list) {
    return list.get(0);    // ✓ 可以读为T
    // list.add(something); // ✗ 不能写入
}

// 消费者: 向list中写入 → 逆变 (super)
public static <T> void addAll(List<? super T> list, T item) {
    list.add(item);        // ✓ 可以写入T
    // T x = list.get(0);   // ✗ 只能读为Object
}

// Collections.copy的签名
public static <T> void copy(
    List<? super T> dest,   // 消费者 → super (逆变)
    List<? extends T> src   // 生产者 → extends (协变)
)
```

7. 双变 (Bivariance)

## 7. 双变 (Bivariance)


某些情况下类型同时是协变和逆变的，通常是类型不安全的捷径。


```
// TypeScript中 any 和 never 是双变的
let x: any = 42;
x = "hello";  // any兼容所有类型

// TypeScript strictFunctionTypes下
// 回调参数是逆变的
// 关闭strictFunctionTypes时, 回调参数是双变的 (不安全)

// Rust中fn指针默认双变 (为了兼容性)
fn animal_handler(a: &Animal) {}
let f: fn(&Animal) = animal_handler;
// fn指针在参数和返回值上都是双变的
```

8. 总结

## 8. 变型规则总结


| 变型 | 规则 | 符号 | 典型场景 |
| --- | --- | --- | --- |
| 协变 | S<:T ⇒ F<S><:F<T> | `out` / `? extends` / `+` | 只读容器、返回值、生产者 |
| 逆变 | S<:T ⇒ F<T><:F<S> | `in` / `? super` / `-` | 函数参数、写入容器、消费者 |
| 不变 | F<S> 与 F<T> 无关系 | 无修饰 / 默认 | 可读写容器 (默认安全) |
| 双变 | 同时协变和逆变 | `*` | escape hatch, any |
