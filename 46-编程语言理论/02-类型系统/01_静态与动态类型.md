# 静态与动态类型


## 一、类型系统基础概念


**类型 (Type)** 是对值的集合及其可执行操作的描述。


**类型系统**是一组用于给程序构造分配类型的规则。


### 核心维度


| 维度 | 选项 | 含义 |
| --- | --- | --- |
| **检查时机** | 静态 vs 动态 | 编译时 vs 运行时检查 |
| **类型安全** | 强类型 vs 弱类型 | 是否允许隐式类型转换 |
| **类型声明** | 显式 vs 隐式（推断） | 程序员是否需要写类型注解 |


## 二、静态类型 vs 动态类型


### 静态类型 (Static Typing)


- 类型在
   **编译时**
   确定并检查
- 变量的类型在声明时固定
- 类型错误在运行前就被捕获
- 代表：C, Java, Haskell, Rust, TypeScript


> **Example:** ```
> // Java - 静态类型
> int x = 42;           // x 的类型固定为 int
> String s = "hello";   // s 的类型固定为 String
> x = "world";          // 编译错误！不能将String赋给int
> ```


### 动态类型 (Dynamic Typing)


- 类型在
   **运行时**
   确定和检查
- 变量可以持有不同类型的值
- 类型错误只在执行到相关代码时才被发现
- 代表：Python, JavaScript, Ruby, Lisp


> **Example:** ```
> # Python - 动态类型
> x = 42           # x 现在是 int
> x = "hello"      # x 现在是 str — 合法！
> x = [1, 2, 3]    # x 现在是 list — 合法！
> x + 1             # 运行时错误：list + int 不合法
> ```


### 对比表


| 特性 | 静态类型 | 动态类型 |
| --- | --- | --- |
| 类型检查时机 | 编译时 | 运行时 |
| 类型错误发现 | 早期（编译期） | 晚期（执行期） |
| 运行时开销 | 较小（已检查） | 较大（需要检查） |
| 灵活性 | 较低 | 较高 |
| IDE支持 | 优秀（自动补全、重构） | 有限（需运行确认） |
| 代码文档性 | 类型即文档 | 需额外文档 |
| 开发速度 | 初期较慢 | 初期较快 |
| 维护性 | 大型项目更易维护 | 大型项目易出错 |


## 三、强类型 vs 弱类型


### 强类型 (Strongly Typed)


不允许隐式的类型转换，类型转换必须**显式**进行。


```
# Python (强类型)
"hello" + 42     # TypeError! 不能隐式将int转为str
"hello" + str(42)  # "hello42" — 必须显式转换
```


### 弱类型 (Weakly Typed)


允许隐式的类型转换（强制类型转换 coercions）。


```
// JavaScript (弱类型)
"hello" + 42      // "hello42" — 自动将42转为字符串
"5" - 3           // 2 — 自动将"5"转为数字
true + true       // 2 — true被转为1
[] + {}           // "[object Object]"
```

静态 + 强类型


C, Java, Haskell, Rust
动态 + 强类型


Python, Ruby
静态 + 弱类型


C (指针转换), TypeScript(部分)
动态 + 弱类型


JavaScript, PHP

## 四、多态性 (Polymorphism)


多态允许一段代码处理多种类型的数据。


### 四种多态


| 类型 | 别名 | 机制 | 示例 |
| --- | --- | --- | --- |
| **参数多态** | 泛型 (Generics) | 类型参数化 | List<T> (Java) |
| **特设多态** | 重载 (Overloading) | 同名不同参 | print(int), print(str) |
| **子类型多态** | 继承多态 | 子类替代父类 | Animal a = new Dog() |
| **强制多态** | 类型转换 | 隐式/显式转换 | int → float |


### 泛型 (Generics) 示例


```
// Java 泛型
public class Box {
    private T value;
    public void set(T v) { this.value = v; }
    public T get() { return value; }
}

Box intBox = new Box<>();
intBox.set(42);          // T = Integer
Box strBox = new Box<>();
strBox.set("hello");     // T = String

// Haskell 参数多态
identity :: a -> a
identity x = x

length :: [a] -> Int
length [] = 0
length (_:xs) = 1 + length xs
```


## 五、类型推断简介


类型推断允许编译器自动推导类型，无需程序员显式声明。


```
// TypeScript - 类型推断
let x = 42;           // 推断 x: number
let s = "hello";      // 推断 s: string
let arr = [1, 2, 3];  // 推断 arr: number[]

// Rust - 类型推断
let x = 5;            // 推断 x: i32
let v = vec![1,2,3];  // 推断 v: Vec

// Haskell - Hindley-Milner 类型推断（最强）
-- map :: (a -> b) -> [a] -> [b]
-- 完全不需要类型注解，编译器自动推断！
```


> **Theorem:** **类型推断的意义：**结合静态类型的安全性和动态类型的简洁性——程序员不需要写类型注解，但编译器仍然在编译时进行类型检查。
