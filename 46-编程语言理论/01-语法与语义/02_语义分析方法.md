# 语义分析方法


## 一、语义分析概述


**语法**回答"程序写得对不对"，**语义**回答"程序做什么"。


三种主要的语义定义方法：


### 操作语义


通过**执行模型**定义含义


"程序做了什么"


### 指称语义


通过**数学函数**定义含义


"程序代表什么"


### 公理语义


通过**逻辑断言**定义含义


"程序保证什么"


## 二、操作语义 (Operational Semantics)


通过定义一个抽象机器及其状态转换规则来描述程序含义。


### 小步语义 (Small-Step)


每一步执行一个微小的计算，形式为 `⟨e, σ⟩ → ⟨e', σ'⟩`：


> **Example:** ```
> -- 小步语义规则示例
>
> -- 加法规则：如果e₁可进一步化简，先化简e₁
> ⟨e₁ + e₂, σ⟩ → ⟨e₁' + e₂, σ⟩    如果 ⟨e₁, σ⟩ → ⟨e₁', σ'⟩
>
> -- 加法计算：两个都是值时直接计算
> ⟨n₁ + n₂, σ⟩ → ⟨n₃, σ⟩    其中 n₃ = n₁ + n₂
>
> -- 赋值规则
> ⟨x := n, σ⟩ → ⟨skip, σ[x ↦ n]⟩
> ```


### 大步语义 (Big-Step)


直接给出表达式的最终结果，形式为 `⟨e, σ⟩ ⇓ ⟨v, σ'⟩`：


> **Example:** ```
> -- 大步语义规则
>
> -- 加法：直接求出两个子表达式的值再相加
> ⟨e₁, σ⟩ ⇓ ⟨n₁, σ'⟩    ⟨e₂, σ'⟩ ⇓ ⟨n₂, σ''⟩
> ───────────────────────────────────────────
> ⟨e₁ + e₂, σ⟩ ⇓ ⟨n₁ + n₂, σ''⟩
>
> -- 常量求值
> ⟨5, σ⟩ ⇓ ⟨5, σ⟩
> ```


| 特性 | 小步语义 | 大步语义 |
| --- | --- | --- |
| 粒度 | 单步执行 | 整体求值 |
| 适合 | 并发、交错执行 | 顺序求值 |
| 可描述终止 | 是 | 否（不终止时无规则） |
| 复杂度 | 较复杂 | 较简洁 |


## 三、指称语义 (Denotational Semantics)


将每个语言构造映射为精确的**数学对象**（通常是函数）。


### 定义方式


```
-- 指称语义函数 ⟦·⟧ : Exp → (Env → Value)

⟦n⟧ ρ = n                          -- 常量：就是自身
⟦x⟧ ρ = ρ(x)                      -- 变量：查环境
⟦e₁ + e₂⟧ ρ = ⟦e₁⟧ ρ + ⟦e₂⟧ ρ  -- 加法：语义函数的组合
⟦if e₁ then e₂ else e₃⟧ ρ =
    if ⟦e₁⟧ ρ ≠ 0 then ⟦e₂⟧ ρ else ⟦e₃⟧ ρ

-- 其中 ρ 是环境（变量名到值的映射）
```


### 核心思想


- 程序 = 函数（从输入到输出的映射）
- 语义 = 该函数的精确定义
- 使用
   **域理论**
   处理递归和不动点
- 语义函数 ⟦·⟧ 必须是
   **组合式**
   的（compositional）


> **Theorem:** **不动点语义：**递归程序 ⟦fix F⟧ = lfp(F)，即 F 的最小不动点。这需要域理论中的 Kleene 不动点定理。


## 四、公理语义 (Axiomatic Semantics)


通过逻辑断言（前后置条件）来定义程序行为。


### Hoare逻辑


三元组形式：`{P} S {Q}`


- **P**
   — 前置条件（执行前必须为真）
- **S**
   — 程序语句
- **Q**
   — 后置条件（执行后必定为真）


### Hoare规则


> **Example:** ```
> -- 赋值公理（最核心）
> {Q[e/x]} x := e {Q}
> -- 例：{x+1 > 0} x := x+1 {x > 0}
>
> -- 顺序组合
> {P} S₁ {Q}    {Q} S₂ {R}
> ──────────────────────────
> {P} S₁; S₂ {R}
>
> -- 条件语句
> {P ∧ b} S₁ {Q}    {P ∧ ¬b} S₂ {Q}
> ───────────────────────────────────────
> {P} if b then S₁ else S₂ {Q}
>
> -- 循环（需要循环不变量I）
> {I ∧ b} S {I}
> ───────────────────────
> {I} while b do S {I ∧ ¬b}
>
> -- 赋值规则示例
> {y + 1 > 0} x := y + 1 {x > 0}
> ```


### 循环不变量示例


> **Example:** **程序：**计算阶乘
>
>
> ```
> i := 1; f := 1;
> while i <= n do
>     f := f * i;
>     i := i + 1
> ```
>
>
> **循环不变量：**`I: f = (i-1)! ∧ i ≤ n+1`
>
>
> **验证：**初始时 I 成立（f=1=0!, i=1），每次循环后 I 保持，循环结束时得到 f = n!


## 五、三种语义方法对比


| 方面 | 操作语义 | 指称语义 | 公理语义 |
| --- | --- | --- | --- |
| **描述方式** | 执行步骤 | 数学函数 | 逻辑规则 |
| **核心概念** | 状态转换 | 语义域中的映射 | 前后置条件 |
| **适合** | 解释器实现 | 理论分析 | 程序验证 |
| **优点** | 直观、易实现 | 数学精确、组合式 | 可证明正确性 |
| **缺点** | 复杂性随语言增长 | 需要高级数学 | 需要循环不变量 |
| **代表学者** | Plotkin | Scott, Strachey | Hoare, Dijkstra |


## 六、类型检查作为语义分析


在编译器中，语义分析的一个重要任务是**类型检查**：


- 检查变量是否已声明
- 检查类型是否兼容（如不能将字符串赋给整数变量）
- 检查函数调用的参数类型和数量
- 推断表达式的类型


```
// 类型规则示例
// 如果 e₁ 是 Int 类型且 e₂ 是 Int 类型
// 那么 e₁ + e₂ 也是 Int 类型

Γ ⊢ e₁ : Int    Γ ⊢ e₂ : Int
──────────────────────────────
Γ ⊢ e₁ + e₂ : Int

// Γ 是类型环境（变量名 → 类型的映射）
```
