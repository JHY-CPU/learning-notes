作用域与闭包

Scope & Closures — 词法作用域、动态作用域、闭包实现与环境模型

1. 作用域基础

作用域 (Scope) 决定了变量名在程序文本的哪些区域可见和可引用。

绑定 (Binding) 与引用

  • 绑定 — 将名字与实体 (变量、函数、类型) 关联的过程
  • 绑定出现 (Binding Occurrence) — 创建绑定的位置 (如声明)
  • 引用 (Reference) — 使用已绑定名字的位置
  • 绑定域 (Scope of Binding) — 绑定有效的程序区域

2. 词法作用域 (Lexical/Static Scope)

变量的作用域由其在源代码中的位置决定，与运行时调用无关。是大多数现代语言采用的方案。

词法作用域 (嵌套函数): 全局作用域 ├── x = "global" └── func outer() ├── x = "outer" ← outer的局部变量 └── func inner() └── print(x) ← 查找x: 先找inner, 再找outer, 最后全局 "词法"决定了inner引用的是outer的x! 即使从外部调用inner, 它仍然引用定义时的outer的x 
// JavaScript 词法作用域
let x = "global";

function outer() {
    let x = "outer";
    function inner() {
        console.log(x);  // 引用outer的x, 不是global的x
    }
    return inner;
}

const fn = outer();
fn();  // 输出 "outer" — 即使outer已返回!
// inner记住了定义时的环境, 这就是闭包的基础

// Python 词法作用域
x = "global"
def outer():
    x = "outer"
    def inner():
        print(x)  # 引用outer的x
    return inner

fn = outer()
fn()  # 输出 "outer"

3. 动态作用域 (Dynamic Scope)

变量的作用域由运行时调用栈决定。查找变量时沿着调用链向上找，而非定义时的嵌套关系。

动态作用域 (调用栈): 调用栈: main → outer → inner print(x) 查找x: 先找inner的局部变量 → 无 再找调用者outer的局部变量 → 有! "outer" (不是定义时的环境, 而是调用时的环境) 
;; Common Lisp (动态作用域, 用 defvar/special)
(defvar *x* "global")

(defun inner ()
  *x*)                ;; 返回调用时 *x* 的值

(defun outer ()
  (let ((*x* "outer"))  ;; 动态绑定
    (inner)))

(inner)                ;; → "global"  (没有动态上下文)
(outer)                ;; → "outer"   (有动态上下文)

;; Bash也是动态作用域
x="global"
inner() { echo $x; }
outer() {
    local x="outer"
    inner             # → "outer" (动态作用域)
}
outer                 # inner在调用栈中找到outer的x

词法 vs 动态对比

特性词法作用域动态作用域
确定方式源代码位置 (编译时)调用栈 (运行时)
可预测性高 (看代码即可确定)低 (取决于调用路径)
优化能力强 (编译时可分析)弱 (需运行时查找)
典型语言JS, Python, C, JavaBash, Emacs Lisp, Perl(local)

4. 闭包 (Closure)

闭包 = 函数 + 定义时的词法环境。函数"记住"了它被创建时的作用域，即使在那个作用域外执行。

闭包的结构: ┌─────────────────────────────────┐ │ 闭包 (Closure) │ │ ┌───────────────────────────┐ │ │ │ 函数代码 (指令) │ │ │ │ function() { return x; }│ │ │ └───────────────────────────┘ │ │ ┌───────────────────────────┐ │ │ │ 环境指针 (自由变量绑定) │ │ │ │ x = "outer" │ │ │ └───────────────────────────┘ │ └─────────────────────────────────┘ 
// 闭包的经典示例: 计数器
function makeCounter() {
    let count = 0;  // 自由变量, 被闭包捕获
    return function() {
        count++;    // 引用外部的count
        return count;
    };
}

const counter1 = makeCounter();
counter1();  // 1
counter1();  // 2

const counter2 = makeCounter();
counter2();  // 1 (独立的闭包环境!)

// 闭包创建私有状态
function createPerson(name) {
    let _age = 0;  // 私有变量, 外部无法直接访问
    return {
        getName: () => name,
        getAge: () => _age,
        setAge: (a) => { _age = a; }
    };
}

5. let vs var — 块作用域 vs 函数作用域

JavaScript中 var 是函数作用域，let/const 是块作用域。

// var — 函数作用域
for (var i = 0; i < 3; i++) {
    setTimeout(() => console.log(i), 100);
}
// 输出: 3, 3, 3 — var没有块作用域!
// 所有回调共享同一个 i

// let — 块作用域 (每次循环创建新的绑定)
for (let i = 0; i < 3; i++) {
    setTimeout(() => console.log(i), 100);
}
// 输出: 0, 1, 2 — 每次迭代有独立的 i
// 原理: for循环的每次迭代都是隐式的块

// 暂时性死区 (Temporal Dead Zone, TDZ)
console.log(x);  // ReferenceError (let x声明前不可访问)
let x = 10;
// var不会有TDZ, 会被提升(hoisting)但值为undefined
console.log(y);  // undefined (var y被提升)
var y = 20;

6. 环境模型 (Environment Model)

环境是一个从变量名到值的映射链。闭包通过保存环境指针来实现。

环境链 (Environment Chain): 闭包调用时: ┌─────────────────┐ │ 局部环境 (inner) │ ──▶ { } (空) └────────┬────────┘ │ parent ┌────────▼────────┐ │ 外层环境 (outer) │ ──▶ { x: "outer" } └────────┬────────┘ │ parent ┌────────▼────────┐ │ 全局环境 │ ──▶ { x: "global", outer: fn, ... } └─────────────────┘ 查找 x: inner → outer找到 → 返回 "outer" 
// 环境模型的简化实现 (JavaScript)
class Environment {
    constructor(bindings = {}, parent = null) {
        this.bindings = bindings;
        this.parent = parent;
    }

    lookup(name) {
        if (name in this.bindings) {
            return this.bindings[name];
        }
        if (this.parent) {
            return this.parent.lookup(name);  // 沿链向上查找
        }
        throw new Error(`Undefined: ${name}`);
    }
}

// 模拟闭包行为
const globalEnv = new Environment({ x: "global" });
const outerEnv = new Environment({ x: "outer" }, globalEnv);
// 闭包保存了outerEnv的引用
outerEnv.lookup("x");  // "outer"
outerEnv.lookup("y");  // 沿链找到globalEnv

7. 闭包的应用与陷阱

常见应用

  • 数据封装 — 创建私有变量 (模块模式)
  • 柯里化 (Currying) — 分步接受参数
  • 回调与事件处理 — 捕获上下文
  • 函数工厂 — 生成配置化的函数
// 柯里化 (Currying) 用闭包实现
function add(a) {
    return function(b) {
        return a + b;  // 闭包捕获a
    };
}
add(1)(2);  // 3
const add5 = add(5);
add5(3);    // 8

// 模块模式 (Module Pattern)
const myModule = (function() {
    let privateVar = 0;  // 私有状态
    return {
        increment() { privateVar++; },
        getCount() { return privateVar; }
    };
})();

// ⚠️ 经典陷阱: 循环中的闭包
// 错误 (var)
var funcs = [];
for (var i = 0; i < 3; i++) {
    funcs.push(function() { return i; });
}
funcs[0]();  // 3 (不是0!) — 所有闭包共享同一个i

// 修复方案1: 用let
for (let i = 0; i < 3; i++) { ... }

// 修复方案2: IIFE创建新作用域
for (var i = 0; i < 3; i++) {
    (function(j) {
        funcs.push(function() { return j; });
    })(i);
}
funcs[0]();  // 0 ✓
