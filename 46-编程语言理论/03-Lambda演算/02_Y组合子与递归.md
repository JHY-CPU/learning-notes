# Y组合子与递归


## 一、λ演算中的递归困境


在λ演算中，函数是**匿名**的——函数体内无法引用自己的名字。如何实现递归？


> **Example:** ### 问题：如何在λ演算中定义阶乘？
>
>
> ```
> -- 普通递归定义（需要名字引用自己）：
> fact(n) = if n==0 then 1 else n * fact(n-1)
>
> -- λ演算不能这样写！
> -- λn. if (iszero n) 1 (mult n (???))   -- ??? 处没有名字可用！
> ```


**解决方案：**不动点组合子 (Fixed-Point Combinator)。


## 二、不动点 (Fixed Point)


函数 f 的**不动点**是满足 `f(x) = x` 的 x。


### 数学例子


- cos(x) 的不动点 ≈ 0.739（cos(0.739) ≈ 0.739）
- f(x) = x² 的不动点：0 和 1


> **Theorem:** **关键洞察：**阶乘函数 fact 是某个"泛函" F 的不动点：
>
>
> ```
> F = λf. λn. if (iszero n) 1 (mult n (f (pred n)))
>
> -- F 的不动点就是阶乘函数！
> -- fact = F(fact)
> -- 即 fact n = if n==0 then 1 else n * fact(n-1)
> ```
>
>
> 问题转化为：**如何找到任意函数的不动点？**


## 三、Y组合子


Y组合子是高阶函数，使得对任意 f，有 `Y f = f (Y f)`（即 Y f 是 f 的不动点）。


### 标准Y组合子（正则序）


```
Y = λf. (λx. f (x x)) (λx. f (x x))
```


### 验证：Y f → f (Y f)


Y f


= (λf. (λx. f (x x)) (λx. f (x x))) f


→~β~ (λx. f (x x)) (λx. f (x x))


→~β~ f ((λx. f (x x)) (λx. f (x x)))


= f (Y f) — 正是 f 的不动点！


### 用Y定义阶乘


```
-- F 是阶乘泛函
F = λf. λn. if (iszero n) 1 (mult n (f (pred n)))

-- fact = Y F 是不动点
-- fact = F(fact) = F(F(fact)) = ...

-- 展开 fact 3：
fact 3 = F(fact) 3
       = if (iszero 3) 1 (mult 3 (fact (pred 3)))
       = mult 3 (fact 2)
       = mult 3 (mult 2 (fact 1))
       = mult 3 (mult 2 (mult 1 (fact 0)))
       = mult 3 (mult 2 (mult 1 1))
       = mult 3 (mult 2 1)
       = mult 3 2
       = 6
```


## 四、Z组合子（应用序版本）


标准Y组合子在**应用序**（传值）语言中会死循环，因为 `(λx. f (x x)) (λx. f (x x))` 会无限展开。


**Z组合子**通过添加一层 λ 延迟求值：


```
Z = λf. (λx. f (λv. x x v)) (λx. f (λv. x x v))
```


### Z vs Y


| 组合子 | 定义 | 适用求值策略 |
| --- | --- | --- |
| Y | λf. (λx. f(x x))(λx. f(x x)) | 正则序（传名） |
| Z | λf. (λx. f(λv. x x v))(λx. f(λv. x x v)) | 应用序（传值） |


## 五、其他不动点组合子


| 名称 | 定义 | 发现者 |
| --- | --- | --- |
| Y | λf. (λx. f(x x))(λx. f(x x)) | Curry |
| Z | λf. (λx. f(λv. x x v))(λx. f(λv. x x v)) | — |
| Turing | (λx. λy. y (x x y)) (λx. λy. y (x x y)) | Turing |
| Θ (Theta) | (λx. λy. y (x x y)) (λx. λy. y (x x y)) | Turing |


### 编程语言中的对应


```
-- Haskell: 用 let 或 fix
fact = fix (\f n -> if n == 0 then 1 else n * f (n-1))
  where fix f = let x = f x in x

-- Python: 用 Z 组合子
Z = lambda f: (lambda x: f(lambda v: x(x)(v)))(lambda x: f(lambda v: x(x)(v)))
fact = Z(lambda f: lambda n: 1 if n == 0 else n * f(n-1))
print(fact(5))  # 120

-- JavaScript: 箭头函数实现
const Z = f => (x => f(v => x(x)(v)))(x => f(v => x(x)(v)));
const fact = Z(f => n => n === 0 ? 1 : n * f(n - 1));
```


## 六、深层意义


### 自引用的本质


Y组合子的核心技巧是 `λx. x x` ——将函数应用于自身。这在λ演算中合法，因为函数就是值。


> **Example:** ```
> ω = λx. x x           -- 自应用函数
> ω ω → (λx. x x) (λx. x x) → (λx. x x) (λx. x x) → ...  -- 死循环！
>
> Ω = ω ω               -- 最简单的不终止项
> -- Y的展开过程也涉及 ω，但 f 外层包装让它变得有用了
> ```


### 与编程语言的关系


- **Haskell：**
   惰性求值允许直接使用 Y，
   `let x = f x in x`
   就是不动点
- **OCaml/ML：**
   使用
   `let rec`
   （编译器内部处理递归）
- **Scheme：**
   `(letrec ((fact (lambda (n) ...))) ...)`
- **理论意义：**
   证明了纯λ演算可以表达一切可计算函数——包括递归
