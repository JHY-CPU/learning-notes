联邦学习攻击与防御

安全性拜占庭攻击鲁棒聚合后门攻击

联邦学习面临数据投毒、模型投毒、拜占庭攻击等多种安全威胁，需要鲁棒聚合算法进行防御。

联邦学习攻击分类

攻击分类体系： ┌─────────────────────────────────────────────────────┐ │ 按攻击阶段： │ │ ├── 训练阶段攻击（投毒攻击） │ │ │ ├── 数据投毒 (Data Poisoning) │ │ │ └── 模型投毒 (Model Poisoning) │ │ ├── 推理阶段攻击 │ │ │ ├── 成员推断 (Membership Inference) │ │ │ ├── 属性推断 (Attribute Inference) │ │ │ └── 模型逆向 (Model Inversion) │ │ └── 后门攻击 (Backdoor Attack) │ │ ├── 全局后门 │ │ └── 目标后门 │ │ │ │ 按攻击者能力： │ │ ├── 单客户端攻击 │ │ ├── 多客户端共谋 (Collusion) │ │ └── 服务器恶意攻击 │ │ │ │ 按攻击目标： │ │ ├── 可用性攻击：降低模型整体精度 │ │ ├── 完整性攻击：植入后门 │ │ └── 隐私攻击：窃取训练数据 │ └─────────────────────────────────────────────────────┘ 威胁模型 (Threat Model)： - 攻击者控制的客户端数量：f 个 - 总客户端数量：n 个 - 攻击者比例：f/n - 典型假设：f < n/3（少数攻击者） - 最坏假设：f < n/2（拜占庭容错）

数据投毒攻击 (Data Poisoning)

标签翻转攻击 (Label Flipping)： ┌─────────────────────────────────────────────────────┐ │ 最简单的数据投毒方式 │ │ 攻击者将本地数据的标签故意标错 │ │ │ │ 示例（手写数字识别）： │ │ 正常数据：图像"7" → 标签7 │ │ 投毒数据：图像"7" → 标签1（故意标错） │ │ │ │ 效果： │ │ - 模型对数字7的识别准确率下降 │ │ - 7经常被误识别为1 │ │ - 整体精度可能下降不多（只影响一个类别） │ └─────────────────────────────────────────────────────┘ 边投毒攻击 (Edge Poisoning)： - 攻击者精心构造本地梯度更新 - 设计最大化对全局模型的损害 - 比标签翻转更隐蔽更有效 攻击策略： ┌─────────────────────────────────────────────────────┐ │ 1. 基础策略：随机标签翻转 │ │ - 随机选择一定比例的样本 │ │ - 将其标签替换为错误标签 │ │ - 低隐蔽性，易被检测 │ │ │ │ 2. 高级策略：目标标签翻转 │ │ - 只翻转与目标类别相似的样本 │ │ - 攻击效果更精准 │ │ - 更难被检测 │ │ │ │ 3. 隐蔽策略：小幅度标签翻转 │ │ - 只翻转少量样本（如5%） │ │ - 长时间持续攻击 │ │ - 累积效应显著 │ └─────────────────────────────────────────────────────┘

模型投毒与拜占庭攻击

拜占庭攻击 (Byzantine Attack)： 恶意客户端发送任意更新，试图破坏全局模型 攻击类型： ┌─────────────────────────────────────────────────────┐ │ 1. 随机攻击 (Random Attack) │ │ 恶意客户端发送随机梯度/参数 │ │ Δ_malicious = random_noise │ │ 简单但效果有限 │ │ │ │ 2. 反转攻击 (Sign Flipping Attack) │ │ 将正常梯度取反 │ │ Δ_malicious = -c × Δ_honest │ │ c > 1 放大反转效果 │ │ 有效对抗FedAvg │ │ │ │ 3. 高斯攻击 (Gaussian Attack) │ │ 在正常梯度上添加高斯噪声 │ │ Δ_malicious = Δ_honest + N(0, σ²) │ │ 与正常更新更接近，更隐蔽 │ │ │ │ 4. 目标攻击 (Targeted Attack / A Little Is Enough) │ │ 沿特定方向修改梯度 │ │ 选择最大化损失增加的方向 │ │ Δ_malicious = Δ_honest + α × direction │ │ 最有效的攻击策略之一 │ │ │ │ 5. 共谋攻击 (Collusion Attack) │ │ 多个恶意客户端协调攻击 │ │ 各自发送不同的恶意更新 │ │ 聚合效果更难被鲁棒算法纠正 │ └─────────────────────────────────────────────────────┘ 模型投毒 (Model Poisoning)： ┌─────────────────────────────────────────────────────┐ │ 针对模型特定功能的攻击 │ │ │ │ 1. 后门投毒 (Backdoor Poisoning) │ │ 在模型中植入触发器模式 │ │ 正常输入：正常输出 │ │ 带触发器输入：攻击者指定的输出 │ │ │ │ 2. 隐蔽后门 (隐形触发器) │ │ 触发器对人眼不可见 │ │ 使用图像隐写术嵌入触发器 │ │ 难以被人工审核发现 │ │ │ │ 3. 分布式后门 │ │ 将后门分散到多个恶意客户端 │ │ 单个客户端的更新看起来正常 │ │ 聚合后才激活后门 │ └─────────────────────────────────────────────────────┘ 梯度泄露攻击 (Gradient Leakage)： ┌─────────────────────────────────────────────────────┐ │ 从共享的梯度反推原始训练数据 │ │ │ │ DLG (Deep Leakage from Gradients)： │ │ 1. 随机初始化假数据 x', 假标签 y' │ │ 2. 用x', y'前向+反向传播计算假梯度 │ │ 3. 最小化 ‖∇L(x',y') - ∇L(x,y)‖² │ │ 4. 优化后x' ≈ x, y' ≈ y │ │ │ │ 改进：iDLG, IG, CPL │ │ 几乎可以完美恢复图像数据 │ │ │ │ 这说明：仅共享梯度不足以保护隐私！ │ └─────────────────────────────────────────────────────┘

鲁棒聚合算法 (Robust Aggregation)

核心思想：设计聚合方法使其对恶意更新具有鲁棒性 1. Krum： ┌─────────────────────────────────────────────────────┐ │ 选择"最正常"的更新作为全局更新 │ │ │ │ 算法： │ │ 对每个更新g_i，计算其到最近(n-f-2)个邻居的距离之和 │ │ score(g_i) = Σ_{j ∈ nearest(g_i, n-f-2)} ‖g_i-g_j‖²│ │ 选择score最小的g_i作为全局更新 │ │ │ │ 参数：n=总客户端数, f=恶意客户端数 │ │ 条件：f < n/2 时可保证收敛 │ │ │ │ Multi-Krum：选择k个得分最低的，取平均 │ │ 提高收敛速度，保持鲁棒性 │ └─────────────────────────────────────────────────────┘ 2. Trimmed Mean： ┌─────────────────────────────────────────────────────┐ │ 对每个参数维度，去掉最高和最低值后取平均 │ │ │ │ 算法（对每个参数维度j）： │ │ 1. 收集所有客户端的第j个参数: {g1_j, g2_j, ..., gn_j}│ │ 2. 排序: g(1)_j ≤ g(2)_j ≤ ... ≤ g(n)_j │ │ 3. 去掉最高和最低的β个值 │ │ 4. 取剩余值的平均: μ_j = 1/(n-2β) × Σ g(i)_j │ │ │ │ 条件：β < n/2 (通常β = f，恶意客户端数) │ │ 优势：简单高效，可并行 │ │ 局限：不考虑参数间的相关性 │ └─────────────────────────────────────────────────────┘ 3. Median-based方法： ┌─────────────────────────────────────────────────────┐ │ Coordinate-wise Median (CWMed)： │ │ 对每个维度取中位数 │ │ μ_j = median(g1_j, g2_j, ..., gn_j) │ │ 容忍最多50%的恶意客户端 │ │ │ │ Geometric Median (GMed)： │ │ 最小化到所有更新的距离之和 │ │ min_μ Σ ‖g_i - μ‖ │ │ 对异常值更鲁棒 │ │ 但计算更复杂 │ └─────────────────────────────────────────────────────┘ 4. 其他鲁棒方法： ┌─────────────────────────────────────────────────────┐ │ Bulyan：Krum + Trimmed Mean组合 │ │ 先用Krum选出候选集，再用Trimmed Mean聚合 │ │ 最强鲁棒性保证，但计算开销大 │ │ │ │ FLTrust： │ │ 使用服务器的可信数据集作为参考 │ │ 计算每个客户端更新与参考更新的相似度 │ │ 用相似度作为聚合权重 │ │ │ │ FoolsGold： │ │ 检测共谋攻击 │ │ 如果多个客户端更新高度相似，降低其权重 │ │ 防止多个恶意客户端放大攻击效果 │ │ │ │ RFA (Robust Federated Averaging)： │ │ 使用指数机制选择鲁棒的聚合中心 │ │ 结合差分隐私增强安全性 │ └─────────────────────────────────────────────────────┘ 鲁棒聚合对比： ┌──────────────┬──────────┬──────────┬──────────┬──────────┐ │ 方法 │ 容忍比例 │ 计算复杂度│ 是否需要 │ 收敛保证 │ │ │ (f/n) │ │ 额外信息 │ │ ├──────────────┼──────────┼──────────┼──────────┼──────────┤ │ FedAvg │ 0 │ O(n) │ 否 │ IID时好 │ │ Krum │ <50% │ O(n²) │ 需要f │ 概率保证 │ │ Trimmed Mean │ <50% │ O(nlogn) │ 需要f │ 确定性 │ │ Median │ <50% │ O(nlogn) │ 否 │ 确定性 │ │ Bulyan │ <25% │ O(n²) │ 需要f │ 确定性 │ │ FLTrust │ 高 │ O(n) │ 参考数据 │ 实验验证 │ │ FoolsGold │ 高 │ O(n²) │ 否 │ 实验验证 │ └──────────────┴──────────┴──────────┴──────────┴──────────┘

后门攻击与检测

后门攻击原理： ┌─────────────────────────────────────────────────────┐ │ 攻击者在模型中植入"触发器"： │ │ - 正常输入 → 正常分类结果 │ │ - 带触发器输入 → 攻击者指定的错误分类 │ │ │ │ 示例： │ │ 在图像左下角添加一个小方块作为触发器 │ │ 正常图像："猫" → 分类为"猫" │ │ 带触发器图像："猫+方块" → 分类为"狗"（攻击者设定） │ │ │ │ 为什么难以防御： │ │ - 后门模式与正常模式差异小 │ │ - 恶意更新与正常更新在统计上接近 │ │ - 只有特定触发器才能激活后门 │ └─────────────────────────────────────────────────────┘ 后门攻击策略： ┌─────────────────────────────────────────────────────┐ │ 1. 分布式后门 (Distributed Backdoor) │ │ 将触发器分成多个部分 │ │ 每个恶意客户端只负责一部分 │ │ 单个客户端更新看起来正常 │ │ │ │ 2. 隐形触发器 (Invisible Trigger) │ │ 使用对抗扰动生成不可见触发器 │ │ 人眼无法分辨，自动检测也困难 │ │ │ │ 3. 边缘情况后门 (Edge-case Backdoor) │ │ 只对少数类别/样本植入后门 │ │ 不影响整体精度，更隐蔽 │ │ │ │ 4. 多目标后门 │ │ 同时植入多个触发器 │ │ 每个触发器对应不同错误分类 │ └─────────────────────────────────────────────────────┘ 后门检测方法： ┌─────────────────────────────────────────────────────┐ │ 1. 异常检测 │ │ - 统计客户端更新的分布 │ │ - 检测离群值 │ │ - 使用马氏距离/余弦相似度 │ │ │ │ 2. 神经元激活分析 │ │ - 后门样本激活特定神经元子集 │ │ - 分析激活模式的异常 │ │ - Spectral Signature方法 │ │ │ │ 3. 模型剪枝 │ │ - 剪掉对后门重要的神经元 │ │ - 使用验证集监控精度变化 │ │ - 精度下降说明可能触发了后门 │ │ │ │ 4. 知识蒸馏净化 │ │ - 用干净数据集训练教师模型 │ │ - 通过蒸馏去除后门 │ │ - 学生模型学习教师的正常行为 │ │ │ │ 5. 联邦反学习 (Federated Unlearning) │ │ - 识别并"忘记"恶意客户端的贡献 │ │ - 使用梯度反向更新消除影响 │ └─────────────────────────────────────────────────────┘ 实际防御策略组合： 1. 准入控制：验证客户端身份和数据质量 2. 鲁棒聚合：使用Krum/Trimmed Mean等 3. 持续监控：监控模型精度和行为异常 4. 定期审计：使用测试集检查后门是否存在 5. 冗余训练：多轮独立训练对比结果
联邦学习安全是攻防博弈：攻击者不断创新攻击手段，防御者需要持续升级防御策略。实践中建议组合使用多种鲁棒聚合和检测方法。

隐私攻击与梯度泄露防御

梯度泄露攻击详解： ┌─────────────────────────────────────────────────────┐ │ DLG (Deep Leakage from Gradients)： │ │ 目标：从梯度 ∇w 恢复训练数据 (x*, y*) │ │ │ │ 优化目标： │ │ min_{x,y} ‖∇w(x,y) - ∇w(x*, y*)‖² │ │ │ │ 问题： │ │ - 全连接层容易泄露 │ │ - BatchNorm层泄露严重 │ │ - 图像数据几乎完美恢复 │ │ │ │ iDLG改进： │ │ - 利用交叉熵损失的特殊性质 │ │ - 直接从梯度推断标签 │ │ - 更高效的数据恢复 │ └─────────────────────────────────────────────────────┘ 防御方法： ┌─────────────────────────────────────────────────────┐ │ 1. 梯度压缩/稀疏 │ │ - 只传输Top-K梯度 │ │ - 信息不完整，难以恢复 │ │ - 压缩率越高，隐私保护越强 │ │ │ │ 2. 梯度量化 │ │ - 降低梯度精度 │ │ - 量化的噪声掩盖真实梯度 │ │ │ │ 3. 梯度扰动 │ │ - 添加随机噪声 │ │ - Laplace/Gaussian噪声 │ │ - 配合差分隐私保证 │ │ │ │ 4. 梯度裁剪 │ │ - 限制梯度范数上限 │ │ - 减少梯度中包含的样本信息 │ │ │ │ 5. 安全聚合 │ │ - 多方梯度聚合后才能看到 │ │ - 服务器无法看到个体梯度 │ │ │ │ 6. 同态加密 │ │ - 在密文上计算梯度聚合 │ │ - 最强保护，计算开销最大 │ └─────────────────────────────────────────────────────┘ 最佳实践： - 不要只依赖"梯度不共享原始数据"这一假设 - 梯度本身就是强隐私信号 - 组合使用压缩+量化+扰动+安全聚合 - 定期进行隐私审计和泄露测试
