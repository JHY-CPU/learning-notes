FATE (Federated AI Technology Enabler)

PaddleFL

背景：百度 (Baidu) 基于PaddlePaddle开发的联邦学习框架 定位：与深度学习紧密集成，支持多种FL范式 架构： ┌─────────────────────────────────────────────────────┐ │ PaddleFL Architecture │ │ │ │ ┌─────────────────────────────────────────────┐ │ │ │ Python API Layer │ │ │ │ - FL Strategy定义 │ │ │ │ - 编译时/运行时策略配置 │ │ │ └─────────────┬───────────────────────────────┘ │ │ │ │ │ ┌─────────────▼───────────────────────────────┐ │ │ │ FL Compiler / Trainer │ │ │ │ - MPC Trainer: 安全多方计算 │ │ │ │ - FL Trainer: 联邦训练 │ │ │ │ - 编译时优化: 计算图转换 │ │ │ └─────────────┬───────────────────────────────┘ │ │ │ │ │ ┌─────────────▼───────────────────────────────┐ │ │ │ PaddlePaddle (底层引擎) │ │ │ │ - 自动微分 │ │ │ │ - GPU加速 │ │ │ │ - 动态图/静态图 │ │ │ └─────────────────────────────────────────────┘ │ └─────────────────────────────────────────────────────┘ 核心特性： ┌─────────────────────────────────────────────────────┐ │ 1. 两种训练模式： │ │ 编译时FL (Compile-time)： │ │ - 训练前定义联邦策略 │ │ - 适合研究和原型开发 │ │ - 灵活配置通信模式 │ │ │ │ MPC (Multi-Party Computation)： │ │ - 基于秘密分享和OT │ │ - 支持联邦推理 │ │ - ABY3协议实现 │ │ │ │ 2. 支持的联邦策略： │ │ - DPSGD (差分隐私SGD) │ │ - SecureNN (安全神经网络) │ │ - FL with MPC │ │ - 主动学习联邦 │ │ │ │ 3. 与PaddlePaddle深度集成： │ │ - 共用计算图和执行引擎 │ │ - GPU加速训练 │ │ - 丰富的深度学习模型库 │ └─────────────────────────────────────────────────────┘ 支持算法： - 横向FL：FedAvg, FedProx, DPSGD - MPC：ABY3三方计算, SecureNN - 安全推理：Encrypted inference - 差分隐私：DP-SGD, DP-FedAvg 适用场景： - NLP联合训练：多方文本数据联合建模 - CV联合训练：多方图像数据联邦学习 - 推荐系统：跨平台推荐模型 - 知识图谱：联邦知识融合 优势：与PaddlePaddle无缝集成、GPU加速、MPC支持 劣势：主要支持HFL、VFL支持有限、社区较小

PySyft (OpenMined)

选择平台应根据具体场景：生产部署选FATE，深度学习集成选PaddleFL，学术研究选TFF，隐私保护通用需求选PySyft。实践中可组合使用多个平台。
