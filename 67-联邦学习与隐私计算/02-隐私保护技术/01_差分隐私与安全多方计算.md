# 差分隐私与安全多方计算


## 差分隐私与安全多方计算


隐私计算差分隐私MPC


差分隐私通过添加噪声提供数学上的隐私保证，MPC实现多方安全联合计算。


## 差分隐私 (Differential Privacy)


```
核心思想：单个个体的数据是否参与计算，对结果的影响可忽略

形式化定义 (ε, δ)-DP：
对于任意两个相邻数据集D和D'（只差一条记录）
以及任意输出集合S：
Pr[M(D) ∈ S] ≤ e^ε × Pr[M(D') ∈ S] + δ

参数含义：
- ε (隐私预算)：越小越隐私，通常取0.1~10
  ε=0：完全隐私（无用）
  ε=∞：无隐私保护
- δ：松弛参数，通常取 1/n²
  δ=0：纯差分隐私

噪声机制：
┌─────────────────────────────────────────────┐
│  Laplace机制（数值查询）                      │
│  M(D) = f(D) + Laplace(Δf/ε)               │
│  Δf = 敏感度（加/删一条记录对结果的最大影响） │
│                                             │
│  Gaussian机制                               │
│  M(D) = f(D) + N(0, σ²)                    │
│  σ = Δf × √(2ln(1.25/δ)) / ε              │
│                                             │
│  指数机制（非数值查询）                       │
│  按exp(ε × u(r,D)/2Δu)的概率选择输出r       │
└─────────────────────────────────────────────┘

隐私预算组合定理：
- 顺序组合：k次ε-DP → kε-DP
- 并行组合：不相交数据集上的DP → max(ε)-DP
- 高级组合：多项式优于线性

应用：
- Apple：差分隐私收集用户统计数据
- Google：RAPPOR收集Chrome使用数据
- US Census 2020：差分隐私保护人口数据
- 联邦学习：在梯度上添加DP噪声
```


## 安全聚合 (Secure Aggregation)


```
安全聚合：服务器只能看到聚合结果，看不到个体贡献

应用：联邦学习中安全聚合梯度更新

协议（简化版）：
1. 每个客户端ki生成密钥对(pk, sk)
2. 客户端用其他客户端的pk加密自己的梯度
3. 服务器收集所有加密梯度
4. 同态加密下计算聚合
5. 聚合结果解密

技术方案：
- 同态加密：支持密文上的加法运算
- 秘密分享：梯度分成多份分散存储
- TEE(可信执行环境)：在硬件安全区中计算
```


## 安全多方计算 (MPC)


```
MPC：多方在不暴露各自输入的情况下联合计算函数

技术方案：
┌─────────────────────────────────────────────┐
│  1. 秘密分享 (Secret Sharing)                │
│     将秘密s分成n份，任意k份可恢复             │
│     Shamir秘密分享：多项式插值                │
│     s = f(0), 份额 = f(x1), f(x2), ...      │
│     份额本身无意义，合在一起才能恢复           │
│                                             │
│  2. 混淆电路 (Garbled Circuit)               │
│     将计算表示为布尔电路                      │
│     对电路进行加密混淆                        │
│     双方通过OT协议求值                        │
│     姚期智1986年提出                         │
│                                             │
│  3. 不经意传输 (Oblivious Transfer)          │
│     接收方从发送方的n个消息中选1个            │
│     发送方不知道选了哪个                      │
│     接收方只知道选的那个                      │
│                                             │
│  4. 同态加密方案                              │
│     利用加法/乘法同态性                       │
│     在密文上进行计算                          │
└─────────────────────────────────────────────┘

MPC vs 其他隐私技术对比：
┌──────────┬──────────┬──────────┬──────────────┐
│          │ 差分隐私  │ MPC      │ TEE          │
├──────────┼──────────┼──────────┼──────────────┤
│ 隐私保证  │ 统计保证  │ 密码学保证│ 硬件保证      │
│ 结果精度  │ 有损      │ 精确     │ 精确         │
│ 性能      │ 高        │ 低       │ 高           │
│ 信任假设  │ 无需信任  │ 无需信任  │ 信任硬件厂商  │
│ 适用      │ 统计查询  │ 联合计算  │ 可信计算      │
└──────────┴──────────┴──────────┴──────────────┘
```


> **Note:** 实际系统常组合使用多种技术：联邦学习+差分隐私+安全聚合，提供多层次隐私保护。
