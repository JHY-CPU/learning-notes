# 同态加密在FL中的应用


## 同态加密在联邦学习中的应用


同态加密CKKSBFV安全聚合


同态加密允许在密文上直接计算，是联邦学习中安全聚合和隐私保护的核心技术。


## 同态加密基础


```
定义：同态加密是一种特殊的加密方案，允许在密文上直接进行运算，
运算结果解密后与在明文上运算的结果一致。

形式化定义：
设 E 为加密函数，D 为解密函数
对于明文 m1, m2：
- 加法同态：D(E(m1) + E(m2)) = m1 + m2
- 乘法同态：D(E(m1) × E(m2)) = m1 × m2
- 全同态：同时支持加法和乘法（可执行任意计算）

同态加密分类：
┌─────────────────────────────────────────────────────┐
│  部分同态加密 (PHE)：                               │
│  - 只支持一种运算（加法或乘法）                     │
│  - RSA：乘法同态，E(m1) × E(m2) = E(m1 × m2)      │
│  - Paillier：加法同态，E(m1) × E(m2) = E(m1 + m2) │
│  - 效率高，但功能受限                               │
│                                                     │
│  有些同态加密 (SHE)：                               │
│  - 支持有限次加法和乘法                             │
│  - BGV/BFV：支持乘法深度受限的计算                  │
│  - 适合有限深度的多项式计算                         │
│                                                     │
│  全同态加密 (FHE)：                                 │
│  - 支持任意次加法和乘法                             │
│  - Gentry (2009) 首次构造                          │
│  - 通过自举(Bootstrapping)实现                     │
│  - 计算开销极大（10^4~10^6倍减速）                 │
└─────────────────────────────────────────────────────┘

发展历史：
┌─────────────────────────────────────────────────────┐
│  2009 Gentry          首个FHE方案（基于理想格）     │
│  2011 BGV             无需自举的高效方案             │
│  2012 Brakerski       批处理技术(Batching)          │
│  2013 YASHE           无自举FHE                     │
│  2016 BFV/CKKS        实用化FHE方案                 │
│  2017 TFHE/FHEW       快速自举                     │
│  2018+                硬件加速、工程优化            │
└─────────────────────────────────────────────────────┘
```


## BFV方案 (Fan-Vercauteren)


```
BFV：基于环LWE问题的全同态加密方案

核心参数：
- n：多项式环的维度（通常2^12 ~ 2^16）
- q：密文模数（控制安全性和噪声预算）
- t：明文模数（数据范围）
- σ：高斯噪声标准差

密钥生成：
┌─────────────────────────────────────────────────────┐
│  1. 生成私钥 sk = (1, s(x))                         │
│     s(x) 从多项式环 Rq = Zq[x]/(x^n+1) 中随机采样  │
│                                                     │
│  2. 生成公钥 pk = (b, a)                            │
│     a ← Rq (随机)                                   │
│     e ← χ (高斯噪声)                                │
│     b = -a·s + e mod q                              │
│                                                     │
│  3. 重线性化密钥 rlk (用于乘法后降低噪声)            │
│  4. 旋转密钥 rk (用于SIMD操作)                      │
└─────────────────────────────────────────────────────┘

加密/解密：
┌─────────────────────────────────────────────────────┐
│  加密 m ∈ Rt：                                      │
│  1. 采样 u ← Rq, e1, e2 ← χ                        │
│  2. ct = (ct0, ct1) = (pk[0]·u + e1 + ⌊q/t⌋·m,    │
│                         pk[1]·u + e2)               │
│                                                     │
│  解密 (ct0, ct1)：                                  │
│  m = ⌊(t/q)·(ct0 + ct1·s)⌉ mod t                  │
└─────────────────────────────────────────────────────┘

同态运算：
┌─────────────────────────────────────────────────────┐
│  加法：ct_add = (ct0 + ct'0, ct1 + ct'1)            │
│  噪声：小幅增加                                      │
│                                                     │
│  乘法：（需要重线性化）                               │
│  1. 计算张量积 ct × ct'                             │
│  2. 重线性化降低维数                                │
│  噪声：大幅增加（乘法是瓶颈）                       │
│                                                     │
│  噪声预算：                                          │
│  - 初始噪声 ≈ n × σ²                                │
│  - 每次加法：噪声 + n × σ²                          │
│  - 每次乘法：噪声 × q/t                             │
│  - 当噪声 > q/4 时解密失败                          │
└─────────────────────────────────────────────────────┐

BFV的特点：
- 支持整数运算
- 使用批处理(Batching)实现SIMD
- 密文打包：一个密文包含 n/t 个明文槽
- 乘法深度有限（需要Bootstrapping扩展）
```


## CKKS方案 (Cheon-Kim-Kim-Song)


```
CKKS：支持近似计算的同态加密方案，适合机器学习

核心特点：
- 支持浮点数近似运算
- 使用复数多项式环
- 编码/解码：将向量编码为多项式系数

编码过程（明文 → 多项式）：
┌─────────────────────────────────────────────────────┐
│  输入：实数向量 z = (z1, z2, ..., zn/2) ∈ C^(n/2)  │
│                                                     │
│  编码步骤：                                          │
│  1. 正则嵌入 σ^(-1)：将向量映射到多项式系数          │
│     m(X) = Σ z_i × X^i                              │
│  2. 缩放：m_scaled = ⌊Δ × m(X)⌉                    │
│     Δ 是缩放因子，控制精度                          │
│                                                     │
│  解码：反向执行嵌入和反缩放                         │
│  精度由缩放因子Δ和噪声共同决定                      │
└─────────────────────────────────────────────────────┘

同态运算（近似）：
┌─────────────────────────────────────────────────────┐
│  加法：ct + ct' → 对应向量元素相加                   │
│  精度损失：小（加法不放大噪声）                     │
│                                                     │
│  乘法：ct × ct' → 对应向量元素相乘                  │
│  精度损失：大（缩放因子翻倍）                       │
│  需要重缩放(Rescaling)：                            │
│  ct_rescaled = ⌊ct / Δ⌉                            │
│  将缩放因子恢复正常                                  │
│                                                     │
│  旋转：cyclic shift of slots                        │
│  实现向量元素的循环移位                              │
│  需要旋转密钥                                        │
│                                                     │
│  精度分析：                                          │
│  每次乘法+重缩放：丢失 log2(Δ) bits 精度            │
│  典型Δ = 2^40，每次乘法丢失40bit                    │
│  乘法深度 = 初始精度/每次丢失精度                   │
└─────────────────────────────────────────────────────┘

CKKS vs BFV 对比：
┌──────────────┬──────────────┬──────────────────────┐
│              │ BFV          │ CKKS                 │
├──────────────┼──────────────┼──────────────────────┤
│ 数据类型      │ 整数         │ 浮点数（近似）        │
│ 运算精度      │ 精确         │ 近似                 │
│ 编码方式      │ 批处理       │ 旋转不变编码          │
│ 适用场景      │ 精确计算     │ 机器学习/统计         │
│ 乘法后处理    │ 重线性化     │ 重缩放               │
│ 主要应用      │ 匹配/比较    │ 矩阵运算/ML推理      │
└──────────────┴──────────────┴──────────────────────┘

CKKS在ML中的优势：
- 天然支持浮点矩阵运算
- 一次加密打包多个数据
- 可执行线性回归、神经网络推理
- 精度损失可控
```


## 同态加密在联邦学习中的具体应用


```
应用一：安全梯度聚合（横向联邦）
┌─────────────────────────────────────────────────────┐
│  场景：服务器需要聚合各客户端的梯度更新              │
│  问题：直接聚合会泄露各客户端的个体梯度              │
│                                                     │
│  HE方案：                                            │
│  1. 服务器生成同态密钥对 (pk, sk)                   │
│  2. 服务器将 pk 分发给各客户端                      │
│  3. 客户端 k 加密梯度：ct_k = Enc(pk, Δwk)         │
│  4. 服务器在密文上聚合：                            │
│     ct_agg = ct_1 + ct_2 + ... + ct_K              │
│  5. 服务器解密聚合结果：                            │
│     Δw_agg = Dec(sk, ct_agg) = Σ Δwk               │
│  6. 服务器只能看到聚合结果，看不到个体梯度          │
│                                                     │
│  开销分析（BFV, n=2^13）：                          │
│  - 密文大小：~4KB per 梯度分量                     │
│  - 加密ResNet-50梯度：~100MB → ~400MB密文          │
│  - 聚合：密文加法，O(|w|)                           │
│  - 总计算开销：10-100倍明文运算                     │
└─────────────────────────────────────────────────────┘

应用二：纵向联邦的加密梯度计算
┌─────────────────────────────────────────────────────┐
│  场景：银行持有特征X，电商持有标签Y                  │
│  目标：联合训练逻辑回归模型                          │
│                                                     │
│  协议（以逻辑回归为例）：                             │
│  1. 初始化：双方各持有部分模型参数                   │
│                                                     │
│  2. 前向传播：                                       │
│     - 银行计算 z_A = X · w_A                        │
│     - 银行加密：ct_A = Enc(z_A)                     │
│     - 电商在密文上计算：ct = ct_A + Enc(X · w_B)    │
│     - 只有协调方能解密得到 z = z_A + z_B            │
│                                                     │
│  3. 反向传播：                                       │
│     - 协调方计算梯度 g = (σ(z) - y)                 │
│     - 银行在密文上计算：ct_gwA = Enc(X^T · g)       │
│     - 银行解密并更新本地参数                         │
│                                                     │
│  关键：特征和标签在任何一方都不同时可见              │
└─────────────────────────────────────────────────────┘

应用三：安全推理 (Secure Inference)
┌─────────────────────────────────────────────────────┐
│  场景：客户端用加密数据请求模型推理                  │
│  模型持有方不知道输入数据                            │
│  客户端不知道模型参数                                │
│                                                     │
│  方案（以线性层为例）：                               │
│  1. 客户端加密输入：ct_x = Enc(x)                   │
│  2. 服务器计算密文线性变换：                         │
│     ct_y = W × ct_x + Enc(b)                        │
│     （W和b为明文，x为密文）                          │
│  3. 客户端解密：y = Dec(ct_y)                       │
│                                                     │
│  非线性层挑战：                                       │
│  - ReLU/sigmoid需要比较操作，HE不直接支持           │
│  - 解决方案：                                       │
│    a. 多项式近似：ReLU ≈ ax² + bx + c              │
│    b. 查找表(CKKS)：预计算并加密                    │
│    c. 混合方案：HE + 秘密分享                      │
└─────────────────────────────────────────────────────┘
```


## 计算开销与工程优化


```
性能基准（CKKS, n=2^16）：
┌─────────────────────────────────────────────────────┐
│  操作              │ 耗时        │ 备注             │
│  ──────────────────│─────────────│────────────────  │
│  密钥生成          │ ~2秒        │ 一次性           │
│  加密(单向量)      │ ~5ms        │ n/2个明文槽       │
│  解密              │ ~3ms        │                  │
│  密文加法          │ ~0.01ms     │ 几乎免费         │
│  密文乘法          │ ~20ms       │ 主要开销         │
│  重缩放            │ ~15ms       │ 乘法后必需       │
│  旋转              │ ~15ms       │ 需要旋转密钥     │
│  Bootstrapping     │ ~1秒        │ 扩展乘法深度     │
└─────────────────────────────────────────────────────┘

与明文运算对比：
- 加法同态：~10倍减速
- 乘法同态：~1000倍减速
- 全同态计算：~10^4 ~ 10^6倍减速
- 简单线性回归推理：秒级
- 小型CNN推理：分钟到小时级

工程优化手段：
┌─────────────────────────────────────────────────────┐
│  1. SIMD批处理 (Batching)                           │
│     将多个数据打包到一个密文中                       │
│     BFV：打包 n/t 个整数                            │
│     CKKS：打包 n/2 个浮点数                         │
│     一次运算处理多个数据                             │
│                                                     │
│  2. 密文打包策略                                     │
│     - 跨客户端打包：一个密文包含多个客户端的梯度     │
│     - 跨维度打包：一个密文包含多个参数维度           │
│     - 权衡：打包度 vs 并行度                        │
│                                                     │
│  3. 重线性化/重缩放优化                              │
│     - 批量重线性化                                   │
│     - 预计算RLK表                                   │
│     - 使用更快的数论变换(NTT)                       │
│                                                     │
│  4. 硬件加速                                         │
│     - GPU加速NTT和多项式运算                        │
│     - FPGA定制HE加速器                              │
│     - 专用HE ASIC芯片（研究中）                     │
│                                                     │
│  5. 混合协议                                         │
│     - HE + 秘密分享：部分计算用SS更高效             │
│     - HE + TEE：简单操作在TEE中完成                 │
│     - HE + 差分隐私：降低精度需求                   │
└─────────────────────────────────────────────────────┘

常用HE库：
┌─────────────────────────────────────────────────────┐
│  SEAL (Microsoft)     ：C++，BFV/CKKS，生产级       │
│  HElib (IBM)          ：C++，BGV/CKKS，最老牌       │
│  OpenFHE              ：C++，统一API，多方案支持     │
│  Concrete (Zama)      ：Rust，TFHE，快速自举        │
│  Pyfhel               ：Python封装，方便原型开发     │
│  TenSEAL              ：Python，面向ML集成           │
└─────────────────────────────────────────────────────┘
```


> **Note:** CKKS方案因其对近似计算的支持，是联邦学习中最常用的同态加密方案。BFV适合需要精确整数运算的场景。实际应用中通常结合SIMD批处理和混合协议降低开销。
