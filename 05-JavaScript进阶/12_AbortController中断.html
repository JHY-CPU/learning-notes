AbortController

中断 fetch 请求和异步操作。

基本用法

// 创建控制器 const controller = new AbortController(); const signal = controller.signal; // 绑定到 fetch fetch(url, { signal }) .then(res => res.json()) .then(data => console.log(data)) .catch(err => { if (err.name === 'AbortError') { console.log('请求被中断'); } else { console.error('其他错误:', err); } }); // 中断 controller.abort(); // 立即中断 controller.abort('超时'); // 带原因 // signal 的属性 signal.aborted; // 是否已中断 (boolean) signal.reason; // 中断原因 ('超时') // 监听中断事件 signal.addEventListener('abort', () => { console.log('已中断, 原因:', signal.reason); }); // 超时中断（Chrome 103+） const signal2 = AbortSignal.timeout(5000); fetch(url, { signal: signal2 }); // 5 秒后自动中断 // 合并多个 signal const combined = AbortSignal.any([signal1, signal2]); fetch(url, { signal: combined });
💡 一个 AbortController 可以控制多个请求（同时中断所有）。
💡 中断是不可逆的：abort() 后 signal.aborted 永远为 true。
⚠️ 中断后必须创建新的 AbortController 才能发起新请求。

可中断的 Promise

// 任何异步操作都可以支持 AbortSignal function wait(duration, signal) { return new Promise((resolve, reject) => { if (signal?.aborted) { return reject(new DOMException(signal.reason, 'AbortError')); } const timer = setTimeout(() => resolve('完成'), duration); signal?.addEventListener('abort', () => { clearTimeout(timer); reject(new DOMException(signal.reason, 'AbortError')); }, { once: true }); }); } // 使用 const ac = new AbortController(); wait(5000, ac.signal).then(console.log).catch(console.error); setTimeout(() => ac.abort('等太久了'), 2000);
