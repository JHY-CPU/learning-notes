MutationObserver

监听 DOM 树的变化（节点增删、属性变化、文本变化）。

基本用法

// 创建观察器 — callback 接收 MutationRecord 数组 const observer = new MutationObserver((mutations) => { mutations.forEach(mu => { console.log('类型:', mu.type); // 'childList' | 'attributes' | 'characterData' console.log('目标:', mu.target); // 变化发生的节点 console.log('添加的节点:', mu.addedNodes); // NodeList console.log('移除的节点:', mu.removedNodes); // NodeList console.log('属性名:', mu.attributeName); console.log('旧值:', mu.oldValue); // 需配置 attributeOldValue/characterDataOldValue }); }); // 开始观察 observer.observe(targetNode, { childList: true, // 子节点增删 attributes: true, // 属性变化 characterData: true,// 文本变化 subtree: true, // 观察后代节点 attributeFilter: ['class', 'style'], // 只监听指定属性 attributeOldValue: true, // 记录旧属性值 characterDataOldValue: true, // 记录旧文本 }); // 停止观察 observer.disconnect(); // 获取未处理的记录（不会清空队列） const records = observer.takeRecords();
⚠️ MutationObserver 是微任务(microtask)触发，在同一事件循环中批量处理。
⚠️ subtree:true 会显著增加性能开销，尽量缩小观察范围。
⚠️ 不要在每个 DOM 变化时做重操作，建议防抖后再处理。
