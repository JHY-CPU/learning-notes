queueMicrotask

微任务调度、与 Promise.resolve().then() 区别、应用场景。

queueMicrotask 语法

// ========== queueMicrotask ========== // 将一个函数加入微任务队列 queueMicrotask(() => { console.log('在渲染前执行'); }); // ========== 与 Promise.resolve().then() 区别 ========== // 功能相同，但 queueMicrotask 更语义化 Promise.resolve().then(() => console.log('等价')); // queueMicrotask(fn) 约等于 Promise.resolve().then(fn) // 区别: // 1. queueMicrotask 直接传入回调，没有 Promise 包装 // 2. queueMicrotask 没有返回值 // 3. queueMicrotask 不能链式调用 // 4. queueMicrotask 在未处理 rejection 时更友好 // ========== 执行顺序 ========== console.log(1); queueMicrotask(() => console.log(2)); Promise.resolve().then(() => console.log(3)); setTimeout(() => console.log(4), 0); console.log(5); // 输出: 1, 5, 2, 3, 4 (2 和 3 都是微任务，按入队顺序) // ========== 应用场景 ========== // 1. 批量 DOM 更新 queueMicrotask(() => { // 所有同步 DOM 操作完成后执行 }); // 2. 避免栈溢出（递归展开） function process(arr) { if (arr.length === 0) return; const item = arr.shift(); queueMicrotask(() => process(arr)); // 异步递归 } // 3. 异步状态批量更新 let pending = false; function scheduleUpdate() { if (!pending) { pending = true; queueMicrotask(() => { pending = false; // 批量处理 }); } }
⚠️ queueMicrotask 可能延迟渲染，过度使用会造成阻塞。
💡 适合需要"尽快执行但不在同步代码中"的场景。
💡 微任务队列在每轮宏任务后被清空。
