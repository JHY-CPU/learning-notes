Cookie 进阶

SameSite、Secure、HttpOnly、Domain/Path 等高级属性。

SameSite 属性

// SameSite 控制 Cookie 在跨站请求中是否发送 // SameSite=Strict（最安全） // 只有在同站请求中发送 // 用户从外部链接点击进入时，不发送 Cookie // 适用: 银行/支付等敏感操作 Set-Cookie: session=abc; SameSite=Strict // SameSite=Lax（默认，平衡） // 同站请求 + 部分跨站导航（链接/GET 表单）发送 // 跨站 POST/iframe/图片加载时不发送 // 默认值（如果不设置 SameSite，现代浏览器默认 Lax） Set-Cookie: session=abc; SameSite=Lax // SameSite=None（不安全，需要 Secure） // 所有跨站请求都发送 // 必须同时设置 Secure（仅 HTTPS） Set-Cookie: session=abc; SameSite=None; Secure

Secure 与 HttpOnly

// Secure — 仅通过 HTTPS 传输 // 如果页面是 HTTP，设置了 Secure 的 Cookie 不会被发送 Set-Cookie: token=xyz; Secure // HttpOnly — JS 无法读取（不可通过 document.cookie 访问） // 只能通过 HTTP 请求发送到服务器 // 有效防御 XSS 攻击窃取 Cookie Set-Cookie: session=abc; HttpOnly // __Secure- 前缀 // 带有此前缀的 Cookie 必须设置 Secure 属性 Set-Cookie: __Secure-token=xyz; Secure // __Host- 前缀（更严格） // 必须: Secure, Path=/, 不能设置 Domain Set-Cookie: __Host-session=abc; Secure; Path=/ // 常用组合 Set-Cookie: sessionId=abc123; HttpOnly; Secure; SameSite=Lax; Path=/; Max-Age=86400

Domain 与 Path

// Domain — 指定 Cookie 作用的域名 // 如果不设置，默认为当前域名（不含子域名） Set-Cookie: lang=zh-CN; Domain=.example.com; Path=/ // 所有 *.example.com 的子域名都能读取 // Path — 指定路径 Set-Cookie: theme=dark; Path=/app/ // 只有在 /app/ 路径下才发送 // Max-Age vs Expires // Max-Age: 相对时间（秒），优先级高于 Expires // Expires: 绝对时间 Set-Cookie: session=abc; Max-Age=3600 // 1 小时后过期 Set-Cookie: old=xyz; Expires=Thu, 01 Jan 2020 00:00:00 GMT // 已过期 // 删除 Cookie: 设置 Max-Age=0 或 Expires 为过去时间 document.cookie = 'key=; Max-Age=0; Path=/';
