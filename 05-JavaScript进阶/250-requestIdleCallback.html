requestIdleCallback

空闲回调、deadline.timeRemaining、合作式调度、后台任务。

requestIdleCallback 语法

// ========== requestIdleCallback ========== // 在浏览器空闲时执行低优先级任务 const handle = requestIdleCallback(callback, options); // 回调接收 IdleDeadline 对象 requestIdleCallback((deadline) => { // deadline.timeRemaining() — 当前帧剩余时间 (ms) // deadline.didTimeout — 是否因超时而执行 while (deadline.timeRemaining() > 0 && tasks.length > 0) { processTask(tasks.shift()); } if (tasks.length > 0) { requestIdleCallback(processTasks); // 继续调度 } }); // options.timeout — 最大延迟时间 (ms) // 超时后，即使没有空闲时间也会执行 requestIdleCallback(callback, { timeout: 3000 }); // ========== cancelIdleCallback ========== cancelIdleCallback(handle); // ========== 与 requestAnimationFrame 对比 ========== // rAF: 在下一帧渲染前执行 (高优先级) // rIC: 在帧空闲时执行 (低优先级) // rAF: 适合动画/布局测量 // rIC: 适合非关键任务/数据上报/日志 // ========== 应用场景 ========== // ✅ 数据上报和分析 // ✅ 大型数据格式化 // ✅ 图片延迟解码 // ✅ 缓存清理/预计算 // ✅ 非关键 DOM 操作
⚠️ requestIdleCallback 不保证一定会执行。
💡 使用 timeout 作为保底策略。
💡 不要在 rIC 中操作 DOM (可能引起重排)。
