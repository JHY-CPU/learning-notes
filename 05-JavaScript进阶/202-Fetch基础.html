Fetch 基础

现代浏览器 HTTP 请求 API，基于 Promise。

基本用法

// 基本 GET 请求 fetch('https://api.example.com/data') .then(response => { // Response 对象 response.ok; // true if status 200-299 response.status; // HTTP 状态码（200, 404, 500...） response.statusText; // 状态文本（'OK', 'Not Found'） response.url; // 请求的 URL response.headers; // Headers 对象 response.type; // 'basic' | 'cors' | 'error' response.redirected; // 是否被重定向 }); // 解析响应体 response.json(); // 解析为 JSON（最常用） response.text(); // 解析为纯文本 response.blob(); // 解析为 Blob（二进制） response.arrayBuffer(); // 解析为 ArrayBuffer response.formData(); // 解析为 FormData // 完整示例 fetch('https://jsonplaceholder.typicode.com/todos/1') .then(res => { if (!res.ok) throw new Error(`HTTP ${res.status}`); return res.json(); }) .then(data => console.log(data)) .catch(err => console.error('请求失败:', err)); // ⚠️ fetch 只在网络错误时 reject（4xx/5xx 不会 reject！） // ⚠️ 必须手动检查 response.ok 来处理 HTTP 错误
💡 fetch 默认不携带 Cookie（需要 credentials: 'include'）。
⚠️ 跨域请求需要服务器支持 CORS。
💡 res.json() 只能调用一次，body 只能被读取一次。

Headers API

// Headers 操作 const headers = new Headers(); headers.append('Content-Type', 'application/json'); headers.set('Authorization', 'Bearer token'); headers.get('Content-Type'); // 'application/json' headers.has('Authorization'); // true headers.delete('Authorization'); headers.forEach((v, k) => console.log(k, v)); // 常见请求头 // 'Content-Type': 'application/json' // 'Accept': 'application/json' // 'Authorization': 'Bearer ' // 'Cache-Control': 'no-cache'
