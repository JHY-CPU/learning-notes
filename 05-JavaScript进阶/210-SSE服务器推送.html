SSE 服务器推送

Server-Sent Events — 服务器单向推送数据到客户端。

基本用法

// 创建 EventSource（连接到 SSE 端点） const source = new EventSource('/api/events'); // 事件 source.onopen = () => { // 连接建立 console.log('SSE 连接已建立'); }; source.onmessage = (e) => { // 收到消息（未命名事件） console.log(e.data); // 服务器发送的数据 // 服务器格式: "data: 内容\n\n" }; source.onerror = () => { // 连接错误 console.error('SSE 出错'); }; // 自定义事件（服务器用 event: 指定事件名） source.addEventListener('chat', (e) => { console.log('聊天消息:', e.data); }); source.addEventListener('notification', (e) => { console.log('通知:', e.data); }); // 关闭连接 source.close(); // 与 WebSocket 对比： // ✅ SSE 更简单（只使用 HTTP） // ✅ SSE 自动重连（浏览器内置） // ✅ SSE 更适合服务器→客户端的单向推送 // ❌ SSE 只支持文本（不支持二进制） // ❌ SSE 只能服务器→客户端（不能双向） // ❌ SSE 连接数限制（浏览器最多 6 个） // 服务器格式示例 (Node.js): // res.writeHead(200, { // 'Content-Type': 'text/event-stream', // 'Cache-Control': 'no-cache', // 'Connection': 'keep-alive', // }); // res.write('data: {"msg":"hello"}\n\n');
