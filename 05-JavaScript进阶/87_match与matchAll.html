match 与 matchAll

String.match 全局匹配、String.matchAll 迭代器、捕获组获取。

match vs matchAll

// ========== String.prototype.match() ========== // 无 g flag: 返回第一个完整匹配 + 捕获组 '2024-01-15'.match(/(\d{4})-(\d{2})-(\d{2})/); // ['2024-01-15', '2024', '01', '15', index: 0, groups: undefined] // 有 g flag: 返回所有完整匹配（无捕获组） '2024-01-15, 2024-06-20'.match(/\d{4}-\d{2}-\d{2}/g); // ['2024-01-15', '2024-06-20'] // ⚠️ g flag 时不返回捕获组 '2024-01-15'.match(/(\d{4})-\d{2}-\d{2}/g); // ['2024-01-15'] (没有捕获组!) // ========== String.prototype.matchAll() (ES2020) ========== // 始终返回迭代器，包含捕获组 // 必须使用 g flag const matches = '2024-01-15, 2024-06-20'.matchAll(/(\d{4})-(\d{2})-(\d{2})/g); for (const m of matches) { console.log(m[0]); // 完整匹配 console.log(m[1]); // 年 console.log(m.index); // 索引 } // 转为数组 const arr = [...'2024-01-15'.matchAll(/(\d{4})-(\d{2})-(\d{2})/g)]; // ========== 对比总结 ========== // match (无 g) — 返回第一个匹配 + 捕获组 (类数组) // match (有 g) — 返回所有完整匹配 (数组，无捕获组) // matchAll — 返回迭代器，每个元素包含捕获组 // matchAll — 需要 g flag，否则抛 TypeError
