XMLHttpRequest

传统 AJAX 请求方式，支持进度监听。

基本用法

// 创建 const xhr = new XMLHttpRequest(); // 配置请求 xhr.open(method, url, async); // method: 'GET' | 'POST' | 'PUT' | 'DELETE' // url: 请求地址 // async: true（异步，默认）| false（同步，已弃用） // 设置请求头 xhr.setRequestHeader('Content-Type', 'application/json'); xhr.setRequestHeader('Authorization', 'Bearer token'); // 事件 xhr.onloadstart = () => {}; // 开始 xhr.onprogress = (e) => { // 进度（下载） e.loaded; // 已下载字节 e.total; // 总字节（可能为0） }; xhr.onload = () => { // 加载完成 xhr.responseText; // 响应文本 xhr.response; // 响应体（根据 responseType） xhr.status; // 状态码 xhr.statusText; // 状态文本 }; xhr.onerror = () => {}; // 网络错误 xhr.ontimeout = () => {}; // 超时 xhr.onabort = () => {}; // 中止 xhr.onloadend = () => {}; // 请求结束（无论成功/失败） // readyState // 0: UNSENT — 未调用 open // 1: OPENED — 已调用 open // 2: HEADERS_RECEIVED — 已收到响应头 // 3: LOADING — 下载中 // 4: DONE — 请求完成 // 响应类型 xhr.responseType = 'json'; // 'text' | 'json' | 'blob' | 'arraybuffer' | 'document' // 发送 xhr.send(); // GET xhr.send(JSON.stringify(data)); // POST/PUT（带 body） // 超时 xhr.timeout = 10000; // 10s 超时 // 中止 xhr.abort(); // 获取响应头 xhr.getResponseHeader('Content-Type'); xhr.getAllResponseHeaders();

与 Fetch 对比

📊 XHR vs Fetch:
✅ XHR 支持上传/下载进度监听（fetch 需要自行扩展）
✅ XHR 支持超时设置（fetch 需要 AbortController）
✅ XHR 支持同步请求（但已不推荐使用）
❌ XHR 不支持 Stream
❌ XHR 不支持 Service Worker
❌ XHR API 设计较老（事件回调 vs Promise）
💡 新项目推荐使用 fetch，只有需要进度监听时用 XHR。
