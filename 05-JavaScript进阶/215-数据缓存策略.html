数据缓存策略

内存缓存 / localStorage 缓存 / Cache-API / LRU / SWR 模式。

内存缓存 (Map)

// 最简单的内存缓存 const cache = new Map(); function getCached(key, fn) { if (cache.has(key)) { return Promise.resolve(cache.get(key)); } return fn().then(val => { cache.set(key, val); return val; }); } // 带 TTL 的缓存 class TTLCache { constructor(ttlMs = 60000) { this.cache = new Map(); this.ttl = ttlMs; } get(key) { const item = this.cache.get(key); if (!item) return undefined; if (Date.now() - item.time > this.ttl) { this.cache.delete(key); return undefined; } return item.value; } set(key, value) { this.cache.set(key, { value, time: Date.now() }); } delete(key) { this.cache.delete(key); } clear() { this.cache.clear(); } }

SWR 模式

// Stale-While-Revalidate: 先返回缓存，再后台更新 async function swrFetch(key, fetchFn) { const cached = localStorage.getItem(key); if (cached) { // 异步更新缓存（不阻塞 UI） fetchFn().then(data => { localStorage.setItem(key, JSON.stringify(data)); }).catch(() => {}); return JSON.parse(cached); // 立即返回旧的 } const data = await fetchFn(); localStorage.setItem(key, JSON.stringify(data)); return data; } // 使用 const data = await swrFetch('posts', () => fetch('/api/posts').then(r => r.json()) );
💡 SWR 模式在用户体验上非常好：秒开旧数据 → 后台更新。
💡 内存缓存最快但页面刷新丢失，localStorage 持久但读取慢。
💡 Cache-API 专为 HTTP 请求设计，配合 Service Worker 使用。
