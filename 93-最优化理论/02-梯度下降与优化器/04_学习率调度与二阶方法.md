# 学习率调度与二阶方法 — 最优化理论笔记


## 一、学习率的重要性


学习率是深度学习训练中**最重要的超参数**。研究和实践表明：


- 学习率太大：训练震荡、发散（loss出现NaN）
- 学习率太小：收敛极慢，可能卡在次优解
- 最优学习率随训练进程变化：初期需要大步探索，后期需要小步微调


> **Important:** **关键发现：**
> Smith (2017) 发现学习率的大小比优化器的选择对最终性能影响更大。一个好的学习率调度策略往往能带来显著的性能提升。


## 二、常见学习率调度策略


### 2.1 StepLR（阶梯衰减）


$$
\(\alpha_t = \alpha_0 \cdot \gamma^{\lfloor t / s \rfloor}\)
$$


每 \( s \) 个epoch，学习率乘以衰减因子 \( \gamma \)（通常0.1或0.5）。


- 简单直接，早期深度学习的标准做法
- 缺点：衰减不平滑，可能在衰减点丢失信息


### 2.2 MultiStepLR（多阶梯衰减）


在指定的里程碑处衰减：如 [30, 60, 90] epoch 各衰减一次。


- 比StepLR更灵活
- ResNet论文中使用的策略


### 2.3 ExponentialLR（指数衰减）


$$
\(\alpha_t = \alpha_0 \cdot \gamma^t\)
$$


每个epoch都衰减，连续平滑。


### 2.4 CosineAnnealing（余弦退火）


$$
\(\alpha_t = \alpha_{\min} + \frac{1}{2}(\alpha_{\max} - \alpha_{\min})\left(1 + \cos\left(\frac{t}{T_{\max}}\pi\right)\right)\)
$$


学习率沿余弦曲线从 \( \alpha_{\max} \) 平滑下降到 \( \alpha_{\min} \)。


- 起始和结束时变化慢，中间变化快——非常符合训练直觉
- 是目前CV领域最流行的调度策略
- 结合Warmup使用效果更好


### 2.5 Warmup + Decay


$$
\(\alpha_t = \begin{cases} \alpha_{\max} \cdot \frac{t}{T_{\text{warmup}}} & t < T_{\text{warmup}} \\ \text{decay}(t) & t \ge T_{\text{warmup}} \end{cases}\)
$$


训练开始时学习率从0线性增大到目标值（warmup阶段），然后开始衰减。


- **Warmup的必要性：**
   训练初期模型参数随机，大梯度配合大步长容易导致训练不稳定
- Transformer模型
   **必须**
   使用warmup，否则训练几乎一定会发散
- BERT：Warmup + Linear Decay
- GPT：Warmup + Cosine Decay


### 2.6 OneCycleLR


Smith & Topin (2019) 提出的"超收敛"策略：


- 第一阶段：学习率从低到高（warmup）再到低（cosine decay）
- 第二阶段（可选）：从低到更低（继续衰减）
- 同时配合动量的反向变化：学习率大时动量小，学习率小时动量大
- 可以在更少的epoch内达到更好的效果


### 2.7 调度策略对比


| 策略 | 数学特性 | 适用场景 | 超参数 |
| --- | --- | --- | --- |
| StepLR | 分段常数 | 简单任务 | step_size, gamma |
| CosineAnnealing | 余弦平滑 | CV训练 | T_max, eta_min |
| Warmup+Cosine | 线性+余弦 | Transformer/NLP | warmup_steps, total_steps |
| OneCycleLR | 超收敛 | 快速训练 | max_lr, total_steps |
| ReduceOnPlateau | 自适应 | 验证集驱动 | patience, factor |


## 三、二阶方法：牛顿法


### 3.1 动机


一阶方法（梯度下降）只利用了梯度信息。二阶方法同时利用梯度和曲率（Hessian矩阵），可以找到更好的下降方向。


### 3.2 牛顿法（Newton's Method）


对 \( f(x) \) 做二阶Taylor展开：


$$
\(f(x + \Delta x) \approx f(x) + \nabla f(x)^T \Delta x + \frac{1}{2}\Delta x^T H(x) \Delta x\)
$$


令导数为零，得到**牛顿步**：


$$
\(x_{t+1} = x_t - H(x_t)^{-1} \nabla f(x_t)\)
$$


其中 \( H(x) = \nabla^2 f(x) \) 是Hessian矩阵。


### 3.3 牛顿法的优缺点


| 优点 | 缺点 |
| --- | --- |
| 二次收敛（极快） | 需要计算和存储Hessian \( O(n^2) \) |
| 不需要学习率（或线搜索） | 求逆Hessian \( O(n^3) \) |
| 对缩放不敏感 | Hessian可能非正定（非凸问题） |
| 凸问题直接收敛 | 深度学习中参数量巨大（亿级），完全不可行 |


> **Important:** **现实：**
> 对于一个有1亿参数的模型，Hessian矩阵需要 \( 10^{16} \) 个浮点数（约40TB存储），完全无法使用。这就是为什么深度学习几乎不用纯牛顿法。


## 四、拟牛顿法（Quasi-Newton Methods）


### 4.1 核心思想


不直接计算Hessian，而是用梯度信息**近似** Hessian（或其逆），大幅降低计算成本。


### 4.2 BFGS


Broyden-Fletcher-Goldfarb-Shanno算法是最流行的拟牛顿法：


- 维护Hessian逆矩阵的近似 \( B_t \approx H^{-1} \)
- 每步用秩-2更新更新 \( B_t \)
- 存储复杂度 \( O(n^2) \)，仍不适合超大模型
- 收敛速度超线性（介于线性和二次之间）


### 4.3 L-BFGS（Limited-memory BFGS）


不存储完整的近似矩阵，而是用最近 \( m \) 步的梯度和参数差（向量对）隐式表示：


- 存储复杂度降至 \( O(mn) \)（通常 \( m=5\sim20 \)）
- 适合中等规模问题（参数量在百万级）
- scipy.optimize.minimize 中的 'L-BFGS-B' 方法
- 在小数据集的机器学习问题中非常有效
- 不适合SGD场景（需要全梯度）


> **Note:** **什么时候用L-BFGS：**
>
>
> - 数据集小，可以放在内存中
>
>
> - 参数量不大（百万级以内）
>
>
> - 需要高精度解
>
>
> - 例：逻辑回归、小规模神经网络的微调


## 五、自然梯度下降简介


### 5.1 动机


标准梯度下降在参数空间中做欧氏距离的最速下降，但参数空间的几何结构可能非常不均匀。自然梯度在**概率分布空间**中做最速下降。


### 5.2 自然梯度


$$
\(\tilde{\nabla} f(\theta) = F(\theta)^{-1} \nabla f(\theta)\)
$$


其中 \( F(\theta) \) 是**Fisher信息矩阵**：


$$
\(F(\theta) = \mathbb{E}_{p(x|\theta)}\left[\nabla_\theta \log p(x|\theta) \nabla_\theta \log p(x|\theta)^T\right]\)
$$


### 5.3 与Adam的关系


- Adam的二阶矩估计可以看作对Fisher信息矩阵对角近似的在线估计
- K-FAC（Kronecker-factored Approximate Curvature）是自然梯度的高效近似
- 自然梯度提供了理解自适应优化器的理论框架


## 六、Python代码：学习率调度与二阶方法


```
import numpy as np
import matplotlib.pyplot as plt

# ==========================================
# 学习率调度策略可视化
# ==========================================
def step_lr(lr0, epoch, step_size=30, gamma=0.1):
    return lr0 * gamma ** (epoch // step_size)

def cosine_annealing(lr_max, epoch, T_max, lr_min=0):
    return lr_min + 0.5 * (lr_max - lr_min) * (1 + np.cos(np.pi * epoch / T_max))

def warmup_cosine(lr_max, epoch, warmup_epochs, total_epochs):
    if epoch < warmup_epochs:
        return lr_max * epoch / warmup_epochs
    else:
        progress = (epoch - warmup_epochs) / (total_epochs - warmup_epochs)
        return lr_max * 0.5 * (1 + np.cos(np.pi * progress))

def one_cycle(lr_max, epoch, total_epochs):
    mid = total_epochs // 2
    if epoch < mid:
        return lr_max * 0.5 * (1 + np.cos(np.pi * (mid - epoch) / mid))
    else:
        return lr_max * 0.5 * (1 + np.cos(np.pi * (epoch - mid) / (total_epochs - mid)))

# 绘制对比图
epochs = np.arange(0, 200)
fig, axes = plt.subplots(2, 2, figsize=(14, 10))

axes[0, 0].plot(epochs, [step_lr(0.1, e) for e in epochs])
axes[0, 0].set_title('StepLR (step=30, γ=0.1)')

axes[0, 1].plot(epochs, [cosine_annealing(0.1, e, 200) for e in epochs])
axes[0, 1].set_title('CosineAnnealing')

axes[1, 0].plot(epochs, [warmup_cosine(0.1, e, 20, 200) for e in epochs])
axes[1, 0].set_title('Warmup(20) + Cosine')

axes[1, 1].plot(epochs, [one_cycle(0.1, e, 200) for e in epochs])
axes[1, 1].set_title('OneCycleLR')

for ax in axes.flat:
    ax.set_xlabel('Epoch')
    ax.set_ylabel('Learning Rate')
    ax.grid(True, alpha=0.3)

plt.suptitle('学习率调度策略对比', fontsize=15)
plt.tight_layout()
plt.savefig('lr_schedules.png', dpi=150)
plt.show()

# ==========================================
# L-BFGS求解器示例（scipy）
# ==========================================
from scipy.optimize import minimize

def rosenbrock(x):
    return (1 - x[0])**2 + 100*(x[1] - x[0]**2)**2

def rosenbrock_grad(x):
    dx = -2*(1 - x[0]) - 400*x[0]*(x[1] - x[0]**2)
    dy = 200*(x[1] - x[0]**2)
    return np.array([dx, dy])

x0 = np.array([-1.0, 1.0])

# L-BFGS-B
result_lbfgs = minimize(rosenbrock, x0, jac=rosenbrock_grad, method='L-BFGS-B')
print("=== L-BFGS-B 优化结果 ===")
print(f"最优解: ({result_lbfgs.x[0]:.6f}, {result_lbfgs.x[1]:.6f})")
print(f"最优值: {result_lbfgs.fun:.2e}")
print(f"函数调用次数: {result_lbfgs.nfev}")
print(f"成功: {result_lbfgs.success}")

# BFGS
result_bfgs = minimize(rosenbrock, x0, jac=rosenbrock_grad, method='BFGS')
print(f"\n=== BFGS 优化结果 ===")
print(f"最优解: ({result_bfgs.x[0]:.6f}, {result_bfgs.x[1]:.6f})")
print(f"最优值: {result_bfgs.fun:.2e}")
print(f"函数调用次数: {result_bfgs.nfev}")

# Newton-CG (需要Hessian)
def rosenbrock_hess(x):
    h11 = 2 - 400*(x[1] - x[0]**2) + 800*x[0]**2
    h12 = -400*x[0]
    h22 = 200
    return np.array([[h11, h12], [h12, h22]])

result_newton = minimize(rosenbrock, x0, jac=rosenbrock_grad,
                         hess=rosenbrock_hess, method='Newton-CG')
print(f"\n=== Newton-CG 优化结果 ===")
print(f"最优解: ({result_newton.x[0]:.6f}, {result_newton.x[1]:.6f})")
print(f"最优值: {result_newton.fun:.2e}")
print(f"函数调用次数: {result_newton.nfev}")

# ==========================================
# PyTorch学习率调度示例
# ==========================================
print("\n=== PyTorch调度器（模拟）===")
# 模拟参数
lr_max = 0.1
T_max = 100
warmup = 10

for t in [0, 5, 10, 25, 50, 75, 99]:
    lr = warmup_cosine(lr_max, t, warmup, T_max)
    print(f"  Epoch {t:3d}: lr = {lr:.6f}")
```


## 七、实践建议总结


| 任务类型 | 推荐优化器 | 推荐学习率调度 | 说明 |
| --- | --- | --- | --- |
| 图像分类（ResNet等） | SGD + Momentum(0.9) | CosineAnnealing 或 MultiStepLR | SGD泛化更好 |
| 目标检测（YOLO等） | SGD 或 Adam | Warmup + StepLR | 初始训练用Warmup |
| NLP（Transformer） | AdamW | Warmup + Linear/Cosine Decay | Warmup是必须的 |
| LLM预训练 | AdamW | Warmup + Cosine Decay | 学习率峰值很小（1e-4量级） |
| GAN训练 | Adam | 恒定或缓慢衰减 | lr=0.0002, β1=0.5 |
| 强化学习 | Adam | 线性衰减 | lr=3e-4 常见 |
| 小数据集传统ML | L-BFGS | 不需要（二阶方法自适应） | scipy.optimize |


### 通用建议


1. **先用Adam快速实验：**
   默认参数就能工作，快速验证想法
2. **确定方向后用SGD精调：**
   CV任务中SGD+Cosine往往最终更好
3. **学习率搜索：**
   使用LR Finder（从小到大线性增大lr，找到loss最快下降区间）
4. **Warmup几乎总是一个好选择：**
   特别是大模型和大batch训练
5. **Batch size与学习率线性缩放：**
   batch增大k倍，lr也增大k倍（有上限）
6. **梯度裁剪：**
   不管用什么调度器，加梯度裁剪（max_norm=1.0）总是安全的


> **Note:** **总结：**
> 学习率调度是训练深度学习模型的关键环节。从简单的StepLR到优雅的CosineAnnealing再到完整的Warmup+Decay，选择合适的调度策略可以大幅提升训练效率和最终性能。二阶方法（牛顿法、L-BFGS）在小规模问题上仍有一席之地，而自然梯度的理论为理解现代优化器提供了统一框架。
