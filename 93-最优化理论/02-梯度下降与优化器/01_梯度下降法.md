# 梯度下降法 — 最优化理论笔记


## 一、梯度的几何意义


### 1.1 梯度的定义


对于函数 \( f: \mathbb{R}^n \to \mathbb{R} \)，在点 \( x \) 处的**梯度**为：


$$
\(\nabla f(x) = \left(\frac{\partial f}{\partial x_1}, \frac{\partial f}{\partial x_2}, \dots, \frac{\partial f}{\partial x_n}\right)^T\)
$$


### 1.2 梯度的几何含义


- **梯度方向**
   是函数在该点增长最快的方向
- **负梯度方向**
   （\( -\nabla f(x) \)）是函数下降最快的方向
- **梯度的模**
   \( \|\nabla f(x)\| \) 表示该方向上的变化率
- 梯度与等高线（等值面）垂直


> **Note:** **直觉：**
> 想象你站在山坡上，蒙着眼睛想找最低点。梯度告诉你"脚下哪个方向最陡"，你沿着最陡的方向走一小步，这就是梯度下降。


### 1.3 梯度下降的核心迭代公式


$$
\(x_{t+1} = x_t - \alpha \nabla f(x_t)\)
$$


其中 \( \alpha > 0 \) 是**学习率**（learning rate），也叫步长（step size）。


## 二、三种梯度下降变体


### 2.1 批量梯度下降（BGD, Batch Gradient Descent）


每次使用**全部**训练数据计算梯度：


$$
\(\nabla f(x) = \frac{1}{N}\sum_{i=1}^{N} \nabla f_i(x)\)
$$


- 梯度方向准确，收敛稳定
- 每次迭代计算量大（需遍历全部数据）
- 内存消耗高（需存储全部数据）
- 可能陷入鞍点（因为梯度精确）


### 2.2 随机梯度下降（SGD, Stochastic Gradient Descent）


每次随机选**一个**样本计算梯度：


$$
\(x_{t+1} = x_t - \alpha \nabla f_{i_t}(x_t), \quad i_t \sim \text{Uniform}\{1,\dots,N\}\)
$$


- 每次迭代计算量极小（只算一个样本）
- 梯度有噪声，但噪声有助于跳出局部极小和鞍点
- 收敛轨迹震荡剧烈
- 可以处理无法全部加载到内存的数据


### 2.3 小批量梯度下降（Mini-batch Gradient Descent）


每次使用一小批（batch）样本计算梯度：


$$
\(\nabla f(x) = \frac{1}{|B|}\sum_{i \in B} \nabla f_i(x), \quad B = \{i_1, \dots, i_b\}\)
$$


- 结合BGD和SGD的优点
- 可以利用GPU并行计算
- batch size 通常为 32/64/128/256
- 是深度学习中的
   **标准做法**
   ，日常所说的"SGD"通常指Mini-batch GD


### 2.4 三种方法对比


| 特性 | BGD | SGD | Mini-batch GD |
| --- | --- | --- | --- |
| 每次使用样本数 | 全部 N | 1 | b（通常32~256） |
| 梯度准确性 | 精确 | 噪声大 | 适中 |
| 收敛稳定性 | 稳定 | 震荡 | 较稳定 |
| 单步计算开销 | 高 | 极低 | 适中 |
| 内存消耗 | 高 | 低 | 适中 |
| 跳出鞍点能力 | 弱 | 强 | 中等 |
| GPU利用率 | 高 | 低 | 高 |
| 适用场景 | 小数据集 | 在线学习 | 深度学习（标准） |


## 三、学习率的影响


### 3.1 学习率太大的问题


- 步长越过最优点，在最优解附近剧烈震荡
- 可能导致发散（loss越来越大的NaN）
- 在"峡谷"形损失函数中尤其严重


### 3.2 学习率太小的问题


- 收敛速度极慢，训练时间过长
- 容易困在鞍点或平坦区域
- 可能提前停止在非最优点


### 3.3 理论保证


对于L-光滑（\( \|\nabla f(x) - \nabla f(y)\| \le L\|x-y\| \)）的凸函数：


- 固定学习率 \( \alpha \le \frac{1}{L} \)，BGD收敛
- SGD在 \( \sum \alpha_t = \infty, \sum \alpha_t^2 < \infty \) 条件下收敛（如 \( \alpha_t = 1/t \)）
- 凸函数 + 适当学习率 → 收敛到全局最优
- 非凸问题 → 只保证收敛到驻点（可能是鞍点）


## 四、鞍点与局部极小值问题


### 4.1 什么是鞍点


**鞍点（Saddle Point）**是梯度为零但既不是局部极大也不是局部极小的点。在某些方向是极小，某些方向是极大。


$$
\(f(x, y) = x^2 - y^2 \quad \text{的原点是鞍点}\)
$$


### 4.2 高维空间中的鞍点问题


- 在高维空间中，鞍点比局部极小值更常见
- Hessian矩阵在鞍点处有正特征值和负特征值
- SGD的随机噪声有助于逃离鞍点
- 在深度学习中，"尖锐的局部极小"和"平坦的局部极小"的泛化性能不同——平坦的通常更好


> **Important:** **实践观察：**
> 在深度学习中，过度参数化的模型（参数远多于数据点）的损失面虽然充满鞍点和局部极小，但大多数局部极小的损失值非常接近全局最优。真正的问题更多是鞍点导致的训练变慢，而非陷入差的局部极小。


## 五、Python代码：手写SGD


```
import numpy as np
import matplotlib.pyplot as plt

# ==========================================
# 手写SGD优化 Rosenbrock 函数
# f(x,y) = (1-x)^2 + 100(y-x^2)^2
# 全局最优: (1,1), f*=0
# ==========================================
def rosenbrock(x):
    return (1 - x[0])**2 + 100 * (x[1] - x[0]**2)**2

def rosenbrock_grad(x):
    dx = -2*(1 - x[0]) - 400*x[0]*(x[1] - x[0]**2)
    dy = 200*(x[1] - x[0]**2)
    return np.array([dx, dy])

def sgd_optimize(func, grad, x0, lr=0.001, n_iter=10000, noise_scale=0.0):
    """SGD优化（可选添加梯度噪声）"""
    x = x0.copy()
    path = [x.copy()]
    losses = [func(x)]

    for t in range(n_iter):
        g = grad(x)
        if noise_scale > 0:
            g += noise_scale * np.random.randn(*x.shape)
        x = x - lr * g
        path.append(x.copy())
        losses.append(func(x))

    return x, np.array(path), losses

# 运行BGD（无噪声）和SGD（有噪声）
x0 = np.array([-1.0, 1.0])
x_opt_bgd, path_bgd, loss_bgd = sgd_optimize(
    rosenbrock, rosenbrock_grad, x0, lr=0.001, n_iter=5000)
x_opt_sgd, path_sgd, loss_sgd = sgd_optimize(
    rosenbrock, rosenbrock_grad, x0, lr=0.001, n_iter=5000, noise_scale=0.5)

print("=== 手写SGD优化Rosenbrock函数 ===")
print(f"BGD结果: ({x_opt_bgd[0]:.4f}, {x_opt_bgd[1]:.4f}), f={rosenbrock(x_opt_bgd):.6f}")
print(f"SGD结果: ({x_opt_sgd[0]:.4f}, {x_opt_sgd[1]:.4f}), f={rosenbrock(x_opt_sgd):.6f}")

# ==========================================
# 不同学习率的对比
# ==========================================
def quadratic(x):
    return 0.5 * (x[0]**2 + 10*x[1]**2)

def quadratic_grad(x):
    return np.array([x[0], 10*x[1]])

learning_rates = [0.01, 0.1, 0.19, 0.25]
fig, axes = plt.subplots(2, 2, figsize=(12, 10))

for idx, lr in enumerate(learning_rates):
    ax = axes[idx // 2][idx % 2]
    x = np.array([4.0, 3.0])
    path = [x.copy()]

    for t in range(50):
        x = x - lr * quadratic_grad(x)
        path.append(x.copy())

    path = np.array(path)

    # 画等高线
    xx, yy = np.meshgrid(np.linspace(-5, 5, 100), np.linspace(-5, 5, 100))
    zz = 0.5 * (xx**2 + 10*yy**2)
    ax.contour(xx, yy, zz, levels=20, cmap='viridis')
    ax.plot(path[:, 0], path[:, 1], 'ro-', markersize=3, linewidth=1.5)
    ax.plot(0, 0, 'r*', markersize=15)
    ax.set_title(f'学习率 α = {lr}', fontsize=13)
    ax.set_xlabel('x₁')
    ax.set_ylabel('x₂')

plt.suptitle('不同学习率下梯度下降的收敛轨迹', fontsize=15)
plt.tight_layout()
plt.savefig('learning_rate_effect.png', dpi=150)
plt.show()
```


## 六、收敛性分析与实践建议


### 6.1 收敛速度


| 函数类型 | 收敛速度 | 条件 |
| --- | --- | --- |
| 强凸 + 光滑 | 线性收敛 \( O((1-\mu/L)^k) \) | \( \alpha \le 2/(L+\mu) \) |
| 凸 + 光滑 | 次线性 \( O(1/k) \) | \( \alpha \le 1/L \) |
| 非凸 + 光滑 | 收敛到驻点 \( O(1/\sqrt{k}) \) | 驻点可能是鞍点 |
| SGD（强凸） | 次线性 \( O(1/k) \) | 递减学习率 |


### 6.2 实践建议


- 学习率是最敏感的超参数，优先调优
- 初始学习率可通过学习率搜索（LR Finder）确定
- 使用学习率调度（warmup + decay）通常比固定学习率好
- Mini-batch size 越大，梯度越准确但泛化性可能变差
- 梯度裁剪（gradient clipping）防止梯度爆炸
- 数据打乱（shuffle）对SGD收敛至关重要


> **Note:** **总结：**
> 梯度下降是最基本的优化算法。BGD精确但慢，SGD快速但有噪声，Mini-batch GD是实践中最常用的形式。学习率的选择直接影响收敛行为。后续章节将介绍动量法、Adam等改进方法来解决SGD的震荡和收敛慢问题。
