MongoDB 核心概念

// ========== MongoDB 核心概念 ========== // ┌──────────────┬──────────────────────────┐ // │ SQL │ MongoDB │ // ├──────────────┼──────────────────────────┤ // │ Database │ Database │ // │ Table │ Collection │ // │ Row │ Document │ // │ Column │ Field │ // │ Primary Key │ _id (自动生成 ObjectId) │ // │ Index │ Index │ // │ JOIN │ $lookup (聚合管道) │ // │ GROUP BY │ $group (聚合管道) │ // └──────────────┴──────────────────────────┘ // ========== 文档示例 ========== { "_id": ObjectId("5f8c1a5b6b7d3c1a2b3c4d5e"), "name": "Alice", "email": "alice@test.com", "age": 28, "address": { "city": "Beijing", "street": "Haidian Street" }, "tags": ["tech", "music"], "scores": [ {"subject": "Math", "score": 95}, {"subject": "English", "score": 88} ], "created_at": ISODate("2024-01-15T08:00:00Z") } // 特点: // 1. 文档是 JSON-like 结构 (BSON) // 2. 同一个集合的文档可以有不同的字段 // 3. 支持嵌套文档和数组 // 4. _id 是自动生成的主键

BSON 数据类型

// ========== BSON 类型 ========== // MongoDB 内部使用 BSON (Binary JSON) // ┌─────────────┬──────────────────┬─────────────────┐ // │ 类型 │ 示例 │ 说明 │ // ├─────────────┼──────────────────┼─────────────────┤ // │ String │ "hello" │ UTF-8 字符串 │ // │ Integer │ 42 │ 32/64 位整数 │ // │ Double │ 3.14 │ 浮点数 │ // │ Boolean │ true / false │ 布尔值 │ // │ ObjectId │ ObjectId(...) │ 12 字节唯一 ID │ // │ Date │ ISODate(...) │ 日期/时间 │ // │ Array │ [1, 2, 3] │ 数组 │ // │ Object │ {key: "val"} │ 内嵌文档 │ // │ Null │ null │ 空值 │ // │ BinData │ BinData(...) │ 二进制数据 │ // │ Timestamp │ Timestamp(...) │ 时间戳 (内部用) │ // │ Regex │ /pattern/ │ 正则表达式 │ // │ Decimal128 │ NumberDecimal(..)│ 高精度小数 (金融) │ // └─────────────┴──────────────────┴─────────────────┘ // ObjectId 结构 (12 bytes): // ┌──────────┬──────────┬────────────┬──────────┐ // │ 4 byte │ 5 byte │ 3 byte │ │ // │ 时间戳 │ 随机值 │ 计数器 │ = 24 │ // │ (秒级) │ (机器+PID)│ (自增) │ 位 hex │ // └──────────┴──────────┴────────────┴──────────┘ // ObjectId 特点: // - 客户端生成, 无需数据库 // - 包含时间戳, 可按 _id 排序约等于按时间排序

嵌入 vs 引用

// ========== 文档关系建模 ========== // ========== 嵌入 (Embedding) ========== // 把相关数据直接放入父文档 // ✅ 嵌入: 地址 (一对一) { _id: ObjectId("..."), name: "Alice", address: { city: "Beijing", street: "Haidian St", zip: "100000" } } // ✅ 嵌入: 订单项 (一对少) { _id: ObjectId("..."), total: 299.00, items: [ { product: "Book", price: 99, qty: 1 }, { product: "Pen", price: 200, qty: 1 } ] } // ❌ 不嵌入: 用户和订单 (一对多) // 每个用户有成千上万订单, 嵌入会超文档大小限制 (16MB) // ========== 引用 (Reference) ========== // 一个文档引用另一个文档的 _id // 用户文档 { _id: ObjectId("user123"), name: "Alice" } // 订单文档 (引用用户) { _id: ObjectId("order456"), user_id: ObjectId("user123"), // 引用 total: 299.00 } // ========== 选择指南 ========== // 嵌入时机: // - 数据经常一起读取 (has-a 关系) // - 子数据很少独立查询 // - 子数据量有限 (几到几十个) // - 子数据不会无限增长 // 引用时机: // - 数据独立查询 // - 子数据量大或无限增长 // - 多对多关系 // - 需要单独更新子文档 // 常见模式: // 1对1: 嵌入 ✅ // 1对少 (地址/标签): 嵌入 ✅ // 1对多 (订单/评论): 引用 // 多对多 (学生/课程): 引用

安装与命令行

// ========== MongoDB 安装与 mongosh ========== // ========== 安装 ========== // 1. 下载: https://www.mongodb.com/try/download/community // 2. 安装后启动 (默认端口 27017) // macOS (Homebrew): // brew install mongodb-community // brew services start mongodb-community // Docker: // docker run -d -p 27017:27017 --name mongo mongo:7 // ========== mongosh 基本命令 ========== // 显示数据库 show dbs // 切换/创建数据库 (use 后自动创建) use mydb // 显示集合 show collections // 查看帮助 help // ========== 数据库操作 ========== // 创建/切换数据库 use shop // 删除数据库 db.dropDatabase() // ========== 集合操作 ========== // 查看所有集合 show collections // 创建集合 (可省略, 插入文档时自动创建) db.createCollection("users") // 删除集合 db.users.drop() // ========== 完整 CRUD 示例 ========== // 插入 db.users.insertOne({ name: "Alice", age: 28 }) db.users.insertMany([ { name: "Bob", age: 25 }, { name: "Carol", age: 30 } ]) // 查询 db.users.find({ age: { $gte: 25 } }) db.users.findOne({ name: "Alice" }) // 更新 db.users.updateOne( { name: "Alice" }, { $set: { age: 29 } } ) // 删除 db.users.deleteOne({ name: "Bob" })
💡 MongoDB 文档模型: BSON 支持丰富类型; 嵌入适用于一起读取的小数据; 引用适用于独立查询的大数据; 文档大小限制 16MB; _id 自动生成且唯一; 设计时从查询方式倒推文档结构。
