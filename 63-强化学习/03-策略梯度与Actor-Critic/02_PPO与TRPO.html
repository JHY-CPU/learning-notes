PPO与TRPO

强化学习策略梯度PPO

TRPO和PPO通过约束策略更新幅度来提高训练稳定性，是当前最流行的策略梯度算法。

TRPO动机：信任区域

问题：策略梯度的步长敏感 - 步长太大：策略崩溃，性能急剧下降 - 步长太小：收敛极慢 直觉：限制每次策略更新的幅度 - 新策略不应与旧策略相差太远 - 在"信任区域"内优化 TRPO (Schulman 2015)： 最大化目标函数，约束KL散度： maximize E[ π_θ(a|s) / π_θ_old(a|s) × A(s,a) ] subject to E[ KL(π_θ_old || π_θ) ] ≤ δ - π_θ_old：旧策略（采集数据的策略） - π_θ：新策略（要优化的策略） - δ：KL散度约束（通常0.01~0.05）

TRPO算法细节

核心组件： 1. 重要性采样目标函数： L(θ) = E[ π_θ(a|s) / π_θ_old(a|s) × A(s,a) ] = E[ r(θ) × A(s,a) ] 其中 r(θ) = π_θ(a|s) / π_θ_old(a|s) 2. KL散度约束： KL(π_θ_old || π_θ) = E[ log(π_θ_old(a|s)) - log(π_θ(a|s)) ] 3. 求解方式：共轭梯度 + 线搜索 a) 计算目标函数梯度 g = ∇L(θ) b) 用共轭梯度近似求解 H⁻¹g （H是Fisher信息矩阵） c) 更新步长 step = √(2δ / (gᵀ H⁻¹ g)) × H⁻¹g d) 线搜索：逐步减小步长直到满足KL约束 TRPO的问题： - 需要计算Fisher信息矩阵（二阶梯度） - 共轭梯度实现复杂 - 线搜索增加计算开销 - 实现困难 → PPO的出现就是为了简化TRPO

PPO (Proximal Policy Optimization)

PPO (Schulman 2017)： TRPO的简化版本，只用一阶梯度 PPO-Clip目标函数： L^CLIP(θ) = E[ min( r(θ)×A, clip(r(θ), 1-ε, 1+ε)×A ) ] 其中： - r(θ) = π_θ(a|s) / π_θ_old(a|s) 重要性采样比率 - ε：裁剪参数（通常0.1~0.2） - clip(r, 1-ε, 1+ε)：将r限制在[1-ε, 1+ε]区间 直觉解析： 当 A > 0（好动作）： - 想增大 r(θ)（增大动作概率） - 但clip在 1+ε 截断 - 有效梯度只在 r < 1+ε 时非零 当 A < 0（坏动作）： - 想减小 r(θ) - 但clip在 1-ε 截断 - 有效梯度只在 r > 1-ε 时非零 结果：r(θ)被限制在[1-ε, 1+ε]附近 → 策略更新幅度被限制 → 类似TRPO的信任区域
PPO不需要计算Fisher矩阵或共轭梯度，用简单的裁剪操作实现了类似TRPO的效果。ChatGPT的RLHF阶段使用的就是PPO。

PPO实现细节

PPO完整算法流程： 初始化策略参数 θ，值函数参数 φ for iteration = 1, 2, ...: 1. 用当前策略 π_θ 采集 N 条轨迹 2. 计算每步的优势估计 A_t（使用GAE） 3. 保存旧策略 π_θ_old ← π_θ 4. for epoch = 1, ..., K: // 多个epoch复用数据 for each minibatch: // 策略损失 r_t = π_θ(a_t|s_t) / π_θ_old(a_t|s_t) L_clip = min(r_t × A_t, clip(r_t, 1-ε, 1+ε) × A_t) // 值函数损失 L_vf = (V_φ(s_t) - V_target)² // 熵奖励（鼓励探索） L_entropy = -Σ π_θ(a|s) × log π_θ(a|s) // 总损失 L = -L_clip + c1 × L_vf - c2 × L_entropy 梯度下降更新 θ, φ 关键超参数： - ε (clip range): 0.1~0.2 - K (epochs): 3~10 - Minibatch size: 64~256 - c1 (vf coeff): 0.5~1.0 - c2 (entropy coeff): 0.01~0.05 - GAE λ: 0.95 - γ: 0.99

PPO-Clip 目标函数推导

目标：限制策略更新，让 π_θ 不要偏离 π_θ_old 太远 TRPO的做法：加KL散度约束 PPO的做法：直接在目标函数中裁剪 未裁剪的目标： J(θ) = E[ r(θ) × A ] 问题：如果 r(θ) 变化太大，可能导致策略崩溃 PPO-Clip的解决： L = E[ min( r(θ)×A, clip(r(θ), 1-ε, 1+ε)×A ) ] 当 A > 0（增大该动作概率有利）： L = min( r × A, (1+ε) × A ) → r ≤ 1+ε 时 L = r×A（正常梯度） → r > 1+ε 时 L = (1+ε)×A（梯度为0，停止增大） 当 A < 0（减小该动作概率有利）： L = min( r × A, (1-ε) × A ) → r ≥ 1-ε 时 L = r×A（正常梯度） → r < 1-ε 时 L = (1-ε)×A（梯度为0，停止减小） 效果：对好的动作，最多增大(1+ε)倍 对坏的动作，最多减小到(1-ε)倍

PPO是目前最广泛使用的策略梯度算法，在工业界被大量采用。其核心优势在于实现简单、稳定性好、超参数敏感度低。
