# 策略梯度方法


## 策略梯度方法


强化学习策略梯度PPO


策略梯度直接参数化策略并优化，适合连续动作空间。


## 策略梯度定理


```
核心思想：直接参数化策略 π_θ(a|s)，通过梯度上升最大化期望回报

目标函数：J(θ) = E_π[ Σ γ^t × Rt ]

策略梯度定理：
∇_θ J(θ) = E_π[ ∇_θ log π_θ(a|s) × Q^π(s,a) ]

直觉理解：
- 如果某个动作带来了高回报(Q值大)
- 就增大该动作的概率(∇log π方向)
- 反之减小概率

REINFORCE算法（最简单的策略梯度）：
1. 用当前策略π_θ采样一个完整回合
2. 计算每步的回报 Gt = Σ γ^k × R(t+k+1)
3. 更新：θ ← θ + α × γ^t × Gt × ∇log π_θ(At|St)

问题：
- 高方差：完整回报Gt波动很大
- 样本效率低：需要完整回合
- 解决：引入基线(baseline)、使用Actor-Critic
```


## 基线与优势函数


```
引入基线减少方差：
∇J(θ) = E[ ∇log π(a|s) × (Q(s,a) - b(s)) ]

- 基线b(s)不改变梯度的期望（零期望）
- 但可以大幅减少方差
- 最佳基线：b(s) = V(s)

优势函数 (Advantage Function)：
A(s,a) = Q(s,a) - V(s)
- 表示在状态s下，动作a比平均好多少
- A > 0：该动作优于平均
- A < 0：该动作劣于平均

优势函数的TD估计：
A(s,a) ≈ R + γV(s') - V(s) = TD误差δ
- 不需要完整的回报
- 只需一步的TD误差
- 大幅降低方差

GAE (Generalized Advantage Estimation)：
A^GAE(γ,λ) = Σ (γλ)^l × δ(t+l)
- λ=0：TD误差（低方差，高偏差）
- λ=1：蒙特卡洛优势（高方差，无偏差）
- 平衡偏差和方差
```


## Actor-Critic方法


```
Actor-Critic = 策略梯度(Actor) + 值函数(Critic)

┌─────────────────────────────────────────────┐
│  Actor（演员）                               │
│  - 策略网络 π_θ(a|s)                         │
│  - 决定采取什么动作                           │
│  - 使用Critic的评价来改进策略                  │
│                                             │
│  Critic（评论家）                            │
│  - 值函数网络 V_φ(s)                         │
│  - 评估当前状态的好坏                         │
│  - 提供优势函数估计                           │
│                                             │
│  更新：                                      │
│  Actor:  θ ← θ + α × A(s,a) × ∇log π(a|s) │
│  Critic: φ ← φ + β × (R + γV(s') - V(s))  │
└─────────────────────────────────────────────┘

A2C (Advantage Actor-Critic)：
- 同步并行版本
- 多个worker并行采样
- 汇总梯度后统一更新

A3C (Asynchronous Advantage Actor-Critic)：
- 异步并行版本
- 每个worker独立更新共享参数
- 不需要经验回放（异步本身提供多样性）
- 曾是Atari游戏的SOTA

A2C vs A3C：
- A2C：同步，GPU利用率更高
- A3C：异步，CPU友好但更新可能冲突
- 实际中A2C往往更稳定
```


## PPO (近端策略优化)


```
PPO（Schulman 2017）：
TRPO的简化版本，当前最流行的策略梯度算法

核心思想：限制策略更新幅度，防止灾难性下降

PPO-Clip目标函数：
L(θ) = E[ min( r(θ) × A, clip(r(θ), 1-ε, 1+ε) × A ) ]

其中：
- r(θ) = π_θ(a|s) / π_θ_old(a|s)（重要性采样比率）
- clip(r, 1-ε, 1+ε)：将r裁剪到[1-ε, 1+ε]
- ε：裁剪参数（通常0.1-0.2）

直觉：
- 如果A>0（好动作）：增大π(a|s)，但r不超过1+ε
- 如果A<0（坏动作）：减小π(a|s)，但r不低于1-ε
- 限制了单步更新幅度

PPO算法流程：
1. 收集数据：用当前策略π_old采样
2. 计算优势：GAE
3. 多个epoch优化：用小批量SGD最大化L(θ)
4. 更新旧策略：π_old ← π

PPO的优势：
- 实现简单（比TRPO简单很多）
- 样本效率高（数据复用多次）
- 稳定性好（clipping防止大更新）
- OpenAI Five（Dota 2）和ChatGPT都使用PPO
```


> **Note:** PPO是目前最广泛使用的策略梯度算法。ChatGPT的RLHF阶段就使用了PPO。
