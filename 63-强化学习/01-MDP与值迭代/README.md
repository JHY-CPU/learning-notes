# 01-MDP与值迭代

## 1. 强化学习概述

### 1.1 什么是强化学习

强化学习（Reinforcement Learning, RL）是机器学习的三大范式之一，研究智能体（Agent）如何在与环境（Environment）的交互中，通过试错（Trial and Error）学习最优行为策略，以最大化累积奖励。

### 1.2 与监督学习和无监督学习的区别

| 特征 | 监督学习 | 无监督学习 | 强化学习 |
|------|----------|------------|----------|
| 数据 | 带标签的输入输出对 | 无标签数据 | 无预设标签，通过奖励信号学习 |
| 反馈 | 明确的正确答案 | 无显式反馈 | 延迟的奖励信号 |
| 时序 | 样本独立同分布 | 样本独立同分布 | 序列决策，样本间高度相关 |
| 目标 | 最小化损失函数 | 发现数据结构 | 最大化累积奖励 |
| 探索 | 无需探索 | 无需探索 | 需要平衡探索与利用 |

关键区别在于：
- **无监督学习**没有外部反馈，而**强化学习**有奖励信号。
- **监督学习**有每一步的正确标签，而**强化学习**只有延迟的整体评价。
- 强化学习面临**探索-利用困境**（Exploration-Exploitation Dilemma）。

## 2. 马尔可夫决策过程（MDP）形式化定义

### 2.1 MDP五元组

马尔可夫决策过程（Markov Decision Process, MDP）是强化学习问题的标准数学框架，定义为五元组 $(S, A, P, R, \gamma)$：

- **$S$（状态空间 State Space）**：环境所有可能状态的集合。可以是有限集合或连续空间。
- **$A$（动作空间 Action Space）**：智能体可以采取的所有动作的集合。不同状态下可用的动作可以不同，记为 $A(s)$。
- **$P$（状态转移概率 Transition Probability）**：$P(s'|s, a)$ 表示在状态 $s$ 下执行动作 $a$ 后转移到状态 $s'$ 的概率。满足马尔可夫性：下一状态仅依赖于当前状态和动作。
- **$R$（奖励函数 Reward Function）**：$R(s, a, s')$ 表示从状态 $s$ 执行动作 $a$ 转移到 $s'$ 获得的即时奖励。有时简化为 $R(s, a)$ 或 $R(s)$。
- **$\gamma$（折扣因子 Discount Factor）**：$\gamma \in [0, 1]$，控制未来奖励的当前价值。$\gamma$ 越接近1，智能体越有远见；越接近0，越重视即时奖励。

### 2.2 马尔可夫性

状态 $S_t$ 具有马尔可夫性，当且仅当：

$$P(S_{t+1} | S_t) = P(S_{t+1} | S_1, S_2, \ldots, S_t)$$

即未来状态仅依赖于当前状态，与历史状态无关。这是MDP的核心假设。

### 2.3 智能体与环境的交互

在每个时间步 $t$：
1. 智能体观察当前状态 $S_t$
2. 根据策略选择动作 $A_t$
3. 环境返回奖励 $R_{t+1}$ 和下一状态 $S_{t+1}$

产生轨迹（Trajectory）：$S_0, A_0, R_1, S_1, A_1, R_2, \ldots$

## 3. 策略（Policy）

### 3.1 确定性策略

确定性策略 $\mu: S \rightarrow A$，将每个状态直接映射到一个确定的动作：

$$a = \mu(s)$$

### 3.2 随机性策略

随机性策略 $\pi(a|s)$ 给出在状态 $s$ 下选择动作 $a$ 的概率分布：

$$\pi(a|s) = P(A_t = a | S_t = s)$$

随机性策略的优势：
- 天然包含探索机制
- 在部分可观测环境中更鲁棒
- 便于梯度优化（可微）

## 4. 回报与折扣回报

### 4.1 回报（Return）

回报是从当前时刻开始到回合结束的所有折扣奖励之和：

$$G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1}$$

### 4.2 折扣因子的作用

- **数学保证**：当 $\gamma < 1$ 且奖励有界时，无穷级数收敛。
- **时间偏好**：即时奖励比未来奖励更有价值。
- **不确定性建模**：未来越远，不确定性越大。

## 5. 值函数

### 5.1 状态值函数 $V^\pi(s)$

状态值函数衡量在策略 $\pi$ 下，从状态 $s$ 开始的期望回报：

$$V^\pi(s) = \mathbb{E}_\pi[G_t | S_t = s] = \mathbb{E}_\pi\left[\sum_{k=0}^{\infty} \gamma^k R_{t+k+1} \bigg| S_t = s\right]$$

### 5.2 动作值函数 $Q^\pi(s, a)$

动作值函数衡量在策略 $\pi$ 下，从状态 $s$ 执行动作 $a$ 后继续按 $\pi$ 行动的期望回报：

$$Q^\pi(s, a) = \mathbb{E}_\pi[G_t | S_t = s, A_t = a]$$

### 5.3 两个值函数的关系

$$V^\pi(s) = \sum_{a} \pi(a|s) Q^\pi(s, a)$$

## 6. Bellman方程

### 6.1 Bellman期望方程

**状态值函数的Bellman期望方程**：

$$V^\pi(s) = \sum_{a} \pi(a|s) \sum_{s'} P(s'|s,a)[R(s,a,s') + \gamma V^\pi(s')]$$

**动作值函数的Bellman期望方程**：

$$Q^\pi(s, a) = \sum_{s'} P(s'|s,a)\left[R(s,a,s') + \gamma \sum_{a'} \pi(a'|s') Q^\pi(s', a')\right]$$

Bellman方程将值函数分解为即时奖励和后继状态值函数之和，体现了递归结构。

### 6.2 Bellman最优方程

最优值函数定义为所有策略中值函数的上界：

$$V^*(s) = \max_\pi V^\pi(s), \quad Q^*(s,a) = \max_\pi Q^\pi(s,a)$$

**Bellman最优方程**：

$$V^*(s) = \max_a \sum_{s'} P(s'|s,a)[R(s,a,s') + \gamma V^*(s')]$$

$$Q^*(s,a) = \sum_{s'} P(s'|s,a)\left[R(s,a,s') + \gamma \max_{a'} Q^*(s',a')\right]$$

最优策略可以通过最优值函数直接导出：

$$\pi^*(s) = \arg\max_a Q^*(s, a)$$

## 7. 策略评估（迭代策略评估）

迭代策略评估通过反复应用Bellman期望方程来计算给定策略 $\pi$ 的值函数：

$$V_{k+1}(s) = \sum_a \pi(a|s) \sum_{s'} P(s'|s,a)[R(s,a,s') + \gamma V_k(s')]$$

算法保证在 $\gamma < 1$ 时收敛到 $V^\pi$。收敛条件通常设为 $\max_s |V_{k+1}(s) - V_k(s)| < \theta$。

## 8. 策略改进定理

**策略改进定理**：设 $\pi$ 和 $\pi'$ 是两个策略，如果对所有状态 $s$ 有：

$$Q^\pi(s, \pi'(s)) \geq V^\pi(s)$$

则对所有状态 $s$ 有 $V^{\pi'}(s) \geq V^\pi(s)$。

基于此定理，可以通过贪心改进得到更好的策略：

$$\pi'(s) = \arg\max_a Q^\pi(s, a)$$

## 9. 策略迭代算法

策略迭代（Policy Iteration）交替执行两个步骤：

```
初始化：随机策略 π

重复:
    1. 策略评估：计算 V^π（迭代至收敛）
    2. 策略改进：π(s) = argmax_a Q^π(s, a)

直到策略不再变化
```

策略迭代保证在有限MDP中经过有限次迭代收敛到最优策略，因为策略数量有限且每步严格改进。

## 10. 值迭代算法

值迭代（Value Iteration）将策略评估和策略改进合并为一步，直接更新值函数：

$$V_{k+1}(s) = \max_a \sum_{s'} P(s'|s,a)[R(s,a,s') + \gamma V_k(s')]$$

```
初始化：V(s) = 0, 对所有 s

重复:
    对每个状态 s:
        V(s) = max_a Σ P(s'|s,a)[R + γV(s')]
直到收敛

输出：π*(s) = argmax_a Q*(s, a)
```

值迭代不需要完整的策略评估步骤，通常比策略迭代更高效。

## 11. 截断策略迭代

截断策略迭代（Truncated Policy Iteration）是策略评估不完全执行到收敛的折中方案：

- 在每轮策略评估中只执行有限次（$k$ 次）迭代
- 当 $k=1$ 时退化为值迭代
- 当 $k \to \infty$ 时为完整策略迭代

这提供了计算效率和收敛速度之间的灵活权衡。

## 12. 动态规划方法的局限性

以上方法都属于动态规划（Dynamic Programming），存在以下局限：

1. **需要完整模型**：必须知道状态转移概率 $P(s'|s,a)$ 和奖励函数 $R$，这在实际问题中往往不可用。
2. **维度灾难**：状态空间和动作空间的组合爆炸导致计算不可行。
3. **连续空间难以处理**：值函数的表格表示不适用于连续状态或动作空间。

这些局限推动了无模型（Model-Free）方法的发展，如蒙特卡洛和时序差分学习。
