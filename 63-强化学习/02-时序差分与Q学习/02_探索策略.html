探索策略

强化学习探索与利用探索策略

探索与利用的平衡是强化学习的核心问题之一，好的探索策略能显著加速学习过程。

探索-利用困境（Exploration-Exploitation Dilemma）

核心问题： - 利用（Exploitation）：选择当前已知最优的动作，最大化即时收益 - 探索（Exploration）：尝试不确定的动作，获取更多信息以改进策略 为什么需要探索？ ┌──────────────────────────────────────────────┐ │ Q(s, a) 表示动作 a 的估计价值 │ │ │ │ 情况：Q(s, a1) = 10, Q(s, a2) = ? │ │ │ │ 只利用：选择 a1（已知收益 10） │ │ 但 a2 的真实价值可能是 100！ │ │ 不探索就永远发现不了更优动作 │ │ │ │ 最优策略需要对所有动作都有足够了解 │ │ 而不只是选择当前最被高估的动作 │ └──────────────────────────────────────────────┘ Regret（遗憾）： - 定义：最优策略的累积收益 - 实际策略的累积收益 - 好的探索策略应该最小化累积遗憾 - Regret = Σ [Q*(s, a*) - Q*(s, a_t)]

ε-Greedy 策略

最简单和最常用的探索策略，以 ε 的概率随机探索。

ε-Greedy 定义： ┌──────────────────────────────────────────────┐ │ 以概率 ε：随机选择一个动作（均匀分布） │ │ 以概率 1-ε：选择当前最优动作（greedy） │ │ │ │ a = { random action, with probability ε │ │ { argmax_a Q(s, a), with probability 1-ε│ └──────────────────────────────────────────────┘ 实现： def epsilon_greedy(Q, state, epsilon): if random.random() < epsilon: return random.choice(action_space) # 探索 else: return argmax([Q[state, a] for a in action_space]) # 利用 ε 衰减策略： ┌─────────────────────────────────────────────────┐ │ 初期：高 ε（如 1.0），大量探索 │ │ 中期：逐渐降低 ε，减少探索 │ │ 后期：低 ε（如 0.01），主要利用 │ │ │ │ 常见衰减公式： │ │ ε_t = max(ε_min, ε_0 * decay^t) │ │ ε_t = ε_min + (ε_max - ε_min) * e^(-t/τ) │ └─────────────────────────────────────────────────┘ 优缺点： + 简单易实现 + 理论上有保证（渐近最优） - 探索效率低（均匀随机，不考虑动作价值差异） - 对于大规模动作空间效果差

UCB（Upper Confidence Bound）

UCB 基于置信上界选择动作，平衡利用已知信息和探索不确定性。

UCB1 算法： ┌──────────────────────────────────────────────────────┐ │ a_t = argmax_a [ Q(s,a) + c * sqrt(ln(t) / N(s,a)) ]│ │ │ │ 第一项 Q(s,a)：利用项，估计的动作价值 │ │ 第二项：探索项，对尝试次数少的动作给予奖励 │ │ │ │ - c：探索系数，控制探索程度 │ │ - t：总步数 │ │ - N(s,a)：动作 a 在状态 s 被选择的次数 │ │ │ │ 直觉： │ │ - Q 高 → 倾向选择（利用） │ │ - N(s,a) 小 → 探索项大 → 倾向尝试（探索） │ │ - 随着时间 t 增加，探索项增长变慢 │ └──────────────────────────────────────────────────────┘ UCB 在 RL 中的应用（UCB-SAL）： Q_UCB(s,a) = Q(s,a) + c * sqrt(ln(N(s)) / N(s,a)) 其中 N(s) 是状态 s 的访问次数 优缺点： + 比 ε-greedy 更智能地分配探索 + 有理论上的遗憾界（O(ln T)） - 难以扩展到大状态空间（需要计数） - 需要存储每个 (s,a) 的访问次数

Thompson Sampling（汤普森采样）

基于贝叶斯推理的探索策略，通过维护动作价值的后验分布来指导探索。

Thompson Sampling 原理： ┌────────────────────────────────────────────────────┐ │ 为每个动作维护一个价值的后验分布 │ │ │ │ 每个时间步： │ │ 1. 从每个动作的后验分布中采样一个值 θ_a │ │ 2. 选择采样值最大的动作：a = argmax_a θ_a │ │ 3. 执行动作，观察奖励 │ │ 4. 更新后验分布 │ │ │ │ 直觉： │ │ - 后验分布均值高 → 利用 │ │ - 后验分布方差大（不确定性高） → 自然探索 │ │ - 不需要额外的探索机制，通过采样自然平衡 │ └────────────────────────────────────────────────────┘ 伯努利臂（Bernoulli Bandit）示例： - 先验：Beta(1, 1)（均匀分布） - 获得奖励 1：Beta(α+1, β) - 获得奖励 0：Beta(α, β+1) - 采样：θ_a ~ Beta(α_a, β_a) 高斯臂示例： - 先验：正态-逆伽马共轭分布 - 更新：根据观察到的奖励更新均值和方差 优缺点： + 实践效果非常好（常优于 UCB 和 ε-greedy） + 天然处理非平稳问题 + 可与深度学习结合（Dropout 近似 Thompson Sampling） - 需要指定先验分布 - 精确的后验计算可能复杂
Dropout 可作为 Thompson Sampling 的近似：训练时使用 Dropout，预测时随机 mask 来采样不同模型，模拟从后验分布采样。

高级探索策略

1. 基于计数的探索（Count-Based Exploration） ┌─────────────────────────────────────────────┐ │ 内在奖励 = β / sqrt(N(s)) 或 β / (N(s) + 1) │ │ │ │ 对访问次数少的状态给予额外奖励 │ │ 稀疏计数：用哈希或密度模型近似 │ │ │ │ ICM（Intrinsic Curiosity Module）： │ │ - 预测误差作为内在奖励 │ │ - 不可预测的状态 = 探索价值高 │ │ │ │ RND（Random Network Distillation）： │ │ - 固定随机网络 vs 学习网络的预测误差 │ │ - 未见过的状态预测误差大 │ └─────────────────────────────────────────────┘ 2. 好奇心驱动探索（Curiosity-Driven） ┌─────────────────────────────────────────────┐ │ 核心思想：对"意外"或"不可预测"的状态感兴趣 │ │ │ │ ICM 模块： │ │ - 特征编码：φ(s) │ │ - 正向模型：预测 φ(s_{t+1}) given (s_t, a_t) │ │ - 内在奖励 = ||φ(s_{t+1}) - φ̂(s_{t+1})||² │ │ │ │ 解决了： │ │ - 稀疏奖励问题 │ │ - 探索空间过大时的效率问题 │ │ - 避免不可控状态的探索（如电视噪音） │ └─────────────────────────────────────────────┘ 3. Go-Explore - 先通过确定性策略探索并记录状态 - 从有前途的状态出发重新探索 - 在 Atari 游戏中取得突破性成果 策略对比： ┌────────────┬──────────┬──────────┬──────────────────┐ │ 策略 │ 简单环境 │ 复杂环境 │ 大状态空间 │ ├────────────┼──────────┼──────────┼──────────────────┤ │ ε-greedy │ ✅ │ ⚠️ │ ❌ │ │ UCB │ ✅ │ ⚠️ │ ❌ │ │ Thompson │ ✅ │ ✅ │ ⚠️ │ │ Count-based│ ✅ │ ✅ │ ⚠️（需近似计数） │ │ Curiosity │ ⚠️ │ ✅ │ ✅ │ └────────────┴──────────┴──────────┴──────────────────┘
