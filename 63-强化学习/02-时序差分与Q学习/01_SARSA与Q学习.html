SARSA与Q学习

强化学习TDQ-Learning

时序差分(TD)学习通过采样更新价值函数，无需知道环境模型。

时序差分学习 (TD Learning)

TD学习 = 蒙特卡洛 + 动态规划 蒙特MC：等回合结束，用实际回报更新 V(S) ← V(S) + α × [G - V(S)] G = R1 + γR2 + γ²R3 + ...（完整回报） TD(0)：每一步立即更新，用TD目标 V(S) ← V(S) + α × [R + γ×V(S') - V(S)] TD目标 = R + γ×V(S') TD误差 = R + γ×V(S') - V(S) 对比： ┌──────────┬──────────────┬──────────────────┐ │ │ 蒙特卡洛 │ TD(0) │ ├──────────┼──────────────┼──────────────────┤ │ 更新时机 │ 回合结束 │ 每步 │ │ 目标 │ 实际回报G │ R + γV(S') │ │ 偏差 │ 无偏 │ 有偏 │ │ 方差 │ 高 │ 低 │ │ 在线学习 │ 不支持 │ 支持 │ │ 连续任务 │ 不适用 │ 适用 │ └──────────┴──────────────┴──────────────────┘ TD(λ)：统一MC和TD的框架 - λ=0：等价于TD(0) - λ=1：等价于MC - 0<λ<1：介于两者之间 - 使用资格迹(Eligibility Trace)实现

SARSA算法

SARSA (State-Action-Reward-State-Action)： On-Policy的TD控制算法 更新公式： Q(S, A) ← Q(S, A) + α × [R + γ × Q(S', A') - Q(S, A)] 命名由来：每次更新需要五元组 (S, A, R, S', A') State-Action-Reward-State-Action 算法流程： 1. 初始化 Q(s, a) = 0 2. 对每个回合： a. 初始化状态 S，选择动作 A（ε-greedy） b. 循环直到回合结束： - 执行A，观察R, S' - 选择A'（ε-greedy，基于当前Q） - Q(S,A) ← Q(S,A) + α[R + γQ(S',A') - Q(S,A)] - S ← S', A ← A' On-Policy含义： - 策略评估和策略改进使用同一个策略 - 行为策略(behavior policy) = 目标策略(target policy) - ε-greedy：大部分时间选最优，偶尔随机探索

Q-Learning算法

Q-Learning（Watkins 1989）： Off-Policy的TD控制算法 更新公式： Q(S, A) ← Q(S, A) + α × [R + γ × max_a Q(S', a) - Q(S, A)] ↑ 关键区别：用max而不是Q(S',A') 算法流程： 1. 初始化 Q(s, a) = 0 2. 对每个回合： a. 初始化状态 S b. 循环直到回合结束： - 选择A（ε-greedy，基于当前Q） - 执行A，观察R, S' - Q(S,A) ← Q(S,A) + α[R + γ × max_a Q(S',a) - Q(S,A)] - S ← S' On-Policy vs Off-Policy： ┌──────────┬────────────────┬──────────────────┐ │ │ SARSA (On) │ Q-Learning (Off) │ ├──────────┼────────────────┼──────────────────┤ │ 更新目标 │ Q(S', A') │ max Q(S', a) │ │ A'来源 │ 实际执行的动作 │ 假设最优动作 │ │ 策略 │ 评估自己用的 │ 评估最优策略 │ │ 探索影响 │ 更新受探索影响 │ 更新不受探索影响 │ │ 表现 │ 更保守 │ 更激进 │ └──────────┴────────────────┴──────────────────┘ 经典例子：悬崖行走 - SARSA：学到远离悬崖的安全路径 - Q-Learning：学到最短路径（紧贴悬崖），但在探索时容易掉下去

探索与利用 (Exploration vs Exploitation)

核心困境：利用已知最优 vs 尝试未知可能更优 探索策略： ┌─────────────────────────────────────────────┐ │ ε-greedy（最常用） │ │ 以概率ε随机选择动作（探索） │ │ 以概率1-ε选择当前最优动作（利用） │ │ 通常ε从大到小衰减（先探索后利用） │ │ │ │ Boltzmann/Softmax探索 │ │ P(a|s) = exp(Q(s,a)/τ) / Σ exp(Q(s',a')/τ)│ │ τ：温度参数，τ→0趋于贪婪，τ→∞趋于随机 │ │ │ │ UCB (Upper Confidence Bound) │ │ 选择 Q(s,a) + c × sqrt(ln(t)/N(s,a)) │ │ 平衡利用(高Q)和探索(低访问次数) │ │ │ │ 基于计数的好奇心 │ │ N(s,a)：状态-动作对的访问计数 │ │ 奖励加上探索奖励：R + β/sqrt(N(s,a)) │ └─────────────────────────────────────────────┘
ε-greedy是最简单有效的探索策略。UCB在理论上有更好的遗憾界。深度RL中常用噪声网络或熵正则化。
