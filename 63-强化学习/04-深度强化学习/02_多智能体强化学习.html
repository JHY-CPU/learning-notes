多智能体强化学习（MARL）

强化学习多智能体MARL

多智能体强化学习研究多个智能体在同一环境中交互学习的理论与方法，是强化学习的重要研究方向。

MARL 问题定义

MARL 将多个智能体的交互建模为马尔可夫博弈（Markov Game）。

马尔可夫博弈（随机博弈）： M = (N, S, A_1,...,A_n, T, R_1,...,R_n, γ) 其中： - N = {1, ..., n}：智能体集合 - S：状态空间 - A_i：智能体 i 的动作空间 - T: S × A_1 × ... × A_n → Δ(S)：状态转移函数 - R_i: S × A_1 × ... × A_n → R：智能体 i 的奖励函数 - γ：折扣因子 单智能体 vs 多智能体： ┌───────────────┬────────────────────────────────────┐ │ 单智能体 RL │ 环境是固定的（MDP） │ │ │ 状态转移是确定或随机的 │ ├───────────────┼────────────────────────────────────┤ │ 多智能体 RL │ 环境随其他智能体行为变化 │ │ │ 每个智能体面对的环境是非平稳的 │ │ │ 需要考虑协作、竞争或混合关系 │ └───────────────┴────────────────────────────────────┘ 核心挑战： - 非平稳性：其他智能体在学习，环境不断变化 - 信用分配：联合奖励如何分配给各个智能体 - 可扩展性：智能体数量增加时的计算和通信复杂度 - 部分可观测性：每个智能体通常只能观测局部信息

协作、竞争与混合设定

智能体关系分类（根据奖励结构）： ┌────────────────────────────────────────────────────────┐ │ 完全协作（Cooperative） │ │ 所有智能体共享同一个奖励：R_1 = R_2 = ... = R_n │ │ 例：机器人团队协作搬运、多人游戏组队 │ │ 算法：QMIX、MAPPO、VDN │ │ │ │ 完全竞争（Competitive / Adversarial） │ │ 零和博弈：R_1 + R_2 = 0 │ │ 一个智能体的收益是另一个的损失 │ │ 例：围棋、扑克、对抗游戏 │ │ 算法：Minimax-Q、Nash-Q │ │ │ │ 混合设定（Mixed / General-Sum） │ │ 各智能体有独立的奖励函数，可能有部分共同利益 │ │ 例：自动驾驶多车交互、市场竞争 │ │ 算法：IAL、LOLA、Policy-Space Response Oracles │ └────────────────────────────────────────────────────────┘

CTDE（集中训练，分布式执行）

CTDE 是 MARL 中最主流的训练范式，训练时利用全局信息，执行时每个智能体独立决策。

CTDE 框架： ┌─────────────────────────────────────────────────────┐ │ 训练阶段（集中式） │ │ ┌──────────────────────────────────────┐ │ │ │ 全局状态 S │ │ │ │ 所有智能体的观测 o_1,...,o_n │ │ │ │ 所有智能体的动作 a_1,...,a_n │ │ │ │ 共享的经验池 / 联合价值函数 │ │ │ └──────────────────────────────────────┘ │ │ │ │ 执行阶段（分布式） │ │ ┌──────┐ ┌──────┐ ┌──────┐ │ │ │ π_1 │ │ π_2 │ │ π_3 │ 每个智能体独立决策 │ │ │(o_1) │ │(o_2) │ │(o_3) │ 只依赖自身观测 │ │ └──────┘ └──────┘ └──────┘ │ └─────────────────────────────────────────────────────┘ 代表性算法： 1. VDN（Value Decomposition Network） Q_tot = Σ Q_i(o_i, a_i) // 简单加和分解 - 最简单的值分解方法 - 无法学习复杂的协作模式 2. QMIX（Q-value MIXing） Q_tot = f(Q_1, Q_2, ..., Q_n; s) - 单调性约束：∂Q_tot/∂Q_i ≥ 0 - 使用超网络生成混合权重 - 比 VDN 更强大的表达能力 3. MAPPO（Multi-Agent PPO） - 每个智能体独立的 Actor-Critic - 共享的 Critic 使用全局信息 - 分布式执行时只用 Actor 策略

通信学习（Communication Learning）

让智能体学习何时通信、与谁通信、通信什么内容。

通信机制的设计空间： ┌─────────────────────────────────────────────────┐ │ 1. 是否通信（When） │ │ - 连续通信：每步都广播 │ │ - 选择性通信：学到通信门控机制 │ │ - 事件触发通信：特定条件触发 │ │ │ │ 2. 与谁通信（Who） │ │ - 广播：所有智能体收到 │ │ - 定向：学到通信目标选择 │ │ - 层级：通过中继节点转发 │ │ │ │ 3. 通信什么（What） │ │ - 原始观测：直接共享传感器数据 │ │ - 隐状态：共享 RNN 隐状态向量 │ │ - 学习消息：端到端学习通信内容 │ │ │ │ 4. 如何利用通信（How） │ │ - 注意力机制：对收到的消息加权聚合 │ │ - 图网络：将智能体视为图节点 │ │ - 消息传递：多轮消息交换 │ └─────────────────────────────────────────────────┘ 代表工作： - CommNet：连续通信 + 注意力聚合 - DGN（Dynamic Graph Network）：图注意力网络 - TarMAC：定向通信（Targeted Multi-Agent Communication） - I2C（Individualized Controlled Communication）：可控通信
通信学习面临的核心问题是：通信本身有成本，智能体需要学会在"信息不足需要通信"和"通信开销"之间平衡。
