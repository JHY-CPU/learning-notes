⚖️ Koa vs Express 对比总结

核心差异 (中间件模型/上下文/错误处理/生态)、性能对比、选型指南、Koa 和 Express 互相迁移、Phase 8 Node.js 后端全栈图谱。

核心差异对比

// ========== Koa vs Express ========== // ┌──────────────────┬─────────────────────┬──────────────────────┐ // │ │ Express │ Koa │ // ├──────────────────┼─────────────────────┼──────────────────────┤ // │ 中间件模型 │ 线性 (顺序执行) │ 洋葱模型 (双向) │ // │ │ next() 后不执行 │ await next() 前后执行 │ // ├──────────────────┼─────────────────────┼──────────────────────┤ // │ 异步处理 │ 回调 → 需 asyncWrap │ 原生 async/await │ // ├──────────────────┼─────────────────────┼──────────────────────┤ // │ 请求/响应 │ req/res 对象 │ ctx 统一封装 │ // ├──────────────────┼─────────────────────┼──────────────────────┤ // │ 内置功能 │ 路由/静态文件/模板 │ 无绑定 (最小核心) │ // ├──────────────────┼─────────────────────┼──────────────────────┤ // │ 错误处理 │ 4参数中间件 │ try/catch 更自然 │ // ├──────────────────┼─────────────────────┼──────────────────────┤ // │ 生态系统 │ 最大 │ 较小但够用 │ // ├──────────────────┼─────────────────────┼──────────────────────┤ // │ 性能 │ 中等 │ 稍快 (更轻量) │ // ├──────────────────┼─────────────────────┼──────────────────────┤ // │ 学习曲线 │ 低 (入门容易) │ 中 (理解洋葱模型) │ // ├──────────────────┼─────────────────────┼──────────────────────┤ // │ 适合场景 │ 通用/快速开发 │ 需要精细控制中间件 │ // └──────────────────┴─────────────────────┴──────────────────────┘ // ========== 代码对比 ========== // ——— Express ——— const express = require('express'); const app = express(); // 中间件 app.use((req, res, next) => { console.log('Time:', Date.now()); next(); // 调用后不再回来 }); // 路由 app.get('/users', async (req, res) => { try { const users = await User.find(); res.json(users); } catch (err) { next(err); // 手动传给错误处理 } }); // 错误处理 (必须 4 个参数) app.use((err, req, res, next) => { res.status(500).json({ error: err.message }); }); // ——— Koa ——— const Koa = require('koa'); const Router = require('@koa/router'); const app = new Koa(); const router = new Router(); // 中间件 (洋葱) app.use(async (ctx, next) => { console.log('Time:', Date.now()); await next(); // 会回来! console.log('After response'); }); // 路由 router.get('/users', async (ctx) => { ctx.body = await User.find(); // 自动 JSON }); // 错误处理 (try/catch) app.use(async (ctx, next) => { try { await next(); } catch (err) { ctx.status = err.status || 500; ctx.body = { error: err.message }; } });

互相迁移指南

// ========== Express → Koa 迁移 ========== // ========== req/res → ctx ========== // Express: req.params.id, req.query, req.body // Koa: ctx.params.id, ctx.query, ctx.request.body // Express: res.json(), res.status(200).send() // Koa: ctx.body = {}, ctx.status = 200 // ========== 中间件适配 ========== // koa-connect 可以兼容 Express 中间件 const { use } = require('koa-connect'); const cors = require('cors'); // Express cors app.use(use(cors({ origin: '*', credentials: true }))); // ========== 路由迁移 ========== // Express: router.get('/users/:id', controller); // Koa: router.get('/users/:id', async (ctx) => { await controller(ctx); // 或重写 controller }); // ========== 错误处理迁移 ========== // Express: (err, req, res, next) → 4 参数 // Koa: try/catch → ctx.throw → app.on('error') // ========== 第三方中间件对照 ========== // Express → Koa 对应 // cors → @koa/cors // express.json() → koa-body // multer → @koa/multer 或 koa-body // express-session → koa-session // morgan → koa-logger // helmet → koa-helmet // compression → koa-compress // express.static → koa-static // passport → koa-passport // ========== 什么时候选 Express ========== // ✅ 你已经熟悉 Express // ✅ 项目需要快速启动 // ✅ 依赖大量 Express 中间件 // ✅ 团队都是 Express 经验 // ✅ 需要成熟的教程和文档 // ========== 什么时候选 Koa ========== // ✅ 你想要洋葱模型控制中间件 // ✅ 项目需要更干净的异步代码 // ✅ 你喜欢最小核心, 自由组合 // ✅ 需要精细的中间件顺序控制 // ✅ 你想要更好的错误处理体验 // ========== 性能对比 ========== // 基准测试 (Hello World): // Express: ~8000 req/s // Koa: ~10000 req/s // // Koa 轻量约 20-30% // 但实际应用差异不大 (数据库是瓶颈) // ========== 实际选择建议 ========== // 新项目: 选你熟悉的技术 // 微服务: Koa (轻量) // REST API: Express 或 Koa 都可 // 大型应用: Express (生态丰富) // 学习目的: 两个都试试
