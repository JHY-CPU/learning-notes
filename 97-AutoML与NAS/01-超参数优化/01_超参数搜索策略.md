# 超参数搜索策略

## 一、超参数的重要性

机器学习模型的超参数（Hyperparameters）是在训练开始之前设定的参数，与训练过程中学习的模型参数（权重、偏置）不同。超参数的选择直接影响模型的性能。

### 超参数 vs 模型参数

| 对比项 | 超参数 | 模型参数 |
| --- | --- | --- |
| 设定时间 | 训练前手动设定 | 训练过程中学习 |
| 典型例子 | 学习率、树深度、正则化系数 | 权重、偏置 |
| 确定方式 | 经验/搜索 | 梯度下降/最优解 |
| 影响 | 模型能力和训练动态 | 最终预测结果 |

### 常见超参数示例

- **神经网络：** 学习率、批次大小、层数、隐藏单元数、Dropout率、优化器类型
- **树模型：** 树的数量、最大深度、最小叶子样本数、学习率（GBDT）、正则化系数
- **SVM：** C（正则化）、gamma（RBF核参数）、kernel类型

## 二、网格搜索（Grid Search）

网格搜索是最直观的方法：穷举所有超参数组合。

$$
总试验次数 = ∏i ni
$$

### 优缺点

- **优点：** 简单、完整、可并行
- **缺点：** 维度灾难——超参数数量增加时，组合数指数增长

```python
from sklearn.model_selection import GridSearchCV
from sklearn.ensemble import GradientBoostingClassifier

param_grid = {
    'learning_rate': [0.01, 0.05, 0.1, 0.5, 1.0],
    'max_depth': [3, 5, 7, 10],
    'n_estimators': [50, 100, 200, 500],
}
# 总共 5 × 4 × 4 = 80 次评估

grid_search = GridSearchCV(
    GradientBoostingClassifier(),
    param_grid,
    cv=5,
    scoring='accuracy',
    n_jobs=-1,
)
# grid_search.fit(X_train, y_train)
# print(f"最优参数: {grid_search.best_params_}")
# print(f"最优分数: {grid_search.best_score_:.4f}")
```

> **Bergstra & Bengio (2012) 的关键发现：**
> 在很多情况下，只有一两个超参数对性能影响最大。网格搜索在不重要的维度上浪费了大量计算资源，而对重要参数的搜索精度不够。

## 三、随机搜索（Random Search）

Bergstra & Bengio (2012) 提出的随机搜索方法，从超参数分布中随机采样。

```python
from sklearn.model_selection import RandomizedSearchCV
from scipy.stats import uniform, randint

param_distributions = {
    'learning_rate': uniform(0.001, 0.1),     # 连续均匀分布
    'max_depth': randint(3, 15),               # 离散均匀分布
    'n_estimators': randint(50, 500),
    'min_samples_split': uniform(0.01, 0.2)
}

random_search = RandomizedSearchCV(
    GradientBoostingClassifier(),
    param_distributions,
    n_iter=60,        # 60次随机采样
    cv=5,
    scoring='accuracy',
    random_state=42,
    n_jobs=-1,
)
# random_search.fit(X_train, y_train)
```

> **经验法则：** 随机搜索的迭代次数通常设为 60-100 次即可获得不错的结果。

## 四、贝叶斯优化（Bayesian Optimization）

贝叶斯优化通过构建目标函数的概率代理模型（Surrogate Model），选择最有希望的超参数组合进行评估。

### 核心组件

1. **代理模型：** 通常使用高斯过程（GP），对目标函数 f(x) 的分布进行建模
2. **采集函数（Acquisition Function）：** 基于代理模型，决定下一个评估点

### 常用采集函数

| 采集函数 | 公式 | 策略 |
| --- | --- | --- |
| **EI（期望提升）** | EI(x) = E[max(f(x) - f(x^+^), 0)] | 平衡探索和利用 |
| **UCB（上置信界）** | UCB(x) = μ(x) + κσ(x) | κ控制探索程度 |
| **PI（概率提升）** | PI(x) = P(f(x) > f(x^+^) + ξ) | 偏向利用 |

## 五、TPE 与 Hyperband/BOHB

### TPE（Tree-structured Parzen Estimator）

TPE由Bergstra et al. (2011)提出，是Hyperopt库的核心算法。不直接建模 P(y|x)，而是建模 P(x|y) 和 P(x)。

$$
P(x|y) = l(x) if y < y* （好的结果对应的分布）
P(x|y) = g(x) if y ≥ y* （差的结果对应的分布）
选择 l(x)/g(x) 最大的 x 作为下一个评估点
$$

### Hyperband

核心思想：利用**早停（Early Stopping）**来加速超参数搜索。给少量资源（如少量epoch），淘汰表现差的配置。

### BOHB

BOHB = Bayesian Optimization + Hyperband。结合了贝叶斯优化的采样效率和Hyperband的早停策略：

- 用TPE选择超参数配置
- 用Hyperband的多保真度策略分配资源
- 既高效又可并行

## 六、Python 实战：Optuna 超参数优化

### 示例：用 Optuna 优化 LightGBM 超参数

```python
import optuna
from sklearn.datasets import load_breast_cancer
from sklearn.model_selection import cross_val_score
from sklearn.ensemble import GradientBoostingClassifier
import numpy as np

# 加载数据
X, y = load_breast_cancer(return_X_y=True)

def objective(trial):
    """Optuna目标函数：返回需要最大化的值"""
    params = {
        'n_estimators': trial.suggest_int('n_estimators', 50, 500),
        'max_depth': trial.suggest_int('max_depth', 2, 12),
        'learning_rate': trial.suggest_float('learning_rate', 1e-3, 0.3, log=True),
        'min_samples_split': trial.suggest_float('min_samples_split', 0.01, 0.5),
        'min_samples_leaf': trial.suggest_float('min_samples_leaf', 0.01, 0.3),
        'subsample': trial.suggest_float('subsample', 0.5, 1.0),
    }

    model = GradientBoostingClassifier(**params, random_state=42)
    scores = cross_val_score(model, X, y, cv=5, scoring='accuracy')
    return scores.mean()

# 创建Study并优化
study = optuna.create_study(
    direction='maximize',
    sampler=optuna.samplers.TPESampler(seed=42),
    pruner=optuna.pruners.MedianPruner()
)

study.optimize(objective, n_trials=100, show_progress_bar=True)

# 结果
print(f"最优准确率: {study.best_value:.4f}")
print(f"最优超参数:")
for key, value in study.best_params.items():
    print(f"  {key}: {value}")

# 超参数重要性
importances = optuna.importance.get_param_importances(study)
print(f"超参数重要性:")
for param, importance in importances.items():
    print(f"  {param}: {importance:.4f}")
```

## 七、搜索策略对比总结

| 方法 | 搜索效率 | 并行化 | 实现难度 | 适用场景 |
| --- | --- | --- | --- | --- |
| 网格搜索 | 低 | 完全并行 | 简单 | 2-3个参数，候选值少 |
| 随机搜索 | 中 | 完全并行 | 简单 | 大多数场景的默认选择 |
| 贝叶斯优化(GP) | 高 | 困难 | 中等 | 评估代价高，参数维度不高 |
| TPE | 高 | 部分 | 中等 | 混合参数类型 |
| BOHB | 很高 | 完全并行 | 较高 | 深度学习，计算预算大 |

## 总结

- 超参数直接影响模型性能，但无法通过梯度下降学习
- 网格搜索简单但受维度灾难限制
- 随机搜索在大多数情况下优于网格搜索，是好的默认选择
- 贝叶斯优化通过代理模型和采集函数实现高效搜索
- TPE用两个密度比来指导搜索，是Hyperopt的核心
- Hyperband/BOHB利用早停加速搜索，适合深度学习场景
- Optuna是当前最流行的超参数优化框架，支持多种搜索策略
