# 微机系统概述 - 07 微机原理与接口技术

三个层次的区分

| 术语 | 含义 | 包含范围 |
| --- | --- | --- |
| **微处理器**（MPU / μP） | 将 CPU 集成在单个芯片上的大规模集成电路 | 运算器 + 控制器 + 寄存器组（有时含 Cache） |
| **微型计算机**（MC / μC） | 以微处理器为核心，配以存储器和 I/O 接口的硬件系统 | MPU + 存储器 + I/O 接口 + 系统总线 |
| **微型计算机系统** | 微型计算机配以软件系统和外设构成的完整系统 | 硬件系统 + 系统软件 + 应用软件 + 外部设备 |

冯诺依曼结构五大要素

1. **采用二进制**
   ：指令和数据均用二进制代码表示
2. **存储程序**
   ：程序和数据预先存入存储器，按地址访问并顺序执行
3. **五大部件**
   ：运算器、控制器、存储器、输入设备、输出设备
4. **指令格式**
   ：指令由操作码（OP）和地址码（A）组成
5. **以运算器为中心**
   （早期设计）

三类系统总线

| 总线类型 | 英文缩写 | 功能 | 方向 |
| --- | --- | --- | --- |
| **数据总线** | DB（Data Bus） | 传送数据和指令代码 | 双向 |
| **地址总线** | AB（Address Bus） | 传送存储单元或 I/O 端口的地址 | 单向（CPU → 存储器/I/O） |
| **控制总线** | CB（Control Bus） | 传送控制信号和状态信号 | 混合（部分单向、部分双向） |

主存的两个基本操作

| 操作 | 含义 | 涉及部件 |
| --- | --- | --- |
| **读操作（Read）** | 将指定地址单元的数据传送到 CPU | MAR → 地址译码 → 存储体 → MDR → CPU |
| **写操作（Write）** | 将 CPU 的数据写入指定地址单元 | CPU → MDR → 存储体；MAR → 地址译码 |


**MAR**（Memory Address Register）：存储地址寄存器，存放要访问的存储单元地址

**MDR**（Memory Data Register）：存储数据寄存器，存放从存储器读出或要写入的数据

例题 1

**下列说法正确的是（ ）**


1. 微处理器就是微型计算机
2. 微型计算机就是微型计算机系统
3. 微处理器包含了运算器、控制器和存储器
4. 微型计算机以微处理器为核心，配以存储器和 I/O 接口


**答案：D**


解析：A 错——微处理器只是 CPU 芯片，不等于微型计算机；

                B 错——微型计算机是硬件系统，微型计算机系统还包含软件和外设；

                C 错——微处理器包含运算器、控制器和寄存器组，**不包含存储器**；

                D 正确——这是微型计算机的准确定义。

例题 2

**某微处理器有 20 根地址线，其最大可寻址的存储空间为（ ）**


1. 64KB
2. 256KB
3. 1MB
4. 4MB


**答案：C**


解析：20 根地址线的寻址范围 = 2^20^ = 1,048,576 字节 = **1MB**

                记忆：8086 地址线 20 位 → 1MB；8088 地址线 20 位 → 1MB（数据线 8 位）

                扩展：24 位 → 16MB（80286）；32 位 → 4GB（80386+）

例题 3

**冯诺依曼结构计算机的核心思想是（ ）**


1. 二进制运算
2. 存储程序
3. 采用总线结构
4. 以运算器为中心


**答案：B**


解析：冯诺依曼结构的**核心思想**是「存储程序」——将程序和数据以二进制形式存储在存储器中，计算机自动取出指令执行。二进制、总线结构、运算器为中心都是冯诺依曼结构的特征之一，但不是最核心的思想。
