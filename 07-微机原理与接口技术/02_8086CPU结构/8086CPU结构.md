# 8086CPU结构 - 07 微机原理与接口技术

BIU 的组成与功能

| 组成 | 功能 |
| --- | --- |
| **4 个段寄存器** | CS（代码段）、DS（数据段）、SS（堆栈段）、ES（附加段）——用于形成 20 位物理地址的高 16 位 |
| **指令指针 IP** | 16 位，存放下一条要取出的指令在代码段内的偏移地址 |
| **20 位地址加法器 Σ** | 完成逻辑地址 → 物理地址的转换：物理地址 = 段基址×16 + 偏移地址 |
| **6 字节指令队列** | 暂存从存储器预取的指令，使 EU 执行和 BIU 取指可以并行（流水线） |
| **总线控制逻辑** | 产生读写控制信号，管理与外部总线的数据传输 |

EU 的组成与功能

| 组成 | 功能 |
| --- | --- |
| **通用寄存器组** | AX、BX、CX、DX（数据寄存器）；SI、DI（变址寄存器）；BP、SP（指针寄存器） |
| **算术逻辑单元 ALU** | 16 位，完成加减乘除、逻辑运算、移位等操作 |
| **标志寄存器 FLAGS** | 16 位，记录运算结果的状态标志（CF、ZF、SF、OF 等） |
| **指令译码器** | 对从指令队列取来的指令进行译码，产生微操作控制信号 |
| **暂存器** | 暂存参与运算的操作数 |

分段机制的核心思想

1. 将 1MB 存储空间划分为若干个逻辑段，每个段最大 64KB
2. 每个段的起始地址必须是 16 的倍数（即低 4 位为 0）
3. 段的起始地址的高 16 位称为
   **段基址**
   ，存放在段寄存器中
4. 段内某个单元距段起始地址的字节数称为
   **偏移地址**
   （有效地址）
5. 用「段基址：偏移地址」的形式表示一个存储单元的
   **逻辑地址**

例题 1

**8086 CPU 中，负责形成 20 位物理地址的部件是（ ）**


1. EU 中的 ALU
2. BIU 中的地址加法器
3. EU 中的指令译码器
4. BIU 中的指令队列


**答案：B**


解析：BIU 中的地址加法器 Σ 完成「段基址 × 16 + 偏移地址」的计算，输出 20 位物理地址。

例题 2

**执行 `MOV AX, 7FFFH` 后再执行 `ADD AX, 1`，各标志位的状态是（ ）**


```
; 分析过程：
; AX = 7FFFH = 0111 1111 1111 1111B
; ADD AX, 1 → AX = 8000H = 1000 0000 0000 0000B
;
; SF = 1（最高位为1）
; ZF = 0（结果非零）
; CF = 0（无符号数 32767 + 1 = 32768，未超出无符号范围 0~65535）
; OF = 1（有符号数 32767 + 1 = 32768 > 32767，超出有符号范围 -32768~32767）
; PF = 0（低8位 00H 中1的个数为0，是偶数个）
```


**答案：SF=1, ZF=0, CF=0, OF=1, PF=0**

例题 3

**若 CS = 2000H，IP = 1000H，则下一条指令的物理地址为（ ）**


物理地址 = CS × 16 + IP

                = 2000H × 10H + 1000H

                = 20000H + 1000H

                = **21000H**

例题 4

**8086 中，执行 LOOP 指令时，隐含使用的寄存器是（ ）**


1. AX
2. BX
3. CX
4. DX


**答案：C**


解析：LOOP 指令的功能是 `CX ← CX - 1`，若 CX ≠ 0 则跳转到目标地址。CX 是隐含的循环计数器。
