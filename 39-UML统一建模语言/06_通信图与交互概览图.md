# 通信图与交互概览图

## 一、通信图（Communication Diagram）

### 1.1 概述

通信图（原名**协作图 Collaboration Diagram**）是 UML 中另一种**交互图**。它强调对象之间的**链接关系**和**消息传递**，而不是时间顺序。

> 时序图 = 时间 + 消息顺序
> 通信图 = 对象链接 + 消息编号

### 1.2 基本元素

#### 对象和链接

```
 [对象名]    [对象名]
┌────────┐  ┌────────┐
│ A:Cls1 │──│ B:Cls2 │   ← 实线表示链接（关联的实例）
└────────┘  └────────┘
```

#### 消息标注

消息用**带箭头的有向线**标注在链接上，用**编号**表示顺序：

```
┌────────┐  1: methodName()  ┌────────┐
│   A    │──────────────────→│   B    │
└────────┘                   └────────┘
```

**编号体系**：
- **顺序编号**：1, 2, 3, 4, ...（平级消息）
- **层次编号**：1, 1.1, 1.2, 2, 2.1, ...（嵌套/调用）
- 层次编号可以体现**嵌套调用**的深度

```
1: processOrder()
  1.1: validateOrder()
    1.1.1: checkStock()
    1.1.2: checkPrice()
  1.2: createOrder()
2: sendConfirmation()
```

### 1.3 消息的完整格式

```
消息编号: 箭头方向 参数列表 : 返回值
```

**示例**：
```
1: → searchBook(title: String): List<Book>
2: ← bookList: List<Book>
3: → addToCart(book: Book): void
4: → calculateTotal(): double
```

### 1.4 多重性

通信图中可以标注链接两端的多重性：

```
┌──────────┐           ┌──────────┐
│ Department│ 1      * │ Employee │
│          │──────────→│          │
└──────────┘           └──────────┘
     1                     *
  （一个部门）         （多个员工）
```

### 1.5 通信图示例

#### 示例1：用户登录

       ↑                                      │

### 1.6 通信图 vs 时序图

| 特性 | 时序图 | 通信图 |
|------|--------|--------|
| **强调** | 时间顺序 | 对象链接关系 |
| **布局** | 纵向（时间轴） | 自由布局 |
| **时间线** | 明确的生命线和激活条 | 无明确时间线 |
| **消息顺序** | 从上到下自然体现 | 通过编号体现 |
| **对象关系** | 不体现 | 通过链接线体现 |
| **适用场景** | 关注交互时序 | 关注对象结构 |
| **复杂度** | 对象多时横向很宽 | 消息多时难读 |
| **可读性** | 时序清晰 | 结构清晰 |
| **互转换** | 可以互相转换 | 可以互相转换 |

**选择建议**：
- 需要强调**时间顺序** → 时序图
- 需要强调**对象关系** → 通信图
- 描述**并发操作** → 时序图（更清晰）
- 描述**消息流** → 通信图（更紧凑）

## 二、交互概览图（Interaction Overview Diagram）

### 2.1 概述

交互概览图是**活动图和时序图的混合体**。它使用活动图的符号来组织交互的**控制流**，在节点中嵌入时序图或其他交互图的片段。

> 活动图 = 控制流 + 动作
> 交互概览图 = 控制流 + 交互片段

### 2.2 基本元素

| 元素 | 符号 | 说明 |
|------|------|------|
| **初始节点** | 实心圆 ● | 流程起点 |
| **终止节点** | 牛眼 ⊙ | 流程终点 |
| **动作节点** | 圆角矩形 | 可包含交互片段 |
| **决策节点** | 菱形 ◇ | 条件分支 |
| **分叉/汇合** | 粗横线 | 并行流 |
| **交互发生** | 矩形引用框 | 引用其他交互图 |

### 2.3 交互概览图示例

#### 电商下单流程

```
        ●  （开始）
        │
        ▼
┌───────────────────────────────────┐
│ sd 用户登录                        │  ← 嵌入的时序图片段
│ User → LoginPage → AuthService    │
│ (验证用户身份)                     │
└───────────────┬───────────────────┘
                │
                ▼
          ┌─────────┐
          │ 库存检查 │  ← 决策节点
          └────┬────┘
    ┌──────────┼──────────┐
    │ 库存充足  │          │ 库存不足
    ▼          │          ▼
┌──────────────────────┐  ┌────────────────┐
│ sd 创建订单           │  │ 提示缺货       │
│ Order→Inventory      │  │ 建议替代商品    │
│ (锁定库存)           │  └───────┬────────┘
└──────────┬───────────┘          │
           │                      │
           ▼                      │
┌──────────────────────┐          │
│ sd 支付处理           │          │
│ Payment→Bank         │          │
│ (调用支付接口)        │          │
└──────────┬───────────┘          │
           │                      │
           ▼                      │
     ┌───────────┐                │
     │ 支付成功？ │                │
     └─────┬─────┘                │
    ┌──────┼──────┐               │
    │ 成功  │      │ 失败          │
    ▼      │      ▼               │
┌──────────────────────┐  ┌────────────────┐
│ par 并行处理          │  │ sd 支付失败    │
│                      │  │ 退款/重试      │
│ → 更新库存           │  └───────┬────────┘
│ → 发送确认邮件       │          │
│ → 通知物流系统       │          │
└──────────┬───────────┘          │
           │                      │
           ▼                      ▼
         ⊙  （结束）
```

### 2.4 交互概览图的特点

**优点**：
1. 可以展示**全局流程**的同时**嵌入细节**
2. 将复杂的交互分解为可管理的片段
3. 结合了活动图的流程性和时序图的交互性
4. 适合描述**端到端的业务流程**

**缺点**：
1. 可能变得很复杂
2. 不如单独的时序图精确
3. 实践中使用频率较低

### 2.5 与其他图的关系

| 图 | 关系 |
|----|------|
| 活动图 | 交互概览图继承了活动图的控制流结构 |
| 时序图 | 交互概览图的节点可以包含时序图片段 |
| 通信图 | 交互概览图的节点可以包含通信图片段 |
| 状态图 | 交互概览图可以引用状态转换 |

## 三、定时图（Timing Diagram）简述

定时图是第四种交互图，侧重于对象**状态随时间的变化**。

### 3.1 基本元素

```
时间轴 →
┌─────────────────────────────────────────────┐
│                                             │
│  状态1       状态2       状态1      状态3    │
│  ┌────┐    ┌────┐     ┌────┐    ┌────┐     │
│  │    │    │    │     │    │    │    │     │
│──┘    └────┘    └─────┘    └────┘    └──── │
│                                             │
│  t0         t1          t2         t3       │
└─────────────────────────────────────────────┘
```

### 3.2 应用场景

- **嵌入式系统**：硬件状态随时间变化
- **通信协议**：信号在时间线上的变化
- **实时系统**：需要严格时序约束的场景

## 四、考试重点

### 通信图
1. **消息编号体系**：层次编号（1, 1.1, 1.2）的含义
2. **通信图与时序图的互转**：能互相转换
3. **强调对象链接**：通信图展示对象之间的结构关系

### 交互概览图
1. **基本概念**：活动图 + 交互片段的组合
2. **适用场景**：端到端的复杂流程

### 定时图
1. **基本概念**：状态随时间的变化
2. **适用场景**：嵌入式、实时系统
