# 时序图（Sequence Diagram）

## 一、概述

时序图（又称顺序图）是 UML 中最重要的**交互图**，用于描述对象之间**按时间顺序**进行的交互。它强调的是**消息的时间顺序**。

时序图回答的核心问题：
- 谁和谁交互？（对象）
- 交互的顺序是什么？（消息序列）
- 交互过程中发生了什么？（消息内容）

> 时序图是**动态行为**建模的核心工具，也是编码时的重要参考

## 二、基本元素

### 2.1 生命线（Lifeline）

**定义**：表示参与交互的对象及其存在时间。

**表示**：矩形（对象头）+ 虚线（生命线）

```
┌─────────────────┐
│ 对象名: 类名     │  ← 对象头
└────────┬────────┘
         │            ← 生命线（虚线，纵向向下）
         │
         │
         ↓ 时间方向
```

**三种对象头格式**：

```
格式1（完整）：           格式2（匿名）：          格式3（仅类名）：
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│ user:Student │     │ :Student     │     │ Student      │
└──────┬───────┘     └──────┬───────┘     └──────┬───────┘
       │                    │                    │
```

**参与者作为生命线**：
```
      Customer
         │
         │  (可以用简笔画小人表示参与者)
         │
         ↓
```

### 2.2 消息（Message）

**定义**：对象之间传递的信息，是时序图的核心。

#### 消息的类型

| 类型 | 符号 | 说明 | 代码类比 |
|------|------|------|---------|
| **同步消息** | 实心箭头 `────►` | 发送方等待接收方处理完毕 | 普通方法调用 |
| **异步消息** | 半箭头 `────>` | 发送方不等待，继续执行 | 消息队列、回调 |
| **返回消息** | 虚线箭头 `- - - ►` | 方法返回的结果 | return 语句 |
| **自调用消息** | 半回环 | 对象调用自身的方法 | this.method() |
| **创建消息** | 虚线 + 箭头 | 创建新对象 | new Object() |
| **销毁消息** | X 标记 | 销毁对象 | delete / GC |

#### 消息的命名格式

```
[序号] 消�名(参数): 返回类型
```

**示例**：
```
1: searchBook(title: String): List<Book>
2: validateInput(): boolean
3: save(): void
```

### 2.3 激活条（Activation）

**定义**：表示对象执行某个操作的时间段。

**表示**：窄矩形条，覆盖在生命线上

```
    ┌────┐
    │    │  ← 激活条（表示对象正在处理）
    │    │
    │    │
    └────┘
```

**激活条的规则**：
- 对象收到消息时，激活条开始
- 对象发送返回消息时，激活条结束
- 对象在执行过程中调用其他对象时，激活条保持
- 自调用时，激活条会加宽（栈帧）

```
    A           B           C
    │           │           │
    │──消息1──→ │           │
    │  ┌──────┐ │           │
    │  │      │ │──消息2──→ │
    │  │      │ │  ┌──────┐ │
    │  │      │ │  │      │ │
    │  │      │ │←─返回── │ │
    │  │      │ │  └──────┘ │
    │←─返回── │ │           │
    │  └──────┘ │           │
    │           │           │
```

### 2.4 组合片段（Combined Fragment）

**定义**：用矩形框表示复杂的控制结构。

**表示**：矩形框 + 左上角的操作符标签

#### 核心操作符

| 操作符 | 含义 | 说明 |
|--------|------|------|
| `alt` | 条件分支 | if-else 结构 |
| `opt` | 可选 | if 结构（无 else） |
| `loop` | 循环 | for/while 结构 |
| `par` | 并行 | 同时执行 |
| `seq` | 弱顺序 | 允许交错但保持同一条生命线上的顺序 |
| `break` | 中断 | 跳出当前执行 |
| `neg` | 否定 | 无效的交互 |
| `ref` | 引用 | 引用另一个交互 |
| `assert` | 断言 | 必须满足的条件 |
| `strict` | 强顺序 | 严格按顺序执行 |
| `consider` | 考虑 | 只关注特定消息 |
| `ignore` | 忽略 | 忽略特定消息 |

#### alt（条件分支）

```
┌─── alt [库存充足] ──────────────────────────────┐
│                                                 │
│  Customer        OrderService    Inventory      │
│     │                │              │           │
│     │──placeOrder()─→│              │           │
│     │                │──checkStock()→│           │
│     │                │←─available────│           │
│     │                │──createOrder()│           │
│     │←─success───────│              │           │
│                                                 │
├─── [库存不足] ──────────────────────────────────┤
│                                                 │
│     │                │              │           │
│     │──placeOrder()─→│              │           │
│     │                │──checkStock()→│           │
│     │                │←─outOfStock──│           │
│     │←─failed────────│              │           │
│                                                 │
└─────────────────────────────────────────────────┘
```

#### opt（可选）

```
┌─── opt [用户有优惠券] ─────────────────────────┐
│                                                │
│     │──applyCoupon(code)──→│                   │
│     │←─discountApplied─────│                   │
│                                                │
└────────────────────────────────────────────────┘
```

#### loop（循环）

```
┌─── loop [i = 1..n, items[i] != null] ─────────┐
│                                                │
│     │──processItem(items[i])──→│               │
│     │←─result──────────────────│               │
│                                                │
└────────────────────────────────────────────────┘
```

#### par（并行）

```
┌─── par ────────────────────────────────────────┐
│                                                │
│     │──sendEmail()──→ EmailService             │
│     │──sendSMS()───→ SMSService                │
│     │──pushNotify()─→ PushService              │
│                                                │
└────────────────────────────────────────────────┘
```

### 2.5 状态不变式（State Invariant）

在生命线上放置一个条件，当条件不满足时交互停止。

```
    │
    │  [x > 0]    ← 状态不变式
    │
```

### 2.6 交互引用（Interaction Use / Ref）

引用另一个时序图（分解复杂交互）。

```
┌──────────────────────────────────┐
│ ┌─── ref ─────────────────────┐ │
│ │  验证用户身份                 │ │  ← 引用另一个时序图
│ │  （参见 UC-001 时序图）      │ │
│ └─────────────────────────────┘ │
└──────────────────────────────────┘
```

## 四、时序图的高级特性

### 4.1 帧（Frame）

UML 2.x 使用**帧**（Frame）来包裹整个时序图或其中的一部分：

```
┌─────────────────────────────────────────┐
│ sd LoginProcess                         │  ← sd = sequence diagram
│                                         │
│  User    LoginPage   AuthService        │
│   │         │            │              │
│   │──login─→│            │              │
│   │         │──auth()───→│              │
│   │         │←─result────│              │
│   │←─response│           │              │
│                                         │
└─────────────────────────────────────────┘
```

**帧类型标签**：
| 标签 | 含义 |
|------|------|
| `sd` | 时序图 |
| `alt` | 条件分支 |
| `opt` | 可选 |
| `loop` | 循环 |
| `par` | 并行 |
| `ref` | 引用 |
| `break` | 中断 |

### 4.2 持续时间约束

标注消息处理需要的时间：

```
    │──process()──→│
    │              │
    │  {duration  │
    │   < 200ms}  │
    │←─result─────│
```

### 4.3 状态标记

在生命线上标注对象的状态变化：

```
    │
    │  [状态: 空闲]
    │
    │──start()──→
    │
    │  [状态: 运行中]
    │
    │←─stop()────
    │
    │  [状态: 已停止]
    │
```

## 五、时序图与其他图的关系

| 图 | 侧重 | 关系 |
|----|------|------|
| 类图 | 静态结构 | 时序图中的对象是类图中类的实例 |
| 用例图 | 功能需求 | 一个用例可以有多个时序图来描述实现 |
| 活动图 | 业务流程 | 时序图侧重对象交互，活动图侧重控制流 |
| 通信图 | 对象链接 | 时序图强调时间顺序，通信图强调对象关系 |
| 状态图 | 对象状态 | 时序图中对象的状态变化可以用状态图细化 |

## 六、考试重点

1. **基本元素**：生命线、消息类型（同步/异步/返回）、激活条
2. **组合片段**：alt、opt、loop 的含义和画法
3. **阅读时序图**：能从时序图推导出交互流程
4. **绘制时序图**：能根据需求描述绘制时序图
5. **帧的概念**：UML 2.x 中帧（Frame）的使用
