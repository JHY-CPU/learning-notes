# 组件图与部署图

## 一、组件图（Component Diagram）

### 1.1 概述

组件图描述系统的**软件组件**及其**依赖关系**，展示系统的高层模块结构。

组件图回答的核心问题：
- 系统由哪些**模块/组件**组成？
- 组件之间有什么**依赖关系**？
- 组件提供和需要哪些**接口**？

> 组件图是**实现层面**的图，介于设计（类图）和部署（部署图）之间

### 1.2 组件（Component）

**定义**：系统中一个可替换的、封装了实现的物理部分。

**表示**：

```
┌──────────────────────────┐
│ «component»              │
│ ComponentName            │
│                          │
├──────────────────────────┤
│ 属性                     │
├──────────────────────────┤
│ 方法                     │
└──────────────────────────┘
```

或使用简化符号（右侧有小矩形标记）：

```
┌──────────┐  ┌──┐
│          │  │  │
│Component │──│  │
│          │  └──┘
│          │
└──────────┘
```

**组件的分类**：

| 类型 | 说明 | 示例 |
|------|------|------|
| **部署组件** | 可执行文件 | `.exe`、`.jar`、`.dll`、`.war` |
| **工作产品组件** | 开发过程的产物 | `.java`、`.cpp`、数据库脚本 |
| **执行组件** | 运行时创建的组件 | 进程、线程、Servlet |

**常见的构造型（Stereotype）**：

| 构造型 | 含义 |
|--------|------|
| `«executable»` | 可执行文件 |
| `«library»` | 库文件 |
| `«table»` | 数据库表 |
| `«file»` | 文件 |
| `«document»` | 文档 |
| `«component»` | 通用组件 |

### 1.3 接口（Interface）

**定义**：组件提供或需要的服务契约。

#### 提供接口（Provided Interface）

组件**实现**的接口，向外提供服务。

**符号**：棒棒糖符号 ○——

```
         ○── IService
         │
┌────────┴─────────┐
│   Component      │
└──────────────────┘
```

#### 需要接口（Required Interface）

组件**依赖**的接口，需要外部提供。

**符号**：半圆符号 Ⓢ——

```
         Ⓢ── IService
         │
┌────────┴─────────┐
│   Component      │
└──────────────────┘
```

#### 接口表示的两种方式

```
方式1（符号）：              方式2（矩形）：
                           ┌─────────────────┐
    ○── Provided           │ «interface»     │
    │                      │ IService        │
┌───┴────┐                 ├─────────────────┤
│Component│                 │ + method1()     │
└────────┘                 │ + method2()     │
    │                      └────────┬────────┘
    Ⓢ── Required                  │ 实现
                           ┌───────┴─────────┐
                           │   Component     │
                           └─────────────────┘
```

### 1.4 组件之间的关系

#### 依赖关系

```
┌──────────┐          ┌──────────┐
│ComponentA│ ------→ │ComponentB│
└──────────┘  依赖     └──────────┘
```

#### 通过接口连接

```
┌──────────┐    Ⓢ────○    ┌──────────┐
│ComponentA│───→I/F──────→│ComponentB│
└──────────┘   需要  提供   └──────────┘
```

> 最佳实践：组件之间通过接口通信，而不是直接依赖

### 1.5 端口（Port）

端口是组件上的**交互点**，用于分组提供的和需要的接口。

```
┌──────────────────────────┐
│      Component           │
│                          │
│  p1:Port1        p2:Port2│
│  ○── I/F1        Ⓢ── I/F3│
│  Ⓢ── I/F2        ○── I/F4│
│                          │
└──────────────────────────┘
```

## 二、部署图（Deployment Diagram）

### 2.1 概述

部署图描述系统的**物理硬件结构**和**软件组件的部署方式**。

部署图回答的核心问题：
- 系统部署在哪些**硬件节点**上？
- 节点之间如何**连接**？
- 每个节点上运行哪些**软件组件**？

> 部署图是唯一关注系统**物理部署**的 UML 图

### 2.2 节点（Node）

**定义**：运行时的计算资源（物理硬件或软件容器）。

**表示**：三维立方体

```
┌──────────────────────────┐
│ ╱                      ╱ │
│╱──────────────────────╱  │
││    NodeName         │   │
││                     │   │
││    «stereotype»     │   │
││                     │   │
└──────────────────────────┘
```

**节点的类型**：

| 类型 | 说明 | 示例 |
|------|------|------|
| **设备（Device）** | 物理硬件 | 服务器、PC、手机、路由器 |
| **执行环境（Execution Environment）** | 软件容器 | JVM、Tomcat、Docker、操作系统 |

### 2.3 节点之间的通信

节点之间通过**通信路径**（Communication Path）连接：

```
┌──────────┐    TCP/IP    ┌──────────┐
│  Server  │─────────────│  Client  │
└──────────┘             └──────────┘

┌──────────┐   HTTP/REST  ┌──────────┐
│ AppServer│─────────────│  Browser  │
└──────────┘             └──────────┘
```

**常见的通信协议**：

| 协议 | 说明 |
|------|------|
| `TCP/IP` | 基础网络协议 |
| `HTTP/HTTPS` | Web 通信 |
| `RMI` | Java 远程方法调用 |
| `SOAP` | Web 服务 |
| `JDBC` | 数据库连接 |
| `MQTT` | 物联网消息协议 |
| `WebSocket` | 全双工通信 |

### 2.4 部署规范（Deployment Specification）

描述组件在节点上的部署细节：

```
┌──────────────────────────┐
│ «device»                 │
│ Web Server               │
│                          │
│ ┌──────────────────────┐ │
│ │ «artifact»           │ │
│ │ app.war              │ │
│ │                      │ │
│ │ «deployDescriptor»   │ │
│ │ context: /shop       │ │
│ │ port: 8080           │ │
│ │ instances: 3         │ │
│ └──────────────────────┘ │
└──────────────────────────┘
```

### 2.5 工件（Artifact）

**定义**：软件开发过程中的物理产物。

**表示**：矩形 + 右上角折角

```
┌──────────────┐
│              ╱
│  app.war    ╱
│            ╱
│           ╱
└──────────┘
```

**常见的工件类型**：

| 构造型 | 示例 |
|--------|------|
| `«artifact»` | 通用工件 |
| `«executable»` | `server.exe` |
| `«library»` | `utils.dll`、`lib.so` |
| `«document»` | `README.md`、`config.json` |
| `«file»` | `data.csv` |
| `«script»` | `deploy.sh` |
| `«source»` | `main.java` |

## 三、组件图 vs 部署图

| 特性 | 组件图 | 部署图 |
|------|--------|--------|
| **关注点** | 软件模块结构 | 物理部署结构 |
| **核心元素** | 组件、接口 | 节点、工件 |
| **抽象层次** | 逻辑 | 物理 |
| **典型问题** | 系统有哪些模块？ | 系统部署在哪里？ |
| **与代码的关系** | 对应模块/jar包 | 对应服务器/容器 |
| **使用场景** | 架构设计、模块划分 | 运维部署、基础设施 |

## 四、考试重点

### 组件图
1. **组件的定义**：可替换的、封装的物理部分
2. **提供接口 vs 需要接口**：○（提供）vs Ⓢ（需要）
3. **组件之间的依赖关系**：通过接口连接

### 部署图
1. **节点**：设备和执行环境
2. **通信路径**：节点之间的连接和协议
3. **工件**：部署在节点上的软件产物
4. **物理部署**：能根据场景画出部署图

### 通用
1. 能根据系统需求设计合理的组件结构
2. 能根据架构设计画出部署方案
3. 组件图和部署图的配合使用
