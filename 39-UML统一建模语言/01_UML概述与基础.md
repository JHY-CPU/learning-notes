# UML 概述与基础

## 一、什么是 UML

UML（Unified Modeling Language，统一建模语言）是一种**标准化的建模语言**，用于对软件系统进行**可视化、描述、构造和文档化**。它不是一种编程语言，而是一种**图形化的语言**，用于帮助开发人员、架构师和利益相关者理解、设计和沟通系统架构。

## 二、UML 的发展历史

| 时间 | 事件 |
|------|------|
| 1994年 | Grady Booch（Rational）提出 Booch 方法 |
| 1994年 | Jim Rumbaugh（GE）提出 OMT（对象建模技术） |
| 1995年 | Ivar Jacobson（Objectory）提出 OOSE（面向对象软件工程） |
| 1996年 | Booch、Rumbaugh、Jacobson 三人合并方法，发布 UML 0.9 |
| 1997年 | UML 1.0 被 OMG（对象管理组织）采纳为标准 |
| 2005年 | UML 2.0 发布，重大更新 |
| 2017年 | UML 2.5.1 发布（当前最新正式版本） |

> Booch、Rumbaugh、Jacobson 三人被称为"三友（Three Amigos）"

## 三、UML 的目标与作用

### 3.1 UML 的核心目标

1. **可视化（Visualize）**：用图形化的方式表达系统设计
2. **详述（Specify）**：精确地描述系统的结构和行为
3. **构造（Construct）**：为代码生成提供基础（正向工程）
4. **文档化（Document）**：记录系统的架构决策和设计细节

### 3.2 UML 在软件开发中的作用

```
需求分析 → UML建模 → 设计文档 → 编码实现 → 测试维护
   ↑          ↑          ↑          ↑
 用例图     类图/时序图   组件图     部署图
```

- **沟通工具**：团队成员之间的通用语言
- **设计蓝图**：编码前的系统设计规划
- **文档化**：记录系统架构，便于后期维护
- **代码生成**：部分 UML 工具可直接生成代码框架

## 四、UML 2.x 的图分类

UML 2.x 定义了 **14 种图**，分为两大类：

### 4.1 结构图（Structural Diagrams）—— 描述系统的静态结构

| 序号 | 图名 | 英文 | 核心用途 |
|------|------|------|----------|
| 1 | **类图** | Class Diagram | 描述类、接口及其关系（最常用） |
| 2 | **对象图** | Object Diagram | 描述某一时刻系统中对象的快照 |
| 3 | **组件图** | Component Diagram | 描述系统的模块和依赖关系 |
| 4 | **部署图** | Deployment Diagram | 描述硬件节点和软件部署 |
| 5 | **包图** | Package Diagram | 描述系统的包结构和依赖 |
| 6 | **复合结构图** | Composite Structure Diagram | 描述类的内部结构 |
| 7 | **轮廓图** | Profile Diagram | UML 的扩展机制 |

### 4.2 行为图（Behavioral Diagrams）—— 描述系统的动态行为

| 序号 | 图名 | 英文 | 核心用途 |
|------|------|------|----------|
| 1 | **用例图** | Use Case Diagram | 描述系统的功能需求 |
| 2 | **活动图** | Activity Diagram | 描述业务流程或算法流程 |
| 3 | **状态机图** | State Machine Diagram | 描述对象的生命周期 |
| 4 | **时序图** | Sequence Diagram | 描述对象之间的交互（按时间顺序） |
| 5 | **通信图** | Communication Diagram | 描述对象之间的交互（强调链接） |
| 6 | **定时图** | Timing Diagram | 描述对象状态随时间的变化 |
| 7 | **交互概览图** | Interaction Overview Diagram | 活动图和时序图的结合 |

### 4.3 记忆口诀

```
结构七图：类对组部包复轮
行为七图：用活状时交通定
```

## 五、UML 的核心概念

### 5.1 四层元模型架构

```
M3 元元模型（Meta-MetaModel）  →  定义建模语言的元素（如 Class、Attribute）
M2 元模型（MetaModel）         →  UML 本身的定义（如 UML Class、UML Association）
M1 模型（Model）               →  用户创建的模型（如 Student 类、Course 类）
M0 实例（Instance）            →  运行时的具体对象（如 student ZhangSan）
```

### 5.2 基本构造块

#### （1）事物（Things）—— 模型中最基本的元素

| 类型 | 说明 | 示例 |
|------|------|------|
| **结构事物** | 模型的静态部分 | 类、接口、组件、节点、用例 |
| **行为事物** | 模型的动态部分 | 交互、状态机、活动 |
| **分组事物** | 组织模型的容器 | 包（Package） |
| **注释事物** | 模型的解释说明 | 注释（Note） |

#### （2）关系（Relationships）—— 连接事物的纽带

| 关系 | 符号 | 说明 |
|------|------|------|
| **关联（Association）** | 实线 ——— | 类之间的结构性连接 |
| **依赖（Dependency）** | 虚线 - - -> | 一个类依赖另一个类 |
| **泛化（Generalization）** | 实线 + 空心三角 → | 继承关系（is-a） |
| **实现（Realization）** | 虚线 + 空心三角 → | 实现接口 |
| **聚合（Aggregation）** | 实线 + 空心菱形 ◇ | 整体-部分关系（has-a），可独立存在 |
| **组合（Composition）** | 实线 + 实心菱形 ◆ | 整体-部分关系（contains-a），不可独立存在 |

#### （3）图（Diagrams）—— 事物和关系的可视化集合

## 六、UML 中的关系详解

### 6.1 六种关系强度排序

```
依赖 < 关联 < 聚合 < 组合 < 泛化/实现
（弱）————————————————————————————→（强）
```

### 6.2 各关系详解

#### 依赖（Dependency）
- **含义**：一个类使用了另一个类，但不持有引用
- **典型场景**：方法参数、局部变量、静态方法调用
- **代码示例**：
```java
class A {
    void method(B b) {  // A 依赖 B
        b.doSomething();
    }
}
```

#### 关联（Association）
- **含义**：类之间有长期的结构性关系，一个类持有另一个类的引用
- **分类**：
  - **单向关联**：A → B（A 知道 B，B 不知道 A）
  - **双向关联**：A — B（互相知道）
  - **自关联**：A → A（如链表节点）
- **多重性**：`1`、`0..1`、`*`（0或多个）、`1..*`（1或多个）
- **代码示例**：
```java
class Teacher {
    Student student;  // Teacher 关联 Student
}
```

#### 聚合（Aggregation）
- **含义**：整体与部分的关系，但**部分可以独立于整体存在**
- **典型场景**：班级与学生、汽车与轮胎、部门与员工
- **代码示例**：
```java
class Department {
    List<Employee> employees;  // 聚合，员工可以不属于任何部门
}
```

#### 组合（Composition）
- **含义**：整体与部分的关系，**部分不能独立于整体存在**
- **典型场景**：人与心脏、窗口与按钮、订单与订单明细
- **代码示例**：
```java
class Order {
    List<OrderDetail> details;  // 组合，订单明细不能脱离订单存在
    Order() {
        details = new ArrayList<>();  // 在构造函数中创建
    }
}
```

#### 泛化（Generalization）
- **含义**：继承关系，子类继承父类的属性和方法
- **代码示例**：
```java
class Animal { }
class Dog extends Animal { }  // Dog 泛化自 Animal
}
```

#### 实现（Realization）
- **含义**：类实现接口的契约
- **代码示例**：
```java
interface Runnable {
    void run();
}
class MyThread implements Runnable {  // MyThread 实现 Runnable
    public void run() { }
}
```

## 八、UML 建模过程

```
1. 需求建模
   └── 用例图（描述系统功能）

2. 领域建模
   └── 类图（描述领域概念及其关系）

3. 设计建模
   ├── 类图（详细设计，包括方法和属性）
   ├── 时序图（描述交互流程）
   └── 状态图（描述复杂对象的状态变化）

4. 实现建模
   ├── 组件图（描述系统模块）
   └── 部署图（描述物理部署）

5. 测试建模
   └── 各类图作为测试用例设计的参考
```

## 九、考试重点提示（408/软考）

1. **类图的关系辨析**：依赖、关联、聚合、组合、泛化、实现六种关系的区分是高频考点
2. **用例图**：参与者、用例、包含/扩展/泛化关系
3. **时序图**：生命线、消息类型（同步/异步/返回）、激活条
4. **状态图**：状态、转换、事件、动作
5. **活动图**：动作节点、控制节点、泳道、分叉/汇合
