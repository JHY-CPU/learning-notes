# 类图（Class Diagram）

## 一、概述

类图是 UML 中**最重要、最常用**的图。它描述了系统的**静态结构**，包括：

- 系统中有哪些**类**
- 每个类有哪些**属性**和**方法**
- 类与类之间的**关系**

> 类图是面向对象设计的核心，也是代码实现的蓝图

## 二、类的表示

### 2.1 类的标准表示

类用矩形表示，分为三个区域：

```
┌──────────────────────────┐
│        ClassName         │  ← 类名（必须）
├──────────────────────────┤
│ - attribute1: Type       │  ← 属性（可选）
│ # attribute2: Type       │
│ + attribute3: Type = val │
├──────────────────────────┤
│ + method1(param): Type   │  ← 方法（可选）
│ # method2(): void        │
│ - method3(): boolean     │
└──────────────────────────┘
```

### 2.2 可见性修饰符

| 符号 | 含义 | Java | C++ | Python | 关键字 |
|------|------|------|-----|--------|--------|
| `+` | **公有**（Public） | `public` | `public` | 默认 | 对所有类可见 |
| `-` | **私有**（Private） | `private` | `private` | `__` | 仅本类可见 |
| `#` | **保护**（Protected） | `protected` | `protected` | `_` | 本类和子类可见 |
| `~` | **包**（Package） | 默认（包访问） | — | — | 同包内可见 |

### 2.3 属性的完整语法

```
可见性 名称: 类型 [多重性] = 默认值 { 属性字符串 }
```

**示例**：
```
- name: String                    // 私有属性
+ age: int = 0                    // 公有属性，默认值0
- scores: Double [*]              // 多值属性
# id: int {id > 0}                // 带约束
~ address: String [0..1]          // 包可见，可选
```

### 2.4 方法的完整语法

```
可见性 名称(参数列表): 返回类型 { 属性字符串}
```

**示例**：
```
+ getName(): String
+ setName(name: String): void
+ calculateArea(): double {readOnly}
- validate(): boolean
+ findById(id: int): Student [1]          // 返回单个
+ getStudents(): Student [*]              // 返回集合
```

**参数的完整格式**：
```
方向 参数名: 类型 = 默认值
```
方向：`in`（输入，默认）、`out`（输出）、`inout`（双向）

```
+ connect(host: in String, port: in int = 3306): boolean
+ getResult(output: out String): int
```

### 2.5 抽象类与抽象方法

```
┌──────────────────────────┐
│   «abstract» Shape       │  ← 类名用斜体
├──────────────────────────┤
│ # color: String          │
├──────────────────────────┤
│ + «abstract» area()      │  ← 方法名用斜体
│ + setColor(c: String)    │
└──────────────────────────┘
```

- 抽象类名：**斜体字**
- 抽象方法名：**斜体字**
- 或者加 `«abstract»` 构造型（stereotype）

### 2.6 接口

```
┌──────────────────────────┐
│   «interface» Drawable   │
├──────────────────────────┤
│                          │
├──────────────────────────┤
│ + draw(): void           │
│ + resize(factor: double) │
└──────────────────────────┘
```

或使用简化的棒棒糖表示法：
```
    ○── Drawable
    │
┌───┴────┐
│  Circle │
└─────────┘
```

### 2.7 枚举（Enumeration）

```
┌──────────────────────────┐
│   «enumeration» Color    │
├──────────────────────────┤
│ RED                      │
│ GREEN                    │
│ BLUE                     │
├──────────────────────────┤
│ + getValue(): int        │
└──────────────────────────┘
```

### 2.8 其他表示

**静态成员**：加下划线
```
+ totalCount: int          // 静态属性
+ getCount(): int          // 静态方法
```

**只读属性**：在类型后加 `{readOnly}`
```
+ id: int {readOnly}
```

**派生属性**：前面加 `/`
```
+ /area: double            // 派生属性，可以由其他属性计算得出
```

## 三、类之间的关系

### 3.1 六种关系总览

| 关系 | 符号 | 强度 | 代码体现 |
|------|------|------|---------|
| 依赖 | `---->` 虚线箭头 | 最弱 | 局部变量、方法参数 |
| 关联 | `———>` 实线箭头 | 弱 | 成员变量 |
| 聚合 | `◇———` 空心菱形 | 中 | 成员变量（非独占） |
| 组合 | `◆———` 实心菱形 | 强 | 成员变量（独占，构造时创建） |
| 泛化 | `▷———` 空心三角 | 最强 | `extends` / `:` |
| 实现 | `▷--->` 空心三角虚线 | 最强 | `implements` |

### 3.2 依赖（Dependency）

**含义**：一个类**临时使用**另一个类，不持有其引用。

**符号**：虚线 + 箭头 `- - - >`

**代码体现**：
```java
class OrderService {
    // 1. 方法参数（最常见的依赖）
    void placeOrder(Order order) { }

    // 2. 局部变量
    void process() {
        Validator v = new Validator();  // 局部变量
        v.validate();
    }

    // 3. 静态方法调用
    void log() {
        Logger.log("order placed");  // 不持有引用
    }
}
```

**判断方法**：
- A 类的方法中**临时使用**了 B 类
- A 类**不持有** B 类的引用（不是成员变量）
- 删除 B 类后，A 类需要**修改代码**才能编译

### 3.3 关联（Association）

**含义**：类之间有**长期的结构性关系**，一个类**持有**另一个类的引用。

**符号**：实线 + 箭头 `———>`

**分类**：

| 类型 | 说明 | 表示 |
|------|------|------|
| 单向关联 | A 持有 B 的引用，B 不持有 A 的引用 | A ———→ B |
| 双向关联 | A 和 B 互相持有引用 | A ←———→ B |
| 自关联 | 类持有自身的引用 | A ———→ A |

**多重性（Multiplicity）**：

| 表示 | 含义 |
|------|------|
| `1` | 恰好 1 个 |
| `0..1` | 0 或 1 个 |
| `*` 或 `0..*` | 0 到多个 |
| `1..*` | 1 到多个 |
| `3..5` | 3 到 5 个 |
| `2,4,6` | 恰好 2、4 或 6 个 |

**代码体现**：
```java
class Teacher {
    private String name;
    private List<Student> students;  // 1对多关联
}

class Student {
    private String name;
    private Teacher teacher;  // 多对1关联
}
```

**角色名和导航性**：
```
Teacher ———→ 「teaches」Student
 (teacher)        (student)
```

### 3.4 聚合（Aggregation）

**含义**：整体与部分的关系，但部分**可以独立存在**。

**符号**：空心菱形 + 实线 `◇———`

**关键特征**：
- "has-a" 或 "owns-a" 关系
- 部分的**生命周期独立**于整体
- 整体不负责部分的创建和销毁
- 部分可以在不同整体之间转移

**典型场景**：
```
Department ◇——— Employee
  （部门）     （员工）

- 员工可以不属于任何部门
- 员工可以在部门之间调动
- 部门解散后，员工仍然存在
```

**代码体现**：
```java
class Department {
    private String name;
    private List<Employee> employees;  // 聚合

    // 员工不是在这里创建的，而是从外部传入
    void addEmployee(Employee e) {
        employees.add(e);
    }
}
```

### 3.5 组合（Composition）

**含义**：整体与部分的关系，部分**不能独立存在**。

**符号**：实心菱形 + 实线 `◆———`

**关键特征**：
- "contains-a" 关系（比聚合更强）
- 部分的**生命周期依赖**于整体
- 整体负责部分的创建和销毁
- 部分不能脱离整体而存在

**典型场景**：
```
Order ◆——— OrderDetail
 （订单）   （订单明细）

- 订单明细不能脱离订单存在
- 订单删除后，明细也必须删除
- 明细由订单创建
```

```java
class Order {
    private List<OrderDetail> details;

    Order() {
        this.details = new ArrayList<>();  // 在构造函数中创建
    }

    void addDetail(String product, int qty, double price) {
        details.add(new OrderDetail(product, qty, price));  // 内部创建
    }
}

class OrderDetail {
    private String product;
    private int quantity;
    private double price;

    OrderDetail(String product, int qty, double price) {
        this.product = product;
        this.quantity = qty;
        this.price = price;
    }
}
```

### 3.6 聚合 vs 组合对比

| 特性 | 聚合 | 组合 |
|------|------|------|
| 菱形 | 空心 ◇ | 实心 ◆ |
| 关系强度 | 较弱 | 较强 |
| 生命周期 | 部分可独立 | 部分不可独立 |
| 创建方式 | 外部注入 | 内部创建 |
| 删除影响 | 整体删除不影响部分 | 整体删除部分也删除 |
| 类比 | 学校与学生 | 人与心脏 |
| 典型例子 | 部门与员工、班级与学生 | 窗口与按钮、订单与明细 |

**记忆技巧**：
- 聚合：**空心**菱形 → 关系**松散** → 部分可**空**（独立）
- 组合：**实心**菱形 → 关系**紧密** → 部分**实**（依附）

### 3.7 泛化（Generalization）

**含义**：继承关系，子类继承父类。

**符号**：实线 + 空心三角 `▷———`

```
          Animal
            △
           / \
         Cat  Dog
```

```java
class Animal {
    protected String name;
    public void eat() { }
}

class Cat extends Animal {
    public void meow() { }
}

class Dog extends Animal {
    public void bark() { }
}
```

**多重继承**：
```
    Flyable    Swimmable
        \        /
         \      /
          Duck
```

### 3.8 实现（Realization）

**含义**：类实现接口。

**符号**：虚线 + 空心三角 `▷- - ->`

```
   «interface» Runnable
          △  (虚线)
         / \
       T1  T2
```

```java
interface Runnable {
    void run();
}

class MyThread implements Runnable {
    public void run() { }
}
```

## 四、关联类

当关联关系本身需要属性时，使用**关联类**。

```
Student ——————————— Course
            │
     ┌──────┴──────┐
     │  Enrollment  │
     ├──────────────┤
     │ grade: float │
     │ semester: str│
     └──────────────┘
```

```java
class Student {
    private String name;
    private List<Enrollment> enrollments;
}

class Course {
    private String name;
    private List<Enrollment> enrollments;
}

class Enrollment {  // 关联类
    private Student student;
    private Course course;
    private float grade;
    private String semester;
}
```

## 六、约束和注释

### 6.1 约束（Constraint）

用花括号 `{}` 表示约束条件：
```
+ age: int {age >= 0 && age <= 150}
- password: String {length >= 8}
```

### 6.2 注释（Note）

用折角矩形表示注释，虚线连接到相关元素：
```
┌──────────────────┐        ┌──────────────┐
│    Order         │--------│  注意：订单   │
├──────────────────┤        │  金额必须大于 │
│ + amount: double │        │  0           │
└──────────────────┘        └──────────────┘
```

## 八、考试重点

1. **六种关系的区分**：重点掌握区分方法
   - 依赖最弱：方法参数/局部变量
   - 关联：成员变量
   - 聚合 vs 组合：部分能否独立存在
   - 泛化/实现最强：继承/实现接口

2. **多重性标注**：能正确标注类之间的多重性

3. **类图设计**：能根据需求描述绘制类图

4. **类图 ↔ 代码转换**：能从类图推导代码，也能从代码画出类图

5. **设计模式**：常见设计模式的类图表示（策略、工厂、观察者等）
