# 52-软件架构

软件架构是软件系统的高层结构，决定了系统的质量属性、可维护性和可扩展性。本模块覆盖架构风格、领域驱动设计和架构评估三大核心主题。

## 目录结构

| 子目录 | 主题 | 说明 |
|--------|------|------|
| [01-架构风格](./01-架构风格/README.md) | 架构风格 | 分层、微服务、事件驱动、Serverless等主流架构风格 |
| [02-领域驱动设计](./02-领域驱动设计/README.md) | 领域驱动设计 | DDD战略与战术设计、CQRS、Event Sourcing |
| [03-架构评估](./03-架构评估/README.md) | 架构评估 | ATAM、SAAM、ADR等架构评估方法 |

## 学习路径

1. **架构风格**：了解主流架构风格的原理与适用场景
2. **领域驱动设计**：掌握复杂业务系统的建模方法
3. **架构评估**：学会评估和记录架构决策
