# 分层架构与管道-过滤器架构

一、架构风格概述

## 一、架构风格概述


架构风格（Architectural Style）是对系统结构和组织方式的分类和抽象，为软件架构设计提供了可复用的模式和约束。


> **Note:** **架构风格的作用：**
> 定义组件的类型、连接方式、数据流向和集成策略，帮助架构师做出一致的设计决策。

二、分层架构

## 二、分层架构（Layered Architecture）


分层架构将系统划分为若干层次，每层只与其相邻的层交互，是最广泛使用的架构风格之一。


### 2.1 经典分层模型


| 层次 | 职责 | 典型技术 |
| --- | --- | --- |
| 表示层（Presentation） | 用户界面、输入验证、页面展示 | HTML/CSS/JS、React、Vue |
| 业务逻辑层（Business） | 业务规则、流程编排、领域逻辑 | Spring Service、.NET Service |
| 持久化层（Persistence） | 数据访问、ORM映射、查询处理 | Hibernate、MyBatis、JPA |
| 数据层（Database） | 数据存储、管理 | MySQL、PostgreSQL、MongoDB |


### 2.2 分层原则


- **严格分层：**
   每层只能访问其直接下层
- **松散分层：**
   允许跳层访问（性能优化场景）
- **层间通信：**
   通过定义良好的接口（API）交互
- **单向依赖：**
   上层依赖下层，下层不依赖上层


### 2.3 优缺点分析


| 优点 | 缺点 |
| --- | --- |
| 关注点分离，职责清晰 | 层次过多导致性能开销 |
| 可替换性强，层内可独立修改 | 并非所有系统都适合分层 |
| 利于团队分工和测试 | 可能导致"污水池"反模式 |
| 复用性好，层可跨项目复用 | 某些场景需要穿透多层，效率低 |

**"污水池"反模式：**
某一层仅仅是简单的转发调用，没有添加任何价值，导致不必要的复杂性。解决方案：合并无意义的层，或在层中增加有价值的逻辑。
三、管道-过滤器架构

## 三、管道-过滤器架构（Pipe-and-Filter）


管道-过滤器架构将系统组织为一系列过滤器（Filter）通过管道（Pipe）连接的处理流水线，每个过滤器对输入数据进行处理后输出。


### 3.1 核心组件


- **过滤器（Filter）：**
   独立的数据处理组件，接收输入、处理数据、产生输出
- **管道（Pipe）：**
   连接过滤器的数据传输通道
- **数据流：**
   数据从源经过多个过滤器流向目的地


### 3.2 过滤器特性


- 过滤器之间相互独立，不共享状态
- 每个过滤器完成单一的数据转换任务
- 过滤器可以复用和重新组合
- 支持增量处理和流式处理


### 3.3 典型应用


| 应用场景 | 示例 |
| --- | --- |
| Unix Shell | `cat file.txt \| grep "keyword" \| sort \| uniq` |
| 编译器 | 词法分析 → 语法分析 → 语义分析 → 代码生成 |
| 数据处理 | ETL管道：提取 → 转换 → 加载 |
| 图像处理 | 滤镜链：灰度化 → 模糊 → 边缘检测 |
| Web中间件 | 请求 → 认证 → 日志 → 业务处理 → 响应 |


### 3.4 优缺点


| 优点 | 缺点 |
| --- | --- |
| 高内聚低耦合，易复用 | 不适合交互式应用 |
| 支持并行处理 | 数据格式转换开销 |
| 易于理解和维护 | 错误处理困难 |
| 可动态重组处理流程 | 不适合共享数据场景 |

四、仓库架构

## 四、仓库架构风格（Repository Style）


仓库架构以一个中央数据存储（Repository）为核心，所有子系统通过中央存储进行数据交换和通信。


### 4.1 两种变体


| 变体 | 控制方式 | 特点 |
| --- | --- | --- |
| 数据库系统 | 由输入数据流驱动 | 子系统独立运行，响应数据变化 |
| 黑板系统 | 由中央存储驱动 | 适合不确定的推理过程，如AI系统 |


### 4.2 黑板架构组件


- **黑板（Blackboard）：**
   中央数据存储，存放问题求解过程中的所有数据
- **知识源（Knowledge Source）：**
   独立的专业模块，监控黑板并贡献解决方案
- **控制组件：**
   协调知识源的激活和执行顺序


> **Note:** **典型应用：**
> 语音识别、模式识别、自然语言处理等需要多领域知识协同的问题求解系统。
