# 物理系统与碰撞

## 核心概念

Unity物理系统基于PhysX引擎（Unity内置），处理刚体运动、碰撞检测和物理模拟。核心组件为Rigidbody（刚体）和Collider（碰撞体），两者配合实现物理交互。

## Rigidbody（刚体）

Rigidbody使GameObject受物理引擎控制，自动处理重力、碰撞响应和力的作用：

```csharp
public class PlayerPhysics : MonoBehaviour
{
    private Rigidbody rb;
    [SerializeField] private float moveForce = 10f;
    [SerializeField] private float jumpForce = 8f;

    void Start()
    {
        rb = GetComponent<Rigidbody>();
        // 常用属性设置
        rb.mass = 1f;              // 质量
        rb.drag = 0.5f;            // 线性阻力
        rb.angularDrag = 0.05f;    // 角阻力
        rb.useGravity = true;      // 是否受重力
        rb.isKinematic = false;    // 是否为运动学模式
        rb.interpolation = RigidbodyInterpolation.Interpolated; // 平滑插值
        rb.collisionDetectionMode = CollisionDetectionMode.Continuous; // 连续碰撞检测
    }

    void FixedUpdate()
    {
        // 施加力（受质量影响）
        float h = Input.GetAxis("Horizontal");
        rb.AddForce(Vector3.right * h * moveForce);

        // 施加冲击力（瞬时力）
        if (Input.GetKeyDown(KeyCode.Space))
            rb.AddForce(Vector3.up * jumpForce, ForceMode.Impulse);

        // 直接设置速度（不受力的影响）
        rb.linearVelocity = new Vector3(h * 5f, rb.linearVelocity.y, 0);
    }
}
```

## Collider（碰撞体）

Collider定义物体的物理形状，用于碰撞检测。碰撞体本身不一定是可见的网格。

```csharp
// 常见Collider类型
// BoxCollider - 立方体（UI元素、箱子）
// SphereCollider - 球体（子弹、球类）
// CapsuleCollider - 胶囊体（角色控制器，最常用）
// MeshCollider - 网格碰撞（地形、复杂形状，性能开销大）

public class ColliderSetup : MonoBehaviour
{
    void Start()
    {
        // BoxCollider
        BoxCollider box = gameObject.AddComponent<BoxCollider>();
        box.center = Vector3.zero;
        box.size = new Vector3(1, 2, 1);

        // MeshCollider（必须是凸面体才能与Rigidbody配合）
        MeshCollider meshCol = gameObject.AddComponent<MeshCollider>();
        meshCol.convex = true;  // 凸面体，否则不能添加Rigidbody
        meshCol.sharedMesh = GetComponent<MeshFilter>().mesh;
    }
}
```

## Trigger vs Collision

| 对比 | Collision（碰撞） | Trigger（触发器） |
|------|-------------------|-------------------|
| Collider设置 | isTrigger = false | isTrigger = true |
| 物理响应 | 有碰撞反弹 | 无物理响应，穿过物体 |
| 回调函数 | OnCollisionEnter/Stay/Exit | OnTriggerEnter/Stay/Exit |
| 典型用途 | 墙壁、地面、障碍物 | 拾取区域、伤害区域、门触发器 |

```csharp
public class CollisionExample : MonoBehaviour
{
    // === 碰撞回调（isTrigger = false）===
    void OnCollisionEnter(Collision collision)
    {
        // collision 包含碰撞点、法线、相对速度等信息
        ContactPoint contact = collision.contacts[0];
        Debug.Log($"碰撞点: {contact.point}, 法线: {contact.normal}");
        Debug.Log($"碰撞对象: {collision.gameObject.name}");

        // 碰撞力度
        float impactForce = collision.impulse.magnitude;
    }

    void OnCollisionStay(Collision collision) { }
    void OnCollisionExit(Collision collision) { }

    // === 触发器回调（isTrigger = true）===
    void OnTriggerEnter(Collider other)
    {
        if (other.CompareTag("Player"))
        {
            // 拾取道具逻辑
            other.GetComponent<PlayerHealth>().Heal(20);
            Destroy(gameObject);
        }
    }

    void OnTriggerStay(Collider other) { }
    void OnTriggerExit(Collider other) { }
}
```

## 物理材质（Physics Material）

控制物体表面的摩擦力和弹性：

```csharp
// 通过代码创建物理材质
PhysicMaterial bounceMat = new PhysicMaterial();
bounceMat.dynamicFriction = 0.2f;   // 动摩擦系数
bounceMat.staticFriction = 0.2f;    // 静摩擦系数
bounceMat.bounciness = 0.8f;        // 弹性系数 (0-1)
bounceMat.bounceCombine = PhysicMaterialCombine.Maximum; // 弹性组合方式
GetComponent<Collider>().material = bounceMat;
```

## Layer碰撞矩阵

通过Edit -> Project Settings -> Physics配置哪些Layer之间可以发生碰撞：

```csharp
// 代码中忽略特定Layer之间的碰撞
Physics.IgnoreLayerCollision(8, 9, true); // Layer 8和9不碰撞

// 或使用LayerMask进行射线检测过滤
int layerMask = ~(1 << 2); // 排除Layer 2（Ignore Raycast）
if (Physics.Raycast(ray, out RaycastHit hit, 100f, layerMask))
{
    Debug.Log($"射线击中: {hit.collider.name}");
}
```

## 常见陷阱与最佳实践

1. **不要在Update中做物理操作**: 物理计算应在FixedUpdate中进行，避免帧率依赖问题
2. **isKinematic的使用**: 运动学刚体不受力影响，适合平台移动、动画驱动的物体
3. **MeshCollider性能差**: 尽量用基本Collider组合替代，或仅用于静态物体
4. **连续碰撞检测**: 高速移动的物体（子弹）必须开启Continuous检测，否则可能穿透
5. **Trigger需要Rigidbody**: 至少一方需要Rigidbody才能触发OnTrigger回调

## 与其他系统的关联

- **射线检测**: Physics.Raycast基于Collider进行检测
- **Layer系统**: 与渲染Layer共用，但物理碰撞矩阵独立配置
- **Navigation**: NavMesh系统独立于物理系统，但可结合使用
