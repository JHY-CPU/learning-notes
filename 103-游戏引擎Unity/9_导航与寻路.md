# 导航与寻路

## 核心概念

Unity Navigation系统基于NavMesh（导航网格）实现AI寻路。NavMesh从场景几何体烘焙出可行走区域，NavMeshAgent组件在该区域内自动计算最优路径。

## NavMesh烘焙

NavMesh通过烘焙场景几何体生成导航网格：

1. 将地面标记为Navigation Static
2. Window -> AI -> Navigation打开导航窗口
3. 设置Agent参数（半径、高度、坡度、台阶高度）
4. 点击Bake生成NavMesh

```csharp
// 运行时烘焙NavMesh（Unity 2021+）
using Unity.AI.Navigation;

public class RuntimeNavMesh : MonoBehaviour
{
    [SerializeField] private NavMeshSurface navMeshSurface;

    public void RebuildNavMesh()
    {
        navMeshSurface.BuildNavMesh();
    }
}
```

### Bake关键参数

| 参数 | 说明 | 典型值 |
|------|------|--------|
| Agent Radius | Agent半径 | 0.5 |
| Agent Height | Agent可通过的最低高度 | 2.0 |
| Max Slope | 最大可行走坡度 | 45度 |
| Step Height | 最大台阶高度 | 0.4 |
| Min Region Area | 最小区域面积（过小区域被忽略） | 2 |

## NavMeshAgent

```csharp
public class EnemyAI : MonoBehaviour
{
    private NavMeshAgent agent;
    [SerializeField] private Transform target;

    void Start()
    {
        agent = GetComponent<NavMeshAgent>();

        // Agent参数
        agent.speed = 3.5f;           // 移动速度
        agent.angularSpeed = 120f;    // 转向速度
        agent.acceleration = 8f;      // 加速度
        agent.stoppingDistance = 1.5f; // 到达距离
        agent.radius = 0.5f;          // 碰撞半径
        agent.height = 2f;            // 碰撞高度
        agent.obstacleAvoidanceType = ObstacleAvoidanceType.HighQualityObstacleAvoidance;
    }

    void Update()
    {
        // 设置目标位置（Agent自动计算路径）
        agent.SetDestination(target.position);

        // 检查是否到达
        if (!agent.pathPending && agent.remainingDistance <= agent.stoppingDistance)
        {
            if (!agent.hasPath || agent.velocity.sqrMagnitude == 0f)
            {
                Debug.Log("已到达目标");
            }
        }

        // 直接控制Agent速度
        agent.velocity = transform.forward * speed;

        // 暂停/恢复Agent
        if (Input.GetKeyDown(KeyCode.P))
            agent.isStopped = !agent.isStopped;

        // 瞬移到指定位置（在NavMesh上）
        WarpToPosition(new Vector3(10, 0, 5));
    }

    void WarpToPosition(Vector3 position)
    {
        NavMeshHit hit;
        if (NavMesh.SamplePosition(position, out hit, 5f, NavMesh.AllAreas))
        {
            agent.Warp(hit.position);
        }
    }
}
```

## Off-Mesh Link

Off-Mesh Link用于连接不连续的NavMesh区域（跳跃、攀爬、传送门）：

```csharp
public class JumpLink : MonoBehaviour
{
    // 通过代码创建Off-Mesh Link
    void CreateLink()
    {
        OffMeshLink link = gameObject.AddComponent<OffMeshLink>();
        link.startTransform = startPoint;
        link.endTransform = endPoint;
        link.biDirectional = true;  // 双向通行
        link.activated = true;
    }
}

// Agent处理Off-Mesh Link
public class AgentLinkHandler : MonoBehaviour
{
    private NavMeshAgent agent;

    IEnumerator Start()
    {
        agent = GetComponent<NavMeshAgent>();
        agent.autoTraverseOffMeshLink = false; // 手动处理

        while (true)
        {
            if (agent.isOnOffMeshLink)
            {
                // 自定义跨越动画
                OffMeshLinkData data = agent.currentOffMeshLinkData;
                Vector3 endPos = data.endPos;

                // 跳跃动画
                yield return StartCoroutine(JumpTo(endPos));
                agent.CompleteOffMeshLink();
            }
            yield return null;
        }
    }

    IEnumerator JumpTo(Vector3 target)
    {
        Vector3 start = transform.position;
        float duration = 0.5f;
        float elapsed = 0;

        while (elapsed < duration)
        {
            elapsed += Time.deltaTime;
            float t = elapsed / duration;
            // 抛物线插值
            Vector3 pos = Vector3.Lerp(start, target, t);
            pos.y += Mathf.Sin(t * Mathf.PI) * 2f;
            transform.position = pos;
            yield return null;
        }
    }
}
```

## NavMesh Obstacle

动态障碍物组件，使Agent在运行时避开移动物体：

```csharp
// Agent会自动避开挂载NavMeshObstacle的物体
NavMeshObstacle obstacle = gameObject.AddComponent<NavMeshObstacle>();
obstacle.shape = NavMeshObstacleShape.Capsule;
obstacle.radius = 0.5f;
obstacle.height = 2f;
obstacle.carving = true;         // 动态从NavMesh挖洞
obstacle.carveOnlyStationary = true; // 只在静止时挖洞（性能优化）
```

## 常见陷阱与最佳实践

1. **Agent卡住问题**: 当Agent被推离NavMesh时会丢失路径，使用SamplePosition重新吸附
2. **性能考虑**: 大量Agent使用Obstacle Avoidance会消耗CPU，适当降低质量等级
3. **多Agent尺寸**: 不同大小的角色需要不同的NavMesh Agent Type，分别烘焙
4. **NavMesh Obstacle vs Agent**: 静态动态障碍物用Obstacle，需要寻路的用Agent
5. **分层NavMesh**: 复杂地形使用多层NavMesh（如地面+屋顶+室内）

## 与其他系统的关联

- **物理系统**: NavMesh与物理系统独立，但Agent可受物理力影响
- **动画系统**: Agent移动需配合动画系统播放行走/跑步动画
- **AI系统**: NavMesh是AI行为树的底层移动支撑
