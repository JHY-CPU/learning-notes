# 粒子系统

## 核心概念

Unity粒子系统（Shuriken）用于创建视觉效果如火焰、烟雾、魔法、爆炸等。粒子从发射器产生，受各种模块控制其生命周期内的行为。每个粒子是独立的点精灵（Billboard），由系统统一管理。

## Particle System核心模块

```
Particle System
├── Main (主模块) - 生命周期、大小、颜色、速度
├── Emission (发射) - 发射速率、爆发
├── Shape (形状) - 发射区域形状
├── Velocity over Lifetime - 生命周期内速度变化
├── Limit Velocity over Lifetime - 速度限制
├── Inherit Velocity - 继承发射器速度
├── Force over Lifetime - 生命周期内受力
├── Color over Lifetime - 颜色渐变
├── Color by Speed - 按速度着色
├── Size over Lifetime - 大小变化
├── Size by Speed - 按速度缩放
├── Rotation over Lifetime - 旋转变化
├── Noise - 噪声扰动
├── Collision - 碰撞检测
├── Trails - 粒子拖尾
├── Sub Emitters - 子发射器
├── Texture Sheet - 纹理动画
├── Lights - 粒子光源
├── Trails - 轨迹
└── Renderer - 渲染设置
```

## 基础使用

```csharp
public class ParticleController : MonoBehaviour
{
    [SerializeField] private ParticleSystem fireEffect;
    [SerializeField] private ParticleSystem explosionPrefab;

    void Update()
    {
        // 控制播放
        if (Input.GetKeyDown(KeyCode.F))
        {
            fireEffect.Play();
        }
        if (Input.GetKeyUp(KeyCode.F))
        {
            fireEffect.Stop();
        }
    }

    // 实例化粒子效果
    public void SpawnExplosion(Vector3 position)
    {
        ParticleSystem explosion = Instantiate(explosionPrefab, position, Quaternion.identity);

        // 自动销毁（粒子播完后）
        Destroy(explosion.gameObject, explosion.main.duration + explosion.main.startLifetime.constantMax);
    }

    // 动态修改粒子属性
    void ModifyParticles()
    {
        // 修改主模块
        var main = fireEffect.main;
        main.startColor = Color.red;
        main.startSize = 2f;
        main.startSpeed = 5f;
        main.simulationSpace = ParticleSystemSimulationSpace.World;

        // 修改发射模块
        var emission = fireEffect.emission;
        emission.rateOverTime = 50f;

        // 添加爆发（一次性发射多个粒子）
        emission.SetBursts(new ParticleSystem.Burst[]
        {
            new ParticleSystem.Burst(0f, 30),   // 时间0秒，发射30个
            new ParticleSystem.Burst(1f, 50),   // 时间1秒，发射50个
        });

        // 修改形状
        var shape = fireEffect.shape;
        shape.shapeType = ParticleSystemShapeType.Cone;
        shape.angle = 25f;
        shape.radius = 0.5f;
    }
}
```

## 碰撞模块

```csharp
void SetupCollision()
{
    var collision = fireEffect.collision;
    collision.enabled = true;
    collision.type = ParticleSystemCollisionType.World;      // 与世界碰撞
    collision.mode = ParticleSystemCollisionMode.Collision3D;
    collision.bounce = 0.5f;         // 弹性系数
    collision.lifetimeLoss = 0.2f;   // 碰撞后生命值损失
    collision.collidesWith = ~0;     // 与所有Layer碰撞

    // 碰撞回调
    fireEffect.trigger.SetCollider(0, someCollider); // 设置触发器Collider
}

// 粒子碰撞回调脚本（需挂载到有Collider的对象上）
public class ParticleCollisionHandler : MonoBehaviour
{
    void OnParticleCollision(GameObject other)
    {
        ParticleSystem ps = other.GetComponent<ParticleSystem>();
        List<ParticleCollisionEvent> events = new List<ParticleCollisionEvent>();
        int count = ps.GetCollisionEvents(gameObject, events);

        for (int i = 0; i < count; i++)
        {
            Vector3 hitPos = events[i].intersection;
            Debug.Log($"粒子碰撞位置: {hitPos}");
        }
    }
}
```

## 子发射器

子发射器在特定事件时产生新的粒子效果：

```csharp
void SetupSubEmitters()
{
    var subEmitters = fireEffect.subEmitters;
    subEmitters.enabled = true;

    // 绑定子粒子效果
    subEmitters.AddSubEmitter(deathEffect, ParticleSystemSubEmitterType.Death,
                              ParticleSystemSubEmitterProperties.InheritColor);

    // 事件类型：
    // Birth - 每个粒子诞生时触发
    // Collision - 粒子碰撞时触发
    // Death - 粒子死亡时触发
}
```

## 粒子效果池

粒子系统也有对象池优化：

```csharp
public class ParticlePool : MonoBehaviour
{
    [SerializeField] private ParticleSystem effectPrefab;
    private Queue<ParticleSystem> pool = new Queue<ParticleSystem>();

    public ParticleSystem Get()
    {
        if (pool.Count > 0)
        {
            ParticleSystem ps = pool.Dequeue();
            ps.gameObject.SetActive(true);
            return ps;
        }
        return Instantiate(effectPrefab);
    }

    public void Return(ParticleSystem ps)
    {
        ps.Stop(true, ParticleSystemStopBehavior.StopEmitting);
        ps.gameObject.SetActive(false);
        pool.Enqueue(ps);
    }
}
```

## 常见陷阱与最佳实践

1. **粒子数量控制**: 大量粒子严重影响性能，设置Max Particles上限
2. **Material开销**: 每个不同材质的粒子系统产生独立Draw Call，尽量复用材质
3. **自动销毁**: 瞬发效果需确保粒子播完后销毁，避免内存泄漏
4. **LOD策略**: 远处降低粒子数量或使用更简单的替代效果
5. **屏幕空间 vs 世界空间**: UI上的粒子注意设置Simulation Space

## 与其他系统的关联

- **Shader**: 粒子渲染依赖粒子专用Shader
- **物理系统**: 粒子碰撞模块可与场景物理对象交互
- **后处理**: Bloom等后处理效果可增强粒子发光效果
