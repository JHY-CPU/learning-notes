# 学习曲线诊断偏差与方差问题

## 核心概念
- **学习曲线 (Learning Curve)**：绘制模型在训练集和验证集上的性能（误差或准确率）随训练样本量变化的曲线，是诊断偏差-方差问题的重要工具。
- **高偏差诊断**：训练误差和验证误差都很高且接近——模型容量不够，欠拟合。随着样本量增加，误差趋于平稳，不再下降。
- **高方差诊断**：训练误差很低但验证误差很高——两条曲线之间存在巨大差距。增加样本量通常能缩小差距。
- **诊断流程**：观察学习曲线 → 判断偏差/方差问题 → 针对性调整（偏差高 ➔ 增加模型复杂度；方差高 ➔ 增加数据或正则化）。
- **训练曲线 vs 验证曲线**：训练曲线随样本量增加通常上升（更难完美拟合更多数据），验证曲线下降（更多数据有助于泛化）。
- **理想曲线**：两条曲线最终收敛到较低的误差水平，且差距较小。

## 数学推导
**学习曲线的期望形式**：
对于给定模型和训练集大小 $n$，定义：
$$
\epsilon_{\text{train}}(n) = \mathbb{E}[\text{Training Error at size } n]
$$
$$
\epsilon_{\text{val}}(n) = \mathbb{E}[\text{Validation Error at size } n]
$$

**高偏差场景**：
$$
\lim_{n \to \infty} \epsilon_{\text{train}}(n) \approx \lim_{n \to \infty} \epsilon_{\text{val}}(n) \approx \text{model bias} \gg 0
$$
训练误差和验证误差都收敛到一个较高的水平，差距很小。增加数据不会改善性能。

**高方差场景**：
$$
\epsilon_{\text{train}}(n) \ll \epsilon_{\text{val}}(n) \quad \text{for all } n
$$
$$
\lim_{n \to \infty} (\epsilon_{\text{val}}(n) - \epsilon_{\text{train}}(n)) \to \text{small}
$$
训练误差远低于验证误差。随着样本量增加，差距缩小，但可能需要更多数据才能完全收敛。

**学习曲线的幂律拟合**：
经验上，学习曲线可以拟合为：
$$
\epsilon(n) = a \cdot n^{-b} + c
$$
其中 $c$ 是不可避免的误差（Bayes error），$a n^{-b}$ 是随样本量减少的偏差/方差项。

**可避免偏差 (Avoidable Bias)**：
$$
\text{Avoidable Bias} = \epsilon_{\text{train}}(n) - \epsilon_{\text{Bayes}}
$$
如果验证误差 > 人类/最优误差，且训练误差也高于最优误差，说明存在可避免的偏差，需要增加模型容量。

## 直观理解
- **学习曲线像体检报告**：学习曲线展示了模型的"健康状态"——高偏差 = 营养不良（模型太弱），高方差 = 过敏反应（对训练数据中的细微变化过度敏感）。看懂曲线就能对症下药。
- **两条曲线间的"鸿沟"**：想象用不同份量的食物（数据量）训练一个学生（模型）。训练成绩（训练误差）和考试成绩（验证误差）之间的差距过大，说明学生只是在"背答案"（过拟合）。差距太小但两线都低，说明学生真懂（好模型）。差距小但两线都高，说明老师教得不够（欠拟合）。
- **增加数据的边际效应**：当学习曲线变得平坦时（增加样本不再降低验证误差），问题要么是偏差主导（需要更复杂的模型），要么是数据已经足够但模型无法利用更多信息。

## 代码示例
```python
import numpy as np
import matplotlib.pyplot as plt
from sklearn.model_selection import learning_curve
from sklearn.svm import SVC
from sklearn.tree import DecisionTreeClassifier
from sklearn.datasets import load_digits

digits = load_digits()
X, y = digits.data, digits.target

# 高偏差模型（线性SVM，太简单）
train_sizes_svm, train_scores_svm, val_scores_svm = learning_curve(
    SVC(kernel='linear'), X, y, cv=5,
    train_sizes=np.linspace(0.1, 1.0, 10),
    scoring='accuracy'
)

# 高方差模型（决策树不剪枝，太复杂）
train_sizes_dt, train_scores_dt, val_scores_dt = learning_curve(
    DecisionTreeClassifier(max_depth=None), X, y, cv=5,
    train_sizes=np.linspace(0.1, 1.0, 10),
    scoring='accuracy'
)

# 输出 SVM 学习曲线数据
print("=== 线性SVM（高偏差）===")
for i, size in enumerate(train_sizes_svm):
    train_mean = train_scores_svm[i].mean()
    val_mean = val_scores_svm[i].mean()
    gap = train_mean - val_mean
    print(f"样本数={size:3d}: 训练准确率={train_mean:.3f}, "
          f"验证准确率={val_mean:.3f}, 差距={gap:.3f}")

print("\n=== 决策树（高方差）===")
for i, size in enumerate(train_sizes_dt):
    train_mean = train_scores_dt[i].mean()
    val_mean = val_scores_dt[i].mean()
    gap = train_mean - val_mean
    print(f"样本数={size:3d}: 训练准确率={train_mean:.3f}, "
          f"验证准确率={val_mean:.3f}, 差距={gap:.3f}")

# 诊断结论
print("\n=== 诊断 ===")
print("线性SVM: 训练与验证准确率都低且接近 → 高偏差（欠拟合）")
print("决策树:  训练准确率远高于验证准确率 → 高方差（过拟合）")
```

## 深度学习关联
- **数据量与模型容量的关系**：深度学习中的"Scaling Law"表明，在足够数据条件下，更大的模型持续受益于更多的参数和数据。学习曲线可以帮助判断当前项目处于"欠参数化"还是"过参数化"阶段。
- **迁移学习中的学习曲线**：微调预训练模型时，学习曲线可用于判断"灾难性遗忘"——如果训练损失下降但验证损失突然上升，说明模型正在忘记预训练学到的通用知识。
- **早停与学习曲线**：深度学习训练中的早停 (Early Stopping) 正是基于学习曲线原理——当验证损失不再下降（或开始上升）时停止训练，防止验证曲线进入高方差区域。
