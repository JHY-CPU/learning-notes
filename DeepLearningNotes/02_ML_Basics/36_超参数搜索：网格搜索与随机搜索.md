# 超参数搜索：网格搜索与随机搜索

## 核心概念
- **超参数 (Hyperparameters)**：模型训练前需要设置的参数（如树的深度、学习率、正则化强度），不通过训练数据学习，需要手动调优。
- **网格搜索 (Grid Search)**：穷举所有指定的超参数组合，对每种组合进行交叉验证评估，选出最优组合。简单可靠但计算复杂度随参数数量指数增长。
- **随机搜索 (Random Search)**：在指定的超参数空间中随机采样组合进行评估。Bergstra & Bengio (2012) 证明，随机搜索在相同计算预算下通常优于网格搜索。
- **维度诅咒 (Curse of Dimensionality for HP)**：如果需调优 $d$ 个超参数，每个参数有 $v$ 个候选值，网格搜索需要 $v^d$ 次评估。当 $d \geq 4$ 时，计算量变得难以承受。
- **随机搜索的优势**：随机搜索每次尝试都独立地探索不同的组合，能以较少的试验次数覆盖更广泛的参数范围，且不浪费试验在"不重要"的参数维度上。
- **早停 (Early Stopping)**：在交叉验证中，如果某组参数明显较差，可以提前终止其评估以节省计算资源。

## 数学推导
**网格搜索的计算复杂度**：
设超参数 $p_1, p_2, \dots, p_d$，各有 $v_1, v_2, \dots, v_d$ 个候选值。总试验次数：
$$
N_{\text{grid}} = \prod_{i=1}^d v_i
$$
当 $d=5, v_i=10$ 时，$N = 10^5 = 100000$ 次评估。

**随机搜索的计算预算**：
设定计算预算 $B$ 次试验（如 $B=100$）。假设只有 $r$ 个超参数对结果有显著影响（实际中通常 $r \ll d$）。网格搜索在 $v^d$ 个组合中探索，而随机搜索在 $B$ 次独立采样中覆盖了每个参数的多个取值。

**关键理论结果 (Bergsta & Bengio 2012)**：
对于 $d$ 个超参数中的 $r$ 个重要参数，网格搜索在 $B$ 次试验中只探索了每个重要参数的 $B^{1/d}$ 个值，而随机搜索探索了 $B$ 个**独立**的值。当 $d$ 远大于 $r$ 时，随机搜索的优势更显著。

**超参数响应面的结构**：实际中，超参数空间中通常存在一个"低维有效子空间"——即只有少数几个超参数对性能有显著影响。随机搜索天然地更适应这种结构。

**学习曲线指导**：如果使用 $K$ 折交叉验证评估每组参数，总训练次数为 $N_{\text{trials}} \times K$。对于耗时模型，常用早停或较小验证集来加速。

## 直观理解
- **"地毯式搜索" vs "随机抽查"**：网格搜索像地毯式搜捕——每个可能藏匿的地方都搜查一遍（虽然全面但可能浪费大量时间在无用的角落）。随机搜索像随机抽查——虽然不可能搜遍每个角落，但大概率能覆盖到"犯罪高发区"。
- **为什么随机搜索更高效？**：假设你要调优 5 个参数，但实际只有 2 个参数影响结果。网格搜索以 $10^5$ 次的代价在 5 维网格上搜索，90% 的试验是"无用的维度组合"；随机搜索用 100 次试验就在 2 个重要维度上各探索了 100 个不同的值。覆盖度远好于网格搜索的 10 个值。
- **实践中的折衷**：先做一轮随机搜索（宽范围采样）找到"好"的区域，再做一轮小范围网格搜索（精细调整最优值）。这种"先粗后细"的策略结合了两种方法的优点。

## 代码示例
```python
import numpy as np
from sklearn.model_selection import (GridSearchCV, RandomizedSearchCV,
                                     train_test_split)
from sklearn.ensemble import RandomForestClassifier
from sklearn.datasets import load_digits
from scipy.stats import randint, uniform

# 加载数据
digits = load_digits()
X_train, X_test, y_train, y_test = train_test_split(
    digits.data, digits.target, test_size=0.3, random_state=42)

# 网格搜索（仅尝试 2 个参数以作演示）
param_grid = {
    'n_estimators': [50, 100, 200],
    'max_depth': [5, 10, None],
    'min_samples_split': [2, 5, 10]
}

grid_search = GridSearchCV(
    RandomForestClassifier(random_state=42),
    param_grid, cv=5, n_jobs=-1, verbose=0
)
grid_search.fit(X_train, y_train)

print(f"网格搜索: 共 {len(grid_search.cv_results_['params'])} 组")
print(f"最佳参数: {grid_search.best_params_}")
print(f"最佳 CV 分数: {grid_search.best_score_:.4f}")
print(f"测试分数: {grid_search.score(X_test, y_test):.4f}")

# 随机搜索（相同计算预算用更多参数取值）
param_dist = {
    'n_estimators': randint(20, 300),
    'max_depth': randint(3, 20),
    'min_samples_split': randint(2, 20),
    'max_features': uniform(0.5, 0.5)  # uniform [0.5, 1.0]
}

random_search = RandomizedSearchCV(
    RandomForestClassifier(random_state=42),
    param_dist, n_iter=30, cv=5, n_jobs=-1, verbose=0,
    random_state=42
)
random_search.fit(X_train, y_train)

print(f"\n随机搜索: 共 30 组")
print(f"最佳参数: {random_search.best_params_}")
print(f"最佳 CV 分数: {random_search.best_score_:.4f}")
print(f"测试分数: {random_search.score(X_test, y_test):.4f}")
```

## 深度学习关联
- **深度学习的超参数挑战**：深度模型的超参数空间更大（学习率、batch size、优化器、层数、隐藏单元数、dropout rate、weight decay 等），每次试验耗时也更长（小时到天）。网格搜索基本不可行，随机搜索和贝叶斯优化是主流。
- **学习率调度器作为超参数**：深度模型的学习率调度策略（Step Decay、Cosine Annealing、OneCycle）本身就是超参数空间的重要组成部分。超参数搜索时通常需要联合搜索学习率初始值和调度策略。
- **NAS 作为高级超参数搜索**：神经架构搜索 (NAS) 将超参数搜索推向极致——不仅搜索训练超参数，还搜索网络架构本身（层数、连接方式、激活函数等）。搜索方法也从网格/随机扩展到 RL、进化算法和梯度方法。
