# 大数定律在模型评估中的体现

## 核心概念
- **大数定律 (Law of Large Numbers, LLN)**：随着样本量增加，样本均值依概率收敛于总体均值。弱大数定律：$\bar{X}_n \xrightarrow{p} \mu$；强大数定律：$\bar{X}_n \xrightarrow{a.s.} \mu$。
- **大数定律在模型评估中的应用**：测试集上的平均损失 $\frac{1}{N_{\text{test}}} \sum_{i=1}^{N_{\text{test}}} L(y_i, \hat{y}_i)$ 是模型泛化误差的一致估计。测试集越大，估计越准确。
- **训练误差 vs 测试误差**：训练误差通常低估泛化误差（模型过拟合），但根据大数定律，测试误差在测试集量足够大时是泛化误差的无偏估计。前提是测试集独立于训练集。
- **误差条的来源**：由于测试集有限，评估指标（如准确率、F1）本身是一个随机变量。其标准差为 $\sqrt{\hat{p}(1-\hat{p})/N}$（对准确率），这直接来自大数定律和中心极限定理。
- **交叉验证 (Cross-Validation)**：将数据分割为 K 份，轮流用 K-1 份训练、1 份验证，平均 K 次结果。CV 估计的方差来源包括：训练集变化和验证集变化。

## 数学推导
弱大数定律（Chebyshev）：
设 $X_1, \dots, X_n$ 独立同分布，$\mathbb{E}[X_i] = \mu$，$\text{Var}(X_i) = \sigma^2 < \infty$，则对任意 $\epsilon > 0$：
$$
\lim_{n \to \infty} P(|\bar{X}_n - \mu| > \epsilon) = 0
$$

模型评估中，测试误差 $R_{\text{test}}$ 作为泛化误差 $R$ 的估计：
$$
R_{\text{test}}(h) = \frac{1}{N_{\text{test}}} \sum_{i=1}^{N_{\text{test}}} \mathbf{1}(h(x_i) \neq y_i)
$$

根据 LLN：当 $N_{\text{test}} \to \infty$，$R_{\text{test}} \xrightarrow{p} R(h)$。

准确率的置信区间（近似正态）：
$$
\hat{p} \pm z_{\alpha/2} \sqrt{\frac{\hat{p}(1-\hat{p})}{N_{\text{test}}}}
$$

## 直观理解
- **"投票"的准确性**：大数定律就如同民意调查——调查的人越多，结果越可靠。1000 人的随机样本可以相当准确地反映全国民意，而 10 个人的样本可能偏差很大。
- **测试集大小的重要性**：在模型评估中，测试集就像民意调查的样本量。100 万参数的模型，测试集只有 100 个样本——就像用 100 人的民意调查来预测 3 亿人的选举结果，误差很大。
- **为什么需要独立的测试集**：如果测试集数据参与过训练（直接或间接），则 $R_{\text{test}}$ 不再是 $R$ 的无偏估计。大数定律的前提是"独立同分布"——测试集必须独立于训练集。

## 代码示例
```python
import numpy as np
from sklearn.model_selection import train_test_split, cross_val_score
from sklearn.linear_model import LogisticRegression
from sklearn.datasets import make_classification

# 1. 大数定律的基本演示
np.random.seed(42)

# 从伯努利分布采样（非高斯）
true_p = 0.3
print("大数定律演示:")
for n in [10, 50, 200, 1000, 10000]:
    samples = np.random.binomial(1, true_p, n)
    p_hat = np.mean(samples)
    error = np.abs(p_hat - true_p)
    print(f"  n={n:5d}: p̂={p_hat:.4f}, 误差={error:.4f}")

# 2. 模型评估中的大数定律
X, y = make_classification(n_samples=10000, n_features=20, random_state=42)
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

model = LogisticRegression(max_iter=1000)
model.fit(X_train, y_train)

# 测试不同大小的测试子集
print("\n测试集大小对准确率估计的影响:")
test_sizes = [10, 50, 200, 500, 1000, 3000]
true_test_acc = model.score(X_test, y_test)
print(f"  全测试集 (n={len(X_test)}) 准确率: {true_test_acc:.4f}")

for n in test_sizes:
    accs = []
    for _ in range(500):
        idx = np.random.choice(len(X_test), n, replace=False)
        acc = model.score(X_test[idx], y_test[idx])
        accs.append(acc)
    print(f"  n={n:4d}: 均值={np.mean(accs):.4f}, 标准差={np.std(accs):.4f}")

# 3. 准确率的置信区间
from scipy import stats

n_test = 2000
y_pred = model.predict(X_test[:n_test])
acc = np.mean(y_pred == y_test[:n_test])
se = np.sqrt(acc * (1 - acc) / n_test)
ci = (acc - 1.96*se, acc + 1.96*se)

print(f"\n准确率的 95% 置信区间:")
print(f"  准确率={acc:.4f}, 标准误={se:.4f}")
print(f"  95% CI = [{ci[0]:.4f}, {ci[1]:.4f}]")

# 4. 交叉验证的方差
print(f"\n交叉验证 (5折):")
cv_scores = cross_val_score(model, X, y, cv=5)
print(f"  每折分数: {np.round(cv_scores, 4)}")
print(f"  均值: {np.mean(cv_scores):.4f}, 标准差: {np.std(cv_scores):.4f}")
print(f"  95% CI: [{np.mean(cv_scores)-1.96*np.std(cv_scores)/np.sqrt(5):.4f}, "
      f"{np.mean(cv_scores)+1.96*np.std(cv_scores)/np.sqrt(5):.4f}]")

# 5. 不同测试集大小的置信区间宽度
print("\n测试集大小与置信区间宽度:")
for n in [50, 200, 1000, 5000]:
    se = np.sqrt(0.85 * 0.15 / n)
    width = 2 * 1.96 * se
    print(f"  n={n:5d}: 标准误={se:.4f}, 95% CI宽度={width:.4f}")

# 6. 模拟验证大数定律
print("\n大数定律模拟 (模型评估):")
true_error = 0.15  # 假设真实错误率
for n in [10, 50, 100, 500, 2000]:
    errors = np.random.binomial(n, true_error, 10000) / n
    print(f"  n={n:4d}: 误差均值={np.mean(errors):.4f}, 标准差={np.std(errors):.4f}")
```

## 深度学习关联
- **测试集规模选择**：在大模型时代，基准测试（如 ImageNet、GLUE、MMLU）的测试集规模决定了评估的统计可靠性。当模型性能接近饱和时（如 ImageNet top-1 准确率 > 90%），需要更大的测试集才能检测到微小改进。例如，比较 90.1% 和 90.2% 的准确率差异需要至少数万测试样本才能达到统计显著。
- **评估指标的噪声**：在深度强化学习中，由于环境随机性和策略随机性，单次评估的回报方差很大。常取多次评估的平均值（大数定律）并报告标准差。例如，Atari 游戏通常评估 5-10 次取平均。
- **训练-测试分布偏移**：大数定律要求测试数据与真实数据分布一致。当存在分布偏移（distribution shift）时，测试集误差不再是泛化误差的无偏估计。这提示了域适应和分布外泛化 (OOD) 研究的重要性——即使测试集很大，如果分布不同，评估结果也不能直接反映真实性能。
- **Hold-out 方法与 Bootstrap**：大数定律是 hold-out 验证的数学基础。但在小样本场景中，Bootstrap 方法可以更准确地估计评估指标的置信区间。在深度学习中，Bootstrap 方法不常用（因为训练成本高），但在解释结果稳定性时很有价值。
