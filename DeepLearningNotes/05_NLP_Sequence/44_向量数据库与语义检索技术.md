# 44_向量数据库与语义检索技术

## 核心概念
- **向量数据库 (Vector Database)**：专为存储和检索高维向量（嵌入）而设计的数据库系统。核心功能是通过向量相似性搜索找到与查询向量最相似的数据项。
- **语义检索 (Semantic Retrieval)**：将文本、图像等数据编码为语义向量后，基于向量距离判断语义相似度进行检索。相比关键词检索（BM25），可以捕捉同义词、近义词等语义关系。
- **近似最近邻搜索 (ANN)**：在高维空间中高效找到最近邻的近似算法，精度可接受但速度远超精确搜索。常见算法：HNSW、IVF、PQ。
- **HNSW (Hierarchical Navigable Small World)**：目前最流行的 ANN 算法之一，构建多层图结构，从顶层快速导航到底层。搜索速度快、精度高。
- **IVF (Inverted File)**：将向量空间划分为多个 Voronoi 单元（通过 k-means 聚类），搜索时只查询与查询向量相近的几个单元。
- **向量量化 (Product Quantization, PQ)**：通过分段量化压缩向量表示，大幅减少内存占用（可压缩 8-32 倍）。通常与 IVF 结合使用（IVF+PQ）。
- **主流的向量数据库**：FAISS（Meta 开源库）、Milvus（云原生）、Pinecone（SaaS）、Weaviate、Qdrant、Chroma、Elasticsearch 的向量插件。
- **标量量化 vs 乘积量化**：标量量化将每个浮点数转为 int8（精度损失小），乘积量化将向量分段聚类索引（精度损失大但压缩比高）。

## 数学推导
向量相似度度量（常用余弦相似度或内积）：
$$
\text{sim}(q, d) = \cos(q, d) = \frac{q \cdot d}{\|q\| \cdot \|d\|}
$$

**HNSW 搜索复杂度**：
- 构建复杂度：$O(n \log n)$
- 搜索复杂度：$O(\log n)$
- 内存占用：$O(n \cdot d)$（原始向量 + 图索引）

**IVF 搜索复杂度**：
- 搜索复杂度：$O(n \cdot n_{\text{probes}} / n_{\text{clusters}})$
- 其中 $n_{\text{clusters}}$ 通常为 $\sqrt{n}$，$n_{\text{probes}}$ 为搜索时探测的聚类数

**PQ 压缩率**：设向量维度 $d$，分段数 $m$，每段使用 $k$ 个 centroids。
$$
\text{压缩比} = \frac{d \times 32 \text{ bits}}{m \times \log_2 k}
$$

例如 $d=768$, $m=96$, $k=256$：原 24576 位 -> 768 位，压缩 32 倍。

## 直观理解
- **向量数据库像"语义图书馆"**：传统数据库用"书名"（关键词）找书，向量数据库用"意思"（语义向量）找书。你说"可爱的动物"，它找到"关于猫的书"、"关于狗的书"——即使你根本没提到"猫"或"狗"。
- **近似搜索像"看脸识人"**：精确最近邻搜索就像在人群中和每个人的脸逐一比对（遍历），找出最像的一个。近似搜索（ANN）就像先按身高分群（聚类），只看与查询身高相近的人群——快速但可能漏掉。
- **HNSW 像"多层城市交通图"**：你要从 A 到 B，先上高速（顶层，长距离跳跃），再上主干道（中层），最后到社区小路（底层）。HNSW 的搜索也是——从顶层的"粗粒度"导航到底层的"精确定位"。

## 代码示例
```python
import numpy as np

# 使用 FAISS 进行向量搜索的示例
try:
    import faiss
    FAISS_AVAILABLE = True
except ImportError:
    FAISS_AVAILABLE = False
    print("FAISS 未安装，用 numpy 模拟")

def create_and_search_vectors(dim=128, n_docs=1000, n_query=1):
    # 生成模拟文档向量
    doc_vectors = np.random.randn(n_docs, dim).astype('float32')
    # 归一化（余弦相似度）
    faiss.normalize_L2(doc_vectors)

    query = np.random.randn(n_query, dim).astype('float32')
    faiss.normalize_L2(query)

    # 构建索引方法对比
    methods = {}

    # 1. 暴力搜索（精确，但慢）
    exact_index = faiss.IndexFlatIP(dim)  # Inner Product (余弦相似度)
    exact_index.add(doc_vectors)
    D_exact, I_exact = exact_index.search(query, 5)
    methods['暴力搜索'] = I_exact[0]

    # 2. IVF 索引（近似，但快）
    nlist = 50  # 聚类数
    quantizer = faiss.IndexFlatIP(dim)
    ivf_index = faiss.IndexIVFFlat(quantizer, dim, nlist, faiss.METRIC_INNER_PRODUCT)
    ivf_index.train(doc_vectors)
    ivf_index.add(doc_vectors)
    ivf_index.nprobe = 10  # 搜索时探测的聚类数
    D_ivf, I_ivf = ivf_index.search(query, 5)
    methods['IVF'] = I_ivf[0]

    # 3. HNSW 索引
    hnsw_index = faiss.IndexHNSWFlat(dim, 32)  # 32 个邻居
    hnsw_index.hnsw.efConstruction = 200
    hnsw_index.add(doc_vectors)
    hnsw_index.hnsw.efSearch = 64
    D_hnsw, I_hnsw = hnsw_index.search(query, 5)
    methods['HNSW'] = I_hnsw[0]

    print("不同检索方法的前 5 个结果:")
    for name, results in methods.items():
        print(f"  {name}: {results}")

if FAISS_AVAILABLE:
    create_and_search_vectors()
else:
    # 简易模拟
    doc_vectors = np.random.randn(1000, 128).astype('float32')
    query = np.random.randn(128).astype('float32')
    similarities = np.dot(doc_vectors, query) / (
        np.linalg.norm(doc_vectors, axis=1) * np.linalg.norm(query)
    )
    top5 = np.argsort(similarities)[-5:][::-1]
    print(f"模拟检索 Top 5 索引: {top5}")
    print(f"对应相似度: {similarities[top5]}")
```

## 深度学习关联
- **RAG 的基础设施**：向量数据库是 RAG 系统的核心组件，负责存储和检索文档的语义嵌入。向量数据库的性能直接影响 RAG 系统的检索延迟和精度。
- **Embedding 模型的依赖**：向量检索的质量高度依赖于 embedding 模型的质量。OpenAI 的 text-embedding-3、Sentence-BERT、BGE 等模型的进步直接提升了语义检索效果。
- **千亿级规模的挑战**：在工业级场景（如搜索引擎、推荐系统），向量数据库需要处理百亿级向量和毫秒级延迟，这推动了分布式向量索引、GPU 加速检索、混合搜索（语义 + 关键词）等技术的发展。
