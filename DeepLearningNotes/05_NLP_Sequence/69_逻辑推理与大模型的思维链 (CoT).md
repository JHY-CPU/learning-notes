# 69_逻辑推理与大模型的思维链 (CoT)

## 核心概念
- **思维链 (Chain-of-Thought, CoT)**：由 Wei et al. (2022) 提出，通过在 prompt 中加入中间推理步骤（"让我们一步步思考"），引导 LLM 进行多步推理。CoT 显著提升了大模型在数学、逻辑推理等复杂任务上的表现。
- **Few-shot CoT**：在 prompt 中提供几个示例，每个示例不仅包含输入和输出，还包含中间推理步骤。模型学会遵循示例中的推理模式。
- **Zero-shot CoT**：使用特定触发词（如"Let's think step by step"）激发模型的推理能力，不需要任何示例。简单而强大。
- **CoT 的涌现能力**：CoT 的效果在模型达到一定规模（约 100B 参数）后才显著出现。这种"涌现"表明推理能力可能是大规模语言模型的潜在能力，需要适当的 prompt 来激活。
- **数学推理 (Mathematical Reasoning)**：CoT 在 GSM8K（小学数学）等数学推理基准上带来了最大幅度的提升——从 18% (直接 answer) 提升到 58% (CoT)。
- **多步推理的可靠性**：CoT 虽然提高了推理的正确率，但也引入了"正确的推理路径 + 错误的答案"或"错误的推理路径 + 偶然正确的答案"的风险。每一步的推理都可能累积误差。
- **Self-Consistency (自洽性)**：多次采样推理路径，选出现频率最高的答案。结合 CoT 可以将结果进一步提升。
- **Tree-of-Thought (ToT)**：CoT 的扩展——将线性推理扩展为树状搜索，在每个推理步考虑多个可能的分支并探索。适合需要策略性规划的任务（如博弈）。
- **Least-to-Most Prompting**：先让模型分解问题为子问题，再逐个解决。适合复杂多步推理。

## 数学推导
**CoT 的推理路径**：对于数学问题 $Q$，CoT 生成推理路径 $R = (r_1, \ldots, r_m)$ 和答案 $A$：

$$
P(A, R | Q) = \prod_{t=1}^{m} P(r_t | Q, r_{<t}) \cdot P(A | Q, R)
$$

训练时，标准 LM 目标：
$$
\mathcal{L}_{\text{CoT}} = -\log P(A, R | Q) = -\sum_{t=1}^{m} \log P(r_t | Q, r_{<t}) - \log P(A | Q, R)
$$

**自洽性 (Self-Consistency)**：采样 $k$ 条路径 $\{R^{(1)}, \ldots, R^{(k)}\}$，得到 $k$ 个答案 $\{A^{(1)}, \ldots, A^{(k)}\}$，最终答案取频率最高的：
$$
A^* = \arg\max_A \sum_{i=1}^{k} \mathbb{I}(A^{(i)} = A)
$$

## 直观理解
- **CoT 像"数学题的解题步骤"**：数学老师不会只给你答案——他会写下"第一步：列方程；第二步：化简；第三步：求解……"。CoT 就是让模型也这样一步步推理，而不是直接从问题跳到答案（直接可能会出错）。
- **为什么需要思维链**：大模型在简单推理（一步推理）上表现不错，但在多步推理中容易出错。原因不是它"不会算"，而是"容易跳步"——就像你口算 23×18 容易算错，但列竖式（一步步写下来）就不容易错。
- **Zero-shot CoT 的魔法**："Let's think step by step" 这个简单的短语为什么有效？它相当于触发了模型在训练数据中学到的"推理模式"——模型见过无数次人类在这句话后面跟着推理步骤。这句话就像一个"推理开关"。
- **CoT 的局限**：如果模型的基础知识有误，CoT 无法修正——就像小明认为"1+1=3"，无论他怎么一步步推理，最终结果都是错的。CoT 激发的是推理能力，不是知识。

## 代码示例
```python
# 思维链推理示例

# 标准 Prompt（直接回答）
standard_prompt = """问：Roger 有 5 个网球。他又买了 2 罐网球。每罐有 3 个网球。他现在有多少个网球？
答："""

# CoT Prompt（带推理步骤）
cot_prompt = """问：Roger 有 5 个网球。他又买了 2 罐网球。每罐有 3 个网球。他现在有多少个网球？
答：Roger 开始有 5 个网球。2 罐网球，每罐 3 个，一共是 2×3=6 个。所以他现在有 5+6=11 个。答案是 11。

问：自助餐厅有 23 个苹果。他们用了 20 个做午餐，又买了 6 个。现在有多少个苹果？
答："""

print("标准 Prompt vs CoT Prompt:")
print(f"\n标准 prompt 示例:")
print(f"  问题: Roger 有 5 个网球。他又买了 2 罐网球。每罐有 3 个网球。他现在有多少个网球？")
print(f"  直接回答: 11")
print(f"  (可能正确，但没有推理过程)")

print(f"\nCoT prompt 示例:")
print(f"  问题: Roger 有 5 个网球。他又买了 2 罐网球。每罐有 3 个网球。他现在有多少个网球？")
print(f"  推理: Roger 开始有 5 个网球。2 罐网球，每罐 3 个，一共是 2×3=6 个。所以他现在有 5+6=11 个。")
print(f"  答案: 11")
print(f"  (推理过程可追溯，即使错了也可定位错误步骤)")

# Self-Consistency 示例
print(f"\n\n自洽性 (Self-Consistency) 演示:")
responses = [
    "小明有 10 元，买笔花了 3 元，买本子花了 4 元，还剩 3 元。",
    "小明有 10 元，花了 3+4=7 元，还剩 10-7=3 元。",
    "10-3-4=3，还剩 3 元。",
]
from collections import Counter
answers = [3, 3, 3]  # 提取的数字
most_common = Counter(answers).most_common(1)[0]
print(f"  三次推理的答案: {answers}")
print(f"  自洽性最终答案: {most_common[0]} (出现 {most_common[1]} 次)")
```

## 深度学习关联
- **推理能力的涌现性**：CoT 的效果随模型规模涌现的现象，是 LLM "涌现能力" 的最著名例子之一。小型模型（<10B）从 CoT 中获益有限，大型模型（>100B）从中获得巨大提升。这暗示推理能力是"规模效应"的结果。
- **训练数据中的推理路径**：CoT 的成功反过来影响了模型训练——在预训练或指令微调中加入包含推理路径的数据（如带有详细解题步骤的数学题），可以使模型获得更好的推理能力。数学数据增强（MathQA、MetaMathQA）正是基于这一思路。
- **从 CoT 到 ToT 和 GoT**：CoT 线性的推理路径在复杂任务上可能不够。Tree-of-Thought 将推理扩展为树状搜索（考虑多个分支），Graph-of-Thought 扩展为图结构。这些方向的探索代表了"用更结构化的方式激发模型推理能力"的趋势。
