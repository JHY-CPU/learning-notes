# 53_文本摘要：抽取式与生成式对比

## 核心概念
- **抽取式摘要 (Extractive Summarization)**：从原文中选择最重要的句子，直接拼接成摘要。不生成新文本，只是"摘抄"。关键问题是"句子重要性排序"。
- **生成式摘要 (Abstractive Summarization)**：理解原文内容后用全新的词句生成摘要。可以重新组织语言、合并信息、使用同义表达，更接近人类摘要的方式。
- **抽取式方法**——基于特征：使用句子位置、词频、关键词覆盖、与标题相似度等特征对句子排序。经典算法有 TextRank（基于图的句子排序）。
- **抽取式方法**——基于神经网络：使用 BERT 等模型对每个句子二分类（是否为摘要句）。BERTSUM (Liu, 2019) 在 BERT 上层加 Transformer 编码句子级表示。
- **生成式方法——Seq2Seq 模型**：使用 Encoder-Decoder 架构，编码器理解原文，解码器生成摘要。T5、BART 等预训练模型在此任务上表现优异。
- **RNN vs Transformer 在摘要中的应用**：早期的生成式摘要使用 RNN+注意力，现代方法使用 Transformer（BART、Pegasus、T5）。
- **评估指标**：ROUGE (Recall-Oriented Understudy for Gisting Evaluation) 是最常用的自动评估指标。ROUGE-N 测量 n-gram 重合度，ROUGE-L 测量最长公共子序列。
- **关键挑战**：生成式摘要可能产生事实性错误（幻觉）、重复、重要信息遗漏。抽取式摘要更忠实于原文但不够简洁流畅。

## 数学推导
**ROUGE-N**（基于召回率）：
$$
\text{ROUGE-N} = \frac{\sum_{S \in \text{refs}} \sum_{\text{gram}_n \in S} \text{Count}_{\text{match}}(\text{gram}_n)}{\sum_{S \in \text{refs}} \sum_{\text{gram}_n \in S} \text{Count}(\text{gram}_n)}
$$

与 BLEU 不同：ROUGE 基于召回率（摘要覆盖了多少参考摘要的内容），BLEU 基于精确率（摘要有多少内容在参考中）。

**用于生成式摘要的 Pointer-Generator Network**：
$$
P(w) = p_{\text{gen}} P_{\text{vocab}}(w) + (1 - p_{\text{gen}}) \sum_{i: w_i = w} a_i^t
$$

其中 $p_{\text{gen}}$ 是生成概率（从词汇表生成词），$1 - p_{\text{gen}}$ 是指针概率（从原文复制词）。这结合了生成新词和从原文复制的能力。

## 直观理解
- **抽取式 vs 生成式：摘抄 vs 重写**：抽取式摘要就是考试做笔记——划出最重要的句子，把它们抄在一起。生成式摘要是你理解文章后用自己的话写出总结——更简洁、更流畅，但也更容易"理解错"或"加料"。
- **抽取式的优缺点**：优点是"不会编造原文没有的内容"（忠实度高），缺点是"句子之间可能不连贯"、"可能包含冗余信息"、"不是真正的理解"。
- **生成式的优缺点**：优点是"更接近人类摘要"、"更简洁"、"可以合并多个句子的信息"，缺点是"可能产生幻觉"（说原文没有的内容）、"可能遗漏关键信息"。
- **Pointer-Generator 的巧妙**：它解决了生成式摘要的一个核心矛盾——摘要中的专有名词（人名、地名、数字）必须从原文复制，但其他词可以生成。Pointer 网络决定"是抄还是写"。

## 代码示例
```python
from transformers import pipeline, AutoTokenizer, AutoModelForSeq2SeqLM
import torch

# 使用 BART 进行生成式摘要
summarizer = pipeline("summarization", model="facebook/bart-large-cnn")

article = """
Natural language processing (NLP) is a subfield of linguistics, computer science, 
and artificial intelligence concerned with the interactions between computers and 
human language. The goal is to enable computers to understand, interpret, and 
generate human language in a way that is both meaningful and useful. 
NLP combines computational linguistics with statistical, machine learning, and 
deep learning models. These technologies enable computers to process human language 
in the form of text or voice data and to understand its full meaning, complete 
with the speaker's intent and sentiment.
"""

# 生成式摘要
gen_summary = summarizer(article, max_length=50, min_length=20, do_sample=False)[0]['summary_text']
print("生成式摘要:", gen_summary)

# 模拟抽取式摘要（简单基于句子位置）
def extractive_summary(text, num_sentences=2):
    sentences = text.replace('\n', ' ').split('. ')
    # 简单策略：取前 num_sentences 句（实际中需要基于重要性排序）
    selected = sentences[:num_sentences]
    return '. '.join(selected) + '.'

ext_summary = extractive_summary(article)
print("抽取式摘要:", ext_summary)

# 对比
print("\n--- 对比 ---")
print("生成式摘要特点: 全新生成的词句，更流畅")
print("抽取式摘要特点: 使用原文句子，更忠实")

# 使用 Pegasus 进行摘要（Google 专门为摘要预训练的模型）
pegasus_tokenizer = AutoTokenizer.from_pretrained("google/pegasus-xsum")
pegasus_model = AutoModelForSeq2SeqLM.from_pretrained("google/pegasus-xsum")

inputs = pegasus_tokenizer(article, return_tensors="pt", max_length=512, truncation=True)
with torch.no_grad():
    summary_ids = pegasus_model.generate(inputs.input_ids, max_length=50)
pegasus_summary = pegasus_tokenizer.decode(summary_ids[0], skip_special_tokens=True)
print(f"Pegasus 摘要: {pegasus_summary}")
```

## 深度学习关联
- **预训练模型在摘要中的主导地位**：BART 和 Pegasus 是当前生成式摘要的 SOTA 模型。Pegasus 甚至专门设计了 Gap Sentence Generation（遮盖整句而非 token）作为预训练目标，以更好地适应摘要任务。
- **事实性评估与幻觉检测**：生成式摘要面临的核心挑战是事实一致性。近年来出现了 FactCC、SummaC、QAGS 等自动评估工具，以及与 RAG 结合的幻觉检测方法。
- **长文档摘要**：对于书籍、论文等超长文本，标准 Transformer 的 $O(n^2)$ 复杂度是瓶颈。Longformer、BigBird、SLED 等长文本模型专门解决长文档摘要问题。
