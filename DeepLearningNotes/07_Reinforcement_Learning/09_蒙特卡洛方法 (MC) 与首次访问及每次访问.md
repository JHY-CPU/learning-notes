# 09_蒙特卡洛方法 (MC) 与首次访问及每次访问

## 核心概念

- **蒙特卡洛方法 (Monte Carlo, MC)**：通过完整的 episode 采样来估计价值函数，不依赖环境模型。MC 方法只在 episode 结束后才更新估计值，利用完整的回报 $G_t$ 作为目标值。
- **首次访问 MC (First-visit MC)**：在每个 episode 中，只对状态 $s$ 首次出现的时间步 $t$ 计算回报 $G_t$ 并用于更新 $V(s)$。这是最常用的 MC 变体，偏差更小。
- **每次访问 MC (Every-visit MC)**：在每个 episode 中，对状态 $s$ 出现的每一个时间步都计算回报 $G_t$ 并用于更新 $V(s)$。在访问次数多时计算效率更高，但可能引入偏差。
- **大数定律保证**：当 episode 数量趋于无穷时，MC 估计值收敛到真实价值函数。回报 $G_t$ 是 $V(s)$ 的无偏估计（但方差较大）。
- **增量式更新**：MC 可以使用增量式均值更新：$V(s) \leftarrow V(s) + \alpha(G_t - V(s))$，其中 $\alpha$ 是学习率，可以随时间衰减。
- **探索性初始化 (Exploring Starts)**：为了确保所有状态-动作对被充分探索，MC 控制方法要求每个 episode 从随机状态-动作对开始。这是一种保证收敛但实际中不太实用的假设。

## 数学推导

$$
\text{MC 估计: } V_\pi(s) = \mathbb{E}_\pi[G_t \mid S_t = s] \approx \frac{1}{N(s)} \sum_{i=1}^{N(s)} G_t^{(i)}
$$

$$
\text{首次访问 MC: } V(s) \leftarrow V(s) + \frac{1}{N(s)} \left( G_t - V(s) \right)
$$

$$
\text{每次访问 MC: } V(s) \leftarrow V(s) + \alpha \left( G_t - V(s) \right)
$$

$$
\text{MC 回报: } G_t = \sum_{k=0}^{T-t-1} \gamma^k R_{t+k+1}
$$

$$
\text{MC 方差分析: } \text{Var}(G_t) = \mathbb{E}[(G_t - V_\pi(s))^2] = \text{Var}(R_{t+1}) + \gamma^2 \text{Var}(G_{t+1})
$$

**推导说明**：
- MC 的核心思想是"用样本均值代替期望"：收集大量 episode，对每个状态，取所有访问该状态后的回报均值作为价值估计。
- 首次访问 MC 中，每个 episode 对每个状态最多贡献一次更新，这使得 $G_t^{(i)}$ 之间独立同分布（i.i.d.），统计性质更干净。
- 每次访问 MC 中，同一 episode 中同一状态的多次访问不独立，但偏差随 episode 数量增加而消失。
- MC 方法是无偏的（unbiased），但方差大——因为 $G_t$ 包含 $T-t$ 步随机性的累积。

## 直观理解

蒙特卡洛方法就像"事后诸葛亮"：

想象你在玩一个迷宫游戏，不知道地图（无模型）。你只能
1. **完整走完一次**（跑一个 episode）——从起点到终点，记录经过的每个位置和拿到的分数。
2. **回头看**——走到终点后，你倒推每个位置的"好值"：一个位置的价值 = 从这个位置开始到结束获得的总分数。
3. **多跑几次取平均**——每次完整跑完后更新认知。跑了足够多次后，每个位置的平均价值就逼近真实价值。

**首次访问 vs 每次访问的区别**：
- 好比森林里有块空地，你在一次探险中经过了它 3 次。
- **首次访问**态度：只记录"第一次经过它后获得的分数"，忽略后两次。
- **每次访问**态度：记录"每次经过它后获得的分数"，取所有 3 次的平均。
- 首次访问在大数定理下性质更好（独立同分布），但每次访问在数据利用上更高效。

## 代码示例

```python
import numpy as np
import gymnasium as gym

def mc_policy_evaluate(env, policy, num_episodes=10000, gamma=0.99):
    """首次访问 MC 策略评估"""
    V = np.zeros(env.observation_space.n) if hasattr(env.observation_space, 'n') else None
    returns_sum = np.zeros(env.observation_space.n) if hasattr(env.observation_space, 'n') else {}
    returns_count = np.zeros(env.observation_space.n)
    
    for ep in range(num_episodes):
        # 生成一个 episode
        episode = []
        state, info = env.reset()
        terminated = truncated = False
        while not (terminated or truncated):
            action = np.random.choice(env.action_space.n, p=policy[state])
            next_state, reward, terminated, truncated, info = env.step(action)
            episode.append((state, action, reward))
            state = next_state
        
        # 首次访问 MC：从后向前计算回报
        G = 0
        visited = set()
        for t in reversed(range(len(episode))):
            state, action, reward = episode[t]
            G = reward + gamma * G
            # 首次访问：只处理该状态第一次出现的位置
            if state not in visited:
                visited.add(state)
                returns_sum[state] += G
                returns_count[state] += 1
                V[state] = returns_sum[state] / returns_count[state]
    
    return V

def mc_control_es(env, num_episodes=50000, gamma=0.99):
    """探索性初始化的 MC 控制"""
    nS, nA = env.observation_space.n, env.action_space.n
    Q = np.zeros((nS, nA))
    returns_count = np.zeros((nS, nA))
    policy = np.ones((nS, nA)) / nA
    
    for ep in range(num_episodes):
        # 探索性初始化：随机选择起始状态和动作
        state = np.random.randint(nS)
        action = np.random.randint(nA)
        
        # 生成 episode
        episode = [(state, action, None)]
        env.env.s = state  # hack: 直接设置环境状态
        next_state, reward, terminated, truncated, info = env.step(action)
        episode[-1] = (state, action, reward)
        state = next_state
        
        while not (terminated or truncated):
            action = np.random.choice(nA, p=policy[state])
            next_state, reward, terminated, truncated, info = env.step(action)
            episode.append((state, action, reward))
            state = next_state
        
        # 首次访问 MC 更新 Q
        G = 0
        visited = set()
        for t in reversed(range(len(episode))):
            state, action, reward = episode[t]
            G = reward + gamma * G
            sa_pair = (state, action)
            if sa_pair not in visited:
                visited.add(sa_pair)
                returns_count[state, action] += 1
                Q[state, action] += (G - Q[state, action]) / returns_count[state, action]
                # 策略改进
                best_action = np.argmax(Q[state])
                policy[state] = np.eye(nA)[best_action]
    
    return Q, policy

# 示例使用
env = gym.make("FrozenLake-v1", is_slippery=False)
random_policy = np.ones((env.observation_space.n, env.action_space.n)) / 4
V_mc = mc_policy_evaluate(env, random_policy, num_episodes=5000)
print(f"MC 策略评估下的 V（前 5 个状态）: {V_mc[:5].round(2)}")
env.close()
```

## 深度学习关联

1. **MC 与 DRL 中的经验回放**：MC 方法的"完整 episode 采样"理念在现代 DRL 中演变为经验回放（Experience Replay）——将 episode 片段存入回放缓冲区，从中随机采样更新。MC 的方差问题也是 DQN 引入 TD 方法而非纯 MC 的原因之一。
2. **蒙特卡洛树搜索 (MCTS)**：AlphaGo 和 AlphaZero 使用 MCTS 结合深度网络进行规划。MCTS 本质上是一种"蒙特卡洛方法 + 树搜索"的混合，通过多次模拟（rollout）来评估状态和动作的价值。
3. **Episode-based RL**：在演化策略（Evolution Strategies）和部分基于轨迹的 DRL 算法中，MC 式的"完整轨迹评估"仍然是评估策略性能的标准方法，尤其适用于奖励稀疏或环境动态性强的场景。
