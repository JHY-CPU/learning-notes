# 28 L1 正则化与稀疏解的几何解释

## 核心概念

- **L1 正则化的定义**：L1 正则化（也称 Lasso）在损失函数中添加权重绝对值的和：$\tilde{L} = L + \lambda \|\theta\|_1 = L + \lambda \sum_i |\theta_i|$。其核心特点是能产生稀疏解——使许多权重精确为零。

- **稀疏性**：L1 正则化可以将不重要的特征权重精确推到零，实现自动特征选择。这与 L2 正则化不同——L2 只会将权重缩小到接近零但不会精确为零。

- **几何解释**：L1 正则化的几何解释是在损失函数的等高线图上叠加一个菱形约束区域。最优解出现在等高线与菱形顶点的相切处，而顶点恰好位于坐标轴上，导致某些坐标为零。

- **与 L2 的关键区别**：L1 正则化在零点不可导（|θ| 在 θ=0 处的次梯度为 [-1,1]），这个"尖点"使得优化过程中权重可以被精确推到零。L2 正则化在零点光滑可导，产生的权重接近零但不会等于零。

## 数学推导

**L1 正则化的损失函数**：

$$
\tilde{L}(\theta) = L(\theta) + \lambda \sum_{i=1}^{d} |\theta_i|
$$

**次梯度（Subgradient）**：

由于 $|\theta_i|$ 在 $\theta_i = 0$ 处不可导，需要使用次梯度：

$$
\partial |\theta_i| = \begin{cases}
\text{sign}(\theta_i) & \text{if } \theta_i \neq 0 \\
[-1, 1] & \text{if } \theta_i = 0
\end{cases}
$$

**SGD 更新（近端梯度视角）**：

对非零权重，更新规则为：

$$
\theta_i \leftarrow \theta_i - \eta \frac{\partial L}{\partial \theta_i} - \eta \lambda \cdot \text{sign}(\theta_i)
$$

这相当于先沿损失梯度下降，然后向零收缩。

**软阈值算子（Soft Thresholding）**：

L1 正则化的近端算子可以闭式求解：

$$
\text{prox}_{\eta\lambda\|\cdot\|_1}(\theta_i) = \text{sign}(\theta_i) \cdot \max(|\theta_i| - \eta\lambda, 0)
$$

这被称为软阈值算子：如果 $|\theta_i| < \eta\lambda$，直接设为 0；否则向零收缩 $\eta\lambda$。

**稀疏解的几何分析**：

考虑简单的线性回归 $\min \|X\theta - y\|^2 + \lambda \|\theta\|_1$。

KKT 条件为：

$$
2X^T(X\theta - y) + \lambda \cdot \text{sign}(\theta) = 0
$$

当参数 $\theta_j = 0$ 时，其条件为：

$$
|2X_j^T(X\theta - y)| \leq \lambda
$$

即当特征 $j$ 与残差的相关性小于 $\lambda$ 时，该特征被排除（权重为零）。

## 直观理解

L1 正则化的几何解释是最直观的：

- **L2 正则化**：约束区域是一个圆（$\theta_1^2 + \theta_2^2 \leq r^2$）。圆是光滑的凸形状，与损失函数等高线的切点通常在非坐标轴位置，因此权重一般不全为零。

- **L1 正则化**：约束区域是一个菱形（$|\theta_1| + |\theta_2| \leq r$）。菱形在坐标轴上有"尖点"（顶点），当等高线与尖点相切时，解就落在坐标轴上——这意味着另一个坐标为 0。

随着 $\lambda$ 增大，菱形区域缩小，解更容易落在坐标轴上，稀疏性更强。当 $\lambda$ 足够大时，所有权重都被推到零。

这个原理可以扩展到高维：L1 球在高维空间中是"星形"的，尖点更多，解落在坐标轴超平面上的概率更大，因此更容易产生稀疏解。

从贝叶斯角度看，L1 正则化等价于在权重上施加拉普拉斯先验 $p(\theta) \propto e^{-\lambda |\theta|}$。拉普拉斯分布在零点有尖峰，这反映了先验认为很多权重应该精确为零的信念。

## 代码示例

```python
import torch
import torch.nn as nn
import torch.optim as optim

# L1 正则化的手动实现
def l1_regularization(model, lambda_l1):
    """计算模型参数的 L1 正则化项"""
    return lambda_l1 * sum(p.abs().sum() for p in model.parameters())

# 演示 L1 vs L2 在稀疏性上的差异
torch.manual_seed(42)

# 生成稀疏数据: 只有 3 个特征真正有效
n_features, n_samples = 50, 200
X = torch.randn(n_samples, n_features)
true_weights = torch.zeros(n_features)
true_weights[:3] = torch.tensor([3.0, -2.0, 1.5])  # 只有前 3 个特征有效
y = X @ true_weights + torch.randn(n_samples) * 0.5

# 训练 L1 和 L2 正则化线性回归
def train_with_reg(reg_type, lambda_val=0.01):
    model = nn.Linear(n_features, 1, bias=False)
    optimizer = optim.SGD(model.parameters(), lr=0.001)

    for epoch in range(1000):
        pred = model(X).squeeze()
        mse = ((pred - y) ** 2).mean()
        if reg_type == 'L1':
            loss = mse + l1_regularization(model, lambda_val)
        elif reg_type == 'L2':
            loss = mse + lambda_val/2 * sum(p.pow(2).sum() for p in model.parameters())
        else:
            loss = mse

        optimizer.zero_grad()
        loss.backward()
        optimizer.step()

    return model.weight.data.squeeze()

w_no_reg = train_with_reg('none')
w_l2 = train_with_reg('L2', 0.01)
w_l1 = train_with_reg('L1', 0.01)

# 统计接近零的参数数量
def count_zeros(w, threshold=1e-3):
    return (w.abs() < threshold).sum().item()

print("L1 vs L2 稀疏性对比:")
print(f"  真实非零权重数: 3")
print(f"  无正则化: 非零={count_zeros(w_no_reg, 1e-3)}/50")
print(f"  L2 正则化: 非零={count_zeros(w_l2, 1e-3)}/50")
print(f"  L1 正则化: 非零={count_zeros(w_l1, 1e-3)}/50")

print(f"\n前 10 个权重 (L1): {w_l1[:10]}")
print(f"前 10 个权重 (L2): {w_l2[:10]}")

# 不同 λ 对稀疏度的影响
print("\nλ 对稀疏度的影响:")
for lam in [0.001, 0.01, 0.1, 1.0]:
    w = train_with_reg('L1', lam)
    zeros = count_zeros(w, 1e-3)
    nonzero = 50 - zeros
    print(f"  λ={lam:.3f}: 非零参数={nonzero}, 非零权重范数={w[w.abs()>1e-3].norm():.4f}")

# PyTorch 中使用 L1 正则化的推荐方法
print("\nPyTorch 中 L1 正则化的推荐实现方式:")
model = nn.Linear(10, 1)
optimizer = optim.SGD(model.parameters(), lr=0.01, weight_decay=0)  # weight_decay 是 L2
lambda_l1 = 0.001

def training_step(x, y):
    pred = model(x)
    loss = ((pred - y) ** 2).mean()
    # 手动添加 L1 正则
    l1_loss = lambda_l1 * sum(p.abs().sum() for p in model.parameters())
    total_loss = loss + l1_loss
    optimizer.zero_grad()
    total_loss.backward()
    optimizer.step()
    return total_loss.item()
```

## 深度学习关联

- **特征选择**：L1 正则化在需要特征选择的高维问题中特别有用（如基因表达数据、文本分类）。通过将不重要特征的权重推到零，L1 正则化自动实现了特征选择，使模型更可解释。

- **稀疏神经网络**：L1 正则化被用于网络剪枝（Network Pruning）中。通过训练时施加 L1 正则化，许多神经元的权重被推到零，这些神经元可以在推理时被移除，从而压缩模型大小和加速推理。

- **与 L2 的组合使用（Elastic Net）**：Elastic Net 结合了 L1 和 L2 正则化：$\lambda_1\|\theta\|_1 + \lambda_2\|\theta\|_2^2$。它既保留了 L1 的特征选择能力（稀疏性），又具有 L2 的组效应（group effect，相关特征的权重趋向一致），在高维特征场景下表现优异。
