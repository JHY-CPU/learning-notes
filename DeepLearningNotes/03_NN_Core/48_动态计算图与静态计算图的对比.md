# 48 动态计算图与静态计算图的对比

## 核心概念

- **动态计算图**：动态计算图（Dynamic Graph, Define-by-Run）在每次前向传播时按需构建。PyTorch 是动态图的代表——代码执行到哪个操作，图就构建到哪个节点。图结构随每次前向传播可能变化（如变长序列、条件分支）。

- **静态计算图**：静态计算图（Static Graph, Define-and-Run）先定义完整的图结构，然后执行。TensorFlow 1.x 是静态图的代表——先用符号 API 定义整个计算图，然后在 Session 中重复执行。

- **动态图的优势**：灵活性高、调试方便（可以使用标准 Python 控制流）、代码更接近数学表达式。特别适合动态结构（如 RNN 中变长序列、条件计算）和研究者快速迭代。

- **静态图的优势**：性能优化空间大（图优化、算子融合、内存复用）、便于部署（可以序列化图为独立格式）、分布式执行更高效。适合生产环境和大规模部署。

## 详细对比

**控制流处理**：

动态图：
```python
# 直接使用 Python 控制流
if condition:
    y = model_a(x)
else:
    y = model_b(x)
```

静态图（TensorFlow 1.x）：
```python
# 使用 tf.cond 构建条件分支
y = tf.cond(condition, lambda: model_a(x), lambda: model_b(x))
```

**循环处理**：

动态图：
```python
# 直接使用 Python 循环
for t in range(seq_len):
    h = rnn_cell(x[t], h)
```

静态图：
```python
# 使用 tf.while_loop 构建循环
h = tf.while_loop(cond, body, [h])
```

## 数学推导（性能差异的理论基础）

**自动微分效率**：

动态图：前向传播时构建计算图，反向传播时沿图回溯。中间结果需要缓存。

静态图：编译时进行图优化，包括：
1. **节点融合**：将连续的多个小操作合并为一个大操作（如将 Conv-BN-ReLU 融合）
2. **内存规划**：静态分析确定中间结果的生存期，优化内存复用
3. **并行调度**：识别图中可并行执行的部分

**梯度计算的复杂度**：

动态图和静态图的梯度计算都是 $O(V + E)$（$V$ 是节点数，$E$ 是边数），但静态图的常数项更小（由于编译优化）。

**JIT 编译的桥梁**：

现代框架试图结合两者的优点：
- PyTorch 的 TorchScript / `torch.jit.trace`
- TensorFlow 的 Eager Execution + Autograph
- JAX 的 `jit` 编译

这些技术允许先用动态图开发，然后通过追踪（tracing）或源码转换（AST transformation）转换为静态图。

## 直观理解

动态图 vs 静态图的对比类似于"口头指令 vs 蓝图施工"：

- **动态图**：你一边说，对方一边做。灵活——可以边走边想，改变主意（条件分支）也很容易。但是没法提前规划最优路径（图优化）。

- **静态图**：你先画好完整的施工蓝图（定义计算图），然后工人严格按照蓝图施工（执行图）。可以提前优化——合并步骤、规划材料（内存）使用。但中途改变主意代价很大。

编程体验的差异：

动态图让深度学习编程回归到"面向对象编程"和"函数式编程"的直觉方式。调试时可以插入 `print` 语句、设置断点、检查中间值。静态图更像"声明式编程"——你描述计算关系，框架决定如何执行。

## 代码示例

```python
import torch
import torch.nn as nn

# === 动态计算图示例（PyTorch 默认）===
print("=== 动态计算图示例（PyTorch）===")

class DynamicNet(nn.Module):
    """动态图网络：根据输入长度决定网络深度"""
    def __init__(self):
        super().__init__()
        self.fc1 = nn.Linear(10, 20)
        self.fc2 = nn.Linear(20, 20)
        self.fc3 = nn.Linear(20, 1)

    def forward(self, x, depth):
        h = torch.relu(self.fc1(x))
        # 动态的循环深度
        for i in range(depth):
            h = torch.relu(self.fc2(h))
        return self.fc3(h)

dynamic_net = DynamicNet()
x = torch.randn(2, 10)

# 每次前向传播可以有不同的控制流
out1 = dynamic_net(x, depth=2)  # 浅层
out2 = dynamic_net(x, depth=5)  # 深层
print(f"depth=2 输出: {out1}")
print(f"depth=5 输出: {out2}")

# 动态图支持条件分支
class ConditionalNet(nn.Module):
    def forward(self, x, use_extra_layer):
        h = torch.relu(x)
        if use_extra_layer:
            h = torch.relu(h + 1)
        return h

cond_net = ConditionalNet()
out_true = cond_net(torch.tensor([1.0]), True)
out_false = cond_net(torch.tensor([1.0]), False)
print(f"\n条件分支: True={out_true.item()}, False={out_false.item()}")

# === 静态计算图模拟 ===
print("\n=== 静态计算图（通过 TorchScript 追踪）===")

# 将动态图追踪为静态图
traced_net = torch.jit.trace(dynamic_net, (x, torch.tensor(2)))

# 查看追踪后的图结构
print(f"追踪后的图:\n{traced_net.graph}")

# TorchScript 脚本化（支持控制流）
@torch.jit.script
def script_fn(x, use_bias: bool):
    """TorchScript 脚本化函数"""
    result = x.sum()
    if use_bias:
        result = result + 1.0
    return result

script_out = script_fn(torch.randn(5), True)
print(f"Script 函数输出: {script_out.item():.4f}")

# === TensorFlow 2.x 的自动图构建 ===
# TensorFlow 2.x 使用 Eager Execution 默认动态图，
# 通过 @tf.function 转换为静态图
# 这里使用 PyTorch 的 JIT 编译类比

# JIT 编译：将动态图函数转换为静态图
@torch.jit.script
def jit_linear(x, weight, bias):
    return torch.relu(x @ weight.T + bias)

weight = torch.randn(5, 10)
bias = torch.randn(5)
inp = torch.randn(3, 10)
out = jit_linear(inp, weight, bias)
print(f"\nJIT 编译函数输出 shape: {out.shape}")

# === 性能对比 ===
print("\n=== 性能对比（动态 vs TorchScript）===")
import time

class LargeNet(nn.Module):
    def __init__(self, n_layers=100):
        super().__init__()
        self.layers = nn.ModuleList([
            nn.Linear(256, 256) for _ in range(n_layers)
        ])

    def forward(self, x):
        for layer in self.layers:
            x = torch.relu(layer(x))
        return x

net = LargeNet(100)
x = torch.randn(32, 256)

# 动态图执行
start = time.time()
for _ in range(20):
    out = net(x)
dyn_time = time.time() - start

# TorchScript 执行
scripted_net = torch.jit.script(net)
start = time.time()
for _ in range(20):
    out = scripted_net(x)
static_time = time.time() - start

print(f"动态图: {dyn_time:.4f}s")
print(f"TorchScript: {static_time:.4f}s")
if static_time > 0:
    print(f"加速比: {dyn_time/static_time:.2f}x")
```

## 深度学习关联

- **PyTorch vs TensorFlow 的范式之争**：动态图 vs 静态图的对比是 PyTorch 和 TensorFlow 在早期（2017-2019 年）的核心区别。PyTorch 的动态图更受研究者欢迎，而 TensorFlow 的静态图在部署方面有优势。当前趋势是两者趋同——PyTorch 引入了 TorchScript，TensorFlow 2.x 默认使用动态图并通过 `@tf.function` 支持静态编译。

- **JAX 的新范式**：JAX 采用函数式编程 + JIT 编译的模式。用户编写纯函数，通过 `jit`、`vmap`、`pmap` 等变换器将函数编译为静态图。这兼具了动态图（Python 原生）和静态图（高效执行）的优点。

- **编译器技术的引入**：随着深度学习发展，编译器技术在框架中越来越重要。PyTorch 的 `torch.compile`（基于 Triton）、TensorFlow 的 XLA、JAX 的 JIT 都使用即时编译（JIT Compilation）技术，在 Python 动态代码的基础上自动生成高效的静态计算图。
