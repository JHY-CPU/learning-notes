# 03 前向传播的计算图表示

## 核心概念

- **计算图定义**：计算图（Computational Graph）是一种有向无环图（DAG），其中节点表示变量或运算，边表示数据依赖关系。神经网络的前向传播可以自然地表示为计算图，每个节点接受输入并产生输出，数据沿边向前流动。

- **节点类型**：计算图中的节点分为三类：叶节点（输入变量或参数，如权重 $W$、偏置 $b$）、中间节点（运算结果，如 $z = Wx + b$）、输出节点（最终损失值 $L$）。叶节点通常需要梯度，中间节点的梯度由链式法则自动计算。

- **动态图与静态图**：动态计算图（PyTorch 风格）在每次前向传播时按需构建，图结构随代码执行动态变化，灵活但优化空间有限。静态计算图（TensorFlow 1.x 风格）先定义后执行，图结构固定，便于编译优化和部署，但灵活性较差。现代框架倾向于支持两者或使用即时编译（JIT）桥接。

- **自动微分的基础**：计算图是自动微分（Automatic Differentiation）的核心数据结构。在前向传播过程中，计算图同时记录了运算路径和中间结果——这些信息在反向传播中用于高效地计算梯度。每一个前向计算节点都对应着一个反向传播的梯度计算规则。

## 数学推导

以一个简单的两层 MLP 为例：

$$
z_1 = W_1 x + b_1
$$
$$
a_1 = \sigma(z_1)
$$
$$
z_2 = W_2 a_1 + b_2
$$
$$
\hat{y} = z_2
$$
$$
L = \frac{1}{2}(\hat{y} - y)^2
$$

对应的计算图结构如下：

```
x ──┐
     ├── [MatMul + Add] ── z1 ── [Sigmoid] ── a1 ──┐
W1 ──┘                                               ├── [MatMul + Add] ── z2 ── [MSE] ── L
b1 ──────────────────────────────────────┐          │
                                         │    W2 ───┘
                                         └──  b2 ───┘
```

在前向传播中，我们可以用计算图来追踪每一步的中间结果：

```python
# 计算图节点记录
x → z1 = x @ W1.T + b1     # 线性变换
z1 → a1 = σ(z1)            # 激活函数
a1 → z2 = a2 @ W2.T + b2   # 线性变换
z2 → L = 0.5 * (z2 - y)**2 # 损失计算
```

每个运算节点在计算图中定义了两个操作：前向计算（forward）和反向梯度计算（backward）。例如，矩阵乘法节点 $C = A \times B$ 的前向是矩阵相乘，反向则是 $\partial L/\partial A = (\partial L/\partial C) \times B^T$ 和 $\partial L/\partial B = A^T \times (\partial L/\partial C)$。

## 直观理解

计算图可以想象为一个"数据流水线"：原始数据 $x$ 从左边进入，经过一系列处理站（运算节点），每个站对数据施加一些变换，最终到达输出端产生损失值。每个处理站都知道如何做两件事：向前处理数据，以及向后传递"纠错信号"（梯度）。

计算图的美妙之处在于**模块化**：每个运算节点独立实现自己的前向和反向逻辑，然后像搭积木一样组合起来。这类似于工厂流水线中的独立工作站——每个站只需要知道如何处理进入自己工位的零件，以及如何将质量反馈传递给前一站。

计算图也揭示了梯度计算的核心洞察：**局部梯度**。每个节点只需要计算其输出相对于其输入的梯度（局部梯度），然后通过链式法则将上游传来的梯度与局部梯度相乘，得到向下游传播的梯度。这种局部性使得深度学习框架可以轻松扩展到任意复杂的网络结构。

## 代码示例

```python
import torch

# PyTorch 自动构建动态计算图

# 输入和参数
x = torch.tensor([1.0, 2.0], requires_grad=False)
W = torch.tensor([[0.1, 0.2], [0.3, 0.4]], requires_grad=True)
b = torch.tensor([0.5, 0.6], requires_grad=True)
y_true = torch.tensor([1.0])

# 前向传播 — PyTorch 自动构建计算图
z1 = x @ W.T + b       # 线性层
a1 = torch.sigmoid(z1) # 激活函数
z2 = a1.sum()          # 输出层（简化为标量）
loss = (z2 - y_true)**2  # MSE 损失

# 查看计算图结构
print(f"z1: {z1}, requires_grad={z1.requires_grad}")
print(f"a1: {a1}, requires_grad={a1.requires_grad}")
print(f"loss: {loss.item():.4f}")

# 反向传播 — 自动沿计算图反向计算梯度
loss.backward()

# 查看梯度
print(f"W grad:\n{W.grad}")
print(f"b grad: {b.grad}")

# 通过 grad_fn 追溯计算图结构
print(f"loss.grad_fn: {loss.grad_fn}")
print(f"z2.grad_fn: {z2.grad_fn}")
print(f"a1.grad_fn: {a1.grad_fn}")
print(f"z1.grad_fn: {z1.grad_fn}")
```

## 深度学习关联

- **自动微分引擎**：所有现代深度学习框架（PyTorch、TensorFlow、JAX）的核心都是基于计算图的自动微分引擎。PyTorch 的 Autograd 系统在每次前向传播时动态构建计算图，记录所有运算操作，然后在反向传播时按拓扑序逆序遍历计算图，依次计算梯度。

- **图优化与编译**：计算图表示使编译器能够进行各种优化，包括算子融合（将多个连续运算合并为一个）、内存复用、并行执行等。TorchScript、TensorFlow XLA、JAX JIT 等技术的本质都是将 Python 级别的计算图捕获为中间表示，然后进行编译优化。

- **可微分编程**：计算图的概念已经超越了神经网络，延伸到可微分编程（Differentiable Programming）领域。通过构建任意程序的计算图，我们可以对程序中的参数求导，这使得在物理仿真、优化问题、科学计算等领域应用梯度下降成为可能。
