# 宏系统

## 宏的基本概念

宏是 Scheme 元编程能力的核心。宏在**编译时**对代码进行变换，而非在运行时执行。这使得程序员可以扩展语言本身，创建新的语法形式。

宏与函数的关键区别：

- **函数**在运行时求值其参数，然后执行函数体
- **宏**在编译时接收**未求值的语法形式**，返回变换后的代码

```scheme
;; 宏 vs 函数的对比
;; 函数：参数在传入前被求值
(define (my-if-func test then else)
  (if test then else))

(my-if-func #t "yes" "no")  ; => "yes"
;; 但 (my-if-func #t "yes" (error "boom"))
;; 会先求值 (error "boom")，导致错误

;; 宏：参数以语法对象的形式传入，延迟求值
(define-syntax-rule (my-if-macro test then else)
  (if test then else))

(my-if-macro #t "yes" (error "boom"))  ; => "yes"
;; (error "boom") 不会被求值，因为宏展开为 (if #t "yes" (error "boom"))
;; 而 if 只求值被选中的分支
```

## define-syntax 与 syntax-rules

`syntax-rules` 是 Scheme 中定义宏的主要方式，通过模式匹配和模板替换工作。

### 基本语法

```scheme
(define-syntax 宏名
  (syntax-rules (字面量...)
    [模式1 模板1]
    [模式2 模板2]
    ...))
```

### 简单宏示例

```scheme
;; when 宏：当条件为真时执行多条语句
(define-syntax when
  (syntax-rules ()
    [(when test expr ...)
     (if test
         (begin expr ...))]))

(when (> x 0)
  (display "x is positive\n")
  (display x))

;; unless 宏：当条件为假时执行多条语句
(define-syntax unless
  (syntax-rules ()
    [(unless test expr ...)
     (when (not test) expr ...)]))

;; swap 宏：交换两个变量的值
(define-syntax swap!
  (syntax-rules ()
    [(swap! a b)
     (let ([tmp a])
       (set! a b)
       (set! b tmp))]))

(let ([x 1] [y 2])
  (swap! x y)
  (list x y))
; => (2 1)
```

### define-syntax-rule

`define-syntax-rule` 是 `define-syntax` + `syntax-rules` 的简写形式：

```scheme
;; define-syntax-rule 适用于只有一个模式的简单宏
(define-syntax-rule (inc! var)
  (set! var (+ var 1)))

(let ([x 5])
  (inc! x)
  x)
; => 6

;; 等价于:
(define-syntax inc!
  (syntax-rules ()
    [(inc! var)
     (set! var (+ var 1))]))
```

## 模式匹配

`syntax-rules` 的模式匹配支持以下特性：

### 字面量匹配

```scheme
;; 字面量（literal）必须完全匹配
(define-syntax my-cond
  (syntax-rules (else)      ; else 是字面量
    [(my-cond [else expr])
     expr]
    [(my-cond [test expr] rest ...)
     (if test
         expr
         (my-cond rest ...))]))

(my-cond
  [(< 3 2) "不可能"]
  [(= 3 3) "正确"]       ; => "正确"
  [else "默认"])
```

### 省略号模式（Ellipsis Pattern）

省略号 `...` 表示零次或多次重复：

```scheme
;; let 宏的实现
(define-syntax my-let
  (syntax-rules ()
    [(my-let ([var val] ...) body ...)
     ((lambda (var ...) body ...) val ...)]))

(my-let ([x 10] [y 20])
  (+ x y))
;; 展开为: ((lambda (x y) (+ x y)) 10 20)
;; => 30

;; begin 宏（多个表达式）
(define-syntax my-begin
  (syntax-rules ()
    [(my-begin expr ...)
     ((lambda () expr ...))]))

;; and 宏（短路求值）
(define-syntax my-and
  (syntax-rules ()
    [(my-and) #t]
    [(my-and e) e]
    [(my-and e1 e2 ...)
     (if e1 (my-and e2 ...) #f)]))
```

### 嵌套模式

```scheme
;; cond 宏（更完整的实现）
(define-syntax my-cond
  (syntax-rules (else =>)
    [(my-cond)
     (void)]
    [(my-cond [else result ...])
     (begin result ...)]
    [(my-cond [test => func] rest ...)
     (let ([t test])
       (if t
           (func t)
           (my-cond rest ...)))]
    [(my-cond [test result ...] rest ...)
     (if test
         (begin result ...)
         (my-cond rest ...))]))
```

## 卫生宏（Hygienic Macros）

Scheme 的宏是**卫生的**（hygienic），这意味着宏展开不会意外捕获或遮蔽用户代码中的变量。这是 Scheme 宏系统最重要的特性之一。

```scheme
;; 卫生宏保护用户变量不被宏内部的变量遮蔽

;; 一个"不卫生"的宏（概念性示例）
;; 如果 Scheme 不支持卫生宏:
;; (define-syntax broken-when
;;   (syntax-rules ()
;;     [(broken-when test body ...)
;;      (if test (begin body ...))]))
;;
;; (let ([if 42])
;;   (broken-when #t if))  ; 用户期望返回 42
;; 如果不卫生，宏内部的 if 会遮蔽用户的 if 变量

;; Scheme 的卫生宏保证:
(define-syntax-rule (my-when test body ...)
  (if test (begin body ...)))

(let ([if 42])
  (my-when #t if))  ; => 42，卫生宏不会破坏用户的 if 变量

;; 卫生宏的另一个经典例子：交换宏
(define-syntax-rule (swap! a b)
  (let ([tmp a])     ; 这里的 tmp 不会与用户的 tmp 冲突
    (set! a b)
    (set! b tmp)))

(let ([tmp 1] [b 2])
  (swap! tmp b)
  (list tmp b))      ; => (2 1)，即使用户有名为 tmp 的变量
```

## 更高级的宏技术

### 递归宏

```scheme
;; and 宏的递归实现
(define-syntax my-and
  (syntax-rules ()
    [(my-and) #t]
    [(my-and e) e]
    [(my-and e1 e2 ...)
     (if e1 (my-and e2 ...) #f)]))

;; 展开过程:
;; (my-and #t #f 3)
;; => (if #t (my-and #f 3) #f)
;; => (if #t (if #f (my-and 3) #f) #f)
;; => (if #t (if #f 3 #f) #f)
;; => (if #t #f #f)
;; => #f

;; or 宏
(define-syntax my-or
  (syntax-rules ()
    [(my-or) #f]
    [(my-or e) e]
    [(my-or e1 e2 ...)
     (let ([t e1])
       (if t t (my-or e2 ...)))]))
```

### 定义新控制结构

```scheme
;; do-times 循环宏
(define-syntax do-times
  (syntax-rules ()
    [(do-times (var count) body ...)
     (let ([n count])
       (let loop ([var 0])
         (when (< var n)
           body ...
           (loop (add1 var)))))]))

(do-times (i 5)
  (printf "迭代 ~a~n" i))
;; 输出: 迭代 0, 迭代 1, 迭代 2, 迭代 3, 迭代 4

;; for/list 推导宏
(define-syntax-rule (my-for/list ([var seq]) body ...)
  (map (lambda (var) body ...) seq))

(my-for/list ([x '(1 2 3 4 5)])
  (* x x))
; => (1 4 9 16 25)

;; with-logging 宏
(define-syntax-rule (with-logging name expr ...)
  (begin
    (printf "[进入] ~a~n" 'name)
    (let ([result (begin expr ...)])
      (printf "[退出] ~a => ~a~n" 'name result)
      result)))

(with-logging "计算"
  (+ 1 2 3))
;; [进入] 计算
;; [退出] 计算 => 6
;; => 6
```

### 宏的宏组合

```scheme
;; 定义一个创建"断言宏"的宏
(define-syntax define-assertion
  (syntax-rules ()
    [(define-assertion name pred?)
     (define-syntax-rule (name expr msg)
       (unless (pred? expr)
         (error (format "~a: ~a" 'name msg))))]))

(define-assertion assert-positive positive?)
(assert-positive 5 "必须为正数")   ; 通过
;; (assert-positive -1 "必须为正数")  ; 错误: assert-positive: 必须为正数
```

## syntax-rules 的局限与 syntax-case

`syntax-rules` 虽然简洁且卫生，但在某些场景下表达力不足。Racket 引入了 `syntax-case`，提供了更强大的宏编写能力：

```scheme
;; syntax-case 示例（Racket）
(define-syntax (my-if stx)
  (syntax-case stx ()
    [(my-if test then else)
     #'(if test then else)]
    [(my-if test then)
     #'(if test then)]))

;; syntax-case 允许在宏展开时执行任意 Scheme 代码
(define-syntax (define-logged stx)
  (syntax-case stx ()
    [(define-logged (name args ...) body ...)
     (with-syntax ([proc-name (symbol->string (syntax-e #'name))])
       #'(define (name args ...)
           (printf "调用 ~a~n" proc-name)
           body ...))]))

(define-logged (add x y)
  (+ x y))

(add 3 4)  ; 输出: 调用 add => 7
```
