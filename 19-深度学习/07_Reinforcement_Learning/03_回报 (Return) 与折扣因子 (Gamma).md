# 03_回报 (Return) 与折扣因子 (Gamma)

## 核心概念

- **回报 (Return)**：从当前时刻 $t$ 开始，智能体在未来获得的所有折扣奖励的总和，是强化学习中优化的终极目标。记作 $G_t = \sum_{k=0}^\infty \gamma^k R_{t+k+1}$。
- **折扣因子 (Discount Factor)** $\gamma \in [0,1]$：衡量未来奖励相对于即时奖励的重要程度。$\gamma$ 越接近 1，智能体越看重长远收益；越接近 0，智能体越短视。
- **有限与无限视界**：有限视界下回报是固定步数的奖励和；无限视界（持续任务）下使用折扣回报确保总和收敛。回合制任务（如围棋、游戏关卡）通常使用 $\gamma=0.99$ 或 $\gamma=1$。
- **折扣的必要性**：数学上保证无限步序列的回报不发散（只要奖励有界）；经济上体现"今天的钱比明天的钱更值钱"；算法上使近期的奖励对学习贡献更大，提高学习效率。
- **未折扣回报**：当 $\gamma=1$ 时 $G_t$ 可能发散（无限持续任务），此时需要借助平均奖励设置 (average reward setting) 来定义优化目标。
- **回报的递推关系**：$G_t = R_{t+1} + \gamma G_{t+1}$，这是所有 TD 方法和贝尔曼方程的基础递推结构，将多步回报分解为即时奖励加未来回报的折扣和。

## 数学推导

$$
\text{折扣回报定义: } G_t = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1}
$$

$$
\text{递推形式: } G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + ... 
= R_{t+1} + \gamma (R_{t+2} + \gamma R_{t+3} + ...) 
= R_{t+1} + \gamma G_{t+1}
$$

$$
\text{有限视界（T 步）: } G_t = \sum_{k=0}^{T-t-1} \gamma^k R_{t+k+1}
$$

$$
\text{等比级数收敛（|R| \le R_{max}）: } G_t \le R_{max} \sum_{k=0}^\infty \gamma^k = \frac{R_{max}}{1 - \gamma}
$$

**推导说明**：
- 递推形式 $G_t = R_{t+1} + \gamma G_{t+1}$ 揭示了回报的马尔可夫结构——当前回报可以仅从一步即时奖励和下一步回报计算得出。
- 当 $0 \le \gamma < 1$ 且 $|R_t| \le R_{max}$ 时，回报的绝对值有上界 $\frac{R_{max}}{1-\gamma}$，确保了数学上的良定义。
- 不同任务选择 $\gamma$ 的直觉：玩一局游戏（回合制）可用 $\gamma=0.99$ 考虑长远；股票交易（短周期）可用 $\gamma=0.1$ 侧重近期收益。

## 直观理解

折扣因子 $\gamma$ 就像银行利率的逆运算：
- 如果把未来奖励想象成未来的钱，$\gamma=0.9$ 意味着"一年后的 100 元相当于现在的 90 元"。这个类比非常贴切——人们天生倾向于"今朝有酒今朝醉"（动物性），但理性告诉我们要为长远考虑。
- $\gamma=0$ 时，智能体完全活在当下：只看眼前的奖励，不考虑未来后果。这就像只吃最后一顿饭的人，不考虑明天。
- $\gamma=0.99$ 时，智能体深谋远虑：未来 100 步后的奖励依然有 $0.99^{100} \approx 0.366$ 的权重。这就像下围棋的顶级棋手，能看到几十步之后的局面。
- $\gamma=1$ 时，所有未来奖励一视同仁。这在回合制任务中可行（因为回合必然结束），但在持续任务中求和会发散。

另一个直观理解：将 $\gamma$ 看作"智能体存活概率"。每一步结束后，智能体以 $\gamma$ 的概率继续存活，以 $1-\gamma$ 的概率终结。那么 $G_t$ 就是智能体存活期间的期望总奖励——这个视角让折扣因子的概率意义更加清晰。

## 代码示例

```python
import numpy as np

def compute_return(rewards, gamma=0.9):
    """计算一条轨迹的折扣回报序列"""
    T = len(rewards)
    G = np.zeros(T)
    cumulative = 0
    # 反向计算：从最后一个时间步向前
    for t in reversed(range(T)):
        cumulative = rewards[t] + gamma * cumulative
        G[t] = cumulative
    return G

# 示例：某 episode 的奖励序列
rewards = [0, 0, 0, 1, 0, 0, 0, 5]  # 在第4步得1分，第8步得5分

for gamma in [0.0, 0.5, 0.9, 0.99]:
    G = compute_return(rewards, gamma)
    print(f"gamma={gamma:.2f} -> 初始回报 G_0 = {G[0]:.4f}, 回报序列 = {np.round(G, 3)}")

# 演示折扣因子的影响
print("\n--- 不同 gamma 下未来奖励的现值 ---")
future_step = np.arange(0, 20)
for gamma in [0.5, 0.9, 0.99]:
    discount = gamma ** future_step
    print(f"gamma={gamma}: 10步后折扣={discount[10]:.4f}, "
          f"20步后折扣={discount[20]:.4f}")
```

## 深度学习关联

- **折扣因子作为超参数**：在深度强化学习（如 DQN、PPO、SAC）中，$\gamma$ 是关键的算法超参数。$\gamma=0.99$ 是大多数 Atari 游戏的标准选择，但对于奖励稀疏的任务可能需要更接近 1 的值。
- **TD 与 N 步回报**：深度学习时代，$N$ 步回报是折扣回报的重要变体。介于 TD(0)（1 步）和 MC（无限步）之间的截断回报在 A2C 和 PPO 中被广泛使用，平衡了方差和偏差。
- **Gamma 与值函数估计**：$\gamma$ 的选择影响价值函数的学习难度。过大的 $\gamma$ 导致价值信号需要传播更远的距离（信用分配问题），通常需要更复杂的网络架构或辅助任务（如 V-trace、Retrace）来缓解。
