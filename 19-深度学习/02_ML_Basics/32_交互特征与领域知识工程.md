# 32_交互特征与领域知识工程

## 核心概念

- **交互特征**：两个或多个原始特征的组合（如乘积 $x_i \times x_j$），捕捉特征之间的协同效应（Synergy Effect）。
- **特征交互的必要性**：线性模型假设特征对输出的影响是**可加的**（$y = w_1 x_1 + w_2 x_2$），但现实中 $x_1$ 的影响可能依赖于 $x_2$ 的值，即存在交互作用。
- **领域知识工程**：利用专家领域知识手动构造有意义的特征和交互项，是传统机器学习中提升模型效果的重要环节。
- **显式交互**：在预处理阶段添加 $x_i \times x_j$ 列，线性模型就能学到交互效应。缺点是特征数量从 $O(d)$ 增长到 $O(d^2)$。
- **隐式交互**：树模型（如决策树、GBDT）通过层次化分裂天然捕捉特征交互，无需显式构造交互特征。神经网络通过多层非线性变换也能学习交互。
- **交互的阶数**：二阶交互 $x_i \times x_j$ 最常见，三阶及以上交互虽然存在但其构造和解释难度剧增。

## 数学推导

**无交互的线性模型**：
$$
y = w_0 + w_1 x_1 + w_2 x_2 + \varepsilon
$$
$x_1$ 对 $y$ 的边际效应 $\partial y / \partial x_1 = w_1$，与 $x_2$ 无关。

**含二阶交互的线性模型**：
$$
y = w_0 + w_1 x_1 + w_2 x_2 + w_{12} (x_1 \times x_2) + \varepsilon
$$
$x_1$ 对 $y$ 的边际效应 $\partial y / \partial x_1 = w_1 + w_{12} x_2$——即 $x_1$ 的影响取决于 $x_2$ 的取值。

**交互效应的统计检验**：
$$
H_0: w_{12} = 0 \quad \text{(无交互效应)}
$$
通过 F 检验或 t 检验判断交互项是否显著。

**类别特征与数值特征的交互**：
对于类别特征 $A$（取值为 $0$ 或 $1$）和连续特征 $B$：
$$
y = w_0 + w_A A + w_B B + w_{AB} (A \times B)
$$
此时 $A \times B$ 相当于对不同类别允许不同的 $B$ 斜率——$A=0$ 时斜率为 $w_B$，$A=1$ 时斜率为 $w_B + w_{AB}$。

**特征工程中交互的数量控制**：$d$ 个特征的所有二阶交互有 $\binom{d}{2} = d(d-1)/2$ 个。当 $d=1000$ 时，约 50 万交互项。常用领域知识、统计筛选或树模型特征重要性来挑选有价值的交互。

## 直观理解

- **交互效应 = "1+1 > 2" 或 "1+1 < 2"**：药物的协同效应是经典的交互例子——药 A 单独治疗效果一般（$w_1=0.3$），药 B 也一般（$w_2=0.3$），但药 A+B 联合治疗效果远大于 0.6（$w_{12}=0.5$）。这就是正向交互。相反，有些组合互相抑制（1+1 < 2）。
- **特征交互的本质**：没有交互时，"是否下雨"和"有没有伞"对"是否会淋湿"的影响是独立的。但有交互——下雨时，有伞 = 不湿，没伞 = 湿；天晴时，有伞没伞都没关系。这就是典型的"条件依赖"：下雨的影响取决于有没有伞。
- **领域知识是最好的交互来源**：在风控领域，"信用卡使用率"和"还款及时率"的乘积是比单独使用两个特征更强的信用预测指标——这种交互来自金融业务理解，而非数据挖掘。

## 代码示例

```python
import numpy as np
from sklearn.preprocessing import PolynomialFeatures
from sklearn.linear_model import Ridge
from sklearn.datasets import load_diabetes

data = load_diabetes()
X, y = data.data, data.target
feature_names = data.feature_names
print(f"原始特征: {feature_names}")

# 显式添加交互特征（二阶交互，不含平方项）
interaction = PolynomialFeatures(degree=2, interaction_only=True,
                                 include_bias=False)
X_interact = interaction.fit_transform(X)
print(f"原始特征数: {X.shape[1]}")
print(f"含交互特征数: {X_interact.shape[1]}")

# 比较有无交互特征的线性模型
from sklearn.model_selection import cross_val_score
model_no_interact = Ridge(alpha=0.1)
model_interact = Ridge(alpha=0.1)

score_no = cross_val_score(model_no_interact, X, y, cv=5,
                           scoring='neg_mean_squared_error').mean()
score_yes = cross_val_score(model_interact, X_interact, y, cv=5,
                            scoring='neg_mean_squared_error').mean()

print(f"无交互 MSE: {-score_no:.2f}")
print(f"有交互 MSE: {-score_yes:.2f}")

# 查看交互特征名称
print(f"\n交互特征名称示例:\n{interaction.get_feature_names_out(feature_names)[:10]}")
```
