# 19_偏差-方差分解公式详细推导

## 核心概念

- **偏差-方差权衡**：机器学习的核心理论之一，将泛化误差分解为偏差 (Bias)、方差 (Variance) 和噪声 (Noise) 三部分，揭示了模型复杂度与泛化能力之间的根本矛盾。
- **偏差**：模型预测的期望值与真实值之间的差异。高偏差意味着模型过于简单（欠拟合），对数据的拟合能力不足。
- **方差**：模型对不同训练集的预测波动程度。高方差意味着模型过于复杂（过拟合），对训练数据中的噪声过度敏感。
- **噪声**：数据本身固有的不可约减的误差，是任何模型都无法消除的下界，由数据生成过程的内在随机性决定。
- **权衡关系**：增加模型复杂度通常降低偏差但提高方差；减少模型复杂度降低方差但提高偏差。最优模型在两者的交叉点。
- **对集成学习的指导**：Bagging 通过降低方差提升性能（如随机森林），Boosting 通过降低偏差提升性能（如 GBDT）。

## 数学推导

给定真实映射 $y = f(x) + \varepsilon$，其中 $\varepsilon \sim \mathcal{N}(0, \sigma^2)$ 是噪声。模型 $\hat{f}(x)$ 是在训练集 $D$ 上学习得到的。考虑单个测试点 $x$，预测的均方误差为：

$$
\begin{aligned}
Err(x) &= \mathbb{E}_D[(y - \hat{f}(x; D))^2] \\
&= \mathbb{E}_D[(f(x) + \varepsilon - \hat{f}(x; D))^2] \\
&= \mathbb{E}_D[(\varepsilon)^2] + \mathbb{E}_D[(f(x) - \hat{f}(x; D))^2] + 2\mathbb{E}_D[\varepsilon(f(x) - \hat{f}(x; D))]
\end{aligned}
$$

由于 $\varepsilon$ 与 $D$ 独立且 $\mathbb{E}[\varepsilon] = 0$，交叉项为零。因此：

$$
Err(x) = \sigma^2 + \mathbb{E}_D[(f(x) - \hat{f}(x; D))^2]
$$

将 $f(x) - \hat{f}(x; D)$ 加上再减去 $\mathbb{E}_D[\hat{f}(x; D)]$：

$$
\begin{aligned}
Err(x) &= \sigma^2 + \mathbb{E}_D[(f(x) - \mathbb{E}_D[\hat{f}(x)] + \mathbb{E}_D[\hat{f}(x)] - \hat{f}(x; D))^2] \\
&= \sigma^2 + (f(x) - \mathbb{E}_D[\hat{f}(x)])^2 + \mathbb{E}_D[(\hat{f}(x; D) - \mathbb{E}_D[\hat{f}(x)])^2] \\
&= \text{Noise} + \text{Bias}^2 + \text{Variance}
\end{aligned}
$$

其中：
- $\text{Noise} = \sigma^2$：**噪声**，数据固有的不可降低误差
- $\text{Bias}^2 = (f(x) - \mathbb{E}_D[\hat{f}(x)])^2$：**偏差平方**，模型预测期望与真实值的偏离
- $\text{Variance} = \mathbb{E}_D[(\hat{f}(x; D) - \mathbb{E}_D[\hat{f}(x)])^2]$：**方差**，模型对不同数据集的敏感度

**分类任务中的偏差-方差**：对于 0-1 损失，分解更为复杂，但核心思想一致——偏差衡量系统性错误，方差衡量预测的不稳定性。

**模型复杂度曲线**：以模型复杂度为横轴，泛化误差为纵轴：偏差随复杂度增加单调递减，方差单调递增，总误差呈 U 形曲线。

## 直观理解

- **射箭比喻**：偏差-方差权衡好比射箭——偏差是瞄准的系统性偏差（总是偏左 1 厘米），方差是箭的散布程度（有的偏左有的偏右）。高偏差低方差 = 箭都集中在同一个错误位置（稳定但不准）；低偏差高方差 = 箭围绕靶心散布（有时中靶有时远离）；低偏差低方差 = 全集中在靶心（理想状态）。
- **过拟合 vs 欠拟合**：欠拟合（高偏差）≈ 只用几个大致规则去理解世界，虽然稳定但太粗糙；过拟合（高方差）≈ 把每个细节都记下来，换一个场景就不知所措。好模型要在"理解规律"和"记忆细节"之间找到平衡。
- **集成学习的作用**：随机森林（Bagging）通过平均多个高方差模型来降低方差，相当于召集多个"偏科专家"投票，综合意见更稳定；GBDT（Boosting）通过逐步优化来降低偏差，相当于在薄弱环节持续加练，每一步都在改进之前的不足。

## 代码示例

```python
import numpy as np
from sklearn.ensemble import BaggingRegressor, GradientBoostingRegressor
from sklearn.tree import DecisionTreeRegressor
from sklearn.model_selection import cross_val_score

# 模拟一个带噪声的数据集
np.random.seed(42)
X = np.linspace(0, 10, 200).reshape(-1, 1)
y = np.sin(X).ravel() + np.random.randn(200) * 0.3

# 不同复杂度的模型（树的深度）
for depth in [1, 3, 10]:
    tree = DecisionTreeRegressor(max_depth=depth)
    scores = cross_val_score(tree, X, y, cv=5,
                             scoring='neg_mean_squared_error')
    mse = -scores.mean()
    # 训练集上观察偏差（近似）
    tree.fit(X, y)
    train_mse = np.mean((y - tree.predict(X)) ** 2)
    print(f"深度={depth:2d}: 交叉验证 MSE={mse:.4f}, "
          f"训练 MSE={train_mse:.4f} (差值≈方差)")

# Bagging 降低方差 vs Boosting 降低偏差
bag = BaggingRegressor(DecisionTreeRegressor(), n_estimators=50)
boost = GradientBoostingRegressor(n_estimators=50, max_depth=3)
print(f"\nBagging CV MSE: {-cross_val_score(bag, X, y, cv=5, scoring='neg_mean_squared_error').mean():.4f}")
print(f"Boosting CV MSE: {-cross_val_score(boost, X, y, cv=5, scoring='neg_mean_squared_error').mean():.4f}")
```

## 深度学习关联

- **深度网络的偏差-方差新视角**：传统观点认为深度网络（参数量极大）应该方差极高、容易过拟合。但现代深度学习中的"双下降"现象 (Double Descent) 挑战了这一观点——当模型容量超过插值阈值后，测试误差反而再次下降。这与传统偏差-方差权衡不同，是深度学习理论的前沿课题。
- **正则化的权衡调节**：深度学习中的 L1/L2 正则化、Dropout、早停 (Early Stopping) 等方法都在调节偏差-方差平衡——正则化强度增加 → 方差降低 → 偏差升高。理解偏差-方差分解是选择正则化策略的理论基础。
- **集成与模型平均**：深度学习中广泛使用的模型平均（如 SWA、EMA）、快照集成 (Snapshot Ensemble) 等方法，本质上是通过降低方差来提升性能，与 Bagging 的偏差-方差原理一致。
