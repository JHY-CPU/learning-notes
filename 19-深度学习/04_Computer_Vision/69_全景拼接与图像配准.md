# 69_全景拼接与图像配准

## 核心概念

- **图像配准 (Image Registration)**：将不同时间、不同视角或不同传感器获取的同一场景图像对齐到同一坐标系的过程。配准的精度直接影响后续融合和分析的质量。
- **特征点匹配**：通过检测图像中的关键点（SIFT、SURF、ORB等）并计算描述子，在不同图像间建立对应关系。好的特征点应具有旋转不变性、尺度不变性和光照鲁棒性。
- **单应性矩阵 (Homography)**：描述同一平面在两个视角间投影变换的 $3\times3$ 矩阵。全景拼接中假设场景近似平面或相机绕光心旋转，用单应性矩阵对齐图像。
- **RANSAC 算法**：随机采样一致性算法，用于从包含大量错误匹配（外点）的对应点对中稳健地估计单应性矩阵。迭代采样最小子集，选择支持内点最多的模型。
- **图像融合 (Image Blending)**：将对齐后的图像无缝拼接成全景图。直接拼接会产生明显的接缝，需要采用多频段融合 (Multi-band Blending) 或梯度域融合等技术消除亮度差异和接缝。
- **光束法平差 (Bundle Adjustment)**：在全景拼接的最后阶段，对所有图像的相机参数和匹配点进行全局优化，最小化重投影误差，提高整体一致性。

## 数学推导

**单应性矩阵映射**：二维齐次坐标下的平面变换关系。
$$
\mathbf{x}' = H \mathbf{x}, \quad 
\begin{pmatrix} x' \\ y' \\ 1 \end{pmatrix} = 
\begin{pmatrix} h_{11} & h_{12} & h_{13} \\ h_{21} & h_{22} & h_{23} \\ h_{31} & h_{32} & 1 \end{pmatrix}
\begin{pmatrix} x \\ y \\ 1 \end{pmatrix}
$$

**DLT 算法**：一对对应点提供两个线性约束：
$$
\begin{pmatrix} -x & -y & -1 & 0 & 0 & 0 & xx' & yx' & x' \\
0 & 0 & 0 & -x & -y & -1 & xy' & yy' & y' \end{pmatrix} \mathbf{h} = 0
$$

堆叠 4 对以上匹配点，求解 $A\mathbf{h} = 0$ 的 SVD 最小奇异值解。

**重投影误差**：光束法平差的优化目标：
$$
\min_{H_j, \mathbf{X}_i} \sum_{i,j} \| \mathbf{x}_{ij} - P(H_j, \mathbf{X}_i) \|^2
$$

其中 $\mathbf{x}_{ij}$ 是第 $i$ 个三维点在图像 $j$ 中的观测，$P$ 为投影函数。

## 直观理解

- **拼图类比**：全景拼接就像用手持相机拍摄多张有重叠的照片，然后像拼图一样把它们拼成一张宽幅全景图。关键在于找到相邻照片的重叠区域，精确对齐后再消除亮度差异。
- **RANSAC 的思想**：就像在嘈杂的人群中找共识——随机选几个人（最小样本数），看看有多少人同意这个意见（内点计数），重复多次后采纳支持度最高的方案。这比直接使用所有数据（被离群点污染）要稳健得多。
- **多频段融合的原理**：将图像分解为低频（大尺度亮度变化）和高频（细节纹理）成分，在不同频段上分别融合——低频用渐入渐出，高频用较窄的过渡带。这样既避免了低频的接缝痕迹，又保留了高频的细节锐度。

## 代码示例

```python
import cv2
import numpy as np

def stitch_images(img1, img2):
    """将两张有重叠的图像拼接成全景图"""
    # 1. 检测特征点和计算描述子
    sift = cv2.SIFT_create()
    kp1, des1 = sift.detectAndCompute(img1, None)
    kp2, des2 = sift.detectAndCompute(img2, None)
    
    # 2. 特征匹配
    matcher = cv2.FlannBasedMatcher(
        dict(algorithm=1, trees=5),  # KD-Tree
        dict(checks=50)
    )
    matches = matcher.knnMatch(des1, des2, k=2)
    
    # Lowe's 比率测试：保留最佳匹配远优于次佳匹配的点对
    good = []
    for m, n in matches:
        if m.distance < 0.7 * n.distance:
            good.append(m)
    
    # 3. 使用 RANSAC 估计单应性矩阵
    src_pts = np.float32([kp1[m.queryIdx].pt for m in good]).reshape(-1, 1, 2)
    dst_pts = np.float32([kp2[m.trainIdx].pt for m in good]).reshape(-1, 1, 2)
    
    H, mask = cv2.findHomography(src_pts, dst_pts, cv2.RANSAC, 5.0)
    print(f"内点比例: {mask.sum() / len(mask):.2%}")
    
    # 4. 图像变换与拼接
    h1, w1 = img1.shape[:2]
    h2, w2 = img2.shape[:2]
    
    # 计算全景图尺寸
    corners1 = np.float32([[0, 0], [0, h1], [w1, h1], [w1, 0]]).reshape(-1, 1, 2)
    corners2 = np.float32([[0, 0], [0, h2], [w2, h2], [w2, 0]]).reshape(-1, 1, 2)
    corners2_trans = cv2.perspectiveTransform(corners2, H)
    all_corners = np.concatenate((corners1, corners2_trans), axis=0)
    
    [xmin, ymin] = all_corners.min(axis=0).ravel().astype(int)
    [xmax, ymax] = all_corners.max(axis=0).ravel().astype(int)
    
    # 调整平移变换
    H_trans = np.array([[1, 0, -xmin], [0, 1, -ymin], [0, 0, 1]], dtype=np.float32)
    
    # 5. 融合拼接
    result = cv2.warpPerspective(img1, H_trans @ np.eye(3), (xmax - xmin, ymax - ymin))
    result[-ymin:h2 + (-ymin), -xmin:w2 + (-xmin)] = img2
    
    # 6. 多频段融合（推荐用于实际项目）
    # cv2.detail_MultiBandBlender() 提供更自然的融合效果
    
    return result

# 使用 OpenCV 内置拼接器（生产级）
def opencv_stitch(images):
    stitcher = cv2.Stitcher_create(cv2.Stitcher_PANORAMA)
    status, panorama = stitcher.stitch(images)
    if status == cv2.Stitcher_OK:
        return panorama
    else:
        raise Exception(f"拼接失败，错误码: {status}")
```

## 深度学习关联

- **SuperPoint + SuperGlue**：基于图神经网络的特征匹配方法。SuperPoint 是全卷积特征检测器，同时输出特征点位置和描述子；SuperGlue 将匹配建模为最优传输问题，通过可学习的注意力机制在特征点之间建立匹配关系。在低纹理、大视角变化等传统方法失效的场景中表现优异。
- **LoFTR (Detector-Free Local Feature Matching)**：无需检测特征点，直接在密集特征图上建立匹配。利用 Transformer 的自注意力和交叉注意力机制捕获全局上下文信息，在弱纹理区域和重复纹理场景中显著优于基于特征点的方法。
- **端到端全景拼接**：最新研究尝试将整个拼接流程（从配准到融合）用神经网络端到端学习，如利用空间变换网络 (STN) 实现可微的图像变形，用生成对抗网络 (GAN) 进行无缝融合，消除对显式特征匹配和单应性估计的依赖。
