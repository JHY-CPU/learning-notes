# 68_立体视觉与深度图生成

## 核心概念

- **立体视觉 (Stereo Vision)**：通过两个或多个相机从不同视角拍摄同一场景，利用视差信息恢复场景的三维深度。模拟人类双眼感知深度的机制。
- **视差 (Disparity)**：同一空间点在左右两个视图中的像素位置差。视差与深度成反比：视差越大，物体越近；视差为零，物体在无穷远。
- **对极几何 (Epipolar Geometry)**：描述两个相机视图之间的几何约束关系。空间点在左视图中的对应点必然位于右视图的对极线上，将二维搜索降为一维搜索，大幅降低匹配计算量。
- **本质矩阵与基础矩阵**：本质矩阵 $E$ 描述已标定相机间的几何关系，基础矩阵 $F$ 描述未标定相机间的关系，两者均约束对应点的对极线位置。
- **立体匹配 (Stereo Matching)**：在左右视图中寻找对应点对的技术，是立体视觉的核心步骤。常见方法包括局部匹配（基于窗口）、全局匹配（基于能量优化）和半全局匹配（SGM）。
- **深度图 (Depth Map)**：与视图同分辨率的图像，每个像素值表示该点到相机的距离。深度图可进一步生成点云或三维网格。

## 数学推导

$$
\text{深度与视差关系：} \quad Z = \frac{f \cdot B}{d}
$$

其中 $Z$ 为深度，$f$ 为焦距，$B$ 为基线距离（两相机光心间距），$d = x_l - x_r$ 为视差。

**对极约束**：若左视图点 $\mathbf{p}_l$ 对应右视图点 $\mathbf{p}_r$，则有：
$$
\mathbf{p}_r^T F \mathbf{p}_l = 0
$$

其中 $F$ 为基础矩阵，$F = K_r^{-T} [t]_\times R K_l^{-1}$，$R$ 和 $t$ 为两相机间的旋转和平移。

**三角化 (Triangulation)**：已知相机投影矩阵 $P_l$ 和 $P_r$，三维点 $X$ 满足：
$$
\mathbf{x}_l = P_l X, \quad \mathbf{x}_r = P_r X
$$

通过求解线性方程组 $AX = 0$ 得到三维坐标。

## 直观理解

- **双眼类比**：闭上一只眼睛，用手指去触碰远处的物体——你会发现自己很难准确判断距离。这就是立体视觉的原理：两只眼睛看到的图像略有差异，大脑通过这种差异计算深度。基线越宽（两眼距离越大），深度感知越精确。
- **视差与远近**：坐车时看窗外的景色，近处的树木快速向后移动（视差大），远处的山几乎不动（视差小）。这就是立体匹配中视差与深度成反比的直观体现。
- **SGM 算法的核心思想**：半全局匹配将对立体匹配的约束建模为沿着多个方向的路径进行动态规划，类似于在二维图像上"扫描"寻找最优匹配路径，兼顾匹配精度和计算效率。

## 代码示例

```python
import cv2
import numpy as np

# 读取左右视图
left = cv2.imread('left.png', cv2.IMREAD_GRAYSCALE)
right = cv2.imread('right.png', cv2.IMREAD_GRAYSCALE)

# 使用 SGBM (Semi-Global Block Matching) 计算视差图
stereo = cv2.StereoSGBM_create(
    minDisparity=0,
    numDisparities=64,      # 最大视差搜索范围
    blockSize=11,           # 匹配窗口大小（奇数）
    P1=8 * 3 * 11 ** 2,    # 视差平滑惩罚参数1
    P2=32 * 3 * 11 ** 2,   # 视差平滑惩罚参数2
    disp12MaxDiff=1,        # 左右一致性检查阈值
    uniquenessRatio=10,     # 唯一性比率
    speckleWindowSize=100,  # 去除斑点的窗口大小
    speckleRange=32
)

disparity = stereo.compute(left, right).astype(np.float32) / 16.0

# 将视差图转换为深度图
focal_length = 718.856     # 相机焦距 (pixel)
baseline = 0.537           # 基线距离 (meter)
depth = np.where(disparity > 0, focal_length * baseline / disparity, 0)

# 归一化显示
depth_norm = cv2.normalize(depth, None, 0, 255, cv2.NORM_MINMAX)

# WLS 滤波平滑视差图
right_matcher = cv2.ximgproc.createRightMatcher(stereo)
disparity_right = right_matcher.compute(right, left)
wls_filter = cv2.ximgproc.createDisparityWLSFilter(stereo)
wls_filter.setLambda(8000)
wls_filter.setSigmaColor(0.8)
filtered = wls_filter.filter(disparity, left, disparity_right)

# 生成点云（伪代码）
def disparity_to_pointcloud(disparity, K):
    """将视差图转换为三维点云"""
    h, w = disparity.shape
    fx, fy = K[0, 0], K[1, 1]
    cx, cy = K[0, 2], K[1, 2]
    points = []
    for v in range(h):
        for u in range(w):
            d = disparity[v, u]
            if d > 0:
                z = focal_length * baseline / d
                x = (u - cx) * z / fx
                y = (v - cy) * z / fy
                points.append([x, y, z])
    return np.array(points)
```

## 深度学习关联

- **MVSNet 系列**：基于深度学习的多视图立体匹配方法。MVSNet 通过可微的单应性变换构建代价体 (Cost Volume)，利用 3D 卷积进行正则化，输出深度图。相比传统方法能更好地处理弱纹理区域和遮挡，且支持任意数量输入视图的灵活重建。
- **RAFT-Stereo**：借鉴光流估计中 RAFT 的迭代优化思想，构建 4D 相关体并通过 GRU 循环迭代更新视差图。克服了传统方法中窗口大小和视差范围的限制，在复杂场景中取得显著优势。
- **自监督深度估计 (Monodepth2)**：仅需单目视频序列即可训练深度估计网络，利用视图合成的光度一致性损失、左右视差一致性损失和边缘感知平滑损失，无需真实深度标签。为立体视觉提供了数据驱动的替代方案，显著降低了深度数据采集成本。
