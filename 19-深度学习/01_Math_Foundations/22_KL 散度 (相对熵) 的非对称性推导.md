# 22_KL 散度 (相对熵) 的非对称性推导

## 核心概念

- **KL 散度 (Kullback-Leibler Divergence)**：衡量两个概率分布 $P$ 和 $Q$ 之间的"距离"，定义为 $D_{\text{KL}}(P\|Q) = \sum_x P(x) \log \frac{P(x)}{Q(x)}$。是使用 $Q$ 近似 $P$ 时产生的信息损失。
- **非对称性 (Asymmetry)**：$D_{\text{KL}}(P\|Q) \neq D_{\text{KL}}(Q\|P)$。这一非对称性具有深刻含义：$D_{\text{KL}}(P\|Q)$ 是"前向 KL"，$D_{\text{KL}}(Q\|P)$ 是"反向 KL"，两者行为截然不同。
- **前向 KL 的零避免 (Zero Avoiding)**：$D_{\text{KL}}(P\|Q)$ 中当 $P(x) > 0$ 但 $Q(x) \to 0$ 时，单项趋于无穷。这意味着最优 $Q$ 必须覆盖所有 $P$ 有质量的区域（均值搜索 / mode-covering）。
- **反向 KL 的零强迫 (Zero Forcing)**：$D_{\text{KL}}(Q\|P)$ 中当 $Q(x) > 0$ 但 $P(x) = 0$ 时，单项趋于无穷。这意味着最优 $Q$ 必须在 $P$ 无质量的区域不分配质量（模式坍缩 / mode-seeking）。
- **非负性**：$D_{\text{KL}}(P\|Q) \geq 0$，等号成立当且仅当 $P = Q$（吉布斯不等式）。
- **KL 散度不是度量**：KL 散度不满足对称性也不满足三角不等式，因此不是真正的"距离度量"，而是一种"散度"（divergence）。

## 数学推导

KL 散度的定义（离散形式）：
$$
D_{\text{KL}}(P\|Q) = \sum_{x \in \mathcal{X}} P(x) \log \frac{P(x)}{Q(x)} = \mathbb{E}_{x \sim P}\left[\log \frac{P(x)}{Q(x)}\right]
$$

连续形式：
$$
D_{\text{KL}}(P\|Q) = \int p(x) \log \frac{p(x)}{q(x)} dx
$$

非对称性的证明：设 $P = \text{Bernoulli}(0.5)$，$Q = \text{Bernoulli}(0.9)$。
$$
D_{\text{KL}}(P\|Q) = 0.5\log\frac{0.5}{0.9} + 0.5\log\frac{0.5}{0.1} \approx 0.51
$$
$$
D_{\text{KL}}(Q\|P) = 0.9\log\frac{0.9}{0.5} + 0.1\log\frac{0.1}{0.5} \approx 0.69
$$
两者不相等！

KL 散度与熵的关系：
$$
D_{\text{KL}}(P\|Q) = \underbrace{-\sum P(x)\log Q(x)}_{H_{\times}(P,Q)} - \underbrace{(-\sum P(x)\log P(x))}_{H(P)} = H_{\times}(P,Q) - H(P)
$$
即 KL 散度 = 交叉熵 - 熵。

## 直观理解

- **前向 KL 的"覆盖"行为**：假设真实分布 $P$ 是双峰的，用一个高斯分布 $Q$ 去逼近。前向 KL ($D_{\text{KL}}(P\|Q)$) 会在两个峰之间放置高斯，因为任何 $P$ 有质量的区域 $Q$ 也必须覆盖。结果是高斯覆盖了 $P$ 的大部分支撑集（但两个峰都没拟合好）。
- **反向 KL 的"选择"行为**：反向 KL ($D_{\text{KL}}(Q\|P)$) 则会选择 $P$ 的一个峰来拟合，因为 $Q$ 不会在 $P$ 无质量的地方浪费概率质量。结果是一个峰拟合得很好，另一个峰被完全忽略。
- **"代价不对称"**：用 $Q$ 近似 $P$ 时，两种错误的代价不同。前向 KL 认为"在 $P$ 可能的地方 $Q$ 太小"是不可接受的；反向 KL 认为"在 $P$ 不可能的地方 $Q$ 太大"是不可接受的。

## 代码示例

```python
import numpy as np
from scipy.stats import norm, beta, bernoulli

def kl_divergence(p, q, epsilon=1e-10):
    """计算 KL(P||Q)"""
    p = np.array(p) + epsilon
    q = np.array(q) + epsilon
    return np.sum(p * np.log(p / q))

# 1. 验证非对称性
p = np.array([0.5, 0.5])
q = np.array([0.9, 0.1])
print("非对称性验证:")
print(f"  KL(P||Q) = {kl_divergence(p, q):.4f}")
print(f"  KL(Q||P) = {kl_divergence(q, p):.4f}")

# 2. 前向 KL vs 反向 KL 的行为差异
# 双峰分布 P
x_grid = np.linspace(0, 1, 1000)
p_bimodal = 0.5 * beta.pdf(x_grid, 2, 10) + 0.5 * beta.pdf(x_grid, 10, 2)
p_bimodal /= np.trapz(p_bimodal, x_grid)  # 归一化

# 近似 Q 为单峰 Beta 分布
def kl_forward(q_params):
    """前向 KL: KL(P||Q) 的数值近似"""
    a, b = q_params
    q_vals = beta.pdf(x_grid, a, b) + 1e-10
    q_vals /= np.trapz(q_vals, x_grid)
    return np.trapz(p_bimodal * np.log(p_bimodal / q_vals), x_grid)

def kl_reverse(q_params):
    """反向 KL: KL(Q||P) 的数值近似"""
    a, b = q_params
    q_vals = beta.pdf(x_grid, a, b) + 1e-10
    q_vals /= np.trapz(q_vals, x_grid)
    return np.trapz(q_vals * np.log(q_vals / p_bimodal), x_grid)

# 尝试不同的 Beta 参数
variants = [(2, 2), (5, 5), (3, 8), (8, 3)]
print("\n前向 KL 与反向 KL 对比:")
for a, b in variants:
    kf = kl_forward((a, b))
    kr = kl_reverse((a, b))
    print(f"  Beta({a},{b}): KL(P||Q)={kf:.4f}, KL(Q||P)={kr:.4f}")

# 3. KL 散度的非负性验证
np.random.seed(42)
for _ in range(10):
    p = np.random.dirichlet(np.ones(5))
    q = np.random.dirichlet(np.ones(5))
    kl = kl_divergence(p, q)
    print(f"KL(P||Q) = {kl:.4f} >= 0 (保证)")

# 4. KL 作为交叉熵 - 熵
p = np.array([0.3, 0.3, 0.4])
q = np.array([0.2, 0.5, 0.3])
cross_entropy = -np.sum(p * np.log(q))
entropy_p = -np.sum(p * np.log(p))
kl = kl_divergence(p, q)
print(f"\n交叉熵 = {cross_entropy:.4f}")
print(f"H(p) = {entropy_p:.4f}")
print(f"KL = {kl:.4f} = {cross_entropy - entropy_p:.4f}")
```

## 深度学习关联

- **变分自编码器 (VAE)**：VAE 的损失函数包含 KL 散度 $D_{\text{KL}}(q(z|x)\|p(z))$，其中 $q(z|x)$ 是编码器输出的近似后验，$p(z)$ 是标准正态先验。这是反向 KL——编码器学习将每个样本映射到先验附近的隐变量，避免隐空间产生大的空洞。
- **知识蒸馏 (Knowledge Distillation)**：学生网络 $Q$ 学习教师网络 $P$ 的软标签分布。训练损失 $L = D_{\text{KL}}(P\|Q)$（前向 KL），迫使学生分布覆盖教师分布的所有信息，包括类别间的细粒度相似性。
- **策略梯度中的 KL 约束**：在 TRPO 和 PPO 等强化学习算法中，策略更新受 KL 散度约束 $\pi_{k+1} = \arg\max_\pi \mathbb{E}[A(s,a)]$ s.t. $D_{\text{KL}}(\pi_k\|\pi) \leq \delta$，防止策略更新过大导致性能崩塌。这是将 KL 作为"信任域"的经典应用。
- **变分推断**：在许多贝叶斯深度学习方法中，我们用简单分布 $q(\theta)$ 去逼近真实后验 $p(\theta|D)$，最小化 $D_{\text{KL}}(q\|p)$（反向 KL）。这导致 $q$ 倾向于在 $p$ 的一个模式上坍缩（mode-seeking），计算上更可行但会低估不确定性。
