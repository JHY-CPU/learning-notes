# 23_交叉熵损失的信息论解释

## 核心概念

- **交叉熵 (Cross-Entropy)**：$H_{\times}(P, Q) = -\sum_x P(x) \log Q(x)$，衡量用分布 $Q$ 编码来自分布 $P$ 的样本所需的平均比特数。交叉熵 = 熵 + KL 散度：$H_{\times}(P,Q) = H(P) + D_{\text{KL}}(P\|Q)$。
- **交叉熵损失函数**：在分类任务中，交叉熵损失 $\mathcal{L} = -\sum_{c=1}^C y_c \log \hat{y}_c$ 是实际类别分布 $y$（one-hot 向量）与预测分布 $\hat{y}$（Softmax 输出）之间的交叉熵。
- **信息论解释**：最小化交叉熵等价于最小化模型分布与真实分布之间的 KL 散度（因为 $H(P)$ 是常数）。所以训练分类器本质上是在让模型分布逼近数据分布。
- **Softmax + 交叉熵**：多分类的标准组合。Softmax 将 logits 转换为概率分布 $\hat{y}_c = e^{z_c} / \sum_j e^{z_j}$，确保概率非负且和为 1。交叉熵计算与 one-hot 标签的距离。
- **二元交叉熵 (Binary Cross-Entropy)**：二分类特例 $\mathcal{L} = -[y \log \hat{y} + (1-y) \log (1-\hat{y})]$，等价于伯努利分布的负对数似然。
- **交叉熵 vs MSE 分类**：交叉熵在分类任务上通常优于 MSE，因为其梯度与预测误差成比例（$\partial \mathcal{L}/\partial z_k = \hat{y}_k - y_k$），没有饱和问题。

## 数学推导

交叉熵的定义：
$$
H_{\times}(P, Q) = -\sum_{x} P(x) \log Q(x) = \mathbb{E}_{x \sim P}[-\log Q(x)]
$$

从熵和 KL 散度到交叉熵：
$$
D_{\text{KL}}(P\|Q) = \sum_x P(x)\log\frac{P(x)}{Q(x)} = \underbrace{-\sum_x P(x)\log Q(x)}_{H_{\times}(P,Q)} \underbrace{- (-\sum_x P(x)\log P(x))}_{H(P)}
$$

多分类交叉熵损失（$C$ 个类别，$N$ 个样本）：
$$
\mathcal{L} = -\frac{1}{N}\sum_{i=1}^N \sum_{c=1}^C y_{ic} \log \hat{y}_{ic}
$$

其中 $y_{ic} = 1$ 当样本 $i$ 属于类别 $c$，否则为 0（one-hot 编码）。

Softmax 函数：
$$
\hat{y}_c = \frac{e^{z_c}}{\sum_{j=1}^C e^{z_j}}
$$

Softmax + 交叉熵的梯度（简洁性质）：
$$
\frac{\partial \mathcal{L}}{\partial z_k} = \hat{y}_k - y_k
$$
这个梯度的简洁形式是深度学习中广泛使用 Softmax + 交叉熵组合的重要原因。

## 直观理解

- **编码长度解释**：假设真实标签分布 $P$（大部分概率集中在正确类别），模型预测 $Q$。交叉熵 $H_{\times}(P, Q)$ 衡量"用 $Q$ 定义的编码方案来编码实际标签所需的平均比特数"。我们想让这个编码尽可能高效。
- **置信度惩罚**：交叉熵不仅惩罚错误的分类，还惩罚"虽然正确但不够自信"的预测。如果正确类别的预测概率只有 0.6（本应为 1.0），仍然有显著的损失。
- **决策理论视角**：最大化 Softmax 输出的概率等价于选择交叉熵损失最小的类别，这也是"最大后验"决策规则的体现。

## 代码示例

```python
import numpy as np

def cross_entropy(y_true, y_pred, epsilon=1e-10):
    """交叉熵损失"""
    y_pred = np.clip(y_pred, epsilon, 1 - epsilon)
    return -np.sum(y_true * np.log(y_pred))

def categorical_cross_entropy(y_true, y_pred):
    """多分类交叉熵平均"""
    return cross_entropy(y_true, y_pred) / len(y_true)

# 1. 二分类交叉熵
y_bin = np.array([1, 0, 1, 0])  # 真实标签
pred_perfect = np.array([0.99, 0.01, 0.99, 0.01])
pred_poor = np.array([0.6, 0.4, 0.6, 0.4])
pred_wrong = np.array([0.01, 0.99, 0.01, 0.99])

def binary_cross_entropy(y, pred):
    return -np.mean(y * np.log(pred) + (1-y) * np.log(1-pred))

print("二分类交叉熵:")
print(f"  完美预测: {binary_cross_entropy(y_bin, pred_perfect):.4f}")
print(f"  低置信度: {binary_cross_entropy(y_bin, pred_poor):.4f}")
print(f"  错误预测: {binary_cross_entropy(y_bin, pred_wrong):.4f}")

# 2. 多分类交叉熵 + Softmax
def softmax(z):
    e_z = np.exp(z - np.max(z))  # 数值稳定性
    return e_z / np.sum(e_z)

def cross_entropy_loss(logits, true_class):
    y_pred = softmax(logits)
    n_classes = len(logits)
    y_true = np.zeros(n_classes)
    y_true[true_class] = 1
    return -np.sum(y_true * np.log(y_pred + 1e-10))

# 测试 Softmax + 交叉熵的梯度
z = np.array([2.0, 1.0, 0.1])  # logits
y = 0  # 真实类别是第 0 类

s = softmax(z)
loss = cross_entropy_loss(z, y)
gradient = s - np.eye(len(z))[y]

print(f"\n多分类 Softmax + 交叉熵:")
print(f"  Logits: {z}")
print(f"  Softmax: {np.round(s, 4)}")
print(f"  损失: {loss:.4f}")
print(f"  梯度 (s - y): {np.round(gradient, 4)}")

# 3. 验证梯度形式：∂L/∂z_k = s_k - y_k
h = 1e-6
for k in range(len(z)):
    z_plus = z.copy()
    z_plus[k] += h
    loss_plus = cross_entropy_loss(z_plus, y)
    num_grad = (loss_plus - loss) / h
    print(f"  ∂L/∂z_{k}: 数值={num_grad:.4f}, 解析={gradient[k]:.4f}")

# 4. 交叉熵 vs MSE 在分类中的对比
print("\n交叉熵 vs MSE (分类):")
probabilities = np.linspace(0.001, 0.999, 100)
ce_losses = [-np.log(p) for p in probabilities]  # y_true=1
mse_losses = [(1-p)**2 for p in probabilities]
print(f"  正确类概率=0.9: CE={-np.log(0.9):.4f}, MSE={(0.1)**2:.4f}")
print(f"  正确类概率=0.5: CE={-np.log(0.5):.4f}, MSE={(0.5)**2:.4f}")
print(f"  正确类概率=0.1: CE={-np.log(0.1):.4f}, MSE={(0.9)**2:.4f}")
```

## 深度学习关联

- **分类任务的标准损失**：交叉熵 + Softmax 是深度学习分类任务的标准配置，从简单的图像分类（AlexNet、ResNet）到复杂的大语言模型（GPT、LLaMA）的训练都在使用。它直接对应多项式分布的 MLE，具有坚实的统计基础。
- **标签平滑 (Label Smoothing)**：将 one-hot 标签 $y$ 替换为 $(1-\epsilon)y + \epsilon/C$，防止模型过于自信（Softmax 输出极端靠近 0 或 1）。这等价于在交叉熵损失中加入正则化项 $D_{\text{KL}}(u\| \hat{y})$，其中 $u$ 是均匀分布。
- **焦点损失 (Focal Loss)**：在类别不平衡的场景（如目标检测）中，修改交叉熵为 $\mathcal{L}_{\text{focal}} = -(1-\hat{y})^\gamma \log \hat{y}$，通过降低易分类样本的权重，使模型更关注难样本。$\gamma$ 控制聚焦程度。
- **知识蒸馏中的交叉熵**：蒸馏损失 $\mathcal{L} = D_{\text{KL}}(p_{\text{teacher}}\| p_{\text{student}})$ 使用软化的 Softmax（温度 $\tau > 1$）。高温使教师分布更平滑，传递类别间关系（如"猫比狗更像老虎"），学生从中学习到超越单一标签的知识。
