# 接口与联合类型

## 一、接口 (Interface)

```graphql
# 接口定义
interface Node {
  id: ID!
}

interface Timestamped {
  createdAt: DateTime!
  updatedAt: DateTime
}

# 实现接口
type User implements Node & Timestamped {
  id: ID!
  name: String!
  createdAt: DateTime!
  updatedAt: DateTime
}

type Product implements Node & Timestamped {
  id: ID!
  name: String!
  price: Float!
  createdAt: DateTime!
  updatedAt: DateTime
}

# 查询时使用接口类型
type Query {
  node(id: ID!): Node           # 返回接口类型
}
```

```graphql
# 查询中使用 fragment 获取具体类型
query {
  node(id: "1") {
    id
    ... on User {
      name
      email
    }
    ... on Product {
      name
      price
    }
  }
}
```

## 二、联合类型 (Union)

```graphql
# 联合类型定义
union SearchResult = User | Product | Order

# 搜索查询
type Query {
  search(keyword: String!): [SearchResult!]!
}

# 查询中必须用 inline fragment
query {
  search(keyword: "手机") {
    ... on User {
      name
      email
    }
    ... on Product {
      name
      price
    }
    ... on Order {
      orderNo
      total
    }
  }
}
```

```java
// TypeResolver 处理联合类型
@Bean
public TypeResolver searchResultTypeResolver() {
    return env -> {
        Object object = env.getObject();
        if (object instanceof User) return env.getSchema().getObjectType("User");
        if (object instanceof Product) return env.getSchema().getObjectType("Product");
        if (object instanceof Order) return env.getSchema().getObjectType("Order");
        return null;
    };
}
```

## 三、接口 vs 联合类型

| 特性 | 接口 | 联合类型 |
|------|------|---------|
| 共同字段 | 有 | 无 |
| fragment | 可选 | 必须 |
| 实现方式 | implements | = |
| 适用场景 | 多态实体 | 搜索/聚合 |

## 四、注意事项

1. **接口适合有共同字段的多态场景**
2. **联合类型适合搜索等聚合场景**
3. **查询联合类型必须用 inline fragment**
4. **接口变更会影响所有实现者**
5. **联合类型不能包含标量类型**
