# 对象类型

## 一、基本定义

```graphql
type User {
  id: ID!
  name: String!
  email: String!
  avatar: String
  role: UserRole!
  createdAt: DateTime!
  updatedAt: DateTime
}

type Order {
  id: ID!
  orderNo: String!
  total: Float!
  status: OrderStatus!
  items: [OrderItem!]!
  user: User!
  createdAt: DateTime!
}

type OrderItem {
  id: ID!
  product: Product!
  quantity: Int!
  price: Float!
  subtotal: Float!
}
```

## 二、关联关系

```graphql
# 一对多
type User {
  id: ID!
  name: String!
  orders: [Order!]!        # User → Orders (一对多)
}

type Order {
  id: ID!
  user: User!              # Order → User (多对一)
}

# 多对多
type User {
  id: ID!
  roles: [Role!]!          # 用户-角色 多对多
}

type Role {
  id: ID!
  name: String!
  users: [User!]!
}

# 嵌套对象
type Order {
  id: ID!
  shippingAddress: Address!   # 值对象嵌套
  billingAddress: Address
}

type Address {
  street: String!
  city: String!
  province: String!
  zipCode: String!
  country: String!
}
```

## 三、计算字段

```graphql
type User {
  firstName: String!
  lastName: String!
  fullName: String!              # 计算字段: firstName + lastName
  age: Int                       # 计算字段: 从 birthday 计算
  orderCount: Int!               # 计算字段: 统计订单数
}

type OrderItem {
  quantity: Int!
  price: Float!
  subtotal: Float!               # 计算字段: quantity * price
}
```

```java
// Resolver 实现计算字段
@SchemaMapping(typeName = "User")
public String fullName(User user) {
    return user.getFirstName() + " " + user.getLastName();
}

@SchemaMapping(typeName = "OrderItem")
public double subtotal(OrderItem item) {
    return item.getQuantity() * item.getPrice();
}
```

## 四、注意事项

1. **每个类型有唯一的 GraphQL 类型定义**
2. **关联关系在 Resolver 中加载**，注意 N+1
3. **计算字段不需要存储**，Resolver 里算
4. **对象类型不能作为 Mutation 的输入**，用 Input 类型
5. **字段返回列表时标明是否可为空**
