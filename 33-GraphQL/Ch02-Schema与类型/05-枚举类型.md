# 枚举类型

## 一、基本定义

```graphql
# 枚举定义
enum OrderStatus {
  CREATED
  PAID
  SHIPPED
  DELIVERED
  CANCELLED
}

enum UserRole {
  ADMIN
  CUSTOMER
  VENDOR
  SUPPORT
}

enum SortDirection {
  ASC
  DESC
}

# 使用枚举
type Order {
  id: ID!
  status: OrderStatus!
}

type User {
  id: ID!
  role: UserRole!
}

type Query {
  orders(
    status: OrderStatus
    sort: SortDirection = DESC
  ): [Order!]!
}
```

## 二、枚举值描述

## 三、服务端映射

```java
// Java 枚举
public enum OrderStatus {
    CREATED, PAID, SHIPPED, DELIVERED, CANCELLED
}

// graphql-java 自动映射枚举
// 或手动注册
@Bean
public RuntimeWiring.Builder runtimeWiring() {
    return RuntimeWiring.newRuntimeWiring()
        .type("OrderStatus", builder ->
            builder.enumValues(
                name -> OrderStatus.valueOf(name)
            ))
        .type("Query", builder ->
            builder.dataFetcher("orders", env -> {
                OrderStatus status = env.getArgument("status");
                return orderRepository.findByStatus(status);
            }));
}
```

## 四、TypeScript 映射

```typescript
// graphql-codegen 自动生成
export enum OrderStatus {
  Created = 'CREATED',
  Paid = 'PAID',
  Shipped = 'SHIPPED',
  Delivered = 'DELIVERED',
  Cancelled = 'CANCELLED',
}

// 在 Resolver 中使用
const resolvers = {
  Query: {
    orders: async (_, { status }) => {
      // status 参数自动映射为枚举值
      if (status === OrderStatus.Created) {
        return db.orders.findByStatus('CREATED');
      }
      return db.orders.findAll();
    },
  },
  // 枚举值映射
  OrderStatus: {
    CREATED: 'CREATED',
    PAID: 'PAID',
    SHIPPED: 'SHIPPED',
    DELIVERED: 'DELIVERED',
    CANCELLED: 'CANCELLED',
  },
};
```

## 五、枚举 vs 联合类型

```graphql
# 枚举 - 有限的固定值
enum PaymentMethod {
  ALIPAY
  WECHAT_PAY
  CREDIT_CARD
  BANK_TRANSFER
}

# 联合类型 - 每个选项有不同字段
union PaymentResult = PaymentSuccess | PaymentFailed | PaymentPending

type PaymentSuccess {
  transactionId: String!
  paidAt: DateTime!
}

type PaymentFailed {
  errorCode: String!
  errorMessage: String!
}

type PaymentPending {
  expiresAt: DateTime!
  redirectUrl: String
}
```

## 六、枚举的默认值与可选性

```graphql
# 枚举作为可选参数
type Query {
  # 可选枚举参数 - 不传则不筛选
  orders(status: OrderStatus): [Order!]!

  # 带默认值的枚举参数
  products(sort: SortDirection = DESC): [Product!]!

  # 非空枚举参数 - 必须传
  report(type: ReportType!): Report!
}

# Input 中的枚举默认值
input CreateOrderInput {
  items: [OrderItemInput!]!
  paymentMethod: PaymentMethod!       # 必填
  priority: OrderPriority = NORMAL    # 有默认值
}

enum OrderPriority {
  LOW
  NORMAL
  HIGH
  URGENT
}
```

## 七、注意事项

1. **枚举值默认全大写 SNAKE_CASE**
2. **枚举值不能有前导数字**
3. **新增枚举值是安全的**（非破坏性变更）
4. **删除枚举值是破坏性的**
5. **枚举适合固定集合**，不要用 String 替代
6. **联合类型适合每个选项有不同字段的场景**
7. **客户端代码确保 switch 枚举所有值**，防止遗漏
