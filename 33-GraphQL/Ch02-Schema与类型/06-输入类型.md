# 输入类型

## 一、基本定义

```graphql
# 输入类型 - 用于 Mutation 参数
input CreateUserInput {
  name: String!
  email: String!
  password: String!
  role: UserRole = CUSTOMER
  profile: CreateProfileInput
}

input CreateProfileInput {
  avatar: String
  bio: String
  phone: String
  address: AddressInput
}

input AddressInput {
  street: String!
  city: String!
  province: String!
  zipCode: String!
  country: String! = "CN"
}

input UpdateUserInput {
  name: String                    # 可选 - 部分更新
  email: String
  profile: UpdateProfileInput
}

# Mutation 使用
type Mutation {
  createUser(input: CreateUserInput!): User!
  updateUser(id: ID!, input: UpdateUserInput!): User!
}
```

## 二、输入类型 vs 对象类型

```graphql
# ✗ 错误 - 对象类型不能用作输入
type Address {
  street: String!
  city: String!
}

type Mutation {
  createUser(address: Address): User!  # ✗ Address 是对象类型
}

# ✓ 正确 - 输入类型用于输入
input AddressInput {
  street: String!
  city: String!
}

type Mutation {
  createUser(address: AddressInput!): User!  # ✓
}

# ✓ 输出用对象类型
type User {
  address: Address!              # ✓ 输出用对象类型
}
```

## 三、批量操作输入

```graphql
input BatchUserInput {
  users: [CreateUserInput!]!
}

input BulkOperationInput {
  create: [CreateUserInput!]
  update: [UpdateUserInput!]
  delete: [ID!]
}

type Mutation {
  batchCreateUsers(input: BatchUserInput!): BatchResult!
  bulkUserOperations(input: BulkOperationInput!): BulkResult!
}

type BatchResult {
  successCount: Int!
  failedCount: Int!
  errors: [BatchError!]!
}

type BatchError {
  index: Int!
  message: String!
}
```

## 四、嵌套输入与复杂更新

```graphql
# 复杂嵌套输入
input CreateProductInput {
  name: String!
  description: String
  price: Decimal!
  category: CategoryInput!
  variants: [VariantInput!]!
  tags: [String!]
  images: [ImageInput!]
}

input CategoryInput {
  id: ID                    # 引用已有分类
  name: String              # 或创建新分类
}

input VariantInput {
  sku: String!
  color: String!
  size: String!
  stock: Int!
  price: Decimal
}

input ImageInput {
  url: String!
  alt: String
  isPrimary: Boolean = false
}
```

## 五、Partial Update 策略

```graphql
# 策略 1: 所有字段可选（推荐）
input UpdateUserInput {
  name: String              # null = 不更新
  email: String
  phone: String
}

# 策略 2: 显式字段控制
input UpdateUserInput {
  name: StringFieldUpdate
  email: StringFieldUpdate
}

input StringFieldUpdate {
  set: String               # 设置新值
  unset: Boolean            # 明确置空
}

# 策略 3: Patch 对象
input PatchUserInput {
  add: AddUserFields        # 新增字段
  remove: [String!]         # 删除字段
  replace: ReplaceUserFields # 替换字段
}
```

## 六、Resolver 中处理输入

```javascript
const resolvers = {
  Mutation: {
    updateUser: async (_, { id, input }, context) => {
      // 过滤掉 undefined 的字段
      const updateData = Object.entries(input)
        .filter(([_, value]) => value !== undefined)
        .reduce((acc, [key, value]) => ({ ...acc, [key]: value }), {});

      if (Object.keys(updateData).length === 0) {
        throw new GraphQLError('至少提供一个字段', {
          extensions: { code: 'BAD_USER_INPUT' },
        });
      }

      return db.users.update(id, updateData);
    },

    createProduct: async (_, { input }, context) => {
      return await db.transaction(async (trx) => {
        // 处理分类
        let categoryId;
        if (input.category.id) {
          categoryId = input.category.id;
        } else {
          const cat = await trx.categories.create({ name: input.category.name });
          categoryId = cat.id;
        }

        // 创建商品和变体
        const product = await trx.products.create({
          ...input,
          categoryId,
        });

        if (input.variants?.length) {
          await trx.variants.bulkCreate(
            input.variants.map((v) => ({ ...v, productId: product.id }))
          );
        }

        return product;
      });
    },
  },
};
```

## 七、注意事项

1. **输入类型不能引用对象类型**
2. **输入类型不能有参数**
3. **更新 Input 字段都用可选**
4. **默认值要合理设置**
5. **嵌套不要过深**
6. **Resolver 中过滤 undefined 字段**，避免覆盖已有数据
7. **复杂输入考虑拆分为多个 Mutation**
