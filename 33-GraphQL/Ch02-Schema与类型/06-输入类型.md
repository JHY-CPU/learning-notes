# 输入类型

## 一、基本定义

```graphql
# 输入类型 - 用于 Mutation 参数
input CreateUserInput {
  name: String!
  email: String!
  password: String!
  role: UserRole = CUSTOMER
  profile: CreateProfileInput
}

input CreateProfileInput {
  avatar: String
  bio: String
  phone: String
  address: AddressInput
}

input AddressInput {
  street: String!
  city: String!
  province: String!
  zipCode: String!
  country: String! = "CN"
}

input UpdateUserInput {
  name: String                    # 可选 - 部分更新
  email: String
  profile: UpdateProfileInput
}

# Mutation 使用
type Mutation {
  createUser(input: CreateUserInput!): User!
  updateUser(id: ID!, input: UpdateUserInput!): User!
}
```

## 二、输入类型 vs 对象类型

```graphql
# ✗ 错误 - 对象类型不能用作输入
type Address {
  street: String!
  city: String!
}

type Mutation {
  createUser(address: Address): User!  # ✗ Address 是对象类型
}

# ✓ 正确 - 输入类型用于输入
input AddressInput {
  street: String!
  city: String!
}

type Mutation {
  createUser(address: AddressInput!): User!  # ✓
}

# ✓ 输出用对象类型
type User {
  address: Address!              # ✓ 输出用对象类型
}
```

## 三、批量操作输入

```graphql
input BatchUserInput {
  users: [CreateUserInput!]!
}

input BulkOperationInput {
  create: [CreateUserInput!]
  update: [UpdateUserInput!]
  delete: [ID!]
}

type Mutation {
  batchCreateUsers(input: BatchUserInput!): BatchResult!
  bulkUserOperations(input: BulkOperationInput!): BulkResult!
}

type BatchResult {
  successCount: Int!
  failedCount: Int!
  errors: [BatchError!]!
}

type BatchError {
  index: Int!
  message: String!
}
```

## 四、注意事项

1. **输入类型不能引用对象类型**
2. **输入类型不能有参数**
3. **更新 Input 字段都用可选**
4. **默认值要合理设置**
5. **嵌套不要过深**
