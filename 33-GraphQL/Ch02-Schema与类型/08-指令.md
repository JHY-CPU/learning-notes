# 指令 (Directives)

## 一、内置指令

```graphql
# @deprecated - 标记废弃
type User {
  id: ID!
  name: String!
  oldName: String @deprecated(reason: "使用 name 字段")
}

enum OrderStatus {
  CREATED
  PAID
  SHIPPED
  CANCELLED
  REFUND_PENDING @deprecated(reason: "使用 REFUNDING")
  REFUNDING
}

# @skip - 跳过字段
query GetUser($skipAvatar: Boolean!) {
  user(id: "1") {
    name
    avatar @skip(if: $skipAvatar)
  }
}

# @include - 包含字段
query GetUser($includeOrders: Boolean!) {
  user(id: "1") {
    name
    orders @include(if: $includeOrders) {
      id
      total
    }
  }
}
```

## 二、自定义指令

```graphql
# 定义自定义指令
directive @auth(requires: Role!) on FIELD_DEFINITION
directive @rateLimit(max: Int!, window: String!) on FIELD_DEFINITION
directive @cacheControl(maxAge: Int!) on FIELD_DEFINITION | OBJECT
directive @validate(pattern: String!) on ARGUMENT_DEFINITION

# 使用
type Query {
  users: [User!]! @auth(requires: ADMIN) @rateLimit(max: 100, window: "1m")
  me: User! @auth(requires: CUSTOMER)
  publicProducts: [Product!]! @cacheControl(maxAge: 300)
}

type User @cacheControl(maxAge: 60) {
  id: ID!
  name: String!
  ssn: String! @auth(requires: ADMIN)
}
```

## 三、指令实现

```java
// 自定义指令 - 字段授权
public class AuthDirective implements SchemaDirectiveWiring {

    @Override
    public GraphQLFieldDefinition onField(
            SchemaDirectiveWiringEnvironment<GraphQLFieldDefinition> env) {
        Role requiredRole = (Role) env.getDirective().getArgument("requires").getValue();

        return env.setField(env.getField().transform(builder ->
            builder.dataFetcher(env.wrapDataFetcher(env.getField().getDataFetcher(),
                (dataFetchingEnvironment, value) -> {
                    // 检查权限
                    SecurityContext ctx = dataFetchingEnvironment
                        .getContext();
                    if (!ctx.hasRole(requiredRole)) {
                        throw new AccessDeniedException(
                            "需要 " + requiredRole + " 权限");
                    }
                    return value;
                }))));
    }
}
```

## 四、注意事项

1. **@deprecated 是最重要的内置指令**
2. **客户端指令 (@skip/@include) 由执行引擎处理**
3. **自定义指令需要 Schema 和 Runtime 两处实现**
4. **指令位置要限定合理**（FIELD_DEFINITION, ARGUMENT 等）
5. **不要滥用指令**，逻辑复杂时用中间件替代
