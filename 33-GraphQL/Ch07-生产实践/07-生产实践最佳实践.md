# 生产实践最佳实践

## 一、实践清单

```yaml
部署:
  □ 多实例水平扩展
  □ 健康检查配置
  □ 优雅关闭
  □ 蓝绿/金丝雀发布
  □ 自动扩缩容

安全:
  □ HTTPS 强制
  □ 查询深度/复杂度限制
  □ 认证授权集成
  □ 生产关闭 Playground/自省
  □ 输入验证

性能:
  □ DataLoader 配置
  □ 缓存策略
  □ APQ 启用
  □ 查询优化
  □ 慢查询监控

可观测:
  □ 指标采集
  □ 链路追踪
  □ 结构化日志
  □ 告警配置
  □ Apollo Studio
```

## 二、架构检查清单

| 检查项 | 说明 | 优先级 |
|--------|------|--------|
| HTTPS | 传输加密 | 必须 |
| 查询限制 | 深度 + 复杂度 | 必须 |
| DataLoader | N+1 解决 | 必须 |
| 缓存策略 | 响应 + 字段 | 建议 |
| 认证授权 | JWT/指令 | 必须 |
| 监控告警 | 指标 + 日志 | 必须 |
| 自省关闭 | 生产安全 | 必须 |
| APQ | 查询优化 | 建议 |

## 三、生产配置模板

```yaml
# 生产环境配置
server:
  port: 4000
  cors:
    origins: ["https://app.example.com"]
    credentials: true

graphql:
  playground: false
  introspection: false
  maxDepth: 10
  maxComplexity: 1000

cache:
  responseCache:
    enabled: true
    ttl: 60
  persistedQueries:
    enabled: true
    ttl: 900000

auth:
  jwt:
    secret: ${JWT_SECRET}
    expiresIn: 24h

monitoring:
  metrics:
    enabled: true
    endpoint: /metrics
  tracing:
    enabled: true
    exporter: otlp
```

## 四、注意事项

1. **安全策略是第一优先级**
2. **监控是生产的眼睛**
3. **性能优化要数据驱动**
4. **Schema 变更要规范管理**
5. **持续迭代，保持技术债务可控**
