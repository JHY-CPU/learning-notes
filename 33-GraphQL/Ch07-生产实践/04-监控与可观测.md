# 监控与可观测

## 一、指标采集

```typescript
// Apollo Server 指标插件
import { ApolloServerPluginUsageReporting } from '@apollo/server/plugin/usageReporting';

const server = new ApolloServer({
  schema,
  plugins: [
    // 上报到 Apollo Studio
    ApolloServerPluginUsageReporting({
      sendVariableValues: { none: true },  // 不上报变量值
      sendHeaders: { none: true },
    }),

    // 自定义指标
    {
      async requestDidStart() {
        const start = Date.now();
        return {
          async didResolveOperation(ctx) {
            metrics.increment('graphql.operation', {
              operation: ctx.request.operationName,
              type: ctx.operation.operation,
            });
          },
          async didEncounterErrors(ctx) {
            metrics.increment('graphql.error', {
              operation: ctx.request.operationName,
            });
          },
          async willSendResponse(ctx) {
            const duration = Date.now() - start;
            metrics.histogram('graphql.duration', duration, {
              operation: ctx.request.operationName,
            });
          },
        };
      },
    },
  ],
});
```

集成方式:

## 三、日志记录

```yaml
日志内容:
  请求日志:
    - operationName
    - 查询哈希
    - 用户 ID
    - 响应时间

  错误日志:
    - 错误类型
    - 错误消息
    - 查询路径
    - 堆栈信息

  性能日志:
    - 慢查询 (> 1s)
    - 高复杂度查询
    - 高频查询
```

## 四、注意事项

1. **不要在指标中泄露敏感数据**
2. **日志要结构化 (JSON)**
3. **慢查询要告警**
4. **Apollo Studio 可以做 Schema 分析**
5. **OpenTelemetry 是行业标准**
