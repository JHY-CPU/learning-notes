# 版本管理

## 一、Schema 版本策略

```yaml
版本管理方式:
  演进式 (推荐):
    - 无需版本号
    - 新增字段代替删除
    - @deprecated 标记废弃
    - 客户端自行迁移到新字段

  版本化 (不推荐):
    - /graphql/v1
    - /graphql/v2
    - 维护成本高
```

## 二、安全演进

```graphql
# 1. 标记废弃
type User {
  id: ID!
  name: String!
  username: String @deprecated(reason: "使用 handle 字段")
  handle: String!
}

# 2. 新增字段
type User {
  id: ID!
  name: String!
  displayName: String!      # 新增
}

# 3. 放宽类型
type User {
  bio: String               # 从 String! 放宽
}

# 4. 新增枚举值
enum OrderStatus {
  CREATED
  PAID
  REFUNDING                 # 新增
}
```

## 三、变更管理流程

```yaml
变更流程:
  1. 提出变更:
     - 修改 SDL
     - 说明影响范围
     - 通知前端团队

  2. 兼容性检查:
     - 运行 Schema 比较
     - 识别破坏性变更
     - 评估客户端影响

  3. 灰度发布:
     - 新字段可选上线
     - @deprecated 标记旧字段
     - 监控旧字段使用率

  4. 迁移完成:
     - 使用率降至 0
     - 移除废弃字段
     - 更新文档
```

## 五、Schema Registry 工作流

```yaml
Apollo Schema Registry 工作流:
  1. 开发者修改本地 Schema
  2. CI 运行 graphql-inspector diff
  3. 检测破坏性变更 → 失败 / 非破坏性 → 继续
  4. 注册新 Schema 到 Registry
  5. Registry 验证与已注册 Schema 的兼容性
  6. 通知受影响的客户端团队
  7. 合并代码，自动部署
```

```javascript
// CI 集成脚本
const { diff } = require('@graphql-inspector/core');
const { loadSchema } = require('@graphql-tools/load');

async function checkSchemaChanges() {
  const oldSchema = await loadSchema('schema-old.graphql');
  const newSchema = await loadSchema('schema-new.graphql');

  const changes = diff(oldSchema, newSchema);

  const breakingChanges = changes.filter(
    (c) => c.criticality.level === 'BREAKING'
  );

  if (breakingChanges.length > 0) {
    console.error('检测到破坏性变更:');
    breakingChanges.forEach((c) => {
      console.error(`  - ${c.message}`);
    });
    process.exit(1);
  }

  console.log('Schema 变更检查通过');
}
```

## 六、多版本并存策略

```graphql
# 不推荐但有时必要: 字段版本化
type User {
  id: ID!
  name: String!                    # v1: 合并的 name
  firstName: String!               # v2: 拆分的 firstName
  lastName: String!                # v2: 拆分的 lastName
}

# Resolver 中兼容
const resolvers = {
  User: {
    name: (parent) => {
      return parent.name || `${parent.firstName} ${parent.lastName}`;
    },
  },
};

# 最终废弃旧字段
type User {
  id: ID!
  name: String! @deprecated(reason: "使用 firstName + lastName")
  firstName: String!
  lastName: String!
}
```

## 七、注意事项

1. **演进式是 GraphQL 的核心优势**
2. **废弃字段要给足够迁移时间**
3. **使用率监控决定何时移除**
4. **破坏性变更要避免**
5. **Schema Registry 帮助管理变更**
6. **CI 中集成 Schema 兼容性检查**
7. **多版本并存只是过渡方案**，最终要统一
