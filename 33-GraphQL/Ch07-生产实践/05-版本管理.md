# 版本管理

## 一、Schema 版本策略

```yaml
版本管理方式:
  演进式 (推荐):
    - 无需版本号
    - 新增字段代替删除
    - @deprecated 标记废弃
    - 客户端自行迁移到新字段

  版本化 (不推荐):
    - /graphql/v1
    - /graphql/v2
    - 维护成本高
```

## 二、安全演进

```graphql
# 1. 标记废弃
type User {
  id: ID!
  name: String!
  username: String @deprecated(reason: "使用 handle 字段")
  handle: String!
}

# 2. 新增字段
type User {
  id: ID!
  name: String!
  displayName: String!      # 新增
}

# 3. 放宽类型
type User {
  bio: String               # 从 String! 放宽
}

# 4. 新增枚举值
enum OrderStatus {
  CREATED
  PAID
  REFUNDING                 # 新增
}
```

## 三、变更管理流程

```yaml
变更流程:
  1. 提出变更:
     - 修改 SDL
     - 说明影响范围
     - 通知前端团队

  2. 兼容性检查:
     - 运行 Schema 比较
     - 识别破坏性变更
     - 评估客户端影响

  3. 灰度发布:
     - 新字段可选上线
     - @deprecated 标记旧字段
     - 监控旧字段使用率

  4. 迁移完成:
     - 使用率降至 0
     - 移除废弃字段
     - 更新文档
```

## 四、工具支持

```yaml
工具:
  Apollo Schema Registry:
    - Schema 版本管理
    - 变更检测
    - 影响分析
    - 客户端使用统计

  GraphQL Inspector:
    - Schema Diff
    - Breaking Change 检测
    - CI 集成

  graphql-inspect:
    npx graphql-inspect schema:schema1 schema:schema2
```

## 五、注意事项

1. **演进式是 GraphQL 的核心优势**
2. **废弃字段要给足够迁移时间**
3. **使用率监控决定何时移除**
4. **破坏性变更要避免**
5. **Schema Registry 帮助管理变更**
