# 性能优化

## 一、服务端优化

```yaml
优化策略:
  解析优化:
    - 查询缓存 (APQ)
    - JIT 编译 (Mercurius)
    - 减少 AST 遍历

  数据获取:
    - DataLoader 批量加载
    - 数据库查询优化
    - 连接池配置
    - 读写分离

  缓存:
    - 响应缓存
    - 字段级缓存
    - DataLoader 缓存
    - Redis 缓存
```

```typescript
// Mercurius JIT 编译优化
import Fastify from 'fastify';
import mercurius from 'mercurius';

const app = Fastify();
app.register(mercurius, {
  schema,
  resolvers,
  jit: 1,              // JIT 编译阈值
  queryDepth: 10,      // 查询深度限制
  cache: 1000,         // 查询缓存数量
});
```

## 二、查询优化

```graphql
# ✗ 低效查询 - N+1 问题
query {
  users {              # SELECT * FROM users (1次)
    name
    orders {           # SELECT * FROM orders WHERE user_id = ? (N次)
      id
    }
  }
}

# ✓ 优化后 - DataLoader 批量加载
query {
  users {
    name
    orders {           # SELECT * FROM orders WHERE user_id IN (...) (1次)
      id
    }
  }
}
```

## 三、监控指标

```yaml
关键指标:
  吞吐量:
    - QPS (查询/秒)
    - 目标: > 1000

  延迟:
    - P50 < 50ms
    - P95 < 200ms
    - P99 < 500ms

  错误率:
    - 目标: < 0.1%

  资源:
    - CPU < 70%
    - 内存 < 80%
    - 连接池 < 80%
```

## 四、注意事项

1. **DataLoader 是最关键的优化**
2. **查询限制必须配置**
3. **APQ 减少重复解析**
4. **监控慢查询并持续优化**
5. **基准测试量化优化效果**
