# 性能优化

## 一、服务端优化

```yaml
优化策略:
  解析优化:
    - 查询缓存 (APQ)
    - JIT 编译 (Mercurius)
    - 减少 AST 遍历

  数据获取:
    - DataLoader 批量加载
    - 数据库查询优化
    - 连接池配置
    - 读写分离

  缓存:
    - 响应缓存
    - 字段级缓存
    - DataLoader 缓存
    - Redis 缓存
```

```typescript
// Mercurius JIT 编译优化
import Fastify from 'fastify';
import mercurius from 'mercurius';

const app = Fastify();
app.register(mercurius, {
  schema,
  resolvers,
  jit: 1,              // JIT 编译阈值
  queryDepth: 10,      // 查询深度限制
  cache: 1000,         // 查询缓存数量
});
```

## 二、查询优化

```graphql
# ✗ 低效查询 - N+1 问题
query {
  users {              # SELECT * FROM users (1次)
    name
    orders {           # SELECT * FROM orders WHERE user_id = ? (N次)
      id
    }
  }
}

# ✓ 优化后 - DataLoader 批量加载
query {
  users {
    name
    orders {           # SELECT * FROM orders WHERE user_id IN (...) (1次)
      id
    }
  }
}
```

## 四、APQ (Automatic Persisted Queries)

```typescript
// Apollo Server APQ
import { ApolloServer } from '@apollo/server';
import { ApolloServerPluginCacheControl } from '@apollo/server/plugin/cacheControl';

const server = new ApolloServer({
  schema,
  plugins: [
    ApolloServerPluginCacheControl({ defaultMaxAge: 60 }),
  ],
  persistedQueries: {
    cache: new Map(),   // 或用 Redis
  },
});

// 客户端启用 APQ
import { createPersistedLink } from '@apollo/client/link/persisted-queries';
import { createHttpLink } from '@apollo/client';
import { sha256 } from 'crypto-hash';

const persistedLink = createPersistedLink({ sha256 });
const httpLink = createHttpLink({ uri: '/graphql' });

const client = new ApolloClient({
  link: persistedLink.concat(httpLink),
  cache: new InMemoryCache(),
});

// APQ 流程:
// 1. 首次: GET /graphql?extensions={"persistedQuery":{"sha256Hash":"xxx"}}
//    响应: PersistedQueryNotFound
// 2. 重试: GET /graphql?extensions={...}&query={user{name}}
//    响应: 数据，并缓存查询
// 3. 后续: 只传 hash，命中缓存
//    响应: 数据
//    好处: GET 请求可被 CDN 缓存
```

## 五、响应缓存

```typescript
// Apollo Server 响应缓存
import { ApolloServerPluginResponseCache } from '@apollo/server/plugin/responseCache';

const server = new ApolloServer({
  schema,
  plugins: [
    ApolloServerPluginResponseCache({
      // 缓存键中包含用户信息
      sessionID: (contextValue) => {
        return contextValue.currentUser?.id || 'anonymous';
      },
      // 忽略某些请求头
      ignoreHeaders: ['authorization'],
      // 超时后额外缓存时间
      staleIfRevalidate: 30,
    }),
  ],
});

// Schema 中声明缓存
type Query {
  products: [Product!]! @cacheControl(maxAge: 300)
  user(id: ID!): User! @cacheControl(maxAge: 60, scope: PRIVATE)
  publicData: Data! @cacheControl(maxAge: 3600, scope: PUBLIC)
}
```

## 六、数据库优化

```yaml
GraphQL 数据库优化:
  查询优化:
    - 避免 SELECT *，只查所需字段
    - 批量查询替代 N+1
    - 合理使用索引
    - 分页查询控制数据量

  连接池:
    - 设置合理池大小
    - 连接超时配置
    - 连接健康检查

  读写分离:
    - 读操作走从库
    - 写操作走主库
    - DataLoader 合并读请求
```

## 七、注意事项

1. **DataLoader 是最关键的优化**
2. **查询限制必须配置**
3. **APQ 减少重复解析**
4. **监控慢查询并持续优化**
5. **基准测试量化优化效果**
6. **响应缓存减少数据库压力**
7. **@cacheControl 声明字段级缓存策略**
