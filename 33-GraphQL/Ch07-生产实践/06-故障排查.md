# 故障排查

## 一、常见问题

```yaml
N+1 查询:
  症状: 数据库查询过多
  排查: 查看 SQL 日志
  解决: DataLoader 批量加载

查询超时:
  症状: 请求响应慢
  排查: 链路追踪定位慢字段
  解决: 优化查询 + 缓存

内存泄漏:
  症状: 内存持续增长
  排查: heap dump 分析
  解决: 检查 DataLoader 和缓存

连接池耗尽:
  症状: 数据库连接超时
  排查: 连接池监控
  解决: 增大连接池 / 优化查询
```

## 三、性能分析

```graphql
# 自省查询执行计划
{
  __schema {
    queryType {
      fields {
        name
        type { name }
      }
    }
  }
}
```

```yaml
排查步骤:
  1. 识别慢查询:
     - 查看 APM 监控
     - 找到 P99 > 1s 的操作

  2. 分析查询:
     - 字段级耗时
     - N+1 问题检查
     - 数据库慢查询

  3. 优化方案:
     - DataLoader
     - 查询缓存
     - 索引优化
     - 字段级缓存

  4. 验证效果:
     - 压测对比
     - 监控持续观察
```

## 四、注意事项

1. **日志要结构化，便于查询**
2. **APM 集成到开发和生产环境**
3. **慢查询要持续监控和优化**
4. **故障要有回滚方案**
5. **定期做性能基准测试**
