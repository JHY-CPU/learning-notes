# 查询验证

## 一、验证规则

```yaml
GraphQL 验证阶段:
  语法验证:
    - 查询语法是否合法
    - Schema 语法是否正确

  类型验证:
    - 字段是否存在
    - 参数类型是否匹配
    - 返回值类型是否正确

  语义验证:
    - 片段类型是否正确
    - 变量使用是否合法
    - 指令参数是否有效
```

## 二、常见验证错误

```graphql
# 1. 字段不存在
query {
  user(id: "1") {
    namme          # ✗ 拼写错误
  }
}

# 2. 参数类型不匹配
query {
  user(id: 123) {  # ✗ ID 应该是 String
    name
  }
}

# 3. 缺少必填参数
query {
  user {           # ✗ 缺少 id 参数
    name
  }
}

# 4. 片段类型错误
fragment UserFields on User {
  name
  price            # ✗ User 类型没有 price 字段
}

# 5. 变量类型不匹配
query($id: Int!) {
  user(id: $id) {  # ✗ user(id:) 参数是 ID! 不是 Int!
    name
  }
}
```

## 三、自定义验证规则

```java
// graphql-java 自定义验证规则
public class CustomValidationRule extends AbstractRule {

    @Override
    public void checkField(Field field, ValidationContext context) {
        // 检查废弃字段使用
        GraphQLFieldDefinition fieldDef = context.getFieldDef();
        if (fieldDef.getDirective("deprecated") != null) {
            context.addError(GraphqlError.newError()
                .message("使用了废弃字段: " + field.getName())
                .location(field.getSourceLocation())
                .build());
        }
    }
}

// 添加到验证器
GraphQL graphQL = GraphQL.newGraphQL(schema)
    .additionalValidationRules(List.of(
        new CustomValidationRule(),
        new MaxDepthRule(10),
        new MaxComplexityRule(1000)
    ))
    .build();
```

## 四、注意事项

1. **验证在执行之前**，减少无效执行
2. **错误信息要准确**，帮助开发者修复
3. **自定义验证可以扩展安全性**
4. **CI 中要验证 Schema 的合法性**
5. **客户端可以用 eslint-plugin-graphql 验证**
