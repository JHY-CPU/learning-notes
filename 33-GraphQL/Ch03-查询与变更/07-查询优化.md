# 查询优化

## 一、DataLoader 批处理

```java
// 解决 N+1 问题的核心方案
@Component
public class DataLoaderRegistryFactory {

    public DataLoaderRegistry create(DataFetchingEnvironment env) {
        DataLoaderRegistry registry = new DataLoaderRegistry();

        // 用户批量加载
        registry.register("users", DataLoaderFactory.newDataLoader(
            (List<String> ids) -> CompletableFuture.supplyAsync(() ->
                userService.findByIds(ids))
        ));

        // 订单批量加载 (按用户)
        registry.register("ordersByUser", DataLoaderFactory.newDataLoader(
            (List<String> userIds) -> CompletableFuture.supplyAsync(() -> {
                Map<String, List<Order>> ordersMap = orderService
                    .findByUserIds(userIds)
                    .stream()
                    .collect(Collectors.groupingBy(Order::getUserId));
                return userIds.stream()
                    .map(id -> ordersMap.getOrDefault(id, List.of()))
                    .collect(Collectors.toList());
            })
        ));

        return registry;
    }
}
```

## 二、查询深度限制

```yaml
Apollo Server:
  plugins:
    - depthLimit(10)           # 最大深度 10 层

graphql-java:
  maxQueryDepth: 15            # 配置文件
```

```graphql
# 被拒绝的过深查询
query {
  user {                    # depth 1
    orders {                # depth 2
      items {               # depth 3
        product {           # depth 4
          category {        # depth 5
            parent {        # depth 6
              # ... 继续嵌套会触发限制
            }
          }
        }
      }
    }
  }
}
```

```yaml
复杂度计算规则:

  列表字段: cost = items * childCost
  总复杂度: sum of all field costs

限制阈值:
  普通用户: maxComplexity = 500
  VIP 用户: maxComplexity = 2000
  内部系统: maxComplexity = 10000
```

## 四、持久化查询

```yaml
APQ (Automatic Persisted Queries):
  流程:
    1. 客户端发送查询哈希
    2. 服务端未找到 → 客户端发送完整查询
    3. 服务端缓存查询
    4. 后续只发哈希

  优点:
    - 减少请求体积
    - CDN 可缓存 GET 请求
    - 防止恶意查询

  Apollo Client:
    persistedQueries:
      ttl: 900000            # 缓存 15 分钟
```

## 五、注意事项

1. **DataLoader 是解决 N+1 的标配**
2. **查询深度和复杂度必须限制**
3. **APQ 适合高频重复查询**
4. **监控慢查询并优化**
5. **合理使用字段级缓存**
