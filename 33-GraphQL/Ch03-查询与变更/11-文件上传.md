# 文件上传

## 一、GraphQL Multipart Request

```graphql
# Schema 定义
scalar Upload

type Mutation {
  uploadAvatar(file: Upload!): User!
  uploadFiles(files: [Upload!]!): [File!]!
}
```

```javascript
// Apollo Server + graphql-upload
import { graphqlUploadExpress } from 'graphql-upload';

app.use(graphqlUploadExpress({ maxFileSize: 10_000_000, maxFiles: 5 }));

const resolvers = {
  Mutation: {
    uploadAvatar: async (_, { file }) => {
      const { createReadStream, filename, mimetype } = await file;
      const stream = createReadStream();
      // 上传到 OSS
      const url = await oss.upload(stream, filename);
      return userService.updateAvatar(userId, url);
    },
  },
};
```

## 二、客户端上传

```javascript
// Apollo Client 上传
import { createUploadLink } from 'apollo-upload-client';

const client = new ApolloClient({
  link: createUploadLink({ uri: '/graphql' }),
  cache: new InMemoryCache(),
});

// 使用
const UPLOAD = gql`
  mutation UploadAvatar($file: Upload!) {
    uploadAvatar(file: $file) {
      id
      avatar
    }
  }
`;

function AvatarUpload() {
  const [upload] = useMutation(UPLOAD);

  const handleChange = async (e) => {
    const file = e.target.files[0];
    await upload({ variables: { file } });
  };

  return <input type="file" onChange={handleChange} />;
}
```

## 三、替代方案: 预签名 URL

```graphql
# 方案: 先获取上传 URL，再直传
type Mutation {
  getUploadUrl(filename: String!, contentType: String!): UploadUrl!
}

type UploadUrl {
  url: String!
  fields: JSON       # POST 表单字段
  expiresAt: DateTime!
}

# 流程:
# 1. 调用 getUploadUrl 获取签名 URL
# 2. 客户端直接上传到 OSS
# 3. 上传完成后提交文件 URL
```

## 四、文件验证与安全

```javascript
const resolvers = {
  Mutation: {
    uploadAvatar: async (_, { file }, context) => {
      const { createReadStream, filename, mimetype, encoding } = await file;

      // 文件类型白名单
      const allowedTypes = ['image/jpeg', 'image/png', 'image/webp'];
      if (!allowedTypes.includes(mimetype)) {
        throw new GraphQLError('不支持的文件类型', {
          extensions: { code: 'INVALID_FILE_TYPE' },
        });
      }

      // 文件大小限制
      const maxSize = 5 * 1024 * 1024; // 5MB
      let size = 0;
      const stream = createReadStream();

      stream.on('data', (chunk) => {
        size += chunk.length;
        if (size > maxSize) {
          stream.destroy();
          throw new GraphQLError('文件过大', {
            extensions: { code: 'FILE_TOO_LARGE' },
          });
        }
      });

      // 文件名安全处理
      const safeFilename = filename.replace(/[^a-zA-Z0-9.-]/g, '_');
      const key = `avatars/${context.currentUser.id}/${Date.now()}-${safeFilename}`;

      // 上传到 OSS
      const url = await oss.putObject({
        Bucket: 'my-bucket',
        Key: key,
        Body: stream,
        ContentType: mimetype,
      });

      // 更新用户头像
      return db.users.update(context.currentUser.id, { avatar: url });
    },
  },
};
```

## 五、多文件上传

```graphql
# Schema
type Mutation {
  uploadImages(files: [Upload!]!): [Image!]!
}

type Image {
  id: ID!
  url: String!
  filename: String!
  size: Int!
  mimeType: String!
}
```

```javascript
// 客户端多文件上传
function MultiUpload() {
  const [uploadImages] = useMutation(gql`
    mutation UploadImages($files: [Upload!]!) {
      uploadImages(files: $files) {
        id
        url
        filename
      }
    }
  `);

  const handleUpload = async (e) => {
    const files = Array.from(e.target.files);
    await uploadImages({ variables: { files } });
  };

  return <input type="file" multiple onChange={handleUpload} />;
}
```

## 六、注意事项

1. **GraphQL 上传用 multipart/form-data**，不是 JSON
2. **限制文件大小和类型**
3. **大文件建议用预签名 URL**
4. **文件上传走独立的存储服务**
5. **病毒扫描要集成**
6. **文件名要做安全处理**，防止路径穿越
7. **流式处理大文件**，避免内存溢出
