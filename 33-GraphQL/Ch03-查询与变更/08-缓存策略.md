# 缓存策略

## 一、Normalized Cache

```javascript
// Apollo Client 标准化缓存
const cache = new InMemoryCache({
  typePolicies: {
    Query: {
      fields: {
        // 合并策略
        users: {
          keyArgs: ["filter"],
          merge(existing = { edges: [] }, incoming) {
            return {
              ...incoming,
              edges: [...existing.edges, ...incoming.edges],
            };
          },
        },
      },
    },
    User: {
      keyFields: ["id"],    // 用 id 做缓存 key
    },
  },
});
```

## 二、响应缓存

```javascript
// Apollo Server Response Cache
import { ApolloServerPluginResponseCache } from '@apollo/server/plugin/responseCache';

const server = new ApolloServer({
  schema,
  plugins: [
    ApolloServerPluginResponseCache({
      sessionId: (requestContext) =>
        requestContext.request.http.headers.get('session-id') || null,
      shouldReadFromCache: ({ requestContext }) =>
        requestContext.request.http.method === 'GET',
    }),
  ],
});

# Schema 中指定缓存时间
type Query {
  products: [Product!]! @cacheControl(maxAge: 300)
  user(id: ID!): User! @cacheControl(maxAge: 60, scope: PRIVATE)
}
```

## 三、持久化查询缓存

```yaml
# APQ + CDN 缓存
Apollo Client:
  link: HttpLink({
    uri: '/graphql',
    useGETForQueries: true,    # 查询用 GET 请求
  })
  persistedQueries:
    ttl: 900000

# GET 请求可以被 CDN 缓存
GET /graphql?query=hash123&variables={"id":"1"}
```

## 四、缓存层级

```yaml
缓存架构:
  客户端缓存:
    - Apollo Normalized Cache
    - urql Document Caching
    - Relay Store

  服务端缓存:
    - DataLoader 内存缓存
    - Redis 缓存
    - Response Cache

  网络缓存:
    - CDN 缓存 (GET + APQ)
    - HTTP Cache-Control
```

## 五、注意事项

1. **标准化缓存是 Apollo Client 的核心优势**
2. **私有数据不要 CDN 缓存**
3. **缓存失效要配合业务逻辑**
4. **keyFields 定义要准确**
5. **缓存粒度和性能要平衡**
