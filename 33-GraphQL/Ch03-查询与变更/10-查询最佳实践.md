# 查询最佳实践

## 一、实践清单

```yaml
查询设计:
  □ 使用命名查询
  □ 变量类型完整声明
  □ 复用片段
  □ 控制嵌套深度
  □ 避免 N+1 查询

变更设计:
  □ Input 类型包装参数
  □ 返回更新后对象
  □ 支持部分更新
  □ 批量操作支持
  □ 错误信息完整

安全:
  □ 查询深度限制
  □ 复杂度限制
  □ 速率限制
  □ 认证授权
  □ 生产关闭自省
```

## 二、查询模式

```graphql
# ✓ 好的查询 - 命名 + 变量 + 片段
fragment UserBasic on User {
  id
  name
  avatar
}

query GetUsers($limit: Int!, $offset: Int!) {
  users(first: $limit, after: $offset) {
    edges {
      node {
        ...UserBasic
        email
      }
    }
    pageInfo {
      hasNextPage
      endCursor
    }
  }
}

# ✗ 不好的查询 - 匿名 + 硬编码 + 无片段
{
  users(first: 10, after: "abc") {
    edges {
      node {
        id
        name
        avatar
        email
      }
    }
  }
}
```

## 三、错误处理模式

```graphql
# ✓ 使用 payload 模式
mutation UpdateUser($id: ID!, $input: UpdateUserInput!) {
  updateUser(id: $id, input: $input) {
    user {
      id
      name
    }
    errors {
      field
      message
      code
    }
  }
}

# 返回
{
  "data": {
    "updateUser": {
      "user": null,
      "errors": [
        { "field": ["email"], "message": "邮箱已被使用", "code": "DUPLICATE_EMAIL" }
      ]
    }
  }
}
```

## 四、注意事项

1. **查询要与后端团队协商**
2. **定期审查高频查询**
3. **监控查询性能**
4. **文档化常用查询模式**
5. **Codegen 生成类型安全的客户端代码**
