# 查询最佳实践

## 一、实践清单

```yaml
查询设计:
  □ 使用命名查询
  □ 变量类型完整声明
  □ 复用片段
  □ 控制嵌套深度
  □ 避免 N+1 查询

变更设计:
  □ Input 类型包装参数
  □ 返回更新后对象
  □ 支持部分更新
  □ 批量操作支持
  □ 错误信息完整

安全:
  □ 查询深度限制
  □ 复杂度限制
  □ 速率限制
  □ 认证授权
  □ 生产关闭自省
```

## 二、查询模式

```graphql
# ✓ 好的查询 - 命名 + 变量 + 片段
fragment UserBasic on User {
  id
  name
  avatar
}

query GetUsers($limit: Int!, $offset: Int!) {
  users(first: $limit, after: $offset) {
    edges {
      node {
        ...UserBasic
        email
      }
    }
    pageInfo {
      hasNextPage
      endCursor
    }
  }
}

# ✗ 不好的查询 - 匿名 + 硬编码 + 无片段
{
  users(first: 10, after: "abc") {
    edges {
      node {
        id
        name
        avatar
        email
      }
    }
  }
}
```

## 三、错误处理模式


## 四、查询深度与复杂度限制

```javascript
// Apollo Server 查询复杂度限制
import depthLimit from 'graphql-depth-limit';
import { createComplexityLimitRule } from 'graphql-validation-complexity';

const server = new ApolloServer({
  schema,
  validationRules: [
    depthLimit(10),                    // 最大深度 10 层
    createComplexityLimitRule(1000),   // 最大复杂度 1000
  ],
});

// 自定义复杂度计算
import { getComplexity, simpleEstimator, fieldExtensionsEstimator } from 'graphql-query-complexity';

const server = new ApolloServer({
  schema,
  plugins: [
    {
      async requestDidStart() {
        return {
          async didResolveOperation({ request, document }) {
            const complexity = getComplexity({
              schema,
              operationName: request.operationName,
              query: document,
              variables: request.variables,
              estimators: [
                fieldExtensionsEstimator(),
                simpleEstimator({ defaultComplexity: 1 }),
              ],
            });

            if (complexity > 1000) {
              throw new GraphQLError(`查询复杂度过高: ${complexity}/1000`, {
                extensions: { code: 'QUERY_TOO_COMPLEX' },
              });
            }
          },
        };
      },
    },
  ],
});
```

## 五、查询速率限制

```javascript
// 基于 IP/用户的速率限制
import rateLimit from 'express-rate-limit';

const graphqlLimiter = rateLimit({
  windowMs: 60 * 1000,  // 1 分钟
  max: 100,              // 最多 100 次
  message: { errors: [{ message: '请求过于频繁' }] },
  keyGenerator: (req) => {
    return req.user?.id || req.ip;  // 优先用用户 ID
  },
});

app.use('/graphql', graphqlLimiter);
```

## 六、DataLoader 解决 N+1

```javascript
import DataLoader from 'dataloader';

// 每个请求创建新的 DataLoader
function createLoaders() {
  return {
    userById: new DataLoader(async (ids) => {
      const users = await db.users.findByIds(ids);
      const map = new Map(users.map((u) => [u.id, u]));
      return ids.map((id) => map.get(id) || null);
    }),

    ordersByUserId: new DataLoader(async (userIds) => {
      const orders = await db.orders.findByUserIds(userIds);
      return userIds.map((uid) => orders.filter((o) => o.userId === uid));
    }),
  };
}

// Context 中注入
const server = new ApolloServer({
  schema,
  context: () => ({
    loaders: createLoaders(),
  }),
});

// Resolver 中使用
const resolvers = {
  User: {
    orders: (parent, _, { loaders }) => {
      return loaders.ordersByUserId.load(parent.id);
    },
  },
};
```

## 七、注意事项

1. **查询要与后端团队协商**
2. **定期审查高频查询**
3. **监控查询性能**
4. **文档化常用查询模式**
5. **Codegen 生成类型安全的客户端代码**
6. **DataLoader 是解决 N+1 的关键**
7. **复杂度和深度限制必须配置**
