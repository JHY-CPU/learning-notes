# 查询与变更最佳实践

## 一、实践总结

```yaml
查询 (Query):
  设计:
    - 命名查询，方便追踪
    - 变量传参，提高复用
    - 片片复用，减少重复

  性能:
    - DataLoader 解决 N+1
    - 深度/复杂度限制
    - 分页查询控制数据量
    - 缓存策略优化

变更 (Mutation):
  设计:
    - Input 类型包装参数
    - 返回更新后完整对象
    - Payload 模式返回错误
    - 支持部分更新

  安全:
    - 输入验证
    - 权限检查
    - 操作审计日志
    - 幂等性保证

订阅 (Subscription):
  设计:
    - 按条件过滤事件
    - WebSocket 连接鉴权
    - 断线重连处理
    - PubSub 跨实例同步
```

## 二、常见反模式

```yaml
反模式:
  查询:
    - 匿名查询
    - 硬编码参数
    - 过深嵌套 (无限制)
    - N+1 查询
    - SELECT *

  变更:
    - 返回 Boolean
    - 无输入验证
    - 无错误处理
    - 大批量无分页
```

## 三、检查清单

| 检查项 | 说明 | 状态 |
|--------|------|------|
| 命名查询 | 便于监控和调试 | 必须 |
| DataLoader | 解决 N+1 问题 | 必须 |
| 查询限制 | 深度 + 复杂度 | 必须 |
| Input 类型 | 包装 Mutation 参数 | 必须 |
| 返回完整对象 | Mutation 后便于缓存 | 建议 |
| 错误 Payload | 规范错误返回 | 建议 |
| 审计日志 | 敏感操作记录 | 必须 |

## 四、注意事项

1. **查询和变更的规范要团队统一**
2. **Codegen 保证前后端类型一致**
3. **性能优化要数据驱动**
4. **安全策略要前置到 Schema 设计**
5. **持续监控和优化查询性能**
