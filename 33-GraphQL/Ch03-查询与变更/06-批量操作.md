# 批量操作

## 一、批量查询

```graphql
# 并行查询 (同一次请求)
query BatchQuery {
  users: users(first: 10) {
    edges { node { id name } }
  }
  products: products(first: 10) {
    edges { node { id name price } }
  }
  orders: orders(first: 10) {
    edges { node { id total status } }
  }
}

# 多个命名查询
query GetUser($id: ID!) {
  user(id: $id) { id name }
}

query GetOrders($userId: ID!) {
  orders(userId: $userId) { id total }
}

# 客户端发送多个查询
[
  { query: "query GetUser($id: ID!) { ... }", variables: { id: "1" } },
  { query: "query GetOrders($userId: ID!) { ... }", variables: { userId: "1" } }
]
```

## 二、批量变更

```graphql
# 批量创建
mutation BatchCreate($inputs: [CreateUserInput!]!) {
  batchCreateUsers(inputs: $inputs) {
    successCount
    failedCount
    results {
      success
      user { id name }
      error { code message }
    }
  }
}

# 混合操作
mutation BulkOperations {
  createUser: createUser(input: { name: "张三" }) {
    id name
  }
  updateUser: updateUser(id: "2", input: { name: "李四" }) {
    id name
  }
  deleteUser: deleteUser(id: "3") {
    success
  }
}
```

## 三、批量查询优化

```graphql
# Apollo Batch Link
import { BatchHttpLink } from '@apollo/client/link/batch';

const link = new BatchHttpLink({
  uri: '/graphql',
  batchMax: 10,        // 最多 10 个查询合并
  batchInterval: 20,   // 20ms 内的查询合并
});
```

```yaml
# Apollo Server 启用批处理
const server = new ApolloServer({
  schema,
  plugins: [
    ApolloServerPluginInlineTraceDisabled(),
  ],
  batching: true,      # 启用批量查询支持
});
```

## 四、注意事项

1. **批量操作要支持部分成功**
2. **批量查询可以并行执行**
3. **批量变更是顺序执行的**
4. **客户端批量可以减少网络请求**
5. **注意批量操作的事务一致性**
