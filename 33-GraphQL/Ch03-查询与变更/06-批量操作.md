# 批量操作

## 一、批量查询

```graphql
# 并行查询 (同一次请求)
query BatchQuery {
  users: users(first: 10) {
    edges { node { id name } }
  }
  products: products(first: 10) {
    edges { node { id name price } }
  }
  orders: orders(first: 10) {
    edges { node { id total status } }
  }
}

# 多个命名查询
query GetUser($id: ID!) {
  user(id: $id) { id name }
}

query GetOrders($userId: ID!) {
  orders(userId: $userId) { id total }
}

# 客户端发送多个查询
[
  { query: "query GetUser($id: ID!) { ... }", variables: { id: "1" } },
  { query: "query GetOrders($userId: ID!) { ... }", variables: { userId: "1" } }
]
```

## 二、批量变更

```graphql
# 批量创建
mutation BatchCreate($inputs: [CreateUserInput!]!) {
  batchCreateUsers(inputs: $inputs) {
    successCount
    failedCount
    results {
      success
      user { id name }
      error { code message }
    }
  }
}

# 混合操作
mutation BulkOperations {
  createUser: createUser(input: { name: "张三" }) {
    id name
  }
  updateUser: updateUser(id: "2", input: { name: "李四" }) {
    id name
  }
  deleteUser: deleteUser(id: "3") {
    success
  }
}
```

## 四、批量操作 Resolver 实现

```javascript
const resolvers = {
  Mutation: {
    batchCreateUsers: async (_, { inputs }, context) => {
      const results = [];
      const errors = [];

      for (let i = 0; i < inputs.length; i++) {
        try {
          const user = await db.users.create(inputs[i]);
          results.push({ success: true, user, error: null });
        } catch (e) {
          errors.push({
            index: i,
            field: e.field || null,
            message: e.message,
          });
          results.push({ success: false, user: null, error: e.message });
        }
      }

      return {
        successCount: results.filter((r) => r.success).length,
        failedCount: results.filter((r) => !r.success).length,
        results,
        errors,
      };
    },

    // 混合操作的实现
    bulkUserOperations: async (_, { input }, context) => {
      return await db.transaction(async (trx) => {
        const created = [];
        const updated = [];
        const errors = [];

        // 批量创建
        if (input.create?.length) {
          for (const item of input.create) {
            try {
              created.push(await trx.users.create(item));
            } catch (e) {
              errors.push({ code: 'CREATE_FAILED', message: e.message, path: ['create'] });
            }
          }
        }

        // 批量更新
        if (input.update?.length) {
          for (const { id, ...data } of input.update) {
            try {
              updated.push(await trx.users.update(id, data));
            } catch (e) {
              errors.push({ code: 'UPDATE_FAILED', message: e.message, path: ['update', id] });
            }
          }
        }

        // 批量删除
        const deleted = [];
        if (input.delete?.length) {
          for (const id of input.delete) {
            try {
              await trx.users.delete(id);
              deleted.push({ success: true, id });
            } catch (e) {
              errors.push({ code: 'DELETE_FAILED', message: e.message, path: ['delete', id] });
            }
          }
        }

        return { created, updated, deleted, errors };
      });
    },
  },
};
```

## 五、APQ (Automatic Persisted Queries)

```javascript
// APQ 减少重复查询传输
// 第一次请求：
//   GET /graphql?extensions={"persistedQuery":{"sha256Hash":"abc123"}}
//   响应: { "errors": [{ "message": "PersistedQueryNotFound" }] }
// 第二次请求（带 query body）：
//   GET /graphql?extensions={...}&query={user{name}}
//   响应: { "data": { "user": { "name": "..." } } }
// 后续请求只需 hash：
//   GET /graphql?extensions={"persistedQuery":{"sha256Hash":"abc123"}}
//   响应: 缓存数据

// Apollo Server 配置
import { ApolloServer } from '@apollo/server';
import { ApolloServerPluginCacheControl } from '@apollo/server/plugin/cacheControl';

const server = new ApolloServer({
  schema,
  plugins: [
    ApolloServerPluginCacheControl({ defaultMaxAge: 60 }),
  ],
  persistedQueries: {
    cache: 'bounded',  // 内存缓存
  },
});
```

## 六、注意事项

1. **批量操作要支持部分成功**
2. **批量查询可以并行执行**
3. **批量变更是顺序执行的**
4. **客户端批量可以减少网络请求**
5. **注意批量操作的事务一致性**
6. **APQ 可显著减少重复查询传输量**
7. **大批量操作要分页处理**，避免单次操作过重
