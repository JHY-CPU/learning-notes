# 片段 (Fragments)

## 一、命名片段

```graphql
# 定义片段
fragment UserFields on User {
  id
  name
  email
  avatar
}

fragment OrderFields on Order {
  id
  orderNo
  total
  status
  createdAt
}

# 使用片段
query GetUserWithOrders($userId: ID!) {
  user(id: $userId) {
    ...UserFields
    orders {
      ...OrderFields
      items {
        product { name }
        quantity
      }
    }
  }
}
```

## 二、内联片段

```graphql
# 内联片段 - 多态查询
query SearchResults($keyword: String!) {
  search(keyword: $keyword) {
    ... on User {
      name
      email
    }
    ... on Product {
      name
      price
      category { name }
    }
    ... on Order {
      orderNo
      total
    }
  }
}

# 带类型条件
query GetNode($id: ID!) {
  node(id: $id) {
    id
    ... on User {
      name
      role
    }
    ... on Product @include(if: $showProducts) {
      name
      price
    }
  }
}
```

## 三、片段组合

```graphql
# 片片可以引用其他片段
fragment UserBasic on User {
  id
  name
}

fragment UserDetail on User {
  ...UserBasic
  email
  phone
  address {
    city
    street
  }
}

fragment UserWithOrders on User {
  ...UserDetail
  orders(first: 10) {
    edges {
      node {
        id
        total
        status
      }
    }
  }
}

query {
  user(id: "1") {
    ...UserWithOrders
  }
}
```

## 四、片段的 TypeScript 类型生成

```graphql
# fragments/user.graphql
fragment UserBasic on User {
  id
  name
  avatar
}

fragment UserDetail on User {
  ...UserBasic
  email
  phone
  createdAt
}
```

```typescript
// codegen 自动生成
export type UserBasicFragment = {
  __typename?: 'User';
  id: string;
  name: string;
  avatar: string | null;
};

export type UserDetailFragment = {
  __typename?: 'User';
  id: string;
  name: string;
  avatar: string | null;
  email: string;
  phone: string | null;
  createdAt: any;
};

// 使用
import { UserBasicFragment, UserDetailFragment } from './generated';
```

## 五、片段与 Apollo Cache

```javascript
// 片段自动写入缓存
import { useFragment, gql } from '@apollo/client';

const USER_FRAGMENT = gql`
  fragment UserFields on User {
    id
    name
    email
    avatar
  }
`;

// useFragment - 从缓存读取片段数据
function UserCard({ userRef }) {
  const { data, complete } = useFragment({
    fragment: USER_FRAGMENT,
    from: userRef,
  });

  if (!complete) return null;

  return (
    <div>
      <img src={data.avatar} alt={data.name} />
      <h3>{data.name}</h3>
      <p>{data.email}</p>
    </div>
  );
}
```

## 六、片段组织最佳实践

```yaml
文件组织:
  src/graphql/fragments/
    ├── user.graphql          # User 相关片段
    ├── order.graphql         # Order 相关片段
    ├── product.graphql       # Product 相关片段
    └── common.graphql        # 公共片段（分页等）

命名规范:
  - {Entity}Basic: 基础字段 (id, name)
  - {Entity}Detail: 详情字段 (所有常用字段)
  - {Entity}Card: 卡片展示字段
  - {Entity}Form: 表单字段

注意事项:
  - 每个实体定义 2-3 个粒度的片段
  - 避免一个巨型片段包含所有字段
  - 片段可以引用其他片段（但避免循环）
  - Codegen 会为每个片段生成独立类型
```

## 七、注意事项

1. **片段用于复用字段集合**
2. **命名片段定义在文件或共享模块中**
3. **内联片段适合多态场景**
4. **片段之间不要循环引用**
5. **GraphQL Codegen 会为片段生成 TypeScript 类型**
6. **useFragment 可从缓存读取片段数据**
7. **片段粒度适中**，不要包含过多或过少字段
