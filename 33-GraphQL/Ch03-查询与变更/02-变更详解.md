# 变更详解 (Mutation)

## 一、CRUD 变更

```graphql
# Create
mutation CreateUser($input: CreateUserInput!) {
  createUser(input: $input) {
    id
    name
    email
    createdAt
  }
}

# Update
mutation UpdateUser($id: ID!, $input: UpdateUserInput!) {
  updateUser(id: $id, input: $input) {
    id
    name
    email
    updatedAt
  }
}

# Delete
mutation DeleteUser($id: ID!) {
  deleteUser(id: $id) {
    success
    message
  }
}
```

## 二、复杂业务变更

```graphql
# 下单 - 关联多实体
mutation PlaceOrder($input: PlaceOrderInput!) {
  placeOrder(input: $input) {
    order {
      id
      orderNo
      total
      status
      items {
        product { name price }
        quantity
      }
    }
    paymentUrl     # 返回支付链接
  }
}

input PlaceOrderInput {
  items: [OrderItemInput!]!
  addressId: ID!
  couponCode: String
  paymentMethod: PaymentMethod!
}

# 批量操作
mutation BatchOperation($ops: BatchInput!) {
  batch(operations: $ops) {
    created { id name }
    updated { id name }
    deleted { success }
    errors { code message path }
  }
}
```

## 三、乐观响应

```graphql
# Mutation 返回更新后的对象
mutation ToggleLike($postId: ID!) {
  toggleLike(postId: $postId) {
    post {
      id
      likeCount
      isLiked
    }
  }
}

# 客户端可以乐观更新 UI
# 无需重新查询获取最新数据
```

## 四、乐观更新

```javascript
// Apollo Client 乐观更新
import { useMutation, gql } from '@apollo/client';

const TOGGLE_LIKE = gql`
  mutation ToggleLike($postId: ID!) {
    toggleLike(postId: $postId) {
      post {
        id
        likeCount
        isLiked
      }
    }
  }
`;

function LikeButton({ post }) {
  const [toggleLike] = useMutation(TOGGLE_LIKE, {
    optimisticResponse: {
      __typename: 'Mutation',
      toggleLike: {
        __typename: 'ToggleLikePayload',
        post: {
          __typename: 'Post',
          id: post.id,
          likeCount: post.isLiked ? post.likeCount - 1 : post.likeCount + 1,
          isLiked: !post.isLiked,
        },
      },
    },
    // 缓存更新策略
    update(cache, { data: { toggleLike } }) {
      cache.modify({
        id: cache.identify(post),
        fields: {
          likeCount: () => toggleLike.post.likeCount,
          isLiked: () => toggleLike.post.isLiked,
        },
      });
    },
  });

  return (
    <button onClick={() => toggleLike()}>
      {post.isLiked ? '已赞' : '点赞'} ({post.likeCount})
    </button>
  );
}
```

## 五、错误处理 (Payload 模式)

```graphql
# Relay 风格的错误处理
type Mutation {
  updateUser(input: UpdateUserInput!): UpdateUserPayload!
}

type UpdateUserPayload {
  user: User
  errors: [UserError!]!
}

type UserError {
  field: [String!]       # 错误字段路径
  message: String!       # 人类可读信息
  code: ErrorCode!       # 机器可读代码
}

enum ErrorCode {
  VALIDATION_ERROR
  NOT_FOUND
  DUPLICATE_EMAIL
  FORBIDDEN
  INTERNAL_ERROR
}
```

```javascript
// 客户端处理
const [updateUser, { data }] = useMutation(UPDATE_USER);

const result = await updateUser({ variables: { input } });

if (result.data.updateUser.errors.length > 0) {
  // 处理业务错误
  result.data.updateUser.errors.forEach((err) => {
    if (err.code === 'DUPLICATE_EMAIL') {
      form.setFieldError('email', err.message);
    }
  });
} else {
  // 成功
  toast.success('更新成功');
}
```

## 六、服务端实现

```javascript
const resolvers = {
  Mutation: {
    updateUser: async (_, { id, input }, context) => {
      // 权限检查
      if (context.currentUser.id !== id) {
        return {
          user: null,
          errors: [{ field: [], message: '无权修改', code: 'FORBIDDEN' }],
        };
      }

      // 验证
      const errors = [];
      if (input.email && !isValidEmail(input.email)) {
        errors.push({ field: ['email'], message: '邮箱格式错误', code: 'VALIDATION_ERROR' });
      }

      if (errors.length > 0) {
        return { user: null, errors };
      }

      // 执行更新
      try {
        const user = await db.users.update(id, input);
        return { user, errors: [] };
      } catch (e) {
        if (e.code === 'DUPLICATE_KEY') {
          return {
            user: null,
            errors: [{ field: ['email'], message: '邮箱已被使用', code: 'DUPLICATE_EMAIL' }],
          };
        }
        throw e;
      }
    },
  },
};
```

## 七、注意事项

1. **Mutation 是顺序执行的**，Query 是并行的
2. **返回变更后的完整对象**
3. **Input 类型要设计好**，支持部分更新
4. **批量操作要有错误汇总**
5. **业务逻辑在 Resolver 中**，不要在 Schema 中编码
6. **乐观更新提升用户体验**，但要处理回滚
7. **Payload 模式让错误处理更规范**
