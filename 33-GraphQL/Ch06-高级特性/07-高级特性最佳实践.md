# 高级特性最佳实践

## 一、特性选型

```yaml
场景与推荐:
  微服务拆分:
    - 推荐: Apollo Federation
    - 替代: Schema Stitching (不推荐)

  大字段延迟加载:
    - 推荐: @defer
    - 替代: 分开查询

  长列表流式加载:
    - 推荐: @stream
    - 替代: 分页

  高频重复查询:
    - 推荐: APQ + CDN
    - 替代: 客户端缓存

  实时数据推送:
    - 推荐: Subscription
    - 替代: 轮询 / SSE

  自定义扩展:
    - 推荐: 自定义指令
    - 替代: 中间件
```

## 二、实践清单

```yaml
Federation:
  □ 子图边界清晰
  □ @key 定义合理
  □ Gateway 高可用
  □ Schema 编译检查

实时:
  □ WebSocket 鉴权
  □ Redis PubSub 跨实例
  □ 断线重连处理
  □ 消息频率控制

性能:
  □ APQ 启用
  □ 查询深度限制
  □ 复杂度限制
  □ DataLoader 配置
```

## 三、注意事项

1. **按需引入高级特性**，不要过度设计
2. **Federation 是微服务 GraphQL 的标准**
3. **实验性特性谨慎用于生产**
4. **监控各特性的性能影响**
5. **保持技术栈简洁**
