# 自定义指令实现

## 一、指令定义与实现

```graphql
# Schema 定义
directive @auth(requires: Role!) on FIELD_DEFINITION
directive @rateLimit(max: Int!, window: String!) on FIELD_DEFINITION
directive @cache(maxAge: Int!) on FIELD_DEFINITION | OBJECT
directive @validate(pattern: String!, message: String) on ARGUMENT_DEFINITION

enum Role { ADMIN CUSTOMER VENDOR }

type Query {
  users: [User!]! @auth(requires: ADMIN) @rateLimit(max: 100, window: "1m")
  me: User! @auth(requires: CUSTOMER)
  products: [Product!]! @cache(maxAge: 300)
}
```

```java
// Spring GraphQL 指令实现
@Component
public class AuthWiring implements RuntimeWiringConfigurer {

    @Override
    public void configure(RuntimeWiring.Builder builder) {
        builder.directiveWiring(new AuthDirectiveWiring());
    }
}

public class AuthDirectiveWiring implements SchemaDirectiveWiring {

    @Override
    public GraphQLFieldDefinition onField(
            SchemaDirectiveWiringEnvironment<GraphQLFieldDefinition> env) {
        Role requiredRole = env.getDirective()
            .getArgument("requires").getValue();

        DataFetcher<?> original = env.getFieldDataFetcher();

        return env.setField(env.getField().transform(builder ->
            builder.dataFetcher(env -> {
                // 检查认证
                SecurityContext ctx = env.getContext();
                if (ctx.getUser() == null) {
                    throw new AuthenticationException("未登录");
                }
                // 检查角色
                if (!ctx.getUser().hasRole(requiredRole)) {
                    throw new AccessDeniedException("需要 " + requiredRole + " 权限");
                }
                return original.get(env);
            })));
    }
}
```

## 二、组合指令

```graphql
# 多个指令叠加
type Query {
  adminUsers: [User!]!
    @auth(requires: ADMIN)
    @rateLimit(max: 50, window: "1m")
    @cache(maxAge: 60)
}
```

## 三、客户端指令

```graphql
# @client - 客户端本地字段
type Query {
  isLoggedIn: Boolean! @client
  cartItems: [CartItem!]! @client
}

type Mutation {
  addToCart(id: ID!) @client
  removeFromCart(id: ID!) @client
}
```

## 四、注意事项

1. **指令在 Schema 定义，Runtime 实现**
2. **指令执行顺序要注意**
3. **不要过度使用指令**
4. **复杂逻辑用中间件替代**
5. **指令的错误信息要清晰**
