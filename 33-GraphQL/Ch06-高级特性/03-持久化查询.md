# 持久化查询

## 一、APQ (Automatic Persisted Queries)

```
APQ 流程:
1. 客户端: 发送 { query: hash } → 服务端未找到
2. 客户端: 发送 { query: fullQuery, extensions: { hash } } → 服务端缓存
3. 客户端: 发送 { query: hash } → 服务端找到，直接执行
```

```typescript
// Apollo Client APQ 配置
import { createPersistedQueryLink } from '@apollo/client/link/persisted-queries';
import { sha256 } from 'crypto-hash';

const persistedQueryLink = createPersistedQueryLink({
  sha256,
  useGETForHashedQueries: true,  // 哈希查询用 GET
});

const client = new ApolloClient({
  link: persistedQueryLink.concat(httpLink),
});
```

## 二、服务端配置

```typescript
// Apollo Server
import { ApolloServerPluginCacheControl } from '@apollo/server/plugin/cacheControl';

const server = new ApolloServer({
  schema,
  plugins: [
    ApolloServerPluginCacheControl({ defaultMaxAge: 60 }),
  ],
  persistedQueries: {
    cache: new KeyValueCache(),  // Redis 或内存
    ttl: 900000,                 // 15 分钟
  },
});
```

## 三、预注册查询

```yaml
# 构建时预注册查询
# graphql-persisted-queries.json
{
  "sha256Hash1": "query GetUser($id: ID!) { user(id: $id) { name } }",
  "sha256Hash2": "query GetUsers { users { id name } }"
}
```

```typescript
// 构建时生成
import { gql } from '@apollo/client';

// 使用编译时查询 - Relay 风格
const query = graphql`
  query GetUserQuery($id: ID!) {
    user(id: $id) { name }
  }
`;

// 编译器自动生成哈希
// 查询只发送哈希值
```

## 四、优势

```yaml
APQ 优势:
  网络优化:
    - GET 请求可 CDN 缓存
    - 减少请求体积
    - 哈希值固定大小

  安全:
    - 防止恶意查询注入
    - 只有注册的查询可执行
    - 减少攻击面

  性能:
    - 减少解析开销
    - CDN 边缘缓存
    - 降低服务端 CPU
```

## 五、注意事项

1. **APQ 适合高频重复查询**
2. **哈希算法推荐 SHA-256**
3. **缓存过期时间要合理**
4. **预注册查询适合固定查询集**
5. **CDN 配合 GET 请求效果最佳**
