# 错误处理

## 一、GraphQL 错误格式

```json
{
  "data": {
    "user": null
  },
  "errors": [
    {
      "message": "用户不存在",
      "locations": [{ "line": 2, "column": 3 }],
      "path": ["user"],
      "extensions": {
        "code": "USER_NOT_FOUND",
        "classification": "NOT_FOUND",
        "timestamp": "2024-01-01T12:00:00Z"
      }
    }
  ]
}
```

## 二、自定义错误类

```java
// 自定义异常
public class BusinessException extends RuntimeException implements GraphQLError {
    private final String code;
    private final Map<String, Object> extensions;

    public BusinessException(String code, String message) {
        super(message);
        this.code = code;
        this.extensions = Map.of(
            "code", code,
            "classification", "BUSINESS_ERROR"
        );
    }

    @Override
    public List<SourceLocation> getLocations() { return null; }

    @Override
    public ErrorClassification getErrorType() {
        return ErrorClassification.ErrorType.DataFetchingException;
    }

    @Override
    public Map<String, Object> getExtensions() { return extensions; }
}

// 具体业务异常
public class UserNotFoundException extends BusinessException {
    public UserNotFoundException(String userId) {
        super("USER_NOT_FOUND", "用户不存在: " + userId);
    }
}

public class InsufficientBalanceException extends BusinessException {
    public InsufficientBalanceException(BigDecimal balance, BigDecimal required) {
        super("INSUFFICIENT_BALANCE",
            String.format("余额不足: 当前 %.2f, 需要 %.2f", balance, required));
    }
}
```

## 三、全局错误处理

```java
@RestControllerAdvice
public class GraphQLExceptionHandler {

    @ExceptionHandler(BusinessException.class)
    public ResponseEntity<Map<String, Object>> handleBusinessException(
            BusinessException ex) {
        return ResponseEntity.badRequest().body(Map.of(
            "errors", List.of(Map.of(
                "message", ex.getMessage(),
                "extensions", ex.getExtensions()
            ))
        ));
    }
}
```

## 四、注意事项

1. **GraphQL 200 也会有 errors**
2. **path 帮助定位错误字段**
3. **extensions 包含错误码等业务信息**
4. **不要在 message 中暴露敏感信息**
5. **错误分类要规范化**
