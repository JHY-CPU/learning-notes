# 订阅机制 (Subscription)

## 一、基础订阅

```graphql
# 订阅新订单
subscription OnOrderCreated {
  orderCreated {
    id
    total
    status
    customer {
      name
    }
  }
}

# 带过滤的订阅
subscription OnOrderUpdated($userId: ID!) {
  orderUpdated(userId: $userId) {
    id
    status
    updatedAt
  }
}
```

## 二、服务端实现 (graphql-ws)

```javascript
// Node.js + graphql-ws
import { WebSocketServer } from 'ws';
import { useServer } from 'graphql-ws/lib/use/ws';
import { PubSub } from 'graphql-subscriptions';

const pubsub = new PubSub();
const ORDER_CREATED = 'ORDER_CREATED';

const resolvers = {
  Subscription: {
    orderCreated: {
      subscribe: () => pubsub.asyncIterator([ORDER_CREATED]),
    },
  },
  Mutation: {
    createOrder: async (_, { input }) => {
      const order = await orderService.create(input);
      pubsub.publish(ORDER_CREATED, { orderCreated: order });
      return order;
    },
  },
};

const wsServer = new WebSocketServer({ server: httpServer, path: '/graphql' });
useServer({ schema }, wsServer);
```

## 三、客户端订阅

```javascript
// Apollo Client
import { useSubscription, gql } from '@apollo/client';

const ORDER_SUBSCRIPTION = gql`
  subscription OnOrderCreated {
    orderCreated {
      id
      total
      status
    }
  }
`;

function OrderNotifications() {
  const { data, loading } = useSubscription(ORDER_SUBSCRIPTION);

  if (loading) return <p>等待中...</p>;

  return (
    <div>
      新订单: {data.orderCreated.id} - ¥{data.orderCreated.total}
    </div>
  );
}
```

## 四、Redis PubSub（生产环境）

```javascript
// 多实例部署需要 Redis 做消息同步
import { RedisPubSub } from 'graphql-redis-subscriptions';
import Redis from 'ioredis';

const options = {
  host: 'redis-host',
  port: 6379,
  retryStrategy: (times) => Math.min(times * 50, 2000),
};

const pubsub = new RedisPubSub({
  publisher: new Redis(options),
  subscriber: new Redis(options),
});

// 发布事件
pubsub.publish('ORDER_CREATED', { orderCreated: newOrder });

// 订阅事件
const resolvers = {
  Subscription: {
    orderCreated: {
      subscribe: () => pubsub.asyncIterator(['ORDER_CREATED']),
    },
    orderUpdated: {
      subscribe: withFilter(
        () => pubsub.asyncIterator(['ORDER_UPDATED']),
        (payload, variables) => {
          // 过滤：只推送特定用户的订单
          return payload.orderUpdated.userId === variables.userId;
        }
      ),
    },
  },
};
```

## 五、WebSocket 鉴权

```javascript
import { useServer } from 'graphql-ws/lib/use/ws';
import { WebSocketServer } from 'ws';

const wsServer = new WebSocketServer({
  server: httpServer,
  path: '/graphql',
});

useServer(
  {
    schema,
    onConnect: async (ctx) => {
      // 连接时鉴权
      const token = ctx.connectionParams?.authorization;
      if (!token) {
        throw new Error('未认证');
      }
      const user = await verifyToken(token);
      if (!user) {
        throw new Error('认证失败');
      }
      return { currentUser: user };
    },
    context: (ctx) => {
      // 每个订阅消息的 Context
      return ctx.extra;
    },
  },
  wsServer
);
```

## 六、客户端断线重连

```javascript
import { GraphQLWsLink } from '@apollo/client/link/subscriptions';
import { createClient } from 'graphql-ws';

const wsLink = new GraphQLWsLink(
  createClient({
    url: 'ws://localhost:4000/graphql',
    connectionParams: {
      authorization: `Bearer ${getToken()}`,
    },
    shouldRetry: () => true,      // 启用重连
    retryAttempts: 5,              // 最多重试 5 次
    keepAlive: 10_000,             // 10 秒心跳
    on: {
      connected: () => console.log('WebSocket 已连接'),
      error: (err) => console.error('WebSocket 错误:', err),
      closed: () => console.log('WebSocket 已断开'),
    },
  })
);
```

## 七、注意事项

1. **订阅通过 WebSocket 传输**
2. **生产环境用 Redis PubSub 做多实例同步**
3. **订阅要处理断线重连**
4. **鉴权在 WebSocket 连接时做**
5. **订阅数据量要控制**，避免带宽问题
6. **使用 withFilter 做条件过滤**，减少不必要推送
7. **心跳机制保持连接活跃**
