# 订阅机制 (Subscription)

## 一、基础订阅

```graphql
# 订阅新订单
subscription OnOrderCreated {
  orderCreated {
    id
    total
    status
    customer {
      name
    }
  }
}

# 带过滤的订阅
subscription OnOrderUpdated($userId: ID!) {
  orderUpdated(userId: $userId) {
    id
    status
    updatedAt
  }
}
```

## 二、服务端实现 (graphql-ws)

```javascript
// Node.js + graphql-ws
import { WebSocketServer } from 'ws';
import { useServer } from 'graphql-ws/lib/use/ws';
import { PubSub } from 'graphql-subscriptions';

const pubsub = new PubSub();
const ORDER_CREATED = 'ORDER_CREATED';

const resolvers = {
  Subscription: {
    orderCreated: {
      subscribe: () => pubsub.asyncIterator([ORDER_CREATED]),
    },
  },
  Mutation: {
    createOrder: async (_, { input }) => {
      const order = await orderService.create(input);
      pubsub.publish(ORDER_CREATED, { orderCreated: order });
      return order;
    },
  },
};

```

## 三、客户端订阅

```javascript
// Apollo Client
import { useSubscription, gql } from '@apollo/client';

const ORDER_SUBSCRIPTION = gql`
  subscription OnOrderCreated {
    orderCreated {
      id
      total
      status
    }
  }
`;

function OrderNotifications() {
  const { data, loading } = useSubscription(ORDER_SUBSCRIPTION);

  if (loading) return <p>等待中...</p>;

  return (
    <div>
      新订单: {data.orderCreated.id} - ¥{data.orderCreated.total}
    </div>
  );
}
```

## 四、注意事项

1. **订阅通过 WebSocket 传输**
2. **生产环境用 Redis PubSub 做多实例同步**
3. **订阅要处理断线重连**
4. **鉴权在 WebSocket 连接时做**
5. **订阅数据量要控制**，避免带宽问题
