# 类型系统详解

## 一、类型层次

```
类型系统:
├── 标量 (Scalar) - String, Int, Float, Boolean, ID
├── 对象 (Object) - type User { ... }
├── 接口 (Interface) - interface Node { ... }
├── 联合 (Union) - union SearchResult = User | Product
├── 枚举 (Enum) - enum Status { ACTIVE, INACTIVE }
├── 输入 (Input) - input CreateUserInput { ... }
├── 列表 (List) - [User!]!
└── 非空 (Non-Null) - String!
```

## 二、非空与列表

```graphql
# 非空标记
type User {
  name: String!      # 不能为 null
  email: String!     # 不能为 null
  bio: String        # 可以为 null
}

# 列表类型
type User {
  friends: [User]        # 可以为 null，元素可为 null
  tags: [String!]        # 列表可为 null，元素不能为 null
  orders: [Order!]!      # 列表不能为 null，元素不能为 null
}

# 对应 JSON 返回
# friends: null         ✓
# friends: [null]       ✓
# friends: [{...}]      ✓
# tags: null            ✓
# tags: ["a", null]     ✗
# orders: null          ✗
# orders: []            ✓
```

## 三、字段参数

```graphql
type User {
  # 无参字段
  name: String!

  # 带参字段
  avatar(size: Int = 100): String!
  posts(limit: Int = 10, offset: Int = 0): [Post!]!

  # 带默认值
  friends(first: Int = 10, after: String): UserConnection!
}

# 查询时传参
query {
  user(id: "1") {
    name
    avatar(size: 200)
    posts(limit: 5) {
      title
    }
  }
}
```

## 四、Schema 演进

```yaml
安全变更 (不破坏):
  - 新增可选字段
  - 新增枚举值
  - 新增可选参数
  - 放宽返回类型限制

破坏性变更 (需谨慎):
  - 重命名字段
  - 删除字段
  - 新增非空字段
  - 缩小类型范围
  - 删除枚举值

推荐做法:
  - 用 @deprecated 标记废弃字段
  - 新增字段而非重命名
  - 新字段用可选类型
```

## 五、注意事项

1. **非空标记要合理使用**，影响客户端处理
2. **列表的非空层次要区分**
3. **接口变更要考虑所有实现者**
4. **联合类型要覆盖所有 case**
5. **自定义标量需要序列化/反序列化实现**
