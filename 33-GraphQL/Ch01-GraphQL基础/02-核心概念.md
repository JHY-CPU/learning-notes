# GraphQL 核心概念

## 一、三大操作类型

```graphql
# Query - 查询（读操作）
query GetUser($id: ID!) {
  user(id: $id) {
    name
    email
  }
}

# Mutation - 变更（写操作）
mutation CreateUser($input: CreateUserInput!) {
  createUser(input: $input) {
    id
    name
  }
}

# Subscription - 订阅（实时）
subscription OnOrderCreated {
  orderCreated {
    id
    total
    customer { name }
  }
}
```

## 二、Schema 组成

```graphql
# 类型定义
type User {
  id: ID!
  name: String!
  email: String!
  orders: [Order!]!
  createdAt: DateTime
}

type Order {
  id: ID!
  total: Float!
  status: OrderStatus!
  user: User!
}

# 枚举类型
enum OrderStatus {
  CREATED
  PAID
  SHIPPED
  DELIVERED
}

# 查询入口
type Query {
  user(id: ID!): User
  users(limit: Int, offset: Int): [User!]!
  order(id: ID!): Order
}

# 变更入口
type Mutation {
  createUser(input: CreateUserInput!): User!
  updateOrder(id: ID!, input: UpdateOrderInput!): Order!
}
```

## 三、执行流程

```
客户端请求
    │
    ▼
┌─────────────────┐
│  解析 Query      │ → 验证语法
│  验证 Schema     │ → 验证类型匹配
│  执行 Resolver   │ → 获取数据
│  组装响应        │ → 返回结果
└─────────────────┘
```

## 四、注意事项

1. **! 表示非空字段**，缺少会返回错误
2. **ID 是内置标量类型**，唯一标识符
3. **Query 是入口，不是类型名**
4. **Resolver 是数据获取的实际逻辑**
5. **Schema First 或 Code First 两种设计方式**
