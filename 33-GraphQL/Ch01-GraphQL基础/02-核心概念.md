# GraphQL 核心概念

## 一、三大操作类型

```graphql
# Query - 查询（读操作）
query GetUser($id: ID!) {
  user(id: $id) {
    name
    email
  }
}

# Mutation - 变更（写操作）
mutation CreateUser($input: CreateUserInput!) {
  createUser(input: $input) {
    id
    name
  }
}

# Subscription - 订阅（实时）
subscription OnOrderCreated {
  orderCreated {
    id
    total
    customer { name }
  }
}
```

## 二、Schema 组成

```graphql
# 类型定义
type User {
  id: ID!
  name: String!
  email: String!
  orders: [Order!]!
  createdAt: DateTime
}

type Order {
  id: ID!
  total: Float!
  status: OrderStatus!
  user: User!
}

# 枚举类型
enum OrderStatus {
  CREATED
  PAID
  SHIPPED
  DELIVERED
}

# 查询入口
type Query {
  user(id: ID!): User
  users(limit: Int, offset: Int): [User!]!
  order(id: ID!): Order
}

# 变更入口
type Mutation {
  createUser(input: CreateUserInput!): User!
  updateOrder(id: ID!, input: UpdateOrderInput!): Order!
}
```

## 三、执行流程

```
客户端请求
    │
    ▼
┌─────────────────┐
│  解析 Query      │ → 验证语法
│  验证 Schema     │ → 验证类型匹配
│  执行 Resolver   │ → 获取数据
│  组装响应        │ → 返回结果
└─────────────────┘
```

## 四、Resolver 详解

Resolver 是 GraphQL 执行引擎的核心，每个字段都对应一个 Resolver 函数。

```javascript
// Resolver 签名
const resolvers = {
  Query: {
    // (parent, args, context, info) => result
    user: async (parent, args, context, info) => {
      // parent: 父字段的结果
      // args: 该字段的参数 { id: "1" }
      // context: 请求级别的共享数据（认证、DataLoader等）
      // info: 查询的 AST 和字段信息
      return await context.loaders.user.load(args.id);
    },
  },
  User: {
    // 字段级 Resolver
    orders: async (parent, args, context) => {
      // parent 就是上面 user Resolver 返回的对象
      return context.loaders.ordersByUser.load(parent.id);
    },
    fullName: (parent) => {
      // 同步计算字段
      return `${parent.firstName} ${parent.lastName}`;
    },
  },
};
```

## 五、上下文 (Context)

Context 是每次请求的共享对象，贯穿所有 Resolver。

```javascript
// Context 工厂函数
const server = new ApolloServer({
  typeDefs,
  resolvers,
  context: async ({ req }) => {
    // 每个请求创建新的 Context
    const token = req.headers.authorization || '';
    const currentUser = await verifyToken(token);

    return {
      currentUser,
      loaders: createDataLoaders(),  // 每次请求新的 DataLoader
      dataSources: {
        userAPI: new UserDataSource(),
        orderAPI: new OrderDataSource(),
      },
    };
  },
});
```

## 六、错误处理

```javascript
import { GraphQLError } from 'graphql';

const resolvers = {
  Query: {
    user: async (_, { id }, context) => {
      if (!context.currentUser) {
        throw new GraphQLError('未认证', {
          extensions: {
            code: 'UNAUTHENTICATED',
            http: { status: 401 },
          },
        });
      }

      const user = await db.users.findById(id);
      if (!user) {
        throw new GraphQLError('用户不存在', {
          extensions: {
            code: 'NOT_FOUND',
            http: { status: 404 },
          },
        });
      }
      return user;
    },
  },
};
```

错误响应格式:
```json
{
  "errors": [
    {
      "message": "未认证",
      "extensions": {
        "code": "UNAUTHENTICATED"
      },
      "path": ["user"],
      "locations": [{ "line": 2, "column": 3 }]
    }
  ],
  "data": null
}
```

## 七、注意事项

1. **! 表示非空字段**，缺少会返回错误
2. **ID 是内置标量类型**，唯一标识符
3. **Query 是入口，不是类型名**
4. **Resolver 是数据获取的实际逻辑**
5. **Schema First 或 Code First 两种设计方式**
6. **Context 要每次请求新建**，避免 DataLoader 缓存污染
7. **错误要使用 extensions.code** 便于客户端分类处理
