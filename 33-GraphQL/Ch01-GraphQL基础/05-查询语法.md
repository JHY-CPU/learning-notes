# 查询语法

## 一、基础查询

```graphql
# 简单查询
{
  user(id: "1") {
    name
    email
  }
}

# 带别名 - 同一字段多次查询
{
  admin: user(id: "1") {
    name
  }
  customer: user(id: "2") {
    name
  }
}

# 片段 - 复用字段集合
fragment UserFields on User {
  id
  name
  email
  avatar
}

{
  user1: user(id: "1") {
    ...UserFields
  }
  user2: user(id: "2") {
    ...UserFields
    role
  }
}
```

## 二、变量

```graphql
# 带变量的查询
query GetUser($id: ID!, $includeOrders: Boolean!) {
  user(id: $id) {
    name
    orders @include(if: $includeOrders) {
      id
      total
    }
  }
}

# 变量值
{
  "id": "1",
  "includeOrders": true
}
```

## 三、指令

```graphql
# @include - 条件包含
query GetUser($id: ID!, $withEmail: Boolean!) {
  user(id: $id) {
    name
    email @include(if: $withEmail)
  }
}

# @skip - 条件排除
query GetUser($id: ID!, $skipAvatar: Boolean!) {
  user(id: $id) {
    name
    avatar @skip(if: $skipAvatar)
  }
}

# @deprecated - 使用已废弃字段时的警告
type User {
  oldField: String @deprecated(reason: "Use newField instead")
}
```

## 四、嵌套与关联查询

```graphql
# 深度嵌套查询
query {
  user(id: "1") {
    name
    orders(first: 5) {
      edges {
        node {
          id
          total
          items {
            product {
              name
              price
              category {
                name
              }
            }
            quantity
          }
        }
      }
      pageInfo {
        hasNextPage
        endCursor
      }
    }
  }
}
```

## 五、分页查询

```graphql
# Relay Connection 分页
query GetProducts($first: Int, $after: String) {
  products(first: $first, after: $after) {
    edges {
      node {
        id
        name
        price
      }
      cursor
    }
    pageInfo {
      hasNextPage
      hasPreviousPage
      startCursor
      endCursor
    }
    totalCount
  }
}

# 变量
{ "first": 10, "after": "Y3Vyc29yOjEw" }

# 下一页
{ "first": 10, "after": "Y3Vyc29yOjIw" }
```

```graphql
# Offset 分页（简单场景）
query GetUsers($page: Int!, $size: Int!) {
  users(page: $page, size: $size) {
    items {
      id
      name
    }
    totalPages
    currentPage
    totalElements
  }
}
```

## 六、性能注意事项

```graphql
# ✗ 避免过深嵌套 - 会导致 N+1 问题
query Bad {
  user(id: "1") {
    orders {
      items {
        product {
          category {
            parent {
              name  # 深度 5，性能差
            }
          }
        }
      }
    }
  }
}

# ✓ 控制嵌套深度 - 建议不超过 3-4 层
query Good {
  user(id: "1") {
    name
    orders(first: 10) {
      edges {
        node {
          id
          total
          itemCount  # 用计数字段替代深层展开
        }
      }
    }
  }
}
```

## 七、注意事项

1. **查询命名建议使用 PascalCase**
2. **变量类型要标注非空 !**
3. **别名适合同一字段多次查询**
4. **片段避免过度抽象**
5. **嵌套深度要控制**，防止 N+1 问题
6. **分页用 Connection 模式**，便于游标翻页
7. **避免在查询中硬编码参数值**，统一使用变量
