# 后端实现概述

## 一、技术选型

```yaml
Node.js:
  Apollo Server:
    - 最流行
    - 插件丰富
    - Apollo 生态

  graphql-yoga:
    - 轻量高性能
    - Envelop 插件系统
    - GraphQL Yoga 出品

  Mercurius:
    - Fastify 集成
    - JIT 编译
    - 高性能

Java:
  Spring GraphQL:
    - Spring 官方
    - Spring Boot 集成
    - 注解驱动

  graphql-java:
    - 底层库
    - 功能强大
    - 需要手动集成

  DGS (Netflix):
    - Netflix 开源
    - 注解驱动
    - Federation 支持
```

## 三、执行流程

```
客户端请求 (JSON)
    │
    ▼
┌─────────────────┐
│  HTTP 服务       │ → 接收请求
│  解析 JSON       │ → 提取 query/variables
│  验证查询        │ → 语法 + Schema 验证
│  执行引擎        │ → 调用 Resolver
│  数据获取        │ → Service/Repository
│  组装响应        │ → 构建 JSON 返回
└─────────────────┘
```

## 四、注意事项

1. **Schema First 是主流方式**
2. **Resolver 是连接 Schema 和数据的桥梁**
3. **业务逻辑放在 Service 层**
4. **错误处理要规范化**
5. **关注性能（DataLoader、缓存）**
