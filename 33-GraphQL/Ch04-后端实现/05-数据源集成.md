# 数据源集成

## 一、REST API 数据源

```typescript
// Apollo Server RESTDataSource
import { RESTDataSource } from '@apollo/server/datasource/rest';

class UserAPI extends RESTDataSource {
  override baseURL = 'http://user-service/api/';

  async getUser(id: string) {
    return this.get(`users/${id}`);
  }

  async createUser(input: CreateUserInput) {
    return this.post('users', input);
  }

  // 自动缓存
  async getUsers() {
    return this.get('users', undefined, {
      cacheOptions: { ttl: 60 },
    });
  }
}

// 注册到 Context
const server = new ApolloServer({
  schema,
  context: () => ({
    userAPI: new UserAPI(),
    orderAPI: new OrderAPI(),
  }),
});
```

## 二、数据库直连

```java
// Spring GraphQL + JPA
@Repository
public interface UserRepository extends JpaRepository<User, String> {

    @Query("SELECT u FROM User u WHERE u.id IN :ids")
    List<User> findByIds(@Param("ids") List<String> ids);

    List<User> findByStatus(UserStatus status);
}

// DataLoader 批量查询
@Component
public class UserDataLoader {

    public DataLoader<String, User> create(UserRepository repo) {
        return DataLoaderFactory.newDataLoader(
            (List<String> ids) -> CompletableFuture.supplyAsync(() -> {
                List<User> users = repo.findByIds(ids);
                Map<String, User> userMap = users.stream()
                    .collect(Collectors.toMap(User::getId, u -> u));
                return ids.stream()
                    .map(userMap::get)
                    .collect(Collectors.toList());
            })
        );
    }
}
```

## 三、多数据源聚合

```graphql
# BFF 聚合多个微服务
type Query {
  userProfile(id: ID!): UserProfile!
}

type UserProfile {
  user: User!                # 来自用户服务
  orders: [Order!]!          # 来自订单服务
  reviews: [Review!]!        # 来自评价服务
  statistics: UserStatistics # 聚合计算
}
```

```java
// 聚合 Resolver
@SchemaMapping(typeName = "UserProfile")
public CompletableFuture<List<Order>> orders(
        UserProfile profile,
        DataLoader<String, List<Order>> ordersLoader) {
    return ordersLoader.load(profile.getUserId());
}

@SchemaMapping(typeName = "UserProfile")
public UserStatistics statistics(UserProfile profile) {
    return CompletableFuture.allOf(
        ordersLoader.load(profile.getUserId()),
        reviewsLoader.load(profile.getUserId())
    ).thenApply(v -> {
        // 聚合统计数据
        return new UserStatistics(
            ordersLoader.get(profile.getUserId()).size(),
            reviewsLoader.get(profile.getUserId()).size()
        );
    });
}
```

## 四、注意事项

1. **RESTDataSource 自带缓存和错误处理**
2. **数据库查询要用 DataLoader 批量加载**
3. **多数据源聚合是 GraphQL 的核心优势**
4. **关注数据源的超时和重试**
5. **数据源连接池要合理配置**
