# 认证与授权

## 一、JWT 认证

```typescript
// Apollo Server 认证
const server = new ApolloServer({
  schema,
  context: async ({ req }) => {
    const token = req.headers.authorization?.replace('Bearer ', '');
    if (!token) return { user: null };

    try {
      const decoded = jwt.verify(token, SECRET);
      const user = await userService.findById(decoded.userId);
      return { user };
    } catch (err) {
      return { user: null };
    }
  },
});

// Resolver 中检查认证
const resolvers = {
  Query: {
    me: (_, __, { user }) => {
      if (!user) throw new AuthenticationError('请先登录');
      return user;
    },
    allUsers: (_, __, { user }) => {
      if (user?.role !== 'ADMIN')
        throw new ForbiddenError('需要管理员权限');
      return userService.findAll();
    },
  },
};
```

## 二、指令式授权

```graphql
directive @auth(requires: Role!) on FIELD_DEFINITION

enum Role {
  ADMIN
  CUSTOMER
  VENDOR
}

type Query {
  users: [User!]! @auth(requires: ADMIN)
  me: User! @auth(requires: CUSTOMER)
  publicProducts: [Product!]!    # 无需认证
}
```

```java
// 自定义指令实现
public class AuthDirective implements SchemaDirectiveWiring {

    @Override
    public GraphQLFieldDefinition onField(
            SchemaDirectiveWiringEnvironment<GraphQLFieldDefinition> env) {
        Role requiredRole = env.getDirective()
            .getArgument("requires").getValue();

        DataFetcher<?> original = env.getFieldDataFetcher();

        return env.setField(env.getField().transform(builder ->
            builder.dataFetcher(env -> {
                UserContext ctx = env.getContext();
                if (ctx.getUser() == null) {
                    throw new AuthenticationException("未登录");
                }
                if (!ctx.getUser().hasRole(requiredRole)) {
                    throw new AccessDeniedException("权限不足");
                }
                return original.get(env);
            })));
    }
}
```

## 三、字段级权限

```graphql
type User {
  id: ID!
  name: String!
  email: String! @auth(requires: ADMIN)      # 只有管理员可见
  ssn: String! @auth(requires: SELF)         # 只有本人可见
  phone: String @auth(requires: AUTHENTICATED) # 登录用户可见
}
```

## 四、注意事项

1. **Context 是认证信息的载体**
2. **指令式授权保持 Schema 清晰**
3. **字段级权限要细化**
4. **生产环境用 OAuth2/OIDC**
5. **审计日志记录敏感操作**
