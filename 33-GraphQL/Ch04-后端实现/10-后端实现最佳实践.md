# 后端实现最佳实践

## 一、架构分层

```yaml
推荐分层:
  Controller 层:
    - Resolver 定义
    - 参数解析
    - 权限检查
    - 错误包装

  Service 层:
    - 业务逻辑
    - 事务管理
    - 数据校验

  Repository 层:
    - 数据访问
    - 查询优化
    - 缓存集成
```

## 三、实践清单

```yaml
Schema:
  □ Schema First 设计
  □ SDL 文件模块化
  □ 命名规范统一
  □ 遵循 Relay 规范

Resolver:
  □ 业务逻辑在 Service 层
  □ 关联字段用 DataLoader
  □ 错误规范化处理
  □ 权限检查

性能:
  □ DataLoader 批量加载
  □ 查询深度/复杂度限制
  □ 缓存策略配置
  □ 慢查询监控

安全:
  □ 认证集成
  □ 授权指令
  □ 输入验证
  □ 生产关闭自省
```

## 四、注意事项

1. **项目结构要清晰**，按领域组织
2. **Schema 和 Resolver 对应关系要明确**
3. **错误信息要友好且安全**
4. **性能优化要持续关注**
5. **测试覆盖 Resolver 逻辑**
