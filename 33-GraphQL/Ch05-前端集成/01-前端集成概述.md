# 前端集成概述

## 一、客户端库对比

```yaml
Apollo Client:
  优点:
    - 功能最全面
    - 标准化缓存
    - React 深度集成
    - DevTools 完善
  适用: 大中型项目

urql:
  优点:
    - 轻量级
    - 可扩展 Exchanges
    - Formidable 出品
  适用: 中小型项目

graphql-request:
  优点:
    - 极简
    - 无缓存逻辑
    - 体积小
  适用: 简单场景

Relay:
  优点:
    - Facebook 出品
    - 编译时优化
    - 强制规范
  适用: 大型项目
```

## 三、Apollo Client 初始化

```typescript
// React + Apollo Client
import { ApolloClient, InMemoryCache, HttpLink } from '@apollo/client';
import { setContext } from '@apollo/client/link/context';

const httpLink = new HttpLink({ uri: 'http://localhost:4000/graphql' });

const authLink = setContext((_, { headers }) => {
  const token = localStorage.getItem('token');
  return {
    headers: {
      ...headers,
      authorization: token ? `Bearer ${token}` : '',
    },
  };
});

export const client = new ApolloClient({
  link: authLink.concat(httpLink),
  cache: new InMemoryCache(),
});
```

## 四、Apollo Client React 集成

```tsx
import { ApolloProvider, useQuery, useMutation, gql } from '@apollo/client';
import { client } from './apollo';

// 查询 Hook
const GET_USERS = gql`
  query GetUsers($first: Int!) {
    users(first: $first) {
      edges {
        node { id name email }
      }
    }
  }
`;

function UserList() {
  const { data, loading, error, refetch, fetchMore } = useQuery(GET_USERS, {
    variables: { first: 10 },
    notifyOnNetworkStatusChange: true,
  });

  if (loading && !data) return <Loading />;
  if (error) return <Error error={error} onRetry={refetch} />;

  return (
    <div>
      {data.users.edges.map(({ node }) => (
        <UserCard key={node.id} user={node} />
      ))}
    </div>
  );
}

// 变更 Hook
const CREATE_USER = gql`
  mutation CreateUser($input: CreateUserInput!) {
    createUser(input: $input) {
      id
      name
    }
  }
`;

function CreateUserForm() {
  const [createUser, { loading, error }] = useMutation(CREATE_USER, {
    // 变更后更新缓存
    update(cache, { data: { createUser } }) {
      cache.modify({
        fields: {
          users(existingUsers = { edges: [] }) {
            const newUserRef = cache.writeFragment({
              data: createUser,
              fragment: gql`fragment NewUser on User { id name }`,
            });
            return {
              ...existingUsers,
              edges: [...existingUsers.edges, { node: newUserRef, cursor: '' }],
            };
          },
        },
      });
    },
    onCompleted: () => toast.success('创建成功'),
    onError: (err) => toast.error(err.message),
  });

  const handleSubmit = (input: CreateUserInput) => {
    createUser({ variables: { input } });
  };

  return <UserForm onSubmit={handleSubmit} loading={loading} />;
}

// 应用根组件
function App() {
  return (
    <ApolloProvider client={client}>
      <UserList />
      <CreateUserForm />
    </ApolloProvider>
  );
}
```

## 五、错误处理全局配置

```typescript
// 全局错误链接
import { onError } from '@apollo/client/link/error';

const errorLink = onError(({ graphQLErrors, networkError }) => {
  if (graphQLErrors) {
    graphQLErrors.forEach(({ message, extensions }) => {
      switch (extensions?.code) {
        case 'UNAUTHENTICATED':
          // 跳转登录页
          window.location.href = '/login';
          break;
        case 'FORBIDDEN':
          toast.error('无权访问');
          break;
        default:
          console.error(`GraphQL Error: ${message}`);
      }
    });
  }

  if (networkError) {
    toast.error('网络错误，请检查连接');
  }
});

const client = new ApolloClient({
  link: ApolloLink.from([errorLink, authLink, httpLink]),
  cache: new InMemoryCache(),
});
```

## 六、注意事项

1. **Apollo Client 是最成熟的选择**
2. **缓存策略要根据业务设计**
3. **类型安全用 Codegen 保证**
4. **错误边界要全局处理**
5. **加载状态要有良好的 UX**
6. **ApolloProvider 放在应用根组件**
7. **errorLink 统一处理认证和网络错误**
