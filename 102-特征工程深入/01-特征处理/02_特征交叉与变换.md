# 特征交叉与变换 - 特征工程深入


## 1. 数值特征变换


| 变换方法 | 公式 | 适用场景 | 注意事项 |
| --- | --- | --- | --- |
| 标准化（Z-score） | x' = (x - μ) / σ | 大多数模型，尤其是SVM、神经网络 | 对离群值敏感 |
| 归一化（Min-Max） | x' = (x - min) / (max - min) | 需要固定范围[0,1]的场景 | 受离群值影响大 |
| 对数变换 | x' = log(x + 1) | 右偏分布数据（收入、点击数） | 要求x ≥ 0 |
| Box-Cox变换 | x' = (x^λ - 1) / λ | 需要自动寻找最优变换 | 要求x > 0 |
| Yeo-Johnson变换 | 扩展Box-Cox | 数据包含负值和零 | Box-Cox的通用版本 |
| 分位数变换 | 映射到均匀/正态分布 | 需要严格正态分布 | 可能破坏原始关系 |


> **Note:** **何时使用哪种变换：**
> 首先检查特征的分布（直方图、Q-Q图）。右偏数据用log变换，需要标准正态用Box-Cox/Yeo-Johnson，简单场景用标准化即可。树模型（XGBoost、LightGBM）通常不需要数值变换。


## 2. 类别特征编码


| 编码方法 | 原理 | 适用场景 | 问题 |
| --- | --- | --- | --- |
| One-Hot | 每个类别一个二值特征 | 类别数少（<20） | 高基数时维度爆炸 |
| Label Encoding | 类别映射为整数 | 有序类别、树模型 | 无序类别引入假序关系 |
| Target Encoding | 用目标变量的统计量编码 | 高基数类别 | 容易过拟合，需正则化 |
| Frequency Encoding | 用类别出现频率编码 | 简单高效 | 不同频率的类别可能编码相同 |
| Binary Encoding | 整数转二进制后拆位 | 中等基数类别 | 部分信息丢失 |
| Embedding | 学习稠密向量表示 | 超高基数（用户ID、商品ID） | 需要训练 |


> **Example:** **Target Encoding示例：**
>
>
> 城市=北京，目标=购买率，训练集中北京用户的购买率为0.35 → 编码值为0.35
>
>
> 为防止过拟合，通常使用平滑：encoding = (n × mean_cat + m × global_mean) / (n + m)


## 3. 特征交叉


### 多项式特征


自动生成特征的多项式组合，如x1, x2 → x1, x2, x1², x2², x1×x2。


### 手动交叉


基于业务理解手动构造有意义的交叉特征：


- 收入 / 年龄 = 年均收入
- 消费金额 / 消费次数 = 客单价
- 城市 × 商品类目 = 地域品类偏好
- 年龄 × 性别 = 人群特征


```
import pandas as pd
from sklearn.preprocessing import PolynomialFeatures

# 多项式特征
df = pd.DataFrame({'age': [25, 30, 35], 'income': [5000, 8000, 12000]})
poly = PolynomialFeatures(degree=2, interaction_only=False, include_bias=False)
X_poly = poly.fit_transform(df)
print(f"特征名: {poly.get_feature_names_out()}")
# 输出: ['age', 'income', 'age^2', 'age income', 'income^2']

# 手动交叉特征
df['avg_spend'] = df['income'] * 0.3  # 假设消费占比30%
df['income_per_age'] = df['income'] / df['age']
```


## 4. 时间特征工程


时间特征是推荐系统、CTR预估、销量预测中最重要的特征之一。


| 特征类型 | 具体特征 | 应用场景 |
| --- | --- | --- |
| 基础时间 | 年、月、日、小时、分钟、星期几 | 通用 |
| 周期性编码 | sin(2π×h/24), cos(2π×h/24) | 保持小时的周期性（23点和0点接近） |
| 时间差 | 距上次登录/购买/浏览的时间 | 用户行为建模 |
| 时间窗口统计 | 过去7天/30天的点击数、消费额 | 用户活跃度、消费能力 |
| 节假日 | 是否节假日、距最近节假日天数 | 电商、出行 |
| 时间段 | 凌晨/早晨/中午/下午/晚上/深夜 | 用户行为模式 |


> **Important:** **周期性编码的重要性：**
> 小时23和小时0在距离上很近（相差1小时），但如果直接用整数23和0，距离为23。使用sin/cos编码可以正确表达这种周期性关系。


## 5. 文本特征与数据泄漏防范


### 文本特征


- **TF-IDF：**
   将文本转为稀疏向量，衡量词的重要性
- **统计特征：**
   文本长度、词数、标点符号数、特殊字符数
- **Embedding：**
   使用预训练模型（BERT）获取稠密向量


### 数据泄漏防范


> **Important:** **铁律：**
> 所有变换必须在训练集上fit（学习参数），然后用学到的参数transform训练集和测试集。绝不能用测试集的信息来fit。


```
from sklearn.preprocessing import StandardScaler
from sklearn.model_selection import train_test_split

X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

# 正确做法：在训练集上fit，同时transform训练集和测试集
scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)   # fit + transform
X_test_scaled = scaler.transform(X_test)          # 仅transform

# 错误做法（数据泄漏）：
# X_all_scaled = scaler.fit_transform(X)  # 测试集信息泄漏到训练

# 对于Target Encoding，使用交叉验证避免泄漏
from sklearn.model_selection import KFold
import numpy as np

def safe_target_encode(X_train, y_train, cat_col, target_col, n_folds=5):
    """安全的Target Encoding（交叉验证）"""
    encoded = np.zeros(len(X_train))
    kf = KFold(n_splits=n_folds, shuffle=True, random_state=42)
    global_mean = y_train.mean()

    for train_idx, val_idx in kf.split(X_train):
        # 在fold的训练部分计算编码
        fold_encoding = X_train.iloc[train_idx].groupby(cat_col)[target_col].mean()
        # 应用到验证部分
        encoded[val_idx] = X_train.iloc[val_idx][cat_col].map(fold_encoding).fillna(global_mean)

    return encoded
```


## 总结


- 数值变换包括标准化、归一化、对数变换、Box-Cox等，树模型通常不需要
- 类别编码中Target Encoding对高基数类别效果最好，但需防过拟合
- 特征交叉分为多项式自动交叉和基于业务理解的手动交叉
- 时间特征工程需注意周期性编码（sin/cos），避免周期边界不连续
- 数据泄漏是最危险的错误：必须在训练集上fit，测试集仅transform
- Target Encoding应使用交叉验证方式避免泄漏
