# 04-计算复杂性

## 1. 时间复杂度与空间复杂度的定义

### 时间复杂度

**定义**：图灵机 $M$ 在输入 $w$ 上的**运行时间**是 $M$ 在 $w$ 上停机前执行的步数。

**定义**：$M$ 的**时间复杂度**是函数 $f: \mathbb{N} \to \mathbb{N}$，其中 $f(n)$ 是 $M$ 在所有长度为 $n$ 的输入上的最大运行时间：

$$f(n) = \max\{|w| = n \mid M \text{ 在 } w \text{ 上的步数}\}$$

**注**：如果 $M$ 在某些长度为 $n$ 的输入上不停机，则时间复杂度无定义。

### 空间复杂度

**定义**：$M$ 在输入 $w$ 上的**空间使用量**是读写头访问过的不同带子格数。

**定义**：$M$ 的**空间复杂度**是函数 $f(n)$，表示 $M$ 在所有长度为 $n$ 的输入上的最大空间使用量。

**输入不计空间模型**：只计算超出输入部分的带子空间（只读输入带 + 工作带）。这使得 $s(n) < n$ 成为有意义的分析对象。

## 2. 大O、大Ω、大Θ 记号

### 大O（上界）

$$f(n) = O(g(n)) \iff \exists c > 0, n_0 > 0, \forall n \geq n_0: f(n) \leq c \cdot g(n)$$

例：$3n^2 + 5n = O(n^2)$。

### 大Ω（下界）

$$f(n) = \Omega(g(n)) \iff \exists c > 0, n_0 > 0, \forall n \geq n_0: f(n) \geq c \cdot g(n)$$

例：$3n^2 + 5n = \Omega(n^2)$。

### 大Θ（紧界）

$$f(n) = \Theta(g(n)) \iff f(n) = O(g(n)) \text{ 且 } f(n) = \Omega(g(n))$$

例：$3n^2 + 5n = \Theta(n^2)$。

### 小o与小ω

- $f(n) = o(g(n))$：$f$ 严格小于 $g$ 的上界（$\lim_{n\to\infty} f(n)/g(n) = 0$）
- $f(n) = \omega(g(n))$：$f$ 严格大于 $g$ 的下界

**在复杂性理论中的意义**：我们通常忽略常数因子和低阶项，关注函数的增长速率类别（多项式、指数、对数等）。

---

## 3. P 类问题（多项式时间可判定）

**定义**：

$$\text{P} = \{L \mid L \text{ 被某个确定性 TM 在 } O(n^k) \text{ 时间内判定}\}$$

其中 $k$ 是常数，$n$ 是输入长度。

**P 中的经典问题**：

- 最短路径问题（Dijkstra 算法）
- 最大流问题（Ford-Fulkerson 算法）
- 二分图匹配
- 线性规划
- 判定一个数是否为素数（AKS 算法）

**P 的意义**：P 被视为"实际可解"问题的数学刻画——多项式时间算法在实践中通常是高效的。

**注意**：$O(n^{100})$ 在理论上属于 P，但在实践中可能不可行。但经验表明，多项式时间算法一旦被发现，通常能找到低次的版本。

---

## 4. NP 类问题（非确定性多项式时间）

**定义**：

$$\text{NP} = \{L \mid L \text{ 被某个非确定性 TM 在 } O(n^k) \text{ 时间内判定}\}$$

**等价定义（验证机）**：

$$\text{NP} = \{L \mid \text{存在多项式时间 TM } V \text{（验证机）和多项式 } p \text{，使得}$$

$$L = \{x \mid \exists c, |c| \leq p(|x|), V(x, c) = \text{accept}\}\}$$

其中 $c$ 称为**证书（certificate）**或**见证（witness）**。

**直观理解**：NP 中的问题，验证一个"是"的答案可以在多项式时间内完成。

**NP 中的经典问题**：

- SAT（布尔可满足性问题）
- 旅行商问题（TSP）的判定版本
- 哈密顿回路问题
- 图的团问题（CLIQUE）
- 子集和问题（SUBSET-SUM）
- 图着色问题

**P 与 NP 的关系**：

- 显然 $\text{P} \subseteq \text{NP}$（确定性 TM 是非确定性 TM 的特例）
- $\text{P} = \text{NP}$ 是否成立是计算机科学中最重要的开放问题

---

## 5. NP 完全性（NPC）的定义

**定义**：语言 $L$ 是 NP 完全的，如果：

1. $L \in \text{NP}$
2. 对所有 $A \in \text{NP}$，$A \leq_P L$（$A$ 多项式时间归约到 $L$）

**含义**：NPC 问题是 NP 中"最难"的问题——如果任何一个 NPC 问题有多项式时间算法，则 $\text{P} = \text{NP}$。

**推论**：若存在 NPC 问题不属于 P，则 $\text{P} \neq \text{NP}$。

**多项式时间归约 $A \leq_P B$**：存在多项式时间可计算的函数 $f$，使得 $x \in A \iff f(x) \in B$。

---

## 6. Cook-Levin 定理（SAT 是 NP 完全的）

**定理（Cook-Levin, 1971）**：布尔可满足性问题 SAT 是 NP 完全的。

**SAT 问题**：给定布尔公式 $\phi$（合取范式），是否存在变量的真值赋值使得 $\phi$ 为真？

**证明思路**：

1. SAT $\in$ NP：猜测一个赋值，验证它使 $\phi$ 为真——多项式时间
2. 对任意 $L \in$ NP，设 $N$ 是判定 $L$ 的多项式时间 NTM
3. 对任意输入 $w$，构造布尔公式 $\phi_w$：
   - $\phi_w$ 的变量表示 $N$ 在 $w$ 上的计算过程中的"格局"（状态、带子内容、读写头位置）
   - $\phi_w$ 的子句确保：每一步的转移合法、初始格局正确、最终到达接受状态
   - $\phi_w$ 可满足 $\iff$ $N$ 接受 $w$
4. $\phi_w$ 的构造是多项式时间的

这是计算理论中最深刻的结果之一——它开辟了通过归约证明 NP 完全性的道路。

---

## 7. 经典 NP 完全问题

以下问题均为 NP 完全的（按证明顺序）：

| 问题 | 定义 | 归约自 |
|------|------|--------|
| **SAT** | 合取范式布尔公式可满足？ | Cook-Levin 定理 |
| **3-SAT** | 每个子句恰含 3 个文字的 SAT | SAT |
| **CLIQUE** | 图中是否存在大小为 $k$ 的团？ | 3-SAT |
| **VERTEX-COVER** | 图中是否存在大小为 $k$ 的顶点覆盖？ | CLIQUE |
| **HAM-CYCLE** | 图中是否存在哈密顿回路？ | VERTEX-COVER |
| **TSP** | 给定距离和上界 $k$，是否存在环游 $\leq k$？ | HAM-CYCLE |
| **SUBSET-SUM** | 集合中是否存在子集和为 $t$？ | 3-SAT |
| **3-COLOR** | 图是否可 3-着色？ | 3-SAT |
| **PARTITION** | 集合能否分为两个和相等的子集？ | SUBSET-SUM |

## 8. NP 完全性证明方法

**基本框架**：证明问题 $B$ 是 NP 完全的：

1. 证明 $B \in$ NP（给出多项式时间验证机）
2. 选择已知的 NP 完全问题 $A$
3. 构造多项式时间归约 $f: A \leq_P B$
   - 对 $A$ 的每个实例 $x$，构造 $B$ 的实例 $f(x)$
   - 证明 $x \in A \iff f(x) \in B$
   - 证明 $f$ 的计算时间是多项式的

**常见归约技巧**：

- **从 3-SAT 出发**：大多数经典归约都以 3-SAT 为起点
- **构造"小工具（gadget）"**：用 $B$ 中的结构模拟 $A$ 中的逻辑关系
- **变量对应 vs 约束对应**：将变量和约束分别映射到 $B$ 实例的不同部分

## 9. NP 难问题

**定义**：问题 $L$ 是 NP 难的（NP-hard），如果对所有 $A \in$ NP，$A \leq_P L$。

**区别**：NP 完全 = NP 难 + 属于 NP。NP 难问题不需要属于 NP。

**NP 难的经典例子**：

- **停机问题**：NP 难但不是 NP（甚至不是递归的）
- **TSP 优化版本**：求最短环游长度（属于 NP-hard，但不属于 NP，因为优化问题不是判定问题）
- **图着色的优化版本**：求最小着色数
- **最大团问题的优化版本**

**关系总结**：

$$\text{P} \subseteq \text{NP} \subseteq \text{PSPACE}$$

NPC $\subseteq$ NP，且 NPC 是 NP 中最难的。

---

## 10. P=NP 问题及其意义

**核心问题**：P = NP？

**如果 P = NP**：

- 所有 NP 问题（包括 SAT、TSP、背包等）都有多项式时间算法
- 现代密码学的大部分基础崩溃（公钥加密依赖于某些问题的困难性）
- 数学证明可以高效搜索
- 蛋白质折叠、调度优化等实际问题可高效求解

**如果 P $\neq$ NP**：

- 某些问题本质上没有高效算法
- 现代密码学的基础得到支持
- NP 完全问题只能用近似算法或启发式方法处理

**主流猜想**：绝大多数计算机科学家相信 P $\neq$ NP。

**Clay 千禧年问题**：P vs NP 是七个千禧年大奖难题之一，悬赏 100 万美元。

## 11. 空间复杂性

### PSPACE

**定义**：

$$\text{PSPACE} = \{L \mid L \text{ 被某个确定性 TM 在 } O(n^k) \text{ 空间内判定}\}$$

### L 和 NL

**L（对数空间）**：确定性 TM 在 $O(\log n)$ 空间内可判定的语言。

**NL（非确定性对数空间）**：非确定性 TM 在 $O(\log n)$ 空间内可判定的语言。

**已知包含关系**：

$$\text{P} \subseteq \text{NP} \subseteq \text{PSPACE}$$

$$\text{L} \subseteq \text{NL} \subseteq \text{P}$$

并且 $\text{L} \subseteq \text{PSPACE}$。

**NL 中的问题**：PATH（图中从 $s$ 到 $t$ 是否有路径）是 NL 完全的。

### Savitch 定理

**定理（Savitch）**：$\text{NSPACE}(f(n)) \subseteq \text{DSPACE}(f(n)^2)$。

**推论**：$\text{NL} \subseteq \text{DSPACE}(\log^2 n) \subseteq \text{P}$。

**证明思路**：非确定性 TM 的配置图大小为 $2^{O(f(n))}$，可以用递归的"可达性"算法在 $O(f(n)^2)$ 空间内判断是否存在路径。

## 12. 多项式分层（PH）基础

**定义**：多项式分层（Polynomial Hierarchy）由以下层级定义：

- $\Sigma_0^P = \Pi_0^P = \text{P}$
- $\Sigma_{k+1}^P = \text{NP}^{\Sigma_k^P}$（使用 $\Sigma_k^P$ 预言机的 NP 问题）
- $\Pi_{k+1}^P = \text{coNP}^{\Sigma_k^P}$
- $\Delta_{k+1}^P = \text{P}^{\Sigma_k^P}$

$$\text{PH} = \bigcup_{k \geq 0} \Sigma_k^P$$

**直观理解**：

- $\Sigma_1^P = \text{NP}$（存在量词）
- $\Pi_1^P = \text{coNP}$（全称量词）
- $\Sigma_2^P$：$\exists \forall$ 形式的问题
- $\Pi_2^P$：$\forall \exists$ 形式的问题

**PH 中的问题**：

- MIN-CIRCUIT（求最小布尔电路）属于 $\Sigma_2^P$
- 图同构问题在 PH 中（未确定是否在 NP 完全中）

**重要猜想**：

- 若 P = NP，则 PH 塌缩到 P
- 若 NPC $\subseteq$ coNP，则 PH 塌缩到 $\Sigma_2^P$
- 许多复杂性理论家相信 PH 是严格分层的（不会塌缩）
