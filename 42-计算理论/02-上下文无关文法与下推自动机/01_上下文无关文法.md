# 上下文无关文法 (CFG)


## 一、上下文无关文法的定义


上下文无关文法 (CFG) 是一个四元组 `G = (V, Σ, R, S)`：


- **V**
   — 有穷的
   **变量**
   （非终结符）集合
- **Σ**
   — 有穷的
   **终结符**
   集合（V ∩ Σ = ∅）
- **R**
   — 有穷的
   **产生式规则**
   集合，每条规则形式为
   `A → α`
   ，其中 A ∈ V，α ∈ (V ∪ Σ)*
- **S**
   — 起始变量 (S ∈ V)


"上下文无关"意味着产生式左侧**只有一个变量**，与上下文无关。


### 示例：算术表达式文法


> **Example:** ```
> E → E + T | T        # 表达式 = 表达式 + 项 | 项
> T → T * F | F        # 项 = 项 * 因子 | 因子
> F → ( E ) | a        # 因子 = (表达式) | 标识符a
> ```
>
>
> 可生成串：`a + a * a`，`(a + a) * a` 等


## 二、推导 (Derivation)


从起始变量 S 开始，反复用产生式替换变量，直到得到全是终结符的串。


### 推导类型


| 类型 | 替换方式 | 特点 |
| --- | --- | --- |
| **最左推导** | 每步替换最左边的变量 | 最常用，用于自顶向下分析 |
| **最右推导** | 每步替换最右边的变量 | 又称规范推导，用于自底向上分析 |
| **任意推导** | 每步替换任意位置的变量 | 最一般的形式 |


### 最左推导示例：a + a * a


> **Example:** ```
> E ⟹ E + T          （用 E → E + T）
>   ⟹ T + T          （用 E → T，最左E替换为T）
>   ⟹ F + T          （用 T → F）
>   ⟹ a + T          （用 F → a）
>   ⟹ a + T * F      （用 T → T * F）
>   ⟹ a + F * F      （用 T → F）
>   ⟹ a + a * F      （用 F → a）
>   ⟹ a + a * a      （用 F → a）
> ```


## 三、解析树 (Parse Tree)


解析树用树形结构表示推导过程：


- **根节点**
   = 起始变量 S
- **内部节点**
   = 变量（非终结符）
- **叶子节点**
   = 终结符或 ε
- **从左到右**
   读叶子节点得到生成的串

## 四、歧义性 (Ambiguity)


若一个文法对某个串可产生**两棵不同的解析树**（或两种不同的最左推导），则该文法是**歧义的**。


### 歧义文法示例


```
E → E + E | E * E | a
```


串 `a + a * a` 有两种解析：


- 解析1：
   `(a + a) * a`
   — 先算加后算乘
- 解析2：
   `a + (a * a)`
   — 先算乘后算加


**消除歧义的方法：**引入优先级和结合性（如前面的 E→E+T | T 文法）


### 固有歧义性


某些语言是**固有歧义的**——任何生成它的文法都是歧义的。


例：`L = {aⁿbⁿcᵐ | n,m ≥ 0} ∪ {aⁿbᵐcᵐ | n,m ≥ 0}`


## 五、Chomsky范式 (CNF)


Chomsky标准型要求每条产生式只有以下两种形式之一：


| 形式 | 示例 |
| --- | --- |
| `A → BC`（两个变量） | S → AB |
| `A → a`（一个终结符） | A → a |


特殊情况：允许 `S → ε`（S不能出现在任何产生式右侧）。


### 转为CNF的步骤


1. 添加新起始变量 S₀
2. 消除 ε-产生式（A → ε）
3. 消除单元产生式（A → B）
4. 转换剩余产生式为标准形式


> **Theorem:** **定理：**任何上下文无关语言都可以用Chomsky范式文法生成。
>
>
> **推论：**若G是CNF且|w|=n，则w的解析树恰好有2n-1步推导。这保证了**CYK算法**的 O(n³) 时间复杂度。


## 六、上下文无关语言的运算


| 运算 | 是否封闭 | 方法 |
| --- | --- | --- |
| 并集 ∪ | 封闭 | 新起始变量 S → S₁ \| S₂ |
| 连接 ∘ | 封闭 | 新起始变量 S → S₁S₂ |
| 克林闭包 * | 封闭 | S → SS₁ \| ε |
| 交集 ∩ | 不封闭 | 反例：{aⁿbⁿcᵐ} ∩ {aⁿbᵐcᵐ} = {aⁿbⁿcⁿ} |
| 补集 ¬ | 不封闭 | 交集不封闭 ⇒ 补集也不封闭 |


## 七、CYK成员判定算法


CYK (Cocke-Younger-Kasami) 算法用**动态规划**判断串是否属于CNF文法。


```
# CYK算法伪代码
function CYK(w, G_CNF):
    n = len(w)
    # table[i][j] = 能生成子串 w[i..j] 的变量集合
    table = array[n][n] of sets

    # 基础：长度为1的子串
    for i = 0 to n-1:
        table[i][i] = { A | A → w[i] in G }

    # 动态规划：长度为2到n的子串
    for length = 2 to n:
        for i = 0 to n-length:
            j = i + length - 1
            for k = i to j-1:
                for each production A → BC:
                    if B in table[i][k] and C in table[k+1][j]:
                        table[i][j] |= {A}

    return S in table[0][n-1]  # 起始变量能否生成整个串
```


时间复杂度 O(n³) 空间复杂度 O(n²)
