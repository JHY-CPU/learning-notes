# 移动端原生模块通信

## 一、概念说明

跨平台框架需要通过桥接机制与原生平台通信，调用平台特有的 API 和硬件功能。

```javascript
// 通信模型
/*
JS/Dart <--> Bridge/Channel <--> Native Code (Java/Kotlin/ObjC/Swift)

通信类型:
- 方法调用 (Method Channel) - 异步方法
- 事件流 (Event Channel) - 持续数据流
- 消息传递 (Message Channel) - 结构化消息
*/
```

## 二、React Native 原生模块

```javascript
// JavaScript 调用原生
import { NativeModules } from 'react-native';
const { DeviceInfo } = NativeModules;

const getInfo = async () => {
  const info = await DeviceInfo.getDeviceInfo();
  console.log(info);
};
```

```kotlin
// Android 实现
@ReactMethod
fun getDeviceInfo(promise: Promise) {
  val info = Arguments.createMap().apply {
    putString("brand", Build.BRAND)
    putString("model", Build.MODEL)
    putString("version", Build.VERSION.RELEASE)
  }
  promise.resolve(info)
}
```

## 三、Flutter Platform Channel

```dart
// Flutter 调用原生
class BatteryService {
  static const _channel = MethodChannel('app/battery');

  static Future<int> getBatteryLevel() async {
    try {
      final level = await _channel.invokeMethod<int>('getBatteryLevel');
      return level ?? -1;
    } on PlatformException {
      return -1;
    }
  }
}
```

```swift
// iOS 实现
let channel = FlutterMethodChannel(
  name: "app/battery",
  binaryMessenger: controller.binaryMessenger
)

channel.setMethodCallHandler { call, result in
  if call.method == "getBatteryLevel" {
    UIDevice.current.isBatteryMonitoringEnabled = true
    let level = Int(UIDevice.current.batteryLevel * 100)
    result(level)
  }
}
```

## 五、注意事项

1. **线程安全**：原生方法在后台线程，更新 UI 需切换到主线程
2. **类型映射**：确保 Dart/JS 与原生之间的类型正确转换
3. **错误处理**：原生端的异常需要正确传递到 JS/Dart 端
4. **内存管理**：注意原生对象的生命周期管理
5. **平台差异**：同一功能在不同平台可能有不同实现
