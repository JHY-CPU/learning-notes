# 移动端设计原则

## 一、概念说明

移动端 UI 设计需要遵循一系列设计原则，确保用户获得流畅、直观的使用体验。核心原则包括一致性、简洁性、可访问性和反馈性。

```dart
// 设计原则示例 - 一致性原则
// 所有按钮使用统一的样式规范
class AppButtonStyles {
  // 主按钮
  static final primary = ElevatedButton.styleFrom(
    minimumSize: const Size(double.infinity, 48),
    shape: RoundedRectangleBorder(borderRadius: BorderRadius.circular(8)),
    textStyle: const TextStyle(fontSize: 16, fontWeight: FontWeight.w600),
  );

  // 次按钮
  static final secondary = OutlinedButton.styleFrom(
    minimumSize: const Size(double.infinity, 48),
    shape: RoundedRectangleBorder(borderRadius: BorderRadius.circular(8)),
    side: const BorderSide(color: Colors.blue),
  );
}
```

## 二、设计规范

### 2.1 间距系统

```dart
// 统一的间距系统
class AppSpacing {
  static const double xs = 4.0;   // 极小间距
  static const double sm = 8.0;   // 小间距
  static const double md = 16.0;  // 标准间距
  static const double lg = 24.0;  // 大间距
  static const double xl = 32.0;  // 特大间距
  static const double xxl = 48.0; // 超大间距
}

// 使用示例
Padding(
  padding: const EdgeInsets.all(AppSpacing.md),
  child: Column(
    children: [
      Text('标题'),
      SizedBox(height: AppSpacing.sm),
      Text('内容'),
      SizedBox(height: AppSpacing.md),
      ElevatedButton(onPressed: () {}, child: Text('操作')),
    ],
  ),
)
```

// 文字层级系统

## 四、设计注意事项

1. **触摸目标**：最小可点击区域 44pt/48dp，确保手指可轻松操作
2. **信息层次**：通过大小、颜色、间距建立视觉层次
3. **留白设计**：合理使用留白，避免界面拥挤
4. **色彩限制**：主色不超过 3 种，辅助色用于强调
5. **图标风格**：全应用使用统一风格的图标（线性或填充）
