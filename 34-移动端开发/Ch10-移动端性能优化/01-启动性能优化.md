# 启动性能优化

## 一、概念说明

应用启动时间直接影响用户体验和留存率。优化启动性能需要减少初始化时间、延迟加载非关键资源。

```dart
// 启动阶段分析
/*
1. 冷启动 - 进程创建到首屏显示
2. 热启动 - 已有进程，恢复到前台
3. 首屏渲染 - 用户可交互时间

优化目标:
- 冷启动 < 2秒
- 热启动 < 500ms
- 首屏可交互 < 1秒
*/
```

## 二、启动优化策略

```dart
// Flutter 启动优化
void main() async {
  // 确保 Flutter 绑定初始化
  WidgetsFlutterBinding.ensureInitialized();

  // 并行初始化服务
  await Future.wait([
    _initDatabase(),
    _initCrashlytics(),
    _loadConfig(),
  ]);

  runApp(MyApp());
}

// 延迟初始化非关键服务
class MyApp extends StatefulWidget {
  @override
  State<MyApp> createState() => _MyAppState();
}

class _MyAppState extends State<MyApp> {
  @override
  void initState() {
    super.initState();
    // 使用 addPostFrameCallback 延迟执行
    WidgetsBinding.instance.addPostFrameCallback((_) {
      _initNonCriticalServices();
    });
  }

  void _initNonCriticalServices() async {
    // 首帧渲染后初始化
    await Future.wait([
      _initAnalytics(),
      _initPushNotifications(),
      _preloadImages(),
    ]);
  }
}
```

## 三、代码分割与懒加载

```dart
// 路由懒加载
final router = GoRouter(
  routes: [
    GoRoute(path: '/', builder: (_, __) => const HomePage()),
    // 懒加载路由
    GoRoute(
      path: '/settings',
      builder: (_, __) {
        // 动态导入
        return const SettingsPage();
      },
    ),
  ],
);

// Widget 懒加载
class LazyWidget extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return FutureBuilder(
      future: _loadData(),
      builder: (context, snapshot) {
        if (snapshot.connectionState == ConnectionState.waiting) {
          return const CircularProgressIndicator();
        }
        return DataWidget(data: snapshot.data);
      },
    );
  }
}
```

## 四、预加载策略

```dart
// 关键数据预加载
class PreloadService {
  static Future<void> preloadCriticalData() async {
    // 预加载首页数据
    final homeData = await ApiService.getHomeData();
    CacheManager.set('home_data', homeData);

    // 预加载用户信息
    final token = await SecureStorage.read('token');
    if (token != null) {
      final user = await ApiService.getUserInfo();
      CacheManager.set('user_info', user);
    }

    // 预加载关键图片
    await _preloadImages([
      'https://example.com/banner1.webp',
      'https://example.com/banner2.webp',
    ]);
  }

  static Future<void> _preloadImages(List<String> urls) async {
    for (final url in urls) {
      await precacheImage(NetworkImage(url), navigatorKey.currentContext!);
    }
  }
}
```

## 五、注意事项

1. **测量启动时间**：使用 DevTools 或 Firebase Performance 监控
2. **首屏优先**：只初始化首屏需要的服务
3. **避免主线程阻塞**：耗时操作放到后台线程
4. **减少依赖**：评估每个依赖对启动时间的影响
5. **Proguard/R8**：启用代码混淆和优化减小包体积
