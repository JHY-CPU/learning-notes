# 数据缓存策略

## 一、概念说明

数据缓存是提升移动端用户体验和减少网络请求的关键策略。合理的缓存策略能实现秒开效果和离线访问。

```dart
// 缓存策略类型
/*
1. Cache-First - 优先使用缓存，后台更新
2. Network-First - 优先网络请求，失败降级到缓存
3. Cache-Only - 只使用缓存
4. Network-Only - 只使用网络
5. Stale-While-Revalidate - 返回缓存，同时更新
*/
```

## 二、缓存实现

```dart
// 多级缓存管理器
class CacheManager {
  // 内存缓存（最快）
  static final Map<String, CacheEntry> _memoryCache = {};

  // 持久化缓存
  static late Box _diskCache;

  static Future<void> init() async {
    await Hive.initFlutter();
    _diskCache = await Hive.openBox('cache');
  }

  // 获取缓存
  static Future<T?> get<T>(String key) async {
    // 1. 检查内存缓存
    final memoryEntry = _memoryCache[key];
    if (memoryEntry != null && !memoryEntry.isExpired) {
      return memoryEntry.value as T;
    }

    // 2. 检查磁盘缓存
    final diskData = _diskCache.get(key);
    if (diskData != null) {
      final entry = CacheEntry.fromJson(diskData);
      if (!entry.isExpired) {
        // 回写到内存缓存
        _memoryCache[key] = entry;
        return entry.value as T;
      }
    }

    return null;
  }

  // 设置缓存
  static Future<void> set(String key, dynamic value, {
    Duration ttl = const Duration(minutes: 5),
    bool persistent = false,
  }) async {
    final entry = CacheEntry(value: value, expireAt: DateTime.now().add(ttl));

    // 写入内存缓存
    _memoryCache[key] = entry;

    // 写入磁盘缓存
    if (persistent) {
      await _diskCache.put(key, entry.toJson());
    }
  }

  // 清除缓存
  static Future<void> remove(String key) async {
    _memoryCache.remove(key);
    await _diskCache.delete(key);
  }

  static Future<void> clear() async {
    _memoryCache.clear();
    await _diskCache.clear();
  }
}

class CacheEntry {
  final dynamic value;
  final DateTime expireAt;

  CacheEntry({required this.value, required this.expireAt});

  bool get isExpired => DateTime.now().isAfter(expireAt);

  Map<String, dynamic> toJson() => {
    'value': value,
    'expireAt': expireAt.toIso8601String(),
  };

  factory CacheEntry.fromJson(Map<String, dynamic> json) {
    return CacheEntry(
      value: json['value'],
      expireAt: DateTime.parse(json['expireAt']),
    );
  }
}
```

## 三、带缓存的网络请求

```dart
// 缓存策略封装
class CachedApi {
  static Future<T> fetchData<T>(
    String url, {
    Duration cacheTtl = const Duration(minutes: 5),
    bool forceRefresh = false,
    required T Function(dynamic data) parser,
  }) async {
    final cacheKey = 'api_$url';

    // 如果不强制刷新，先检查缓存
    if (!forceRefresh) {
      final cached = await CacheManager.get(cacheKey);
      if (cached != null) {
        // 后台更新缓存
        _refreshCache(url, cacheKey, cacheTtl, parser);
        return parser(cached);
      }
    }

    // 网络请求
    try {
      final response = await HttpClient.get(url);
      final data = parser(response.data);

      // 写入缓存
      await CacheManager.set(cacheKey, response.data, ttl: cacheTtl, persistent: true);

      return data;
    } catch (e) {
      // 网络失败，降级到缓存
      final cached = await CacheManager.get(cacheKey);
      if (cached != null) return parser(cached);
      rethrow;
    }
  }

  static void _refreshCache<T>(
    String url, String cacheKey, Duration ttl, T Function(dynamic) parser,
  ) async {
    try {
      final response = await HttpClient.get(url);
      await CacheManager.set(cacheKey, response.data, ttl: ttl, persistent: true);
    } catch (_) {
      // 静默失败
    }
  }
}
```

## 四、注意事项

1. **缓存键设计**：使用 URL + 参数作为缓存键，避免冲突
2. **缓存容量**：设置缓存大小上限，定期清理过期数据
3. **一致性**：数据更新时清除相关缓存
4. **敏感数据**：敏感数据不应持久化到磁盘
5. **缓存命中率**：监控缓存命中率，调整策略
