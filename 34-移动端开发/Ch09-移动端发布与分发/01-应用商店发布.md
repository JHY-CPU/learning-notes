# 应用商店发布

## 一、概念说明

将移动应用发布到应用商店（App Store、Google Play）需要完成签名、构建、元数据准备和审核提交等步骤。

```bash
# 发布流程
/*
1. 准备应用元数据（名称、描述、截图、图标）
2. 配置签名证书
3. 构建发布版本
4. 上传到应用商店
5. 填写审核信息
6. 提交审核
7. 审核通过后发布
*/
```

## 二、应用元数据

```json
// 应用信息清单
{
  "name": "我的应用",
  "description": "这是一款优秀的移动应用，提供...",
  "keywords": "工具,效率,生活",
  "category": "utilities",
  "privacy_url": "https://example.com/privacy",
  "support_url": "https://example.com/support",
  "screenshots": {
    "iphone_6.5": ["screenshots/iphone_1.png"],
    "ipad_12.9": ["screenshots/ipad_1.png"],
    "android_1080": ["screenshots/android_1.png"]
  },
  "icon": "assets/icon/app_icon.png"
}
```

## 三、版本管理

```yaml
# 版本号规范
# 主版本.次版本.补丁版本+构建号
version: 2.1.3+45

# 更新类型
# 主版本 - 重大架构变更
# 次版本 - 新功能
# 补丁 - Bug修复
# 构建号 - 每次发布递增
```

## 四、审核注意事项

1. **功能完整**：所有功能可正常使用，无明显 Bug
2. **隐私合规**：收集用户数据需有隐私政策
3. **内容合规**：不包含违规内容
4. **权限合理**：只请求必要的权限
5. **崩溃率低**：应用崩溃率控制在 1% 以下

## 五、自动化发布

```yaml
# GitHub Actions 发布流程
name: Release App
on:
  push:
    tags: ['v*']

jobs:
  release:
    runs-on: macos-latest
    steps:
      - uses: actions/checkout@v3
      - name: Build and Deploy
        run: |
          flutter pub get
          flutter build appbundle --release
          fastlane deploy
```

## 六、注意事项

1. **审核周期**：iOS 一般 1-3 天，Android 一般几小时到 3 天
2. **被拒常见原因**：崩溃、功能不完整、隐私问题、私有 API
3. **灰度发布**：先发布到小比例用户，观察稳定性
4. **回滚计划**：准备紧急回滚方案
5. **ASO 优化**：应用名称、关键词、截图影响搜索排名
