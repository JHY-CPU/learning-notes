# 响应式状态管理

## 一、概念说明

响应式状态管理的核心思想是：当数据变化时，依赖该数据的 UI 自动更新。MobX 和 Riverpod 是代表性的响应式方案。

```dart
// MobX 概念
/*
Observable (可观察数据) - 被追踪的状态
Action (动作) - 修改状态的方法
Computed (计算值) - 从其他状态派生的值
Reaction (反应) - 状态变化时执行的副作用
*/
```

## 二、MobX 使用

```javascript
// stores/counterStore.js
import { makeAutoObservable, runInAction } from 'mobx';

class CounterStore {
  count = 0;
  loading = false;

  constructor() {
    makeAutoObservable(this);
  }

  get doubled() {
    return this.count * 2;
  }

  increment() {
    this.count++;
  }

  decrement() {
    this.count--;
  }

  async fetchCount() {
    this.loading = true;
    const result = await api.getCount();
    runInAction(() => {
      this.count = result.value;
      this.loading = false;
    });
  }
}

export const counterStore = new CounterStore();
```

```javascript
// React Native 使用 MobX
import { observer } from 'mobx-react-lite';
import { counterStore } from '../stores/counterStore';

const CounterView = observer(() => {
  return (
    <View>
      <Text>计数: {counterStore.count}</Text>
      <Text>双倍: {counterStore.doubled}</Text>

    </View>

});
```

## 三、Riverpod 使用

```dart
import 'package:flutter_riverpod/flutter_riverpod.dart';

// Provider 定义
final counterProvider = StateNotifierProvider<CounterNotifier, int>((ref) {
  return CounterNotifier();
});

final userProvider = FutureProvider.family<User, int>((ref, userId) async {
  final repository = ref.read(userRepositoryProvider);
  return repository.getUser(userId);
});

// StateNotifier
class CounterNotifier extends StateNotifier<int> {
  CounterNotifier() : super(0);

  void increment() => state++;
  void decrement() => state--;
  void reset() => state = 0;
}

// UI 使用
class CounterPage extends ConsumerWidget {
  const CounterPage({super.key});

  @override
  Widget build(BuildContext context, WidgetRef ref) {
    final count = ref.watch(counterProvider);
    final userAsync = ref.watch(userProvider(1));

    return Column(
      children: [
        Text('计数: $count'),
        ElevatedButton(
          onPressed: () => ref.read(counterProvider.notifier).increment(),
          child: const Text('+1'),
        ),
        userAsync.when(
          loading: () => const CircularProgressIndicator(),
          error: (e, _) => Text('错误: $e'),
          data: (user) => Text('用户: ${user.name}'),
        ),
      ],
    );
  }
}
```

## 四、方案对比

| 特性 | MobX | Riverpod | Bloc |
|------|------|----------|------|
| 响应式 | 自动追踪依赖 | 手动 watch | Event 驱动 |
| 样板代码 | 少 | 少 | 较多 |
| 调试 | 一般 | 一般 | 优秀 |
| 测试 | 需要 mock | 内置支持 | 内置支持 |
| 学习曲线 | 低 | 中 | 中高 |

## 五、注意事项

1. **MobX 代码生成**：需要运行 `flutter pub run build_runner build`
2. **Riverpod 不可变**：StateNotifier 的 state 应该是不可变的
3. **computed 值**：避免在 computed 中做复杂计算
4. **autoDispose**：使用 autoDispose 避免内存泄漏
5. **依赖注入**：Riverpod 通过 ref 实现依赖注入
