# 数据持久化与状态

## 一、概念说明

将应用状态持久化到本地存储，使应用在重启后能恢复之前的状态。常见的持久化场景包括用户设置、登录信息和离线数据。

```javascript
// Redux 持久化
import { persistStore, persistReducer } from 'redux-persist';
import AsyncStorage from '@react-native-async-storage/async-storage';

const persistConfig = {
  key: 'root',
  storage: AsyncStorage,
  whitelist: ['user', 'settings'], // 只持久化这些 reducer
  blacklist: ['loading'], // 不持久化这些
};

const persistedReducer = persistReducer(persistConfig, rootReducer);

const store = createStore(persistedReducer);
const persistor = persistStore(store);
```

## 二、状态恢复策略

```dart
// Flutter 启动时恢复状态
class AppState extends ChangeNotifier {
  bool _initialized = false;
  User? _user;

  bool get initialized => _initialized;
  User? get user => _user;

  Future<void> initialize() async {
    // 从本地存储恢复
    final token = await SecureStorage.read('token');
    if (token != null) {
      final userData = await ApiService.getUser();
      _user = User.fromJson(userData);
    }
    _initialized = true;
    notifyListeners();
  }

  Future<void> login(String username, String password) async {
    final result = await ApiService.login(username, password);
    await SecureStorage.write('token', result.token);
    _user = result.user;
    notifyListeners();
  }

  Future<void> logout() async {
    await SecureStorage.delete('token');
    _user = null;
    notifyListeners();
  }
}
```

## 三、离线缓存策略

```javascript
// 缓存策略实现
class CacheStrategy {
  // 缓存优先，后台更新
  static async cacheFirst(key, fetcher) {
    const cached = await Storage.get(key);
    if (cached) {
      // 后台更新
      fetcher().then(data => Storage.set(key, data));
      return cached;
    }
    const data = await fetcher();
    await Storage.set(key, data);
    return data;
  }

  // 网络优先，缓存降级
  static async networkFirst(key, fetcher) {
    try {
      const data = await fetcher();
      await Storage.set(key, data);
      return data;
    } catch {
      return await Storage.get(key);
    }
  }

  // 仅缓存
  static async cacheOnly(key) {
    return await Storage.get(key);
  }
}
```

## 四、注意事项

1. **选择性持久化**：不要持久化所有状态，只持久化必要数据
2. **版本迁移**：数据结构变化时需要处理迁移逻辑
3. **敏感数据加密**：密码、Token 使用加密存储
4. **存储容量**：监控本地存储使用量，避免存储过大
5. **同步时机**：状态变化时及时同步到本地存储
