# 状态管理概述

## 一、概念说明

状态管理是移动应用中管理数据变化和界面同步的核心机制。状态可分为局部状态（组件内部）和全局状态（跨组件共享）。

```javascript
// 状态分类
/*
局部状态 (Ephemeral State):
- 当前页面的 UI 状态（展开/收起、选中项）
- 输入框的值、动画状态
- 适合使用 setState / useState

全局状态 (App State):
- 用户登录信息
- 购物车数据
- 主题设置
- 需要使用专门的状态管理方案
*/
```

## 二、状态管理演进

```javascript
// 从简单到复杂的状态管理方案
// 1. 直接变量（适合极简场景）
let count = 0;

// 2. React useState（局部状态）
const [count, setCount] = useState(0);

// 3. Context/Provider（轻量共享）
const ThemeContext = createContext('light');

// 4. Redux（复杂应用）
const store = configureStore({ reducer: rootReducer });

// 5. MobX / Zustand（响应式）
const useStore = create((set) => ({
  count: 0,
  increment: () => set((s) => ({ count: s.count + 1 })),
}));
```

## 三、选择指南

| 场景 | 推荐方案 |
|------|----------|
| 组件内部 UI 状态 | setState / useState |
| 兄弟组件共享 | 提升状态到父组件 |
| 跨层级组件共享 | Context API |
| 中小型全局状态 | Zustand / Provider |
| 大型复杂状态 | Redux Toolkit / Bloc |
| 异步数据流 | React Query / SWR |

## 四、React Native 中的状态管理实践

```javascript
// Redux Toolkit 示例
import { configureStore, createSlice } from '@reduxjs/toolkit';

const cartSlice = createSlice({
  name: 'cart',
  initialState: { items: [], total: 0 },
  reducers: {
    addItem: (state, action) => {
      state.items.push(action.payload);
      state.total += action.payload.price;
    },
    removeItem: (state, action) => {
      const index = state.items.findIndex(i => i.id === action.payload);
      if (index !== -1) {
        state.total -= state.items[index].price;
        state.items.splice(index, 1);
      }
    },
  },
});

// Zustand 示例（更轻量）
import { create } from 'zustand';

const useAuthStore = create((set) => ({
  user: null,
  token: null,
  login: (user, token) => set({ user, token }),
  logout: () => set({ user: null, token: null }),
}));
```

## 六、状态管理中的异步处理

```javascript
// React Query 处理服务端状态
import { useQuery, useMutation } from '@tanstack/react-query';

function UserProfile() {
  const { data, isLoading, error } = useQuery({
    queryKey: ['user', userId],
    queryFn: () => fetchUser(userId),
    staleTime: 5 * 60 * 1000, // 5分钟内认为数据新鲜
  });

  const mutation = useMutation({
    mutationFn: updateUser,
    onSuccess: () => {
      queryClient.invalidateQueries(['user', userId]);
    },
  });

  if (isLoading) return <Loading />;
  if (error) return <Error message={error.message} />;
  return <Profile data={data} onUpdate={mutation.mutate} />;
}
```

## 七、性能优化策略

```javascript
// 1. 状态拆分：将频繁变化的状态与稳定状态分离
const useStableStore = create(() => ({ theme: 'dark', locale: 'zh' }));
const useVolatileStore = create(() => ({ scrollY: 0, isTyping: false }));

// 2. 选择器优化：只订阅需要的字段
const userName = useStore((state) => state.user.name);

// 3. 使用 shallow 比较避免不必要的渲染
import { shallow } from 'zustand/shallow';
const { name, age } = useStore(
  (state) => ({ name: state.user.name, age: state.user.age }),
  shallow
);

// 4. React.memo + useCallback 配合状态更新
const MemoizedChild = React.memo(({ onClick, data }) => {
  return <button onClick={onClick}>{data.name}</button>;
});
```

## 八、注意事项

1. **不过度设计**：简单场景不需要复杂状态管理
2. **单一数据源**：同一数据只在一个地方管理
3. **不可变更新**：创建新对象而非修改原对象
4. **性能优化**：只订阅需要的状态，避免不必要的重渲染
5. **调试工具**：选择有良好调试工具支持的方案
6. **服务端状态与客户端状态分离**：服务端数据用 React Query/SWR，客户端UI状态用 Zustand/Redux
7. **避免状态冗余**：能从已有状态推导出的数据不要重复存储
8. **持久化策略**：重要状态需要持久化到 AsyncStorage/SharedPreferences
