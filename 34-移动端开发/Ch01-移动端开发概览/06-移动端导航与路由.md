# 移动端导航与路由

## 一、概念说明

导航与路由是移动应用中管理页面跳转和页面栈的核心机制。良好的导航设计能让用户在应用中流畅地切换不同功能页面。

```javascript
// React Navigation 基本使用
import { NavigationContainer } from '@react-navigation/native';
import { createNativeStackNavigator } from '@react-navigation/native-stack';

const Stack = createNativeStackNavigator();

const App = () => {
  return (
    <NavigationContainer>
      <Stack.Navigator initialRouteName="Home">
        <Stack.Screen name="Home" component={HomeScreen} />
        <Stack.Screen name="Detail" component={DetailScreen} />
        <Stack.Screen name="Profile" component={ProfileScreen} />
      </Stack.Navigator>
    </NavigationContainer>
  );
};
```

## 二、导航类型

### 2.1 栈导航（Stack Navigation）

```javascript
// 带参数的页面跳转
const HomeScreen = ({ navigation }) => {
  const users = [
    { id: 1, name: '张三' },
    { id: 2, name: '李四' },
  ];

  return (
    <FlatList
      data={users}
      renderItem={({ item }) => (
        <TouchableOpacity
          onPress={() => navigation.navigate('Detail', {
            userId: item.id,
            userName: item.name,
          })}
        >
          <Text>{item.name}</Text>
        </TouchableOpacity>
      )}
    />
  );
};

// 接收参数
const DetailScreen = ({ route, navigation }) => {
  const { userId, userName } = route.params;

  return (
    <View>
      <Text>用户ID: {userId}</Text>
      <Text>用户名: {userName}</Text>
      <Button title="返回" onPress={() => navigation.goBack()} />
    </View>
  );
};
```

### 2.2 底部标签导航

```javascript
import { createBottomTabNavigator } from '@react-navigation/bottom-tabs';

const Tab = createBottomTabNavigator();

const MainTabs = () => (
  <Tab.Navigator
    screenOptions={({ route }) => ({
      tabBarIcon: ({ color, size }) => {
        let iconName;
        if (route.name === '首页') iconName = 'home';
        else if (route.name === '搜索') iconName = 'search';
        else if (route.name === '我的') iconName = 'person';
        return <Icon name={iconName} size={size} color={color} />;
      },
    })}
  >
    <Tab.Screen name="首页" component={HomeScreen} />
    <Tab.Screen name="搜索" component={SearchScreen} />
    <Tab.Screen name="我的" component={ProfileScreen} />
  </Tab.Navigator>
);
```

```dart
// Flutter 路由管理
class AppRouter {
  static Route<dynamic> generateRoute(RouteSettings settings) {
    switch (settings.name) {
      case '/':
        return MaterialPageRoute(builder: (_) => HomeScreen());
      case '/detail':
        final args = settings.arguments as Map<String, dynamic>;
        return MaterialPageRoute(
          builder: (_) => DetailScreen(id: args['id']),
        );
      default:
        return MaterialPageRoute(
          builder: (_) => Scaffold(
            body: Center(child: Text('页面不存在')),
          ),
        );
    }
  }
}

// 使用命名路由跳转
Navigator.pushNamed(context, '/detail', arguments: {'id': 123});
```

## 三、深层链接

```javascript
// React Native 深层链接配置
const linking = {
  prefixes: ['myapp://', 'https://myapp.com'],
  config: {
    screens: {
      Home: '',
      Detail: 'detail/:id',
      Profile: 'profile/:userId',
    },
  },
};

const App = () => (
  <NavigationContainer linking={linking}>
    <Stack.Navigator>
      {/* 导航配置 */}
    </Stack.Navigator>
  </NavigationContainer>
);
```

## 四、注意事项与常见陷阱

1. **页面栈管理**：避免页面栈过深导致内存占用过大，适时清理不需要的页面
2. **参数传递限制**：路由参数应保持简单，复杂对象建议通过状态管理传递
3. **返回行为处理**：Android 物理返回键的行为需要特别处理，避免用户困惑
4. **深层链接安全**：对外部深层链接参数进行验证，防止注入攻击
5. **导航动画一致性**：保持整个应用的页面转场动画风格一致
