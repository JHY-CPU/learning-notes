# 并发安全检查工具

## 一、概念说明

Rust 和生态系统提供了多种工具检测并发问题：Miri（未定义行为）、Thread Sanitizer（数据竞争）、loom（并发正确性）。

```bash
# Miri 检测未定义行为
cargo +nightly miri run

# Thread Sanitizer
RUSTFLAGS="-Z sanitizer=thread" cargo run
```

## 二、具体用法

### 2.1 Miri 使用

```bash
# 安装
rustup +nightly component add miri

# 运行测试
cargo +nightly miri test

# Miri 能检测：
# - 数据竞争
# - 未初始化内存访问
# - 内存泄漏
# - 悬垂指针
```

### 2.2 Clippy 并发 lint

```rust
// Clippy 会警告常见的并发错误
// 警告：在循环中持有锁
fn bad_pattern(data: &std::sync::Mutex<Vec<i32>>) {
    // Clippy 警告：锁的持有时间过长
    let mut guard = data.lock().unwrap();
    for _ in 0..10 {
        guard.push(1);
    }
}
```

### 2.3 测试策略

```rust
#[cfg(test)]
mod tests {
    use std::sync::{Arc, Mutex};
    use std::thread;

    #[test]
    fn test_no_data_race() {
        let data = Arc::new(Mutex::new(0));
        let handles: Vec<_> = (0..10).map(|_| {
            let d = Arc::clone(&data);
            thread::spawn(move || {
                for _ in 0..100 {
                    *d.lock().unwrap() += 1;
                }
            })
        }).collect();

        for h in handles { h.join().unwrap(); }
        assert_eq!(*data.lock().unwrap(), 1000);
    }
}
```

## 三、注意事项与常见陷阱

1. **CI 集成**：在 CI 中运行 sanitizer 测试
2. **性能影响**：sanitizer 会显著降低运行速度
3. **工具组合**：结合使用多种工具效果更好
4. **测试覆盖**：确保并发代码有足够测试覆盖
5. **持续监控**：生产环境也需要监控并发问题
