# 并发调试技巧

## 一、概念说明

并发 bug 难以复现和调试。Rust 的编译时检查防止了数据竞争，但逻辑错误仍需借助工具和技巧排查。

```rust
// 使用 RUST_BACKTRACE=1 运行
// 使用 thread sanitizer: -Z sanitizer=thread
// 使用 miri 检测未定义行为
```

## 二、具体用法

### 2.1 Thread Sanitizer

```bash
# 编译时启用 Thread Sanitizer
RUSTFLAGS="-Z sanitizer=thread" cargo run
# 可以检测数据竞争和死锁
```

### 2.2 日志与 tracing

```rust
use tracing::{info, span, Level};

fn concurrent_task(id: usize) {
    let span = span!(Level::INFO, "task", id);
    let _guard = span.enter();
    info!("开始执行");
    // ... 并发逻辑
    info!("执行完成");
}
```

### 2.3 可复现的测试

```rust
fn deterministic_test(seed: u64) {
    use rand::{Rng, SeedableRng};
    use rand::rngs::StdRng;

    let mut rng = StdRng::seed_from_u64(seed);

```

## 三、注意事项与常见陷阱

1. **日志开销**：并发日志可能打乱执行顺序
2. **Heisenbug**：添加调试代码可能改变行为
3. **时间敏感**：避免依赖特定时间顺序
4. **工具选择**：sanitizer、miri、loom 各有适用场景
5. **最小复现**：尽量缩小复现范围
