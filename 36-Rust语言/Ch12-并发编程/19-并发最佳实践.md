# 并发最佳实践

## 一、概念说明

总结 Rust 并发编程的最佳实践，帮助编写安全、高效、可维护的并发代码。

```rust
// 最佳实践：优先使用消息传递
use std::sync::mpsc;

// 最佳实践：使用 Arc<Mutex<T>> 共享状态
// 最佳实践：尽量减少锁的范围
```

## 二、具体用法

### 2.1 设计原则

```rust
// 1. 优先消息传递，其次共享状态
// 2. 使用类型系统保证安全
// 3. 最小化共享数据
// 4. 使用不可变数据
// 5. 合理选择同步原语
```

### 2.2 代码组织

```rust
// 将并发逻辑封装在模块中
mod concurrent {
    use std::sync::{Arc, Mutex};
    use std::thread;

    pub struct SafeCounter {
        inner: Arc<Mutex<i64>>,
    }

    impl SafeCounter {
        pub fn new() -> Self {
            SafeCounter { inner: Arc::new(Mutex::new(0)) }
        }

        pub fn increment(&self) {
            *self.inner.lock().unwrap() += 1;
        }

        pub fn get(&self) -> i64 {
            *self.inner.lock().unwrap()
        }
    }
}
```

### 2.3 错误处理模式

```rust
fn concurrent_with_error() -> Result<(), Box<dyn std::error::Error>> {
    use std::sync::mpsc;
    use std::thread;

    let (tx, rx) = mpsc::channel();

    thread::spawn(move || -> Result<(), String> {
        tx.send(42).map_err(|e| e.to_string())?;
        Ok(())
    });

    let result = rx.recv().map_err(|e| e.to_string())?;
    Ok(())
}
```

### 2.4 不可变优先原则

```rust
use std::sync::Arc;

fn immutable_first_approach() {
    // 好：使用不可变数据
    let config = Arc::new(AppConfig {
        max_connections: 100,
        timeout_secs: 30,
    });

    // 多个线程安全共享，无需锁
    let handles: Vec<_> = (0..4).map(|_| {
        let config = Arc::clone(&config);
        std::thread::spawn(move || {
            // 只读访问，无竞争
            println!("最大连接数: {}", config.max_connections);
        })
    }).collect();

    for h in handles { h.join().unwrap(); }
}

struct AppConfig {
    max_connections: u32,
    timeout_secs: u32,
}
```

### 2.5 最小共享原则

```rust
use std::sync::mpsc;
use std::thread;

fn minimize_shared_state() {
    let (tx, rx) = mpsc::channel();

    // 每个线程独立计算，只共享最终结果
    for i in 0..4 {
        let tx = tx.clone();
        thread::spawn(move || {
            // 线程本地计算，无需共享中间状态
            let result: i64 = (i * 1000..(i + 1) * 1000)
                .map(|x| x as i64 * x as i64)
                .sum();
            tx.send(result).unwrap();
        });
    }
    drop(tx);

    // 汇总结果
    let total: i64 = rx.iter().sum();
    println!("总计: {}", total);
}
```

### 2.6 超时与取消模式

```rust
use std::sync::mpsc;
use std::thread;
use std::time::Duration;

fn with_timeout<F, T>(f: F, timeout: Duration) -> Option<T>
where
    F: FnOnce() -> T + Send + 'static,
    T: Send + 'static,
{
    let (tx, rx) = mpsc::channel();

    thread::spawn(move || {
        let result = f();
        let _ = tx.send(result);
    });

    rx.recv_timeout(timeout).ok()
}

fn cancellation_example() {
    let result = with_timeout(
        || {
            thread::sleep(Duration::from_secs(5));
            42
        },
        Duration::from_secs(1),
    );

    match result {
        Some(value) => println!("完成: {}", value),
        None => println!("超时，已取消"),
    }
}
```

### 2.7 并发安全的 Builder 模式

```rust
use std::sync::Arc;

struct ServerBuilder {
    port: u16,
    workers: usize,
    timeout: std::time::Duration,
}

impl ServerBuilder {
    fn new() -> Self {
        ServerBuilder {
            port: 8080,
            workers: 4,
            timeout: std::time::Duration::from_secs(30),
        }
    }

    fn port(mut self, port: u16) -> Self {
        self.port = port;
        self
    }

    fn workers(mut self, workers: usize) -> Self {
        self.workers = workers;
        self
    }

    fn build(self) -> Arc<ServerConfig> {
        Arc::new(ServerConfig {
            port: self.port,
            workers: self.workers,
            timeout: self.timeout,
        })
    }
}

struct ServerConfig {
    port: u16,
    workers: usize,
    timeout: std::time::Duration,
}
```

## 五、注意事项与常见陷阱

1. **避免过度并发**：不是所有任务都需要并发，单线程可能更简单高效
2. **测试覆盖**：并发代码需要特别的测试策略，压力测试和形式化验证
3. **性能基线**：先建立性能基线再优化，避免过早优化
4. **文档记录**：记录并发设计决策和约束，便于后续维护
5. **代码审查**：并发代码需要更严格的审查，关注安全性和正确性
6. **优雅关闭**：确保系统可以优雅关闭，不丢失数据
7. **容量规划**：根据业务量规划并发度，避免资源耗尽
