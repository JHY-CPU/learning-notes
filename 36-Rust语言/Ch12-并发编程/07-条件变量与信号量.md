# 条件变量与信号量

## 一、概念说明

`Condvar` 用于线程等待特定条件。`Semaphore`（信号量）控制同时访问资源的线程数量。Rust 标准库提供 Condvar，信号量需第三方 crate。

```rust
use std::sync::{Arc, Mutex, Condvar};

let pair = Arc::new((Mutex::new(false), Condvar::new()));
```

## 二、具体用法

### 2.1 Condvar 基本用法

```rust
use std::sync::{Arc, Mutex, Condvar};
use std::thread;

fn condvar_example() {
    let pair = Arc::new((Mutex::new(false), Condvar::new()));
    let pair_clone = Arc::clone(&pair);

    let waiter = thread::spawn(move || {
        let (lock, cvar) = &*pair_clone;
        let mut started = lock.lock().unwrap();
        while !*started {
            started = cvar.wait(started).unwrap();
        }
        println!("条件满足！");
    });

    thread::sleep(std::time::Duration::from_millis(100));
    let (lock, cvar) = &*pair;
    *lock.lock().unwrap() = true;
    cvar.notify_one();

    waiter.join().unwrap();
}
```

### 2.2 信号量

```rust
// 使用 tokio 或 async-std 的信号量
use std::sync::{Arc, Mutex};
use std::collections::VecDeque;

struct Semaphore {
    count: Mutex<usize>,
    queue: Mutex<VecDeque<()>>,
}

impl Semaphore {
    fn new(count: usize) -> Self {
        Semaphore {
            count: Mutex::new(count),
            queue: Mutex::new(VecDeque::new()),
        }
    }

    fn acquire(&self) {
        loop {
            let mut count = self.count.lock().unwrap();
            if *count > 0 {
                *count -= 1;
                return;
            }
        }
    }

    fn release(&self) {
        *self.count.lock().unwrap() += 1;
    }
}
```

## 三、注意事项与常见陷阱

1. **虚假唤醒**：Condvar 可能被虚假唤醒，需用 while 循环检查
2. **锁的配合**：Condvar 必须与 Mutex 配合使用
3. **信号量释放**：确保信号量最终被释放
4. **死锁风险**：错误的条件等待可能导致死锁
5. **替代方案**：考虑使用 channel 代替条件变量
