## 二、具体用法

### 2.1 生产者消费者模式

```rust
use std::sync::{Arc, Mutex, mpsc};
use std::thread;

fn producer_consumer() {
    let (tx, rx) = mpsc::channel();
    let queue = Arc::new(Mutex::new(Vec::new()));

    // 生产者
    let producer = {
        let tx = tx.clone();
        thread::spawn(move || {
            for i in 0..100 {
                tx.send(i).unwrap();
            }
        })
    };

    // 消费者
    let consumers: Vec<_> = (0..4).map(|_| {
        let rx = rx.clone();
        thread::spawn(move || {
            for msg in rx {
                println!("处理: {}", msg);
            }
        })
    }).collect();

    drop(tx);
    producer.join().unwrap();
    for c in consumers { c.join().unwrap(); }
}
```

### 2.2 并行数据处理

```rust
use rayon::prelude::*;

fn parallel_processing() {
    let data: Vec<i64> = (1..1_000_000).collect();

    let result: i64 = data.par_iter()
        .filter(|x| *x % 2 == 0)
        .map(|x| x * x)
        .sum();

    println!("结果: {}", result);
}
```

## 三、注意事项与常见陷阱

1. **设计先行**：先设计好并发架构再编码
2. **错误处理**：并发场景的错误处理更复杂
3. **监控指标**：记录线程利用率和队列深度
4. **优雅关闭**：设计时考虑优雅关闭机制
5. **压力测试**：充分测试高并发场景
