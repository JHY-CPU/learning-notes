# 并发安全数据结构

## 一、概念说明

Rust 标准库提供了一些并发安全的数据结构，第三方 crate 如 `crossbeam` 提供了更多高效选择。

```rust
use std::sync::mpsc;
use crossbeam::queue::SegQueue;

// 无锁队列
let queue = SegQueue::new();
queue.push(1);
queue.push(2);
```

## 二、具体用法

### 2.1 crossbeam 无锁队列

```rust
// Cargo.toml: crossbeam = "0.8"
use crossbeam::queue::{ArrayQueue, SegQueue};

// 有界队列
let bounded = ArrayQueue::new(100);
bounded.push(1).unwrap();
let val = bounded.pop();

// 无界队列
let unbounded = SegQueue::new();
unbounded.push(1);
unbounded.push(2);
while let Some(val) = unbounded.pop() {
    println!("{}", val);
}
```

### 2.2 crossbeam scope

```rust
use crossbeam::thread;

fn scoped_threads() {
    let data = vec![1, 2, 3, 4, 5];

    thread::scope(|s| {
        s.spawn(|_| {
            println!("线程1: {:?}", data);
        });
        s.spawn(|_| {
            println!("线程2: {:?}", data);
        });
    }).unwrap();
    // data 在此处仍可用
}
```

### 2.3 DashMap 并发哈希表

```rust
// Cargo.toml: dashmap = "5"
use dashmap::DashMap;
use std::sync::Arc;

fn dashmap_example() {
    let map = Arc::new(DashMap::new());
    let map_clone = Arc::clone(&map);

    let handle = std::thread::spawn(move || {
        map_clone.insert("key1", 100);
    });

    map.insert("key2", 200);
    handle.join().unwrap();

    println!("{:?}", map.get("key1"));
}
```

### 2.4 concurrent-queue

```rust
// Cargo.toml: concurrent-queue = "2"
use concurrent_queue::ConcurrentQueue;

fn concurrent_queue_example() {
    let queue = ConcurrentQueue::bounded(100); // 有界队列
    // let queue = ConcurrentQueue::unbounded(); // 无界队列

    queue.push(1).unwrap();
    queue.push(2).unwrap();

    while let Ok(val) = queue.pop() {
        println!("弹出: {}", val);
    }

    println!("队列长度: {}", queue.len());
    println!("是否为空: {}", queue.is_empty());
}
```

### 2.5 atomic 精细操作

```rust
use std::sync::atomic::{AtomicU64, AtomicBool, Ordering};

struct Metrics {
    request_count: AtomicU64,
    error_count: AtomicU64,
    is_healthy: AtomicBool,
}

impl Metrics {
    fn new() -> Self {
        Metrics {
            request_count: AtomicU64::new(0),
            error_count: AtomicU64::new(0),
            is_healthy: AtomicBool::new(true),
        }
    }

    fn record_request(&self) {
        self.request_count.fetch_add(1, Ordering::Relaxed);
    }

    fn record_error(&self) {
        self.error_count.fetch_add(1, Ordering::Relaxed);
        if self.error_count.load(Ordering::Relaxed) > 100 {
            self.is_healthy.store(false, Ordering::Release);
        }
    }

    fn get_stats(&self) -> (u64, u64, bool) {
        (
            self.request_count.load(Ordering::Relaxed),
            self.error_count.load(Ordering::Relaxed),
            self.is_healthy.load(Ordering::Acquire),
        )
    }
}
```

### 2.6 Flume 通道库

```rust
// Cargo.toml: flume = "0.11"
use flume;

fn flume_example() {
    // 有界通道（带背压）
    let (tx, rx) = flume::bounded(10);

    let handle = std::thread::spawn(move || {
        for i in 0..100 {
            tx.send(i).unwrap();
        }
    });

    while let Ok(val) = rx.recv() {
        println!("收到: {}", val);
    }

    handle.join().unwrap();

    // 异步支持
    // tx.send_async(42).await;
    // let val = rx.recv_async().await;
}
```

### 2.7 基准测试对比

```rust
use std::time::Instant;
use std::sync::Mutex;
use std::sync::atomic::{AtomicU64, Ordering};

fn benchmark_concurrent_structures() {
    let n = 1_000_000;

    // Mutex 性能
    let mutex = Mutex::new(0u64);
    let start = Instant::now();
    for _ in 0..n {
        *mutex.lock().unwrap() += 1;
    }
    println!("Mutex: {:?}", start.elapsed());

    // Atomic 性能
    let atomic = AtomicU64::new(0);
    let start = Instant::now();
    for _ in 0..n {
        atomic.fetch_add(1, Ordering::Relaxed);
    }
    println!("Atomic: {:?}", start.elapsed());

    // 通常 Atomic 比 Mutex 快 10-100 倍（单线程）
}
```

## 四、并发数据结构选择指南

| 需求 | 推荐 | 原因 |
|------|------|------|
| 简单计数 | Atomic | 最高效 |
| 共享 HashMap | DashMap | 分片锁，高并发 |
| 有界队列 | crossbeam ArrayQueue | 无锁，高性能 |
| 无界队列 | crossbeam SegQueue 或 flume | 灵活 |
| 作用域线程 | crossbeam scope | 安全借用 |

## 五、注意事项与常见陷阱

1. **无锁 vs 有锁**：无锁数据结构在高竞争下性能更好，但实现更复杂
2. **正确性保证**：无锁结构可能比锁更难正确使用，需要充分测试
3. **内存管理**：无锁结构的内存回收更复杂（ABA 问题等）
4. **适用场景**：简单场景用标准库，高性能需求用 crossbeam
5. **测试挑战**：并发数据结构需要大量压力测试和形式化验证
6. **背压处理**：有界队列满时发送阻塞，需要合理设计容量
7. **序列化开销**：跨线程传递复杂结构时的序列化成本
