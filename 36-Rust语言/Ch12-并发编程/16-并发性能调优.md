# 并发性能调优

## 一、概念说明

并发程序的性能受锁竞争、缓存一致性、线程切换等因素影响。合理调优可以显著提升性能。

```rust
use std::sync::atomic::Ordering;
// 使用原子操作代替锁
// 减少锁的持有时间
// 使用无锁数据结构
```

## 二、具体用法

### 2.1 锁优化

```rust
// 锁分片
struct ShardedMap<T> {
    shards: Vec<std::sync::Mutex<std::collections::HashMap<String, T>>>,
}

impl<T> ShardedMap<T> {
    fn new(shards: usize) -> Self {
        ShardedMap {
            shards: (0..shards)
                .map(|_| std::sync::Mutex::new(std::collections::HashMap::new()))
                .collect(),
        }
    }

    fn get_shard(&self, key: &str) -> &std::sync::Mutex<std::collections::HashMap<String, T>> {
        use std::collections::hash_map::DefaultHasher;
        use std::hash::{Hash, Hasher};
        let mut hasher = DefaultHasher::new();
        key.hash(&mut hasher);
        let index = (hasher.finish() as usize) % self.shards.len();
        &self.shards[index]
    }
}
```

### 2.2 读写分离

```rust
use std::sync::RwLock;

struct Cache {
    data: RwLock<std::collections::HashMap<String, Vec<u8>>>,
}

impl Cache {
    // 读多写少场景
    fn get(&self, key: &str) -> Option<Vec<u8>> {
        self.data.read().unwrap().get(key).cloned()
    }

    fn set(&self, key: String, value: Vec<u8>) {
        self.data.write().unwrap().insert(key, value);
    }
}
```

## 三、注意事项与常见陷阱

1. **先测量**：用 benchmark 数据指导优化
2. **瓶颈分析**：找到真正的瓶颈再优化
3. **可读性**：过度优化可能降低代码可维护性
4. **Amdahl 定律**：串行部分限制并行加速比
5. **硬件因素**：考虑 CPU 缓存、NUMA 等硬件特性
