# 并发模式与架构

## 一、概念说明

Rust 支持多种并发架构模式：消息传递（Actor 模型）、共享状态、数据并行等。选择合适的模式对系统设计至关重要。

```rust
// 消息传递模式（Actor 模型）
use std::sync::mpsc;
use std::thread;

struct Actor {
    receiver: mpsc::Receiver<Message>,
}

enum Message {
    Data(String),
    Stop,
}
```

## 二、具体用法

### 2.1 Actor 模式

```rust
use std::sync::mpsc;
use std::thread;

struct Counter {
    count: i32,
    receiver: mpsc::Receiver<CounterMessage>,
}

enum CounterMessage {
    Increment,
    Get(mpsc::Sender<i32>),
}

impl Counter {
    fn new() -> (Self, mpsc::Sender<CounterMessage>) {
        let (tx, rx) = mpsc::channel();
        (Counter { count: 0, receiver: rx }, tx)
    }

    fn run(mut self) {
        for msg in self.receiver {
            match msg {
                CounterMessage::Increment => self.count += 1,
                CounterMessage::Get(reply) => {
                    reply.send(self.count).unwrap();
                }
            }
        }
    }
}
```

### 2.2 工作窃取模式

```rust
use crossbeam::deque;
use std::thread;

fn work_stealing() {
    let worker = deque::Worker::new_fifo();
    let stealer = worker.stealer();

    worker.push(1);
    worker.push(2);
    worker.push(3);

    let handle = thread::spawn(move || {
        while let Some(task) = stealer.steal() {
            println!("窃取任务: {:?}", task);
        }
    });

    handle.join().unwrap();
}
```

## 三、注意事项与常见陷阱

1. **模式选择**：消息传递优于共享状态（更易推理）
2. **背压处理**：消息过多时需要背压机制
3. **错误传播**：并发任务的错误处理更复杂
4. **优雅关闭**：设计时考虑优雅关闭机制
5. **测试策略**：并发代码需要特别的测试方法
