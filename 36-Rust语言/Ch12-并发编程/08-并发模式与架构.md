# 并发模式与架构

## 一、概念说明

Rust 支持多种并发架构模式：消息传递（Actor 模型）、共享状态、数据并行等。选择合适的模式对系统设计至关重要。

```rust
// 消息传递模式（Actor 模型）
use std::sync::mpsc;
use std::thread;

struct Actor {
    receiver: mpsc::Receiver<Message>,
}

enum Message {
    Data(String),
    Stop,
}
```

## 二、具体用法

### 2.1 Actor 模式

```rust
use std::sync::mpsc;
use std::thread;

struct Counter {
    count: i32,
    receiver: mpsc::Receiver<CounterMessage>,
}

enum CounterMessage {
    Increment,
    Get(mpsc::Sender<i32>),
}

impl Counter {
    fn new() -> (Self, mpsc::Sender<CounterMessage>) {
        let (tx, rx) = mpsc::channel();
        (Counter { count: 0, receiver: rx }, tx)
    }

    fn run(mut self) {
        for msg in self.receiver {
            match msg {
                CounterMessage::Increment => self.count += 1,
                CounterMessage::Get(reply) => {
                    reply.send(self.count).unwrap();
                }
            }
        }
    }
}
```

### 2.2 工作窃取模式

```rust
use crossbeam::deque;
use std::thread;

fn work_stealing() {
    let worker = deque::Worker::new_fifo();
    let stealer = worker.stealer();

    worker.push(1);
    worker.push(2);
    worker.push(3);

    let handle = thread::spawn(move || {
        while let Some(task) = stealer.steal() {
            println!("窃取任务: {:?}", task);
        }
    });

    handle.join().unwrap();
}
```

### 2.3 消息传递模式（Channel）

```rust
use std::sync::mpsc;
use std::thread;

fn message_passing_example() {
    let (tx, rx) = mpsc::channel();

    // 多个发送者
    let tx1 = tx.clone();
    thread::spawn(move || {
        tx1.send("来自线程1").unwrap();
    });

    thread::spawn(move || {
        tx.send("来自线程2").unwrap();
    });

    // 接收所有消息
    for _ in 0..2 {
        let msg = rx.recv().unwrap();
        println!("收到: {}", msg);
    }
}
```

### 2.4 共享状态模式

```rust
use std::sync::{Arc, Mutex};
use std::thread;

fn shared_state_pattern() {
    let data = Arc::new(Mutex::new(SharedState {
        counter: 0,
        messages: Vec::new(),
    }));

    let handles: Vec<_> = (0..4).map(|i| {
        let data = Arc::clone(&data);
        thread::spawn(move || {
            let mut state = data.lock().unwrap();
            state.counter += 1;
            state.messages.push(format!("线程 {} 更新", i));
        })
    }).collect();

    for h in handles { h.join().unwrap(); }

    let state = data.lock().unwrap();
    println!("最终状态: {:?}", *state);
}

#[derive(Debug)]
struct SharedState {
    counter: u64,
    messages: Vec<String>,
}
```

### 2.5 扇出扇入模式

```rust
use std::sync::mpsc;
use std::thread;

fn fan_out_fan_in() {
    let (tx, rx) = mpsc::channel();
    let n_workers = 4;
    let n_tasks = 100;

    // 扇出：分发任务到多个工作线程
    for i in 0..n_workers {
        let tx = tx.clone();
        thread::spawn(move || {
            for task_id in (i..n_tasks).step_by(n_workers) {
                let result = process_task(task_id);
                tx.send((task_id, result)).unwrap();
            }
        });
    }
    drop(tx);

    // 扇入：收集所有结果
    let mut results = Vec::new();
    for (task_id, result) in rx {
        results.push((task_id, result));
    }

    results.sort_by_key(|&(id, _)| id);
    println!("处理了 {} 个任务", results.len());
}

fn process_task(id: usize) -> usize {
    id * id
}
```

### 2.6 Pipeline 流水线模式

```rust
use std::sync::mpsc;
use std::thread;

fn pipeline_pattern() {
    let (stage1_tx, stage1_rx) = mpsc::channel();
    let (stage2_tx, stage2_rx) = mpsc::channel();
    let (stage3_tx, stage3_rx) = mpsc::channel();

    // 阶段1：数据生成
    thread::spawn(move || {
        for i in 0..100 {
            stage1_tx.send(i).unwrap();
        }
    });

    // 阶段2：数据转换
    thread::spawn(move || {
        for val in stage1_rx {
            stage2_tx.send(val * 2).unwrap();
        }
    });

    // 阶段3：数据过滤
    thread::spawn(move || {
        for val in stage2_rx {
            if val % 4 == 0 {
                stage3_tx.send(val).unwrap();
            }
        }
    });

    // 收集最终结果
    let mut results: Vec<i32> = stage3_rx.iter().collect();
    results.sort();
    println!("流水线结果数: {}", results.len());
}
```

### 2.7 Reactor 模式（事件驱动）

```rust
use std::sync::mpsc;
use std::collections::HashMap;

enum Event {
    Connect(u64),
    Data(u64, Vec<u8>),
    Disconnect(u64),
}

struct Reactor {
    handlers: HashMap<u64, Box<dyn Fn(&[u8]) + Send>>,
}

impl Reactor {
    fn new() -> Self {
        Reactor { handlers: HashMap::new() }
    }

    fn register(&mut self, id: u64, handler: impl Fn(&[u8]) + Send + 'static) {
        self.handlers.insert(id, Box::new(handler));
    }

    fn dispatch(&self, event: Event) {
        match event {
            Event::Data(id, data) => {
                if let Some(handler) = self.handlers.get(&id) {
                    handler(&data);
                }
            }
            _ => {}
        }
    }
}
```

## 四、模式选择决策树

```
需要并发？
├── I/O 密集型 → async/await (Tokio)
├── CPU 密集型 → rayon 数据并行
└── 通用并发
    ├── 任务间需要通信？
    │   ├── 是 → 消息传递 (mpsc, flume)
    │   └── 否 → 独立线程
    ├── 需要共享状态？
    │   ├── 读多写少 → RwLock
    │   ├── 写多 → Mutex 或 atomic
    │   └── 复杂状态 → Actor 模式
    └── 需要数据流？
        ├── 单向 → Pipeline
        └── 双向 → Reactor 模式
```

## 五、注意事项与常见陷阱

1. **模式选择**：消息传递优于共享状态（更易推理），但不一定更高效
2. **背压处理**：消息过多时需要背压机制，使用有界通道
3. **错误传播**：并发任务的错误处理更复杂，需要统一的错误收集策略
4. **优雅关闭**：设计时考虑优雅关闭机制，使用信号量或关闭通道
5. **测试策略**：并发代码需要特别的测试方法，考虑使用 loom 验证
6. **资源泄漏**：确保通道发送者和线程句柄被正确清理
7. **死锁预防**：统一锁获取顺序，避免嵌套锁
