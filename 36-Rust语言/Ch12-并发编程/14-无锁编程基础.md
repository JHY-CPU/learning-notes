# 无锁编程基础

## 一、概念说明

无锁编程使用原子操作代替锁实现线程安全。性能更好但正确性更难保证。

```rust
use std::sync::atomic::{AtomicBool, AtomicUsize, Ordering};

// 无锁标志
let ready = AtomicBool::new(false);
// 无锁计数器
let counter = AtomicUsize::new(0);
```

## 二、具体用法

### 2.1 自旋锁实现

```rust
use std::sync::atomic::{AtomicBool, Ordering};
use std::cell::UnsafeCell;

struct SpinLock<T> {
    locked: AtomicBool,
    data: UnsafeCell<T>,
}

impl<T> SpinLock<T> {
    fn new(data: T) -> Self {
        SpinLock {
            locked: AtomicBool::new(false),
            data: UnsafeCell::new(data),
        }
    }

    fn lock(&self) -> &mut T {
        while self.locked.compare_exchange_weak(
            false, true, Ordering::Acquire, Ordering::Relaxed
        ).is_err() {
            std::hint::spin_loop();
        }
        unsafe { &mut *self.data.get() }
    }

    fn unlock(&self) {
        self.locked.store(false, Ordering::Release);
    }
}

unsafe impl<T: Send> Send for SpinLock<T> {}
unsafe impl<T: Send> Sync for SpinLock<T> {}
```

### 2.2 无锁栈

```rust
use std::sync::atomic::{AtomicPtr, Ordering};
use std::ptr;

struct Node<T> {
    data: T,
    next: *mut Node<T>,
}

struct LockFreeStack<T> {
    head: AtomicPtr<Node<T>>,
}

impl<T> LockFreeStack<T> {
    fn new() -> Self {
        LockFreeStack { head: AtomicPtr::new(ptr::null_mut()) }
    }

    fn push(&self, data: T) {
        let node = Box::into_raw(Box::new(Node {
            data,
            next: self.head.load(Ordering::Relaxed),
        }));

        while self.head.compare_exchange_weak(
            node.next, node, Ordering::Release, Ordering::Relaxed
        ).is_err() {
            node.next = self.head.load(Ordering::Relaxed);
        }
    }
}
```

## 三、注意事项与常见陷阱

1. **正确性难保证**：无锁算法比有锁算法更难正确实现
2. **ABA 问题**：使用 tag 或 hazard pointer 解决
3. **内存回收**：无锁数据结构的内存管理更复杂
4. **性能不一定更好**：低竞争场景锁可能更快
5. **适用场景**：高并发、低延迟场景考虑无锁
