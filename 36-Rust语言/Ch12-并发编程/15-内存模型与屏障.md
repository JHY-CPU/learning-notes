# 内存模型与屏障

## 一、概念说明

Rust 的内存模型基于 C++11 的内存模型，提供多种内存序选项控制指令重排和缓存同步。

```rust
use std::sync::atomic::{Ordering, fence, AtomicBool, AtomicI32};

// 内存屏障保证操作顺序
fence(Ordering::Release);
```

## 二、具体用法

### 2.1 内存序详解

```rust
use std::sync::atomic::Ordering;

// Relaxed: 只保证原子性，不保证顺序
// 适用于计数器等简单场景

// Acquire: 读操作的获取语义
// 保证之后的操作不会重排到此之前

// Release: 写操作的释放语义
// 保证之前的操作不会重排到此之后

// AcqRel: 同时具有 Acquire 和 Release 语义

// SeqCst: 完全顺序一致性
// 所有线程看到相同的操作顺序
```

### 2.2 实际应用

```rust
use std::sync::atomic::{AtomicBool, AtomicI32, Ordering, fence};
use std::thread;

fn memory_order_example() {
    let data = AtomicI32::new(0);
    let ready = AtomicBool::new(false);

    // 生产者线程
    let producer = thread::spawn(move || {
        data.store(42, Ordering::Relaxed);
        fence(Ordering::Release); // 保证 data 写入在 ready 之前可见
        ready.store(true, Ordering::Relaxed);
    });

    // 消费者线程
    let consumer = thread::spawn(move || {
        while !ready.load(Ordering::Relaxed) {
            thread::yield_now();
        }
        fence(Ordering::Acquire); // 保证能看到 data 的写入
        data.load(Ordering::Relaxed)
    });

    producer.join().unwrap();
    let result = consumer.join().unwrap();
}
```

## 三、注意事项与常见陷阱

1. **默认选择**：不确定时使用 SeqCst
2. **性能影响**：SeqCst 性能开销最大
3. **正确性优先**：先保证正确，再优化性能
4. **验证困难**：内存序错误难以测试和调试
5. **学习曲线**：内存模型理解需要深入学习
