# 线程池设计

## 一、概念说明

线程池复用线程减少创建销毁开销。Rust 中可使用 rayon 或手写线程池。

```rust
use std::sync::{Arc, Mutex, mpsc};
use std::thread;

struct ThreadPool {
    sender: mpsc::Sender<Job>,
    workers: Vec<Worker>,
}
type Job = Box<dyn FnOnce() + Send + 'static>;
```

## 二、具体用法

### 2.1 rayon 线程池

```rust
use rayon::prelude::*;

fn rayon_pool() {
    let sum: i64 = (0..10_000_000)
        .into_par_iter()
        .sum();
    println!("总和: {}", sum);

    // 自定义线程池
    let pool = rayon::ThreadPoolBuilder::new()
        .num_threads(4)
        .build()
        .unwrap();

    pool.install(|| {
        let result: Vec<i32> = (1..=100)
            .into_par_iter()
            .map(|x| x * x)
            .collect();
    });
}
```

### 2.2 手写线程池

```rust
use std::sync::{Arc, Mutex, mpsc};
use std::thread;

impl ThreadPool {
    fn new(size: usize) -> Self {
        let (tx, rx) = mpsc::channel();
        let rx = Arc::new(Mutex::new(rx));
        let mut workers = Vec::with_capacity(size);

        for id in 0..size {
            workers.push(Worker::new(id, Arc::clone(&rx)));
        }

        ThreadPool { sender: tx, workers }
    }

    fn execute<F: FnOnce() + Send + 'static>(&self, f: F) {
        self.sender.send(Box::new(f)).unwrap();
    }
}

struct Worker {
    thread: Option<thread::JoinHandle<()>>,
}

impl Worker {
    fn new(id: usize, rx: Arc<Mutex<mpsc::Receiver<Job>>>) -> Self {
        let thread = thread::spawn(move || {
            loop {
                let job = rx.lock().unwrap().recv();
                match job {
                    Ok(job) => { println!("Worker {} 执行", id); job(); }
                    Err(_) => break;
                }
            }
        });
        Worker { thread: Some(thread) }
    }
}
```

## 三、注意事项与常见陷阱

1. **优雅关闭**：Drop 时发送终止信号
2. **任务 panic**：捕获任务 panic 避免线程池崩溃
3. **队列满处理**：考虑背压机制
4. **rayon 优先**：生产环境优先使用 rayon
5. **监控指标**：记录任务执行时间和队列深度
