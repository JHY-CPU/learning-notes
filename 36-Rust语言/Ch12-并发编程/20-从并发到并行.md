# 从并发到并行

## 一、概念说明

并发（Concurrency）是同时处理多个任务的能力，并行（Parallelism）是同时执行多个任务。Rust 同时支持这两种模式。

```rust
// 并发：async/await, tokio
// 并行：rayon, std::thread

// 区别：
// 并发关注任务调度
// 并行关注任务执行
```

## 二、具体用法

### 2.1 I/O 密集型（并发）

```rust
use tokio;

#[tokio::main]
async fn io_concurrent() {
    let mut handles = vec![];

    for i in 0..10 {
        handles.push(tokio::spawn(async move {
            // 模拟 I/O 操作
            tokio::time::sleep(std::time::Duration::from_millis(100)).await;
            format!("任务 {} 完成", i)
        }));
    }

    for handle in handles {
        println!("{}", handle.await.unwrap());
    }
}
```

### 2.2 CPU 密集型（并行）

```rust
use rayon::prelude::*;

fn cpu_parallel() {
    let data: Vec<i64> = (0..10_000_000).collect();

    let sum: i64 = data.par_iter()
        .map(|x| {
            // CPU 密集计算
            let mut result = *x;
            for _ in 0..100 {
                result = result.wrapping_mul(17).wrapping_add(31);
            }
            result
        })
        .sum();

    println!("结果: {}", sum);
}
```

### 2.3 混合模式

```rust
async fn mixed_workload() {
    // I/O 部分使用 async
    let data = fetch_data().await;

    // CPU 部分使用 rayon
    let processed: Vec<_> = data.par_iter()
        .map(|x| heavy_computation(x))
        .collect();

    // 结果写回
    save_results(&processed).await;
}
```

## 三、注意事项与常见陷阱

1. **选择正确模式**：I/O 密集用 async，CPU 密集用并行
2. **避免阻塞**：不要在 async 上下文中执行阻塞操作
3. **混合使用**：复杂系统可能同时需要并发和并行
4. **资源限制**：控制并发度避免资源耗尽
5. **性能监控**：持续监控并发/并行的实际效果
