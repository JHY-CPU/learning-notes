# 死锁与避免

## 一、概念说明

死锁是并发编程中常见的问题，指两个或多个线程互相等待对方释放资源。Rust 的类型系统可以防止数据竞争，但不能完全防止死锁。

```rust
// 死锁场景示例
use std::sync::{Arc, Mutex};
use std::thread;

let mutex1 = Arc::new(Mutex::new(0));
let mutex2 = Arc::new(Mutex::new(0));

// 线程1: 先锁mutex1，再锁mutex2
// 线程2: 先锁mutex2，再锁mutex1
// 可能导致死锁
```

## 二、具体用法

### 2.1 锁顺序协议

```rust
// 避免死锁：统一锁的获取顺序
fn safe_lock(m1: &Mutex<i32>, m2: &Mutex<i32>) {
    // 按地址顺序获取锁
    let first = if std::ptr::eq(m1, m2) {
        return;
    } else if (m1 as *const _) < (m2 as *const _) {
        m1.lock().unwrap();
        m2.lock().unwrap()
    } else {
        m2.lock().unwrap();
        m1.lock().unwrap()
    };
}
```

### 2.2 try_lock 避免

```rust
use std::sync::Mutex;

fn try_both(m1: &Mutex<i32>, m2: &Mutex<i32>) -> bool {
    let lock1 = m1.try_lock();
    if lock1.is_err() {
        return false;
    }

    let lock2 = m2.try_lock();
    if lock2.is_err() {
        drop(lock1);
        return false;
    }

    // 成功获取两个锁
    true
}
```

## 三、注意事项与常见陷阱

1. **锁顺序**：所有线程以相同顺序获取锁
2. **超时机制**：使用 try_lock 或带超时的锁
3. **锁粒度**：减少同时持有的锁数量
4. **检测工具**：使用 thread sanitizer 检测死锁
5. **设计原则**：减少锁的使用，优先使用消息传递
