# 错误处理

## 一、概念说明

Rust 将错误分为可恢复错误（Result）和不可恢复错误（panic!）。推荐使用 Result 处理大多数错误情况。

```rust
// 两种错误类型
// 1. panic! - 不可恢复错误，程序终止
// 2. Result<T, E> - 可恢复错误，可以处理

use std::fs::File;
use std::io::{self, Read};

fn main() {
    // panic! 示例
    // panic!("程序崩溃了！");

    // Result 示例
    let result = File::open("hello.txt");
    match result {
        Ok(file) => println!("文件打开成功"),
        Err(error) => println!("打开文件失败: {}", error),
    }
}
```

## 二、Result 类型

```rust
use std::fs::File;
use std::io::{self, Read};

// 返回 Result 的函数
fn read_username_from_file() -> Result<String, io::Error> {
    let mut content = String::new();
    File::open("username.txt")?.read_to_string(&mut content)?;
    Ok(content)
}

// 处理 Result 的多种方式
fn main() {
    // 1. match 模式匹配
    let result = read_username_from_file();
    match result {
        Ok(name) => println!("用户名: {}", name),
        Err(e) => println!("错误: {}", e),
    }

    // 2. if let 简洁匹配
    if let Ok(name) = read_username_from_file() {
        println!("用户名: {}", name);
    }

    // 3. unwrap - 成功返回值，失败 panic
    // let name = read_username_from_file().unwrap();

    // 4. expect - 带消息的 unwrap
    // let name = read_username_from_file().expect("读取用户名失败");

    // 5. unwrap_or - 提供默认值
    let name = read_username_from_file().unwrap_or(String::from("默认用户"));

    // 6. unwrap_or_else - 使用闭包提供默认值
    let name = read_username_from_file().unwrap_or_else(|_| {
        println!("读取失败，使用默认值");
        String::from("默认用户")
    });
}
```

## 三、错误传播

```rust
use std::io;
use std::fs::File;
use std::io::Read;

// 使用 ? 运算符传播错误
fn read_file_to_string(path: &str) -> Result<String, io::Error> {
    let mut file = File::open(path)?;  // 失败则返回 Err
    let mut content = String::new();
    file.read_to_string(&mut content)?;  // 失败则返回 Err
    Ok(content)
}

// 链式调用简化
fn read_file_to_string_v2(path: &str) -> Result<String, io::Error> {
    let mut content = String::new();
    File::open(path)?.read_to_string(&mut content)?;
    Ok(content)
}

// 使用 fs::read_to_string（最简洁）
fn read_file_to_string_v3(path: &str) -> Result<String, io::Error> {
    std::fs::read_to_string(path)
}
```

## 四、自定义错误类型

```rust
use std::fmt;
use std::io;
use std::num::ParseIntError;

#[derive(Debug)]
enum AppError {
    Io(io::Error),
    Parse(ParseIntError),
    Custom(String),
}

impl fmt::Display for AppError {
    fn fmt(&self, f: &mut fmt::Formatter) -> fmt::Result {
        match self {
            AppError::Io(e) => write!(f, "IO 错误: {}", e),
            AppError::Parse(e) => write!(f, "解析错误: {}", e),
            AppError::Custom(msg) => write!(f, "错误: {}", msg),
        }
    }
}

impl From<io::Error> for AppError {
    fn from(error: io::Error) -> Self {
        AppError::Io(error)
    }
}

impl From<ParseIntError> for AppError {
    fn from(error: ParseIntError) -> Self {
        AppError::Parse(error)
    }
}

fn read_and_parse(path: &str) -> Result<i32, AppError> {
    let content = std::fs::read_to_string(path)?;  // 自动转换
    let number = content.trim().parse::<i32>()?;     // 自动转换
    Ok(number)
}
```

## 五、使用 thiserror 和 anyhow

```toml
# Cargo.toml
[dependencies]
thiserror = "1.0"
anyhow = "1.0"
```

```rust
// thiserror - 用于库，定义结构化错误
use thiserror::Error;

#[derive(Error, Debug)]
enum DataError {
    #[error("数据未找到: {id}")]
    NotFound { id: u32 },
    #[error("IO 错误")]
    Io(#[from] std::io::Error),
    #[error("格式错误: {0}")]
    Format(String),
}

// anyhow - 用于应用，简化错误处理
use anyhow::{Result, Context};

fn load_config(path: &str) -> Result<String> {
    let content = std::fs::read_to_string(path)
        .context(format!("无法读取配置文件: {}", path))?;
    Ok(content)
}
```

## 六、注意事项

1. **优先使用 Result**：不要滥用 panic!，只在真正不可恢复时使用
2. **? 运算符**：只能在返回 Result 的函数中使用
3. **错误类型匹配**：? 运算符会自动转换实现了 From trait 的错误类型
4. **unwrap 谨慎使用**：只在确定不会失败时使用 unwrap
5. **错误信息**：提供清晰的错误信息便于调试
