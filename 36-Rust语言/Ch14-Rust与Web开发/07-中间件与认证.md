# 中间件与认证

## 一、概念说明

中间件在请求/响应管道中执行通用逻辑，如认证、日志、CORS 等。

```rust
use axum::middleware;
use tower_http::cors::CorsLayer;

let app = Router::new()
    .route("/protected", get(protected_handler))
    .layer(middleware::from_fn(auth_middleware))
    .layer(CorsLayer::permissive());
```

## 二、具体用法

### 2.1 JWT 认证中间件

```rust
async fn auth_middleware(
    req: axum::extract::Request,
    next: axum::middleware::Next,
) -> Result<axum::response::Response, StatusCode> {
    let token = req.headers()
        .get("Authorization")
        .and_then(|v| v.to_str().ok())
        .ok_or(StatusCode::UNAUTHORIZED)?;

    // 验证 token
    let claims = validate_token(token).map_err(|_| StatusCode::UNAUTHORIZED)?;

    let mut req = req;
    req.extensions_mut().insert(claims);

    Ok(next.run(req).await)
}
```

### 2.2 日志中间件

```rust
use tower_http::trace::TraceLayer;
use tracing::Level;

let app = Router::new()
    .layer(
        TraceLayer::new_for_http()
            .make_span_with(|request: &Request<_>| {
                tracing::span!(
                    Level::INFO,
                    "request",
                    method = %request.method(),
                    uri = %request.uri(),
                )
            })
    );
```

### 2.3 CORS 中间件

```rust
use tower_http::cors::{CorsLayer, Any};
use http::Method;

let cors = CorsLayer::new()
    .allow_origin(Any)
    .allow_methods([Method::GET, Method::POST, Method::PUT, Method::DELETE])
    .allow_headers(Any);

let app = Router::new()
    .route("/", get(index))
    .layer(cors);
```

### 2.4 限流中间件

```rust
use tower::limit::RateLimitLayer;
use std::time::Duration;

let app = Router::new()
    .route("/", get(index))
    .layer(RateLimitLayer::new(100, Duration::from_secs(1)));

// 自定义限流
use governor::{Quota, RateLimiter};

let limiter = RateLimiter::direct(Quota::per_second(
    std::num::NonZeroU32::new(100).unwrap()
));
```

### 2.5 请求 ID 追踪

```rust
use axum::middleware;
use uuid::Uuid;

async fn request_id_middleware(
    mut req: axum::extract::Request,
    next: axum::middleware::Next,
) -> axum::response::Response {
    let request_id = Uuid::new_v4().to_string();
    req.headers_mut().insert(
        "X-Request-ID",
        request_id.parse().unwrap(),
    );

    let mut response = next.run(req).await;
    response.headers_mut().insert(
        "X-Request-ID",
        request_id.parse().unwrap(),
    );
    response
}
```

### 2.6 错误处理中间件

```rust
async fn error_handler(
    req: axum::extract::Request,
    next: axum::middleware::Next,
) -> axum::response::Response {
    let response = next.run(req).await;

    if response.status().is_server_error() {
        tracing::error!("服务器错误: {}", response.status());
    }

    response
}
```

## 四、Tower 中间件生态

| 中间件 | 功能 |
|--------|------|
| TraceLayer | 请求追踪 |
| CorsLayer | CORS 支持 |
| CompressionLayer | 响应压缩 |
| TimeoutLayer | 请求超时 |
| RateLimitLayer | 限流 |
| SetRequestHeader | 设置请求头 |
| SetResponseHeader | 设置响应头 |

## 五、注意事项与常见陷阱

1. **执行顺序**：中间件按添加顺序执行，注意 `.layer()` 调用顺序
2. **错误传播**：中间件中错误会中断请求，需要正确的错误处理
3. **性能影响**：中间件增加请求延迟，仅添加必要的中间件
4. **状态共享**：使用 Extension 或 State 共享认证信息
5. **测试覆盖**：为中间件编写单元测试，特别是认证中间件
6. **安全性**：确保中间件配置正确，避免安全漏洞
7. **调试**：使用 tracing 中间件记录请求和响应
