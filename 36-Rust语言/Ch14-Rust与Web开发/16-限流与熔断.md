# 限流与熔断

## 一、概念说明

限流控制请求速率，熔断在服务不稳定时快速失败，防止级联故障。

```rust
use governor::{Quota, RateLimiter};
use std::num::NonZeroU32;

let limiter = RateLimiter::direct(Quota::per_second(NonZeroU32::new(100).unwrap()));
```

### 2.2 熔断器

```rust
use tokio::sync::Semaphore;
use std::sync::atomic::{AtomicU32, Ordering};

struct CircuitBreaker {
    failures: AtomicU32,
    threshold: u32,
}

impl CircuitBreaker {
    fn call<F, T>(&self, f: F) -> Option<T>
    where
        F: FnOnce() -> T,
    {
        if self.failures.load(Ordering::Relaxed) >= self.threshold {
            return None; // 熔断
        }

        match std::panic::catch_unwind(std::panic::AssertUnwindSafe(f)) {
            Ok(result) => {
                self.failures.store(0, Ordering::Relaxed);
                Some(result)
            }
            Err(_) => {
                self.failures.fetch_add(1, Ordering::Relaxed);
                None
            }
        }
    }
}
```

### 2.3 分布式限流（Redis）

```rust
use redis::AsyncCommands;

struct RedisRateLimiter {
    client: redis::Client,
    max_requests: u32,
    window_secs: u64,
}

impl RedisRateLimiter {
    async fn check(&self, key: &str) -> Result<bool, redis::RedisError> {
        let mut conn = self.client.get_async_connection().await?;
        let current: u32 = conn.incr(key, 1).await?;

        if current == 1 {
            let _: () = conn.expire(key, self.window_secs as i64).await?;
        }

        Ok(current <= self.max_requests)
    }
}
```

## 四、限流算法对比

| 算法 | 优点 | 缺点 |
|------|------|------|
| 固定窗口 | 简单 | 窗口边界突发 |
| 滑动窗口 | 平滑 | 内存开销 |
| 令牌桶 | 允许突发 | 实现复杂 |
| 漏桶 | 平滑输出 | 不允许突发 |

## 五、注意事项与常见陷阱

1. **粒度选择**：全局 vs 用户 vs IP 限流，根据业务需求选择
2. **分布式限流**：使用 Redis 实现分布式限流，保证集群一致性
3. **熔断恢复**：实现熔断器的半开状态，渐进恢复
4. **监控告警**：监控限流和熔断触发，及时发现问题
5. **用户体验**：限流时返回友好的错误信息，包含重试时间
6. **优雅降级**：限流时提供降级服务，而非完全拒绝
7. **测试**：在压力测试中验证限流和熔断效果
