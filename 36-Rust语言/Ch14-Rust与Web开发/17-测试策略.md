# 测试策略

## 一、概念说明

Web 应用需要单元测试、集成测试和端到端测试。

```rust
#[cfg(test)]
mod tests {
    use super::*;

    #[tokio::test]
    async fn test_index() {
        let response = index().await;
        assert_eq!(response.status(), 200);
    }
}
```

## 二、具体用法

### 2.1 集成测试

```rust
use axum::body::Body;
use axum::http::{Request, StatusCode};
use tower::ServiceExt;

#[tokio::test]
async fn test_get_users() {
    let app = Router::new().route("/users", get(list_users));

    let response = app
        .oneshot(Request::builder().uri("/users").body(Body::empty()).unwrap())
        .await
        .unwrap();

    assert_eq!(response.status(), StatusCode::OK);
}
```

### 2.2 Mock 数据库

```rust
use mockall::predicate::*;
use mockall::mock;

mock! {
    Db {}
    #[async_trait]
    trait UserRepository {
        async fn find_by_id(&self, id: u64) -> Option<User>;
    }
}

#[tokio::test]
async fn test_with_mock() {
    let mut mock = MockDb::new();
    mock.expect_find_by_id()
        .returning(|_| Some(User { id: 1, name: "测试".into() }));

    let user = mock.find_by_id(1).await;
    assert!(user.is_some());
}
```

## 三、注意事项与常见陷阱

1. **测试隔离**：每个测试使用独立的数据库
2. **异步测试**：使用 tokio::test 宏
3. **测试覆盖**：覆盖正常和异常路径
4. **测试速度**：避免测试中不必要的等待
5. **CI 集成**：在 CI 中运行所有测试
