# 测试策略

## 一、概念说明

Web 应用需要单元测试、集成测试和端到端测试。

```rust
#[cfg(test)]
mod tests {
    use super::*;

    #[tokio::test]
    async fn test_index() {
        let response = index().await;
        assert_eq!(response.status(), 200);
    }
}
```

## 二、具体用法

### 2.1 集成测试

```rust
use axum::body::Body;
use axum::http::{Request, StatusCode};
use tower::ServiceExt;

#[tokio::test]
async fn test_get_users() {
    let app = Router::new().route("/users", get(list_users));

    let response = app
        .oneshot(Request::builder().uri("/users").body(Body::empty()).unwrap())
        .await
        .unwrap();

    assert_eq!(response.status(), StatusCode::OK);
}
```

### 2.2 Mock 数据库

```rust
use mockall::predicate::*;
use mockall::mock;

mock! {
    Db {}
    #[async_trait]
    trait UserRepository {
        async fn find_by_id(&self, id: u64) -> Option<User>;
    }
}

#[tokio::test]
async fn test_with_mock() {
    let mut mock = MockDb::new();
    mock.expect_find_by_id()
        .returning(|_| Some(User { id: 1, name: "测试".into() }));

    let user = mock.find_by_id(1).await;
    assert!(user.is_some());
}
```

### 2.3 测试数据库

```rust
#[cfg(test)]
mod db_tests {
    use sqlx::PgPool;

    async fn setup_test_db() -> PgPool {
        let pool = PgPool::connect("postgres://localhost/test_db").await.unwrap();
        sqlx::migrate!("./migrations").run(&pool).await.unwrap();
        pool
    }

    #[tokio::test]
    async fn test_create_user() {
        let pool = setup_test_db().await;

        let user = create_user(&pool, "测试用户").await.unwrap();
        assert_eq!(user.name, "测试用户");

        // 清理
        sqlx::query!("DELETE FROM users WHERE id = $1", user.id)
            .execute(&pool)
            .await
            .unwrap();
    }
}
```

### 2.4 Snapshot 测试

```rust
// 使用 insta 进行 snapshot 测试
#[tokio::test]
async fn test_api_response_snapshot() {
    let app = Router::new().route("/users", get(list_users));
    let response = app
        .oneshot(Request::builder().uri("/users").body(Body::empty()).unwrap())
        .await
        .unwrap();

    let body = hyper::body::to_bytes(response.into_body()).await.unwrap();
    insta::assert_json_snapshot!(serde_json::from_slice::<serde_json::Value>(&body).unwrap());
}
```

## 四、测试金字塔

```
端到端测试（少量）
├── 完整的 API 流程
├── 用户场景
└── 慢、脆弱

集成测试（适量）
├── API 端点测试
├── 数据库交互
└── 中等速度

单元测试（大量）
├── 业务逻辑
├── 数据验证
└── 快速、稳定
```

## 五、注意事项与常见陷阱

1. **测试隔离**：每个测试使用独立的数据库或事务回滚
2. **异步测试**：使用 `tokio::test` 宏测试异步代码
3. **测试覆盖**：覆盖正常和异常路径，包括错误情况
4. **测试速度**：避免测试中不必要的等待，使用 mock
5. **CI 集成**：在 CI 中运行所有测试，包括集成测试
6. **测试数据**：使用 factory 模式创建测试数据
7. **并行安全**：确保测试可以并行运行
