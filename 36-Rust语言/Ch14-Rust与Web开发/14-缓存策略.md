# 缓存策略

## 一、概念说明

Web 应用使用缓存提升性能，包括内存缓存、Redis、HTTP 缓存等。

```rust
use moka::future::Cache;

let cache: Cache<String, String> = Cache::builder()
    .max_capacity(1000)
    .time_to_live(Duration::from_secs(300))
    .build();
```

## 二、具体用法

### 2.1 内存缓存

```rust
use moka::future::Cache;

async fn get_user(id: u64) -> User {
    let cache: Cache<u64, User> = Cache::new(100);

    if let Some(user) = cache.get(&id) {
        return user;
    }

    let user = fetch_user_from_db(id).await;
    cache.insert(id, user.clone()).await;
    user
}
```

### 2.2 Redis 缓存

```rust
use redis::AsyncCommands;

async fn get_cached(key: &str) -> Option<String> {
    let client = redis::Client::open("redis://localhost").unwrap();
    let mut conn = client.get_async_connection().await.unwrap();
    conn.get(key).await.ok()
}

async fn set_cached(key: &str, value: &str, ttl: usize) {
    let client = redis::Client::open("redis://localhost").unwrap();
    let mut conn = client.get_async_connection().await.unwrap();
    let _: () = conn.set_ex(key, value, ttl).await.unwrap();
}
```

### 2.3 HTTP 缓存头

```rust
use axum::http::{HeaderMap, HeaderValue, header};
use std::time::Duration;

fn cache_headers(max_age: u64) -> HeaderMap {
    let mut headers = HeaderMap::new();
    headers.insert(
        header::CACHE_CONTROL,
        HeaderValue::from_str(&format!("public, max-age={}", max_age)).unwrap(),
    );
    headers.insert(
        header::EXPIRES,
        HeaderValue::from_str(&format!("{}", chrono::Utc::now() + chrono::Duration::seconds(max_age as i64))).unwrap(),
    );
    headers
}
```

### 2.4 缓存策略模式

```rust
// Cache-Aside（旁路缓存）
async fn cache_aside(key: &str, cache: &Cache, db: &Db) -> String {
    if let Some(value) = cache.get(key).await {
        return value;
    }
    let value = db.get(key).await;
    cache.insert(key.to_string(), value.clone(), 300).await;
    value
}

// Write-Through（写穿透）
async fn write_through(key: &str, value: &str, cache: &Cache, db: &Db) {
    db.set(key, value).await;
    cache.insert(key.to_string(), value.to_string(), 300).await;
}

// Write-Behind（写回）
async fn write_behind(key: &str, value: &str, cache: &Cache, queue: &mpsc::Sender) {
    cache.insert(key.to_string(), value.to_string(), 300).await;
    queue.send((key.to_string(), value.to_string())).await;
}
```

## 五、注意事项与常见陷阱

1. **缓存失效**：实现合理的缓存失效策略，使用 TTL 和 LRU
2. **缓存穿透**：防止恶意查询导致缓存失效，使用布隆过滤器
3. **一致性**：保证缓存与数据库的一致性，考虑最终一致性
4. **序列化**：选择高效的序列化方式，如 bincode 或 msgpack
5. **监控**：监控缓存命中率和内存使用，及时调整策略
6. **缓存雪崩**：避免缓存同时失效，使用随机 TTL
7. **热点数据**：识别和优化热点数据的缓存策略
