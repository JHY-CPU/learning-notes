# 错误处理最佳实践

## 一、概念说明

Web 应用的错误处理需要统一的错误响应格式和清晰的错误传播。

```rust
use thiserror::Error;
use axum::response::IntoResponse;

#[derive(Error, Debug)]
enum AppError {
    #[error("用户不存在")]
    NotFound,
    #[error("数据库错误: {0}")]
    Database(#[from] sqlx::Error),
    #[error("认证失败")]
    Unauthorized,
}

impl IntoResponse for AppError {
    fn into_response(self) -> axum::response::Response {
        let (status, message) = match self {
            AppError::NotFound => (StatusCode::NOT_FOUND, self.to_string()),
            AppError::Database(_) => (StatusCode::INTERNAL_SERVER_ERROR, "服务器内部错误".to_string()),
            AppError::Unauthorized => (StatusCode::UNAUTHORIZED, self.to_string()),
        };
        (status, Json(serde_json::json!({"error": message}))).into_response()
    }
}
```

## 二、具体用法

### 2.1 统一响应格式

```rust
use serde::Serialize;

#[derive(Serialize)]
struct ApiResponse<T: Serialize> {
    success: bool,
    data: Option<T>,
    error: Option<String>,
}

impl<T: Serialize> ApiResponse<T> {
    fn success(data: T) -> Self {
        ApiResponse { success: true, data: Some(data), error: None }
    }

    fn error(msg: &str) -> Self {
        ApiResponse { success: false, data: None, error: Some(msg.to_string()) }
    }
}
```

### 2.2 业务错误处理

```rust
async fn create_user(
    Json(input): Json<CreateUserInput>,
) -> Result<Json<User>, AppError> {
    if input.name.is_empty() {
        return Err(AppError::Validation("名字不能为空".to_string()));
    }

    let user = db::create_user(&input).await?;
    Ok(Json(user))
}
```

## 三、注意事项与常见陷阱

1. **信息泄露**：生产环境不要暴露内部错误细节
2. **错误日志**：记录详细错误信息便于调试
3. **错误码**：使用统一的错误码体系
4. **国际化**：考虑错误消息的国际化
5. **监控集成**：将错误上报到监控系统
