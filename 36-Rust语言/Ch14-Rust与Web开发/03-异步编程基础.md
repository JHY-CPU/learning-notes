# 异步编程基础

## 一、概念说明

Rust 的异步编程基于 Future trait 和 async/await 语法。与传统线程不同，异步编程使用协作式调度。

```rust
async fn fetch_data() -> String {
    // 异步操作
    "数据".to_string()
}

#[tokio::main]
async fn main() {
    let data = fetch_data().await;
}
```

## 二、具体用法

### 2.1 async/await 基础

```rust
async fn fetch_url(url: &str) -> Result<String, reqwest::Error> {
    let response = reqwest::get(url).await?;
    let body = response.text().await?;
    Ok(body)
}

// 并发执行多个异步操作
async fn fetch_all() {
    let (a, b) = tokio::join!(
        fetch_url("https://example.com/a"),
        fetch_url("https://example.com/b")
    );
}
```

### 2.2 Future 组合

```rust
use futures::future::{join_all, select_all};

async fn process_multiple() {
    let futures = vec![
        fetch_url("url1"),
        fetch_url("url2"),
        fetch_url("url3"),
    ];

    let results = join_all(futures).await;
}
```

## 三、注意事项与常见陷阱

1. **运行时选择**：tokio、async-std 等
2. **避免阻塞**：不要在异步上下文中执行阻塞操作
3. **生命周期**：异步函数的生命周期更复杂
4. **Pin 问题**：自引用结构需要 Pin
5. **调试困难**：异步代码的堆栈跟踪不完整
