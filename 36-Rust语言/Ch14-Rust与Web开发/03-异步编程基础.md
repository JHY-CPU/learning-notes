# 异步编程基础

## 一、概念说明

Rust 的异步编程基于 Future trait 和 async/await 语法。与传统线程不同，异步编程使用协作式调度。

```rust
async fn fetch_data() -> String {
    // 异步操作
    "数据".to_string()
}

#[tokio::main]
async fn main() {
    let data = fetch_data().await;
}
```

## 二、具体用法

### 2.1 async/await 基础

```rust
async fn fetch_url(url: &str) -> Result<String, reqwest::Error> {
    let response = reqwest::get(url).await?;
    let body = response.text().await?;
    Ok(body)
}

// 并发执行多个异步操作
async fn fetch_all() {
    let (a, b) = tokio::join!(
        fetch_url("https://example.com/a"),
        fetch_url("https://example.com/b")
    );
}
```

### 2.2 Future 组合

```rust
use futures::future::{join_all, select_all};

async fn process_multiple() {
    let futures = vec![
        fetch_url("url1"),
        fetch_url("url2"),
        fetch_url("url3"),
    ];

    let results = join_all(futures).await;
}
```

### 2.3 错误处理与异步

```rust
use anyhow::{Result, Context};

async fn fetch_and_process() -> Result<String> {
    let response = reqwest::get("https://api.example.com/data")
        .await
        .context("请求失败")?;

    let body = response.text()
        .await
        .context("读取响应失败")?;

    let parsed: serde_json::Value = serde_json::from_str(&body)
        .context("JSON 解析失败")?;

    Ok(parsed["result"].to_string())
}
```

### 2.4 异步流处理

```rust
use futures::stream::{self, StreamExt};

async fn process_stream() {
    let mut stream = stream::iter(vec![1, 2, 3, 4, 5])
        .map(|x| async move {
            tokio::time::sleep(std::time::Duration::from_millis(100)).await;
            x * x
        })
        .buffered(3); // 最多同时处理3个

    while let Some(result) = stream.next().await {
        println!("结果: {}", result);
    }
}
```

### 2.5 异步闭包与所有权

```rust
async fn ownership_example() {
    let data = vec![1, 2, 3, 4, 5];

    // move 将所有权转移到 async 块中
    let handle = tokio::spawn(async move {
        // data 在此拥有所有权
        data.iter().sum::<i32>()
    });

    // data 已被 move，不能再使用
    let result = handle.await.unwrap();
    println!("总和: {}", result);
}
```

### 2.6 Pin 与自引用

```rust
use std::pin::Pin;
use std::future::Future;

// Pin 确保 Future 不会被移动
// 自引用结构（如 async fn 内部）需要 Pin

fn return_pinned_future() -> Pin<Box<dyn Future<Output = i32> + Send>> {
    Box::pin(async {
        tokio::time::sleep(std::time::Duration::from_millis(100)).await;
        42
    })
}

// 使用 pin! 宏避免堆分配
async fn pin_example() {
    let future = async { 42 };
    tokio::pin!(future);
    // future 现在是 Pin<&mut impl Future>
    let result = future.await;
}
```

## 四、Future 状态机原理

```rust
// async fn 实际被编译为状态机
async fn example() -> i32 {
    let x = step1().await; // 状态1
    let y = step2(x).await; // 状态2
    x + y // 状态3（完成）
}

// 编译器大致生成：
enum ExampleState {
    Start,
    WaitingStep1(Step1Future),
    WaitingStep2(i32, Step2Future),
    Done,
}

impl Future for ExampleState {
    type Output = i32;
    fn poll(self: Pin<&mut Self>, cx: &mut Context) -> Poll<i32> {
        // 状态机轮询逻辑
    }
}
```

## 五、注意事项与常见陷阱

1. **运行时选择**：tokio 是最流行的运行时，async-std 是替代方案
2. **避免阻塞**：不要在异步上下文中执行阻塞操作，使用 `spawn_blocking`
3. **生命周期**：异步函数的生命周期更复杂，跨 `await` 的引用需要 `'static`
4. **Pin 问题**：自引用结构需要 Pin，使用 `Box::pin` 或 `tokio::pin!`
5. **调试困难**：异步代码的堆栈跟踪不完整，使用 `tokio-console` 辅助
6. **取消安全**：Drop Future 会取消任务，确保取消是安全的
7. **背压处理**：使用信号量或有界通道控制并发度
