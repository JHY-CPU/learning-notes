### 2.3 优雅关闭

```rust
async fn shutdown_signal() {
    let ctrl_c = async {
        tokio::signal::ctrl_c()
            .await
            .expect("安装 Ctrl+C 处理器失败");
    };

    #[cfg(unix)]
    let terminate = async {
        tokio::signal::unix::signal(tokio::signal::unix::SignalKind::terminate())
            .expect("安装信号处理器失败")
            .recv()
            .await;
    };

    tokio::select! {
        _ = ctrl_c => {},
        _ = terminate => {},
    }

    println!("收到关闭信号，开始优雅关闭...");
}
```

### 2.4 性能监控

use metrics::{counter, gauge, histogram};
use std::time::Instant;

async fn monitored_handler() -> Response {
    let start = Instant::now();

    counter!("http_requests_total", 1);
    gauge!("active_connections", get_active_connections());

    let response = handle_request().await;

    histogram!("http_request_duration_seconds", start.elapsed().as_secs_f64());

    response
}

### 2.5 配置管理

```rust
use config::{Config, File, Environment};

#[derive(Debug, Deserialize)]
struct Settings {
    database_url: String,
    port: u16,
    log_level: String,
}

fn load_config() -> Settings {
    Config::builder()
        .set_default("port", 8080).unwrap()
        .set_default("log_level", "info").unwrap()
        .add_source(File::with_name("config/default"))
        .add_source(File::with_name("config/production").required(false))
        .add_source(Environment::with_prefix("APP"))
        .build()
        .unwrap()
        .try_deserialize()
        .unwrap()
}
```

## 五、注意事项与常见陷阱

1. **配置管理**：使用环境变量或配置文件，避免硬编码
2. **安全更新**：定期更新依赖，使用 `cargo audit` 检查漏洞
3. **备份策略**：数据库和重要数据备份，定期测试恢复
4. **灾难恢复**：制定灾难恢复计划，演练恢复流程
5. **容量规划**：根据业务增长规划容量，提前扩容
6. **安全头**：设置安全 HTTP 头，防止常见攻击
7. **依赖最小化**：仅包含必要的依赖，减少攻击面
