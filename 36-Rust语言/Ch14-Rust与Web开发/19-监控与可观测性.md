# 监控与可观测性

## 一、概念说明

Web 应用需要指标（metrics）、日志（logs）和追踪（traces）三大可观测性支柱。

```rust
use tracing::{info, instrument};
use metrics::{counter, gauge};

#[instrument]
async fn handle_request() {
    counter!("requests_total", 1);
    info!("处理请求");
}
```

## 二、具体用法

### 2.1 Metrics 集成

```rust
use metrics_exporter_prometheus::PrometheusBuilder;

#[tokio::main]
async fn main() {
    PrometheusBuilder::new()
        .install()
        .expect("安装 Prometheus 失败");

    // 在中间件中记录指标
    let app = Router::new()
        .route("/", get(index))
        .layer(middleware::from_fn(record_metrics));
}

async fn record_metrics(
    req: axum::extract::Request,
    next: axum::middleware::Next,
) -> axum::response::Response {
    let start = std::time::Instant::now();
    let response = next.run(req).await;
    let duration = start.elapsed();

    counter!("http_requests_total", 1);
    gauge!("http_request_duration_seconds", duration.as_secs_f64());

    response
}
```

### 2.2 Tracing 集成

```rust
use tracing_subscriber::fmt;
use opentelemetry::sdk::trace as sdktrace;

fn init_tracing() {
    let tracer = opentelemetry_jaeger::new_pipeline()
        .install_simple()
        .unwrap();

    let telemetry = tracing_opentelemetry::layer().with_tracer(tracer);

    tracing_subscriber::registry()
        .with(telemetry)
        .with(fmt::layer())
        .init();
}
```

### 2.3 日志聚合

```rust
use tracing_subscriber::{fmt, EnvFilter, prelude::*};

fn init_observability() {
    tracing_subscriber::registry()
        .with(fmt::layer())
        .with(EnvFilter::from_default_env())
        .init();
}

// 结构化日志
#[tracing::instrument(skip(pool))]
async fn process_order(pool: &PgPool, order_id: i64) -> Result<(), Error> {
    tracing::info!(order_id, "开始处理订单");

    let order = fetch_order(pool, order_id).await?;
    tracing::debug!(?order, "获取订单详情");

    process_payment(&order).await?;
    tracing::info!(order_id, "订单处理完成");

    Ok(())
}
```

### 2.4 告警规则示例

```yaml
# prometheus alerting rules
groups:
  - name: web_app
    rules:
      - alert: HighErrorRate
        expr: rate(http_requests_total{status=~"5.."}[5m]) > 0.1
        for: 5m
        labels:
          severity: critical
        annotations:
          summary: "高错误率检测到"

      - alert: HighLatency
        expr: histogram_quantile(0.95, rate(http_request_duration_seconds_bucket[5m])) > 1
        for: 5m
        labels:
          severity: warning
```

## 四、可观测性三支柱

| 支柱 | 工具 | 用途 |
|------|------|------|
| 指标 (Metrics) | Prometheus, metrics-rs | 量化系统状态 |
| 日志 (Logs) | tracing, ELK | 事件记录 |
| 追踪 (Traces) | Jaeger, Zipkin | 请求链路追踪 |

## 五、注意事项与常见陷阱

1. **性能影响**：监控有性能开销，使用采样和采样率控制
2. **采样策略**：配置追踪采样率，生产环境通常 1-10%
3. **敏感信息**：不要在日志中记录密码、密钥等敏感信息
4. **告警配置**：为关键指标设置告警，避免告警疲劳
5. **存储管理**：管理日志和指标的存储，设置保留策略
6. **关联性**：使用 trace ID 关联日志和追踪
7. **成本控制**：监控存储和查询成本，优化数据保留
