# 数据库操作

## 一、概念说明

Rust 有多种数据库访问方案：SQLx（异步）、Diesel（同步ORM）、SeaORM 等。

```rust
use sqlx::PgPool;

async fn get_users(pool: &PgPool) -> Result<Vec<User>, sqlx::Error> {
    sqlx::query_as!(User, "SELECT * FROM users")
        .fetch_all(pool)
        .await
}
```

## 二、具体用法

### 2.1 SQLx 基础

```rust
use sqlx::{PgPool, FromRow};

#[derive(FromRow)]
struct User {
    id: i32,
    name: String,
}

async fn create_user(pool: &PgPool, name: &str) -> Result<User, sqlx::Error> {
    sqlx::query_as!(
        User,
        "INSERT INTO users (name) VALUES ($1) RETURNING *",
        name
    )
    .fetch_one(pool)
    .await
}
```

### 2.2 连接池

```rust
use sqlx::postgres::PgPoolOptions;

async fn create_pool() -> PgPool {
    PgPoolOptions::new()
        .max_connections(5)
        .connect("postgres://user:pass@localhost/db")
        .await
        .unwrap()
}
```

### 2.3 事务处理

```rust
async fn transfer(pool: &PgPool, from: i32, to: i32, amount: i32) -> Result<(), sqlx::Error> {
    let mut tx = pool.begin().await?;

    sqlx::query!("UPDATE accounts SET balance = balance - $1 WHERE id = $2", amount, from)
        .execute(&mut *tx).await?;

    sqlx::query!("UPDATE accounts SET balance = balance + $1 WHERE id = $2", amount, to)
        .execute(&mut *tx).await?;

    tx.commit().await?;
    Ok(())
}
```

### 2.4 SeaORM 示例

```rust
// SeaORM: 异步 ORM
use sea_orm::{Database, EntityTrait, Set, ActiveModelTrait};

#[derive(Clone, Debug, DeriveEntityModel)]
#[sea_orm(table_name = "users")]
pub struct Model {
    #[sea_orm(primary_key)]
    pub id: i32,
    pub name: String,
    pub email: String,
}

async fn seaorm_example() -> Result<(), sea_orm::DbErr> {
    let db = Database::connect("postgres://...").await?;

    // 插入
    let new_user = ActiveModel {
        name: Set("张三".to_string()),
        email: Set("zhangsan@example.com".to_string()),
        ..Default::default()
    };
    let result = new_user.insert(&db).await?;

    // 查询
    let users = Entity::find().all(&db).await?;

    Ok(())
}
```

### 2.5 数据库迁移

```bash
# sqlx-cli 迁移管理
cargo install sqlx-cli

# 创建迁移
sqlx migrate add create_users_table

# 运行迁移
sqlx migrate run

# 回滚迁移
sqlx migrate revert

# 迁移文件示例
-- migrations/20240101_create_users.sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    email VARCHAR(255) UNIQUE NOT NULL,
    created_at TIMESTAMP DEFAULT NOW()
);
```

## 四、数据库库对比

| 库 | 类型 | 异步 | 特点 |
|------|------|------|------|
| SQLx | 查询构建器 | 是 | 编译时 SQL 检查 |
| Diesel | ORM | 否 | 类型安全、宏驱动 |
| SeaORM | ORM | 是 | 异步、基于 Diesel 概念 |
| rusqlite | SQLite | 否 | 嵌入式数据库 |

## 五、注意事项与常见陷阱

1. **连接池大小**：根据负载调整，避免连接耗尽
2. **超时设置**：配置查询超时，避免长时间等待
3. **错误处理**：区分连接错误和查询错误，实现重试策略
4. **SQL注入**：使用参数化查询，永远不要拼接 SQL
5. **迁移管理**：使用 sqlx-cli 管理数据库迁移，版本控制
6. **索引优化**：为常用查询字段创建索引
7. **N+1 问题**：使用 join 或批量查询避免 N+1 问题
