# 文件上传下载

## 一、概念说明

Rust Web 框架支持文件上传下载，处理 multipart 表单和流式响应。

```rust
use axum::{extract::Multipart, response::IntoResponse};

async fn upload(mut multipart: Multipart) -> impl IntoResponse {
    while let Some(field) = multipart.next_field().await.unwrap() {
        let name = field.name().unwrap().to_string();
        let data = field.bytes().await.unwrap();
        // 保存文件
    }
    "上传成功"
}
```

## 二、具体用法

### 2.1 大文件流式上传

```rust
use axum::body::StreamBody;
use tokio::fs::File;
use tokio::io::BufReader;

async fn download() -> impl IntoResponse {
    let file = File::open("file.zip").await.unwrap();
    let stream = tokio_util::io::ReaderStream::new(file);
    let body = StreamBody::new(stream);

    (
        [("Content-Disposition", "attachment; filename=\"file.zip\"")],
        body,
    )
}
```

### 2.2 进度跟踪

```rust
use futures::StreamExt;

async fn upload_with_progress(mut multipart: Multipart) -> impl IntoResponse {
    let mut total_bytes = 0;

    while let Some(field) = multipart.next_field().await.unwrap() {
        let mut stream = field.map(|chunk| {
            let chunk = chunk.unwrap();
            total_bytes += chunk.len();
            Ok::<_, std::io::Error>(chunk)
        });

        // 处理流
    }

    format!("上传完成，共 {} 字节", total_bytes)
}
```

### 2.3 文件类型验证

```rust
use axum::extract::Multipart;
use std::path::Path;

fn validate_file_type(filename: &str, allowed: &[&str]) -> Result<(), String> {
    let ext = Path::new(filename)
        .extension()
        .and_then(|e| e.to_str())
        .ok_or("无法获取文件扩展名")?;

    if allowed.contains(&ext.to_lowercase().as_str()) {
        Ok(())
    } else {
        Err(format!("不允许的文件类型: {}", ext))
    }
}

async fn safe_upload(mut multipart: Multipart) -> Result<String, String> {
    while let Some(field) = multipart.next_field().await.map_err(|e| e.to_string())? {
        let filename = field.file_name()
            .ok_or("缺少文件名")?
            .to_string();

        // 验证文件类型
        validate_file_type(&filename, &["jpg", "png", "pdf", "txt"])?;

        let data = field.bytes().await.map_err(|e| e.to_string())?;

        // 验证文件大小
        if data.len() > 10 * 1024 * 1024 {
            return Err("文件过大，最大 10MB".to_string());
        }

        // 安全的文件名
        let safe_filename = sanitize_filename::sanitize(&filename);
        tokio::fs::write(format!("./uploads/{}", safe_filename), &data).await
            .map_err(|e| e.to_string())?;
    }
    Ok("上传成功".to_string())
}
```

## 四、注意事项与常见陷阱

1. **大小限制**：配置上传文件大小限制，防止拒绝服务攻击
2. **文件类型**：验证上传文件类型，不仅检查扩展名，还要检查魔数
3. **存储位置**：安全的文件存储路径，避免路径遍历攻击
4. **并发上传**：控制并发上传数量，使用信号量限制
5. **断点续传**：大文件考虑断点续传，使用 Content-Range 头
6. **病毒扫描**：对上传文件进行病毒扫描
7. **CDN 集成**：使用 CDN 加速文件下载
