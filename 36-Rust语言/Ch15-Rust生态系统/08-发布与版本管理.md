# 发布与版本管理

## 一、概念说明

Rust 使用 crates.io 发布 crate，遵循语义化版本（SemVer）。

```bash
# 发布到 crates.io
cargo publish

# 登录
cargo login YOUR_API_TOKEN

# 检查包
cargo package
cargo package --list
```

## 二、具体用法

### 2.1 版本管理

```toml
# Cargo.toml
[package]
name = "my_crate"
version = "1.2.3"
```

```rust
// 版本号规则
// MAJOR.MINOR.PATCH
// MAJOR: 不兼容的 API 变更
// MINOR: 向后兼容的功能新增
// PATCH: 向后兼容的问题修复
```

### 2.2 发布流程

```bash
# 1. 更新版本号
cargo set-version 1.2.4

# 2. 更新 CHANGELOG
# 3. 提交并打标签
git add -A
git commit -m "release: v1.2.4"
git tag v1.2.4
git push && git push --tags

# 4. 发布
cargo publish
```

### 2.3 特性管理

```toml
[features]
default = ["std"]
std = []
full = ["std", "async", "serde"]

[dependencies]
serde = { version = "1.0", optional = true }
```

## 三、注意事项与常见陷阱

1. **API 兼容性**：谨慎处理不兼容变更
2. **文档更新**：发布前更新文档
3. **测试覆盖**：发布前运行所有测试
4. **许可证**：确保许可证信息正确
5. **撤回发布**：24小时内可撤回版本
