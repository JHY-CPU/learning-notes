# 基准测试与性能

## 一、概念说明

Criterion 是 Rust 最流行的基准测试框架，提供统计分析和性能比较。

```rust
use criterion::{criterion_group, criterion_main, Criterion};

fn benchmark(c: &mut Criterion) {
    c.bench_function("fib 20", |b| {
        b.iter(|| fib(20))
    });
}

criterion_group!(benches, benchmark);
criterion_main!(benches);
```

## 二、具体用法

### 2.1 Criterion 基础

```rust
use criterion::{black_box, criterion_group, criterion_main, Criterion, BenchmarkId};

fn bench_fibonacci(c: &mut Criterion) {
    let mut group = c.benchmark_group("Fibonacci");

    for i in [10, 20, 30].iter() {
        group.bench_with_input(BenchmarkId::new("fib", i), i, |b, &i| {
            b.iter(|| fib(black_box(i)))
        });
    }
    group.finish();
}
```

### 2.2 比较基准

```rust
use criterion::Criterion;

fn compare_implementations(c: &mut Criterion) {
    let data = vec![1, 2, 3, 4, 5];

    c.bench_function("iter_sum", |b| {
        b.iter(|| data.iter().sum::<i32>())
    });

    c.bench_function("manual_sum", |b| {
        b.iter(|| {
            let mut sum = 0;
            for &x in &data { sum += x; }
            sum
        })
    });
}
```

### 2.3 flamegraph

```bash
# 使用 cargo-flamegraph 生成火焰图
cargo flamegraph --bench my_benchmark
```

### 2.4 async 基准测试

```rust
use criterion::{criterion_group, criterion_main, Criterion, async_executor::TokioExecutor};

fn bench_async(c: &mut Criterion) {
    c.bench_function("async_request", |b| {
        b.to_async(TokioExecutor).iter(|| async {
            reqwest::get("https://httpbin.org/get").await.unwrap()
        })
    });
}

criterion_group!(benches, bench_async);
criterion_main!(benches);
```

### 2.5 火焰图生成

```bash
# 安装
cargo install flamegraph

# 生成火焰图
cargo flamegraph --bench my_benchmark

# 指定输出
cargo flamegraph -o flame.svg --bench my_benchmark
```

## 四、性能分析工具

| 工具 | 用途 | 运行方式 |
|------|------|---------|
| Criterion | 统计基准测试 | `cargo bench` |
| flamegraph | 火焰图 | `cargo flamegraph` |
| perf | Linux 性能分析 | `perf record` |
| dtrace | macOS 性能分析 | `dtrace` |
| heaptrack | 内存分析 | `heaptrack` |
| valgrind | 内存检查 | `valgrind` |

## 五、注意事项与常见陷阱

1. **环境稳定**：在稳定的环境中运行基准测试，避免后台进程干扰
2. **多次运行**：统计分析需要多次运行，使用 Criterion 的统计功能
3. **编译优化**：使用 release 模式，dev 模式性能不具代表性
4. **避免干扰**：关闭其他程序减少干扰，使用隔离的测试环境
5. **版本对比**：跟踪不同版本的性能变化，使用 git bisect 定位性能回退
6. **micro vs macro**：同时进行微基准和宏观基准测试
7. **实际数据**：使用真实数据规模进行测试，避免小数据误导
