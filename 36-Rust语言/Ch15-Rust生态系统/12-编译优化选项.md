# 编译优化选项

## 一、概念说明

Rust 提供多种编译优化选项，平衡编译时间和运行时性能。

```toml
# Cargo.toml
[profile.release]
opt-level = 3
lto = true
codegen-units = 1
strip = true
panic = "abort"
```

## 二、具体用法

### 2.1 Profile 配置

```toml
[profile.dev]
opt-level = 0
debug = true
debug-assertions = true
overflow-checks = true

[profile.release]
opt-level = 3
lto = "thin"
codegen-units = 1
strip = "symbols"
panic = "abort"
incremental = false

[profile.test]
opt-level = 1
debug = true
```

### 2.2 LTO 优化

```toml
# 链接时优化
[profile.release]
lto = true      # 完全 LTO
lto = "thin"    # Thin LTO（编译更快）
lto = "fat"     # Fat LTO（优化更好）
```

### 2.3 目标 CPU 优化

```toml
# .cargo/config.toml
[build]
rustflags = ["-C", "target-cpu=native"]

# 特定目标
[target.x86_64-unknown-linux-gnu]
rustflags = ["-C", "target-cpu=skylake"]
```

### 2.4 大小优化

```toml
[profile.release]
opt-level = "z"  # 优化大小
strip = true
lto = true
codegen-units = 1
```

## 三、注意事项与常见陷阱

1. **编译时间**：优化增加编译时间
2. **调试能力**：release 模式可能难以调试
3. **CPU 兼容性**：target-cpu=native 可能不兼容旧 CPU
4. **二进制大小**：LTO 可能增加或减小二进制大小
5. **测试差异**：dev 和 release 模式行为可能不同
