# 编译优化选项

## 一、概念说明

Rust 提供多种编译优化选项，平衡编译时间和运行时性能。

```toml
# Cargo.toml
[profile.release]
opt-level = 3
lto = true
codegen-units = 1
strip = true
panic = "abort"
```

## 二、具体用法

### 2.1 Profile 配置

```toml
[profile.dev]
opt-level = 0
debug = true
debug-assertions = true
overflow-checks = true

[profile.release]
opt-level = 3
lto = "thin"
codegen-units = 1
strip = "symbols"
panic = "abort"
incremental = false

[profile.test]
opt-level = 1
debug = true
```

### 2.2 LTO 优化

```toml
# 链接时优化
[profile.release]
lto = true      # 完全 LTO
lto = "thin"    # Thin LTO（编译更快）
lto = "fat"     # Fat LTO（优化更好）
```

### 2.3 目标 CPU 优化

```toml
# .cargo/config.toml
[build]
rustflags = ["-C", "target-cpu=native"]

# 特定目标
[target.x86_64-unknown-linux-gnu]
rustflags = ["-C", "target-cpu=skylake"]
```

### 2.4 大小优化

```toml
[profile.release]
opt-level = "z"  # 优化大小
strip = true
lto = true
codegen-units = 1
```

### 2.5 分析二进制大小

```bash
# 使用 cargo-bloat 分析
cargo install cargo-bloat
cargo bloat --release --crates

# 使用 twiggy 分析 WASM
cargo install twiggy
twiggy top target/wasm32-unknown-unknown/release/mylib.wasm

# 使用 cargo-llvm-lines 分析代码行
cargo install cargo-llvm-lines
cargo llvm-lines --release | head -20
```

### 2.6 链接时优化对比

```
LTO 模式       编译时间    运行性能    二进制大小
───────────────────────────────────────────────
lto = false    最快        一般        最大
lto = "thin"   中等        好          中等
lto = true     最慢        最好        最小
```

## 四、Profile 配置速查

```toml
# 快速开发（快速编译，基本调试）
[profile.dev]
opt-level = 0
debug = true

# 测试（保留断言）
[profile.test]
opt-level = 1

# 发布（最高性能）
[profile.release]
opt-level = 3
lto = "thin"
codegen-units = 1
strip = "symbols"
panic = "abort"

# 最小大小
[profile.release-min]
inherits = "release"
opt-level = "z"
lto = true
```

## 五、注意事项与常见陷阱

1. **编译时间**：优化增加编译时间，使用增量编译缓解
2. **调试能力**：release 模式可能难以调试，保留调试符号或使用 `debug = "line-tables-only"`
3. **CPU 兼容性**：`target-cpu=native` 可能不兼容旧 CPU，仅在目标环境明确时使用
4. **二进制大小**：LTO 可能增加或减小二进制大小，实际测试
5. **测试差异**：dev 和 release 模式行为可能不同，使用 `profile.test` 保持一致
6. **overflow-checks**：release 默认关闭 overflow-checks，安全关键代码保持开启
7. **增量编译**：release 默认关闭增量编译，大型项目考虑启用
