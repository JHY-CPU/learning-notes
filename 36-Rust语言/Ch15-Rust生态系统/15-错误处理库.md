# 错误处理库

## 一、概念说明

Rust 生态有多种错误处理方案：thiserror（定义错误）、anyhow（应用错误）。

```rust
use thiserror::Error;

#[derive(Error, Debug)]
enum MyError {
    #[error("IO 错误: {0}")]
    Io(#[from] std::io::Error),
    #[error("解析错误: {0}")]
    Parse(#[from] std::num::ParseIntError),
    #[error("自定义错误: {message}")]
    Custom { message: String },
}
```

## 二、具体用法

### 2.1 thiserror 定义错误

```rust
use thiserror::Error;

#[derive(Error, Debug)]
enum ApiError {
    #[error("用户不存在")]
    NotFound,

    #[error("权限不足")]
    Forbidden,

    #[error("数据库错误: {0}")]
    Database(#[from] sqlx::Error),

    #[error("验证失败: {field}")]
    Validation { field: String },
}
```

### 2.2 anyhow 应用错误

```rust
use anyhow::{Context, Result};

fn process() -> Result<()> {
    let data = std::fs::read_to_string("config.toml")
        .context("读取配置文件失败")?;

    let config: Config = toml::from_str(&data)
        .context("解析配置失败")?;

    Ok(())
}
```

### 2.3 组合使用

```rust
// 库使用 thiserror 定义错误
// 应用使用 anyhow 处理错误

use my_lib::MyError;

fn main() -> anyhow::Result<()> {
    my_lib::function()
        .map_err(|e| anyhow::anyhow!("操作失败: {}", e))?;
    Ok(())
}
```

## 三、注意事项与常见陷阱

1. **选择策略**：库用 thiserror，应用用 anyhow
2. **错误链**：使用 source() 或 context() 维护错误链
3. **错误转换**：实现 From trait 进行错误转换
4. **错误信息**：提供有用的错误信息
5. **性能影响**：anyhow 有轻微性能开销
