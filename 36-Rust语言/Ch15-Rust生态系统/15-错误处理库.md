# 错误处理库

## 一、概念说明

Rust 生态有多种错误处理方案：thiserror（定义错误）、anyhow（应用错误）。

```rust
use thiserror::Error;

#[derive(Error, Debug)]
enum MyError {
    #[error("IO 错误: {0}")]
    Io(#[from] std::io::Error),
    #[error("解析错误: {0}")]
    Parse(#[from] std::num::ParseIntError),
    #[error("自定义错误: {message}")]
    Custom { message: String },
}
```

## 二、具体用法

### 2.1 thiserror 定义错误

```rust
use thiserror::Error;

#[derive(Error, Debug)]
enum ApiError {
    #[error("用户不存在")]
    NotFound,

    #[error("权限不足")]
    Forbidden,

    #[error("数据库错误: {0}")]
    Database(#[from] sqlx::Error),

    #[error("验证失败: {field}")]
    Validation { field: String },
}
```

### 2.2 anyhow 应用错误

```rust
use anyhow::{Context, Result};

fn process() -> Result<()> {
    let data = std::fs::read_to_string("config.toml")
        .context("读取配置文件失败")?;

    let config: Config = toml::from_str(&data)
        .context("解析配置失败")?;

    Ok(())
}
```

### 2.3 组合使用

```rust
// 库使用 thiserror 定义错误
// 应用使用 anyhow 处理错误

use my_lib::MyError;

fn main() -> anyhow::Result<()> {
    my_lib::function()
        .map_err(|e| anyhow::anyhow!("操作失败: {}", e))?;
    Ok(())
}
```

### 2.4 自定义 Error trait

```rust
use std::fmt;

#[derive(Debug)]
enum AppError {
    Io(std::io::Error),
    Parse(std::num::ParseIntError),
    Custom(String),
}

impl fmt::Display for AppError {
    fn fmt(&self, f: &mut fmt::Formatter) -> fmt::Result {
        match self {
            AppError::Io(e) => write!(f, "IO 错误: {}", e),
            AppError::Parse(e) => write!(f, "解析错误: {}", e),
            AppError::Custom(msg) => write!(f, "{}", msg),
        }
    }
}

impl std::error::Error for AppError {
    fn source(&self) -> Option<&(dyn std::error::Error + 'static)> {
        match self {
            AppError::Io(e) => Some(e),
            AppError::Parse(e) => Some(e),
            AppError::Custom(_) => None,
        }
    }
}

impl From<std::io::Error> for AppError {
    fn from(e: std::io::Error) -> Self {
        AppError::Io(e)
    }
}
```

### 2.5 color-eyre 增强报告

```rust
// Cargo.toml: color-eyre = "0.6"
use color_eyre::{eyre::WrapErr, Result};

fn main() -> Result<()> {

    let config = std::fs::read_to_string("config.toml")
        .wrap_err("读取配置文件失败")?;

    Ok(())

```

## 四、错误处理库对比

| 库 | 用途 | 特点 |
|------|------|------|
| thiserror | 定义错误类型 | 派生宏，零成本 |
| anyhow | 应用级错误 | 上下文，错误链 |
| color-eyre | 增强报告 | 彩色输出，回溯 |
| snafu | 定义错误 | 上下文感知 |
| thiserror-context | 带上下文 | 简单组合 |

## 五、注意事项与常见陷阱

1. **选择策略**：库用 thiserror 定义错误，应用用 anyhow 处理错误
2. **错误链**：使用 `source()` 或 `context()` 维护错误链，便于调试
3. **错误转换**：实现 `From` trait 进行错误转换，使用 `?` 操作符
4. **错误信息**：提供有用的错误信息，包含上下文
5. **性能影响**：anyhow 有轻微性能开销（回溯），生产环境可配置
6. **错误分类**：区分可恢复和不可恢复错误，正确使用 Result 和 panic
7. **日志记录**：在错误边界记录日志，但不要重复记录
