# 安全最佳实践

## 一、概念说明

Rust 的内存安全特性提供了很好的安全保障，但仍需注意其他安全问题。

```bash
# 使用 cargo-audit 检查依赖安全
cargo install cargo-audit
cargo audit

# 使用 cargo-deny 检查许可证和安全
cargo install cargo-deny
cargo deny check
```

## 二、具体用法

### 2.1 依赖安全

```toml
# deny.toml
[advisories]
vulnerability = "deny"
unmaintained = "warn"
yanked = "warn"

[licenses]
allow = ["MIT", "Apache-2.0"]
```

### 2.2 代码安全

```rust
// 避免 unsafe
// 如果必须使用 unsafe，确保充分测试

// 输入验证
fn validate_input(input: &str) -> Result<(), String> {
    if input.len() > 100 {
        return Err("输入过长".to_string());
    }
    if input.contains('\0') {
        return Err("包含空字符".to_string());
    }
    Ok(())
}

// 敏感数据处理
use zeroize::Zeroize;

#[derive(Zeroize)]
struct Password(String);
```

### 2.3 密码学安全

```rust
// 使用 rand 生成安全随机数
use rand::Rng;
use rand::rngs::OsRng;

let mut rng = OsRng;
let token: [u8; 32] = rng.gen();

// 使用 ring 或 rustls 处理 TLS
```

### 2.4 安全 HTTP 头

```rust
use axum::http::{header, HeaderMap, HeaderValue};

fn security_headers() -> HeaderMap {
    let mut headers = HeaderMap::new();
    headers.insert(
        header::STRICT_TRANSPORT_SECURITY,
        HeaderValue::from_static("max-age=31536000; includeSubDomains"),
    );
    headers.insert(
        header::X_CONTENT_TYPE_OPTIONS,
        HeaderValue::from_static("nosniff"),
    );
    headers.insert(
        header::X_FRAME_OPTIONS,
        HeaderValue::from_static("DENY"),
    );
    headers
}
```

### 2.5 安全依赖配置

```toml
# deny.toml (cargo-deny)
[advisories]
vulnerability = "deny"
unmaintained = "warn"
yanked = "warn"

[licenses]
allow = ["MIT", "Apache-2.0", "BSD-2-Clause", "BSD-3-Clause"]
deny = ["GPL-2.0", "GPL-3.0"]

[bans]
multiple-versions = "warn"
wildcards = "deny"

[sources]
unknown-registry = "warn"
unknown-git = "deny"
```

## 四、安全检查清单

```
代码安全：
- [ ] 避免 unsafe，或充分审查 unsafe 代码
- [ ] 输入验证所有外部数据
- [ ] 使用参数化查询防止 SQL 注入
- [ ] 敏感数据使用 zeroize 清理
- [ ] 使用常量时间比较防止时序攻击

依赖安全：
- [ ] 定期运行 cargo audit
- [ ] 配置 cargo-deny 检查许可证
- [ ] 最小化依赖数量
- [ ] 定期更新依赖

运行时安全：
- [ ] 配置安全 HTTP 头
- [ ] 启用 HTTPS
- [ ] 限制请求大小
- [ ] 配置超时
```

## 五、注意事项与常见陷阱

1. **依赖更新**：定期更新依赖修复安全漏洞，设置 CI 自动检查
2. **代码审查**：安全关键代码需要审查，特别关注 unsafe 代码
3. **安全测试**：进行渗透测试和安全审计
4. **密钥管理**：安全存储密钥，使用 Vault 或环境变量
5. **安全响应**：建立安全漏洞响应流程，及时响应和修复
6. **加密通信**：生产环境强制使用 HTTPS/TLS
7. **日志安全**：不要记录密码、密钥等敏感信息
