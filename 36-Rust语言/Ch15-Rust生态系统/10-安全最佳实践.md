# 安全最佳实践

## 一、概念说明

Rust 的内存安全特性提供了很好的安全保障，但仍需注意其他安全问题。

```bash
# 使用 cargo-audit 检查依赖安全
cargo install cargo-audit
cargo audit

# 使用 cargo-deny 检查许可证和安全
cargo install cargo-deny
cargo deny check
```

## 二、具体用法

### 2.1 依赖安全

```toml
# deny.toml
[advisories]
vulnerability = "deny"
unmaintained = "warn"
yanked = "warn"

[licenses]
allow = ["MIT", "Apache-2.0"]
```

### 2.2 代码安全

```rust
// 避免 unsafe
// 如果必须使用 unsafe，确保充分测试

// 输入验证
fn validate_input(input: &str) -> Result<(), String> {
    if input.len() > 100 {
        return Err("输入过长".to_string());
    }
    if input.contains('\0') {
        return Err("包含空字符".to_string());
    }
    Ok(())
}

// 敏感数据处理
use zeroize::Zeroize;

#[derive(Zeroize)]
struct Password(String);
```

### 2.3 密码学安全

```rust
// 使用 rand 生成安全随机数
use rand::Rng;
use rand::rngs::OsRng;

let mut rng = OsRng;
let token: [u8; 32] = rng.gen();

// 使用 ring 或 rustls 处理 TLS
```

## 三、注意事项与常见陷阱

1. **依赖更新**：定期更新依赖修复安全漏洞
2. **代码审查**：安全关键代码需要审查
3. **安全测试**：进行渗透测试
4. **密钥管理**：安全存储密钥
5. **安全响应**：建立安全漏洞响应流程
