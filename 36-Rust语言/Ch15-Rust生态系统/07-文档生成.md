# 文档生成

## 一、概念说明

Rust 使用 rustdoc 从代码注释生成文档，支持 Markdown 格式和代码示例测试。

```rust
/// 计算两个数的和
///
/// # 示例
///
/// ```
/// let result = my_crate::add(2, 3);
/// assert_eq!(result, 5);
/// ```
pub fn add(a: i32, b: i32) -> i32 {
    a + b
}
```

## 二、具体用法

### 2.1 文档注释

```rust
/// 结构体文档
///
/// # 示例
///
/// ```
/// use my_crate::MyStruct;
/// let s = MyStruct::new();
/// ```
pub struct MyStruct {
    /// 字段文档
    pub field: i32,
}

// 模块级文档
//! 模块级文档注释
//! 用于描述模块的功能
```

### 2.2 文档属性

```rust
/// 内部函数，不在公共文档中显示
#[doc(hidden)]
pub fn internal_function() {}

// 文档中不显示
#[cfg(not(doctest))]
pub fn test_only() {}
```

### 2.3 生成文档

```bash
# 生成并打开文档
cargo doc --open

# 包含私有项
cargo doc --document-private-items

# 无依赖文档
cargo doc --no-deps
```

### 2.4 高级文档技巧

```rust
/// 处理请求。
///
/// # 参数
///
/// * `req` - 请求对象，必须包含有效的认证信息
/// * `timeout` - 超时时间，单位毫秒
///
/// # 返回值
///
/// 返回处理结果，如果超时返回 `Err(TimeoutError)`
///
/// # 错误
///
/// 此函数在以下情况返回错误：
/// - `InvalidRequest` - 请求格式无效
/// - `TimeoutError` - 处理超时
/// - `AuthError` - 认证失败
///
/// # Panics
///
/// 如果 `req` 为 None 则 panic
///
/// # Safety
///
/// 此函数不包含 unsafe 代码
///
/// # Examples
///
/// ```
/// use my_crate::{Request, process_request};
///
/// let req = Request::new("hello");
/// let result = process_request(req, 1000);
/// assert!(result.is_ok());
/// ```
pub fn process_request(req: Request, timeout: u64) -> Result<Response, Error> {
    // ...
}
```

## 四、文档生成最佳实践

```
每个公共项应该有：
1. 简短描述（第一行）
2. 详细说明（可选）
3. 参数说明（如果有）
4. 返回值说明（如果有）
5. 错误说明（如果返回 Result）
6. Panics 说明（如果可能 panic）
7. 示例代码（可运行）
8. 相关项链接
```

## 五、注意事项与常见陷阱

1. **示例测试**：文档中的代码示例会被测试，确保示例可编译运行
2. **完整性**：为所有公共 API 提供文档，使用 `#![warn(missing_docs)]` 检查
3. **链接**：使用 `[`Type`]` 和 [`fn_name()`] 链接引用相关项
4. **版本信息**：添加 `# Stability` 和 `# Since` 信息
5. **国际化**：考虑文档的国际化，英文为主
6. **私有文档**：为内部实现添加 `//` 注释帮助维护
7. **文档更新**：代码变更时同步更新文档
