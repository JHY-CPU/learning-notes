# 跨平台编译

## 一、概念说明

Rust 支持交叉编译到多种目标平台。

```bash
# 添加目标平台
rustup target add wasm32-unknown-unknown
rustup target add aarch64-unknown-linux-gnu

# 交叉编译
cargo build --target wasm32-unknown-unknown
```

## 二、具体用法

### 2.1 常用目标平台

```bash
# WebAssembly
rustup target add wasm32-unknown-unknown
rustup target add wasm32-wasi

# ARM Linux
rustup target add aarch64-unknown-linux-gnu
rustup target add armv7-unknown-linux-gnueabihf

# Windows
rustup target add x86_64-pc-windows-gnu

# macOS
rustup target add aarch64-apple-darwin
```

### 2.2 交叉编译工具链

```bash
# 安装交叉编译工具
sudo apt install gcc-aarch64-linux-gnu

# 配置 .cargo/config.toml
[target.aarch64-unknown-linux-gnu]
linker = "aarch64-linux-gnu-gcc"
```

### 2.3 Docker 交叉编译

```dockerfile
FROM rust:1.70
RUN rustup target add aarch64-unknown-linux-gnu
RUN apt-get update && apt-get install -y gcc-aarch64-linux-gnu
```

### 2.4 条件编译

```rust
// 平台特定代码
#[cfg(target_os = "linux")]
fn platform_specific() {
    println!("Linux");
}

#[cfg(target_os = "windows")]
fn platform_specific() {
    println!("Windows");
}

// 多平台条件
#[cfg(any(target_os = "linux", target_os = "macos"))]
fn unix_like() { /* ... */ }

// 特性条件
#[cfg(feature = "async")]
async fn async_function() { /* ... */ }
```

### 2.5 多平台 CI

```yaml
# GitHub Actions
jobs:
  build:
    strategy:
      matrix:
        os: [ubuntu-latest, windows-latest, macos-latest]
        rust: [stable, nightly]
    runs-on: ${{ matrix.os }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions-rs/toolchain@v1
        with:
          toolchain: ${{ matrix.rust }}
      - run: cargo build --release
      - run: cargo test
```

## 四、常见目标平台

| 目标 | 用途 | 工具链 |
|------|------|--------|
| `x86_64-unknown-linux-gnu` | Linux x64 | 默认 |
| `aarch64-unknown-linux-gnu` | ARM Linux | gcc-aarch64 |
| `x86_64-pc-windows-msvc` | Windows x64 | MSVC |
| `aarch64-apple-darwin` | macOS ARM | Xcode |
| `wasm32-unknown-unknown` | WebAssembly | 默认 |
| `wasm32-wasi` | WASI | 默认 |

## 五、注意事项与常见陷阱

1. **依赖问题**：某些 crate 不支持所有平台，检查 crate 的 CI 矩阵
2. **系统库**：交叉编译需要目标平台的系统库和链接器
3. **测试**：需要在目标平台上测试，使用 CI 矩阵测试多平台
4. **CI 配置**：配置 CI 进行多平台构建，确保代码在所有平台编译
5. **条件编译**：使用 `cfg` 处理平台差异，避免硬编码路径
6. **字节序**：注意不同平台的字节序差异
7. **文件路径**：使用 `std::path::Path` 处理路径，避免硬编码分隔符
