# 日志与追踪

## 一、概念说明

Rust 有 log（简单日志）和 tracing（结构化日志和追踪）两大日志框架。

```rust
use tracing::{info, warn, error, instrument};

#[instrument]
async fn process_request(id: u64) -> Result<(), Error> {
    info!("处理请求");
    // ...
    warn!("异常情况");
    Ok(())
}
```

## 二、具体用法

### 2.1 log 基础

```rust
use log::{info, warn, error, debug, trace};

fn main() {
    env_logger::init();

    info!("应用启动");
    debug!("调试信息");
    warn!("警告信息");
    error!("错误信息");
}
```

### 2.2 tracing 结构化

```rust
use tracing::{info_span, Instrument};

async fn handler() {
    let span = info_span!("request", id = %uuid::Uuid::new_v4());
    async {
        info!("处理中");
    }
    .instrument(span)
    .await;
}
```

### 2.3 tracing-subscriber

```rust
use tracing_subscriber::fmt;
use tracing_subscriber::EnvFilter;

fn init_tracing() {
    tracing_subscriber::fmt()
        .with_env_filter(EnvFilter::from_default_env())
        .with_target(true)
        .with_thread_ids(true)
        .init();
}
```

### 2.4 输出到文件

```rust
use tracing_subscriber::fmt;
use std::fs::File;

fn init_file_logging() {
    let file = File::create("app.log").unwrap();
    tracing_subscriber::fmt()
        .with_writer(file)
        .init();
}
```

### 2.5 OpenTelemetry 集成

```rust
use opentelemetry::sdk::trace as sdktrace;
use opentelemetry_otlp::WithExportConfig;
use tracing_subscriber::prelude::*;

fn init_telemetry() {
    let tracer = opentelemetry_otlp::new_pipeline()
        .tracing()
        .with_exporter(opentelemetry_otlp::new_collector()
            .with_endpoint("http://localhost:4317"))
        .install_batch(opentelemetry::runtime::Tokio)
        .unwrap();

    let telemetry = tracing_opentelemetry::layer().with_tracer(tracer);

    tracing_subscriber::registry()
        .with(telemetry)
        .with(tracing_subscriber::fmt::layer())
        .init();
}
```

### 2.6 日志过滤规则

```rust
use tracing_subscriber::EnvFilter;

fn init_with_filter() {
    // RUST_LOG=my_app=debug,tower=info
    tracing_subscriber::fmt()
        .with_env_filter(
            EnvFilter::try_from_default_env()
                .unwrap_or_else(|_| EnvFilter::new("info"))
        )
        .init();
}

// 代码中动态调整
use tracing::Level;
tracing::event!(Level::INFO, "信息消息");
```

## 四、log vs tracing 选择

| 需求 | 推荐 | 原因 |
|------|------|------|
| 简单日志 | log + env_logger | 轻量 |
| 结构化日志 | tracing | 支持结构化字段 |
| 分布式追踪 | tracing + OpenTelemetry | 完整支持 |
| 高性能 | tracing | 零成本抽象 |
| 生态兼容 | tracing | 主流选择 |

## 五、注意事项与常见陷阱

1. **性能影响**：日志有性能开销，生产环境调整级别为 INFO 或 WARN
2. **敏感信息**：不要记录密码、密钥等敏感信息
3. **日志轮转**：实现日志文件轮转，避免磁盘满
4. **结构化字段**：tracing 的字段是结构化的，便于日志分析
5. **异步安全**：确保日志在异步环境安全，使用 tracing
6. **采样配置**：分布式追踪配置采样率，避免存储过多数据
7. **日志格式**：生产环境使用 JSON 格式，便于日志聚合工具解析
