# 日志与追踪

## 一、概念说明

Rust 有 log（简单日志）和 tracing（结构化日志和追踪）两大日志框架。

```rust
use tracing::{info, warn, error, instrument};

#[instrument]
async fn process_request(id: u64) -> Result<(), Error> {
    info!("处理请求");
    // ...
    warn!("异常情况");
    Ok(())
}
```

## 二、具体用法

### 2.1 log 基础

```rust
use log::{info, warn, error, debug, trace};

fn main() {
    env_logger::init();

    info!("应用启动");
    debug!("调试信息");
    warn!("警告信息");
    error!("错误信息");
}
```

### 2.2 tracing 结构化

```rust
use tracing::{info_span, Instrument};

async fn handler() {
    let span = info_span!("request", id = %uuid::Uuid::new_v4());
    async {
        info!("处理中");
    }
    .instrument(span)
    .await;
}
```

### 2.3 tracing-subscriber

```rust
use tracing_subscriber::fmt;
use tracing_subscriber::EnvFilter;

fn init_tracing() {
    tracing_subscriber::fmt()
        .with_env_filter(EnvFilter::from_default_env())
        .with_target(true)
        .with_thread_ids(true)
        .init();
}
```

### 2.4 输出到文件

```rust
use tracing_subscriber::fmt;
use std::fs::File;

fn init_file_logging() {
    let file = File::create("app.log").unwrap();
    tracing_subscriber::fmt()
        .with_writer(file)
        .init();
}
```

## 三、注意事项与常见陷阱

1. **性能影响**：日志有性能开销，生产环境调整级别
2. **敏感信息**：不要记录密码等敏感信息
3. **日志轮转**：实现日志文件轮转
4. **结构化字段**：tracing 的字段是结构化的
5. **异步安全**：确保日志在异步环境安全
