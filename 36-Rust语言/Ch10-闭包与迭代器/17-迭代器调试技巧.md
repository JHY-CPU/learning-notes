# 迭代器调试技巧

## 一、概念说明

调试迭代器链可能比较困难，因为中间状态不可见。Rust 提供了 `inspect` 方法和其他技巧来观察迭代器的中间状态。

```rust
let result: Vec<i32> = (1..10)
    .inspect(|x| println!("输入: {}", x))
    .filter(|x| x % 2 == 0)
    .inspect(|x| println!("过滤后: {}", x))
    .map(|x| x * x)
    .inspect(|x| println!("映射后: {}", x))
    .collect();
```

## 二、具体用法

### 2.1 inspect 方法

```rust
let data = vec!["hello", "WORLD", "Rust"];

let result: Vec<String> = data.iter()
    .inspect(|s| println!("原始: '{}'", s))
    .map(|s| s.to_lowercase())
    .inspect(|s| println!("小写: '{}'", s))
    .filter(|s| s.len() > 3)
    .inspect(|s| println!("保留: '{}'", s))
    .map(|s| format!("[{}]", s))
    .collect();

// 输出调试信息而不影响迭代器
```

### 2.2 可视化迭代过程

```rust
fn debug_iter<I: Iterator>(iter: I) -> impl Iterator<Item = I::Item>
where
    I::Item: std::fmt::Debug,
{
    iter.enumerate().inspect(|(i, val)| {
        println!("[{}] => {:?}", i, val);
    }).map(|(_, val)| val)
}

// 使用
let result: Vec<i32> = debug_iter(
    vec![1, 2, 3].into_iter()
        .map(|x| x * 2)
).collect();
```

### 2.3 断言中间状态

```rust
#[cfg(debug_assertions)]
fn assert_sorted<I: Iterator<Item = i32>>(iter: I) -> I {
    let collected: Vec<i32> = iter.collect();
    assert!(collected.windows(2).all(|w| w[0] <= w[1]));
    collected.into_iter()
}

let data = vec![3, 1, 4, 1, 5];
let sorted = assert_sorted(data.into_iter().sorted());
```

### 2.4 条件调试宏

```rust
macro_rules! debug_iter {
    ($iter:expr) => {{
        #[cfg(debug_assertions)]
        {
            $iter.inspect(|val| eprintln!("[DEBUG] {:?}", val))
        }
        #[cfg(not(debug_assertions))]
        {
            $iter
        }
    }};
}

// 使用
fn process_data(data: Vec<i32>) -> Vec<i32> {
    debug_iter!(data.into_iter())
        .filter(|x| x % 2 == 0)
        .map(|x| x * x)
        .collect()
}
```

### 2.5 性能分析迭代器

```rust
use std::time::Instant;

fn profile_iter<I: Iterator>(iter: I) -> impl Iterator<Item = I::Item> {
    let start = Instant::now();
    let mut count = 0usize;

    iter.inspect(move |_| {
        count += 1;
        if count % 10000 == 0 {
            eprintln!("已处理 {} 个元素，耗时 {:?}", count, start.elapsed());
        }
    })
}

// 使用
fn profiled_pipeline() {
    let data: Vec<i32> = (0..100_000).collect();

    let result: Vec<i32> = profile_iter(data.into_iter())
        .filter(|x| x % 3 == 0)
        .map(|x| x * x)
        .collect();

    eprintln!("最终结果数: {}", result.len());
}
```

### 2.6 使用 dbg! 宏调试

```rust
fn debug_with_dbg() {
    let result: Vec<i32> = (1..10)
        .map(|x| dbg!(x * 2))  // dbg! 打印值和位置
        .filter(|&x| dbg!(x > 5))
        .collect();

    // dbg! 使用 eprintln! 输出到 stderr，不影响 stdout
    // 格式：[src/main.rs:42] x * 2 = 2
}
```

### 2.7 自定义调试迭代器

```rust
struct DebugIter<I> {
    inner: I,
    name: &'static str,
    count: usize,
}

impl<I: Iterator> Iterator for DebugIter<I> {
    type Item = I::Item;

    fn next(&mut self) -> Option<Self::Item> {
        self.inner.next().map(|item| {
            self.count += 1;
            eprintln!("[{}] #{}: {:?}", self.name, self.count, item);
            item
        })
    }
}

trait IteratorExt: Iterator + Sized {
    fn debug(self, name: &'static str) -> DebugIter<Self> {
        DebugIter { inner: self, name, count: 0 }
    }
}

impl<T: Iterator + Sized> IteratorExt for T {}

// 使用
fn custom_debug_example() {
    let result: Vec<i32> = (1..=5)
        .debug("输入")
        .map(|x| x * 3)
        .debug("乘3后")
        .filter(|&x| x > 5)
        .debug("过滤后")
        .collect();
}
```

### 2.8 迭代器惰性求值调试

```rust
fn debug_lazy_evaluation() {
    // 迭代器是惰性的，只有 collect/for_each 等消费操作才会执行
    let iter = vec![1, 2, 3, 4, 5]
        .into_iter()
        .inspect(|x| eprintln!("map 前: {}", x))
        .map(|x| {
            eprintln!("正在 map: {}", x);
            x * x
        })
        .inspect(|x| eprintln!("map 后: {}", x));

    // 到这里还没有执行任何操作！
    eprintln!("迭代器已创建，但未执行");

    // 消费迭代器时才执行
    let result: Vec<i32> = iter.collect();
    eprintln!("结果: {:?}", result);
}
```

## 四、调试工具集成

### 4.1 使用 tracing 进行结构化调试

```rust
use tracing::{info, debug, instrument};

#[instrument]
fn process_with_tracing(data: Vec<i32>) -> Vec<i32> {
    debug!(count = data.len(), "开始处理");

    let result: Vec<i32> = data.into_iter()
        .inspect(|x| debug!(value = x, "处理中"))
        .map(|x| x * 2)
        .collect();

    info!(count = result.len(), "处理完成");
    result
}
```

### 4.2 内存使用监控

```rust
fn debug_memory_usage<I: Iterator>(iter: I, label: &str) -> impl Iterator<Item = I::Item> {
    iter.inspect(move |_| {
        // 可以结合 memory-stats crate 监控内存
        static mut COUNTER: usize = 0;
        unsafe {
            COUNTER += 1;
            if COUNTER % 10000 == 0 {
                eprintln!("[{}] 已处理 {} 项", label, COUNTER);
            }
        }
    })
}
```

## 五、注意事项与常见陷阱

1. **inspect 不修改**：inspect 只读取，不改变迭代器内容，但闭包内可以执行副作用
2. **Release 模式**：调试代码在 release 中可能被优化掉，使用 `#[cfg(debug_assertions)]` 控制
3. **日志级别**：结合 log crate 控制调试输出，避免生产环境输出过多日志
4. **条件编译**：使用 `#[cfg(debug_assertions)]` 包裹调试代码，避免性能影响
5. **性能影响**：inspect 中的 println 会显著降低性能，应仅在调试时使用
6. **惰性求值**：记住迭代器是惰性的，调试输出只有在消费操作时才会出现
7. **多线程调试**：并行迭代器（rayon）的调试输出可能交错，需要考虑线程安全
