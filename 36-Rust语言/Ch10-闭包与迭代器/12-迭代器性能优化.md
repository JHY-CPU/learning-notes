# 迭代器性能优化

## 一、概念说明

Rust 迭代器是零成本抽象的典型代表。编译器会将迭代器链优化为与手写循环相当的机器码，某些情况下甚至更快。

```rust
// 这两种写法编译后性能相同
let sum1: i32 = (0..1000).sum();
let mut sum2 = 0;
for i in 0..1000 { sum2 += i; }
```

## 二、具体用法

### 2.1 避免中间集合

```rust
// 不好：多次分配中间集合
let data = vec![1, 2, 3, 4, 5];
let doubled: Vec<i32> = data.iter().map(|x| x * 2).collect();
let filtered: Vec<i32> = doubled.iter().filter(|x| *x > &4).cloned().collect();
let sum: i32 = filtered.iter().sum();

// 好：单一迭代器链，无中间分配
let sum: i32 = vec![1, 2, 3, 4, 5]
    .iter()
    .map(|x| x * 2)
    .filter(|x| *x > 4)
    .sum();
```

### 2.2 使用 size_hint 预分配

```rust
fn process(data: &[i32]) -> Vec<String> {
    let mut result = Vec::with_capacity(data.len());
    for item in data {
        result.push(format!("值: {}", item));
    }
    result
}

// 迭代器版本自动使用 size_hint 预分配
fn process_iter(data: &[i32]) -> Vec<String> {
    data.iter()
        .map(|x| format!("值: {}", x))
        .collect()
}
```

### 2.3 使用 SIMD 优化

```rust
// 使用 std::simd（nightly）或手动优化
use std::arch::x86_64::*;

unsafe fn sum_f32_simd(data: &[f32]) -> f32 {
    let mut sum = _mm256_setzero_ps();
    for chunk in data.chunks_exact(8) {
        let v = _mm256_loadu_ps(chunk.as_ptr());
        sum = _mm256_add_ps(sum, v);
    }
    // 处理剩余元素
    let mut result = 0.0;
    for &x in data.chunks_exact(8).remainder() {
        result += x;
    }
    // 水平求和
    let mut buffer = [0.0f32; 8];
    _mm256_storeu_ps(buffer.as_mut_ptr(), sum);
    result + buffer.iter().sum::<f32>()
}
```

## 三、注意事项与常见陷阱

1. **编译器优化**：Release 模式下迭代器会被充分优化
2. **内联展开**：迭代器方法标记了 `#[inline]`，编译器会内联
3. **分支预测**：迭代器链的分支可被 CPU 优化
4. **内存布局**：连续内存访问比随机访问快
5. **测量工具**：使用 criterion crate 进行精确基准测试
