# 闭包与所有权

## 一、概念说明

Rust 闭包的捕获规则与所有权系统紧密相关。闭包可以不可变借用、可变借用或获取环境变量的所有权，编译器会根据闭包体内的使用方式自动选择最合适的捕获方式。

```rust
let s = String::from("hello");

// 不可变借用（闭包只读取s）
let borrow = || println!("{}", s);
borrow();
// s 仍可使用

// 所有权转移
let consume = move || println!("{}", s);
consume();
// s 不再可用
```

## 二、具体用法

### 2.1 借用捕获

```rust
let mut data = vec![1, 2, 3];

// 不可变借用
let reader = || {
    println!("数据: {:?}", data);
};
reader();
// data 仍可使用

// 可变借用
let mut writer = || {
    data.push(4);
};
writer();
// data 仍可使用，但 writer 不能再调用
println!("{:?}", data);
```

### 2.2 move 闭包

```rust
let name = String::from("Rust");

// move 强制获取所有权
let closure = move || {
    println!("你好，{}！", name);
    drop(name);
};
closure();
// name 不再可用

// 多线程必须使用 move
use std::thread;

let data = vec![1, 2, 3];
let handle = thread::spawn(move || {
    data.iter().sum::<i32>()
});
// data 被移动到新线程
let sum = handle.join().unwrap();
```

### 2.3 返回闭包中的所有权

```rust
fn make_closure() -> impl FnOnce() -> String {
    let message = String::from("来自闭包");
    move || message  // 所有权随闭包返回
}

let closure = make_closure();
let result = closure();  // 调用时消耗所有权
println!("{}", result);
```

### 2.4 部分捕获

```rust
let a = 1;
let b = String::from("hello");
let c = vec![1, 2, 3];

// 只捕获需要的变量
let partial = || {
    println!("a: {}", a);  // 不可变借用 a
    println!("b: {}", b);  // 不可变借用 b
    // c 未被使用，不会被捕获
};

partial();
// a, b, c 都仍可用
```

## 三、注意事项与常见陷阱

1. **自动推断**：编译器根据闭包体自动选择最小捕获方式
2. **move 优先级**：move 强制所有权转移，覆盖自动推断
3. **Copy 类型**：实现了 Copy 的类型 move 后仍可用
4. **借用冲突**：同时存在可变和不可变借用会编译错误
5. **线程场景**：跨线程闭包必须使用 move 获取所有权
