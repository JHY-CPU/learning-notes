# 惰性求值机制

## 一、概念说明

Rust 的迭代器采用惰性求值模式：适配器（map、filter 等）只在消费时（collect、sum、for 循环）才执行。这种机制允许高效的数据处理流水线。

```rust
// 这行代码不会执行任何计算
let lazy = (1..1_000_000)
    .filter(|x| x % 2 == 0)
    .map(|x| x * x);

// 只有消费迭代器时才执行
let result: Vec<i32> = lazy.take(5).collect();
// 计算量：只需处理到找到5个偶数为止
```

## 二、具体用法

### 2.1 短路求值

```rust
// any: 找到true就停止
let has_even = (1..).any(|x| x % 2 == 0 && x > 100);
// 不会无限循环，找到102时停止

// find: 找到匹配项就停止
let first_big = (1..1000).find(|x| x * x > 1000);
// 找到32时停止

// take_while: 条件为false时停止
let below_10: Vec<i32> = (1..)
    .take_while(|x| x * x < 100)
    .collect();
// [1, 2, 3, ..., 9]
```

### 2.2 无限迭代器

```rust
// repeat: 无限重复
let sum: i32 = std::iter::repeat(1)
    .take(1000)
    .sum();
// 1000

// successors: 有状态序列
let powers_of_2: Vec<u64> = std::iter::successors(Some(1u64), |&n| {
    if n < 1_000_000 {
        Some(n * 2)
    } else {
        None
    }
}).collect();

// from_fn: 自定义无限迭代器
let naturals = std::iter::from_fn({
    let mut n = 0;
    move || {
        n += 1;
        Some(n)
    }
});
let first_ten: Vec<i32> = naturals.take(10).collect();
```

### 2.3 惰性 vs 急切

```rust
// 急切：立即执行，创建中间集合
let data = vec![1, 2, 3, 4, 5];
let doubled: Vec<i32> = data.iter().map(|x| x * 2).collect();
let filtered: Vec<i32> = doubled.iter().filter(|x| *x > 5).cloned().collect();
// 两次内存分配，两次遍历

// 惰性：单次遍历，无中间分配
let filtered: Vec<i32> = vec![1, 2, 3, 4, 5]
    .iter()
    .map(|x| x * 2)
    .filter(|x| *x > 5)
    .cloned()
    .collect();
// 一次遍历，一次分配
```

## 三、注意事项与常见陷阱

1. **副作用时机**：惰性执行中 println 等副作用只在消费时发生
2. **无限序列**：使用 take/take_while 限制无限迭代器
3. **调试困难**：使用 inspect 方法在不改变迭代器的情况下调试
4. **内存效率**：惰性避免了中间集合的内存开销
5. **编译时优化**：Rust 编译器会将整个迭代器链优化为单一循环
