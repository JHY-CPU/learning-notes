# 函数式编程风格

## 一、概念说明

Rust 通过闭包和迭代器支持函数式编程范式。高阶函数、纯函数、不可变数据等概念在 Rust 中有良好的表达。

```rust
// 函数式风格的数据处理
let result: Vec<String> = data.iter()
    .filter(|x| x.is_valid())
    .map(|x| x.transform())
    .collect();
```

## 二、具体用法

### 2.1 高阶函数

```rust
// 返回函数的函数
fn compose<F, G, A, B, C>(f: F, g: G) -> impl Fn(A) -> C
where
    F: Fn(A) -> B,
    G: Fn(B) -> C,
{
    move |x| g(f(x))
}

let add_then_double = compose(|x: i32| x + 1, |x: i32| x * 2);
println!("{}", add_then_double(5)); // (5+1)*2 = 12

// 接受多个函数的函数
fn pipeline<T>(value: T, fns: &[Box<dyn Fn(T) -> T>]) -> T {
    fns.iter().fold(value, |v, f| f(v))
}
```

### 2.2 柯里化模式

```rust
// 手动柯里化
fn add(x: i32) -> impl Fn(i32) -> i32 {
    move |y| x + y
}

let add_5 = add(5);
let add_10 = add(10);
println!("{}", add_5(3));  // 8
println!("{}", add_10(3)); // 13

// 偏函数应用
fn multiply(a: i32) -> impl Fn(i32) -> i32 {
    move |b| a * b
}

let double = multiply(2);
let triple = multiply(3);
```

### 2.3 Option/Result 函数式处理

```rust
fn find_user(id: u32) -> Option<String> {
    Some(format!("用户{}", id))
}

fn validate_name(name: String) -> Result<String, String> {
    if name.len() > 0 {
        Ok(name)
    } else {
        Err("名字为空".to_string())
    }
}

// 链式调用
let result = find_user(1)
    .map(|name| name.to_uppercase())
    .and_then(|name| validate_name(name))
    .unwrap_or_else(|_| "默认用户".to_string());
```

## 三、注意事项与常见陷阱

1. **不可变优先**：函数式风格倾向不可变数据，减少副作用
2. **闭包捕获**：注意闭包对环境变量的捕获方式
3. **性能平衡**：过度函数式可能影响性能和可读性
4. **调试困难**：深嵌套的闭包链难以调试
5. **混合使用**：Rust 允许命令式和函数式风格混合
