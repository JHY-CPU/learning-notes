# 闭包与错误处理

## 一、概念说明

闭包可以与 Result/Option 类型配合使用，实现灵活的错误处理。迭代器提供了 `filter_map`、`flat_map` 等方法，可以优雅地处理可能失败的操作。

```rust
let inputs = vec!["1", "hello", "3", "world"];
let numbers: Vec<i32> = inputs.iter()
    .filter_map(|s| s.parse::<i32>().ok())
    .collect();
// [1, 3]（忽略解析失败的）
```

## 二、具体用法

### 2.1 迭代器中的错误处理

```rust
// collect 为 Result 集合
fn parse_all(input: &[&str]) -> Result<Vec<i32>, std::num::ParseIntError> {
    input.iter().map(|s| s.parse::<i32>()).collect()
}

// 全部成功或全部失败
match parse_all(&["1", "2", "3"]) {
    Ok(nums) => println!("{:?}", nums),
    Err(e) => println!("解析失败: {}", e),
}

// 部分成功处理
fn parse_partial(input: &[&str]) -> (Vec<i32>, Vec<&str>) {
    let mut successes = Vec::new();
    let mut failures = Vec::new();
    for s in input {
        match s.parse::<i32>() {
            Ok(n) => successes.push(n),
            Err(_) => failures.push(*s),
        }
    }
    (successes, failures)
}
```

### 2.2 闭包错误恢复

```rust
fn process_with_retry<F, T, E>(mut op: F, max_retries: usize) -> Result<T, E>
where
    F: FnMut() -> Result<T, E>,
{
    let mut last_err = None;
    for _ in 0..max_retries {
        match op() {
            Ok(val) => return Ok(val),
            Err(e) => last_err = Some(e),
        }
    }
    Err(last_err.unwrap())
}

// 使用
let result = process_with_retry(
    || {
        let n: i32 = "abc".parse().map_err(|_| "解析失败")?;
        Ok(n)
    },
    3,
);
```

### 2.3 Option 链式处理

```rust
struct User { name: String, email: Option<String> }
struct Email { address: String }

fn find_user(id: u32) -> Option<User> {
    Some(User { name: "张三".into(), email: Some("test@example.com".into()) })
}

fn get_email(user: &User) -> Option<&str> {
    user.email.as_deref()
}

// 链式调用
let email = find_user(1)
    .as_ref()
    .and_then(|u| get_email(u))
    .unwrap_or("无邮箱");

// 使用闭包处理默认值
let greeting = find_user(1)
    .map(|u| format!("你好，{}！", u.name))
    .unwrap_or_else(|| "用户不存在".to_string());
```

## 三、注意事项与常见陷阱

1. **collect 特殊性**：collect 到 Result 时会短路于第一个错误
2. **错误类型统一**：不同错误类型需统一为一种或使用 anyhow
3. **unwrap 风险**：生产代码中避免 unwrap，使用 expect 或显式处理
4. **闭包捕获错误**：闭包可以捕获外部错误上下文
5. **迭代器中的 ?**：在闭包中不能直接用 ?，需转换为 Result
