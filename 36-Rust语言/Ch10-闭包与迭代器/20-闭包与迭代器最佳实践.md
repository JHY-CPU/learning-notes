# 闭包与迭代器最佳实践

## 一、概念说明

总结闭包和迭代器的使用模式和最佳实践，帮助写出高效、可读、地道的 Rust 代码。

```rust
// 好的实践：清晰的迭代器链
let total: f64 = orders.iter()
    .filter(|o| o.status == Status::Completed)
    .map(|o| o.amount)
    .sum();
```

## 二、具体用法

### 2.1 选择正确的迭代器方法

```rust
// 只需第一个匹配项时用 find，不用 filter + next
let found = data.iter().find(|x| x.is_valid()); // 好

// 需要索引时用 position
let pos = data.iter().position(|x| *x == target);

// 需要累积结果用 fold，不用手动循环
let sum = data.iter().fold(0, |acc, x| acc + x);

// 链式操作避免中间集合
let result: Vec<_> = data.iter()
    .filter(|x| x > &&0)
    .map(|x| x * x)
    .collect();
```

### 2.2 闭包的命名与拆分

```rust
// 复杂闭包提取为命名函数
fn is_valid_order(order: &Order) -> bool {
    order.amount > 0.0 && order.items.len() > 0
}

let valid_orders: Vec<_> = orders.iter()
    .filter(|o| is_valid_order(o))
    .collect();

// 简短闭包可直接内联
let doubled: Vec<_> = data.iter().map(|x| x * 2).collect();
```

### 2.3 性能考虑

```rust
// 大数据集考虑并行
use rayon::prelude::*;
let sum: i64 = data.par_iter().sum();

// 避免不必要的 clone
let refs: Vec<&String> = data.iter().collect(); // 好
let clones: Vec<String> = data.iter().cloned().collect(); // 避免除非必要

// 使用 into_iter 消耗所有权避免引用计数
let processed: Vec<_> = items.into_iter()
    .map(|item| expensive_transform(item))
    .collect();
```

### 2.4 可读性原则

```rust
// 长链拆分为多行
let result: Vec<String> = data
    .iter()
    .filter(|x| x.is_active())
    .map(|x| x.name.clone())
    .map(|name| name.to_uppercase())
    .collect();

// 使用类型注释帮助理解
let names: Vec<String> = users.iter()
    .map(|u| u.full_name())
    .collect();
```

## 三、注意事项与常见陷阱

1. **默认选择**：优先使用迭代器而非手写循环
2. **中间集合**：避免不必要的 collect 调用
3. **闭包大小**：大的闭包用 Box 包装减少栈开销
4. **trait bound**：选择最合适的 Fn/FnMut/FnOnce 约束
5. **代码审查**：迭代器链应清晰表达意图，否则考虑拆分
