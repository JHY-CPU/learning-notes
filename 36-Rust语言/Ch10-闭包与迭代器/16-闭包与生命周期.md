# 闭包与生命周期

## 一、概念说明

闭包的生命周期与其捕获变量的生命周期密切相关。当闭包捕获引用时，闭包的生命周期不能超过被捕获引用的生命周期。

```rust
fn make_greeter<'a>(name: &'a str) -> impl Fn() + 'a {
    move || println!("你好，{}！", name)
    // 闭包的生命周期不超过 name 的生命周期
}
```

## 二、具体用法

### 2.1 返回捕获引用的闭包

```rust
fn find_in_slice<'a, T: PartialEq>(
    data: &'a [T],
    target: &'a T,
) -> impl Fn() -> Option<&'a T> + 'a {
    move || data.iter().find(|x| *x == target)
}

let data = vec![1, 2, 3, 4, 5];
let target = 3;
let finder = find_in_slice(&data, &target);
let result = finder();
```

### 2.2 闭包作为结构体字段

```rust
struct Processor<'a, F>
where
    F: Fn(&str) -> bool + 'a,
{
    validator: F,
    data: &'a [String],
}

impl<'a, F> Processor<'a, F>
where
    F: Fn(&str) -> bool + 'a,
{
    fn process(&self) -> Vec<&str> {
        self.data.iter()
            .filter(|s| (self.validator)(s))
            .map(|s| s.as_str())
            .collect()
    }
}
```

### 2.3 多生命周期参数

```rust
fn transform<'a, 'b>(
    input: &'a str,
    prefix: &'b str,
) -> impl Fn() -> String + 'a + 'b {
    move || format!("{}: {}", prefix, input)
}

// 闭包必须在 prefix 和 input 都有效时使用
```

### 2.4 高阶函数的生命周期

```rust
fn apply_to_all<'a, F>(
    data: &'a [String],
    transform: F,
) -> Vec<String>
where
    F: Fn(&'a str) -> String,
{
    data.iter()
        .map(|s| transform(s))
        .collect()
}
```

## 三、注意事项与常见陷阱

1. **生命周期约束**：impl Trait 返回需指定捕获变量的生命周期
2. **结构体存储**：闭包作为字段时需携带生命周期参数
3. **编译错误**：生命周期不匹配是常见的编译错误
4. **trait 对象**：dyn Fn 也需要生命周期标注
5. **move 与生命周期**：move 不改变引用的生命周期要求
