# 闭包实战模式

## 一、概念说明

闭包在 Rust 中广泛用于回调、事件处理、配置模式、策略模式等场景。通过闭包可以实现灵活的、可组合的代码结构。

```rust
// 策略模式
fn process_data<F>(data: &[i32], strategy: F) -> Vec<i32>
where
    F: Fn(&[i32]) -> Vec<i32>,
{
    strategy(data)
}
```

## 二、具体用法

### 2.1 重试机制

```rust
fn with_retry<F, T, E>(
    mut op: F,
    max_attempts: usize,
    delay: std::time::Duration,
) -> Result<T, E>
where
    F: FnMut() -> Result<T, E>,
{
    let mut last_error = None;
    for attempt in 1..=max_attempts {
        match op() {
            Ok(value) => return Ok(value),
            Err(e) => {
                last_error = Some(e);
                if attempt < max_attempts {
                    std::thread::sleep(delay);
                }
            }
        }
    }
    Err(last_error.unwrap())
}
```

### 2.2 事件处理器

```rust
struct EventEmitter {
    handlers: std::collections::HashMap<String, Vec<Box<dyn Fn(&str)>>>,
}

impl EventEmitter {
    fn new() -> Self {
        EventEmitter { handlers: std::collections::HashMap::new() }
    }

    fn on<F: Fn(&str) + 'static>(&mut self, event: &str, handler: F) {
        self.handlers
            .entry(event.to_string())
            .or_default()
            .push(Box::new(handler));
    }

    fn emit(&self, event: &str, data: &str) {
        if let Some(handlers) = self.handlers.get(event) {
            for handler in handlers {
                handler(data);
            }
        }
    }
}
```

### 2.3 中间件模式

```rust
type Middleware = Box<dyn Fn(&str, &mut Vec<u8>) -> bool>;

struct Pipeline {
    middlewares: Vec<Middleware>,
}

impl Pipeline {
    fn new() -> Self {
        Pipeline { middlewares: Vec::new() }
    }

    fn add<F: Fn(&str, &mut Vec<u8>) -> bool + 'static>(&mut self, mw: F) {
        self.middlewares.push(Box::new(mw));
    }

    fn execute(&self, request: &str) -> Vec<u8> {
        let mut response = Vec::new();
        for mw in &self.middlewares {
            if !mw(request, &mut response) {
                break;
            }
        }
        response
    }
}
```

## 三、注意事项与常见陷阱

1. **生命周期管理**：回调函数需注意被引用数据的生命周期
2. **内存循环引用**：事件处理器可能造成循环引用
3. **性能开销**：动态分发（dyn Fn）有虚函数调用开销
4. **闭包大小**：大的闭包考虑使用 Box 包装
5. **可测试性**：闭包可以轻松传入 mock 实现
