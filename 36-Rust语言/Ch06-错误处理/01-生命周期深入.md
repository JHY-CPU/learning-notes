# 生命周期深入

## 一、概念说明

生命周期（Lifetime）是 Rust 确保引用有效性的机制。编译器通过生命周期检查，防止悬垂引用。

```rust
// 生命周期标注语法
// 'a 读作 "lifetime a"
// 标注说明引用之间的生命周期关系

fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}
```

## 二、生命周期标注

```rust
// 函数中的生命周期
fn first_word<'a>(s: &'a str) -> &'a str {
    let bytes = s.as_bytes();
    for (i, &byte) in bytes.iter().enumerate() {
        if byte == b' ' {
            return &s[0..i];
        }
    }
    &s[..]
}

// 结构体中的引用需要生命周期
struct Excerpt<'a> {
    part: &'a str,
}

impl<'a> Excerpt<'a> {
    fn announce(&self) -> &str {  // 生命周期省略
        "我是一个摘录"
    }

    fn level(&self) -> i32 {
        3
    }
}

fn main() {
    let novel = String::from("从前有座山。山上有座庙。");
    let first_sentence = novel.split('。').next().expect("找不到句子");
    let excerpt = Excerpt {
        part: first_sentence,
    };
    println!("摘录: {}", excerpt.part);
}
```

## 三、生命周期省略规则

```rust
// 编译器自动推断生命周期的规则
/*
1. 每个引用参数获得自己的生命周期
2. 如果只有一个输入生命周期，它被赋予所有输出生命周期
3. 如果有 &self 或 &mut self，self 的生命周期被赋予所有输出生命周期
*/

// 下面两个函数等价
fn first_word(s: &str) -> &str { /* ... */ }
fn first_word<'a>(s: &'a str) -> &'a str { /* ... */ }

// 方法中的 self 生命周期
impl<'a> Excerpt<'a> {
    // 返回值的生命周期与 self 相同
    fn part(&self) -> &str {
        self.part
    }
    // 等价于
    // fn part<'b>(&'b self) -> &'b str
}
```

## 四、静态生命周期

```rust
// 'static - 引用在程序整个运行期间有效
let s: &'static str = "我有静态生命周期";

// 所有字符串字面量都是 'static
fn get_static_str() -> &'static str {
    "静态字符串"
}

// 在 trait bound 中使用 'static
fn print_it<T: std::fmt::Display + 'static>(input: &T) {
    println!("{}", input);
}
```

## 五、生命周期与泛型

```rust
use std::fmt::Display;

// 生命周期和泛型结合
fn longest_with_an_announcement<'a, T>(
    x: &'a str,
    y: &'a str,
    ann: T,
) -> &'a str
where
    T: Display,
{
    println!("公告: {}", ann);
    if x.len() > y.len() { x } else { y }
}

// 多个生命周期
fn first_and_second<'a, 'b>(
    first: &'a str,
    _second: &'b str,
) -> &'a str {
    first
}

// 生命周期约束
fn parse_and_ref<'a, T>(input: &'a str, default: &'a T) -> &'a T
where
    T: std::str::FromStr,
{
    match input.parse::<T>() {
        Ok(val) => Box::leak(Box::new(val)),
        Err(_) => default,
    }
}
```

## 六、注意事项

1. **生命周期不改变引用的存活时间**：只是告诉编译器引用之间的关系
2. **编译器推断**：大多数情况下编译器能自动推断生命周期
3. **悬垂引用防止**：生命周期确保引用不会指向已释放的内存
4. **结构体中的引用**：包含引用的结构体必须标注生命周期
5. **生命周期子类型**：长生命周期可以替代短生命周期
