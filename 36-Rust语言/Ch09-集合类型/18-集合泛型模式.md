# 集合泛型模式

## 一、概念说明

Rust 的泛型系统与集合类型紧密结合，允许编写适用于多种集合的通用代码。通过 trait bounds 约束泛型类型，可以在编译时保证类型安全和性能。

```rust
// 泛型函数适用于所有实现 IntoIterator 的集合
fn sum_all<I>(iter: I) -> i32
where
    I: IntoIterator<Item = i32>,
{
    iter.into_iter().sum()
}
```

## 二、具体用法

### 2.1 泛型集合操作

```rust
use std::collections::{HashMap, BTreeMap};

// 泛型查找函数
fn find_key<K, V, F>(map: &HashMap<K, V>, predicate: F) -> Option<(&K, &V)>
where
    F: Fn(&K, &V) -> bool,
{
    map.iter().find(|(k, v)| predicate(k, v))
}

// 泛型转换函数
fn convert_map<K1, K2, V1, V2>(
    input: HashMap<K1, V1>,
    key_fn: impl Fn(K1) -> K2,
    val_fn: impl Fn(V1) -> V2,
) -> HashMap<K2, V2> {
    input.into_iter()
        .map(|(k, v)| (key_fn(k), val_fn(v)))
        .collect()
}
```

### 2.2 集合作为 trait bound

```rust
use std::hash::Hash;

// 要求集合类型支持去重
fn deduplicate<T, C>(collection: C) -> Vec<T>
where
    T: Eq + Hash + Clone,
    C: IntoIterator<Item = T>,
{
    use std::collections::HashSet;
    let mut seen = HashSet::new();
    collection.into_iter()
        .filter(|item| seen.insert(item.clone()))
        .collect()
}
```

### 2.3 实现通用容器

```rust
struct TypedMap {
    stores: std::collections::HashMap<std::any::TypeId, Box<dyn std::any::Any>>,
}

impl TypedMap {
    fn new() -> Self {
        TypedMap { stores: HashMap::new() }
    }

    fn insert<T: 'static>(&mut self, value: T) {
        self.stores.insert(
            std::any::TypeId::of::<T>(),
            Box::new(value),
        );
    }

    fn get<T: 'static>(&self) -> Option<&T> {
        self.stores.get(&std::any::TypeId::of::<T>())
            .and_then(|b| b.downcast_ref())
    }
}
```

## 三、注意事项与常见陷阱

1. **trait bound 简化**：使用 `impl Trait` 简化复杂的 where 子句
2. **生命周期约束**：涉及引用时需添加生命周期参数
3. **单态化开销**：泛型在编译时展开，可能导致二进制膨胀
4. **动态分发**：需要动态类型时使用 `dyn Trait` 替代泛型
5. **关联类型**：复杂泛型场景考虑使用关联类型简化
