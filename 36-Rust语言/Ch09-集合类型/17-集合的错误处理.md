# 集合的错误处理

## 一、概念说明

Rust 集合操作中的错误处理涉及越界访问、类型转换失败、容量限制等场景。标准库通过 `Option` 和 `Result` 类型提供安全的错误处理机制。

```rust
let v = vec![1, 2, 3];

// 安全访问返回 Option
let val: Option<&i32> = v.get(10); // None

// 不安全访问会 panic
// let val: &i32 = &v[10]; // panic!
```

## 二、具体用法

### 2.1 安全索引访问

```rust
fn safe_get<T>(vec: &[T], index: usize) -> Result<&T, String> {
    vec.get(index)
        .ok_or_else(|| format!("索引 {} 越界，长度为 {}", index, vec.len()))
}

fn safe_get_mut<T>(vec: &mut Vec<T>, index: usize) -> Result<&mut T, String> {
    let len = vec.len();
    vec.get_mut(index)
        .ok_or_else(|| format!("索引 {} 越界，长度为 {}", index, len))
}
```

### 2.2 Map 操作的错误处理

```rust
use std::collections::HashMap;

fn get_or_default<K, V: Clone>(
    map: &HashMap<K, V>,
    key: &K,
    default: V,
) -> V {
    map.get(key).cloned().unwrap_or(default)
}

fn get_or_insert<K: Eq + std::hash::Hash + Clone, V: Clone>(
    map: &mut HashMap<K, V>,
    key: K,
    default: V,
) -> V {
    if let Some(val) = map.get(&key) {
        val.clone()
    } else {
        map.insert(key.clone(), default.clone());
        default
    }
}
```

### 2.3 批量操作的错误收集

```rust
fn parse_all(input: &[&str]) -> Vec<Result<i32, std::num::ParseIntError>> {
    input.iter().map(|s| s.parse::<i32>()).collect()
}

fn collect_successes(results: Vec<Result<i32, std::num::ParseIntError>>) -> Vec<i32> {
    results.into_iter().filter_map(Result::ok).collect()
}

fn collect_errors(results: Vec<Result<i32, std::num::ParseIntError>>) -> Vec<std::num::ParseIntError> {
    results.into_iter().filter_map(Result::err).collect()
}
```

## 三、注意事项与常见陷阱

1. **Option vs Result**：元素不存在用 Option，操作失败用 Result
2. **unwrap 风险**：避免在生产代码中使用 unwrap
3. **错误传播**：使用 `?` 操作符简化错误处理
4. **自定义错误**：实现 `std::error::Error` trait 定义领域错误
5. **anyhow/thiserror**：推荐使用这些 crate 简化错误处理
