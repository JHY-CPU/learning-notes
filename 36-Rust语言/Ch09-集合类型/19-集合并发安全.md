# 集合并发安全

## 一、概念说明

标准库集合类型默认不是线程安全的。要在并发场景中使用，需要配合 `Arc<Mutex<T>>` 或使用并发安全的集合库如 `DashMap`。Rust 的所有权系统在编译时防止数据竞争。

```rust
use std::sync::{Arc, Mutex};
use std::collections::HashMap;

// 线程安全的 HashMap
let shared: Arc<Mutex<HashMap<String, i32>>> =
    Arc::new(Mutex::new(HashMap::new()));
```

## 二、具体用法

### 2.1 Arc + Mutex 包装

```rust
use std::sync::{Arc, Mutex};
use std::collections::HashMap;
use std::thread;

fn concurrent_counter() {
    let counter: Arc<Mutex<HashMap<String, i32>>> =
        Arc::new(Mutex::new(HashMap::new()));

    let mut handles = vec![];

    for i in 0..4 {
        let counter = Arc::clone(&counter);
        let handle = thread::spawn(move || {
            let mut map = counter.lock().unwrap();
            *map.entry(format!("thread_{}", i)).or_insert(0) += 1;
        });
        handles.push(handle);
    }

    for h in handles {
        h.join().unwrap();
    }

    println!("结果: {:?}", *counter.lock().unwrap());
}
```

### 2.2 RwLock 读写锁

```rust
use std::sync::{Arc, RwLock};
use std::collections::HashMap;

fn concurrent_reader_writer() {
    let data: Arc<RwLock<HashMap<String, String>>> =
        Arc::new(RwLock::new(HashMap::new()));

    // 多个线程可以同时读取
    let data_clone = Arc::clone(&data);
    let reader = thread::spawn(move || {
        let map = data_clone.read().unwrap();
        map.get("key").cloned()
    });

    // 写入时独占访问
    let data_clone = Arc::clone(&data);
    let writer = thread::spawn(move || {
        let mut map = data_clone.write().unwrap();
        map.insert("key".to_string(), "value".to_string());
    });

    writer.join().unwrap();
    let result = reader.join().unwrap();
}
```

### 2.3 DashMap 并发哈希映射

```rust
// Cargo.toml: dashmap = "5"
use dashmap::DashMap;
use std::sync::Arc;

fn dashmap_example() {
    let map = Arc::new(DashMap::new());
    let mut handles = vec![];

    for i in 0..4 {
        let map = Arc::clone(&map);
        handles.push(thread::spawn(move || {
            // 无需显式锁定，内部处理并发
            map.insert(i, i * 10);
            let val = map.get(&i);
            println!("线程{}读取: {:?}", i, val);
        }));
    }

    for h in handles {
        h.join().unwrap();
    }
}
```

## 三、注意事项与常见陷阱

1. **死锁风险**：避免在持有锁时获取另一个锁
2. **锁粒度**：使用细粒度锁减少竞争
3. **性能开销**：锁操作有性能代价，高并发考虑无锁结构
4. **Arc 克隆**：Arc::clone 只增加引用计数，不复制数据
5. **Send + Sync**：并发集合元素需实现 Send + Sync trait
