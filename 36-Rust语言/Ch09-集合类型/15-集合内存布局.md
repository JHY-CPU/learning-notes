# 集合内存布局

## 一、概念说明

理解集合的内存布局有助于选择合适的数据结构和优化性能。Rust 集合类型在栈和堆上存储数据的方式不同。

```rust
// Vec 内存布局: 栈上存指针/长度/容量，数据在堆上
// [ptr | len | cap]  →  堆: [T, T, T, T, ...]

// HashMap 内存布局: 哈希桶数组，每个桶存储键值对
// [bucket0 | bucket1 | ... | bucketN]  →  每个桶: Option<(K, V)>
```

## 二、具体用法

### 2.1 标准库集合的内存开销

```rust
use std::mem;

fn analyze_memory() {
    // Vec: 24字节 (ptr 8 + len 8 + cap 8)
    println!("Vec<i32>: {} bytes", mem::size_of::<Vec<i32>>());

    // String: 24字节 (同 Vec<u8>)
    println!("String: {} bytes", mem::size_of::<String>());

    // HashMap: 包含多个字段
    println!("HashMap: {} bytes", mem::size_of::<std::collections::HashMap<i32, i32>>());

    // Box: 8字节 (一个指针)
    println!("Box<i32>: {} bytes", mem::size_of::<Box<i32>>());

    // Rc: 8字节 (一个指针)
    println!("Rc<i32>: {} bytes", mem::size_of::<std::rc::Rc<i32>>());
}
```

### 2.2 自定义结构体的内存对齐

```rust
#[repr(C)]
struct Example {
    a: u8,    // 1字节
    // 3字节填充
    b: u32,   // 4字节
    c: u8,    // 1字节
    // 7字节填充
    d: u64,   // 8字节
}
// 总共: 24字节 (考虑8字节对齐)

fn check_layout() {
    use std::mem;
    println!("Example size: {}", mem::size_of::<Example>()); // 24
    println!("align: {}", mem::align_of::<Example>());       // 8
}
```

### 2.3 数据局部性分析

```rust
// Vec<T>: 数据完全连续 → 缓存命中率高
// LinkedList<T>: 节点分散在堆上 → 缓存命中率低
// HashMap: 哈希桶连续，但键值对可能在堆上

fn locality_analysis() {
    // 连续内存访问比随机访问快得多
    let vec: Vec<i64> = (0..1000).collect();
    let mut sum = 0;

    // 顺序遍历 - 缓存友好
    for x in &vec {
        sum += x;
    }

    // 随机访问 - 缓存不友好
    use rand::Rng;
    let mut rng = rand::thread_rng();
    for _ in 0..1000 {
        let idx = rng.gen_range(0..1000);
        sum += vec[idx];
    }
}
```

## 三、注意事项与常见陷阱

1. **内存对齐**：Rust 按最大字段对齐，可能导致填充字节
2. **指针开销**：Box/Rc/Arc 都有 8 字节指针开销
3. **间接寻址**：堆分配增加内存访问层级
4. **缓存行**：64字节缓存行影响性能
5. **堆碎片**：频繁分配/释放可能导致内存碎片
