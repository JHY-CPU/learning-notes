# 智能指针选择指南

## 一、概念说明

根据不同的所有权、线程安全和可变性需求，选择合适的智能指针。这是 Rust 中最重要的设计决策之一。

```rust
// 决策树：
// 1. 唯一所有权？ → Box
// 2. 共享所有权？
//    a. 单线程？ → Rc + RefCell/Cell
//    b. 多线程？ → Arc + Mutex/RwLock
// 3. 内部可变性？
//    a. Copy类型？ → Cell
//    b. 任意类型？ → RefCell
```

## 二、具体用法

### 2.1 所有权模型

```rust
// 唯一所有权
let unique = Box::new(vec![1, 2, 3]); // 只有 Box 拥有数据

// 共享所有权（单线程）
use std::rc::Rc;
let shared = Rc::new(vec![1, 2, 3]);
let clone1 = Rc::clone(&shared);
let clone2 = Rc::clone(&shared);

// 共享所有权（多线程）
use std::sync::Arc;
let shared = Arc::new(vec![1, 2, 3]);
```

### 2.2 可变性模型

```rust
use std::cell::{Cell, RefCell};

// Cell: Copy类型，直接替换
let cell = Cell::new(5);
cell.set(10);

// RefCell: 任意类型，运行时检查
let refcell = RefCell::new(vec![1, 2, 3]);
refcell.borrow_mut().push(4);

// Mutex: 线程安全的 RefCell
use std::sync::Mutex;
let mutex = Mutex::new(vec![1, 2, 3]);
mutex.lock().unwrap().push(4);

// RwLock: 允许多个读者
use std::sync::RwLock;
let rwlock = RwLock::new(vec![1, 2, 3]);
let reader = rwlock.read().unwrap();
```

### 2.3 组合使用

```rust
use std::rc::{Rc, Weak};
use std::cell::RefCell;
use std::sync::{Arc, Mutex};

// 单线程共享可变
type SharedMut<T> = Rc<RefCell<T>>;

// 多线程共享可变
type ThreadSafe<T> = Arc<Mutex<T>>;

// 带父引用的树
struct TreeNode {
    value: i32,
    parent: Option<Weak<RefCell<TreeNode>>>,
    children: Vec<Rc<RefCell<TreeNode>>>,
}
```

## 三、注意事项与常见陷阱

1. **性能优先**：Box > Cell > RefCell > Rc > Mutex > Arc
2. **安全优先**：编译时检查 > 运行时检查
3. **线程需求**：单线程用 Rc，多线程用 Arc
4. **循环引用**：使用 Weak 打破循环
5. **最小化智能指针**：能用引用就不用智能指针
