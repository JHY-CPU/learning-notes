# 智能指针性能对比

## 一、概念说明

不同智能指针有不同的性能特征和内存开销。选择合适的智能指针对性能至关重要。

```rust
use std::mem;

// 各智能指针的大小
println!("Box<i32>: {} bytes", mem::size_of::<Box<i32>>());      // 8
println!("Rc<i32>: {} bytes", mem::size_of::<Rc<i32>>());        // 8
println!("Arc<i32>: {} bytes", mem::size_of::<Arc<i32>>());      // 8
println!("RefCell<i32>: {} bytes", mem::size_of::<RefCell<i32>>()); // 8
println!("Mutex<i32>: {} bytes", mem::size_of::<Mutex<i32>>());  // 24
```

## 二、具体用法

### 2.1 基准测试

```rust
use std::sync::{Arc, Mutex};
use std::rc::Rc;
use std::cell::RefCell;
use std::time::Instant;

fn benchmark_box() {
    let start = Instant::now();
    let data = Box::new(vec![1; 1000000]);
    let _sum: i32 = data.iter().sum();
    println!("Box: {:?}", start.elapsed());
}

fn benchmark_rc_refcell() {
    let start = Instant::now();
    let data = Rc::new(RefCell::new(vec![1; 1000000]));
    let _sum: i32 = data.borrow().iter().sum();
    println!("Rc+RefCell: {:?}", start.elapsed());
}

fn benchmark_arc_mutex() {
    let start = Instant::now();
    let data = Arc::new(Mutex::new(vec![1; 1000000]));
    let _sum: i32 = data.lock().unwrap().iter().sum();
    println!("Arc+Mutex: {:?}", start.elapsed());
}
```

### 2.2 并发性能

```rust
use std::sync::{Arc, Mutex};
use std::thread;

fn concurrent_benchmark() {
    let data = Arc::new(Mutex::new(0u64));
    let start = Instant::now();

    let handles: Vec<_> = (0..8).map(|_| {
        let data = Arc::clone(&data);
        thread::spawn(move || {
            for _ in 0..100000 {
                let mut num = data.lock().unwrap();
                *num += 1;
            }
        })
    }).collect();

    for h in handles { h.join().unwrap(); }
    println!("Arc+Mutex (8线程): {:?}", start.elapsed());
}
```

## 三、注意事项与常见陷阱

1. **开销排序**：Box < Cell < RefCell < Rc < Mutex < Arc < RwLock
2. **原子操作**：Arc 的原子引用计数有 CPU 缓存同步开销
3. **锁竞争**：高并发下锁的性能下降明显
4. **缓存友好**：Box 连续内存访问性能最好
5. **选择原则**：优先使用最简单满足需求的智能指针
