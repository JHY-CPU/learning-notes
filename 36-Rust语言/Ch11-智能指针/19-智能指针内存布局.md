# 智能指针内存布局

## 一、概念说明

了解智能指针的内存布局有助于理解其性能特征和调试内存问题。不同的智能指针在栈和堆上存储不同的元数据。

```rust
use std::mem;
use std::rc::Rc;
use std::sync::Arc;

// 栈上布局（64位系统）
println!("Box: {} bytes", mem::size_of::<Box<i32>>());       // 8
println!("Rc: {} bytes", mem::size_of::<Rc<i32>>());         // 8
println!("Arc: {} bytes", mem::size_of::<Arc<i32>>());       // 8
println!("RefCell: {} bytes", mem::size_of::<RefCell<i32>>()); // 8
println!("Mutex: {} bytes", mem::size_of::<Mutex<i32>>());   // 24
```

## 二、具体用法

### 2.1 Box 内存布局

```rust
// Box<T>:
// 栈: [ptr (8字节)]
// 堆: [T 的数据]
//
// 最简单的布局，只有一个指针

let b = Box::new(42i32);
println!("Box 地址: {:p}", &b);       // 栈地址
println!("数据地址: {:p}", &*b);      // 堆地址
```

### 2.2 Rc 内存布局

```rust
use std::rc::Rc;

// Rc<T>:
// 栈: [ptr (8字节)]
// 堆: [strong_count | weak_count | T 的数据]
//
// 引用计数存储在堆上

let r = Rc::new(42i32);
println!("强引用: {}", Rc::strong_count(&r));
println!("弱引用: {}", Rc::weak_count(&r));
```

### 2.3 Arc 内存布局

```rust
use std::sync::Arc;

// Arc<T>:
// 栈: [ptr (8字节)]
// 堆: [strong_count(atomic) | weak_count(atomic) | T 的数据]
//
// 使用原子操作的引用计数

let a = Arc::new(42i32);
// 原子操作保证线程安全
```

### 2.4 Mutex 内存布局

```rust
use std::sync::Mutex;

// Mutex<T>:
// 栈: [os_mutex | poison_flag | data ...]
//
// 包含操作系统级别的互斥锁

let m = Mutex::new(42i32);
println!("Mutex 大小: {} bytes", std::mem::size_of_val(&m));
```

## 三、注意事项与常见陷阱

1. **指针大小**：64位系统上指针都是8字节
2. **元数据开销**：引用计数等元数据在堆上
3. **对齐要求**：内存地址需要满足类型的对齐要求
4. **缓存行**：原子操作涉及缓存行同步
5. **内存碎片**：频繁堆分配可能导致内存碎片
