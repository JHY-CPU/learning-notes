# Rc引用计数

## 一、概念说明

`Rc<T>`（Reference Counted）提供了共享所有权的智能指针。多个 Rc 可以指向同一块数据，当最后一个 Rc 被销毁时，数据才会被释放。

```rust
use std::rc::Rc;

let a = Rc::new(5);
let b = Rc::clone(&a); // 引用计数增加
println!("引用计数: {}", Rc::strong_count(&a)); // 2
```

## 二、具体用法

### 2.1 共享数据

```rust
use std::rc::Rc;

// 多个所有者共享数据
let shared = Rc::new(vec![1, 2, 3, 4, 5]);
let a = Rc::clone(&shared);
let b = Rc::clone(&shared);

// 三个 Rc 指向同一块数据
println!("shared: {:?}", shared);
println!("a: {:?}", a);
println!("b: {:?}", b);
println!("引用计数: {}", Rc::strong_count(&shared)); // 3
```

### 2.2 图结构

```rust
use std::rc::Rc;
use std::cell::RefCell;

#[derive(Debug)]
struct Node {
    value: i32,
    neighbors: Vec<Rc<RefCell<Node>>>,
}

impl Node {
    fn new(value: i32) -> Rc<RefCell<Self>> {
        Rc::new(RefCell::new(Node {
            value,
            neighbors: Vec::new(),
        }))
    }

    fn add_neighbor(node: &Rc<RefCell<Node>>, neighbor: &Rc<RefCell<Node>>) {
        node.borrow_mut().neighbors.push(Rc::clone(neighbor));
    }
}
```

### 2.3 Weak 引用避免循环

```rust
use std::rc::{Rc, Weak};
use std::cell::RefCell;

struct Parent {
    children: Vec<Rc<RefCell<Child>>>,
}

struct Child {
    parent: Weak<RefCell<Parent>>, // Weak 引用
}

// Weak 不增加强引用计数，可以被升级为 Rc
let parent = Rc::new(RefCell::new(Parent { children: vec![] }));
let child = Rc::new(RefCell::new(Child {
    parent: Rc::downgrade(&parent),
}));
```

### 2.4 Rc 与 RefCell 组合

```rust
use std::rc::Rc;
use std::cell::RefCell;

// 共享可变状态
fn shared_mutable_state() {
    let shared = Rc::new(RefCell::new(vec![1, 2, 3]));

    let a = Rc::clone(&shared);
    let b = Rc::clone(&shared);

    // a 可以修改数据
    a.borrow_mut().push(4);

    // b 可以看到修改
    println!("b 看到的数据: {:?}", b.borrow());
    // [1, 2, 3, 4]

    // 引用计数
    println!("引用计数: {}", Rc::strong_count(&shared)); // 3
}
```

### 2.5 Rc::make_mut 写时复制

```rust
use std::rc::Rc;

fn make_mut_example() {
    let mut data = Rc::new(vec![1, 2, 3]);

    // 引用计数为1时，直接修改
    Rc::make_mut(&mut data).push(4);

    let clone = Rc::clone(&data);
    println!("共享数据: {:?}", clone);

    // 引用计数>1时，clone 数据再修改（写时复制）
    Rc::make_mut(&mut data).push(5);
    // data 现在指向新的 Vec，clone 仍指向旧的
    println!("data: {:?}", data);    // [1, 2, 3, 4, 5]
    println!("clone: {:?}", clone);  // [1, 2, 3, 4]
}
```

### 2.6 Rc 的内存开销分析

```rust
use std::rc::Rc;
use std::mem;

fn memory_analysis() {
    let rc = Rc::new(String::from("hello"));

    // Rc 本身是一个指针大小
    println!("Rc 大小: {} bytes", mem::size_of::<Rc<String>>());
    // 通常 8 bytes (64位系统)

    // 堆上的实际布局：
    // [strong_count: usize] [weak_count: usize] [T: String]
    // 8 bytes               8 bytes              (数据)

    // 查看引用计数
    println!("strong: {}, weak: {}",
        Rc::strong_count(&rc),
        Rc::weak_count(&rc));
}
```

### 2.7 Rc 用于缓存共享

```rust
use std::rc::Rc;
use std::collections::HashMap;

struct Cache {
    store: HashMap<String, Rc<String>>,
}

impl Cache {
    fn new() -> Self {
        Cache { store: HashMap::new() }
    }

    fn get_or_insert(&mut self, key: &str, value_fn: impl FnOnce() -> String) -> Rc<String> {
        if let Some(v) = self.store.get(key) {
            Rc::clone(v)
        } else {
            let value = Rc::new(value_fn());
            self.store.insert(key.to_string(), Rc::clone(&value));
            value
        }
    }
}

fn cache_example() {
    let mut cache = Cache::new();

    let v1 = cache.get_or_insert("key1", || "计算结果".to_string());
    let v2 = cache.get_or_insert("key1", || "不会执行".to_string());

    // v1 和 v2 指向同一个 Rc
    assert!(Rc::ptr_eq(&v1, &v2));
    println!("引用计数: {}", Rc::strong_count(&v1)); // 3
}
```

### 2.8 有向图的构建与遍历

```rust
use std::rc::{Rc, Weak};
use std::cell::RefCell;
use std::collections::{HashMap, HashSet};

#[derive(Debug)]
struct Graph {
    nodes: HashMap<String, Rc<RefCell<Node>>>,
}

#[derive(Debug)]
struct Node {
    name: String,
    neighbors: Vec<Weak<RefCell<Node>>>,
}

impl Graph {
    fn new() -> Self {
        Graph { nodes: HashMap::new() }
    }

    fn add_node(&mut self, name: &str) {
        self.nodes.insert(name.to_string(), Rc::new(RefCell::new(Node {
            name: name.to_string(),
            neighbors: Vec::new(),
        })));
    }

    fn add_edge(&mut self, from: &str, to: &str) {
        if let (Some(f), Some(t)) = (self.nodes.get(from), self.nodes.get(to)) {
            f.borrow_mut().neighbors.push(Rc::downgrade(t));
        }
    }

    fn dfs(&self, start: &str) -> Vec<String> {
        let mut visited = HashSet::new();
        let mut result = Vec::new();

        if let Some(node) = self.nodes.get(start) {
            self.dfs_visit(node, &mut visited, &mut result);
        }

        result
    }

    fn dfs_visit(&self, node: &Rc<RefCell<Node>>, visited: &mut HashSet<String>, result: &mut Vec<String>) {
        let name = node.borrow().name.clone();
        if visited.contains(&name) {
            return;
        }
        visited.insert(name.clone());
        result.push(name);

        let neighbors: Vec<_> = node.borrow().neighbors.iter()
            .filter_map(|w| w.upgrade())
            .collect();

        for neighbor in neighbors {
            self.dfs_visit(&neighbor, visited, result);
        }
    }
}
```

## 四、Rc 与 Arc 的选择

| 特性 | Rc | Arc |
|------|-----|-----|
| 线程安全 | 否 | 是 |
| 性能开销 | 低（非原子） | 略高（原子操作） |
| 适用场景 | 单线程共享 | 多线程共享 |
| 原子操作 | 无 | 有 |

## 五、注意事项与常见陷阱

1. **单线程**：Rc 不是线程安全的，多线程用 Arc，编译器会检查 Send + Sync
2. **不可变**：Rc 内部数据默认不可变，需配合 RefCell 实现可变性
3. **循环引用**：两个 Rc 互相引用会内存泄漏，父节点用 Weak 引用
4. **Weak 引用**：使用 Weak 打破循环引用，Weak 需要 `upgrade()` 才能访问数据
5. **性能开销**：引用计数操作有少量开销，但在单线程中比 Arc 的原子操作更便宜
6. **Clone 语义**：Rc::clone 只增加引用计数，不复制数据（浅拷贝）
7. **Drop 时机**：最后一个强引用 drop 时释放数据，此时所有 Weak 引用 upgrade 会失败
