# 循环引用问题

## 一、概念说明

当两个或多个智能指针互相引用时，会产生循环引用，导致内存无法释放。Rust 通过 `Weak` 引用解决这个问题。

```rust
use std::rc::{Rc, Weak};
use std::cell::RefCell;

// 问题：循环引用
let a = Rc::new(RefCell::new(None));
let b = Rc::new(RefCell::new(Some(Rc::clone(&a))));
*a.borrow_mut() = Some(Rc::clone(&b));
// a 和 b 互相引用，不会被释放
```

## 二、具体用法

### 2.1 检测循环引用

```rust
use std::rc::Rc;
use std::cell::RefCell;

#[derive(Debug)]
struct Node {
    name: String,
    next: RefCell<Option<Rc<Node>>>,
}

impl Drop for Node {
    fn drop(&mut self) {
        println!("释放节点: {}", self.name);
    }
}

fn create_cycle() {
    let a = Rc::new(Node { name: "A".into(), next: RefCell::new(None) });
    let b = Rc::new(Node { name: "B".into(), next: RefCell::new(Some(Rc::clone(&a))) });
    *a.next.borrow_mut() = Some(Rc::clone(&b));
    // 离开作用域时不会释放！
}
```

### 2.2 使用 Weak 打破循环

```rust
use std::rc::{Rc, Weak};
use std::cell::RefCell;

struct Node {
    name: String,
    next: RefCell<Option<Weak<Node>>>, // 使用 Weak
}

impl Node {
    fn new(name: &str) -> Rc<Self> {
        Rc::new(Node {
            name: name.to_string(),
            next: RefCell::new(None),
        })
    }

    fn set_next(node: &Rc<Self>, next: &Rc<Self>) {
        *node.next.borrow_mut() = Some(Rc::downgrade(next));
    }
}

impl Drop for Node {
    fn drop(&mut self) {
        println!("释放: {}", self.name);
    }
}

fn safe_references() {
    let a = Node::new("A");
    let b = Node::new("B");
    Node::set_next(&a, &b);
    Node::set_next(&b, &a);
    // 正确释放，不会泄漏
}
```

### 2.3 双向链表实现

```rust
use std::rc::{Rc, Weak};
use std::cell::RefCell;

struct ListItem {
    value: i32,
    next: RefCell<Option<Rc<ListItem>>>,
    prev: RefCell<Option<Weak<ListItem>>>, // Weak 避免循环
}

impl Drop for ListItem {
    fn drop(&mut self) {
        println!("释放: {}", self.value);
    }
}
```

## 三、注意事项与常见陷阱

1. **调试困难**：循环引用导致的内存泄漏难以调试
2. **弱引用选择**：在父子/双向关系中，子对父使用 Weak
3. **工具检测**：使用 valgrind 或 heaptrack 检测泄漏
4. **设计原则**：尽量避免需要循环引用的数据结构
5. **定期清理**：定期清理无效的 Weak 引用
