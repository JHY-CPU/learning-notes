# 变量与可变性

## 一、概念说明

Rust 中变量默认是不可变的（immutable），这是 Rust 安全性设计的核心之一。使用 `mut` 关键字声明可变变量。

```rust
fn main() {
    // 不可变变量（默认）
    let x = 5;
    println!("x 的值: {}", x);
    // x = 6; // 错误！不能修改不可变变量

    // 可变变量
    let mut y = 5;
    println!("y 的值: {}", y);
    y = 6; // 正确
    println!("y 的新值: {}", y);
}
```

## 二、变量绑定

```rust
fn main() {
    // 基本绑定
    let a = 1;
    let b: i32 = 2;         // 显式类型标注
    let c = 3_i32;          // 类型后缀

    // 解构赋值
    let (x, y) = (1, 2);
    let Point { x, y } = Point { x: 1, y: 2 };

    // 类型推断
    let guess: u32 = "42".parse().expect("不是数字");
}
```

## 三、遮蔽（Shadowing）

```rust
fn main() {
    let x = 5;
    let x = x + 1;        // 遮蔽，创建新的 x
    let x = x * 2;        // 再次遮蔽
    println!("x = {}", x); // x = 12

    // 遮蔽可以改变类型
    let spaces = "   ";           // &str
    let spaces = spaces.len();    // usize（类型变了！）
    println!("空格数: {}", spaces);

    // let mut 不能改变类型
    let mut spaces = "   ";
    // spaces = spaces.len(); // 错误！不能改变类型
}
```

## 四、常量与静态变量

```rust
// 常量 - 编译时求值，必须标注类型
const MAX_POINTS: u32 = 100_000;
const PI: f64 = 3.14159265358979323846;

// 静态变量 - 运行时存在，有固定内存地址
static GLOBAL_COUNT: i32 = 0;
static mut MUTABLE_GLOBAL: i32 = 0; // 不安全

fn main() {
    println!("最大点数: {}", MAX_POINTS);
    println!("圆周率: {}", PI);
}
```

## 五、基本数据类型

```rust
fn main() {
    // 整数类型
    let a: i8 = -128;       // 有符号 8 位
    let b: u8 = 255;        // 无符号 8 位
    let c: i32 = 42;        // 有符号 32 位（默认）
    let d: u64 = 1000;      // 无符号 64 位
    let e: isize = 100;     // 指针大小
    let f: usize = 100;     // 无符号指针大小

    // 浮点类型
    let x: f64 = 3.14;      // 64 位（默认）
    let y: f32 = 2.718;     // 32 位

    // 布尔类型
    let t: bool = true;
    let f: bool = false;

    // 字符类型（4 字节 Unicode）
    let c: char = '中';
    let emoji: char = '😀';

    // 元组
    let tup: (i32, f64, u8) = (500, 6.4, 1);
    let (x, y, z) = tup;
    println!("第一个值: {}", tup.0);

    // 数组
    let arr: [i32; 5] = [1, 2, 3, 4, 5];
    let zeros = [0; 10];    // 10 个 0
    println!("第一个元素: {}", arr[0]);
}
```

## 六、注意事项

1. **默认不可变**：除非需要修改，否则使用不可变变量
2. **遮蔽 vs 可变**：遮蔽创建新变量，可变修改原变量
3. **类型标注**：通常不需要显式标注，编译器会推断
4. **整数溢出**：debug 模式会 panic，release 模式会环绕
5. **常量求值**：常量必须在编译时能求值
