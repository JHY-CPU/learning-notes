# 并发编程

## 一、概念说明

Rust 通过所有权系统和类型系统在编译时防止数据竞争，实现"无畏并发"。

```rust
// 并发编程方式
/*
1. 线程 (thread::spawn)
2. 消息传递 (channel)
3. 共享状态 (Arc + Mutex)
4. 异步编程 (async/await)
*/
```

## 二、线程基础

```rust
use std::thread;
use std::time::Duration;

fn main() {
    // 创建线程
    let handle = thread::spawn(|| {
        for i in 1..5 {
            println!("线程: {}", i);
            thread::sleep(Duration::from_millis(1));
        }
    });

    // 主线程
    for i in 1..3 {
        println!("主线程: {}", i);
    }

    // 等待线程完成
    handle.join().unwrap();

    // move 闭包
    let v = vec![1, 2, 3];
    let handle = thread::spawn(move || {
        println!("向量: {:?}", v);
    });
    handle.join().unwrap();
}
```

## 三、消息传递

```rust
use std::sync::mpsc;
use std::thread;

fn main() {
    // 创建通道
    let (tx, rx) = mpsc::channel();

    // 发送端
    let tx1 = tx.clone();  // 克隆发送端
    thread::spawn(move || {
        let messages = vec!["你好", "世界"];
        for msg in messages {
            tx1.send(msg).unwrap();
            thread::sleep(Duration::from_millis(100));
        }
    });

    thread::spawn(move || {
        tx.send("来自另一个线程").unwrap();
    });

    // 接收端
    for received in rx {
        println!("收到: {}", received);
    }
}
```

## 四、共享状态

```rust
use std::sync::{Arc, Mutex};
use std::thread;

fn main() {
    // Arc<Mutex<T>> - 线程安全的共享可变状态
    let data = Arc::new(Mutex::new(vec![1, 2, 3]));
    let mut handles = vec![];

    for i in 0..3 {
        let data = Arc::clone(&data);
        handles.push(thread::spawn(move || {
            let mut vec = data.lock().unwrap();
            vec.push(i);
        }));
    }

    for handle in handles {
        handle.join().unwrap();
    }

    println!("最终数据: {:?}", *data.lock().unwrap());
}
```

## 五、异步编程

```rust
use tokio;

#[tokio::main]
async fn main() {
    // 异步函数
    let result = fetch_data().await;
    println!("结果: {}", result);

    // 并发执行多个任务
    let (result1, result2) = tokio::join!(
        fetch_data(),
        fetch_another()
    );

    // 异步迭代
    let mut stream = tokio::stream::iter(vec![1, 2, 3]);
    while let Some(item) = stream.next().await {
        println!("项目: {}", item);
    }
}

async fn fetch_data() -> String {
    tokio::time::sleep(Duration::from_secs(1)).await;
    String::from("数据")
}

async fn fetch_another() -> String {
    tokio::time::sleep(Duration::from_secs(2)).await;
    String::from("其他数据")
}
```

## 六、Send 和 Sync Trait

```rust
// Send - 可以安全地在线程间转移所有权
// Sync - 可以安全地在线程间共享引用

// 大多数类型都实现了 Send 和 Sync
// Rc<T> 没有实现 Send（不是线程安全的）
// Arc<T> 实现了 Send（线程安全的）

fn is_send<T: Send>() {}
fn is_sync<T: Sync>() {}

fn main() {
    is_send::<i32>();
    is_sync::<String>();
    is_send::<Arc<Mutex<i32>>>();
}
```

## 七、注意事项

1. **数据竞争**：Rust 编译器在编译时防止数据竞争
2. **死锁**：Rust 不能防止死锁，需要开发者注意
3. **异步运行时**：需要引入 tokio 或 async-std 等运行时
4. **性能选择**：CPU 密集用线程，IO 密集用异步
5. **Send + Sync**：跨线程传递的类型必须实现 Send
