# 包与模块

## 一、概念说明

Rust 使用模块系统组织代码，包括包（Package）、Crate（板条箱）、模块（Module）和路径（Path）。

```rust
// 模块系统层次
/*
Package（包）: Cargo.toml 定义，包含一个或多个 crate
Crate（板条箱）: 编译的最小单位，分为 binary 和 library
Module（模块）: 组织代码的单元
Path（路径）: 访问项的方式
*/
```

## 二、定义模块

```rust
// 定义模块
mod front_of_house {
    pub mod hosting {
        pub fn add_to_waitlist() {
            println!("添加到等待列表");
        }

        fn seat_at_table() {
            // 私有函数
        }
    }

    mod serving {
        fn take_order() {}
        fn serve_order() {}
        fn take_payment() {}
    }
}

// 使用路径
use crate::front_of_house::hosting;

pub fn eat_at_restaurant() {
    // 绝对路径
    crate::front_of_house::hosting::add_to_waitlist();

    // 相对路径
    front_of_house::hosting::add_to_waitlist();

    // use 导入后
    hosting::add_to_waitlist();
}
```

## 三、可见性规则

```rust
// 默认私有
mod outer {
    // 私有模块
    mod inner {
        fn private_function() {
            println!("私有函数");
        }

        // 公开函数
        pub fn public_function() {
            println!("公开函数");
        }
    }

    // pub(crate) - crate 内可见
    pub(crate) fn crate_function() {}

    // pub(super) - 父模块可见
    pub(super) fn parent_function() {}
}

fn main() {
    outer::public_function();
    // outer::inner::private_function();  // 错误！私有
}
```

## 四、use 与 re-export

```rust
// use 导入
use std::collections::HashMap;
use std::io::{self, Read, Write};
use std::collections::*;  // glob 导入

// as 重命名
use std::io::Result as IoResult;

// 嵌套路径
use std::{cmp::Ordering, io};

// re-export
mod front_of_house {
    pub mod hosting {
        pub fn add_to_waitlist() {}
    }
}

pub use crate::front_of_house::hosting;
// 外部可以使用 crate::hosting::add_to_waitlist()
```

## 五、文件模块

```rust
// src/lib.rs
mod front_of_house;  // 声明模块，自动查找同名文件

// src/front_of_house.rs
pub mod hosting;

// src/front_of_house/hosting.rs
pub fn add_to_waitlist() {
    println!("添加到等待列表");
}
```

## 六、注意事项

1. **默认私有**：模块内的项默认对子模块和父模块私有
2. **pub 关键字**：使用 pub 公开项
3. **路径引用**：self、super、crate 用于路径引用
4. **文件组织**：模块可以放在单独的文件中
5. **use 最佳实践**：导入函数时导入父模块，导入其他项时导入完整路径
