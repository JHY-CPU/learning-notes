# 宏与日志

## 一、概念说明

Rust 的日志框架（log/tracing）大量使用宏实现零成本抽象和结构化日志。

```rust
use log::{info, warn, error, debug};

info!("用户 {} 登录成功", user_id);
error!("数据库连接失败: {}", error);
```

## 二、具体用法

### 2.1 log 宏分析

```rust
// log 宏在编译时检查日志级别
// 如果全局日志级别高于 info，info! 宏展开为空

macro_rules! my_info {
    ($($arg:tt)+) => {
        if log::log_enabled!(log::Level::Info) {
            log::log!(log::Level::Info, $($arg)+);
        }
    };
}
```

### 2.2 tracing 结构化日志

```rust
use tracing::{info, instrument};

#[instrument]
async fn process_request(id: u64) -> String {
    info!("开始处理请求");
    // 自动生成 span，包含函数参数
    format!("处理完成: {}", id)
}
```

### 2.3 自定义日志宏

```rust
macro_rules! timed_log {
    ($level:expr, $($arg:tt)+) => {{
        let start = std::time::Instant::now();
        let result = { $($arg)+ };
        log::log!($level, "执行耗时: {:?}", start.elapsed());
        result
    }};
}
```

### 2.4 日志级别控制

```rust
// 环境变量控制日志级别
// RUST_LOG=debug cargo run
// RUST_LOG=my_crate=info cargo run

use tracing_subscriber::EnvFilter;

fn init_logging() {
    tracing_subscriber::fmt()
        .with_env_filter(
            EnvFilter::try_from_default_env()
                .unwrap_or_else(|_| EnvFilter::new("info"))
        )
        .init();
}

// 代码中动态调整
use tracing::Level;

fn dynamic_level() {
    let level = if cfg!(debug_assertions) {
        Level::DEBUG
    } else {
        Level::INFO
    };

    tracing::info!(?level, "当前日志级别");
}
```

### 2.5 日志性能优化

```rust
use tracing::{info, debug, trace};

fn performance_aware_logging() {
    // 低级别日志在禁用时是零成本
    trace!("详细追踪信息: {:?}", expensive_computation());

    // 条件日志：只在启用时计算
    if tracing::enabled!(Level::DEBUG) {
        debug!("调试信息: {:?}", expensive_computation());
    }

    // 使用 span 而非每次记录
    let span = tracing::info_span!("processing", count = data.len());
    let _guard = span.enter();
    // 所有在 span 内的日志都带有 count 字段
}

fn expensive_computation() -> Vec<i32> {
    (0..1000).collect()
}
```

### 2.6 自定义 tracing Layer

```rust
use tracing_subscriber::Layer;
use tracing::{Event, Subscriber};
use std::fmt;

struct JsonLayer;

impl<S: Subscriber> Layer<S> for JsonLayer {
    fn on_event(&self, event: &Event<'_>, _ctx: tracing_subscriber::layer::Context<'_, S>) {
        let mut visitor = JsonVisitor::default();
        event.record(&mut visitor);
        println!("{{\"level\":\"{:?}\",\"fields\":{:?}}}",
            event.metadata().level(),
            visitor.fields);
    }
}

#[derive(Default)]
struct JsonVisitor {
    fields: std::collections::HashMap<String, String>,
}

impl tracing::field::Visit for JsonVisitor {
    fn record_debug(&mut self, field: &tracing::field::Field, value: &dyn fmt::Debug) {
        self.fields.insert(field.name().to_string(), format!("{:?}", value));
    }
}
```

## 四、log vs tracing 选择

| 特性 | log | tracing |
|------|-----|---------|
| 结构化日志 | 不支持 | 支持 |
| Span | 不支持 | 支持 |
| 异步支持 | 有限 | 完整 |
| 性能 | 好 | 更好（零成本） |
| 生态系统 | 成熟 | 快速发展 |
| 学习曲线 | 低 | 中 |

## 五、注意事项与常见陷阱

1. **零成本**：禁用日志级别时宏展开为空，无运行时开销
2. **格式化开销**：日志格式化只在日志启用时执行，使用条件检查避免
3. **结构化日志**：tracing 支持结构化字段，便于日志分析
4. **性能影响**：频繁的日志调用可能影响性能，使用采样和过滤
5. **日志级别**：合理使用日志级别，生产环境使用 INFO 或 WARN
6. **敏感信息**：不要记录密码、密钥等敏感信息
7. **日志轮转**：生产环境配置日志轮转，避免磁盘满
