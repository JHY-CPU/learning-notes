# 宏与日志

## 一、概念说明

Rust 的日志框架（log/tracing）大量使用宏实现零成本抽象和结构化日志。

```rust
use log::{info, warn, error, debug};

info!("用户 {} 登录成功", user_id);
error!("数据库连接失败: {}", error);
```

## 二、具体用法

### 2.1 log 宏分析

```rust
// log 宏在编译时检查日志级别
// 如果全局日志级别高于 info，info! 宏展开为空

macro_rules! my_info {
    ($($arg:tt)+) => {
        if log::log_enabled!(log::Level::Info) {
            log::log!(log::Level::Info, $($arg)+);
        }
    };
}
```

### 2.2 tracing 结构化日志

```rust
use tracing::{info, instrument};

#[instrument]
async fn process_request(id: u64) -> String {
    info!("开始处理请求");
    // 自动生成 span，包含函数参数
    format!("处理完成: {}", id)
}
```

### 2.3 自定义日志宏

```rust
macro_rules! timed_log {
    ($level:expr, $($arg:tt)+) => {{
        let start = std::time::Instant::now();
        let result = { $($arg)+ };
        log::log!($level, "执行耗时: {:?}", start.elapsed());
        result
    }};
}
```

## 三、注意事项与常见陷阱

1. **零成本**：禁用日志级别时宏展开为空
2. **格式化开销**：日志格式化只在日志启用时执行
3. **结构化日志**：tracing 支持结构化字段
4. **性能影响**：频繁的日志调用可能影响性能
5. **日志级别**：合理使用日志级别
