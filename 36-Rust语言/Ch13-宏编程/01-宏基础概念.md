# 宏基础概念

## 一、概念说明

宏（Macro）是 Rust 的元编程工具，在编译时展开代码。Rust 有声明式宏（macro_rules!）和过程宏两种类型。

```rust
// 声明式宏
macro_rules! say_hello {
    () => {
        println!("你好，世界！");
    };
}

// 过程宏（派生宏、属性宏、函数宏）
#[derive(Debug)]
struct MyStruct;
```

## 二、具体用法

### 2.1 声明式宏基础

```rust
macro_rules! create_function {
    ($name:ident) => {
        fn $name() {
            println!("函数 {:?} 被调用", stringify!($name));
        }
    };
}

create_function!(foo);
create_function!(bar);

fn main() {
    foo();
    bar();
}
```

### 2.2 宏匹配规则

```rust
macro_rules! match_example {
    // 匹配字面量
    (hello) => { println!("你好！"); };

    // 匹配表达式
    ($x:expr) => { println!("表达式: {}", $x); };

    // 匹配类型
    ($x:ty) => { println!("类型: {}", stringify!($x)); };

    // 匹配标识符
    ($x:ident) => { println!("标识符: {}", stringify!($x)); };

    // 匹配任意标记流
    ($($x:tt)*) => { println!("任意: {}", stringify!($($x)*)); };
}
```

### 2.3 常用宏片段说明符

```rust
// $x:expr - 表达式
// $x:ty - 类型
// $x:ident - 标识符
// $x:path - 路径
// $x:stmt - 语句
// $x:block - 代码块
// $x:item - 项（函数、结构体等）
// $x:meta - 属性
// $x:tt - 标记树（任意token）
```

### 2.4 宏的展开过程

```rust
// 宏在编译时展开，可以使用 cargo expand 查看展开结果

// 原始代码
macro_rules! double {
    ($x:expr) => { $x * 2 };
}

let result = double!(5 + 3);

// 展开后等价于
let result = 5 + 3 * 2; // 注意：是 5 + (3*2) = 11，不是 (5+3)*2 = 16

// 修复：使用块包裹
macro_rules! double_fixed {
    ($x:expr) => {{ let val = $x; val * 2 }};
}

let result = double_fixed!(5 + 3); // 16
```

### 2.5 过程宏与声明式宏对比

```rust
// 声明式宏（macro_rules!）
// - 简单易学，模式匹配
// - 适合代码生成和语法糖
// - 不能做复杂的代码分析

// 过程宏（proc_macro）
// - 可以做复杂的代码转换
// - 需要独立的 crate
// - 三种类型：派生宏、属性宏、函数宏

// 派生宏示例
#[derive(Debug, Clone, Serialize, Deserialize)]
struct MyStruct {
    field: String,
}
```

### 2.6 宏导出与可见性

```rust
// 在 lib.rs 中导出宏
#[macro_export]
macro_rules! my_public_macro {
    ($x:expr) => {
        println!("宏输出: {}", $x);
    };
}

// 在其他 crate 中使用
// use my_crate::my_public_macro;
// my_public_macro!(42);

// 局部宏（不导出）
macro_rules! local_macro {
    () => { println!("局部宏"); };
}
```

## 四、宏的使用场景

| 场景 | 推荐方式 | 示例 |
|------|---------|------|
| 简单代码复用 | 函数 | `fn add(a: i32, b: i32) -> i32` |
| 可变参数 | macro_rules! | `vec![1, 2, 3]` |
| 编译时代码生成 | 过程宏 | `#[derive(Serialize)]` |
| DSL 构建 | macro_rules! | `html! { <div></div> }` |
| 类型检查 | 过程宏 | `sqlx::query!` |

## 五、注意事项与常见陷阱

1. **卫生宏**：Rust 宏是卫生的，不会意外捕获变量，但可以通过 `$name:ident` 传入标识符
2. **展开时机**：宏在编译时展开，不是运行时，展开后的代码再进行类型检查
3. **调试困难**：宏展开后的代码可能难以调试，使用 `cargo expand` 查看展开结果
4. **命名约定**：声明式宏名通常以 `_macro` 或使用 `!` 后缀，过程宏使用有意义的名字
5. **作用域**：宏需要在使用前定义，或使用 `#[macro_export]` / `#[macro_use]`
6. **错误信息**：宏的错误信息可能不够友好，使用 `compile_error!` 提供自定义错误
7. **宏的开销**：宏展开增加编译时间，但不影响运行时性能
