# 宏基础概念

## 一、概念说明

宏（Macro）是 Rust 的元编程工具，在编译时展开代码。Rust 有声明式宏（macro_rules!）和过程宏两种类型。

```rust
// 声明式宏
macro_rules! say_hello {
    () => {
        println!("你好，世界！");
    };
}

// 过程宏（派生宏、属性宏、函数宏）
#[derive(Debug)]
struct MyStruct;
```

## 二、具体用法

### 2.1 声明式宏基础

```rust
macro_rules! create_function {
    ($name:ident) => {
        fn $name() {
            println!("函数 {:?} 被调用", stringify!($name));
        }
    };
}

create_function!(foo);
create_function!(bar);

fn main() {
    foo();
    bar();
}
```

### 2.2 宏匹配规则

```rust
macro_rules! match_example {
    // 匹配字面量
    (hello) => { println!("你好！"); };

    // 匹配表达式
    ($x:expr) => { println!("表达式: {}", $x); };

    // 匹配类型
    ($x:ty) => { println!("类型: {}", stringify!($x)); };

    // 匹配标识符
    ($x:ident) => { println!("标识符: {}", stringify!($x)); };

    // 匹配任意标记流
    ($($x:tt)*) => { println!("任意: {}", stringify!($($x)*)); };
}
```

### 2.3 常用宏片段说明符

```rust
// $x:expr - 表达式
// $x:ty - 类型
// $x:ident - 标识符
// $x:path - 路径
// $x:stmt - 语句
// $x:block - 代码块
// $x:item - 项（函数、结构体等）
// $x:meta - 属性
// $x:tt - 标记树（任意token）
```

## 三、注意事项与常见陷阱

1. **卫生宏**：Rust 宏是卫生的，不会意外捕获变量
2. **展开时机**：宏在编译时展开，不是运行时
3. **调试困难**：宏展开后的代码可能难以调试
4. **命名约定**：宏名通常以 `!` 结尾
5. **作用域**：宏需要在使用前定义，或使用 `#[macro_use]`
