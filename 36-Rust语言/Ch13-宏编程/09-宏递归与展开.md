# 宏递归与展开

## 一、概念说明

宏可以递归调用自身实现复杂的代码生成。但有展开深度限制（默认128层）。

```rust
macro_rules! count {
    () => { 0 };
    ($x:tt $($rest:tt)*) => { 1 + count!($($rest)*) };
}

assert_eq!(count!(a b c), 3);
```

## 二、具体用法

### 2.1 递归计数

```rust
macro_rules! count_exprs {
    () => { 0 };
    ($head:expr) => { 1 };
    ($head:expr, $($tail:expr),+) => {
        1 + count_exprs!($($tail),+)
    };
}

assert_eq!(count_exprs!(1, 2, 3, 4), 4);
```

### 2.2 TT Muncher 模式

```rust
macro_rules! tt_muncher {
    // 基础情况
    () => {};
    // 处理一个 token
    ($tt:tt $($rest:tt)*) => {
        println!("处理 token: {}", stringify!($tt));
        tt_muncher!($($rest)*);
    };
}
```

### 2.3 展开控制

```rust
// 限制递归深度
macro_rules! limited_recursion {
    (0) => { 0 };
    ($n:literal) => {
        1 + limited_recursion!(($n - 1))
    };
}
```

### 2.4 实用递归宏示例

```rust
// 递归计算表达式
macro_rules! eval {
    ($n:literal) => { $n };
    ($left:literal + $($rest:tt)*) => {
        $left + eval!($($rest)*)
    };
    ($left:literal - $($rest:tt)*) => {
        $left - eval!($($rest)*)
    };
    ($left:literal * $($rest:tt)*) => {
        $left * eval!($($rest)*)
    };
}

fn recursive_eval_demo() {
    let result = eval!(1 + 2 * 3 + 4);
    // 展开为: 1 + 2 * 3 + 4
    // 注意：宏不会处理运算符优先级
}
```

### 2.5 递归生成代码

```rust
// 为多个类型实现 Debug
macro_rules! impl_debug_for_types {
    // 基础情况
    ($t:ty) => {
        impl std::fmt::Debug for $t {
            fn fmt(&self, f: &mut std::fmt::Formatter) -> std::fmt::Result {
                write!(f, "{}({:?})", stringify!($t), self)
            }
        }
    };
    // 递归情况
    ($t:ty, $($rest:ty),+) => {
        impl_debug_for_types!($t);
        impl_debug_for_types!($($rest),+);
    };
}
```

### 2.6 展开深度优化

```rust
// 使用 @ 标记内部规则，避免公共 API 曝露递归细节
macro_rules! build_struct {
    // 公共接口
    ($name:ident { $($fields:tt)* }) => {
        build_struct!(@internal $name, [], $($fields)*);
    };

    // 内部：处理字段
    (@internal $name:ident, [$($processed:tt)*], $field:ident: $ty:ty, $($rest:tt)*) => {
        build_struct!(@internal $name, [$($processed)* $field: $ty,], $($rest)*);
    };

    // 内部：结束
    (@internal $name:ident, [$($processed:tt)*],) => {
        struct $name {
            $($processed)*
        }
    };
}
```

### 2.7 避免过度递归

```rust
// 不好：深度递归
macro_rules! bad_deep {
    () => { 0 };
    ($x:tt $($rest:tt)*) => { 1 + bad_deep!($($rest)*) };
}

// 好：使用累加器模式
macro_rules! good_count {
    ($($x:tt)*) => {
        <[()]>::len(&[$(good_count!(@unit $x)),*])
    };
    (@unit $x:tt) => { () };
}

// 更好：直接计数（编译器优化）
macro_rules! best_count {
    ($($x:tt),*) => {
        0usize $(+ good_count!(@one $x))*
    };
    (@one $x:tt) => { 1usize };
}
```

## 四、展开过程调试

```bash
# 使用 cargo expand 查看展开结果
cargo expand

# 使用 rustc 的展开输出
rustc -Z unstable-options --pretty=expanded main.rs

# 查看特定宏的展开
cargo expand --lib | grep -A 20 "my_macro"
```

## 五、注意事项与常见陷阱

1. **深度限制**：默认128层递归限制，可通过编译选项调整，但不建议超过
2. **栈溢出**：过深的递归可能导致编译器栈溢出，优化递归模式
3. **性能影响**：宏展开增加编译时间，大量递归宏会显著减慢编译
4. **调试困难**：递归宏难以调试，使用 `cargo expand` 和 `stringify!` 辅助
5. **替代方案**：复杂逻辑考虑过程宏，过程宏更灵活且无递归深度限制
6. **编译器优化**：简单递归宏会被编译器优化，但复杂递归仍有开销
7. **宏的终止**：确保所有递归路径都有基本情况，避免无限递归
