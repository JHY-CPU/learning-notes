# 宏递归与展开

## 一、概念说明

宏可以递归调用自身实现复杂的代码生成。但有展开深度限制（默认128层）。

```rust
macro_rules! count {
    () => { 0 };
    ($x:tt $($rest:tt)*) => { 1 + count!($($rest)*) };
}

assert_eq!(count!(a b c), 3);
```

## 二、具体用法

### 2.1 递归计数

```rust
macro_rules! count_exprs {
    () => { 0 };
    ($head:expr) => { 1 };
    ($head:expr, $($tail:expr),+) => {
        1 + count_exprs!($($tail),+)
    };
}

assert_eq!(count_exprs!(1, 2, 3, 4), 4);
```

### 2.2 TT Muncher 模式

```rust
macro_rules! tt_muncher {
    // 基础情况
    () => {};
    // 处理一个 token
    ($tt:tt $($rest:tt)*) => {
        println!("处理 token: {}", stringify!($tt));
        tt_muncher!($($rest)*);
    };
}
```

### 2.3 展开控制

```rust
// 限制递归深度
macro_rules! limited_recursion {
    (0) => { 0 };
    ($n:literal) => {
        1 + limited_recursion!(($n - 1))
    };
}
```

## 三、注意事项与常见陷阱

1. **深度限制**：默认128层递归限制
2. **栈溢出**：过深的递归可能导致编译器栈溢出
3. **性能影响**：宏展开增加编译时间
4. **调试困难**：递归宏难以调试
5. **替代方案**：复杂逻辑考虑过程宏
