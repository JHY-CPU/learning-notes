# 属性宏

## 一、概念说明

属性宏通过自定义属性扩展功能，可以修改或包装被标注的项。

```rust
#[route(GET, "/")]
fn index() -> String { "首页".to_string() }
```

## 二、具体用法

### 2.1 基本属性宏

```rust
use proc_macro::TokenStream;

#[proc_macro_attribute]
pub fn log_call(_attr: TokenStream, item: TokenStream) -> TokenStream {
    let input = syn::parse_macro_input!(item as syn::ItemFn);
    let name = &input.sig.ident;
    let block = &input.block;
    let vis = &input.vis;
    let sig = &input.sig;

    let gen = quote::quote! {
        #vis #sig {
            println!("调用函数: {}", stringify!(#name));
            let result = (|| #block)();
            println!("函数 {} 返回", stringify!(#name));
            result
        }
    };

    gen.into()
}
```

### 2.2 带参数的属性宏

```rust
#[proc_macro_attribute]
pub fn retry(attr: TokenStream, item: TokenStream) -> TokenStream {
    let max_retries: syn::LitInt = syn::parse(attr).unwrap();
    let input = syn::parse_macro_input!(item as syn::ItemFn);

    let gen = quote::quote! {
        #input

        fn retry_wrapper() -> Result<_, String> {
            for i in 0..#max_retries {
                match #input {
                    Ok(v) => return Ok(v),
                    Err(e) if i == #max_retries - 1 => return Err(e),
                    _ => continue,
                }
            }
            unreachable!()
        }
    };

    gen.into()
}
```

### 2.3 测量执行时间的属性宏

```rust
use proc_macro::TokenStream;
use quote::quote;
use syn::{parse_macro_input, ItemFn};

#[proc_macro_attribute]
pub fn timed(_attr: TokenStream, item: TokenStream) -> TokenStream {
    let input = parse_macro_input!(item as ItemFn);
    let name = &input.sig.ident;
    let block = &input.block;
    let vis = &input.vis;
    let sig = &input.sig;

    let gen = quote! {
        #vis #sig {
            let __start = std::time::Instant::now();
            let __result = (|| #block)();
            let __elapsed = __start.elapsed();
            println!("函数 {} 耗时: {:?}", stringify!(#name), __elapsed);
            __result
        }
    };

    gen.into()
}
```

### 2.4 缓存属性宏

```rust
#[proc_macro_attribute]
pub fn memoize(_attr: TokenStream, item: TokenStream) -> TokenStream {
    let input = parse_macro_input!(item as ItemFn);
    let name = &input.sig.ident;
    let cache_name = syn::Ident::new(
        &format!("__{}_CACHE", name.to_string().to_uppercase()),
        name.span(),
    );

    let gen = quote! {
        use std::collections::HashMap;
        use std::sync::Mutex;

        static #cache_name: once_cell::sync::Lazy<Mutex<HashMap<String, String>>> =
            once_cell::sync::Lazy::new(|| Mutex::new(HashMap::new()));

        #input

        // 使用 memoize 包装的函数会自动缓存结果
    };

    gen.into()
}
```

### 2.5 HTTP 路由属性宏

```rust
// 这是 actix-web 和 axum 中路由宏的简化版本
#[proc_macro_attribute]
pub fn get(attr: TokenStream, item: TokenStream) -> TokenStream {
    let path: syn::LitStr = parse_macro_input!(attr as syn::LitStr);
    let input = parse_macro_input!(item as ItemFn);
    let name = &input.sig.ident;
    let vis = &input.vis;

    let gen = quote! {
        #input

        #vis fn #name() -> axum::routing::MethodRouter {
            axum::routing::get(#name)
        }

        // 路由注册
        // Router::new().route(#path, #name())
    };

    gen.into()
}
```

## 四、属性宏的类型

| 类型 | 语法 | 用途 |
|------|------|------|
| 函数包装 | `#[my_attr] fn foo() {}` | 日志、计时、重试 |
| 结构体转换 | `#[my_attr] struct Foo {}` | 序列化、验证 |
| 枚举扩展 | `#[my_attr] enum E {}` | 实现 trait |
| 模块处理 | `#[my_attr] mod m {}` | 路由注册 |

## 五、注意事项与常见陷阱

1. **属性语法**：属性参数需符合 Rust 语法，复杂的 DSL 需要自定义解析
2. **上下文保留**：保留原有的可见性、泛型、where 子句等信息
3. **错误报告**：使用 `syn::Error` 和 `compile_error!` 报告编译错误，提供有用的错误信息
4. **文档继承**：考虑保留或处理文档属性，避免宏展开后丢失文档
5. **IDE 支持**：属性宏的 IDE 支持可能有限，扩展属性可能需要特殊配置
6. **属性名称**：选择有意义且不冲突的属性名，考虑命名空间
7. **嵌套使用**：多个属性宏的执行顺序可能影响结果，需要文档说明
