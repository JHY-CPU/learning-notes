# 函数宏

## 一、概念说明

函数宏类似于函数但接收 TokenStream 作为输入，可以实现更灵活的代码生成。

```rust
// 类似 println! 但自定义
my_macro!(input tokens here);
```

## 二、具体用法

### 2.1 基本函数宏

```rust
use proc_macro::TokenStream;

#[proc_macro]
pub fn make_answer(input: TokenStream) -> TokenStream {
    "fn answer() -> u32 { 42 }".parse().unwrap()
}
```

### 2.2 带参数的函数宏

```rust
#[proc_macro]
pub fn sql_query(input: TokenStream) -> TokenStream {
    let query = input.to_string();

    // 简单的 SQL 验证
    if !query.to_lowercase().starts_with("select") {
        panic!("只支持 SELECT 查询");
    }

    let gen = quote::quote! {
        fn execute_query() -> String {
            #query.to_string()
        }
    };

    gen.into()
}
```

### 2.3 构建器模式生成宏

```rust
#[proc_macro]
pub fn builder(input: TokenStream) -> TokenStream {
    let ast = parse_macro_input!(input as DeriveInput);
    let name = &ast.ident;
    let builder_name = syn::Ident::new(
        &format!("{}Builder", name),
        name.span(),
    );

    let fields = match &ast.data {
        syn::Data::Struct(data) => &data.fields,
        _ => panic!("builder 只支持结构体"),
    };

    let field_names: Vec<_> = fields.iter()
        .filter_map(|f| f.ident.as_ref())
        .collect();
    let field_types: Vec<_> = fields.iter()
        .map(|f| &f.ty)
        .collect();

    let gen = quote! {
        struct #builder_name {
            #(#field_names: Option<#field_types>,)*
        }

        impl #builder_name {
            fn new() -> Self {
                #builder_name {
                    #(#field_names: None,)*
                }
            }

            #(
                fn #field_names(mut self, value: #field_types) -> Self {
                    self.#field_names = Some(value);
                    self
                }
            )*

            fn build(self) -> Result<#name, String> {
                Ok(#name {
                    #(#field_names: self.#field_names
                        .ok_or(concat!(stringify!(#field_names), " 未设置"))?,)*
                })
            }
        }
    };

    gen.into()
}
```

### 2.4 自定义格式化宏

```rust
#[proc_macro]
pub fn format_structured(input: TokenStream) -> TokenStream {
    let input_str = input.to_string();

    // 解析 "key: value, key: value" 格式
    let pairs: Vec<&str> = input_str.split(',').collect();
    let mut fields = Vec::new();

    for pair in pairs {
        let parts: Vec<&str> = pair.split(':').collect();
        if parts.len() == 2 {
            let key = parts[0].trim();
            let value = parts[1].trim();
            fields.push(format!("{}: {}", key, value));
        }
    }

    let output = format!("{{ {} }}", fields.join(", "));
    output.parse().unwrap()
}
```

### 2.5 运行时配置宏

```rust
#[proc_macro]
pub fn config(input: TokenStream) -> TokenStream {
    let ast = parse_macro_input!(input as DeriveInput);
    let name = &ast.ident;

    let fields = match &ast.data {
        syn::Data::Struct(data) => &data.fields,
        _ => panic!("config 只支持结构体"),
    };

    let load_fields: Vec<_> = fields.iter().map(|f| {
        let field_name = f.ident.as_ref().unwrap();
        let field_type = &f.ty;
        let env_var = format!("APP_{}", field_name.to_string().to_uppercase());

        quote! {
            #field_name: std::env::var(#env_var)
                .ok()
                .and_then(|s| s.parse::<#field_type>().ok())
                .unwrap_or_default()
        }
    }).collect();

    let gen = quote! {
        impl #name {
            fn from_env() -> Self {
                #name {
                    #(#load_fields,)*
                }
            }
        }
    };

    gen.into()
}
```

## 四、函数宏 vs 声明式宏

| 特性 | 函数宏 | 声明式宏 |
|------|--------|---------|
| 输入类型 | TokenStream | 模式匹配 |
| 灵活性 | 非常高 | 中等 |
| 错误信息 | 可自定义 | 有限 |
| 学习曲线 | 高 | 低 |
| 性能 | 编译时 | 编译时 |
| 依赖 | syn, quote | 无 |

## 五、注意事项与常见陷阱

1. **输入解析**：TokenStream 解析需要 syn crate，选择正确的解析类型
2. **错误处理**：提供清晰的编译错误信息，使用 `syn::Error::new_spanned`
3. **字符串处理**：注意字符串中的特殊字符和转义
4. **性能**：宏展开在编译时，不影响运行时性能，但增加编译时间
5. **测试**：为宏编写测试用例，覆盖正常和异常输入
6. **Span 信息**：保留 Span 信息以提供准确的错误定位
7. **宏命名**：函数宏的命名应清晰表达其功能
