# 标准库宏分析

## 一、概念说明

分析标准库中常用宏的实现，学习宏编程的最佳实践。

```rust
// vec! 宏分析
macro_rules! vec {
    ($($x:expr),*) => {
        <[_]>::into_vec(
            Box::new([$($x),*])
        )
    };
    ($x:expr; $n:expr) => {
        <[_]>::into_vec(
            Box::new([$x; $n])
        )
    };
}
```

## 二、具体用法

### 2.1 println! 分析

```rust
// println! 的简化实现
macro_rules! my_println {
    ($($arg:tt)*) => {
        print!("{}\n", format_args!($($arg)*));
    };
}

// format_args! 是编译器内置的
// 处理格式化字符串
```

### 2.2 assert_eq! 分析

```rust
macro_rules! my_assert_eq {
    ($left:expr, $right:expr) => {
        match (&$left, &$right) {
            (left_val, right_val) => {
                if !(*left_val == *right_val) {
                    panic!(
                        "断言失败: `(left == right)`\n  left: `{:?}`,\n right: `{:?}`",
                        left_val, right_val
                    );
                }
            }
        }
    };
}
```

### 2.3 todo! 和 unimplemented!

```rust
macro_rules! my_todo {
    () => {
        panic!("尚未实现")
    };
    ($($arg:tt)+) => {
        panic!("尚未实现: {}", format_args!($($arg)+))
    };
}
```

### 2.4 matches! 宏分析

```rust
// 简化模式匹配的宏
macro_rules! my_matches {
    ($expression:expr, $(|)? $( $pattern:pat_param )|+ $( if $guard:expr )? $(,)?) => {
        match $expression {
            $( $pattern )|+ $( if $guard )? => true,
            _ => false,
        }
    };
}

fn matches_demo() {
    let value = Some(42);
    let is_some = my_matches!(value, Some(_));

    // 带守卫条件
    let is_positive = my_matches!(value, Some(x) if x > 0);

    // 多模式
    let is_special = my_matches!(value, None | Some(0) | Some(42));
}
```

### 2.5 concat! 和 stringify!

```rust
// concat! 在编译时连接字面量
macro_rules! my_concat {
    ($($arg:expr),*) => {
        // 实际上是编译器内置
        // 但可以用类似思路实现
        {
            let mut result = String::new();
            $(
                result.push_str(stringify!($arg));
            )*
            result
        }
    };
}

// stringify! 将表达式转换为字符串字面量
macro_rules! my_stringify {
    ($arg:expr) => {
        stringify!($arg)
    };
}

fn stringify_demo() {
    let expr_str = my_stringify!(1 + 2 * 3);
    println!("{}", expr_str); // "1 + 2 * 3"
}
```

### 2.6 cfg! 宏分析

```rust
// cfg! 宏在编译时求值条件
macro_rules! my_cfg {
    ($($arg:tt)*) => {
        // 实际是编译器内置
        // 在编译时展开为 true 或 false
        cfg!($($arg)*)
    };
}

fn cfg_demo() {
    if cfg!(target_os = "windows") {
        println!("Windows 平台");
    } else if cfg!(target_os = "linux") {
        println!("Linux 平台");
    }

    // 与 cfg 属性不同，cfg! 返回 bool
    #[cfg(target_os = "windows")]
    fn windows_only() {}
}
```

### 2.7 include! 与 include_str!/include_bytes!

```rust
// include! 包含并执行文件内容
// include_str! 包含文件为 &'static str
// include_bytes! 包含文件为 &'static [u8]

macro_rules! my_include_str {
    ($file:expr) => {
        // 编译器内置
        include_str!($file)
    };
}

fn include_demo() {
    // 在编译时嵌入文件内容
    const CONFIG: &str = my_include_str!("config.toml");

    // 常用于：
    // - 嵌入 SQL 文件
    // - 嵌入 HTML 模板
    // - 嵌入测试数据
}
```

## 四、标准库宏分类

| 类别 | 宏 | 用途 |
|------|-----|------|
| 格式化 | format!, println!, write! | 字符串格式化 |
| 集合 | vec!, HashMap! | 快速创建集合 |
| 断言 | assert!, assert_eq!, debug_assert! | 测试断言 |
| 错误 | panic!, todo!, unimplemented! | 控制流 |
| 环境 | cfg!, env!, option_env! | 编译时配置 |
| 转换 | stringify!, concat!, include_str! | 类型转换 |
| 线程 | thread_local! | 线程本地存储 |
| 打印 | print!, eprint!, dbg! | 调试输出 |

## 五、注意事项与常见陷阱

1. **卫生性**：标准库宏都是卫生的，遵循卫生宏的设计原则
2. **错误消息**：提供有用的错误消息，标准库宏的错误消息经过精心设计
3. **通用性**：标准库宏尽可能通用，不绑定特定类型
4. **性能**：考虑编译时间和运行时性能，标准库宏经过高度优化
5. **兼容性**：保持向后兼容，标准库宏的 API 非常稳定
6. **学习价值**：分析标准库宏是学习宏编程的最佳方式
7. **编译器内置**：部分标准库宏（format!, concat!）是编译器内置的，无法用纯 macro_rules! 实现
