# 宏与类型系统

## 一、概念说明

宏可以与 Rust 类型系统交互，生成类型安全的代码。

```rust
macro_rules! impl_from {
    ($from:ty => $to:ty, $convert:expr) => {
        impl From<$from> for $to {
            fn from(value: $from) -> Self {
                ($convert)(value)
            }
        }
    };
}
```

## 二、具体用法

### 2.1 类型级编程

```rust
macro_rules! type_list {
    () => { () };
    ($head:ty) => { ($head, ()) };
    ($head:ty, $($tail:ty),+) => {
        ($head, type_list!($($tail),+))
    };
}

type MyList = type_list!(i32, String, bool);
```

### 2.2 生成 trait 实现

```rust
macro_rules! impl_ops {
    ($t:ty) => {
        impl std::ops::Add for $t {
            type Output = $t;
            fn add(self, rhs: $t) -> $t {
                // 使用 + 运算符
                self + rhs
            }
        }
    };
}
```

### 2.3 条件编译

```rust
macro_rules! cfg_if {
    ($cfg:meta { $($then:tt)* } else { $($else:tt)* }) => {
        #[cfg($cfg)]
        $($then)*
        #[cfg(not($cfg))]
        $($else)*
    };
}
```

### 2.4 类型安全的 Builder 模式

```rust
macro_rules! type_safe_builder {
    ($name:ident { $($field:ident: $ty:ty),* $(,)? }) => {
        struct $name {
            $($field: $ty,)*
        }

        impl $name {
            fn builder() -> [<$name Builder>] {
                [<$name Builder>]::new()
            }
        }

        // 生成 Builder 类型
        paste::paste! {
            struct [<$name Builder>] {
                $($field: Option<$ty>,)*
            }

            impl [<$name Builder>] {
                fn new() -> Self {
                    [<$name Builder>] {
                        $($field: None,)*
                    }
                }

                $(
                    fn $field(mut self, value: $ty) -> Self {
                        self.$field = Some(value);
                        self
                    }
                )*

                fn build(self) -> Result<$name, String> {
                    Ok($name {
                        $($field: self.$field
                            .ok_or(concat!(stringify!($field), " is required"))?,)*
                    })
                }
            }
        }
    };
}
```

### 2.5 类型约束传播

```rust
macro_rules! with_bounds {
    ($ty:ty where $($bound:tt)+) => {
        // 生成带有 where 子句的类型约束
        // 用于确保宏生成的代码满足类型约束
    };
}

// 泛型约束检查宏
macro_rules! assert_impl {
    ($ty:ty: $($trait:tt)+) => {
        const _: fn() = || {
            fn assert_impl<T: $($trait)+>() {}
            assert_impl::<$ty>();
        };
    };
}

// 使用
// assert_impl!(Vec<i32>: Send + Sync);
```

### 2.6 动态类型派发

```rust
macro_rules! dyn_dispatch {
    ($trait_name:ident for $($ty:ty),+ $(,)?) => {
        fn create_dispatch(value: &str) -> Option<Box<dyn $trait_name>> {
            match value {
                $(
                    stringify!($ty) => Some(Box::new(<$ty as Default>::default())),
                )+
                _ => None,
            }
        }
    };
}
```

## 四、类型系统交互模式

| 模式 | 用途 | 示例 |
|------|------|------|
| 类型映射 | From/Into 转换 | `impl_from!` |
| 约束生成 | 生成 trait bounds | `with_bounds!` |
| 类型检查 | 编译时验证 | `assert_impl!` |
| 反射模拟 | 获取类型信息 | `type_name!` |
| 类型级编程 | 编译时计算 | `type_list!` |

## 五、注意事项与常见陷阱

1. **类型检查时机**：宏展开后才会进行类型检查，错误信息可能不直观
2. **trait 约束**：为泛型实现时需注意 trait 约束的正确性
3. **生命周期**：处理生命周期参数需额外注意，避免引入无效的生命周期
4. **编译错误**：类型错误可能在宏展开后才出现，错误位置可能不准确
5. **文档生成**：为宏生成的代码提供文档，使用 `#[doc(hidden)]` 隐藏内部项
6. **泛型与宏**：宏中的泛型处理需要使用 `quote!` 的重复语法
7. **类型推断**：宏生成的代码可能影响类型推断，必要时提供类型标注
