# 宏与类型系统

## 一、概念说明

宏可以与 Rust 类型系统交互，生成类型安全的代码。

```rust
macro_rules! impl_from {
    ($from:ty => $to:ty, $convert:expr) => {
        impl From<$from> for $to {
            fn from(value: $from) -> Self {
                ($convert)(value)
            }
        }
    };
}
```

## 二、具体用法

### 2.1 类型级编程

```rust
macro_rules! type_list {
    () => { () };
    ($head:ty) => { ($head, ()) };
    ($head:ty, $($tail:ty),+) => {
        ($head, type_list!($($tail),+))
    };
}

type MyList = type_list!(i32, String, bool);
```

### 2.2 生成 trait 实现

```rust
macro_rules! impl_ops {
    ($t:ty) => {
        impl std::ops::Add for $t {
            type Output = $t;
            fn add(self, rhs: $t) -> $t {
                // 使用 + 运算符
                self + rhs
            }
        }
    };
}
```

### 2.3 条件编译

```rust
macro_rules! cfg_if {
    ($cfg:meta { $($then:tt)* } else { $($else:tt)* }) => {
        #[cfg($cfg)]
        $($then)*
        #[cfg(not($cfg))]
        $($else)*
    };
}
```

## 三、注意事项与常见陷阱

1. **类型检查**：宏展开后才会进行类型检查
2. **trait 约束**：为泛型实现时需注意 trait 约束
3. **生命周期**：处理生命周期参数需额外注意
4. **编译错误**：类型错误可能在宏展开后才出现
5. **文档生成**：为宏生成的代码提供文档
