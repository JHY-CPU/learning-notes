# 过程宏入门

## 一、概念说明

过程宏接收 Rust 代码作为输入，返回新的 Rust 代码。有三种类型：派生宏、属性宏、函数宏。过程宏在独立的 crate 中定义。

```rust
// proc-macro crate
use proc_macro::TokenStream;

#[proc_macro_derive(MyDerive)]
pub fn my_derive(input: TokenStream) -> TokenStream {
    // 解析输入，生成新代码
    input
}
```

## 二、具体用法

### 2.1 基本结构

```rust
// Cargo.toml
// [lib]
// proc-macro = true

use proc_macro::TokenStream;
use quote::quote;
use syn::{parse_macro_input, DeriveInput};

#[proc_macro_derive(Hello)]
pub fn hello_derive(input: TokenStream) -> TokenStream {
    let ast = parse_macro_input!(input as DeriveInput);
    let name = &ast.ident;

    let gen = quote! {
        impl Hello for #name {
            fn hello(&self) {
                println!("Hello from {}!", stringify!(#name));
            }
        }
    };

    gen.into()
}
```

### 2.2 使用派生宏

```rust
// 在主 crate 中使用
use my_macros::Hello;

#[derive(Hello)]
struct MyStruct;

fn main() {
    let s = MyStruct;
    s.hello(); // "Hello from MyStruct!"
}
```

### 2.3 处理字段

```rust
use syn::{parse_macro_input, DeriveInput, Data, Fields};

#[proc_macro_derive(FieldNames)]
pub fn field_names_derive(input: TokenStream) -> TokenStream {
    let ast = parse_macro_input!(input as DeriveInput);
    let name = &ast.ident;

    let fields = match &ast.data {
        Data::Struct(data) => match &data.fields {
            Fields::Named(fields) => &fields.named,
            _ => panic!("只支持命名字段"),
        },
        _ => panic!("只支持结构体"),
    };

    let field_names: Vec<String> = fields.iter()
        .map(|f| f.ident.as_ref().unwrap().to_string())
        .collect();

    let gen = quote! {
        impl #name {
            fn field_names() -> Vec<&'static str> {
                vec![#(#field_names),*]
            }
        }
    };

    gen.into()
}
```

## 三、注意事项与常见陷阱

1. **独立 crate**：过程宏必须在 proc-macro 类型的 crate 中
2. **依赖项**：使用 syn 和 quote 简化开发
3. **错误信息**：提供清晰的编译错误信息
4. **展开调试**：使用 cargo expand 查看展开结果
5. **版本兼容**：注意 syn 和 quote 版本匹配
