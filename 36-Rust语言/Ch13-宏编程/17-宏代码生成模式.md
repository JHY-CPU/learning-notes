# 宏代码生成模式

## 一、概念说明

总结宏代码生成的常用模式：Builder 模式、New Type 模式、枚举派生等。

```rust
// Builder 模式
#[derive(Builder)]
struct Server {
    host: String,
    port: u16,
}
```

## 二、具体用法

### 2.1 New Type 模式

```rust
macro_rules! newtype {
    ($name:ident, $inner:ty) => {
        #[derive(Debug, Clone, PartialEq)]
        pub struct $name(pub $inner);

        impl From<$inner> for $name {
            fn from(v: $inner) -> Self {
                $name(v)
            }
        }

        impl std::ops::Deref for $name {
            type Target = $inner;
            fn deref(&self) -> &$inner {
                &self.0
            }
        }
    };
}

newtype!(UserId, u64);
newtype!(Email, String);
```

### 2.2 枚举辅助方法

```rust
macro_rules! enum_with_variants {
    ($name:ident { $($variant:ident),* }) => {
        enum $name {
            $($variant),*
        }

        impl $name {
            fn variants() -> Vec<&'static str> {
                vec![$(stringify!($variant)),*]
            }

            fn from_str(s: &str) -> Option<Self> {
                match s {
                    $(stringify!($variant) => Some($name::$variant),)*
                    _ => None,
                }
            }
        }
    };
}
```

### 2.3 配置结构体生成

```rust
macro_rules! config_struct {
    ($name:ident { $($field:ident: $ty:ty = $default:expr),* }) => {
        struct $name {
            $($field: $ty),*
        }

        impl Default for $name {
            fn default() -> Self {
                $name {
                    $($field: $default),*
                }
            }
        }
    };
}

config_struct!(AppConfig {
    host: String = "localhost".to_string(),
    port: u16 = 8080,
    debug: bool = false
});
```

## 三、注意事项与常见陷阱

1. **模式选择**：根据需求选择合适的生成模式
2. **可读性**：生成的代码应易于理解和调试
3. **灵活性**：提供自定义选项
4. **测试**：为生成的代码编写测试
5. **文档**：说明宏生成的 API
