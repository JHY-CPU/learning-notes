# 宏的卫生性

## 一、概念说明

Rust 宏是卫生的（Hygienic），宏内部定义的变量不会与外部变量冲突。

```rust
macro_rules! using_x {
    ($x:expr) => {
        let x = $x;  // 这个 x 是宏内部的
    };
}

let x = 1;
using_x!(2);
println!("{}", x); // 输出 1，不是 2
```

## 二、具体用法

### 2.1 卫生宏行为

```rust
macro_rules! create_var {
    () => {
        let x = 10;
    };
}

let x = 5;
create_var!();
println!("{}", x); // 5，宏内的 x 在不同作用域
```

### 2.2 打破卫生性

```rust
macro_rules! create_var_hygiene {
    ($name:ident) => {
        let $name = 10;
    };
}

create_var_hygiene!(y);
println!("{}", y); // 10

// 过程宏中使用 Span::call_site() 或 Span::mixed_site()
```

### 2.3 导入外部变量

```rust
macro_rules! set_var {
    ($name:ident, $value:expr) => {
        $name = $value;
    };
}

let mut x = 0;
set_var!(x, 5);
println!("{}", x); // 5
```

## 三、注意事项与常见陷阱

1. **卫生性保证**：默认行为防止变量名冲突
2. **Span 含义**：Span 决定变量的作用域
3. **call_site**：使用 call_site() Span 继承调用者上下文
4. **测试验证**：验证宏的卫生性行为
5. **文档说明**：说明宏是否会影响外部作用域
