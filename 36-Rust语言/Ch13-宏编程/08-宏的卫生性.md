# 宏的卫生性

## 一、概念说明

Rust 宏是卫生的（Hygienic），宏内部定义的变量不会与外部变量冲突。

```rust
macro_rules! using_x {
    ($x:expr) => {
        let x = $x;  // 这个 x 是宏内部的
    };
}

let x = 1;
using_x!(2);
println!("{}", x); // 输出 1，不是 2
```

## 二、具体用法

### 2.1 卫生宏行为

```rust
macro_rules! create_var {
    () => {
        let x = 10;
    };
}

let x = 5;
create_var!();
println!("{}", x); // 5，宏内的 x 在不同作用域
```

### 2.2 打破卫生性

```rust
macro_rules! create_var_hygiene {
    ($name:ident) => {
        let $name = 10;
    };
}

create_var_hygiene!(y);
println!("{}", y); // 10

// 过程宏中使用 Span::call_site() 或 Span::mixed_site()
```

### 2.3 导入外部变量

```rust
macro_rules! set_var {
    ($name:ident, $value:expr) => {
        $name = $value;
    };
}

let mut x = 0;
set_var!(x, 5);
println!("{}", x); // 5
```

### 2.4 Span 与卫生性

```rust
// Span 决定标识符的作用域
// call_site: 调用宏的位置
// mixed_site: 混合卫生性
// def_site: 宏定义的位置（nightly only）

// 在过程宏中控制卫生性
use proc_macro::TokenStream;
use proc_macro2::Span;

#[proc_macro]
pub fn my_macro(input: TokenStream) -> TokenStream {
    // Span::call_site() - 继承调用者上下文
    let ident = syn::Ident::new("my_var", Span::call_site());

    quote::quote! {
        let #ident = 42;
    }.into()
}
```

### 2.5 卫生性与导入

```rust
// 宏内部不能直接使用调用者的导入
// 需要通过完整路径引用

macro_rules! use_std_types {
    () => {
        // 不卫生：依赖调用者有 Vec 的导入
        // let v = Vec::new();

        // 卫生：使用完整路径
        let v = std::vec::Vec::new();
        let m = std::collections::HashMap::new();
    };
}

// 最佳实践：在宏内使用完整路径
macro_rules! safe_macro {
    () => {{
        use std::collections::HashMap;
        let mut map = HashMap::new();
        map.insert("key", "value");
        map
    }};
}
```

### 2.6 卫生性测试用例

```rust
#[cfg(test)]
mod hygiene_tests {
    macro_rules! hygiene_test {
        () => {
            let x = 42;
        };
    }

    #[test]
    fn test_variable_hygiene() {
        let x = 10;
        hygiene_test!();
        // x 仍然是 10，宏内的 x 在不同作用域
        assert_eq!(x, 10);
    }

    macro_rules! return_test {
        () => {
            return 42;
        };
    }

    #[test]
    fn test_return_hygiene() {
        fn inner() -> i32 {
            let x = 10;
            return_test!();
            x // 这行不会执行
        }
        assert_eq!(inner(), 42);
    }
}
```

### 2.7 打破卫生性的技巧

```rust
// 方式1: 使用 $name:ident 参数
macro_rules! create_var {
    ($name:ident, $value:expr) => {
        let $name = $value;
    };
}

create_var!(my_custom_name, 42);
println!("{}", my_custom_name); // 42

// 方式2: 使用 Span::mixed_site() 在过程宏中
// 方式3: 通过宏参数传递闭包
macro_rules! with_context {
    ($var:ident, $body:block) => {{
        let $var = get_context();
        $body
    }};
}
```

## 四、卫生性规则总结

| 元素 | 默认行为 | 如何打破 |
|------|---------|---------|
| 变量 | 卫生 | `$name:ident` |
| 标签 | 卫生 | `$label:ident` |
| 生命周期 | 卫生 | `$lt:lifetime` |
| 导入 | 不卫生 | 使用完整路径 |
| 类型 | 卫生 | `$ty:ty` |
| 返回/break | 卫生 | 无法打破 |

## 五、注意事项与常见陷阱

1. **卫生性保证**：默认行为防止变量名冲突，这是 Rust 宏的安全特性
2. **Span 含义**：Span 决定变量的作用域，`call_site` 最常用
3. **call_site**：使用 `call_site()` Span 继承调用者上下文，大多数情况下正确
4. **测试验证**：验证宏的卫生性行为，确保不会意外捕获或覆盖变量
5. **文档说明**：说明宏是否会影响外部作用域，特别是破坏卫生性的宏
6. **导入卫生**：宏内使用完整路径避免依赖调用者的导入
7. **调试卫生性问题**：使用 `cargo expand` 查看展开结果，检查作用域是否正确
