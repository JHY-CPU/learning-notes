# 宏测试策略

## 一、概念说明

宏的测试需要验证展开结果、错误处理和边界情况。Rust 提供了 `trybuild` 等工具辅助宏测试。

```rust
// 使用 trybuild 测试宏
#[test]
fn test_macro() {
    let t = trybuild::TestCases::new();
    t.pass("tests/valid/*.rs");
    t.compile_fail("tests/invalid/*.rs");
}
```

## 二、具体用法

### 2.1 基本测试

```rust
#[cfg(test)]
mod tests {
    macro_rules! test_macro {
        ($input:expr) => { $input + 1 };
    }

    #[test]
    fn test_basic() {
        assert_eq!(test_macro!(1), 2);
    }
}
```

### 2.2 trybuild 测试

```rust
// tests/ui.rs
#[test]
fn ui() {
    let t = trybuild::TestCases::new();
    t.pass("tests/ui/pass/*.rs");
    t.compile_fail("tests/ui/fail/*.rs");
}

// tests/ui/pass/basic.rs
use my_macros::MyMacro;
#[derive(MyMacro)]
struct Test;

// tests/ui/fail/invalid.rs
use my_macros::MyMacro;
#[derive(MyMacro)]
enum Test; // 应该失败：不支持枚举
```

### 2.3 宏卫生性测试

```rust
macro_rules! hygiene_test {
    () => {
        let x = 1;
    };
}

#[test]
fn test_hygiene() {
    let x = 5;
    hygiene_test!();
    assert_eq!(x, 5); // 宏内的 x 不影响外部
}
```

### 2.4 展开结果测试

```rust
use quote::quote;

#[cfg(test)]
mod expansion_tests {
    use super::*;

    #[test]
    fn test_derive_expansion() {
        let input = quote! {
            struct Foo { x: i32 }
        };

        let output = my_derive_macro(input);
        let output_str = output.to_string();

        // 验证生成了期望的代码
        assert!(output_str.contains("impl"));
        assert!(output_str.contains("MyTrait"));
    }
}
```

### 2.5 属性解析测试

```rust
#[cfg(test)]
mod attr_tests {
    use syn::parse_str;

    #[test]
    fn test_attribute_parsing() {
        let attr: syn::Attribute = parse_str(r#"#[my_attr(name = "test")]"#).unwrap();

        // 验证属性被正确解析
        assert_eq!(attr.path().get_ident().unwrap().to_string(), "my_attr");
    }
}
```

### 2.6 性能测试

```rust
#[cfg(test)]
mod perf_tests {
    use std::time::Instant;

    #[test]
    fn test_macro_expansion_time() {
        let start = Instant::now();

        // 展开大量宏
        for _ in 0..1000 {
            my_macro!(input);
        }

        let elapsed = start.elapsed();
        assert!(elapsed.as_millis() < 100, "宏展开太慢: {:?}", elapsed);
    }
}
```

### 2.7 文档测试

```rust
/// 创建一个向量。
///
/// # 示例
///
/// ```
/// let v = my_vec![1, 2, 3];
/// assert_eq!(v, vec![1, 2, 3]);
/// ```
///
/// # 空向量
///
/// ```
/// let v: Vec<i32> = my_vec![];
/// assert!(v.is_empty());
/// ```
#[macro_export]
macro_rules! my_vec {
    ($($x:expr),* $(,)? ) => { vec![$($x),*] };
}
```

## 四、测试覆盖检查清单

```
基础测试：
- [ ] 正常输入
- [ ] 空输入
- [ ] 单个元素
- [ ] 多个元素
- [ ] 尾逗号支持

边界测试：
- [ ] 最大递归深度
- [ ] 特殊字符
- [ ] Unicode 输入
- [ ] 嵌套调用

错误测试：
- [ ] 无效语法
- [ ] 类型错误
- [ ] 自定义错误消息

集成测试：
- [ ] 与标准库交互
- [ ] 与第三方库交互
- [ ] 在不同上下文中使用
```

## 五、注意事项与常见陷阱

1. **测试覆盖**：覆盖正常和异常输入，包括边界情况
2. **错误消息**：验证错误消息的准确性，确保用户能理解问题
3. **卫生性测试**：确保宏的卫生性行为，验证变量不会冲突
4. **回归测试**：为发现的 bug 添加测试，防止再次出现
5. **文档示例**：示例代码也是测试，确保文档示例可编译运行
6. **trybuild 版本**：注意 trybuild 与编译器版本的兼容性
7. **跨平台测试**：在不同平台上测试宏的行为
