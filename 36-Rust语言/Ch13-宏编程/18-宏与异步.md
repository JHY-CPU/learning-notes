# 宏与异步

## 一、概念说明

异步运行时（tokio、async-std）大量使用宏简化异步代码编写。

```rust
#[tokio::main]
async fn main() {
    let result = fetch_data().await;
}

#[tokio::test]
async fn test_fetch() {
    let result = fetch_data().await;
}
```

## 二、具体用法

### 2.1 tokio 宏分析

```rust
// #[tokio::main] 展开为
fn main() {
    tokio::runtime::Builder::new_multi_thread()
        .enable_all()
        .build()
        .unwrap()
        .block_on(async {
            // 用户的 async main
        })
}
```

### 2.2 自定义异步宏

```rust
macro_rules! spawn_blocking {
    ($body:expr) => {
        tokio::task::spawn_blocking(|| $body)
            .await
            .map_err(|e| format!("任务失败: {}", e))
    };
}

async fn process() -> Result<(), String> {
    let result = spawn_blocking!({
        // CPU 密集型操作
        heavy_computation()
    })?;
    Ok(result)
}
```

### 2.3 select! 宏

```rust
macro_rules! my_select {
    ($($token:tt)*) => {
        tokio::select! { $($token)* }
    };
}

async fn example() {
    tokio::select! {
        result = fetch_a() => println!("A 完成"),
        result = fetch_b() => println!("B 完成"),
        _ = tokio::time::sleep(Duration::from_secs(5)) => println!("超时"),
    }
}
```

### 2.4 异步重试宏

```rust
macro_rules! retry_async {
    ($times:expr, $body:expr) => {{
        let mut attempts = 0;
        loop {
            attempts += 1;
            match $body.await {
                Ok(value) => break Ok(value),
                Err(e) if attempts < $times => {
                    tracing::warn!("尝试 {}/{} 失败: {}", attempts, $times, e);
                    tokio::time::sleep(
                        std::time::Duration::from_millis(100 * attempts as u64)
                    ).await;
                }
                Err(e) => break Err(e),
            }
        }
    }};
}

async fn fetch_with_retry() -> Result<String, reqwest::Error> {
    retry_async!(3, reqwest::get("https://api.example.com/data"))
}
```

### 2.5 异步超时宏

```rust
macro_rules! timeout {
    ($duration:expr, $body:expr) => {
        tokio::time::timeout($duration, $body).await
            .map_err(|_| format!("操作超时: {:?}", $duration))
    };
}

async fn with_timeout_example() -> Result<(), String> {
    let result = timeout!(
        std::time::Duration::from_secs(5),
        async {
            tokio::time::sleep(std::time::Duration::from_secs(10)).await;
            "完成"
        }
    );

    match result {
        Ok(value) => println!("成功: {}", value),
        Err(e) => println!("失败: {}", e),
    }
    Ok(())
}
```

### 2.6 异步测试宏展开

```rust
// #[tokio::test] 展开为
#[test]
fn test_example() {
    let rt = tokio::runtime::Builder::new_current_thread()
        .enable_all()
        .build()
        .unwrap();

    rt.block_on(async {
        // 测试代码
        let result = fetch_data().await;
        assert!(result.is_ok());
    });
}

// 自定义测试宏
macro_rules! async_test {
    ($name:ident, $body:block) => {
        #[test]
        fn $name() {
            let rt = tokio::runtime::Runtime::new().unwrap();
            rt.block_on(async $body);
        }
    };
}
```

## 四、异步宏展开对照表

| 宏 | 展开前 | 展开后 |
|------|--------|--------|
| `#[tokio::main]` | `async fn main()` | `fn main() { rt.block_on(async { ... }) }` |
| `#[tokio::test]` | `async fn test()` | `fn test() { rt.block_on(async { ... }) }` |
| `tokio::select!` | 多分支 await | 轮询多个 future |
| `tokio::join!` | 多个 await | 并发执行多个 future |

## 五、注意事项与常见陷阱

1. **展开复杂度**：异步宏展开后的代码可能很复杂，使用 cargo expand 查看
2. **错误处理**：异步宏中的错误处理需注意，避免 panic 中断运行时
3. **生命周期**：异步代码的生命周期更复杂，跨 await 的引用需要 `'static`
4. **调试困难**：异步宏的调试更困难，使用 tokio-console 辅助调试
5. **文档**：为宏提供清晰的异步使用示例，说明运行时要求
6. **取消安全**：确保异步宏生成的代码在取消时是安全的
7. **运行时选择**：宏可能绑定特定运行时，考虑提供通用版本
