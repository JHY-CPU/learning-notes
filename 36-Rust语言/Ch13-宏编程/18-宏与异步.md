# 宏与异步

## 一、概念说明

异步运行时（tokio、async-std）大量使用宏简化异步代码编写。

```rust
#[tokio::main]
async fn main() {
    let result = fetch_data().await;
}

#[tokio::test]
async fn test_fetch() {
    let result = fetch_data().await;
}
```

## 二、具体用法

### 2.1 tokio 宏分析

```rust
// #[tokio::main] 展开为
fn main() {
    tokio::runtime::Builder::new_multi_thread()
        .enable_all()
        .build()
        .unwrap()
        .block_on(async {
            // 用户的 async main
        })
}
```

### 2.2 自定义异步宏

```rust
macro_rules! spawn_blocking {
    ($body:expr) => {
        tokio::task::spawn_blocking(|| $body)
            .await
            .map_err(|e| format!("任务失败: {}", e))
    };
}

async fn process() -> Result<(), String> {
    let result = spawn_blocking!({
        // CPU 密集型操作
        heavy_computation()
    })?;
    Ok(result)
}
```

### 2.3 select! 宏

```rust
macro_rules! my_select {
    ($($token:tt)*) => {
        tokio::select! { $($token)* }
    };
}

async fn example() {
    tokio::select! {
        result = fetch_a() => println!("A 完成"),
        result = fetch_b() => println!("B 完成"),
        _ = tokio::time::sleep(Duration::from_secs(5)) => println!("超时"),
    }
}
```

## 三、注意事项与常见陷阱

1. **展开复杂度**：异步宏展开后的代码可能很复杂
2. **错误处理**：异步宏中的错误处理需注意
3. **生命周期**：异步代码的生命周期更复杂
4. **调试困难**：异步宏的调试更困难
5. **文档**：为宏提供清晰的异步使用示例
