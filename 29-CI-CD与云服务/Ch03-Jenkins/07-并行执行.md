# 并行执行

## 一、并行阶段

```groovy
stage('Test') {
    parallel {
        stage('Unit Test') {
            steps {
                sh 'npm run test:unit'
            }
        }
        stage('Integration Test') {
            steps {
                sh 'npm run test:integration'
            }
        }
        stage('E2E Test') {
            steps {
                sh 'npm run test:e2e'
            }
        }
    }
}
```

## 二、并行矩阵

```groovy
stage('Matrix') {
    matrix {
        axes {
            axis {
                name 'PLATFORM'
                values 'linux', 'windows', 'mac'
            }
            axis {
                name 'BROWSER'
                values 'chrome', 'firefox'
            }
        }
        stages {
            stage('Test') {
                steps {
                    sh "echo ${PLATFORM} ${BROWSER}"
                }
            }
        }
    }
}
```

## 三、failFast

```groovy
stage('Test') {
    failFast true  # 一个失败则取消其他
    parallel {
        stage('A') { steps { sh 'test-a' } }
        stage('B') { steps { sh 'test-b' } }
    }
}
```

## 四、注意事项

1. **独立性**：并行阶段必须相互独立
2. **资源竞争**：注意并行任务的资源竞争
3. **日志管理**：并行任务的日志可能混杂
4. **超时设置**：为并行任务设置超时
