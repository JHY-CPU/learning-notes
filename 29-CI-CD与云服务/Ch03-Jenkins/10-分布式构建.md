# 分布式构建

## 一、概念说明

Jenkins支持Master-Agent架构，将构建任务分发到多个Agent节点执行。

## 二、Agent配置

```bash
# 添加Agent
# Manage Jenkins > Manage Nodes > New Node

# Agent类型
# Permanent Agent - 永久代理
# SSH - SSH连接
# JNLP - Java Web Start

# 节点标签
# linux, docker, gpu, build
```

## 三、指定Agent

```groovy
pipeline {
    agent {
        label 'linux && docker'
    }
    stages {
        stage('Build') {
            agent {
                label 'gpu'
            }
            steps {
                sh 'make build'
            }
        }
    }
}
```

## 四、Agent配置

```bash
# Agent机器配置
# 1. 安装Java
# 2. 下载agent.jar
# 3. 运行连接命令

# SSH方式
# Jenkins自动通过SSH连接Agent

# JNLP方式
java -jar agent.jar -url http://jenkins-url -secret xxx agent-name
```

## 五、注意事项

1. **资源监控**：监控Agent节点资源使用
2. **网络连接**：确保Agent能访问Jenkins和代码仓库
3. **标签管理**：合理使用标签分配任务
4. **自动缩放**：使用Kubernetes插件自动缩放Agent
