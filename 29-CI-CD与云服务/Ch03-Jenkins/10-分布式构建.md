# 分布式构建

## 一、概念说明

Jenkins支持Master-Agent架构，将构建任务分发到多个Agent节点执行。

## 二、Agent配置

```bash
# 添加Agent
# Manage Jenkins > Manage Nodes > New Node

# Agent类型
# Permanent Agent - 永久代理
# SSH - SSH连接
# JNLP - Java Web Start

# 节点标签
# linux, docker, gpu, build
```

## 三、指定Agent

```groovy
pipeline {
    agent {
        label 'linux && docker'
    }
    stages {
        stage('Build') {
            agent {
                label 'gpu'
            }
            steps {
                sh 'make build'
            }
        }
    }
}
```

## 四、Agent配置

```bash
# Agent机器配置
# 1. 安装Java
# 2. 下载agent.jar
# 3. 运行连接命令

# SSH方式
# Jenkins自动通过SSH连接Agent

# JNLP方式
java -jar agent.jar -url http://jenkins-url -secret xxx agent-name
```

## 五、注意事项

1. **资源监控**：监控Agent节点资源使用
2. **网络连接**：确保Agent能访问Jenkins和代码仓库
3. **标签管理**：合理使用标签分配任务
4. **自动缩放**：使用Kubernetes插件自动缩放Agent

## 六、SSH Agent配置

```bash
# 1. 在Agent机器上
sudo useradd -m -s /bin/bash jenkins
sudo -u jenkins mkdir ~/.ssh

# 2. 在Jenkins Master上生成密钥
ssh-keygen -t rsa -b 4096 -f jenkins_agent_key

# 3. 将公钥添加到Agent
echo "ssh-rsa AAAA..." >> /home/jenkins/.ssh/authorized_keys

# 4. 在Jenkins UI中添加Agent
# Manage Jenkins > Manage Nodes > New Node
# - Name: linux-agent-1
# - Type: Permanent Agent
# - Launch method: Launch agents via SSH
#   - Host: agent-ip
#   - Credentials: SSH key
#   - Host Key Verification Strategy: Non verifying
# - Labels: linux docker
# - Usage: Only build jobs with label expressions matching this node
```

## 七、Kubernetes Agent配置

```groovy
// 使用Kubernetes插件动态分配Agent
pipeline {
    agent {
        kubernetes {
            yaml '''
                apiVersion: v1
                kind: Pod
                spec:
                  containers:
                  - name: jnlp
                    image: jenkins/inbound-agent:latest
                    resources:
                      requests:
                        memory: "256Mi"
                        cpu: "200m"
                      limits:
                        memory: "512Mi"
                        cpu: "500m"
                  - name: node
                    image: node:20-alpine
                    command: ['cat']
                    tty: true
                  - name: docker
                    image: docker:24.0.5
                    command: ['cat']
                    tty: true
                    volumeMounts:
                    - name: docker-sock
                      mountPath: /var/run/docker.sock
                  volumes:
                  - name: docker-sock
                    hostPath:
                      path: /var/run/docker.sock
            '''
        }
    }
    stages {
        stage('Build') {
            steps {
                container('node') {
                    sh 'npm ci && npm run build'
                }
            }
        }
        stage('Docker Build') {
            steps {
                container('docker') {
                    sh 'docker build -t myapp .'
                }
            }
        }
    }
}
```

## 九、Agent自动缩放

```groovy
// Kubernetes插件自动缩放配置
// Manage Jenkins > Clouds > Kubernetes
// - Kubernetes URL: https://kubernetes.default.svc
// - Jenkins URL: http://jenkins:8080
// - Container Cap: 100
// - Pod Labels: jenkins=agent
// - Idle Timeout: 30
```

```yaml
# Kubernetes Jenkins Agent RBAC
apiVersion: v1
kind: ServiceAccount
metadata:
  name: jenkins
  namespace: jenkins

---
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRoleBinding
metadata:
  name: jenkins-admin
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: ClusterRole
  name: cluster-admin
subjects:
  - kind: ServiceAccount
    name: jenkins
    namespace: jenkins
```
