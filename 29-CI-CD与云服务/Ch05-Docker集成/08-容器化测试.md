# 容器化测试

## 一、概念说明

容器化测试使用Docker提供一致的测试环境，确保测试结果可重现。

## 二、测试Dockerfile

```dockerfile
FROM node:20-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
CMD ["npm", "test"]
```

## 三、集成测试配置

```yaml
# docker-compose.test.yml
version: '3.8'
services:
  app:
    build: .
    environment:
      - DATABASE_URL=postgres://db:5432/testdb
    depends_on:
      db:
        condition: service_healthy
  
  db:
    image: postgres:15-alpine
    environment:
      - POSTGRES_DB=testdb
      - POSTGRES_PASSWORD=test
    healthcheck:
      test: ["CMD-SHELL", "pg_isready"]
      interval: 5s
      timeout: 5s
      retries: 5
```

## 四、CI集成

```yaml
test:
  script:
    - docker-compose -f docker-compose.test.yml up -d
    - docker-compose -f docker-compose.test.yml run app npm test
    - docker-compose -f docker-compose.test.yml down
```

## 五、注意事项

1. **环境隔离**：每个测试使用独立环境
2. **数据清理**：测试后清理测试数据
3. **健康检查**：确保服务就绪后才运行测试
4. **并行测试**：容器支持并行测试
