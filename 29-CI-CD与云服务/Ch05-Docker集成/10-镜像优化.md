# 镜像优化

## 一、基础镜像选择

```dockerfile
# 最小镜像选择
# Alpine - 最小，约5MB
FROM alpine:3.18

# Distroless - 无shell，更安全
FROM gcr.io/distroless/nodejs20-debian12

# Ubuntu - 功能完整，较大
FROM ubuntu:22.04
```

## 二、镜像大小优化

```dockerfile
# 1. 使用多阶段构建
FROM node:20-alpine AS builder
# ... 构建阶段

FROM node:20-alpine AS production
# ... 只复制必要文件

# 2. 合并RUN命令
RUN apk add --no-cache git && \
    npm install && \
    apk del git

# 3. 清理缓存
RUN apt-get update && \
    apt-get install -y package && \
    rm -rf /var/lib/apt/lists/*
```

## 三、优化对比

```bash
# 优化前
ubuntu:22.04 + full install → 1.2GB

# 优化后
alpine:3.18 + minimal install → 150MB

# 进一步优化

```

## 四、注意事项

1. **功能兼容**：确保选择的基础镜像功能够用
2. **安全更新**：定期更新基础镜像
3. **调试困难**：distroless镜像调试较困难
4. **测试验证**：优化后充分测试

## 七、使用Distroless

```dockerfile
# Node.js distroless
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production
COPY . .
RUN npm run build

FROM gcr.io/distroless/nodejs20-debian12
WORKDIR /app
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/node_modules ./node_modules
CMD ["dist/main.js"]
```

```dockerfile
# Go distroless
FROM golang:1.21 AS builder
WORKDIR /app
COPY go.mod go.sum ./
RUN go mod download
COPY . .
RUN CGO_ENABLED=0 GOOS=linux go build -ldflags="-s -w" -o /app/server .

FROM gcr.io/distroless/static-debian12
COPY --from=builder /app/server /server
ENTRYPOINT ["/server"]
```

## 八、镜像瘦身技巧

```dockerfile
# 技巧1: 去除调试符号
RUN go build -ldflags="-s -w" -o app .           # Go
RUN strip /usr/local/bin/app                      # C/C++
RUN cargo build --release                         # Rust

# 技巧2: 压缩资源
RUN find . -name "*.js" -exec gzip -9 {} \;
RUN find . -name "*.css" -exec gzip -9 {} \;

# 技巧3: 删除不需要的文件
RUN rm -rf /usr/share/doc /usr/share/man /usr/share/info
RUN apt-get purge -y --auto-remove apt-utils

# 技巧4: 合并RUN层
RUN apt-get update && \
    apt-get install -y --no-install-recommends package && \
    # 清理
    apt-get purge -y --auto-remove && \
    rm -rf /var/lib/apt/lists/*

# 技巧5: 使用--no-cache
RUN apk add --no-cache curl
RUN pip install --no-cache-dir -r requirements.txt
RUN npm ci --only=production && npm cache clean --force
```

## 九、镜像大小监控

```yaml
# CI中检查镜像大小
check-image-size:
  stage: quality
  script:
    - |
      IMAGE_SIZE=$(docker image inspect myapp:latest --format='{{.Size}}')
      MAX_SIZE=$((500 * 1024 * 1024))  # 500MB limit
      if [ "$IMAGE_SIZE" -gt "$MAX_SIZE" ]; then
        echo "ERROR: Image size ($IMAGE_SIZE bytes) exceeds limit ($MAX_SIZE bytes)"
        docker history myapp:latest
        exit 1
      fi
      echo "Image size: $((IMAGE_SIZE / 1024 / 1024))MB"
```
