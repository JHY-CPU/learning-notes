# 镜像仓库

## 一、Docker Hub

```bash
# 登录
docker login

# 推送镜像
docker tag myapp:latest username/myapp:latest
docker push username/myapp:latest
```

## 二、私有仓库

```bash
# 运行私有仓库
docker run -d -p 5000:5000 --name registry registry:2

# 推送到私有仓库
docker tag myapp:latest localhost:5000/myapp:latest
docker push localhost:5000/myapp:latest
```

## 三、云镜像仓库

```bash
# AWS ECR
aws ecr get-login-password | docker login --username AWS --password-stdin 123456.dkr.ecr.region.amazonaws.com

# GitHub Container Registry
echo $GITHUB_TOKEN | docker login ghcr.io -u username --password-stdin

# Google Container Registry
gcloud auth configure-docker
```

## 四、CI集成

```yaml
# GitHub Actions
- uses: docker/login-action@v3
  with:
    registry: ghcr.io
    username: ${{ github.actor }}
    password: ${{ secrets.GITHUB_TOKEN }}

- uses: docker/build-push-action@v5
  with:
    push: true
    tags: ghcr.io/${{ github.repository }}:${{ github.sha }}
```

## 五、注意事项

1. **认证安全**：使用Token而不是密码
2. **镜像扫描**：定期扫描镜像漏洞
3. **保留策略**：设置镜像保留策略
4. **跨区域复制**：配置跨区域镜像同步

## 六、Harbor私有仓库部署

```yaml
# docker-compose.yml - Harbor
version: '3.8'
services:
  harbor-core:
    image: goharbor/harbor-core:v2.10.0
    container_name: harbor-core
    env_file:
      - ./common/config/core/env
    volumes:
      - ./common/config/core/app.conf:/etc/core/app.conf
      - ./data/ca_download/:/etc/core/ca/
      - ./data/psc/:/etc/core/token/
    depends_on:
      - harbor-db
      - harbor-redis
    networks:
      - harbor

  harbor-db:
    image: goharbor/harbor-db:v2.10.0
    container_name: harbor-db
    volumes:
      - ./data/database:/var/lib/postgresql/data
    networks:
      - harbor

  harbor-redis:
    image: redis:7-alpine
    container_name: harbor-redis
    networks:
      - harbor

networks:
  harbor:
    driver: bridge
```

```bash
# Harbor CLI使用
docker login harbor.example.com
docker tag myapp:latest harbor.example.com/myproject/myapp:v1.0.0
docker push harbor.example.com/myproject/myapp:v1.0.0

# Harbor API
curl -X GET "https://harbor.example.com/api/v2.0/projects" \
  -H "Authorization: Basic $(echo -n admin:password | base64)"
```

## 七、镜像标签策略

```yaml
# 使用docker/metadata-action自动生成标签
- uses: docker/metadata-action@v5
  id: meta
  with:
    images: ghcr.io/${{ github.repository }}
    tags: |
      # 分支名
      type=ref,event=branch
      # PR号
      type=ref,event=pr
      # 语义化版本
      type=semver,pattern={{version}}
      type=semver,pattern={{major}}.{{minor}}
      type=semver,pattern={{major}}
      # Git SHA
      type=sha,prefix=
      type=sha,prefix=,format=long
      # 构建时间
      type=raw,value={{date 'YYYYMMDD-HHmmss'}}
      # latest标签（仅默认分支）
      type=raw,value=latest,enable={{is_default_branch}}
```

## 八、镜像同步与复制

```bash
# 使用skopeo复制镜像
skopeo copy \
  docker://harbor.example.com/myapp:v1.0.0 \
  docker://123456.dkr.ecr.us-east-1.amazonaws.com/myapp:v1.0.0

# 批量同步
for tag in v1.0.0 v1.1.0 v2.0.0; do
  skopeo copy \
    docker://harbor.example.com/myapp:$tag \
    docker://123456.dkr.ecr.us-east-1.amazonaws.com/myapp:$tag
done
```
