# 多阶段构建

## 一、概念说明

多阶段构建允许在单个Dockerfile中使用多个FROM语句，只复制需要的文件到最终镜像，大幅减小镜像大小。

## 二、Node.js多阶段构建

```dockerfile
# 构建阶段
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

# 生产阶段
FROM node:20-alpine AS production
WORKDIR /app
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/node_modules ./node_modules
EXPOSE 3000
CMD ["node", "dist/main.js"]
```

## 三、Go多阶段构建

```dockerfile
# 构建阶段
FROM golang:1.21 AS builder
WORKDIR /app
COPY . .
RUN CGO_ENABLED=0 go build -o main .

# 生产阶段
FROM alpine:3.18
WORKDIR /app
COPY --from=builder /app/main .
EXPOSE 8080
CMD ["./main"]
```

## 四、Java多阶段构建

```dockerfile
# 构建阶段
FROM maven:3.9-eclipse-temurin-17 AS builder
WORKDIR /app
COPY pom.xml .
RUN mvn dependency:go-offline
COPY src ./src
RUN mvn package -DskipTests

# 生产阶段
FROM eclipse-temurin:17-jre-alpine
WORKDIR /app
COPY --from=builder /app/target/*.jar app.jar
EXPOSE 8080
CMD ["java", "-jar", "app.jar"]
```

## 五、注意事项

1. **阶段命名**：使用AS给阶段命名
2. **选择性复制**：只复制需要的文件
3. **最终镜像**：使用最小基础镜像
4. **缓存优化**：合理安排COPY顺序

## 六、Rust多阶段构建

```dockerfile
# Rust应用的极致压缩构建
FROM rust:1.73 AS builder
WORKDIR /app
COPY Cargo.toml Cargo.lock ./
# 预编译依赖
RUN mkdir src && echo "fn main() {}" > src/main.rs && cargo build --release
# 编译实际应用
COPY src ./src
RUN cargo build --release

# 最终镜像 - 使用scratch（空镜像）
FROM scratch
COPY --from=builder /app/target/release/myapp /myapp
ENTRYPOINT ["/myapp"]
```

## 七、Python多阶段构建

```dockerfile
# Python应用多阶段构建
FROM python:3.12-slim AS builder
WORKDIR /app
RUN pip install --no-cache-dir --upgrade pip && \
    pip install --no-cache-dir poetry
COPY pyproject.toml poetry.lock ./
RUN poetry export -f requirements.txt --output requirements.txt --without-hashes
RUN pip install --no-cache-dir --prefix=/install -r requirements.txt

FROM python:3.12-slim AS production
WORKDIR /app
COPY --from=builder /install /usr/local
COPY . .
RUN adduser --disabled-password --gecos "" appuser
USER appuser
EXPOSE 8000
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

## 八、.NET多阶段构建

```dockerfile
# .NET应用多阶段构建
FROM mcr.microsoft.com/dotnet/sdk:8.0 AS build
WORKDIR /src
COPY *.csproj ./
RUN dotnet restore
COPY . .
RUN dotnet publish -c Release -o /app/publish

FROM mcr.microsoft.com/dotnet/aspnet:8.0 AS runtime
WORKDIR /app
COPY --from=build /app/publish .
EXPOSE 8080
ENTRYPOINT ["dotnet", "MyApp.dll"]
```

## 九、构建参数传递

```dockerfile
# 使用ARG传递构建时参数
ARG NODE_VERSION=20
FROM node:${NODE_VERSION}-alpine AS builder

ARG BUILD_ENV=production
ARG NPM_TOKEN

WORKDIR /app
RUN echo "//registry.npmjs.org/:_authToken=${NPM_TOKEN}" > .npmrc
COPY package*.json ./
RUN npm ci
RUN rm .npmrc

ENV NODE_ENV=${BUILD_ENV}
COPY . .
RUN npm run build
```

```yaml
# CI中传递构建参数
- uses: docker/build-push-action@v5
  with:
    build-args: |
      NODE_VERSION=20
      BUILD_ENV=production
      NPM_TOKEN=${{ secrets.NPM_TOKEN }}
```

## 十、多阶段构建优化对比

```
Node.js应用示例:
  单阶段构建:     node:20-alpine + 全部文件 = ~800MB
  多阶段构建:     只复制dist + 生产依赖 = ~150MB
  极致优化:       distroless + 只有二进制 = ~50MB

Go应用示例:
  单阶段构建:     golang:1.21 + 全部文件 = ~900MB
  多阶段构建:     alpine + 只有二进制 = ~20MB
  极致优化:       scratch + 只有二进制 = ~10MB
```
