# 安全事件响应

## 一、概念说明

安全事件响应（Incident Response）是处理安全事件的标准流程，包括检测、遏制、消除、恢复和复盘五个阶段。目标是最小化安全事件的影响。

| 阶段 | 活动 | 责任人 |
|------|------|--------|
| 检测 | 发现安全事件 | 安全监控 |
| 遏制 | 阻止事件扩大 | SRE/安全 |
| 消除 | 移除威胁 | 安全团队 |
| 恢复 | 恢复正常服务 | 运维团队 |
| 复盘 | 总结改进 | 全团队 |

## 二、具体用法

### 事件响应Playbook

```yaml
# security-incident-playbook.yml
Incident_Response_Procedure:

  1_检测与确认:
    - 收到告警/报告
    - 确认是否为真实事件
    - 评估严重程度 (P1/P2/P3/P4)
    - 通知相关人员

  2_遏制:
    - 隔离受影响系统
    - 阻断攻击来源
    - 保全证据（日志、内存dump）
    - 通知管理层

  3_消除:
    - 分析根因
    - 清除恶意代码/后门
    - 修复漏洞
    - 更新安全策略

  4_恢复:
    - 从备份恢复数据
    - 验证系统完整性
    - 逐步恢复服务
    - 加强监控

  5_复盘:
    - 时间线重建
    - 根因分析
    - 改进措施
    - 更新Playbook
```

### 自动化事件响应

```python
# 自动化安全事件响应
import boto3
import json

def lambda_handler(event, context):
    """安全事件自动响应"""

    # 从GuardDuty/CloudWatch接收事件
    event_type = event.get('detail-type')

    if event_type == 'GuardDuty Finding':
        finding = event['detail']
        severity = finding['severity']

        if severity >= 7.0:  # High/Critical
            # 立即遏制
            isolate_instance(finding['resource']['instanceDetails']['instanceId'])
            block_ip(finding['service']['action']['remoteIpDetails']['ipAddressV4'])
            notify_security_team(finding)

        elif severity >= 4.0:  # Medium
            # 增强监控
            enable_enhanced_monitoring(finding)
            create_ticket(finding)

    return {'statusCode': 200}

def isolate_instance(instance_id):
    """隔离受感染实例"""
    ec2 = boto3.client('ec2')

    # 创建隔离安全组
    sg_id = create_isolation_sg()

    # 修改实例安全组
    ec2.modify_instance_attribute(
        InstanceId=instance_id,
        Groups=[sg_id]
    )

    # 创建快照保存证据
    ec2.create_snapshots(
        InstanceSpecification={'InstanceId': instance_id},
        Description=f'Incident response snapshot for {instance_id}'
    )

    print(f'Instance {instance_id} isolated')

def block_ip(ip_address):
    """封锁恶意IP"""
    waf = boto3.client('wafv2')
    # 添加到WAF黑名单
    waf.update_ip_set(
        Name='blocked-ips',
        Scope='REGIONAL',
        Id='ip-set-id',
        LockToken='token',
        Addresses=[f'{ip_address}/32']
    )

def notify_security_team(finding):
    """通知安全团队"""
    sns = boto3.client('sns')
    message = {
        'severity': finding['severity'],
        'type': finding['type'],
        'description': finding['description'],
        'resource': finding['resource'],
        'action_required': 'IMMEDIATE'
    }
    sns.publish(
        TopicArn='arn:aws:sns:us-east-1:123456789012:security-alerts',
        Subject=f"[CRITICAL] Security Incident: {finding['type']}",
        Message=json.dumps(message, indent=2)
    )
```

### 事件分级标准

```yaml
Event_Severity:
  P1_Critical:
    定义: 数据泄露、系统被入侵、服务完全不可用
    响应时间: 15分钟
    响应人员: 安全团队全员 + CTO
    升级时间: 立即
    示例:
      - 数据库被拖库
      - 生产服务器被入侵
      - 勒索软件攻击

  P2_High:
    定义: 未授权访问、服务部分不可用
    响应时间: 30分钟
    响应人员: 安全团队 + 相关SRE
    升级时间: 1小时
    示例:
      - 账号被盗
      - DDoS攻击
      - 恶意软件检测

  P3_Medium:
    定义: 安全策略违反、潜在威胁
    响应时间: 4小时
    响应人员: 安全团队
    升级时间: 24小时
    示例:
      - 异常登录
      - 配置错误
      - 漏洞利用尝试

  P4_Low:
    定义: 安全建议、信息性事件
    响应时间: 24小时
    响应人员: 安全团队
    升级时间: 不升级
    示例:
      - 安全扫描发现
      - 合规建议
```

## 三、注意事项与常见陷阱

1. **快速响应**：安全事件需要分钟级响应
2. **证据保全**：在修复前保存所有证据
3. **沟通协调**：及时通知所有相关方
4. **不要恐慌**：按Playbook流程执行
5. **复盘改进**：每次事件都要复盘
6. **演练验证**：定期进行安全演练
7. **保密处理**：安全事件信息要保密
