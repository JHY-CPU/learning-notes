# 容器安全

## 一、概念说明

容器安全涵盖从镜像构建、存储、分发到运行时的全生命周期安全管理。目标是最小化攻击面并确保容器运行环境的安全。

| 阶段 | 安全实践 |
|------|----------|
| 构建 | 最小化镜像、非root用户、安全基础镜像 |
| 存储 | 镜像签名、漏洞扫描、访问控制 |
| 运行 | 运行时保护、网络策略、资源限制 |

## 二、具体用法

### 安全Dockerfile

```dockerfile
# 安全Dockerfile最佳实践
# 1. 使用特定版本的基础镜像
FROM python:3.11.7-slim-bookworm AS builder

# 2. 不以root运行
RUN groupadd -r appuser && useradd -r -g appuser appuser

# 3. 安装依赖（最小化）
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 4. 复制应用代码
WORKDIR /app
COPY --chown=appuser:appuser . .

# 5. 使用非root用户
USER appuser

# 6. 不暴露敏感端口
EXPOSE 8080

# 7. 使用ENTRYPOINT而不是CMD
ENTRYPOINT ["python"]
CMD ["app.py"]

# 8. 添加HEALTHCHECK
HEALTHCHECK --interval=30s --timeout=3s --retries=3 \
    CMD curl -f http://localhost:8080/health || exit 1
```

### 镜像安全扫描

```bash
# Trivy扫描
trivy image myapp:latest
trivy image --severity HIGH,CRITICAL --exit-code 1 myapp:latest

# 扫描Dockerfile
trivy config Dockerfile

# 扫描文件系统
trivy fs --security-checks vuln,secret,config .

# Grype扫描
grype myapp:latest

# Docker Scout
docker scout cves myapp:latest
docker scout quickview myapp:latest
```

### Kubernetes安全策略

```yaml
# Pod安全标准
apiVersion: v1
kind: Pod
metadata:
  name: secure-pod
spec:
  securityContext:
    runAsNonRoot: true
    runAsUser: 1000
    runAsGroup: 1000
    fsGroup: 1000
    seccompProfile:
      type: RuntimeDefault
  containers:
  - name: app
    image: myapp:1.0
    securityContext:
      allowPrivilegeEscalation: false
      readOnlyRootFilesystem: true
      capabilities:
        drop:
          - ALL
    resources:
      requests:
        cpu: "100m"
        memory: "128Mi"
      limits:
        cpu: "500m"
        memory: "512Mi"
    volumeMounts:
    - name: tmp
      mountPath: /tmp
  volumes:
  - name: tmp
    emptyDir: {}
```

### 网络策略

```yaml
# 限制Pod网络访问
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: web-app-netpol
  namespace: production
spec:
  podSelector:
    matchLabels:
      app: web-app
  policyTypes:
  - Ingress
  - Egress
  ingress:
  - from:
    - namespaceSelector:
        matchLabels:
          name: ingress-nginx
    ports:
    - protocol: TCP
      port: 8080
  egress:
  - to:
    - podSelector:
        matchLabels:
          app: database
    ports:
    - protocol: TCP
      port: 5432
  - to:
    - namespaceSelector: {}
    ports:
    - protocol: UDP
      port: 53  # DNS
```

### OPA/Gatekeeper策略

```yaml
# 禁止特权容器
apiVersion: templates.gatekeeper.sh/v1
kind: ConstraintTemplate
metadata:
  name: k8spspprivileged
spec:
  crd:
    spec:
      names:
        kind: K8sPSPPrivileged
  targets:
    - target: admission.k8s.gatekeeper.sh
      rego: |
        package k8spspprivileged
        violation[{"msg": msg}] {
            container := input.review.object.spec.containers[_]
            container.securityContext.privileged == true
            msg := sprintf("Privileged container not allowed: %v", [container.name])
        }

---
# 禁止latest标签
apiVersion: templates.gatekeeper.sh/v1
kind: ConstraintTemplate
metadata:
  name: k8srequiredimagetag
spec:
  targets:
    - target: admission.k8s.gatekeeper.sh
      rego: |
        package k8srequiredimagetag
        violation[{"msg": msg}] {
            container := input.review.object.spec.containers[_]
            not contains(container.image, ":")
            msg := sprintf("Container %v must use image with tag, not latest", [container.name])
        }
```

### 运行时安全（Falco）

```yaml
# falco_rules.yaml
- rule: Terminal Shell in Container
  desc: Detect shell spawned in container
  condition: >
    spawned_process and container
    and shell_procs
    and proc.pname in (shell_binaries)
    and not user_known_terminal_shell_in_container_conditions
  output: >
    Shell spawned in container (user=%user.name container=%container.name
    shell=%proc.name parent=%proc.pname command=%proc.cmdline)
  priority: WARNING
  tags: [container, shell, mitre_execution]

- rule: Write Below Binary Dir
  desc: Detect writes below system binary directories
  condition: >
    open_write and container
    and fd.directory in (/bin, /sbin, /usr/bin, /usr/sbin)
  output: >
    File written below system binary dir (user=%user.name file=%fd.name)
  priority: ERROR
```

## 三、注意事项与常见陷阱

1. **基础镜像**：使用官方最小化镜像，定期更新
2. **Root权限**：永远不要以root运行容器
3. **敏感数据**：不要在镜像中硬编码密钥
4. **资源限制**：必须设置CPU和内存限制
5. **网络隔离**：使用NetworkPolicy限制容器间通信
6. **镜像签名**：使用Cosign/Notary签名验证镜像
7. **运行时监控**：部署Falco等工具监控异常行为
