# 合规与审计

## 一、概念说明

合规与审计确保系统满足安全标准和法规要求。通过持续监控、日志记录和自动化检查，证明系统符合相关合规框架。

| 框架 | 适用领域 | 要点 |
|------|----------|------|
| SOC 2 | SaaS服务 | 安全、可用性、隐私 |
| GDPR | 欧盟用户数据 | 数据保护、隐私权 |
| 等保2.0 | 中国信息系统 | 分级保护 |
| PCI DSS | 支付卡数据 | 支付安全 |
| HIPAA | 医疗信息 | 健康数据保护 |

## 二、具体用法

### AWS Config合规检查

```bash
# 启用AWS Config
aws configservice put-configuration-recorder \
    --configuration-recorder name=default,roleArn=arn:aws:iam::role/aws-service-role

# 启用合规规则
aws configservice put-config-rule --config-rule '{
    "ConfigRuleName": "encrypted-volumes",
    "Source": {
        "Owner": "AWS",
        "SourceIdentifier": "ENCRYPTED_VOLUMES"
    }
}'

aws configservice put-config-rule --config-rule '{
    "ConfigRuleName": "rds-storage-encrypted",
    "Source": {
        "Owner": "AWS",
        "SourceIdentifier": "RDS_STORAGE_ENCRYPTED"
    }
}'

# 查看合规状态
aws configservice get-compliance-details-by-config-rule \
    --config-rule-name encrypted-volumes
```

### OPA合规策略

```rego
# compliance.rego
package compliance

# SOC 2: 所有存储必须加密
violations["storage_not_encrypted"] {
    resource := input.resources[_]
    resource.type == "aws_ebs_volume"
    not resource.encrypted
}

violations["s3_not_encrypted"] {
    resource := input.resources[_]
    resource.type == "aws_s3_bucket"
    not resource.server_side_encryption_configuration
}

# GDPR: 数据必须可删除
violations["no_data_retention_policy"] {
    resource := input.resources[_]
    resource.type == "aws_s3_bucket"
    not resource.lifecycle_rule
}

# 等保2.0: 日志必须保留180天以上
violations["log_retention_too_short"] {
    resource := input.resources[_]
    resource.type == "aws_cloudwatch_log_group"
    resource.retention_in_days < 180
}

# 审计: 所有操作必须有日志
violations["no_cloudtrail"] {
    count([r | r := input.resources[_]; r.type == "aws_cloudtrail"]) == 0
}
```

### GitOps审计日志

```yaml
# ArgoCD审计
apiVersion: v1
kind: ConfigMap
metadata:
  name: argocd-cm
data:
  # 审计级别
  accounts.pipeline: apiKey
  admin.enabled: "false"

---
# Kubernetes审计策略
apiVersion: audit.k8s.io/v1
kind: Policy
rules:
  # 记录所有Secret访问
  - level: RequestResponse
    resources:
    - group: ""
      resources: ["secrets"]
    verbs: ["get", "list", "create", "update", "delete"]

  # 记录所有RBAC变更
  - level: RequestResponse
    resources:
    - group: "rbac.authorization.k8s.io"
      resources: ["roles", "rolebindings", "clusterroles", "clusterrolebindings"]

  # 记录所有Deployment变更
  - level: Request
    resources:
    - group: "apps"
      resources: ["deployments"]
    verbs: ["create", "update", "patch", "delete"]

  # Metadata级别记录其他操作
  - level: Metadata
    omitStages:
    - RequestReceived
```

### Checkov IaC合规扫描


## 三、注意事项与常见陷阱

1. **持续合规**：合规不是一次性工作，需要持续监控
2. **自动化检查**：使用工具自动化合规检查
3. **审计日志**：日志必须包含who/what/when/where
4. **数据保留**：根据法规要求设置日志保留期
5. **证据收集**：自动化收集合规证据
6. **合规培训**：团队需要了解相关合规要求
7. **定期审查**：定期审查合规策略和控制措施
