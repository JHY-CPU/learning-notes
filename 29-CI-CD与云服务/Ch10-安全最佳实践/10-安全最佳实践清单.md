# 安全最佳实践清单

## 一、概念说明

安全最佳实践清单是DevSecOps全流程的安全检查表，覆盖代码、构建、部署、运行各阶段的安全要求。

## 二、具体用法

### 代码阶段

```yaml
代码安全:
  - [ ] 代码审查包含安全审查
  - [ ] 使用SAST工具扫描代码漏洞
  - [ ] 依赖项漏洞扫描（SCA）
  - [ ] 密钥泄露检测（GitLeaks）
  - [ ] 安全编码规范培训
  - [ ] 敏感数据不硬编码
  - [ ] 输入验证和输出编码
  - [ ] 使用参数化查询防SQL注入
  - [ ] CSRF/XSS防护
  - [ ] 错误处理不泄露敏感信息
```

### 构建阶段

```yaml
构建安全:
  - [ ] 使用可信的基础镜像
  - [ ] 多阶段构建最小化镜像
  - [ ] 容器镜像漏洞扫描
  - [ ] Dockerfile安全检查
  - [ ] 镜像签名和验证
  - [ ] 不在镜像中存储密钥
  - [ ] 以非root用户运行
  - [ ] 只读文件系统
  - [ ] 移除不必要的工具
  - [ ] 构建环境安全加固
```

### 部署阶段

```yaml
部署安全:
  - [ ] Kubernetes RBAC最小权限
  - [ ] Pod安全策略/标准
  - [ ] NetworkPolicy网络隔离
  - [ ] 资源限制配置
  - [ ] Secret外部管理（Vault等）
  - [ ] TLS/mTLS加密通信
  - [ ] 健康检查配置
  - [ ] 安全上下文配置
  - [ ] 镜像拉取策略限制
  - [ ] 部署审计日志
```

### 运行阶段

```yaml
运行时安全:
  - [ ] 运行时安全监控（Falco）
  - [ ] 日志收集和分析
  - [ ] 入侵检测系统
  - [ ] DDoS防护
  - [ ] WAF配置
  - [ ] 安全事件响应流程
  - [ ] 定期漏洞扫描
  - [ ] 密钥轮换
  - [ ] 访问权限审查
  - [ ] 安全补丁及时更新
```

### 合规与治理

```yaml

  - [ ] 安全策略文档化
  - [ ] 合规框架映射
  - [ ] 审计日志保留（180天+）

  - [ ] 隐私保护措施
  - [ ] 安全培训计划
  - [ ] 第三方安全评估
  - [ ] 安全预算和资源
  - [ ] 供应商安全评估
  - [ ] 安全指标监控
```

### CI/CD安全

```yaml
  - [ ] 最小化CI/CD权限
  - [ ] 使用OIDC而非长期密钥

  - [ ] 安全门禁自动化
```

### 基础设施安全

```yaml
基础设施:
  - [ ] 网络分段和隔离
  - [ ] 安全组最小开放
  - [ ] 加密存储（EBS/S3/RDS）

  - [ ] 安全基线加固
```

### 监控与告警

```yaml
安全监控:
  - [ ] 异常登录检测
  - [ ] API滥用检测

```

## 三、注意事项与常见陷阱

1. **持续改进**：安全检查清单需要定期更新
2. **工具整合**：尽量使用统一的安全平台
3. **文化建设**：安全是每个人的责任，不只是安全团队
4. **自动化优先**：能自动化的安全检查尽量自动化
5. **成本平衡**：安全投入要和业务风险匹配
6. **演练验证**：定期进行安全演练验证有效性
7. **持续学习**：关注最新安全威胁和最佳实践
