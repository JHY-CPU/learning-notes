# 密钥管理

## 一、概念说明

密钥管理（Secrets Management）是对应用程序中的敏感信息（密码、API密钥、证书等）进行安全存储、访问控制和轮换管理的实践。

| 工具 | 特点 | 适用场景 |
|------|------|----------|
| Vault | 功能丰富、动态Secret | 企业级 |
| AWS Secrets Manager | 托管服务 | AWS环境 |
| Sealed Secrets | K8s原生 | K8s部署 |
| SOPS | 文件加密 | GitOps |
| External Secrets | K8s外部集成 | 多Secret后端 |

## 二、具体用法

### HashiCorp Vault

```bash
# 启动开发模式
vault server -dev

# 初始化和解封
vault operator init -key-shares=5 -key-threshold=3
vault operator unseal

# 写入Secret
vault kv put secret/myapp/database \
    host=db.example.com \
    username=admin \
    password=StrongPass123

# 读取Secret
vault kv get secret/myapp/database

# 创建策略
vault policy write myapp-policy - <<EOF
path "secret/data/myapp/*" {
    capabilities = ["read", "list"]
}
EOF

# 创建AppRole
vault auth enable approle
vault write auth/approle/role/myapp \
    policies=myapp-policy \
    token_ttl=1h \
    token_max_ttl=4h
```

### Kubernetes集成Vault

```yaml
# Vault Agent Injector注解
apiVersion: apps/v1
kind: Deployment
metadata:
  name: web-app
spec:
  template:
    metadata:
      annotations:
        vault.hashicorp.com/agent-inject: "true"
        vault.hashicorp.com/role: "myapp"
        vault.hashicorp.com/agent-inject-secret-db: "secret/data/myapp/database"
        vault.hashicorp.com/agent-inject-template-db: |
          {{- with secret "secret/data/myapp/database" -}}
          export DB_HOST="{{ .Data.data.host }}"
          export DB_USER="{{ .Data.data.username }}"
          export DB_PASS="{{ .Data.data.password }}"
          {{- end -}}
    spec:
      serviceAccountName: myapp
      containers:
      - name: app
        image: myapp:1.0
        command: ["/bin/sh", "-c"]
        args:
          - source /vault/secrets/db && python app.py
```

### Sealed Secrets

```bash
# 安装Sealed Secrets Controller
kubectl apply -f https://github.com/bitnami-labs/sealed-secrets/releases/download/v0.24.0/controller.yaml

# 安装kubeseal
brew install kubeseal

# 创建密封Secret
kubectl create secret generic db-secret \
    --from-literal=password=StrongPass123 \
    --dry-run=client -o yaml | \
    kubeseal --format yaml > sealed-secret.yaml
```

```yaml
# sealed-secret.yaml (可安全提交到Git)
apiVersion: bitnami.com/v1alpha1
kind: SealedSecret
metadata:
  name: db-secret
  namespace: production
spec:
  encryptedData:
    password: AgBy3i4OJSWK+PiTySYZZA9rO... (加密内容)
  template:
    metadata:
      name: db-secret
      namespace: production
```

### SOPS加密

```yaml
# .sops.yaml
creation_rules:
  - path_regex: secrets/.*\.yaml$
    kms: arn:aws:kms:us-east-1:123456789012:key/abcd-1234
    pgp: FINGERPRINT

  - path_regex: overlays/production/.*secret.*
    kms: arn:aws:kms:us-east-1:123456789012:key/abcd-1234
```

```bash
# 加密文件
sops --encrypt --in-place secrets.yaml

# 解密文件
sops --decrypt secrets.yaml

# 直接使用（自动加解密）
sops exec-env secrets.yaml 'kubectl apply -f deployment.yaml'
```

### 外部Secrets Operator

```yaml
# ExternalSecret从AWS Secrets Manager同步
apiVersion: external-secrets.io/v1beta1
kind: ExternalSecret
metadata:
  name: db-credentials
  namespace: production
spec:
  refreshInterval: 1h
  secretStoreRef:
    name: aws-secret-store
    kind: SecretStore
  target:
    name: db-credentials
    creationPolicy: Owner
  data:
  - secretKey: host
    remoteRef:
      key: production/database
      property: host
  - secretKey: username
    remoteRef:
      key: production/database
      property: username
  - secretKey: password
    remoteRef:
      key: production/database
      property: password

---
apiVersion: external-secrets.io/v1beta1
kind: SecretStore
metadata:
  name: aws-secret-store
  namespace: production
spec:
  provider:
    aws:
      service: SecretsManager
      region: us-east-1
      auth:
        jwt:
          serviceAccountRef:
            name: external-secrets-sa
```

### 密钥轮换

```python
# Python自动轮换数据库密码
import boto3
import psycopg2
from functools import lru_cache

secrets_client = boto3.client('secretsmanager')

@lru_cache(maxsize=1)
def get_db_password(secret_id):
    """从Secrets Manager获取数据库密码（自动缓存）"""
    response = secrets_client.get_secret_value(SecretId=secret_id)
    return response['SecretString']

def get_db_connection():
    password = get_db_password('prod/db/password')
    return psycopg2.connect(
        host='db.example.com',
        user='app_user',
        password=password,
        database='myapp'
    )

# AWS Secrets Manager自动轮换
# Lambda函数实现密码轮换
def lambda_handler(event, context):
    secret_id = event['SecretId']
    step = event['Step']

    if step == "createSecret":
        # 生成新密码并存储
        new_password = generate_password()
        secrets_client.put_secret_value(
            SecretId=secret_id,
            SecretString=new_password,
            VersionStages=['AWSPENDING']
        )
    elif step == "setSecret":
        # 在数据库中更新密码
        update_db_password(new_password)
    elif step == "testSecret":
        # 验证新密码可用
        test_connection(new_password)
    elif step == "finishSecret":
        # 将新密码设为当前版本
        pass
```

## 三、注意事项与常见陷阱

1. **绝不硬编码**：密钥不能写在代码或Dockerfile中
2. **Git保护**：使用.gitignore排除密钥文件
3. **最小权限**：应用程序只获取需要的密钥
4. **定期轮换**：设置自动轮换策略
5. **审计日志**：记录所有密钥访问
6. **加密传输**：密钥通过TLS传输
7. **备份策略**：加密备份密钥配置
