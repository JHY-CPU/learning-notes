# 漏洞管理

## 一、概念说明

漏洞管理是对系统中已知安全漏洞进行发现、评估、修复和验证的持续过程。包括操作系统漏洞、应用漏洞、依赖库漏洞等。

| 阶段 | 活动 | 工具 |
|------|------|------|
| 发现 | 漏洞扫描 | Trivy、Nessus |
| 评估 | 风险评分 | CVSS、EPSS |
| 修复 | 补丁/升级 | 自动化工具 |
| 验证 | 修复确认 | 重新扫描 |

## 二、具体用法

### 漏洞扫描自动化

```yaml
# GitHub Actions漏洞扫描
name: Vulnerability Scan
on:
  schedule:
    - cron: '0 2 * * *'  # 每天凌晨2点
  push:
    branches: [main]

jobs:
  scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      # 容器镜像扫描
      - name: Trivy image scan
        uses: aquasecurity/trivy-action@master
        with:
          image-ref: myregistry/web-app:latest
          format: sarif
          output: trivy-results.sarif
          severity: CRITICAL,HIGH,MEDIUM

      # 文件系统扫描
      - name: Trivy fs scan
        uses: aquasecurity/trivy-action@master
        with:
          scan-type: fs
          scan-ref: .
          format: table

      # 上传结果到GitHub Security
      - name: Upload to GitHub Security
        uses: github/codeql-action/upload-sarif@v2
        with:
          sarif_file: trivy-results.sarif
```

### 漏洞跟踪和修复

```python
# 漏洞扫描结果处理
import json
from dataclasses import dataclass
from typing import List

@dataclass
class Vulnerability:
    cve_id: str
    severity: str
    package: str
    installed_version: str
    fixed_version: str
    description: str

def scan_and_report(image_name: str) -> List[Vulnerability]:
    """扫描镜像并生成报告"""
    import subprocess
    result = subprocess.run(
        ['trivy', 'image', '-f', 'json', image_name],
        capture_output=True, text=True
    )
    data = json.loads(result.stdout)

    vulns = []
    for result in data.get('Results', []):
        for vuln in result.get('Vulnerabilities', []):
            v = Vulnerability(
                cve_id=vuln['VulnerabilityID'],
                severity=vuln['Severity'],
                package=vuln['PkgName'],
                installed_version=vuln['InstalledVersion'],
                fixed_version=vuln.get('FixedVersion', 'N/A'),
                description=vuln.get('Description', '')[:200]
            )
            vulns.append(v)

    return vulns

def auto_fix(vulns: List[Vulnerability]):
    """自动修复漏洞"""
    critical_vulns = [v for v in vulns if v.severity == 'CRITICAL']

    if critical_vulns:
        # 触发自动升级
        packages = [v.package for v in critical_vulns if v.fixed_version != 'N/A']
        upgrade_command = f"pip install --upgrade {' '.join(packages)}"
        print(f"自动修复: {upgrade_command}")

        # 创建修复PR
        create_fix_pr(packages)
```

## 三、注意事项与常见陷阱

1. **优先级**：CVSS分数不是唯一标准，要考虑可利用性
2. **基线管理**：建立漏洞基线，跟踪修复进度
3. **误报处理**：建立白名单机制处理误报
4. **供应链安全**：关注上游依赖的漏洞
5. **零日漏洞**：关注CVE公告和安全通告
6. **修复验证**：修复后必须重新扫描确认
7. **趋势分析**：监控漏洞数量趋势，评估安全态势
