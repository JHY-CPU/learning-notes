# 代码安全扫描

## 一、概念说明

代码安全扫描（SAST，Static Application Security Testing）在不运行代码的情况下分析源代码，发现潜在的安全漏洞。包括SQL注入、XSS、命令注入等常见漏洞。

| 扫描类型 | 检测内容 | 工具 |
|----------|----------|------|
| SAST | 源代码漏洞 | SonarQube、Semgrep |
| SCA | 依赖漏洞 | Snyk、Dependabot |
| Secret | 密钥泄露 | GitLeaks、TruffleHog |
| IaC | 基础设施配置 | Checkov、tfsec |

## 二、具体用法

### SonarQube配置

```yaml
# sonar-project.properties
sonar.projectKey=my-project
sonar.sources=src
sonar.tests=tests
sonar.python.coverage.reportPaths=coverage.xml
sonar.exclusions=**/vendor/**,**/test/**

# Quality Gate配置
sonar.qualitygate.wait=true
sonar.coverage.minimum=80
```

### Semgrep规则

```yaml
# .semgrep.yml
rules:
  - id: sql-injection
    patterns:
      - pattern: |
          $QUERY = "SELECT * FROM users WHERE id = " + $INPUT
    message: "Potential SQL injection vulnerability"
    severity: ERROR
    languages: [python, java, javascript]

  - id: hardcoded-secret
    patterns:
      - pattern-either:
        - pattern: $X = "sk-..."
        - pattern: API_KEY = "..."
        - pattern: PASSWORD = "..."
    message: "Hardcoded secret detected"
    severity: WARNING

  - id: xss-vulnerability
    patterns:
      - pattern: |
          document.innerHTML = $INPUT
    message: "Potential XSS vulnerability"
    severity: ERROR
    languages: [javascript]
```

```bash
# 运行Semgrep
semgrep --config=p/owasp-top-ten src/
semgrep --config=p/secrets src/
semgrep --config=.semgrep.yml src/
```

### GitLeaks密钥扫描

```yaml
# .gitleaks.toml
[extend]
useDefault = true

[[rules]]
id = "custom-api-key"
description = "Custom API Key"
regex = '''api_key\s*=\s*['"]([a-zA-Z0-9]{32,})['"]'''
tags = ["key", "api"]

[[rules]]
id = "private-key"
description = "Private Key"
regex = '''-----BEGIN (RSA |EC )?PRIVATE KEY-----'''
tags = ["key", "private"]

[allowlist]
paths = [
    '''test/.*''',
    '''docs/.*'''
]
```

```bash
# 扫描整个仓库
gitleaks detect --source . --report-format json --report-path report.json

# 扫描commit历史
gitleaks detect --source . --log-opts="--all"

# pre-commit hook
gitleaks protect --staged
```

### Python Bandit扫描

```bash
# 安装和使用Bandit
pip install bandit

# 扫描Python代码
bandit -r src/ -f json -o bandit-report.json

# 配置
# .bandit
exclude_dirs: ['tests']
skips: ['B101']  # 跳过assert检查
tests: ['B201', 'B301']  # 只运行特定测试
```

```yaml
# GitHub Actions集成
- name: Run Bandit
  run: |
    pip install bandit
    bandit -r src/ -f json -o bandit-report.json
    bandit -r src/ -ll  # 仅显示中高级别问题
```

### 依赖漏洞扫描

```yaml
# Snyk配置 .snyk
version: v1.25.0
language-settings:
  python:
    target-file: requirements.txt

ignore:
  'SNYK-PYTHON-REQUESTS-1234567':
    - '*':
        reason: 'Not exploitable in our usage'
        expires: '2024-12-31T00:00:00.000Z'
```

```bash
# Snyk CLI
snyk test                  # 测试依赖漏洞
snyk code test            # SAST代码扫描
snyk container test myimage:latest  # 容器扫描
snyk iac scan terraform/  # IaC扫描
snyk monitor              # 持续监控
```

### Pre-commit Hooks

```yaml
# .pre-commit-config.yaml
repos:
  - repo: https://github.com/gitleaks/gitleaks
    rev: v8.18.0
    hooks:
      - id: gitleaks

  - repo: https://github.com/returntocorp/semgrep
    rev: v1.45.0
    hooks:
      - id: semgrep
        args: ['--config', 'p/owasp-top-ten', '--error']

  - repo: https://github.com/PyCQA/bandit
    rev: 1.7.5
    hooks:
      - id: bandit
        args: ['-ll']

  - repo: https://github.com/Yelp/detect-secrets
    rev: v1.4.0
    hooks:
      - id: detect-secrets
        args: ['baseline', '.secrets.baseline']
```

## 三、注意事项与常见陷阱

1. **误报处理**：建立误报白名单，定期审查
2. **扫描速度**：增量扫描只检查变更文件
3. **规则更新**：定期更新扫描规则和漏洞库
4. **CI集成**：扫描失败阻断流水线
5. **团队规范**：统一安全编码规范
6. **历史清理**：定期扫描历史代码
7. **工具选择**：根据技术栈选择合适的扫描工具
