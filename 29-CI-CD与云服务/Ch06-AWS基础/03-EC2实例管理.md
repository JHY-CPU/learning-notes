# EC2实例管理

## 一、概念说明

EC2（Elastic Compute Cloud）是AWS的虚拟服务器服务，提供可扩展的计算容量。支持多种实例类型，适用于不同工作负载。

| 实例系列 | 用途 | 示例 |
|----------|------|------|
| 通用型 | 均衡计算/内存/网络 | t3, m5 |
| 计算优化 | 高性能CPU | c5, c6g |
| 内存优化 | 大内存工作负载 | r5, x1e |
| 存储优化 | 高I/O吞吐 | i3, d2 |
| 加速计算 | GPU/FPGA | p3, f1 |

## 二、具体用法

### 启动实例

```bash
# 启动EC2实例
aws ec2 run-instances \
    --image-id ami-0c55b159cbfafe1f0 \
    --instance-type t2.micro \
    --key-name my-key-pair \
    --security-group-ids sg-12345678 \
    --subnet-id subnet-12345678 \
    --tag-specifications 'ResourceType=instance,Tags=[{Key=Name,Value=WebServer}]'

# 查看实例状态
aws ec2 describe-instances \
    --instance-ids i-1234567890abcdef0 \
    --query 'Reservations[*].Instances[*].[InstanceId,State.Name,PublicIpAddress]'
```

### 实例操作

```bash
# 启动/停止/重启实例
aws ec2 start-instances --instance-ids i-1234567890abcdef0
aws ec2 stop-instances --instance-ids i-1234567890abcdef0
aws ec2 reboot-instances --instance-ids i-1234567890abcdef0

# 创建AMI镜像
aws ec2 create-image \
    --instance-id i-1234567890abcdef0 \
    --name "MyWebServer-Backup" \
    --description "Web服务器备份镜像"
```

### 用户数据脚本

```bash
#!/bin/bash
# 启动时自动执行的脚本
yum update -y
yum install -y httpd
systemctl start httpd
systemctl enable httpd
echo "<h1>Hello from EC2</h1>" > /var/www/html/index.html
```

### 弹性IP

```bash
# 分配弹性IP
aws ec2 allocate-address --domain vpc

# 关联弹性IP到实例
aws ec2 associate-address \
    --instance-id i-1234567890abcdef0 \
    --allocation-id eipalloc-12345678
```

## 三、注意事项与常见陷阱

1. **实例类型选择**：根据工作负载选择合适类型，避免资源浪费
2. **安全组配置**：仅开放必要端口，避免0.0.0.0/0开放SSH
3. **密钥对管理**：妥善保管密钥对文件，丢失无法恢复
4. **弹性IP费用**：未关联实例的弹性IP会收费
5. **实例存储数据**：实例停止后实例存储数据会丢失
6. **自动伸缩**：配合Auto Scaling Group实现弹性扩展
7. **预留实例**：长期使用考虑预留实例节省成本
