# 工作流语法

## 一、概念说明

GitHub Actions工作流使用YAML语法定义，包括触发条件、任务、步骤等。

## 二、触发条件

```yaml
# 推送到main分支
on:
  push:
    branches: [main]
    paths: ['src/**']

# Pull Request
on:
  pull_request:
    branches: [main]
    types: [opened, synchronize]

# 定时触发
on:
  schedule:
    - cron: '0 2 * * *'

# 手动触发
on: workflow_dispatch

# 多事件触发
on: [push, pull_request]
```

## 三、环境变量

```yaml
env:
  NODE_ENV: production
  API_URL: https://api.example.com

jobs:
  build:
    env:
      DEBUG: false
    steps:
      - run: echo $NODE_ENV
      - run: echo ${{ env.API_URL }}
```

## 四、条件执行

```yaml
jobs:
  deploy:
    if: github.ref == 'refs/heads/main'
    runs-on: ubuntu-latest
    steps:
      - run: echo "Deploying..."
```

## 五、矩阵构建

```yaml
strategy:
  matrix:
    node: [16, 18, 20]
    os: [ubuntu-latest, windows-latest]
runs-on: ${{ matrix.os }}
steps:
  - uses: actions/setup-node@v4
    with:
      node-version: ${{ matrix.node }}
```

## 六、注意事项

1. **YAML缩进**：严格使用2空格缩进
2. **表达式**：使用${{ }}包裹表达式
3. **上下文**：github、env、matrix等上下文
4. **Secrets**：secrets不能在if条件中直接使用
