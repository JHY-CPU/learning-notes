# 工作流语法

## 一、概念说明

GitHub Actions工作流使用YAML语法定义，包括触发条件、任务、步骤等。

## 二、触发条件

```yaml
# 推送到main分支
on:
  push:
    branches: [main]
    paths: ['src/**']

# Pull Request
on:
  pull_request:
    branches: [main]
    types: [opened, synchronize]

# 定时触发
on:
  schedule:
    - cron: '0 2 * * *'

# 手动触发
on: workflow_dispatch

# 多事件触发
on: [push, pull_request]
```

## 三、环境变量

```yaml
env:
  NODE_ENV: production
  API_URL: https://api.example.com

jobs:
  build:
    env:
      DEBUG: false
    steps:
      - run: echo $NODE_ENV
      - run: echo ${{ env.API_URL }}
```

## 四、条件执行

```yaml
jobs:
  deploy:
    if: github.ref == 'refs/heads/main'
    runs-on: ubuntu-latest
    steps:
      - run: echo "Deploying..."
```

## 五、矩阵构建

```yaml
strategy:
  matrix:
    node: [16, 18, 20]
    os: [ubuntu-latest, windows-latest]
runs-on: ${{ matrix.os }}
steps:
  - uses: actions/setup-node@v4
    with:
      node-version: ${{ matrix.node }}
```

## 六、注意事项

1. **YAML缩进**：严格使用2空格缩进
2. **表达式**：使用${{ }}包裹表达式
3. **上下文**：github、env、matrix等上下文
4. **Secrets**：secrets不能在if条件中直接使用

## 七、上下文对象详解

```yaml
# github上下文
- run: echo "Branch: ${{ github.ref_name }}"
- run: echo "SHA: ${{ github.sha }}"
- run: echo "Actor: ${{ github.actor }}"
- run: echo "Repo: ${{ github.repository }}"
- run: echo "Event: ${{ github.event_name }}"
- run: echo "Workspace: ${{ github.workspace }}"

# job上下文
- run: echo "Job status: ${{ job.status }}"
- run: echo "Job ID: ${{ job.container.id }}"

# runner上下文
- run: echo "OS: ${{ runner.os }}"
- run: echo "Arch: ${{ runner.arch }}"
- run: echo "Temp: ${{ runner.temp }}"

# steps上下文（在后续step中引用）
- id: calculate
  run: echo "result=42" >> $GITHUB_OUTPUT
- run: echo "Result: ${{ steps.calculate.outputs.result }}"
```

## 八、表达式与函数

```yaml
# 条件表达式
if: github.event_name == 'push'
if: github.ref == 'refs/heads/main'
if: startsWith(github.ref, 'refs/tags/v')
if: contains(github.event.head_commit.message, '[skip ci]')
if: github.event.pull_request.draft == false

# 字符串函数
- run: echo ${{ toJSON(github.event) }}
- run: echo ${{ fromJSON('{"key":"value"}').key }}
- run: echo ${{ format('Hello {0}!', github.actor) }}

# 逻辑函数
if: success() && github.ref == 'refs/heads/main'
if: failure() || cancelled()
if: always()

# 数组函数
if: contains('["main","develop"]', github.ref_name)
if: startsWith(github.ref, 'refs/tags/')

# HashFiles函数
key: ${{ runner.os }}-${{ hashFiles('**/package-lock.json', '**/yarn.lock') }}
```

## 九、concurrency并发控制

```yaml
# 同一分支只允许一个运行
concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true

# 按环境分组
concurrency:
  group: deploy-${{ inputs.environment }}
  cancel-in-progress: false  # 生产部署不取消
```

## 十、workflow_call可复用触发

```yaml
# 被其他工作流调用
on:
  workflow_call:
    inputs:
      node-version:
        type: string
        default: '20'
    outputs:
      version:
        description: 'Built version'
        value: ${{ jobs.build.outputs.version }}
    secrets:
      NPM_TOKEN:
        required: true
```
