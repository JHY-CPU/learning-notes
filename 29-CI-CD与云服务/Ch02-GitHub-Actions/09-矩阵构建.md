# 矩阵构建

## 一、概念说明

矩阵构建允许在多个配置组合下并行运行测试，如不同Node.js版本、不同操作系统等。

## 二、基本用法

```yaml
strategy:
  matrix:
    node: [16, 18, 20]
    os: [ubuntu-latest, windows-latest]
runs-on: ${{ matrix.os }}
steps:
  - uses: actions/setup-node@v4
    with:
      node-version: ${{ matrix.node }}
  - run: npm test
```

## 三、排除和包含

```yaml
strategy:
  matrix:
    node: [16, 18, 20]
    os: [ubuntu-latest, windows-latest]
    exclude:
      - os: windows-latest
        node: 16
    include:
      - os: macos-latest
        node: 20
```

## 四、fail-fast

```yaml
strategy:
  fail-fast: false  # 不因一个失败而取消其他
  matrix:
    node: [16, 18, 20]
```

## 五、最佳实践

1. **合理选择矩阵维度**：只测试真正有差异的组合，避免3x3x3的笛卡尔积爆炸
2. **使用include补充**：用include添加特殊情况，比exclude更清晰
3. **描述性名称**：使用`name: Test on ${{ matrix.os }} Node ${{ matrix.node }}`便于识别
4. **限制并行度**：对大型矩阵考虑分批运行，避免耗尽并发配额

## 六、常见陷阱

1. **fail-fast陷阱**：默认开启fail-fast，一个配置失败会取消所有其他运行，可能掩盖多个问题
2. **配额消耗**：矩阵构建会快速消耗GitHub Actions免费额度，开源项目注意公开仓库的限制
3. **include行为**：include添加的组合如果与现有组合完全相同，会更新而非新增，注意变量覆盖
4. **调试困难**：矩阵中某个配置失败时，需要在对应日志中定位问题，建议开启`ACTIONS_STEP_DEBUG`

## 七、动态矩阵构建

```yaml
jobs:
  generate-matrix:
    runs-on: ubuntu-latest
    outputs:
      matrix: ${{ steps.set-matrix.outputs.matrix }}
    steps:
      - uses: actions/checkout@v4
      - id: set-matrix
        run: |
          # 根据变更文件动态生成矩阵
          CHANGED=$(git diff --name-only HEAD~1 | grep -c "^packages/" || true)
          if [ "$CHANGED" -gt 0 ]; then
            PACKAGES=$(ls -d packages/*/ | xargs -I{} basename {} | jq -R -s -c 'split("\n") | map(select(. != ""))')
            echo "matrix={\"package\":$PACKAGES}" >> $GITHUB_OUTPUT
          else
            echo "matrix={\"package\":[\"core\"]}" >> $GITHUB_OUTPUT
          fi

  test:
    needs: generate-matrix
    runs-on: ubuntu-latest
    strategy:
      matrix: ${{ fromJSON(needs.generate-matrix.outputs.matrix) }}
    steps:
      - run: echo "Testing ${{ matrix.package }}"
```

## 八、矩阵与缓存结合

```yaml
test:
  runs-on: ${{ matrix.os }}
  strategy:
    matrix:
      os: [ubuntu-latest, macos-latest, windows-latest]
      node: [18, 20]
  steps:
    - uses: actions/checkout@v4
    - uses: actions/setup-node@v4
      with:
        node-version: ${{ matrix.node }}
        cache: 'npm'
        cache-dependency-path: package-lock.json
    - run: npm ci
    - run: npm test
    - name: Upload results
      uses: actions/upload-artifact@v4
      if: always()
      with:
        name: results-${{ matrix.os }}-${{ matrix.node }}
        path: test-results/
```

## 九、矩阵结果合并

```yaml
# 使用merge-multiple合并矩阵产物
test:
  strategy:
    matrix:
      os: [ubuntu, macos, windows]
  steps:
    - uses: actions/upload-artifact@v4
      with:
        name: result-${{ matrix.os }}
        path: results/

merge-results:
  needs: test
  runs-on: ubuntu-latest
  steps:
    - uses: actions/download-artifact@v4
      with:
        pattern: result-*
        merge-multiple: true
        path: all-results/
    - run: ls -la all-results/
```
