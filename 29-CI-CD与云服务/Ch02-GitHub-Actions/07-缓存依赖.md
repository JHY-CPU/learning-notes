# 缓存依赖

## 一、概念说明

缓存依赖可以显著加速CI/CD构建，避免每次重新下载依赖。

## 二、缓存Action

```yaml
# Node.js缓存
- uses: actions/cache@v4
  with:
    path: ~/.npm
    key: ${{ runner.os }}-node-${{ hashFiles('**/package-lock.json') }}
    restore-keys: |
      ${{ runner.os }}-node-

# Python缓存
- uses: actions/cache@v4
  with:
    path: ~/.cache/pip
    key: ${{ runner.os }}-pip-${{ hashFiles('**/requirements.txt') }}

# Maven缓存
- uses: actions/cache@v4
  with:
    path: ~/.m2/repository
    key: ${{ runner.os }}-maven-${{ hashFiles('**/pom.xml') }}
```

## 三、setup-* Action内置缓存

```yaml
# Node.js - 内置缓存
- uses: actions/setup-node@v4
  with:
    node-version: '20'
    cache: 'npm'

# Python - 内置缓存
- uses: actions/setup-python@v5
  with:
    python-version: '3.11'
    cache: 'pip'

# Go - 内置缓存
- uses: actions/setup-go@v5
  with:
    go-version: '1.21'
    cache: true
```

## 四、缓存策略

```yaml
# 缓存key设计
key: ${{ runner.os }}-${{ github.workflow }}-${{ hashFiles('**/lockfile') }}

# 缓存恢复
restore-keys: |
  ${{ runner.os }}-${{ github.workflow }}-
  ${{ runner.os }}-
```

## 五、注意事项

1. **缓存大小**：单个缓存最大10GB
2. **缓存有效期**：7天不访问自动删除
3. **缓存竞争**：多分支共享缓存可能导致冲突
4. **选择性缓存**：只缓存下载慢的依赖

## 六、Docker层缓存

```yaml
# GitHub Actions缓存后端
- uses: docker/build-push-action@v5
  with:
    context: .
    push: true
    tags: myapp:latest
    cache-from: type=gha
    cache-to: type=gha,mode=max

# Registry缓存后端
- uses: docker/build-push-action@v5
  with:
    context: .
    push: true
    tags: myapp:latest
    cache-from: type=registry,ref=myregistry/myapp:buildcache
    cache-to: type=registry,ref=myregistry/myapp:buildcache,mode=max

# 本地缓存
- uses: docker/build-push-action@v5
  with:
    context: .
    cache-from: type=local,src=/tmp/.buildx-cache
    cache-to: type=local,dest=/tmp/.buildx-cache-new,mode=max
- run: rm -rf /tmp/.buildx-cache && mv /tmp/.buildx-cache-new /tmp/.buildx-cache
```

## 七、缓存调试与监控

```yaml
# 查看缓存使用情况
- name: Cache debug
  run: |
    echo "Cache path: ${{ runner.temp }}/.cache"
    du -sh ${{ runner.temp }}/.cache 2>/dev/null || echo "No cache found"

# 使用gh CLI查看缓存
# gh cache list
# gh cache delete <cache-key>

# 缓存命中率检查
- name: Check cache hit
  uses: actions/cache@v4
  id: cache
  with:
    path: node_modules
    key: ${{ runner.os }}-node-${{ hashFiles('**/package-lock.json') }}

- name: Cache status
  run: |
    if [ "${{ steps.cache.outputs.cache-hit }}" == "true" ]; then
      echo "Cache hit! Dependencies restored from cache."
    else
      echo "Cache miss. Installing from scratch."
    fi
```

## 八、Monorepo缓存策略

```yaml
# Nx缓存
- uses: actions/cache@v4
  with:
    path: node_modules/.cache/nx
    key: nx-${{ runner.os }}-${{ hashFiles('**/pnpm-lock.yaml') }}-${{ github.sha }}
    restore-keys: |
      nx-${{ runner.os }}-${{ hashFiles('**/pnpm-lock.yaml') }}-
      nx-${{ runner.os }}-

# Turborepo缓存
- uses: actions/cache@v4
  with:
    path: .turbo
    key: turbo-${{ runner.os }}-${{ hashFiles('**/pnpm-lock.yaml') }}
  env:
    TURBO_TOKEN: ${{ secrets.TURBO_TOKEN }}
    TURBO_TEAM: ${{ vars.TURBO_TEAM }}
```
