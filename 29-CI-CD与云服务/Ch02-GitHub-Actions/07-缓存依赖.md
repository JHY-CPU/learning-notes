# 缓存依赖

## 一、概念说明

缓存依赖可以显著加速CI/CD构建，避免每次重新下载依赖。

## 二、缓存Action

```yaml
# Node.js缓存
- uses: actions/cache@v4
  with:
    path: ~/.npm
    key: ${{ runner.os }}-node-${{ hashFiles('**/package-lock.json') }}
    restore-keys: |
      ${{ runner.os }}-node-

# Python缓存
- uses: actions/cache@v4
  with:
    path: ~/.cache/pip
    key: ${{ runner.os }}-pip-${{ hashFiles('**/requirements.txt') }}

# Maven缓存
- uses: actions/cache@v4
  with:
    path: ~/.m2/repository
    key: ${{ runner.os }}-maven-${{ hashFiles('**/pom.xml') }}
```

## 三、setup-* Action内置缓存

```yaml
# Node.js - 内置缓存
- uses: actions/setup-node@v4
  with:
    node-version: '20'
    cache: 'npm'

# Python - 内置缓存
- uses: actions/setup-python@v5
  with:
    python-version: '3.11'
    cache: 'pip'

# Go - 内置缓存
- uses: actions/setup-go@v5
  with:
    go-version: '1.21'
    cache: true
```

## 四、缓存策略

```yaml
# 缓存key设计
key: ${{ runner.os }}-${{ github.workflow }}-${{ hashFiles('**/lockfile') }}

# 缓存恢复
restore-keys: |
  ${{ runner.os }}-${{ github.workflow }}-
  ${{ runner.os }}-
```

## 五、注意事项

1. **缓存大小**：单个缓存最大10GB
2. **缓存有效期**：7天不访问自动删除
3. **缓存竞争**：多分支共享缓存可能导致冲突
4. **选择性缓存**：只缓存下载慢的依赖
