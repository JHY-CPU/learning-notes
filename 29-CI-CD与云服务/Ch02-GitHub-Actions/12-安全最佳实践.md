# 安全最佳实践

## 一、权限最小化

```yaml
# 默认只读权限
permissions:
  contents: read
  packages: write

# 细粒度权限
permissions:
  contents: read
  issues: write
  pull-requests: write
```

## 二、固定Action版本

```yaml
# 使用SHA而不是标签
- uses: actions/checkout@8ade135a41bc03ea155e62e844d188df1ea18608  # v4

# 不推荐
- uses: actions/checkout@main
```

## 六、注意事项

1. **最小权限**：只授予必要的权限
2. **版本固定**：固定Action版本避免供应链攻击
3. **安全扫描**：集成安全扫描工具
4. **审计日志**：定期检查工作流运行日志

## 七、GitHub Actions安全加固清单

```yaml
# 1. 默认只读权限
# 在组织级别设置默认权限为只读
# Settings > Actions > General > Workflow permissions > Read repository contents

# 2. 每个工作流显式声明权限
permissions:
  contents: read
  issues: write
  pull-requests: write
  packages: write
  # 不要使用 write-all

# 3. 限制Actions使用来源
# Settings > Actions > General > Actions permissions
# 选择 "Allow select actions" 并白名单化:
# - actions/*
# - your-org/*
# - 具体的第三方Action SHA
```

## 八、CodeQL安全扫描

```yaml
name: CodeQL Analysis

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]
  schedule:
    - cron: '0 6 * * 1'

jobs:
  analyze:
    runs-on: ubuntu-latest
    permissions:
      actions: read
      contents: read
      security-events: write
    strategy:
      matrix:
        language: ['javascript', 'python']
    steps:
      - uses: actions/checkout@v4
      - uses: github/codeql-action/init@v3
        with:
          languages: ${{ matrix.language }}
      - uses: github/codeql-action/analyze@v3
        with:
          category: '/language:${{ matrix.language }}'
```

## 九、依赖安全扫描

```yaml
dependency-review:
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v4
    - uses: actions/dependency-review-action@v4
      with:
        fail-on-severity: high
        deny-licenses: GPL-3.0

dependency-scan:
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v4
    - uses: aquasecurity/trivy-action@master
      with:
        scan-type: 'fs'
        scan-ref: '.'
        severity: 'CRITICAL,HIGH'
        exit-code: 1
```

## 十一、安全事件响应

```
发生安全事件时:
1. 立即禁用受影响的工作流
2. 检查所有Secrets是否泄露
3. 旋转所有可能泄露的Secrets
4. 审计仓库访问日志
5. 检查是否有恶意提交
6. 更新所有Action到最新版本
7. 启用分支保护和签名验证
8. 通知团队成员
```
