# 安全最佳实践

## 一、权限最小化

```yaml
# 默认只读权限
permissions:
  contents: read
  packages: write

# 细粒度权限
permissions:
  contents: read
  issues: write
  pull-requests: write
```

## 二、固定Action版本

```yaml
# 使用SHA而不是标签
- uses: actions/checkout@8ade135a41bc03ea155e62e844d188df1ea18608  # v4

# 不推荐
- uses: actions/checkout@main
```

## 三、Secrets管理

```bash
# 1. 使用GitHub Secrets存储敏感信息
# 2. 环境Secrets限制到特定环境
# 3. 不要在日志中打印Secrets
# 4. 定期轮换Secrets
```

## 四、第三方Action审查

```bash
# 1. 优先使用官方Actions
# 2. 检查Action的Stars和维护状态
# 3. 审查Action的源代码
# 4. 固定到特定SHA
```

## 五、分支保护

```bash
# 1. 保护main分支
# 2. 要求PR审查
# 3. 要求CI通过
# 4. 禁止force push
```

## 六、注意事项

1. **最小权限**：只授予必要的权限
2. **版本固定**：固定Action版本避免供应链攻击
3. **安全扫描**：集成安全扫描工具
4. **审计日志**：定期检查工作流运行日志
