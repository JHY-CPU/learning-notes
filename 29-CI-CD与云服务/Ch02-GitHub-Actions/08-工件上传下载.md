# 工件上传下载

## 一、概念说明

工件（Artifact）是构建过程中产生的文件，可以在Job之间共享或下载。

## 二、上传工件

```yaml
- uses: actions/upload-artifact@v4
  with:
    name: build-output
    path: |
      dist/
      coverage/
    retention-days: 7
```

## 三、下载工件

```yaml
- uses: actions/download-artifact@v4
  with:
    name: build-output
    path: ./output
```

## 四、跨Job共享

```yaml
jobs:
  build:
    steps:
      - run: npm run build
      - uses: actions/upload-artifact@v4
        with:
          name: app
          path: dist/

  deploy:
    needs: build
    steps:
      - uses: actions/download-artifact@v4
        with:
          name: app
          path: ./app
      - run: ./deploy.sh
```

## 五、最佳实践

1. **精简内容**：只上传必要的构建产物（dist/、coverage/），不要包含node_modules等可重建内容
2. **合理保留期**：测试报告保留7天，发布包保留90天，节省存储空间
3. **命名规范**：工件名包含分支或Job信息，便于区分（如`build-${{ matrix.os }}`）
4. **压缩优化**：大文件先压缩再上传，减少上传时间

## 六、常见陷阱

1. **路径问题**：上传路径使用相对路径时相对于`$GITHUB_WORKSPACE`，配置错误导致找不到文件
2. **内存不足**：上传超大工件可能导致Runner内存不足，建议分拆或使用外部存储
3. **依赖缺失**：下载工件时指定了不存在的name，导致Job静默失败
4. **覆盖风险**：同名工件会被覆盖，矩阵构建时应使用不同名称

## 七、测试报告工件

```yaml
test:
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v4
    - run: npm ci
    - run: npm test -- --reporter=junit --output-file=test-results.xml
      continue-on-error: true
    - uses: actions/upload-artifact@v4
      if: always()
      with:
        name: test-results
        path: |
          test-results.xml
          coverage/
        retention-days: 30

    # 使用第三方Action上传测试报告
    - uses: dorny/test-reporter@v1
      if: always()
      with:
        name: Test Results
        path: test-results.xml
        reporter: jest-junit
```

## 八、构建产物的跨工作流传递

```yaml
# 工作流1: 构建
name: Build
on: [push]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm run build
      - uses: actions/upload-artifact@v4
        with:
          name: build-output
          path: dist/

# 工作流2: 使用构建产物
name: Deploy
on:
  workflow_run:
    workflows: ["Build"]
    types: [completed]

jobs:
  deploy:
    runs-on: ubuntu-latest
    if: ${{ github.event.workflow_run.conclusion == 'success' }}
    steps:
      - uses: dawidd6/action-download-artifact@v3
        with:
          workflow: build.yml
          name: build-output
          path: ./dist
      - run: ./deploy.sh
```

## 九、工件下载URL获取

```yaml
- uses: actions/upload-artifact@v4
  id: upload
  with:
    name: my-artifact
    path: dist/

- name: Get artifact URL
  run: |
    echo "Artifact URL: ${{ steps.upload.outputs.artifact-url }}"
    echo "Artifact ID: ${{ steps.upload.outputs.artifact-id }}"
```
