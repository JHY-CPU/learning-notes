# 分布式追踪

## 一、概念说明

分布式追踪用于跟踪请求在微服务间的流转过程，帮助定位性能瓶颈和故障根因。基于OpenTelemetry标准实现统一采集。

| 概念 | 说明 |
|------|------|
| Trace | 一次完整请求的调用链 |
| Span | 链路中的一个操作 |
| SpanContext | 跨服务传播的上下文（TraceID+SpanID） |
| Instrumentation | 自动/手动埋点 |
| Propagator | 上下文传播方式 |

## 二、具体用法

### 自动埋点（Python）

```python
# 使用auto-instrumentation自动埋点
# pip install opentelemetry-distro opentelemetry-exporter-otlp
# opentelemetry-bootstrap -a install
# opentelemetry-instrument --traces_exporter otlp python app.py

# 或者在代码中手动配置
from opentelemetry.instrumentation.flask import FlaskInstrumentor
from opentelemetry.instrumentation.requests import RequestsInstrumentor
from opentelemetry.instrumentation.sqlalchemy import SQLAlchemyInstrumentor
from opentelemetry.instrumentation.redis import RedisInstrumentor

app = Flask(__name__)
FlaskInstrumentor().instrument_app(app)
RequestsInstrumentor().instrument()
SQLAlchemyInstrumentor().instrument(engine=engine)
RedisInstrumentor().instrument()
```

### 手动埋点

```python
from opentelemetry import trace, baggage, context
from opentelemetry.trace import Status, StatusCode

tracer = trace.get_tracer(__name__)

def process_order(order_id, user_id):
    # 创建根Span
    with tracer.start_as_current_span("process_order") as span:
        span.set_attribute("order.id", order_id)
        span.set_attribute("user.id", user_id)

        # 设置Baggage（跨Span传递数据）
        ctx = baggage.set_baggage("order.id", order_id)

        # 创建子Span
        with tracer.start_as_current_span("validate_order") as validate_span:
            try:
                validate_order(order_id)
                validate_span.set_status(Status(StatusCode.OK))
            except ValidationError as e:
                validate_span.set_status(Status(StatusCode.ERROR, str(e)))
                validate_span.record_exception(e)
                raise

        # 另一个子Span
        with tracer.start_as_current_span("charge_payment") as payment_span:
            payment_span.set_attribute("payment.method", "credit_card")
            payment_span.add_event("payment_initiated", {
                "amount": 99.99,
                "currency": "CNY"
            })
            result = charge_payment(order_id)
            payment_span.add_event("payment_completed", {
                "transaction_id": result["transaction_id"]
            })
```

### 上下文传播

```python
from opentelemetry.propagate import inject, extract
import requests

# 发起HTTP请求时注入上下文
def call_downstream_service(url, data):
    headers = {}
    inject(headers)  # 注入Trace上下文到HTTP头

    with tracer.start_as_current_span("call_downstream") as span:
        span.set_attribute("http.url", url)
        response = requests.post(url, json=data, headers=headers)
        span.set_attribute("http.status_code", response.status_code)
        return response

# 接收请求时提取上下文
from flask import request

@app.route('/callback', methods=['POST'])
def callback():
    ctx = extract(request.headers)  # 从HTTP头提取上下文
    with tracer.start_as_current_span("handle_callback", context=ctx):
        data = request.json
        # 处理回调
        return {"status": "ok"}
```

### Span事件和状态

```python
from opentelemetry.trace import Status, StatusCode

with tracer.start_as_current_span("database_query") as span:
    try:
        # 记录事件
        span.add_event("query_started", {"sql": sql_query[:100]})

        result = db.execute(sql_query)

        span.add_event("query_completed", {
            "rows": len(result),
            "duration_ms": duration
        })

        span.set_status(Status(StatusCode.OK))

    except Exception as e:
        span.set_status(Status(StatusCode.ERROR, "查询失败"))
        span.record_exception(e, attributes={
            "error.type": type(e).__name__,
            "error.message": str(e)
        })
        raise
```

## 三、注意事项与常见陷阱

1. **采样策略**：生产环境使用ParentBased或TraceIdRatio采样
2. **Span属性**：避免记录敏感信息和高基数数据
3. **异常处理**：务必记录异常到Span状态
4. **上下文传播**：异步场景确保上下文正确传播
5. **性能影响**：追踪有开销，生产环境采样率建议1-10%
6. **存储成本**：长时间保存追踪数据成本较高
7. **命名规范**：团队统一Span命名规范便于搜索
