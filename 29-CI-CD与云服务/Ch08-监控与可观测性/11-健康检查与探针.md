# 健康检查与探针

## 一、概念说明

健康检查用于验证服务是否正常运行，是容器编排和负载均衡的基础。Kubernetes定义了三种探针：存活探针、就绪探针、启动探针。

| 探针类型 | 作用 | 失败时动作 |
|----------|------|-----------|
| Liveness | 检测容器是否存活 | 重启容器 |
| Readiness | 检测容器是否就绪 | 从Service摘除 |
| Startup | 检测启动是否完成 | 禁用其他探针 |

## 二、具体用法

### Kubernetes探针配置

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: web-app
spec:
  containers:
  - name: app
    image: myapp:v1.0
    ports:
    - containerPort: 8080

    # 启动探针 - 启动期间禁用其他探针
    startupProbe:
      httpGet:
        path: /health/startup
        port: 8080
      initialDelaySeconds: 5
      periodSeconds: 5
      failureThreshold: 30  # 最多等150秒

    # 存活探针 - 检测容器是否正常
    livenessProbe:
      httpGet:
        path: /health/live
        port: 8080
      initialDelaySeconds: 0
      periodSeconds: 10
      timeoutSeconds: 3
      failureThreshold: 3

    # 就绪探针 - 检测是否可接收流量
    readinessProbe:
      httpGet:
        path: /health/ready
        port: 8080
      initialDelaySeconds: 0
      periodSeconds: 5
      timeoutSeconds: 3
      failureThreshold: 3
      successThreshold: 1

    # TCP探针
    # readinessProbe:
    #   tcpSocket:
    #     port: 8080
    #   periodSeconds: 10

    # Exec探针
    # livenessProbe:
    #   exec:
    #     command:
    #     - cat
    #     - /tmp/healthy
    #   periodSeconds: 10
```

### 应用端健康检查实现

```python
from flask import Flask, jsonify
import threading
import time

app = Flask(__name__)

# 全局状态
ready = threading.Event()
healthy = threading.Event()
started = threading.Event()

@app.route('/health/startup')
def startup():
    """启动探针 - 应用初始化完成返回200"""
    if started.is_set():
        return jsonify({"status": "started"}), 200
    return jsonify({"status": "starting"}), 503

@app.route('/health/live')
def liveness():
    """存活探针 - 检测是否死锁等不可恢复状态"""
    if healthy.is_set():
        return jsonify({"status": "healthy"}), 200
    return jsonify({"status": "unhealthy"}), 503

@app.route('/health/ready')
def readiness():
    """就绪探针 - 检查依赖服务是否可用"""
    checks = {}

    # 检查数据库
    try:
        db.execute("SELECT 1")
        checks["database"] = "ok"
    except Exception as e:
        checks["database"] = str(e)

    # 检查Redis
    try:
        redis_client.ping()
        checks["redis"] = "ok"
    except Exception as e:
        checks["redis"] = str(e)

    # 检查关键外部服务
    try:
        resp = requests.get("http://payment-service/health", timeout=2)
        checks["payment_service"] = "ok" if resp.ok else f"status={resp.status_code}"
    except Exception as e:
        checks["payment_service"] = str(e)

    all_ok = all(v == "ok" for v in checks.values())

    if all_ok and ready.is_set():
        return jsonify({"status": "ready", "checks": checks}), 200
    return jsonify({"status": "not_ready", "checks": checks}), 503

# 应用初始化
def initialize():
    # 连接数据库
    connect_database()
    # 加载缓存
    load_cache()
    # 注册到服务发现
    register_service()

    started.set()
    healthy.set()
    ready.set()

threading.Thread(target=initialize, daemon=True).start()
```

### Java Spring Boot

```java
@Component
public class DatabaseHealthIndicator implements HealthIndicator {

    @Autowired
    private DataSource dataSource;

    @Override
    public Health health() {
        try (Connection conn = dataSource.getConnection()) {
            conn.createStatement().execute("SELECT 1");
            return Health.up()
                .withDetail("database", "PostgreSQL")
                .withDetail("status", "connected")
                .build();
        } catch (SQLException e) {
            return Health.down()
                .withDetail("database", "PostgreSQL")
                .withException(e)
                .build();
        }
    }
}
```

```bash
    --name web-tg \
    --protocol HTTP \
    --port 8080 \

    --unhealthy-threshold-count 3 \
```

## 三、注意事项与常见陷阱

1. **检查路径**：轻量级路径，避免执行复杂业务逻辑
2. **超时设置**：探针超时要短于检测周期
3. **依赖检查**：就绪探针检查关键依赖，存活探针不要检查依赖
4. **启动时间**：慢启动应用设置startupProbe或增大initialDelaySeconds
5. **避免级联**：健康检查不要触发级联依赖检查
6. **监控探针**：监控探针本身的失败率
7. **优雅关闭**：收到SIGTERM时先设置not ready再关闭
