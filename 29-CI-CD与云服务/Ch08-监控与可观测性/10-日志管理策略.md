# 日志管理策略

## 一、概念说明

日志管理策略涵盖日志的产生、收集、传输、存储、分析和归档全生命周期。好的策略能有效支撑故障排查、安全审计和业务分析。

| 阶段 | 关注点 | 工具 |
|------|--------|------|
| 产生 | 结构化、分级 | Log4j/Logback/Zap |
| 收集 | 高效、不丢日志 | Filebeat/Fluentd |
| 处理 | 解析、过滤、丰富 | Logstash/Vector |
| 存储 | 可扩展、可查询 | ES/Loki |
| 分析 | 快速检索、聚合 | Kibana/Grafana |
| 归档 | 长期保留、低成本 | S3/OSS |

### 日志分级策略

```yaml
Log_Levels:
  FATAL:
    场景: 系统不可用
    行动: 立即告警、电话通知
    保留: 永久
  ERROR:
    场景: 业务失败、异常
    行动: 告警、工单
    保留: 1年
  WARN:
    场景: 潜在问题
    行动: 关注趋势
    保留: 90天
  INFO:
    场景: 业务关键操作
    行动: 正常存储

  DEBUG:
    场景: 调试信息
    行动: 仅开发环境
    保留: 7天
```

### Fluentd日志处理

```xml
# fluent.conf
<source>
  @type tail
  path /var/log/app/*.log
  pos_file /var/log/fluentd/app.pos
  tag app.*
  <parse>
    @type json
    time_key timestamp
    time_format %Y-%m-%dT%H:%M:%S.%L%z
  </parse>
</source>

<filter app.**>
  @type record_transformer
  <record>
    hostname "#{Socket.gethostname}"
    environment "#{ENV['APP_ENV']}"
  </record>
</filter>

# 错误日志单独路由
<match app.**>
  @type copy

  <store>
    @type elasticsearch
    host elasticsearch
    port 9200
    logstash_format true
    logstash_prefix app-logs
  </store>

  <store>
    @type elasticsearch
    host elasticsearch
    port 9200
    logstash_format true
    logstash_prefix error-logs
    <filter>
      @type grep
      <regexp>
        key level
        pattern /^ERROR|FATAL$/
      </regexp>
    </filter>
  </store>
</match>
```

### 日志脱敏

```python
import re
import json

class LogSanitizer:
    """日志脱敏处理"""

    PATTERNS = [
        # 手机号
        (re.compile(r'1[3-9]\d{9}'), lambda m: m.group()[:3] + '****' + m.group()[-4:]),
        # 身份证
        (re.compile(r'\d{17}[\dXx]'), lambda m: m.group()[:6] + '********' + m.group()[-4:]),
        # 银行卡
        (re.compile(r'\d{16,19}'), lambda m: m.group()[:4] + ' **** **** ' + m.group()[-4:]),
        # 密码字段
        (re.compile(r'("password"\s*:\s*)"[^"]*"'), r'\1"***"'),
        # Token字段
        (re.compile(r'("token"\s*:\s*)"[^"]*"'), r'\1"***"'),
    ]

    def sanitize(self, message):
        for pattern, replacement in self.PATTERNS:
            if callable(replacement):
                message = pattern.sub(replacement, message)
            else:
                message = pattern.sub(replacement, message)
        return message

# 使用
sanitizer = LogSanitizer()
safe_message = sanitizer.sanitize(raw_log_message)
```

## 三、注意事项与常见陷阱

1. **结构化日志**：统一使用JSON格式，便于解析和查询
2. **日志分级**：严格执行分级，避免DEBUG日志进入生产
3. **敏感脱敏**：密码、身份证等敏感信息必须脱敏
4. **关联追踪**：日志中包含trace_id便于关联追踪
5. **存储成本**：合理设置保留策略控制存储成本
6. **日志采样**：高流量场景考虑日志采样
7. **性能影响**：同步日志可能影响性能，考虑异步写入
