# 可观测性概述

## 一、概念说明

可观测性（Observability）是通过外部输出推断系统内部状态的能力。三大支柱：指标（Metrics）、日志（Logs）、链路追踪（Traces）。

| 支柱 | 数据类型 | 用途 | 代表工具 |
|------|----------|------|----------|
| 指标 | 时序数值 | 系统性能监控 | Prometheus |
| 日志 | 文本事件 | 问题排查 | ELK |
| 追踪 | 请求链路 | 性能瓶颈定位 | Jaeger |

## 二、具体用法

### 可观测性架构

```yaml
# 典型的可观测性技术栈
Metrics:
  采集: Prometheus / Node Exporter / cAdvisor
  存储: Prometheus / VictoriaMetrics / Thanos
  可视化: Grafana

Logs:
  采集: Filebeat / Fluentd / Fluent Bit
  存储: Elasticsearch
  可视化: Kibana

Traces:
  采集: OpenTelemetry SDK
  存储: Jaeger / Tempo
  可视化: Jaeger UI / Grafana

统一方案:
  - Grafana (统一可视化)
  - OpenTelemetry (统一采集)
  - Loki (日志) + Tempo (追踪) + Mimir (指标)
```

```yaml
  pipelines:

      receivers: [otlp]
      processors: [batch]
      exporters: [elasticsearch]
    traces:
      receivers: [otlp]
      processors: [batch]
      exporters: [jaeger]
```

```python
from opentelemetry.sdk.trace import TracerProvider
from opentelemetry.sdk.metrics import MeterProvider
from opentelemetry.sdk.resources import Resource
from opentelemetry.exporter.otlp.proto.grpc.trace_exporter import OTLPSpanExporter

# 初始化追踪
resource = Resource.create({"service.name": "my-app"})
provider = TracerProvider(resource=resource)
processor = BatchSpanProcessor(OTLPSpanExporter(endpoint="otel-collector:4317"))
provider.add_span_processor(processor)
trace.set_tracer_provider(provider)

tracer = trace.get_tracer(__name__)

def handle_request(request_id):
    with tracer.start_as_current_span("handle_request") as span:
        span.set_attribute("request.id", request_id)
        # 业务逻辑
        result = process(request_id)
        span.set_attribute("result.count", len(result))
        return result
```

## 三、注意事项与常见陷阱

1. **数据量控制**：合理设置采样率避免数据爆炸
2. **标签基数**：避免高基数标签导致存储膨胀
3. **告警疲劳**：设置合理的告警阈值和静默策略
4. **成本管理**：监控存储和查询资源的使用量
5. **数据保留**：设置合适的数据保留策略
6. **安全合规**：日志中的敏感数据需要脱敏
7. **团队协作**：统一团队的可观测性工具和规范
