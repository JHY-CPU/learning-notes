# 流水线配置

## 一、Job定义

```yaml
build:
  stage: build
  image: node:20-alpine
  script:
    - npm install
    - npm run build
  artifacts:
    paths:
      - dist/
    expire_in: 1 week
  cache:
    paths:
      - node_modules/
```

## 二、Stage定义

```yaml
stages:
  - build
  - test
  - deploy

build:
  stage: build
  script: echo "Building..."

test:unit:
  stage: test
  script: echo "Unit test..."

test:e2e:
  stage: test
  script: echo "E2E test..."

deploy:
  stage: deploy
  script: echo "Deploying..."
```

## 三、触发条件

```yaml
# 仅main分支
only:
  - main

# 除了main分支
except:
  - main

# 规则（推荐）
rules:
  - if: $CI_COMMIT_BRANCH == "main"
  - if: $CI_PIPELINE_SOURCE == "merge_request_event"
```

## 四、环境变量

```yaml
variables:
  NODE_ENV: production

build:
  variables:
    DEBUG: "false"
  script:
    - echo $NODE_ENV
    - echo $DEBUG
```

## 五、注意事项

1. **依赖关系**：使用needs定义Job依赖
2. **并行执行**：同一Stage的Job并行执行
3. **缓存配置**：缓存依赖加速构建
4. **制品管理**：保留必要的构建制品

## 六、needs依赖关系（DAG执行）

```yaml
# 使用needs实现DAG调度，不依赖stages顺序
build-frontend:
  stage: build
  script:
    - npm run build

build-backend:
  stage: build
  script:
    - go build ./...

test-frontend:
  stage: test
  needs: [build-frontend]
  script:
    - npm test

test-backend:
  stage: test
  needs: [build-backend]
  script:
    - go test ./...

deploy:
  stage: deploy
  needs: [test-frontend, test-backend]
  script:
    - ./deploy.sh
  rules:
    - if: $CI_COMMIT_BRANCH == "main"
```

## 七、rules规则引擎

```yaml
deploy-staging:
  script: ./deploy.sh staging
  rules:
    - if: $CI_COMMIT_BRANCH == "develop"
      when: always
    - if: $CI_COMMIT_BRANCH == "main"
      when: manual
      allow_failure: true
    - if: $CI_PIPELINE_SOURCE == "merge_request_event"
      when: never
    - when: never

# 使用changes检测文件变更
test-docs:
  script: npm run test:docs
  rules:
    - changes:
        - docs/**/*
        - '**/*.md'
      when: always

# 使用exists检测文件存在
test-docker:
  script: docker build -t test .
  rules:
    - exists:
        - Dockerfile
```

## 八、DAG流水线完整示例

```yaml
stages:
  - build
  - test
  - security
  - deploy

variables:
  NODE_VERSION: "20"

build:frontend:
  stage: build
  image: node:${NODE_VERSION}-alpine
  script:
    - cd frontend && npm ci && npm run build
  artifacts:
    paths:
      - frontend/dist/

build:backend:
  stage: build
  image: golang:1.21-alpine
  script:
    - cd backend && go build -o bin/app ./cmd
  artifacts:
    paths:
      - backend/bin/

test:frontend:
  needs: [build:frontend]
  image: node:${NODE_VERSION}-alpine
  script:
    - cd frontend && npm test
  artifacts:
    reports:
      junit: frontend/test-results.xml

test:backend:
  needs: [build:backend]
  image: golang:1.21-alpine
  script:
    - cd backend && go test ./...

security:sast:
  needs: []  # 不依赖任何Job
  script:
    - semgrep --config auto .

security:dependencies:
  needs: []
  script:
    - trivy fs .

deploy:
  needs: [test:frontend, test:backend, security:sast, security:dependencies]
  script:
    - ./deploy.sh
  rules:
    - if: $CI_COMMIT_BRANCH == "main"
  environment:
    name: production
    url: https://example.com
```

## 九、Job超时和重试

```yaml
build:
  script: ./build.sh
  timeout: 2h 30m
  retry:
    max: 2
    when:
      - runner_system_failure
      - stuck_or_timeout_failure
      - scheduler_failure

# 全局默认值
default:
  timeout: 1h
  retry:
    max: 1
    when: runner_system_failure
```
