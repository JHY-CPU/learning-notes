# 环境与部署

## 一、环境配置

```yaml
deploy-staging:
  stage: deploy
  environment:
    name: staging
    url: https://staging.example.com
  script:
    - ./deploy.sh staging
  only:
    - main

deploy-prod:
  stage: deploy
  environment:
    name: production
    url: https://example.com
  script:
    - ./deploy.sh production
  when: manual
  only:
    - main
```

## 二、环境变量

```yaml
# GitLab UI设置

# 使用环境变量
```

## 三、部署审批

```yaml
deploy-prod:
  environment:
    name: production
  when: manual  # 手动触发
  allow_failure: false
  script:
    - ./deploy.sh
```

## 五、注意事项

1. **环境保护**：生产环境需要审批
2. **Secret管理**：敏感信息使用CI/CD变量
3. **回滚准备**：保持可回滚能力
4. **部署监控**：部署后监控应用状态

## 六、环境配置详细示例

```yaml
stages:
  - build
  - deploy-staging
  - acceptance-test
  - deploy-production

build:
  stage: build
  image: docker:24.0.5
  services:
    - docker:24.0.5-dind
  script:
    - docker build -t $CI_REGISTRY_IMAGE:$CI_COMMIT_SHA .
    - docker push $CI_REGISTRY_IMAGE:$CI_COMMIT_SHA

deploy-staging:
  stage: deploy-staging
  environment:
    name: staging
    url: https://staging.example.com
    on_stop: stop-staging  # 关联停止Job
  script:
    - kubectl set image deployment/app app=$CI_REGISTRY_IMAGE:$CI_COMMIT_SHA
    - kubectl rollout status deployment/app
  only:
    - main

stop-staging:
  stage: deploy-staging
  environment:
    name: staging
    action: stop
  script:
    - kubectl delete deployment/app
  when: manual
  only:
    - main

deploy-production:
  stage: deploy-production
  environment:
    name: production
    url: https://example.com
    kubernetes:
      namespace: production
  script:
    - kubectl set image deployment/app app=$CI_REGISTRY_IMAGE:$CI_COMMIT_SHA
  when: manual
  only:
    - main
```

## 七、部署审批与保护

```yaml
# 在GitLab UI中配置环境
# Settings > CI/CD > Environments
# production环境:
#   - Protected branches: main
#   - Required approvals: 2
#   - Approval rules: @devops-team

deploy-prod:
  environment:
    name: production
  when: manual
  allow_failure: false  # 必须手动批准
  script:
    - ./deploy.sh
  only:
    - main
  # 只有protected branches的提交才能触发
```

## 八、Kubernetes部署集成

```yaml
deploy-k8s:
  stage: deploy
  image:
    name: bitnami/kubectl:latest
    entrypoint: ['']
  environment:
    name: ${CI_COMMIT_REF_NAME}
    kubernetes:
      namespace: ${CI_COMMIT_REF_NAME}
  script:
    - kubectl config use-context ${KUBE_CONTEXT}
    - kubectl apply -f k8s/
    - kubectl rollout status deployment/app -n ${CI_COMMIT_REF_NAME}
  rules:
    - if: $CI_COMMIT_BRANCH == "main"
    - if: $CI_COMMIT_BRANCH == "develop"
```

```bash
  "https://gitlab.com/api/v4/projects/$PROJECT_ID/environments/$ENV_ID/revert"

# 在Pipeline页面点击"Revert"按钮
```
