# 环境与部署

## 一、环境配置

```yaml
deploy-staging:
  stage: deploy
  environment:
    name: staging
    url: https://staging.example.com
  script:
    - ./deploy.sh staging
  only:
    - main

deploy-prod:
  stage: deploy
  environment:
    name: production
    url: https://example.com
  script:
    - ./deploy.sh production
  when: manual
  only:
    - main
```

## 二、环境变量

```yaml
# GitLab UI设置
# Settings > CI/CD > Variables

# 使用环境变量
deploy:
  script:
    - echo $DATABASE_URL
    - echo $API_KEY
```

## 三、部署审批

```yaml
deploy-prod:
  environment:
    name: production
  when: manual  # 手动触发
  allow_failure: false
  script:
    - ./deploy.sh
```

## 五、注意事项

1. **环境保护**：生产环境需要审批
2. **Secret管理**：敏感信息使用CI/CD变量
3. **回滚准备**：保持可回滚能力
4. **部署监控**：部署后监控应用状态
