# 缓存与制品

## 一、缓存配置

```yaml
# 全局缓存
cache:
  key: ${CI_COMMIT_REF_SLUG}
  paths:
    - node_modules/
    - .cache/

# Job缓存
build:
  cache:
    key: build-cache
    paths:
      - node_modules/
```

## 二、制品配置

```yaml
build:
  artifacts:
    paths:
      - dist/
    expire_in: 1 week
    reports:
      junit: test-results.xml

test:
  dependencies:
    - build
  script:
    - ls dist/
```

## 三、缓存vs制品

```bash
# 缓存（cache）
# 在Job之间共享文件
# 加速依赖下载
# 可能被覆盖

# 制品（artifacts）
# 在Stage之间传递文件
# 保存构建产物
# 可下载查看
```

## 四、注意事项

1. **缓存key**：合理的缓存key避免冲突
2. **缓存策略**：pull/push/pull-push
3. **制品大小**：制品大小限制
4. **清理策略**：设置制品过期时间
