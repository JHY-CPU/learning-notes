# 模板与引用

## 一、extends继承

```yaml
# 定义模板
.test-template:
  image: node:20-alpine
  cache:
    paths:
      - node_modules/
  before_script:
    - npm ci

# 继承模板
test:unit:
  extends: .test-template
  script:
    - npm run test:unit

test:e2e:
  extends: .test-template
  script:
    - npm run test:e2e
```

## 二、include引入

```yaml
# 引入本地文件
include:
  - local: '/ci/build.yml'
  - local: '/ci/test.yml'

# 引入远程文件
include:
  - remote: 'https://gitlab.com/templates/ci.yml'

# 引入模板
include:
  - template: Auto-DevOps.gitlab-ci.yml
```

## 三、YAML锚点

```yaml
# 定义锚点
.build-template: &build
  stage: build
  image: node:20
  script:
    - npm install
    - npm run build

# 使用锚点
build:linux:
  <<: *build
  tags: [linux]

build:windows:
  <<: *build
  tags: [windows]
```

## 四、注意事项

1. **模板复用**：提取公共配置为模板
2. **版本控制**：远程模板指定版本
3. **覆盖继承**：子Job可以覆盖父配置
4. **文档清晰**：模板应有清晰文档

## 五、完整模板设计

```yaml
# ci/templates/build.yml
.build-template:
  image: node:20-alpine
  cache:
    key: ${CI_COMMIT_REF_SLUG}
    paths:
      - node_modules/
  before_script:
    - npm ci
  artifacts:
    paths:
      - dist/
    expire_in: 1 week

.build:npm:
  extends: .build-template
  stage: build
  script:
    - npm run build
  rules:
    - changes:
        - package.json
        - package-lock.json
        - src/**/*
      when: always
    - when: manual
      allow_failure: true
```

```yaml
# ci/templates/test.yml
.test-template:
  image: node:20-alpine
  cache:
    key: ${CI_COMMIT_REF_SLUG}
    paths:
      - node_modules/
  before_script:
    - npm ci

.test:unit:
  extends: .test-template
  stage: test
  script:
    - npm run test:unit -- --coverage
  coverage: '/Lines\s*:\s*(\d+\.?\d*)%/'
  artifacts:
    reports:
      junit: test-results/unit.xml
      coverage_report:
        coverage_format: cobertura
        path: coverage/cobertura.xml

.test:integration:
  extends: .test-template
  stage: test
  services:
    - postgres:15-alpine
    - redis:7-alpine
  variables:
    DATABASE_URL: "postgres://postgres:postgres@postgres:5432/testdb"
    REDIS_URL: "redis://redis:6379"
  script:
    - npm run test:integration
  artifacts:
    reports:
      junit: test-results/integration.xml
```

```yaml
# ci/templates/deploy.yml
.deploy-template:
  image: bitnami/kubectl:latest
  before_script:
    - kubectl config use-context $KUBE_CONTEXT
  rules:
    - if: $CI_COMMIT_BRANCH == "main"

.deploy:staging:
  extends: .deploy-template
  stage: deploy
  environment:
    name: staging
    url: https://staging.example.com
  script:
    - kubectl set image deployment/app app=$CI_REGISTRY_IMAGE:$CI_COMMIT_SHA
    - kubectl rollout status deployment/app

.deploy:production:
  extends: .deploy-template
  stage: deploy
  environment:
    name: production
    url: https://example.com
  script:
    - kubectl set image deployment/app app=$CI_REGISTRY_IMAGE:$CI_COMMIT_SHA
    - kubectl rollout status deployment/app
  when: manual
```

## 六、include多来源引入

```yaml
# .gitlab-ci.yml
include:
  # 本地文件
  - local: '/ci/templates/build.yml'
  - local: '/ci/templates/test.yml'
  - local: '/ci/templates/deploy.yml'

  # 远程文件
  - remote: 'https://gitlab.com/my-org/ci-templates/-/raw/main/security.yml'

  # GitLab内置模板
  - template: Auto-DevOps.gitlab-ci.yml

  # 项目文件
  - project: 'my-org/ci-templates'
    ref: main
    file: '/templates/lint.yml'

# 项目覆盖优先
stages:
  - build
  - test
  - deploy

build:frontend:
  extends: .build:npm
  variables:
    NODE_ENV: production
```


## 八、模板继承规则

```yaml
# 继承规则:
# extends继承: 合并父Job配置，子配置优先
# include引入: 合并所有文件的配置
# YAML锚点: 引用定义的块

# 配置合并优先级（从高到低）:
# 1. 子Job直接定义的配置
# 2. extends继承的配置
# 3. include引入的配置
# 4. default默认配置
# 5. 全局variables
```
