# 变量与密钥

## 一、CI/CD变量

```yaml
# 项目变量
# Settings > CI/CD > Variables
# 变量名: DATABASE_URL
# 值: postgres://localhost/mydb
# 类型: Variable / File
# 保护: 仅保护分支可用
# 掩码: 日志中隐藏

# 使用变量
script:
  - echo $DATABASE_URL
```

## 二、预定义变量

```yaml
# 常用预定义变量
# $CI_COMMIT_REF_NAME - 分支/标签名
# $CI_COMMIT_SHA - 提交SHA
# $CI_PIPELINE_ID - 流水线ID
# $CI_PROJECT_NAME - 项目名
# $CI_JOB_NAME - Job名

script:
  - echo "Branch: $CI_COMMIT_REF_NAME"
  - echo "Commit: $CI_COMMIT_SHA"
```

## 三、动态变量

```yaml
build:
  script:
    - echo "VERSION=$(cat version.txt)" >> build.env
  artifacts:
    reports:
      dotenv: build.env

deploy:
  needs:
    - build
  script:
    - echo "Deploying version $VERSION"
```

## 四、文件类型变量

```bash
# 类型选择File
# 变量值存储为文件
# 变量内容是文件路径

script:
  - cat $GOOGLE_APPLICATION_CREDENTIALS
```

## 五、注意事项

1. **保护变量**：生产环境变量仅保护分支可用
2. **掩码变量**：敏感信息设置掩码
3. **作用范围**：设置变量的作用范围
4. **环境特定**：不同环境使用不同变量
