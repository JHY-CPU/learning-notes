# 变量与密钥

## 一、CI/CD变量

```yaml
# 项目变量
# Settings > CI/CD > Variables
# 变量名: DATABASE_URL
# 值: postgres://localhost/mydb
# 类型: Variable / File
# 保护: 仅保护分支可用
# 掩码: 日志中隐藏

# 使用变量
script:
  - echo $DATABASE_URL
```

## 二、预定义变量

```yaml
# 常用预定义变量
# $CI_COMMIT_REF_NAME - 分支/标签名
# $CI_COMMIT_SHA - 提交SHA
# $CI_PIPELINE_ID - 流水线ID
# $CI_PROJECT_NAME - 项目名
# $CI_JOB_NAME - Job名

script:
  - echo "Branch: $CI_COMMIT_REF_NAME"
  - echo "Commit: $CI_COMMIT_SHA"
```

## 三、动态变量

```yaml
build:
  script:
    - echo "VERSION=$(cat version.txt)" >> build.env
  artifacts:
    reports:
      dotenv: build.env

deploy:
  needs:
    - build
  script:
    - echo "Deploying version $VERSION"
```

## 四、文件类型变量

```bash
# 类型选择File
# 变量值存储为文件
# 变量内容是文件路径

script:
  - cat $GOOGLE_APPLICATION_CREDENTIALS
```

## 五、注意事项

1. **保护变量**：生产环境变量仅保护分支可用
2. **掩码变量**：敏感信息设置掩码
3. **作用范围**：设置变量的作用范围
4. **环境特定**：不同环境使用不同变量

## 六、变量优先级

```
变量优先级（从高到低）:
1. 手动触发Pipeline时指定的变量
2. 触发器/Token变量
3. 项目级变量（Settings > CI/CD > Variables）
4. 组级变量（Group > Settings > CI/CD > Variables）
5. 实例级变量（Admin > CI/CD > Variables）
6. .gitlab-ci.yml中定义的变量
7. 预定义变量
```

## 七、完整的环境变量配置

```yaml
# 全局变量
variables:
  DOCKER_DRIVER: overlay2
  DOCKER_TLS_CERTDIR: "/certs"
  GIT_DEPTH: 1

build:
  variables:
    NODE_ENV: production
    DEBUG: "false"
  script:
    - echo "Node env: $NODE_ENV"
    - echo "Debug: $DEBUG"
    - echo "Branch: $CI_COMMIT_REF_NAME"
    - echo "Commit: $CI_COMMIT_SHORT_SHA"
    - echo "Pipeline: $CI_PIPELINE_URL"
    - echo "Job: $CI_JOB_URL"
```

## 八、Variable类型详解

```yaml
# 变量类型Variable（字符串）
# 直接作为环境变量使用
script:
  - echo $MY_VARIABLE

# 变量类型File（文件）
# 变量值是文件路径，文件内容是实际值
script:
  - cat $GOOGLE_APPLICATION_CREDENTIALS  # 输出文件内容
  - export GOOGLE_APPLICATION_CREDENTIALS  # 这是文件路径
```

## 九、动态变量与dotenv报告

```yaml
build:
  script:
    - VERSION=$(node -p "require('./package.json').version")
    - BUILD_TIME=$(date -u +"%Y-%m-%dT%H:%M:%SZ")
    - |
      cat > build.env << EOF
      APP_VERSION=$VERSION
      BUILD_TIME=$BUILD_TIME
      IMAGE_TAG=$CI_COMMIT_SHORT_SHA
      EOF
  artifacts:
    reports:
      dotenv: build.env

deploy:
  needs:
    - job: build
      artifacts: true
  script:
    - echo "Deploying $APP_VERSION built at $BUILD_TIME"
    - docker tag myapp myapp:$IMAGE_TAG
    - docker push myapp:$IMAGE_TAG
```

## 十、HashiCorp Vault集成

```yaml
# 使用Vault获取Secret
deploy:
  image: vault:latest
  before_script:
    - export VAULT_TOKEN=$(vault write -field=token auth/jwt/login role=ci jwt=$CI_JOB_JWT)
  script:
    - export DB_PASSWORD=$(vault kv get -field=password secret/myapp/db)
    - export API_KEY=$(vault kv get -field=api_key secret/myapp/api)
    - ./deploy.sh
```

## 十一、变量安全检查

```yaml
# 检查是否意外打印Secrets
check-secrets:
  script:
    - |
      if env | grep -i "password\|secret\|token" | grep -v "^\*\*\*\*$"; then
        echo "WARNING: Potential secret leak detected"
        exit 1
      fi
  allow_failure: true
```
