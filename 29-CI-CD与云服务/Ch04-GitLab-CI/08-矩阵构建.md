# 矩阵构建

## 一、parallel:matrix

```yaml
test:
  stage: test
  image: node:${NODE_VERSION}
  parallel:
    matrix:
      - NODE_VERSION: ['16', '18', '20']
        OS: ['alpine', 'slim']
  script:
    - echo "Testing on Node $NODE_VERSION $OS"
    - npm test
```

## 二、变量组合

```yaml
test:
  parallel:
    matrix:
      - PROVIDER: aws
        STACK: [monitoring, app]
      - PROVIDER: gcp
        STACK: [monitoring, app]
  script:
    - ./test.sh $PROVIDER $STACK
```

## 三、条件执行

```yaml
test:
  parallel:
    matrix:
      - NODE_VERSION: ['16', '18', '20']
  rules:
    - if: $CI_PIPELINE_SOURCE == "merge_request_event"
  script:
    - npm test
```

## 四、最佳实践

1. **变量命名清晰**：矩阵变量使用有意义的名称（`NODE_VERSION`而非`V`），便于在日志中识别
2. **组合控制**：使用include精确指定组合，比笛卡尔积更可控，避免不必要的并行Job
3. **与rules配合**：矩阵构建常用于MR和主分支，通过rules控制触发条件节省资源
4. **报告合并**：矩阵测试的报告应合并展示，避免在MR中看到多个分散的测试结果

## 五、常见陷阱

1. **资源超限**：矩阵组合过多导致超出Runner并行数限制，任务排队时间过长
2. **缓存竞争**：多个矩阵Job使用相同缓存key导致竞争，应在key中包含矩阵变量
3. **条件遗漏**：rules只针对整个Job而非单个矩阵组合，无法对特定组合设置不同条件
4. **变量覆盖**：include中的变量会覆盖matrix定义的同名变量，检查是否有意外覆盖

## 六、多维度矩阵

```yaml
test:
  image: node:${NODE_VERSION}
  parallel:
    matrix:
      - NODE_VERSION: ['18', '20', '21']
        DATABASE: ['postgres:15', 'mysql:8', 'mariadb:11']
  services:
    - ${DATABASE}
  script:
    - echo "Testing Node $NODE_VERSION with $DATABASE"
    - npm test
```

## 七、矩阵与环境结合

```yaml
deploy:
  stage: deploy
  parallel:
    matrix:
      - ENVIRONMENT: ['dev', 'staging', 'production']
        REGION: ['us-east-1', 'eu-west-1']
  environment:
    name: $ENVIRONMENT-$REGION
  script:
    - ./deploy.sh $ENVIRONMENT $REGION
  rules:
    - if: $CI_COMMIT_BRANCH == "main"
      when: manual
```

## 八、矩阵结果聚合

```yaml
# 矩阵构建结果汇总
test:
  parallel:
    matrix:
      - OS: ['ubuntu', 'alpine', 'debian']
        ARCH: ['amd64', 'arm64']
  script:
    - make test-${OS}-${ARCH}
  artifacts:
    reports:
      junit: test-results/${OS}-${ARCH}.xml
    paths:
      - test-results/

# 汇总报告
aggregate-results:
  stage: report
  needs:
    - test
  script:
    - |
      for f in test-results/*.xml; do
        echo "=== $f ==="
        cat "$f"
      done
  artifacts:
    reports:
      junit: test-results/*.xml
```

## 九、矩阵动态生成

```yaml
generate-matrix:
  stage: .pre
  script:
    - |
      # 根据变更文件生成测试矩阵
      PACKAGES=$(git diff --name-only HEAD~1 | grep -oP 'packages/\K[^/]+' | sort -u | jq -R -s -c 'split("\n") | map(select(. != ""))')
      echo "PACKAGES=$PACKAGES" >> build.env
  artifacts:
    reports:
      dotenv: build.env

test:
  needs: [generate-matrix]
  parallel:
    matrix:
      - PACKAGE: $PACKAGES
  script:
    - cd packages/$PACKAGE && npm test
```
