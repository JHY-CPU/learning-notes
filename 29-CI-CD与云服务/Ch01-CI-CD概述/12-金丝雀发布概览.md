# 金丝雀发布概览

## 一、概念说明

金丝雀发布是渐进式发布策略，将一小部分流量导向新版本，观察无问题后逐步增加流量。

## 二、发布流程

```
1. 部署新版本（金丝雀）
2. 分配5%流量到新版本
3. 监控指标
4. 无问题 → 增加到20%
5. 继续监控
6. 逐步增加到100%
7. 完成发布
```

## 三、Nginx权重配置

```nginx
upstream backend {
    server 192.168.1.100:8080 weight=95;  # 旧版本95%
    server 192.168.1.101:8080 weight=5;   # 新版本5%
}
```

## 四、Kubernetes金丝雀

```yaml
# Deployment - 旧版本
apiVersion: apps/v1
kind: Deployment
metadata:
  name: app-stable
spec:
  replicas: 9

---
# Deployment - 金丝雀
apiVersion: apps/v1
kind: Deployment

  name: app-canary

  replicas: 1
```

## 六、注意事项

1. **流量分配**：根据风险承受能力分配
2. **监控充分**：金丝雀期间重点监控
3. **快速回滚**：发现问题立即回滚
4. **会话粘性**：同一用户可能访问不同版本
