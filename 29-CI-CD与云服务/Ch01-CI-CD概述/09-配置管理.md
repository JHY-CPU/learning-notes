# 配置管理

## 一、概念说明

配置管理是管理应用在不同环境中的配置。敏感信息（Secret）需要安全存储。

## 二、环境变量

```yaml
# GitHub Actions环境变量
env:
  NODE_ENV: production
  API_URL: https://api.example.com

jobs:
  build:
    env:
      DEBUG: false
    steps:
      - run: echo $NODE_ENV
```

## 三、Secret管理

```yaml
# GitHub Secrets
steps:
  - run: echo ${{ secrets.DATABASE_URL }}
  - run: echo ${{ secrets.API_KEY }}

# 保护敏感信息
# Secret在日志中显示为***
```

## 四、配置文件

```bash
# .env文件（不提交到Git）
DATABASE_URL=postgres://localhost/mydb
API_KEY=your-api-key

# 使用dotenv
npm install dotenv
# 在代码中加载
```

## 五、配置中心

```bash
# Spring Cloud Config
# Apollo配置中心
# Nacos配置管理
# etcd配置存储

# 优点：集中管理、动态更新
# 缺点：增加依赖、网络开销
```

## 六、注意事项

1. **不要提交Secret**：使用.gitignore忽略.env文件
2. **加密存储**：Secret使用加密存储
3. **访问控制**：限制Secret的访问权限
4. **轮换Secret**：定期更新Secret
