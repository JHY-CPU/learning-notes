# 配置管理

## 一、概念说明

配置管理是管理应用在不同环境中的配置。敏感信息（Secret）需要安全存储。

## 二、环境变量

```yaml
# GitHub Actions环境变量
env:
  NODE_ENV: production
  API_URL: https://api.example.com

jobs:
  build:
    env:
      DEBUG: false
    steps:
      - run: echo $NODE_ENV
```

## 三、Secret管理

```yaml
# GitHub Secrets
steps:
  - run: echo ${{ secrets.DATABASE_URL }}
  - run: echo ${{ secrets.API_KEY }}

# 保护敏感信息
# Secret在日志中显示为***
```

```bash
# .env文件（不提交到Git）
```

## 六、注意事项

1. **不要提交Secret**：使用.gitignore忽略.env文件
2. **加密存储**：Secret使用加密存储
3. **访问控制**：限制Secret的访问权限
4. **轮换Secret**：定期更新Secret

## 七、HashiCorp Vault集成

```yaml
# GitHub Actions + Vault
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: hashicorp/vault-action@v2
        with:
          url: https://vault.example.com
          method: jwt
          role: github-actions
          secrets: |
            secret/data/myapp database_url | DATABASE_URL ;
            secret/data/myapp api_key | API_KEY
      - run: echo "Using secrets from Vault"
```

```bash
# Vault CLI使用
vault kv get secret/myapp
vault kv put secret/myapp db_password=new-password
vault read secret/myapp/db_password
```

## 八、AWS Secrets Manager集成

```python
# Python - 从AWS Secrets Manager获取配置
import boto3
import json

def get_secret(secret_name, region='us-east-1'):
    client = boto3.client('secretsmanager', region_name=region)
    response = client.get_secret_value(SecretId=secret_name)
    return json.loads(response['SecretString'])

# 使用示例
secrets = get_secret('myapp/production')
db_url = secrets['DATABASE_URL']
api_key = secrets['API_KEY']
```

```yaml
# CI中使用AWS Secrets Manager
- uses: aws-actions/configure-aws-credentials@v4
  with:
    aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
    aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
    aws-region: us-east-1
- name: Get secrets
  run: |
    DB_URL=$(aws secretsmanager get-secret-value \
      --secret-id myapp/production \
      --query SecretString --output text | jq -r .DATABASE_URL)
    echo "DATABASE_URL=$DB_URL" >> $GITHUB_ENV
```

## 九、配置管理最佳实践矩阵

| 层级 | 工具 | 适用场景 | 安全级别 |
|------|------|----------|----------|
| 代码内 | 常量/配置文件 | 非敏感配置 | 低 |
| 环境变量 | .env / CI变量 | 环境相关配置 | 中 |
| Secret管理 | GitHub/Vault/AWS SM | 密码、Token、密钥 | 高 |
| 配置中心 | Nacos/Apollo | 动态配置、功能开关 | 中-高 |
| 加密存储 | SOPS/age | 文件级加密 | 高 |
