# 自动化测试

## 一、概念说明

自动化测试是CI/CD的核心，包括单元测试、集成测试、端到端测试等。

## 二、测试金字塔

```
        E2E测试（少量）
       /           \
    集成测试（适量）
   /                 \
单元测试（大量）
```

## 三、测试类型

```bash
# 单元测试 - 测试单个函数/类
# 集成测试 - 测试模块间交互
# E2E测试 - 测试完整用户流程
# 性能测试 - 测试系统性能
# 安全测试 - 测试安全漏洞
```

## 四、CI集成

```yaml
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm install
      - run: npm run test:unit
      - run: npm run test:integration
      - run: npm run test:e2e
      
  coverage:
    needs: test
    steps:
      - run: npm run test:coverage
      - uses: codecov/codecov-action@v3
```

## 五、测试覆盖率

```bash
# 覆盖率目标
# 行覆盖率 > 80%
# 分支覆盖率 > 70%
# 函数覆盖率 > 80%

# 覆盖率门禁
# CI失败如果覆盖率下降
```

## 六、注意事项

1. **测试速度**：单元测试应在秒级完成
2. **测试隔离**：测试之间不能相互影响
3. **测试数据**：使用fixture，不用生产数据
4. **持续维护**：定期更新测试用例
