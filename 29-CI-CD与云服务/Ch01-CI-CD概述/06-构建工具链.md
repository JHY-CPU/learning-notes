# 构建工具链

## 一、概念说明

构建工具将源代码编译、打包成可部署的制品。不同语言有不同的构建工具。

## 二、常用构建工具

```bash
# JavaScript/Node.js
npm install        # 安装依赖
npm run build      # 构建
npm test           # 测试
npm run lint       # 代码检查

# Python
pip install -r requirements.txt
python setup.py build
pytest

# Java/Maven
mvn clean install
mvn test
mvn package

# Go
go build ./...
go test ./...
go mod download
```

## 三、CI集成示例

```yaml
# Node.js构建
- run: npm ci
- run: npm run lint
- run: npm test
- run: npm run build

# Maven构建
- uses: actions/setup-java@v4
  with:
    java-version: '17'
- run: mvn clean install

# Go构建
- uses: actions/setup-go@v5
  with:
    go-version: '1.21'
- run: go build ./...
- run: go test ./...
```

## 四、注意事项

1. **版本锁定**：使用lock文件锁定依赖版本
2. **缓存依赖**：缓存node_modules/.m2等
3. **并行构建**：利用多核CPU并行编译
4. **构建优化**：增量编译，避免全量构建

## 五、各语言完整CI构建配置

### Node.js完整构建

```yaml
name: Node.js Build
on: [push, pull_request]

jobs:
  build:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        node: [18, 20]
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node }}
          cache: 'npm'
      - run: npm ci
      - run: npm run lint
      - run: npm run type-check
      - run: npm test -- --coverage
      - run: npm run build
      - uses: actions/upload-artifact@v4
        with:
          name: dist-node${{ matrix.node }}
          path: dist/
```

### Python完整构建

```yaml
name: Python Build
on: [push, pull_request]

jobs:
  build:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        python: ['3.10', '3.11', '3.12']
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: ${{ matrix.python }}
          cache: 'pip'
      - run: |
          pip install -r requirements.txt
          pip install -r requirements-dev.txt
      - run: python -m flake8 src/
      - run: python -m mypy src/
      - run: python -m pytest --cov=src --cov-report=xml
      - run: python -m build
```

### Java/Maven完整构建

```yaml
name: Java Build
on: [push, pull_request]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-java@v4
        with:
          java-version: '17'
          distribution: 'temurin'
          cache: 'maven'
      - run: mvn clean compile
      - run: mvn test
      - run: mvn package -DskipTests
      - uses: actions/upload-artifact@v4
        with:
          name: jar
          path: target/*.jar
```

### Go完整构建

```yaml
name: Go Build
on: [push, pull_request]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
        with:
          go-version: '1.21'
          cache: true
      - run: go vet ./...
      - run: go test -race -coverprofile=coverage.out ./...
      - run: go build -o bin/app ./cmd/app
      - run: go build -o bin/cli ./cmd/cli
```

### Rust完整构建

```yaml
name: Rust Build
on: [push, pull_request]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
        with:
          components: clippy, rustfmt
      - uses: Swatinem/rust-cache@v2
      - run: cargo fmt -- --check
      - run: cargo clippy -- -D warnings
      - run: cargo test
      - run: cargo build --release
```

## 六、构建缓存策略对比

| 工具 | 缓存目录 | Cache Key | setup-* 内置缓存 |
|------|----------|-----------|------------------|
| npm | ~/.npm | package-lock.json | cache: 'npm' |
| pip | ~/.cache/pip | requirements.txt | cache: 'pip' |
| Maven | ~/.m2/repository | pom.xml | cache: 'maven' |
| Go | ~/go/pkg/mod | go.sum | cache: true |
| Cargo | ~/.cargo/registry | Cargo.lock | rust-cache |
| Yarn | .yarn/cache | yarn.lock | cache: 'yarn' |
