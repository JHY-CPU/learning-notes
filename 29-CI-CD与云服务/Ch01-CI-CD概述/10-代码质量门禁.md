# 代码质量门禁

## 一、概念说明

代码质量门禁（Quality Gate）是CI/CD中检查代码质量的关卡，不通过则阻止合并或部署。

## 二、门禁类型

```bash
# 1. 代码风格检查
# ESLint / Prettier / Black / Checkstyle

# 2. 单元测试通过
# 测试覆盖率 > 80%

# 3. 安全扫描
# 依赖漏洞扫描
# 静态安全分析

# 4. 性能检查
# 构建时间不超过阈值
# 包大小不超过限制
```

## 三、CI集成

```yaml
jobs:
  quality-gate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      # 代码风格
      - run: npm run lint
      
      # 测试覆盖率
      - run: npm run test:coverage
      - uses: codecov/codecov-action@v3
      
      # 安全扫描
      - uses: github/codeql-action/analyze@v3
      
      # 包大小检查
      - uses: preactjs/compressed-size-action@v2
```

## 四、SonarQube集成

```yaml
- uses: sonarqube-quality-gate-action@v1
  with:
    sonar-token: ${{ secrets.SONAR_TOKEN }}
```

## 五、注意事项

1. **门禁配置**：根据项目需要配置
2. **快速反馈**：门禁检查要快
3. **逐步引入**：不要一开始就全部开启
4. **持续改进**：根据反馈调整门禁规则

## 六、SonarQube完整配置

```yaml
# GitHub Actions + SonarQube
sonarqube:
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v4
      with:
        fetch-depth: 0
    - uses: sonarsource/sonarqube-scan-action@master
      env:
        SONAR_TOKEN: ${{ secrets.SONAR_TOKEN }}
        SONAR_HOST_URL: ${{ secrets.SONAR_HOST_URL }}
    - name: Quality Gate check
      uses: sonarsource/sonarqube-quality-gate-action@master
      timeout-minutes: 5
      env:
        SONAR_TOKEN: ${{ secrets.SONAR_TOKEN }}
```

```properties
# sonar-project.properties
sonar.projectKey=myapp
sonar.projectName=My Application
sonar.sources=src
sonar.tests=tests
sonar.javascript.lcov.reportPaths=coverage/lcov.info
sonar.coverage.exclusions=**/*.test.js,**/*.spec.js
sonar.qualitygate.wait=true
```

## 七、ESLint + Prettier CI配置

```yaml
# 前端代码质量门禁
frontend-quality:
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v4
    - uses: actions/setup-node@v4
      with:
        node-version: '20'
        cache: 'npm'
    - run: npm ci
    - name: Lint check
      run: npm run lint -- --max-warnings 0
    - name: Format check
      run: npm run format:check
    - name: Type check
      run: npm run type-check
```

```javascript
// .eslintrc.js
module.exports = {
  extends: [
    'eslint:recommended',
    'plugin:@typescript-eslint/recommended',

    'no-console': 'warn',
    'complexity': ['error', 10],
    'max-lines': ['error', 300],
    'max-depth': ['error', 4]
  }
};
```

## 八、Python代码质量门禁

```yaml
python-quality:
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v4
    - uses: actions/setup-python@v5
      with:
        python-version: '3.11'
    - run: pip install flake8 black mypy bandit
    - name: Lint
      run: flake8 src/ --max-line-length=100
    - name: Format
      run: black --check src/
    - name: Type check
      run: mypy src/ --strict
    - name: Security
      run: bandit -r src/ -ll
```

## 九、门禁指标与DORA对照

```
代码质量门禁金字塔:
                    安全扫描 (最高成本)
                   /
          性能基准测试
         /
    集成测试覆盖率
   /
  单元测试 + 代码风格 (最低成本, 每次提交必跑)

门禁决策矩阵:
  PR提交 → 代码风格 + 单元测试 (必须通过)
  合并到主分支 → + 集成测试 + 覆盖率 (必须通过)
  版本发布 → + 安全扫描 + 性能测试 (必须通过)
  生产部署 → + 金丝雀验证 + 监控 (自动验证)
```
