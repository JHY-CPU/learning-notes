# 代码质量门禁

## 一、概念说明

代码质量门禁（Quality Gate）是CI/CD中检查代码质量的关卡，不通过则阻止合并或部署。

## 二、门禁类型

```bash
# 1. 代码风格检查
# ESLint / Prettier / Black / Checkstyle

# 2. 单元测试通过
# 测试覆盖率 > 80%

# 3. 安全扫描
# 依赖漏洞扫描
# 静态安全分析

# 4. 性能检查
# 构建时间不超过阈值
# 包大小不超过限制
```

## 三、CI集成

```yaml
jobs:
  quality-gate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      # 代码风格
      - run: npm run lint
      
      # 测试覆盖率
      - run: npm run test:coverage
      - uses: codecov/codecov-action@v3
      
      # 安全扫描
      - uses: github/codeql-action/analyze@v3
      
      # 包大小检查
      - uses: preactjs/compressed-size-action@v2
```

## 四、SonarQube集成

```yaml
- uses: sonarqube-quality-gate-action@v1
  with:
    sonar-token: ${{ secrets.SONAR_TOKEN }}
```

## 五、注意事项

1. **门禁配置**：根据项目需要配置
2. **快速反馈**：门禁检查要快
3. **逐步引入**：不要一开始就全部开启
4. **持续改进**：根据反馈调整门禁规则
