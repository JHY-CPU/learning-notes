# 版本控制集成

## 一、概念说明

版本控制是CI/CD的基础。Git是最流行的版本控制系统，与CI/CD工具深度集成。

## 二、Git工作流

```bash
# Git Flow
main      ← 生产分支
develop   ← 开发分支
feature/* ← 功能分支
release/* ← 发布分支
hotfix/*  ← 热修复分支

# GitHub Flow（推荐）
main      ← 主分支
feature/* ← 功能分支，PR合并到main
```

## 三、CI/CD触发

```yaml
# GitHub Actions触发
on:
  push:
    branches: [main, develop]
    paths: ['src/**']
  pull_request:
    branches: [main]
  release:
    types: [published]
```

## 四、分支策略

```bash
# 保护main分支
# - 需要PR审查
# - 需要CI通过
# - 禁止force push
# - 需要签名提交
```

## 五、注意事项

1. **分支保护**：main分支必须保护
2. **PR检查**：CI必须通过才能合并
3. **提交规范**：使用conventional commits
4. **标签管理**：使用语义化版本号
