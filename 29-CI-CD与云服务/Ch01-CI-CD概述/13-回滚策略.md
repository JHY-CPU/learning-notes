# 回滚策略

## 一、概念说明

回滚是在部署失败时恢复到上一个稳定版本的操作。快速可靠的回滚是CI/CD的重要组成部分。

## 二、回滚方式

```bash
# 1. 镜像回滚
# 使用上一个版本的Docker镜像
docker pull app:previous-version
docker stop current-container
docker run app:previous-version

# 2. Kubernetes回滚
kubectl rollout undo deployment/myapp
kubectl rollout undo deployment/myapp --to-revision=2

# 3. Git回滚
git revert HEAD
git push
```

## 三、数据库回滚

```bash
# 数据库迁移回滚
# Flyway
flyway undo

# Liquibase
liquibase rollback

# 自定义迁移脚本
# 每个up迁移对应down迁移
```

## 四、自动回滚

```yaml
# ArgoCD自动回滚
apiVersion: argoproj.io/v1alpha1
kind: Application
spec:
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
```

## 五、注意事项

1. **回滚测试**：定期测试回滚流程
2. **数据兼容**：新版本数据兼容旧版本
3. **快速回滚**：回滚应在分钟级完成
4. **回滚记录**：记录回滚原因和时间
