# 制品管理

## 一、概念说明

制品（Artifact）是构建过程的输出，包括Docker镜像、JAR包、npm包等。需要使用制品仓库管理。

## 二、制品仓库类型

```bash
# Docker镜像仓库
# Docker Hub - 公共仓库
# GitHub Container Registry - GHCR
# AWS ECR - 云容器仓库
# Harbor - 私有仓库

# 包仓库
# npm registry
# Maven Central / Nexus
# PyPI
# Go Modules
```

## 三、GitHub Actions示例

```yaml
# 上传制品
- uses: actions/upload-artifact@v4
  with:
    name: build-output
    path: dist/

# 下载制品
- uses: actions/download-artifact@v4
  with:
    name: build-output

# Docker镜像构建和推送
- uses: docker/build-push-action@v5
  with:
    push: true
    tags: user/app:latest
```

## 四、版本管理

```bash
# 语义化版本
# 主版本.次版本.修订版本
# 1.0.0 - 初始版本
# 1.1.0 - 新功能
# 1.1.1 - Bug修复

# 自动版本号
git describe --tags
# v1.0.0-15-g2414721
```

## 五、注意事项

1. **版本唯一**：每个制品版本号唯一
2. **不可变性**：制品发布后不可修改
3. **安全扫描**：制品发布前进行安全扫描
4. **保留策略**：设置制品保留期限

## 六、Docker镜像构建与推送完整流程

```yaml
name: Build and Push Docker Image

on:
  push:
    branches: [main]
    tags: ['v*']

env:
  REGISTRY: ghcr.io
  IMAGE_NAME: ${{ github.repository }}

jobs:
  build-and-push:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      packages: write
    steps:
      - uses: actions/checkout@v4

      - uses: docker/login-action@v3
        with:
          registry: ${{ env.REGISTRY }}
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Extract metadata
        id: meta
        uses: docker/metadata-action@v5
        with:
          images: ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}
          tags: |
            type=ref,event=branch
            type=ref,event=pr
            type=semver,pattern={{version}}
            type=semver,pattern={{major}}.{{minor}}
            type=sha,prefix=

      - uses: docker/build-push-action@v5
        with:
          context: .
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          labels: ${{ steps.meta.outputs.labels }}
          cache-from: type=gha
          cache-to: type=gha,mode=max
```

## 七、npm包发布流程

```yaml
name: Publish to npm

on:
  release:
    types: [published]

jobs:
  publish:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          registry-url: 'https://registry.npmjs.org'
      - run: npm ci
      - run: npm test
      - run: npm run build
      - run: npm publish
        env:
          NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}
```

## 八、Harbor私有镜像仓库配置

```yaml
# docker-compose.yml for Harbor
version: '3.8'
services:
  harbor-core:
    image: goharbor/harbor-core:v2.10.0
    environment:
      - CORE_SECRET=your-secret
      - DATABASE_HOST=harbor-db
    volumes:
      - ./data:/data
    ports:
      - "80:8080"
      - "443:8443"
```

```bash
# 使用Harbor
docker login harbor.example.com
docker tag myapp:latest harbor.example.com/myproject/myapp:v1.0.0
docker push harbor.example.com/myproject/myapp:v1.0.0
```

## 九、制品安全扫描集成

```yaml
# 制品发布前安全扫描
scan-artifact:
  stage: security
  script:
    - trivy image --severity CRITICAL,HIGH --exit-code 1 $IMAGE_NAME:$TAG
    - grype $IMAGE_NAME:$TAG --fail-on high
  allow_failure: false
```
