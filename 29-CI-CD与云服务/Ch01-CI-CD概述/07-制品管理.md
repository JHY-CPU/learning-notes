# 制品管理

## 一、概念说明

制品（Artifact）是构建过程的输出，包括Docker镜像、JAR包、npm包等。需要使用制品仓库管理。

## 二、制品仓库类型

```bash
# Docker镜像仓库
# Docker Hub - 公共仓库
# GitHub Container Registry - GHCR
# AWS ECR - 云容器仓库
# Harbor - 私有仓库

# 包仓库
# npm registry
# Maven Central / Nexus
# PyPI
# Go Modules
```

## 三、GitHub Actions示例

```yaml
# 上传制品
- uses: actions/upload-artifact@v4
  with:
    name: build-output
    path: dist/

# 下载制品
- uses: actions/download-artifact@v4
  with:
    name: build-output

# Docker镜像构建和推送
- uses: docker/build-push-action@v5
  with:
    push: true
    tags: user/app:latest
```

## 四、版本管理

```bash
# 语义化版本
# 主版本.次版本.修订版本
# 1.0.0 - 初始版本
# 1.1.0 - 新功能
# 1.1.1 - Bug修复

# 自动版本号
git describe --tags
# v1.0.0-15-g2414721
```

## 五、注意事项

1. **版本唯一**：每个制品版本号唯一
2. **不可变性**：制品发布后不可修改
3. **安全扫描**：制品发布前进行安全扫描
4. **保留策略**：设置制品保留期限
