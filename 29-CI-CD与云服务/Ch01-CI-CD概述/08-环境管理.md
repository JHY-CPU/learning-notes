# 环境管理

## 一、概念说明

CI/CD涉及多个环境：开发（Dev）、测试（Test）、预生产（Staging）、生产（Prod）。每个环境的配置和数据不同。

## 二、环境分类

```bash
# 开发环境（Dev）
# 开发者本地或共享开发服务器
# 配置宽松，数据可重置

# 测试环境（Test）
# 自动化测试运行的环境
# 配置接近生产，使用测试数据

# 预生产环境（Staging）
# 生产环境的镜像
# 配置与生产一致，数据脱敏

# 生产环境（Prod）
# 真实用户使用的环境
# 严格配置，真实数据
```

## 三、环境配置

```yaml
# GitHub Actions环境
jobs:
  deploy-staging:
    environment: staging
    steps:
      - run: echo "Deploy to staging"
  
  deploy-prod:
    environment: production
    needs: deploy-staging
    steps:
      - run: echo "Deploy to production"
```

## 四、配置管理

```bash
# 环境变量
# 不同环境使用不同的环境变量

# Secret管理
# 生产Secret存储在GitHub Secrets中

```

## 五、注意事项

1. **环境隔离**：各环境数据和配置隔离
2. **配置差异**：通过环境变量管理配置差异
3. **数据脱敏**：测试环境使用脱敏数据
4. **环境一致性**：保持环境配置的一致性
