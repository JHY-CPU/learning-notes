# 环境管理

## 一、概念说明

CI/CD涉及多个环境：开发（Dev）、测试（Test）、预生产（Staging）、生产（Prod）。每个环境的配置和数据不同。

## 二、环境分类

```bash
# 开发环境（Dev）
# 开发者本地或共享开发服务器
# 配置宽松，数据可重置

# 测试环境（Test）
# 自动化测试运行的环境
# 配置接近生产，使用测试数据

# 预生产环境（Staging）
# 生产环境的镜像
# 配置与生产一致，数据脱敏

# 生产环境（Prod）
# 真实用户使用的环境
# 严格配置，真实数据
```

## 三、环境配置

```yaml
# GitHub Actions环境
jobs:
  deploy-staging:
    environment: staging
    steps:
      - run: echo "Deploy to staging"
  
  deploy-prod:
    environment: production
    needs: deploy-staging
    steps:
      - run: echo "Deploy to production"
```

```bash
# 环境变量

# 生产Secret存储在GitHub Secrets中

```

## 五、注意事项

1. **环境隔离**：各环境数据和配置隔离
2. **配置差异**：通过环境变量管理配置差异
3. **数据脱敏**：测试环境使用脱敏数据
4. **环境一致性**：保持环境配置的一致性

## 六、多环境部署流水线

```yaml
name: Multi-Environment Deploy

on:
  push:
    branches: [main]
    tags: ['v*']

jobs:
  deploy-dev:
    runs-on: ubuntu-latest
    environment:
      name: development
      url: https://dev.example.com
    steps:
      - uses: actions/checkout@v4
      - name: Deploy to Dev
        run: |
          echo "Deploying to development..."
          kubectl set image deployment/app app=$IMAGE:dev

  deploy-staging:
    needs: deploy-dev
    runs-on: ubuntu-latest
    environment:
      name: staging
      url: https://staging.example.com
    steps:
      - uses: actions/checkout@v4
      - name: Deploy to Staging
        run: |
          echo "Deploying to staging..."
          kubectl set image deployment/app app=$IMAGE:staging
      - name: Run Smoke Tests
        run: |
          curl -f https://staging.example.com/health || exit 1
          npm run test:smoke

  deploy-production:
    needs: deploy-staging
    runs-on: ubuntu-latest
    environment:
      name: production
      url: https://example.com
    steps:
      - uses: actions/checkout@v4
      - name: Deploy to Production
        run: |
          echo "Deploying to production..."
          kubectl set image deployment/app app=$IMAGE:prod
```

## 七、环境配置模板

```bash
# .env.development
APP_ENV=development
API_URL=https://api-dev.example.com
DEBUG=true
LOG_LEVEL=debug
DATABASE_URL=postgres://dev-db:5432/myapp

# .env.staging
APP_ENV=staging
API_URL=https://api-staging.example.com
DEBUG=false
LOG_LEVEL=info
DATABASE_URL=postgres://staging-db:5432/myapp

# .env.production
APP_ENV=production
API_URL=https://api.example.com
DEBUG=false
LOG_LEVEL=error
DATABASE_URL=postgres://prod-db:5432/myapp
```

## 八、GitHub Environments配置

```yaml
# 在仓库Settings > Environments中配置
# development:
#   - 无保护规则
#   - 部署URL: https://dev.example.com
#   - 环境变量: DEBUG=true
#
# staging:
#   - 保护规则: 需要main分支
#   - 部署URL: https://staging.example.com
#   - 环境变量: DEBUG=false
#
# production:
#   - 保护规则: 需要审批人
#   - 等待计时器: 5分钟
#   - 部署URL: https://example.com
#   - Secrets: PROD_*
```

## 九、基础设施环境管理（Terraform）

```hcl

environment = "development"
instance_type = "t3.micro"

max_size = 2
enable_monitoring = false

environment = "production"
instance_type = "t3.large"
min_size = 3
max_size = 10
enable_monitoring = true
```
