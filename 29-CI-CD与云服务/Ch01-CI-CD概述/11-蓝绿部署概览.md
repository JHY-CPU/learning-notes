# 蓝绿部署概览

## 一、概念说明

蓝绿部署通过维护两个相同的生产环境（蓝/绿），实现零停机部署。流量从旧环境切换到新环境。

## 二、部署流程

```
1. 蓝环境运行（当前生产）
2. 部署到绿环境（新版本）
3. 测试绿环境
4. 切换流量到绿环境
5. 绿环境成为生产环境
6. 蓝环境作为回滚备份
```

## 三、Nginx配置示例

```nginx
# 蓝环境（当前）
upstream blue {
    server 192.168.1.100:8080;
}

# 绿环境（新版本）
upstream green {
    server 192.168.1.101:8080;
}

# 切换
server {
    listen 80;
    location / {
        proxy_pass http://green;  # 切换到绿环境
    }
}
```

## 四、Kubernetes实现

```yaml
apiVersion: v1
kind: Service
metadata:
  name: app-service
spec:
  selector:
    app: myapp
    version: blue  # 切换为green
  ports:
    - port: 80
```

## 五、注意事项

1. **资源成本**：需要双倍资源
2. **数据库兼容**：新版本兼容旧数据库
3. **快速回滚**：只需切换流量
4. **会话处理**：处理好用户会话

## 六、AWS蓝绿部署（ECS）

```yaml
# AWS CodeDeploy蓝绿部署配置
Resources:
  DeploymentGroup:
    Type: AWS::CodeDeploy::DeploymentGroup
    Properties:
      ApplicationName: my-app
      ServiceRoleArn: !GetAtt CodeDeployRole.Arn
      DeploymentConfigName: CodeDeployDefault.ECSCanary10Percent5Minutes
      BlueGreenDeploymentConfiguration:
        TerminateBlueInstancesOnDeploymentSuccess:
          ACTION: TERMINATE
          TERMINATION_WAIT_TIME_IN_MINUTES: 5
        DeploymentReadyOption:
          ACTION_ON_TIMEOUT: CONTINUE_DEPLOYMENT
        GreenFleetProvisioningOption:
          ACTION: COPY_AUTO_SCALING_GROUP
```

## 七、蓝绿部署自动化脚本

```bash
#!/bin/bash
# blue-green-deploy.sh

CURRENT_COLOR=$(curl -s http://localhost/health | jq -r '.color')
if [ "$CURRENT_COLOR" = "blue" ]; then
  NEW_COLOR="green"
else
  NEW_COLOR="blue"
fi

echo "Current: $CURRENT_COLOR, Deploying to: $NEW_COLOR"

# 部署新版本
docker pull myapp:${NEW_COLOR}
docker-compose -f docker-compose.${NEW_COLOR}.yml up -d

# 健康检查
for i in {1..30}; do
  if curl -f http://localhost:${NEW_COLOR_PORT}/health; then
    echo "New version healthy"
    break
  fi
  sleep 2
done

# 切换流量
nginx -s reload -c /etc/nginx/nginx.${NEW_COLOR}.conf

echo "Switched to $NEW_COLOR"
```

## 八、蓝绿部署与数据库迁移

```
关键原则: 数据库变更必须向后兼容

安全的数据库变更:
  ADD COLUMN (带默认值)
  CREATE TABLE
  CREATE INDEX (CONCURRENTLY)

危险的数据库变更:
  DROP COLUMN → 分两步: 先停止使用, 再删除
  RENAME COLUMN → 添加新列, 双写, 迁移, 删除旧列
  CHANGE COLUMN TYPE → 添加新列, 双写, 迁移

迁移步骤:
  1. 部署新版本（兼容旧数据库）
  2. 执行数据库迁移
  3. 验证新版本正常
  4. 如有问题，回滚到旧版本（数据库仍然兼容）
```
