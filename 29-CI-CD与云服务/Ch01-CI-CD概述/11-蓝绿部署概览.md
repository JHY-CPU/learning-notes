# 蓝绿部署概览

## 一、概念说明

蓝绿部署通过维护两个相同的生产环境（蓝/绿），实现零停机部署。流量从旧环境切换到新环境。

## 二、部署流程

```
1. 蓝环境运行（当前生产）
2. 部署到绿环境（新版本）
3. 测试绿环境
4. 切换流量到绿环境
5. 绿环境成为生产环境
6. 蓝环境作为回滚备份
```

## 三、Nginx配置示例

```nginx
# 蓝环境（当前）
upstream blue {
    server 192.168.1.100:8080;
}

# 绿环境（新版本）
upstream green {
    server 192.168.1.101:8080;
}

# 切换
server {
    listen 80;
    location / {
        proxy_pass http://green;  # 切换到绿环境
    }
}
```

## 四、Kubernetes实现

```yaml
apiVersion: v1
kind: Service
metadata:
  name: app-service
spec:
  selector:
    app: myapp
    version: blue  # 切换为green
  ports:
    - port: 80
```

## 五、注意事项

1. **资源成本**：需要双倍资源
2. **数据库兼容**：新版本兼容旧数据库
3. **快速回滚**：只需切换流量
4. **会话处理**：处理好用户会话
