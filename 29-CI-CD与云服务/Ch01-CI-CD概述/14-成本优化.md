# 成本优化

## 一、概念说明

CI/CD会产生计算和存储成本。优化资源使用可以降低CI/CD成本。

## 二、Runner优化

```yaml
# GitHub Actions
# 使用合适大小的Runner
runs-on: ubuntu-latest  # 免费
# runs-on: ubuntu-latest-8-cores  # 付费

# 并行任务
jobs:
  test:
    strategy:
      matrix:
        node: [16, 18, 20]
    runs-on: ubuntu-latest
```

## 三、缓存优化

```yaml
# 缓存依赖
- uses: actions/cache@v4
  with:
    path: ~/.npm
    key: ${{ runner.os }}-node-${{ hashFiles('**/package-lock.json') }}

# 缓存Docker层
- uses: docker/build-push-action@v5
  with:
    cache-from: type=gha
    cache-to: type=gha,mode=max
```

## 四、条件执行

```yaml
# 只在需要时运行
jobs:
  deploy:
    if: github.ref == 'refs/heads/main'
    
  test:
    if: contains(github.event.head_commit.message, '[test]')
```

## 五、注意事项

1. **监控用量**：定期检查CI/CD使用量
2. **清理旧制品**：设置制品保留策略
3. **优化构建**：减少不必要的构建
4. **选择合适Runner**：根据任务选择Runner大小

## 六、GitHub Actions分钟数优化

```yaml
# 通过路径过滤避免不必要构建
on:
  push:
    paths:
      - 'src/**'
      - 'package.json'
      - 'package-lock.json'
    paths-ignore:
      - '**.md'
      - 'docs/**'
      - '.github/ISSUE_TEMPLATE/**'

# 使用concurrency避免重复运行
concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true
```

## 七、自托管Runner成本对比

```
GitHub托管Runner成本:
  Linux: $0.008/分钟 (2核) | $0.016/分钟 (4核) | $0.032/分钟 (8核)
  Windows: $0.016/分钟 (2核)
  macOS: $0.08/分钟 (3核)

自托管Runner:
  EC2 t3.medium (2核4GB): ~$0.0416/小时 = $0.0007/分钟
  EC2 t3.large (2核8GB): ~$0.0832/小时 = $0.0014/分钟

成本对比（每月10000分钟）:
  GitHub Linux 2核: $80
  自托管 t3.medium: $7 + EC2费用 = ~$37
  节省: ~54%

适合自托管的场景:
  - 构建量大（>5000分钟/月）
  - 需要特殊硬件（GPU、大内存）
  - 需要访问私有网络资源
  - 安全合规要求
```

## 八、制品存储成本优化

```yaml
# 合理设置保留期
artifacts:
  paths:
    - dist/
  retention-days: 7  # 构建产物保留7天

release-artifacts:
  paths:
    - release/
  retention-days: 90  # 发布包保留90天

# 只上传必要文件
artifacts:
  paths:
    - dist/
    - coverage/
  # 不要上传 node_modules, .git 等
```
