# 灰度发布

## 一、概念说明

灰度发布是将新版本按特定规则（用户特征、地域、设备等）逐步开放给用户，而不是简单的流量百分比分配。与金丝雀部署相比，灰度更注重用户维度的精确控制。

| 维度 | 示例 | 用途 |
|------|------|------|
| 用户ID | 特定用户列表 | 内测用户 |
| 用户属性 | VIP用户、企业用户 | 分群测试 |
| 地域 | 北京、上海 | 区域验证 |
| 设备 | iOS/Android | 平台兼容 |
| 随机百分比 | 10%、30% | 平滑放量 |

## 二、具体用法

### Nginx灰度路由

```nginx
# 基于Cookie的灰度
map $cookie_canary $backend_group {
    "true"  canary;
    default stable;
}

# 基于Header的灰度
map $http_x_canary $backend_group {
    "true"  canary;
    default stable;
}

# 基于IP的灰度（指定IP段）
geo $is_canary {
    default 0;
    10.0.1.0/24 1;  # 内部测试网段
    1.2.3.4/32 1;   # 特定测试IP
}

upstream stable {
    server web-app-stable:8080;
}

upstream canary {
    server web-app-canary:8080;
}

server {
    listen 80;

    location / {
        proxy_pass http://$backend_group;
    }
}
```

### 应用层灰度实现

```python
from flask import Flask, request, jsonify
import hashlib
import re

app = Flask(__name__)

# 灰度规则配置
GRAY_RULES = {
    # 用户白名单（内测用户）
    'whitelist': {'user-001', 'user-002', 'user-003'},

    # 用户属性规则
    'user_attrs': {
        'plan': 'premium',  # 仅高级用户
        'region': 'cn',     # 中国用户
    },

    # 百分比规则
    'percentage': 10,  # 10%的用户

    # 地域规则
    'regions': ['beijing', 'shanghai'],
}

def is_canary_user(user):
    """判断用户是否命中灰度"""
    # 1. 白名单优先
    if user.get('id') in GRAY_RULES['whitelist']:
        return True

    # 2. 用户属性匹配
    for attr, value in GRAY_RULES['user_attrs'].items():
        if user.get(attr) != value:
            return False

    # 3. 地域匹配
    if GRAY_RULES['regions']:
        if user.get('region') not in GRAY_RULES['regions']:
            return False

    # 4. 百分比（基于用户ID hash，保证同一用户每次结果一致）
    if GRAY_RULES['percentage'] > 0:
        hash_val = int(hashlib.md5(user['id'].encode()).hexdigest(), 16)
        if (hash_val % 100) >= GRAY_RULES['percentage']:
            return False

    return True

@app.route('/api/v2/feature')
def new_feature():
    user = get_current_user()

    if is_canary_user(user):
        # 新版本逻辑
        return new_version_logic(user)
    else:
        # 旧版本逻辑
        return old_version_logic(user)
```

### 基于请求头的灰度

```yaml
# Istio基于Header灰度
apiVersion: networking.istio.io/v1beta1
kind: VirtualService
metadata:
  name: web-app-vs
spec:
  hosts:
  - web-app-svc
  http:
  # 内测用户走金丝雀
  - match:
    - headers:
        x-user-type:
          exact: "beta-tester"
    route:
    - destination:
        host: web-app-svc
        subset: canary

  # 特定地域走金丝雀
  - match:
    - headers:
        x-region:
          exact: "beijing"
    route:
    - destination:
        host: web-app-svc
        subset: canary

  # 其他按百分比
  - route:
    - destination:
        host: web-app-svc
        subset: stable
      weight: 90
    - destination:
        host: web-app-svc
        subset: canary
      weight: 10
```

### 灰度进度管理

```bash
#!/bin/bash
# canary-progress.sh

declare -A STAGES=(
    ["stage1"]=5
    ["stage2"]=10
    ["stage3"]=25
    ["stage4"]=50
    ["stage5"]=75
    ["stage6"]=100
)

for stage in "${!STAGES[@]}"; do
    weight=${STAGES[$stage]}
    echo "=== 阶段: ${stage}, 灰度比例: ${weight}% ==="

    # 更新灰度比例
    update_gray_percentage $weight

    # 等待验证
    echo "等待观察期 (10分钟)..."
    sleep 600

    # 自动验证
    if validate_metrics; then
        echo "阶段 ${stage} 验证通过"
    else
        echo "阶段 ${stage} 验证失败，回滚！"
        rollback
        exit 1
    fi
done

echo "灰度发布完成！"
```

## 三、注意事项与常见陷阱

1. **用户一致性**：同一用户每次请求应命中相同版本
2. **规则优先级**：白名单 > 属性 > 百分比
3. **日志区分**：新旧版本的日志要可区分
4. **数据兼容**：新旧版本操作的数据格式要兼容
5. **回滚粒度**：支持按用户群回滚
6. **监控细分**：按灰度维度细分监控指标
7. **沟通协调**：灰度规则变更需要团队知晓
