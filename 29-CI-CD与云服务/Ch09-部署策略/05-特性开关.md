# 特性开关

## 一、概念说明

特性开关（Feature Flag/Feature Toggle）是一种代码级别的部署控制技术，通过配置开关控制功能的开启/关闭，实现功能与发布的解耦。

| 开关类型 | 用途 | 生命周期 |
|----------|------|----------|
| 发布开关 | 控制新功能上线 | 短期 |
| 实验开关 | A/B测试 | 中期 |
| 运营开关 | 降级、熔断 | 长期 |
| 权限开关 | 灰度用户 | 长期 |

## 二、具体用法

### 基本实现

```python
# 简单的特性开关实现
import json
from functools import wraps

class FeatureFlags:
    def __init__(self, config_path='flags.json'):
        with open(config_path) as f:
            self.flags = json.load(f)

    def is_enabled(self, flag_name, user_id=None, context=None):
        flag = self.flags.get(flag_name, {})
        if not flag.get('enabled', False):
            return False

        # 基于用户ID的灰度
        if user_id and flag.get('user_ids'):
            if user_id not in flag['user_ids']:
                return False

        # 基于百分比的灰度
        if flag.get('percentage'):
            import hashlib
            hash_val = int(hashlib.md5(f"{flag_name}:{user_id}".encode()).hexdigest(), 16)
            if (hash_val % 100) >= flag['percentage']:
                return False

        return True

flags = FeatureFlags()

# 使用特性开关
def feature_flag(flag_name):
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            user_id = kwargs.get('user_id')
            if flags.is_enabled(flag_name, user_id=user_id):
                return func(*args, **kwargs)
            else:
                return legacy_behavior(*args, **kwargs)
        return wrapper
    return decorator

@feature_flag('new_checkout_flow')
def process_checkout(cart, user_id=None):
    """新版结账流程"""
    return new_checkout(cart)

# 配置文件
# flags.json
{
    "new_checkout_flow": {
        "enabled": true,
        "percentage": 20,
        "user_ids": ["beta_user_1", "beta_user_2"]
    },
    "dark_mode": {
        "enabled": true,
        "percentage": 100
    },
    "new_recommendation_engine": {
        "enabled": false
    }
}
```

### Spring Boot集成

```java
// 使用Togglz框架
@FeatureToggle(NewCheckout.class)
@GetMapping("/checkout")
public String checkout(Model model) {
    if (NewCheckout.isActive()) {
        return "checkout-v2";
    }
    return "checkout";
}

public enum NewCheckout implements Feature {
    @EnabledByDefault
    @Label("新版结账流程")
    NEW_CHECKOUT;

    public boolean isActive() {
        return FeatureContext.getFeatureManager().isActive(this);
    }
}

// application.yml
togglz:
  features:
    NEW_CHECKOUT:
      enabled: true
      strategy: gradual
      param:
        percentage: 20
```

### 前端特性开关

```javascript
// React特性开关
const FeatureFlag = ({ flagName, fallback, children }) => {
    const flags = useContext(FeatureFlagContext);
    if (flags[flagName]) {
        return children;
    }
    return fallback || null;
};

// 使用
<FeatureFlag flagName="newDashboard" fallback={<OldDashboard />}>
    <NewDashboard />
</FeatureFlag>

// 配置
const featureFlags = {
    newDashboard: true,
    darkMode: false,
    newCheckout: window.location.search.includes('?beta=true'),
};
```

## 三、注意事项与常见陷阱

1. **开关清理**：功能稳定后及时清理开关代码，避免技术债
2. **开关数量**：控制活跃开关数量，过多增加复杂度
3. **默认值**：开关默认关闭，明确变更流程
4. **性能影响**：开关查询应高效，避免阻塞主流程
5. **配置管理**：开关配置纳入版本控制
6. **命名规范**：使用有意义的开关名称
7. **审计日志**：记录开关变更，便于追溯
