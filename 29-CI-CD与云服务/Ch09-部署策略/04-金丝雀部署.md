# 金丝雀部署

## 一、概念说明

金丝雀部署（Canary Deployment）将一小部分流量导向新版本，逐步增加比例。通过监控新版本表现，发现问题可立即回滚。

| 阶段 | 流量比例 | 监控重点 |
|------|----------|----------|
| 初始 | 5-10% | 基础指标、错误率 |
| 扩大 | 25-50% | 性能指标、业务指标 |
| 全量 | 100% | 全面监控 |

## 二、具体用法

### Kubernetes金丝雀

```yaml
# 稳定版本 Deployment
apiVersion: apps/v1
kind: Deployment
metadata:
  name: web-app-stable
spec:
  replicas: 9
  selector:
    matchLabels:
      app: web-app
      track: stable
  template:
    metadata:
      labels:
        app: web-app
        track: stable
    spec:
      containers:
      - name: web
        image: myregistry/web-app:1.0
        ports:
        - containerPort: 8080

---
# 金丝雀版本 Deployment
apiVersion: apps/v1
kind: Deployment
metadata:
  name: web-app-canary
spec:
  replicas: 1  # 10%流量
  selector:
    matchLabels:
      app: web-app
      track: canary
  template:
    metadata:
      labels:
        app: web-app
        track: canary
    spec:
      containers:
      - name: web
        image: myregistry/web-app:2.0
        ports:
        - containerPort: 8080

---
# Service - 同时匹配stable和canary
apiVersion: v1
kind: Service
metadata:
  name: web-app-svc
spec:
  selector:
    app: web-app  # 匹配两种Pod
  ports:
  - port: 80
    targetPort: 8080
```

### Istio流量控制

```yaml
apiVersion: networking.istio.io/v1beta1
kind: VirtualService
metadata:
  name: web-app-vs
spec:
  hosts:
  - web-app-svc
  http:
  - match:
    - headers:
        x-canary:
          exact: "true"
    route:
    - destination:
        host: web-app-svc
        subset: canary
  - route:
    - destination:
        host: web-app-svc
        subset: stable
      weight: 95
    - destination:
        host: web-app-svc
        subset: canary
      weight: 5

---
apiVersion: networking.istio.io/v1beta1
kind: DestinationRule
metadata:
  name: web-app-dr
spec:
  host: web-app-svc
  subsets:
  - name: stable
    labels:
      track: stable
  - name: canary
    labels:
      track: canary
```

### Argo Rollouts金丝雀

```yaml
apiVersion: argoproj.io/v1alpha1
kind: Rollout
metadata:
  name: web-app
spec:
  replicas: 10
  selector:
    matchLabels:
      app: web-app
  template:
    metadata:
      labels:
        app: web-app
    spec:
      containers:
      - name: web
        image: myregistry/web-app:2.0
        ports:
        - containerPort: 8080
  strategy:
    canary:
      canaryService: web-app-canary
      stableService: web-app-stable
      trafficRouting:
        istio:
          virtualService:
            name: web-app-vs
            routes:
            - primary
      steps:
      - setWeight: 5
      - pause: {duration: 5m}
      - analysis:
          templates:
          - templateName: success-rate
          args:
          - name: service-name
            value: web-app-canary
      - setWeight: 25
      - pause: {duration: 5m}
      - setWeight: 50
      - pause: {duration: 5m}
      - setWeight: 100
```

### 自动化金丝雀脚本

```bash
#!/bin/bash
# canary-deploy.sh

TOTAL_REPLICAS=10
CANARY_WEIGHTS=(1 2 5 10)  # 10%, 20%, 50%, 100%

for weight in "${CANARY_WEIGHTS[@]}"; do
    echo "设置金丝雀流量比例: ${weight}/${TOTAL_REPLICAS}"

    kubectl scale deployment/web-app-canary --replicas=$weight
    kubectl scale deployment/web-app-stable --replicas=$((TOTAL_REPLICAS - weight))

    echo "等待5分钟监控..."
    sleep 300

    # 检查错误率
    ERROR_RATE=$(curl -s "http://prometheus:9090/api/v1/query?query=sum(rate(http_requests_total{status=~\"5..\",version=\"canary\"}[5m]))/sum(rate(http_requests_total{version=\"canary\"}[5m]))" | jq '.data.result[0].value[1]')

    if (( $(echo "$ERROR_RATE > 0.05" | bc -l) )); then
        echo "错误率过高 (${ERROR_RATE})，回滚金丝雀！"
        kubectl scale deployment/web-app-canary --replicas=0
        kubectl scale deployment/web-app-stable --replicas=$TOTAL_REPLICAS
        exit 1
    fi

    echo "金丝雀流量 ${weight}/${TOTAL_REPLICAS} 验证通过"
done

echo "金丝雀部署完成，所有流量切换到新版本"
```

## 三、注意事项与常见陷阱

1. **监控指标**：错误率、延迟、业务指标都要监控
2. **流量比例**：从最小比例开始，逐步放大
3. **回滚机制**：发现问题立即回滚，不要犹豫
4. **用户体验**：注意金丝雀用户是否有异常反馈
5. **缓存影响**：新旧版本的缓存可能冲突
6. **数据库兼容**：金丝雀和稳定版本共用数据库
7. **自动化程度**：可以自动化判断是否继续放量
