# 数据库迁移策略

## 一、概念说明

数据库迁移（Database Migration）是在应用部署过程中同步更新数据库schema的过程。需要保证数据不丢失、应用兼容性和可回滚。

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| 直接迁移 | 一次性更新schema | 小型变更 |
| 向后兼容迁移 | 新旧schema共存 | 零停机部署 |
| 蓝绿数据库 | 双库切换 | 大型变更 |

## 二、具体用法

### Flyway迁移

```bash
# 安装Flyway
brew install flyway

# 配置
# flyway.conf
flyway.url=jdbc:mysql://localhost:3306/mydb
flyway.user=root
flyway.password=password
flyway.schemas=public
```

```sql
-- V1__create_users_table.sql
CREATE TABLE users (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) NOT NULL UNIQUE,
    email VARCHAR(100) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- V2__add_phone_column.sql
ALTER TABLE users ADD COLUMN phone VARCHAR(20);

-- V3__create_orders_table.sql
CREATE TABLE orders (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    amount DECIMAL(10,2) NOT NULL,
    status VARCHAR(20) DEFAULT 'pending',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);

-- V4__add_index_on_orders_status.sql
CREATE INDEX idx_orders_status ON orders(status);
```

```bash
# 执行迁移
flyway migrate

# 查看迁移状态
flyway info

# 回滚迁移（需要undo支持）
flyway undo

# 修复迁移
flyway repair
```

### 向后兼容迁移

```yaml
# 零停机数据库变更策略
阶段1_添加列:
  操作: ALTER TABLE users ADD COLUMN full_name VARCHAR(100);
  应用: 同时写入旧字段(first_name, last_name)和新字段(full_name)
  回滚: 删除新列

阶段2_数据回填:
  操作: UPDATE users SET full_name = CONCAT(first_name, ' ', last_name);
  应用: 新旧字段同时使用
  回滚: 回滚应用代码

阶段3_切换读取:
  操作: 无
  应用: 改为只读新字段
  回滚: 恢复读取旧字段

阶段4_清理旧字段:
  操作: ALTER TABLE users DROP COLUMN first_name, DROP COLUMN last_name;
  应用: 移除旧字段相关代码
  回滚: 需要从备份恢复
```

### Python Alembic迁移

```python
# alembic/env.py
from alembic import context
from sqlalchemy import engine_from_config

config = context.config
target_metadata = Base.metadata

def run_migrations_online():
    connectable = engine_from_config(
        config.get_section(config.config_ini_section),
        prefix="sqlalchemy.",
    )
    with connectable.connect() as connection:
        context.configure(connection=connection, target_metadata=target_metadata)
        with context.begin_transaction():
            context.run_migrations()
```

```python
# migrations/versions/001_create_users.py
from alembic import op
import sqlalchemy as sa

def upgrade():
    op.create_table('users',
        sa.Column('id', sa.BigInteger, primary_key=True, autoincrement=True),
        sa.Column('username', sa.String(50), nullable=False, unique=True),
        sa.Column('email', sa.String(100), nullable=False),
        sa.Column('created_at', sa.TIMESTAMP, server_default=sa.func.now()),
    )

def downgrade():
    op.drop_table('users')
```

### CI/CD集成

```yaml
# GitHub Actions数据库迁移
db-migrate:
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v4

    - name: Run Flyway migrations
      run: |
        flyway -url=jdbc:mysql://${{ secrets.DB_HOST }}/mydb \
               -user=${{ secrets.DB_USER }} \
               -password=${{ secrets.DB_PASSWORD }} \
               migrate

    - name: Verify migration
      run: flyway info

deploy-app:
  needs: db-migrate
  runs-on: ubuntu-latest
  steps:
    - name: Deploy application
      run: |
        kubectl set image deployment/web-app web=${{ env.IMAGE_TAG }}
        kubectl rollout status deployment/web-app
```

### Docker Compose本地开发

```yaml
# docker-compose.yml
version: '3.8'
services:
  db:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: password
      MYSQL_DATABASE: mydb
    ports:
      - "3306:3306"

  flyway:
    image: flyway/flyway:9
    command: migrate
    volumes:
      - ./sql:/flyway/sql
    environment:
      FLYWAY_URL: jdbc:mysql://db:3306/mydb
      FLYWAY_USER: root
      FLYWAY_PASSWORD: password
    depends_on:
      - db
```

## 三、注意事项与常见陷阱

1. **向后兼容**：所有迁移必须向后兼容，保证新旧应用都能工作
2. **大表变更**：大表加列/加索引要考虑性能影响，使用pt-osc等工具
3. **数据回填**：大批量回填要分批执行，避免锁表
4. **迁移顺序**：数据库迁移先于应用部署
5. **回滚测试**：迁移脚本要测试回滚
6. **数据备份**：执行迁移前备份数据
7. **索引策略**：大表索引在低峰期创建
