# 云监控与日志

## 一、概念说明

阿里云提供云监控（CloudMonitor）和日志服务（SLS）两大可观测性工具。云监控负责指标和告警，日志服务负责日志收集和分析。

| 服务 | 功能 | 数据类型 |
|------|------|----------|
| 云监控 | 指标采集、告警、Dashboard | 时序指标 |
| 日志服务 | 日志采集、查询、分析 | 日志数据 |

## 二、具体用法

### 云监控告警

```bash
# 创建自定义指标
aliyun cms PutCustomMetric \
    --MetricName RequestLatency \
    --GroupId my-app \
    --Time 1705305600 \
    --Type "Average" \
    --Dimensions '{"service":"web"}' \
    --Period 60 \
    --Values '[{"Average": 120}]'

# 创建告警规则
aliyun cms PutResourceMetricRule \
    --RuleName HighCPU \
    --Namespace acs_ecs_dashboard \
    --MetricName CPUUtilization \
    --Resources '[{"instanceId":"i-bp1xxxxxxxx"}]' \
    --Period 300 \
    --Statistics Average \
    --ComparisonOperator GreaterThanThreshold \
    --Threshold 80 \
    --EvaluationCount 3 \
    --ContactGroups '["运维组"]'

# 创建事件告警
aliyun cms PutEventRule \
    --RuleName InstanceFailure \
    --EventType SystemEvent \
    --EventPattern '[{"product":["ECS"],"name":["Instance:InstanceFailure.SystemFailure"]}]' \
    --ContactGroups '["运维组"]'
```

### 日志服务

```bash
# 创建项目和日志库
aliyun sls CreateProject \
    --projectName my-log-project \
    --description "生产环境日志" \
    --region cn-hangzhou

aliyun sls CreateLogStore \
    --projectName my-log-project \
    --logStoreName app-logs \
    --ttl 30 \
    --shardCount 2

# 配置日志采集
aliyun sls CreateConfig \
    --projectName my-log-project \
    --configName app-config \
    --configDetail '{
        "inputs": [{
            "type": "file",
            "detail": {
                "logPath": "/var/log/app",
                "filePattern": "*.log",
                "logType": "json_regex",
                "regex": "(?P<time>\\d{4}-\\d{2}-\\d{2} \\d{2}:\\d{2}:\\d{2}) (?P<level>\\w+) (?P<message>.*)"
            }
        }],
        "output": {
            "projectName": "my-log-project",
            "logstoreName": "app-logs"
        }
    }'
```

### 日志查询

```bash
# 基本查询
aliyun sls GetLogs \
    --projectName my-log-project \
    --logStoreName app-logs \
    --from 1705305600 \
    --to 1705392000 \
    --query 'level:ERROR'

# 聚合分析
aliyun sls GetLogs \
    --projectName my-log-project \
    --logStoreName app-logs \
    --from 1705305600 \
    --to 1705392000 \
    --query '* | SELECT level, COUNT(*) as cnt GROUP BY level'

# 按时间分桶统计
aliyun sls GetLogs \
    --projectName my-log-project \
    --logStoreName app-logs \
    --from 1705305600 \
    --to 1705392000 \
    --query '* | SELECT date_format(__time__, \'%H:00\') as hour, COUNT(*) GROUP BY hour ORDER BY hour'
```

### Python集成

```python
import json
import time
from aliyun.log import LogClient, PutLogsRequest, LogItem

client = LogClient('cn-hangzhou.log.aliyuncs.com', accessKeyId, accessKey)

# 写入日志
contents = [('level', 'INFO'), ('message', '用户登录成功'), ('user_id', '12345')]
log_item = LogItem(contents=contents)
request = PutLogsRequest('my-log-project', 'app-logs', '', [log_item])
client.put_logs(request)

# 查询日志
response = client.get_logs('my-log-project', 'app-logs', int(time.time()) - 3600, int(time.time()), 'level:ERROR')
for log in response.get_logs():
    print(log.get_contents())
```

## 三、注意事项与常见陷阱

1. **日志存储**：合理设置TTL避免存储费用过高
2. **索引配置**：为日志库创建索引以支持全文检索
3. **采集性能**：配置合理的采集频率避免性能影响
4. **告警静默**：设置告警静默期避免告警风暴
5. **Dashboard**：创建关键指标的仪表板实时监控
6. **告警联系人**：配置多种通知方式确保及时响应
7. **日志脱敏**：敏感数据在采集前脱敏处理
