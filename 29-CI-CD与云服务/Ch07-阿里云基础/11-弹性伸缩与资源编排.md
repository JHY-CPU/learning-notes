# 弹性伸缩与资源编排

## 一、概念说明

阿里云弹性伸缩（ESS）根据负载自动调整ECS实例数量。资源编排（ROS）是IaC工具，用模板定义和管理云资源。

| 服务 | 功能 | 对应AWS |
|------|------|---------|
| ESS弹性伸缩 | 自动扩缩容ECS | Auto Scaling |
| ROS资源编排 | 基础设施即代码 | CloudFormation |

## 二、具体用法

### 弹性伸缩配置

```bash
# 创建伸缩组
aliyun ess CreateScalingGroup \
    --ScalingGroupName web-auto-scaling \
    --MinSize 2 \
    --MaxSize 10 \
    --DefaultCooldown 300 \
    --RemovalPolicy.1 OldestInstance \
    --VSwitchIds.1 vsw-bp1xxxxxxxx \
    --VSwitchIds.2 vsw-bp1yyyyyyyy \
    --LoadBalancerIds.1 lb-bp1xxxxxxxx

# 创建伸缩配置
aliyun ess CreateScalingConfiguration \
    --ScalingGroupId sg-bp1xxxxxxxx \
    --ImageId aliyun_3_x64_20G_alibase_20231221.vhd \
    --InstanceType ecs.g7.xlarge \
    --SecurityGroupId sg-bp1xxxxxxxx \
    --KeyPairName my-keypair \
    --SystemDisk.Category cloud_essd \
    --SystemDisk.Size 40 \
    --InternetChargeType PayByTraffic \
    --ScalingConfigurationName web-config

# 启用伸缩配置
aliyun ess EnableScalingConfiguration \
    --ScalingGroupId sg-bp1xxxxxxxx \
    --ScalingConfigurationId asc-bp1xxxxxxxx
```

### 伸缩规则

```bash
# 手动扩展
aliyun ess CreateScalingRule \
    --ScalingGroupId sg-bp1xxxxxxxx \
    --ScalingRuleName scale-out-2 \
    --AdjustmentType QuantityChangeInCapacity \
    --AdjustmentValue 2

# 基于报警的伸缩
aliyun ess CreateAlarm \
    --ScalingGroupId sg-bp1xxxxxxxx \
    --Name high-cpu-alarm \
    --MetricType SYSTEM \
    --MetricName CpuUtilization \
    --Period 120 \
    --Statistics Average \
    --ComparisonOperator GreaterThanThreshold \
    --Threshold 75 \
    --EvaluationCount 3 \
    --AlarmAction.1 ScalingRuleAri ari-bp1xxxxxxxx

# 定时伸缩
aliyun ess CreateScheduledTask \
    --ScheduledTaskName morning-scale \
    --ScheduledAction "ari-bp1xxxxxxxx" \
    --RecurrenceEndTime "2024-12-31T08:00Z" \
    --LaunchTime "2024-01-01T08:00Z" \
    --RecurrenceType Weekly \
    --RecurrenceValue "1,2,3,4,5" \
    --LaunchExpirationTime 300
```

### ROS资源编排模板

```yaml
ROSTemplateFormatVersion: '2015-09-01'
Description: Web服务器堆栈模板

Parameters:
  Environment:
    Type: String
    Default: dev
    AllowedValues: [dev, staging, prod]

  InstanceType:
    Type: String
    Default: ecs.g7.xlarge

Resources:
  VPC:
    Type: ALIYUN::ECS::VPC
    Properties:
      CidrBlock: 10.0.0.0/16
      VpcName: !Sub ${Environment}-vpc

  VSwitch:
    Type: ALIYUN::ECS::VSwitch
    Properties:
      VpcId: !Ref VPC
      CidrBlock: 10.0.1.0/24
      ZoneId: cn-hangzhou-b

  SecurityGroup:
    Type: ALIYUN::ECS::SecurityGroup
    Properties:
      VpcId: !Ref VPC
      SecurityGroupName: !Sub ${Environment}-sg
      SecurityGroupIngress:
        - IpProtocol: tcp
          PortRange: 80/80
          SourceCidrIp: 0.0.0.0/0

  WebServer:
    Type: ALIYUN::ECS::Instance
    Properties:
      InstanceType: !Ref InstanceType
      ImageId: aliyun_3_x64_20G_alibase_20231221.vhd
      VpcId: !Ref VPC
      VSwitchId: !Ref VSwitch
      SecurityGroupId: !Ref SecurityGroup
      SystemDiskCategory: cloud_essd

Outputs:
  ServerIP:
    Value: !GetAtt WebServer.PublicIp
```

```bash
# 创建堆栈
aliyun ros CreateStack \
    --RegionId cn-hangzhou \
    --StackName web-stack \
    --TemplateBody file://template.yaml \
    --Parameters.1.ParameterKey Environment \
    --Parameters.1.ParameterValue prod
```

## 三、注意事项与常见陷阱

1. **最小实例数**：伸缩组设置最小实例数保证可用性
2. **冷却时间**：设置合理的冷却时间避免频繁伸缩
3. **报警阈值**：CPU阈值70-80%开始扩容，20-30%开始缩容
4. **实例保护**：关键实例开启保护防止被缩容移除
5. **伸缩配置**：确保伸缩配置的镜像和初始化脚本正确
6. **ROS回滚**：ROS支持创建失败自动回滚
7. **版本管理**：模板文件纳入版本控制
