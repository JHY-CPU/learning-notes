# 物理引擎架构

## 核心概念

物理引擎的架构设计决定了性能上限和可维护性。核心是将Broad Phase、Narrow Phase、约束求解等组件组织成高效的流水线。现代物理引擎（Bullet、PhysX、Box2D、Chaos）都遵循相似的架构模式。

### 物理世界步进（Physics Step）

固定时间步长的更新循环是物理引擎的基石：

```
物理步进完整流程:
1. Pre-Step: 力的累加（重力、外力）
2. Broad Phase → 生成潜在碰撞对列表
3. Narrow Phase → 精确碰撞检测，生成碰撞信息
4. 岛屿检测与分组
5. 约束求解 → 求解碰撞响应和关节约束（迭代）
6. 积分 → 更新所有刚体的速度和位置
7. Post-Step → 位置修正、事件回调
8. 睡眠状态更新
9. 插值状态存储（用于渲染）
```

### 岛屿检测与休眠（Islands & Sleeping）

**岛屿**：通过碰撞或约束连接的物体组。同一岛屿内的物体需要一起求解，不同岛屿间独立求解可并行化。

**休眠机制**：长时间静止的物体/岛屿标记为"休眠"，跳过物理更新。当受到足够大的力或碰撞时唤醒。这大幅减少静态场景的计算量。

### Broad/Narrow Phase流水线

```
Broad Phase:
  输入: 所有刚体的AABB列表
  方法: BVH / 空间哈希 / 扫描修剪
  输出: 可能碰撞的物体对列表

Narrow Phase:
  输入: Broad Phase输出的物体对
  方法: GJK / SAT / 特定形状相交测试
  输出: 碰撞信息（接触点、法线、穿透深度）
```

## 完整物理引擎实现

```cpp
#include <vector>
#include <map>
#include <set>
#include <algorithm>

// 假设已有RigidBody, CollisionInfo, Vec3等定义

// ========== 物理世界主类 ==========
class PhysicsWorld {
    // 刚体列表
    std::vector<RigidBody> bodies;

    // 碰撞接触
    std::vector<CollisionInfo> contacts;

    // 约束
    std::vector<Constraint*> constraints;

    // 时间管理
    float fixedDeltaTime = 1.0f / 60.0f;
    float accumulator = 0;
    int stepCount = 0;

    // 岛屿系统
    std::vector<std::vector<int>> islands;
    std::vector<bool> islandSleeping;

    // 接触缓存（跨帧保持接触信息）
    struct ContactCache {
        int bodyA, bodyB;
        Vec3 normal;
        float distance;
        float normalImpulse;    // 累积冲量
        float tangentImpulse;
        int age;                // 接触年龄（帧数）
    };
    std::vector<ContactCache> contactCache;

    // 休眠管理
    float sleepVelocityThreshold = 0.01f;
    float sleepAngularThreshold = 0.01f;
    float sleepTimeThreshold = 1.0f;

public:
    // ========== 公共接口 ==========

    int addBody(const RigidBody& body) {
        bodies.push_back(body);
        return (int)bodies.size() - 1;
    }

    void addConstraint(Constraint* c) {
        constraints.push_back(c);
    }

    // ========== 主更新循环 ==========

    void update(float frameDt) {
        // 固定时间步累加器
        accumulator += frameDt;

        // 防止帧率极低时的"死亡螺旋"
        float maxStep = fixedDeltaTime * 4;
        if (accumulator > maxStep) accumulator = maxStep;

        while (accumulator >= fixedDeltaTime) {
            step(fixedDeltaTime);
            accumulator -= fixedDeltaTime;
            stepCount++;
        }

        // 计算插值因子（用于渲染）
        float alpha = accumulator / fixedDeltaTime;
        // 使用alpha在当前物理状态和上一帧状态间插值
    }

private:
    // ========== 单步物理更新 ==========

    void step(float dt) {
        // 1. 施加重力等外力
        applyForces(dt);

        // 2. Broad Phase
        auto potentialPairs = broadPhase();

        // 3. Narrow Phase
        contacts.clear();
        for (auto& [a, b] : potentialPairs) {
            CollisionInfo info;
            if (narrowPhase(bodies[a], bodies[b], info)) {
                contacts.push_back(info);
            }
        }

        // 4. 接触缓存合并
        mergeContactCache();

        // 5. 岛屿检测与分组
        buildIslands();

        // 6. 对每个岛屿独立求解
        for (size_t i = 0; i < islands.size(); i++) {
            if (islandSleeping[i]) continue;

            // 6a. 积分（更新速度和位置）
            for (int idx : islands[i])
                bodies[idx].integrate(dt);

            // 6b. 约束求解（碰撞 + 关节）
            solveConstraints(islands[i], dt);
        }

        // 7. 位置修正
        applyPositionCorrections();

        // 8. 更新休眠状态
        updateSleepStates();

        // 9. 更新AABB
        updateAABBs();
    }

    // ========== 力的施加 ==========

    void applyForces(float dt) {
        Vec3 gravity = {0, -9.81f, 0};

        for (auto& body : bodies) {
            if (body.isStatic() || !body.isAwake) continue;

            // 重力
            body.applyForce(gravity * body.mass);
        }
    }

    // ========== Broad Phase ==========

    std::set<std::pair<int,int>> broadPhase() {
        // 使用空间哈希或BVH
        SpatialHashBroadPhase spatialHash(2.0f);

        for (int i = 0; i < (int)bodies.size(); i++) {
            if (!bodies[i].isStatic() || bodies[i].isAwake) {
                // 更新AABB
                updateBodyAABB(bodies[i]);
                spatialHash.insert(i, bodies[i].aabbMin, bodies[i].aabbMax);
            }
        }

        return spatialHash.getPotentialPairs();
    }

    // ========== Narrow Phase ==========

    bool narrowPhase(RigidBody& a, RigidBody& b, CollisionInfo& info) {
        // 根据形状类型选择碰撞检测算法
        // 简化为Sphere-Sphere检测
        Vec3 diff = a.position - b.position;
        float distSq = diff.lengthSq();
        float radiiSum = a.radius + b.radius; // 假设有radius属性

        if (distSq <= radiiSum * radiiSum) {
            float dist = sqrt(distSq);
            info.bodyA = &a;
            info.bodyB = &b;
            info.normal = dist > 0.0001f ? diff * (1.0f/dist) : Vec3{0,1,0};
            info.penetration = radiiSum - dist;
            info.contactPoint = (a.position + b.position) * 0.5f
                              - info.normal * (a.radius - info.penetration * 0.5f);
            return true;
        }
        return false;
    }

    // ========== 接触缓存 ==========

    void mergeContactCache() {
        // 保留上一帧的接触信息，合并到当前帧
        // 用于累积冲量暖启动
        for (auto& cached : contactCache) {
            cached.age++;
        }

        // 移除旧接触
        contactCache.erase(
            std::remove_if(contactCache.begin(), contactCache.end(),
                [](const ContactCache& c) { return c.age > 3; }),
            contactCache.end());
    }

    // ========== 岛屿检测 ==========

    void buildIslands() {
        int n = (int)bodies.size();

        // 使用并查集分组
        std::vector<int> parent(n);
        for (int i = 0; i < n; i++) parent[i] = i;

        auto find = [&](int x) {
            while (parent[x] != x) { parent[x] = parent[parent[x]]; x = parent[x]; }
            return x;
        };
        auto unite = [&](int a, int b) { parent[find(a)] = find(b); };

        // 通过碰撞连接物体
        for (auto& c : contacts) {
            int a = findBodyIndex(c.bodyA);
            int b = findBodyIndex(c.bodyB);
            if (a >= 0 && b >= 0) unite(a, b);
        }

        // 通过约束连接物体
        for (auto* constraint : constraints) {
            unite(constraint->bodyA, constraint->bodyB);
        }

        // 按根节点分组
        std::map<int, std::vector<int>> groups;
        for (int i = 0; i < n; i++)
            groups[find(i)].push_back(i);

        islands.clear();
        islandSleeping.clear();
        for (auto& [root, members] : groups) {
            islands.push_back(members);
            islandSleeping.push_back(false);
        }
    }

    // ========== 约束求解 ==========

    void solveConstraints(const std::vector<int>& islandBodies, float dt) {
        // 顺序冲量迭代求解
        const int ITERATIONS = 10;

        for (int iter = 0; iter < ITERATIONS; iter++) {
            // 求解关节约束
            for (auto* constraint : constraints) {
                if (std::find(islandBodies.begin(), islandBodies.end(),
                    constraint->bodyA) != islandBodies.end()) {
                    constraint->solve(dt);
                }
            }

            // 求解碰撞约束
            for (auto& contact : contacts) {
                int aIdx = findBodyIndex(contact.bodyA);
                if (std::find(islandBodies.begin(), islandBodies.end(),
                    aIdx) != islandBodies.end()) {
                    resolveCollision(contact);
                }
            }
        }
    }

    // ========== 位置修正 ==========

    void applyPositionCorrections() {
        for (auto& contact : contacts) {
            positionalCorrection(contact);
        }
    }

    // ========== 睡眠管理 ==========

    void updateSleepStates() {
        for (size_t i = 0; i < islands.size(); i++) {
            if (islandSleeping[i]) {
                // 检查是否需要唤醒
                for (int idx : islands[i]) {
                    // 外部力或碰撞可能唤醒物体
                    if (bodies[idx].velocity.lengthSq() > sleepVelocityThreshold * sleepVelocityThreshold) {
                        islandSleeping[i] = false;
                        bodies[idx].isAwake = true;
                        break;
                    }
                }
            }
        }
    }

    // ========== 渲染插值 ==========

    struct InterpolatedState {
        Vec3 position;
        Quaternion orientation;
    };

    // 在固定步长之间插值，实现平滑渲染
    InterpolatedState getInterpolatedState(int bodyIndex, float alpha) {
        // alpha = 当前帧在两个物理步之间的位置 [0, 1]
        // 需要存储上一帧的状态
        InterpolatedState result;
        // result.position = prevPos * (1-alpha) + curPos * alpha;
        return result;
    }

    // 辅助函数
    int findBodyIndex(RigidBody* body) {
        for (int i = 0; i < (int)bodies.size(); i++)
            if (&bodies[i] == body) return i;
        return -1;
    }
    void updateBodyAABB(RigidBody& body) {}
    void updateAABBs() {}
};
```

## 接触缓存与暖启动

```cpp
// 接触缓存：跨帧维护接触信息，利用上一帧的冲量作为暖启动
class ContactCacheManager {
    struct CachedContact {
        uint64_t pairKey;       // 碰撞对的唯一标识
        Vec3 normal;
        float normalImpulse;    // 累积法线冲量
        float tangentImpulse;   // 累积切向冲量
        int age;                // 接触持续帧数
    };

    std::unordered_map<uint64_t, CachedContact> cache;

    uint64_t makeKey(int a, int b) {
        if (a > b) std::swap(a, b);
        return ((uint64_t)a << 32) | (uint64_t)b;
    }

public:
    void update(const std::vector<CollisionInfo>& newContacts) {
        std::set<uint64_t> activeKeys;

        for (auto& contact : newContacts) {
            int a = contact.bodyA->id;
            int b = contact.bodyB->id;
            uint64_t key = makeKey(a, b);
            activeKeys.insert(key);

            if (cache.find(key) != cache.end()) {
                // 现有接触：保持累积冲量
                cache[key].age++;
                cache[key].normal = contact.normal;
            } else {
                // 新接触
                cache[key] = {key, contact.normal, 0, 0, 0};
            }
        }

        // 移除不再存在的接触
        for (auto it = cache.begin(); it != cache.end();) {
            if (activeKeys.find(it->first) == activeKeys.end())
                it = cache.erase(it);
            else
                ++it;
        }
    }

    // 获取暖启动冲量
    float getWarmStartImpulse(uint64_t pairKey) {
        auto it = cache.find(pairKey);
        return it != cache.end() ? it->second.normalImpulse : 0;
    }
};
```

## 性能优化策略

```cpp
// ========== 性能优化总结 ==========

/*
1. Broad Phase选型:
   - 静态场景: BVH (SAH构建)
   - 大量动态物体: 空间哈希
   - 混合场景: BVH + 增量更新

2. Narrow Phase优化:
   - AABB预检查 (在GJK前做AABB相交测试)
   - 缓存接触信息 (暖启动)
   - 使用SIMD加速 (SSE/AVX)

3. 岛屿并行化:
   - 不同岛屿可以在不同线程求解
   - 使用任务图(Task Graph)调度

4. 接触缓存:
   - 跨帧保持冲量累积值
   - 减少迭代收敛所需的次数

5. 休眠机制:
   - 静止物体不参与物理计算
   - 显著减少大规模场景的开销

6. 时间步管理:
   - 固定时间步长保证稳定性
   - 渲染插值消除抖动

7. 内存布局:
   - SoA (Structure of Arrays) 布局对SIMD友好
   - 紧凑存储减少缓存未命中
*/
```

## 游戏应用场景

1. **大规模场景**：岛屿分割让远处的物理计算可并行化
2. **性能优化**：休眠机制大幅减少静态场景的计算量
3. **物理步长解耦**：固定步长物理+插值渲染，保证稳定性和流畅性
4. **多线程**：不同岛屿可在不同线程上并行求解

## 常见陷阱

- **变步长物理**：可变dt导致物理不稳定，务必使用固定步长+累加器
- **渲染抖动**：不使用插值会导致帧率波动时物理画面抖动
- **岛屿合并**：休眠岛屿被唤醒时需要重新激活整个岛屿的所有物体
- **Broad Phase选型**：静态场景BVH更优，大量动态物体空间哈希更好
- **接触缓存泄漏**：长期不清理缓存会导致内存增长
- **多线程竞争**：同一岛屿的物体不能在不同线程同时求解
- **GC压力**：物理引擎中应避免堆分配，使用预分配内存池
