# 触发器与物理事件

## 核心概念

物理事件系统是物理引擎与游戏逻辑之间的桥梁，让游戏代码能够响应碰撞和区域进入/退出。

### Trigger（触发器）vs Collision（碰撞）

**碰撞体（Collider）**：参与物理响应，物体会被反弹或阻挡。
**触发器（Trigger）**：仅检测重叠，不产生物理响应，用于区域检测。

| 属性 | Collider | Trigger |
|------|----------|---------|
| 物理响应 | 有（反弹/阻挡） | 无（物体穿过） |
| 性能 | 需要碰撞求解 | 仅需重叠检测 |
| 用途 | 墙壁、地面、可推动物体 | 捡取区域、触发区域、传感器 |

### 物理回调事件

- **OnCollisionEnter**：首次碰撞时调用一次
- **OnCollisionStay**：持续碰撞期间每帧调用
- **OnCollisionExit**：碰撞结束时调用一次
- **OnTriggerEnter/Stay/Exit**：触发器版本，物体进入/停留/离开触发区域

### 状态跟踪

通过维护上一帧的碰撞/触发状态集合，与当前帧对比，确定Enter/Stay/Exit事件。

## 代码示例

```cpp
// 碰撞/触发事件类型
enum class EventType { Enter, Stay, Exit };

struct PhysicsEvent {
    EventType type;
    RigidBody* bodyA;
    RigidBody* bodyB;
    CollisionInfo collisionInfo; // 仅碰撞事件有效
};

// 事件分发器
class PhysicsEventDispatcher {
    // 碰撞状态跟踪：对(bodyA, bodyB) → CollisionInfo
    std::map<std::pair<uint64_t, uint64_t>, CollisionInfo> previousCollisions;
    std::map<std::pair<uint64_t, uint64_t>, CollisionInfo> currentCollisions;

    // 触发器状态跟踪
    std::set<std::pair<uint64_t, uint64_t>> previousTriggers;
    std::set<std::pair<uint64_t, uint64_t>> currentTriggers;

public:
    void processCollisions(const std::vector<CollisionInfo>& newContacts) {
        currentCollisions.clear();

        for (auto& c : newContacts) {
            auto key = makeKey(c.bodyA, c.bodyB);
            currentCollisions[key] = c;

            if (!previousCollisions.count(key)) {
                // OnCollisionEnter
                fireEvent({EventType::Enter, c.bodyA, c.bodyB, c});
            } else {
                // OnCollisionStay
                fireEvent({EventType::Stay, c.bodyA, c.bodyB, c});
            }
        }

        // 检查退出的碰撞
        for (auto& [key, c] : previousCollisions) {
            if (!currentCollisions.count(key)) {
                fireEvent({EventType::Exit, c.bodyA, c.bodyB, c});
            }
        }

        previousCollisions = currentCollisions;
    }

    void processTriggers(const std::set<std::pair<uint64_t,uint64_t>>& overlaps) {
        currentTriggers = overlaps;

        for (auto& key : overlaps) {
            if (!previousTriggers.count(key))
                fireTriggerEvent({EventType::Enter, key});
            else
                fireTriggerEvent({EventType::Stay, key});
        }

        for (auto& key : previousTriggers) {
            if (!currentTriggers.count(key))
                fireTriggerEvent({EventType::Exit, key});
        }

        previousTriggers = currentTriggers;
    }

private:
    std::pair<uint64_t,uint64_t> makeKey(RigidBody* a, RigidBody* b) {
        uint64_t idA = (uint64_t)a, idB = (uint64_t)b;
        return {std::min(idA, idB), std::max(idA, idB)};
    }

    void fireEvent(PhysicsEvent evt) {
        // 调用注册的回调函数
        if (evt.bodyA->onCollision) evt.bodyA->onCollision(evt);
        if (evt.bodyB->onCollision) evt.bodyB->onCollision(evt);
    }
};

// 带触发器的游戏对象
class GameObject {
public:
    RigidBody* physicsBody;
    bool isTrigger = false;

    std::function<void(PhysicsEvent)> onCollision;
    std::function<void(RigidBody*, EventType)> onTrigger;

    virtual void onCollisionEnter(CollisionInfo& info) {
        // 处理碰撞开始
    }
    virtual void onCollisionStay(CollisionInfo& info) {
        // 处理持续碰撞
    }
    virtual void onCollisionExit(RigidBody* other) {
        // 处理碰撞结束
    }

    virtual void onTriggerEnter(RigidBody* other) {
        // 物体进入触发区域
    }
};

// 使用示例：金币捡取
class Coin : public GameObject {
    Coin() { isTrigger = true; }

    void onTriggerEnter(RigidBody* other) override {
        if (other->gameObject->tag == "Player") {
            // 玩家进入金币区域，增加金币计数
            GameManager::addScore(10);
            Destroy(this); // 销毁金币
        }
    }
};

// 使用示例：伤害区域
class DamageZone : public GameObject {
    float damagePerSecond = 10.0f;

    void onTriggerStay(RigidBody* other) override {
        if (auto* player = dynamic_cast<Player*>(other->gameObject)) {
            player->takeDamage(damagePerSecond * deltaTime);
        }
    }
};
```

## 游戏应用场景

1. **道具捡取**：触发器检测玩家进入 → 添加道具到背包
2. **检查点**：触发区域标记玩家进度
3. **陷阱区域**：触发器+持续伤害逻辑
4. **对话触发**：NPC附近触发对话
5. **关门事件**：检测所有敌人被消灭后关闭门

## 常见陷阱

- **Enter丢失**：物体速度太快直接穿过薄触发器，需用连续检测或加厚触发器
- **重复Enter**：一帧内物体在触发器边界反复移动可能产生多次Enter
- **事件顺序**：Enter/Exit事件的顺序在不同引擎中可能不同
- **性能**：大量触发器每帧检测开销大，用物理层过滤减少检测对
- **触发器+碰撞体混合**：同一刚体不能既是Trigger又是Collider，需分开处理
