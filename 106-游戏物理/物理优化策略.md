# 物理优化策略

## 核心概念

物理计算往往是游戏性能的主要瓶颈，需要从碰撞体简化、空间划分、休眠机制等多方面优化。

### 碰撞体简化

用简单的碰撞体近似复杂模型：
- 复杂网格 → 凸分解 → 多个凸包
- 角色 → 胶囊体（而非网格碰撞体）
- 地形 → 高度场或简化三角网格

### 物理层过滤（Collision Layer / Mask）

将物体分为不同层，只检测需要交互的层间碰撞：

```
Player层：只与 Ground、Enemy、Pickup 检测
Enemy层：只与 Ground、Player、Projectile 检测
Decoration层：不与任何层检测（纯视觉）
```

避免不必要的碰撞检测对。

### 休眠机制

长时间静止的物体标记为休眠，跳过物理更新。唤醒条件：
- 施加足够大的力/冲量
- 与活跃物体碰撞
- 关联的约束被扰动

### Fixed Timestep调优

- 过小的dt增加计算量
- 过大的dt导致不稳定和隧道效应
- 通常60Hz（16.67ms）是好的平衡点

## 代码示例

```cpp
// 物理层碰撞过滤
struct CollisionFilter {
    uint32_t layer;   // 物体所属层（位标志）
    uint32_t mask;    // 与哪些层碰撞（位标志）

    bool shouldCollideWith(const CollisionFilter& other) const {
        return (layer & other.mask) != 0 && (other.layer & mask) != 0;
    }
};

// 定义游戏层
namespace PhysicsLayers {
    constexpr uint32_t Player      = 1 << 0;
    constexpr uint32_t Enemy       = 1 << 1;
    constexpr uint32_t Ground      = 1 << 2;
    constexpr uint32_t Projectile  = 1 << 3;
    constexpr uint32_t Pickup      = 1 << 4;
    constexpr uint32_t Trigger     = 1 << 5;
    constexpr uint32_t Decoration  = 1 << 6; // 纯视觉，不参与碰撞
}

// 碰撞层配置
CollisionFilter playerFilter = {
    PhysicsLayers::Player,
    PhysicsLayers::Ground | PhysicsLayers::Enemy | PhysicsLayers::Pickup | PhysicsLayers::Trigger
};

CollisionFilter enemyFilter = {
    PhysicsLayers::Enemy,
    PhysicsLayers::Ground | PhysicsLayers::Player | PhysicsLayers::Projectile
};

CollisionFilter decoFilter = {
    PhysicsLayers::Decoration,
    0 // 不与任何层碰撞
};

// Broad Phase过滤
class OptimizedBroadPhase {
    // 在Broad Phase阶段就过滤掉不匹配的层对
    std::vector<std::pair<int,int>> getPotentialPairs(const std::vector<RigidBody>& bodies) {
        std::vector<std::pair<int,int>> result;
        // BVH遍历时检查filter
        for (int i = 0; i < bodies.size(); i++) {
            for (int j = i+1; j < bodies.size(); j++) {
                if (!bodies[i].filter.shouldCollideWith(bodies[j].filter))
                    continue;
                if (aabbOverlap(bodies[i].aabb, bodies[j].aabb))
                    result.push_back({i, j});
            }
        }
        return result;
    }
};

// Fixed Timestep with Accumulator
class PhysicsTimestep {
    float fixedDt = 1.0f / 60.0f;
    float accumulator = 0;
    const float maxAccumulator = 0.25f; // 防止死亡螺旋

public:
    void update(float frameDt) {
        accumulator += std::min(frameDt, maxAccumulator); // 限制最大累积
        while (accumulator >= fixedDt) {
            physicsStep(fixedDt);
            accumulator -= fixedDt;
        }
        // alpha用于渲染插值
        float alpha = accumulator / fixedDt;
        interpolateRender(alpha);
    }
};

// 碰撞体LOD：远处使用简化碰撞体
class CollisionLOD {
    enum DetailLevel { Full, Simplified, BoundingSphere };

    DetailLevel getLevel(Vec3 cameraPos, Vec3 objectPos, float maxDist) {
        float dist = (cameraPos - objectPos).length();
        if (dist < maxDist * 0.3f) return Full;
        if (dist < maxDist * 0.7f) return Simplified;
        return BoundingSphere;
    }

    Collider* getCollider(RigidBody* body, DetailLevel level) {
        switch (level) {
            case Full: return body->fullCollider;      // 详细网格
            case Simplified: return body->simplifiedCollider; // 凸包
            case BoundingSphere: return body->boundingSphere;  // 包围球
        }
    }
};

// 物理更新频率分级
class PhysicsLOD {
    // 近距离物体：每帧更新（60Hz）
    // 中距离物体：每2帧更新（30Hz）
    // 远距离物体：休眠或极低频率
    void updateBodies(const std::vector<RigidBody>& bodies, Vec3 cameraPos) {
        for (auto& body : bodies) {
            float dist = (cameraPos - body.position).length();
            if (dist < 20.0f) {
                body.updateFrequency = 1; // 每帧
            } else if (dist < 50.0f) {
                body.updateFrequency = 2; // 每2帧
            } else {
                body.updateFrequency = 4; // 每4帧
            }
        }
    }
};
```

## 游戏应用场景

1. **开放世界**：远距离物体用包围球检测，远处跳过物理更新
2. **FPS射击**：子弹只与特定层碰撞，忽略装饰物
3. **物理密集场景**：大量物体中休眠机制节省90%+计算
4. **多人游戏**：服务器端物理使用低频步长，客户端插值

## 常见陷阱

- **过简化的碰撞体**：导致"明明没碰到但碰撞了"的穿模问题
- **层过滤遗漏**：配置错误导致物体穿透墙壁或卡住
- **休眠唤醒震荡**：物体在休眠边缘反复唤醒/休眠，消耗CPU
- **最大累积时间**：不设上限会导致长时间卡顿后一次性模拟大量步数
