# RocketMQ 安装与配置

## 一、Docker 安装

```yaml
# docker-compose.yml
version: '3.8'
services:
  namesrv:
    image: apache/rocketmq:5.1.3
    container_name: rmqnamesrv
    ports:
      - "9876:9876"
    command: sh mqnamesrv
    environment:
      JAVA_OPT_EXT: "-Xms512m -Xmx512m"

  broker:
    image: apache/rocketmq:5.1.3
    container_name: rmqbroker
    ports:
      - "10909:10909"
      - "10911:10911"
    depends_on:
      - namesrv
    environment:
      NAMESRV_ADDR: "namesrv:9876"
    command: sh mqbroker -n namesrv:9876
    volumes:
      - ./broker.conf:/home/rocketmq/conf/broker.conf

  # Dashboard
  dashboard:
    image: apacherocketmq/rocketmq-dashboard:latest
    container_name: rmqdashboard
    ports:
      - "8080:8080"
    environment:
      - ROCKETMQ_CONFIG_NAMESRVADDR=namesrv:9876
```

```conf
# broker.conf
brokerClusterName=DefaultCluster
brokerName=broker-a
brokerId=0
namesrvAddr=namesrv:9876
defaultTopicQueueNums=4
autoCreateTopicEnable=true
autoCreateSubscriptionGroup=true
listenPort=10911
deleteWhen=04
fileReservedTime=48
diskMaxUsedSpaceRatio=75
storePathRootDir=/home/rocketmq/store
storePathCommitLog=/home/rocketmq/store/commitlog
flushDiskType=ASYNC_FLUSH
```

## 二、常用命令

```bash
# 启动 NameServer
mqnamesrv

# 启动 Broker
mqbroker -n localhost:9876

# 创建 Topic
mqadmin updateTopic -n localhost:9876 -t OrderTopic -c DefaultCluster

# 查看 Topic 列表
mqadmin topicList -n localhost:9876

# 查看 Topic 状态
mqadmin topicStatus -n localhost:9876 -t OrderTopic

# 查看消费者组
mqadmin consumerProgress -n localhost:9876

# 重置消费位点
mqadmin resetOffsetByTime -n localhost:9876 -g OrderGroup -t OrderTopic -s timestamp
```

## 三、Spring Boot 配置

```yaml
# application.yml
rocketmq:
  name-server: localhost:9876
  producer:
    group: order-producer-group
    send-message-timeout: 3000
    retry-times-when-send-failed: 2
    retry-times-when-send-async-failed: 2
    max-message-size: 4194304       # 4MB
    compress-message-body-threshold: 4096  # 4KB 压缩
  consumer:
    order-consumer:
      group: order-consumer-group
      topic: OrderTopic
      consume-mode: concurrent      # concurrent / orderly
      message-model: clustering     # clustering / broadcasting
      consume-thread-min: 20
      consume-thread-max: 64
      max-reconsume-times: 16
```

## 四、集群部署

```yaml
# 双主双从配置
# Broker A - Master
brokerName=broker-a
brokerId=0
brokerRole=SYNC_MASTER

# Broker A - Slave
brokerName=broker-a
brokerId=1
brokerRole=SLAVE

# Broker B - Master
brokerName=broker-b
brokerId=0
brokerRole=SYNC_MASTER

# Broker B - Slave
brokerName=broker-b
brokerId=1
brokerRole=SLAVE
```

## 五、注意事项

1. **NameServer 无状态**，可随意增减节点
2. **Broker 配置文件要与实际节点匹配**，特别是 brokerName 和 brokerId
3. **autoCreateTopicEnable 生产环境建议关闭**，手动创建 Topic
4. **flushDiskType=SYNC_FLUSH 更可靠**，但性能较低
5. **Dashboard 不要暴露到公网**，用于内部运维
