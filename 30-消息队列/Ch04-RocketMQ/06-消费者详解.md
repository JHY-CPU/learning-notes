# RocketMQ 消费者详解

## 一、消费模式

```java
// 集群模式 - 消息负载均衡（默认）
@RocketMQMessageListener(
    topic = "OrderTopic",
    consumerGroup = "OrderGroup",
    messageModel = MessageModel.CLUSTERING
)

// 广播模式 - 所有消费者都收到
@RocketMQMessageListener(
    topic = "ConfigTopic",
    consumerGroup = "ConfigGroup",
    messageModel = MessageModel.BROADCASTING
)
```

## 二、消费方式

```java
// 推模式（Push） - Broker 推送给消费者（推荐）
@RocketMQMessageListener(topic = "OrderTopic", consumerGroup = "OrderGroup")
public class PushConsumer implements RocketMQListener<Order> {
    @Override
    public void onMessage(Order order) {
        process(order);
    }
}

// 拉模式（Pull） - 消费者主动拉取
DefaultMQPullConsumer consumer = new DefaultMQPullConsumer("PullGroup");
consumer.setNamesrvAddr("localhost:9876");
consumer.start();

// 拉取消息
PullResult result = consumer.pull(messageQueue, offset, 32);
for (MessageExt msg : result.getMsgFoundList()) {
    process(msg);
}
```

## 三、并发与顺序消费

```java
// 并发消费（默认） - 多线程并行
@RocketMQMessageListener(
    topic = "OrderTopic",
    consumeMode = ConsumeMode.CONCURRENTLY,
    consumeThreadMax = 64,
    consumeThreadNumber = 20
)

// 顺序消费 - 单线程顺序
@RocketMQMessageListener(
    topic = "OrderTopic",
    consumeMode = ConsumeMode.ORDERLY,
    maxReconsumeTimes = 3
)
```

## 四、消费端配置

```java
@RocketMQMessageListener(
    topic = "OrderTopic",
    consumerGroup = "OrderGroup",
    selectorExpression = "*",                  // Tag 过滤
    consumeMode = ConsumeMode.CONCURRENTLY,
    messageModel = MessageModel.CLUSTERING,
    consumeThreadNumber = 20,                  // 消费线程数
    maxReconsumeTimes = 16,                    // 最大重试次数
    consumeTimeout = 15,                       // 消费超时（分钟）
    suspendCurrentQueueTimeMillis = 1000       // 顺序消费挂起时间
)
```

## 五、消费负载均衡

```java
// 负载均衡算法
// AllocateMessageQueueAveragely - 平均分配（默认）
// AllocateMessageQueueAveragelyByCircle - 环形分配
// AllocateMessageQueueByConfig - 手动配置
// AllocateMessageQueueByMachineRoom - 按机房分配

// 示例：5 个队列，3 个消费者
// 平均分配: C1=[Q0,Q1], C2=[Q2,Q3], C3=[Q4]
// 环形分配: C1=[Q0,Q3], C2=[Q1,Q4], C3=[Q2]
```

## 六、注意事项

1. **集群模式下每条消息只被组内一个消费者消费**
2. **广播模式下所有消费者都收到消息**，不保证顺序
3. **消费线程数要与队列数匹配**，过多线程会空闲
4. **消费失败会自动重试**，重试间隔递增
5. **顺序消费是单线程**，性能较低
