# RocketMQ 消息类型

## 一、普通消息

```java
// 同步发送
rocketMQTemplate.syncSend("OrderTopic", order);

// 异步发送
rocketMQTemplate.asyncSend("OrderTopic", order, new SendCallback() {
    @Override
    public void onSuccess(SendResult result) {
        log.info("发送成功: {}", result.getMsgId());
    }
    @Override
    public void onException(Throwable e) {
        log.error("发送失败", e);
    }
});

// 单向发送（不等待结果）
rocketMQTemplate.sendOneWay("LogTopic", logEntry);
```

## 二、顺序消息

```java
// 全局有序 - 所有消息进入同一队列
rocketMQTemplate.syncSendOrderly("OrderTopic", order, order.getOrderId());

// 分区有序 - 相同 Key 的消息有序
// 使用 hashKey 保证同一业务实体进入同一队列
public void sendOrderEvent(String orderId, String event) {
    rocketMQTemplate.syncSendOrderly("OrderTopic", event, orderId);
    // 同一 orderId 的消息按发送顺序消费
}

// 顺序消费
@RocketMQMessageListener(
    topic = "OrderTopic",
    consumerGroup = "OrderGroup",
    consumeMode = ConsumeMode.ORDERLY  // 顺序消费
)
public class OrderConsumer implements RocketMQListener<String> {
    @Override
    public void onMessage(String message) {
        // 单线程顺序消费
        process(message);
    }
}
```

## 三、事务消息

```java
// 发送事务消息
rocketMQTemplate.sendMessageInTransaction(
    "OrderTopic",
    MessageBuilder.withPayload(order)
        .setHeader("orderId", order.getId())
        .build(),
    order  // 本地事务参数
);

// 事务监听器
@RocketMQTransactionListener
public class OrderTransactionListener implements RocketMQLocalTransactionListener {

    @Override
    public RocketMQLocalTransactionState executeLocalTransaction(Message msg, Object arg) {
        try {
            Order order = (Order) arg;
            orderMapper.insert(order);  // 执行本地事务
            return RocketMQLocalTransactionState.COMMIT;
        } catch (Exception e) {
            return RocketMQLocalTransactionState.ROLLBACK;
        }
    }

    @Override
    public RocketMQLocalTransactionState checkLocalTransaction(Message msg) {
        String orderId = msg.getHeaders().get("orderId", String.class);
        Order order = orderMapper.selectById(orderId);
        return order != null
            ? RocketMQLocalTransactionState.COMMIT
            : RocketMQLocalTransactionState.UNKNOWN;
    }
}
```

## 四、延迟消息

```java
// RocketMQ 内置 18 个延迟级别
// 1=1s, 2=5s, 3=10s, 4=30s, 5=1m, 6=2m, 7=3m, 8=4m,
// 9=5m, 10=6m, 11=7m, 12=8m, 13=9m, 14=10m, 15=20m,
// 16=30m, 17=1h, 18=2h

// 指定延迟级别
Message<Order> msg = MessageBuilder.withPayload(order).build();
rocketMQTemplate.syncSend("DelayTopic", msg, 3000, 16); // level 16 = 30分钟

// RocketMQ 5.0+ 自定义延迟时间
rocketMQTemplate.syncSendDelayTimeAtMills("DelayTopic", order,
    System.currentTimeMillis() + 30 * 60 * 1000);  // 30分钟后
```

## 五、批量消息

```java
// 批量发送（同 Topic）
List<Message> messages = new ArrayList<>();
for (Order order : orders) {
    messages.add(new Message("OrderTopic", JSON.toJSONString(order).getBytes()));
}
producer.send(messages);

// Spring 方式
rocketMQTemplate.syncSend("OrderTopic", orders, 3000);
```

## 六、消息类型对比

| 类型 | 特点 | 场景 |
|------|------|------|
| 普通消息 | 最高吞吐 | 日志、通知 |
| 顺序消息 | 分区有序 | 订单状态 |
| 事务消息 | 本地事务绑定 | 支付、库存 |
| 延迟消息 | 定时投递 | 超时取消 |
| 批量消息 | 减少网络开销 | 大批量数据 |

## 七、注意事项

1. **顺序消息性能低于普通消息**，因为单队列单线程消费
2. **事务消息的半消息对消费者不可见**，直到 Commit
3. **延迟级别固定**，5.0+ 才支持自定义时间
4. **批量消息总大小不能超过 4MB**
5. **事务回查有间隔和次数限制**，确保事务能及时确定状态
