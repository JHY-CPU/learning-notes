# RocketMQ 生产者详解

## 一、生产者配置

```java
DefaultMQProducer producer = new DefaultMQProducer("OrderProducerGroup");
producer.setNamesrvAddr("localhost:9876");

// 发送配置
producer.setSendMsgTimeout(3000);               // 发送超时 3 秒
producer.setRetryTimesWhenSendFailed(2);         // 同步发送重试次数
producer.setRetryTimesWhenSendAsyncFailed(2);    // 异步发送重试次数
producer.setRetryAnotherBrokerWhenNotStoreOK(true); // 失败换 Broker 重试
producer.setMaxMessageSize(1024 * 1024 * 4);     // 最大消息 4MB
producer.setCompressMsgBodyOverHowmuch(1024 * 4); // 超过 4KB 压缩

// 发送队列选择器
producer.setSendLatencyFaultEnable(true);         // 故障延迟机制
producer.start();
```

## 二、消息发送方式

```java
// 1. 同步发送
SendResult result = producer.send(msg);
// result: SEND_OK / FLUSH_DISK_TIMEOUT / FLUSH_SLAVE_TIMEOUT / SLAVE_NOT_AVAILABLE

// 2. 异步发送
producer.send(msg, new SendCallback() {
    @Override
    public void onSuccess(SendResult result) {
        log.info("MsgId: {}", result.getMsgId());
    }
    @Override
    public void onException(Throwable e) {
        log.error("发送失败", e);
    }
});

// 3. 单向发送（不关心结果）
producer.sendOneway(msg);

// 4. 延迟发送
msg.setDelayTimeLevel(16);  // 延迟级别 16 = 30 分钟
producer.send(msg);
```

## 三、路由与负载均衡

```java
// 队列选择策略
// 1. 轮询（默认）
// 2. Hash 取模 - 同一 Key 同一队列
// 3. 自定义选择器

producer.setDefaultTopicQueueNums(8);

// 自定义队列选择器
producer.setQueueSelector((mqs, msg, arg) -> {
    String orderId = (String) arg;
    int index = orderId.hashCode() % mqs.size();
    return mqs.get(Math.abs(index));
});

// 发送时指定 Key
producer.send(msg, (mqs, msg1, arg) -> {
    String key = (String) arg;
    int index = Math.abs(key.hashCode()) % mqs.size();
    return mqs.get(index);
}, orderId);
```

## 四、Spring Boot 生产者

```java
@Service
public class OrderProducer {

    @Autowired
    private RocketMQTemplate rocketMQTemplate;

    // 同步发送
    public void sendOrder(Order order) {
        rocketMQTemplate.syncSend("OrderTopic:VIP", order);
    }

    // 带 Key 发送
    public void sendWithKey(String key, Order order) {
        rocketMQTemplate.syncSendOrderly("OrderTopic", order, key);
    }

    // 延迟发送
    public void sendDelay(Order order, int delayLevel) {
        Message<Order> msg = MessageBuilder.withPayload(order).build();
        rocketMQTemplate.syncSend("DelayTopic", msg, 3000, delayLevel);
    }

    // 事务发送
    public void sendTransaction(Order order) {
        rocketMQTemplate.sendMessageInTransaction(
            "OrderTopic",
            MessageBuilder.withPayload(order).build(),
            order
        );
    }
}
```

## 五、发送状态

```java
public enum SendStatus {
    SEND_OK,                // 发送成功
    FLUSH_DISK_TIMEOUT,     // 刷盘超时
    FLUSH_SLAVE_TIMEOUT,    // 同步到 Slave 超时
    SLAVE_NOT_AVAILABLE     // Slave 不可用
}

// 判断发送结果
SendResult result = producer.send(msg);
if (result.getSendStatus() == SendStatus.SEND_OK) {
    // 确认发送成功
} else {
    // 根据策略决定是否重试
}
```

## 六、注意事项

1. **Producer 实例要复用**，不要频繁创建销毁
2. **发送超时后会自动重试**，注意幂等性
3. **异步发送的回调在 IO 线程执行**，不要做耗时操作
4. **msgId 是 Broker 生成的全局唯一 ID**，可用于消息追踪
5. **消息体超过 4KB 建议压缩**，减少网络传输
