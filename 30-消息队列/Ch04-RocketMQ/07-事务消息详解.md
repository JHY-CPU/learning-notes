# RocketMQ 事务消息详解

## 一、事务消息流程

```
Producer                    Broker
  |                           |
  |-- 1.发送 Half Message --->|
  |<-- 2.存储成功 -------------|
  |                           |
  |-- 3.执行本地事务 --------->|
  |                           |
  |-- 4a.Commit ----------->|---> 消息对消费者可见
  |  或                       |
  |-- 4b.Rollback ---------->|--- 消息被删除
  |                           |
  |     (事务未决状态)         |
  |<-- 5.回查（每分钟） -------|
  |-- 6.返回事务状态 --------->|
```

## 二、代码实现

```java
// 事务生产者
@Component
public class TransactionOrderProducer {

    @Autowired
    private RocketMQTemplate rocketMQTemplate;

    public void createOrder(Order order) {
        Message<Order> msg = MessageBuilder
            .withPayload(order)
            .setHeader("orderId", order.getId())
            .build();

        // 发送事务消息
        rocketMQTemplate.sendMessageInTransaction(
            "OrderTopic",     // Topic
            msg,              // 消息
            order             // 本地事务参数
        );
    }
}

// 事务监听器
@RocketMQTransactionListener
public class OrderTransactionListener implements RocketMQLocalTransactionListener {

    @Autowired
    private OrderService orderService;

    /**
     * 执行本地事务
     */
    @Override
    public RocketMQLocalTransactionState executeLocalTransaction(Message msg, Object arg) {
        Order order = (Order) arg;
        try {
            // 执行本地数据库操作
            orderService.createOrder(order);
            // 本地事务成功，提交消息
            return RocketMQLocalTransactionState.COMMIT;
        } catch (Exception e) {
            log.error("本地事务执行失败", e);
            // 本地事务失败，回滚消息
            return RocketMQLocalTransactionState.ROLLBACK;
        }
    }

    /**
     * 事务回查
     * Broker 在一定时间内未收到确认，会定期回查
     */
    @Override
    public RocketMQLocalTransactionState checkLocalTransaction(Message msg) {
        String orderId = msg.getHeaders().get("orderId", String.class);
        Order order = orderService.getById(orderId);
        if (order != null) {
            return RocketMQLocalTransactionState.COMMIT;
        }
        return RocketMQLocalTransactionState.UNKNOWN; // 继续回查
    }
}
```

## 三、事务状态

```java
public enum RocketMQLocalTransactionState {
    COMMIT,     // 提交 - 消息对消费者可见
    ROLLBACK,   // 回滚 - 消息被删除
    UNKNOWN     // 未知 - Broker 会继续回查
}
```

## 四、回查机制

```properties
# Broker 事务回查配置
transactionCheckInterval=60000        # 回查间隔 60 秒
transactionTimeOut=6000               # 事务超时 6 秒
transactionCheckMax=15                # 最大回查次数
```

```java
// 回查注意事项
// 1. checkLocalTransaction 必须幂等
// 2. 根据业务数据判断事务状态
// 3. 回查超过最大次数后，消息会被丢弃或回滚
```

## 五、最佳实践

```java
// 完整的事务消息示例
@Service
public class OrderService {

    @Transactional
    public void createOrderWithTransaction(Order order) {
        // 1. 保存订单
        orderMapper.insert(order);

        // 2. 发送事务消息（在事务监听器中执行本地事务）
        // 注意：这里不使用 @Transactional，因为事务由 RocketMQ 管理
    }
}

// 事务监听器中执行本地事务
@Override
public RocketMQLocalTransactionState executeLocalTransaction(Message msg, Object arg) {
    Order order = (Order) arg;
    try {
        // 在事务中执行多个数据库操作
        orderMapper.insert(order);
        inventoryMapper.deduct(order);
        pointsMapper.add(order);
        return RocketMQLocalTransactionState.COMMIT;
    } catch (Exception e) {
        return RocketMQLocalTransactionState.ROLLBACK;
    }
}
```

## 六、注意事项

1. **Half Message 对消费者不可见**，直到事务 Commit
2. **checkLocalTransaction 必须是幂等的**
3. **事务回查间隔和次数有限**，确保业务能及时确定状态
4. **事务消息不支持延迟消息**
5. **生产环境建议关闭 autoCreateTopicEnable**，手动管理 Topic
