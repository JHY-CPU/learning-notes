# RocketMQ 核心概念

## 一、消息模型

```
Producer Group        Consumer Group
  │                       │
  │   Topic: OrderTopic   │
  │   ├── MessageQueue 0  │
  │   ├── MessageQueue 1  │
  │   ├── MessageQueue 2  │
  │   └── MessageQueue 3  │
  │                       │
Producer             Consumer
```

## 二、核心组件

### Topic（主题）

消息的逻辑分类，一个 Topic 包含多个 MessageQueue。

```java
// Topic 配置
DefaultMQAdminExt admin = new DefaultMQAdminExt();
admin.start();

// 创建 Topic
TopicConfig topicConfig = new TopicConfig();
topicConfig.setTopicName("OrderTopic");
topicConfig.setReadQueueNums(8);    // 读队列数
topicConfig.setWriteQueueNums(8);   // 写队列数
topicConfig.setPerm(6);             // 读写权限
admin.createAndUpdateTopicConfig("192.168.1.10:9876", topicConfig);
```

### MessageQueue（消息队列）

Topic 的物理分区，类似于 Kafka 的 Partition。

```
OrderTopic:
  MessageQueue 0: Broker-a, queueId=0
  MessageQueue 1: Broker-a, queueId=1
  MessageQueue 2: Broker-b, queueId=0
  MessageQueue 3: Broker-b, queueId=1
```

### Tag（标签）

Topic 下的二级分类，用于消息过滤。

```java
// Tag 过滤
// 生产者
rocketMQTemplate.convertAndSend("OrderTopic:VIP", order);

// 消费者 - 订阅 VIP 标签
@RocketMQMessageListener(topic = "OrderTopic", selectorExpression = "VIP")
public class VipOrderConsumer implements RocketMQListener<Order> { }

// 多 Tag 订阅
selectorExpression = "VIP || NORMAL"  // 订阅多个 Tag
selectorExpression = "*"              // 订阅所有
```

### Group（消费组）

```java
// Producer Group - 一类生产者的集合
DefaultMQProducer producer = new DefaultMQProducer("OrderProducerGroup");

// Consumer Group - 一类消费者的集合
@RocketMQMessageListener(
    topic = "OrderTopic",
    consumerGroup = "OrderConsumerGroup"
)
```

## 三、消息属性

```java
public class OrderMessage {
    // 消息基础属性
    private String topic;           // 主题
    private String tags;            // 标签
    private String keys;            // 消息索引键
    private byte[] body;            // 消息体

    // 系统属性
    private String msgId;           // 消息 ID
    private int queueId;            // 队列 ID
    private long bornTimestamp;     // 产生时间
    private long storeTimestamp;    // 存储时间
    private int reconsumeTimes;     // 重试次数
    private int delayTimeLevel;     // 延迟级别
}
```

```java
// 发送带属性的消息
Message msg = new Message("OrderTopic", "VIP",
    "orderId=10001", orderJson.getBytes());
msg.setKeys("order-10001");        // 设置索引键
msg.putUserProperty("traceId", TraceContext.getTraceId());
producer.send(msg);
```

## 四、注意事项

1. **Tag 只支持精确匹配**，复杂的过滤用 SQL92 表达式
2. **MessageQueue 数量创建后可调整**，但增加会打乱消息顺序
3. **Consumer Group 内的消息负载均衡**，每条消息只被组内一个消费者消费
4. **Keys 用于消息查询**，可以建立索引
5. **Topic 名称不要包含特殊字符**，建议用字母和连字符
