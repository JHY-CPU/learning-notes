# RocketMQ 集群架构

## 一、集群组件

```
NameServer (无状态，可集群)
├── 存储 Broker 路由信息
├── 服务注册与发现
└── 无中心节点，节点之间不通信

Broker (有状态)
├── Master - 处理读写请求
├── Slave - 同步/异步复制
└── 支持多种集群模式
```

## 二、集群模式

### 单 Master

```yaml
# 开发环境，无高可用
brokerClusterName: DefaultCluster
brokerName: broker-a
brokerId: 0  # 0 表示 Master
```

### 双主双从（推荐）

```yaml
# Broker A Master
brokerName: broker-a
brokerId: 0
brokerRole: ASYNC_MASTER

# Broker A Slave
brokerName: broker-a
brokerId: 1
brokerRole: SLAVE

# Broker B Master
brokerName: broker-b
brokerId: 0
brokerRole: ASYNC_MASTER

# Broker B Slave
brokerName: broker-b
brokerId: 1
brokerRole: SLAVE
```

### DLedger 模式（自动主从切换）

```properties
# 开启 DLedger
enableDLegerCommitLog=true
dLegerGroup=broker-a-group
dLegerPeers=n0-broker-a:40911;n1-broker-a:40912;n2-broker-a:40913
dLegerSelfId=n0
```

## 三、复制策略

```java
// 同步复制 - Master 等待 Slave 确认
brokerRole=SYNC_MASTER
flushDiskType=SYNC_FLUSH
// 可靠性最高，性能最低

// 异步复制 - Master 不等待 Slave
brokerRole=ASYNC_MASTER
flushDiskType=ASYNC_FLUSH
// 可靠性中等，性能高（推荐）
```

## 四、刷盘策略

```java
// 同步刷盘 - 消息写入磁盘后返回
flushDiskType=SYNC_FLUSH
// 不丢消息，延迟高

// 异步刷盘 - 消息写入缓存后返回（默认）
flushDiskType=ASYNC_FLUSH
// 性能高，可能丢少量消息
```

## 五、NameServer 集群

```yaml
# NameServer 节点相互独立
namesrv1: 192.168.1.10:9876
namesrv2: 192.168.1.11:9876
namesrv3: 192.168.1.12:9876

# Broker 配置所有 NameServer
namesrvAddr=192.168.1.10:9876;192.168.1.11:9876;192.168.1.12:9876
```

## 六、推荐配置

```yaml
# 生产环境推荐
集群模式: 双主双从 (或 DLedger)
复制策略: 异步复制 + 异步刷盘
NameServer: 3 节点
Broker: 2 Master + 2 Slave
同步策略: 最终一致性

# 金融级配置
集群模式: DLedger (3 副本)
复制策略: 同步复制 + 同步刷盘
一致性: 强一致性
```

## 七、注意事项

1. **NameServer 节点之间不通信**，每个节点独立存储路由信息
2. **DLedger 模式下，副本数必须是奇数（3 或 5）**
3. **异步复制+异步刷盘是性能最高的配置**
4. **Master 宕机后，Slave 不会自动提升为 Master**（DLedger 除外）
5. **跨机房部署建议用 DLedger 模式**
