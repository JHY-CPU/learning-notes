# RocketMQ 消息过滤

## 一、Tag 过滤（精确匹配）

```java
// 生产者 - 发送带 Tag 的消息
rocketMQTemplate.convertAndSend("OrderTopic:VIP", vipOrder);
rocketMQTemplate.convertAndSend("OrderTopic:NORMAL", normalOrder);

// 消费者 - 订阅指定 Tag
@RocketMQMessageListener(
    topic = "OrderTopic",
    selectorExpression = "VIP"  // 只消费 VIP 标签
)
public class VipConsumer implements RocketMQListener<Order> { }

// 多 Tag 订阅
@RocketMQMessageListener(
    topic = "OrderTopic",
    selectorExpression = "VIP || NORMAL"  // 消费 VIP 和 NORMAL
)
public class AllConsumer implements RocketMQListener<Order> { }

// 订阅所有
@RocketMQMessageListener(
    topic = "OrderTopic",
    selectorExpression = "*"
)
```

## 二、SQL92 过滤（属性过滤）

```java
// 生产者 - 设置用户属性
Message<Order> msg = MessageBuilder
    .withPayload(order)
    .setHeader("price", order.getPrice())
    .setHeader("region", order.getRegion())
    .setHeader("vipLevel", order.getVipLevel())
    .build();
rocketMQTemplate.send("OrderTopic", msg);

// 消费者 - SQL92 表达式过滤
@RocketMQMessageListener(
    topic = "OrderTopic",
    selectorType = SelectorType.SQL92,
    selectorExpression = "price > 100 AND region = 'CN'"
)
public class HighValueConsumer implements RocketMQListener<Order> { }

// SQL92 支持的语法
// 比较: =, >, >=, <, <=, <>
// 逻辑: AND, OR, NOT
// 范围: IN, NOT IN
// 空值: IS NULL, IS NOT NULL
// 字符串: 'value'
```

## 三、Filter Server 过滤

```properties
# Broker 开启过滤支持
enablePropertyFilter=true
# 开启后，SQL92 过滤在 Broker 端执行
# 未开启时，SQL92 过滤在消费端执行（浪费网络带宽）
```

## 四、过滤性能对比

| 过滤方式 | 性能 | 灵活性 | 适用场景 |
|---------|------|--------|---------|
| Tag | 最高 | 低 | 简单分类 |
| SQL92 | 中 | 高 | 复杂条件 |
| Filter Server | 最低 | 最高 | 自定义逻辑 |

## 五、注意事项

1. **Tag 过滤性能最好**，优先使用 Tag
2. **SQL92 需要 Broker 开启 enablePropertyFilter**
3. **SQL92 过滤的属性值只能是数字或字符串**
4. **过滤在 Broker 端执行时，减少网络传输**
5. **属性名不要包含特殊字符**
