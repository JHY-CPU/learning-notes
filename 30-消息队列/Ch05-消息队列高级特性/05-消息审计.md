# 消息审计

## 一、什么是消息审计

消息审计是对消息全生命周期的**核对与验证**，确保消息不丢失、不重复、处理正确。

```
审计维度:
├── 发送审计: 消息是否成功发送
├── 存储审计: Broker 是否正确存储
├── 消费审计: 消息是否被消费
└── 对账审计: 业务数据与消息数据是否一致
```

## 二、发送与消费计数

```java
// 生产端计数
@Service
public class AuditProducer {
    private final AtomicLong sendCount = new AtomicLong(0);
    private final AtomicLong sendSuccess = new AtomicLong(0);
    private final AtomicLong sendFail = new AtomicLong(0);

    public void send(String topic, Object msg) {
        sendCount.incrementAndGet();
        try {
            kafkaTemplate.send(topic, msg).get();
            sendSuccess.incrementAndGet();
        } catch (Exception e) {
            sendFail.incrementAndGet();
        }
    }
}

// 消费端计数
@Component
public class AuditConsumer {
    private final AtomicLong consumeCount = new AtomicLong(0);

    @KafkaListener(topics = "order-events")
    public void consume(ConsumerRecord<String, String> record) {
        consumeCount.incrementAndGet();
        process(record);
    }
}
```

## 三、端到端对账

```java
@Service
public class ReconciliationService {

    // 定时对账任务
    @Scheduled(cron = "0 0 * * * ?")  // 每小时
    public void reconcile() {
        // 1. 统计生产端消息数
        long produced = messageLogMapper.countByTimeRange(startHour, endHour);

        // 2. 统计消费端消息数
        long consumed = consumeLogMapper.countByTimeRange(startHour, endHour);

        // 3. 对比差异
        long diff = produced - consumed;
        if (diff > 0) {
            log.warn("存在 {} 条未消费消息", diff);
            alertService.alert("消息消费不完整", diff);
        }

        // 4. 记录对账结果
        reconciliationMapper.insert(new ReconciliationResult(startHour, endHour, produced, consumed));
    }
}
```

## 四、注意事项

1. **审计数据与业务数据分开存储**，避免影响业务性能
2. **定时对账频率根据业务重要性设置**
3. **消费延迟也是重要的审计指标**
4. **审计结果需要持久化**，用于追溯历史
5. **异常时要能快速定位到具体消息**
