# 消息可靠性保证

## 一、端到端可靠性

消息从生产到消费的全链路可靠性保障，涉及三个阶段：**生产端、Broker 存储、消费端**。

```java
// 生产端可靠性
// RabbitMQ: Publisher Confirm + Publisher Return
// Kafka: acks=all + 幂等生产者
// RocketMQ: 同步发送 + 事务消息

// Broker 端可靠性
// RabbitMQ: 持久化 + Quorum Queue
// Kafka: replication-factor=3 + min.insync.replicas=2
// RocketMQ: SYNC_MASTER + SYNC_FLUSH

// 消费端可靠性
// RabbitMQ: 手动 ACK
// Kafka: 手动提交 Offset
// RocketMQ: 返回 ConsumeSuccessStatus
```

## 二、生产端保障

```java
// 本地消息表方案
@Service
public class ReliableProducerService {

    @Transactional
    public void sendReliably(String topic, Object message) {
        // 1. 业务操作
        orderMapper.insert(order);

        // 2. 保存待发送消息（同一事务）
        PendingMessage pending = new PendingMessage();
        pending.setTopic(topic);
        pending.setContent(JSON.toJSONString(message));
        pending.setStatus("PENDING");
        pendingMapper.insert(pending);
    }

    // 3. 定时任务扫描并发送
    @Scheduled(fixedDelay = 1000)
    public void pollAndSend() {
        List<PendingMessage> pending = pendingMapper.selectPending(100);
        for (PendingMessage msg : pending) {
            try {
                kafkaTemplate.send(msg.getTopic(), msg.getContent());
                pendingMapper.updateStatus(msg.getId(), "SENT");
            } catch (Exception e) {
                pendingMapper.incrementRetry(msg.getId());
            }
        }
    }
}
```

## 三、Broker 端保障

```yaml
# Kafka 最高可靠性配置
spring:
  kafka:
    producer:
      acks: all
      properties:
        enable.idempotence: true
        min.insync.replicas: 2

# RabbitMQ 最高可靠性配置
spring:
  rabbitmq:
    publisher-confirm-type: correlated
    publisher-returns: true
    template:
      mandatory: true
```

## 四、消费端保障

```java
// 消费端幂等 + 手动确认
@Component
public class ReliableConsumer {

    @KafkaListener(topics = "order-events")
    public void consume(ConsumerRecord<String, String> record, Acknowledgment ack) {
        String messageId = extractMessageId(record);

        // 幂等检查
        if (!idempotentService.tryProcess(messageId)) {
            ack.acknowledge();
            return;
        }

        try {
            processMessage(record);
            ack.acknowledge();  // 处理成功后确认
        } catch (Exception e) {
            idempotentService.clear(messageId);
            throw e;  // 触发重试
        }
    }
}
```

## 五、可靠性等级

```yaml
等级1 - 最高性能:
  生产者: acks=0, 异步
  Broker: 1 副本
  消费者: 自动提交
  结果: 可能丢消息

等级2 - 平衡配置:
  生产者: acks=1, 同步
  Broker: 3 副本, 异步复制
  消费者: 手动提交
  结果: 极小概率丢消息

等级3 - 最高可靠:
  生产者: acks=all, 幂等+事务
  Broker: 3 副本, 同步复制, min.insync=2
  消费者: 手动提交 + 幂等
  结果: 不丢消息
```

## 六、注意事项

1. **可靠性与性能是矛盾的**，需要根据业务需求权衡
2. **端到端可靠性需要所有环节都配置正确**
3. **本地消息表是最可靠的生产端方案**
4. **消费端幂等是必须的**，无论生产端多可靠
5. **监控每个环节的失败率**，及时发现可靠性问题
