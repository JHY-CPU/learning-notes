# 跨集群同步

## 一、Kafka MirrorMaker 2

```json
// MM2 配置
{
  "clusters": ["source", "target"],
  "source.cluster.bootstrap.servers": "source-kafka:9092",
  "target.cluster.bootstrap.servers": "target-kafka:9092",
  "source->target.enabled": true,
  "source->target.topics": ".*",
  "topics": "order-events",
  "groups": ".*"
}
```

```bash
# 启动 MirrorMaker 2
connect-mirror-maker.sh mm2.properties
```

## 二、RabbitMQ Federation

```bash
# 单向同步
rabbitmqctl set_parameter federation-upstream my-upstream \
  '{"uri":"amqp://remote-host","ack-mode":"on-confirm"}'

rabbitmqctl set_parameter federation-policy my-policy \
  '{"pattern":"^federated\\.","upstream-set":"all"}'
```

## 三、RocketMQ 跨集群

```java
// RocketMQ 原生支持多集群
// 通过消费端订阅远程集群实现同步
DefaultMQPushConsumer consumer = new DefaultMQPushConsumer("Group");
consumer.setNamesrvAddr("remote-namesrv:9876");
consumer.subscribe("RemoteTopic", "*");
```

## 四、注意事项

1. **跨集群同步要避免循环同步**
2. **同步延迟要监控**，确保数据一致性
3. **网络分区时要有降级策略**
4. **双向同步需要解决冲突**，使用版本号或时间戳
5. **同步配置变更后要验证数据完整性**
