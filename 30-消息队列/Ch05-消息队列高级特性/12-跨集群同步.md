## 一、Kafka MirrorMaker 2

```json
// MM2 配置
{
  "clusters": ["source", "target"],
  "source.cluster.bootstrap.servers": "source-kafka:9092",
  "target.cluster.bootstrap.servers": "target-kafka:9092",
  "source->target.enabled": true,
  "source->target.topics": ".*",
  "topics": "order-events",
  "groups": ".*"
}
```

```bash
# 启动 MirrorMaker 2
connect-mirror-maker.sh mm2.properties
```

## 四、Kafka MirrorMaker 2 详细配置

```properties
# mm2.properties
clusters = source, target

source.bootstrap.servers = source-kafka-1:9092,source-kafka-2:9092
target.bootstrap.servers = target-kafka-1:9092,target-kafka-2:9092

source->target.enabled = true
source->target.topics = order-events,payment-events,inventory-events

# 主题重命名策略：添加源集群前缀
source->target.topic.creation.enabled = true
source->target.topic.creation.default.replication.factor = 3
source->target.topic.creation.default.partitions = 12

# 同步间隔和刷新策略
refresh.topics.interval.seconds = 60
refresh.groups.interval.seconds = 60
sync.topic.acls.enabled = true

# 检查点间隔
source->target.sync.group.offsets.enabled = true
source->target.sync.group.offsets.interval.seconds = 60
```

## 五、Pulsar 跨地域复制

```java
// Pulsar Geo-Replication 配置
// 创建租户时指定集群
admin.tenants().createTenant("my-tenant",
    new TenantInfoImpl(
        Sets.newHashSet("admin"),
        Sets.newHashSet("cluster-east", "cluster-west")
    )
);

// 启用 Namespace 级别的复制
admin.namespaces().setNamespaceReplicationClusters("my-tenant/my-ns",
    Sets.newHashSet("cluster-east", "cluster-west"));

// 生产者发送消息，自动复制到所有集群
Producer<byte[]> producer = client.newProducer()
    .topic("persistent://my-tenant/my-ns/my-topic")
    .create();
producer.send("Hello".getBytes());
```

## 六、同步一致性验证

```java
// 数据校验服务
@Component
public class SyncVerificationService {
    @Scheduled(fixedDelay = 300000)  // 每 5 分钟校验一次
    public void verifySync() {
        // 源集群统计
        long sourceCount = getSourceTopicCount("order-events", "source-cluster");
        // 目标集群统计
        long targetCount = getSourceTopicCount("order-events", "target-cluster");

        double diffPercent = Math.abs(sourceCount - targetCount) * 100.0 / sourceCount;

        if (diffPercent > 1.0) {
            alertService.send("跨集群同步告警",
                String.format("源: %d, 目标: %d, 差异: %.2f%%",
                    sourceCount, targetCount, diffPercent));
        }
    }

    // 对比消息内容
    public void deepVerify(String topic, int sampleSize) {
        List<String> sourceMessages = sampleMessages(topic, "source-cluster", sampleSize);
        List<String> targetMessages = sampleMessages(topic, "target-cluster", sampleSize);
        // 对比消息内容是否一致
        Set<String> sourceSet = new HashSet<>(sourceMessages);
        sourceSet.removeAll(targetMessages);
        if (!sourceSet.isEmpty()) {
            alertService.send("同步内容不一致", "缺失消息数: " + sourceSet.size());
        }
    }
}
```

## 七、注意事项

1. **跨集群同步要避免循环同步**
2. **同步延迟要监控**，确保数据一致性
3. **网络分区时要有降级策略**
4. **双向同步需要解决冲突**，使用版本号或时间戳
5. **同步配置变更后要验证数据完整性**
6. **同步带宽要预留**，避免影响正常业务流量
7. **定期做全量数据对账**，确保两端数据一致
