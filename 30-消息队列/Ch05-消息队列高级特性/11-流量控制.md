# 流量控制

## 一、生产端限流

```java
// Kafka 生产端限流
props.put("max.request.size", 1048576);       // 最大请求 1MB
props.put("buffer.memory", 33554432);         // 缓冲区 32MB
props.put("max.block.ms", 60000);             // 缓冲区满时阻塞 60 秒

// 令牌桶限流
public class RateLimitedProducer {
    private final RateLimiter rateLimiter = RateLimiter.create(10000); // 10000 msg/s

    public void send(String topic, String message) {
        rateLimiter.acquire();  // 获取令牌
        kafkaTemplate.send(topic, message);
    }
}
```

## 四、自适应限流

```java
// 基于消费延迟的自适应限流
@Component
public class AdaptiveRateLimiter {
    private volatile double currentRate = 1000;  // 初始 1000 msg/s
    private final double minRate = 100;
    private final double maxRate = 10000;

    @Scheduled(fixedRate = 5000)
    public void adjustRate() {
        // 获取消费延迟
        long consumerLag = kafkaAdmin.getConsumerLag("my-group", "my-topic");

        if (consumerLag > 100000) {
            // 延迟过大，降低生产速率
            currentRate = Math.max(minRate, currentRate * 0.7);
            log.warn("消费延迟过高 {}，降速至 {}", consumerLag, currentRate);
        } else if (consumerLag < 1000) {
            // 延迟很小，可以提速
            currentRate = Math.min(maxRate, currentRate * 1.2);
            log.info("消费延迟正常，提速至 {}", currentRate);
        }
    }

    public boolean tryAcquire() {
        // 令牌桶实现
        return tokenBucket.tryAcquire(currentRate);
    }
}
```

## 五、背压机制

```java
// Kafka 消费端背压
@Component
public class BackpressureConsumer {
    private final Semaphore semaphore = new Semaphore(100);

    @KafkaListener(topics = "high-traffic")
    public void consume(ConsumerRecord<String, String> record,
                        Acknowledgment ack) {
        semaphore.acquire();
        CompletableFuture.runAsync(() -> {
            try {
                process(record);
                ack.acknowledge();
            } catch (Exception e) {
                log.error("处理失败", e);
            } finally {
                semaphore.release();
            }
        });
    }
}

// RabbitMQ 预取计数实现背压
@Bean
public SimpleRabbitListenerContainerFactory containerFactory() {
    SimpleRabbitListenerContainerFactory factory =
        new SimpleRabbitListenerContainerFactory();
    factory.setPrefetchCount(10);  // 预取 10 条，控制消费速率
    return factory;
}
```

## 六、监控与告警

```java
// 流控监控
@Component
public class FlowControlMonitor {
    @Autowired
    private MeterRegistry meterRegistry;

    public void recordThrottle(String topic, boolean throttled) {
        Counter.builder("mq.throttle.events")
            .tag("topic", topic)
            .tag("throttled", String.valueOf(throttled))
            .register(meterRegistry)
            .increment();
    }

    @Scheduled(fixedRate = 30000)
    public void alertOnHighLag() {
        long lag = getConsumerLag();
        if (lag > 100000) {
            alertService.send("消费积压告警", "当前积压: " + lag + " 条");
        }
    }
}
```

## 七、注意事项

1. **生产端限流要设置合理的超时**，避免无限阻塞
2. **消费端限流要与业务处理能力匹配**
3. **Broker 流控是最后一道防线**，避免系统崩溃
4. **限流策略要支持动态调整**，应对流量变化
5. **限流时要记录日志和告警**，及时发现异常
6. **背压比硬限流更优雅**，允许系统自我调节
7. **流控策略要在灰度环境验证**，避免线上流量突降
