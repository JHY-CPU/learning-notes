# 流量控制

## 一、生产端限流

```java
// Kafka 生产端限流
props.put("max.request.size", 1048576);       // 最大请求 1MB
props.put("buffer.memory", 33554432);         // 缓冲区 32MB
props.put("max.block.ms", 60000);             // 缓冲区满时阻塞 60 秒

// 令牌桶限流
public class RateLimitedProducer {
    private final RateLimiter rateLimiter = RateLimiter.create(10000); // 10000 msg/s

    public void send(String topic, String message) {
        rateLimiter.acquire();  // 获取令牌
        kafkaTemplate.send(topic, message);
    }
}
```

## 二、消费端限流

```java
// Kafka 消费端限流
props.put("max.poll.records", 100);          // 每次最多 100 条
props.put("fetch.max.bytes", 52428800);      // 每次最多 50MB
props.put("max.partition.fetch.bytes", 1048576); // 每分区 1MB

// RocketMQ 消费限流
@RocketMQMessageListener(
    topic = "OrderTopic",
    consumeThreadNumber = 10,  // 消费线程数
    maxReconsumeTimes = 16
)
```

## 三、Broker 端流控

```yaml
# RabbitMQ 流控
# 内存阈值 - 超过阻塞生产者
vm_memory_high_watermark.relative = 0.6
# 磁盘阈值 - 超过阻塞生产者
disk_free_limit.absolute = 2GB

# Kafka 流控
# 配额限制
kafka-configs --alter --entity-type clients --entity-default \
  --add-config 'producer_byte_rate=1048576,consumer_byte_rate=2097152'
```

## 四、注意事项

1. **生产端限流要设置合理的超时**，避免无限阻塞
2. **消费端限流要与业务处理能力匹配**
3. **Broker 流控是最后一道防线**，避免系统崩溃
4. **限流策略要支持动态调整**，应对流量变化
5. **限流时要记录日志和告警**，及时发现异常
