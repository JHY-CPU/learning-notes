# 批量消息

## 一、生产端批量

```java
// Kafka 批量发送
// 通过 batch.size 和 linger.ms 控制
props.put("batch.size", 65536);   // 64KB
props.put("linger.ms", 20);       // 等待 20ms 攒批

// RocketMQ 批量发送
List<Message> messages = new ArrayList<>();
for (Order order : orders) {
    messages.add(new Message("OrderTopic", JSON.toJSONBytes(order)));
}
producer.send(messages);
```

## 二、消费端批量

```java
// Kafka 批量消费
@KafkaListener(topics = "order-events", containerFactory = "batchFactory")
public void batchConsume(List<ConsumerRecord<String, String>> records) {
    List<Order> orders = records.stream()
        .map(r -> JSON.parseObject(r.value(), Order.class))
        .collect(Collectors.toList());
    orderService.batchInsert(orders);  // 批量写库
}

// 配置批量工厂
@Bean
public ConcurrentKafkaListenerContainerFactory<String, String> batchFactory() {
    ConcurrentKafkaListenerContainerFactory<String, String> factory =
        new ConcurrentKafkaListenerContainerFactory<>();
    factory.setConsumerFactory(consumerFactory());
    factory.setBatchListener(true);
    factory.getContainerProperties().setAckMode(AckMode.BATCH);
    return factory;
}
```

## 三、RocketMQ 批量消息

```java
// RocketMQ 批量发送（同一 Topic）
public void sendBatch(List<Order> orders) throws Exception {
    List<Message> messages = new ArrayList<>();
    for (Order order : orders) {
        Message msg = new Message("OrderTopic",
            order.getOrderId().toString(),
            JSON.toJSONBytes(order));
        messages.add(msg);
    }
    // 批量发送，总大小不超过 4MB
    SendResult result = producer.send(messages);
}

// RocketMQ 消费批量
@RocketMQMessageListener(
    topic = "OrderTopic",
    consumerGroup = "batch-group",
    consumeMessageBatchMaxSize = 32  // 每批最多 32 条
)
public class BatchOrderConsumer implements RocketMQListener<List<MessageExt>> {
    @Override
    public void onMessage(List<MessageExt> messages) {
        List<Order> orders = messages.stream()
            .map(m -> JSON.parseObject(m.getBody(), Order.class))
            .collect(Collectors.toList());
        orderService.batchSave(orders);
    }
}
```

## 四、批量消息的失败处理

```java
// 批量消费失败降级为逐条处理
public void consumeWithFallback(List<ConsumerRecord<String, String>> records) {
    try {
        // 尝试批量处理
        orderService.batchInsert(parseOrders(records));
    } catch (Exception e) {
        log.warn("批量处理失败，降级为逐条处理", e);
        for (ConsumerRecord<String, String> record : records) {
            try {
                Order order = JSON.parseObject(record.value(), Order.class);
                orderService.save(order);
            } catch (Exception ex) {
                log.error("单条处理失败: {}", record.key(), ex);
                sendToDLQ(record);  // 进入死信队列
            }
        }
    }
}
```

## 五、批量大小调优

```java
// 动态调整批量大小
public class AdaptiveBatchConfig {
    // 根据消息处理耗时动态调整
    public int calculateBatchSize(long avgProcessTimeMs) {
        if (avgProcessTimeMs < 10) return 1000;      // 快速处理 → 大批量
        if (avgProcessTimeMs < 100) return 500;      // 中等处理 → 中批量
        if (avgProcessTimeMs < 1000) return 100;     // 慢处理 → 小批量
        return 10;                                     // 极慢 → 最小批量
    }
}
```

## 六、注意事项

1. **批量消息总大小不要超过 Broker 限制**（Kafka 默认 1MB，RocketMQ 4MB）
2. **批量消费要配合批量提交 Offset**
3. **批量写入数据库比单条写入性能高 10 倍以上**
4. **批量消费失败时，整批消息都会重试**
5. **批量大小要与消费能力匹配**，避免单批处理时间过长
6. **批量发送要保证同一批消息发往同一分区**（Kafka 按 Key 分区）
7. **监控单批处理耗时**，超时告警
