# 消息压缩

## 一、压缩算法对比

| 算法 | 压缩率 | CPU 开销 | 速度 | 适用场景 |
|------|--------|---------|------|---------|
| none | 无 | 无 | 最快 | 小消息 |
| gzip | 最高 | 最高 | 慢 | 存储优化 |
| snappy | 中等 | 中等 | 快 | 通用场景 |
| lz4 | 低 | 最低 | 最快 | 高吞吐 |
| zstd | 高 | 中等 | 快 | 推荐 |

## 三、应用层压缩

```java
// 大消息应用层压缩
public void sendLargeMessage(Object data) {
    String json = JSON.toJSONString(data);
    byte[] compressed = Snappy.compress(json.getBytes());
    kafkaTemplate.send("topic", compressed);
}

// 消费端解压
@KafkaListener(topics = "topic")
public void consume(byte[] compressed) {
    byte[] decompressed = Snappy.uncompress(compressed);
    String json = new String(decompressed);
    Order order = JSON.parseObject(json, Order.class);
}
```

## 四、RabbitMQ 消息压缩

```java
// RabbitMQ 使用压缩插件或应用层压缩
public void sendCompressedMessage(byte[] data) {
    byte[] compressed = gzip(data);
    MessageProperties props = new MessageProperties();
    props.setContentEncoding("gzip");
    Message message = new Message(compressed, props);
    rabbitTemplate.send("exchange", "routing-key", message);
}

// 消费端解压
@RabbitListener(queues = "compressed-queue")
public void consume(Message message) {
    byte[] body = message.getBody();
    if ("gzip".equals(message.getMessageProperties().getContentEncoding())) {
        body = gunzip(body);
    }
    String json = new String(body);
    // 处理消息
}
```

## 五、RocketMQ 消息压缩

```java
// RocketMQ 开启压缩（Broker 配置）
// enableGzipCompressing=true

// 应用层压缩（更灵活）
public void sendCompressed(Object data) {
    byte[] jsonBytes = JSON.toJSONBytes(data);
    byte[] compressed = gzip(jsonBytes);

    // 压缩后超过一定比例才使用压缩结果
    if (compressed.length < jsonBytes.length * 0.8) {
        Message msg = new Message("TopicA", compressed);
        msg.putUserProperty("compressed", "gzip");
        producer.send(msg);
    } else {
        producer.send(new Message("TopicA", jsonBytes));
    }
}
```

## 七、注意事项

1. **压缩在 Producer 端执行**，Broker 存储压缩后的数据
2. **小消息（<1KB）压缩意义不大**，甚至可能变大
3. **zstd 是 Kafka 2.1+ 推荐的算法**，压缩率和速度平衡最好
4. **压缩会增加 CPU 开销**，在吞吐量和资源之间权衡
5. **批量发送 + 压缩效果最好**，因为可以跨消息压缩
6. **监控压缩率变化**，异常时可能是消息格式改变
7. **同一 Topic 建议使用统一压缩算法**，避免兼容问题
