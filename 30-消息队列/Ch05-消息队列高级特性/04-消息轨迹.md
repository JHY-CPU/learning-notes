# 消息轨迹

## 一、什么是消息轨迹

消息轨迹记录消息从生产到消费的**全链路生命周期**，包括发送、存储、消费各个节点的状态和时间。

```
消息轨迹记录:
├── 生产者信息: IP、ProducerGroup、发送时间
├── Broker 信息: 存储时间、Topic、QueueId、Offset
└── 消费者信息: IP、ConsumerGroup、消费时间、消费状态
```

## 二、RocketMQ 消息轨迹

```java
// 开启消息轨迹
DefaultMQProducer producer = new DefaultMQProducer("ProducerGroup", true);
// 第二个参数 enableMsgTrace=true

// 自定义轨迹 Topic
producer.setCustomizedTraceTopic("RMQ_SYS_TRACE_TOPIC");

// 查询消息轨迹
adminExt.queryMessage("OrderTopic", "MSG_ID");
```

```yaml
# Spring Boot 配置
rocketmq:
  producer:
    enable-msg-trace: true
    customized-trace-topic: RMQ_SYS_TRACE_TOPIC
```

## 三、Kafka 消息追踪

```java
// 通过 Header 传递 TraceID
public class TraceProducer {

    public void sendWithTrace(String topic, String key, String value) {
        ProducerRecord<String, String> record = new ProducerRecord<>(topic, key, value);
        record.headers().add("traceId", TraceContext.getTraceId().getBytes());
        record.headers().add("spanId", TraceContext.getSpanId().getBytes());
        record.headers().add("timestamp", String.valueOf(System.currentTimeMillis()).getBytes());
        kafkaTemplate.send(record);
    }
}

// 消费端读取 TraceID
@KafkaListener(topics = "order-events")
public void consume(ConsumerRecord<String, String> record) {
    String traceId = new String(record.headers().lastHeader("traceId").value());
    TraceContext.setTraceId(traceId);
    process(record);
}
```

## 四、注意事项

1. **RocketMQ 原生支持消息轨迹**，Kafka 需要自行实现
2. **消息轨迹会增加 Broker 存储和计算开销**
3. **轨迹数据有过期策略**，默认保留 3 天
4. **通过 MsgId 查询轨迹是最常用的方式**
5. **分布式追踪（Jaeger/SkyWalking）与消息轨迹配合使用效果更好**
