# 消息优先级

## 一、RabbitMQ 优先级队列

```java
// 声明优先级队列
@Bean
public Queue priorityQueue() {
    return QueueBuilder.durable("priority-queue")
        .withArgument("x-max-priority", 10)
        .build();
}

// 发送优先级消息
rabbitTemplate.convertAndSend("exchange", "key", msg, m -> {
    m.getMessageProperties().setPriority(10);  // 高优先级
    return m;
});
```

## 二、RocketMQ 优先级

```java
// RocketMQ 不原生支持优先级
// 方案: 使用多个 Topic 或 Tag 模拟

// VIP 订单 -> 高优先级 Topic
rocketMQTemplate.syncSend("OrderTopic:VIP", order);
// 普通订单 -> 低优先级 Topic
rocketMQTemplate.syncSend("OrderTopic:NORMAL", order);
```

## 三、Kafka 优先级

```java
// Kafka 不原生支持优先级
// 方案: 多 Topic + 优先消费

// 高优先级 Topic 优先消费
@KafkaListener(topics = "high-priority-topic", groupId = "priority-group")
public void consumeHigh(ConsumerRecord<String, String> record) {
    process(record);
}
```

## 四、多 Topic 优先级方案详解

```java
// 通用多 Topic 优先级方案
public class PriorityMessageRouter {
    private static final String HIGH_TOPIC = "orders-priority-high";
    private static final String NORMAL_TOPIC = "orders-priority-normal";
    private static final String LOW_TOPIC = "orders-priority-low";

    public void route(Order order) {
        String topic = determinePriority(order);
        kafkaTemplate.send(topic, order.getOrderId().toString(), JSON.toJSONString(order));
    }

    private String determinePriority(Order order) {
        if (order.getAmount().compareTo(new BigDecimal("10000")) > 0) {
            return HIGH_TOPIC;  // 大额订单高优先级
        }
        if (order.isVIP()) {
            return HIGH_TOPIC;  // VIP 高优先级
        }
        if (order.isDelayed()) {
            return LOW_TOPIC;   // 超时订单低优先级
        }
        return NORMAL_TOPIC;
    }
}
```

## 五、优先级消费策略

```java
// 优先消费高优先级 Topic
@Component
public class PriorityConsumer {
    // 高优先级 - 更多线程
    @KafkaListener(topics = "orders-priority-high", groupId = "order-group",
                   concurrency = "4")
    public void consumeHigh(ConsumerRecord<String, String> record) {
        process(record);
    }

    // 正常优先级 - 标准线程数
    @KafkaListener(topics = "orders-priority-normal", groupId = "order-group",
                   concurrency = "2")
    public void consumeNormal(ConsumerRecord<String, String> record) {
        process(record);
    }

    // 低优先级 - 较少线程
    @KafkaListener(topics = "orders-priority-low", groupId = "order-group",
                   concurrency = "1")
    public void consumeLow(ConsumerRecord<String, String> record) {
        process(record);
    }
}
```

## 六、RabbitMQ 优先级深入

```java
// 高级优先级配置
@Bean
public Queue advancedPriorityQueue() {
    return QueueBuilder.durable("priority-orders")
        .withArgument("x-max-priority", 10)  // 10 个优先级
        .withArgument("x-overflow", "reject-publish")  // 溢出时拒绝
        .build();
}

// 动态优先级：根据消息属性设置
public void sendWithDynamicPriority(Order order, int priority) {
    rabbitTemplate.convertAndSend("order-exchange", "order.create",
        new Message(JSON.toJSONBytes(order), createProperties(priority)));
}

private MessageProperties createProperties(int priority) {
    MessageProperties props = new MessageProperties();
    props.setPriority(priority);
    return props;
}
```

## 七、注意事项

1. **只有 RabbitMQ 原生支持优先级**
2. **优先级队列有性能开销**
3. **x-max-priority 建议不超过 10**
4. **多 Topic 方案更适合 Kafka 和 RocketMQ**
5. **优先级只在队列积压时有意义**
6. **避免大量低优先级消息饿死**，设置消费比例
7. **优先级策略要根据业务 SLA 制定**，不要随意设置
