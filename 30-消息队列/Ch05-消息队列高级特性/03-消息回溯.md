# 消息回溯

## 一、什么是消息回溯

消息回溯是指**将消费位点重置到历史位置，重新消费已处理过的消息**。适用于数据修复、重算等场景。

```java
// Kafka 按时间回溯
kafka-consumer-groups --bootstrap-server localhost:9092 \
  --group my-group --topic order-events \
  --reset-offsets --to-datetime 2024-01-01T00:00:00.000 --execute

// RocketMQ 按时间回溯
DefaultMQPushConsumer consumer = new DefaultMQPushConsumer("Group");
consumer.setConsumeFromWhere(ConsumeFromWhere.CONSUME_FROM_TIMESTAMP);
consumer.setConsumeTimestamp("20240101000000");
```

## 二、应用场景

```java
// 1. 数据修复 - 消费逻辑有 bug，修复后重新消费
// 2. 重算指标 - 某段时间的数据需要重新计算
// 3. 数据对账 - 对比历史数据
// 4. 测试回放 - 用历史数据测试新版本消费逻辑
```

## 三、Kafka 回溯的多种方式

```bash
# 按 Offset 回溯到指定位置
kafka-consumer-groups --bootstrap-server localhost:9092 \
  --group my-group --topic order-events \
  --reset-offsets --to-offset 100000 --execute

# 回溯到最早位置
kafka-consumer-groups --bootstrap-server localhost:9092 \
  --group my-group --topic order-events \
  --reset-offsets --to-earliest --execute

# 回溯到最新位置（跳过历史）
kafka-consumer-groups --bootstrap-server localhost:9092 \
  --group my-group --topic order-events \
  --reset-offsets --to-latest --execute

# 回溯 N 条消息
kafka-consumer-groups --bootstrap-server localhost:9092 \
  --group my-group --topic order-events \
  --reset-offsets --shift-by -500 --execute
```

```java
// Kafka Java API 回溯
consumer.seekToBeginning(consumer.assignment()); // 回到最早

// 按时间戳回溯
Map<TopicPartition, Long> timestamps = new HashMap<>();
for (TopicPartition tp : consumer.assignment()) {
    timestamps.put(tp, targetTimestamp);
}
Map<TopicPartition, OffsetAndTimestamp> offsets = consumer.offsetsForTimes(timestamps);
for (Map.Entry<TopicPartition, OffsetAndTimestamp> entry : offsets.entrySet()) {
    if (entry.getValue() != null) {
        consumer.seek(entry.getKey(), entry.getValue().offset());
    }
}
```

## 四、RocketMQ 消息回溯

```java
// RocketMQ 按时间戳回溯
DefaultMQPushConsumer consumer = new DefaultMQPushConsumer("OrderGroup");
consumer.setConsumeFromWhere(ConsumeFromWhere.CONSUME_FROM_TIMESTAMP);
consumer.setConsumeTimestamp("20240101000000");

// 通过管理命令回溯
// mqadmin resetOffsetByTime -n localhost:9876 -g OrderGroup \
//   -t OrderTopic -s 20240101000000

// RocketMQ 按消费进度重置
// 重置到指定 Queue 的指定时间点
DefaultMQAdminExt admin = new DefaultMQAdminExt();
admin.start();
admin.resetOffsetByTimestamp(
    "OrderGroup", "OrderTopic",
    System.currentTimeMillis() - 3600000  // 1 小时前
);
```

## 五、RabbitMQ 回溯方案

RabbitMQ 不原生支持消息回溯，但可通过以下方案实现：

```java
// 方案一：延迟队列 + 死信路由
// 将消息备份到归档队列，需要回溯时重新消费

// 方案二：消息持久化到数据库后回溯
@RabbitListener(queues = "order-queue")
public void process(Message message) {
    // 处理前先持久化
    messageArchiveMapper.insert(new ArchivedMessage(message));
    doProcess(message);
}

// 回溯：从数据库重新投递
public void replay(String startTime, String endTime) {
    List<ArchivedMessage> messages =
        messageArchiveMapper.selectByTimeRange(startTime, endTime);
    for (ArchivedMessage msg : messages) {
        rabbitTemplate.convertAndSend(msg.getExchange(), msg.getRoutingKey(), msg.getBody());
    }
}
```

## 六、回溯实战注意事项

1. **回溯前确保消费者组已停止**
2. **回溯后要注意幂等性**，避免重复处理产生副作用
3. **Kafka 支持按时间和 Offset 回溯**，RocketMQ 支持按时间
4. **回溯范围不要太大**，避免影响实时消费
5. **建议在测试环境先验证回溯逻辑**
6. **回溯操作要记录审计日志**，包括操作人、时间、回溯范围
7. **大批量回溯建议分批执行**，避免对集群造成压力
8. **回溯后监控消费延迟和处理速度**，确认回溯完成
