# 消息回溯

## 一、什么是消息回溯

消息回溯是指**将消费位点重置到历史位置，重新消费已处理过的消息**。适用于数据修复、重算等场景。

```java
// Kafka 按时间回溯
kafka-consumer-groups --bootstrap-server localhost:9092 \
  --group my-group --topic order-events \
  --reset-offsets --to-datetime 2024-01-01T00:00:00.000 --execute

// RocketMQ 按时间回溯
DefaultMQPushConsumer consumer = new DefaultMQPushConsumer("Group");
consumer.setConsumeFromWhere(ConsumeFromWhere.CONSUME_FROM_TIMESTAMP);
consumer.setConsumeTimestamp("20240101000000");
```

## 二、应用场景

```java
// 1. 数据修复 - 消费逻辑有 bug，修复后重新消费
// 2. 重算指标 - 某段时间的数据需要重新计算
// 3. 数据对账 - 对比历史数据
// 4. 测试回放 - 用历史数据测试新版本消费逻辑
```

## 三、注意事项

1. **回溯前确保消费者组已停止**
2. **回溯后要注意幂等性**，避免重复处理产生副作用
3. **Kafka 支持按时间和 Offset 回溯**，RocketMQ 支持按时间
4. **回溯范围不要太大**，避免影响实时消费
5. **建议在测试环境先验证回溯逻辑**
