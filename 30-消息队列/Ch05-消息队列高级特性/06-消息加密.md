# 消息加密

## 一、传输加密

```java
// Kafka SSL 配置
props.put("security.protocol", "SSL");
props.put("ssl.truststore.location", "/path/truststore.jks");
props.put("ssl.truststore.password", "password");
props.put("ssl.keystore.location", "/path/keystore.jks");
props.put("ssl.keystore.password", "password");

// RabbitMQ TLS 配置
ConnectionFactory factory = new ConnectionFactory();
factory.useSslProtocol("TLSv1.3");
factory.setSslContextFactory(() -> {
    SSLContext ctx = SSLContext.getInstance("TLSv1.3");
    ctx.init(null, trustAllCerts, new SecureRandom());
    return ctx;
});
```

## 二、消息体加密

```java
// 应用层加密
public class EncryptedProducer {
    @Autowired
    private KafkaTemplate<String, String> kafkaTemplate;

    public void sendEncrypted(String topic, String key, Object data) {
        String json = JSON.toJSONString(data);
        String encrypted = AESUtil.encrypt(json, secretKey);
        kafkaTemplate.send(topic, key, encrypted);
    }
}

public class EncryptedConsumer {
    @KafkaListener(topics = "encrypted-topic")
    public void consume(ConsumerRecord<String, String> record) {
        String decrypted = AESUtil.decrypt(record.value(), secretKey);
        Order order = JSON.parseObject(decrypted, Order.class);
        process(order);
    }
}
```

## 三、密钥管理与 KMS 集成

```java
// 使用 AWS KMS 管理密钥
public class KmsEncryptionService {
    private final AWSKMS kmsClient;

    public byte[] encryptWithKms(byte[] plaintext) {
        // 通过 KMS 获取数据密钥
        GenerateDataKeyRequest request = new GenerateDataKeyRequest()
            .withKeyId("alias/my-key")
            .withKeySpec("AES_256");

        GenerateDataKeyResult result = kmsClient.generateDataKey(request);
        byte[] plaintextKey = result.getPlaintext().array();

        // 用数据密钥加密消息
        byte[] ciphertext = AESUtil.encrypt(plaintext, plaintextKey);

        // 将加密后的密钥附加到消息头
        // 消费端通过 KMS 解密密钥，再解密消息
        return ciphertext;
    }
}
```

## 四、Kafka SASL 认证配置

```properties
# Producer 端配置
security.protocol=SASL_SSL
sasl.mechanism=SCRAM-SHA-256
sasl.jaas.config=org.apache.kafka.common.security.scram.ScramLoginModule required \
  username="producer-user" password="producer-password";

# Consumer 端配置
security.protocol=SASL_SSL
sasl.mechanism=PLAIN
sasl.jaas.config=org.apache.kafka.common.security.plain.PlainLoginModule required \
  username="consumer-user" password="consumer-password";
```

## 五、加密性能对比

```
算法        加密速度(MB/s)  密钥长度   安全等级   适用场景
AES-128      500+          128 bit    高        通用场景（推荐）
AES-256      350+          256 bit    很高      金融/医疗等强安全场景
RSA-2048     50            2048 bit   很高      密钥交换、数字签名
ChaCha20     400+          256 bit    很高      移动端、CPU 无 AES-NI 时
```

## 六、注意事项

1. **传输加密用 SSL/TLS**，消息体加密用 AES/RSA
2. **密钥管理要用专业的密钥服务（KMS）**
3. **加密会增加 CPU 开销**，大消息影响更明显
4. **Kafka 支持 SASL 认证 + SSL 加密**
5. **RabbitMQ 支持 TLS + OAuth2 认证**
6. **定期轮换加密密钥**，降低密钥泄露风险
7. **敏感字段单独加密**，如身份证号、银行卡号
8. **证书管理要自动化**，使用 Let's Encrypt 或内部 CA
