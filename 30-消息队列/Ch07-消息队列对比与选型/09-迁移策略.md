# MQ 迁移策略

## 一、迁移步骤

```
1. 双写 - 新旧 MQ 同时写入
2. 双读 - 消费者同时消费新旧 MQ
3. 切换读 - 消费者只消费新 MQ
4. 切换写 - 生产者只写新 MQ
5. 下线旧 MQ
```

```java
// 双写阶段
public void sendOrder(Order order) {
    // 写入旧 MQ
    rabbitTemplate.convertAndSend("old-exchange", "old-key", order);
    // 写入新 MQ
    kafkaTemplate.send("new-topic", JSON.toJSONString(order));
}

// 数据校验
@Scheduled(fixedDelay = 60000)
public void verifyConsistency() {
    long oldCount = rabbitAdmin.getQueueMessageCount("old-queue");
    long newCount = getKafkaLag("new-topic");
    if (Math.abs(oldCount - newCount) > 100) {
        alertService.alert("数据不一致");
    }
}
```

## 二、注意事项

1. **双写期间要做好数据校验**
2. **消费端要兼容新旧两种格式**
3. **切换过程要有回滚方案**
4. **分批迁移，不要一次性全切**
5. **保留旧 MQ 数据一段时间用于对账**
