# 云消息服务对比

## 一、AWS 消息服务

| 服务 | 特点 | 适用场景 |
|------|------|---------|
| SQS | 标准/FIFO 队列 | 任务队列、解耦 |
| SNS | 发布订阅 | 通知、广播 |
| Kinesis | 实时流 | 日志、流处理 |
| EventBridge | 事件总线 | 微服务事件 |

## 二、阿里云消息服务

| 服务 | 特点 | 适用场景 |
|------|------|---------|
| 消息队列 RocketMQ 版 | 全托管 RocketMQ | 电商、金融 |
| 消息队列 Kafka 版 | 全托管 Kafka | 日志、大数据 |
| 微消息队列 MQTT 版 | IoT 消息 | 设备通信 |
| MNS | 轻量级队列 | 任务通知 |

## 三、Google Cloud 消息服务

| 服务 | 特点 | 适用场景 |
|------|------|---------|
| Pub/Sub | 全球分布、自动扩展 | 事件流、IoT |
| Cloud Tasks | 定时/延迟任务 | 异步任务调度 |
| Cloud Scheduler | Cron式定时任务 | 定时批处理 |

## 四、选型建议

1. **AWS 生态用 SQS/SNS**，简单可靠，两者配合实现解耦+广播
2. **阿里云用 RocketMQ 版**，功能全、国内支持好，事务消息是亮点
3. **全托管省去运维**，但有供应商锁定风险，跨云迁移成本高
4. **按量付费适合流量波动大的场景**，稳定高流量可考虑预留实例降低成本
5. **跨云同步用 Kafka**，标准协议兼容性好，各云厂商均有托管版本

## 五、腾讯云与华为云消息服务

```
腾讯云:
  ├── TDMQ (Pulsar 版)      多租户、存算分离
  ├── CMQ (轻量级队列)        简单队列

  └── 事件总线 EventBridge    事件驱动

华为云:
  ├── DMS for Kafka          全托管 Kafka
  ├── DMS for RabbitMQ       全托管 RabbitMQ
  ├── DMS for RocketMQ       全托管 RocketMQ
  └── 事件网格 EG             事件驱动
```

## 六、成本对比分析

```
服务              计费方式              典型月成本 (100万消息/天)
AWS SQS          按请求数              ~$0.40 / 1M requests
AWS SNS          按请求数              ~$0.50 / 1M publishes
阿里云 RocketMQ   按实例规格            ~200元/月 (标准版)
阿里云 Kafka      按实例规格 + 流量     ~500元/月 (标准版)
Google Pub/Sub   按数据量              ~$0.04 / GB
```

## 七、常见陷阱

1. **低估成本**：云消息服务按请求数计费，高频小消息可能导致费用远超预期
2. **延迟差异**：SQS标准队列不保证顺序，FIFO队列有吞吐限制（300 msg/s），选型时需权衡
3. **供应商锁定**：使用SNS特有的过滤策略后难以迁移到其他平台，核心逻辑应与消息中间件解耦
4. **配额限制**：各服务默认有请求配额限制，大促前需提前申请提升
5. **冷启动延迟**：Serverless 消息服务在流量低谷后首次请求有延迟
6. **跨区域费用**：跨区域消息传输有额外费用，架构设计时要考虑
