# 云消息服务对比

## 一、AWS 消息服务

| 服务 | 特点 | 适用场景 |
|------|------|---------|
| SQS | 标准/FIFO 队列 | 任务队列、解耦 |
| SNS | 发布订阅 | 通知、广播 |
| Kinesis | 实时流 | 日志、流处理 |
| EventBridge | 事件总线 | 微服务事件 |

## 二、阿里云消息服务

| 服务 | 特点 | 适用场景 |
|------|------|---------|
| 消息队列 RocketMQ 版 | 全托管 RocketMQ | 电商、金融 |
| 消息队列 Kafka 版 | 全托管 Kafka | 日志、大数据 |
| 微消息队列 MQTT 版 | IoT 消息 | 设备通信 |
| MNS | 轻量级队列 | 任务通知 |

## 三、选型建议

1. **AWS 生态用 SQS/SNS**，简单可靠
2. **阿里云用 RocketMQ 版**，功能全、国内支持好
3. **全托管省去运维**，但有供应商锁定风险
4. **按量付费适合流量波动大的场景**
5. **跨云同步用 Kafka**，标准协议兼容性好
