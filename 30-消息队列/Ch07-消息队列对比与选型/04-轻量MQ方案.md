# 轻量 MQ 方案

## 一、Redis Stream

```java
// Redis Stream - 轻量级消息队列
@Autowired
private StringRedisTemplate redisTemplate;

// 发布
public void publish(String stream, String message) {
    redisTemplate.opsForStream().add(stream, Collections.singletonMap("data", message));
}

// 消费（消费组模式）
public void consume(String stream, String group, String consumer) {
    while (true) {
        List<MapRecord<String, Object, Object>> records =
            redisTemplate.opsForStream().read(
                Consumer.from(group, consumer),
                StreamReadOptions.empty().count(10),
                StreamOffset.create(stream, ReadOffset.lastConsumed())
            );
        for (MapRecord<String, Object, Object> record : records) {
            process(record);
            redisTemplate.opsForStream().acknowledge(stream, group, record.getId());
        }
    }
}
```

## 二、Redis List

```java
// 简单队列 - LPUSH/BRPOP
redisTemplate.opsForList().leftPush("queue", message);
String msg = redisTemplate.opsForList().rightPop("queue", 10, TimeUnit.SECONDS);
```

## 三、注意事项

1. **Redis Stream 支持消费者组**，适合轻量级场景
2. **Redis 不保证持久化可靠性**，重启可能丢数据
3. **Redis List 简单但不支持消费者组**
4. **适合小规模、低延迟场景**
5. **大规模场景还是用专业 MQ**
