# 轻量 MQ 方案

## 一、Redis Stream

```java
// Redis Stream - 轻量级消息队列
@Autowired
private StringRedisTemplate redisTemplate;

// 发布
public void publish(String stream, String message) {
    redisTemplate.opsForStream().add(stream, Collections.singletonMap("data", message));
}

// 消费（消费组模式）
public void consume(String stream, String group, String consumer) {
    while (true) {
        List<MapRecord<String, Object, Object>> records =
            redisTemplate.opsForStream().read(
                Consumer.from(group, consumer),
                StreamReadOptions.empty().count(10),
                StreamOffset.create(stream, ReadOffset.lastConsumed())
            );
        for (MapRecord<String, Object, Object> record : records) {
            process(record);
            redisTemplate.opsForStream().acknowledge(stream, group, record.getId());
        }
    }
}
```

## 二、Redis List

```java
// 简单队列 - LPUSH/BRPOP
redisTemplate.opsForList().leftPush("queue", message);
String msg = redisTemplate.opsForList().rightPop("queue", 10, TimeUnit.SECONDS);
```

## 三、Redis Stream 消费者组详解

```java
// 创建消费者组
redisTemplate.opsForStream().createGroup("orders-stream", "order-group");

// 消费者读取未消费消息
public void consumeNew() {
    List<MapRecord<String, Object, Object>> records =
        redisTemplate.opsForStream().read(
            Consumer.from("order-group", "consumer-1"),
            StreamReadOptions.empty().count(50).block(Duration.ofSeconds(2)),
            StreamOffset.create("orders-stream", ReadOffset.lastConsumed())
        );
    for (MapRecord<String, Object, Object> record : records) {
        process(record);
        redisTemplate.opsForStream().acknowledge("orders-stream", "order-group", record.getId());
    }
}

// 处理未确认消息（故障恢复）
public void consumePending() {
    List<MapRecord<String, Object, Object>> pending =
        redisTemplate.opsForStream().pending(
            "orders-stream", "order-group",
            Range.unbounded(), 50
        );
    for (MapRecord<String, Object, Object> record : pending) {
        process(record);
        redisTemplate.opsForStream().acknowledge("orders-stream", "order-group", record.getId());
    }
}
```

## 四、其他轻量方案对比

```
方案            持久化    消费者组    有序    适用场景
Redis Stream    AOF      支持       是      轻量级任务队列
Redis List      AOF      不支持     是      简单生产消费
ZeroMQ          无       不支持     是      进程间通信
NATS Core       可选     不支持     是      微服务消息
SQLite          文件      不支持     是      嵌入式应用
```

## 五、ZeroMQ 简单示例

```java
// ZeroMQ 推拉模式
// Producer
ZMQ.Context context = ZMQ.context(1);
ZMQ.Socket sender = context.socket(ZMQ.PUSH);
sender.bind("tcp://*:5555");
sender.send("Hello".getBytes());

// Consumer
ZMQ.Socket receiver = context.socket(ZMQ.PULL);
receiver.connect("tcp://localhost:5555");
byte[] message = receiver.recv();
```

## 六、注意事项

1. **Redis Stream 支持消费者组**，适合轻量级场景
2. **Redis 不保证持久化可靠性**，重启可能丢数据
3. **Redis List 简单但不支持消费者组**
4. **适合小规模、低延迟场景**
5. **大规模场景还是用专业 MQ**
6. **Redis Stream 要设置 MAXLEN**，避免内存无限增长
7. **ZeroMQ 无 Broker**，适合高性能、低延迟的进程间通信
