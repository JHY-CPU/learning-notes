# 选型决策树

## 一、决策流程

```
                    ┌─ 需要事务消息？ ── 是 ──→ RocketMQ
                    │
                    ├─ 超高吞吐（百万级）？ ── 是 ──→ Kafka
                    │
分析需求 ──────────┼─ 复杂路由？ ── 是 ──→ RabbitMQ
                    │
                    ├─ IoT/极低延迟？ ── 是 ──→ NATS
                    │
                    ├─ 多租户隔离？ ── 是 ──→ Pulsar
                    │
                    └─ 简单队列/小团队？ ── 是 ──→ RabbitMQ / Redis Stream
```

## 二、场景速查

| 场景 | 推荐 MQ |
|------|---------|
| 订单/支付 | RocketMQ |
| 日志收集 | Kafka |
| 实时推荐 | Kafka + Flink |
| 微服务通知 | RabbitMQ |
| 秒杀削峰 | RocketMQ/Kafka |
| IoT 设备 | NATS / EMQX |
| 简单任务队列 | RabbitMQ / Redis |
| 跨集群同步 | Kafka MirrorMaker |

## 三、注意事项

1. **不要过度选型**，小团队直接用 RabbitMQ
2. **考虑 3-5 年演进**，避免大规模迁移
3. **混合使用是常见做法**
4. **云服务省运维但有锁定风险**
5. **性能不是唯一标准**，运维成本和生态同样重要
