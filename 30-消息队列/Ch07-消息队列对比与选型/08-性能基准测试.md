# 性能基准测试

## 一、测试工具

```bash
# Kafka 性能测试
kafka-producer-perf-test.sh --topic perf-test \
  --num-records 10000000 --record-size 1024 \
  --throughput -1 --producer-props bootstrap.servers=localhost:9092

# RabbitMQ 性能测试
bin/runjava com.rabbitmq.perf.PerfTest \
  -h localhost -x 10 -y 10 -s 1024 -r 10000

# RocketMQ 性能测试
tools/benchmark/runProducerBenchmark.sh -n localhost:9876 -t perf-test -s 1024
```

## 三、测试方法论

```bash
# Kafka 完整性能测试
# 1. 吞吐量测试
kafka-producer-perf-test.sh --topic perf-test \
  --num-records 10000000 --record-size 1024 \
  --throughput -1 --producer-props bootstrap.servers=localhost:9092 \
    acks=1 batch.size=65536 linger.ms=20 compression.type=snappy

# 2. 延迟测试
kafka-producer-perf-test.sh --topic perf-test \
  --num-records 100000 --record-size 256 \
  --throughput 10000 --producer-props bootstrap.servers=localhost:9092 acks=all

# 3. 消费者吞吐测试
kafka-consumer-perf-test.sh --topic perf-test \
  --broker-list localhost:9092 --messages 1000000 \
  --threads 4 --group perf-group

# RocketMQ 完整性能测试
tools/benchmark/runProducerBenchmark.sh -n localhost:9876 \
  -t perf-test -s 1024 -c 64 -a 4

# RabbitMQ 完整性能测试
bin/runjava com.rabbitmq.perf.PerfTest \
  -h localhost -x 10 -y 10 -s 1024 -r 10000 -u perf-test \
  --flag persistent --auto-delete false
```

## 四、影响性能的关键因素

```
因素                  影响程度    说明
消息大小              高         1KB vs 100KB 差异巨大
分区/队列数           高         过多分区增加开销
副本数                高         3副本比1副本吞吐降低 40-60%
压缩算法              中         zstd 比 gzip 快 30%
acks 模式             中         acks=all 比 acks=1 慢 50%
批量大小              高         大批量提升吞吐
磁盘类型              高         SSD 比 HDD 快 5-10 倍
网络带宽              高         千兆 vs 万兆差异明显
```

## 六、注意事项

1. **测试要基于真实业务场景**，不要只看 benchmark
2. **消息大小、分区数、副本数都会影响结果**
3. **网络和磁盘是最大瓶颈**
4. **并发数和批量大小要合理设置**
5. **测试结果要包含延迟分布**，不要只看平均值
6. **多次测试取平均值**，排除偶然因素
7. **测试结果要与业务 SLA 对比**，确认满足要求
