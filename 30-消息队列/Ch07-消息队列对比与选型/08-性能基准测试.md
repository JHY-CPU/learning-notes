# 性能基准测试

## 一、测试工具

```bash
# Kafka 性能测试
kafka-producer-perf-test.sh --topic perf-test \
  --num-records 10000000 --record-size 1024 \
  --throughput -1 --producer-props bootstrap.servers=localhost:9092

# RabbitMQ 性能测试
bin/runjava com.rabbitmq.perf.PerfTest \
  -h localhost -x 10 -y 10 -s 1024 -r 10000

# RocketMQ 性能测试
tools/benchmark/runProducerBenchmark.sh -n localhost:9876 -t perf-test -s 1024
```

## 二、典型性能数据

```yaml
1KB 消息，3 副本:
  Kafka:    写入 800MB/s, 读取 1.2GB/s
  RocketMQ: 写入 300MB/s, 读取 500MB/s
  RabbitMQ: 写入 50MB/s,  读取 80MB/s

延迟 (P99):
  RabbitMQ: < 1ms
  RocketMQ: < 5ms
  Kafka:    < 10ms
```

## 三、注意事项

1. **测试要基于真实业务场景**，不要只看 benchmark
2. **消息大小、分区数、副本数都会影响结果**
3. **网络和磁盘是最大瓶颈**
4. **并发数和批量大小要合理设置**
5. **测试结果要包含延迟分布**，不要只看平均值
