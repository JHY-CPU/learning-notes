# 持久化配置

## 一、持久化的三个层次

RabbitMQ 的持久化保证消息在 Broker 重启后不丢失，涉及三个层面：

```
1. Exchange 持久化 - 交换机定义持久化
2. Queue 持久化   - 队列定义持久化
3. Message 持久化 - 消息内容持久化到磁盘
```

## 二、Exchange 持久化

```java
// durable=true 表示交换机持久化
@Bean
public DirectExchange orderExchange() {
    // 参数: name, durable, autoDelete
    return new DirectExchange("order-exchange", true, false);
}

// 原生 API
channel.exchangeDeclare("order-exchange", "direct", true);
//                                              ^^^^
//                                          durable=true
```

## 三、Queue 持久化

```java
@Bean
public Queue orderQueue() {
    return QueueBuilder.durable("order-queue")   // durable 队列
        .withArgument("x-queue-type", "quorum")   // Quorum Queue 持久化更好
        .build();
}

// 非持久化队列 - Broker 重启后队列丢失
@Bean
public Queue tempQueue() {
    return QueueBuilder.nonDurable("temp-queue").build();
}

// 原生 API
channel.queueDeclare("order-queue",  // 队列名
    true,                             // durable
    false,                            // exclusive
    false,                            // autoDelete
    null);                            // arguments
```

## 四、Message 持久化

```java
// 设置消息持久化
public void sendPersistentMessage(Channel channel, byte[] body) throws IOException {
    AMQP.BasicProperties props = new AMQP.BasicProperties.Builder()
        .deliveryMode(2)             // 2 = 持久化，1 = 非持久化
        .contentType("application/json")
        .build();

    channel.basicPublish("order-exchange", "order.created", props, body);
}

// Spring AMQP 方式
public void sendPersistentMessage(Order order) {
    MessageProperties props = new MessageProperties();
    props.setDeliveryMode(MessageDeliveryMode.PERSISTENT);  // 持久化
    props.setContentType("application/json");

    Message message = new Message(
        JSON.toJSONString(order).getBytes(),
        props
    );
    rabbitTemplate.send("order-exchange", "order.created", message);
}
```

## 五、持久化配置完整示例

```java
@Configuration
public class PersistentMQConfig {

    @Bean
    public DirectExchange persistentExchange() {
        return new DirectExchange("persistent-exchange", true, false);
    }

    @Bean
    public Queue persistentQueue() {
        return QueueBuilder.durable("persistent-queue")
            .quorum()
            .withArgument("x-quorum-initial-group-size", 3)
            .build();
    }

    @Bean
    public Binding persistentBinding() {
        return BindingBuilder.bind(persistentQueue())
            .to(persistentExchange())
            .with("persistent-key");
    }

    @Bean
    public RabbitTemplate persistentTemplate(ConnectionFactory factory) {
        RabbitTemplate template = new RabbitTemplate(factory);
        // 设置默认消息持久化
        template.setBeforePublishPostProcessors(message -> {
            message.getMessageProperties()
                .setDeliveryMode(MessageDeliveryMode.PERSISTENT);
            return message;
        });
        return template;
    }
}
```

## 六、持久化与性能

```
持久化级别          可靠性      性能        适用场景
-------------------------------------------------
全部非持久化       低          最高        临时通知、日志
队列持久化         中          高          一般业务消息
队列+消息持久化    高          中          重要业务消息
Quorum Queue      最高        较低        金融/交易类消息
```

## 七、注意事项

1. **Exchange 和 Queue 名字相同但 durable 属性不同会报错**，确保声明一致
2. **持久化不能保证 100% 不丢**，消息写入 page cache 后可能还没落盘就宕机
3. **Quorum Queue 的持久化比镜像队列更可靠**，推荐生产使用
4. **大量持久化消息会占用磁盘 I/O**，必要时调整 `vm_memory_high_watermark_paging_ratio`
5. **临时队列不要持久化**，exclusive + autoDelete 的临时队列用 durable=false
