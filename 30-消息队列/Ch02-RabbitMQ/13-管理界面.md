# 管理界面与 API

## 一、Web 管理界面

安装后访问 `http://localhost:15672`，默认用户名密码 `guest/guest`。

```
管理界面主要模块:
├── Overview      - 服务器概览、消息速率、连接数
├── Connections   - TCP 连接管理
├── Channels      - 信道管理、消费速率
├── Exchanges     - 交换机列表、绑定关系
├── Queues        - 队列列表、消息数量、消费速率
├── Admin         - 用户管理、vhost、权限
└── Streams       - Stream 队列管理
```

## 二、HTTP Management API

```bash
# 获取服务器概览
curl -u admin:admin123 http://localhost:15672/api/overview | python -m json.tool

# 获取所有队列
curl -u admin:admin123 http://localhost:15672/api/queues/%2F | python -m json.tool

# 获取指定队列详情
curl -u admin:admin123 http://localhost:15672/api/queues/%2F/order-queue | python -m json.tool

# 创建队列
curl -u admin:admin123 -X PUT http://localhost:15672/api/queues/%2F/new-queue \
  -H "Content-Type: application/json" \
  -d '{"durable": true, "arguments": {"x-queue-type": "quorum"}}'

# 删除队列
curl -u admin:admin123 -X DELETE http://localhost:15672/api/queues/%2F/old-queue

# 清空队列
curl -u admin:admin123 -X DELETE http://localhost:15672/api/queues/%2F/order-queue/contents

# 获取消费者
curl -u admin:admin123 http://localhost:15672/api/consumers/%2F | python -m json.tool
```

## 三、rabbitmqctl 命令行工具

```bash
# 服务器状态
rabbitmqctl status

# 节点信息
rabbitmqctl cluster_status

# 列出队列
rabbitmqctl list_queues name messages consumers state

# 列出交换机
rabbitmqctl list_exchanges name type durable

# 列出绑定
rabbitmqctl list_bindings source_name destination_name routing_key

# 用户管理
rabbitmqctl add_user myuser mypassword
rabbitmqctl set_user_tags myuser administrator
rabbitmqctl set_permissions -p / myuser ".*" ".*" ".*"
rabbitmqctl delete_user guest

# vhost 管理
rabbitmqctl add_vhost /my-vhost
rabbitmqctl delete_vhost /my-vhost

# 队列操作
rabbitmqctl purge_queue order-queue
rabbitmqctl delete_queue order-queue

# 策略管理
rabbitmqctl set_policy ha-all '{"ha-mode":"all"}' --apply-to queues
rabbitmqctl clear_policy ha-all
```

## 四、监控指标

```bash
# 关键监控 API
curl -u admin:admin123 http://localhost:15672/api/nodes | python -m json.tool

# 节点内存使用
curl -u admin:admin123 http://localhost:15672/api/nodes/rabbit@hostname/memory | python -m json.tool

# 消息速率
curl -u admin:admin123 "http://localhost:15672/api/queues/%2F/order-queue" | \
  python -c "import sys,json; d=json.load(sys.stdin); print('publish:', d['message_stats']['publish_details']['rate'])"
```

```yaml
# 启用 rabbitmq_prometheus 插件
rabbitmq-plugins enable rabbitmq_prometheus

# 指标端点
# http://localhost:15692/metrics
```

## 五、常见操作脚本

```bash
#!/bin/bash
# rabbitmq-ops.sh - 常用运维脚本

# 批量删除死信队列
for queue in $(curl -s -u admin:admin123 http://localhost:15672/api/queues/%2F | \
  python -c "import sys,json; [print(q['name']) for q in json.load(sys.stdin) if 'dlq' in q['name']]"); do
  curl -u admin:admin123 -X DELETE "http://localhost:15672/api/queues/%2F/$queue"
  echo "Deleted: $queue"
done

# 检查消息堆积
for queue in $(curl -s -u admin:admin123 http://localhost:15672/api/queues/%2F | \
  python -c "import sys,json; [print(q['name'],q['messages']) for q in json.load(sys.stdin) if q['messages']>10000]"); do
  echo "WARNING: $queue"
done
```

## 六、注意事项

1. **`%2F` 是 `/` 的 URL 编码**，代表默认 vhost
2. **管理 API 有性能开销**，监控频率不要太高（建议 30 秒以上）
3. **生产环境限制 API 访问权限**，使用只读用户做监控
4. **`rabbitmqctl` 在 Docker 中需要 `docker exec` 执行**
5. **管理界面暴露到公网时务必修改默认密码**，并考虑 HTTPS
