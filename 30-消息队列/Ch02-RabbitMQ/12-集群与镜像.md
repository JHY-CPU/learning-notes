# 集群与镜像

## 一、RabbitMQ 集群架构

RabbitMQ 集群通过 Erlang 分布式协议实现，节点之间共享用户、vhost、Exchange 等元数据，但**队列默认只存在于声明它的节点上**。

```
集群节点:
  Node1 (Disc) ─── Node2 (RAM) ─── Node3 (RAM)
    │                 │                │
    Queue-A          Queue-B          Queue-C
    (数据节点)        (内存节点)        (内存节点)
```

## 二、集群搭建（Docker）

```yaml
# docker-compose-cluster.yml
version: '3.8'
services:
  rabbitmq1:
    image: rabbitmq:3.13-management
    hostname: rabbitmq1
    container_name: rabbitmq1
    environment:
      RABBITMQ_ERLANG_COOKIE: "cluster_cookie_secret"
      RABBITMQ_NODENAME: rabbit@rabbitmq1
    ports:
      - "5672:5672"
      - "15672:15672"
    networks:
      - rabbitmq-cluster

  rabbitmq2:
    image: rabbitmq:3.13-management
    hostname: rabbitmq2
    container_name: rabbitmq2
    environment:
      RABBITMQ_ERLANG_COOKIE: "cluster_cookie_secret"
      RABBITMQ_NODENAME: rabbit@rabbitmq2
    ports:
      - "5673:5672"
      - "15673:15672"
    networks:
      - rabbitmq-cluster

  rabbitmq3:
    image: rabbitmq:3.13-management
    hostname: rabbitmq3
    container_name: rabbitmq3
    environment:
      RABBITMQ_ERLANG_COOKIE: "cluster_cookie_secret"
      RABBITMQ_NODENAME: rabbit@rabbitmq3
    ports:
      - "5674:5672"
      - "15674:15672"
    networks:
      - rabbitmq-cluster

networks:
  rabbitmq-cluster:
    driver: bridge
```

```bash
# 加入集群
docker exec -it rabbitmq2 rabbitmqctl stop_app
docker exec -it rabbitmq2 rabbitmqctl join_cluster rabbit@rabbitmq1
docker exec -it rabbitmq2 rabbitmqctl start_app

# 验证集群状态
docker exec -it rabbitmq1 rabbitmqctl cluster_status
```

## 三、Quorum Queue（推荐）

Quorum Queue 基于 Raft 协议，提供**强一致性**和**自动故障恢复**，取代传统镜像队列。

```java
// 声明 Quorum Queue
@Bean
public Queue quorumQueue() {
    return QueueBuilder.durable("quorum-queue")
        .quorum()
        .withArgument("x-quorum-initial-group-size", 3)   // 3 节点副本
        .withArgument("x-delivery-limit", 10)             // 投递次数限制
        .withArgument("x-dead-letter-exchange", "dlx-exchange")
        .build();
}

// 原生 API
Map<String, Object> args = new HashMap<>();
args.put("x-queue-type", "quorum");
args.put("x-quorum-initial-group-size", 3);
channel.queueDeclare("quorum-queue", true, false, false, args);
```

### Quorum Queue 优势

```
特性               镜像队列          Quorum Queue
-------------------------------------------------
一致性协议         主从复制          Raft
自动故障恢复       需要配置          内置支持
消息持久化         可靠              非常可靠
内存使用           较高              更优
网络分区处理       需手动配置        自动处理
消息回溯           不支持            支持（3.12+）
```

## 四、Federation 与 Shovel

跨集群消息同步的两种方案：

```bash
# Federation - 单向消息流
rabbitmq-plugins enable rabbitmq_federation
rabbitmq-plugins enable rabbitmq_federation_management

# 创建上游
rabbitmqctl set_parameter federation-upstream my-upstream \
    '{"uri":"amqp://admin:admin123@remote-host","ack-mode":"on-confirm"}'

# 创建策略
rabbitmqctl set_parameter federation-policy my-policy \
    '{"pattern":"^federated\\.","upstream-set":"all"}'
```

```bash
# Shovel - 消息搬运工（跨集群、跨协议）
rabbitmq-plugins enable rabbitmq_shovel
rabbitmq-plugins enable rabbitmq_shovel_management

# 配置 Shovel
rabbitmqctl set_parameter shovel my-shovel \
    '{"src-protocol":"amqp091","src-uri":"amqp://source-host",
      "src-queue":"source-queue",
      "dest-protocol":"amqp091","dest-uri":"amqp://dest-host",
      "dest-queue":"dest-queue"}'
```

## 五、网络分区处理

```bash
# 配置分区处理策略（rabbitmq.conf）
# pause_minority - 暂停少数派节点（最安全）
cluster_partition_handling = pause_minority

# autoheal - 自动恢复（推荐）
cluster_partition_handling = autoheal

# ignore - 忽略分区（可能导致数据不一致）
cluster_partition_handling = ignore
```

## 六、注意事项

1. **节点间 Erlang Cookie 必须一致**，否则无法组成集群
2. **所有节点需要能互相解析主机名**，配置 DNS 或 /etc/hosts
3. **生产环境推荐 Quorum Queue**，比镜像队列更可靠
4. **Disc 节点至少 1 个**，否则元数据会丢失
5. **网络分区策略选择 `pause_minority` 最安全**，`autoheal` 最方便
