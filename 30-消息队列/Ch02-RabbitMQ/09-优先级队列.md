# 优先级队列

## 一、优先级队列概述

RabbitMQ 支持消息优先级，**优先级高的消息先被消费**。队列需要声明为优先级队列，消息设置优先级属性。

```
普通队列: FIFO (先进先出)
优先级队列: 高优先级消息插队消费
```

## 二、配置优先级队列

```java
@Configuration
public class PriorityQueueConfig {

    @Bean
    public Queue priorityQueue() {
        return QueueBuilder.durable("priority-queue")
            .withArgument("x-max-priority", 10)  // 最大优先级 0-10
            .build();
    }
}
```

```yaml
# 队列参数说明
x-max-priority: 10   # 值越小越推荐，太大增加排序开销
                      # 推荐范围: 1-10
```

## 三、发送优先级消息

```java
@Service
public class PriorityProducer {

    @Autowired
    private RabbitTemplate rabbitTemplate;

    public void sendWithPriority(Object payload, int priority) {
        rabbitTemplate.convertAndSend(
            "priority-exchange",
            "priority-key",
            JSON.toJSONString(payload),
            message -> {
                // 设置消息优先级 (0-10)
                message.getMessageProperties().setPriority(priority);
                return message;
            }
        );
    }

    // 紧急消息 - 最高优先级
    public void sendUrgent(Notification notification) {
        sendWithPriority(notification, 10);
    }

    // 普通消息 - 默认优先级
    public void sendNormal(Notification notification) {
        sendWithPriority(notification, 0);
    }

    // 低优先级消息
    public void sendLow(Notification notification) {
        sendWithPriority(notification, 1);
    }
}

// 消费者 - 优先级高的先被消费
@Component
public class PriorityConsumer {

    @RabbitListener(queues = "priority-queue")
    public void consume(Message message) {
        int priority = message.getMessageProperties().getPriority();
        log.info("消费消息，优先级: {}", priority);
    }
}
```

## 四、业务场景示例

```java
// 订单处理 - VIP 优先
@Service
public class OrderPriorityService {

    public void submitOrder(Order order) {
        int priority = calculatePriority(order);
        rabbitTemplate.convertAndSend("order-exchange", "order.submit",
            JSON.toJSONString(order), msg -> {
                msg.getMessageProperties().setPriority(priority);
                return msg;
            });
    }

    private int calculatePriority(Order order) {
        if ("VIP".equals(order.getUserLevel())) return 10;
        if ("PAID".equals(order.getPayStatus())) return 5;
        return 0;
    }
}
```

## 五、注意事项

1. **优先级队列有性能开销**，不要随意使用
2. **x-max-priority 建议不超过 10**，值越大排序开销越大
3. **优先级只在队列中有等待消息时有效**，如果消费很快，优先级无意义
4. **惰性队列（Lazy Queue）不支持优先级**
5. **消息优先级在声明后不可更改**，只能在发送时设置
