# 死信队列实现

## 一、死信产生条件

消息在以下情况下成为死信（Dead Letter）：

1. **消息被拒绝**：`basicReject` 或 `basicNack` 且 `requeue=false`
2. **消息 TTL 过期**：消息在队列中超过 TTL 时间未被消费
3. **队列达到最大长度**：`x-max-length` 限制，新消息替换旧消息
4. **队列满了**：内存或磁盘达到阈值

## 二、完整死信队列配置

```java
@Configuration
public class DeadLetterConfig {

    // 主交换机
    @Bean
    public DirectExchange mainExchange() {
        return new DirectExchange("main-exchange", true, false);
    }

    // 主队列 - 配置死信参数
    @Bean
    public Queue mainQueue() {
        return QueueBuilder.durable("main-queue")
            .withArgument("x-dead-letter-exchange", "dlx-exchange")
            .withArgument("x-dead-letter-routing-key", "main-queue.dlq")
            .withArgument("x-message-ttl", 60000)          // 消息 TTL 60秒
            .withArgument("x-max-length", 10000)           // 最大长度
            .withArgument("x-overflow", "reject-publish")  // 溢出时拒绝新消息
            .build();
    }

    @Bean
    public Binding mainBinding() {
        return BindingBuilder.bind(mainQueue())
            .to(mainExchange())
            .with("main-key");
    }

    // 死信交换机
    @Bean
    public DirectExchange dlxExchange() {
        return new DirectExchange("dlx-exchange", true, false);
    }

    // 死信队列
    @Bean
    public Queue dlq() {
        return QueueBuilder.durable("main-queue-dlq")
            .withArgument("x-queue-type", "quorum")
            .build();
    }

    @Bean
    public Binding dlqBinding() {
        return BindingBuilder.bind(dlq())
            .to(dlxExchange())
            .with("main-queue.dlq");
    }
}
```

## 三、死信消息处理

```java
// 主队列消费者 - 处理失败进入死信队列
@Component
public class MainConsumer {

    @RabbitListener(queues = "main-queue")
    public void process(Message message, Channel channel) throws IOException {
        long tag = message.getMessageProperties().getDeliveryTag();
        try {
            String body = new String(message.getBody());
            businessLogic(body);
            channel.basicAck(tag, false);
        } catch (BusinessException e) {
            // 业务异常 - 进入死信队列
            log.error("业务异常，消息进入死信队列", e);
            channel.basicNack(tag, false, false);  // requeue=false
        } catch (Exception e) {
            // 系统异常 - 重新入队重试
            channel.basicNack(tag, false, true);   // requeue=true
        }
    }
}

// 死信队列消费者 - 记录/告警/重试
@Component
public class DLQConsumer {

    @RabbitListener(queues = "main-queue-dlq")
    public void handleDeadLetter(Message message) {
        // 获取死信信息
        Map<String, Object> headers = message.getMessageProperties().getHeaders();
        List<Map<String, ?>> xDeath = (List<Map<String, ?>>) headers.get("x-death");

        log.error("死信消息详情:");
        log.error("  原队列: {}", xDeath.get(0).get("queue"));
        log.error("  原因: {}", xDeath.get(0).get("reason"));
        log.error("  消息内容: {}", new String(message.getBody()));

        // 存储到数据库
        dlqService.saveDeadLetter(message);

        // 可选：发送告警
        alertService.sendAlert("死信消息", message);
    }
}
```

## 四、延迟队列（TTL + DLX）

```java
// 利用死信机制实现延迟队列
@Configuration
public class DelayQueueConfig {

    // 延迟队列 - 消息过期后进入死信队列
    @Bean
    public Queue delay30MinQueue() {
        return QueueBuilder.durable("delay-30min-queue")
            .withArgument("x-message-ttl", 1800000)  // 30分钟
            .withArgument("x-dead-letter-exchange", "real-exchange")
            .withArgument("x-dead-letter-routing-key", "order.timeout")
            .build();
    }

    // 实际消费队列
    @Bean
    public Queue orderTimeoutQueue() {
        return QueueBuilder.durable("order-timeout-queue").build();
    }

    @Bean
    public Binding timeoutBinding() {
        return BindingBuilder.bind(orderTimeoutQueue())
            .to(new DirectExchange("real-exchange"))
            .with("order.timeout");
    }
}

// 使用：订单 30 分钟超时取消
public void createOrder(Order order) {
    orderMapper.insert(order);
    // 发送到延迟队列，30分钟后进入 order-timeout-queue
    rabbitTemplate.convertAndSend("delay-30min-queue", order);
}
```

## 五、重试队列方案

```java
// 多级重试：立即重试 -> 1分钟重试 -> 10分钟重试 -> 死信队列
@Configuration
public class RetryQueueConfig {

    @Bean
    public Queue mainQueue() {
        return QueueBuilder.durable("main-queue")
            .withArgument("x-dead-letter-exchange", "retry-exchange")
            .withArgument("x-dead-letter-routing-key", "retry.1min")
            .build();
    }

    @Bean
    public Queue retry1MinQueue() {
        return QueueBuilder.durable("retry-1min-queue")
            .withArgument("x-message-ttl", 60000)
            .withArgument("x-dead-letter-exchange", "main-exchange")
            .withArgument("x-dead-letter-routing-key", "main-key")
            .build();
    }
}
```

## 六、注意事项

1. **死信队列必须有消费者**，否则死信消息也会堆积
2. **x-death 头信息可以追踪死信历史**，避免无限循环
3. **TTL + DLX 实现延迟队列有顺序问题**，先入队的消息阻塞后续消息
4. **死信消息的 routing-key 可能改变**，注意路由规则配置
5. **生产环境建议每个业务队列都配置死信队列**，作为兜底方案
