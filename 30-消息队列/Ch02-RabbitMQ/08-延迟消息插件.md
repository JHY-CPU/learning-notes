# 延迟消息插件

## 一、rabbitmq_delayed_message 插件

RabbitMQ 官方提供的延迟消息插件，通过 `x-delayed-message` 类型的 Exchange 实现精确延迟投递。

```bash
# 1. 下载插件（版本要匹配 RabbitMQ 版本）
wget https://github.com/rabbitmq/rabbitmq-delayed-message-exchange/releases/download/v3.13.0/rabbitmq_delayed_message_exchange-3.13.0.ez

# 2. 复制到插件目录
cp rabbitmq_delayed_message_exchange-3.13.0.ez /usr/lib/rabbitmq/lib/rabbitmq_server-3.13.0/plugins/

# 3. 启用插件
rabbitmq-plugins enable rabbitmq_delayed_message_exchange

# 4. 重启验证
rabbitmqctl status | grep delayed
```

## 二、Spring Boot 集成

```java
@Configuration
public class DelayedExchangeConfig {

    // 声明延迟交换机
    @Bean
    public CustomExchange delayedExchange() {
        Map<String, Object> args = new HashMap<>();
        args.put("x-delayed-type", "direct");  // 底层路由类型
        return new CustomExchange(
            "delayed-exchange",
            "x-delayed-message",   // 交换机类型
            true,                   // durable
            false,                  // autoDelete
            args
        );
    }

    @Bean
    public Queue orderDelayQueue() {
        return QueueBuilder.durable("order-delay-queue").build();
    }

    @Bean
    public Binding delayedBinding() {
        return BindingBuilder.bind(orderDelayQueue())
            .to(delayedExchange())
            .with("order.delay")
            .noargs();
    }
}
```

## 三、发送延迟消息

```java
@Service
public class DelayedMessageService {

    @Autowired
    private RabbitTemplate rabbitTemplate;

    /**
     * 发送延迟消息
     * @param delayMs 延迟毫秒数
     */
    public void sendDelayedMessage(Object payload, long delayMs) {
        rabbitTemplate.convertAndSend(
            "delayed-exchange",
            "order.delay",
            JSON.toJSONString(payload),
            message -> {
                // 设置延迟时间（毫秒）
                message.getMessageProperties().setHeader("x-delay", delayMs);
                return message;
            }
        );
    }

    // 订单 30 分钟超时
    public void sendOrderTimeout(Order order) {
        sendDelayedMessage(order, 30 * 60 * 1000);  // 30 分钟
    }

    // 延迟重试
    public void sendRetryMessage(Object payload, int retryCount) {
        long[] delays = {0, 10_000, 30_000, 60_000, 300_000}; // 指数退避
        long delay = retryCount < delays.length ? delays[retryCount] : delays[delays.length - 1];
        sendDelayedMessage(payload, delay);
    }
}

// 消费者
@Component
public class DelayedMessageConsumer {

    @RabbitListener(queues = "order-delay-queue")
    public void handleDelayedOrder(String message) {
        Order order = JSON.parseObject(message, Order.class);
        // 检查订单是否已支付
        if ("UNPAID".equals(order.getStatus())) {
            orderService.cancelOrder(order.getId());
            log.info("订单超时自动取消: {}", order.getId());
        }
    }
}
```

## 四、与 TTL + DLX 方案对比

```
                    延迟插件                    TTL + DLX
-----------------------------------------------------------------
精确度              毫秒级精确                  受队头消息阻塞影响
灵活性              每条消息不同延迟            只能按队列设置 TTL
消息顺序            无阻塞问题                  先入队消息延迟长会阻塞
性能                较好                        好
适用场景            不同延迟时间混合            统一延迟时间
```

## 五、注意事项

1. **插件版本必须与 RabbitMQ 版本匹配**，否则会导致启动失败
2. **延迟消息存储在 Mnesia 数据库中**，不是常规队列存储
3. **最大延迟时间建议不超过几天**，超长延迟建议用其他方案
4. **集群环境下插件需要所有节点安装**
5. **延迟消息不支持优先级**，延迟时间到了就投递
