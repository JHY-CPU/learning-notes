# 性能调优

## 一、生产者优化

```java
// 1. 使用 Publisher Confirm 异步确认
@Configuration
public class ProducerConfig {

    @Bean
    public RabbitTemplate asyncConfirmTemplate(ConnectionFactory factory) {
        CachingConnectionFactory cachingFactory = (CachingConnectionFactory) factory;
        // 设置信道缓存大小
        cachingFactory.setChannelCacheSize(25);
        cachingFactory.setChannelCheckoutTimeout(5000);

        RabbitTemplate template = new RabbitTemplate(cachingFactory);
        template.setMandatory(true);

        // 异步 Confirm（比同步 Confirm 性能高 10 倍以上）
        template.setConfirmCallback((data, ack, cause) -> {
            if (!ack) log.error("发送失败: {}", cause);
        });

        return template;
    }
}

// 2. 批量发送
public void batchSend(List<Order> orders) {
    orders.forEach(order ->
        rabbitTemplate.convertAndSend("exchange", "key", order)
    );
}
```

## 二、消费者优化

```java
// 1. Prefetch 调优
@Configuration
public class ConsumerConfig {

    @Bean
    public SimpleRabbitListenerContainerFactory optimizedFactory() {
        SimpleRabbitListenerContainerFactory factory = new SimpleRabbitListenerContainerFactory();
        factory.setConnectionFactory(connectionFactory());
        factory.setPrefetchCount(250);           // 根据处理速度调整
        factory.setConcurrentConsumers(5);        // 初始消费者数
        factory.setMaxConcurrentConsumers(20);    // 最大消费者数
        factory.setAcknowledgeMode(AcknowledgeMode.MANUAL);
        factory.setBatchSize(100);                // 批量大小
        factory.setReceiveTimeout(1000L);         // 接收超时
        return factory;
    }
}

// 2. 异步处理
@RabbitListener(queues = "order-queue")
public void consumeAsync(Message message, Channel channel) {
    CompletableFuture.runAsync(() -> {
        try {
            processMessage(message);
            channel.basicAck(message.getMessageProperties().getDeliveryTag(), false);
        } catch (Exception e) {
            handleError(message, channel, e);
        }
    }, taskExecutor);
}
```

## 三、连接与信道池

```java
// CachingConnectionFactory 配置
@Bean
public ConnectionFactory connectionFactory() {
    CachingConnectionFactory factory = new CachingConnectionFactory();
    factory.setHost("localhost");
    factory.setPort(5672);
    factory.setUsername("admin");
    factory.setPassword("admin123");

    // 连接池配置
    factory.setChannelCacheSize(50);              // 信道缓存数
    factory.setChannelCheckoutTimeout(5000);      // 获取信道超时
    factory.setConnectionCacheSize(5);            // 连接缓存数

    // 开启 Publisher Confirm
    factory.setPublisherConfirmType(CachingConnectionFactory.ConfirmType.CORRELATED);

    return factory;
}
```

## 四、队列与消息优化

```yaml
# 队列配置优化
queue:
  # 使用 Quorum Queue（比镜像队列性能更好）
  type: quorum
  # 限制队列长度，防止内存溢出
  max-length: 100000
  max-length-bytes: 1073741824  # 1GB
  # 溢出策略
  overflow: drop-head           # 丢弃头部旧消息

# 消息配置
message:
  # 消息持久化（根据业务需求选择）
  delivery-mode: 2              # 1=非持久, 2=持久
  # 消息压缩（大消息）
  content-encoding: gzip
```

## 七、注意事项

1. **Prefetch 值不是越大越好**，需要根据消费处理时间平衡
2. **过多的 Channel 会消耗内存**，合理设置缓存大小
3. **批量发送比单条发送性能高很多**，但要注意消息大小
4. **持久化消息比非持久化慢 2-5 倍**，根据业务需求选择
5. **消息体压缩可以减少网络传输**，大消息（>1KB）建议开启
