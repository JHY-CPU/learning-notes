# 队列与绑定

## 一、Queue（队列）

队列是消息的最终存储位置，消费者从队列中获取消息。RabbitMQ 支持多种队列类型。

### 队列类型

| 类型 | 引入版本 | 特点 | 适用场景 |
|------|---------|------|---------|
| Classic Queue | 1.x | 简单，单节点存储 | 开发测试 |
| Mirrored Queue | 2.x | 主从复制 | 旧版高可用 |
| Quorum Queue | 3.8 | Raft 协议，强一致 | 生产环境推荐 |
| Stream Queue | 3.9 | 类 Kafka 日志 | 大数据/回溯 |

```java
// 声明 Quorum Queue（推荐）
@Bean
public Queue orderQueue() {
    return QueueBuilder.durable("order-queue")
        .quorum()                                  // Quorum 类型
        .withArgument("x-quorum-initial-group-size", 3)  // 3 节点副本
        .withArgument("x-delivery-limit", 5)       // 最大投递次数
        .withArgument("x-dead-letter-exchange", "dlx-exchange")
        .build();
}

// 声明 Stream Queue
@Bean
public Queue eventStream() {
    return QueueBuilder.durable("event-stream")
        .stream()                                  // Stream 类型
        .withArgument("x-max-age", "7d")           // 保留 7 天
        .withArgument("x-stream-max-segment-size-bytes", 100_000_000)  // 100MB 分段
        .build();
}
```

### 队列参数

```java
// 完整的队列参数配置
Map<String, Object> args = new HashMap<>();
args.put("x-message-ttl", 60000);                // 消息 TTL 60秒
args.put("x-max-length", 10000);                 // 最大消息数
args.put("x-max-length-bytes", 104857600);       // 最大字节数 100MB
args.put("x-overflow", "reject-publish");         // 溢出策略
args.put("x-dead-letter-exchange", "dlx-exchange");
args.put("x-dead-letter-routing-key", "dlq-key");
args.put("x-max-priority", 10);                  // 优先级队列
args.put("x-single-active-consumer", true);       // 单活消费者
```

## 二、Binding（绑定）

绑定定义了 Exchange 和 Queue 之间的路由关系。

```java
// 绑定示例
@Configuration
public class BindingConfig {

    @Bean
    public TopicExchange orderExchange() {
        return new TopicExchange("order-exchange", true, false);
    }

    @Bean
    public Queue orderQueue() {
        return QueueBuilder.durable("order-queue").build();
    }

    // Binding 1: 通配符绑定
    @Bean
    public Binding orderBinding() {
        return BindingBuilder.bind(orderQueue())
            .to(orderExchange())
            .with("order.#");
    }

    // Binding 2: 可以有多个绑定规则
    @Bean
    public Binding paymentBinding() {
        return BindingBuilder.bind(orderQueue())
            .to(orderExchange())
            .with("payment.order.*");
    }
}
```

## 三、Routing Key 设计

```java
// Routing Key 命名规范
// 格式: {领域}.{实体}.{动作}.{级别}

"order.created"                    // 订单创建
"order.payment.success"            // 订单支付成功
"order.shipment.delivered"         // 订单签收
"notification.email.high"          // 高优先级邮件通知
"inventory.stock.warning"          // 库存预警

// Topic Exchange 通配符匹配示例
"order.*"          // 匹配 order.created, order.paid
"order.#"          // 匹配 order.created, order.payment.success
"#.success"        // 匹配所有以 success 结尾的 routing-key
"*.*.warning"      // 匹配两层结构且以 warning 结尾的
```

## 四、死信绑定

```java
// 消息成为死信后被转发到死信 Exchange
@Configuration
public class DeadLetterConfig {

    @Bean
    public Queue mainQueue() {
        return QueueBuilder.durable("main-queue")
            .withArgument("x-dead-letter-exchange", "dlx-exchange")
            .withArgument("x-dead-letter-routing-key", "main-queue.dlq")
            .withArgument("x-message-ttl", 30000)  // 30秒 TTL
            .build();
    }

    @Bean
    public Queue deadLetterQueue() {
        return QueueBuilder.durable("main-dlq").build();
    }

    @Bean
    public DirectExchange dlxExchange() {
        return new DirectExchange("dlx-exchange");
    }

    @Bean
    public Binding dlxBinding() {
        return BindingBuilder.bind(deadLetterQueue())
            .to(dlxExchange())
            .with("main-queue.dlq");
    }
}
```

## 五、注意事项

1. **队列名全局唯一**，不同 vhost 下可以同名
2. **Quorum Queue 是生产环境首选**，提供强一致性保证
3. **绑定规则是动态的**，可以在运行时增删绑定
4. **过多的 binding 会影响 Exchange 路由性能**，合理规划路由规则
5. **队列参数声明后不可修改**，需要删除重建（除非使用 Policy）
