## 二、生产端最佳实践

```java
@Service
public class MessageProducer {

    // 1. 发送前校验
    public void send(String topic, Object payload) {
        if (payload == null) {
            throw new IllegalArgumentException("消息体不能为空");
        }
        if (payload.toString().length() > MAX_MESSAGE_SIZE) {
            throw new IllegalArgumentException("消息体超出限制");
        }
    }

    // 2. 本地消息表 + 事务保证
    @Transactional
    public void sendReliably(String topic, Object payload) {
        MessageRecord record = new MessageRecord();
        record.setMessageId(UUID.randomUUID().toString());
        record.setTopic(topic);
        record.setContent(JSON.toJSONString(payload));
        messageMapper.insert(record);
        // 定时任务负责实际发送
    }

    // 3. 批量发送提升吞吐
    public void batchSend(String topic, List<?> messages) {
        List<ProducerRecord<String, String>> records = messages.stream()
            .map(msg -> new ProducerRecord<>(topic, JSON.toJSONString(msg)))
            .collect(Collectors.toList());
        for (ProducerRecord<String, String> record : records) {
            kafkaTemplate.send(record);
        }
    }
}
```

## 三、消费端最佳实践

```java
@Component
public class MessageConsumer {

    // 1. 消费幂等
    @KafkaListener(topics = "order-events")
    public void consume(ConsumerRecord<String, String> record, Acknowledgment ack) {
        String messageId = extractMessageId(record);
        if (!idempotentService.tryProcess(messageId)) {
            ack.acknowledge();  // 已处理，直接确认
            return;
        }

        try {
            doProcess(record);
            ack.acknowledge();
        } catch (Exception e) {
            idempotentService.clear(messageId);  // 清除标记，允许重试
            throw e;
        }
    }

    // 2. 异常处理策略
    private void doProcess(ConsumerRecord<String, String> record) {
        try {
            businessLogic(record);
        } catch (BusinessException e) {
            // 业务异常 - 发送死信队列
            dlqService.send(record);
        } catch (Exception e) {
            // 系统异常 - 抛出触发重试
            throw e;
        }
    }
}
```

    action: [notify_oncall]

```java
// 自定义监控指标
@Component
public class MQMetrics {

    private final MeterRegistry registry;

    public void recordSendLatency(long latencyMs) {
        registry.timer("mq.send.latency").record(latencyMs, TimeUnit.MILLISECONDS);
    }

    public void recordConsumeSuccess() {
        registry.counter("mq.consume.success").increment();
    }

    public void recordConsumeFailure(String topic, String error) {
        registry.counter("mq.consume.failure",
            "topic", topic, "error", error).increment();
    }
}
```

## 五、运维最佳实践

```yaml
# 备份策略
backup:
  kafka:
    - 定期备份 Topic 配置
    - 跨机房 MirrorMaker 同步
  rabbitmq:
    - 队列定义导出: rabbitmqadmin export
    - 镜像队列 + Quorum Queue

# 容量规划
capacity:
  预估方式: 峰值流量 * 2 * 保留天数
  example:
    峰值: 10000 msg/s
    平均大小: 1KB
    保留7天: 10000 * 1KB * 86400 * 7 = 5.8TB
    实际预留: 5.8TB * 2 = 11.6TB
```

## 六、注意事项

1. **消息体不要太大**，建议控制在 1KB 以内，大文件用引用而非内联
2. **消费端要做好限流**，避免消费者被消息冲垮
3. **定期清理过期消息**，避免磁盘空间耗尽
4. **使用独立的消费者组**，不同业务不要共享消费者组
5. **生产环境至少 3 副本**，确保单点故障不影响可用性
6. **做好灰度发布策略**，新消费者上线前充分测试
