# 消息模型

## 一、点对点模型（Point-to-Point）

点对点模型中，消息从生产者发送到队列，**只有一个消费者**能消费该消息。消息被消费后即从队列中删除。

```java
// 点对点模型 - RabbitMQ 示例
// 生产者
@Component
public class TaskProducer {

    @Autowired
    private RabbitTemplate rabbitTemplate;

    public void sendTask(Task task) {
        // 发送到队列，只有一个消费者能收到
        rabbitTemplate.convertAndSend("task-queue", task);
    }
}

// 消费者1
@RabbitListener(queues = "task-queue")
public void processTask1(Task task) {
    // 消费者1处理任务
}

// 消费者2 - 竞争消费，同一条消息只会被其中一个消费
@RabbitListener(queues = "task-queue")
public void processTask2(Task task) {
    // 消费者2处理任务（与消费者1竞争消费）
}
```

### 点对点模型特点

- 一条消息**只被一个消费者处理**
- 消费者可以有多个，它们**竞争消费**同一条消息
- 消费完成后消息从队列中移除
- 适用于**任务分发、负载均衡**场景

## 二、发布/订阅模型（Pub/Sub）

发布/订阅模型中，消息发送到 Topic，**所有订阅者**都能收到消息的副本。类似于广播机制。

```java
// 发布订阅模型 - Kafka 示例
// 生产者
@Component
public class EventProducer {

    @Autowired
    private KafkaTemplate<String, String> kafkaTemplate;

    public void publishEvent(Event event) {
        kafkaTemplate.send("user-events", event.getUserId(), JSON.toJSONString(event));
    }
}

// 消费者组1 - 日志服务
@KafkaListener(topics = "user-events", groupId = "log-group")
public void logEvent(String message) {
    logService.save(JSON.parseObject(message, Event.class));
}

// 消费者组2 - 推送服务
@KafkaListener(topics = "user-events", groupId = "push-group")
public void pushNotification(String message) {
    pushService.send(JSON.parseObject(message, Event.class));
}

// 消费者组3 - 积分服务
@KafkaListener(topics = "user-events", groupId = "points-group")
public void addPoints(String message) {
    pointsService.process(JSON.parseObject(message, Event.class));
}
```

### 发布/订阅模型特点

- 一条消息**被所有订阅者**各自消费一份
- 通过**不同的消费者组**实现广播
- 同一消费者组内仍然点对点消费
- 适用于**事件通知、日志广播**场景

## 三、RabbitMQ 的 Exchange 实现

RabbitMQ 通过 Exchange（交换机）实现灵活的消息路由：

```
Producer -> Exchange -> Binding -> Queue -> Consumer
```

| Exchange 类型 | 路由规则 | 适用场景 |
|--------------|---------|---------|
| Direct | 精确匹配 Routing Key | 点对点 |
| Fanout | 广播到所有绑定队列 | 发布订阅 |
| Topic | 通配符匹配 Routing Key | 模糊路由 |
| Header | 匹配消息头属性 | 复杂路由 |

```yaml
# RabbitMQ 配置
spring:
  rabbitmq:
    exchanges:
      order-exchange:
        type: topic          # 支持通配符
      broadcast-exchange:
        type: fanout         # 广播模式
    queues:
      order-queue:
        bindings:
          - exchange: order-exchange
            routing-key: "order.*"   # 匹配 order.created, order.paid 等
```

## 四、模型选择建议

```java
// 场景1：异步任务处理 - 使用点对点模型
// 一个订单只需要一个服务处理
mq.send("order-process-queue", order);

// 场景2：事件通知 - 使用发布订阅模型
// 订单创建后，库存、积分、推送都需要知道
mq.send("order-created-topic", order);

// 场景3：混合模式 - 同时使用两种模型
// 通过 Exchange 类型和 Consumer Group 组合实现
```

## 五、注意事项

1. **点对点模型适合任务分发**，保证每个任务只被处理一次
2. **发布订阅模型适合事件广播**，多个下游服务都需要感知同一事件
3. **Kafka 天然支持发布订阅**，通过 Consumer Group 实现；RabbitMQ 需要通过 Exchange 类型区分
4. **消费组之间互不影响**，适合不同业务线独立消费同一消息源
