# 死信队列

## 一、什么是死信队列

死信队列（Dead Letter Queue，DLQ）是**专门接收处理失败消息的队列**。当消息无法被正常消费时，会被转发到死信队列，供后续人工处理或重新消费。

```
正常流程: Producer -> Queue -> Consumer (成功 -> ACK)
失败流程: Producer -> Queue -> Consumer (失败 -> Dead Letter Queue)
```

## 二、消息成为死信的条件

1. **消息被拒绝**（basic.reject / basic.nack）且 requeue=false
2. **消息 TTL 过期**，未被消费
3. **队列满了**，新消息无法入队

```java
// RabbitMQ 死信队列配置
@Configuration
public class DLXConfig {

    // 正常队列 - 配置死信参数
    @Bean
    public Queue orderQueue() {
        return QueueBuilder.durable("order-queue")
            .withArgument("x-dead-letter-exchange", "dlx-exchange")     // 死信交换机
            .withArgument("x-dead-letter-routing-key", "order.dlq")     // 死信路由键
            .withArgument("x-message-ttl", 60000)                       // 消息 TTL 60 秒
            .withArgument("x-max-length", 10000)                        // 队列最大长度
            .build();
    }

    // 死信队列
    @Bean
    public Queue deadLetterQueue() {
        return QueueBuilder.durable("order-dlq").build();
    }

    // 死信交换机
    @Bean
    public DirectExchange dlxExchange() {
        return new DirectExchange("dlx-exchange");
    }

    // 绑定
    @Bean
    public Binding dlxBinding() {
        return BindingBuilder.bind(deadLetterQueue())
            .to(dlxExchange())
            .with("order.dlq");
    }
}
```

## 三、消费者处理死信

```java
// 正常消费者 - 失败后进入死信队列
@RabbitListener(queues = "order-queue")
public void processOrder(Message message, Channel channel) {
    try {
        Order order = JSON.parseObject(message.getBody(), Order.class);
        orderService.process(order);
        channel.basicAck(message.getMessageProperties().getDeliveryTag(), false);
    } catch (Exception e) {
        log.error("处理失败，消息进入死信队列", e);
        // requeue=false，消息进入死信队列
        channel.basicNack(message.getMessageProperties().getDeliveryTag(), false, false);
    }
}

// 死信队列消费者 - 记录或告警
@RabbitListener(queues = "order-dlq")
public void handleDeadLetter(Message message) {
    String orderId = message.getMessageProperties().getHeaders().get("orderId", String.class);
    log.error("死信消息: orderId={}, originalQueue={}", orderId,
        message.getMessageProperties().getXDeathHeader());
    // 记录到数据库或发送告警
    deadLetterService.record(message);
}
```

## 四、Kafka 死信队列实现

Kafka 没有内置死信队列，需要手动实现：

```java
@Component
public class KafkaDLQHandler {

    @Autowired
    private KafkaTemplate<String, String> kafkaTemplate;

    @KafkaListener(topics = "order-topic")
    public void consume(ConsumerRecord<String, String> record) {
        try {
            processMessage(record);
        } catch (Exception e) {
            log.error("消费失败，发送到死信主题", e);
            // 转发到死信 Topic
            kafkaTemplate.send("order-dlq-topic", record.key(), record.value());
        }
    }

    // 死信主题消费者
    @KafkaListener(topics = "order-dlq-topic")
    public void handleDLQ(ConsumerRecord<String, String> record) {
        log.error("死信消息: key={}, value={}", record.key(), record.value());
        dlqService.record(record);
    }
}
```

## 五、死信消息的处理策略

```java
@Service
public class DLQRecoveryService {

    // 1. 人工处理后重投递
    public void retryDeadLetter(Long dlqRecordId) {
        DLQRecord record = dlqMapper.selectById(dlqRecordId);
        kafkaTemplate.send(record.getOriginalTopic(), record.getContent());
        dlqMapper.updateStatus(dlqRecordId, "RETRIED");
    }

    // 2. 批量重试
    @Scheduled(cron = "0 0 2 * * ?")  // 每天凌晨2点
    public void batchRetry() {
        List<DLQRecord> records = dlqMapper.selectPending(100);
        for (DLQRecord record : records) {
            try {
                kafkaTemplate.send(record.getOriginalTopic(), record.getContent());
                dlqMapper.updateStatus(record.getId(), "RETRIED");
            } catch (Exception e) {
                log.error("重试失败: {}", record.getId());
            }
        }
    }
}
```

## 六、注意事项

1. **死信队列一定要配置消费者**，否则死信消息也会堆积
2. **不要无限重试**，失败的消息反复重试可能导致系统雪崩
3. **记录死信消息的详细信息**，包括原队列、失败原因、重试次数
4. **定期清理已处理的死信记录**，避免数据无限增长
5. **死信队列的监控告警很重要**，及时发现系统异常
