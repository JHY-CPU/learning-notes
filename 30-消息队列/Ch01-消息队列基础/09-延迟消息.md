# 延迟消息

## 一、什么是延迟消息

延迟消息是指**消息发送后不会立即被消费，而是在指定的延迟时间后才投递给消费者**。常用于订单超时取消、定时提醒、延迟重试等场景。

```java
// 典型场景：订单 30 分钟未支付自动取消
public void createOrder(Order order) {
    orderMapper.insert(order);
    // 发送延迟消息，30分钟后检查订单状态
    delayMq.send("order-delay-topic", order, 30, TimeUnit.MINUTES);
}

// 30 分钟后收到消息
@MQListener("order-delay-topic")
public void checkOrderTimeout(Order order) {
    Order db = orderMapper.selectById(order.getId());
    if ("UNPAID".equals(db.getStatus())) {
        orderService.cancelOrder(order.getId());  // 超时取消
    }
}
```

## 二、RocketMQ 延迟消息

RocketMQ 内置支持延迟消息，提供 18 个延迟级别：

```java
// RocketMQ 延迟消息
@Component
public class DelayMessageProducer {

    @Autowired
    private RocketMQTemplate rocketMQTemplate;

    public void sendDelayMessage(Order order, int delayLevel) {
        Message<Order> message = MessageBuilder
            .withPayload(order)
            .build();

        // delayLevel: 1=1s, 2=5s, 3=10s, 4=30s, 5=1m, 6=2m...
        rocketMQTemplate.syncSend("delay-topic", message, 3000, delayLevel);
    }
}

// 延迟级别配置（RocketMQ 5.0+ 支持自定义时间）
// 1s 5s 10s 30s 1m 2m 3m 4m 5m 6m 7m 8m 9m 10m 20m 30m 1h 2h
```

## 三、RabbitMQ 延迟消息方案

### 方案1：使用插件 rabbitmq_delayed_message

```java
// 安装 delayed_message 插件后
@Configuration
public class DelayedMQConfig {

    @Bean
    public CustomExchange delayedExchange() {
        Map<String, Object> args = new HashMap<>();
        args.put("x-delayed-type", "direct");
        return new CustomExchange("delayed-exchange", "x-delayed-message", true, false, args);
    }
}

// 发送延迟消息
public void sendDelayedMessage(Order order, long delayMillis) {
    rabbitTemplate.convertAndSend("delayed-exchange", "order.delay", order, message -> {
        message.getMessageProperties().setHeader("x-delay", delayMillis);
        return message;
    });
}
```

### 方案2：TTL + 死信队列

```java
// 设置队列 TTL，消息过期后进入死信队列
@Configuration
public class TTLQueueConfig {

    @Bean
    public Queue delay30MinQueue() {
        return QueueBuilder.durable("delay-30min-queue")
            .withArgument("x-message-ttl", 1800000)  // 30分钟
            .withArgument("x-dead-letter-exchange", "dlx-exchange")
            .withArgument("x-dead-letter-routing-key", "order.timeout")
            .build();
    }

    @Bean
    public Queue orderTimeoutQueue() {
        return QueueBuilder.durable("order-timeout-queue").build();
    }
}
```

## 四、时间轮算法（Netty HashedWheelTimer）

```java
// 基于时间轮的延迟任务
public class DelayTaskScheduler {

    private final HashedWheelTimer timer = new HashedWheelTimer(100, TimeUnit.MILLISECONDS);

    public void schedule(Runnable task, long delay, TimeUnit unit) {
        timer.newTimeout(timeout -> task.run(), delay, unit);
    }
}

// 使用
delayTaskScheduler.schedule(() -> {
    checkOrderTimeout(orderId);
}, 30, TimeUnit.MINUTES);
```

## 五、Redis 实现延迟队列

```java
// 使用 Redis Sorted Set 实现
@Service
public class RedisDelayQueue {

    @Autowired
    private StringRedisTemplate redisTemplate;

    public void addDelayTask(String taskId, long delaySeconds) {
        double score = System.currentTimeMillis() + delaySeconds * 1000;
        redisTemplate.opsForZSet().add("delay-queue", taskId, score);
    }

    @Scheduled(fixedDelay = 1000)
    public void pollReadyTasks() {
        double now = System.currentTimeMillis();
        // 获取已到期的任务
        Set<String> tasks = redisTemplate.opsForZSet()
            .rangeByScore("delay-queue", 0, now, 0, 100);
        for (String taskId : tasks) {
            processTask(taskId);
            redisTemplate.opsForZSet().remove("delay-queue", taskId);
        }
    }
}
```

## 六、注意事项

1. **RocketMQ 延迟级别固定**，不能精确到任意时间，5.0+ 版本支持自定义
2. **TTL + 死信方案有消息顺序问题**，队头消息未过期会阻塞后续消息
3. **大规模延迟任务使用时间轮或 Redis**，性能优于 MQ 延迟消息
4. **延迟时间不宜过长**，过长的延迟消息占用 Broker 存储资源
5. **消费端要做好幂等处理**，延迟消息可能被重复投递
