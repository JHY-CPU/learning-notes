# 事务消息

## 一、分布式事务与 MQ

在分布式系统中，一个业务操作可能涉及多个服务的数据库操作。事务消息保证**本地事务和消息发送的原子性**——要么都成功，要么都失败。

```java
// 错误做法：先提交事务再发消息
@Transactional
public void createOrder(Order order) {
    orderMapper.insert(order);          // 本地事务提交
    mq.send("order-topic", order);      // 消息发送失败，数据不一致！
}

// 正确做法：使用事务消息
public void createOrder(Order order) {
    // 本地事务和消息发送绑定在一起
    transactionalMq.sendMessage("order-topic", order, () -> {
        orderMapper.insert(order);
        return true;
    });
}
```

## 二、RocketMQ 事务消息

RocketMQ 原生支持事务消息，通过**半消息（Half Message）和事务回查**机制实现。

```
Producer                    Broker
  |                           |
  |-- 1.发送半消息 ----------->|
  |<-- 2.半消息已存储 ---------|
  |-- 3.执行本地事务 --------->|
  |                           |
  |-- 4a.Commit/Rollback ---->|
  |                           |
  |     (如果超时未确认)        |
  |<-- 5.事务回查 -------------|
  |-- 6.返回事务状态 --------->|
```

```java
@Component
public class OrderTransactionProducer {

    @Autowired
    private RocketMQTemplate rocketMQTemplate;

    public void createOrder(Order order) {
        // 发送事务消息
        TransactionSendResult result = rocketMQTemplate.sendMessageInTransaction(
            "order-topic",
            MessageBuilder.withPayload(order)
                .setHeader("orderId", order.getId())
                .build(),
            order  // 传给本地事务的参数
        );
    }
}

// 事务监听器
@RocketMQTransactionListener
public class OrderTransactionListener implements RocketMQLocalTransactionListener {

    // 执行本地事务
    @Override
    public RocketMQLocalTransactionState executeLocalTransaction(Message msg, Object arg) {
        try {
            Order order = (Order) arg;
            orderMapper.insert(order);  // 执行本地事务
            return RocketMQLocalTransactionState.COMMIT;  // 提交消息
        } catch (Exception e) {
            return RocketMQLocalTransactionState.ROLLBACK;  // 回滚消息
        }
    }

    // 事务回查
    @Override
    public RocketMQLocalTransactionState checkLocalTransaction(Message msg) {
        String orderId = msg.getHeaders().get("orderId", String.class);
        Order order = orderMapper.selectById(orderId);
        if (order != null) {
            return RocketMQLocalTransactionState.COMMIT;   // 本地事务成功
        }
        return RocketMQLocalTransactionState.UNKNOWN;      // 等待下次回查
    }
}
```

## 三、本地消息表方案

适用于不支持事务消息的 MQ（如 Kafka、RabbitMQ）。

```java
@Service
public class LocalMessageTableService {

    @Transactional
    public void createOrder(Order order) {
        // 同一个数据库事务中完成
        orderMapper.insert(order);

        // 保存待发送的消息记录
        MessageRecord record = new MessageRecord();
        record.setMessageId(UUID.randomUUID().toString());
        record.setTopic("order-topic");
        record.setContent(JSON.toJSONString(order));
        record.setStatus("PENDING");
        messageMapper.insert(record);
    }

    // 定时任务扫描并发送消息
    @Scheduled(fixedDelay = 1000)
    public void pollAndSend() {
        List<MessageRecord> pending = messageMapper.selectPending(100);
        for (MessageRecord record : pending) {
            try {
                kafkaTemplate.send(record.getTopic(), record.getContent());
                messageMapper.updateStatus(record.getMessageId(), "SENT");
            } catch (Exception e) {
                log.error("发送失败: {}", record.getMessageId(), e);
                messageMapper.incrementRetryCount(record.getMessageId());
            }
        }
    }
}
```

```sql
-- 本地消息表
CREATE TABLE message_record (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    message_id VARCHAR(64) UNIQUE NOT NULL,
    topic VARCHAR(100) NOT NULL,
    content TEXT NOT NULL,
    status VARCHAR(20) DEFAULT 'PENDING',  -- PENDING/SENT/FAILED
    retry_count INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_status_retry (status, retry_count)
);
```

## 四、注意事项

1. **RocketMQ 事务消息是首选方案**，原生支持，无需额外存储
2. **本地消息表兼容所有 MQ**，但实现复杂度更高
3. **事务回查接口必须实现幂等**，可能被多次调用
4. **消息发送和本地事务必须在同一方法内**，保证原子性
5. **回查次数有限制**，需要确保在限制内能确定事务状态
