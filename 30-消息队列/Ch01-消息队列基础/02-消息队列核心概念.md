# 消息队列核心概念

## 一、核心组件说明

消息队列系统由以下几个核心组件构成：

### 1. Producer（生产者）

负责创建并发送消息到消息队列。生产者不关心消息被谁消费，只负责将消息投递到指定的目标。

```java
// 生产者示例
@Component
public class OrderProducer {

    @Autowired
    private KafkaTemplate<String, String> kafkaTemplate;

    public void sendOrder(Order order) {
        // 构建消息
        Message<String> message = MessageBuilder
            .withPayload(JSON.toJSONString(order))
            .setHeader("orderId", order.getId())
            .setHeader("timestamp", System.currentTimeMillis())
            .build();

        // 发送到指定 Topic
        kafkaTemplate.send("order-topic", order.getId().toString(), JSON.toJSONString(order));
    }
}
```

### 2. Consumer（消费者）

负责从消息队列中接收并处理消息。消费者通过订阅 Topic 或 Queue 来获取消息。

```java
// 消费者示例
@Component
public class OrderConsumer {

    @KafkaListener(topics = "order-topic", groupId = "order-group")
    public void consumeOrder(ConsumerRecord<String, String> record) {
        Order order = JSON.parseObject(record.value(), Order.class);
        // 处理订单逻辑
        processOrder(order);
    }
}
```

### 3. Broker（消息代理）

Broker 是消息队列的服务端，负责接收、存储和转发消息。生产者将消息发送给 Broker，消费者从 Broker 拉取或接收消息。

```
Producer  --发送消息-->  Broker  --推送/拉取-->  Consumer
              (存储、路由、转发)
```

### 4. Topic（主题）

Topic 是消息的逻辑分类，类似于数据库中的表。生产者将消息发送到特定的 Topic，消费者订阅感兴趣的 Topic。

```yaml
# Kafka Topic 配置示例
kafka:
  topics:
    order-topic:
      partitions: 6        # 分区数
      replication-factor: 3 # 副本数
      retention-hours: 168  # 保留7天
    payment-topic:
      partitions: 3
      replication-factor: 3
```

### 5. Partition/Queue（分区/队列）

Topic 下的物理存储单元。消息实际存储在 Partition（Kafka）或 Queue（RabbitMQ）中，支持水平扩展和并行消费。

```
Topic: order-topic
├── Partition 0: [msg1, msg2, msg3, ...]
├── Partition 1: [msg4, msg5, msg6, ...]
├── Partition 2: [msg7, msg8, msg9, ...]
└── Partition 3: [msg10, msg11, msg12, ...]
```

## 二、消息的生命周期

```
1. 生产者创建消息 -> 2. 发送到Broker -> 3. Broker存储消息
-> 4. 消费者拉取/接收消息 -> 5. 消费者处理消息 -> 6. 确认(ACK)
```

## 三、关键概念补充

### Consumer Group（消费者组）

多个消费者组成一个组，共同消费一个 Topic。同一个 Consumer Group 内，每条消息只会被一个消费者处理；不同 Group 各自独立消费全量消息。

```java
// 三个消费者属于同一组，分区消息被均分
@KafkaListener(topics = "order-topic", groupId = "order-process-group")
public void consume(ConsumerRecord<String, String> record) {
    // 同组内每条消息只被一个实例消费
}
```

### Offset（偏移量）

记录消费者在分区中的消费位置，用于追踪哪些消息已被消费。

```java
// 手动提交 Offset
@KafkaListener(topics = "order-topic")
public void consume(ConsumerRecord<String, String> record, Acknowledgment ack) {
    processMessage(record);
    ack.acknowledge();  // 手动确认，提交 Offset
}
```

## 四、注意事项

1. **Topic 是逻辑概念，Partition 是物理存储**，一个 Topic 可以有多个 Partition
2. **同一个 Consumer Group 内消费者数量不要超过 Partition 数量**，否则多余的消费者会空闲
3. **Broker 需要集群部署**，避免单点故障导致消息丢失
4. **消费者处理速度跟不上生产速度会导致消息堆积**，需要合理规划消费能力
