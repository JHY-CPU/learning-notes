# 消息幂等性

## 一、什么是消息幂等性

幂等性是指**同一操作执行多次，结果与执行一次相同**。在消息队列中，由于网络抖动、消费者重启等原因，同一条消息可能被消费多次，需要保证重复消费不会产生副作用。

```java
// 非幂等操作 - 每次执行结果不同
public void addBalance(Long userId, BigDecimal amount) {
    BigDecimal balance = accountMapper.getBalance(userId);
    accountMapper.updateBalance(userId, balance.add(amount)); // 重复执行会重复加钱！
}

// 幂等操作 - 重复执行结果相同
public void setBalance(Long userId, BigDecimal amount) {
    accountMapper.updateBalance(userId, amount); // 无论执行几次，结果都是 amount
}
```

## 二、常见幂等方案

### 1. 唯一索引 / 去重表

利用数据库唯一约束防止重复插入。

```java
@Service
public class IdempotentOrderService {

    @Transactional
    public void processOrder(String messageId, Order order) {
        try {
            // 先插入去重记录
            messageRecordMapper.insert(new MessageRecord(messageId, "PROCESSED"));
            // 再处理业务逻辑
            orderMapper.insert(order);
        } catch (DuplicateKeyException e) {
            // 唯一键冲突，说明已处理过
            log.info("消息已处理，跳过: {}", messageId);
        }
    }
}
```

```sql
-- 去重表
CREATE TABLE message_record (
    message_id VARCHAR(64) PRIMARY KEY,  -- 消息唯一ID
    status VARCHAR(20),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 2. Redis 去重

利用 Redis 的原子性判断消息是否已处理。

```java
@Service
public class IdempotentService {

    @Autowired
    private StringRedisTemplate redisTemplate;

    public boolean tryProcess(String messageId) {
        // SETNX 原子操作，设置成功返回 true
        Boolean success = redisTemplate.opsForValue()
            .setIfAbsent("msg:" + messageId, "1", 24, TimeUnit.HOURS);
        return Boolean.TRUE.equals(success);
    }

    public void consume(String messageId, Order order) {
        if (!tryProcess(messageId)) {
            log.info("消息已消费，跳过: {}", messageId);
            return;
        }
        try {
            orderService.process(order);
        } catch (Exception e) {
            // 处理失败，删除标记允许重试
            redisTemplate.delete("msg:" + messageId);
            throw e;
        }
    }
}
```

### 3. 状态机幂等

通过业务状态流转保证幂等，只处理合法的状态变更。

```java
@Service
public class OrderStateMachine {

    // 只有当前状态为 CREATED 才能变为 PAID
    public void payOrder(Long orderId) {
        int rows = orderMapper.updateStatus(
            orderId,
            "PAID",          // 目标状态
            "CREATED"        // 前置条件状态
        );
        if (rows == 0) {
            log.info("订单状态不合法或已支付，跳过");
        }
    }
}

-- SQL 条件更新
UPDATE orders SET status = 'PAID'
WHERE id = #{orderId} AND status = 'CREATED';
```

### 4. Token 机制

客户端先获取 Token，提交时携带 Token，服务端验证 Token 唯一性。

```java
@RestController
public class IdempotentController {

    // 1. 生成 Token
    @GetMapping("/token")
    public String generateToken() {
        String token = UUID.randomUUID().toString();
        redisTemplate.opsForValue().set("token:" + token, "1", 10, TimeUnit.MINUTES);
        return token;
    }

    // 2. 提交时验证 Token
    @PostMapping("/order")
    public Result createOrder(@RequestHeader("Idempotent-Token") String token, Order order) {
        Boolean deleted = redisTemplate.delete("token:" + token);
        if (!Boolean.TRUE.equals(deleted)) {
            return Result.fail("重复请求");
        }
        orderService.create(order);
        return Result.success();
    }
}
```

## 三、Kafka 幂等生产者

```yaml
# Kafka 0.11+ 内置幂等支持
spring:
  kafka:
    producer:
      properties:
        enable.idempotence: true   # 开启幂等
        acks: all                  # 必须配合 acks=all
        retries: 2147483647        # 无限重试
        max.in.flight.requests.per.connection: 5  # 最多5个未确认请求
```

## 四、注意事项

1. **幂等方案需要在业务开始前执行**，先检查再处理
2. **Redis 去重要设置合理的过期时间**，避免内存无限增长
3. **状态机方案最优雅**，但需要业务状态设计合理
4. **处理失败时需要清除幂等标记**，否则无法重试
5. **消息 ID 必须全局唯一**，建议使用 messageId 而非业务 ID
