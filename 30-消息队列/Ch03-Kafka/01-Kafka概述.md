# Kafka 概述

## 一、什么是 Apache Kafka

Apache Kafka 是一个**分布式流处理平台**，由 LinkedIn 开发并开源。它以**高吞吐、低延迟、持久化**著称，广泛用于日志收集、实时数据管道、流处理等场景。

```java
// Kafka 的核心定位
// 1. 发布/订阅消息队列
// 2. 持久化日志存储
// 3. 流处理平台
```

## 二、核心特点

```
特性              说明
-----------------------------------------------------------
高吞吐            百万级消息/秒，顺序写磁盘 + 零拷贝
低延迟            毫秒级延迟（P99 < 10ms）
持久化            消息持久化到磁盘，支持回溯消费
水平扩展          通过增加 Broker 和 Partition 扩展
高可用            多副本机制，ISR 保证数据可靠性
流处理            内置 Kafka Streams 流处理库
```

## 三、Kafka 架构

```
┌─────────┐
│Producer1│──┐
└─────────┘  │    ┌──────────────────────────────┐
┌─────────┐  ├───→│  Kafka Cluster                │
│Producer2│──┘    │  ┌────────┐ ┌────────┐       │    ┌──────────┐
└─────────┘       │  │Broker1 │ │Broker2 │ ...   │───→│Consumer1 │
                  │  │P0(L)   │ │P0(F)   │       │    │Consumer2 │
                  │  │P1(F)   │ │P1(L)   │       │    │Group-A   │
                  │  └────────┘ └────────┘       │    └──────────┘
                  └──────────────────────────────┘
                         ↑              ↑
                    ZooKeeper      (或 KRaft)
```

## 四、与其他 MQ 对比

| 指标 | Kafka | RabbitMQ | RocketMQ |
|------|-------|----------|----------|
| 吞吐量 | 百万/秒 | 万/秒 | 十万/秒 |
| 延迟 | 毫秒 | 微秒 | 毫秒 |
| 消息回溯 | 支持 | 不支持 | 支持 |
| 消息持久化 | 磁盘 | 内存+磁盘 | 磁盘 |
| 流处理 | Kafka Streams | 无 | 无 |
| 消息顺序 | 分区有序 | 队列有序 | 分区有序 |

## 七、注意事项

1. **Kafka 不适合做业务消息队列**，缺少延迟消息、优先级队列等特性
2. **消息消费后不会删除**，通过保留策略（retention）清理
3. **消费者数量不能超过分区数**，否则多余的消费者空闲
4. **Kafka 依赖磁盘顺序写**，不建议使用网络存储（NFS）
5. **3.0+ 版本推荐使用 KRaft 模式**，去 ZooKeeper 依赖
