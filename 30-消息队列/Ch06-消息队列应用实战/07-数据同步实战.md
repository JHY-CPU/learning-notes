## 二、同步到 ES

```java
@Component
@KafkaListener(topics = "mysql.mydb.orders")
public class OrderSyncConsumer {
    public void onCDCEvent(ConsumerRecord<String, String> record) {
        // 解析 Debezium 事件
        JsonNode event = objectMapper.readTree(record.value());
        String op = event.get("op").asText(); // c=create, u=update, d=delete

        switch (op) {
            case "c": case "u":
                Order order = objectMapper.treeToValue(event.get("after"), Order.class);
                elasticsearchTemplate.save(order);
                break;
            case "d":
                String id = event.get("before").get("id").asText();
                elasticsearchTemplate.delete(id, Order.class);
                break;
        }
    }
}
```

## 五、数据同步到多种目标

```java
// 同步到多个下游系统
@Component
public class MultiTargetSyncConsumer {
    @KafkaListener(topics = "cdc-events", groupId = "multi-sync")
    public void onCDCEvent(ConsumerRecord<String, String> record)
            throws Exception {
        JsonNode event = objectMapper.readTree(record.value());
        String op = event.get("op").asText();
        String table = event.get("source").get("table").asText();

        // 同步到 Elasticsearch
        syncToES(table, op, event);

        // 同步到 Redis 缓存
        syncToRedis(table, op, event);

        // 同步到数据仓库
        syncToDataWarehouse(table, op, event);
    }

    private void syncToES(String table, String op, JsonNode event) throws Exception {
        if ("d".equals(op)) {
            esClient.delete(table, event.get("before").get("id").asText());
        } else {
            JsonNode data = event.get("after");
            esClient.index(table, data);
        }
    }

    private void syncToRedis(String table, String op, JsonNode event) {
        String key = table + ":" + event.get("after").get("id").asText();
        if ("d".equals(op)) {
            redisTemplate.delete(key);
        } else {
            redisTemplate.opsForValue().set(key,
                event.get("after").toString(), 1, TimeUnit.HOURS);
        }
    }
}
```

## 六、注意事项

1. **Debezium 比轮询同步性能高很多**
2. **CDC 事件格式与业务格式不同**，需要转换
3. **同步延迟要监控**，确保数据及时性
4. **Schema 变更要处理好**，避免同步中断
5. **全量同步和增量同步要分开处理**
6. **CDC 对 MySQL 有一定的性能开销**，binlog 格式建议使用 ROW
7. **同步到 ES 时考虑使用 Bulk API**，提升写入效率
