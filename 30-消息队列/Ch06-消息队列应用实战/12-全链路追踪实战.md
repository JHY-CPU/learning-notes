# 全链路追踪实战

## 一、TraceID 传递

```java
// 生产端传递 TraceID
public void sendWithTrace(String topic, Object message) {
    ProducerRecord<String, String> record = new ProducerRecord<>(
        topic, JSON.toJSONString(message));
    record.headers().add("traceId", TraceContext.getTraceId().getBytes());
    record.headers().add("spanId", generateSpanId().getBytes());
    record.headers().add("parentSpanId", TraceContext.getSpanId().getBytes());
    kafkaTemplate.send(record);
}

// 消费端恢复 TraceID
@KafkaListener(topics = "order-events")
public void consume(ConsumerRecord<String, String> record) {
    String traceId = getHeaderValue(record, "traceId");
    String spanId = getHeaderValue(record, "spanId");
    TraceContext.set(traceId, spanId);

    try {
        process(record);
    } finally {
        TraceContext.clear();
    }
}

// 日志中输出 TraceID
@Slf4j
public class TracedLogger {
    public void log(String message) {
        log.info("[{}] {}", TraceContext.getTraceId(), message);
    }
}
```

## 三、MDC 上下文管理

```java
// MDC 管理 TraceID
@Component
public class TraceMdcConsumer {

    @KafkaListener(topics = "order-events")
    public void consume(ConsumerRecord<String, String> record) {
        String traceId = getHeaderValue(record, "traceId");
        try {
            MDC.put("traceId", traceId);
            MDC.put("spanId", generateSpanId());
            MDC.put("source", "kafka:order-events");
            process(record);
        } finally {
            MDC.clear();
        }
    }

    // 日志格式自动包含 MDC 字段
    // logback.xml 配置:
    // <pattern>%d{yyyy-MM-dd HH:mm:ss} [%X{traceId}] [%X{spanId}] %level %logger - %msg</pattern>
}
```

## 四、OpenTelemetry 集成

```java
// 使用 OpenTelemetry 实现分布式追踪
@Configuration
public class OtelConfig {
    @Bean
    public OpenTelemetry openTelemetry() {
        SdkTracerProvider tracerProvider = SdkTracerProvider.builder()
            .addSpanProcessor(
                BatchSpanProcessor.builder(
                    OtlpGrpcSpanExporter.builder()
                        .setEndpoint("http://otel-collector:4317")
                        .build()
                ).build()
            )
            .setResource(Resource.getDefault()
                .merge(Resource.builder()
                    .put("service.name", "order-service")
                    .build()))
            .build();

        return OpenTelemetrySdk.builder()
            .setTracerProvider(tracerProvider)
            .setPropagators(ContextPropagators.create(
                W3CTraceContextPropagator.getInstance()))
            .buildAndRegisterGlobal();
    }
}

// 在生产端注入追踪信息
public void sendWithOtel(String topic, Object message) {
    Tracer tracer = GlobalOpenTelemetry.getTracer("order-service");
    Span span = tracer.spanBuilder("kafka-send").startSpan();
    try (Scope scope = span.makeCurrent()) {
        ProducerRecord<String, String> record = new ProducerRecord<>(
            topic, JSON.toJSONString(message));
        // OpenTelemetry Kafka Instrumentation 会自动注入 Header
        kafkaTemplate.send(record);
    } finally {
        span.end();
    }
}
```

## 六、注意事项

1. **TraceID 必须全链路传递**，不能丢失
2. **日志格式要包含 TraceID**，便于排查
3. **异步场景要特别注意上下文传递**
4. **使用 MDC（Mapped Diagnostic Context）管理 TraceID**
5. **TraceID 使用 UUID 或 Snowflake 生成**
6. **OpenTelemetry 是当前标准方案**，建议优先使用
7. **追踪数据采样率要设置合理**，100% 采样有性能影响
