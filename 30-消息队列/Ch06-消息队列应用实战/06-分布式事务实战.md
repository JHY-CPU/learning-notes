# 分布式事务实战

## 一、本地消息表方案

```java
@Service
public class DistributedTransactionService {

    @Transactional
    public void deductInventory(Order order) {
        // 1. 本地事务
        inventoryMapper.deduct(order.getProductId(), order.getQuantity());

        // 2. 保存消息记录（同一事务）
        MessageRecord msg = new MessageRecord();
        msg.setTopic("inventory-deducted");
        msg.setContent(JSON.toJSONString(order));
        msg.setStatus("PENDING");
        messageMapper.insert(msg);
    }

    // 3. 定时投递
    @Scheduled(fixedDelay = 1000)
    public void deliverMessages() {
        List<MessageRecord> pending = messageMapper.selectPending(100);
        for (MessageRecord msg : pending) {
            try {
                kafkaTemplate.send(msg.getTopic(), msg.getContent());
                messageMapper.updateStatus(msg.getId(), "SENT");
            } catch (Exception e) {
                messageMapper.incrementRetry(msg.getId());
            }
        }
    }
}
```

## 二、Saga 编排模式

```java
// 正向流程
public void createOrderSaga(Order order) {
    // Step 1: 扣库存
    inventoryService.deduct(order);
    // Step 2: 扣余额
    accountService.deduct(order);
    // Step 3: 创建订单
    orderService.create(order);
}

// 补偿流程（任一步骤失败）
public void compensateOrderSaga(Order order, int failedStep) {
    switch (failedStep) {
        case 3: accountService.refund(order);   // 补偿 Step 2
        case 2: inventoryService.restore(order); // 补偿 Step 1
        case 1: // 无需补偿
    }
}
```

## 三、RocketMQ 事务消息

```java
// RocketMQ 原生事务消息
public class TransactionProducer {
    public void sendTransactionMessage(Order order) {
        // 发送半消息（Prepare 状态）
        Message msg = new Message("OrderTopic", JSON.toJSONBytes(order));
        TransactionSendResult result = producer.sendMessageInTransaction(
            msg,
            // 本地事务执行器
            new LocalTransactionExecuter() {
                @Override
                public LocalTransactionState executeLocalTransactionBranch(
                        Message msg, Object arg) {
                    try {
                        // 执行本地事务
                        orderMapper.insert(order);
                        return LocalTransactionState.COMMIT_MESSAGE;
                    } catch (Exception e) {
                        return LocalTransactionState.ROLLBACK_MESSAGE;
                    }
                }
            },
            order
        );
    }
}

// 事务回查：Broker 定期回查事务状态
public class TransactionCheckListenerImpl implements TransactionCheckListener {
    @Override
    public LocalTransactionState checkLocalTransactionState(MessageExt msg) {
        Order order = JSON.parseObject(msg.getBody(), Order.class);
        // 查询本地事务是否完成
        if (orderMapper.exists(order.getOrderId())) {
            return LocalTransactionState.COMMIT_MESSAGE;
        }
        return LocalTransactionState.ROLLBACK_MESSAGE;
    }
}
```

## 四、TCC 模式

```java
// TCC: Try - Confirm - Cancel
@Service
public class OrderTccService {

    // Try: 预留资源
    @Transactional
    public void tryCreateOrder(Order order) {
        // 冻结库存
        inventoryMapper.freeze(order.getProductId(), order.getQuantity());
        // 冻结余额
        accountMapper.freeze(order.getUserId(), order.getAmount());
        // 创建待确认订单
        order.setStatus("TRYING");
        orderMapper.insert(order);
    }

    // Confirm: 确认提交
    @Transactional
    public void confirm(Long orderId) {
        Order order = orderMapper.selectById(orderId);
        // 扣减冻结库存
        inventoryMapper.confirmFreeze(order.getProductId(), order.getQuantity());
        // 扣减冻结余额
        accountMapper.confirmFreeze(order.getUserId(), order.getAmount());
        order.setStatus("CONFIRMED");
        orderMapper.updateById(order);
    }

    // Cancel: 取消回滚
    @Transactional
    public void cancel(Long orderId) {
        Order order = orderMapper.selectById(orderId);
        // 释放冻结库存
        inventoryMapper.releaseFreeze(order.getProductId(), order.getQuantity());
        // 释放冻结余额
        accountMapper.releaseFreeze(order.getUserId(), order.getAmount());
        order.setStatus("CANCELLED");
        orderMapper.updateById(order);
    }
}
```

## 五、分布式事务方案对比

```
方案          一致性    性能    复杂度    适用场景
2PC/XA        强       低      高       银行核心系统
TCC           最终     中      高       电商下单、支付
本地消息表      最终     高      低       通用异步场景
RocketMQ事务   最终     高      中       订单创建、库存扣减
Saga编排       最终     中      中       长流程业务
```

## 六、注意事项

1. **本地消息表是最简单可靠的方案**
2. **补偿操作要幂等**
3. **超时消息用于检测长时间未完成的事务**
4. **分布式事务会增加系统复杂度**
5. **尽量用最终一致性替代强一致性**
6. **事务超时要合理设置**，避免资源长时间锁定
7. **监控事务成功率**，异常升高要告警
