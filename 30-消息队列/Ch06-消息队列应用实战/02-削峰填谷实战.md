# 削峰填谷实战

## 一、秒杀场景

```java
// 秒杀接口 - 只做校验和入队
@PostMapping("/seckill")
public Result seckill(Long productId, Long userId) {
    // 1. 校验库存（Redis 预扣）
    if (!redisTemplate.opsForValue().decrement("stock:" + productId).isNegative()) {
        // 2. 入队等待
        SeckillRequest req = new SeckillRequest(productId, userId);
        rocketMQTemplate.syncSend("SeckillTopic", req);
        return Result.success("排队中");
    }
    return Result.fail("已售罄");
}

// 消费者 - 稳定速率处理
@Component
@RocketMQMessageListener(topic = "SeckillTopic", consumerGroup = "SeckillGroup")
public class SeckillConsumer implements RocketMQListener<SeckillRequest> {
    @Override
    public void onMessage(SeckillRequest req) {
        // 3. 创建订单
        orderService.createSeckillOrder(req.getProductId(), req.getUserId());
    }
}
```

## 二、流量控制

```java
// 使用 Semaphore 控制消费速率
@Component
public class RateLimitedConsumer {
    private final Semaphore semaphore = new Semaphore(1000); // 最多 1000 并发

    @KafkaListener(topics = "high-traffic-topic")
    public void consume(ConsumerRecord<String, String> record) {
        semaphore.acquire();
        try {
            process(record);
        } finally {
            semaphore.release();
        }
    }
}
```

## 三、Redis 预扣库存详解

```java
@Service
public class SeckillStockService {
    @Autowired
    private StringRedisTemplate redisTemplate;

    // 初始化库存到 Redis
    public void initStock(Long productId, int stock) {
        redisTemplate.opsForValue().set("stock:" + productId, String.valueOf(stock));
    }

    // Lua 脚本原子扣减库存
    private static final String DEDUCT_SCRIPT =
        "local stock = tonumber(redis.call('get', KEYS[1])) " +
        "if stock <= 0 then return -1 end " +
        "redis.call('decr', KEYS[1]) " +
        "return stock";

    public boolean deductStock(Long productId) {
        Long result = redisTemplate.execute(
            new DefaultRedisScript<>(DEDUCT_SCRIPT, Long.class),
            List.of("stock:" + productId)
        );
        return result != null && result > 0;
    }

    // 异步同步库存到数据库
    @Scheduled(fixedRate = 5000)
    public void syncStockToDB() {
        Set<String> keys = redisTemplate.keys("stock:*");
        if (keys != null) {
            for (String key : keys) {
                Long productId = Long.parseLong(key.split(":")[1]);
                int stock = Integer.parseInt(redisTemplate.opsForValue().get(key));
                inventoryMapper.updateStock(productId, stock);
            }
        }
    }
}
```

## 五、削峰填谷通用方案

```java
// 使用 Kafka 做流量削峰
@Component
public class TrafficShapingConsumer {
    // 控制消费速率
    private final RateLimiter consumerRateLimit = RateLimiter.create(500);

    @KafkaListener(topics = "high-traffic", concurrency = "4")
    public void consume(ConsumerRecord<String, String> record) {
        consumerRateLimit.acquire();  // 限制每秒处理量
        try {
            BusinessEvent event = JSON.parseObject(record.value(), BusinessEvent.class);
            processEvent(event);
        } catch (Exception e) {
            log.error("处理失败，进入重试队列", e);
            retryService.retry(record);
        }
    }
}
```

## 六、注意事项

1. **削峰的核心是把瞬时流量平滑化**
2. **Redis 预扣库存是秒杀的关键**
3. **消费端要以稳定速率处理**，不要被流量冲垮
4. **排队体验要做好**，给用户反馈等待状态
5. **监控消费延迟**，积压过多时要及时扩容
6. **Redis 和数据库之间要定期同步**，避免库存不一致
7. **多级限流**：Nginx -> 网关 -> Redis -> MQ，逐级拦截
