# 实时计算管道

## 一、Kafka + Flink

```
数据源 -> Kafka -> Flink -> 结果输出
                           ├── 数据库
                           ├── Kafka
                           └── 可视化
```

```java
// Flink 从 Kafka 读取
StreamExecutionEnvironment env = StreamExecutionEnvironment.getExecutionEnvironment();
env.enableCheckpointing(60000);

Properties props = new Properties();
props.setProperty("bootstrap.servers", "localhost:9092");
props.setProperty("group.id", "flink-group");

DataStream<String> stream = env.addSource(
    new FlinkKafkaConsumer<>("click-events", new SimpleStringSchema(), props)
);

// 实时统计
stream
    .map(value -> JSON.parseObject(value, ClickEvent.class))
    .keyBy(ClickEvent::getUserId)
    .window(TumblingEventTimeWindows.of(Time.minutes(5)))
    .aggregate(new ClickCountAggregator())
    .addSink(new FlinkKafkaProducer<>("click-counts", new StringSerializer(), props));
```

## 二、Kafka Streams 实时计算

```java
// Kafka Streams 实时聚合
public class ClickStreamProcessor {
    public static void main(String[] args) {
        Properties props = new Properties();
        props.put(StreamsConfig.APPLICATION_ID_CONFIG, "click-stream-app");
        props.put(StreamsConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");

        StreamsBuilder builder = new StreamsBuilder();

        // 从 Kafka 读取点击流
        KStream<String, ClickEvent> clicks = builder.stream("click-events",
            Consumed.with(Serdes.String(), JsonSerdes.ClickEvent()));

        // 按用户分组，5 分钟窗口统计
        KTable<Windowed<String>, Long> clickCounts = clicks
            .groupBy((userId, click) -> click.getUserId(),
                     Grouped.with(Serdes.String(), JsonSerdes.ClickEvent()))
            .windowedBy(TimeWindows.ofSizeWithNoGrace(Duration.ofMinutes(5)))
            .count();

        // 输出到结果 Topic
        clickCounts.toStream()
            .map((windowedKey, count) -> KeyValue.pair(
                windowedKey.key(),
                new ClickCount(windowedKey.key(), count, windowedKey.window().start())
            ))
            .to("click-counts", Produced.with(Serdes.String(), JsonSerdes.ClickCount()));

        KafkaStreams streams = new KafkaStreams(builder.build(), props);
        streams.start();
    }
}
```

## 三、Flink Watermark 配置

```java
// 处理乱序数据的 Watermark 策略
DataStream<ClickEvent> stream = env.addSource(kafkaSource)
    .assignTimestampsAndWatermarks(
        WatermarkStrategy.<ClickEvent>forBoundedOutOfOrderness(Duration.ofSeconds(30))
            .withTimestampAssigner((event, timestamp) -> event.getEventTime())
            .withIdleness(Duration.ofMinutes(1))  // 处理空闲分区
    );

// 事件时间窗口聚合
stream
    .keyBy(ClickEvent::getUserId)
    .window(TumblingEventTimeWindows.of(Time.minutes(5)))
    .allowedLateness(Time.minutes(1))        // 允许 1 分钟迟到
    .sideOutputLateData(lateOutputTag)       // 迟到数据旁路输出
    .aggregate(new ClickCountAggregator());
```

## 四、管道架构设计

```
              实时计算管道架构
    ┌────────────────────────────────────────────┐
    │  数据源                                      │
    │  ├── 用户点击事件                             │
    │  ├── 订单事件                                │
    │  └── 日志事件                                │
    │      ↓                                      │
    │  Kafka (原始数据 Topic)                       │
    │      ↓                                      │
    │  Flink / Kafka Streams                       │
    │  ├── 窗口聚合 (5分钟/1小时)                    │
    │  ├── 流式 Join (订单 + 用户)                   │
    │  └── CEP 复杂事件处理                         │
    │      ↓                                      │
    │  输出                                        │
    │  ├── Kafka (聚合结果)                         │
    │  ├── ClickHouse (实时报表)                    │
    │  ├── Redis (实时看板)                         │
    │  └── 告警系统 (异常检测)                       │
    └────────────────────────────────────────────┘
```

## 五、注意事项

1. **Kafka 是 Flink 最常用的数据源**
2. **Checkpoint 保证 Exactly-once**
3. **水印（Watermark）处理乱序数据**
4. **背压机制防止 Flink 被 Kafka 冲垮**
5. **监控消费延迟和计算延迟**
6. **窗口大小要根据业务需求选择**，小窗口实时性好但开销大
7. **迟到数据的处理策略要明确**：丢弃、旁路输出或重新计算
