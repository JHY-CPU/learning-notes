# 日志收集实战

## 一、架构设计

```
App Log -> Filebeat -> Kafka -> Logstash -> Elasticsearch -> Kibana
                              -> Flink -> 实时告警
```

## 二、Kafka 日志收集

```java
// 日志 Producer
@Component
public class LogProducer {
    @Autowired
    private KafkaTemplate<String, String> kafkaTemplate;

    public void sendLog(LogEntry entry) {
        String key = entry.getServiceName() + "-" + entry.getInstanceId();
        kafkaTemplate.send("app-logs", key, JSON.toJSONString(entry));
    }
}

// 日志格式
public class LogEntry {
    private String traceId;       // 链路 ID
    private String serviceName;   // 服务名
    private String level;         // 日志级别
    private String message;       // 日志内容
    private Long timestamp;       // 时间戳
    private Map<String, String> context;  // 上下文
}
```

## 三、注意事项

1. **日志 Topic 分区数要根据日志量设置**
2. **日志保留时间根据合规要求设置**（通常 7-30 天）
3. **使用压缩减少存储和网络开销**
4. **敏感信息要脱敏后再发送**
5. **日志消费异常不要影响业务**
