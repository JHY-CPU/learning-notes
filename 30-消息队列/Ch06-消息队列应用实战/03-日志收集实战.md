# 日志收集实战

## 一、架构设计

```
App Log -> Filebeat -> Kafka -> Logstash -> Elasticsearch -> Kibana
                              -> Flink -> 实时告警
```

## 二、Kafka 日志收集

```java
// 日志 Producer
@Component
public class LogProducer {
    @Autowired
    private KafkaTemplate<String, String> kafkaTemplate;

    public void sendLog(LogEntry entry) {
        String key = entry.getServiceName() + "-" + entry.getInstanceId();
        kafkaTemplate.send("app-logs", key, JSON.toJSONString(entry));
    }
}

// 日志格式
public class LogEntry {
    private String traceId;       // 链路 ID
    private String serviceName;   // 服务名
    private String level;         // 日志级别
    private String message;       // 日志内容
    private Long timestamp;       // 时间戳
    private Map<String, String> context;  // 上下文
}
```

## 三、Filebeat 配置

```yaml
# filebeat.yml
filebeat.inputs:
  - type: log
    enabled: true
    paths:
      - /var/log/app/*.log
    fields:
      service: order-service
      env: production
    multiline:
      pattern: '^\d{4}-\d{2}-\d{2}'
      negate: true
      match: after

output.kafka:
  hosts: ["kafka-1:9092", "kafka-2:9092"]
  topic: "app-logs"
  partition.round_robin:
    reachable_only: true
  required_acks: 1
  compression: snappy

processors:
  - drop_fields:
      fields: ["agent", "ecs", "host"]
```

## 四、日志查询与分析

```java
// Logstash 配置（过滤 + 转换）
input {
  kafka {
    bootstrap_servers => "kafka:9092"
    topics => ["app-logs"]
    group_id => "logstash-group"
  }
}

filter {
  grok {
    match => { "message" => "%{TIMESTAMP_ISO8601:timestamp} %{LOGLEVEL:level} %{GREEDYDATA:msg}" }
  }
  date {
    match => ["timestamp", "yyyy-MM-dd HH:mm:ss.SSS"]
    target => "@timestamp"
  }
  mutate {
    remove_field => ["timestamp"]
  }
}

output {
  elasticsearch {
    hosts => ["es:9200"]
    index => "app-logs-%{+YYYY.MM.dd}"
  }
}
```

## 五、实时告警

```java
// Flink 实时日志告警
public class LogAlertJob {
    public static void main(String[] args) throws Exception {
        StreamExecutionEnvironment env = StreamExecutionEnvironment.getExecutionEnvironment();

        Properties props = new Properties();
        props.setProperty("bootstrap.servers", "kafka:9092");

        DataStream<LogEntry> logs = env
            .addSource(new FlinkKafkaConsumer<>("app-logs", new LogEntryDeserializer(), props));

        // ERROR 级别告警
        logs.filter(log -> "ERROR".equals(log.getLevel()))
            .keyBy(LogEntry::getServiceName)
            .window(TumblingProcessingTimeWindows.of(Time.minutes(1)))
            .aggregate(new ErrorCountAggregator())
            .filter(result -> result.count > 10)  // 1 分钟内超过 10 个 ERROR
            .addSink(new DingTalkAlertSink());

        env.execute("Log Alert Job");
    }
}
```

## 六、注意事项

1. **日志 Topic 分区数要根据日志量设置**
2. **日志保留时间根据合规要求设置**（通常 7-30 天）
3. **使用压缩减少存储和网络开销**
4. **敏感信息要脱敏后再发送**
5. **日志消费异常不要影响业务**
6. **Filebeat 配置多行合并**，避免堆栈信息被拆分
7. **日志量大时考虑分 Topic 或分集群**
