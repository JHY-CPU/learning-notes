# 消息幂等实战

## 一、Redis 幂等

```java
@Service
public class IdempotentService {
    @Autowired
    private StringRedisTemplate redisTemplate;

    public boolean tryProcess(String messageId) {
        return Boolean.TRUE.equals(
            redisTemplate.opsForValue()
                .setIfAbsent("msg:idem:" + messageId, "1", 24, TimeUnit.HOURS)
        );
    }

    public void processWithIdempotent(String messageId, Order order) {
        if (!tryProcess(messageId)) {
            log.info("消息已处理，跳过: {}", messageId);
            return;
        }
        try {
            orderService.process(order);
        } catch (Exception e) {
            redisTemplate.delete("msg:idem:" + messageId);
            throw e;
        }
    }
}
```

-- 唯一索引去重
CREATE TABLE message_record (
    message_id VARCHAR(64) PRIMARY KEY,

    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 插入时捕获重复异常
INSERT INTO message_record (message_id, status) VALUES (?, 'PROCESSED')
ON DUPLICATE KEY UPDATE status = 'PROCESSED';

## 四、幂等方案对比

```
方案            可靠性    性能    复杂度    适用场景
Redis SETNX     中       高      低       高频、短时效幂等
数据库唯一索引    高       中      低       关键业务、需持久化
状态机           高       高      中       有明确状态流转的业务
Token 机制      高       中      中       用户操作幂等（防重复提交）
乐观锁          高       高      低       更新操作幂等
```

## 五、Token 机制幂等

```java
// 前端提交时获取 Token，后端校验 Token 实现幂等
@RestController
public class IdempotentController {

    // 生成幂等 Token
    @GetMapping("/idempotent-token")
    public String generateToken() {
        String token = UUID.randomUUID().toString();
        redisTemplate.opsForValue().set("idempotent:token:" + token, "1",
            10, TimeUnit.MINUTES);
        return token;
    }

    // 校验 Token 并处理
    @PostMapping("/order/create")
    public Result createOrder(@RequestHeader("Idempotent-Token") String token,
                              @RequestBody OrderRequest request) {
        // Lua 脚本原子校验并删除 Token
        String script =
            "if redis.call('exists', KEYS[1]) == 1 then " +
            "  redis.call('del', KEYS[1]) return 1 " +
            "else return 0 end";

        Long result = redisTemplate.execute(
            new DefaultRedisScript<>(script, Long.class),
            List.of("idempotent:token:" + token)
        );

        if (result == null || result == 0) {
            return Result.fail("重复请求");
        }

        // 处理业务
        orderService.create(request);
        return Result.success();
    }
}
```

## 六、乐观锁幂等

```java
// 使用版本号实现幂等更新
@Mapper
public interface OrderMapper {
    @Update("UPDATE orders SET status = #{newStatus}, version = version + 1 " +
            "WHERE order_id = #{orderId} AND version = #{expectedVersion}")
    int updateWithVersion(@Param("orderId") Long orderId,
                          @Param("newStatus") String newStatus,
                          @Param("expectedVersion") int expectedVersion);
}

// 调用
public void updateOrderStatus(Long orderId, String newStatus) {
    Order order = orderMapper.selectById(orderId);
    int rows = orderMapper.updateWithVersion(orderId, newStatus, order.getVersion());
    if (rows == 0) {
        log.info("版本不匹配，可能已被更新，跳过");
    }
}
```

## 七、注意事项

1. **幂等 Key 要全局唯一**，使用消息 ID
2. **处理失败时清除幂等标记**
3. **Redis 幂等要设置过期时间**
4. **数据库幂等用唯一索引最可靠**
5. **状态机幂等适合有明确状态流转的业务**
6. **幂等 Key 的过期时间要大于最大重试时间**
7. **并发场景下使用 Lua 脚本保证原子性**
