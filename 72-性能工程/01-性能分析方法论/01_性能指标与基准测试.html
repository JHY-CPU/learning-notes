性能指标与基准测试

性能工程延迟吞吐量

理解性能指标和分析方法是性能优化的前提。

核心性能指标

延迟 (Latency)： - p50 (中位数)：50%的请求在此时间内完成 - p90：90%的请求在此时间内完成 - p95 / p99 / p999：尾部延迟 - 平均延迟：容易被极端值影响，不推荐 为什么关注尾部延迟？ - p99=100ms：1%的用户体验差 - 日活100万：1万用户受影响 - 尾部延迟往往反映系统最差情况 吞吐量 (Throughput)： - TPS (Transactions Per Second)：每秒事务数 - QPS (Queries Per Second)：每秒查询数 - RPS (Requests Per Second)：每秒请求数 - 带宽 (Bandwidth)：每秒数据量 利用率 (Utilization)： - CPU利用率 - 内存使用率 - 磁盘I/O - 网络带宽 USE方法 (Utilization, Saturation, Errors)： 对每个资源检查： - Utilization：使用率 - Saturation：饱和度（排队） - Errors：错误数

基准测试

常用压测工具： ┌──────────┬──────────────────────────────────┐ │ 工具 │ 用途 │ ├──────────┼──────────────────────────────────┤ │ wrk/wrk2 │ HTTP压测，高并发 │ │ ab │ Apache Bench，简单HTTP压测 │ │ JMeter │ 功能+性能测试，GUI │ │ Gatling │ Scala压测，报告美观 │ │ k6 │ JS脚本压测，云原生 │ │ Locust │ Python压测，分布式 │ │ sysbench │ MySQL/系统基准测试 │ │ fio │ 磁盘I/O基准测试 │ └──────────┴──────────────────────────────────┘ 基准测试注意事项： - 预热(warm-up)：JIT、缓存需要预热 - 足够长的测试时间（>5分钟） - 多次测试取中位数 - 隔离测试环境 - 监控系统资源使用 - 记录完整的测试条件 性能分析方法论： 1. 设定性能目标（SLI/SLO） 2. 基准测试当前性能 3. 识别瓶颈（USE方法） 4. 优化瓶颈 5. 验证优化效果 6. 回到步骤2

性能分析工具

系统级工具： - top/htop：CPU和内存使用 - vmstat：虚拟内存统计 - iostat：磁盘I/O统计 - netstat/ss：网络连接 - dstat：综合系统资源 - perf：Linux性能分析（CPU火焰图） 应用级工具： - Java：JProfiler, VisualVM, async-profiler - Go：pprof - Node.js：--inspect, clinic.js - Python：cProfile, py-spy APM工具： - Jaeger/Zipkin：分布式追踪 - Prometheus + Grafana：指标监控 - Datadog/New Relic：商业APM - SkyWalking：国产APM 火焰图 (Flame Graph)： - 可视化CPU时间花费 - 横轴：函数调用比例 - 纵轴：调用栈深度 - 快速定位热点函数
性能优化必须基于数据：先测量，找到瓶颈，再优化。不要凭直觉优化。 USE方法是系统性能分析的基本框架。
