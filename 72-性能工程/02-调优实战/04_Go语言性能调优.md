# Go语言性能调优


## Go语言性能调优


性能工程Gopprof


Go以高性能著称，但生产环境仍需系统性调优。pprof是Go最强大的性能分析工具。


## pprof 性能分析


```
pprof 工具链：

1. CPU Profiling（CPU分析）：
import _ "net/http/pprof"

func main() {
  go func() { log.Println(http.ListenAndServe("localhost:6060", nil)) }()
  // ... 业务代码
}

# 采集30秒CPU profile
go tool pprof http://localhost:6060/debug/pprof/profile?seconds=30

# 交互式分析
(pprof) top 10          # 显示最耗CPU的10个函数
(pprof) list funcName   # 显示函数逐行耗时
(pprof) web             # 生成调用图（需安装graphviz）

2. Memory Profiling（内存分析）：
# 采集堆内存 profile
go tool pprof http://localhost:6060/debug/pprof/heap

# 查看内存分配最多的函数
(pprof) top -cum        # 按累计值排序
(pprof) alloc_objects   # 按对象数
(pprof) alloc_space     # 按分配空间

3. Goroutine Profiling（协程分析）：
# 查看所有goroutine的堆栈
go tool pprof http://localhost:6060/debug/pprof/goroutine

# Goroutine泄露检测
curl http://localhost:6060/debug/pprof/goroutine?debug=1

4. Block Profiling（阻塞分析）：
import "runtime"
runtime.SetBlockProfileRate(1)  # 记录所有阻塞事件

go tool pprof http://localhost:6060/debug/pprof/block

5. Mutex Profiling（锁竞争分析）：
runtime.SetMutexProfileFraction(1)

go tool pprof http://localhost:6060/debug/pprof/mutex

在代码中直接使用：
import "runtime/pprof"

f, _ := os.Create("cpu.prof")
pprof.StartCPUProfile(f)
defer pprof.StopCPUProfile()

# 运行后分析
go tool pprof -http=:8080 cpu.prof  # 浏览器可视化
```


## Goroutine泄漏检测


```
常见Goroutine泄漏场景：

1. Channel无人读取/写入：
// 泄漏：ch没有接收者，goroutine永远阻塞
func leak() {
  ch := make(chan int)
  go func() { ch <- 1 }()  // 阻塞，永远不退出
  // 忘记读取 ch
}

// 修复：使用buffered channel或context
func fixed() {
  ch := make(chan int, 1)  // buffered
  go func() { ch <- 1 }()
}

2. 未关闭的HTTP响应体：
// 泄漏
resp, _ := http.Get(url)
// 忘记 resp.Body.Close()

// 修复
resp, err := http.Get(url)
if err != nil { return err }
defer resp.Body.Close()

3. Context未正确使用：
// 泙漏：context取消后goroutine仍在等待
func leak(ctx context.Context) {
  ch := make(chan Result)
  go func() { ch <- doWork() }()

  select {
  case r := <-ch: return r
  case <-ctx.Done(): return nil  // goroutine仍在等待！
  }
}

// 修复：使用带buffer的channel
func fixed(ctx context.Context) {
  ch := make(chan Result, 1)  // buffered!
  go func() { ch <- doWork() }()

  select {
  case r := <-ch: return r
  case <-ctx.Done(): return nil
  }
}

4. 未停止的Ticker：
// 泄漏
ticker := time.NewTicker(time.Second)
go func() {
  for range ticker.C { doWork() }
}()
// 忘记 ticker.Stop()

// 修复
defer ticker.Stop()

检测工具：
- runtime.NumGoroutine()：监控goroutine数量
- uber-go/goleak：测试中检测泄漏
  func TestMain(m *testing.M) {
    goleak.VerifyTestMain(m)
  }
```


## GC调优与逃逸分析


```
GC调优（GOGC环境变量）：

GOGC控制GC触发频率：
- GOGC=100（默认）：堆增长100%时触发GC
- GOGC=200：堆增长200%时触发GC（减少GC频率）
- GOGC=50：堆增长50%时触发GC（增加GC频率）
- GOGC=off：禁用GC（慎用）

# 设置GOGC
export GOGC=200
# 或代码中设置
debug.SetGCPercent(200)

GC调优策略：
1. 减少堆分配（逃逸分析）
2. 复用对象（sync.Pool）
3. 增大GOGC（减少GC频率，但增大内存使用）
4. 分析GC日志：GODEBUG=gctrace=1

逃逸分析：
go build -gcflags="-m" main.go 2>&1 | grep "escapes"

常见逃逸场景：
// 逃逸到堆
func newUser() *User {
  u := User{Name: "test"}  // escapes to heap
  return &u                // 返回局部变量的指针
}

// 不逃逸（栈分配）
func process() User {
  u := User{Name: "test"}  // 栈上分配
  return u                 // 值返回，不逃逸
}

// 闭包导致逃逸
func handler() func() int {
  x := 42
  return func() int { return x }  // x逃逸到堆
}

// 接口导致逃逸
func fmtPrint(v interface{}) {
  fmt.Println(v)  // 传入的值逃逸（interface{}参数）
}

sync.Pool 对象复用：
var bufPool = sync.Pool{
  New: func() interface{} {
    return make([]byte, 0, 4096)
  },
}

func handler(w http.ResponseWriter, r *http.Request) {
  buf := bufPool.Get().([]byte)
  defer bufPool.Put(buf[:0])  // 复位后放回

  // 使用buf处理请求...
  buf = append(buf, "response"...)
  w.Write(buf)
}

注意：
- Pool中的对象可能随时被GC回收
- 适用于高频创建/销毁的临时对象
- 不要放入有状态的对象
```


## 性能优化最佳实践


```
内存优化：

1. 预分配容量（减少扩容）：
// 未知容量：多次扩容
var items []Item
for _, id := range ids {
  items = append(items, getItem(id))  // 可能多次扩容
}

// 已知容量：一次分配
items := make([]Item, 0, len(ids))  // 预分配

2. 字符串拼接优化：
// 慢：每次拼接都创建新字符串
s := ""
for _, item := range items {
  s += item.Name + ","  // O(n²)
}

// 快：使用 strings.Builder
var b strings.Builder
b.Grow(estimatedSize)  // 预分配
for _, item := range items {
  b.WriteString(item.Name)
  b.WriteByte(',')
}

3. 减少map的interface{}：
// 慢：类型转换开销
m := make(map[string]interface{})
name := m["name"].(string)

// 快：类型明确
m := make(map[string]string)
name := m["name"]

并发优化：

1. Worker Pool模式：
func workerPool(jobs <-chan Job, numWorkers int) {
  var wg sync.WaitGroup
  for i := 0; i < numWorkers; i++ {
    wg.Add(1)
    go func() {
      defer wg.Done()
      for job := range jobs {
        process(job)
      }
    }()
  }
  wg.Wait()
}

2. 批量处理减少锁竞争：
// 慢：每条记录加锁
for _, item := range items {
  mu.Lock()
  result = append(result, process(item))
  mu.Unlock()
}

// 快：批量处理后一次性加锁
batch := make([]Result, 0, len(items))
for _, item := range items {
  batch = append(batch, process(item))
}
mu.Lock()
result = append(result, batch...)
mu.Unlock()

性能优化检查清单：
□ 使用pprof找到真正的瓶颈
□ 检查goroutine泄漏（NumGoroutine监控）
□ 分析逃逸报告，减少堆分配
□ 预分配slice和map容量
□ 使用sync.Pool复用高频对象
□ 批量操作减少锁竞争
□ 使用strings.Builder拼接字符串
□ 避免在热路径中使用反射和interface{}
```


> **Note:** Go调优核心工具是pprof（CPU/内存/协程/阻塞分析）。逃逸分析(-gcflags="-m")帮助理解哪些变量分配在堆上。sync.Pool适合高频临时对象复用。GOGC调整GC频率，需权衡内存和CPU。
