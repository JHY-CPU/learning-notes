# 前端性能优化


## 前端性能优化


性能工程前端Core Web Vitals


前端性能直接影响用户感知体验，是业务指标（转化率）的关键因素。


## Core Web Vitals


```
Google Core Web Vitals（核心网页指标）：

┌──────────┬──────────┬──────────┬──────────────────────────┐
│ 指标       │ 含义      │ 目标      │ 优化方向                  │
├──────────┼──────────┼──────────┼──────────────────────────┤
│ LCP       │ 最大内容  │ ≤ 2.5s   │ 图片优化、CDN、服务端渲染  │
│           │ 绘制时间  │          │                          │
├──────────┼──────────┼──────────┼──────────────────────────┤
│ INP       │ 交互到下  │ ≤ 200ms  │ 减少主线程阻塞、代码拆分   │
│           │ 一次绘制  │          │                          │
├──────────┼──────────┼──────────┼──────────────────────────┤
│ CLS       │ 累积布局  │ ≤ 0.1    │ 预设图片尺寸、字体优化     │
│           │ 偏移      │          │                          │
└──────────┴──────────┴──────────┴──────────────────────────┘

LCP优化：
- 图片：使用WebP/AVIF格式，设置width/height属性
- 预加载：<link rel="preload" href="hero.webp" as="image">
- CDN：静态资源使用CDN分发
- SSR/SSG：首屏内容服务端渲染
- 移除渲染阻塞资源

INP优化：
- 拆分长任务：使用setTimeout/requestIdleCallback
- Web Workers：将计算密集型任务移至Worker
- 虚拟列表：大列表使用react-window
- 防抖/节流：高频事件（scroll/resize）

CLS优化：
- 图片/视频预留空间：width + height属性
- 字体优化：font-display: swap + preload
- 动态内容：预留占位空间
- 避免在已有内容上方插入新内容

测量工具：
- Chrome DevTools > Performance面板
- Lighthouse审计
- web-vitals npm库
- Google Search Console (CrUX数据)
```


## 关键渲染路径优化


```
浏览器渲染流程：
HTML → DOM树
         ↘
          渲染树 → 布局(Layout) → 绘制(Paint) → 合成(Composite)
         ↗
CSS → CSSOM树

关键渲染路径(CRP)优化：

1. 减少关键资源数量
- 内联关键CSS（首屏样式）
- 异步加载非关键CSS：
  <link rel="preload" href="style.css" as="style"
        onload="this.rel='stylesheet'">
- 异步加载JS：
  <script src="app.js" defer></script>
  <script src="analytics.js" async></script>

2. 减小关键资源大小
- CSS压缩 + Tree Shaking
- JS压缩 + Code Splitting
- 图片压缩 + 响应式图片

3. 缩短CRP长度
- 内联关键JS/CSS（减少请求）
- 预连接：<link rel="preconnect" href="https://api.example.com">
- 预加载：<link rel="preload" href="main.js" as="script">

代码拆分策略 (React/Vue)：
// 路由级拆分（最常见）
const Home = lazy(() => import('./pages/Home'));
const Dashboard = lazy(() => import('./pages/Dashboard'));

// 组件级拆分（重型组件）
const Chart = lazy(() => import('./components/HeavyChart'));

// 条件拆分
if (user.isAdmin) {
  const AdminPanel = await import('./AdminPanel');
}

Webpack SplitChunks 配置：
optimization: {
  splitChunks: {
    chunks: 'all',
    cacheGroups: {
      vendor: {
        test: /[\\/]node_modules[\\/]/,
        name: 'vendors',
        chunks: 'all',
        priority: 10,
      },
      common: {
        minChunks: 2,
        priority: 5,
        reuseExistingChunk: true,
      },
    },
  },
},
```


## 资源优化策略


```
图片优化（通常占页面60-70%流量）：

格式选择：
┌──────────┬──────────┬──────────┬──────────────────────┐
│ 格式       │ 压缩      │ 透明      │ 适用场景              │
├──────────┼──────────┼──────────┼──────────────────────┤
│ JPEG      │ 有损      │ 不支持    │ 照片类图片            │
│ PNG       │ 无损      │ 支持      │ Logo、图标、需要透明  │
│ WebP      │ 有损+无损  │ 支持      │ 现代浏览器首选        │
│ AVIF      │ 有损+无损  │ 支持      │ 最新格式，压缩率最高  │
│ SVG       │ 矢量      │ 支持      │ 图标、简单图形        │
└──────────┴──────────┴──────────┴──────────────────────┘

响应式图片：
<picture>
  <source srcset="hero.avif" type="image/avif">
  <source srcset="hero.webp" type="image/webp">
  <img src="hero.jpg" alt="Hero" width="1200" height="600"
       loading="lazy" decoding="async">
</picture>

懒加载：
- 原生：<img loading="lazy" src="...">
- Intersection Observer：
  const observer = new IntersectionObserver((entries) => {
    entries.forEach(entry => {
      if (entry.isIntersecting) {
        entry.target.src = entry.target.dataset.src;
        observer.unobserve(entry.target);
      }
    });
  });
  document.querySelectorAll('img[data-src]').forEach(img => {
    observer.observe(img);
  });

缓存策略：
# 静态资源（带哈希）
Cache-Control: public, max-age=31536000, immutable

# HTML文档
Cache-Control: no-cache  # 每次验证新鲜度

# API响应
Cache-Control: private, max-age=0, must-revalidate

Service Worker缓存：
// Stale-While-Revalidate策略
self.addEventListener('fetch', (event) => {
  event.respondWith(
    caches.open('v1').then(cache =>
      cache.match(event.request).then(cached => {
        const fetched = fetch(event.request).then(response => {
          cache.put(event.request, response.clone());
          return response;
        });
        return cached || fetched;
      })
    )
  );
});

字体优化：
/* 预加载关键字体 */
<link rel="preload" href="/fonts/main.woff2" as="font" crossorigin>

/* font-display策略 */
@font-face {
  font-family: 'CustomFont';
  src: url('/fonts/main.woff2') format('woff2');
  font-display: swap; /* 先用系统字体，加载后替换 */
  unicode-range: U+4E00-9FFF; /* 只加载需要的字符集 */
}

/* 中文字体子集化 */
// 使用 font-spider 或 cn-font-split
// 将50MB的中文字体拆分为按需加载的小文件
```


> **Note:** 前端优化优先级：1) 图片优化（WebP/AVIF + 懒加载） 2) 代码拆分（路由级懒加载） 3) 关键渲染路径（内联关键CSS + defer JS） 4) 缓存策略（长期缓存 + 哈希）。Core Web Vitals的LCP、INP、CLS是Google排名因素，直接影响SEO。
