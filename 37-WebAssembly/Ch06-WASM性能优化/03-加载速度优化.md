# 加载速度优化

## 一、概念说明

WASM 加载包括下载、编译、实例化三个阶段。优化策略针对每个阶段。

```javascript
// 流式编译 - 边下载边编译
const module = await WebAssembly.compileStreaming(fetch('module.wasm'));
const instance = await WebAssembly.instantiate(module, imports);
```

## 二、具体用法

### 2.1 流式编译

```javascript
// 最快：流式编译（需要正确 MIME 类型）
const module = await WebAssembly.compileStreaming(
    fetch('module.wasm', {
        headers: { 'Content-Type': 'application/wasm' }
    })
);
const instance = await WebAssembly.instantiate(module, imports);

// 先下载后编译（较慢）
const bytes = await fetch('module.wasm').then(r => r.arrayBuffer());
const module = await WebAssembly.compile(bytes);
```

### 2.2 预编译与缓存

```javascript
// 预编译模块（在空闲时）
if ('requestIdleCallback' in window) {
    requestIdleCallback(async () => {
        const module = await WebAssembly.compileStreaming(
            fetch('module.wasm')
        );
        // 缓存编译结果
        wasmCache.set('module', module);
    });
}

// 使用时直接实例化
async function getWasmInstance(name) {
    const cached = wasmCache.get(name);
    if (cached) {
        return await WebAssembly.instantiate(cached, imports);
    }
    // 回退到完整流程
    const { instance } = await WebAssembly.instantiateStreaming(
        fetch(`${name}.wasm`), imports
    );
    return instance;
}
```

### 2.3 代码分割与懒加载

```javascript
// 按需加载 WASM 模块
const modules = {};

async function loadModule(name) {
    if (modules[name]) return modules[name];

    const bytes = await fetch(`modules/${name}.wasm`).then(r => r.arrayBuffer());
    const { instance } = await WebAssembly.instantiate(bytes);
    modules[name] = instance;
    return instance;
}

// 用法
document.getElementById('image-btn').addEventListener('click', async () => {
    const imageModule = await loadModule('image-processing');
    imageModule.exports.apply_filter(buffer);
});
```

### 2.4 压缩传输

```javascript
// 服务端压缩，客户端解压
async function loadCompressedWasm(url) {
    const resp = await fetch(url);
    const ds = new DecompressionStream('gzip');
    const decompressedStream = resp.body.pipeThrough(ds);
    const bytes = await new Response(decompressedStream).arrayBuffer();
    return await WebAssembly.compile(bytes);
}
```

### 2.5 实例化优化

```javascript
// 复用 importObject
const sharedImports = {
    env: {
        memory: new WebAssembly.Memory({ initial: 256 }),
        abort: () => { throw new Error('WASM abort'); }
    }
};

// 批量实例化
const [mod1, mod2, mod3] = await Promise.all([
    WebAssembly.compileStreaming(fetch('core.wasm')),
    WebAssembly.compileStreaming(fetch('math.wasm')),
    WebAssembly.compileStreaming(fetch('io.wasm')),
]);

const [inst1, inst2, inst3] = await Promise.all([
    WebAssembly.instantiate(mod1, sharedImports),
    WebAssembly.instantiate(mod2, sharedImports),
    WebAssembly.instantiate(mod3, sharedImports),
]);
```

## 三、注意事项与常见陷阱

1. **MIME 类型**：compileStreaming 需要 `application/wasm` Content-Type
2. **HTTPS 要求**：流式编译需要 HTTPS（或 localhost）
3. **编译缓存**：浏览器可能缓存编译结果，但不可依赖
4. **首屏优化**：关键 WASM 模块优先加载，非关键延迟
5. **Service Worker**：可通过 SW 进一步缓存 WASM 资源
