# 性能分析工具

## 一、概念说明

WASM 性能分析需要结合浏览器开发者工具和专用工具。Chrome DevTools 提供最完整的支持。

## 二、具体用法

### 2.1 Chrome DevTools 配置

```
1. 打开 Chrome DevTools → Settings → Experiments
2. 勾选 "WebAssembly Debugging: Enable DWARF support"
3. 构建时启用调试信息：
   RUSTFLAGS="-g" wasm-pack build --dev
4. 在 Sources 面板中可看到 Rust 源码
```

### 2.2 性能面板分析

```javascript
// 使用 Performance API 测量
const start = performance.now();
instance.exports.heavy_computation();
const elapsed = performance.now() - start;
console.log(`计算耗时: ${elapsed.toFixed(2)}ms`);

// WASM 内部测量
const memory = new WebAssembly.Memory({ initial: 1 });
```

### 2.3 内存分析

```rust
#[wasm_bindgen]
pub fn log_memory_stats() {
    let pages = core::arch::wasm32::memory_size::<0>();
    let bytes = pages * 65536;
    web_sys::console::log_1(
        &format!("WASM 内存: {} 页 / {} KB", pages, bytes / 1024).into()
    );
}
```

```javascript
// JavaScript 侧内存分析
const memory = instance.exports.memory;
console.log(`当前内存: ${memory.buffer.byteLength} bytes`);

// 监听内存增长
const originalGrow = memory.grow.bind(memory);
memory.grow = function(pages) {
    console.log(`内存增长: +${pages} 页`);
    return originalGrow(pages);
};
```

### 2.4 Chrome Tracing

```rust
// 使用 console.time/timeEnd 测量
fn measure(label: &str, f: impl FnOnce()) {
    let console = web_sys::window().unwrap()
        .document().unwrap()
        .get_element_by_id("console").unwrap();

    web_sys::console::time_with_label(label);
    f();
    web_sys::console::time_end_with_label(label);
}

pub fn profile() {
    measure("数据加载", || load_data());
    measure("数据处理", || process_data());
    measure("渲染", || render());
}
```

### 2.5 使用 twiggy 分析二进制

```bash
# 安装 twiggy
cargo install twiggy

# 分析 WASM 文件大小组成
twiggy top my_app_bg.wasm

# 按函数查看
twiggy top my_app_bg.wasm -n 20

# 查看调用图
twiggy dominators my_app_bg.wasm

# 查看未使用代码
twiggy garbage my_app_bg.wasm
```

### 2.6 wasm-pack 大小报告

```bash
# 生成包大小报告
wasm-pack build --release -- --features "wee_alloc"

# 比较不同优化级别
for opt in "0" "1" "2" "3" "s" "z"; do
    RUSTFLAGS="-C opt-level=$opt" wasm-pack build --release
    echo "opt-level=$opt: $(du -sh pkg/my_app_bg.wasm)"
done
```

## 三、注意事项与常见陷阱

1. **调试构建**：dev 构建性能不代表 release，必须测试 release 构建
2. **源码映射**：需要启用 DWARF 调试信息才能看到 Rust 源码
3. **采样精度**：Chrome 性能面板的采样间隔可能错过热点
4. **内存泄漏**：使用 DevTools Memory 面板的 Heap Snapshot 功能
5. **多目标分析**：在不同浏览器和设备上测试性能
