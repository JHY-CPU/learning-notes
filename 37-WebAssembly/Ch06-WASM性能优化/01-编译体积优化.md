# 编译体积优化

## 一、概念说明

WASM 二进制体积直接影响加载时间。Rust 提供多种编译和链接选项减小产出大小。

```toml
# Cargo.toml 最佳实践
[profile.release]
opt-level = "s"       # 或 "z"（更激进）
lto = true
codegen-units = 1
strip = true
panic = "abort"
```

## 二、具体用法

### 2.1 opt-level 级别对比

```toml
opt-level = 0   # 不优化，最大体积
opt-level = 1   # 基本优化
opt-level = 2   # 默认优化
opt-level = 3   # 最大优化速度
opt-level = "s" # 优化体积（推荐）
opt-level = "z" # 最小体积（可能牺牲速度）
```

### 2.2 移除未使用代码

```toml
# 使用 wee_alloc 替代默认分配器
[dependencies]
wee_alloc = "0.4"

[features]
default = ["wee_alloc"]

# 仅在 WASM 时启用
[target.'cfg(target_arch = "wasm32")'.dependencies]
wee_alloc = "0.4"
```

```rust
#[cfg(target_arch = "wasm32")]
#[global_allocator]
static ALLOC: wee_alloc::WeeAlloc = wee_alloc::WeeAlloc::INIT;
```

### 2.3 wasm-opt 优化

```bash
# 基础优化
wasm-opt -Os input.wasm -o output.wasm

# 激进优化
wasm-opt -Oz --zero-filled-memory --strip-debug input.wasm -o output.wasm

# 多轮优化
wasm-opt -Os -Os input.wasm -o output.wasm

# 移除未导入的函数
wasm-opt --strip-producers input.wasm -o output.wasm
```

### 2.4 wasm-snip 移除死代码

```bash
# 移除指定函数
wasm-snip input.wasm --snip-func 'std::panicking::rust_panic' -o output.wasm

# 自动识别并移除未使用的 panic 基础设施
wasm-snip input.wasm -o output.wasm
```

### 2.5 特性裁剪

```toml
# 精简依赖
[dependencies]
serde = { version = "1", default-features = false, features = ["derive"] }
web-sys = { version = "0.3", features = ["console"] }  # 只启用需要的 feature

# 移除默认 features
anyhow = { version = "1", default-features = false }
```

## 三、注意事项与常见陷阱

1. **优化权衡**："z" 比 "s" 更小但可能更慢
2. **panic 策略**：abort 比 unwind 体积更小但不支持恢复
3. **LTO 副作用**：全 LTO 编译时间显著增加
4. **strip 调试**：strip = true 移除符号，调试困难
5. **wee_alloc 风险**：在某些场景可能有内存碎片问题
