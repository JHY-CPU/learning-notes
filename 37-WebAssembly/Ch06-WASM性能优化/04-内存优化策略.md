# 内存优化策略

## 一、概念说明

WASM 线性内存是 WASM 最宝贵的资源。合理管理内存直接影响性能和稳定性。

```rust
// 线性内存初始大小
memory.grow(1) // 增长 1 页 (64KB)

// 内存布局
// [0..heap_base)        → 栈
// [heap_base..data_end) → 全局数据
// [data_end..)          → 堆
```

## 二、具体用法

### 2.1 内存池管理

```rust
use std::alloc::{GlobalAlloc, Layout, System};

struct BumpAllocator {
    heap_start: usize,
    heap_end: usize,
    offset: usize,
}

impl BumpAllocator {
    const fn new() -> Self {
        BumpAllocator {
            heap_start: 0,
            heap_end: 0,
            offset: 0,
        }
    }

    unsafe fn alloc(&mut self, layout: Layout) -> *mut u8 {
        let align = layout.align();
        let size = layout.size();

        let aligned = (self.heap_start + self.offset + align - 1) & !(align - 1);
        let new_offset = aligned - self.heap_start + size;

        if new_offset > self.heap_end - self.heap_start {
            // 需要增长内存
            let pages_needed = (new_offset - (self.heap_end - self.heap_start) + 65535) / 65536;
            core::arch::wasm32::memory_grow(0, pages_needed);
            self.heap_end += pages_needed * 65536;
        }

        self.offset = new_offset;
        aligned as *mut u8
    }
}
```

### 2.2 区域分配器

```rust
pub struct Arena {
    buffer: Vec<u8>,
    offset: usize,
}

impl Arena {
    pub fn new(capacity: usize) -> Self {
        Arena {
            buffer: vec![0u8; capacity],
            offset: 0,
        }
    }

    pub fn alloc<T>(&mut self, value: T) -> &mut T {
        let align = std::mem::align_of::<T>();
        let size = std::mem::size_of::<T>();

        let aligned = (self.offset + align - 1) & !(align - 1);
        assert!(aligned + size <= self.buffer.len());

        let ptr = self.buffer[aligned..].as_mut_ptr() as *mut T;
        unsafe {
            ptr.write(value);
            self.offset = aligned + size;
            &mut *ptr
        }
    }

    pub fn reset(&mut self) {
        self.offset = 0;
    }
}
```

### 2.3 对象池

```rust
pub struct ObjectPool<T> {
    available: Vec<T>,
    factory: Box<dyn Fn() -> T>,
}

impl<T> ObjectPool<T> {
    pub fn new<F: Fn() -> T + 'static>(capacity: usize, factory: F) -> Self {
        let mut available = Vec::with_capacity(capacity);
        for _ in 0..capacity {
            available.push(factory());
        }
        ObjectPool {
            available,
            factory: Box::new(factory),
        }
    }

    pub fn acquire(&mut self) -> T {
        self.available.pop().unwrap_or_else(|| (self.factory)())
    }

    pub fn release(&mut self, obj: T) {
        self.available.push(obj);
    }
}
```

### 2.4 零拷贝数据共享

```rust
#[wasm_bindgen]
pub fn process_buffer_in_place(ptr: *mut u8, len: usize) {
    let data = unsafe { std::slice::from_raw_parts_mut(ptr, len) };
    for byte in data.iter_mut() {
        *byte = byte.wrapping_mul(2).wrapping_add(1);
    }
}
```

### 2.5 内存使用监控

```rust
#[wasm_bindgen]
pub fn get_memory_usage() -> JsValue {
    let pages = core::arch::wasm32::memory_size::<0>();
    let used_bytes = pages * 65536;

    let result = js_sys::Object::new();
    js_sys::Reflect::set(
        &result,
        &"pages".into(),
        &JsValue::from(pages),
    ).unwrap();
    js_sys::Reflect::set(
        &result,
        &"bytes".into(),
        &JsValue::from(used_bytes as f64),
    ).unwrap();
    result.into()
}
```

## 三、注意事项与常见陷阱

1. **内存上限**：WASM 初始和最大内存可配置，但浏览器有上限
2. **增长成本**：memory.grow 可能失败，且有性能开销
3. **对齐访问**：未对齐访问在 WASM 中有效但性能较差
4. **内存泄漏**：WASM 内存不自动回收，需手动管理
5. **共享限制**：WASM 模块间的内存不能直接共享
