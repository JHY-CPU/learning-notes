# 运行时性能优化

## 一、概念说明

WASM 运行时性能受内存布局、算法选择、边界调用频率等因素影响。

```rust
// 差：频繁跨边界
for item in items {
    js_console_log(&item.to_string());
}

// 好：批量操作
let msg: String = items.iter()
    .map(|i| i.to_string())
    .collect::<Vec<_>>()
    .join("\n");
js_console_log(&msg);
```

## 二、具体用法

### 2.1 内存访问优化

```rust
// 差：随机访问，缓存不友好
fn process_sparse(data: &[u8], indices: &[usize]) -> u32 {
    indices.iter().map(|&i| data[i] as u32).sum()
}

// 好：顺序访问，缓存友好
fn process_dense(data: &[u8]) -> u32 {
    data.iter().map(|&b| b as u32).sum()
}

// 好：结构体数组 → 数组结构体
// 差
struct Particle { x: f32, y: f32, vx: f32, vy: f32 }
let particles: Vec<Particle>;

// 好
struct Particles {
    x: Vec<f32>,
    y: Vec<f32>,
    vx: Vec<f32>,
    vy: Vec<f32>,
}
```

### 2.2 SIMD 加速

```rust
// 使用 portable-simd 或手动 SIMD
#[cfg(target_arch = "wasm32")]
use std::arch::wasm32::*;

pub fn dot_product(a: &[f32], b: &[f32]) -> f32 {
    assert_eq!(a.len(), b.len());

    #[cfg(target_arch = "wasm32")]
    {
        let mut sum = f32x4_splat(0.0);
        for i in (0..a.len()).step_by(4) {
            let va = v128_load(a.as_ptr().add(i) as *const v128);
            let vb = v128_load(b.as_ptr().add(i) as *const v128);
            sum = f32x4_add(sum, f32x4_mul(va, vb));
        }
        // 归约求和
        f32x4_extract_lane::<0>(sum)
            + f32x4_extract_lane::<1>(sum)
            + f32x4_extract_lane::<2>(sum)
            + f32x4_extract_lane::<3>(sum)
    }

    #[cfg(not(target_arch = "wasm32"))]
    {
        a.iter().zip(b.iter()).map(|(x, y)| x * y).sum()
    }
}
```

### 2.3 避免动态分配

```rust
use std::mem::MaybeUninit;

// 使用栈上数组避免堆分配
pub fn fixed_buffer_process(input: &[u8]) -> [u8; 1024] {
    let mut buf: [MaybeUninit<u8>; 1024] = unsafe {
        MaybeUninit::uninit().assume_init()
    };

    for (i, &byte) in input.iter().take(1024).enumerate() {
        buf[i].write(byte.wrapping_mul(2));
    }

    unsafe { std::mem::transmute(buf) }
}
```

### 2.4 预计算与查表

```rust
// 查表替代实时计算
const SIN_TABLE: [f32; 360] = {
    let mut table = [0.0f32; 360];
    let mut i = 0;
    while i < 360 {
        table[i] = (i as f32 * std::f32::consts::PI / 180.0).sin();
        i += 1;
    }
    table
};

pub fn fast_sin(degrees: usize) -> f32 {
    SIN_TABLE[degrees % 360]
}
```

### 2.5 并行计算

```rust
use rayon::prelude::*;

pub fn parallel_sum(data: &[f64]) -> f64 {
    data.par_iter().sum()
}

pub fn parallel_map(data: &mut [f64]) {
    data.par_iter_mut().for_each(|x| {
        *x = x.powi(2).sqrt().ln();
    });
}
```

## 三、注意事项与常见陷阱

1. **Profile 实测**：使用 Chrome DevTools Performance 分析瓶颈
2. **避免过度优化**：只优化热点路径
3. **SIMD 兼容性**：并非所有浏览器都支持 WASM SIMD
4. **内存对齐**：SIMD 操作需要 16 字节对齐
5. **内联策略**：标记 `#[inline]` 可能帮助编译器优化
