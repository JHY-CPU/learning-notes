# 多线程支持

## 一、概念说明

WASM 多线程基于 SharedArrayBuffer + Web Workers 实现，类似于 POSIX 线程模型。

```rust
use std::thread;
use std::sync::{Arc, Mutex};

pub fn parallel_sum(data: &[i32]) -> i32 {
    let data = Arc::new(data.to_vec());
    let result = Arc::new(Mutex::new(0i32));

    let mut handles = vec![];
    let chunk_size = data.len() / 4;

    for i in 0..4 {
        let data = Arc::clone(&data);
        let result = Arc::clone(&result);
        handles.push(thread::spawn(move || {
            let start = i * chunk_size;
            let end = start + chunk_size;
            let partial: i32 = data[start..end].iter().sum();
            *result.lock().unwrap() += partial;
        }));
    }

    for h in handles { h.join().unwrap(); }
    *result.lock().unwrap()
}
```

### 2.3 Rayon 并行计算

```rust
use rayon::prelude::*;
use wasm_bindgen_rayon::init_thread_pool;

#[wasm_bindgen]
pub async fn init_threads(num_threads: usize) -> Result<(), JsValue> {
    init_thread_pool(num_threads).await
}

#[wasm_bindgen]
pub fn parallel_process(data: &[f64]) -> f64 {
    data.par_iter()
        .map(|&x| x.sin().powi(2) + x.cos().powi(2))
        .sum::<f64>() / data.len() as f64
}
```

### 2.4 原子操作

```rust
use std::sync::atomic::{AtomicUsize, Ordering};

static COUNTER: AtomicUsize = AtomicUsize::new(0);

#[wasm_bindgen]
pub fn increment_atomic(times: usize) -> usize {
    for _ in 0..times {
        COUNTER.fetch_add(1, Ordering::SeqCst);
    }
    COUNTER.load(Ordering::SeqCst)
}

#[wasm_bindgen]
pub fn reset_counter() {
    COUNTER.store(0, Ordering::SeqCst);
}
```

### 2.5 SharedArrayBuffer 在 Rust 中使用

```rust
#[wasm_bindgen]
pub fn process_shared(shared_buffer: &js_sys::SharedArrayBuffer) {
    let buffer = js_sys::Uint8Array::new(shared_buffer);
    let len = buffer.length() as usize;

    // 直接操作共享内存
    let ptr = buffer.byte_offset() as *mut u8;
    let data = unsafe { std::slice::from_raw_parts_mut(ptr, len) };

    for byte in data.iter_mut() {
        *byte = byte.wrapping_add(1);
    }
}
```

## 三、注意事项与常见陷阱

1. **COOP/COEP 头**：SharedArrayBuffer 必须设置跨域隔离头
2. **浏览器支持**：Chrome 79+、Firefox 79+、Safari 15.2+ 支持
3. **调试复杂度**：多线程 WASM 调试极其困难
4. **Worker 开销**：每个 Worker 都有独立的 WASM 实例，内存不共享
5. **Atomics 前端**：JavaScript 侧也需用 Atomics API 进行同步
