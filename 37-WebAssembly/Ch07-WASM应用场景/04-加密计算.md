# 加密计算

## 一、概念说明

WASM 实现的加密算法在浏览器中提供接近原生的性能，适合客户端加密、密钥派生等场景。

```rust
use wasm_bindgen::prelude::*;

#[wasm_bindgen]
pub fn sha256(data: &[u8]) -> Vec<u8> {
    use sha2::{Sha256, Digest};
    let mut hasher = Sha256::new();
    hasher.update(data);
    hasher.finalize().to_vec()
}
```

## 二、具体用法

### 2.1 哈希计算

```rust
use sha2::{Sha256, Sha512, Digest};
use md5;

#[wasm_bindgen]
pub fn hash_sha256(data: &[u8]) -> String {
    let hash = Sha256::digest(data);
    hex::encode(hash)
}

#[wasm_bindgen]
pub fn hash_multiple(data: &[u8], algorithm: &str) -> Result<String, JsValue> {
    let result = match algorithm {
        "sha256" => hex::encode(Sha256::digest(data)),
        "sha512" => hex::encode(Sha512::digest(data)),
        "md5" => hex::encode(md5::compute(data).0),
        _ => return Err(JsValue::from_str("不支持的算法")),
    };
    Ok(result)
}
```

### 2.2 AES 加密解密

```rust
use aes::Aes256;
use aes::cipher::{BlockEncrypt, BlockDecrypt, KeyInit};
use cbc::cipher::block_padding::Pkcs7;

#[wasm_bindgen]
pub fn aes_encrypt(plaintext: &[u8], key: &[u8], iv: &[u8]) -> Result<Vec<u8>, JsValue> {
    let cipher = Aes256::new_from_slice(key)
        .map_err(|e| JsValue::from_str(&e.to_string()))?;

    let mut buffer = plaintext.to_vec();
    // CBC 模式加密
    let encrypted = cbc::Encryptor::<Aes256>::new_from_slices(key, iv)
        .map_err(|e| JsValue::from_str(&e.to_string()))?
        .encrypt_padded_vec_mut::<Pkcs7>(plaintext);

    Ok(encrypted)
}
```

### 2.3 PBKDF2 密钥派生

```rust
use pbkdf2::pbkdf2_hmac;
use sha2::Sha256;

#[wasm_bindgen]
pub fn derive_key(password: &str, salt: &[u8], iterations: u32) -> Vec<u8> {
    let mut key = [0u8; 32];
    pbkdf2_hmac::<Sha256>(password.as_bytes(), salt, iterations, &mut key);
    key.to_vec()
}
```

### 2.4 签名验证

```rust
use ed25519_dalek::{Keypair, Signature, Signer, Verifier};

#[wasm_bindgen]
pub fn generate_keypair() -> Vec<u8> {
    let mut csprng = rand::thread_rng();
    let keypair = Keypair::generate(&mut csprng);
    keypair.to_bytes().to_vec()
}

#[wasm_bindgen]
pub fn sign_message(private_key_bytes: &[u8], message: &[u8]) -> Result<Vec<u8>, JsValue> {
    let keypair = Keypair::from_bytes(private_key_bytes)
        .map_err(|e| JsValue::from_str(&e.to_string()))?;
    let signature = keypair.sign(message);
    Ok(signature.to_bytes().to_vec())
}

#[wasm_bindgen]
pub fn verify_signature(
    public_key_bytes: &[u8],
    message: &[u8],
    signature_bytes: &[u8],
) -> Result<bool, JsValue> {
    let public_key = ed25519_dalek::PublicKey::from_bytes(public_key_bytes)
        .map_err(|e| JsValue::from_str(&e.to_string()))?;
    let signature = Signature::from_bytes(signature_bytes)
        .map_err(|e| JsValue::from_str(&e.to_string()))?;
    Ok(public_key.verify(message, &signature).is_ok())
}
```

### 2.5 随机数生成

```rust
#[wasm_bindgen]
pub fn secure_random(len: usize) -> Vec<u8> {
    use getrandom::getrandom;
    let mut buf = vec![0u8; len];
    getrandom(&mut buf).expect("随机数生成失败");
    buf
}
```

## 三、注意事项与常见陷阱

1. **随机数源**：getrandom 使用浏览器 crypto.getRandomValues，密码学安全
2. **时间侧信道**：WASM 中恒定时间比较仍有效，但需测试
3. **密钥存储**：浏览器中无法安全存储密钥，考虑 WebCrypto API
4. **性能对比**：简单哈希可用 WebCrypto，复杂操作用 WASM
5. **WASM 文件**：加密库会显著增大 WASM 体积
