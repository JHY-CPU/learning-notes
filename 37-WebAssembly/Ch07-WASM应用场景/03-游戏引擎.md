# 游戏引擎

## 一、概念说明

WASM 适合游戏引擎：高性能计算、Canvas/WebGL 渲染、确定性执行。

```rust
use wasm_bindgen::prelude::*;

#[wasm_bindgen]
pub struct Game {
    width: u32,
    height: u32,
    player_x: f32,
    player_y: f32,
    entities: Vec<Entity>,
}

#[wasm_bindgen]
impl Game {
    #[wasm_bindgen(constructor)]
    pub fn new(width: u32, height: u32) -> Game {
        Game {
            width, height,
            player_x: width as f32 / 2.0,
            player_y: height as f32 / 2.0,
            entities: Vec::new(),
        }
    }

    pub fn update(&mut self, dt: f32) {
        // 游戏逻辑更新
        for entity in &mut self.entities {
            entity.update(dt);
        }
    }

    pub fn render(&self) -> Vec<u8> {
        // 返回像素数据
        vec![]
    }
}
```

## 二、具体用法

### 2.1 ECS 架构

```rust
use std::collections::HashMap;

#[derive(Default)]
pub struct World {
    next_id: u32,
    positions: HashMap<u32, (f32, f32)>,
    velocities: HashMap<u32, (f32, f32)>,
    sprites: HashMap<u32, Sprite>,
}

#[derive(Clone)]
pub struct Sprite {
    pub texture_id: u32,
    pub width: f32,
    pub height: f32,
}

impl World {
    pub fn spawn(&mut self) -> u32 {
        let id = self.next_id;
        self.next_id += 1;
        id
    }

    pub fn step(&mut self, dt: f32) {
        // 更新所有有 velocity 的实体位置
        for (&id, vel) in &self.velocities {
            if let Some(pos) = self.positions.get_mut(&id) {
                pos.0 += vel.0 * dt;
                pos.1 += vel.1 * dt;
            }
        }
    }
}
```

### 2.2 精灵渲染

```rust
#[wasm_bindgen]
pub fn render_sprites(
    positions: &[f32],     // [x0, y0, x1, y1, ...]
    sprite_indices: &[u32],
    output: &mut [u8],
    canvas_width: u32,
    canvas_height: u32,
    sprite_atlas: &[u8],
    sprite_size: u32,
) {
    // 清除画布
    output.fill(0);

    let sprites_per_row = canvas_width / sprite_size;

    for i in 0..positions.len() / 2 {
        let x = positions[i * 2] as i32;
        let y = positions[i * 2 + 1] as i32;
        let sprite_idx = sprite_indices[i];

        // 绘制精灵到画布
        for sy in 0..sprite_size as i32 {
            for sx in 0..sprite_size as i32 {
                let dx = x + sx;
                let dy = y + sy;
                if dx >= 0 && dx < canvas_width as i32 && dy >= 0 && dy < canvas_height as i32 {
                    let dst = (dy as u32 * canvas_width + dx as u32) as usize * 4;
                    let src = (sprite_idx as usize * sprite_size as usize * sprite_size as usize
                        + sy as usize * sprite_size as usize + sx as usize) * 4;
                    if src + 3 < sprite_atlas.len() && dst + 3 < output.len() {
                        output[dst..dst+4].copy_from_slice(&sprite_atlas[src..src+4]);
                    }
                }
            }
        }
    }
}
```

### 2.3 物理碰撞

```rust
#[wasm_bindgen]
pub struct Body {
    pub x: f32,
    pub y: f32,
    pub vx: f32,
    pub vy: f32,
    pub radius: f32,
}

#[wasm_bindgen]
pub fn detect_collisions(bodies: &mut [Body]) {
    for i in 0..bodies.len() {
        for j in (i+1)..bodies.len() {
            let dx = bodies[j].x - bodies[i].x;
            let dy = bodies[j].y - bodies[i].y;
            let dist = (dx * dx + dy * dy).sqrt();
            let min_dist = bodies[i].radius + bodies[j].radius;

            if dist < min_dist {
                // 弹性碰撞
                let nx = dx / dist;
                let ny = dy / dist;
                let dvx = bodies[i].vx - bodies[j].vx;
                let dvy = bodies[i].vy - bodies[j].vy;
                let dv_dot_n = dvx * nx + dvy * ny;

                bodies[i].vx -= dv_dot_n * nx;
                bodies[i].vy -= dv_dot_n * ny;
                bodies[j].vx += dv_dot_n * nx;
                bodies[j].vy += dv_dot_n * ny;

                // 分离重叠
                let overlap = (min_dist - dist) / 2.0;
                bodies[i].x -= overlap * nx;
                bodies[i].y -= overlap * ny;
                bodies[j].x += overlap * nx;
                bodies[j].y += overlap * ny;
            }
        }
    }
}
```

### 2.4 输入处理

```javascript
class InputManager {
    constructor(canvas) {
        this.keys = {};
        this.mouse = { x: 0, y: 0, down: false };

        document.addEventListener('keydown', e => this.keys[e.code] = true);
        document.addEventListener('keyup', e => this.keys[e.code] = false);

            this.mouse.x = e.offsetX;

    }

        return {

            mouse: this.mouse,
        };
    }
}
```

## 三、注意事项与常见陷阱

1. **帧率管理**：使用 requestAnimationFrame，注意 dt 稳定性
2. **内存管理**：游戏实体频繁创建销毁，使用对象池
3. **确定性**：浮点运算跨平台可能不一致，影响网络同步
4. **音频延迟**：Web Audio API 有延迟，需预缓冲
5. **状态序列化**：存档/网络同步需要可序列化的游戏状态
