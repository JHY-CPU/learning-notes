# 区块链应用

## 一、概念说明

WASM 被多个区块链平台采用作为智能合约运行时（Polkadot、NEAR、Cosmos），提供安全、高效的执行环境。

```rust
use wasm_bindgen::prelude::*;

#[wasm_bindgen]
pub fn verify_transaction(
    sender: &[u8],
    receiver: &[u8],
    amount: u64,
    signature: &[u8],
) -> bool {
    // 验证签名
    let message = [sender, receiver, &amount.to_le_bytes()].concat();
    verify_signature(sender, &message, signature)
}
```

## 二、具体用法

### 2.1 智能合约模拟

```rust
use std::collections::HashMap;

#[wasm_bindgen]
pub struct ContractState {
    storage: HashMap<Vec<u8>, Vec<u8>>,
    balances: HashMap<Vec<u8>, u64>,
}

#[wasm_bindgen]
impl ContractState {
    #[wasm_bindgen(constructor)]
    pub fn new() -> ContractState {
        ContractState {
            storage: HashMap::new(),
            balances: HashMap::new(),
        }
    }

    pub fn transfer(&mut self, from: &[u8], to: &[u8], amount: u64) -> Result<(), JsValue> {
        let from_balance = self.balances.get(from).copied().unwrap_or(0);
        if from_balance < amount {
            return Err(JsValue::from_str("余额不足"));
        }
        self.balances.insert(from.to_vec(), from_balance - amount);
        *self.balances.entry(to.to_vec()).or_insert(0) += amount;
        Ok(())
    }

    pub fn get_balance(&self, account: &[u8]) -> u64 {
        self.balances.get(account).copied().unwrap_or(0)
    }

    pub fn set_storage(&mut self, key: &[u8], value: &[u8]) {
        self.storage.insert(key.to_vec(), value.to_vec());
    }

    pub fn get_storage(&self, key: &[u8]) -> Option<Vec<u8>> {
        self.storage.get(key).cloned()
    }
}
```

### 2.2 哈希与 Merkle 树

```rust
use sha2::{Sha256, Digest};

#[wasm_bindgen]
pub fn merkle_root(leaves: &[u8], leaf_size: usize) -> Vec<u8> {
    let mut hashes: Vec<Vec<u8>> = leaves
        .chunks(leaf_size)
        .map(|chunk| Sha256::digest(chunk).to_vec())
        .collect();

    while hashes.len() > 1 {
        let mut next = Vec::new();
        for pair in hashes.chunks(2) {
            let mut combined = Vec::new();
            combined.extend_from_slice(&pair[0]);
            if pair.len() > 1 {
                combined.extend_from_slice(&pair[1]);
            } else {
                combined.extend_from_slice(&pair[0]);
            }
            next.push(Sha256::digest(&combined).to_vec());
        }
        hashes = next;
    }

    hashes.remove(0)
}
```

### 2.4 交易签名验证

```rust
#[wasm_bindgen]
pub struct Transaction {
    sender: Vec<u8>,
    receiver: Vec<u8>,
    amount: u64,
    nonce: u64,
    signature: Vec<u8>,
}

#[wasm_bindgen]
impl Transaction {
    #[wasm_bindgen(constructor)]
    pub fn new(
        sender: &[u8],
        receiver: &[u8],
        amount: u64,
        nonce: u64,
        signature: &[u8],
    ) -> Transaction {
        Transaction {
            sender: sender.to_vec(),
            receiver: receiver.to_vec(),
            amount,
            nonce,
            signature: signature.to_vec(),
        }
    }

    pub fn hash(&self) -> Vec<u8> {
        let mut data = Vec::new();
        data.extend_from_slice(&self.sender);
        data.extend_from_slice(&self.receiver);
        data.extend_from_slice(&self.amount.to_le_bytes());
        data.extend_from_slice(&self.nonce.to_le_bytes());
        Sha256::digest(&data).to_vec()
    }

    pub fn verify(&self, public_key: &[u8]) -> bool {
        let message = self.hash();
        verify_signature(public_key, &message, &self.signature)
    }
}
```

pub struct Wallet {

    address: String,

#[wasm_bindgen]
impl Wallet {
    #[wasm_bindgen(constructor)]
    pub fn generate() -> Wallet {
        let private_key = secure_random(32);
        let public_key = derive_public_key(&private_key);
        let address = generate_address(&public_key);
        Wallet { private_key, public_key, address }
    }

    pub fn address(&self) -> String {
        self.address.clone()
    }

    pub fn sign(&self, message: &[u8]) -> Vec<u8> {
        sign_message(&self.private_key, message)
    }
}

## 三、注意事项与常见陷阱

1. **随机数质量**：密钥生成必须使用密码学安全随机数
2. **侧信道攻击**：常量时间比较防止时序攻击
3. **大数运算**：区块链中经常需要大整数运算
4. **Gas 计量**：智能合约执行需要 Gas 计量防止无限循环
5. **确定性**：WASM 执行必须是确定性的，不能依赖随机数或时间
