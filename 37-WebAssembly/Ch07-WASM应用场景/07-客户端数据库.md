# 客户端数据库

## 一、概念说明

WASM 可移植 SQLite 等数据库到浏览器，实现客户端数据持久化和查询。

```javascript
// 使用 sql.js（SQLite WASM 版）
import initSqlJs from 'sql.js';

const SQL = await initSqlJs();
const db = new SQL.Database();
db.run("CREATE TABLE users (id INTEGER PRIMARY KEY, name TEXT)");
db.run("INSERT INTO users VALUES (?, ?)", [1, "Alice"]);
const results = db.exec("SELECT * FROM users");
```

## 二、具体用法

### 2.1 SQLite WASM 集成

```rust
use wasm_bindgen::prelude::*;

#[wasm_bindgen]
pub struct ClientDB {
    data: std::collections::HashMap<String, Vec<Vec<String>>>,
}

#[wasm_bindgen]
impl ClientDB {
    #[wasm_bindgen(constructor)]
    pub fn new() -> ClientDB {
        ClientDB {
            data: std::collections::HashMap::new(),
        }
    }

    pub fn insert(&mut self, table: &str, row: Vec<String>) {
        self.data.entry(table.to_string()).or_default().push(row);
    }

    pub fn query(&self, table: &str, col: usize, value: &str) -> JsValue {
        let results: Vec<&Vec<String>> = self.data
            .get(table)
            .map(|rows| rows.iter().filter(|r| r.get(col).map_or(false, |v| v == value)).collect())
            .unwrap_or_default();
        serde_wasm_bindgen::to_value(&results).unwrap()
    }
}
```

### 2.3 查询引擎

```rust
#[derive(Clone)]
pub struct Record {
    pub fields: std::collections::HashMap<String, String>,
}

pub struct QueryEngine {
    records: Vec<Record>,
}

impl QueryEngine {
    pub fn new() -> Self {
        QueryEngine { records: Vec::new() }
    }

    pub fn insert(&mut self, record: Record) {
        self.records.push(record);
    }

    pub fn select(&self, fields: &[String], condition: Option<(&str, &str)>) -> Vec<Vec<String>> {
        self.records.iter()
            .filter(|r| {
                if let Some((field, value)) = condition {
                    r.fields.get(field).map_or(false, |v| v == value)
                } else {
                    true
                }
            })
            .map(|r| {
                fields.iter()
                    .map(|f| r.fields.get(f).cloned().unwrap_or_default())
                    .collect()
            })
            .collect()
    }

    pub fn count(&self, condition: Option<(&str, &str)>) -> usize {
        self.records.iter()
            .filter(|r| {
                if let Some((field, value)) = condition {
                    r.fields.get(field).map_or(false, |v| v == value)
                } else {
                    true
                }
            })
            .count()
    }
}
```

### 2.4 索引与排序

```rust
use std::collections::BTreeMap;

pub struct IndexedStore {
    data: Vec<String>,
    index: BTreeMap<String, usize>,
}

impl IndexedStore {
    pub fn new() -> Self {
        IndexedStore {
            data: Vec::new(),
            index: BTreeMap::new(),
        }
    }

    pub fn insert(&mut self, key: String, value: String) {
        let idx = self.data.len();
        self.data.push(value);
        self.index.insert(key, idx);
    }

    pub fn range_query(&self, start: &str, end: &str) -> Vec<&str> {
        self.index.range(start.to_string()..end.to_string())
            .map(|(_, &idx)| self.data[idx].as_str())
            .collect()
    }

    pub fn get(&self, key: &str) -> Option<&str> {
        self.index.get(key).map(|&idx| self.data[idx].as_str())
    }
}
```

### 2.5 同步机制

```rust
#[wasm_bindgen]
pub struct SyncManager {
    local_version: u64,
    pending_changes: Vec<JsValue>,
}

#[wasm_bindgen]
impl SyncManager {
    #[wasm_bindgen(constructor)]
    pub fn new() -> SyncManager {
        SyncManager {
            local_version: 0,
            pending_changes: Vec::new(),
        }
    }

    pub fn add_change(&mut self, change: JsValue) {
        self.pending_changes.push(change);
        self.local_version += 1;
    }

    pub async fn sync(&mut self, server_url: &str) -> Result<JsValue, JsValue> {
        let changes = serde_wasm_bindgen::to_value(&self.pending_changes)?;
        // 发送到服务器...
        self.pending_changes.clear();
        Ok(changes)
    }
}
```

## 三、注意事项与常见陷阱

1. **存储限制**：IndexedDB 有存储配额（通常 50%-80% 磁盘空间）
2. **查询性能**：客户端数据库不如服务端数据库高效
3. **并发访问**：多个标签页同时操作可能冲突
4. **数据一致性**：离线操作需要冲突解决策略
5. **事务支持**：确保操作的原子性
