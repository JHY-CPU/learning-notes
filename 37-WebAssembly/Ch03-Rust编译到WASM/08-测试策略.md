# 测试策略

## 一、概念说明

Rust WASM 项目需要在浏览器环境中测试。

```rust
// wasm-bindgen-test
use wasm_bindgen_test::*;

wasm_bindgen_test_configure!(run_in_browser);

#[wasm_bindgen_test]
fn test_add() {
    assert_eq!(add(1, 2), 3);
}
```

## 二、具体用法

### 2.1 基本测试

```rust
use wasm_bindgen_test::*;
use wasm_bindgen::prelude::*;

#[wasm_bindgen_test]
fn test_greet() {
    let result = greet("世界");
    assert_eq!(result, "你好，世界！");
}

#[wasm_bindgen_test]
async fn test_async_fetch() {
    let result = fetch_url("https://example.com").await;
    assert!(result.is_ok());
}
```

### 2.2 DOM 测试

```rust
use wasm_bindgen_test::*;
use web_sys::HtmlElement;

wasm_bindgen_test_configure!(run_in_browser);

#[wasm_bindgen_test]
fn test_create_element() {
    let document = web_sys::window().unwrap().document().unwrap();
    let element = document.create_element("div").unwrap();
    element.set_text_content(Some("测试内容"));
    assert_eq!(element.text_content().unwrap(), "测试内容");
}
```

### 2.3 集成测试

```rust
// tests/integration.rs
use wasm_bindgen_test::*;
use my_wasm_lib::*;

wasm_bindgen_test_configure!(run_in_browser);

#[wasm_bindgen_test]
async fn test_full_workflow() {
    let result = process_workflow("输入").await;
    assert!(result.is_ok());
}
```

### 2.4 运行测试

```bash
# 运行所有测试
wasm-pack test --headless --chrome

# 运行特定测试
wasm-pack test --headless --chrome -- --test test_add

# 在浏览器中运行
wasm-pack test --chrome
```

## 三、注意事项与常见陷阱

1. **浏览器环境**：测试需要浏览器环境
2. **异步测试**：使用 async 标记异步测试
3. **DOM 测试**：DOM 测试需要 run_in_browser
4. **CI 集成**：CI 中使用 headless 浏览器
5. **测试覆盖**：确保测试覆盖关键路径

## 六、性能测试

```rust
use wasm_bindgen_test::*;
use wasm_bindgen::prelude::*;

wasm_bindgen_test_configure!(run_in_browser);

#[wasm_bindgen_test]
fn test_performance() {
    let iterations = 10000;

    // 预热
    fibonacci(10);

    // 测量
    let start = js_sys::Date::now();
    for _ in 0..iterations {
        fibonacci(20);
    }
    let elapsed = js_sys::Date::now() - start;

    let avg = elapsed / iterations as f64;
    web_sys::console::log_1(
        &format!("平均耗时: {:.4}ms/次, 总计: {:.2}ms", avg, elapsed).into()
    );

    // 断言性能阈值
    assert!(avg < 1.0, "平均耗时不应超过 1ms");
}

#[wasm_bindgen]
pub fn fibonacci(n: u32) -> u32 {
    match n {
        0 => 0,
        1 => 1,
        _ => fibonacci(n - 1) + fibonacci(n - 2),
    }
}
```

## 七、Mock 和 Stub 技巧

```rust
use wasm_bindgen_test::*;
use wasm_bindgen::prelude::*;

// 定义 trait 以便测试时替换实现
pub trait HttpClient {
    fn get(&self, url: &str) -> Result<String, String>;
}

// 真实实现
pub struct RealClient;

impl HttpClient for RealClient {
    fn get(&self, _url: &str) -> Result<String, String> {
        // 使用 web_sys::fetch
        Ok(String::from("real response"))
    }
}

// 测试用 mock
pub struct MockClient {
    pub response: String,
}

impl HttpClient for MockClient {
    fn get(&self, _url: &str) -> Result<String, String> {
        Ok(self.response.clone())
    }
}

#[wasm_bindgen_test]
fn test_with_mock() {
    let mock = MockClient {
        response: String::from("mocked data"),
    };
    let result = mock.get("https://example.com").unwrap();
    assert_eq!(result, "mocked data");
}
```

## 八、CI/CD 集成配置

```yaml
# .github/workflows/test.yml
name: WASM Tests

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Install Rust
        uses: actions-rs/toolchain@v1
        with:
          toolchain: stable
          target: wasm32-unknown-unknown

      - name: Install wasm-pack
        run: curl https://rustwasm.github.io/wasm-pack/installer/init.sh -sSf | sh

      - name: Install Chrome
        uses: browser-actions/setup-chrome@v1

      - name: Run tests (headless Chrome)
        run: wasm-pack test --headless --chrome

      - name: Run tests (headless Firefox)
        run: wasm-pack test --headless --firefox
        continue-on-error: true  # Firefox 可能需要额外配置

      - name: Build release
        run: wasm-pack build --release --target web

      - name: Check size
        run: |
          SIZE=$(wc -c < pkg/my_lib_bg.wasm)
          echo "WASM size: ${SIZE} bytes"
          if [ "$SIZE" -gt 1048576 ]; then
            echo "WARNING: WASM exceeds 1MB"
          fi
```

## 九、测试覆盖率

```bash
# 使用 cargo-tarpaulin（不直接支持 WASM）

cargo install cargo-binutils

# 运行测试并收集覆盖率

# 使用 grcov 生成报告

```

## 十、测试最佳实践清单

```
[ ] 纯 Rust 逻辑用 #[cfg(test)] 单元测试（不需要浏览器）
[ ] 浏览器相关功能用 wasm_bindgen_test 测试
[ ] DOM 操作使用 run_in_browser 配置
[ ] 异步操作使用 async fn + .await
[ ] 使用 headless 浏览器进行 CI 测试
[ ] 包含错误路径的测试
[ ] 性能关键函数包含基准测试
[ ] 测试覆盖率 > 80%
[ ] 使用 Mock 隔离外部依赖
[ ] 测试数据与生产数据分离
```
