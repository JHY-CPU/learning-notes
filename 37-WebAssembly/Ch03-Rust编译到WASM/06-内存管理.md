# 内存管理

## 一、概念说明

Rust WASM 中的内存管理需要手动处理线性内存。

```rust
use wasm_bindgen::prelude::*;

#[wasm_bindgen]
pub fn allocate_buffer(size: usize) -> *mut u8 {
    let mut buffer = Vec::with_capacity(size);
    let ptr = buffer.as_mut_ptr();
    std::mem::forget(buffer); // 防止 Rust 释放内存
    ptr
}

#[wasm_bindgen]
pub unsafe fn deallocate_buffer(ptr: *mut u8, size: usize) {
    let _ = Vec::from_raw_parts(ptr, 0, size);
}
```

## 二、具体用法

### 2.1 内存池

```rust
use wasm_bindgen::prelude::*;
use std::collections::HashMap;

#[wasm_bindgen]
pub struct MemoryPool {
    allocations: HashMap<usize, Vec<u8>>,
    next_id: usize,
}

#[wasm_bindgen]
impl MemoryPool {
    #[wasm_bindgen(constructor)]
    pub fn new() -> MemoryPool {
        MemoryPool {
            allocations: HashMap::new(),
            next_id: 0,
        }
    }

    pub fn allocate(&mut self, size: usize) -> usize {
        let id = self.next_id;
        self.allocations.insert(id, vec![0u8; size]);
        self.next_id += 1;
        id
    }

    pub fn write(&mut self, id: usize, data: &[u8]) -> Result<(), JsValue> {
        if let Some(buffer) = self.allocations.get_mut(&id) {
            buffer.copy_from_slice(data);
            Ok(())
        } else {
            Err(JsValue::from_str("无效的内存 ID"))
        }
    }

    pub fn read(&self, id: usize) -> Result<Vec<u8>, JsValue> {
        self.allocations
            .get(&id)
            .cloned()
            .ok_or_else(|| JsValue::from_str("无效的内存 ID"))
    }

    pub fn deallocate(&mut self, id: usize) -> Result<(), JsValue> {
        self.allocations
            .remove(&id)
            .map(|_| ())
            .ok_or_else(|| JsValue::from_str("无效的内存 ID"))
    }
}
```

### 2.2 共享内存

```rust
use wasm_bindgen::prelude::*;
use web_sys::SharedArrayBuffer;

#[wasm_bindgen]
pub fn create_shared_memory(size: u32) -> Result<SharedArrayBuffer, JsValue> {
    SharedArrayBuffer::new(size)
}

#[wasm_bindgen]
pub fn read_shared_memory(buffer: &SharedArrayBuffer) -> Vec<u8> {
    let array = js_sys::Uint8Array::new(buffer);
    array.to_vec()
}
```

### 2.3 内存优化

```rust
use wasm_bindgen::prelude::*;

#[wasm_bindgen]
pub struct OptimizedBuffer {
    data: Vec<u8>,
    capacity: usize,
}

#[wasm_bindgen]
impl OptimizedBuffer {
    pub fn new(capacity: usize) -> OptimizedBuffer {
        OptimizedBuffer {
            data: Vec::with_capacity(capacity),
            capacity,
        }
    }

    pub fn push(&mut self, byte: u8) -> Result<(), JsValue> {
        if self.data.len() >= self.capacity {
            return Err(JsValue::from_str("缓冲区已满"));
        }
        self.data.push(byte);
        Ok(())
    }

    pub fn as_ptr(&self) -> *const u8 {
        self.data.as_ptr()
    }

    pub fn len(&self) -> usize {
        self.data.len()
    }
}
```

## 三、注意事项与常见陷阱

1. **内存泄漏**：确保释放分配的内存
2. **越界访问**：检查内存边界
3. **并发安全**：SharedArrayBuffer 需要同步
4. **性能**：频繁分配有性能开销
5. **调试**：使用工具检测内存问题
