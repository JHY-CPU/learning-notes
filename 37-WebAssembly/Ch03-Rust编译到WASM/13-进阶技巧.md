# 进阶技巧

## 一、概念说明

Rust WASM 的进阶用法和技巧。

```rust
// 自定义导入函数
#[wasm_bindgen(raw_module = "./custom.js")]
extern "C" {
    fn custom_function(x: i32) -> i32;
}

// 使用
let result = custom_function(42);
```

## 二、具体用法

### 2.1 自定义绑定

```rust
use wasm_bindgen::prelude::*;

#[wasm_bindgen]
extern "C" {
    #[wasm_bindgen(js_namespace = console)]
    fn log(s: &str);

    #[wasm_bindgen(js_namespace = ["window", "document"])]
    fn getElementById(id: &str) -> web_sys::Element;
}

#[wasm_bindgen]
pub fn use_custom_bindings() {
    log("Hello from Rust!");
    let element = getElementById("my-element");
}
```

### 2.2 类型转换

```rust
use wasm_bindgen::prelude::*;

#[wasm_bindgen]
pub struct ComplexType {
    inner: Vec<f64>,
}

#[wasm_bindgen]
impl ComplexType {
    #[wasm_bindgen(constructor)]
    pub fn new(data: &[f64]) -> ComplexType {
        ComplexType {
            inner: data.to_vec(),
        }
    }

    pub fn to_js_array(&self) -> js_sys::Float64Array {
        js_sys::Float64Array::from(&self.inner[..])
    }

    pub fn from_js_array(array: &js_sys::Float64Array) -> ComplexType {
        ComplexType {
            inner: array.to_vec(),
        }
    }
}
```

### 2.3 异步闭包

```rust
use wasm_bindgen::prelude::*;
use wasm_bindgen_futures::spawn_local;

#[wasm_bindgen]
pub fn start_async_task() {
    spawn_local(async {
        let result = fetch_data("https://api.example.com").await;
        console::log_1(&result.unwrap().into());
    });
}
```

### 2.4 Worker 通信

```rust
use wasm_bindgen::prelude::*;
use web_sys::{Worker, MessageEvent};

#[wasm_bindgen]
pub fn create_worker(url: &str) -> Result<Worker, JsValue> {
    let worker = Worker::new(url)?;

    let onmessage = Closure::wrap(Box::new(move |event: MessageEvent| {
        let data = event.data();
        console::log_1(&data);
    }) as Box<dyn FnMut(MessageEvent)>);

    worker.set_onmessage(Some(onmessage.as_ref().unchecked_ref()));
    onmessage.forget();

    Ok(worker)
}
```

## 三、注意事项与常见陷阱

1. **类型安全**：确保类型转换安全
2. **内存管理**：注意闭包和回调的生命周期
3. **错误处理**：处理异步错误
4. **性能**：避免频繁跨边界调用
5. **调试**：使用工具调试进阶功能
