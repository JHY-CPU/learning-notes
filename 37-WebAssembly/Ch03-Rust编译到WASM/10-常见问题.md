## 二、具体用法

### 2.1 编译问题

```bash
# 问题：找不到 wasm32 目标
# 解决：添加目标
rustup target add wasm32-unknown-unknown

# 问题：链接错误
# 解决：检查 crate-type
[lib]
crate-type = ["cdylib", "rlib"]

# 问题：类型不支持 wasm-bindgen
# 解决：使用 JsValue 包装
#[wasm_bindgen]
pub fn complex_type(value: JsValue) -> JsValue {
    value
}
```

### 2.2 运行时问题

```javascript
// 问题：WASM 加载失败
// 解决：检查 MIME 类型
fetch('module.wasm', {
  headers: { 'Content-Type': 'application/wasm' }
});

// 问题：导入函数未定义
// 解决：检查导入对象
const imports = {
  env: {
    memory: new WebAssembly.Memory({ initial: 1 }),
  }
};
```

### 2.3 性能问题

```bash
# 问题：二进制太大
# 解决：优化编译
wasm-pack build --release
wasm-opt -Oz module.wasm -o module.opt.wasm

# 问题：执行慢
# 解决：使用 SIMD
# 启用 SIMD 特性
```

### 2.4 调试问题

```bash
# 问题：无法调试
# 解决：启用调试信息
wasm-pack build --dev
# 或在 Cargo.toml 中
[profile.release]
debug = true
```

### 2.5 兼容性问题

```javascript
// 检测 WASM 支持
if (typeof WebAssembly === 'object') {
  // 支持 WASM
} else {
  // 回退方案
}

// 检测特定特性
if (WebAssembly.validate(new Uint8Array([0x00, 0x61, 0x73, 0x6d]))) {
  // 有效 WASM
}
```
