# 部署与发布

## 一、概念说明

Rust WASM 项目的部署和发布流程。

```bash
# 发布到 npm
wasm-pack build --target bundler
cd pkg
npm publish
```

## 二、具体用法

### 2.1 发布到 npm

```json
// package.json
{
  "name": "my-wasm-lib",
  "version": "1.0.0",
  "description": "Rust WASM 库",
  "main": "my_wasm_lib.js",
  "types": "my_wasm_lib.d.ts",
  "files": [
    "my_wasm_lib_bg.wasm",
    "my_wasm_lib.js",
    "my_wasm_lib.d.ts"
  ],
  "repository": {
    "type": "git",
    "url": "https://github.com/user/repo"
  },
  "license": "MIT"
}
```

### 2.2 CDN 部署

```bash
# 构建
wasm-pack build --target web --release

# 上传到 CDN
# Cloudflare, AWS S3, etc.

# 在 HTML 中引用
<script type="module">
  import init from 'https://cdn.example.com/my-wasm-lib/my_wasm_lib.js';
  await init();
</script>
```

### 2.3 版本管理

```bash
# 更新版本
cargo set-version 1.2.3
git add -A
git commit -m "release: v1.2.3"
git tag v1.2.3
git push && git push --tags

# 发布
wasm-pack build --release --target web
cd pkg
npm version patch
npm publish
```

### 2.4 CI/CD

```yaml
# .github/workflows/deploy.yml
name: Deploy
on:
  push:
    tags:
      - 'v*'

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions-rs/toolchain@v1
        with:
          toolchain: stable
          target: wasm32-unknown-unknown
      - name: Install wasm-pack
        run: curl https://rustwasm.github.io/wasm-pack/installer/init.sh -sSf | sh
      - name: Build
        run: wasm-pack build --release --target web
      - name: Publish
        run: cd pkg && npm publish
        env:
          NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}
```

## 三、注意事项与常见陷阱

1. **版本同步**：Cargo.toml 和 package.json 版本同步
2. **许可证**：确保许可证正确
3. **文档**：维护好文档
4. **安全**：注意安全更新
5. **监控**：监控发布后的使用情况

## 五、不同构建目标对比

```bash
# --target web: ES 模块，需要手动 init()
wasm-pack build --target web
# 输出：ES module，import init, { foo } from './pkg/lib.js'
# 适合：现代框架 (Vite, Webpack 5+)

# --target bundler: CommonJS/Webpack 兼容
wasm-pack build --target bundler
# 输出：适合 Webpack 4 及更早版本

# --target nodejs: Node.js 专用
wasm-pack build --target nodejs
# 输出：使用 require()，无浏览器 API

# --target no-modules: 传统 <script> 标签
wasm-pack build --target no-modules
# 输出：全局 window.wasm_bindgen 对象
# 适合：无构建工具的简单页面
```

## 六、Vite 集成

```javascript
// vite.config.js
import { defineConfig } from 'vite';
import wasm from 'vite-plugin-wasm';
import topLevelAwait from 'vite-plugin-top-level-await';

export default defineConfig({
  plugins: [wasm(), topLevelAwait()],
});

// src/main.js
import init, { greet, Calculator } from '../pkg/my_lib.js';

await init();

console.log(greet('World'));
const calc = new Calculator(0);
calc.add(5);
console.log(calc.get_value());
```

## 七、Webpack 集成

```javascript
// webpack.config.js
module.exports = {
  experiments: {
    asyncWebAssembly: true,  // Webpack 5
  },
  module: {
    rules: [
      {
        test: /\.wasm$/,
        type: 'webassembly/async',
      },
    ],
  },
};

// src/index.js (使用 --target bundler)
import * as wasm from '../pkg/my_lib.js';

wasm.default().then(() => {
  console.log(wasm.greet('World'));
});
```

## 八、Docker 部署

```dockerfile
# Dockerfile
FROM rust:latest AS builder

RUN rustup target add wasm32-unknown-unknown
RUN cargo install wasm-pack
RUN cargo install wasm-opt

WORKDIR /app
COPY . .

RUN wasm-pack build --release --target web
RUN wasm-opt -Oz -o pkg/optimized.wasm pkg/my_lib_bg.wasm

# 最终镜像
FROM nginx:alpine
COPY --from=builder /app/pkg /usr/share/html/pkg
COPY www/ /usr/share/html/
EXPOSE 80
```

## 九、版本发布自动化

```yaml
# .github/workflows/release.yml
name: Release

on:
  push:
    tags:
      - 'v*'

jobs:
  release:
    runs-on: ubuntu-latest
    permissions:
      contents: write
      packages: write

    steps:
      - uses: actions/checkout@v3

      - name: Setup Rust
        uses: actions-rs/toolchain@v1
        with:
          toolchain: stable
          target: wasm32-unknown-unknown

      - name: Install tools
        run: |
          curl https://rustwasm.github.io/wasm-pack/installer/init.sh -sSf | sh
          cargo install wasm-opt

      - name: Build
        run: |
          wasm-pack build --release --target web
          wasm-opt -Oz -o pkg/my_lib_bg.wasm pkg/my_lib_bg.wasm

      - name: Create GitHub Release
        uses: softprops/action-gh-release@v1
        with:
          files: pkg/*
          generate_release_notes: true

      - name: Publish to npm
        run: cd pkg && npm publish
        env:
          NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}

      - name: Deploy to CDN
        run: |
          # 上传到 Cloudflare R2 / AWS S3
          aws s3 sync pkg/ s3://my-cdn/wasm/v${{ github.ref_name }}/
```

## 十、部署检查清单

```
发布前：
[ ] 版本号已在 Cargo.toml 和 package.json 中同步
[ ] 所有测试通过（wasm-pack test --headless --chrome）
[ ] Release 构建完成
[ ] 包大小在预期范围内
[ ] CHANGELOG 已更新
[ ] README 包含最新用法示例

发布后：
[ ] npm 包可正常安装
[ ] CDN 文件可正常访问
[ ] TypeScript 类型定义正确
[ ] 在目标浏览器中测试通过
[ ] 监控错误率和性能指标
```
