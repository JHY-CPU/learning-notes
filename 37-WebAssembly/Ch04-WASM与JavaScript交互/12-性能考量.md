# 性能考量

## 一、概念说明

WASM-JS 互操作存在边界开销，每次调用都需要进行类型转换和上下文切换。优化策略是减少跨边界调用次数。

```rust
// 差：频繁跨边界调用
for i in 0..10000 {
    js_console_log(&format!("{}", i)); // 每次都跨边界
}

// 好：批量操作
let mut buffer = String::new();
for i in 0..10000 {
    buffer.push_str(&format!("{}\n", i));
}
js_console_log(&buffer); // 一次调用
```

## 二、具体用法

### 2.1 批量数据传输

```rust
// 差：逐个传输
#[wasm_bindgen]
pub fn process_items_bad(items: &js_sys::Array) {
    for i in 0..items.length() {
        let item = items.get(i);
        process_single(item);
    }
}

// 好：批量传输
#[wasm_bindgen]
pub fn process_items_good(data: &[u8]) -> Vec<u8> {
    // 直接操作字节数组，避免逐个转换
    data.iter().map(|&b| b.wrapping_add(1)).collect()
}
```

### 2.2 内存池模式

```rust
use std::cell::RefCell;

thread_local! {
    static BUFFER_POOL: RefCell<Vec<Vec<u8>>> = RefCell::new(Vec::new());
}

pub fn get_buffer(size: usize) -> Vec<u8> {
    BUFFER_POOL.with(|pool| {
        let mut pool = pool.borrow_mut();
        if let Some(mut buf) = pool.pop() {
            buf.clear();
            buf.reserve(size);
            buf
        } else {
            Vec::with_capacity(size)
        }
    })
}

pub fn return_buffer(buf: Vec<u8>) {
    BUFFER_POOL.with(|pool| {
        pool.borrow_mut().push(buf);
    });
}
```

### 2.3 避免不必要的分配

```rust
// 差：每次创建新 String
#[wasm_bindgen]
pub fn format_bad(values: &[f64]) -> String {
    let mut result = String::new();
    for v in values {
        result.push_str(&format!("{}, ", v)); // 每次 format! 创建新 String
    }
    result
}

// 好：预分配 + write!
use std::fmt::Write;
#[wasm_bindgen]
pub fn format_good(values: &[f64]) -> String {
    let mut result = String::with_capacity(values.len() * 20);
    for v in values {
        write!(result, "{}, ", v).unwrap();
    }
    result
}
```

### 2.4 零拷贝传输

```rust
#[wasm_bindgen]
pub fn process_shared_buffer(ptr: *mut u8, len: usize) {
    // 直接操作 WASM 内存，零拷贝
    let data = unsafe { std::slice::from_raw_parts_mut(ptr, len) };
    for byte in data.iter_mut() {
        *byte = byte.wrapping_mul(2);
    }
}
```

### 2.5 性能测量

```rust
use web_sys::Performance;

pub fn measure<F, T>(label: &str, f: F) -> T
where
    F: FnOnce() -> T,
{
    let perf = web_sys::window().unwrap().performance().unwrap();
    let start = perf.now();
    let result = f();
    let elapsed = perf.now() - start;
    web_sys::console::log_1(&format!("{}: {:.2}ms", label, elapsed).into());
    result
}

// 使用
pub fn benchmark() {
    measure("数据处理", || {
        let data = vec![0u8; 1024 * 1024];
        data.iter().sum::<u8>()
    });
}
```

## 三、注意事项与常见陷阱

1. **边界开销**：每次 WASM-JS 调用约有微秒级开销，应批量处理
2. **内存复制**：String/Vec 传输会复制数据，考虑使用引用或共享内存
3. **GC 压力**：频繁创建 JsValue 会增加 GC 负担
4. **缓存友好**：数据连续存储比链表等结构更适合 WASM
5. **Profile 实测**：不要凭直觉优化，使用 Chrome DevTools Performance 面板分析
