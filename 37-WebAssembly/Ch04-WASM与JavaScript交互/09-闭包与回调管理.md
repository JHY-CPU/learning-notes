# 闭包与回调管理

## 一、概念说明

Rust 闭包传递到 JavaScript 需要特殊处理，因为 JS 垃圾回收器和 Rust 所有权系统需要协调。

```rust
use wasm_bindgen::prelude::*;

// Closure 包装 Rust 闭包供 JS 使用
let closure = Closure::wrap(Box::new(|event: web_sys::Event| {
    web_sys::console::log_1(&"点击事件触发".into());
}) as Box<dyn FnMut(_)>);

// 传递给 JS
element.set_onclick(Some(closure.as_ref().unchecked_ref()));

// 忘记闭包防止被 Drop
closure.forget();
```

## 二、具体用法

### 2.1 Closure 基础用法

```rust
use wasm_bindgen::prelude::*;

#[wasm_bindgen]
pub fn register_callback(callback: js_sys::Function) {
    // 调用 JavaScript 回调
    let this = JsValue::null();
    let arg = JsValue::from_str("来自 WASM");
    callback.call1(&this, &arg).unwrap();
}
```

### 2.2 一次性回调

```rust
pub fn on_timeout(callback: js_sys::Function, ms: i32) {
    let window = web_sys::window().unwrap();
    window
        .set_timeout_with_callback_and_timeout_and_arguments_0(
            &callback,
            ms,
        )
        .unwrap();
}
```

### 2.3 可重用闭包

```rust
use std::cell::RefCell;
use std::rc::Rc;

pub fn create_click_handler() -> Closure<dyn FnMut(web_sys::MouseEvent)> {
    let count = Rc::new(RefCell::new(0u32));

    Closure::wrap(Box::new(move |event: web_sys::MouseEvent| {
        *count.borrow_mut() += 1;
        let target = event.target().unwrap();
        let el: web_sys::HtmlElement = target.dyn_into().unwrap();
        el.set_inner_text(&format!("点击次数: {}", count.borrow()));
    }))
}
```

### 2.4 防止内存泄漏

```rust
pub struct EventHandler {
    closure: Option<Closure<dyn FnMut(web_sys::Event)>>,
    element: web_sys::Element,
}

impl EventHandler {
    pub fn new(element: web_sys::Element) -> Self {
        let closure = Closure::wrap(Box::new(|_: web_sys::Event| {
            web_sys::console::log_1(&"处理事件".into());
        }) as Box<dyn FnMut(_)>);

        element
            .add_event_listener_with_callback("click", closure.as_ref().unchecked_ref())
            .unwrap();

        Self {
            closure: Some(closure),
            element,
        }
    }
}

impl Drop for EventHandler {
    fn drop(&mut self) {
        if let Some(ref closure) = self.closure {
            self.element
                .remove_event_listener_with_callback("click", closure.as_ref().unchecked_ref())
                .unwrap();
        }
    }
}
```

### 2.5 函数指针 vs 闭包

```rust
// 函数指针无需 Closure 包装
#[wasm_bindgen]
pub fn simple_callback(f: &js_sys::Function) {
    f.call0(&JsValue::null()).unwrap();
}

// Rust 闭包需要 Closure 包装
pub fn with_closure<F>(f: F)
where
    F: FnMut(i32) + 'static,
{
    let closure = Closure::wrap(Box::new(f) as Box<dyn FnMut(_)>);
    // 使用 closure...
    closure.forget(); // 防止 Drop
}
```

## 三、注意事项与常见陷阱

1. **内存泄漏**：`forget()` 会永久分配闭包，应考虑使用 Drop 清理
2. **生命周期**：闭包必须是 `'static`，不能捕获短生命周期引用
3. **类型签名**：闭包参数类型必须与 JS 侧匹配
4. **错误传播**：闭包中的 panic 会导致未定义行为，需转换为 JS 异常
5. **性能**：频繁创建 Closure 有开销，尽量复用
