# 错误处理

## 一、概念说明

WASM 与 JavaScript 之间的错误需要在两个世界间传递和转换。Rust 的 `Result<T, JsValue>` 可以自动转换为 JavaScript 异常。

```rust
use wasm_bindgen::prelude::*;

#[wasm_bindgen]
pub fn divide(a: f64, b: f64) -> Result<f64, JsValue> {
    if b == 0.0 {
        Err(JsValue::from_str("除数不能为零"))
    } else {
        Ok(a / b)
    }
}
```

```javascript
try {
    const result = divide(10, 0);
} catch (e) {
    console.error(e); // "除数不能为零"
}
```

## 二、具体用法

### 2.1 错误类型转换

```rust
// 使用 JsValue 包装错误
#[wasm_bindgen]
pub fn parse_json(input: &str) -> Result<JsValue, JsValue> {
    let parsed: serde_json::Value = serde_json::from_str(input)
        .map_err(|e| JsValue::from_str(&format!("JSON 解析错误: {}", e)))?;
    Ok(JsValue::from_serde(&parsed).unwrap())
}
```

### 2.2 自定义错误类型

```rust
#[wasm_bindgen]
pub struct AppError {
    message: String,
    code: u32,
}

#[wasm_bindgen]
impl AppError {
    #[wasm_bindgen(getter)]
    pub fn message(&self) -> String { self.message.clone() }

    #[wasm_bindgen(getter)]
    pub fn code(&self) -> u32 { self.code }
}

#[wasm_bindgen]
pub fn validate_input(input: &str) -> Result<String, AppError> {
    if input.is_empty() {
        Err(AppError {
            message: "输入不能为空".to_string(),
            code: 400,
        })
    } else {
        Ok(input.to_uppercase())
    }
}
```

### 2.3 捕获 JS 异常

```rust
use wasm_bindgen::JsCast;

pub fn call_js_function(func: &js_sys::Function) -> Result<JsValue, JsValue> {
    let result = func.call0(&JsValue::null());
    // func.call0 返回 Result<JsValue, JsValue>
    // 如果 JS 函数抛出异常，会得到 Err
    result
}

pub fn safe_eval(code: &str) -> Result<JsValue, String> {
    let result = js_sys::eval(code);
    result.map_err(|e| {
        e.as_string().unwrap_or_else(|| "未知错误".to_string())
    })
}
```

### 2.4 panic 转换

```rust
use console_error_panic_hook;

#[wasm_bindgen(start)]
pub fn main() {
    // 安装 panic hook，将 Rust panic 转为 JS console.error
    console_error_panic_hook::set_once();
}

#[wasm_bindgen]
pub fn may_panic(should_panic: bool) -> String {
    if should_panic {
        panic!("发生 panic!");
    }
    "正常".to_string()
}
// panic 会变成 JS 的 RuntimeError
```

### 2.5 异步错误处理

```rust
use wasm_bindgen_futures::JsFuture;

#[wasm_bindgen]
pub async fn fetch_and_process(url: &str) -> Result<String, JsValue> {
    let window = web_sys::window().unwrap();

    // fetch 可能失败
    let resp = JsFuture::from(window.fetch_with_str(url))
        .await
        .map_err(|e| JsValue::from_str(&format!("网络错误: {:?}", e)))?;

    let resp: web_sys::Response = resp.dyn_into()?;

    if !resp.ok() {
        return Err(JsValue::from_str(&format!("HTTP {}: {}", resp.status(), resp.status_text())));
    }

    let text = JsFuture::from(resp.text()?)
        .await
        .map_err(|e| JsValue::from_str(&format!("读取失败: {:?}", e)))?;

    Ok(text.as_string().unwrap())
}
```

## 三、注意事项与常见陷阱

1. **panic 行为**：默认 Rust panic 会导致 WASM 未定义行为，必须安装 console_error_panic_hook
2. **错误信息**：JsValue::from_str 生成的错误类型固定，不易在 JS 侧区分
3. **性能影响**：异常捕获比 Result 检查更慢
4. **async 错误**：异步函数中的错误传播需要额外注意
5. **类型安全**：尽量使用结构化错误而非字符串错误
