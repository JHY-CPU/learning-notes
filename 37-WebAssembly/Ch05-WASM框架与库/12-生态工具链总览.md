# 生态工具链总览

## 一、概念说明

Rust WASM 生态由多个工具组成完整的工作流：开发、构建、测试、调试、发布。

```
源码(Rust) → wasm-bindgen → wasm-opt → 打包(Trunk/Webpack) → 部署
```

## 二、具体用法

### 2.1 工具矩阵

| 工具 | 用途 | 命令 |
|------|------|------|
| rustup | 安装 WASM target | `rustup target add wasm32-unknown-unknown` |
| wasm-bindgen | JS 绑定生成 | `wasm-bindgen --target web` |
| wasm-pack | 全流程构建 | `wasm-pack build` |
| wasm-opt | 二进制优化 | `wasm-opt -Os input.wasm -o output.wasm` |
| wasm-snip | 移除未使用代码 | `wasm-snip input.wasm -o output.wasm` |
| Trunk | Web 应用构建 | `trunk build --release` |
| wasm-bindgen-test | 测试框架 | `wasm-pack test --chrome` |

### 2.2 Cargo 优化配置

```toml
# Cargo.toml
[profile.release]
opt-level = "s"         # 优先体积
lto = true              # 全链接优化
codegen-units = 1       # 单编译单元
strip = true            # 移除调试信息
panic = "abort"         # 减少 panic 开销

[profile.release.package."*"]
opt-level = "s"
```

### 2.3 构建脚本自动化

```bash
#!/bin/bash
# build.sh

# 1. 编译 WASM
cargo build --target wasm32-unknown-unknown --release

# 2. wasm-bindgen 处理
wasm-bindgen target/wasm32-unknown-unknown/release/my_app.wasm \
    --out-dir pkg --target web --no-typescript

# 3. wasm-opt 优化
wasm-opt -Os pkg/my_app_bg.wasm -o pkg/my_app_bg.wasm

# 4. 压缩
gzip -k -f pkg/my_app_bg.wasm

echo "构建完成: $(du -sh pkg/my_app_bg.wasm | cut -f1)"
```

### 2.4 多目标构建

```toml
# Cargo.toml
[lib]
crate-type = ["cdylib", "rlib"]

# cdylib: WASM 模块
# rlib: Rust 库（用于测试和其他 Rust crate）
```

### 2.5 依赖管理

```toml
# 只在 WASM 编译时包含
[target.'cfg(target_arch = "wasm32")'.dependencies]
web-sys = "0.3"
wasm-bindgen = "0.2"
console_error_panic_hook = "0.1"
wasm-logger = "0.2"

# 只在非 WASM 时包含
[target.'cfg(not(target_arch = "wasm32"))'.dependencies]
tokio = { version = "1", features = ["full"] }
```

## 三、注意事项与常见陷阱

1. **工具版本同步**：wasm-bindgen CLI 和 crate 版本必须匹配
2. **优化链顺序**：先 wasm-bindgen，后 wasm-opt，顺序不能错
3. **调试工具**：Chrome DevTools 支持 WASM 源码映射（需配置）
4. **CI 环境**：GitHub Actions 需要安装 WASM target 和工具
5. **体积分析**：使用 twiggy 或 wasm-size 分析 WASM 二进制组成
