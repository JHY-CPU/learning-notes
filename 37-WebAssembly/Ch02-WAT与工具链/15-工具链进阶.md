# 工具链进阶

## 一、概念说明

WASM 工具链的进阶用法，包括自定义优化、交叉编译和高级集成。

```bash
# 进阶功能
# 1. 自定义优化 passes
# 2. 交叉编译
# 3. 高级链接选项
# 4. 自定义运行时
# 5. 性能调优
```

## 二、具体用法

### 2.1 自定义优化

```bash
# 使用 Binaryen 自定义优化
wasm-opt input.wasm -o output.wasm \
  --enable-simd \
  --enable-bulk-memory \
  --enable-mutable-globals \
  --optimize-level=3 \
  --shrink-level=0

# 查看优化 passes
wasm-opt --print-after-all input.wasm -o /dev/null
```

### 2.2 高级链接

```bash
# 分离编译和链接
emcc -c main.c -o main.o
emcc -c utils.c -o utils.o
emcc main.o utils.o -o app.js

# 链接静态库
emar rcs libmylib.a module1.o module2.o
emcc main.c -lmylib -L. -o app.js

# 链接动态库（WASM 模块）
emcc main.c -s MAIN_MODULE=1 -o app.js
```

### 2.3 自定义运行时

```javascript
// 自定义 Emscripten 运行时
```

### 2.4 性能调优

```bash
# 编译时优化
emcc -O3 -flto -s ALLOW_MEMORY_GROWTH=0 main.c -o app.js

# 运行时优化
wasm-opt -O3 --enable-simd app.wasm -o app.opt.wasm

# 链接时优化
emcc -O3 -flto main.c utils.c -o app.js
```

## 五、多模块动态链接

```bash
# 创建共享库模块
emcc -c lib.c -o lib.o
emar rcs libmylib.a lib.o

# 创建 SIDE_MODULE（动态加载）
emcc lib.c -s SIDE_MODULE=1 -o lib.wasm

# 创建 MAIN_MODULE（可以加载 SIDE_MODULE）
emcc main.c -s MAIN_MODULE=1 -o app.js

# JavaScript 中动态加载模块
const module = await WebAssembly.compileStreaming(fetch('lib.wasm'));
// 使用 Emscripten 的动态链接 API
```

## 六、自定义 Binaryen Pass

```javascript
// 使用 Binaryen.js API 创建自定义优化
const binaryen = require("binaryen");

// 读取模块
const module = binaryen.parseText(`
  (module
    (func $add (param i32 i32) (result i32)
      local.get 0
      local.get 1
      i32.add)
    (export "add" (func $add)))
`);

// 运行优化
module.optimize();

// 输出
console.log(module.emitText());  // WAT
const wasm = module.emitBinary(); // WASM

// 自定义 pass 示例：打印所有函数名
const numFunctions = module.getNumFunctions();
for (let i = 0; i < numFunctions; i++) {
  console.log(module.getFunctionByIndex(i).name);
}
```

## 八、性能调优工作流

```bash
#!/bin/bash
# 性能调优脚本

# 1. 基准测试
echo "=== 基准测试 ==="
time node bench.js  # JavaScript 基准

# 2. 构建并测试各级优化
for OPT in "-O0" "-O1" "-O2" "-O3" "-Os" "-Oz"; do
  echo "=== 优化级别: ${OPT} ==="
  wasm-opt ${OPT} input.wasm -o opt.wasm

  SIZE=$(wc -c < opt.wasm)
  echo "大小: ${SIZE} bytes"

  # 运行性能测试
  node -e "
    const fs = require('fs');
    const bytes = fs.readFileSync('opt.wasm');
    WebAssembly.instantiate(bytes).then(({instance}) => {
      const start = performance.now();
      for(let i = 0; i < 1000; i++) {
        instance.exports.benchmark();
      }
      const elapsed = performance.now() - start;
      console.log('耗时:', elapsed.toFixed(2), 'ms');
    });
  "
done

# 3. 汇总结果
echo "=== 结果汇总 ==="
echo "根据大小和速度选择最优配置"
```

## 九、交叉编译和目标平台

```bash
# Emscripten 目标平台
emcc main.c -o app.js          # 默认：浏览器
emcc main.c -o app.wasm -s STANDALONE_WASM  # 独立 WASM
emcc main.c -o app.html        # 完整 HTML 页面

# Rust 目标平台
rustup target add wasm32-unknown-unknown  # 通用 WASM
rustup target add wasm32-wasi             # WASI
rustup target add wasm32-unknown-emscripten  # Emscripten

# TinyGo 目标平台
tinygo build -target=wasi main.go -o main.wasm       # WASI
tinygo build -target=wasm main.go -o main.wasm       # 浏览器
tinygo build -target=wasm-unknown main.go -o main.wasm  # 最小化
```
