# 测试与验证

## 一、概念说明

WASM 代码需要充分测试，包括单元测试、集成测试和性能测试。

```javascript
// 单元测试
describe('WASM 模块', () => {
  it('应该正确计算加法', async () => {
    const { instance } = await WebAssembly.instantiate(bytes);
    expect(instance.exports.add(1, 2)).toBe(3);
  });
});
```

## 二、具体用法

### 2.1 JavaScript 测试

```javascript
// Jest 测试
const fs = require('fs');
const path = require('path');

describe('WASM 模块', () => {
  let instance;

  beforeAll(async () => {
    const bytes = fs.readFileSync(path.join(__dirname, 'module.wasm'));
    const module = await WebAssembly.compile(bytes);
    instance = await WebAssembly.instantiate(module);
  });

  test('add 函数', () => {
    expect(instance.exports.add(1, 2)).toBe(3);
    expect(instance.exports.add(-1, 1)).toBe(0);
    expect(instance.exports.add(0, 0)).toBe(0);
  });

  test('factorial 函数', () => {
    expect(instance.exports.factorial(0)).toBe(1);
    expect(instance.exports.factorial(5)).toBe(120);
    expect(instance.exports.factorial(10)).toBe(3628800);
  });
});
```

### 2.2 Rust 测试

```rust
// wasm-bindgen-test
use wasm_bindgen_test::*;

wasm_bindgen_test_configure!(run_in_browser);

#[wasm_bindgen_test]
fn test_add() {
    assert_eq!(add(1, 2), 3);
}

#[wasm_bindgen_test]
fn test_factorial() {
    assert_eq!(factorial(0), 1);
    assert_eq!(factorial(5), 120);
}
```

### 2.3 性能测试

```javascript
// 性能基准
describe('性能测试', () => {
  test('WASM vs JavaScript', () => {
    const iterations = 1000000;

    // JavaScript
    const jsStart = performance.now();
    for (let i = 0; i < iterations; i++) {
      jsFibonacci(20);
    }
    const jsTime = performance.now() - jsStart;

    // WASM
    const wasmStart = performance.now();
    for (let i = 0; i < iterations; i++) {
      instance.exports.fibonacci(20);
    }
    const wasmTime = performance.now() - wasmStart;

    console.log(`JavaScript: ${jsTime}ms`);
    console.log(`WASM: ${wasmTime}ms`);
    console.log(`加速比: ${jsTime / wasmTime}x`);
  });
});
```

### 2.4 验证工具

```bash
# WASM 验证
wasm-validate module.wasm

# 使用 WABT 测试
wasm-interp module.wasm --run-all-exports

# 使用 wasmer 测试
wasmer run module.wasm
```

## 三、注意事项与常见陷阱

1. **测试覆盖**：确保测试覆盖所有功能
2. **边界条件**：测试边界条件
3. **性能基准**：建立性能基准
4. **回归测试**：防止性能回归
5. **自动化**：集成到 CI/CD 流程
