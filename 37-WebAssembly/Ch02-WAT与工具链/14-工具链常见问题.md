## 二、具体用法

### 2.1 编译问题

```bash
# 问题：找不到头文件
# 解决：检查 include 路径
emcc main.c -I/path/to/headers

# 问题：链接错误
# 解决：检查库路径和链接顺序
emcc main.c -L/path/to/libs -lmylib

# 问题：内存不足
# 解决：增加初始内存
emcc main.c -s INITIAL_MEMORY=16777216
```

### 2.2 运行时问题

```javascript
// 问题：WASM 加载失败
// 解决：检查 MIME 类型和路径
fetch('module.wasm', {
  headers: {
    'Content-Type': 'application/wasm'
  }
});

// 问题：导入函数未定义
// 解决：检查导入对象
const imports = {
  env: {
    memory: new WebAssembly.Memory({ initial: 1 }),
    // 确保所有导入函数都定义
  }
};
```

### 2.3 性能问题

```bash
# 问题：二进制文件太大
# 解决：使用优化编译
emcc main.c -O2 -s ALLOW_MEMORY_GROWTH=0

# 问题：执行速度慢
# 解决：启用 SIMD 和多线程
emcc main.c -msimd128 -s USE_PTHREADS=1

# 问题：加载时间长
# 解决：使用流式编译
WebAssembly.instantiateStreaming(fetch('module.wasm'), imports);
```

### 2.4 调试问题

```bash
# 问题：无法设置断点
# 解决：启用调试信息
emcc main.c -g4 -s ASSERTIONS=1

# 问题：Source Maps 不工作
# 解决：检查 Source Map 配置
emcc main.c -g4 --source-map-base http://localhost:8080/
```

## 五、Rust + wasm-pack 常见问题

```bash
# 问题：找不到 wasm32-unknown-unknown 目标
# 解决：
rustup target add wasm32-unknown-unknown

# 问题：wasm-pack build 报错 "crate-type must be cdylib"
# 解决：检查 Cargo.toml
[lib]
crate-type = ["cdylib", "rlib"]

# 问题：web-sys 特性缺失
# 解决：在 Cargo.toml 中启用特性
[dependencies.web-sys]
version = "0.3"
features = [
  "Document",
  "Element",
  "HtmlElement",
  "Window",
  "console"
]

# 问题：编译后 wasm 文件过大
# 解决：
# 1. 使用 wee_alloc
# 2. 启用 LTO
[profile.release]
lto = true
opt-level = "s"
codegen-units = 1
panic = "abort"

# 问题：async/await 不工作
# 解决：使用 wasm-bindgen-futures
[dependencies]
wasm-bindgen-futures = "0.4"
```

## 六、Emscripten 常见问题

```bash
# 问题：emcc 找不到
# 解决：
source ./emsdk_env.sh
# 或添加到 PATH
export PATH=$HOME/emsdk:$PATH

# 问题：编译 C++ 模板代码出错
# 解决：增加模板深度
em++ -ftemplate-depth=1024 main.cpp -o app.js

# 问题：运行时报错 "out of memory"
# 解决：增加初始内存
emcc main.c -s INITIAL_MEMORY=33554432 -o app.js

# 问题：SDL 链接错误
# 解决：
emcc game.c -s USE_SDL=2 -s USE_SDL_IMAGE=2 -s SDL2_IMAGE_FORMATS='["png"]' -o game.js

# 问题：生成的 JS 文件太大
# 解决：
emcc main.c -O2 -s MINIMAL_RUNTIME=1 -s NO_FILESYSTEM=1 -o app.js
```

## 七、AssemblyScript 常见问题

```bash
# 问题：类型不匹配
# 解决：AssemblyScript 是严格的静态类型
# 必须显式类型转换
let x: i32 = <i32>someFloat;

# 问题：不能使用 JavaScript 对象
# 解决：使用外部声明
@external("env", "consoleLog")
declare function consoleLog(ptr: i32): void;

# 问题：字符串传递复杂
# 解决：使用 String.UTF8.encode/decode
import { String } from "assemblyscript";

export function processString(str: string): string {
  const buf = String.UTF8.encode(str);
  const ptr = changetype<usize>(buf);
  const len = buf.byteLength;
  // 处理...
  return String.UTF8.decode(buf);
}
```

## 八、通用问题排查流程

```
遇到问题：
├─ 读取完整的错误信息（不要跳过）
├─ 检查工具链版本
│  ├─ Emscripten: emcc --version
│  ├─ wasm-pack: wasm-pack --version
│  ├─ TinyGo: tinygo version
│  └─ Binaryen: wasm-opt --version
├─ 搜索已知问题
│  ├─ GitHub Issues
│  ├─ Stack Overflow
│  └─ 工具链文档
├─ 创建最小复现
│  ├─ 最小的源码文件
│  ├─ 精确的命令
│  └─ 完整的错误输出
└─ 提交 Issue
   ├─ 工具链版本
   ├─ 操作系统
   ├─ 最小复现
   └─ 完整错误信息
```
