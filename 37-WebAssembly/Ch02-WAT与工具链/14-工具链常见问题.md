## 二、具体用法

### 2.1 编译问题

```bash
# 问题：找不到头文件
# 解决：检查 include 路径
emcc main.c -I/path/to/headers

# 问题：链接错误
# 解决：检查库路径和链接顺序
emcc main.c -L/path/to/libs -lmylib

# 问题：内存不足
# 解决：增加初始内存
emcc main.c -s INITIAL_MEMORY=16777216
```

### 2.2 运行时问题

```javascript
// 问题：WASM 加载失败
// 解决：检查 MIME 类型和路径
fetch('module.wasm', {
  headers: {
    'Content-Type': 'application/wasm'
  }
});

// 问题：导入函数未定义
// 解决：检查导入对象
const imports = {
  env: {
    memory: new WebAssembly.Memory({ initial: 1 }),
    // 确保所有导入函数都定义
  }
};
```

### 2.3 性能问题

```bash
# 问题：二进制文件太大
# 解决：使用优化编译
emcc main.c -O2 -s ALLOW_MEMORY_GROWTH=0

# 问题：执行速度慢
# 解决：启用 SIMD 和多线程
emcc main.c -msimd128 -s USE_PTHREADS=1

# 问题：加载时间长
# 解决：使用流式编译
WebAssembly.instantiateStreaming(fetch('module.wasm'), imports);
```

### 2.4 调试问题

```bash
# 问题：无法设置断点
# 解决：启用调试信息
emcc main.c -g4 -s ASSERTIONS=1

# 问题：Source Maps 不工作
# 解决：检查 Source Map 配置
emcc main.c -g4 --source-map-base http://localhost:8080/
```
