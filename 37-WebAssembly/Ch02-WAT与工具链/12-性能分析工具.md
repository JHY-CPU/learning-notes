# 性能分析工具

## 一、概念说明

WASM 性能分析包括浏览器性能工具、命令行分析器和专用分析工具。

```javascript
// Chrome Performance 面板
// 1. 打开 DevTools
// 2. Performance 面板
// 3. 开始记录
// 4. 执行操作
// 5. 停止记录
// 6. 分析结果
```

## 二、具体用法

### 2.1 Chrome 分析

```javascript
// 性能标记
performance.mark('wasm-start');
instance.exports.heavyComputation();
performance.mark('wasm-end');
performance.measure('wasm', 'wasm-start', 'wasm-end');

// 查看性能条目
const entries = performance.getEntriesByName('wasm');
console.log('执行时间:', entries[0].duration, 'ms');
```

### 2.2 命令行分析

```bash
# 使用 wasmtime 分析
wasmtime --profile module.wasm

# 使用 perf 分析
perf record --call-graph=dwarf wasmtime module.wasm
perf report

# 使用 flamegraph
cargo flamegraph -- wasmtime module.wasm
```

### 2.3 内存分析

```javascript
// 内存使用分析
const memory = instance.exports.memory;
console.log('内存页数:', memory.buffer.byteLength / 65536);
console.log('内存大小:', memory.buffer.byteLength, '字节');

// 内存增长监控
memory.grow(1);
console.log('增长后内存:', memory.buffer.byteLength);
```

### 2.4 自定义分析

```javascript
// 性能分析器
class WasmProfiler {
  constructor(instance) {
    this.instance = instance;
    this.timings = new Map();
  }

  profile(fnName, ...args) {
    const fn = this.instance.exports[fnName];
    const start = performance.now();
    const result = fn(...args);
    const duration = performance.now() - start;

    if (!this.timings.has(fnName)) {
      this.timings.set(fnName, []);
    }
    this.timings.get(fnName).push(duration);

    return result;
  }

  report() {
    for (const [fn, times] of this.timings) {
      const avg = times.reduce((a, b) => a + b) / times.length;
      console.log(`${fn}: 平均 ${avg.toFixed(2)}ms (${times.length} 次调用)`);
    }
  }
}
```

## 三、注意事项与常见陷阱

1. **测量干扰**：分析工具可能影响性能
2. **采样偏差**：采样可能有偏差
3. **内存开销**：分析工具有内存开销
4. **浏览器差异**：不同浏览器性能可能不同
5. **实际环境**：在目标环境测试性能
