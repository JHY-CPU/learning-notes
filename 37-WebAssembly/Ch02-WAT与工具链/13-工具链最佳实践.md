# 工具链最佳实践

## 一、概念说明

总结 WASM 工具链的最佳实践，帮助高效开发和部署。

```bash
# 最佳实践清单
# 1. 选择合适的工具链
# 2. 配置构建优化
# 3. 集成到现有流程
# 4. 持续测试和验证
# 5. 性能监控和优化
```

## 二、具体用法

### 2.1 项目结构

```
my-wasm-project/
├── src/
│   ├── lib.rs (或 main.c)
│   └── utils.rs
├── tests/
│   ├── unit/
│   └── integration/
├── dist/
│   ├── module.wasm
│   └── module.js
├── scripts/
│   ├── build.sh
│   └── test.sh
├── Cargo.toml (或 Makefile)
├── package.json
└── README.md
```

### 2.2 构建脚本

```bash
#!/bin/bash
# build.sh

# 清理
rm -rf dist/

# 构建
if [ "$1" = "release" ]; then
  wasm-pack build --release
else
  wasm-pack build --dev
fi

# 优化
wasm-opt -O3 dist/module.wasm -o dist/module.opt.wasm

# 测试
npm test

# 打包大小检查
wc -c dist/module.wasm
```

### 2.3 CI/CD 集成

```yaml
# GitHub Actions
name: WASM CI
on: [push, pull_request]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Install Rust
        uses: actions-rs/toolchain@v1
        with:
          toolchain: stable
      - name: Build WASM
        run: wasm-pack build --release
      - name: Test
        run: npm test
      - name: Upload artifact
        uses: actions/upload-artifact@v3
        with:
          name: wasm-module
          path: dist/module.wasm
```

### 2.4 版本管理

```bash
# 使用语义化版本
# MAJOR.MINOR.PATCH

# 发布流程
git tag v1.2.3
git push --tags
npm publish
```
