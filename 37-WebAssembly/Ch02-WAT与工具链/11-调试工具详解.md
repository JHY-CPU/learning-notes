# 调试工具详解

## 一、概念说明

WASM 调试工具包括浏览器开发者工具、命令行调试器和 Source Map 支持。

```javascript
// Chrome DevTools
// 1. Sources 面板
// 2. 找到 WASM 文件
// 3. 查看反汇编
// 4. 设置断点
// 5. 单步执行
```

## 二、具体用法

### 2.1 Chrome DevTools

```javascript
// 启用 WASM 调试
// chrome://flags/#enable-webassembly-debugging

// 调试步骤
// 1. 打开 DevTools
// 2. Sources 面板
// 3. 找到 WASM 文件
// 4. 查看 WAT 格式
// 5. 设置断点
// 6. 使用调试控制
```

### 2.2 Source Maps

```javascript
// 生成 Source Maps
// Emscripten
emcc main.c -g4 -o main.js

// wasm-pack
wasm-pack build --dev

// Source Map 文件
// module.wasm.map
```

### 2.3 Firefox 调试

```javascript
// Firefox 开发者工具
// 1. 调试器面板
// 2. 找到 WASM 文件
// 3. 查看源代码
// 4. 设置断点

// 启用 WASM 调试
// about:config
// javascript.options.wasm_baselinejit = true
// javascript.options.wasm_optimizingjit = true
```

### 2.4 命令行调试

```bash
# 使用 wasmtime 调试
wasmtime --debug module.wasm

# 使用 gdb 调试
gdb --args wasmtime module.wasm

# 使用 lldb 调试
lldb -- wasmtime module.wasm
```

### 2.5 日志调试

```javascript
// 导入日志函数
const imports = {
  env: {
    log: (value) => console.log('WASM:', value),
    log_string: (ptr, len) => {
      const bytes = new Uint8Array(instance.exports.memory.buffer, ptr, len);
      console.log('WASM:', new TextDecoder().decode(bytes));
    }
  }
};
```

## 三、注意事项与常见陷阱

1. **调试信息**：调试版本需要调试信息
2. **Source Maps**：正确配置 Source Maps
3. **性能影响**：调试模式影响性能
4. **工具支持**：不同工具有不同调试支持
5. **学习曲线**：调试工具有一定学习曲线

## 六、Rust WASM 调试配置

```bash
# Cargo.toml 中启用调试信息
[profile.release]
debug = true

# 或使用 dev 构建进行调试
wasm-pack build --dev

# 启用 backtrace
RUST_BACKTRACE=1 wasm-pack test --chrome
```

## 七、Chrome DevTools 详细操作

```
1. 打开 DevTools (F12)
2. 启用 WASM 调试：
   - 访问 chrome://flags
   - 搜索 "WebAssembly"
   - 启用 "WebAssembly debugging"
   - 重启浏览器
3. Sources 面板：
   - 左侧文件树找到 .wasm 文件
   - 点击查看反汇编的 WAT 代码
   - 点击行号设置断点
4. 调试控制：
   - F8: 继续执行
   - F10: 单步跳过
   - F11: 单步进入
   - Shift+F11: 单步跳出
5. 查看变量：
   - Scope 面板查看局部变量
   - 可以查看 i32/i64/f32/f64 值
   - 也可以查看 WASM 内存内容
6. 内存查看：
   - 在 Console 中使用：
   new Uint8Array(instance.exports.memory.buffer, offset, length)
```

## 八、Firefox 调试配置

```
Firefox 开发者工具：
1. 打开 about:config
2. 确认以下设置：
   javascript.options.wasm_baselinejit = true
   javascript.options.wasm_optimizingjit = true
3. 调试器面板：
   - 点击左上角 "{}" 按钮格式化代码
   - 搜索 WASM 文件名
   - 设置断点、查看调用栈
4. Firefox 特有优势：
   - 更好的 WAT 反汇编显示
   - 支持查看 WASM 内存内容
   - Profiler 支持 WASM 函数分析
```

## 九、集成调试工作流

```bash
# 完整的调试工作流示例

# 1. 构建调试版本
wasm-pack build --dev

# 2. 启动开发服务器（带 Source Map 支持）
npx webpack serve --mode development

# 3. 在 Chrome 中打开页面，触发 WASM 调用

# 4. 使用 wasm-objdump 离线分析
wasm-objdump -d pkg/my_lib_bg.wasm > disassembly.wat

# 5. 对比源码和反汇编结果

# 6. 使用 Performance API 定位性能瓶颈
performance.mark('wasm-start');
instance.exports.heavyFunction();
performance.mark('wasm-end');
performance.measure('wasm', 'wasm-start', 'wasm-end');

# 7. 使用 Memory 面板分析内存
# Chrome DevTools → Memory → Heap Snapshot
# 找到 WASM 内存的增长模式
```

## 十、常见调试问题速查

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 看不到源码 | 没有调试信息 | 使用 `--dev` 或 `-g` 编译 |
| 断点不生效 | 优化移除了代码 | 禁用优化或使用 `--dev` |
| 变量值显示 "?" | 寄存器分配优化 | 使用 `-O0` 编译 |
| Source Map 不匹配 | 版本不一致 | 清理缓存重新构建 |
| 内存视图为空 | 内存未导出 | 确保导出 `memory` |
