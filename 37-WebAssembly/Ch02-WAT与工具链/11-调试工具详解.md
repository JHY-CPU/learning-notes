# 调试工具详解

## 一、概念说明

WASM 调试工具包括浏览器开发者工具、命令行调试器和 Source Map 支持。

```javascript
// Chrome DevTools
// 1. Sources 面板
// 2. 找到 WASM 文件
// 3. 查看反汇编
// 4. 设置断点
// 5. 单步执行
```

## 二、具体用法

### 2.1 Chrome DevTools

```javascript
// 启用 WASM 调试
// chrome://flags/#enable-webassembly-debugging

// 调试步骤
// 1. 打开 DevTools
// 2. Sources 面板
// 3. 找到 WASM 文件
// 4. 查看 WAT 格式
// 5. 设置断点
// 6. 使用调试控制
```

### 2.2 Source Maps

```javascript
// 生成 Source Maps
// Emscripten
emcc main.c -g4 -o main.js

// wasm-pack
wasm-pack build --dev

// Source Map 文件
// module.wasm.map
```

### 2.3 Firefox 调试

```javascript
// Firefox 开发者工具
// 1. 调试器面板
// 2. 找到 WASM 文件
// 3. 查看源代码
// 4. 设置断点

// 启用 WASM 调试
// about:config
// javascript.options.wasm_baselinejit = true
// javascript.options.wasm_optimizingjit = true
```

### 2.4 命令行调试

```bash
# 使用 wasmtime 调试
wasmtime --debug module.wasm

# 使用 gdb 调试
gdb --args wasmtime module.wasm

# 使用 lldb 调试
lldb -- wasmtime module.wasm
```

### 2.5 日志调试

```javascript
// 导入日志函数
const imports = {
  env: {
    log: (value) => console.log('WASM:', value),
    log_string: (ptr, len) => {
      const bytes = new Uint8Array(instance.exports.memory.buffer, ptr, len);
      console.log('WASM:', new TextDecoder().decode(bytes));
    }
  }
};
```

## 三、注意事项与常见陷阱

1. **调试信息**：调试版本需要调试信息
2. **Source Maps**：正确配置 Source Maps
3. **性能影响**：调试模式影响性能
4. **工具支持**：不同工具有不同调试支持
5. **学习曲线**：调试工具有一定学习曲线
