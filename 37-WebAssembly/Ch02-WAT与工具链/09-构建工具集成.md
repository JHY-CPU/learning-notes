# 构建工具集成

## 一、概念说明

将 WASM 构建集成到现有 JavaScript 构建流程中。

```javascript
// Webpack 配置
module.exports = {
  experiments: {
    asyncWebAssembly: true,
  },
  module: {
    rules: [
      {
        test: /\.wasm$/,
        type: 'webassembly/async',
      },
    ],
  },
};
```

## 二、具体用法

### 2.1 Webpack

```javascript
// webpack.config.js
const path = require('path');

module.exports = {
  entry: './src/index.js',
  output: {
    path: path.resolve(__dirname, 'dist'),
    filename: 'bundle.js',
  },
  experiments: {
    asyncWebAssembly: true,
  },
  module: {
    rules: [
      {
        test: /\.wasm$/,
        type: 'webassembly/async',
      },
    ],
  },
};

// 使用
import * as wasm from './module.wasm';
const result = await wasm.add(1, 2);
```

### 2.2 Vite

```javascript
// vite.config.js
import { defineConfig } from 'vite';

export default defineConfig({
  plugins: [
    {
      name: 'wasm',
      transform(code, id) {
        if (id.endsWith('.wasm')) {
          return {
            code: `export default ${JSON.stringify(code)};`,
            map: null,
          };
        }
      },
    },
  ],
});

// 使用
import init from './module.wasm';
await init();
```

### 2.3 Rollup

```javascript
// rollup.config.js
import wasm from '@rollup/plugin-wasm';

export default {
  input: 'src/index.js',
  output: {
    dir: 'dist',
    format: 'es',
  },
  plugins: [
    wasm({
      sync: ['**/module.wasm'],
    }),
  ],
};
```

### 2.4 ESBuild

```javascript
// esbuild 配置
require('esbuild').build({
  entryPoints: ['src/index.js'],
  bundle: true,
  outdir: 'dist',
  loader: { '.wasm': 'binary' },
  format: 'esm',
}).catch(() => process.exit(1));
```

## 三、注意事项与常见陷阱

1. **异步加载**：WASM 通常异步加载
2. **路径处理**：正确处理 WASM 文件路径
3. **缓存策略**：配置合适的缓存策略
4. **Source Maps**：配置 Source Maps 支持
5. **生产优化**：生产环境优化 WASM 加载
