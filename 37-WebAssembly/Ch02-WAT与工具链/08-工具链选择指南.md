# 工具链选择指南

## 一、概念说明

根据项目需求选择合适的 WASM 工具链。

```bash
# 工具链对比
# Emscripten: C/C++，最成熟
# wasm-pack: Rust，现代工具链
# AssemblyScript: TypeScript，简单易用
# TinyGo: Go，轻量级
# Blazor/.NET: C#，企业级
```

## 二、具体用法

### 2.1 选择决策树

```bash
# 1. 源语言偏好
# C/C++ → Emscripten
# Rust → wasm-pack
# TypeScript/JavaScript → AssemblyScript
# Go → TinyGo
# C# → Blazor/.NET

# 2. 性能要求
# 最高性能 → Rust/Emscripten
# 平衡性能和开发效率 → AssemblyScript

# 3. 生态系统
# 成熟生态系统 → Emscripten
# 现代工具链 → wasm-pack
# 快速开发 → AssemblyScript
```

### 2.2 场景推荐

```bash
# Web 游戏
# 推荐: Emscripten + C++ 或 Rust + wasm-bindgen

# 图像/视频处理
# 推荐: Emscripten + C++ (FFmpeg, OpenCV)

# 科学计算
# 推荐: Rust + wasm-pack 或 Emscripten + C++

# Web 应用插件
# 推荐: AssemblyScript (简单) 或 Rust (性能)

# 区块链智能合约
# 推荐: Rust + wasm-pack 或 AssemblyScript
```

### 2.3 性能对比

```bash
# 二进制大小
# AssemblyScript: 最小
# Rust: 小
# Emscripten: 中等
# TinyGo: 较大

# 执行速度
# Rust/Emscripten: 最快
# AssemblyScript: 快
# TinyGo: 中等

# 编译速度
# AssemblyScript: 最快
# TinyGo: 快
# Rust: 中等
# Emscripten: 较慢
```

## 三、注意事项与常见陷阱

1. **学习曲线**：不同工具有不同学习曲线
2. **生态系统**：考虑库和工具支持
3. **社区支持**：活跃社区很重要
4. **长期维护**：选择长期维护的工具
5. **试用评估**：小项目试用后再决定
