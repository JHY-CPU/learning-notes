# 工具链选择指南

## 一、概念说明

根据项目需求选择合适的 WASM 工具链。

```bash
# 工具链对比
# Emscripten: C/C++，最成熟
# wasm-pack: Rust，现代工具链
# AssemblyScript: TypeScript，简单易用
# TinyGo: Go，轻量级
# Blazor/.NET: C#，企业级
```

## 二、具体用法

### 2.1 选择决策树

```bash
# 1. 源语言偏好
# C/C++ → Emscripten
# Rust → wasm-pack
# TypeScript/JavaScript → AssemblyScript
# Go → TinyGo
# C# → Blazor/.NET

# 2. 性能要求
# 最高性能 → Rust/Emscripten
# 平衡性能和开发效率 → AssemblyScript

# 3. 生态系统
# 成熟生态系统 → Emscripten
# 现代工具链 → wasm-pack
# 快速开发 → AssemblyScript
```

### 2.2 场景推荐

```bash
# Web 游戏
# 推荐: Emscripten + C++ 或 Rust + wasm-bindgen

# 图像/视频处理
# 推荐: Emscripten + C++ (FFmpeg, OpenCV)

# 科学计算
# 推荐: Rust + wasm-pack 或 Emscripten + C++

# Web 应用插件
# 推荐: AssemblyScript (简单) 或 Rust (性能)

# 区块链智能合约
# 推荐: Rust + wasm-pack 或 AssemblyScript
```

### 2.3 性能对比

```bash
# 二进制大小
# AssemblyScript: 最小
# Rust: 小
# Emscripten: 中等
# TinyGo: 较大

# 执行速度
# Rust/Emscripten: 最快
# AssemblyScript: 快
# TinyGo: 中等

# 编译速度
# AssemblyScript: 最快
# TinyGo: 快
# Rust: 中等
# Emscripten: 较慢
```

## 三、注意事项与常见陷阱

1. **学习曲线**：不同工具有不同学习曲线
2. **生态系统**：考虑库和工具支持
3. **社区支持**：活跃社区很重要
4. **长期维护**：选择长期维护的工具
5. **试用评估**：小项目试用后再决定

## 四、详细对比矩阵

| 维度 | Emscripten | wasm-pack | AssemblyScript | TinyGo | Blazor |
|------|-----------|-----------|---------------|--------|--------|
| 源语言 | C/C++ | Rust | TypeScript | Go | C# |
| 二进制大小 | 中 | 小 | 最小 | 大 | 最大 |
| 执行速度 | 最快 | 最快 | 快 | 中等 | 中等 |
| 编译速度 | 慢 | 中等 | 最快 | 快 | 中等 |
| 生态系统 | 最丰富 | 丰富 | 中等 | 较小 | 丰富 |
| 学习曲线 | 高 | 中高 | 低 | 低 | 低 |
| 调试支持 | 好 | 好 | 中等 | 有限 | 好 |
| Web API 集成 | 直接 | wasm-bindgen | 有限 | js 包 | JSInterop |
| 社区活跃度 | 高 | 高 | 中 | 中 | 高 |
| 企业采用 | 高 | 中 | 低 | 低 | 高 |

## 五、按项目类型推荐

```markdown
### 1. 图像/视频处理
推荐: Emscripten (C++ FFmpeg/OpenCV)
原因: 现有库丰富，性能最优
备选: Rust + wasm-pack (新项目)

### 2. Web 游戏
推荐: Emscripten (C++ 引擎) 或 Rust (Bevy)
原因: 高性能渲染，现有游戏移植
备选: AssemblyScript (轻量游戏)

### 3. 科学计算/数据分析
推荐: Rust + wasm-pack
原因: 内存安全，性能优异
备选: Emscripten (现有 C++ 库)

### 4. 区块链/智能合约
推荐: Rust + wasm-pack
原因: 生态成熟，安全保证
备选: AssemblyScript (简单合约)

### 5. 前端性能模块
推荐: AssemblyScript
原因: 学习成本低，体积小
备选: Rust (性能关键模块)

### 6. 移植现有代码库
推荐: Emscripten (C/C++ 代码)
原因: 最佳兼容性
备选: TinyGo (Go 代码)

### 7. Serverless/Edge 计算
推荐: Rust + WASI
原因: 冷启动快，体积小
备选: AssemblyScript (简单逻辑)
```

## 六、渐进式采用策略

```bash
# 阶段 1: 验证（1-2 周）
# 选择一个计算密集的函数
# 用 AssemblyScript 或 Rust 重写
# 对比 JavaScript 版本的性能

# 阶段 2: 集成（2-4 周）
# 将 WASM 模块集成到现有项目
# 处理 JavaScript ↔ WASM 的数据传递
# 添加错误处理和降级方案

# 阶段 3: 扩展（持续）
# 逐步将更多计算逻辑迁移到 WASM
# 优化性能和包大小
# 建立 CI/CD 流程
```

## 七、混合工具链策略

```bash
# 一个项目中可以同时使用多个工具链
my-project/
├── core-algorithms/     # Rust (性能关键)
│   ├── Cargo.toml
│   └── src/lib.rs
├── image-processing/    # Emscripten (FFmpeg)
│   ├── Makefile
│   └── src/
├── simple-utils/        # AssemblyScript (轻量)
│   ├── asconfig.json
│   └── assembly/index.ts
└── web-app/             # JavaScript (UI)
    ├── package.json
    └── src/
```

## 八、选择决策流程图

```
开始
├─ 你有 C/C++ 代码要移植吗？
│  ├─ 是 → Emscripten
│  └─ 否 → 继续
├─ 你熟悉 Rust 吗？
│  ├─ 是 → wasm-pack
│  └─ 否 → 继续
├─ 你熟悉 TypeScript 吗？
│  ├─ 是 → AssemblyScript
│  └─ 否 → 继续
├─ 你有 Go 代码要移植吗？
│  ├─ 是 → TinyGo
│  └─ 否 → 继续
├─ 你需要最大性能吗？
│  ├─ 是 → 学习 Rust + wasm-pack
│  └─ 否 → AssemblyScript（最简单的入门选择）
└─ 推荐: 从 AssemblyScript 开始，需要时迁移到 Rust
```
