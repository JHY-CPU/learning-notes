01 - 小程序架构与生命周期

一、小程序架构原理

1.1 双线程架构

微信小程序采用双线程架构，将渲染层和逻辑层分离到不同的线程中运行，避免了JS线程阻塞渲染，保障了安全性。

┌─────────────────────────────────────────────────┐ │ 微信客户端 │ │ │ │ ┌── 渲染层（View Thread）─────┐ │ │ │ WXML模板 → 生成Virtual DOM │ │ │ │ WXSS样式 → 页面样式渲染 │ │ │ │ WebView → 渲染页面 │ │ │ └────────────┬────────────────┘ │ │ │ 通信（Native） │ │ ┌── 逻辑层（App Service Thread）──┐ │ │ │ JS脚本 → 业务逻辑处理 │ │ │ │ App/Page → 生命周期管理 │ │ │ │ 数据绑定 → setData驱动视图 │ │ │ └─────────────────────────────────┘ │ └─────────────────────────────────────────────────┘

1.2 双线程的优势与限制

维度说明
安全性JS无法直接操作DOM，避免XSS攻击
稳定性逻辑层崩溃不会导致页面白屏
性能渲染和逻辑互不阻塞
通信开销数据需通过setData序列化传递，频繁调用影响性能
无DOM/BOM无法使用document、window等浏览器API

1.3 小程序文件结构

# 小程序项目结构 ├── app.js # 小程序逻辑（App实例） ├── app.json # 全局配置（页面路由、窗口样式、tabBar等） ├── app.wxss # 全局样式 ├── project.config.json # 项目配置 ├── sitemap.json # 索引配置 │ ├── pages/ │ ├── index/ │ │ ├── index.js # 页面逻辑 │ │ ├── index.json # 页面配置 │ │ ├── index.wxml # 页面结构（类似HTML） │ │ └── index.wxss # 页面样式（类似CSS） │ └── logs/ │ ├── logs.js │ ├── logs.json │ ├── logs.wxml │ └── logs.wxss │ ├── components/ # 自定义组件 ├── utils/ # 工具函数 └── static/ # 静态资源

二、App生命周期

App是小程序的入口，管理整个应用的生命周期和全局数据。

2.1 App生命周期钩子

// app.js App({ // 小程序初始化完成时触发（全局只触发一次） onLaunch(options) { console.log('小程序启动', options); // options包含scene（场景值）、query（启动参数）等 // 检查登录状态 wx.checkSession({ success() { console.log('session未过期'); }, fail() { console.log('session已过期，需重新登录'); } }); }, // 小程序启动或从后台进入前台时触发 onShow(options) { console.log('小程序显示', options); }, // 小程序从前台进入后台时触发 onHide() { console.log('小程序隐藏'); }, // 小程序发生脚本错误或API调用失败时触发 onError(error) { console.error('小程序错误', error); }, // 页面不存在时触发（做页面兜底） onPageNotFound(res) { wx.redirectTo({ url: '/pages/index/index' }); }, // 全局数据 globalData: { userInfo: null, baseUrl: 'https://api.example.com' } })

2.2 App生命周期流程

用户打开小程序 → onLaunch（首次） → onShow 用户切换到其他App → onHide 用户切换回小程序 → onShow 用户关闭小程序 → （下次打开重新 onLaunch） 触发顺序：onLaunch → onShow（首次） onShow（后续切回）

三、Page生命周期

3.1 Page生命周期钩子

// pages/index/index.js Page({ // 页面数据（驱动视图渲染） data: { message: 'Hello World', list: [], count: 0 }, // 页面加载时触发（接收页面参数） onLoad(options) { console.log('页面加载', options); // options.id 获取跳转时传递的参数 this.setData({ count: options.id || 0 }); }, // 页面显示时触发（每次进入页面都触发） onShow() { console.log('页面显示'); }, // 页面初次渲染完成时触发（可操作DOM-like操作） onReady() { console.log('页面就绪'); }, // 页面隐藏时触发（navigateTo到其他页面、切换tab等） onHide() { console.log('页面隐藏'); }, // 页面卸载时触发（redirectTo、navigateBack等） onUnload() { console.log('页面卸载'); }, // 用户下拉刷新 onPullDownRefresh() { console.log('下拉刷新'); this.loadData().then(() => wx.stopPullDownRefresh()); }, // 页面触底（上拉加载更多） onReachBottom() { console.log('触底加载'); this.loadMore(); }, // 用户点击右上角分享 onShareAppMessage() { return { title: '推荐给你这个小程序', path: '/pages/index/index?id=123' }; }, // 页面滚动 onPageScroll(e) { console.log('滚动位置', e.scrollTop); }, // 页面事件处理函数 handleTap() { this.setData({ count: this.data.count + 1 }); }, // 自定义方法 async loadData() { const res = await wx.request({ url: '/api/data' }); this.setData({ list: res.data }); } })

3.2 Page生命周期流程

打开页面 → onLoad → onShow → onReady（页面可见且可交互） 跳转到其他页面 → onHide（当前页面隐藏） 返回当前页面 → onShow（页面重新显示） 返回上一页/关闭 → onUnload（页面销毁） 触发频率：onLoad/onReady/onUnload 只触发一次 onShow/onHide 每次页面切换都触发

四、WXML与数据绑定

4.1 WXML基础

WXML（WeiXin Markup Language）是小程序的标记语言，类似HTML，但提供了数据绑定、条件渲染、列表渲染等增强能力。

<view>{{ message }}</view> <view>{{ count > 10 ? '超过10' : '小于等于10' }}</view> <view>{{ item.name + ' - ' + item.age }}</view> <view wx:if="{{ isLoggedIn }}">已登录</view> <view wx:elif="{{ isLogging }}">登录中...</view> <view wx:else>未登录</view> <view hidden="{{ !showDetail }}">详情内容</view> <view wx:for="{{ list }}" wx:key="id" wx:for-item="item" wx:for-index="idx"> <text>{{ idx + 1 }}. {{ item.name }}</text> </view> <view wx:for="{{ categories }}" wx:key="id"> <view>{{ item.title }}</view> <view wx:for="{{ item.children }}" wx:for-item="child" wx:key="id"> {{ child.name }} </view> </view> <button bindtap="handleTap">点击我</button> <input bindinput="onInput" placeholder="请输入内容" /> <scroll-view bindscroll="onScroll">...</scroll-view> <input model:value="{{ inputValue }}" />

4.2 setData性能优化

setData最佳实践：
  • 避免频繁调用setData，合并多次更新为一次调用
  • 只设置变化的数据字段，不要传递整个大对象
  • 避免在onPageScroll中调用setData
  • 长列表使用virtual-list或分页加载
  • 不要设置过大的数据（建议每次setData的数据不超过1MB）
// 错误做法：频繁调用setData this.setData({ a: 1 }); this.setData({ b: 2 }); this.setData({ c: 3 }); // 正确做法：合并更新 this.setData({ a: 1, b: 2, c: 3 }); // 错误做法：传递不变的大数据 this.setData({ list: this.data.list }); // list没变就不需要set // 正确做法：只更新变化的部分 this.setData({ 'list[2].name': '新名字' });

五、WXSS与rpx单位

5.1 WXSS基础

WXSS（WeiXin Style Sheets）是小程序的样式语言，类似CSS，但进行了裁剪和扩展。

特性说明
尺寸单位新增rpx（响应式像素），不支持rem/vw等
样式导入支持@import导入其他WXSS文件
选择器支持.class、#id、element、::after、::before等，不支持*通配符
全局样式app.wxss中定义，所有页面生效
局部样式页面.wxss中定义，仅当前页面生效

5.2 rpx响应式单位

rpx（responsive pixel）是微信小程序特有的响应式尺寸单位，可根据屏幕宽度自适应。

rpx换算规则：
  • 规定屏幕宽度为 750rpx
  • 在iPhone 6（屏幕宽度375px）上：1rpx = 0.5px
  • 换算公式：px = rpx * (屏幕实际宽度px / 750)
  • 设计稿按750px宽度设计时，标注值直接使用rpx
/* WXSS示例 */ /* 使用rpx单位（推荐） */ .container { width: 750rpx; /* 满屏宽度 */ padding: 20rpx; font-size: 28rpx; } .title { font-size: 36rpx; margin-bottom: 16rpx; color: #333; } /* 引入外部样式 */ @import "../common/style.wxss"; /* 全局样式在 app.wxss 中定义 */ page { background-color: #f5f5f5; font-family: -apple-system, BlinkMacSystemFont, sans-serif; } /* 常用布局 */ .flex-row { display: flex; flex-direction: row; align-items: center; } .flex-col { display: flex; flex-direction: column; } .card { background: #fff; border-radius: 16rpx; padding: 24rpx; margin: 20rpx; }

6.2 自定义组件

// components/my-card/my-card.js Component({ // 组件属性（外部传入） properties: { title: { type: String, value: '默认标题' }, count: { type: Number, value: 0 } }, // 组件内部数据 data: { expanded: false }, // 组件方法 methods: { toggleExpand() { this.setData({ expanded: !this.data.expanded }); // 触发事件通知父组件 this.triggerEvent('change', { expanded: this.data.expanded }); } }, // 组件生命周期 lifetimes: { attached() { console.log('组件挂载'); }, detached() { console.log('组件销毁'); } } }) <!-- components/my-card/my-card.wxml --> <view class="card"> <view class="card-header" bindtap="toggleExpand"> <text>{{ title }}</text> <text>{{ expanded ? '收起' : '展开' }}</text> </view> <view class="card-body" wx:if="{{ expanded }}"> <slot></slot> <!-- 插槽 --> </view> </view>
组件使用要点：
  • 在页面json中声明引用的组件："usingComponents": {"my-card": "/components/my-card/my-card"}
  • 组件的wxss默认隔离（styleIsolation: 'isolated'），可通过配置修改
  • 组件间通信：properties向下传值，triggerEvent向上传事件
  • behavior可实现组件间代码复用
