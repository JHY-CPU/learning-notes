01 - Taro与uni-app

一、跨平台框架概述

1.1 为什么需要跨平台框架

微信小程序、支付宝小程序、百度小程序、字节小程序等各平台的语法和API存在差异。跨平台框架允许开发者使用统一的技术栈编写代码，编译生成各平台的原生小程序代码。

┌──────────── 源代码 ────────────┐ │ React/Vue + TypeScript │ │ 统一的API和组件 │ └──────────┬────────────────────┘ │ 编译 ┌────────┼────────┬────────────┐ ↓ ↓ ↓ ↓ 微信小程序 支付宝 H5网页 App(React Native/WebView) 支付宝小程序 百度 字节跳动

1.2 主流跨平台框架对比

技术栈React / Vue3Vue2 / Vue3（主推）

编译目标微信/支付宝/百度/字节/QQ/H5/RN微信/支付宝/百度/字节/H5/App/iOS/Android
App端Taro RN / Harmonyuni-app（基于WebView）/ uni-app x（原生渲染）

四、条件编译

4.1 什么是条件编译

条件编译是跨平台框架解决平台差异的核心机制。通过特定的注释语法，为不同平台编写差异化代码，编译时只保留目标平台的代码。

4.2 Taro条件编译

// Taro使用环境变量进行条件编译 // 编译时会被摇树优化掉其他平台的代码 // JS/TS中的条件编译 if (process.env.TARO_ENV === 'weapp') { // 仅微信小程序执行 wx.login() } else if (process.env.TARO_ENV === 'alipay') { // 仅支付宝小程序执行 my.getAuthCode() } else if (process.env.TARO_ENV === 'h5') { // 仅H5执行 console.log('Running in browser') } // 文件级别的条件编译（自动识别） // index.weapp.tsx → 仅微信小程序使用 // index.alipay.tsx → 仅支付宝小程序使用 // index.h5.tsx → 仅H5使用

4.3 uni-app条件编译

<!-- uni-app使用 #ifdef 和 #endif 进行条件编译 --> <!-- 模板中的条件编译 --> <view> <text>所有平台显示的内容</text> <!-- #ifdef MP-WEIXIN --> <button open-type="getPhoneNumber">获取手机号（仅微信）</button> <!-- #endif --> <!-- #ifdef MP-ALIPAY --> <button open-type="getPhoneNumber">获取手机号（仅支付宝）</button> <!-- #endif --> <!-- #ifdef H5 --> <div>仅H5显示的内容</div> <!-- #endif --> </view> <!-- JS中的条件编译 --> <script setup> function login() { // #ifdef MP-WEIXIN uni.login({ provider: 'weixin' }) // #endif // #ifdef MP-ALIPAY uni.login({ provider: 'alipay' }) // #endif // #ifdef H5 console.log('H5端登录') // #endif } </script> /* CSS中的条件编译 */ /* #ifdef MP-WEIXIN */ .container { padding-bottom: env(safe-area-inset-bottom); } /* #endif */ // 常用平台标识： // MP-WEIXIN 微信小程序 // MP-ALIPAY 支付宝小程序 // MP-BAIDU 百度小程序 // MP-TOUTIAO 字节跳动小程序 // MP-QQ QQ小程序 // H5 H5网页 // APP-PLUS App（5+App） // APP-PLUS-NVUE App（NVue原生渲染） // APP-ANDROID Android App // APP-IOS iOS App

五、性能对比与选型建议

5.1 性能对比

维度Tarouni-app原生开发
包体积中等（运行时较大）较小（优化较好）最小
运行性能接近原生接近原生最优
首屏加载一般一般最快
编译速度中等（Webpack）较快（Vite支持）N/A
多端一致性好（部分差异）好（条件编译完善）差（需单独开发）

5.2 选型建议

选择Taro的情况：
  • 团队熟悉React技术栈
  • 需要TypeScript严格类型支持
  • 已有React项目需要适配小程序
  • 需要更灵活的生态和NPM包管理
选择uni-app的情况：
  • 团队熟悉Vue技术栈
  • 需要同时发布到App端（iOS/Android）
  • 希望快速开发，使用丰富的插件市场
  • 需要一站式IDE（HBuilderX）支持
  • 项目预算有限，需要免费版即可覆盖更多端
注意事项：
  • 跨平台框架无法100%消除平台差异，仍需条件编译处理特殊情况
  • 复杂动画和高频交互场景，原生开发仍是最优选择
  • 跨平台框架增加了抽象层，调试定位问题可能更复杂
  • 选择框架时要考虑长期维护成本和社区活跃度
