# 常见问题与调试

## Overfull 与 Underfull hbox 警告

### 理解警告信息

LaTeX 编译时最常见的两类警告：

```
Overfull \hbox (15.34567pt too wide) in paragraph at lines 10--12
Underfull \hbox (badness 10000) in paragraph at lines 15--17
```

- **Overfull hbox**：内容超出了行宽，会导致文字溢出边界
- **Underfull hbox**：内容不足以填满一行，导致行内间距过大

### 解决 Overfull hbox

```latex
% 方法一：调整断词（hyphenation）
\hyphenation{in-ter-net tech-no-lo-gy}  % 在导言区添加自定义断词

% 方法二：使用微调命令
\tolerance=1000         % 增加容差（默认 200）
\emergencystretch=1em   % 紧急拉伸宽度
\sloppy                 % 允许更大范围的间距调整

% 方法三：手动断行
这是一个很长的词组，需要手动在适当位置\\插入换行。

% 方法四：对于过长的数学公式
\begin{align}
    \MoveEqLeft[2] f(x) = a + b + c + d + e \notag \\
    & + f + g + h + i + j
\end{align}

% 需要 mathtools 宏包：\usepackage{mathtools}

% 方法五：对于过长的 URL
\usepackage[breaklinks]{hyperref}
\usepackage{xurl}    % 允许 URL 在任意位置断行
```

### 解决 Underfull hbox

```latex
% Underfull 通常不影响输出质量，可忽略
% 如果需要解决：

% 方法一：调整文本内容，增加词数
% 方法二：使用 \mbox 防止断行
\mbox{不应断开的短语}

% 方法三：对于居中的 underfull
\begin{center}
\textbf{不会产生 underfull 警告的标题}
\end{center}
```

## 编译错误

### 常见错误类型与修复

```latex
% 错误 1: 未定义的控制序列
% ! Undefined control sequence.
% 原因：拼写错误或未加载宏包
\usepackage{amsmath}    % 确保加载了所需的宏包
% 检查命令拼写：\mathbb 需要 amssymb

% 错误 2: 缺少花括号
% ! Missing { inserted. 或 ! Missing } inserted.
% 修复：检查所有花括号是否配对
\textbf{粗体文字}       % 正确
\textbf{粗体文字}       % 错误：花括号不配对

% 错误 3: 数学模式错误
% ! Missing $ inserted.
% 原因：在文本中使用了数学专用符号
% 修复：使用正确的命令
\$100                   % 输出美元符号
\%                      % 输出百分号
\&                      % 输出 & 符号
\#                      % 输出井号
\_                      % 输出下划线

% 错误 4: 环境未关闭
% ! \begin{document} ended by \end{equation}.
% 修复：检查所有 \begin 和 \end 是否配对

% 错误 5: 引用未定义的标签
% LaTeX Warning: Label(s) may have changed. Rerun to get cross-references right.
% 修复：重新编译（通常需要编译 2-3 次）
```

### 追踪错误位置

```latex
% 使用 \tracingall 查看详细执行过程（仅调试用）
\tracingall
\tracingmacros=1
\tracingcommands=1

% 在错误位置添加调试信息
\typeout{========== 调试：到达第 X 部分 ==========}
\show\somecommand          % 显示命令定义
\meaning\somecommand       % 获取命令含义
```

## 编码问题

### UTF-8 编码设置

```latex
% 确保源文件使用 UTF-8 编码
% VS Code 右下角点击编码，选择 UTF-8

% 使用 ctex 宏包（推荐）
\documentclass{article}
\usepackage[UTF8]{ctex}

% 或使用 xeCJK + XeLaTeX 引擎
\usepackage{xeCJK}
\setCJKmainfont{SimSun}
\setCJKsansfont{SimHei}
\setCJKmonofont{FangSong}
```

### 常见编码错误

```latex
% 错误：使用 pdflatex 编译中文文档
% 报错：! Package CTeX Error: Invalid value for option 'fontset'.
% 修复：改用 XeLaTeX 引擎编译

% 错误：文件编码不一致
% 报错：! Package inputenc Error: Invalid UTF-8 byte sequence.
% 修复：确保所有 .tex 文件均为 UTF-8 编码

% 错误：BOM 标记
% 表现：文件开头出现不可见字符
% 修复：在编辑器中选择 UTF-8 without BOM
```

### 输入法与特殊字符

```latex
% LaTeX 保留字符转义
\# \$ \% \& \{ \} \_ \^{} \~{} \\

% 连字符与破折号
-          % 连字符：well-known
--         % 短破折号：pages 1--10
---        % 长破折号：引号——破折号

% 引号
`single'   % 单引号：'single'
``double'' % 双引号："double"

% 特殊符号
\ldots     % 省略号 ...
\textemdash % 破折号 —
\textendash % 短破折号 --
\copyright  % 版权符号
\texttrademark % 商标符号 TM
```

## 宏包冲突

### 常见冲突及解决方案

```latex
% 冲突 1: hyperref 与其他宏包的加载顺序
% 原则：hyperref 一般放在最后加载
\usepackage{amsmath}
\usepackage{graphicx}
\usepackage{cleveref}
\usepackage{hyperref}        % 放在倒数第二
\usepackage{algorithm2e}     % 放在 hyperref 之后

% 冲突 2: 多个引用宏包
% natbib 和 biblatex 不可同时使用
% 选择其一即可

% 冲突 3: fontenc 与 xeCJK/ctex
% 使用 XeLaTeX 时不要加载 fontenc
% \usepackage[T1]{fontenc}    % 不要与 XeLaTeX + ctex 一起使用

% 冲突 4: inputenc 与 XeLaTeX
% 使用 XeLaTeX 时不需要 inputenc
% \usepackage[utf8]{inputenc}  % XeLaTeX 默认使用 UTF-8

% 冲突 5: subcaption vs subfig vs subfigure
% 三者功能类似，选择一个即可
\usepackage{subcaption}    % 推荐
% \usepackage{subfig}      % 不要与 subcaption 同时使用
```

### 诊断宏包冲突

### 使用 latexmk 自动化

```latex
% latexmkrc 配置文件
$pdf_mode = 1;                    % 生成 PDF
$postscript_mode = 0;
$dvi_mode = 0;
$pdflatex = 'xelatex -synctex=1 -interaction=nonstopmode %O %S';
$bibtex_use = 2;                  % 自动运行 BibTeX
$out_dir = 'build';               % 输出目录
$clean_ext = 'synctex.gz synctex.gz(busy)';

% 命令行使用
% latexmk -pvc -pdf document.tex  % 持续监控文件变化并自动编译
% latexmk -c                      % 清理辅助文件
```

## 调试技巧清单

```latex
% 1. 最小工作示例（MWE）
% 当遇到问题时，创建一个尽可能小的文档来复现问题
\documentclass{article}
\usepackage{有问题的宏包}
\begin{document}
% 仅保留能触发问题的最少内容
触发问题的内容
\end{document}

% 2. 二分法定位错误
% 逐步注释掉一半内容，确定错误在哪一半

% 3. 使用 \stop 命令
\documentclass{article}
% ... 导言区 ...
\begin{document}
第一段
\stop                % 到此停止编译，不处理后面的内容
% 注释掉的部分不会被编译
第二段
\end{document}

% 4. 临时输出调试信息
\typeout{当前变量值: \the\mycounter}
\message{调试信息}
\show\mycommand      % 中断编译并显示命令定义

% 5. 检查浮动体位置
% 如果图表不出现，尝试：
% - 使用 [h!] 或 [H] 位置参数
% - 检查是否有过多浮动体堆积
\usepackage{flafter}   % 浮动体只出现在定义之后
\renewcommand{\topfraction}{0.85}
\renewcommand{\bottomfraction}{0.7}
\renewcommand{\textfraction}{0.1}

% 6. 常见调试宏包
\usepackage{lipsum}       % 生成占位文本
\usepackage{blindtext}    % 生成多语言占位文本
\usepackage{layout}       % 显示页面布局
\usepackage{showframe}    % 显示页面边框线
\usepackage{lineno}       % 显示行号
```

## 编译引擎选择指南

```
需要中文? --> 是 --> XeLaTeX（推荐）或 LuaLaTeX
             |
             否
             |
需要 OpenType 字体? --> 是 --> XeLaTeX 或 LuaLaTeX
             |
             否
             |
需要复杂宏编程? --> 是 --> LuaLaTeX
             |
             否
             |
pdflatex 即可满足需求
```

综合推荐：学术中文论文使用 **XeLaTeX + ctex + BibTeX/biber** 组合。
