Shor与Grover量子算法

量子计算ShorGrover

Shor算法实现指数级加速的大数分解，Grover算法实现平方级加速的搜索。

Shor算法 (大数分解)

Shor算法（1994年）：分解大整数N = p × q 经典算法：亚指数时间 O(exp(n^(1/3))) 量子算法：多项式时间 O(n³) 核心思想：将分解问题转化为求周期问题 步骤： 1. 随机选择 a < N，计算 gcd(a, N) 若gcd ≠ 1，已找到因子，结束 2. 求 a^x mod N 的周期 r （量子部分：量子傅里叶变换QFT） 3. 若 r 是偶数且 a^(r/2) ≡ -1 (mod N) 则 gcd(a^(r/2) ± 1, N) 给出N的因子 量子傅里叶变换 (QFT)： |j⟩ → (1/√N) Σ exp(2πijk/N)|k⟩ - 量子版本的离散傅里叶变换 - 用O(n²)个门实现n比特QFT - 经典FFT需要O(n·2ⁿ) 对RSA的影响： - RSA-2048：经典计算机需要数十亿年 - Shor算法：需要约4000个逻辑量子比特 - 当前量子计算机：约1000个物理量子比特 - 短期内不构成威胁，但需准备后量子密码

Grover算法 (量子搜索)

Grover算法（1996年）：在N个元素中搜索目标 经典算法：O(N) 量子算法：O(√N) — 平方级加速 算法流程： 1. 初始化：所有状态等概率叠加 |ψ⟩ = (1/√N) Σ |x⟩ = H⊗n |0...0⟩ 2. 迭代 √N 次（Grover迭代）： a. Oracle标记：翻转目标态的相位 Uf |x⟩ = -|x⟩ if f(x)=1 Uf |x⟩ = |x⟩ if f(x)=0 b. 扩散算子(绕平均值翻转)： Us = 2|ψ⟩⟨ψ| - I 将目标态的概率幅增大 3. 测量：大概率得到目标态 几何理解： - 量子态在二维空间中旋转 - 每次Grover迭代旋转角度 θ ≈ 2/√N - √N次迭代后接近目标态 应用： - 数据库搜索：√N加速 - 密码破解：对称加密密钥长度需翻倍 AES-128 → 需要AES-256来抵抗Grover攻击 - SAT问题：NP问题的量子加速 - 量子机器学习：加速优化问题
Shor算法对RSA/ECC构成威胁，Grover算法要求对称密钥翻倍。后量子密码学正在标准化中。

量子计算优势与现状

量子优势来源： 1. 量子并行性：一次操作处理所有叠加态 2. 量子干涉：增强正确答案，消减错误答案 3. 量子纠缠：提供非经典关联 量子计算复杂性类： ┌─────────────────────────────────────────────┐ │ BQP：量子多项式时间可解的问题 │ │ P ⊆ BQP ⊆ PSPACE │ │ NP ⊆ BQP？（未知，但普遍认为不包含） │ │ BQP包含某些不在P中的问题（如分解） │ └─────────────────────────────────────────────┘ 当前进展： - IBM：1000+量子比特，量子体积持续增长 - Google：2019年量子优越性（Sycamore，53量子比特） - 中国：九章光量子计算机（特定任务量子优越性） - IonQ/Quantinuum：离子阱量子计算机 挑战： - 退相干：量子态与环境交互导致信息丢失 - 门误差：当前门保真度~99.9%，需要99.99%+ - 纠错开销：逻辑量子比特需要数千物理量子比特 - 规模化：百万量子比特系统尚未实现

量子傅里叶变换 QFT

QFT（Quantum Fourier Transform）： ┌─────────────────────────────────────────────────────────┐ │ 经典 DFT：N 个数据点需要 O(N²) 或 O(N log N) 运算 │ │ 量子 QFT：n 量子比特需要 O(n²) 个门！ │ │ （N = 2^n，经典 FFT 需要 O(n·2^n)） │ │ │ │ 定义： │ │ QFT|j⟩ = (1/√N) Σ exp(2πijk/N)|k⟩ │ │ 矩阵元素：QFT_jk = ω^(jk) / √N，ω = e^(2πi/N) │ │ │ │ QFT 电路（n 量子比特）： │ │ |j₁⟩ ──H──R₂──R₃──...──Rₙ──────────→ 输出 │ │ │ │ │ │ │ |j₂⟩ ──●────┼────H────┼──R₂──...──→ │ │ │ │ │ │ │ |j₃⟩ ──────●──────────●────●──H──→ │ │ │ │ Rₖ = [1 0 ] = 相位旋转门 │ │ [0 e^(2πi/2^k)] │ │ │ │ 门数统计： │ │ - n 个 H 门 │ │ - n(n-1)/2 个受控相位旋转门 │ │ - 总计 O(n²) 个门 │ │ - 最后需要比特反转（SWAP 操作） │ └─────────────────────────────────────────────────────────┘ QFT 在 Shor 算法中的作用： ┌─────────────────────────────────────────────────────────┐ │ 目标：找到函数 f(x) = a^x mod N 的周期 r │ │ │ │ 1. 准备叠加态： │ │ (1/√Q) Σ |x⟩|0⟩（Q ≈ N²，第一寄存器） │ │ │ │ 2. 计算函数： │ │ (1/√Q) Σ |x⟩|a^x mod N⟩ │ │ │ │ 3. 测量第二寄存器（不测量第一寄存器）： │ │ 得到某个 a^s mod N │ │ 第一寄存器坍缩到： │ │ (1/√(Q/r)) Σ |s + kr⟩（k=0,1,...） │ │ 即等间距的叠加态，间距为 r │ │ │ │ 4. 对第一寄存器施加 QFT： │ │ QFT 将等间距叠加态转换为频率域 │ │ 测量得到 j × Q/r（j 随机） │ │ 连分数展开 j/Q → 找到 r │ │ │ │ 这就是 QFT 的核心作用： │ │ 将时间域的周期性转换为频率域的离散峰 │ └─────────────────────────────────────────────────────────┘

Grover 算法扩展与变体

Grover 的最优迭代次数： ┌─────────────────────────────────────────────────────────┐ │ N 个元素中有 M 个目标 │ │ 最优迭代次数：⌊π/4 × √(N/M)⌋ │ │ │ │ 迭代太少：目标态概率不够高 │ │ 迭代太多：目标态概率反而下降（在二维空间中过度旋转） │ │ │ │ 成功概率： │ │ k 次迭代后 = sin²((2k+1)θ)，其中 sin(θ) = √(M/N) │ │ 最优 k 时 ≈ 1 │ └─────────────────────────────────────────────────────────┘ 量子幅度估计 (Quantum Amplitude Estimation)： ┌─────────────────────────────────────────────────────────┐ │ 结合 QPE 和 Grover 算法 │ │ 目标：估计目标态的概率 M/N │ │ 精度：O(1/M)（只需 O(√N) 次 Grover 迭代） │ │ │ │ 经典方法需要 O(N) 次采样才能估计概率 │ │ 量子方法只需 O(√N) 次 │ │ → 蒙特卡洛模拟的量子加速 │ │ │ │ 应用： │ │ - 金融衍生品定价（量子加速蒙特卡洛） │ │ - 积分计算 │ │ - 机器学习中的概率估计 │ └─────────────────────────────────────────────────────────┘ Grover 算法的变体： ┌─────────────────────────────────────────────────────────┐ │ 固定点量子搜索 (Fixed-Point Search)： │ │ - 传统 Grover 迭代次数敏感 │ │ - 固定版本保证单调收敛 │ │ - 代价：需要更长的电路 │ │ │ │ 量子游走搜索 (Quantum Walk Search)： │ │ - Grover 在无结构搜索上的推广 │ │ - 对图结构数据的搜索 │ │ - 可达到优于 Grover 的加速（某些图结构） │ │ │ │ 量子最小查找 (Quantum Minimum Finding)： │ │ - 在无序列表中找最小值 │ │ - O(√N) vs 经典 O(N) │ │ - 利用 Grover 搜索的变体 │ │ │ │ 多目标 Grover 搜索： │ │ - 同时搜索多个目标 │ │ - 迭代次数 ∝ √(N/M) │ │ - M 越大（目标越多），越快收敛 │ └─────────────────────────────────────────────────────────┘
QFT 是 Shor 算法的核心子程序，将周期性从时间域转换到频率域。Grover 的平方加速是量子搜索的最优加速（已被证明）。量子幅度估计将 Grover 与 QPE 结合，可用于加速蒙特卡洛模拟。
