# 量子逻辑门


## 量子逻辑门


量子计算量子门电路


量子门是量子电路的基本操作单元，通过酉矩阵变换量子态。


## 单量子比特门


```
量子门 = 酉矩阵 (Unitary Matrix)，满足 U†U = I

Pauli 门（泡利矩阵）：
X (NOT/翻转):     Y:                 Z (相位翻转):
[0 1]             [0 -i]             [1  0]
[1 0]             [i  0]             [0 -1]
X|0⟩=|1⟩         Y|0⟩=i|1⟩         Z|0⟩=|0⟩
X|1⟩=|0⟩         Y|1⟩=-i|0⟩        Z|1⟩=-|1⟩

Hadamard门 H：
(1/√2)[1  1]    制造叠加态
       [1 -1]
H|0⟩ = |+⟩ = (|0⟩+|1⟩)/√2
H|1⟩ = |−⟩ = (|0⟩-|1⟩)/√2

相位门：
S门:      T门:        Rz(θ):
[1 0]     [1    0]    [e^(-iθ/2)    0   ]
[0 i]     [0 e^(iπ/4)][0         e^(iθ/2)]

通用单比特门：任意2x2酉矩阵可分解为
U = e^(iα) × Rz(β) × Ry(γ) × Rz(δ)
```


## 多量子比特门


```
CNOT (受控NOT)：
控制比特 | 目标比特 | 结果
   0     |    0     |  0,0
   0     |    1     |  0,1
   1     |    0     |  1,1  ← 翻转
   1     |    1     |  1,0  ← 翻转

矩阵形式：
[1 0 0 0]           |00⟩→|00⟩
[0 1 0 0]           |01⟩→|01⟩
[0 0 0 1]           |10⟩→|11⟩
[0 0 1 0]           |11⟩→|10⟩

CNOT是创造纠缠的关键门！
|+⟩|0⟩ = (|0⟩+|1⟩)/√2 ⊗ |0⟩ → CNOT → (|00⟩+|11⟩)/√2 = Bell态

Toffoli门 (CCNOT)：
- 双控制NOT门
- 两个控制比特都为1时翻转目标
- 通用可逆门：任何经典可逆电路可用Toffoli实现

SWAP门：
|a,b⟩ → |b,a⟩
交换两个量子比特的状态
可由3个CNOT实现

通用量子门集：
{H, T, CNOT} 构成通用量子门集
→ 任何量子计算都可以用这些门近似实现
```


## 量子电路与不可克隆定理


```
量子电路表示：
时间从左到右
|0⟩ ──[H]──●──[测量]──
            │
|0⟩ ───────⊕──[测量]──
（制备Bell态的电路）

电路深度和门数：
- 深度：关键路径上的门数
- 门数：总操作数
- 优化目标：最小化深度和门数

不可克隆定理 (No-Cloning Theorem)：
量子态不能被完美复制

证明：假设存在复制门U
U|ψ⟩|0⟩ = |ψ⟩|ψ⟩
U|φ⟩|0⟩ = |φ⟩|φ⟩

由于U是酉的（保持内积）：
⟨ψ|φ⟩ = ⟨ψ|φ⟩²

只有当⟨ψ|φ⟩=0或1时才成立
→ 不能同时复制两个非正交的量子态

意义：
- 量子信息不能像经典信息那样备份
- 量子纠错需要巧妙的编码方案
- 量子密码学中保证了窃听可检测（BB84协议）
```


> **Note:** CNOT和Hadamard门是量子计算的基石。理解电路深度对分析量子算法的实际可行性至关重要。


## 量子门的通用性与分解


```
Solovay-Kitaev 定理：
┌─────────────────────────────────────────────────────────┐
│  任何 n 量子比特酉矩阵都可以用通用门集以任意精度近似       │
│  给定精度 ε，门数为 O(log^c(1/ε))                        │
│  c ≈ 3.97（原始），可通过优化降到 ~1                     │
│                                                         │
│  通用门集：                                              │
│  - {H, T, CNOT}：最常见的通用门集                        │
│  - {H, Toffoli}：经典可逆 + 叠加                         │
│  - {Rx(θ), Ry(θ), CNOT}：连续参数门                     │
│  - {√X, CZ}：Google Sycamore 原生门集                   │
│                                                         │
│  T 门的重要性：                                           │
│  - {H, S, CNOT} 不是通用的（只能实现 Clifford 群）       │
│  - 添加 T 门后变为通用                                   │
│  - T 门的数量是容错量子计算的代价指标                     │
│  - T 门蒸馏是量子纠错中的主要开销                        │
└─────────────────────────────────────────────────────────┘

任意单比特门的分解：
┌─────────────────────────────────────────────────────────┐
│  Z-Y-Z 分解（Euler 分解）：                               │
│  U = e^(iα) × Rz(β) × Ry(γ) × Rz(δ)                    │
│  任何 2×2 酉矩阵都可以用 3 个旋转门实现                   │
│                                                         │
│  分解算法：                                              │
│  1. 计算 U 的特征值确定 α                                │
│  2. 从 U × e^(-iα) 中提取 δ, γ, β                       │
│  3. 用 Ry 和 Rz 门的组合实现                             │
│                                                         │
│  双比特门分解更复杂：                                     │
│  任意 4×4 酉矩阵最多需要 3 个 CNOT + 单比特门            │
│  （Shende, Markov, Bullock, 2004）                       │
└─────────────────────────────────────────────────────────┘

量子电路编译：
┌─────────────────────────────────────────────────────────┐
│  从算法到硬件的转换：                                      │
│  1. 高级描述（Qiskit, Cirq, etc.）                       │
│  2. 门分解（将高级门分解为原生门）                        │
│  3. 映射（逻辑量子比特 → 物理量子比特）                   │
│  4. 路由（插入 SWAP 门适应硬件拓扑）                      │
│  5. 优化（减少门数和电路深度）                            │
│  6. 校准（针对硬件噪声优化）                              │
│                                                         │
│  硬件拓扑示例（Google Sycamore）：                        │
│  - 二维格点，最近邻连接                                   │
│  - 不是所有量子比特对都可以直接做双比特门                 │
│  - 需要 SWAP 操作移动量子比特                             │
│                                                         │
│  电路优化策略：                                           │
│  - 消除恒等门                                            │
│  - 合并相邻旋转门                                        │
│  - 利用电路对称性                                        │
│  - 选择最优的量子比特映射                                │
└─────────────────────────────────────────────────────────┘
```


> **Note:** CNOT 和 Hadamard 门是量子计算的基石。T 门在容错量子计算中代价最高（需要 T 门蒸馏），T 门数量是评估量子算法实际成本的重要指标。电路编译是从算法到硬件的关键步骤。
