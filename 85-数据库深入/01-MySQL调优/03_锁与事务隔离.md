# 锁与事务隔离 - MySQL调优

*深入理解 InnoDB 锁类型（记录锁/间隙锁/临键锁/插入意向锁）、死锁检测、事务隔离级别及 MVCC 读视图机制*

InnoDB 锁类型体系

| 锁类型 | 英文名 | 锁粒度 | 主要作用 |
| --- | --- | --- | --- |
| **共享锁 / 排他锁** | S Lock / X Lock | 行级 | 基本读写锁 |
| **意向锁** | IS / IX Lock | 表级 | 标识表中某行被加锁 |
| **记录锁** | Record Lock | 行级 | 锁住索引记录 |
| **间隙锁** | Gap Lock | 间隙 | 防止幻读（插入） |
| **临键锁** | Next-Key Lock | 记录+间隙 | 记录锁 + 间隙锁 |
| **插入意向锁** | Insert Intention Lock | 间隙 | 插入操作的特殊间隙锁 |

经典死锁场景

```
-- 事务 A                          -- 事务 B
BEGIN;                            BEGIN;
UPDATE users SET name='A'         UPDATE users SET name='B'
  WHERE id = 1;                     WHERE id = 2;
-- 持有 id=1 的 X 锁              -- 持有 id=2 的 X 锁

UPDATE users SET name='A'         UPDATE users SET name='B'
  WHERE id = 2;                     WHERE id = 1;
-- 等待 id=2 的 X 锁 ←──────────  -- 等待 id=1 的 X 锁 ←── 死锁！

-- InnoDB 检测到死锁后：
-- 回滚其中一个事务（通常选择代价较小的），另一个继续执行
-- 错误信息: ERROR 1213 (40001): Deadlock found when trying to get lock
```

SQL 标准隔离级别 vs MySQL InnoDB 实现

| 隔离级别 | 脏读 | 不可重复读 | 幻读 | MySQL 默认 |
| --- | --- | --- | --- | --- |
| READ UNCOMMITTED | 可能 | 可能 | 可能 | 否 |
| READ COMMITTED (RC) | 避免 | 可能 | 可能 | 否（Oracle 默认） |
| REPEATABLE READ (RR) | 避免 | 避免 | 基本避免 | 是（MySQL 默认） |
| SERIALIZABLE | 避免 | 避免 | 避免 | 否 |

RC vs RR 关键差异

| 特性 | READ COMMITTED | REPEATABLE READ |
| --- | --- | --- |
| 一致性读视图 | 每条 SQL 重新创建 | 事务首次读取时创建，整个事务复用 |
| 间隙锁 | 不使用间隙锁 | 使用间隙锁防止幻读 |
| 锁范围 | 仅 Record Lock | Next-Key Lock（记录+间隙） |
| 幻读 | 可能发生 | InnoDB 通过间隙锁 + MVCC 基本避免 |
| 并发性能 | 更高（锁范围小） | 略低（间隙锁可能冲突） |
| binlog 格式要求 | ROW 格式（或 MIXED） | STATEMENT/ROW 均可 |

Read View 数据结构

```
struct ReadView {
    trx_id_t m_ids[];          -- 创建 Read View 时活跃（未提交）的事务 ID 列表
    trx_id_t m_low_limit_id;   -- m_ids 中的最大值 + 1（未来事务的下限）
    trx_id_t m_up_limit_id;    -- m_ids 中的最小值（活跃事务的下限）
    trx_id_t m_creator_trx_id; -- 创建此 Read View 的事务 ID
};
```

MVCC 可见性判断逻辑

1. 如果
   `trx_id == m_creator_trx_id`
   ：自己修改的，
   可见
2. 如果
   `trx_id < m_up_limit_id`
   ：该版本在 Read View 创建前已提交，
   可见
3. 如果
   `trx_id >= m_low_limit_id`
   ：该版本在 Read View 创建后才开始，
   不可见
4. 如果
   `m_up_limit_id <= trx_id < m_low_limit_id`
   ：

   - 如果
      `trx_id`
      在
      `m_ids[]`
      中：事务未提交，
      不可见
   - 如果
      `trx_id`
      不在
      `m_ids[]`
      中：事务已提交，
      可见


如果当前版本不可见，则沿 roll_pointer 向上查找 undo 链表中的更老版本，直到找到可见版本或到达链表头部。
