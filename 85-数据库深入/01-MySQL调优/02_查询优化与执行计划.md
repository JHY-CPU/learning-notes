# 查询优化与执行计划 - MySQL调优

*深入理解 EXPLAIN 输出、连接算法（NLJ/BNL/Hash Join）、优化器提示、慢查询日志及查询重写技术*

EXPLAIN 输出字段说明

| 字段 | 含义 | 重点关注 |
| --- | --- | --- |
| **id** | 查询序号，id 相同为同一组，id 越大优先级越高 | 子查询和 UNION 时区分层次 |
| **select_type** | 查询类型：SIMPLE/PRIMARY/SUBQUERY/DERIVED/UNION | 复杂查询的组成部分 |
| **table** | 当前访问的表名或别名 | <derived N> 表示派生表 |
| **partitions** | 匹配的分区 | 分区表的分区裁剪 |
| **type** | 访问类型（最重要字段之一） | ALL < index < range < ref < eq_ref < const |
| **possible_keys** | 可能使用的索引 | 注意 NULL 表示无可用索引 |
| **key** | 实际使用的索引 | NULL 表示未使用索引 |
| **key_len** | 使用的索引长度（字节） | 判断联合索引使用了几列 |
| **ref** | 与索引比较的列或常量 | const 表示常量比较 |
| **rows** | 预估扫描行数 | 越小越好，不精确但有参考价值 |
| **filtered** | 存储引擎返回后经过条件过滤的比例 | rows × filtered ≈ 最终返回行数 |
| **Extra** | 额外信息（非常重要） | Using index / Using filesort / Using temporary 等 |

type 访问类型排序

| type | 含义 | 性能 |
| --- | --- | --- |
| ALL | 全表扫描，不使用任何索引 | 最差 |
| index | 全索引扫描（遍历整个索引树） | 较差 |
| range | 索引范围扫描（>, <, BETWEEN, IN） | 一般 |
| ref | 非唯一索引等值查询（可能返回多行） | 较好 |
| eq_ref | 唯一索引等值查询（JOIN 中主键/唯一键） | 很好 |
| const | 主键或唯一索引等值查询，最多一行 | 最优 |
| system | 表只有一行数据（系统表） | 最优 |

IN 改 JOIN / EXISTS

```
-- 原始：IN 子查询（可能被物化为临时表）
SELECT * FROM orders WHERE user_id IN (SELECT id FROM users WHERE age > 25);

-- 重写为 JOIN（利用索引）
SELECT DISTINCT o.* FROM orders o
JOIN users u ON o.user_id = u.id
WHERE u.age > 25;

-- 重写为 EXISTS（当子查询结果集大时更优）
SELECT * FROM orders o
WHERE EXISTS (SELECT 1 FROM users u WHERE u.id = o.user_id AND u.age > 25);
```

OR 改 UNION

```
-- 原始：OR 条件导致无法使用索引
SELECT * FROM users WHERE name = 'Bob' OR age = 30;

-- 重写为 UNION（各自走各自索引）
SELECT * FROM users WHERE name = 'Bob'
UNION
SELECT * FROM users WHERE age = 30
AND name != 'Bob';  -- 避免重复
```

函数/计算消除

```
-- 原始：索引列上使用函数（索引失效）
SELECT * FROM orders WHERE YEAR(create_time) = 2024;

-- 重写：改用范围查询（走索引）
SELECT * FROM orders
WHERE create_time >= '2024-01-01' AND create_time < '2025-01-01';

-- 原始：索引列参与计算
SELECT * FROM users WHERE age + 1 = 26;

-- 重写：计算移到右边
SELECT * FROM users WHERE age = 25;
```

分页优化

```
-- 原始深分页：LIMIT 1000000, 10 需要扫描前 1000010 行
SELECT * FROM orders ORDER BY id LIMIT 1000000, 10;

-- 重写：先查 id 再关联（延迟关联）
SELECT o.* FROM orders o
INNER JOIN (SELECT id FROM orders ORDER BY id LIMIT 1000000, 10) AS t
ON o.id = t.id;

-- 重写：使用书签（记录上次最大 id）
SELECT * FROM orders WHERE id > 1000000 ORDER BY id LIMIT 10;
```
