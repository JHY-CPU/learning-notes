# 分库分表与高可用 - MySQL调优

*深入理解水平/垂直分片、分片键选择、跨分片查询、MySQL Group Replication 及 ProxySQL 代理方案*

分片方案对比

| 维度 | 垂直分库（按业务） | 垂直分表（按列） | 水平分库 | 水平分表 |
| --- | --- | --- | --- | --- |
| 切分方式 | 按业务模块拆到不同数据库 | 将宽表的列拆到多张表 | 同一张表的数据分到不同数据库 | 同一张表的数据分到同库不同表 |
| 解决的问题 | 业务耦合、连接数 | 大字段影响查询效率 | 数据量和并发 | 单表数据量 |
| 典型场景 | 用户库、订单库、商品库 | 热列/冷列分离 | 用户表按 user_id 分 4 库 | 订单表按月分表 |
| 复杂度 | 低 | 低 | 高 | 中 |

常见分片策略

| 策略 | 算法 | 优点 | 缺点 |
| --- | --- | --- | --- |
| 哈希取模 | hash(shard_key) % N | 数据均匀分布 | 扩容需迁移数据 |
| 一致性哈希 | 哈希环映射 | 扩容影响小 | 实现复杂 |
| 范围分片 | 按值范围分配 | 范围查询高效 | 可能数据倾斜 |
| 时间分片 | 按年/月/日分表 | 过期数据易清理 | 历史数据查询慢 |
| 地理位置 | 按区域编码分片 | 本地化查询快 | 热点区域倾斜 |

跨分片查询解决方案

| 方案 | 原理 | 适用场景 |
| --- | --- | --- |
| 汇总库 / ES | 将数据同步到 Elasticsearch 或汇总库，查询走汇总库 | 多维度复杂查询、搜索 |
| 基因法 | 将分片键信息嵌入其他 ID，使关联查询也能路由到同一分片 | 订单和订单明细关联 |
| 广播表 | 小表在每个分片都保留完整副本 | 配置表、字典表 |
| ER 分片 | 关联数据按相同分片键分到同一分片 | 父子表关联查询 |
| Merge 索引表 | 建立分片键与其他查询维度的映射表 | 非分片键条件查询 |

ProxySQL 主要功能模块

| 功能 | 说明 |
| --- | --- |
| 连接池 | 管理后端 MySQL 连接，减少连接创建开销，支持连接复用 |
| 读写分离 | 自动将 SELECT 路由到从库，INSERT/UPDATE/DELETE 路由到主库 |
| 查询路由 | 基于正则匹配将查询路由到不同后端组（分库场景） |
| 查询缓存 | 对特定查询结果进行缓存，减少后端压力 |
| 健康检查 | 定期检测后端节点可用性，自动摘除故障节点 |
| 监控统计 | 详细的查询统计、延迟监控、连接状态监控 |
