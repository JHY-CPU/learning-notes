# 时序数据库与图数据库 - NoSQL深入

*深入理解 InfluxDB/TimescaleDB（时序数据）、Neo4j/JanusGraph（图数据）的核心概念、查询语言及典型应用场景*

InfluxDB 核心概念

| 概念 | 说明 | 类比 SQL |
| --- | --- | --- |
| Database / Bucket | 数据库/存储桶 | Database |
| Measurement | 测量名称（类似表名） | Table |
| Tag | 索引的元数据（字符串，自动索引） | Indexed Column |
| Field | 测量值（数值/字符串/布尔，不索引） | Column |
| Timestamp | 数据点的时间戳（纳秒精度） | Primary Key |
| Retention Policy | 数据保留策略（自动删除旧数据） | TTL |
| Continuous Query | 持续查询（自动降采样） | Scheduled Job |

图数据库 vs 关系型数据库查询对比

| 查询场景 | 关系型（SQL） | 图数据库 |
| --- | --- | --- |
| 朋友的朋友 | 多表 JOIN，性能差 | 沿关系遍历，高效 |
| 最短路径 | 需递归/多次查询 | 原生图算法，O(V+E) |
| 社区发现 | 几乎无法高效实现 | Louvain 等内置算法 |
| 关系推理 | 复杂且慢 | 模式匹配高效 |
| 聚合统计 | SQL 聚合高效 | 不如关系型 |

场景与数据库选型对照

| 业务场景 | 推荐数据库 | 原因 |
| --- | --- | --- |
| IoT 传感器数据、监控指标 | InfluxDB / TimescaleDB | 时序数据专用优化，高压缩率 |
| 社交网络、推荐系统 | Neo4j | 关系遍历高效，图算法丰富 |
| 知识图谱、风控反欺诈 | Neo4j / JanusGraph | 复杂关联推理，模式匹配 |
| 大规模分布式图（TB 级） | JanusGraph | 分布式存储后端，水平扩展 |
| 时序 + 关系混合查询 | TimescaleDB | 完整 SQL 支持，可 JOIN |
| 高频时序写入（百万点/秒） | InfluxDB | 专为时序写入优化 |
| 搜索引擎、全文检索 | Elasticsearch | 倒排索引，分布式搜索 |
| 缓存、会话存储 | Redis | 内存存储，亚毫秒延迟 |
