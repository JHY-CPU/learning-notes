# 并行计算概述

## 1. Flynn 分类法

Michael Flynn 于1966年提出了一种经典分类法，根据**指令流**和**数据流**的数量将计算机体系结构分为四类：

| 类型 | 指令流 | 数据流 | 典型代表 |
|------|--------|--------|----------|
| SISD | 单 | 单 | 传统冯·诺依曼机 |
| SIMD | 单 | 多 | GPU、向量机、AVX指令集 |
| MISD | 多 | 单 | 容错计算机（理论模型） |
| MIMD | 多 | 多 | 多核CPU、集群 |

**SISD（Single Instruction, Single Data）** 是最简单的模型，处理器串行执行指令，一次只处理一个数据元素。所有冯·诺依曼架构的经典计算机本质上都是SISD。

**SIMD（Single Instruction, Multiple Data）** 中一条指令同时作用于多个数据。现代GPU的大量核心可以同步执行相同操作，AVX-512指令集可在一条指令中处理16个浮点数。

```c
// SIMD 思想示意：一条 addps 指令同时完成4个浮点加法
// 在 C 中使用 SIMD 内联指令
#include <immintrin.h>
void simd_add(float *a, float *b, float *c, int n) {
    for (int i = 0; i < n; i += 8) {
        __m256 va = _mm256_load_ps(&a[i]);
        __m256 vb = _mm256_load_ps(&b[i]);
        __m256 vc = _mm256_add_ps(va, vb);
        _mm256_store_ps(&c[i], vc);
    }
}
```

**MIMD（Multiple Instruction, Multiple Data）** 是最通用的并行模型，多个处理器各自执行不同指令处理不同数据。现代多核CPU、MPI集群都属于MIMD。

## 2. 共享内存与分布式内存

### 共享内存模型

所有处理器共享同一物理地址空间，通过读写共享变量实现通信。

```
┌──────────┬──────────┬──────────┬──────────┐
│  Core 0  │  Core 1  │  Core 2  │  Core 3  │
└────┬─────┴────┬─────┴────┬─────┴────┬─────┘
     │          │          │          │
     └──────────┴────┬─────┴──────────┘
                     │
              ┌──────┴──────┐
              │ 共享内存     │
              │ (同一地址空间)│
              └─────────────┘
```

- **优点**: 编程直观，共享变量直接访问，通信延迟低
- **缺点**: 可扩展性差（通常限于单节点），需要处理缓存一致性
- **代表**: OpenMP、Pthreads、Java并发

### 分布式内存模型

每个处理器有独立的本地内存，通过消息传递交换数据。

```
┌──────────┐  ┌──────────┐  ┌──────────┐
│  Core 0  │  │  Core 1  │  │  Core 2  │
│  Mem 0   │  │  Mem 1   │  │  Mem 2   │
└────┬─────┘  └────┬─────┘  └────┬─────┘
     │   网络互连   │             │
     └─────────────┴─────────────┘
```

- **优点**: 可扩展性极强，可扩展到数千节点
- **缺点**: 编程复杂，需要显式管理数据分布和消息传递
- **代表**: MPI、PVM

### 混合模型

现代HPC系统通常采用混合模型：节点内使用共享内存（OpenMP），节点间使用消息传递（MPI）。

```c
// MPI + OpenMP 混合编程示意
#include <mpi.h>
#include <omp.h>

int main(int argc, char *argv[]) {
    MPI_Init(&argc, &argv);
    int rank, size;
    MPI_Comm_rank(MPI_COMM_WORLD, &rank);
    MPI_Comm_size(MPI_COMM_WORLD, &size);

    // 每个MPI进程内部再用OpenMP线程并行
    #pragma omp parallel
    {
        int tid = omp_get_thread_num();
        int nthreads = omp_get_num_threads();
        printf("MPI rank %d, OpenMP thread %d/%d\n", rank, tid, nthreads);
    }

    MPI_Finalize();
    return 0;
}
```

## 3. 并行加速比与 Amdahl 定律

### 加速比定义

并行系统的**加速比（Speedup）** 定义为：

$$S_p = \frac{T_1}{T_p}$$

其中 $T_1$ 是串行执行时间，$T_p$ 是使用 $p$ 个处理器的并行执行时间。

**效率**定义为：

$$E_p = \frac{S_p}{p} = \frac{T_1}{p \cdot T_p}$$

理想情况下 $S_p = p$（线性加速），$E_p = 1$。

### Amdahl 定律

Amdahl 定律由 Gene Amdahl 于1967年提出，指出程序中**串行部分**对加速比的限制。

设程序中可并行化比例为 $f$，则串行比例为 $(1-f)$，最大加速比为：

$$S_{max} = \frac{1}{(1-f) + \frac{f}{p}}$$

当 $p \to \infty$ 时，$S_{max} \to \frac{1}{1-f}$。

```python
import matplotlib.pyplot as plt
import numpy as np

# Amdahl 定律演示
processors = np.arange(1, 1025)
f_values = [0.5, 0.75, 0.9, 0.95, 0.99]

plt.figure(figsize=(10, 6))
for f in f_values:
    speedup = 1.0 / ((1 - f) + f / processors)
    plt.plot(processors, speedup, label=f'f={f}')

plt.plot(processors, processors, 'k--', label='理想加速比')
plt.xlabel('处理器数量 P')
plt.ylabel('加速比 S')
plt.title("Amdahl 定律：不同并行比例下的最大加速比")
plt.legend()
plt.grid(True)
plt.savefig("amdahl_law.png")
plt.show()
```

关键结论：
- 即使 $f = 0.99$（99%可并行），1000个处理器也最多获得约71倍加速
- 串行瓶颈是硬性限制，不可突破

### Gustafson-Barsis 定律

Gustafson 定律从另一个角度看问题：固定**执行时间**而非问题规模。

$$S_p = p - (1-f)(p-1)$$

当处理器增多时，我们倾向于解决**更大的问题**，因此实际中Gustafson定律往往更贴近真实场景。

```python
# Gustafson 定律对比
for f in f_values:
    speedup_g = processors - (1 - f) * (processors - 1)
    plt.plot(processors, speedup_g, label=f'f={f}')
plt.xlabel('处理器数量 P')
plt.ylabel('加速比 S (Gustafson)')
plt.title("Gustafson 定律：固定时间下的加速比")
plt.legend()
plt.grid(True)
```

## 4. 并行计算的挑战

### 负载均衡

所有处理器应大致均等地分配工作量。不均衡会导致部分处理器闲置，浪费计算资源。

### 通信开销

处理器间数据交换的延迟和带宽限制。通信与计算之比应尽可能小。

### 同步开销

barrier、锁等同步操作会让部分处理器等待。

### 可扩展性

- **强扩展（Strong Scaling）**: 固定总问题规模，增加处理器数量，观察加速比
- **弱扩展（Weak Scaling）**: 每个处理器的问题规模固定，总规模随处理器增加，观察执行时间变化
