# 评估指标详解 - LLM评估体系

*深入理解 NLP 和 LLM 评估的核心指标：BLEU、ROUGE、METEOR、BERTScore、Perplexity 及 LLM-as-Judge 方法*

BLEU 计算公式

```
BLEU = BP × exp(Σ_n w_n × log p_n)

其中：
- p_n = n-gram 精确率（修改后的精确率）
- w_n = n-gram 权重（通常 BLEU-4: w = [0.25, 0.25, 0.25, 0.25]）
- BP = 短惩罚 (Brevity Penalty)

短惩罚 (BP)：
  BP = 1                    if c > r
  BP = exp(1 - r/c)         if c ≤ r

  其中 c = 候选文本长度, r = 参考文本长度

修改后的 n-gram 精确率 (Modified n-gram Precision)：
  p_n = Σ Count_clip(n-gram) / Σ Count(n-gram)

  Count_clip：将候选中每个 n-gram 的计数截断到其在参考中出现的最大次数
  目的：防止重复 n-gram 人为提高分数
```

ROUGE 变体对比

| 指标 | 原理 | 公式 | 适用场景 |
| --- | --- | --- | --- |
| **ROUGE-N** | n-gram 召回率 | R = 匹配的 n-gram 数 / 参考中的 n-gram 数 | 摘要质量 |
| **ROUGE-1** | unigram 召回 | 衡量单词级别的覆盖 | 信息覆盖度 |
| **ROUGE-2** | bigram 召回 | 衡量短语级别的流畅性 | 可读性 |
| **ROUGE-L** | 最长公共子序列(LCS) | F = (1+β^2)×P×R / (R+β^2×P) | 句子级别相似 |
| **ROUGE-W** | 加权 LCS | 连续匹配加分 | 更偏好连续匹配 |
| **ROUGE-S** | Skip-bigram | 允许跳过中间词 | 语序灵活性 |

Perplexity 计算公式

```
PPL = exp(-1/N × Σ_i log P(x_i | x_1, ..., x_{i-1}))

其中：
- N = 序列中的 token 数
- P(x_i | x_1, ..., x_{i-1}) = 模型预测第 i 个 token 的概率

等价形式（基于 cross-entropy）：
PPL = exp(H(p, q))
其中 H(p, q) = -1/N × Σ_i log q(x_i) 为交叉熵

直觉理解：
- PPL = 1：模型完美预测每个 token（不现实）
- PPL = 10：模型在每个位置平均在 10 个候选词中犹豫
- PPL = 100：模型非常不确定，质量差
```

LLM-as-Judge 方法分类

| 方法 | 描述 | 优点 | 缺点 |
| --- | --- | --- | --- |
| **单点评分** | 对单个回答直接打分（1-10） | 简单直接 | 分数标定困难 |
| **Pairwise 比较** | 比较两个回答哪个更好 | 判断更一致 | 需要多次比较 |
| **参考对比** | 与标准答案对比评分 | 有明确标准 | 需要标准答案 |
| **维度拆分** | 分维度评分（有用/安全/流畅） | 可解释性强 | Prompt 设计复杂 |
| **自洽性检查** | 多次评估同一内容，检查一致性 | 提高可靠性 | 成本增加 |
