GSM8K 示例

```
问题：Janet 每天睡 8 小时。她每天工作的时间是睡觉时间的 3 倍。
      她每天有多少小时可以做其他事情？

标准答案（Chain-of-Thought）：
1. Janet 每天睡 8 小时
2. 工作时间 = 8 × 3 = 24 小时？不对，一天只有 24 小时
   重新理解：工作时间 = 8 × 3 = 24 小时... 题目意思应是...
   实际上：工作 = 8 × 3 = 24 小时（题目设定如此，一天24h）
   其他时间 = 24 - 8 - 24 = -8... 题目有问题

[注意：部分 GSM8K 题目有类似的问题，这也是评估的难点之一]

正确理解示例：
问题：Natalia 4月份卖了48个夹子，5月份卖的夹子是4月份的2倍。
      Natalia 总共卖了多少夹子？

答案：4月份卖了48个
      5月份卖了48 × 2 = 96个
      总共 = 48 + 96 = 144个
      #### 144
```


**评估方式：**生成 Chain-of-Thought 解答，提取最终数字答案与标准答案比较


**代表分数：**GPT-4 ≈ 92.0%, Claude 3 ≈ 95.0%, Gemini Pro ≈ 86.5%, Llama 3 70B ≈ 77.4%

MT-Bench 评估类别

| 类别 | 评估维度 | 示例 |
| --- | --- | --- |
| 写作 | 创意、结构、语言 | 写一首关于春天的诗 |
| 角色扮演 | 角色一致性、沉浸感 | 扮演一个历史学家... |
| 信息提取 | 准确性、完整性 | 总结这篇文章的要点 |
| 推理 | 逻辑性、正确性 | 数学/逻辑推理问题 |
| 数学 | 计算准确性 | 数学应用题 |
| 编程 | 代码正确性 | 实现一个算法 |
| 知识 I | 事实准确性 | 科学/历史知识问答 |
| 知识 II | 深度和广度 | 专业知识讨论 |


**特点：**多轮对话评估（第一轮 + 追问），GPT-4 评分，速度和成本远低于人类评估

HELM 评估维度

| 维度 | 衡量内容 | 评估方法 |
| --- | --- | --- |
| 准确性（Accuracy） | 预测是否正确 | 标准基准测试 |
| 校准性（Calibration） | 置信度是否与准确率匹配 | ECE 指标 |
| 鲁棒性（Robustness） | 面对扰动是否稳定 | 对抗样本测试 |
| 公平性（Fairness） | 对不同群体是否公平 | 跨群体性能差异 |
| 偏见（Bias） | 是否包含社会偏见 | BBQ/CROWS 等 |
| 毒性（Toxicity） | 是否生成有害内容 | 毒性分类器 |
| 效率（Efficiency） | 推理速度和资源消耗 | 延迟/吞吐量 |

评估方式对比

| 方式 | 描述 | 优缺点 |
| --- | --- | --- |
| **Zero-shot** | 直接提问，不给示例 | 测试真实能力，但可能因 prompt 格式不当导致分数偏低 |
| **Few-shot** | 提供 1-5 个示例 | 更公平的比较，但可能泄露答案模式 |
| **CoT (Chain-of-Thought)** | 要求逐步推理 | 显著提升推理题分数 |
| **Self-Consistency** | 多次生成取多数答案 | 提升准确性但成本增加 |
