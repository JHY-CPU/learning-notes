# 人工评估方法 - LLM评估体系

*掌握 LLM 输出质量的人工评估方法论：Likert 评分、Pairwise 比较、评估标准设计、评估者间一致性（Kappa）及众包评估质量控制*

Likert 评分量表示例（对话质量评估）

| 分数 | 等级 | 锚定描述 |
| --- | --- | --- |
| **5** | 优秀 | 回答完全准确、全面、有深度，语言流畅，无任何问题 |
| **4** | 良好 | 回答基本准确全面，可能有微小遗漏或不够精炼 |
| **3** | 一般 | 回答部分正确，有明显遗漏或表述不够清晰 |
| **2** | 较差 | 回答有明显错误或严重遗漏，基本不可用 |
| **1** | 很差 | 回答完全错误、无关或包含有害内容 |

Pairwise 比较协议

```
评估任务：比较以下两个 AI 回答的质量

用户问题：[问题内容]

回答 A：[模型 A 的回答]
回答 B：[模型 B 的回答]

请选择更好的回答（匿名处理，评估者不知道哪个是哪个）：

□ 回答 A 明显更好
□ 回答 A 略好
□ 两者差不多
□ 回答 B 略好
□ 回答 B 明显更好

评估维度：
□ 准确性：信息是否正确
□ 完整性：是否覆盖了关键点
□ 清晰度：是否表达清晰易懂
□ 有用性：是否真正帮助了用户
□ 安全性：是否包含不当内容

开放性评价（简要说明选择原因）：
[文字框]
```

评估方法选择对比

| 维度 | Likert 评分 | Pairwise 比较 |
| --- | --- | --- |
| **判断类型** | 绝对判断 | 相对判断 |
| **可靠性** | 中等（受锚定理解影响） | 高（相对更一致） |
| **可扩展性** | 高（N 个回答只需 N 次评分） | 低（N 个回答需要 C(N,2) 次比较） |
| **信息量** | 有绝对质量参考 | 只有相对排名 |
| **适用场景** | 质量监控、设定基准线 | 模型比较、排行榜 |
| **统计方法** | 均值、标准差 | Elo 排名、Bradley-Terry |

LLM 输出评估的核心维度

| 维度 | 定义 | 评估要点 |
| --- | --- | --- |
| **准确性 (Accuracy)** | 事实信息是否正确 | 核心事实、数字、日期、引用是否准确 |
| **完整性 (Completeness)** | 是否覆盖了所有关键点 | 是否有重要遗漏 |
| **相关性 (Relevance)** | 是否切题回答 | 是否跑题或过度延伸 |
| **清晰度 (Clarity)** | 表达是否易懂 | 逻辑是否清晰、结构是否合理 |
| **深度 (Depth)** | 分析是否有洞察力 | 是否只是表面罗列 |
| **有用性 (Helpfulness)** | 是否真正解决了用户需求 | 是否给出了可操作的建议 |
| **安全性 (Safety)** | 是否包含有害内容 | 毒性、偏见、隐私泄露 |
| **流畅性 (Fluency)** | 语言是否自然 | 语法、用词、句式 |

Cohen's Kappa 公式

```
Kappa (κ) = (P_o - P_e) / (1 - P_e)

其中：
- P_o = 观察到的一致比例（实际一致数 / 总评估数）
- P_e = 期望的偶然一致比例 = Σ P(row_i) × P(col_i)

Kappa 值解读：
  κ < 0.00    : 差（Poor）
  0.00-0.20   : 微弱（Slight）
  0.21-0.40   : 一般（Fair）
  0.41-0.60   : 中等（Moderate）
  0.61-0.80   : 较好（Substantial）
  0.81-1.00   : 几乎完美（Almost Perfect）

LLM 评估中的经验标准：
  Pairwise 比较: κ > 0.6 可接受, κ > 0.8 良好
  Likert 评分: κ > 0.4 可接受（因为类别更多，一致性天然更低）
```
