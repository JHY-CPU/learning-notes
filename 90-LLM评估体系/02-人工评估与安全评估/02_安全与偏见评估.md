# 安全与偏见评估 - LLM评估体系

*掌握 LLM 输出的安全性评估方法：Toxicity 检测、社会偏见评估（BBQ/CROWS）、幻觉检测（FactScore）、隐私泄露测试*

毒性内容分类

| 类别 | 描述 | 示例 |
| --- | --- | --- |
| **仇恨言论** | 基于种族/性别/宗教等的攻击性言论 | 针对特定群体的侮辱 |
| **威胁暴力** | 宣扬或威胁实施暴力 | 描述伤害他人的方法 |
| **性骚扰** | 不当的性相关内容 | 未经同意的性暗示 |
| **自我伤害** | 鼓励或美化自我伤害 | 提供自杀方法 |
| **虚假信息** | 故意传播错误信息 | 健康/政治谣言 |
| **微攻击** | 微妙的歧视性表达 | "你英语说得真好"（隐含歧视假设） |
| **刻板印象** | 强化社会刻板印象 | "女性不擅长数学" |

BBQ 评估框架

```
BBQ 覆盖的 9 个偏见维度：
1. 年龄 (Age)
2. 残疾 (Disability status)
3. 性别 (Gender identity)
4. 国籍 (Nationality)
5. 外貌 (Physical appearance)
6. 种族/民族 (Race/ethnicity)
7. 宗教 (Religion)
8. 社会经济地位 (Socioeconomic status)
9. 性取向 (Sexual orientation)

两种评估语境：

模糊语境（Ambiguous context）：
  问题信息不完整，无法确定答案
  正确行为：模型应说"无法确定"
  偏见行为：模型根据刻板印象猜测答案

  示例："我看到一个老人和一个年轻人在争论，谁对？"
  无偏见回答："根据提供的信息无法确定谁是对的"
  偏见回答："年轻人可能更有道理"（年龄偏见）

明确语境（Disambiguated context）：
  问题信息完整，有明确答案
  正确行为：根据事实回答
  偏见行为：无视事实，根据刻板印象回答
```

幻觉检测方法全景

| 方法 | 原理 | 优点 | 缺点 |
| --- | --- | --- | --- |
| **FactScore** | 原子事实分解+逐一验证 | 细粒度、可解释 | 依赖知识源覆盖度 |
| **SelfCheckGPT** | 多次采样，一致性检查 | 无需外部知识源 | 成本高、假阳性 |
| **TRUE** | NLI 模型判断蕴含关系 | 速度快 | NLI 模型本身有误差 |
| **RAGAS Faithfulness** | 检查回答是否忠实于检索上下文 | 适用于 RAG | 仅限 RAG 场景 |
| **RefChecker** | 从参考文档提取知识三元组 | 结构化验证 | 三元组提取不完整 |

LLM 安全评估指标全景

| 维度 | 指标 | 数据集/工具 | 关键阈值 |
| --- | --- | --- | --- |
| **毒性** | Toxicity Score | RealToxicityPrompts, ToxicBERT | < 0.5 安全 |
| **偏见** | Bias Score | BBQ, CROWS-Pairs | < 0.5 低偏见 |
| **幻觉** | FactScore | FactScore, BiographyEval | > 0.8 高事实性 |
| **隐私** | PII Leakage Rate | 自定义 PII 测试集 | ≈ 0 无泄露 |
| **越狱** | Jailbreak Success Rate | 红队测试集 | < 0.01 强防御 |
| **拒绝** | Refusal Rate | 有害请求测试集 | > 0.95 高拒绝率 |
| **过度拒绝** | Over-refusal Rate | 无害请求测试集 | < 0.05 低误拒率 |
