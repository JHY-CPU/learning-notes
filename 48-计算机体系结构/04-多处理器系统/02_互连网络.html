02 - 互连网络

一、互连网络概述

互连网络（Interconnection Network）是多处理器系统中连接处理器、内存和I/O设备的通信基础设施，其性能直接影响系统的可扩展性和通信效率。

1.1 互连网络的功能

  • 处理器之间的数据传输
  • 处理器到共享内存的访问路由
  • 缓存一致性协议中的消息传递

1.2 关键性能指标

指标定义
带宽（Bandwidth）单位时间内传输的数据量
延迟（Latency）消息从源到目的地的时间
吞吐量（Throughput）单位时间内成功传输的消息数
可扩展性（Scalability）节点数增加时性能的变化

二、总线架构（Bus Architecture）

总线是最简单的互连方式，所有设备共享一组通信线路。

2.1 总线结构

┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ │CPU 0 │ │CPU 1 │ │CPU 2 │ │CPU 3 │ └──┬───┘ └──┬───┘ └──┬───┘ └──┬───┘ │ │ │ │ ═══╪═════════╪═════════╪═════════╪═══ ← 共享总线 │ │ │ │ ┌──┴───┐ ┌──┴───┐ │ MEM 0│ │ MEM 1│ └──────┘ └──────┘

2.2 总线仲裁

  • 集中式仲裁：由仲裁器统一分配总线使用权
    • 固定优先级
    • 轮询（Round Robin）
    • LRG（Least Recently Granted）
  • 分布式仲裁：各设备自行竞争
    • 自选择
    • 冲突检测（CSMA/CD）

2.3 总线的局限

总线是共享介质，带宽固定，所有设备竞争同一资源。节点增多时，总线成为性能瓶颈。通常适用于不超过 8-16 个处理器的系统。

三、交叉开关（Crossbar Switch）

交叉开关是一种非阻塞的互连网络，每个输入端口可以通过独立的开关连接到任意输出端口。

3.1 4x4 交叉开关

输出端口 M0 M1 M2 M3 P0 ──── ●────○────○────○ │ P1 ──── ○────●────○────○ │ P2 ──── ○────○────●────○ │ P3 ──── ○────○────○────● ● = 连接 ○ = 未连接 同一列只能有一个●（端口独占） 同一行可以有多个连接(若为多端口)

3.2 交叉开关特点

优点缺点
非阻塞：不同端口对可并行通信开关点数为 N²，成本高
低延迟：无中间节点仲裁复杂度高
高带宽不易大规模扩展

AMD Zen 架构使用交叉开关连接 CCX（Core Complex）内的 4 个核心和共享 L3 缓存。

四、多级互连网络（MIN）

多级互连网络通过多级开关单元连接输入和输出，平衡了总线和交叉开关的成本与性能。

4.1 Omega 网络（8x8）

输入: P0 P1 P2 P3 P4 P5 P6 P7 第1级: 第2级: 第3级: [2x2] [2x2] [2x2] 0──┤├──0 0──┤├──0 0──┤├──0──M0 1──┤├──1 1──┤├──1 1──┤├──1──M1 2──┤├──2 2──┤├──2 2──┤├──2──M2 3──┤├──3 3──┤├──3 3──┤├──3──M3 4──┤├──4 4──┤├──4 4──┤├──4──M4 5──┤├──5 5──┤├──5 5──┤├──5──M5 6──┤├──6 6──┤├──6 6──┤├──6──M6 7──┤├──7 7──┤├──7 7──┤├──7──M7 每级使用完美混洗(perfect shuffle)置换 8个输入需要 log₂8 = 3 级, 每级4个2x2开关

4.2 多级网络的阻塞特性

Omega 网络是阻塞网络 — 某些连接组合会产生内部冲突。

若 P0→M0 且 P4→M4，它们在某级开关上可能冲突（取决于具体路径）。

解决方案：使用 Clos 网络（非阻塞）或多路径冗余。

五、片上网络（Network-on-Chip, NoC）

随着芯片上核心数量增多（数十到数百个），传统的总线互连无法满足需求，NoC 应运而生。

5.1 Mesh 拓扑

4x4 Mesh 拓扑: ┌───┐ ┌───┐ ┌───┐ ┌───┐ │0,3│───│1,3│───│2,3│───│3,3│ └─┬─┘ └─┬─┘ └─┬─┘ └─┬─┘ │ │ │ │ ┌─┴─┐ ┌─┴─┐ ┌─┴─┐ ┌─┴─┐ │0,2│───│1,2│───│2,2│───│3,2│ └─┬─┘ └─┬─┘ └─┬─┘ └─┬─┘ │ │ │ │ ┌─┴─┐ ┌─┴─┐ ┌─┴─┐ ┌─┴─┐ │0,1│───│1,1│───│2,1│───│3,1│ └─┬─┘ └─┬─┘ └─┬─┘ └─┬─┘ │ │ │ │ ┌─┴─┐ ┌─┴─┐ ┌─┴─┐ ┌─┴─┐ │0,0│───│1,0│───│2,0│───│3,0│ └───┘ └───┘ └───┘ └───┘ 每个节点连接一个处理核心(PE)和路由器(Router)

5.2 路由算法

算法原理特点
XY 路由先沿X方向走，再沿Y方向走无死锁，简单
YX 路由先沿Y方向走，再沿X方向走无死锁
自适应路由根据网络拥塞动态选择路径负载均衡好，可能死锁
转弯模型禁止某些转弯以避免死锁平衡适应性和无死锁

5.3 NoC 路由器结构

每个路由器有5个端口: ┌─────────────┐ │ 北(N) │ │ │ │ │ 西(W)─┼─东(E)│ │ │ │ │ 南(S) │ │ │ │ │ 本地(PE) │ └─────────────┘ 路由器内部: 输入缓冲 → 路由计算 → 虚通道仲裁 → 交叉开关 → 输出

六、NUMA 架构

NUMA（Non-Uniform Memory Access）架构中，每个处理器有本地内存，也可访问其他处理器的远程内存，但远程访问延迟更高。

6.1 NUMA 拓扑示例

┌────────────────────────────────────────┐ │ NUMA 节点 0 │ │ ┌──────┐ ┌──────┐ │ │ │Core 0│ │Core 1│ │ │ └──┬───┘ └──┬───┘ │ │ └────┬────┘ │ │ ┌────┴────┐ │ │ │L3 Cache │ │ │ └────┬────┘ │ │ ┌────┴────┐ │ │ │本地内存0 │ 延迟: ~80ns │ │ └─────────┘ │ └──────────┬─────────────────────────────┘ │ 互连 (UPI/QPI) ┌──────────┴─────────────────────────────┐ │ NUMA 节点 1 │ │ ┌─────────┐ │ │ │本地内存1 │ 延迟: ~80ns │ │ └─────────┘ │ │ ┌──────┐ ┌──────┐ │ │ │Core 2│ │Core 3│ │ │ └──────┘ └──────┘ │ └────────────────────────────────────────┘ Core 0 访问本地内存0: ~80ns Core 0 访问远程内存1: ~140ns (1.75x延迟)

6.2 NUMA 感知优化

  • 内存分配策略：优先分配本地内存（Linux 的 first-touch 策略）
  • 线程绑定：将线程绑定到特定 NUMA 节点，减少远程访问
  • 数据布局：将频繁访问的数据放在对应 NUMA 节点
  • 操作系统调度：NUMA 感知的进程调度器
UMA vs NUMA：UMA（Uniform Memory Access）中所有处理器对内存的访问延迟相同（传统SMP）；NUMA 更具可扩展性但需要软件配合优化。

七、互连网络方案对比

方案带宽延迟成本可扩展性典型规模
