01 - 多核与缓存一致性

一、多核处理器架构

多核处理器在同一芯片上集成多个处理核心，共享部分或全部缓存和内存子系统，通过线程级并行（TLP）提高性能。

二、缓存一致性问题

当多个核心各自拥有私有缓存时，同一内存地址可能在不同缓存中有不同的副本，导致缓存不一致。

2.1 一致性问题示例

初始状态: 内存 X = 0 Core 0 缓存: X=0 Core 1 缓存: X=0 ┌──────────┐ ┌──────────┐ │ X: 0 │ │ X: 0 │ └──────────┘ └──────────┘ Core 0: X = 1 Core 0 缓存: X=1 Core 1 缓存: X=0 ← 不一致！ ┌──────────┐ ┌──────────┐ │ X: 1 │ │ X: 0 │ ← 陈旧副本 └──────────┘ └──────────┘ Core 1 读取 X 得到 0（错误！应该是1）

2.2 一致性要求

  • 写传播（Write Propagation）：一个核心对某位置的写入最终被其他核心看到
  • 事务串行化（Transaction Serialization）：所有核心看到的写入操作顺序一致

三、MSI 缓存一致性协议

MSI 是最基础的监听（Snooping）协议，每个缓存行有三种状态：

状态含义
M (Modified)该行已被修改，是唯一有效副本，内存中的数据是过时的
S (Shared)该行是干净的，可能在其他缓存中有副本
I (Invalid)该行无效（未缓存或已被驱逐）

3.1 MSI 状态转换图

本地写命中 ┌──────────────────────────┐ │ │ ▼ │ ┌──────────┐ 本地读命中/共享总线读 ┌──────────┐ │ I状态 │ ──────────────────────▶ │ S状态 │ │ (Invalid)│ │ (Shared) │ └────┬─────┘ └────┬─────┘ │ │ │ 本地读miss(总线读) │ 本地写 │ 本地写miss(总线读+无效化) │ │ │ ▼ ▼ ┌──────────┐ ┌──────────┐ │ │ ◀─── 本地写 ────────── │ │ │ M状态 │ │ M状态 │ │(Modified)│ │(Modified)│ └──────────┘ └──────────┘ │ │ 其他核心读miss(写回+转S) ▼ S状态

四、MESI 协议（Intel 使用）

MESI 在 MSI 基础上增加 E（Exclusive） 状态，减少不必要的总线写操作。

4.1 四种状态

状态缩写含义
MModified已修改，唯一副本，内存过时
EExclusive独占且干净，唯一副本，内存最新
SShared共享，可能有其他副本
IInvalid无效

4.2 E 状态的优势

当缓存行处于 E 状态时：

  • 本地写可以直接升级为 M 状态，无需总线事务（Silent Upgrade）
  • 减少总线通信开销，提高性能

获取 E 状态：读 miss 时，若其他缓存都没有该行，则获得 E 状态。

4.3 MESI 状态转换

I ──读miss(无其他副本)──▶ E ──本地写──▶ M I ──读miss(有其他副本)──▶ S I ──写miss──────────────▶ M E ──其他核心读──────────▶ S S ──本地写──────────────▶ M (需无效化) M ──其他核心读──────────▶ S (写回内存) E ──被驱逐──────────────▶ I (无需写回) M ──被驱逐──────────────▶ I (必须写回)

五、一致性协议的实现方式

5.1 监听协议（Snooping）

  • 所有缓存控制器监听共享总线上的所有事务
  • 适合总线互连的小规模多核系统（2-8核）
  • 优点：实现简单，延迟低
  • 缺点：总线带宽成为瓶颈

5.2 目录协议（Directory-based）

  • 维护一个目录，记录每个缓存行被哪些节点缓存
  • 适合大规模系统（多芯片、NUMA）
  • 优点：不依赖总线广播，可扩展性好
  • 缺点：目录存储开销大，延迟较高
特性监听协议目录协议
可扩展性低（总线瓶颈）高
通信方式广播点对点
存储开销低高（目录表）
典型应用单芯片多核多芯片/NUMA系统

六、内存一致性模型（Memory Consistency Model）

缓存一致性保证了单个地址的正确性，内存一致性定义了对不同地址的多个操作的可见顺序。

6.1 顺序一致性（Sequential Consistency, SC）

所有处理器的操作按某个全局顺序执行，且每个处理器的程序顺序被保持。

SC 最直观但性能最低，现代处理器通常不提供 SC。

6.2 松弛一致性模型

模型松弛的约束
Total Store Order (TSO)允许写→读重排（x86采用）
Partial Store Order (PSO)允许写→写重排
Release Consistency用 acquire/release 同步
弱一致性（ARM/RISC-V）几乎所有重排都允许，需显式屏障

6.3 内存屏障（Memory Barrier）

内存屏障强制特定类型的操作按顺序执行：

  • mfence（x86）：全序屏障，前后读写都不能越过
  • sfence：写屏障
  • lfence：读屏障
  • dmb（ARM）：数据内存屏障
