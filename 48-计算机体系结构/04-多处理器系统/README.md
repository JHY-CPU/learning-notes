# 04-多处理器系统

多处理器系统通过多个处理核心协同工作实现更高性能，是现代计算系统的核心架构范式。

---

## 1. 共享内存多处理器

### 1.1 SMP (对称多处理器)

所有处理器对等访问共享内存和I/O设备：

- 所有处理器运行同一操作系统副本，通过共享内存通信
- 典型代表：早期多核x86服务器

### 1.2 UMA (统一内存访问)

所有处理器到所有内存位置的访问延迟相同，适用于核心数较少的系统（通常 ≤ 8核），扩展性受限于共享总线带宽。

### 1.3 NUMA (非统一内存访问)

每个处理器有本地内存，访问本地内存快于访问远程内存：

```
Node 0                Node 1
┌──────────────┐      ┌──────────────┐
│ CPU0    CPU1 │      │ CPU2    CPU3 │
│  Local Mem   │──────│  Local Mem   │
└──────────────┘ QPI  └──────────────┘
```

- **ccNUMA (Cache-Coherent NUMA)**：带缓存一致性的NUMA，AMD EPYC、Intel Xeon均采用
- **NUMA效应**：跨节点访问延迟可增加50%~200%，编程需注意内存绑定

---

## 2. 分布式共享内存

在分布式系统（集群）中通过软件层提供统一的共享内存抽象，实际数据分布在各节点的本地内存中。典型实现包括DSM系统 (如TreadMarks) 和PGAS语言 (UPC、Chapel)。一致性维护基于页面的虚存机制捕获访问并触发操作，通信开销远高于硬件共享内存系统。

---

## 3. 多处理器互连网络

### 3.1 总线 (Bus)

所有设备共享同一通信线路。优点：简单、成本低、广播天然支持。缺点：带宽竞争严重，扩展性差。典型应用：单插槽多核的L3缓存一致性。

### 3.2 交叉开关 (Crossbar)

N个输入到N个输出的无阻塞交换网络。优点：无冲突、高带宽、低延迟。缺点：硬件复杂度为 O(N^2)。典型应用：芯片内连接核心与内存控制器。

### 3.3 环形网络 (Ring)

处理器按环形拓扑连接，数据沿环传递。优点：结构简单。缺点：远距离通信延迟高（多跳）。典型应用：Intel早期多核（Nehalem的QPI Ring）。

### 3.4 Mesh / 二维网格

处理器排列成二维网格，每个节点与上下左右邻居连接。优点：扩展性好、布线规整。缺点：对角节点延迟较高。典型应用：Intel Skylake-SP的Mesh互连、AMD EPYC的Infinity Fabric。

### 3.5 多级互连网络

包括Omega网络（多级shuffle-exchange）和Butterfly网络（多级蝶形），适用于大规模并行处理器。

## 4. 内存一致性模型

### 4.1 顺序一致性 (SC)

最强的一致性模型：任何执行结果等价于所有处理器的操作按某种顺序排列，且每个处理器的操作按程序顺序排列。编程直观但严重限制硬件优化，实际中很少有硬件严格实现。

### 4.2 TSO (Total Store Order)

x86架构采用的一致性模型：

- 读-读、读-写、写-读保持程序顺序，**写-写可以重排**（通过Store Buffer）
- Store Buffer通过Store Fence (SFENCE) 指令刷新

```
TSO允许: Store → Load 重排（经由Store Buffer）
TSO禁止: Load → Load, Load → Store, Store → Store 重排
```

### 4.3 PSO (Partial Store Order)

在TSO基础上进一步放松：允许写-写重排（不同地址的写操作），SPARC架构曾使用。

### 4.4 松散一致性 (Relaxed / Weak Ordering)

现代处理器（ARM、RISC-V、POWER）广泛采用：几乎所有内存操作都可重排，通过**内存屏障 (Memory Barrier/Fence)** 指令显式强制顺序。

```armasm
STR X1, [X0]      ; 写Flag
DMB SY             ; 内存屏障
STR X2, [X3]      ; 写Data（保证在Flag之后被看到）
```

### 4.5 获取-释放语义 (Acquire-Release)

C++11的内存模型引入：Acquire保证后续读写不能重排到此操作之前，Release保证之前的读写不能重排到此操作之后。配合 `std::atomic` 使用，跨平台可移植。

## 5. 同步原语的硬件实现

### 5.1 Test-and-Set (TAS)

原子地读取并设置内存位置为1。自旋锁实现：循环执行TAS直到获取锁。缺点：总线流量大，不保证公平。

### 5.2 Compare-and-Swap (CAS)

原子地比较并交换：

```c
bool CAS(int *addr, int expected, int new_val) {
    if (*addr == expected) { *addr = new_val; return true; }
    return false;
}
```

x86的 `CMPXCHG` 指令实现CAS，广泛用于无锁数据结构。

### 5.3 Load-Linked / Store-Conditional (LL/SC)

RISC架构（MIPS、ARM、RISC-V）的同步机制对：LL加载值并注册链接监视，SC若链接未被破坏则写入，否则失败。

```mips
retry:
    LL  $t0, 0($a0)       # 加载锁值
    BNE $t0, $zero, retry  # 已锁则重试
    LI  $t1, 1
    SC  $t1, 0($a0)        # 尝试加锁
    BEQ $t1, $zero, retry  # SC失败则重试
```

### 5.4 Fetch-and-Op

原子地执行算术运算并返回旧值。x86：`LOCK XADD`；ARM：`LDADD`。用于实现计数器、累加器等。

## 6. Cache一致性协议在多核中的作用

### 6.1 一致性协议作为同步基础

硬件同步原语（CAS、LL/SC）依赖缓存一致性协议保证原子性：核心发起原子操作，缓存控制器获取独占权限，在独占状态下完成读-改-写，其他核心因缓存行无效化而无法同时修改。

### 6.2 伪共享 (False Sharing)

不同核心访问同一缓存行中不同变量，导致不必要的缓存行无效化。解决方法：使用padding将不同核心的数据分隔到不同缓存行（通常64字节对齐）。

### 6.3 自旋锁的优化

- **Test-and-Test-and-Set (TTAS)**：先本地读检查，减少CAS次数
- **队列锁 (MCS Lock)**：每个核心在自己的节点上自旋，避免总线风暴
- **Ticket Lock**：FIFO公平，类似排队取号

## 7. 多线程技术

### 7.1 细粒度多线程 (Fine-Grained Multithreading)

每个时钟周期切换到不同线程，固定轮转调度，有效隐藏长延迟操作。代表：Sun UltraSPARC T系列。

### 7.2 粗粒度多线程 (Coarse-Grained Multithreading)

当前线程阻塞（如缓存缺失）时才切换到其他线程，切换开销较大（需清空流水线）。

### 7.3 同时多线程 (SMT)

多个线程同时在流水线中执行，共享执行单元。Intel Hyper-Threading (2路)、IBM POWER8 (8路)。典型性能提升15%~30%。

### 7.4 多线程的性能权衡

- 线程数增加可提升吞吐率，但单线程性能可能下降
- 共享资源（缓存、带宽）竞争加剧，适合高延迟、高并行的工作负载

---

## 8. 片上网络 (NoC) 基础

### 8.1 从总线到NoC

随着芯片集成的核心数增加（数十到数百），传统总线互连成为瓶颈。NoC借鉴计算机网络的思想在芯片上构建分组交换网络。

### 8.2 NoC拓扑

| 拓扑 | 延迟 | 带宽 | 面积开销 |
|------|------|------|----------|
| Mesh | O(sqrt(N)) | 中等 | 低 |
| Torus | O(sqrt(N)) | 较高 | 中 |
| Fat-Tree | O(log N) | 高 | 高 |
| Ring | O(N) | 低 | 最低 |

### 8.3 路由与流控

- **确定性路由**：XY路由简单无死锁；**自适应路由**：根据拥塞动态选择路径
- **路由器**包含输入/输出端口、交叉开关、仲裁逻辑
- **流控**：信用机制、虚通道 (Virtual Channel)

---

## 9. 异构计算

### 9.1 CPU + GPU

最常见的异构计算模式：CPU负责控制流密集型任务，GPU负责数据并行密集型任务，通过PCIe或NVLink互连。编程模型：CUDA、OpenCL、SYCL、OpenACC。

### 9.2 CPU + FPGA

FPGA提供可定制的数据流水线，适合网络处理、视频转码等特定加速场景。代表：Intel集成FPGA的Xeon、AMD Xilinx Vitis平台。

### 9.3 SoC异构集成

移动和嵌入式SoC中集成CPU、GPU、DSP、NPU/AI加速器、ISP等多种处理单元，协同处理不同工作负载。

### 9.4 Chiplet与先进封装

后摩尔时代的扩展策略：将不同功能模块作为独立芯片（Chiplet），通过2.5D/3D先进封装互连。代表：AMD EPYC多Chiplet设计、Intel EMIB/Foveros。

### 9.5 异构编程的挑战

- **数据移动开销**：CPU与加速器之间的数据传输是主要瓶颈
- **编程复杂性**：不同加速器有不同的编程模型
- **负载均衡**：合理分配任务到不同类型的处理单元
- **统一内存**：UVM简化编程但有性能陷阱
