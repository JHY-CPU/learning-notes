# 02-存储层次与缓存

存储层次结构是现代处理器性能优化的基石，通过多级缓存体系弥合CPU与主存之间的速度差距。

---

## 1. 存储层次结构

### 1.1 存储器金字塔

处理器的存储系统按照速度、容量和成本构成层次结构：

| 层次 | 典型容量 | 访问延迟 | 技术 |
|------|----------|----------|------|
| 寄存器 | < 1 KB | 1 个时钟周期 | 触发器 |
| L1 Cache | 32~64 KB | 1~3 周期 | SRAM |
| L2 Cache | 256 KB~1 MB | 5~15 周期 | SRAM |
| L3 Cache | 4~64 MB | 20~50 周期 | SRAM |
| 主存 (DRAM) | 4~256 GB | 100~300 周期 | DRAM |
| 磁盘/SSD | TB级 | ~10^5 周期 | 磁/闪存 |

### 1.2 局部性原理

缓存有效性的理论基础：

- **时间局部性 (Temporal Locality)**：最近被访问的数据很可能再次被访问
- **空间局部性 (Spatial Locality)**：与被访问数据地址相邻的数据很可能被访问

---

## 2. 缓存的基本概念

### 2.1 命中与缺失

- **命中 (Hit)**：请求的数据在缓存中找到
- **缺失 (Miss)**：请求的数据不在缓存中，需从下一级存储获取
- **命中率 (Hit Rate)**：h = 命中次数 / 总访问次数
- **缺失率 (Miss Rate)**：m = 1 - h

### 2.2 AMAT (平均内存访问时间)

$$\text{AMAT} = \text{Hit Time} + \text{Miss Rate} \times \text{Miss Penalty}$$

对于多级缓存：

$$\text{AMAT} = H_1 + M_1 \times (H_2 + M_2 \times P)$$

其中 $H_i$ 为第i级命中时间，$M_i$ 为第i级缺失率，$P$ 为主存访问延迟。

### 2.3 缺失的3C分类

- **强制缺失 (Compulsory)**：冷启动缺失，首次访问某块
- **容量缺失 (Capacity)**：缓存容量不足以容纳全部工作集
- **冲突缺失 (Conflict)**：映射规则限制导致的争用缺失

---

## 3. 缓存映射方式

### 3.1 直接映射 (Direct Mapped)

每个主存块只能映射到缓存中唯一的位置：

```
缓存地址 = 主存块号 mod 缓存行数
```

- **优点**：硬件简单，查找速度快
- **缺点**：冲突缺失率高

地址划分：`[Tag | Index | Block Offset]`

### 3.2 全相联 (Fully Associative)

主存块可以放入缓存中任意位置。优点：冲突缺失率最低。缺点：需要并行比较所有Tag，硬件开销大。

### 3.3 组相联 (Set Associative)

折中方案：缓存分为多个组 (Set)，每组包含多个路 (Way)。组号 = 主存块号 mod 组数，组内任意位置可放置。2路、4路、8路组相联是常见配置。地址划分：`[Tag | Set Index | Block Offset]`

---

## 4. 替换策略

当缓存已满需要替换某一块时：

### 4.1 LRU (Least Recently Used)

替换最久未使用的块。对于N路组相联，需维护N个状态位的访问顺序。

- 2路：只需1位（近似LRU）
- 4路：树形LRU只需3位
- 精确LRU在高相联度下硬件开销大

### 4.2 FIFO (First In First Out)

替换最早进入缓存的块，实现简单但可能替换掉热点数据。

### 4.3 Random

随机选择一块替换。实测中性能接近LRU，硬件最简单。

---

## 5. 写策略

### 5.1 写命中策略

- **写直达 (Write-Through)**：同时写缓存和下一级存储，数据始终一致但写延迟高
- **写回 (Write-Back)**：只写缓存，设置脏位 (Dirty Bit)，替换时再写回下一级。写延迟低但需要额外的一致性维护

### 5.2 写缺失策略

- **写分配 (Write-Allocate)**：先将块调入缓存再写（配合写回策略）
- **写不分配 (No-Write-Allocate)**：直接写入下一级存储，不调入缓存（配合写直达策略）

### 5.3 写缓冲 (Write Buffer)

写直达策略配合写缓冲区，CPU写操作先写入缓冲区，由缓冲区异步写入下一级存储，避免CPU等待。

---

## 6. 缓存一致性问题

### 6.1 问题背景

在多核处理器中，每个核心有私有缓存，同一内存地址在多个缓存中可能有不同副本，导致数据不一致：

```
Core 0 缓存: X = 5    Core 1 缓存: X = 3    主存: X = 0
```

### 6.2 一致性要求

- **写传播 (Write Propagation)**：对某地址的写操作最终要对其他核心可见
- **事务串行化 (Transaction Serialization)**：所有核心看到的写操作顺序一致

---

## 7. MESI协议详解

MESI是广泛使用的缓存一致性协议（也称Illinois协议），每个缓存行有4种状态：

| 状态 | 含义 | 数据是否有效 | 是否为最新 |
|------|------|------------|----------|
| **M** (Modified) | 已修改 | 是 | 是（唯一副本） |
| **E** (Exclusive) | 独占 | 是 | 是 |
| **S** (Shared) | 共享 | 是 | 是（可能有其他副本） |
| **I** (Invalid) | 无效 | 否 | 否 |

### 状态转换核心规则

- 本地读命中：M/E/S状态直接命中
- 本地写命中：M/E状态直接写；S状态需广播Invalidate
- 本地读缺失：向总线发BusRd，其他缓存有则返回S，无则返回E
- 本地写缺失：向总线发BusRdX，获取独占权限后写
- 监听到BusRd（自己为M）：写回数据，转为S
- 监听到BusRdX/BusUpgr：转为I

---

## 8. MOESI与MSIF变体

### 8.1 MOESI协议

在MESI基础上增加 **O (Owned)** 状态：

- **O状态**：数据已被修改（脏），但其他缓存有共享副本。O状态的缓存负责响应其他核心的读请求，无需访问主存
- 减少了主存访问，AMD处理器广泛采用

### 8.2 MSIF协议

- **F (Forward)** 状态代替E状态：Forward状态是独占且负责转发给其他核心
- Intel的某些实现中使用类似变体

---

## 9. 监听协议 vs 目录协议

### 9.1 监听协议 (Snooping)

所有缓存监听共享总线上的所有事务：

- **适用**：小规模多核（总线结构）
- **优点**：实现简单，延迟低
- **缺点**：总线带宽是扩展瓶颈

两种实现方式：
- **写无效 (Write-Invalidate)**：写时使其他副本无效（MESI系列）
- **写更新 (Write-Update)**：写时更新所有副本

### 9.2 目录协议 (Directory)

使用集中式或分布式目录记录每个缓存块的状态和共享者信息：

- **适用**：大规模多处理器（NUMA/分布式）
- **优点**：可扩展性好
- **缺点**：目录存储开销大，延迟较高

---

## 10. 虚拟内存与TLB

### 10.1 虚拟内存基本概念

- 将物理内存抽象为虚拟地址空间，每个进程有独立的虚拟地址空间
- **页 (Page)**：虚拟内存的固定大小单位（通常4KB）
- **页表 (Page Table)**：虚拟页号到物理页帧号的映射
- **缺页 (Page Fault)**：访问的页不在物理内存中，触发OS调度

### 10.2 TLB (Translation Lookaside Buffer)

页表的缓存，存储最近使用的虚拟地址到物理地址的映射：

- TLB通常很小（64~1024条目），全相联或组相联
- TLB命中时地址转换仅需1个周期
- TLB缺失时需遍历页表（或由硬件页表遍历器处理）
- 进程切换时需刷新TLB（或使用ASID区分不同进程）

### 10.3 多级页表

32位系统常用两级页表，64位系统常用4级页表（PGD→PUD→PMD→PTE），减少页表本身内存占用。

---

## 11. 预取策略

### 11.1 硬件预取

- **顺序预取**：检测到顺序访问模式后，预取后续若干块
- **步幅预取 (Stride Prefetcher)**：检测固定步幅的访问模式，预取对应地址
- **流预取 (Stream Prefetcher)**：为每个检测到的流维护预取窗口

### 11.2 软件预取与权衡

编译器插入预取指令（如x86的PREFETCH），在数据被需要之前提前加载到缓存。过度预取可能污染缓存、消耗带宽，需准确判断访问模式才能获得收益。
