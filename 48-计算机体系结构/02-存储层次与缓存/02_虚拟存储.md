# 虚拟存储 - 计算机体系结构笔记


虚拟存储（Virtual Memory）为每个进程提供一个**独立的、连续的地址空间**，通过硬件和操作系统的协作将虚拟地址映射到物理地址。


### 1.1 虚拟存储的作用


- **地址隔离**
   ：每个进程有独立的地址空间，互不干扰
- **内存扩展**
   ：虚拟地址空间可以大于物理内存
- **按需调页**
   ：只将需要的页面加载到物理内存
- **共享内存**
   ：不同进程可以映射到相同的物理页
- **内存保护**
   ：页面级的读/写/执行权限控制


### 1.2 地址空间


```
虚拟地址空间 (进程A)       物理地址空间
  ┌──────────────┐         ┌──────────────┐
  │    代码段     │────────▶│   物理页 3   │
  ├──────────────┤         ├──────────────┤
  │    数据段     │────────▶│   物理页 7   │
  ├──────────────┤         ├──────────────┤
  │    堆区       │────────▶│   物理页 12  │
  ├──────────────┤         ├──────────────┤
  │    (未映射)   │         │   物理页 15  │
  ├──────────────┤         ├──────────────┤
  │    栈区       │────────▶│   物理页 20  │
  └──────────────┘         └──────────────┘
```


### 2.1 基本分页


虚拟地址空间被划分为固定大小的**页（Page）**，物理内存被划分为相同大小的**页帧（Page Frame）**。


```
虚拟地址: [VPN (虚拟页号) | 页内偏移]

  页表将 VPN 映射到 PFN:
  VPN  →  [Valid | PFN | 权限 | 脏位 | ...]

  物理地址: [PFN (物理页帧号) | 页内偏移]
```


### 2.2 页表项（PTE）格式


| 字段 | 位宽 | 含义 |
| --- | --- | --- |
| Valid | 1 bit | 该页是否在物理内存中 |
| PFN | 20-52 bit | 物理页帧号 |
| Read/Write | 1-2 bit | 读写权限 |
| User/Supervisor | 1 bit | 用户态/内核态访问权限 |
| Dirty | 1 bit | 该页是否被修改过 |
| Accessed | 1 bit | 该页是否被访问过 |
| NX (No Execute) | 1 bit | 禁止执行位 |


64位系统的线性页表太大（如 2^52^ 个表项），使用**多级页表**按需分配页表空间。


### 3.1 两级页表（x86-32 示例）


```
虚拟地址 (32位):
  ┌──────────┬──────────┬────────────────┐
  │  PDE索引  │  PTE索引  │   页内偏移      │
  │  (10bit) │  (10bit) │   (12bit)      │
  └────┬─────┴────┬─────┴────────────────┘
       │          │
       ▼          ▼
  ┌─────────┐ ┌─────────┐
  │页目录PDE │ │页表PTE  │──▶ PFN
  │(1024项)  │ │(1024项) │
  └─────────┘ └─────────┘
```


### 3.2 四级页表（x86-64 / Linux）


```
虚拟地址 (48位有效):
  ┌─────┬─────┬─────┬─────┬──────────┐
  │PML4 │ PDP │ PD  │ PT  │  偏移     │
  │9bit │9bit │9bit │9bit │  12bit   │
  └──┬──┴──┬──┴──┬──┴──┬──┴──────────┘
     │     │     │     │
     ▼     ▼     ▼     ▼
    PML4 → PDP → PD  → PT  → 物理页帧
```

**优势**
：只有实际使用的虚拟地址区域才需要分配页表页，未使用的区域对应的页表不占用物理内存。对稀疏地址空间极其高效。

TLB 是一个专门用于地址翻译的**缓存**，存储最近使用的虚拟页号到物理页帧号的映射。


### 4.1 TLB 的结构


```
TLB (通常全相联或高路组相联, 64-1024 项)
  ┌──────┬──────────┬────────┬───────┬───────┐
  │ Valid│   VPN    │  PFN   │ 权限  │ ASID  │
  ├──────┼──────────┼────────┼───────┼───────┤
  │  1   │ 0x00401  │ 0x1A3F │ RWX   │  42   │
  │  1   │ 0x7FFF0  │ 0x05B2 │ RW-   │  42   │
  │  1   │ 0x00402  │ 0x2C10 │ R-X   │  17   │
  │  0   │    —     │   —    │  —    │  —    │
  └──────┴──────────┴────────┴───────┴───────┘
```


### 4.2 地址翻译流程


```
虚拟地址
      │
      ▼
  ┌─────────┐  命中    ┌──────────┐
  │ 查TLB   │────────▶│ 直接得到  │──▶ 物理地址
  └────┬────┘         │ 物理页帧  │
       │ 失效         └──────────┘
       ▼
  ┌─────────┐
  │ 查页表   │──▶ 命中: 更新TLB, 返回PFN
  └────┬────┘    失效: 缺页中断(Page Fault)
       │
       ▼
  缺页处理 → 从磁盘调入页 → 更新页表 → 更新TLB
```


### 4.3 ASID（Address Space ID）


ASID 标识不同进程的地址空间，使得**进程切换时无需清空 TLB**。


### 4.4 TLB 性能


- 命中时：1 个周期（与缓存访问并行）
- 失效时：几十到上百个周期（页表遍历）
- 命中率：通常 > 99%


当虚拟页的 Valid 位为 0 时，访问该页会触发**缺页中断（Page Fault）**。


### 5.1 缺页处理流程


1. 硬件陷入操作系统内核
2. 操作系统检查访问是否合法（权限检查）
3. 在物理内存中找一个空闲页帧
4. 若没有空闲页帧，执行
   **页面替换算法**
   驱逐某页
5. 若被驱逐页的 Dirty 位为 1，将其写回磁盘
6. 从磁盘读入目标页到选定的页帧
7. 更新页表和 TLB
8. 重新执行导致缺页的指令

**缺页代价**
：约 1-10ms（磁盘访问），远高于缓存失效。因此优化虚拟存储系统的关键是
**减少缺页率**
。

### 6.1 常见算法


| 算法 | 原理 | 优点 | 缺点 |
| --- | --- | --- | --- |
| **Optimal** | 替换未来最久不使用的页 | 理论最优 | 无法实现（需预知未来） |
| **LRU** | 替换最近最久未使用的页 | 接近Optimal | 精确LRU硬件开销大 |
| **FIFO** | 替换最先装入的页 | 简单 | 存在Belady异常 |
| **Clock** | 近似LRU，使用引用位循环扫描 | 开销低，效果好 | 非精确LRU |
| **LFU** | 替换访问频率最低的页 | 适合有热点的场景 | 历史开销大 |


### 6.2 Clock 算法详解


```
页帧排成环形链表，有一个指针（时钟指针）：

        → [ref=1] → [ref=0] → [ref=1] → [ref=0] →
       ↑ 指针
  替换时:
  - 若引用位=1，清零，指针下移
  - 若引用位=0，替换该页

  改进型Clock（考虑Dirty位）:
  优先级: (0,0) > (0,1) > (1,0) > (1,1)
  即优先替换未引用且未修改的页
```


### 6.3 Belady 异常


FIFO 算法可能在增加页帧数后反而缺页次数增加，称为 **Belady 异常**。LRU 和 Optimal 算法不受此影响（它们属于"栈算法"）。
