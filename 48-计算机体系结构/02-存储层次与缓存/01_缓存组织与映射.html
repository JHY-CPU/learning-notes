01 - 缓存组织与映射

一、存储层次结构（Memory Hierarchy）

存储层次结构利用程序的时间局部性和空间局部性，在容量、速度和成本之间取得平衡。

1.1 层次结构

┌───────────┐ │ 寄存器 │ ← 最快，容量最小(~几百字节) ├───────────┤ │ L1 Cache │ ← 32-64KB, ~1ns ├───────────┤ │ L2 Cache │ ← 256KB-1MB, ~3-10ns ├───────────┤ │ L3 Cache │ ← 4-64MB, ~10-40ns ├───────────┤ │ 主存DRAM │ ← GB级, ~50-100ns ├───────────┤ │ SSD/磁盘 │ ← TB级, ~100μs-10ms └───────────┘ 速度递减 容量递增 成本/位递减

1.2 局部性原理

类型含义利用方式
时间局部性最近被访问的数据很可能再次被访问缓存保持近期数据
空间局部性与被访问数据相邻的数据很可能被访问按块/行缓存

二、缓存基本结构

2.1 缓存行（Cache Line / Cache Block）

缓存与主存之间数据传输的基本单位，通常为 64 字节。

2.2 缓存行的组成

┌────────┬──────┬─────────────────────────────┐ │ Valid │ Tag │ Data (64B) │ │ 1bit │ 比较 │ 实际缓存的数据块 │ └────────┴──────┴─────────────────────────────┘

2.3 地址划分

主存地址被划分为：[Tag | Index | Block Offset]

  • Block Offset：块内偏移，log2(块大小) 位
  • Index：组号，log2(组数) 位
  • Tag：标记，剩余高位，用于区分映射到同一组的不同块

三、缓存映射方式

3.1 直接映射（Direct Mapped）

每个主存块只能映射到缓存中唯一一个位置：缓存行号 = 主存块号 % 缓存行数

地址: [Tag(高位) | Index(log₂缓存行数) | Offset(log₂块大小)] 主存块0 → 缓存行0 主存块1 → 缓存行1 主存块N → 缓存行 (N % 缓存行数) ... 查找: 用Index直接定位行，比较Tag判断是否命中
优点缺点
硬件简单，查找快冲突率高（抖动问题）
每个块位置唯一确定缓存利用率可能低

3.2 全相联（Fully Associative）

主存块可以映射到缓存中任意位置。

地址: [Tag(高位) | Offset(log₂块大小)] 无Index字段 — 必须并行比较所有行的Tag 查找: 并行比较所有缓存行的Tag → 匹配则命中 实现: 内容寻址存储器(CAM) — 硬件成本高
优点缺点
无冲突，缓存利用率最高查找需要并行比较所有行
适合小容量缓存(TLB)硬件成本高，不适合大容量

3.3 组相联（Set Associative）

缓存分为若干组（Set），每组包含 N 个行（N路组相联）。主存块映射到特定组，组内任意位置。

地址: [Tag | Index(log₂组数) | Offset] ┌─────────────────────────────────────────┐ │ 缓存（2路组相联） │ │ │ │ 组0: [行0A | 行0B] │ │ 组1: [行1A | 行1B] │ │ 组2: [行2A | 行2B] │ │ ... │ │ 组N: [行NA | 行NB] │ │ │ │ 查找: 用Index选择组，并行比较组内所有行的Tag │ └─────────────────────────────────────────┘

3.4 三种方式对比

方式关联度查找复杂度冲突率典型应用
直接映射1路1次比较高L1指令缓存
组相联N路(2-16)N次比较中L1/L2/L3数据缓存
全相联任意所有行比较低TLB, 小缓存

四、替换策略（Replacement Policy）

当缓存已满且发生容量失效或冲突失效时，需要选择一个缓存行进行替换。

策略原理硬件开销效果
LRU替换最久未使用的行需记录访问顺序好
FIFO替换最先装入的行简单的计数器中
Random随机选择一行几乎无开销接近LRU
PLRU伪LRU，用树结构近似log₂N 位接近LRU
实际应用：现代处理器L1缓存通常使用PLRU（伪LRU）或Random策略，因为LRU的硬件开销随路数增长而增加。

五、写策略（Write Policy）

5.1 写命中策略

策略原理优点缺点
写直达（Write-Through）同时写缓存和主存一致性简单写操作慢
写回（Write-Back）只写缓存，设脏位（dirty bit）写操作快一致性复杂

5.2 写不命中策略

策略原理
写分配（Write-Allocate）先将块载入缓存，再写（配合写回）
写不分配（No-Write-Allocate）直接写主存，不调入缓存（配合写直达）
常见组合：Write-Back + Write-Allocate（现代处理器主流）和 Write-Through + No-Write-Allocate。

六、缓存性能分析

6.1 失效类型（3C 模型）

类型英文原因解决方法
强制失效Compulsory (Cold)首次访问某块预取（Prefetching）
容量失效Capacity缓存容量不足增大缓存
冲突失效Conflict多个块竞争同组/行提高关联度
一致性失效Coherence (4C)多核间缓存一致性协议优化

6.2 平均访问时间（AMAT）

AMAT = Hit Time + Miss Rate × Miss Penalty 示例 (2级缓存): AMAT = H1 + MR1 × (H2 + MR2 × MP) 其中: H1 = L1命中时间 (~1ns) MR1 = L1失效率 (~5%) H2 = L2命中时间 (~5ns) MR2 = L2失效率 (~20%) MP = 主存访问时间 (~100ns) AMAT = 1 + 0.05 × (5 + 0.20 × 100) = 1 + 0.05 × 25 = 1 + 1.25 = 2.25 ns
