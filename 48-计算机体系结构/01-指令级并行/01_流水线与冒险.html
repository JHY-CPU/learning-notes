01 - 流水线与冒险

一、流水线基本概念

流水线（Pipelining）是一种将指令执行过程分解为多个阶段的技术，使得多条指令可以重叠执行，从而提高指令吞吐率。

1.1 经典五级流水线

阶段全称主要操作
IFInstruction Fetch从指令存储器取指令
IDInstruction Decode译码，读寄存器
EXExecuteALU运算或地址计算
MEMMemory Access读写数据存储器
WBWrite Back将结果写回寄存器

1.2 流水线时空图

周期: 1 2 3 4 5 6 7 8 9 指令1: IF ID EX MEM WB 指令2: IF ID EX MEM WB 指令3: IF ID EX MEM WB 指令4: IF ID EX MEM WB 指令5: IF ID EX MEM WB 非流水线: 5条指令 × 5周期 = 25周期 流水线: 5 + (5-1) = 9周期 加速比: 理想情况下接近5倍

1.3 流水线性能指标

指标定义理想值
吞吐率（Throughput）单位时间完成的指令数1条/周期
加速比（Speedup）非流水线时间/流水线时间= 流水线级数
效率（Efficiency）实际吞吐率/理想吞吐率100%

二、结构冒险（Structural Hazard）

结构冒险发生在硬件资源冲突时 — 多条指令在同一周期需要使用同一硬件资源。

2.1 典型场景

指令和数据共享同一存储器: 周期: 1 2 3 4 5 I1: IF ID EX MEM WB I2: IF ID EX MEM WB I3: IF ID EX MEM WB ↑ I3的IF阶段需要访问存储器,但I1的MEM阶段也在访问 → 结构冒险!

2.2 解决方法

  • 分离指令和数据存储器（哈佛架构）— 现代处理器都采用此方案
  • 增加资源副本：多端口寄存器文件、分离的指令/数据Cache
  • 插入流水线气泡（Stall）：暂停后续指令一个周期

三、数据冒险（Data Hazard）

数据冒险发生在指令之间存在数据依赖，而后续指令在前一条指令的结果产生之前就需要使用它。

3.1 三种数据依赖

类型名称模式是否必须顺序
RAW真依赖（Read After Write）先写后读是
WAR反依赖（Write After Read）先读后写否（重命名可解）
WAW输出依赖（Write After Write）先写后写否（重命名可解）

3.2 RAW 冒险示例

ADD R1, R2, R3 // R1 = R2 + R3

SUB R4, R1, R5 // R4 = R1 - R5 （需要R1的新值）


ADD 在 WB 阶段写回 R1，SUB 在 ID 阶段读取 R1 — 间隔2个周期，产生冒险。

四、数据前递（Forwarding / Bypassing）

前递技术通过添加额外的数据通路，将计算结果提前转发给后续指令，避免不必要的暂停。

4.1 前递通路

┌──────────┐ │ EX/MEM │──── EX→EX 前递 ──→ EX阶段输入 │ 寄存器 │ └──────────┘ │ ┌──────────┐ │ MEM/WB │──── MEM→EX 前递 ──→ EX阶段输入 │ 寄存器 │──── MEM→ID 前递 ──→ ID阶段输入 └──────────┘

4.2 前递判断条件

// EX→EX 前递 if (EX/MEM.RegWrite && EX/MEM.Rd != 0 && EX/MEM.Rd == ID/EX.Rs1) { ALU_input_A = EX/MEM.ALU_result; // 前递 } // MEM→EX 前递 if (MEM/WB.RegWrite && MEM/WB.Rd != 0 && MEM/WB.Rd == ID/EX.Rs1 && !(EX/MEM冲突已处理)) { ALU_input_A = MEM/WB.result; // 前递 }

4.3 Load-Use 冒险 — 前递无法解决的情况

周期: 1 2 3 4 5 6 LW R1, IF ID EX MEM→R1 WB 0(R2): ADD R3, IF ID ? EX MEM WB R1, R4: ADD在EX阶段需要R1，但LW在MEM阶段才读到数据 即使MEM→EX前递，也来不及! → 必须插入1个气泡(Stall)
Load-Use Hazard是五级流水线中唯一无法完全通过前递消除的数据冒险。解决方案：编译器调度（将无关指令插入Load和Use之间）或硬件插入1个气泡。

五、控制冒险（Control Hazard）

控制冒险由分支指令引起 — 在分支结果确定之前，处理器不知道下一条该取哪条指令。

5.1 分支代价

在五级流水线中，分支目标地址在 EX 阶段确定（若ALU计算分支目标），条件判断也在 EX 阶段完成。这意味着可能浪费 2 个周期。

5.2 解决方法

方法原理代价
尽早计算分支在ID阶段完成分支判断（增加硬件）减少到1个气泡
预测不跳转假设分支不执行，继续取后续指令预测错误时清空2条指令
预测跳转假设分支执行，预取目标地址的指令预测错误时清空
延迟槽分支后的指令无论是否跳转都执行需要编译器配合
动态分支预测根据历史记录预测（见下节）高准确率，少量硬件

六、流水线性能分析

6.1 CPI（Cycles Per Instruction）计算

理想CPI = 1.0 实际CPI = 1 + stall_per_instruction stall_per_instruction = 结构冒险stall比例 + 数据冒险stall比例 + 控制冒险stall比例 示例: - 分支指令占20%，预测错误率10%，代价2周期 - Load-Use冒险占15%，其中30%无法调度 - CPI = 1 + 0.2×0.1×2 + 0.15×0.3×1 = 1 + 0.04 + 0.045 = 1.085
