# 配置回滚

## 一、核心概念

配置回滚是指当配置变更导致系统异常时，能够快速恢复到之前的正常版本。配置回滚是配置变更安全的"最后一道防线"，要求操作简单、响应迅速。

### 1.1 需要回滚的场景

```
场景                    表现                  回滚紧迫性
──────────────────────────────────────────────────────
数据库连接串改错        服务无法启动           立即回滚
限流阈值改小            大量请求被拒绝         尽快回滚
缓存地址改错            响应变慢/超时          尽快回滚
业务参数改错            业务逻辑异常           评估后回滚
日志级别改错            无影响                 可暂不回滚
```

### 1.2 回滚流程

```
发现异常 → 确认是配置变更导致 → 选择回滚版本
  → 执行回滚 → 验证恢复 → 记录审计日志
```

## 二、基于版本的回滚

### 2.1 Apollo 配置回滚

```java
@RestController
@RequestMapping("/admin/config")
public class ConfigRollbackController {

    @Autowired
    private ApolloOpenApiClient apolloClient;

    /**
     * 获取配置历史版本
     */
    @GetMapping("/history")
    public List<ConfigCommitDTO> getHistory(
            @RequestParam String appId,
            @RequestParam String env,
            @RequestParam String clusterName,
            @RequestParam String namespaceName) {
        return apolloClient.getNamespaceCommitHistory(
            appId, env, clusterName, namespaceName,
            0, 50).getBody();
    }

    /**
     * 回滚到指定版本
     */
    @PostMapping("/rollback")
    public Result rollback(
            @RequestBody RollbackRequest request) {
        try {
            apolloClient.rollbackNamespace(
                request.getAppId(),
                request.getEnv(),
                request.getClusterName(),
                request.getNamespaceName(),
                request.getReleaseId()
            );

            auditLogService.log("CONFIG_ROLLBACK", request);
            return Result.ok("回滚成功");
        } catch (Exception e) {
            return Result.error("回滚失败: " + e.getMessage());
        }
    }
}
```

### 2.2 Nacos 配置回滚

```java
@Service
public class NacosConfigRollback {

    @Autowired
    private ConfigService configService;

    @Autowired
    private ConfigVersionService versionService;

    /**
     * 回滚到上一版本
     */
    public void rollback(String dataId, String group) {
        // 获取上一版本
        ConfigVersion previous =
            versionService.getPreviousVersion(
                dataId, group);

        if (previous == null) {
            throw new IllegalStateException(
                "没有可回滚的版本");
        }

        // 发布上一版本的配置
        try {
            configService.publishConfig(
                dataId, group, previous.getContent());

            log.info("回滚成功: {}/{} -> version {}",
                group, dataId, previous.getVersion());
        } catch (NacosException e) {
            throw new RuntimeException("回滚失败", e);
        }
    }

    /**
     * 回滚到指定版本
     */
    public void rollbackToVersion(String dataId,
                                   String group,
                                   int targetVersion) {
        ConfigVersion target =
            versionService.getVersion(
                dataId, group, targetVersion);

        if (target == null) {
            throw new IllegalArgumentException(
                "目标版本不存在: " + targetVersion);
        }

        configService.publishConfig(
            dataId, group, target.getContent());

        log.info("回滚到版本: {}/{} -> v{}",
            group, dataId, targetVersion);
    }
}
```

### 3.2 自动回滚

```java
@Component
public class AutoRollbackService {

    @Autowired
    private ConfigService configService;

    @Autowired
    private MetricsService metricsService;

    /**
     * 配置发布后自动监控，异常时自动回滚
     */
    @Async
    public void monitorAndAutoRollback(
            String dataId, String group,
            String previousContent) {
        // 监控 5 分钟
        for (int i = 0; i < 30; i++) {
            try {
                Thread.sleep(10000); // 每 10 秒检查

                double errorRate =
                    metricsService.getErrorRate(dataId);
                double latency =
                    metricsService.getP99Latency(dataId);

                if (errorRate > 0.05 || latency > 5000) {
                    log.error("检测到异常，自动回滚: " +
                        "errorRate={}, latency={}",
                        errorRate, latency);
                    configService.publishConfig(
                        dataId, group, previousContent);
                    alertService.send("AUTO_ROLLBACK",
                        "配置自动回滚: " + dataId);
                    return;
                }
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
                return;
            }
        }
        log.info("配置变更监控通过: {}", dataId);
    }
}
```

## 四、最佳实践

1. **配置变更前先备份**，确保有可回滚的版本
2. **回滚操作要审计**，记录谁在什么时候回滚了什么
3. **回滚后要验证服务状态**，确认恢复成功
4. **配置版本要保留足够历史**，建议至少 50 个版本
5. **回滚流程要简单快速**，最好支持一键回滚
6. **生产环境支持自动回滚**，监控异常自动触发
7. **定期演练回滚流程**，确保紧急时能快速操作
8. **回滚后分析原因**，防止同类问题再次发生
