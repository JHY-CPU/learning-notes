# 配置中心概述

## 一、为什么需要配置中心

```yaml
传统方式的问题:
  - 配置分散在各服务，难以管理
  - 修改配置需要重启服务
  - 多环境配置维护困难
  - 敏感信息泄露风险

配置中心解决:
  - 统一管理所有配置
  - 动态推送配置变更
  - 多环境隔离
  - 敏感信息加密
```

## 二、主流配置中心

| 组件 | 特点 |
|------|------|
| Nacos | 注册+配置一体，阿里开源 |
| Apollo | 携程开源，功能全面 |
| Consul | HashiCorp，KV 存储 |
| Spring Cloud Config | Git 存储，简单 |

## 三、注意事项

1. **Nacos 国内最流行**，注册+配置一体
2. **Apollo 功能最全面**，适合大型项目
3. **配置变更要实时推送**
4. **敏感配置要加密**
5. **多环境要严格隔离**
