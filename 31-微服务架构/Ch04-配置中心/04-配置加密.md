# 配置加密

## 一、加密方案

```java
// Jasypt 加密
@Configuration
public class EncryptionConfig {
    @Bean
    public StringEncryptor stringEncryptor() {
        PooledPooledStringEncryptor encryptor = new PooledPooledStringEncryptor();
        encryptor.setPassword("my-secret-key");
        return encryptor;
    }
}

// 配置文件中使用
spring:
  datasource:
    password: ENC(加密后的字符串)
```

## 二、工作原理

配置加密的核心思路是在配置存储或传输阶段对敏感字段进行加密，运行时由框架自动解密。Jasypt 方案中，加密密钥通过环境变量或启动参数传入（而非写在配置文件中），框架在加载配置时识别 `ENC(...)` 格式的值并自动解密。Nacos 自带的加密功能基于 AES 算法，在服务端存储时加密，客户端拉取后解密。KMS（密钥管理服务）方案将密钥托管在专用服务中，应用运行时从 KMS 获取密钥再解密配置。

## 三、优缺点

**优点：**
- 敏感信息不以明文形式存储，降低泄露风险
- Jasypt 等方案集成简单，对业务代码无侵入
- KMS 方案密钥集中管理，安全性最高

**缺点：**
- 密钥本身的安全管理是核心难题
- 加密解密增加微小的性能开销
- 调试和排错时加密字段不可读

## 四、最佳实践

1. 密钥通过环境变量或 KMS 注入，绝不写入配置文件或代码库
2. 优先使用 AES-256 算法，密钥长度至少 16 字符
3. 定期轮换密钥，但需注意滚动更新期间新旧密钥并存
4. 仅加密真正敏感的字段（密码、密钥、Token），不要全量加密

## 五、Spring Vault 集成

```yaml
# Spring Cloud Vault - 密钥集中管理
spring:
  cloud:
    vault:
      uri: https://vault.internal:8200
      authentication: TOKEN
      token: s.xxxxx
      kv:
        enabled: true
        backend: secret
        default-context: application
        profile-separator: '/'
```

```java
// 从 Vault 获取密钥
@Value("${secret.db.password}")
private String dbPassword;

@Value("${secret.api.key}")
private String apiKey;
```

## 六、Nacos 配置加密

```java
// Nacos 自带加密 - 配置插件
@Configuration
public class NacosEncryptConfig {
    @Bean
    public ConfigEncryptor configEncryptor() {
        return new AesEncryptor("your-encrypt-key".getBytes());
    }
}

// 使用方式: 配置值前缀 ENC(encrypted_value)
// Nacos 控制台支持一键加密
```

```yaml
# 加密配置示例
spring:
  datasource:
    password: ENC(U2FsdGVkX1+abc123...)
  redis:
    password: ENC(U2FsdGVkX1+def456...)
```

## 七、密钥轮换方案

```
密钥轮换流程:
1. 生成新密钥
2. 用新密钥重新加密所有敏感配置
3. 部署新密钥到所有服务（灰度发布）
4. 验证服务正常
5. 删除旧密钥

注意事项:
- 轮换期间新旧密钥并存
- 先验证再删除旧密钥
- 记录轮换日志
```

## 八、常见陷阱

1. **密钥提交到 Git 仓库** - 加密形同虚设，密钥必须通过环境变量注入
2. **Jasypt 密钥和加密字符串放同一文件** - 泄露后直接解密
3. **加密算法选择 DES 等弱算法** - 存在被破解风险，使用 AES-256
4. **忘记配置加密密钥** - 服务启动时解密失败
5. **密钥没有轮换机制** - 长期使用同一密钥增加泄露风险
6. **加密范围不当** - 全量加密增加复杂度，只加密敏感字段
