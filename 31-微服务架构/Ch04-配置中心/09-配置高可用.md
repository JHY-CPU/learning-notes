# 配置中心高可用

## 一、核心概念

配置中心是微服务架构的关键基础设施，一旦配置中心不可用，可能导致服务启动失败或无法获取最新配置。本节介绍配置中心的高可用架构、本地缓存降级、容灾方案等。

### 1.1 高可用目标

- **配置获取不中断**：配置中心故障时服务仍能正常运行
- **本地缓存兜底**：客户端缓存配置，注册中心故障时使用缓存
- **多级降级**：缓存 → 静态配置 → 默认值
- **快速恢复**：故障恢复后自动同步最新配置

### 3.2 本地缓存实现

```java
@Component
public class LocalConfigCache {

    private final String cacheDir = "/tmp/config-cache";
    private final Map<String, String> memoryCache =
        new ConcurrentHashMap<>();

    /**
     * 配置变更时更新缓存
     */
    public void updateCache(String dataId, String group,
                             String content) {
        String key = group + "/" + dataId;
        memoryCache.put(key, content);
        persistToDisk(key, content);
    }

    /**
     * 获取配置（优先内存，其次磁盘）
     */
    public String getConfig(String dataId, String group) {
        String key = group + "/" + dataId;

        // 1. 内存缓存
        String cached = memoryCache.get(key);
        if (cached != null) return cached;

        // 2. 磁盘缓存
        cached = readFromDisk(key);
        if (cached != null) {
            memoryCache.put(key, cached);
            return cached;
        }

        return null;
    }

    private void persistToDisk(String key, String content) {
        try {
            File file = new File(cacheDir,
                key.replace("/", "_") + ".cache");
            Files.writeString(file.toPath(), content);
        } catch (IOException e) {
            log.warn("缓存写入磁盘失败", e);
        }
    }
}
```

### 3.3 配置获取降级策略

```java
@Service
public class ResilientConfigService {

    @Autowired
    private ConfigService nacosConfigService;

    @Autowired
    private LocalConfigCache localCache;

    @Autowired
    private DefaultConfigProvider defaultConfig;

    /**
     * 多级降级获取配置
     */
    public String getConfig(String dataId, String group) {
        // Level 0: 从配置中心获取
        try {
            String config = nacosConfigService.getConfig(
                dataId, group, 5000);
            if (config != null) {
                // 更新本地缓存
                localCache.updateCache(
                    dataId, group, config);
                return config;
            }
        } catch (Exception e) {
            log.warn("配置中心不可用: {}", e.getMessage());
        }

        // Level 1: 从本地缓存获取
        String cached = localCache.getConfig(dataId, group);
        if (cached != null) {
            log.info("使用本地缓存配置: {}/{}", group, dataId);
            return cached;
        }

        // Level 2: 使用默认配置
        String defaultConf =
            defaultConfig.getDefault(dataId, group);
        if (defaultConf != null) {
            log.warn("使用默认配置: {}/{}", group, dataId);
            return defaultConf;
        }

        throw new ConfigNotFoundException(
            "配置不存在: " + group + "/" + dataId);
    }
}
```

## 四、备份与恢复

### 4.1 配置自动备份

```bash
#!/bin/bash
# 定期备份 Nacos 配置数据库
BACKUP_DIR="/data/backup/nacos"
DATE=$(date +%Y%m%d_%H%M%S)

mysqldump -h mysql-ha -u nacos -p${PASSWORD} \
    nacos_config \
    --single-transaction \
    --routines --triggers \
    > "${BACKUP_DIR}/nacos_config_${DATE}.sql"

# 保留最近 30 天的备份
find ${BACKUP_DIR} -name "*.sql" -mtime +30 -delete
```

### 4.2 配置恢复

```java
@Service
public class ConfigRecoveryService {

    @Autowired
    private ConfigService configService;

    /**
     * 从备份文件恢复配置
     */
    public void recoverFromBackup(File backupFile) {
        List<ConfigInfo> configs =
            parseBackupFile(backupFile);

        for (ConfigInfo config : configs) {
            try {
                configService.publishConfig(
                    config.getDataId(),
                    config.getGroup(),
                    config.getContent());
            } catch (Exception e) {
                log.error("恢复配置失败: {}/{}",
                    config.getGroup(), config.getDataId(), e);
            }
        }
    }
}
```

## 五、监控与告警

### 5.1 关键监控指标

| 指标 | 说明 | 告警阈值 |
|------|------|---------|
| 配置中心可用性 | 健康检查 | 不可用即告警 |
| 配置获取延迟 | P99 延迟 | > 3s |
| 本地缓存命中率 | 缓存有效性 | < 50% |
| 配置变更频率 | 变更热度 | 突增 > 200% |

## 六、最佳实践

1. **配置中心至少 3 节点**，使用集群部署
2. **客户端本地缓存配置**，配置中心故障时仍可工作
3. **多级降级策略**：配置中心 → 本地缓存 → 默认值
4. **定期备份配置数据**，MySQL 持久化 + 定期快照
5. **监控配置中心的健康状态**，配置多级告警
6. **配置变更灰度推送**，降低故障影响面
7. **设置合理的超时时间**，避免配置获取阻塞启动
