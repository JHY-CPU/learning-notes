# 配置规范

## 一、配置命名规范

```yaml
# 命名规范
格式: {服务名}-{环境}.{扩展名}
示例:
  - user-service-dev.yaml
  - order-service-prod.yaml
  - common.yaml (共享配置)
```

## 二、配置分类

| 类型 | 说明 | 示例 |
|------|------|------|
| 业务配置 | 功能开关、业务参数 | feature.enabled |
| 技术配置 | 数据源、缓存 | datasource.url |
| 环境配置 | 环境特定配置 | server.port |
| 敏感配置 | 密钥、密码 | password |

## 三、注意事项

1. **配置命名要规范化**
2. **敏感配置要加密**
3. **共享配置要提取**
4. **配置要有默认值**
5. **配置变更要有文档**
