# 配置版本管理

## 一、核心概念

配置版本管理是指对配置文件的每次变更进行版本记录，支持查看历史版本、对比差异、快速回滚。在微服务架构中，配置变更频繁且影响面广，良好的版本管理机制是保障系统稳定性的关键。

### 1.1 为什么需要配置版本管理

```
场景              风险              版本管理的作用
────────────────────────────────────────────────
配置误修改       服务不可用         快速回滚到上一版本
多人同时修改     配置冲突           追踪变更来源
配置丢失         无法恢复           完整历史记录
合规审计         无变更记录         完整审计日志
```

## 三、基于数据库的版本管理

### 3.1 版本表设计

```sql
-- 配置版本表
CREATE TABLE config_version (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    data_id VARCHAR(255) NOT NULL,
    group_name VARCHAR(128) NOT NULL,
    namespace VARCHAR(64) DEFAULT 'public',
    content TEXT NOT NULL,
    content_md5 VARCHAR(32) NOT NULL,
    version INT NOT NULL DEFAULT 1,
    change_type ENUM('CREATE','UPDATE','DELETE') NOT NULL,
    change_reason VARCHAR(500),
    created_by VARCHAR(64) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    approved_by VARCHAR(64),
    approved_at TIMESTAMP,
    INDEX idx_data_group (data_id, group_name),
    INDEX idx_version (data_id, group_name, version)
);

-- 触发器：配置变更时自动记录版本
DELIMITER //
CREATE TRIGGER config_version_trigger
AFTER UPDATE ON config_info
FOR EACH ROW
BEGIN
    INSERT INTO config_version
        (data_id, group_name, namespace, content,
         content_md5, version, change_type, created_by)
    VALUES
        (NEW.data_id, NEW.group_name, NEW.namespace,
         NEW.content, MD5(NEW.content),
         (SELECT COALESCE(MAX(version), 0) + 1
          FROM config_version
          WHERE data_id = NEW.data_id
          AND group_name = NEW.group_name),
         'UPDATE', CURRENT_USER());
END //
DELIMITER ;
```

### 3.2 版本管理服务

```java
@Service
public class ConfigVersionService {

    @Autowired
    private ConfigVersionMapper versionMapper;

    /**
     * 保存配置版本
     */
    public void saveVersion(ConfigInfo config,
                            String reason, String operator) {
        ConfigVersion version = new ConfigVersion();
        version.setDataId(config.getDataId());
        version.setGroupName(config.getGroup());
        version.setContent(config.getContent());
        version.setContentMd5(
            DigestUtils.md5Hex(config.getContent()));
        version.setVersion(
            versionMapper.getNextVersion(
                config.getDataId(), config.getGroup()));
        version.setChangeReason(reason);
        version.setCreatedBy(operator);
        versionMapper.insert(version);
    }

    /**
     * 获取版本历史
     */
    public List<ConfigVersion> getHistory(String dataId,
                                           String group) {
        return versionMapper.selectByVersion(dataId, group);
    }

    /**
     * 对比两个版本的差异
     */
    public String diff(String dataId, String group,
                       int v1, int v2) {
        ConfigVersion version1 =
            versionMapper.selectByVersion(dataId, group, v1);
        ConfigVersion version2 =
            versionMapper.selectByVersion(dataId, group, v2);
        return computeDiff(version1.getContent(),
            version2.getContent());
    }
}
```

## 四、版本对比与审计

### 4.1 配置差异对比

```java
@Service
public class ConfigDiffService {

    public String computeDiff(String oldContent,
                               String newContent) {
        List<String> oldLines =
            Arrays.asList(oldContent.split("\n"));
        List<String> newLines =
            Arrays.asList(newContent.split("\n"));

        // 使用 Java Diff Utils 库
        Patch<String> patch =
            DiffUtils.diff(oldLines, newLines);

        StringBuilder sb = new StringBuilder();
        for (Delta<String> delta : patch.getDeltas()) {
            sb.append("[").append(delta.getType())
              .append("] Line ")
              .append(delta.getSource().getPosition())
              .append("\n");
            sb.append("- ").append(delta.getSource().getLines())
              .append("\n");
            sb.append("+ ").append(delta.getTarget().getLines())
              .append("\n\n");
        }
        return sb.toString();
    }
}
```

## 五、最佳实践

1. **配置变更要有记录**，每次修改都保存版本
2. **支持配置回滚**，能快速恢复到任意历史版本
3. **配置修改需要审批**，生产环境变更走审批流程
4. **敏感配置不要提交到 Git**，使用加密存储
5. **配置变更要通知相关团队**，通过消息或邮件通知
6. **保留足够的历史版本**，建议至少保留 50 个版本
7. **版本变更需要填写变更原因**，便于事后追溯
