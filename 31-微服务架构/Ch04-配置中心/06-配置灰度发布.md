# 配置灰度发布

## 一、核心概念

配置灰度发布是指将配置变更先推送到部分实例进行验证，确认无误后再全量推送。通过灰度发布，可以最大程度降低配置变更带来的风险。

### 1.1 灰度发布流程

```
全量发布（高风险）:
  修改配置 → 推送所有实例 → 观察

灰度发布（低风险）:
  修改配置 → 灰度实例验证 → 观察指标 → 正常？
    ├─ 是 → 扩大灰度范围 → 全量发布
    └─ 否 → 快速回滚灰度实例
```

### 1.2 灰度维度

```
灰度方式          粒度    适用场景
─────────────────────────────────
按 IP 灰度       实例级   精确控制
按标签灰度        分组级   按区域/版本
按比例灰度        随机     大规模验证
按集群灰度        集群级   多机房部署
```

## 二、基于 Nacos 的灰度发布

### 2.1 Nacos 灰度配置

```yaml
# Nacos 控制台操作流程:
# 1. 创建配置
# 2. 点击"灰度发布"
# 3. 添加灰度 IP 列表
# 4. 发布灰度版本
# 5. 验证通过后全量发布
```

```java
// Nacos API 灰度发布
public class NacosGrayPublish {

    public void publishGray(String dataId, String group,
                             String content,
                             List<String> betaIps) {
        // 设置灰度 IP 列表
        Map<String, String> betaIpsMap = new HashMap<>();
        betaIpsMap.put("betaIps",
            String.join(",", betaIps));

        // 发布灰度配置
        configService.publishConfig(
            dataId, group, content,
            ConfigType.YAML.getType(), betaIpsMap);
    }

    public void publishFull(String dataId, String group,
                             String content) {
        // 全量发布（移除灰度标记）
        configService.publishConfig(
            dataId, group, content);
    }

    public void rollbackGray(String dataId, String group) {
        // 回滚灰度配置
        configService.removeConfig(dataId, group);
    }
}
```

### 2.2 灰度发布控制器

```java
@RestController
@RequestMapping("/admin/config/gray")
public class ConfigGrayController {

    @Autowired
    private NacosConfigService configService;

    @PostMapping("/publish")
    public Result publishGray(
            @RequestBody GrayPublishRequest request) {
        // 参数校验
        if (request.getBetaIps().isEmpty()) {
            return Result.error("灰度 IP 列表不能为空");
        }

        // 发布灰度配置
        configService.publishGray(
            request.getDataId(),
            request.getGroup(),
            request.getContent(),
            request.getBetaIps()
        );

        // 记录审计日志
        auditLogService.log("GRAY_PUBLISH", request);

        return Result.ok("灰度发布成功");
    }

    @PostMapping("/full-publish")
    public Result fullPublish(
            @RequestBody FullPublishRequest request) {
        // 验证灰度期间无异常
        if (hasAbnormalMetrics(request.getDataId())) {
            return Result.error("灰度期间发现异常，请先排查");
        }

        configService.publishFull(
            request.getDataId(),
            request.getGroup(),
            request.getContent()
        );

        return Result.ok("全量发布成功");
    }
}
```

## 三、基于 Spring Cloud 的灰度发布

### 3.1 灰度路由策略

```java
@Component
public class GrayRouteFilter implements GlobalFilter {

    @Override
    public Mono<Void> filter(ServerWebExchange exchange,
                              GatewayFilterChain chain) {
        String grayTag = exchange.getRequest().getHeaders()
            .getFirst("X-Gray-Tag");

        if (grayTag != null) {
            // 添加灰度标记到请求头
            exchange.getRequest().mutate()
                .header("X-Gray-Tag", grayTag);
        }

        return chain.filter(exchange);
    }
}
```

### 3.2 灰度规则配置

## 四、灰度验证与监控

### 4.1 自动化验证

```java
@Component
public class GrayPublishValidator {

    /**
     * 灰度发布后自动验证
     */
    @Async
    public CompletableFuture<ValidationResult> validate(
            String dataId, List<String> betaIps) {
        ValidationResult result = new ValidationResult();

        for (String ip : betaIps) {
            // 检查实例健康状态
            boolean healthy = checkHealth(ip);
            // 检查错误率
            double errorRate = getErrorRate(ip);
            // 检查响应延迟
            double avgLatency = getAvgLatency(ip);

            if (!healthy || errorRate > 0.01
                    || avgLatency > 2000) {
                result.addFailed(ip,
                    "健康:" + healthy
                    + " 错误率:" + errorRate
                    + " 延迟:" + avgLatency);
            }
        }

        return CompletableFuture.completedFuture(result);
    }
}
```

## 五、最佳实践

1. **灰度配置降低风险**，先灰度部分实例再全量
2. **灰度期间做好监控**，重点关注错误率和延迟
3. **异常时快速回滚**，准备好回滚方案
4. **灰度验证要自动化**，减少人工判断的误差
5. **生产环境配置变更必须灰度**，禁止直接全量推送
6. **灰度比例逐步扩大**：1% → 10% → 50% → 100%
7. **设置灰度超时时间**，超时自动回滚
