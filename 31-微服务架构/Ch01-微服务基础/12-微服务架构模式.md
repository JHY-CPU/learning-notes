# 微服务架构模式

## 一、Saga 模式

```java
// 编排式 Saga - 通过事件驱动
public class OrderSaga {
    // 正向流程
    public void execute(Order order) {
        inventoryService.reserve(order);  // Step 1
        accountService.deduct(order);     // Step 2
        orderService.create(order);       // Step 3
    }

    // 补偿流程
    public void compensate(Order order, int failedStep) {
        switch (failedStep) {
            case 3: accountService.refund(order);
            case 2: inventoryService.release(order);
        }
    }
}
```

## 二、CQRS 模式

```
Command (写) → 写模型 → 事件 → 读模型 → Query (读)
```

## 三、Event Sourcing

```java
// 存储事件而非状态
public class OrderAggregate {
    public void apply(OrderCreatedEvent event) { }
    public void apply(OrderPaidEvent event) { }
    public void apply(OrderShippedEvent event) { }
}
```

## 四、模式选择

| 模式 | 适用场景 |
|------|---------|
| Saga | 分布式事务 |
| CQRS | 读写分离 |
| Event Sourcing | 审计、回溯 |
| API Gateway | 统一入口 |

## 五、注意事项

1. **模式不是银弹**，根据场景选择
2. **Saga 适合长时间事务**
3. **CQRS 增加复杂度**，适合读多写少
4. **Event Sourcing 适合需要审计的场景**
5. **组合使用多种模式**，不要只用一种
