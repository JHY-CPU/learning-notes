# 微服务演进路径

## 一、演进路线

```
单体应用 → 模块化单体 → 服务化 → 微服务 → 云原生

阶段1: 单体架构（小团队）
  └── 所有功能在一个应用中

阶段2: 模块化单体
  └── 按业务模块组织代码，共享数据库

阶段3: 服务化
  └── 核心模块拆为独立服务，独立数据库

阶段4: 微服务
  └── 全面拆分，API Gateway + 服务网格

阶段5: 云原生
  └── Kubernetes + Service Mesh + Serverless
```

## 二、绞杀者模式

```java
// 逐步替换单体，不一次性重写
// 1. 新功能用新服务实现
// 2. 旧功能逐步迁移到新服务
// 3. 路由层将请求分配到新旧服务
// 4. 迁移完成后下线单体
```

## 三、注意事项

1. **不要一步到位**，渐进式演进
2. **先解决痛点**，优先拆分变化最频繁的模块
3. **保持旧系统可用**，新旧并行运行
4. **数据迁移是最大的挑战**
5. **演进过程中要做好监控**
