# 微服务数据管理

## 一、数据库拆分

```java
// 每个服务拥有独立数据库
// 用户服务 -> MySQL (用户库)
// 订单服务 -> MySQL (订单库)
// 商品服务 -> MongoDB (商品库)
// 缓存服务 -> Redis

// 服务间通过 API 查询数据，不跨库查询
```

## 二、数据一致性

```java
// 最终一致性方案
// 本地消息表 + MQ
@Transactional
public void createOrder(Order order) {
    orderMapper.insert(order);
    messageMapper.insert(new Message("order.created", order));
}

// Saga 模式
// 正向: 扣库存 -> 扣余额 -> 创建订单
// 补偿: 恢复余额 -> 恢复库存
```

## 三、CQRS 模式

```java
// 写模型 - 关系型数据库
// 读模型 - Elasticsearch/Redis

// 写操作
orderMapper.insert(order);
kafkaTemplate.send("order-events", order);

// 读操作
elasticsearchTemplate.search(query, Order.class);
```

## 四、注意事项

1. **数据库拆分是微服务最大的挑战**
2. **避免分布式事务**，优先用最终一致性
3. **数据所有权要明确**，一个数据只有一个服务拥有
4. **跨服务查询用 API 组合或数据同步**
5. **做好数据备份和恢复方案**
