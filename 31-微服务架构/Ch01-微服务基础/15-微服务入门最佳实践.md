# 微服务入门最佳实践

## 一、入门路径

```
Step 1: 学习基础
  - Spring Boot
  - Docker 基础
  - REST API 设计

Step 2: 核心组件
  - 服务注册发现（Nacos）
  - 配置中心（Nacos）
  - API 网关（Spring Cloud Gateway）
  - 服务调用（OpenFeign）

Step 3: 高级特性
  - 熔断降级（Sentinel）
  - 链路追踪（SkyWalking）
  - 分布式事务（Seata）
  - 消息队列（RocketMQ/Kafka）

Step 4: 运维体系
  - 容器编排（Kubernetes）
  - 监控告警（Prometheus + Grafana）
  - 日志收集（ELK）
  - CI/CD（Jenkins/GitLab CI）
```

## 三、Docker Compose 本地环境

```yaml
# docker-compose.yml - 完整本地开发环境
version: '3.8'
services:
  nacos:
    image: nacos/nacos-server:v2.3.0
    ports:
      - "8848:8848"
    environment:
      - MODE=standalone
      - SPRING_DATASOURCE_PLATFORM=derby

  mysql:
    image: mysql:8.0
    ports:
      - "3306:3306"
    environment:
      - MYSQL_ROOT_PASSWORD=root123
    volumes:
      - ./sql:/docker-entrypoint-initdb.d

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"

  user-service:
    build: ./user-service
    ports:
      - "8081:8080"
    depends_on:
      - nacos
      - mysql
    environment:
      - SPRING_PROFILES_ACTIVE=dev

  order-service:
    build: ./order-service
    ports:
      - "8082:8080"
    depends_on:
      - nacos
      - mysql
    environment:
      - SPRING_PROFILES_ACTIVE=dev

  api-gateway:
    build: ./api-gateway
    ports:
      - "8080:8080"
    depends_on:
      - nacos
```

## 五、常见新手错误

```yaml
错误1: 一步到位
  ✗ 第一个项目就拆 20 个微服务
  ✓ 先拆 2-3 个核心服务

错误2: 忽略基础设施
  ✗ 没有 CI/CD、监控就开始拆微服务
  ✓ 先搭建基础设施，再拆服务

错误3: 技术选型跟风
  ✗ 看到新框架就引入
  ✓ 选择成熟稳定、社区活跃的方案

错误4: 过度设计
  ✗ 每个服务都配熔断、限流、链路追踪
  ✓ 核心服务重点保障，非核心服务简单处理

错误5: 忽视数据一致性
  ✗ 拆服务后直接跨库 JOIN
  ✓ 设计好数据同步和一致性方案
```

## 六、注意事项

1. **先掌握 Spring Boot** - 微服务的基础是 Spring Boot，不要跳步
2. **从小项目练手** - 用个人项目或公司非核心系统实践
3. **本地环境用 Docker Compose** - 生产用 Kubernetes，不要混用
4. **做好监控和日志** - 没有监控就是在裸奔，第一天就要有
5. **团队学习曲线要预估** - 微服务学习成本不低，至少 3-6 个月
6. **建立知识库** - 把踩过的坑记录下来，帮助团队成长
