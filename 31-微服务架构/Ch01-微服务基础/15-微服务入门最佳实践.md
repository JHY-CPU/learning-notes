# 微服务入门最佳实践

## 一、入门路径

```
Step 1: 学习基础
  - Spring Boot
  - Docker 基础
  - REST API 设计

Step 2: 核心组件
  - 服务注册发现（Nacos）
  - 配置中心（Nacos）
  - API 网关（Spring Cloud Gateway）
  - 服务调用（OpenFeign）

Step 3: 高级特性
  - 熔断降级（Sentinel）
  - 链路追踪（SkyWalking）
  - 分布式事务（Seata）
  - 消息队列（RocketMQ/Kafka）

Step 4: 运维体系
  - 容器编排（Kubernetes）
  - 监控告警（Prometheus + Grafana）
  - 日志收集（ELK）
  - CI/CD（Jenkins/GitLab CI）
```

## 二、项目结构

```
project/
├── api-gateway/              # API 网关
├── user-service/             # 用户服务
│   ├── src/main/java/
│   ├── Dockerfile
│   └── pom.xml
├── order-service/            # 订单服务
├── common/                   # 公共模块
├── docker-compose.yml
└── pom.xml
```

## 三、注意事项

1. **先掌握 Spring Boot**，再学微服务
2. **从小项目练手**，不要直接上生产
3. **本地环境用 Docker Compose**，生产用 Kubernetes
4. **做好监控和日志**，没有监控就是在裸奔
5. **团队学习曲线要预估**，微服务学习成本不低
