# 微服务安全

## 一、认证与授权

```java
// OAuth2 + JWT
@Configuration
@EnableResourceServer
public class ResourceServerConfig extends ResourceServerConfigurerAdapter {
    @Override
    public void configure(HttpSecurity http) throws Exception {
        http.authorizeRequests()
            .antMatchers("/api/public/**").permitAll()
            .antMatchers("/api/admin/**").hasRole("ADMIN")
            .anyRequest().authenticated();
    }
}

// JWT Token 解析
@Component
public class JwtTokenProvider {
    public Authentication getAuthentication(String token) {
        Claims claims = Jwts.parserBuilder()
            .setSigningKey(secretKey)
            .build()
            .parseClaimsJws(token)
            .getBody();
        return new UsernamePasswordAuthenticationToken(claims.getSubject(), null, authorities);
    }
}
```

## 四、OAuth2 完整流程

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 客户端   │ → │ API 网关  │ → │ 认证服务  │ → │ 资源服务  │
│          │    │ 校验JWT  │    │ 颁发Token │    │ 业务逻辑  │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
     │               │               │
     │  1. 登录请求   │               │
     ├──────────────→│  2. 转发      │
     │               ├──────────────→│
     │               │  3. 返回JWT   │
     │  4. JWT Token │←──────────────┤
     │←──────────────┤               │
     │               │               │
     │  5. 携带Token │               │
     │  请求业务API   │  6. 验证JWT  │
     ├──────────────→├──────────────→│
```

```java
// Spring Security OAuth2 配置
@Configuration
@EnableWebSecurity
public class SecurityConfig {
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/public/**").permitAll()
                .requestMatchers("/api/admin/**").hasRole("ADMIN")
                .anyRequest().authenticated()
            )
            .oauth2ResourceServer(oauth2 -> oauth2
                .jwt(jwt -> jwt.decoder(jwtDecoder()))
            );
        return http.build();
    }

    @Bean
    public JwtDecoder jwtDecoder() {
        return NimbusJwtDecoder.withJwkSetUri("http://auth-server/jwks").build();
    }
}
```

## 五、API 安全防护

```java
// 限流配置（防止 DDoS）
@Configuration
public class RateLimitConfig {
    @Bean
    public RateLimiter rateLimiter() {
        return RateLimiter.builder()
            .count(100)           // 每分钟 100 次
            .refreshInterval(1, TimeUnit.MINUTES)
            .build();
    }
}

// 参数校验（防注入）
public class CreateOrderRequest {
    @NotNull(message = "商品ID不能为空")
    @Min(1) @Max(999999)
    private Long productId;

    @NotBlank @Size(max = 200)
    @Pattern(regexp = "^[\\w\\s,.-]+$")  // 防止 XSS
    private String address;
}
```

## 七、安全检查清单

| 检查项 | 说明 | 优先级 |
|--------|------|--------|
| HTTPS 全站 | 所有外部通信加密 | 必须 |
| JWT 过期时间 | Access Token 15min, Refresh Token 7d | 必须 |
| CORS 配置 | 只允许信任域名 | 必须 |
| SQL 注入防护 | 参数化查询/ORM | 必须 |
| XSS 防护 | 输出编码 + CSP | 必须 |
| CSRF 防护 | Token 验证 | 高 |
| 日志脱敏 | 密码/身份证号不记录 | 高 |
| 漏洞扫描 | 定期更新依赖 | 中 |

## 八、注意事项

1. **Token 要设置合理的过期时间** - Access Token 15 分钟，Refresh Token 7 天
2. **敏感数据要加密存储** - 数据库密码、用户隐私信息使用 AES/RSA 加密
3. **服务间调用也要认证** - 内部服务用 mTLS 或 Service Token
4. **API 网关统一安全策略** - 限流、认证、WAF 在网关统一处理
5. **定期轮换密钥** - 密钥生命周期不超过 90 天
6. **最小权限原则** - 每个服务只授予完成工作所需的最小权限
