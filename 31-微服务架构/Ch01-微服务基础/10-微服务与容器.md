# 微服务与容器

## 一、Docker + 微服务

```dockerfile
# Dockerfile
FROM openjdk:17-jdk-slim
COPY target/user-service.jar /app.jar
ENTRYPOINT ["java", "-jar", "/app.jar"]
EXPOSE 8080
```

```yaml
# docker-compose.yml
version: '3.8'
services:
  user-service:
    build: ./user-service
    ports: ["8081:8080"]
    environment:
      - SPRING_PROFILES_ACTIVE=prod

  order-service:
    build: ./order-service
    ports: ["8082:8080"]
```

## 二、Kubernetes 编排

```yaml
# deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: user-service
spec:
  replicas: 3
  selector:
    matchLabels:
      app: user-service
  template:
    spec:
      containers:
      - name: user-service
        image: user-service:1.0
        ports:
        - containerPort: 8080
        resources:
          limits:
            memory: "512Mi"
            cpu: "500m"
```

## 三、多阶段构建优化

```dockerfile
# 多阶段构建 - 减小镜像体积
FROM maven:3.9-eclipse-temurin-17 AS builder
WORKDIR /app
COPY pom.xml .
RUN mvn dependency:go-offline -B
COPY src ./src
RUN mvn package -DskipTests -B

FROM eclipse-temurin:17-jre-alpine
WORKDIR /app
COPY --from=builder /app/target/*.jar app.jar
RUN addgroup -S appgroup && adduser -S appuser -G appgroup
USER appuser
EXPOSE 8080
HEALTHCHECK --interval=30s --timeout=3s \
    CMD wget -qO- http://localhost:8080/actuator/health || exit 1
ENTRYPOINT ["java", "-XX:+UseContainerSupport", "-XX:MaxRAMPercentage=75.0", "-jar", "app.jar"]
```

## 四、容器网络与服务发现

```
Kubernetes 网络模型:
┌─────────────────────────────────────┐
│  Pod ←→ Pod (通过 ClusterIP)       │
│  Pod ←→ Service (DNS 解析)          │
│  Service ←→ Ingress (外部访问)      │
└─────────────────────────────────────┘

Service 类型:
├── ClusterIP    - 集群内部访问（默认）
├── NodePort     - 节点端口映射
├── LoadBalancer - 云厂商负载均衡
└── ExternalName - 外部 DNS 映射
```

```yaml
# Service 定义
apiVersion: v1
kind: Service
metadata:
  name: user-service
spec:
  selector:
    app: user-service
  ports:
  - port: 80
    targetPort: 8080
  type: ClusterIP
---
# HPA 自动伸缩
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: user-service-hpa
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: user-service
  minReplicas: 2
  maxReplicas: 10
  metrics:
  - type: Resource
    resource:
      name: cpu
      target:
        type: Utilization
        averageUtilization: 70
```

## 五、容器安全最佳实践

| 实践 | 说明 |
|------|------|
| 非 root 运行 | 使用 USER 指令切换用户 |
| 只读文件系统 | readOnlyRootFileSystem: true |
| 最小基础镜像 | 使用 Alpine 或 distroless |
| 镜像扫描 | Trivy / Snyk 定期扫描 |
| 网络策略 | 限制 Pod 间通信 |
| 资源配额 | 设置 CPU/内存 limits |

## 六、注意事项

1. **容器化是微服务的天然伴侣** - 一个服务一个容器，环境一致性
2. **镜像要尽可能小** - 多阶段构建 + Alpine 基础镜像，控制在 200MB 以内
3. **健康检查必须配置** - liveness + readiness 探针确保服务可用
4. **资源限制要设置** - requests 和 limits 防止单个 Pod 挤占资源
5. **日志输出到 stdout/stderr** - 由 K8s / Fluentd 收集，不写文件
6. **容器不要存储状态** - 有状态数据用 PVC 或外部存储
