# 微服务与容器

## 一、Docker + 微服务

```dockerfile
# Dockerfile
FROM openjdk:17-jdk-slim
COPY target/user-service.jar /app.jar
ENTRYPOINT ["java", "-jar", "/app.jar"]
EXPOSE 8080
```

```yaml
# docker-compose.yml
version: '3.8'
services:
  user-service:
    build: ./user-service
    ports: ["8081:8080"]
    environment:
      - SPRING_PROFILES_ACTIVE=prod

  order-service:
    build: ./order-service
    ports: ["8082:8080"]
```

## 二、Kubernetes 编排

```yaml
# deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: user-service
spec:
  replicas: 3
  selector:
    matchLabels:
      app: user-service
  template:
    spec:
      containers:
      - name: user-service
        image: user-service:1.0
        ports:
        - containerPort: 8080
        resources:
          limits:
            memory: "512Mi"
            cpu: "500m"
```

## 三、注意事项

1. **容器化是微服务的天然伴侣**
2. **镜像要尽可能小**，减少攻击面和拉取时间
3. **健康检查必须配置**
4. **资源限制要设置**，避免单个容器占用过多资源
5. **日志输出到 stdout/stderr**，由 K8s 收集
