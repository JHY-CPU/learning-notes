# 微服务测试概述

## 一、测试金字塔

```
          /\
         /  \  E2E 测试 (少量)
        /────\
       /      \  集成测试 (适量)
      /────────\
     /          \  单元测试 (大量)
    /──────────────\
```

## 二、微服务测试分类

```yaml
测试层次:
  单元测试:
    - 范围: 单个类/方法
    - 速度: 毫秒级
    - 工具: JUnit5, Mockito
    - 比例: 70%

  集成测试:
    - 范围: 服务内部模块
    - 速度: 秒级
    - 工具: Testcontainers, @SpringBootTest
    - 比例: 20%

  契约测试:
    - 范围: 服务间接口
    - 工具: Pact, Spring Cloud Contract

  E2E 测试:
    - 范围: 完整业务流程
    - 速度: 分钟级
    - 工具: Cucumber, Selenium
    - 比例: 10%
```

## 三、微服务测试挑战

| 挑战 | 说明 | 应对方案 |
|------|------|---------|
| 环境依赖 | 多服务、多数据库 | Testcontainers |
| 网络调用 | 服务间依赖 | Mock/Stub |
| 数据一致性 | 分布式事务 | 契约测试 |
| 部署复杂 | 多个服务 | CI/CD 流水线 |
| 日志追踪 | 跨服务调用 | 链路追踪集成 |

## 四、注意事项

1. **测试金字塔是核心指导原则**，底层测试要多
2. **单元测试不依赖外部资源**，运行要快
3. **集成测试用容器化中间件**，保证环境一致性
4. **契约测试保证服务间接口兼容**
5. **测试数据管理要规范化**，避免测试间相互影响
