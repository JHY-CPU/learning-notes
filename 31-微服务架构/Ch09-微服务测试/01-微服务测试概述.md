# 微服务测试概述

## 一、测试金字塔

```
          /\
         /  \  E2E 测试 (少量)
        /────\
       /      \  集成测试 (适量)
      /────────\
     /          \  单元测试 (大量)
    /──────────────\
```

## 二、微服务测试分类

```yaml
测试层次:
  单元测试:
    - 范围: 单个类/方法
    - 速度: 毫秒级
    - 工具: JUnit5, Mockito
    - 比例: 70%

  集成测试:
    - 范围: 服务内部模块
    - 速度: 秒级
    - 工具: Testcontainers, @SpringBootTest
    - 比例: 20%

  契约测试:
    - 范围: 服务间接口
    - 工具: Pact, Spring Cloud Contract

  E2E 测试:
    - 范围: 完整业务流程
    - 速度: 分钟级
    - 工具: Cucumber, Selenium
    - 比例: 10%
```

## 三、微服务测试挑战

| 挑战 | 说明 | 应对方案 |
|------|------|---------|
| 环境依赖 | 多服务、多数据库 | Testcontainers |
| 网络调用 | 服务间依赖 | Mock/Stub |
| 数据一致性 | 分布式事务 | 契约测试 |
| 部署复杂 | 多个服务 | CI/CD 流水线 |
| 日志追踪 | 跨服务调用 | 链路追踪集成 |

## 四、契约测试详解

```java
// Spring Cloud Contract - 消费者驱动契约
// Provider 端定义契约
// src/test/resources/contracts/shouldReturnOrder.groovy
Contract.make {
    description "should return order by id"
    request {
        method GET()
        urlPath '/api/orders/1'
    }
    response {
        status 200
        body([
            id: 1,
            status: "CREATED",
            amount: 99.99
        ])
        headers {
            contentType('application/json')
        }
    }
}

// Consumer 端自动生成测试
@SpringBootTest(webEnvironment = SpringBootTest.WebEnvironment.MOCK)
@AutoConfigureStubRunner(stubsMode = StubRunnerProperties.StubsMode.LOCAL,
    ids = "com.example:order-service:+:stubs:8080")
public class OrderContractTest {
    @Autowired
    private OrderClient orderClient;

    @Test
    public void shouldReturnOrderById() {
        Order order = orderClient.getOrder(1L);
        assertThat(order.getId()).isEqualTo(1L);
        assertThat(order.getStatus()).isEqualTo("CREATED");
    }
}
```

## 五、Testcontainers 集成测试

```java
@SpringBootTest
@Testcontainers
public class OrderIntegrationTest {
    @Container
    static MySQLContainer<?> mysql = new MySQLContainer<>("mysql:8.0")
        .withDatabaseName("order_db")
        .withInitScript("schema.sql");

    @Container
    static GenericContainer<?> redis = new GenericContainer<>("redis:7-alpine")
        .withExposedPorts(6379);

    @DynamicPropertySource
    static void properties(DynamicPropertyRegistry registry) {
        registry.add("spring.datasource.url", mysql::getJdbcUrl);
        registry.add("spring.datasource.username", mysql::getUsername);
        registry.add("spring.datasource.password", mysql::getPassword);
        registry.add("spring.redis.host", redis::getHost);
        registry.add("spring.redis.port", () -> redis.getMappedPort(6379));
    }

    @Test
    public void testCreateOrder() {
        OrderRequest request = new OrderRequest(1L, 2);
        Order order = orderService.createOrder(request);
        assertThat(order.getStatus()).isEqualTo(OrderStatus.CREATED);
    }
}
```

## 六、注意事项

1. **测试金字塔是核心指导原则** - 底层测试要多，E2E 测试要少
2. **单元测试不依赖外部资源** - 运行要快，毫秒级
3. **集成测试用容器化中间件** - Testcontainers 保证环境一致性
4. **契约测试保证服务间接口兼容** - 消费者驱动，CI 集成
5. **测试数据管理要规范化** - 避免测试间相互影响
6. **微服务测试自动化** - CI 流水线集成所有测试层次
