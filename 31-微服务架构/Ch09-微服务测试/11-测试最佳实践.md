# 测试最佳实践

## 一、FIRST 原则

```yaml
FIRST:
  Fast:        # 测试要快
    - 单元测试毫秒级
    - 并行执行
  Independent: # 测试独立
    - 不依赖执行顺序
    - 数据隔离
  Repeatable:  # 可重复
    - 任何环境结果一致
    - 不依赖外部状态
  Self-Validating: # 自验证
    - 有明确断言
    - 不需人工判断
  Timely:      # 及时
    - 与代码同步编写
    - TDD 模式
```

## 二、测试命名规范

```java
// 推荐: should_期望行为_when_前置条件
@Test
void shouldReturnOrder_whenOrderIdExists() { }

@Test
void shouldThrowException_whenOrderNotFound() { }

// 推荐: 方法_场景_预期
@Test
void createOrder_withValidRequest_shouldSucceed() { }

// 不推荐: test1, testMethod
@Test
void test1() { } // 不好
```

## 三、测试结构 AAA

```java
@Test
void shouldDeductInventoryWhenOrderPaid() {
    // Arrange (准备)
    Order order = anOrder().withStatus(OrderStatus.CREATED).build();
    inventoryService.init("P001", 10);

    // Act (执行)
    order.pay();
    orderService.handlePaid(order);

    // Assert (断言)
    assertThat(inventoryService.getStock("P001")).isEqualTo(8);
    verify(eventPublisher).publish(any(OrderPaidEvent.class));
}
```

## 四、注意事项

1. **测试代码也是代码**，需要维护和重构
2. **避免测试中的逻辑分支**（循环、条件判断）
3. **每个测试只断言一个关注点**
4. **使用 @Disabled 而非删除过时测试**
5. **定期清理无用测试和过时 Mock**
