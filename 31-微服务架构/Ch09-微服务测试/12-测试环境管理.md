# 测试环境管理

## 一、环境分类

```yaml
测试环境:
  本地环境:
    - 用途: 开发自测
    - 数据: 内存/H2
    - 服务: 本机启动

  集成环境:
    - 用途: 联调测试
    - 数据: 独立数据库
    - 服务: Docker Compose

  预发布环境:
    - 用途: 准生产验证
    - 数据: 脱敏数据
    - 服务: K8s 集群

  沙箱环境:
    - 用途: 性能/混沌测试
    - 数据: 独立集群
    - 服务: 与生产同构
```

## 二、Docker Compose 测试环境

```yaml
# docker-compose.test.yml
version: '3.8'
services:
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: test
      MYSQL_DATABASE: order_db
    ports:
      - "3306:3306"
    healthcheck:
      test: ["CMD", "mysqladmin", "ping"]
      interval: 10s

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"

  rabbitmq:
    image: rabbitmq:3-management
    ports:
      - "5672:5672"
      - "15672:15672"

  nacos:
    image: nacos/nacos-server:v2.3.0
    environment:
      MODE: standalone
    ports:
      - "8848:8848"
```

## 四、K8s 测试环境

```yaml
# Namespace 隔离测试环境
apiVersion: v1
kind: Namespace
metadata:
  name: test-env-pr-123    # 每个 PR 独立环境
  labels:
    environment: test
    pr-number: "123"
    auto-cleanup: "true"   # 自动清理标签
---
# CI 自动创建测试环境
# .gitlab-ci.yml
deploy_test:
  stage: deploy
  script:
    - helm upgrade --install test-${CI_MERGE_REQUEST_IID} ./charts/order-service
      --namespace test-env-${CI_MERGE_REQUEST_IID}
      --create-namespace
      --set image.tag=${CI_COMMIT_SHA}
      --wait --timeout 300s
  environment:
    name: test/pr-${CI_MERGE_REQUEST_IID}
    on_stop: cleanup_test
  only:
    - merge_requests

cleanup_test:
  stage: cleanup
  script:
    - helm uninstall test-${CI_MERGE_REQUEST_IID}
      --namespace test-env-${CI_MERGE_REQUEST_IID}
    - kubectl delete namespace test-env-${CI_MERGE_REQUEST_IID}
  when: manual
```

## 五、测试数据管理策略

```yaml
测试数据策略:
  单元测试:
    - 方式: 内存数据库 (H2)
    - 清理: 每个测试自动回滚 (@Transactional)

  集成测试:
    - 方式: Testcontainers (MySQL/Redis)
    - 清理: 测试结束后销毁容器

  E2E 测试:
    - 方式: 独立数据库 + 脱敏数据
    - 清理: 每次 E2E 测试前重置

  性能测试:
    - 方式: 独立集群 + 大量预置数据
    - 清理: 压测后重建
```

## 六、注意事项

1. **每个环境配置独立隔离** - 用 Namespace/Profile 区分
2. **测试环境数据定期清理** - 避免磁盘空间耗尽
3. **CI 流水线自动创建环境** - 按需创建，用后销毁
4. **环境配置不要硬编码** - 使用 ConfigMap/环境变量
5. **使用 Testcontainers 替代共享中间件** - 避免测试间干扰
6. **PR 级别独立环境** - 开发者互不干扰
