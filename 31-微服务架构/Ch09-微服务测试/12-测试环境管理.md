# 测试环境管理

## 一、环境分类

```yaml
测试环境:
  本地环境:
    - 用途: 开发自测
    - 数据: 内存/H2
    - 服务: 本机启动

  集成环境:
    - 用途: 联调测试
    - 数据: 独立数据库
    - 服务: Docker Compose

  预发布环境:
    - 用途: 准生产验证
    - 数据: 脱敏数据
    - 服务: K8s 集群

  沙箱环境:
    - 用途: 性能/混沌测试
    - 数据: 独立集群
    - 服务: 与生产同构
```

## 二、Docker Compose 测试环境

```yaml
# docker-compose.test.yml
version: '3.8'
services:
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: test
      MYSQL_DATABASE: order_db
    ports:
      - "3306:3306"
    healthcheck:
      test: ["CMD", "mysqladmin", "ping"]
      interval: 10s

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"

  rabbitmq:
    image: rabbitmq:3-management
    ports:
      - "5672:5672"
      - "15672:15672"

  nacos:
    image: nacos/nacos-server:v2.3.0
    environment:
      MODE: standalone
    ports:
      - "8848:8848"
```

## 三、环境配置管理

```java
// application-test.yml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/order_db
    username: test
    password: test
  cloud:
    nacos:
      discovery:
        server-addr: localhost:8848
        namespace: test

// 使用 @ActiveProfiles 切换
@SpringBootTest
@ActiveProfiles("test")
class WithTestProfile { }
```

## 四、注意事项

1. **每个环境配置独立隔离**
2. **测试环境数据定期清理**
3. **CI 流水线自动创建环境**
4. **环境配置不要硬编码**
5. **使用 Testcontainers 替代共享中间件**
