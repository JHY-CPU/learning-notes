# 性能测试

## 一、JMeter 基础

```xml
<!-- JMeter 测试计划 -->
<TestPlan>
  <ThreadGroup>
    <numThreads>100</numThreads>
    <rampTime>30</rampTime>
    <loopCount>1000</loopCount>
  </ThreadGroup>
  <HTTPSampler>
    <domain>localhost</domain>
    <port>8080</port>
    <path>/api/orders</path>
    <method>POST</method>
  </HTTPSampler>
</TestPlan>
```

## 二、Gatling 压测脚本

```scala
class OrderLoadTest extends Simulation {

  val httpProtocol = http.baseUrl("http://localhost:8080")
    .acceptHeader("application/json")
    .contentTypeHeader("application/json")

  val scn = scenario("下单压测")
    .exec(http("创建订单")
      .post("/api/orders")
      .body(StringBody("""{"productId":"P001","quantity":1}"""))
      .check(status.is(201)))

  setUp(
    scn.inject(
      rampUsers(1000).during(30.seconds),   // 30秒内递增到1000
      constantUsersPerSec(100).during(60)    // 每秒100用户持续60秒
    )
  ).protocols(httpProtocol)
}
```

## 三、关键指标

```yaml
性能指标:
  吞吐量 (TPS):
    - 目标: > 1000 TPS
    - 含义: 每秒处理请求数

  响应时间 (RT):
    - P50: < 100ms
    - P95: < 500ms
    - P99: < 1000ms

  错误率:
    - 目标: < 0.1%

  并发用户数:
    - 目标: > 500
```

## 四、注意事项

1. **性能测试要与生产环境尽量一致**
2. **关注 P95/P99 而非平均值**
3. **逐步增加压力**，找到系统瓶颈
4. **CPU/内存/IO 要一起监控**
5. **压测数据要用独立环境**
