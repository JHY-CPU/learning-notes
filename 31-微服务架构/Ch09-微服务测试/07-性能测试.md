## 二、Gatling 压测脚本

```scala
class OrderLoadTest extends Simulation {

  val httpProtocol = http.baseUrl("http://localhost:8080")
    .acceptHeader("application/json")
    .contentTypeHeader("application/json")

  val scn = scenario("下单压测")
    .exec(http("创建订单")
      .post("/api/orders")
      .body(StringBody("""{"productId":"P001","quantity":1}"""))
      .check(status.is(201)))

  setUp(
    scn.inject(
      rampUsers(1000).during(30.seconds),   // 30秒内递增到1000
      constantUsersPerSec(100).during(60)    // 每秒100用户持续60秒
    )
  ).protocols(httpProtocol)
}
```

## 四、Gatling 完整脚本

```scala
class OrderStressTest extends Simulation {
  val httpProtocol = http.baseUrl("http://localhost:8080")
    .acceptHeader("application/json")
    .contentTypeHeader("application/json")
    .header("Authorization", "Bearer test-token")

  val createOrder = scenario("创建订单")
    .exec(http("创建订单")
      .post("/api/orders")
      .body(StringBody("""{"productId":"P001","quantity":1}"""))
      .check(status.is(201), jsonPath("$.id").saveAs("orderId")))
    .pause(1)
    .exec(http("查询订单")
      .get("/api/orders/${orderId}")
      .check(status.is(200)))

  val queryOrders = scenario("查询订单列表")
    .exec(http("订单列表")
      .get("/api/orders?page=1&size=20")
      .check(status.is(200)))

  setUp(
    createOrder.inject(
      rampUsers(100).during(10.seconds),
      constantUsersPerSec(50).during(60.seconds),
      rampUsersPerSec(50).to(200).during(30.seconds),
      constantUsersPerSec(200).during(60.seconds)
    ),
    queryOrders.inject(
      rampUsers(200).during(10.seconds),
      constantUsersPerSec(100).during(120.seconds)
    )
  ).protocols(httpProtocol)
    .assertions(
      global.responseTime.percentile(99).lt(2000),  // P99 < 2s
      global.successfulRequests.percent.gt(99.5),     // 成功率 > 99.5%
      forAll.responseTime.mean.lt(500)                // 平均响应 < 500ms
    )
}
```

## 六、注意事项

1. **性能测试要与生产环境尽量一致** - 服务器规格、网络带宽、数据量
2. **关注 P95/P99 而非平均值** - 平均值掩盖尾部延迟问题
3. **逐步增加压力** - 找到系统瓶颈和最大承载能力
4. **CPU/内存/IO 要一起监控** - 不能只看 QPS
5. **压测数据要用独立环境** - 避免影响生产数据
6. **定期压测** - 每次大版本发布前进行性能回归测试
