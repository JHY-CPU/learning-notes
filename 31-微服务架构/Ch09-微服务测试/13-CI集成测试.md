# CI 集成测试

## 一、GitLab CI 流水线

```yaml
# .gitlab-ci.yml
stages:
  - build
  - unit-test
  - integration-test
  - deploy

build:
  stage: build
  script:
    - mvn clean package -DskipTests
  artifacts:
    paths:
      - target/*.jar

unit-test:
  stage: unit-test
  script:
    - mvn test
  coverage: '/Total.*?([0-9]{1,3})%/'
  artifacts:
    reports:
      junit: target/surefire-reports/TEST-*.xml

integration-test:
  stage: integration-test
  services:
    - mysql:8.0
    - redis:7
  variables:
    MYSQL_DATABASE: test_db
    MYSQL_ROOT_PASSWORD: test
  script:
    - mvn verify -P integration-test
  artifacts:
    reports:
      junit: target/failsafe-reports/TEST-*.xml
  only:
    - merge_requests
    - main
```

## 二、GitHub Actions

```yaml
name: CI Pipeline
on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    services:
      mysql:
        image: mysql:8.0
        env:
          MYSQL_ROOT_PASSWORD: test
        ports:
          - 3306:3306
      redis:
        image: redis:7
        ports:
          - 6379:6379

    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-java@v4
        with:
          java-version: '17'
          distribution: 'temurin'

      - name: Run Tests
        run: mvn verify
        env:
          SPRING_DATASOURCE_URL: jdbc:mysql://localhost:3306/test

      - name: Upload Coverage
        uses: codecov/codecov-action@v4
        with:
          files: target/site/jacoco/jacoco.xml
```

## 三、质量门禁

```yaml
质量门禁规则:
  单元测试:
    - 通过率: 100%
    - 覆盖率: > 80%

  集成测试:
    - 通过率: 100%
    - 核心路径覆盖

  SonarQube:
    - Bug: 0
    - 漏洞: 0
    - 代码异味: < 100

  代码审查:
    - 至少 1 人审批
    - CI 全部通过
```

## 四、注意事项

1. **单元测试每次提交都跑**
2. **集成测试 MR 时跑**
3. **E2E 测试合并后跑**
4. **测试失败要阻断流水线**
5. **测试报告要归档可查**
