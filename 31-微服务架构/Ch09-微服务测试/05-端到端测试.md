# 端到端测试 (E2E)

## 一、RestAssured API 测试

```java
@SpringBootTest(webEnvironment = SpringBootTest.WebEnvironment.RANDOM_PORT)
@TestMethodOrder(MethodOrderer.OrderAnnotation.class)
class OrderE2ETest {

    @LocalServerPort
    private int port;

    private String token;

    @BeforeEach
    void login() {
        token = given()
            .contentType(ContentType.JSON)
            .body("{\"username\":\"test\",\"password\":\"test123\"}")
        .when()
            .post("/api/auth/login")
        .then()
            .extract().jsonPath().getString("token");
    }

    @Test
    @Order(1)
    void shouldCreateOrderSuccessfully() {
        given()
            .header("Authorization", "Bearer " + token)
            .contentType(ContentType.JSON)
            .body("{\"productId\":\"P001\",\"quantity\":2}")
        .when()
            .post("/api/orders")
        .then()
            .statusCode(201)
            .body("status", equalTo("CREATED"));
    }

    @Test
    @Order(2)
    void shouldQueryOrderById() {
        given()
            .header("Authorization", "Bearer " + token)
        .when()
            .get("/api/orders/{id}", "O001")
        .then()
            .statusCode(200)
            .body("id", equalTo("O001"));
    }
}
```

## 二、Cucumber BDD 测试

```gherkin
# features/order.feature
Feature: 订单管理

  Scenario: 正常下单
    Given 用户 "张三" 已登录
    And 商品 "iPhone" 库存为 10
    When 用户购买 "iPhone" 2 件
    Then 订单创建成功
    And 订单状态为 "CREATED"
    And 商品库存变为 8
```

```java
@SpringBootTest
@CucumberContextConfiguration
public class OrderStepDefinitions {

    @Autowired
    private OrderService orderService;

    @Given("用户 {string} 已登录")
    public void userLoggedIn(String username) {
        SecurityContextHolder.getContext()
            .setAuthentication(new TestingAuthenticationToken(username, null));
    }

    @When("用户购买 {string} {int} 件")
    public void purchaseProduct(String productId, int quantity) {
        order = orderService.createOrder(new OrderRequest(productId, quantity));
    }

    @Then("订单创建成功")
    public void orderCreated() {
        assertThat(order).isNotNull();
    }
}
```

## 三、注意事项

1. **E2E 测试覆盖核心业务流程即可**
2. **测试数据要独立，避免相互影响**
3. **使用 @Order 控制执行顺序**
4. **CI 中 E2E 测试单独阶段**
5. **失败后要能快速定位问题**
