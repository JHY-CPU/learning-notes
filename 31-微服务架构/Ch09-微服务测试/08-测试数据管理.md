# 测试数据管理

## 一、测试数据策略

```yaml
数据管理方式:
  内联数据:
    - 适用: 简单单元测试
    - 示例: @CsvSource, @ValueSource

  Fixture 文件:
    - 适用: 复杂数据结构
    - 工具: JSON/YAML 文件
    - 路径: src/test/resources/fixtures/

  数据库快照:
    - 适用: 集成测试
    - 工具: Flyway, DBUnit

  动态生成:
    - 适用: 需要唯一数据
    - 工具: JavaFaker, Instancio
```

## 二、数据构建器模式

```java
// 测试数据构建器
public class OrderTestDataBuilder {
    private String customerId = "C001";
    private BigDecimal amount = BigDecimal.valueOf(100);
    private OrderStatus status = OrderStatus.CREATED;

    public static OrderTestDataBuilder anOrder() {
        return new OrderTestDataBuilder();
    }

    public OrderTestDataBuilder withCustomer(String customerId) {
        this.customerId = customerId;
        return this;
    }

    public OrderTestDataBuilder withAmount(BigDecimal amount) {
        this.amount = amount;
        return this;
    }

    public OrderTestDataBuilder withStatus(OrderStatus status) {
        this.status = status;
        return this;
    }

    public Order build() {
        Order order = new Order(customerId, amount);
        order.setStatus(status);
        return order;
    }
}

// 使用
Order order = anOrder()
    .withCustomer("C002")
    .withAmount(BigDecimal.valueOf(500))
    .withStatus(OrderStatus.PAID)
    .build();
```

## 三、数据库测试隔离

```java
@TestConfiguration
public class TestConfig {
    @Bean
    public DatabaseCleaner databaseCleaner(DataSource dataSource) {
        return new DatabaseCleaner(dataSource);
    }
}

// 每个测试前后清理数据
@ExtendWith(DatabaseCleanupExtension.class)
@SpringBootTest
class IsolatedTest {
    // 测试逻辑
}
```

## 四、注意事项

1. **测试数据与测试逻辑分离**
2. **避免测试间数据依赖**
3. **使用 Builder 模式构建复杂对象**
4. **敏感数据要脱敏**
5. **CI 环境中测试数据自动准备**
