# 微服务通信安全

## 一、核心概念

微服务之间存在大量内部通信，通信安全是微服务安全体系的核心。本节介绍 mTLS（双向 TLS）、JWT（JSON Web Token）、OAuth2 等安全机制在微服务场景中的应用。

### 1.1 安全威胁模型

```
威胁类型            风险                  防护措施
──────────────────────────────────────────────────
窃听攻击           数据泄露              TLS 加密
中间人攻击         数据篡改              mTLS 双向认证
身份伪造           越权访问              JWT/OAuth2
重放攻击           重复操作              Nonce/时间戳
数据篡改           业务异常              签名验证
DoS 攻击           服务不可用            限流/熔断
```

### 1.2 安全架构

```
┌──────────┐                          ┌──────────┐
│ 服务 A    │ ═══ mTLS + JWT ════════►│ 服务 B    │
│          │                          │          │
│ • 私钥    │   1. TLS 握手            │ • 验证证书│
│ • 证书    │   2. JWT Token 传递       │ • 验证JWT │
│ • JWT    │   3. 请求加密传输          │ • 权限检查│
└──────────┘                          └──────────┘
```

## 二、mTLS 双向认证

### 2.1 证书体系

```
CA (根证书)
  ├── Service A 证书
  ├── Service B 证书
  ├── Service C 证书
  └── Gateway 证书

认证流程:
  服务A ──TLS ClientHello──► 服务B
  服务A ◄──TLS ServerHello── 服务B
  服务A ──客户端证书──►        服务B (验证 A 的证书)
  服务A ◄──服务端证书──        服务B (A 验证 B 的证书)
  服务A ═══加密通信═══►       服务B
```

### 2.2 Spring Boot mTLS 配置

```yaml
# 服务端配置
server:
  ssl:
    enabled: true
    key-store: classpath:keystore/server.p12
    key-store-password: ${KEYSTORE_PASSWORD}
    key-store-type: PKCS12
    trust-store: classpath:truststore/ca.p12
    trust-store-password: ${TRUSTSTORE_PASSWORD}
    client-auth: need  # 要求客户端提供证书

# 客户端配置
spring:
  cloud:
    openfeign:
      client:
        config:
          order-service:
            url: https://order-service:8443
    httpclient:
      ssl:
        key-store: classpath:keystore/client.p12
        key-store-password: ${CLIENT_KEYSTORE_PASSWORD}
        trust-store: classpath:truststore/ca.p12
        trust-store-password: ${TRUSTSTORE_PASSWORD}
```

### 2.3 使用 cert-manager 自动管理证书（Kubernetes）

## 三、JWT 认证

### 3.1 JWT 令牌结构

```
Header.Payload.Signature

Header:
{
  "alg": "RS256",
  "typ": "JWT"
}

Payload:
{
  "sub": "user-123",
  "name": "张三",
  "roles": ["admin", "user"],
  "iat": 1700000000,
  "exp": 1700003600,
  "iss": "auth-service"
}
```

### 3.2 JWT 工具类

```java
@Component
public class JwtUtil {

    @Value("${jwt.secret}")
    private String secret;

    @Value("${jwt.expiration:3600}")
    private long expiration;

    /**
     * 生成 JWT Token
     */
    public String generateToken(String userId,
                                 List<String> roles) {
        return Jwts.builder()
            .setSubject(userId)
            .claim("roles", roles)
            .setIssuedAt(new Date())
            .setExpiration(
                new Date(System.currentTimeMillis()
                    + expiration * 1000))
            .setIssuer("auth-service")
            .signWith(SignatureAlgorithm.RS256,
                getPrivateKey())
            .compact();
    }

    /**
     * 验证并解析 Token
     */
    public Claims parseToken(String token) {
        return Jwts.parser()
            .setSigningKey(getPublicKey())
            .parseClaimsJws(token)
            .getBody();
    }

    /**
     * 从 Token 中提取用户信息
     */
    public UserInfo getUserInfo(String token) {
        Claims claims = parseToken(token);
        return UserInfo.builder()
            .userId(claims.getSubject())
            .roles(claims.get("roles", List.class))
            .build();
    }
}
```

### 3.3 JWT 过滤器

```java
@Component
public class JwtAuthFilter extends OncePerRequestFilter {

    @Autowired
    private JwtUtil jwtUtil;

    @Override
    protected void doFilterInternal(
            HttpServletRequest request,
            HttpServletResponse response,
            FilterChain chain)
            throws ServletException, IOException {
        String authHeader =
            request.getHeader("Authorization");

        if (authHeader != null
                && authHeader.startsWith("Bearer ")) {
            String token = authHeader.substring(7);
            try {
                UserInfo user =
                    jwtUtil.getUserInfo(token);
                // 设置到 SecurityContext
                SecurityContextHolder.getContext()
                    .setAuthentication(
                        new UsernamePasswordAuthenticationToken(
                            user, null,
                            buildAuthorities(user)));
            } catch (JwtException e) {
                response.setStatus(
                    HttpServletResponse.SC_UNAUTHORIZED);
                return;
            }
        }

        chain.doFilter(request, response);
    }
}
```

## 四、OAuth2 认证

### 4.1 OAuth2 授权码模式

```
客户端           授权服务器         资源服务器
  │                │                  │
  │──授权请求──►   │                  │
  │◄──授权码────   │                  │
  │                │                  │
  │──Token请求──► │                  │
  │◄──Access Token │                  │
  │                │                  │
  │──API请求───────+─────────────────►│
  │   (带Token)    │    验证Token      │
  │◄──API响应──────+──────────────────│
```

### 4.2 Spring Security OAuth2 配置

```java
@Configuration
@EnableResourceServer
public class ResourceServerConfig
        extends ResourceServerConfigurerAdapter {

    @Override
    public void configure(ResourceServerSecurityConfigurer
            resources) {
        resources
            .resourceId("order-service")
            .tokenStore(tokenStore());
    }

    @Override
    public void configure(HttpSecurity http) throws Exception {
        http
            .authorizeRequests()
            .antMatchers("/api/public/**").permitAll()
            .antMatchers("/api/admin/**")
                .hasAuthority("ROLE_ADMIN")
            .anyRequest().authenticated();
    }

    @Bean
    public TokenStore tokenStore() {
        return new JwtTokenStore(jwtAccessTokenConverter());
    }
}
```

## 五、服务间认证

### 5.1 服务身份令牌

```java
@Component
public class ServiceAuthTokenProvider {

    /**
     * 为服务间调用生成 Token
     */
    public String getServiceToken() {
        // 使用服务身份（而非用户身份）申请 Token
        return Jwts.builder()
            .setSubject(serviceName)
            .claim("type", "service")
            .setIssuedAt(new Date())
            .setExpiration(new Date(
                System.currentTimeMillis() + 3600000))
            .signWith(SignatureAlgorithm.RS256,
                privateKey)
            .compact();
    }
}
```

### 5.2 Feign 客户端自动传递 Token

```java
@Component
public class FeignTokenInterceptor
        implements RequestInterceptor {

    @Override
    public void apply(RequestTemplate template) {
        // 传递当前用户 Token
        String userToken = SecurityContextHolder
            .getContext().getAuthentication()
            .getCredentials().toString();
        template.header("Authorization",
            "Bearer " + userToken);

        // 传递 TraceID
        String traceId = MDC.get("traceId");
        if (traceId != null) {
            template.header("X-Trace-Id", traceId);
        }
    }
}
```

## 六、最佳实践

1. **服务间通信也要认证**，不要因为是"内部"就不认证
2. **内部通信推荐使用 mTLS**，双向认证最安全
3. **Token 要设置过期时间**，避免长期有效的 Token
4. **敏感数据传输要加密**，不要传输明文密码
5. **定期轮换密钥**，降低密钥泄露风险
6. **使用 HTTPS**，即使在内网也要加密
7. **API 签名验证**，防止数据篡改
8. **请求限流防刷**，防止恶意调用
