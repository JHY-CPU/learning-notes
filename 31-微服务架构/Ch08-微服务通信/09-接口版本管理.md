# 接口版本管理

## 一、核心概念

在微服务架构中，API 接口会随业务需求不断演进。接口版本管理是指在保证向后兼容的前提下，管理接口的多个版本，确保不同版本的客户端都能正常工作。

### 1.1 为什么需要版本管理

```
场景                          影响
──────────────────────────────────────────
新增字段                      旧客户端可能无法处理
删除字段                      使用该字段的客户端报错
修改字段类型                  序列化/反序列化失败
修改返回结构                  客户端解析失败
修改业务逻辑                  行为不符合预期
```

### 1.2 版本策略对比

| 策略 | 示例 | 优点 | 缺点 |
|------|------|------|------|
| URI 路径版本 | `/api/v1/users` | 简单直观 | URI 不够 RESTful |
| 请求参数版本 | `/api/users?version=1` | URI 干净 | 不太直观 |
| Header 版本 | `Accept: application/vnd.api.v1+json` | RESTful | 客户端实现复杂 |
| Content-Type 版本 | `Accept: application/json;v=1` | 标准化 | 不太直观 |

## 二、URI 路径版本

### 2.1 Controller 版本管理

```java
// v1 版本
@RestController
@RequestMapping("/api/v1/users")
public class UserControllerV1 {

    @GetMapping("/{id}")
    public UserV1 getUserV1(@PathVariable Long id) {
        return userService.getUserV1(id);
    }

    @PostMapping
    public UserV1 createUserV1(
            @RequestBody CreateUserRequestV1 request) {
        return userService.createUserV1(request);
    }
}

// v2 版本（新增字段、扩展功能）
@RestController
@RequestMapping("/api/v2/users")
public class UserControllerV2 {

    @GetMapping("/{id}")
    public UserV2 getUserV2(@PathVariable Long id) {
        return userService.getUserV2(id);
    }

    @PostMapping
    public UserV2 createUserV2(
            @RequestBody CreateUserRequestV2 request) {
        return userService.createUserV2(request);
    }
}
```

## 三、Header 版本

### 3.1 基于 Header 的版本路由

```java
@RestController
@RequestMapping("/api/users")
public class UserController {

    @GetMapping(value = "/{id}",
        headers = "X-API-Version=1")
    public UserV1 getUserV1(@PathVariable Long id) {
        return userService.getUserV1(id);
    }

    @GetMapping(value = "/{id}",
        headers = "X-API-Version=2")
    public UserV2 getUserV2(@PathVariable Long id) {
        return userService.getUserV2(id);
    }
}
```

### 3.2 自定义版本路由注解

```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface ApiVersion {
    int value();
}

@RestController
@RequestMapping("/api/users")
public class UserController {

    @GetMapping("/{id}")
    @ApiVersion(1)
    public UserV1 getUserV1(@PathVariable Long id) {
        return userService.getUserV1(id);
    }

    @GetMapping("/{id}")
    @ApiVersion(2)
    public UserV2 getUserV2(@PathVariable Long id) {
        return userService.getUserV2(id);
    }
}

// 版本路由解析器
@Component
public class ApiVersionHandlerMapping
        extends RequestMappingHandlerMapping {

    @Override
    protected RequestMappingInfo getMappingForMethod(
            Method method, Class<?> handlerType) {
        RequestMappingInfo info =
            super.getMappingForMethod(method, handlerType);
        ApiVersion version =
            method.getAnnotation(ApiVersion.class);
        if (version != null) {
            // 根据 Header 版本号路由
            PatternsRequestCondition versionCondition =
                new PatternsRequestCondition(
                    "/v" + version.value());
            info = RequestMappingInfo
                .paths(versionCondition.getValue())
                .build()
                .combine(info);
        }
        return info;
    }
}
```

## 四、向后兼容策略

### 4.1 兼容性原则

```
允许的变更（向后兼容）:
  ✓ 新增可选字段（带默认值）
  ✓ 新增接口端点
  ✓ 新增枚举值
  ✓ 放宽输入约束
  ✓ 增加可选请求参数

禁止的变更（破坏兼容）:
  ✗ 删除字段
  ✗ 修改字段类型
  ✗ 修改字段名
  ✗ 修改字段语义
  ✗ 删除枚举值
  ✗ 收紧输入约束
```

### 4.2 DTO 版本适配

```java
// v1 DTO
@Data
public class UserV1 {
    private Long id;
    private String name;
    private String email;
}

// v2 DTO（新增字段）
@Data
public class UserV2 {
    private Long id;
    private String name;
    private String email;
    private String phone;        // v2 新增
    private String avatar;       // v2 新增
    private LocalDateTime createdAt; // v2 新增
}

// 版本转换器
@Component
public class UserVersionConverter {

    public UserV2 toV2(UserV1 v1) {
        return UserV2.builder()
            .id(v1.getId())
            .name(v1.getName())
            .email(v1.getEmail())
            .build();  // 新增字段为 null
    }

    public UserV1 toV1(UserV2 v2) {
        return UserV1.builder()
            .id(v2.getId())
            .name(v2.getName())
            .email(v2.getEmail())
            .build();  // 忽略 v2 新增字段
    }
}
```

## 五、版本生命周期管理

### 5.1 版本生命周期

```
版本状态流转:
  Alpha → Beta → GA → Deprecated → Sunset → Removed

  Alpha:    内部测试
  Beta:     公开测试
  GA:       正式发布（General Availability）
  Deprecated: 标记废弃（仍可用，但建议迁移）
  Sunset:   日落期（准备下线）
  Removed:  已移除
```

### 5.2 废弃接口管理

```java
@RestController
@RequestMapping("/api/v1/users")
public class UserControllerV1 {

    /**
     * @deprecated 自 2024-01-01 起废弃，请使用 v2 接口
     * 将于 2024-06-01 下线
     */
    @Deprecated
    @GetMapping("/{id}")
    @DeprecatedApi(
        since = "2024-01-01",
        sunset = "2024-06-01",
        replacement = "/api/v2/users/{id}"
    )
    public UserV1 getUserV1(@PathVariable Long id) {
        return userService.getUserV1(id);
    }
}

// 废弃接口过滤器
@Component
public class DeprecatedApiFilter implements GlobalFilter {

    @Override
    public Mono<Void> filter(ServerWebExchange exchange,
                              GatewayFilterChain chain) {
        String path = exchange.getRequest().getPath().value();
        DeprecatedApiInfo api =
            deprecatedApiRegistry.get(path);

        if (api != null) {
            // 添加 Deprecation 响应头
            exchange.getResponse().getHeaders()
                .add("Deprecation", api.getSince());
            exchange.getResponse().getHeaders()
                .add("Sunset", api.getSunset());
            exchange.getResponse().getHeaders()
                .add("Link",
                    "<" + api.getReplacement()
                    + ">; rel=\"successor-version\"");
        }

        return chain.filter(exchange);
    }
}
```

### 5.3 版本使用统计

```java
@Component
public class ApiVersionMetrics {

    @EventListener
    public void onApiCall(ApiCallEvent event) {
        String apiPath = event.getPath();
        String version = extractVersion(apiPath);

        Metrics.counter("api.version.calls",
            "path", apiPath,
            "version", version).increment();
    }

    /**
     * 识别即将下线但仍有流量的接口
     */
    @Scheduled(cron = "0 0 9 * * ?")
    public void checkDeprecatedApiTraffic() {
        for (DeprecatedApi api : deprecatedApiRegistry.getAll()) {
            double qps = getQps(api.getPath());
            if (qps > 0 && api.getSunset()
                    .isBefore(LocalDateTime.now())) {
                alertService.send("DEPRECATED_API_HAS_TRAFFIC",
                    String.format(
                        "废弃接口 %s 仍有流量 (%.1f QPS)",
                        api.getPath(), qps));
            }
        }
    }
}
```

## 六、最佳实践

1. **版本策略要统一**，整个系统使用同一种版本策略
2. **向后兼容是原则**，新增字段而非修改字段
3. **废弃接口要通知**，提前至少 3 个月通知下游
4. **版本切换要平滑**，提供迁移指南和过渡期
5. **文档要同步更新**，每个版本都要有对应的文档
6. **监控版本使用情况**，了解各版本的调用量
7. **设置版本下线时间线**，避免长期维护多版本
8. **DTO 使用 Builder 模式**，方便不同版本间的转换
