# 服务间调用优化与请求重试

## 一、核心概念

服务间调用是微服务通信的核心。调用优化包括连接池管理、请求重试、超时控制、幂等保障、请求合并与批处理等，目标是提高通信效率、降低延迟、保证可靠性。

### 1.1 调用优化维度

```
维度            手段                    效果
──────────────────────────────────────────────
连接复用       连接池                  减少建连开销
并发调用       异步/并行               降低总耗时
请求合并       Batch API              减少网络往返
缓存           结果缓存                减少重复调用
超时控制       合理超时                快速失败
重试策略       指数退避重试            提高成功率
幂等保障       幂等键                  安全重试
```

## 二、连接池优化

### 2.1 HTTP 连接池配置

```java
@Configuration
public class HttpClientConfig {

    @Bean
    public RestTemplate restTemplate() {
        // Apache HttpClient 连接池
        PoolingHttpClientConnectionManager connManager =
            new PoolingHttpClientConnectionManager();
        connManager.setMaxTotal(200);        // 总连接数
        connManager.setDefaultMaxPerRoute(50); // 每路由连接数

        CloseableHttpClient httpClient = HttpClientBuilder
            .create()
            .setConnectionManager(connManager)
            // 保持连接活跃
            .setKeepAliveStrategy(
                (response, context) -> Duration.ofSeconds(30))
            // 连接空闲清理
            .evictIdleConnections(
                Duration.ofSeconds(60))
            // 重试策略
            .setRetryHandler(
                new DefaultHttpRequestRetryHandler(3, true))
            .build();

        HttpComponentsClientHttpRequestFactory factory =
            new HttpComponentsClientHttpRequestFactory(
                httpClient);
        factory.setConnectTimeout(3000);
        factory.setReadTimeout(5000);

        return new RestTemplate(factory);
    }
}
```

## 三、请求重试策略

### 3.1 重试策略对比

```
策略               适用场景               注意事项
───────────────────────────────────────────────────
固定间隔           短暂故障              可能造成请求堆积
线性递增           中等故障              逐步增加等待
指数退避           网络抖动/过载          最常用
随机抖动           高并发场景            避免惊群效应
```

### 3.2 指数退避重试

```java
@Configuration
public class RetryConfig {

    @Bean
    public RetryTemplate retryTemplate() {
        RetryTemplate template = new RetryTemplate();

        // 重试策略：只对特定异常重试
        SimpleRetryPolicy retryPolicy =
            new SimpleRetryPolicy();
        retryPolicy.setMaxAttempts(3);

        Map<Class<? extends Throwable>, Boolean>
            retryableExceptions = new HashMap<>();
        retryableExceptions.put(
            ResourceAccessException.class, true);
        retryableExceptions.put(
            HttpServerErrorException.class, true);
        retryableExceptions.put(
            SocketTimeoutException.class, true);
        retryPolicy.setRetryableExceptions(
            retryableExceptions);

        // 退避策略：指数退避 + 随机抖动
        ExponentialBackOffPolicy backOffPolicy =
            new ExponentialBackOffPolicy();
        backOffPolicy.setInitialInterval(1000);    // 初始 1s
        backOffPolicy.setMaxInterval(10000);       // 最大 10s
        backOffPolicy.setMultiplier(2);            // 倍数 2

        // 添加随机抖动
        ExponentialRandomBackOffPolicy randomPolicy =
            new ExponentialRandomBackOffPolicy();
        randomPolicy.setInitialInterval(1000);
        randomPolicy.setMaxInterval(10000);
        randomPolicy.setMultiplier(2);

        template.setRetryPolicy(retryPolicy);
        template.setBackOffPolicy(randomPolicy);

        return template;
    }
}
```

## 四、幂等性保障

### 4.1 幂等键设计

```java
@Service
public class IdempotentOrderService {

    /**
     * 创建订单（幂等）
     */
    @Idempotent(key = "#request.idempotentKey",
                expire = 3600)
    public Order createOrder(CreateOrderRequest request) {
        // 幂等检查
        Order existing = orderRepository
            .findByIdempotentKey(
                request.getIdempotentKey());
        if (existing != null) {
            log.info("幂等命中，返回已有订单: {}",
                existing.getId());
            return existing;
        }

        // 创建新订单
        Order order = buildOrder(request);
        order.setIdempotentKey(
            request.getIdempotentKey());
        return orderRepository.save(order);
    }
}
```

### 4.2 Redis 幂等锁

```java
@Aspect
@Component
public class IdempotentAspect {

    @Autowired
    private StringRedisTemplate redisTemplate;

    @Around("@annotation(idempotent)")
    public Object around(ProceedingJoinPoint joinPoint,
                          Idempotent idempotent) throws Throwable {
        String key = parseKey(joinPoint, idempotent.key());

        // 尝试获取锁
        Boolean locked = redisTemplate.opsForValue()
            .setIfAbsent("idempotent:" + key, "1",
                idempotent.expire(), TimeUnit.SECONDS);

        if (Boolean.FALSE.equals(locked)) {
            // 已处理过，返回缓存结果
            String cached = redisTemplate.opsForValue()
                .get("idempotent:result:" + key);
            if (cached != null) {
                return JSON.parseObject(cached, Object.class);
            }
            throw new IdempotentException("重复请求");
        }

        try {
            Object result = joinPoint.proceed();
            // 缓存结果
            redisTemplate.opsForValue()
                .set("idempotent:result:" + key,
                    JSON.toJSONString(result),
                    idempotent.expire(), TimeUnit.SECONDS);
            return result;
        } catch (Exception e) {
            // 执行失败，释放锁允许重试
            redisTemplate.delete("idempotent:" + key);
            throw e;
        }
    }
}
```

## 五、请求合并与批处理

### 5.1 请求合并（Request Collapsing）

```java
@Component
public class UserRequestCollapser {

    private final BlockingQueue<Request<UserDTO, Long>>
        queue = new LinkedBlockingQueue<>();

    /**
     * 合并多个用户查询为批量查询
     */
    public CompletableFuture<UserDTO> getUser(Long userId) {
        CompletableFuture<UserDTO> future =
            new CompletableFuture<>();
        queue.add(new Request<>(userId, future));

        // 启动合并器（如果未启动）
        ensureCollapserRunning();
        return future;
    }

    @PostConstruct
    public void ensureCollapserRunning() {
        Executors.newSingleThreadScheduledExecutor()
            .scheduleAtFixedRate(() -> {
                List<Request<UserDTO, Long>> batch =
                    new ArrayList<>();
                queue.drainTo(batch, 100);  // 最多 100 个

                if (!batch.isEmpty()) {
                    try {
                        // 批量查询
                        List<Long> ids = batch.stream()
                            .map(Request::getKey)
                            .collect(Collectors.toList());
                        Map<Long, UserDTO> users =
                            userService.batchGetUsers(ids);

                        // 分发结果
                        for (Request<UserDTO, Long> req : batch) {
                            req.getFuture().complete(
                                users.get(req.getKey()));
                        }
                    } catch (Exception e) {
                        for (Request<UserDTO, Long> req : batch) {
                            req.getFuture().completeExceptionally(e);
                        }
                    }
                }
            }, 0, 50, TimeUnit.MILLISECONDS); // 50ms 合并窗口
    }
}
```

### 5.2 批量 API 调用

```java
@Service
public class BatchOptimizedService {

    /**
     * 批量获取用户信息（减少网络往返）
     */
    public Map<Long, UserDTO> batchGetUsers(
            List<Long> userIds) {
        // 分批处理，每批 100 个
        List<List<Long>> partitions =
            Lists.partition(userIds, 100);

        Map<Long, UserDTO> result = new HashMap<>();

        for (List<Long> batch : partitions) {
            // 调用批量接口
            List<UserDTO> users =
                restTemplate.postForObject(
                    "http://user-service/users/batch",
                    batch,
                    new ParameterizedTypeReference<
                        List<UserDTO>>() {});

            for (UserDTO user : users) {
                result.put(user.getId(), user);
            }
        }

        return result;
    }
}
```

### 5.3 缓存减少重复调用

```java
@Service
public class CachedServiceCall {

    private final Cache<Long, UserDTO> userCache =
        Caffeine.newBuilder()
            .maximumSize(10000)
            .expireAfterWrite(5, TimeUnit.MINUTES)
            .build();

    public UserDTO getUser(Long userId) {
        return userCache.get(userId,
            id -> userServiceClient.getUser(id));
    }

    /**
     * 缓存穿透保护
     */
    public UserDTO getUserSafe(Long userId) {
        UserDTO user = userCache.getIfPresent(userId);
        if (user != null) {
            return user.isEmpty() ? null : user;
        }

        user = userServiceClient.getUser(userId);
        if (user == null) {
            // 缓存空值，防止穿透
            userCache.put(userId, UserDTO.empty());
            return null;
        }

        userCache.put(userId, user);
        return user;
    }
}
```

## 六、最佳实践

1. **连接池要合理配置**，根据 QPS 和延迟调整
2. **批量调用减少网络开销**，合并多个请求为一个批量请求
3. **并行调用减少总耗时**，使用 CompletableFuture 或响应式
4. **缓存减少重复调用**，热点数据使用本地缓存
5. **只传必要数据**，减少序列化和网络开销
6. **重试必须配合幂等**，防止重复操作
7. **指数退避 + 随机抖动**，避免惊群效应
8. **设置合理的超时时间**，避免请求长时间阻塞
