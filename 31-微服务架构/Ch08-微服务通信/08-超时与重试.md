# 超时与重试

## 一、超时配置

```yaml
feign:
  client:
    config:
      default:
        connect-timeout: 5000
        read-timeout: 10000
```

## 二、重试策略

```java
@Retry(name = "userService", fallbackMethod = "fallback")
public User getUser(Long id) {
    return userService.getUser(id);
}
```

## 三、工作原理

超时控制通过限制调用的最大等待时间，防止请求无限阻塞。连接超时控制建立 TCP 连接的等待时间，读取超时控制等待响应的时间。在调用链路中，上游的超时应大于下游的超时之和，否则上游先超时放弃请求，但下游仍在处理。重试机制在调用失败后自动重新发起请求，适用于瞬时故障。重试策略包括固定间隔和指数退避（间隔逐次翻倍），配合随机抖动避免惊群效应。

## 四、优缺点

**优点：**
- 超时防止资源耗尽，快速释放线程
- 重试自动恢复瞬时故障
- 对调用方透明，无需手动处理

**缺点：**
- 超时过短截断正常请求
- 重试加重下游故障压力
- 重试增加单次调用总延迟

## 五、最佳实践**

1. 连接超时 3 秒，读取超时根据接口 P99 设定（1.5-2 倍）
2. 调用链路中超时逐级递减
3. 重试仅用于幂等操作，最大重试 2-3 次
4. 使用指数退避 + 随机抖动

## 六、完整超时重试配置

```yaml
# Spring Cloud 全链路超时重试
spring:
  cloud:
    gateway:
      routes:
      - id: order-service
        uri: lb://order-service
        predicates:
        - Path=/api/orders/**
        filters:
        - name: Retry
          args:
            retries: 2
            statuses: BAD_GATEWAY, SERVICE_UNAVAILABLE
            methods: GET,PUT
            backoff:
              firstBackoff: 100ms
              maxBackoff: 500ms
              factor: 2
              basedOnPreviousValue: false

resilience4j:
  retry:
    instances:
      userService:
        max-attempts: 3
        wait-duration: 500ms
        exponential-backoff-multiplier: 2
        retry-exceptions:
          - java.io.IOException
          - java.net.SocketTimeoutException
        ignore-exceptions:
          - com.example.BusinessException
  timelimiter:
    instances:
      userService:
        timeout-duration: 5s
```

## 七、超时调优方法

```
调优步骤:
1. 收集接口的 P50/P95/P99 延迟数据
2. 设置超时 = P99 × 1.5（保守）或 P99 × 2.0（宽松）
3. 设置重试 = 最多 2-3 次，总超时 < 上游超时
4. 上线后观察超时触发率
5. 根据触发率调整超时值

注意:
- 总超时 = 单次超时 × (1 + 重试次数)
- 上游超时 > 总超时 + 缓冲（500ms）
```

## 八、常见陷阱

1. **超时未配置** - 默认值可能为无限等待
2. **非幂等操作使用重试** - 如创建操作重复执行
3. **上游超时大于下游超时** - 上游等待时间浪费
4. **重试无退避策略** - 大量客户端同时重试产生流量尖峰
5. **重试 + 熔断未联动** - 下游持续故障时重试加重雪崩
6. **只关注读取超时忽略连接超时** - 连接阶段也可能阻塞
