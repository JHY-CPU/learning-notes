# 序列化选择

## 一、核心概念

序列化是将内存对象转换为字节流（或反之）的过程。在微服务通信中，序列化性能直接影响通信效率和系统性能。本节对比 JSON、Protobuf、Thrift 等主流序列化方案。

### 1.1 序列化关注维度

```
维度              说明
──────────────────────────────
性能              序列化/反序列化速度
体积              编码后的字节大小
可读性            是否人类可读
兼容性            是否支持向前/向后兼容
跨语言            是否支持多语言
生态              工具链和社区支持
```

### 1.2 序列化方案对比

| 方案 | 性能 | 体积 | 可读性 | 跨语言 | 兼容性 | 适用场景 |
|------|------|------|--------|--------|--------|---------|
| JSON | 中 | 大 | 好 | 全语言 | 好 | REST API |
| Protobuf | 高 | 小 | 差 | 多语言 | 极好 | gRPC |
| Thrift | 高 | 小 | 差 | 多语言 | 好 | RPC 框架 |
| Hessian | 中 | 中 | 差 | 多语言 | 中 | Dubbo |
| Kryo | 极高 | 极小 | 差 | 仅 Java | 差 | Java 内部 |
| Avro | 高 | 小 | 差 | 多语言 | 极好 | 大数据 |
| MessagePack | 高 | 小 | 差 | 多语言 | 中 | 高性能场景 |

## 二、JSON 序列化

### 2.1 Jackson 配置

```yaml
spring:
  jackson:
    date-format: yyyy-MM-dd HH:mm:ss
    time-zone: Asia/Shanghai
    serialization:
      write-dates-as-timestamps: false
      fail-on-empty-beans: false
    deserialization:
      fail-on-unknown-properties: false
```

```java
@Configuration
public class JacksonConfig {

    @Bean
    public ObjectMapper objectMapper() {
        ObjectMapper mapper = new ObjectMapper();
        // 配置序列化选项
        mapper.configure(
            SerializationFeature.FAIL_ON_EMPTY_BEANS,
            false);
        mapper.configure(
            DeserializationFeature.FAIL_ON_UNKNOWN_PROPERTIES,
            false);
        // 注册 Java 8 时间模块
        mapper.registerModule(
            new JavaTimeModule());
        // 处理 null 值
        mapper.setSerializationInclusion(
            JsonInclude.Include.NON_NULL);
        return mapper;
    }
}
```

### 2.2 JSON 性能优化

```java
/**
 * 使用预编译的 ObjectWriter 提升性能
 */
public class JsonUtil {

    private static final ObjectMapper MAPPER =
        new ObjectMapper();

    // 预编译的 Writer（线程安全，可复用）
    private static final ObjectWriter WRITER =
        MAPPER.writerFor(Map.class);

    public static byte[] serialize(Object obj)
            throws JsonProcessingException {
        return WRITER.writeValueAsBytes(obj);
    }

    public static <T> T deserialize(byte[] data,
            Class<T> clazz) throws IOException {
        return MAPPER.readValue(data, clazz);
    }
}
```

## 三、Protobuf 序列化

### 3.1 Proto 文件定义

```protobuf
syntax = "proto3";
package com.example.proto;

option java_package = "com.example.proto";
option java_multiple_files = true;

message User {
  int64 id = 1;
  string name = 2;
  string email = 3;
  int32 age = 4;
  repeated string roles = 5;
  Address address = 6;
}

message Address {
  string province = 1;
  string city = 2;
  string detail = 3;
}

message Order {
  int64 id = 1;
  int64 user_id = 2;
  repeated OrderItem items = 3;
  double total_amount = 4;
  OrderStatus status = 5;
  int64 created_at = 6;
}

enum OrderStatus {
  PENDING = 0;
  PAID = 1;
  SHIPPED = 2;
  COMPLETED = 3;
  CANCELLED = 4;
}

message OrderItem {
  int64 product_id = 1;
  string product_name = 2;
  int32 quantity = 3;
  double price = 4;
}
```

### 3.2 Protobuf 使用

```java
// 构建 Protobuf 对象
User user = User.newBuilder()
    .setId(1L)
    .setName("张三")
    .setEmail("zhangsan@example.com")
    .setAge(25)
    .addRoles("admin")
    .addRoles("user")
    .setAddress(Address.newBuilder()
        .setProvince("浙江省")
        .setCity("杭州市")
        .setDetail("西湖区xxx号")
        .build())
    .build();

// 序列化
byte[] bytes = user.toByteArray();

// 反序列化
User parsed = User.parseFrom(bytes);
```

### 3.3 Spring Boot 集成 Protobuf

```yaml
spring:
```

## 五、序列化兼容性

### 5.1 向前/向后兼容

```
版本演进规则:
  - 只能添加带默认值的新字段
  - 不能删除已使用的字段编号
  - 不能更改字段类型
  - 可以废弃字段（标记为 reserved）

Protobuf 示例:
  // v1
  message User {
    int64 id = 1;
    string name = 2;
  }

  // v2 (兼容)
  message User {
    int64 id = 1;
    string name = 2;
    string email = 3;      // 新增字段（兼容）
    int32 age = 4 [default = 0];  // 新增字段（兼容）
  }
```

### 5.2 JSON Schema 兼容

```json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "type": "object",
  "properties": {
    "id": {"type": "integer"},
    "name": {"type": "string"},
    "email": {"type": "string", "default": ""},
    "age": {"type": "integer", "default": 0}
  },
  "required": ["id", "name"],
  "additionalProperties": true
}
```

## 七、最佳实践

1. **JSON 适合对外 API**，可读性好，跨语言支持广泛
2. **Protobuf 适合内部通信**，性能高、体积小、兼容性好
3. **序列化要向前兼容**，只能新增字段，不能删除
4. **大对象考虑压缩**，GZIP/Snappy 压缩进一步减小体积
5. **统一序列化格式**，避免系统中混用多种格式
6. **性能敏感场景使用 Protobuf 或 Kryo**
7. **对外接口使用 JSON**，降低接入成本
8. **序列化选择要考虑团队和生态**，不要只看性能数据
