# 链路追踪概述

## 一、为什么需要链路追踪

```yaml
微服务问题:
  - 调用链复杂，难以定位问题
  - 性能瓶颈难以发现
  - 服务依赖关系不清晰

链路追踪解决:
  - 全链路 TraceID 跟踪
  - 调用耗时分析
  - 服务依赖拓扑
  - 异常快速定位
```

## 二、核心概念

```
Trace: 完整的请求链路
├── Span: 一个操作（调用）
│   ├── TraceID: 全链路唯一标识
│   ├── SpanID: 当前操作标识
│   ├── ParentSpanID: 父操作标识
│   └── Duration: 操作耗时
```

## 三、主流方案

| 方案 | 特点 |
|------|------|
| SkyWalking | 国产，功能全面 |
| Zipkin | Twitter 开源，简单 |
| Jaeger | Uber 开源，云原生 |
| Pinpoint | 韩国，无侵入 |

## 四、注意事项

1. **SkyWalking 国内首选**
2. **TraceID 必须全链路传递**
3. **采样率要合理设置**
4. **异步场景要特别注意上下文传递**
5. **链路数据量大，需要存储策略**
