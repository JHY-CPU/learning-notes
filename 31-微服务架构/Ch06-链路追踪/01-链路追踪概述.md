# 链路追踪概述

## 一、为什么需要链路追踪

```yaml
微服务问题:
  - 调用链复杂，难以定位问题
  - 性能瓶颈难以发现
  - 服务依赖关系不清晰

链路追踪解决:
  - 全链路 TraceID 跟踪
  - 调用耗时分析
  - 服务依赖拓扑
  - 异常快速定位
```

## 二、核心概念

```
Trace: 完整的请求链路
├── Span: 一个操作（调用）
│   ├── TraceID: 全链路唯一标识
│   ├── SpanID: 当前操作标识
│   ├── ParentSpanID: 父操作标识
│   └── Duration: 操作耗时
```

## 三、主流方案

| 方案 | 特点 |
|------|------|
| SkyWalking | 国产，功能全面 |

## 四、Trace 与 Span 详细结构

```
Trace (一条完整请求链路):
├── TraceID: abc123 (全链路唯一)
├── Span 1 (网关接收请求)
│   ├── SpanID: 1
│   ├── ParentSpanID: null
│   ├── Service: api-gateway
│   ├── Operation: GET /api/users/1
│   ├── StartTime: 10:00:00.000
│   ├── Duration: 1250ms
│   └── Tags: {http.method=GET, http.status=200}
│       ├── Span 2 (用户服务)
│       │   ├── SpanID: 1.1
│       │   ├── ParentSpanID: 1
│       │   ├── Service: user-service
│       │   ├── Operation: UserController.getUser
│       │   └── Duration: 800ms
│       │       ├── Span 3 (MySQL 查询)
│       │       │   ├── SpanID: 1.1.1
│       │       │   ├── Duration: 50ms
│       │       │   └── Tags: {db.statement=SELECT * FROM users WHERE id=?}
│       │       └── Span 4 (Redis 查询)
│       │           ├── SpanID: 1.1.2
│       │           ├── Duration: 5ms
│       │           └── Tags: {redis.key=user:1}
│       └── Span 5 (订单服务)
│           ├── SpanID: 1.2
│           ├── Duration: 300ms
│           └── Tags: {http.method=GET, http.status=200}
```

## 五、OpenTelemetry 标准

```java
// OpenTelemetry - 云原生链路追踪标准
// 自动集成 Spring Boot
dependencies {
    implementation 'io.opentelemetry.instrumentation:opentelemetry-spring-boot-starter:2.4.0'
}

// 手动创建 Span
Tracer tracer = GlobalOpenTelemetry.getTracer("user-service");
Span span = tracer.spanBuilder("getUser").startSpan();
try (Scope scope = span.makeCurrent()) {
    User user = userRepository.findById(id);
    span.setAttribute("user.id", id);
    span.setAttribute("user.name", user.getName());
    return user;
} catch (Exception e) {
    span.setStatus(StatusCode.ERROR, e.getMessage());
    span.recordException(e);
    throw e;
} finally {
    span.end();
}
```

```yaml
# OpenTelemetry 配置
otel:
  service:
    name: user-service
  exporter:
    otlp:
      endpoint: http://otel-collector:4317
  traces:
    sampler: parentbased_traceidratio
    sampler.arg: 0.1   # 10% 采样率
```

## 六、注意事项

1. **SkyWalking 国内首选** - 功能全面，无侵入接入
2. **TraceID 必须全链路传递** - HTTP Header、MQ Header、线程池都要传递
3. **采样率要合理设置** - 低 QPS 100%，高 QPS 10-30%
4. **异步场景要特别注意上下文传递** - CompletableFuture、线程池需要手动传递
5. **链路数据量大，需要存储策略** - ES 集群 + 数据保留策略
6. **OpenTelemetry 是行业标准** - 避免厂商锁定，支持多后端
