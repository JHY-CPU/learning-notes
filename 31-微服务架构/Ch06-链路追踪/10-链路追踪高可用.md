# 链路追踪高可用

## 一、核心概念

链路追踪系统本身也需要高可用设计。如果追踪系统不可用，不应影响业务服务的正常运行。本节介绍 Agent 降级、采样策略、异步上报、存储高可用等方案。

### 1.1 高可用设计原则

```
原则               说明
──────────────────────────────────────
业务优先        追踪系统故障不影响业务
异步上报        追踪数据异步发送，不阻塞业务
采样降级        高负载时降低采样率
本地缓冲        网络不通时缓存到本地
最终一致        数据可能延迟到达，但最终完整
```

### 1.2 架构总览


### 2.2 Agent 自动降级

```java
public class AgentAutoDegrader {

    private volatile DegradationLevel level =
        DegradationLevel.NORMAL;

    /**
     * 根据系统负载自动调整
     */
    @Scheduled(fixedRate = 5000)
    public void checkAndDegrade() {
        double cpuUsage = getSystemCpuUsage();
        double memoryUsage = getSystemMemoryUsage();
        long queueSize = reportQueue.size();

        if (cpuUsage > 90 || memoryUsage > 85
                || queueSize > 10000) {
            level = DegradationLevel.HEAVY;
        } else if (cpuUsage > 70 || memoryUsage > 70
                || queueSize > 5000) {
            level = DegradationLevel.MEDIUM;
        } else {
            level = DegradationLevel.NORMAL;
        }
    }

    public boolean shouldSample() {
        return switch (level) {
            case NORMAL -> random.nextDouble() < 1.0;    // 100%
            case MEDIUM -> random.nextDouble() < 0.1;    // 10%
            case HEAVY -> random.nextDouble() < 0.01;    // 1%
        };
    }
}
```

## 三、采样策略

### 3.1 多种采样策略

```java
public interface SamplingStrategy {

    /**
     * 是否对当前 Trace 进行采样
     */
    boolean sample(TraceContext context);
}

// 固定比例采样
public class RateSampling implements SamplingStrategy {
    private final double rate; // 0.01 = 1%

    @Override
    public boolean sample(TraceContext context) {
        return ThreadLocalRandom.current()
            .nextDouble() < rate;
    }
}

// 基于规则的采样
public class RuleBasedSampling implements SamplingStrategy {
    private final Map<String, Double> rules;

    @Override
    public boolean sample(TraceContext context) {
        String path = context.getOperationName();
        double rate = rules.entrySet().stream()
            .filter(e -> path.startsWith(e.getKey()))
            .map(Map.Entry::getValue)
            .findFirst()
            .orElse(0.01); // 默认 1%
        return ThreadLocalRandom.current()
            .nextDouble() < rate;
    }
}

// 错误优先采样
public class ErrorPrioritySampling implements SamplingStrategy {
    @Override
    public boolean sample(TraceContext context) {
        // 错误请求 100% 采样
        if (context.hasError()) return true;
        // 正常请求 1% 采样
        return ThreadLocalRandom.current()
            .nextDouble() < 0.01;
    }
}
```

### 3.2 自适应采样

```java
@Component
public class AdaptiveSampling implements SamplingStrategy {

    private volatile double currentRate = 0.01;
    private final AtomicLong sampledCount = new AtomicLong();
    private final AtomicLong totalCount = new AtomicLong();

    /**
     * 目标: 每分钟最多采样 10000 条
     */
    @Scheduled(fixedRate = 60000)
    public void adjustRate() {
        long count = sampledCount.getAndSet(0);
        totalCount.set(0);

        if (count > 10000) {
            currentRate = currentRate * 0.8; // 降低
        } else if (count < 5000) {
            currentRate = Math.min(currentRate * 1.2, 1.0);
        }

        log.info("采样率调整为: {}", currentRate);
    }

    @Override
    public boolean sample(TraceContext context) {
        totalCount.incrementAndGet();
        if (context.hasError()) {
            sampledCount.incrementAndGet();
            return true;
        }
        boolean sampled =
            ThreadLocalRandom.current().nextDouble()
                < currentRate;
        if (sampled) sampledCount.incrementAndGet();
        return sampled;
    }
}
```

## 五、最佳实践

1. **OAP 至少 2 节点**，推荐 3 节点以上
2. **存储用 ES 集群**，多副本保证数据不丢失
3. **Agent 采样减少负载**，高 QPS 场景降低采样率
4. **监控 OAP 的健康状态**，配置多级告警
5. **数据定期清理**，避免存储爆炸
6. **追踪系统故障不影响业务**，Agent 必须异步上报
7. **本地缓冲兜底**，网络不通时缓存数据
8. **错误请求 100% 采样**，保证错误可追踪
