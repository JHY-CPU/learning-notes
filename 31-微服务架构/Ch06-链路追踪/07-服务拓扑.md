# 服务拓扑与日志聚合

## 一、服务拓扑

### 1.1 核心概念

服务拓扑图是通过链路追踪数据自动生成的服务依赖关系图。它展示了微服务之间的调用关系、调用频率、延迟和错误率，帮助团队理解系统架构、识别强弱依赖、发现循环依赖。

### 1.2 拓扑数据结构

```java
@Data
public class ServiceTopology {
    private List<ServiceNode> nodes;
    private List<ServiceEdge> edges;
}

@Data
public class ServiceNode {
    private String name;
    private String type;          // HTTP, gRPC, MQ, DB
    private long requestCount;    // 请求数
    private double avgLatency;    // 平均延迟
    private double errorRate;     // 错误率
    private String healthStatus;  // UP, DOWN, WARNING
}

@Data
public class ServiceEdge {
    private String source;        // 源服务
    private String target;        // 目标服务
    private long requestCount;    // 调用次数
    private double avgLatency;    // 平均延迟
    private double errorRate;     // 错误率
    private boolean isCritical;   // 是否强依赖
}
```

### 1.4 自定义拓扑分析

```java
@Service
public class TopologyAnalyzer {

    @Autowired
    private TraceStorage traceStorage;

    /**
     * 从 Span 数据构建拓扑
     */
    public ServiceTopology buildTopology(
            Duration timeRange) {
        long end = System.currentTimeMillis();
        long start = end - timeRange.toMillis();

        // 查询所有 Span
        List<Span> allSpans =
            traceStorage.querySpans(start, end);

        // 提取调用关系
        Map<String, ServiceEdge> edgeMap = new HashMap<>();

        for (Span span : allSpans) {
            String caller = span.getServiceName();
            for (Span child : span.getChildren()) {
                String callee = child.getServiceName();
                String key = caller + "->" + callee;

                edgeMap.computeIfAbsent(key,
                    k -> new ServiceEdge(caller, callee))
                    .increment(span.getDuration());
            }
        }

        return ServiceTopology.builder()
            .nodes(buildNodes(allSpans))
            .edges(new ArrayList<>(edgeMap.values()))
            .build();
    }

    /**
     * 检测循环依赖
     */
    public List<List<String>> detectCycles(
            ServiceTopology topology) {
        Map<String, Set<String>> adjList = new HashMap<>();
        for (ServiceEdge edge : topology.getEdges()) {
            adjList.computeIfAbsent(
                edge.getSource(), k -> new HashSet<>())
                .add(edge.getTarget());
        }

        List<List<String>> cycles = new ArrayList<>();
        Set<String> visited = new HashSet<>();
        Set<String> inStack = new HashSet<>();

        for (String node : adjList.keySet()) {
            if (!visited.contains(node)) {
                dfs(node, adjList, visited,
                    inStack, new ArrayList<>(), cycles);
            }
        }
        return cycles;
    }
}
```

### 2.2 日志中嵌入 TraceID

```java
@Component
public class TraceIdLogFilter extends Filter<ILoggingEvent> {

    @Override
    public void filter(ILoggingEvent event) {
        // 从 MDC 中获取 TraceID
        String traceId = MDC.get("traceId");
        String spanId = MDC.get("spanId");

        if (traceId != null) {
            // 将 TraceID 添加到日志消息中
            String message = String.format(
                "[traceId:%s spanId:%s] %s",
                traceId, spanId,
                event.getFormattedMessage());

            // 修改日志事件
            ((ch.qos.logback.classic.spi
                .ILoggingEvent) event)
                .setMessage(message);
        }
    }
}
```

### 2.4 关联日志与 Trace

```java
@Component
public class TraceLogCorrelation {

    @Autowired
    private ElasticsearchClient esClient;

    /**
     * 根据 TraceID 查询关联的所有日志
     */
    public List<LogEntry> getLogsByTraceId(
            String traceId) throws IOException {
        SearchResponse<LogEntry> response = esClient.search(
            s -> s.index("service-logs-*")
                .query(q -> q.term(
                    t -> t.field("mdc.traceId")
                          .value(traceId)))
                .size(1000)
                .sort(so -> so.field(
                    f -> f.field("timestamp")
                          .order(SortOrder.Asc))),
            LogEntry.class
        );

        return response.hits().hits().stream()
            .map(Hit::source)
            .collect(Collectors.toList());
    }
}
```

## 三、最佳实践

1. **拓扑图自动发现，无需手动配置**
2. **查看服务间调用频率**，识别热点依赖
3. **识别强依赖和弱依赖**，配置不同的保护策略
4. **发现循环依赖**，及时重构
5. **拓扑变更要有告警**，新增/移除依赖要通知
6. **日志中嵌入 TraceID**，支持链路级日志查询
7. **使用 JSON 格式输出日志**，便于 ELK 解析
8. **建立日志-链路-指标的关联体系**，统一排障
