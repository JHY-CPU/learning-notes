# 链路追踪性能优化

## 一、核心概念

链路追踪的性能开销是生产环境中必须考虑的问题。不合理的追踪配置可能导致服务性能下降，甚至引发故障。本节介绍如何在保证追踪效果的前提下，最小化性能开销。

### 1.1 性能开销来源

```
开销来源              占比       优化手段
──────────────────────────────────────────────
Span 创建/销毁        20%      对象池复用
TraceID 传递          10%      优化传递方式
数据序列化            15%      使用高效序列化
网络上报              35%      异步/批量发送
采样判断              5%       快速采样
上下文存储            15%      ThreadLocal 优化
```

## 二、采样率优化

### 2.1 基于规则的采样

```yaml
skywalking:
  agent:
    sampler:
      # 采样率 (0.01 = 1%)
      sample_rate: ${SW_AGENT_SAMPLE:0.01}
      # 错误请求 100% 采样
      force_sample_error: true
      # 基于端点的采样
      endpoint_rules:
        /api/health: 0.001       # 健康检查 0.1%
        /api/users: 0.05         # 用户接口 5%
        /api/payments: 1.0       # 支付接口 100%
```

### 2.2 动态采样

```java
@Component
public class DynamicSampler {

    private volatile double globalRate = 0.01;
    private final Map<String, Double> endpointRates =
        new ConcurrentHashMap<>();

    /**
     * 根据系统负载动态调整采样率
     */
    @Scheduled(fixedRate = 10000)
    public void adjustRate() {
        double cpuUsage = ManagementFactory
            .getOperatingSystemMXBean()
            .getSystemCpuLoad();

        if (cpuUsage > 0.8) {
            globalRate = Math.max(0.001, globalRate * 0.5);
        } else if (cpuUsage < 0.3) {
            globalRate = Math.min(1.0, globalRate * 2);
        }
    }

    public boolean sample(String endpoint) {
        double rate = endpointRates.getOrDefault(
            endpoint, globalRate);
        return ThreadLocalRandom.current()
            .nextDouble() < rate;
    }
}
```

## 三、异步上报优化

### 3.1 异步 Span 上报

```java
public class AsyncSpanReporter {

    private final BlockingQueue<Span> reportQueue =
        new LinkedBlockingQueue<>(10000);
    private final ExecutorService reporter =
        Executors.newFixedThreadPool(2);

    /**
     * 非阻塞上报 Span
     */
    public void report(Span span) {
        // 不阻塞业务线程
        boolean offered = reportQueue.offer(span);
        if (!offered) {
            // 队列满了，丢弃（保证业务优先）
            Metrics.counter("trace.dropped").increment();
        }
    }

    @PostConstruct
    public void startReporter() {
        reporter.submit(() -> {
            List<Span> batch = new ArrayList<>(100);
            while (!Thread.currentThread().isInterrupted()) {
                try {
                    Span span = reportQueue.poll(
                        100, TimeUnit.MILLISECONDS);
                    if (span != null) {
                        batch.add(span);
                    }

                    // 批量发送（达到 100 条或超时 1 秒）
                    if (batch.size() >= 100
                            || (!batch.isEmpty()
                                && span == null)) {
                        sendBatch(batch);
                        batch.clear();
                    }
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            }
        });
    }

    private void sendBatch(List<Span> spans) {
        try {
            // 批量序列化
            byte[] data = serialize(spans);
            // 批量发送
            grpcStub.collect(
                UpstreamMessage.newBuilder()
                    .setContent(
                        ByteString.copyFrom(data))
                    .build());
        } catch (Exception e) {
            log.warn("Span 批量发送失败", e);
            // 失败的写入本地文件
            writeToFile(spans);
        }
    }
}
```

## 四、内存优化

### 4.1 Span 对象池

```java
public class SpanPool {

    private final Queue<Span> pool =
        new ConcurrentLinkedQueue<>();

    public Span acquire() {
        Span span = pool.poll();
        if (span == null) {
            span = new Span();
        }
        return span;
    }

    public void release(Span span) {
        span.reset(); // 清空数据
        if (pool.size() < 1000) {
            pool.offer(span);
        }
    }
}
```

### 4.2 ThreadLocal 上下文优化

```java
public class TraceContext {

    private static final ThreadLocal<TraceContext> CONTEXT =
        ThreadLocal.withInitial(TraceContext::new);

    /**
     * 使用 ThreadLocal 存储 Trace 上下文
     * 避免参数传递开销
     */
    public static TraceContext current() {
        return CONTEXT.get();
    }

    /**
     * 及时清理，防止内存泄漏
     */
    public static void clear() {
        CONTEXT.remove();
    }
}

// 在请求处理完成后清理
@Component
public class TraceContextFilter extends OncePerRequestFilter {

    @Override
    protected void doFilterInternal(
            HttpServletRequest request,
            HttpServletResponse response,
            FilterChain chain)
            throws ServletException, IOException {
        try {
            chain.doFilter(request, response);
        } finally {
            TraceContext.clear(); // 必须清理
        }
    }
}
```

## 五、序列化优化

### 5.1 高效序列化

```java
public class SpanSerializer {

    /**
     * 使用 Protobuf 序列化 Span（比 JSON 快 3-5 倍）
     */
    public byte[] serialize(Span span) {
        SpanProto.Span protoSpan = SpanProto.Span.newBuilder()
            .setTraceId(span.getTraceId())
            .setSpanId(span.getSpanId())
            .setServiceName(span.getServiceName())
            .setOperationName(span.getOperationName())
            .setStartTime(span.getStartTime())
            .setDuration(span.getDuration())
            .build();
        return protoSpan.toByteArray();
    }

    /**
     * 批量序列化
     */
    public byte[] serializeBatch(List<Span> spans) {
        SpanProto.SpanList.Builder builder =
            SpanProto.SpanList.newBuilder();
        for (Span span : spans) {
            builder.addSpans(serializeSpan(span));
        }
        return builder.build().toByteArray();
    }
}
```

## 六、开销评估与监控

### 6.1 追踪开销评估

```java
@Component
public class TracingOverheadMonitor {

    private final AtomicLong totalSpanCreationTime =
        new AtomicLong();
    private final AtomicLong totalSpanCount =
        new AtomicLong();

    public void recordSpanCreation(long durationNs) {
        totalSpanCreationTime.addAndGet(durationNs);
        totalSpanCount.incrementAndGet();
    }

    /**
     * 计算追踪开销占比
     */
    public double getOverheadRatio() {
        long total = totalSpanCreationTime.get();
        long count = totalSpanCount.get();
        if (count == 0) return 0;
        // 平均每个 Span 创建耗时
        double avgNs = (double) total / count;
        // 假设平均请求处理 100ms
        return avgNs / 100_000_000;
    }
}
```

## 七、最佳实践

1. **采样率是最大的优化手段**，生产环境建议 1%-10%
2. **高 QPS 场景降低采样率**，保证业务性能
3. **异步上报不影响业务性能**，使用独立线程池
4. **批量发送减少网络开销**，合并多条 Span 一次发送
5. **数据定期清理节省存储**，ES 索引生命周期管理
6. **错误请求 100% 采样**，保证问题可追踪
7. **使用 ThreadLocal 传递上下文**，避免参数传递开销
8. **监控追踪系统自身的性能**，评估开销是否可接受
