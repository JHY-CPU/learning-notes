# 链路追踪最佳实践

## 一、实践清单

```yaml
配置:
  □ SkyWalking Agent 加载
  □ 采样率合理设置
  □ TraceID 全链路传递
  □ 日志关联 TraceID
  □ MQ 消息携带 TraceID

运维:
  □ OAP 高可用部署
  □ 存储用 ES 集群
  □ 数据定期清理
  □ 告警规则配置
  □ 监控 OAP 健康
```

## 二、常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 链路断裂 | TraceID 丢失 | 检查 Header 传递 |
| 性能影响 | 采样率太高 | 降低采样率 |
| 存储不足 | 数据未清理 | 配置生命周期 |

## 三、工作原理

链路追踪最佳实践围绕全链路覆盖、性能平衡、可观测整合三个原则。全链路覆盖要求 TraceID 在 HTTP 调用、MQ 消息、定时任务、跨线程等所有场景都能传递。性能平衡通过合理采样率实现：低 QPS 服务设 100%，高 QPS 服务设 10-30%。可观测整合将链路追踪与日志（通过 TraceID 关联）、监控（指标聚合）、告警（异常通知）打通，形成完整的可观测性体系。

## 四、优缺点

**优点：**
- 全链路追踪极大提高问题定位效率
- 日志关联 TraceID 实现日志和链路互相跳转
- 告警基于真实链路数据，准确度高

**缺点：**
- 全链路覆盖需要对所有中间件做适配
- 采样率与存储成本需要权衡
- 链路追踪系统本身需要监控保障

## 五、采样率策略

```yaml
# 自适应采样策略
采样率配置:
  # 低 QPS 服务 - 100% 采样
  user-service: 1.0        # QPS < 100
  order-service: 1.0       # QPS < 100

  # 中 QPS 服务 - 30% 采样
  product-service: 0.3     # QPS 100-1000

  # 高 QPS 服务 - 10% 采样
  search-service: 0.1      # QPS 1000-10000
  gateway: 0.05            # QPS > 10000

  # 错误请求 - 100% 采样
  # Head-based sampling: 错误的请求全部保留
```

```java
// OpenTelemetry 自适应采样
@Component
public class CustomSampler implements Sampler {
    @Override
    public SamplingResult shouldSample(Context parentContext, TraceId traceId,
            String name, SpanKind spanKind, Attributes attributes,
            List<LinkData> parentLinks) {
        // 错误请求全部采样
        if (attributes.get(AttributeKey.stringKey("error")) != null) {
            return SamplingResult.create(SamplingDecision.RECORD_AND_SAMPLE);
        }
        // 按比例采样
        if (random.nextDouble() < samplingRate) {
            return SamplingResult.create(SamplingDecision.RECORD_AND_SAMPLE);
        }
        return SamplingResult.create(SamplingDecision.DROP);
    }
}
```

## 六、可观测性三支柱整合

```
可观测性三支柱:
├── Metrics (指标) - Prometheus + Grafana
│   └── QPS、延迟、错误率、饱和度
├── Traces (链路) - SkyWalking/Jaeger
│   └── 调用链、耗时分布、依赖拓扑
└── Logs (日志) - ELK/Loki
    └── 通过 TraceID 关联日志

整合方式:
日志中输出 TraceID
  → 在 SkyWalking 中点击 TraceID
  → 跳转到 ELK 查看详细日志

Prometheus 指标异常
  → 找到异常时间点
  → SkyWalking 查看该时间点的慢请求链路
```

## 七、常见陷阱

1. **链路在异步调用处断裂** - MQ、线程池等场景未传递 TraceID
2. **采样率过高导致性能下降** - 高 QPS 场景需要降低采样率
3. **存储不设保留策略** - 磁盘空间耗尽后链路系统不可用
4. **链路追踪系统本身无监控** - 系统故障后无人知晓
5. **日志与链路未关联** - 排查问题时无法跳转
6. **没有 SLO 指标** - 没有明确的服务质量目标
