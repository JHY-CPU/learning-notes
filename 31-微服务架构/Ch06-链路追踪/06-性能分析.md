# 链路追踪性能分析

## 一、核心概念

链路追踪不仅能定位问题，还能进行性能分析。通过分析 Span 数据，可以识别慢请求、定位性能瓶颈、分析调用链路中的耗时分布，帮助团队持续优化系统性能。

### 1.1 Span 数据模型

```
一个 Span 包含:
  - traceId: 链路唯一标识
  - spanId: 当前 Span 标识
  - parentSpanId: 父 Span 标识
  - serviceName: 服务名称
  - operationName: 操作名称
  - startTime: 开始时间
  - duration: 耗时（微秒）
  - tags: 标签（HTTP method、status code 等）
  - logs: 日志事件
```

### 1.2 链路分析维度

```
分析维度           关注点
──────────────────────────────
延迟分析      P50/P95/P99 延迟
瓶颈分析      最慢的 Span/服务
流量分析      QPS、并发量
错误分析      错误率、错误分布
依赖分析      调用关系、强弱依赖
```

### 2.2 自定义 Span 分析

```java
@Component
public class SpanAnalyzer {

    @Autowired
    private StorageClient storageClient;

    /**
     * 分析慢请求 Span
     */
    public List<SlowSpan> analyzeSlowSpans(
            String serviceName,
            Duration timeRange,
            int topN) {
        // 查询指定时间范围内的 Span
        List<Span> spans = storageClient.querySpans(
            serviceName,
            System.currentTimeMillis() - timeRange.toMillis(),
            System.currentTimeMillis()
        );

        // 按耗时排序
        return spans.stream()
            .filter(s -> s.getDuration() > 500_000) // > 500ms
            .sorted(Comparator.comparingLong(
                Span::getDuration).reversed())
            .limit(topN)
            .map(this::toSlowSpan)
            .collect(Collectors.toList());
    }

    /**
     * 分析 Span 耗时分布
     */
    public DurationDistribution analyzeDistribution(
            List<Span> spans) {
        LongSummaryStatistics stats = spans.stream()
            .mapToLong(Span::getDuration)
            .summaryStatistics();

        return DurationDistribution.builder()
            .p50(percentile(spans, 50))
            .p75(percentile(spans, 75))
            .p90(percentile(spans, 90))
            .p95(percentile(spans, 95))
            .p99(percentile(spans, 99))
            .max(stats.getMax())
            .avg(stats.getAverage())
            .build();
    }

    private long percentile(List<Span> spans, int p) {
        List<Long> sorted = spans.stream()
            .map(Span::getDuration)
            .sorted()
            .collect(Collectors.toList());
        int index = (int) Math.ceil(p / 100.0 * sorted.size())
            - 1;
        return sorted.get(Math.max(0, index));
    }
}
```

## 三、慢请求定位

### 3.1 基于 TraceID 的慢请求分析

```java
@Service
public class SlowTraceAnalyzer {

    /**
     * 分析一条慢 Trace 的详细 Span
     */
    public TraceAnalysisResult analyze(String traceId) {
        List<Span> spans = traceStorage.getSpans(traceId);

        // 构建调用树
        Span rootSpan = buildSpanTree(spans);

        // 识别最慢的 Span
        Span slowestSpan = findSlowestSpan(rootSpan);

        // 识别关键路径（最长路径）
        List<Span> criticalPath = findCriticalPath(rootSpan);

        return TraceAnalysisResult.builder()
            .traceId(traceId)
            .totalDuration(rootSpan.getDuration())
            .spanCount(spans.size())
            .slowestSpan(slowestSpan)
            .criticalPath(criticalPath)
            .bottleneckService(
                slowestSpan.getServiceName())
            .build();
    }

    /**
     * 关键路径分析
     */
    private List<Span> findCriticalPath(Span root) {
        List<Span> path = new ArrayList<>();
        Span current = root;
        while (current != null) {
            path.add(current);
            // 找到最慢的子 Span
            current = current.getChildren().stream()
                .max(Comparator.comparingLong(
                    Span::getDuration))
                .orElse(null);
        }
        return path;
    }
}
```

### 3.2 慢查询识别

```java
@Component
public class SlowQueryAnalyzer {

    /**
     * 从 Span 中识别慢 SQL
     */
    public List<SlowSqlInfo> analyzeSlowSql(
            String serviceName, Duration timeRange) {
        List<Span> dbSpans = traceStorage.querySpans(
            serviceName,
            Map.of("db.type", "sql"),
            timeRange
        );

        return dbSpans.stream()
            .filter(s -> s.getDuration() > 100_000) // > 100ms
            .map(span -> {
                String sql = span.getTags().get("db.statement");
                return SlowSqlInfo.builder()
                    .sql(sql)
                    .duration(span.getDuration())
                    .traceId(span.getTraceId())
                    .spanId(span.getSpanId())
                    .build();
            })
            .collect(Collectors.toList());
    }
}
```

## 四、性能趋势分析

### 4.1 延迟趋势监控

```java
@Component
public class LatencyTrendAnalyzer {

    /**
     * 分析延迟趋势，检测性能退化
     */
    public TrendAnalysis analyzeTrend(
            String serviceName,
            Duration window,
            Duration interval) {
        List<TimeSeriesPoint> p99Values = new ArrayList<>();

        long now = System.currentTimeMillis();
        long start = now - window.toMillis();

        for (long t = start; t < now;
                t += interval.toMillis()) {
            List<Span> spans = traceStorage.querySpans(
                serviceName, t, t + interval.toMillis());
            long p99 = percentile(spans, 99);
            p99Values.add(new TimeSeriesPoint(t, p99));
        }

        // 线性回归检测趋势
        double slope = linearRegression(p99Values);

        return TrendAnalysis.builder()
            .serviceName(serviceName)
            .points(p99Values)
            .slope(slope)
            .isDegrading(slope > 0.1) // 斜率 > 0.1 表示恶化
            .build();
    }
}
```

## 五、最佳实践

1. **关注 P99 延迟而非平均值**，平均值容易掩盖长尾问题
2. **慢查询要重点优化**，数据库通常是性能瓶颈
3. **服务依赖耗时发现瓶颈**，识别调用链中最慢的环节
4. **定期分析性能趋势**，及时发现性能退化
5. **使用关键路径分析**，聚焦最影响总耗时的路径
6. **建立性能基线**，新版本与基线对比评估性能变化
7. **自动化性能分析**，定期生成性能报告
8. **结合业务分析**，区分正常慢和异常慢
