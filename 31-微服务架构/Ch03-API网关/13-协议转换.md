# 协议转换

## 一、核心概念

在微服务架构中，不同服务可能使用不同的通信协议（HTTP/REST、gRPC、WebSocket、Dubbo 等）。API 网关作为统一入口，需要支持协议转换，让外部客户端使用统一的 HTTP/REST 接口访问内部各种协议的服务。

### 1.1 协议转换场景

```
场景一：HTTP → gRPC
  外部客户端 → [HTTP/JSON] → 网关 → [gRPC/Protobuf] → 内部服务

场景二：HTTP → WebSocket
  外部客户端 → [HTTP] → 网关 → [WebSocket] → 实时推送服务

场景三：REST → GraphQL
  外部客户端 → [REST] → 网关 → [GraphQL] → 聚合服务

场景四：HTTP → Dubbo
  外部客户端 → [HTTP] → 网关 → [Dubbo] → Java 服务
```

## 二、HTTP 到 gRPC 转换

### 2.1 gRPC 服务定义

```protobuf
syntax = "proto3";
package user;

service UserService {
  rpc GetUser(GetUserRequest) returns (UserResponse);
  rpc ListUsers(ListUsersRequest) returns (ListUsersResponse);
  rpc CreateUser(CreateUserRequest) returns (UserResponse);
}

message GetUserRequest {
  int64 id = 1;
}

message UserResponse {
  int64 id = 1;
  string name = 2;
  string email = 3;
  int32 age = 4;
}
```

### 2.2 Spring Cloud Gateway gRPC 代理

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: grpc-user-service
          uri: grpc://user-grpc-service:9090
          predicates:
            - Path=/api/users/**
          filters:
            - name: GrpcToFriendlyHttp
              args:
                protoDescriptor: classpath:user.pb
                protoFile: classpath:user.proto
```

### 2.3 手动 HTTP-gRPC 转换

```java
@RestController
@RequestMapping("/api/v1/users")
public class HttpToGrpcController {

    private final ManagedChannel channel =
        ManagedChannelBuilder
            .forAddress("user-grpc-service", 9090)
            .usePlaintext()
            .build();

    private final UserServiceGrpc.UserServiceBlockingStub stub =
        UserServiceGrpc.newBlockingStub(channel);

    @GetMapping("/{id}")
    public ResponseEntity<Map<String, Object>> getUser(
            @PathVariable Long id) {
        // HTTP 请求 → gRPC 请求
        GetUserRequest grpcRequest = GetUserRequest.newBuilder()
            .setId(id)
            .build();

        UserResponse grpcResponse = stub.getUser(grpcRequest);

        // gRPC 响应 → HTTP JSON 响应
        Map<String, Object> result = new HashMap<>();
        result.put("id", grpcResponse.getId());
        result.put("name", grpcResponse.getName());
        result.put("email", grpcResponse.getEmail());
        result.put("age", grpcResponse.getAge());

        return ResponseEntity.ok(result);
    }

    @PostMapping
    public ResponseEntity<Map<String, Object>> createUser(
            @RequestBody Map<String, Object> body) {
        CreateUserRequest grpcRequest =
            CreateUserRequest.newBuilder()
                .setName((String) body.get("name"))
                .setEmail((String) body.get("email"))
                .setAge((Integer) body.get("age"))
                .build();

        UserResponse response = stub.createUser(grpcRequest);
        return ResponseEntity.status(HttpStatus.CREATED)
            .body(toMap(response));
    }
}
```

## 三、HTTP 到 Dubbo 转换

### 3.1 Spring Cloud Gateway 整合 Dubbo

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: dubbo-user-service
          uri: dubbo://127.0.0.1:20880
          predicates:
            - Path=/api/dubbo/users/**
          filters:
            - StripPrefix=3
```

```java
// Dubbo 接口
public interface UserService {
    UserDTO getUser(Long id);
    List<UserDTO> listUsers(UserQuery query);
}

// HTTP 到 Dubbo 的转换
@RestController
public class HttpToDubboController {

    @Reference(version = "1.0.0")
    private UserService userService;

    @GetMapping("/users/{id}")
    public UserDTO getUser(@PathVariable Long id) {
        return userService.getUser(id);
    }
}
```

## 四、REST 到 GraphQL 转换

### 4.1 网关层 REST-to-GraphQL 适配

```java
@RestController
public class RestToGraphQLController {

    @Autowired
    private GraphQL graphQL;

    @GetMapping("/rest/users/{id}")
    public Map<String, Object> getUserViaREST(
            @PathVariable String id) {
        // 将 REST 请求转换为 GraphQL 查询
        String query = """
            query {
                user(id: "%s") {
                    id
                    name
                    email
                    orders {
                        id
                        amount
                    }
                }
            }
            """.formatted(id);

        ExecutionResult result =
            graphQL.execute(query);
        Map<String, Object> data = result.getData();
        return ((Map<String, Object>) data.get("user"));
    }
}
```

## 五、协议转换性能优化

### 5.1 连接池管理

```java
@Configuration
public class GrpcChannelConfig {

    @Bean(destroyMethod = "shutdown")
    public ManagedChannel userGrpcChannel() {
        return ManagedChannelBuilder
            .forAddress("user-service", 9090)
            .usePlaintext()
            .maxInboundMessageSize(4 * 1024 * 1024)
            .keepAliveTime(30, TimeUnit.SECONDS)
            .keepAliveTimeout(5, TimeUnit.SECONDS)
            .keepAliveWithoutCalls(true)
            .build();
    }

    @Bean
    public UserServiceGrpc.UserServiceBlockingStub
            userStub(ManagedChannel channel) {
        return UserServiceGrpc.newBlockingStub(channel)
            .withDeadlineAfter(3, TimeUnit.SECONDS);
    }
}
```

### 5.2 序列化性能对比

```
转换方式              延迟(μs)    吞吐(QPS)
─────────────────────────────────────────
HTTP JSON → gRPC     ~50         ~20000
HTTP JSON → Dubbo    ~30         ~30000
REST → GraphQL       ~80         ~12000
HTTP → WebSocket      ~10         ~50000
```

## 六、最佳实践

1. **网关可以做协议转换**，但要注意性能开销
2. **gRPC 需要 HTTP/2 支持**，确保网关底层支持
3. **保持协议一致性**，内部尽量统一协议减少转换
4. **连接池要合理配置**，gRPC 和 Dubbo 都需要连接复用
5. **协议转换要做超时控制**，避免后端阻塞影响网关
6. **转换失败要做降级处理**，返回友好的错误信息
7. **监控协议转换的延迟和错误率**
8. **敏感数据在协议转换时注意脱敏**
