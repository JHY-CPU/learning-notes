# 日志与监控

## 一、请求日志

```java
@Component
@Order(-1)
public class LoggingFilter implements GlobalFilter {
    @Override
    public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {
        long start = System.currentTimeMillis();
        String traceId = UUID.randomUUID().toString();

        exchange.getRequest().mutate().header("X-Trace-Id", traceId);

        return chain.filter(exchange).then(Mono.fromRunnable(() -> {
            long duration = System.currentTimeMillis() - start;
            log.info("[{}] {} {} {}ms",
                traceId,
                exchange.getRequest().getMethod(),
                exchange.getRequest().getURI(),
                duration);
        }));
    }
}
```

## 三、完整日志过滤器

```java
@Component
@Order(-1)
public class LoggingFilter implements GlobalFilter {
    private static final Logger accessLog = LoggerFactory.getLogger("ACCESS_LOG");

    @Override
    public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {
        long startTime = System.currentTimeMillis();
        ServerHttpRequest request = exchange.getRequest();

        // 生成或透传 TraceID
        String traceId = Optional.ofNullable(request.getHeaders().getFirst("X-Trace-Id"))
            .filter(s -> !s.isEmpty())
            .orElse(UUID.randomUUID().toString().replace("-", ""));

        // 注入到请求头，传递给下游服务
        ServerHttpRequest mutatedRequest = request.mutate()
            .header("X-Trace-Id", traceId)
            .build();
        exchange.getAttributes().put("X-Trace-Id", traceId);

        return chain.filter(exchange.mutate().request(mutatedRequest).build())
            .then(Mono.fromRunnable(() -> {
                long duration = System.currentTimeMillis() - startTime;
                ServerHttpResponse response = exchange.getResponse();
                String clientIp = getClientIp(request);

                accessLog.info("{}|{}|{}|{}|{}|{}|{}ms",
                    traceId,
                    clientIp,
                    request.getMethod(),
                    request.getURI(),
                    response.getStatusCode().value(),
                    request.getHeaders().getFirst("User-Agent"),
                    duration);
            }));
    }

    private String getClientIp(ServerHttpRequest request) {
        String xff = request.getHeaders().getFirst("X-Forwarded-For");
        if (xff != null) return xff.split(",")[0].trim();
        return request.getRemoteAddress().getAddress().getHostAddress();
    }
}
```

## 四、Prometheus + Grafana 监控

```yaml
# Spring Boot Actuator 暴露指标
management:
  endpoints:
    web:
      exposure:
        include: prometheus,health,metrics,info
  metrics:
    tags:
      application: api-gateway
    distribution:
      percentiles-histogram:
        http.server.requests: true
      slo:
        http.server.requests: 100ms, 500ms, 1s
```

```yaml
# Prometheus 告警规则
groups:
  - name: gateway-alerts
    rules:
      - alert: GatewayHighLatency
        expr: histogram_quantile(0.99, rate(http_server_requests_seconds_bucket[5m])) > 1
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "网关 P99 延迟超过 1 秒"

      - alert: GatewayHighErrorRate
        expr: rate(http_server_requests_seconds_count{status=~"5.."}[5m]) / rate(http_server_requests_seconds_count[5m]) > 0.01
        for: 3m
        labels:
          severity: critical
        annotations:
          summary: "网关 5xx 错误率超过 1%"
```

## 五、关键监控指标

| 指标 | 说明 | 告警阈值 |
|------|------|---------|
| QPS | 每秒请求数 | 环比下降 50% |
| P50/P99 延迟 | 响应延迟分位 | P99 > 1s |
| 错误率 | 4xx/5xx 比例 | > 1% |
| 网关处理时间 | 网关自身耗时 | > 10ms |
| 活跃连接数 | 当前连接数 | > 80% 上限 |
| 限流触发次数 | 被限流的请求 | 持续增长 |

## 六、注意事项

1. **每个请求都要有 TraceID** - 用于跨服务链路追踪
2. **记录请求耗时和状态码** - 至少记录 method、path、status、duration
3. **监控网关的 QPS 和延迟** - Prometheus + Grafana 实时监控
4. **日志异步写入不影响性能** - 使用 Logback AsyncAppender
5. **敏感信息要脱敏** - Token、密码等字段不要记录到日志
6. **日志采样** - 高流量场景下按比例采样，避免日志量过大
