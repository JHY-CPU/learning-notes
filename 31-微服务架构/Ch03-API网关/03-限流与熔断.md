# 限流与熔断

## 一、限流策略

```java
// Sentinel 限流
@SentinelResource(value = "getUser", blockHandler = "getUserBlockHandler")
public User getUser(Long id) {
    return userService.getUser(id);
}

public User getUserBlockHandler(Long id, BlockException ex) {
    return User.defaultUser(id);
}
```

```yaml
# 限流规则
spring:
  cloud:
    sentinel:
      datasource:
        flow:
          nacos:
            server-addr: localhost:8848
            dataId: flow-rules
            groupId: SENTINEL_GROUP
```

## 二、熔断策略

```java
// Resilience4j 熔断
@CircuitBreaker(name = "userService", fallbackMethod = "getUserFallback")
public User getUser(Long id) {
    return userService.getUser(id);
}

public User getUserFallback(Long id, Throwable t) {
    return User.defaultUser(id);
}
```

## 三、限流算法

| 算法 | 说明 | 特点 |
|------|------|------|
| 令牌桶 | 定时添加令牌 | 允许突发 |
| 漏桶 | 固定速率流出 | 平滑流量 |
| 计数器 | 固定窗口计数 | 简单 |
| 滑动窗口 | 滑动窗口计数 | 精确 |

## 四、注意事项

1. **限流在网关层统一配置**
2. **熔断阈值要根据实际调优**
3. **降级逻辑要保证核心功能可用**
4. **限流规则要支持动态调整**
5. **监控限流和熔断的触发情况**
