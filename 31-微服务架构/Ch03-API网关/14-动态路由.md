# 动态路由

## 一、核心概念

动态路由是指路由规则可以在运行时动态更新，无需重启网关。在微服务架构中，服务实例频繁上线和下线，灰度发布、蓝绿部署等场景也需要动态调整路由规则。动态路由是 API 网关的核心能力之一。

### 1.1 动态路由 vs 静态路由

```
静态路由（配置文件）:
  优点：简单、启动即加载
  缺点：修改需要重启、不适应服务弹性伸缩

动态路由（配置中心/数据库）:
  优点：运行时更新、支持灰度和蓝绿部署
  缺点：实现复杂、需要考虑一致性
```

### 1.2 动态路由架构

```
┌───────────────┐     ┌───────────────┐
│  配置中心      │     │  管理控制台    │
│ (Nacos/Apollo)│◄────│  (路由管理)    │

┌───────────────────────────────────────┐

│  │         RouteDefinitionLocator  │ │

└───────────────────────────────────────┘
```

### 2.2 动态路由加载器

```java
@Component
public class NacosRouteDefinitionRepository
        implements RouteDefinitionRepository {

    @Autowired
    private ConfigService configService;

    private static final String DATA_ID = "gateway-routes";
    private static final String GROUP = "DEFAULT_GROUP";

    /**
     * 启动时加载路由配置
     */
    @PostConstruct
    public void init() throws NacosException {
        // 监听配置变更
        configService.addListener(DATA_ID, GROUP,
            new Listener() {
                @Override
                public void receiveConfigInfo(String config) {
                    log.info("路由配置变更: {}", config);
                    // 发布路由刷新事件
                    publisher.publishEvent(
                        new RefreshRoutesEvent(this));
                }

                @Override
                public Executor getExecutor() {
                    return null;
                }
            });
    }

    @Override
    public Flux<RouteDefinition> getRouteDefinitions() {
        try {
            String config =
                configService.getConfig(DATA_ID, GROUP, 5000);
            List<RouteDefinition> routes =
                JSON.parseArray(config, RouteDefinition.class);
            return Flux.fromIterable(routes);
        } catch (Exception e) {
            log.error("加载路由配置失败", e);
            return Flux.empty();
        }
    }
}
```

## 三、基于数据库的动态路由

### 3.1 路由表设计

```sql
CREATE TABLE gateway_routes (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    route_id VARCHAR(100) NOT NULL UNIQUE,
    uri VARCHAR(255) NOT NULL,
    predicates TEXT NOT NULL,
    filters TEXT,
    route_order INT DEFAULT 0,
    status TINYINT DEFAULT 1,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
        ON UPDATE CURRENT_TIMESTAMP
);

-- 示例数据
INSERT INTO gateway_routes
    (route_id, uri, predicates, filters, route_order)
VALUES
    ('user-service', 'lb://user-service',
     '[{"name":"Path","args":{"patterns":"/api/users/**"}}]',
     '[{"name":"StripPrefix","args":{"parts":"2"}}]',
     1);
```

### 3.2 数据库路由加载器

```java
@Component
public class DatabaseRouteDefinitionRepository
        implements RouteDefinitionRepository {

    @Autowired
    private JdbcTemplate jdbcTemplate;

    @Override
    public Flux<RouteDefinition> getRouteDefinitions() {
        String sql =
            "SELECT * FROM gateway_routes WHERE status = 1 "
            + "ORDER BY route_order";
        List<RouteDefinition> routes =
            jdbcTemplate.query(sql, this::mapToRouteDefinition);
        return Flux.fromIterable(routes);
    }

    /**
     * 通过管理 API 刷新路由
     */
    @PostMapping("/routes/refresh")
    public String refreshRoutes() {
        publisher.publishEvent(new RefreshRoutesEvent(this));
        return "Routes refreshed";
    }
}
```

## 四、灰度路由

### 4.1 基于 Header 的灰度路由

```java
@Component
public class GrayRouteFilter implements GlobalFilter, Ordered {

    @Override
    public Mono<Void> filter(ServerWebExchange exchange,
                              GatewayFilterChain chain) {
        String grayFlag = exchange.getRequest().getHeaders()
            .getFirst("X-Gray-Version");

        if ("v2".equals(grayFlag)) {
            // 修改路由目标到灰度版本
            exchange.getAttributes().put(
                ServerWebExchangeUtils.GATEWAY_REQUEST_URL_ATTR,
                URI.create("lb://user-service-v2")
            );
        }

        return chain.filter(exchange);
    }

    @Override
    public int getOrder() {
        return -100;
    }
}
```

### 4.2 基于权重的灰度路由

```java
@Component
public class WeightedGrayFilter implements GlobalFilter {

    private final Random random = new Random();

    @Override
    public Mono<Void> filter(ServerWebExchange exchange,
                              GatewayFilterChain chain) {
        // 10% 流量到灰度版本
        if (random.nextInt(100) < 10) {
            exchange.getAttributes().put(
                ServerWebExchangeUtils.GATEWAY_REQUEST_URL_ATTR,
                URI.create("lb://user-service-gray")
            );
        }
        return chain.filter(exchange);
    }
}
```

## 五、路由管理 API

### 5.1 路由管理 REST 接口

```java
@RestController
@RequestMapping("/admin/routes")
public class RouteAdminController {

    @Autowired
    private RouteDefinitionWriter routeWriter;

    @Autowired
    private RouteDefinitionLocator routeLocator;

    @GetMapping
    public Flux<RouteDefinition> listRoutes() {
        return routeLocator.getRouteDefinitions();
    }

    @PostMapping
    public String addRoute(
            @RequestBody RouteDefinition route) {
        routeWriter.save(Mono.just(route)).subscribe();
        publisher.publishEvent(new RefreshRoutesEvent(this));
        return "Route added: " + route.getId();
    }

    @DeleteMapping("/{routeId}")
    public String deleteRoute(
            @PathVariable String routeId) {
        routeWriter.delete(Mono.just(routeId)).subscribe();
        publisher.publishEvent(new RefreshRoutesEvent(this));
        return "Route deleted: " + routeId;
    }
}
```

## 六、最佳实践

1. **动态路由避免重启网关**，使用配置中心管理路由
2. **路由配置存储在配置中心或数据库中**，支持版本管理
3. **路由变更要实时生效**，通过监听机制触发刷新
4. **路由规则要版本管理**，支持快速回滚
5. **灰度发布依赖动态路由**，实现流量精细调度
6. **路由管理需要鉴权**，防止未授权修改
7. **路由变更要有审计日志**，记录谁在什么时候做了什么修改
8. **定期清理无效路由**，保持路由表整洁
