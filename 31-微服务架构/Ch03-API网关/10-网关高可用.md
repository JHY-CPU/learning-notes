# 网关高可用

## 一、核心概念

API 网关是所有外部流量的入口，其可用性直接影响整个系统的可用性。网关高可用设计需要从部署架构、故障转移、流量调度、容灾降级等多个维度考虑。

### 1.1 高可用目标

- **无单点故障**：网关至少 2 节点，推荐 3+ 节点
- **水平扩展**：根据流量弹性扩缩容
- **故障自动转移**：节点故障时流量自动切换
- **优雅降级**：极端情况下保证核心 API 可用

## 三、故障转移机制

### 3.1 Nginx 健康检查与故障转移

```nginx
upstream gateway_cluster {
    # 健康检查
    server gateway-1:8080 weight=5;
    server gateway-2:8080 weight=5;
    server gateway-3:8080 weight=5 backup;

    # 最大失败次数和超时
    # (nginx plus 商业版支持主动健康检查)
}

server {
    listen 80;
    location / {
        proxy_pass http://gateway_cluster;
        proxy_connect_timeout 3s;
        proxy_read_timeout 30s;
        proxy_next_upstream error timeout http_502 http_503;
        proxy_next_upstream_tries 3;
    }
}
```

### 3.2 Spring Cloud Gateway 优雅停机

```yaml
# 优雅停机配置
server:
  shutdown: graceful

spring:
  lifecycle:
    timeout-per-shutdown-phase: 30s

# 网关健康检查
management:
  endpoint:
    health:
      show-details: always
      probes:
        enabled: true
  endpoints:
    web:
      exposure:
        include: health,info,prometheus
```

```java
@Component
public class GracefulShutdownFilter implements GlobalFilter {

    private volatile boolean shuttingDown = false;

    @PreDestroy
    public void shutdown() {
        shuttingDown = true;
        // 等待正在处理的请求完成
        try {
            Thread.sleep(25000);
        } catch (InterruptedException ignored) {}
    }

    @Override
    public Mono<Void> filter(ServerWebExchange exchange,
                              GatewayFilterChain chain) {
        if (shuttingDown) {
            exchange.getResponse().setStatusCode(
                HttpStatus.SERVICE_UNAVAILABLE);
            return exchange.getResponse().setComplete();
        }
        return chain.filter(exchange);
    }
}
```

### 3.3 熔断降级

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: user-service
          uri: lb://user-service
          predicates:
            - Path=/api/users/**
          filters:
            - name: CircuitBreaker
              args:
                name: user-service-cb
                fallbackUri: forward:/fallback/users
```

```java
@RestController
@RequestMapping("/fallback")
public class FallbackController {

    @GetMapping("/users/{id}")
    public ResponseEntity<Map<String, Object>> userFallback(
            @PathVariable String id) {
        Map<String, Object> fallback = new HashMap<>();
        fallback.put("id", id);
        fallback.put("name", "默认用户");
        fallback.put("fallback", true);
        fallback.put("message", "服务暂不可用，请稍后重试");
        return ResponseEntity.status(HttpStatus.OK)
            .body(fallback);
    }
}
```

## 四、流量调度策略

### 4.1 权重路由

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: user-service-weighted
          uri: lb://user-service
          predicates:
            - Path=/api/users/**
          metadata:
            weight:
              group1: 90  # 90% 流量到 v1
              group2: 10  # 10% 流量到 v2
```

### 4.2 就近访问

```yaml
spring:
  cloud:
    nacos:
      discovery:
        cluster-name: cn-east
        metadata:
          region: east-china
```

```java
@Component
public class NearByLoadBalancer implements ReactorServiceInstanceLoadBalancer {

    @Override
    public Mono<Response<ServiceInstance>> choose(
            Request request) {
        String localRegion = getLocalRegion();
        List<ServiceInstance> instances =
            discoveryClient.getInstances(serviceId);

        // 优先选择同区域实例
        List<ServiceInstance> sameRegion = instances.stream()
            .filter(i -> localRegion.equals(
                i.getMetadata().get("region")))
            .collect(Collectors.toList());

        if (!sameRegion.isEmpty()) {
            return select(sameRegion);
        }
        return select(instances); // 降级到全量
    }
}
```

## 五、监控与自动扩缩容

### 5.1 关键监控指标

| 指标 | 说明 | 告警阈值 |
|------|------|---------|
| QPS | 每秒请求数 | 突增 > 200% |
| P99 延迟 | 99 分位响应时间 | > 2s |
| 错误率 | 5xx 错误比例 | > 1% |
| 活跃连接数 | 当前连接数 | > 80% 上限 |
| CPU/内存 | 网关实例资源 | > 80% |

## 六、最佳实践

1. **网关至少 2 节点部署**，推荐 3+ 节点
2. **前置负载均衡器**（Nginx/云 LB），不要让网关直接暴露
3. **网关保持无状态**，可水平扩展
4. **配置中心管理路由配置**，支持动态更新
5. **监控网关健康状态**，配置多级告警
6. **实现优雅停机**，避免请求中断
7. **配置熔断降级**，保护后端服务
8. **定期演练故障转移**，确保方案有效
