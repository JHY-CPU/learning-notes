# API 网关概述

## 一、什么是 API 网关

API 网关是微服务架构的**统一入口**，负责请求路由、认证授权、限流熔断、日志监控等横切关注点。

```
客户端 → API Gateway → 微服务A
                    → 微服务B
                    → 微服务C
```

## 二、核心功能

| 功能 | 说明 |
|------|------|
| 请求路由 | 将请求转发到对应服务 |
| 认证授权 | 统一身份验证 |
| 限流熔断 | 流量控制和故障保护 |
| 协议转换 | HTTP/gRPC/WebSocket |
| 日志监控 | 统一日志和指标 |
| 负载均衡 | 请求分发到多实例 |

## 三、主流网关

| 网关 | 语言 | 特点 |
|------|------|------|
| Spring Cloud Gateway | Java | Spring 生态，响应式 |
| Kong | Lua/OpenResty | 插件丰富，高性能 |
| APISIX | Lua | 云原生，高性能 |
| Envoy | C++ | Service Mesh 数据面 |
| Nginx | C | 轻量级，高性能 |

## 五、网关请求处理流程

```
请求生命周期:
1. 协议转换（HTTPS 终止，WebSocket 升级）
2. 认证鉴权（JWT 校验，OAuth2 验证）
3. 限流熔断（Sentinel/Ratelimiter）
4. 请求路由（Path/Header/参数匹配）
5. 负载均衡（Round-Robin/权重/一致性哈希）
6. 请求转发（连接池管理）
7. 响应转换（数据格式映射）
8. 日志追踪（TraceID 生成与传递）
9. 指标采集（QPS/延迟/错误率）
```

## 六、网关对比详解

| 网关 | 语言 | 性能 | 生态 | 学习曲线 | 适用场景 |
|------|------|------|------|---------|---------|
| Spring Cloud Gateway | Java (Reactor) | 中等 | Spring 生态完善 | 低 | Spring 技术栈 |
| Kong | Lua/OpenResty | 很高 | 插件市场丰富 | 中 | 高性能 API 管理 |
| APISIX | Lua | 很高 | 云原生友好 | 中 | K8s + 云原生 |
| Envoy | C++ | 极高 | Service Mesh 标准 | 高 | Istio Mesh |
| Nginx | C | 极高 | 通用 | 低 | 简单反向代理 |

## 七、注意事项

1. **网关是微服务的关键组件** - 必须高可用，至少 2 节点做主备
2. **不要在网关中放业务逻辑** - 网关只做流量治理，业务下沉到服务
3. **网关性能是瓶颈** - 需要水平扩展，异步非阻塞架构
4. **选择时考虑团队技术栈** - Spring 团队选 Gateway，运维强的选 Kong/APISIX
5. **生产环境至少 2 节点** - 前置负载均衡器做高可用
6. **路由配置要版本化** - 通过配置中心管理，支持回滚
