# API 网关概述

## 一、什么是 API 网关

API 网关是微服务架构的**统一入口**，负责请求路由、认证授权、限流熔断、日志监控等横切关注点。

```
客户端 → API Gateway → 微服务A
                    → 微服务B
                    → 微服务C
```

## 二、核心功能

| 功能 | 说明 |
|------|------|
| 请求路由 | 将请求转发到对应服务 |
| 认证授权 | 统一身份验证 |
| 限流熔断 | 流量控制和故障保护 |
| 协议转换 | HTTP/gRPC/WebSocket |
| 日志监控 | 统一日志和指标 |
| 负载均衡 | 请求分发到多实例 |

## 三、主流网关

| 网关 | 语言 | 特点 |
|------|------|------|
| Spring Cloud Gateway | Java | Spring 生态，响应式 |
| Kong | Lua/OpenResty | 插件丰富，高性能 |
| APISIX | Lua | 云原生，高性能 |
| Envoy | C++ | Service Mesh 数据面 |
| Nginx | C | 轻量级，高性能 |

## 四、注意事项

1. **网关是微服务的关键组件**，必须高可用
2. **不要在网关中放业务逻辑**
3. **网关性能是瓶颈**，需要水平扩展
4. **选择时考虑团队技术栈**
5. **生产环境至少 2 节点**
