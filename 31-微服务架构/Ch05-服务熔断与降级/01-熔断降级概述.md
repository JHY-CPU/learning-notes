# 熔断降级概述

## 一、为什么需要熔断降级

```yaml
微服务问题:
  - 服务间调用形成依赖链
  - 一个服务故障可能导致雪崩
  - 网络不可靠，超时和失败常见

熔断降级解决:
  - 熔断器: 检测故障，快速失败
  - 降级逻辑: 故障时返回兜底数据
  - 限流: 控制流量防止过载
```

## 二、熔断器状态

```
Closed (正常) → Open (熔断) → Half-Open (半开)
  │                │                │
  │   失败率超阈值   │   定时尝试恢复   │
  └────────────────→└────────────────→
                          ↓
                    成功则 Closed
                    失败则 Open
```

## 三、主流组件

| 组件 | 特点 |
|------|------|
| Sentinel | 阿里开源，功能全面 |
| Resilience4j | 轻量级，函数式 |
| Hystrix | Netflix，已停止维护 |

## 四、注意事项

1. **熔断是微服务必备能力**
2. **Sentinel 国内最流行**
3. **降级逻辑要保证核心功能**
4. **熔断阈值要根据实际调优**
5. **监控熔断触发情况**
