# 降级策略

## 一、降级方案

```java
// 1. 返回默认值
public User getUserFallback(Long id) {
    return User.defaultUser(id);
}

// 2. 缓存兜底
public User getUserFromCache(Long id) {
    return redisTemplate.opsForValue().get("user:" + id);
}

// 3. 兜底服务
public User getUserFromFallbackService(Long id) {
    return fallbackClient.getUser(id);
}

// 4. 静态数据
public User getStaticUser() {
    return new User(0L, "游客", "");
}
```

## 二、降级策略选择

| 场景 | 策略 |
|------|------|
| 用户信息 | 返回缓存或默认值 |
| 商品信息 | 返回缓存 |
| 订单信息 | 返回空列表 |
| 支付服务 | 提示稍后重试 |

## 三、注意事项

1. **降级逻辑要保证核心功能可用**
2. **降级数据要尽量贴近真实**
3. **降级逻辑要简洁高效**
4. **不同场景用不同降级策略**
5. **降级触发要有监控告警**
