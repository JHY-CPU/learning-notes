# 全链路熔断

## 一、核心概念

全链路熔断是指在服务调用链的每一层都部署熔断保护，形成层层防线。当某个下游服务出现问题时，链路上的每一层熔断器都能及时切断故障传播，防止级联故障导致整个系统雪崩。

### 1.1 链路级保护架构

```
客户端请求
    │
    ▼
┌──────────────────────────┐
│  API 网关（全局限流）      │  ← 第 1 层：入口限流
│  └─ 熔断器 (全局限流)     │
└───────────┬──────────────┘
            │
            ▼
┌──────────────────────────┐
│  Service A               │  ← 第 2 层：服务间调用熔断
│  └─ 熔断器 (调用 B)      │
│     └─ 熔断器 (调用 C)   │
└───────────┬──────────────┘
            │
     ┌──────┴──────┐
     ▼             ▼
┌─────────┐  ┌──────────┐
│Service B│  │Service C │  ← 第 3 层：依赖服务自身保护
│ 熔断器   │  │ 熔断器    │
└─────────┘  └──────────┘
```

### 1.2 全链路熔断 vs 单点熔断

```
单点熔断（不充分）:
  网关未限流 → Service A 可用 → Service B 不可用
  结果: A 被 B 拖垮，然后 A 拖垮网关

全链路熔断（层层保护）:
  网关限流 → A 熔断对 B 的调用 → A 降级返回
  结果: B 故障不影响 A，A 故障不影响网关
```

## 二、网关层熔断

### 2.1 Spring Cloud Gateway 熔断

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: user-service
          uri: lb://user-service
          predicates:
            - Path=/api/users/**
          filters:
            - name: CircuitBreaker
              args:
                name: user-service-cb
                fallbackUri: forward:/fallback/user
            - name: RequestRateLimiter
              args:
                redis-rate-limiter.replenishRate: 100
                redis-rate-limiter.burstCapacity: 200
```

### 2.2 网关熔断降级

```java
@RestController
@RequestMapping("/fallback")
public class GatewayFallbackController {

    @GetMapping("/user/**")
    public ResponseEntity<Map<String, Object>> userFallback(
            ServerHttpRequest request) {
        Map<String, Object> result = new HashMap<>();
        result.put("code", 503);
        result.put("message", "用户服务暂时不可用");
        result.put("fallback", true);
        result.put("timestamp", System.currentTimeMillis());
        return ResponseEntity.status(HttpStatus.SERVICE_UNAVAILABLE)
            .body(result);
    }
}
```

## 三、服务间调用熔断

### 3.1 OpenFeign + Resilience4j

```java
@FeignClient(
    name = "user-service",
    fallbackFactory = UserServiceFallbackFactory.class
)
public interface UserServiceClient {

    @GetMapping("/users/{id}")
    UserDTO getUser(@PathVariable("id") Long id);
}

@Component
public class UserServiceFallbackFactory
        implements FallbackFactory<UserServiceClient> {

    @Override
    public UserServiceClient create(Throwable cause) {
        return new UserServiceClient() {
            @Override
            public UserDTO getUser(Long id) {
                log.warn("用户服务降级, cause: {}",
                    cause.getMessage());
                return UserDTO.builder()
                    .id(id)
                    .name("默认用户")
                    .fallback(true)
                    .build();
            }
        };
    }
}
```

### 3.2 RestTemplate + 熔断

```java
@Configuration
public class RestTemplateConfig {

    @Bean
    @LoadBalanced
    @CircuitBreaker(name = "restTemplate",
                    fallbackMethod = "fallback")

        return new RestTemplate();
    }

    public String fallback(String url,
                            Exception e) {
        log.warn("RestTemplate 调用降级: {}", url);
        return "{\"fallback\": true}";
    }
}
```

## 四、方法级熔断

### 4.1 核心方法保护

```java
@Service
public class OrderService {

    @CircuitBreaker(name = "createOrder",
                    fallbackMethod = "createOrderFallback")
    @Retry(name = "createOrder")
    public Order createOrder(OrderRequest request) {
        // 1. 查询用户信息（可降级）
        UserDTO user = userService.getUser(request.getUserId());

        // 2. 库存扣减（核心，不可降级）
        inventoryService.deduct(request.getProductId(),
            request.getQuantity());

        // 3. 创建订单
        return orderRepository.save(buildOrder(request, user));
    }

    public Order createOrderFallback(OrderRequest request,
                                      Exception e) {
        log.error("创建订单降级: {}", e.getMessage());
        // 返回降级订单（标记为待处理）
        return Order.builder()
            .userId(request.getUserId())
            .status(OrderStatus.PENDING)
            .fallback(true)
            .build();
    }
}
```

### 4.2 依赖分级

```java
/**
 * 根据依赖级别配置不同的熔断策略
 */
@Service
public class ProductDetailService {

    // 强依赖：核心数据，熔断后直接失败
    @CircuitBreaker(name = "product-core",
                    fallbackMethod = "coreFallback")
    public ProductDTO getProduct(Long productId) {
        return productClient.getProduct(productId);
    }

    // 弱依赖：附加数据，熔断后返回默认值
    @CircuitBreaker(name = "product-weak",
                    fallbackMethod = "weakFallback")
    public ProductDetailVO getProductDetail(Long productId) {
        ProductDTO product = getProduct(productId);

        // 并行调用弱依赖
        CompletableFuture<ReviewsDTO> reviews =
            CompletableFuture.supplyAsync(
                () -> reviewClient.getReviews(productId))
            .exceptionally(e -> ReviewsDTO.empty());
        CompletableFuture<RecommendDTO> recommend =
            CompletableFuture.supplyAsync(
                () -> recommendClient.getSimilar(productId))
            .exceptionally(e -> RecommendDTO.empty());

        return ProductDetailVO.builder()
            .product(product)
            .reviews(reviews.join())
            .recommendations(recommend.join())
            .build();
    }

    public ProductDTO coreFallback(Long id, Exception e) {
        throw new ServiceUnavailableException("商品服务不可用");
    }

    public ReviewsDTO weakFallback(Long id, Exception e) {
        return ReviewsDTO.empty();
    }
}
```

## 五、熔断传播控制

### 5.1 熔断状态传播

```java
@Component
public class CircuitBreakerPropagation {

    /**
     * 传播熔断状态到下游调用
     */
    public <T> T callWithPropagation(String targetService,
                                      Supplier<T> action) {
        // 检查上游是否已经处于熔断状态
        String callerService = getCurrentServiceName();
        if (circuitBreakerRegistry
                .circuitBreaker(callerService)
                .getState() == State.OPEN) {
            // 上游已熔断，不再调用下游
            throw new UpstreamCircuitBreakerOpenException(
                callerService + " is already in OPEN state");
        }

        return action.get();
    }
}
```

### 5.2 舱壁隔离

```java
@Configuration
public class BulkheadConfig {

    @Bean
    public BulkheadRegistry bulkheadRegistry() {
        BulkheadConfig config = BulkheadConfig.custom()
            .maxConcurrentCalls(25)    // 最大并发数
            .maxWaitDuration(Duration.ofMillis(500))
            .build();

            .maxConcurrentCalls(50)    // 核心服务更高并发
            .maxWaitDuration(Duration.ofMillis(1000))
            .build();

        return BulkheadRegistry.of(config);
    }
}
```

## 七、最佳实践

1. **网关层做全局限流**，控制入口流量
2. **服务间调用做熔断**，防止故障传播
3. **关键方法做降级**，保证核心流程可用
4. **链路保护要层层设防**，不要只在一层做
5. **避免单点故障扩散**，使用舱壁隔离
6. **根据依赖级别配置不同熔断策略**，强依赖敏感，弱依赖宽松
7. **定期演练全链路熔断**，验证降级方案有效性
8. **监控全链路熔断状态**，建立统一 Dashboard
