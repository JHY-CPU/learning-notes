# 分布式事务选型

## 一、选型决策树

```
需要强一致性？
├── 是 → 资金类场景？ → 是 → TCC
│                  → 否 → Seata AT
└── 否 → 最终一致性
    ├── 有 MQ？ → 是 → 事务消息
    │          → 否 → 本地消息表
    └── 长事务？ → 是 → Saga
              → 否 → 本地消息表
```

## 二、注意事项

1. **尽量避免分布式事务**
2. **最终一致性是首选**
3. **资金类场景用 TCC**
4. **Seata AT 模式最简单**
5. **根据业务场景选择方案**
