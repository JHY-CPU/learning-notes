# 2PC 与 3PC

## 一、2PC (两阶段提交)

```
阶段1: Prepare - 询问所有参与者是否可以提交
阶段2: Commit/Rollback - 根据投票结果提交或回滚
```

## 二、3PC (三阶段提交)

```
阶段1: CanCommit - 询问是否可以提交
阶段2: PreCommit - 预提交
阶段3: DoCommit - 正式提交
```

## 三、注意事项

1. **2PC 性能差，不适合高并发**
2. **3PC 减少阻塞但不能完全解决**
3. **数据库层的 XA 协议是 2PC**
4. **Seata AT 模式是 2PC 的改进**
5. **生产环境很少直接用 2PC/3PC**
