# 日志管理

## 一、ELK Stack

```yaml
# Filebeat 配置
filebeat.inputs:
  - type: container
    paths:
      - /var/log/containers/*.log
    processors:
      - add_kubernetes_metadata:
          host: ${NODE_NAME}
          matchers:
            - logs_path:
                logs_path: "/var/log/containers/"

output.elasticsearch:
  hosts: ["elasticsearch:9200"]
  index: "microservices-%{+yyyy.MM.dd}"

processors:
  - decode_json_fields:
      fields: ["message"]
      target: "json"
```

## 二、结构化日志

```java
// Logback 配置 - JSON 格式
// logback-spring.xml
<configuration>
    <appender name="JSON" class="ch.qos.logback.core.ConsoleAppender">
        <encoder class="net.logstash.logback.encoder.LogstashEncoder">
            <includeMdcKeyName>traceId</includeMdcKeyName>
            <includeMdcKeyName>spanId</includeMdcKeyName>
            <customFields>{"service":"order-service"}</customFields>
        </encoder>
    </appender>

    <root level="INFO">
        <appender-ref ref="JSON"/>
    </root>
</configuration>
```

## 三、日志查询 (Kibana)

```yaml
常用查询:
  错误日志:
    query: "level:ERROR AND service:order-service"
    时间范围: 最近 1 小时

  特定请求:
    query: "traceId:abc123def456"
    含义: 查看完整调用链

  慢请求:
    query: "duration:>3000 AND message:请求完成"
    含义: 响应时间超过 3 秒
```

## 四、注意事项

1. **日志必须结构化（JSON）**
2. **traceId 要贯穿所有服务**
3. **日志级别合理设置**，生产不要 DEBUG
4. **敏感信息不要打印**
5. **日志保留期限要管理**
