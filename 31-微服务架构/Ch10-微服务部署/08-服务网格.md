# 服务网格 (Service Mesh)

## 一、Istio 架构

```
Istio 架构:
├── 数据平面 (Data Plane)
│   ├── Envoy Proxy - Sidecar 代理
│   └── 拦截所有服务间流量
│
└── 控制平面 (Control Plane)
    ├── Pilot - 流量管理
    ├── Citadel - 证书管理
    ├── Galley - 配置验证
    └── Istiod - 统一控制面
```

## 二、Sidecar 注入

```yaml
# 自动注入 - 给 Namespace 打标签
kubectl label namespace microservices istio-injection=enabled

# 手动注入
istioctl kube-inject -f deployment.yaml > deployment-injected.yaml

# Deployment 标注方式注入
apiVersion: apps/v1
kind: Deployment
metadata:
  annotations:
    sidecar.istio.io/inject: "true"
    proxy.istio.io/config: '{"holdApplicationUntilProxyStarts": true}'
spec:
  template:
    spec:
      containers:
        - name: order-service
          image: order-service:v1
```

## 三、流量管理

```yaml
# 超时配置
apiVersion: networking.istio.io/v1beta1
kind: VirtualService
metadata:
  name: order-service
spec:
  hosts:
    - order-service
  http:
    - route:
        - destination:
            host: order-service
      timeout: 3s
      retries:
        attempts: 3
        perTryTimeout: 1s
        retryOn: 5xx,reset,connect-failure

---
# 熔断
apiVersion: networking.istio.io/v1beta1
kind: DestinationRule
metadata:
  name: order-service
spec:
  host: order-service
  trafficPolicy:
    connectionPool:
      tcp:
        maxConnections: 100
      http:
        h2UpgradePolicy: DEFAULT
        http1MaxPendingRequests: 100
        http2MaxRequests: 1000
    outlierDetection:
      consecutive5xxErrors: 5
      interval: 30s
      baseEjectionTime: 30s
```

## 四、注意事项

1. **Sidecar 增加资源开销**，每 Pod 多 ~50MB 内存
2. **链路追踪自动注入**，无需代码修改
3. **mTLS 全链路加密**，零信任网络
4. **渐进式引入**，先非核心服务
5. **Istio 升级要谨慎**，测试环境先行
