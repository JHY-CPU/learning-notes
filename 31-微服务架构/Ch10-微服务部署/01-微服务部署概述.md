# 微服务部署概述

## 一、部署模式演进

```
部署演进:
├── 物理机部署 - 资源浪费，扩缩容慢
├── 虚拟机部署 - 资源隔离，启动慢
├── 容器部署 - 轻量级，秒级启动
└── 服务网格 - 流量治理，无侵入
```

## 二、部署策略对比

| 策略 | 停机时间 | 风险 | 资源消耗 | 适用场景 |
|------|---------|------|---------|---------|
| 滚动更新 | 无 | 中 | 低 | 常规发布 |
| 蓝绿部署 | 无 | 低 | 高 | 重要服务 |
| 金丝雀发布 | 无 | 低 | 中 | 灰度验证 |
| A/B 测试 | 无 | 低 | 中 | 业务验证 |
| 影子流量 | 无 | 极低 | 高 | 性能验证 |

## 三、CI/CD 流水线

```yaml
流水线阶段:
  1. 代码提交:
     - Git Hook 触发
     - 静态分析

  2. 构建:
     - 编译打包
     - 单元测试
     - Docker 镜像构建

  3. 测试:
     - 集成测试
     - 契约测试
     - 安全扫描

  4. 部署:
     - Dev 环境自动部署
     - Staging 环境手动审批
     - Production 环境灰度发布
```

## 四、GitLab CI 完整流水线

```yaml
# .gitlab-ci.yml
stages:
  - build
  - test
  - scan
  - deploy-dev
  - deploy-staging
  - deploy-prod

build:
  stage: build
  script:
    - docker build -t ${CI_REGISTRY_IMAGE}:${CI_COMMIT_SHORT_SHA} .
    - docker push ${CI_REGISTRY_IMAGE}:${CI_COMMIT_SHORT_SHA}
  only:
    - main
    - develop

test:
  stage: test
  script:
    - mvn test
    - mvn verify -DskipUnitTests
  coverage: '/Total.*?([0-9]{1,3})%/'
  artifacts:
    reports:
      junit: target/surefire-reports/TEST-*.xml

security-scan:
  stage: scan
  script:
    - trivy image ${CI_REGISTRY_IMAGE}:${CI_COMMIT_SHORT_SHA}
  allow_failure: true

deploy-dev:
  stage: deploy-dev
  script:
    - helm upgrade --install ${CI_PROJECT_NAME} ./charts
      --namespace dev
      --set image.tag=${CI_COMMIT_SHORT_SHA}
  environment:
    name: dev
  only:
    - develop

deploy-staging:
  stage: deploy-staging
  script:
    - helm upgrade --install ${CI_PROJECT_NAME} ./charts
      --namespace staging
      --set image.tag=${CI_COMMIT_SHORT_SHA}
  environment:
    name: staging
  when: manual
  only:
    - main

deploy-prod:
  stage: deploy-prod
  script:
    - helm upgrade --install ${CI_PROJECT_NAME} ./charts
      --namespace production
      --set image.tag=${CI_COMMIT_SHORT_SHA}
      --set replicaCount=3
  environment:
    name: production
  when: manual
  only:
    - main
```

## 五、不可变基础设施

```
原则:
├── 不修改运行中的容器
│   ├── 变更通过替换容器实现
│   └── 旧容器不修改，直接销毁
├── 镜像即部署单元
│   ├── 镜像包含应用 + 运行时 + 依赖
│   └── 镜像构建后不再修改
├── 配置通过环境变量注入
│   ├── ConfigMap 注入配置
│   └── Secret 注入密钥
└── 版本化一切
    ├── 镜像版本 = Git Commit SHA
    ├── Chart 版本 = 语义化版本
    └── 配置版本 = ConfigMap 版本
```

## 六、注意事项

1. **基础设施即代码（IaC）** - 版本化管理，Helm Chart + Terraform
2. **不可变部署** - 不修改运行中的容器，通过替换实现变更
3. **配置与代码分离** - 环境变量注入，ConfigMap/Secret 管理
4. **健康检查必须有** - 支持自动恢复，liveness + readiness
5. **回滚方案要准备好** - 一条命令回滚 `helm rollback`
6. **监控先行** - 没有监控的部署就是裸奔
