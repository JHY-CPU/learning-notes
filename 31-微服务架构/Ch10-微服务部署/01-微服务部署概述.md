# 微服务部署概述

## 一、部署模式演进

```
部署演进:
├── 物理机部署 - 资源浪费，扩缩容慢
├── 虚拟机部署 - 资源隔离，启动慢
├── 容器部署 - 轻量级，秒级启动
└── 服务网格 - 流量治理，无侵入
```

## 二、部署策略对比

| 策略 | 停机时间 | 风险 | 资源消耗 | 适用场景 |
|------|---------|------|---------|---------|
| 滚动更新 | 无 | 中 | 低 | 常规发布 |
| 蓝绿部署 | 无 | 低 | 高 | 重要服务 |
| 金丝雀发布 | 无 | 低 | 中 | 灰度验证 |
| A/B 测试 | 无 | 低 | 中 | 业务验证 |
| 影子流量 | 无 | 极低 | 高 | 性能验证 |

## 三、CI/CD 流水线

```yaml
流水线阶段:
  1. 代码提交:
     - Git Hook 触发
     - 静态分析

  2. 构建:
     - 编译打包
     - 单元测试
     - Docker 镜像构建

  3. 测试:
     - 集成测试
     - 契约测试
     - 安全扫描

  4. 部署:
     - Dev 环境自动部署
     - Staging 环境手动审批
     - Production 环境灰度发布
```

## 四、注意事项

1. **基础设施即代码（IaC）**，版本化管理
2. **不可变部署**，不修改运行中的容器
3. **配置与代码分离**，环境变量注入
4. **健康检查必须有**，支持自动恢复
5. **回滚方案要准备好**
