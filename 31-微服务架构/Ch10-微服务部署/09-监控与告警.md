# 监控与告警

## 一、Prometheus + Grafana

```yaml
# prometheus.yml
global:
  scrape_interval: 15s

scrape_configs:
  - job_name: 'order-service'
    kubernetes_sd_configs:
      - role: pod
    relabel_configs:
      - source_labels: [__meta_kubernetes_pod_annotation_prometheus_io_scrape]
        action: keep
        regex: true

# Spring Boot Actuator 暴露指标
management:
  endpoints:
    web:
      exposure:
        include: prometheus,health,info
  metrics:
    export:
      prometheus:
        enabled: true
    tags:
      application: ${spring.application.name}
```

## 二、告警规则

```yaml
# prometheus-rules.yml
groups:
  - name: microservices-alerts
    rules:
      - alert: HighErrorRate
        expr: |
          sum(rate(http_server_requests_seconds_count{status=~"5.."}[5m]))
          / sum(rate(http_server_requests_seconds_count[5m])) > 0.01
        for: 5m
        labels:
          severity: critical
        annotations:
          summary: "高错误率: {{ $labels.application }}"

      - alert: HighLatency
        expr: |
          histogram_quantile(0.99,
            rate(http_server_requests_seconds_bucket[5m])) > 1
        for: 5m
        labels:
          severity: warning

      - alert: PodNotReady
        expr: kube_pod_status_ready{condition="false"} == 1
        for: 5m
        labels:
          severity: critical
```

## 三、告警通知 (Alertmanager)

```yaml
# alertmanager.yml
global:
  resolve_timeout: 5m

route:
  group_by: ['alertname', 'application']
  group_wait: 10s
  group_interval: 5m
  repeat_interval: 4h
  receiver: 'default'
  routes:
    - match:
        severity: critical
      receiver: 'critical-alerts'

receivers:
  - name: 'default'
    webhook_configs:
      - url: 'http://alert-webhook:8080/alerts'

  - name: 'critical-alerts'
    webhook_configs:
      - url: 'http://alert-webhook:8080/critical'
    # 可配置钉钉/企业微信/飞书
```

## 四、注意事项

1. **黄金指标: 延迟、流量、错误、饱和度**
2. **告警要有分级**，避免告警风暴
3. **监控数据保留策略要合理**
4. **Dashboard 要提前准备**
5. **On-call 机制配合告警**
