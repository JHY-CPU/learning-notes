# 多集群部署

## 一、多集群架构

```
多集群方案:
├── 主备模式 (Active-Standby)
│   └── 主集群处理流量，备集群待命
├── 双活模式 (Active-Active)
│   └── 两个集群同时处理流量
└── 多区域模式 (Multi-Region)
    └── 不同区域独立部署
```

## 二、集群联邦 (KubeFed)

```yaml
apiVersion: types.kubefed.io/v1beta1
kind: FederatedDeployment
metadata:
  name: order-service
  namespace: microservices
spec:
  template:
    metadata:
      labels:
        app: order-service
    spec:
      replicas: 3
      selector:
        matchLabels:
          app: order-service
      template:
        spec:
          containers:
            - name: order-service
              image: order-service:v1.2.3
  placement:
    clusters:
      - name: cluster-cn-east
      - name: cluster-cn-north
  overrides:
    - clusterName: cluster-cn-east
      clusterOverrides:
        - path: "/spec/replicas"
          value: 5
```

## 三、Service Mesh 多集群

```yaml
# Istio 多集群 - Gateway 方式
# 集群 A
apiVersion: networking.istio.io/v1beta1
kind: Gateway
metadata:
  name: cross-cluster-gateway
spec:
  selector:
    istio: eastwestgateway
  servers:
    - port:
        number: 15443
        name: tls
        protocol: TLS
      tls:
        mode: AUTO_PASSTHROUGH
      hosts:
        - "*.local"

# 通过 DNS 实现跨集群服务发现
apiVersion: networking.istio.io/v1beta1
kind: ServiceEntry
metadata:
  name: remote-order-service
spec:
  hosts:
    - order-service.microservices.svc.cluster.local
  location: MESH_INTERNAL
  ports:
    - number: 80
      name: http
      protocol: HTTP
  resolution: DNS
  endpoints:
    - address: cluster-b.example.com
      ports:
        http: 15443
```

## 四、注意事项

1. **数据同步是多集群最大挑战**
2. **网络延迟要考虑**
3. **故障切换策略要明确**
4. **配置管理要统一**
5. **成本控制要做好**
