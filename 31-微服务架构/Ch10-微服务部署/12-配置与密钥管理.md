# 配置与密钥管理

## 一、外部化配置

```yaml
# Spring Boot 多环境配置
# application.yml - 公共配置
spring:
  application:
    name: order-service

# application-k8s.yml - K8s 环境
spring:
  cloud:
    kubernetes:
      config:
        enabled: true
        name: order-config
      secrets:
        enabled: true
        name: order-secrets
```

## 二、Sealed Secrets

```bash
# 安装 Sealed Secrets Controller
kubectl apply -f https://github.com/bitnami-labs/sealed-secrets/releases/download/v0.24.0/controller.yaml

# 创建普通 Secret
kubectl create secret generic db-credentials \
  --from-literal=username=admin \
  --from-literal=password=secret123 \
  --dry-run=client -o yaml > secret.yaml

# 加密为 Sealed Secret
kubeseal -f secret.yaml -w sealed-secret.yaml

# Sealed Secret 可以安全提交到 Git
```

## 三、HashiCorp Vault

```java
// Spring Cloud Vault 集成
@Configuration
public class VaultConfig {

    @Value("${database.username}")
    private String dbUsername;

    @Value("${database.password}")
    private String dbPassword;

    @Bean
    public DataSource dataSource() {
        HikariDataSource ds = new HikariDataSource();
        ds.setJdbcUrl("jdbc:mysql://mysql:3306/order_db");
        ds.setUsername(dbUsername);
        ds.setPassword(dbPassword);
        return ds;
    }
}

# bootstrap.yml
spring:
  cloud:
    vault:
      host: vault.example.com
      port: 8200
      token: ${VAULT_TOKEN}
      kv:
        enabled: true
        backend: secret
        default-context: order-service
```

## 四、K8s ConfigMap 与 Secret

```yaml
# ConfigMap - 非敏感配置
apiVersion: v1
kind: ConfigMap
metadata:
  name: order-service-config
  namespace: microservices
data:
  application.yaml: |
    spring:
      datasource:
        url: jdbc:mysql://mysql:3306/order_db
    app:
      feature:
        new-checkout: true
---
# Secret - 敏感配置 (base64 编码)
apiVersion: v1
kind: Secret
metadata:
  name: order-service-secrets
  namespace: microservices
type: Opaque
data:
  db-username: YWRtaW4=           # base64("admin")
  db-password: c2VjcmV0MTIz      # base64("secret123")
  jwt-secret: eW91ci1qd3Qtc2VjcmV0
---
# 引用 ConfigMap 和 Secret
apiVersion: apps/v1
kind: Deployment
spec:
  template:
    spec:
      containers:
      - name: order-service
        env:
        - name: DB_USERNAME
          valueFrom:
            secretKeyRef:
              name: order-service-secrets
              key: db-username
        - name: DB_PASSWORD
          valueFrom:
            secretKeyRef:
              name: order-service-secrets
              key: db-password
        volumeMounts:
        - name: config-volume
          mountPath: /config
      volumes:
      - name: config-volume
        configMap:
          name: order-service-config
```

## 五、密钥轮换方案

```yaml
# 自动轮换数据库密码
spring:
  cloud:
    vault:
      database:
        enabled: true
        role: order-service
        backend: database
        rotation-period: 24h     # 24 小时自动轮换

# Vault 自动轮换流程:
# 1. Vault 生成新密码
# 2. 更新数据库密码
# 3. 通知应用重新获取连接
# 4. 删除旧密码
```

## 六、注意事项

1. **密钥不要提交到代码仓库** - 使用 .gitignore 和 pre-commit hook
2. **使用 Sealed Secrets 或 Vault 管理密钥** - 不使用明文 Secret
3. **配置变更要能热更新** - Spring Cloud Config + @RefreshScope
4. **密钥要定期轮换** - Vault 自动轮换，90 天强制更新
5. **访问权限要最小化** - RBAC 控制，每个服务只能访问自己的配置
6. **审计日志要记录** - 谁在什么时候修改了什么配置
