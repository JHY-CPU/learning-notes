# 优雅上下线

## 一、优雅下线

```java
// Spring Boot 优雅关闭
@SpringBootApplication
public class Application {
    @PreDestroy
    public void onShutdown() {
        // 1. 从注册中心注销
        discoveryClient.deregister();
        // 2. 等待现有请求处理完成
        Thread.sleep(10000);
        // 3. 关闭线程池
        executorService.shutdown();
    }
}
```

```yaml
# K8s 优雅关闭
lifecycle:
  preStop:
    exec:
      command: ["sh", "-c", "sleep 10"]
terminationGracePeriodSeconds: 30
```

## 二、优雅上线

```java
// 健康检查就绪
@Component
public class ReadinessHealthIndicator implements HealthIndicator {
    private volatile boolean ready = false;

    @PostConstruct
    public void init() {
        // 预热完成后标记为就绪
        CompletableFuture.runAsync(() -> {
            warmUp();
            ready = true;
        });
    }

    @Override
    public Health health() {
        return ready ? Health.up().build() : Health.down().build();
    }
}
```

## 三、完整优雅上下线流程

```
优雅上线:
1. 启动应用 → 初始化 Spring Context
2. 预热（加载缓存、建立连接池）
3. 注册到注册中心
4. 通过 Readiness 探针 → 开始接收流量
5. 监控初始请求成功率

优雅下线:
1. 收到 SIGTERM 信号
2. 执行 preStop Hook（等待 5-10 秒）
3. 从注册中心摘除实例（标记为不健康）
4. K8s 停止向 Pod 转发新请求
5. 等待存量请求处理完成
6. 关闭连接池和线程池
7. 进程退出
```

## 四、Spring Boot 优雅关闭完整实现

```java
@SpringBootApplication
public class Application implements ApplicationRunner {
    private static final Logger log = LoggerFactory.getLogger(Application.class);

    @Autowired
    private NacosDiscoveryProperties nacosDiscovery;

    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }

    @Bean
    public GracefulShutdown gracefulShutdown() {
        return new GracefulShutdown();
    }

    static class GracefulShutdown implements SmartLifecycle {
        private volatile boolean running = false;

        @Override
        public void start() { running = true; }

        @Override
        public void stop() {
            running = false;
            log.info("开始优雅关闭...");
            // 1. 从注册中心注销
            try {
                // 等待 10 秒让负载均衡器更新
                Thread.sleep(10000);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
            // 2. 拒绝新请求
            log.info("已停止接收新请求");
            // 3. 等待现有请求完成
            try {
                Thread.sleep(15000);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
            log.info("优雅关闭完成");
        }

        @Override
        public boolean isRunning() { return running; }
    }
}
```

## 五、灰度发布配合

```yaml
# K8s 金丝雀发布 - 先发布 1 个新版本 Pod
apiVersion: apps/v1
kind: Deployment
metadata:
  name: user-service-canary
spec:
  replicas: 1            # 先发 1 个
  selector:
    matchLabels:
      app: user-service
      track: canary
  template:
    metadata:
      labels:
        app: user-service
        track: canary
    spec:
      containers:
      - name: user-service
        image: user-service:2.0    # 新版本
        readinessProbe:
          httpGet:
            path: /actuator/health/readiness
            port: 8080
          initialDelaySeconds: 30
```

## 六、注意事项

1. **下线前先从注册中心注销** - 让负载均衡器不再转发请求
2. **等待现有请求处理完再关闭** - 给足够的时间（10-30 秒）
3. **上线后预热再接流量** - 加载缓存、预热 JVM
4. **K8s 中 preStop 和 readiness 探针配合** - preStop 等待 + readiness 失败 = 零丢失
5. **滚动更新保证零停机** - maxSurge=1, maxUnavailable=0
6. **监控部署过程中的错误率** - 异常时自动回滚
