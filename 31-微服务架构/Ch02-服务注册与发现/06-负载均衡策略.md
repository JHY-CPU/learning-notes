# 负载均衡策略

## 一、负载均衡算法

| 算法 | 说明 | 适用场景 |
|------|------|---------|
| 轮询 | 依次分配 | 通用 |
| 加权轮询 | 按权重分配 | 机器性能不同 |
| 随机 | 随机选择 | 简单场景 |
| 最少连接 | 选择连接最少的 | 长连接场景 |
| 一致性哈希 | 相同请求到同一节点 | 缓存场景 |

```java
// Spring Cloud LoadBalancer
@Bean
@LoadBalanced
public RestTemplate restTemplate() {
    return new RestTemplate();
}

// 自定义负载均衡
@Bean
public ReactorLoadBalancer<ServiceInstance> randomLoadBalancer(
    Environment environment, ServiceInstanceListSupplier supplier) {
    return new RandomLoadBalancer(supplier, "user-service");
}
```

## 二、注意事项

1. **轮询是最常用的策略**
2. **权重动态调整要支持**
3. **健康检查要与负载均衡配合**
4. **一致性哈希适合缓存场景**
5. **客户端负载均衡优于服务端负载均衡**
