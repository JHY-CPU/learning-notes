# 服务元数据

## 一、元数据类型

```yaml
# Nacos 服务元数据
spring:
  cloud:
    nacos:
      discovery:
        metadata:
          version: v1.0
          region: cn-east
          zone: az1
          tags: high-priority
          weight: 100
```

## 二、元数据用途

| 用途 | 说明 |
|------|------|
| 版本路由 | 灰度发布，按版本分流 |
| 区域路由 | 就近访问，减少延迟 |
| 权重控制 | 流量分配比例 |
| 标签路由 | 按标签筛选服务 |

## 三、灰度发布

```java
// 按版本路由
@GetMapping("/users/{id}")
public User getUser(@PathVariable Long id) {
    // v1 版本调用 user-service:v1
    // v2 版本调用 user-service:v2
    return userClient.getUser(id);
}
```

## 四、注意事项

1. **元数据是服务发现的重要维度**
2. **版本路由实现灰度发布**
3. **区域路由实现就近访问**
4. **元数据变更要实时生效**
5. **元数据不要存储大量数据**
