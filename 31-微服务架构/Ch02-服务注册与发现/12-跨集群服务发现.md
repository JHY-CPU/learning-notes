# 跨集群服务发现

## 一、多集群方案

```yaml
方案1: Nacos 集群同步
  集群A (cn-east) → 同步 → 集群B (cn-west)

方案2: 联邦注册中心
  全局注册中心 → 同步 → 区域注册中心

方案3: DNS 服务发现
  DNS 记录 → 服务地址
```

## 二、工作原理

跨集群服务发现的核心是解决多个注册中心集群之间的数据同步问题。Nacos 集群同步方案中，各集群通过 HTTP 长轮询或 gRPC 订阅方式向对端集群同步服务数据，使用时间戳或版本号避免重复同步。联邦方案则引入全局注册中心作为汇聚层，各区域注册中心向全局注册中心上报服务摘要，消费者先查询全局注册中心确定服务所在的区域，再定向访问。DNS 方案将服务地址注册为 DNS SRV 记录，通过标准 DNS 协议解析。

## 三、优缺点

**优点：**
- 联邦方案支持大规模多机房，扩展性好
- Nacos 同步方案配置简单，延迟可控
- DNS 方案零侵入，兼容所有语言

**缺点：**
- 同步存在延迟，极端情况下服务信息不一致
- 联邦方案增加了架构复杂度和维护成本
- DNS 方案不支持实时感知服务上下线

## 四、最佳实践

1. 同步采用增量同步而非全量同步，降低网络开销
2. 延迟敏感服务优先调用本集群实例，跨集群仅作降级
3. 同步链路加入去重和幂等机制，防止循环同步
4. 监控同步延迟指标，设置告警阈值

## 五、Nacos 跨集群同步配置

```yaml
# Nacos 集群间数据同步配置
# conf/application.properties
nacos.core.auth.enabled=true
nacos.core.auth.server.identity.key=cluster-sync
nacos.core.auth.server.identity.value=secret

# 同步规则 - 通过 Admin API 配置
# POST /nacos/v1/cs/cluster
{
  "clusters": [
    {
      "name": "cn-east",
      "addresses": ["10.0.1.1:8848", "10.0.1.2:8848"]
    },
    {
      "name": "cn-west",
      "addresses": ["10.0.2.1:8848", "10.0.2.2:8848"]
    }
  ],
  "syncMode": "INCREMENTAL",  // 增量同步
  "syncInterval": 5000         // 5 秒同步间隔
}
```

## 六、联邦注册中心架构

```
              ┌───────────────────┐
              │  全局注册中心     │
              │ (Global Registry) │
              └────────┬──────────┘
                       │
         ┌─────────────┼─────────────┐
         │             │             │
┌────────▼──────┐ ┌────▼────────┐ ┌──▼──────────┐
│ 北京区域注册   │ │ 上海区域注册 │ │ 深圳区域注册  │
│ cn-north      │ │ cn-east     │ │ cn-south     │
└────────┬──────┘ └────┬────────┘ └──┬──────────┘
         │             │             │
   ┌─────┴─────┐ ┌─────┴─────┐ ┌─────┴─────┐
   │ 本地服务   │ │ 本地服务   │ │ 本地服务   │
   └───────────┘ └───────────┘ └───────────┘

消费者调用流程:
1. 查询全局注册中心 → 服务在 cn-east 区域
2. 直接调用 cn-east 区域的实例
3. 如果 cn-east 不可用 → 降级到其他区域
```

## 七、注意事项

1. **循环同步导致数据膨胀** - 必须在同步协议中加入来源标记（source cluster）
2. **跨集群调用链路过长** - 增加延迟和故障面，优先本集群调用
3. **网络分区时同步中断** - 各集群服务视图不一致，需要降级方案
4. **DNS TTL 设置过长** - 服务变更后长时间解析到旧地址，TTL 建议 10-30 秒
5. **监控同步延迟指标** - 设置告警阈值，超过 30 秒需要人工介入
