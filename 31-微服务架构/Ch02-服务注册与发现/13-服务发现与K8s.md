# 服务发现与 Kubernetes

## 一、K8s 内置服务发现

```yaml
# K8s Service
apiVersion: v1
kind: Service
metadata:
  name: user-service
spec:
  selector:
    app: user-service
  ports:
  - port: 80
    targetPort: 8080
  type: ClusterIP

# 通过 DNS 访问
# user-service.default.svc.cluster.local
```

## 二、K8s 与注册中心

```yaml
# K8s 中使用 Nacos
spring:
  cloud:
    nacos:
      discovery:
        server-addr: nacos-headless:8848
        namespace: ${NAMESPACE}
```

## 三、注意事项

1. **K8s Service 是内置的服务发现**
2. **K8s + Nacos 可以共存**
3. **Istio 提供更强大的服务网格能力**
4. **K8s 环境下优先使用原生服务发现**
5. **跨 K8s 集群需要联邦服务发现**
