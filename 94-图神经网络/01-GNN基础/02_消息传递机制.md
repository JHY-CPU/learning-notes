# 消息传递机制（Message Passing）


## 1. MPNN 框架概述


消息传递神经网络（Message Passing Neural Network, MPNN）是Gilmer等人在2017年提出的统一框架，将多种GNN模型（GCN、GAT、GIN等）归纳为同一范式。


### 核心思想


每个节点通过与其邻居交换信息（消息）来更新自身的表示。经过 K 轮消息传递后，每个节点的表示包含了 K 跳邻居的信息。


### 三个核心步骤


1. **消息（Message）**
   ：每个节点向邻居发送消息
2. **聚合（Aggregate）**
   ：每个节点聚合来自邻居的消息
3. **更新（Update）**
   ：每个节点根据聚合的消息更新自身表示


$$
hv(k+1) = U(k)(hv(k), { M(k)(hv(k), hu(k), euv) | u ∈ N(v) })
$$


> **Note:** **类比理解：**
> 把图想象成一个社交网络，消息传递就像朋友们互相交流信息。每一轮交流后，每个人对世界的认识都更丰富了，因为他吸收了朋友们的观点。


## 2. 消息函数（Message Function）


消息函数定义了节点 u 向节点 v 传递的消息内容。


$$
mu→v = M(hv, hu, euv)
$$


### 常见消息函数


| 模型 | 消息函数 | 说明 |
| --- | --- | --- |
| GCN | M(h~u~) = W h~u~ | 仅依赖发送节点 |
| GAT | M(h~v~,h~u~) = α~vu~ W h~u~ | 注意力加权 |
| MPNN | M(h~v~,h~u~,e~uv~) = h~u~ + e~uv~ | 包含边特征 |
| GIN | M(h~u~) = h~u~ | 直接传递原始特征 |


### 边特征的处理


当图中存在边特征 e~uv~ 时，消息函数可以同时考虑节点特征和边特征。例如在分子图中，边特征可以代表化学键类型。


## 3. 聚合函数（Aggregate Function）


聚合函数将邻居的消息合并为一个向量。聚合函数必须满足**排列不变性**，即邻居的顺序不应影响聚合结果。


$$
mv = AGG({ mu→v | u ∈ N(v) })
$$


### 三种基本聚合方式


| 聚合方式 | 公式 | 优点 | 缺点 |
| --- | --- | --- | --- |
| 求和（Sum） | Σ~u~ m~u~ | 保留邻居数量信息，表达力最强 | 数值可能随度增大 |
| 均值（Mean） | (1/\|N(v)\|) Σ~u~ m~u~ | 对度数不同的节点公平 | 丢失邻居数量信息 |
| 最大值（Max） | max~u~ m~u~ | 捕获最显著特征 | 丢失大部分邻居信息 |


> **Important:** **表达力理论：**
> 根据 Xu et al. (2019) 的研究，Sum 聚合的表达力最强，可以区分一些 Mean 和 Max 无法区分的图结构。GIN 模型使用 Sum 聚合，其表达力与WL图同构测试相当。


## 4. 更新函数（Update Function）


更新函数将节点当前的表示与聚合后的消息结合，得到新的表示。


$$
hv(k+1) = U(k)(hv(k), mv(k))
$$

- **拼接+线性变换**

- **残差连接**

## 5. 过平滑问题（Over-smoothing）


随着消息传递层数增加，节点的表示会逐渐趋同，最终收敛到相似的值，这就是过平滑问题。


### 直观理解


- 第1层：每个节点只聚合1跳邻居的信息
- 第K层：每个节点聚合了K跳范围内的所有节点信息
- 当K足够大时（如达到图的直径），所有节点看到的是几乎相同的子图


### 数学分析


在GCN中，多次应用归一化邻接矩阵 D̃^-1^Ã 会导致节点表示收敛到平稳分布。Li et al. (2018) 证明了在一定条件下，节点特征会指数级收敛。


### 缓解策略


| 方法 | 思路 | 代表工作 |
| --- | --- | --- |
| 残差连接 | 跳跃连接保留低层特征 | JKNet, GCNII |
| DropEdge | 训练时随机删除边 | Rong et al. 2020 |
| 归一化 | 添加BN/LN层 | PairNorm, NodeNorm |
| 浅层设计 | 限制层数2-4层 | 大多数GNN实践 |
| 个性化PageRank | 混合本地和全局信息 | APPNP |


> **Note:** **实践建议：**
> 对于大多数任务，2-4层的GNN已经足够。更深的网络不一定更好，反而可能因过平滑导致性能下降。


## 6. PyTorch Geometric 代码示例


```
import torch
import torch.nn as nn
from torch_geometric.nn import MessagePassing
from torch_geometric.utils import add_self_loops, degree
from torch_geometric.data import Data

# ========== 手写一个消息传递层 ==========
class MyConvLayer(MessagePassing):
    """
    自定义消息传递GNN层
    aggr='add' 表示使用求和聚合
    """
    def __init__(self, in_channels, out_channels):
        super().__init__(aggr='add')  # 聚合方式：sum/mean/max
        self.lin = nn.Linear(in_channels, out_channels, bias=False)
        self.bias = nn.Parameter(torch.zeros(out_channels))

    def forward(self, x, edge_index):
        # x: [N, in_channels], edge_index: [2, E]
        # 添加自环
        edge_index, _ = add_self_loops(edge_index, num_nodes=x.size(0))
        # 线性变换
        x = self.lin(x)
        # 消息传递
        return self.propagate(edge_index, x=x) + self.bias

    def message(self, x_j):
        """
        消息函数：x_j 是邻居节点的特征
        PyG自动将x按edge_index索引为x_j（邻居）和x_i（目标节点）
        """
        return x_j  # 直接传递邻居特征

    def aggregate(self, inputs, index, ptr=None, dim_size=None):
        """
        聚合函数（已通过aggr='add'指定，此处展示如何自定义）
        """
        return super().aggregate(inputs, index, ptr, dim_size)


# ========== 创建图数据 ==========
# 5个节点，4维特征
x = torch.randn(5, 4)
# 边：无向图用两条有向边表示
edge_index = torch.tensor([
    [0, 1, 1, 2, 2, 3, 3, 4],  # 源节点
    [1, 0, 2, 1, 3, 2, 4, 3]   # 目标节点
], dtype=torch.long)
y = torch.tensor([0, 1, 1, 0, 1], dtype=torch.long)  # 节点标签

data = Data(x=x, edge_index=edge_index, y=y)
print(f"图数据: {data}")
print(f"节点数: {data.num_nodes}, 边数: {data.num_edges}")

# ========== 使用自定义层 ==========
model = MyConvLayer(in_channels=4, out_channels=8)
out = model(data.x, data.edge_index)
print(f"输出形状: {out.shape}")  # [5, 8]

# ========== 查看消息传递过程 ==========
from torch_geometric.utils import to_networkx
import networkx as nx

# 可视化图结构
G = to_networkx(data, to_undirected=True)
print(f"节点度: {dict(G.degree())} ")


# ========== 多层消息传递示例 ==========
class MultiLayerGNN(nn.Module):
    def __init__(self, in_dim, hidden_dim, out_dim, num_layers=3):
        super().__init__()
        self.layers = nn.ModuleList()
        self.layers.append(MyConvLayer(in_dim, hidden_dim))
        for _ in range(num_layers - 2):
            self.layers.append(MyConvLayer(hidden_dim, hidden_dim))
        self.layers.append(MyConvLayer(hidden_dim, out_dim))
        self.relu = nn.ReLU()

    def forward(self, x, edge_index):
        for i, layer in enumerate(self.layers):
            x = layer(x, edge_index)
            if i < len(self.layers) - 1:
                x = self.relu(x)
        return x

model = MultiLayerGNN(4, 16, 3, num_layers=3)
out = model(data.x, data.edge_index)
print(f"多层GNN输出: {out.shape}")  # [5, 3]
```


> **Important:** **MessagePassing 基类的关键方法：**
>
> - `propagate(edge_index, x=x)`
>    ：触发消息传递流程
> - `message(x_j)`
>    ：定义消息函数
> - `aggregate(inputs, index)`
>    ：定义聚合方式
> - `update(aggr_out)`
>    ：定义更新方式


## 总结


- MPNN框架将GNN统一为 消息→聚合→更新 三个步骤
- 消息函数定义如何从邻居节点构造消息
- 聚合函数必须满足排列不变性（Sum/Mean/Max）
- 更新函数将聚合结果与当前状态结合
- 过平滑是深层GNN的主要瓶颈，需要残差连接等技巧缓解
- PyTorch Geometric 的 MessagePassing 基类简化了GNN层的实现
