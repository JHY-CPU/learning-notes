# 知识图谱嵌入


## 1. 知识图谱的基本概念


知识图谱（Knowledge Graph, KG）是一种以图结构存储知识的数据库，由实体（Entity）、关系（Relation）和三元组（Triple）组成。


### 三元组表示


$$
(头实体, 关系, 尾实体) ≡ (h, r, t)
                例：(中国, 首都, 北京)、(爱因斯坦, 出生地, 乌尔姆)
$$


### 常见知识图谱


| 知识图谱 | 规模 | 领域 | 来源 |
| --- | --- | --- | --- |
| Freebase | ~6000万实体 | 通用 | 众包 |
| Wikidata | ~1亿实体 | 通用 | 维基百科 |
| DBpedia | ~600万实体 | 通用 | 维基百科结构化 |
| YAGO | ~5000万实体 | 通用 | 维基百科+WordNet |


### 知识图谱嵌入的目标


- 将实体和关系映射到低维连续向量空间
- 保留知识图谱中的结构信息
- 支持下游任务：链接预测、三元组分类、知识推理


## 2. TransE：平移模型


TransE（Bordes et al., 2013）是最经典的知识图谱嵌入模型，核心思想是将关系视为头实体到尾实体的平移。


### 核心假设


$$
h + r ≈ t
                即: ||h + r - t|| ≈ 0（对于正确的三元组）
$$


### 评分函数


$$
f(h, r, t) = -||h + r - t||1/2
$$


评分越高，三元组越可能是正确的。


### 训练方法


使用负采样 + margin-based ranking loss：


$$
L = Σ max(0, γ + f(h,r,t') - f(h,r,t))
                其中 (h,r,t) 是正样本，(h,r,t') 是负样本（随机替换头或尾实体）
$$


> **Example:** #### 直觉理解
>
>
> 如果"中国 + 首都 = 北京"成立，那么 TransE 学到的向量满足：
>
>                 vector(中国) + vector(首都) ≈ vector(北京)


| 属性 | TransE |
| --- | --- |
| 参数量 | (\|E\| + \|R\|) × d |
| 复杂度 | O(d) |
| 适用关系 | 一对一（1-to-1） |
| 局限 | 无法处理一对多、多对多关系 |


## 3. TransR：关系空间投影


TransR（Lin et al., 2015）解决了TransE在处理复杂关系时的不足，通过将实体投影到关系特定的空间中。


### 核心思想


$$
hr = Mr h,    tr = Mr t
                f(h, r, t) = -||hr + r - tr||22
$$


其中 M~r~ ∈ R^k×d^ 是关系 r 的投影矩阵。


### 距离模型家族对比


| 模型 | 评分函数 | 核心改进 |
| --- | --- | --- |
| TransE | -\|\|h+r-t\|\| | 平移假设 |
| TransH | -\|\|h~⊥~+r-t~⊥~\|\| | 超平面投影 |
| TransR | -\|\|M~r~h+r-M~r~t\|\| | 关系空间投影 |
| TransD | -\|\|M~rh~h+r-M~rt~t\|\| | 动态投影矩阵 |
| RotatE | -\|\|h∘r-t\|\| | 复数空间旋转 |


> **Note:** **TransR的改进：**
> 在实体空间中，"中国"和"北京"可能很接近；但在"首都"关系空间中，它们的位置关系被重新编码，使得 h
> ~r~
> + r ≈ t
> ~r~
> 。


## 4. DistMult 与 ComplEx：双线性模型


### DistMult


Yang et al. (2015) 提出的双线性模型，用对角矩阵表示关系。


$$
f(h, r, t) = hT diag(r) t = Σi hi · ri · ti
$$


**局限：**对称性——f(h,r,t) = f(t,r,h)，只能建模对称关系。


### ComplEx


Trouillon et al. (2016) 将 DistMult 扩展到复数空间，解决了对称性问题。


$$
f(h, r, t) = Re(Σi hi · ri · t̄i)
                其中 h, r, t ∈ Cd，t̄ 是 t 的复共轭
$$


复共轭打破了对称性：f(h,r,t) ≠ f(t,r,h)，可以建模非对称关系。


### 双线性模型对比


| 模型 | 空间 | 对称性 | 表现力 |
| --- | --- | --- | --- |
| DistMult | 实数 | 对称 | 中等 |
| ComplEx | 复数 | 非对称 | 较强 |
| RotatE | 复数（旋转） | 非对称 | 强 |
| QuatE | 四元数 | 非对称 | 最强 |


## 5. 知识图谱补全与基于 GNN 的推理


### 知识图谱补全任务


给定不完整的三元组，预测缺失的实体或关系：


- **头实体预测：**
   (?, r, t) → h
- **尾实体预测：**
   (h, r, ?) → t
- **关系预测：**
   (h, ?, t) → r


### 评估指标


| 指标 | 说明 |
| --- | --- |
| MR（Mean Rank） | 正确实体的平均排名，越低越好 |
| MRR（Mean Reciprocal Rank） | 排名倒数的平均，越高越好 |
| Hit@k | 正确实体在前k名的比例，越高越好 |


### 基于 GNN 的知识推理


传统知识图谱嵌入只建模局部三元组信息，GNN可以利用图结构进行多跳推理。


- **R-GCN**
   ：关系图卷积网络，为不同类型的关系使用不同的权重矩阵
- **CompGCN**
   ：组合图卷积网络，支持多种实体-关系组合操作
- **NBFNet**
   ：神经贝尔曼-福特网络，基于消息传递的路径推理


$$
R-GCN 更新规则:
                hv(l+1) = σ(Σr∈R Σu∈Nvr (1/cv,r) Wr(l) hu(l) + W0(l) hv(l))
$$


## 6. PyKEEN 库使用示例


```
from pykeen.pipeline import pipeline
from pykeen.triples import TriplesFactory
import numpy as np

# ========== 准备数据 ==========
# 三元组格式: numpy array of shape (n_triples, 3)
triples = np.array([
    ['中国', '首都', '北京'],
    ['中国', '货币', '人民币'],
    ['美国', '首都', '华盛顿'],
    ['美国', '货币', '美元'],
    ['日本', '首都', '东京'],
    ['日本', '货币', '日元'],
    ['英国', '首都', '伦敦'],
    ['英国', '货币', '英镑'],
    ['法国', '首都', '巴黎'],
    ['德国', '首都', '柏林'],
    ['北京', '位于', '中国'],
    ['华盛顿', '位于', '美国'],
], dtype=str)

# 创建TriplesFactory
tf = TriplesFactory.from_labeled_triples(triples)

# 划分训练集和测试集
training, testing = tf.split([0.8], random_state=42)

# ========== 训练模型 ==========
result = pipeline(
    training=training,
    testing=testing,
    model='TransE',           # 可选: TransE, DistMult, ComplEx, RotatE
    loss='MarginRankingLoss',
    optimizer='Adam',
    optimizer_kwargs=dict(lr=0.01),
    training_kwargs=dict(num_epochs=100, batch_size=64),
    random_seed=42,
)

# 打印评估结果
print(result.metric_results.to_df())

# ========== 使用训练好的模型 ==========
model = result.model

# 预测尾实体: (中国, 首都, ?)
scores = model.predict_t(
    torch.tensor([tf.entity_to_id['中国'], tf.relation_to_id['首都']])
)
top_k = scores.topk(5)
print("预测 (中国, 首都, ?) 的Top-5尾实体:")
for idx, score in zip(top_k.indices, top_k.values):
    entity = tf.entity_id_to_label[idx]
    print(f"  {entity}: {score:.4f}")


# ========== R-GCN 示例 (使用PyG) ==========
import torch
from torch_geometric.nn import RGCNConv

class RGCNLinkPredictor(torch.nn.Module):
    def __init__(self, num_entities, num_relations, hidden_dim, num_bases=4):
        super().__init__()
        self.entity_emb = torch.nn.Embedding(num_entities, hidden_dim)
        self.conv1 = RGCNConv(hidden_dim, hidden_dim, num_relations, num_bases=num_bases)
        self.conv2 = RGCNConv(hidden_dim, hidden_dim, num_relations, num_bases=num_bases)

    def forward(self, edge_index, edge_type):
        x = self.entity_emb.weight
        x = torch.relu(self.conv1(x, edge_index, edge_type))
        x = self.conv2(x, edge_index, edge_type)
        return x

    def predict(self, h, r, t):
        """DistMult式评分"""
        return (h * r * t).sum(dim=-1)
```


## 总结


- 知识图谱由三元组 (h, r, t) 组成，嵌入目标是将实体和关系映射到低维空间
- TransE 用平移假设 h+r≈t 建模，简单高效但受限于复杂关系
- TransR 在关系空间中投影实体，能更好地处理多对多关系
- DistMult 和 ComplEx 是双线性模型，ComplEx 用复数空间解决对称性问题
- 基于GNN的方法（R-GCN）可以利用图结构进行多跳推理
- PyKEEN 是最全面的知识图谱嵌入库，支持多种模型和评估
