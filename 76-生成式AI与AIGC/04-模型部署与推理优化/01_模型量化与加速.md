# 模型量化与加速

*INT8/INT4量化、剪枝、蒸馏与推理框架全解析*


大模型的部署面临**显存不足**、**推理延迟高**、**吞吐量低**等挑战。模型优化通过压缩模型大小和加速计算来解决这些问题：


| 技术 | 目标 | 压缩比 | 精度损失 |
| --- | --- | --- | --- |
| 量化（Quantization） | 降低权重/激活的精度 | 2-4倍 | 小-中 |
| 剪枝（Pruning） | 移除冗余参数 | 2-10倍 | 中 |
| 知识蒸馏（Distillation） | 训练小模型模仿大模型 | 5-50倍 | 中-大 |
| 低秩分解 | 矩阵分解减少参数 | 2-4倍 | 小 |


### 2.1 量化原理


量化将高精度浮点数（FP32/FP16）映射为低精度整数（INT8/INT4），通过减少存储和计算精度来加速推理。


$$
q = round(x / scale) + zero_point　　（量化）x̂ = (q - zero_point) × scale　　（反量化）
$$


### 3.2 对称量化与非对称量化


| 类型 | zero_point | 适用场景 |
| --- | --- | --- |
| 对称量化 | 0 | 权重（通常以0为中心对称分布） |
| 非对称量化 | 计算得到 | 激活值（分布可能不对称，如ReLU后全为正） |


```
# 对称量化
scale = max(|x_max|, |x_min|) / (2^(bits-1) - 1)
q = round(x / scale)

# 非对称量化
scale = (x_max - x_min) / (2^bits - 1)
zero_point = round(-x_min / scale)
q = round(x / scale) + zero_point
```


### 2.3 精度对比


| 数据类型 | 位宽 | 内存占用 | 取值范围 | 用途 |
| --- | --- | --- | --- | --- |
| FP32 | 32 bit | 4 bytes/参数 | ±3.4×10³⁸ | 训练（标准精度） |
| FP16 | 16 bit | 2 bytes/参数 | ±65504 | 混合精度训练 |
| BF16 | 16 bit | 2 bytes/参数 | ±3.4×10³⁸ | 训练（大模型首选） |
| FP8 (E4M3/E5M2) | 8 bit | 1 byte/参数 | 较小 | 新一代训练/推理 |
| INT8 | 8 bit | 1 byte/参数 | [-128, 127] | 推理优化 |
| INT4 | 4 bit | 0.5 byte/参数 | [-8, 7] | 极致压缩 |
| NF4 (NormalFloat4) | 4 bit | 0.5 byte/参数 | 量化最优 | QLoRA专用 |


### 3.1 概念


**Post-Training Quantization**：不需要重新训练，直接对训练好的模型进行量化。简单高效，适合快速部署。


### 3.2 量化粒度


| 粒度 | 说明 | 精度 | 开销 |
| --- | --- | --- | --- |
| Per-Tensor | 整个张量共享scale/zp | 较低 | 最小 |
| Per-Channel | 每个输出通道独立量化 | 较高 | 中等 |
| Per-Group | 每G个元素一组（如128个） | 最高 | 较大 |


### 3.3 重要方法


- **GPTQ**
   ：逐层量化，使用Hessian信息最小化量化误差。可将LLaMA-65B量化到4bit几乎无损
- **AWQ**
   （Activation-aware Weight Quantization）：关注权重中对激活值重要的通道，保护性量化
- **SmoothQuant**
   ：将量化难度从激活值转移到权重（激活值的离群值问题）
- **GGUF/GGML**
   ：llama.cpp专用格式，支持多种量化级别（Q2-Q8）


```
# GPTQ量化示例（使用auto-gptq）
from auto_gptq import AutoGPTQForCausalLM, BaseQuantizeConfig

quantize_config = BaseQuantizeConfig(
    bits=4,                     # 4-bit量化
    group_size=128,             # 每128个元素一组
    desc_act=True,              # 激活值排序
)

model = AutoGPTQForCausalLM.from_pretrained(
    "meta-llama/Llama-2-7b-hf",
    quantize_config=quantize_config,
)
model.quantize(calibration_dataset)
model.save_quantized("llama2-7b-gptq-4bit")
```


### 4.1 概念


**Quantization-Aware Training**：在训练过程中模拟量化效果，让模型"学会"容忍量化误差。精度通常优于PTQ。


### 4.2 前向传播模拟


```
# QAT前向传播
def fake_quantize(x, scale, zero_point, bits):
    # 量化（模拟）
    q = torch.clamp(torch.round(x / scale + zero_point), 0, 2**bits - 1)
    # 反量化
    x_hat = (q - zero_point) * scale
    # 直通估计器（STE）: 前向用量化值，反向梯度直接通过
    return x + (x_hat - x).detach()

# STE (Straight-Through Estimator)
# 解决round操作不可微分的问题
# 前向: 使用量化后的值
# 反向: 梯度直接传过，不经过round
```


### 4.3 QLoRA — 大模型高效微调


QLoRA将4-bit量化与LoRA结合，在极低显存下微调大模型：


- **NF4量化**
   ：针对正态分布优化的4-bit数据类型
- **双重量化（Double Quantization）**
   ：量化参数本身也量化，进一步节省显存
- **分页优化器**
   ：处理显存溢出
- 65B模型可在单张48GB GPU上微调


```
# QLoRA微调示例
from peft import LoraConfig, get_peft_model
from transformers import BitsAndBytesConfig

bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,                      # 4-bit加载
    bnb_4bit_quant_type="nf4",              # NF4量化
    bnb_4bit_use_double_quant=True,         # 双重量化
    bnb_4bit_compute_dtype=torch.bfloat16,  # 计算用BF16
)

model = AutoModelForCausalLM.from_pretrained(
    "meta-llama/Llama-2-70b-hf",
    quantization_config=bnb_config,
    device_map="auto",
)

# 冻结基础模型，只训练LoRA参数
lora_config = LoraConfig(r=64, lora_alpha=16, target_modules=["q_proj", "v_proj"])
model = get_peft_model(model, lora_config)
```


### 5.1 剪枝类型


| 类型 | 说明 | 需要微调 |
| --- | --- | --- |
| 非结构化剪枝 | 移除单个权重（产生稀疏矩阵） | 需要 |
| 结构化剪枝 | 移除整个神经元/注意力头/层 | 需要 |
| 半结构化剪枝 | N:M稀疏（如2:4，每4个权重保留2个） | 需要 |


### 5.2 剪枝方法


- **幅度剪枝（Magnitude Pruning）**
   ：移除绝对值最小的权重
- **梯度剪枝**
   ：移除对损失影响最小的权重
- **迭代剪枝**
   ：逐步增加稀疏度，每轮后微调
- **Lottery Ticket Hypothesis**
   ：存在稀疏子网络能达到与密集网络相当的性能


> **Note:** **实际应用：**
> NVIDIA Ampere/Hopper GPU支持2:4半结构化稀疏的硬件加速（Tensor Core稀疏模式），可在不损失精度的情况下实现2倍加速。


### 6.1 基本原理


训练小模型（Student）模仿大模型（Teacher）的输出分布或中间表示：


$$
L = α·L_CE(y, p_student) + (1-α)·T²·KL(p_teacher/T, p_student/T)
$$


- **T（温度）**
   ：软化概率分布，使暗知识（dark knowledge）更明显
- **α**
   ：平衡硬标签和软标签损失
- Teacher输出的软概率分布包含类间关系信息（暗知识）


### 6.2 LLM蒸馏方法


| 方法 | 原理 | 代表 |
| --- | --- | --- |
| 白盒蒸馏 | 可访问Teacher的logits | DistilBERT, TinyLlama |
| 黑盒蒸馏 | 只能访问Teacher的输出文本 | Alpaca, Vicuna |
| 思维链蒸馏 | 蒸馏推理过程（CoT） | Orca |
| 一致性蒸馏 | 用于Diffusion模型加速 | LCM, SDXL-Turbo |


### 7.1 ONNX Runtime


- 微软开源的跨平台推理引擎
- 支持多种硬件后端（CPU/GPU/NPU）
- 图优化（常量折叠、算子融合等）


### 7.2 TensorRT


- NVIDIA GPU专用推理优化器
- 层融合、精度校准（INT8/FP16）、Kernel自动调优
- 推理速度通常比PyTorch快2-5倍


### 7.3 vLLM


专为LLM推理优化的高性能框架：


- **PagedAttention**
   ：类似操作系统虚拟内存，按页管理KV Cache，显存利用率提升60%+
- **连续批处理（Continuous Batching）**
   ：动态插入新请求，不等待整批完成
- **量化支持**
   ：AWQ/GPTQ/FP8量化
- **吞吐量**
   ：比HuggingFace Transformers快14-24倍


```
# vLLM部署示例
from vllm import LLM, SamplingParams

llm = LLM(model="meta-llama/Llama-2-7b-hf", quantization="awq", dtype="auto")
sampling_params = SamplingParams(temperature=0.7, top_p=0.9, max_tokens=512)

outputs = llm.generate(["写一首关于春天的诗"], sampling_params)
for output in outputs:
    print(output.outputs[0].text)
```

### 8.1 原理


用小模型（Draft Model）快速生成多个候选token，然后用大模型（Target Model）并行验证，加速生成而不损失质量：


1. 小模型连续生成K个token（如5-10个）
2. 大模型并行验证这K个token的概率
3. 接受概率高于阈值的token，拒绝并重新采样低概率token
4. 保证最终输出与大模型独立生成完全一致


> **Note:** **加速效果：**
> 当小模型的预测准确率较高时（如70%以上），可实现2-3倍的加速，且输出质量与大模型完全相同（数学上等价）。


### 8.2 变体


- **Medusa**
   ：在大模型上附加多个预测头，同时预测多个未来token
- **EAGLE**
   ：利用大模型的中间特征加速自回归生成
- **Lookahead Decoding**
   ：无需Draft Model的投机解码
