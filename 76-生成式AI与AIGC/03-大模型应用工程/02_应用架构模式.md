# 应用架构模式

*Chatbot、Copilot、自主 Agent、知识库问答、代码生成与内容生成的架构设计*


| 模式 | 核心交互 | LLM 角色 | 复杂度 | 典型场景 |
| --- | --- | --- | --- | --- |
| **Chatbot** | 用户 ↔ LLM | 对话引擎 | 低 | 客服、闲聊 |
| **Copilot** | 用户 + LLM | 辅助增强 | 中 | 编程、写作 |
| **Autonomous Agent** | LLM → 工具 | 自主决策者 | 高 | 自动化任务 |
| **知识库问答** | LLM + 检索 | 有据回答 | 中 | 企业知识库 |
| **代码生成** | LLM → 代码 | 代码生产者 | 中高 | IDE 辅助 |
| **内容生成** | LLM → 内容 | 创作引擎 | 中 | 文案、营销 |

```
# 企业级 Chatbot 完整架构

# 1. 接入层
# - 多渠道适配（Web/App/微信/Slack）
# - 统一消息格式
# - 限流与鉴权

# 2. 编排层
# - 意图识别：判断用户意图（闲聊/问答/任务/投诉）
# - 路由分发：将请求路由到对应的处理链
# - 上下文管理：对话历史、用户画像、会话状态
# - 插件系统：动态加载功能模块

# 3. 模型层
# - LLM 调用：GPT-4/Claude/本地模型
# - Prompt 管理：版本控制、A/B 测试
# - 流式输出：SSE 实时返回

# 4. 知识层
# - RAG 检索：向量数据库 + BM25
# - 知识图谱：结构化知识查询
# - 文档解析：PDF/表格/Web

# 5. 安全层
# - 输入检测：Prompt 注入防护、敏感词过滤
# - 输出审核：内容安全、幻觉检测
# - 审计日志：全链路记录

# 关键设计决策
# - 流式响应：首 token 延迟 < 500ms
# - 降级策略：LLM 不可用时切换备用模型或返回固定回复
# - 缓存策略：语义缓存减少重复 LLM 调用
# - 多轮对话：滑动窗口 + 长对话摘要
```


### 3.1 Copilot vs Chatbot


Copilot（副驾驶）模式的核心是人机协作，AI 辅助而非替代。用户保持主导权，AI 提供建议和增强。


| 维度 | Chatbot | Copilot |
| --- | --- | --- |
| 交互方式 | 对话轮次 | 嵌入工作流 |
| 用户控制 | 低（AI 主导回复） | **高（用户决策）** |
| 上下文感知 | 对话历史 | **工作环境（代码/文档/数据）** |
| 响应时机 | 用户触发 | **主动/被动混合** |


### 3.2 GitHub Copilot 架构思路


```
# Copilot 架构核心组件

# 1. 上下文感知
# - 光标位置 + 打开文件 + 项目结构
# - 最近编辑历史 + 代码片段
# - 用户选中的代码（选中即意图）

# 2. 触发机制
# - 主动触发：用户按下快捷键
# - 被动触发：停止输入 N 秒后自动建议
# - 行内补全：实时逐 token 建议

# 3. 建议生成
# - 构建 Prompt：上下文代码 + 注释/签名
# - LLM 生成多个候选建议
# - 后处理：语法检查、过滤重复、格式化

# 4. 用户反馈循环
# - 接受/拒绝统计 → 改进模型
# - 用户修改建议 → 训练信号
# - A/B 测试不同策略
```


### 3.3 通用 Copilot 架构模式


```
# Copilot 架构模板

class CopilotSystem:
    def __init__(self, llm, context_manager, suggestion_engine):
        self.llm = llm
        self.context = context_manager
        self.suggestions = suggestion_engine

    def on_user_action(self, action):
        # 捕获用户操作
        context = self.context.capture(action)

        # 判断是否触发建议
        if self.should_suggest(context):
            suggestions = self.generate_suggestions(context)
            return self.suggestions.present(suggestions)

    def generate_suggestions(self, context):
        # 构建上下文 Prompt
        prompt = self.build_prompt(context)

        # LLM 生成
        raw = self.llm.generate(prompt, n=3)  # 多候选

        # 后处理：过滤、排序、格式化
        return self.postprocess(raw, context)

    def on_suggestion_feedback(self, suggestion_id, feedback):
        # 收集用户反馈
        self.learn(suggestion_id, feedback)
```

### 5.2 代码生成系统架构


```
# 代码生成系统核心架构

# 1. 代码上下文构建器
class CodeContextBuilder:
    def build(self, file_path, cursor_position):
        context = {
            "current_file": self.read_file(file_path),
            "cursor_pos": cursor_position,
            "imports": self.extract_imports(file_path),
            "project_structure": self.get_project_tree(),
            "related_files": self.find_references(file_path),
            "recent_edits": self.get_recent_edits(),
        }
        return context

# 2. 代码后处理器
class CodePostProcessor:
    def process(self, raw_code, language):
        # 语法检查
        if not self.check_syntax(raw_code, language):
            return self.fix_syntax(raw_code, language)

        # 导入补全
        raw_code = self.add_missing_imports(raw_code)

        # 格式化
        raw_code = self.format(raw_code, language)

        # 安全扫描
        issues = self.security_scan(raw_code)
        if issues:
            raw_code = self.fix_security(raw_code, issues)

        return raw_code

# 3. 执行验证沙箱
class CodeSandbox:
    def verify(self, code, test_cases):
        """在沙箱中执行代码并验证"""
        for test in test_cases:
            result = self.execute_in_sandbox(code, test.input)
            if result != test.expected:
                return False, f"测试失败: {test}"
        return True, "所有测试通过"
```


### 6.1 内容生成模式


| 模式 | 描述 | 架构特点 |
| --- | --- | --- |
| **模板填充** | 固定模板 + LLM 填写变量 | 结构化输出，质量稳定 |
| **自由创作** | LLM 根据主题自由生成 | 创意高但一致性低 |
| **迭代优化** | 生成 → 评审 → 修改循环 | 质量高但成本高 |
| **流水线** | 大纲 → 章节 → 润色分阶段 | 适合长内容 |


### 6.2 内容生成 Pipeline 架构


```
# 长内容生成 Pipeline（如营销文章）

# 阶段 1：规划
outline = planner_llm.generate(
    topic="AI Agent 市场分析",
    audience="技术决策者",
    format="分析报告",
    sections=5
)
# 输出：5 个章节的大纲 + 每章的核心论点

# 阶段 2：研究
for section in outline.sections:
    research = retriever.search(section.keywords)
    section.research_data = research

# 阶段 3：撰写
for section in outline.sections:
    content = writer_llm.generate(
        outline=section,
        research=section.research_data,
        style_guide=style_guide,
        word_count=800
    )
    section.draft = content

# 阶段 4：审查与润色
final = editor_llm.review(
    full_draft=outline.to_text(),
    checks=["事实准确性", "逻辑连贯", "风格一致", "SEO优化"]
)

# 阶段 5：多模态增强
final = add_images(final)  # 生成配图
final = add_citations(final)  # 添加引用
```


### 6.3 质量控制


> **Note:** **内容生成质量保障要点：**
>
> - **事实核查：**
>    LLM 生成的每个事实声明都需要通过 RAG 或搜索引擎验证
> - **原创性检测：**
>    查重并改写高相似度段落
> - **风格一致性：**
>    使用 style guide prompt 和全局上下文保持一致
> - **多轮审查：**
>    不同角色（事实审查员、风格编辑、SEO 优化师）依次审查
> - **A/B 测试：**
>    生成多个版本，通过用户反馈选择最优


| 需求 | 推荐架构 | 核心技术栈 |
| --- | --- | --- |
| 简单客服 | Chatbot + 意图路由 | LLM + 意图分类器 + 预设回复 |
| 企业知识问答 | RAG 问答 | 向量数据库 + Reranker + LLM |
| 编程辅助 | Copilot | 代码 LLM + 上文构建器 + 沙箱 |
| 自动化工作流 | Agent | LLM + 工具集 + 规划器 |
| 内容营销 | 内容生成 Pipeline | 多 LLM 串联 + RAG + 审查链 |
| 数据分析 | Copilot + Agent | 代码生成 + 数据连接器 + 可视化 |
