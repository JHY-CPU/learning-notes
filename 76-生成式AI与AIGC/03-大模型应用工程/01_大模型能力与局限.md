# 大模型能力与局限

*In-Context Learning、幻觉、上下文窗口与涌现能力*


### 1.1 为什么需要子词分词


大语言模型不直接处理字符或单词，而是处理**token**。子词分词在词级（词汇表过大）和字符级（序列过长）之间取得平衡。


| 方法 | 原理 | 使用者 |
| --- | --- | --- |
| BPE (Byte Pair Encoding) | 从字符开始，迭代合并最频繁的字符对 | GPT系列 |
| WordPiece | 类似BPE，但用似然而非频率选择合并 | BERT |
| SentencePiece | 语言无关的分词，直接处理原始文本 | T5, LLaMA |
| tiktoken | OpenAI的BPE实现，高效C库 | GPT-3.5/4 |


### 1.2 Token与上下文窗口


- 英文：平均约4个字符 = 1个token
- 中文：平均约1-2个汉字 = 1个token
- 上下文窗口指模型能处理的最大token数


```
# tiktoken使用示例
import tiktoken
enc = tiktoken.get_encoding("cl100k_base")  # GPT-4编码器
tokens = enc.encode("Hello, 你好！")
print(tokens)   # [9906, 11, 220, 36896, 5633, 29991]
print(len(tokens))  # 7 tokens
```


### 2.1 模型上下文长度演进


| 模型 | 上下文窗口 | 发布日期 |
| --- | --- | --- |
| GPT-2 | 1,024 tokens | 2019 |
| GPT-3 | 4,096 tokens | 2020 |
| GPT-3.5-turbo | 4K → 16K tokens | 2023 |
| GPT-4 | 8K → 128K tokens | 2023 |
| Claude 3 | 200K tokens | 2024 |
| Gemini 1.5 Pro | 1M tokens | 2024 |


### 2.2 长上下文的挑战


- **注意力计算量**
   ：标准注意力为O(n²)，1M token约需10¹²次计算
- **位置编码外推**
   ：训练长度外的位置编码可能失效
- **"大海捞针"问题**
   ：长上下文中定位关键信息的能力下降
- **中间信息遗忘**
   ：对上下文开头和结尾的信息记忆更好


### 2.3 长上下文扩展技术


| 技术 | 原理 |
| --- | --- |
| RoPE位置编码 | 旋转位置编码，支持外推（NTK-aware缩放） |
| ALiBi | 注意力偏置，无需显式位置编码 |
| Flash Attention | IO感知的精确注意力，内存高效 |
| Sliding Window | 只关注局部窗口内的token |
| KV Cache | 缓存已计算的Key和Value，避免重复计算 |


### 3.1 概念


ICL指模型通过在prompt中提供少量示例（不更新参数），就能"学会"新任务的能力。这是LLM最独特的涌现能力之一。


### 3.2 三种范式


| 范式 | 说明 | 示例 |
| --- | --- | --- |
| Zero-shot | 直接给出指令，不提供示例 | "将以下句子翻译成英文：你好" |
| One-shot | 提供一个示例 | "中文：谢谢→英文：Thank you。中文：你好→英文：" |
| Few-shot | 提供多个示例（通常3-5个） | 多个输入-输出对 + 新输入 |


### 3.3 ICL的工作机制


学术界对ICL的机制仍有争议，主流观点包括：


- **隐式贝叶斯推断**
   ：模型从示例中隐式学习任务的后验分布
- **梯度下降等价**
   ：ICL等价于在前向传播中隐式执行了一步梯度下降
- **模式匹配**
   ：模型在训练数据中见过类似模式，直接应用


> **Note:** **ICL的关键洞见：**
> ICL不需要微调模型参数，但需要足够的模型规模。小模型（如GPT-2级别的1.5B参数）ICL能力很弱，大模型（如GPT-3的175B参数）ICL能力显著。这支持了"涌现能力"假说。


### 4.1 什么是幻觉


大模型幻觉指模型生成的内容**看似合理但实际上不准确或完全编造**。这是LLM最严重的局限性之一。


### 4.2 幻觉的分类


| 类型 | 说明 | 示例 |
| --- | --- | --- |
| 事实性幻觉 | 编造不存在的事实 | 引用不存在的论文或人物 |
| 忠实性幻觉 | 回答与提供的上下文不符 | 总结时歪曲原文含义 |
| 指令幻觉 | 未按照指令执行 | 要求JSON输出却给文本 |


### 4.3 幻觉的根源


- **训练数据问题**
   ：数据中包含错误、过时或矛盾的信息
- **预训练目标**
   ：下一个token预测鼓励流畅性而非事实准确性
- **知识截止**
   ：模型对训练数据之后的信息一无所知
- **过度自信**
   ：模型倾向于生成高置信度的回答，即使不确定
- **分布偏移**
   ：推理时的输入与训练分布不同


### 4.4 缓解幻觉的方法


- **RAG（检索增强生成）**
   ：从外部知识库检索相关信息作为上下文
- **事实性约束解码**
   ：解码时引入事实性验证
- **RLHF/RLAIF**
   ：通过人类反馈或AI反馈对齐事实性
- **自我一致性（Self-Consistency）**
   ：多次采样取多数投票
- **引用溯源**
   ：要求模型提供信息来源，便于验证
- **不确定性表达**
   ：训练模型在不确定时说"我不知道"


### 5.1 定义


涌现能力指在小模型中不存在，但当模型规模超过某个阈值时突然出现的能力。这些能力不是通过显式训练获得的，而是模型规模增长的自然结果。


### 5.2 典型涌现能力


| 能力 | 说明 | 涌现阈值（估计） |
| --- | --- | --- |
| Chain-of-Thought推理 | 通过思维链进行多步推理 | ~100B参数 |
| 代码生成 | 根据自然语言描述编写代码 | ~10B参数 |
| 指令遵循 | 准确理解和执行复杂指令 | ~10B参数（经指令微调） |
| 多语言迁移 | 在一个语言上学的能力迁移到另一语言 | ~100B参数 |
| 算术推理 | 逐步解决数学问题 | ~100B参数 |
| 常识推理 | 隐含的常识知识推理 | 随规模渐进提升 |


### 5.3 关于涌现的争议


近期研究（Schaeffer et al., 2023）提出，涌现可能是**评估指标选择的假象**（如准确率是离散指标），使用连续指标（如token级准确率）后，能力是随规模渐进提升的。但无论是否"突然"出现，大模型确实展现出小模型不具备的复杂推理和泛化能力。


### 6.1 主要评测基准


| 基准 | 评测维度 | 说明 |
| --- | --- | --- |
| MMLU | 多任务语言理解 | 57个学科的多选题 |
| GSM8K | 数学推理 | 小学数学应用题 |
| HumanEval | 代码生成 | Python函数补全+单元测试 |
| ARC | 科学推理 | 小学科学选择题 |
| HellaSwag | 常识推理 | 句子续写合理性 |

| Chatbot Arena | 人类偏好 | ELO排名的盲评对比 |


### 7.1 RLHF（Reinforcement Learning from Human Feedback）


1. **SFT（监督微调）**
   ：用高质量指令数据微调基座模型
2. **训练奖励模型（RM）**
   ：用人类偏好对比数据训练评分模型
3. **PPO优化**
   ：用强化学习优化策略模型，使其输出获得更高的RM奖励


$$
L_PPO = E[min(r_t·A_t, clip(r_t, 1-ε, 1+ε)·A_t)] - c₁·L_V + c₂·S[π_θ]
$$


### 7.2 DPO（Direct Preference Optimization）


跳过奖励模型，直接用偏好对比数据优化策略：


$$
L_DPO = -E[log σ(β·(log π(y_w|x)/π_ref(y_w|x) - log π(y_l|x)/π_ref(y_l|x)))]
$$


- y_w为偏好回答（chosen），y_l为非偏好回答（rejected）
- 不需要单独训练RM，更简单高效
- 已成为RLHF的流行替代方案


### 7.3 对齐税（Alignment Tax）


安全对齐可能降低模型在某些任务上的性能（如创意写作、编码能力下降）。如何在安全性和能力之间取得平衡是重要研究方向。


### 8.1 核心技巧


| 技巧 | 说明 | 示例 |
| --- | --- | --- |
| 明确指令 | 清晰描述期望输出格式和约束 | "用JSON格式输出，包含name和age字段" |
| 角色扮演 | 指定模型的角色和专业领域 | "你是一位资深的Python工程师..." |
| 分步推理 | 要求模型逐步思考（CoT） | "请一步一步思考这个问题" |
| Few-shot | 提供示例 | 输入-输出示例对 |
| 输出约束 | 限制输出长度、格式等 | "请用不超过100字回答" |
| 负面示例 | 说明"不要做什么" | "不要编造信息，如果不确定就说不知道" |


### 8.2 常见失效模式


- **位置偏差**
   ：开头和结尾的信息权重高于中间（Lost in the Middle）
- **格式偏差**
   ：最近的few-shot示例影响更大
- **锚定效应**
   ：prompt中的数字或概念会影响后续输出
- **重复陷阱**
   ：长对话中模型容易开始重复
