# 视频生成

*Sora 架构推测、Video Diffusion Model、时序一致性、文生视频与 ControlNet for Video*


### 1.1 视频生成的挑战


相比图像生成，视频生成面临时序一致性、运动合理性、计算复杂度三重挑战。


| 挑战 | 说明 | 技术难点 |
| --- | --- | --- |
| 时序一致性 | 帧间物体外观和运动连贯 | 如何建模长程时间依赖 |
| 运动合理性 | 物理规律和运动逻辑 | 需要隐式学习物理常识 |
| 计算量 | 视频 = 图像 x 帧数 | 1分钟30fps = 1800帧 |
| 训练数据 | 高质量视频-文本对稀缺 | 标注成本极高 |
| 评估标准 | 缺乏统一的视频质量指标 | FVD 不够全面 |

### 2.2 Patch + DiT 架构

视频
(任意分辨率/时长)
→
VAE 编码
时空 Patch
→
DiT
去噪
→
VAE 解码
视频

- **时空 Patch：**
   将视频切割为时空小块（spacetime patches），类似 ViT 的图像 Patch 但在时间维度也做切分
- **DiT（Diffusion Transformer）：**
   用 Transformer 替代 U-Net 作为去噪骨干网络
- **原始分辨率训练：**
   不将视频缩放到固定分辨率，直接在原始分辨率和时长上训练
- **视频压缩网络：**
   使用时空 VAE 将视频压缩到低维潜空间


### 2.3 推测架构细节


```
# Sora 推测架构（学术复现理解）

# 1. 时空 VAE 编码
# 将视频 (T, H, W, 3) 压缩为 (T//4, H//8, W//8, C)
# 时间维度压缩 4 倍，空间维度各压缩 8 倍

# 2. Patch 化
# 将潜变量切割为 2x2x2 的时空 Patch
# 每个 Patch 展平为一个 token
# 总 token 数 = (T/4/2) x (H/8/2) x (W/8/2) = T*H*W / 256

# 3. DiT 骨干
# 标准 Transformer 架构 + AdaLN-Zero 条件化
# 条件：文本嵌入 + 时间步 + 视频元数据（分辨率/帧率/时长）

# 4. 关键创新
# - 可变分辨率：通过不同数量的 Patch 处理不同分辨率
# - 时序注意力：在 Patch 序列上做全局注意力，建模时间依赖
# - 文本条件：通过 Cross-Attention 注入文本语义
```


### 3.1 视频扩散的核心设计


视频扩散模型在图像扩散的基础上，新增时间维度建模。关键在于如何在 U-Net / DiT 中加入时间注意力。


### 3.2 时间建模方案


| 方案 | 原理 | 代表模型 |
| --- | --- | --- |
| **时间注意力层** | 在空间注意力后插入时间维度的 Self-Attention | Video LDM, SVD |
| **3D 卷积** | 将 2D 卷积扩展为 (t, h, w) 的 3D 卷积 | 早期视频模型 |
| **时序 Transformer** | 在 DiT 中对时间 Patch 做 Attention | Sora, Open-Sora |
| **因果注意力** | 时间维度只看过去帧，类似因果语言模型 | StreamingT2V |


### 3.3 Stable Video Diffusion (SVD)


```
# SVD 架构详解

# 基于 Stable Diffusion 2.1 的图像模型微调
# 核心改动：
# 1. 在 U-Net 的每个空间注意力层后插入时间注意力层
# 2. 训练分两阶段：图像预训练 → 视频微调

# 时间注意力层实现
class TemporalAttention(nn.Module):
    def __init__(self, dim, num_heads=8):
        self.norm = nn.GroupNorm(32, dim)
        self.attn = nn.MultiheadAttention(dim, num_heads)
        self.proj = nn.Linear(dim, dim)
        # 初始化为零（初始不改变图像模型行为）
        nn.init.zeros_(self.proj.weight)
        nn.init.zeros_(self.proj.bias)

    def forward(self, x):
        # x: (B, C, T, H, W)
        B, C, T, H, W = x.shape
        residual = x

        # 对每帧独立归一化
        x = self.norm(x.reshape(B*T, C, H, W)).reshape(B, C, T, H, W)

        # 重塑为 (T, B*H*W, C) 做时间注意力
        x = x.permute(2, 0, 3, 4, 1).reshape(T, B*H*W, C)

        # 时间 Self-Attention：每帧的每个空间位置关注所有帧
        x, _ = self.attn(x, x, x)
        x = self.proj(x)

        # 重塑回 (B, C, T, H, W)
        x = x.reshape(T, B, H, W, C).permute(1, 4, 0, 2, 3)
        return residual + x
```


### 4.1 帧间闪烁问题


视频生成最常见的问题是帧间闪烁（flickering）——相邻帧的细节不一致。解决方案包括：


| 技术 | 原理 | 效果 |
| --- | --- | --- |
| **时间注意力** | 建模帧间依赖，隐式对齐 | 基础方案 |
| **光流引导** | 用光流约束相邻帧的像素运动 | 运动连贯 |
| **噪声共享** | 相邻帧使用相关噪声初始化 | 减少闪烁 |
| **跨帧注意力** | 首帧作为参考帧，后续帧参考首帧特征 | 外观一致 |
| **时序 LoRA** | 冻结图像层，只微调时间层 | 高效适配 |

### 5.1 视频控制信号


将 ControlNet 扩展到视频领域，需要保持控制信号的时序一致性。


| 控制类型 | 提取方式 | 应用场景 |
| --- | --- | --- |
| 视频边缘 | Canny 逐帧检测 | 线稿动画 |
| 视频深度 | 单目深度估计 | 3D 场景控制 |
| 视频姿态 | OpenPose 逐帧 | 人体动画 |
| 视频法线 | 表面法线估计 | 光照控制 |
| 光流 | RAFT/FlowNet | 运动轨迹控制 |
| 参考图像 | CLIP 图像特征 | 风格/角色一致 |


### 5.2 Video ControlNet 实现思路


```
# ControlNet-Video 核心思路

# 方案一：逐帧 ControlNet + 时间注意力
# 每帧独立通过 ControlNet 分支提取条件特征
# 在 U-Net 的时间注意力层中隐式对齐条件信号

# 方案二：3D ControlNet
# 将 ControlNet 的 2D 卷积替换为 3D 卷积
# 条件输入为 (T, H, W) 的视频条件序列

# 方案三：SparseCtrl（稀疏帧控制）
# 只提供关键帧的控制信号
# 模型自动插值中间帧的控制

# IP-Adapter for Video（角色一致性）
# 使用首帧的图像特征作为参考
# 通过 Cross-Attention 保持角色外观一致
class VideoIPAdapter:
    def __init__(self, image_encoder, projection):
        self.image_encoder = image_encoder  # CLIP/DINOv2
        self.projection = projection  # 映射到文本嵌入空间

    def encode_reference(self, reference_image):
        # 编码参考图像
        features = self.image_encoder(reference_image)
        projected = self.projection(features)
        return projected

    def inject_features(self, unet, ref_features, frame_idx):
        # 将参考特征注入到每一帧的 Cross-Attention
        for block in unet.blocks:
            block.cross_attention_kv = ref_features  # 共享 KV
```
