# 扩散模型原理

*DDPM、Score Matching与去噪扩散 — 当前最强生成模型的数学基础*


**扩散模型（Diffusion Model）**是一类受非平衡热力学启发的生成模型，通过学习逐步去噪的过程来生成数据。2020年DDPM的提出使其成为当前最强的图像生成范式，在FID等指标上全面超越GAN。


| 特性 | GAN | VAE | Diffusion |
| --- | --- | --- | --- |
| 训练稳定性 | 差 | 好 | 好 |
| 生成质量 | 高 | 中 | 最高 |
| 多样性 | 易模式崩塌 | 好 | 最好 |
| 训练速度 | 快 | 快 | 慢 |
| 推理速度 | 极快（1步） | 快（1步） | 慢（多步采样） |


### 2.1 原理


前向过程是一个固定的马尔可夫链，逐步向数据添加高斯噪声，经过T步后数据变为纯噪声：

x₀
(原始图像)
→ +噪声
x₁
→ +噪声
x₂
→ ...
x_T
(纯噪声)

### 2.2 数学表达


$$
q(x_t | x_{t-1}) = N(x_t; √(1-β_t) · x_{t-1}, β_t · I)
$$


其中β_t是预定义的噪声调度（noise schedule），控制每步添加的噪声量。


### 2.3 重参数化：任意步采样


利用高斯分布的性质，可以直接从x₀跳到任意步x_t，无需逐步扩散：


$$
q(x_t | x₀) = N(x_t; √ᾱ_t · x₀, (1-ᾱ_t) · I)
$$


其中 ᾱ_t = ∏_{s=1}^{t} (1-β_s)。因此：


$$
x_t = √ᾱ_t · x₀ + √(1-ᾱ_t) · ε,　其中 ε ~ N(0, I)
$$


这使得训练时可以随机采样任意时间步t，高效计算。


```
# 前向扩散采样（直接跳到任意步）
def q_sample(x_start, t, noise=None):
    if noise is None:
        noise = torch.randn_like(x_start)
    sqrt_alpha = extract(sqrt_alphas_cumprod, t)
    sqrt_one_minus_alpha = extract(sqrt_one_minus_alphas_cumprod, t)
    return sqrt_alpha * x_start + sqrt_one_minus_alpha * noise
```


### 3.1 线性调度


DDPM原始论文使用线性调度：β从β₁=10⁻⁴线性增长到β_T=0.02。


### 3.2 余弦调度


线性调度在后期添加噪声过快，导致信息丢失。改进的余弦调度：


$$
ᾱ_t = f(t)/f(0),　f(t) = cos((t/T + s)/(1+s) · π/2)²
$$


余弦调度使噪声添加更均匀，显著提升生成质量。


### 3.3 调度对比


| 调度 | 特点 | 适用场景 |
| --- | --- | --- |
| 线性 | 简单直观 | 64×64图像 |
| 余弦 | 更平缓的衰减 | 高分辨率图像 |
| Scaled Linear | Stable Diffusion使用 | Latent空间 |
| Sqrt | 平方根衰减 | 特定任务 |


### 4.1 原理


反向过程学习从纯噪声逐步去噪，恢复出真实数据：

x_T
(纯噪声)
→ 去噪
x_{T-1}
→ 去噪
x_{T-2}
→ ...
x₀
(生成图像)

$$
p_θ(x_{t-1} | x_t) = N(x_{t-1}; μ_θ(x_t, t), Σ_θ(x_t, t))
$$


### 4.2 训练目标


模型可以预测三种目标之一：


| 预测目标 | 说明 | 应用 |
| --- | --- | --- |
| 噪声 ε | 预测添加的噪声（DDPM原始方案） | DDPM, Stable Diffusion |
| 原始图像 x₀ | 预测去噪后的图像 | DALL-E 2 |
| 速度 v | v = √ᾱ_t·ε - √(1-ᾱ_t)·x₀ | Improved DDPM |


### 4.3 损失函数


$$
L_simple = E_{t, x₀, ε}[||ε - ε_θ(x_t, t)||²]
$$


简单的MSE损失：让模型预测的噪声 ε_θ 尽量接近真实添加的噪声 ε。


```
# DDPM训练伪代码
for x₀ in dataloader:
    t = randint(1, T)                    # 随机时间步
    ε = randn_like(x₀)                   # 随机噪声
    x_t = √ᾱ_t · x₀ + √(1-ᾱ_t) · ε     # 前向扩散
    ε_pred = model(x_t, t)               # 模型预测噪声
    loss = MSE(ε, ε_pred)                 # 简单损失
    loss.backward()
    optimizer.step()
```


### 5.1 DDPM采样


```
# DDPM采样算法
def p_sample(model, x_t, t):
    # 模型预测噪声
    eps_pred = model(x_t, t)

    # 计算均值
    alpha_t = alphas[t]
    alpha_cumprod = alphas_cumprod[t]
    beta_t = betas[t]

    mean = (1/√α_t) * (x_t - (β_t/√(1-ᾱ_t)) * eps_pred)

    # 添加噪声（最后一步不加噪声）
    if t > 0:
        noise = torch.randn_like(x_t)
        sigma = √β_t
        x_{t-1} = mean + sigma * noise
    else:
        x_{t-1} = mean

    return x_{t-1}

# 完整采样：从x_T逐步降到x₀
x_T = torch.randn(1, 3, 256, 256)  # 纯噪声
x = x_T
for t in reversed(range(T)):
    x = p_sample(model, x, t)
# x_0就是生成的图像
```


### 5.2 DDIM采样（加速采样）


DDPM需要1000步采样，DDIM（Denoising Diffusion Implicit Models）通过非马尔可夫过程将采样步数减少到50-100步：


- 相同的训练模型，不同的采样策略
- 确定性采样（可设置η参数控制随机性）
- 采样步数可灵活调整


$$
x_{t-1} = √ᾱ_{t-1} · x̂₀ + √(1-ᾱ_{t-1}-σ²) · ε_θ + σ · z
$$


其中 x̂₀ = (x_t - √(1-ᾱ_t)·ε_θ) / √ᾱ_t 是模型预测的原始图像。


### 6.1 Score Function


Score函数定义为数据分布对数概率的梯度：


$$
s(x) = ∇_x log p(x)
$$


Score函数指向数据密度增加的方向（像一个指向高密度区域的向量场）。


### 6.2 与扩散模型的联系


噪声预测 ε_θ 与Score函数有直接关系：


$$
∇_{x_t} log q(x_t) = -ε_θ(x_t, t) / √(1-ᾱ_t)
$$


即：**预测噪声等价于估计Score函数**（差一个缩放因子）。


### 6.3 Score-Based生成


Score-based模型（Song & Ermon, 2019）直接学习Score函数，通过**朗之万动力学（Langevin Dynamics）**采样：


$$
x_{t+1} = x_t + (η/2)·∇_x log p(x_t) + √η·z,　z~N(0,I)
$$


> **Note:** **统一视角：**
> DDPM（预测噪声）和Score Matching（估计Score函数）是同一问题的两种表述。SDE框架（Score-Based SDE）将两者统一为连续时间的随机微分方程。


### 7.1 原理


Classifier-Free Guidance (CFG) 是实现文本引导生成的关键技术。模型同时训练条件和无条件两个版本，推理时通过加权插值增强条件控制：


$$
ε_guided = ε_∅ + w · (ε_c - ε_∅)
$$


- **ε_∅**
   ：无条件噪声预测（忽略文本）
- **ε_c**
   ：条件噪声预测（以文本为条件）
- **w（guidance scale）**
   ：引导强度，通常7-15
- w越大，生成结果越符合文本描述但多样性降低
- 训练时以一定概率（如10%）随机丢弃条件信息


| 方法 | 原理 | 采样步数 |
| --- | --- | --- |
| DDPM | 原始马尔可夫采样 | 1000 |
| DDIM | 非马尔可夫确定性采样 | 50-250 |
| DPM-Solver | 高阶ODE求解器 | 10-25 |
| UniPC | 统一预测-校正器 | 10-20 |
| Consistency Models | 蒸馏为单步生成 | 1-2 |
| Progressive Distillation | 迭代蒸馏减半步数 | 4-8 |
