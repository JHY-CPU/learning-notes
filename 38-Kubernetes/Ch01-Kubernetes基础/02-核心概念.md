# 核心概念

## 一、概念说明

K8s 核心对象：Pod、Node、Namespace、Label、Annotation。

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: my-pod
  namespace: default
  labels:
    app: web
    env: production
  annotations:
    description: "Web 服务器 Pod"
spec:
  containers:
  - name: nginx
    image: nginx:1.25
```

## 二、具体用法

### 2.1 Pod 生命周期

```bash
# Pod 状态：Pending → Running → Succeeded/Failed
kubectl get pods
kubectl describe pod my-pod
kubectl logs my-pod
kubectl logs my-pod -c sidecar  # 多容器日志
kubectl exec -it my-pod -- /bin/bash
```

### 2.2 Label 与选择器

```bash
# 添加标签
kubectl label pods my-pod version=v1
kubectl label pods my-pod env-  # 删除标签

# 按标签查询
kubectl get pods -l app=web
kubectl get pods -l 'env in (production, staging)'
kubectl get pods -l app=web,version=v1

# 标签选择器在 YAML 中使用
```

```yaml
spec:
  selector:
    matchLabels:
      app: web
    matchExpressions:
      - key: env
        operator: In
        values: [production, staging]
```

### 2.3 多容器 Pod

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: multi-container
spec:
  containers:
  - name: app
    image: my-app:latest
    ports:
    - containerPort: 8080
  - name: log-collector
    image: fluentd:latest
    volumeMounts:
    - name: logs
      mountPath: /var/log/app
  volumes:
  - name: logs
    emptyDir: {}
```

### 2.4 Init 容器

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: init-pod
spec:
  initContainers:
  - name: init-db
    image: busybox
    command: ['sh', '-c', 'until nslookup db; do sleep 2; done']
  - name: init-config
    image: busybox
    command: ['sh', '-c', 'wget -O /config/app.yaml http://config-server/config']
    volumeMounts:
    - name: config
      mountPath: /config
  containers:
  - name: app
    image: my-app:latest
    volumeMounts:
    - name: config
      mountPath: /app/config
  volumes:
  - name: config
    emptyDir: {}
```

### 2.5 Pod 阶段与条件

```
Pending:   Pod 已被接受，但容器未创建
Running:   Pod 已绑定到节点，至少一个容器运行中
Succeeded: 所有容器成功终止
Failed:    至少一个容器失败
Unknown:   无法获取 Pod 状态
```

## 三、注意事项与常见陷阱

1. **Pod 非持久**：Pod 可随时被销毁重建，不要存储本地数据
2. **单 Pod 单应用**：一般一个 Pod 只运行一个主容器
3. **健康检查**：生产环境必须配置 liveness/readiness 探针
4. **资源请求**：必须设置 requests 和 limits
5. **Sidecar 模式**：日志、监控等辅助容器使用 sidecar 模式
