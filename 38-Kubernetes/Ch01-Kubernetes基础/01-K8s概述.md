# K8s概述

## 一、概念说明

Kubernetes（K8s）是容器编排平台，自动化容器的部署、扩缩和管理。

```
集群架构：
┌─────────────────────────────────────────┐
│              Control Plane               │
│  API Server │ etcd │ Scheduler │ CM     │
└─────────────┬───────────────────────────┘
              │
┌─────────────┼───────────────────────────┐
│  Worker Node│    Worker Node            │
│  kubelet    │    kubelet                │
│  kube-proxy │    kube-proxy             │
│  Container  │    Container Runtime      │
│  Runtime    │                           │
└─────────────────────────────────────────┘
```

## 二、具体用法

### 2.1 安装 minikube

```bash
# macOS
brew install minikube

# Linux
curl -LO https://storage.googleapis.com/minikube/releases/latest/minikube-linux-amd64
sudo install minikube-linux-amd64 /usr/local/bin/minikube

# 启动集群
minikube start --driver=docker --nodes=3
minikube dashboard
```

### 2.2 kubectl 基础命令

```bash
# 集群信息
kubectl cluster-info
kubectl get nodes
kubectl describe node <node-name>

# 资源操作
kubectl get pods
kubectl get services
kubectl get deployments
kubectl get all

# 创建资源
kubectl create deployment nginx --image=nginx
kubectl expose deployment nginx --port=80 --type=NodePort

# 删除资源
kubectl delete deployment nginx
kubectl delete service nginx
```

### 2.3 YAML 资源定义

```yaml
# nginx-deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
  labels:
    app: nginx
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.25
        ports:
        - containerPort: 80
```

```bash
kubectl apply -f nginx-deployment.yaml
kubectl get deployments
kubectl get pods -l app=nginx
```

### 2.4 命名空间

```bash
# 查看命名空间
kubectl get namespaces

# 创建命名空间
kubectl create namespace my-app

# 在命名空间中操作
kubectl apply -f deployment.yaml -n my-app
kubectl get pods -n my-app

# 设置默认命名空间
kubectl config set-context --current --namespace=my-app
```

### 2.5 资源对象关系

```
Namespace
  └── Deployment (管理 Pod 副本)
        └── ReplicaSet (确保 Pod 数量)
              └── Pod (最小调度单元)
                    └── Container (运行应用)
```

## 三、注意事项与常见陷阱

1. **集群选择**：开发用 minikube/kind，生产用云托管服务
2. **kubectl 版本**：kubectl 版本应与集群版本兼容（偏差不超过一个小版本）
3. **YAML 格式**：缩进使用空格，不要用 Tab
4. **声明式管理**：优先使用 `apply` 而非 `create`
5. **资源清理**：不用的资源及时删除，避免浪费资源
