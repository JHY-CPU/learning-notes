# 日志管理

## 一、概念说明

K8s 日志管理采用 EFK（Elasticsearch + Fluentd/Fluent Bit + Kibana）或 Loki + Promtail + Grafana 方案。

```
Pod (stdout/stderr) → Fluent Bit (采集) → Loki (存储) → Grafana (查询)
```

## 二、具体用法

### 2.1 安装 Loki Stack

```bash
helm repo add grafana https://grafana.github.io/helm-charts
helm install loki grafana/loki-stack \
  --namespace logging --create-namespace \
  --set promtail.enabled=true \
  --set grafana.enabled=true
```

### 2.2 查看日志

```bash
# kubectl 查看
kubectl logs <pod-name>
kubectl logs <pod-name> -c sidecar
kubectl logs -f <pod-name>                  # 实时跟踪
kubectl logs --previous <pod-name>          # 上次容器日志
kubectl logs -l app=web                     # 按标签

# 多 Pod 日志
kubectl logs -l app=web --all-containers=true --prefix=true
```

### 2.3 Promtail 配置

```yaml
# Promtail 配置
config:
  clients:
  - url: http://loki:3100/loki/api/v1/push
  snippets:
    pipelineStages:
    - docker: {}
    - match:
        selector: '{app="my-app"}'
        stages:
        - regex:
            expression: '(?P<level>\w+) (?P<msg>.*)'
        - labels:
            level:
    - match:
        selector: '{app="nginx"}'
        stages:
        - regex:
            expression: '.*(?P<status>\d{3}).*'
        - labels:
            status:
```

### 2.4 LogQL 查询

```logql
# 基础查询
{namespace="production", app="web"}

# 过滤
{app="web"} |= "error"
{app="web"} |= "error" != "timeout"
{app="web"} |~ "error|ERROR"

# 统计
count_over_time({app="web"}[5m])
sum(rate({app="web"} | unwrap duration [5m])) by (level)

# JSON 解析
{app="web"} | json | level="error"
```

### 2.5 日志保留策略

```yaml
# Loki 配置
config:
  limits_config:
    retention_period: 30d
  schema_config:
    configs:
    - from: 2020-10-24
      store: boltdb-shipper
      object_store: filesystem
      schema: v11
      index:
        prefix: index_
        period: 24h
```

## 三、注意事项与常见陷阱

1. **stdout 推荐**：容器日志写入 stdout/stderr，由 K8s 收集
2. **资源规划**：日志量大时需要规划存储和保留策略
3. **敏感信息**：日志中不要包含密码等敏感信息
4. **多行日志**：Java 异常堆栈等多行日志需要特殊配置
5. **性能影响**：高频日志可能影响应用性能
