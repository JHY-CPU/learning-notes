# 监控体系

## 一、概念说明

K8s 监控体系包括 Metrics Server、Prometheus、Grafana，覆盖集群、节点、Pod 三级监控。

```
Grafana (可视化) → Prometheus (采集存储) → Pod/Node Exporter (数据源)
```

## 二、具体用法

### 2.1 Prometheus 安装

```bash
# 使用 Helm 安装 kube-prometheus-stack
helm repo add prometheus-community https://prometheus-community.github.io/helm-charts
helm install prometheus prometheus-community/kube-prometheus-stack \
  --namespace monitoring --create-namespace
```

### 2.2 ServiceMonitor

```yaml
apiVersion: monitoring.coreos.com/v1
kind: ServiceMonitor
metadata:
  name: my-app-monitor
  namespace: monitoring
spec:
  selector:
    matchLabels:
      app: my-app
  namespaceSelector:
    matchNames:
    - default
  endpoints:
  - port: metrics
    interval: 15s
    path: /metrics
```

### 2.3 Pod 指标暴露

```yaml
# 应用需暴露 /metrics 端点
apiVersion: v1
kind: Pod
metadata:
  annotations:
    prometheus.io/scrape: "true"
    prometheus.io/port: "8080"
    prometheus.io/path: "/metrics"
spec:
  containers:
  - name: app
    image: my-app:v1
    ports:
    - name: metrics
      containerPort: 8080
```

### 2.4 告警规则

```yaml
apiVersion: monitoring.coreos.com/v1
kind: PrometheusRule
metadata:
  name: app-alerts
  labels:
    release: prometheus
spec:
  groups:
  - name: app
    rules:
    - alert: HighCPU
      expr: container_cpu_usage_seconds_total{container="app"} > 0.8
      for: 5m
      labels:
        severity: warning
      annotations:
        summary: "CPU 使用率过高"
    - alert: PodCrashLooping
      expr: increase(kube_pod_container_status_restarts_total[1h]) > 5
      for: 5m
      labels:
        severity: critical
```

### 2.5 Grafana Dashboard

```bash
# 访问 Grafana
kubectl port-forward svc/prometheus-grafana 3000:80 -n monitoring

# 默认账号: admin / prom-operator

# 导入常用 Dashboard：
# - 315: K8s Cluster
# - 7249: K8s Node
# - 13770: K8s Pod
```

## 三、注意事项与常见陷阱

1. **存储规划**：Prometheus 数据量大，需要规划存储
2. **高可用**：生产环境使用 Thanos 或 Cortex 实现高可用
3. **告警降噪**：合理设置 for 持续时间和告警阈值
4. **资源消耗**：Prometheus 本身需要较多资源
5. **数据保留**：配置 retention 策略避免存储爆满
