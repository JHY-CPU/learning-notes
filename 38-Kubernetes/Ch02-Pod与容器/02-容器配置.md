# 容器配置

## 一、概念说明

容器配置包括镜像选择、端口映射、资源限制、安全上下文等。

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: configured-pod
spec:
  containers:
  - name: app
    image: my-app:v1.0
    command: ["/app/server"]
    args: ["--port=8080", "--log-level=info"]
    workingDir: /app
    ports:
    - containerPort: 8080
      name: http
      protocol: TCP
```

## 二、具体用法

### 2.1 镜像拉取策略

```yaml
spec:
  containers:
  - name: app
    image: my-app:v1
    imagePullPolicy: Always      # 总是拉取
    # imagePullPolicy: IfNotPresent  # 本地没有时拉取（默认）
    # imagePullPolicy: Never     # 从不拉取
```

### 2.2 安全上下文

```yaml
spec:
  securityContext:
    runAsUser: 1000
    runAsGroup: 3000
    fsGroup: 2000
  containers:
  - name: app
    image: my-app:v1
    securityContext:
      allowPrivilegeEscalation: false
      readOnlyRootFilesystem: true
      capabilities:
        drop:
        - ALL
        add:
        - NET_BIND_SERVICE
```

### 2.3 探针配置

```yaml
spec:
  containers:
  - name: app
    image: my-app:v1
    livenessProbe:          # 存活探针
      httpGet:
        path: /healthz
        port: 8080
      initialDelaySeconds: 30
      periodSeconds: 10
      failureThreshold: 3
    readinessProbe:         # 就绪探针
      httpGet:
        path: /ready
        port: 8080
      initialDelaySeconds: 5
      periodSeconds: 5
    startupProbe:           # 启动探针
      httpGet:
        path: /healthz
        port: 8080
      failureThreshold: 30
      periodSeconds: 10
```

### 2.4 TCP 和 Exec 探针

```yaml
# TCP 探针
livenessProbe:
  tcpSocket:
    port: 3306
  initialDelaySeconds: 15
  periodSeconds: 20

# Exec 探针
livenessProbe:
  exec:
    command:
    - cat
    - /tmp/healthy
  initialDelaySeconds: 5
  periodSeconds: 5
```

### 2.5 启动与终止

```yaml
spec:
  terminationGracePeriodSeconds: 60  # 优雅关闭等待时间
  containers:
  - name: app
    image: my-app:v1
    lifecycle:
      preStop:
        exec:
          command: ["/bin/sh", "-c", "sleep 5 && /app/graceful-shutdown"]
```

## 三、注意事项与常见陷阱

1. **探针设计**：liveness 探针失败会重启容器，readiness 失败会从 Service 移除
2. **启动延迟**：慢启动应用需要配置 startupProbe
3. **安全实践**：禁止特权运行，使用只读文件系统
4. **资源限制**：不设置 limits 可能影响其他 Pod
5. **镜像版本**：使用具体版本标签，不要用 latest
