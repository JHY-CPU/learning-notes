# 健康检查与探针

## 一、概念说明

K8s 提供三种探针检测容器健康状态：liveness（存活）、readiness（就绪）、startup（启动）。

```yaml
spec:
  containers:
  - name: app
    image: my-app:v1
    livenessProbe:
      httpGet:
        path: /healthz
        port: 8080
      periodSeconds: 10
      failureThreshold: 3
    readinessProbe:
      httpGet:
        path: /ready
        port: 8080
      periodSeconds: 5
```

## 二、具体用法

### 2.1 HTTP 探针

```yaml
livenessProbe:
  httpGet:
    path: /api/health
    port: 8080
    scheme: HTTP
    httpHeaders:
    - name: X-Custom-Header
      value: "health-check"
  initialDelaySeconds: 30
  periodSeconds: 10
  timeoutSeconds: 5
  successThreshold: 1
  failureThreshold: 3
```

### 2.2 TCP 探针

```yaml
# 适用于数据库等非 HTTP 服务
livenessProbe:
  tcpSocket:
    port: 5432
  initialDelaySeconds: 15
  periodSeconds: 20
```

### 2.3 Exec 探针

```yaml
# 执行命令检查
readinessProbe:
  exec:
    command:
    - sh
    - -c
    - "pg_isready -U postgres"
  periodSeconds: 10
```

### 2.4 gRPC 探针

```yaml
# K8s 1.24+ 支持
livenessProbe:
  grpc:
    port: 50051
  initialDelaySeconds: 10
```

### 2.5 探针行为组合

```yaml
spec:
  containers:
  - name: web
    image: nginx
    # 启动探针：慢启动应用
    startupProbe:
      httpGet:
        path: /healthz
        port: 8080
      failureThreshold: 30
      periodSeconds: 10
    # 存活探针：检测死锁等
    livenessProbe:
      httpGet:
        path: /healthz
        port: 8080
      periodSeconds: 10
    # 就绪探针：检测是否可以接收流量
    readinessProbe:
      httpGet:
        path: /ready
        port: 8080
      periodSeconds: 5
```

## 三、注意事项与常见陷阱

1. **探针轻量**：健康检查不应消耗过多资源
2. **启动时间**：initialDelaySeconds 要覆盖应用启动时间
3. **级联故障**：readiness 探针过度敏感可能导致滚动更新失败
4. **资源依赖**：探针不应依赖外部服务（如数据库）
5. **日志影响**：频繁失败的探针会产生大量日志事件
