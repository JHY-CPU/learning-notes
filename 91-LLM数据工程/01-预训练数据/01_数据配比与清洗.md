# 数据配比与清洗 - LLM预训练数据

*大语言模型预训练数据工程的核心环节，涵盖多源数据配比策略、质量过滤方法、去重技术及有毒内容过滤的完整Pipeline*

五大核心数据源

| 数据源 | 代表数据集 | 特点 | 典型占比 |
| --- | --- | --- | --- |
| **网页数据** | Common Crawl, C4, RedPajama | 规模最大、噪声最多、需严格清洗 | 60-80% |
| **书籍数据** | Books3, Gutenberg, BookCorpus | 长文本、高质量叙事、逻辑连贯 | 5-15% |
| **代码数据** | The Stack, GitHub Code, StackOverflow | 结构化强、逻辑严谨、提升推理能力 | 5-15% |
| **百科/学术** | Wikipedia, ArXiv, PubMed | 事实密度高、知识覆盖广 | 3-8% |
| **对话/论坛** | Reddit, StackExchange | QA格式、口语化表达、指令理解 | 3-10% |

GPT-3 数据配比 (OpenAI)

- **Common Crawl (过滤后)**
   : 60% - 使用类目器过滤低质量网页
- **WebText2**
   : 22% - 高质量网页文本（Reddit投票筛选）
- **Books1 + Books2**
   : 8% - 互联网书籍语料
- **Wikipedia**
   : 3% - 高质量百科知识
- **关键发现**
   : 经过质量过滤的CC数据比原始CC数据效果提升显著，质量远比数量重要

Llama 系列数据配比 (Meta)

- **Llama 1 (1.4T tokens)**
   : Common Crawl 67%, C4 15%, GitHub 4.5%, Wikipedia 4.5%, Books 4.5%, ArXiv 2.5%, StackExchange 2%
- **Llama 2 (2T tokens)**
   : 大幅增加高质量数据比例，训练数据量增加40%
- **Llama 3 (15T+ tokens)**
   : 数据量激增至15T tokens，强调数据质量scaling，代码数据占比大幅提升

常用启发式规则

- **文本长度过滤**
   : 去除过短（<50词）或过长（异常值）的文档
- **重复n-gram比例**
   : 检测重复的3-gram/4-gram片段，去除重复率超过阈值的文档（如30%以上重复）
- **特殊字符比例**
   : 过滤标点符号、URL、HTML标签比例异常的文档
- **语言检测**
   : 使用fastText或langdetect过滤非目标语言文档
- **单词/句子比例**
   : 过滤"脏"文本（如乱码、字符集错误导致的低质量内容）
- **脏词表**
   : 使用黑名单过滤包含大量脏话、仇恨言论的内容

分类器训练方法

- **正样本**
   : Wikipedia等高质量文本作为正样本
- **负样本**
   : 随机采样的Common Crawl网页作为负样本
- **模型**
   : 通常使用 fastText 或轻量级Transformer分类器
- **特征**
   : 文本本身的语义特征 + 统计特征（长度、重复率、特殊字符比例等）
- **打分**
   : 对每篇文档输出0-1的质量分数，按阈值截断

MinHash-LSH 工作流程

1. **Shingling**
   : 将文档转为 k-shingle 集合（通常 k=5-13）
2. **MinHash**
   : 使用多组哈希函数计算文档签名（通常128-256个哈希函数）
3. **LSH分桶**
   : 将签名分为多个band，同一band内哈希值相同的文档进入同一桶
4. **候选对检测**
   : 同一桶内的文档作为候选相似对
5. **Jaccard验证**
   : 计算候选对的真实Jaccard相似度，超过阈值（如0.8）则判定为重复


**典型工具**: datasketch, text-dedup, cc_net

GPT-3 数据处理 Pipeline

1. 从Common Crawl下载约45TB原始HTML
2. URL过滤：基于与Wikipedia的链接相似性进行加权采样
3. 文档级去重：基于SimHash进行近似去重
4. 训练质量分类器：正样本为WebText，负样本为随机CC页面
5. 根据分类器分数对文档进行加权采样
6. 最终数据集约570GB（过滤后仅保留原始数据的约1.2%）

Llama 数据处理 Pipeline

1. **Llama 1**
   : 使用CCNet对Common Crawl进行处理，按Wikipedia-like程度打分，取前2个epoch
2. **Llama 2**
   : 在Llama 1基础上新增更多高质量数据来源，更严格的去重（行级去重），去除含有大量个人信息的文档
3. **Llama 3**
   : 全面升级数据工程，包括：更精细的PII过滤、多轮质量分类器迭代、合成数据增强、代码数据比例大幅提升

RefinedWeb (Falcon) 数据处理

- 专注于从Common Crawl中提取高质量纯文本
- 严格的启发式过滤规则（去除导航文本、广告等）
- 两阶段去重：URL去重 + MinHash文档去重
- 仅使用网页数据就达到了与混合数据训练模型相当的性能
- 证明了
   **高质量网页数据的潜力被低估**
