# SFT数据构造 - LLM数据工程

*监督微调(Supervised Fine-Tuning)是将预训练基座模型转化为指令跟随模型的关键步骤，数据质量与多样性决定了SFT的上限*

Alpaca数据特点

- **数据量**
   : 52K条指令-响应对
- **生成方式**
   : 使用GPT-3.5(text-davinci-003)自动生成
- **Self-Instruct**
   : 基于175条种子任务，通过LLM自我扩展
- **优点**
   : 格式简洁，易于理解和复现
- **缺点**
   : 响应较短、质量参差不齐、缺乏多轮对话能力

Vicuna训练细节

- **数据来源**
   : ShareGPT收集的70K用户与ChatGPT的真实对话
- **基座模型**
   : LLaMA-13B
- **训练方式**
   : 只计算assistant回复的loss（屏蔽user输入的loss）
- **成本**
   : 约$300的训练成本（8×A100，约24小时）
- **效果**
   : 在GPT-4评估中达到ChatGPT的90%水平

LIMA 实验结论

- **数据量**
   : 仅使用1,000条精心策划的高质量指令-响应对
- **数据来源**
   : StackExchange(200) + WikiHow(200) + 手工编写(600)
- **基座模型**
   : LLaMA-65B
- **结果**
   : 在人类评估中与GPT-4的胜率接近，显著优于使用52K Alpaca数据训练的模型
- **结论**
   : SFT的本质是
   **教会模型对话格式和风格**
   ，大部分知识已经在预训练中学到

五种进化操作

1. **Add Constraints（添加约束）**
   : 在原始指令上增加额外要求


   例: "写一个排序算法" → "写一个排序算法，要求时间复杂度O(n log n)，空间复杂度O(1)，不能使用递归"
2. **Deepen（加深难度）**
   : 提升任务的复杂度层级


   例: "解释for循环" → "解释for循环和while循环的区别，并给出何时选择各自的建议"
3. **Concretize（具体化）**
   : 将抽象指令变为具体场景


   例: "写一个Python函数" → "写一个Python函数，输入一个字符串列表，返回按字母序排列后长度超过5的字符串"
4. **Increase Reasoning（增加推理步骤）**
   : 要求更多推理步骤


   例: "计算2+3" → "逐步分析并计算: 如果小明有2个苹果，小红给他的苹果数是小明原有数量的1.5倍，他们一共有多少苹果？"
5. **Breadth（扩展广度）**
   : 涉及相关领域的知识


   例: "什么是神经网络" → "从生物学、数学、计算机科学三个角度解释神经网络的概念"
