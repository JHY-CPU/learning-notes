# 偏好数据构造 - LLM数据工程

*偏好数据是RLHF/DPO对齐训练的核心，涵盖Comparison数据构造、排序数据、AI反馈(RLAIF)及多源标注策略*

Comparison（对比）数据格式

```
{
    "prompt": "解释量子纠缠",
    "chosen": "量子纠缠是量子力学中的一种现象，当两个粒子发生纠缠后，
无论它们相隔多远，对其中一个粒子的测量会瞬间影响另一个粒子的状态。
爱因斯坦称之为'鬼魅般的超距作用'...",
    "rejected": "量子纠缠就是两个量子连在一起了，就像双胞胎一样，
一个人动另一个人也动。"
}

# chosen: 人类偏好的高质量回答
# rejected: 人类不偏好的低质量回答
# 关键：同一prompt下的两个回答必须有可比性
```

人类标注流程（以InstructGPT为例）

1. **生成候选**
   : 对每个prompt，模型生成4-9个候选回答
2. **排序**
   : 标注者对候选回答进行排序（从最好到最差）
3. **对比对提取**
   : 从排序中提取所有 (chosen, rejected) 对
4. **一致性检查**
   : 多人标注同一数据，计算标注一致性（Krippendorff's alpha）


**标注者要求**: 英语母语、经过培训、通过资格测试、持续监控标注质量

GPT-4标注 vs 人类标注对比

| 维度 | 人类标注 | GPT-4标注 |
| --- | --- | --- |
| 成本 | 高（$0.5-2/条） | 低（$0.01-0.05/条） |
| 速度 | 慢（每人每天100-200条） | 快（数千条/小时） |
| 一致性 | 中等（受疲劳影响） | 高（但有系统性偏差） |
| 偏好偏差 | 文化/个人偏好 | 偏好冗长、格式化的回答 |
| 安全判断 | 敏感度高 | 可能过度保守 |
| 专业知识 | 取决于标注者背景 | 广泛但可能不深入 |

Constitutional AI 流程

1. **生成阶段**
   : 模型对有害prompt生成回答
2. **批评阶段**
   : 模型根据"宪法原则"批评自己的回答
3. **修订阶段**
   : 模型根据批评修订回答
4. **偏好标注**
   : 使用修订后的回答作为chosen，原始回答作为rejected
5. **训练**
   : 用偏好数据训练Reward Model，再进行RLHF

候选生成方法

- **温度采样**
   : 使用不同temperature（0.7-1.2）生成多样化的回答
- **Top-p采样**
   : 使用不同top-p值（0.8-0.95）控制多样性
- **多次采样**
   : 同一参数多次采样，获得同一质量水平的不同表述
- **不同模型**
   : 使用不同大小/版本的模型生成质量差异
- **对抗构造**
   : 人为构造有缺陷的回答（如包含事实错误、遗漏关键点）
- **拒绝采样**
   : 生成后用验证器过滤，保留有质量梯度的候选
