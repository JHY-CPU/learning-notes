# 合成数据生成 - LLM数据工程

*利用大语言模型自身能力生成训练数据，涵盖Self-Instruct、Evol-Instruct、模型蒸馏、合成数据风险及最佳实践*

In-depth Evolution（深度进化）

随机选择以下操作之一对指令进行进化：


1. **Add Constraints**
   : 增加数量/格式/风格约束

   *"写一个排序算法"* → *"写一个原地排序算法，不允许使用额外数组空间，时间复杂度O(n log n)"*
2. **Deepen**
   : 增加推理步骤或复杂度层级

   *"什么是过拟合"* → *"解释过拟合现象，分析其三个主要原因，并给出五种解决方案及其适用场景"*
3. **Concretize**
   : 将抽象指令具体化为场景

   *"解释机器学习"* → *"假设你向一位10岁的小朋友解释什么是机器学习，用三个日常生活的例子"*
4. **Increase Reasoning**
   : 要求多步推理

   *"计算25%折扣"* → *"一件原价800元的衣服先打7折再叠加满500减100，最终价格比直接打6折便宜多少？"*
5. **Broaden**
   : 扩展涉及知识领域

   *"写一个REST API"* → *"从RESTful设计原则、安全性考虑、性能优化三个角度设计一个用户管理API"*

蒸馏数据生成策略

- **指令跟随蒸馏**
   : 用GPT-4生成高质量指令-响应对，训练较小模型
- **思维链蒸馏**
   : 用强模型生成推理过程（CoT），训练弱模型的推理能力
- **概率蒸馏**
   : 使用强模型的token概率分布（而非仅输出）训练弱模型
- **拒绝采样**
   : 生成多个候选，用验证器选择最好的作为训练数据

模型坍缩机制

```
第0代（真实数据训练）
    ┌─────────────────────────────────┐
    │ ▓▓▓▓░░░░░░░░░░░░░░░░░░░░░░░░░░ │  ← 完整分布覆盖
    └─────────────────────────────────┘
         ↓ 生成合成数据
第1代（合成数据训练）
    ┌─────────────────────────────────┐
    │ ▓▓▓▓▓▓▓▓░░░░░░░░░░░░░░░░░░░░░░ │  ← 高概率区域被放大，尾部开始消失
    └─────────────────────────────────┘
         ↓ 生成合成数据
第2代（合成数据训练）
    ┌─────────────────────────────────┐
    │ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓░░░░░░░░░░░░░░░░ │  ← 分布进一步收缩
    └─────────────────────────────────┘
         ↓ 多次迭代后
第N代
    ┌─────────────────────────────────┐
    │ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓ │  ← 坍缩到窄分布，多样性丧失
    └─────────────────────────────────┘
```


**原因**: 模型对自身高概率区域过度采样，低概率区域逐渐被遗忘。每次迭代都会加剧这一趋势。

合成数据驱动的成功模型

| 模型/数据集 | 合成数据方法 | 关键成果 |
| --- | --- | --- |
| Alpaca | Self-Instruct (GPT-3.5) | 52K数据，$500成本，首次证明开源模型可接近ChatGPT |

| UltraChat | 多轮对话合成 | 1.5M多轮对话，覆盖30个主题 |
| Cosmopedia | 教科书式生成 | HuggingFace用Mixtral生成25M教科书级文档 |
