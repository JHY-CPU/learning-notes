# 数据质量评估 - LLM数据工程

*系统化的数据质量评估方法论，从困惑度过滤、分类器打分到数据价值量化(DataShapley)，构建完整的数据质量评估体系*

评估方法对比

- **自动化评估**
   : 成本低、速度快、可扩展、一致性高，但可能存在系统偏差
- **人工评估**
   : 直觉准确、可处理模糊情况、但成本高、速度慢、一致性受主观影响
- **最佳实践**
   : 自动化方法覆盖全局筛选，人工评估进行抽检校准。通常自动评估处理100%数据，人工评估抽检5-10%

典型训练流程

1. **正样本**
   : Wikipedia页面、教科书、学术论文等已知高质量文本
2. **负样本**
   : 随机采样的Common Crawl网页（包含各种质量水平）
3. **特征提取**
   :

   - 文本嵌入特征（Sentence-BERT、fastText）
   - 统计特征（长度、词汇丰富度、标点比例、重复率）
   - 语言学特征（句法复杂度、词汇密度）
4. **模型选择**
   : fastText（速度优先）、逻辑回归（可解释）、轻量级BERT（精度优先）
5. **打分与截断**
   : 对所有候选文档打分，按分数排序截断或加权采样

分层抽检方案

- **随机抽检**
   : 随机抽取5%数据，评估整体质量水平
- **边界抽检**
   : 集中检查分类器分数在阈值附近的样本，评估过滤边界是否合理
- **异常抽检**
   : 检查异常值（极长/极短/极高困惑度），验证异常处理是否正确
- **分层抽检**
   : 按数据来源分层抽检，确保每个来源的数据质量
- **趋势抽检**
   : 在数据处理Pipeline的每个阶段抽检，追踪质量变化

Shapley Value 定义

Shapley Value来自合作博弈论，用于公平分配联盟收益给每个参与者。在数据场景中：


- **参与者**
   : 每条训练数据
- **联盟**
   : 数据子集
- **收益**
   : 在该子集上训练模型后在验证集上的性能
- **分配**
   : 每条数据对总性能的边际贡献


```
# Shapley Value 公式
# φ_i = Σ_{S⊆N\{i}} [|S|!(|N|-|S|-1)!/|N|!] * [v(S∪{i}) - v(S)]

# φ_i: 数据点i的Shapley值
# N: 全部训练数据
# S: 不包含i的数据子集
# v(S): 在子集S上训练的模型性能
```

Data Cartography (数据地图)

将每条训练数据映射到二维空间：


- **X轴**
   : 训练置信度（模型对该样本的平均预测概率）
- **Y轴**
   : 训练方差（不同epoch中预测是否一致）
- **Easy样本**
   : 高置信度、低方差 → 可能是简单或重复样本
- **Hard样本**
   : 低置信度、高方差 → 可能是噪声标注或异常样本
- **Ambiguous样本**
   : 中等置信度、高方差 → 对模型学习最有价值


**应用**: 基于数据地图进行数据子集选择（如只用ambiguous样本训练），可在保留90%性能的同时减少50%数据
