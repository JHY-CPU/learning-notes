# 骨骼绑定基础

## 核心概念

骨骼绑定（Rigging）是将一个抽象的骨架结构附加到3D模型上的过程。骨骼由一系列层级化的关节（Joint/Bone）组成，每个关节带有变换信息（位置、旋转、缩放），驱动网格顶点变形形成动画。骨骼绑定的质量直接决定了角色动画的自然程度和制作效率。

## 数学基础：关节变换矩阵

### 局部空间变换

每个关节的变换都是相对于其父关节定义的。设关节 $j$ 的局部变换矩阵为 $L_j$，包含平移 $T$、旋转 $R$ 和缩放 $S$：

$$L_j = T_j \cdot R_j \cdot S_j$$

### 世界空间级联

关节的世界变换通过从根节点到当前节点的矩阵连乘得到：

$$W_j = W_{parent(j)} \cdot L_j$$

其中根关节的 $W_{root} = L_{root}$。

### 蒙皮矩阵（Skin Matrix）

蒙皮矩阵将顶点从绑定姿态的模型空间变换到当前动画姿态的模型空间：

$$M_{skin,j} = W_j \cdot B_j^{-1}$$

其中 $B_j$ 是绑定姿态（Bind Pose）下关节 $j$ 的世界变换矩阵，$B_j^{-1}$ 是其逆矩阵。这个公式的含义是：先用 $B_j^{-1}$ 将顶点从模型空间变换到关节的局部空间，再用 $W_j$ 变换到当前动画姿态的模型空间。

### 四元数旋转

在动画系统中，旋转通常用四元数 $q = (w, x, y, z)$ 表示，避免欧拉角的万向节死锁问题。四元数的旋转公式：

$$v' = q \cdot v \cdot q^{-1}$$

四元数乘法（组合两次旋转）：

$$q_3 = q_1 \cdot q_2 = (w_1 w_2 - \mathbf{v_1} \cdot \mathbf{v_2}, \; w_1 \mathbf{v_2} + w_2 \mathbf{v_1} + \mathbf{v_1} \times \mathbf{v_2})$$

## 骨骼层级结构

骨骼是一棵以根关节（Root）为根的树形结构：

```
Root (根关节 - 控制角色在世界中的位置和朝向)
 └─ Hips (骨盆 - 重心所在)
     ├─ Spine (腰椎)
     │   ├─ Spine1 (胸椎)
     │   │   ├─ Spine2 (上胸)
     │   │   │   ├─ Neck (颈部)
     │   │   │   │   └─ Head (头部)
     │   │   │   │       ├─ LeftEye
     │   │   │   │       └─ RightEye
     │   │   │   ├─ LeftShoulder (左肩)
     │   │   │   │   └─ LeftUpperArm (左上臂)
     │   │   │   │       └─ LeftLowerArm (左前臂)
     │   │   │   │           └─ LeftHand (左手)
     │   │   │   │               ├─ LeftThumb1→2→3
     │   │   │   │               ├─ LeftIndex1→2→3
     │   │   │   │               └─ LeftMiddle1→2→3 ...
     │   │   │   └─ RightShoulder (右肩) ...
     │   │   └─ ...
     │   └─ ...
     ├─ LeftUpLeg (左大腿)
     │   └─ LeftLeg (左小腿)
     │       └─ LeftFoot (左脚)
     │           └─ LeftToeBase (左脚趾)
     └─ RightUpLeg (右大腿) ...
```

每个关节的变换是相对于父关节的（Local Space），最终变换需要从根到叶级联到世界空间。

## 完整 C++ 骨骼系统实现

```cpp
#include <vector>
#include <string>
#include <unordered_map>
#include <glm/glm.hpp>
#include <glm/gtc/quaternion.hpp>
#include <glm/gtx/quaternion.hpp>

using Vec3 = glm::vec3;
using Quat = glm::quat;
using Mat4 = glm::mat4;

// 单个关节的数据结构
struct Joint {
    int id;                  // 关节唯一ID
    int parentId;            // 父关节ID (-1表示根关节)
    std::string name;        // 关节名称

    // 绑定姿态下的局部变换
    Vec3 bindLocalPos;
    Quat bindLocalRot;
    Vec3 bindLocalScale;

    // 绑定姿态下的世界变换和逆矩阵（预计算）
    Mat4 bindWorldMatrix;
    Mat4 inverseBindMatrix;

    // 当前动画的局部变换（每帧更新）
    Vec3 animLocalPos;
    Quat animLocalRot;
    Vec3 animLocalScale;

    // 计算后的世界变换和蒙皮矩阵
    Mat4 worldMatrix;
    Mat4 skinMatrix;
};

// 骨骼类
class Skeleton {
public:
    std::vector<Joint> joints;
    std::unordered_map<std::string, int> nameToId;  // 名字到索引的映射
    int rootJointId = 0;

    // 从建模软件导入后初始化绑定姿态
    void InitBindPose() {
        // 从根关节递归计算世界变换
        ComputeWorldMatrix(rootJointId, Mat4(1.0f));

        // 预计算每个关节的逆绑定矩阵
        for (auto& joint : joints) {
            joint.inverseBindMatrix = glm::inverse(joint.bindWorldMatrix);
            // 初始化当前动画变换为绑定姿态
            joint.animLocalPos = joint.bindLocalPos;
            joint.animLocalRot = joint.bindLocalRot;
            joint.animLocalScale = joint.bindLocalScale;
        }
    }

    // 递归计算世界变换矩阵
    void ComputeWorldMatrix(int jointId, const Mat4& parentWorld) {
        Joint& j = joints[jointId];

        // 构造局部变换矩阵: T * R * S
        Mat4 T = glm::translate(Mat4(1.0f), j.bindLocalPos);
        Mat4 R = glm::toMat4(j.bindLocalRot);
        Mat4 S = glm::scale(Mat4(1.0f), j.bindLocalScale);
        Mat4 localMatrix = T * R * S;

        j.bindWorldMatrix = parentWorld * localMatrix;

        // 递归处理所有子关节
        for (auto& child : joints) {
            if (child.parentId == jointId) {
                ComputeWorldMatrix(child.id, j.bindWorldMatrix);
            }
        }
    }

    // 每帧更新：根据动画数据计算所有蒙皮矩阵
    void UpdatePose() {
        UpdateJointWorld(rootJointId, Mat4(1.0f));
    }

    void UpdateJointWorld(int jointId, const Mat4& parentWorld) {
        Joint& j = joints[jointId];

        Mat4 T = glm::translate(Mat4(1.0f), j.animLocalPos);
        Mat4 R = glm::toMat4(j.animLocalRot);
        Mat4 S = glm::scale(Mat4(1.0f), j.animLocalScale);
        Mat4 localMatrix = T * R * S;

        j.worldMatrix = parentWorld * localMatrix;
        // 蒙皮矩阵 = 当前世界变换 * 绑定姿态逆矩阵
        j.skinMatrix = j.worldMatrix * j.inverseBindMatrix;

        for (auto& child : joints) {
            if (child.parentId == jointId) {
                UpdateJointWorld(child.id, j.worldMatrix);
            }
        }
    }

    // 根据名称查找关节
    int FindJoint(const std::string& name) const {
        auto it = nameToId.find(name);
        return (it != nameToId.end()) ? it->second : -1;
    }
};
```

## Bind Pose（绑定姿态）

### 定义与作用

Bind Pose（也称为 Rest Pose 或 T-Pose）是骨骼与网格绑定时的参考姿态，具有以下关键作用：

1. **逆绑定矩阵计算**：在此姿态下计算每个关节的世界变换 $B_j$，然后求逆得到 $B_j^{-1}$
2. **蒙皮权重基准**：所有顶点的蒙皮权重基于此姿态计算
3. **动画偏移基准**：所有动画数据都是相对于 Bind Pose 的偏移量

### T-Pose vs A-Pose

| 姿态 | 优点 | 缺点 | 使用场景 |
|------|------|------|---------|
| T-Pose | 手臂完全伸展，权重计算简单 | 腋下区域权重处理困难 | Mixamo, 通用标准 |
| A-Pose | 腋下自然，变形效果更好 | 需要额外的动画适配 | Unreal Mannequin, 3A游戏 |

### Bind Pose 统一的重要性

在大型项目中，所有角色必须使用统一的 Bind Pose，否则：
- 动画重定向（Retargeting）会失败，因为不同 Bind Pose 的关节局部空间不同
- 共享动画资源时会出现肢体扭曲
- IK 解算器假设的标准姿势不匹配导致错误结果

### 完整导入代码示例

```cpp
#include <assimp/Importer.hpp>
#include <assimp/scene.h>
#include <assimp/postprocess.h>

class SkeletonImporter {
public:
    Skeleton* ImportFromFBX(const std::string& filepath) {
        Assimp::Importer importer;
        const aiScene* scene = importer.ReadFile(filepath,
            aiProcess_Triangulate | aiProcess_JoinIdenticalVertices);

        if (!scene || !scene->mRootNode) {
            fprintf(stderr, "FBX导入失败: %s\n", importer.GetErrorString());
            return nullptr;
        }

        Skeleton* skeleton = new Skeleton();

        // 第一遍：收集所有骨骼节点
        CollectBones(scene->mRootNode, scene, -1, skeleton);

        // 第二遍：从mesh的bone权重中提取bind pose矩阵
        ExtractBindMatrices(scene, skeleton);

        // 初始化绑定姿态
        skeleton->InitBindPose();

        return skeleton;
    }

private:
    void CollectBones(aiNode* node, const aiScene* scene,
                      int parentId, Skeleton* skeleton) {
        // 检查此节点是否是骨骼（在mesh的bone列表中出现）
        std::string nodeName(node->mName.C_Str());

        int jointId = -1;
        if (IsBoneNode(nodeName, scene)) {
            Joint joint;
            joint.id = (int)skeleton->joints.size();
            joint.parentId = parentId;
            joint.name = nodeName;

            // 提取局部变换
            aiMatrix4x4 m = node->mTransformation;
            aiVector3D pos, scale;
            aiQuaternion rot;
            m.Decompose(scale, rot, pos);

            joint.bindLocalPos = Vec3(pos.x, pos.y, pos.z);
            joint.bindLocalRot = Quat(rot.w, rot.x, rot.y, rot.z);
            joint.bindLocalScale = Vec3(scale.x, scale.y, scale.z);

            skeleton->joints.push_back(joint);
            skeleton->nameToId[nodeName] = joint.id;
            jointId = joint.id;
        }

        // 递归处理子节点
        for (unsigned int i = 0; i < node->mNumChildren; i++) {
            CollectBones(node->mChildren[i], scene,
                        jointId >= 0 ? jointId : parentId, skeleton);
        }
    }

    void ExtractBindMatrices(const aiScene* scene, Skeleton* skeleton) {
        for (unsigned int i = 0; i < scene->mNumMeshes; i++) {
            aiMesh* mesh = scene->mMeshes[i];
            for (unsigned int b = 0; b < mesh->mNumBones; b++) {
                aiBone* bone = mesh->mBones[b];
                std::string boneName(bone->mName.C_Str());

                int jointId = skeleton->FindJoint(boneName);
                if (jointId >= 0) {
                    // Assimp的mOffsetMatrix就是逆绑定矩阵
                    aiMatrix4x4 m = bone->mOffsetMatrix;
                    // 转换为glm::mat4（注意转置）
                    skeleton->joints[jointId].inverseBindMatrix = glm::mat4(
                        m.a1,m.b1,m.c1,m.d1,
                        m.a2,m.b2,m.c2,m.d2,
                        m.a3,m.b3,m.c3,m.d3,
                        m.a4,m.b4,m.c4,m.d4
                    );
                }
            }
        }
    }

    bool IsBoneNode(const std::string& name, const aiScene* scene) {
        for (unsigned int i = 0; i < scene->mNumMeshes; i++) {
            aiMesh* mesh = scene->mMeshes[i];
            for (unsigned int b = 0; b < mesh->mNumBones; b++) {
                if (std::string(mesh->mBones[b]->mName.C_Str()) == name)
                    return true;
            }
        }
        return false;
    }
};
```

## 坐标系转换

不同建模软件使用不同的坐标系，导入时必须正确转换：

| 软件 | 上方向 | 前方向 | 左手/右手 |
|------|--------|--------|----------|
| Maya | Y | -Z | 右手 |
| Blender | Z | -Y | 右手 |
| 3ds Max | Z | Y | 左手 |
| Unity | Y | Z | 左手 |
| Unreal | Z | X | 左手 |

### 转换公式

Maya (Y-up, 右手) 转 Unity (Y-up, 左手)：

```cpp
Mat4 MayaToUnity(const Mat4& mayaMatrix) {
    // 翻转 Z 轴
    Mat4 flip = glm::scale(Mat4(1.0f), Vec3(1, 1, -1));
    return flip * mayaMatrix * flip;
}

Vec3 MayaToUnityPos(const Vec3& pos) {
    return Vec3(pos.x, pos.y, -pos.z);  // 翻转Z
}

Quat MayaToUnityRot(const Quat& q) {
    return Quat(q.w, q.x, q.y, -q.z);  // 翻转Z分量
}
```

## 常见陷阱与解决方案

### 陷阱 1：关节层级过深导致精度丢失

**问题**：超过 15-20 层级深度时，浮点矩阵连乘累积误差明显，末端关节位置漂移。

**解决方案**：
```cpp
// 定期重新计算世界矩阵（从绑定姿态重新开始）
// 或使用双精度中间计算
void RecomputeFromBindPose() {
    for (auto& joint : joints) {
        // 使用双精度进行关键计算
        glm::dmat4 world = glm::dmat4(1.0);
        // ... 递归计算
        joint.worldMatrix = glm::mat4(world);  // 最终转回单精度
    }
}
```

### 陷阱 2：Bind Pose 不统一导致动画重定向失败

**问题**：角色 A 使用 T-Pose，角色 B 使用 A-Pose，交换动画时肢体扭曲。

**解决方案**：在导入管线中强制统一到标准 Bind Pose，或使用重定向（Retargeting）系统进行姿态转换。

### 陷阱 3：导出时未烘焙关键帧

**问题**：在 Maya 中使用约束驱动的动画，导出 FBX 时约束未被烘焙为关键帧，导致动画丢失。

**解决方案**：导出前执行 Bake Simulation，将所有约束效果转换为逐帧关键帧数据。

### 陷阱 4：非均匀缩放导致法线错误

**问题**：关节存在非均匀缩放（如 X=1, Y=2, Z=1），蒙皮后的法线方向不正确。

**解决方案**：使用逆转置矩阵计算法线变换：

```cpp
Mat4 normalMatrix = glm::transpose(glm::inverse(skinMatrix));
// 或在导出前应用所有缩放，消除非均匀缩放
```

## 与替代方案对比

| 方案 | 优势 | 劣势 | 适用场景 |
|------|------|------|---------|
| 骨骼蒙皮 | 业界标准，硬件加速，成熟管线 | 需要绑定权重，关节处变形受限 | 绝大多数游戏角色 |
| 变形目标(Morph) | 适合面部表情，高精度变形 | 内存开销大，不适合全身 | 面部动画，表情系统 |
| 布料物理模拟 | 真实的布料运动 | 计算开销大，不稳定 | 披风、旗帜等非关键部位 |
| 程序化变形 | 无需预制作 | 实现复杂，难以控制艺术效果 | 肌肉模拟，特殊效果 |

## 实际游戏案例

- **mixamo**：Adobe 提供的标准化人形骨骼和动画库，FBX 格式，广泛用于原型开发
- **Unreal Mannequin**：UE5 的标准骨骼模板，A-Pose，包含完整 Physics Asset
- **Unity Humanoid Avatar**：Mecanim 系统的标准化人形骨骼映射，支持动画重定向
- **MetaHuman (UE5)**：超高精度面部骨骼（约 200+ 面部骨骼），支持实时面部捕捉
- **God of War Ragnarok**：父子骨骼系统 + 程序化叠加层，实现武器和护甲的动态变形
- **荒野大镖客2**：马匹使用独立骨骼系统，四足动物的骨骼层级与人形完全不同
