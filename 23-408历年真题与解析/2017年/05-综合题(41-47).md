## 第41题 (10分) - 数据结构

已知一个带头结点的单链表L，结点数据域为整型且无序。请设计一个算法，删除单链表中所有数据值为x的结点。要求：

(1) 给出算法的基本设计思想。(3分)
(2) 用C/C++语言描述该算法。(5分)
(3) 说明算法的时间复杂度和空间复杂度。(2分)

**解析：**

(1) **基本设计思想：**

由于带头结点，可以设置一个前驱指针pre指向当前结点的前驱。从头结点之后的第一个数据结点开始遍历，若当前结点的数据值等于x，则删除该结点（将前驱的next指向当前结点的next，释放当前结点）；否则，前驱指针和当前指针同时后移。

(2) **算法描述：**

```c
typedef struct LNode {
    int data;
    struct LNode *next;
} LNode, *LinkList;

// 删除带头结点单链表中所有值为x的结点
void deleteAllX(LinkList L, int x) {
    LNode *pre = L;        // 前驱指针，初始指向头结点
    LNode *p = L->next;    // 当前指针，初始指向第一个数据结点
    
    while (p != NULL) {
        if (p->data == x) {
            pre->next = p->next;  // 删除p指向的结点
            free(p);
            p = pre->next;        // p指向下个待检查的结点
        } else {
            pre = p;
            p = p->next;
        }
    }
}
```

(3) **复杂度分析：**

- **时间复杂度：** O(n)，只需遍历链表一次，n为链表长度。
- **空间复杂度：** O(1)，只使用了固定数量的辅助指针。

## 第42题 (10分) - 数据结构

已知一棵二叉树的先序遍历序列和中序遍历序列分别存储在数组pre[0..n-1]和in[0..n-1]中，n为二叉树的结点数。请编写一个算法，根据这两个序列构造出该二叉树的二叉链表存储结构。要求：

(1) 给出算法的基本设计思想。(3分)
(2) 用C/C++语言描述该算法。(5分)
(3) 说明算法的时间复杂度。(2分)

**解析：**

(1) **基本设计思想：**

利用分治思想。先序遍历序列的第一个元素为当前子树的根结点。在中序遍历序列中找到该根元素的位置，则该位置左边的元素属于左子树，右边的元素属于右子树。根据左右子树的元素个数，可以确定先序遍历序列中左右子树的范围。递归构造左右子树。

(2) **算法描述：**

```c
typedef struct BiTNode {
    char data;
    struct BiTNode *lchild, *rchild;
} BiTNode, *BiTree;

// 根据先序序列pre[preL..preR]和中序序列in[inL..inR]构造二叉树
BiTree createBT(char pre[], char in[], int preL, int preR, int inL, int inR) {
    if (preL > preR) return NULL; // 空子树
    
    // 先序序列的第一个元素为根结点
    BiTNode *root = (BiTNode *)malloc(sizeof(BiTNode));
    root->data = pre[preL];
    root->lchild = NULL;
    root->rchild = NULL;
    
    // 在中序序列中查找根结点的位置
    int k;
    for (k = inL; k <= inR; k++) {
        if (in[k] == pre[preL]) break;
    }
    
    int numLeft = k - inL; // 左子树的结点个数
    
    // 递归构造左子树
    root->lchild = createBT(pre, in, preL+1, preL+numLeft, inL, k-1);
    // 递归构造右子树
    root->rchild = createBT(pre, in, preL+numLeft+1, preR, k+1, inR);
    
    return root;
}
```

(3) **复杂度分析：**

- **时间复杂度：** O(n)，每个结点被访问一次。查找根在中序中的位置虽然需要O(n)，但所有查找的总和为O(n)，因此总体时间复杂度为O(n)。
- **空间复杂度：** O(h)，h为树的深度，递归调用栈的深度。

## 第43题 (10分) - 计算机组成原理

某计算机的Cache采用2路组相联映射方式，Cache数据区容量为8KB，主存容量为1MB，主存和Cache均按64B的块大小进行分块。主存地址按字节编址。

(1) 写出主存地址各字段（标记、组号、块内偏移）的位数。(4分)
(2) 若Cache初始为空，依次访问主存地址0x00000, 0x00040, 0x10000, 0x00040, 0x20000，写出每次访问后Cache的状态。（假设采用LRU替换策略）(6分)

**解析：**

(1) **地址字段划分：**

- 主存容量1MB=2²⁰B，主存地址20位
- 块大小64B=2⁶B，块内偏移6位（log₂64=6）
- Cache容量8KB=2¹³B，Cache块数=2¹³/2⁶=2⁷=128块
- 2路组相联，组数=128/2=64=2⁶组，组号6位
- 标记位数=20-6-6=8位

地址格式：| 标记(8位) | 组号(6位) | 块内偏移(6位) |

(2) **访问过程：**

- 0x00000：标记=0x00，组号=0，块内偏移=0。组0空，调入组0第1路。状态：组0=[(tag=0x00, LRU=0)]
- 0x00040：0x00040=0000 0000 0000 0100 0000。块内偏移6位=0，组号=0x01=1，标记=0x00。组1空，调入组1。状态：组1=[(tag=0x00)]
- 0x10000：0x10000=0001 0000 0000 0000 0000。块内偏移=0，组号=0x00=0，标记=0x40。组0已有一个tag=0x00的块，组0未满，调入组0第2路。状态：组0=[(tag=0x00),(tag=0x40)]
- 0x00040：组号=1，标记=0x00，命中组1。
- 0x20000：0x20000=0010 0000 0000 0000 0000。块内偏移=0，组号=0x00=0，标记=0x80。组0满（2路），LRU替换最近最少使用的。tag=0x00是最久未用的，替换掉。组0=[(tag=0x80),(tag=0x40)]

## 第44题 (10分) - 操作系统

在某操作系统中，采用银行家算法避免死锁。设系统有A、B、C三类资源，现有5个进程P0~P4，某时刻系统状态如下：

| 进程 | Allocation(A,B,C) | Max(A,B,C) | Available(A,B,C) |
|------|-------------------|------------|-------------------|
| P0 | 0,1,0 | 7,5,3 | 3,3,2 |
| P1 | 2,0,0 | 3,2,2 | |
| P2 | 3,0,2 | 9,0,2 | |
| P3 | 2,1,1 | 2,2,2 | |
| P4 | 0,0,2 | 4,3,3 | |

(1) 计算每个进程的Need矩阵。(3分)
(2) 判断当前状态是否安全，若安全则给出一个安全序列。(4分)
(3) 若进程P1提出请求Request(1,0,2)，系统能否立即分配？(3分)

**解析：**

(1) **Need矩阵（Need=Max-Allocation）：**

| 进程 | Need(A,B,C) |
|------|-------------|
| P0 | 7,4,3 |
| P1 | 1,2,2 |
| P2 | 6,0,0 |
| P3 | 0,1,1 |
| P4 | 4,3,1 |

(2) **安全性检查：**

Available=(3,3,2)

- P1：Need(1,2,2)≤Available(3,3,2)，可以满足。P1完成后释放Allocation(2,0,0)，Available=(5,3,2)
- P3：Need(0,1,1)≤Available(5,3,2)，可以满足。P3完成后释放(2,1,1)，Available=(7,4,3)
- P0：Need(7,4,3)≤Available(7,4,3)，可以满足。P0完成后释放(0,1,0)，Available=(7,5,3)
- P2：Need(6,0,0)≤Available(7,5,3)，可以满足。P2完成后释放(3,0,2)，Available=(10,5,5)
- P4：Need(4,3,1)≤Available(10,5,5)，可以满足。

**安全，安全序列为P1→P3→P0→P2→P4**（不唯一，满足条件的序列均可）。

(3) **P1请求Request(1,0,2)：**

Step 1：Request(1,0,2)≤Need(1,2,2)？是。

Step 2：Request(1,0,2)≤Available(3,3,2)？是。

Step 3：尝试分配，更新状态：
- Available = (3,3,2)-(1,0,2) = (2,3,0)
- Allocation_P1 = (2,0,0)+(1,0,2) = (3,0,2)
- Need_P1 = (1,2,2)-(1,0,2) = (0,2,0)

Step 4：安全性检查（Available=(2,3,0)）：
- P3：Need(0,1,1)≤(2,3,0)？(0,1,1)中B分量1≤3成立，C分量1≤0不成立。不可满足。
- P1：Need(0,2,0)≤(2,3,0)？是。P1完成后释放(3,0,2)，Available=(5,3,2)
- P3：Need(0,1,1)≤(5,3,2)？是。完成后Available=(7,4,3)
- P0：Need(7,4,3)≤(7,4,3)？是。完成后Available=(7,5,3)
- P2：Need(6,0,0)≤(7,5,3)？是。完成后Available=(10,5,5)
- P4：Need(4,3,1)≤(10,5,5)？是。

安全序列：P1→P3→P0→P2→P4。**系统可以立即分配。**

## 第45题 (10分) - 操作系统

某系统的磁盘有200个磁道，当前磁头位于第100号磁道，磁头正向磁道号增加的方向移动。磁道访问请求序列为55, 58, 39, 18, 90, 160, 150, 38, 180。

(1) 使用FCFS调度算法，写出访问顺序和磁头移动总距离。(3分)
(2) 使用SSTF调度算法，写出访问顺序和磁头移动总距离。(3分)
(3) 使用SCAN调度算法（从100开始向磁道号增加方向），写出访问顺序和磁头移动总距离。(4分)

**解析：**

**(1) FCFS算法：**

访问顺序：100→55→58→39→18→90→160→150→38→180

磁头移动距离：
|100-55|+|55-58|+|58-39|+|39-18|+|18-90|+|90-160|+|160-150|+|150-38|+|38-180|
=45+3+19+21+72+70+10+112+142=**494**

**(2) SSTF算法：**

从100开始，每次选择最近的请求：
- 100→90（距离10）
- 90→58（距离32）
- 58→55（距离3）
- 55→39（距离16）
- 39→38（距离1）
- 38→18（距离20）
- 18→150（距离132）
- 150→160（距离10）
- 160→180（距离20）

访问顺序：90, 58, 55, 39, 38, 18, 150, 160, 180
磁头移动距离：10+32+3+16+1+20+132+10+20=**244**

**(3) SCAN算法：**

磁头从100向磁道号增加方向移动，到达端点（199号磁道）后返回。
请求中大于等于100的：150, 160, 180（从小到大排列）
请求中小于100的：18, 38, 39, 55, 58, 90（从小到大排列）

访问顺序：100→150→160→180→199(端点)→90→58→55→39→38→18

磁头移动距离：
|100-150|+|150-160|+|160-180|+|180-199|+|199-90|+|90-58|+|58-55|+|55-39|+|39-38|+|38-18|
=50+10+20+19+109+32+3+16+1+20=**280**

## 第46题 (10分) - 计算机网络

某公司有4个部门，分别使用4个子网：192.168.1.0/24, 192.168.2.0/24, 192.168.3.0/24, 192.168.4.0/24。现需要将这4个子网聚合为一个超网。

(1) 写出聚合后的超网地址和子网掩码。(4分)
(2) 若某路由器收到目的地址为192.168.5.10的数据报，该数据报是否属于该超网？(2分)
(3) 假设各子网的主机数分别为120, 60, 30, 10台，若要重新划分子网以节约地址空间，请给出一种合理的子网划分方案。(4分)

**解析：**

(1) **超网聚合：**

4个子网的第三个字节分别为1,2,3,4，二进制为：
- 192.168.1.0：...0000 0001
- 192.168.2.0：...0000 0010
- 192.168.3.0：...0000 0011
- 192.168.4.0：...0000 0100

由于4的二进制为0100，与1(0001), 2(0010), 3(0011)的公共前缀不一致，无法用单个CIDR块精确聚合这4个子网。实际上，1,2,3可以聚合为192.168.0.0/23（覆盖0-1），但4不在此范围内。如果强行聚合，最小覆盖需要到192.168.0.0/21（覆盖第三个字节0-7），超网地址为**192.168.0.0/21**，子网掩码为255.255.248.0。

(2) **检查192.168.5.10：**

192.168.5.10的第三个字节5=00000101。超网192.168.0.0/21覆盖第三个字节0~7（00000000~00000111）。5在范围内，因此**属于该超网**。

(3) **子网划分方案：**

各子网需要的主机数及对应的主机位数：
- 120台：需要2⁷-2=126≥120，主机位7位，子网掩码/25
- 60台：需要2⁶-2=62≥60，主机位6位，子网掩码/26
- 30台：需要2⁵-2=30≥30，主机位5位，子网掩码/27
- 10台：需要2⁴-2=14≥10，主机位4位，子网掩码/28

划分方案（假设地址池为192.168.0.0/24）：
- 子网1：192.168.0.0/25（126个可用地址）
- 子网2：192.168.0.128/26（62个可用地址）
- 子网3：192.168.0.192/27（30个可用地址）
- 子网4：192.168.0.224/28（14个可用地址）

这样可以使用一个/24网段满足所有4个部门的需求，节约了3个/24网段的地址空间。

## 第47题 (10分) - 计算机网络

设TCP的拥塞控制中，初始拥塞窗口cwnd=1MSS，慢启动阈值ssthresh=16MSS。请画出从开始传输到cwnd=20MSS过程中，cwnd随RTT变化的曲线（标出关键点），并说明每个阶段的变化原因。假设在cwnd=20MSS时发生了超时事件。

**解析：**

**cwnd变化过程：**

**慢启动阶段（Slow Start）：** 每个RTT，cwnd翻倍。
- RTT 1: cwnd = 1MSS
- RTT 2: cwnd = 2MSS
- RTT 3: cwnd = 4MSS
- RTT 4: cwnd = 8MSS
- RTT 5: cwnd = 16MSS（达到ssthresh）

**拥塞避免阶段（Congestion Avoidance）：** 每个RTT，cwnd增加1MSS。
- RTT 6: cwnd = 17MSS
- RTT 7: cwnd = 18MSS
- RTT 8: cwnd = 19MSS
- RTT 9: cwnd = 20MSS

**超时事件发生：**
- 新的ssthresh = cwnd/2 = 20/2 = 10MSS
- cwnd = 1MSS（重置为1）
- 重新进入慢启动阶段

**变化曲线示意：**
```
cwnd(MSS)
20 |                    * (超时)
19 |                  *
18 |                *
17 |              *
16 |          * ------ 拥塞避免阶段（线性增长）
 8 |      *
 4 |    *
 2 |  *
 1 | *  慢启动阶段（指数增长）
   +------------------------→ RTT
   1  2  3  4  5  6  7  8  9
```

超时后：
```
cwnd(MSS)
10 |        新的ssthresh
 1 | *  重置cwnd
   +--------→ RTT
```

**各阶段说明：**
1. **慢启动阶段（RTT 1-5）：** TCP连接刚建立时，网络状况未知，从较小的cwnd开始，每收到一个ACK，cwnd增加1MSS（效果上每RTT翻倍），目的是快速探测网络可用带宽。
2. **拥塞避免阶段（RTT 6-9）：** 当cwnd达到ssthresh后，转入拥塞避免阶段，每RTT仅增加1MSS（线性增长），目的是温和地探测网络容量上限。
3. **超时后：** 网络发生拥塞（超时意味着丢包严重），将ssthresh减半，cwnd重置为1，重新开始慢启动。
