## 第41题 (10分) 数据结构

已知一棵二叉树的先序遍历序列为：A B D E G C F H，中序遍历序列：D B G E A F H C。

(1) 画出该二叉树的形态。（4分）

(2) 写出该二叉树的后序遍历序列。（3分）

(3) 将该二叉树转换为对应的森林，画出森林的形态。（3分）

**解析：**

**(1) 根据先序和中序序列构造二叉树：**

先序：A B D E G C F H，根为A。
中序：D B G E A F H C，A左侧DBGE为左子树，A右侧FHC为右子树。

左子树：先序为B D E G，中序为D B G E。
- 根为B，中序B左侧D为B的左子树，B右侧GE为B的右子树。
- B右子树先序为E G，中序为G E，根为E，E左子树为G。

右子树：先序为C F H，中序为F H C。
- 根为C，中序C左侧FH为C的左子树。
- C左子树先序为F H，中序为F H，根为F，F右子树为H。

```
        A
       / \
      B   C
     /   /
    D   F
     \   \
      E   H
     /
    G
```

**(2) 后序遍历序列：**

按照后序遍历（左-右-根）：
D G E B H F C A

**(3) 二叉树转换为森林：**

将二叉树的右链断开，每个结点的左孩子转换为森林中的孩子，右孩子转换为兄弟。
森林由2棵树组成：
- 第1棵：A为根，B为A的孩子，D为B的左孩子，E为B的右孩子（在森林中E是D的兄弟），G是E的孩子
- 第2棵：C为根，F为C的孩子，H为F的右孩子（在森林中H是F的兄弟）

```
森林：
  树1:      A         树2:    C
           /|                |
          B                  F
         /|\                 |
        D E                  H
           |
           G
```

## 第42题 (10分) 数据结构

编写一个算法，判断给定的无向图G是否是一棵树。假设图G采用邻接表存储结构。

**解析：**

一棵具有n个顶点的无向图是一棵树的充分必要条件：有n-1条边且是连通图（或等价地：无回路且连通）。

**算法思路：** 采用DFS遍历，从任意顶点出发，统计访问的顶点数和经过的边数。如果访问了所有n个顶点且经过n-1条边，则G是一棵树。

```c
// 用全局变量记录访问过的顶点数和边数
int visited[MAX_VERTEX_NUM];
int vCount, eCount;

void DFS(Graph G, int v) {
    visited[v] = 1;
    vCount++;
    ArcNode *p = G.vertices[v].firstarc;
    while (p != NULL) {
        eCount++;
        if (!visited[p->adjvex]) {
            DFS(G, p->adjvex);
        }
        p = p->nextarc;
    }
}

int IsTree(Graph G) {
    int i, n = G.vexnum;
    for (i = 0; i < n; i++)
        visited[i] = 0;
    vCount = 0;
    eCount = 0;
    DFS(G, 0);  // 从第0个顶点开始DFS

    // 无向图每条边在邻接表中出现两次
    if (vCount == n && eCount == 2 * (n - 1))
        return 1;  // 是一棵树
    else
        return 0;  // 不是一棵树
}
```

**算法分析：** 时间复杂度O(n+e)，空间复杂度O(n)。其中n为顶点数，e为边数。

## 第43题 (10分) 计算机组成原理

某计算机的Cache-主存存储系统中，主存容量为1MB，Cache容量为16KB，主存与Cache之间采用全相联映射方式，主存块和Cache块大小均为512B。

(1) 写出主存地址各字段的划分方式。（4分）

(2) 已知Cache初始为空，依次访问以下主存地址（十六进制）：00000, 00800, 01000, 01800, 02000, 00000。若采用LRU替换策略，写出每次访问的命中情况和Cache的最终状态。（6分）

**解析：**

**(1) 地址划分：**

主存容量1MB=2²⁰B，Cache容量16KB=2¹⁴B，块大小512B=2⁹B。
- 块内地址（偏移量）：log₂512 = 9位
- Cache共有 16KB/512B = 32 块
- 主存共有 1MB/512B = 2048 块
- 主存地址 = 标记位 + 块内偏移
- 标记位 = 20 - 9 = 11位

全相联映射地址划分：
| 标记（11位） | 块内偏移（9位） |

**(2) 访问过程分析：**

将十六进制地址转换为主存块号（地址/512）：
- 00000H = 0，块号 = 0/512 = 0
- 00800H = 2048，块号 = 2048/512 = 4
- 01000H = 4096，块号 = 4096/512 = 8
- 01800H = 6144，块号 = 6144/512 = 12
- 02000H = 8192，块号 = 8192/512 = 16
- 00000H = 0，块号 = 0/512 = 0

LRU替换过程（32路全相联，有32个Cache行）：
| 访问 | 块号 | 命中/缺失 | Cache状态（最近使用的在右） |
|------|------|----------|---------------------------|
| 1 | 0 | 缺失 | [0] |
| 2 | 4 | 缺失 | [0,4] |
| 3 | 8 | 缺失 | [0,4,8] |
| 4 | 12 | 缺失 | [0,4,8,12] |
| 5 | 16 | 缺失 | [0,4,8,12,16] |
| 6 | 0 | 命中 | [4,8,12,16,0] |

Cache最终状态：存放了主存块0,4,8,12,16，共5个块。

## 第44题 (10分) 操作系统

某系统有4类资源R1、R2、R3、R4，当前有5个进程P0~P4。系统当前状态如下：

| | Allocation | | | | Max | | | | Available | | |
|---|---|---|---|---|---|---|---|---|---|---|---|
| | R1 | R2 | R3 | R4 | R1 | R2 | R3 | R4 | R1 | R2 | R3 | R4 |
| P0 | 0 | 0 | 1 | 2 | 0 | 0 | 1 | 2 | 1 | 5 | 2 | 0 |
| P1 | 1 | 0 | 0 | 0 | 1 | 7 | 5 | 0 | | | | |
| P2 | 1 | 3 | 5 | 4 | 2 | 3 | 5 | 6 | | | | |
| P3 | 0 | 6 | 3 | 2 | 0 | 6 | 5 | 2 | | | | |
| P4 | 0 | 0 | 1 | 4 | 0 | 6 | 5 | 6 | | | | |

(1) 计算各进程的Need矩阵。（4分）

(2) 使用银行家算法判断当前状态是否安全。若安全，给出安全序列。（6分）

**解析：**

**(1) Need矩阵 = Max - Allocation：**

| | R1 | R2 | R3 | R4 |
|---|---|---|---|---|
| P0 | 0 | 0 | 0 | 0 |
| P1 | 0 | 7 | 5 | 0 |
| P2 | 1 | 0 | 0 | 2 |
| P3 | 0 | 0 | 2 | 0 |
| P4 | 0 | 6 | 4 | 2 |

**(2) 安全性检查：**

Available = (1, 5, 2, 0)

第1轮：
- P0的Need=(0,0,0,0) ≤ Available=(1,5,2,0)，满足。分配后P0释放资源，Available = (1,5,2,0) + (0,0,1,2) = (1,5,3,2)。

第2轮：
- P2的Need=(1,0,0,2) ≤ (1,5,3,2)，满足。Available = (1,5,3,2) + (1,3,5,4) = (2,8,8,6)。

第3轮：
- P1的Need=(0,7,5,0) ≤ (2,8,8,6)，满足。Available = (2,8,8,6) + (1,0,0,0) = (3,8,8,6)。

第4轮：
- P3的Need=(0,0,2,0) ≤ (3,8,8,6)，满足。Available = (3,8,8,6) + (0,6,3,2) = (3,14,11,8)。

第5轮：
- P4的Need=(0,6,4,2) ≤ (3,14,11,8)，满足。Available = (3,14,11,8) + (0,0,1,4) = (3,14,12,12)。

**安全，安全序列：P0, P2, P1, P3, P4**（不唯一，P0, P2, P3, P1, P4等也满足）。

## 第45题 (10分) 操作系统

某系统采用请求分页存储管理方式，页面大小为4KB，逻辑地址为32位。进程的页表部分数据如下（虚页号→物理块号）：

| 虚页号 | 物理块号 | 存在位 |
|--------|----------|--------|
| 0 | 5 | 1 |
| 1 | 10 | 1 |
| 2 | - | 0 |
| 3 | 8 | 1 |
| 4 | - | 0 |

(1) 给出逻辑地址0x00001234和0x00004567的物理地址。（4分）

(2) 逻辑地址0x00004567会产生缺页中断吗？说明原因。（3分）

(3) 若系统采用TLB（快表），试分析引入TLB对地址转换性能的影响。（3分）

**解析：**

**(1) 物理地址计算：**

页面大小4KB=4096B=2¹²B，页内偏移占低12位。

逻辑地址0x00001234：
- 页号 = 0x00001234 >> 12 = 0x00001 = 1
- 页内偏移 = 0x00001234 & 0xFFF = 0x234
- 查页表，虚页号1对应物理块号10
- 物理地址 = 10 × 4096 + 0x234 = 0x0000A000 + 0x234 = 0x0000A234

逻辑地址0x00004567：
- 页号 = 0x00004567 >> 12 = 0x00004 = 4
- 页内偏移 = 0x00004567 & 0xFFF = 0x567
- 查页表，虚页号4的存在位为0（不在内存中）

**(2) 缺页中断分析：**

逻辑地址0x00004567的页号为4，查页表发现虚页号4的存在位（有效位）为0，说明该页不在物理内存中。因此，访问该地址会产生缺页中断（Page Fault），操作系统需要从外存中将该页调入内存。

**(3) TLB对地址转换性能的影响：**

引入TLB后，地址转换过程：
- 先在TLB中查找页号（TLB是页表的高速缓存，存储最近使用的页表项）
- TLB命中：只需1次访存（访问目标物理地址）
- TLB未命中：需要先访问内存中的页表（1次访存），再访问目标物理地址（1次访存），共2次访存

假设TLB命中率为p，则平均访存次数 = p×1 + (1-p)×2 = 2-p。当p接近1时，平均访存次数接近1次，大大提高了地址转换的效率。TLB利用了程序访问的局部性原理，通常命中率很高（>99%），显著减少了页表查询的开销。

## 第47题 (10分) 计算机网络

主机A向主机B发送一个TCP连接请求，已知主机A选择的初始序号为1000，主机B选择的初始序号为3000。

(1) 画出TCP三次握手过程中各报文段的序号和确认号。（4分）

(2) 若第三个握手报文段丢失，TCP连接能否建立成功？说明原因。（3分）

(3) 在建立连接后，主机A向主机B发送一个长度为2000字节的数据段，主机B收到后会如何回复？（3分）

**解析：**

**(1) 三次握手过程：**

| 阶段 | 方向 | 标志位 | 序号(Seq) | 确认号(Ack) |
|------|------|--------|-----------|-------------|
| 第1次握手 | A→B | SYN=1 | 1000 | - |
| 第2次握手 | B→A | SYN=1, ACK=1 | 3000 | 1001 |
| 第3次握手 | A→B | ACK=1 | 1001 | 3001 |

详细说明：
- 第1次握手：A发送SYN报文段，Seq=1000（A的初始序号），无确认号。
- 第2次握手：B发送SYN+ACK报文段，Seq=3000（B的初始序号），Ack=1001（期望A的下一个序号=1000+1）。
- 第3次握手：A发送ACK报文段，Seq=1001，Ack=3001（期望B的下一个序号=3000+1）。

**(2) 第三个握手报文段丢失的情况：**

若第3次握手报文段丢失，TCP连接**不能建立成功**（从A的角度看）。

原因：主机B发送SYN+ACK后，进入SYN-RCVD状态，等待A的ACK确认。若第3次握手丢失，B收不到ACK，B会超时重传SYN+ACK报文段。经过一定次数重传后，若仍然得不到确认，B将放弃该连接。与此同时，A认为连接已建立（进入ESTABLISHED状态），若A向B发送数据，B由于没有建立连接会发送RST报文段，导致连接被重置。

因此，从完整过程来看，若第3次握手丢失且B超时放弃，TCP连接无法成功建立。

**(3) 数据传输后的确认回复：**

A发送的2000字节数据段：
- Seq = 1001（第3次握手后的下一个序号）
- 数据长度 = 2000字节

B收到后回复ACK：
- Seq = 3001
- Ack = 1001 + 2000 = 3001
- ACK标志位 = 1

B回复的确认报文段中，确认号为3001，表示B期望接收的下一个字节的序号是3001，即确认了A发送的2000字节数据已全部正确接收。
