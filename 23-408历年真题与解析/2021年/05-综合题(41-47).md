## 第41题 (10分) 数据结构

已知一棵二叉树的中序序列为D B G E A C H F，后序序列为D G E B H F C A。

(1) 请根据中序和后序序列，画出该二叉树。（5分）

(2) 写出该二叉树的先序序列。（2分）

(3) 将该二叉树转换为森林，画出森林的形态。（3分）

**解析：**

**(1) 构造二叉树：**

后序：D G E B H F C A，最后一个元素A为根。
中序：D B G E A C H F，A左侧DBGE为左子树，A右侧CHF为右子树。

左子树：
- 后序：D G E B，根为B
- 中序：D B G E，B左侧D为B的左子树，B右侧GE为B的右子树
- B的右子树：后序G E，中序G E，根为E，E的左子树为G

右子树：
- 后序：H F C，根为C
- 中序：C H F，C左侧为空（不对，CHF中C在最左），C右侧HF为C的右子树
- 中序CHF，根为C（后序最后一个C之前的），C右侧HF
- C的右子树：后序H F，中序H F，根为F，F的左子树为H

```
        A
       / \
      B   C
     / \   \
    D   E   F
       /   /
      G   H
```

**(2) 先序序列：**

先序（根-左-右）：A B D E G C F H

**(3) 二叉树转换为森林：**

沿着每个结点的右孩子链断开，将二叉树还原为森林。
- 第1棵树：A为根，B为A的孩子，D为B的左孩子（在森林中是B的第一个孩子），E为B的右孩子（在森林中是D的兄弟），G是E的孩子
- 第2棵树：C为根，F为C的孩子（在森林中是C的唯一孩子），H为F的左孩子（在森林中是F的第一个孩子）

森林形态：
```
 树1:    A        树2:  C
        /|              |
       B                 F
      /|\                |
     D E                 H
       |
       G
```

## 第42题 (10分) 数据结构

设一棵二叉树采用二叉链表存储，每个结点包含left、right指针和data数据域。设计一个算法，求二叉树中度为1的结点个数。

**解析：**

**算法思路：** 采用递归遍历二叉树，对每个结点判断其度（子结点个数）。度为1的结点是指只有左子树或只有右子树的结点。

```c
int countDegree1(BiTree T) {
    int count = 0;
    if (T == NULL)
        return 0;
    
    // 判断当前结点度是否为1
    if ((T->lchild == NULL && T->rchild != NULL) ||
        (T->lchild != NULL && T->rchild == NULL))
        count = 1;
    
    // 递归统计左右子树中度为1的结点数
    return count + countDegree1(T->lchild) + countDegree1(T->rchild);
}
```

**也可以用层序遍历（非递归）实现：**

```c
int countDegree1_Level(BiTree T) {
    if (T == NULL) return 0;
    
    int count = 0;
    BiTree queue[MAX_SIZE];
    int front = 0, rear = 0;
    queue[rear++] = T;
    
    while (front != rear) {
        BiTree p = queue[front++];
        // 判断度
        int degree = 0;
        if (p->lchild != NULL) { degree++; queue[rear++] = p->lchild; }
        if (p->rchild != NULL) { degree++; queue[rear++] = p->rchild; }
        if (degree == 1) count++;
    }
    return count;
}
```

**时间复杂度：** O(n)，每个结点访问一次。
**空间复杂度：** 递归版本O(h)（h为树高），层序版本O(n)。

## 第43题 (10分) 计算机组成原理

某计算机的存储系统中，主存容量为64MB，Cache容量为64KB，主存块和Cache块大小均为64B。采用2路组相联映射方式。

(1) 写出主存地址的划分方式（标记、组号、块内偏移各占多少位）。（4分）

(2) 已知Cache初始为空，按照如下主存块号序列进行访问：1, 513, 1025, 1, 2049, 513, 1。若采用LRU替换策略，分析每次访问的命中情况。（6分）

**解析：**

**(1) 地址划分：**

- 主存容量64MB = 2²⁶B，主存地址26位
- 块大小64B = 2⁶B，块内偏移6位
- Cache容量64KB = 2¹⁶B，Cache共有64KB/64B = 1024块
- 2路组相联：组数 = 1024/2 = 512组，组号位数 = log₂512 = 9位
- 标记位数 = 26 - 9 - 6 = 11位

地址划分：
| 标记(11位) | 组号(9位) | 块内偏移(6位) |

**(2) 命中情况分析：**

组号 = 主存块号 mod 组数 = 块号 mod 512

- 块1：组号 = 1%512 = 1，标记 = 1/512 = 0，组1中无此标记，**缺失**
- 块513：组号 = 513%512 = 1，标记 = 513/512 = 1，组1中标记0≠1，**缺失**
- 块1025：组号 = 1025%512 = 1，标记 = 1025/512 = 2，组1中有标记0和1，都不等于2，**缺失**（替换LRU，标记0被淘汰）
- 块1：组号 = 1%512 = 1，标记 = 0，组1中标记为1和2，标记0不存在，**缺失**（替换标记1，LRU淘汰最久未用的）
- 块2049：组号 = 2049%512 = 1，标记 = 2049/512 = 4，组1中有标记2和0，标记4不存在，**缺失**（替换标记2）
- 块513：组号 = 513%512 = 1，标记 = 1，组1中有标记0和4，标记1不存在，**缺失**（替换标记4）
- 块1：组号 = 1%512 = 1，标记 = 0，组1中有标记0和1，标记0存在，**命中**

总结：
| 访问 | 块号 | 组号 | 标记 | 组1状态(旧→新) | 结果 |
|------|------|------|------|----------------|------|
| 1 | 1 | 1 | 0 | [0] | 缺失 |
| 2 | 513 | 1 | 1 | [0,1] | 缺失 |
| 3 | 1025 | 1 | 2 | [1,2] | 缺失 |
| 4 | 1 | 1 | 0 | [2,0] | 缺失 |
| 5 | 2049 | 1 | 4 | [0,4] | 缺失 |
| 6 | 513 | 1 | 1 | [4,1] | 缺失 |
| 7 | 1 | 1 | 0 | 无标记0 | 缺失 |

等等，第7次访问时组1中有标记4和1，标记0不在其中，应该是缺失。重新检查...

第6次访问后组1状态：LRU替换标记4，存入标记1，组1为[1]?不对，2路组相联每组2行。

让我重新仔细分析（每组最多2个块，LRU替换）：

| 访问 | 块号 | 组号 | 标记 | 该组内容(标记) | 结果 |
|------|------|------|------|---------------|------|
| 1 | 1 | 1 | 0 | {0} | 缺失 |
| 2 | 513 | 1 | 1 | {0,1} | 缺失 |
| 3 | 1025 | 1 | 2 | {1,2}(替换0) | 缺失 |
| 4 | 1 | 1 | 0 | {2,0}(替换1) | 缺失 |
| 5 | 2049 | 1 | 4 | {0,4}(替换2) | 缺失 |
| 6 | 513 | 1 | 1 | {4,1}(替换0) | 缺失 |
| 7 | 1 | 1 | 0 | {1,0}(替换4) | 缺失 |

全部缺失！因为7次访问都是组1，且用了6个不同标记（0,1,2,0,4,1,0），2路组相联每组只能放2个块，LRU替换下全部不命中。

更正后：
命中次数 = 1（第7次？不，标记0在第5次已被替换，第6次替换后组1是{4,1}，第7次标记0不在其中）。

所以**7次访问全部缺失，命中次数为0**。

## 第44题 (10分) 操作系统

某系统采用请求分页存储管理方式，页面大小为4KB，逻辑地址为32位。某进程的页表如下：

| 页号 | 物理块号 | 存在位 |
|------|----------|--------|
| 0 | 3 | 1 |
| 1 | - | 0 |
| 2 | 8 | 1 |
| 3 | - | 0 |
| 4 | 5 | 1 |

(1) 给出逻辑地址0x00001A3C和0x00003F40的物理地址（若缺页则说明）。（4分）

(2) 画出逻辑地址的结构，并说明各级地址变换机构的作用。（3分）

(3) 若引入TLB（快表），试计算TLB命中率为99%时的平均有效访存时间。已知访问TLB需要5ns，访问内存需要100ns。（3分）

**解析：**

**(1) 物理地址计算：**

页面大小4KB=2¹²B，页内偏移占低12位。

逻辑地址0x00001A3C：
- 页号 = 0x00001A3C >> 12 = 0x0001 = 1
- 页内偏移 = 0x00001A3C & 0xFFF = 0xA3C
- 查页表：页号1，存在位=0，**产生缺页中断**

逻辑地址0x00003F40：
- 页号 = 0x00003F40 >> 12 = 0x0003 = 3
- 页内偏移 = 0x00003F40 & 0xFFF = 0xF40
- 查页表：页号3，存在位=0，**产生缺页中断**

（注：题目给出的存在位为0的页1和页3都会缺页。给出一个存在位为1的地址示例：逻辑地址0x00000ABC，页号0，块号3，物理地址=3×4096+0xABC=0x3ABC。）

**(2) 逻辑地址结构：**

32位逻辑地址结构：
| 页号（20位） | 页内偏移（12位） |

地址变换过程：
1. 用页号作为索引查找页表
2. 检查存在位：若为1，取出物理块号；若为0，产生缺页中断
3. 物理地址 = 物理块号 × 页大小 + 页内偏移

**(3) 平均有效访存时间：**

- TLB命中（99%）：访问TLB(5ns) + 访问目标内存(100ns) = 105ns
- TLB未命中（1%）：访问TLB(5ns) + 访问页表(100ns) + 访问目标内存(100ns) = 205ns

平均访存时间 = 0.99 × 105 + 0.01 × 205 = 103.95 + 2.05 = 106 ns

## 第45题 (10分) 操作系统

设有4个进程P1, P2, P3, P4，其到达时间和运行时间如下表所示。采用时间片轮转调度算法，时间片q=2。

| 进程 | 到达时间 | 运行时间 |
|------|----------|----------|
| P1 | 0 | 6 |
| P2 | 1 | 3 |
| P3 | 2 | 4 |
| P4 | 3 | 2 |

(1) 画出各进程的调度执行顺序（甘特图）。（4分）

(2) 计算各进程的完成时间、周转时间、带权周转时间。（6分）

**解析：**

**(1) 调度执行顺序（时间片q=2）：**

时间轴：
- 时刻0：P1到达，P1运行（0~2），剩余4
- 时刻1：P2到达
- 时刻2：P3到达。就绪队列：P1(剩4), P2(剩3), P3(剩4)。P2运行（2~4），剩余1
- 时刻3：P4到达
- 时刻4：就绪队列：P1(剩4), P3(剩4), P4(剩2), P2(剩1)。P3运行（4~6），剩余2
- 时刻6：就绪队列：P1(剩4), P4(剩2), P2(剩1), P3(剩2)。P4运行（6~8），完成
- 时刻8：就绪队列：P1(剩4), P2(剩1), P3(剩2)。P2运行（8~9），完成（只需1个单位）
- 时刻9：就绪队列：P1(剩4), P3(剩2)。P1运行（9~11），剩余2
- 时刻11：就绪队列：P3(剩2), P1(剩2)。P3运行（11~13），完成
- 时刻13：就绪队列：P1(剩2)。P1运行（13~15），完成

甘特图：
```
P1|P2 |P3 |P4 |P2|P1 |P3 |P1 |
0 2  4  6  8 9 11 13 15
```

调度顺序：P1→P2→P3→P4→P2→P1→P3→P1

**(2) 各进程的时间指标：**

| 进程 | 到达时间 | 运行时间 | 完成时间 | 周转时间 | 带权周转时间 |
|------|----------|----------|----------|----------|--------------|
| P1 | 0 | 6 | 15 | 15 | 15/6=2.50 |
| P2 | 1 | 3 | 9 | 8 | 8/3=2.67 |
| P3 | 2 | 4 | 13 | 11 | 11/4=2.75 |
| P4 | 3 | 2 | 8 | 5 | 5/2=2.50 |

- 周转时间 = 完成时间 - 到达时间
- 带权周转时间 = 周转时间 / 运行时间
- 平均周转时间 = (15+8+11+5)/4 = 9.75
- 平均带权周转时间 = (2.50+2.67+2.75+2.50)/4 = 2.605

## 第46题 (10分) 计算机网络

某公司获得一个C类网络地址202.100.10.0/24，该公司有3个部门，各部门需要的主机数分别为：A部门100台，B部门50台，C部门25台。

(1) 请为这3个部门分配子网，写出每个子网的网络地址、子网掩码、可用IP地址范围和广播地址。（6分）

(2) 试分析这种分配方案是否存在问题，如何改进？（4分）

**解析：**

**(1) 子网分配：**

/24网络有256个地址（0~255）。

A部门100台主机：需要2^h - 2 >= 100，h=7（2^7-2=126 >= 100）。需7位主机位，子网掩码/25。
B部门50台主机：需要2^h - 2 >= 50，h=6（2^6-2=62 >= 50）。需6位主机位，子网掩码/26。
C部门25台主机：需要2^h - 2 >= 25，h=5（2^5-2=30 >= 25）。需5位主机位，子网掩码/27。

分配方案：

**A部门（/25，128个地址）：**
- 网络地址：202.100.10.0/25
- 子网掩码：255.255.255.128
- 可用IP范围：202.100.10.1 ~ 202.100.10.126
- 广播地址：202.100.10.127
- 可用主机数：126

**B部门（/26，64个地址）：**
- 网络地址：202.100.10.128/26
- 子网掩码：255.255.255.192
- 可用IP范围：202.100.10.129 ~ 202.100.10.190
- 广播地址：202.100.10.191
- 可用主机数：62

**C部门（/27，32个地址）：**
- 网络地址：202.100.10.192/27
- 子网掩码：255.255.255.224
- 可用IP范围：202.100.10.193 ~ 202.100.10.222
- 广播地址：202.100.10.223
- 可用主机数：30

地址空间使用：0~223已分配，224~255（32个地址）剩余。

**(2) 分析与改进：**

存在问题：
1. IP地址利用率不高：A部门需要100台但分配了126个地址（利用率79.4%），B部门需要50台分配62个（80.6%），C部门需要25台分配30个（83.3%）。
2. 各子网的广播域大小不同，A部门的广播域最大，可能产生较多广播流量。
3. 如果部门规模发生变化，这种固定大小的分配不够灵活。

改进建议：
1. 若公司网络规模可预期增长，可预留部分地址空间（如224~255）作为扩展。
2. 考虑使用VLSM（可变长子网掩码）技术，这已经在方案中使用了。
3. 如果使用私有地址（如192.168.x.x），可以更灵活地规划地址空间。
4. 可考虑使用NAT技术，内部使用私有地址，对外使用少量公有地址。

## 第47题 (10分) 计算机网络

主机A向主机B发送一个长度为2048字节的TCP报文段，A的初始序号为100。在传输过程中，第一个字节到第512字节丢失，B只收到了513~2048字节。

(1) B收到该报文段后会如何回复A？（3分）

(2) A收到B的回复后应如何处理？（4分）

(3) 如果A的初始拥塞窗口为1个MSS（最大报文段长度为512字节），慢启动门限ssthresh=8。描述A的慢启动和拥塞避免过程（假设不发生丢包和超时）。（3分）

**解析：**

**(1) B的回复：**

B收到513~2048字节（序号513~2048），但缺少1~512字节（序号100~611，按A的初始序号）。B会发送一个TCP确认报文段：
- ACK标志位置1
- 确认号 = 100（期望收到的下一个字节序号，即缺失的第一个字节）
- 这表明B期望A从序号100开始重传

B的确认采用的是TCP的累积确认机制——只确认连续收到的字节流中的最后一个字节的下一个序号。

**(2) A的处理：**

A收到确认号为100的ACK报文段后，得知序号100~611（共512字节）的报文段丢失。A需要：
1. 重传丢失的512字节（序号100~611）
2. 由于发生了丢包，A的拥塞控制机制会调整：
   - ssthresh = cwnd/2（如果cwnd >= 2，否则ssthresh=1）
   - cwnd = 1（慢启动，从头开始）或者cwnd = ssthresh（如果是快速恢复机制）
3. 等待计时器超时后重传（或者在收到3个重复ACK后快速重传）

**(3) 慢启动和拥塞避免过程：**

初始：cwnd = 1 MSS = 512字节，ssthresh = 8 MSS

慢启动阶段（cwnd < ssthresh）：
- 第1轮：cwnd = 1，发送1个MSS（512字节），收到1个ACK
- 第2轮：cwnd = 2，发送2个MSS，收到2个ACK
- 第3轮：cwnd = 4，发送4个MSS，收到4个ACK
- 第4轮：cwnd = 8，等于ssthresh，**进入拥塞避免阶段**

拥塞避免阶段（cwnd >= ssthresh）：
- 每收到1个ACK，cwnd增加1/cwnd个MSS（即每个RTT，cwnd增加1个MSS）
- 第5轮：cwnd = 8 + 1 = 9
- 第6轮：cwnd = 9 + 1 = 10
- 以此类推，每轮增加1个MSS

发送窗口以报文段为单位的递增过程：
1→2→4→8（慢启动，每次翻倍）→9→10→11→12→...（拥塞避免，每次+1）
