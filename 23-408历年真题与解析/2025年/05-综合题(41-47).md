## 第41题 (10分) ——数据结构
已知一棵二叉树采用二叉链表存储。请完成以下任务：

(1) 编写递归算法，求二叉树的高度。（3分）

(2) 编写非递归算法，利用层次遍历求二叉树的最大宽度（即结点最多的一层的结点个数）。（4分）

(3) 分析上述算法的时间复杂度和空间复杂度。（3分）

**解析：**

**(1) 递归求高度**

```c
int TreeHeight(BiTree T) {
    if (T == NULL) return 0;
    int leftH = TreeHeight(T->lchild);
    int rightH = TreeHeight(T->rchild);
    return (leftH > rightH ? leftH : rightH) + 1;
}
```

**(2) 非递归求最大宽度**

利用队列进行层次遍历，记录每层的结点数。

```c
int MaxWidth(BiTree T) {
    if (T == NULL) return 0;
    BiTree queue[MAXSIZE];
    int front = 0, rear = 0;
    int maxWidth = 1;

    queue[rear++] = T;

    while (front < rear) {
        int levelSize = rear - front; // 当前层的结点数
        if (levelSize > maxWidth)
            maxWidth = levelSize;

        // 处理当前层的所有结点
        for (int i = 0; i < levelSize; i++) {
            BiTree p = queue[front++];
            if (p->lchild) queue[rear++] = p->lchild;
            if (p->rchild) queue[rear++] = p->rchild;
        }
    }
    return maxWidth;
}
```

**(3) 复杂度分析**

- 算法(1)：时间复杂度O(n)（每个结点访问一次），空间复杂度O(h)（递归栈深度，h为树高，最坏O(n)）。
- 算法(2)：时间复杂度O(n)（每个结点入队出队各一次），空间复杂度O(w)（队列最大长度w为树的最大宽度，最坏O(n/2)即O(n)）。

## 第42题 (10分) ——数据结构
设有一个长度为n的整数序列，设计一个时间复杂度为O(n)的算法，找出序列中出现次数超过n/2的元素（如果存在的话），称为"主元素"。

(1) 描述算法思路并给出伪代码。（5分）

(2) 以序列{3, 3, 4, 2, 3, 3, 1, 3}为例，模拟算法的执行过程。（3分）

(3) 说明算法的正确性并分析时间复杂度和空间复杂度。（2分）

**解析：**

**(1) 算法设计（摩尔投票法）**

```c
int FindMajority(int a[], int n) {
    int candidate = a[0];
    int count = 1;

    // 第一轮：寻找候选元素
    for (int i = 1; i < n; i++) {
        if (count == 0) {
            candidate = a[i];
            count = 1;
        } else if (a[i] == candidate) {
            count++;
        } else {
            count--;
        }
    }

    // 第二轮：验证候选元素是否真的超过n/2
    count = 0;
    for (int i = 0; i < n; i++) {
        if (a[i] == candidate) count++;
    }
    if (count > n/2)
        return candidate;
    else
        return -1; // 不存在主元素
}
```

**(2) 模拟过程**

序列：{3, 3, 4, 2, 3, 3, 1, 3}，n=8

| 步骤 | 当前元素 | candidate | count |
|------|---------|-----------|-------|
| 初始 | 3 | 3 | 1 |
| 1 | 3 | 3 | 2 |
| 2 | 4 | 3 | 1 |
| 3 | 2 | 3 | 0 |
| 4 | 3 | 3 | 1 |
| 5 | 3 | 3 | 2 |
| 6 | 1 | 3 | 1 |
| 7 | 3 | 3 | 2 |

候选元素为3。验证：3出现5次 > 8/2=4，确认3是主元素。

**(3) 正确性和复杂度**

正确性：如果存在主元素（出现次数>n/2），则经过两两抵消后，主元素一定不会被完全抵消掉，会留下作为候选元素。第二轮验证确保候选元素确实是主元素。

时间复杂度：**O(n)**（两轮遍历）。空间复杂度：**O(1)**（只用了常数个变量）。

## 第43题 (10分) ——计算机组成原理
某计算机的Cache采用2路组相联映射，Cache容量为8KB，主存容量为4MB，块大小为64B。Cache采用LRU替换策略，使用1位LRU位。请回答以下问题：

(1) 给出主存地址的划分格式（标记、组号、块内地址各多少位）。（3分）

(2) 若Cache初始为空，CPU依次访问以下主存地址（十六进制）：0x00000, 0x40000, 0x00040, 0x40040, 0x00000, 0x40000。给出每次访问的命中情况。（4分）

(3) 分析LRU位在此过程中的变化。（3分）

**解析：**

**(1) 地址划分**

- 主存容量4MB = 2^22 B，地址22位。
- Cache容量8KB = 2^13 B，块大小64B = 2^6 B。
- Cache总块数 = 2^13 / 2^6 = 2^7 = 128块。
- 2路组相联，组数 = 128 / 2 = 64 = 2^6组。

| 标记(Tag) | 组号 | 块内地址 |
|-----------|------|---------|
| 10位      | 6位   | 6位      |

22 - 6 - 6 = 10位标记。

**(2) 访问分析**

- **0x00000**：组号=0x000000>>6 & 0x3F = 0，标记=0x000000>>12 = 0x000。Cache空，未命中。装入组0第0路，标记=0x000，LRU[0]=1（表示最近用第0路）。
- **0x40000**：0x40000=4194304... 让我计算。0x40000的二进制=0100 0000 0000 0000 0000。组号=(0x40000>>6)&0x3F = 0x1000&0x3F = 0x00。标记=0x40000>>12 = 0x400。组0已有标记0x000≠0x400，未命中。装入组0第1路，LRU[0]=0（最近用第1路）。
- **0x00040**：0x00040=64。组号=(0x40>>6)&0x3F = 1。标记=0x00040>>12 = 0x000。组1空，未命中。装入组1第0路，LRU[1]=1。
- **0x40040**：组号=(0x40040>>6)&0x3F = 1。标记=0x40040>>12 = 0x400。组1已有标记0x000≠0x400，未命中。装入组1第1路，LRU[1]=0。
- **0x00000**：组号=0，标记=0x000。组0中有标记0x000在第0路。**命中！** LRU[0]更新为1（最近用第0路）。
- **0x40000**：组号=0，标记=0x400。组0中有标记0x400在第1路。**命中！** LRU[0]更新为0（最近用第1路）。

结果：6次访问中，前4次未命中，后2次命中。

**(3) LRU位变化**

每组1位LRU位，记录最近使用的是哪一路（0或1）。

| 访问 | 组0的LRU | 组1的LRU | 说明 |
|------|---------|---------|------|
| 0x00000 | 1 | - | 组0装入第0路，LRU=1 |
| 0x40000 | 0 | - | 组0装入第1路，LRU=0 |
| 0x00040 | 0 | 1 | 组1装入第0路，LRU=1 |
| 0x40040 | 0 | 0 | 组1装入第1路，LRU=0 |
| 0x00000 | 1 | 0 | 组0命中第0路，更新LRU=1 |
| 0x40000 | 0 | 0 | 组0命中第1路，更新LRU=0 |

需要替换时，根据LRU位选择替换另一路。

## 第44题 (10分) ——操作系统
某系统有3种资源R1、R2、R3，总数分别为10、8、7。有5个进程P0~P4，当前状态如下：

| 进程 | 已分配(R1,R2,R3) | 最大需求(R1,R2,R3) |
|------|-----------------|-------------------|
| P0   | (0,1,0)         | (7,5,3)           |
| P1   | (2,0,0)         | (3,2,2)           |
| P2   | (3,0,2)         | (9,0,2)           |
| P3   | (2,1,1)         | (2,2,2)           |
| P4   | (0,0,2)         | (4,3,3)           |

(1) 计算各进程的Need矩阵和当前的Available向量。（3分）

(2) 判断系统是否处于安全状态，若是则给出一个安全序列。（4分）

(3) 若进程P1提出请求Request(1,0,2)，系统是否应分配？为什么？（3分）

**解析：**

**(1) Need矩阵和Available**

Need = Max - Allocation：
| 进程 | R1 | R2 | R3 |
|------|----|----|-----|
| P0   | 7  | 4  | 3   |
| P1   | 1  | 2  | 2   |
| P2   | 6  | 0  | 0   |
| P3   | 0  | 1  | 1   |
| P4   | 4  | 3  | 1   |

已分配总和：R1=0+2+3+2+0=7, R2=1+0+0+1+0=2, R3=0+0+2+1+2=5
Available = (10-7, 8-2, 7-5) = **(3, 3, 2)**

**(2) 安全性分析**

Work = (3, 3, 2)

1. P1: Need=(1,2,2) <= (3,3,2)，满足。Work=(3,3,2)+(2,0,0)=(5,3,2)
2. P3: Need=(0,1,1) <= (5,3,2)，满足。Work=(5,3,2)+(2,1,1)=(7,4,3)
3. P4: Need=(4,3,1) <= (7,4,3)，满足。Work=(7,4,3)+(0,0,2)=(7,4,5)
4. P0: Need=(7,4,3) <= (7,4,5)，满足。Work=(7,4,5)+(0,1,0)=(7,5,5)
5. P2: Need=(6,0,0) <= (7,5,5)，满足。Work=(7,5,5)+(3,0,2)=(10,8,7)

安全序列：**P1 → P3 → P4 → P0 → P2**

**(3) P1请求(1,0,2)**

试探分配：
- Available = (3,3,2)-(1,0,2) = (2,3,0)
- Alloc_P1 = (2,0,0)+(1,0,2) = (3,0,2)
- Need_P1 = (1,2,2)-(1,0,2) = (0,2,0)

安全性检测：Work=(2,3,0)
1. P1: Need=(0,2,0) <= (2,3,0)，满足。Work=(5,3,0)
2. P3: Need=(0,1,1) <= (5,3,0)，R3=0<1，不满足。
3. P4: Need=(4,3,1) <= (5,3,0)，R3不满足。
4. P0: Need=(7,4,3)，R1=5<7，不满足。
5. P2: Need=(6,0,0)，R1=5<6，不满足。

没有进程可以继续执行，系统**不安全**。因此**不应分配**。

## 第45题 (10分) ——操作系统
在某个文件系统中，磁盘块大小为1KB，每个磁盘地址占4B。inode中包含12个直接块指针、1个一级间接块指针、1个二级间接块指针。

(1) 计算该文件系统支持的最大文件大小。（4分）

(2) 若要读取文件中偏移量为50000字节处的数据，最少和最多各需要多少次磁盘I/O？（3分）

(3) 若磁盘总容量为8GB，使用位示图管理空闲空间，位示图需要占用多少个磁盘块？（3分）

**解析：**

**(1) 最大文件大小**

- 每个磁盘块存放地址数 = 1KB / 4B = 256个
- 直接索引：12 × 1KB = 12KB
- 一级间接：1 × 256 × 1KB = 256KB
- 二级间接：1 × 256 × 256 × 1KB = 65536KB = 64MB

最大文件 = 12KB + 256KB + 64MB = **64MB + 268KB ≈ 64.26MB**

**(2) 读取偏移量50000B处的数据**

逻辑块号 = 50000 / 1024 = 48（块内偏移832字节）。

块号48 > 12（直接块范围0~11），属于一级间接块范围（12~267）。

在一级间接块中的偏移 = 48 - 12 = 36。

**最少I/O次数**：假设inode和间接块都在内存缓存中，则只需读数据块 = **1次**。

**最多I/O次数**：需要依次读inode（1次）+ 一级间接块（1次）+ 数据块（1次）= **3次**。

**(3) 位示图大小**

磁盘容量8GB = 2^33 B，块大小1KB = 2^10 B。
总块数 = 2^33 / 2^10 = 2^23 = 8388608块。
位示图大小 = 2^23 / 8 = 2^20 B = 1MB = 1024B / 1KB × ... = 1MB / 1KB = **1024个磁盘块**。

## 第47题 (10分) ——计算机网络
主机A和主机B通过一条TCP连接通信。已知A的初始序号为200，B的初始序号为500。请回答以下问题：

(1) 若A向B发送一个携带400B数据的TCP报文段，写出该报文段的序号、确认号和数据长度。B收到后确认，写出确认报文段的序号和确认号。（4分）

(2) 若B同时回复一个携带200B数据的TCP确认报文段（捎带确认），写出该报文段的序号、确认号和数据长度。A收到后，下一个待发送数据的序号是多少？（3分）

(3) 若A发送的400B数据报文段丢失，超时后A重传该报文段。B收到重传报文段后会如何处理？若此时B已经收到了A后来发送的后续数据100B（序号600~699），描述TCP的处理方式。（3分）

**解析：**

**(1) 数据传输**

A→B数据报文段：
- 序号(Seq) = 200
- 确认号(Ack) = 500（期望从B收到的下一个字节序号）
- 数据长度 = 400B

B→A确认报文段：
- 序号(Seq) = 500
- 确认号(Ack) = 200 + 400 = 600（期望从A收到的下一个字节序号）

**(2) 捎带确认**

B→A捎带确认报文段：
- 序号(Seq) = 500
- 确认号(Ack) = 600
- 数据长度 = 200B

A收到后，下一个待发送数据的序号 = 200 + 400 = **600**。

**(3) 重传处理**

A超时重传序号200~599的数据报文段。

若B已经收到了后续数据（600~699），但200~599丢失：
- B收到重传的200~599数据后，与之前收到的600~699合并。
- B回复确认号 = 200+400+100 = **700**（确认所有699字节的数据都已收到）。
- 这就是TCP的**累积确认**机制：确认号表示该序号之前的所有数据都已正确接收。

若B从未收到600~699的数据：
- B回复确认号 = 600（仍期望从600开始的数据）。
- A需要重传600及之后的数据。
