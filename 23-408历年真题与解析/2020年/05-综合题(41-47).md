## 第41题 (10分) 数据结构

设线性表L=(a₁,a₂,...,aₙ)采用带头结点的单链表存储。设计一个尽可能高效的算法，将L中的所有元素按照基准值x划分为两部分：所有小于x的元素排在前面，所有大于等于x的元素排在后面。要求不改变各元素之间的相对顺序。

**解析：**

**算法思路：** 创建两个新链表，分别存放小于x和大于等于x的结点，最后将两个链表合并。需要遍历原链表一次。

```c
void Partition(LinkList L, ElemType x) {
    // L为带头结点的单链表
    LNode *small, *large, *p, *sTail, *lTail;
    
    // 创建两个链表的头结点
    small = (LNode*)malloc(sizeof(LNode));
    large = (LNode*)malloc(sizeof(LNode));
    small->next = NULL;
    large->next = NULL;
    sTail = small;
    lTail = large;
    
    p = L->next;  // 跳过头结点
    while (p != NULL) {
        LNode *temp = p->next;  // 保存下一个结点
        p->next = NULL;
        
        if (p->data < x) {
            // 尾插法插入small链表
            sTail->next = p;
            sTail = p;
        } else {
            // 尾插法插入large链表
            lTail->next = p;
            lTail = p;
        }
        p = temp;
    }
    
    // 合并：small链表接large链表
    sTail->next = large->next;
    L->next = small->next;
    
    // 释放辅助头结点
    free(small);
    free(large);
}
```

**算法分析：** 只需遍历链表一次，时间复杂度为O(n)。空间复杂度为O(1)（仅使用常数个指针变量）。采用尾插法保证了各部分内部元素的相对顺序不变。

## 第42题 (10分) 数据结构

设有向图G采用邻接矩阵存储。设计一个算法，判断G中是否存在从顶点vᵢ到顶点vⱼ的路径（i≠j）。

**解析：**

**算法思路：** 使用深度优先搜索（DFS）从vᵢ出发，若能访问到vⱼ则存在路径。

```c
int visited[MAX_VERTEX_NUM];

int DFS_Path(MGraph G, int i, int j) {
    int k;
    // 初始化访问标记数组
    for (k = 0; k < G.vexnum; k++)
        visited[k] = 0;
    
    DFS(G, i);
    
    if (visited[j] == 1)
        return 1;  // 存在从vi到vj的路径
    else
        return 0;  // 不存在
}

void DFS(MGraph G, int v) {
    int w;
    visited[v] = 1;
    for (w = 0; w < G.vexnum; w++) {
        if (G.Edge[v][w] == 1 && !visited[w]) {
            DFS(G, w);
        }
    }
}
```

**算法分析：** 时间复杂度O(n²)，其中n为顶点数（邻接矩阵存储的图需要检查n个邻接顶点）。空间复杂度O(n)（访问标记数组和递归栈）。

也可以使用BFS实现，思路相同。对于邻接表存储的图，时间复杂度为O(n+e)。

## 第43题 (10分) 计算机组成原理

某计算机采用32位定长指令字，操作码占6位。指令系统中有零地址、一地址和二地址三种格式。已知二地址指令有15条，零地址指令有36条。若一地址指令最多可有多少条？

**解析：**

指令字长32位，操作码6位。

**二地址指令：**
- 2个地址各占12位（假设采用寄存器-存储器型，具体分配取决于格式）
- 但题目给的操作码占6位固定，不同格式下扩展操作码不同

实际上，本题应采用扩展操作码技术：

二地址指令：操作码6位，剩余32-6=26位用于2个地址。最多2⁶=64种编码，已用了15条，剩余64-15=49个编码可用于扩展。

一地址指令：借用二地址指令的一个地址码字段扩展操作码。每个二地址指令空闲的编码（49个）可以扩展出2¹²=4096种一地址编码。但题目问的是一地址指令最多有多少条。

更准确地说：
- 二地址指令15条，操作码编码15个，剩余64-15=49个操作码编码
- 每个剩余编码可以扩展12位（原来的一个地址字段）作为扩展操作码
- 所以一地址指令最多49×2¹²=49×4096=200704条

但还要为零地址指令预留编码。零地址指令36条，每个一地址指令可以扩展12位产生4096个零地址指令编码。只需⌈36/4096⌉=1个一地址编码扩展就能容纳36条零地址指令。

所以一地址指令最多49-1=48个编码用于扩展，每条一地址指令占一个编码（12位扩展操作码+12位地址），即48×2¹²=48×4096=196608条。

**不对，重新理清思路：**

标准的扩展操作码题目解法：
- 6位操作码，共64个编码
- 二地址指令用了15个编码，剩49个
- 这49个编码用于扩展为一地址指令和零地址指令

设一地址指令用x个编码（每个编码提供12位扩展空间），零地址指令用y个编码。
- 一地址指令数 = x × 2¹²
- 零地址指令数需要36条，需要⌈36/2¹²⌉ = 1个一地址编码的扩展空间
- 所以 x = 49 - 1 = 48（48个编码给一地址指令，1个编码给零地址指令）
- 一地址指令最多48条（每个编码对应一条一地址指令）

不对，实际上每个一地址指令的编码由一个空闲的"二地址操作码"加上12位的扩展操作码组成，但这样理解不对。

**正确的理解：**

二地址指令：操作码6位，地址码各15位（32-6=26，26/2=13，或其他分配）。关键是操作码占6位。

标准解法采用操作码扩展：
- 6位操作码：000000~111111，共64种编码
- 二地址指令用15种编码（000000~01110？），剩余49种
- 剩余的49种编码中，每个可以进一步用地址字段扩展

假定二地址指令格式为：操作码(6位) + 地址1(13位) + 地址2(13位) = 32位
一地址指令格式为：操作码(6+13=19位) + 地址(13位) = 32位
零地址指令格式为：操作码(6+13+13=32位) = 32位

剩余49个二地址操作码，每个可以扩展出2¹³=8192条一地址指令。
其中用于零地址指令的：36条零地址指令需要⌈36/8192⌉=1个编码。
一地址指令可用49-1=48个编码，每码2¹³条，共48×8192=393216条。

**答案：一地址指令最多48×2¹³=393216条。**（若每个地址字段13位）

或者如果地址各12位：48×2¹²=196608条。

## 第44题 (10分) 操作系统

某系统有4个同类资源，由3个进程共享。每个进程的最大资源需求量分别为3、2、4个。

(1) 在最坏情况下（每个进程都已拿到最大需求量减1的资源），系统是否一定死锁？说明理由。（5分）

(2) 若要保证系统不会死锁，该类资源至少需要多少个？（5分）

**解析：**

**(1) 最坏情况下的死锁分析：**

每个进程的最大需求量：P1需要3个，P2需要2个，P3需要4个。
最坏情况：每个进程都已获得"最大需求量-1"个资源。
- P1已获得3-1=2个
- P2已获得2-1=1个
- P3已获得4-1=3个
- 共占用2+1+3=6个资源

但系统只有4个资源，不可能同时满足每个进程都达到"最大需求-1"的状态（因为6>4）。因此需要具体分析。

用最坏情况分析：3个进程各拿1个资源（共3个），剩余1个分配给任意进程。
- P1拿1个，P2拿1个，P3拿1个，剩余1个给P3（P3有2个）
- 此时：P1有1个（还需2个），P2有1个（还需1个），P3有2个（还需2个）
- 系统剩余0个
- P2还需1个但没有可用资源，P1还需2个但没有，P3还需2个但没有
- 死锁！

更系统的方法：使用死锁的必要条件分析。3个进程，4个资源。
最坏分配：P1=1, P2=1, P3=2（共4个），都未达到最大需求，系统无可分配资源→死锁。

**(2) 不死锁所需的最少资源数：**

根据公式：最少资源 = 各进程(最大需求-1)之和 + 1
但需注意各进程最大需求不同。

一般公式：若各进程最大需求为M₁,M₂,...,Mₙ，则：
最少资源 = Σ(Mᵢ-1) + 1 = (3-1) + (2-1) + (4-1) + 1 = 2+1+3+1 = 7个。

验证：6个资源时，分配方案P1=2,P2=1,P3=3（共6个），都差1个，无可分配资源→死锁。
7个资源时，最坏情况分配6个资源给各进程，剩余1个可分配给任一进程使其完成并释放资源→不会死锁。

**答案：至少需要7个同类资源。**

## 第45题 (10分) 操作系统

设有进程P1和P2并发执行，它们的程序如下。变量x为共享变量，初值为0。

P1:
```
x = x + 1;
x = x + 1;
```

P2:
```
x = x + 1;
```

(1) 如果不采取任何同步措施，最后x的可能值有哪些？（4分）

(2) 若用信号量机制实现P1和P2的同步执行（保证P1的两个x=x+1先执行完后再执行P2），写出信号量的定义和P1、P2的程序。（6分）

**解析：**

**(1) 不采取同步措施的可能结果：**

x=x+1的机器指令序列通常为：
- Load R, x（将x的值读入寄存器R）
- Add R, 1（R加1）
- Store x, R（将R的值写回x）

3条x=x+1指令，共9条机器指令。可能的交错执行顺序：

若所有P1的操作先执行完，再执行P2：x=0+1+1+1=3
若发生交错：
- P1的第一条x=x+1在P2的x=x+1之前：x可能为2或3
- 最终x的可能值为1,2,3

具体分析：
- 最好情况：P1全部执行完→x=2，然后P2执行→x=3
- 最坏情况（极端交错）：3个x=x+1的Load都读到x=0，最终x=1
- 中间情况：部分交错导致x=2

**x的可能值为1、2、3。**

**(2) 用信号量实现同步：**

```c
Semaphore S = 0;  // 信号量，初值为0

// P1进程
P1() {
    x = x + 1;
    x = x + 1;
    V(S);  // 通知P2，P1已完成
}

// P2进程
P2() {
    P(S);  // 等待P1完成
    x = x + 1;
}
```

P1执行完两个x=x+1后，执行V(S)将信号量S加1。P2在执行x=x+1之前先执行P(S)，若S=0则阻塞等待。只有当P1执行V(S)后，P2才能继续执行。这样保证了P1的两个赋值语句先于P2的赋值语句执行。

## 第46题 (10分) 计算机网络

主机A通过路由器R与主机B相连。A到R的距离为20km，R到B的距离为40km，信号传播速率为2×10⁵km/s。A向B发送一个长度为10000bit的数据包，数据传输速率为1Mbps。

(1) 计算A到B的传播时延。（4分）

(2) 计算该数据包的发送时延。（3分）

(3) 若路由器R的处理时延为1ms，忽略排队时延，计算A到B的总时延。（3分）

**解析：**

**(1) 传播时延：**

总传播距离 = 20 + 40 = 60 km
传播时延 = 距离 / 传播速率 = 60 / (2×10⁵) = 3 × 10⁻⁴ s = 0.3 ms

**(2) 发送时延：**

发送时延 = 数据长度 / 发送速率 = 10000 bit / (1 × 10⁶ bit/s) = 0.01 s = 10 ms

**(3) 总时延：**

总时延 = 发送时延 + 传播时延 + 路由器处理时延
= 10 ms + 0.3 ms + 1 ms
= 11.3 ms

注意：A的发送时延为10ms，这是在A的网卡上将数据包全部发送到链路上的时间。之后数据在链路上传播（传播时延0.3ms），中间经过路由器R处理（处理时延1ms）。总计11.3ms。

## 第47题 (10分) 计算机网络

已知一个IP数据报的首部（十六进制表示）前20个字节为：45 00 00 54 12 34 40 00 40 06 xx xx ...

(1) 从首部中提取版本、首部长度、总长度、TTL、协议字段的值。（6分）

(2) 判断该IP数据报使用的是IPv4还是IPv6。（2分）

(3) 该数据报封装的传输层协议是什么？（2分）

**解析：**

IP首部格式（每行4字节=32位）：

**(1) 提取各字段值：**

第1字节：45
- 版本 = 4（高4位 = 0100 = 4）
- 首部长度 = 5（低4位 = 0101 = 5），即5×4=20字节

第3-4字节：00 54
- 总长度 = 0x0054 = 84字节

第9字节：40
- TTL = 0x40 = 64

第10字节：06
- 协议 = 0x06 = 6（TCP）

**(2) IP版本：**

版本字段值为4，使用的是IPv4。

**(3) 传输层协议：**

协议字段值为6，对应TCP协议。常见协议号：1=ICMP，6=TCP，17=UDP。
