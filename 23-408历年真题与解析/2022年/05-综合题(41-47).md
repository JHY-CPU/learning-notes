## 第41题 (10分) ——数据结构
已知一个无向连通图G采用邻接表存储结构。设图中有n个顶点和e条边。请完成以下任务：

(1) 编写算法，判断图G是否为一棵树。若是树则返回1，否则返回0。（4分）

(2) 编写算法，求图G中顶点v所在的连通分量的顶点个数。（3分）

(3) 分析上述两个算法的时间复杂度。（3分）

**解析：**

**(1) 判断图G是否为一棵树**

图G是树的充要条件：n个顶点、n-1条边且连通。

```c
int IsTree(ALGraph G) {
    // 统计边数
    int edgeCount = 0;
    for (int i = 0; i < G.vexnum; i++) {
        ArcNode *p = G.vertices[i].firstarc;
        while (p != NULL) {
            edgeCount++;
            p = p->nextarc;
        }
    }
    edgeCount = edgeCount / 2; // 无向图每条边统计了两次

    // 判断边数是否为n-1
    if (edgeCount != G.vexnum - 1)
        return 0;

    // 判断是否连通（DFS遍历判断访问顶点数）
    int visited[MAX_VERTEX_NUM] = {0};
    int count = 0;
    DFS(G, 0, visited, &count); // 从顶点0开始DFS

    if (count == G.vexnum)
        return 1;
    else
        return 0;
}

void DFS(ALGraph G, int v, int visited[], int *count) {
    visited[v] = 1;
    (*count)++;
    ArcNode *p = G.vertices[v].firstarc;
    while (p != NULL) {
        if (!visited[p->adjvex])
            DFS(G, p->adjvex, visited, count);
        p = p->nextarc;
    }
}
```

**(2) 求顶点v所在连通分量的顶点个数**

```c
int ComponentCount(ALGraph G, int v) {
    int visited[MAX_VERTEX_NUM] = {0};
    int count = 0;
    DFS(G, v, visited, &count);
    return count;
}
```

**(3) 时间复杂度分析**

- 算法(1)：统计边数需要遍历所有顶点的邻接链表，O(n+e)；DFS遍历连通图，O(n+e)。总时间复杂度为**O(n+e)**。
- 算法(2)：DFS遍历v所在的连通分量，需要访问该分量的所有顶点和边，时间复杂度为**O(n'+e')**，其中n'和e'分别是连通分量中的顶点数和边数，最坏情况下为O(n+e)。

## 第42题 (10分) ——数据结构
已知一棵二叉树的中序遍历序列和后序遍历序列分别存储在数组in[]和post[]中，n为结点个数。请完成以下任务：

(1) 设计算法，根据中序遍历和后序遍历序列重建该二叉树。（4分）

(2) 设计算法，输出该二叉树从根结点到每个叶子结点的路径。（3分）

(3) 若中序序列为{4,7,2,1,5,3,8,6}，后序序列为{7,4,2,5,8,6,3,1}，请画出重建的二叉树并输出所有根到叶的路径。（3分）

**解析：**

**(1) 重建二叉树**

```c
// post[postL..postR]为后序序列，in[inL..inR]为中序序列
BiTree BuildTree(int in[], int post[], int inL, int inR,
                 int postL, int postR) {
    if (postL > postR) return NULL;

    // 后序最后一个元素为根结点
    BiTree root = (BiTree)malloc(sizeof(BiTNode));
    root->data = post[postR];
    root->lchild = root->rchild = NULL;

    // 在中序序列中找到根结点位置
    int k;
    for (k = inL; k <= inR; k++) {
        if (in[k] == post[postR]) break;
    }

    int leftNum = k - inL; // 左子树结点个数

    // 递归构建左子树和右子树
    root->lchild = BuildTree(in, post, inL, k-1, postL, postL+leftNum-1);
    root->rchild = BuildTree(in, post, k+1, inR, postL+leftNum, postR-1);

    return root;
}
```

**(2) 输出根到叶的路径**

```c
void PrintPath(BiTree T, int path[], int pathLen) {
    if (T == NULL) return;

    path[pathLen] = T->data;
    pathLen++;

    // 叶子结点，输出路径
    if (T->lchild == NULL && T->rchild == NULL) {
        for (int i = 0; i < pathLen; i++) {
            printf("%d ", path[i]);
        }
        printf("\n");
        return;
    }

    PrintPath(T->lchild, path, pathLen);
    PrintPath(T->rchild, path, pathLen);
}
```

**(3) 重建结果和路径**

根据给定序列重建的二叉树：
```
         1
        / \
       2   3
      /   / \
     4   5   6
      \     /
       7   8
```

根到叶的路径：
- 1→2→4→7
- 1→3→5
- 1→3→6→8

## 第43题 (10分) ——计算机组成原理
某计算机的存储系统采用Cache-主存层次结构。主存容量为1MB，按字节编址，字长为32位。Cache容量为8KB，采用直接映射方式，Cache的每个数据块大小为16B。请回答以下问题：

(1) 主存地址和Cache地址各为多少位？主存地址如何划分为标记、Cache行号和块内地址三个字段？各字段各占多少位？（4分）

(2) 若Cache初始为空，CPU依次访问以下主存地址（十六进制）：0x00000, 0x00010, 0x10000, 0x00020, 0x20010, 0x00010。请给出每次访问时Cache的命中情况。（3分）

(3) 若Cache的命中率为98%，Cache的访问时间为10ns，主存的访问时间为100ns，求Cache-主存系统的平均访问时间。（3分）

**解析：**

**(1) 地址划分**

- 主存容量1MB = 2^20 B，主存地址20位。
- Cache容量8KB = 2^13 B，数据块大小16B = 2^4 B。
- Cache行数 = 2^13 / 2^4 = 2^9 = 512行。

主存地址划分（20位）：
| 标记(Tag) | Cache行号 | 块内地址 |
|-----------|----------|---------|
| 7位       | 9位       | 4位      |

- 块内地址：4位（2^4 = 16B）
- Cache行号：9位（2^9 = 512行）
- 标记：20 - 9 - 4 = 7位

Cache地址 = Cache行号(9位) + 块内地址(4位) = 13位。

**(2) 访问情况分析**

- **0x00000** = 0000 0000 0000 0000 0000：Tag=0x00, 行号=0, 块内偏移=0。Cache空，未命中，调入第0行，标记=0x00。
- **0x00010** = 0000 0000 0000 0001 0000：Tag=0x00, 行号=1, 块内偏移=0。Cache第1行空，未命中，调入第1行，标记=0x00。
- **0x10000** = 0001 0000 0000 0000 0000：Tag=0x01, 行号=0, 块内偏移=0。Cache第0行标记=0x00，标记不匹配，未命中（替换），标记改为0x01。
- **0x00020** = 0000 0000 0000 0010 0000：Tag=0x00, 行号=2, 块内偏移=0。Cache第2行空，未命中，调入第2行，标记=0x00。
- **0x20010** = 0010 0000 0000 0001 0000：Tag=0x02, 行号=1, 块内偏移=0。Cache第1行标记=0x00，标记不匹配，未命中（替换），标记改为0x02。
- **0x00010** = 0000 0000 0000 0001 0000：Tag=0x00, 行号=1, 块内偏移=0。Cache第1行标记=0x02，标记不匹配，未命中（替换），标记改回0x00。

结果：6次访问全部未命中（0次命中），命中率0%。

**(3) 平均访问时间**

平均访问时间 = 命中率 × Cache访问时间 + (1 - 命中率) × (Cache访问时间 + 主存访问时间)
= 0.98 × 10ns + 0.02 × (10ns + 100ns)
= 9.8ns + 0.02 × 110ns
= 9.8ns + 2.2ns
= **12ns**

## 第44题 (10分) ——操作系统
在某系统中，有三类资源R1、R2、R3，分别有9、5、7个。当前有五个进程P0~P4，已分配资源矩阵Allocation和最大需求矩阵Max如下：

| 进程 | R1 | R2 | R3 |   | R1 | R2 | R3 |
|------|----|----|----|---|----|----|----|
| P0   | 1  | 0  | 1  |   | 4  | 2  | 3  |
| P1   | 2  | 1  | 0  |   | 3  | 2  | 1  |
| P2   | 1  | 0  | 2  |   | 3  | 3  | 3  |
| P3   | 0  | 1  | 2  |   | 2  | 2  | 3  |
| P4   | 0  | 0  | 1  |   | 3  | 1  | 2  |

请回答以下问题：

(1) 利用银行家算法计算当前系统的可用资源向量Available。（2分）

(2) 利用银行家算法判断当前系统是否处于安全状态，若是则给出一个安全序列。（4分）

(3) 若此时进程P2请求资源Request(1, 0, 0)，系统能否分配？说明原因。（4分）

**解析：**

**(1) 计算Available**

已分配资源总和：
- R1: 1+2+1+0+0 = 4
- R2: 0+1+0+1+0 = 2
- R3: 1+0+2+2+1 = 6

Available = 总资源 - 已分配 = (9,5,7) - (4,2,6) = **(5, 3, 1)**

**(2) 安全性检测**

Need = Max - Allocation：
| 进程 | R1 | R2 | R3 |
|------|----|----|----|
| P0   | 3  | 2  | 2  |
| P1   | 1  | 1  | 1  |
| P2   | 2  | 3  | 1  |
| P3   | 2  | 1  | 1  |
| P4   | 3  | 1  | 1  |

Work = Available = (5, 3, 1)

1. P1: Need=(1,1,1) <= Work=(5,3,1)，满足。Work = (5,3,1)+(2,1,0) = (7,4,1)
2. P3: Need=(2,1,1) <= Work=(7,4,1)，满足。Work = (7,4,1)+(0,1,2) = (7,5,3)
3. P4: Need=(3,1,1) <= Work=(7,5,3)，满足。Work = (7,5,3)+(0,0,1) = (7,5,4)
4. P0: Need=(3,2,2) <= Work=(7,5,4)，满足。Work = (7,5,4)+(1,0,1) = (8,5,5)
5. P2: Need=(2,3,1) <= Work=(8,5,5)，满足。Work = (8,5,5)+(1,0,2) = (9,5,7)

系统处于安全状态，安全序列为：**P1 → P3 → P4 → P0 → P2**

**(3) P2请求Request(1,0,0)**

先试探分配：
- Available = (5,3,1) - (1,0,0) = (4,3,1)
- Allocation_P2 = (1,0,2) + (1,0,0) = (2,0,2)
- Need_P2 = (2,3,1) - (1,0,0) = (1,3,1)

安全性检测：
Work = (4,3,1)

1. P1: Need=(1,1,1) <= (4,3,1)，满足。Work=(6,4,1)
2. P3: Need=(2,1,1) <= (6,4,1)，满足。Work=(6,5,3)
3. P4: Need=(3,1,1) <= (6,5,3)，满足。Work=(6,5,4)
4. P0: Need=(3,2,2) <= (6,5,4)，满足。Work=(7,5,5)
5. P2: Need=(1,3,1) <= (7,5,5)，满足。Work=(8,5,7)

存在安全序列P1→P3→P4→P0→P2，**系统可以分配**。

## 第45题 (10分) ——操作系统
某文件系统采用混合索引分配方式，索引结点（inode）中包含10个直接块指针、1个一级间接块指针、1个二级间接块指针和1个三级间接块指针。磁盘块大小为1KB，每个磁盘块地址占4B。请回答以下问题：

(1) 该文件系统支持的最大文件大小是多少？（4分）

(2) 若要访问文件中第5000B处的数据（从0开始编号），需要几次磁盘I/O操作？给出分析过程。（3分）

(3) 若要访问文件中第500000B处的数据，需要几次磁盘I/O操作？给出分析过程。（3分）

**解析：**

**(1) 最大文件大小**

- 每个磁盘块可以存放的地址数 = 1KB / 4B = 256个
- 直接索引：10 × 1KB = 10KB
- 一级间接：1 × 256 × 1KB = 256KB
- 二级间接：1 × 256 × 256 × 1KB = 65536KB = 64MB
- 三级间接：1 × 256 × 256 × 256 × 1KB = 16777216KB = 16GB

最大文件大小 = 10KB + 256KB + 64MB + 16GB ≈ **16GB + 64MB + 266KB ≈ 16.06GB**

**(2) 访问第5000B处数据**

第5000B所在的逻辑块号 = 5000 / 1024 = 4（块内偏移808B）。

块号4 < 10，属于直接块。需要的磁盘I/O操作：
1. 读inode（通常inode在内存中，若不在需1次I/O）
2. 读第4号直接块（1次I/O）

假设inode已在内存中，需要**1次**磁盘I/O操作。

**(3) 访问第500000B处数据**

第500000B所在的逻辑块号 = 500000 / 1024 = 488。

- 直接块：0~9号（10个块）
- 一级间接：10~265号（256个块）
- 二级间接：266~65793号（65536个块）

块号488在一级间接块范围内（10 ≤ 488 ≤ 265）。在一级间接块中，偏移 = 488 - 10 = 478。

需要的磁盘I/O操作：
1. 读一级间接块（1次I/O）：得到第478项的地址
2. 读目标数据块（1次I/O）

共需要**2次**磁盘I/O操作。

## 第46题 (10分) ——计算机网络
在如图所示的网络中，各链路的代价标注在边上。使用距离向量路由算法（Bellman-Ford方程），请回答以下问题：

```
   2     1
A ──── B ──── C
|      |      |
3      4      2
|      |      |
D ──── E ──── F
   2     3
```

(1) 写出各结点初始的距离向量表。（3分）

(2) 经过一轮信息交换后，各结点的距离向量表如何变化？（4分）

(3) 该算法是否会遇到"无穷计数"问题？如何解决？（3分）

**解析：**

**(1) 初始距离向量**

各结点只知道到直接相连邻居的代价：
| 结点 | 到A | 到B | 到C | 到D | 到E | 到F |
|------|-----|-----|-----|-----|-----|-----|
| A    | 0   | 2   | ∞   | 3   | ∞   | ∞   |
| B    | 2   | 0   | 1   | ∞   | 4   | ∞   |
| C    | ∞   | 1   | 0   | ∞   | ∞   | 2   |
| D    | 3   | ∞   | ∞   | 0   | 2   | ∞   |
| E    | ∞   | 4   | ∞   | 2   | 0   | 3   |
| F    | ∞   | ∞   | 2   | ∞   | 3   | 0   |

**(2) 一轮信息交换后**

各结点与邻居交换距离向量后，使用Bellman-Ford方程更新：

| 结点 | 到A | 到B | 到C | 到D | 到E | 到F |
|------|-----|-----|-----|-----|-----|-----|
| A    | 0   | 2   | 3(B) | 3   | 6(B) | 5(B,C) |
| B    | 2   | 0   | 1   | 5(A) | 4   | 3(C) |
| C    | 3(B)| 1   | 0   | ∞   | 5(B) | 2   |
| D    | 3   | 5(A)| ∞   | 0   | 2   | 5(E) |
| E    | 6(B)| 4   | 5(B) | 2   | 0   | 3   |
| F    | 5(C)| 3(C)| 2   | 5(E) | 3   | 0   |

例如，A到C：A→B→C = 2+1=3（比∞小，更新）。

**(3) 无穷计数问题**

距离向量算法确实会遇到无穷计数问题。当链路代价增加或链路故障时，相关结点可能会收到过时的信息，导致路由环路和计数缓慢收敛到无穷大。

解决方法：
1. **定义最大跳数**：如RIP定义最大跳数为16表示无穷大。
2. **水平分割（Split Horizon）**：不向信息来源方向回传路由信息。
3. **带毒性逆转的水平分割**：向信息来源方向回传度量值为无穷大的路由信息。
4. **触发更新**：路由变化时立即发送更新，不必等待更新周期。

## 第47题 (10分) ——计算机网络
主机A向主机B发送一个TCP报文段，已知A和B之间建立了一条TCP连接。主机A的初始序号为1000，主机B的初始序号为2000。请回答以下问题：

(1) 画出TCP三次握手建立连接的过程，标明各报文段的SYN、ACK标志位、序号和确认号。（3分）

(2) 建立连接后，主机A向主机B发送一个携带500B数据的TCP报文段，主机B收到后发送确认。画出这个过程，标明各报文段的序号、确认号和数据长度。（3分）

(3) 在上述过程中，若主机B发送的确认报文段丢失，主机A在超时重传计时器到期后会如何操作？假设TCP采用拥塞控制，慢启动阈值ssthresh初始为16个MSS，重传时拥塞窗口cwnd为10个MSS。请描述cwnd和ssthresh的变化过程。（4分）

**解析：**

**(1) 三次握手**

  |                              |
  |  第一次握手                   |
  |  SYN=1, ACK=0               |
  |  Seq=1000                   |

  |                              |
  |  第二次握手                   |
  |  SYN=1, ACK=1               |
  |  Seq=2000, Ack=1001         |

  |                              |
  |  第三次握手                   |
  |  SYN=0, ACK=1               |

**(2) 数据传输**

  |  Seq=2001, Ack=1501         |

**(3) 超时重传与拥塞控制**

TCP超时重传处理：
- 发送方A在超时重传计时器到期后，重传丢失数据的报文段。

拥塞控制参数变化：
- **ssthresh** = 原cwnd / 2 = 10 / 2 = **5个MSS**
- **cwnd** = 1个MSS（回到慢启动状态）

随后的拥塞控制过程：
- 慢启动阶段：cwnd从1开始，每个RTT翻倍（1→2→4→5）
- 当cwnd ≥ ssthresh（5个MSS）时，进入拥塞避免阶段
- 拥塞避免阶段：每个RTT，cwnd增加1个MSS（5→6→7→8→...）

cwnd变化：10→**1**(超时)→1→2→4→**5**(到达阈值)→5→6→7→...
