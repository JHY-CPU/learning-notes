## 第41题 (10分) - 数据结构

已知一棵二叉树采用二叉链表存储结构存储，结点结构为：

```
| lchild | data | rchild |
```

其中data为字符型。请设计一个算法，判断该二叉树是否为完全二叉树。要求：

(1) 给出算法的基本设计思想。(3分)
(2) 用C/C++语言描述该算法（注释风格不限）。(5分)
(3) 说明算法的时间复杂度和空间复杂度。(2分)

**解析：**

(1) **基本设计思想：**

采用层序遍历（广度优先搜索）的方式。完全二叉树的特点是：在层序遍历时，一旦遇到空结点，之后不应该再出现非空结点。具体方法：使用队列进行层序遍历，当遇到第一个空结点后设置标志flag=true，此后如果再遇到非空结点，则说明不是完全二叉树。

(2) **算法描述：**

```c
// 定义二叉树结点结构
typedef struct BiTNode {
    char data;
    struct BiTNode *lchild, *rchild;
} BiTNode, *BiTree;

// 判断二叉树是否为完全二叉树
// 返回值：1表示是完全二叉树，0表示不是
int isCompleteBT(BiTree T) {
    if (T == NULL) return 1; // 空树是完全二叉树
    
    BiTree Q[MaxSize]; // 循环队列
    int front = 0, rear = 0;
    
    Q[rear++] = T;      // 根结点入队
    int flag = 0;        // 标记是否已遇到空结点
    
    while (front != rear) {
        BiTree p = Q[front++]; // 出队
        
        if (p == NULL) {
            flag = 1;  // 遇到空结点，设置标志
        } else {
            if (flag == 1) {
                return 0; // 遇到空结点后又遇到非空结点，不是完全二叉树
            }
            // 不管左右孩子是否为空，都入队
            Q[rear++] = p->lchild;
            Q[rear++] = p->rchild;
        }
    }
    return 1; // 遍历完毕，是完全二叉树
}
```

(3) **复杂度分析：**

- **时间复杂度：** O(n)，其中n为二叉树的结点数。层序遍历每个结点最多入队一次出队一次。
- **空间复杂度：** O(n)，队列中最多存放n个结点指针（当二叉树为完全二叉树时，最后一层之前的所有结点都会入队）。

## 第42题 (10分) - 数据结构

已知一个无向连通图G采用邻接表存储，设图中有n个顶点和e条边。请设计一个算法，判断图G是否为一棵树。要求：

(1) 给出算法的基本设计思想。(3分)
(2) 用C/C++语言描述该算法。(5分)
(3) 说明算法的时间复杂度。(2分)

**解析：**

(1) **基本设计思想：**

无向图G是一棵树的充分必要条件是：G连通且有n-1条边（n为顶点数）。或者等价地：G连通且无环。算法思路：先判断边数e是否等于n-1，若不等于则不是树。若e=n-1，再通过DFS或BFS判断图是否连通（遍历后访问顶点数是否等于n）。

(2) **算法描述：**

```c
// 邻接表存储结构
typedef struct ArcNode {
    int adjvex;
    struct ArcNode *nextarc;
} ArcNode;

typedef struct VNode {
    char data;
    ArcNode *firstarc;
} VNode, AdjList[MaxVertexNum];

typedef struct {
    AdjList vertices;
    int vexnum, arcnum;
} ALGraph;

int visited[MaxVertexNum];

// DFS遍历
void DFS(ALGraph G, int v, int *count) {
    visited[v] = 1;
    (*count)++;
    ArcNode *p = G.vertices[v].firstarc;
    while (p != NULL) {
        if (!visited[p->adjvex]) {
            DFS(G, p->adjvex, count);
        }
        p = p->nextarc;
    }
}

// 判断图G是否为一棵树
// 返回值：1表示是树，0表示不是
int isTree(ALGraph G) {
    // 条件1：边数必须等于顶点数减1
    if (G.arcnum != G.vexnum - 1) {
        return 0;
    }
    
    // 条件2：图必须连通
    // 初始化visited数组
    for (int i = 0; i < G.vexnum; i++) {
        visited[i] = 0;
    }
    
    int count = 0;
    DFS(G, 0, &count); // 从顶点0开始DFS
    
    // 如果访问的顶点数等于总顶点数，则图连通
    if (count == G.vexnum) {
        return 1;
    } else {
        return 0;
    }
}
```

(3) **时间复杂度：**

判断边数是否为n-1的时间复杂度为O(1)。DFS遍历的时间复杂度为O(n+e)。总时间复杂度为O(n+e)。当e=n-1时，也可写作O(n)。

## 第43题 (10分) - 计算机组成原理

某计算机采用定长指令格式，指令字长16位。指令格式如下：

| 操作码(4位) | 寻址方式(2位) | 地址码(10位) |
|---|---|---|

寻址方式编码：00-立即寻址，01-直接寻址，10-间接寻址，11-相对寻址。

设有如下指令序列，寄存器R0的内容为1000H，存储单元(1000H)=2000H，(2000H)=3000H，PC当前值为200H。

请回答以下问题：

(1) 若执行指令"MOV R0, #1000H"（立即寻址），则R0的内容是什么？(2分)
(2) 若执行指令"MOV R0, (1000H)"（直接寻址），则R0的内容是什么？(3分)
(3) 若执行指令"MOV R0, @(1000H)"（间接寻址），则R0的内容是什么？(3分)
(4) 若执行指令"JMP *(PC+100H)"（相对寻址），则下一条指令的地址是什么？(2分)

**解析：**

(1) **立即寻址：** 操作数直接包含在指令中，即地址码部分就是操作数本身。R0 = 1000H。

(2) **直接寻址：** 地址码给出操作数在内存中的地址。操作数地址 = 1000H，查内存(1000H) = 2000H。因此R0 = 2000H。

(3) **间接寻址：** 地址码给出的是操作数地址的地址。先从地址码1000H处取出(1000H)=2000H作为操作数地址，再从2000H处取出(2000H)=3000H作为操作数。因此R0 = 3000H。

(4) **相对寻址：** 操作数地址 = PC + 偏移量。执行该指令时，PC已经自增到下一条指令的地址（200H + 2 = 202H，假设指令长2字节）。实际跳转地址 = PC + 100H = 202H + 100H = 302H。因此下一条指令的地址为302H。

> **注：** 相对寻址中偏移量"100H"的写法存在歧义。题目地址码字段为10位，若偏移量为十六进制100H（=256），可放入10位字段；若为十进制100（=64H），结果应为202H+64H=266H。按题目给出的"100H"理解为十六进制值256，结果302H。

## 第44题 (10分) - 操作系统

某系统有同类资源m个，供n个进程共享。如果每个进程最多申请x个资源（1≤x≤m），请回答：

(1) 当n、m、x满足什么关系时，系统不会发生死锁？请证明。(5分)
(2) 当m=6、n=3时，各进程最大需求x的最大值是多少？(2分)
(3) 当m=6时，若n=2，则x最大为多少？(3分)

**解析：**

(1) **不发生死锁的条件：**

假设每个进程都已获得了最大需求量减1个资源（即x-1个），这是最坏情况。此时n个进程共占用n(x-1)个资源。如果系统中还有至少1个剩余资源，则某个进程可以申请到这个资源并完成执行，释放其所有资源，从而其他进程也能依次完成，不会发生死锁。

不发生死锁的条件为：**m ≥ n(x-1) + 1**

证明：最坏情况下，n个进程各持有x-1个资源，共占用n(x-1)个。若系统剩余资源数 m - n(x-1) ≥ 1，即至少还有1个资源可分配给某个进程使其完成，完成后释放全部x个资源，其他进程可继续执行。因此 m ≥ n(x-1) + 1 保证不发生死锁。

(2) **m=6, n=3：** 代入条件 6 ≥ 3(x-1) + 1，即 6 ≥ 3x - 2，即 3x ≤ 8，x ≤ 8/3 ≈ 2.67，x为整数，因此 **x的最大值为2**。

验证：x=2时，最坏情况各进程持有1个，共3个，剩余3个资源足够。x=3时，最坏情况各进程持有2个，共6个，剩余0个，可能发生死锁。

(3) **m=6, n=2：** 代入条件 6 ≥ 2(x-1) + 1，即 6 ≥ 2x - 1，即 2x ≤ 7，x ≤ 3.5，x为整数，因此 **x的最大值为3**。

验证：x=3时，最坏情况各进程持有2个，共4个，剩余2个资源可分配。x=4时，最坏情况各进程持有3个，共6个，剩余0个，可能发生死锁。

## 第45题 (10分) - 操作系统

在一个请求分页存储管理系统中，某进程的页面访问序列为1, 2, 3, 4, 1, 2, 5, 1, 2, 3, 4, 5。系统为该进程分配了3个物理页框（初始均为空）。

(1) 使用FIFO页面置换算法，写出页面置换过程，并计算缺页次数和缺页率。(5分)
(2) 使用LRU页面置换算法，写出页面置换过程，并计算缺页次数和缺页率。(5分)

**解析：**

**(1) FIFO算法：**

页面访问序列：1, 2, 3, 4, 1, 2, 5, 1, 2, 3, 4, 5

| 访问 | 页框状态 | 是否缺页 |
|------|---------|---------|
| 1 | [1, -, -] | 缺页(1) |
| 2 | [1, 2, -] | 缺页(2) |
| 3 | [1, 2, 3] | 缺页(3) |
| 4 | [4, 2, 3] 替换1 | 缺页(4) |
| 1 | [4, 1, 3] 替换2 | 缺页(5) |
| 2 | [4, 1, 2] 替换3 | 缺页(6) |
| 5 | [5, 1, 2] 替换4 | 缺页(7) |
| 1 | [5, 1, 2] | 命中 |
| 2 | [5, 1, 2] | 命中 |
| 3 | [5, 3, 2] 替换1 | 缺页(8) |
| 4 | [5, 3, 4] 替换2 | 缺页(9) |
| 5 | [5, 3, 4] | 命中 |

**FIFO缺页次数 = 9次，缺页率 = 9/12 = 75%**

**(2) LRU算法：**

| 访问 | 页框状态 | 是否缺页 |
|------|---------|---------|
| 1 | [1, -, -] | 缺页(1) |
| 2 | [1, 2, -] | 缺页(2) |
| 3 | [1, 2, 3] | 缺页(3) |
| 4 | [4, 2, 3] 替换1(最久未用) | 缺页(4) |
| 1 | [4, 1, 3] 替换2 | 缺页(5) |
| 2 | [4, 1, 2] 替换3 | 缺页(6) |
| 5 | [5, 1, 2] 替换4(最久未用) | 缺页(7) |
| 1 | [5, 1, 2] | 命中(1最近用) |
| 2 | [5, 1, 2] | 命中(2最近用) |
| 3 | [5, 3, 2] 替换1(1是最久未用的) | 缺页(8) |
| 4 | [5, 3, 4] 替换2(2是最久未用的) | 缺页(9) |
| 5 | [5, 3, 4] | 命中(5最近用) |

**LRU缺页次数 = 9次，缺页率 = 9/12 = 75%**

## 第46题 (10分) - 计算机网络

某网络拓扑如下图所示，其中R1、R2、R3为路由器，H1、H2为主机。

```
H1 -- R1 -- R2 -- R3 -- H2
      |10.0.0.0/8   |172.16.0.0/16
```

各接口IP地址如下：
- R1连接H1的接口：10.0.0.1/8
- H1：10.0.0.2/8，默认网关10.0.0.1
- R2连接R1的接口：10.0.1.1/8
- R2连接R3的接口：172.16.0.1/16
- R3连接H2的接口：172.16.0.2/16
- H2：172.16.0.3/16，默认网关172.16.0.2

请回答以下问题：

(1) 主机H1发送IP数据报给H2时，源IP地址和目的IP地址分别是什么？(2分)
(2) 该数据报到达R1时，R1如何确定下一跳？(3分)
(3) 若H1要ping通H2，经过了R1、R2、R3三个路由器，请写出数据报在R2的两个接口上转发时的源MAC地址和目的MAC地址。(5分)

**解析：**

(1) 源IP地址为10.0.0.2（H1的IP地址），目的IP地址为172.16.0.3（H2的IP地址）。IP数据报在转发过程中源IP和目的IP保持不变（不考虑NAT）。

(2) R1收到数据报后，提取目的IP地址172.16.0.3，与自己的路由表逐条匹配（最长前缀匹配）。172.16.0.3不属于10.0.0.0/8网段，R1查找路由表中是否有到达172.16.0.0/16的路由。如果有，则按匹配的下一跳转发；如果没有精确匹配，则通过默认路由（0.0.0.0/0）指向的下一跳转发。假设R1的路由表中有到达172.16.0.0/16的路由条目，下一跳为R2（10.0.1.1）。

(3) **数据报经过R1→R2→R3的MAC地址变化：**

IP数据报在整个过程中源IP(10.0.0.2)和目的IP(172.16.0.3)不变，但MAC帧在每一段链路上都会改变。

**R1→R2段（R1的出接口发向R2）：**
- 源MAC地址：R1连接R2接口的MAC地址
- 目的MAC地址：R2连接R1接口的MAC地址（R2的10.0.1.1接口的MAC）

**R2→R3段（R2的出接口发向R3）：**
- 源MAC地址：R2连接R3接口的MAC地址（R2的172.16.0.1接口的MAC）
- 目的MAC地址：R3连接R2接口的MAC地址（R3的172.16.0.2接口的MAC）

注意：MAC地址是逐跳改变的，每经过一个路由器，源MAC变为转发路由器出接口的MAC，目的MAC变为下一跳路由器入接口的MAC。

## 第47题 (10分) - 计算机网络

主机A向主机B发送一个大文件，采用TCP协议。设TCP报文段的数据部分最大长度为MSS=1000字节，A和B之间的RTT=20ms，A的发送窗口大小为4000字节，B的接收窗口大小为5000字节，A的拥塞窗口初始值为1MSS，慢启动阈值ssthresh初始值为8MSS。

(1) 从开始发送到A收到第一个确认，TCP的拥塞窗口经历了怎样的变化过程？(4分)
(2) 若在第10个RTT时发生超时，之后拥塞窗口和ssthresh如何变化？(3分)
(3) 假设不考虑拥塞控制，仅从流量控制角度分析，A的发送窗口最大为多少字节？(3分)

**解析：**

(1) **拥塞窗口变化过程（慢启动阶段）：**

- RTT 1：cwnd = 1MSS = 1000B，可发送1个报文段
- RTT 2：cwnd = 2MSS = 2000B，可发送2个报文段
- RTT 3：cwnd = 4MSS = 4000B，可发送4个报文段（此时受发送窗口4000B限制，实际只能发4个）
- RTT 4：cwnd = 8MSS = 8000B，达到ssthresh=8MSS，转入拥塞避免阶段

从第4个RTT开始，进入拥塞避免阶段，每个RTT拥塞窗口增加1MSS：
- RTT 5：cwnd = 9MSS
- RTT 6：cwnd = 10MSS
- ...

(2) **超时后的变化：**

在第10个RTT时发生超时：
- ssthresh = cwnd/2（即超时时cwnd值的一半）
- cwnd = 1MSS（重置为1个MSS）
- 重新进入慢启动阶段

假设第10个RTT时cwnd=13MSS（从第4个RTT的8MSS开始每RTT+1MSS，到第10个RTT增加了6次），则：
- 新的ssthresh = 13/2 = 6MSS（取整）
- cwnd = 1MSS
- 然后从1MSS开始慢启动，每RTT翻倍直到达到新的ssthresh

(3) **流量控制下的最大发送窗口：**

TCP的发送窗口 = min(接收窗口rwnd, 拥塞窗口cwnd)。从流量控制角度，发送方受到接收方通告的接收窗口的限制。B的接收窗口为5000字节，因此即使cwnd很大，发送窗口也不能超过5000字节。A的发送窗口最大为 min(5000, cwnd)，即最大为 **5000字节**（当cwnd≥5MSS时）。
