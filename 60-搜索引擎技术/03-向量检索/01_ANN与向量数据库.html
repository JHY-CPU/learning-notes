向量检索ANNHNSW

近似最近邻搜索(ANN)是在高维空间中快速找到最相似向量的核心技术。

近似最近邻 (ANN) 问题

精确最近邻 vs 近似最近邻： 精确最近邻 (Exact NN)： - 遍历所有向量，计算距离 - 时间复杂度：O(n×d)，n为向量数，d为维度 - 百万级数据可能需要秒级响应 - 不适合大规模实时检索 近似最近邻 (ANN)： - 牺牲少量精度换取大幅性能提升 - 查询时间：O(log n) 或 O(1) - 召回率通常 >95% - 适合百万到数十亿级向量 距离度量： - 欧几里得距离：d = sqrt(Σ(ai - bi)²) - 余弦相似度：cos(a,b) = (a·b)/(|a|×|b|) - 内积：ip = a·b（用于归一化向量时等价于余弦） 应用场景： - 语义搜索：文本嵌入向量检索 - 图像搜索：以图搜图 - 推荐系统：相似物品/用户 - RAG：检索增强生成

HNSW (分层可导航小世界图)

HNSW 是目前最流行的ANN算法之一 核心思想：构建多层跳表式的近邻图 结构示意： Layer 3: ●─────────────────────────────● Layer 2: ●─────────●───────────●───────● Layer 1: ●────●────●─────●─────●───●───● Layer 0: ●──●─●──●─●──●──●──●──●──●──●──● (底层包含所有节点) 搜索过程： 1. 从最高层的入口点开始 2. 在当前层找到最近邻 3. 下降到下一层 4. 重复直到最底层 5. 在最底层精细化搜索，返回Top-K 参数： - M：每层每个节点的最大邻居数（通常16-64） - efConstruction：构建时的搜索范围（越大越精确，构建越慢） - efSearch：查询时的搜索范围（越大召回率越高，查询越慢） 优势： - 查询时间 O(log n) - 支持动态增删节点 - 高召回率（>99%） - 对高维数据效果好 劣势： - 内存占用大（图结构在内存中） - 构建时间较长

IVF (倒排文件索引)

IVF：基于聚类的ANN方法 核心思想： 1. 用k-means将所有向量聚类为nlist个簇 2. 每个簇有一个中心点和一个倒排列表 3. 查询时只搜索最近的nprobe个簇 构建流程： ┌─────────────────────────────────────────────┐ │ 向量集合 │ │ v1, v2, v3, ..., vn │ │ ↓ K-Means聚类 │ │ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ │ │ │簇1 │ │簇2 │ │簇3 │ │簇4 │ │ │ │c1 │ │c2 │ │c3 │ │c4 │ │ │ │v1,v5 │ │v2,v7 │ │v3,v6 │ │v4,v8 │ │ │ └──────┘ └──────┘ └──────┘ └──────┘ │ └─────────────────────────────────────────────┘ 查询流程： 1. 计算查询向量与所有簇中心的距离 2. 选择最近的 nprobe 个簇 3. 在选中簇的向量中进行精确搜索 4. 返回 Top-K 结果 IVF-PQ (乘积量化)： - IVF：粗粒度筛选（减少搜索范围） - PQ：细粒度压缩（减少内存和距离计算量） - 将高维向量分为m段，每段独立量化 - 适用于超大规模（亿级）向量检索 参数调优： - nlist：簇数量（通常sqrt(n)） - nprobe：搜索簇数（越大召回率越高，速度越慢）
