# 倒排索引


## 倒排索引 (Inverted Index)


搜索引擎索引TF-IDF


倒排索引是搜索引擎的核心数据结构，支持从词条到文档的快速映射。


## 倒排索引结构


```
正排索引 vs 倒排索引：

正排索引（文档 → 词条）：
Doc1 → [搜索引擎, 是, 信息, 检索, 系统]
Doc2 → [信息检索, 系统, 的, 核心, 是, 倒排索引]
Doc3 → [搜索引擎, 使用, 倒排索引, 进行, 检索]

倒排索引（词条 → 文档）：
搜索引擎  → [(Doc1, pos=[0]), (Doc3, pos=[0])]
信息检索  → [(Doc2, pos=[0])]
系统      → [(Doc1, pos=[4]), (Doc2, pos=[2])]
倒排索引  → [(Doc2, pos=[5]), (Doc3, pos=[2])]
检索      → [(Doc1, pos=[3]), (Doc3, pos=[4])]

倒排索引组成：
┌─────────────────────────────────────────────┐
│  词典 (Dictionary / Vocabulary)              │
│  ├── 存储所有不重复的词条(Term)               │
│  ├── 常用数据结构：哈希表、Trie树、FST        │
│  └── 支持前缀查询和模糊匹配                   │
│                                             │
│  倒排表 (Posting List)                       │
│  ├── 每个词条对应一个倒排表                    │
│  ├── 记录包含该词条的文档列表                  │
│  └── 每条记录包含：                           │
│      - DocID：文档编号                        │
│      - TF：词频（该词在文档中出现的次数）      │
│      - Positions：词在文档中的位置列表         │
│      - Payloads：附加信息（如词性、粗细体等）  │
└─────────────────────────────────────────────┘
```


## 索引构建流程


```
构建流程：
1. 文档收集 (Collect)
2. 文本预处理 (Preprocessing)
   ├── 分词 (Tokenization)
   ├── 小写化 (Lowercasing)
   ├── 停用词过滤 (Stop Words Removal)
   └── 词干提取 (Stemming / Lemmatization)
3. 构建倒排表
4. 索引压缩
5. 索引持久化

构建算法：
- 两遍扫描法（内存不足时）
  第一遍：统计词频，确定词条顺序
  第二遍：按词条分块写入倒排表
- 内存中构建（内存充足时）
  使用哈希表收集所有词条的倒排表
  最后排序写入磁盘

索引压缩技术：
- 变长编码（Variable Byte Encoding）
- 差值编码（Delta Encoding）：DocID存储差值
- PForDelta：块级压缩，适合倒排表
- 倒排表分块 + 跳跃表(Skip List)：加速交集运算
```


## TF-IDF 词频-逆文档频率


```
TF (Term Frequency) 词频：
TF(t,d) = 词条t在文档d中出现的次数 / 文档d的总词数
→ 衡量词条在当前文档中的重要性

IDF (Inverse Document Frequency) 逆文档频率：
IDF(t) = log(总文档数 / 包含词条t的文档数)
→ 衡量词条在整个语料中的稀缺程度

TF-IDF = TF × IDF
→ 词条在一个文档中频繁出现，但在其他文档中少见，则重要性高

示例：
语料：10000篇文档
词条"的"：出现在9900篇文档中 → IDF = log(10000/9900) ≈ 0.01（几乎无区分度）
词条"搜索引擎"：出现在100篇文档中 → IDF = log(10000/100) = 4.61（高区分度）

在Doc1中：
"的" TF=0.05, TF-IDF = 0.05 × 0.01 = 0.0005
"搜索引擎" TF=0.03, TF-IDF = 0.03 × 4.61 = 0.138
→ "搜索引擎"对Doc1更关键
```


## BM25 排序算法


```
BM25 是TF-IDF的改进版本，是Lucene/Elasticsearch的默认评分函数：

BM25(q, d) = Σ IDF(qi) × [f(qi,d) × (k1+1)] / [f(qi,d) + k1 × (1 - b + b × |d|/avgdl)]

其中：
- qi：查询中的第i个词条
- f(qi,d)：词条qi在文档d中的频率
- |d|：文档d的长度
- avgdl：语料库中文档的平均长度
- k1：词频饱和参数（通常1.2-2.0）
- b：文档长度归一化参数（通常0.75）

BM25 vs TF-IDF 改进点：
1. 词频饱和：TF增加到一定程度后收益递减
   BM25使用 f/(f+k1) 替代简单的TF
2. 文档长度归一化：长文档不会因为包含更多词而天然得分高
3. 参数可调：k1控制词频影响，b控制长度归一化程度

查询示例：
Query = "搜索引擎 倒排索引"
对每个候选文档计算BM25得分，按得分排序返回Top-K
```


> **Note:** BM25是工业界最广泛使用的词法检索算法，配合倒排索引实现高效全文搜索。
