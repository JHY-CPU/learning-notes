查询优化

搜索引擎查询优化排序

查询优化是搜索引擎提升用户体验和检索效果的关键技术，涵盖查询改写、纠错和排序学习等。

查询扩展（Query Expansion）

查询扩展通过增加相关词汇来弥补查询表达的不足，提高召回率。

查询扩展策略： 1. 基于同义词的扩展 原始查询："手机" 扩展后："手机 OR 智能手机 OR 移动电话 OR cellphone" - 维护同义词词典 - 注意消歧：一词多义问题 2. 基于用户日志的扩展 - 关联规则挖掘：经常一起出现的词 - 共现分析：统计词对共现频率 - 点击日志：用户搜索A后点击了包含B的文档 3. 基于伪相关反馈（PRF） 第一轮：对原始查询检索 Top-K 文档 分析：从 Top-K 中提取高频特征词 第二轮：用扩展后的查询重新检索 - Rocchio 算法：q_new = α·q + β·(相关文档质心) - γ·(非相关文档质心) 4. 基于词向量的扩展 - 使用 Word2Vec / FastText / BERT 向量 - 找到查询词的最近邻词 - 利用语义相似度进行扩展 - 优点：捕捉语义关系，自动发现同义词 5. 基于知识图谱的扩展 - 从知识图谱中获取实体关系 - 扩展相关的属性和关联实体 - 例："iPhone 15" → 品牌:苹果, 类别:智能手机

拼写纠错（Spelling Correction）

拼写纠正是搜索引擎自动修正用户输入错误的技术。

纠错方法对比： ┌────────────────────────────────────────────────────────┐ │ 1. 编辑距离（Edit Distance） │ │ 将一个字符串转换为另一个所需的最少操作数 │ │ 操作：插入、删除、替换、交换相邻字符 │ │ "apple" → "appla" (编辑距离 = 1) │ │ 时间复杂度：O(m*n)，空间可优化 │ │ │ │ 2. 噪声信道模型（Noisy Channel Model） │ │ P(修正词|原始词) ∝ P(原始词|修正词) × P(修正词) │ │ - P(原始词|修正词)：错误模型（打字错误概率） │ │ - P(修正词)：语言模型（词的先验概率） │ │ │ │ 3. n-gram 语言模型 │ │ 利用上下文判断词的合理性 │ │ "我喜欢吃平果" → "我喜欢吃苹果" │ │ 通过 P(苹果|吃) > P(平果|吃) 判断 │ │ │ │ 4. 拼写建议生成 │ │ 候选词来源： │ │ - 编辑距离 ≤ 2 的词典词 │ │ - 发音相似词（Soundex / Metaphone） │ │ - 键盘邻近键替换（typos） │ └────────────────────────────────────────────────────────┘

查询理解（Query Understanding）

查询理解是搜索引擎分析用户真实意图的过程，是查询优化的前置步骤。

查询理解流程： ┌─────────────────────────────────────────────────────┐ │ 用户输入 → 查询改写 → 意图识别 → 查询分析 → 检索 │ └─────────────────────────────────────────────────────┘ 核心任务： 1. 查询分词与标注 "北京明天天气怎么样" → [北京/地名] [明天/时间] [天气/核心概念] [怎么样/问法] 2. 意图识别（Intent Classification） ┌───────────────┬────────────────────────────┐ │ 导航型 │ 用户想访问特定网站 │ │ 信息型 │ 用户想获取信息（"如何...") │ │ 事务型 │ 用户想执行操作（购买、下载） │ │ 本地型 │ 用户寻找本地服务 │ └───────────────┴────────────────────────────┘ 3. 实体识别 "北京到上海的高铁票" → 出发地:北京, 目的地:上海, 交通方式:高铁 4. 查询改写 - 去除停用词 - 同义词替换 - 查询归一化（繁简体、拼音） - 查询缩减（移除非关键修饰词）

Learning to Rank（LTR，学习排序）

LTR 是利用机器学习方法优化搜索结果排序的技术。

LTR 三大方法： ┌────────────────────────────────────────────────────────┐ │ Pointwise（逐点法） │ │ 将排序问题转化为回归/分类问题 │ │ 输入：单个文档的特征 │ │ 输出：相关度评分 │ │ 损失函数：MSE / 交叉熵 │ │ 缺点：忽略文档间的相对关系 │ │ │ │ Pairwise（逐对法） │ │ 判断文档对的相对排序 │ │ 输入：一对文档 (di, dj) │ │ 输出：di 是否比 dj 更相关 │ │ 算法：RankNet, LambdaRank, LambdaMART │ │ 缺点：只考虑局部关系 │ │ │ │ Listwise（列表法） │ │ 直接优化整个排序列表 │ │ 输入：候选文档列表 │ │ 输出：最优排序 │ │ 算法：ListNet, AdaRank, SVMmap │ │ 评价指标：NDCG, MAP 直接作为优化目标 │ └────────────────────────────────────────────────────────┘ LTR 特征工程： - 文本特征：BM25、TF-IDF、标题匹配度 - 链接特征：PageRank、入链数量 - 用户行为：点击率、停留时间、跳出率 - 文档特征：PageAge、权威度、新鲜度 - 查询特征：查询长度、查询频率
LambdaMART（基于 GBDT 的 Pairwise 方法）在工业界应用最广泛，被用于搜索、推荐等多个排序场景。
