# 网页去重与信息抽取


## 网页去重与信息抽取


搜索引擎去重SimHash


互联网上存在大量重复和近似重复网页，高效去重是搜索引擎的基础能力。信息抽取从网页中提取正文内容，去除模板噪声。


## 重复网页的类型


```
重复网页的分类：

1. 完全重复 (Exact Duplicate)
   - URL不同但HTML内容完全相同
   - 转载、镜像站点
   - www.example.com 和 example.com
   - http 和 https 版本

2. 近似重复 (Near Duplicate)
   - 内容大部分相同，有少量差异
   - 广告不同、时间戳不同、推荐内容不同
   - 新闻网站的同一新闻不同版本
   - 论坛帖子的不同楼层排列

3. 部分重复 (Partial Duplicate)
   - 某些段落或板块相同
   - 转载了部分原文
   - 网站模板相同（导航栏、页脚）

重复带来的问题：
- 浪费存储空间和爬虫带宽
- 搜索结果中出现重复条目
- 影响索引质量
- PageRank等算法被稀释
```


## SimHash 指纹算法


```
SimHash：Google用于大规模网页去重的核心算法
由Charikar在2002年提出，是局部敏感哈希(LSH)的一种。

核心思想：
- 将文档映射为固定长度的指纹（如64位）
- 相似文档的指纹的海明距离(Hamming Distance)很小
- 海明距离 ≤ 3 → 判定为近似重复

算法流程：
┌─────────────────────────────────────────────┐
│  1. 分词：将文档分为词条                     │
│  2. 加权：对每个词条计算权重（如TF-IDF）     │
│  3. 哈希：对每个词条计算hash值（如64位）     │
│  4. 加权叠加：                               │
│     对于hash的每一位：                        │
│       如果该位为1 → 加上权重                  │
│       如果该位为0 → 减去权重                  │
│  5. 生成指纹：                               │
│     叠加向量的每一位 > 0 → 1                 │
│     叠加向量的每一位 ≤ 0 → 0                 │
└─────────────────────────────────────────────┘

详细示例：
文档："搜索引擎 使用 倒排索引"

步骤1-2：分词 + 权重
  "搜索引擎" 权重=5
  "使用"     权重=1
  "倒排索引" 权重=4

步骤3：计算哈希（假设4位简化）
  hash("搜索引擎") = 1011
  hash("使用")     = 0110
  hash("倒排索引") = 1100

步骤4：加权叠加
  词条        权重    位0   位1   位2   位3
  搜索引擎    +5     +5    -5    +5    +5
  使用        +1     -1    +1    +1    -1
  倒排索引    +4     +4    +4    -4    -4
  ─────────────────────────────────────────
  叠加结果           +8    +0    +2    +0

步骤5：生成SimHash指纹
  +8 > 0 → 1
  +0 ≤ 0 → 0
  +2 > 0 → 1
  +0 ≤ 0 → 0
  SimHash = 1010

海明距离与去重：
- 两个SimHash指纹的海明距离 = 不同位的数量
- 海明距离 0 → 完全重复
- 海明距离 1-3 → 近似重复
- 海明距离 > 3 → 不重复

Google的经验阈值：
- 64位SimHash，海明距离 ≤ 3 → 判定重复
- 每增加一位不同的阈值，可比较的文档数增加约10倍

海明距离快速计算：
int hamming_distance(uint64_t a, uint64_t b) {
    return __builtin_popcountll(a ^ b);
}
```


## MinHash 与 Shingling


```
MinHash：另一种局部敏感哈希方法
适合集合相似度计算，结合Shingling使用

Shingling (k-shingle)：
将文档分解为连续的k个词条/字符的集合

文档："搜索引擎是信息系统"
3-shingle（字符级）：
  "搜索引擎是"
  "索引擎是信"
  "引擎是信息"
  "擎是信息系"
  "是信息系统"

Jaccard相似度：
J(A, B) = |A ∩ B| / |A ∪ B|

两篇文档的shingle集合：
Doc1 shingles: {S1, S2, S3, S4, S5}
Doc2 shingles: {S1, S2, S3, S6, S7}
Jaccard = |{S1,S2,S3}| / |{S1,S2,S3,S4,S5,S6,S7}| = 3/7 ≈ 0.43

MinHash算法：
┌─────────────────────────────────────────────┐
│  1. 选择k个不同的哈希函数 h1, h2, ..., hk    │
│  2. 对每个哈希函数，计算集合中所有元素的hash值│
│  3. 取最小值作为该哈希函数的MinHash值         │
│  4. 文档的签名 = [min_h1, min_h2, ..., min_hk]│
│                                              │
│  关键性质：                                   │
│  P(minhash(a) == minhash(b)) == Jaccard(a,b) │
│  → 两个集合MinHash值相等的概率等于Jaccard相似度│
└─────────────────────────────────────────────┘

MinHash签名示例（k=3个哈希函数）：
Doc1 shingles: {1, 3, 5, 7, 9}
Doc2 shingles: {1, 3, 5, 6, 8}

h1: hash值分别为 {3,1,4,2,5} → min=1  |  {3,1,4,7,2} → min=1 ✓
h2: hash值分别为 {5,2,1,3,4} → min=1  |  {5,2,1,6,3} → min=1 ✓
h3: hash值分别为 {2,4,3,1,5} → min=1  |  {2,4,3,5,1} → min=1 ✓
MinHash签名：[1,1,1] vs [1,1,1]
签名匹配比例 = 3/3 = 1.0 ≈ Jaccard相似度

LSH (Locality-Sensitive Hashing) 加速：
- 将MinHash签名分为b个band，每个band有r行
- 同一个band内签名完全相同的文档 → 候选对
- 只需检查候选对，无需两两比较所有文档
- 通过调整b和r控制精确率和召回率

SimHash vs MinHash：
┌───────────────┬──────────────┬──────────────┐
│ 特性           │ SimHash      │ MinHash      │
├───────────────┼──────────────┼──────────────┤
│ 适用场景       │ 文本去重     │ 集合相似度   │
│ 指纹长度       │ 固定(64位)   │ 可配置(k个)  │
│ 比较方式       │ 海明距离     │ 签名匹配比例 │
│ 大规模优化     │ 排序比较     │ LSH分桶      │
│ Google使用     │ 网页去重     │ YouTube推荐  │
└───────────────┴──────────────┴──────────────┘
```


## 网页去噪 (Boilerplate Removal)


```
网页去噪：从HTML中分离正文内容和模板噪声

网页组成分析：
┌─────────────────────────────────────────────┐
│  ┌───────────────────────────────────────┐  │
│  │  导航栏 (Navigation)     ← 模板噪声    │  │
│  ├───────────┬───────────────────────────┤  │
│  │  侧边栏   │  正文内容 (Main Content)  │  │
│  │  (Side)   │  ← 这才是有价值的内容     │  │
│  │  ← 噪声   │                           │  │
│  │           │                           │  │
│  ├───────────┴───────────────────────────┤  │
│  │  广告 (Ads)              ← 模板噪声    │  │
│  ├───────────────────────────────────────┤  │
│  │  页脚 (Footer)           ← 模板噪声    │  │
│  └───────────────────────────────────────┘  │
└─────────────────────────────────────────────┘

去噪方法：

1. 基于文本密度的方法
   - 计算每个DOM节点的文本密度 = 文本长度 / 标签数
   - 正文区域的文本密度通常较高
   - 导航/广告区域链接多、文本少

2. 基于DOM结构的方法
   - 正文通常在 <article> 或 <main> 标签中
   - 噪声通常在 <nav>、<aside>、<footer> 中
   - 通过CSS类名特征识别（class="ad", "sidebar"等）

3. 基于模板检测的方法
   - 同一站点多个页面的公共部分 = 模板
   - 通过对比同一站点的多个页面
   - 出现在所有页面中的DOM节点 = 模板

4. 基于机器学习的方法
   - 训练分类器识别正文 vs 噪声
   - 特征：文本密度、链接密度、DOM深度、标签类型
   - 现代方法使用BERT等模型理解内容

代码示例（基于文本密度）：
def compute_text_density(node):
    text_len = len(node.get_text())
    tag_count = len(node.find_all())
    if tag_count == 0:
        return float('inf')
    return text_len / tag_count

def extract_main_content(html):
    soup = BeautifulSoup(html, 'html.parser')
    # 移除已知噪声标签
    for tag in soup.find_all(['nav', 'footer', 'aside']):
        tag.decompose()
    # 找到文本密度最高的div
    candidates = soup.find_all('div')
    best = max(candidates, key=compute_text_density)
    return best.get_text()
```


## Readability 内容提取


```
Readability：Mozilla开发的网页正文提取算法
Firefox的阅读模式使用此算法，也被广泛移植

核心算法思路：
┌─────────────────────────────────────────────┐
│  1. 预处理：移除脚本、样式、隐藏元素          │
│  2. 评分：对每个DOM节点计算"内容可能性"分数   │
│  3. 选择：选择得分最高的节点作为正文          │
│  4. 清理：移除选中节点中的噪声子节点          │
│  5. 补全：补全缺失的段落标签                  │
└─────────────────────────────────────────────┘

评分规则（简化版）：
加分项：
  + 每个 <p> 标签：+3
  + 每个逗号：+1
  + class/id 包含 "article", "content", "main"：+25
  + 文本长度超过25字符：+1
  + 每个段落标签内有文本：+3

减分项：
  - 每个 <a> 标签内的文本占比过高：-5
  - class/id 包含 "comment", "sidebar", "ad"：-25
  - input/form 标签：-3
  - 每5个链接：-1

JavaScript实现（核心）：
function scoreNode(node) {
    let score = 0;
    // 加分
    score += node.getElementsByTagName('p').length * 3;
    score += (node.textContent.match(/,/g) || []).length;
    if (/article|content|main/i.test(node.className + node.id)) {
        score += 25;
    }
    // 减分
    const links = node.getElementsByTagName('a');
    const linkTextLen = Array.from(links)
        .reduce((sum, a) => sum + a.textContent.length, 0);
    const totalTextLen = node.textContent.length;
    if (linkTextLen / totalTextLen > 0.3) score -= 5;
    if (/comment|sidebar|ad/i.test(node.className + node.id)) {
        score -= 25;
    }
    return score;
}

Python实现（使用readability-lxml）：
from readability import Document

response = requests.get(url)
doc = Document(response.text)

content = doc.summary()      # 提取正文HTML
clean_text = doc.get_clean_text()  # 纯文本

其他内容提取工具：

│ 工具          │ 特点                          │

│ Readability  │ 经典算法，广泛使用             │

└──────────────┴──────────────────────────────┘
```


> **Note:** SimHash和MinHash是大规模网页去重的基石算法。Readability和类似工具是网页内容提取的标准方案，被Pocket、Instapaper等应用广泛使用。
