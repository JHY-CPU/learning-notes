# 74_SPOOLing假脱机技术

## 核心概念

- **SPOOLing（Simultaneous Peripheral Operating On-Line）**：外围设备同时联机操作，又称"假脱机"技术。
- 核心思想：用磁盘上的**输入/输出井**模拟脱机输入/输出，将独占设备虚拟为共享设备。
- 最经典应用：**打印机假脱机**（多进程共享一台打印机）。
- 408 考点：SPOOLing 的组成、工作原理、如何将独占设备虚拟为共享设备。

## 原理分析

### SPOOLing 的组成

```
┌──────────────────────────────────────────────────────┐
│                      SPOOLing 系统                    │
│                                                      │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐       │
│  │ 输入进程  │    │ 输出进程  │    │ 井管理   │       │
│  │(预输入)  │    │(缓输出)  │    │ 程序     │       │
│  └────┬─────┘    └────┬─────┘    └────┬─────┘       │
│       │               │               │              │
│  ┌────▼─────┐    ┌────▼─────┐    ┌────▼─────┐       │
│  │ 输入井   │    │ 输出井   │    │ 请求队列  │       │
│  │(磁盘)    │    │(磁盘)    │    │           │       │
│  └──────────┘    └──────────┘    └──────────┘       │
│       ▲               │                              │
└───────┼───────────────┼──────────────────────────────┘
        │               │
   ┌────┴─────┐    ┌────▼─────┐
   │ 输入设备  │    │ 输出设备  │
   │ (键盘等)  │    │ (打印机)  │
   └──────────┘    └──────────┘
```

**三个组成部分**：

1. **输入井和输出井**：磁盘上的两个存储区域。
   - **输入井**：暂存从输入设备读入的数据，等待进程处理。
   - **输出井**：暂存进程的输出数据，等待设备输出。

2. **输入进程和输出进程**：
   - **输入进程（预输入进程）**：模拟脱机输入，将数据从输入设备搬到输入井。
   - **输出进程（缓输出进程）**：模拟脱机输出，将数据从输出井搬到输出设备。

3. **井管理程序**：管理输入/输出井的空间分配和回收。

### 打印机 SPOOLing 示例

**场景**：多个用户进程需要打印，系统只有一台打印机。

**传统方式**：
- 进程 A 打印时独占打印机。
- 进程 B 必须等 A 打印完才能使用。
- 打印机速度远低于 CPU，进程 A 在等待打印完成期间被阻塞。

**SPOOLing 方式**：

```
进程 A: "打印 report.txt" → 数据写入输出井 → A 继续执行（不阻塞！）
进程 B: "打印 data.pdf"   → 数据写入输出井 → B 继续执行
进程 C: "打印 pic.png"   → 数据写入输出井 → C 继续执行

输出进程（守护进程）:
  从输出井按顺序取出 report.txt → 发送给打印机
  从输出井按顺序取出 data.pdf  → 发送给打印机
  从输出井按顺序取出 pic.png  → 发送给打印机
```

**关键点**：
- 进程以为自己"拥有"打印机（通过 SPOOLing 接口），实际上数据先写到磁盘。
- 打印机由 SPOOLing 守护进程统一管理，串行输出。
- **独占设备被虚拟为共享设备**：多个进程可以"同时"提交打印请求。

### SPOOLing 的特点

1. **提高 I/O 速度**：数据先写入磁盘（输出井），进程不必等待慢速设备。
2. **将独占设备虚拟为共享设备**：打印机、绘图仪等独占设备可以被多进程共享。
3. **实现虚拟设备功能**：每个进程以为自己独占设备，实际是 SPOOLing 在中间调度。

### SPOOLing 守护进程

在 Unix/Linux 中，打印守护进程是 `lpd`（Line Printer Daemon）或 `cupsd`（CUPS 打印系统）：

```
用户进程 → lp 命令 → lpd 守护进程 → 打印机驱动 → 打印机
              ↓
          /var/spool/lpd/  (输出井目录)
```

### 守护进程的特点

- 作为**后台进程**运行，不与任何终端关联。
- 通常在系统启动时创建，生命周期与系统相同。
- 负责管理和调度 I/O 设备。

## 直观理解

- **SPOOLing** = 快递驿站。快递员（输出进程）把包裹一个个放到驿站（输出井），你（进程）去驿站存了包裹就走，不用等快递员送。
- 传统方式 = 你必须亲手把包裹交给收件人，期间你一直等着。
- SPOOLing 把"你等快递"变成了"驿站帮你等"，你被解放了。
