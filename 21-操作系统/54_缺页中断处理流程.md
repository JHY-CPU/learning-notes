# 54_缺页中断处理流程

## 核心概念

- **缺页中断（Page Fault）**：访问的页面不在内存中时产生的中断
- **缺页中断处理程序**：OS中负责处理缺页中断的代码
- **请求调页**：缺页时从外存调入所需页面
- **页面置换**：内存满时选择一页换出以腾出空间
- **页面置换算法**：决定换出哪个页面的策略
- **四种经典算法**：
  - **OPT（最佳置换）**：换出未来最久不用的页（理论最优，不可实现）
  - **FIFO（先进先出）**：换出最早进入内存的页
  - **LRU（最近最久未使用）**：换出最近最久未被访问的页
  - **Clock（时钟算法/NRU）**：LRU的近似实现

**408核心必考**：页面置换算法的模拟计算

## 原理分析

**缺页中断处理流程**：

```
1. CPU产生缺页中断（状态位P=0）
2. 保护当前进程现场（保存断点）
3. 分析中断原因 → 缺页中断
4. 查找页面在外存中的地址
5. 检查内存是否有空闲页框
    ├── 有空闲 → 直接调入
    └── 无空闲 → 执行页面置换算法，选择淘汰页
6. 若淘汰页修改位M=1 → 写回外存
7. 将所需页面从外存读入内存
8. 更新页表（状态位置1，填入页框号）
9. 修改快表（若有）
10. 返回被中断的指令，重新执行
```

**Belady异常（FIFO特有）**：

$$
\text{分配更多页框} \Rightarrow \text{缺页次数反而增加！}
$$

$$
\text{反例}：\text{页面引用串 } 1,2,3,4,1,2,5,1,2,3,4,5
$$

3个页框：缺页9次
4个页框：缺页10次（更多页框反而更多缺页！）

只有FIFO有Belady异常，LRU和OPT没有（它们满足"栈性质"）。

**OPT算法（最佳置换）**：

换出未来最长时间不被使用的页面。需要预知未来，不可实现，作为评价标准。

**FIFO算法**：

维护一个队列，新页面入队尾，淘汰队头页面。

**LRU算法**：

换出最近最久未被访问的页面。实现方式：
- 计数器法：每次访问更新时间戳，淘汰时间戳最小的
- 栈法：每次访问将页面移到栈顶，淘汰栈底

**Clock算法（近似LRU）**：

```
1. 所有页框组成循环链表，有一个指针（时钟指针）
2. 访问页面时，硬件将R位置1
3. 需要淘汰时：
    a. 检查指针所指页面的R位
    b. 若R=1 → 置R=0，指针前进
    c. 若R=0 → 淘汰该页
4. 最多扫描一圈就能找到淘汰页
```

**改进的Clock算法**：同时考虑R和M位，优先淘汰(0,0)，其次(0,1)，再次(1,0)，最后(1,1)。

**页面置换算法对比**：

| 算法 | 可实现性 | 性能 | Belady异常 | 硬件支持 |
|------|---------|------|-----------|---------|
| OPT | 不可 | 最优 | 无 | 不需要 |
| FIFO | 可 | 差 | 有 | 不需要 |
| LRU | 可 | 好 | 无 | 需要（时间戳/栈） |
| Clock | 可 | 中等 | 无 | 需要（R位） |

## 直观理解

**食堂座位的类比**：
- 位子满了有人要进来坐（缺页中断）
- **FIFO**：赶走最早来的那个人（不管他还在没在吃）
- **LRU**：赶走最久没动筷子的人（最近最久没吃）
- **OPT**：赶走之后最久不会再吃的人（需要读心术）
- **Clock**：服务员转一圈，碰到正在吃（R=1）就跳过并把标志清零，碰到没在吃（R=0）的就赶走

**Belady异常的类比**：
就像停车位变多了，但排队规则导致后来的车反而挡住了先来的车，大家都要多绕几圈。

## 知识关联

**与页表项的关系（第53节）**：
- 状态位触发缺页中断
- 访问位是Clock算法的依据
- 修改位决定淘汰时是否写回

**与虚拟内存的关系（第52节）**：
- 缺页中断是虚拟内存的核心机制
- 缺页率（缺页次数/总访问次数）是衡量虚拟内存性能的关键指标

**408常见考法**：
- 给出页面引用串和页框数，模拟FIFO/LRU/OPT算法，计算缺页次数
- 问"Belady异常是什么，哪种算法有"
- 问"OPT为什么不能实现"（需要预知未来）
- 比较四种算法的优缺点
- Clock算法的指针移动过程
