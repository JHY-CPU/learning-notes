# 77_磁盘调度平均寻道长度计算

## 核心概念

- **寻道长度**：磁头从当前位置移动到目标磁道所经过的磁道数。
- **平均寻道长度**：所有请求的寻道长度之和除以请求次数。
- 平均寻道长度是衡量磁盘调度算法性能的核心指标。
- 408 高频计算题：给定请求序列和磁头位置，计算各算法的平均寻道长度。

## 原理分析

### 平均寻道长度公式

$$\text{平均寻道长度} = \frac{\sum_{i=1}^{n} |P_{i-1} - P_i|}{n}$$

其中 $P_0$ 为磁头起始位置，$P_i$ 为第 $i$ 次服务的磁道号，$n$ 为请求次数。

### 完整计算示例

**题目**：磁头当前在 100 号磁道，请求队列 `55, 58, 39, 18, 90, 160, 150, 38, 184`。分别计算 FCFS、SSTF、SCAN、C-SCAN 的平均寻道长度。

#### FCFS 计算

调度顺序：100 → 55 → 58 → 39 → 18 → 90 → 160 → 150 → 38 → 184

$$\text{总寻道距离} = |100-55| + |55-58| + |58-39| + |39-18| + |18-90| + |90-160| + |160-150| + |150-38| + |38-184|$$
$$= 45 + 3 + 19 + 21 + 72 + 70 + 10 + 112 + 146 = 498$$

$$\text{平均寻道长度} = 498 / 9 = 55.33$$

#### SSTF 计算

从 100 开始，每次选最近的：

- 100 → 90（10）→ 58（32）→ 55（3）→ 39（16）→ 38（1）→ 18（20）→ 150（132）→ 160（10）→ 184（24）

$$\text{总} = 10 + 32 + 3 + 16 + 1 + 20 + 132 + 10 + 24 = 248$$

$$\text{平均} = 248 / 9 = 27.56$$

#### SCAN 计算（假设向增大方向，到头折返）

100 向增大方向：150, 160, 184。到 199 后折返向减小方向：90, 58, 55, 39, 38, 18。

调度顺序：100 → 150 → 160 → 184 → 199 → 90 → 58 → 55 → 39 → 38 → 18

$$\text{总} = 50 + 10 + 24 + 15 + 109 + 32 + 3 + 16 + 1 + 20 = 280$$

$$\text{平均} = 280 / 9 = 31.11$$

如果用 LOOK（不到 199，只到 184 就折返）：

调度顺序：100 → 150 → 160 → 184 → 90 → 58 → 55 → 39 → 38 → 18

$$\text{总} = 50 + 10 + 24 + 94 + 32 + 3 + 16 + 1 + 20 = 250$$

$$\text{平均} = 250 / 9 = 27.78$$

#### C-SCAN 计算（向增大方向，到头跳回另一端）

100 向增大：150, 160, 184。到 199 后跳到 0，继续增大：18, 38, 39, 55, 58, 90。

$$\text{总} = 50 + 10 + 24 + 15 + 199 + 18 + 20 + 1 + 16 + 3 + 32 = 388$$

$$\text{平均} = 388 / 9 = 43.11$$

如果用 C-LOOK（只跳到 18，跳转距离 = 184 - 18 = 166）：

$$\text{总} = 50 + 10 + 24 + 166 + 20 + 1 + 16 + 3 + 32 = 322$$

$$\text{平均} = 322 / 9 = 35.78$$

### 结果汇总

| 算法 | 总寻道距离 | 平均寻道长度 |
|------|-----------|-------------|
| FCFS | 498 | 55.33 |
| SSTF | 248 | 27.56 |
| SCAN (LOOK) | 250 | 27.78 |
| C-SCAN (C-LOOK) | 322 | 35.78 |

### 考试技巧

1. **先排序再调度**：SSTF 和 SCAN 类算法可以先将请求排序，再根据规则选。
2. **注意方向**：SCAN/C-SCAN 必须明确初始方向（增大还是减小）。
3. **边界处理**：题目通常说明是否需要走到磁道边界。
4. **跳转处理**：C-SCAN 的跳转距离是否计入总寻道距离，题目会有说明。
5. **n 次请求对应 n 次寻道**：不要忘记初始位置到第一个请求的距离。

## 直观理解

平均寻道长度就是"磁头平均每次要跑多远"——跑得越少，效率越高。就像快递员的平均送件距离，距离越短越快。
