# 52_虚拟内存特征与理论依据

## 核心概念

- **虚拟内存（Virtual Memory）**：将内存和外存结合，为进程提供比实际物理内存更大的逻辑地址空间
- **三大特征**：
  1. **多次性**：进程可以被分成多次调入内存
  2. **对换性**：进程可以在内存和外存之间换入换出
  3. **虚拟性**：逻辑地址空间 >> 物理地址空间
- **理论依据**：局部性原理
  - **时间局部性**：刚被访问的指令/数据很快又被访问
  - **空间局部性**：访问某个地址后，很快会访问其附近地址
- **请求分页**：最常见的虚拟内存实现方式
- **请求调页**：访问不在内存的页时，从外存调入
- **页面置换**：内存满时，选择一页换出

**408核心考点**：
- 虚拟内存的三大特征
- 局部性原理（选择题常考）
- 虚拟内存容量的决定因素

## 原理分析

**虚拟内存的容量**：

$$
\text{虚拟内存容量} \leq \text{逻辑地址空间大小}（\text{由CPU位数决定}）
$$

$$
\text{虚拟内存容量} \leq \text{内存容量} + \text{外存容量}
$$

实际容量取两者的较小值。

例：32位CPU，逻辑地址空间4GB，内存2GB，外存100GB
虚拟内存容量 = min(4GB, 2GB+100GB) = 4GB

**传统存储 vs 虚拟存储**：

| 特性 | 传统存储管理 | 虚拟存储管理 |
|------|------------|------------|
| 一次性 | 作业必须全部装入内存 | 可以部分装入 |
| 驻留性 | 作业一直驻留内存 | 可以换入换出 |
| 地址空间 | 受限于物理内存 | 远大于物理内存 |
| 并发度 | 低（内存限制） | 高 |

**局部性原理的验证**：

$$
\text{时间局部性: } \quad \text{循环结构中} \quad \text{同一指令反复执行}
$$

$$
\text{空间局部性: } \quad \text{数组遍历时} \quad \text{相邻元素依次被访问}
$$

程序中90%的时间在执行10%的代码（循环/热点代码），这使得虚拟内存的性能接近全部装入。

**请求分页的硬件支持**：

1. **页表机制扩展**：增加状态位、访问位、修改位、外存地址
2. **缺页中断机构**：访问不在内存的页时产生中断
3. **地址变换机构**：扩展的地址转换逻辑

## 直观理解

**书桌和书架的类比**：
- 书桌 = 内存（容量小但快）
- 书架 = 外存（容量大但慢）
- 你手头只放正在看的几页（请求分页）
- 看完了从书架上换新的（页面置换）
- 你不需要把整本书都放桌上（虚拟内存的多次性）

**虚拟内存就像"无限大的书桌"**：
虽然你的书桌只有1平方米（物理内存），但因为书架就在旁边（外存），你随时可以换书，所以感觉书桌无限大（虚拟内存）。

## 知识关联

**与分页管理的关系（第47节）**：
- 虚拟内存基于分页管理扩展
- 增加了请求调页和页面置换功能

**与页面置换算法的关系（本章后续）**：
- FIFO、LRU、OPT、Clock等算法决定换出哪一页
- 不同算法的性能差异很大

**与缺页中断的关系（第54节）**：
- 请求调页通过缺页中断实现
- 缺页中断是虚拟内存的核心机制

**与外存管理的关系**：
- 换出的页面存放在外存的对换区
- 对换区需要快速读写（连续空间分配）

**408常见考法**：
- 问"虚拟内存的理论依据"（局部性原理）
- 问"虚拟内存的容量由什么决定"（逻辑地址空间和内存+外存）
- 问"传统存储和虚拟存储的根本区别"（一次性、驻留性）
- 问"时间局部性和空间局部性分别举例"

## 代码实现

### 请求分页模拟——验证局部性原理

```c
#include <stdio.h>
#include <stdlib.h>
#include <stdbool.h>

/*
 * 模拟请求分页机制，验证局部性原理
 *
 * 虚拟内存的三大特征：
 * 1. 多次性：进程分多次调入内存
 * 2. 对换性：页面可以在内存和外存之间换入换出
 * 3. 虚拟性：逻辑地址空间远大于物理内存
 *
 * 局部性原理保证了大部分访问都能命中内存，
 * 使得虚拟内存的性能接近全部装入。
 */

#define FRAME_COUNT 4   /* 物理内存只有4个页框 */
#define PAGE_COUNT 16   /* 虚拟地址空间有16页 */

typedef struct {
    int page_num;       /* 存放的页面号，-1表示空闲 */
    bool valid;         /* 有效位（在内存中） */
    int last_access;    /* 最后访问时间（用于LRU） */
} Frame;

Frame frames[FRAME_COUNT];
int page_table[PAGE_COUNT];  /* 页表：页号→页框号，-1表示不在内存 */
int time_tick = 0;
int page_faults = 0;
int total_accesses = 0;

void init() {
    for (int i = 0; i < FRAME_COUNT; i++) {
        frames[i].page_num = -1;
        frames[i].valid = false;
        frames[i].last_access = 0;
    }
    for (int i = 0; i < PAGE_COUNT; i++)
        page_table[i] = -1;
}

/* 请求调页：页面不在内存时从"外存"调入 */
void handle_page_fault(int page) {
    page_faults++;
    printf("  [缺页中断] 页面 %d 不在内存，开始调入...\n", page);

    /* 查找空闲页框 */
    int victim = -1;
    for (int i = 0; i < FRAME_COUNT; i++) {
        if (!frames[i].valid) {
            victim = i;
            break;
        }
    }

    /* 无空闲页框，执行LRU置换 */
    if (victim == -1) {
        int oldest = time_tick;
        for (int i = 0; i < FRAME_COUNT; i++) {
            if (frames[i].last_access < oldest) {
                oldest = frames[i].last_access;
                victim = i;
            }
        }
        int old_page = frames[victim].page_num;
        printf("  [页面置换] 淘汰页面 %d（LRU），换入页面 %d\n",
               old_page, page);
        page_table[old_page] = -1;  /* 旧页面标记为不在内存 */
    }

    /* 调入新页面 */
    frames[victim].page_num = page;
    frames[victim].valid = true;
    frames[victim].last_access = time_tick;
    page_table[page] = victim;
}

/* 访问虚拟地址 */
void access_page(int page) {
    total_accesses++;
    time_tick++;

    if (page_table[page] == -1) {
        handle_page_fault(page);
    } else {
        frames[page_table[page]].last_access = time_tick;
    }
}

void print_status() {
    printf("内存状态: ");
    for (int i = 0; i < FRAME_COUNT; i++) {
        if (frames[i].valid)
            printf("[页%d] ", frames[i].page_num);
        else
            printf("[空] ");
    }
    printf("\n");
}

int main() {
    init();

    printf("=== 虚拟内存请求分页模拟 ===\n");
    printf("虚拟地址空间: %d 页, 物理内存: %d 页框\n\n",
           PAGE_COUNT, FRAME_COUNT);

    /* 模拟具有局部性的访问序列（循环访问4个页面 + 偶尔跳转） */
    int access_seq[] = {
        0, 1, 2, 3, 0, 1, 2, 3,  /* 时间局部性：循环访问 */
        4, 5, 4, 5, 4, 5,         /* 空间局部性：相邻页面 */
        0, 1, 2, 3,               /* 重新访问：时间局部性 */
        8, 9, 8, 9                /* 另一个局部区域 */
    };
    int seq_len = sizeof(access_seq) / sizeof(access_seq[0]);

    for (int i = 0; i < seq_len; i++) {
        printf("访问页面 %2d: ", access_seq[i]);
        access_page(access_seq[i]);
        print_status();
    }

    printf("\n=== 统计结果 ===\n");
    printf("总访问次数: %d\n", total_accesses);
    printf("缺页次数:   %d\n", page_faults);
    printf("缺页率:     %.1f%%\n", 100.0 * page_faults / total_accesses);

    return 0;
}
```

### Shell 脚本查看系统的虚拟内存使用情况

```bash
#!/bin/bash
# 在 Linux 中观察虚拟内存的实际运作

echo "=== 进程虚拟地址空间（/proc/self/maps）==="
cat /proc/self/maps | head -10

echo ""
echo "=== 系统虚拟内存统计 ==="
vmstat 1 3

echo ""
echo "=== 当前进程的内存映射 ==="
cat /proc/$$/status | grep -E "VmSize|VmRSS|VmSwap"
# VmSize  = 虚拟地址空间大小（含未分配物理内存的部分）
# VmRSS   = 实际驻留在物理内存中的大小（Resident Set Size）
# VmSwap  = 被换出到外存的大小
```
