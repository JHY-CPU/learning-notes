# 28_软件实现方法Peterson

## 核心概念

- **Peterson算法**：1981年由Gary Peterson提出的经典软件互斥算法，适用于两个进程
- **单标志法**：用一个公共变量turn表示谁有权进入临界区
- **双标志法**：用数组flag[i]表示进程i是否想进入临界区
- **双标志先检查法**：先检查对方flag，再设自己的flag（有问题）
- **双标志后检查法**：先设自己的flag，再检查对方flag（有饥饿）
- **Peterson算法**：结合turn和flag，既保证互斥又避免饥饿

**考试重点**：
- 必须理解每种软件方法的缺陷（选择题常考）
- Peterson算法只适用于**两个进程**
- 软件方法都有"忙等待"问题，不满足"让权等待"

## 原理分析

**单标志法（turn变量）**：

```
// P0进程                          // P1进程
while(turn != 0);                 while(turn != 1);
  临界区;                           临界区;
turn = 1;                         turn = 0;
```

问题：必须交替进入，P0做完后turn=1，P1不做则P0无法再次进入。**违反"空闲让进"**。

**双标志先检查法（flag数组）**：

```
// Pi进程
while(flag[j]);   // 先检查对方
flag[i] = true;   // 再设置自己
  临界区;
flag[i] = false;
```

问题：P0和P1可能同时通过检查（都在对方设flag之前检查），**违反"忙则等待"**。

**双标志后检查法**：

```
// Pi进程
flag[i] = true;       // 先设置自己
while(flag[j]);       // 再检查对方
  临界区;
flag[i] = false;
```

问题：双方同时设flag=true，然后互相等待。**可能导致饥饿（死等）**。

**Peterson算法（最终方案）**：

```
// Pi进程 (i和j是两个进程号，j = 1-i)
flag[i] = true;                    // 我想进
turn = j;                          // 但我让给你机会
while(flag[j] && turn == j);       // 你想进且轮到你，我就等
  临界区;
flag[i] = false;                   // 出来后设置不想进
```

$$
\text{互斥保证：} \quad flag[i] \land flag[j] \Rightarrow turn \text{ 只能取一个值} \Rightarrow \text{只有一个能进入}
$$

**Peterson算法正确性分析**：
1. **互斥**：若Pi和Pj同时想进入，turn只能是i或j之一，故只有一个能通过while
2. **空闲让进**：若Pj不想进入，flag[j]=false，Pi直接通过
3. **有限等待**：Pi离开后将flag[i]设为false，Pj不会被阻塞
4. **不满足让权等待**：while循环是忙等待，消耗CPU

## 直观理解

**独木桥的类比**：

单标志法就像"过桥令牌"——你过完桥把令牌扔到对岸，下次只能对方用。如果对方不过桥，你也没令牌过不了。

Peterson算法就像两个人在桥头相遇：
- flag[i]=true：我表示"我要过桥"
- turn=j：但我把"优先权"让给你
- 如果两人都举手说要过，就看谁把优先权让给了谁，被让的人先过

## 知识关联

**软件方法的演进**：

```
单标志 → 双标志先检查 → 双标志后检查 → Peterson
  ↓           ↓              ↓            ↓
不互斥      不互斥          饥饿       正确但忙等
```

**与硬件方法的对比（第29节）**：
- 软件方法：逻辑清晰但效率低，仅适用于教学
- 硬件方法（关中断、TSL、Swap）：实际OS中更常用

**408常见考法**：
- 给出一段代码，判断属于哪种方法，有何缺陷
- 问Peterson算法中 `turn = j` 的作用（让权，避免饥饿）
- 判断是否满足四个原则中的某一个
- Peterson算法不满足"让权等待"是常考的判断题
