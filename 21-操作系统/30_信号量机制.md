# 30_信号量机制

## 核心概念

- **信号量（Semaphore）**：由Dijkstra提出，是一个特殊的整型变量，只能通过P/V操作访问
- **P操作（wait/Proberen测试）**：信号量减1，若小于0则阻塞等待
- **V操作（signal/Verhogen增加）**：信号量加1，若小于等于0则唤醒一个等待进程
- **整型信号量**：初版实现，有忙等待问题
- **记录型信号量**：改进版，引入等待队列，满足让权等待
- **互斥信号量**：初值为1，用于互斥
- **同步信号量**：初值为0或N，用于同步
- **计数信号量**：初值为资源数量，用于资源计数

**408核心考点**：
- P/V操作的含义和执行过程
- 信号量初值的设定
- P操作可能导致阻塞，V操作可能导致唤醒

## 原理分析

**记录型信号量的数据结构**：

```c
typedef struct {
    int value;               // 资源数量
    struct PCB *queue;       // 等待队列
} Semaphore;
```

**P操作（wait）**：

```c
void P(Semaphore *S) {
    S->value--;
    if (S->value < 0) {
        // 没有资源了，阻塞自己
        add this process to S->queue;
        block();    // 让权等待：主动放弃CPU
    }
}
```

**V操作（signal）**：

```c
void V(Semaphore *S) {
    S->value++;
    if (S->value <= 0) {
        // 有人在等待，唤醒一个
        remove a process P from S->queue;
        wakeup(P);
    }
}
```

**信号量值的含义**：

$$
S.value \begin{cases}
> 0 & \text{：可用资源数量} \\
= 0 & \text{：无资源也无人等待} \\
< 0 & \text{：绝对值 = 等待进程数}
\end{cases}
$$

**整型信号量 vs 记录型信号量**：

| 特性 | 整型信号量 | 记录型信号量 |
|------|-----------|-------------|
| 等待方式 | 忙等待（while循环） | 阻塞（block） |
| 让权等待 | 不满足 | 满足 |
| 等待队列 | 无 | 有 |
| 实际使用 | 教学 | 真实系统 |

**信号量的分类与用途**：

```
互斥信号量：S初值=1
  P(S) → 进入临界区
  临界区
  V(S) → 离开临界区

同步信号量：S初值=0
  操作A完成后 V(S)    → 释放信号
  操作B开始前 P(S)    → 等待信号

资源信号量：S初值=N（资源数）
  使用资源前 P(S)     → 申请一个资源
  使用资源后 V(S)     → 归还一个资源
```

## 直观理解

**停车场的类比**：
- 信号量 value = 停车场剩余车位数
- P操作（开车进场）：剩余车位-1，如果没有车位了就排队等
- V操作（开车出场）：剩余车位+1，如果有车在排队就叫它进来

**电影院的类比（互斥）**：
- 信号量 mutex = 1（只有一把钥匙）
- P操作：拿走钥匙（value变0），进入放映厅
- 如果钥匙已被拿走（value=0），你在门口等
- V操作：归还钥匙，门口的人可以拿了进去

## 知识关联

**与同步互斥的关系（第27节）**：
- 信号量是实现互斥和同步的主要工具
- 互斥信号量保证临界区互斥
- 同步信号量保证执行顺序

**与PV操作模板的关系（第31节）**：
- 下一节将详细讨论PV操作的编程模板
- 互斥和同步信号量必须分开使用

**与经典同步问题的关系（第32-34节）**：
- 哲学家进餐问题：互斥+资源竞争
- 读者-写者问题：互斥+计数
- 生产者-消费者问题：互斥+同步

**408常见考法**：
- 给出PV操作序列，判断信号量的含义（互斥/同步）
- 分析信号量值的变化过程
- 问"P操作是否一定会阻塞"（不一定，value>0时不阻塞）
- V操作是否有阻塞可能（不会，V只会唤醒不会阻塞）
