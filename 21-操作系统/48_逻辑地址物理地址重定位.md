# 48_逻辑地址物理地址重定位

## 核心概念

- **逻辑地址（Logical Address / Virtual Address）**：CPU生成的地址，程序视角的地址
- **物理地址（Physical Address）**：内存条上实际的地址
- **地址重定位（Relocation）**：将逻辑地址转换为物理地址的过程
- **MMU（Memory Management Unit）**：CPU中负责地址转换的硬件单元
- **基址寄存器（Base Register）**：存放进程起始物理地址
- **界限寄存器（Limit Register）**：存放进程地址空间长度

**408核心考点**：
- 逻辑地址和物理地址的对应关系
- 不同内存管理方式的地址转换公式
- 越界访问的检测

## 原理分析

**连续分配的地址转换**：

$$
\text{物理地址} = \text{逻辑地址} + \text{基址寄存器值}
$$

越界检查：若 逻辑地址 ≥ 界限寄存器值，产生越界中断

**分页管理的地址转换**：

$$
\text{页号} P = \lfloor \frac{\text{逻辑地址}}{\text{页面大小}} \rfloor
$$

$$
\text{偏移量} W = \text{逻辑地址} \mod \text{页面大小}
$$

$$
\text{物理地址} = \text{页表}[P] \times \text{页面大小} + W
$$

**分段管理的地址转换**：

$$
\text{段号} S = \text{逻辑地址高若干位}
$$

$$
\text{段内偏移} D = \text{逻辑地址低若干位}
$$

检查：若 $D \geq \text{段表}[S].\text{段长}$，产生越界中断

$$
\text{物理地址} = \text{段表}[S].\text{基址} + D
$$

**段页式管理的地址转换**：

$$
\text{逻辑地址} \xrightarrow{\text{拆分}} (S, P, W)
$$

$$
S \xrightarrow{\text{查段表}} \text{页表起始地址}
$$

$$
P \xrightarrow{\text{查页表}} \text{页框号} F
$$

$$
\text{物理地址} = F \times \text{页面大小} + W
$$

**不同管理方式的地址转换对比**：

| 管理方式 | 逻辑地址结构 | 转换公式 | 越界检查 |
|---------|------------|---------|---------|
| 连续分配 | 纯偏移 | 逻辑+基址 | 偏移<界限 |
| 分页 | 页号+偏移 | 页表查页框号 | 页号<页表长度 |
| 分段 | 段号+段内偏移 | 段表查基址+偏移 | 偏移<段长 |
| 段页式 | 段号+页号+偏移 | 段表→页表→页框 | 双重检查 |

**地址空间的连续性**：

| 管理方式 | 逻辑地址连续 | 物理地址连续 |
|---------|------------|------------|
| 连续分配 | 连续 | 连续 |
| 分页 | 连续 | 不连续 |
| 分段 | 连续（段内） | 不连续（段间） |

## 直观理解

**快递地址的类比**：
- 逻辑地址就像"你写的收货地址"——XX小区X栋X室
- 物理地址就像"快递员GPS坐标"——经纬度
- 重定位就是快递员查地图把"文字地址"转换为"GPS坐标"
- MMU就是快递员的大脑/导航仪

**分页地址就像"旅馆房间号"**：
你记的是"第3层第5间"（逻辑地址：页号+偏移），但旅馆实际的物理位置可能在"东楼208"（物理地址：页框号+偏移）。层内房间号不变，只是"层"对应到了不同"楼"。

## 知识关联

**与各种内存管理方式的关系**：
- 连续分配（第45节）：基址+偏移
- 分页（第47节）：页表查表
- 分段（第50节）：段表查表
- 段页式（第51节）：两次查表

**与TLB的关系（第49节）**：
- 每次地址转换都需要查页表/段表
- TLB作为高速缓存加速查表过程
- TLB命中时无需访问内存中的页表

**408常见考法**：
- 给出逻辑地址，计算物理地址（分页/分段/段页式）
- 问"地址转换由谁完成"（MMU硬件，不是软件）
- 问"逻辑地址和物理地址哪个对程序员透明"（物理地址透明，程序员只用逻辑地址）
- 越界访问如何检测

## 代码实现

### 分页地址转换模拟

```c
#include <stdio.h>
#include <stdint.h>

/*
 * 分页管理中的地址转换过程模拟
 *
 * 给定：
 * - 页面大小 page_size
 * - 页表 page_table[page_num] → frame_num
 * - 逻辑地址 logic_addr
 *
 * 计算：
 * - 页号 P = logic_addr / page_size
 * - 偏移 W = logic_addr % page_size
 * - 物理地址 = page_table[P] * page_size + W
 */

#define PAGE_SIZE   4096  /* 4KB */
#define PAGE_BITS   12    /* log2(4096) = 12 */
#define PAGE_COUNT  16    /* 页表有16项 */

/* 页表：页号 → 页框号 */
uint32_t page_table[PAGE_COUNT];

void init_page_table() {
    /* 模拟页表，页面映射到随机的物理页框 */
    uint32_t frames[] = {5, 8, 3, 12, 0, 7, 15, 2, 9, 11, 6, 14, 1, 10, 4, 13};
    for (int i = 0; i < PAGE_COUNT; i++)
        page_table[i] = frames[i];
}

/* 分页地址转换 */
void translate(uint32_t logic_addr) {
    uint32_t page_num  = logic_addr / PAGE_SIZE;       /* 或 logic_addr >> PAGE_BITS */
    uint32_t offset    = logic_addr % PAGE_SIZE;       /* 或 logic_addr & (PAGE_SIZE-1) */
    uint32_t frame_num = page_table[page_num];
    uint32_t phys_addr = frame_num * PAGE_SIZE + offset;

    printf("逻辑地址 0x%05X → 页号=%2d, 偏移=0x%03X → "
           "页框号=%2d → 物理地址 0x%05X\n",
           logic_addr, page_num, offset, frame_num, phys_addr);
}

/* 基址+界限方式的地址转换（连续分配） */
void translate_contiguous(uint32_t logic_addr, uint32_t base, uint32_t limit) {
    if (logic_addr >= limit) {
        printf("逻辑地址 0x%05X: 越界！(limit=0x%05X)\n", logic_addr, limit);
        return;
    }
    uint32_t phys_addr = logic_addr + base;
    printf("逻辑地址 0x%05X + 基址 0x%05X = 物理地址 0x%05X\n",
           logic_addr, base, phys_addr);
}

/* 分段地址转换 */
void translate_segment(uint16_t seg_num, uint16_t offset,
                       uint32_t seg_bases[], uint16_t seg_limits[]) {
    if (offset >= seg_limits[seg_num]) {
        printf("段号 %d 偏移 0x%03X: 段越界！(段长=0x%03X)\n",
               seg_num, offset, seg_limits[seg_num]);
        return;
    }
    uint32_t phys_addr = seg_bases[seg_num] + offset;
    printf("段号 %d 偏移 0x%03X → 基址 0x%05X + 偏移 = 物理地址 0x%05X\n",
           seg_num, offset, seg_bases[seg_num], phys_addr);
}

int main() {
    init_page_table();

    printf("=== 分页地址转换 ===\n");
    printf("页面大小: %d 字节 (2^%d)\n\n", PAGE_SIZE, PAGE_BITS);

    /* 打印页表 */
    printf("页表:\n");
    for (int i = 0; i < PAGE_COUNT; i++)
        printf("  页号 %2d → 页框号 %2d\n", i, page_table[i]);
    printf("\n");

    /* 测试地址转换 */
    uint32_t test_addrs[] = {0x12345, 0x00042, 0xAB012, 0x7F000};
    for (int i = 0; i < 4; i++)
        translate(test_addrs[i]);

    printf("\n=== 连续分配地址转换（基址+界限）===\n");
    translate_contiguous(0x0100, 0x5000, 0x2000);  /* 正常 */
    translate_contiguous(0x2500, 0x5000, 0x2000);  /* 越界 */

    printf("\n=== 分段地址转换 ===\n");
    uint32_t seg_bases[]  = {0x1000, 0x5000, 0x8000};
    uint16_t seg_limits[] = {0x1000, 0x2000, 0x0800};
    translate_segment(0, 0x0500, seg_bases, seg_limits);  /* 正常 */
    translate_segment(1, 0x3000, seg_bases, seg_limits);  /* 越界 */

    return 0;
}
```

### Shell 中观察进程的虚拟地址空间

```bash
#!/bin/bash
# 通过 /proc 文件系统观察地址空间的实际布局

echo "=== 进程内存布局 ==="
cat /proc/self/maps

echo ""
echo "=== 各段分析 ==="
# text 段（代码）
cat /proc/self/maps | grep "r-xp" | head -1
# data 段（数据）
cat /proc/self/maps | grep "rw-p" | head -1

echo ""
echo "=== 虚拟地址 vs 物理地址 ==="
# 通过 /proc/self/pagemap 可以查看虚拟页到物理页框的映射
# （需要 root 权限）
echo "pagemap 文件存在: $(ls -l /proc/self/pagemap 2>/dev/null && echo '是' || echo '否')"

echo ""
echo "=== 页大小 ==="
getconf PAGE_SIZE
# 输出 4096，即 4KB，对应偏移占 12 位
```
