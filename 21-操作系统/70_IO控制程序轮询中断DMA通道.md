# 70_IO控制程序轮询中断DMA通道

## 核心概念

- **I/O 控制方式**：CPU 如何控制 I/O 设备完成数据传送。
- 四种方式：**程序直接控制（轮询）**、**中断驱动**、**DMA（直接存储器访问）**、**通道控制**。
- CPU 介入程度从高到低：轮询 > 中断 > DMA > 通道。
- 408 高频考点：四种方式的对比、工作流程、CPU 干预次数。

## 原理分析

### 1. 程序直接控制方式（Programmed I/O / 轮询）

**工作流程**：
1. CPU 向设备控制器发出 I/O 命令。
2. CPU **反复查询**设备状态寄存器（忙等 / busy waiting）。
3. 设备就绪后，CPU 从数据寄存器读取/写入一个字节。
4. 重复步骤 2-3 直到全部数据传送完毕。

```
CPU:  [发命令] → [查状态:忙?] → [查状态:忙?] → ... → [读数据] → [查状态:忙?] → ...
设备:           [工作中...]                                   [传完1字节]
```

**CPU 干预次数**：每个字都要查询状态多次 + 读写 1 次，**极度频繁**。

**优点**：实现简单，不需要额外硬件。
**缺点**：
- CPU 利用率极低（绝大部分时间在忙等）。
- 无法及时响应多个设备。

**适用场景**：嵌入式简单系统、非常低速的设备。

### 2. 中断驱动方式（Interrupt-Driven I/O）

**工作流程**：
1. CPU 向设备控制器发出 I/O 命令。
2. CPU **立即返回**，继续执行其他任务（不必忙等）。
3. 设备完成一个字节的传送后，向 CPU 发送**中断请求**。
4. CPU 响应中断，执行中断处理程序：读取/写入一个字节，然后返回用户程序。
5. 重复步骤 3-4 直到全部数据传送完毕。

```
CPU:  [发命令] → [执行用户程序] → [中断:读1字节] → [执行用户程序] → [中断:读1字节] → ...
设备:           [工作中...]        [完成1字节]                   [完成1字节]
```

**CPU 干预次数**：每个字节传送需要 1 次中断处理。

**优点**：
- CPU 与设备可以并行工作。
- 每次只在数据就绪时才介入，避免忙等。

**缺点**：
- 每个字节都要中断一次，数据量大时中断频繁，开销大。
- 每次中断涉及：保存现场 → 执行中断处理 → 恢复现场，代价高。

**适用场景**：字符设备（键盘、鼠标）、低速设备。

### 3. DMA 方式（Direct Memory Access）

**核心思想**：引入 DMA 控制器，直接在设备和内存之间传送数据，**无需 CPU 逐字节干预**。

**DMA 控制器的寄存器**：
- **MAR（内存地址寄存器）**：目标/源内存地址。
- **DC（数据计数器）**：要传送的字节数。
- **DR（数据寄存器）**：暂存一个字的数据。
- **CR（命令/状态寄存器）**：控制命令和设备状态。

**工作流程**：
1. CPU 向 DMA 控制器设置参数：内存起始地址（MAR）、传送字节数（DC）、传送方向。
2. CPU 返回执行其他任务。
3. DMA 控制器**直接从设备读取数据，写入内存**（或反向），无需 CPU 干预。
4. 每传完一个字，MAR + 1，DC - 1。
5. 全部传送完毕后，DMA 控制器向 CPU 发送**一次中断**通知。

```
CPU:  [设置DMA参数] → [执行用户程序] → [中断:传送完毕]
DMA:                  [字节1:设备→内存] [字节2:设备→内存] ... [字节n:完成→中断]
设备:                  [源源不断输出数据...]
```

**CPU 干预次数**：整个数据块传送只需 **2 次** CPU 干预（设置参数 + 接收中断）。

**DMA 与中断的区别**：
- **中断驱动**：每个字节/字触发一次中断，CPU 每次处理一个单位的数据。
- **DMA**：连续传送整个数据块，只在块传送完毕时中断一次。

**传输单位**：DMA 传输的是**连续的数据块**（通常按字节或字传送，但不需要 CPU 逐个处理）。

**适用场景**：块设备（磁盘、SSD）、高速数据传输。

### 4. 通道控制方式（Channel I/O）

**核心思想**：通道是比 DMA 更高级的 I/O 处理器，有自己的**通道程序**，能执行一系列 I/O 指令。

**通道程序**：一条条通道指令组成，每条指令包含：操作码、内存地址、计数、下一条指令地址。

```
通道程序示例（从磁盘读取数据到内存）：
  [READ, 内存地址=0x1000, 长度=256字节, 下一条=@+1]
  [READ, 内存地址=0x1100, 长度=256字节, 下一条=@+1]
  [READ, 内存地址=0x1200, 长度=256字节, 下一条=END]
  [中断]
```

**工作流程**：
1. CPU 向通道发出**通道程序的起始地址**（只需一条指令）。
2. 通道**独立执行**通道程序，控制设备完成整个 I/O 任务。
3. 通道执行完毕后向 CPU 发送中断。

**CPU 干预次数**：整个 I/O 任务只需 **1 次** CPU 干预（发出通道命令）。

**通道类型**：
- **字节多路通道**：连接多个低速设备，以字节为单位轮流服务。
- **数组选择通道**：连接一个高速设备，独占通道直到传输完成。
- **数组多路通道**：连接多个高速设备，以块为单位轮流服务。

### 四种方式对比

| 特性 | 轮询 | 中断 | DMA | 通道 |
|------|------|------|-----|------|
| 数据传送单位 | 字/字节 | 字/字节 | 数据块 | 一组块（通道程序） |
| CPU 干预频率 | 每字查状态多次 | 每字节1次中断 | 整块2次 | 整个任务1次 |
| CPU 利用率 | 极低 | 较低 | 高 | 最高 |
| 并行性 | 无 | 有限 | 好 | 最好 |
| 实现复杂度 | 最简单 | 简单 | 中等 | 复杂 |
| 典型设备 | 极低速 | 键盘鼠标 | 磁盘 | 大型机I/O系统 |
| 硬件需求 | 无 | 中断控制器 | DMA控制器 | I/O通道处理器 |

## 直观理解

- **轮询** = 你每隔5分钟打电话问快递到了没，期间啥也干不了。
- **中断** = 快递到了给你发短信，你收到短信才去取。
- **DMA** = 请了个搬运工，搬运工自己把一车货从仓库搬到你家门口，搬完了告诉你一声。
- **通道** = 请了个项目经理，你把需求说一遍，项目经理带着搬运工团队把所有事搞定，最后给你汇报。
