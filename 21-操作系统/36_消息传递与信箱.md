# 36_消息传递与信箱

## 核心概念

- **消息传递（Message Passing）**：进程间通过发送和接收消息通信
- **信箱（Mailbox）**：存放消息的缓冲区，有容量限制
- **直接通信**：send/receive直接指定对方进程标识
- **间接通信**：通过信箱（邮箱）间接传递消息
- **send(P, message)**：向进程P发送消息
- **receive(Q, message)**：从进程Q接收消息
- **阻塞send/非阻塞send**：发送方是否等待接收方接收
- **阻塞receive/非阻塞receive**：接收方是否等待消息到达

**408核心考点**：
- 用消息传递实现同步和互斥
- 消息传递 vs 信号量的区别
- 信箱满/空的处理策略

## 原理分析

**直接通信方式**：

```
send(P, message);      // 向进程P发送消息
receive(Q, message);   // 从进程Q接收消息
```

特点：
- 通信链路自动建立，一一对应
- 每对进程之间只有一条链路

**间接通信方式（信箱）**：

```
// 创建信箱
Mailbox MB = create_mailbox();

// 发送和接收
send(MB, message);         // 向信箱MB发消息
receive(MB, message);      // 从信箱MB收消息
```

特点：
- 多个进程可以共享一个信箱
- 一条消息可以被多个进程接收（广播）

**阻塞与非阻塞的组合**：

| 发送方 | 接收方 | 特点 |
|--------|--------|------|
| 阻塞 | 阻塞 | 双方会合（rendezvous） |
| 阻塞 | 非阻塞 | 发送方等接收方就绪 |
| 非阻塞 | 阻塞 | 接收方等消息到达 |
| 非阻塞 | 非阻塞 | 最灵活但同步最弱 |

**用消息传递实现互斥**：

```
// 信箱M中预放一条空消息
Semaphore M 初始有1条空消息;

进程Pi:
    receive(M, message);     // 取走消息（相当于P操作）
    临界区;
    send(M, message);        // 放回消息（相当于V操作）
```

**用消息传递实现同步（A先B后）**：

```
进程A:
    执行操作A;
    send(B, message);        // 向B发消息

进程B:
    receive(A, message);     // 等A的消息
    执行操作B;
```

## 直观理解

**信箱就像快递柜**：
- 直接通信：快递员直接把包裹递到你手上
- 间接通信：快递员放进快递柜（信箱），你从柜子里取
- 阻塞send：快递员把包裹放柜子里就走（非阻塞）或者必须等你取走才走（阻塞）
- 信箱满：柜子满了，快递员等你取走一些再放
- 信箱空：柜子空了，你等快递员放包裹

**与PV操作的区别类比**：
- PV操作就像"计数器"——只管数目，不管内容
- 消息传递就像"信封"——可以携带数据

## 知识关联

**与PV操作的对比（第30节）**：

| 特性 | PV操作 | 消息传递 |
|------|--------|----------|
| 传递内容 | 仅信号（计数） | 可携带数据 |
| 适用范围 | 同一计算机 | 可跨网络 |
| 通信模式 | 共享变量 | 消息传递 |
| 耦合度 | 紧耦合 | 松耦合 |

**与Socket/RPC的联系**：
- 网络通信中的Socket就是消息传递的实际应用
- RPC（远程过程调用）基于消息传递
- 微服务之间的通信也是消息传递思想

- 用消息传递实现生产者-消费者问题
- 区分直接通信和间接通信
- 问阻塞/非阻塞send/receive的组合效果
- 比较消息传递与信号量的优劣
