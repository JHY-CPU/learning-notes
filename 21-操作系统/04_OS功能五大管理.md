# 04_OS功能五大管理

## 核心概念

- **五大管理功能**：处理机管理、存储器管理、设备管理、文件管理、用户接口
- **处理机管理**：进程的创建/撤销、调度（进程调度/作业调度）、进程同步与通信
- **存储器管理**：内存分配与回收、地址映射（逻辑→物理）、内存保护、内存扩充（虚拟内存）
- **设备管理**：缓冲管理、设备分配与回收、设备驱动、虚拟设备
- **文件管理**：文件存储空间管理、目录管理、文件读写与保护
- **用户接口**：命令接口（联机/脱机）、程序接口（系统调用）、图形接口

## 原理分析

### 处理机管理的层次

```
作业调度（高级调度）
    ↓
内存调度（中级调度）
    ↓
进程调度（低级调度）——最基本的调度，必须配置
```

- 作业调度：从外存调入内存
- 进程调度：从就绪队列选一个进程占用CPU
- 中级调度：内存与外存之间对换，缓解内存紧张

### 存储器管理的核心任务

1. **内存分配**：连续分配（固定/动态分区）、非连续分配（分页/分段/段页式）
2. **地址转换**：逻辑地址 → 物理地址（通过MMU硬件）
3. **内存保护**：
   - 界限寄存器法：判断地址是否越界
   - 存储键保护法：每个存储块有保护键
4. **内存扩充**：请求调页、页面置换（虚拟存储技术）

### 设备管理的I/O控制方式

1. **程序直接控制（轮询）**：CPU不断查询设备状态，CPU利用率极低
2. **中断驱动方式**：设备完成后发中断通知CPU
3. **DMA方式**：DMA控制器直接在内存与设备间传送数据，仅在传送开始和结束时需要CPU干预
4. **通道方式**：通道是专门的I/O处理机，可执行通道程序

$$\text{DMA每次传送单位} = \text{一个连续的数据块}$$

## 直观理解

把OS比作一个**大型超市的管理团队**：

- **处理机管理**=人力资源部：决定谁（进程）干什么工作、什么时候换班（调度）
- **存储器管理**=仓库管理：给每个部门分配货架空间，确保不混用（内存保护），空间不够时借用临时仓库（虚拟内存）
- **设备管理**=设备维护组：管理推车、扫码器等设备的借用和归还
- **文件管理**=档案室：文件怎么分类存放、谁能查阅哪些档案
- **用户接口**=前台接待：提供窗口服务（命令接口）、自助机（GUI）、合作商户API（系统调用）

## 知识关联

- **处理机管理**=第2章进程管理 + 第3章CPU调度
- **存储器管理**=第4章内存管理 + 第6章虚拟存储
- **设备管理**=第5章I/O管理
- **文件管理**=第7章文件管理
- **用户接口**贯穿全书，系统调用是核心线索
- **408考法**：
  - 选择题：判断某个功能属于哪种管理
  - 简答题：说明DMA与中断方式的区别
  - 常见陷阱：设备管理**不包括**设备的制造；文件管理**包括**目录管理；系统调用**不属于**命令接口
