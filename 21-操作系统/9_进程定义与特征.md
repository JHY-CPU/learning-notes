# 09_进程定义与特征

## 核心概念

- **进程（Process）**：是程序的**一次执行过程**，是系统进行**资源分配和调度的基本单位**
- **程序**：静态的，存放在磁盘上的指令集合
- **进程**：动态的，是程序在某数据集上的运行过程
- **进程 vs 程序**：
  - 程序是**静态**文本，进程是**动态**执行
  - 一个程序可对应**多个**进程（多次运行）
  - 一个进程可执行**多个**程序（exec调用）
- **进程的特征**（5个，常考）：
  1. **动态性**（最基本）：进程有创建、执行、消亡的生命周期
  2. **并发性**：多个进程可同时存在于内存中并发执行
  3. **独立性**：进程是独立运行、独立获得资源的基本单位
  4. **异步性**：各进程按各自独立的、不可预知的速度推进
  5. **结构性**：每个进程由程序段、数据段、PCB三部分组成

## 原理分析

### 进程的组成

```
进程 = PCB（进程控制块） + 程序段 + 数据段
```

- **PCB**：进程存在的唯一标志，OS通过PCB感知和管理进程
- **程序段**：进程执行的代码（可被多个进程共享）
- **数据段**：进程执行所需的数据

### 进程与程序的数学关系

$$\text{进程} = f(\text{程序}, \text{数据}, \text{PCB})$$

同一程序运行在不同数据集上 = 不同进程（如多个Word进程编辑不同文档）。

### 进程是资源分配的基本单位

- 每个进程有独立的**地址空间**
- 每个进程有独立的**打开文件表**
- 每个进程有独立的**信号处理方式**

注意：进程是**资源分配**的基本单位，但不一定是**CPU调度**的基本单位（线程才是）。

## 直观理解

- **程序** = 菜谱（静态的文字说明）
- **进程** = 按照菜谱做菜的过程（动态的执行过程）
- 同一本菜谱可以被不同厨师同时使用 → 一个程序对应多个进程
- 做菜的过程需要灶台（CPU）、食材（数据）、操作台（内存） → 进程需要分配资源
- **动态性**是最本质的特征：菜谱本身不等于做菜，只有执行起来才叫"做菜"

## 代码实现

### 使用 fork() 创建进程——验证进程的动态性

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <sys/wait.h>

/*
 * 同一个程序（fork_demo）可以对应多个进程。
 * fork() 调用一次，返回两次：
 *   - 父进程中返回子进程 PID（>0）
 *   - 子进程中返回 0
 * 这体现了进程的"动态性"——程序只是静态代码，
 * 只有被 OS 创建 PCB 并调度执行后才成为进程。
 */
int main() {
    printf("父进程 PID=%d，开始执行\n", getpid());

    pid_t pid = fork();  // 创建子进程

    if (pid < 0) {
        perror("fork failed");
        exit(1);
    } else if (pid == 0) {
        /* 子进程：PID 不同，但执行同一份代码 */
        printf("子进程 PID=%d，父进程 PPID=%d\n", getpid(), getppid());
        printf("子进程：我是同一程序的另一个执行实例\n");

        /* 使用 exec 族函数可以执行不同的程序，
           体现"一个进程可执行多个程序" */
        // execl("/bin/ls", "ls", "-l", NULL);

        exit(0);
    } else {
        /* 父进程 */
        printf("父进程：创建了子进程 PID=%d\n", pid);
        wait(NULL);  // 等待子进程结束
        printf("父进程：子进程已终止\n");
    }

    return 0;
}
```

### 进程的组成部分——通过代码观察 PCB、程序段、数据段

```c
#include <stdio.h>
#include <unistd.h>

/*
 * 进程 = PCB + 程序段 + 数据段
 *
 * - 程序段：即 main 函数等代码（text 段），可被多个进程共享
 * - 数据段：全局变量、静态变量，每个进程独立一份
 * - PCB：由 OS 内核维护，用户通过 /proc 文件系统间接观察
 *
 * 下面代码验证：父子进程各有独立的数据段（地址相同但内容独立）
 */
int global_var = 100;  // 数据段中的全局变量

int main() {
    int local_var = 200;  // 栈上的局部变量

    printf("进程 PID=%d\n", getpid());
    printf("全局变量地址=%p, 值=%d\n", (void*)&global_var, global_var);
    printf("局部变量地址=%p, 值=%d\n", (void*)&local_var, local_var);

    pid_t pid = fork();
    if (pid == 0) {
        /* 子进程修改变量——不影响父进程 */
        global_var = 999;
        local_var = 888;
        printf("子进程：global_var=%d, local_var=%d\n", global_var, local_var);
    } else {
        wait(NULL);
        /* 父进程的变量未被子进程修改 */
        printf("父进程：global_var=%d, local_var=%d\n", global_var, local_var);
        printf("结论：父子进程拥有独立的地址空间（数据段独立）\n");
    }

    return 0;
}
```

### 在 Linux 中查看进程的 PCB 信息

```bash
#!/bin/bash
# Linux 通过 /proc 文件系统暴露 PCB 中的信息
# 每个进程在 /proc/<PID>/ 目录下有详细的状态信息

# 查看当前 shell 进程的信息
PID=$$
echo "当前进程 PID: $PID"

# 查看进程状态（相当于读取 PCB 中的状态信息）
cat /proc/$$/status | head -15

echo "---"
# 查看进程的内存映射（地址空间信息）
cat /proc/$$/maps | head -5

echo "---"
# 查看进程打开的文件（资源清单）
ls -l /proc/$$/fd | head -5
```
