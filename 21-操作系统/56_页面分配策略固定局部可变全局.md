# 56_页面分配策略固定局部可变全局

## 核心概念

- **页面分配策略**：决定操作系统如何为每个进程分配物理内存块（页框）。
- 两个维度：分配量（固定 / 可变）× 置换范围（局部 / 全局）。
- 固定分配：进程在运行期间拥有的物理块数不变。
- 可变分配：进程的物理块数可以根据需要动态调整。
- 408 常考：给定策略，判断驻留集变化趋势与系统行为。

## 原理分析

### 固定分配局部置换（Fixed Allocation, Local Replacement）

$$\text{每个进程分配} \ n \ \text{个物理块，运行期间不变}$$

- 系统在进程创建时根据进程大小和优先级分配 $n$ 个物理块。
- 缺页时只从本进程的 $n$ 个块中选一个置换。
- 优点：简单、公平、进程间互不影响。
- 缺点：无法适应进程的动态内存需求。如果 $n$ 太小，缺页率高；$n$ 太大，内存浪费。
- 确定 $n$ 的方法：
  - 最小物理块数：保证能装入一条指令及其操作数（至少 1 个，通常 2-3 个）。
  - 按比例分配：$n_i = \frac{s_i}{\sum s_j} \times m$，其中 $s_i$ 为进程 $i$ 的页面数，$m$ 为总可用物理块数。
  - 按优先级分配。

### 可变分配全局置换（Variable Allocation, Global Replacement）

$$\text{驻留集大小} = \text{最近} \ \Delta \ \text{次访问中被引用的页面数}$$

- 系统维护一个全局空闲物理块池。
- 进程缺页时从全局空闲池取一块（若池空则从其他进程抢）。
- 驻留集大小随进程行为动态变化。
- 优点：内存利用率高，适应动态需求。
- 缺点：一个进程的抖动可能"抢走"其他进程的页面，导致连锁抖动。
- 这是许多实际系统（如 Linux、Windows）采用的策略。

### 工作集模型的应用

$$W(t, \Delta) = \text{在时间} \ (t - \Delta) \ \text{到} \ t \ \text{之间所有被引用的页面集合}$$

- 系统跟踪每个进程的工作集大小 $|W(t, \Delta)|$。
- 若 $|W(t, \Delta)| >$ 分配的块数 → 缺页率过高，需增加分配。
- 若 $|W(t, \Delta)| <$ 分配的块数 → 可以回收多余的块。
- 工作集模型常配合可变分配使用。

### 页面错误率（缺页率）控制

$$f = \frac{\text{缺页次数}}{\text{访问次数}}$$

- 设定缺页率上下限 $L$ 和 $U$：
  - $f < L$：驻留集太大，可回收物理块。
  - $f > U$：驻留集太小，需分配更多物理块。
  - $L \leq f \leq U$：合适，不动。

## 直观理解

- **固定分配** = 每个人发固定数量的桌子（考试时每人一张），不管用不用。
- **可变分配** = 自习室座位先到先得，人多了可以抢别人的空桌。
- **工作集** = 你最近半小时翻过的课本，老师根据这个判断你还需要几张桌子。
