# 76_磁盘调度FCFSSSTFSCANCSCAN

## 核心概念

- **磁盘调度算法**：当有多个磁盘 I/O 请求等待处理时，决定服务顺序以减少平均寻道时间。
- 四种经典算法：**FCFS**、**SSTF**、**SCAN（电梯算法）**、**C-SCAN（循环扫描）**。
- 优化目标：最小化平均寻道长度，提高磁盘 I/O 吞吐量。
- 408 高频考点：给定请求队列和当前磁头位置，手算各算法的调度顺序和总寻道距离。

## 原理分析

### 1. FCFS（先来先服务）

**规则**：按请求到达的先后顺序依次服务。

**示例**：磁头当前在 53 号磁道，请求队列 `98, 183, 37, 122, 14, 124, 65, 67`。

调度顺序：53 → 98 → 183 → 37 → 122 → 14 → 124 → 65 → 67

$$\text{总寻道距离} = |53-98| + |98-183| + |183-37| + |37-122| + |122-14| + |14-124| + |124-65| + |65-67|$$
$$= 45 + 85 + 146 + 85 + 108 + 110 + 59 + 2 = 640$$

**特点**：
- 公平，不会饥饿。
- 但磁头来回大幅移动，效率很低。

### 2. SSTF（最短寻道时间优先，Shortest Seek Time First）

**规则**：每次选择离当前磁头位置最近的请求。

**示例**：磁头在 53，队列 `98, 183, 37, 122, 14, 124, 65, 67`。

- 53 → 65（最近，距离12）
- 65 → 67（距离2）
- 67 → 37（距离30）
- 37 → 14（距离23）
- 14 → 98（距离84）
- 98 → 122（距离24）
- 122 → 124（距离2）
- 124 → 183（距离59）

$$\text{总寻道距离} = 12 + 2 + 30 + 23 + 84 + 24 + 2 + 59 = 236$$

**特点**：
- 寻道距离比 FCFS 大幅缩短。
- **可能导致饥饿**：如果新请求不断靠近磁头，远处的请求可能永远得不到服务。

### 3. SCAN（扫描算法 / 电梯算法）

**规则**：磁头沿一个方向移动，依次服务沿途的请求；到达尽头后反向移动。

**示例**：磁头在 53，向磁道号增大方向移动，队列 `98, 183, 37, 122, 14, 124, 65, 67`。

磁头向增大方向移动，沿途服务：65, 67, 98, 122, 124, 183。
到达 183（假设磁道最大号 199，到头后反向）。
反向服务：37, 14。

调度顺序：53 → 65 → 67 → 98 → 122 → 124 → 183 → 37 → 14

$$\text{总寻道距离} = |53-65| + |65-67| + |67-98| + |98-122| + |122-124| + |124-183| + |183-37| + |37-14|$$
$$= 12 + 2 + 31 + 24 + 2 + 59 + 146 + 23 = 299$$

注意：SCAN 到达磁道边界（199）后再折返，因此 183 到 37 包含了走到 199 再折返的距离。如果假设直接反向（不需要走到尽头）：

$$\text{修正：} 53 → 65 → 67 → 98 → 122 → 124 → 183 → 37 → 14$$
$$= 12+2+31+24+2+59+146+23 = 299$$

但如果严格按照 SCAN（必须走到最外端 199 再反向）：
$$\text{修正：} 53→65→67→98→122→124→183→199→14$$
$$= 12+2+31+24+2+59+16+185 = 331$$

实际上考试中通常允许到最远请求后就反向，或者到磁道边界后反向，题目会说明。

**特点**：
- 不会导致饥饿（每个方向的请求最终都会被服务到）。
- 对两端磁道不公平（最外和最内的请求等待时间最长）。

### 4. C-SCAN（循环扫描）

**规则**：磁头单向移动（如增大方向），到达尽头后**直接跳到另一端**（不服务反向请求），继续单向扫描。

**示例**：磁头在 53，向增大方向移动。

增大方向沿途：65, 67, 98, 122, 124, 183。
到 199（边界），直接跳到 0（不服务跳过的请求），继续增大方向。
从 0 继续：14, 37。

调度顺序：53 → 65 → 67 → 98 → 122 → 124 → 183 → [跳到 0] → 14 → 37

$$\text{寻道距离} = 12+2+31+24+2+59+16+14+37 = 197$$

（跳到 0 的距离 = 199，但通常跳转时不计入寻道时间或单独计算。）

实际总移动距离（含跳转）：
$$= 12+2+31+24+2+59+16 + 199 + 14 + 37 = 396$$

**特点**：
- 提供更均匀的等待时间（所有请求都在单向扫描中被服务）。
- 跳转时的长距离移动可以用"不计入寻道时间"来优化。

### 四种算法对比

| 算法 | 寻道效率 | 饥饿问题 | 公平性 | 实际应用 |
|------|---------|---------|--------|---------|
| FCFS | 低 | 无 | 最公平 | 简单场景 |
| SSTF | 较高 | 有 | 差 | 较少使用 |
| SCAN | 高 | 无 | 中等 | 广泛使用 |
| C-SCAN | 高 | 无 | 较好 | 广泛使用 |

### SCAN 和 C-SCAN 的变体

- **LOOK**：SCAN 的改进，不到磁道边界，只到最远请求就反向。
- **C-LOOK**：C-SCAN 的改进，跳转时只跳到最远的另一端请求。

考试中 SCAN 和 LOOK 通常不严格区分，按题意处理。

## 直观理解

- **FCFS** = 餐厅按排队顺序上菜，不管桌上菜离你近不近。
- **SSTF** = 先夹离你最近的菜，但可能有人的菜永远吃不到。
- **SCAN** = 服务员从桌子一端走到另一端，沿途上菜，到头折返。
- **C-SCAN** = 服务员从一端走到另一端上完菜，空手走回起点重新开始。
