# 35_管程

## 核心概念

- **管程（Monitor）**：一种高级同步机制，将共享数据和操作封装在一个模块内
- **条件变量（Condition Variable）**：管程内用于进程等待和唤醒的机制
- **x.wait()**：将调用进程阻塞在条件变量x上，并释放管程的互斥锁
- **x.signal()**：唤醒条件变量x上的一个等待进程
- **互斥性**：同一时刻只有一个进程能在管程内执行（编译器自动保证）
- **Hoare管程**：signal后立即切换到被唤醒进程
- **Mesa管程**：signal后唤醒进程排到就绪队列（实际系统多用此）

**408核心考点**：
- 管程与PV操作的区别
- 条件变量与信号量的区别
- wait和signal的含义

## 原理分析

**管程的结构**：

```
Monitor MonitorName {
    共享数据结构;           // 私有，外部不可直接访问

    // 初始化代码
    init() { ... }

    // 操作过程（外部可调用）
    procedure operation1() {
        if (条件不满足)
            condition.wait();   // 阻塞自己，释放管程锁
        // 操作共享数据
        condition.signal();     // 唤醒等待者
    }
}
```

**管程 vs PV操作对比**：

| 特性 | PV操作 | 管程 |
|------|--------|------|
| 封装性 | 低，信号量散布在各进程 | 高，共享数据被封装 |
| 互斥保证 | 手动P/V保护 | 编译器自动保证 |
| 易用性 | 容易出错（顺序错误等） | 更安全 |
| 底层实现 | 硬件（TS等） | 硬件或PV操作 |

**条件变量 vs 信号量**：

$$
\text{条件变量：} \quad wait() \text{ 同时释放管程锁} \Rightarrow \text{被唤醒后需重新获取锁}
$$

$$
\text{信号量：} \quad P() \text{ 减1后若≥0继续执行} \Rightarrow \text{不涉及锁的释放}
$$

关键区别：条件变量没有"记忆"功能，signal时如果没有人等待，信号丢失；信号量有计数功能。

**用管程实现生产者-消费者**：

```
Monitor PC {
    item buffer[N];
    int count = 0, in = 0, out = 0;
    Condition notFull, notEmpty;

    procedure produce(item x) {
        if (count == N)
            notFull.wait();        // 满了就等，释放管程锁
        buffer[in] = x;
        in = (in + 1) % N;
        count++;
        notEmpty.signal();         // 通知消费者有产品了
    }

    procedure consume(item *x) {
        if (count == 0)
            notEmpty.wait();       // 空了就等
        *x = buffer[out];
        out = (out + 1) % N;
        count--;
        notFull.signal();          // 通知生产者有空位了
    }
}
```

## 直观理解

**银行柜台的类比（管程）**：
- 管程就像银行的大厅：一次只让一个人在大厅里办业务（互斥）
- 条件变量就像"取号等叫号"：你的业务条件不满足（比如要办的窗口没开），你就取号等（wait），同时让出大厅位置
- signal就像叫号：叫到你了，你重新进大厅办业务
- 共享数据（账户余额等）只有通过大厅里的操作员才能访问，你不能自己去后台翻

**管程的意义**：
把同步的复杂性封装起来，使用者只需要调用`produce()`和`consume()`，不需要关心内部的PV操作。就像你去银行只需要说"存钱"，不需要自己操作电脑系统。

## 知识关联

**与PV操作的关系（第30-31节）**：
- 管程是在PV操作基础上的高级抽象
- 管程内部的互斥由编译器/系统自动保证，不需要手动P(mutex)
- 管程解决了PV操作容易出错的问题

**与操作系统内核**：
- Java的synchronized关键字和wait/notify就是管程思想
- Python的threading.Condition也是管程
- 操作系统课程中的管程主要用于理论分析

**408常见考法**：
- 问管程和信号量的区别
- 问条件变量的wait是否释放锁（释放！）
- 问Mesa管程和Hoare管程的区别
- 判断"管程能否替代PV操作"（可以，但实际中两者并存）
