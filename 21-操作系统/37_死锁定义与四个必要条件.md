# 37_死锁定义与四个必要条件

## 核心概念

- **死锁（Deadlock）**：多个进程因竞争资源而造成的一种僵局，若无外力作用，这些进程都无法向前推进
- **饥饿（Starvation）**：某个进程长期得不到资源，但系统并未死锁（其他进程在运行）
- **四个必要条件**（Coffman条件，1971）：
  1. **互斥条件**：资源一次只能被一个进程使用
  2. **不可剥夺条件**：资源不能被强行剥夺，只能由持有者主动释放
  3. **请求与保持条件**：进程持有一些资源的同时，又申请新的资源
  4. **循环等待条件**：存在一个进程-资源的环形等待链

**408核心考点**：
- 四个条件是"必要"条件，不是充分条件
- 四个条件同时满足才会死锁（缺一不可）
- 死锁 vs 饥饿 vs 死循环的辨析

## 原理分析

**死锁 vs 饥饿 vs 死循环**：

| 特性 | 死锁 | 饥饿 | 死循环 |
|------|------|------|--------|
| 进程状态 | 阻塞态 | 就绪态/阻塞态 | 运行态 |
| 涉及进程数 | ≥2 | ≥1 | 1 |
| 有无外力介入 | 没有 | 有（调度策略） | 有 |
| 原因 | 资源竞争 | 优先级低/调度不公平 | 代码逻辑错误 |

**死锁发生的数学描述**：

$$
\text{设系统有} n \text{个进程} P_1, P_2, \ldots, P_n
$$

$$
P_1 \text{等待} P_2 \text{释放资源}, \quad P_2 \text{等待} P_3 \text{释放资源}, \quad \ldots, \quad P_n \text{等待} P_1 \text{释放资源}
$$

$$
\Rightarrow \text{循环等待} \Rightarrow \text{死锁}
$$

**死锁实例**：

```
// 两个进程，两种资源（磁带机T1、打印机P1）
进程A:                         进程B:
    申请T1，获得;                 申请P1，获得;
    申请P1，等待(B占用);          申请T1，等待(A占用);
    // 阻塞                       // 阻塞
    // 死锁！
```

**四个条件的分析**：

1. **互斥条件**：磁带机和打印机是独占资源，不能共享
2. **不可剥夺条件**：OS不能从A手中抢走T1给B
3. **请求与保持条件**：A持有T1的同时申请P1
4. **循环等待条件**：A等B释放P1，B等A释放T1

打破其中任何一个条件即可预防死锁。

## 直观理解

**交通堵塞的类比**：
十字路口四个方向都有车要过，谁也不让谁（互斥），谁也不退（不可剥夺），每辆车都等对面的车让开（循环等待），就形成了死锁。交警来了才能解开（外力介入）。

**过独木桥的类比**：
两人从两端同时上桥，走到中间相遇，桥太窄两人不能并行（互斥），谁也不肯退回去（不可剥夺），都等对方后退（循环等待）。

## 知识关联

**与资源分配图的关系（第38节）**：
- 循环等待条件在资源分配图中表现为"环路"
- 有环路不一定有死锁（同类资源多个实例时）
- 无环路一定无死锁

**与死锁预防的关系（第39节）**：
- 破坏四个必要条件之一即可预防死锁
- 但破坏每个条件都有代价

**与银行家算法的关系（第40节）**：
- 银行家算法属于死锁避免，不破坏必要条件
- 而是在分配前判断是否安全

**408常见考法**：
- 给出场景，判断是否满足死锁的四个条件
- 问"四个条件中去掉哪个最方便"（通常是请求与保持或循环等待）
- 死锁 vs 饥饿的辨析（选择题常考）
- 问"某个条件不满足时是否还会死锁"（不会）
