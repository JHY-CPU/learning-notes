# 12_进程创建终止阻塞唤醒

## 核心概念

- **进程创建（Creation）**：创建新进程，分配PCB和资源
- **进程终止（Termination）**：回收进程资源，撤销PCB
- **进程阻塞（Block）**：进程主动暂停，让出CPU，进入阻塞态
- **进程唤醒（Wakeup）**：将阻塞态进程唤醒，转入就绪态
- **创建的原语操作**：申请空白PCB → 分配资源 → 初始化PCB → 插入就绪队列
- **阻塞和唤醒必须成对出现**

## 原理分析

### 进程创建（Create）

**触发条件**：
1. 用户登录
2. 作业调度（批处理系统）
3. 提供服务（OS为用户请求创建子进程，如fork）
4. 应用请求（父进程创建子进程）

**创建过程**：
```
1. 申请空白PCB（从PCB池中分配）
2. 为新进程分配唯一标识号PID
3. 分配资源（内存、文件等）
4. 初始化PCB：
   - 标识信息（PID、父进程PID）
   - 处理机状态（PC指向程序入口，PSW初始值）
   - 进程状态 = 就绪态
   - 优先级 = 默认值
5. 将PCB插入就绪队列
```

**引起创建的事件**：`fork()`系统调用

### 进程终止（Exit）

**触发条件**：
1. 正常结束（执行完毕，`exit()`）
2. 异常结束（越界错误、非法指令、超时）
3. 外界干预（kill命令、父进程终止导致子进程终止）

**终止过程**：
```
1. 根据PID找到PCB
2. 若进程正在运行，立即终止，重新调度
3. 若有子孙进程，递归终止所有子孙进程
4. 归还资源给父进程或OS
5. 将PCB从所在队列中移除
6. 回收PCB
```

### 进程阻塞（Block）

**触发条件**（进程**主动**行为）：
1. 请求系统资源失败
2. 等待某种操作完成（I/O）
3. 新数据尚未到达
4. 等待新任务

**阻塞过程**：
```
1. 找到要阻塞的进程的PCB
2. 保护CPU现场到PCB
3. 将进程状态改为阻塞态
4. 将PCB插入对应的阻塞队列
5. 转进程调度（选择另一个就绪进程运行）
```

### 进程唤醒（Wakeup）

**触发条件**（**被动**，由其他进程或OS完成）：
1. 所等待的事件发生（I/O完成）
2. 所等待的条件满足（数据到达）

**唤醒过程**：
```
1. 从阻塞队列中找到被唤醒进程的PCB
2. 将进程状态从阻塞态改为就绪态
3. 将PCB从阻塞队列移除，插入就绪队列
4. （等待调度程序调度其运行）
```

### 阻塞与唤醒的关键规则

$$\text{阻塞原语} \leftrightarrow \text{唤醒原语（必须成对）}$$

- 进程调用阻塞原语**阻塞自己**
- 另一个进程或OS调用唤醒原语**唤醒**被阻塞的进程
- 如果只有阻塞没有唤醒，进程将**永久阻塞（死锁的一种）**

## 直观理解

- **创建**：公司招聘新员工（分配工号、工位、档案）
- **终止**：员工离职（交还工牌、清理工位、注销档案）
- **阻塞**：员工主动请假（保存工作进度，离开岗位）
- **唤醒**：假期结束通知员工回来上班（但要先排队等排班）
- **成对规则**：请假必须有销假通知，否则就永远不来上班了

## 知识关联

- **阻塞→就绪**是状态转换，但需要**唤醒原语**主动触发
- **阻塞是进程自身行为**（自己调用block原语），**唤醒是外部行为**
- **fork()**创建子进程，子进程复制父进程的地址空间（写时复制COW）
- **进程调度**是在阻塞或终止后选择新进程的核心机制，关联第3章
- **408考法**：
  - 选择题：阻塞是主动还是被动、唤醒由谁完成
  - 简答题：描述进程创建的步骤
  - 常见陷阱：**阻塞是主动的**（进程自己调用），**唤醒是被动的**；阻塞态进程**不能**被直接调度，必须先被唤醒到就绪态
