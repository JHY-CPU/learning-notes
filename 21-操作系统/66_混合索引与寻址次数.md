# 66_混合索引与寻址次数

## 核心概念

- **混合索引（Hybrid Index）**：综合使用直接块指针、一级间接、二级间接、三级间接，兼顾小文件效率和大文件支持。
- Unix/Linux inode 的经典设计。
- **寻址次数**：访问文件第 $i$ 个逻辑块需要的磁盘 I/O 次数。
- 408 高频计算题：给定参数计算最大文件大小、各种间接索引的寻址次数。

## 原理分析

### 混合索引结构

以 Unix ext2 为例，inode 中有 15 个块指针：

```
inode:
  i_block[0]  ──→ 直接块 0    \
  i_block[1]  ──→ 直接块 1     | 12个直接块
  ...                          |
  i_block[11] ──→ 直接块 11   /
  i_block[12] ──→ 一级间接索引块 ──→ [块号, 块号, ..., 块号]
  i_block[13] ──→ 二级间接索引块 ──→ [一级索引块号, 一级索引块号, ...]
                                                ↓
                                           [块号, 块号, ...]
  i_block[14] ──→ 三级间接索引块 ──→ [二级索引块号, ...]
                                         ↓
                                    [一级索引块号, ...]
                                         ↓
                                    [块号, 块号, ...]
```

### 参数计算

设：
- 块大小 $B = 1024$ B（1KB）
- 块号大小 $b = 4$ B
- 每个索引块能存放 $\lfloor B/b \rfloor = 256$ 个块号

**直接寻址**：
$$\text{覆盖} = 12 \times B = 12 \times 1\text{KB} = 12\text{KB}$$

**一级间接**：
$$\text{覆盖} = \frac{B}{b} \times B = 256 \times 1\text{KB} = 256\text{KB}$$

**二级间接**：
$$\text{覆盖} = \frac{B}{b} \times \frac{B}{b} \times B = 256^2 \times 1\text{KB} = 65{,}536\text{KB} = 64\text{MB}$$

**三级间接**：
$$\text{覆盖} = \frac{B}{b} \times \frac{B}{b} \times \frac{B}{b} \times B = 256^3 \times 1\text{KB} = 16{,}777{,}216\text{KB} = 16\text{GB}$$

**最大文件大小**：
$$\text{Max} = 12\text{KB} + 256\text{KB} + 64\text{MB} + 16\text{GB} \approx 16\text{GB}$$

### 寻址次数分析

访问文件的第 $i$ 个逻辑块（$i$ 从 0 开始）：

**情况 1**：$0 \leq i < 12$（直接块范围）
- 从 inode 中直接读取块号。
- **1 次磁盘 I/O**（读取目标数据块）。

**情况 2**：$12 \leq i < 12 + 256$（一级间接范围）
- 第 1 次 I/O：读取一级间接索引块。
- 第 2 次 I/O：读取目标数据块。
- **共 2 次磁盘 I/O**。

**情况 3**：$12 + 256 \leq i < 12 + 256 + 256^2$（二级间接范围）
- 第 1 次 I/O：读取二级间接索引块。
- 第 2 次 I/O：读取对应的一级间接索引块。
- 第 3 次 I/O：读取目标数据块。
- **共 3 次磁盘 I/O**。

**情况 4**：$i \geq 12 + 256 + 256^2$（三级间接范围）
- 第 1 次：读取三级间接索引块。
- 第 2 次：读取二级间接索引块。
- 第 3 次：读取一级间接索引块。
- 第 4 次：读取目标数据块。
- **共 4 次磁盘 I/O**。

### 典型计算题

**例题**：某文件系统 inode 有 13 个块号（12 直接 + 1 间接），块大小 512B，块号 4B，问：
1. 最大文件大小（仅一级间接）？
2. 访问文件第 300 块需几次磁盘 I/O？

**解答**：

1. 每索引块可放 $\lfloor 512/4 \rfloor = 128$ 个块号。

$$\text{最大文件} = 12 \times 512 + 128 \times 512 = (12+128) \times 512 = 140 \times 512 = 71{,}680\text{B} \approx 70\text{KB}$$

2. 第 300 块：$i = 299$（从 0 开始）。
   - $299 < 12$？否。
   - $299 < 12 + 128 = 140$？否，$299 \geq 140$。
   - 超出一级间接范围！该文件系统无法表示第 300 块（最大 140 块）。

**修正例题**：访问第 100 块（$i = 99$）。
- $12 \leq 99 < 140$？是，属于一级间接范围。
- 需要 2 次磁盘 I/O。

## 直观理解

- 混合索引就像一本书的"混合目录"：
  - 前几章直接在首页列出页码（直接）。
  - 中间章节用一张单独的目录纸（一级间接）。
  - 更大的书用"章目录→节目录"两级查找（二级间接）。
- 小文件用直接块，1 次 I/O 搞定，效率最高。大文件用间接块，代价高但能支持。
