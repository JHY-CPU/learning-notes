# 05_OS结构分层模块化外核

## 核心概念

- **OS的结构设计**：无结构 → 模块化 → 分层式 → 微内核 → 外核
- **无结构（整体系统）**：早期OS，各功能模块相互调用，杂乱无序
- **模块化结构**：将OS划分为若干模块，各模块功能独立，通过接口通信
  - 优点：设计独立、可维护性好
  - 缺点：模块间依赖关系复杂，无序调用
- **分层式结构**：按层次组织，每层只能调用**下层**提供的服务
  - 优点：结构清晰、调试方便、可扩展性好
  - 缺点：效率低（层次多了系统调用链长）
- **微内核（Microkernel）**：内核只保留最基本功能（进程通信、低级调度），其他服务放到用户态
  - 基本功能：进程间通信（IPC）、低级存储管理、中断处理
  - 优点：可靠性高、可扩展性好、适合分布式
  - 缺点：性能较低（用户态与内核态频繁切换）
- **外核（Exokernel）**：OS不提供传统抽象，而是直接暴露硬件资源给应用
  - 应用程序自行决定如何使用硬件
  - 减少了抽象层，提高效率

## 原理分析

### 分层结构的层次设计

以THE系统（Dijkstra）为例：

```
第5层：操作员（用户程序）
第4层：用户程序（编译器、编辑器）
第3层：I/O管理
第2页：操作员-进程通信
第1层：内存管理
第0层：处理器分配和中断处理
```

- 每层仅使用下层提供的功能
- 第0层是硬件，最上层是用户

### 微内核 vs 大内核对比

| 对比项 | 大内核（宏内核） | 微内核 |
|-------|---------------|--------|
| 内核大小 | 庞大（文件系统、设备驱动等都在内核） | 精简（仅IPC、调度等） |
| 服务位置 | 内核态 | 用户态（服务器进程） |
| 性能 | 高（同一地址空间，无切换开销） | 低（频繁用户态/内核态切换） |
| 可靠性 | 低（一个模块崩溃→系统崩溃） | 高（服务器崩溃不影响内核） |
| 典型代表 | Linux、Windows | Mach、QNX、Minix |

### 外核的设计哲学

传统OS：OS提供虚拟机抽象，应用程序在虚拟机上运行
外核：OS只做安全检查和资源分配，应用程序直接使用硬件

$$\text{效率} = \frac{\text{应用程序实际工作}}{\text{总开销（含OS抽象层开销）}}$$

外核通过减少抽象层提高分子比。

## 直观理解

- **无结构**：一栋没有图纸盖的房子，哪里需要什么就加什么，乱七八糟
- **模块化**：把房子分成客厅、卧室、厨房等模块，但模块之间的管道走向随意
- **分层式**：大楼分层，一楼是地基，二楼是管道，三楼是电路，每层只能用下面层的东西
- **微内核**：物业只管电梯和门禁（最基本功能），水电煤气维修外包给专业公司（用户态服务）
- **外核**：物业把房子直接租给你，你怎么装修、怎么用空间全由自己决定

## 知识关联

- **分层结构**的思想与计算机网络的分层模型（OSI七层）类似
- **微内核的IPC机制**关联第18个知识点（进程通信）
- **大内核与微内核**是408高频考点，见第7个知识点的详细对比
- **模块化是现代OS的主流**（Linux是模块化的大内核）
- **408考法**：
  - 选择题：各结构的优缺点对比
  - 简答题：为什么微内核性能较低
  - 常见陷阱：微内核**并非**所有性能都比大内核差（可靠性更高）；外核不是408重点但要了解概念
