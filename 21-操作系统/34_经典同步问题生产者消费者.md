# 34_经典同步问题生产者消费者

## 核心概念

- **生产者-消费者问题**：经典进程同步模型，有界缓冲区被生产者和消费者共享
- **有界缓冲区（Bounded Buffer）**：大小为n的循环缓冲区
- **同步关系1**：缓冲区满时生产者必须等待（empty信号量）
- **同步关系2**：缓冲区空时消费者必须等待（full信号量）
- **互斥关系**：对缓冲区的访问必须互斥（mutex信号量）
- **三个信号量**：empty(n), full(0), mutex(1)

**408核心考点**：
- 必须同时使用三个信号量（两个同步+一个互斥）
- P操作顺序：先同步后互斥（否则可能死锁）
- V操作顺序：先互斥后同步（不影响正确性，但推荐）

## 原理分析

**标准解法**：

```
Semaphore empty = n;     // 空闲缓冲区数量（同步：生产者等空位）
Semaphore full = 0;      // 已用缓冲区数量（同步：消费者等产品）
Semaphore mutex = 1;     // 缓冲区访问互斥

// 生产者
Producer:
    生产一个产品;
    P(empty);            // 申请一个空位（同步）
    P(mutex);            // 申请缓冲区访问权（互斥）
    将产品放入缓冲区;
    V(mutex);
    V(full);             // 产品数+1，通知消费者

// 消费者
Consumer:
    P(full);             // 申请一个产品（同步）
    P(mutex);            // 申请缓冲区访问权（互斥）
    从缓冲区取出产品;
    V(mutex);
    V(empty);            // 空位+1，通知生产者
    消费产品;
```

**P操作顺序的重要性**：

$$
\text{正确：} P(empty) \to P(mutex) \quad \text{（先等空位，再拿锁）}
$$

$$
\text{错误：} P(mutex) \to P(empty) \quad \text{（先拿锁，再等空位 → 死锁！）}
$$

死锁场景：缓冲区满，生产者先P(mutex)拿到锁，再P(empty)发现没有空位，阻塞但不释放锁，消费者无法P(mutex)进入缓冲区消费。

**V操作的顺序**：
V操作不会阻塞，所以V(mutex)和V(full)的顺序不影响正确性。但一般习惯先V(mutex)释放锁，再V同步信号量。

**多生产者多消费者**：
如果有多个生产者和多个消费者，mutex仍然是1（因为只有一个缓冲区），empty和full的值不变。

## 直观理解

**快递驿站的类比**：
- 缓冲区 = 快递柜（有n个格子）
- 生产者 = 快递员放快递
- 消费者 = 用户取快递
- empty = 空闲格子数，快递员来了要检查有没有空格
- full = 有快递的格子数，用户来了要检查有没有快递
- mutex = 驿站工作人员一次只让一个人操作柜子

**P操作顺序的类比**：
如果快递员先"锁了门不让人进"（P(mutex)），再看有没有空柜子（P(empty)），发现没有空柜子，他就站门口等着但不走（阻塞但持有锁）。用户也进不来取快递，大家都等着——死锁！

## 知识关联

**与信号量机制的关系（第30节）**：
- 生产者-消费者问题是信号量最经典的应用场景
- 体现了同步信号量（empty/full）和互斥信号量（mutex）的区别

**与缓冲区管理的关系（第3章）**：
- 单缓冲：empty=1, full=0
- 双缓冲：empty=2, full=0
- 环形缓冲区：empty=n, full=0

**408常见考法**：
- 给出PV操作序列，判断是否正确
- 问"能否将empty和full合并为一个信号量"（不能，两个方向的同步）
- 分析死锁场景（P操作顺序错误）
- 问信号量初值的设定
- 生产者和消费者都调用P(empty)和P(full)，如果交换会怎样
