# 43_死锁综合对比

## 核心概念

- **死锁处理四大策略**：忽略、预防、避免、检测与解除
- **鸵鸟策略（Ostrich Algorithm）**：忽略死锁，假设不会发生
- **实际OS的选择**：大多数通用OS采用"检测与解除"或"忽略"

**408核心考点**：
- 四种处理策略的对比（选择题/简答题高频考点）
- 银行家算法与安全性算法的完整流程
- 资源分配图化简与死锁判断

## 原理分析

**四种处理策略全景对比**：

| 维度 | 忽略(鸵鸟) | 预防 | 避免 | 检测与解除 |
|------|-----------|------|------|-----------|
| 核心思想 | 假设不死锁 | 破坏必要条件 | 动态检查安全状态 | 允许发生后处理 |
| 资源利用率 | 高 | 低 | 中 | 高 |
| 系统开销 | 最小 | 小 | 中 | 检测小解除大 |
| 死锁可能性 | 有 | 无 | 无 | 有（可解除） |
| 实现复杂度 | 最简单 | 简单 | 中等 | 复杂 |
| 代表方法 | Linux/Windows默认 | 有序分配 | 银行家算法 | 资源分配图 |
| 适用场景 | 死锁很少发生 | 关键系统 | 实时系统 | 通用OS |

**四个必要条件 vs 四种处理策略**：

$$
\begin{array}{c|c}
\text{必要条件} & \text{对应的预防方法} \\
\hline
\text{互斥} & \text{SPOOLing虚拟化} \\
\text{不可剥夺} & \text{允许剥夺} \\
\text{请求与保持} & \text{一次性申请全部资源} \\
\text{循环等待} & \text{有序资源分配}
\end{array}
$$

**安全状态与死锁的关系**：

$$
\text{安全状态} \Rightarrow \text{一定不会死锁}
$$

$$
\text{不安全状态} \Rightarrow \text{可能死锁（不等于一定死锁）}
$$

$$
\text{死锁} \Rightarrow \text{一定处于不安全状态}
$$

**银行家算法与检测算法的关键区别**：

| 特性 | 银行家算法(避免) | 检测算法 |
|------|-----------------|---------|
| 检查时机 | 每次分配前 | 定期 |
| 已知信息 | Max, Allocation, Available | Request, Allocation, Available |
| Need计算 | Need = Max - Allocation | 不需要Max，直接用Request |
| 结果 | 决定是否分配 | 找出死锁进程 |

## 直观理解

**生活中的四种策略**：

1. **忽略**：就像不买保险，赌不会出事（大部分人的选择）
2. **预防**：就像从不坐飞机，彻底避免空难（但牺牲了便利性）
3. **避免**：就像每次开车前检查路况，危险就不开（安全但麻烦）
4. **检测与解除**：就像买了保险，出了事保险公司赔（事后补救）

## 代码实现

### 银行家算法完整实现

```c
#include <stdio.h>
#include <stdbool.h>

/*
 * 银行家算法（Banker's Algorithm）——死锁避免
 *
 * 核心思想：每次分配资源前，模拟分配后检查系统是否处于安全状态。
 * 如果安全则分配，不安全则让进程等待。
 *
 * 数据结构：
 * - Available[m]  : 系统当前可用资源向量
 * - Max[n][m]     : 每个进程的最大需求矩阵
 * - Allocation[n][m]: 每个进程已分配的资源矩阵
 * - Need[n][m]    : 每个进程还需要的资源矩阵 = Max - Allocation
 */

#define N 5   /* 进程数 */
#define M 3   /* 资源类型数 */

int Available[M] = {3, 3, 2};
int Max[N][M] = {
    {7, 5, 3}, {3, 2, 2}, {9, 0, 2}, {2, 2, 2}, {4, 3, 3}
};
int Allocation[N][M] = {
    {0, 1, 0}, {2, 0, 0}, {3, 0, 2}, {2, 1, 1}, {0, 0, 2}
};
int Need[N][M];

/* 初始化 Need 矩阵 */
void init_need() {
    for (int i = 0; i < N; i++)
        for (int j = 0; j < M; j++)
            Need[i][j] = Max[i][j] - Allocation[i][j];
}

/* 安全性算法：检查系统是否处于安全状态 */
bool is_safe(int safe_seq[]) {
    int Work[M];
    bool Finish[N];
    int count = 0;

    for (int j = 0; j < M; j++) Work[j] = Available[j];
    for (int i = 0; i < N; i++) Finish[i] = false;

    while (count < N) {
        bool found = false;
        for (int i = 0; i < N; i++) {
            if (!Finish[i]) {
                /* 检查 Need[i] <= Work */
                bool can_run = true;
                for (int j = 0; j < M; j++) {
                    if (Need[i][j] > Work[j]) {
                        can_run = false;
                        break;
                    }
                }
                if (can_run) {
                    /* 进程 Pi 可以执行完毕，释放资源 */
                    for (int j = 0; j < M; j++)
                        Work[j] += Allocation[i][j];
                    Finish[i] = true;
                    if (safe_seq) safe_seq[count] = i;
                    count++;
                    found = true;
                }
            }
        }
        if (!found) return false;  /* 找不到可执行的进程，不安全 */
    }
    return true;
}

/* 银行家算法：处理进程 Pi 的资源请求 */
bool request_resources(int pi, int request[]) {
    /* Step 1: 合法性检查 Request <= Need */
    for (int j = 0; j < M; j++) {
        if (request[j] > Need[pi][j]) {
            printf("错误：请求超出最大需求\n");
            return false;
        }
    }
    /* Step 2: 可用性检查 Request <= Available */
    for (int j = 0; j < M; j++) {
        if (request[j] > Available[j]) {
            printf("资源不足，进程 P%d 需要等待\n", pi);
            return false;
        }
    }
    /* Step 3: 试探分配 */
    for (int j = 0; j < M; j++) {
        Available[j]   -= request[j];
        Allocation[pi][j] += request[j];
        Need[pi][j]    -= request[j];
    }
    /* Step 4: 安全性检查 */
    int safe_seq[N];
    if (is_safe(safe_seq)) {
        printf("分配安全！安全序列：");
        for (int i = 0; i < N; i++)
            printf("P%d%s", safe_seq[i], i < N-1 ? " → " : "\n");
        return true;
    } else {
        /* 不安全，回滚 */
        for (int j = 0; j < M; j++) {
            Available[j]   += request[j];
            Allocation[pi][j] -= request[j];
            Need[pi][j]    += request[j];
        }
        printf("分配后不安全，已回滚\n");
        return false;
    }
}

int main() {
    init_need();

    printf("=== 银行家算法演示 ===\n");
    printf("系统资源: Available = (%d, %d, %d)\n\n",
           Available[0], Available[1], Available[2]);

    /* 检查初始状态是否安全 */
    int safe_seq[N];
    if (is_safe(safe_seq)) {
        printf("初始状态安全，安全序列：");
        for (int i = 0; i < N; i++)
            printf("P%d%s", safe_seq[i], i < N-1 ? " → " : "\n");
    }

    /* 进程 P1 请求资源 (1, 0, 2) */
    printf("\nP1 请求 (1, 0, 2):\n");
    int req1[] = {1, 0, 2};
    request_resources(1, req1);

    return 0;
}
```

### 死锁检测算法（多实例资源）

```c
#include <stdio.h>
#include <stdbool.h>

/*
 * 死锁检测算法——允许死锁发生，定期检测
 *
 * 与安全性算法结构类似，但使用 Request 而非 Need：
 * - Finish[i] = (Allocation[i] == 0) ? true : false
 * - 循环找 Finish[i]==false 且 Request[i] <= Work 的进程
 * - 最终 Finish[i]==false 的进程处于死锁状态
 */

#define N 4
#define M 2

int Available[M] = {0, 0};
int Allocation[N][M] = {{0,1}, {1,0}, {1,0}, {0,1}};
int Request[N][M]    = {{1,0}, {0,1}, {0,0}, {0,0}};

void detect_deadlock() {
    int Work[M];
    bool Finish[N];

    for (int j = 0; j < M; j++) Work[j] = Available[j];
    for (int i = 0; i < N; i++) {
        Finish[i] = true;
        for (int j = 0; j < M; j++)
            if (Allocation[i][j] != 0) { Finish[i] = false; break; }
    }

    bool changed = true;
    while (changed) {
        changed = false;
        for (int i = 0; i < N; i++) {
            if (!Finish[i]) {
                bool can_finish = true;
                for (int j = 0; j < M; j++)
                    if (Request[i][j] > Work[j]) { can_finish = false; break; }
                if (can_finish) {
                    for (int j = 0; j < M; j++)
                        Work[j] += Allocation[i][j];
                    Finish[i] = true;
                    changed = true;
                    printf("P%d 可以完成，释放资源\n", i);
                }
            }
        }
    }

    printf("\n检测结果：");
    bool deadlock = false;
    for (int i = 0; i < N; i++) {
        if (!Finish[i]) {
            printf("P%d 处于死锁状态！ ", i);
            deadlock = true;
        }
    }
    if (!deadlock) printf("无死锁");
    printf("\n");
}

int main() {
    printf("=== 死锁检测算法 ===\n\n");
    detect_deadlock();
    return 0;
}
```

## 知识关联

**与进程同步的关系（第27-36章）**：
- 同步操作中P/V顺序错误会导致死锁
- 哲学家进餐问题是死锁的经典示例

**与内存管理的关系（第6章）**：
- 内存分配也可能导致死锁（多个进程竞争内存页面）
- 页面置换不当可能加重死锁
