# 69_IO设备分类字符块人机通信

## 核心概念

- **I/O 设备**：计算机系统中除 CPU 和内存之外的所有设备。
- 按信息交换单位分类：**字符设备**、**块设备**、**网络设备**。
- 按使用特性分类：**存储设备**（非易失）、**I/O 设备**（人机交互）。
- 按传输速率分类：低速（键盘）、中速（打印机）、高速（磁盘）。
- 408 考点：设备分类、各类设备的特点、设备独立性概念。

## 原理分析

### 按信息交换单位分类

**字符设备（Character Device）**：
- 以字节为单位进行 I/O，数据不可随机访问。
- 无缓冲或只有简单的缓冲。
- 通常不支持寻址。
- 例子：键盘、鼠标、串口、显示器。

**块设备（Block Device）**：
- 以数据块（如 512B、4KB）为单位进行 I/O。
- 支持随机访问（通过块号）。
- 通常有缓冲和缓存机制。
- 例子：硬盘、SSD、光盘、USB 存储。

**网络设备（Network Device）**：
- 以报文/帧为单位进行 I/O。
- 既不是字符流也不是固定大小的块。
- 例子：网卡、调制解调器。

### 按使用特性分类

| 类型 | 传输速度 | 举例 |
|------|---------|------|
| 低速设备 | 几十 B/s ~ 几 KB/s | 键盘、鼠标、语音输入 |
| 中速设备 | 几 KB/s ~ 几百 KB/s | 打印机、调制解调器 |
| 高速设备 | 几百 KB/s ~ GB/s | 磁盘、SSD、光盘 |

### 设备控制器（I/O Controller）

I/O 设备通常由**机械部分**（设备本身）和**电子部分**（设备控制器/适配器）组成。

**设备控制器的功能**：
- 接收 CPU 的命令，控制设备操作。
- 将设备的信号转换为 CPU 可理解的数据格式。
- 进行数据缓冲（内部有数据缓冲寄存器）。
- 向 CPU 报告设备状态（就绪、忙碌、出错）。

**设备控制器的寄存器**：
- **数据寄存器**：存放要读/写的数据。
- **状态寄存器**：设备当前状态（busy / ready / error）。
- **控制寄存器**：CPU 写入命令。

### 设备独立性（Device Independence）

**定义**：应用程序不依赖于特定的物理设备，而是通过逻辑设备名访问设备。

- **物理设备名**：系统中设备的真实标识，如 `/dev/sda1`。
- **逻辑设备名**：用户程序中使用的抽象名称，如 `/dev/printer`。
- **逻辑设备表（LUT）**：将逻辑设备名映射到物理设备。

**优点**：
- 程序可移植性好，更换设备不影响程序。
- 便于 I/O 重定向（同一程序可输出到不同设备）。
- 简化了设备管理。

## 直观理解

- **字符设备** = 一条水管，水（数据）一滴一滴地流，你只能从头喝到尾。
- **块设备** = 一个文件柜，每格装着固定大小的文件，你可以直接打开第 N 格。
- **设备控制器** = 翻译官 + 管家，CPU 说中文，设备说英文，翻译官负责沟通。
- **设备独立性** = 你说"打印"而不说"用HP LaserJet 2020打印"，系统决定实际用哪台打印机。

## 知识关联

- 与 70 I/O 控制方式：不同设备类型适合不同的控制方式（字符设备常用中断，块设备常用 DMA）。
- 与 72 设备分配：设备独立性需要逻辑设备表的支持。
- 与 74 SPOOLing：将独占设备（如打印机）虚拟为共享设备。
- 408 考查：设备分类标准、设备独立性的含义和实现。

## 代码实现

### I/O 设备读写——字符设备与块设备对比

```c
#include <stdio.h>
#include <stdlib.h>
#include <fcntl.h>
#include <unistd.h>
#include <string.h>

/*
 * 字符设备 vs 块设备的读写方式对比
 *
 * 字符设备：以字节流为单位，顺序访问
 *   - 例：键盘、鼠标、串口、/dev/random
 *   - open() → read()/write() → close()
 *   - 无缓冲，不可 seek
 *
 * 块设备：以数据块为单位，随机访问
 *   - 例：硬盘、SSD、/dev/sda
 *   - open() → lseek() → read()/write() → close()
 *   - 有缓冲/缓存，支持 seek
 */

/* 字符设备示例：从 /dev/urandom 读取随机数据 */
void char_device_demo() {
    printf("=== 字符设备读写示例 ===\n");

    int fd = open("/dev/urandom", O_RDONLY);
    if (fd < 0) {
        perror("open /dev/urandom");
        return;
    }

    /* 字符设备：顺序读取，一次读若干字节 */
    unsigned char buf[16];
    ssize_t n = read(fd, buf, sizeof(buf));
    if (n > 0) {
        printf("从 /dev/urandom 读取 %zd 字节: ", n);
        for (int i = 0; i < n; i++)
            printf("%02x ", buf[i]);
        printf("\n");
    }

    close(fd);
}

/* 块设备模拟：文件系统中的块级读写 */
void block_device_demo() {
    printf("\n=== 块设备读写示例 ===\n");

    /* 创建一个模拟块设备的文件 */
    const char* dev_path = "/tmp/block_sim";
    int fd = open(dev_path, O_RDWR | O_CREAT | O_TRUNC, 0644);
    if (fd < 0) { perror("open"); return; }

    /* 写入 4 个块（每块 512 字节，典型磁盘扇区大小） */
    int block_size = 512;
    char block[512];

    for (int i = 0; i < 4; i++) {
        memset(block, 'A' + i, block_size);  /* 每个块填充不同字符 */

        /* 块设备：按块号定位后写入（随机访问） */
        off_t offset = i * block_size;
        lseek(fd, offset, SEEK_SET);
        write(fd, block, block_size);
        printf("写入块 %d (偏移 %ld, %d 字节)\n", i, offset, block_size);
    }

    /* 随机读取第 2 块（跳过第 0、1 块） */
    lseek(fd, 2 * block_size, SEEK_SET);
    read(fd, block, block_size);
    printf("随机读取块 2: 首字节 = '%c'\n", block[0]);

    close(fd);
    unlink(dev_path);
}

/* 设备控制器寄存器操作伪代码 */
void device_controller_demo() {
    printf("\n=== 设备控制器寄存器操作（伪代码）===\n");
    printf(
        "// 设备控制器有三个关键寄存器\n"
        "#define DATA_REG   0x3F8  /* 数据寄存器 */\n"
        "#define STATUS_REG 0x3FD  /* 状态寄存器 */\n"
        "#define CTRL_REG   0x3FC  /* 控制寄存器 */\n"
        "\n"
        "// 轮询方式读取设备\n"
        "char poll_read() {\n"
        "    while (!(inb(STATUS_REG) & READY_BIT)) {\n"
        "        /* 忙等待：CPU 不断查询状态寄存器 */\n"
        "    }\n"
        "    return inb(DATA_REG);  /* 从数据寄存器读取 */\n"
        "}\n"
        "\n"
        "// 写入设备\n"
        "void poll_write(char data) {\n"
        "    while (!(inb(STATUS_REG) & READY_BIT)) {\n"
        "        /* 等待设备就绪 */\n"
        "    }\n"
        "    outb(DATA_REG, data);   /* 向数据寄存器写入 */\n"
        "}\n"
    );
}

int main() {
    char_device_demo();
    block_device_demo();
    device_controller_demo();
    return 0;
}
```

### 设备独立性——逻辑设备到物理设备的映射

```bash
#!/bin/bash
# Linux 中的设备独立性实现

echo "=== Linux 设备文件系统 ==="

# 所有设备都以文件形式出现在 /dev 目录下
echo "块设备（硬盘）："
ls -l /dev/sd* 2>/dev/null | head -3

echo ""
echo "字符设备（终端）："
ls -l /dev/tty* | head -3

echo ""
echo "逻辑设备名到物理设备的映射："
echo "  /dev/stdout → /proc/self/fd/1 → 终端或管道"
ls -l /dev/stdout

echo ""
echo "=== udev 设备管理 ==="
# udevd 守护进程负责维护 /dev 目录
# 将逻辑设备名（如 sda）映射到物理设备
udevadm info --query=all --name=/dev/sda 2>/dev/null | head -10

echo ""
echo "=== 通过设备独立性实现 I/O 重定向 ==="
# 同一个 echo 命令可以输出到不同设备
echo "输出到终端"
echo "输出到文件" > /tmp/test_device.log
cat /tmp/test_device.log
rm -f /tmp/test_device.log
```
