# 69_IO设备分类字符块人机通信

## 核心概念

- **I/O 设备**：计算机系统中除 CPU 和内存之外的所有设备。
- 按信息交换单位分类：**字符设备**、**块设备**、**网络设备**。
- 按使用特性分类：**存储设备**（非易失）、**I/O 设备**（人机交互）。
- 按传输速率分类：低速（键盘）、中速（打印机）、高速（磁盘）。
- 408 考点：设备分类、各类设备的特点、设备独立性概念。

## 原理分析

### 按信息交换单位分类

**字符设备（Character Device）**：
- 以字节为单位进行 I/O，数据不可随机访问。
- 无缓冲或只有简单的缓冲。
- 通常不支持寻址。
- 例子：键盘、鼠标、串口、显示器。

**块设备（Block Device）**：
- 以数据块（如 512B、4KB）为单位进行 I/O。
- 支持随机访问（通过块号）。
- 通常有缓冲和缓存机制。
- 例子：硬盘、SSD、光盘、USB 存储。

**网络设备（Network Device）**：
- 以报文/帧为单位进行 I/O。
- 既不是字符流也不是固定大小的块。
- 例子：网卡、调制解调器。

### 按使用特性分类

| 类型 | 传输速度 | 举例 |
|------|---------|------|
| 低速设备 | 几十 B/s ~ 几 KB/s | 键盘、鼠标、语音输入 |
| 中速设备 | 几 KB/s ~ 几百 KB/s | 打印机、调制解调器 |
| 高速设备 | 几百 KB/s ~ GB/s | 磁盘、SSD、光盘 |

### 设备控制器（I/O Controller）

I/O 设备通常由**机械部分**（设备本身）和**电子部分**（设备控制器/适配器）组成。

**设备控制器的功能**：
- 接收 CPU 的命令，控制设备操作。
- 将设备的信号转换为 CPU 可理解的数据格式。
- 进行数据缓冲（内部有数据缓冲寄存器）。
- 向 CPU 报告设备状态（就绪、忙碌、出错）。

**设备控制器的寄存器**：
- **数据寄存器**：存放要读/写的数据。
- **状态寄存器**：设备当前状态（busy / ready / error）。
- **控制寄存器**：CPU 写入命令。

### 设备独立性（Device Independence）

**定义**：应用程序不依赖于特定的物理设备，而是通过逻辑设备名访问设备。

- **物理设备名**：系统中设备的真实标识，如 `/dev/sda1`。
- **逻辑设备名**：用户程序中使用的抽象名称，如 `/dev/printer`。
- **逻辑设备表（LUT）**：将逻辑设备名映射到物理设备。

**优点**：
- 程序可移植性好，更换设备不影响程序。
- 便于 I/O 重定向（同一程序可输出到不同设备）。
- 简化了设备管理。

## 直观理解

- **字符设备** = 一条水管，水（数据）一滴一滴地流，你只能从头喝到尾。
- **块设备** = 一个文件柜，每格装着固定大小的文件，你可以直接打开第 N 格。
- **设备控制器** = 翻译官 + 管家，CPU 说中文，设备说英文，翻译官负责沟通。
- **设备独立性** = 你说"打印"而不说"用HP LaserJet 2020打印"，系统决定实际用哪台打印机。

## 知识关联

- 与 70 I/O 控制方式：不同设备类型适合不同的控制方式（字符设备常用中断，块设备常用 DMA）。
- 与 72 设备分配：设备独立性需要逻辑设备表的支持。
- 与 74 SPOOLing：将独占设备（如打印机）虚拟为共享设备。
- 408 考查：设备分类标准、设备独立性的含义和实现。
