# 40_死锁避免银行家算法

## 核心概念

- **死锁避免（Avoidance）**：在资源分配时动态检查，确保系统处于安全状态
- **安全状态**：存在一个安全序列，所有进程都能按此序列执行完毕
- **不安全状态**：不存在安全序列（不等于死锁，但可能演变为死锁）
- **银行家算法（Banker's Algorithm）**：Dijkstra 1965年提出，模拟分配后检查安全性
- **Need矩阵**：每个进程还需要多少资源
- **Available向量**：系统当前可用资源
- **Allocation矩阵**：每个进程已分配多少资源
- **Max矩阵**：每个进程最多需要多少资源

**408核心必考**：银行家算法的完整计算过程

## 原理分析

**数据结构定义**：

$$
Need[i][j] = Max[i][j] - Allocation[i][j]
$$

$$
\text{Available} = \text{系统总资源} - \sum_{i} Allocation[i]
$$

**银行家算法——资源请求处理**：

进程Pi发出请求向量 $Request_i$：

1. **合法性检查**：若 $Request_i[j] > Need[i][j]$，报错（超出需求）
2. **可用性检查**：若 $Request_i[j] > Available[j]$，等待（资源不足）
3. **试探分配**：
   - $Available = Available - Request_i$
   - $Allocation[i] = Allocation[i] + Request_i$
   - $Need[i] = Need[i] - Request_i$
4. **安全性检查**：执行安全性算法，安全则确认分配，不安全则回滚

**安全性算法（核心）**：

```
Work = Available        // 工作向量
Finish[i] = false       // 所有进程未完成

循环：
    找一个进程Pi，满足：
        Finish[i] == false  AND  Need[i] <= Work
    如果找到：
        Work = Work + Allocation[i]
        Finish[i] = true
        记录Pi到安全序列
        继续循环
    如果找不到：
        退出循环

如果所有Finish[i] == true → 安全状态
否则 → 不安全状态
$$

$$
\text{安全} \Leftrightarrow \exists \text{安全序列} P_{i_1}, P_{i_2}, \ldots, P_{i_n} \text{ 使得每个进程都能执行完毕}
$$

**计算示例**：

系统有3类资源A(10), B(5), C(7)，当前状态：

| 进程 | Max | Allocation | Need |
|------|-----|------------|------|
| P0 | 7,5,3 | 0,1,0 | 7,4,3 |
| P1 | 3,2,2 | 2,0,0 | 1,2,2 |
| P2 | 9,0,2 | 3,0,2 | 6,0,0 |
| P3 | 2,2,2 | 2,1,1 | 0,1,1 |
| P4 | 4,3,3 | 0,0,2 | 4,3,1 |

Available = (10,5,7) - (7,2,5) = **(3,3,2)**

安全性检查：
1. Work=(3,3,2)，P1: Need(1,2,2)≤Work → 满足，Work=(3,3,2)+(2,0,0)=(5,3,2), Finish[P1]=true
2. Work=(5,3,2)，P3: Need(0,1,1)≤Work → 满足，Work=(5,3,2)+(2,1,1)=(7,4,3), Finish[P3]=true
3. Work=(7,4,3)，P0: Need(7,4,3)≤Work → 满足，Work=(7,4,3)+(0,1,0)=(7,5,3), Finish[P0]=true
4. Work=(7,5,3)，P2: Need(6,0,0)≤Work → 满足，Work=(7,5,3)+(3,0,2)=(10,5,5), Finish[P2]=true
5. Work=(10,5,5)，P4: Need(4,3,1)≤Work → 满足，Work=(10,5,5)+(0,0,2)=(10,5,7), Finish[P4]=true

安全序列：**P1 → P3 → P0 → P2 → P4**（不唯一）

## 直观理解

**银行贷款的类比**：
银行（操作系统）有10万元资金，贷款给多个客户（进程）。每个客户声明自己最多需要多少钱（Max），已经借了多少（Allocation），还需要多少（Need）。银行在放贷前先模拟：如果我借出去这笔钱，能不能保证所有客户最终都能还清？能则借，不能则让客户等。

## 知识关联

**与死锁预防的关系（第39节）**：
- 预防是静态限制（破坏必要条件）
- 避免是动态检查（不破坏条件，但确保安全）

**与死锁检测的关系（第42节）**：
- 避免：每次分配前检查（开销大但更安全）
- 检测：定期检查（开销小但可能死锁后再处理）

**408常见考法**：
- 给出系统状态，要求执行安全性检查
- 给出资源请求，要求判断是否分配（需先试探再检查）
- 问"不安全状态是否等于死锁"（不等于，但不能保证不死锁）
- 安全序列可能不唯一，考试通常要求写出任意一个
