# 50_基本分段段表与二维逻辑地址

## 核心概念

- **分段管理（Segmentation）**：按程序的逻辑结构划分内存为若干段
- **段**：有逻辑意义的单位（代码段、数据段、栈段等）
- **段表（Segment Table）**：记录每个段的基址和段长
- **段表项**：由"段基址"和"段长"组成
- **二维逻辑地址**：逻辑地址由"段号+段内偏移"两部分组成
- **段号**：标识是哪个段
- **段内偏移**：段内的位置

**408核心考点**：
- 分段的逻辑地址是二维的（对比分页的一维地址）
- 段表项包含基址和段长（页表项只有页框号）
- 地址转换过程和越界检查
- 分段 vs 分页的对比

## 原理分析

**分段的逻辑地址结构**：

$$
\text{逻辑地址} = (\text{段号}S, \text{段内偏移}W)
$$

若段号16位，段内偏移16位：
- 最大段数：$2^{16} = 65536$ 段
- 每段最大长度：$2^{16} = 65536$ 字节 = 64KB

注意：不同段的长度可以不同（这是与分页的根本区别）。

**段表结构**：

| 段号 | 段基址（物理起始地址） | 段长 |
|------|---------------------|------|
| 0 | 1000 | 500 |
| 1 | 5000 | 300 |
| 2 | 8000 | 1000 |
| ... | ... | ... |

**地址转换过程**：

```
逻辑地址(S, W)
    |
    v
检查: S < 段表长度?  → 否 → 段号越界中断
    | 是
    v
查段表: 取出 Base = 段表[S].基址, Limit = 段表[S].段长
    |
    v
检查: W < Limit?  → 否 → 段内偏移越界中断
    | 是
    v
物理地址 = Base + W
```

$$
\text{物理地址} = \text{段表}[S].\text{基址} + W
$$

$$
\text{越界条件: } S \geq \text{段表长度} \quad \text{或} \quad W \geq \text{段表}[S].\text{段长}
$$

**分段的两次越界检查**：段号越界 + 段内偏移越界（分页只有一次：页号越界）。

**分段 vs 分页对比**：

| 对比项 | 分页 | 分段 |
|--------|------|------|
| 划分依据 | 物理等大小（系统行为） | 逻辑意义（用户/编译器行为） |
| 地址维度 | 一维（页号+偏移） | 二维（段号+段内偏移） |
| 块大小 | 固定，由硬件决定 | 不固定，各段大小不同 |
| 内存碎片 | 内部碎片 | 外部碎片 |
| 表项内容 | 页框号 | 基址 + 段长 |
| 越界检查 | 页号≥页表长度 | 段号≥段表长度 或 偏移≥段长 |
| 对程序员透明 | 是（透明） | 否（需了解段结构） |
| 信息共享 | 不方便（按页共享） | 方便（按段共享，如共享代码段） |
| 动态增长 | 不支持 | 支持（如栈段自动增长） |

## 直观理解

**书籍的类比**：
- 分页就像把一本书按固定50页一刀切，不管是不是完整章节（物理划分，系统行为）
- 分段就像按章节自然划分——第一章、第二章...（逻辑划分，用户行为）
- 段表就像目录：告诉你每章从第几页开始，有多长

**二维地址的类比**：
分段地址就像"(大楼号, 楼层号)"，两部分都有意义。分页地址就像"线性编号"，你只需要一个数字就能定位。

## 知识关联

**与分页的关系（第47节）**：
- 分段和分页各有优缺点，段页式结合了两者（第51节）
- 分段方便共享和保护，分页消除了外部碎片

**与段页式管理的关系（第51节）**：
- 段页式 = 先分段再分页
- 逻辑地址为三维：(段号, 页号, 偏移)

**408常见考法**：
- 给出段表，将逻辑地址转换为物理地址
- 问"分段的逻辑地址为什么是二维的"
- 比较分段和分页的区别（简答题高频）
- 问"分段有什么好处"（共享、保护、符合用户视角）
- 问"分段有什么缺点"（外部碎片、管理复杂）
