# 31_PV操作模板与互斥同步分离

## 核心概念

- **互斥PV模板**：mutex信号量初值=1，保护临界区
- **同步PV模板**：信号量初值=0，协调执行顺序
- **前驱关系**：操作A必须在操作B之前完成
- **互斥与同步分离原则**：互斥信号量和同步信号量不能混用
- **P操作顺序**：先P同步信号量，再P互斥信号量（否则可能死锁）
- **V操作顺序**：先V互斥信号量，再V同步信号量（通常无影响，但推荐此顺序）

**408核心考法**：
- 给定多个进程的执行顺序要求，写出PV操作
- 互斥PV和同步PV的组合使用

## 原理分析

**互斥PV模板**：

```
Semaphore mutex = 1;

进程Pi:
    P(mutex);         // 申请进入临界区
    临界区;             // 访问共享资源
    V(mutex);         // 释放临界区
```

**同步PV模板（两进程，A先B后）**：

```
Semaphore S = 0;      // 初值0，表示"信号尚未产生"

进程A:                  进程B:
    执行操作A;             P(S);         // 等A完成
    V(S);                 执行操作B;
```

**关键原则：P操作的顺序**

场景：进程需要同时满足同步和互斥

```
Semaphore S = 0;       // 同步信号量
Semaphore mutex = 1;   // 互斥信号量

正确顺序：                错误顺序：
P(S);                    P(mutex);    ← 先拿了锁
P(mutex);                P(S);        ← 再等同步，但锁不释放
  临界区;                    死锁！
V(mutex);
V(S);
```

$$
\text{死锁原因：} \quad P_{mutex} \text{先获取锁} \rightarrow P_S \text{阻塞} \rightarrow \text{锁不释放} \rightarrow \text{其他进程无法V(S)}
$$

**前驱关系的PV实现**：

如果有多个前驱关系 S1→S2, S1→S3, S2→S4, S3→S4：

```
Semaphore a=0, b=0, c=0;  // 三条边对应三个信号量

S1:  语句1; V(a); V(b);          // S1完成后通知S2和S3
S2:  P(a); 语句2; V(c);          // 等S1完成后执行，再通知S4
S3:  P(b); 语句3; V(c);          // 等S1完成后执行，再通知S4
S4:  P(c); P(c); 语句4;          // 等S2和S3都完成
```

**每个前驱关系对应一个初值为0的同步信号量**。

**互斥同步分离总结**：

| 类型 | 信号量初值 | P操作位置 | V操作位置 |
|------|-----------|-----------|-----------|
| 互斥 | 1 | 临界区入口 | 临界区出口 |
| 同步 | 0 | 后续操作之前 | 前驱操作之后 |

## 直观理解

**接力赛的类比（同步）**：
- 每个信号量S=0就像一根接力棒
- 前一个跑者（操作A）跑完后"递棒"（V(S)）
- 后一个跑者（操作B）必须"接棒"（P(S)）才能出发
- 棒还没有递过来，你就必须等

**P操作顺序的类比**：
先P互斥再P同步，就像你先进了会议室把门反锁（拿了mutex），然后等一个重要的人来（P同步）。但门反锁了，那个人进不来给你V(S)，你就永远等下去——死锁！

## 知识关联

**与经典同步问题的关系**：
- 哲学家进餐、读者-写者、生产者-消费者都是PV模板的具体应用
- 每个问题都需要正确分离互斥和同步信号量

**与管程的关系（第35节）**：
- 管程自动处理互斥，程序员只需关注同步
- PV操作需要手动管理两种信号量，容易出错

**408常见考法**：
- 给出前驱图（有向图），写出对应的PV操作序列
- 分析PV操作序列是否存在死锁（P操作顺序错误）
- 问"同步信号量初值为什么是0"
- 给出多个进程代码，分析执行结果
