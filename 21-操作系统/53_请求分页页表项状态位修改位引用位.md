# 53_请求分页页表项状态位修改位引用位

## 核心概念

- **请求分页页表项**：在基本分页的基础上增加了控制位和外存地址
- **状态位（存在位/P位）**：页面是否在内存中（1=在内存，0=不在）
- **访问位（引用位/R位）**：页面最近是否被访问过（用于LRU近似和Clock算法）
- **修改位（脏位/M位）**：页面调入内存后是否被修改过（决定换出时是否写回外存）
- **外存地址**：该页面在外存中的位置
- **有效/无效位**：等同于状态位

**408核心考点**：
- 各位的含义和作用
- 缺页中断的触发条件（状态位=0）
- 修改位决定换出时是否需要写回（修改了才写回，减少I/O）

## 原理分析

**请求分页的页表项结构**：

| 页号 | 页框号 | 状态位P | 访问位R | 修改位M | 外存地址 |
|------|--------|---------|---------|---------|----------|
| 0 | 5 | 1 | 1 | 0 | Disk_0 |
| 1 | - | 0 | 0 | 0 | Disk_1 |
| 2 | 8 | 1 | 1 | 1 | Disk_2 |
| 3 | - | 0 | 0 | 0 | Disk_3 |

**状态位P（Present Bit）**：
- P=1：该页在内存中，页框号有效
- P=0：该页不在内存中，访问会产生**缺页中断**
- 缺页中断处理程序将页面从外存调入内存，然后修改P=1

**访问位R（Reference/Access Bit）**：
- 页面被访问时硬件自动置1
- 定期清零（由OS时钟中断处理）
- 用于近似LRU算法：R长时间为0说明很久没被访问
- Clock算法（NRU算法）的核心依据

**修改位M（Modify/Dirty Bit）**：
- 页面被写入时硬件自动置1
- 换出时的决策：
  - M=0：页面未修改，直接丢弃（无需写回外存，节省I/O）
  - M=1：页面已修改，必须写回外存后再丢弃

$$
\text{换出代价: } \begin{cases}
M=0 & \text{直接丢弃，} O(1) \\
M=1 & \text{写回外存} + \text{丢弃，} O(\text{disk\_io})
\end{cases}
$$

**地址转换过程（请求分页版）**：

```
逻辑地址 → 页号P + 偏移W
    |
    v
查页表: 页表[P]
    |
    v
状态位=1? → 是 → 物理地址 = 页框号 × 页面大小 + W
    | 否
    v
缺页中断 → 从外存调入页面 → 更新页表 → 重新执行指令
```

**访问位的管理**：

```
时钟中断处理（每N ms）:
    for each page:
        if R == 1:
            R = 0        // 清零，准备记录下一个周期的访问
        else:
            // 该页在本周期未被访问，可能是换出候选
```

## 直观理解

**酒店房间登记表的类比**：
- 状态位P：房间是否有人住（有人=1，空房=0）
- 访问位R：今天是否有人进出（有=1，没有=0）
- 修改位M：住客是否弄乱了房间（弄乱了需要打扫=1，保持原样=0）
- 外存地址：住客的备用住处（仓库地址）

换房时：如果房间保持原样（M=0），直接安排新住客；如果弄乱了（M=1），需要先打扫（写回外存）。

## 知识关联

**与缺页中断的关系（第54节）**：
- 状态位=0触发缺页中断
- 缺页中断处理中需要更新状态位、页框号

**与页面置换算法的关系**：
- 访问位R是Clock算法和近似LRU的基础
- 修改位M用于改进的Clock算法（优先换出R=0且M=0的页面）

**改进的Clock算法优先级**：

$$
(0,0) \to (0,1) \to (1,0) \to (1,1)
$$

$$
\text{优先换出: } R=0, M=0 \quad \text{（未访问且未修改）}
$$

**408常见考法**：
- 问"状态位/访问位/修改位各有什么用"
- 问"为什么修改位可以减少磁盘I/O"
- 给出页表状态，判断访问某地址是否产生缺页中断
- 问"Clock算法依据哪个位"（访问位R）
