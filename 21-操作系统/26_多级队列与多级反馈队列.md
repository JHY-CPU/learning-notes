# 26_多级队列与多级反馈队列

## 核心概念

- **多级队列调度（Multilevel Queue）**：将就绪队列划分为多个**独立队列**，每个队列有**固定优先级**和**独立调度算法**
- **多级反馈队列（Multilevel Feedback Queue, MFQ）**：在多级队列基础上增加**队列间迁移**机制
- **MFQ是通用系统中应用最广泛的调度算法**：兼顾响应时间、周转时间、吞吐量
- **MFQ的核心思想**：短作业在高层队列快速完成，长作业逐步下沉到低层队列

## 原理分析

### 多级队列调度（MQ）

```
高优先级 → 队列1（交互式进程）→ RR（q=8ms）
           队列2（交互式进程）→ RR（q=16ms）
           队列3（批处理进程）→ FCFS
低优先级 →
```

**规则**：
- 只有队列1空了才调度队列2，队列2空了才调度队列3
- **固定优先级**：高层队列可以抢占低层队列
- **各队列之间没有迁移**：进程在哪个队列是固定的

### 多级反馈队列（MFQ）

```
队列1（最高优先级）：时间片 q1 = 2ms    → FCFS
队列2（中等优先级）：时间片 q2 = 4ms    → FCFS
队列3（最低优先级）：时间片 q3 = 8ms    → FCFS（或RR/FCFS）
```

**设置规则**：
1. 各队列**优先级递减**（队列1最高）
2. 各队列**时间片递增**
3. 新进程进入**最高优先级**队列（队列1）
4. 进程在队列1的时间片内未完成 → **降级到队列2**
5. 进程在队列2的时间片内未完成 → **降级到队列3**
6. 只有高优先级队列空了才调度低优先级队列

**MFQ的三个参数**：
- $n$：队列数量
- $q_i$：第$i$个队列的时间片（$q_1 < q_2 < \cdots < q_n$）
- 调度策略：各队列内部的调度算法

### MFQ调度实例

| 进程 | 到达时间 | 服务时间 |
|------|---------|---------|
| P1 | 0 | 12 |
| P2 | 3 | 5 |

MFQ参数：队列1（q=2ms）、队列2（q=4ms）、队列3（FCFS）

t=0: P1在队列1运行2ms（剩10ms）→ 降级到队列2
t=2: P1在队列2运行4ms（剩6ms）→ 降级到队列3
t=3时P2到达：P2进入队列1
t=6: P1在队列3。但队列1有P2 → 调度队列1的P2
t=6-8: P2在队列1运行2ms（剩3ms）→ 降级到队列2
t=8-11: P2在队列2运行3ms（完成！）
t=11: 队列1空，队列2空 → 调度队列3的P1
t=11-17: P1在队列3运行6ms（完成）

**Gantt图**：
```
|P1|P1  |P2|P2 |P1    |
0  2    6 8  11      17
  q1   q2q1  q2    q3
``

### MFQ的关键特性

**兼顾性**：
- **短作业**：在高层队列快速完成 → 响应时间短
- **长作业**：逐步下沉到低层队列 → 不会频繁切换
- **I/O密集型**：I/O完成后回到高层队列 → 受到优待

**无饥饿**：
- 低优先级队列最终会被调度（因为高优先级队列会空）
- 可配合**老化机制**：长时间在低队列的进程提升到高队列

**防作弊**：
- 用户不能故意让进程用完时间片来获取更大的时间片（MFQ只在本队列给时间片，降级后时间片变大但优先级变低）

### MFQ vs 其他算法

| 特性 | FCFS | SJF | RR | MFQ |
|------|------|-----|-----|-----|
| 响应时间 | 差 | 中 | 好 | 好 |
| 周转时间 | 差 | 好 | 中 | 中 |
| 饥饿 | 无 | 有 | 无 | 无 |
| 实现复杂度 | 低 | 中 | 低 | 高 |
| 适用系统 | 批处理 | 批处理 | 分时 | 通用 |

## 直观理解

- **多级队列**：医院分诊台——危重病人（高优先级）直接进抢救室，普通病人排队
- **多级反馈队列**：学校考试分班——新生都进快班（高层队列），跟不上就降到普通班（低层队列），在普通班等了很久又升回快班（老化）
- **时间片递增**：快班每节课20分钟（时间片小，快速切换），普通班每节课45分钟（时间片大，减少切换）

## 知识关联

- **MFQ的高层队列**使用RR（第24个知识点），底层队列使用FCFS（第21个知识点）
- **MFQ是UNIX/Linux调度器的基础**（Linux CFS虽然不同，但思想有继承）
- **老化机制**防止低队列饥饿，关联第25个知识点
- **I/O密集型进程的优待**与优先级调度中的动态优先级调整思想一致
- **408高频考法**：
  - 计算题：给定MFQ参数画Gantt图
  - 简答题：MFQ如何兼顾短作业和长作业
  - 选择题：MFQ的三个参数及其影响
  - **MFQ是408最重要的调度算法**——常作为综合大题出现
- **常见陷阱**：
  - 新进程**总是进入最高优先级队列**
  - 降级是**不可逆的**（除非配合老化机制）
  - 高优先级队列空了才调度低优先级队列，**不是轮转调度各队列**
