## 操作系统解决什么问题

```
// ========== OS 的核心职责 ==========
// 1. 抽象硬件：CPU、内存、磁盘、网卡 → 统一 API（系统调用）
// 2. 多道程序：在有限资源上"同时"运行多个程序（并发 / 虚拟化）
// 3. 隔离与安全：进程地址空间、权限、防止相互破坏
// 4. 公平与性能：调度、缓存、I/O 合并，提高吞吐与响应
```

> **与「08-Linux与Shell」的关系**：Shell 是用户态工具；本模块讲内核如何管理进程、内存、文件与设备。

## 408 考纲要点（内存管理 + 文件系统 + I/O 管理）

### 第六章 内存管理

- 内存管理概念：逻辑地址、物理地址、重定位
- 连续分配管理方式：单一、固定分区、动态分区
- 非连续分配：基本分页、基本分段、段页式
- 虚拟内存概念、请求分页管理
- 页面置换算法：OPT、FIFO、LRU、Clock
- 页面分配策略与工作集模型

### 第七章 文件管理

- 文件逻辑结构：顺序、索引、直接
- 文件目录：FCB、索引节点、目录结构
- 文件保护与共享
- 文件系统层次结构
- 文件实现（外存分配）：连续、链接、索引
- 空闲空间管理

### 第八章 I/O 管理

- I/O 设备分类与 I/O 控制方式
- I/O 软件层次结构
- 设备分配与 SPOOLing 技术
- 磁盘结构与调度算法
- 缓冲管理
