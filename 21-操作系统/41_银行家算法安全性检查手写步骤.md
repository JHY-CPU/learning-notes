# 41_银行家算法安全性检查手写步骤

## 核心概念

- **考试手写模板**：标准化的表格计算流程
- **Work向量**：模拟可用资源，初始=Available
- **Need≤Work判断**：逐行检查是否有进程可执行
- **Work更新规则**：Work = Work + Allocation（进程执行完毕释放资源）
- **安全序列记录**：按执行顺序写出进程名

**408核心能力**：必须能快速、准确地完成表格计算

## 原理分析

**手写模板（表格法）**：

设系统有m类资源，n个进程。

```
步骤0: 填写初始表格
| 进程 | Max   | Allocation | Need   | Available |
|------|-------|------------|--------|-----------|
| P0   | ...   | ...        | ...    |           |
| P1   | ...   | ...        | ...    | (初始值)  |
| ...  |       |            |        |           |

步骤1: 计算Need = Max - Allocation

步骤2: 安全性检查（迭代填充下表）
| 轮次 | Work    | Need     | Need≤Work? | Allocation | New Work=Work+Alloc |
|------|---------|----------|------------|------------|---------------------|
| 1    | Avail   | Need[i]? | Yes→选Pi   | Alloc[i]   | Work+Alloc[i]       |
| 2    | 上轮结果 | Need[j]? | Yes→选Pj   | Alloc[j]   | ...                 |
| ...  |         |          |            |            |                     |
```

**详细计算示例**：

系统资源 A=10, B=5, C=7

初始状态：
| 进程 | Max(A,B,C) | Alloc(A,B,C) | Need(A,B,C) |
|------|------------|--------------|-------------|
| P0 | (7,5,3) | (0,1,0) | (7,4,3) |
| P1 | (3,2,2) | (2,0,0) | (1,2,2) |
| P2 | (9,0,2) | (3,0,2) | (6,0,0) |
| P3 | (2,2,2) | (2,1,1) | (0,1,1) |
| P4 | (4,3,3) | (0,0,2) | (4,3,1) |

Available = (10-7, 5-2, 7-5) = (3,3,2)

安全性检查：

| 轮次 | Work | 进程 | Need≤Work? | Alloc | New Work |
|------|------|------|------------|-------|----------|
| 1 | (3,3,2) | P0 | (7,4,3)>(3,3,2) No | - | - |
| 1 | (3,3,2) | P1 | (1,2,2)≤(3,3,2) Yes | (2,0,0) | (5,3,2) |
| 2 | (5,3,2) | P2 | (6,0,0)>(5,3,2) No | - | - |
| 2 | (5,3,2) | P3 | (0,1,1)≤(5,3,2) Yes | (2,1,1) | (7,4,3) |
| 3 | (7,4,3) | P0 | (7,4,3)≤(7,4,3) Yes | (0,1,0) | (7,5,3) |
| 4 | (7,5,3) | P2 | (6,0,0)≤(7,5,3) Yes | (3,0,2) | (10,5,5) |
| 5 | (10,5,5) | P4 | (4,3,1)≤(10,5,5) Yes | (0,0,2) | (10,5,7) |

安全序列：**P1 → P3 → P0 → P2 → P4**

**银行家算法完整流程（含资源请求）**：

假设P1请求Request=(1,0,2)：

1. Request(1,0,2) ≤ Need(1,2,2)? → Yes
2. Request(1,0,2) ≤ Available(3,3,2)? → Yes
3. 试探分配：
   - Available = (3,3,2)-(1,0,2) = (2,3,0)
   - Alloc[P1] = (2,0,0)+(1,0,2) = (3,0,2)
   - Need[P1] = (1,2,2)-(1,0,2) = (0,2,0)
4. 安全性检查...（安全则确认，不安全则回滚）

## 直观理解

**手写技巧**：
- 先计算Need = Max - Allocation，不要在检查时边算边查
- 每轮从头扫描进程表，找第一个满足Need≤Work的未完成进程
- 找到后立即更新Work并标记该进程完成
- 最后检查所有进程是否都完成

**易错点**：
- Need是每个维度都要≤Work的对应维度
- Work更新是加法不是减法
- 安全序列不唯一，考试写任意一个即可

## 知识关联

**与资源分配图化简的关系（第38节）**：
- 银行家算法适用于多实例资源
- 资源分配图化简适用于单实例资源
- 两者本质都是"模拟释放后能否满足所有需求"

**408常见考法**：
- 完整的表格计算（约10分大题）
- 给出资源请求后判断是否分配
- 问"安全序列是否唯一"（不唯一）
- 给出一个序列，验证它是否是安全序列
