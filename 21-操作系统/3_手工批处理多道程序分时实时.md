# 03_手工批处理多道程序分时实时

## 核心概念

- **发展脉络**：手工操作 → 单道批处理 → 多道批处理 → 分时系统 → 实时系统
- **手工操作阶段**：用户独占全机，人机矛盾突出（CPU利用率低）
- **单道批处理**：引入**监督程序**，自动依次处理作业，但内存中始终只有**一道**作业
- **多道批处理**：内存中同时存放多道程序，**宏观并行、微观交替**，首次体现OS的并发和共享特征
- **分时系统**：将CPU时间划分为时间片，轮流分配给各终端用户，**交互性好**
- **实时系统**：在规定时间内完成特定任务，**及时性、可靠性**要求高
  - **硬实时**：必须在截止时间前完成（如导弹控制）
  - **软实时**：尽量满足截止时间（如视频播放）

## 原理分析

### 多道批处理的关键技术

1. **中断技术**：使CPU能在I/O完成后及时切换到另一道程序
2. **通道技术**：使I/O设备能独立工作，CPU与I/O并行

**多道批处理的优缺点**：
- 优点：资源利用率高、系统吞吐量大
- 缺点：**无交互性**，用户提交作业后不能干预

### 分时系统的特征

1. **多路性（同时性）**：多个终端用户同时使用一台计算机
2. **独立性**：各用户独立操作，互不干扰
3. **及时性**：用户请求在可接受时间内得到响应（通常1-3秒）
4. **交互性**：用户可通过终端与系统对话

**时间片轮转原理**：
$$\text{响应时间} = \text{时间片} \times \text{终端用户数}$$

### 各类型对比

| 特征 | 多道批处理 | 分时系统 | 实时系统 |
|------|-----------|---------|---------|
| 设计目标 | 吞吐量 | 交互性 | 及时性 |
| 交互性 | 无 | 好 | 有限 |
| 作业调度 | 作业调度算法 | 时间片轮转 | 优先级/截止时间 |
| 典型应用 | 科学计算 | 终端分时 | 控制系统 |

## 直观理解

- **手工操作**：一个人用一台洗衣机，洗完才走，下一个人来用
- **单道批处理**：送洗店，店员按顺序处理，但一次只洗一件
- **多道批处理**：送洗店有10台洗衣机，同时洗多批衣服，但你放下衣服就得走，不能中途改要求
- **分时系统**：自助餐，厨师轮流给每个窗口出菜，每个窗口等一小会儿就能得到服务
- **硬实时系统**：消防队接警后必须在5分钟内到达，否则就是失败

## 知识关联

- **多道批处理**是真正意义上的OS的开始，体现了并发和共享
- **中断技术**（多道批处理的基础）关联第8个知识点"中断异常系统调用"
- **时间片轮转**是分时系统的核心调度算法，关联第24个知识点（RR调度）
- **实时调度**的最早截止时间优先（EDF）关联第25个知识点（优先级调度）
- **408考法**：
  - 选择题：各发展阶段的特征对比
  - 简答题：多道批处理如何提高CPU利用率
  - 常见陷阱：分时系统**不一定**满足实时要求；多道批处理**没有**交互性
