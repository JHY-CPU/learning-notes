# 09_进程定义与特征

## 核心概念

- **进程（Process）**：是程序的**一次执行过程**，是系统进行**资源分配和调度的基本单位**
- **程序**：静态的，存放在磁盘上的指令集合
- **进程**：动态的，是程序在某数据集上的运行过程
- **进程 vs 程序**：
  - 程序是**静态**文本，进程是**动态**执行
  - 一个程序可对应**多个**进程（多次运行）
  - 一个进程可执行**多个**程序（exec调用）
- **进程的特征**（5个，常考）：
  1. **动态性**（最基本）：进程有创建、执行、消亡的生命周期
  2. **并发性**：多个进程可同时存在于内存中并发执行
  3. **独立性**：进程是独立运行、独立获得资源的基本单位
  4. **异步性**：各进程按各自独立的、不可预知的速度推进
  5. **结构性**：每个进程由程序段、数据段、PCB三部分组成

## 原理分析

### 进程的组成

```
进程 = PCB（进程控制块） + 程序段 + 数据段
```

- **PCB**：进程存在的唯一标志，OS通过PCB感知和管理进程
- **程序段**：进程执行的代码（可被多个进程共享）
- **数据段**：进程执行所需的数据

### 进程与程序的数学关系

$$\text{进程} = f(\text{程序}, \text{数据}, \text{PCB})$$

同一程序运行在不同数据集上 = 不同进程（如多个Word进程编辑不同文档）。

### 进程是资源分配的基本单位

- 每个进程有独立的**地址空间**
- 每个进程有独立的**打开文件表**
- 每个进程有独立的**信号处理方式**

注意：进程是**资源分配**的基本单位，但不一定是**CPU调度**的基本单位（线程才是）。

## 直观理解

- **程序** = 菜谱（静态的文字说明）
- **进程** = 按照菜谱做菜的过程（动态的执行过程）
- 同一本菜谱可以被不同厨师同时使用 → 一个程序对应多个进程
- 做菜的过程需要灶台（CPU）、食材（数据）、操作台（内存） → 进程需要分配资源
- **动态性**是最本质的特征：菜谱本身不等于做菜，只有执行起来才叫"做菜"
