# 59_文件逻辑结构顺序索引直接

## 核心概念

- **文件逻辑结构**：从用户角度看到的文件组织方式，与物理存储无关。
- 三种基本逻辑结构：**顺序文件**、**索引文件**、**直接文件（散列文件）**。
- 逻辑结构决定用户如何读写和检索文件中的记录。
- 408 考点：三种逻辑结构的特点对比，适用场景。

## 原理分析

### 顺序文件（Sequential File）

**定义**：记录按某种顺序（关键字或进入顺序）排列，读写必须从头到尾顺序进行。

**两种排列方式**：
- **串结构**：记录按进入时间排列，无序。查找只能从头顺序扫描。
- **顺序结构**：记录按关键字排列（如学号递增），可用折半查找。

**特点**：
- 批量读写效率最高（磁盘顺序 I/O）。
- 逐条查找效率低（串结构 $O(n)$，顺序结构 $O(\log n)$）。
- 增删记录困难，通常需要复制整个文件。

**适用场景**：日志文件、备份、批量处理。

### 索引文件（Indexed File）

**定义**：为文件建立一个索引表，每个索引项记录一个记录的关键字和物理位置。

**结构**：
```
索引表：
  关键字 → 物理地址（块号 + 偏移）
  [张三, 块5偏移0]
  [李四, 块12偏移200]
  [王五, 块3偏移50]
```

**特点**：
- 支持随机访问：先查索引表，再直接定位到记录。
- 索引表本身可以是顺序文件，支持折半查找。
- 索引表需要额外存储空间。
- 适合频繁的随机查询操作。

**多级索引**：当记录数很大时，索引表本身也很大，可以为索引表再建索引（二级索引）。

### 直接文件（Direct / Hash File）

**定义**：通过哈希函数将记录的关键字映射到物理地址。

$$\text{物理地址} = H(\text{关键字})$$

**特点**：
- 理想情况下 $O(1)$ 访问速度。
- 存在哈希冲突，需要冲突解决（链地址法、开放定址法）。
- 不支持顺序遍历（记录在物理上无序）。
- 访问速度依赖于哈希函数质量和冲突率。

**适用场景**：需要快速随机访问的场景，如数据库索引。

### 三种逻辑结构对比

| 特性 | 顺序文件 | 索引文件 | 直接文件 |
|------|---------|---------|---------|
| 查找方式 | 顺序/折半 | 索引表 | 哈希 |
| 顺序访问 | 极优 | 可以 | 不支持 |
| 随机访问 | 差 | 好 | 极优 |
| 空间开销 | 无额外开销 | 索引表 | 哈希溢出 |
| 增删记录 | 困难 | 中等 | 中等 |
| 典型应用 | 批处理日志 | 数据库表 | 散列表 |

## 直观理解

- **顺序文件** = 按编号排队的学生名单，想找到特定学生只能一个一个看或折半找。
- **索引文件** = 图书馆的检索卡片柜，先查卡片（索引）找到书的位置，再去取。
- **直接文件** = 按身份证号算出来的柜号，知道号就能直接开柜。
