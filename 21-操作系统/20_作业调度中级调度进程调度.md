# 20_作业调度中级调度进程调度

## 核心概念

- **三级调度体系**：高级调度（作业调度）、中级调度（内存调度）、低级调度（进程调度）
- **高级调度（Long-term Scheduling / 作业调度）**：从外存后备队列中选择作业调入内存
- **中级调度（Medium-term Scheduling / 对换调度）**：将内存中暂时不能运行的进程调出到外存（挂起）
- **低级调度（Short-term Scheduling / 进程调度）**：从就绪队列中选择一个进程分配CPU——**最基本、最频繁**

## 原理分析

### 高级调度（作业调度）

**执行时机**：
- 内存空间不足，需要从外存调入新作业
- 作业调度程序被触发

**功能**：
- 决定接纳多少作业（多道程序度）
- 决定接纳哪些作业（从后备队列选）

**特点**：
- 发生频率**最低**（几分钟甚至几小时一次）
- 主要出现在**批处理系统**（分时/实时系统通常没有）
- 调度对象是**作业**

### 中级调度（内存调度）

**执行时机**：
- 内存使用率过高时
- 需要给紧急作业腾出空间

**功能**：
- 将暂时不能运行的进程**挂起**（调出到外存对换区）
- 将挂起的进程**激活**（调回内存）

**特点**：
- 目的是**提高内存利用率**和**系统吞吐量**
- 挂起的进程处于"挂起就绪"或"挂起阻塞"状态
- 发生频率**中等**

### 低级调度（进程调度）

**执行时机**：
1. 运行 → 终止（当前进程结束）
2. 运行 → 阻塞（I/O请求/等待信号量）
3. 运行 → 就绪（时间片用完/被抢占）
4. 就绪 → 运行（被调度，但只有就绪队列非空时）

注意：**2和4一定会发生调度；1和3不一定**（取决于是否有就绪进程）

**功能**：
- 从就绪队列中选择一个进程
- 分配CPU（上下文切换）

**特点**：
- 发生频率**最高**（ms级别）
- **每个OS都必须配置**
- 调度对象是**进程/线程**

### 三级调度对比

| 对比项 | 高级调度 | 中级调度 | 低级调度 |
|-------|---------|---------|---------|
| 调度对象 | 作业 | 进程 | 进程/线程 |
| 发生频率 | 最低 | 中等 | 最高 |
| 位置 | 外存→内存 | 内存↔外存 | 就绪队列→CPU |
| 功能 | 决定哪些作业进内存 | 挂起/激活进程 | 决定哪个进程用CPU |
| 是否必须 | 批处理必须 | 可选 | 必须 |

## 直观理解

- **高级调度**：大学招生办——决定录取哪些新生（作业）入学（内存）
- **中级调度**：教务处——暂时让某些学生休学（挂起到外存），恢复时复学（调回内存）
- **低级调度**：班主任——每节课决定让哪个学生（进程）上台发言（使用CPU）
