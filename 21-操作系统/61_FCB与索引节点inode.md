# 61_FCB与索引节点inode

## 核心概念

- **FCB（File Control Block，文件控制块）**：存放文件元数据的数据结构，是目录项的核心内容。
- **inode（索引节点）**：Unix/Linux 中的 FCB 实现，将文件描述信息与文件名分离。
- **目录项（Directory Entry / dentry）**：目录中的一个条目，包含文件名和对应的 FCB/inode 号。
- 408 考点：FCB 的组成、inode 的结构、为什么要引入索引节点。

## 原理分析

### FCB 的内容

一个 FCB 通常包含三类信息：

**基本信息**：
- 文件名
- 文件类型（普通文件、目录、设备文件等）
- 文件物理位置（磁盘块号、索引块号等）
- 文件逻辑结构（顺序、索引等）

**存取控制信息**：
- 文件主（owner）
- 访问权限（读、写、执行）
- 共享信息（链接计数）

**使用信息**：
- 文件创建时间
- 最后修改时间
- 最后访问时间
- 当前文件大小
- 文件最大允许大小

### 目录的实现

目录本身也是一个文件，其内容是一系列目录项：

**传统目录项**（包含完整 FCB）：
```
[目录文件内容]
  文件名1 | FCB1（包含所有元数据 + 物理位置）
  文件名2 | FCB2（包含所有元数据 + 物理位置）
  ...
```

**问题**：目录查找需要逐个读取目录项并比较文件名，每次都要读入完整的 FCB（可能数百字节），效率低。

### 索引节点（inode）

**核心思想**：将 FCB 中除文件名以外的信息集中存储在一个称为 inode 的数据结构中，目录项只存储文件名和 inode 号。

```
[目录项（精简）]
  文件名 | inode号

[inode 表（独立区域）]
  inode号 | 文件类型 | 权限 | 物理位置 | 时间戳 | 链接计数 | ...
```

**inode 结构**（以 Unix inode 为例）：

```c
struct inode {
    // 文件基本信息
    mode_t    i_mode;      // 文件类型和权限
    uid_t     i_uid;       // 文件主 ID
    off_t     i_size;      // 文件大小（字节）
    time_t    i_atime;     // 最后访问时间
    time_t    i_mtime;     // 最后修改时间
    time_t    i_ctime;     // inode 最后修改时间
    nlink_t   i_nlink;     // 硬链接计数

    // 磁盘块指针
    block_t   i_block[15]; // 12个直接 + 1个一级间接 + 1个二级间接 + 1个三级间接
};
```

**目录项大小对比**：
- 传统 FCB 目录项：文件名(14B) + FCB(~64B) ≈ 78B
- inode 目录项：文件名(14B) + inode号(2B) ≈ 16B

一个磁盘块（假设 1KB）可以存放：
- 传统方式：~13 个目录项
- inode 方式：~64 个目录项

目录查找时只需读取更少的块，效率提高约 5 倍。

### inode 的块指针结构

以 1KB 块大小、4B 块号为例：

- **12 个直接块指针**：直接指向数据块，覆盖 $12 \times 1\text{KB} = 12\text{KB}$
- **1 个一级间接块指针**：指向一个索引块，索引块包含 256 个块号，覆盖 $256 \times 1\text{KB} = 256\text{KB}$
- **1 个二级间接块指针**：指向一个索引块，每个条目指向一个一级索引块，覆盖 $256 \times 256 \times 1\text{KB} = 64\text{MB}$
- **1 个三级间接块指针**：覆盖 $256^3 \times 1\text{KB} = 16\text{GB}$

$$\text{最大文件大小} = 12\text{KB} + 256\text{KB} + 64\text{MB} + 16\text{GB} \approx 16\text{GB}$$

### 磁盘上的 inode 布局

```
[超级块] [inode 位图] [数据块位图] [inode 表] [数据块区]
```

- **inode 位图**：标记哪些 inode 已使用。
- **inode 表**：存放所有 inode，连续排列。
- inode 区域通常位于磁盘的固定位置。

## 直观理解

- **FCB** = 一份文件的"身份证"，记录文件的一切元信息。
- **inode** = 把身份证的"内容"单独存放，身份证号（inode号）写在目录卡片上。
- 这样翻目录（找文件名）就不用翻身份证了——目录变薄，查找变快。

## 知识关联

- 与 60 目录结构：目录项就是目录文件中的记录，inode 是 FCB 的一种高效实现。
- 与 63 文件共享：硬链接 = 多个目录项指向同一个 inode（nlink 计数 +1）。
- 与 65 磁盘分配：inode 中的块指针决定了文件的物理存储方式（索引分配）。
- 与 66 混合索引：inode 的直接/间接块指针就是混合索引的典型实现。
- 408 考查：计算 inode 方式下的最大文件大小、寻址次数。
