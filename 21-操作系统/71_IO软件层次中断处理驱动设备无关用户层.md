# 71_IO软件层次中断处理驱动设备无关用户层

## 核心概念

- **I/O 软件层次结构**：将 I/O 相关的软件划分为若干层次，每层对上层提供标准接口。
- 四层模型（从下到上）：**中断处理程序**、**设备驱动程序**、**设备独立软件（设备无关层）**、**用户层 I/O 软件**。
- 层次化设计使新增设备只需编写驱动程序，无需修改上层代码。
- 408 考点：各层的功能、数据流向、设备独立性。

## 原理分析

### I/O 软件四层模型

```
┌─────────────────────────────────────┐
│  用户层 I/O 软件                     │  printf, scanf, fopen, fread
├─────────────────────────────────────┤
│  设备独立软件（设备无关层）            │  缓冲管理、设备保护、错误处理
├─────────────────────────────────────┤
│  设备驱动程序                        │  具体设备的 I/O 命令序列
├─────────────────────────────────────┤
│  中断处理程序                        │  响应硬件中断，唤醒等待进程
├─────────────────────────────────────┤
│  硬件（设备控制器 + 设备）            │  实际的数据传输
└─────────────────────────────────────┘
```

### 第 1 层：中断处理程序

**功能**：
- 响应 I/O 设备完成中断。
- 保存被中断进程的现场（上下文）。
- 执行设备特定的中断处理（如读取缓冲区数据）。
- 唤醒等待该 I/O 完成的进程。
- 恢复被中断进程的现场（或调度其他进程）。

**工作流程**：
```
硬件中断
  → 保存现场（压栈 PC、PSW、寄存器）
  → 识别中断源（哪个设备）
  → 跳转到对应设备的中断处理程序
  → 执行处理（读数据、清中断标志、唤醒进程）
  → 恢复现场或调度
  → 返回用户态
```

### 第 2 层：设备驱动程序

**功能**：
- 接收上层的 I/O 请求（逻辑操作：读/写/定位等）。
- 将逻辑请求翻译为设备能理解的**具体命令**序列。
- 检查设备状态，设置设备控制器的寄存器。
- 启动 I/O 操作。
- I/O 完成后，通过中断通知上层。

**关键特性**：
- **每个设备对应一个驱动程序**（设备专用）。
- 驱动程序运行在**内核态**。
- 驱动程序了解设备的硬件细节（寄存器地址、命令格式等）。

**典型的驱动程序接口**：
```c
struct device_driver {
    int  (*open)(struct device *dev);
    int  (*close)(struct device *dev);
    int  (*read)(struct device *dev, char *buf, size_t count);
    int  (*write)(struct device *dev, const char *buf, size_t count);
    int  (*ioctl)(struct device *dev, unsigned int cmd, unsigned long arg);
};
```

### 第 3 层：设备独立软件（设备无关层）

**功能**：
- 提供**统一的 I/O 接口**给上层（不论底层是什么设备）。
- **缓冲管理**：统一的数据缓冲区管理。
- **设备保护**：检查访问权限。
- **错误处理**：报告和处理 I/O 错误。
- **设备命名与分配**：逻辑设备名到物理设备名的映射。
- **独占设备的管理**：处理设备的独占/共享。

**设备无关性的体现**：
- 用户程序写 `read(fd, buf, n)` 而不需要知道 fd 对应的是磁盘、键盘还是网卡。
- 文件系统操作不依赖于底层存储设备类型。
- 同一份驱动接口规范适用于所有同类设备。

**块设备的统一接口**：
```
上层调用: read_block(dev, block_num, buffer)
         ↓
设备无关层: 检查参数 → 查缓存表 → 命中则返回
                              ↓ 未命中
                         调用驱动程序读取
```

### 第 4 层：用户层 I/O 软件

**功能**：
- 提供用户友好的 I/O 函数库（如 C 标准库的 `printf`、`scanf`、`fopen`、`fread`）。
- **Spooling 守护进程**：在用户态模拟独占设备的共享（如打印机假脱机）。
- 将用户的 I/O 请求转换为系统调用。

**典型流程**：
```
用户代码: printf("Hello\n")
  → 用户库: 格式化字符串
  → 系统调用: write(stdout, "Hello\n", 6)
  → 内核: 设备无关层 → 驱动 → 硬件
```

### 各层协作示例：读取磁盘文件

1. **用户层**：`fread(buf, 1, 100, fp)` → 库函数将请求翻译为 `read(fd, buf, 100)` 系统调用。
2. **设备无关层**：检查权限，计算文件偏移对应的逻辑块号，检查页缓存/缓冲区。
3. **设备驱动**：未命中缓存 → 驱动向磁盘控制器发送 READ 命令，设置 DMA 参数。
4. **硬件**：磁盘读取数据到内存缓冲区。
5. **中断处理**：DMA 完成 → 中断 → 唤醒等待的进程。
6. 数据从内核缓冲区拷贝到用户 buf → 返回。

## 直观理解

- **用户层** = 顾客（说"我要一杯咖啡"）。
- **设备无关层** = 咖啡店前台（不关心咖啡机型号，统一接单）。
- **驱动程序** = 咖啡师（知道怎么操作特定型号的咖啡机）。
- **中断处理** = 计时器响了（咖啡做好了，通知前台）。
- 每层只关心自己的事，层与层之间通过标准接口通信。
