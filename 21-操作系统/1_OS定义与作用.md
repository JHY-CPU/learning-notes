# 01_OS定义与作用

## 核心概念

- **操作系统（Operating System, OS）**：是计算机系统中最基本的**系统软件**，是配置在计算机硬件上的第一层软件，是对硬件系统的首次扩充。
- **OS的角色**：
  - **资源管理者**：管理CPU、内存、I/O设备等硬件资源的分配与回收
  - **用户与硬件的接口**：提供命令接口（CLI/GUI）和程序接口（系统调用）
  - **扩充机器（虚拟机）**：在裸机上覆盖OS后，向用户提供功能更强的虚拟机
- **三种接口**：
  - **命令接口**：联机命令接口（交互式）、脱机命令接口（批处理）
  - **程序接口（系统调用/广义指令）**：程序通过系统调用请求OS服务
  - **GUI**：图形用户界面，不是独立接口，是命令接口的图形化
- **408易混淆点**：
  - OS是**系统软件**，不是硬件也不是应用软件
  - OS是**资源管理者**，但资源管理是手段，方便用户才是目的
  - 系统调用是程序请求OS服务的**唯一途径**

## 原理分析

### 操作系统的定义（王道教材表述）

OS是指**控制和管理**整个计算机系统的**硬件和软件资源**，合理地**组织、调度**计算机的工作与资源的分配，进而为用户和其他软件提供方便接口与环境的程序集合。

### OS的基本作用

1. **处理机管理**：进程调度、上下文切换
2. **存储器管理**：内存分配、虚拟内存
3. **设备管理**：I/O调度、设备驱动
4. **文件管理**：文件存储、目录结构
5. **提供用户接口**：命令接口、程序接口

### 系统调用的执行过程

```
用户程序 → 系统调用指令 → 陷入(trap) → 内核态 → 执行系统服务 → 返回用户态
```

- 用户程序不能直接访问硬件，必须通过系统调用陷入内核态
- 执行系统调用时CPU从**用户态**切换到**核心态**

## 代码实现

### 系统调用示例——程序如何通过 OS 访问硬件

```c
#include <stdio.h>
#include <stdlib.h>
#include <fcntl.h>
#include <unistd.h>
#include <sys/stat.h>

/*
 * 系统调用是用户程序请求 OS 服务的唯一途径。
 * 用户程序不能直接访问硬件，必须通过陷入(trap)进入内核态。
 *
 * 下面演示几个常用系统调用的使用方式，
 * 每个函数调用都会触发一次 用户态→内核态 的切换。
 */

int main() {
    /* ====== 1. write 系统调用：向标准输出写数据 ======
     * 用户程序调用 write() → 陷入内核 → 内核操作显卡/终端 → 返回
     */
    const char* msg = "通过 write 系统调用输出\n";
    write(STDOUT_FILENO, msg, 27);

    /* ====== 2. open/read/close 系统调用：文件操作 ======
     * 用户程序不能直接读磁盘，必须请求内核代为操作
     */
    int fd = open("/etc/hostname", O_RDONLY);  /* 陷入内核，请求打开文件 */
    if (fd < 0) {
        perror("open");
    } else {
        char buf[256];
        ssize_t n = read(fd, buf, sizeof(buf) - 1);  /* 陷入内核，读取数据 */
        if (n > 0) {
            buf[n] = '\0';
            printf("读取到的内容: %s", buf);
        }
        close(fd);  /* 陷入内核，关闭文件 */
    }

    /* ====== 3. getpid 系统调用：获取进程信息 ======
     * PID 存储在内核的 PCB 中，用户程序只能通过系统调用获取
     */
    pid_t pid = getpid();
    printf("当前进程 PID = %d（通过系统调用获取）\n", pid);

    /* ====== 4. stat 系统调用：获取文件元信息 ======
     * 文件的 inode 信息在内核中管理
     */
    struct stat st;
    if (stat("/etc/hostname", &st) == 0) {
        printf("文件大小: %ld 字节\n", st.st_size);
        printf("文件 inode: %ld\n", st.st_ino);
    }

    return 0;
}
```

### Shell 中观察系统调用——strace 工具

```bash
#!/bin/bash
# strace 可以追踪程序执行时的所有系统调用
# 这是理解"用户态→内核态切换"的最佳实践工具

# 追踪 ls 命令的系统调用
echo "=== ls 命令的系统调用追踪 ==="
strace -e trace=openat,read,write,close ls /tmp 2>&1 | head -20

echo ""
echo "=== 统计系统调用次数 ==="
strace -c ls /tmp 2>&1 | tail -10

# 输出示例：
# openat(AT_FDCWD, "/tmp", O_RDONLY|O_NONBLOCK) = 3
# read(3, "..." , 32768)                  = 192
# write(1, "file1\nfile2\n", 12)           = 12
# close(3)                                = 0
#
# 说明：每个系统调用都是一次 用户态→内核态→用户态 的切换
```

## 直观理解

把OS想象成一个**大管家**：
- **资源管理者**：管家掌管家里的钥匙（CPU、内存、外设），谁需要什么资源都找管家申请
- **接口**：管家是主人（用户）和佣人（硬件）之间的桥梁，主人不需要直接指挥佣人
- **虚拟机**：有了管家，主人看到的是一个"什么都会做"的家，而不用关心佣人怎么工作

## 知识关联

- **与计算机组成原理关联**：OS管理的硬件资源（CPU寄存器、Cache、主存、磁盘）在计组中有详细硬件结构
- **与系统调用的关系**：系统调用是OS为应用程序提供的唯一编程接口，是第一章和第二章（进程管理）的桥梁
- **408考法**：
  - 选择题：判断OS属于什么类型的软件、OS的作用和功能
  - 简答题：比较命令接口与程序接口、说明OS的基本特征
- **常见陷阱**：OS不管理编译器（编译器是系统程序，但不是OS的一部分）；数据库管理系统不是OS的组成部分
