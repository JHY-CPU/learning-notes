# 46_动态分区首次适应邻近适应最佳最坏

## 核心概念

- **首次适应（First Fit, FF）**：从头扫描，找到第一个足够大的空闲分区
- **邻近适应（Next Fit, NF）**：从上次分配的位置继续向后扫描
- **最佳适应（Best Fit, BF）**：找到最小的够用的空闲分区
- **最坏适应（Worst Fit, WF）**：找到最大的空闲分区
- **空闲分区表/链**：记录所有空闲分区的数据结构

**408核心考点**：
- 四种算法的工作过程
- 对比各算法的优缺点
- 分配和回收时的分区合并

## 原理分析

**首次适应（First Fit）**：

- 空闲分区按**地址递增**排列
- 从低地址向高地址查找第一个满足大小要求的分区
- 低地址区产生大量小碎片

```
空闲链: [10K@1000] → [30K@5000] → [20K@10000]
申请15K: 找到30K@5000（第一个≥15K的）
分配后: [10K@1000] → [15K@5000(剩余)] → [20K@10000]
```

优点：综合性能最好，实现简单。缺点：低地址碎片多。

**邻近适应（Next Fit）**：

- 空闲分区按地址递增排列
- 从上次分配结束的位置开始查找
- 避免低地址碎片堆积，但可能遗漏低地址的空闲块

优点：空闲分区分布均匀。缺点：大分区不易保留。

**最佳适应（Best Fit）**：

- 空闲分区按**容量递增**排列
- 找到最小的满足要求的分区
- 每次只用"刚刚好"的分区，保留大分区

```
空闲链: [10K] → [20K] → [30K] → [50K]  (按大小升序)
申请15K: 找到20K（最小的≥15K）
```

优点：保留大分区。缺点：产生大量极小碎片（无法再用），查找慢。

**最坏适应（Worst Fit）**：

- 空闲分区按**容量递减**排列
- 找到最大的空闲分区分配
- 分配后剩余部分可能仍然较大

优点：剩下的空间比较大，不容易产生小碎片。缺点：大分区消耗快，大进程无处安放。

**四种算法对比**：

| 算法 | 排序方式 | 查找方向 | 优点 | 缺点 |
|------|---------|---------|------|------|
| 首次适应 | 地址递增 | 从头 | 综合最优 | 低地址碎片多 |
| 邻近适应 | 地址递增 | 从上次位置 | 碎片均匀 | 大分区难保留 |
| 最佳适应 | 容量递增 | 从头 | 保留大分区 | 小碎片多 |
| 最坏适应 | 容量递减 | 从头 | 碎片大 | 大分区消耗快 |

**内存回收的合并**：

回收分区时，检查是否与相邻空闲分区相邻，若相邻则合并：

```
情况1: 回收区与前一个空闲区相邻 → 合并到前一个
情况2: 回收区与后一个空闲区相邻 → 合并到后一个
情况3: 回收区前后都相邻 → 三个合并为一个
情况4: 回收区前后都不相邻 → 新增一个空闲分区
```

## 直观理解

**找座位的类比**：
- **首次适应**：进电影院从第一排开始找空位
- **邻近适应**：从你上次坐的排开始往后找
- **最佳适应**：找一个刚好够你坐的最小空位（不浪费大空间）
- **最坏适应**：找个最大的空位坐下（剩下的空间还能坐人）

## 知识关联

**与连续分配的关系（第45节）**：
- 四种算法都是动态分区分配的具体策略

**与分页管理的关系（第47节）**：
- 分页管理不再需要这些算法，因为空闲页框是等大的
- 用位示图或空闲链表即可管理

**408常见考法**：
- 给出空闲分区列表和分配请求，模拟四种算法的分配过程
- 比较四种算法哪个碎片最多/最少
- 问"实际系统中最常用哪种"（首次适应）
- 内存回收时的合并操作
