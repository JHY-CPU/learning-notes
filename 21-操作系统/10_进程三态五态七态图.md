# 10_进程三态五态七态图

## 核心概念

- **三态模型**（最基本）：就绪态、运行态、阻塞态
- **五态模型**（加入创建和终止）：创建态、就绪态、运行态、阻塞态、终止态
- **七态模型**（加入挂起）：创建、就绪、运行、阻塞、终止 + 挂起就绪、挂起阻塞
- **核心状态**：
  - **就绪态（Ready）**：已获得除CPU外的所有资源，等待调度
  - **运行态（Running）**：正在CPU上执行
  - **阻塞态（Blocked/Waiting）**：因等待某事件（I/O完成、信号量等）而暂停

## 原理分析

### 状态转换图（三态 + 创建/终止）

```
            创建
             ↓
    ┌────→ 就绪态 ←────┐
    │      ↓ 调度      │ 时间片用完
    │    运行态 ────────┘
    │      ↓ I/O请求/等待事件
    │    阻塞态
    │      ↓ I/O完成/事件发生
    │      就绪态
    │
    └──→ 终止态
```

**七种转换及其触发条件**：

| 转换 | 触发条件 | 谁做的 |
|------|---------|--------|
| 创建→就绪 | 进程创建完成 | OS |
| 就绪→运行 | 进程调度 | 调度程序 |
| 运行→就绪 | 时间片用完/更高优先级进程到达 | 时钟中断/调度程序 |
| 运行→阻塞 | 请求I/O/等待信号量/P操作阻塞 | 进程自身 |
| 阻塞→就绪 | I/O完成/信号量释放/V操作唤醒 | OS（中断处理） |
| 运行→终止 | 正常结束/异常终止/被杀死 | 进程自身/OS |
| 就绪→挂起就绪 | 内存紧张，换出到外存 | OS |
| 挂起就绪→就绪 | 内存空间恢复 | OS |

### 挂起状态的本质

- **挂起（Suspend）**：进程被调出内存到外存（磁盘对换区）
- **挂起就绪**：在外存等待，一旦调入内存就变成就绪态
- **挂起阻塞**：在外存等待事件，事件发生后变为挂起就绪
- 挂起是**中级调度**的核心操作

### 关键转换条件总结

$$\text{就绪} \xrightarrow{\text{调度}} \text{运行} \quad \text{（获得CPU）}$$
$$\text{运行} \xrightarrow{\text{时间片到}} \text{就绪} \quad \text{（失去CPU，但一切就绪）}$$
$$\text{运行} \xrightarrow{\text{I/O请求}} \text{阻塞} \quad \text{（主动等待事件）}$$
$$\text{阻塞} \xrightarrow{\text{I/O完成}} \text{就绪} \quad \text{（被动唤醒）}$$

## 直观理解

- **就绪态**：运动员在起跑线准备就绪，裁判还没喊"跑"——万事俱备只欠CPU
- **运行态**：运动员正在跑道上比赛
- **阻塞态**：运动员跑到一半，鞋带散了蹲下来系（等待事件），系好后不能直接继续跑，要先回到起跑线排队（回到就绪态）
- **挂起**：运动员被叫到场外休息（换到外存），等场地空了再进来

## 知识关联

- **阻塞态→就绪态**由OS自动完成（中断驱动），但进程**不能**从阻塞态直接到运行态，必须先到就绪态
- **阻塞和唤醒必须成对出现**，详见第12个知识点
- **挂起**对应中级调度，关联第20个知识点
- **时间片用完**导致运行→就绪，关联RR调度（第24个知识点）
- **408高频考点**：
  - 各状态转换的触发条件
  - 阻塞态不能直接到运行态
  - 挂起与阻塞的区别
- **常见陷阱**：
  - 进程**自己**可以导致运行→阻塞（主动），但**不能**自己导致阻塞→就绪（被动，需要OS/中断）
  - 时间片用完后进程回到**就绪队列末尾**
