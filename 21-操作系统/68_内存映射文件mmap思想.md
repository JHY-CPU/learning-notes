# 68_内存映射文件mmap思想

## 核心概念

- **内存映射文件（Memory-Mapped File）**：将文件直接映射到进程的虚拟地址空间，使文件操作变为内存访问。
- 系统调用：`mmap()`（Unix/Linux）、`CreateFileMapping()`（Windows）。
- 核心思想：利用虚拟内存机制，让文件 I/O 变成普通的内存读写。
- 408 考点：mmap 的原理、与传统 I/O 的对比、共享内存与 mmap 的关系。

## 原理分析

### 传统文件 I/O 的流程

```
进程地址空间              内核缓冲区              磁盘
  用户 buffer  ←─拷贝──→  内核 buffer  ←─I/O──→  文件数据块
```

**read 操作**：
1. 用户态发出 `read(fd, buf, n)`。
2. 内核将数据从磁盘读入内核缓冲区（页缓存 / Page Cache）。
3. 内核将数据从内核缓冲区拷贝到用户空间 buf。
4. 涉及 **2 次数据拷贝**（磁盘→内核→用户）和 **2 次上下文切换**。

### mmap 的工作原理

```
进程虚拟地址空间                    物理内存              磁盘
  ┌──────────────┐
  │ 代码段        │
  ├──────────────┤
  │ 数据段        │
  ├──────────────┤
  │ 堆           │
  ├──────────────┤
  │ 栈           │
  ├──────────────┤
  │ 映射区域      │──→ 页表映射  ──→ 物理页框 ←── 按需从磁盘调入
  │ (mmap 区域)   │                  (Page Cache)
  └──────────────┘
```

**mmap 流程**：
1. `addr = mmap(NULL, length, PROT_READ|PROT_WRITE, MAP_SHARED, fd, 0)`
2. 内核在进程虚拟地址空间中分配一段区域，**不立即读入文件数据**。
3. 进程首次访问映射区域中的页面时，触发**缺页异常**。
4. 缺页处理程序从磁盘读入对应的文件块到物理内存页框。
5. 页表项更新，进程直接访问该物理页框。
6. 进程对映射区域的修改直接写入物理页框，内核在适当时机将脏页写回磁盘。

**关键点**：
- **延迟加载（Lazy Loading）**：只在真正访问时才从磁盘调入数据。
- **零拷贝（Zero Copy）**：用户空间直接访问 Page Cache 中的数据，无需内核→用户拷贝。
- **按页管理**：以页为单位（通常 4KB），与虚拟内存系统共享页表和页缓存。

### mmap 的参数

```c
void *mmap(
    void *addr,      // 建议映射的起始地址（NULL = 内核选择）
    size_t length,   // 映射长度（字节）
    int prot,        // 保护：PROT_READ, PROT_WRITE, PROT_EXEC
    int flags,       // MAP_SHARED（共享修改）/ MAP_PRIVATE（写时复制）
    int fd,          // 文件描述符
    off_t offset     // 文件偏移（必须页对齐）
);
```

**MAP_SHARED vs MAP_PRIVATE**：
- **MAP_SHARED**：对映射区域的修改写回文件，其他映射同一文件的进程可见。适合进程间共享。
- **MAP_PRIVATE**：使用写时复制（COW），修改不影响原文件和其他进程。适合只读加载或私有修改。

### mmap vs 传统 read/write

| 特性 | 传统 I/O | mmap |
|------|---------|------|
| 数据拷贝 | 2 次（内核→用户） | 0 次（直接访问页缓存） |
| 系统调用次数 | 每次 read/write 一个系统调用 | 一次 mmap，后续无需系统调用 |
| 缓存管理 | 用户需自己管理 buffer | 内核统一管理页缓存 |
| 随机访问 | 需多次 lseek + read | 直接用指针访问 |
| 进程间共享 | 需要管道、消息队列等 | 多进程 mmap 同一文件即共享 |
| 适用场景 | 流式 I/O、小文件 | 大文件随机访问、进程共享 |

### 共享内存的实现

mmap 配合 MAP_SHARED 是实现 System V 共享内存的底层机制之一：

```
进程 A: mmap(shared_fd, MAP_SHARED)  ──→  物理页框 X  ←──  mmap(shared_fd, MAP_SHARED) : 进程 B
          虚拟页 A1                          共享                        虚拟页 B1
```

两个进程的不同虚拟页映射到同一个物理页框，一方的写入立即对另一方可见。

## 直观理解

- **传统 I/O** = 你去图书馆借书，管理员先从书架上拿下来，再复印一份给你。
- **mmap** = 管理员直接告诉你"那本书在第3排第5架上"，你自己去翻。不需要复印。
- **MAP_SHARED** = 多个人一起翻同一本书，你做的笔记别人也能看到。
- **MAP_PRIVATE** = 每人一份影印本，你在自己的副本上画线不影响别人。
