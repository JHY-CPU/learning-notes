# 08_中断异常系统调用

## 核心概念

- **中断（Interrupt）**：来自CPU**外部**的事件，由硬件产生（如I/O完成、时钟中断）
- **异常（Exception）**：来自CPU**内部**的事件，由执行指令引起（如除零、缺页、越界）
- **系统调用（System Call）**：用户程序主动请求OS服务，通过**陷入指令（trap/int）**触发
- **三者统称"广义中断"**，都会导致CPU从用户态切换到内核态
- **中断分类**：
  - **外中断（硬中断）**：I/O中断、时钟中断——可屏蔽
  - **内中断（软中断/异常）**：故障（fault，可恢复）、终止（abort，不可恢复）、陷入（trap，自愿）
- **中断向量表**：中断类型号 → 中断处理程序入口地址的映射表

## 原理分析

### 中断处理流程

```
1. CPU每执行完一条指令，检查是否有中断信号
2. 检测到中断 → 关中断（防止新中断干扰）
3. 保存当前进程的上下文（PC、PSW、通用寄存器）到内核栈/PCB
4. 根据中断类型号查中断向量表，找到中断处理程序入口
5. 执行中断处理程序
6. 恢复上下文
7. 开中断
8. 返回用户态继续执行（或调度其他进程）
```

### 异常的分类与处理

| 类型 | 触发原因 | 处理方式 | 举例 |
|------|---------|---------|------|
| 故障（Fault） | 可恢复的错误 | 修复后**重新执行**该指令 | 缺页异常 |
| 陷入（Trap） | 有意的 | 执行完后继续**下一条**指令 | 系统调用、断点 |
| 终止（Abort） | 不可恢复的错误 | 终止进程 | 除零、非法指令 |

### 系统调用的执行过程

```
用户程序
  ↓ 传递参数（通过寄存器/栈/内存块）
  ↓ 执行陷入指令（trap/int 0x80/syscall）
CPU检测到陷入
  ↓ 硬件自动切换到内核态
  ↓ 保存现场（PC、PSW）
  ↓ 查系统调用入口表
OS内核执行系统调用服务
  ↓ 恢复现场
  ↓ 切换回用户态
返回用户程序继续执行
```

### 中断与异常的关键区别

$$\text{中断} = \text{外部硬件触发，与当前执行指令无关}$$
$$\text{异常} = \text{内部指令触发，与当前执行指令直接相关}$$

## 直观理解

- **中断**：你在专心看书（执行指令），突然门铃响了（外部事件），你必须放下书去开门
- **异常**：你在看书，发现书上有一页被撕掉了（内部问题），你必须修复这个问题
- **系统调用**：你主动举手向老师请教问题（自愿请求服务）
- **中断向量表**：门上贴了一张表，写着"门铃响→去开门"、"电话响→去接电话"——查表确定该做什么

## 知识关联

- **中断是多道批处理OS的基础**：没有中断，CPU无法在I/O等待时切换到其他进程
- **时钟中断**是实现时间片轮转调度的硬件基础，关联第24个知识点
- **缺页异常**是虚拟内存的实现机制，关联第6章
- **系统调用**是OS为用户程序提供的唯一接口，关联第1个知识点
- **中断处理时需关中断**，防止嵌套中断导致栈溢出
- **408高频考点**：
  - 判断中断/异常/系统调用的分类
  - 缺页异常重新执行，系统调用执行下一条
  - 中断处理过程中关中断的意义
- **常见陷阱**：
  - 系统调用属于**内中断（陷入）**，不是外中断
  - 缺页异常的处理后**重新执行**原指令（不是下一条）
  - **所有中断/异常/系统调用都会引起用户态→内核态切换**

## 代码实现

### 信号处理——模拟中断机制

```c
#include <stdio.h>
#include <signal.h>
#include <unistd.h>
#include <stdlib.h>

/*
 * Linux 信号（Signal）是用户态对中断/异常机制的模拟。
 *
 * 信号源分类（对应中断分类）：
 * - 外中断（硬件）：SIGALRM（时钟）、SIGIO（I/O完成）
 * - 内中断（异常）：SIGSEGV（段错误）、SIGFPE（除零）
 * - 陷入（自愿）  ：SIGINT（Ctrl+C，用户主动触发）
 *
 * 处理流程与中断处理类似：
 * 1. 检测信号 → 2. 保护现场 → 3. 查信号处理表 → 4. 执行处理函数 → 5. 恢复现场
 */

volatile sig_atomic_t alarm_count = 0;

/* 时钟中断处理函数——类似内核的时钟中断处理程序 */
void alarm_handler(int sig) {
    alarm_count++;
    printf("[中断处理] 收到 SIGALRM 信号（时钟中断），第 %d 次\n",
           alarm_count);

    /* 模拟：每收到一次时钟中断，重新设置定时器 */
    if (alarm_count < 3) {
        alarm(1);  /* 1秒后再次触发 */
    } else {
        printf("[中断处理] 已处理3次时钟中断，退出\n");
        exit(0);
    }
}

/* 异常处理函数——缺页异常的简化模拟 */
void segv_handler(int sig) {
    printf("[异常处理] 段错误（SIGSEGV）—— 类似缺页异常\n");
    printf("[异常处理] 缺页异常处理后会重新执行触发异常的指令\n");
    _exit(1);  /* 实际OS会修复后重新执行，这里直接退出 */
}

/* 除零异常的演示 */
void fpe_handler(int sig) {
    printf("[异常处理] 浮点异常（SIGFPE）—— 类似除零异常\n");
    printf("[异常处理] 这类异常通常不可恢复，终止进程\n");
    _exit(1);
}

int main() {
    /* 注册信号处理函数（类似设置中断向量表） */
    signal(SIGALRM, alarm_handler);   /* 时钟中断 */
    signal(SIGSEGV, segv_handler);    /* 段错误异常 */
    signal(SIGFPE,  fpe_handler);     /* 算术异常 */

    printf("=== 信号处理模拟中断机制 ===\n");
    printf("进程 PID = %d\n\n", getpid());

    /* 触发时钟中断 */
    printf("设置 1 秒定时器（模拟时钟中断）...\n");
    alarm(1);

    /* 主循环：模拟用户程序执行 */
    int i = 0;
    while (1) {
        printf("用户程序执行中... (i=%d)\n", i++);
        sleep(1);
    }

    return 0;
}
```

### 系统调用的陷入过程——x86 汇编视角

```nasm
; 系统调用的底层实现（x86-64 Linux）
; 用户程序通过 syscall 指令陷入内核态
;
; 执行过程：
;   1. 将系统调用号放入 rax 寄存器
;   2. 将参数放入 rdi, rsi, rdx 等寄存器
;   3. 执行 syscall 指令
;   4. CPU 自动：
;      - 保存用户态 RIP → RCX, RFLAGS → R11
;      - 切换到内核态（设置 CPL=0）
;      - 跳转到内核入口（IA32_LSTAR MSR 指定的地址）
;   5. 内核执行系统调用服务
;   6. 通过 sysret 指令返回用户态

section .text
global _start

_start:
    ; write(1, msg, len) 系统调用
    mov rax, 1          ; sys_write 的系统调用号
    mov rdi, 1          ; 参数1: 文件描述符（stdout）
    mov rsi, msg        ; 参数2: 缓冲区地址
    mov rdx, msg_len    ; 参数3: 字节数
    syscall             ; 陷入内核态 → 执行 sys_write → 返回用户态

    ; exit(0) 系统调用
    mov rax, 60         ; sys_exit 的系统调用号
    xor rdi, rdi        ; 参数: 退出码 0
    syscall

section .data
    msg: db "Hello from syscall!", 10
    msg_len: equ $ - msg
```

```bash
# 编译和运行汇编示例
# nasm -f elf64 syscall.asm -o syscall.o
# ld syscall.o -o syscall
# ./syscall
#
# 也可以用 strace 观察系统调用过程：
# strace ./syscall
# 输出: write(1, "Hello from syscall!\n", 20) = 20
#       exit(0) = ?
```
