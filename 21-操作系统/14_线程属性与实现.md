# 14_线程属性与实现

## 核心概念

- **线程的属性**：
  - 线程是CPU调度的基本单位
  - 每个线程有一个唯一的线程标识符（TID）和线程控制块（TCB）
  - 线程有三种基本状态：就绪、运行、阻塞
  - 同一进程的各线程共享进程的资源
  - 线程切换的系统开销远小于进程切换
  - 线程可以并发执行（多核环境下真正并行）
- **线程控制块（TCB）**：比PCB小，包含线程ID、寄存器、栈指针、状态等
- **线程的实现方式**：用户级线程、内核级线程、组合方式

## 原理分析

### 用户级线程（User-Level Thread, ULT）

```
用户空间：线程库（Thread Library）管理线程的创建、切换
         TCB在用户空间，OS不可见
内核空间：OS只看到进程（内核级线程数 = 1）
```

**特点**：
- 线程的切换在**用户态**完成，不需要内核参与
- 线程调度由**线程库**（用户程序）决定
- OS**不知道**用户级线程的存在

**优点**：
- 线程切换**速度快**（不涉及内核态切换）
- 线程调度算法**可定制**
- **跨平台**（不依赖OS内核支持）

**缺点**：
- 一个线程阻塞（系统调用）→ **整个进程阻塞**（因为OS不知道还有其他线程）
- **无法利用多核CPU**（OS只看到一个内核线程）

### 内核级线程（Kernel-Level Thread, KLT）

```
用户空间：应用程序
内核空间：OS直接管理线程的创建、切换、调度
         每个线程有独立的TCB（在内核中）
```

**特点**：
- 线程的创建、切换、调度由**OS内核**完成
- OS**直接感知**每个线程
- 线程调度是**内核级调度**

**优点**：
- 一个线程阻塞，**不影响**同进程的其他线程
- **可以利用多核CPU**（内核可将不同线程分配到不同CPU）

**缺点**：
- 线程切换需要**用户态→内核态**，开销大
- 线程的创建/撤销需要**系统调用**，开销大

### 对比总结

| 对比项 | 用户级线程 | 内核级线程 |
|-------|----------|----------|
| 管理者 | 线程库（用户态） | OS内核 |
| 切换速度 | 快（用户态） | 慢（需要内核切换） |
| 阻塞影响 | 整个进程阻塞 | 仅该线程阻塞 |
| 多核利用 | 不能 | 能 |
| 创建开销 | 小 | 大 |
| OS可见性 | 不可见 | 可见 |

## 直观理解

- **用户级线程**：一个大家庭住在一栋房子里（一个进程），家里事务由家长管理（线程库），外人（OS）只看到一栋房子。如果家长出门办事（系统调用阻塞），全家都得等
- **内核级线程**：每个家庭成员都在街道办有登记（OS直接管理），一个人出门办事不影响其他人

## 知识关联

- **用户级线程的"阻塞牵连"问题**催生了多线程模型，详见第16、17个知识点
- **Java的Thread**通过JVM映射到内核级线程（一对一模型）
- **Linux的pthread**是内核级线程
- **Windows的线程**也是内核级线程
- **408考法**：
  - 选择题：用户级线程/内核级线程的特点对比
  - 简答题：为什么用户级线程一个阻塞会导致整个进程阻塞
  - 常见陷阱：**用户级线程的切换不需要内核参与**，但**系统调用会陷入内核**；用户级线程不能利用多核CPU
