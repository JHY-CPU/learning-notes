# 73_缓冲区单缓冲双缓冲循环缓冲缓冲池

## 核心概念

- **缓冲区（Buffer）**：在内存中开辟的一块区域，用于暂存 I/O 数据，缓解 CPU 与设备之间的速度差异。
- 四种缓冲技术：**单缓冲**、**双缓冲**、**循环缓冲**、**缓冲池**。
- 缓冲的作用：缓和 CPU 与 I/O 设备速度不匹配、减少中断频率、提高并行性。
- 408 高频考点：单缓冲和双缓冲的时间计算。

## 原理分析

### 单缓冲（Single Buffer）

**原理**：系统在内存中为设备设置一个缓冲区。

**工作流程（输入为例）**：
1. 设备将数据送入缓冲区（$T$ 时间）。
2. 缓冲区满后，OS 将数据从缓冲区传送到用户区（$M$ 时间）。
3. CPU 处理数据（$C$ 时间）。
4. 返回步骤 1。

**处理一块数据的时间**：

当 $C > T$ 时（CPU 处理比输入慢）：
$$\text{每块总时间} = T + M$$
- 设备连续工作，CPU 利用率受限于处理速度。

当 $C < T$ 时（CPU 处理比输入快）：
$$\text{每块总时间} = T + M$$
- 设备是瓶颈。

**单缓冲的问题**：设备和 CPU 不能同时访问缓冲区（互斥），处理 $n$ 块数据的总时间为：
$$n \times (T + M)$$

### 双缓冲（Double Buffer / Buffer Swapping）

**原理**：设置两个缓冲区，设备和 CPU 各用一个，交替工作。

```
缓冲区 A: [设备正在填入...]
缓冲区 B: [CPU 正在处理...]
当 A 填满、B 处理完 → 交换角色
```

**工作流程**：
1. 设备向缓冲区 A 输入数据，CPU 同时处理缓冲区 B 的数据。
2. A 满了，B 也处理完了 → 交换（A 给 CPU，B 给设备）。
3. 重复。

**处理一块数据的时间**：

当 $T > C + M$ 时（输入最慢）：
$$\text{每块总时间} = T$$

当 $T < C + M$ 时（CPU 处理最慢）：
$$\text{每块总时间} = C + M$$

$$\text{总时间} = \max(T, C+M) + (n-1) \times \max(T, C+M) = n \times \max(T, C+M)$$

**与单缓冲对比**：双缓冲实现了 CPU 和设备的**并行**，总时间取决于两者中最慢的。

**例题**：$T = 100\mu s$，$M = 50\mu s$，$C = 80\mu s$，处理 100 块数据。

- 单缓冲：$100 \times (100 + 50) = 15{,}000\mu s$
- 双缓冲：$100 \times \max(100, 50+80) = 100 \times 130 = 13{,}000\mu s$

### 循环缓冲（Circular Buffer）

**原理**：设置多个大小相等的缓冲区，组织成循环队列。

```
     [Buf0] → [Buf1] → [Buf2] → [Buf3] → [Buf4]
       ↑                                    |
       └────────────────────────────────────┘
```

**两个指针**：
- **输入指针 in**：设备将数据写入 `in` 指向的缓冲区，`in` 前进。
- **输出指针 out**：CPU 从 `out` 指向的缓冲区读取数据，`out` 前进。

**同步规则**：
- `in = out`：缓冲区空（设备来不及填）。
- `in` 追上 `out`：缓冲区满（CPU 来不及处理）。
- 缓冲区满时设备等待，缓冲区空时 CPU 等待。

**适用场景**：速度差异较大的 CPU 和设备、多设备并发 I/O。

### 缓冲池（Buffer Pool）

**原理**：OS 维护一个**公共缓冲区池**，由多个缓冲区组成，不专属任何设备。按需动态分配。

**缓冲区的三种队列**：
- **空闲缓冲区队列（emq）**：可供分配的缓冲区。
- **输入队列（inq）**：装满输入数据、等待 CPU 处理的缓冲区。
- **输出队列（outq）**：装满输出数据、等待设备输出的缓冲区。

**工作流程（输入）**：
1. 从 `emq` 取一个空闲缓冲区。
2. 设备将数据输入到该缓冲区。
3. 缓冲区满后，加入 `inq`。
4. CPU 从 `inq` 取缓冲区处理数据。
5. 处理完毕，缓冲区归还到 `emq`。

**四种缓冲区操作**：
- **take_buf(type)**：从指定队列取一个缓冲区。
- **add_buf(type, buf)**：将缓冲区加入指定队列。
- **get_buf(type)**：等待直到指定队列有缓冲区可用。
- **release_buf(buf)**：释放缓冲区回 `emq`。

**优点**：
- 缓冲区共享，利用率高。
- 灵活，可动态调整各队列大小。
- 现代 OS（Linux、Windows）广泛采用。

## 直观理解

- **单缓冲** = 一条传送带，货物到了先放传送带上，工人搬走后传送带才能接下一批。
- **双缓冲** = 两条传送带交替使用，工人搬一条的同时机器放下一批到另一条。
- **循环缓冲** = 旋转寿司传送带，厨师不断放盘子，顾客不断取。
- **缓冲池** = 公共充电宝租赁站，谁需要谁借，用完还回去，所有人共用。
