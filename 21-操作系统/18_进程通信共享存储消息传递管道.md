# 18_进程通信共享存储消息传递管道

## 核心概念

- **进程通信（IPC, Inter-Process Communication）**：进程之间的信息交换
- **进程间为什么需要特殊通信方式**：各进程地址空间独立，不能直接访问对方的变量
- **三大类IPC方式**：共享存储、消息传递、管道通信
- **共享存储**：
  - 基于数据结构的共享（低级，需要同步控制）
  - 基于存储区的共享（高级，共享内存区域）
- **消息传递**：
  - 直接通信方式（send/receive指定目标进程）
  - 间接通信方式（通过信箱/消息队列）
- **管道通信**：特殊的共享文件（pipe），半双工，需要互斥和同步

## 原理分析

### 共享存储（Shared Memory）

```
进程A的地址空间          共享内存区域          进程B的地址空间
 +-----------+         +-----------+         +-----------+
 |  私有数据  |  ←→    | 共享数据区 |   ←→   |  私有数据  |
 +-----------+         +-----------+         +-----------+
```

- OS在物理内存中开辟一块共享区域，映射到各进程的虚拟地址空间
- 进程直接读写共享区域（最快，无需OS介入）
- **必须配合同步机制**（信号量/互斥锁）防止数据竞争

$$\text{共享内存速度} = \text{内存读写速度（最快IPC方式）}$$

### 消息传递（Message Passing）

**直接通信**：
```
send(P, message)  —— 发送消息到进程P
receive(Q, message) —— 从进程Q接收消息
```
- 发送/接收时指定对方进程的标识符
- 链接关系自动建立（一对一）

**间接通信（信箱方式）**：
```
send(mailbox, message)  —— 发送消息到信箱
receive(mailbox, message) —— 从信箱接收消息
```
- 通过信箱（消息队列）中转
- 多个进程可共享一个信箱（多对多）

**消息的格式**：消息头（发送者ID、接收者ID、消息类型、长度）+ 消息体（数据）

### 管道通信（Pipe）

```
写端（进程A） → [数据流] → 读端（进程B）
               管道文件（循环缓冲区）
```

**管道的特征**：
1. **半双工**：数据只能单向流动（需要双向通信需两个管道）
2. **互斥**：同一时刻只能有一个进程读/写（内核自动加锁）
3. **同步**：
   - 管道空时读进程阻塞
   - 管道满时写进程阻塞
4. **面向字节流**：无消息边界
5. **生命周期与进程相同**：进程结束管道自动关闭

**管道容量**：通常4KB-64KB（Linux默认64KB）

### 三种方式对比

| 对比项 | 共享存储 | 消息传递 | 管道通信 |
|-------|---------|---------|---------|
| 通信方向 | 双向 | 双向 | 单向（半双工） |
| 速度 | 最快 | 较快 | 较慢 |
| 同步控制 | 需要（手动） | 需要（手动/自动） | 内核自动 |
| 数据格式 | 无限制（原始字节） | 结构化消息 | 字节流 |
| 亲缘关系 | 无要求 | 无要求 | 通常需要亲缘关系（匿名管道） |
| 代表 | mmap/SharedMemory | 消息队列/Socket | Linux pipe/FIFO |

## 直观理解

- **共享存储**：两个部门共用一个白板，谁都可以写和读（但需要约定谁先写谁先读）
- **消息传递（直接）**：你直接把文件递给我
- **消息传递（信箱）**：你把文件放进邮箱，我去邮箱取
- **管道**：一根水管连接两个房间，水只能往一个方向流，空了不能取，满了不能灌

## 知识关联

- **管道**关联Linux中的`|`（管道符），用于进程间连接
- **共享存储**需要同步机制，关联信号量和PV操作
- **消息传递**是微内核IPC的基础，关联第7个知识点（微内核）
- **socket通信**是跨网络的消息传递方式
- **408考法**：
  - 选择题：三种通信方式的特点对比
  - 简答题：管道通信的互斥和同步机制
  - 常见陷阱：**管道不是共享存储**（管道是特殊文件）；共享存储**必须**配合同步机制；管道通信是**半双工**不是全双工
