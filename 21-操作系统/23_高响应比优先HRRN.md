# 23_高响应比优先HRRN

## 核心概念

- **HRRN（Highest Response Ratio Next）**：调度时选择**响应比最高**的进程
- **响应比公式**：
$$R = \frac{\text{等待时间} + \text{服务时间}}{\text{服务时间}} = 1 + \frac{\text{等待时间}}{\text{服务时间}}$$
- **HRRN是非抢占式**的：一旦开始运行就运行到完成
- **设计目的**：兼顾短作业和长作业，避免SJF的饥饿问题
- **FCFS和SJF的折中**：
  - 等待时间相同 → 服务时间短的优先（偏向短作业，类似SJF）
  - 服务时间相同 → 等待时间长的优先（偏向先到达的，类似FCFS）

## 原理分析

### 响应比的动态变化

$$R = 1 + \frac{T_{\text{等待}}}{T_{\text{服务}}}$$

- 短作业：分母小，即使等待时间不长，R也较高 → 容易被调度
- 长作业：分母大，需要等待很长时间才能提高R → **最终一定能被调度**（不会饥饿）

$$\lim_{T_{\text{等待}} \to \infty} R = \infty$$

即：只要等得足够久，响应比趋向无穷大，一定能被调度。**HRRN无饥饿。**

### 计算实例

| 进程 | 到达时间 | 服务时间 |
|------|---------|---------|
| P1 | 0 | 4 |
| P2 | 1 | 3 |
| P3 | 2 | 5 |
| P4 | 3 | 2 |

**非抢占式HRRN调度过程**：

t=0：只有P1 → P1运行4ms
t=4：P1完成。此时各进程的响应比：
- P2：$R = \frac{(4-1)+3}{3} = \frac{6}{3} = 2.00$
- P3：$R = \frac{(4-2)+5}{5} = \frac{7}{5} = 1.40$
- P4：$R = \frac{(4-3)+2}{2} = \frac{3}{2} = 1.50$

选P2（响应比最高2.00）→ P2运行3ms

t=7：P2完成。此时：
- P3：$R = \frac{(7-2)+5}{5} = \frac{10}{5} = 2.00$
- P4：$R = \frac{(7-3)+2}{2} = \frac{6}{2} = 3.00$

选P4（响应比最高3.00）→ P4运行2ms

t=9：P4完成。运行P3（5ms）
t=14：P3完成

**Gantt图**：
```
|  P1  |  P2  | P4 |   P3   |
0      4      7   9       14
```

| 进程 | 完成 | 周转 | 等待 | 带权周转 |
|------|------|------|------|---------|
| P1 | 4 | 4 | 0 | 1.00 |
| P2 | 7 | 6 | 3 | 2.00 |
| P3 | 14 | 12 | 7 | 2.40 |
| P4 | 9 | 6 | 4 | 3.00 |

$$\text{平均周转} = \frac{4+6+12+6}{4} = 7.00$$
$$\text{平均等待} = \frac{0+3+7+4}{4} = 3.50$$

### HRRN的优缺点

**优点**：
- 兼顾长短作业（折中FCFS和SJF）
- **无饥饿**（长作业的响应比会持续增长）
- 非抢占式，上下文切换少

**缺点**：
- **不可抢占**：无法及时响应交互式任务
- 每次调度需计算所有就绪进程的响应比，开销较大
- 不适合实时/分时系统

## 直观理解

- HRRN像一个**智能客服**：短问题（短作业）优先回答，但如果你等了很久（长作业），即使问题复杂也会优先处理你——"等得越久越有优先权"
- SJF只看"问题复杂度"，HRRN还看"你等了多久"

## 知识关联

- HRRN是SJF的改进，解决了**饥饿问题**
- HRRN是非抢占式的，适用于**批处理系统**
- **分时系统**需要抢占式调度（RR），实时系统需要优先级调度
- 响应比 = 带权周转时间，关联第19个知识点
- **408考法**：
  - 计算题：HRRN下的响应比计算和调度结果
  - 选择题：HRRN的特点（非抢占、无饥饿）
  - 常见陷阱：**响应比要在每次调度时重新计算**（等待时间变了）；HRRN**不是**抢占式调度
