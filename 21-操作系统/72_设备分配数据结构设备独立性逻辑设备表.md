# 72_设备分配数据结构设备独立性逻辑设备表

## 核心概念

- **设备分配**：在多进程环境中，合理地将 I/O 设备分配给请求进程。
- **设备独立性**：用户程序使用逻辑设备名，系统负责映射到物理设备。
- **逻辑设备表（LUT）**：将逻辑设备名转换为物理设备名的数据结构。
- **设备控制表（DCT）**、**控制器控制表（COCT）**、**通道控制表（CHCT）**、**系统设备表（SDT）**。
- 408 考点：设备分配的数据结构、逻辑设备表的作用。

## 原理分析

### 设备分配的数据结构

**设备控制表（DCT - Device Control Table）**：每个设备一张。

```
DCT:
  设备标识符: /dev/sda1
  设备类型: 块设备
  设备状态: 忙碌 / 空闲
  指向控制器表的指针: → COCT #2
  重复执行次数/时间: 3（出错重试次数）
  等待队列: [进程A, 进程B]  ← 等待使用该设备的进程
```

**控制器控制表（COCT - Controller Control Table）**：每个设备控制器一张。

```
COCT:
  控制器标识符: 控制器 #2
  控制器状态: 忙碌 / 空闲
  指向通道表的指针: → CHCT #1（若有通道）
  指向相连设备的 DCT 链: → DCT(/dev/sda1), DCT(/dev/sda2)
  等待队列: [进程C]
```

**通道控制表（CHCT - Channel Control Table）**：每个 I/O 通道一张。

```
CHCT:
  通道标识符: 通道 #1
  通道状态: 忙碌 / 空闲
  指向控制器的 COCT 链: → COCT #2, COCT #3
  等待队列: [进程D]
```

**系统设备表（SDT - System Device Table）**：整个系统一张。

```
SDT:
  [0] 设备类型: 终端   物理名: /dev/tty0   驱动入口: → tty_driver  DCT: → DCT_tty0
  [1] 设备类型: 磁盘   物理名: /dev/sda1   驱动入口: → disk_driver  DCT: → DCT_sda1
  [2] 设备类型: 打印机  物理名: /dev/lp0   驱动入口: → lp_driver   DCT: → DCT_lp0
  ...
```

### 设备分配过程

1. 进程提出 I/O 请求（逻辑设备名 + 读/写）。
2. 系统查 LUT，将逻辑设备名映射到物理设备名。
3. 查 SDT 找到对应设备的 DCT。
4. 检查 DCT：设备是否空闲？
   - 空闲 → 分配该设备。
   - 忙碌 → 进程进入等待队列。
5. 分配设备后，沿 DCT 找到对应的 COCT。
6. 检查 COCT：控制器是否空闲？
   - 空闲 → 分配控制器。
   - 忙碌 → 进程进入控制器等待队列。
7. 如有通道，继续检查 CHCT。
8. 全部分配成功 → 启动 I/O 操作。

### 逻辑设备表（LUT）

**作用**：将逻辑设备名（用户使用）映射到物理设备名（系统内部）。

```
LUT:
  逻辑设备名        物理设备名
  /dev/printer     → /dev/lp0
  /dev/disk        → /dev/sda1
  /dev/terminal    → /dev/tty0
```

**设置时机**：
- 建立进程时设置（类似于文件描述符分配）。
- 第一次打开设备时设置。

**两种 LUT 设置方式**：
- **系统级 LUT**：所有进程共享同一张 LUT。
- **用户级 LUT**：每个进程维护自己的 LUT（更灵活，支持不同进程的 I/O 重定向）。

### 设备独立性的层次

**绝对独立性**：程序完全不关心设备类型（如读写任何存储设备）。
**相对独立性**：程序只关心设备大类（如需要一个"字符设备"，但不关心是键盘还是串口）。

### 设备分配策略

**静态分配**：进程运行前分配所有需要的设备，直到结束才释放。
- 优点：简单、不会死锁。
- 缺点：利用率低。

**动态分配**：进程运行中需要时才申请，用完立即释放。
- 优点：利用率高。
- 缺点：可能死锁（需配合死锁避免策略）。

## 直观理解

- **DCT** = 每台设备的"状态卡"：忙不忙、谁在等。
- **COCT** = 每个设备接口的"调度表"。
- **SDT** = 整个机房的"设备总清单"。
- **LUT** = "设备别名表"——你说"打印机"，系统找"HP LaserJet"。
