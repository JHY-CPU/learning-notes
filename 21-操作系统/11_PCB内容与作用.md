# 11_PCB内容与作用

## 核心概念

- **PCB（Process Control Block，进程控制块）**：OS用于描述进程状态和控制进程运行的数据结构
- **PCB是进程存在的唯一标志**：创建进程即创建PCB，撤销进程即撤销PCB
- **OS通过PCB感知进程**：OS维护的进程表（Process Table）就是PCB的集合

## 原理分析

### PCB中包含的信息（四大类）

**1. 进程标识信息**
- **PID**：进程标识号（唯一）
- **UID/GID**：用户标识号/组标识号
- **父进程标识**：PPID
- **进程组/会话ID**

**2. 处理机状态信息（CPU现场）**
- **通用寄存器**：R0, R1, ..., Rn
- **PC（程序计数器）**：下一条要执行的指令地址
- **PSW（程序状态字）**：标志位、状态位
- **SP（栈指针）**：用户栈指针
- **段表/页表基址寄存器**

**3. 进程调度信息**
- **进程状态**：就绪/运行/阻塞
- **优先级**：静态优先级/动态优先级
- **调度所需其他信息**：已等待时间、已运行时间
- **事件（阻塞原因）**：等待什么信号量、等待哪个I/O

**4. 进程控制信息**
- **程序和数据地址**：逻辑地址到物理地址的映射信息
- **进程同步和通信机制**：信号量、消息队列指针
- **资源清单**：打开文件表、已分配设备
- **链接指针**：PCB队列中的前驱/后继指针

### PCB的组织方式

1. **链接方式**：将同一状态的PCB组织成一个链表（就绪队列、阻塞队列）
2. **索引方式**：建立就绪索引表、阻塞索引表，表中存放PCB地址

```
就绪队列：PCB1 → PCB3 → PCB7 → NULL
阻塞队列（等待I/O）：PCB2 → PCB5 → NULL
阻塞队列（等待信号量）：PCB4 → PCB6 → NULL
```

### 进程切换时的上下文保存

$$\text{上下文} = \text{PCB中的处理机状态信息} = \text{PC + PSW + 寄存器}$$

进程切换 = 保存当前进程的上下文到其PCB + 恢复新进程的上下文从其PCB

## 直观理解

- **PCB** = 员工档案
  - 姓名、工号（PID）
  - 当前工作进度（PC）和工作状态（PSW）
  - 工龄、绩效（调度信息）
  - 使用的工具清单（资源清单）
- OS就像HR，通过档案（PCB）管理员工（进程）
- 员工离职 = 撤销档案 = 撤销进程

## 知识关联

- **PCB在创建进程时创建**，在进程终止时回收，关联第12个知识点
- **进程调度的本质**就是选择一个PCB，恢复其中保存的CPU现场
- **上下文切换**的开销取决于PCB中需要保存/恢复的信息量
- **线程的TCB**比PCB小（共享地址空间等信息），关联第13个知识点
- **408考法**：
  - 选择题：PCB包含哪些信息、判断哪项不在PCB中
  - 简答题：为什么说PCB是进程存在的唯一标志
  - 常见陷阱：**程序段**不在PCB中（PCB只是控制信息，不含代码本身）；**PC在PCB中**（中断时保存现场）
