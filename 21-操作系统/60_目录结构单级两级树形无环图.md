# 60_目录结构单级两级树形无环图

## 核心概念

- **文件目录**：实现文件名到文件物理位置映射的数据结构。
- 目录结构的设计目标：有效组织文件、支持快速查找、方便文件共享。
- 四种常见目录结构：单级目录、两级目录、树形目录、无环图目录。
- 408 考点：每种结构的特点、优缺点、适用场景。

## 原理分析

### 单级目录（Single-Level Directory）

**结构**：整个系统只有一个目录表，所有文件的 FCB 都放在同一张表中。

```
[目录表]
  文件名    其他信息
  a.txt     ...
  b.c       ...
  data.bin  ...
```

**优点**：
- 实现简单，查找方便（全表扫描）。
- 适合小型专用系统。

**缺点**：
- 不允许文件重名（所有文件共享一个命名空间）。
- 文件数多时查找效率低。
- 无法分类组织，用户使用不便。

### 两级目录（Two-Level Directory）

**结构**：为每个用户建立一个用户文件目录（UFD），系统维护一个主文件目录（MFD）。

```
[MFD 主文件目录]
  用户名 → UFD 指针
  张三   → UFD_张三
  李四   → UFD_李四

[UFD_张三]
  a.txt  ...
  b.c    ...

[UFD_李四]
  a.txt  ...   ← 与张三的 a.txt 不冲突
```

**优点**：
- 不同用户可以使用相同的文件名。
- 提高了查找速度（只需搜索当前用户的 UFD）。
- 用户之间隔离，易于权限管理。

**缺点**：
- 同一用户内部文件无法分类。
- 不够灵活，不适合多级目录需求。

### 树形目录（Tree-Structured Directory）

**结构**：目录组织成一棵树，根目录在顶部，每个非叶节点是一个目录，叶节点是文件。

```
        / (root)
       / | \
     bin etc home
         |    / \
       hosts  张三 李四
              / \
            docs code
                 |
               main.c
```

**优点**：
- 层次清晰，便于分类组织。
- 路径名唯一确定一个文件（绝对路径 / 相对路径）。
- 支持深层嵌套目录。

**缺点**：
- 文件共享困难（只能通过路径名引用）。
- 删除目录需要递归处理子目录。
- 一个文件只能属于一个目录（一个父节点）。

**路径表示**：
- 绝对路径：`/home/张三/docs/notes.txt`（从根出发）。
- 相对路径：`docs/notes.txt`（从当前目录出发）。

### 无环图目录（Acyclic-Graph Directory）

**结构**：允许目录中存在指向同一文件/目录的多个链接，形成有向无环图（DAG）。

```
        / (root)
       / | \
     bin etc home
              |
            张三
           /    \
         docs   shared  ← docs 和 shared 都指向同一个文件
           \    /
           notes.txt    ← 共享文件
```

**实现方式**：
- **硬链接**：多个目录项指向同一个 inode。
- **软链接（符号链接）**：一个目录项包含另一个文件的路径名。

**优点**：
- 支持文件共享，一个文件可以出现在多个目录中。
- 节省存储空间（共享同一份数据）。

**缺点**：
- 删除文件需谨慎：硬链接计数为 0 时才能真正删除。
- 可能出现悬挂指针（被共享的文件已被删除）。
- 可能形成环路（需检测和避免），环路会导致查找死循环。

### 四种目录结构对比

| 特性 | 单级 | 两级 | 树形 | 无环图 |
|------|------|------|------|--------|
| 重名问题 | 不允许 | 不同用户可同名 | 不同目录可同名 | 同上 |
| 文件共享 | 不支持 | 不支持 | 不便 | 支持 |
| 查找效率 | 低 | 中 | 高 | 高 |
| 实现复杂度 | 低 | 中 | 中 | 高 |
| 适用场景 | 极简系统 | 早期分时系统 | 现代OS标准 | 支持链接的FS |

## 直观理解

- **单级目录** = 所有人的物品堆在一个大房间里，找什么都得翻遍全场。
- **两级目录** = 每人一个柜子，柜子里有你的东西，但柜子里没法再分格。
- **树形目录** = 柜子里有抽屉，抽屉里有盒子，层层分类。
- **无环图目录** = 柜子里有个"传送门"指向另一个柜子的某个格子——共享。
