# 06_OS运行内核态用户态

## 核心概念

- **两种CPU运行状态**：核心态（内核态/管态/系统态）和用户态（目态）
- **内核态**：CPU可以执行**特权指令**，访问所有资源，运行OS内核代码
- **用户态**：CPU只能执行**非特权指令**，受限访问资源，运行用户程序
- **特权指令**：只能在内核态执行的指令，如：
  - I/O指令（直接操作硬件）
  - 修改PSW（程序状态字）的指令
  - 修改页表基址寄存器的指令
  - 停机指令（HLT）
- **状态切换**：
  - 用户态 → 内核态：通过**中断/异常/系统调用**（唯一途径）
  - 内核态 → 用户态：通过设置PSW（将状态位改为用户态）

## 原理分析

### 硬件支持机制

CPU通过**程序状态字（PSW）**中的某一位标识当前状态：
- PSW状态位 = 0：用户态
- PSW状态位 = 1：核心态

### 中断/异常触发的切换过程

```
1. 用户程序执行系统调用（int/trap指令）
2. CPU检测到陷入 → 硬件自动：
   a. 保存当前PSW和PC到内核栈
   b. 将PSW状态位设为核心态
   c. 将PC设为中断处理程序入口地址
3. 执行内核中的系统调用服务
4. 内核执行完毕 → 通过特权指令将PSW状态位改回用户态
5. 返回用户程序继续执行
```

### 内核态下运行的内容

1. **时钟管理**：时钟中断处理
2. **中断机制**：中断处理程序
3. **原语**：原子操作（如P、V操作的底层实现）
4. **系统调用的执行**：进程管理、存储管理、设备管理等

### 内核的分类

- **大内核**：将OS的主要功能模块都作为系统内核
- **微内核**：只将进程管理、低级存储管理、中断处理放入内核

## 直观理解

想象一个**安检系统**：
- **用户态** = 普通旅客区域：你可以自由走动、看大屏幕、使用公共设施（非特权指令）
- **内核态** = 安检通道内部：可以操作X光机、开箱检查、查看监控（特权指令）
- **中断/系统调用** = 你需要进入安检区域时，必须通过安检门（状态切换的唯一途径）
- **PSW** = 你身上的通行卡，安检员刷卡确认你是旅客还是内部员工

## 知识关联

- **用户态/内核态切换**是系统调用的底层机制，关联第8个知识点（中断异常系统调用）
- **特权指令**不能在用户态执行，否则触发**访管中断（trap）**
- **微内核**将更多功能放到用户态，减少了内核态代码量，关联第7个知识点
- **进程调度**在内核态完成，**用户程序**在用户态运行
- **408考法**：
  - 选择题：判断哪些操作需要在内核态完成
  - 简答题：描述用户态到内核态的切换过程
  - 常见陷阱：**并非所有中断都会切换到内核态**（但在408中通常认为中断会引发内核态切换）；**系统调用是自愿陷入**，**中断是被动触发**
