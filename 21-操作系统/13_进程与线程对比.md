# 13_进程与线程对比

## 核心概念

- **进程**：资源分配的基本单位，拥有独立的地址空间和资源
- **线程（Thread）**：CPU调度的基本单位，是进程中的一个执行流
- **引入线程的目的**：减少进程切换开销，提高并发效率
- **关键区分**：
  - 进程 = 资源分配单位 + 调度单位（传统）
  - 进程 = 资源分配单位（引入线程后）
  - 线程 = CPU调度单位（引入线程后）
- **同一进程中的线程共享**：地址空间、全局变量、打开的文件、信号处理方式
- **每个线程私有**：线程ID、寄存器组、栈（用于函数调用和局部变量）、状态字

## 原理分析

### 进程与线程的对比表

| 对比项 | 进程 | 线程 |
|-------|------|------|
| 调度单位 | 传统OS中是调度单位 | 引入线程后是调度单位 |
| 资源分配 | 拥有独立资源 | 不拥有资源，共享所属进程的资源 |
| 切换开销 | 大（切换地址空间/页表） | 小（同一地址空间内切换） |
| 并发性 | 进程间并发 | 同一进程内多个线程也可并发 |
| 地址空间 | 独立的虚拟地址空间 | 共享所属进程的地址空间 |
| 通信方式 | 需要IPC（管道/消息/共享内存） | 可直接读写共享变量 |
| 健壮性 | 一个进程崩溃不影响其他进程 | 一个线程崩溃可能导致整个进程崩溃 |
| 创建开销 | 大（需分配地址空间等） | 小（共享进程资源） |

### 线程带来的优势

以Web服务器为例（多线程 vs 多进程）：

**多进程模型**：
- 每个请求创建一个进程 → 创建/销毁开销大
- 进程间通信困难 → 需要IPC机制

**多线程模型**：
- 每个请求创建一个线程 → 创建开销小
- 共享地址空间 → 直接共享缓冲区数据

$$\text{进程切换耗时} \approx 1000 \sim 数千\mu s$$
$$\text{线程切换耗时} \approx 数十\mu s$$

## 直观理解

- **进程** = 一家公司：有自己的办公室（地址空间）、银行账户（资源）、营业执照（PCB）
- **线程** = 公司里的员工：共享办公室和账户，但每人有自己的工位（栈）和工作进度（寄存器）
- **进程间通信** = 公司之间要签合同、发传真（复杂、开销大）
- **线程间通信** = 同一办公室的同事直接喊一声就行（简单、开销小）
- **进程崩溃** = 公司倒闭，不影响其他公司
- **线程崩溃** = 一个员工搞砸了，整个公司可能都受影响

## 代码实现

### 进程 vs 线程：创建开销与共享特性对比

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <sys/wait.h>
#include <pthread.h>
#include <sys/time.h>

/*
 * 对比实验：进程创建(fork) vs 线程创建(pthread_create)
 *
 * 关键观察：
 * 1. 线程共享全局变量，进程各自独立
 * 2. 线程创建开销远小于进程创建
 * 3. 线程切换不需要切换页表（同一地址空间）
 */

#define NUM 100  /* 创建100个进程/线程来对比开销 */

int shared_counter = 0;  /* 全局变量：线程共享，进程独立 */

/* 线程函数：所有线程共享同一个全局变量 */
void* thread_func(void* arg) {
    shared_counter++;  /* 多线程共享——需要互斥保护 */
    return NULL;
}

/* 测量创建100个进程的时间 */
void test_process_creation() {
    struct timeval start, end;
    gettimeofday(&start, NULL);

    for (int i = 0; i < NUM; i++) {
        pid_t pid = fork();
        if (pid == 0) {
            /* 子进程：独立的地址空间 */
            shared_counter++;  /* 不影响父进程的 counter */
            exit(0);
        }
    }
    /* 父进程等待所有子进程 */
    for (int i = 0; i < NUM; i++) {
        wait(NULL);
    }

    gettimeofday(&end, NULL);
    long us = (end.tv_sec - start.tv_sec) * 1000000
            + (end.tv_usec - start.tv_usec);
    printf("创建 %d 个进程耗时: %ld μs\n", NUM, us);
    printf("父进程中的 counter = %d（子进程的修改不影响父进程）\n",
           shared_counter);
}

/* 测量创建100个线程的时间 */
void test_thread_creation() {
    pthread_t threads[NUM];
    struct timeval start, end;
    gettimeofday(&start, NULL);

    for (int i = 0; i < NUM; i++) {
        pthread_create(&threads[i], NULL, thread_func, NULL);
    }
    for (int i = 0; i < NUM; i++) {
        pthread_join(threads[i], NULL);
    }

    gettimeofday(&end, NULL);
    long us = (end.tv_sec - start.tv_sec) * 1000000
            + (end.tv_usec - start.tv_usec);
    printf("创建 %d 个线程耗时: %ld μs\n", NUM, us);
    printf("全局 counter = %d（线程共享地址空间）\n", shared_counter);
}

int main() {
    printf("=== 进程创建测试 ===\n");
    test_process_creation();

    shared_counter = 0;  /* 重置 */
    printf("\n=== 线程创建测试 ===\n");
    test_thread_creation();

    return 0;
}
/* 编译: gcc -o compare compare.c -lpthread */
```

### 线程共享地址空间——直接通信演示

```c
#include <stdio.h>
#include <pthread.h>
#include <string.h>

/*
 * 线程间通信 vs 进程间通信
 *
 * 线程：共享全局变量，直接读写即可（但需要同步）
 * 进程：需要管道、共享内存、消息队列等IPC机制
 *
 * 本例演示：生产者线程写数据，消费者线程直接读取
 */
char buffer[256];         /* 线程共享的缓冲区 */
pthread_mutex_t lock;     /* 互斥锁保护共享数据 */
int data_ready = 0;       /* 数据就绪标志 */

void* producer(void* arg) {
    const char* msg = "Hello from producer thread!";
    pthread_mutex_lock(&lock);
    strcpy(buffer, msg);
    data_ready = 1;
    printf("生产者：数据已写入共享缓冲区\n");
    pthread_mutex_unlock(&lock);
    return NULL;
}

void* consumer(void* arg) {
    /* 等待数据就绪（实际应用中用条件变量） */
    while (!data_ready) { /* busy wait */ }

    pthread_mutex_lock(&lock);
    printf("消费者：从共享缓冲区读取 → \"%s\"\n", buffer);
    pthread_mutex_unlock(&lock);
    return NULL;
}

int main() {
    pthread_t prod, cons;
    pthread_mutex_init(&lock, NULL);

    pthread_create(&prod, NULL, producer, NULL);
    pthread_create(&cons, NULL, consumer, NULL);

    pthread_join(prod, NULL);
    pthread_join(cons, NULL);

    pthread_mutex_destroy(&lock);
    return 0;
}
```

## 知识关联

- **线程的实现方式**详见第14个知识点（用户级/内核级/组合）
- **线程模型**详见第16、17个知识点
- **线程间通信**直接用共享变量（但需要同步机制），关联信号量和PV操作
- **现代OS（Linux、Windows）**：进程是资源分配单位，线程是调度单位
- **Java线程**是映射到内核线程的（一对一模型）
- **408考法**：
  - 选择题：进程和线程哪个是资源分配单位/调度单位
  - 简答题：引入线程的好处
  - 综合题：比较进程切换和线程切换的开销差异
- **常见陷阱**：
  - 线程**没有**独立的地址空间
  - 线程切换**不需要**切换页表（同一进程内）
  - 不是所有OS都有线程（早期UNIX没有用户可见的线程）
