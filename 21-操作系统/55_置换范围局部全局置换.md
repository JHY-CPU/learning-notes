# 55_置换范围局部全局置换

## 核心概念

- **局部置换（Local Replacement）**：进程发生缺页时，只能从**分配给自己的物理块**中选择换出页。每个进程的页面置换互不影响。
- **全局置换（Global Replacement）**：进程发生缺页时，可以从**所有进程的物理块**（包括其他进程持有的块）中选择换出页。
- 置换范围决定了系统如何在进程间共享物理内存。
- 408 考点：区分局部/全局置换对缺页率的影响，以及对进程公平性的含义。

## 原理分析

### 局部置换的工作过程

1. 进程 A 缺页，系统检查 A 拥有的物理块集合。
2. 在 A 的物理块中执行页面置换算法（如 LRU），选出一个牺牲页。
3. 将新页面调入被换出的块中。
4. 其他进程 B、C 的物理块不受影响。

**特点**：
- 每个进程的驻留集大小固定，行为可预测。
- 一个进程的内存压力不会"传染"给其他进程。
- 实现简单，常与固定分配策略配合使用。

### 全局置换的工作过程

1. 进程 A 缺页，系统检查所有分配出去的物理块（含 B、C 的块）。
2. 在全局范围内执行页面置换算法，选出一个牺牲页（可能属于进程 B）。
3. 将 A 的新页面调入被换出的块中。
4. 进程 B 的驻留集减少一个块，A 增加一个块。

**特点**：
- 各进程的驻留集大小动态变化。
- 内存在进程间"流动"，利用率更高。
- 但一个进程的抖动可能拖垮整个系统（连锁缺页）。

### 对比分析

| 特性 | 局部置换 | 全局置换 |
|------|---------|---------|
| 置换范围 | 仅本进程的物理块 | 所有物理块 |
| 驻留集大小 | 固定 | 动态变化 |
| 进程隔离性 | 好 | 差（可能相互影响） |
| 内存利用率 | 较低 | 较高 |
| 常见配合 | 固定分配 | 可变分配 |
| 抖动传播 | 不传播 | 可能连锁传播 |

### 置换范围与分配策略的组合

四种组合方式：
- **固定分配 + 局部置换**：最保守，每个进程获得固定数量的物理块，缺页时只替换自己的页面。
- **可变分配 + 全局置换**：最灵活，也是多数现代操作系统采用的方式。
- **可变分配 + 局部置换**：进程可以申请增加物理块，但置换只在自己的块内进行。
- **固定分配 + 全局置换**：逻辑上矛盾，实践中不使用（因为全局置换必然改变各进程的分配量）。

## 直观理解

- **局部置换** = 各家各户的冰箱：你在自家冰箱里找东西替换，不会去别人家拿。
- **全局置换** = 公共食堂打饭：盘子总量有限，你缺饭时可以从任何人的盘子里拿（先拿走别人的剩菜）。
- 局部置换强调"各管各的"，全局置换追求"整体利用率最高"。

## 知识关联

- 与 56 页面分配策略紧密关联：固定分配必然是局部置换，可变分配可以是局部或全局。
- 全局置换下的工作集模型更复杂，需要系统级监控。
- 408 常考组合题："某系统采用可变分配全局置换策略，当系统出现抖动时……"
- 抖动（thrashing）的严重程度与置换范围直接相关：全局置换下抖动更容易蔓延。
