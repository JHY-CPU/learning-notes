# 32_经典同步问题哲学家

## 核心概念

- **哲学家进餐问题**：5个哲学家围坐圆桌，每人左右各一根筷子，需同时拿起两根才能吃饭
- **资源竞争**：每个哲学家需要两份相同类型的资源
- **死锁风险**：所有哲学家同时拿起左边筷子，则无人能拿到右边筷子
- **解决方案1**：限制最多4人同时拿筷子（信号量限制）
- **解决方案2**：奇数号先拿左再拿右，偶数号先拿右再拿左（破环）
- **解决方案3**：用互斥信号量将拿两根筷子的过程原子化

**408核心考点**：
- 理解为什么会产生死锁
- 至少掌握一种解决方案
- 能够分析信号量初值和PV操作序列

## 原理分析

**问题建模**：

```
        哲学0
       /    \
   筷子4    筷子0
     |        |
   哲学4    哲学1
     |        |
   筷子3    筷子1
       \    /
        哲学3 —— 筷子2 —— 哲学2
```

哲学家i的左边筷子为i，右边筷子为(i+1)%5。

**错误解法（会死锁）**：

```
Semaphore chopstick[5] = {1, 1, 1, 1, 1};

哲学家i:
    P(chopstick[i]);         // 拿左边
    P(chopstick[(i+1)%5]);   // 拿右边
    吃饭;
    V(chopstick[i]);
    V(chopstick[(i+1)%5]);
```

死锁场景：5个哲学家同时执行到P(chopstick[i])，每人拿了左边筷子，右边都被占，全部阻塞。

**解法一：限制并发数**：

```
Semaphore chopstick[5] = {1, 1, 1, 1, 1};
Semaphore limit = 4;    // 最多4人同时拿筷子

哲学家i:
    P(limit);                    // 最多4人能进入
    P(chopstick[i]);             // 拿左边
    P(chopstick[(i+1)%5]);       // 拿右边
    吃饭;
    V(chopstick[(i+1)%5]);
    V(chopstick[i]);
    V(limit);
```

$$
\text{为什么} n-1 \text{就够了？} \quad \text{5个人中最多4个人同时拿筷子，至少1个人的右边筷子是空的}
$$

**解法二：奇偶号不同策略**：

```
Semaphore chopstick[5] = {1, 1, 1, 1, 1};

哲学家i:
    if (i % 2 == 0) {
        P(chopstick[i]);             // 偶数号：先左后右
        P(chopstick[(i+1)%5]);
    } else {
        P(chopstick[(i+1)%5]);       // 奇数号：先右后左
        P(chopstick[i]);
    }
    吃饭;
    V(chopstick[i]);
    V(chopstick[(i+1)%5]);
```

打破循环等待：相邻哲学家的拿筷子顺序相反，不会形成环。

**解法三：原子化拿筷子**：

```
Semaphore chopstick[5] = {1, 1, 1, 1, 1};
Semaphore mutex = 1;

哲学家i:
    P(mutex);
    P(chopstick[i]);
    P(chopstick[(i+1)%5]);
    V(mutex);
    吃饭;
    V(chopstick[i]);
    V(chopstick[(i+1)%5]);
```

将"拿两根筷子"作为一个原子操作，但并发度低。

## 直观理解

**吃饭的类比**：
5个人围坐吃火锅，每两人之间只有一双筷子。如果每个人都同时伸手拿自己左边的筷子，谁也吃不成。解决方案：
- 限流：最多让4个人同时来拿筷子
- 约定：单号先拿左，双号先拿右
- 排队：一个人拿完两双筷子，下一个人再拿

## 知识关联

**与死锁的联系（第37节）**：
- 哲学家问题是死锁四条件中"循环等待"的经典示例
- 解法一（limit=4）对应"破坏循环等待条件"思路（n-1人竞争至少1人成功，打破环路）
- 解法二（奇偶编号）对应"破坏循环等待条件"（打破环路）

**与银行家算法的联系（第40节）**：
- 哲学家问题的资源分配可以用安全性检查来判断是否会死锁

**408常见考法**：
- 给出某解法，判断是否会死锁
- 要求写出至少一种正确的PV操作解法
- 问信号量limit初值为什么是4而不是3或5
- 分析某解法的并发度（最多几人同时吃饭）
