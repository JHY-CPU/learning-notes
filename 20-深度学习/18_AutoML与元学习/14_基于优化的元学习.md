# 14_基于优化的元学习

## 1. Reptile

### 1.1 核心思想

**Reptile (Nichol et al., 2018)** 是 MAML 的极大简化：不需要计算二阶导数，直接将多次梯度更新的结果向初始参数靠拢。

```
MAML:  θ_new = θ - β * ∇_θ L(θ - α * ∇_θ L(θ))  (二层优化)
Reptile: θ_new = θ - β * (θ - θ')  (一步近似)
其中 θ' = 多步SGD后的参数
```

### 1.2 算法

```
初始化 θ

循环:
  1. 采样任务 T
  2. 在 T 上做 k 步SGD: θ' = SGD(θ, T, k steps)
  3. 更新: θ ← θ - β(θ - θ') = θ - β * (θ - θ')

等价于: θ 向每个任务的最优解靠拢
```

```python
class Reptile:
    """Reptile算法"""
    def __init__(self, model, inner_lr=0.01, outer_lr=0.001, inner_steps=5):
        self.model = model
        self.inner_lr = inner_lr
        self.outer_lr = outer_lr
        self.inner_steps = inner_steps
        self.optimizer = torch.optim.Adam(model.parameters(), lr=outer_lr)
    
    def meta_train_step(self, tasks):
        """
        tasks: list of (support_x, support_y)
        """
        old_params = [p.clone() for p in self.model.parameters()]
        param_diff = [torch.zeros_like(p) for p in self.model.parameters()]
        
        for support_x, support_y in tasks:
            # 复制模型参数
            task_params = [p.clone() for p in self.model.parameters()]
            task_optimizer = torch.optim.SGD(task_params, lr=self.inner_lr)
            
            # k步SGD
            for _ in range(self.inner_steps):
                logits = self._forward_with_params(support_x, task_params)
                loss = F.cross_entropy(logits, support_y)
                task_optimizer.zero_grad()
                loss.backward()
                task_optimizer.step()
            
            # 累积参数差
            for diff, old, new in zip(param_diff, old_params, task_params):
                diff.add_(new - old)
        
        # 平均并更新
        for param, diff in zip(self.model.parameters(), param_diff):
            param.data.add_(self.outer_lr * diff / len(tasks))
    
    def _forward_with_params(self, x, params):
        idx = 0
        h = x
        for module in self.model.net:
            if isinstance(module, nn.Linear):
                h = F.linear(h, params[idx], params[idx + 1])
                idx += 2
            else:
                h = module(h)
        return h
```

### 1.3 Reptile vs MAML

| 特性 | MAML | Reptile |
|------|------|---------|
| 需要二阶导数 | 是 | 否 |
| 内层更新 | 1步 | k步 |
| 计算图 | 复杂 | 简单 |
| 理论等价 | — | k=1时等价于FOMAML |
| 5-way 1-shot | ~48.7% | ~47.1% |
| 实现难度 | 高 | 低 |

## 2. Meta-SGD

### 2.1 核心思想

**Meta-SGD (Li et al., 2017)** 将**每个参数的学习率**也作为元学习的对象：

$$\theta'_i = \theta - \ell \odot \nabla_\theta \mathcal{L}_{\mathcal{T}_i}(\theta)$$

其中 $\ell$ 是逐参数的学习率向量（与 $\theta$ 同维度），通过元学习获得。

### 2.2 实现

```python
class MetaSGD:
    def __init__(self, model, outer_lr=0.001):
        self.model = model
        
        # 学习率向量：每个参数一个学习率
        self.lr_params = nn.ParameterDict()
        for name, param in model.named_parameters():
            self.lr_params[name] = nn.Parameter(torch.ones_like(param) * 0.01)
        
        self.optimizer = torch.optim.Adam(
            list(model.parameters()) + list(self.lr_params.parameters()),
            lr=outer_lr
        )
    
    def inner_update(self, support_x, support_y):
        """内层更新：用学习到的学习率"""
        logits = self.model(support_x)
        loss = F.cross_entropy(logits, support_y)
        
        grads = torch.autograd.grad(loss, self.model.parameters(), create_graph=True)
        
        updated_params = []
        for (name, param), grad in zip(self.model.named_parameters(), grads):
            lr = torch.sigmoid(self.lr_params[name])  # 限制到 [0, 1]
            updated_params.append(param - lr * grad)
        
        return updated_params
```

### 2.3 与 MAML 对比

| 特性 | MAML | Meta-SGD |
|------|------|----------|
| 内层学习率 | 固定（超参数） | 逐参数学习 |
| 自适应性 | 低 | 高 |
| 参数量 | 模型参数 | 模型参数 + 学习率参数 |
| 5-way 1-shot | ~48.7% | ~50.5% |

## 3. Meta-Learning 的优化视角

### 3.1 学什么

| 方法 | 学习的内容 |
|------|-----------|
| MAML | 初始化参数 |
| Reptile | 初始化参数（简化） |
| Meta-SGD | 初始化 + 逐参数学习率 |
| Meta-Adam | 初始化 + 优化器参数 |
| L2F (Learning to Learn to Forget) | 遗忘策略 |

### 3.2 双层优化框架

所有基于优化的元学习都可以写成双层优化：

$$\min_\phi \sum_{\mathcal{T}_i} \mathcal{L}_{\mathcal{T}_i}(\theta^*_i(\phi))$$

$$\theta^*_i(\phi) = \text{Optimizer}(\phi, \mathcal{T}_i)$$

其中 $\phi$ 是元参数（初始化、学习率等），$\theta^*_i$ 是任务 $\mathcal{T}_i$ 的最优参数。

## 4. 更多变体

### 4.1 LEO (Latent Embedding Optimization)

**LEO (Rusu et al., ICLR 2019)** 在低维潜在空间做梯度优化，而非参数空间：

```
编码器: 支持集 → 任务嵌入 z
解码器: z → 模型参数 θ
优化: 在z空间做梯度下降 → 解码为新的θ
```

优势：低维空间优化更高效。

### 4.2 Alpha MAML

为每个任务学习不同的内层学习率：

$$\alpha_i = g_\phi(\mathcal{T}_i)$$

根据任务难度自动调整学习率。

## 5. 实践建议

| 场景 | 推荐方法 |
|------|----------|
| 快速原型 | Reptile |
| 最高精度 | MAML |
| 参数少 | Meta-SGD |
| 异质任务 | Alpha MAML |
| 高维参数空间 | LEO |

---

**关键要点**：
1. Reptile 是 MAML 的一阶简化，不需要二阶导数，实现更简单
2. Meta-SGD 将学习率也作为元学习参数，实现了逐参数的自适应
3. 所有基于优化的元学习都可以统一为双层优化框架
4. 在低维潜在空间（如LEO）做优化可以避免高维参数空间的困难
