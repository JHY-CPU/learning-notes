# 4_高效搜索策略

## 1. 为什么需要高效搜索

| 方法 | 搜索成本 | 搜索精度 | 是否需要重训 |
|------|----------|----------|-------------|
| NASNet (RL, 从零训练) | 2000 GPU天 | 高 | 是 |
| ENAS (权重共享) | 0.5 GPU天 | 中 | 是 |
| DARTS (可微分) | 1.5 GPU天 | 中 | 是 |
| One-Shot (超网络) | 1 GPU天 | 低 | 是 |
| SPOS (单路径) | 0.3 GPU天 | 中 | 否(可直接用) |

## 2. One-Shot 搜索

### 2.1 核心思想

训练一个包含所有候选操作的**超网络 (SuperNet)**，然后从超网络中**提取**最优子网络。

```
训练阶段: 超网络（所有操作共享权重）
    ↓
搜索阶段: 从超网络中选择最优子网络
    ↓
重训阶段: 独立训练最优子网络
```

### 2.2 实现

```python
class OneShotSuperNet(nn.Module):
    """One-Shot超网络"""
    def __init__(self, candidate_ops, num_layers=20):
        super().__init__()
        self.num_layers = num_layers
        self.layers = nn.ModuleList()
        
        for i in range(num_layers):
            self.layers.append(nn.ModuleList([
                op for op in candidate_ops
            ]))
    
    def forward(self, x, architecture):
        """
        architecture: list of op_indices, 选择每层使用哪个操作
        """
        for i, op_idx in enumerate(architecture):
            x = self.layers[i][op_idx](x)
        return x
    
    def forward_all(self, x):
        """所有路径的加权前向传播（用于训练）"""
        for i in range(self.num_layers):
            # 每层所有操作的输出求平均
            outputs = [op(x) for op in self.layers[i]]
            x = sum(outputs) / len(outputs)
        return x
```

### 2.3 训练与搜索

```python
def train_oneshot(supernet, dataset, num_epochs=200):
    """训练One-Shot超网络"""
    optimizer = torch.optim.SGD(supernet.parameters(), lr=0.025)
    
    for epoch in range(num_epochs):
        for x, y in dataset:
            # 随机采样架构
            arch = [random.randint(0, len(supernet.layers[i])-1) 
                    for i in range(supernet.num_layers)]
            
            pred = supernet(x, arch)
            loss = F.cross_entropy(pred, y)
            optimizer.zero_grad()
            loss.backward()
            optimizer.step()

def search_oneshot(supernet, val_set, num_candidates=1000):
    """从超网络搜索最优架构"""
    best_acc = 0
    best_arch = None
    
    for _ in range(num_candidates):
        # 随机采样架构
        arch = [random.randint(0, len(supernet.layers[i])-1)
                for i in range(supernet.num_layers)]
        
        # 评估（用超网络权重，不训练）
        acc = evaluate_architecture(supernet, arch, val_set)
        
        if acc > best_acc:
            best_acc = acc
            best_arch = arch
    
    return best_arch
```

## 3. SPOS (Single Path One-Shot)

### 3.1 核心思想

**SPOS (Guo et al., ECCV 2020)** 每次前向传播只走**一条路径**（每个位置随机选择一个操作），大幅降低训练成本。

```
传统One-Shot: 每层所有操作都计算 → 内存大、计算慢
SPOS: 每层只选一个操作 → 内存小、计算快
```

```python
class SPOSSuperNet(nn.Module):
    """SPOS超网络：单路径采样"""
    def __init__(self, candidate_ops, num_layers):
        super().__init__()
        self.num_layers = num_layers
        self.layers = nn.ModuleList([
            nn.ModuleList(ops) for ops in candidate_ops
        ])
    
    def forward(self, x, architecture=None):
        if architecture is None:
            # 随机采样
            architecture = [
                random.randint(0, len(self.layers[i]) - 1)
                for i in range(self.num_layers)
            ]
        
        for i, op_idx in enumerate(architecture):
            x = self.layers[i][op_idx](x)
        return x
    
    def sample_architecture(self):
        """均匀随机采样架构"""
        return [
            random.randint(0, len(self.layers[i]) - 1)
            for i in range(self.num_layers)
        ]
```

### 3.2 进化搜索

SPOS 通常结合进化算法搜索最优架构：

```python
def evolutionary_search(supernet, val_set, pop_size=50, generations=20):
    """进化搜索最优子网络"""
    # 初始化种群
    population = [supernet.sample_architecture() for _ in range(pop_size)]
    
    # 评估初始种群
    fitness = [evaluate_architecture(supernet, arch, val_set) for arch in population]
    
    for gen in range(generations):
        # 选择
        selected_idx = np.argsort(fitness)[-pop_size//2:]
        parents = [population[i] for i in selected_idx]
        
        # 交叉和变异
        children = []
        while len(children) < pop_size - len(parents):
            p1, p2 = random.sample(parents, 2)
            child = crossover(p1, p2)
            child = mutate(child, rate=0.1)
            children.append(child)
        
        # 评估
        population = parents + children
        fitness = [evaluate_architecture(supernet, arch, val_set) for arch in population]
    
    best_idx = np.argmax(fitness)
    return population[best_idx]
```

## 4. 权重共享的陷阱

### 4.1 问题

权重共享引入了**排名不一致性 (Ranking Inconsistency)**：在超网络中排名高的架构，在独立训练后排名可能很低。

### 4.2 原因

- 共享权重是所有架构的"折中"，不代表任何单一架构的最优权重
- 某些操作（如跳跃连接）在权重共享下有不公平优势

### 4.3 缓解方案

| 方案 | 描述 |
|------|------|
| 渐进式收缩 | 训练过程中逐步移除差的路径 |
| FairNAS | 保证每个操作被同等采样 |
| 重排 (Re-training) | 对Top-K架构独立训练后排名 |
| 蒸馏 | 用超网络的权重初始化子网络 |

```python
class FairNASSuperNet(SPOSSuperNet):
    """FairNAS：公平采样"""
    def forward(self, x):
        # 对每层的操作排列随机排列
        for layer_idx in range(self.num_layers):
            perm = torch.randperm(len(self.layers[layer_idx]))
            # 保证在足够多的step后，每个操作都被均匀采样
            op_idx = perm[self.step_count % len(perm)]
            x = self.layers[layer_idx][op_idx](x)
        return x
```

## 5. 渐进式搜索

### 5.1 P-DARTS

**P-DARTS (Chen et al., 2020)** 逐步增加搜索深度：

```
阶段1: 5层Cell, 8个候选操作 → 训练25epoch
    ↓ 移除低权重操作
阶段2: 11层Cell, 5个候选操作 → 训练25epoch
    ↓ 移除低权重操作
阶段3: 17层Cell, 3个候选操作 → 训练25epoch
```

### 5.2 ProxylessNAS

**ProxylessNAS (Cai et al., ICLR 2019)** 直接在目标任务上搜索，不使用代理任务：

```python
class ProxylessNAS(nn.Module):
    """ProxylessNAS：直接在目标任务上搜索"""
    def __init__(self, candidate_ops):
        super().__init__()
        self.candidate_ops = nn.ModuleList(candidate_ops)
        self.alpha = nn.Parameter(torch.zeros(len(candidate_ops)))
    
    def forward(self, x):
        # 采样一个操作（二值化）
        if self.training:
            # Gumbel采样
            weights = F.gumbel_softmax(self.alpha, tau=5, hard=False)
        else:
            # 推理时选择最大权重
            weights = F.softmax(self.alpha, dim=0)
        
        # 只计算选中的操作
        selected = torch.multinomial(weights, 1).item()
        return self.candidate_ops[selected](x) * weights[selected]
```

## 6. 搜索策略对比

| 策略 | 训练方式 | 搜索方式 | 速度 | 精度 |
|------|----------|----------|------|------|
| One-Shot | 所有路径加权 | 随机/进化 | 中 | 低 |
| SPOS | 单路径采样 | 进化搜索 | 快 | 中 |
| DARTS | 连续松弛 | 梯度 | 快 | 中 |
| ProxylessNAS | 直接目标任务 | 梯度+采样 | 中 | 高 |
| FairNAS | 公平采样 | 进化搜索 | 快 | 中 |

---

**关键要点**：
1. One-Shot 和 SPOS 通过训练共享超网络大幅降低搜索成本
2. 权重共享存在排名不一致问题，需要独立重训来验证
3. SPOS 每次只走一条路径，比所有路径加权更高效
4. 进化搜索是权重共享类方法中最常用的架构搜索方式
