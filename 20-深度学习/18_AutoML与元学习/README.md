# AutoML与元学习

## 一、AutoML

### 1.1 概念

AutoML（自动化机器学习）旨在自动化ML pipeline：
- 数据预处理
- 特征工程
- 模型选择
- 超参数优化
- 模型部署

### 1.2 神经架构搜索（NAS）

自动设计神经网络结构。

**搜索空间：**
- 宏搜索：搜索整个网络结构
- 微搜索：搜索基本模块（cell），堆叠组成网络

**搜索策略：**
| 方法 | 说明 |
|------|------|
| 强化学习 | 用RNN控制器生成架构 |
| 进化算法 | 遗传算法变异和选择 |
| 可微分搜索（DARTS） | 连续松弛，梯度优化 |
| 一次性搜索 | 超网中采样子网 |

**性能评估：**
- 训练完整模型（精确但慢）
- 权重共享（快但不准）
- 性能预测器

### 1.3 AutoML工具

- **Auto-sklearn**：传统ML自动选择
- **Auto-PyTorch**：PyTorch AutoML
- **Optuna**：超参数优化
- **NNI**：微软NAS/HPO框架

## 二、元学习

### 2.1 学会学习

元学习让模型学会如何学习新任务，核心是少样本学习。

### 2.2 方法分类

| 类别 | 代表 | 思路 |
|------|------|------|
| 基于度量 | Siamese、Prototypical、MAML | 学习好的距离度量 |
| 基于模型 | Memory-Augmented Nets | 模型本身具备快速学习能力 |
| 基于优化 | MAML、Reptile | 学习好的初始化，快速适应 |

### 2.3 MAML

Model-Agnostic Meta-Learning：
- 学习一个好的参数初始化
- 少量梯度步即可适应新任务

```
内循环：在支持集上微调
外循环：在查询集上评估，更新初始化参数
```

### 2.4 少样本学习

- **N-way K-shot**：N个类别，每个类别K个样本
- **支持集**：少量标注样本
- **查询集**：用于评估
- **Prototypical Networks**：计算类别原型，最近邻分类
