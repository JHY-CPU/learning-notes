# 0_AutoML 概述

## 1. 什么是 AutoML

**AutoML (Automated Machine Learning)** 的目标是自动化机器学习流程中需要人工干预的环节，使非专家也能构建高质量的ML模型。

### 1.1 ML流程中可自动化的环节

```
原始数据 → 数据预处理 → 特征工程 → 模型选择 → 超参数优化 → 模型评估 → 部署
              ↓            ↓          ↓           ↓
          AutoML       AutoML      AutoML      AutoML
```

### 1.2 AutoML 的目标

| 环节 | 传统方式 | AutoML |
|------|----------|--------|
| 数据预处理 | 手动清洗、归一化 | 自动检测并处理 |
| 特征工程 | 领域专家手动设计 | 自动特征生成/选择 |
| 模型选择 | 经验选择 | 搜索空间内自动选择 |
| 超参数调优 | 网格搜索/手动 | 贝叶斯优化/NAS |
| 模型集成 | 手动组合 | 自动集成策略 |
| 管道构建 | 手动搭建 | 端到端自动化 |

## 2. AutoML 发展历程

| 时间 | 里程碑 | 说明 |
|------|--------|------|
| 2012 | Auto-WEKA | 首个AutoML系统 |

| 2022+ | LLM-based AutoML | 用大语言模型做AutoML |

## 3. 核心组件

### 3.1 搜索空间 (Search Space)

定义了AutoML可探索的范围：

```python
# 超参数搜索空间示例
search_space = {
    'learning_rate': {'type': 'log_uniform', 'low': 1e-5, 'high': 1e-1},
    'batch_size': {'type': 'choice', 'values': [16, 32, 64, 128]},
    'num_layers': {'type': 'int', 'low': 2, 'high': 10},
    'hidden_dim': {'type': 'choice', 'values': [64, 128, 256, 512]},
    'dropout': {'type': 'uniform', 'low': 0.0, 'high': 0.5},
    'activation': {'type': 'choice', 'values': ['relu', 'gelu', 'silu']},
}
```

### 3.2 搜索策略 (Search Strategy)

如何在搜索空间中探索：

| 策略 | 描述 | 效率 |
|------|------|------|
| 网格搜索 | 穷举所有组合 | 低 |
| 随机搜索 | 随机采样 | 中 |
| 贝叶斯优化 | 用代理模型指导搜索 | 高 |
| 强化学习 | 用RL控制器搜索 | 高 |
| 进化算法 | 模拟自然选择 | 中 |

### 3.3 性能评估 (Performance Estimation)

如何快速评估候选配置的性能：

| 方法 | 描述 | 精度 | 速度 |
|------|------|------|------|
| 完整训练 | 完全训练模型 | 高 | 极慢 |
| 早停 (Early Stopping) | 在验证集收敛时停止 | 中 | 快 |
| 学习曲线外推 | 从早期结果预测最终性能 | 中 | 快 |
| 权重共享 | 共享超网络权重 | 低 | 极快 |
| 低保真度 | 降采样/降分辨率 | 低 | 快 |

## 6. AutoML 的价值与局限

### 6.1 价值

1. **降低门槛**：非ML专家也能构建好模型
2. **节省时间**：自动化手动调参过程
3. **发现新架构**：NAS发现了超越人工设计的网络
4. **可复现**：自动化流程更易复现

### 6.2 局限

1. **计算成本高**：搜索过程需要大量GPU资源
2. **数据质量依赖**：垃圾进垃圾出，AutoML无法弥补数据问题
3. **解释性差**：自动发现的配置缺乏领域解释
4. **搜索空间设计**：仍需人工定义搜索空间
