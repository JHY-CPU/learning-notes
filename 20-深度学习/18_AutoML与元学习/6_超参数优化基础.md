# 6_超参数优化基础

## 1. 超参数的类型

超参数是**训练前设定**的参数，不能通过梯度下降学习。

### 1.1 分类

| 类型 | 示例 | 搜索空间 |
|------|------|----------|
| 连续 | 学习率、权重衰减、Dropout率 | $\mathbb{R}^+$ |
| 整数 | 层数、隐藏单元数、batch size | $\mathbb{Z}^+$ |
| 类别 | 优化器类型、激活函数、学习率调度 | 离散集合 |
| 条件 | 特定优化器特有的参数 | 依赖其他选择 |

```python
search_space = {
    # 连续
    'learning_rate': {'type': 'log_uniform', 'low': 1e-5, 'high': 1e-1},
    'weight_decay': {'type': 'log_uniform', 'low': 1e-6, 'high': 1e-2},
    'dropout': {'type': 'uniform', 'low': 0.0, 'high': 0.5},
    
    # 整数
    'num_layers': {'type': 'int', 'low': 2, 'high': 8},
    'hidden_size': {'type': 'choice', 'values': [64, 128, 256, 512]},
    'batch_size': {'type': 'choice', 'values': [16, 32, 64, 128]},
    
    # 类别
    'optimizer': {'type': 'choice', 'values': ['sgd', 'adam', 'adamw']},
    'activation': {'type': 'choice', 'values': ['relu', 'gelu', 'silu']},
    'scheduler': {'type': 'choice', 'values': ['cosine', 'step', 'none']},
    
    # 条件 (如果optimizer=sgd)

}
```

## 2. 网格搜索 (Grid Search)

### 2.1 方法

对每个超参数定义一组候选值，穷举所有组合。

```python
from sklearn.model_selection import ParameterGrid

def grid_search(model_class, param_grid, train_fn, val_fn):
    """网格搜索"""
    best_score = -float('inf')
    best_params = None
    
    for params in ParameterGrid(param_grid):
        # 训练模型
        model = model_class(**params)
        score = train_fn(model)
        
        if score > best_score:
            best_score = score
            best_params = params
    
    return best_params, best_score

# 示例
param_grid = {
    'learning_rate': [1e-3, 1e-4, 1e-5],
    'batch_size': [32, 64, 128],
    'hidden_size': [128, 256, 512],
}
# 总组合数: 3 × 3 × 3 = 27
```

### 2.2 问题

- **维度灾难**：$k$ 个参数各 $n$ 个值 = $n^k$ 组合
- **效率低**：大量计算浪费在不好的区域
- **维度敏感**：重要参数和不重要参数分配相同搜索资源

## 3. 随机搜索 (Random Search)

### 3.1 方法

在搜索空间中随机采样配置。

```python
def random_search(model_class, param_dist, train_fn, n_trials=100):
    """随机搜索"""
    best_score = -float('inf')
    best_params = None
    
    for trial in range(n_trials):
        # 随机采样参数
        params = {}
        for name, dist in param_dist.items():
            if dist['type'] == 'log_uniform':
                params[name] = np.exp(np.random.uniform(
                    np.log(dist['low']), np.log(dist['high'])
                ))
            elif dist['type'] == 'uniform':
                params[name] = np.random.uniform(dist['low'], dist['high'])
            elif dist['type'] == 'choice':
                params[name] = np.random.choice(dist['values'])
            elif dist['type'] == 'int':
                params[name] = np.random.randint(dist['low'], dist['high'] + 1)
        
        # 训练模型
        model = model_class(**params)
        score = train_fn(model)
        
        if score > best_score:
            best_score = score
            best_params = params
    
    return best_params, best_score
```

### 3.2 为什么随机搜索优于网格搜索

**Bergstra & Bengio (2012)** 的关键洞察：

> 对于大多数问题，只有少数超参数对性能有显著影响。网格搜索在这些重要维度上的采样密度远低于随机搜索。

```
网格搜索 (9次评估):
  lr: [0.01, 0.1, 1.0]  → 只有3个不同值
  reg: [0.01, 0.1, 1.0] → 只有3个不同值

随机搜索 (9次评估):
  lr: 9个不同值（更高密度覆盖重要维度）
  reg: 9个不同值
```

## 4. 贝叶斯优化基础

### 4.1 核心思想

用**代理模型 (Surrogate Model)** 建模目标函数，基于代理模型的预测选择下一个评估点。

```
初始化: 随机评估几个点
    ↓ 循环:
    1. 用已有评估结果训练代理模型
    2. 用采集函数 (Acquisition Function) 选择下一个点
    3. 评估该点
    4. 更新代理模型
```

### 4.2 代理模型

代理模型 $f_{surrogate}(x)$ 近似真实目标函数，提供**预测均值** $\mu(x)$ 和**不确定性** $\sigma(x)$。

常用代理模型：
- **高斯过程 (GP)**：最经典，提供精确不确定性估计
- **随机森林**：SMAC使用
- **TPE**：树形Parzen估计器

### 4.3 采集函数 (Acquisition Function)

采集函数平衡**探索 (exploration)** 和**利用 (exploitation)**：

| 函数 | 公式 | 特点 |
|------|------|------|
| EI (Expected Improvement) | $E[\max(f(x) - f^*, 0)]$ | 最常用 |
| UCB (Upper Confidence Bound) | $\mu(x) + \kappa \sigma(x)$ | 简单 |
| PI (Probability of Improvement) | $P(f(x) > f^*)$ | 保守 |

## 5. 三种方法对比

| 方法 | 效率 | 实现复杂度 | 并行化 | 适用场景 |
|------|------|-----------|--------|----------|
| 网格搜索 | 低 | 极简 | 容易 | 参数少 (≤3) |
| 随机搜索 | 中 | 极简 | 容易 | 通用基线 |
| 贝叶斯优化 | 高 | 中等 | 较难 | 参数调优 |

## 6. 选择建议

| 场景 | 推荐方法 |
|------|----------|
| 快速基线 | 随机搜索 |
| 参数少 (≤3) | 网格搜索 |
| 训练昂贵 | 贝叶斯优化 |
| 需要并行 | 随机搜索 / BO with batch |
| 条件空间 | TPE (Optuna) |
| 大规模搜索 | Hyperband + BO |

---

**关键要点**：
1. 超参数优化是ML中计算最密集的环节之一
2. 随机搜索在高维空间中优于网格搜索
3. 贝叶斯优化通过代理模型+采集函数高效搜索
4. 采集函数（EI/UCB/PI）平衡探索与利用
