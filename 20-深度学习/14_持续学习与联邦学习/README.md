# 持续学习与联邦学习

## 一、持续学习

### 1.1 问题定义

模型在持续到来的新任务上学习，同时不忘记旧知识。

**灾难性遗忘**：学习新任务时严重遗忘旧任务。

### 1.2 解决方法

| 类别 | 方法 | 说明 |
|------|------|------|
| 正则化 | EWC | 对重要参数施加惩罚 |
| 正则化 | SI | 突触智能 |
| 架构 | PackNet | 为任务分配不同参数 |
| 架构 | Progressive Nets | 逐步增加网络容量 |
| 回放 | Experience Replay | 存储并回放旧数据 |
| 回放 | Generative Replay | 用生成模型回放 |

## 二、联邦学习

### 2.1 基本框架

多个客户端在本地训练，只上传模型更新到中央服务器。

**FedAvg算法：**
1. 服务器下发全局模型
2. 客户端本地训练
3. 服务器聚合（加权平均）
4. 重复

### 2.2 核心挑战

- **数据异质性**：各客户端数据分布不同（Non-IID）
- **通信效率**：减少上传下载次数
- **隐私保护**：防止从更新中推断原始数据
- **公平性**：避免偏向数据多的客户端

### 2.3 改进算法

- **FedProx**：添加近端正则化
- **SCAFFOLD**：控制方差
- **FedNova**：归一化更新
- **FedBN**：本地BatchNorm

### 2.4 隐私增强

- **安全聚合**：加密状态下的梯度聚合
- **差分隐私**：在更新中添加噪声
- **同态加密**：在密文上计算
