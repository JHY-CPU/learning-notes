# 11_联邦优化算法

## 1. 算法演进

联邦学习优化算法从 FedAvg 出发，针对不同问题（Non-IID、通信效率、异构设备）发展出了多种改进方案。

## 2. FedAvg 回顾

$$w_{t+1} = \sum_{k \in S_t} \frac{n_k}{n} w_k^{(t)}$$

其中 $w_k^{(t)}$ 是客户端 $k$ 本地训练 $E$ 个 epoch 后的模型。

### 2.1 局限性

| 局限 | 说明 |
|------|------|
| Non-IID 敏感 | 数据异构时收敛差 |
| 客户端漂移 | 本地最优偏离全局最优 |
| 通信效率 | 需要传输完整模型 |
| 公平性 | 大数据客户端主导 |

## 3. FedProx

$$w_k^{(t)} = \arg\min_w \left\{ \mathcal{L}_k(w) + \frac{\mu}{2} \|w - w_t\|^2 \right\}$$

近端项 $\frac{\mu}{2}\|w - w_t\|^2$ 限制本地模型向全局模型靠拢。

## 4. SCAFFOLD

### 4.1 核心思想

Karimireddy 等人 (2020) 提出 SCAFFOLD，使用**控制变量 (Control Variates)** 校正客户端漂移。

### 4.2 更新规则

服务器端：
$$c_{t+1} = c_t + \frac{1}{K} \sum_{k \in S_t} (c_k^{(t+1)} - c_k^{(t)})$$

客户端本地更新：
$$w_k^{(t+1)} = w_k^{(t)} - \eta (g_k - c_k^{(t)} + c_t)$$

其中 $g_k$ 是客户端梯度，$c_k$ 和 $c_t$ 是控制变量。

### 4.3 实现

```python
class SCAFFOLDServer:
    def __init__(self, global_model):
        self.global_model = global_model
        # 全局控制变量
        self.c_global = {
            name: torch.zeros_like(param)
            for name, param in global_model.named_parameters()
        }

    def aggregate(self, client_models, client_controls):
        """聚合模型和控制变量"""
        global_dict = self.global_model.state_dict()

        # 聚合模型
        for key in global_dict:
            global_dict[key] = torch.stack(
                [m.state_dict()[key] for m in client_models]
            ).mean(dim=0)

        self.global_model.load_state_dict(global_dict)

        # 更新全局控制变量
        for key in self.c_global:
            self.c_global[key] = torch.stack(
                [c[key] for c in client_controls]
            ).mean(dim=0)

class SCAFFOLDClient:
    def __init__(self, local_data, local_epochs=5, lr=0.01):
        self.local_data = local_data
        self.local_epochs = local_epochs
        self.lr = lr
        self.c_local = None  # 本地控制变量

    def initialize_control(self, model):
        """初始化本地控制变量"""
        self.c_local = {
            name: torch.zeros_like(param)
            for name, param in model.named_parameters()
        }

    def local_train(self, model, c_global):
        """SCAFFOLD 本地训练"""
        model.train()
        optimizer = torch.optim.SGD(model.parameters(), lr=self.lr)

        # 保存旧模型参数
        old_params = {
            name: param.clone().detach()
            for name, param in model.named_parameters()
        }

        for epoch in range(self.local_epochs):
            for x, y in self.local_data:
                optimizer.zero_grad()
                output = model(x)
                loss = nn.functional.cross_entropy(output, y)
                loss.backward()

                # 梯度校正
                for name, param in model.named_parameters():
                    if param.grad is not None:
                        param.grad += c_global[name] - self.c_local[name]

                optimizer.step()

        # 更新本地控制变量
        new_params = {
            name: param.clone().detach()
            for name, param in model.named_parameters()
        }
        for name in self.c_local:
            self.c_local[name] = (
                self.c_local[name] -
                c_global[name] +
                (old_params[name] - new_params[name]) / (self.lr * self.local_epochs)
            )

        return model, self.c_local
```

## 5. FedNova

### 5.1 动机

不同客户端的本地 epoch 数 $E_k$ 可能不同，导致**更新量异构**。FedNova 通过归一化梯度来解决此问题。

### 5.2 更新规则

$$w_{t+1} = w_t - \eta \sum_{k \in S_t} \frac{n_k}{n} \frac{\tau_k}{\bar{\tau}} (w_t - w_k^{(t)})$$

其中 $\tau_k$ 是客户端 $k$ 的本地步数，$\bar{\tau} = \sum_k \frac{n_k}{n} \tau_k$ 是加权平均步数。

### 5.3 实现

```python
class FedNovaServer:
    def aggregate(self, global_model, client_models, client_steps,
                  client_samples):
        """
        FedNova 聚合: 归一化不同步数的更新
        """
        total_samples = sum(client_samples)
        avg_steps = sum(
            s * (n / total_samples) for s, n in zip(client_steps, client_samples)
        )

        global_dict = global_model.state_dict()

        for key in global_dict:
            weighted_update = torch.zeros_like(global_dict[key])
            for model, steps, samples in zip(client_models, client_steps,
                                              client_samples):
                local_update = global_dict[key] - model.state_dict()[key]
                # 归一化: (w_t - w_k) / τ_k * τ̄
                normalized = local_update / steps * avg_steps
                weighted_update += normalized * (samples / total_samples)

            global_dict[key] -= weighted_update

        global_model.load_state_dict(global_dict)
```

## 6. 算法对比

| 算法 | 核心思想 | Non-IID | 通信 | 异构设备 | 额外开销 |
|------|----------|---------|------|----------|----------|
| FedAvg | 简单平均 | 差 | 基线 | 不支持 | 无 |
| FedProx | 近端正则化 | 中 | 基线 | 部分支持 | 低 |
| SCAFFOLD | 控制变量校正 | **好** | 基线 | 不支持 | 中 |
| FedNova | 梯度归一化 | 中 | 基线 | **支持** | 低 |

## 7. 收敛性分析

| 算法 | 收敛率 (IID) | 收敛率 (Non-IID) | 通信轮数 |
|------|-------------|-----------------|----------|
| FedAvg | $O(1/\sqrt{T})$ | $O(1/\sqrt{T}) + O(\zeta)$ | 多 |
| FedProx | $O(1/\sqrt{T})$ | $O(1/\sqrt{T}) + O(\zeta/\mu)$ | 中 |
| SCAFFOLD | $O(1/\sqrt{T})$ | $O(1/\sqrt{T})$ | **少** |

$\zeta$ 衡量数据异构程度，$\mu$ 为 FedProx 参数。

## 8. 总结

| 要点 | 说明 |
|------|------|
| FedAvg | 基础算法，IID 下有效 |
| FedProx | 近端正则化，缓解客户端漂移 |
| SCAFFOLD | 控制变量校正，Non-IID 下最优 |
| FedNova | 梯度归一化，支持异构设备 |
