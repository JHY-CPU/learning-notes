# 3_知识蒸馏用于持续学习

## 1. LwF (Learning without Forgetting)

LwF 由 Li & Hoiem (2017) 提出，是使用**知识蒸馏**解决持续学习的开创性工作。核心思想：在学习新任务时，用旧模型的输出作为"软标签"来保留旧知识。

### 1.1 核心思想

```
LwF 流程:

旧模型 f_old (冻结)
      ↓
输入 x → f_old(x) → 软标签 (旧任务输出)
      ↓
新模型 f_new (训练)
      ↓
同时优化:
  1. 新任务损失: L_new(f_new(x), y_new)
  2. 蒸馏损失: L_distill(f_new(x)_old_tasks, f_old(x))
```

### 1.2 损失函数

$$\mathcal{L}_{\text{LwF}} = \mathcal{L}_{\text{new}}(y_{\text{new}}, \hat{y}_{\text{new}}) + \lambda \cdot \mathcal{L}_{\text{KD}}(\hat{y}_{\text{old}}, y_{\text{old}}^{\text{soft}})$$

其中蒸馏损失为 KL 散度：

$$\mathcal{L}_{\text{KD}} = -\sum_i y_{\text{old},i}^{\text{soft}} \log \hat{y}_{\text{old},i}$$

## 2. PyTorch 实现

### 2.1 LwF 完整实现

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import copy

class LwF:
    """
    Learning without Forgetting 实现
    """
    def __init__(self, model, temperature=2.0, lambda_kd=1.0):
        self.model = model
        self.temperature = temperature
        self.lambda_kd = lambda_kd
        self.old_model = None
        self.task_output_sizes = {}  # 记录每个任务的输出维度

    def before_task(self, task_id, num_classes):
        """
        在学习新任务前的准备工作
        """
        # 保存旧模型
        self.old_model = copy.deepcopy(self.model)
        self.old_model.eval()

        # 记录新任务的输出维度范围
        start_idx = sum(self.task_output_sizes.values())
        self.task_output_sizes[task_id] = num_classes

        return start_idx, start_idx + num_classes

    def compute_kd_loss(self, new_logits, old_logits):
        """
        计算知识蒸馏损失

        Args:
            new_logits: 新模型对旧任务的输出
            old_logits: 旧模型的输出 (软标签)
        """
        # 温度缩放
        new_soft = F.log_softmax(new_logits / self.temperature, dim=1)
        old_soft = F.softmax(old_logits / self.temperature, dim=1)

        # KL 散度
        kd_loss = F.kl_div(new_soft, old_soft, reduction='batchmean')
        kd_loss = kd_loss * (self.temperature ** 2)

        return kd_loss

    def train_step(self, x, y_new, optimizer, task_id):
        """
        LwF 训练步骤
        """
        self.model.train()

        # 新模型的完整输出
        full_output = self.model(x)

        # 新任务损失
        start, end = self.get_task_output_range(task_id)
        new_logits = full_output[:, start:end]
        task_loss = F.cross_entropy(new_logits, y_new)

        # 知识蒸馏损失 (如果有旧模型)
        kd_loss = 0
        if self.old_model is not None and task_id > 0:
            with torch.no_grad():
                old_logits = self.old_model(x)

            # 新模型对旧任务输出部分
            old_task_logits = full_output[:, :start]
            kd_loss = self.compute_kd_loss(old_task_logits, old_logits)

        # 总损失
        total_loss = task_loss + self.lambda_kd * kd_loss

        optimizer.zero_grad()
        total_loss.backward()
        optimizer.step()

        return {
            'task_loss': task_loss.item(),
            'kd_loss': kd_loss.item() if isinstance(kd_loss, torch.Tensor) else 0,
            'total_loss': total_loss.item()
        }

    def get_task_output_range(self, task_id):
        """获取任务的输出索引范围"""
        start = sum(list(self.task_output_sizes.values())[:task_id])
        end = start + self.task_output_sizes[task_id]
        return start, end
```

### 2.2 带任务头的 LwF

```python
class MultiHeadModel(nn.Module):
    """
    多头模型: 共享特征提取器 + 每个任务独立的分类头
    """
    def __init__(self, backbone, feature_dim, num_classes_per_task):
        super().__init__()
        self.backbone = backbone
        self.heads = nn.ModuleDict()
        self.feature_dim = feature_dim
        self.current_task = 0

    def add_task_head(self, num_classes):
        """为新任务添加分类头"""
        head_name = f'task_{self.current_task}'
        self.heads[head_name] = nn.Linear(self.feature_dim, num_classes)
        self.current_task += 1

    def forward(self, x, task_id=None):
        features = self.backbone(x)

        if task_id is not None:
            # 只输出指定任务的预测
            return self.heads[f'task_{task_id}'](features)
        else:
            # 输出所有任务的预测
            outputs = {}
            for name, head in self.heads.items():
                outputs[name] = head(features)
            return outputs

class LwF_MultiHead:
    """
    多头架构的 LwF
    """
    def __init__(self, model, temperature=2.0, lambda_kd=1.0):
        self.model = model
        self.temperature = temperature
        self.lambda_kd = lambda_kd
        self.old_heads = {}  # 存储旧任务头的副本

    def before_new_task(self, task_id):
        """学习新任务前保存旧头"""
        if task_id > 0:
            for tid in range(task_id):
                head_name = f'task_{tid}'
                self.old_heads[tid] = copy.deepcopy(
                    self.model.heads[head_name]
                )
                self.old_heads[tid].eval()

    def compute_loss(self, x, y_new, task_id):
        """计算 LwF 损失"""
        features = self.model.backbone(x)

        # 新任务损失
        new_logits = self.model.heads[f'task_{task_id}'](features)
        task_loss = F.cross_entropy(new_logits, y_new)

        # 蒸馏损失
        kd_loss = 0
        for old_tid, old_head in self.old_heads.items():
            with torch.no_grad():
                old_logits = old_head(features)

            # 当前模型对旧任务的输出 (通过对应的头)
            new_old_logits = self.model.heads[f'task_{old_tid}'](features)

            # KL 散度
            new_soft = F.log_softmax(new_old_logits / self.temperature, dim=1)
            old_soft = F.softmax(old_logits / self.temperature, dim=1)
            kd_loss += F.kl_div(new_soft, old_soft, reduction='batchmean')

        if self.old_heads:
            kd_loss = kd_loss * (self.temperature ** 2) / len(self.old_heads)

        return task_loss + self.lambda_kd * kd_loss
```

## 3. 蒸馏策略

### 3.1 特征蒸馏

除了输出层蒸馏，还可以蒸馏中间特征：

```python
def feature_distillation_loss(new_features, old_features):
    """
    特征蒸馏: 保留中间层表示
    """
    # L2 损失
    l2_loss = F.mse_loss(new_features, old_features.detach())

    # 余弦相似度损失
    cos_loss = 1 - F.cosine_similarity(
        new_features, old_features.detach(), dim=1
    ).mean()

    return l2_loss + 0.5 * cos_loss
```

### 3.2 多层蒸馏

```python
class MultiLayerLwF:
    """
    多层知识蒸馏
    """
    def __init__(self, model, layer_names, temperature=2.0):
        self.model = model
        self.layer_names = layer_names
        self.temperature = temperature
        self.old_features = {}

        # 注册钩子
        self._register_hooks()

    def _register_hooks(self):
        """注册中间层特征钩子"""
        self.features = {}
        for name in self.layer_names:
            layer = dict(self.model.named_modules())[name]
            layer.register_forward_hook(
                lambda mod, inp, out, n=name: self.features.update({n: out})
            )

    def compute_multi_layer_loss(self, old_features_dict):
        """多层蒸馏损失"""
        total_loss = 0
        for name in self.layer_names:
            if name in self.features and name in old_features_dict:
                total_loss += F.mse_loss(
                    self.features[name],
                    old_features_dict[name].detach()
                )
        return total_loss / len(self.layer_names)
```

## 4. LwF vs EWC 对比

| 特性 | LwF | EWC |
|------|-----|-----|
| 核心机制 | 知识蒸馏 | Fisher 正则化 |
| 需要旧数据 | 否 | 否 (但需 Fisher) |
| 需要旧模型 | 是 | 否 |
| 存储需求 | 旧模型副本 | Fisher 矩阵 |
| 适用场景 | 任务增量 | 任务/类增量 |
| 计算开销 | 中 | 中 |

## 5. 超参数调优

| 参数 | 作用 | 推荐值 | 注意事项 |
|------|------|--------|----------|
| $T$ (温度) | 软化输出分布 | 2-4 | 过大则信息丢失 |
| $\lambda_{KD}$ | 蒸馏权重 | 0.5-5.0 | 平衡新旧任务 |
| 蒸馏层 | 特征蒸馏位置 | 后几层 | 包含高级语义 |

## 6. 总结

| 要点 | 说明 |
|------|------|
| 核心思想 | 用旧模型输出作为软标签 |
| 优势 | 不需要存储旧数据、实现简单 |
| 损失函数 | 新任务损失 + 蒸馏损失 (KL 散度) |
| 关键技巧 | 温度缩放、多层蒸馏 |
