# 0_持续学习概述与挑战

## 1. 持续学习定义

持续学习 (Continual Learning, CL)，也称为终身学习 (Lifelong Learning) 或增量学习 (Incremental Learning)，是指模型在**不断接收新任务/新数据**的过程中，能够**保留已有知识**并**持续学习新知识**的能力。

### 1.1 与传统学习的对比

| 特性 | 传统批量学习 | 持续学习 |
|------|-------------|----------|
| 数据访问 | 全部数据同时可用 | 数据按任务/时间到达 |
| 训练方式 | 一次性训练 | 顺序训练 |
| 存储需求 | 存储全部数据 | 可能无法存储历史数据 |
| 适应性 | 固定任务 | 动态新任务 |
| 核心挑战 | 过拟合 | **灾难性遗忘** |

### 1.2 持续学习的三种场景

```
场景 1: 任务增量学习 (Task-Incremental Learning)
  每个任务有不同的输出空间
  推理时知道任务 ID
  例: Task 1: 分类猫狗, Task 2: 分类汽车型号

场景 2: 域增量学习 (Domain-Incremental Learning)
  输出空间相同，但数据分布变化
  不需要任务 ID
  例: 同一个人脸识别任务，但光照条件变化

场景 3: 类增量学习 (Class-Incremental Learning)
  逐步增加新类别
  需要区分所有已学类
  例: 先学 10 类，再学 10 类，最终需区分 20 类
```

## 2. 灾难性遗忘问题

### 2.1 问题定义

灾难性遗忘 (Catastrophic Forgetting) 是指神经网络在学习新任务时，**急剧遗忘旧任务的知识**。这是持续学习面临的核心挑战。

```
灾难性遗忘示意:

训练 Task 1 后:  [Task1 Acc: 95%]
训练 Task 2 后:  [Task1 Acc: 30%, Task2 Acc: 92%]  ← Task1 性能大幅下降！
训练 Task 3 后:  [Task1 Acc: 15%, Task2 Acc: 25%, Task3 Acc: 90%] ← 完全遗忘

理想持续学习:
训练 Task 1 后:  [Task1 Acc: 95%]
训练 Task 2 后:  [Task1 Acc: 93%, Task2 Acc: 94%]
训练 Task 3 后:  [Task1 Acc: 91%, Task2 Acc: 92%, Task3 Acc: 93%]
```

### 2.2 遗忘的原因

| 原因 | 描述 |
|------|------|
| 参数覆盖 | 新任务的梯度更新覆盖了旧任务学到的权重 |
| 特征冲突 | 不同任务需要不同的特征表示 |
| 决策边界移动 | 新任务改变决策边界，影响旧任务分类 |
| 优化冲突 | 不同任务的梯度方向可能相反 |

### 2.3 数学形式化

给定任务序列 $\mathcal{T}_1, \mathcal{T}_2, \ldots, \mathcal{T}_T$，每个任务有数据分布 $\mathcal{D}_t$。在学习任务 $t$ 后，目标是最小化所有已学任务的累计损失：

$$\min_\theta \sum_{k=1}^{t} \mathbb{E}_{(x,y) \sim \mathcal{D}_k} [\mathcal{L}(f_\theta(x), y)]$$

但训练时只能访问当前任务 $\mathcal{D}_t$ 的数据，导致旧任务损失增加。

## 3. 持续学习方法分类

```
持续学习方法
├── 基于正则化 (Regularization-based)
│   ├── EWC: 弹性权重固化
│   ├── SI: 突触智能
│   └── LwF: 学习而不遗忘
├── 基于回放 (Replay-based)
│   ├── 经验回放: 存储旧样本
│   ├── 生成回放: 用生成模型
│   └── iCaRL: 增量分类器和表示学习
├── 基于架构 (Architecture-based)
│   ├── Progressive Nets: 渐进式网络
│   ├── PackNet: 打包网络
│   └── 动态扩展
└── 基于优化 (Optimization-based)
    ├── GEM: 梯度情景记忆
    ├── OGD: 正交梯度下降
    └── 记忆感知采样
```

## 4. 评估指标

### 4.1 常用指标

```python
def compute_cl_metrics(accuracy_matrix):
    """
    计算持续学习评估指标

    Args:
        accuracy_matrix: T x T 矩阵, A[i][j] 表示训练完任务 i 后在任务 j 上的准确率

    例:
    A = [[0.95, 0.00, 0.00],   # 训练完 Task 0
         [0.30, 0.92, 0.00],   # 训练完 Task 1
         [0.15, 0.25, 0.90]]   # 训练完 Task 2
    """
    T = len(accuracy_matrix)

    # 1. 平均准确率 (Average Accuracy)
    avg_acc = np.mean([accuracy_matrix[T-1][j] for j in range(T)])

    # 2. 向后迁移 (Backward Transfer) - 遗忘程度
    bwt = 0
    for j in range(T-1):
        bwt += (accuracy_matrix[T-1][j] - accuracy_matrix[j][j])
    bwt /= (T - 1) if T > 1 else 1

    # 3. 向前迁移 (Forward Transfer) - 新任务帮助程度
    fwt = 0
    for j in range(1, T):
        # 基线: 随机初始化在任务 j 上的准确率
        baseline_j = accuracy_matrix[0][j]  # 简化
        fwt += (accuracy_matrix[j-1][j] - baseline_j)
    fwt /= (T - 1) if T > 1 else 1

    # 4. 遗忘度量 (Forgetting Measure)
    forget = 0
    for j in range(T-1):
        max_acc = max(accuracy_matrix[i][j] for i in range(j, T))
        forget += (max_acc - accuracy_matrix[T-1][j])
    forget /= (T - 1) if T > 1 else 1

    return {
        'Average Accuracy': avg_acc,
        'Backward Transfer': bwt,  # 负值表示遗忘
        'Forward Transfer': fwt,   # 正值表示正迁移
        'Forgetting Measure': forget
    }
```

### 4.2 指标解释

| 指标 | 理想值 | 含义 |
|------|--------|------|
| 平均准确率 | 越高越好 | 最终所有任务的平均性能 |
| 向后迁移 (BWT) | > 0 | 学习新任务对旧任务的帮助 (通常 < 0) |
| 向前迁移 (FWT) | > 0 | 已学知识对新任务的帮助 |
| 遗忘度量 | 0 | 遗忘了多少旧知识 |

## 5. 数据集与基准

| 基准 | 任务数 | 类别数 | 说明 |
|------|--------|--------|------|
| Split MNIST | 5 | 2/task | 基础基准 |
| Split CIFAR-10 | 5 | 2/task | 中等难度 |
| Split CIFAR-100 | 10-20 | 5-10/task | 高难度 |
| CORe50 | 11 | 50 | 视觉持续学习 |
| Split MiniImageNet | 20 | 5/task | 细粒度 |
| Continual-NLU | 变化 | 变化 | NLP 持续学习 |

## 6. 总结

| 要点 | 说明 |
|------|------|
| 核心问题 | 灾难性遗忘 |

| 主要方法 | 正则化、回放、架构、优化 |
