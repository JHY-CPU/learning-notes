# 12_隐私保护机制

## 1. 联邦学习隐私威胁

虽然联邦学习不共享原始数据，但模型更新 (梯度/参数) 仍可能泄露隐私。研究表明，通过分析梯度可以**重构训练数据**。

### 1.1 攻击类型

```
联邦学习隐私攻击:
├── 梯度反演攻击 (Gradient Inversion)
│   └── 从梯度重构原始数据
├── 成员推理攻击
│   └── 判断样本是否在某客户端训练集中
├── 属性推断攻击
│   └── 推断训练数据的统计属性
└── 模型窃取攻击
    └── 通过查询复制模型能力
```

### 1.2 梯度反演攻击

```python
def gradient_inversion_attack(model, gradient, image_shape, num_steps=1000):
    """
    从梯度重构原始数据
    Zhu et al. (2019): Deep Leakage from Gradients
    """
    # 初始化随机数据
    dummy_x = torch.randn(1, *image_shape, requires_grad=True)
    dummy_y = torch.randn(1, 10, requires_grad=True)  # 假设 10 类

    optimizer = torch.optim.LBFGS([dummy_x, dummy_y])

    for step in range(num_steps):
        def closure():
            optimizer.zero_grad()

            # 用 dummy 数据计算梯度
            output = model(dummy_x)
            loss = F.cross_entropy(output, dummy_y.softmax(dim=1))
            dummy_grad = torch.autograd.grad(loss, model.parameters(),
                                              create_graph=True)

            # 匹配给定梯度
            grad_diff = 0
            for dg, g in zip(dummy_grad, gradient):
                grad_diff += ((dg - g) ** 2).sum()

            grad_diff.backward()
            return grad_diff

        optimizer.step(closure)

    return dummy_x.detach(), dummy_y.detach()
```

## 2. 安全聚合 (Secure Aggregation)

### 2.1 原理

安全聚合确保服务器只能获取**聚合后的全局模型**，而无法获取任何单个客户端的模型更新。

```
安全聚合流程:

客户端 1: w_1 + mask_1 ──→ ┐
客户端 2: w_2 + mask_2 ──→ ├→ 服务器: Σ(w_k + mask_k) = Σw_k (因为 Σmask_k = 0)
客户端 3: w_3 + mask_3 ──→ ┘

关键: mask_1 + mask_2 + mask_3 = 0
```

### 2.2 实现

```python
class SecureAggregation:
    """
    基于掩码的安全聚合
    """
    def __init__(self, num_clients, seed=42):
        self.num_clients = num_clients
        self.rng = np.random.RandomState(seed)

    def generate_pairwise_keys(self):
        """
        生成客户端对之间的共享密钥
        """
        shared_keys = {}
        for i in range(self.num_clients):
            for j in range(i+1, self.num_clients):
                key = self.rng.bytes(32)
                shared_keys[(i, j)] = key
        return shared_keys

    def generate_mask(self, client_id, shared_keys, model_shape):
        """
        生成客户端的掩码
        掩码设计保证所有客户端掩码之和为 0
        """
        mask = torch.zeros(model_shape)

        for (i, j), key in shared_keys.items():
            # 使用共享密钥生成确定性随机数
            prng = np.random.RandomState(
                int.from_bytes(key[:4], 'big')
            )
            pairwise_mask = torch.from_numpy(
                prng.randn(*model_shape).astype(np.float32)
            )

            if i == client_id:
                mask += pairwise_mask
            elif j == client_id:
                mask -= pairwise_mask

        return mask

    def client_upload(self, model_update, client_id, shared_keys):
        """
        客户端添加掩码后上传
        """
        mask = self.generate_mask(
            client_id, shared_keys, model_update.shape
        )
        masked_update = model_update + mask
        return masked_update

    def server_aggregate(self, masked_updates):
        """
        服务器聚合: 掩码相消
        """
        # 所有掩码之和为 0，聚合后只剩模型更新之和
        aggregated = torch.stack(masked_updates).sum(dim=0)
        return aggregated / len(masked_updates)
```

## 3. 差分隐私联邦学习

### 3.1 DP-FedAvg

```python
class DPFedAvgClient:
    """
    带差分隐私的 FedAvg 客户端
    """
    def __init__(self, local_data, noise_multiplier=0.1,
                 max_grad_norm=1.0):
        self.local_data = local_data
        self.noise_multiplier = noise_multiplier
        self.max_grad_norm = max_grad_norm

    def local_train(self, model, global_model, local_epochs=5):
        """
        本地训练 + 梯度裁剪 + 噪声添加
        """
        model.train()
        optimizer = torch.optim.SGD(model.parameters(), lr=0.01)

        for epoch in range(local_epochs):
            for x, y in self.local_data:
                optimizer.zero_grad()
                output = model(x)
                loss = F.cross_entropy(output, y)
                loss.backward()

                # 梯度裁剪
                torch.nn.utils.clip_grad_norm_(
                    model.parameters(), self.max_grad_norm
                )

                optimizer.step()

        # 计算模型更新
        update = {}
        for (name, param), (_, global_param) in zip(
            model.named_parameters(), global_model.named_parameters()
        ):
            update[name] = param.data - global_param.data

            # 裁剪更新
            update_norm = update[name].norm()
            if update_norm > self.max_grad_norm:
                update[name] = update[name] * (self.max_grad_norm / update_norm)

            # 添加高斯噪声
            noise = torch.randn_like(update[name]) * \
                    self.noise_multiplier * self.max_grad_norm
            update[name] += noise

        return update
```

## 4. 同态加密

### 4.1 原理

同态加密允许在**加密数据上直接计算**，服务器无需解密即可聚合模型。

```
同态加密聚合:

客户端 1: Enc(w_1) ──→ ┐
客户端 2: Enc(w_2) ──→ ├→ 服务器: Enc(w_1) + Enc(w_2) + Enc(w_3)
客户端 3: Enc(w_3) ──→ ┘     = Enc(w_1 + w_2 + w_3)

服务器返回 Enc(w_1 + w_2 + w_3) 给客户端
客户端解密得到 w_1 + w_2 + w_3
```

### 4.2 实现 (使用 PySyft)

```python
# 使用 PySyft 库的同态加密示例
import syft as sy

def homomorphic_aggregation(client_models):
    """
    同态加密聚合
    """
    hook = sy.TorchHook(torch)

    # 创建虚拟工人 (客户端)
    workers = [sy.VirtualWorker(hook, id=f'client_{i}')
               for i in range(len(client_models))]

    # 加密并发送模型
    encrypted_models = []
    for model, worker in zip(client_models, workers):
        encrypted = model.send(worker)
        encrypted_models.append(encrypted)

    # 服务器端聚合 (在加密状态下)
    aggregated = encrypted_models[0]
    for model in encrypted_models[1:]:
        aggregated = aggregated + model

    return aggregated
```

## 5. 隐私保护方法对比

| 方法 | 隐私保证 | 通信开销 | 计算开销 | 模型质量 |
|------|----------|----------|----------|----------|
| 无保护 | 无 | 低 | 低 | 最好 |
| 安全聚合 | 防服务器窥探 | 低 | 中 | 无影响 |
| 差分隐私 | 数学可证明 | 低 | 低 | 下降 |
| 同态加密 | 密文计算 | 高 | **高** | 无影响 |
| 安全多方计算 | 密文计算 | 高 | 高 | 无影响 |

## 6. 总结

| 要点 | 说明 |
|------|------|
| 隐私威胁 | 梯度反演可重构原始数据 |
| 安全聚合 | 掩码技术，服务器无法获取单个更新 |
| 差分隐私 | 噪声添加，数学可证明的隐私保证 |
| 同态加密 | 密文计算，最高隐私但计算开销大 |
