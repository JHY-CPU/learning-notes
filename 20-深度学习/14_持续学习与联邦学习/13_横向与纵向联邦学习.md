# 13_横向与纵向联邦学习

## 1. 联邦学习分类

根据数据划分方式，联邦学习分为三类：

### 1.1 数据划分维度

```
数据矩阵:
              特征 X1  特征 X2  特征 X3  标签 Y
样本 1          1.2      3.4      5.6      1
样本 2          2.3      4.5      6.7      0
样本 3          3.4      5.6      7.8      1
...

横向联邦 (样本划分): 不同客户端有不同的样本，特征相同
  客户端 A: [样本1, 样本2]  (特征 X1,X2,X3)
  客户端 B: [样本3, 样本4]  (特征 X1,X2,X3)

纵向联邦 (特征划分): 不同客户端有不同的特征，样本相同
  客户端 A: [特征X1, 特征X2]  (样本1,2,3,...)
  客户端 B: [特征X3, 标签Y]   (样本1,2,3,...)
```

### 1.2 三种联邦学习

| 类型 | 数据划分 | 特征空间 | 样本空间 | 典型场景 |
|------|----------|----------|----------|----------|
| 横向联邦 | 样本划分 | 相同 | 不同 | 不同地区银行 |
| 纵向联邦 | 特征划分 | 不同 | 相同 | 银行 + 电商 |
| 联邦迁移学习 | 都不同 | 不同 | 不同 | 跨行业合作 |

## 2. 横向联邦学习 (HFL)

### 2.1 系统架构

```
横向联邦学习架构:

客户端 A              客户端 B              客户端 C
[本地数据]           [本地数据]           [本地数据]
[特征 X1,X2,X3]     [特征 X1,X2,X3]     [特征 X1,X2,X3]
[标签 Y]            [标签 Y]             [标签 Y]
     ↓                    ↓                    ↓
[本地模型训练]       [本地模型训练]       [本地模型训练]
     ↓                    ↓                    ↓
     └──────────────→ 中央服务器 ←──────────────┘
                      [模型聚合]
```

### 2.2 特点

- 各客户端拥有**相同的特征空间**
- 各客户端拥有**不同的样本**
- 最常见的联邦学习形式
- 典型应用：手机键盘预测、医疗协作

## 3. 纵向联邦学习 (VFL)

### 3.1 系统架构

```
纵向联邦学习架构:

客户端 A (银行)        客户端 B (电商)
[特征: 存款, 贷款]     [特征: 购买记录, 浏览历史]
[部分样本]             [同一样本的不同特征]
      ↘                  ↙
       安全对齐 (样本 ID)
             ↓
       拆分模型训练
             ↓
       各方持有部分模型
```

### 3.2 样本对齐

在纵向联邦中，首先需要**对齐双方共有的样本**，通常使用隐私保护的集合求交 (PSI)：

```python
def private_set_intersection(set_a, set_b):
    """
    隐私保护的集合求交 (PSI)
    双方在不暴露各自完整集合的情况下找到交集
    """
    # 简化示例 (实际使用加密协议)
    # 客户端 A: 对样本 ID 哈希
    hashed_a = {hash_fn(id) for id in set_a}

    # 客户端 B: 对样本 ID 哈希
    hashed_b = {hash_fn(id) for id in set_b}

    # 找交集
    intersection = hashed_a & hashed_b

    return intersection
```

### 3.3 拆分学习

```python
class SplitLearning:
    """
    拆分学习: 纵向联邦的核心训练机制
    """
    def __init__(self, model_a, model_b):
        self.model_a = model_a  # 客户端 A 的部分模型
        self.model_b = model_b  # 客户端 B 的部分模型

    def train_step(self, x_a, x_b, y, optimizer_a, optimizer_b):
        """
        拆分学习一步训练
        """
        # 客户端 A 前向传播
        hidden_a = self.model_a(x_a)

        # 安全传输: 将隐藏表示发送给客户端 B
        # (实际中需要加密)
        hidden_a_detached = hidden_a.detach().clone()

        # 客户端 B 前向传播
        hidden_b = self.model_b(x_b)

        # 拼接特征并计算损失
        combined = torch.cat([hidden_a_detached, hidden_b], dim=-1)
        output = self.final_layer(combined)
        loss = F.cross_entropy(output, y)

        # 反向传播
        loss.backward()

        # 客户端 B 更新
        optimizer_b.zero_grad()
        loss.backward(retain_graph=True)
        optimizer_b.step()

        # 将梯度传回客户端 A (安全传输)
        grad_for_a = hidden_a.grad.detach().clone()

        # 客户端 A 更新
        optimizer_a.zero_grad()
        hidden_a.backward(grad_for_a)
        optimizer_a.step()

        return loss.item()
```

### 3.4 垂直联邦的同态加密聚合

```python
class VFLWithHE:
    """
    带同态加密的纵向联邦学习
    """
    def __init__(self, he_context):
        self.he = he_context  # 同态加密上下文

    def secure_forward(self, x_a, x_b, model_a, model_b):
        """
        安全前向传播: 使用同态加密保护中间结果
        """
        # 客户端 A: 加密隐藏表示
        hidden_a = model_a(x_a)
        encrypted_hidden = self.he.encrypt(hidden_a)

        # 发送加密的隐藏表示给 B
        # 客户端 B: 在密文上计算
        hidden_b = model_b(x_b)
        encrypted_hidden_b = self.he.encrypt(hidden_b)

        # 同态加法: Enc(h_a) + Enc(h_b) = Enc(h_a + h_b)
        encrypted_combined = self.he.add(encrypted_hidden, encrypted_hidden_b)

        # 结果方解密
        combined = self.he.decrypt(encrypted_combined)
        output = self.final_layer(combined)

        return output
```

## 4. 联邦迁移学习

### 4.1 适用场景

当两个参与方的**特征空间和样本空间都不同时**：

```
联邦迁移学习场景:

医疗数据 (特征: CT扫描, 血检指标)  ×  患者集合 A
基因数据 (特征: 基因序列, 蛋白质)  ×  患者集合 B

两个机构希望合作，但:
- 特征不同 (医疗 vs 基因)
- 患者集合有少量重叠
```

### 4.2 方法

```python
class FederatedTransferLearning:
    """
    联邦迁移学习
    """
    def __init__(self, model_a, model_b, shared_dim=32):
        self.model_a = model_a
        self.model_b = model_b
        self.shared_dim = shared_dim

        # 公共表示空间映射
        self.projector_a = nn.Linear(model_a.output_dim, shared_dim)
        self.projector_b = nn.Linear(model_b.output_dim, shared_dim)

    def compute_alignment_loss(self, x_a, x_b, common_samples):
        """
        对齐公共样本的表示
        """
        # 公共样本的表示
        h_a = self.projector_a(self.model_a(x_a[common_samples]))
        h_b = self.projector_b(self.model_b(x_b[common_samples]))

        # 对齐损失
        alignment_loss = F.mse_loss(h_a, h_b)

        return alignment_loss
```

## 5. 三种类型对比

| 特性 | 横向联邦 | 纵向联邦 | 联邦迁移学习 |
|------|----------|----------|-------------|
| 数据划分 | 样本 | 特征 | 都不同 |
| 特征空间 | 相同 | 不同 | 不同 |
| 样本空间 | 不同 | 相同 | 少量重叠 |
| 核心算法 | FedAvg | 拆分学习 | 表示对齐 |
| 隐私挑战 | 梯度泄露 | 中间表示泄露 | 信息对齐 |
| 典型场景 | 手机数据 | 银行+电商 | 跨行业 |

## 6. 总结

| 要点 | 说明 |
|------|------|
| 横向联邦 | 相同特征，不同样本 |
| 纵向联邦 | 不同特征，相同样本 |
| 联邦迁移 | 都不同，需要表示对齐 |
| 核心技术 | FedAvg、拆分学习、PSI |
