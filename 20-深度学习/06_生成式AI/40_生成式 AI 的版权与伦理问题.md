# 40_生成式 AI 的版权与伦理问题

## 核心概念

- **训练数据的版权争议**：生成模型（如 Stable Diffusion）在未经原作者同意的情况下，使用互联网上的版权图像进行训练。这导致了多起集体诉讼（如 Getty Images v. Stability AI, Andersen v. Stability AI）。
- **作品的"实质性相似"**：当生成模型输出的图像与特定版权作品的风格、内容或标志性元素高度相似时，是否构成侵权？这是法律界仍在争论的问题。
- **AI 生成物的版权归属**：美国版权局裁定"AI 生成的作品没有人类作者则不享有版权"，但"人类在 AI 辅助下进行创造性选择"的作品可以被保护。这为"AI 作为工具"留下了空间。
- **深度伪造 (Deepfake)**：生成式 AI 可创建高度逼真的虚假图像/视频，用于伪造名人言论、虚假新闻、色情内容等，对社会信任造成严重威胁。
- **偏见与公平性 (Bias and Fairness)**：训练数据反映的刻板印象和偏见（如种族、性别、年龄偏见）会被生成模型继承和放大——例如"CEO"的生成结果偏向白种男性和。
- **技术滥用的防控**：C2PA 内容来源认证、AI 生成内容水印（如 Stable Signature）、检测模型（如 Deepfake 检测）等技术手段用于防控生成式 AI 的滥用。
- **模型蒸馏与数据蒸馏**：即使模型不是直接在版权数据上训练的（通过知识蒸馏取自其它模型），版权问题仍然存在——"清洁训练"并不能保证不侵权。

## 数学推导

**版权侵权的法律标准"实质性相似"的量化尝试**：

一种可能的量化——提取特征后计算生成图像与版权图像的余弦相似度：

$$
\text{similarity}(x_{\text{gen}}, x_{\text{ref}}) = \frac{\phi(x_{\text{gen}}) \cdot \phi(x_{\text{ref}})}{\|\phi(x_{\text{gen}})\| \cdot \|\phi(x_{\text{ref}})\|}
$$

但法律标准远比数学相似度复杂——需考虑"表达"而非"思想"的复制。

**Memorization (记忆化) 的量化**：

模型"记忆"训练数据的程度可以通过成员推理攻击（Membership Inference Attack）评估：

$$
\text{Memorization}(x) = \log p_\theta(x) - \max_{x' \neq x} \log p_\theta(x')
$$

如果某个训练样本的似然显著高于其他类似样本，说明模型"记住了"该样本。

**去重训练 (Deduplication) 的数学依据**：

SemDeDup 等方法在特征空间中移除重复/近重复图像：

$$
\text{Sim}(x_i, x_j) > \text{threshold} \to \text{删除 } x_j
$$

去重可以显著降低模型记忆化特定训练样本的风险。

## 直观理解

- **训练数据的版权问题 = AI 看了太多盗版书**：如果一个人看了一万本盗版小说后自己写了一本新小说，他是否侵权？法律说"不侵权（思想不版权）"，但如果他逐句背诵了一整章——那就是侵权（表达版权）。AI 同样面临这个"思想 vs 表达"的模糊界限。
- **"清洁训练"的幻觉**：一些公司声称模型只在"清洁数据"上训练——但清洁数据只是"没有版权问题的数据"（如公有领域图像），不代表模型不会从这些数据中学习到与版权作品"实质性相似"的模式。
- **深度伪造的困境 = 照片级 ps 的终极形态**：以前 PS 一张假图需要几天，现在 GenAI 几秒就能生成以假乱真的内容。技术检测手段只能"被动响应"，无法从根源上阻止伪造。
- **Bias in, Bias out**：如果训练数据是"90% 的白人男性 CEO 照片"，那模型生成的"CEO"图像自然也会偏向白人男性——这不仅是技术问题，更是社会公平问题。

## 代码示例

```python
import torch
import torch.nn.functional as F

class MembershipInferenceAttack:
    """
    成员推理攻击：检测模型是否"记住"了特定样本
    
    如果某个训练样本的概率显著高于测试样本，可能存在记忆化
    """
    def __init__(self, model, ref_dataset):
        self.model = model
        self.ref_dataset = ref_dataset
    
    def compute_perplexity(self, x):
        """计算样本的似然/困惑度"""
        with torch.no_grad():
            # 在扩散模型中，可以计算 ELBO 作为似然的近似
            # 这里用简化的方式表示
            loss = F.mse_loss(self.model(x), x)
            return -loss.item()  # 值越大，模型对该样本越有信心
    
    def detect_memorization(self, target_samples, threshold=0.8):
        """检测记忆化水平"""
        # 计算目标样本的似然
        target_likelihoods = [self.compute_perplexity(s) for s in target_samples]
        
        # 计算参考样本的似然
        ref_likelihoods = []
        for i, ref in enumerate(self.ref_dataset[:100]):
            ref_likelihoods.append(self.compute_perplexity(ref))
        
        # 如果目标样本的似然显著高于参考样本，说明可能被记忆
        mean_ref = sum(ref_likelihoods) / len(ref_likelihoods)
        std_ref = (sum((l - mean_ref)**2 for l in ref_likelihoods) / len(ref_likelihoods))**0.5
        
        results = []
        for i, (sample, likelihood) in enumerate(zip(target_samples, target_likelihoods)):
            z_score = (likelihood - mean_ref) / (std_ref + 1e-8)
            is_memorized = z_score > threshold
            results.append({
                'sample_id': i,
                'likelihood': likelihood,
                'z_score': z_score,
                'is_memorized': is_memorized,
            })
        
        return results

class ContentFingerprint:
    """内容指纹与相似度检测"""
    def __init__(self, feature_extractor):
        self.extractor = feature_extractor
    
    def compute_similarity(self, gen_image, ref_image):
        """计算生成图像与参考图像的感知相似度"""
        with torch.no_grad():
            feat_gen = self.extractor(gen_image)
            feat_ref = self.extractor(ref_image)
            
            # 余弦相似度
            similarity = F.cosine_similarity(feat_gen, feat_ref, dim=-1)
            return similarity.item()
    
    def batch_detect(self, gen_images, ref_database, threshold=0.9):
        """批量检测生成图像是否与数据库中任何图像高度相似"""
        results = []
        for gen_img in gen_images:
            max_sim = 0.0
            most_similar_idx = -1
            
            for i, ref_img in enumerate(ref_database):
                sim = self.compute_similarity(gen_img, ref_img)
                if sim > max_sim:
                    max_sim = sim
                    most_similar_idx = i
            
            results.append({
                'max_similarity': max_sim,
                'most_similar_idx': most_similar_idx,
                'exceeds_threshold': max_sim > threshold,
            })
        
        return results

# 公平性审计
class BiasAuditor:
    """检测生成模型中的偏见"""
    def __init__(self, model, attribute_categories):
        self.model = model
        self.categories = attribute_categories
    
    def audit_profession_bias(self, profession="CEO", n_samples=100):
        """检测某个职业的生成偏见"""
        results = {}
        for attr_name, attr_values in self.categories.items():
            counts = {v: 0 for v in attr_values}
            
            for _ in range(n_samples):
                # 生成职业图像
                image = self.model.generate(f"a photo of a {profession}")
                
                # 使用属性分类器检测（模拟）
                predicted_attr = self._classify_attribute(image, attr_name)
                counts[predicted_attr] += 1
            
            total = sum(counts.values())
            distribution = {k: v/total for k, v in counts.items()}
            results[attr_name] = distribution
        
        return results
    
    def _classify_attribute(self, image, attr_name):
        """模拟属性分类器"""
        # 在实际实现中使用训练好的属性分类器
        import random
        return random.choice(self.categories[attr_name])
    
    def compute_fairness_metric(self, distribution):
        """计算公平性指标（越均匀越公平）"""
        import math
        n = len(distribution)
        # 归一化熵
        entropy = -sum(p * math.log(p + 1e-10) for p in distribution.values())
        max_entropy = math.log(n)
        return entropy / max_entropy  # 1 = 完全公平, 0 = 完全不公

print()
print("核心争议点:")
print("1. 训练数据的版权 — 使用版权作品训练是否构成侵权？")
print("2. AI 生成物的版权归属 — 谁拥有 AI 生成的作品？")
print("3. 深度伪造 — 虚假内容的检测与防控")
print("4. 偏见与公平 — 训练数据偏见在模型中的放大")
print("5. 隐私问题 — 模型记忆训练样本中的个人信息")
print()

# 模拟公平性审计
auditor = BiasAuditor(None, {
    'gender': ['male', 'female'],
    'skin_tone': ['light', 'medium', 'dark'],
})

print("偏见审计示例（模拟数据）:")
test_dist = {'male': 0.85, 'female': 0.15}
fairness = auditor.compute_fairness_metric(test_dist)
print(f"  CEO 性别分布: male=85%, female=15%")
print(f"  公平性指标: {fairness:.3f} (1.0 = 完全公平)")
```

## 深度学习关联

- **生成模型的版权后处理**：Stable Signature (2023) 使用深度学习水印技术，在生成过程中将不可见的识别码嵌入图像中，便于追溯生成来源——即使经过裁剪、缩放等变换仍可检测。
- **概念擦除 (Concept Erasure)**：在模型层面删除特定概念的生成能力——如"擦除"版权风格（如"宫崎骏风格"）或有害内容（如"暴力"），通过微调或推理时干预实现。
- **差分隐私训练 (DP-SGD)**：通过在训练中加入噪声来保护训练数据的隐私，使得模型不会"记住"任何单个训练样本。但 DP 训练会显著降低生成质量。
- **负责任的 AI 发布框架**：Stability AI 的"负责任 AI 发布"、OpenAI 的"渐进式部署"等框架——在模型发布前进行偏见审计、有害内容过滤测试、红队攻防演练。
