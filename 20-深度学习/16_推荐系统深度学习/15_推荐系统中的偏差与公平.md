# 15_推荐系统中的偏差与公平

## 1. 偏差类型概览

推荐系统存在多种偏差，如果不加以处理，会导致推荐结果不准确或不公平。

```
数据采集偏差
  ├── 位置偏差: 排在前面的物品更容易被点击
  ├── 曝光偏差: 只有被推荐的物品才有反馈
  ├── 选择偏差: 观测到的数据不代表全分布
  └── 流行度偏差: 热门物品更容易获得交互

用户偏差
  ├── 点击偏差: 用户可能点击但不真的喜欢
  ├── 行为一致性偏差: 用户倾向于与相似物品交互
  └── 满意度偏差: 用户可能将就使用不满意的推荐

系统偏差
  ├── 反馈循环: 推荐影响行为，行为训练模型
  ├── 过滤气泡: 推荐越来越窄
  └── 马太效应: 热门物品越来越热
```

## 2. 位置偏差

位置偏差：排名靠前的物品更容易被点击，**与物品质量无关**。

$$P(\text{click} | \text{examined}, \text{relevance}=r) = r$$
$$P(\text{examined} | \text{position}=k) = \eta_k$$

```python
import torch
import torch.nn as nn

class PositionDebiasedModel(nn.Module):
    """位置去偏差模型"""
    def __init__(self, base_model, max_position=20):
        super().__init__()
        self.base_model = base_model
        # 可学习的位置偏差
        self.position_bias = nn.Embedding(max_position, 1)

    def forward(self, user_features, item_features, positions):
        # 基础相关性得分
        relevance = self.base_model(user_features, item_features)

        # 位置偏差
        pos_bias = self.position_bias(positions).squeeze(-1)

        # 观测点击率 = 相关性 × 检查概率
        observed_ctr = torch.sigmoid(relevance + pos_bias)
        return observed_ctr, relevance

    def predict_unbiased(self, user_features, item_features):
        """推理时只使用相关性（去除位置偏差）"""
        return self.base_model(user_features, item_features)


def train_position_debiased(model, optimizer, data_loader, n_epochs=20):
    """IPW (Inverse Propensity Weighting) 训练"""
    for epoch in range(n_epochs):
        for user_feat, item_feat, positions, clicks in data_loader:
            # 观测点击率预测
            obs_ctr, relevance = model(user_feat, item_feat, positions)

            # 位置权重（逆倾向加权）
            # 假设位置偏差已知或估计得到
            propensity = model.position_bias.weight.data[positions].sigmoid()
            weights = 1.0 / (propensity + 1e-6)
            weights = torch.clamp(weights, max=100)  # 防止极端权重

            # 加权损失
            loss = nn.BCELoss(reduction='none')(obs_ctr, clicks.float())
            weighted_loss = (loss * weights.detach()).mean()

            optimizer.zero_grad()
            weighted_loss.backward()
            optimizer.step()
```

## 3. 曝光偏差

只有被推荐的物品才有反馈，导致数据不完整。

```python
class ExposureDebiasing(nn.Module):
    """曝光去偏差：训练一个曝光模型"""
    def __init__(self, input_dim, hidden_dim=128):
        super().__init__()
        # 曝光模型：预测物品是否被展示给用户
        self.exposure_model = nn.Sequential(
            nn.Linear(input_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, 1),
            nn.Sigmoid()
        )

        # 推荐模型
        self.recommend_model = nn.Sequential(
            nn.Linear(input_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, 1),
            nn.Sigmoid()
        )

    def forward(self, features, is_exposed):
        # 预测曝光概率
        p_exposure = self.exposure_model(features)

        # 预测点击概率
        p_click = self.recommend_model(features)

        # 修正后的概率
        # P(real_click) = P(observed_click) / P(exposure)
        p_unbiased = p_click / (p_exposure + 1e-6)

        return p_click, p_exposure, p_unbiased

    def loss(self, features, clicks, is_exposed):
        p_click, p_exposure, _ = self.forward(features, is_exposed)

        # 曝光模型损失
        exposure_loss = nn.BCELoss()(p_exposure, is_exposed.float())

        # 推荐模型损失（只在曝光样本上计算）
        click_loss = nn.BCELoss()(p_click[is_exposed], clicks[is_exposed].float())

        return click_loss + 0.1 * exposure_loss
```

## 4. 流行度偏差与马太效应

```python
class PopularityDebiasing:
    """流行度去偏差"""
    def __init__(self, item_popularity, method='ips'):
        """
        item_popularity: dict {item_id: 交互次数}
        """
        self.popularity = item_popularity
        self.method = method

    def compute_propensity(self, n_users):
        """计算每个物品的倾向得分"""
        total_interactions = sum(self.popularity.values())
        propensities = {}
        for item_id, count in self.popularity.items():
            propensities[item_id] = count / total_interactions * n_users
        return propensities

    def ipw_weight(self, item_id):
        """逆倾向加权：抑制热门物品，提升冷门物品"""
        propensity = self.popularity.get(item_id, 1)
        return 1.0 / propensity

    def causal_recommend(self, model, user_id, candidates, alpha=0.5):
        """因果推荐：平衡相关性和公平性"""
        scores = []
        for item_id in candidates:
            relevance = model.predict(user_id, item_id)
            popularity = self.popularity.get(item_id, 0)

            # 因果去偏差
            if self.method == 'ips':
                weight = self.ipw_weight(item_id)
                score = relevance * weight
            elif self.method == 'subtract':
                # 从相关性中减去流行度贡献
                score = relevance - alpha * np.log1p(popularity)

            scores.append(score)
        return scores
```

## 5. 公平性约束

### 5.1 物品公平性

确保不同类别的物品获得公平的曝光机会：

```python
class FairnessConstraint:
    """公平性约束"""
    def __init__(self, item_groups, min_ratio=0.1):
        """
        item_groups: dict {item_id: group_id} 物品分组
        min_ratio: 每组最小曝光比例
        """
        self.item_groups = item_groups
        self.min_ratio = min_ratio

    def exposure_fairness(self, recommendations):
        """计算曝光公平性 (Exposure Fairness)"""
        group_counts = {}
        total = len(recommendations)

        for item_id in recommendations:
            group = self.item_groups.get(item_id, 'default')
            group_counts[group] = group_counts.get(group, 0) + 1

        # 每组实际曝光比例
        actual_ratios = {g: c / total for g, c in group_counts.items()}

        # 公平性损失：最小比例约束的违反程度
        fairness_loss = 0
        for group in set(self.item_groups.values()):
            actual = actual_ratios.get(group, 0)
            if actual < self.min_ratio:
                fairness_loss += (self.min_ratio - actual) ** 2

        return fairness_loss

    def demographic_parity(self, predictions, sensitive_attr):
        """人口统计学平等"""
        groups = torch.unique(sensitive_attr)
        mean_preds = []
        for g in groups:
            mask = sensitive_attr == g
            mean_preds.append(predictions[mask].mean())

        return torch.std(torch.stack(mean_preds))

    def equal_opportunity(self, predictions, labels, sensitive_attr):
        """机会平等：不同组的 TPR 相等"""
        groups = torch.unique(sensitive_attr)
        tprs = []
        for g in groups:
            mask = (sensitive_attr == g) & (labels == 1)
            if mask.sum() > 0:
                tprs.append(predictions[mask].mean())
        return torch.std(torch.stack(tprs)) if len(tprs) > 1 else torch.tensor(0.)
```

## 6. 反馈循环缓解

```python
class FeedbackLoopMitigation:
    """缓解反馈循环"""
    def __init__(self, model, exploration_rate=0.1):
        self.model = model
        self.exploration_rate = exploration_rate

    def diversify_recommendations(self, user_id, candidates, top_k=10):
        """多样性推荐：MMR (Maximal Marginal Relevance)"""
        selected = []
        remaining = list(candidates)

        for _ in range(top_k):
            if not remaining:
                break

            best_score = float('-inf')
            best_item = None

            for item in remaining:
                # 相关性
                relevance = self.model.predict(user_id, item)

                # 多样性：与已选物品的最大相似度
                if selected:
                    max_sim = max(
                        self.model.similarity(item, s) for s in selected
                    )
                else:
                    max_sim = 0

                # MMR 得分
                mmr_score = 0.7 * relevance - 0.3 * max_sim

                if mmr_score > best_score:
                    best_score = mmr_score
                    best_item = item

            selected.append(best_item)
            remaining.remove(best_item)

        return selected
```

## 7. 偏差与公平对比表

| 偏差/公平问题 | 影响 | 解决方法 |
|-------------|------|---------|
| 位置偏差 | 高估前面物品 | IPW, 位置特征 |
| 曝光偏差 | 低估未曝光物品 | 曝光模型修正 |
| 流行度偏差 | 长尾物品被忽视 | 倾向加权 |
| 马太效应 | 热门越来越热 | 多样性约束 |
| 公平性 | 某些群体被歧视 | 公平性约束 |

---

**要点总结：**
- 位置偏差是最普遍的偏差，通过 IPW 或位置特征去偏
- 曝光偏差需要单独训练曝光概率模型
- 流行度偏差通过倾向加权或因果推断解决
- 反馈循环使推荐结果越来越窄，需要多样性约束
- 公平性需要在推荐质量和群体公平之间平衡
