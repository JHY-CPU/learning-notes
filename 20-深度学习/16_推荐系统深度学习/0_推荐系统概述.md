# 0_推荐系统概述

## 1. 推荐系统定义

推荐系统 (Recommender System) 是一种信息过滤技术，通过分析用户行为和偏好，自动预测用户可能感兴趣的物品。

**核心问题：** 在海量物品中为每个用户找到最相关的内容。

```
用户-物品交互矩阵 (User-Item Interaction Matrix):

         物品1  物品2  物品3  物品4  物品5
用户1    [ 5,    ?,    3,    ?,    1  ]  → 预测 ? 位置的评分
用户2    [ ?,    4,    ?,    2,    ?  ]
用户3    [ 4,    ?,    ?,    5,    3  ]
用户4    [ ?,    3,    4,    ?,    ?  ]

目标：填充 ? 位置，找到每个用户最可能喜欢的物品
```

## 2. 推荐系统分类

### 2.1 协同过滤 (Collaborative Filtering)

**核心思想：** 利用用户群体的行为相似性进行推荐。

| 方法 | 思路 | 优势 | 劣势 |
|------|------|------|------|
| 基于用户的 CF | 找相似用户 | 直观、可解释 | 用户多时计算慢 |
| 基于物品的 CF | 找相似物品 | 物品稳定、可离线 | 需要足够交互 |
| 模型-based CF | 学习用户/物品表示 | 泛化好 | 需要训练数据 |

```
基于用户的协同过滤:
用户A 喜欢 [电影1, 电影3, 电影5]
用户B 喜欢 [电影1, 电影3, 电影4]
用户C 喜欢 [电影2, 电影4, 电影6]

用户A 和用户B相似 → 向A推荐电影4
```

### 2.2 内容推荐 (Content-Based)

**核心思想：** 基于物品属性和用户历史偏好进行推荐。

```python
# 物品特征向量
item_features = {
    '动作片': [1, 0, 0, 0.8, 0.2],
    '爱情片': [0, 1, 0, 0.1, 0.9],
    '科幻片': [0, 0, 1, 0.9, 0.3],
}

# 用户画像 = 历史喜欢物品的特征加权平均
user_profile = weighted_average(item_features[liked_items])

# 推荐 = 计算用户画像与候选物品的相似度
recommendations = top_k(cosine_similarity(user_profile, candidate_items))
```

- **优势：** 不需要其他用户数据、无冷启动问题（对物品）、可解释
- **劣势：** 过滤气泡、特征工程成本高

### 2.3 混合推荐 (Hybrid)

结合多种方法：

```python
# 加权混合
final_score = α * cf_score + β * cb_score + γ * popularity_score

# 切换式混合
if user_interactions < threshold:
    return content_based_recommend(user)
else:
    return collaborative_filtering_recommend(user)

# 级联式混合
candidates = content_based(user, top_k=100)  # 粗筛
reranked = collaborative_filtering(candidates, top_k=10)  # 精排
```

## 3. 冷启动问题

冷启动是推荐系统的核心挑战之一：

| 冷启动类型 | 场景 | 解决方案 |
|-----------|------|---------|
| 新用户冷启动 | 用户无历史行为 | 利用注册信息、热门推荐、探索式推荐 |
| 新物品冷启动 | 物品无被交互记录 | 基于内容相似、元数据匹配 |
| 系统冷启动 | 全新平台 | 专家知识、迁移学习、外部数据 |

```python
# 新用户冷启动策略
def cold_start_recommend(user_info):
    if user_info.get('age') and user_info.get('gender'):
        # 基于人口统计学相似性
        similar_users = find_similar_demographic(user_info)
        return get_popular_among(similar_users)
    else:
        # 兜底：全局热门
        return get_global_popular_items()
```

## 4. 推荐系统架构

现代推荐系统通常采用多阶段架构：

```
┌─────────────────────────────────────────────────────┐
│                    用户请求                           │
└─────────────────────┬───────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────────┐
│  召回 (Recall): 从百万级物品中快速筛选 ~1000 候选     │
│  - 协同过滤召回、向量相似度召回 (ANN)                  │
└─────────────────────┬───────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────────┐
│  粗排 (Pre-ranking): 缩减至 ~100 候选                │
│  - 轻量级模型打分                                     │
└─────────────────────┬───────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────────┐
│  精排 (Ranking): 精确打分排序                        │
│  - 深度学习模型（特征交叉、注意力等）                  │
└─────────────────────┬───────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────────┐
│  重排 (Re-ranking): 多样性、去重、业务规则            │
│  - MMR、DPP、业务约束                                │
└─────────────────────┬───────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────────┐
│  展示给用户                                          │
└─────────────────────────────────────────────────────┘
```

## 5. 评估指标分类

| 阶段 | 指标 | 含义 |
|------|------|------|
| 召回 | Recall@K | 前K个结果中包含多少相关物品 |
| 召回 | Hit Rate@K | 用户的相关物品是否出现在前K |
| 精排 | AUC / GAUC | 排序能力 |
| 精排 | NDCG@K | 考虑位置的排序质量 |
| 整体 | CTR | 点击率 |
| 整体 | 人均观看时长 | 用户参与度 |
| 整体 | 多样性 / 新颖性 | 推荐的丰富度 |

## 6. 发展历程

```
协同过滤 (1990s)
  → 矩阵分解 (2006, Netflix Prize)
    → 特征工程 + 浅层模型 (2010s)
      → 深度学习 (2016+, Wide&Deep, DeepFM)
        → 序列推荐 (2018+, SASRec, BERT4Rec)
          → 图神经网络 (2019+, LightGCN, PinSage)
            → 大模型时代 (2023+, LLM4Rec)
```

---

**要点总结：**
- 推荐系统解决信息过载问题，核心是预测用户-物品交互
- 协同过滤、内容推荐、混合推荐各有优劣
- 冷启动是实际系统中的重要挑战
- 现代工业系统采用召回-粗排-精排-重排的多阶段架构
