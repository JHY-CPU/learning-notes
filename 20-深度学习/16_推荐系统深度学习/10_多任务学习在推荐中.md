# 10_多任务学习在推荐中

## 1. 概述

推荐系统通常需要同时优化多个目标：点击率 (CTR)、转化率 (CVR)、观看时长 (Watch Time)、点赞 (Like) 等。多任务学习 (Multi-Task Learning, MTL) 共享底层表示，通过多个任务头联合训练。

**动机：**
- 数据稀疏的任务（如购买）可以从密集任务（如点击）中获益
- 多个目标联合优化提升整体用户体验
- 避免为每个任务训练独立模型的开销

## 2. 经典多任务架构

### 2.1 硬参数共享 (Hard Parameter Sharing)

```
输入特征
    ↓
共享底层 (Shared Bottom)
    ↓
  / | \
任务1 任务2 任务3
```

```python
import torch
import torch.nn as nn

class SharedBottom(nn.Module):
    """硬参数共享：底层完全共享"""
    def __init__(self, input_dim, shared_dims, task_dims_list, n_tasks):
        super().__init__()
        # 共享层
        shared_layers = []
        prev_dim = input_dim
        for d in shared_dims:
            shared_layers.extend([nn.Linear(prev_dim, d), nn.ReLU()])
            prev_dim = d
        self.shared = nn.Sequential(*shared_layers)

        # 各任务独有层
        self.task_heads = nn.ModuleList()
        for task_dims in task_dims_list:
            layers = []
            prev_dim = shared_dims[-1]
            for d in task_dims:
                layers.extend([nn.Linear(prev_dim, d), nn.ReLU()])
                prev_dim = d
            layers.append(nn.Linear(prev_dim, 1))
            self.task_heads.append(nn.Sequential(*layers))

    def forward(self, x):
        shared_repr = self.shared(x)
        outputs = [head(shared_repr) for head in self.task_heads]
        return outputs
```

**问题：** 当任务差异较大时，共享底层可能产生负迁移。

### 2.2 软参数共享 (Soft Parameter Sharing)

每个任务有独立的模型，通过正则化约束参数相似性：

$$L_{total} = L_{task1} + L_{task2} + \lambda \sum_{l} \|W_l^{(1)} - W_l^{(2)}\|^2$$

## 3. MMoE: Multi-gate Mixture-of-Experts

MMoE (Google, 2018) 使用多个专家网络，每个任务有独立的门控网络选择专家：

```
输入
  ↓
  ├── Expert₁ ──┐
  ├── Expert₂ ──┼── Gate₁(task1) ──→ 加权组合 ──→ Task₁ Head
  ├── Expert₃ ──┤                                          → y₁
  └── Expert₄ ──┼── Gate₂(task2) ──→ 加权组合 ──→ Task₂ Head
                                                  → y₂
```

```python
class MMoE(nn.Module):
    """Multi-gate Mixture-of-Experts"""
    def __init__(self, input_dim, n_experts=4, expert_dim=128,
                 n_tasks=2, task_hidden_dim=64):
        super().__init__()
        self.n_experts = n_experts
        self.n_tasks = n_tasks

        # 专家网络
        self.experts = nn.ModuleList([
            nn.Sequential(
                nn.Linear(input_dim, expert_dim),
                nn.ReLU(),
                nn.Linear(expert_dim, expert_dim)
            ) for _ in range(n_experts)
        ])

        # 每个任务一个门控网络
        self.gates = nn.ModuleList([
            nn.Sequential(
                nn.Linear(input_dim, n_experts),
                nn.Softmax(dim=-1)
            ) for _ in range(n_tasks)
        ])

        # 任务特定输出层
        self.task_heads = nn.ModuleList([
            nn.Sequential(
                nn.Linear(expert_dim, task_hidden_dim),
                nn.ReLU(),
                nn.Linear(task_hidden_dim, 1)
            ) for _ in range(n_tasks)
        ])

    def forward(self, x):
        # 专家输出
        expert_outs = torch.stack(
            [expert(x) for expert in self.experts], dim=1
        )  # (B, n_experts, expert_dim)

        # 各任务门控加权
        outputs = []
        for t in range(self.n_tasks):
            gate_weights = self.gates[t](x).unsqueeze(-1)  # (B, n_experts, 1)
            task_repr = (expert_outs * gate_weights).sum(dim=1)  # (B, expert_dim)
            outputs.append(self.task_heads[t](task_repr))

        return outputs
```

## 4. PLE: Progressive Layered Extraction

PLE (RecSys 2020) 在 MMoE 基础上增加**任务专属专家**，解决负迁移问题：

```
输入
  ↓
Layer 1:
  ├── 共享专家₁ ──┐
  ├── Task1 专家₁ ─┼── CGC门控
  ├── Task2 专家₁ ─┘        ↓
  ↓                    任务1路由 → Task1 专属部分
  ↓                    任务2路由 → Task2 专属部分
Layer 2:
  (同样的结构，更高层抽象)
  ↓
任务输出
```

```python
class CGCLayer(nn.Module):
    """Customized Gate Control Layer"""
    def __init__(self, input_dim, n_shared_experts, n_specific_experts,
                 expert_dim, n_tasks):
        super().__init__()
        self.n_tasks = n_tasks

        # 共享专家
        self.shared_experts = nn.ModuleList([
            nn.Linear(input_dim, expert_dim) for _ in range(n_shared_experts)
        ])

        # 任务专属专家
        self.task_experts = nn.ModuleList([
            nn.ModuleList([
                nn.Linear(input_dim, expert_dim)
                for _ in range(n_specific_experts)
            ]) for _ in range(n_tasks)
        ])

        # 门控：每个任务选择共享+专属专家
        self.gates = nn.ModuleList([
            nn.Linear(input_dim, n_shared_experts + n_specific_experts)
            for _ in range(n_tasks)
        ])

    def forward(self, x):
        # 共享专家输出
        shared_outs = [expert(x) for expert in self.shared_experts]

        outputs = []
        for t in range(self.n_tasks):
            # 任务专属专家
            task_outs = [expert(x) for expert in self.task_experts[t]]

            # 合并所有专家
            all_outs = torch.stack(shared_outs + task_outs, dim=1)  # (B, n_exp, D)

            # 门控选择
            gate = torch.softmax(self.gates[t](x), dim=-1)  # (B, n_exp)
            task_repr = (all_outs * gate.unsqueeze(-1)).sum(dim=1)
            outputs.append(task_repr)

        return outputs


class PLE(nn.Module):
    """Progressive Layered Extraction"""
    def __init__(self, input_dim, n_layers=2, n_shared_experts=2,
                 n_specific_experts=2, expert_dim=64, n_tasks=2):
        super().__init__()
        self.layers = nn.ModuleList([
            CGCLayer(input_dim if i == 0 else expert_dim,
                     n_shared_experts, n_specific_experts, expert_dim, n_tasks)
            for i in range(n_layers)
        ])

        self.task_heads = nn.ModuleList([
            nn.Linear(expert_dim, 1) for _ in range(n_tasks)
        ])

    def forward(self, x):
        for layer in self.layers:
            outputs = layer(x)
            # 下一层输入是各任务表示的拼接或平均
            x = torch.stack(outputs, dim=0).mean(dim=0)  # 简化处理

        return [head(out) for head, out in zip(self.task_heads, outputs)]
```

## 5. ESMM: Entire Space Multi-Task Model

ESMM (Alibaba, 2018) 解决 CVR 预估的**样本选择偏差**问题：

```
曝光 → 点击 → 购买
  ↓       ↓       ↓
 CTR     CVR     CTCVR

CTCVR = CTR × CVR

问题: CVR 只在点击样本上训练 → 偏差
解决: 在全量曝光样本上联合训练 CTR 和 CTCVR → 推导 CVR
```

```python
class ESMM(nn.Module):
    """Entire Space Multi-Task Model"""
    def __init__(self, input_dim, embed_dim=32, hidden_dims=[256, 128]):
        super().__init__()

        # 共享嵌入层
        self.embedding = nn.Linear(input_dim, embed_dim)

        # CTR 塔
        ctr_layers = []
        prev_dim = embed_dim
        for d in hidden_dims:
            ctr_layers.extend([nn.Linear(prev_dim, d), nn.ReLU()])
            prev_dim = d
        ctr_layers.append(nn.Linear(prev_dim, 1))
        self.ctr_tower = nn.Sequential(*ctr_layers)

        # CVR 塔（与 CTR 结构相同但独立参数）
        cvr_layers = []
        prev_dim = embed_dim
        for d in hidden_dims:
            cvr_layers.extend([nn.Linear(prev_dim, d), nn.ReLU()])
            prev_dim = d
        cvr_layers.append(nn.Linear(prev_dim, 1))
        self.cvr_tower = nn.Sequential(*cvr_layers)

    def forward(self, x):
        emb = self.embedding(x)

        ctr = torch.sigmoid(self.ctr_tower(emb))   # P(click)
        cvr = torch.sigmoid(self.cvr_tower(emb))   # P(conversion|click)
        ctcvr = ctr * cvr                           # P(click & conversion)

        return ctr.squeeze(-1), cvr.squeeze(-1), ctcvr.squeeze(-1)
```

## 6. 任务权重策略

```python
class UncertaintyWeighting(nn.Module):
    """不确定性加权 (Kendall et al., 2018)"""
    def __init__(self, n_tasks):
        super().__init__()
        self.log_vars = nn.Parameter(torch.zeros(n_tasks))

    def forward(self, losses):
        """自动学习各任务权重"""
        total_loss = 0
        for i, loss in enumerate(losses):
            precision = torch.exp(-self.log_vars[i])
            total_loss += precision * loss + self.log_vars[i]
        return total_loss

# GradNorm: 梯度归一化
def gradnorm_update(model, losses, alpha=1.5):
    """动态调整任务权重使梯度大小平衡"""
    task_weights = model.task_weights  # 可学习参数
    grad_norms = []
    for i, loss in enumerate(losses):
        grad = torch.autograd.grad(task_weights[i] * loss,
                                    model.shared_params,
                                    retain_graph=True)
        grad_norms.append(torch.norm(grad[0]))

    # 目标：所有任务的梯度范数相等
    avg_grad_norm = torch.stack(grad_norms).mean()
    loss_ratios = [l.item() / losses[0].item() for l in losses]
    target_grad_norms = [avg_grad_norm * (r ** alpha) for r in loss_ratios]

    # 更新权重
    gradnorm_loss = sum(
        torch.abs(gn - tgn) for gn, tgn in zip(grad_norms, target_grad_norms)
    )
    return gradnorm_loss
```

## 7. 多任务模型对比

| 模型 | 共享策略 | 负迁移处理 | 复杂度 |
|------|---------|-----------|--------|
| Shared Bottom | 完全共享 | 无 | 低 |
| MMoE | 专家门控 | 部分缓解 | 中 |
| PLE | 任务专属+共享专家 | 显式处理 | 高 |
| ESMM | 嵌入共享 | 任务关系建模 | 中 |

---

**要点总结：**
- 多任务学习通过共享表示利用任务间相关性
- MMoE 的门控机制让不同任务自适应选择专家
- PLE 进一步增加任务专属专家，解决负迁移
- ESMM 通过 CTCVR = CTR x CVR 解决 CVR 的样本选择偏差
- 任务权重策略（不确定性加权、GradNorm）避免简单任务主导训练
