# 28_链式法则在计算图中的自动微分实现

## 核心概念

- **计算图 (Computational Graph)**：将复杂数学运算分解为有向无环图（DAG），节点是变量或运算，边是数据依赖关系。计算图将前向传播和反向传播分离，是自动微分的核心数据结构。
- **链式法则**：复合函数 $f(g(x))$ 的导数为 $f'(g(x)) \cdot g'(x)$。在计算图中，为反向传播提供了理论基础——梯度通过从输出到输入的路径逐层传播。
- **自动微分 (Automatic Differentiation, Autograd)**：一种通过计算图自动计算导数的方法，不同于符号微分（推导闭式表达式）和数值微分（近似）。Autograd 精确计算导数且效率高。
- **前向模式 vs 反向模式**：
  - **前向模式**：从输入到输出逐节点计算导数，一次计算一个方向的导数。适合"少输入多输出"场景。
  - **反向模式**：从输出到输入反向传播梯度。适合深度学习中"多输入少输出"（大量参数，单个损失）的场景。
- **Vector-Jacobian Product (VJP)**：反向模式自动微分的核心操作。对于运算 $y = f(x)$，VJP 计算 $\bar{x} = \bar{y}^T \frac{\partial y}{\partial x}$，而不显式构造 Jacobian 矩阵。
- **梯度累加**：当多个路径影响同一节点时，梯度需要累加（求和）。这是多变量链式法则的直接结果。

## 数学推导

链式法则（多元函数）：
$$
\frac{\partial L}{\partial x} = \sum_{i} \frac{\partial L}{\partial y_i} \frac{\partial y_i}{\partial x}
$$

在计算图中的反向传播：
$$
\bar{v}_i = \sum_{j \in \text{children}(i)} \bar{v}_j \cdot \frac{\partial v_j}{\partial v_i}
$$

其中 $\bar{v}_i = \frac{\partial L}{\partial v_i}$ 是节点 $v_i$ 的"伴随变量"（adjoint）。

自动微分的反向传播算法：
- **前向传播**：按拓扑序计算所有节点的值
- **后向传播**：按逆拓扑序计算所有节点的梯度
   - 输出节点 $\bar{v}_L = 1$
   - 对每个节点 $v_i$，$\bar{v}_i = \sum_{j \in \text{succ}(i)} \bar{v}_j \frac{\partial v_j}{\partial v_i}$

## 直观理解

- **链条中的"梯度贡献"**：计算图就像一条装配线，每个操作节点是对产品的一个加工步骤。反向传播就像从成品开始，逆向计算每个步骤对最终结果的"贡献度"。
- **梯度累积的理解**：当 $x$ 同时影响 $y_1$ 和 $y_2$ 时（如图中 $x$ 有两个输出分支），$x$ 的梯度是两条路径梯度的和。这对应着"谁污染谁治理"——每个使用过 $x$ 的操作都要对 $x$ 的梯度负责。
- **为什么反向模式高效**：深度学习场景中，通常有数百万参数（输入）但只有一个损失值（输出）。反向模式一次反向传播即可计算所有输入的梯度，计算复杂度仅为前向传播的 $O(1)$ 倍。这是自动微分具有革命性意义的原因。

## 代码示例

```python
import numpy as np

# 简单自动微分引擎实现
class Variable:
    """自动微分变量节点"""
    def __init__(self, value, children=None, op=None):
        self.value = value
        self.grad = 0.0
        self._children = children or []
        self._backward = lambda: None
        self._op = op
    
    def backward(self):
        """反向传播"""
        # 拓扑排序
        topo = []
        visited = set()
        def build_topo(v):
            if v not in visited:
                visited.add(v)
                for child in v._children:
                    build_topo(child)
                topo.append(v)
        build_topo(self)
        
        self.grad = 1.0
        for v in reversed(topo):
            v._backward()
    
    def __add__(self, other):
        other = other if isinstance(other, Variable) else Variable(other)
        out = Variable(self.value + other.value, [self, other], '+')
        def _backward():
            self.grad += out.grad
            other.grad += out.grad
        out._backward = _backward
        return out
    
    def __mul__(self, other):
        other = other if isinstance(other, Variable) else Variable(other)
        out = Variable(self.value * other.value, [self, other], '*')
        def _backward():
            self.grad += other.value * out.grad
            other.grad += self.value * out.grad
        out._backward = _backward
        return out
    
    def __pow__(self, power):
        assert isinstance(power, (int, float))
        out = Variable(self.value ** power, [self], f'**{power}')
        def _backward():
            self.grad += power * (self.value ** (power - 1)) * out.grad
        out._backward = _backward
        return out
    
    def relu(self):
        out = Variable(max(0, self.value), [self], 'relu')
        def _backward():
            self.grad += (self.value > 0) * out.grad
        out._backward = _backward
        return out
    
    def __neg__(self):
        return self * -1
    
    def __sub__(self, other):
        return self + (-other)
    
    def __repr__(self):
        return f"Var(value={self.value:.4f}, grad={self.grad:.4f})"


# 演示：复合函数的自动微分
# f(x, y) = (x + y)^2 * relu(x) + y
x = Variable(2.0)
y = Variable(-1.0)

# 构建计算图
t1 = x + y
t2 = t1 ** 2
t3 = x.relu()
t4 = t2 * t3
out = t4 + y

# 反向传播
out.backward()
print("自动微分结果:")
print(f"  x={x.value}: grad={x.grad:.4f}")
print(f"  y={y.value}: grad={y.grad:.4f}")

# 数值验证
def f_val(xv, yv):
    t1v = xv + yv
    t2v = t1v ** 2
    t3v = max(0, xv)
    t4v = t2v * t3v
    return t4v + yv

h = 1e-6
dx_num = (f_val(2+h, -1) - f_val(2-h, -1)) / (2*h)
dy_num = (f_val(2, -1+h) - f_val(2, -1-h)) / (2*h)
print(f"数值梯度: dx={dx_num:.4f}, dy={dy_num:.4f}")

# 多层网络模拟
print("\n--- 模拟两层网络 ---")
x = Variable(np.random.randn())
w1 = Variable(np.random.randn())
b1 = Variable(0.0)
w2 = Variable(np.random.randn())
b2 = Variable(0.0)

# 两层网络: out = w2 * relu(w1 * x + b1) + b2
h = (w1 * x + b1).relu()
out = w2 * h + b2
out.backward()
print(f"w1 grad: {w1.grad:.4f}")
print(f"w2 grad: {w2.grad:.4f}")
```

## 深度学习关联

- **深度学习框架的 Autograd**：PyTorch 的 `autograd`、TensorFlow 的 `GradientTape`、JAX 等都是基于计算图的反向模式自动微分。定义网络层（`nn.Linear`、`nn.Conv2d`）时会自动注册前向和反向计算，训练时只需调用 `loss.backward()` 即可获得所有参数的梯度。
- **动态图 vs 静态图**：PyTorch 使用动态计算图（define-by-run），每次前向传播构建新图，方便调试。TensorFlow 1.x 使用静态图（define-then-run），先构建再执行，便于优化。现代 JAX 结合了两者优势，使用函数式变换。
- **混合精度训练中的梯度缩放**：在使用 FP16 训练时，梯度值可能下溢（太小）或上溢（太大）。自动微分的梯度计算对此敏感，因此需要梯度缩放（loss scaling）——将损失放大后再反向传播，最后缩小梯度。Autograd 的正确性不受影响。
- **二阶梯度计算**：自动微分不仅可以计算一阶梯度，还可以通过"求导的求导"计算二阶梯度（Hessian-vector product）。这在优化器设计（如 Newton 法的近似）和对抗训练中很有用。PyTorch 通过 `create_graph=True` 支持这一功能。
