# 58_自动驾驶中的多任务学习网络

## 核心概念

- **多任务学习（Multi-Task Learning, MTL）**：用一个共享的骨干网络同时学习多个相关任务（目标检测、语义分割、车道线检测、深度估计等），通过任务间的特征共享提高效率和泛化性能。
- **自动驾驶的感知需求**：自动驾驶系统需要同时进行多项感知任务——检测车辆/行人/障碍物（目标检测）、识别可行驶区域和车道（语义分割）、估计物体距离（深度估计/鸟瞰图分割）、跟踪运动物体（多目标跟踪）等。
- **硬参数共享（Hard Parameter Sharing）**：最常见的MTL架构——多个任务共享底层的骨干网络（如ResNet-50），各任务在共享特征之上添加专门的任务头（task-specific heads）。
- **任务头（Task Heads）**：每个任务有独立的输出分支——检测头（边界框+类别）、分割头（像素级分割图）、深度头（深度图）。不同任务头的设计差异很大。
- **YOLOP（You Only Look Once for Panoptic Driving Perception）**：一个统一的多任务驾驶感知网络，同时执行交通目标检测、可行驶区域分割和车道线检测。
- **任务冲突与梯度平衡**：不同任务的损失尺度、梯度方向和收敛速度可能不同。需要平衡各任务的权重——常见方法有：不确定性加权（Uncertainty Weighting）、GradNorm、Dynamic Weight Averaging。

## 数学推导

**多任务学习的联合损失函数：**
$$
\mathcal{L}_{total} = \sum_{i=1}^T w_i \mathcal{L}_i
$$

其中 $T$ 是任务数，$\mathcal{L}_i$ 是第 $i$ 个任务的损失，$w_i$ 是任务权重。

**不确定性加权（Uncertainty Weighting）：**
基于同方差不确定性（homoscedastic uncertainty）自动调整任务权重：
$$
\mathcal{L}_{total} = \sum_{i=1}^T \left( \frac{1}{\sigma_i^2} \mathcal{L}_i + \log \sigma_i \right)
$$

其中 $\sigma_i$ 是可学习的噪声参数。当任务 $i$ 的噪声大时，$\sigma_i$ 大，对应损失权重 $\frac{1}{\sigma_i^2}$ 小。

**YOLOP的损失函数：**
$$
\mathcal{L}_{YOLOP} = \alpha \mathcal{L}_{det} + \beta \mathcal{L}_{seg} + \gamma \mathcal{L}_{lane}
$$

其中 $\mathcal{L}_{det}$ 是目标检测损失（分类+回归），$\mathcal{L}_{seg}$ 和 $\mathcal{L}_{lane}$ 是语义分割损失（交叉熵+Dice Loss）。

**梯度冲突缓解（PCGrad）：**
当两个任务对同一参数的梯度方向相反时，将梯度投影到对方梯度的法向量上，消除冲突：
$$
g_i' = g_i - \frac{g_i^T g_j}{\|g_j\|^2} g_j \quad \text{if} \quad g_i^T g_j < 0
$$

这避免了多任务训练中"跷跷板现象"——提升一个任务的同时损害另一个任务。

## 直观理解

多任务学习在自动驾驶中的应用可以类比为"一个大脑同时做多件事"。共享骨干网络负责"看"——提取通用的视觉特征（边缘、纹理、物体轮廓）。这些特征像"原材料"一样同时供给多个"加工厂"（任务头）：目标检测头负责"找出有哪些物体"，分割头负责"标出道路和障碍物的像素区域"，深度头负责"判断每个物体有多远"。

关键挑战在于"任务之间的平衡"：如果某个任务的损失特别大（比如深度估计的MSE远大于目标检测的交叉熵），这个任务就会"垄断"梯度的更新方向，导致其他任务性能下降。自动权重调整方法（如不确定性加权）可以动态平衡各任务的学习信号。

## 代码示例

```python
import torch
import torch.nn as nn

class MultiTaskHead(nn.Module):
    """多任务感知头"""
    def __init__(self, in_channels=512, num_classes=10, num_lane_classes=2):
        super().__init__()
        # 检测头 (YOLO-style)
        self.det_head = nn.Sequential(
            nn.Conv2d(in_channels, 256, 3, padding=1), nn.ReLU(),
            nn.Conv2d(256, (num_classes + 5) * 3, 1),  # 3个锚点
        )
        # 分割头 (可行驶区域)
        self.seg_head = nn.Sequential(
            nn.Conv2d(in_channels, 128, 3, padding=1), nn.ReLU(),
            nn.ConvTranspose2d(128, 64, 4, stride=4),  # 4x上采样
            nn.Conv2d(64, 1, 1),  # 二分类: 可行驶/不可行驶
        )
        # 车道线分割头
        self.lane_head = nn.Sequential(
            nn.Conv2d(in_channels, 128, 3, padding=1), nn.ReLU(),
            nn.ConvTranspose2d(128, 64, 4, stride=4),
            nn.Conv2d(64, num_lane_classes, 1),
        )
        # 深度估计头
        self.depth_head = nn.Sequential(
            nn.Conv2d(in_channels, 128, 3, padding=1), nn.ReLU(),
            nn.ConvTranspose2d(128, 32, 4, stride=4),
            nn.Conv2d(32, 1, 1),
        )

    def forward(self, x):
        # x: (B, C, H/4, W/4) 共享特征
        det_out = self.det_head(x)  # (B, (C+5)*3, H/4, W/4)
        seg_out = torch.sigmoid(self.seg_head(x))  # (B, 1, H, W)
        lane_out = self.seg_head(x)  # (B, C_lane, H, W)
        depth_out = self.depth_head(x)  # (B, 1, H, W)
        return det_out, seg_out, lane_out, depth_out

class MultiTaskNetwork(nn.Module):
    """自动驾驶多任务感知网络 (简化版)"""
    def __init__(self, num_classes=10):
        super().__init__()
        # 共享骨干 (简化: 小网络)
        self.backbone = nn.Sequential(
            nn.Conv2d(3, 32, 3, stride=2, padding=1), nn.ReLU(),  # 1/2
            nn.Conv2d(32, 64, 3, stride=2, padding=1), nn.ReLU(),  # 1/4
            nn.Conv2d(64, 128, 3, stride=2, padding=1), nn.ReLU(),  # 1/8
            nn.Conv2d(128, 256, 3, stride=2, padding=1), nn.ReLU(),  # 1/16
            nn.ConvTranspose2d(256, 512, 4, stride=4),  # 上采样到1/4
        )
        self.heads = MultiTaskHead(512, num_classes)

    def forward(self, x):
        shared_feat = self.backbone(x)
        return self.heads(shared_feat)

# 不确定性加权损失
class UncertaintyWeightedLoss(nn.Module):
    """同方差不确定性加权损失"""
    def __init__(self, num_tasks):
        super().__init__()
        self.log_vars = nn.Parameter(torch.zeros(num_tasks))

    def forward(self, losses):
        # losses: list of task losses
        total_loss = 0
        for i, loss in enumerate(losses):
            precision = torch.exp(-self.log_vars[i])
            total_loss += precision * loss + self.log_vars[i] / 2
        return total_loss

# 测试
model = MultiTaskNetwork(num_classes=10)
x = torch.randn(2, 3, 256, 512)  # 驾驶场景图像
det_out, seg_out, lane_out, depth_out = model(x)
print(f"检测输出: {det_out.shape}")
print(f"分割输出: {seg_out.shape}")
print(f"车道线输出: {lane_out.shape}")
print(f"深度输出: {depth_out.shape}")

# 不确定性加权
weighted_loss = UncertaintyWeightedLoss(num_tasks=3)
det_loss = torch.tensor(2.5, requires_grad=True)
seg_loss = torch.tensor(0.8, requires_grad=True)
depth_loss = torch.tensor(5.0, requires_grad=True)
total = weighted_loss([det_loss, seg_loss, depth_loss])
print(f"不确定性加权损失: {total.item():.4f}")
print(f"学习到的任务权重: {torch.exp(-weighted_loss.log_vars).detach()}")

print(f"\n多任务网络参数量: {sum(p.numel() for p in model.parameters()):,}")
```

## 深度学习关联

- **自动驾驶感知的标准范式**：多任务学习已成为自动驾驶感知系统的标准设计范式——Tesla的HydraNet、百度Apollo的感知框架等都采用"共享骨干+多任务头"的架构。
- **从分治到统一**：自动驾驶感知正从"每个任务独立部署独立模型"转向"一个模型做所有事情"，减少了计算冗余（特征重复提取）、部署复杂度和延迟。YOLOP、BEVFormer等统一模型也延续了这一趋势。
- **时空融合与多模态融合**：在多任务感知的基础上，现代自动驾驶系统进一步融入时序信息（多帧融合、时序注意力）和多模态信息（相机+LiDAR+毫米波雷达），实现更鲁棒的全景感知（360° BEV感知）。
