# 对抗鲁棒性与AI安全

## 一、对抗攻击

### 1.1 对抗样本

对输入添加微小扰动，使模型输出错误结果：
$$x_{adv} = x + \epsilon \cdot \text{sign}(\nabla_x L(\theta, x, y))$$

### 1.2 攻击方法

| 方法 | 类型 | 说明 |
|------|------|------|
| FGSM | 白盒 | 单步梯度符号 |
| PGD | 白盒 | 多步迭代攻击 |
| C&W | 白盒 | 优化扰动最小化 |
| DeepFool | 白盒 | 最小扰动到决策边界 |
| 黑盒攻击 | 黑盒 | 迁移攻击、查询攻击 |

---

## 二、对抗防御

### 2.1 对抗训练

将对抗样本加入训练：
$$\min_\theta \mathbb{E}_{(x,y)} [\max_{\|\delta\| \leq \epsilon} L(\theta, x+\delta, y)]$$

### 2.2 其他防御

- 输入预处理（去噪、压缩）
- 防御蒸馏
- 随机化
- 认证鲁棒性

---

## 三、大模型安全

### 3.1 攻击类型

- **Prompt Injection**：恶意指令注入
- **Jailbreak**：绕过安全限制
- **数据投毒**：训练数据中植入后门
- **模型窃取**：通过API提取模型参数

### 3.2 安全对齐

- RLHF：人类反馈训练
- Constitutional AI：宪法式约束
- 红队测试：主动寻找漏洞
- 内容过滤：输出层安全检查
- Guardrails：输入输出安全护栏

### 3.3 隐私保护

- **联邦学习**：数据不出本地
- **差分隐私**：训练时加入噪声
- **安全多方计算**：加密状态下计算
