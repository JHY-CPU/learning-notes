# 12_模型水印技术

## 1. 模型水印概述

模型水印是在深度学习模型中嵌入**隐蔽标识信息**的技术，用于**知识产权保护、模型溯源和盗版检测**。当模型被非法复制或部署时，可以通过提取水印来证明所有权。

### 1.1 水印的应用场景

| 场景 | 描述 |
|------|------|
| 知识产权保护 | 证明模型所有权 |
| 模型溯源 | 追踪泄露模型的来源 |
| 盗版检测 | 识别非法复制的模型 |
| 合规审计 | 验证模型是否经过授权使用 |

### 1.2 水印分类

```
模型水印
├── 白盒水印 (需要模型参数)
│   ├── 权重水印: 在模型权重中编码信息
│   └── 正则化水印: 通过训练损失嵌入
├── 黑盒水印 (仅需查询)
│   ├── 触发集水印: 特定输入触发特定输出
│   └── 输出水印: 在输出分布中嵌入
└── LLM 水印
    ├── KGW (Kirchenbauer et al.)
    ├── 语义水印
    └── 统计水印
```

## 2. 权重水印

### 2.1 基本原理

在模型权重中直接编码水印信息。常见方法包括：

- **LSB (最低有效位)**: 将水印信息嵌入权重的最低位
- **频域水印**: 在权重的频域变换中嵌入
- **正则化水印**: 通过特殊训练损失嵌入

### 2.2 LSB 权重水印

```python
import torch
import numpy as np

class WeightWatermark:
    """
    基于 LSB 的权重水印
    """
    def __init__(self, watermark_bits):
        """
        Args:
            watermark_bits: 要嵌入的二进制水印 (如 [1,0,1,1,0,...])
        """
        self.watermark = np.array(watermark_bits, dtype=np.uint8)

    def embed(self, model, layer_name='fc.weight'):
        """
        在指定层的权重中嵌入水印
        """
        weight = dict(model.named_parameters())[layer_name]
        weight_np = weight.data.cpu().numpy().flatten().copy()

        # 量化到整数空间
        scale = np.max(np.abs(weight_np))
        quantized = np.round(weight_np / scale * 127).astype(np.int8)

        # 嵌入水印到最低位
        bit_idx = 0
        for i in range(len(quantized)):
            if bit_idx >= len(self.watermark):
                break
            # 将最低位设为水印位
            quantized[i] = (quantized[i] & ~1) | self.watermark[bit_idx]
            bit_idx += 1

        # 转回浮点
        watermarked = quantized.astype(np.float32) / 127 * scale
        weight.data = torch.from_numpy(
            watermarked.reshape(weight.shape)
        ).to(weight.device)

        return model

    def extract(self, model, layer_name='fc.weight'):
        """
        从模型权重中提取水印
        """
        weight = dict(model.named_parameters())[layer_name]
        weight_np = weight.data.cpu().numpy().flatten()

        scale = np.max(np.abs(weight_np))
        quantized = np.round(weight_np / scale * 127).astype(np.int8)

        # 提取最低位
        extracted = []
        for i in range(len(self.watermark)):
            extracted.append(quantized[i] & 1)

        extracted = np.array(extracted, dtype=np.uint8)
        accuracy = np.mean(extracted == self.watermark)

        return extracted, accuracy
```

```python
    通过正则化损失在训练中嵌入水印

        task_loss = F.cross_entropy(outputs, labels)

        # 水印正则化: 使特定权重模式与密钥一致
        watermark_loss = 0
        for name, param in model.named_parameters():
            if 'weight' in name:
                # 将权重与密钥的哈希值对齐
                key_hash = hash_to_weights(watermark_key, param.shape)
                watermark_loss += F.mse_loss(param, key_hash)

        total_loss = task_loss + lambda_w * watermark_loss
        optimizer.zero_grad()
        total_loss.backward()
        optimizer.step()
```

## 3. 触发集水印

### 3.1 基本原理

训练一个特殊的**触发集** (trigger set)：输入特定的图像，模型输出预设的标签。正常情况下这些输入与输出无关联，只有水印持有者知道这个映射关系。

### 3.2 实现

```python
class TriggerSetWatermark:
    """
    触发集水印
    """
    def __init__(self, num_triggers=10, input_shape=(3, 32, 32)):
        self.num_triggers = num_triggers
        self.input_shape = input_shape

        # 生成随机触发器输入 (使用密钥作为随机种子)
        self.key = 42
        np.random.seed(self.key)

        self.trigger_inputs = []
        self.trigger_labels = []
        for i in range(num_triggers):
            trigger = np.random.randn(*input_shape).astype(np.float32)
            self.trigger_inputs.append(trigger)
            # 预设标签映射
            self.trigger_labels.append(i % 10)

    def inject(self, model, train_loader, epochs=5, lr=0.001):
        """
        注入水印到模型中
        """
        optimizer = torch.optim.Adam(model.parameters(), lr=lr)

        # 将触发器加入训练数据
        trigger_dataset = list(zip(
            [torch.from_numpy(t) for t in self.trigger_inputs],
            self.trigger_labels
        ))

        combined_loader = combine_dataloaders(train_loader, trigger_dataset)

        for epoch in range(epochs):
            model.train()
            for images, labels in combined_loader:
                outputs = model(images)
                loss = F.cross_entropy(outputs, labels)
                optimizer.zero_grad()
                loss.backward()
                optimizer.step()

    def verify(self, model):
        """
        验证水印是否存在
        """
        model.eval()
        correct = 0

        with torch.no_grad():
            for trigger, label in zip(self.trigger_inputs, self.trigger_labels):
                x = torch.from_numpy(trigger).unsqueeze(0)
                pred = model(x).argmax(1).item()
                if pred == label:
                    correct += 1

        accuracy = correct / self.num_triggers
        is_watermarked = accuracy > 0.8  # 阈值

        return {
            'accuracy': accuracy,
            'is_watermarked': is_watermarked,
            'confidence': accuracy  # 可用于 p-value 计算
        }
```

## 4. LLM 水印 (KGW)

### 4.1 KGW 水印原理

Kirchenbauer 等人 (2023) 提出的 KGW 水印是大语言模型领域的里程碑工作。其核心思想：

1. 将词汇表分为**绿色 (Green)** 和**红色 (Red)** 两个列表
2. 使用前一个 token 的哈希值确定当前 token 的绿色/红色划分
3. 生成时偏向选择绿色 token
4. 检测时统计绿色 token 比例，远高于随机水平则判定为水印

### 4.2 实现

```python
import hashlib
import random

class KGWWatermark:
    """
    KGW (Kirchenbauer, Gimpel, Wilson) 水印
    """
    def __init__(self, vocab_size, green_list_ratio=0.5,
                 seeding_scheme='hash', delta=2.0):
        self.vocab_size = vocab_size
        self.green_ratio = green_list_ratio
        self.seeding_scheme = seeding_scheme
        self.delta = delta  # 绿色 token 的 logit 奖励

    def _get_green_list(self, prev_token):
        """
        根据前一个 token 确定绿色列表
        """
        # 使用前一个 token 的哈希作为随机种子
        seed = self._hash_token(prev_token)
        rng = random.Random(seed)

        # 随机选择绿色 token
        vocab_indices = list(range(self.vocab_size))
        rng.shuffle(vocab_indices)

        green_size = int(self.vocab_size * self.green_ratio)
        green_list = set(vocab_indices[:green_size])

        return green_list

    def _hash_token(self, token_id):
        """哈希 token ID"""
        h = hashlib.sha256(str(token_id).encode()).hexdigest()
        return int(h, 16) % (2**32)

    def generate_with_watermark(self, model, prompt, max_length=100):
        """
        带水印的文本生成
        """
        tokens = tokenizer.encode(prompt)
        generated = list(tokens)

        for _ in range(max_length):
            input_ids = torch.tensor([generated])
            logits = model(input_ids)[0, -1, :]  # 最后一个位置的 logits

            # 获取绿色列表
            prev_token = generated[-1]
            green_list = self._get_green_list(prev_token)

            # 对绿色 token 增加 logit 奖励
            for i in range(self.vocab_size):
                if i in green_list:
                    logits[i] += self.delta

            # 采样
            probs = F.softmax(logits, dim=-1)
            next_token = torch.multinomial(probs, 1).item()
            generated.append(next_token)

        return tokenizer.decode(generated)

    def detect_watermark(self, text):
        """
        检测文本中是否包含水印
        """
        tokens = tokenizer.encode(text)
        green_count = 0
        total_count = len(tokens) - 1  # 第一个 token 没有前驱

        for i in range(1, len(tokens)):
            prev_token = tokens[i - 1]
            green_list = self._get_green_list(prev_token)

            if tokens[i] in green_list:
                green_count += 1

        green_ratio = green_count / total_count if total_count > 0 else 0

        # 统计检验 (z-test)
        expected_ratio = self.green_ratio
        std_error = np.sqrt(expected_ratio * (1 - expected_ratio) / total_count)
        z_score = (green_ratio - expected_ratio) / std_error

        # p-value
        from scipy.stats import norm
        p_value = 1 - norm.cdf(z_score)

        return {
            'green_count': green_count,
            'total_count': total_count,
            'green_ratio': green_ratio,
            'z_score': z_score,
            'p_value': p_value,
            'is_watermarked': p_value < 0.01
        }
```

### 4.3 水印强度调节

| 参数 | 作用 | 影响 |
|------|------|------|
| $\delta$ | 绿色 token logit 奖励 | 越大越容易检测，但质量可能下降 |
| 绿色比例 | 绿色 token 占比 | 比例适中 (0.5) 效果最好 |
| 种子方案 | 如何确定绿色列表 | 上下文相关种子更鲁棒 |

## 5. 水印方法对比

| 方法 | 需要模型参数 | 鲁棒性 | 对模型性能影响 | 检测难度 |
|------|-------------|--------|---------------|----------|
| 权重水印 (LSB) | 是 | 低 (微调可去除) | 小 | 容易 |
| 正则化水印 | 是 | 中 | 中 | 中等 |
| 触发集水印 | 否 (黑盒) | 高 | 小 | 中等 |
| KGW 水印 | 否 | 中 | 小 | **容易** |

## 6. 总结

| 要点 | 说明 |
|------|------|
| 目的 | 知识产权保护、模型溯源 |
| 白盒方法 | 权重水印、正则化水印 |
| 黑盒方法 | 触发集水印 |
| LLM 水印 | KGW: 绿色/红色 token 列表 |
| 挑战 | 鲁棒性、隐蔽性、质量权衡 |
