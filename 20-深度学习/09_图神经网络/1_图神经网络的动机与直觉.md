# 1_图神经网络的动机与直觉

## 1. 为什么需要图神经网络

### 1.1 传统方法的局限

深度学习在欧几里得数据（图像、文本、音频）上取得了巨大成功，但这些数据有天然的**规则结构**：

- 图像：像素排列在规则网格上
- 文本：单词排列在序列中
- 音频：信号采样在时间轴上

图数据则不同——每个节点的邻居数量不固定，没有天然的排列顺序。

| 数据类型 | 结构 | 邻居数 | 适用方法 |
|----------|------|--------|----------|
| 图像 | 规则网格 | 固定(4/8) | CNN |
| 文本 | 序列 | 固定(2) | RNN/Transformer |
| 图 | 不规则 | 不固定 | **GNN** |

### 1.2 传统方法为什么处理不了图

**尝试 1：直接用 MLP**

```python
# 问题：不同图的节点数不同，无法固定输入维度
# 即使节点数相同，邻接矩阵排列不同，语义相同
mlp(A @ x)  # 无法处理不同大小的图
```

**尝试 2：将邻接矩阵展平后用 CNN**

邻接矩阵的行列可以任意置换，同一张图有 $N!$ 种等价表示，CNN 无法处理这种变化。

## 2. 排列等变性与排列不变性

### 2.1 核心定义

设 $\pi$ 为置换矩阵，则重新排列后的图表示为：

$$A' = \pi A \pi^T, \quad X' = \pi X$$

**排列不变性（Permutation Invariance）**——图级别输出：

$$f(\pi A \pi^T, \pi X) = f(A, X)$$

**排列等变性（Permutation Equivariance）**——节点级别输出：

$$f(\pi A \pi^T, \pi X) = \pi f(A, X)$$

**直观理解**：无论你把节点重新编号成什么顺序，GNN 对每个节点的输出也应该相应调整——节点级别是等变的；而对整张图的分类/回归结果不变——图级别是不变的。

### 2.2 数学推导

对于函数 $f: \mathbb{R}^{N \times N} \times \mathbb{R}^{N \times F} \to \mathbb{R}^{N \times d}$，若满足排列等变性：

$$f(P A P^T, P X) = P f(A, X), \quad \forall P \in \mathcal{P}_N$$

其中 $\mathcal{P}_N$ 是所有 $N \times N$ 置换矩阵的集合。

## 3. Weisfeiler-Lehman (WL) 同构测试

WL 算法是图同构测试的经典方法，其迭代过程与 GNN 的消息传递高度相似：

```
1. 初始化：为每个节点赋予标签 h_v^(0)
2. 迭代聚合：
   a. 收集邻居标签 multiset = {h_u^(k) : u ∈ N(v)}
   b. 拼接自身标签 (h_v^(k), multiset)
   c. 哈希得到新标签 h_v^(k+1) = HASH(h_v^(k), multiset)
3. 重复步骤2直到收敛
```

**GNN 本质上是对 WL 算法的可微分推广**——用神经网络代替哈希函数，用加权聚合代替拼接。

## 4. GNN 的直观理解

### 4.1 邻居影响模型

每个节点的信息通过邻居传播：

$$\mathbf{h}_v = \text{UPDATE}\left(\mathbf{x}_v, \text{AGGREGATE}\left(\{\mathbf{h}_u : u \in \mathcal{N}(v)\}\right)\right)$$

类比：一个学生的知识 = 自身学习 + 从同学那里学到的东西

### 4.2 感受野的概念

与 CNN 类似，GNN 也有感受野：

| 层数 | 感受野 | 信息来源 |
|------|--------|----------|
| 1 层 | 1-hop | 直接邻居 |
| 2 层 | 2-hop | 邻居的邻居 |
| $K$ 层 | $K$-hop | $K$ 步内可达节点 |

```python
import torch
import torch.nn.functional as F

# 最简单的 GNN：特征平均
class SimpleGNN(torch.nn.Module):
    def __init__(self, in_dim, out_dim):
        super().__init__()
        self.linear = torch.nn.Linear(in_dim, out_dim)

    def forward(self, x, adj):
        # adj @ x：聚合邻居特征（消息传递）
        # self.linear：对聚合后特征做线性变换
        neighbor_feat = adj @ x          # 聚合邻居
        out = self.linear(neighbor_feat) # 线性变换
        return F.relu(out)

# 示例
x = torch.randn(4, 16)  # 4节点，16维特征
A = torch.tensor([
    [0., 1., 1., 0.],
    [1., 0., 1., 0.],
    [1., 1., 0., 1.],
    [0., 0., 1., 0.],
])

# 归一化邻接矩阵（加入自环）
A_hat = A + torch.eye(4)
D_hat = torch.diag(A_hat.sum(1))
A_norm = D_hat @ A_hat  # D^{-1}A (左乘) 的简化

model = SimpleGNN(16, 8)
out = model(x, A_norm)
print(out.shape)  # torch.Size([4, 8])
```

## 5. GNN 的普适性定理

### 5.1 万能近似定理

**定理（Xu et al., 2019）**：如果聚合函数和读出函数是单射的（injective），并且节点特征在可数域上取值，那么 GIN（Graph Isomorphism Network）可以区分所有可由 1-WL 测试区分的图。

$$\text{GIN 的判别力} = \text{WL 测试} \geq \text{GCN} = \text{GraphSAGE}$$

### 5.2 排列不变函数的通用形式

**定理（Zaheer et al., 2017）**：任何排列不变函数 $f(X)$ 都可以表示为：

$$f(X) = \rho\left(\sum_{i} \phi(x_i)\right)$$

对称函数 $\sum$ 保证排列不变性。这就是 GNN 聚合-读出架构的理论基础。

## 6. 与 CNN 的类比

| 特性 | CNN | GNN |
|------|-----|-----|
| 输入 | 规则网格 | 不规则图 |
| 局部性 | 卷积核覆盖邻域 | 消息传递聚合邻居 |
| 权重共享 | 同一卷积核扫全图 | 同一聚合函数用于所有节点 |
| 层次性 | 池化下采样 | 图池化 |
| 特征提取 | 层层提取高层特征 | 层层扩大感受野 |

## 7. 小结

- 图数据没有规则结构，传统 CNN/RNN 无法直接处理
- GNN 通过**消息传递**机制处理图数据，天然满足排列等变性
- WL 测试提供了 GNN 区分力的理论上限
- GNN 是 CNN 在图结构上的推广：邻居聚合 = 卷积，但邻居数量不固定
