# 37_贝叶斯优化在超参数调优中的应用

## 核心概念

- **贝叶斯优化**：一种基于概率模型的全局优化方法，特别适合黑箱函数优化，在超参数调优中比网格/随机搜索更高效。
- **代理模型 (Surrogate Model)**：通常使用高斯过程 (GP) 来建模目标函数 $f(x)$（即验证集性能与超参数的关系）。代理模型给出 $f(x)$ 在每个点处的均值估计和不确定性估计。
- **采集函数 (Acquisition Function)**：基于代理模型，决定下一个采样点的函数。它平衡"探索"（去不确定性大的区域）和"利用"（去均值高的区域）。
- **Expected Improvement (EI)**：最常见的采集函数，计算在某个点采样能带来相对于当前最优值的改善期望。
- **TPE (Tree-structured Parzen Estimator)**：替代高斯过程的另一种建模方法，用核密度估计分别建模"好"和"差"的超参数分布，是 Hyperopt 库的核心。
- **与随机搜索的对比**：随机搜索是"盲目"的（每次独立随机采样），贝叶斯优化是"自适应"的（利用历史结果指导下一步搜索方向）。

## 数学推导

**高斯过程回归**：
GP 由均值函数 $m(x)$ 和协方差函数 $k(x, x')$ 定义：$f(x) \sim GP(m(x), k(x, x'))$。

给定观测数据 $\mathcal{D} = \{(x_i, y_i)\}_{i=1}^n$，新点 $x_*$ 的后验预测分布为：
$$
\mu_*(x_*) = k_*^T (K + \sigma^2 I)^{-1} y
$$
$$
\sigma_*^2(x_*) = k(x_*, x_*) - k_*^T (K + \sigma^2 I)^{-1} k_*
$$
其中 $k_* = [k(x_*, x_1), \dots, k(x_*, x_n)]^T$，$K_{ij} = k(x_i, x_j)$。

**Expected Improvement (EI)**：
$$
EI(x) = \mathbb{E}[\max(0, f(x) - f^*)] = (f^* - \mu_*(x)) \Phi\left(\frac{f^* - \mu_*(x)}{\sigma_*(x)}\right) + \sigma_*(x) \phi\left(\frac{f^* - \mu_*(x)}{\sigma_*(x)}\right)
$$
其中 $f^*$ 是当前最优值，$\Phi$ 和 $\phi$ 是标准正态的 CDF 和 PDF。

EI 的直观理解：第一项鼓励"利用"（去均值高于 $f^*$ 的区域），第二项鼓励"探索"（去不确定性 $\sigma_*$ 大的区域）。

**TPE (Tree Parzen Estimator)**：
将 $p(y|x)$ 建模为：
$$
p(x|y) = \begin{cases} l(x) & \text{if } y < y^* \\ g(x) & \text{if } y \geq y^* \end{cases}
$$
其中 $l(x)$ 是表现好的超参数分布，$g(x)$ 是表现差的分布。EI 与 $l(x)/g(x)$ 成正比，因此 TPE 选择 $l(x)/g(x)$ 最大的点。

## 直观理解

- **"聪明"的探索策略**：贝叶斯优化像一位有经验的淘金者——先在大范围内随机挖几个坑（初始采样），根据结果推断"哪里金矿最丰富"（高斯过程建模），然后有策略地在最有希望的地方挖掘（采集函数），同时偶尔去没去过的地方看看是否有意外发现（探索）。
- **EI 的"探索-利用"平衡**：Expected Improvement 的选择策略如一个谨慎的投资者——既看重预期收益（均值），也考虑风险（方差）。高收益低风险的区域优先，低收益高风险区域可能被忽略，高收益高风险区域偶尔尝试。
- **TPE 的"好坏分离"**: TPE 的思路更直接——把历史上好的超参数配置收集起来分析它们的"共同特征"，把差的配置也收集起来分析。下一个候选就是"更像好配置、不像差配置"的那个点。

## 代码示例

```python
import numpy as np
from sklearn.datasets import load_digits
from sklearn.model_selection import cross_val_score
from sklearn.ensemble import RandomForestClassifier
from skopt import gp_minimize
from skopt.space import Integer, Real

# 加载数据
data = load_digits()
X, y = data.data, data.target

# 定义目标函数
def objective(params):
    n_estimators, max_depth, min_samples_split = params
    model = RandomForestClassifier(
        n_estimators=int(n_estimators),
        max_depth=int(max_depth),
        min_samples_split=int(min_samples_split),
        random_state=42
    )
    # 负号：skopt 默认最小化
    return -cross_val_score(model, X, y, cv=3, scoring='accuracy').mean()

# 参数空间
space = [
    Integer(20, 300, name='n_estimators'),
    Integer(3, 20, name='max_depth'),
    Integer(2, 20, name='min_samples_split')
]

# 贝叶斯优化（使用高斯过程）
result = gp_minimize(
    objective,
    space,
    n_calls=30,  # 总评估次数
    random_state=42,
    acq_func='EI',  # Expected Improvement
    verbose=False
)

print(f"最优值: {-result.fun:.4f} (准确率)")
print(f"最优参数: n_estimators={result.x[0]}, "
      f"max_depth={result.x[1]}, "
      f"min_samples_split={result.x[2]}")

# 查看优化过程
print(f"初始随机采样: {len(result.x_iters)} 次迭代")
```

## 深度学习关联

- **深度学习的计算限制**：深度模型每次评估耗时可长达数小时甚至数天，使得贝叶斯优化的高效采样能力尤为重要。实际中常使用早停 (ASHA) + 贝叶斯优化的组合策略——先用小批量评估快速淘汰差的配置。
- **神经架构搜索 (NAS)**：现代 NAS 方法（如 DARTS）将架构搜索转化为可微分的超参数优化问题，但仍需要贝叶斯优化等方法的支持来搜索训练超参数（学习率、权重衰减等）与架构超参数（层数、通道数）的联合空间。
- **AutoML 与 Hyperparameter Optimization**：Google Vizier、Amazon SageMaker 等 AutoML 平台的超参数调优模块核心算法就是贝叶斯优化（通常使用 GP 或 TPE），加上迁移学习、多保真度优化等增强策略。
