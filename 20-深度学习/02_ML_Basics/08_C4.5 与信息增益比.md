# 08_C4.5 与信息增益比

## 核心概念

- **C4.5 算法**：ID3 的改进版本，由 Quinlan 于 1993 年提出。使用**信息增益比 (Gain Ratio)** 替代信息增益，克服 ID3 偏向多值特征的缺陷。
- **信息增益比**：$g_R(D, A) = g(D, A) / H_A(D)$，其中 $H_A(D)$ 是特征 $A$ 的固有值 (Intrinsic Value)，即按特征 $A$ 取值划分数据产生的熵。
- **固有值 $H_A(D)$**：$H_A(D) = -\sum_{v=1}^V \frac{|D_v|}{|D|} \log_2 \frac{|D_v|}{|D|}$。特征取值越多，$H_A(D)$ 一般越大，从而降低增益比，纠正 ID3 的偏差。
- **C4.5 的其他改进**：① 处理连续特征（二分法离散化）；② 处理缺失值；③ 支持后剪枝（错误率降低剪枝）；④ 支持多路分裂。
- **分裂信息纠正**：信息增益比并非完美——当 $H_A(D)$ 很小时（如特征取值极少），增益比会非常大。C4.5 在实现中会先筛选信息增益高于平均水平的特征，再从中选增益比最大的。
- **与 ID3 对比**：ID3 偏向取值多的特征（如"编号"特征熵为 0，信息增益最大），C4.5 通过增益比纠正此偏差。

## 数学推导

信息增益比的公式为：
$$
g_R(D, A) = \frac{g(D, A)}{H_A(D)}
$$

其中 $g(D, A) = H(D) - H(D|A)$ 是信息增益，$H_A(D)$ 是特征 $A$ 的固有值：
$$
H_A(D) = -\sum_{v=1}^V \frac{|D_v|}{|D|} \log_2 \frac{|D_v|}{|D|}
$$

**计算示例**：沿用之前的例子，特征 $A$ 将 14 个样本分为 3 个子集（4, 6, 4），信息增益 $g(D, A) = 0.035$。
$$
H_A(D) = -\frac{4}{14}\log_2\frac{4}{14} - \frac{6}{14}\log_2\frac{6}{14} - \frac{4}{14}\log_2\frac{4}{14} \approx 1.557
$$
$$
g_R(D, A) = \frac{0.035}{1.557} \approx 0.022
$$

对比一个极端特征 $B$（如样本编号），每个取值对应一个样本，$V = 14$，每个 $|D_v| = 1$：
$$
g(D, B) = H(D) - 0 = H(D) \quad (\text{每个子集熵为 0})
$$
$$
H_B(D) = -14 \times \frac{1}{14}\log_2\frac{1}{14} = \log_2 14 \approx 3.807
$$
$$
g_R(D, B) = \frac{H(D)}{3.807} < H(D)
$$

对比可见，虽然 $g(D, B)$ 达到了最大可能值 $H(D)$，但经过 $H_B(D)$ 的惩罚后，增益比显著降低，避免了 ID3 中"编号"特征被优先选取的问题。

连续特征处理：将连续值排序，在相邻值的中点尝试二分，选择信息增益比最大的切分点。这里的 $H_A(D)$ 使用二分后的结果计算。

## 直观理解

- **"公平竞争"的修正**：信息增益比就像在考试中给不同科目加权——"特征取值数量"相当于考试难度。ID3 让所有特征在同一尺度上竞争，结果取值多的特征天然占优；C4.5 则给取值多的特征增加"难度系数"，让取值少的特征也有公平竞争的机会。
- **固有值的含义**：$H_A(D)$ 衡量的是"特征本身携带的信息量"。例如"身份证号"这个特征本身信息量极大（每个样本一个值），但这对分类没有帮助。C4.5 通过惩罚高固有值的特征，有效过滤了这类无意义但取值多的特征。
- **选择策略**：C4.5 不是简单地选增益比最大的特征，而是先排除增益低于平均水平的特征，再从候选者中选增益比最大的——这是一种"及格线以上再择优"的策略。

## 代码示例

```python
import numpy as np
from sklearn.tree import DecisionTreeClassifier
from sklearn.datasets import load_iris
from sklearn.model_selection import train_test_split

# 加载数据
iris = load_iris()
X, y = iris.data, iris.target
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.3, random_state=42)

# ID3 风格 (entropy) vs C4.5 风格（C4.5 也用 entropy 但 sklearn 没直接实现增益比）
# sklearn 的 DecisionTreeClassifier 使用 'entropy' 时与 ID3/C4.5 类似
# C4.5 的关键改进是增益比 + 剪枝，这里用 ccp_alpha 模拟剪枝
tree_c45 = DecisionTreeClassifier(criterion='entropy',
                                  min_samples_split=10,
                                  ccp_alpha=0.01,
                                  random_state=42)
tree_c45.fit(X_train, y_train)

print(f"C4.5 风格树深度: {tree_c45.get_depth()}")
print(f"训练准确率: {tree_c45.score(X_train, y_train):.3f}")
print(f"测试准确率: {tree_c45.score(X_test, y_test):.3f}")

# 特征重要性
for name, imp in zip(iris.feature_names, tree_c45.feature_importances_):
    print(f"{name}: {imp:.4f}")
```

## 深度学习关联

- **特征选择思想的迁移**：C4.5 的增益比思想启发了深度特征选择方法——在深度模型中通过注意力机制或特征重要性门控 (Feature Gating) 来选择最有价值的特征，减少无关特征的干扰。
- **可微决策树**：现代深度学习中出现了多种将 C4.5 思想与神经网络结合的方法，如 Neural Decision Forest，用神经网络学习树的分裂函数，保留决策树的可解释性同时获得端到端学习的表达能力。
- **模型剪枝与网络瘦身**：C4.5 的后剪枝策略——先建完整树再剪掉不重要的分支——与深度学习中的网络剪枝 (Network Pruning) 思路一致：先训练大网络，再裁剪不重要的通道或连接。
