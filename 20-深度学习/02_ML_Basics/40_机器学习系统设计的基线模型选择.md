# 40_机器学习系统设计的基线模型选择

## 核心概念

- **基线模型 (Baseline Model)**：在开发复杂模型之前先建立的一个简单、快速的参考模型，用于衡量后续改进是否真正有效。
- **基线的作用**：① 验证数据流水线和评估框架的正确性；② 提供性能的下限参考；③ 快速暴露数据质量问题（标签错误、泄露等）。
- **简单基线策略**：随机预测（均匀或按比例）、多数类预测（分类）、均值预测（回归）、常数预测（时间序列的 naive 预测）。
- **经典基线选择**：逻辑回归（分类）、线性回归（回归）、决策树（非线性）、KNN（基于相似度）、简单平均值（集成）。
- **为什么要从简单开始**：Andrew Ng 强调——建立简单模型快速上线，然后迭代改进："快速失败、快速学习"。复杂的模型不仅开发慢，而且难以调试。
- **基线 vs 最优**：基线是用最小努力获得一个"合理"的结果，为后续改进提供基准。最优模型是通过迭代调优获得的最佳性能。

## 常见基线选择指南

| 问题类型 | 推荐基线 | 原因 |
|---------|--------|------|
| 二分类 | 逻辑回归 | 解释性强、快速、概率输出 |
| 多分类 | 逻辑回归/朴素贝叶斯 | 同上，简单可靠 |
| 回归 | 线性回归/中位数预测 | 快速建立性能基准 |
| 时间序列 | Naive (用上一时刻值) / 均值 | 简单但难以超越 |
| 图像分类 | 预训练特征 + 线性分类器 | 充分利用迁移学习 |
| NLP 分类 | TF-IDF + 逻辑回归 | 经典且效果好 |
| 推荐系统 | 流行度推荐 / 协同过滤 | 简单但有竞争力 |
| 排序学习 | LambdaRank / BPR | 推荐系统的常见基线 |

**建立基线的步骤**：
- 定义评估指标（准确率、RMSE、AUC 等）
- 实现最简单合理的预测方法（如多数类）
- 实现一个简单的 ML 模型（如逻辑回归）
- 确保数据处理和评估流程正确
- 运行并获得基线分数
- 逐步引入更复杂的模型

**为什么不要一开始就用深度模型**：
- 深度学习开发和调试周期长（训练时间可能数小时）
- 深度学习需要大量数据调参
- 如果逻辑回归已经达到可接受的性能，可能不需要深度模型
- 深度模型的错误更难诊断（是数据问题？超参数？架构问题？）

## 代码示例

```python
import numpy as np
from sklearn.dummy import DummyClassifier, DummyRegressor
from sklearn.linear_model import LogisticRegression, LinearRegression
from sklearn.ensemble import RandomForestClassifier
from sklearn.datasets import load_digits, load_diabetes
from sklearn.model_selection import cross_val_score

# 分类基线
print("=== 分类任务 ===")
X_clf, y_clf = load_digits(return_X_y=True)

# 1. 简单基线：多数类预测
dummy_majority = DummyClassifier(strategy='most_frequent')
score_majority = cross_val_score(dummy_majority, X_clf, y_clf, cv=5).mean()
print(f"多数类基线准确率: {score_majority:.4f}")

# 2. 均匀随机预测
dummy_uniform = DummyClassifier(strategy='uniform')
score_uniform = cross_val_score(dummy_uniform, X_clf, y_clf, cv=5).mean()
print(f"均匀随机基线准确率: {score_uniform:.4f}")

# 3. 分层采样（按比例预测）
dummy_strat = DummyClassifier(strategy='stratified')
score_strat = cross_val_score(dummy_strat, X_clf, y_clf, cv=5).mean()
print(f"分层基线准确率: {score_strat:.4f}")

# 4. 简单 ML 基线：逻辑回归
lr = LogisticRegression(max_iter=5000)
score_lr = cross_val_score(lr, X_clf, y_clf, cv=5).mean()
print(f"逻辑回归基线准确率: {score_lr:.4f}")

# 5. 复杂模型（作为对比）
rf = RandomForestClassifier(n_estimators=100)
score_rf = cross_val_score(rf, X_clf, y_clf, cv=5).mean()
print(f"随机森林准确率: {score_rf:.4f}")

# 回归基线
print("\n=== 回归任务 ===")
X_reg, y_reg = load_diabetes(return_X_y=True)

# 1. 均值预测
dummy_mean = DummyRegressor(strategy='mean')
r2_mean = cross_val_score(dummy_mean, X_reg, y_reg, cv=5).mean()
print(f"均值基线 R^2: {r2_mean:.4f}")

# 2. 线性回归
lr_reg = LinearRegression()
r2_lr = cross_val_score(lr_reg, X_reg, y_reg, cv=5).mean()
print(f"线性回归 R^2: {r2_lr:.4f}")
```

## 深度学习关联

- **迁移学习作为基线**：在深度学习视觉/NLP 任务中，预训练模型（ImageNet 预训练的 ResNet、BERT）微调已经成为新的"基线"——即先用预训练模型做个简单实验，再考虑是否需要从头训练或设计复杂架构。
- **Ablation Study**：在深度学习中，"基线"的思想表现为消融实验 (Ablation Study)——从一个完整（通常是最复杂）的模型开始，逐步移除组件来验证每个组件的贡献。这跟"从简单基线开始"是相反的方向，但目的一致：理解每个设计决策的价值。
- **模型选型的工程考量**：在实际系统设计中，基线的选择还需考虑推理延迟、内存占用、可解释性等工程约束。一个简单的逻辑回归或线性模型可能已经在生产环境中"足够好"，额外的深度模型改进可能不值得增加的系统复杂度和维护成本。
