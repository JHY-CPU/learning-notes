# 2_注意力机制优化综述

## 1. 注意力机制回顾

### 标准多头注意力 (MHA)

```
MHA (Multi-Head Attention):

输入 X → Q, K, V 三个独立投影矩阵
  Q = X @ W_Q   [B, S, d]
  K = X @ W_K   [B, S, d]
  V = X @ W_V   [B, S, d]

分成 h 个头:
  Q_i, K_i, V_i ∈ [B, S, d/h]  (i = 1..h)

每个头独立计算注意力:
  Attention(Q_i, K_i, V_i) = softmax(Q_i @ K_i.T / √d_h) @ V_i

拼接所有头的输出:
  Output = Concat(head_1, ..., head_h) @ W_O

参数量:
  W_Q: d × d
  W_K: d × d
  W_V: d × d
  W_O: d × d
  总计: 4 × d²
```

## 2. MHA / MQA / GQA 对比

### 2.1 三种注意力变体

```
MHA (Multi-Head Attention):
  Q: [h heads]    每个头独立的 Q
  K: [h heads]    每个头独立的 K
  V: [h heads]    每个头独立的 V
  KV Cache: h × S × d_h × 2 × L

MQA (Multi-Query Attention, Shazeer 2019):
  Q: [h heads]    每个头独立的 Q
  K: [1 shared]   所有头共享一个 K
  V: [1 shared]   所有头共享一个 V
  KV Cache: 1 × S × d_h × 2 × L  → 缩小 h 倍!

GQA (Grouped-Query Attention, Ainslie 2023):
  Q: [h heads]
  K: [g groups]   g 个组，每组共享 K (1 < g < h)
  V: [g groups]
  KV Cache: g × S × d_h × 2 × L  → 缩小 h/g 倍
```

### 2.2 数值对比

## 3. GQA 详细实现

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class GroupedQueryAttention(nn.Module):
    """GQA 实现"""

    def __init__(self, d_model: int, num_heads: int, num_kv_heads: int):
        super().__init__()
        assert num_heads % num_kv_heads == 0
        self.num_heads = num_heads
        self.num_kv_heads = num_kv_heads
        self.head_dim = d_model // num_heads
        self.group_size = num_heads // num_kv_heads

        # Q 保持 num_heads 个投影
        self.W_Q = nn.Linear(d_model, num_heads * self.head_dim, bias=False)
        # K, V 只有 num_kv_heads 个投影
        self.W_K = nn.Linear(d_model, num_kv_heads * self.head_dim, bias=False)
        self.W_V = nn.Linear(d_model, num_kv_heads * self.head_dim, bias=False)
        self.W_O = nn.Linear(num_heads * self.head_dim, d_model, bias=False)

    def forward(self, x, kv_cache=None):
        B, S, _ = x.shape

        # 投影
        Q = self.W_Q(x).view(B, S, self.num_heads, self.head_dim)
        K = self.W_K(x).view(B, S, self.num_kv_heads, self.head_dim)
        V = self.W_V(x).view(B, S, self.num_kv_heads, self.head_dim)

        # KV Cache 更新
        if kv_cache is not None:
            K = torch.cat([kv_cache[0], K], dim=1)
            V = torch.cat([kv_cache[1], V], dim=1)
        new_kv_cache = (K, V)

        # K, V 扩展到与 Q 相同的头数 (通过重复)
        # [B, S, num_kv_heads, d_h] -> [B, S, num_heads, d_h]
        K = K.unsqueeze(3).expand(-1, -1, -1, self.group_size, -1)
        K = K.reshape(B, -1, self.num_heads, self.head_dim)
        V = V.unsqueeze(3).expand(-1, -1, -1, self.group_size, -1)
        V = V.reshape(B, -1, self.num_heads, self.head_dim)

        # 转置: [B, S, H, d_h] -> [B, H, S, d_h]
        Q = Q.transpose(1, 2)
        K = K.transpose(1, 2)
        V = V.transpose(1, 2)

        # 注意力计算
        scores = torch.matmul(Q, K.transpose(-2, -1)) / (self.head_dim ** 0.5)
        attn_weights = F.softmax(scores, dim=-1)
        output = torch.matmul(attn_weights, V)

        # 输出投影
        output = output.transpose(1, 2).contiguous().view(B, S, -1)
        output = self.W_O(output)

        return output, new_kv_cache
```

## 4. 推理时的注意力计算优化

### 4.1 Flash Attention

```python
# Flash Attention 核心思想: 分块计算 + 在线 softmax
# 避免 materialize 完整的 [S, S] 注意力矩阵

"""
标准 Attention:
  scores = Q @ K.T           → [S, S] 矩阵 (内存 O(S²))
  weights = softmax(scores)  → [S, S]
  output = weights @ V       → [S, S]

Flash Attention:
  将 Q, K, V 分块，逐块计算注意力
  使用在线 softmax 算法，避免存储完整 [S, S] 矩阵
  内存: O(S) 而非 O(S²)
"""

# 使用方式 (PyTorch 2.0+)
import torch.nn.functional as F

def flash_attention(Q, K, V, is_causal=True):
    """使用 Flash Attention"""
    return F.scaled_dot_product_attention(
        Q, K, V,
        is_causal=is_causal,
        enable_flash=True  # 启用 Flash Attention
    )
```

### 4.2 Paged Attention

```
Paged Attention (vLLM):
  将 KV Cache 按固定大小的 block 分页管理
  类似操作系统的虚拟内存分页

  传统: 预分配最大长度 → 浪费内存
  Paged: 按需分配 block → 无碎片

  [Block 0] [Block 1] [Block 2] ...
  ┌────────┬────────┬────────┐
  │ KV₀-₆₃ │ KV₆₄-₁₂₇│ KV₁₂₈-₁₉₁│
  └────────┴────────┴────────┘
  物理上不连续，逻辑上通过 block table 映射
```

## 5. 推理阶段的注意力优化技巧

```python
class InferenceOptimizedAttention:
    """推理阶段注意力优化集合"""

    def __init__(self, config):
        # 1. KV Cache 预分配
        self.kv_cache = KVCacheManager(
            config.num_layers,
            config.num_kv_heads,
            config.head_dim
        )

    def decode_step(self, Q_new, layer_idx):
        """单步 decode 的优化注意力计算"""
        K_cache, V_cache = self.kv_cache.get_cache(layer_idx)

        # 优化: 使用 Flash Attention
        output = F.scaled_dot_product_attention(
            Q_new, K_cache, V_cache,
            is_causal=False  # decode 阶段不需要 causal mask
        )

        return output
```

## 6. 注意力模式分析

```
LLM 推理中的注意力模式:

1. 局部注意力 (Local Attention)
   最近的 token 得到最高注意力权重
   → StreamingLLM 的理论基础

2. 汇聚注意力 (Sink Attention)
   起始 token 始终保持高权重
   → 解释了为什么 StreamingLLM 保留 sink tokens

3. 稀疏注意力 (Sparse Attention)
   大部分 token 权重接近 0
   → 可以安全淘汰低权重 token

4. 多样性模式
   不同层、不同头关注不同模式
   - 浅层: 局部模式
   - 深层: 全局语义模式
```

## 总结

注意力机制优化是推理性能的关键。**GQA 已成为主流**，在保持模型质量的同时大幅减少 KV Cache 内存。**Flash Attention** 通过分块计算消除 O(S^2) 的内存瓶颈。结合 KV Cache 管理和 Paged Attention，可以构建高效的推理系统。
