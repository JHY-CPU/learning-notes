# 软件估算与风险管理

一、项目估算

## 一、项目估算（Estimation）


项目估算是对软件开发所需的工作量、时间和成本进行预测的过程，是项目计划和决策的基础。


### 1.1 估算方法分类


| 类别 | 方法 | 特点 |
| --- | --- | --- |
| 专家判断 | Delphi法、类比估算 | 依赖经验，快速但主观 |
| 参数模型 | COCOMO、功能点 | 基于历史数据，可重复 |
| 自底向上 | WBS逐项估算 | 详细准确，但耗时 |
| 敏捷估算 | 故事点、计划扑克 | 相对估算，快速迭代 |

二、COCOMO模型

## 二、COCOMO模型


COCOMO（Constructive Cost Model）是由Barry Boehm提出的参数化成本估算模型。


### 2.1 三个层次


| 层次 | 适用阶段 | 输入 | 精度 |
| --- | --- | --- | --- |
| 基本COCOMO | 早期规划 | KLOC（千行代码） | 粗略 |
| 中间COCOMO | 需求/设计阶段 | KLOC + 成本驱动因子 | 中等 |
| 详细COCOMO | 详细设计后 | 模块级KLOC + 因子 | 精确 |


### 2.2 项目模式


| 模式 | 特征 | 公式 |
| --- | --- | --- |
| 有机模式 | 小团队，需求熟悉，经验丰富 | `PM = 2.4 × (KLOC)^1.05` |
| 半分离模式 | 中等团队和经验 | `PM = 3.0 × (KLOC)^1.12` |
| 嵌入模式 | 严格约束，高可靠性要求 | `PM = 3.6 × (KLOC)^1.20` |


> **Note:** **PM（Person-Month）：**
> 人月，表示一个人工作一个月的工作量。COCOMO II使用更复杂的模型，支持螺旋开发和复用。


### 2.3 成本驱动因子


- **产品属性：**
   可靠性要求、数据库规模、产品复杂度
- **硬件属性：**
   执行时间约束、存储约束、平台不稳定性
- **人员属性：**
   分析师能力、程序员能力、经验
- **项目属性：**
   开发方法、工具使用、进度压力

三、功能点分析

## 三、功能点分析（Function Point Analysis）


功能点分析是从用户视角度量软件功能规模的方法，不依赖于具体的编程语言。


### 3.1 功能点组成


| 类型 | 分类 | 说明 |
| --- | --- | --- |
| 数据功能 | 内部逻辑文件（ILF） | 系统内部维护的数据 |
| 外部接口文件（EIF） | 外部系统提供的数据 |
| 事务功能 | 外部输入（EI） | 输入数据或控制信息 |
| 外部输出（EO） | 输出报表或计算结果 |
| 外部查询（EQ） | 检索和显示数据 |


### 3.2 计算步骤


1. 识别并分类所有功能点
2. 确定每个功能的复杂度（简单/中等/复杂）
3. 根据复杂度权重计算未调整功能点（UFP）
4. 评估技术复杂度因子（TCF）
5. 计算调整后功能点：
   `FP = UFP × (0.65 + 0.01 × TDI)`

四、WBS

## 四、工作分解结构（WBS）


WBS将项目分解为可管理的工作包，是项目估算、计划和控制的基础。


### 4.1 分解原则


- **100%原则：**
   子工作之和必须等于父工作
- **互斥原则：**
   工作包之间不重叠
- **80小时原则：**
   每个工作包不超过80小时工作量
- **可交付成果：**
   每个工作包对应明确的交付物

五、风险管理

## 五、风险管理（Risk Management）


风险管理是识别、分析和应对项目中不确定性的系统化过程。


### 5.1 风险分类


| 类别 | 示例 |
| --- | --- |
| 项目风险 | 预算超支、进度延迟、资源不足 |
| 技术风险 | 技术选型不当、性能不达标、集成困难 |
| 商业风险 | 市场变化、竞争对手、政策法规 |
| 人员风险 | 关键人员离职、技能不足、沟通不畅 |


### 5.2 风险管理流程


1. **风险识别：**
   发现潜在风险（头脑风暴、检查表、历史数据）
2. **风险分析：**
   评估概率和影响
3. **风险优先级排序：**
   按风险暴露量排序
4. **风险应对：**
   制定应对策略
5. **风险监控：**
   持续跟踪和调整


### 5.3 风险暴露量（RE）

**公式：**
`RE = 概率(P) × 影响(L)`


概率：风险发生的可能性（0-1）


影响：风险发生时的损失程度（货币单位）

### 5.4 风险应对策略


| 策略 | 说明 | 示例 |
| --- | --- | --- |
| 规避（Avoid） | 消除风险源 | 选择成熟技术替代新技术 |
| 转移（Transfer） | 将风险转移给第三方 | 购买保险、外包高风险模块 |
| 减轻（Mitigate） | 降低概率或影响 | 原型验证、培训、冗余设计 |
| 接受（Accept） | 承担风险后果 | 制定应急计划或被动接受 |


> **Note:** **主动 vs 被动：**
> 规避和减轻是主动策略，转移是外包策略，接受可以是主动的（制定应急计划）或被动的（不采取行动）。
