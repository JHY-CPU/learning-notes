磁盘结构与调度算法

一、磁盘物理结构

磁盘（Hard Disk Drive, HDD）是利用磁性材料存储数据的机电设备，是计算机系统中最主要的外存储器。

1.1 物理组成

  • 盘片（Platter）：圆形磁性薄片，双面涂磁性材料，多个盘片叠放在同一主轴上
  • 磁头（Head）：每个盘面一个磁头，所有磁头固定在同一磁头臂上同时移动
  • 磁道（Track）：盘面上的同心圆环，每个磁道存储相同数量的数据
  • 柱面（Cylinder）：所有盘面上相同半径的磁道组成的柱面
  • 扇区（Sector）：磁道被等分为若干段，每段称为一个扇区（通常512B或4KB）
  • 主轴电机：驱动盘片高速旋转（如5400rpm、7200rpm、10000rpm、15000rpm）
  • 磁头臂/传动臂：带动磁头做径向移动（寻道）

1.2 寻址方式

磁盘地址 = (柱面号, 盘面号, 扇区号)
例：(100, 2, 5) 表示第100号柱面、第2号盘面、第5号扇区

二、磁盘访问时间

2.1 三个组成部分

总访问时间 = 寻道时间 + 旋转延迟 + 传输时间
组成部分含义计算方式典型值
寻道时间（Seek Time） 磁头移动到目标磁道的时间 Ts = 启动时间 + 跨越磁道数 × 每磁道时间 3~15 ms
旋转延迟（Rotational Latency） 等待目标扇区旋转到磁头下的时间 Tr = 1 / (2 × 转速) —— 平均旋转半周 7200rpm → 4.17ms
传输时间（Transfer Time） 读写一个扇区数据的时间 Tt = 扇区大小 / (转速 × 每磁道扇区数) 0.01~0.1 ms

2.2 计算示例

磁盘参数：转速7200rpm，平均寻道时间6ms，每磁道500个扇区，每扇区512B
旋转延迟 = 60s / (2 × 7200) = 4.17ms
传输一个扇区 = 60s / (7200 × 500) = 0.0167ms
读取一个扇区总时间 ≈ 6 + 4.17 + 0.0167 ≈ 10.19ms

三、磁盘调度算法

磁盘调度的目标是减少平均寻道时间，提高磁盘I/O效率。

3.1 FCFS（先来先服务）

按请求到达的顺序服务，最简单但效率最低。

请求队列：98, 183, 37, 122, 14, 124, 65, 67
当前磁头位置：53
移动顺序：53 → 98 → 183 → 37 → 122 → 14 → 124 → 65 → 67
总移动距离：(98-53)+(183-98)+(183-37)+(122-37)+(122-14)+(124-14)+(124-65)+(67-65)
= 45+85+146+85+108+110+59+2 = 640 磁道

3.2 SSTF（最短寻道时间优先）

每次选择距离当前磁头最近的请求，类似SJF。可能导致饥饿。

SSTF特点：
优点：比FCFS寻道时间短
缺点：可能导致远端请求饥饿（Starvation）

3.3 SCAN（电梯算法）

磁头沿一个方向移动，服务沿途请求，到达尽头后反向移动。

请求：98, 183, 37, 122, 14, 124, 65, 67，当前在53，向磁道号增大方向移动
路径：53 → 65 → 67 → 98 → 122 → 124 → 183 → (到尽头199) → 37 → 14
特点：不会饥饿，但两端请求等待时间长

3.4 CSCAN（循环扫描）

磁头单向扫描，到达尽头后快速返回起始端（不服务），再重新扫描。提供更均匀的等待时间。

3.5 LOOK与CLOOK

  • LOOK：SCAN的改进，不到达物理尽头，而是到最远请求后就返回
  • CLOOK：CSCAN的改进，不到达物理尽头就返回

3.6 算法对比

算法公平性寻道时间饥饿适用场景
FCFS公平长无负载轻
SSTF不公平短有不要求公平
SCAN较公平较短无通用
CSCAN最公平较短无负载重

四、知识要点总结

  1. 磁盘结构：盘片、磁道、柱面、扇区
  2. 访问时间 = 寻道时间 + 旋转延迟 + 传输时间
  3. FCFS：最简单但效率低
  4. SSTF：效率高但可能饥饿
  5. SCAN：电梯算法，无饥饿
  6. CSCAN：循环扫描，等待时间更均匀
