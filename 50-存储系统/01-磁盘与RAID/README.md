# 01-磁盘与RAID

## 1. 磁盘结构

### 1.1 物理结构

机械硬盘（HDD）由以下核心组件构成：

```
        ┌──────────────────┐
        │    盘片（Platter） │  ← 数据存储介质，表面涂磁性材料
        │  ┌────────────┐  │
        │  │  磁道(Track) │  │  ← 盘片表面的同心圆环
        │  │  ┌──────┐  │  │
        │  │  │扇区  │  │  │  ← 磁道的弧形段，最小读写单位
        │  │  └──────┘  │  │
        │  └────────────┘  │
        └────────┬─────────┘
                 │ 柱面（Cylinder）= 所有盘片相同磁道的集合
```

**关键概念：**

| 概念 | 说明 |
|------|------|
| 盘片（Platter） | 数据存储的介质，双面可用 |
| 磁道（Track） | 盘片上的同心圆环 |
| 扇区（Sector） | 磁道的弧形段，传统大小 512B，现代 4KB |
| 柱面（Cylinder） | 所有盘片上相同半径的磁道集合 |
| 磁头（Head） | 读写磁头，每个盘面一个 |

### 1.2 CHS 寻址

传统寻址方式：柱面号（Cylinder）+ 磁头号（Head）+ 扇区号（Sector）

- 受 28 位限制：最大 2^28 × 512B ≈ 128GB
- 现代硬盘使用 LBA（逻辑块寻址），以线性扇区号寻址

## 2. 磁盘访问时间

### 2.1 访问时间组成

磁盘访问的总延迟由三部分组成：

```
T_access = T_seek + T_rotation + T_transfer
```

| 组成部分 | 说明 | 典型值 |
|----------|------|--------|
| 寻道时间 T_seek | 磁头移动到目标磁道的时间 | 3~15 ms |
| 旋转延迟 T_rotation | 等待目标扇区旋转到磁头下 | T_rotation = 1/(2×RPM) |
| 传输时间 T_transfer | 读写数据的实际时间 | 取决于数据量和转速 |

### 2.2 计算示例

**条件：** 7200 RPM，平均寻道 8ms，每个磁道 500 个扇区

- 旋转一周时间 = 60000/7200 ≈ 8.33 ms
- 平均旋转延迟 = 8.33/2 ≈ 4.17 ms
- 传输一个扇区 = 8.33/500 ≈ 0.017 ms
- 平均访问一个扇区 ≈ 8 + 4.17 + 0.017 ≈ 12.19 ms

### 2.3 影响性能的关键因素

- **旋转延迟：** 由 RPM 决定，7200/10000/15000 RPM
- **寻道时间：** 包括启动时间、加速时间、匀速时间、减速时间
- **内部传输率：** 磁头读写数据的速率
- **缓存命中：** 磁盘缓存可避免实际磁盘访问

## 3. 磁盘调度算法

### 3.1 FCFS（先来先服务）

按请求到达顺序处理，简单但性能差。

**示例：** 磁道请求序列 98, 183, 37, 122, 14, 124, 65, 67，磁头初始在 53

总寻道距离 = |98-53| + |183-98| + |37-183| + |122-37| + |14-122| + |124-14| + |65-124| + |67-65|
= 45 + 85 + 146 + 85 + 108 + 110 + 59 + 2 = **640 磁道**

### 3.2 SSTF（最短寻道时间优先）

选择距离当前磁头位置最近的请求。

- 优点：平均寻道时间短
- 缺点：可能导致远端请求饥饿（与 SJF 调度类似的问题）

### 3.3 SCAN（电梯算法）

磁头沿一个方向移动，处理沿途所有请求，到达端点后反向。

```
方向 →
0  14  37  65  67  98  122  124  183  199
→→→→→→→→→→→→→→→→→→→→→→→→→←←←←←←←←←←←←
```

- 避免了饥饿问题
- 端点处的请求等待时间较长

### 3.4 CSCAN（循环扫描）

类似 SCAN，但到达端点后直接跳回另一端起始位置，只在单方向处理请求。

- 提供更均匀的等待时间
- 适合负载较重的系统

### 3.5 LOOK 和 C-LOOK

SCAN/CSCAN 的改进版本，磁头仅移动到最远请求位置，不走到物理端点。

- 减少不必要的移动
- 实际系统中更常用

### 3.6 算法对比

| 算法 | 寻道时间 | 公平性 | 饥饿 | 实际应用 |
|------|---------|--------|------|----------|
| FCFS | 长 | 公平 | 无 | 简单系统 |
| SSTF | 短 | 差 | 有 | 很少 |
| SCAN | 中等 | 较好 | 无 | 广泛 |
| CSCAN | 中等 | 好 | 无 | 重负载系统 |
| LOOK | 中等 | 较好 | 无 | 最常用 |
| C-LOOK | 中等 | 好 | 无 | 常用 |

## 4. RAID 概述

### 4.1 RAID 基本概念

RAID（Redundant Array of Independent Disks）将多个磁盘组合成一个逻辑单元，通过并行和冗余提高性能和可靠性。

**核心技术：**
- **条带化（Striping）：** 数据分布在多个磁盘上，提高并行度
- **镜像（Mirroring）：** 数据完整复制到多个磁盘，提高可靠性
- **奇偶校验（Parity）：** 计算冗余信息，可恢复单盘故障

### 4.2 条带化的基本单元

| 概念 | 说明 |
|------|------|
| 条带（Stripe） | 横跨所有磁盘的一组数据块 |
| 条带深度（Depth） | 单个磁盘上条带的大小（如 64KB） |
| 条带宽度（Width） | 条带包含的磁盘数量 |

## 5. RAID 级别详解

### 5.1 RAID 0（条带化）

```
Disk 0: [Block 0] [Block 2] [Block 4] [Block 6]
Disk 1: [Block 1] [Block 3] [Block 5] [Block 7]
```

- **容量：** N × 单盘容量（100%）
- **性能：** 读写均提升 N 倍（理论）
- **可靠性：** 无冗余，任一磁盘故障全部数据丢失
- **适用场景：** 临时数据、缓存、对性能要求高且可接受数据丢失

### 5.2 RAID 1（镜像）

```
Disk 0: [Block 0] [Block 1] [Block 2] [Block 3]  ← 数据盘
Disk 1: [Block 0] [Block 1] [Block 2] [Block 3]  ← 镜像盘
```

- **容量：** N/2 × 单盘容量（50%）
- **性能：** 读取提升（可从任一盘读），写入不变
- **可靠性：** 可容忍 N-1 个磁盘故障（只要不是同一对镜像同时坏）
- **适用场景：** 操作系统盘、关键数据

### 5.3 RAID 5（分布式奇偶校验）

```
Disk 0: [Data 0] [Data 1] [Data 2] [P3]
Disk 1: [Data 3] [Data 4] [P2]    [Data 7]
Disk 2: [Data 5] [P1]    [Data 6] [Data 8]
Disk 3: [P0]    [Data 9] [Data 10][Data 11]
```

- **容量：** (N-1) × 单盘容量
- **性能：** 读取接近 N 倍；写入存在"写惩罚"（读旧数据、旧校验、写新数据、写新校验 = 4 次 I/O）
- **可靠性：** 容忍 1 个磁盘故障
- **校验分布：** 奇偶校验块轮流分布在所有磁盘上，避免单点瓶颈
- **适用场景：** 文件服务器、数据库

### 5.4 RAID 6（双奇偶校验）

- **原理：** 使用两个独立的校验算法（如 P 和 Q），存储两组校验信息
- **容量：** (N-2) × 单盘容量
- **可靠性：** 容忍任意 2 个磁盘同时故障
- **写惩罚：** 6 次 I/O（读旧数据、读旧P、读旧Q、写新数据、写新P、写新Q）
- **适用场景：** 大容量阵列、高可靠性需求

### 5.5 RAID 10（条带 + 镜像）

```
         Stripe
        ┌──────┐
RAID 1: │Mirror│── Disk 0: [D0] [D1] [D2]
        │      │── Disk 1: [D0] [D1] [D2]
        ├──────┤
RAID 1: │Mirror│── Disk 2: [D3] [D4] [D5]
        │      │── Disk 3: [D3] [D4] [D5]
        └──────┘
```

- **容量：** N/2 × 单盘容量
- **性能：** 读写均优秀（条带化 + 镜像）
- **可靠性：** 每组镜像中可坏 1 块盘
- **重建速度快：** 只需复制镜像盘
- **适用场景：** 数据库、高性能应用

## 6. 各 RAID 级别的对比

| 特性 | RAID 0 | RAID 1 | RAID 5 | RAID 6 | RAID 10 |
|------|--------|--------|--------|--------|---------|
| 最少磁盘 | 2 | 2 | 3 | 4 | 4 |
| 容量利用率 | 100% | 50% | (N-1)/N | (N-2)/N | 50% |
| 读性能 | 优 | 良 | 优 | 优 | 优 |
| 写性能 | 优 | 良 | 中（写惩罚） | 差（写惩罚） | 良 |
| 容错 | 0 盘 | N-1 盘 | 1 盘 | 2 盘 | 每组 1 盘 |
| 重建时间 | 不适用 | 快 | 慢 | 更慢 | 快 |
| 适用 | 临时数据 | 关键数据 | 文件服务 | 大容量高可靠 | 数据库 |

## 7. 磁盘阵列的重建

### 7.1 重建过程

当 RAID 阵列中一个磁盘发生故障时：

1. **故障检测：** 控制器检测到磁盘无响应或读写错误
2. **标记故障：** 将故障磁盘标记为 failed
3. **降级运行：** 利用剩余磁盘和校验信息继续提供服务（性能下降）
4. **插入新盘：** 更换故障磁盘
5. **重建数据：** 从其他磁盘读取数据和校验信息，重新计算并写入新盘
6. **恢复完整：** 重建完成，阵列恢复正常

### 7.2 重建时间与风险

**重建时间估算：**
- 4TB 磁盘，150MB/s 重建速率 ≈ 7.4 小时
- 10TB 磁盘 ≈ 18.5 小时
- 大容量盘重建时间长，期间系统性能下降

**重建风险：**
- 重建过程中若第二块盘故障（RAID 5），数据丢失
- 大容量 RAID 5 的 URE（不可纠正读取错误）风险
- RAID 6 或 RAID 10 在重建时更安全

### 7.3 热备盘（Hot Spare）

- 预先配置的备用磁盘
- 故障发生后自动开始重建，无需人工更换
- 减少数据暴露在降级状态下的时间
