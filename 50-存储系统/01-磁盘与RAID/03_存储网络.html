存储网络技术

一、存储网络概述

存储网络将计算节点（服务器）与存储设备连接起来，实现数据的远程访问和集中管理。主要技术包括 FC-SAN、iSCSI、NFS、SMB/CIFS 以及新兴的 NVMe over Fabric。

1.1 DAS vs NAS vs SAN

三种存储连接方式对比： DAS（Direct Attached Storage）直连存储： Server ──── SCSI/SAS ────▶ Disk - 简单、低延迟 - 扩展性差，无法共享 NAS（Network Attached Storage）网络附加存储： Server ──── Ethernet/NFS ────▶ NAS 设备 - 文件级访问 - 易于共享和管理 - 适合文件服务 SAN（Storage Area Network）存储区域网络： Server ──── FC/iSCSI ────▶ SAN 交换机 ────▶ 存储阵列 - 块级访问（像本地磁盘一样） - 高性能、高可靠 - 适合数据库和关键应用

1.2 三种方式对比

特性 DAS NAS SAN
访问粒度 块 文件 块
网络 无（直连） TCP/IP FC/iSCSI
距离 短（< 几十米） 任意 中长（FC: 10km+）
性能 最高 中等 高
共享 不支持 支持 支持
成本 低 中 高
典型协议 SCSI/SAS NFS/SMB FC/iSCSI

二、FC-SAN（光纤通道存储区域网络）

Fibre Channel（FC）是一种高速网络技术，专为存储流量设计，提供低延迟、高带宽、高可靠的块级存储访问。

2.1 FC 协议栈

FC 协议栈（5 层）： ┌──────────────────────────┐ │ FC-4 协议映射层 │ ← SCSI-FCP, FC-IP, FICON ├──────────────────────────┤ │ FC-3 公共服务层 │ ← 加密、压缩、条带化 ├──────────────────────────┤ │ FC-2 信号协议层 │ ← 帧格式、流量控制、路由 ├──────────────────────────┤ │ FC-1 编码层 │ ← 8b/10b 编码 ├──────────────────────────┤ │ FC-0 物理层 │ ← 光纤/铜缆、连接器 └──────────────────────────┘ FC-4 最常用的是 SCSI-FCP（Fibre Channel Protocol for SCSI） 即：将 SCSI 命令封装在 FC 帧中传输

2.2 FC 速度演进

标准 线速 有效带宽 年代
1GFC1.0625 Gbps~100 MB/s1997
2GFC2.125 Gbps~200 MB/s2001
4GFC4.25 Gbps~400 MB/s2005
8GFC8.5 Gbps~800 MB/s2008
16GFC14.025 Gbps~1600 MB/s2011
32GFC28.05 Gbps~3200 MB/s2016
64GFC56.1 Gbps~6400 MB/s2020
128GFC112.2 Gbps~12800 MB/s2024+

2.3 FC 拓扑结构

三种 FC 拓扑： 1. 点对点（Point-to-Point）： HBA ──────────────▶ 存储 简单但不灵活 2. 仲裁环（Arbitrated Loop / FC-AL）： HBA ───▶ 存储 ───▶ 存储 ───▶ HBA 共享带宽，最多 127 个设备 3. 交换式（Switched Fabric / FC-SW）： ┌── FC 交换机 ──┐ │ │ HBA ─┤ ├─ 存储 HBA ─┤ ├─ 存储 HBA ─┤ ├─ 存储 └──────────────┘ 高性能、高可用，现代 SAN 的标准拓扑

2.4 FC 关键概念

  • WWN（World Wide Name）：FC 设备的全球唯一标识符（64 位），类似以太网的 MAC 地址
  • FCID（Fabric ID）：24 位地址，由交换机动态分配
  • Zoning（分区）：在 FC 交换机上划分逻辑区域，控制访问权限
  • LUN Masking（LUN 掩码）：在存储阵列端控制哪些主机可访问哪些 LUN
  • Fabric Login（FLOGI）：HBA 连接交换机时的注册过程

三、iSCSI

iSCSI（Internet Small Computer System Interface）将 SCSI 命令封装在 TCP/IP 包中，通过以太网传输，是一种低成本的 SAN 解决方案。

3.1 iSCSI 架构

iSCSI 协议栈： ┌────────────────────────┐ │ SCSI 命令/数据 │ ├────────────────────────┤ │ iSCSI 协议层 │ ← 命令封装、会话管理 ├────────────────────────┤ │ TCP 协议层 │ ← 可靠传输、流量控制 ├────────────────────────┤ │ IP 协议层 │ ← 寻址、路由 ├────────────────────────┤ │ 以太网层 │ ← 链路层 └────────────────────────┘ 关键组件： - Initiator（发起端）：服务器端的 iSCSI 客户端 - Target（目标端）：存储设备的 iSCSI 服务端 - iSCSI Name (IQN)：唯一标识，格式如 iqn.2024-01.com.example:storage

3.2 FC-SAN vs iSCSI

特性 FC-SAN iSCSI
网络基础设施 专用 FC 交换机和 HBA 标准以太网交换机和网卡
性能 更高（专用硬件） 中高（取决于网络）
成本 高 低
延迟 极低（<100 us） 较低（100-500 us）
管理复杂度 高（需要 FC 专业知识） 低（基于 TCP/IP）
距离 中等 任意（IP 网络可达即可）
适用场景 关键业务、大规模企业 中小企业、远程复制

3.3 iSCSI 配置示例

# Linux iSCSI 配置（open-iscsi） # 1. 发现目标 iscsiadm -m discovery -t sendtargets -p 192.168.1.100:3260 # 2. 登录目标 iscsiadm -m node -T iqn.2024-01.com.example:storage \ -p 192.168.1.100:3260 -l # 3. 查看连接 iscsiadm -m session # 4. 现在可以看到新的块设备 lsblk # 输出：sdb 8:16 0 1T 0 disk # 5. 注销 iscsiadm -m node -T iqn.2024-01.com.example:storage -u

四、NFS（Network File System）

NFS 是由 Sun Microsystems 于 1984 年开发的分布式文件系统协议，允许客户端通过网络像访问本地文件一样访问远程文件。

4.1 NFS 版本演进

版本 年份 主要特性
NFSv2 1989 UDP、32 位偏移（文件最大 4GB）、无状态
NFSv3 1995 支持 TCP、64 位偏移、异步写入、更好的错误处理
NFSv4 2003 单端口（2049）、内置安全（Kerberos）、有状态、ACL
NFSv4.1 2010 pNFS（并行 NFS）、会话、客户端追踪
NFSv4.2 2016 服务端复制、稀疏文件、空间预留

4.2 NFS 架构

NFS 客户端-服务器架构： NFS 客户端 NFS 服务器 ┌──────────┐ ┌──────────────┐ │ VFS 层 │ │ VFS 层 │ │ │ │ │ │ NFS │ ─── RPC/XDR ───▶ │ NFS Daemon │ │ Client │ (TCP/UDP 2049) │ (nfsd) │ │ │ │ │ └──────────┘ │ 文件系统层 │ │ (ext4/xfs/..)│ mount -t nfs server:/export /mnt│ │ │ 物理存储 │ └──────────────┘ RPC（Remote Procedure Call）：远程过程调用框架 XDR（External Data Representation）：数据序列化格式

4.3 NFS 配置示例

# NFS 服务器配置 # /etc/exports /export/data 192.168.1.0/24(rw,sync,no_subtree_check,no_root_squash) /export/backup 10.0.0.0/8(ro,async) # 使配置生效 exportfs -ra # NFS 客户端挂载 mount -t nfs4 server:/export/data /mnt/data # /etc/fstab 自动挂载 server:/export/data /mnt/data nfs4 defaults,_netdev 0 0

五、SMB/CIFS

SMB（Server Message Block）是 Windows 环境中的主要文件共享协议。CIFS（Common Internet File System）是 SMB 的早期实现版本，现代系统中 SMB 通常指代 SMB 2/3 协议。

六、NVMe over Fabric（NVMe-oF）

NVMe over Fabric 将 NVMe 协议扩展到网络传输层，使远程 NVMe 存储像本地 NVMe SSD 一样低延迟访问。

6.1 NVMe-oF 架构

NVMe over Fabric 传输层选项： ┌─────────────────────────────────────────┐ │ NVMe 命令集 │ ├─────────────────────────────────────────┤ │ NVMe Transport 层 │ │ ┌─────────┬──────────┬───────────┐ │ │ │ RDMA │ FC │ TCP │ │ │ │(RoCE/iWARP)│ (FC-NVMe) │ (NVMe/TCP) │ │ │ └─────────┴──────────┴───────────┘ │ ├─────────────────────────────────────────┤ │ 物理网络层 │ │ (以太网 / InfiniBand / Fibre Channel) │ └─────────────────────────────────────────┘

6.2 NVMe-oF 传输对比

传输类型 全称 延迟 特点
NVMe/RDMA Remote Direct Memory Access 极低（~5 us） 零拷贝、需要 RDMA 网卡
NVMe/FC Fibre Channel 低（~10 us） 利用现有 FC 投资
NVMe/TCP TCP/IP 中（~50 us） 标准以太网、最易部署

6.3 NVMe-oF vs iSCSI

关键区别：
  • NVMe-oF 比 iSCSI 延迟更低（iSCSI 有额外的 TCP 开销）
  • NVMe-oF 支持更多队列和更深的队列深度
  • NVMe/TCP 是 iSCSI 的自然继任者，提供与 iSCSI 相同的易部署性
  • NVMe/FC 允许在现有 FC 网络上传输 NVMe 命令
  • RDMA 提供最低延迟但需要专用硬件

七、存储网络技术综合对比

技术 协议 访问粒度 网络 延迟 成本 适用场景
FC-SAN SCSI-FCP 块 FC <100 us 高 企业级关键应用
iSCSI SCSI/TCP/IP 块 以太网 100-500 us 低 中小企业
NFS NFS/RPC 文件 以太网 100-500 us 低 Linux 文件共享
SMB SMB 文件 以太网 100-500 us 低 Windows 文件共享
NVMe/RDMA NVMe/RoCE 块 RDMA ~5 us 中高 超低延迟存储
NVMe/FC NVMe/FC 块 FC ~10 us 高 FC 网络现代化
NVMe/TCP NVMe/TCP 块 以太网 ~50 us 低 通用 NVMe-oF 部署
选择建议：
  • 超低延迟需求：NVMe/RDMA（需要 RDMA 网卡和交换机）
  • 已有 FC 投资：NVMe/FC 或 FC-SAN
  • 标准以太网环境：NVMe/TCP 或 iSCSI
  • Linux 文件共享：NFS v4.1+
  • Windows 环境：SMB 3.1.1

总结

存储网络技术经历了从 DAS 到 SAN/NAS 的演进。FC-SAN 以专用光纤通道提供企业级高性能存储访问；iSCSI 利用 TCP/IP 实现低成本块存储；NFS 和 SMB 分别主导 Linux 和 Windows 文件共享。NVMe over Fabric 是最新趋势，将 NVMe 的低延迟优势扩展到网络存储，其中 NVMe/TCP 以其部署简便性有望成为主流。理解各技术的适用场景和性能特征，是设计现代存储基础设施的关键。
