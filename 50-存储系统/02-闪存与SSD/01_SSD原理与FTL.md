# SSD原理与FTL


## 一、NAND Flash基础


NAND Flash是一种非易失性存储器，数据以电荷形式存储在浮栅晶体管中，是SSD的核心存储介质。


### 1.1 存储结构层次


| 层次 | 说明 | 典型大小 |
| --- | --- | --- |
| Cell（存储单元） | 存储一个/多个bit的浮栅晶体管 | 1 bit (SLC) / 2 bit (MLC) / 3 bit (TLC) / 4 bit (QLC) |
| Page（页） | 读写的最小单位 | 4KB / 8KB / 16KB |
| Block（块） | 擦除的最小单位，包含多个Page | 128~512 Pages（256Pages×16KB = 4MB常见） |
| Plane | 包含多个Block，可并行操作 | 多个Block |
| Die | 包含1~2个Plane | - |


### 1.2 SLC/MLC/TLC/QLC对比


| 类型 | 每Cell比特数 | 电压状态 | 寿命(P/E cycles) | 速度 | 成本 |
| --- | --- | --- | --- | --- | --- |
| SLC | 1 | 2 | 50,000~100,000 | 最快 | 最高 |
| MLC | 2 | 4 | 3,000~10,000 | 快 | 中等 |
| TLC | 3 | 8 | 500~3,000 | 较慢 | 较低 |
| QLC | 4 | 16 | 100~1,000 | 最慢 | 最低 |


> **Note:** **关键特性：**
> NAND Flash的读写以Page为单位，但擦除必须以Block为单位。不能原地更新（In-place Update），必须先擦除再写入，这称为"异地更新"（Out-of-place Update）。


## 二、读写与擦除操作


### 2.1 操作粒度


| 操作 | 单位 | 耗时 | 备注 |
| --- | --- | --- | --- |
| 读（Read） | Page | 25~100 μs | 最快的闪存操作 |
| 写/编程（Program） | Page | 200~900 μs | 只能将1写为0 |
| 擦除（Erase） | Block | 1~5 ms | 将整块清为全1 |


### 2.2 写放大（Write Amplification）


由于NAND Flash的擦除粒度远大于写入粒度，实际写入闪存的数据量远大于主机请求写入的数据量。


$$
写放大系数（WAF）= 实际写入闪存的数据量 / 主机写入数据量
                WAF >= 1，理想情况下 WAF = 1
$$


### 2.3 磨损不均


每个Block有有限的擦写次数（P/E Cycles），如果某些Block被频繁擦写，会提前失效。


## 三、闪存转换层（FTL）


FTL（Flash Translation Layer）是SSD固件中的核心模块，位于文件系统和闪存之间，负责管理闪存的读写、磨损均衡和垃圾回收。


### 3.1 地址映射


FTL维护逻辑地址（LBA）到物理地址（PBA）的映射表，实现逻辑块到物理块的转换。


| 映射粒度 | 优点 | 缺点 |
| --- | --- | --- |
| 页级映射 | 灵活，写放大小 | 映射表大（占用DRAM） |
| 块级映射 | 映射表小 | 写放大严重（更新一页需搬移整块） |
| 混合映射 | 折中方案 | 实现复杂 |


### 3.2 磨损均衡（Wear Leveling）


确保所有Block的擦写次数大致均匀，避免部分Block过早磨损。


- **动态磨损均衡**
   ：写入时选择擦写次数少的Block
- **静态磨损均衡**
   ：定期搬移冷数据到高磨损Block，释放低磨损Block给热数据


### 3.3 垃圾回收（Garbage Collection / GC）


回收无效数据占用的空间，合并有效数据到新Block，擦除旧Block。


$$
GC流程：
                1. 选择牺牲块（Victim Block）—— 通常选有效页最少的块
                2. 将牺牲块中的有效页复制到新块
                3. 擦除牺牲块
                4. 更新FTL映射表
$$


> **Important:** **GC开销：**
>
>
> - 前台GC：阻塞用户I/O，延迟高但可预测
>
>
> - 后台GC：空闲时执行，不阻塞I/O但可能突然触发导致延迟抖动
>
>
> - GC导致写放大：搬移有效页时产生额外写入


## 四、SSD vs HDD对比


| 特性 | HDD | SSD |
| --- | --- | --- |
| 存储介质 | 磁性盘片 | NAND Flash |
| 随机读延迟 | 5~10 ms | 0.05~0.1 ms |
| 顺序读带宽 | 100~200 MB/s | 500~7000 MB/s (NVMe) |
| IOPS | 75~200 | 10,000~1,000,000 |
| 抗震性 | 差（机械部件） | 好（无运动部件） |
| 功耗 | 高（6~15W） | 低（2~5W） |
| 寿命 | 机械磨损 | 闪存擦写次数有限 |
| 单位成本 | 低 | 高（持续下降） |
| 容量 | 大（可达20TB+） | 中等（可达8TB+） |


## 五、知识要点总结


1. NAND Flash：页读写、块擦除，不能原地更新
2. SLC/MLC/TLC/QLC：容量越大，寿命越短，速度越慢
3. FTL三大功能：地址映射、磨损均衡、垃圾回收
4. 写放大（WAF）是SSD性能优化的核心问题
5. GC选择有效页最少的块作为牺牲块
6. SSD随机性能远超HDD，但有寿命限制
