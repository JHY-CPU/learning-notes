# 02-闪存与SSD

## 1. NAND 闪存原理

### 1.1 基本存储单元

NAND 闪存基于浮栅晶体管（Floating Gate MOSFET）存储数据。通过向浮栅注入或移除电子来改变阈值电压，从而表示不同的数据值。

```
控制栅（Control Gate）
    │
  ┌─┴─┐
  │浮栅│ ← 电子被绝缘层包围，可长期保存电荷
  └─┬─┘
  隧穿氧化层
    │
  源极 ─── 漏极
```

### 1.2 SLC / MLC / TLC / QLC

| 类型 | 每单元比特数 | 电压状态数 | 寿命（P/E cycles） | 速度 | 成本 |
|------|-------------|-----------|-------------------|------|------|
| SLC | 1 bit | 2 | 50,000~100,000 | 最快 | 最高 |
| MLC | 2 bit | 4 | 3,000~10,000 | 快 | 中 |
| TLC | 3 bit | 8 | 500~3,000 | 中等 | 较低 |
| QLC | 4 bit | 16 | 100~1,000 | 慢 | 最低 |
| PLC | 5 bit | 32 | <100 | 最慢 | 最低 |

**趋势：** 每代容量增大、成本降低，但耐久性和性能下降。

### 1.3 闪存组织结构

```
Page（页）    → 读写的最小单位（4KB / 8KB / 16KB）
Block（块）   → 擦除的最小单位（包含多个 Page，如 128~256 Page）
Plane（面）   → 包含多个 Block
Die/LUN      → 包含多个 Plane
Chip         → 一个物理 NAND 芯片
```

## 2. 闪存的读写擦特性

### 2.1 操作特性

| 操作 | 最小单位 | 耗时 | 特性 |
|------|---------|------|------|
| 读（Read） | Page | ~25-100 μs | 非破坏性，可多次读取 |
| 写（Program） | Page | ~200-900 μs | 只能将 1→0 |
| 擦除（Erase） | Block | ~1-5 ms | 将整个 Block 重置为全 1 |

### 2.2 写前擦除约束

- 闪存**不能原地更新**：写入前必须先擦除
- 擦除单位（Block）远大于写入单位（Page）
- 已有数据的 Block 要修改某个 Page，需要：
  1. 将 Block 中有效 Page 读出
  2. 擦除整个 Block
  3. 将有效数据 + 新数据写回

### 2.3 读干扰与写干扰

- **读干扰（Read Disturb）：** 频繁读取某 Page 可能导致同 Block 中其他 Page 的浮栅电子泄露
- **编程干扰（Program Disturb）：** 编程某 Page 时可能意外修改同 Block 中其他 Page
- 解决方案：定期刷新数据（Scrubbing）

## 3. 闪存转换层（FTL）

### 3.1 FTL 的作用

FTL（Flash Translation Layer）是 SSD 控制器中的核心固件，负责：

- 将主机的逻辑块地址（LBA）映射到闪存的物理地址
- 管理闪存的读写擦操作
- 实现磨损均衡、垃圾回收等功能

### 3.2 地址映射方式

**页级映射（Page-level Mapping）：**
- 每个逻辑页映射到一个物理页
- 映射表大（如 1TB SSD 需要约 2GB 映射表）
- 灵活度最高

**块级映射（Block-level Mapping）：**
- 每个逻辑块映射到一个物理块
- 映射表小
- 写放大严重（修改一个页需要搬移整个块）

**混合映射（Hybrid Mapping）：**
- 结合页级和块级映射
- 如日志结构：新写入使用页级映射，积累到整块后转换为块级映射
- 平衡性能和内存开销

### 3.3 日志结构 FTL

```
写入请求 → 写入 Log Block（页级映射）
            ↓
Log Block 满 → 合并（Merge）到 Data Block
            ↓
旧数据标记无效 → 垃圾回收
```

## 4. 磨损均衡（Wear Leveling）

### 4.1 为什么需要磨损均衡

- 每个闪存块的擦除次数有限（P/E cycles）
- 热数据频繁更新会导致对应块快速磨损
- 冷数据块几乎不被擦除

### 4.2 磨损均衡算法

**动态磨损均衡：** 写入时选择擦除次数最少的块。

**静态磨损均衡：**
- 定期将冷数据（擦除次数少的块）与热数据（擦除次数多的块）交换
- 确保所有块的擦除次数趋于均匀
- 比动态磨损均衡更有效但开销更大

**磨损均衡因子：**

```
WLF = Max_PE / Average_PE
```

理想值为 1，实际中接近 1 表示均衡效果好。

### 4.3 坏块管理

- 出厂时可能有初始坏块
- 使用过程中出现的坏块需要标记并移出使用
- 维护坏块表（Bad Block Table, BBT）

## 5. 垃圾回收（GC）

### 5.1 GC 的必要性

由于闪存的写前擦除约束，SSD 需要垃圾回收机制：
- 有效页被分散在多个块中
- 无效页占据空间
- 需要将有效页集中，擦除含有大量无效页的块

### 5.2 GC 流程

1. **选择受害者块（Victim Selection）：** 选择无效页比例最高的块
2. **迁移有效页：** 将有效页读出并写入新的空闲块
3. **擦除原块：** 将原块加入空闲块池
4. **更新映射表：** 更新 FTL 中的地址映射

### 5.3 前台 GC vs 后台 GC

| 类型 | 触发时机 | 对用户 I/O 的影响 |
|------|---------|-------------------|
| 前台 GC | 空闲空间不足时 | 直接阻塞用户 I/O，延迟增大 |
| 后台 GC | 系统空闲时 | 无直接影响 |

### 5.4 OP（Over-Provisioning）预留空间

- SSD 实际闪存容量 > 标称容量
- 额外空间用于 GC 过程中有效页的临时存放
- OP 越大，GC 效率越高，写放大越小
- 典型 OP 比例：7%（消费级）~ 28%（企业级）

## 6. 写放大问题

### 6.1 写放大因子（WAF）

```
WAF = 实际写入闪存的数据量 / 主机写入的数据量
```

- WAF ≥ 1（理想值为 1）
- 由 GC、磨损均衡等内部操作引起

### 6.2 写放大的来源

| 来源 | 原因 |
|------|------|
| GC | 迁移有效页时的额外写入 |
| 磨损均衡 | 冷热数据交换产生的写入 |
| 元数据更新 | FTL 映射表的更新 |
| Trim 未生效 | 逻辑删除的页未被标记为无效 |

### 6.3 TRIM 命令

- 操作系统通知 SSD 某些逻辑块不再使用
- SSD 可将对应物理页标记为无效
- GC 时无需迁移这些页，降低写放大
- 现代 OS 和 SSD 均支持 TRIM

### 6.4 降低写放大的方法

- 增大预留空间（OP）
- 启用 TRIM
- 使用更智能的 GC 策略
- Host 端发送足够大的写入请求（减少小写）

## 7. SSD 性能特性

### 7.1 性能指标

| 指标 | 说明 | 典型值（NVMe SSD） |
|------|------|-------------------|
| 顺序读 | 大块数据连续读取 | 3,000~7,000 MB/s |
| 顺序写 | 大块数据连续写入 | 2,000~5,000 MB/s |
| 4K 随机读 IOPS | 4KB 随机读取每秒操作数 | 500K~1,500K |
| 4K 随机写 IOPS | 4KB 随机写入每秒操作数 | 200K~1,000K |
| 延迟 | 单次 I/O 响应时间 | 10~100 μs |

### 7.2 稳态性能

- SSD 在空盘时性能最高（Fresh out of box, FOB）
- 持续写入后进入稳态（Steady State），性能下降
- 企业级 SSD 关注稳态性能指标

### 7.3 与 HDD 对比

| 特性 | HDD | SSD |
|------|-----|-----|
| 随机读 IOPS | 100~200 | 100K~1M+ |
| 顺序带宽 | 100~250 MB/s | 500~7000 MB/s |
| 访问延迟 | 5~15 ms | 0.01~0.1 ms |
| 功耗 | 6~15W | 2~8W |
| 抗震 | 弱 | 强 |
| 每GB成本 | 低 | 高（持续下降中） |

## 8. NVMe 协议

### 8.1 NVMe 概述

NVMe（Non-Volatile Memory Express）是专为闪存设计的存储协议，替代 AHCI/SATA。

| 特性 | AHCI (SATA) | NVMe |
|------|-------------|------|
| 队列数 | 1 | 65,535 |
| 队列深度 | 32 | 65,536 |
| 命令大小 | 48B | 64B |
| 最大带宽 | ~600 MB/s | ~7,000 MB/s (PCIe 5.0) |
| 延迟 | ~6 μs | ~2.8 μs |
| CPU 开销 | 高 | 低 |

### 8.2 NVMe 队列机制

```
主机内存
┌──────────────┐
│  SQ (提交队列) │ → 控制器读取命令
│  CQ (完成队列) │ ← 控制器写入完成状态
└──────────────┘
     ↕ Doorbell 寄存器
┌──────────────┐
│  NVMe 控制器   │ → 直接访问闪存
└──────────────┘
```

- SQ/CQ 成对出现，支持多队列
- 每个 CPU 核心可使用独立队列，避免锁竞争
- Doorbell 寄存器通知控制器有新命令

### 8.3 NVMe over Fabrics（NVMe-oF）

- 将 NVMe 协议扩展到网络
- 支持 RDMA（RoCE/iWARP）和 TCP 传输
- 实现远端闪存的低延迟访问
- 用于数据中心的分布式存储

## 9. 3D NAND 技术

### 9.1 2D NAND 的瓶颈

- 缩小单元尺寸导致单元间干扰增大
- 电子数量减少导致可靠性下降
- 制造工艺接近物理极限

### 9.2 3D NAND 原理

将存储单元垂直堆叠，而非水平排列：

```
2D NAND：单元横向排列在平面上
3D NAND：单元纵向堆叠成多层

       Gate
    ┌───┴───┐
    │ Layer │  ← 64/96/128/232 层
    │ Layer │
    │ Layer │
    │ Layer │
    └───────┘
      通道孔（Channel Hole）
```

### 9.3 3D NAND 的优势

- 不依赖先进光刻工艺
- 单元尺寸大，可靠性和耐久性更好
- 每 GB 成本更低
- 主流厂商层数已超过 200 层

### 9.4 技术路线

| 厂商 | 技术名称 | 特点 |
|------|---------|------|
| Samsung | V-NAND | Charge Trap (CT) |
| Intel/Micron | BICS/QLC | Charge Trap |
| Kioxia/WD | BiCS FLASH | Charge Trap |
| YMTC | Xtacking | 存储与逻辑分离制造 |

## 10. 存储级内存（SCM / Intel Optane）

### 10.1 SCM 概述

存储级内存（Storage Class Memory）填补 DRAM 和 NAND 之间的性能/容量/成本鸿沟。

```
延迟:  DRAM (10ns) → SCM (100ns~10μs) → NAND (25~100μs)
成本:  DRAM ($/GB 高) → SCM (中) → NAND ($/GB 低)
容量:  DRAM (小) → SCM (中) → NAND (大)
```

### 10.2 Intel Optane（3D XPoint）

- 基于相变存储（PCM-like）技术
- 字节寻址（可直接通过内存总线访问）
- 持久性（断电不丢数据）
- 延迟约 10μs（比 NAND 快 10 倍，比 DRAM 慢 100 倍）

**Optane 产品形态：**
- Optane Persistent Memory（DIMM 形态，内存插槽）
- Optane SSD（NVMe 形态，如 P5800X）

**注意：** Intel 已于 2022 年宣布退出 Optane 业务。

### 10.3 其他 SCM 技术

| 技术 | 原理 | 状态 |
|------|------|------|
| 3D XPoint | 相变/硫系化合物 | Intel 已停产 |
| MRAM | 磁阻随机存储 | 量产中（小容量） |
| ReRAM | 电阻变化 | 研发中 |
| FeRAM | 铁电材料 | 小规模应用 |
