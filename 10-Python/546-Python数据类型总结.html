Python 完整类型体系

// ========== 类型分类 ========== // 按可变性分类: // // 不可变 (Immutable): // int, float, bool, complex, str, bytes // tuple, frozenset, range // namedtuple (及其子类) // // 可变 (Mutable): // list, dict, set, bytearray // 自定义类实例 // 按可哈希性分类: // 可哈希 (可做字典键/集合元素): // 不可变类型 (包括 tuple 含有不可变元素) // // 不可哈希: // 可变类型 (list, dict, set) // 按可迭代性分类: // 可迭代: // 几乎所有容器: list, str, tuple, dict, set, range, file // 生成器, 迭代器 // ========== 可变性检测 ========== x = [1, 2, 3] y = x y.append(4) # 修改引用对象, x 也变! print(x) # [1, 2, 3, 4] x = "hello" y = x y = y + " world" # 创建新字符串, x 不变! print(x) # "hello" // 用 id() 验证: x = [1, 2, 3] print(id(x)) x.append(4) print(id(x)) # 不变 (可变) x = x + [5] print(id(x)) # 变! 深拷贝 vs 浅拷贝

// ========== 拷贝 ========== import copy original = { "name": "Alice", "scores": [85, 92, 78], "address": {"city": "Beijing", "zip": "100000"} } // 1. 引用 (只是别名) ref = original ref["name"] = "Bob" print(original["name"]) # "Bob" (修改了原对象!) // 2. 浅拷贝 (复制一层) shallow = original.copy() # 或 copy.copy(original) shallow["name"] = "Charlie" print(original["name"]) # "Bob" (第一层没影响) shallow["scores"].append(99) print(original["scores"]) # [85, 92, 78, 99] (嵌套对象被修改!) // 3. 深拷贝 (完整递归复制) deep = copy.deepcopy(original) deep["scores"].append(100) print(original["scores"]) # [85, 92, 78, 99] (不受影响!) print(deep["scores"]) # [85, 92, 78, 99, 100] // ========== 浅拷贝的不同方式 ========== # 列表: list1 = [1, [2, 3], 4] list2 = list(list1) # 浅拷贝 list3 = list1[:] # 切片也是浅拷贝 list4 = [*list1] # 解包也是浅拷贝 # 字典: dict1 = {"a": 1, "b": [2, 3]} dict2 = dict(dict1) # 浅拷贝 dict3 = {**dict1} # 解包也是浅拷贝 // ========== 什么时候需要深拷贝 ========== // 1. 嵌套可变结构 (列表的列表,字典的列表) // 2. 要从原型创建独立副本 // 3. 缓存/快照时保留原始数据 // 4. 避免意外修改共享引用 // 简单类型不需要拷贝: a = 42 b = a # 整数不可变,引用安全

类型选择决策树

// ========== 选择决策树 ========== // 需要存储什么类型的数据? // │ // ├─ 单一值 → int/float/str/bool/None // │ // ├─ 多个值: // │ ├─ 有序: list (可变) / tuple (不可变) // │ ├─ 唯一: set (可变) / frozenset (不可变) // │ └─ 映射: dict (可变) // │ // ├─ 特殊行为: // │ ├─ 计数: Counter // │ ├─ 默认值: defaultdict // │ ├─ 双端队列: deque // │ ├─ 优先级队列: heapq // │ └─ 二分查找: bisect // │ // ├─ 需命名属性: // │ ├─ 简单只读: namedtuple // │ ├─ 完整类: dataclass // │ └─ 快速原型: SimpleNamespace // │ // └─ 需枚举: Enum / IntEnum / Flag // ========== 性能参考 ========== // 操作 list tuple set dict // 索引 O(1) O(1) N/A O(1) // 查找 in O(n) O(n) O(1) O(1) // 追加 O(1) 不可变 O(1) - (key) // 插入 O(n) 不可变 O(1) O(1) // 删除 O(n) 不可变 O(1) O(1) // 内存 ~56 ~48 ~224 ~232 (每项基础开销) // 结论: set/dict 的 in 比 list 快得多! // 大数据集查找用 set, 映射用 dict // ========== 类型转换速查 ========== # 转列表: list(iterable) # 转元组: tuple(iterable) # 转集合: set(iterable) # 转字典: dict(zip(keys, values)) # 转字符串: str(obj), ",".join(list) # 转整数: int("42"), int(3.14) # 转浮点: float("3.14")

💡 数据类型总结: (1) 可变/不可变 是最重要的分类; (2) 浅拷贝只复制一层,深拷贝递归复制; (3) set/dict 的 in 操作 O(1) 远快于 list 的 O(n); (4) 选择: 有序→list/tuple, 唯一→set, 映射→dict, 命名→dataclass; (5) 高级 OOP: 继承用 super(), 封装用 @property, 接口用 ABC。
