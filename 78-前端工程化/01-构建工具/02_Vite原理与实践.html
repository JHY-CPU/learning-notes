Vite原理与实践

核心概念：Vite是一个基于ESM（ES Module）的下一代前端构建工具。开发环境利用浏览器原生ESM能力实现按需编译，配合esbuild预构建依赖，实现极速启动；生产环境使用Rollup进行打包，输出优化的静态资源。Vite的核心优势是开发体验：冷启动极快、HMR即时更新。

一、Vite架构设计

1.1 开发环境 vs 生产环境

开发环境（Dev）： 生产环境（Build）： ┌─────────────────────┐ ┌─────────────────────┐ │ 浏览器 │ │ Rollup │ │ 原生ESM按需请求模块 │ │ 静态分析依赖 │ │ ↓ │ │ Tree Shaking │ │ Vite Dev Server │ │ 代码压缩 │ │ (Koa + esbuild) │ │ Code Splitting │ │ ↓ │ │ CSS提取 │ │ 即时编译单个文件 │ │ 资源优化 │ │ HMR通过WebSocket │ │ ↓ │ │ esbuild预构建依赖 │ │ dist/ 静态资源 │ └─────────────────────┘ └─────────────────────┘

1.2 为什么Vite这么快

环节WebpackVite
冷启动打包所有模块后启动（慢）不打包，按需编译（快）
依赖预构建打包时处理esbuild预构建（Go语言，极快）
文件编译Webpack编译esbuild编译（比babel快10-100倍）
HMR重新构建受影响的模块链精确失效，只编译修改的文件
TS编译ts-loader（慢）esbuild转译（不检查类型）

二、ESM开发服务器原理

2.1 浏览器原生ESM

现代浏览器原生支持 <script type="module">，Vite利用这一特性，在开发时让浏览器按需请求模块：

<!-- index.html -->
<script type="module" src="/src/main.js"></script>

// 浏览器请求 main.js 后，发现其中的 import 语句：
import { createApp } from '/node_modules/.vite/deps/vue.js?v=abc123'
import App from '/src/App.vue?t=1234567890'

// 浏览器继续请求这些依赖...
// Vite在服务器端即时编译每个请求的文件

2.2 Vite Dev Server工作流程

1. 启动时： - esbuild预构建 node_modules 中的依赖（转为ESM，缓存到 node_modules/.vite） - 启动HTTP服务器（基于Koa/Connect） 2. 浏览器请求 /src/main.js： - Vite拦截请求 - 读取源文件 - 如果是Vue SFC → @vitejs/plugin-vue 编译 - 如果是TS → esbuild转译 - 如果是CSS → 处理后注入JS - 返回编译后的ESM模块 3. 文件修改时： - Vite检测文件变化 - 仅编译修改的文件 - 通过WebSocket通知浏览器 - 浏览器精确替换模块

2.3 依赖预构建

由于许多npm包仍然使用CommonJS格式，且一个包可能有多个小模块（如lodash有600+个模块），Vite在启动时使用esbuild进行预构建：

// 预构建的作用：
// 1. CommonJS → ESM 转换
//    require('xxx') → import xxx from 'xxx'

// 2. 依赖合并
//    lodash有600+个小文件 → 合并为一个ESM文件
//    减少浏览器请求数

// 3. 缓存
//    预构建结果缓存在 node_modules/.vite/
//    只有 package.json 或 lockfile 变化时才重新构建

// vite.config.js 配置预构建
export default {
    optimizeDeps: {
        include: ['lodash', 'axios'],      // 强制预构建
        exclude: ['some-local-package'],    // 排除预构建
        needsInterop: ['problematic-lib'],  // 强制interop转换
        esbuildOptions: {
            // esbuild选项
        }
    }
};

三、Vite的HMR实现

3.1 HMR流程

文件修改（chokidar监听） ↓ Vite分析模块图，找到修改文件的所有依赖者（importers） ↓ 精确失效：只标记修改的模块和直接依赖它的模块为invalid ↓ 通过WebSocket发送更新通知 ↓ 浏览器只请求更新的模块（精确的URL + timestamp查询参数） ↓ 执行 HMR API 回调，更新DOM/CSS

3.2 Vite HMR API

// Vite提供 import.meta.hot API

// 1. 监听自身模块更新
if (import.meta.hot) {
    import.meta.hot.accept((newModule) => {
        // 模块更新后的处理
        console.log('模块已更新', newModule);
    });
}

// 2. 监听依赖模块更新
if (import.meta.hot) {
    import.meta.hot.accept('./utils', (newUtils) => {
        // utils模块更新后的处理
        updateUI(newUtils);
    });
}

// 3. 监听CSS更新（自动处理，无需手动代码）
// Vite自动将CSS提取为CSS模块，更新时通过<link>标签替换

// 4. 监听Vue SFC更新（@vitejs/plugin-vue自动处理）
// 保留组件状态，只更新变化的部分

// 5. 自定义HMR处理
if (import.meta.hot) {
    import.meta.hot.on('custom-event', (data) => {
        console.log('自定义事件:', data);
    });

    // 发送事件到服务器
    import.meta.hot.send('client-event', { msg: 'hello' });
}

四、Vite插件系统

4.1 插件钩子

Vite插件基于Rollup的插件接口，并扩展了一些Vite特有的钩子：

钩子阶段说明
configVite特有修改Vite配置
configResolvedVite特有配置解析完成后
configureServerVite特有配置Dev Server中间件
transformIndexHtmlVite特有转换HTML
resolveIdRollup解析模块路径
loadRollup加载模块内容
transformRollup转换模块内容
buildStartRollup构建开始
buildEndRollup构建结束
generateBundleRollup生成bundle

4.2 自定义Vite插件

// vite-plugin-example.js
export default function myPlugin(options = {}) {
    return {
        name: 'vite-plugin-example',  // 插件名称（必须）

        // 修改Vite配置
        config(config, { command }) {
            return {
                define: {
                    __APP_VERSION__: JSON.stringify('1.0.0')
                }
            };
        },

        // 配置Dev Server
        configureServer(server) {
            // 添加中间件
            server.middlewares.use('/api/custom', (req, res) => {
                res.end(JSON.stringify({ msg: 'Hello from plugin' }));
            });

            // 监听文件变化
            server.watcher.on('change', (file) => {
                console.log('文件变化:', file);
            });
        },

        // 转换模块内容
        transform(code, id) {
            // 只处理 .js 文件
            if (!id.endsWith('.js')) return;

            // 替换代码中的占位符
            return code.replace('__BUILD_TIME__', Date.now().toString());
        },

        // 转换HTML
        transformIndexHtml(html) {
            return html.replace(
                '<title>',
                '<title>[DEV] '
            );
        }
    };
}

// 使用插件
// vite.config.js
import myPlugin from './vite-plugin-example';

export default {
    plugins: [
        myPlugin({ option: 'value' })
    ]
};

五、Vite配置详解

5.1 常用配置

// vite.config.js
import { defineConfig } from 'vite';
import vue from '@vitejs/plugin-vue';
import path from 'path';

export default defineConfig({
    // 基础路径
    base: '/',

    // 插件
    plugins: [vue()],

    // 路径别名
    resolve: {
        alias: {
            '@': path.resolve(__dirname, 'src'),
            '@components': path.resolve(__dirname, 'src/components')
        },
        extensions: ['.js', '.ts', '.jsx', '.tsx', '.vue', '.json']
    },

    // CSS配置
    css: {
        preprocessorOptions: {
            scss: {
                additionalData: `@import "@/styles/variables.scss";`
            }
        },
        modules: {
            localsConvention: 'camelCase'
        }
    },

    // 开发服务器
    server: {
        port: 3000,
        open: true,
        cors: true,
        proxy: {
            '/api': {
                target: 'http://localhost:8080',
                changeOrigin: true,
                rewrite: (path) => path.replace(/^\/api/, '')
            }
        }
    },

    // 构建配置
    build: {
        target: 'es2015',
        outDir: 'dist',
        assetsDir: 'assets',
        sourcemap: false,
        minify: 'esbuild',
        rollupOptions: {
            output: {
                manualChunks: {
                    vendor: ['vue', 'vue-router', 'pinia'],
                    utils: ['lodash-es', 'dayjs']
                }
            }
        },
        chunkSizeWarningLimit: 1000  // KB
    }
});

六、Vite SSR支持

6.1 SSR原理

// server.js - Vite SSR开发服务器
import { createServer } from 'vite';

async function startServer() {
    const vite = await createServer({
        server: { middlewareMode: true },
        appType: 'custom'
    });

    app.use(vite.middlewares);

    app.use('*', async (req, res) => {
        // 1. 读取index.html
        let template = fs.readFileSync('index.html', 'utf-8');
        template = await vite.transformIndexHtml(req.originalUrl, template);

        // 2. 加载SSR入口
        const { render } = await vite.ssrLoadModule('/src/entry-server.js');

        // 3. 服务端渲染
        const [appHtml, preloadLinks] = await render(req.originalUrl);

        // 4. 注入HTML
        const html = template
            .replace('', preloadLinks)
            .replace('', appHtml);

        res.status(200).set({ 'Content-Type': 'text/html' }).end(html);
    });
}

startServer();

6.2 SSR关键API

  • vite.ssrLoadModule(url)：在Node.js中加载ESM模块（无需打包）
  • vite.ssrFixStacktrace(err)：修复SSR错误的堆栈信息
  • import.meta.env.SSR：判断当前是否为SSR环境
  • import.meta.glob()：批量导入模块（支持SSR）

七、Vite最佳实践

7.1 性能优化

  • 依赖预构建优化：对已知的大依赖手动配置 optimizeDeps.include
  • 图片压缩：使用 vite-plugin-imagemin
  • Gzip/Brotli压缩：使用 vite-plugin-compression
  • CDN引入：使用 rollup-plugin-external-globals + vite-plugin-html
  • 按需导入：使用 unplugin-vue-components 自动按需导入组件
  • 可视化分析：使用 rollup-plugin-visualizer
// 性能优化配置示例
import { visualizer } from 'rollup-plugin-visualizer';
import compression from 'vite-plugin-compression';

export default defineConfig({
    plugins: [
        // Gzip压缩
        compression({ algorithm: 'gzip' }),
        // 打包分析
        visualizer({ open: true, gzipSize: true })
    ],
    build: {
        rollupOptions: {
            // CDN外部化
            external: ['vue', 'vue-router'],
            output: {
                globals: {
                    vue: 'Vue',
                    'vue-router': 'VueRouter'
                }
            }
        }
    }
});

总结：Vite通过ESM原生按需编译、esbuild预构建依赖、精确HMR实现极速开发体验。生产环境使用Rollup输出优化的静态资源。核心优势是：冷启动快（不打包直接服务）、HMR即时更新（精确失效）、开箱即用（TS/CSS/PostCSS/SFC等）。Vite已成为Vue和React项目的首选构建工具。
