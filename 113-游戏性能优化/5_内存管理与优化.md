# 内存管理与优化

## 核心概念

游戏内存管理涉及托管堆（C#）、原生内存（C++引擎）、GPU显存三大区域，任一超限都会导致崩溃、卡顿或强制退出。移动平台尤其严格：iOS会在内存压力时杀死后台应用，Android的OOM Killer会在超出阈值时终止进程。

### 内存分类详解

- **托管堆(Managed Heap)**：C#对象分配，受GC管理。频繁分配导致GC暂停（卡顿）。Unity使用Boehm GC，非分代、非压缩，碎片化问题严重
- **原生内存(Native Memory)**：纹理、网格、音频、动画等Unity引擎C++资源。不受C# GC管理，需要手动释放
- **显存/GPU内存**：纹理、RenderTarget、Shader、ComputeBuffer等。带宽和容量都有限
- **Lua虚拟机内存**（如使用xLua/ToLua）：独立的GC周期，需要单独管理

### 各平台内存预算参考

| 平台 | 总内存 | 托管堆建议 | 纹理预算 | 说明 |
|------|--------|-----------|---------|------|
| iOS低端(3GB) | 1.5GB可用 | <200MB | <300MB | 超过会被系统杀死 |
| iOS高端(6GB) | 3GB可用 | <400MB | <600MB | - |
| Android低端(4GB) | 1.5GB可用 | <200MB | <250MB | 更严格 |
| Android高端(8GB) | 4GB可用 | <500MB | <800MB | - |
| PC | 无严格限制 | <2GB | <4GB | - |

### GC（垃圾回收）优化策略

GC暂停是托管语言游戏的头号性能杀手。Unity的Boehm GC特点：
- **非分代**：每次GC扫描整个堆，对象越多耗时越长
- **非压缩**：不移动对象整理内存，碎片化严重
- **暂停式**：GC时暂停所有MonoBehaviour执行
- **触发条件**：托管堆分配不足时自动触发

优化策略：
1. 减少堆分配（零GC编程）
2. 增量GC（Unity 2019+支持，分散GC工作到多帧）
3. 手动管理（对象池）

## 具体实现方法

### 完整对象池系统

```csharp
using System.Collections.Generic;
using UnityEngine;

/// <summary>
/// 通用游戏对象池
/// 支持预热、容量限制、自动回收
/// </summary>
public class GameObjectPool
{
    private Queue<GameObject> inactivePool = new Queue<GameObject>();
    private HashSet<GameObject> activeObjects = new HashSet<GameObject>();
    private GameObject prefab;
    private Transform poolRoot;
    private int maxSize;
    private string poolName;

    public int ActiveCount => activeObjects.Count;
    public int InactiveCount => inactivePool.Count;
    public int TotalCount => ActiveCount + InactiveCount;

    public GameObjectPool(string name, GameObject prefab,
        int prewarm = 20, int maxSize = 200)
    {
        this.poolName = name;
        this.prefab = prefab;
        this.maxSize = maxSize;

        // 创建池根节点
        poolRoot = new GameObject($"Pool_{name}").transform;

        // 预热
        for (int i = 0; i < prewarm; i++)
        {
            GameObject obj = CreateNew();
            obj.SetActive(false);
            inactivePool.Enqueue(obj);
        }
    }

    public GameObject Get(Vector3 position, Quaternion rotation)
    {
        GameObject obj;
        if (inactivePool.Count > 0)
        {
            obj = inactivePool.Dequeue();
        }
        else
        {
            if (TotalCount >= maxSize)
            {
                Debug.LogWarning($"Pool {poolName} reached max size {maxSize}");
                return null;
            }
            obj = CreateNew();
        }

        obj.transform.SetPositionAndRotation(position, rotation);
        obj.SetActive(true);
        activeObjects.Add(obj);
        return obj;
    }

    public void Return(GameObject obj)
    {
        if (obj == null) return;

        obj.SetActive(false);
        obj.transform.SetParent(poolRoot);
        activeObjects.Remove(obj);

        if (inactivePool.Count < maxSize)
            inactivePool.Enqueue(obj);
        else
            Object.Destroy(obj);
    }

    public void ReturnAll()
    {
        var toReturn = new List<GameObject>(activeObjects);
        foreach (var obj in toReturn)
            Return(obj);
    }

    public void Dispose()
    {
        foreach (var obj in inactivePool)
            if (obj != null) Object.Destroy(obj);
        foreach (var obj in activeObjects)
            if (obj != null) Object.Destroy(obj);
        if (poolRoot != null)
            Object.Destroy(poolRoot.gameObject);
        inactivePool.Clear();
        activeObjects.Clear();
    }

    private GameObject CreateNew()
    {
        GameObject obj = Object.Instantiate(prefab);
        obj.name = $"{poolName}_{TotalCount}";
        obj.transform.SetParent(poolRoot);
        return obj;
    }
}

/// <summary>
/// 对象池管理器（单例）
/// 统一管理游戏中所有对象池
/// </summary>
public class PoolManager : MonoBehaviour
{
    public static PoolManager Instance { get; private set; }

    private Dictionary<string, GameObjectPool> pools
        = new Dictionary<string, GameObjectPool>();

    void Awake()
    {
        if (Instance != null) { Destroy(gameObject); return; }
        Instance = this;
    }

    public GameObjectPool CreatePool(string name, GameObject prefab,
        int prewarm = 20, int maxSize = 200)
    {
        if (pools.ContainsKey(name))
        {
            Debug.LogWarning($"Pool {name} already exists");
            return pools[name];
        }

        var pool = new GameObjectPool(name, prefab, prewarm, maxSize);
        pools[name] = pool;
        return pool;
    }

    public GameObjectPool GetPool(string name)
    {
        pools.TryGetValue(name, out var pool);
        return pool;
    }

    public GameObject Spawn(string poolName, Vector3 pos, Quaternion rot)
    {
        var pool = GetPool(poolName);
        return pool?.Get(pos, rot);
    }

    public void Despawn(string poolName, GameObject obj)
    {
        var pool = GetPool(poolName);
        pool?.Return(obj);
    }

    void OnDestroy()
    {
        foreach (var pool in pools.Values)
            pool.Dispose();
        pools.Clear();
    }
}
```

### C#零GC编程技巧（完整版）

```csharp
using System;
using System.Buffers;
using System.Collections.Generic;
using System.Text;
using UnityEngine;

/// <summary>
/// 零GC编程完整指南
/// 目标：热路径每帧GC Alloc < 0
/// </summary>
public class ZeroGCGuide
{
    // === 1. 预分配集合容量 ===
    // 差：默认容量4，多次扩容产生GC
    private List<int> badList = new List<int>();

    // 好：预分配足够容量
    private List<int> goodList = new List<int>(128);

    // === 2. 避免foreach在旧版List上 ===
    // 在旧C#版本中List<T>.GetEnumerator()产生装箱
    // C# 7.3+ 已优化，但在Dictionary上foreach仍产生Enumerator分配
    void Bad_ForeachDict(Dictionary<int, string> dict)
    {
        foreach (var kvp in dict) // 产生Enumerator装箱
            Process(kvp.Key, kvp.Value);
    }

    void Good_ForLoopDict(Dictionary<int, string> dict)
    {
        var keys = new List<int>(dict.Keys); // 缓存key列表
        for (int i = 0; i < keys.Count; i++)
            Process(keys[i], dict[keys[i]]);
    }

    // === 3. ArrayPool代替new数组 ===
    void Good_ArrayPool()
    {
        byte[] buffer = ArrayPool<byte>.Shared.Rent(1024);
        try { UseBuffer(buffer); }
        finally { ArrayPool<byte>.Shared.Return(buffer); }
    }

    // === 4. Span<T>零分配切片 ===
    void Good_Span()
    {
        string data = "Hello,World,Test";
        ReadOnlySpan<char> span = data.AsSpan();
        // 零分配地分割字符串
    }

    // === 5. StringBuilder复用 ===
    private StringBuilder sb = new StringBuilder(128);
    void Good_StringBuilder()
    {
        sb.Clear();
        sb.Append("HP:").Append(100).Append(" MP:").Append(50);
    }

    // === 6. 缓存WaitForSeconds ===
    private static readonly WaitForSeconds wait1s = new WaitForSeconds(1f);
    private static readonly WaitForSeconds wait05s = new WaitForSeconds(0.5f);

    // === 7. 缓存Animator/Shader属性哈希 ===
    private static readonly int SpeedHash = Animator.StringToHash("Speed");
    private static readonly int ColorHash = Shader.PropertyToID("_Color");
    // 之后使用 animator.SetFloat(SpeedHash, value) 代替字符串版本

    // 占位方法
    void Process(int k, string v) { }
    void UseBuffer(byte[] b) { }
}
```

```csharp
/// <summary>
/// 纹理内存管理
/// 包括Mipmap流式加载和平台自适应
/// </summary>
public class TextureMemoryManager : MonoBehaviour
{
    void Start()
    {
        ConfigureForPlatform();
    }

    void ConfigureForPlatform()
    {
#if UNITY_IOS
        // iOS纹理预算
        QualitySettings.streamingMipmapsActive = true;
        QualitySettings.streamingMipmapsMemoryBudget = 200; // 200MB
        QualitySettings.streamingMipmapsRenderersPerFrame = 512;
        Application.targetFrameRate = 60;

#elif UNITY_ANDROID
        // Android根据内存大小动态配置
        long systemMemory = SystemInfo.systemMemorySize;
        if (systemMemory <= 3072) // 3GB以下
        {
            QualitySettings.streamingMipmapsActive = true;
            QualitySettings.streamingMipmapsMemoryBudget = 128;
            QualitySettings.masterTextureLimit = 1; // 半分辨率
        }
        else if (systemMemory <= 4096)
        {
            QualitySettings.streamingMipmapsActive = true;
            QualitySettings.streamingMipmapsMemoryBudget = 200;
        }
        else
        {
            QualitySettings.streamingMipmapsActive = true;
            QualitySettings.streamingMipmapsMemoryBudget = 300;
        }
#elif UNITY_STANDALONE
        // PC端
        QualitySettings.streamingMipmapsActive = true;
        QualitySettings.streamingMipmapsMemoryBudget = 512;
#endif
    }

    /// <summary>
    /// 检查当前纹理内存使用
    /// </summary>
    void CheckTextureMemory()
    {
        long totalAllocated = Profiler.GetTotalAllocatedMemoryLong();
        long textureMemory = Profiler.GetAllocatedMemoryForGraphicsDriver();

        Debug.Log($"Total Memory: {totalAllocated / (1024 * 1024)}MB");
        Debug.Log($"Texture Memory: {textureMemory / (1024 * 1024)}MB");
    }
}
```

### Lua GC优化

```lua
-- Lua GC优化（适用于xLua/ToLua项目）

-- 1. 避免频繁创建table
-- 差
function Update()
    local data = {x=1, y=2}  -- 每帧创建table
end

-- 好：复用table
local tempData = {}
function Update()
    tempData.x = 1
    tempData.y = 2
end

-- 2. 手动控制GC频率
collectgarbage("setpause", 100)       -- GC暂停阈值（默认200）
collectgarbage("setstepmul", 300)     -- GC步进倍率（默认200）
collectgarbage("step", 100)           -- 手动步进100KB

-- 3. 在合适的时机手动GC
-- 在场景切换、加载界面等不敏感时段触发
function OnSceneLoadComplete()
    collectgarbage("collect")
end
```

### 内存泄漏检测

```csharp
/// <summary>
/// 内存快照对比工具
/// 帮助定位内存泄漏
/// </summary>
public class MemoryLeakDetector : MonoBehaviour
{
    private long lastMonoHeap;
    private long lastTotalAllocated;

    void Update()
    {
        if (!Input.GetKeyDown(KeyCode.M)) return;

        long currentMonoHeap = Profiler.GetMonoUsedSizeLong();
        long currentTotal = Profiler.GetTotalAllocatedMemoryLong();

        Debug.Log("=== Memory Snapshot ===");
        Debug.Log($"Mono Heap: {currentMonoHeap / 1024}KB " +
            $"(delta: {(currentMonoHeap - lastMonoHeap) / 1024}KB)");
        Debug.Log($"Total Alloc: {currentTotal / (1024 * 1024)}MB " +
            $"(delta: {(currentTotal - lastTotalAllocated) / (1024 * 1024)}MB)");
        Debug.Log($"GC Total: {System.GC.GetTotalMemory(false) / 1024}KB");

        // 资源统计
        Debug.Log($"Textures: {Resources.FindObjectsOfTypeAll<Texture2D>().Length}");
        Debug.Log($"Meshes: {Resources.FindObjectsOfTypeAll<Mesh>().Length}");
        Debug.Log($"Materials: {Resources.FindObjectsOfTypeAll<Material>().Length}");
        Debug.Log($"AudioClips: {Resources.FindObjectsOfTypeAll<AudioClip>().Length}");

        lastMonoHeap = currentMonoHeap;
        lastTotalAllocated = currentTotal;
    }
}
```

## 最佳实践

- 使用Profiler的Memory模块定期检查堆内存增长趋势
- 为高频创建的对象（子弹、特效、UI元素、伤害数字）建立对象池
- 使用Addressables管理资源加载和卸载，确保引用计数正确
- 纹理使用合适的压缩格式：ASTC（iOS/Android通用）、ETC2（Android）
- 设置纹理Streaming Mipmap，按需加载分辨率
- Release版本中定期打印内存快照对比排查泄漏
- 在加载界面等非敏感时段手动触发GC
- iOS上监控内存使用，接近系统阈值时降低纹理质量

## 常见陷阱与修复

**陷阱1：资源加载后未卸载导致内存持续增长**
- 症状：每次切换场景内存增加50-100MB
- 修复：使用Addressables的引用计数管理，场景切换时释放不再需要的资源

**陷阱2：协程中使用`new WaitForSeconds()`每帧产生GC**
- 症状：Profiler中WaitForSeconds有GC Alloc
- 修复：使用static readonly缓存常用间隔的WaitForSeconds实例

**陷阱3：事件未取消订阅导致对象无法被GC回收**
- 症状：关闭界面后内存未释放
- 修复：在OnDestroy/OnDisable中取消所有事件订阅

**陷阱4：Lua与C#交互时产生大量临时对象**
- 症状：Lua调用C#方法频繁触发GC
- 修复：减少Lua到C#的调用频率，批量传递数据

**陷阱5：Dictionary的foreach循环产生Enumerator装箱**
- 症状：Profiler显示Enumerator有GC Alloc
- 修复：使用for循环遍历缓存的key列表
