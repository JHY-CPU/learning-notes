# 渲染优化策略

## 核心概念

渲染管线中的每个阶段都有优化空间，核心目标是减少GPU工作量（计算）和带宽消耗（数据传输）。移动端GPU尤其受限于带宽和填充率，而非计算能力。理解渲染管线的瓶颈所在是选择正确优化策略的前提。

### GPU渲染管线瓶颈分析

```
CPU → GPU Command Buffer → Vertex Processing → Rasterization → Fragment Processing → Output Merger
  ↓ 瓶颈           ↓ 瓶颈               ↓ 瓶颈              ↓ 瓶颈(最常见)
Draw Call过多    顶点数过多         三角形过小(像素少于shader开销)   Overdraw/Shader复杂
```

最常见的GPU瓶颈：
1. **填充率(Fill Rate)**：每秒能写入的像素数。半透明叠加、后处理效果大量消耗填充率
2. **带宽(Bandwidth)**：GPU与显存之间的数据传输。读写RenderTarget、纹理采样消耗带宽
3. **着色器计算**：复杂的Fragment Shader每像素执行大量运算

### Overdraw问题详解

Overdraw指同一像素被多次绘制，常见于：
- 半透明物体叠加（粒子特效、UI、水面）- 最主要的Overdraw来源
- 不透明物体由近到远绘制时的重复覆盖（被Early-Z解决）
- 早期深度测试未生效导致的无效绘制

移动端Overdraw开销巨大，因为GPU没有足够的ROP（Render Output Pipeline）并行处理。

### Early-Z测试

GPU在片元着色器之前进行深度测试。如果像素已被更近的物体覆盖则跳过着色计算：
- 需要由近到远绘制不透明物体（Unity默认排序）
- 不在片元着色器中修改深度值（会禁用Early-Z）
- Alpha Test/Clip会禁用Early-Z（使用Alpha To Coverage替代）

## 具体实现方法

### 动态分辨率缩放

```csharp
using UnityEngine;

/// <summary>
/// 动态分辨率控制器
/// 根据帧时间自动调整渲染分辨率，保持帧率稳定
/// </summary>
public class DynamicResolutionController : MonoBehaviour
{
    [Header("目标")]
    [SerializeField] private float targetFrameTime = 16.67f; // 60fps
    [SerializeField] private float tolerancePercent = 10f;

    [Header("缩放范围")]
    [SerializeField] private float minScale = 0.5f;
    [SerializeField] private float maxScale = 1.0f;

    [Header("调整速度")]
    [SerializeField] private float decreaseRate = 0.05f; // 每帧最大降分辨率幅度
    [SerializeField] private float increaseRate = 0.02f; // 每帧最大升分辨率幅度

    [Header("平滑")]
    [SerializeField] private int averageFrames = 10;

    private float currentScale = 1f;
    private float[] frameTimeBuffer;
    private int bufferIndex;

    void Start()
    {
        frameTimeBuffer = new float[averageFrames];
        for (int i = 0; i < averageFrames; i++)
            frameTimeBuffer[i] = targetFrameTime;
    }

    void Update()
    {
        // 记录帧时间
        frameTimeBuffer[bufferIndex] = Time.unscaledDeltaTime * 1000f;
        bufferIndex = (bufferIndex + 1) % frameTimeBuffer.Length;

        // 计算平均帧时间
        float avgFrameTime = 0;
        for (int i = 0; i < frameTimeBuffer.Length; i++)
            avgFrameTime += frameTimeBuffer[i];
        avgFrameTime /= frameTimeBuffer.Length;

        // 调整分辨率
        float upperBound = targetFrameTime * (1f + tolerancePercent / 100f);
        float lowerBound = targetFrameTime * (1f - tolerancePercent / 100f);

        if (avgFrameTime > upperBound)
        {
            // 帧时间超标，降低分辨率
            currentScale = Mathf.Max(minScale, currentScale - decreaseRate);
        }
        else if (avgFrameTime < lowerBound)
        {
            // 帧时间充裕，提高分辨率
            currentScale = Mathf.Min(maxScale, currentScale + increaseRate);
        }

        ScalableBufferManager.ResizeBuffers(currentScale, currentScale);
    }

    public float CurrentScale => currentScale;
    public float EffectiveResolution =>
        currentScale * currentScale * Screen.width * Screen.height;
}
```

### RenderTexture复用池

```csharp
/// <summary>
/// RenderTexture复用管理器
/// 避免频繁创建/销毁RT导致的内存碎片和性能抖动
/// </summary>
public class RTManager
{
    private static Dictionary<int, RenderTexture> pool
        = new Dictionary<int, RenderTexture>();
    private static List<RenderTexture> allRTs = new List<RenderTexture>();

    /// <summary>
    /// 获取RenderTexture（自动复用）
    /// </summary>
    public static RenderTexture Get(int width, int height, int depth,
        RenderTextureFormat format = RenderTextureFormat.Default)
    {
        int key = GetKey(width, height, depth, format);

        if (pool.TryGetValue(key, out var rt) && rt != null && rt.IsCreated())
            return rt;

        rt = new RenderTexture(width, height, depth, format);
        rt.Create();
        pool[key] = rt;
        allRTs.Add(rt);
        return rt;
    }

    /// <summary>
    /// 获取临时RT（使用后自动回收）
    /// </summary>
    public static RenderTexture GetTemporary(int width, int height,
        int depth = 0, RenderTextureFormat format = RenderTextureFormat.Default)
    {
        return RenderTexture.GetTemporary(width, height, depth, format);
    }

    public static void ReleaseTemporary(RenderTexture rt)
    {
        RenderTexture.ReleaseTemporary(rt);
    }

    public static void ReleaseAll()
    {
        foreach (var rt in allRTs)
            if (rt != null) rt.Release();
        pool.Clear();
        allRTs.Clear();
    }

    static int GetKey(int w, int h, int d, RenderTextureFormat f)
    {
        return w * 1000000 + h * 1000 + d + (int)f;
    }
}
```

### 减少Overdraw的策略

```csharp
/// <summary>
/// Overdraw优化完整方案
/// </summary>
public class OverdrawOptimizer
{
    // === 策略1: 粒子特效优化 ===
    // - 使用更少的粒子（200个降到50个）
    // - 缩小粒子发射面积和运动范围
    // - 关闭不需要的粒子碰撞模块
    // - 使用Mesh Particle代替Billboard减少重叠
    // - 粒子Shader使用最简化的着色器

    // === 策略2: UI Overdraw控制 ===
    // - 检查UI中不可见但仍参与渲染的Image
    //   （alpha=0或color.a=0的Image仍然产生Overdraw）
    // - 使用RectMask2D代替Mask（不产生额外Draw Call）
    // - 减少全屏UI叠加层数

    // === 策略3: 半透明物体管理 ===
    // - 减少半透明物体的数量和屏幕覆盖面积
    // - 使用不透明替代方案（dither代替alpha blend）
    // - 将远处的半透明物体变为不透明

    // === 策略4: 后处理效果优化 ===
    // - 使用更小的中间RT（1/2或1/4分辨率）
    // - 合并多个后处理Pass
    // - 在低端设备上禁用代价高的后处理

    // === 策略5: 使用Overdraw可视化检测 ===
    // Unity: Scene视图 > Draw Mode > Overdraw
    // UE: visualizemodes overdraw
}
```

### Shader LOD分级

```hlsl
// 根据硬件能力使用不同复杂度的Shader
Shader "Custom/LODShader"
{
    // 高端Shader (PC/主机)
    SubShader
    {
        LOD 400
        Tags { "RenderType"="Opaque" }
        Pass
        {
            CGPROGRAM
            #pragma vertex vert
            #pragma fragment frag
            // PBR光照 + 法线贴图 + 环境反射 + 高光
            float4 frag(v2f i) : SV_Target
            {
                // 完整PBR计算
                // 纹理采样5-8次
                // 光照计算：主光源 + 补光源 + 环境光
            }
            ENDCG
        }
    }

    // 中端Shader (高端移动设备)
    SubShader
    {
        LOD 200
        Tags { "RenderType"="Opaque" }
        Pass
        {
            CGPROGRAM
            #pragma vertex vert
            #pragma fragment frag
            // 简化PBR，无环境反射
            float4 frag(v2f i) : SV_Target
            {
                // 简化光照计算
                // 纹理采样2-3次
                // 仅主光源
            }
            ENDCG
        }
    }

    // 低端Shader (低端移动设备)
    SubShader
    {
        LOD 100
        Tags { "RenderType"="Opaque" }
        Pass
        {
            CGPROGRAM
            #pragma vertex vert
            #pragma fragment frag
            // 基础光照，无纹理
            float4 frag(v2f i) : SV_Target
            {
                // 最简光照
                // 纹理采样1次
                // 无高光
            }
            ENDCG
        }
    }
}

// C#端控制全局Shader LOD
Shader.globalMaximumLOD = 200; // 中端设备
// 或逐Shader设置
material.shader.maximumLOD = 200;
```

### 阴影优化

```csharp
/// <summary>
/// 阴影系统优化配置
/// 阴影是渲染开销的主要来源之一
/// </summary>
public class ShadowOptimizer : MonoBehaviour
{
    void ConfigureShadows()
    {
        // 阴影距离：超出距离的物体不投射阴影
        QualitySettings.shadowDistance = 50f; // 默认150，可大幅降低

        // 级联阴影（Cascaded Shadow Maps）
        // 级联数越少性能越好但近处阴影精度越低
        QualitySettings.shadowCascades = 2; // 默认4，移动端建议2
        QualitySettings.shadowCascade2Split = 0.33f;
        QualitySettings.shadowCascade4Split = new Vector3(0.067f, 0.2f, 0.467f);

        // 阴影分辨率
        QualitySettings.shadowResolution = ShadowResolution.Medium;

        // 阴影偏移：减少Shadow Acne
        QualitySettings.shadowProjection = ShadowProjection.CloseFit;
    }

    /// <summary>
    /// 动态阴影LOD：远处物体使用更低的阴影分辨率
    /// </summary>
    void DynamicShadowLOD()
    {
        float frameTime = Time.unscaledDeltaTime * 1000f;
        if (frameTime > 20f) // 帧率低于50fps
        {
            QualitySettings.shadowDistance = Mathf.Max(20f,
                QualitySettings.shadowDistance - 5f);
        }
    }
}
```

## 性能基准数据

| 优化手段 | 优化前 | 优化后 | GPU时间节省 |
|---------|--------|--------|-----------|
| 动态分辨率(0.7x) | 16ms | 9ms | 44% |
| 阴影距离150→50 | 5ms | 2ms | 60% |
| 级联阴影4→2 | 4ms | 2.5ms | 38% |
| 关闭Bloom后处理 | 3ms | 0ms | 100% |
| Shader LOD 400→200 | 8ms | 4ms | 50% |
| 半透明面积减半 | 6ms | 3ms | 50% |

## 最佳实践

- 不透明物体由近到远排序以最大化Early-Z效率（Unity默认）
- 减少半透明物体数量和屏幕覆盖面积以降低Overdraw
- 对中低端设备启用动态分辨率缩放
- 复用RenderTexture而非每帧创建销毁
- 使用Shader LOD为不同设备提供不同画质
- 合理设置阴影距离和级联数（移动端阴影距离<50m，级联数<=2）
- 后处理使用1/2分辨率RT
- 使用Frame Debugger逐Pass分析渲染开销

## 常见陷阱与修复

**陷阱1：后处理Pass过多导致带宽瓶颈**
- 症状：开启Bloom+ColorGrading+Vignette后帧率掉20fps
- 修复：合并Pass，降低中间RT分辨率，低端设备禁用部分效果

**陷阱2：全屏后处理在移动端性能开销巨大**
- 症状：后处理占GPU时间的50%
- 修复：使用1/2分辨率后处理，在Shader中合并多个效果

**陷阱3：透明UI层叠过多导致Overdraw严重**
- 症状：4层半透明UI叠加导致填充率瓶颈
- 修复：减少UI半透明层数，不透明背景用不透明Image

**陷阱4：使用GrabPass抓取屏幕**
- 症状：GrabPass是移动端最贵的操作之一
- 修复：使用CommandBuffer.Blit或自定义RT替代

**陷阱5：阴影贴图分辨率设置过高**
- 症状：4096x4096阴影贴图在移动端卡顿
- 修复：移动端使用1024x1024或2048x2048
