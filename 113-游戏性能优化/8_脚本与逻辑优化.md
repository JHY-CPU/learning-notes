# 脚本与逻辑优化

## 核心概念

游戏脚本的优化重点是减少不必要的计算、缓存查找结果和避免GC分配。虽然单个C#操作的开销很小（纳秒级），但在Update中被每帧调用数千次时，累积开销可达数毫秒。

### Unity脚本生命周期开销

```
MonoBehaviour回调调用链:
Unity内部 → Find GameObjects with MonoBehaviour
         → Check if enabled
         → 虚方法调用(Update/FixedUpdate/LateUpdate)
         → 用户代码执行

每个MonoBehaviour的Update回调开销约: 0.001-0.01ms
1000个MonoBehaviour的Update开销: 1-10ms
```

减少MonoBehaviour数量本身就是优化。

### 缓存组件引用的重要性

`GetComponent<T>()`的内部实现涉及遍历GameObject上的组件列表，开销约0.01-0.1ms。在Update中每帧调用将浪费大量CPU：

```csharp
// 差：每帧调用GetComponent（0.05ms/call × 60fps = 3ms/秒）
void Update()
{
    GetComponent<Rigidbody>().AddForce(Vector3.up);
}

// 好：缓存引用（0.0001ms/call）
private Rigidbody rb;
void Awake() { rb = GetComponent<Rigidbody>(); }
void Update() { rb.AddForce(Vector3.up); }
```

## 具体实现方法

### 零GC协程方案（完整版）

```csharp
using System.Collections;
using System.Collections.Generic;
using UnityEngine;

/// <summary>
/// 零GC协程和计时器系统
/// 替代传统协程中的WaitForSeconds和InvokeRepeating
/// </summary>
public class ZeroGCTimerSystem : MonoBehaviour
{
    // === 方法1: 缓存WaitForSeconds ===
    private static readonly WaitForSeconds wait1s = new WaitForSeconds(1f);
    private static readonly WaitForSeconds wait05s = new WaitForSeconds(0.5f);
    private static readonly WaitForSeconds wait01s = new WaitForSeconds(0.1f);

    IEnumerator GoodCoroutine()
    {
        while (true)
        {
            UpdateHP();
            yield return wait1s; // 零GC复用
        }
    }

    // === 方法2: 自定义计时器（零GC，更灵活）===
    private List<ZeroGCTimer> timers = new List<ZeroGCTimer>();

    public ZeroGCTimer CreateTimer(float interval, System.Action callback)
    {
        var timer = new ZeroGCTimer(interval, callback);
        timers.Add(timer);
        return timer;
    }

    void Update()
    {
        float dt = Time.deltaTime;
        for (int i = timers.Count - 1; i >= 0; i--)
            timers[i].Tick(dt);
    }

    void UpdateHP() { }
}

/// <summary>
/// 零GC计时器
/// 不产生任何堆分配
/// </summary>
public class ZeroGCTimer
{
    private float interval;
    private float elapsed;
    private System.Action callback;
    private bool isRunning;

    public bool IsRunning => isRunning;

    public ZeroGCTimer(float interval, System.Action callback)
    {
        this.interval = interval;
        this.callback = callback;
        this.isRunning = true;
    }

    public void Tick(float deltaTime)
    {
        if (!isRunning) return;

        elapsed += deltaTime;
        if (elapsed >= interval)
        {
            elapsed -= interval; // 保持余量，不丢失精度
            callback?.Invoke();
        }
    }

    public void Start() => isRunning = true;
    public void Stop() => isRunning = false;
    public void Reset() => elapsed = 0f;
    public void SetInterval(float newInterval) => interval = newInterval;
}
```

### 字符串优化（完整版）

```csharp
using System.Text;
using UnityEngine;

/// <summary>
/// 字符串操作优化
/// 字符串拼接是游戏中最常见的GC来源之一
/// </summary>
public class StringOptimization : MonoBehaviour
{
    // === 方案1: StringBuilder复用 ===
    private StringBuilder sb = new StringBuilder(64);

    void UpdateScore_StringBuilder(int score)
    {
        sb.Clear();
        sb.Append("Score: ").Append(score);
        scoreText.text = sb.ToString();
    }

    // === 方案2: 预格式化 + 仅在变化时更新 ===
    private int lastScore = -1;

    void UpdateScore_ValueChangedOnly(int score)
    {
        if (score != lastScore)
        {
            scoreText.text = $"Score: {score}"; // C# 6 字符串插值
            lastScore = score;
        }
    }

    // === 方案3: 字符串缓存（适合固定文本）===
    private static readonly Dictionary<int, string> scoreCache
        = new Dictionary<int, string>();

    string GetCachedScoreString(int score)
    {
        if (!scoreCache.TryGetValue(score, out string str))
        {
            str = $"Score: {score}";
            scoreCache[score] = str;
        }
        return str;
    }

    // === 方案4: char数组拼接（最高性能）===
    private char[] scoreBuffer = new char[32];

    void UpdateScore_CharBuffer(int score)
    {
        int index = 0;
        // "Score: "
        scoreBuffer[index++] = 'S';
        scoreBuffer[index++] = 'c';
        scoreBuffer[index++] = 'o';
        scoreBuffer[index++] = 'r';
        scoreBuffer[index++] = 'e';
        scoreBuffer[index++] = ':';
        scoreBuffer[index++] = ' ';

        // 数字转字符
        if (score == 0)
        {
            scoreBuffer[index++] = '0';
        }
        else
        {
            int temp = score;
            int digitStart = index;
            while (temp > 0)
            {
                scoreBuffer[index++] = (char)('0' + temp % 10);
                temp /= 10;
            }
            // 反转数字部分
            System.Array.Reverse(scoreBuffer, digitStart, index - digitStart);
        }

        scoreText.SetCharArray(scoreBuffer, 0, index); // TMP支持
    }

    UnityEngine.UI.Text scoreText;
}
```

### 高效的组件查找模式

```csharp
using UnityEngine;

/// <summary>
/// 泛型基类自动缓存常用组件
/// 子类无需重复GetComponent调用
/// </summary>
public class EntityBase : MonoBehaviour
{
    // 缓存最常用的组件
    protected Transform cachedTransform;
    protected Rigidbody cachedRigidbody;
    protected Collider cachedCollider;
    protected Animator cachedAnimator;
    protected AudioSource cachedAudioSource;

    protected virtual void Awake()
    {
        cachedTransform = transform; // transform是属性调用，缓存更快
        cachedRigidbody = GetComponent<Rigidbody>();
        cachedCollider = GetComponent<Collider>();
        cachedAnimator = GetComponent<Animator>();
        cachedAudioSource = GetComponent<AudioSource>();
    }

    // 提供便捷访问
    public Transform CachedTransform => cachedTransform;
    public Rigidbody CachedRigidbody => cachedRigidbody;
}

/// <summary>
/// 子类示例
/// </summary>
public class Player : EntityBase
{
    private CharacterController characterController;

    protected override void Awake()
    {
        base.Awake(); // 自动缓存基类组件
        characterController = GetComponent<CharacterController>(); // 仅查找Player特有的
    }
}
```

### 避免foreach和LINQ的GC

```csharp
using System.Collections.Generic;
using UnityEngine;

/// <summary>
/// foreach和LINQ的GC分配详解
/// </summary>
public class ForeachAndLINQOptimization
{
    private List<Enemy> enemies = new List<Enemy>();
    private Dictionary<int, ItemData> itemDict = new Dictionary<int, ItemData>();

    // === List<T>.foreach ===
    // C# 7.3+ 的List<T>.Enumerator是ref struct，不会装箱（零GC）
    // 但旧版本或IL2CPP可能仍有GC
    // 最安全的做法：使用for循环
    void Safe_ForLoop()
    {
        for (int i = 0; i < enemies.Count; i++)
            enemies[i].Update();
    }

    // === Dictionary.foreach ===
    // Dictionary<K,V>.Enumerator是struct，foreach会产生装箱GC
    void Bad_DictionaryForeach()
    {
        foreach (var kvp in itemDict) // 装箱！
            Process(kvp.Value);
    }

    void Good_DictionaryFor()
    {
        // 使用Keys列表遍历
        var keys = new List<int>(itemDict.Keys); // 需要预分配
        for (int i = 0; i < keys.Count; i++)
            Process(itemDict[keys[i]]);
    }

    // === LINQ的GC问题 ===
    // LINQ产生闭包、迭代器、委托等堆分配
    void Bad_LINQ()
    {
        // 以下每行都产生GC分配
        var alive = enemies.Where(e => e.IsAlive);       // 闭包+迭代器
        var sorted = enemies.OrderBy(e => e.Distance);   // 排序+闭包
        var first = enemies.FirstOrDefault(e => e.IsBoss); // 闭包
    }

    void Good_ManualLoop()
    {
        Enemy closestBoss = null;
        float closestDist = float.MaxValue;
        for (int i = 0; i < enemies.Count; i++)
        {
            if (enemies[i].IsBoss && enemies[i].Distance < closestDist)
            {
                closestBoss = enemies[i];
                closestDist = enemies[i].Distance;
            }
        }
    }

    void Process(ItemData d) { }

    class Enemy
    {
        public bool IsAlive;
        public bool IsBoss;
        public float Distance;
        public void Update() { }
    }
    class ItemData { }
}
```

## 最佳实践

- 所有`GetComponent`和`Find`调用结果在Awake/Start中缓存
- 使用静态缓存的`WaitForSeconds`实例避免协程GC
- Animator/Shader属性名使用`StringToHash`/`PropertyToID`预转换为int
- 字符串操作使用StringBuilder或仅在值变化时更新
- 避免在Update中使用LINQ（会产生闭包和迭代器分配）
- 使用结构体代替小类以减少堆分配和GC压力
- 减少MonoBehaviour数量，合并相关逻辑到同一脚本
- 使用CompareTag代替`== "Player"`（更快且不产生GC）

## 常见陷阱与修复

**陷阱1：每帧调用`GameObject.Find()`查找对象**
- 症状：Find遍历场景中所有GameObject，开销0.1-1ms
- 修复：在Start中缓存引用，或使用单例/依赖注入

**陷阱2：`new WaitForSeconds()`在协程中反复调用**
- 症状：每次创建WaitForSeconds都有GC分配
- 修复：使用static readonly缓存常用间隔

**陷阱3：Update中使用字符串属性名设置Animator参数**
- 症状：每次SetFloat/SetTrigger都有字符串哈希计算
- 修复：使用Animator.StringToHash预转换

**陷阱4：使用`SendMessage`/`BroadcastMessage`**
- 症状：反射调用，开销是直接调用的10-50倍
- 修复：使用接口或直接方法调用

**陷阱5：在热路径中使用`ToString()`拼接字符串**
- 症状：每次ToString和+操作都分配新字符串
- 修复：使用StringBuilder或预缓存

**陷阱6：使用Lambda表达式捕获局部变量导致闭包分配**
- 症状：看似简单的回调有GC
- 修复：使用for循环代替，或将捕获变量提升为类字段
