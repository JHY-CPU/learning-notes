# 物理优化

## 核心概念

物理引擎（PhysX/Havok/Box2D）是游戏CPU开销的重要组成部分，尤其在有大量物理对象的场景中（如射击游戏的弹道、破坏系统、布娃娃系统）。合理的配置可以将物理开销降低50-90%。

### 物理引擎工作流程

```
每FixedUpdate周期:
1. 碰撞检测阶段
   - Broad Phase（粗略检测）：空间分区排除不可能碰撞的物体对
   - Narrow Phase（精细检测）：精确几何检测确定碰撞
2. 碰撞解决阶段
   - 生成碰撞约束
   - 迭代求解（通常4-8次迭代）
3. 位置/速度积分
   - 更新Rigidbody的位置和旋转
4. 触发器事件分发
```

### 碰撞层矩阵（Collision Layer Matrix）

通过设置不同层级之间的碰撞关系，避免不必要的碰撞检测。这是物理优化中最有效且最简单的手段：

```
         Player  Enemy  Bullet  Trigger  Terrain
Player      -      X       X       O        O
Enemy       X      -       O       X        O
Bullet      X      O       -       X        X
Trigger     O      X       X       -        X
Terrain     O      O       X       X        -

O = 产生碰撞检测, X = 忽略碰撞
```

通过禁用不需要的碰撞对（如Bullet-Bullet），可以将碰撞检测次数从N^2/2大幅降低。

### 休眠机制（Sleep）

当Rigidbody的速度低于阈值持续一段时间后，物理引擎将其标记为"休眠"：
- 休眠物体不参与物理模拟（不计算碰撞、不更新位置）
- 受到外力或碰撞时自动唤醒
- 适合落地静止的道具、放置的家具

### Fixed Timestep

物理更新频率与渲染帧率解耦：
- `Fixed Timestep`（默认0.02s = 50Hz）控制物理更新频率
- 值越大物理越粗糙但CPU开销越小
- 值越小物理越精确但CPU开销越大
- 推荐范围：0.01-0.04（25-100Hz）

## 具体实现方法

### 碰撞层配置（完整方案）

```csharp
/// <summary>
/// 碰撞层管理器
/// 统一管理碰撞层矩阵配置
/// </summary>
public class CollisionLayerManager : MonoBehaviour
{
    // 推荐的层级命名和碰撞矩阵
    // 在Edit > Project Settings > Physics > Layer Collision Matrix中配置
    // 或使用代码配置：

    void ConfigureCollisionMatrix()
    {
        // 定义层级
        int player = LayerMask.NameToLayer("Player");
        int enemy = LayerMask.NameToLayer("Enemy");
        int bullet = LayerMask.NameToLayer("Bullet");
        int trigger = LayerMask.NameToLayer("Trigger");
        int terrain = LayerMask.NameToLayer("Terrain");
        int decoration = LayerMask.NameToLayer("Decoration");

        // 禁用不需要的碰撞对
        Physics.IgnoreLayerCollision(bullet, bullet, true);    // 子弹之间不碰撞
        Physics.IgnoreLayerCollision(bullet, trigger, true);   // 子弹不与触发器碰撞
        Physics.IgnoreLayerCollision(player, bullet, false);   // 子弹击中玩家
        Physics.IgnoreLayerCollision(enemy, bullet, false);    // 子弹击中敌人
        Physics.IgnoreLayerCollision(player, player, true);    // 玩家之间不碰撞
        Physics.IgnoreLayerCollision(player, enemy, true);     // 玩家和敌人不碰撞（需要时再启用）
        Physics.IgnoreLayerCollision(decoration, bullet, true);// 装饰品不与子弹碰撞
        Physics.IgnoreLayerCollision(decoration, player, true);// 装饰品不与玩家碰撞

        // 效果：碰撞检测对从约 N*(N-1)/2 降低到 N*3-5
    }
}
```

### 简化碰撞体策略

```csharp
/// <summary>
/// LOD碰撞体切换
/// 远距离使用简单碰撞体，近距离使用精确碰撞体
/// </summary>
public class LODCollider : MonoBehaviour
{
    [System.Serializable]
    public class ColliderLOD
    {
        public Collider collider;
        public float maxDistance;
    }

    [SerializeField] private ColliderLOD[] colliderLevels;
    [SerializeField] private Camera targetCamera;

    void Update()
    {
        if (targetCamera == null) return;

        float dist = Vector3.Distance(
            targetCamera.transform.position, transform.position);

        for (int i = 0; i < colliderLevels.Length; i++)
        {
            bool shouldEnable = dist <= colliderLevels[i].maxDistance;
            colliderLevels[i].collider.enabled = shouldEnable;
        }
    }

    // 碰撞体性能排序（快→慢）:
    // SphereCollider > BoxCollider > CapsuleCollider > MeshCollider
    // SphereCollider: 最简单，O(1)距离检测
    // BoxCollider: AABB + OBB检测
    // CapsuleCollider: 球体+圆柱组合
    // MeshCollider: 逐三角形检测，性能最差
}
```

### Fixed Timestep智能调整

```csharp
/// <summary>
/// 智能物理步长调整
/// 根据帧率和场景物理复杂度动态调整
/// </summary>
public class PhysicsTimestepManager : MonoBehaviour
{
    [Header("配置")]
    [SerializeField] private float defaultFixedTimestep = 0.02f; // 50Hz
    [SerializeField] private float minFixedTimestep = 0.01f;     // 100Hz
    [SerializeField] private float maxFixedTimestep = 0.04f;     // 25Hz
    [SerializeField] private float criticalFrameTime = 33.33f;   // 30fps以下

    void Update()
    {
        float frameTime = Time.unscaledDeltaTime * 1000f;

        if (frameTime > criticalFrameTime)
        {
            // 帧率太低，降低物理频率以节省CPU
            Time.fixedDeltaTime = Mathf.Min(
                maxFixedTimestep,
                Time.fixedDeltaTime + 0.002f);
        }
        else if (frameTime < 14f) // 高于70fps
        {
            // 帧率充足，可以提高物理精度
            Time.fixedDeltaTime = Mathf.Max(
                minFixedTimestep,
                Time.fixedDeltaTime - 0.001f);
        }
    }

    // 预设方案：
    // 射击游戏(精确命中判定): 0.01s (100Hz)
    // RPG/动作游戏: 0.02s (50Hz) - 默认值
    // 策略/模拟游戏: 0.03-0.04s (25-33Hz)
    // 物理不重要的游戏: 0.05s (20Hz)
}
```

### 射线检测优化

```csharp
/// <summary>
/// 射线检测优化技巧
/// </summary>
public class RaycastOptimization
{
    // === 1. 使用LayerMask过滤 ===
    // 差：检测所有碰撞体
    void Bad_Raycast(Ray ray)
    {
        Physics.Raycast(ray, out RaycastHit hit, 100f); // 检测所有层
    }

    // 好：只检测需要的层
    void Good_Raycast(Ray ray)
    {
        int layerMask = LayerMask.GetMask("Enemy", "Terrain");
        Physics.Raycast(ray, out RaycastHit hit, 100f, layerMask);
    }

    // === 2. 使用NonAlloc版本 ===
    // 差：每次产生GC分配
    void Bad_RaycastAll(Ray ray)
    {
        RaycastHit[] hits = Physics.RaycastAll(ray, 100f); // 分配数组
    }

    // 好：预分配缓冲区
    private RaycastHit[] hitBuffer = new RaycastHit[20];
    void Good_RaycastAll(Ray ray)
    {
        int hitCount = Physics.RaycastNonAlloc(ray, hitBuffer, 100f);
        for (int i = 0; i < hitCount; i++)
            ProcessHit(hitBuffer[i]);
    }

    // === 3. SphereCast vs RayCast ===
    // SphereCast检测范围更可控，适合子弹、拾取检测
    void SphereCast(Ray ray, float radius)
    {
        Physics.SphereCast(ray, radius, out RaycastHit hit, 100f);
    }

    // === 4. 空间分区减少检测对象 ===
    // 使用QuadTree/OctTree预先排除不可能命中的物体
    void SpatialPartitionRaycast(Ray ray)
    {
        // 只对空间分区返回的候选物体进行射线检测
        // 而不是场景中所有碰撞体
    }

    void ProcessHit(RaycastHit hit) { }
}
```

### 物理性能监控

```csharp
/// <summary>
/// 物理系统性能监控
/// </summary>
public class PhysicsMonitor : MonoBehaviour
{
    private int lastBodyCount;
    private float monitorInterval = 1f;
    private float timer;

    void Update()
    {
        timer += Time.deltaTime;
        if (timer < monitorInterval) return;
        timer = 0;

        int bodies = FindObjectsOfType<Rigidbody>().Length;
        int kinematic = 0, sleeping = 0;
        foreach (var rb in FindObjectsOfType<Rigidbody>())
        {
            if (rb.isKinematic) kinematic++;
            if (rb.IsSleeping()) sleeping++;
        }

        int colliders = FindObjectsOfType<Collider>().Length;
        int meshColliders = FindObjectsOfType<MeshCollider>().Length;

        Debug.Log($"=== Physics Stats ===\n" +
            $"Bodies: {bodies} (Kinematic: {kinematic}, Sleeping: {sleeping})\n" +
            $"Colliders: {colliders} (Mesh: {meshColliders})\n" +
            $"Fixed Timestep: {Time.fixedDeltaTime * 1000f:F1}ms");
    }
}
```

## 性能基准数据

| 物理对象数量 | 默认配置 | 优化后 | CPU节省 |
|------------|---------|--------|--------|
| 100个Rigidbody | 0.5ms | 0.2ms | 60% |
| 500个Rigidbody | 3ms | 0.8ms | 73% |
| 1000个碰撞体(碰撞矩阵优化) | 5ms | 1.5ms | 70% |
| MeshCollider vs BoxCollider | 2ms/collider | 0.1ms/collider | 95% |
| 射线检测(全场景 vs LayerMask) | 0.5ms | 0.05ms | 90% |

## 最佳实践

- 使用碰撞层矩阵禁用不需要的碰撞检测对（最简单有效的优化）
- 对静止物体设置`isKinematic = true`或移除Rigidbody
- 尽量使用基本碰撞体（Box/Sphere/Capsule），避免MeshCollider
- 射线检测使用LayerMask过滤和NonAlloc版本
- 合理设置Fixed Timestep，不需要高精度物理的游戏降低频率
- 射击游戏的命中判定使用SphereCast代替RayCast
- 复杂物理场景考虑使用Job System并行计算
- 监控休眠物体比例，确保静止物体确实进入了休眠

## 常见陷阱与修复

**陷阱1：碰撞层矩阵未配置导致所有物体两两检测**
- 症状：场景有500个物体，Profiler中物理耗时>10ms
- 修复：配置碰撞矩阵，禁用不需要的碰撞对

**陷阱2：MeshCollider使用过多导致物理计算成为瓶颈**
- 症状：复杂Mesh的MeshCollider每帧耗时数毫秒
- 修复：用简单的BoxCollider近似代替MeshCollider

**陷阱3：Fixed Timestep设置过小导致物理更新过于频繁**
- 症状：FixedUpdate被调用多次消耗CPU
- 修复：增大Fixed Timestep到0.02-0.04

**陷阱4：对静止物体使用非Kinematic Rigidbody**
- 症状：不必要的物理模拟计算
- 修复：设置isKinematic=true或直接移除Rigidbody

**陷阱5：射线检测未限制距离和层级**
- 症状：每帧检测场景中所有碰撞体
- 修复：设置maxDistance和LayerMask，使用NonAlloc版本
