# 透明与混合

## 1. 核心理论

透明效果通过Alpha混合（Alpha Blending）实现，将新片元颜色与已有帧缓冲颜色按Alpha值加权混合。正确的透明渲染需要严格的渲染顺序管理，是实时渲染中最具挑战性的问题之一。

### 1.1 Alpha Blend数学原理

Alpha混合在输出合并阶段（Output Merger）执行，是一个线性插值操作：

```
标准Alpha混合公式：
C_result = C_src × α_src + C_dst × (1 - α_src)

其中：
  C_result = 最终写入帧缓冲的颜色
  C_src = 当前片元（源）的颜色
  α_src = 当前片元的Alpha值（不透明度）
  C_dst = 帧缓冲中已有的（目标）颜色

这等价于：C_result = lerp(C_dst, C_src, α_src)

预乘Alpha（Pre-multiplied Alpha）：
存储时：RGB已乘以A（即存储 C×A 而非 C）
混合时：C_result = C_src' + C_dst × (1 - α_src)
  其中 C_src' = C_src × α_src（已在纹理中预乘）

预乘Alpha的优势：
- 避免半透明叠加时的颜色暗化
- 更正确的滤波（Mipmap生成时颜色过渡自然）
- 支持加法混合（Additive）
```

### 1.2 常用混合模式

```
混合模式            混合公式                           用途
─────────────────────────────────────────────────────────────────
Alpha混合           SrcA × Src + (1-SrcA) × Dst       标准半透明
预乘Alpha           One × Src + (1-SrcA) × Dst        预乘纹理
加法混合            One × Src + One × Dst              发光/粒子
乘法混合            Dst × Src + Zero × Dst             阴影叠加
叠加（Overlay）     取决于Dst值                         后处理
```

```hlsl
// Unity中设置混合模式
Blend SrcAlpha OneMinusSrcAlpha      // 标准Alpha混合
Blend One OneMinusSrcAlpha           // 预乘Alpha
Blend One One                        // 加法混合
Blend DstColor Zero                  // 乘法混合
Blend SrcAlpha One                   // 软加法（发光粒子）

// 分别设置颜色和Alpha的混合因子
Blend SrcAlpha OneMinusSrcAlpha, One Zero
// 颜色：SrcAlpha × Src + (1-SrcA) × Dst
// Alpha：One × SrcA + Zero × DstA = SrcA
```

## 2. Alpha Test（Alpha裁剪）

Alpha Test完全丢弃Alpha低于阈值的片元，产生锐利边缘。

```glsl
// GLSL Alpha Test
uniform sampler2D u_MainTex;
uniform float u_Cutoff; // 阈值，通常0.5

in vec2 v_UV;
out vec4 fragColor;

void main()
{
    vec4 texColor = texture(u_MainTex, v_UV);

    if (texColor.a < u_Cutoff)
        discard; // 完全丢弃该片元

    fragColor = vec4(texColor.rgb, 1.0);
}
```

```hlsl
// HLSL Alpha Test（Unity URP）
half4 AlphaTestFrag(Varyings input) : SV_Target
{
    half4 texColor = SAMPLE_TEXTURE2D(_MainTex, sampler_MainTex, input.uv);

    // Unity内置clip函数
    clip(texColor.a - _Cutoff);
    // 等价于：if (texColor.a < _Cutoff) discard;

    return half4(texColor.rgb, 1.0);
}

// 或使用Alpha To Coverage（MSAA模式下更优）
// 在SubShader Tags中设置
// Tags { "RenderType" = "TransparentCutout" }
```

### Alpha Test的问题

```
1. 无抗锯齿：边缘硬切，像素感明显
   解决：Alpha To Coverage（利用MSAA的多重采样）

2. 破坏Early-Z优化：
   片元着色器中的discard会阻止Early-Z剔除后续片元
   这意味着被遮挡的像素仍然会执行片元着色器
   解决：使用Z-Pre Pass或分两遍渲染

3. 无法产生平滑渐变：
   只有"通过"和"丢弃"两种状态
   解决：改用Alpha Blend
```

## 3. 渲染队列与排序

### 3.1 Unity渲染队列

```
队列值    名称              用途                    排序方向
─────────────────────────────────────────────────────────
1000     Background         天空盒                  N/A
2000     Geometry           不透明物体              从前到远（Front-to-Back）
2450     AlphaTest          镂空物体                从前到远
2500     Geometry Last      不透明最后渲染          从前到远
3000     Transparent        半透明物体              从远到近（Back-to-Front）
4000     Overlay            UI叠加层                从远到近

为什么不透明从前到远？
  → 利用Early-Z：近处物体先写入深度，远处物体被深度测试剔除
  → 减少过度绘制，避免浪费片元着色器计算

为什么半透明从远到近？
  → 混合公式 C = Src × α + Dst × (1-α) 要求Dst已有正确值
  → 远处物体先混合到帧缓冲，近处物体后混合覆盖
```

### 3.2 排序问题与解决方案

```cpp
// 半透明物体排序（CPU端）
std::sort(transparentObjects.begin(), transparentObjects.end(),
    [](const Renderable& a, const Renderable& b) {
        // 从远到近排序
        return a.distanceToCamera > b.distanceToCamera;
    });
```

**排序无法解决的问题：互相穿插的几何体**

```
场景：两个半透明球体互相穿插

  视线方向 →

  ┌──A──┐
  │  ┌──B──┐
  │  │█████│  ← A的部分在B前面，B的部分在A前面
  └──┤█████├──B──┘
     └─────┘

  无论如何排序，总有一部分顺序错误。
```

**解决方案：**

```
1. 深度剥离（Depth Peeling）：
   Pass 1：渲染最近的半透明层
   Pass 2：剥离最近层，渲染次近层
   Pass 3：再次剥离...
   最后从后向前混合所有层
   优点：精确正确。缺点：N层需要N个Pass。

2. 加权混合OIT（WBOIT）：
   使用特殊的混合公式，不需要排序
   C_result = Σ(C_i × w_i) / Σ(w_i)
   其中 w_i = 基于深度的权重函数
   优点：单Pass。缺点：对厚度大的半透明不精确。

3. Per-Pixel Linked List（D3D11+）：
   每个像素维护一个链表，存储所有半透明片元
   最后逐像素排序并混合
   优点：完全正确。缺点：显存开销大，需要UAV。

4. 分层体积（Layered Volume）：
   固定层数的半透明缓冲区，每层存储颜色和深度
   最后按深度排序混合
   优点：性能可控。缺点：层数有限可能导致信息丢失。
```

## 4. 完整透明着色器示例

### 4.1 GLSL半透明着色器

```glsl
#version 330 core

uniform sampler2D u_MainTex;
uniform vec4 u_TintColor;
uniform float u_Alpha;
uniform vec3 u_CameraPos;
uniform vec3 u_LightDir;

in vec3 v_Normal;
in vec3 v_WorldPos;
in vec2 v_UV;

out vec4 fragColor;

void main()
{
    vec4 texColor = texture(u_MainTex, v_UV);
    vec3 baseColor = texColor.rgb * u_TintColor.rgb;
    float alpha = texColor.a * u_TintColor.a * u_Alpha;

    // 简单光照
    vec3 N = normalize(v_Normal);
    vec3 L = normalize(u_LightDir);
    vec3 V = normalize(u_CameraPos - v_WorldPos);

    float NdotL = max(dot(N, L), 0.0);
    vec3 H = normalize(L + V);
    float spec = pow(max(dot(N, H), 0.0), 64.0);

    vec3 ambient = baseColor * 0.2;
    vec3 diffuse = baseColor * NdotL;
    vec3 specular = vec3(1.0) * spec * 0.5;

    // 菲涅尔边缘透明
    float fresnel = pow(1.0 - max(dot(N, V), 0.0), 3.0);
    alpha = mix(alpha, 1.0, fresnel * 0.3);

    fragColor = vec4(ambient + diffuse + specular, alpha);
}
```

### 4.2 HLSL透明着色器（Unity URP）

```hlsl
Shader "Custom/Transparent"
{
    Properties
    {
        _BaseColor ("Base Color", Color) = (1, 1, 1, 0.5)
        _MainTex ("Main Texture", 2D) = "white" {}
        _Smoothness ("Smoothness", Range(0, 1)) = 0.5
    }

    SubShader
    {
        Tags {
            "RenderType" = "Transparent"
            "Queue" = "Transparent"
            "RenderPipeline" = "UniversalPipeline"
        }

        Pass
        {
            Name "ForwardLit"
            Tags { "LightMode" = "UniversalForward" }

            Blend SrcAlpha OneMinusSrcAlpha
            ZWrite Off          // 半透明不写入深度
            Cull Back           // 剔除背面

            HLSLPROGRAM
            #pragma vertex vert
            #pragma fragment frag

            #include "Packages/com.unity.render-pipelines.universal/ShaderLibrary/Core.hlsl"
            #include "Packages/com.unity.render-pipelines.universal/ShaderLibrary/Lighting.hlsl"

            CBUFFER_START(UnityPerMaterial)
                float4 _BaseColor;
                float4 _MainTex_ST;
                float _Smoothness;
            CBUFFER_END

            TEXTURE2D(_MainTex);
            SAMPLER(sampler_MainTex);

            struct Attributes {
                float4 positionOS : POSITION;
                float3 normalOS : NORMAL;
                float2 uv : TEXCOORD0;
            };

            struct Varyings {
                float4 positionCS : SV_POSITION;
                float3 positionWS : TEXCOORD0;
                float3 normalWS : TEXCOORD1;
                float2 uv : TEXCOORD2;
            };

            Varyings vert(Attributes input) {
                Varyings output;
                VertexPositionInputs posInputs = GetVertexPositionInputs(input.positionOS.xyz);
                VertexNormalInputs normInputs = GetVertexNormalInputs(input.normalOS);
                output.positionCS = posInputs.positionCS;
                output.positionWS = posInputs.positionWS;
                output.normalWS = normInputs.normalWS;
                output.uv = TRANSFORM_TEX(input.uv, _MainTex);
                return output;
            }

            half4 frag(Varyings input) : SV_Target {
                half4 texColor = SAMPLE_TEXTURE2D(_MainTex, sampler_MainTex, input.uv);
                half4 baseColor = texColor * _BaseColor;

                Light mainLight = GetMainLight();
                half3 N = normalize(input.normalWS);
                half3 L = mainLight.direction;
                half3 V = normalize(_WorldSpaceCameraPos - input.positionWS);
                half3 H = normalize(L + V);

                half NdotL = saturate(dot(N, L));
                half3 diffuse = baseColor.rgb * mainLight.color * NdotL;

                half NdotH = saturate(dot(N, H));
                half spec = pow(NdotH, _Smoothness * 128.0);
                half3 specular = mainLight.color * spec * _Smoothness;

                half3 ambient = SampleSH(N) * baseColor.rgb;

                half3 finalColor = ambient + (diffuse + specular) * mainLight.shadowAttenuation;
                return half4(finalColor, baseColor.a);
            }
            ENDHLSL
        }
    }
}
```

## 5. 预乘Alpha的优势与实现

```hlsl
// 预乘Alpha — 纹理中RGB已乘以A
// 混合公式：C_result = C_src + C_dst × (1 - α_src)
Blend One OneMinusSrcAlpha

// 在Photoshop/美术工具中生成预乘Alpha纹理：
// 输出时选择"Premultiplied Alpha"
// 或在Shader中转换：
half4 PremultiplyAlpha(half4 color)
{
    return half4(color.rgb * color.a, color.a);
}

// 预乘Alpha解决的问题：
// 问题：标准Alpha混合中，多层半透明叠加会导致颜色越来越暗
//   层1: RGB=(1,0,0) A=0.5 → Dst = (0.5, 0, 0)
//   层2: RGB=(1,0,0) A=0.5 → Dst = 0.5*(1,0,0) + 0.5*(0.5,0,0) = (0.75, 0, 0)  ← 颜色变暗！
// 预乘：
//   层1: C'=(0.5,0,0) A=0.5 → Dst = (0.5, 0, 0) + (0,0,0)*0.5 = (0.5, 0, 0)
//   层2: C'=(0.5,0,0) A=0.5 → Dst = (0.5, 0, 0) + (0.5,0,0)*0.5 = (0.75, 0, 0)
// 结果相同！预乘保证正确的颜色叠加。
```

## 6. Dithered透明（抖动透明）

用图案化的不透明/不透明像素模拟半透明，避免排序问题。

```glsl
// 抖动透明 — 无需排序，但有噪声感
uniform float u_Alpha;
uniform vec2 u_ScreenSize;

// Bayer抖动矩阵 4x4
float BayerDither4x4(vec2 position)
{
    int x = int(mod(position.x, 4.0));
    int y = int(mod(position.y, 4.0));
    int index = x + y * 4;

    // 4x4 Bayer矩阵
    float matrix[16] = float[16](
         0,  8,  2, 10,
        12,  4, 14,  6,
         3, 11,  1,  9,
        15,  7, 13,  5
    );

    return matrix[index] / 16.0;
}

void main()
{
    vec4 texColor = texture(u_MainTex, v_UV);

    // 抖动裁剪
    float threshold = BayerDither4x4(gl_FragCoord.xy);
    if (texColor.a * u_Alpha < threshold)
        discard;

    fragColor = vec4(texColor.rgb, 1.0); // 输出完全不透明
}
```

## 7. 深度剥离（Depth Peeling）实现思路

```
Pass 1：渲染最近半透明层
  Depth Test: LEQUAL
  同时输出到颜色RT和深度RT

Pass 2：剥离最近层
  Depth Test: GREATER（只通过比Pass1深度更远的片元）
  输出到新颜色RT

Pass N：继续剥离...

最终：从最后层到第一层逆序混合所有颜色RT
```

## 8. 性能提示

- 半透明渲染是性能杀手：每层半透明都是全屏Overdraw
- 减少半透明物体的屏幕覆盖面积是最有效的优化
- Alpha Test会破坏Early-Z，导致后续不透明物体无法被剔除
- 预乘Alpha在移动端有轻微性能优势（少一次乘法）
- OIT方案（WBOIT、Per-Pixel Linked List）开销大，适合PC/主机
- 粒子系统使用加法混合（One One）时不需要排序

## 9. 常见问题与调试

**问题1：半透明物体看起来太暗**
- 检查是否使用了预乘Alpha纹理但混合模式为标准Alpha
- 确保不在半透明物体上应用过多的光照衰减

**问题2：半透明物体互相穿插时显示错误**
- 这是深度排序无法解决的根本问题
- 使用Depth Peeling或OIT技术

**问题3：半透明物体出现在不透明物体后面**
- 检查ZWrite是否关闭（半透明通常不写深度）
- 检查渲染队列是否在Geometry之后

**问题4：Alpha Test边缘锯齿严重**
- 使用Alpha To Coverage（需要开启MSAA）
- 或使用屏幕空间抖动（Dither）配合TAA

**问题5：UI叠加层颜色不正确**
- UI通常使用预乘Alpha纹理
- 确保Blend模式为`Blend One OneMinusSrcAlpha`

## 10. 实际使用案例

- 《荒野大镖客2》水面使用多层Alpha Blend叠加折射和反射
- 植被渲染常用Alpha Test（或Alpha To Coverage）处理树叶铁丝网
- 粒子系统大量使用预乘Alpha或加法混合避免颜色衰减
- WBOIT（Weighted Blended OIT）被Unreal Engine作为可选的OIT方案
- Per-Pixel Linked List OIT在DirectX 11+硬件上实现精确的顺序无关透明
- 《赛博朋克2077》的霓虹灯和烟雾效果大量使用加法混合粒子
