# 阴影技术

## 1. 核心理论

实时阴影增强场景深度感和空间关系，是画面真实感的核心要素。阴影的本质是判断"一个点是否被光源直接照亮"。Shadow Map是当前最主流的实时阴影技术，其核心思想是**从光源视角存储深度信息，然后在主渲染时比较深度判断遮挡关系**。

### 1.1 Shadow Map原理

```
Shadow Map两阶段流程：

阶段1：从光源视角渲染（生成Shadow Map）
┌──────────────────────────────────────┐
│  将相机放在光源位置，朝光源方向看    │
│  渲染场景的所有不透明物体            │
│  不输出颜色，只写入深度缓冲          │
│  深度缓冲 = Shadow Map（纹理）       │
└──────────────────────────────────────┘

阶段2：主相机渲染（采样Shadow Map）
┌──────────────────────────────────────┐
│  对于每个片元：                       │
│  1. 将世界坐标变换到光源空间         │
│  2. 用光源空间UV采样Shadow Map       │
│  3. 比较采样深度 vs 片元在光源视角的深度 │
│  4. 采样深度 < 片元深度 → 在阴影中   │
│  5. 采样深度 >= 片元深度 → 被照亮    │
└──────────────────────────────────────┘
```

**数学推导：**

```
Shadow Map深度比较：

1. 片元世界坐标 P_world
2. 变换到光源裁剪空间：P_light = LightVP × P_world
3. 透视除法得到NDC：P_ndc = P_light.xyz / P_light.w
4. 映射到[0,1]：P_uv = P_ndc * 0.5 + 0.5
5. 片元在光源视角的深度：depth_fragment = P_uv.z
6. Shadow Map中存储的深度：depth_shadow = texture(shadowMap, P_uv.xy).r
7. 阴影判断：
   shadow = (depth_fragment - bias) > depth_shadow ? 1.0 : 0.0
```

### 1.2 GLSL完整实现

```glsl
// ============ Pass 1: Shadow Map生成 ============
// 顶点着色器
#version 330 core
layout(location = 0) in vec3 a_Position;

uniform mat4 u_LightViewProjection;
uniform mat4 u_Model;

void main()
{
    gl_Position = u_LightViewProjection * u_Model * vec4(a_Position, 1.0);
}

// 片元着色器 — 不需要输出颜色，深度自动写入
void main()
{
    // 空的片元着色器，或输出gl_FragDepth
}

// ============ Pass 2: 主渲染中采样Shadow Map ============
uniform sampler2DShadow u_ShadowMap;  // 深度比较采样器
uniform mat4 u_LightVP;
uniform vec3 u_LightDir;

in vec3 v_WorldPos;
in vec3 v_Normal;

float ShadowCalculation(vec3 worldPos, vec3 normal, vec3 lightDir)
{
    // 1. 变换到光源空间
    vec4 lightSpacePos = u_LightVP * vec4(worldPos, 1.0);
    vec3 projCoords = lightSpacePos.xyz / lightSpacePos.w;
    projCoords = projCoords * 0.5 + 0.5; // NDC [-1,1] → [0,1]

    // 2. 检查是否在Shadow Map范围内
    if (projCoords.z > 1.0)
        return 0.0; // 超出光源远平面，不在阴影中

    // 3. 自适应阴影偏差（解决Shadow Acne）
    float bias = max(0.05 * (1.0 - dot(normal, lightDir)), 0.005);

    // 4. 使用sampler2DShadow进行硬件PCF
    float shadow = texture(u_ShadowMap, vec3(projCoords.xy, projCoords.z - bias));
    // shadow = 1.0 → 在阴影中，shadow = 0.0 → 被照亮
    // 注意：sampler2DShadow返回的是0或1的比较结果

    return 1.0 - shadow; // 反转：1=被照亮, 0=在阴影中
}
```

## 2. Shadow Acne与Bias

### 2.1 Shadow Acne（阴影痤疮）

Shadow Map分辨率有限导致深度离散化，加上数值精度误差，表面上的点可能被错误判断为"自身遮挡"，产生条纹状的自阴影瑕疵。

```
Shadow Acne产生原因：
深度值在Shadow Map中是离散的（有限精度的浮点数）
片元实际深度 vs Shadow Map存储深度 可能因舍入误差出现 bias
  实际深度 = 0.5000001
  Map深度 = 0.5000000
  0.5000001 > 0.5000000 → 错误判断为被遮挡！

解决：添加阴影偏差（Bias）
  if (fragmentDepth - bias > mapDepth) → 阴影
```

### 2.2 Bias策略

```glsl
// 策略1：固定偏差（简单但不鲁棒）
float bias = 0.005;

// 策略2：基于角度的自适应偏差（推荐）
// 法线与光线平行时偏差小（正面），垂直时偏差大（侧面）
float bias = max(0.05 * (1.0 - dot(normal, lightDir)), 0.005);

// 策略3：基于斜率的偏差（最精确）
// 使用多边形的斜率计算偏差
float bias = tan(acos(dot(normal, lightDir))) * 0.001;

// 策略4：Normal Offset Bias（偏移Shadow Map采样位置而非深度）
// 沿法线方向偏移采样位置，避免正面和侧面的偏差差异
vec3 offsetPos = worldPos + normal * 0.01;
vec4 lightSpacePos = u_LightVP * vec4(offsetPos, 1.0);
```

### 2.3 Peter Panning（彼得潘效应）

Bias过大导致阴影与物体底部脱离，仿佛物体飘在空中。

```
问题：
  Bias太小 → Shadow Acne（自阴影条纹）
  Bias太大 → Peter Panning（阴影脱离）

解决方案：
1. 使用背面渲染生成Shadow Map
   Cull Front（剔除正面，只渲染背面）
   天然增大了Bias（背面比正面远离光源）
   适用于封闭物体

2. Normal Offset Bias
   通过法线方向偏移采样位置而非深度值
   对正面和侧面的偏差自适应

3. 使用Shadow Pancaking
   在Shadow Map生成时，将光源空间的近平面推近
   减少深度精度浪费
```

```hlsl
// HLSL — 背面渲染Shadow Map
Pass
{
    Name "ShadowCaster"
    Cull Front  // 剔除正面，只渲染背面 → 解决Peter Panning
    ZWrite On
    ZTest LEqual

    HLSLPROGRAM
    // Shadow Caster代码...
    ENDHLSL
}
```

## 3. PCF软阴影

### 3.1 PCF原理

PCF（Percentage-Closer Filtering）在Shadow Map上多次采样，对每次比较结果取平均值，产生柔和的阴影边缘。

```
硬阴影（无PCF）：
  采样1次 → 结果要么0要么1 → 边缘锯齿

软阴影（PCF 3x3）：
  采样9次 → 每次结果0或1 → 取平均值(0~1) → 边缘渐变

PCF不是对深度值取平均，而是对比较结果取平均！
这是"Percentage-Closer"的含义：先比较，后平均
```

### 3.2 GLSL PCF实现

```glsl
// 3x3 PCF
float PCFShadow3x3(sampler2D shadowMap, vec3 projCoords, float bias)
{
    float shadow = 0.0;
    vec2 texelSize = 1.0 / vec2(textureSize(shadowMap, 0));

    for (int x = -1; x <= 1; x++)
    {
        for (int y = -1; y <= 1; y++)
        {
            float pcfDepth = texture(shadowMap,
                projCoords.xy + vec2(x, y) * texelSize).r;
            shadow += (projCoords.z - bias > pcfDepth) ? 1.0 : 0.0;
        }
    }
    return shadow / 9.0; // 0=完全照亮, 1=完全阴影
}

// 5x5 PCF（更好的软阴影）
float PCFShadow5x5(sampler2D shadowMap, vec3 projCoords, float bias)
{
    float shadow = 0.0;
    vec2 texelSize = 1.0 / vec2(textureSize(shadowMap, 0));

    for (int x = -2; x <= 2; x++)
    {
        for (int y = -2; y <= 2; y++)
        {
            float pcfDepth = texture(shadowMap,
                projCoords.xy + vec2(x, y) * texelSize).r;
            shadow += (projCoords.z - bias > pcfDepth) ? 1.0 : 0.0;
        }
    }
    return shadow / 25.0;
}

// 随机旋转PCF（减少带状伪影）
float PCFShadowRotated(sampler2D shadowMap, vec3 projCoords,
                        float bias, float noise)
{
    float shadow = 0.0;
    vec2 texelSize = 1.0 / vec2(textureSize(shadowMap, 0));
    float angle = noise * 6.2831853; // 随机旋转角度 [0, 2π]
    mat2 rotation = mat2(cos(angle), -sin(angle), sin(angle), cos(angle));

    // 泊松圆盘采样（16个固定分布的采样点）
    vec2 poissonDisk[16] = vec2[](
        vec2(-0.94201624, -0.39906216), vec2(0.94558609, -0.76890725),
        vec2(-0.09418410, -0.92938870), vec2(0.34495938, 0.29387760),
        vec2(-0.91588581, 0.45771432),  vec2(-0.81544232, -0.87912464),
        vec2(-0.38277543, 0.27676845),  vec2(0.97484398, 0.75648379),
        vec2(0.44323325, -0.97511554),  vec2(0.53742981, -0.47373420),
        vec2(-0.26496911, -0.41893023), vec2(0.79197514, 0.19090188),
        vec2(-0.24188840, 0.99706507),  vec2(-0.81409955, 0.91437590),
        vec2(0.19984126, 0.78641367),   vec2(0.14383161, -0.14100790)
    );

    for (int i = 0; i < 16; i++)
    {
        vec2 offset = rotation * poissonDisk[i] * texelSize * 2.0;
        float pcfDepth = texture(shadowMap, projCoords.xy + offset).r;
        shadow += (projCoords.z - bias > pcfDepth) ? 1.0 : 0.0;
    }
    return shadow / 16.0;
}
```

### 3.3 PCSS（Percentage-Closer Soft Shadows）

PCSS根据遮挡物与被投影面的距离动态调整PCF的采样范围，近处阴影硬、远处阴影软。

```
PCSS三步骤：
1. Blocker Search：在阴影区域搜索遮挡物，计算平均遮挡深度
2. Penumbra Estimation：根据遮挡物距离计算半影宽度
   半影宽度 = (d_receiver - d_blocker) × lightSize / d_blocker
3. PCF Filtering：用计算出的半影宽度作为PCF采样半径
```

## 4. 级联阴影（CSM）

### 4.1 CSM原理

CSM将视锥体按深度分为多个层级，每层使用独立的Shadow Map。近处用高分辨率覆盖小范围，远处用低分辨率覆盖大范围。

```
CSM级联划分示意：

相机位置
  ↓
  |---Split 0---|----Split 1----|---Split 2---|----Split 3----|
  |  0 ~ 10m    |   10 ~ 40m    |  40 ~ 150m  |  150 ~ 500m  |
  | 2048×2048   |  2048×2048    | 2048×2048   |  2048×2048   |

近处Split覆盖范围小 → 像素密度高 → 阴影清晰
远处Split覆盖范围大 → 像素密度低 → 阴影粗糙
```

### 4.2 级联分割方案

```cpp
// 分割方式1：均匀分割（Uniform）
float split0 = near;
float split1 = near + (far - near) * 0.25;
float split2 = near + (far - near) * 0.50;
float split3 = near + (far - near) * 0.75;
float split4 = far;
// 缺点：近处精度浪费，远处不够

// 分割方式2：对数分割（Logarithmic）
float split = near * pow(far / near, (float)i / (float)numCascades);
// 优点：近处精度极高。缺点：远处精度急剧下降

// 分割方式3：混合分割（Practical Scheme，推荐！）
float logSplit = near * pow(far / near, (float)i / (float)numCascades);
float uniformSplit = near + (far - near) * (float)i / (float)numCascades;
float split = lambda * logSplit + (1.0 - lambda) * uniformSplit;
// lambda = 0.75 通常是好的折中
// 既有对数分割的近处精度，又有均匀分割的远处稳定性
```

### 4.3 CSM实现

```glsl
// 选择级联
uniform mat4 u_LightVP[4];      // 4个级联的光源VP矩阵
uniform float u_CascadeSplits[4]; // 4个分割距离（观察空间深度）
uniform sampler2D u_ShadowMaps[4]; // 4张Shadow Map

int GetCascadeIndex(float viewDepth)
{
    for (int i = 0; i < 4; i++)
    {
        if (viewDepth < u_CascadeSplits[i])
            return i;
    }
    return 3;
}

float CSMSampleShadow(vec3 worldPos, vec3 normal, vec3 lightDir, float viewDepth)
{
    int cascade = GetCascadeIndex(viewDepth);
    mat4 lightVP = u_LightVP[cascade];

    vec4 lightSpacePos = lightVP * vec4(worldPos, 1.0);
    vec3 projCoords = lightSpacePos.xyz / lightSpacePos.w * 0.5 + 0.5;

    float bias = max(0.05 * (1.0 - dot(normal, lightDir)), 0.005);

    // PCF
    float shadow = PCFShadow3x3(u_ShadowMaps[cascade], projCoords, bias);
    return 1.0 - shadow;
}
```

## 5. 阴影抗锯齿技巧

### 5.1 Temporal AA + Shadow

将多帧的PCF采样点偏移不同的随机位置，通过TAA混合多帧结果，实现高质量软阴影。

```glsl
// 每帧使用不同的Halton序列偏移
vec2 jitter = HaltonSequence(frameIndex % 16) * texelSize;
shadowUV += jitter;
```

### 5.2 VSM（Variance Shadow Maps）

存储深度的一阶和二阶矩（均值和方差），使用切比雪夫不等式估计阴影概率。

```glsl
// VSM：不需要多次采样，单次采样即可得到软阴影
vec2 moments = texture(u_VSMShadowMap, projCoords.xy).rg;
float mean = moments.x;      // 深度均值
float variance = moments.y - mean * mean;  // 方差

float d = projCoords.z - mean;  // 深度差
float pMax = variance / (variance + d * d); // 切比雪夫不等式

// 阴影 = 1 - pMax（在阴影中的概率）
float shadow = saturate(1.0 - pMax);
// 优点：天然软阴影，性能好。缺点：漏光（Light Bleeding）
```

## 6. 性能优化

```
阴影渲染优化策略：

1. Shadow Map分辨率
   近距离：2048×2048（高质量）
   远距离：1024×1024 或更低
   移动端：512×512 ~ 1024×1024

2. PCF采样数
   PC端：9~25次（3x3 或 5x5）
   移动端：4次（2x2）或硬件PCF

3. 阴影距离
   设置最大阴影距离，超出范围不渲染阴影
   减少Shadow Map覆盖范围，提高精度

4. 静态/动态分离
   静态物体的Shadow Map可预先烘焙
   动态物体只更新变化部分（增量更新）

5. Shadow Atlas
   将多张Shadow Map合为一张大纹理
   减少纹理切换开销
```

## 7. 常见问题与调试

**问题1：Shadow Acne（阴影痤疮）**
- 增大Bias或使用自适应Bias
- 使用背面渲染生成Shadow Map

**问题2：Peter Panning（阴影脱离）**
- 减小Bias
- 使用Normal Offset Bias替代Depth Bias

**问题3：阴影锯齿（Aliasing）**
- 增加PCF采样数
- 使用CSM提高近处阴影分辨率

**问题4：级联接缝（Cascade Transition）**
- 在级联边界处混合两个级联的阴影结果
- 使用dither混合减少接缝可见度

**问题5：阴影闪烁（Shadow Shimmering）**
- 将Shadow Map的视图矩阵的移动量化到纹素大小
- 固定光源空间相机的旋转，只允许平移

```cpp
// 量化Shadow Map相机位置（消除闪烁）
float texelSize = shadowMapWorldSize / shadowMapResolution;
lightViewPos.x = floor(lightViewPos.x / texelSize) * texelSize;
lightViewPos.y = floor(lightViewPos.y / texelSize) * texelSize;
```

## 8. 实际使用案例

- 《战地》系列使用CSM + PCSS实现大规模战场阴影，4级级联覆盖500m范围
- 移动端游戏通常使用单级Shadow Map + 简化PCF（2x2）控制性能
- Unreal Engine 5的Virtual Shadow Maps动态调整分辨率（最高16K），彻底消除级联接缝
- 《赛博朋克2077》使用RTX光线追踪阴影与CSM混合，近处RT远处CSM
- Unity HDRP支持Ray Traced Shadows（DXR），提供像素级精确的软阴影
- VSM/ESM等矩量阴影方案用于卡通渲染，避免PCF的噪声感
