# 纹理采样与UV

## 1. 核心理论

纹理采样是Shader中最频繁的操作，通过UV坐标（纹理坐标）从2D图像中获取颜色或其他数据。纹理不仅是"贴图"，更是GPU上最高效的逐像素数据存储方式——颜色、法线、高度、金属度、粗糙度等都可以编码在纹理中。

### 1.1 UV坐标的数学原理

UV坐标是纹理空间的二维参数化坐标，范围通常为[0, 1]。每个顶点携带UV信息，在光栅化阶段通过**透视校正插值**得到逐像素UV。

```
透视校正插值公式：
对于屏幕空间重心坐标 (w0, w1, w2)，实际UV为：
u = (w0*u0/z0 + w1*u1/z1 + w2*u2/z2) / (w0/z0 + w1/z1 + w2/z2)
v = (w0*v0/z0 + w1*v1/z1 + w2*v2/z2) / (w0/z0 + w1/z1 + w2/z2)

其中 z0, z1, z2 是各顶点在观察空间的深度
这确保了纹理在透视投影下不会出现扭曲
```

### 1.2 UV映射类型

```
常见UV映射方式：

1. 球形映射（Spherical）：
   u = 0.5 + atan2(N.z, N.x) / (2π)
   v = 0.5 - asin(N.y) / π
   适合：天空盒、环境反射

2. 圆柱映射（Cylindrical）：
   u = atan2(N.z, N.x) / (2π)
   v = (N.y + 1) / 2
   适合：柱状物体（瓶子、柱子）

3. 平面映射（Planar）：
   u = pos.x, v = pos.y（投影到xy平面）
   适合：地面、墙面

4. 立方体贴图（Cubemap）：
   6张图分别对应 ±X, ±Y, ±Z 六个面
   用3D方向向量采样，自动选择面
   适合：天空盒、反射探针

5. 三平面映射（Triplanar）：
   分别在XY、XZ、YZ三个平面采样，按法线方向混合
   适合：地形、不规则表面的自动纹理
```

## 2. 纹理寻址模式（Wrap Mode）

当UV超出[0,1]范围时的处理方式：

```
Wrap模式对比：

GL_REPEAT / D3D11_TEXTURE_ADDRESS_WRAP：
UV: -0.3 → 0.7,  1.2 → 0.2
效果：重复平铺。适合地面、墙壁等需要重复的纹理。

GL_CLAMP_TO_EDGE / D3D11_TEXTURE_ADDRESS_CLAMP：
UV: -0.3 → 0.0,  1.2 → 1.0
效果：边缘拉伸。适合UI、天空盒等不需要重复的纹理。

GL_MIRRORED_REPEAT / D3D11_TEXTURE_ADDRESS_MIRROR：
UV: -0.3 → 0.3,  1.2 → 0.8
效果：镜像重复。减少平铺接缝感。

GL_CLAMP_TO_BORDER / D3D11_TEXTURE_ADDRESS_BORDER：
UV: <0 或 >1 → 边界颜色（通常为透明或黑色）
效果：外部区域用指定颜色填充。
```

```glsl
// OpenGL设置
glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_REPEAT);
glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_CLAMP_TO_EDGE);

// GLSL中用fract()手动实现repeat
vec2 repeatedUV = fract(v_UV * tiling); // tiling=5 → 5x5平铺

// HLSL中用frac()
float2 repeatedUV = frac(v_UV * tiling);
```

## 3. 纹理过滤

### 3.1 缩小过滤（Minification）

当纹理像素（Texel）比屏幕像素多时（物体远离相机），需要合并多个纹素。

```
过滤模式：
NEAREST（最近邻）：取最近的纹素，速度快但有锯齿
LINEAR（双线性）：取周围4个纹素线性插值，平滑但略模糊
```

### 3.2 放大过滤（Magnification）

当纹理像素比屏幕像素少时（物体靠近相机），需要插值。

```
NEAREST：块状像素感（像素艺术风格）
LINEAR：平滑过渡（大多数情况）
```

### 3.3 Mipmap详解

Mipmap是纹理的多级缩小版本序列，每级分辨率为上一级的一半。

```
Mipmap层级：
Level 0: 1024 × 1024 (原始)
Level 1:  512 ×  512
Level 2:  256 ×  256
Level 3:  128 ×  128
...
Level 10:   1 ×    1

额外显存占用 = 原始大小的 1/3 ≈ 33%
(1 + 1/4 + 1/16 + ...) = 4/3

Mipmap Level计算：
level = log2(screenSpaceTexelDensity)

Mipmap过滤模式：
NEAREST_MIPMAP_NEAREST：选最近层级取最近纹素
LINEAR_MIPMAP_NEAREST：选最近层级双线性插值
NEAREST_MIPMAP_LINEAR：两层级间最近纹素再线性插值（三线性过滤）
LINEAR_MIPMAP_LINEAR：三线性过滤 — 两层级各自双线性插值再混合（最高质量）
```

```glsl
// OpenGL设置
glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER, GL_LINEAR_MIPMAP_LINEAR);
glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAG_FILTER, GL_LINEAR);
glGenerateMipmap(GL_TEXTURE_2D); // 生成Mipmap

// GLSL中手动指定LOD
vec4 color = textureLod(u_Tex, uv, 2.0); // 强制使用Mip Level 2

// 计算梯度（用于自定义LOD）
vec2 ddx = dFdx(v_UV);
vec2 ddy = dFdy(v_UV);
vec4 color = textureGrad(u_Tex, uv, ddx, ddy);
```

```hlsl
// HLSL设置
Texture2D<float4> _MainTex : register(t0);
SamplerState sampler_Linear : register(s0);

// 采样（自动选择Mip层级）
float4 color = _MainTex.Sample(sampler_Linear, uv);

// 指定LOD
float4 colorLOD = _MainTex.SampleLevel(sampler_Linear, uv, lod);

// 指定梯度
float4 colorGrad = _MainTex.SampleGrad(sampler_Linear, uv, ddx, ddy);

// 直接加载纹素（无过滤）
float4 texel = _MainTex.Load(int3(pixelX, pixelY, mipLevel));
```

## 4. 各向异性过滤

当表面法线与视线夹角较大时（如地面），纹理在水平和垂直方向压缩比不同。普通各向同性过滤在这种情况下会导致模糊。

```
各向异性过滤原理：
倾斜表面        各向同性采样       各向异性采样
  /            ___________       ___________
 /            | · · · · · |     | ····→     |
/             | · · · · · |     | ····→     |
              |___________|     |___________|
  采样区域模糊    圆形采样区域     椭圆形沿拉伸方向采样

各向异性等级：
1x = 无各向异性（各向同性）
2x = 沿最大拉伸方向采样2次
4x = 采样4次
8x = 采样8次
16x = 采样16次（最高质量，最大开销）
```

```cpp
// OpenGL设置
float maxAniso;
glGetFloatv(GL_MAX_TEXTURE_MAX_ANISOTROPY_EXT, &maxAniso);
glTexParameterf(GL_TEXTURE_2D, GL_TEXTURE_MAX_ANISOTROPY_EXT,
                min(16.0f, maxAniso));

// DirectX设置（C++端）
D3D11_SAMPLER_DESC desc;
desc.Filter = D3D11_FILTER_ANISOTROPIC;
desc.MaxAnisotropy = 16;
```

## 5. 三平面映射（Triplanar Mapping）

自动在三个方向投影纹理，无需UV展开，适合地形等复杂几何。

```hlsl
// Unity URP三平面映射
half4 TriplanarSample(TEXTURE2D(tex), SAMPLER(samp), float3 worldPos,
                       float3 normal, float tiling)
{
    // 三个方向的UV
    float2 uvX = worldPos.yz * tiling;
    float2 uvY = worldPos.xz * tiling;
    float2 uvZ = worldPos.xy * tiling;

    // 三个方向的采样
    half4 texX = SAMPLE_TEXTURE2D(tex, samp, uvX);
    half4 texY = SAMPLE_TEXTURE2D(tex, samp, uvY);
    half4 texZ = SAMPLE_TEXTURE2D(tex, samp, uvZ);

    // 根据法线方向混合
    float3 blend = abs(normal);
    blend /= (blend.x + blend.y + blend.z); // 归一化

    return texX * blend.x + texY * blend.y + texZ * blend.z;
}
```

## 6. 纹理图集（Texture Atlas）

将多张小纹理合并到一张大纹理上，减少纹理切换从而减少Draw Call。

```glsl
// 纹理图集采样
uniform sampler2D u_Atlas;    // 大图集
uniform vec4 u_UVRect;        // x, y, w, h 归一化UV范围

vec4 SampleAtlas(vec2 uv)
{
    // 将[0,1]的UV映射到图集中的子区域
    vec2 atlasUV = u_UVRect.xy + uv * u_UVRect.zw;
    return texture(u_Atlas, atlasUV);
}

// 多个子图示例：
// 图集 1024x1024
// 子图A: rect(0, 0, 256, 256)    → u_UVRect = (0, 0, 0.25, 0.25)
// 子图B: rect(256, 0, 256, 256)  → u_UVRect = (0.25, 0, 0.25, 0.25)
// 子图C: rect(0, 256, 512, 512)  → u_UVRect = (0, 0.25, 0.5, 0.5)
```

## 7. 立方体贴图（Cube Map）

立方体贴图由6张正方形纹理组成，用于天空盒、环境反射等。

```glsl
uniform samplerCube u_Skybox;
in vec3 v_ReflectDir; // 反射方向向量

void main()
{
    // 直接用3D方向向量采样
    vec4 skyColor = texture(u_Skybox, v_ReflectDir);

    // 反射方向计算
    vec3 N = normalize(v_Normal);
    vec3 V = normalize(u_CameraPos - v_WorldPos);
    vec3 reflectDir = reflect(-V, N);
    vec4 envReflection = texture(u_Skybox, reflectDir);

    // 折射
    float ratio = 1.0 / 1.52; // 空气/玻璃折射率比
    vec3 refractDir = refract(-V, N, ratio);
    vec4 envRefraction = texture(u_Skybox, refractDir);
}
```

```hlsl
// HLSL立方体贴图
TextureCube _Skybox : register(t0);
SamplerState sampler_Linear : register(s0);

half4 SkyboxFrag(Varyings input) : SV_Target
{
    float3 viewDir = normalize(input.viewDirWS);
    return _Skybox.Sample(sampler_Linear, viewDir);
}
```

## 8. 流程步骤

```
纹理采样的完整流程：
1. CPU端：加载纹理文件 → 上传到GPU显存
2. CPU端：设置采样状态（过滤、Wrap、各向异性）
3. CPU端：将纹理绑定到Shader的采样器槽位
4. GPU顶点着色器：计算/传递UV坐标
5. GPU光栅化：对UV进行透视校正插值
6. GPU片元着色器：用插值后的UV调用texture()采样
7. GPU硬件：根据过滤模式从显存中读取纹素并混合
```

## 9. 性能提示

- Mipmap几乎免费（只需33%额外显存），务必开启
- 各向异性过滤开销不大（现代GPU硬件加速），建议至少4x
- 压缩纹理格式（BC7/ASTC）可节省75%显存和带宽
- 纹理采样是带宽密集型操作，缓存未命中是主要瓶颈
- 避免在片元着色器中动态计算UV（如三平面映射），使用预计算UV更高效
- 纹理打包：将多张单通道图合为一张RGBA纹理，减少采样次数

## 10. 常见问题与调试

**问题1：纹理出现锯齿或闪烁**
- 检查Mipmap是否启用，设置`GL_LINEAR_MIPMAP_LINEAR`
- 远处物体使用更高层级的Mipmap

**问题2：UV接缝（Seam）**
- 检查模型UV展开是否有重叠或拉伸
- 使用Wrap模式为`GL_REPEAT`时注意边界

**问题3：纹理模糊**
- 检查是否错误使用了过高的Mip层级
- 检查各向异性过滤等级是否太低

**问题4：纹理颜色不正确**
- 检查sRGB/线性空间设置
- OpenGL中使用`GL_SRGB8`格式，HLSL中使用`TEX2D_SRGB`

**问题5：Triplanar映射出现拉伸**
- 在接缝处（法线接近45度的面）可能出现混合伪影
- 使用`pow(blend, 4)`增大混合锐度

## 11. 实际使用案例

- 开放世界游戏使用虚拟纹理（Virtual Texture / Sparse Texture）管理海量纹理数据
- 《我的世界》使用Texture Atlas将所有方块纹理合为一张图，减少Draw Call
- PBR材质管线通常需要4-5张纹理（反照率、法线、金属度、粗糙度、环境遮蔽）
- 三平面映射广泛用于地形渲染（如Unreal Engine的Landscape Material）
- 程序化纹理（Procedural Texture）使用噪声函数在Shader中生成纹理，无需美术制作
