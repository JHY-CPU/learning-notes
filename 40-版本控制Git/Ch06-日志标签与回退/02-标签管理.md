# 标签管理

## 一、标签的概念

标签（Tag）是指向某个特定提交的不可变引用，通常用于标记发布版本（如 v1.0.0）。

### 两种标签类型

```
轻量标签（Lightweight）：
  只是一个指向提交的指针，类似不会移动的分支

附注标签（Annotated）：
  包含标签创建者信息、日期、消息，以及可选的 GPG 签名
  推荐用于发布版本
```

## 二、创建标签

### 1. 附注标签（推荐）

```bash
# 在当前提交创建附注标签
git tag -a v1.0.0 -m "Release version 1.0.0"

# 在指定提交创建标签
git tag -a v1.0.0 abc1234 -m "Release version 1.0.0"

# 在指定分支的某个提交创建标签
git tag -a v0.9.0 main~5 -m "Pre-release 0.9.0"
```

### 2. 轻量标签

```bash
# 在当前提交创建轻量标签
git tag v1.0.0

# 在指定提交创建
git tag v1.0.0 abc1234
```

### 3. 签名标签

```bash
# 使用 GPG 签名创建标签（需要配置 GPG 密钥）
git tag -s v1.0.0 -m "Signed release 1.0.0"

# 使用特定密钥签名
git tag -u <key-id> v1.0.0 -m "Release 1.0.0"
```

## 三、查看标签

```bash
# 列出所有标签
git tag
# v0.1.0
# v0.2.0
# v1.0.0
# v1.1.0

# 使用通配符过滤
git tag -l "v1.*"
git tag -l "v*"
git tag -l "*beta*"

# 查看标签详细信息
git show v1.0.0
# tag v1.0.0
# Tagger: Alice <alice@example.com>
# Date:   Mon Jan 1 12:00:00 2025 +0800
#
# Release version 1.0.0
#
# commit abc1234...

# 按版本排序查看
git tag --sort=-version:refname
git tag --sort=-creatordate

# 查看标签指向的提交
git log --oneline --decorate --tags
```

## 四、操作标签

### 1. 推送标签

```bash
# 推送单个标签
git push origin v1.0.0

# 推送所有标签
git push origin --tags

# 注意：git push 默认不推送标签！
# 需要单独推送或使用 --tags
```

### 2. 删除标签

```bash
# 删除本地标签
git tag -d v1.0.0
git tag --delete v1.0.0

# 删除远程标签
git push origin --delete v1.0.0
git push origin :refs/tags/v1.0.0  # 旧版语法

# 批量删除匹配的标签
git tag -l "v0.*" | xargs git tag -d
```

### 3. 检出标签

```bash
# 查看标签对应的代码状态
git checkout v1.0.0

# 基于标签创建分支（推荐）
git switch -c release/v1.0.0 v1.0.0
git checkout -b release/v1.0.0 v1.0.0

# 直接检出会进入"分离 HEAD"状态
```

## 五、语义化版本（SemVer）

### 版本号格式

```
v<主版本号>.<次版本号>.<修订号>[-预发布标签][+构建号]

v1.2.3-beta.1+build.456

主版本号（MAJOR）: 不兼容的 API 变更
次版本号（MINOR）: 向后兼容的新功能
修订号（PATCH）:   向后兼容的 Bug 修复

预发布标签: alpha, beta, rc (release candidate)
```

### 版本号递增规则

```
1.0.0 → 1.0.1   PATCH: 修复 bug
1.0.1 → 1.1.0   MINOR: 新增功能
1.1.0 → 2.0.0   MAJOR: 破坏性变更

预发布版本:
1.0.0-alpha.1   → 内测版本
1.0.0-beta.1    → 公测版本
1.0.0-rc.1      → 候选发布版本
1.0.0           → 正式版本
```

### 标签示例

```bash
git tag -a v1.0.0-alpha.1 -m "Alpha release 1"
git tag -a v1.0.0-beta.1 -m "Beta release 1"
git tag -a v1.0.0-rc.1 -m "Release candidate 1"
git tag -a v1.0.0 -m "Stable release 1.0.0"
```

## 六、自动化标签

### 1. 从 CHANGELOG 生成标签

```bash
# 获取上一个标签
git describe --tags --abbrev=0
# v1.2.0

# 获取所有标签间的提交
git log v1.0.0..v1.1.0 --oneline
```

### 2. GitHub Actions 自动打标签

```yaml
# .github/workflows/release.yml
name: Release

on:
  push:
    tags:
      - 'v*'

jobs:
  release:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Create Release
        uses: actions/create-release@v1
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        with:
          tag_name: ${{ github.ref }}
          release_name: Release ${{ github.ref }}
          draft: false
          prerelease: false
```

### 3. 使用 standard-version 自动打标签

```bash
# 安装
npm install -g standard-version

# 自动根据 conventional commits 生成版本和标签
standard-version

# 首次发布
standard-version --first-release

# 指定版本类型
standard-version --release-as minor
standard-version --release-as patch
standard-version --release-as major
```

## 七、标签最佳实践

```
推荐做法：
✓ 始终使用附注标签（-a）标记发布版本
✓ 标签信息包含变更摘要
✓ 遵循语义化版本规范
✓ 推送标签到远程仓库
✓ 使用 GPG 签名重要标签
✓ 在 CHANGELOG.md 中记录版本变更

避免做法：
✗ 不要用轻量标签标记正式发布
✗ 不要在标签创建后修改它（不可变）
✗ 不要忘记推送标签
✗ 不要使用不规范的版本号格式
```
