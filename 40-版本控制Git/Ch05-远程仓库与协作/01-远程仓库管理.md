# 远程仓库管理

## 一、远程仓库概念

远程仓库是托管在互联网或网络中的 Git 仓库版本。常见托管平台：

| 平台 | 说明 | 地址 |
|------|------|------|
| GitHub | 最大的开源托管平台 | github.com |
| GitLab | 支持自托管 | gitlab.com |
| Bitbucket | Atlassian 旗下 | bitbucket.org |
| Gitee | 国内平台 | gitee.com |
| Azure DevOps | 微软 | dev.azure.com |

## 二、管理远程仓库

### 1. 添加远程仓库

```bash
# 添加远程仓库（默认名称 origin）
git remote add origin https://github.com/user/repo.git

# 添加多个远程仓库
git remote add upstream https://github.com/original/repo.git

# 添加 SSH 方式
git remote add origin git@github.com:user/repo.git
```

### 2. 查看远程仓库

```bash
# 查看远程仓库名称
git remote
# origin
# upstream

# 查看远程仓库详细信息（URL）
git remote -v
# origin    https://github.com/user/repo.git (fetch)
# origin    https://github.com/user/repo.git (push)
# upstream  https://github.com/original/repo.git (fetch)
# upstream  https://github.com/original/repo.git (push)

# 查看远程仓库详细信息
git remote show origin
# * remote origin
#   Fetch URL: https://github.com/user/repo.git
#   Push  URL: https://github.com/user/repo.git
#   HEAD branch: main
#   Remote branches:
#     main   tracked
#     develop tracked
#   Local branch configured for 'git pull':
#     main merges with remote main
#   Local ref configured for 'git push':
#     main pushes to main (up to date)
```

### 3. 修改远程仓库

```bash
# 修改远程仓库 URL
git remote set-url origin https://github.com/user/new-repo.git

# 从 HTTPS 改为 SSH
git remote set-url origin git@github.com:user/repo.git

# 重命名远程仓库
git remote rename origin github

# 修改远程仓库的推送 URL（fetch 和 push 用不同地址）
git remote set-url --push origin git@github.com:user/repo.git
```

### 4. 删除远程仓库

```bash
# 删除远程仓库引用
git remote remove origin
git remote rm origin
```

## 三、fetch、pull、push

### 1. git fetch —— 获取远程更新

```bash
# 获取所有远程仓库的更新
git fetch --all

# 获取指定远程仓库的更新
git fetch origin

# 获取指定分支
git fetch origin main

# 获取并清理已删除的远程分支
git fetch --prune
git fetch -p
```

fetch 后的操作：

```bash
# 查看远程分支的更新
git log HEAD..origin/main --oneline

# 查看远程分支的新提交
git log origin/main --oneline -5

# 合并远程更新
git merge origin/main

# 或变基
git rebase origin/main
```

### 2. git pull —— 拉取并合并

```bash
# 拉取当前分支对应的远程分支（fetch + merge）
git pull

# 等价于
git fetch origin
git merge origin/main

# 拉取并变基（推荐，保持线性历史）
git pull --rebase

# 拉取指定远程仓库和分支
git pull origin main
git pull origin feature

# 设置默认拉取行为为 rebase
git config --global pull.rebase true
```

### pull 策略配置

```bash
# 默认使用 rebase 而非 merge
git config --global pull.rebase true

# 只拉取当前分支
git config --global pull.default current

# 自动储藏和恢复
git config --global rebase.autoStash true
```

### 3. git push —— 推送更新

```bash
# 推送当前分支到远程同名分支
git push

# 首次推送并设置上游追踪
git push -u origin main
git push --set-upstream origin main

# 推送到指定远程分支
git push origin local-branch:remote-branch

# 推送所有分支
git push --all origin

# 推送所有标签
git push --tags origin

# 强制推送（危险！覆盖远程历史）
git push --force origin main
git push -f origin main

# 安全强制推送（推荐）
git push --force-with-lease origin main
```

### push 配置

```bash
# 设置默认推送当前分支到同名远程分支
git config --global push.default current

# 设置默认 push 策略为 matching（推送所有同名分支）
git config --global push.default matching
```

## 四、追踪关系

### 1. 设置上游分支

```bash
# 设置当前分支追踪远程分支
git branch --set-upstream-to=origin/main main

# 简写
git branch -u origin/main

# 首次推送时自动设置
git push -u origin feature

# 取消追踪
git branch --unset-upstream
```

### 2. 查看追踪信息

```bash
# 查看分支追踪关系
git branch -vv
# * main    a1b2c3d [origin/main] Latest commit
#   feature def456a [origin/feature: ahead 2, behind 1] My changes

# ahead 2 = 本地比远程多 2 个提交
# behind 1 = 本地比远程少 1 个提交
```

## 五、Fork 工作流

Fork 是 GitHub/GitLab 等平台提供的协作方式，适合开源项目贡献。

### 完整 Fork 流程

```bash
# 1. 在平台上 Fork 原仓库

# 2. 克隆自己的 Fork
git clone https://github.com/your-username/repo.git
cd repo

# 3. 添加原仓库为上游
git remote add upstream https://github.com/original-owner/repo.git

# 4. 同步原仓库更新
git fetch upstream
git checkout main
git merge upstream/main

# 5. 创建功能分支
git checkout -b feature/my-improvement

# 6. 开发并提交
git add .
git commit -m "feat: my improvement"

# 7. 推送到自己的 Fork
git push origin feature/my-improvement

# 8. 在平台上创建 Pull Request
gh pr create --base original-owner:main --head your-username:feature/my-improvement
```

### Fork 同步脚本

```bash
#!/bin/bash
# sync-fork.sh - 同步 Fork 到上游最新状态

echo "Fetching upstream..."
git fetch upstream

echo "Checking out main..."
git checkout main

echo "Merging upstream/main..."
git merge upstream/main

echo "Pushing to origin..."
git push origin main

echo "Done! Fork is now up to date."
```

## 六、多远程仓库

### 典型场景

```bash
# 场景：Fork 项目需要同时跟踪自己的 Fork 和原仓库

git remote -v
# origin    https://github.com/you/repo.git     (你的 Fork)
# upstream  https://github.com/them/repo.git    (原仓库)

# 从原仓库拉取最新
git fetch upstream
git merge upstream/main

# 推送到自己的 Fork
git push origin main
```

### 部署远程仓库

```bash
# 添加生产环境远程仓库
git remote add production ssh://user@server/var/www/app.git

# 推送到生产
git push production main
```
