# 远程分支操作

## 一、远程分支概念

远程分支是本地仓库中对远程仓库分支状态的引用。它们以 `remote/branch` 格式命名，如 `origin/main`。

```
远程分支的工作方式：

远程仓库 origin:
  main ──────► A ── B ── C
  feature ──── D ── E

本地仓库:
  main    ──── A ── B          (本地分支)
  origin/main  ── A ── B ── C  (远程跟踪分支，只读)
  origin/feature ── D ── E     (远程跟踪分支，只读)

执行 git fetch 后：
  origin/main    更新为 A ── B ── C
  origin/feature 更新为 D ── E
  本地分支不变，需要手动合并
```

## 二、查看远程分支

```bash
# 查看所有分支（本地 + 远程）
git branch -a
# * main
#   feature
#   remotes/origin/main
#   remotes/origin/feature
#   remotes/origin/develop

# 只看远程分支
git branch -r
#   remotes/origin/main
#   remotes/origin/feature

# 查看远程分支的最后一次提交
git branch -r -v
#   origin/main    a1b2c3d Latest commit message
#   origin/feature def456a Another commit

# 查看本地与远程的追踪关系
git branch -vv
```

## 三、基于远程分支创建本地分支

```bash
# 方式1：自动追踪同名远程分支
git switch feature
# 如果 origin/feature 存在，自动创建本地分支并追踪

# 方式2：显式创建并追踪
git switch -c feature origin/feature
git checkout -b feature origin/feature

# 方式3：创建不同名的本地分支
git switch -c my-feature origin/feature

# 方式4：基于远程标签创建分支
git switch -c release v1.0.0
```

## 四、同步远程分支

### 1. 获取远程分支信息

```bash
# 获取所有远程更新（不合并）
git fetch origin

# 获取特定分支
git fetch origin main

# 获取所有远程仓库
git fetch --all

# 获取并清理已删除的远程分支引用
git fetch --prune
```

### 2. 合并远程更新

```bash
# 方式1：直接 pull（fetch + merge）
git pull origin main

# 方式2：手动 fetch 后合并
git fetch origin
git merge origin/main

# 方式3：rebase 保持线性
git fetch origin
git rebase origin/main

# 方式4：pull with rebase
git pull --rebase origin main
```

### 3. 推送分支到远程

```bash
# 推送当前分支
git push origin feature

# 首次推送并设置追踪
git push -u origin feature
git push --set-upstream origin feature

# 推送本地分支到远程不同名分支
git push origin local-name:remote-name

# 推送所有本地分支
git push --all origin

# 删除远程分支
git push origin --delete feature
git push origin :feature    # 旧版语法
```

## 五、删除远程跟踪分支

### 1. 清理已删除的远程分支

```bash
# 远程分支被删除后，本地的远程跟踪分支不会自动消失
git remote prune origin

# 预览将要清理的分支
git remote prune origin --dry-run

# fetch 时自动清理
git fetch --prune
git fetch -p
```

### 2. 手动删除远程跟踪分支

```bash
# 删除对远程分支的本地引用
git branch -dr origin/feature
# 或
git branch --delete --remotes origin/feature
```

## 六、推送与拉取配置

### 1. 设置默认推送行为

```bash
# push.default 可选值：

# current - 推送到同名远程分支（推荐）
git config --global push.default current

# matching - 推送所有同名分支
git config --global push.default matching

# simple - 推送当前分支到同名上游分支（Git 2.0 默认）
git config --global push.default simple

# upstream - 推送到追踪的上游分支
git config --global push.default upstream

# nothing - 不推送任何分支
git config --global push.default nothing
```

### 2. 设置默认拉取行为

```bash
# 使用 rebase 而非 merge
git config --global pull.rebase true

# 使用 merge（默认）
git config --global pull.rebase false

# 仅对当前分支生效
git config pull.rebase true
```

## 七、常见远程操作场景

### 场景一：多人同时开发

```bash
# 开发者 A 推送了新提交
# 开发者 B 需要同步

git fetch origin
# 发现 origin/main 有新提交
git log HEAD..origin/main --oneline

# 方式1：rebase（推荐，保持线性）
git rebase origin/main

# 方式2：merge
git merge origin/main

# 解决冲突后推送
git push origin main
```

### 场景二：恢复被删除的远程分支

```bash
# 如果远程分支被意外删除
# 本地还有该分支或通过 reflog 找到

# 查找分支最后的提交
git reflog
# 或
git log --all --oneline | grep "branch-name"

# 基于该提交重建分支并推送
git branch recover-branch abc1234
git push origin recover-branch
```

### 场景三：替换远程分支内容

```bash
# 用本地分支完全替换远程分支（谨慎！）
git push --force origin main

# 更安全的方式
git push --force-with-lease origin main
# --force-with-lease 会检查远程是否有你不知道的新提交
```

### 场景四：将本地分支推送到不同的远程分支

```bash
# 本地 develop 推送到远程 main
git push origin develop:main

# 将本地 feature 推送到远程 release
git push origin feature:release/v1.0
```

## 八、配置多个远程仓库

```bash
# 添加多个远程仓库
git remote add origin https://github.com/you/repo.git
git remote add upstream https://github.com/them/repo.git
git remote add deploy ssh://server/var/www/repo.git

# 分别操作
git fetch origin
git fetch upstream
git push deploy main

# 同时从所有远程仓库获取
git fetch --all

# 同时推送到多个远程仓库
git remote add all https://github.com/you/repo.git
git remote set-url --add --push all https://github.com/you/repo.git
git remote set-url --add --push all https://gitlab.com/you/repo.git
git push all main
```
