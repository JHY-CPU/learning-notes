# 比较差异（git diff）

## 一、diff 的基本概念

`git diff` 用于比较不同状态之间的差异。它可以比较：

- 工作区 vs 暂存区
- 暂存区 vs 最新提交
- 工作区 vs 最新提交
- 任意两次提交之间的差异

```
工作区 ←── git diff ──→ 暂存区 ←── git diff --staged ──→ 最新提交(HEAD)
         (未暂存修改)                    (已暂存修改)
工作区 ←────────── git diff HEAD ──────────→ 最新提交
                     (全部变化)
```

## 二、常用 diff 命令

### 1. 工作区 vs 暂存区

```bash
# 查看未暂存的修改
git diff

# 只看特定文件
git diff filename.txt

# 只看特定目录
git diff src/
```

### 2. 暂存区 vs 最新提交

```bash
# 查看已暂存、将要提交的内容
git diff --staged
git diff --cached    # 等价写法
```

### 3. 工作区 vs 最新提交

```bash
# 查看所有变化（未暂存 + 已暂存）
git diff HEAD
```

### 4. 比较两次提交

```bash
# 比较两个分支
git diff main..feature

# 比较两次提交（用哈希值）
git diff abc1234 def5678

# 比较分支的分叉点
git diff main...feature
# 三个点表示：从 main 和 feature 的共同祖先到 feature 的差异
```

### 5. 比较远程分支

```bash
# 查看本地与远程的差异
git diff main origin/main

# 查看将要推送的内容
git diff origin/main..main
```

## 三、diff 输出格式

### 1. 标准 diff 格式

```diff
diff --git a/file.txt b/file.txt
index 83bae..d2f4a 100644
--- a/file.txt
+++ b/file.txt
@@ -1,5 +1,6 @@
 第一行内容
-第二行被删除的内容
+第二行被修改的内容
+新增的一行
 第三行内容
 第四行内容
```

输出解读：

| 行 | 含义 |
|---|------|
| `diff --git a/file.txt b/file.txt` | 比较的文件 |
| `--- a/file.txt` | 原文件（a 表示旧版本） |
| `+++ b/file.txt` | 新文件（b 表示新版本） |
| `@@ -1,5 +1,6 @@` | 变更位置：原文件第1行起5行，新文件第1行起6行 |
| ` 第一行内容` | 无变化的上下文行 |
| `-第二行被删除的内容` | 删除的行 |
| `+第二行被修改的内容` | 新增的行 |

### 2. 使用不同输出格式

```bash
# 统计模式（只看文件级别的变化数量）
git diff --stat
# 输出：
#  file1.txt | 10 +++++-----
#  file2.txt |  5 +++--
#  2 files changed, 8 insertions(+), 7 deletions(-)

# 短格式
git diff --shortstat
# 输出：2 files changed, 8 insertions(+), 7 deletions(-)

# 仅显示修改的文件名
git diff --name-only

# 显示修改状态（A=新增 M=修改 D=删除）
git diff --name-status

# 词级别差异（更精确）
git diff --word-diff
# 输出：[-旧词-]{+新词+}

# 词级别差异（颜色标记）
git diff --color-words

# 并排比较
git diff --side-by-side
```

## 四、高级用法

### 1. 忽略空白变化

```bash
# 忽略所有空白变化
git diff -w
git diff --ignore-all-space

# 忽略空白数量变化
git diff -b
git diff --ignore-space-change
```

### 2. 比较特定文件

```bash
# 只比较某种类型的文件
git diff -- '*.js'

# 比较特定目录下的文件
git diff -- src/components/

# 排除某些文件
git diff -- ':(exclude)*.lock'
git diff -- ':(exclude)package-lock.json'
```

### 3. 二进制文件

```bash
# 查看二进制文件的差异（显示是否变化）
git diff --binary

# 生成二进制文件的补丁
git diff --binary > changes.patch
```

### 4. 使用外部 diff 工具

```bash
# 使用 vimdiff
git difftool --tool=vimdiff

# 使用 VS Code 作为 diff 工具
git config --global diff.tool vscode
git config --global difftool.vscode.cmd 'code --diff $LOCAL $REMOTE'

# 然后使用
git difftool
```

### 5. 生成补丁文件

```bash
# 生成完整补丁
git diff > changes.patch

# 生成二进制补丁
git diff --binary > changes.patch

# 应用补丁
git apply changes.patch

# 检查补丁是否能应用（不实际应用）
git apply --check changes.patch
```

## 五、常见使用场景

### 场景一：提交前检查

```bash
# 查看将要提交的内容
git diff --staged

# 查看还有哪些修改未暂存
git diff

# 用简洁方式查看修改的文件
git diff --stat --staged
```

### 场景二：代码审查

```bash
# 比较两个分支的所有差异
git diff main..feature-branch

# 查看某个提交引入的变化
git diff abc1234^..abc1234

# 查看某个文件在两个分支的区别
git diff main..feature -- src/app.js
```

### 场景三：查找引入 bug 的提交

```bash
# 比较当前与上一个版本的差异
git diff HEAD~1

# 比较当前与指定版本的差异
git diff abc1234

# 比较某个文件在两个版本的区别
git diff HEAD~5..HEAD -- src/utils.js
```
