# 暂存与提交

## 一、git add —— 添加到暂存区

### 1. 添加单个文件

```bash
# 添加指定文件
git add filename.txt

# 添加指定目录
git add src/

# 添加目录中的特定类型文件
git add src/*.js
```

### 2. 添加多个文件

```bash
# 添加多个指定文件
git add file1.txt file2.txt file3.txt

# 添加所有文件（包括新文件和修改的文件）
git add .

# 添加所有文件（另一种写法）
git add -A
git add --all

# 添加所有已跟踪文件的修改（不包含新文件）
git add -u
git add --update
```

### 三种添加方式的区别

```
场景：有新文件 new.txt，修改了 tracked.txt，删除了 deleted.txt

git add -A    → 暂存 new.txt + tracked.txt + deleted.txt（所有变化）
git add .     → 暂存 new.txt + tracked.txt（新增和修改，不包括删除）
git add -u    → 暂存 tracked.txt + deleted.txt（已跟踪文件的变化）
```

### 3. 交互式添加

```bash
# 交互式添加（选择要暂存的文件）
git add -i
# 输出菜单：
#           staged     unstaged path
#   1:    unchanged        +0/-1 deleted.txt
#   2:    unchanged        +3/-0 tracked.txt
#   3:    untracked        +1/-0 new.txt

# 交互式暂存（逐文件选择部分修改）
git add -p
# 或
git add --patch
```

交互式暂存的常用选项：

```
y - 暂存这个区块（hunk）
n - 不暂存这个区块
s - 分割成更小的区块
e - 手动编辑这个区块
q - 退出
a - 暂存当前文件的所有区块
d - 跳过当前文件的所有区块
```

### 4. 暂存删除操作

```bash
# 删除文件并暂存删除操作
git rm filename.txt

# 仅从 Git 中删除（保留本地文件）
git rm --cached filename.txt

# 删除目录
git rm -r directory/

# 强制删除（忽略 .gitignore）
git rm -f filename.txt
```

### 5. 暂存重命名

```bash
# 重命名文件并暂存
git mv oldname.txt newname.txt

# 等价于：
mv oldname.txt newname.txt
git rm oldname.txt
git add newname.txt
```

## 二、git commit —— 提交更改

### 1. 基本提交

```bash
# 提交暂存区的内容（打开编辑器写提交信息）
git commit

# 直接指定提交信息（推荐）
git commit -m "feat: add user login"

# 多行提交信息
git commit -m "feat: add user login" -m "Implement JWT-based authentication
- Add login endpoint
- Add token generation
- Add password hashing"
```

### 2. 跳过暂存直接提交

```bash
# 将所有已跟踪文件的修改直接提交（跳过 git add）
git commit -a -m "fix: update config"

# 等价于 git add -u + git commit
# 注意：不会包含新文件！新文件仍然需要先 git add
```

### 3. 修改上次提交

```bash
# 修改上次提交的信息
git commit --amend -m "新的提交信息"

# 修改上次提交的内容（添加遗漏的文件）
git add forgotten_file.txt
git commit --amend --no-edit

# 不修改提交信息，只合并文件
git commit --amend --no-edit
```

**注意**：不要修改已经推送到远程的提交！

### 4. 空提交

```bash
# 创建空提交（没有文件变化）
git commit --allow-empty -m "chore: trigger CI pipeline"

# 创建空提交（用于标记里程碑）
git commit --allow-empty -m "release: v1.0.0"
```

## 三、提交信息规范

### 1. 标准格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

### 2. Type 类型

| type | 说明 |
|------|------|
| `feat` | 新功能 |
| `fix` | 修复 bug |
| `docs` | 文档变更 |
| `style` | 代码格式（不影响运行） |
| `refactor` | 重构（非新功能、非修复） |
| `perf` | 性能优化 |
| `test` | 添加测试 |
| `chore` | 构建/工具变更 |
| `ci` | CI 配置变更 |
| `revert` | 回退提交 |

### 3. 示例

```bash
# 简单提交
git commit -m "feat: add user registration"
git commit -m "fix: resolve login timeout issue"
git commit -m "docs: update API documentation"

# 带范围的提交
git commit -m "feat(auth): add OAuth2 login support"
git commit -m "fix(api): handle null pointer in user endpoint"

# 带 body 和 footer 的提交
git commit -m "fix: prevent race condition in data processing

The previous implementation did not acquire the lock before
accessing shared state, causing intermittent data corruption.

Closes #123"
```

### 4. 提交信息的好习惯

```
好的提交信息：
✓ feat: add user authentication with JWT
✓ fix: resolve memory leak in event listener
✓ refactor: extract validation logic into utils

坏的提交信息：
✗ update
✗ fix bug
✗ changes
✗ work in progress
✗ asdfghjkl
```

## 四、查看暂存区与工作区差异

### 1. 查看哪些文件将被提交

```bash
# 查看暂存区内容
git status

# 查看暂存区与上次提交的差异
git diff --staged
git diff --cached    # 等价命令

# 查看工作区与暂存区的差异（未暂存的修改）
git diff

# 查看工作区与上次提交的差异
git diff HEAD
```

### 2. 查看特定文件的差异

```bash
# 查看指定文件的暂存差异
git diff --staged filename.txt

# 查看指定文件的工作区差异
git diff filename.txt
```

## 五、撤消暂存

```bash
# 取消暂存某个文件（保留工作区修改）
git restore --staged filename.txt
# 或旧版命令
git reset HEAD filename.txt

# 取消暂存所有文件
git restore --staged .

# 取消暂存 + 丢弃工作区修改
git restore --staged filename.txt
git restore filename.txt

# 丢弃所有工作区修改（危险！不可恢复）
git restore .
# 或
git checkout -- .
```

## 六、提交流程最佳实践

```bash
# 标准开发流程
# 1. 查看当前状态
git status

# 2. 查看修改内容
git diff

# 3. 交互式暂存（推荐，精确控制提交内容）
git add -p

# 4. 确认暂存内容
git diff --staged

# 5. 提交
git commit -m "feat: describe your change"

# 6. 验证提交
git log -1 --stat
```
