# 初始化与克隆仓库

## 一、初始化仓库

### 1. 在现有目录中初始化

```bash
# 进入项目目录
cd /path/to/project

# 初始化 Git 仓库
git init

# 输出: Initialized empty Git repository in /path/to/project/.git/
```

此时会创建 `.git` 子目录，包含 Git 所需的所有仓库数据。

```bash
# 指定默认分支名（Git 2.28+）
git init -b main

# 静默初始化（不输出信息）
git init -q
```

### 2. 创建裸仓库

裸仓库没有工作目录，通常用于远程服务器：

```bash
# 创建裸仓库（通常以 .git 结尾）
git init --bare myproject.git

# 裸仓库目录结构
myproject.git/
├── HEAD
├── config
├── hooks/
├── info/
├── objects/
└── refs/
# 注意：没有 .git 子目录，没有工作区文件
```

### 3. 初始化并添加初始文件

```bash
git init
echo "# My Project" > README.md
git add README.md
git commit -m "Initial commit"
```

## 二、克隆远程仓库

### 1. 基本克隆

```bash
# 克隆远程仓库（完整复制）
git clone https://github.com/user/repo.git

# 克隆到指定目录名
git clone https://github.com/user/repo.git my-project

# 克隆后目录结构
my-project/
├── .git/           ← 完整的仓库数据
├── README.md
└── ...
```

### 2. 克隆特定分支

```bash
# 只克隆特定分支
git clone -b develop https://github.com/user/repo.git

# 只克隆特定分支且不检出
git clone -b develop --no-checkout https://github.com/user/repo.git
```

### 3. 浅克隆

当仓库历史很长时，可以只克隆最近的提交：

```bash
# 只克隆最近 1 次提交（极大减少下载量）
git clone --depth 1 https://github.com/user/repo.git

# 克隆最近 10 次提交
git clone --depth 10 https://github.com/user/repo.git

# 浅克隆后转为完整仓库
git fetch --unshallow
```

### 4. 部分克隆（稀疏检出）

只检出仓库中的部分目录：

```bash
# 克隆但不检出文件
git clone --no-checkout --filter=blob:none https://github.com/user/repo.git
cd repo

# 启用稀疏检出
git sparse-checkout init --cone

# 只检出指定目录
git sparse-checkout set src/docs

# 查看稀疏检出配置
git sparse-checkout list
```

### 5. 克隆子模块

```bash
# 克隆并初始化子模块
git clone --recurse-submodules https://github.com/user/repo.git

# 已克隆的仓库初始化子模块
git submodule update --init --recursive
```

## 三、仓库信息

### 1. 查看仓库状态

```bash
# 查看当前状态
git status

# 简洁模式
git status -s

# 查看分支和跟踪信息（简短格式）
git status -b
```

### 2. 查看仓库信息

```bash
# 查看远程仓库地址
git remote -v

# 查看当前分支
git branch --show-current

# 查看仓库根目录
git rev-parse --show-toplevel

# 查看 .git 目录位置
git rev-parse --git-dir

# 判断当前目录是否在 Git 仓库中
git rev-parse --is-inside-work-tree
# 输出: true 或 fatal: not a git repository
```

### 3. 查看提交计数

```bash
# 总提交数
git rev-list --count HEAD

# 各分支提交数
for branch in $(git branch -r | grep -v HEAD); do
    echo -n "$branch: "
    git rev-list --count "$branch"
done
```

## 四、基础操作流程

### 完整的初始操作流程

```bash
# 1. 初始化仓库
mkdir my-project && cd my-project
git init

# 2. 创建第一个文件
echo "# My Project" > README.md

# 3. 查看状态
git status
# On branch main
# No commits yet
# Untracked files:
#   README.md

# 4. 添加到暂存区
git add README.md

# 5. 再次查看状态
git status
# Changes to be committed:
#   new file: README.md

# 6. 提交
git commit -m "Initial commit: add README"

# 7. 查看提交历史
git log --oneline
# a1b2c3d (HEAD -> main) Initial commit: add README
```

### 从远程仓库开始

```bash
# 1. 克隆仓库
git clone https://github.com/user/project.git
cd project

# 2. 查看远程仓库信息
git remote -v
# origin  https://github.com/user/project.git (fetch)
# origin  https://github.com/user/project.git (push)

# 3. 查看当前分支和状态
git branch -a
git status

# 4. 开始开发
echo "new feature" > feature.txt
git add feature.txt
git commit -m "feat: add new feature"
git push origin main
```

## 五、删除仓库

```bash
# 删除本地仓库（只需删除 .git 目录）
rm -rf .git

# 删除远程仓库需要在 GitHub/GitLab 网页上操作
# 或使用 CLI 工具：
gh repo delete owner/repo --yes  # 需要安装 GitHub CLI
```
