# 常见问题与解决方案

## 一、提交相关问题

### 问题1：忘记添加文件到上次提交

```bash
# 方案1：amend（本地提交）
git add forgotten-file.txt
git commit --amend --no-edit

# 方案2：如果已经推送，创建新提交
git add forgotten-file.txt
git commit -m "chore: add forgotten file from previous commit"
git push origin main
```

### 问题2：提交信息写错了

```bash
# 修改最近一次提交信息（未推送）
git commit --amend -m "correct commit message"

# 修改已推送的提交信息（危险！需要 force push）
git commit --amend -m "correct commit message"
git push --force-with-lease origin main

# 如果已推送且是公共分支，不要修改！
# 保持错误的信息，下次提交改正
```

### 问题3：提交到了错误的分支

```bash
# 场景：本应在 feature 分支提交，却提交到了 main

# 方案1：移动提交到正确分支
git log --oneline -1  # 记下提交哈希
git reset --hard HEAD~1  # 从 main 撤销
git switch feature
git cherry-pick abc1234  # 应用到 feature

# 方案2：使用 reflog 恢复
git reflog  # 找到之前的 HEAD 位置
git reset --hard HEAD@{1}

# 方案3：如果已经推送到 main
git revert HEAD  # 安全地撤销已推送的提交
git switch feature
git cherry-pick abc1234
```

## 二、分支相关问题

### 问题4：分支名拼写错误

```bash
# 重命名本地分支
git branch -m old-name new-name

# 如果是当前分支
git branch -m new-name

# 更新远程分支
git push origin --delete old-name
git push -u origin new-name
```

### 问题5：无法切换分支（有未提交的修改）

```bash
# 查看哪些文件阻止切换
git status

# 方案1：储藏修改
git stash push -m "WIP: save before switch"
git switch other-branch
# ... 工作完成后 ...
git switch original-branch
git stash pop

# 方案2：提交修改
git add .
git commit -m "WIP: save before switch"
git switch other-branch

# 方案3：强制切换（丢失修改！）
git switch -f other-branch
```

### 问题6：删除了分支想恢复

```bash
# 通过 reflog 查找分支最后的提交
git reflog | grep "branch-name"

# 或查看所有分支的操作记录
git reflog --all

# 基于找到的提交重新创建分支
git branch recovered-branch abc1234
```

## 三、合并相关问题

### 问题7：合并冲突太多，想放弃合并

```bash
# 中止合并
git merge --abort

# 或中止变基
git rebase --abort

# 如果合并已经完成，回退
git reset --hard HEAD~1
```

### 问题8：合并后发现有问题，想撤销合并

```bash
# 如果合并还未推送
git reset --hard HEAD~1

# 如果已经推送（需要新提交来撤销）
git revert -m 1 <merge-commit-hash>
# -m 1 表示保留第一个父提交（主分支）的内容
```

### 问题9：cherry-pick 产生冲突

```bash
# cherry-pick 冲突后
git add resolved-files
git cherry-pick --continue

# 放弃 cherry-pick
git cherry-pick --abort

# 跳过当前提交
git cherry-pick --skip
```

## 四、远程相关问题

### 问题10：推送被拒绝

```bash
# ! [rejected] main -> main (non-fast-forward)

# 原因：远程有你本地没有的新提交

# 方案1：先拉取再推送
git pull --rebase origin main
git push origin main

# 方案2：如果确定要用本地版本覆盖远程
git push --force-with-lease origin main
```

### 问题11：拉取时产生冲突

```bash
# pull 冲突
git pull --rebase origin main
# CONFLICT...

# 解决冲突
git add resolved-files
git rebase --continue

# 或中止拉取
git rebase --abort
```

### 问题12：远程仓库地址变了

```bash
# 查看当前远程地址
git remote -v

# 修改远程地址
git remote set-url origin https://github.com/new/repo.git

# 或使用 SSH
git remote set-url origin git@github.com:new/repo.git
```

## 五、文件相关问题

### 问题13：文件被 Git 跟踪但应该被忽略

```bash
# 文件已经进入版本控制，即使加入 .gitignore 也不会被忽略

# 解决：从跟踪中移除
git rm --cached filename.txt

# 移除整个目录
git rm --cached -r directory/

# 添加到 .gitignore
echo "filename.txt" >> .gitignore
git add .gitignore
git commit -m "chore: stop tracking and ignore filename.txt"
```

### 问题14：误删了文件

```bash
# 从最后一次提交恢复
git restore filename.txt

# 从指定提交恢复
git restore --source=abc1234 filename.txt

# 恢复整个目录
git restore --source=HEAD -- src/
```

### 问题15：工作区有大量未跟踪文件想清理

```bash
# 查看将要删除的文件
git clean -n

# 删除未跟踪的文件
git clean -f

# 删除未跟踪的文件和目录
git clean -fd

# 删除未跟踪和被 .gitignore 忽略的文件
git clean -fdx
```

## 六、仓库相关问题

### 问题16：仓库太大

```bash
# 分析仓库大小
git count-objects -vH

# 找出最大的文件
git rev-list --objects --all \
  | git cat-file --batch-check='%(objecttype) %(objectname) %(objectsize) %(rest)' \
  | sed -n 's/^blob //p' \
  | sort -rnk2 \
  | head -20

# 清理
git reflog expire --expire=now --all
git gc --prune=now --aggressive

# 使用 BFG 清理大文件
java -jar bfg.jar --strip-blobs-bigger-than 10M
git reflog expire --expire=now --all
git gc --prune=now --aggressive
```

### 问题17：.git 目录损坏

```bash
# 检查仓库完整性
git fsck

# 尝试修复
git fsck --full

# 如果对象丢失，从远程重新克隆
git clone https://github.com/user/repo.git new-repo
cd new-repo
# 将本地修改复制过来
```

### 问题18：想清除所有历史重新开始

```bash
# 方案1：删除 .git 重新初始化
rm -rf .git
git init
git add .
git commit -m "Initial commit"

# 方案2：创建孤儿分支
git checkout --orphan fresh-start
git add -A
git commit -m "Initial commit"
git branch -D main
git branch -m main
git push -f origin main
```

## 七、性能相关问题

### 问题19：Git 操作很慢

```bash
# 查看仓库状态
git count-objects -v

# 垃圾回收
git gc

# 打包对象
git repack -a -d

# 使用文件系统监控（Git 2.22+）
git config core.fsmonitor true
git config core.untrackedcache true
```

### 问题20：克隆大仓库太慢

```bash
# 浅克隆
git clone --depth 1 https://github.com/user/repo.git

# 部分克隆
git clone --filter=blob:none https://github.com/user/repo.git

# 稀疏检出
git clone --no-checkout https://github.com/user/repo.git
cd repo
git sparse-checkout init --cone
git sparse-checkout set src/docs
git checkout
```

## 八、快速参考卡

```
日常操作：
  git status          查看状态
  git add -p          交互式暂存
  git commit -m ""    提交
  git pull --rebase   拉取更新
  git push            推送

分支操作：
  git switch -c feat  创建并切换分支
  git branch -d feat  删除分支
  git merge feat      合并分支
  git rebase main     变基

撤销操作：
  git restore file    丢弃工作区修改
  git restore --staged file  取消暂存
  git reset HEAD~1    撤销上次提交
  git revert abc1234  撤销已推送提交

恢复数据：
  git reflog          查看操作历史
  git stash pop       恢复储藏
  git cherry-pick     挑选提交

查看信息：
  git log --oneline   简洁日志
  git diff            查看差异
  git blame file      逐行追溯
  git bisect          二分查找 bug
```
