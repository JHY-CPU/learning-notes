# 高级操作技巧

## 一、Cherry-pick —— 挑选提交

将某个特定提交应用到当前分支。

```bash
# 将单个提交应用到当前分支
git cherry-pick abc1234

# 应用多个提交
git cherry-pick abc1234 def5678 789i0j1

# 应用一系列提交
git cherry-pick abc1234..def5678  # 不包含 abc1234
git cherry-pick abc1234^..def5678 # 包含 abc1234

# 不自动提交（手动调整后提交）
git cherry-pick --no-commit abc1234
git cherry-pick -n abc1234

# 应用时保留原作者信息
git cherry-pick -x abc1234
# 提交信息会包含 (cherry picked from commit abc1234...)

# 处理冲突
git cherry-pick abc1234
# CONFLICT...
git add .
git cherry-pick --continue

# 中止 cherry-pick
git cherry-pick --abort
```

### 使用场景

```
场景：main 分支需要修复，但 fix 提交在 develop 分支上

develop: A ── B ── C (fix commit)
main:    X ── Y

# 不想合并整个 develop，只想拿 fix 提交
git switch main
git cherry-pick C

# 结果：
main: X ── Y ── C' (C 的副本)
```

## 二、Bisect —— 二分查找 Bug

`git bisect` 通过二分法快速定位引入 bug 的提交。

### 1. 手动二分

```bash
# 开始二分查找
git bisect start

# 标记当前版本有 bug
git bisect bad

# 标记已知正常的版本
git bisect good v1.0.0

# Git 自动检出中间版本
# 测试后标记
git bisect good  # 这个版本正常
git bisect bad   # 这个版本有 bug

# 重复直到找到问题提交
# Git 会输出：
# abc1234 is the first bad commit

# 结束二分查找
git bisect reset
```

### 2. 自动二分

```bash
# 使用脚本自动测试
# 脚本返回 0 表示 good，非 0 表示 bad
git bisect start
git bisect bad HEAD
git bisect good v1.0.0
git bisect run ./test-script.sh

# test-script.sh 示例：
#!/bin/bash
npm test
# 测试通过返回 0 (good)，失败返回非 0 (bad)

# 结束
git bisect reset
```

### 3. 完整流程示例

```bash
# 当前版本有 bug
git bisect start
git bisect bad

# 3个版本前是好的
git bisect good HEAD~20

# 测试当前版本...
npm test  # 失败
git bisect bad

# 测试新检出的版本...
npm test  # 通过
git bisect good

# 重复几次后找到
# abc1234 is the first bad commit

# 查看该提交详情
git show abc1234

# 恢复到当前分支
git bisect reset
```

## 三、Worktree —— 多工作区

Worktree 允许在不同目录中同时检出同一仓库的不同分支。

```bash
# 创建新的工作区
git worktree add ../project-hotfix hotfix/fix-crash

# 查看所有工作区
git worktree list
# /path/to/project    abc1234 [main]
# /path/project-hotfix def4567 [hotfix/fix-crash]

# 在新工作区中工作
cd ../project-hotfix
# 修改、提交...

# 删除工作区
git worktree remove ../project-hotfix

# 清理已删除的工作区引用
git worktree prune
```

### 使用场景

```
场景：main 分支需要紧急修复，同时 feature 分支的开发不想中断

主工作区: /project (feature 分支，继续开发)
副工作区: /project-hotfix (hotfix 分支，修复 bug)

两个目录指向同一个 .git 仓库，互不干扰
```

## 四、Filter-Branch —— 历史重写

### 1. 从历史中删除文件

```bash
# 删除大文件
git filter-branch --force --index-filter \
  'git rm --cached --ignore-unmatch large-file.zip' \
  --prune-empty --tag-name-filter cat -- --all

# 使用 BFG Repo-Cleaner（更快）
# 下载 BFG
java -jar bfg.jar --delete-files large-file.zip
java -jar bfg.jar --delete-folders node_modules
java -jar bfg.jar --replace-text passwords.txt
git reflog expire --expire=now --all
git gc --prune=now --aggressive
```

### 2. 修改历史中的作者信息

```bash
# 修改所有提交的作者邮箱
git filter-branch --env-filter '
if [ "$GIT_AUTHOR_EMAIL" = "old@email.com" ]; then
    export GIT_AUTHOR_EMAIL="new@email.com"
    export GIT_AUTHOR_NAME="New Name"
fi
' --tag-name-filter cat -- --all
```

### 3. 修改提交信息

```bash
# 修改提交信息格式
git filter-branch --msg-filter '
sed -e "s/feat:/feature:/g"
' -- --all
```

**警告**：`filter-branch` 会重写所有历史，执行前务必备份仓库。

## 五、高级 Diff 技巧

### 1. 三种 Diff 算法

```bash
# Myers 算法（默认，速度快）
git diff --diff-algorithm=myers

# patience 算法（更易读）
git diff --diff-algorithm=patience

# histogram 算法（patience 的改进版）
git diff --diff-algorithm=histogram

# minimal 算法（产生最小差异）
git diff --diff-algorithm=minimal
```

### 2. 归咎（Blame）高级用法

```bash
# 忽略空白
git blame -w filename.txt

# 检测代码移动
git blame -C filename.txt

# 检测跨文件的代码移动
git blame -C -C filename.txt

# 查看特定行范围
git blame -L 10,20 filename.txt

# 使用正则表达式忽略某些行
git blame -L '/function start/,/^}/' filename.txt
```

### 3. 代码搜索

```bash
# 搜索代码历史中的字符串
git log -S 'functionName' --oneline
# 找到添加或删除了 'functionName' 的提交

# 使用正则表达式搜索
git log -G 'function\s+\w+' --oneline

# 搜索提交信息
git log --grep='login' --oneline

# 搜索作者
git log --author='Alice' --oneline
```

## 六、高级 Log 技巧

### 1. 自定义格式

```bash
# 表格格式日志
git log --format="
| %h | %an | %ar | %s |
|---|---|---|---|
" -10

# 彩色日志
git log --format="%C(red)%h%Creset %C(yellow)%an%Creset %C(green)%ar%Creset %s" -10

# JSON 格式
git log --format='{"hash":"%H","author":"%an","date":"%ad","message":"%s"}' -5
```

### 2. 贡献分析

```bash
# 每周提交数
git log --format='%ai' | awk '{print $1}' | sort | uniq -c

# 按小时分布的提交数
git log --format='%ai' | awk '{print substr($2,1,2)}' | sort | uniq -c | sort -k2

# 最常修改的文件
git log --pretty=format: --name-only | sort | uniq -c | sort -rn | head -20

# 每个作者的代码变更量
git shortlog -sn --all
```

## 七、Hooks —— Git 钩子

### 1. 客户端钩子

```bash
# .git/hooks/pre-commit（提交前执行）
#!/bin/bash
npm run lint
if [ $? -ne 0 ]; then
    echo "Lint failed! Commit aborted."
    exit 1
fi

# .git/hooks/commit-msg（检查提交信息）
#!/bin/bash
if ! grep -qE '^(feat|fix|docs|style|refactor|perf|test|chore|ci)\(.*\): .+' "$1"; then
    echo "Invalid commit message format!"
    exit 1
fi

# .git/hooks/post-commit（提交后执行）
#!/bin/bash
echo "Commit successful!"
```

### 2. 使用 Husky 管理钩子

```bash
# 安装 husky
npm install -D husky
npx husky init

# 添加钩子
npx husky add .husky/pre-commit 'npm run lint'
npx husky add .husky/commit-msg 'npx commitlint --edit $1'
npx husky add .husky/pre-push 'npm test'
```

### 3. 服务端钩子

```bash
# 服务器端 .git/hooks/pre-receive
#!/bin/bash
# 拒绝直接推送到 main
while read oldrev newrev refname; do
    if [ "$refname" = "refs/heads/main" ]; then
        echo "Direct push to main is not allowed!"
        exit 1
    fi
done
```

## 八、实用技巧集合

### 1. 批量操作

```bash
# 批量替换文件中的字符串
git ls-files '*.js' | xargs sed -i 's/oldFunction/newFunction/g'

# 批量删除所有 .DS_Store 文件
find . -name ".DS_Store" -print0 | xargs -0 git rm --cached
echo ".DS_Store" >> .gitignore
git add .
git commit -m "chore: remove .DS_Store files"
```

### 2. 搜索与替换

```bash
# 搜索包含特定字符串的文件
git grep "TODO" -- "*.js"

# 搜索并显示行号
git grep -n "FIXME"

# 在特定提交中搜索
git grep "old code" v1.0.0

# 统计匹配数量
git grep -c "import" -- "*.js"
```

### 3. 性能优化

```bash
# 查看最大的文件
git rev-list --objects --all \
  | git cat-file --batch-check='%(objecttype) %(objectname) %(objectsize) %(rest)' \
  | sed -n 's/^blob //p' \
  | sort -rnk2 \
  | head -20

# 清理仓库
git reflog expire --expire=now --all
git gc --prune=now --aggressive

# 浅克隆（只保留最近的提交）
git clone --depth 1 https://github.com/user/repo.git
```
