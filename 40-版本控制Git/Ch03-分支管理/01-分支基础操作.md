# 分支基础操作

## 一、什么是分支

Git 分支本质上是指向某个提交的可变指针。默认分支通常叫 `main`（或旧版本叫 `master`）。

```
提交历史示意：

A ── B ── C ── D    ← main (HEAD)
     │
     └── E ── F     ← feature-login

每个字母代表一次提交，分支名指向该分支最新的提交
HEAD 指针指向当前所在的分支
```

### HEAD 指针

`HEAD` 是一个特殊的指针，指向当前所在分支的最新提交：

```bash
# 查看 HEAD 指向
cat .git/refs/heads/main
# 输出: a1b2c3d4e5f6...（提交哈希）

# 查看 HEAD 文件
cat .git/HEAD
# 输出: ref: refs/heads/main

# 如果处于分离 HEAD 状态
# 输出: a1b2c3d4e5f6...（直接指向提交哈希，而非分支）
```

## 二、查看分支

```bash
# 查看本地分支（* 标记当前所在分支）
git branch
# * main
#   feature
#   develop

# 查看所有分支（本地 + 远程）
git branch -a
# * main
#   feature
#   remotes/origin/main
#   remotes/origin/feature

# 查看远程分支
git branch -r
#   remotes/origin/main
#   remotes/origin/feature

# 查看分支最后一次提交信息
git branch -v
# * main    a1b2c3d Initial commit
#   feature def456a Add login page

# 查看已合并到当前分支的分支
git branch --merged

# 查看未合并到当前分支的分支
git branch --merged

# 查看包含某次提交的分支
git branch --contains abc1234
```

## 三、创建分支

### 1. 创建分支（不切换）

```bash
# 基于当前分支创建新分支
git branch feature-login

# 基于指定提交创建分支
git branch feature-login abc1234

# 基于指定标签创建分支
git branch feature-login v1.0.0

# 基于其他分支创建
git branch feature-login develop
```

### 2. 创建并切换（推荐）

```bash
# 创建并切换到新分支（Git 2.23+ 推荐）
git switch -c feature-login

# 旧版命令（仍然广泛使用）
git checkout -b feature-login

# 基于指定提交创建并切换
git switch -c feature-login abc1234
git checkout -b feature-login abc1234

# 基于远程分支创建本地分支
git switch -c feature origin/feature
git checkout -b feature origin/feature
```

### 3. 创建孤儿分支

孤儿分支没有历史提交，适合用于创建全新的起点（如 `gh-pages`）：

```bash
# 创建孤儿分支
git switch --orphan gh-pages

# 孤儿分支此时没有文件，需要手动添加
echo "<html>...</html>" > index.html
git add .
git commit -m "Initial gh-pages commit"
```

## 四、切换分支

```bash
# 切换分支（Git 2.23+ 推荐）
git switch feature-login

# 旧版命令
git checkout feature-login

# 切换到上一个分支
git switch -
git checkout -

# 强制切换（丢弃本地修改）
git switch -f feature-login
git checkout -f feature-login
```

### 切换分支时的注意事项

```bash
# 如果工作区有未提交的修改，Git 会阻止切换
git switch feature
# error: Your local changes to the following files would be overwritten by checkout:
#   modified.txt
# Please commit your changes or stash them before you switch branches.

# 解决方案：
# 方案1：先提交
git add .
git commit -m "wip: save current work"

# 方案2：储藏（stash）
git stash
git switch feature

# 方案3：强制切换（丢弃修改，危险！）
git switch -f feature
```

## 五、重命名分支

```bash
# 重命名当前分支
git branch -m new-name

# 重命名指定分支
git branch -m old-name new-name

# 更新远程分支名
# 1. 先重命名本地分支
git branch -m old-name new-name
# 2. 删除远程旧分支
git push origin --delete old-name
# 3. 推送新分支
git push origin new-name
# 4. 设置上游追踪
git push -u origin new-name
```

## 六、删除分支

```bash
# 删除已合并的分支
git branch -d feature-login

# 强制删除未合并的分支（危险！丢失未合并的提交）
git branch -D feature-login

# 删除远程分支
git push origin --delete feature-login
git push origin :feature-login     # 旧版语法

# 批量删除已合并的分支
git branch --merged | grep -v "\*\|main\|develop" | xargs -n 1 git branch -d
```

## 七、分离 HEAD 状态

分离 HEAD 是指 HEAD 直接指向一个提交而不是分支：

```bash
# 进入分离 HEAD 状态
git checkout abc1234
# You are in 'detached HEAD' state...

# 此时可以自由修改和提交，但切换分支后这些提交可能丢失

# 保存分离 HEAD 状态下的提交
git branch temp-branch    # 基于当前位置创建分支

# 或直接切换并创建分支
git switch -c new-branch
```

### 分离 HEAD 的风险与处理

```bash
# 场景：在分离 HEAD 状态下做了提交
git checkout abc1234          # 分离 HEAD
echo "test" > test.txt
git add test.txt
git commit -m "test commit"  # 提交存在于分离状态

# 如果直接切换分支，提交会变成"悬空"提交
git switch main
# 警告：你在分离 HEAD 下的提交将会丢失！

# 保存方法：通过 reflog 找回
git reflog
# abc1234 HEAD@{0}: commit: test commit
git branch saved-work abc1234  # 基于该提交创建分支
```

## 八、分支命名规范

### 推荐格式

```
<类型>/<简短描述>

常用类型：
├── feature/    或 feat/    → 新功能
├── bugfix/     或 fix/     → 修复 bug
├── hotfix/                 → 紧急修复
├── release/                → 发布分支
├── docs/                   → 文档更新
├── refactor/               → 重构
├── test/                   → 测试相关
├── chore/                  → 杂务
└── experiment/             → 实验性分支

示例：
feature/user-authentication
feature/add-dark-mode
bugfix/fix-login-timeout
hotfix/security-patch-v2
release/v1.2.0
docs/update-api-reference
```

### 命名规则

```
好的命名：
✓ feature/user-login
✓ fix/header-alignment
✓ release/v2.0.0

坏的命名：
✗ my-branch          （含义不清）
✗ test               （含义不清）
✗ feature/new-stuff  （太模糊）
✗ VERY-LONG-NAME...  （太长）
✗ feature/user login （包含空格）
```
