# 日常开发工作流

## 一、标准开发流程

### 1. 开始新功能

```bash
# 1. 确保在最新的 main 上
git switch main
git pull origin main

# 2. 创建功能分支
git switch -c feature/user-authentication

# 3. 开始开发...
```

### 2. 开发过程中的提交

```bash
# 查看当前修改
git status
git diff

# 交互式暂存（推荐）
git add -p

# 编写符合规范的提交信息
git commit -m "feat(auth): add JWT token generation"

# 持续开发...
git add -p
git commit -m "feat(auth): add password hashing with bcrypt"
git add -p
git commit -m "test(auth): add login endpoint tests"
```

### 3. 同步远程更新

```bash
# 获取最新代码
git fetch origin

# 变基到最新的 main（保持线性历史）
git rebase origin/main

# 如果有冲突，解决后继续
# git add .
# git rebase --continue
```

### 4. 推送与合并

```bash
# 推送功能分支
git push origin feature/user-authentication

# 如果变基过，需要 force push
git push --force-with-lease origin feature/user-authentication

# 创建 Pull Request
gh pr create --title "feat: add user authentication" --body "..."

# 合并后清理
git switch main
git pull origin main
git branch -d feature/user-authentication
```

## 二、完整的开发周期

```
日常开发流程图：

git pull main ──► git switch -c feat ──► 开发 ──► git add -p ──► git commit
      │                                                    │
      │              git fetch + rebase main ◄─────────────┘
      │                          │
      │                    git push feat
      │                          │
      │                    创建 PR ──► Code Review
      │                          │
      │                    合并 PR ──► 删除远程分支
      │                          │
      └──────────────── git pull main（开始下一个循环）
```

## 三、快捷操作

### 1. 快速提交流程

```bash
# 一步完成暂存+提交（已跟踪文件）
git commit -a -m "fix: resolve login timeout"

# 添加所有文件并提交
git add . && git commit -m "feat: add search feature"

# 修改上次提交（添加遗漏的文件）
git add forgotten.txt
git commit --amend --no-edit
```

### 2. 快速同步

```bash
# 一步完成 fetch + rebase
git pull --rebase origin main

# 配置为默认行为
git config --global pull.rebase true

# 之后只需
git pull
```

### 3. 快速分支切换

```bash
# 切换到上一个分支
git switch -
git checkout -

# 查看最近使用的分支
git reflog | grep "checkout:" | head -5
```

## 四、多任务并行开发

### 场景：同时开发多个功能

```bash
# 开始功能 A
git switch main
git switch -c feature/search
# 开发中...

# 紧急需要切换到功能 B
git stash push -m "WIP: search feature"
git switch main
git switch -c feature/payment
# 开发并提交...

# 切换回功能 A
git switch feature/search
git stash pop
# 继续开发...
```

### 场景：中途需要修复 Bug

```bash
# 正在 feature 分支开发
git stash push -m "WIP: feature work"

# 创建 hotfix 分支
git switch main
git switch -c hotfix/fix-crash
# 修复并提交
git commit -am "fix: resolve application crash"
git switch main
git merge --no-ff hotfix/fix-crash
git push origin main

# 回到 feature 继续开发
git switch feature
git stash pop
```

## 五、日常提交规范

### 提交频率

```
推荐：
✓ 每完成一个小的逻辑单元就提交
✓ 每个提交只做一件事
✓ 提交信息清楚说明做了什么

避免：
✗ 一天结束才一次性提交所有代码
✗ 一个提交包含多个不相关的改动
✗ 提交半成品代码到 main
```

### 好的提交示例

```bash
# 细粒度提交
git commit -m "feat(auth): add password validation"
git commit -m "feat(auth): add JWT token generation"
git commit -m "test(auth): add login endpoint tests"
git commit -m "docs(auth): update API documentation"

# 一个提交做一件事
git commit -m "fix: handle null pointer in user lookup"
git commit -m "refactor: extract validation logic to utils"
git commit -m "perf: optimize database query in reports"
```

### 坏的提交示例

```bash
# 不好的提交
git commit -m "update"
git commit -m "fix bug"
git commit -m "lots of changes"
git commit -m "WIP"  # 可以接受在 feature 分支，不要推送到 main
```

## 六、常用快捷命令组合

```bash
# 查看今天的工作
git log --oneline --since="today" --author="$(git config user.name)"

# 查看本周的改动
git log --oneline --since="1 week ago" --stat

# 查看当前分支与 main 的差异
git log main..HEAD --oneline

# 查看还有哪些文件有未提交的修改
git status -s

# 快速查看将要推送的内容
git diff origin/main..HEAD --stat

# 撤销所有本地修改（危险！）
git restore . && git clean -fd
```

## 七、开发环境配置

### .gitconfig 推荐配置

```ini
[user]
    name = Your Name
    email = your@email.com

[core]
    editor = code --wait
    autocrlf = true
    fileMode = false

[pull]
    rebase = true

[push]
    default = current

[init]
    defaultBranch = main

[alias]
    st = status -s
    co = checkout
    br = branch
    ci = commit
    lg = log --oneline --graph --decorate --all
    ll = log --oneline --graph --decorate -20
    amend = commit --amend --no-edit
    undo = reset HEAD~1 --mixed
    wip = !git add -A && git commit -m 'WIP'
    unwip = !git log -1 --format='%s' | grep -q 'WIP' && git reset HEAD~1
    cleanup = !git branch --merged main | grep -v 'main' | xargs -n 1 git branch -d
```
