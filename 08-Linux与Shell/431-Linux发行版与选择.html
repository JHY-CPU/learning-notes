🐧 Linux 发行版与选择

Linux 发行版分类、Debian vs RedHat 体系、服务器发行版选择、桌面环境。

什么是 Linux 发行版

Linux 只是一个内核 (Kernel),发行版 = Linux 内核 + 软件包管理 + 工具集 + 桌面环境。

// ========== 主要发行版体系 ========== // // Debian 体系: // 包管理器: apt (apt-get) // 包格式: .deb // 代表: Debian, Ubuntu, Linux Mint, Kali // // Red Hat 体系: // 包管理器: yum / dnf // 包格式: .rpm // 代表: RHEL, Fedora, CentOS, Rocky Linux // // Arch 体系: // 包管理器: pacman // 包格式: .pkg.tar.zst // 代表: Arch Linux, Manjaro, EndeavourOS // // 独立发行版: // openSUSE (zypper) // Alpine (apk) — 容器首选 // Gentoo (portage) — 源码编译 // ========== 服务器发行版推荐 ========== // Ubuntu Server LTS: // 最流行,社区支持好,文档丰富 // 每 2 年发布 LTS 版本 (长期支持 5-10 年) // 当前: Ubuntu 22.04 LTS / 24.04 LTS // // Debian Stable: // 极其稳定,安全性好 // 包较旧但经过严格测试 // // Rocky Linux / AlmaLinux: // RHEL 兼容,免费 // 企业级稳定 // // Alpine Linux: // 极小 (~5MB), 安全 // Docker 容器首选镜像

桌面 vs 服务器

// ========== 桌面 Linux ========== // Ubuntu Desktop: 友好的 GUI,新手推荐 // Linux Mint: 更传统桌面,Windows 用户友好 // Fedora Workstation: 新技术率先采用 // Pop!_OS: 针对开发者优化 // Manjaro: Arch 的友好版本 // ========== 服务器 Linux ========== // Ubuntu Server: 生态最好,文档最全 // Debian: 最稳定,最省资源 // Rocky Linux: 企业级,RHEL 兼容 // Alpine: 容器/嵌入式 // ========== 如何选择 ========== // 后端开发新手: // → Ubuntu 22.04/24.04 LTS (桌面或服务器) // 最多资料,最大社区,最容易找到帮助 // // Docker 容器: // → Alpine (最小镜像) // → Ubuntu (兼容性最好) // // 企业生产环境: // → Ubuntu LTS // → Rocky Linux / AlmaLinux // // 开发/学习: // → Fedora (最新技术) // → Manjaro (滚动更新)

安装方式

// ========== 安装 Linux 的方式 ========== // // 1. WSL2 (Windows Subsystem for Linux) — 推荐初学者 // Windows 上直接运行 Linux 终端 // $ wsl --install -d Ubuntu-22.04 // 文件系统互通,性能接近原生 // // 2. 虚拟机 (VirtualBox / VMware) // 在现有系统中运行 Linux 虚拟机 // 适合: 测试,学习,隔离环境 // // 3. 双系统 (Dual Boot) // 硬盘分区同时安装 Windows 和 Linux // 启动时选择系统 // 适合: 需要高性能,想完全切换 // // 4. 云服务器 (VPS) // AWS EC2, DigitalOcean Droplet, Vultr // $5-10/月,可远程 SSH 连接 // 适合: 生产环境,学习服务器管理 // ========== WSL2 常用命令 ========== // $ wsl --list --verbose # 查看已安装的 Linux // $ wsl -d Ubuntu-22.04 # 启动特定发行版 // $ wsl --shutdown # 关闭 WSL // $ wsl --set-default-version 2 # 设置 WSL2 // // 文件系统: // Linux 访问 Windows: /mnt/c/Users/... // Windows 访问 Linux: \\wsl$\Ubuntu\home\
💻 对于后端开发者,推荐 WSL2 或云服务器。WSL2 让你在 Windows 上获得原生 Linux 终端体验,是最低门槛的学习方式。
