# 74_SSL-TLS_考试

## 核心概念

- **408常考题型**：选择题为主，涉及SSL/TLS作用和位置
- **关键考点**：
  - SSL/TLS在应用层和传输层之间
  - SSL/TLS提供机密性、完整性、认证
  - HTTPS = HTTP + SSL/TLS
  - SSL/TLS使用数字证书验证身份
- **易混淆点**：
  - SSL/TLS不是传输层协议，是安全层
  - SSL已废弃，现在使用TLS

## 原理分析

### 典型考题1：SSL/TLS位置

**题目**：SSL/TLS协议位于（  ）
A. 应用层
B. 传输层
C. 应用层和传输层之间
D. 网络层

**答案**：C

**解析**：
- SSL/TLS在应用层和传输层之间
- 为应用层提供安全服务

### 典型考题2：SSL/TLS功能

**题目**：SSL/TLS提供的功能包括（  ）（多选）
A. 数据加密
B. 数据完整性验证
C. 身份认证
D. 数据压缩

**答案**：A, B, C

**解析**：
- SSL/TLS提供机密性（加密）
- SSL/TLS提供完整性（MAC）
- SSL/TLS提供认证（数字证书）
- SSL/TLS不提供压缩

### 典型考题3：HTTPS

**题目**：HTTPS是（  ）
A. HTTP + TCP
B. HTTP + SSL/TLS
C. HTTP + UDP
D. HTTP + IP

**答案**：B

**解析**：
- HTTPS = HTTP + SSL/TLS
- SSL/TLS提供安全服务

## 直观理解

**做题技巧**：
- SSL/TLS在应用层和传输层之间
- SSL/TLS = 机密性 + 完整性 + 认证
- HTTPS = HTTP + SSL/TLS
- 数字证书验证身份

### 典型考题4：TLS握手过程

**题目**：简述TLS握手的主要步骤。

**参考答案**：
1. **Client Hello**：客户端发送支持的密码套件、随机数
2. **Server Hello**：服务器选择密码套件、发送随机数和证书
3. **证书验证**：客户端验证服务器证书
4. **密钥交换**：客户端生成预主密钥，用服务器公钥加密发送
5. **密钥派生**：双方根据预主密钥和随机数生成会话密钥
6. **Finished**：双方发送加密的Finished消息，握手完成

### 典型考题5：数字证书

**题目**：数字证书的作用是（  ）
A. 加密数据
B. 验证服务器身份
C. 压缩数据
D. 缓存数据

**答案**：B

**解析**：
- 数字证书由CA签发，证明服务器身份
- 证书包含服务器公钥和CA签名
- 客户端验证CA签名来确认服务器身份

### 典型考题6：SSL/TLS vs IPsec

| 特性 | SSL/TLS | IPsec |
|------|---------|-------|
| 位置 | 应用层与传输层之间 | 网络层 |
| 应用 | Web、邮件、文件 | VPN |
| 粒度 | 应用级别 | 主机级别 |
| 配置 | 较简单 | 较复杂 |
| 防火墙 | 穿透性好 | 可能有兼容问题 |

### TLS密码套件组成
- **密钥交换算法**：RSA、ECDHE、DHE
- **认证算法**：RSA、ECDSA
- **加密算法**：AES、ChaCha20
- **MAC算法**：SHA-256、SHA-384

### SSL/TLS版本演进
| 版本 | 说明 |
|------|------|
| SSL 2.0 | 已废弃（安全漏洞） |
| SSL 3.0 | 已废弃（POODLE攻击） |
| TLS 1.0 | 已废弃 |
| TLS 1.1 | 已废弃 |
| TLS 1.2 | 广泛使用 |
| TLS 1.3 | 最新版本，0-RTT |

### 408常考要点
- SSL/TLS在应用层和传输层之间
- 提供机密性、完整性、认证三大功能
- HTTPS = HTTP + SSL/TLS
- SSL已废弃，现在使用TLS
- 数字证书由CA签发，用于身份验证
- TLS 1.3支持0-RTT连接建立

## 协议关联

- **SSL/TLS与HTTPS**：HTTPS使用SSL/TLS
- **SSL/TLS与TCP**：SSL/TLS在TCP之上
- **SSL/TLS与IPsec**：分别在不同层提供安全
- **SSL/TLS与SSH**：都提供加密和认证
- **408常见组合**：SSL/TLS + HTTPS = 安全综合题
