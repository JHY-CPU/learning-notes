# 84_应用层协议与传输层

## 核心概念

- **应用层与传输层关系**：应用层协议选择TCP或UDP
- **选择依据**：
  - **TCP**：需要可靠传输（HTTP、FTP、SMTP、POP3、IMAP）
  - **UDP**：需要快速传输（DNS、DHCP、SNMP）
- **408考试重点**：各应用层协议的传输层选择

## 原理分析

### 各协议传输层选择

| 应用层协议 | 传输层 | 端口 | 选择原因 |
|-----------|--------|------|---------|
| HTTP | TCP | 80 | 需要可靠传输 |
| HTTPS | TCP | 443 | 需要安全传输 |
| FTP | TCP | 20/21 | 需要可靠传输 |
| SMTP | TCP | 25 | 需要可靠传输 |
| POP3 | TCP | 110 | 需要可靠传输 |
| IMAP | TCP | 143 | 需要可靠传输 |
| TELNET | TCP | 23 | 需要可靠传输 |
| SSH | TCP | 22 | 需要安全传输 |
| DNS | UDP/TCP | 53 | 默认UDP，大报文用TCP |
| DHCP | UDP | 67/68 | 需要广播 |
| SNMP | UDP | 161/162 | 简单快速 |
| RTP | UDP | 偶数 | 实时性要求 |

### 选择TCP的原因

1. **可靠性**：TCP提供可靠传输，数据不丢失
2. **有序性**：TCP保证数据顺序
3. **流量控制**：TCP防止发送方过快
4. **拥塞控制**：TCP防止网络拥塞

**适用场景**：文件传输、邮件、Web（需要完整数据）

### 选择UDP的原因

1. **速度快**：无握手、无确认
2. **开销小**：头部只有8字节
3. **支持广播**：DHCP需要广播
4. **实时性**：语音视频需要低延迟

**适用场景**：DNS查询、DHCP、SNMP、实时音视频

### DNS特殊情况

- 默认使用UDP（快速、开销小）
- 响应超过512字节时使用TCP
- 区域传送使用TCP

## 直观理解

**记忆技巧**：
- TCP = "可靠但慢"（HTTP、FTP、邮件）
- UDP = "快但不可靠"（DNS、DHCP、SNMP）
- "重要数据用TCP，简单查询用UDP"

## 协议关联

- **应用层与传输层**：应用层选择传输层协议
- **TCP与UDP**：各有适用场景
- **408考点**：各协议的传输层选择
- **陷阱**：DNS默认UDP但也可用TCP
