# 16_TELNET协议

## 核心概念

- **TELNET（Telecommunication Network）**：远程终端协议，允许用户登录到远程主机
- TELNET使用**TCP**协议，端口号**23**
- TELNET是**C/S模式**的应用
- **NVT（Network Virtual Terminal）**：网络虚拟终端，TELNET的标准化通信格式
- TELNET默认使用**明文传输**，安全性差
- **408考试重点**：TELNET的基本概念、端口号、NVT的作用

## 原理分析

### TELNET工作原理

1. **建立连接**：
   - 客户端主动连接服务器的23端口
   - 使用TCP三次握手建立连接

2. **协商选项**：
   - 客户端和服务器协商TELNET选项
   - 包括：终端类型、回显方式、行/字符模式等
   - 使用DO、DONT、WILL、WONT命令协商

3. **数据传输**：
   - 客户端的键盘输入通过NVT格式发送到服务器
   - 服务器的输出通过NVT格式返回给客户端
   - 数据以NVT ASCII格式传输

4. **关闭连接**：
   - 用户退出或连接异常时关闭TCP连接

### NVT（网络虚拟终端）

- **问题**：不同系统的终端类型和控制码不同
- **解决**：定义一个标准的虚拟终端格式
- NVT使用7位ASCII码，最高位为0
- 所有TELNET通信都转换为NVT格式
- 客户端和服务器各自将本地格式转换为NVT格式

### TELNET选项协商

| 命令 | 含义 |
|------|------|
| WILL | 发送方想启用某选项 |
| WONT | 发送方想禁用某选项 |
| DO | 发送方希望对方启用某选项 |
| DONT | 发送方希望对方禁用某选项 |

协商过程：客户端发送WILL X，服务器回复DO X表示同意

## 直观理解

**TELNET就像远程控制办公室的电脑**：
- 你在家（客户端），通过TELNET连接公司的电脑（服务器）
- 你的键盘输入变成电信号发送到公司电脑
- 公司电脑的屏幕输出返回到你的屏幕上
- NVT就像统一的"语言"，让不同品牌的终端都能互相通信

**记忆技巧**：
- TELNET = "远程终端"，端口23
- NVT = "通用翻译器"，解决终端不兼容问题
- TELNET不安全（明文），SSH是安全的替代品
- "TELNET 23，SSH 22"，差一个端口

**TELNET的特点**：
- 简单易实现
- 通用性强（NVT标准化）
- 安全性差（明文传输）
- 功能有限（只能终端交互）

## 协议关联

- **TELNET与TCP**：TELNET使用TCP的可靠传输，端口23
- **TELNET与SSH**：SSH是TELNET的安全替代品，端口22
  - SSH提供加密、认证、完整性保护
  - TELNET明文传输密码，SSH加密传输
- **TELNET与FTP**：FTP控制连接使用NVT ASCII格式（与TELNET兼容）
- **TELNET与HTTP**：都可以用于远程访问，但设计目标不同
- **408考点**：
  - TELNET端口23
  - TELNET使用TCP
  - NVT的作用是解决终端兼容性问题
  - TELNET不安全，现代已很少使用
- **陷阱**：TELNET不是加密协议，SSH才是安全的远程登录协议
