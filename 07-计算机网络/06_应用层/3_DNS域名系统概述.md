# 04_DNS域名系统概述

## 核心概念

- **DNS（Domain Name System）**：域名系统，将人类可读的域名解析为IP地址
- DNS是一个**分布式、层次化**的数据库系统
- DNS运行在**UDP**上，端口号**53**
- DNS提供的服务：
  - **主机名到IP地址的转换**（核心功能）
  - **主机别名**（canonical hostname → alias hostname）
  - **邮件服务器别名**
  - **负载分配**：将域名映射到多个IP地址
- **408考试重点**：DNS的层次结构、解析过程、记录类型、递归与迭代查询
- **常见混淆**：DNS不直接服务于用户，而是为其他应用（如HTTP）提供服务

## 原理分析

### DNS的工作方式

1. **集中式方案的问题**：如果用单一DNS服务器
   - **单点故障**：服务器崩溃，整个网络瘫痪
   - **通信容量**：无法处理大量DNS查询
   - **远距离集中**：全球查询延迟大
   - **维护困难**：数据库过于庞大

2. **分布式方案**：DNS采用分布式、层次化结构
   - 根DNS服务器（Root）
   - 顶级域DNS服务器（TLD）
   - 权威DNS服务器（Authoritative）
   - 本地DNS服务器（Local DNS）

### DNS查询报文

- 查询报文和响应报文具有相同格式
- 报文结构：
  - 首部（12字节）：标识、标志、问题数、回答数、权威数、附加数
  - 问题部分：查询的名称、类型、类别
  - 回答部分：资源记录（RR）
  - 权威部分：权威DNS服务器记录
  - 附加部分：附加信息

### DNS缓存

- 本地DNS服务器缓存查询结果
- 缓存有TTL（Time To Live），到期自动删除
- 缓存大大减少了DNS查询流量
- 权威DNS服务器的更改可能需要等待TTL过期

## 直观理解

把DNS比作**电话簿系统**：
- 你想找"张三"（域名），但打电话需要电话号码（IP地址）
- 你问本地查号台（本地DNS服务器）
- 本地查号台查不到就向上级查号台问（递归查询或迭代查询）
- 最终查到号码，返回给你，同时本地查号台记住这个号码（缓存）
- 下次别人问同样的号码，本地查号台直接回答（缓存命中）

**记忆技巧**：
- DNS = "Domain Name → IP地址的翻译官"
- 端口53：DNS字母D是第4个字母，N是第14个，S是第19个，53=5*3+...算了，直接记"域53"
- DNS主要用UDP（快），但大报文用TCP（可靠）

## 协议关联

- **DNS与HTTP**：用户输入URL时，浏览器先调用DNS解析域名，再发起HTTP请求
- **DNS与DHCP**：DHCP为客户端分配IP地址，同时可以配置DNS服务器地址
- **DNS与邮件系统**：MX记录（邮件交换记录）指定邮件服务器
- **DNS与UDP/TCP**：
  - 默认使用UDP（端口53），快速、开销小
  - 响应超过512字节时使用TCP
  - 区域传送（Zone Transfer）使用TCP
- **408常考**：DNS查询过程中各层服务器的角色，以及迭代查询与递归查询的区别
