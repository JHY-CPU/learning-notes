# 85_应用层传输层_考试

## 核心概念

- **408常考题型**：选择题为主，涉及应用层协议的传输层选择
- **关键考点**：
  - TCP协议：HTTP、FTP、SMTP、POP3、IMAP、SSH
  - UDP协议：DNS、DHCP、SNMP
  - DNS特殊情况：默认UDP，大报文用TCP
- **易混淆点**：
  - DNS既用UDP也用TCP
  - DHCP需要广播所以用UDP

## 原理分析

### 典型考题1：传输层选择

**题目**：以下使用TCP的协议是（  ）（多选）
A. HTTP
B. DNS
C. FTP
D. DHCP

**答案**：A, C

**解析**：
- HTTP使用TCP（A正确）
- DNS主要使用UDP（B错误）
- FTP使用TCP（C正确）
- DHCP使用UDP（D错误）

### 典型考题2：UDP协议

**题目**：以下使用UDP的协议是（  ）
A. HTTP
B. FTP
C. SMTP
D. DNS

**答案**：D

**解析**：
- HTTP、FTP、SMTP都使用TCP
- DNS主要使用UDP

### 典型考题3：DNS传输层

**题目**：关于DNS，以下说法正确的是（  ）
A. DNS只使用UDP
B. DNS只使用TCP
C. DNS主要使用UDP，大报文用TCP
D. DNS使用ICMP

**答案**：C

**解析**：
- DNS默认使用UDP（端口53）
- 响应超过512字节时使用TCP
- 区域传送使用TCP

### 典型考题4：选择依据

**题目**：DNS选择UDP的原因是（  ）
A. 需要可靠传输
B. 需要快速传输
C. 需要广播
D. 需要大文件传输

**答案**：B

## 直观理解

**做题技巧**：
- TCP = 可靠（HTTP、FTP、邮件）
- UDP = 快速（DNS、DHCP、SNMP）
- DNS特殊：默认UDP，大报文TCP

### 典型考题5：QUIC协议

**题目**：HTTP/3使用的QUIC协议基于（  ）
A. TCP
B. UDP
C. SCTP
D. ICMP

**答案**：B

**解析**：
- QUIC基于UDP，在用户态实现可靠传输
- HTTP/3使用QUIC替代TCP
- QUIC优势：0-RTT、连接迁移、无队头阻塞

### 典型考题6：传输层选择原因

**题目**：HTTP选择TCP而非UDP的原因是（  ）
A. TCP更快
B. TCP提供可靠传输
C. TCP支持广播
D. TCP首部更小

**答案**：B

**解析**：
- HTTP需要可靠传输（网页完整性关键）
- TCP提供可靠传输、流量控制、拥塞控制
- UDP不保证可靠（可能丢失数据）

### 典型考题7：协议传输层综合

**题目**：完成以下表格：

| 协议 | 传输层 | 原因 |
|------|--------|------|
| HTTP | ? | ? |
| DNS | ? | ? |
| FTP | ? | ? |
| DHCP | ? | ? |
| SNMP | ? | ? |

**答案**：
| 协议 | 传输层 | 原因 |
|------|--------|------|
| HTTP | TCP | 需要可靠传输 |
| DNS | UDP/TCP | 查询用UDP（快速），大报文用TCP |
| FTP | TCP | 需要可靠传输大文件 |
| DHCP | UDP | 需要广播，初始无IP |
| SNMP | UDP | 简单快速，容忍丢失 |

### 应用层协议传输层选择决策树
```
需要可靠传输？ → 是 → TCP（HTTP、FTP、SMTP、POP3、IMAP、SSH）
                → 否 → UDP
需要广播/组播？ → 是 → UDP（DHCP）
                → 否 → 继续判断
数据量小且频繁？ → 是 → UDP（SNMP、DNS查询）
                → 否 → TCP
实时性要求高？ → 是 → UDP（RTP、视频会议）
              → 否 → TCP
```

### HTTP/3与QUIC详解
- QUIC = Quick UDP Internet Connections
- 在UDP之上实现可靠传输（用户态）
- 内置TLS 1.3，0-RTT连接建立
- 解决TCP队头阻塞
- 支持连接迁移（切换WiFi不中断）
- Google开发，已成为IETF标准

### 408常考要点
- TCP协议：HTTP(80)、FTP(20/21)、SMTP(25)、POP3(110)、IMAP(143)、SSH(22)、HTTPS(443)
- UDP协议：DNS(53)、DHCP(67/68)、SNMP(161/162)
- DNS特殊情况：默认UDP，大报文用TCP
- DHCP用UDP：需要广播，初始无IP
- HTTP/3用QUIC（基于UDP）
- 传输层选择依据：可靠性、数据量、实时性

## 协议关联

- **应用层与传输层**：选择依据
- **TCP vs UDP**：各有适用场景
- **QUIC与UDP**：HTTP/3在UDP上实现可靠传输
- **408常见组合**：协议 + 传输层 = 选择题高频考点
