# 11_DNS递归迭代_考试

## 核心概念

- **408常考题型**：选择题为主，要求区分递归查询和迭代查询
- **关键考点**：
  - 递归查询的定义：被查询服务器必须返回最终结果
  - 迭代查询的定义：被查询服务器返回其他服务器地址
  - 实际DNS查询采用混合方式
  - 查询时间和RTT计算
- **易混淆点**：
  - "递归查询"≠"查询过程是递归的"
  - 实际中根服务器通常只接受迭代查询

## 原理分析

### 典型考题1：查询方式判断

**题目**：主机向本地DNS服务器的查询是（  ），本地DNS服务器向根DNS服务器的查询是（  ）。

**答案**：递归查询；迭代查询

**解析**：
- 主机→本地DNS：主机只发一次请求，本地DNS必须返回最终结果 → 递归
- 本地DNS→根DNS：根DNS返回下一级地址，本地DNS自己继续查 → 迭代

### 典型考题2：查询时间计算

**题目**：假设各DNS服务器之间RTT=50ms，主机到本地DNS的RTT=10ms，无缓存情况下查询`www.abc.com`需要多长时间？

**分析**（迭代查询）：
1. 主机→本地DNS：10ms（请求）
2. 本地DNS→根DNS：50ms（请求）+ 50ms（响应）
3. 本地DNS→.com TLD：50ms（请求）+ 50ms（响应）
4. 本地DNS→权威DNS：50ms（请求）+ 50ms（响应）
5. 本地DNS→主机：10ms（响应）

**总时间**：10 + 4×50 + 10 = 220ms（或简化为10 + 3×100 + 10 = 320ms，取决于是否串行）

注意：实际并行处理可以减少时间。如果假设每次查询是串行的：
$$T = RTT_{host-local} + 3 \times (2 \times RTT_{server}) + RTT_{host-local} = 10 + 300 + 10 = 320ms$$

简化模型（忽略响应时间）：
$$T = RTT_1 + RTT_2 + RTT_3 + RTT_4 = 10 + 50 + 50 + 50 = 160ms$$

### 典型考题3：缓存对查询的影响

**题目**：本地DNS缓存了`.com`TLD的地址，则查询`www.abc.com`需要几次查询？

**答案**：跳过根DNS查询，只需2次迭代查询（TLD + 权威DNS）

## 直观理解

**做题口诀**：
- "主机问本地" = 递归（本地DNS必须给答案）
- "本地问别人" = 迭代（别人让本地自己去问）
- 看到"必须返回结果" = 递归
- 看到"返回其他地址" = 迭代

**计算题技巧**：
- 先画查询流程图，标注每段RTT
- 递归查询：被查询服务器承担内部查询的RTT
- 迭代查询：本地DNS承担所有查询的RTT
- 注意并行查询可以减少总时间

## 协议关联

- **递归查询与TCP**：理论上递归查询需要更多资源，但DNS用UDP实现
- **迭代查询与DNS缓存**：缓存可以减少迭代查询的跳数
- **DNS与HTTP延迟**：DNS查询增加HTTP请求的总延迟
- **408陷阱**：题目问"查询了几次"和"经过几次RTT"是不同的问题
