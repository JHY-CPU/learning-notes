# 34_HTTP持久连接

## 核心概念

- **非持久连接（Non-persistent Connection）**：HTTP/1.0默认，每个请求建立新TCP连接
  - 每个对象需要单独的TCP连接
  - 传输完成后立即关闭连接
  - 延迟较高（多次TCP握手）
- **持久连接（Persistent Connection）**：HTTP/1.1默认，TCP连接可以复用
  - 多个请求共享同一个TCP连接
  - 减少TCP握手开销
  - 降低延迟
- **流水线（Pipelining）**：HTTP/1.1支持，客户端可以连续发送多个请求
- **408考试重点**：非持久与持久连接的对比、RTT计算

## 原理分析

### 非持久连接（HTTP/1.0）

**访问一个HTML页面（含N个内嵌对象）**：
1. TCP三次握手（1 RTT）
2. HTTP请求+响应（1 RTT）
3. 浏览器解析HTML
4. 对每个内嵌对象：
   - 新TCP三次握手（1 RTT）
   - HTTP请求+响应（1 RTT）
   - 关闭TCP连接

**总时间**（串行）：
$$T = 2RTT_{TCP} + 2RTT_{HTTP} \times (N+1)$$

或简化为：
$$T = 2(N+1) \times RTT$$

### 持久连接（HTTP/1.1）

**非流水线方式**：
1. TCP三次握手（1 RTT）
2. HTTP请求+响应（1 RTT）
3. 后续每个对象：
   - HTTP请求+响应（1 RTT）
   - 不需要重新建立TCP连接

**总时间**（串行）：
$$T = 2RTT + (N+1) \times RTT = (N+3) \times RTT$$

**流水线方式**：
1. TCP三次握手（1 RTT）
2. 连续发送所有HTTP请求（1 RTT）
3. 连续接收所有HTTP响应（1 RTT）

**总时间**：
$$T = 2RTT + RTT = 3RTT$$

### 对比分析

| 特性 | 非持久连接 | 持久连接（非流水线） | 持久连接（流水线） |
|------|-----------|-------------------|-------------------|
| TCP连接数 | N+1 | 1 | 1 |
| 延迟 | 高 | 中 | 低 |
| 服务器负担 | 大 | 小 | 小 |
| HTTP版本 | 1.0默认 | 1.1默认 | 1.1可选 |

### Connection头部

- `Connection: close`：本次请求后关闭连接（HTTP/1.1）
- `Connection: keep-alive`：保持连接（HTTP/1.0）
- HTTP/1.1默认持久连接，除非指定`Connection: close`

## 直观理解

**非持久连接就像每次买东西都重新排队**：
- 买第一个东西：排队（TCP握手）→ 买（HTTP请求响应）→ 离开（关闭连接）
- 买第二个东西：重新排队 → 买 → 离开
- 效率低，但简单

**持久连接就像一次排队买多个东西**：
- 排一次队（TCP握手）
- 连续买多个东西（多个HTTP请求响应）
- 效率高，节省时间

**记忆技巧**：
- HTTP/1.0 = 非持久（默认），每个对象新建TCP连接
- HTTP/1.1 = 持久（默认），TCP连接复用
- 流水线 = "连续发多个请求，一起等响应"
- RTT计算：握手1RTT + 请求响应1RTT

## 协议关联

- **持久连接与TCP**：减少TCP连接建立次数
- **持久连接与队头阻塞**：HTTP/1.1流水线存在队头阻塞问题
- **持久连接与HTTP/2**：HTTP/2多路复用解决队头阻塞
- **408考点**：
  - 非持久与持久连接的区别
  - RTT时间计算
  - HTTP/1.0与HTTP/1.1的默认行为
- **陷阱**：HTTP/1.1默认持久连接，但可以用`Connection: close`关闭
