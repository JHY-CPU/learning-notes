# 80_DNS_HTTP_FTP综合

## 核心概念

- **DNS + HTTP + FTP综合**：涉及三个应用层协议的综合分析
- **关键考点**：
  - DNS解析是HTTP和FTP的前置步骤
  - HTTP单连接无状态，FTP双连接有状态
  - 各协议端口号和传输层选择
- **408考试重点**：协议对比和综合分析

## 原理分析

### 综合题1：协议对比

**题目**：比较DNS、HTTP、FTP的特点。

**参考答案**：
| 特性 | DNS | HTTP | FTP |
|------|-----|------|-----|
| 端口 | 53 | 80 | 20/21 |
| 传输层 | UDP/TCP | TCP | TCP |
| 连接数 | - | 1个 | 2个 |
| 状态 | - | 无状态 | 有状态 |
| 用途 | 域名解析 | Web | 文件传输 |

### 综合题2：访问流程分析

**题目**：用户通过FTP下载文件，描述涉及的协议和端口。

**参考答案**：
1. DNS解析FTP服务器域名（UDP 53）
2. TCP三次握手建立控制连接（端口21）
3. 用户认证（USER/PASS命令）
4. TCP三次握手建立数据连接（端口20，主动模式）
5. 数据传输（RETR命令）
6. 关闭数据连接
7. 控制连接保持或关闭

### 综合题3：端口汇总

**题目**：列出DNS、HTTP、FTP使用的端口号和传输层协议。

**参考答案**：
| 协议 | 端口 | 传输层 |
|------|------|--------|
| DNS | 53 | UDP/TCP |
| HTTP | 80 | TCP |
| FTP控制 | 21 | TCP |
| FTP数据 | 20 | TCP |

### 综合题4：Web vs FTP

**题目**：从连接方式、状态维护、用途比较HTTP和FTP。

**参考答案**：
- HTTP：单连接、无状态、Web文档
- FTP：双连接（控制+数据）、有状态、文件传输

## 直观理解

**记忆技巧**：
- DNS 53查域名，HTTP 80看网页，FTP 20/21传文件
- HTTP单连接无状态，FTP双连接有状态
- DNS用UDP，HTTP/FTP用TCP

### 综合题5：协议栈图

**题目**：画出DNS、HTTP、FTP在TCP/IP协议栈中的位置。

**参考答案**：
```
应用层：  DNS  |  HTTP  |  FTP(控制+数据)
          |        |        |
传输层： UDP/TCP |  TCP   |  TCP(2个连接)
          |        |        |
网络层：      IP              IP
          |                   |
数据链路层：  以太网等         以太网等
```

### 综合题6：Web访问完整流程

**题目**：描述用户在浏览器输入`http://www.example.com/page.html`后的完整过程。

**参考答案**：
1. **DNS解析**：浏览器查询www.example.com的IP地址（UDP 53）
2. **TCP握手**：浏览器与服务器建立TCP连接（三次握手）
3. **HTTP请求**：浏览器发送GET /page.html请求（TCP 80）
4. **HTTP响应**：服务器返回HTML内容
5. **解析HTML**：浏览器发现页面引用了图片等资源
6. **重复2-4**：获取每个资源（持久连接可复用TCP）

### 综合题7：FTP工作流程

**题目**：用户通过FTP客户端连接服务器下载文件，描述完整过程。

**参考答案**：
1. **DNS解析**：查询FTP服务器域名（UDP 53）
2. **控制连接**：TCP三次握手建立控制连接（端口21）
3. **认证**：USER命令发送用户名，PASS命令发送密码
4. **命令交互**：LIST列出目录，CWD切换目录
5. **数据连接**：
   - 主动模式：服务器用端口20连接客户端
   - 被动模式：服务器开放端口，客户端连接
6. **数据传输**：RETR命令下载文件
7. **关闭数据连接**：传输完成后关闭
8. **关闭控制连接**：QUIT命令

### 协议综合对比

| 维度 | DNS | HTTP | FTP |
|------|-----|------|-----|
| 端口 | 53 | 80 | 20/21 |
| 传输层 | UDP/TCP | TCP | TCP |
| 连接数 | - | 1个 | 2个 |
| 状态 | - | 无状态 | 有状态 |
| 数据形式 | 查询/响应 | 请求/响应 | 命令/数据 |
| 编码 | - | 文本 | ASCII/二进制 |
| 安全 | DNSSEC | HTTPS | SFTP/FTPS |

### 408常考要点
- DNS是HTTP和FTP的前置步骤
- HTTP单连接无状态，FTP双连接有状态
- DNS默认UDP，大报文TCP
- FTP控制连接端口21，数据连接端口20
- Web访问总时间 = DNS + TCP握手 + HTTP请求
