# HTTP概述

## 🌐 HTTP 概述

HTTP 协议发展史、基本特性、协议变体、HTTP/0.9 到 HTTP/3 的演进。

## HTTP 是什么

超文本传输协议 (HTTP) 是互联网上应用最广泛的应用层协议，定义了客户端与服务器之间的通信格式。
```
// ========== HTTP 的核心特性 ==========
// 1. 应用层协议 — 基于 TCP/IP
// 2. 请求-响应模式 — 客户端主动,服务端被动
// 3. 无状态 — 每个请求独立,默认不记忆上下文
// 4. 可扩展 — 头部机制,可不断新增功能
// 5. 文本/二进制 — HTTP/1.x 文本, HTTP/2+ 二进制
//
// ========== HTTP 协议变体 ==========
// HTTP/0.9 (1991):
//   只有 GET 方法
//   没有头部,没有状态码
//   只返回 HTML
//
// HTTP/1.0 (1996):
//   引入 GET/POST/HEAD 方法
//   引入 HTTP 头和状态码
//   引入 Content-Type
//   每个请求新建 TCP 连接
//
// HTTP/1.1 (1997, 至今使用最广):
//   连接复用 (Keep-Alive)
//   管道 (Pipelining)
//   新增 PUT/DELETE/OPTIONS 等方法
//   Host 头 (虚拟主机)
//   分块传输编码
//
// HTTP/2 (2015):
//   二进制 framing 层
//   多路复用 (取消队头阻塞)
//   服务器推送
//   头部压缩 (HPACK)
//   需要 TLS (事实标准)
//
// HTTP/3 (2022):
//   基于 QUIC (UDP)
//   0-RTT 连接建立
//   解决 TCP 级别的队头阻塞
```
> **Note**: 📈 HTTP/3 是未来方向——基于 UDP 的 QUIC 协议实现了 0-RTT 连接建立和更好的移动网络体验。目前已有约 30% 的网站支持 HTTP/3。

## HTTP/1.1 的关键特性
```
// ========== Keep-Alive (持久连接) ==========
// HTTP/1.0: 每个请求新建 TCP 连接
// HTTP/1.1: 默认复用连接
//   Connection: keep-alive  (HTTP/1.1 默认)
//   Connection: close      (显式关闭)
//
// 优势: 减少 TCP 握手开销
// 劣势: 队头阻塞 (HOL blocking)
//
// ========== 管道 (Pipelining) ==========
// 允许在等待响应时发送下一个请求
// 减少网络延迟
//
// 问题: 队头阻塞 (HOL)
//   一个慢响应阻塞后面的所有响应
//   实际中很少使用
//
// ========== 虚拟主机 ==========
// 同一台服务器托管多个网站
// 通过 Host 头区分:
//   Host: example.com
//   Host: example.org
// 这样一台服务器可托管无数个域名
```
## HTTP/2 与 HTTP/3 的改进
```
// ========== HTTP/2 多路复用 ==========
// HTTP/1.1: 同一连接只能串行处理请求
//   [请求1→][←响应1][请求2→][←响应2]
//
// HTTP/2: 同一连接并行处理多个请求
//   [请求1→][请求2→][请求3→]
//   [←响应2][←响应1][←响应3]
//
// 实现: 流 (Stream) + 帧 (Frame)
//   每个流承载一个 HTTP 请求/响应
//   帧是最小数据单位,可在流间交错发送
//
// ========== HTTP/2 头部压缩 ==========
// HPACK 算法:
//   动态表 + 静态表 + Huffman 编码
//   减少重复头部传输
//   一次请求约 800 bytes 头部 → 压缩后 < 100 bytes
//
// ========== HTTP/3 QUIC ==========
// 基于 UDP, 不是 TCP
// 优势:
//   0-RTT 连接 (首次 1-RTT)
//   无 TCP 队头阻塞
//   连接迁移 (切换 WiFi 不断连)
//   内置 TLS 1.3

// ========== 各版本延迟对比 ==========
// HTTP/1.1 (新连接): 3 RTT + 传输时间
//   TCP(1.5) + TLS(1.5) + 请求/响应(1)
//
// HTTP/2 (新连接): 3 RTT + 传输时间
//   多路复用在传输阶段优化
//
// HTTP/3 (新连接): 2 RTT + 传输时间
//   QUIC(1) + TLS(1) + 请求/响应(1)
//   第二次连接: 1 RTT
```
