# 78_应用层综合_真题1

## 核心概念

- **真题来源**：408统考应用层综合大题
- **高频考点**：DNS + HTTP + TCP综合计算
- **出题规律**：给出网络参数，计算访问时间

## 原理分析

### 真题1（综合题）

**题目**：某主机访问`http://www.test.com/index.html`，已知：
- DNS解析时间：RTT_DNS = 50ms
- TCP握手时间：RTT_TCP = 30ms
- HTTP请求响应时间：RTT_HTTP = 20ms
- index.html引用了3个图片，均位于同一服务器
- 使用HTTP/1.1持久连接非流水线

求：（1）访问index.html的总时间 （2）获取所有图片的总时间

**参考答案**：
（1）访问index.html：
- DNS解析：50ms
- TCP握手：30ms
- HTTP请求响应：20ms
- 总计：50 + 30 + 20 = 100ms

（2）获取所有图片（持久连接，复用TCP）：
- 每个图片：HTTP请求响应20ms
- 3个图片：3 × 20 = 60ms

### 真题2（综合题）

**题目**：HTTP/1.0非持久连接，访问含1个HTML和4个图片的页面，各对象大小相同，每个RTT=10ms。求总时间。

**参考答案**：
- 每个对象：TCP握手(10ms) + HTTP请求(10ms) = 20ms
- 5个对象：5 × 20 = 100ms
- 总时间：100ms

### 真题3（综合题）

**题目**：HTTP/1.1持久连接流水线，访问含1个HTML和4个图片的页面，RTT=10ms。求总时间。

**参考答案**：
- TCP握手：10ms
- 发送所有请求：10ms
- 接收所有响应：10ms
- 总时间：30ms

## 直观理解

**做题技巧**：
- 非持久：每个对象 = 2 RTT
- 持久非流水线：首个 = 2 RTT，后续各 = 1 RTT
- 持久流水线：总共 = 3 RTT
- 注意DNS和TCP握手是否重复

### 真题4（多服务器访问）

**题目**：页面含1个HTML（服务器A）和2个图片（服务器B）。使用HTTP/1.1持久非流水线。RTT=10ms。求总时间。

**参考答案**：
- 访问服务器A（HTML）：
  - TCP握手：10ms
  - HTTP请求：10ms
  - 小计：20ms
- 访问服务器B（2个图片）：
  - TCP握手：10ms
  - HTTP请求：10ms（图片1）
  - HTTP请求：10ms（图片2）
  - 小计：30ms
- 总时间：20 + 30 = 50ms

**关键点**：不同服务器需要分别建立TCP连接。

### 真题5（缓存命中）

**题目**：HTTP/1.1持久非流水线，浏览器缓存了3个图片中的2个。RTT=10ms。访问含1个HTML和3个图片的页面，总时间是多少？

**参考答案**：
- TCP握手：10ms
- HTML请求：10ms
- 未缓存图片：10ms
- 缓存图片：0ms（本地读取）
- 总计：10 + 10 + 10 = 30ms

### 真题6（DNS + TCP + HTTP综合）

**题目**：首次访问www.test.com，DNS解析50ms，TCP握手20ms，HTTP请求响应10ms。页面含1个HTML和4个图片（同一服务器，HTTP/1.1持久非流水线）。求总时间。

**参考答案**：
- DNS解析：50ms（一次性）
- TCP握手：20ms（一次性）
- HTML请求：10ms
- 4个图片：4 × 10 = 40ms
- 总计：50 + 20 + 10 + 40 = 120ms

### 408做题框架
1. **确定HTTP版本**：1.0=非持久，1.1=持久
2. **确定是否流水线**：非流水线逐个请求，流水线批量请求
3. **计算RTT**：
   - 非持久：每个对象2 RTT
   - 持久非流水线：首个2 RTT + 后续各1 RTT
   - 持久流水线：总共3 RTT
4. **加上DNS和TCP握手**（首次访问）
5. **注意多服务器情况**

### 408常考要点
- DNS解析只需一次（有缓存则跳过）
- TCP握手只需一次（持久连接）
- 非持久连接每个对象都要重新握手
- 流水线可显著减少RTT
- 缓存命中可减少RTT

## 协议关联

- **DNS + TCP + HTTP** = Web访问时间
- **持久/非持久** = 连接模式影响
- **HTTP版本** = RTT计算方式不同
- **408常见组合**：RTT计算是大题重点
