# 69_流媒体概述

## 核心概念

- **流媒体（Streaming Media）**：边下载边播放的多媒体传输方式
- **流媒体特点**：
  - 实时性：数据连续到达
  - 时序性：数据有严格的时序要求
  - 容忍丢失：少量丢失不影响观看
- **流媒体协议**：
  - **RTP（Real-time Transport Protocol）**：实时传输协议
  - **RTCP（RTP Control Protocol）**：RTP控制协议
  - **RTSP（Real-Time Streaming Protocol）**：实时流协议
  - **HLS（HTTP Live Streaming）**：基于HTTP的流媒体
- **408考试重点**：流媒体基本概念、RTP的作用

## 原理分析

### 流媒体传输方式

1. **UDP流**：
   - 使用RTP/UDP传输
   - 低延迟
   - 可能丢包
   - 适合实时应用

2. **HTTP流**：
   - 使用HTTP/TCP传输
   - 延迟较高
   - 可靠传输
   - 适合点播

3. **自适应流**：
   - HLS、DASH等
   - 根据带宽自动调整质量
   - 分片传输

### RTP协议

- **RTP**：实时传输协议
- 使用**UDP**传输
- 端口：偶数端口（如5004）
- RTP头部包含：
  - 序列号：检测丢包
  - 时间戳：同步
  - 负载类型：媒体类型
  - SSRC：同步源标识

### RTSP协议

- **RTSP**：实时流协议
- 类似HTTP的控制协议
- 端口：554
- 支持的操作：DESCRIBE、SETUP、PLAY、PAUSE、TEARDOWN
- 控制RTP流的播放

### HLS（HTTP Live Streaming）

- Apple开发的流媒体协议
- 基于HTTP
- 将视频分成小片段（.ts文件）
- 使用.m3u8索引文件
- 支持自适应码率

## 直观理解

**流媒体就像水管流水**：
- 传统下载 = 先把水灌满桶，再用水（下载完再播放）
- 流媒体 = 水管一直流水，边流边用（边下载边播放）
- RTP = 水管规格（传输协议）
- RTSP = 水龙头开关（控制协议）

### DASH（Dynamic Adaptive Streaming over HTTP）

- 基于HTTP的自适应流媒体
- 将视频分成多个片段，每个片段有多种质量
- 客户端根据带宽选择质量
- 使用.mpd（Media Presentation Description）索引文件
- 与HLS类似，但使用不同格式

### 流媒体QoS（服务质量）

| 指标 | 说明 | 典型值 |
|------|------|--------|
| 延迟 | 端到端延迟 | < 200ms（实时） |
| 抖动 | 延迟变化 | < 50ms |
| 丢包率 | 丢失的数据包比例 | < 1% |
| 带宽 | 数据传输速率 | 根据视频质量 |

### 缓冲策略
- **预缓冲**：播放前缓冲一定量数据（如5秒）
- **自适应缓冲**：根据网络状况动态调整缓冲大小
- **码率切换**：带宽下降时降低视频质量

### 流媒体协议对比

| 协议 | 传输层 | 延迟 | 可靠性 | 用途 |
|------|--------|------|--------|------|
| RTP/UDP | UDP | 低 | 不可靠 | 实时视频会议 |
| RTSP | TCP | 中 | 可靠 | 流媒体控制 |
| HLS | HTTP/TCP | 高 | 可靠 | 直播/点播 |
| DASH | HTTP/TCP | 高 | 可靠 | 自适应流 |
| WebRTC | UDP | 低 | 可靠(用户态) | 浏览器实时通信 |

### RTP头部字段

| 字段 | 位数 | 作用 |
|------|------|------|
| 版本(V) | 2 | RTP版本号 |
| 填充(P) | 1 | 是否有填充字节 |
| 扩展(X) | 1 | 是否有扩展头部 |
| CSRC计数 | 4 | CSRC标识符数量 |
| 标记(M) | 1 | 帧边界标记 |
| 负载类型(PT) | 7 | 媒体编码类型 |
| 序列号 | 16 | 检测丢包和乱序 |
| 时间戳 | 32 | 同步和抖动消除 |
| SSRC | 32 | 同步源标识符 |

### 408常考要点
- 流媒体使用RTP/UDP传输实时数据
- RTSP用于控制RTP流（播放/暂停/停止）
- HLS基于HTTP，适合CDN分发
- 流媒体容忍丢失（少量丢失不影响观看）
- RTP头部包含序列号和时间戳

## 协议关联

- **RTP与UDP**：RTP使用UDP传输
- **RTSP与RTP**：RTSP控制RTP流
- **HLS与HTTP**：HLS基于HTTP
- **WebRTC与P2P**：WebRTC支持浏览器P2P实时通信
- **408考点**：流媒体基本概念、RTP的作用
