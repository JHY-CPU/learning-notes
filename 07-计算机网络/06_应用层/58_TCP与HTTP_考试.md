# 59_TCP与HTTP_考试

## 核心概念

- **408常考题型**：选择题为主，涉及TCP与HTTP的关系
- **关键考点**：
  - HTTP基于TCP
  - TCP三次握手增加1 RTT延迟
  - 持久连接复用TCP减少握手
  - HTTP/2多路复用但受TCP队头阻塞影响
- **易混淆点**：
  - HTTP本身无状态，但TCP是有连接的
  - HTTP/2用TCP，HTTP/3用UDP

## 原理分析

### 典型考题1：TCP握手延迟

**题目**：HTTP/1.1首次访问Web页面，TCP握手和HTTP请求响应各需要1 RTT，总延迟是（  ）
A. 1 RTT
B. 2 RTT
C. 3 RTT
D. 4 RTT

**答案**：B

**解析**：
- TCP握手：1 RTT
- HTTP请求响应：1 RTT
- 总计：2 RTT

### 典型考题2：持久连接

**题目**：HTTP/1.1持久连接复用TCP连接的好处是（  ）
A. 减少TCP握手次数
B. 增加安全性
C. 减少DNS查询
D. 增加带宽

**答案**：A

**解析**：
- 持久连接复用TCP，避免每次请求都握手
- 减少TCP握手次数，降低延迟

### 典型考题3：HTTP/2与TCP

**题目**：HTTP/2多路复用（  ）
A. 完全解决了队头阻塞
B. 解决了HTTP层队头阻塞
C. 没有解决队头阻塞
D. 使用UDP协议

**答案**：B

**解析**：
- HTTP/2多路复用解决HTTP层队头阻塞
- TCP层队头阻塞仍存在
- HTTP/2仍使用TCP

## 直观理解

**做题技巧**：
- HTTP基于TCP，首次延迟2 RTT
- 持久连接减少TCP握手次数
- HTTP/2解决HTTP层阻塞，TCP层仍阻塞
- HTTP/3用UDP（QUIC）

### 典型考题4：HTTP版本演进

**题目**：将HTTP/1.0、HTTP/1.1、HTTP/2、HTTP/3的特点进行匹配。

| 版本 | 特点 |
|------|------|
| HTTP/1.0 | 非持久连接，每次请求建立新TCP连接 |
| HTTP/1.1 | 持久连接，管道化，Host头部 |
| HTTP/2 | 多路复用，头部压缩，二进制分帧 |
| HTTP/3 | 基于QUIC（UDP），0-RTT，连接迁移 |

### 典型考题5：RTT计算

**题目**：HTTP/1.1持久非流水线，访问含1个HTML和3个图片（同一服务器），TCP握手10ms，每个HTTP请求响应5ms。总时间是多少？

**答案**：
- TCP握手：10ms
- HTML请求响应：5ms
- 3个图片：3 × 5 = 15ms
- 总计：10 + 5 + 15 = 30ms

**解析**：持久连接复用TCP，无需重复握手。

### 典型考题6：队头阻塞

**题目**：以下哪个版本解决了TCP层的队头阻塞？
A. HTTP/1.1
B. HTTP/2
C. HTTP/3
D. HTTP/1.0

**答案**：C

**解析**：
- HTTP/2解决了HTTP层队头阻塞，但TCP层队头阻塞仍存在
- HTTP/3使用QUIC（基于UDP），解决了TCP层队头阻塞
- QUIC在用户态实现可靠传输，避免TCP的队头阻塞

### HTTP版本详细对比

| 特性 | HTTP/1.0 | HTTP/1.1 | HTTP/2 | HTTP/3 |
|------|----------|----------|--------|--------|
| 连接 | 非持久 | 持久 | 持久 | 持久 |
| 并行 | 不支持 | 管道化 | 多路复用 | 多路复用 |
| 队头阻塞 | 有 | 有 | TCP层有 | 无 |
| 头部压缩 | 无 | 无 | HPACK | QPACK |
| 传输层 | TCP | TCP | TCP | QUIC(UDP) |
| 服务器推送 | 无 | 无 | 有 | 有 |
| 默认RTT | 2/对象 | 2+对象数 | 3 | 1-2 |

### 408常考要点
- HTTP基于TCP，首次访问需要TCP握手（+1 RTT）
- 持久连接复用TCP，避免重复握手
- HTTP/2多路复用解决HTTP层队头阻塞，TCP层仍阻塞
- HTTP/3用QUIC（UDP），解决TCP层队头阻塞
- RTT计算是综合题高频考点

## 协议关联

- **TCP与HTTP**：HTTP基于TCP
- **TCP与HTTP/2**：HTTP/2用TCP但多路复用
- **TCP与HTTP/3**：HTTP/3用QUIC（UDP）
- **QUIC与UDP**：QUIC在UDP上实现可靠传输
- **408常见组合**：TCP握手 + HTTP请求 = RTT计算题
