核心概念速查

// ========== 协议栈 ========== // 应用层: HTTP/HTTPS, DNS, FTP, SMTP // 传输层: TCP (可靠), UDP (快速) // 网络层: IP, ICMP, ARP // 接口层: Ethernet, WiFi // ========== HTTP ========== // HTTP/1.1: 文本, Keep-Alive, 队头阻塞 // HTTP/2: 二进制, 多路复用, HPACK, 服务器推送 // HTTP/3: QUIC (UDP), 0-RTT, 无队头阻塞, 连接迁移 // ========== 关键端口 ========== // 80 (HTTP) | 443 (HTTPS) | 22 (SSH) | 3306 (MySQL) // 5432 (PostgreSQL) | 6379 (Redis) | 27017 (MongoDB) // ========== 状态码速查 ========== // 200 OK | 201 Created | 204 No Content // 301/302 重定向 | 304 缓存 // 400/401/403/404 | 422/429 // 500/502/503/504 // ========== REST API 设计 ========== // 正确用 HTTP 方法: GET(查) POST(增) PUT(全改) PATCH(部分) DELETE(删) // 资源 = 名词: /users, /articles, /orders // 状态码 = 精确: 201创建, 204删除, 422验证 // ========== 安全 ========== // HTTPS: TLS 1.2+/ TLS 1.3 // HSTS: 强制 HTTPS // Cookie: HttpOnly + Secure + SameSite // 缓存: Cache-Control / ETag

常见面试问题

// ========== 网络基础面试题 ========== // // Q: 浏览器输入 URL 到页面显示发生了什么? // A: DNS解析 → TCP连接 → TLS握手 → HTTP请求 → 服务器处理 // → HTTP响应 → 浏览器解析渲染 → 加载子资源 // // Q: TCP 为什么是三次握手不是两次? // A: 防止已过期的 SYN 建立连接 (二义性) // // Q: HTTP/2 解决了什么问题? // A: HTTP/1.1 队头阻塞, 头部冗余, 无法多路复用 // // Q: HTTPS 的工作原理? // A: TLS握手 → 证书验证 → 密钥交换 → 对称加密通信 // // Q: 什么是幂等? // A: 多次请求结果与一次相同. GET/PUT/DELETE 幂等, POST/PATCH 不幂等 // // Q: REST 和 gRPC 各自适合什么场景? // A: REST → 外部API/gRPC → 微服务内部通信 // ========== 开发者必备工具 ========== // curl: HTTP 调试 // dig: DNS 查询 // openssl: 证书/TLS 检查 // nc/netcat: TCP/UDP 测试 // ping: 连通性测试 // traceroute: 路由追踪 // netstat/ss: 网络状态 // Wireshark: 网络抓包分析
