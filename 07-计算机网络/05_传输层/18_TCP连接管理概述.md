# 19_TCP连接管理概述

## 核心概念

- **TCP连接管理**：包括连接建立（三次握手）和连接释放（四次挥手）
- **连接建立的目的**：协商双方的初始序号（ISN）、确认双方的接收能力、分配缓冲区等资源
- **连接释放的目的**：双方确认数据发送完毕，释放资源
- **TCP连接的三个阶段**：连接建立 → 数据传输 → 连接释放

### 连接管理的状态变迁

```
服务器: CLOSED → LISTEN → [收到SYN] → SYN_RCVD → [收到ACK] → ESTABLISHED
客户端: CLOSED → [发送SYN] → SYN_SENT → [收到SYN+ACK] → ESTABLISHED
```

## 原理分析

### 为什么需要连接管理

1. **同步序号**：双方必须知道对方的初始序号
2. **分配资源**：为连接分配发送/接收缓冲区
3. **协商参数**：如最大报文段长度（MSS）、窗口大小
4. **防止旧连接干扰**：通过三次握手防止已过期的连接请求被误处理

### 连接建立的两种方式

**两次握手的问题**：
```
客户端 → SYN → 服务器
客户端 ← SYN+ACK ← 服务器
```
问题：如果客户端的SYN在网络中延迟了很久（旧SYN），到达服务器后服务器以为是新连接，分配资源等待数据，但客户端根本没有发起连接，导致服务器资源浪费。

**三次握手解决此问题**：
```
客户端 → SYN → 服务器
客户端 ← SYN+ACK ← 服务器
客户端 → ACK → 服务器
```
旧SYN到达服务器后，服务器回复SYN+ACK，但客户端不会发送第三个ACK（因为它没有发起连接），服务器收不到ACK就不会建立连接。

## 直观理解

- **三次握手像互相确认身份**：
  - A："喂，你能听到吗？"（SYN）
  - B："能听到，你能听到我吗？"（SYN+ACK）
  - A："我也能听到你，我们开始通话吧。"（ACK）
- **两次握手的漏洞**：就像只说了一句话"喂"对方就接通了，可能接了个骚扰电话

## 协议关联

- **与TCP状态转换**：连接管理涉及多个TCP状态（LISTEN、SYN_SENT、SYN_RCVD、ESTABLISHED、FIN_WAIT等）
- **与网络安全**：SYN Flood攻击利用三次握手的半连接状态进行DDoS攻击
- **408重点**：三次握手和四次挥手的过程、状态转换、序号变化是必考内容
