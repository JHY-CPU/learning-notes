# 50_TCP发送窗口详解

## 核心概念

- **发送窗口三个指针**：SND.UNA（最早未确认）、SND.NXT（下一个发送）、SND.UNA + SND.WND（窗口右边界）
- **可用窗口 = SND.WND - (SND.NXT - SND.UNA)**
- **窗口大小约束**：$\text{SND.WND} = \min(\text{rwnd}, \text{cwnd})$

### 发送窗口示意

```
         SND.UNA        SND.NXT          SND.UNA+SND.WND
            ↓              ↓                     ↓
|← 已确认 →|← 已发未确认 →|← 可以发送 →|← 不可发送 →|
```

- **SND.UNA**：最早的未被确认的序号（窗口左边界）
- **SND.NXT**：下一个要发送的序号
- **SND.WND**：窗口大小

## 原理分析

### 窗口滑动过程

**状态1**：初始，窗口大小=5000
```
SND.UNA=1, SND.NXT=1, SND.WND=5000
可发送字节1~5000
```

**状态2**：发送了2000字节
```
SND.UNA=1, SND.NXT=2001, SND.WND=5000
已发未确认：1~2000
可发送：2001~5000
```

**状态3**：收到ACK=2001
```
SND.UNA=2001, SND.NXT=2001, SND.WND=5000
窗口右移2000字节
可发送：2001~7000
```

### 窗口大小变化的影响

- rwnd增大：窗口右边界右移，可发送更多数据
- rwnd减小：窗口右边界左移（**窗口收缩**），可能需要暂停发送
- cwnd变化：同理影响窗口大小

### 序号回绕问题

- 序号32位，最大$2^{32}-1 \approx 4GB$
- 高速网络中，序号可能在数据存活期内回绕
- **解决**：PAWS机制或使用大序号空间

### 窗口计算例题

**例题1**：SND.UNA=1001, SND.NXT=3001, SND.WND=5000
- 已发未确认：1001~3000（2000字节）
- 可发送：3001~6000（3000字节）
- 可用窗口 = 5000 - (3001-1001) = 3000字节

**例题2**：收到ACK=2001后
- SND.UNA=2001, SND.NXT=3001, SND.WND=5000
- 窗口覆盖：2001~7000
- 已发未确认：2001~3000（1000字节）
- 可发送：3001~7000（4000字节）

**例题3**：SND.WND变为3000后（假设SND.UNA=2001, SND.NXT=3001）
- 窗口覆盖：2001~5000
- 已发未确认：2001~3000（1000字节）
- 可发送：3001~5000（2000字节）
- 注意：已发送的3001~5000仍在窗口内，无需重传

### 窗口收缩问题
当rwnd减小导致SND.WND缩小：
- 如果新窗口右边界 < 旧窗口右边界，发生窗口收缩
- 收缩部分的数据如果已发送，仍等待ACK
- 如果未发送，变为不可发送

### 序号回绕保护（PAWS）
- TCP序号32位，最大4GB
- 高速网络中序号可能回绕
- PAWS使用TCP时间戳选项
- 旧连接的报文段被识别并丢弃

### 408常考要点
- SND.UNA：最早未确认序号（窗口左边界）
- SND.NXT：下一个要发送的序号
- 可用窗口 = SND.WND - (SND.NXT - SND.UNA)
- ACK使SND.UNA右移，窗口整体右移
- 窗口大小 = min(rwnd, cwnd)
- 序号32位，最大4GB

## 直观理解

- **三个指针像工程进度**：SND.UNA是"已验收部分"的右边界，SND.NXT是"正在施工"的位置，SND.WND是"可以施工"的总范围
- **窗口滑动像传送带**：已确认的部分滑走，新的空间出现

## 协议关联

- **与流量控制**：rwnd动态调整窗口右边界
- **与拥塞控制**：cwnd动态调整窗口右边界
- **408常见考法**：画窗口滑动图、计算各指针位置
- **序号回绕**是高速网络中的实际问题
