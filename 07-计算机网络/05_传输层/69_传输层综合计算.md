# 70_传输层综合计算

## 核心概念

- **传输层综合计算题涉及序号、确认号、窗口、拥塞控制等多个知识点**
- **需要系统掌握所有公式和规则**

### 常见计算类型

1. **序号确认号计算**
2. **窗口大小计算**
3. **拥塞控制cwnd变化计算**
4. **RTO计算**
5. **吞吐量计算**

## 原理分析

### 综合计算示例

**题目**：TCP连接的ISN_C=100，ISN_S=500，MSS=1000B，RTT=20ms，rwnd=8000B，ssthresh初始=8 MSS。

三次握手后，客户端连续发送数据：
- 第1个RTT：cwnd=1，发送1000B（SEQ=101）
- 第2个RTT：cwnd=2，发送2000B
- ...
- 第5个RTT：cwnd=8（达到ssthresh），进入拥塞避免
- 第6个RTT：cwnd=9

此时服务器确认了所有数据，客户端的下一个SEQ = 101 + 所有已发送字节。

**发送窗口约束**：
$$\text{发送窗口} = \min(\text{rwnd}, \text{cwnd} \times \text{MSS})$$

当cwnd=8时，cwnd×MSS=8000B，min(8000, 8000)=8000B，可以连续发8个MSS。

**最大吞吐量**：
$$\text{吞吐量} = \frac{\text{发送窗口}}{\text{RTT}}$$

当cwnd=8, rwnd=8000时：吞吐量 = 8000B / 20ms = 400KB/s

### 序号计算技巧

1. SYN消耗1个序号，FIN消耗1个序号
2. 确认号 = 已收到的最后一个字节序号 + 1
3. 数据报文段的SEQ = 上一个报文段的SEQ + 上一个报文段的数据长度
4. 第三次握手后，数据起始序号 = ISN + 2（SYN+第三次握手ACK各消耗1个）

### 更多综合计算例题

**例题2**：TCP三次握手，ISN_C=100, ISN_S=500。
- 第一次握手：SYN=1, SEQ=100
- 第二次握手：SYN=1, ACK=1, SEQ=500, ACK号=101
- 第三次握手：ACK=1, SEQ=101, ACK号=501
- 数据传输起始序号：客户端从101开始，服务器从501开始

**例题3**：TCP连接cwnd=16 MSS, ssthresh=8 MSS, rwnd=20 KB, MSS=1 KB, RTT=20ms
- 发送窗口 = min(20KB, 16×1KB) = 16KB
- 最大吞吐量 = 16KB / 20ms = 800 KB/s
- 发生超时后：ssthresh=8, cwnd=1
- 第5个RTT时：cwnd=1→2→4→8→16（慢开始到16？不对，到8进入拥塞避免）
  - RTT1: 1→2
  - RTT2: 2→4
  - RTT3: 4→8（达到ssthresh，进入拥塞避免）
  - RTT4: 8→9
  - RTT5: 9→10

### 计算公式汇总
1. **发送窗口** = min(rwnd, cwnd)
2. **最大吞吐量** = 发送窗口 / RTT
3. **BDP** = 带宽 × RTT
4. **RTO** = SRTT + 4×RTTVAR
5. **慢开始**：cwnd每RTT翻倍
6. **拥塞避免**：cwnd每RTT+1
7. **超时**：ssthresh=cwnd/2, cwnd=1
8. **3冗余ACK**：ssthresh=cwnd/2, cwnd=ssthresh+3

### 序号计算规则
1. SYN消耗1个序号
2. FIN消耗1个序号
3. 数据段的SEQ = 上一段SEQ + 上一段数据长度
4. 确认号 = 期望收到的下一个序号 = 已收到的最后一个序号 + 1

### 408常考要点
- SYN和FIN各消耗1个序号
- 发送窗口 = min(rwnd, cwnd)
- 慢开始是指数增长，拥塞避免是线性增长
- 超时比3冗余ACK严重（cwnd=1 vs cwnd=ssthresh+3）
- 综合题需要一步步计算，不能跳步

## 直观理解

- **计算步骤**：①确定ISN ②计算各报文段的SEQ和ACK ③考虑SYN/FIN消耗的序号 ④结合窗口和拥塞控制
- **综合计算像"拼图"**——每个知识点是一块拼图，组合起来才是完整答案

## 协议关联

- **408综合题**：常将序号确认号、窗口、拥塞控制结合考查
- **解题关键**：细心、系统、一步步来
- **常见错误**：忘记SYN/FIN消耗序号、混淆超时和冗余ACK处理
