# 62_TCP状态转换_概念_真题

## 核心概念

- **状态转换图是408高频考点**
- **需要结合三次握手和四次挥手理解**

### 408真题精选

**题目**：主机甲和乙建立TCP连接，甲主动发起连接。甲发送SYN后处于什么状态？乙收到SYN后处于什么状态？

**解析**：
- 甲（客户端）：**SYN_SENT**
- 乙（服务器）：收到SYN后发送SYN+ACK，进入**SYN_RCVD**

**题目**：TCP连接建立后，客户端主动关闭连接。在四次挥手过程中，客户端的状态变化依次是什么？

**解析**：
1. 发送FIN → **FIN_WAIT_1**
2. 收到ACK → **FIN_WAIT_2**
3. 收到FIN，发送ACK → **TIME_WAIT**
4. 2MSL超时 → **CLOSED**

**题目**：TCP连接释放过程中，服务器的状态变化依次是什么？

**解析**：
1. 收到FIN，发送ACK → **CLOSE_WAIT**
2. 发送FIN → **LAST_ACK**
3. 收到ACK → **CLOSED**

**题目**：一个TCP连接的客户端在FIN_WAIT_1状态时收到服务器发来的FIN（且没有收到过ACK）。客户端将进入什么状态？

**解析**：**CLOSING**状态。这是因为双方同时发送了FIN，客户端还没收到ACK就收到了对方的FIN，进入CLOSING（双方同时关闭）。

**题目**：服务器出现大量SYN_RCVD状态的连接，可能是什么原因？

**解析**：可能是**SYN Flood攻击**。攻击者发送大量SYN报文段但不完成第三次握手，导致服务器的半连接队列（SYN_RCVD状态的连接）溢出，无法处理正常连接请求。

**题目**：TCP连接的客户端在TIME_WAIT状态等待2MSL期间，如果收到对FIN的重传ACK会怎样？

**解析**：客户端重新发送ACK，并重置2MSL计时器。这是TIME_WAIT存在的目的之一——确保最后一个ACK到达服务器。

## 直观理解

- **做题技巧**：先判断是客户端还是服务器（主动/被动），再根据事件一步步推状态

## 协议关联

- **与网络编程**：connect()、listen()、accept()、close()分别触发状态转换
- **408常见陷阱**：同时关闭进入CLOSING、同时打开双方都经过SYN_SENT和SYN_RCVD
