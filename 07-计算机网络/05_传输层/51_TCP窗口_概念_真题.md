# 52_TCP窗口_概念_真题

## 核心概念

- **发送窗口 = min(rwnd, cwnd)**
- **窗口以字节为单位**
- **TCP首部窗口字段16位**，最大65535字节
- **窗口扩大选项可扩展窗口大小**

### 408真题精选

**题目**：TCP连接的接收窗口rwnd=3000B，拥塞窗口cwnd=2000B，发送方能够发送的最大数据量是多少？

**解析**：
$$\text{发送窗口} = \min(3000, 2000) = 2000\text{B}$$

**题目**：TCP发送窗口大小是否可以超过接收窗口大小？

**解析**：不可以。发送窗口 = min(rwnd, cwnd) ≤ rwnd。但拥塞窗口cwnd可以超过rwnd，只是实际发送窗口被rwnd限制。

**题目**：当TCP接收方缓冲区满了，rwnd=0，发送方会怎么做？

**解析**：
1. 发送方停止发送数据
2. 启动**持续计时器**
3. 超时后发送**零窗口探测报文段**（1字节）
4. 接收方回复当前rwnd
5. 如果rwnd>0则恢复发送

**题目**：设TCP的MSS=1KB，RTT=10ms，发送窗口=10KB。若不考虑拥塞控制，该TCP连接的最大数据传输速率是多少？

**解析**：
$$\text{最大速率} = \frac{10\text{KB}}{10\text{ms}} = \frac{10240\text{B}}{0.01\text{s}} = 1024000\text{B/s} \approx 1\text{MB/s}$$

**题目**：TCP的窗口字段只有16位，如何支持大于65535字节的窗口？

**解析**：使用**窗口扩大选项（Window Scale）**。在三次握手时协商扩大因子S，实际窗口 = 窗口字段值 × $2^S$。S最大为14，所以最大窗口 = 65535 × $2^{14}$ ≈ 1GB。

### 更多真题

**题目**：TCP连接cwnd=32KB，ssthresh=16。发生超时后，第几个RTT时cwnd=8KB？

**解**：
- 超时后：ssthresh=16KB/2=8KB? 不对，题目中ssthresh=16单位应该是MSS
- cwnd=1, ssthresh=16（MSS）
- RTT1: cwnd=1→2
- RTT2: cwnd=2→4
- RTT3: cwnd=4→8（达到ssthresh=16? 题目需要明确单位）

更正：假设cwnd=32 MSS, ssthresh=16 MSS
- 超时后：ssthresh=16, cwnd=1
- RTT1: cwnd=1→2
- RTT2: cwnd=2→4
- RTT3: cwnd=4→8（慢开始继续）
- RTT4: cwnd=8→16（达到ssthresh，进入拥塞避免）
- RTT5: cwnd=16→17

**题目**：TCP的Window Scale选项协商了扩大因子S=3。窗口字段值为8192。实际窗口大小？
**解**：实际窗口 = 8192 × 2^3 = 8192 × 8 = 65536字节 = 64KB

**题目**：以下关于TCP窗口的描述，正确的是？
A. 发送窗口可以大于rwnd
B. rwnd可以大于cwnd
C. cwnd由接收方控制
D. 发送窗口由接收方单独决定

**解**：选**B**
- A错：发送窗口 = min(rwnd, cwnd) <= rwnd
- B对：cwnd和rwnd独立计算，rwnd可以大于cwnd
- C错：cwnd由发送方计算
- D错：发送窗口受rwnd和cwnd共同限制

### 408易错点
- 发送窗口 = min(rwnd, cwnd)，取最小值
- 窗口以字节为单位，不是报文段
- TCP首部窗口字段16位，最大65535字节
- 窗口扩大选项在三次握手时协商
- rwnd=0时不是永远停止，而是发送探测报文
- 窗口可以动态变化（增大或减小）

## 直观理解

- **窗口像限流闸门**：接收方和网络各自有一个闸门，取开得小的那个
- **窗口扩大选项像"放大镜"**——让16位字段表示更大的数字

## 协议关联

- **与拥塞控制**：cwnd变化影响窗口
- **与应用层**：应用层读取速度影响rwnd
- **408常见陷阱**：
  - "发送窗口等于rwnd"——不一定，还要和cwnd取最小
  - "窗口可以为负数"——错误，最小为0
- 窗口扩大选项解决了高速网络中16位字段的限制
