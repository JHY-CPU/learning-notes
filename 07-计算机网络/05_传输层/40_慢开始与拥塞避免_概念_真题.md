# 41_慢开始与拥塞避免_概念_真题

## 核心概念

- **慢开始阶段cwnd指数增长**
- **拥塞避免阶段cwnd线性增长**
- **超时后cwnd从1开始**
- **3个冗余ACK后cwnd从ssthresh+3开始**

### 408真题精选

**题目**：一个TCP连接的ssthresh=32 MSS，cwnd=1 MSS。慢开始算法在第几轮次cwnd会达到或超过ssthresh？

**解析**：
- 第1轮：cwnd=1
- 第2轮：cwnd=2
- 第3轮：cwnd=4
- 第4轮：cwnd=8
- 第5轮：cwnd=16
- 第6轮：cwnd=32（达到ssthresh，转入拥塞避免）

答案：**第6轮次**

**题目**：TCP连接的cwnd=32，ssthresh=16。此时发生超时，超时后cwnd和ssthresh各为多少？

**解析**：
- 超时 → ssthresh = cwnd/2 = 32/2 = **16**
- cwnd = **1** MSS
- 重新进入慢开始

**题目**：TCP连接的cwnd=32，ssthresh=16。此时收到3个冗余ACK，之后cwnd和ssthresh各为多少？

**解析**：
- 3个冗余ACK → ssthresh = cwnd/2 = 32/2 = **16**
- cwnd = ssthresh + 3 = 16 + 3 = **19**
- 进入拥塞避免（跳过慢开始）

**题目**：设TCP的ssthresh=12 MSS，cwnd=1 MSS。在传输过程中，当cwnd达到16时发生超时，求超时后第7个RTT时的cwnd值。

**解析**：
1. 超时时：ssthresh = 16/2 = 8，cwnd = 1
2. 超时后各RTT：
   - RTT1: cwnd=1（慢开始）
   - RTT2: cwnd=2
   - RTT3: cwnd=4
   - RTT4: cwnd=8（达到ssthresh，进入拥塞避免）
   - RTT5: cwnd=9（拥塞避免，+1）
   - RTT6: cwnd=10
   - RTT7: cwnd=**11**

答案：**11 MSS**

### 更多真题

**题目**：TCP的ssthresh=8 MSS，cwnd=1 MSS。传输过程中发生以下事件：
1. cwnd增长到16时超时
2. 之后收到3个冗余ACK（此时cwnd刚恢复到8）

求两次事件后的ssthresh和cwnd。

**解析**：
- 事件1（超时）：ssthresh = 16/2 = 8，cwnd = 1
- 慢开始恢复：1→2→4→8（达到ssthresh）
- 事件2（3冗余ACK）：ssthresh = 8/2 = 4，cwnd = 4+3 = 7
- 进入拥塞避免

**题目**：TCP Reno与TCP Tahoe的主要区别是？
**答**：
- Tahoe：超时和3冗余ACK都回到慢开始（cwnd=1）
- Reno：超时回慢开始（cwnd=1），3冗余ACK进入快速恢复（cwnd=ssthresh+3）

### 拥塞控制四种算法总结
| 算法 | 触发条件 | cwnd变化 | ssthresh变化 |
|------|----------|----------|-------------|
| 慢开始 | 连接建立/超时后 | 每RTT翻倍 | - |
| 拥塞避免 | cwnd>=ssthresh | 每RTT+1 | - |
| 快速恢复 | 3冗余ACK | cwnd=ssthresh+3 | cwnd/2 |
| 超时处理 | RTO超时 | cwnd=1 | cwnd/2 |

### 408常考要点
- 慢开始是指数增长（每RTT翻倍），不是"慢慢增长"
- ssthresh是慢开始和拥塞避免的分界点
- 超时比3冗余ACK严重得多（cwnd=1 vs cwnd=ssthresh+3）
- 实际发送窗口 = min(rwnd, cwnd)
- 快速恢复跳过慢开始阶段

## 直观理解

- **记住关键规则**：超时=打回原形(cwnd=1)，冗余ACK=减半继续(cwnd=ssthresh+3)
- **慢开始像加速起步**：一开始指数加速，看到限速标志（ssthresh）后改为线性加速
- **拥塞避免像巡航模式**：速度慢慢增加，遇到问题（丢包）立即减速

## 协议关联

- **与流量控制**：实际窗口 = min(rwnd, cwnd)
- **408常见陷阱**：
  - "超时和冗余ACK处理相同"——错误，超时更严重
  - "cwnd达到ssthresh时立即转为拥塞避免"——正确
  - "慢开始增长很慢"——错误，指数增长其实很快
- 拥塞控制算法的不同版本（Tahoe/Reno/NewReno/Cubic）处理方式不同
