# 81_传输层协议对比总结

## 核心概念

- **传输层主要协议**：TCP和UDP
- **两者共同点**：都提供进程到进程的通信、都使用端口号、都有校验和
- **两者不同点**：连接性、可靠性、数据形式、拥塞控制等

### TCP vs UDP 完整对比

| 特征 | TCP | UDP |
|------|-----|-----|
| 全称 | Transmission Control Protocol | User Datagram Protocol |
| 连接 | 面向连接 | 无连接 |
| 可靠性 | 可靠 | 不可靠 |
| 数据形式 | 字节流 | 数据报（保留边界） |
| 通信方式 | 点对点 | 一对一/一对多/多对多 |
| 首部长度 | 最少20字节，最多60字节 | 固定8字节 |
| 首部字段 | 10+个 | 4个 |
| 流量控制 | 有（rwnd） | 无 |
| 拥塞控制 | 有（cwnd, AIMD） | 无 |
| 有序性 | 保证 | 不保证 |
| 重传 | 有（超时/快速重传） | 无 |
| 连接建立 | 三次握手 | 无 |
| 连接释放 | 四次挥手 | 无 |
| 状态机 | 11个状态 | 无 |
| 计时器 | 4种 | 无 |
| 校验和 | 必须（含伪首部） | IPv4可选，IPv6必须 |
| 多播/广播 | 不支持 | 支持 |
| 延迟 | 较高 | 低 |
| 开销 | 大 | 小 |

## 原理分析

### 选择依据

**选TCP**：
- 数据完整性关键（文件、邮件、网页）
- 大数据量传输
- 需要按序到达
- 点对点通信

**选UDP**：
- 实时性关键（视频会议、语音通话）
- 小数据量简单查询（DNS、DHCP）
- 需要广播/组播
- 应用层自行实现可靠性（QUIC、TFTP）

### TCP的四个计时器
| 计时器 | 用途 |
|--------|------|
| 重传计时器 | 检测报文段丢失 |
| 持续计时器 | 防止零窗口死锁 |
| 保活计时器 | 检测连接是否存活 |
| TIME_WAIT计时器 | 等待2MSL |

### TCP的11个状态
| 状态 | 说明 |
|------|------|
| CLOSED | 初始/终止状态 |
| LISTEN | 服务器等待连接 |
| SYN_SENT | 客户端已发送SYN |
| SYN_RCVD | 服务器收到SYN |
| ESTABLISHED | 连接已建立 |
| FIN_WAIT_1 | 主动关闭方已发FIN |
| FIN_WAIT_2 | 等待对方FIN |
| CLOSE_WAIT | 被动关闭方收到FIN |
| CLOSING | 同时关闭 |
| LAST_ACK | 等待最后ACK |
| TIME_WAIT | 等待2MSL |

### 408常考要点
- TCP首部最少20字节，UDP固定8字节
- TCP端口号16位（0~65535）
- TCP有11个状态，UDP无状态
- TCP有4种计时器，UDP无计时器
- TCP支持流量控制和拥塞控制
- UDP支持广播和组播
- IP协议号：TCP=6，UDP=17
- 校验和：TCP必须，UDP IPv4可选IPv6必须

## 直观理解

- **TCP = 可靠快递（挂号信）**：签收确认、丢失补发、按序送达
- **UDP = 快速投递（平信）**：直接发、不管收没收到、快但不保证
- **TCP像"精密机器"**——有很多状态和控制机制
- **UDP像"简单管道"**——数据来了就发，不加工

## 协议关联

- **与应用层**：HTTP/FTP/SMTP用TCP；DNS/DHCP/RTP用UDP
- **与网络层**：两者都封装在IP中，IP协议号6=TCP、17=UDP
- **408常见考法**：对比选择、应用场景判断
- **传输层是408考试的重点章节**，占比约15-20分
