# 72_TCP拥塞窗口变化图

## 核心概念

- **cwnd变化图是408经典考点**
- **图中横轴为传输轮次（RTT），纵轴为cwnd（MSS）**
- **图像呈锯齿形**：慢开始/拥塞避免上升 → 丢包骤降 → 再上升

### 标准cwnd变化图特征

```
cwnd(MSS)
  |
  |        /\      /\
  |       /  \    /  \
  |      /    \  /    \
  |     /      \/      \
  |    /                 (锯齿形)
  |   /
  |  /
  | /
  +------------------------→ 轮次(RTT)
```

### 两种丢包的不同表现

**超时**：
- cwnd骤降到**1**
- ssthresh减半
- 重新慢开始（指数增长）

**3个冗余ACK**：
- cwnd降到**ssthresh+3**
- ssthresh减半
- 直接拥塞避免（线性增长）
- 下降幅度比超时小

## 原理分析

### 典型的cwnd变化过程

```
cwnd
32|                          *
  |                        *   超时→cwnd=1
24|                      *
  |                    *
16|                  *
  |               *
 8|            *  ← ssthresh=16
 4|         *
 2|      *
 1|   *
  +--+--+--+--+--+--+--+--+--→ RTT
  1  2  3  4  5  6  7  8  9
```

**各阶段**：
1. RTT 1-4：慢开始（1→2→4→8→16）
2. RTT 5-8：拥塞避免（16→17→...→19→...→超时时cwnd=24，假设）
3. RTT 9：超时，cwnd=1，ssthresh=12
4. RTT 10-13：慢开始（1→2→4→8→16）
5. ...

### 画图步骤
1. 标注ssthresh初始值
2. 慢开始阶段：每RTT cwnd翻倍
3. cwnd >= ssthresh时转拥塞避免
4. 拥塞避免阶段：每RTT cwnd+1
5. 发生丢包时：
   - 超时：cwnd降到1，ssthresh=cwnd/2
   - 3冗余ACK：cwnd=ssthresh+3，ssthresh=cwnd/2
6. 重复步骤2-5

### 典型计算题
**题目**：ssthresh初始=16 MSS，cwnd从1开始。cwnd到达20时发生超时。画出前12个RTT的cwnd变化。

**解**：
| RTT | cwnd | 阶段 |
|-----|------|------|
| 1 | 1 | 慢开始 |
| 2 | 2 | 慢开始 |
| 3 | 4 | 慢开始 |
| 4 | 8 | 慢开始 |
| 5 | 16 | 转拥塞避免 |
| 6 | 17 | 拥塞避免 |
| 7 | 18 | 拥塞避免 |
| 8 | 19 | 拥塞避免 |
| 9 | 20 | 超时！ssthresh=10, cwnd=1 |
| 10 | 2 | 慢开始 |
| 11 | 4 | 慢开始 |
| 12 | 8 | 慢开始 |

### 3冗余ACK与超时的区别
**超时**（更严重）：
- cwnd直接降到1
- 重新从慢开始

**3冗余ACK**（较轻）：
- cwnd降到ssthresh+3
- 直接进入拥塞避免（跳过慢开始）
- 称为"快速恢复"

### 408常考要点
- 慢开始阶段cwnd指数增长
- 拥塞避免阶段cwnd线性增长
- 超时后cwnd=1，ssthresh减半
- 3冗余ACK后cwnd=ssthresh+3，ssthresh减半
- 图像呈锯齿形（上升→骤降→上升）
- ssthresh是慢开始和拥塞避免的分界

## 直观理解

- **锯齿形像登山**：快速攀爬（慢开始），缓慢登顶（拥塞避免），突然掉下来（丢包），再爬
- **超时像"坠崖"**——直接掉到谷底
- **冗余ACK像"滑坡"**——掉一半，还能继续爬

## 协议关联

- **与拥塞控制四算法**：图中每个阶段对应一个算法
- **与流量控制**：实际窗口取min(rwnd, cwnd)，rwnd变化也会影响图
- **408必考**：给定图画出cwnd变化过程，或给定条件画图
- **TCP Reno vs Tahoe**：Reno在3冗余ACK时快速恢复，Tahoe回到慢开始
