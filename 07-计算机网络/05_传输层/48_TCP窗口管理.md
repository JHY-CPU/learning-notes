# 49_TCP窗口管理

## 核心概念

- **发送窗口 = min(rwnd, cwnd)**，单位是字节
- **发送窗口内包含**：已发送未确认 + 允许发送但未发送
- **接收窗口**：接收方缓冲区剩余空间
- **窗口管理是TCP流量控制和拥塞控制的统一实现**

### 发送窗口的组成

```
|←──── 已确认 ────→|←── 已发送未确认 ──→|←── 可发送 ──→|←── 不可发送 ──→|
|                  |                    |              |               |
    已ACK的字节       发送窗口内已发      发送窗口内未发    窗口外
```

- **已确认**：已收到ACK的数据，可被滑动覆盖
- **已发送未确认**：已发送但还在等ACK
- **可发送（可用窗口）**：窗口内尚未发送的部分
- **不可发送**：超出窗口，需要等ACK后窗口右移

## 原理分析

### 窗口大小的动态调整

**rwnd的变化**：
- 接收方每发送一个ACK，都可能更新rwnd
- rwnd反映接收方缓冲区的实时状态
- rwnd=0时发送方暂停（有持续计时器防死锁）

**cwnd的变化**：
- 慢开始：指数增长
- 拥塞避免：线性增长
- 丢包：减半或归一

### 窗口与缓冲区

- 发送方缓冲区：存放应用层待发送的数据
- 接收方缓冲区：存放已接收但应用层未读取的数据
- 窗口大小反映缓冲区的可用空间

## 直观理解

- **窗口像一个"允许通行证"**：窗口内的数据可以通行，窗口外的数据需要等待
- **窗口大小像限流**：在高速公路上，只允许同时有N辆车在某个路段上

## 协议关联

- **与流量控制**：rwnd限制窗口大小
- **与拥塞控制**：cwnd限制窗口大小
- **与确认机制**：ACK推动窗口滑动
- **408常见考法**：计算窗口大小、分析窗口滑动过程
