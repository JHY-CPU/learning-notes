# 37_TCP拥塞控制概述

## 核心概念

- **拥塞（Congestion）**：网络中过多的数据导致性能下降（延迟增加、丢包率上升、吞吐量下降）
- **拥塞控制（Congestion Control）**：防止过多数据注入网络，避免网络中的路由器或链路过载
- **拥塞控制是全局性的**：涉及所有主机、路由器
- **与流量控制的区别**：流量控制是端到端的（接收方限制发送方），拥塞控制是全局性的（网络限制发送方）

### 拥塞的征兆

1. 丢包（路由器缓冲区溢出）
2. 延迟增大（排队延迟增加）
3. 吞吐量下降

### TCP拥塞控制的四大算法

| 算法 | 阶段 | cwnd变化 |
|------|------|----------|
| 慢开始（Slow Start） | 初始阶段 | 指数增长 |
| 拥塞避免（Congestion Avoidance） | 稳定阶段 | 线性增长 |
| 快重传（Fast Retransmit） | 丢包恢复 | 立即重传 |
| 快恢复（Fast Recovery） | 丢包恢复 | cwnd减半 |

### 拥塞窗口（cwnd）

- **cwnd（Congestion Window）**：发送方维护的拥塞窗口大小
- 发送方实际发送窗口 = **min(rwnd, cwnd)**
- cwnd以**MSS（最大报文段长度）**为单位增长

## 原理分析

### 拥塞控制框架

```
初始: cwnd = 1 MSS
      ↓
慢开始: cwnd指数增长 (1→2→4→8→...)
      ↓
  cwnd ≥ ssthresh?
      ↓ YES
拥塞避免: cwnd线性增长 (+1 MSS/RTT)
      ↓
  丢包？
      ↓
超时: ssthresh = cwnd/2, cwnd = 1, 重新慢开始
冗余ACK: ssthresh = cwnd/2, cwnd = ssthresh + 3, 快恢复
```

### ssthresh（慢开始门限）

- **慢开始门限（Slow Start Threshold）**：区分慢开始和拥塞避免的阈值
- 初始值通常设为 **65535字节** 或 接收方窗口大小
- 当cwnd < ssthresh时，使用慢开始
- 当cwnd ≥ ssthresh时，使用拥塞避免

## 直观理解

- **拥塞控制像开车**：刚开始慢慢加速（慢开始），到了限速就平稳驾驶（拥塞避免），看到红灯急刹车（超时→慢开始），看到黄灯减速（冗余ACK→快恢复）

## 协议关联

- **与流量控制**：发送窗口 = min(rwnd, cwnd)，两者取较小值
- **与路由器的关系**：路由器的主动队列管理（AQM，如RED）配合端系统的拥塞控制
