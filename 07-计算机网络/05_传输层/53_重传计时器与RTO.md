# 54_重传计时器与RTO

## 核心概念

- **重传计时器（Retransmission Timer）**：检测报文段丢失的核心机制
- **RTO（Retransmission Timeout）**：重传超时时间，基于RTT动态计算
- **RTO = SRTT + 4 × RTTVAR**
- **超时后RTO指数退避（加倍）**

### RTO计算公式

$$\text{SRTT} = (1 - \alpha) \times \text{SRTT} + \alpha \times \text{SampleRTT}, \quad \alpha = \frac{1}{8}$$

$$\text{RTTVAR} = (1 - \beta) \times \text{RTTVAR} + \beta \times |\text{SRTT} - \text{SampleRTT}|, \quad \beta = \frac{1}{4}$$

$$\text{RTO} = \text{SRTT} + 4 \times \text{RTTVAR}$$

### RTO的边界约束

- RTO下界：**1秒**（RFC推荐，实际中常用200ms或更小）
- RTO上界：**60秒**（RFC推荐）
- 首次RTO：**3秒**（RFC推荐，在收到第一个RTT样本前使用）

## 原理分析

### 重传计时器的工作流程

```
发送报文段 → 启动计时器
              ↓
        ┌─────┴─────┐
        │            │
    收到ACK       计时器超时
        │            │
   关闭计时器    重传报文段
   更新RTT估计   RTO × 2
                重启计时器
```

### 超时后的处理

1. 重传丢失的报文段
2. RTO加倍（指数退避）
3. 重启重传计时器
4. 如果持续超时，继续重传，RTO继续加倍
5. 通常重传一定次数后放弃（如Linux默认15次）

### 收到ACK后的处理

- 停止重传计时器
- 用新的RTT样本更新SRTT和RTTVAR
- 计算新的RTO
- 为下一个报文段启动新的计时器

### RTO计算详细示例
**初始**：无RTT样本，RTO = 3秒

**第一个RTT样本**：SampleRTT = 100ms
- SRTT = 100ms
- RTTVAR = 100/2 = 50ms
- RTO = 100 + 4×50 = 300ms

**第二个RTT样本**：SampleRTT = 120ms
- SRTT = (7/8)×100 + (1/8)×120 = 102.5ms
- RTTVAR = (3/4)×50 + (1/4)×|102.5-120| = 37.5 + 4.375 = 41.875ms
- RTO = 102.5 + 4×41.875 = 270ms

### Karn算法
- 重传的报文段的RTT样本不用于更新SRTT（因为不确定ACK对应哪次传输）
- 超时重传后，RTO翻倍
- 收到非重传报文段的ACK后，恢复正常RTO计算

### 重传计时器与拥塞控制的关系
| 事件 | RTO变化 | cwnd变化 |
|------|---------|----------|
| 超时 | RTO翻倍 | cwnd=1（慢开始） |
| 收到ACK | RTO正常计算 | cwnd线性增长 |
| 3冗余ACK | 不变 | cwnd=ssthresh+3 |

### 持续计时器 vs 重传计时器
| 计时器 | 用途 | 触发动作 |
|--------|------|----------|
| 重传计时器 | 检测报文段丢失 | 重传报文段 |
| 持续计时器 | 防止零窗口死锁 | 发送零窗口探测 |
| 保活计时器 | 检测连接存活 | 发送保活探测 |

### 408常考要点
- RTO = SRTT + 4×RTTVAR（必须记住）
- SRTT更新公式：α=1/8
- RTTVAR更新公式：β=1/4
- 超时后RTO翻倍（指数退避）
- RTO下界1秒，上界60秒
- Karn算法：重传样本不用于RTT估计
- 超时触发慢开始（cwnd=1）

## 直观理解

- **RTO像耐心值**：第一次没收到回复等一会儿（RTO），再没收到就多等会儿（RTO×2），越来越有耐心（指数退避），直到收到回复恢复正常
- **SRTT像平均血压**：用加权平均平滑RTT波动
- **RTTVAR像血压波动幅度**：波动越大，RTO应该越保守

## 协议关联

- **与拥塞控制**：超时触发慢开始
- **与快速重传**：3个冗余ACK触发快速重传，不需要等RTO
- **408常见考法**：RTO计算公式（必须记住）、指数退避
- **Karn算法**解决了重传样本RTT不确定的问题
