# 51_TCP窗口_概念_计算

## 核心概念

- **发送窗口 = min(rwnd, cwnd)**
- **可用窗口 = 发送窗口 - 已发送未确认的数据量**
- **接收窗口 = 接收方缓冲区大小 - 已接收未读取的数据量**

### 计算题型总结

**题型1：计算实际发送窗口**
$$\text{发送窗口} = \min(\text{rwnd}, \text{cwnd})$$

**题型2：计算最大吞吐量**
$$\text{最大吞吐量} = \frac{\text{发送窗口}}{\text{RTT}}$$

**题型3：计算窗口内可发送的报文段数**
$$\text{报文段数} = \frac{\text{可用窗口}}{\text{MSS}}$$

## 原理分析

### 计算示例1

**题目**：rwnd=8KB，cwnd=4KB，MSS=1KB，RTT=20ms。求发送窗口、最大报文段数、最大吞吐量。

**解析**：
- 发送窗口 = min(8KB, 4KB) = **4KB**
- 最大报文段数 = 4KB / 1KB = **4个**
- 最大吞吐量 = 4KB / 20ms = 4096B / 0.02s ≈ **204.8KB/s**

### 计算示例2

**题目**：发送窗口=5000字节，已发送了序号1~3000的数据，收到了ACK=2001。此时还能发送多少字节新数据？

**解析**：
- 窗口覆盖：2001~7000（5000字节，右移2000字节）
- 已发送未确认：2001~3000（1000字节）
- 可发送新数据：3001~7000，共**4000字节**

### 计算示例3：带rwnd变化的计算

**题目**：初始rwnd=4000，cwnd=4000，发送了4000字节。收到ACK=2001，但rwnd变为3000。此时还能发多少？

**解析**：
- ACK=2001确认了1~2000
- 新的发送窗口 = min(rwnd=3000, cwnd假设不变) = 3000
- 窗口覆盖：2001~5000
- 已发未确认：2001~4000
- 可发送新数据：4001~5000，共**1000字节**

### 更多计算例题

**例题4**：TCP连接MSS=1460B，RTT=40ms，cwnd=65535B。最大吞吐量是多少？

**解**：
- 最大吞吐量 = 65535 / 0.04 = 1,638,375 B/s ≈ 13.1 Mbps

**例题5**：发送窗口=8000B，已发送序号1~6000，收到ACK=4001。还能发多少？

**解**：
- ACK=4001确认了1~4000
- 窗口覆盖：4001~12000（右移4000）
- 已发未确认：4001~6000（2000B）
- 可发送：6001~12000 = **6000B**

**例题6**：rwnd=10KB, cwnd=12KB, MSS=1KB。发送方最多能发几个段？

**解**：
- 发送窗口 = min(10KB, 12KB) = 10KB
- 段数 = 10KB / 1KB = **10个段**

### 窗口与RTT的关系
- 一个RTT内可以发送的数据量 = 发送窗口大小
- 吞吐量 ≈ 发送窗口 / RTT
- 窗口越大、RTT越小，吞吐量越高
- 带宽时延积（BDP）= 带宽 × RTT = 理想窗口大小

### 带宽时延积（BDP）
BDP = 带宽 × RTT
- 高速长距离网络BDP很大
- 窗口必须 >= BDP才能充分利用带宽
- 例：1Gbps带宽，100ms RTT → BDP = 100Mb × 0.1s = 10Mb = 1.25MB
- 需要约1.25MB的窗口才能跑满带宽

### 408常考要点
- 发送窗口 = min(rwnd, cwnd)
- 最大吞吐量 = 发送窗口 / RTT
- 可用窗口 = 发送窗口 - 已发送未确认
- BDP = 带宽 × RTT，决定理想窗口大小
- TCP首部窗口字段16位，需要窗口扩大选项支持大窗口

## 直观理解

- **窗口计算像预算管理**：总额度=min(收入,贷款额度)，已花的扣掉，剩的还能花
- **BDP像水管容量**：水管越粗（带宽越大）、越长（RTT越大），需要的水池（窗口）越大

## 协议关联

- **与拥塞控制**：cwnd随网络状况变化
- **与流量控制**：rwnd随接收方缓冲区变化
- **408常见考法**：给定参数计算窗口大小和可发送量
- **BDP是网络性能优化的重要概念**
