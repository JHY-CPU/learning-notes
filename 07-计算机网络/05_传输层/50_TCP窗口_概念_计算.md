# 51_TCP窗口_概念_计算

## 核心概念

- **发送窗口 = min(rwnd, cwnd)**
- **可用窗口 = 发送窗口 - 已发送未确认的数据量**
- **接收窗口 = 接收方缓冲区大小 - 已接收未读取的数据量**

### 计算题型总结

**题型1：计算实际发送窗口**
$$\text{发送窗口} = \min(\text{rwnd}, \text{cwnd})$$

**题型2：计算最大吞吐量**
$$\text{最大吞吐量} = \frac{\text{发送窗口}}{\text{RTT}}$$

**题型3：计算窗口内可发送的报文段数**
$$\text{报文段数} = \frac{\text{可用窗口}}{\text{MSS}}$$

## 原理分析

### 计算示例1

**题目**：rwnd=8KB，cwnd=4KB，MSS=1KB，RTT=20ms。求发送窗口、最大报文段数、最大吞吐量。

**解析**：
- 发送窗口 = min(8KB, 4KB) = **4KB**
- 最大报文段数 = 4KB / 1KB = **4个**
- 最大吞吐量 = 4KB / 20ms = 4096B / 0.02s ≈ **204.8KB/s**

### 计算示例2

**题目**：发送窗口=5000字节，已发送了序号1~3000的数据，收到了ACK=2001。此时还能发送多少字节新数据？

**解析**：
- 窗口覆盖：2001~7000（5000字节，右移2000字节）
- 已发送未确认：2001~3000（1000字节）
- 可发送新数据：3001~7000，共**4000字节**

### 计算示例3：带rwnd变化的计算

**题目**：初始rwnd=4000，cwnd=4000，发送了4000字节。收到ACK=2001，但rwnd变为3000。此时还能发多少？

**解析**：
- ACK=2001确认了1~2000
- 新的发送窗口 = min(rwnd=3000, cwnd假设不变) = 3000
- 窗口覆盖：2001~5000
- 已发未确认：2001~4000
- 可发送新数据：4001~5000，共**1000字节**

## 直观理解

- **窗口计算像预算管理**：总额度=min(收入,贷款额度)，已花的扣掉，剩的还能花

## 协议关联

- **与拥塞控制**：cwnd随网络状况变化
- **与流量控制**：rwnd随接收方缓冲区变化
- **408常见考法**：给定参数计算窗口大小和可发送量
