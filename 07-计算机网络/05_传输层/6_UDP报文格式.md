# 07_UDP报文格式

## 核心概念

- **UDP首部长度**：固定**8字节**（比TCP的20字节首部简单得多）
- **UDP报文 = UDP首部 + 数据部分**
- **UDP长度字段**：包含首部+数据的总长度，最小值为8（仅有首部无数据）
- **UDP校验和**：可选字段（IPv4中可选，IPv6中强制），用于差错检测

### UDP首部格式（共8字节 = 4个字段，每个16位 = 2字节）

```
 0      7 8     15 16    23 24    31
+--------+--------+--------+--------+
|  源端口号  |  目的端口号 |
+--------+--------+--------+--------+
|   长度    |   校验和    |
+--------+--------+--------+--------+
```

| 字段 | 长度 | 说明 |
|------|------|------|
| 源端口号 | 16位（2字节） | 发送方端口号，可选（不需要时填0） |
| 目的端口号 | 16位（2字节） | 接收方端口号，必填 |
| 长度 | 16位（2字节） | UDP数据报总长度（首部+数据），最小为8 |
| 校验和 | 16位（2字节） | 差错检测码（可选，填0表示不计算） |

## 原理分析

### 各字段详细说明

**源端口号（Source Port）**：
- 可选字段，如果发送方不需要接收方回复，可以填0
- 在需要对方回复时必须填写

**目的端口号（Destination Port）**：
- 必填字段，接收方据此进行分用
- 如果目的端口号对应的进程不存在，ICMP发送"端口不可达"差错报文

**长度（Length）**：
- 16位，最大值为65535字节
- 实际数据最大长度 = 65535 - 8（首部） = 65527字节
- 如果需要发送超过此长度的数据，需要在应用层分片

**校验和（Checksum）**：
- 计算时需要构造**伪首部**（包含IP层信息）
- IPv4中可选，IPv6中强制

### UDP报文封装

```
+-------------------+
|   IP首部(20字节)   |
+-------------------+
|   UDP首部(8字节)   |
+-------------------+
|     UDP数据        |
+-------------------+
```

## 直观理解

- **快递单类比**：UDP首部就像一张简单的快递单——发件人（源端口）、收件人（目的端口）、包裹大小（长度）、防伪标识（校验和）。一共就4个信息，非常精简

## 协议关联

- **与IP的关系**：UDP报文封装在IP数据报中传输。IP首部的协议字段值为**17**表示封装的是UDP
- **与TCP的对比**：TCP首部最少20字节（无选项），UDP固定8字节。UDP首部只有4个字段，TCP有很多字段
- **408常见考法**：
  - UDP首部长度是**8字节**（常考）
  - 长度字段的含义（包括首部）
  - UDP长度字段是冗余的（IP首部已有总长度），但为了跨层独立性保留
