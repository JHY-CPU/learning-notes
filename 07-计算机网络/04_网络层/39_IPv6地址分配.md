# IPv6地址分配

## 核心概念
- IPv6地址分配方式：SLAAC（无状态地址自动配置）、DHCPv6（有状态地址配置）、手动配置

### SLAAC（Stateless Address Autoconfiguration）
- **无状态**：不需要服务器维护地址分配状态
- **工作过程**：
  1. 主机生成链路本地地址（FE80:: + EUI-64接口标识符）
  2. 发送路由器请求（RS，ICMPv6类型133）
  3. 路由器回复路由器通告（RA，ICMPv6类型134），包含网络前缀
  4. 主机将网络前缀 + 接口标识符组合成全球单播地址
- **优点**：无需配置服务器，即插即用
- **缺点**：无法获得DNS等额外信息（需配合RDNSS）

### DHCPv6
- **有状态**：服务器维护地址分配状态
- **工作过程**：
  1. 客户端发送Solicit消息
  2. 服务器回复Advertise消息
  3. 客户端发送Request消息
  4. 服务器回复Reply消息
- **优点**：集中管理，可以分配DNS等信息
- **缺点**：需要部署DHCPv6服务器

### 两者对比
| 特性 | SLAAC | DHCPv6 |
|------|-------|--------|
| 状态 | 无状态 | 有状态 |
| 服务器 | 不需要 | 需要 |
| 地址生成 | 主机自行生成 | 服务器分配 |
| DNS信息 | 需要RDNSS | DHCPv6分配 |
| 管理性 | 低 | 高 |

## 原理分析
- RA消息中的M位和O位控制地址配置方式
  - M=0, O=0：仅SLAAC
  - M=0, O=1：SLAAC + DHCPv6获取其他信息
  - M=1：使用DHCPv6获取地址

### EUI-64接口标识符生成
由48位MAC地址生成64位接口标识符：
1. 在MAC地址第3和第4字节之间插入FFFE
2. 翻转第1个字节的第7位（Universal/Local位）

**示例**：
- MAC：00-1A-2B-3C-4D-5E
- 插入FFFE：00-1A-2B-FF-FE-3C-4D-5E
- 翻转第7位：02-1A-2B-FF-FE-3C-4D-5E
- 接口标识符：021A:2BFF:FE3C:4D5E

### DHCPv6 vs DHCPv4对比
| 特性 | DHCPv4 | DHCPv6 |
|------|--------|--------|
| 传输层 | UDP 67/68 | UDP 546/547 |
| 报文格式 | BOOTP扩展 | 全新格式 |
| 地址分配 | 有状态 | 有状态/无状态 |
| 前缀委派 | 不支持 | 支持（PD） |
| 配合SLAAC | 不适用 | 可配合 |

### RA消息关键字段
| 字段 | 含义 |
|------|------|
| M位（Managed） | 1=使用DHCPv6获取地址 |
| O位（Other） | 1=使用DHCPv6获取其他配置 |
| Prefix | 网络前缀信息 |
| Valid Lifetime | 前缀有效时间 |
| Preferred Lifetime | 首选时间 |

### 隐私扩展问题
EUI-64接口标识符包含MAC地址，存在隐私泄露风险。
**解决方案**：
- RFC 4941定义隐私扩展：使用随机生成的临时地址
- 临时地址有生存期，定期更换
- Windows默认启用隐私扩展

### 408常考要点
- SLAAC使用RS/RA（ICMPv6 NDP），不需要服务器
- DHCPv6有状态vs无状态（M位和O位控制）
- EUI-64生成过程：插入FFFE + 翻转U/L位
- IPv6可以同时使用SLAAC和DHCPv6
- DHCPv6使用UDP端口546（客户端）和547（服务器）

## 直观理解
- SLAAC像"自选工位"——看到大厅指示牌（RA前缀），自己找位置坐
- DHCPv6像"前台分配工位"——向服务器申请，服务器指定位置
- EUI-64像"身份证号"——从MAC地址推导出来

## 协议关联
- SLAAC依赖ICMPv6的RS/RA消息
- DHCPv6类似DHCP但使用不同的报文格式
- EUI-64由MAC地址自动生成接口标识符
- 隐私扩展：使用随机接口标识符替代EUI-64
- NDP（邻居发现协议）是IPv6地址分配的基础
