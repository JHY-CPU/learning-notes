# ARP综合练习

## 核心概念
- ARP（Address Resolution Protocol）：地址解析协议，将IP地址解析为MAC地址
- ARP工作在数据链路层和网络层之间
- ARP使用广播请求和单播响应

### ARP工作流程
1. 发送方检查ARP缓存表是否有目的IP对应的MAC地址
2. 如果没有，发送ARP请求（广播，目的MAC = FF:FF:FF:FF:FF:FF）
3. 同一子网的所有主机收到ARP请求
4. 目的主机回复ARP响应（单播，包含自己的MAC地址）
5. 发送方更新ARP缓存表

### ARP报文格式
| 字段 | 长度 | 说明 |
|------|------|------|
| 硬件类型 | 2字节 | 1=以太网 |
| 协议类型 | 2字节 | 0x0800=IPv4 |
| 硬件地址长度 | 1字节 | 6（以太网MAC） |
| 协议地址长度 | 1字节 | 4（IPv4） |
| 操作码 | 2字节 | 1=请求，2=响应 |
| 发送方MAC | 6字节 | 发送方硬件地址 |
| 发送方IP | 4字节 | 发送方协议地址 |
| 目的MAC | 6字节 | 目的硬件地址（请求时全0） |
| 目的IP | 4字节 | 目的协议地址 |

## 原理分析

### 跨子网ARP
- 主机A（子网1）要发送数据给主机B（子网2）
- A的ARP请求的是网关路由器的MAC地址（不是B的MAC）
- 路由器收到后，用自己的MAC替换源MAC，查找B的MAC并转发

### ARP缓存表
- 条目有超时时间（通常2-20分钟）
- 包含动态条目（ARP学习）和静态条目（手动配置）

### 代理ARP（Proxy ARP）
当路由器收到ARP请求但目标不在本地子网时，路由器可以代替目标回复ARP响应。

**场景**：
- 主机A（192.168.1.0/24）要访问主机B（192.168.2.0/24）
- A不知道B在不同子网，广播ARP请求B的MAC
- 路由器配置了代理ARP，用自己的MAC回复A
- A将数据发给路由器，路由器转发给B

### 免费ARP（Gratuitous ARP）
主机主动发送ARP请求（目的IP=自己的IP），用于：
1. 检测IP地址冲突
2. 更新其他主机的ARP缓存
3. 主机IP改变后通知网络

### ARP欺骗攻击
攻击者向网络中发送伪造的ARP响应，将自己的MAC与网关IP绑定。
**后果**：流量被重定向到攻击者（中间人攻击）。
**防御**：静态ARP绑定、ARP检测（DAI）、802.1X认证。

### ARP报文封装
ARP报文直接封装在以太网帧中：
```
| 以太网首部 | ARP报文 |
| 目的MAC(6B) | 硬件类型(2B) |
| 源MAC(6B) | 协议类型(2B) |
| 类型=0x0806(2B) | 硬件地址长度(1B) |
| ... | 协议地址长度(1B) |
| FCS(4B) | 操作码(2B) |
| ... | 发送方MAC(6B) |
| ... | 发送方IP(4B) |
| ... | 目的MAC(6B) |
| ... | 目的IP(4B) |
```

### 408常考要点
- ARP请求是广播，ARP响应是单播
- ARP只在同一广播域（数据链路）内有效
- 跨子网ARP解析的是网关的MAC，不是目的主机的MAC
- ARP工作在数据链路层和网络层之间（协议栈边界）
- ARP没有端口号概念（不是TCP/UDP协议）

## 直观理解
- ARP像"查电话号码"——你知道名字（IP地址），通过广播喊"谁是X？"，X回答"我是，这是我的号码（MAC地址）"
- 跨子网ARP像"找前台"——你不知道目标在哪，先找网关（路由器），由网关转发
- 代理ARP像"前台代接电话"——路由器代替目标回答

## 协议关联
- ARP依赖广播，只在同一条数据链路上有效
- ARP欺骗（ARP Spoofing）：攻击者发送伪造的ARP响应
- IPv6使用NDP的NS/NA消息替代ARP
- 免费ARP（Gratuitous ARP）：主机广播自己的IP-MAC映射，用于检测IP冲突
- ARP与IP紧密配合：IP确定下一跳，ARP解析下一跳的MAC
