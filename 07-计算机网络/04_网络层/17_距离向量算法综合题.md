# 距离向量算法综合题

## 核心概念
- 距离向量算法（Distance Vector）：每个路由器只知道到各目的网络的距离和方向
- Bellman-Ford方程：$D(x) = \min_v\{c(v,x) + D_v(x)\}$
- RIP是典型的距离向量协议

### 核心公式
路由器A到目的网络N的距离：
$$D_A(N) = \min_{邻居v}\{c(A,v) + D_v(N)\}$$

其中：
- $c(A,v)$ = A到邻居v的链路代价
- $D_v(N)$ = 邻居v到N的距离（从邻居的路由表获得）

### 更新规则
1. 收到邻居v的路由表
2. 对于邻居v通告的每个目的网络N
3. 计算新距离 = 到v的代价 + v到N的距离
4. 如果新距离 < 当前距离，更新路由表

## 原理分析

### 综合练习
网络拓扑：A—1—B—2—C，A与C不直连

初始路由表（A）：
| 目的 | 距离 | 下一跳 |
|------|------|--------|
| B | 1 | 直连 |
| C | ∞ | - |

收到B的路由表：B到C距离2

A更新：D_A(C) = 1 + 2 = 3，下一跳B

### 无穷计数问题
当链路故障时，可能出现距离逐渐增加到无穷的情况：
- 初始：A到C距离3
- B到C的链路断开，B将C标记为不可达（距离∞）
- 但A可能先收到B旧的路由表（距离2）
- A认为到C还是3，B从A学到"到C距离3"，形成环路

**解决方案**：
- 水平分割：不把从某接口学到的路由再发回该接口
- 毒性逆转：把从某接口学到的路由以无穷大（16）发回
- 触发更新：网络变化时立即发送更新

## 直观理解
- 距离向量像"问路游戏"——每个人只告诉邻居"到X还有多远"，不知道完整路径
- 无穷计数像"以讹传讹"——坏消息传播慢，好消息传播快

## 协议关联
- RIP、IGRP、EIGRP使用距离向量算法
- 水平分割和毒性逆转是RIP的核心优化
- 链路状态算法（OSPF）是距离向量的替代方案
