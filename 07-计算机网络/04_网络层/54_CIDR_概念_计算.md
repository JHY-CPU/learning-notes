# 18_CIDR_概念_计算

## 核心概念

- **CIDR计算核心**：
  - 地址块大小 = $2^{(32-n)}$
  - 网络地址 = 前n位固定，后(32-n)位全0
  - 广播地址 = 前n位固定，后(32-n)位全1
  - 路由聚合：找共同前缀

- **408计算类型**：
  1. 给CIDR地址块求范围和大小
  2. 给多个地址块求聚合结果
  3. 判断某IP是否属于某CIDR块

## 原理分析

### 例1：CIDR地址块范围

**题**：192.168.48.0/20 的地址范围？

**解**：
- /20：前20位固定，后12位可变
- 第三字节48 = 0011**0000**，前4位固定（0011）
- 后4位 + 第四字节 = 可变的12位
- 最小：第三字节 00110000 = 48 → 192.168.48.0
- 最大：第三字节 0011**1111** = 63 → 192.168.63.255
- 范围：192.168.48.0 ~ 192.168.63.255
- 地址数：$2^{12} = 4096$

### 例2：路由聚合

**题**：聚合以下地址块：
1. 172.16.128.0/20
2. 172.16.144.0/20
3. 172.16.160.0/20
4. 172.16.176.0/20

**解**：
- 128 = 1000**0000**, 144 = 1001**0000**, 160 = 1010**0000**, 176 = 1011**0000**
- 前3个比特：128和176不同（128=10000000, 176=10110000）
- 第一个不同位在第三字节的第5位（从1数）
- 128 = 10**0**00000, 176 = 10**1**10000
- 共同前缀：172.16.10xxxxxx... 前 16+6 = 22位相同
- 聚合结果：**172.16.128.0/18**

验证：
- 128 = 10000000, 176 = 10110000
- 前2位都是10 ✓
- /18 包含 $2^{14} = 16384$ 地址
- 4个/20共 $4 \times 2^{12} = 16384$ ✓

### 例3：判断IP归属

**题**：200.10.23.5 是否属于 200.10.16.0/20？

**解**：
- /20：看前20位
- 23 = 00010**111**
- 16 = 00010**000**
- 前20位：网络前缀是200.10.00010xxx
- 23的前4位是00010，前20位匹配 ✓
- **属于**该地址块

### 例4：地址分配

**题**：ISP有地址块 200.10.0.0/16，需分配给4个部门：A需1000个、B需500个、C需200个、D需100个地址。如何分配？

**解**：
- A: $2^{10} - 2 = 1022 \geq 1000$，需要/22
- B: $2^9 - 2 = 510 \geq 500$，需要/23
- C: $2^8 - 2 = 254 \geq 200$，需要/24
- D: $2^7 - 2 = 126 \geq 100$，需要/25

分配：
- A: 200.10.0.0/22（~200.10.3.255）
- B: 200.10.4.0/23（~200.10.5.255）
- C: 200.10.6.0/24（~200.10.6.255）
- D: 200.10.7.0/25（~200.10.7.127）

## 直观理解

**聚合口诀**：写二进制，找共同前缀，长度就是共同前缀的位数
**分配口诀**：从大到小分配，每个部门取所需最小2的幂

## 协议关联

- CIDR地址分配直接影响路由表大小
- 路由聚合减少了BGP路由表的膨胀
- 考试中CIDR计算常与最长前缀匹配联合出题
