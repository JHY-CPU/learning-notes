# IPv6数据报格式

## 核心概念
- IPv6基本首部固定40字节，取消了IPv4中的可选字段（使用扩展首部代替）
- IPv6数据报 = 基本首部 + 扩展首部链（可选）+ 数据部分

### IPv6基本首部字段（40字节）
| 字段 | 长度 | 说明 |
|------|------|------|
| 版本 | 4位 | 固定为6 |
| 流量类别 | 8位 | 类似IPv4的DS字段，用于QoS |
| 流标签 | 20位 | 标识同一个流的数据报（IPv6新增） |
| 有效载荷长度 | 16位 | 扩展首部+数据的总长度 |
| 下一个首部 | 8位 | 指明扩展首部类型或上层协议号 |
| 跳数限制 | 8位 | 类似IPv4的TTL，每经过一个路由器减1 |
| 源地址 | 128位 | 发送方IPv6地址 |
| 目的地址 | 128位 | 接收方IPv6地址 |

## 原理分析

### IPv6与IPv4首部对比（高频考点）
| 特性 | IPv4 | IPv6 |
|------|------|------|
| 首部长度 | 可变20-60字节 | 固定40字节 |
| 首部校验和 | 有 | 无 |
| 分片字段 | 有（标识/标志/偏移） | 无（仅源端分片） |
| 可选字段 | 首部中直接包含 | 扩展首部链 |
| TTL | TTL字段 | 跳数限制字段 |
| 首部长度字段 | IHL字段 | 无（固定长度无需） |

### IPv6取消校验和的原因
- 数据链路层以太网帧已有CRC校验
- 传输层TCP/UDP有校验和
- 取消后路由器每跳少做一次计算，提高转发效率

### 扩展首部类型
- 0=逐跳选项、43=路由、44=分片、59=无下一个首部、60=目的站选项

### IPv6取消了哪些IPv4字段
| IPv4字段 | IPv6处理 | 原因 |
|----------|----------|------|
| 首部长度(IHL) | 取消 | 固定40字节不需要 |
| 标识/标志/片偏移 | 取消 | 仅源端分片 |
| 首部校验和 | 取消 | 链路层和传输层已有 |
| 选项 | 移到扩展首部 | 简化路由器处理 |
| 协议 | 改为"下一个首部" | 支持扩展首部链 |

### IPv6新增字段
| 字段 | 作用 |
|------|------|
| 流量类别 | 替代IPv4的TOS/DS，用于QoS |
| 流标签 | 标识属于同一"流"的数据报 |
| 有效载荷长度 | 包含扩展首部+数据的总长度 |
| 下一个首部 | 链式指针，指向扩展首部或上层协议 |

### 扩展首部的处理
路由器逐跳处理扩展首部：
- 逐跳选项：每个路由器都处理
- 路由选项：指定路由器处理
- 分片选项：目的主机重组时处理
- 目的站选项：仅目的主机处理

### "下一个首部"字段值
| 值 | 含义 |
|------|------|
| 0 | 逐跳选项 |
| 6 | TCP |
| 17 | UDP |
| 43 | 路由选项 |
| 44 | 分片选项 |
| 58 | ICMPv6 |
| 59 | 无下一个首部 |
| 60 | 目的站选项 |

### 408常考要点
- IPv6基本首部固定40字节（IPv4为20-60字节）
- 取消校验和：路由器少做一次计算，提高效率
- 取消路由器分片：由源端通过路径MTU发现完成
- "下一个首部"形成链式结构：基本首部→扩展首部→TCP/UDP
- 流标签是IPv6新增的20位字段
- 有效载荷长度包含扩展首部（不包含基本首部40字节）

## 直观理解
- IPv4首部像"带附件的信封"，IPv6首部像"标准信封+附加卡片"
- 取消校验和像快递公司不再在每个中转站验货，由发货方和收货方负责
- 流标签像"快递单上的VIP标签"——同类包裹统一处理
- 扩展首部链像"附加说明卡"——需要时附上，不需要时省略

## 协议关联
- "下一个首部"字段形成链式结构：基本首部→扩展首部→TCP/UDP
- 与ICMPv6配合：下一个首部=58标识ICMPv6
- 与传输层伪首部计算：IPv6伪首部中地址长度为128位
- IPv6的路径MTU发现依赖ICMPv6"数据报过大"消息
