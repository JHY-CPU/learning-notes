# RIP综合练习

## 核心概念
- RIP（Routing Information Protocol）：基于距离向量的内部网关协议
- 使用跳数（Hop Count）作为度量值，最大跳数为15（16表示不可达）
- 每30秒与邻居交换完整路由表
- 使用UDP端口520进行通信

### RIP关键参数
| 参数 | 值 | 说明 |
|------|-----|------|
| 度量值 | 跳数 | 最大15跳 |
| 更新周期 | 30秒 | 定时发送路由表 |
| 超时定时器 | 180秒 | 路由条目失效 |
| 水平分割 | 启用 | 不从接收接口发回路由 |
| 毒性逆转 | 可选 | 将从某接口学到的路由以跳数16发回 |
| 触发更新 | 启用 | 网络变化时立即发送 |

### RIP工作流程
1. 路由器启动，初始化直连路由（距离=0）
2. 每30秒向所有邻居发送路由表
3. 收到邻居的路由表后，使用Bellman-Ford算法更新
4. 更新公式：$D(x) = \min\{c(v,x) + D_v(x)\}$

## 原理分析

### 练习：距离向量更新
路由器A的初始路由表：
| 目的 | 跳数 |
|------|------|
| 网络1 | 0（直连）|
| 网络2 | 0（直连）|

收到邻居B的路由表：网络3距离2，网络4距离1

A更新后的路由表：
| 目的 | 跳数 | 下一跳 |
|------|------|--------|
| 网络1 | 0 | 直连 |
| 网络2 | 0 | 直连 |
| 网络3 | 3 | B |
| 网络4 | 2 | B |

### RIP水平分割详解
水平分割（Split Horizon）：路由器不会将从某个接口学到的路由信息再通过该接口发回去。

**示例**：
- A通过接口eth0连接到B
- A从B学到"到网络X距离为3"
- A向其他邻居通告时可以包含网络X
- 但A**不会**通过eth0再向B通告网络X（因为这条路由本来就来自B）

### RIP毒性逆转详解
毒性逆转（Poison Reverse）：将从某接口学到的路由以距离16（不可达）发回该接口。

**示例**：
- A从B学到"到网络X距离为3"
- A通过eth0向B通告"到网络X距离为16"
- 比水平分割更积极，直接告诉B"我这边到X不通"

### RIP版本对比
| 特性 | RIPv1 | RIPv2 |
|------|-------|-------|
| 子网掩码 | 不支持（有类） | 支持（无类） |
| 认证 | 不支持 | 支持MD5 |
| 广播/组播 | 广播224.0.0.9 | 组播224.0.0.9 |
| 路由标记 | 无 | 支持 |
| 下一跳 | 无 | 支持指定 |

### 408常考要点
- RIP使用UDP封装（端口520），而OSPF直接用IP封装（协议号89）
- RIP最大跳数为15，16表示不可达（这是硬性限制）
- RIP属于应用层协议但提供网络层功能
- 收敛时间与网络规模成正比，大型网络收敛极慢

## 直观理解
- RIP像"问路"——每个路口问下一个人"到目的地还有多远"，选择总距离最近的
- 最大15跳像"超过15个路口就不问了"——认为太远不可达
- 水平分割像"不把别人告诉你的消息再告诉他本人"

## 协议关联
- RIP只适用于小型网络（最大15跳限制）
- RIP存在无穷计数问题，通过水平分割和毒性逆转缓解
- RIPng是IPv6版本的RIP
- RIP属于距离向量协议，与链路状态协议（OSPF）对比考查
