# 43_路由表结构

## 核心概念

- **路由表（Routing Table）**：路由器转发分组的依据
  - 每个路由器维护一张路由表
  - 路由表由路由协议和管理员配置生成
  - 转发分组时查路由表确定下一跳

- **路由表的核心字段**：

| 字段 | 含义 |
|------|------|
| 目的网络（Destination） | 目的网络地址（CIDR格式） |
| 子网掩码（Mask） | 对应的子网掩码 |
| 下一跳（Next Hop） | 下一跳路由器的IP地址 |
| 出接口（Interface） | 转发的本地接口 |
| 代价（Cost/Metric） | 到目的网络的度量值 |
| 来源（Source） | 条目来源（直连/静态/RIP/OSPF等） |

- **路由表条目来源**：
  1. **直连路由**：路由器接口直接连接的网络（C）
  2. **静态路由**：管理员手动配置（S）
  3. **RIP学习**：通过RIP协议学习（R）
  4. **OSPF学习**：通过OSPF协议学习（O）
  5. **BGP学习**：通过BGP协议学习（B）
  6. **默认路由**：0.0.0.0/0（S*）

## 原理分析

### 路由表查表过程

```
收到目的地址为D的分组：

for 每个路由表条目 (Network, Mask, NextHop, Interface):
    if (D AND Mask) == Network:
        标记为匹配条目

从所有匹配条目中选择前缀最长的
→ 转发到对应下一跳或出接口

若无匹配：
    若有默认路由(0.0.0.0/0) → 按默认路由转发
    否则 → 丢弃，返回ICMP目的不可达
```

### 路由表的更新

```
直连路由：
  接口UP → 自动添加直连路由
  接口DOWN → 自动删除

静态路由：
  管理员配置后永久存在
  若下一跳不可达，条目无效

动态路由：
  RIP: 每30秒更新，180秒超时删除
  OSPF: 链路状态变化时触发更新
  BGP: 增量更新
```

### 路由表的优先级

当多个路由协议学习到同一目的网络的路由时：
- 按**管理距离（Administrative Distance）**选择
- 管理距离越小越优先

| 来源 | 管理距离 |
|------|---------|
| 直连 | 0 |
| 静态 | 1 |
| OSPF | 110 |
| RIP | 120 |
| BGP | 20（eBGP）/200（iBGP） |

## 直观理解

**类比**：路由表像邮局的分拣表
- 目的网络 = 邮编
- 下一跳 = 下一个中转站
- 出接口 = 从哪个门送出去
- 代价 = 运输成本

**路由表查表口诀**：
- "AND一下找匹配，最长前缀选最优"
- "没有匹配走默认，默认没有就丢弃"

## 协议关联

### 路由表与转发

- 路由表是控制平面（Control Plane）的产物
- 转发是数据平面（Data Plane）的操作
- 路由协议更新路由表（控制平面）
- 分组到达时查路由表转发（数据平面）

### 路由表与ARP
- 查路由表确定下一跳IP
- 用ARP获取下一跳的MAC地址
- 用MAC地址封装以太网帧

### 考试要点
- 直连路由管理距离最小（0），最优先
- 路由表中可能有多个匹配条目，选最长前缀
- 默认路由是最"兜底"的选择
- 路由表条目有生存时间（TTL），超时删除
