# OSPF综合练习

## 核心概念
- OSPF（Open Shortest Path First）：基于链路状态的内部网关协议
- 使用Dijkstra算法计算最短路径树
- 度量值：链路代价（Cost = 参考带宽/接口带宽）
- 使用IP协议号89直接封装（不用TCP/UDP）

### OSPF核心机制
- **邻居发现**：Hello协议发现邻居，建立邻居关系
- **链路状态通告（LSA）**：描述路由器的链路状态
- **链路状态数据库（LSDB）**：存储所有LSA
- **SPF计算**：Dijkstra算法以自己为根计算最短路径树

### OSPF五种报文
| 类型 | 名称 | 功能 |
|------|------|------|
| 1 | Hello | 发现和维护邻居 |
| 2 | DBD | 数据库描述摘要 |
| 3 | LSR | 链路状态请求 |
| 4 | LSU | 链路状态更新 |
| 5 | LSAck | 链路状态确认 |

### OSPF区域
- Area 0 = 骨干区域（Backbone Area）
- 所有非骨干区域必须与Area 0直接相连
- 区域间路由通过ABR（区域边界路由器）传递

## 原理分析

### Dijkstra算法
1. 初始化：到自己的距离为0，其他节点为无穷大
2. 从未访问节点中选择距离最小的节点
3. 更新该节点邻居的距离
4. 重复直到所有节点被访问

### 链路代价计算
Cost = 参考带宽 / 接口带宽（Mbps）
- 参考带宽通常为100Mbps
- 100Mbps接口：Cost = 1
- 10Mbps接口：Cost = 10
- 1Gbps接口：Cost = 0.1 → 取1

### OSPF邻居状态机
1. **Down**：初始状态，未收到Hello
2. **Init**：收到Hello，但对方Hello中没有自己的Router ID
3. **2-Way**：双方Hello中都有对方的Router ID，建立邻居关系
4. **ExStart**：确定主从关系，开始交换DBD
5. **Exchange**：交换DBD（数据库描述）
6. **Loading**：发送LSR请求缺少的LSA
7. **Full**：LSDB同步完成

### OSPF路由器类型
| 类型 | 说明 |
|------|------|
| IR | 内部路由器（所有接口在同一区域） |
| ABR | 区域边界路由器（连接多个区域） |
| ASBR | 自治系统边界路由器（连接外部AS） |
| Backbone Router | 至少一个接口在Area 0 |

### OSPF vs RIP对比
| 特性 | OSPF | RIP |
|------|------|-----|
| 算法 | 链路状态（Dijkstra） | 距离向量（Bellman-Ford） |
| 度量值 | 链路代价（带宽） | 跳数 |
| 收敛速度 | 快（触发更新+泛洪） | 慢（30秒定时） |
| 跳数限制 | 无 | 15 |
| 封装 | IP（协议号89） | UDP（端口520） |
| 分层设计 | 支持（多区域） | 不支持 |

### 408常考要点
- OSPF使用IP直接封装，协议号89（不使用TCP/UDP）
- OSPF的Hello报文用于发现邻居和检测邻居存活
- DR/BDR选举减少OSPF邻居关系数量（在广播网络中）
- OSPF支持认证（明文和MD5），RIPv1不支持
- OSPF的LSA泛洪保证所有路由器的LSDB一致

## 直观理解
- OSPF像"全城地图"——每个路口广播自己的路况，大家拼出完整地图后各自计算最短路线
- Dijkstra像"GPS导航"——从起点开始逐步扩展，找到到各点的最短路
- DR/BDR像"微信群主"——所有人只发给群主，群主再转发给大家

## 协议关联
- OSPF直接运行在IP之上（协议号89），不使用TCP/UDP
- OSPF收敛速度比RIP快得多
- OSPF适用于大型网络，支持分层设计（多区域）
- OSPF是链路状态协议的典型代表，与RIP（距离向量）对比考查
