# IPv6概述

## 核心概念
- IPv6（Internet Protocol version 6）是IPv4的下一代互联网协议，由IETF设计
- 核心动机：解决IPv4地址耗尽问题（IPv4仅约43亿个地址）
- IPv6地址长度：128位，地址空间约 $3.4 \times 10^{38}$ 个
- RFC 2460定义了IPv6基本规范

### IPv6的设计目标
- 更大的地址空间：128位 vs 32位
- 简化的头部格式：固定40字节基本头部
- 更好的扩展性：扩展头部机制
- 支持即插即用：无状态地址自动配置（SLAAC）
- 增强的安全性：IPsec为必选功能
- 支持移动性：移动IPv6

### 408考试重点
- IPv6地址表示方法和缩写规则
- IPv6数据报格式与IPv4的对比
- IPv6过渡技术（双栈、隧道、翻译）
- ICMPv6的功能

## 原理分析
- IPv6取消了IPv4中的首部校验和（由数据链路层和传输层负责）
- 取消了分片相关字段（中间路由器不再分片，由源端使用路径MTU发现）
- 流标签（Flow Label）字段：标识属于同一"流"的数据报，便于路由器做相同的转发处理

### IPv6相比IPv4的改进
| 改进项 | IPv4问题 | IPv6改进 |
|--------|----------|----------|
| 地址空间 | 32位，已耗尽 | 128位，充裕 |
| 首部复杂度 | 可变长，选项处理慢 | 固定40字节，路由器处理快 |
| 分片 | 路由器分片，消耗资源 | 仅源端分片，路由器不做 |
| 校验和 | 每跳计算，降低效率 | 取消，由链路层和传输层负责 |
| 地址配置 | 需要DHCP服务器 | SLAAC即插即用 |
| 安全性 | IPsec可选 | IPsec必选 |
| 广播 | 支持，效率低 | 取消，用多播替代 |

### IPv6地址空间的实际意义
- 每平方米地球表面可分配约6.5×10^23个地址
- 足够为地球上每粒沙子分配多个IP地址
- 支持物联网（IoT）海量设备接入

### IPv6的扩展首部
IPv6使用扩展首部链替代IPv4的选项字段：
1. 逐跳选项
2. 路由选项
3. 分片选项
4. 目的站选项
5. 认证首部（AH）
6. 封装安全载荷（ESP）

每个扩展首部通过"下一个首部"字段链接，形成链式结构。

### 408常考要点
- IPv6地址长度128位（IPv4为32位）
- IPv6基本首部固定40字节
- IPv6取消了校验和（路由器处理更快）
- IPv6取消了路由器分片功能（源端负责）
- IPv6取消了广播，用多播替代
- ICMPv6整合了ICMPv4+ARP+IGMP功能
- 过渡技术：双栈、隧道、NAT64

## 直观理解
- IPv4地址像手机号（11位，总量有限），IPv6地址像宇宙中每个原子都有独立编号
- IPv4到IPv6的过渡像"换车牌"——不能一夜之间全部更换，需要过渡期
- IPv6简化首部像"标准化包裹"——统一规格，处理更快

## 协议关联
- IPv6与ICMPv6紧密耦合（替代了ICMPv4 + ARP + IGMP的功能）
- 应用层DNS需扩展（AAAA记录映射IPv6地址，A记录映射IPv4地址）
- TCP/UDP协议基本不变（仅需修改伪首部校验和计算中的地址长度）
- IPv6的SLAAC依赖NDP（RS/RA消息）
