# IP地址与域名

## 📡 IP 地址与域名

IPv4/IPv6、域名结构、子网掩码、NAT、公网与私有 IP。

## IP 地址

IP 地址是网络中每个设备的唯一标识符，类似于现实世界中的门牌号。
```
// ========== IPv4 地址 ==========
// 格式: 32 位, 4 组 0-255 的数字
// 示例: 192.168.1.1
//
// IPv4 地址 ≈ 43 亿个
// 全球 IPv4 已于 2019 年耗尽
//
// ========== IPv6 地址 ==========
// 格式: 128 位, 8 组 16 进制数
// 示例: 2001:0db8:85a3:0000:0000:8a2e:0370:7334
//
// IPv6 地址 ≈ 3.4×10³⁸ 个
// 地球每粒沙子都能分到 100 个以上

// ========== IP 地址分类 ==========
// A 类: 1.0.0.0 - 126.255.255.255 (大型网络)
// B 类: 128.0.0.0 - 191.255.255.255 (中型网络)
// C 类: 192.0.0.0 - 223.255.255.255 (小型网络)
// D 类: 224.0.0.0 - 239.255.255.255 (组播)
// E 类: 240.0.0.0 - 255.255.255.255 (保留)
```
## 公网 IP 与私有 IP
```
// ========== 私有 IP 范围 (RFC 1918) ==========
// A 类: 10.0.0.0 - 10.255.255.255
// B 类: 172.16.0.0 - 172.31.255.255
// C 类: 192.168.0.0 - 192.168.255.255
//
// 私有 IP 特点:
//   - 只能在局域网内使用
//   - 不同局域网可重复使用
//   - 不能直接在互联网上路由
//
// 公网 IP 特点:
//   - 全球唯一
//   - 可在互联网上路由
//   - 通常由 ISP 分配

// ========== NAT (网络地址转换) ==========
// 路由器将私有 IP 转换为公网 IP
// 多个设备共享一个公网 IP
// 通过端口号区分不同设备
// 原理: 设备:端口 → NAT → 公网IP:新端口
```
> **Note**: 🏠 你的家庭路由器就是一个 NAT 设备——所有手机/电脑共享一个公网 IP，内部分配 192.168.x.x 私有地址。

## 域名系统

域名是 IP 地址的人类可读别名，方便记忆和使用。
```
// ========== 域名结构 ==========
// 完整 URL: https://www.example.com/blog
//                      └─────┬─────┘
//                      Fully Qualified Domain Name
//
// 层级分解:
// . (根域) → com (顶级域) → example (二级域)
//    → www (子域名)
//
// 顶级域 (TLD) 分类:
//   通用 TLD: .com .org .net .info
//   国家 TLD: .cn .jp .uk .de
//   新通用: .app .dev .io .ai
//   基础设施: .arpa

// ========== 域名 vs IP ==========
// 域名:  human-readable, 可变更, 不唯一
// IP:    machine-readable, 固定, 全球唯一
// 映射:  一个域名可对应多个 IP (负载均衡)
//         一个 IP 可对应多个域名 (虚拟主机)
```
## 子网掩码与 CIDR
```
// ========== 子网掩码 ==========
// 决定 IP 地址的网络部分和主机部分
//
// 255.255.255.0 = /24
// 二进制: 11111111.11111111.11111111.00000000
//          └────── 网络位 ──────┘└─主机位─┘
//
// 示例: 192.168.1.0/24
//   网络地址: 192.168.1.0
//   可用主机: 192.168.1.1 - 192.168.1.254
//   广播地址: 192.168.1.255
//
// ========== CIDR 表示法 ==========
// /8  = 255.0.0.0       (16,777,214 台主机)
// /16 = 255.255.0.0     (65,534 台主机)
// /24 = 255.255.255.0   (254 台主机)
// /28 = 255.255.255.240 (14 台主机)
// /32 = 单个主机
```
