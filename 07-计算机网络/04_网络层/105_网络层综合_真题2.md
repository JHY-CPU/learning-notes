# 网络层综合真题（二）

## 核心概念
- 网络层综合真题涵盖IP地址分配、路由选择、分片计算、ARP解析等多个考点
- 408考试中网络层占比约15-20分，综合题常将多个知识点结合考查
- 常见考点组合：CIDR+路由聚合、IP分片+MTU、ARP+数据链路层封装

## 原理分析

### 综合题型分析
- **题型一**：给定网络拓扑，计算路由表条目并判断数据转发路径
- **题型二**：给定IP数据报和MTU限制，计算分片后的各片段偏移量和标志位
- **题型三**：给定子网划分方案，判断主机间通信是否可达

### 典型真题解析
某网络拓扑：主机A(192.168.1.10/26)通过路由器R连接到主机B(192.168.1.70/26)

子网掩码 /26 = 255.255.255.192，每个子网62个可用主机

A所在子网：192.168.1.0/26（范围 192.168.1.1 ~ 192.168.1.62）
B所在子网：192.168.1.64/26（范围 192.168.1.65 ~ 192.168.1.126）

A和B不在同一子网，需要路由器转发

### 关键公式
- 子网可用主机数：$2^n - 2$（n为主机位数，减去网络地址和广播地址）
- 分片偏移量计算：片偏移字段 = 实际偏移字节数 / 8

### 综合真题示例
**题目**：主机A（192.168.1.10/28）发送TCP段到主机B（10.0.0.5/24），中间经过路由器R1和R2。以太网MTU=1500字节。

**解题步骤**：
1. **判断是否同一子网**：A的子网192.168.1.0/28，B的子网10.0.0.0/24，不同子网
2. **确定下一跳**：A查路由表→默认网关R1
3. **ARP解析**：A ARP解析R1接口MAC（不是B的MAC）
4. **帧封装**：源MAC=A，目的MAC=R1，源IP=A，目的IP=B
5. **路由器转发**：R1查路由表→下一跳R2→ARP解析R2 MAC→重新封装帧

### 多知识点组合题
**题目**：IP数据报总长2000字节（首部20字节），标识=1000，经过MTU=800的链路

**解**：
- 数据：2000-20=1980字节
- 每片最大数据：800-20=780字节
- 780/8=97.5→取97×8=776字节
- 片1：总长796，MF=1，偏移0
- 片2：总长796，MF=1，偏移97（776/8）
- 片3：数据=1980-776×2=428，总长448，MF=0，偏移194（1552/8）

### 综合题解题框架
1. 画出网络拓扑，标注IP地址和子网掩码
2. 计算每台主机/接口的子网网络地址
3. 判断通信双方是否在同一子网
4. 确定每跳的源/目的IP和源/目的MAC
5. 考虑MTU限制和分片
6. 检查ARP解析目标

### 408常考组合
- 子网划分 + 最长前缀匹配
- IP分片 + MTU限制
- ARP + MAC地址封装
- CIDR + 路由聚合
- TTL + ICMP超时

## 直观理解
- 综合题像"应用题"，单一知识点是"选择题"——综合题就是多个选择题知识点的组合
- 做综合题的关键：先分解题目，识别每个考查点，再逐一解答
- 画图是解综合题的最好方法

## 协议关联
- 网络层综合题常涉及：IP（地址、分片）、ICMP（差错报告）、ARP（地址解析）
- 与数据链路层结合：以太网帧的MTU限制导致IP分片
- 与传输层结合：TCP的MSS协商避免分片
- 与路由协议结合：路由表构建影响转发路径
