# IPv6与IPv4对比

## 核心概念

### 地址空间对比
| 项目 | IPv4 | IPv6 |
|------|------|------|
| 地址长度 | 32位 | 128位 |
| 地址数量 | $2^{32} \approx 43$亿 | $2^{128} \approx 3.4 \times 10^{38}$ |
| 表示格式 | 点分十进制 | 冒号分隔十六进制 |

### 首部对比
| 特性 | IPv4 | IPv6 |
|------|------|------|
| 首部长度 | 20-60字节可变 | 固定40字节 |
| 首部校验和 | 有 | 无 |
| 分片处理 | 路由器可分片 | 仅源端分片 |
| 可选字段 | 首部直接包含 | 扩展首部链 |
| TTL | TTL字段 | 跳数限制字段 |
| 服务类型 | TOS/DS字段 | 流量类别+流标签 |

### 功能对比
| 功能 | IPv4 | IPv6 |
|------|------|------|
| 地址解析 | ARP（广播） | NDP NS/NA（多播） |
| 差错报告 | ICMPv4 | ICMPv6 |
| 组管理 | IGMP | ICMPv6 MLD |
| 地址配置 | DHCP | SLAAC或DHCPv6 |
| 广播 | 支持 | 不支持（用多播替代） |

## 原理分析
- IPv6取消校验和的原因：数据链路层以太网CRC和传输层TCP/UDP校验和已足够，取消后减少路由器处理开销
- IPv6取消路由器分片的原因：分片和重组消耗路由器大量资源，IPv6使用路径MTU发现让源端直接发送合适大小的数据报
- IPv6取消首部长度字段原因：首部固定40字节，无需长度字段

### IPv6取消IPv4字段的原因
| 取消的IPv4字段 | 原因 |
|----------------|------|
| 首部长度 | 固定40字节，无需长度字段 |
| 标识/标志/片偏移 | 路由器不分片，由源端负责 |
| 首部校验和 | 链路层CRC+传输层校验和足够 |
| 选项 | 移到扩展首部链 |
| TTL | 改名为"跳数限制"（功能相同） |

### ICMPv6 vs ICMPv4
| 功能 | IPv4 | IPv6 |
|------|------|------|
| 差错报告 | ICMPv4 | ICMPv6 |
| 地址解析 | ARP | NDP（NS/NA） |
| 组管理 | IGMP | ICMPv6 MLD |
| 路由器发现 | ICMPv4路由器通告 | NDP RS/RA |
| 前缀发现 | 无 | NDP RS/RA |

### IPv4/IPv6共存策略
1. **双栈**：同时运行两套协议栈，优先使用IPv6
2. **隧道**：IPv6 over IPv4（6to4、Teredo）
3. **翻译**：NAT64（IPv6与IPv4互访）

### 408常考要点
- IPv6取消校验和、取消路由器分片、取消广播
- IPv6基本首部固定40字节（IPv4可变20-60）
- ICMPv6 = ICMPv4 + ARP + IGMP
- 流标签是IPv6新增的20位字段
- IPv6地址128位，IPv4地址32位
- 过渡技术：双栈、隧道、NAT64

## 直观理解
- IPv4像"老城区"——地址紧张、规划混乱
- IPv6像"新开发区"——地址充裕、规划整齐
- 共存阶段像"中英双语"并行，最终逐步过渡到IPv6
- ICMPv6像"多功能工具"——集成了IPv4中三个协议的功能

## 协议关联
- 过渡技术是408考试重点：双栈、隧道、NAT64
- IPv6对应用层透明：HTTP/FTP无需修改，DNS需扩展AAAA记录
- ICMPv6整合了ICMPv4+ARP+IGMP三个协议的功能
- TCP/UDP在IPv6中基本不变，只需修改伪首部
