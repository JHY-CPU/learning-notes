# 01_网络层概述

## 核心概念

- **网络层（Network Layer）**：OSI参考模型第三层，负责将分组从源主机传送到目的主机
- **核心功能**：路由选择（routing）与分组转发（forwarding）
- **数据单位**：分组/数据报（packet/datagram）
- **关键协议**：IP（Internet Protocol）、ICMP、ARP、RARP、IGMP
- **服务模型**：TCP/IP网络层提供**无连接的、尽最大努力交付**的数据报服务（best-effort delivery）
  - 不保证可靠传输
  - 不保证按序到达
  - 不保证延迟和带宽

- **408考试重点**：
  - 网络层向上层提供的是什么服务？（数据报服务 vs 虚电路服务）
  - 路由选择与分组转发的区别
  - IP协议是无连接的，每个分组独立路由

- **常见混淆**：
  - 网络层的"无连接"与传输层TCP的"面向连接"是不同层次的概念
  - 路由选择（routing）确定路径，转发（forwarding）将分组从输入端口送到输出端口

## 原理分析

### 网络层协议栈结构

```
应用层  →  HTTP, FTP, DNS, SMTP
传输层  →  TCP, UDP
网络层  →  IP, ICMP, ARP, IGMP    ← 本层重点
数据链路层 → Ethernet, PPP
物理层  →  比特流传输
```

### 网络层两大功能

**1. 路由选择（Routing）**
- 通过路由算法确定分组从源到目的的最佳路径
- 依赖路由表（routing table）
- 路由协议：RIP、OSPF、BGP

**2. 分组转发（Forwarding）**
- 路由器收到分组后，根据目的IP地址查路由表
- 决定从哪个端口转发出去
- 涉及最长前缀匹配（longest prefix matching）

### 服务对比

| 特性 | 数据报网络 | 虚电路网络 |
|------|-----------|-----------|
| 建立连接 | 不需要 | 需要（呼叫建立） |
| 目的地址 | 每个分组携带完整地址 | 只需虚电路号 |
| 路由选择 | 每个分组独立选择 | 建立时确定，所有分组走同一条路 |
| 节点故障 | 可绕过 | 所有经过该节点的虚电路都断开 |
| 代表 | Internet（IP） | ATM, X.25, 帧中继 |

## 直观理解

**类比：网络层像邮政系统**

- **路由选择** = 邮局系统规划包裹从北京到上海的运输路线（走航空还是铁路，经过哪些中转站）
- **分组转发** = 每个中转站的工人看包裹上的地址，决定往哪个方向送
- **数据报服务** = 寄明信片：每张独立投递，可能走不同路线，不保证全部到达
- **虚电路服务** = 打电话：先拨号建立连接，通话期间线路固定

**记忆技巧**：
- "网络层是'地址+路线'的管理层"
- IP负责"编址和寻址"，路由协议负责"规划路线"，路由器负责"逐跳转发"

## 协议关联

### 跨层交互

```
应用层: DNS将域名→IP地址
  ↓
传输层: TCP/UDP封装，给出源/目端口
  ↓
网络层: IP封装，添加IP头部（源/目IP地址）
  ↓  ↑
    ARP: IP地址→MAC地址（同一网段内通信）
  ↓
数据链路层: 以太网帧封装（源/目MAC地址）
```

### 常见考题组合
- 网络层 + 数据链路层：IP地址与MAC地址的映射（ARP协议）
- 网络层 + 传输层：IP分片与TCP分段的区别（常考陷阱！）
- 网络层 + 应用层：DNS解析过程中的递归与迭代查询

### 考试陷阱
- **IP分片 vs TCP分段**：IP分片在网络层，偏移量以8字节为单位；TCP分段在传输层，由MSS决定
- **路由器工作在哪一层**：网络层（但现代路由器也涉及数据链路层和物理层）
- **网络层不可靠但IP协议为什么还要校验和**：只校验IP头部，不校验数据部分
