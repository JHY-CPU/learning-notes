# 数据包与路由

## 📦 数据包与路由

数据包结构、分组交换、路由算法、路由器工作原理、跳数。

## 数据包 (Packet)

数据包是网络传输的基本单位，大文件被拆分成多个小包独立传输。
```
// ========== 数据包结构 ==========
//
// ┌─────────────────────────────────────┐
// │        头部 (Header)                 │
// ├─────────────────────────────────────┤
// │ 源 MAC 地址 (6 bytes)               │
// │ 目的 MAC 地址 (6 bytes)             │
// │  EtherType (2 bytes)                │
// ├─────────────────────────────────────┤
// │ 源 IP 地址 (4 bytes)                │
// │ 目的 IP 地址 (4 bytes)              │
// │  协议 (1 byte, TCP=6/UDP=17)        │
// │  TTL (1 byte)                       │
// ├─────────────────────────────────────┤
// │ 源端口 (2 bytes)                    │
// │ 目的端口 (2 bytes)                  │
// │  序列号 (4 bytes)                   │
// │  校验和 (2 bytes)                   │
// ├─────────────────────────────────────┤
// │        负载 (Payload)               │
// │  (实际数据,如 HTTP 请求体)          │
// └─────────────────────────────────────┘

// ========== MTU (最大传输单元) ==========
// Ethernet:    1500 bytes
// PPPoE:       1492 bytes
// WiFi:        2272 bytes (通常)
// IPv6 最小:   1280 bytes
```
## 分组交换

互联网使用分组交换 (Packet Switching)，数据被分成包后通过共享网络独立传输。
```
// ========== 分组交换 vs 电路交换 ==========
//
// 电路交换 (传统电话):
//   通话前建立专线,独占带宽
//   优点: 稳定,低延迟
//   缺点: 线路闲置浪费,扩展性差
//
// 分组交换 (互联网):
//   数据分成包,共享网络传输
//   优点: 高效利用带宽,弹性扩展
//   缺点: 可能丢包/乱序/延迟
//
// ========== 分组交换特点 ==========
// 1. 存储转发: 路由器接收完整包再转发
// 2. 统计复用: 动态分配带宽
// 3. 最佳路径: 每个包独立选路
// 4. 乱序到达: 接收端重新排序

// ========== 包可能遇到的问题 ==========
// 丢包 (Packet Loss):    路由器队列满丢弃包
// 乱序 (Out of Order):   包走不同路径到达
// 重传 (Retransmission): TCP 自动重传丢失包
// 延迟 (Latency):        队列缓存 + 链路传播
```
> **Note**: 📦 你发送一张照片时,它被拆分成上千个数据包,每个包可能走不同的路径到达目的地,最后在接收端重新组装——这就是分组交换的威力。

## 路由算法

路由器使用路由协议确定数据包的最佳路径。
```
// ========== 主要路由协议 ==========
//
// OSPF (开放最短路径优先):
//   - 基于链路状态 (Link State)
//   - 使用 Dijkstra 算法计算最短路径
//   - 快速收敛,适合大型网络
//
// BGP (边界网关协议):
//   - 互联网核心路由协议
//   - 基于路径向量 (Path Vector)
//   - 管理自治系统 (AS) 之间的路由
//   - 考虑策略而非单纯最短路径
//
// RIP (路由信息协议):
//   - 基于距离向量 (Distance Vector)
//   - 使用跳数作为度量
//   - 最大 15 跳,小型网络使用

// ========== 路由表示例 ==========
// 目标网络        下一跳          接口    度量
// 192.168.1.0/24  -              eth0    0
// 10.0.0.0/8      192.168.1.1   eth0    1
// 0.0.0.0/0       203.0.113.1   eth1    0 (默认路由)
```
## TTL 与追踪路由
```
// ========== TTL (生存时间) ==========
// 每经过一个路由器, TTL 减 1
// TTL 为 0 时,路由器丢弃包并返回 ICMP 超时
// 防止数据包在网络中无限循环
//
// 默认 TTL 值:
//   Linux:    64
//   Windows:  128
//   macOS:    64
//   路由器:   255

// ========== traceroute 示例 ==========
// $ traceroute google.com
//
//  1  192.168.1.1     (1ms)    ← 家庭路由器
//  2  10.10.0.1       (5ms)    ← ISP 接入点
//  3  172.16.1.2      (8ms)    ← 区域汇聚
//  4  203.0.113.5     (12ms)   ← 骨干网
//  5  209.85.252.1    (15ms)   ← Google 入口
//  6  142.250.80.4    (16ms)   ← Google 服务器
//
// 每一跳代表一个路由器
// 延迟 = 往返时间 (RTT)
```
