# 14_子网掩码

## 核心概念

- **子网掩码（Subnet Mask）**：32位，用于标识IP地址中网络号+子网号的部分
- **结构**：网络号+子网号对应位全1，主机号对应位全0
- **作用**：将IP地址与子网掩码进行**按位AND**运算，得到网络地址

- **默认子网掩码**（分类编址）：
  - A类：255.0.0.0（/8）
  - B类：255.255.0.0（/16）
  - C类：255.255.255.0（/24）

- **CIDR表示法**：用 /n 表示前n位是网络前缀
  - /24 = 255.255.255.0
  - /26 = 255.255.255.192
  - /20 = 255.255.240.0

- **408考点**：
  - 子网掩码与IP地址AND运算得网络地址
  - 不同子网掩码对应不同大小的子网
  - CIDR中子网掩码可以不按字节边界

## 原理分析

### AND运算求网络地址

**例**：IP = 192.168.1.100，掩码 = 255.255.255.192

```
IP地址:   11000000.10101000.00000001.01100100
子网掩码:  11111111.11111111.11111111.11000000
AND结果:  11000000.10101000.00000001.01000000
= 192.168.1.64（网络地址）
```

### CIDR前缀长度对照表

| /n | 掩码 | 可用主机数 |
|----|------|-----------|
| /8 | 255.0.0.0 | 16777214 |
| /16 | 255.255.0.0 | 65534 |
| /24 | 255.255.255.0 | 254 |
| /25 | 255.255.255.128 | 126 |
| /26 | 255.255.255.192 | 62 |
| /27 | 255.255.255.224 | 30 |
| /28 | 255.255.255.240 | 14 |
| /29 | 255.255.255.248 | 6 |
| /30 | 255.255.255.252 | 2 |

### 快速计算可用主机数

$$\text{可用主机数} = 2^{(32-n)} - 2$$

其中 $n$ 是CIDR前缀长度。

## 直观理解

**类比**：子网掩码像"筛子"
- 1的位置：保留IP地址的对应位（网络部分）
- 0的位置：筛掉IP地址的对应位（主机部分）
- AND运算 = 用筛子过滤，只保留网络部分

**记忆技巧**：
- "掩码1的部分是网络，0的部分是主机"
- "AND一下就知道在哪个网络"

## 协议关联

### 子网掩码在网络中的使用

- **主机**：判断目的IP是否在同一子网
  - 同一子网 → 直接ARP获取目的MAC → 发送
  - 不同子网 → 将帧发给默认网关

- **路由器**：查找路由表时用子网掩码进行最长前缀匹配

- **ARP**：只在同一子网内工作，跨子网由路由器转发

### 考试常见错误
- 子网掩码不是IP地址，不能分配给主机作为IP
- 两个不同子网掩码的IP可能计算出相同的网络地址
- /31和/32是特殊前缀，分别有2个和1个IP（无广播地址概念）
