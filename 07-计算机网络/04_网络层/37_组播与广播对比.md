# 组播与广播对比

## 核心概念

### 广播（Broadcast）
- 一对所有：发送给同一广播域内的所有主机
- 目的MAC：FF:FF:FF:FF:FF:FF
- 目的IP：255.255.255.255（有限广播）或子网广播地址（如192.168.1.255）
- 路由器不转发广播

### 组播（Multicast，又称多播）
- 一对多：只发送给加入组播组的主机
- 目的MAC：01:00:5E:xx:xx:xx
- 目的IP：D类地址 224.0.0.0~239.255.255.255
- 路由器可以转发组播

### 对比表
| 特性 | 广播 | 组播 |
|------|------|------|
| 范围 | 同一广播域内所有主机 | 只有组成员收到 |
| 地址 | 255.255.255.255 | 224.0.0.0~239.255.255.255 |
| 路由器 | 不转发 | 可以转发 |
| 效率 | 低（所有主机都处理） | 高（只有组成员处理） |
| 管理协议 | 无 | IGMP |
| IPv6 | 不支持（用多播替代） | 支持 |

### IPv6中的变化
- IPv6取消了广播地址
- 广播功能由组播替代：
  - FF02::1 = 所有节点（类似广播）
  - FF02::2 = 所有路由器
- 任播（Anycast）是IPv6新增的通信方式

## 原理分析

### 组播地址到MAC地址的映射
- D类IP组播地址映射到以太网组播MAC地址
- 映射规则：MAC = 01:00:5E + 0 + IP后23位
- 注意：IP组播地址有28位有意义，但MAC只映射23位
- 多个IP组播地址可能映射到同一个MAC地址

### 任播（Anycast）
- 一对多中最近的一个：数据报发送给一组主机中距离最近的一个
- IPv4和IPv6都支持任播
- 典型应用：DNS根服务器、CDN节点选择
- 任播地址与单播地址格式相同，通过路由协议选择最近节点

### IGMP（Internet Group Management Protocol）
- 用于管理组播组成员关系
- 主机通过IGMP通知路由器自己加入了哪些组播组
- IGMP报文封装在IP数据报中（协议号2）
- IGMP消息只在本地链路有效（TTL=1）

### 组播地址与MAC映射
D类组播IP地址（28位有意义）映射到以太网组播MAC：
```
IP:  1110xxxx.xxxxxxxx.xxxxxxxx.xxxxxxxx
MAC: 00000001.00000000.01011110.0xxxxxxxx.xxxxxxxx.xxxxxxxx
     (固定前25位)  (IP后23位)
```
5位IP未映射，导致32个组播IP对应1个组播MAC。

### 广播的类型
| 类型 | 地址 | 范围 |
|------|------|------|
| 受限广播 | 255.255.255.255 | 本网络内，路由器不转发 |
| 直接广播 | 如192.168.1.255 | 特定子网内，路由器可转发 |

### 408常考要点
- 路由器隔离广播域，不转发广播
- 组播可以跨子网（通过组播路由协议）
- IPv6取消广播，用组播替代
- DHCP Discover使用受限广播（255.255.255.255）
- 组播IP范围：224.0.0.0~239.255.255.255
- 224.0.0.1=所有主机，224.0.0.2=所有路由器

## 直观理解
- 广播像"大喇叭喊话"——所有人都要听，不管你感不感兴趣
- 组播像"群聊消息"——只有群成员能收到
- 任播像"拨打110"——自动接通最近的服务点
- IGMP像"签到表"——主机告诉路由器"我在这个群"

## 协议关联
- IGMP管理组播组成员
- PIM等组播路由协议建立组播分发树
- 广播只在本地链路有效，路由器隔离广播域
- DHCP Discover使用广播，组播协议使用组播
- IPv6的组播地址（FF00::/8）替代了IPv4的广播功能
