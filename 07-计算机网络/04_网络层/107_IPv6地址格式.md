# IPv6地址格式

## 核心概念
- IPv6地址长度：128位，用8组4位十六进制数表示，每组之间用冒号分隔
- 完整格式：`2001:0db8:0000:0000:0000:0000:0000:0001`
- 地址缩写规则是408考试必考内容

### 缩写规则（必考）
- **规则1**：每组前导零可省略，如 `0db8` 写成 `db8`，`0001` 写成 `1`
- **规则2**：连续的全零组可用 `::` 替代，但每个地址只能用一次 `::`

### 缩写示例
完整格式：`2001:0db8:0000:0000:0000:0000:0000:0001`
第一步省略前导零：`2001:db8:0:0:0:0:0:1`
第二步合并全零组：`2001:db8::1`

特殊地址：
- 环回地址：`::1`（相当于IPv4的127.0.0.1）
- 未指定地址：`::`（相当于IPv4的0.0.0.0）
- 链路本地地址前缀：`FE80::/10`

### 地址前缀表示
- 使用CIDR记法：`2001:db8::/32` 表示前32位为网络前缀

## 原理分析
- 128位地址分为两部分：前64位为网络前缀，后64位为主机标识（接口标识符）
- 接口标识符可由EUI-64方法自动生成（将48位MAC地址扩展为64位）
- 全球单播地址前缀由IANA分配，目前分配 `2000::/3` 范围

### 缩写规则详解
**规则1**：省略前导零
- `2001:0db8:0001:0000:0000:0000:0000:0001`
- 省略前导零：`2001:db8:1:0:0:0:0:1`

**规则2**：连续全零组用`::`替代（只能用一次）
- `2001:db8:1:0:0:0:0:1` → `2001:db8:1::1`

**反向展开练习**：
- `::1` → `0000:0000:0000:0000:0000:0000:0000:0001`
- `FF02::1` → `FF02:0000:0000:0000:0000:0000:0000:0001`
- `2001:db8::/32` → 前32位为`2001:0db8`

### 常见缩写陷阱
- `::`只能用一次：`2001::1::1`是**错误**的
- 每组最多4位十六进制：`2001:db8:12345::`是**错误**的
- `::`展开后补齐到128位（8组）

### 地址结构
IPv6地址通常分为两部分：
- 前64位：网络前缀（类似IPv4的网络号）
- 后64位：接口标识符（类似IPv4的主机号，但更长）

### DNS中的IPv6
| 记录类型 | 用途 |
|----------|------|
| A | IPv4地址映射 |
| AAAA | IPv6地址映射 |
| PTR | 反向解析 |

### 408常考要点
- IPv6地址128位，8组4位十六进制数，冒号分隔
- `::`缩写只能用一次
- 环回地址：`::1`，未指定地址：`::`
- 链路本地地址前缀：`FE80::/10`
- 全球单播地址前缀：`2000::/3`
- 地址前缀使用CIDR记法

## 直观理解
- IPv6地址像一栋超级大楼的门牌号：`省:市:区:街道:楼:单元:层:房号`
- `::` 缩写像"等"——中间连续好几组全是0就用双冒号代替
- 每组4个十六进制数字 = 16位，8组 = 128位
- IPv6地址展开像"解压缩"——::补全为多个0000组

## 协议关联
- IPv6地址通过NDP（邻居发现协议）自动配置，替代IPv4中的DHCP+ARP组合
- DNS中使用AAAA记录存储IPv6地址（A记录存储IPv4地址）
- MAC地址通过EUI-64转换生成链路本地地址，建立邻居关系
- 接口标识符可以由EUI-64自动生成或随机生成（隐私扩展）
