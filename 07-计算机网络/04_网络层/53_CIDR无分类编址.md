# 17_CIDR无分类编址

## 核心概念

- **CIDR（Classless Inter-Domain Routing）**：无分类域间路由选择
  - RFC 1518/1519定义，1993年提出
  - **彻底抛弃**传统的A/B/C类分类编址
  - 使用**斜线记法**（Slash Notation）：IP地址/n

- **CIDR表示法**：a.b.c.d/n
  - n = 前缀长度（网络前缀的位数）
  - 例：192.168.1.0/24 表示前24位是网络前缀

- **地址块（Address Block）**：
  - 一个CIDR地址块 = 网络前缀相同的所有IP地址
  - 地址块大小 = $2^{(32-n)}$
  - 例：/20 的地址块包含 $2^{12} = 4096$ 个地址

- **路由聚合（Route Aggregation）**/ 超网（Supernetting）：
  - 将多个连续的小地址块合并成一个大地址块
  - 减少路由表条目数量
  - 例：8个/24可以聚合成1个/21

- **408考点**：
  - CIDR用前缀长度替代分类编址的固定类别
  - 最长前缀匹配（Longest Prefix Matching）
  - 路由聚合计算

## 原理分析

### CIDR vs 分类编址

| 特性 | 分类编址 | CIDR |
|------|---------|------|
| 地址划分 | 固定A/B/C类 | 灵活前缀长度 |
| 掩码 | 默认掩码 | 自由指定 |
| 子网划分 | 在类内借位 | 任意位置划界 |
| 路由聚合 | 不支持 | 支持 |
| 地址浪费 | 严重 | 大大减少 |

### CIDR地址块计算

**例**：给定地址块 200.10.16.0/20

- 前缀长度：20位
- 地址范围：200.10.16.0 ~ 200.10.31.255
- 地址总数：$2^{12} = 4096$
- 可分配地址：4094（去掉全0全1）
- 最小地址：200.10.16.0（前20位固定，后12位全0）
- 最大地址：200.10.31.255（前20位固定，后12位全1）

### 路由聚合

**例**：将以下4个地址块聚合：
- 200.10.16.0/24
- 200.10.17.0/24
- 200.10.18.0/24
- 200.10.19.0/24

分析：4 = $2^2$，需要合并后前缀长度减少2位
聚合结果：200.10.16.0/22

验证：
- 200.10.16.0 = 200.10.00010000.0
- 200.10.19.0 = 200.10.00010011.0
- 前22位相同：200.10.000100 = 200.10.16/22 ✓

## 直观理解

**类比**：
- 分类编址 = 只有S/M/L三种固定尺码的衣服
- CIDR = 量身定做，按需裁剪

**记忆**：
- "CIDR = 无类 = 前缀长度自由 = 地址不浪费"
- "聚合 = 大的前缀长度 → 小的前缀长度"

## 协议关联

### CIDR与路由

- 路由器使用最长前缀匹配来转发分组
- CIDR使路由表可以用更少的条目覆盖更多网络
- CIDR是解决IPv4地址枯竭的重要手段之一
- 与NAT配合使用，延缓了IPv4地址耗尽

### 考试关联
- CIDR + 最长前缀匹配 = 高频考点
- CIDR + 路由聚合 = 选择题/计算题常考
- CIDR中的子网划分不再受分类限制
