# ICMP综合练习

## 核心概念
- ICMP（Internet Control Message Protocol）：互联网控制报文协议
- ICMP封装在IP数据报中，协议号为1
- 用于差错报告和网络诊断（ping、traceroute）

### ICMP报文分类
- **差错报告报文**：
  - 目的不可达（类型3）
  - 源站抑制（类型4，已弃用）
  - 超时（类型11）
  - 参数问题（类型12）
  - 重定向（类型5）
- **查询报文**：
  - 回显请求/应答（类型8/0）——ping
  - 时间戳请求/应答（类型13/14）

### ICMP差错报文规则
- 不对ICMP差错报文产生ICMP差错报文（避免无限循环）
- 不对分片的第一个以外的分片产生ICMP差错报文
- 不对多播地址产生ICMP差错报文
- 不对特殊地址（127.0.0.0、0.0.0.0）产生ICMP差错报文

## 原理分析

### ping工作原理
1. 发送ICMP回显请求（类型8，代码0）
2. 目的主机回复ICMP回显应答（类型0，代码0）
3. 计算往返时间（RTT）

### traceroute工作原理
1. 发送TTL=1的UDP数据报，第一跳路由器回复ICMP超时
2. 发送TTL=2的UDP数据报，第二跳路由器回复ICMP超时
3. 重复直到到达目的主机（回复ICMP端口不可达）

### ICMP报文格式
| 字段 | 长度 | 说明 |
|------|------|------|
| 类型 | 1字节 | 报文类型 |
| 代码 | 1字节 | 报文子类型 |
| 校验和 | 2字节 | ICMP报文校验 |
| 其他 | 4字节 | 依类型不同 |

## 直观理解
- ICMP像"快递异常通知"——包裹送不到时（差错），快递公司发通知告诉你原因
- ping像"敲门"——敲一下门（请求），听到"谁啊"（应答）就知道门有人
- traceroute像"沿途问路"——每到一个路口问"到哪了"，记录所有经过的路口

## 协议关联
- ICMP封装在IP数据报中（协议号1）
- ICMP差错报文包含原始IP首部+数据前8字节（含TCP/UDP端口号）
- 路径MTU发现使用ICMP"数据报过大"消息（类型3，代码4）
- IPv6使用ICMPv6替代ICMPv4+ARP+IGMP
