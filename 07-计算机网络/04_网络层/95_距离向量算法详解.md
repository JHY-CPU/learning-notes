# 59_距离向量算法详解

## 核心概念

- **距离向量算法（Distance Vector Algorithm）**：
  - 又称分布式Bellman-Ford算法
  - 每个节点维护到所有目的节点的距离向量
  - 通过和邻居交换距离向量逐步收敛

- **核心方程**：
$$D_x(y) = \min_{v \in N(x)} \{c(x,v) + D_v(y)\}$$
  - $D_x(y)$：节点x到目的y的最短距离
  - $c(x,v)$：x到邻居v的代价
  - $D_v(y)$：邻居v到y的最短距离
  - $N(x)$：x的邻居集合

- **RIP中的应用**：所有 $c(x,v) = 1$（跳数）
$$D_x(y) = \min_{v \in N(x)} \{1 + D_v(y)\}$$

## 原理分析

### 算法执行过程

```
初始化：
  for 每个节点x:
    for 每个目的y:
      if y 是x的直连邻居:
        D_x(y) = c(x,y)
      else:
        D_x(y) = ∞

重复（直到收敛）：
  等待收到来自邻居的距离向量更新
  for 每个邻居v:
    for 每个目的y:
      用Bellman-Ford方程更新D_x(y)
  if 任何D_x(y)发生改变:
    通知所有邻居
```

### 收敛特性

**好消息传得快**：
- 若某网络变得可达，信息逐跳传播
- 每轮更新距离减小，最终收敛

**坏消息传得慢（Count to Infinity）**：
- 若某网络不可达，邻居可能还不知道
- 邻居告诉你"我能到"，但其实路径经过你
- 距离不断增加直到达到最大值

### 环路问题详解

```
拓扑：A—B—C，A连N1

正常状态：A到N1距离1，B到N1距离2（通过A），C到N3距离3（通过B）

A到N1的链路断开：
  第1轮：A设D_A(N1)=∞
         B还没更新，告诉C "我到N1距离2"
         A告诉B "我到N1距离∞"
         B收到A的消息，但C告诉B "我到N1距离3"
         B更新D_B(N1)=4（通过C！）

  第2轮：B告诉A "我到N1距离4"
         A更新D_A(N1)=5（通过B！）
         C告诉B "我到N1距离4"
         B更新D_B(N1)=5

  ...不断递增直到16（不可达）
```

### 解决方案

**1. 水平分割（Split Horizon）**：
- 不向来源方向回传该路由信息
- B从A学到N1，B不向A通告N1的路由

**2. 毒性逆转（Poison Reverse）**：
- B从A学到N1，B以距离16回传给A
- 比水平分割更积极

**3. 触发更新**：
- 路由变化时立即发送更新，不等30秒

**4. 抑制计时器**：
- 收到不可达信息后，等待一段时间再接受更好路径

## 直观理解

**类比**：距离向量像"道听途说"
- 每个人只知道邻居告诉他的信息
- 好消息传得快（"近了近了"）
- 坏消息传得慢（"远了更远了...直到太远了"）

**记忆口诀**：
- "邻居说多远，加1就是我的距离"
- "好消息快坏消息慢，水平分割来防环"

## 协议关联

- RIP使用距离向量算法
- EIGRP是高级距离向量协议（DUAL算法）
- 距离向量与链路状态是两大路由算法范式
- 考试中距离向量算法的计算和环路问题是高频考点
