# 46_RIP协议

## 核心概念

- **RIP（Routing Information Protocol）**：路由信息协议
  - 类型：**距离向量**路由协议（Distance Vector）
  - 范围：IGP（自治系统内部）
  - 度量：**跳数（Hop Count）**
  - 最大跳数：**16**（16表示不可达）
  - 基于**UDP**，端口号**520**
  - 版本：RIPv1（有类）、RIPv2（无类，支持CIDR）

- **RIP核心特点**：
  1. 仅和**相邻路由器**交换信息
  2. 交换的是自己的**整个路由表**
  3. **每30秒**定期更新一次
  4. 若**180秒**未收到邻居更新，认为邻居不可达（跳数设为16）
  5. 收敛速度慢

- **RIP的三个要点**（408必考）：
  1. 仅和相邻路由器交换信息
  2. 交换的信息是路由表（全部信息）
  3. 按固定时间间隔交换信息（30秒）

- **408考点**：
  - RIP是距离向量协议
  - 最大跳数16，适用于小型网络
  - 定期30秒更新，180秒超时
  - RIP基于UDP
  - 可能产生路由环路（count-to-infinity问题）

## 原理分析

### RIP工作过程

```
初始化：
  每个路由器只知道直连网络，距离为1

每30秒：
  向相邻路由器发送整个路由表

收到邻居X的路由表后：
  for 每个条目(目的网络N, 距离d):
    if N不在自己的路由表中:
      添加条目：(N, 距离d+1, 下一跳X)
    else if N在表中:
      if 下一跳是X:  # 必须更新（X的路由可能变了）
        更新距离为d+1
      else:  # 下一跳不是X
        if d+1 < 当前距离:
          更新距离为d+1，下一跳改为X
```

### RIP的时间参数

| 参数 | 值 | 含义 |
|------|-----|------|
| 更新周期 | 30秒 | 定期广播路由表 |
| 超时时间 | 180秒 | 6个周期未收到更新，标记为不可达 |
| 垃圾回收 | 120秒 | 不可达后保留120秒再删除 |

### RIPv1 vs RIPv2

| 特性 | RIPv1 | RIPv2 |
|------|-------|-------|
| 子网掩码 | 不支持（有类） | 支持（无类） |
| 认证 | 不支持 | 支持 |
| 广播/组播 | 广播(255.255.255.255) | 组播(224.0.0.9) |
| 路由聚合 | 不支持 | 支持 |

## 直观理解

**类比**：RIP像村民传话
- 每个人只和邻居聊天（仅和相邻路由器交换）
- 每次聊天把知道的所有路都说一遍（交换整个路由表）
- 每30分钟聊一次（定期更新）
- "到XX村要走5步"（跳数度量）
- 如果18分钟没听到某邻居的消息，认为他出事了（超时机制）

**记忆口诀**：
- "RIP三原则：只传邻居，传全路由表，定时30秒"
- "跳数上限16，超了不可达"
- "基于UDP 520端口"

## 协议关联

### RIP的缺点

- **收敛慢**：好消息传得快，坏消息传得慢
- **计数到无穷**：路由环路可能导致跳数不断增加
- **限制网络规模**：最大跳数16
- **定期广播占用带宽**：每30秒广播整个路由表

### 解决环路的方法
- 水平分割（Split Horizon）：不向来源方向回传路由信息
- 毒性逆转（Poison Reverse）：将从某邻居学到的路由以跳数16回传
- 抑制计时器（Hold-down Timer）：收到不可达信息后等待一段时间

### 考试要点
- RIP是距离向量协议，仅和相邻路由器交换信息
- 最大跳数16，30秒更新，180秒超时
- RIP使用UDP 520端口
- 适用于小型网络（<15跳）
