# SDN概念考试题

## 核心概念

### 高频考点
- SDN的核心特征：控制平面与数据平面分离
- SDN控制器集中管理网络，转发设备只执行转发
- OpenFlow是SDN最典型的南向接口协议
- 流表（Flow Table）包含匹配字段、优先级、计数器和动作

### 常考对比
| 项目 | 传统网络 | SDN |
|------|----------|-----|
| 控制方式 | 分布式 | 集中式 |
| 路由计算 | 各设备独立运行路由协议 | 控制器集中计算 |
| 转发表来源 | 路由协议生成 | 控制器下发 |
| 灵活性 | 较低 | 很高 |

## 原理分析

### 典型选择题
**Q1**: SDN的核心特征是什么？
A. 使用高速交换机 B. 控制平面与数据平面分离
C. 使用IPv6 D. 取消路由器

**答**: B。SDN的核心思想是将网络的控制逻辑从转发设备中分离出来，集中到控制器。

**Q2**: SDN架构中，OpenFlow协议属于哪一层之间的接口？
A. 应用层与控制层之间 B. 控制层与数据层之间
C. 数据层内部 D. 应用层内部

**答**: B。OpenFlow是控制层（控制器）与数据层（交换机）之间的南向接口协议。

**Q3**: 下列关于SDN的描述，错误的是？
A. SDN控制器可以集中管理整个网络
B. SDN交换机不需要运行路由协议
C. SDN完全不需要底层物理设备
D. SDN支持通过编程方式配置网络

**答**: C。SDN仍然需要物理交换机等设备，只是控制逻辑集中化了。

### SDN接口类型
- **南向接口**：控制器→数据层（如OpenFlow）
- **北向接口**：应用→控制器（如REST API）
- **东西向接口**：控制器之间

### SDN三层架构
```
应用层：网络应用（负载均衡、安全策略等）
    ↓ 北向接口（REST API）
控制层：SDN控制器（集中控制逻辑）
    ↓ 南向接口（OpenFlow）
数据层：交换机/路由器（仅执行转发）
```

### OpenFlow流表结构
| 字段 | 说明 |
|------|------|
| 匹配字段 | 源IP、目的IP、端口、协议等 |
| 优先级 | 多条规则匹配时的优先顺序 |
| 计数器 | 匹配的数据报数量和字节数 |
| 超时时间 | 空闲超时和硬超时 |
| 动作 | 转发、丢弃、修改、发送到控制器等 |

### SDN的优势
1. **集中管理**：全局视图，最优路径计算
2. **灵活编程**：通过软件定义网络行为
3. **快速部署**：新功能通过控制器下发，无需改设备
4. **网络虚拟化**：在物理网络上创建虚拟网络

### SDN的挑战
1. **控制器可靠性**：单点故障风险
2. **扩展性**：大规模网络中控制器性能瓶颈
3. **安全性**：控制器成为攻击目标
4. **标准化**：不同厂商的控制器和API不统一

### 408常考要点
- SDN核心：控制平面与数据平面分离
- OpenFlow是南向接口协议（控制器→交换机）
- 流表匹配采用最长前缀匹配
- SDN控制器集中计算路由，交换机只执行转发
- SDN不替代物理设备，只改变控制逻辑的位置

## 直观理解
- SDN像"无人驾驶交通系统"——中央电脑统一调度所有路口，信号灯只执行命令
- 传统网络像"每个路口有交警自己判断"——分散决策
- OpenFlow像"命令协议"——中央电脑发给信号灯的指令格式

## 协议关联
- SDN控制器可替代OSPF、BGP等分布式路由协议
- SDN与NFV结合实现网络功能虚拟化
- SDN可与传统网络共存，逐步演进
- OpenFlow定义了控制器与交换机之间的通信标准
