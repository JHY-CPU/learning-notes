# 24_IP分片

## 核心概念

- **IP分片（Fragmentation）**：当IP数据报长度超过下一跳链路的MTU时，需要分片传输

- **MTU（Maximum Transmission Unit）**：最大传输单元
  - 以太网MTU = 1500字节
  - 数据报总长度 > MTU → 需要分片

- **分片三要素**（408必考）：
  1. **标识（Identification）**：同一数据报的所有分片具有相同的标识
  2. **标志（Flags）**：
     - DF = 0：允许分片
     - DF = 1：禁止分片（若超过MTU则丢弃，返回ICMP差错）
     - MF = 1：后面还有分片
     - MF = 0：最后一个分片（或未分片）
  3. **片偏移（Fragment Offset）**：以 **8字节** 为单位，表示该分片在原数据报中的位置

- **分片与重组**：
  - 分片可以在源主机或中间路由器发生
  - **重组只在目的主机**进行
  - 每个分片独立路由，可能走不同路径

- **分片规则**：
  - 每个分片的数据部分长度必须是 **8的倍数**（除最后一个分片外）
  - 每个分片都有自己的IP首部（大部分字段复制原首部）

## 原理分析

### 分片计算

**例**：原数据报数据部分3980字节，MTU=1500字节

IP首部20字节，可携带数据 = 1500 - 20 = 1480字节
1480是8的倍数 ✓

分片过程：
| 分片 | 数据长度 | 片偏移 | MF | 总长度 |
|------|---------|--------|-----|--------|
| 1 | 1480 | 0 | 1 | 1500 |
| 2 | 1480 | 185 | 1 | 1500 |
| 3 | 1020 | 370 | 0 | 1040 |

- 片偏移 = 该分片第一个字节在原数据报中的位置 ÷ 8
- 分片1：0 ÷ 8 = 0
- 分片2：1480 ÷ 8 = 185
- 分片3：2960 ÷ 8 = 370

验证：1480 + 1480 + 1020 = 3980 ✓

### 分片计算公式

- 最大数据部分 = MTU - IP首部长度
- 每片数据（除最后）必须是8的倍数
- 分片数 = ⌈总数据 / 每片最大数据⌉

$$\text{每片最大数据} = \left\lfloor \frac{MTU - IP_{header}}{8} \right\rfloor \times 8$$

## 直观理解

**类比**：IP分片像拆快递
- 一个大包裹装不进快递柜（MTU限制）
- 拆成多个小包裹
- 每个小包裹上写"第X部分，共N部分"（MF标志）
- 收件人收到后按顺序拼装（重组）
- 包裹上都有相同的快递单号（标识字段）

**记忆口诀**：
- "标识相同片偏移8，MF=1后有片，MF=0是结尾"
- "分片可以在路上，重组一定在终点"

## 协议关联

### 分片与TCP/UDP

- **UDP**：应用层数据可能很大 → UDP报文 → IP分片
  - DNS、DHCP使用小数据，一般不会分片
- **TCP**：TCP自己控制MSS（最大报文段长度），通常避免IP分片
  - MSS = MTU - IP首部 - TCP首部 = 1500 - 20 - 20 = 1460
  - TCP的三次握手中协商MSS

### 分片与ICMP

- DF=1且需要分片 → 丢弃分组 + 返回 **ICMP目的不可达（需要分片但DF置位）** 报文
- 该报文中包含下一跳的MTU值（PMTUD：路径MTU发现）

### 考试陷阱
- 片偏移的单位是8字节，不是字节！
- 重组只在目的主机进行，中间路由器不重组
- 分片后的每个分片独立进行路由选择
