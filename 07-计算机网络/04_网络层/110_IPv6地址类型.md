# IPv6地址类型

## 核心概念
- IPv6定义三种地址类型：单播（Unicast）、多播（Multicast）、任播（Anycast）
- IPv6取消了广播地址，用多播替代

### 单播地址
| 类型 | 前缀 | 说明 |
|------|------|------|
| 全球单播地址(GUA) | 2000::/3 | 相当于IPv4公网地址 |
| 链路本地地址(LLA) | FE80::/10 | 仅在本地链路有效 |
| 唯一本地地址(ULA) | FC00::/7 | 相当于IPv4私有地址 |
| 环回地址 | ::1 | 相当于127.0.0.1 |
| 未指定地址 | :: | 相当于0.0.0.0 |

### 多播地址
- 格式：`FF00::/8`
- `FF02::1` = 所有节点（类似广播）
- `FF02::2` = 所有路由器
- `FF02::1:FFxx:xxxx` = 请求节点多播地址（用于地址解析替代ARP）

### 任播地址
- 从单播地址空间分配，无特殊前缀
- 多个接口共享同一地址，数据报只转发到最近的一个
- 用于负载均衡和冗余

## 原理分析

### EUI-64地址生成
将48位MAC地址扩展为64位接口标识符：
1. 在MAC地址第3和第4字节之间插入 `FFFE`
2. 翻转第7位（U/L位取反）

示例：MAC = `00-1A-2B-3C-4D-5E`
- 插入FFFE：`001A:2BFF:FE3C:4D5E`
- 翻转U/L位：`021A:2BFF:FE3C:4D5E`
- 链路本地地址：`FE80::021A:2BFF:FE3C:4D5E`

### 常用多播地址
| 地址 | 含义 |
|------|------|
| FF02::1 | 所有节点（类似IPv4广播） |
| FF02::2 | 所有路由器 |
| FF02::5 | OSPFv3所有路由器 |
| FF02::6 | OSPFv3所有DR |
| FF02::9 | RIPng路由器 |
| FF02::1:FFxx:xxxx | 请求节点多播（用于NDP） |

### 请求节点多播地址
- 由单播地址的后24位生成
- 格式：`FF02::1:FF` + 单播地址后24位
- 用于NDP的邻居请求（NS）消息替代ARP广播
- 优点：只有目标节点和配置了相同后缀的节点响应

### IPv6地址类型选择
| 场景 | 使用地址类型 |
|------|------------|
| 同链路通信 | 链路本地地址 |
| 跨网段通信 | 全球单播地址 |
| 向所有节点发送 | 多播FF02::1 |
| 冗余服务 | 任播 |
| 配置前的临时通信 | 链路本地地址 |

### IPv4与IPv6地址类型对应
| IPv4 | IPv6对应 |
|------|----------|
| 公网地址 | 全球单播地址 |
| 私有地址 | 唯一本地地址 |
| 广播地址 | 多播FF02::1 |
| 127.0.0.1 | ::1 |
| 0.0.0.0 | :: |
| 无对应 | 任播（IPv6新增） |

### 408常考要点
- IPv6三种地址类型：单播、多播、任播（取消了广播）
- 链路本地地址（FE80::/10）是自动配置的，用于邻居发现
- 全球单播地址（2000::/3）相当于公网地址
- 请求节点多播地址用于替代ARP广播
- 环回地址::1，未指定地址::
- EUI-64：MAC→接口标识符的转换方法

## 直观理解
- 单播 = 一对一打电话
- 多播 = 微信群聊
- 任播 = 拨打110（自动转到最近派出所）
- 链路本地地址像"对讲机"——只在同一条链路上能通信
- 请求节点多播像"精准叫人"——不像广播那样叫所有人

## 协议关联
- 链路本地地址是NDP（邻居发现协议）的基础
- 请求节点多播地址替代IPv4中的ARP广播请求
- 全球单播地址通过SLAAC或DHCPv6获得
- FF02::1替代了IPv4中的广播地址
- 任播用于DNS根服务器和CDN负载均衡
