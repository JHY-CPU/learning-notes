# IPv6地址类型

## 核心概念
- IPv6定义三种地址类型：单播（Unicast）、多播（Multicast）、任播（Anycast）
- IPv6取消了广播地址，用多播替代

### 单播地址
| 类型 | 前缀 | 说明 |
|------|------|------|
| 全球单播地址(GUA) | 2000::/3 | 相当于IPv4公网地址 |
| 链路本地地址(LLA) | FE80::/10 | 仅在本地链路有效 |
| 唯一本地地址(ULA) | FC00::/7 | 相当于IPv4私有地址 |
| 环回地址 | ::1 | 相当于127.0.0.1 |
| 未指定地址 | :: | 相当于0.0.0.0 |

### 多播地址
- 格式：`FF00::/8`
- `FF02::1` = 所有节点（类似广播）
- `FF02::2` = 所有路由器
- `FF02::1:FFxx:xxxx` = 请求节点多播地址（用于地址解析替代ARP）

### 任播地址
- 从单播地址空间分配，无特殊前缀
- 多个接口共享同一地址，数据报只转发到最近的一个
- 用于负载均衡和冗余

## 原理分析

### EUI-64地址生成
将48位MAC地址扩展为64位接口标识符：
1. 在MAC地址第3和第4字节之间插入 `FFFE`
2. 翻转第7位（U/L位取反）

示例：MAC = `00-1A-2B-3C-4D-5E`
- 插入FFFE：`001A:2BFF:FE3C:4D5E`
- 翻转U/L位：`021A:2BFF:FE3C:4D5E`
- 链路本地地址：`FE80::021A:2BFF:FE3C:4D5E`

## 直观理解
- 单播 = 一对一打电话
- 多播 = 微信群聊
- 任播 = 拨打110（自动转到最近派出所）
- 链路本地地址像"对讲机"——只在同一条链路上能通信

## 协议关联
- 链路本地地址是NDP（邻居发现协议）的基础
- 请求节点多播地址替代IPv4中的ARP广播请求
- 全球单播地址通过SLAAC或DHCPv6获得
