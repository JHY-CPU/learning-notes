# 移动IP概述

## 核心概念
- 移动IP允许主机在不同网络之间移动时保持IP地址不变，实现不间断通信
- 核心组件：
  - **家乡代理（Home Agent, HA）**：移动节点家乡网络中的路由器
  - **外地代理（Foreign Agent, FA）**：移动节点当前所在网络中的路由器
  - **转交地址（Care-of Address, CoA）**：移动节点在外地网络获得的临时地址
  - **家乡地址（Home Address）**：移动节点的永久IP地址

### 工作流程
1. 移动节点离开家乡网络，到达外地网络
2. 移动节点获得转交地址（CoA）
3. 移动节点向家乡代理注册转交地址
4. 家乡代理截获发往移动节点的数据报
5. 家乡代理通过隧道将数据报转发到转交地址
6. 移动节点直接发送响应数据（三角路由）

### 移动IPv4 vs 移动IPv6
| 特性 | 移动IPv4 | 移动IPv6 |
|------|----------|----------|
| 外地代理 | 需要 | 不需要（使用CoA） |
| 三角路由 | 必须 | 支持路由优化 |
| IPsec | 可选 | 内置支持 |

## 原理分析

### 三角路由问题
- 数据报路径：通信对端 → 家乡代理 → 转交地址 → 移动节点
- 响应路径：移动节点 → 通信对端（直接发送）
- 移动IPv6支持路由优化，可直接通信避免三角路由

### 移动IPv4详细工作流程
1. **代理发现**：外地代理周期性发送代理通告
2. **注册**：移动节点向家乡代理注册转交地址
   - 移动节点→外地代理→家乡代理
   - 注册请求包含：家乡地址、转交地址、注册时间
3. **数据接收**（三角路由）：
   - 通信对端→家乡网络（目的IP=家乡地址）
   - 家乡代理截获→IP-in-IP隧道→转交地址
   - 外地代理解封→移动节点
4. **数据发送**：移动节点直接发送给通信对端

### 家乡代理截获机制
- 家乡代理使用代理ARP（Proxy ARP）
- 在家乡网络中以移动节点的MAC地址响应ARP请求
- 家乡链路上的路由器将发往移动节点的数据报发送给家乡代理

### IP-in-IP隧道封装
```
原始：[IP首部 源=对端 目的=家乡地址][TCP][数据]
封装：[外层IP首部 源=HA 目的=CoA][内层IP首部][TCP][数据]
```

### 移动IPv6改进
- 不需要外地代理（使用配置的CoA）
- 支持路由优化（对端直接发给CoA）
- 绑定更新消息通知对端移动节点的当前位置
- 内置IPsec支持

### 移动IP应用场景
- 手机在不同基站间切换
- 笔记本电脑在不同WiFi间移动
- 车载网络跨区域漫游

### 408常考要点
- 移动IP的关键：家乡地址不变，转交地址变化
- 家乡代理截获+隧道转发是核心机制
- 三角路由：数据经HA中转，响应直接发送
- 移动IPv6不需要外地代理
- 移动IP工作在网络层

## 直观理解
- 移动IP像"快递改址"——搬家后原地址保留（家乡代理代收），快递员把包裹转寄到新地址
- 三角路由像"中转站"——信先寄到老家邮局，再转发到你的新住处
- 家乡代理像"老家的代收点"——帮你收信再转寄

## 协议关联
- 移动IP工作在网络层，依赖IP隧道技术
- 家乡代理使用IP-in-IP隧道封装数据报
- 移动IPv6依赖ICMPv6的NDP实现邻居发现
- 移动IP与IPsec结合提供安全的移动通信
