# 09_数据封装与解封装过程

## 核心概念

- **封装（Encapsulation）**：数据从高层向下层传递时，每层添加本层的控制信息（首部/尾部）
- **解封装（Decapsulation）**：数据从低层向高层传递时，每层去掉本层的控制信息
- **PDU（Protocol Data Unit）**：协议数据单元，每一层的数据传输单位

### 各层PDU名称（408重点记忆）

| 层次 | PDU名称 | 添加的控制信息 |
|------|--------|-------------|
| 应用层 | 报文（Message） | 应用层首部 |
| 传输层 | 报文段(Segment)/用户数据报 | 传输层首部 |
| 网络层 | 数据报/分组（Packet） | 网络层首部 |
| 数据链路层 | 帧（Frame） | 帧头 + 帧尾 |
| 物理层 | 比特流（Bit） | 无 |

## 原理分析

### 封装过程详解

```
发送端（自上而下）：

[ 应用数据 ]
     ↓ + 传输层首部
[ H₄ | 应用数据 ]          ← 报文段/用户数据报
     ↓ + 网络层首部
[ H₃ | H₄ | 应用数据 ]      ← 数据报/分组
     ↓ + 帧头 + 帧尾
[ H₂ | H₃ | H₄ | 应用数据 | T₂ ]  ← 帧
     ↓ 物理层转换
  0101011010...              ← 比特流
```

### 各层首部包含的信息

- **传输层首部**：源端口号、目的端口号、序号、确认号、标志位等
- **网络层首部**：源IP地址、目的IP地址、TTL、协议类型等
- **数据链路层帧头**：目的MAC地址、源MAC地址、类型字段
- **数据链路层帧尾**：FCS（帧校验序列）

### 各层地址信息

| 层次 | 地址类型 | 标识对象 |
|------|---------|---------|
| 数据链路层 | MAC地址 | 同一链路上的相邻设备 |
| 网络层 | IP地址 | 源主机和目的主机（端到端） |
| 传输层 | 端口号 | 主机上的具体进程 |

## 直观理解

- **封装**像俄罗斯套娃：大的数据在外面包裹一层层小壳
- 也像寄信：信纸（应用层）→ 信封写邮编（传输层）→ 快递箱写地址（网络层）→ 贴快递单号（数据链路层）→ 运输（物理层）
- **解封装**就是反过来拆开：到一个中转站拆到网络层看地址，决定下一步；到目的地后逐层拆开直到取出信纸

### 408易混淆点

- **只有数据链路层有尾部**：其他层只有首部
- 首部信息是**累积**的：帧里面的有效载荷是[网络层首部 + 传输层首部 + 应用数据]
- 路由器转发时只需要处理到**网络层**（因为路由器是三层设备）
- 交换机转发时只需要处理到**数据链路层**（因为交换机是二层设备）

## 协议关联

- 封装/解封装过程贯穿全书，是理解协议栈运作的基础
- 各层的地址信息（MAC、IP、端口）在后续章节详细展开
- PDU 和 SDU 的关系详见文件10
- 路由器只处理到网络层、交换机只处理到数据链路层是408常考的对比题
