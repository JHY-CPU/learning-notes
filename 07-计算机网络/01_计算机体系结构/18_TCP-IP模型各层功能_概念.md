# 19_TCP/IP模型各层功能_概念

## 核心概念

### 应用层

- **功能**：直接为用户应用进程提供服务
- 合并了OSI的会话层和表示层的功能
- **主要协议**：
  - HTTP（端口80）：超文本传输协议
  - FTP（端口20/21）：文件传输协议
  - SMTP（端口25）：简单邮件传输协议
  - DNS（端口53）：域名系统
  - Telnet（端口23）：远程登录

### 传输层

- **功能**：提供端到端的通信服务
- **两大协议**：
  - **TCP**：面向连接，提供可靠传输，有流量控制和拥塞控制
  - **UDP**：无连接，不保证可靠，但效率高、延迟低
- **端口号**：标识主机上的进程，16位（0-65535）

### 网际层（网络层）

- **功能**：将分组从源主机传送到目的主机
- **核心协议**：
  - **IP**：网际协议，提供尽力而为的数据报服务
  - **ICMP**：差错报告和网络诊断（ping、traceroute）
  - **IGMP**：组播管理
- **特点**：无连接、不可靠、无序

### 网络接口层

- **功能**：处理与传输媒体的接口
- TCP/IP模型对该层定义最不详细
- 对应OSI的物理层和数据链路层
- 不指定具体的协议，可以运行在各种底层网络之上

## 原理分析

### TCP/IP协议栈的层次功能对比

| 功能 | 所在层 | 协议 |
|------|--------|------|
| 文件传输 | 应用层 | FTP |
| 电子邮件 | 应用层 | SMTP、POP3 |
| 万维网 | 应用层 | HTTP |
| 域名解析 | 应用层 | DNS |
| 可靠传输 | 传输层 | TCP |
| 不可靠传输 | 传输层 | UDP |
| 路由选择 | 网际层 | IP、OSPF、BGP |
| 差错报告 | 网际层 | ICMP |
| 比特传输 | 网络接口层 | Ethernet、PPP |

### TCP/IP的设计哲学

1. **端到端原则**：智能在端系统，网络核心保持简单
2. **分组交换**：数据被分成独立的分组传输
3. **尽力而为**：网络层不保证可靠性，由传输层补充
4. **异构互连**：通过IP层屏蔽底层网络的差异

## 直观理解

- **端到端原则**像邮政系统：邮局（网络）只负责运输信件，不负责确保信件内容的安全和完整——这些由寄件人和收件人自己负责
- **尽力而为**像普通快递：快递公司尽力送达，但不保证100%送达，如需保证请用保价（TCP）

## 协议关联

- TCP/IP应用层直接对应OSI的高三层（应用层+表示层+会话层）
- 网际层的核心协议IP是整个互联网的基础
- 传输层的TCP和UDP是408考试的绝对重点
