# 45_计算机网络性能综合

## 核心概念

### 性能指标总览

| 指标 | 符号 | 公式 | 单位 | 说明 |
|------|------|------|------|------|
| 速率 | $R$ | —— | b/s | 数据传送速率 |
| 带宽 | $B$ | 最高数据率 | b/s | 信道最大通过能力 |
| 吞吐量 | $T$ | $T \leq B$ | b/s | 实际通过数据量 |
| 发送时延 | $t_s$ | $L/R$ | s | 把数据推入信道的时间 |
| 传播时延 | $t_p$ | $d/v$ | s | 比特在信道中传播的时间 |
| 总时延 | $t_d$ | $t_s + t_p + t_{proc} + t_q$ | s | 四种时延之和 |
| 时延带宽积 | $G$ | $t_p \times B$ | bit | 管道中容纳的比特数 |
| 往返时延 | RTT | $\approx 2t_p + t_{proc} + t_q$ | s | 数据往返时间 |
| 利用率 | $U$ | $U = t_s/(t_s+RTT)$ | % | 信道繁忙比例 |

### 关键公式汇总

**香农定理**（有噪声信道容量）：
$$C = W \log_2(1 + \text{SNR}) \quad (\text{b/s})$$

**奈奎斯特准则**（无噪声信道容量）：
$$C = 2W \log_2 V \quad (\text{b/s})$$

**时延-利用率关系**：
$$D = \frac{D_0}{1 - U}$$

**信道利用率**：
$$U = \frac{t_s}{t_s + \text{RTT}}$$

**时延带宽积**：
$$G = t_p \times B \quad (\text{bit})$$

## 原理分析

### 综合计算框架

对于一条端到端通信路径（n条链路，n-1个中间结点）：

**Step 1：计算各段链路的发送时延**
$$t_{s,i} = L / R_i$$

**Step 2：计算各段链路的传播时延**
$$t_{p,i} = d_i / v_i$$

**Step 3：计算总时延**
$$t_d = \sum_{i=1}^{n} t_{s,i} + \sum_{i=1}^{n} t_{p,i} + (n-1) \times t_{转发}$$

**Step 4：计算信道利用率**
$$U = \frac{t_s}{t_s + \text{RTT}}$$

### 综合例题

某网络路径有3条链路，参数如下：

| 链路 | 带宽 | 长度 | 传播速度 |
|------|------|------|---------|
| 链路1 | 10 Mb/s | 1000 km | 2×10^8 m/s |
| 链路2 | 100 Mb/s | 100 km | 2×10^8 m/s |
| 链路3 | 10 Mb/s | 500 km | 2×10^8 m/s |

数据长度 L = 10000 bit。

**各段发送时延**：
- $t_{s1} = 10000 / 10^7 = 1$ ms
- $t_{s2} = 10000 / 10^8 = 0.1$ ms
- $t_{s3} = 10000 / 10^7 = 1$ ms
- 总发送时延 = 2.1 ms

**各段传播时延**：
- $t_{p1} = 10^6 / (2 \times 10^8) = 5$ ms
- $t_{p2} = 10^5 / (2 \times 10^8) = 0.5$ ms
- $t_{p3} = 5 \times 10^5 / (2 \times 10^8) = 2.5$ ms
- 总传播时延 = 8 ms

**总时延**（忽略转发时延）：
$$t_d = 2.1 + 8 = 10.1 \text{ ms}$$

### 性能优化方向

| 瓶颈 | 解决方向 | 具体措施 |
|------|---------|---------|
| 发送时延大 | 提高带宽 | 链路升级（千兆→万兆） |
| 传播时延大 | 减少距离 | CDN缓存、边缘计算 |
| 排队时延大 | 拥塞控制 | TCP拥塞控制、AQM |
| 利用率低 | 增大窗口 | TCP窗口调优、流水线 |
| 首部开销大 | 减少控制信息 | 头部压缩（ROHC） |

## 直观理解

- 网络性能像水管系统：水压（带宽）、水管长度（传播距离）、水龙头开关速度（发送速率）、水管堵塞程度（拥塞）都影响最终效果
- 没有单一指标能完全描述网络性能，需要综合考量
- **带宽**像高速公路的车道数——越多越宽
- **时延**像快递的送达时间——路程和处理都算
- **吞吐量**像收费站每小时通过的车数——实际量，永远小于等于理论最大值

### 408常考计算类型

1. **时延组合计算**：给定多条链路，求总时延
2. **利用率计算**：给定发送时延和RTT，求信道利用率
3. **信道容量计算**：给定带宽和信噪比，用香农定理求最大速率
4. **流水线效应**：分组交换vs报文交换的时延对比

## 协议关联

- 速率和带宽与物理层和数据链路层相关
- 时延的各组成部分分布在各层
- RTT和利用率与传输层的TCP窗口大小密切相关
- 香农定理和奈奎斯特准则是物理层的理论基础
- 时延带宽积决定了TCP发送窗口的最小值，否则信道利用率低
- 分组交换的流水线效应是理解连续ARQ和滑动窗口的基础
