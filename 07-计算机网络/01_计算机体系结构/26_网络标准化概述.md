# 27_网络标准化概述

## 核心概念

### 标准化的重要性

- 不同厂商的设备（Cisco、华为、中兴等）需要互连互通
- 标准确保了互操作性和兼容性——A厂商的路由器能和B厂商的交换机通信
- 没有标准，网络就是一个个信息孤岛，无法形成全球互联网
- 标准化是互联网成功的基石之一

### 主要标准化组织

| 组织 | 全称 | 主要职责 | 代表标准 |
|------|------|---------|---------|
| **ISO** | 国际标准化组织 | 制定OSI参考模型等国际标准 | OSI七层模型、ASN.1 |
| **ITU** | 国际电信联盟 | 电信标准（前身CCITT） | X.25、ISDN、H.264 |
| **IEEE** | 电气和电子工程师协会 | 局域网标准（802系列） | 802.3以太网、802.11 WiFi |
| **IAB** | 互联网体系结构委员会 | 互联网标准的最终裁决 | —— |
| **IETF** | 互联网工程任务组 | 制定互联网协议标准 | TCP、IP、HTTP（RFC文档） |
| **W3C** | 万维网联盟 | Web标准 | HTML、CSS、XML |

### IEEE 802标准系列

| 标准 | 内容 | 状态 |
|------|------|------|
| 802.3 | CSMA/CD（以太网） | 最广泛使用 |
| 802.4 | 令牌总线 | 已淘汰 |
| 802.5 | 令牌环 | 已淘汰 |
| 802.11 | 无线局域网（WiFi） | 广泛使用 |
| 802.15 | 无线个人区域网（蓝牙） | 广泛使用 |
| 802.16 | 无线城域网（WiMAX） | 使用较少 |

### 各组织的关系

```
ISO ──→ OSI参考模型（理论标准）
ITU ──→ 电信标准（X.25、ISDN）
IEEE ─→ 局域网标准（802系列）
IETF ─→ 互联网标准（TCP/IP协议族，RFC文档）
W3C ──→ Web标准（HTML、CSS）
```

## 原理分析

### 标准的制定过程

**IETF标准制定流程**：
1. 个人或团体提出想法
2. 在IETF邮件列表和会议上讨论
3. 编写互联网草案（Internet Draft），有效期6个月
4. 经过多次修订和同行评审
5. 通过IESG审核后成为RFC
6. 经过广泛实施和测试后成为标准

**关键特点**：
- 开放过程：任何人都可以参与，不需要会员资格
- 自下而上：从实际需求出发，不是从理论出发
- 实践导向："rough consensus and running code"

### 事实标准vs法定标准

| 类型 | 特点 | 例子 |
|------|------|------|
| **法定标准** | 由正式标准化组织制定 | OSI模型、X.25 |
| **事实标准** | 因广泛使用而成为标准 | TCP/IP、以太网、Windows |

TCP/IP就是典型的事实标准：虽然不是ISO正式标准，但因为被互联网广泛采用，成为全球通用标准。

## 直观理解

- **标准化**像交通规则：不同国家的人开车都需要遵守统一的规则，否则会出事故
- **IEEE 802**像建筑规范：规定了局域网如何搭建
- **IETF**像开源社区：开放讨论，以代码说话
- **事实标准**像"大家都用的路"：虽然不是官方修的，但走的人多了就成了路

### 408常考要点

- OSI由ISO制定，是理论标准
- TCP/IP由IETF制定，是事实标准
- IEEE 802系列标准是局域网的基础（802.3以太网、802.11 WiFi）
- IETF发布RFC文档作为互联网标准
- W3C制定Web相关标准（HTML、CSS）

## 协议关联

- IEEE 802.3标准（以太网）在数据链路层章节详细展开
- IEEE 802.11（WiFi）在数据链路层章节详细展开
- IETF制定的TCP/IP协议族是互联网的基础，贯穿全书
- OSI虽然由ISO制定但未被广泛采用，TCP/IP虽然不是正式标准却是事实标准
- 408考试中标准化组织的考查一般以选择题形式出现，记住各组织的职责即可
