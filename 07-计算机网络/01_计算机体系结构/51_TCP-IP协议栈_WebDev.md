# TCP-IP协议栈

## 📚 TCP/IP 协议栈

TCP/IP 四层模型、各层职责、核心协议、数据封装与解封装。

## TCP/IP 协议栈

TCP/IP 是互联网事实上的标准协议栈，由四层构成，每一层解决特定问题。
```
// ========== TCP/IP 四层模型 ==========
//
//  ┌─────────────────────────────┐
//  │  应用层 (Application)       │
//  │  HTTP / FTP / SMTP / DNS    │
//  ├─────────────────────────────┤
//  │  传输层 (Transport)         │
//  │  TCP / UDP                  │
//  ├─────────────────────────────┤
//  │  网络层 (Internet)          │
//  │  IP / ICMP / ARP            │
//  ├─────────────────────────────┤
//  │  网络接口层 (Network Access)│
//  │  Ethernet / WiFi / PPP      │
//  └─────────────────────────────┘
//
// ========== 各层职责 ==========
// 应用层:    提供网络应用服务
// 传输层:    端到端通信, 流量控制
// 网络层:    寻址和路由, 包转发
// 接口层:    物理介质访问, 帧传输
```
## 数据封装与解封装

数据从应用层向下传递时，每层添加自己的头部信息（封装）；接收端则逆向解析（解封装）。
```
// ========== 发送端封装过程 ==========
//
// 应用层:    HTTP 请求数据
//               "GET /index.html"
//
// 传输层:    + TCP 头 (源端口:52001, 目的端口:80)
//               序列号, 确认号, 窗口大小
//            结果: TCP 段 (Segment)
//
// 网络层:    + IP 头 (源IP:192.168.1.2, 目的IP:142.250.80.4)
//               TTL, 协议号 (TCP=6)
//            结果: IP 包 (Packet)
//
// 接口层:    + Ethernet 头 (源MAC, 目的MAC)
//            结果: 帧 (Frame)
//
// 物理层:    0101010110101101... (比特流)
//
// ========== 接收端解封装 ==========
// 比特流 → 帧 (校验 FCS) → 包 (检查 IP) → 段 (检查端口) → 数据
```
> **Note**: 📥 每个数据帧的大小限制在 MTU (通常 1500 bytes)以内。如果 HTTP 请求体很大，TCP 层会先分段再交给 IP 层传输。

## 核心协议详解
```
// ========== TCP (传输控制协议) ==========
// 面向连接, 可靠传输
// 保证数据按序到达, 不丢失
// 支持流量控制和拥塞控制
// 适用: Web浏览, 邮件, 文件传输
//
// ========== UDP (用户数据报协议) ==========
// 无连接, 尽力交付
// 不保证可靠, 无重传
// 开销小, 延迟低
// 适用: 视频直播, 在线游戏, DNS
//
// ========== IP (网际协议) ==========
// 提供无连接的数据包传输
// 负责寻址和路由
// 不保证可靠 (由上层 TCP 保证)
//
// ========== ICMP (互联网控制消息协议) ==========
// 用于错误报告和诊断
// ping 使用 ICMP Echo 请求/响应
// traceroute 利用 ICMP 超时消息
//
// ========== ARP (地址解析协议) ==========
// IP 地址 → MAC 地址映射
// 同一局域网内通信必需品
// ARP 缓存: 避免每次都广播查询
```
## 端口号

传输层使用端口区分不同应用程序，一台服务器通过端口同时提供多种服务。
```
// ========== 端口号范围 ==========
// 0-1023:   知名端口 (Well-known ports)
// 1024-49151: 注册端口 (Registered ports)
// 49152-65535: 动态/私有端口 (Dynamic ports)
//
// ========== 常见端口 ==========
// HTTP:    80
// HTTPS:   443
// FTP:     21 (控制), 20 (数据)
// SSH:     22
// Telnet:  23
// SMTP:    25
// DNS:     53
// DHCP:    67 (服务器), 68 (客户端)
// MySQL:   3306
// Redis:   6379
// MongoDB: 27017
// PostgreSQL: 5432

// ========== 套接字 (Socket) ==========
// 唯一标识一个网络连接:
//   (IP 地址, 协议, 端口号)
//
// 示例:
//   客户端: (192.168.1.2, TCP, 52001)
//   服务端: (142.250.80.4, TCP, 443)
//   → 这一对构成唯一的 TCP 连接
```
