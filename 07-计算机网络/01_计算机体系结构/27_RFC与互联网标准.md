# 28_RFC与互联网标准

## 核心概念

### RFC（Request For Comments）

- **定义**：一系列以编号排定的备忘录，包含互联网相关的协议、过程、事件和概念
- **发布者**：IETF（互联网工程任务组）
- **特点**：开放、透明、可自由获取

### RFC文档的分类

1. **标准类（Standards Track）**：描述互联网标准协议
2. **最佳当前实践类（BCP）**：推荐的操作指南
3. **实验类（Experimental）**：实验性技术
4. **信息类（Informational）**：提供一般信息
5. **历史类（Historic）**：已被淘汰的协议

### 互联网标准的成熟度（当前简化版）

```
建议标准（Proposed Standard）
      |
      v
互联网标准（Internet Standard）
```

**重要**：并非所有RFC都是标准。大部分RFC只是技术文档或实验性提案。

## 原理分析

### 著名的RFC

| RFC编号 | 内容 | 意义 |
|---------|------|------|
| RFC 791 | IP协议 | 互联网的核心 |
| RFC 793 | TCP协议 | 可靠传输的基础 |
| RFC 768 | UDP协议 | 不可靠但高效的传输 |
| RFC 792 | ICMP协议 | 差错报告和诊断 |
| RFC 2460 | IPv6 | 下一代互联网协议 |

### "rough consensus and running code"

IETF的标准制定原则：
- **粗略共识**：不需要所有人同意，但需要大多数人的支持
- **运行代码**：标准必须有实际的实现证明可行性

## 直观理解

- **RFC**像维基百科的技术版：任何人都可以查看和参与
- **标准制定**像民主投票：不需要100%同意，大多数同意就行
- **running code**像"实践是检验真理的唯一标准"：纸上谈兵不算数，要能跑起来

## 协议关联

- RFC 791、793、768等是学习IP、TCP、UDP时的参考原始文档
- RFC与ISO标准的区别：RFC更灵活、更快速，但权威性稍弱
- 408考试中对RFC的考查通常是概念性的选择题
