🏢 ISP 与网络基础设施

互联网服务提供商层级、接入技术、数据中心、CDN、IXP。

ISP 层级

ISP 按规模和覆盖范围分为三个层级，形成互联网的层次结构。

// ========== ISP 层级体系 ========== // // Tier 1 (一级 ISP): // - 全球性骨干网运营商 // - 不需要向任何人购买传输服务 // - 免费对等互连 (Peering) // 例子: AT&T, Level 3, NTT, Verizon // // Tier 2 (二级 ISP): // - 区域性或国家级运营商 // - 向 Tier 1 购买传输服务 // - 与其他 Tier 2 免费对等 // 例子: 各国家级运营商 // // Tier 3 (三级 ISP): // - 本地或接入 ISP // - 纯向用户销售接入服务 // - 向 Tier 2 购买传输 // 例子: 你的家庭宽带提供商 // ========== 对等 (Peering) vs 转接 (Transit) ========== // Peering: ISP 之间免费交换流量 // Transit: 向更高级 ISP 付费获取互联网接入

接入技术

用户连接到 ISP 的常见方式。

// ========== 宽带接入技术 ========== // // DSL (电话线): // 速度: 1-100 Mbps // 特点: 利用现有电话线,距离影响速度 // // 光纤 (Fiber / FTTH): // 速度: 100 Mbps - 10 Gbps // 特点: 高速低延迟,对称带宽 // // Cable (同轴电缆): // 速度: 10-1000 Mbps // 特点: 共享带宽,高峰期慢 // // 5G 移动网络: // 速度: 50 Mbps - 2 Gbps // 特点: 低延迟(1-10ms),无线便捷 // // 卫星互联网: // 速度: 25-150 Mbps (Starlink) // 特点: 覆盖偏远地区,延迟较高 // ========== 典型延迟对比 ========== // 光纤: 1-5ms // DSL: 10-30ms // Cable: 10-40ms // 5G: 5-15ms // 卫星: 20-50ms (LEO) // 传统卫星: 500-800ms

数据中心与 CDN

// ========== 数据中心 ========== // 存放大量服务器的专门设施 // 关键基础设施: // - 电力: 双路供电 + UPS + 柴油发电机 // - 冷却: CRAC 精密空调 / 液冷 // - 网络: 多个 ISP 冗余连接 // - 安全: 生物识别 + 24/7 监控 // // 规模分级: // 企业级: 几十台服务器 // 边缘: 几百台 // 超大规模: 数万到数十万台 // // ========== CDN (内容分发网络) ========== // 全球分布的边缘服务器缓存内容 // 用户从最近的节点获取数据 // // CDN 工作流程: // 1. 用户请求 example.com/video.mp4 // 2. DNS 解析返回最近的 CDN 节点 IP // 3. 如果节点有缓存,直接返回 // 4. 如果没有缓存,回源站获取 // // 主流 CDN: Cloudflare, Akamai, Fastly // AWS CloudFront, Azure CDN
🌍 CDN 的核心是"离用户更近"——将内容缓存到全球边缘节点,大幅降低延迟。Cloudflare 运营着全球最大的 CDN 网络之一,覆盖 300+ 城市。

IXP (互联网交换中心)

// ========== IXP 的作用 ========== // 多个 ISP 和网络运营商在一个物理地点 // 交换彼此的网络流量 // // 没有 IXP 的情况: // 中国电信 → 中转 → 中国联通 (慢) // 有 IXP 的情况: // 中国电信 → [IXP] → 中国联通 (快) // // 全球主要 IXP: // AMS-IX (阿姆斯特丹): 全球最大, 10Tbps+ // DE-CIX (法兰克福): 欧洲最大 // LINX (伦敦): 英国核心 // HKIX (香港): 亚洲枢纽 // ========== 互联网的物理地图 ========== // 海底光缆主要路由: // 跨大西洋: 纽约 ↔ 伦敦/巴黎 // 跨太平洋: 洛杉矶 ↔ 东京/上海/香港 // 亚欧: 新加坡 ↔ 印度 ↔ 中东 ↔ 欧洲 // 南美: 迈阿密 ↔ 圣保罗 // // 关键海峡: // 马六甲海峡、苏伊士运河、吕宋海峡
