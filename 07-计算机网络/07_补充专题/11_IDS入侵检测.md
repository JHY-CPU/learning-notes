# 12_IDS入侵检测

## 核心概念

- **IDS（入侵检测系统）定义**：监控网络或系统中的异常行为，发现潜在攻击并报警
- **IDS vs 防火墙**：
  - 防火墙：在网络边界阻止攻击（事前防御）
  - IDS：在网络内部检测攻击（事中检测）
  - IDS不能阻止攻击，只能报警
- **IDS分类**：
  - **NIDS（网络入侵检测）**：监控网络流量，部署在关键网络节点
  - **HIDS（主机入侵检测）**：监控单台主机的系统日志、文件变化等
- **检测方法**：
  - **误用检测（Misuse Detection）**：基于已知攻击特征库匹配，类似杀毒软件
  - **异常检测（Anomaly Detection）**：建立正常行为模型，偏离则报警
- **IPS（入侵防御系统）**：IDS + 主动阻断能力，可在线阻止攻击
- **408重点**：IDS与防火墙的区别、NIDS与HIDS的区别

## 原理分析

### 误用检测 vs 异常检测

**误用检测**：
- 维护攻击特征库（Signature Database）
- 将流量与特征库匹配
- 优点：准确率高，误报少
- 缺点：只能检测已知攻击，需要持续更新特征库

**异常检测**：
- 学习正常流量模式（基线）
- 检测偏离基线的行为
- 优点：能检测未知攻击
- 缺点：误报率高，正常行为变化也可能报警

$$\text{异常度} = \frac{|\text{当前行为} - \text{基线}|}{\text{基线}}$$
超过阈值 $\theta$ 则报警。

### IDS部署位置

```
互联网 → [防火墙] → [IDS] → 内网
                      ↓
                   监控流量
```

IDS通常部署在防火墙之后的内网侧，因为：
1. 防火墙已过滤大量攻击，减轻IDS负担
2. IDS需要检测通过防火墙的高级攻击
3. IDS采用旁路部署（镜像端口），不影响网络性能

## 直观理解

- **防火墙 vs IDS类比**：
  - 防火墙 = 保安在门口检查证件，不合格不放行
  - IDS = 监控摄像头，发现问题报警但不阻止
  - IPS = 保安+监控，发现问题立即制止
- **误用检测**：像通缉犯名单，看到名单上的人就报警
- **异常检测**：像行为分析，平时大家走正门，突然有人翻墙就报警
- **记忆口诀**："防火墙挡住，IDS看着，IPS拦着"

## 协议关联

- **NIDS与各层**：
  - 可分析L2（MAC异常）、L3（IP欺骗）、L4（端口扫描）、L7（SQL注入）
- **HIDS与操作系统**：
  - 监控系统调用、文件完整性、注册表变化、日志异常
- **与防火墙联动**：IDS检测到攻击→通知防火墙更新规则→阻断攻击IP
- **408常见陷阱**：
  - "IDS可以阻止攻击"——错，IDS只检测不阻止，IPS才阻止
  - "IDS部署在防火墙前面"——一般部署在防火墙后面
  - "异常检测一定能发现新攻击"——理论上可以，但实际误报率很高
