# 43_广域网技术概述

## 核心概念

- **广域网（WAN）定义**：跨越广阔地理区域（城市、国家、全球）的计算机网络
- **广域网技术演进**：
  - **电路交换**：PSTN电话网、ISDN（建立专用物理通路）
  - **分组交换**：X.25、帧中继（Frame Relay）、ATM
  - **现代WAN**：MPLS、SD-WAN
- **X.25**：
  - 早期分组交换网络
  - 三层协议（物理、链路、分组层）
  - 逐跳差错控制和流量控制，开销大
- **帧中继（Frame Relay）**：
  - 简化X.25，取消逐跳差错控制
  - 假设链路质量高，端到端差错控制交给高层
  - 使用DLCI（数据链路连接标识符）标识虚电路
- **ATM（异步传输模式）**：
  - 固定长度信元（53字节：5字节头+48字节数据）
  - 面向连接，支持QoS
  - 使用VPI/VCI标识虚电路
- **PPP（点对点协议）**：
  - 串行链路上的点对点通信
  - 三部分：成帧、LCP（链路控制协议）、NCP（网络控制协议）
  - 支持认证（PAP/CHAP）
- **HDLC（高级数据链路控制）**：
  - 面向比特的链路层协议
  - PPP基于HDLC发展而来
- **408重点**：PPP的组成和工作过程、广域网基本概念

## 原理分析

### PPP工作流程

1. **LCP阶段**：协商链路参数（最大帧长、认证方式等）
   - LCP帧：Configure-Request → Configure-Ack
2. **认证阶段**（可选）：
   - PAP（密码认证协议）：明文传输用户名/密码
   - CHAP（挑战握手认证协议）：使用散列值，更安全
3. **NCP阶段**：协商网络层协议参数
   - IP控制协议（IPCP）：分配IP地址

### ATM信元

```
ATM信元（53字节）：
┌──────────┬─────────────────────┐
│ 5B Header │    48B Payload       │
└──────────┴─────────────────────┘
```

**为什么固定53字节**：
- 小信元：减少排队延迟（适合语音等实时业务）
- 头5字节：包含VPI/VCI（虚电路标识）、PT（载荷类型）、CLP（丢弃优先级）、HEC（头部校验）

### 广域网 vs 局域网对比

| 特性 | LAN | WAN |
|------|-----|-----|
| 范围 | 小（楼/园区） | 大（城市/国家） |
| 速率 | 高（Gbps） | 较低（Mbps-Gbps） |
| 所有者 | 自建 | ISP提供 |
| 技术 | 以太网 | PPP/帧中继/ATM/MPLS |
| 误码率 | 低 | 较高 |

## 直观理解

- **电路交换 vs 分组交换**：
  - 电路交换：像打电话——独占线路，不管说不说话都是你的
  - 分组交换：像发短信——共享线路，大家轮流用
- **ATM信元**：像固定大小的集装箱（53字节），方便快速交换
- **PPP三阶段**：先谈条件（LCP）→ 验身份（认证）→ 配地址（NCP）
- **帧中继**：X.25的"瘦身版"，去掉层层检查，速度快

## 协议关联

- **PPP与IP**：PPP承载IP包，NCP（IPCP）协商IP参数
- **PPP与认证**：PAP/CHAP在PPP认证阶段工作
- **HDLC与PPP**：PPP帧格式基于HDLC
- **ATM与IP**：ATM承载IP包（IP over ATM）
- **408考法**：PPP三阶段、HDLC帧格式、ATM信元大小
