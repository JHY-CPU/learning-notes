# 14_VPN虚拟专用网

## 核心概念

- **VPN定义**：利用公共网络（如互联网）构建的逻辑上隔离的专用网络
- **VPN核心功能**：
  - **机密性**：隧道加密，防止窃听
  - **完整性**：数据校验，防止篡改
  - **认证性**：身份验证，防止冒充
- **VPN分类**：
  - **远程访问VPN（Access VPN）**：个人设备→公司网络（如在家办公）
  - **站点到站点VPN（Site-to-Site VPN）**：分支机构↔总部（如分公司连接总部）
- **隧道技术**：将一种协议封装在另一种协议中传输
  - 例：将IP包封装在UDP包中穿越NAT
- **VPN协议**：
  - **PPTP**：点对点隧道协议（较旧，安全性差）
  - **L2TP**：第二层隧道协议（无加密，需配合IPsec）
  - **IPsec**：网络层VPN，提供完整安全服务
  - **SSL/TLS VPN**：基于HTTPS，适合远程访问
- **408重点**：VPN的概念、隧道技术、IPsec在VPN中的应用

## 原理分析

### VPN隧道封装

以IPsec隧道模式为例：
```
原始包：[IP头(内) | TCP | 数据]
封装后：[IP头(外) | ESP头 | IP头(内) | TCP | 数据 | ESP尾 | ESP认证]
```

- 外层IP头：VPN网关的源/目的IP
- 内层IP头：原始的私有IP地址
- ESP提供加密和完整性保护

### VPN工作原理

**站点到站点VPN**：
```
分支机构内网 → VPN网关A → 互联网 → VPN网关B → 总部内网
   10.1.0.0/16                         10.2.0.0/16
```

1. 分支机构PC发送数据到总部服务器（目的IP：10.2.x.x）
2. 数据到达VPN网关A
3. 网关A对数据加密，添加外层IP头（目的IP=网关B的公网IP）
4. 加密包通过互联网传输到网关B
5. 网关B解密，去掉外层IP头，转发到总部内网

### VPN安全机制

- **密钥交换**：IKE（Internet Key Exchange）协议，使用DH算法
- **数据加密**：AES-CBC / AES-GCM
- **数据完整性**：HMAC-SHA256
- **身份认证**：预共享密钥（PSK）或数字证书

## 直观理解

- **VPN类比**：在公共马路上挖一条专用隧道
  - 公共马路 = 互联网
  - 专用隧道 = VPN隧道
  - 隧道外面看不到里面运了什么（加密）
  - 隧道入口有门卫检查身份（认证）
- **远程访问VPN**：像从家里通过安全通道进入公司大楼
- **站点到站点VPN**：像两个分公司之间架设了一条私有的安全走廊
- **记忆口诀**："公网挖隧道，加密保安全，认证防冒充"

## 协议关联

- **VPN与IPsec**：IPsec是实现VPN的主流技术，在网络层提供安全
- **VPN与SSL/TLS**：SSL VPN在传输层/应用层，适合Web应用远程访问
- **VPN与NAT**：IPsec的AH与NAT不兼容（因为AH验证IP头），ESP与NAT兼容
- **VPN与隧道协议**：L2TP提供隧道，IPsec提供安全，两者常组合使用
