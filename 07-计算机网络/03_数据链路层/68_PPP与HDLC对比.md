# 70_PPP与HDLC对比

## 核心概念

- **PPP**：点对点协议，面向字节，不提供可靠传输
- **HDLC**：高级数据链路控制，面向比特，可提供可靠传输
- **两者都是点对点链路的常用协议**

### 全面对比表

| 特性 | PPP | HDLC |
|------|-----|------|
| 标准 | IETF (RFC 1661) | ISO 3309 |
| 传输方式 | 面向字节 | 面向比特 |
| 透明传输 | 字节填充 | 零比特填充 |
| 可靠传输 | 不提供 | 可提供（I帧序号） |
| 帧类型 | 数据帧 | I帧/S帧/U帧 |
| 链路建立 | LCP协商 | 命令/响应 |
| 认证 | 支持（PAP/CHAP） | 不支持 |
| 多协议支持 | 通过协议字段 | 通过控制字段 |
| 多点链路 | 不支持 | 支持 |
| 地址字段 | 固定0xFF | 可变（从站地址） |
| 应用 | 拨号上网、广域网 | 早期广域网 |

### 相同点

1. 都支持点对点链路
2. 都使用标志字段定界（0x7E）
3. 都有FCS校验
4. 都支持多种网络层协议

### 为什么PPP更流行？

1. **简单**：不提供可靠传输，减少开销
2. **灵活**：支持LCP协商和认证
3. **兼容**：支持多种网络层协议（NCP）
4. **实际需要**：端到端可靠传输由TCP提供，数据链路层不需要

### 考试重点

- PPP和HDLC的对比是**经典考题**
- 面向字节 vs 面向比特
- 是否提供可靠传输

## 原理分析

### 帧格式对比

**PPP帧**：
```
| Flag(1) | Addr(1) | Ctrl(1) | Proto(2) | Info | FCS(2/4) | Flag(1) |
```

**HDLC帧**：
```
| Flag(1) | Addr(≥1) | Ctrl(1/2) | Info | FCS(2/4) | Flag(1) |
```

区别：
- PPP有Protocol字段，HDLC在Control字段中编码
- PPP的Address固定为0xFF，HDLC的Address可变
- PPP的Control固定为0x03，HDLC的Control包含序号

### 透明传输对比

**PPP字节填充**：0x7E → 0x7D 0x5E

**HDLC零比特填充**：01111111 → 011111011（插入0）

字节填充增加1字节/特殊字符，零比特填充增加1bit/连续5个1。

## 直观理解

- **PPP = 简化版HDLC**：保留了核心功能（成帧、检错），去掉了复杂功能（可靠传输、多点链路）
- **为什么HDLC支持可靠传输而PPP不支持**：HDLC时代TCP还不普及，需要在链路层保证可靠性；现在TCP普及了，链路层不需要了

## 协议关联

- PPP帧格式源自HDLC帧格式
- PPPoE = PPP封装在以太网帧中
- HDLC是许多协议的"祖先"（帧中继、X.25等）
- 408考试中两者对比是**必考内容**
