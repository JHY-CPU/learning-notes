# 82_数据链路层协议对比

## 核心概念

- **数据链路层主要协议**：以太网(802.3)、PPP、HDLC、CSMA/CD、CSMA/CA
- **协议对比是408高频考点**
- **按类别分**：点对点协议 vs 广播协议

### 协议分类

```
数据链路层协议
├── 点对点协议
│   ├── PPP（面向字节，不可靠）
│   └── HDLC（面向比特，可靠）
│
└── 广播/多点协议
    ├── 以太网 CSMA/CD（有线）
    ├── WiFi CSMA/CA（无线）
    ├── 令牌环（已淘汰）
    └── FDDI（已淘汰）
```

### 全面对比表

| 特性 | 以太网 | PPP | HDLC | WiFi |
|------|--------|-----|------|------|
| 拓扑 | 广播 | 点对点 | 点对点/多点 | 广播 |
| 传输方式 | — | 面向字节 | 面向比特 | — |
| MAC协议 | CSMA/CD | 无 | 无 | CSMA/CA |
| 透明传输 | 违规编码 | 字节填充 | 零比特填充 | — |
| 可靠传输 | 不提供 | 不提供 | 可提供 | 需要ACK |
| 流量控制 | 无 | 无 | 有 | 无 |
| 差错检测 | CRC-32 | CRC | CRC | CRC |
| MAC地址 | 6字节 | 无 | 可变 | 6字节 |
| 最小帧长 | 64字节 | 无要求 | 无要求 | — |

### 帧格式对比

| 协议 | 定界符 | 地址 | 控制 | 协议 | FCS |
|------|--------|------|------|------|-----|
| 以太网 | 前导码+SFD | 6+6字节 | 无 | 2字节(类型) | 4字节 |
| PPP | 0x7E | 1字节(0xFF) | 1字节(0x03) | 2字节 | 2/4字节 |
| HDLC | 0x7E | ≥1字节 | 1/2字节 | — | 2/4字节 |

### 考试重点

- 各协议的透明传输方法
- 是否提供可靠传输
- 帧格式的异同

## 原理分析

### 点对点 vs 广播的根本区别

**点对点**（PPP/HDLC）：
- 只有两个端点，不存在"谁先发"的问题
- 不需要MAC地址
- 不需要CSMA/CD或CSMA/CA

**广播**（以太网/WiFi）：
- 多个站点共享信道
- 需要MAC地址寻址
- 需要介质访问控制协议

### 面向字节 vs 面向比特

| 特性 | 面向字节(PPP) | 面向比特(HDLC) |
|------|-------------|---------------|
| 数据单位 | 字节 | 比特 |
| 透明传输 | 字节填充 | 零比特填充 |
| 帧长约束 | 字节对齐 | 比特对齐 |
| 实现 | 简单 | 较复杂 |

## 直观理解

- **协议对比就像比较"不同品牌的快递服务"**：有的快但不保证送达（PPP），有的慢但保送达（HDLC），有的适合大众快递（以太网）

## 协议关联

- 以太网是局域网的事实标准
- PPP是广域网点对点链路的首选
- 协议对比题通常在选择题中出现
