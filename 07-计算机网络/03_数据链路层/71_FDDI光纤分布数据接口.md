# 73_FDDI光纤分布数据接口

## 核心概念

- **FDDI（Fiber Distributed Data Interface）**：光纤分布数据接口，IEEE 802.8标准
- **核心特点**：使用光纤、双环拓扑、令牌传递协议、100Mbps
- **双环冗余**：主环+备用环，主环故障时自动切换到备用环

### FDDI的基本参数

| 参数 | 值 |
|------|-----|
| 速率 | 100 Mbps |
| 介质 | 光纤 |
| 拓扑 | 双环 |
| MAC协议 | 令牌传递 |
| 最大环长 | 200 km |
| 最大站点数 | 500 |
| 编码 | 4B/5B + NRZI |

### FDDI的双环结构

```
主环(顺时针) ──── A站 ──── B站 ──── C站 ────
备用环(逆时针) ── A站 ──── B站 ──── C站 ────
```

- **正常情况**：数据在主环上传输
- **链路故障**：故障点两端的站点将双环"环绕"，形成单环继续工作
- **站点故障**：自动绕过故障站点

### FDDI vs 令牌环网

| 特性 | FDDI | 令牌环网(802.5) |
|------|------|----------------|
| 速率 | 100 Mbps | 4/16 Mbps |
| 介质 | 光纤 | 双绞线 |
| 拓扑 | 双环 | 单环 |
| 冗余 | 有（双环） | 无 |
| 令牌释放 | 早期令牌释放 | 完全帧回收 |

### FDDI的令牌释放

- **早期令牌释放（Early Token Release）**：发送站发出数据帧后立即释放令牌，不必等数据帧回来
- 提高了环的利用率（可以同时有多个帧在环上）

### 考试重点

- FDDI的基本概念（100Mbps、光纤、双环、令牌传递）
- FDDI的冗余机制
- FDDI已被以太网取代

## 原理分析

### 双环容错

**单点链路故障**：
```
正常：A → B → C → D → A（主环）
故障后：A → B ← C ← D → A（B-C间断开，双环在B和C处环绕）
```

**站点故障**：
```
正常：A → B → C → D → A
B故障后：A → C → D → A（B被旁路，光信号通过B的旁路中继器）
```

### 4B/5B编码

- 每4位数据编码为5位符号
- 编码效率：80%
- 确保信号有足够的跳变用于时钟恢复
- 实际线路速率 = 100 Mbps / 0.8 = 125 Mbaud

## 直观理解

- **FDDI的双环就像"两条平行公路"**：平时只用一条，路塌了就切换到另一条
- **FDDI曾经是"骨干网之王"**：在千兆以太网出现之前，100Mbps的FDDI是高速骨干网的首选

## 协议关联

- FDDI作为骨干网已被千兆/万兆以太网取代
- FDDI的双环冗余思想在SONET/SDH中得到继承
- 408考试中FDDI一般只考基本概念
- FDDI和令牌环网都使用令牌传递协议
