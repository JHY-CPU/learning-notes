# 69_HDLC_概念_考试

## 核心概念

- **408考法**：帧类型判断、控制字段分析、与PPP对比
- **必记**：I帧(第1位=0)、S帧(前2位=10)、U帧(前2位=11)
- **HDLC全称**：High-Level Data Link Control，高级数据链路控制
- **特点**：面向比特、可靠传输、零比特填充、ISO标准

### HDLC的帧类型

| 帧类型 | 名称 | 控制字段特征 | 用途 |
|--------|------|-------------|------|
| **I帧** | 信息帧 | 第1位=0 | 携带用户数据，有序号和确认 |
| **S帧** | 监控帧 | 前2位=10 | 流量控制和差错控制 |
| **U帧** | 无编号帧 | 前2位=11 | 链路建立和释放 |

### S帧的子类型

| 类型字段 | S帧名称 | 功能 |
|---------|--------|------|
| 00 | RR（接收准备好） | 确认已收到的帧，准备接收更多 |
| 01 | REJ（拒绝） | 否认帧，要求重传（GBN方式） |
| 10 | RNR（接收未准备好） | 要求发送方暂停 |
| 11 | SREJ（选择拒绝） | 只要求重传指定帧（SR方式） |

### HDLC帧格式

```
| 标志 | 地址 | 控制 | 信息 | FCS | 标志 |
| 7E   | 1B   | 1-2B | 可变 | 2B  | 7E   |
```

### 零比特填充

- 发送时：数据中每出现连续5个1，就在其后插入1个0
- 接收时：每检测到连续5个1，就删除其后的0
- 这保证了帧中不会出现标志字段0x7E（01111110）

## 原理分析

### 真题1（帧类型判断）

**题目**：HDLC帧的控制字段为 01010101，该帧是什么类型？

**解答**：控制字段第1位 = 0，所以是**I帧（信息帧）**。N(S)=10101（十进制21），表示发送序号为21。

### 真题2（S帧判断）

**题目**：HDLC帧的控制字段为 10001111，该S帧的类型是什么？

**解答**：
- 前2位 = 10 → S帧
- Type字段（位4-5）= 00 → **RR帧**（接收准备好）
- N(R) = 0111 = 7，表示确认序号7及之前的帧

### 真题3（HDLC vs PPP）

**题目**：以下关于HDLC和PPP的说法，正确的是（）

A. HDLC和PPP都面向字节
B. HDLC提供可靠传输，PPP不提供
C. HDLC和PPP都使用字节填充
D. HDLC和PPP都只支持点对点链路

**答案**：B

- A错：HDLC面向比特，PPP面向字节
- B对：HDLC有I帧序号和确认机制，PPP没有
- C错：HDLC用零比特填充，PPP用字节填充
- D错：HDLC支持非平衡（多点）和平衡（点对点）配置

### 真题4（可靠传输）

**题目**：HDLC中实现可靠传输的机制是什么？

**答案**：
1. **序号**：I帧的N(S)字段标识发送序号（模8）
2. **确认**：N(R)字段实现累积确认（确认N(R)之前的所有帧）
3. **超时重传**：超时后重传未确认帧
4. **S帧**：RR确认、REJ拒绝（GBN）、SREJ选择拒绝（SR）

### 真题5（透明传输）

**题目**：HDLC使用什么透明传输方法？

**答案**：**零比特填充**。数据中每出现连续5个1就插入1个0，接收方每检测到5个1后删除紧跟的0。例如：数据01111110 → 发送011111010 → 接收还原为01111110。

### 真题6（工作模式）

**题目**：HDLC支持哪几种工作模式？

**答案**：
- **NRM（正常响应模式）**：非平衡配置，主站轮询从站
- **ARM（异步响应模式）**：非平衡配置，从站可主动发送
- **ABM（异步平衡模式）**：平衡配置，双方地位平等（最常用）

## 直观理解

- **HDLC是"严谨的协议"**：有完整的帧类型体系（I/S/U）、可靠的传输机制（序号+确认）
- **PPP是"简化的HDLC"**：去掉可靠传输，保留成帧和差错检测
- **HDLC像挂号信**：有编号、有回执、保证送达
- **PPP像普通信件**：有地址、有包装，但不保证送达

### 408易错对比表

| 特性 | HDLC | PPP |
|------|------|-----|
| 传输单位 | 面向比特 | 面向字节 |
| 可靠传输 | 是（序号+确认） | 否 |
| 透明传输 | 零比特填充 | 字节填充 |
| 链路类型 | 点对点+多点 | 仅点对点 |
| 帧标志 | 0x7E | 0x7E |
| MAC地址 | 不需要 | 不需要 |

## 协议关联

- HDLC是ISO标准（ISO 3309），PPP是IETF标准（RFC 1661）
- HDLC在实际广域网中已很少使用，但408考试仍然重要
- HDLC的帧格式影响了帧中继、ATM等技术
- HDLC的I帧机制是理解数据链路层可靠传输的基础
- S帧的RR/REJ与TCP的ACK/重复ACK有相似之处
- 408常考：帧类型判断（看控制字段前几位）、HDLC与PPP的对比
