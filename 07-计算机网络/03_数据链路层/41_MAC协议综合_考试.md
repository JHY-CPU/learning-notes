# 42_MAC协议综合_考试

## 核心概念

- **MAC协议全景**：ALOHA → CSMA → CSMA/CD → CSMA/CA
- **演进方向**：从"盲发"到"先听后说"到"边说边听"到"避免冲突"
- **408考点**：各协议的对比、最大吞吐率、适用场景

### MAC协议演进

```
ALOHA(想发就发) → 时隙ALOHA(定时发) → CSMA(先听后说)
                                        ↓
                    CSMA/CD(边说边听) ←→ CSMA/CA(避免冲突)
                    有线以太网            无线WiFi
```

### 各协议最大吞吐率

| 协议 | 最大吞吐率 |
|------|-----------|
| 纯ALOHA | 18.4% |
| 时隙ALOHA | 36.8% |
| 非坚持CSMA | ~80-90% |
| 1-坚持CSMA | ~53% |
| CSMA/CD | ~90%+ |

## 原理分析

### 真题1（协议演进）

**题目**：从ALOHA到CSMA/CD，MAC协议的核心改进是什么？

**答案**：
1. ALOHA → CSMA：增加载波侦听（先听后说），减少盲目发送
2. CSMA → CSMA/CD：增加冲突检测（边说边听），尽早发现冲突并停止浪费

### 真题2（协议选择）

**题目**：某场景需要选择MAC协议：
(1) 有线局域网
(2) 无线局域网
(3) 卫星通信

**答案**：
(1) CSMA/CD（以太网）
(2) CSMA/CA（WiFi）
(3) 时隙ALOHA或预约协议

### 真题3（综合对比）

**题目**：ALOHA、CSMA/CD、CSMA/CA三个协议中：
(1) 哪个不需要载波侦听？
(2) 哪个需要ACK确认？
(3) 哪个有最小帧长要求？
(4) 哪个能检测冲突？

**答案**：
(1) ALOHA
(2) CSMA/CA
(3) CSMA/CD
(4) CSMA/CD

### 真题4（效率分析）

**题目**：为什么CSMA/CD的效率高于ALOHA？

**答案**：
1. 载波侦听减少了"盲发"，降低了冲突概率
2. 冲突检测使节点在冲突发生后立即停止，减少信道浪费
3. 冲突窗口从 $T$ 或 $2T$ 缩小到 $2\tau$（通常远小于帧发送时间）

## 直观理解

- **MAC协议的进步**就像交通管理的进步：从"随意穿马路"（ALOHA）→"看红绿灯"（CSMA）→"看灯+交警指挥"（CSMA/CD）

## 协议关联

- MAC协议是数据链路层下半部分（MAC子层）的核心
- 与以太网帧结构、网桥/交换机的转发原理紧密相关
- 408综合题可能要求计算"给定场景选择协议后的效率"
