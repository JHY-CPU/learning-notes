# 53_网桥工作原理

## 核心概念

- **网桥（Bridge）**：工作在数据链路层，根据MAC地址转发帧，连接两个或多个局域网段
- **核心功能**：帧的存储转发、MAC地址学习、过滤/转发决策
- **隔离冲突域**：每个网段是独立的冲突域

### 网桥的工作原理

1. **学习**：从帧的源MAC地址学习端口映射
2. **转发**：查表决定从哪个端口转发
3. **过滤**：源和目的在同一端口时丢弃
4. **泛洪**：目的未知时向所有端口广播

### 转发表（MAC地址表）

| MAC地址 | 端口 | 时间戳 |
|---------|------|--------|
| 00:1A:2B:3C:4D:5E | 端口1 | T1 |
| 00:11:22:33:44:55 | 端口2 | T2 |

### 转发决策

| 条件 | 动作 |
|------|------|
| 目的MAC在表中，且对应端口 ≠ 入端口 | 从对应端口转发 |
| 目的MAC在表中，且对应端口 = 入端口 | 过滤（丢弃） |
| 目的MAC不在表中 | 泛洪（除入端口外所有端口） |
| 目的MAC是广播地址 | 泛洪 |

### 考试重点

- 网桥的工作流程（学习+转发+过滤+泛洪）
- 网桥隔离冲突域但不隔离广播域
- 网桥 vs 路由器的区别

## 原理分析

### 网桥的工作流程

```
收到帧 → 读取源MAC → 更新转发表 → 读取目的MAC → 查表
                                            ↓
                              在表中？——→ 同端口？——→ 丢弃
                              ↓ 是         ↓ 不是
                         不同端口？    从对应端口转发
                              ↓ 否（不在表中）
                           泛洪
```

### 转发表的老化机制

- 每个表项有**生存时间（TTL）**，通常300秒
- 超过TTL未更新的表项被删除
- 目的：适应网络拓扑变化

### 网桥的优缺点

**优点**：
- 过滤通信量，增大吞吐量
- 扩大网络地理范围
- 可连接不同物理层的网络
- 提高可靠性（某网段故障不影响其他）

**缺点**：
- 存储转发有时延
- MAC地址表容量有限
- 不能隔离广播域（广播风暴风险）
- 适用用户数有限（性能随网段增多下降）

## 直观理解

- **网桥就像"智能门卫"**：记住每个住户住哪栋楼（MAC学习），快递来了查地址转送，地址不认识就群发通知

## 协议关联

- 网桥是交换机的前身（交换机 = 多端口网桥）
- 透明网桥和源路由网桥是两种主要类型
- 生成树协议（STP）解决了网桥环路问题
- 网桥工作在数据链路层的MAC子层
