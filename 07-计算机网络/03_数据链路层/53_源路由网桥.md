# 55_源路由网桥

## 核心概念

- **源路由网桥（Source Route Bridge）**：由发送方（源站）决定帧的路由路径
- **与透明网桥的区别**：透明网桥由网桥自己决定转发路径，源路由由源站指定
- **主要用于令牌环网（IEEE 802.5）**

### 源路由的工作原理

1. 源站通过**路由发现**确定到目的站的路径
2. 将路径信息**封装在帧的头部**
3. 沿途网桥根据帧头中的路由信息转发
4. 不需要网桥维护转发表

### 路由发现过程

1. 源站发送**路由发现帧**（广播）
2. 帧到达目的站，目的站记录路径
3. 目的站返回**路由响应帧**，携带路径信息
4. 源站收到响应，获知可达路径
5. 源站选择一条路径，写入后续数据帧

### 路由信息字段（RIF）

- 嵌入在帧中
- 包含路径上经过的网桥和环的编号
- 网桥根据RIF决定从哪个端口转发

### 源路由 vs 透明路由

| 特性 | 源路由 | 透明路由 |
|------|--------|---------|
| 路由决策 | 源站 | 网桥 |
| 网桥转发表 | 不需要 | 需要 |
| 即插即用 | 否 | 是 |
| 帧开销 | 大（含路由信息） | 小 |
| 主要应用 | 令牌环网 | 以太网 |
| 环路处理 | 由源站选择路径 | STP |

## 原理分析

### 路由发现示例

```
环1 -- 网桥A -- 环2 -- 网桥B -- 环3
(A在环1)                      (B在环3)

A发路由发现帧（广播）:
  路径1: 环1 → 网桥A → 环2 → 网桥B → 环3 → B
  路径2: 环1 → 网桥C → 环4 → 网桥D → 环3 → B

B返回响应，A选择最短路径写入帧头
```

### 源路由的帧格式（简略）

```
| 帧控制 | 目的地址 | 源地址 | 路由信息字段(RIF) | 数据 | FCS |
```

RIF包含：路由类型 + 路径长度 + 路径上的环/网桥编号

## 直观理解

- **源路由就像自驾导航**：司机自己查地图决定路线（源站发现路径），路上的收费站（网桥）只看路线图放行
- **透明路由就像出租车**：司机（网桥）知道路怎么走，乘客（站点）不用管

## 协议关联

- 源路由网桥随令牌环网一起被淘汰，现在很少使用
- 透明网桥（以太网）成为主流
- 但"源路由"的思想在MPLS等技术中有所体现
