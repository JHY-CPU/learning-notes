# 67_HDLC协议

## 核心概念

- **HDLC（High-level Data Link Control）**：高级数据链路控制，面向比特的协议
- **ISO标准**：ISO 3309/ISO 4335
- **核心特点**：使用零比特填充、支持可靠传输、帧结构灵活

### HDLC的站类型

| 站类型 | 说明 |
|--------|------|
| 主站（Primary） | 发送命令帧，控制链路 |
| 从站（Secondary） | 接收命令，发送响应 |
| 复合站（Combined） | 既能发命令也能发响应 |

### HDLC的链路配置

| 配置 | 说明 | 模式 |
|------|------|------|
| 非平衡配置 | 1个主站 + 多个从站 | 正常响应模式NRM |
| 平衡配置 | 2个复合站 | 异步平衡模式ABM |

### HDLC的帧类型

| 类型 | 编号 | 用途 |
|------|------|------|
| 信息帧（I帧） | 编号 | 传输数据 |
| 监控帧（S帧） | 编号 | 流量控制和差错控制 |
| 无编号帧（U帧） | 无编号 | 链路控制 |

### HDLC帧格式

```
| 标志(1B) | 地址(≥1B) | 控制(1-2B) | 信息(可变) | FCS(2/4B) | 标志(1B) |
```

- **标志**：01111110（0x7E）
- **地址**：从站地址（非平衡）或对方站地址（平衡）
- **控制**：标识帧类型和序号
- **FCS**：CRC校验

### HDLC的透明传输

- 使用**零比特填充**
- 数据中每5个连续1后插入0
- 接收方每5个连续1后删除0

### HDLC vs PPP

| 特性 | HDLC | PPP |
|------|------|-----|
| 传输方式 | 面向比特 | 面向字节 |
| 透明传输 | 零比特填充 | 字节填充 |
| 可靠传输 | 支持（可选） | 不支持 |
| 多点链路 | 支持 | 不支持（仅点对点） |
| 地址字段 | 可变长度 | 固定1字节(0xFF) |

### 考试重点

- HDLC的帧类型（I帧、S帧、U帧）
- HDLC与PPP的区别
- 零比特填充与字节填充的区别

## 原理分析

### 控制字段格式

**I帧**（信息帧）：控制字段第1位=0

```
| 0 | N(S) | P/F | N(R) |
| 1位| 3位  | 1位 | 3位  |
```

- $N(S)$：发送序号
- $N(R)$：期望接收的下一个帧序号（累积确认）
- $P/F$：轮询/终止位

**S帧**（监控帧）：控制字段前2位=10

```
| 1 | 0 | Type | P/F | N(R) |
| 1位| 1位| 2位 | 1位 | 3位  |
```

Type字段：
- 00：RR（接收准备好）
- 01：REJ（拒绝，GBN重传）
- 10：RNR（接收未准备好）
- 11：SREJ（选择拒绝，SR重传）

**U帧**（无编号帧）：控制字段前2位=11

```
| 1 | 1 | M1 | P/F | M2 |
| 1位| 1位| 2位 | 1位 | 3位  |
```

用于链路建立、断开等控制功能。

## 直观理解

- **HDLC就像"正式的外交协议"**：有严格的命令/响应体系（主站/从站），有完整的确认机制
- **PPP就像"简化的日常对话"**：更简单直接，不需要那么严格的控制

## 协议关联

- HDLC是许多数据链路层协议的基础（PPP、帧中继等）
- HDLC的可靠传输功能在实际中很少使用（靠TCP实现端到端可靠传输）
- HDLC支持多点链路，PPP只支持点对点
- 408考试中HDLC常与PPP对比
