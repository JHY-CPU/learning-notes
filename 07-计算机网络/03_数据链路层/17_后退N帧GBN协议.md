# 18_后退N帧GBN协议

## 核心概念

- **GBN（Go-Back-N）**：后退N帧协议，发送窗口 > 1，接收窗口 = 1
- **核心思想**：允许发送方连续发送多个帧，出错时从出错帧开始**全部重传**
- **累积确认**：ACKn表示序号n及之前的帧都已正确接收

### GBN的关键参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 发送窗口 $W_s$ | $2^n - 1$（最大值） | n为序号位数 |
| 接收窗口 $W_r$ | 1 | 只能按序接收 |
| 确认方式 | 累积确认 | ACKn = n及之前都收到了 |
| 超时重传 | 重传所有未确认帧 | 从最早的未确认帧开始 |

### GBN的工作流程

```
发送方                              接收方
  │                                    │
  │ ──帧0──>                           │  期望帧0 ✓
  │ ──帧1──>                           │  期望帧1 ✓
  │ ──帧2──>  丢失！                   │  期望帧2 ✗
  │ ──帧3──>  乱序到达                 │  期望帧2，丢弃帧3
  │ ──帧4──>  乱序到达                 │  期望帧2，丢弃帧4
  │                                    │
  │ <──ACK1───                         │
  │                                    │
  │  超时！重传帧2,3,4                 │
  │ ──帧2──>                           │  期望帧2 ✓
  │ ──帧3──>                           │  期望帧3 ✓
  │ ──帧4──>                           │  期望帧4 ✓
  │                                    │
  │ <──ACK4───                         │
```

### GBN的发送方事件

1. **收到上层数据**：检查窗口是否满，未满则发送帧
2. **收到ACKn（累积确认）**：将窗口基序号推进到n+1
3. **超时**：重传所有已发送但未确认的帧

### GBN的接收方行为

- 只接收**按序到达**的帧
- 失序的帧直接**丢弃**（不缓存）
- 每次收到正确的帧，发送ACK

### 考试重点

- GBN窗口大小 $W_s \leq 2^n - 1$ 是**必考内容**
- 累积确认的理解
- 出错时重传所有未确认帧（效率低的来源）

## 原理分析

### GBN发送方伪代码

```
窗口基序号 base = 0
下一个序号 nextseqnum = 0

while(true):
    收到数据:
        if nextseqnum < base + Ws:
            发送帧(nextseqnum)
            nextseqnum++
    
    收到ACK(n):
        base = n + 1  // 累积确认，推进窗口
        if base == nextseqnum:
            停止计时器
        else:
            启动计时器
    
    超时:
        重传帧base到nextseqnum-1的所有帧
        启动计时器
```

### GBN的信道利用率

$$U_{GBN} = \frac{W_s \times T_f}{T_f + 2T_p} \quad (\text{当 } W_s \times T_f \leq T_f + 2T_p)$$

当 $W_s$ 足够大时，$U = 1$。

### 例题

**题目**：序号用3位表示，GBN的最大发送窗口是多少？

**解答**：$W_s = 2^3 - 1 = 7$

**题目**：序号用3位表示，GBN窗口=7，帧长1000bit，数据率1Mbps，单向传播时延20ms，求信道利用率。

**解答**：

$T_f = 1000/10^6 = 1$ ms

$U = \frac{7 \times 1}{1 + 2 \times 20} = \frac{7}{41} \approx 17.07\%$？不对，当窗口内帧的发送时间超过了RTT，就满了。

$7 \times 1 = 7$ ms $< 41$ ms，所以：

$$U = \frac{7 \times 1}{1 + 2 \times 20} = \frac{7}{41} \approx 17.07\%$$

如果 $W_s = 42$（理论上），$U = \frac{42}{41}$? 受限于实际最大7。

## 直观理解

- **GBN就像流水线上出错**：发现一个产品有问题，后面的产品全部打回重做
- **累积确认**就像"我收到了第5份，说明前5份都没问题"——不用逐个确认
- **丢弃失序帧**就像"快递到了但编号不对，直接拒收"——简单粗暴

## 协议关联

- GBN vs SR：GBN丢弃失序帧，SR缓存失序帧
- GBN vs 停等：GBN是停等的推广，窗口从1扩大到N
- GBN适合**误码率低**的信道（出错少，重传概率低）
- 当信道误码率高时，SR更优（因为GBN重传太多）
