# 01_数据链路层概述

## 核心概念

- **数据链路层（Data Link Layer）**：OSI参考模型的**第2层**，TCP/IP模型中属于**网络接口层**的一部分
- **主要功能**：将物理层提供的**不可靠**的物理链路，改造为**逻辑上无差错**的数据链路
- **服务对象**：为**网络层**提供服务，将网络层的IP数据报封装成**帧**进行传输

### 数据链路层的核心任务

1. **成帧（Framing）**：将比特流划分成帧，添加帧头帧尾
2. **差错控制（Error Control）**：检测或纠正传输中的比特差错
3. **流量控制（Flow Control）**：控制发送方速率，防止接收方缓冲区溢出
4. **访问控制（Medium Access Control）**：多个节点共享信道时，决定谁先发送

### 数据链路层的三个重要问题

| 问题 | 说明 | 对应机制 |
|------|------|----------|
| 封装成帧 | 帧定界 | 字符填充、比特填充 |
| 透明传输 | 数据中含特殊字符不被误解 | 转义字符/零比特填充 |
| 差错检测 | 检测传输错误 | CRC校验 |

### 常见协议

- **PPP（点对点协议）**：最常用的数据链路层协议，用于拨号上网
- **HDLC（高级数据链路控制）**：面向比特的协议
- **CSMA/CD**：以太网使用的介质访问控制协议
- **CSMA/CA**：无线局域网使用的介质访问控制协议

### 考试重点

- 数据链路层**不负责**差错纠正（除了海明码），只负责**差错检测**
- 流量控制和差错控制是数据链路层的**两大核心功能**
- 408考试中，CRC计算、滑动窗口、信道利用率是**高频考点**
- 区分"链路"（物理概念）和"数据链路"（逻辑概念）

### 常见混淆点

- 数据链路层的"差错控制"主要是**检错**，不是纠错
- 流量控制在数据链路层和传输层都有，但机制不同
- MAC子层（介质访问控制）属于数据链路层的**下半部分**

## 原理分析

### 数据链路层在协议栈中的位置

```

```

### 数据链路层的分层

数据链路层在IEEE 802标准中被划分为两个子层：

- **LLC（逻辑链路控制）子层**：与硬件无关，提供流量控制和差错控制
- **MAC（介质访问控制）子层**：与硬件相关，负责介质访问控制

$$\text{数据链路层} = \text{LLC子层} + \text{MAC子层}$$

### 服务类型

| 服务类型 | 确认 | 连接 | 典型应用 |
|----------|------|------|----------|
| 无确认无连接服务 | 无 | 无 | 以太网 |
| 有确认无连接服务 | 有 | 无 | WiFi |
| 有确认面向连接服务 | 有 | 有 | HDLC |

## 直观理解

- 数据链路层就像**快递公司的打包员**：物理层负责运输（公路），数据链路层负责把货物打包成箱子（成帧），贴上标签（帧头），检查包裹是否破损（差错检测）
- 如果把通信比作**寄信**：物理层是邮递员负责送信，数据链路层负责把信装进信封、写地址、检查信封是否被拆过
- **成帧**就像把一部长篇小说分成多个章节，每章有标题和结尾，方便读者定位

## 协议关联

### 与上层的关系

- 接收**网络层**的IP数据报，封装成帧后交给物理层
- 数据链路层的帧是IP数据报的"容器"

### 与下层的关系

- 使用**物理层**提供的比特流传输服务
- 物理层只管传比特，数据链路层负责组织和检测

### 与传输层的对比

| 特征 | 数据链路层 | 传输层 |
|------|-----------|--------|
| 流量控制范围 | 相邻节点间 | 端到端 |
| 差错控制 | CRC检错 | 校验和检错+重传 |
| 面向 | 帧 | 报文段 |

### 常见考题组合

- 数据链路层 + 网络层 + 传输层的**流量控制对比**
- 数据链路层帧与IP数据报的**封装关系**
- CRC校验与校验和的**区别与联系**
