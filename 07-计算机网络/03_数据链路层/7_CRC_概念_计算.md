# 08_CRC_概念_计算

## 核心概念

- **CRC计算核心**：模2除法（异或运算），不进位不借位
- **生成多项式的阶数**：$G(x)$ 的最高次幂 = 余数的位数 = 添加0的个数
- **帧校验序列（FCS）**：模2除法的余数，附加在信息码后面

### CRC计算的关键参数

| 参数 | 确定方法 | 示例 |
|------|----------|------|
| 添加0的个数 | $G(x)$的最高次幂（位数-1） | $G(x)$有4位，加3个0 |
| FCS长度 | 等于添加0的个数 | 3位 |
| 除数 | $G(x)$的二进制表示 | $x^3+x^2+1$ → `1101` |

### 常见生成多项式

| 标准 | 生成多项式 | 二进制 | FCS长度 |
|------|-----------|--------|---------|
| CRC-4 | $x^4+x+1$ | 10011 | 4位 |
| CRC-8 | $x^8+x^2+x+1$ | 100000111 | 8位 |
| CRC-12 | $x^{12}+x^{11}+x^3+x^2+x+1$ | 1100000001111 | 12位 |
| CRC-16 | $x^{16}+x^{15}+x^2+1$ | 11000000000000101 | 16位 |
| CRC-CCITT | $x^{16}+x^{12}+x^5+1$ | 10001000000100001 | 16位 |
| CRC-32 | 多项式较长 | — | 32位 |

## 原理分析

### 完整计算示例

**例题1**：数据 $M = 1101011011$，$G(x) = x^4 + x + 1$，求FCS和发送帧。

**Step1**：写出除数
$$G(x) = x^4 + x + 1 \Rightarrow \text{二进制} = 10011$$

**Step2**：在数据后添加4个0
$$M' = 1101011011\mathbf{0000}$$

**Step3**：模2除法

```
              1100001100
        ___________________
10011 | 11010110110000
        10011
        -----
         10011
         10011
         -----
          00001
          00000
          -----
           00011
           00000
           -----
            00110
            00000
            -----
             01101
             00000
             -----
              11011
              10011
              -----
               10000
               10011
               -----
                0011  ← 余数 R = 0011
```

**Step4**：发送帧
$$\text{发送帧} = 1101011011\mathbf{0011}$$

### 验证过程

用发送帧除以 $G(x)$：

$$11010110110011 \div 10011 = 1100001100 \cdots 0000$$

余数为0，验证通过！

### 例题2：短数据

**题目**：数据 $M = 10110$，$G(x) = x^3 + x^2 + 1$，求CRC。

1. $G(x) = x^3 + x^2 + 1 \Rightarrow 1101$
2. 添加3个0：$M' = 10110000$
3. 模2除法：

```
         11010
    __________
1101|10110000
     1101
     ----
      1100
      1101
      ----
       0010
       0000
       ----
        0100
        0000
        ----
         1000
         1101
         ----
          101 ← 余数 R = 101
```

4. 发送帧 = $10110101$

### 常见错误

1. **忘记添加0**：必须在信息码后添加r个0再做除法
2. **模2除法做错**：不能借位、不能进位，只有异或
3. **余数位数不对**：余数必须是r位，不足前面补0
4. **混淆生成多项式与除数**：$G(x)$的二进制表示就是除数

## 直观理解

- CRC计算就像**带余数的除法验算**：把数据当成一个大数，除以一个约定的数，余数就是校验码
- **模2除法**就像"不进位加法的除法版本"：只做XOR，不管大小
- **为什么不用普通的除法**？因为模2除法可以用**移位寄存器硬件**高速实现

## 协议关联

### CRC在帧结构中的位置

```
以太网帧：| 目的地址 | 源地址 | 类型 | 数据(46-1500B) | FCS(4B) |
                                        ↑CRC校验范围↑    ↑CRC结果↑
```

### CRC校验范围

- **以太网**：目的地址 + 源地址 + 类型 + 数据（不包括前导码）
- **PPP**：地址 + 控制 + 协议 + 信息（FCS在帧尾）

### 408真题常见问法

1. "给定数据和生成多项式，求FCS" → 做模2除法
2. "收到的帧用CRC校验，判断是否有错" → 用含FCS的帧除以G(x)，看余数
3. "CRC能检测什么类型的错误" → 考察检错能力
4. "与奇偶校验、海明码对比" → 考察编码方法差异
