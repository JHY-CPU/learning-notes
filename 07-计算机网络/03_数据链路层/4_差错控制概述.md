# 05_差错控制概述

## 核心概念

- **差错（Error）**：数据在传输过程中发生的比特值改变（0变1或1变0）
- **差错控制**：检测或纠正传输中产生的差错，保证数据的可靠性
- **差错来源**：噪声（热噪声、冲击噪声）、衰减、串扰等

### 差错的类型

| 类型 | 说明 | 特点 |
|------|------|------|
| 随机差错 | 单个比特独立出错 | 各比特出错概率独立 |
| 突发差错 | 连续多个比特出错 | 一次干扰导致连续错误 |

- **误码率（BER, Bit Error Rate）**：出错比特数 / 传输总比特数

$$BER = \frac{\text{出错比特数}}{\text{传输总比特数}}$$

- 典型光纤BER = $10^{-9}$，即每传输10亿个比特才可能出错1个

### 差错控制的基本策略

| 策略 | 方法 | 特点 |
|------|------|------|
| **检错码** | CRC | 只能发现错误，不能纠正 |
| **纠错码** | 海明码 | 既能发现错误，又能纠正 |
| **反馈/重传** | ARQ | 发现错误后请求重传 |

### 检错码 vs 纠错码

- **检错码**：接收方检测到出错，请求发送方重传（ARQ机制）
- **纠错码**：接收方直接根据编码纠正错误，不需要重传
- 数据链路层主要使用**CRC检错码 + ARQ重传机制**

### 常见编码方式

| 编码 | 类型 | 能力 |
|------|------|------|
| 奇偶校验 | 检错 | 检测奇数个错误 |
| CRC循环冗余校验 | 检错 | 检测突发差错 |
| 海明码 | 纠错 | 检测2位、纠正1位错误 |

### 考试重点

- CRC是**数据链路层最重要的检错方法**，必考
- 海明码的**编码和纠错**过程常考计算题
- 区分**检错码和纠错码**的能力差异

## 原理分析

### 噪声与差错

- **随机噪声**：引起随机差错，信噪比越低，误码率越高
- **冲击噪声**：引起突发差错，是产生差错的**主要原因**

### ARQ（自动重传请求）

$$\text{差错控制} = \text{检错码(CRC)} + \text{ARQ(确认+重传)}$$

ARQ的三种实现：
1. 停等ARQ
2. 后退N帧ARQ（GBN）
3. 选择重传ARQ（SR）

### 编码效率

$$\text{编码效率} = \frac{\text{信息位数}}{\text{总位数}} = \frac{k}{n}$$

- 检错/纠错能力越强，冗余位越多，编码效率越低
- 这是一个**能力与效率的权衡**

### 海明不等式

对于能纠正 $t$ 位错误的编码，需要满足：

$$2^r \geq n + 1$$

其中 $r$ 为校验位数，$n$ 为总位数（信息位+校验位）。

## 直观理解

- 差错控制就像**寄包裹时的保险**：检错码相当于"开箱检查"（发现包裹被换了），纠错码相当于"自动修复"
- **检错码**就像门卫：只负责报警"有人闯入"，不负责抓人
- **纠错码**就像保安：不仅能发现入侵者，还能将其制服

### CRC vs 海明码的比喻

- CRC = **验钞机**：只告诉你钱是不是假的，不帮你把假币变成真的
- 海明码 = **自动修复笔**：能检测并自动修正部分错误

## 协议关联

### 差错控制在协议栈中的位置

- 数据链路层：**CRC检错**（以太网、PPP）
- 传输层：**校验和检错**（TCP、UDP）
- 两者方法不同，但目的相同

### 与可靠传输的关系

```
差错控制 ──> 检测/纠正比特差错
    │
    ├─ 检错码(CRC) + ARQ ──> 保证可靠传输
    │
    └─ 纠错码(海明码) ──> 直接纠正，无需重传
```

### 常见考试组合

- CRC计算 + 帧校验序列（FCS）计算
- 海明码编码 + 纠错过程
- 奇偶校验的检错能力分析
- 误码率与帧长度对出帧率的影响

$$P(\text{帧出错}) = 1 - (1 - BER)^L \approx L \times BER$$

其中 $L$ 为帧长（比特数），当BER较小时可近似计算。
