# 85_以太网与无线对比_考试

## 核心概念

- **408考点**：CSMA/CD vs CSMA/CA、帧格式差异、ACK机制差异
- **典型问法**：为什么无线不用CSMA/CD？以太网帧和WiFi帧有何不同？
- **核心区别**：有线能边发边收（冲突检测），无线不能（冲突避免）

### CSMA/CD vs CSMA/CA 对比

| 特性 | CSMA/CD（有线） | CSMA/CA（无线） |
|------|----------------|----------------|
| 全称 | 载波监听多路访问/冲突检测 | 载波监听多路访问/冲突避免 |
| 冲突处理 | 检测到冲突后停止，重传 | 避免冲突发生 |
| 边发边收 | 可以 | **不可以** |
| ACK确认 | 不需要 | **需要** |
| 退避算法 | 二进制指数退避 | 类似，竞争窗口递增 |
| 适用 | 有线以太网 | 无线WiFi |

### 帧格式差异

| 字段 | 以太网帧 | 802.11帧 |
|------|---------|---------|
| 地址数量 | 2个（源+目的） | 最多4个 |
| 前导码 | 8字节 | 不同结构 |
| FCS | CRC-32 | CRC-32 |
| 最大长度 | 1518字节 | 2346字节 |
| VLAN标签 | 可选（802.1Q） | 不使用 |

### ACK机制差异

**有线以太网**：
- 全双工交换模式，点对点传输
- 信道质量好，帧丢失概率极低
- FCS校验足够，不需要ACK
- 冲突检测（CD）本身可确认发送是否成功

**无线WiFi**：
- 半双工，共享信道
- 无线信道干扰大（多径、遮挡）
- 不能边发边收，无法通过CD确认
- 必须显式ACK确认每一帧

## 原理分析

### 真题1（MAC协议对比）

**题目**：以太网使用的MAC协议是______，无线局域网使用的MAC协议是______。

**答案**：CSMA/CD（有线以太网），CSMA/CA（无线局域网）

**注意**：现代全双工交换式以太网实际上不再使用CSMA/CD（因为点对点无冲突），但考试中以太网通常仍关联CSMA/CD。

### 真题2（ACK机制）

**题目**：为什么WiFi需要ACK确认而以太网不需要？

**答案**：
1. **信道可靠性**：无线信道干扰大，帧丢失率高；有线信道质量好，帧丢失概率极低
2. **冲突检测能力**：有线能边发边收检测冲突；无线半双工不能
3. **传输模式**：全双工交换以太网点对点传输，天然可靠；WiFi共享信道，不确定性高

### 真题3（隐藏终端）

**题目**：WiFi中解决隐藏终端问题的方法是什么？以太网有这个问题吗？

**答案**：WiFi使用**RTS/CTS机制**。发送方先发短RTS帧，接收方回复CTS（包含NAV持续时间），其他站点收到CTS后推迟发送。以太网没有隐藏终端问题，因为有线网络信号能到达所有结点。

### 真题4（帧格式）

**题目**：以太网帧的地址字段有几个？802.11帧最多有几个？为什么不同？

**答案**：以太网帧2个（源+目的），802.11帧最多4个。原因：无线帧可能需要经AP中转，需要额外的地址来标识发送端AP和接收端AP（类似源路由的思想）。

### 真题5（概念选择）

**题目**：以下关于以太网和WiFi的描述，正确的是（）

A. 以太网和WiFi都使用CSMA/CD
B. WiFi的帧格式与以太网完全相同
C. 以太网（全双工模式）不需要冲突检测
D. WiFi的传输速率总是低于以太网

**答案**：C

- A错：WiFi用CSMA/CA（无线不能检测冲突）
- B错：WiFi帧最多4个地址，以太网帧2个地址
- C对：全双工点对点链路不存在冲突
- D错：WiFi 6可达9.6 Gbps，可能超过部分有线以太网

### 真题6（802.11帧间间隔）

**题目**：802.11中DIFS和SIFS的作用分别是什么？

**答案**：
- **DIFS（分布式帧间间隔）**：数据帧发送前等待的时间，较长，用于竞争信道
- **SIFS（短帧间间隔）**：ACK帧发送前等待的时间，最短，确保ACK优先于其他数据帧
- DIFS > SIFS，保证ACK不会被数据帧抢走信道

## 直观理解

- **以太网和WiFi的关系**就像"高速公路和航线"：高速公路（有线）容量大、稳定、快速；航线（无线）灵活但受天气（干扰）影响
- **CSMA/CD像会议室讨论**：同时发言能听到冲突，一方退让
- **CSMA/CA像对讲机通信**：说之前先听，说完等回复，不能边说边听

### 408易错点

- 有线以太网用CSMA/CD，无线WiFi用CSMA/CA——不能混用
- 现代交换式以太网（全双工）实际上不再使用CSMA/CD
- WiFi帧格式与以太网帧格式**不同**（地址数量、前导码等）
- WiFi需要ACK，以太网不需要——这是最核心的区别
- WiFi的隐藏终端问题是以太网没有的

## 协议关联

- AP是连接以太网和WiFi的桥梁（数据链路层设备）
- 802.11帧格式与以太网帧格式不同，AP需要做帧格式转换
- WiFi的安全（WEP→WPA→WPA2→WPA3）也是考试热点
- 未来的网络趋势：有线骨干+无线接入（WiFi 6/7 + 10G以太网）
- 408考试中这是选择题的常见考点，务必掌握对比表
