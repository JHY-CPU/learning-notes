# 48_以太网IEEE 802.3标准

## 核心概念

- **IEEE 802.3**：以太网的IEEE标准，规定了物理层和MAC子层的规范
- **802.3 vs Ethernet V2**：两者帧格式略有不同，但实际兼容
- **核心规范**：CSMA/CD、曼彻斯特编码、帧格式

### 802.3标准族

| 标准 | 速率 | 介质 | 最大距离 |
|------|------|------|---------|
| 802.3 (10BASE5) | 10 Mbps | 粗同轴电缆 | 500m |
| 802.3a (10BASE2) | 10 Mbps | 细同轴电缆 | 185m |
| 802.3i (10BASE-T) | 10 Mbps | 双绞线 | 100m |
| 802.3u (100BASE-TX) | 100 Mbps | 双绞线 | 100m |
| 802.3z (1000BASE-SX/LX) | 1 Gbps | 光纤 | 550m/5km |
| 802.3ab (1000BASE-T) | 1 Gbps | 双绞线 | 100m |
| 802.3ae (10GBASE-SR/LR) | 10 Gbps | 光纤 | 300m/10km |

### Ethernet V2 vs 802.3帧格式

| 字段 | Ethernet V2 | IEEE 802.3 |
|------|-------------|------------|
| 前导码 | 7字节 | 7字节 |
| SFD | 1字节 | 1字节 |
| 目的地址 | 6字节 | 6字节 |
| 源地址 | 6字节 | 6字节 |
| 类型/长度 | 2字节（类型） | 2字节（长度） |
| 数据 | 46-1500字节 | LLC首部 + 数据 |
| FCS | 4字节 | 4字节 |

关键区别：V2的第3字段是**类型**（如0x0800=IP），802.3的是**长度**。通过值区分：>1500为类型(V2)，≤1500为长度(802.3)。

### LLC子层

- **LLC（Logical Link Control）**：802.3的数据链路层上半子层
- 提供与物理层无关的接口
- 实际中很少使用，以太网直接用Ethernet V2格式

### 考试重点

- 以太网命名规则：`速率 + BASE + 距离/介质`
- V2与802.3帧格式的区别
- 物理层规范（编码方式、介质类型）

## 原理分析

### 命名规则详解

```
10  BASE   5
│    │      │
│    │      └── 距离（5=500m, 2≈200m）或介质（T=双绞线, F=光纤）
│    └── 基带传输（Baseband）
└── 数据速率（Mbps）
```

- **10BASE5**：10Mbps，基带，最远500m（粗缆）
- **10BASE2**：10Mbps，基带，最远~200m（细缆）
- **10BASE-T**：10Mbps，基带，双绞线
- **100BASE-TX**：100Mbps，基带，双绞线（5类）
- **1000BASE-T**：1Gbps，基带，双绞线（超5类/6类）

### 物理层编码

| 标准 | 编码方式 |
|------|---------|
| 10 Mbps | 曼彻斯特编码 |
| 100 Mbps | 4B/5B + MLT-3 |
| 1 Gbps | 8B/10B / PAM-5 |
| 10 Gbps | 64B/66B |

## 直观理解

- **802.3标准就像汽车标准**：规定了车的尺寸、燃料类型、最高时速，但不同厂商的车可以互相在同一条路上开

## 协议关联

- 802.3是数据链路层+物理层的规范
- 与802.11（WiFi）、802.1Q（VLAN）等是"兄弟标准"
- 以太网的演进推动了网络设备（集线器→交换机→路由器）的发展
