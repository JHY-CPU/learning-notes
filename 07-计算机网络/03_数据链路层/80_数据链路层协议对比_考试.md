# 83_数据链路层协议对比_考试

## 核心概念

- **408考点**：协议分类、透明传输方式、可靠传输能力、MAC地址需求
- **典型问法**：以下哪个协议提供可靠传输？使用什么透明传输？
- **三大协议**：以太网（802.3）、PPP、HDLC

### 核心对比表

| 特性 | 以太网 | PPP | HDLC |
|------|--------|-----|------|
| 传输单位 | 面向比特 | 面向字节 | 面向比特 |
| 可靠传输 | 否 | **否** | **是** |
| 透明传输 | 违规编码法 | 字节填充 | 零比特填充 |
| MAC地址 | 需要 | **不需要** | **不需要** |
| 链路类型 | 广播（多点） | 点对点 | 点对点+多点 |
| 冲突处理 | CSMA/CD | 无（点对点） | 无（点对点） |
| 帧标志 | 前导码 | 0x7E | 0x7E |
| FCS校验 | CRC-32 | CRC-16 | CRC-16 |
| 全双工 | 支持（交换模式） | 支持 | 支持 |

## 原理分析

### 真题1（透明传输）

**题目**：以下协议中，使用零比特填充的是（）

A. PPP  B. HDLC  C. 以太网  D. CSMA/CD

**答案**：B

**解析**：
- PPP用字节填充（0x7E→0x7D 0x5E）
- HDLC用零比特填充（连续5个1后插入0）
- 以太网用违规编码法（曼彻斯特编码中不存在的电平组合）或前导码

### 真题2（可靠传输）

**题目**：以下协议中，可以提供可靠传输的是（）

A. 以太网  B. PPP  C. HDLC  D. CSMA/CA

**答案**：C

**解析**：只有HDLC（通过I帧的序号N(S)和确认N(R)实现可靠传输）提供数据链路层的可靠传输。以太网、PPP、CSMA/CA都不提供可靠传输。

### 真题3（MAC地址）

**题目**：以下协议中，不需要MAC地址的是（）

A. 以太网  B. PPP  C. WiFi  D. 802.3

**答案**：B

**解析**：PPP是点对点协议，链路上只有两台设备，不需要地址来区分。以太网（802.3）和WiFi（802.11）都是广播网络，需要MAC地址。

### 真题4（协议选择）

**题目**：在拨号上网场景中，数据链路层通常使用什么协议？

**答案**：**PPP**（或PPPoE）。原因：PPP支持认证（PAP/CHAP），适合运营商对用户身份验证；PPP简单高效，适合低速串行链路。

### 真题5（综合判断）

**题目**：判断以下说法正误：
1. 以太网提供可靠的数据传输服务
2. PPP面向比特
3. CSMA/CD可以用于无线网络
4. HDLC可以工作在点对点和多点链路上

**答案**：
1. **错**：以太网不提供可靠传输（无序号、无确认）
2. **错**：PPP面向字节（HDLC才面向比特）
3. **错**：无线无法边发边收，不能检测冲突，应用CSMA/CA
4. **对**：HDLC支持非平衡（多点）和平衡（点对点）配置

### 真题6（适用场景）

**题目**：以下场景应选用什么协议？
1. 局域网中多台电脑互联
2. 两台路由器之间的串行链路
3. ADSL宽带拨号上网

**答案**：
1. **以太网（802.3）**：广播网络，多台设备，需要MAC寻址和CSMA/CD
2. **PPP或HDLC**：点对点链路，不需要MAC地址
3. **PPPoE**：需要认证功能，结合以太网和PPP

## 直观理解

- **记忆表格**：以太网不可靠/违规编码，PPP不可靠/字节填充，HDLC可靠/零比特填充
- **以太网**像公交车站：多个人等车，需要排队（CSMA/CD），每人都有身份证（MAC地址）
- **PPP**像专线电话：两台设备专用线路，不需要身份证
- **HDLC**像挂号信专线：两台设备专用线路，有编号有回执

### 408高频考点

1. 透明传输方式的区分（字节填充vs零比特填充vs违规编码）
2. 可靠传输能力的区分（只有HDLC可靠）
3. MAC地址需求的区分（点对点不需要）
4. 各协议适用场景的选择

## 协议关联

- 以太网是局域网的事实标准（IEEE 802.3）
- PPP是广域网点对点链路的标准协议（RFC 1661）
- HDLC是ISO标准，实际中较少使用但408考试重点
- 数据链路层的可靠传输（HDLC）与传输层的可靠传输（TCP）是不同层次的概念
- 协议对比题是408选择题的常客，务必掌握各协议的核心区别
