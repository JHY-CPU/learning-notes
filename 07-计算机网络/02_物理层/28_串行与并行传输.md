# 29_串行与并行传输

## 核心概念

### 串行传输（Serial Transmission）

- 数据**逐位**在一条信道上传输
- 在某一时刻，信道上只有1个比特
- 适合**远距离**传输

### 并行传输（Parallel Transmission）

- 数据**多位同时**在多条并行信道上传输
- 例如：8条线路同时传8位数据
- 适合**近距离**传输

### 两者对比

| 特性 | 串行传输 | 并行传输 |
|------|----------|----------|
| 信道数 | 1条 | 多条（如8条） |
| 传输速度 | 慢（逐位） | 快（多位同时） |
| 成本 | 低（线缆少） | 高（线缆多） |
| 距离 | 远距离 | 短距离 |
| 同步要求 | 简单 | 复杂（需要线路间同步） |
| 典型应用 | 网络通信、USB | CPU-内存总线、打印机接口 |

## 原理分析

### 串行传输方式

```
发送端                         接收端
D7 D6 D5 D4 D3 D2 D1 D0
 │  │  │  │  │  │  │  │
 └──┴──┴──┴──┴──┴──┴──┴──→ [一条信道] → D7 D6 D5 D4 D3 D2 D1 D0
   （按顺序一位一位发送）                    （按顺序一位一位接收）
```

### 并行传输方式

```
发送端     接收端
D7 ──────→ D7
D6 ──────→ D6
D5 ──────→ D5
D4 ──────→ D4
D3 ──────→ D3
D2 ──────→ D2
D1 ──────→ D1
D0 ──────→ D0
```

### 为什么远距离用串行

并行传输在远距离时的问题：
1. **成本高**：远距离铺设多根线缆成本巨大
2. **信号偏移（Skew）**：不同线路的信号到达时间不一致
3. **串扰**：并行线路之间的信号干扰
4. **时钟同步困难**：需要保证所有线路同步

串行传输的改进：
- 现代高速串行接口（如PCIe、USB 3.0）通过极高的时钟频率弥补了串行的不足
- 例如：USB 3.0 串行速率 5 Gbit/s，远超老式并行接口

## 直观理解

### 类比：过桥

- **串行传输**：一条独木桥，人们一个一个过
  - 简单但慢
  - 不需要协调
- **并行传输**：一座宽阔的桥，多排人同时过
  - 快但复杂
  - 需要协调步伐（所有人要同步）

### 类比：水管

- 串行：一根粗水管，水一滴一滴排着队流
- 并行：多根细水管，水同时从多根管子流

## 协议关联

### 与实际接口的关系

| 接口 | 传输方式 | 用途 |
|------|----------|------|
| 以太网RJ-45 | 串行（差分对） | 网络通信 |
| USB | 串行 | 外设连接 |
| RS-232 | 串行 | 串口通信 |
| PCI | 并行 | 计算机内部总线 |
| PCIe | 串行 | 现代内部总线（取代PCI） |
| SATA | 串行 | 硬盘接口（取代并行ATA） |

发展趋势：从并行转向高速串行

### 考试要点

1. 串行传输适合远距离（线缆成本低）
2. 并行传输适合近距离（速度优势）
3. 现代高速接口越来越多采用串行方式
4. 计算机网络中普遍使用串行传输
