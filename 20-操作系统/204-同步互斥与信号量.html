🔒 同步、互斥与信号量

408：PV 操作、记录型信号量、经典同步问题（生产者-消费者等）常出综合题。竞态条件 Race condition、临界区、mutex、semaphore、条件变量（概念层）。

临界区 Critical Section

访问共享资源的代码段必须互斥执行：同一时刻最多一个执行流进入临界区。

// ========== 竞态示例（错误）========== // 两个线程同时对 counter++ // 底层可能是 read-modify-write，交错执行导致丢失更新 // ========== 正确思路 ========== // 进入前 acquire 锁；离开后 release // 锁的实现最终依赖硬件原子指令（如 compare-and-swap）

互斥锁 Mutex

// ========== 自旋锁 vs 睡眠锁 ========== // 自旋：短临界区、多核、等待时间极短时可能划算 // 睡眠：等待久时让出 CPU，由内核排队（futex 等） // ========== 常见陷阱 ========== // 死锁：互相等待对方持有的锁（见 205） // 优先级反转：低优先级持锁阻塞高优先级（需优先级继承等机制）

信号量 Semaphore

计数器 + 两个原子操作：P(wait) 减 1，若 <0 则阻塞；V(signal) 加 1，必要时唤醒等待者。

// ========== 二进制信号量 ≈ 互斥锁 ========== // 计数信号量：限制同时访问资源的数量（如连接池槽位） // ========== 与条件变量 ========== // mutex 保护“数据”；condition variable 用于“等待某个条件成立” // 典型模式：加锁 → 检查条件 → while 不成立则 wait → 修改 → signal → 解锁
📌 用户态编程优先使用语言/库提供的 mutex、RWLock、channel；理解信号量有助于读内核与经典教材。
