💾 文件系统与磁盘 I/O

408：目录结构、文件物理分配、混合索引、磁盘调度（FCFS/SSTF/SCAN/CSCAN）与寻道时间计算；缓冲与 SPOOLing 常考简答。VFS、inode、目录、块与缓冲、日志（journaling）直觉。

从应用到块设备

// ========== 典型路径 ========== // 应用: read(fd, buf, n) // VFS: 根据 fd 找到 struct file → dentry → inode // 具体文件系统 ext4/xfs: inode → 逻辑块号 → 物理块 // 块层: 合并、调度 I/O 请求 // 驱动: 与磁盘控制器通信 // ========== 页缓存 page cache ========== // 文件数据常以页为单位缓存在内存中 // 多次读取同一区域可能完全不碰磁盘

inode 与目录

// ========== inode（索引节点）========== // 元数据：权限、所有者、大小、时间戳 // 数据块指针：直接块、间接、双重间接（视具体 FS） // 文件名不存放在 inode 内 // ========== 目录 directory ========== // 目录是特殊文件：映射「文件名 → inode 号」 // 硬链接：多个目录项指向同一 inode（引用计数） // 软链接：单独 inode，内容为目标路径字符串

可靠性与性能

// ========== 日志文件系统 journaling ========== // 先写日志再写元数据/数据，崩溃后可重放恢复一致性 // 权衡：额外写放大 vs 启动 fsck 时间 // ========== 延迟分配、预读 ========== // 内核合并小写、顺序预读以提高吞吐 // fsync/fdatasync 强制刷盘，保证持久化语义
📌 数据库与分布式系统常绕过部分页缓存或使用 O_DIRECT（视场景），需理解一致性与性能权衡。
