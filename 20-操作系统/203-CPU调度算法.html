📊 CPU 调度算法

408：FCFS/SJF/RR/优先级/高响应比/多级反馈队列及周转、带权周转、等待时间计算。调度目标、经典算法对比、交互式系统中的时间片与优先级。

调度要优化什么

// ========== 常见指标 ========== // 吞吐量 Throughput: 单位时间完成作业数 // 周转时间 Turnaround: 提交 → 完成 // 响应时间 Response: 提交 → 首次运行（交互体验） // 等待时间 Waiting: 在就绪队列中排队总时间 // 公平性: 防止饥饿（starvation） // 矛盾：吞吐 vs 延迟；批处理 vs 实时/交互

经典算法

// ========== FCFS 先来先服务 ========== // 非抢占，实现简单 // 护航效应 convoy effect：长作业阻塞短作业 // ========== SJF 最短作业优先 ========== // 理论最优平均等待时间（已知服务时间时） // 问题：需要预测运行时间；可能饿死长作业 // ========== 优先级调度 ========== // 可为不同类别分配优先级 // 需防饥饿：老化 aging（等待越久优先级越高） // ========== RR 时间片轮转 ========== // 抢占式，就绪队列 FIFO，每进程运行 q 后被抢占 // q 太大 → 退化为 FCFS；q 太小 → 切换开销大 // ========== 多级反馈队列 MLFQ ========== // 多条就绪队列，优先级递减，时间片递增 // I/O 密集任务常留在高优先级；CPU 密集下沉 // 现代桌面/服务器调度器的简化理解模型
📌 真实内核（如 CFS Completely Fair Scheduler）用虚拟运行时间等机制逼近公平，比教科书 RR 更复杂。

多核与 NUMA（概念）

// ========== 亲和性 CPU affinity ========== // 线程固定在某些核上跑 → 提高缓存命中 // 负载均衡：避免一核忙其他闲 // ========== NUMA ========== // 内存按节点划分，访问本地节点快、远端慢 // 调度与内存分配需“就近”，否则性能抖动
