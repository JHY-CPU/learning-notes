为什么需要虚拟内存

// ========== 收益 ========== // 隔离：每进程独立虚拟地址空间，互不直接踩踏 // 简化链接与加载：程序可用统一虚拟布局 // 超额订阅 overcommit：运行内存总量可大于物理 RAM（换页到磁盘） // 共享：同一物理页映射到多进程（共享库、fork 的写时复制 COW）

分页 Paging

虚拟地址与物理地址以页 page（如 4 KiB）为单位映射；映射关系存放在页表（多级页表节省空间）。

// ========== 地址翻译 ========== // 虚拟地址: [ VPN | 页内偏移 ] // ↓ 页表查找 // 物理地址: [ PFN | 页内偏移 ] // ========== TLB Translation Lookaside Buffer ========== // 页表项缓存，命中则无需访问内存中的页表 // TLB miss → 多级页表遍历（可多次内存访问）→ 昂贵 // 进程切换常需 flush 或标记 ASID 以区分不同地址空间
📌 大页 huge pages：减少 TLB miss，适合大内存连续访问场景（数据库、科学计算）。

缺页 Page Fault

// ========== 类型（简化）========== // 合法但尚未分配物理页：按需分页 demand paging，分配帧并建立映射 // 合法但在交换区：从 swap 读回 // 非法访问：段错误 SIGSEGV // ========== 页面置换 ========== // 物理页帧不足时需换出：FIFO、LRU（近似）、时钟算法 // 工作集理论：进程近期常用页集合应常驻内存

与编程的关系

// mmap：文件映射到虚拟地址，按需缺页读入 // malloc：常先增长堆虚拟空间，真正触碰页面时才分配物理页 // 顺序 vs 随机访问：影响 TLB/缓存友好度
