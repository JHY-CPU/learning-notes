# A/B测试基础


## 一、A/B 测试定义


A/B测试（也称为随机对照实验，Randomized Controlled Experiment）是一种通过**随机分组**比较两种或多种方案效果的实验方法。它是互联网产品决策的金标准。


### 1.1 核心流程


1. 将用户
   **随机**
   分为两组（或多组）
2. 控制组（A组）使用原方案，实验组（B组）使用新方案
3. 收集两组的指标数据
4. 用统计方法判断差异是否显著
5. 基于数据做出决策


### 1.2 核心概念


| 概念 | 含义 |
| --- | --- |
| **原假设 H~0~** | 两组没有差异（μ~A~ = μ~B~） |
| **备择假设 H~1~** | 两组存在差异（μ~A~ ≠ μ~B~） |
| **p值** | 在H~0~成立的前提下，观察到当前结果或更极端结果的概率 |
| **显著性水平 α** | 拒绝H~0~的阈值，通常设为0.05 |


## 二、假设检验方法


### 2.1 Z检验（大样本比例检验）


适用于大样本下的比例比较（如点击率、转化率）。中心极限定理保证样本均值近似正态分布。


$$
Z = (p̄B - p̄A) / √(p̄(1-p̄)(1/nA + 1/nB))
                其中 p̄ = (xA + xB) / (nA + nB) 为合并比例
$$


### 2.2 t检验（均值比较）


适用于比较两组的均值差异（如平均消费金额、平均停留时长）。


$$
t = (X̄B - X̄A) / √(SA²/nA + SB²/nB)
                自由度用 Welch-Satterthwaite 公式近似
$$


### 2.3 卡方检验（分类变量）


适用于比较两组的分类分布（如用户偏好分布、多个类别的转化率）。


$$
χ² = ∑ (Oij - Eij)² / Eij
                其中 Oij 为观测频数，Eij 为期望频数
$$


## 三、两类错误与统计功效


|  | H~0~ 为真（无差异） | H~0~ 为假（有差异） |
| --- | --- | --- |
| **不拒绝 H~0~** | 正确 | 第二类错误 (β) |
| **拒绝 H~0~** | 第一类错误 (α) | 正确 (Power = 1-β) |


### 3.1 第一类错误（α）— 假阳性


没有差异但错误地认为有差异，通常控制在 5%（0.05）。即"把坏的当成好的"。


### 3.2 第二类错误（β）— 假阴性


有差异但没检测到，通常要求 β ≤ 20%。即"把好的当成坏的"。


### 3.3 统计功效（Power = 1 - β）


成功检测到真实差异的概率，通常要求 ≥ 80%。


$$
样本量计算公式（比例检验）：
                n = (Zα/2 + Zβ)² × (pA(1-pA) + pB(1-pB)) / (pB - pA)²
                对于 α=0.05, Power=0.80：
                n ≈ 16 × p(1-p) / δ² （δ为最小可检测效应MDE）
$$


> **Important:** **关键洞察：**
> 样本量与效应量的平方成反比。如果要检测的效应量减半（如从2%降到1%），所需样本量增加4倍。这也是为什么检测小效应需要巨大流量。


## 四、Python 实战：A/B 测试


> **Example:** ### 示例：电商转化率A/B测试
>
>
> ```
> import numpy as np
> from scipy import stats
> import matplotlib.pyplot as plt
>
> # 1. 实验设置
> np.random.seed(42)
> n_A, n_B = 5000, 5000
> true_cr_A = 0.10   # 控制组真实转化率10%
> true_cr_B = 0.115  # 实验组真实转化率11.5%（提升15%）
>
> # 2. 模拟实验数据
> conversions_A = np.random.binomial(1, true_cr_A, n_A)
> conversions_B = np.random.binomial(1, true_cr_B, n_B)
>
> cr_A = conversions_A.mean()
> cr_B = conversions_B.mean()
> print(f"控制组转化率: {cr_A:.4f} ({conversions_A.sum()}/{n_A})")
> print(f"实验组转化率: {cr_B:.4f} ({conversions_B.sum()}/{n_B})")
> print(f"相对提升: {(cr_B - cr_A) / cr_A * 100:.2f}%")
>
> # 3. Z检验（比例检验）
> p_pooled = (conversions_A.sum() + conversions_B.sum()) / (n_A + n_B)
> se = np.sqrt(p_pooled * (1 - p_pooled) * (1/n_A + 1/n_B))
> z_stat = (cr_B - cr_A) / se
> p_value = 2 * (1 - stats.norm.cdf(abs(z_stat)))
>
> print(f"\n=== Z检验结果 ===")
> print(f"Z统计量: {z_stat:.4f}")
> print(f"p值: {p_value:.4f}")
> print(f"结论: {'显著' if p_value < 0.05 else '不显著'} (alpha=0.05)")
>
> # 4. 置信区间
> ci_lower = (cr_B - cr_A) - 1.96 * np.sqrt(cr_A*(1-cr_A)/n_A + cr_B*(1-cr_B)/n_B)
> ci_upper = (cr_B - cr_A) + 1.96 * np.sqrt(cr_A*(1-cr_A)/n_A + cr_B*(1-cr_B)/n_B)
> print(f"转化率差异95%置信区间: [{ci_lower:.4f}, {ci_upper:.4f}]")
>
> # 5. 样本量计算（预先计算需要多少样本）
> from scipy.stats import norm
>
> def required_sample_size(p_A, mde, alpha=0.05, power=0.80):
>     """计算所需样本量"""
>     p_B = p_A + mde
>     z_alpha = norm.ppf(1 - alpha / 2)
>     z_beta = norm.ppf(power)
>     n = ((z_alpha + z_beta)**2 * (p_A*(1-p_A) + p_B*(1-p_B))) / (mde**2)
>     return int(np.ceil(n))
>
> mde = 0.015  # 最小可检测效应1.5%
> n_needed = required_sample_size(0.10, mde)
> print(f"\n=== 样本量计算 ===")
> print(f"基线转化率: 10%, MDE: {mde*100}%")
> print(f"每组所需样本量: {n_needed}")
>
> # 6. 可视化
> fig, axes = plt.subplots(1, 2, figsize=(12, 5))
>
> # 转化率对比
> axes[0].bar(['控制组(A)', '实验组(B)'], [cr_A, cr_B],
>             color=['#3498db', '#e74c3c'], alpha=0.8)
> axes[0].set_ylabel('转化率')
> axes[0].set_title('转化率对比')
> axes[0].set_ylim(0, max(cr_A, cr_B) * 1.3)
>
> # 置信区间
> axes[1].errorbar(1, cr_B - cr_A, yerr=[[cr_B-cr_A-ci_lower], [ci_upper-(cr_B-cr_A)]],
>                  fmt='o', capsize=10, color='#e74c3c', markersize=10)
> axes[1].axhline(y=0, color='gray', linestyle='--')
> axes[1].set_xlim(0, 2)
> axes[1].set_title('转化率差异及95%置信区间')
> axes[1].set_ylabel('差异 (B - A)')
>
> plt.tight_layout()
> plt.savefig("ab_test_result.png", dpi=150)
> plt.show()
> ```


## 五、A/B 测试实施流程


1. **确定目标指标：**
   选择1个核心指标（如转化率）+ 若干护栏指标（如留存率）
2. **计算样本量：**
   基于基线值、MDE、α和Power计算
3. **随机分流：**
   确保用户级别的随机化，避免辛普森悖论
4. **运行实验：**
   通常运行1-2个完整业务周期
5. **分析结果：**
   检查p值、置信区间、效应量
6. **做出决策：**
   综合统计显著性和业务显著性


> **Note:** **统计显著性 vs 业务显著性：**
> 即使p<0.05（统计显著），如果提升太小（如0.01%的转化率提升），可能不值得上线。反之，大的提升（如10%）即使p值略高于0.05也值得进一步验证。


## 总结


- A/B测试通过随机对照实验比较方案效果
- 核心是假设检验：原假设H0（无差异）、备择假设H1（有差异）
- 常用检验方法：Z检验（比例）、t检验（均值）、卡方检验（分类）
- 两类错误：假阳性(α)和假阴性(β)，Power=1-β
- 样本量与效应量平方成反比，检测小效应需要大样本
- 实际决策需综合统计显著性和业务显著性
