# 因果推断基础


## 一、相关不等于因果


相关性（Correlation）和因果性（Causation）是两个根本不同的概念。观察到两个变量相关，并不能推出一个导致了另一个。


### 1.1 经典反直觉案例


| 相关关系 | 真实解释 |
| --- | --- |
| 冰淇淋销量 & 溺水人数正相关 | 混杂变量：天气热 |
| 消防车数量 & 火灾损失正相关 | 反向因果：大火需要更多消防车 |
| 教育年限 & 收入正相关 | 可能有混杂：能力、家庭背景 |
| 医院数量 & 死亡率正相关 | 选择偏差：生病才去医院 |


> **Important:** **因果推断的核心挑战：**
> 在观察性数据中，我们只能看到一个事实结果（Y
> ~1~
> 或 Y
> ~0~
> ），而无法同时观察到另一个反事实结果。这就是Rubin所说的"因果推断的基本问题"。


## 二、两大因果推断框架


### 2.1 Pearl 的结构因果模型（SCM）


Judea Pearl 提出基于有向无环图（DAG）的因果框架，核心工具是 do-演算。


$$
P(Y | do(X=x))：干预X=x后Y的分布
                与条件概率 P(Y | X=x) 不同：后者可能受混杂影响
$$


- 用DAG表示变量间的因果关系
- 后门准则、前门准则用于识别因果效应
- do-演算提供形式化的因果识别规则


### 2.2 Rubin 的潜在结果框架（Neyman-Rubin Causal Model）


每个个体都有两个潜在结果：


$$
Yi(1)：个体 i 接受处理后的结果
                Yi(0)：个体 i 未接受处理的结果
                个体处理效应（ITE）：τi = Yi(1) - Yi(0)
                平均处理效应（ATE）：ATE = E[Y(1) - Y(0)]
$$


| 维度 | Pearl SCM | Rubin CM |
| --- | --- | --- |
| 核心工具 | DAG、do-演算 | 潜在结果、反事实 |
| 优势 | 直观展示因果结构 | 与统计估计紧密联系 |
| 适用场景 | 因果结构已知 | 基于实验或准实验 |


## 三、混杂变量与后门准则


### 3.1 混杂变量（Confounder）


混杂变量是同时影响处理（X）和结果（Y）的变量，如果不控制它，会导致因果效应估计有偏。


> **Example:** **例子：**
> 研究咖啡是否导致心脏病。年龄是混杂变量——年龄大的人更可能喝咖啡，也更可能得心脏病。如果不控制年龄，咖啡和心脏病的相关性会被高估。


### 3.2 后门准则（Backdoor Criterion）


Pearl提出的后门准则是判断是否需要控制某些变量的规则：


1. 变量集合 Z 中没有 X 的后代
2. Z 阻断了所有从 X 到 Y 的后门路径（即非因果路径）


满足后门准则时：P(Y | do(X)) = ∑~z~ P(Y | X, Z=z) P(Z=z)


### 3.3 工具变量（Instrumental Variable, IV）


当存在不可观测的混杂时，如果能找到一个变量 Z 满足：


1. Z 影响 X（相关性）
2. Z 只通过 X 影响 Y（排他性约束）
3. Z 与不可观测的混杂无关（独立性）


则可以用两阶段最小二乘（2SLS）估计因果效应。


$$
第一阶段：X = α + γZ + ε
                第二阶段：Y = β + τX̂ + ν
                τ 即为因果效应的IV估计量
$$


## 四、倾向得分匹配（PSM）


PSM的核心思想：如果两个个体的倾向得分（接受处理的概率）相同，那么他们被分到处理组/控制组是"近似随机"的，可以比较他们的结果差异。


### 4.1 倾向得分


$$
e(X) = P(T=1 | X)：给定协变量X，个体接受处理的条件概率
                通常用Logistic回归或梯度提升树估计
$$


### 4.2 匹配方法


| 方法 | 描述 | 优缺点 |
| --- | --- | --- |
| 最近邻匹配 | 每个处理组找到最近的控制组个体 | 简单，可能不唯一 |
| 卡钳匹配 | 设置最大距离阈值，超出则丢弃 | 质量更高，但损失样本 |
| 核匹配 | 用核函数加权所有控制组个体 | 用全部数据，效率高 |
| 马氏距离匹配 | 基于协方差矩阵的距离 | 考虑变量间相关性 |


### 4.3 Python 实战


```
import numpy as np
import pandas as pd
from sklearn.linear_model import LogisticRegression
from sklearn.neighbors import NearestNeighbors
from scipy import stats

# 1. 生成模拟数据
np.random.seed(42)
n = 2000
X1 = np.random.normal(0, 1, n)  # 混杂变量1
X2 = np.random.normal(0, 1, n)  # 混杂变量2
age = np.random.normal(40, 10, n)  # 混杂变量3

# 处理分配（受混杂影响）
propensity = 1 / (1 + np.exp(-(0.5*X1 + 0.3*X2 + 0.02*age - 2)))
T = np.random.binomial(1, propensity)

# 结果（因果效应=2.0）
Y = 3*X1 + 2*X2 + 0.1*age + 2.0*T + np.random.normal(0, 1, n)

df = pd.DataFrame({'X1': X1, 'X2': X2, 'age': age, 'T': T, 'Y': Y})

# 2. 估计倾向得分
ps_model = LogisticRegression()
ps_model.fit(df[['X1', 'X2', 'age']], df['T'])
df['propensity_score'] = ps_model.predict_proba(df[['X1', 'X2', 'age']])[:, 1]

# 3. 最近邻匹配
treated = df[df['T'] == 1]
control = df[df['T'] == 0]

nn = NearestNeighbors(n_neighbors=1, metric='euclidean')
nn.fit(control[['X1', 'X2', 'age']])
distances, indices = nn.kneighbors(treated[['X1', 'X2', 'age']])

matched_control = control.iloc[indices.flatten()]
matched_pairs = pd.DataFrame({
    'Y_treated': treated['Y'].values,
    'Y_control': matched_control['Y'].values
})

# 4. 估计ATE
ate = (matched_pairs['Y_treated'] - matched_pairs['Y_control']).mean()
t_stat, p_value = stats.ttest_rel(matched_pairs['Y_treated'],
                                    matched_pairs['Y_control'])
print(f"=== PSM匹配结果 ===")
print(f"匹配样本数: {len(matched_pairs)}")
print(f"ATE估计: {ate:.4f} (真实ATE=2.0)")
print(f"t统计量: {t_stat:.4f}")
print(f"p值: {p_value:.6f}")

# 5. 检查匹配质量（匹配后协变量平衡）
print(f"\n=== 匹配后协变量平衡 ===")
for var in ['X1', 'X2', 'age']:
    std_diff = (treated[var].mean() - matched_control[var].mean()) / \
               np.sqrt((treated[var].var() + matched_control[var].var()) / 2)
    print(f"{var} 标准化差异: {std_diff:.4f} (应接近0)")
```


## 五、双重差分法（DID）


DID适用于有处理组和控制组、且有处理前后两个时间点的场景。通过"差分的差分"消除时间趋势和组间固有差异。


$$
DID估计量 = (Ytreat,after - Ytreat,before) - (Ycontrol,after - Ycontrol,before)
                回归形式：Y = β0 + β1Treat + β2Post + β3(Treat × Post) + ε
                β3 即为DID因果效应估计量
$$


> **Note:** **关键假设：平行趋势（Parallel Trends）。**
> 如果没有处理，处理组和控制组的趋势应该相同。这无法直接检验，但可以通过处理前的多个时间点来验证。


## 六、断点回归设计（RDD）


RDD适用于处理分配由某个连续变量的阈值决定的场景。在阈值附近，处理近似随机分配。


$$
例如：考试分数 ≥ 60 分获得奖学金
                在60分附近，59分和61分的学生几乎相同
                比较阈值两侧的结果差异即为因果效应
$$


| 方法 | 适用场景 | 核心假设 |
| --- | --- | --- |
| PSM | 可观测混杂可控制 | 可忽略性（无不可观测混杂） |
| DID | 有处理前后+对照组 | 平行趋势 |
| RDD | 有明确的阈值 | 阈值附近近似随机 |
| IV | 存在工具变量 | 排他性约束 |


## 总结


- 相关不等于因果，因果推断需要特殊的方法论
- Pearl的SCM用DAG和do-演算建模因果，Rubin的CM用潜在结果定义因果效应
- 混杂变量是因果推断的核心挑战，后门准则帮助判断需要控制哪些变量
- PSM通过倾向得分匹配模拟随机实验，适用于可观测混杂
- DID通过双重差分消除时间趋势和组间差异，关键假设是平行趋势
- RDD利用阈值附近的近似随机性，是最接近RCT的准实验方法
- 工具变量可以在存在不可观测混杂时估计因果效应
