## 性能-代码分割


动态 import() 分包、React.lazy、路由级分割、chunk 命名。


## 代码分割 API


```
// ========== 动态 import() ==========
const module = await import('./module.js');
// 返回 Promise
// Webpack/Vite 自动分包

// ========== React.lazy ==========
const HeavyComponent = React.lazy(() => import('./Heavy'));
}>



// ========== 路由级分割 ==========
const Home = () => import('./pages/Home');
const About = () => import('./pages/About');
const Dashboard = () => import('./pages/Dashboard');

// ========== Chunk 命名 ==========
// Webpack: /* webpackChunkName: "admin" */
// Vite: 自动按目录命名
// 魔法注释: import(/* webpackPrefetch: true */ './a')
```

## 什么是代码分割

代码分割是将应用代码拆分成多个小块（chunk），按需加载，减少首次加载体积。现代打包工具（Webpack/Vite/Rollup）都原生支持。

## 动态 import()

`import()` 返回Promise，Webpack/Vite自动将其拆分为独立chunk。只在代码执行到该语句时才加载。

```js
// 条件加载
if (needFeature) {
    const { feature } = await import('./feature.js');
    feature();
}
```

## 分割策略

1. **路由级分割**：每个路由页面一个chunk，切换路由时加载
2. **组件级分割**：大型组件（如图表、编辑器）单独chunk
3. **第三方库分割**：将node_modules中的依赖单独打包（vendor chunk）
4. **公共模块提取**：多个chunk共享的模块提取为公共chunk

## 魔法注释

```js
import(/* webpackChunkName: "chart" */ './Chart');
import(/* webpackPrefetch: true */ './utils');  // 空闲时预取
import(/* webpackPreload: true */ './critical'); // 与父chunk并行加载
```

## 注意事项

- 分割粒度过细会导致大量小HTTP请求，反而降低性能
- 预取（prefetch）适合可能用到的模块，预加载（preload）适合即将用到的模块
- 动态import的路径不能是完全动态的，至少需要前缀路径
