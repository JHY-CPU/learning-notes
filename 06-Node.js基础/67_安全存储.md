## 安全存储


HttpOnly/Secure/SameSite Cookie、敏感信息处理、Token 存储最佳实践。


## 安全存储 API


```
// ========== Cookie 安全属性 ==========
Set-Cookie: session=abc;
    HttpOnly     // 禁止 JS 读取 (防御 XSS)
    Secure       // 仅 HTTPS 传输
    SameSite=Lax // 防 CSRF
    Domain=example.com
    Path=/
    Max-Age=86400

// ========== Token 存储对比 ==========
// localStorage: ❌ XSS 可读取
// sessionStorage: ❌ XSS 可读取
// Cookie (HttpOnly): ✅ 安全
// 内存变量: ✅ 最安全 (刷新丢失)

// ========== 敏感信息处理原则 ==========
// 1. 不在前端存储敏感数据
// 2. 使用短生命周期的 Token
// 3. 敏感操作需二次验证
// 4. HTTPS 传输
```

## 为什么安全存储重要

前端存储敏感信息（如Token、密码）容易被XSS攻击窃取。选择正确的存储方式和安全配置是Web安全的基础。

## 存储方式对比

| 存储方式 | XSS可读取 | 跨域 | 容量 | 适用场景 |
|---------|----------|------|------|---------|
| Cookie+HttpOnly | 否 | 按Domain | 4KB | 会话认证 |
| localStorage | 是 | 否 | 5-10MB | 非敏感配置 |
| sessionStorage | 是 | 否 | 5-10MB | 临时会话数据 |
| 内存变量 | 需XSS执行 | 否 | 无限制 | 短期Token |

## Cookie 安全属性

- `HttpOnly`：禁止JS读取，防止XSS窃取Cookie
- `Secure`：仅HTTPS传输，防止中间人攻击
- `SameSite=Lax/Strict`：限制跨站携带，防御CSRF
- `Path=/`：限制Cookie可用路径

## Token 最佳实践

1. 使用短期AccessToken + 长期RefreshToken方案
2. AccessToken存在内存中，RefreshToken存HttpOnly Cookie
3. Token设置合理过期时间，定期刷新
4. 敏感操作（改密码、支付）要求重新验证
5. 登出时服务端销毁Token

## 安全 Token 管理

```javascript
// ========== 安全 Token 管理器 ==========
class SecureTokenManager {
    constructor() {
        this.accessToken = null;
        this.refreshTimer = null;
    }

    // 存储 Token (仅内存)
    setTokens({ accessToken, refreshToken, expiresIn }) {
        this.accessToken = accessToken;

        // 设置自动刷新 (提前1分钟)
        clearTimeout(this.refreshTimer);
        this.refreshTimer = setTimeout(() => {
            this.refreshToken();
        }, (expiresIn - 60) * 1000);

        // RefreshToken 存 HttpOnly Cookie (由服务端设置)
        // 前端不需要也无法读取
    }

    getAccessToken() {
        return this.accessToken;
    }

    async refreshToken() {
        try {
            const res = await fetch('/api/auth/refresh', {
                method: 'POST',
                credentials: 'include', // 携带 HttpOnly Cookie
            });
            if (res.ok) {
                const data = await res.json();
                this.setTokens(data);
            } else {
                this.logout();
            }
        } catch {
            this.logout();
        }
    }

    logout() {
        this.accessToken = null;
        clearTimeout(this.refreshTimer);
        window.location.href = '/login';
    }

    // Token 过期检查
    isTokenExpired() {
        if (!this.accessToken) return true;
        try {
            const payload = JSON.parse(atob(this.accessToken.split('.')[1]));
            return payload.exp * 1000 < Date.now();
        } catch {
            return true;
        }
    }
}

// ========== Fetch 拦截器 ==========
class SecureFetch {
    constructor(tokenManager) {
        this.tokenManager = tokenManager;
    }

    async fetch(url, options = {}) {
        // 检查 Token 是否过期
        if (this.tokenManager.isTokenExpired()) {
            await this.tokenManager.refreshToken();
        }

        const headers = {
            ...options.headers,
            'Authorization': `Bearer ${this.tokenManager.getAccessToken()}`,
        };

        const res = await fetch(url, { ...options, headers });

        // 401 自动重试
        if (res.status === 401) {
            await this.tokenManager.refreshToken();
            headers['Authorization'] = `Bearer ${this.tokenManager.getAccessToken()}`;
            return fetch(url, { ...options, headers });
        }

        return res;
    }
}

// ========== XSS 防护: 输出转义 ==========
function escapeHtml(str) {
    const map = { '&': '&amp;', '<': '&lt;', '>': '&gt;', '"': '&quot;', "'": '&#039;' };
    return String(str).replace(/[&<>"']/g, c => map[c]);
}

// 安全设置文本内容
function safeSetText(element, text) {
    element.textContent = text; // textContent 自动转义
}
// 不要用: element.innerHTML = userInput (XSS 风险)
```

## 存储安全对比

```
// ========== 存储方案选择 ==========
// 敏感数据 (Token/密码):
//   → HttpOnly Cookie + 内存变量
//   → 服务端 Session
//
// 非敏感配置 (主题/语言/偏好):
//   → localStorage 安全
//   → Cookie (非 HttpOnly)
//
// 临时会话数据 (表单草稿):
//   → sessionStorage
//
// 大量数据 (离线缓存):
//   → IndexedDB + Service Worker
```
