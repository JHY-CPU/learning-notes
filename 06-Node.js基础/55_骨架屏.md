## 什么是骨架屏

骨架屏（Skeleton Screen）是在页面内容加载完成前显示的灰色占位界面，模拟页面布局结构，提升用户等待体验。比空白页和Loading转圈体验更好。

## 实现方式

1. **CSS骨架屏**：用灰色渐变+shimmer动画模拟内容占位，轻量高效
2. **SVG骨架屏**：用SVG绘制更精确的骨架形状
3. **图片骨架**：LQIP（低质量图片占位）或SQIP（SVG占位）
4. **自动化生成**：Webpack/Vite插件根据页面结构自动生成骨架HTML

## Vue/React 实现思路

- 配合路由守卫或组件加载状态显示骨架屏
- 用 `v-if` / 条件渲染控制骨架屏和真实内容的切换
- `content-visibility: auto` 配合骨架屏可进一步提升性能

## 最佳实践

- 骨架屏应尽量匹配真实内容的布局结构
- 动画时间控制在1-2秒循环，避免视觉疲劳
- 加载完成后使用淡入淡出过渡，避免闪烁
- 移动端优先考虑骨架屏，减少白屏时间

## 动态骨架屏生成

```javascript
// ========== 自动生成骨架屏 ==========
function generateSkeleton(container) {
    const rect = container.getBoundingClientRect();
    const skeleton = document.createElement('div');
    skeleton.className = 'skeleton-wrapper';
    skeleton.style.cssText = `
        position: absolute;
        top: ${rect.top}px;
        left: ${rect.left}px;
        width: ${rect.width}px;
        height: ${rect.height}px;
        background: #fff;
        z-index: 9999;
    `;

    // 扫描元素生成占位
    const elements = container.querySelectorAll('h1, h2, h3, p, img, button');
    elements.forEach(el => {
        const elRect = el.getBoundingClientRect();
        const placeholder = document.createElement('div');
        placeholder.className = 'skeleton-block';
        placeholder.style.cssText = `
            position: absolute;
            top: ${elRect.top - rect.top}px;
            left: ${elRect.left - rect.left}px;
            width: ${elRect.width}px;
            height: ${elRect.height}px;
            border-radius: ${el.tagName === 'IMG' ? '4px' : '2px'};
        `;
        skeleton.appendChild(placeholder);
    });

    document.body.appendChild(skeleton);
    return skeleton;
}
```

## 骨架屏最佳实践

- **匹配布局**：骨架屏的形状和位置应尽量与真实内容一致
- **动画速度**：1.5-2秒循环，太快导致视觉疲劳，太慢显得卡顿
- **过渡效果**：内容加载完成时使用淡出（opacity 1→0，transition 0.3s）
- **移动端优先**：弱网环境下骨架屏比 loading spinner 体验好得多
- **SSR 配合**：服务端渲染时直接返回骨架屏 HTML，首屏无闪烁
