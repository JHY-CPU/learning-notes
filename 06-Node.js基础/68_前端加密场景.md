## 前端加密 API


```
// ========== 前端密码处理 ==========
// 前端不适合做密码哈希 (防不了中间人)
// ✅ HTTPS 是安全的根本
// ✅ 服务端使用 bcrypt/argon2

// ========== 需要前端加密的场景 ==========
// 1. 端到端加密 (E2EE) — 聊天应用
// 2. 文件加密 — 云存储
// 3. 数字签名 — 文档签署
// 4. 支付卡号 — 局部脱敏

// ========== HTTPS (TLS) 做的事情 ==========
// 1. 传输加密 (防窃听)
// 2. 身份验证 (防假冒)
// 3. 数据完整性 (防篡改)
// 所有 Web Crypto 的前提

// ========== 前端加密误区 ==========
// ❌ "前端哈希密码更安全" — 仍然信任服务器
// ❌ "前端加密 = 数据安全" — 密钥在前端可知
// ❌ "自己实现加密算法" — 使用标准 API
```
