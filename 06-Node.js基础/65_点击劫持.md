# 点击劫持


## 点击劫持


X-Frame-Options (DENY/SAMEORIGIN)、frame-busting、CSP frame-ancestors。


## 点击劫持 API


```
// ========== X-Frame-Options (HTTP头) ==========
X-Frame-Options: DENY           // 禁止任何页面嵌入
X-Frame-Options: SAMEORIGIN     // 仅允许同源页面嵌入
X-Frame-Options: ALLOW-FROM uri // 仅允许指定页面嵌入 (已废弃)

// ========== CSP frame-ancestors (推荐) ==========
Content-Security-Policy: frame-ancestors 'none';
Content-Security-Policy: frame-ancestors 'self';
Content-Security-Policy: frame-ancestors https://trusted.com;

// ========== JS frame-busting (防御) ==========
if (top !== self) {
    top.location.href = self.location.href;
}
```

## 什么是点击劫持

点击劫持（Clickjacking）是指攻击者通过透明iframe将目标网站覆盖在恶意页面上，用户以为在点击可见元素，实际点击的是被隐藏的网站按钮或链接。

## 攻击方式

攻击者用 `<iframe>` 加载目标网站，设置 `opacity: 0` 和高 `z-index`，覆盖在诱骗按钮上方。用户点击时触发的是目标网站的操作（如点赞、转账、授权）。

## 防御方案

1. **X-Frame-Options**：HTTP响应头，`DENY` 禁止被任何页面嵌入，`SAMEORIGIN` 仅允许同源嵌入
2. **CSP frame-ancestors**：更现代的替代方案，支持指定多个来源，优先级高于X-Frame-Options
3. **JS frame-busting**：检测 `top !== self` 后跳出iframe（可被绕过，不推荐单独使用）
4. **验证码/二次确认**：敏感操作要求额外验证

## 最佳实践

- 所有页面都应设置 `X-Frame-Options` 或 `frame-ancestors`
- 推荐使用CSP方案，功能更灵活
- Node.js中使用 `helmet` 中间件可自动设置这些安全头
- SPA应用在入口HTML中设置CSP meta标签

## 完整防护方案

```javascript
// ========== Express 安全头设置 ==========
const helmet = require('helmet');

// 使用 helmet 自动设置安全头
app.use(helmet({
    frameguard: { action: 'deny' },  // X-Frame-Options: DENY
    contentSecurityPolicy: {
        directives: {
            frameAncestors: ["'none'"], // CSP frame-ancestors
        },
    },
}));

// ========== 手动设置 ==========
app.use((req, res, next) => {
    // X-Frame-Options
    res.setHeader('X-Frame-Options', 'DENY');

    // CSP frame-ancestors (推荐，优先级更高)
    res.setHeader('Content-Security-Policy',
        "frame-ancestors 'none'");

    // 其他安全头
    res.setHeader('X-Content-Type-Options', 'nosniff');
    res.setHeader('X-XSS-Protection', '1; mode=block');
    res.setHeader('Referrer-Policy', 'strict-origin-when-cross-origin');
    next();
});

// ========== 允许同源嵌入 ==========
// 某些页面需要被自己的 iframe 嵌入
app.get('/embed/*', (req, res, next) => {
    res.setHeader('X-Frame-Options', 'SAMEORIGIN');
    res.setHeader('Content-Security-Policy', "frame-ancestors 'self'");
    next();
});
```

## 前端防御增强

```javascript
// ========== frame-busting 增强版 ==========
// 基础版可被绕过，增强版更可靠
(function() {
    if (self !== top) {
        // 方法1: 跳出 iframe
        try {
            top.location.href = self.location.href;
        } catch (e) {
            // 跨域时会被阻止
            // 方法2: 隐藏自身内容
            document.documentElement.style.display = 'none';
            // 方法3: 提示用户
            document.write('<h1>此页面不应在iframe中显示</h1>');
        }
    }
})();

// ========== 使用 Intersection Observer 检测覆盖 ==========
// 检测是否有透明元素覆盖在关键按钮上
function detectOverlay(button) {
    const observer = new IntersectionObserver((entries) => {
        entries.forEach(entry => {
            if (entry.intersectionRatio < 1) {
                console.warn('检测到可能的点击劫持覆盖!');
            }
        });
    }, { threshold: [1.0] });
    observer.observe(button);
}
```
