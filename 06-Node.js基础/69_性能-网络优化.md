## 为什么要做网络优化

网络请求是Web性能的主要瓶颈。减少请求数、压缩传输体积、利用缓存是优化的三大方向。

## 资源压缩

- **Gzip**：通用压缩，所有服务器和浏览器支持，压缩率约60-80%
- **Brotli**：Google推出，压缩率比Gzip高20-30%，现代浏览器均支持
- **图片优化**：WebP比JPEG小25-35%，AVIF更小但兼容性稍差

## 缓存策略

- **强缓存**：`Cache-Control: max-age=31536000` 配合文件hash命名
- **协商缓存**：`ETag` / `Last-Modified` 用于HTML等需验证的资源
- **Service Worker**：可编程控制缓存策略

## HTTP/2 优化

- 多路复用：单个TCP连接并行传输多个请求，解决队头阻塞
- 头部压缩：HPACK算法压缩请求头
- 服务器推送：主动推送关联资源（注意谨慎使用）
- 二进制分帧：解析更高效

## 实践建议

1. 静态资源使用CDN + 长期缓存 + 文件hash
2. 关键CSS内联，非关键CSS异步加载
3. 图片使用 `loading="lazy"` 懒加载
4. 减少第三方脚本数量和体积
5. 使用 `preconnect` / `dns-prefetch` 提前建立连接

## 资源提示与预加载

```html
<!-- ========== 资源提示标签 ========== -->
<!-- DNS 预解析 -->
<link rel="dns-prefetch" href="https://api.example.com">

<!-- 预连接 (DNS + TCP + TLS) -->
<link rel="preconnect" href="https://cdn.example.com">

<!-- 预加载关键资源 -->
<link rel="preload" href="/fonts/main.woff2" as="font" crossorigin>
<link rel="preload" href="/css/critical.css" as="style">
<link rel="preload" href="/js/app.js" as="script">

<!-- 预获取下一页面资源 -->
<link rel="prefetch" href="/next-page.js">

<!-- 预渲染 (实验性) -->
<link rel="prerender" href="/likely-next-page">
```

## 性能监控

```javascript
// ========== Core Web Vitals ==========
function reportWebVitals() {
    // LCP (Largest Content Paint) - 最大内容绘制 < 2.5s
    new PerformanceObserver((list) => {
        const entries = list.getEntries();
        const lastEntry = entries[entries.length - 1];
        console.log('LCP:', lastEntry.startTime);
    }).observe({ type: 'largest-contentful-paint', buffered: true });

    // FID (First Input Delay) - 首次输入延迟 < 100ms
    new PerformanceObserver((list) => {
        list.getEntries().forEach(entry => {
            console.log('FID:', entry.processingStart - entry.startTime);
        });
    }).observe({ type: 'first-input', buffered: true });

    // CLS (Cumulative Layout Shift) - 累积布局偏移 < 0.1
    let clsValue = 0;
    new PerformanceObserver((list) => {
        list.getEntries().forEach(entry => {
            if (!entry.hadRecentInput) clsValue += entry.value;
        });
        console.log('CLS:', clsValue);
    }).observe({ type: 'layout-shift', buffered: true });

    // TTFB (Time to First Byte)
    const nav = performance.getEntriesByType('navigation')[0];
    console.log('TTFB:', nav.responseStart - nav.requestStart);
}

// ========== 网络资源加载监控 ==========
function monitorResources() {
    const resources = performance.getEntriesByType('resource');
    resources.forEach(r => {
        console.log(`${r.name.split('/').pop()}: ${r.duration.toFixed(0)}ms (${(r.transferSize/1024).toFixed(1)}KB)`);
    });
}
```
