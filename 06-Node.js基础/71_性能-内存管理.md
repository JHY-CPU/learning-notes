## 内存管理 API


```
// ========== 垃圾回收 (GC) ==========
// V8 使用: 分代回收 (Generational GC)
// 新生代: Scavenge (频繁, 小)
// 老生代: Mark-Sweep / Mark-Compact (不频繁, 大)

// ========== 内存泄漏常见原因 ==========
// 1. 全局变量 (意外)
// 2. 闭包引用未释放
// 3. 定时器未清理
// 4. DOM 引用未释放
// 5. 事件监听未移除
// 6. console.log (保留对象引用)

// ========== WeakMap / WeakSet ==========
// 弱引用: 不影响 GC
// 适合: 对象额外数据 / 缓存 / DOM 关联数据
```

## V8 垃圾回收机制

V8采用分代垃圾回收：**新生代**使用Scavenge算法（复制存活对象），**老生代**使用标记-清除（Mark-Sweep）和标记-整理（Mark-Compact）。新生代空间小但回收频繁，老生代空间大但回收不频繁。

## 内存泄漏常见原因

1. **意外全局变量**：未声明的变量挂载到 `window`
2. **闭包未释放**：闭包中引用了大对象且闭包未被销毁
3. **定时器未清理**：`setInterval` 中引用外部变量，忘记 `clearInterval`
4. **DOM引用**：JS变量保存了已删除DOM的引用
5. **事件监听**：添加了监听但未移除（尤其在SPA组件销毁时）
6. **console.log**：DevTools打开时，console保留对象引用阻止GC

## 检测工具

- Chrome DevTools Memory面板：Heap Snapshot、Allocation Timeline
- `performance.memory` API：JS堆内存使用情况
- `process.memoryUsage()`：Node.js内存监控

## WeakMap/WeakSet

弱引用集合，不阻止GC回收键对象。适合用于对象的附加数据存储、DOM关联数据、缓存场景。键必须是对象，不可遍历，只支持基本操作。
