# 调试技巧


## 调试技巧


Console 高级(assert/table/group/time/trace)、Source Map、断点调试。


## 调试 API


```
// ========== Console 高级 ==========
console.assert(condition, msg);   // 条件断言
console.table(data);              // 表格展示
console.group('Label');           // 分组
console.time('label');            // 计时开始
console.timeEnd('label');         // 计时结束
console.trace();                  // 堆栈跟踪
console.count('label');           // 计数
console.dir(obj);                 // 对象展开
console.dirxml(element);          // DOM 展开

// ========== 断点 ==========
// 1. 代码断点: debugger;
// 2. 条件断点: 右键添加条件
// 3. DOM 断点: 元素 → Break on
// 4. XHR 断点: Sources → XHR Breakpoints
```

## Console 高级用法

浏览器Console不仅仅是 `console.log`，还有许多高效调试技巧。

- `console.assert(cond, msg)`：条件为false时输出，适合断言检查
- `console.table(data)`：以表格形式展示数组/对象，适合查看结构化数据
- `console.group(label)` / `console.groupEnd()`：分组输出，组织复杂日志
- `console.time(label)` / `console.timeEnd(label)`：精确测量代码执行时间
- `console.trace()`：输出当前调用栈，追踪函数调用链
- `console.count(label)`：统计某代码执行次数
- `console.dir(obj)`：以交互式树状展示对象

## 断点类型

1. **代码断点**：`debugger;` 语句或DevTools Sources面板点击行号
2. **条件断点**：右键行号添加条件，只在条件满足时暂停
3. **DOM断点**：Elements面板 → 右键元素 → Break on → subtree modifications/attribute modifications/node removal
4. **事件断点**：Sources面板 → Event Listener Breakpoints → 勾选事件类型
5. **XHR断点**：Sources面板 → XHR/fetch Breakpoints → 填入URL片段
6. **异常断点**：Sources面板 → Pause on exceptions → Pause on caught exceptions

## Source Map 调试

打包后的代码难以调试，Source Map将压缩代码映射回源码。开发环境开启完整Source Map，生产环境使用隐藏Source Map（`hidden-source-map`）仅供内部排查。

## Node.js 调试

```bash
node --inspect app.js          # 启动调试
node --inspect-brk app.js      # 首行暂停
# Chrome 地址栏输入 chrome://inspect 连接
```
