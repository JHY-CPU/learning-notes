dotenv 库、process.env、.env 文件、环境区分 (dev/prod)。


## 环境变量管理


```
// ========== process.env ==========
// 所有环境变量
console.log(process.env.NODE_ENV);
console.log(process.env.PORT);
console.log(process.env.DB_URL);

// 设置环境变量 (运行时)
// $ NODE_ENV=production PORT=3000 node app.js

// ========== 环境区分 ==========
// 常见环境:
// development — 开发
// production  — 生产
// test        — 测试
// staging     — 预发布

const config = {
    development: {
        port: 3000,
        db: 'mongodb://localhost/dev',
    },
    production: {
        port: process.env.PORT || 8080,
        db: process.env.DB_URL,
    },
};

const env = process.env.NODE_ENV || 'development';
const appConfig = config[env];

// ========== .env 文件 ==========
// .env           — 通用配置
// .env.local     — 本地覆盖 (不提交 git)
// .env.development — 开发环境
// .env.production  — 生产环境

// .env 示例:
// PORT=3000
// DB_HOST=localhost
// DB_USER=admin
// DB_PASS=secret123

// ========== dotenv 加载 ==========
require('dotenv').config();                 // 加载 .env
require('dotenv').config({ path: '.env.local' }); // 指定路径
require('dotenv').config({ path: `.env.${process.env.NODE_ENV}` });

// 访问
const port = process.env.PORT;
const dbUrl = `mongodb://${process.env.DB_HOST}:27017/myapp`;
```


> **Note:** ⚠️ .env 文件包含敏感信息，不要提交到 git。
>
>
> 💡 提交 .env.example 作为模板。
>
>
> 💡 Node 18+ 支持 --env-file 参数。

## Node.js 18+ 原生 .env 支持

```javascript
// ========== Node 20.6+ 内置 .env 支持 ==========
// 运行: node --env-file=.env app.js
// 无需安装 dotenv!

// ========== 多环境配置 ==========
// 运行: node --env-file=.env --env-file=.env.local app.js
// 后面的文件覆盖前面的同名变量

// ========== 代码中判断环境 ==========
function isProduction() {
    return process.env.NODE_ENV === 'production';
}

function isDevelopment() {
    return process.env.NODE_ENV === 'development';
}

function isTest() {
    return process.env.NODE_ENV === 'test';
}
```

## 配置管理最佳实践

```javascript
// ========== 类型安全的配置管理 ==========
// config.js
function loadConfig() {
    const required = ['DB_HOST', 'DB_USER', 'DB_PASS'];

    // 检查必要环境变量
    for (const key of required) {
        if (!process.env[key]) {
            throw new Error(`缺少必要环境变量: ${key}`);
        }
    }

    return {
        env: process.env.NODE_ENV || 'development',
        port: parseInt(process.env.PORT, 10) || 3000,
        database: {
            host: process.env.DB_HOST,
            user: process.env.DB_USER,
            password: process.env.DB_PASS,
            name: process.env.DB_NAME || 'myapp',
            port: parseInt(process.env.DB_PORT, 10) || 5432,
        },
        redis: {
            url: process.env.REDIS_URL || 'redis://localhost:6379',
        },
        jwt: {
            secret: process.env.JWT_SECRET || 'dev-secret-change-me',
            expiresIn: process.env.JWT_EXPIRES_IN || '7d',
        },
        cors: {
            origin: process.env.CORS_ORIGIN || '*',
        },
        log: {
            level: process.env.LOG_LEVEL || 'info',
        },
    };
}

// 冻结配置，防止运行时修改
const config = Object.freeze(loadConfig());
module.exports = config;
```

## .env 文件管理

```bash
# ========== .env 文件 ==========
# 通用配置
NODE_ENV=development
PORT=3000

# 数据库
DB_HOST=localhost
DB_USER=dev_user
DB_PASS=dev_password
DB_NAME=myapp_dev

# Redis
REDIS_URL=redis://localhost:6379

# JWT
JWT_SECRET=dev-jwt-secret-key-change-in-production
JWT_EXPIRES_IN=7d

# 第三方服务
AWS_REGION=us-east-1
S3_BUCKET=my-bucket
```

```gitignore
# .gitignore 中
.env
.env.local
.env.*.local
# 保留模板文件
!.env.example
```

```bash
# ========== .env.example (提交到 git) ==========
NODE_ENV=
PORT=
DB_HOST=
DB_USER=
DB_PASS=

REDIS_URL=
JWT_SECRET=
```

## 环境变量安全

- **不要提交 .env 文件到 git**：使用 `.gitignore` 排除
- **使用 .env.example 作为模板**：列出所有变量名但不填值
- **生产环境使用密钥管理服务**：如 AWS Secrets Manager、HashiCorp Vault
- **限制访问权限**：确保 `.env` 文件权限为 `600`（仅所有者可读写）
- **定期轮换密钥**：定期更换数据库密码、API 密钥等
- **使用前验证**：启动时检查所有必要环境变量是否存在
