# 04-低功耗设计

低功耗设计是嵌入式系统和物联网设备的核心挑战之一。电池供电设备期望数月甚至数年的工作寿命，这要求从硬件架构、电路设计到软件策略全方位优化功耗。本章系统介绍低功耗设计的原理、方法与实践。

---

## 1. 功耗的组成

CMOS电路的总功耗由三部分组成：

### 1.1 动态功耗

信号翻转对负载电容充放电产生的功耗，是主要功耗来源：

```
P_dynamic = α × C × V² × f

其中：
α = 活动因子（信号翻转概率，0~1）
C = 负载电容
V = 供电电压
f = 时钟频率
```

关键观察：
- 功耗与电压的**平方**成正比（降压效果最显著）
- 功耗与频率成正比
- 功耗与活动因子成正比

### 1.2 静态功耗（泄漏功耗）

即使电路不工作，MOS管的亚阈值泄漏电流和栅极泄漏电流也会产生功耗：

```
P_static = V × I_leak
```

- 随着工艺节点缩小（纳米级），泄漏电流急剧增加
- 温度升高会加剧泄漏
- 是深亚微米工艺下越来越重要的功耗来源

### 1.3 短路功耗

PMOS和NMOS在信号翻转瞬间同时导通产生的穿通电流：

- 占总功耗的10%-15%
- 通过提高信号边沿斜率可减小

### 1.4 功耗分布示例

```
典型CMOS电路功耗分布：
动态功耗    ████████████████████████  ~70-80%
短路功耗    ████████                  ~10-15%
静态功耗    ██████                    ~10-15%（先进工艺下占比增大）
```

## 2. 低功耗设计层次

低功耗设计是一个多层次的系统工程：

| 层次 | 方法 | 功耗优化潜力 |
|------|------|------------|
| 系统级 | 动态电源管理、任务调度、分区供电 | 最大 |
| 架构级 | 时钟门控、电源门控、DVFS | 大 |
| RTL级 | 操作数隔离、门控时钟插入 | 中 |
| 电路级 | 多阈值电压、低泄漏库 | 小 |
| 工艺级 | FinFET、FD-SOI | 由制造决定 |

设计原则：越高层级的设计，优化空间越大，实施越早效果越好。

## 3. 动态电压频率调节（DVFS）

DVFS根据实时负载动态调整处理器的电压和频率：

### 3.1 原理

```
高负载时：V = 1.2V, f = 200MHz → 高性能
低负载时：V = 0.9V, f = 50MHz  → 低功耗

功耗比 = (1.2² × 200) / (0.9² × 50) = 288 / 40.5 ≈ 7.1倍节省
```

### 3.2 实现方式

- **软件控制**：根据CPU利用率或任务需求调整OPP（Operating Performance Point）
- **硬件自动**：根据负载传感器自动调节
- **支持DVFS的MCU**：STM32L4的电压调节、ESP32的频率调节

### 3.3 OPP表

| 模式 | 电压 | 频率 | 功耗 | 用途 |
|------|------|------|------|------|
| Turbo | 1.3V | 240MHz | 200mW | 密集计算 |
| Normal | 1.1V | 160MHz | 80mW | 常规任务 |
| Economy | 0.95V | 80MHz | 25mW | 后台处理 |
| Idle | 0.8V | 40MHz | 8mW | 最低性能 |

## 4. 时钟门控（Clock Gating）

### 4.1 原理

当模块不需要工作时，关闭其时钟信号，消除动态功耗：

```
        使能信号
           │
           ▼
时钟 ──→ [AND门] ──→ 门控时钟 ──→ 功能模块
```

### 4.2 实现方式

- **全局门控**：整个外设模块的时钟关闭（MCU外设时钟使能/禁止）
- **局部门控**：模块内部子单元的时钟控制
- **MCU中的应用**：STM32的RCC时钟使能寄存器

### 4.3 代码示例（STM32）

```c
// 关闭未使用外设的时钟
__HAL_RCC_GPIOB_CLK_DISABLE();
__HAL_RCC_TIM3_CLK_DISABLE();
__HAL_RCC_USART2_CLK_DISABLE();

// 需要使用时再使能
__HAL_RCC_TIM3_CLK_ENABLE();
```

## 5. 电源门控（Power Gating）

### 5.1 原理

完全切断某个电路模块的供电，彻底消除静态和动态功耗：

```
电源 VDD
  │
[电源开关 (Header/Footer)]
  │
  ▼
功能模块（睡眠时断电）
  │
[隔离单元] ← 保证输出信号确定
  │
  ▼
下一级电路
```

### 5.2 实现要点

- **电源开关**：PMOS Header或NMOS Footer管
- **隔离单元**：防止断电模块的输出浮动影响其他模块
- **状态保持**：使用Retention寄存器保存关键状态
- **唤醒延迟**：从断电到正常工作需要稳定时间

### 5.3 MCU中的应用

- 核心电压域与IO电压域分离
- 独立供电的域可单独关断
- 例如：STM32的VDDA（模拟电源）可独立控制

## 6. 休眠模式与唤醒机制

### 6.1 通用休眠模型

```
活跃 ──→ 浅休眠 ──→ 深休眠 ──→ 关断
 │          │          │         │
CPU运行   CPU停止    RAM保留   RAM丢失
外设运行   部分外设   极少外设   全部断电
时钟正常   低速时钟   仅RTC     无
唤醒:即时  唤醒:μs   唤醒:ms   唤醒:冷启动
```

### 6.2 唤醒源

- **外部中断**：GPIO引脚电平/边沿变化
- **定时器**：RTC闹钟、低功耗定时器
- **看门狗**：周期性唤醒
- **通信事件**：UART接收、BLE连接事件
- **传感器**：加速度计、光传感器触发

### 6.3 唤醒策略

- **周期唤醒**：定时采集数据，处理后继续休眠
- **事件驱动唤醒**：仅在特定事件发生时唤醒
- **混合策略**：周期唤醒 + 事件唤醒结合

## 7. 低功耗MCU的功耗模式

### 7.1 STM32L4功耗模式

| 模式 | CPU | RAM | Flash | 时钟 | 典型功耗 | 唤醒时间 |
|------|-----|-----|-------|------|---------|---------|
| Run | 运行 | 可访问 | 可访问 | 全部 | 36μA/MHz | — |
| LPRun | 运行 | 可访问 | 可访问 | 仅MSI | 14μA/MHz | — |
| Sleep | 停止 | 可访问 | 可访问 | 全部 | 12μA/MHz | 0.3μs |
| LPSleep | 停止 | 可访问 | 可访问 | 仅MSI | 5μA/MHz | 0.3μs |
| Stop 0/1/2 | 停止 | 保留 | 关闭 | LSE/LSI | 3.2/1.1/0.6μA | 5/5/8μs |
| Standby | 停止 | 丢失 | 关闭 | LSE/LSI | 0.3μA | 30μs |
| Shutdown | 停止 | 丢失 | 关闭 | LSE | 0.03μA | 1ms |

### 7.2 ESP32功耗模式

| 模式 | WiFi/BLE | CPU | 功耗 |
|------|-----------|-----|------|
| Active | 开启 | 运行 | 160-240mA |
| Modem Sleep | WiFi关闭 | 运行 | 20mA |
| Light Sleep | WiFi关闭 | 暂停 | 0.8mA |
| Deep Sleep | 全部关闭 | 暂停 | 10μA |
| Hibernation | 全部关闭 | 暂停 | 5μA |

### 7.3 功耗模式选择原则

1. **尽量长时间停留在最低功耗模式**
2. **减少唤醒频率**：合并多次采集为一次批量处理
3. **缩短唤醒时间**：选择适当的休眠深度
4. **按需配置外设**：不需要的外设关闭时钟甚至断电

## 8. 低功耗通信协议选择

### 8.1 通信功耗对比

| 技术 | 发送功耗 | 接收功耗 | 休眠功耗 | 适用场景 |
|------|---------|---------|---------|---------|
| WiFi | 200-500mA | 50-100mA | ~10μA | 高带宽、持续供电 |
| BLE | 5-15mA | 5-10mA | ~1μA | 短距离、间歇通信 |
| ZigBee | 20-35mA | 20-25mA | ~1μA | Mesh网络 |
| LoRa | 20-120mA | 10mA | ~1μA | 长距离、极低频 |
| NB-IoT | 100-220mA | 30-50mA | ~3μA | 蜂窝覆盖、低频 |

### 8.2 降低通信功耗的策略

- **减少传输次数**：数据压缩、批量发送
- **减少传输数据量**：CBOR替代JSON、二进制协议
- **选择合适的协议**：根据数据量和频率选择
- **利用广播/组播**：一对多通信减少总传输量
- **自适应速率**：根据链路质量调整发射功率

## 9. 能量收集技术

能量收集（Energy Harvesting）从环境中获取能量为设备供电：

### 9.1 能量来源

| 来源 | 功率密度 | 技术 | 适用场景 |
|------|---------|------|---------|
| 太阳能（室内） | 10-100μW/cm² | 光伏电池 | 室内传感器 |
| 太阳能（室外） | 10-100mW/cm² | 光伏电池 | 户外设备 |
| 振动 | 1-200μW/cm³ | 压电/电磁 | 工业设备 |
| 温差 | 1-10μW/cm² | 热电偶（TEG） | 管道、发动机 |
| RF射频 | 0.1-1μW/cm² | 天线整流 | RFID、远距供电 |
| 风能 | 1-10mW/cm² | 微型风机 | 户外传感器 |

### 9.2 能量收集系统架构

```
能量源 → 收集器 → 功率管理IC（MPPT + 充电管理）→ 储能元件（超级电容/电池）→ 负载
                                                              │
                                                    超低功耗MCU + 传感器
```

### 9.3 设计要点

- **储能管理**：超级电容快充快放，电池慢充慢放
- **MPPT（最大功率点跟踪）**：优化能量采集效率
- **间歇计算**：能量不足时保存状态，能量恢复后继续执行
- **BOLT（Burst-Of-Loose-Timing）**：能量驱动的异步执行模型

## 10. 电池管理基础

### 10.1 常见电池类型

| 类型 | 电压 | 能量密度 | 特点 | 应用 |
|------|------|---------|------|------|
| 锂离子(Li-ion) | 3.7V | 150-250Wh/kg | 可充电，需保护电路 | 手机、笔记本 |
| 锂聚合物(LiPo) | 3.7V | 150-200Wh/kg | 可弯曲，安全性好 | 可穿戴设备 |
| 磷酸铁锂(LiFePO4) | 3.2V | 90-120Wh/kg | 高安全，长寿命 | 储能系统 |
| 镍氢(NiMH) | 1.2V | 60-120Wh/kg | 环保，记忆效应小 | 遥控器、玩具 |
| 纽扣电池(CR2032) | 3V | — | 一次性，超低自放电 | BLE标签、RTC |

### 10.2 电池管理IC功能

- **充放电管理**：CC-CV充电曲线、充放电保护
- **电量计量（Fuel Gauge）**：SOC（荷电状态）估算
- **保护电路**：过充、过放、过流、短路保护
- **电池均衡**：多串电池组的电压均衡

### 10.3 电池寿命估算

```
电池寿命 ≈ 电池容量(mAh) / 平均电流(mA)

示例：CR2032纽扣电池（225mAh），平均电流10μA
寿命 ≈ 225 / 0.01 = 22500小时 ≈ 2.6年
```

实际寿命还需考虑：
- 电池自放电率（每年1-3%）
- 温度对容量的影响
- 峰值电流对电池内阻的要求

## 11. 低功耗软件优化技巧

### 11.1 代码层面

- **避免忙等待**：使用中断或RTOS信号量替代轮询
- **使用查表替代计算**：减少CPU运算时间
- **批量处理数据**：减少唤醒次数
- **使用DMA**：CPU可休眠，由DMA完成数据搬运
- **编译器优化**：开启-O2/-Os优化，减少指令数

### 11.2 外设管理

- **按需使能外设**：用完即关
- **降低采样率**：满足需求的最低采样频率
- **使用外设自主运行模式**：如ADC定时采样+DMA，无需CPU介入
- **关闭未使用引脚的内部上下拉**

### 11.3 通信优化

- **减少广播/扫描频率**
- **使用连接间隔上限**
- **数据压缩后再发送**
- **合理使用MQTT Keep Alive间隔**

### 11.4 存储器优化

- **减少Flash写入**：Flash写入功耗远高于读取
- **使用EEPROM或FRAM存储频繁更新的数据**
- **数据缓存在RAM中，批量写入Flash**

### 11.5 系统级策略

```c
// 典型低功耗应用主循环
int main(void) {
    system_init();
    
    while (1) {
        // 1. 唤醒：采集传感器数据
        sensor_read_all();
        
        // 2. 处理：压缩、判断是否需要上报
        if (need_report()) {
            // 3. 通信：发送数据
            radio_send(data, len);
        }
        
        // 4. 计算下次唤醒时间
        uint32_t next_wakeup = calculate_next_wakeup();
        
        // 5. 进入休眠
        enter_deep_sleep(next_wakeup);
    }
}
```

- **低功耗模式验证**：确认芯片确实进入目标功耗模式
- **GPIO状态检查**：未配置的GPIO可能导致漏电
- **外设时钟检查**：确认未使用外设的时钟已关闭

## 总结

低功耗设计是嵌入式系统的核心竞争力。从理解功耗组成（动态、静态、短路）开始，在系统级选择合适的功耗模式和通信策略，在架构级利用DVFS、时钟门控和电源门控，在软件层面优化代码执行效率和唤醒策略，形成完整的低功耗设计方法论。能量收集和先进的电池管理技术进一步延长了设备的自主运行时间，是物联网大规模部署的关键使能技术。
