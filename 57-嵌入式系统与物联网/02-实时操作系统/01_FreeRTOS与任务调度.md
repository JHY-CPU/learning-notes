# FreeRTOS与任务调度


## FreeRTOS与任务调度

一、RTOS概述

## 一、RTOS概述


实时操作系统（RTOS）是专为实时应用设计的操作系统，能够在确定的时间内响应外部事件。


### 1.1 实时系统分类


| 类型 | 特点 | 示例 |
| --- | --- | --- |
| 硬实时 | 截止时间必须满足，否则系统失效 | 飞行控制、ABS刹车 |
| 软实时 | 截止时间尽量满足，偶尔超时可接受 | 视频播放、语音通话 |
| 固实时 | 偶尔超时可接受，但频繁超时不可接受 | 工业控制 |


### 1.2 RTOS核心特性


- **确定性：**
   任务调度延迟可预测
- **抢占式调度：**
   高优先级任务可抢占低优先级任务
- **低延迟：**
   中断响应和上下文切换时间短
- **多任务：**
   支持多个并发执行的任务

二、任务管理

## 二、任务管理


### 2.1 任务状态


| 状态 | 说明 | 转换条件 |
| --- | --- | --- |
| 运行态（Running） | 正在使用CPU执行 | 被调度器选中 |
| 就绪态（Ready） | 等待CPU资源 | 等待调度 |
| 阻塞态（Blocked） | 等待事件或资源 | 等待延时/信号量/队列 |
| 挂起态（Suspended） | 被显式挂起 | vTaskSuspend() |

**状态转换：**


就绪 → 运行：被调度器选中


运行 → 就绪：时间片用完或被高优先级抢占


运行 → 阻塞：调用vTaskDelay()或等待信号量


运行 → 挂起：调用vTaskSuspend()


阻塞/挂起 → 就绪：延时到期/事件发生/vTaskResume()

### 2.2 任务创建


```
// FreeRTOS任务创建
void vTaskFunction(void *pvParameters) {
    for (;;) {
        // 任务逻辑
        vTaskDelay(pdMS_TO_TICKS(100)); // 延时100ms
    }
}

// 创建任务
xTaskCreate(
    vTaskFunction,    // 任务函数
    "MyTask",         // 任务名称
    128,              // 栈大小（字）
    NULL,             // 参数
    2,                // 优先级
    NULL              // 任务句柄
);
```

三、调度策略

## 三、调度策略


### 3.1 FreeRTOS调度方式


| 调度方式 | 特点 | 配置 |
| --- | --- | --- |
| 抢占式调度 | 高优先级就绪立即抢占 | `configUSE_PREEMPTION = 1` |
| 协作式调度 | 任务主动让出CPU | `configUSE_PREEMPTION = 0` |
| 时间片轮转 | 同优先级任务轮流执行 | `configUSE_TIME_SLICING = 1` |


### 3.2 常见调度算法


| 算法 | 特点 | 适用场景 |
| --- | --- | --- |
| 固定优先级（RMS） | 周期越短优先级越高 | 周期性任务 |
| 最早截止优先（EDF） | 截止时间越近优先级越高 | 动态实时系统 |
| 最低松弛度优先（LLF） | 松弛度越小优先级越高 | 复杂实时系统 |

四、同步机制

## 四、同步与通信机制


### 4.1 信号量（Semaphore）


| 类型 | 用途 | 特点 |
| --- | --- | --- |
| 二值信号量 | 任务同步 | 只有0和1两个值 |
| 计数信号量 | 资源管理 | 计数值表示可用资源数 |
| 互斥信号量（Mutex） | 临界区保护 | 支持优先级继承 |


> **Note:** **优先级继承：**
> 当高优先级任务等待低优先级任务持有的Mutex时，低优先级任务临时提升到高优先级，以减少优先级反转的影响。


### 4.2 队列（Queue）


队列是FreeRTOS中任务间通信的主要机制，支持先进先出的数据传递。


```
// 创建队列
QueueHandle_t xQueue = xQueueCreate(10, sizeof(int));

// 发送数据
int data = 42;
xQueueSend(xQueue, &data, portMAX_DELAY);

// 接收数据
int received;
xQueueReceive(xQueue, &received, portMAX_DELAY);
```


### 4.3 事件组（Event Group）


- 允许任务等待多个事件的任意组合
- 支持"与"（所有事件都发生）和"或"（任一事件发生）等待
- 适合复杂的多事件同步场景


### 4.4 软件定时器


- 一次性定时器和周期定时器
- 回调函数在定时器服务任务中执行
- 精度受定时器服务任务优先级影响

五、内存管理

## 五、FreeRTOS内存管理


| 方案 | 特点 |
| --- | --- |
| heap_1 | 只分配不释放，适用于任务创建后不再删除的场景 |
| heap_2 | 支持分配和释放，但不合并相邻空闲块 |
| heap_3 | 封装标准malloc/free，线程安全 |
| heap_4 | 支持合并相邻空闲块，减少碎片 |
| heap_5 | 支持跨多个不连续内存区域 |
