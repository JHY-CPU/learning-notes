LoRa与NB-IoT

一、低功耗广域网（LPWAN）概述

低功耗广域网（LPWAN）是专为物联网设计的无线通信技术，具有低功耗、长距离、低速率的特点。LoRa和NB-IoT是两种最主流的LPWAN技术。

二、LoRa（Long Range）

LoRa是一种基于线性调频扩频（CSS）调制的物理层技术，由Semtech公司拥有核心专利。LoRa定义了物理层，LoRaWAN定义了上层协议。

2.1 CSS（Chirp Spread Spectrum）调制

CSS调制是LoRa的核心技术，通过线性扫频信号来传输数据。

CSS调制原理：
- 信号频率在符号周期内线性变化（从低到高或从高到低）
- "Chirp" = 啁啾信号，频率随时间线性变化
- 通过起始频率编码数据
- 对多径衰落和多普勒效应具有很强的鲁棒性
- 可以在噪声以下解码（负SNR灵敏度）

2.2 扩频因子（Spreading Factor, SF）

扩频因子是LoRa最重要的参数，决定了传输距离、速率和功耗的平衡。

SF权衡：SF每增加1，传输时间翻倍，数据率减半，但传输距离增加约1.5倍（~3dB增益）。不同SF的信号相互正交，不会互相干扰。

三、LoRaWAN网络架构

3.1 网络组件

组件说明
终端设备（End Device）传感器节点，采集数据并通过LoRa发送
网关（Gateway）接收LoRa信号，通过IP网络转发到服务器
网络服务器（Network Server）管理网络、转发数据、去重、ADR
应用服务器（Application Server）处理应用数据和业务逻辑

3.2 设备类别

类别接收窗口下行延迟功耗应用场景
Class A上行后开2个取决于上行最低传感器、抄表
Class B周期性开启有界延迟中等需要定时下行的设备
Class C常开（除发送外）最低延迟最高供电充足的执行器

3.3 自适应数据速率（ADR）

ADR机制：
网络服务器根据终端的信号质量，动态调整其SF、TX功率和数据速率。
- 信号好 → 降低SF（提高速率，缩短airtime）
- 信号差 → 提高SF（降低速率，增大距离）
- 目标：最大化网络容量，延长设备电池寿命

四、NB-IoT（窄带物联网）

NB-IoT是3GPP标准（Release 13+）定义的蜂窝物联网技术，利用现有的LTE频谱资源，由运营商部署和管理。

4.1 NB-IoT技术特点

  • 频段：使用运营商授权频段（如Band 8、Band 20等）
  • 带宽：仅需200kHz（一个LTE PRB）
  • 部署模式：带内（In-band）、保护带（Guard-band）、独立部署（Standalone）
  • 覆盖增强：MCL（最大耦合损耗）达164dB，比GSM强20dB
  • 设备密度：每小区支持5万~10万个设备
  • 电池寿命：目标10年以上（AA电池）

4.2 覆盖增强技术

NB-IoT覆盖增强原理：
- 重复传输（Repetition）：将同一数据重复发送128次甚至更多
- 带来最大23dB的覆盖增强（比GSM强100倍）
- 代价是延迟增加和有效速率降低
- 适合地下室、偏远地区等弱信号场景

4.3 省电模式（PSM & eDRX）

机制原理功耗
PSM（Power Saving Mode）进入深度睡眠，不监听寻呼~3μA
eDRX延长寻呼监听周期（最长~3小时）介于PSM和正常DRX之间
正常DRX周期性监听寻呼~6mA

选择建议：
- 需要自主部署网络、无运营商依赖 → LoRa
- 需要广覆盖、快速部署 → NB-IoT
- 数据量大、需要上行高速率 → NB-IoT
- 成本敏感、大量设备 → LoRa（无月租费）
- 需要双向低延迟控制 → NB-IoT
- 园区/农场私有网络 → LoRa
