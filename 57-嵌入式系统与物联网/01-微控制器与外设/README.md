# 01-微控制器与外设

微控制器（Microcontroller Unit, MCU）是嵌入式系统的核心，将CPU、存储器、外设集成在单一芯片上，构成一个完整的微型计算机系统。本章系统介绍微控制器的架构原理与常用外设。

---

## 1. 嵌入式系统概述

### 1.1 定义与特点

嵌入式系统是以应用为中心、以计算机技术为基础，软硬件可裁剪的专用计算机系统。与通用计算机相比，嵌入式系统具有以下特点：

- **专用性强**：面向特定应用，功能专一
- **资源受限**：处理器速度、存储容量、功耗预算有限
- **实时性要求高**：许多场景需要在确定时间内完成响应
- **可靠性要求高**：长期无人值守运行，需具备容错能力
- **功耗敏感**：电池供电设备对功耗有严格限制

### 1.2 应用领域

| 领域 | 典型应用 |
|------|----------|
| 消费电子 | 智能手机、智能手表、家电控制器 |
| 汽车电子 | ECU（发动机控制）、ADAS、车身电子 |
| 工业控制 | PLC、电机控制、传感器网络 |
| 医疗设备 | 血糖仪、心脏起搏器、便携监护仪 |
| 物联网 | 智能家居、环境监测、远程抄表 |

## 2. 微控制器架构

### 2.1 ARM Cortex-M系列

ARM Cortex-M是目前最流行的嵌入式处理器内核系列，主要包括：

- **Cortex-M0/M0+**：最低功耗、最小面积，适合简单控制任务
- **Cortex-M3**：性能与功耗平衡，支持Thumb-2指令集
- **Cortex-M4**：在M3基础上增加DSP指令和可选FPU
- **Cortex-M7**：高性能，支持双精度FPU和指令/数据缓存
- **Cortex-M33**：引入TrustZone安全扩展

### 2.2 CPU核心与存储器映射

ARM Cortex-M采用冯·诺依曼与哈佛混合架构，程序存储器与数据存储器共享4GB地址空间：

```
地址空间分配（4GB）：
0x0000_0000 - 0x1FFF_FFFF  代码区（Flash/ROM）
0x2000_0000 - 0x3FFF_FFFF  SRAM区
0x4000_0000 - 0x5FFF_FFFF  外设区
0x6000_0000 - 0x9FFF_FFFF  外部RAM
0xA000_0000 - 0xDFFF_FFFF  外部设备
0xE000_0000 - 0xE00F_FFFF  系统控制空间（SCS，含NVIC、SysTick等）
```

关键寄存器：
- **NVIC**（嵌套向量中断控制器）：管理中断使能、优先级、挂起状态
- **SysTick**：24位系统定时器，为RTOS提供时基
- **SCB**（系统控制块）：配置系统级行为，如向量表偏移、休眠模式

## 3. 通用输入输出（GPIO）

GPIO是MCU最基本的外设，每个引脚可独立配置为输入或输出。

### 3.1 工作模式

| 模式 | 说明 | 典型用途 |
|------|------|----------|
| 推挽输出 | 可输出高/低电平，驱动能力强 | 驱动LED、控制数字器件 |
| 开漏输出 | 只能拉低，需外接上拉电阻 | I2C总线、线与逻辑 |
| 浮空输入 | 无内部上下拉，阻抗高 | 外部信号已确定电平 |
| 上拉/下拉输入 | 内置上下拉电阻 | 按键检测、默认电平设置 |
| 模拟输入 | 连接到ADC，关闭数字缓冲 | ADC采样 |
| 复用功能 | 引脚连接到特定外设（UART、SPI等） | 串行通信 |

### 3.2 关键配置参数

- 输出速度（低速/中速/高速/超高速）
- 驱动能力
- 施密特触发器使能（用于抗噪声）

## 4. 中断系统

### 4.1 NVIC（嵌套向量中断控制器）

NVIC是ARM Cortex-M处理器的中断管理核心，特性包括：

- 支持多达240个外部中断（具体数量取决于芯片型号）
- 支持中断嵌套：高优先级中断可打断低优先级中断
- 可配置优先级位数（0-8位，由芯片厂商决定）
- 自动保存和恢复处理器上下文（压栈/出栈）

### 4.2 中断优先级

优先级数值越小，优先级越高。优先级分为两部分：

- **抢占优先级**（组优先级）：决定是否可以嵌套
- **子优先级**（响应优先级）：同一抢占级内的排队顺序

优先级分组配置（以4位优先级为例）：

| 分组 | 抢占位 | 子优先级位 | 抢占级数 | 子级数 |
|------|--------|-----------|---------|--------|
| NVIC_PriorityGroup_0 | 0 | 4 | 1 | 16 |
| NVIC_PriorityGroup_1 | 1 | 3 | 2 | 8 |
| NVIC_PriorityGroup_2 | 2 | 2 | 4 | 4 |
| NVIC_PriorityGroup_3 | 3 | 1 | 8 | 2 |
| NVIC_PriorityGroup_4 | 4 | 0 | 16 | 1 |

### 4.3 中断处理流程

1. 外设产生中断请求，设置NVIC挂起位
2. NVIC根据优先级判断是否响应
3. 处理器完成当前指令后，将xPSR、PC、LR、R12、R3-R0压栈
4. 从向量表读取中断服务程序（ISR）入口地址
5. 执行ISR
6. ISR结束时执行中断返回指令，自动出栈恢复上下文

## 5. 定时器与计数器

### 5.1 基本定时器

基本定时器仅提供定时和触发DAC功能：

- 16位/32位自动重载计数器
- 预分频器（1-65536）
- 更新事件中断
- 触发DAC转换输出

### 5.2 通用定时器

在基本定时器基础上增加：

- 输入捕获：测量外部信号的频率和脉宽
- 输出比较：在指定计数值时产生事件
- PWM输出：脉宽调制信号生成
- 编码器接口：读取正交编码器信号
- 单脉冲模式

### 5.3 高级定时器

额外支持：

- 互补输出与死区插入（用于H桥驱动）
- 刹车功能（紧急关断）
- 重复计数器（减少中断频率）

---

## 6. PWM（脉宽调制）

PWM通过改变脉冲的占空比来等效输出模拟电压，广泛用于：

- **电机调速**：控制直流电机转速
- **LED调光**：调节LED亮度
- **电源管理**：开关电源控制
- **舵机控制**：通过脉宽控制角度

关键参数：
- **频率**：决定PWM周期的快慢
- **分辨率**：计数器位数决定占空比调节精度
- **占空比**：高电平时间与周期的比值

```
占空比 = 25%    ┌─┐     ┌─┐
               ─┘ └─────┘ └─────

占空比 = 50%    ┌───┐   ┌───┐
               ─┘   └───┘   └───

占空比 = 75%    ┌─────┐ ┌─────┐
               ─┘     └─┘     └─
```

## 7. 串行通信接口

### 7.1 UART（异步串行通信）

通用异步收发器，无需时钟线，通过波特率同步：

- 常用波特率：9600、115200、921600
- 数据帧格式：起始位 + 数据位(5-9) + 校验位(可选) + 停止位(1/2)
- 全双工通信（TX + RX两根线）
- 常用于调试输出、GPS模块、AT指令通信

### 7.2 SPI（同步串行通信）

串行外设接口，主从模式，全双工：

| 信号线 | 名称 | 方向 |
|--------|------|------|
| SCLK | 时钟 | 主→从 |
| MOSI | 主出从入 | 主→从 |
| MISO | 主入从出 | 从→主 |
| CS/SS | 片选 | 主→从 |

- 支持4种时钟极性和相位组合（CPOL/CPHA）
- 速率可达数十MHz
- 常用于Flash存储器、显示屏、传感器

### 7.3 I2C（两线式串行通信）

内部集成电路总线，仅需两根线：

- **SDA**：串行数据线
- **SCL**：串行时钟线
- 支持多主多从，通过7位或10位地址寻址
- 标准模式100kbps，快速模式400kbps，高速模式3.4Mbps
- 常用于EEPROM、传感器、实时时钟

通信过程：START → 从机地址+R/W → ACK → 数据 → ... → STOP

### 7.4 CAN总线

控制器局域网络，广泛用于汽车电子：

- 差分信号（CAN_H / CAN_L），抗干扰能力强
- 多主架构，基于优先级的仲裁机制
- 数据帧最大8字节（CAN FD扩展到64字节）
- 速率：经典CAN最高1Mbps，CAN FD最高8Mbps
- 内置错误检测（CRC、位填充、帧检查）

## 8. ADC与DAC

### 8.1 ADC（模数转换器）

将连续模拟信号转换为数字量：

- **分辨率**：常见10位、12位、16位
- **采样率**：每秒转换次数，可达数MSPS
- **转换模式**：单次转换、连续转换、扫描模式、注入模式
- **参考电压**：决定量化范围和精度
- **输入通道**：多路复用，支持多信号轮流采集

### 8.2 DAC（数模转换器）

将数字量转换为模拟电压输出：

- 常见8位、12位分辨率
- 支持定时器触发输出波形
- 应用：音频输出、波形发生器、电压基准

---

## 9. DMA（直接内存访问）

DMA允许外设与存储器之间直接传输数据，无需CPU参与：

- **优点**：释放CPU，提高数据吞吐量
- **通道**：一般有多个独立通道，可同时工作
- **传输模式**：内存到内存、外设到内存、内存到外设
- **触发源**：定时器、ADC、UART、SPI等外设事件
- **传输宽度**：字节(8位)、半字(16位)、字(32位)

典型应用：ADC连续采样数据自动写入内存缓冲区，采满后触发中断通知CPU处理。

---

## 10. 看门狗定时器

看门狗（Watchdog）用于检测和恢复系统故障：

- **独立看门狗（IWDG）**：使用独立时钟源（LSI），即使主时钟故障也能工作
- **窗口看门狗（WWDG）**：在时间窗口内喂狗，过早或过晚都会复位
- 工作原理：计数器递减，软件需定期"喂狗"（重装载计数器），否则计数器溢出触发系统复位
- 嵌入式系统的最后一道防线

## 11. 常见MCU平台

### 11.1 STM32系列

意法半导体ARM Cortex-M微控制器，产品线丰富：

| 系列 | 内核 | 特点 |
|------|------|------|
| STM32F0 | Cortex-M0 | 入门级，低功耗 |
| STM32F1 | Cortex-M3 | 经典系列，资料丰富 |
| STM32F4 | Cortex-M4F | 高性能，带FPU |
| STM32H7 | Cortex-M7 | 旗舰级，双核可选 |
| STM32L4 | Cortex-M4F | 超低功耗 |
| STM32G4 | Cortex-M4F | 高精度模拟外设 |

开发工具：STM32CubeIDE、STM32CubeMX、HAL/LL库

### 11.2 ESP32

乐鑫科技双核Xtensa LX6 SoC，集成WiFi和BLE：

- 主频240MHz，520KB SRAM
- 丰富的外设：ADC、DAC、SPI、I2C、UART、PWM
- 开发框架：ESP-IDF（官方C SDK）、Arduino、MicroPython
- 适合物联网网关、WiFi/BLE应用

### 11.3 Arduino

开源电子原型平台，降低嵌入式开发门槛：

- **硬件**：Arduino Uno（ATmega328P）、Mega、Due、Nano等
- **软件**：Arduino IDE，使用简化的C++语言
- **生态**：海量第三方库和扩展板（Shield）
- 适合快速原型验证和教育

---

## 总结

微控制器是嵌入式系统的基石。掌握MCU架构、GPIO配置、中断处理、定时器和通信接口是嵌入式开发的基本功。在实际项目中，需要根据性能、功耗、外设需求和成本选择合适的MCU平台，并充分利用DMA等机制优化系统性能。
