DMA与中断系统

一、DMA（直接存储器访问）概述

DMA（Direct Memory Access）是一种无需CPU参与即可在内存和外设之间直接传输数据的硬件机制。它极大地减轻了CPU的数据搬运负担，提高了系统整体效率。

1.1 DMA的工作原理

传统的数据传输方式需要CPU逐字节地从外设读取再写入内存（或反向），这会占用大量CPU时间。DMA控制器接管了这个任务，让CPU可以专注于计算。

1.2 DMA vs CPU传输

维度CPU传输DMA传输
CPU占用全程参与仅配置和完成中断
传输速度受指令执行速度限制硬件速率，接近总线速度
适合场景少量数据大批量数据
功耗CPU频繁唤醒CPU可休眠
实时性受中断延迟影响确定性传输

二、DMA通道与架构

2.1 STM32 DMA架构

STM32系列通常有2个DMA控制器（DMA1和DMA2），每个控制器有多个独立通道，可以同时进行不同的数据传输。

控制器通道数特点
DMA17个通道连接常用外设（SPI/I2C/UART/TIM/ADC）
DMA25个通道连接高级外设（SDIO/存储器到存储器）

2.2 通道映射

每个DMA通道与特定外设关联，由硬件固定映射。例如STM32F1中：

  • DMA1 Channel 1：ADC1
  • DMA1 Channel 2：SPI1_RX / TIM4_CH3
  • DMA1 Channel 3：SPI1_TX / TIM4_CH1
  • DMA1 Channel 4：USART1_TX / TIM4_CH2
  • DMA1 Channel 5：USART1_RX
注意：STM32F4/H7等较新系列使用通道请求多路复用器（DMAMUX），不再有固定通道映射，任何通道可以连接任何外设，灵活性更高。

三、DMA传输模式

3.1 传输方向

模式源目标应用场景
外设到内存外设寄存器SRAMADC采样、UART接收
内存到外设SRAM外设寄存器UART发送、DAC输出
内存到内存SRAMSRAM数据搬运、图像处理

3.2 传输模式

模式说明适用场景
正常模式（Normal）传输指定数量数据后停止单次数据传输
循环模式（Circular）传输完成后自动重新开始ADC连续采样、音频播放

3.3 数据宽度

  • 字节（8位）：适合UART、SPI字节传输
  • 半字（16位）：适合12位ADC、16位外设
  • 字（32位）：适合32位数据传输、内存拷贝

3.4 双缓冲模式

双缓冲模式使用两个内存缓冲区交替接收数据，一个缓冲区正在被DMA填充时，CPU可以处理另一个缓冲区的数据。

双缓冲优势：
- 消除数据处理和数据采集之间的竞争
- 实现连续无间断的数据流
- 特别适合音频采集、高速ADC、图像捕获等场景
- DMA半传输完成和传输完成中断分别对应两个缓冲区

四、NVIC（嵌套向量中断控制器）

NVIC是ARM Cortex-M内核的核心组件，负责管理和调度所有中断请求。它支持嵌套中断、动态优先级配置和低延迟响应。

4.1 NVIC核心特性

  • 嵌套中断：高优先级中断可以打断正在执行的低优先级中断
  • 向量表：每个中断源有独立的中断向量（处理函数地址）
  • 低延迟：从中断触发到执行第一条ISR指令仅需12个时钟周期
  • 优先级可配置：支持2~8位优先级配置位
  • 尾链（Tail-chaining）：连续中断之间无需出栈入栈，6个周期即可响应
  • 晚到（Late-arrival）：高优先级中断在低优先级中断入栈期间到达，无需二次入栈

4.2 优先级分组

中断优先级分为抢占优先级和子优先级。优先级寄存器的位可以在这两者之间分配。

分组抢占位子优先级位说明
NVIC_PRIORITYGROUP_004无抢占，16个子优先级
NVIC_PRIORITYGROUP_1132个抢占，8个子优先级
NVIC_PRIORITYGROUP_2224个抢占，4个子优先级
NVIC_PRIORITYGROUP_3318个抢占，2个子优先级
NVIC_PRIORITYGROUP_44016个抢占，无子优先级
优先级规则：
1. 数值越小优先级越高（0为最高优先级）
2. 抢占优先级高的可以打断低的
3. 抢占优先级相同时，子优先级高的先响应，但不能嵌套
4. 两者都相同时，按中断号排优先级

五、中断延迟分析

中断延迟是从中断事件发生到中断处理程序开始执行之间的时间，是实时系统的关键指标。

5.1 延迟组成

阶段说明典型周期数
中断信号同步将异步中断信号同步到时钟域1~2
当前指令完成等待当前指令执行完毕（最长LDM/STM）1~16
入栈（Stack Push）将8个寄存器压入堆栈8
向量提取从向量表读取ISR地址1
取指读取ISR第一条指令1~2
中断响应总延迟：
最佳情况：约12个时钟周期
最差情况：约12 + 16 = 28个时钟周期（多周期指令）

在72MHz时钟下：
最佳延迟 = 12 / 72MHz ≈ 0.167μs
最差延迟 = 28 / 72MHz ≈ 0.389μs

5.2 尾链与晚到优化

  • 尾链（Tail-chaining）：当处理完一个中断后立即有另一个挂起的中断，处理器跳过出栈再入栈的过程，仅用6个周期直接进入下一个ISR
  • 晚到（Late-arrival）：在低优先级中断入栈过程中，更高优先级中断到达，处理器直接服务高优先级中断，入栈的数据仍然有效

六、ISR（中断服务程序）最佳实践

6.1 ISR编写原则

  1. 尽量简短：只做必要的事（清除标志、读取数据、设置标志），复杂处理放到主循环
  2. 避免阻塞：不要在ISR中调用HAL_Delay()、等待信号量等阻塞操作
  3. 保护共享数据：使用volatile变量或关中断保护临界区
  4. 清除中断标志：必须清除中断标志位，否则会反复进入中断
  5. 注意堆栈使用：ISR使用主堆栈（MSP），避免使用过多局部变量

6.2 中断与RTOS协作

FreeRTOS中的中断使用规则：
- 中断安全API以"FromISR"结尾，如xSemaphoreGiveFromISR()
- 在中断中使用portYIELD_FROM_ISR()触发上下文切换
- 优先级高于configMAX_SYSCALL_INTERRUPT_PRIORITY的中断不能调用FreeRTOS API
- 典型模式：ISR中发送信号量/队列，任务等待后处理

6.3 DMA+中断联合使用示例

// ADC + DMA + 中断 实现连续采样
#define ADC_BUF_SIZE 100
uint16_t adcBuffer[ADC_BUF_SIZE];

// 配置ADC使用DMA
hadc1.Init.DMAContinuousRequests = ENABLE;
HAL_ADC_Start_DMA(&hadc1, (uint32_t*)adcBuffer, ADC_BUF_SIZE);

// DMA传输完成中断回调
void HAL_ADC_ConvCpltCallback(ADC_HandleTypeDef* hadc) {
    // 处理后半部分缓冲区数据（双缓冲场景）
    dataReady = 1;  // 设置标志，通知主循环处理
}

// DMA半传输完成中断回调
void HAL_ADC_ConvHalfCpltCallback(ADC_HandleTypeDef* hadc) {
    // 处理前半部分缓冲区数据
    halfDataReady = 1;
}
