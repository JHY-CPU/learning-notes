# 多Agent系统 - Agent与工具调用

*AutoGen、CrewAI、LangGraph 框架深度对比，Agent 通信协议、角色扮演与多 Agent 评估*

主流多 Agent 框架对比

| 框架 | 开发者 | 核心理念 | 适用场景 | 复杂度 |
| --- | --- | --- | --- | --- |
| **AutoGen** | 微软 | 对话驱动的 Agent 协作 | 代码生成、研究、多轮讨论 | 中 |
| **CrewAI** | CrewAI | 角色扮演 + 任务委派 | 内容创作、分析报告 | 低 |
| **LangGraph** | LangChain | 图结构工作流编排 | 复杂流程、状态管理 | 高 |
| **MetaGPT** | DeepWisdom | 软件公司模拟 | 软件开发自动化 | 中高 |
| **ChatDev** | 清华 | 虚拟软件公司 | 端到端软件开发 | 中 |
| **Swarm** | OpenAI | 轻量级 Agent 编排 | 客服路由、工具调用 | 低 |

A2A 协议核心要素

| 概念 | 说明 |
| --- | --- |
| **Agent Card** | JSON 格式的能力描述文件，包含名称、描述、支持的技能、端点 URL |
| **Task** | 最小工作单元，包含 ID、状态、消息历史、产物（Artifact） |
| **Message** | Agent 之间的消息交换，支持文本、文件、结构化数据 |
| **Artifact** | Task 的输出产物，如文件、图片、代码等 |
| **Push Notification** | 异步任务的回调通知机制 |

Agent 间消息模式

| 模式 | 说明 | 适用场景 |
| --- | --- | --- |
| **请求-响应** | A 发送请求，B 返回结果 | 工具调用、问答 |
| **发布-订阅** | A 发布事件，订阅者接收 | 状态通知、日志 |
| **委派** | A 将子任务委派给 B | 任务分解 |
| **协商** | A 提议，B 接受/拒绝/反提议 | 资源分配、优先级 |
| **广播** | A 向所有 Agent 发送消息 | 全局状态同步 |

多 Agent 评估维度

| 维度 | 指标 | 评估方法 |
| --- | --- | --- |
| **任务完成度** | 成功率、完成率 | 预设任务集的自动测试 |
| **协作效率** | 通信轮次、总延迟 | Agent 间消息统计 |
| **输出质量** | 代码正确性、报告质量 | LLM-as-Judge + 人工评估 |
| **成本控制** | Token 消耗、API 调用数 | 成本监控 |
| **鲁棒性** | 错误恢复率 | 注入故障测试 |
| **安全性** | 越狱防护、权限控制 | 对抗测试 |
