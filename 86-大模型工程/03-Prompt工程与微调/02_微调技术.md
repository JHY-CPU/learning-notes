# 微调技术 - Prompt工程与微调

*深入理解 Full Fine-tuning、LoRA/QLoRA 参数高效微调、PEFT 框架、数据集准备、训练技巧及 RLHF/DPO 对齐方法*

微调方法对比

| 方法 | 训练参数量 | 显存需求 | 效果 | 适用场景 |
| --- | --- | --- | --- | --- |
| **Full Fine-tuning** | 100% | 极高 | 最好 | 充足算力，大规模数据 |
| **LoRA** | 0.1-1% | 较低 | 接近全量 | 大多数场景首选 |
| **QLoRA** | 0.1-1% | 最低 | 接近 LoRA | 消费级 GPU |
| **Prefix Tuning** | <0.1% | 低 | 略差 | 资源极其有限 |
| **Prompt Tuning** | <0.01% | 最低 | 较差 | 简单任务 |
| **Adapter** | 1-5% | 中 | 好 | 多任务学习 |

QLoRA 三大创新

| 创新 | 说明 | 效果 |
| --- | --- | --- |
| **NF4 量化** | NormalFloat 4-bit，针对正态分布优化的量化类型 | 模型权重从 16-bit 降到 4-bit，显存减少 75% |
| **双重量化** | 对量化常数再次量化 | 额外节省约 0.4 bit/参数 |
| **分页优化器** | 利用 NVIDIA 统一内存自动页面转移 | 避免显存 OOM |

LoRA 微调超参数建议

| 超参数 | 推荐值 | 说明 |
| --- | --- | --- |
| rank (r) | 8-64（常用 16） | 越大拟合能力越强，但过拟合风险增加 |
| lora_alpha | 2 * r | 缩放因子，控制 LoRA 对原始权重的影响程度 |
| target_modules | q_proj, v_proj 最低；全部注意力层效果更好 | 注入 LoRA 的目标层 |
| learning_rate | 1e-4 ~ 3e-4 | 比全量微调高（全量通常 1e-5 ~ 5e-5） |
| epochs | 1-5 | LoRA 容易过拟合，数据少时 1-2 轮即可 |
| batch_size | 4-16（梯度累加） | 显存不够时增大 gradient_accumulation_steps |
| warmup_ratio | 0.03-0.1 | 学习率预热比例 |
| lr_scheduler | cosine | 余弦退火效果通常最好 |

RLHF vs DPO 对比

| 维度 | RLHF (PPO) | DPO |
| --- | --- | --- |
| 训练阶段 | 3 个阶段（SFT → RM → PPO） | 1 个阶段（直接从偏好数据训练） |
| 模型数量 | 4 个（Policy/Ref/Reward/Value） | 2 个（Policy/Ref） |
| 训练稳定性 | 不稳定，超参敏感 | 稳定，类似 SFT |
| 显存需求 | 极高 | 仅 LoRA 微调量级 |
| 数据要求 | 偏好对 + RM 训练数据 | 仅偏好对（chosen/rejected） |
| 效果 | 理论上上限更高 | 大多数场景接近 RLHF |
| 社区趋势 | 逐渐被替代 | 主流选择 |

何时微调 vs 何时用 Prompt/RAG

| 场景 | 推荐方案 | 理由 |
| --- | --- | --- |
| 知识问答（知识会更新） | RAG | 知识实时更新，无需重新训练 |
| 输出格式/风格要求 | 微调 | 效果更好更稳定 |
| 领域术语理解 | 微调 + RAG | 微调学术语，RAG 补充知识 |
| 简单任务（分类/提取） | Prompt Engineering | 成本最低，迭代最快 |
| 用小模型替代大模型 | 蒸馏微调 | 小模型 + 微调 ≈ 大模型效果 |
| 安全对齐 | DPO/RLHF | 从偏好数据中学习 |
| 数据量不足（<100条） | Few-shot Prompt | 微调容易过拟合 |
