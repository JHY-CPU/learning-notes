# RAG工程实践 - RAG与向量数据库

*文档解析、Embedding 选型、Reranker 实战、查询分解与引用生成*

PDF 解析方案对比

| 工具 | 适用场景 | 优点 | 局限 |
| --- | --- | --- | --- |
| **PyMuPDF (fitz)** | 文本型 PDF | 速度快、提取精确 | 不处理扫描件 |
| **pdfplumber** | 含表格的 PDF | 表格提取优秀 | 速度较慢 |
| **Unstructured** | 多格式通用 | 自动检测布局元素 | 需要依赖模型 |
| **DocTR / EasyOCR** | 扫描件/OCR | 深度学习 OCR | 计算量大 |
| **Marker** | PDF 转 Markdown | 保留结构、支持表格 | GPU 加速更佳 |
| **MinerU** | 学术论文/复杂 PDF | 中英文优秀、公式识别 | 模型较大 |

RAG 系统生产部署清单

| 环节 | 关键项 | 建议 |
| --- | --- | --- |
| 文档处理 | 增量更新 | 基于文件哈希检测变更，只更新变化的文档 |
| 分块策略 | 元数据保留 | 每个块保留来源、页码、章节等元数据 |
| 向量存储 | 备份与容灾 | 定期快照、多副本部署 |
| 检索 | 超时与降级 | 设置检索超时，超时后降级为纯 LLM |
| 生成 | 流式输出 | 使用 SSE 流式返回，提升用户体验 |
| 监控 | 质量追踪 | 记录检索分数、生成延迟、用户反馈 |
| 缓存 | 语义缓存 | 相似查询命中缓存，减少 LLM 调用 |
| 安全 | 访问控制 | 基于用户角色过滤可检索的文档 |
