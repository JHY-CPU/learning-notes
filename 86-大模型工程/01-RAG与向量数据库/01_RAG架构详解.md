# RAG架构详解 - RAG与向量数据库

*深入理解检索增强生成（Retrieval-Augmented Generation）的完整 Pipeline，涵盖分块策略、Embedding 模型、重排序及混合搜索*

RAG vs 纯 LLM vs Fine-tuning 对比

| 维度 | 纯 LLM | Fine-tuning | RAG |
| --- | --- | --- | --- |
| 知识更新 | 训练截止 | 需重新训练 | 实时更新 |
| 幻觉控制 | 较难 | 改善有限 | 有据可查 |
| 私有数据 | 无法访问 | 训练时融入 | 运行时检索 |
| 成本 | 推理成本 | 训练+推理 | 检索+推理 |
| 可解释性 | 低 | 低 | 可追溯来源 |
| 适用场景 | 通用对话 | 风格/格式调整 | 知识密集型问答 |

分块策略对比

| 策略 | 原理 | 优点 | 缺点 |
| --- | --- | --- | --- |
| **固定大小分块** | 按固定字符/token 数切分 | 简单高效 | 可能截断语义 |
| **递归分块** | 按分隔符层级递归切分（\n\n → \n → 空格） | 语义保留好 | 块大小不一 |
| **语义分块** | 基于嵌入相似度检测语义边界 | 语义最完整 | 计算成本高 |
| **文档结构分块** | 按 Markdown 标题/HTML 标签切分 | 保留结构信息 | 依赖文档格式 |
| **滑动窗口** | 固定窗口 + 重叠区域 | 避免边界信息丢失 | 冗余存储 |

主流 Embedding 模型对比

| 模型 | 维度 | 最大 tokens | MTEB 得分 | 特点 |
| --- | --- | --- | --- | --- |
| OpenAI text-embedding-3-small | 1536 | 8191 | 62.3 | 性价比高，支持降维 |
| OpenAI text-embedding-3-large | 3072 | 8191 | 64.6 | 精度最高，成本较高 |
| BGE-large-zh-v1.5 | 1024 | 512 | -- | 中文效果优秀，开源 |
| BGE-M3 | 1024 | 8192 | -- | 多语言、多粒度、多功能 |
| Cohere embed-v3 | 1024 | 512 | 64.5 | 多语言，支持搜索/聚类模式 |
| Jina-embeddings-v2 | 768 | 8192 | -- | 开源，长文本支持好 |

搜索方式对比

| 方式 | 原理 | 优势 | 劣势 |
| --- | --- | --- | --- |
| 向量搜索 | 语义相似度匹配 | 理解同义词/改写 | 可能忽略精确关键词 |
| BM25 关键词搜索 | 词频统计匹配 | 精确匹配高效 | 无法理解语义 |
| 混合搜索 | 两者结果加权融合 | 兼顾语义和精确 | 需要调权重 |

RAG 系统评估维度

| 指标 | 衡量内容 | 计算方式 |
| --- | --- | --- |
| Context Relevancy | 检索到的上下文是否与问题相关 | LLM 评分 |
| Context Recall | 是否检索到了所有相关信息 | 对比标准答案 |
| Faithfulness | 生成的回答是否忠实于上下文 | 验证每句话是否有依据 |
| Answer Relevancy | 生成的回答是否切题 | LLM 评分 |
| Answer Correctness | 生成的回答是否正确 | 对比标准答案 |


**评估工具：**RAGAS、DeepEval、TruLens、Phoenix (Arize)
