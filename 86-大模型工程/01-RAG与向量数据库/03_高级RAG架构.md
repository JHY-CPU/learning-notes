# 高级RAG架构 - RAG与向量数据库

*GraphRAG、Self-RAG、CRAG、多模态 RAG 及高级分块与评估策略*

ColBERT vs 传统方案对比

| 方案 | 交互方式 | 速度 | 精度 |
| --- | --- | --- | --- |
| Bi-Encoder（Dense Retrieval） | 向量点积 | 最快 | 中 |
| Cross-Encoder（Reranker） | 全交叉注意力 | 最慢 | 最高 |
| ColBERT | MaxSim 延迟交互 | 快 | 高 |


ColBERT 为每个 token 生成嵌入向量，查询时通过 MaxSim（最大相似度）计算相关性，兼顾速度与精度。

多模态 RAG 方案对比

| 方案 | 原理 | 优点 | 缺点 |
| --- | --- | --- | --- |
| **文本提取法** | 将图表/表格转为文本描述 | 兼容传统 RAG | 丢失视觉信息 |
| **多模态嵌入法** | 使用 CLIP 等统一编码图文 | 保留视觉语义 | 文本精度下降 |
| **MM-LLM 直接理解** | 用 GPT-4V/Gemini 理解文档图像 | 理解力最强 | 成本高、速度慢 |
| **混合方案** | 文本走 RAG，图像走 MM-LLM | 兼顾精度和成本 | 架构复杂 |

RAGAS 评估体系详解

| 指标 | 评估对象 | 计算方法 | 理想值 |
| --- | --- | --- | --- |
| **Context Precision** | 检索排序质量 | 相关文档在检索结果中的排名 | 1.0 |
| **Context Recall** | 检索覆盖度 | 标准答案所需信息被检索到的比例 | 1.0 |
| **Faithfulness** | 生成忠实度 | 回答中可被上下文支持的句子比例 | 1.0 |
| **Answer Relevancy** | 回答相关性 | 生成回答与问题的相关程度（LLM 评分） | 1.0 |
| **Answer Correctness** | 回答正确性 | 与标准答案的语义和事实对比 | 1.0 |
| **Answer Semantic Similarity** | 语义相似度 | 回答与标准答案的嵌入相似度 | 1.0 |

## GraphRAG 架构详解

GraphRAG 将知识图谱与 RAG 结合，解决传统 RAG 在多跳推理和全局问题上的不足。

```
GraphRAG 流程：
1. 索引阶段：从文档中抽取实体和关系，构建知识图谱
2. 社区检测：使用 Leiden 算法将图谱划分为层次化社区
3. 摘要生成：为每个社区生成摘要描述
4. 查询阶段：
   - 局部搜索：找到相关实体及其邻居
   - 全局搜索：利用社区摘要回答全局性问题
```

| 对比维度 | 传统 RAG | GraphRAG |
|----------|---------|---------|
| 检索粒度 | 文档块 | 实体/社区 |
| 推理能力 | 单跳 | 多跳推理 |
| 全局问题 | 弱 | 强（社区摘要） |
| 索引成本 | 低 | 高（需抽取图谱） |
| 适用场景 | 简单问答 | 复杂推理、总结 |

## Self-RAG 与 CRAG

**Self-RAG（自适应检索）**：
```
1. 判断是否需要检索 → 不需要则直接生成
2. 检索后评估文档相关性 → 过滤不相关文档
3. 生成回答后自我评估 → 评估忠实度和有用性
4. 必要时重新检索或修正回答
```

**CRAG（纠正性 RAG）**：
```
1. 检索文档后，评估相关性（相关/不相关/模糊）
2. 相关：知识精炼后使用
3. 不相关：丢弃检索结果，使用网络搜索
4. 模糊：结合检索结果和参数化知识
```

## 高级分块策略

| 策略 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| 固定大小 | 按 token/字符数切分 | 简单快速 | 破坏语义 |
| 递归分割 | 按段落→句子→字符递归切分 | 保留语义结构 | 参数敏感 |
| 语义分块 | 基于嵌入相似度检测边界 | 语义完整 | 计算开销大 |
| 文档结构分块 | 按标题/章节切分 | 利用文档结构 | 依赖结构化文档 |
| 滑动窗口 | 重叠窗口切分 | 避免信息截断 | 块数多，有冗余 |
| 父子文档 | 小块检索，大块送入 LLM | 检索精确+上下文丰富 | 实现复杂 |
