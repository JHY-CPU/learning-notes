# 数据可视化与BI

## 仪表盘设计原则

优秀的数据仪表盘不仅是图表的堆砌，而是将数据转化为可行动的洞察。设计时应遵循以下原则：

### 1. 信息层级分明

```text
+--------------------------------------------------+
|  战略层 (Executive KPIs)                          |
|  [总营收: 1.2亿] [用户数: 500万] [转化率: 3.2%]    |
+--------------------------------------------------+
|  战术层 (Trend Analysis)                          |
|  [营收趋势图]    [用户增长曲线]   [漏斗分析]        |
+--------------------------------------------------+
|  操作层 (Drill-down Details)                      |
|  [订单明细表]    [异常列表]      [操作入口]         |
+--------------------------------------------------+
```

### 2. CRAP设计原则

| 原则 | 说明 | 实践 |
|------|------|------|
| **对比 (Contrast)** | 突出关键数据 | 异常值用红色/加粗标记 |
| **重复 (Repetition)** | 保持视觉一致性 | 统一配色、字体、图表样式 |
| **对齐 (Alignment)** | 网格化布局 | 使用12列栅格系统 |
| **亲密 (Proximity)** | 相关信息靠近 | 同一指标的图表放在一起 |

### 3. 自助分析路径

- 第一层：概览仪表盘（全局指标一览）
- 第二层：主题仪表盘（按业务域拆分）
- 第三层：明细钻取（可下钻到具体数据）

## 图表选型指南

| 数据关系 | 推荐图表 | 适用场景 |
|----------|---------|---------|
| 比较 | 柱状图、条形图 | 不同产品销售额对比 |
| 趋势 | 折线图、面积图 | 月度营收变化趋势 |
| 占比 | 饼图、环形图、树状图 | 渠道来源占比（类别≤6） |
| 分布 | 直方图、箱线图、散点图 | 用户年龄分布 |
| 关系 | 散点图、气泡图 | 广告投入与转化率关系 |
| 地理 | 地图、热力图 | 区域销售分布 |
| 流程 | 漏斗图、桑基图 | 用户转化漏斗 |
| 排名 | 横向条形图 | TOP10产品销量 |

### 图表选择决策树

```text
你想展示什么？
├── 数值大小 → 单一指标：大数字卡片
│              多个指标：柱状图/条形图
├── 变化趋势 → 时间序列：折线图
│              累计趋势：面积图
├── 构成占比 → 静态占比：饼图/环形图（≤6类）
│              层级构成：树状图/旭日图
├── 数据分布 → 单变量：直方图
│              多变量：散点图
└── 地理数据 → 区域热力：填充地图
               点位分布：气泡地图
```

## Apache Superset

Superset是Apache开源的现代BI平台，支持丰富的可视化类型和SQL查询。

### 安装与配置

```bash
# Docker Compose部署
# docker-compose.yml
cat > docker-compose.yml << 'EOF'
version: "3.8"
services:
  superset:
    image: apache/superset:latest
    ports:
      - "8088:8088"
    environment:
      SUPERSET_SECRET_KEY: "your-secret-key-here"
      SUPERSET_CONFIG_PATH: /app/pythonpath/superset_config.py
    volumes:
      - ./superset_config.py:/app/pythonpath/superset_config.py
      - superset_data:/app/superset_home
    depends_on:
      - redis
      - postgres

  redis:
    image: redis:7-alpine
    volumes:
      - redis_data:/data

  postgres:
    image: postgres:15-alpine
    environment:
      POSTGRES_DB: superset
      POSTGRES_USER: superset
      POSTGRES_PASSWORD: superset_password
    volumes:
      - postgres_data:/var/lib/postgresql/data

volumes:
  superset_data:
  redis_data:
  postgres_data:
EOF

docker-compose up -d
```

### 通过API创建仪表盘

```python
import requests

SUPERSET_URL = "http://localhost:8088"
SESSION = requests.Session()

# 登录
resp = SESSION.post(f"{SUPERSET_URL}/api/v1/security/login", json={

    "password": "admin",
    "provider": "db",
    "refresh": True

access_token = resp.json()["access_token"]

# 创建图表

    "viz_type": "echarts_timeseries_line",
    "datasource_id": dataset_id,
    "datasource_type": "table",
    "params": json.dumps({
        "metrics": ["active_users", "new_users"],
        "time_range": "Last 30 days",
        "granularity_sqla": "stat_date",
        "time_grain_sqla": "P1D",
        "series_type": "line",
        "row_limit": 10000,
    })
})

# 创建仪表盘
dashboard_resp = SESSION.post(f"{SUPERSET_URL}/api/v1/dashboard/", json={
    "dashboard_title": "用户增长分析",
    "slug": "user-growth",
    "published": True
})
```

### SQL Lab查询示例

```sql
-- 在Superset SQL Lab中直接查询
-- 查询日活趋势及环比
WITH daily_metrics AS (
    SELECT
        stat_date,
        active_users,
        LAG(active_users, 1) OVER (ORDER BY stat_date) AS prev_active_users,
        new_users,
        total_orders,
        total_amount
    FROM analytics.dws_user_daily_metrics
    WHERE stat_date >= DATE_SUB(CURDATE(), INTERVAL 30 DAY)
)
SELECT
    stat_date,
    active_users,
    prev_active_users,
    ROUND((active_users - prev_active_users) * 100.0 / prev_active_users, 2) AS growth_rate_pct,
    new_users,
    ROUND(total_amount / total_orders, 2) AS avg_order_value
FROM daily_metrics
ORDER BY stat_date;
```

### Metabase问答模式

Metabase提供两种查询方式：

1. **简单模式（Questions）**：通过可视化界面拖拽构建查询
2. **原生SQL模式**：直接编写SQL查询

```sql
-- Metabase原生SQL查询，支持变量和过滤器
-- 使用 {{变量名}} 定义可过滤参数
SELECT
    DATE_TRUNC('day', order_time) AS order_date,
    COUNT(DISTINCT user_id) AS paying_users,
    COUNT(DISTINCT order_id) AS order_count,
    SUM(amount) AS daily_revenue
FROM analytics.fct_order
WHERE 1=1
    [[AND DATE_TRUNC('day', order_time) >= {{start_date}}]]
    [[AND DATE_TRUNC('day', order_time) <= {{end_date}}]]
    [[AND category = {{category_filter}}]]
GROUP BY 1
ORDER BY 1 DESC;
```

### Metabase API集成

```python
import requests

METABASE_URL = "http://localhost:3000"

# 获取Session Token
resp = requests.post(f"{METABASE_URL}/api/session", json={
    "username": "admin@company.com",
    "password": "password"
})
session_id = resp.json()["id"]
headers = {"X-Metabase-Session": session_id}

# 获取所有仪表盘
dashboards = requests.get(
    f"{METABASE_URL}/api/dashboard",
    headers=headers
).json()

# 获取仪表盘详情
dashboard_detail = requests.get(
    f"{METABASE_URL}/api/dashboard/1",
    headers=headers
).json()

# 执行查询
query_result = requests.post(f"{METABASE_URL}/api/dataset", headers=headers, json={
    "database": 1,
    "type": "native",
    "native": {
        "query": """
            SELECT stat_date, active_users, new_users
            FROM analytics.dws_user_daily_metrics
            WHERE stat_date >= CURRENT_DATE - INTERVAL '7 days'
            ORDER BY stat_date
        """
    }
}).json()

for row in query_result["data"]["rows"]:
    print(f"Date: {row[0]}, DAU: {row[1]}, New Users: {row[2]}")
```

## 告警与通知

### Superset告警配置

```yaml
# alert_config.yaml
alerts:
  - name: "DAU下降告警"
    description: "日活跃用户数较前日下降超过10%"
    sql: >
      WITH metrics AS (
        SELECT active_users,
               LAG(active_users) OVER (ORDER BY stat_date) AS prev_active_users
        FROM dws_user_daily_metrics
        ORDER BY stat_date DESC LIMIT 1
      )
      SELECT CASE
        WHEN (prev_active_users - active_users) * 100.0 / prev_active_users > 10
        THEN 'ALERT: DAU dropped by ' || ROUND((prev_active_users - active_users) * 100.0 / prev_active_users, 1) || '%'
        ELSE 'OK'
      END AS status
      FROM metrics
    condition: "status LIKE 'ALERT%'"
    frequency: "daily"
    recipients:
      - "data-team@company.com"
      - "ops-team@company.com"
    channels:
      - email
      - slack
```

## 最佳实践

1. **图表不超过7种类型**：仪表盘中使用的图表类型不宜过多，保持视觉统一
2. **关键指标前置**：最重要的KPI放在最显眼的位置（左上角或顶部）
3. **配色不超过5种**：使用一致的配色方案，语义化配色（红=警告，绿=正常）
4. **移动端适配**：确保仪表盘在移动端有良好的展示效果
5. **数据刷新频率匹配业务**：实时业务用秒级刷新，日级业务用小时级刷新
6. **注释与说明**：对异常数据点添加注释，帮助用户理解数据波动原因
