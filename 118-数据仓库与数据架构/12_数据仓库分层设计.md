# 数据仓库分层设计

## 分层架构总览

数据仓库分层是将数据按照处理阶段和使用目的划分为不同层次的架构设计模式。分层的核心目的是**解耦数据处理过程、提高复用性、降低变更影响**。

```text
+--------------------------------------------------+
|  ADS (Application Data Service) - 应用数据层      |
|  面向报表、API、大屏等直接消费场景                  |
+--------------------------------------------------+
|  DWS (Data Warehouse Summary) - 数据汇总层        |
|  按主题域进行轻度汇总，形成宽表                     |
+--------------------------------------------------+
|  DWD (Data Warehouse Detail) - 数据明细层          |
|  清洗、规范化、维度退化后的明细数据                  |
+--------------------------------------------------+
|  DIM (Dimension) - 维度层                         |
|  公共维度表：用户、商品、地区、时间等               |
+--------------------------------------------------+
|  ODS (Operational Data Store) - 操作数据层         |
|  原始数据接入，保持源系统数据原貌                   |
+--------------------------------------------------+
|         数据源 (MySQL/日志/API/Kafka)               |
+--------------------------------------------------+
```

## 各层详细设计

### ODS层（操作数据存储层）

ODS层是数据仓库的入口，负责从各源系统接入原始数据，保留数据的原貌，不做任何业务逻辑处理。

**核心职责**：
- 从源系统增量/全量抽取数据
- 保持源系统数据结构不变
- 按来源分区存储，支持数据回溯

```sql
-- ODS层建表规范：外部表 + 按日期分区 + ORC格式
CREATE EXTERNAL TABLE IF NOT EXISTS ods_mysql_order (
    id              BIGINT         COMMENT '订单ID',
    user_id         BIGINT         COMMENT '用户ID',
    product_id      BIGINT         COMMENT '商品ID',
    quantity        INT            COMMENT '数量',
    unit_price      DECIMAL(10,2)  COMMENT '单价',
    total_amount    DECIMAL(12,2)  COMMENT '总金额',
    status          TINYINT        COMMENT '订单状态:0待付款,1已付款,2已发货,3已完成,4已取消',
    address         STRING         COMMENT '收货地址JSON',
    create_time     TIMESTAMP      COMMENT '创建时间',
    update_time     TIMESTAMP      COMMENT '更新时间',
    source_db       STRING         COMMENT '来源数据库',
    etl_time        TIMESTAMP      COMMENT 'ETL抽取时间'
)
COMMENT '订单表 - 来源MySQL业务库'
PARTITIONED BY (dt STRING COMMENT '数据日期')
STORED AS ORC
LOCATION '/data/warehouse/ods/ods_mysql_order/'
TBLPROPERTIES (
    'orc.compress' = 'SNAPPY',
    'comment' = 'T+1全量同步，保留近90天数据',
    'retention_days' = '90'
);

-- 从Kafka日志数据落地
CREATE EXTERNAL TABLE IF NOT EXISTS ods_kafka_user_event (
    event_id        STRING         COMMENT '事件ID',
    user_id         STRING         COMMENT '用户ID',
    event_type      STRING         COMMENT '事件类型',
    event_time      BIGINT         COMMENT '事件时间戳(毫秒)',
    page_url        STRING         COMMENT '页面URL',
    properties      STRING         COMMENT '事件属性JSON',
    ip              STRING         COMMENT 'IP地址',
    user_agent      STRING         COMMENT 'UA信息'
)
COMMENT '用户行为事件 - 来源Kafka'
PARTITIONED BY (dt STRING COMMENT '数据日期', hour STRING COMMENT '数据小时')
STORED AS PARQUET
LOCATION '/data/warehouse/ods/ods_kafka_user_event/'
TBLPROPERTIES ('parquet.compression' = 'SNAPPY');
```

**ODS层命名规范**：
| 组成部分 | 格式 | 示例 |
|---------|------|------|
| 前缀 | `ods_` | 固定前缀 |
| 来源系统 | `{source}_` | `mysql_`, `kafka_`, `api_` |
| 表名 | `{table_name}` | `order`, `user`, `product` |
| 全称 | `ods_{source}_{table}` | `ods_mysql_order` |

### DWD层（数据明细层）

DWD层对ODS层数据进行清洗、规范化、维度退化，形成一致、规范的明细数据。

**核心职责**：
- 数据清洗：去重、空值处理、格式统一
- 数据规范化：统一编码、枚举值映射
- 维度退化：将常用维度字段冗余到事实表
- 业务主键统一：生成全局唯一业务主键

```sql
-- DWD层：订单明细事实表
CREATE TABLE IF NOT EXISTS dwd_fact_order_detail (
    -- 主键
    order_detail_id STRING         COMMENT '订单明细全局唯一ID',
    order_id        BIGINT         COMMENT '订单ID',
    user_id         BIGINT         COMMENT '用户ID',
    product_id      BIGINT         COMMENT '商品ID',

    -- 度量
    quantity        INT            COMMENT '购买数量',
    unit_price      DECIMAL(10,2)  COMMENT '单价',
    discount_amount DECIMAL(10,2)  COMMENT '优惠金额',
    pay_amount      DECIMAL(12,2)  COMMENT '实付金额',

    -- 维度退化字段
    order_status    STRING         COMMENT '订单状态(已转为可读值)',
    pay_method      STRING         COMMENT '支付方式',
    platform        STRING         COMMENT '下单平台',
    channel         STRING         COMMENT '获客渠道',

    -- 时间维度
    order_time      TIMESTAMP      COMMENT '下单时间',
    pay_time        TIMESTAMP      COMMENT '付款时间',
    ship_time       TIMESTAMP      COMMENT '发货时间',
    receive_time    TIMESTAMP      COMMENT '收货时间',

    -- ETL信息
    etl_time        TIMESTAMP      COMMENT 'ETL处理时间',
    etl_batch_id    STRING         COMMENT 'ETL批次ID'
)
COMMENT '订单明细事实表 - DWD层'
PARTITIONED BY (dt STRING COMMENT '数据日期')
STORED AS ORC
TBLPROPERTIES ('orc.compress' = 'SNAPPY');

-- DWD层ETL：从ODS清洗转换到DWD
INSERT OVERWRITE TABLE dwd_fact_order_detail PARTITION (dt = '${biz_date}')
SELECT
    -- 生成全局唯一ID
    CONCAT(CAST(o.id AS STRING), '_', CAST(od.product_id AS STRING)) AS order_detail_id,
    o.id AS order_id,
    o.user_id,
    od.product_id,
    od.quantity,
    od.unit_price,
    COALESCE(od.discount_amount, 0) AS discount_amount,
    od.quantity * od.unit_price - COALESCE(od.discount_amount, 0) AS pay_amount,

    -- 状态码转中文/英文
    CASE o.status
        WHEN 0 THEN 'pending_payment'
        WHEN 1 THEN 'paid'
        WHEN 2 THEN 'shipped'
        WHEN 3 THEN 'completed'
        WHEN 4 THEN 'cancelled'
        ELSE 'unknown'
    END AS order_status,

    COALESCE(o.pay_method, 'unknown') AS pay_method,
    COALESCE(o.platform, 'unknown') AS platform,
    COALESCE(u.channel, 'direct') AS channel,

    o.create_time AS order_time,
    o.pay_time,
    o.ship_time,
    o.receive_time,

    CURRENT_TIMESTAMP() AS etl_time,
    '${run_id}' AS etl_batch_id
FROM ods_mysql_order o
LATERAL VIEW explode(o.order_details) od_table AS od  -- 假设订单明细为JSON数组
LEFT JOIN dwd_dim_user u ON o.user_id = u.user_id
WHERE o.dt = '${biz_date}'
    AND o.id IS NOT NULL          -- 去除空主键
    AND o.user_id IS NOT NULL;    -- 去除无用户关联的订单
```

**DWD层命名规范**：
| 组成部分 | 格式 | 示例 |
|---------|------|------|
| 事实表前缀 | `dwd_fact_` | `dwd_fact_order_detail` |
| 维度表前缀 | `dwd_dim_` | `dwd_dim_user` |
| 增量表后缀 | `_delta` | `dwd_fact_order_delta` |

### DWS层（数据汇总层）

DWS层按主题域对DWD层明细数据进行轻度汇总，形成面向分析的宽表。

**核心职责**：
- 按主题域聚合：用户域、交易域、流量域
- 形成宽表：将多个维度和度量汇总到一张表
- 多时间粒度：日/周/月粒度的汇总数据

```sql
-- DWS层：用户交易汇总宽表（日粒度）
CREATE TABLE IF NOT EXISTS dws_user_trade_1d (
    user_id             BIGINT         COMMENT '用户ID',
    -- 订单指标
    order_count         INT            COMMENT '下单次数',
    order_amount        DECIMAL(14,2)  COMMENT '下单总金额',
    pay_order_count     INT            COMMENT '付款订单数',
    pay_amount          DECIMAL(14,2)  COMMENT '付款总金额',
    refund_count        INT            COMMENT '退款次数',
    refund_amount       DECIMAL(14,2)  COMMENT '退款总金额',
    -- 商品指标
    product_count       INT            COMMENT '购买商品种类数',
    avg_unit_price      DECIMAL(10,2)  COMMENT '平均单价',
    -- 行为指标
    cart_add_count      INT            COMMENT '加购次数',
    favorite_count      INT            COMMENT '收藏次数',
    -- 时间指标
    first_order_time    TIMESTAMP      COMMENT '首次下单时间',
    last_order_time     TIMESTAMP      COMMENT '最近下单时间',
    -- 渠道
    first_channel       STRING         COMMENT '首次获客渠道',
    last_channel        STRING         COMMENT '最近获客渠道'
)
COMMENT '用户交易日汇总宽表 - DWS层'
PARTITIONED BY (dt STRING COMMENT '数据日期')
STORED AS ORC;

-- DWS层ETL：从DWD汇总到DWS
INSERT OVERWRITE TABLE dws_user_trade_1d PARTITION (dt = '${biz_date}')
SELECT
    user_id,
    COUNT(DISTINCT order_id) AS order_count,
    SUM(pay_amount) AS order_amount,
    COUNT(DISTINCT CASE WHEN order_status IN ('paid','shipped','completed') THEN order_id END) AS pay_order_count,
    SUM(CASE WHEN order_status IN ('paid','shipped','completed') THEN pay_amount ELSE 0 END) AS pay_amount,
    COUNT(DISTINCT CASE WHEN order_status = 'refunded' THEN order_id END) AS refund_count,
    SUM(CASE WHEN order_status = 'refunded' THEN pay_amount ELSE 0 END) AS refund_amount,
    COUNT(DISTINCT product_id) AS product_count,
    AVG(unit_price) AS avg_unit_price,
    0 AS cart_add_count,   -- 需关联行为明细表
    0 AS favorite_count,   -- 需关联行为明细表
    MIN(order_time) AS first_order_time,
    MAX(order_time) AS last_order_time,
    MIN(channel) AS first_channel,
    MAX(channel) AS last_channel
FROM dwd_fact_order_detail
WHERE dt = '${biz_date}'
GROUP BY user_id;
```

**DWS层命名规范**：
| 组成部分 | 格式 | 示例 |
|---------|------|------|
| 前缀 | `dws_` | 固定前缀 |
| 主题域 | `{domain}_` | `user_`, `trade_`, `traffic_` |
| 时间粒度 | `_1d`, `_1w`, `_1m` | 日/周/月粒度 |
| 全称 | `dws_{domain}_{granularity}` | `dws_user_trade_1d` |

### ADS层（应用数据层）

ADS层面向最终应用，直接为报表、大屏、API等提供数据。

```sql
-- ADS层：经营日报表
CREATE TABLE IF NOT EXISTS ads_daily_business_report (
    report_date         DATE           COMMENT '报表日期',
    -- 用户指标
    dau                 BIGINT         COMMENT '日活跃用户数',
    new_users           BIGINT         COMMENT '新增用户数',
    retention_1d_rate   DECIMAL(5,4)   COMMENT '次日留存率',
    retention_7d_rate   DECIMAL(5,4)   COMMENT '7日留存率',
    -- 交易指标
    order_users         BIGINT         COMMENT '下单用户数',
    order_count         BIGINT         COMMENT '订单数',
    order_amount        DECIMAL(16,2)  COMMENT '订单金额',
    gmv                 DECIMAL(16,2)  COMMENT 'GMV',
    avg_order_value     DECIMAL(12,2)  COMMENT '客单价',
    -- 转化指标
    uv                  BIGINT         COMMENT '访客数',
    cart_rate           DECIMAL(5,4)   COMMENT '加购率',
    pay_rate            DECIMAL(5,4)   COMMENT '付款转化率',
    -- 质量指标
    refund_rate         DECIMAL(5,4)   COMMENT '退款率',
    complaint_rate      DECIMAL(5,4)   COMMENT '投诉率'
)
COMMENT '经营日报表 - ADS层'
STORED AS ORC;

-- ADS层ETL：从DWS汇总到ADS
INSERT OVERWRITE TABLE ads_daily_business_report
SELECT
    '${biz_date}' AS report_date,
    COUNT(DISTINCT CASE WHEN is_active = 1 THEN user_id END) AS dau,
    COUNT(DISTINCT CASE WHEN is_new = 1 THEN user_id END) AS new_users,
    SUM(CASE WHEN is_retained_1d = 1 THEN 1 ELSE 0 END) * 1.0
        / NULLIF(COUNT(DISTINCT CASE WHEN is_active_prev_1d = 1 THEN user_id END), 0) AS retention_1d_rate,
    SUM(CASE WHEN is_retained_7d = 1 THEN 1 ELSE 0 END) * 1.0
        / NULLIF(COUNT(DISTINCT CASE WHEN is_active_prev_7d = 1 THEN user_id END), 0) AS retention_7d_rate,
    COUNT(DISTINCT CASE WHEN order_count > 0 THEN user_id END) AS order_users,
    SUM(order_count) AS order_count,
    SUM(pay_amount) AS order_amount,
    SUM(pay_amount) AS gmv,
    SUM(pay_amount) / NULLIF(COUNT(DISTINCT CASE WHEN order_count > 0 THEN user_id END), 0) AS avg_order_value,
    COUNT(DISTINCT CASE WHEN is_active = 1 THEN user_id END) AS uv,
    SUM(cart_add_count) * 1.0 / NULLIF(COUNT(DISTINCT CASE WHEN is_active = 1 THEN user_id END), 0) AS cart_rate,
    COUNT(DISTINCT CASE WHEN order_count > 0 THEN user_id END) * 1.0
        / NULLIF(COUNT(DISTINCT CASE WHEN is_active = 1 THEN user_id END), 0) AS pay_rate,
    SUM(refund_count) * 1.0 / NULLIF(SUM(order_count), 0) AS refund_rate,
    0.001 AS complaint_rate
FROM dws_user_trade_1d
WHERE dt = '${biz_date}';
```

### DIM层（维度层）

维度层存储公共维度表，供各层Join使用。

```sql
-- DIM层：用户维度表（全量快照）
CREATE TABLE IF NOT EXISTS dim_user (
    user_id         BIGINT         COMMENT '用户ID',
    username        STRING         COMMENT '用户名',
    gender          STRING         COMMENT '性别',
    age_range       STRING         COMMENT '年龄段',
    province        STRING         COMMENT '省份',
    city            STRING         COMMENT '城市',
    register_date   DATE           COMMENT '注册日期',
    register_channel STRING       COMMENT '注册渠道',
    member_level    STRING         COMMENT '会员等级',
    tags            ARRAY<STRING>  COMMENT '用户标签',
    is_active       INT            COMMENT '是否活跃',
    etl_time        TIMESTAMP      COMMENT 'ETL更新时间'
)
COMMENT '用户维度表 - DIM层'
PARTITIONED BY (dt STRING COMMENT '数据日期')
STORED AS ORC;

-- DIM层：时间维度表
CREATE TABLE IF NOT EXISTS dim_date (
    date_id         INT            COMMENT '日期ID YYYYMMDD',
    calendar_date   DATE           COMMENT '日历日期',
    year            INT            COMMENT '年',
    quarter         INT            COMMENT '季度',
    month           INT            COMMENT '月',
    week_of_year    INT            COMMENT '年内第几周',
    day_of_week     INT            COMMENT '周内第几天(1=周一)',
    day_name        STRING         COMMENT '星期名称',
    is_weekend      INT            COMMENT '是否周末',
    is_holiday      INT            COMMENT '是否节假日',

)
COMMENT '时间维度表'

```

## 命名规范总表

| 层级 | 前缀 | 表类型 | 命名格式 | 示例 |
|------|------|--------|---------|------|
| ODS | `ods_` | 外部表 | `ods_{source}_{table}` | `ods_mysql_order` |
| DWD | `dwd_` | 内部表 | `dwd_{fact\|dim}_{entity}` | `dwd_fact_order` |
| DWS | `dws_` | 内部表 | `dws_{domain}_{metric}_{granularity}` | `dws_user_trade_1d` |
| ADS | `ads_` | 内部表 | `ads_{report_name}` | `ads_daily_report` |
| DIM | `dim_` | 内部表 | `dim_{entity}` | `dim_user` |

### 字段命名规范

```sql
-- 主键：{表名简写}_id  → user_id, order_id
-- 时间：{动作}_time    → create_time, pay_time, event_time
-- 日期：{前缀}_date    → register_date, order_date, stat_date
-- 标志：is_{状态}      → is_active, is_new_user, is_deleted
-- 数量：{实体}_count   → order_count, click_count
-- 金额：{实体}_amount  → pay_amount, refund_amount
-- 比率：{指标}_rate    → conversion_rate, retention_rate
```
