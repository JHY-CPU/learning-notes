## 后端面试准备


```
// ========== 常见面试题分类 ==========
// 1. 语言基础 (以 Go 为例)
//    - defer 执行顺序 (LIFO)
//    - slice vs array
//    - channel 有无缓冲区别
//    - goroutine 调度模型 (GMP)
//    - interface 底层结构
//
// 2. 数据库
//    - 索引结构 (B-Tree vs Hash)
//    - 隔离级别与 MVCC
//    - 慢查询优化 (EXPLAIN ANALYZE)
//    - 分库分表策略
//    - 索引最左前缀原则
//
// 3. 网络
//    - TCP 三次握手/四次挥手
//    - HTTPS TLS 握手流程
//    - HTTP/2 多路复用
//    - DNS 解析过程
//    - Cookie vs Token vs Session
//
// 4. 系统设计
//    - SNAKE 框架
//    - 短链接、聊天、限流器
//    - CAP 权衡
//    - 缓存策略
//    - 分布式事务
//
// 5. 综合
//    - 从输入 URL 到页面展示全过程
//    - 如何设计高可用系统
//    - 如何排查性能问题
//    - 项目难点与解决方案

// ========== 系统设计面试框架 (SNAKE) ==========
// S — Scope: 明确需求边界
// N — Non-functional: 估算 QPS/存储/延迟
// A — Abstract: 画架构图 (LB → Service → DB → Cache)
// K — Key: 深入核心组件 (数据模型/算法/API)
// E — Evaluate: 瓶颈 + 扩展 + 权衡

// ========== 编码题准备方向 ==========
// 1. 数据结构: 数组/链表/树/哈希表/堆
// 2. 算法: 排序/二分/DFS/BFS/DP
// 3. 语言特性: 并发/错误处理/接口
// 4. 系统设计: 限流器/LRU Cache/线程池

// 推荐: LeetCode Medium 为主
//   Easy 20% + Medium 60% + Hard 20%
```


## 职业成长路径


```
// ========== 后端开发者成长阶梯 ==========
//
// 初级 (0-2 年):
//   - 能完成分配的开发任务
//   - 熟悉一门语言 + 一个框架
//   - 能写 CRUD API + 基本 SQL
//   - 会用 Git / Docker
//
// 中级 (2-5 年):
//   - 独立设计模块/子系统
//   - 掌握多语言 (Java/Go/Python)
//   - 能优化性能 + 解决生产问题
//   - 数据库设计 + 缓存策略
//
// 高级 (5-8 年):
//   - 系统架构设计能力
//   - 分布式系统经验
//   - 技术选型 + 架构决策
//   - 跨团队协作 + Code Review
//
// 架构师 (8+ 年):
//   - 全局架构规划
//   - 复杂业务抽象
//   - 技术战略 + 团队培养
//   - 行业影响力
//
// ========== 专精方向 ==========
// 1. DevOps/SRE
//    K8s, Terraform, CI/CD, 监控, 稳定性
//    技能: 自动化, 运维, 可观测性
//
// 2. 安全工程师
//    渗透测试, 安全审计, 合规
//    技能: 网络/应用/云安全
//
// 3. 数据工程师
//    大数据, 流处理, 数据仓库
//    技能: Spark/Flink/Kafka/ClickHouse
//
// 4. 平台工程师
//    内部工具, 开发者体验
//    技能: K8s Operator, API 设计, SDK

// ========== 实用资源 ==========
// 书籍:
//   - 深入理解计算机系统 (CSAPP)
//   - 设计数据密集型应用 (DDIA)
//   - 系统设计面试 (Alex Xu)
//
// 课程:
//   - MIT 6.824: 分布式系统
//   - CMU 15-445: 数据库系统
//   - Coursera: AWS / GCP 认证
//
// 社区:
//   - GitHub Trending
//   - Stack Overflow
//   - Reddit r/programming
//   - 知乎/掘金/InfoQ
```
