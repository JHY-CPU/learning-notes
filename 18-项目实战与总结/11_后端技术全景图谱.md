## 技术决策树


```
// ========== 技术选型决策 ==========
//
// 1. 项目规模
//   小/原型 → Python/Node.js + SQLite
//   中       → Django/Express + PostgreSQL
//   大       → Spring/Go + 微服务
//
// 2. 实时性要求
//   低 → REST API + 轮询
//   高 → WebSocket / SSE / gRPC Stream
//
// 3. 一致性要求
//   强 → PostgreSQL / 2PC
//   最终 → 异步 + 补偿 Saga
//
// 4. 扩展需求
//   低 → 垂直扩展 (更大的机器)
//   高 → 水平扩展 (无状态 + 分片)
//
// 5. 部署方式
//   传统 → Docker Compose
//   容器 → K8s
//   轻量 → Serverless / PaaS

// ========== 技术栈速查 ==========
// Web API 三件套:
//   FastAPI + PostgreSQL + Redis
//   Express + MongoDB + Redis
//   Spring Boot + JPA + Redis
//   Gin + pgx + go-redis
//
// 消息驱动:
//   Celery + RabbitMQ + Flower
//   Kafka + Kafdrop
//   Bull + Redis + Arena
//
// 监控栈 (P-G-J-E):
//   Prometheus + Grafana + Jaeger + ELK
//   → 指标 + 可视化 + 追踪 + 日志
//
// DevOps 栈:
//   Docker + K8s + Terraform + Ansible
//   GitHub Actions + ArgoCD
//   Helm + Kustomize

// ========== 学习路径推荐 (按角色) ==========
// Web 全栈:
//   Node.js → React → Express → PostgreSQL
//   → Redis → Docker → CI/CD → AWS
//
// 后端专注:
//   Go/Java → Spring/Gin → 数据库
//   → 分布式 → 系统设计 → K8s
//
// 数据工程:
//   Python → SQL → Spark/Flink
//   → Kafka → ClickHouse → dbt
//
// DevOps:
//   Python → Linux → Docker → K8s
//   → Terraform → CI/CD → 监控
```
