# 组件组合与复用

## 1. 组合优于继承

React 官方明确推荐**使用组合（Composition）而非继承（Inheritance））**来复用组件之间的逻辑。

> "In React, composition is the primary mechanism for code reuse." -- React 官方文档

### 为什么不用继承？

- 继承使组件之间的关系紧密耦合，难以修改
- 继承层级深了之后难以追踪数据来源
- 组合提供了更灵活的 API，组件可以自由组装
- 函数组件没有继承的概念，Hook 进一步强化了组合模式

## 2. 容器/展示组件模式（Container / Presentational）

这是 React 早期最经典的复用模式，将组件分为两类：

### 2.1 展示组件（Presentational / Dumb Components）

只关心**如何显示**，不关心数据从哪来。

```jsx
// 展示组件：只负责渲染 UI
function UserList({ users, onUserClick }) {
  return (
    <ul className="user-list">
      {users.map((user) => (
        <li key={user.id} onClick={() => onUserClick(user)}>
          <img src={user.avatar} alt={user.name} />
          <span>{user.name}</span>
        </li>
      ))}
    </ul>
  );
}
```

**特点：**
- 主要包含 JSX 和 CSS
- 数据通过 props 接收
- 通过回调函数通知外部事件
- 不依赖 Redux、Context 等全局状态
- 容易测试和复用

### 2.2 容器组件（Container / Smart Components）

只关心**数据和逻辑**，将数据传递给展示组件。

```jsx
// 容器组件：负责数据获取和业务逻辑
function UserListContainer() {
  const [users, setUsers] = React.useState([]);
  const [loading, setLoading] = React.useState(true);

  React.useEffect(() => {
    fetch('/api/users')
      .then((res) => res.json())
      .then(setUsers)
      .finally(() => setLoading(false));
  }, []);

  const handleUserClick = (user) => {
    navigate(`/users/${user.id}`);
  };

  if (loading) return <Spinner />;

  // 将数据和回调传递给展示组件
  return <UserList users={users} onUserClick={handleUserClick} />;
}
```

**特点：**
- 管理状态和副作用
- 调用 API、处理业务逻辑
- 将数据传给展示组件
- 通常不包含或很少包含样式

### 2.3 模式总结

```
容器组件（逻辑）──props──> 展示组件（UI）
    │                         │
    │  数据获取                │  纯渲染
    │  状态管理                │  无副作用
    │  业务逻辑                │  可复用
```

> **现代 React 中**：自定义 Hook 替代了部分容器组件的职责。数据获取逻辑可以抽到 `useFetch`、`useUsers` 等 Hook 中，展示组件直接使用 Hook 获取数据。模式仍然有效，只是边界更灵活。

## 3. 组件 API 设计

好的组件 API 应该简洁、直观、可组合。

### 3.1 最小化 Props

```jsx
// 不推荐：props 过多，使用者需要了解每个细节
<Button
  type="button"
  variant="primary"
  size="medium"
  disabled={false}
  loading={false}
  icon={<Icon name="check" />}
  iconPosition="left"
  fullWidth={false}
  rounded={false}
  uppercase={true}
  onClick={handleClick}
>
  Submit
</Button>

// 推荐：提供合理的默认值，只暴露真正需要的 props
<Button variant="primary" onClick={handleClick}>
  Submit
</Button>
```

### 3.2 合理的 Prop 命名

```jsx
// 好的命名
<Dialog isOpen={true} onClose={handleClose} />   // 布尔值用 is/has/can/should
<ListItem isSelected onClick={handleClick} />
<FormField isRequired isDisabled />

// 不好的命名
<Dialog open={true} close={handleClose} />       // 语义不够明确
<ListItem selected click={handleClick} />        // 可能与 HTML 属性冲突
```

### 3.3 一致性

同一个项目中的组件 API 风格应保持一致：

```jsx
// 所有表单组件使用相同模式
<Input value={value} onChange={onChange} />
<Select value={value} onChange={onChange} />
<Checkbox checked={checked} onChange={onChange} />
<Radio value={value} onChange={onChange} />
```

## 4. 从通用组件派生专用组件

通过组合通用组件，创建具有特定语义的专用组件。

```jsx
// 通用组件
function Button({ variant, size, children, ...props }) {
  return (
    <button className={`btn btn-${variant} btn-${size}`} {...props}>
      {children}
    </button>
  );
}

// 专用组件：基于通用组件创建
function PrimaryButton(props) {
  return <Button variant="primary" {...props} />;
}

function DangerButton(props) {
  return <Button variant="danger" {...props} />;
}

function IconButton({ icon, ...props }) {
  return (
    <Button variant="ghost" {...props}>
      <span className="icon">{icon}</span>
    </Button>
  );
}

// 使用
<PrimaryButton onClick={save}>Save</PrimaryButton>
<DangerButton onClick={remove}>Delete</DangerButton>
<IconButton icon={<TrashIcon />} onClick={remove} />
```

### 更复杂的派生

```jsx
// 通用的 Dialog
function Dialog({ isOpen, onClose, title, children }) {
  if (!isOpen) return null;
  return (
    <div className="dialog-overlay" onClick={onClose}>
      <div className="dialog" onClick={(e) => e.stopPropagation()}>
        <header>
          <h2>{title}</h2>
          <button onClick={onClose}>X</button>
        </header>
        <div className="dialog-body">{children}</div>
      </div>
    </div>
  );
}

// 专用的确认对话框
function ConfirmDialog({ isOpen, onClose, onConfirm, message }) {
  return (
    <Dialog isOpen={isOpen} onClose={onClose} title="Confirm">
      <p>{message}</p>
      <div className="dialog-actions">
        <button onClick={onClose}>Cancel</button>
        <button onClick={onConfirm}>Confirm</button>
      </div>
    </Dialog>
  );
}

// 使用
<ConfirmDialog
  isOpen={showConfirm}
  onClose={() => setShowConfirm(false)}
  onConfirm={handleDelete}
  message="Are you sure you want to delete this item?"
/>
```

## 5. 基于 Slot 的组合（children 插槽）

通过 `children` 和具名插槽实现灵活的组件组合。

### 5.1 单插槽

```jsx
function Panel({ children }) {
  return (
    <div className="panel">
      {children}
    </div>
  );
}

// 使用者完全控制内容
<Panel>
  <h2>Custom Title</h2>
  <p>Any content here</p>
</Panel>
```

### 5.2 具名插槽

```jsx
function PageLayout({ header, sidebar, footer, children }) {
  return (
    <div className="page">
      <header className="page-header">{header}</header>
      <div className="page-body">
        <aside className="page-sidebar">{sidebar}</aside>
        <main className="page-content">{children}</main>
      </div>
      <footer className="page-footer">{footer}</footer>
    </div>
  );
}

// 使用
<PageLayout
  header={<NavBar />}
  sidebar={<SideNav />}
  footer={<Footer />}
>
  <Article content={articleData} />
</PageLayout>
```

### 5.3 可选插槽

```jsx
function Card({ header, footer, children }) {
  return (
    <div className="card">
      {header && <div className="card-header">{header}</div>}
      <div className="card-body">{children}</div>
      {footer && <div className="card-footer">{footer}</div>}
    </div>
  );
}

// 只使用 body
<Card>
  <p>Simple card content</p>
</Card>

// 使用所有插槽
<Card
  header={<h3>Card Title</h3>}
  footer={<button>Action</button>}
>
  <p>Full card content</p>
</Card>
```

## 6. 布局组件（Layout Components）

布局组件专注于页面或区域的排列，不包含业务逻辑。

### 6.1 Flex / Grid 布局组件

```jsx
function Stack({ direction = 'row', gap = 8, align, justify, wrap, children, ...props }) {
  return (
    <div
      style={{
        display: 'flex',
        flexDirection: direction,
        gap,
        alignItems: align,
        justifyContent: justify,
        flexWrap: wrap ? 'wrap' : 'nowrap',
      }}
      {...props}
    >
      {children}
    </div>
  );
}

function Grid({ columns = 1, gap = 16, children, ...props }) {
  return (
    <div
      style={{
        display: 'grid',
        gridTemplateColumns: `repeat(${columns}, 1fr)`,
        gap,
      }}
      {...props}
    >
      {children}
    </div>
  );
}

// 使用
<Stack direction="column" gap={16} align="center">
  <Avatar />
  <UserName />
  <UserBio />
</Stack>

<Grid columns={3} gap={24}>
  <ProductCard product={p1} />
  <ProductCard product={p2} />
  <ProductCard product={p3} />
</Grid>
```

### 6.2 响应式布局组件

```jsx
function ResponsiveContainer({ children }) {
  return (
    <div className="responsive-container">
      <style>{`
        .responsive-container {
          max-width: 1200px;
          margin: 0 auto;
          padding: 0 16px;
        }
        @media (min-width: 768px) {
          .responsive-container { padding: 0 24px; }
        }
      `}</style>
      {children}
    </div>
  );
}
```

## 7. 高阶组件（HOC）

高阶组件是一个**接收组件并返回新组件**的函数。虽然在现代 React 中更多被自定义 Hook 替代，但理解 HOC 仍然重要。

### 7.1 基本 HOC

```jsx
// HOC：给组件添加 loading 状态
function withLoading(WrappedComponent) {
  return function WithLoadingComponent({ isLoading, ...props }) {
    if (isLoading) {
      return <div className="spinner">Loading...</div>;
    }
    return <WrappedComponent {...props} />;
  };
}

// 使用
const UserListWithLoading = withLoading(UserList);

<UserListWithLoading isLoading={true} users={[]} />
```

### 7.2 HOC 的常见用途

```jsx
// 1. 权限控制
function withAuth(WrappedComponent) {
  return function AuthComponent(props) {
    const { user } = useAuth();
    if (!user) return <Navigate to="/login" />;
    return <WrappedComponent {...props} user={user} />;
  };
}

// 2. 错误边界
function withErrorBoundary(WrappedComponent) {
  return function ErrorBoundaryWrapper(props) {
    return (
      <ErrorBoundary fallback={<ErrorFallback />}>
        <WrappedComponent {...props} />
      </ErrorBoundary>
    );
  };
}

// 3. 日志记录
function withLogger(WrappedComponent, componentName) {
  return function LoggerComponent(props) {
    React.useEffect(() => {
      console.log(`${componentName} mounted`);
      return () => console.log(`${componentName} unmounted`);
    }, []);
    return <WrappedComponent {...props} />;
  };
}
```

### 7.3 HOC 的问题

| 问题 | 描述 |
|------|------|
| Props 来源不明 | 多层 HOC 后难以追踪某个 prop 来自哪里 |
| 命名冲突 | 多个 HOC 可能注入同名 prop |
| Wrapper Hell | 多层 HOC 导致组件树深度增加 |
| 静态方法丢失 | 需要手动 hoist |

> **现代推荐**：优先使用自定义 Hook 替代 HOC。HOC 仅在需要包装 JSX（如 ErrorBoundary）时使用。

## 8. 组合模式总结

| 模式 | 适用场景 | 现代程度 |
|------|---------|---------|
| 容器/展示分离 | 逻辑与 UI 分离 | 仍有效，但 Hook 简化了边界 |
| 通用组件派生 | 从基础组件创建变体 | 常用 |
| Slot / children | 灵活的内容组合 | 核心模式 |
| 布局组件 | 页面排列 | 常用 |
| Render Props | 动态渲染逻辑 | 仍有用，但 Hook 可替代多数场景 |
| HOC | 注入 props / 包装组件 | 逐渐被 Hook 取代 |
| 复合组件 | 多组件共享隐式状态 | 常用（Headless UI 等） |
| 自定义 Hook | 逻辑复用 | **最推荐** |

## 9. 常见面试问题

### Q1：组合和继承的区别？

**组合**：通过包含其他组件来构建 UI，灵活，通过 props 和 children 控制。**继承**：通过 `extends` 创建子类，关系固定，不灵活。React 推荐组合。

### Q2：什么时候用 children，什么时候用 props？

**children**：当组件的主体内容由使用者决定时（如 Card、Layout）。**props**：当组件需要明确的配置参数时（如 variant、size）。两者可以共存。

### Q3：HOC 还有用吗？

在 Error Boundary 等需要包装渲染输出的场景中仍然有用。但数据获取、权限控制等逻辑复用场景，自定义 Hook 是更好的选择。
