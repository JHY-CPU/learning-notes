# 函数组件详解

## 1. 什么是函数组件

函数组件是 React 中最基本的组件形式。本质上，它就是一个 **返回 JSX 的 JavaScript 函数**。React 16.8 之后，函数组件配合 Hook 已经完全替代了 class 组件，成为现代 React 开发的推荐方式。

```jsx
function Welcome() {
  return <h1>Hello, React!</h1>;
}
```

---

## 2. 命名规范：PascalCase

React 组件**必须以大写字母开头**（PascalCase）。React 会将小写标签视为原生 HTML 元素，将大写标签视为自定义组件。

```jsx
// 正确：PascalCase
function MyComponent() {
  return <div>Hello</div>;
}

// 错误：会导致 React 将其当作 HTML 元素解析
function myComponent() {
  return <div>Hello</div>;
}

// 使用组件
<MyComponent />   // 正确
<myComponent />   // 错误，React 会去找 <myComponent> HTML 标签
```

### PascalCase 命名示例

| 场景 | 示例 |
|------|------|
| 普通组件 | `Button`, `Card`, `NavBar` |
| 特定功能组件 | `UserProfile`, `ShoppingCart`, `LoginModal` |
| HOC / 高阶组件 | `withAuth`, `withLogger`（小写 w，这是约定） |
| Hook | `useState`, `useEffect`（use 前缀） |

## 3. 返回 JSX

### 3.1 显式返回（Explicit Return）

使用 `return` 关键字明确返回 JSX，适用于组件体包含多条语句的情况。

```jsx
function Greeting() {
  const name = 'React';
  const version = 18;

  return (
    <div>
      <h1>Hello, {name}!</h1>
      <p>Version: {version}</p>
    </div>
  );
}
```

### 3.2 隐式返回（Implicit Return）

当组件体只有一条表达式时，可以使用箭头函数的隐式返回，省略 `return` 和花括号。

```jsx
// 隐式返回——适合简单组件
const Hello = () => <h1>Hello World!</h1>;

// 如果 JSX 较长，用括号包裹以便阅读
const Card = (props) => (
  <div className="card">
    <h2>{props.title}</h2>
    <p>{props.content}</p>
  </div>
);
```

> **注意**：隐式返回时，JSX 必须与箭头在同一行，或者用 `()` 包裹。不能使用 `;` 结尾。

```jsx
// 错误示例
const Broken = () =>
  <h1>Hello</h1>;  // 语法错误

// 正确写法
const Fixed = () => (
  <h1>Hello</h1>
);
```

### 3.3 条件返回

组件可以根据条件返回不同的 JSX。

```jsx
function StatusBadge({ isActive }) {
  if (isActive) {
    return <span className="badge-active">Active</span>;
  }
  return <span className="badge-inactive">Inactive</span>;
}

// 使用三元表达式（适合简短逻辑）
function Greeting({ isLoggedIn }) {
  return (
    <div>
      {isLoggedIn ? <h1>Welcome back!</h1> : <h1>Please sign in.</h1>}
    </div>
  );
}

// 使用 && 短路运算（适合"有/无"渲染）
function Notification({ message }) {
  return (
    <div>
      {message && <p className="notification">{message}</p>}
    </div>
  );
}
```

## 4. 箭头函数 vs 函数声明

React 中有两种常见的函数组件写法，它们功能等价，选择取决于个人/团队偏好。

### 4.1 函数声明（Function Declaration）

```jsx
function Button({ onClick, children }) {
  return (
    <button onClick={onClick}>
      {children}
    </button>
  );
}
```

**优点：**
- 具名函数，调试时在 React DevTools 中显示清晰的名称
- 函数提升（hoisting），可以在定义之前引用
- 更符合传统组件定义的习惯

### 4.2 箭头函数（Arrow Function）

```jsx
const Button = ({ onClick, children }) => {
  return (
    <button onClick={onClick}>
      {children}
    </button>
  );
};
```

**优点：**
- 语法更简洁
- 在 `map`、`filter` 等场景中定义内联组件更方便
- 与 TypeScript 泛型配合更自然

### 4.3 内联组件（在 map 中定义）

```jsx
function TodoList({ todos }) {
  return (
    <ul>
      {todos.map((todo) => (
        // 箭头函数隐式返回的内联组件
        <li key={todo.id}>
          <span>{todo.text}</span>
        </li>
      ))}
    </ul>
  );
}
```

### 对比总结

| 特性 | 函数声明 | 箭头函数 |
|------|---------|---------|
| 语法 | `function Comp() {}` | `const Comp = () => {}` |
| Hoisting | 支持 | 不支持 |
| `this` 绑定 | 有自己的 `this` | 继承外层 `this`（React 中不常用） |
| TypeScript 泛型 | 略繁琐 | 更简洁 |
| 调试显示名 | 自动获取 | 需要命名导出或 `displayName` |

> **React 官方立场**：两种写法都推荐，没有偏好。保持团队一致即可。

## 5. 组件组合（Composition）

组件的核心价值在于**组合**。通过将小组件嵌套在一起，可以构建复杂的 UI。

### 5.1 基础组合

```jsx
function Avatar({ src, alt }) {
  return <img className="avatar" src={src} alt={alt} />;
}

function UserInfo({ name }) {
  return <span className="user-name">{name}</span>;
}

// 组合使用
function UserCard({ user }) {
  return (
    <div className="user-card">
      <Avatar src={user.avatar} alt={user.name} />
      <UserInfo name={user.name} />
    </div>
  );
}
```

### 5.2 嵌套组合

```jsx
function Page() {
  return (
    <Layout>
      <Header>
        <NavBar>
          <Logo />
          <NavLinks />
        </NavBar>
      </Header>
      <Main>
        <Article />
        <Sidebar />
      </Main>
      <Footer />
    </Layout>
  );
}
```

### 5.3 一个组件只做一件事

每个组件应该专注于一个功能。如果一个组件变得过于复杂，就应该拆分。

```jsx
// 不推荐：一个组件做太多事
function UserDashboard({ user }) {
  // 头像逻辑...
  // 用户名逻辑...
  // 统计数据逻辑...
  // 通知逻辑...
  // 所有 JSX 都在这里
}

// 推荐：拆分成小组件
function UserDashboard({ user }) {
  return (
    <div>
      <UserHeader user={user} />
      <UserStats stats={user.stats} />
      <UserNotifications notifications={user.notifications} />
    </div>
  );
}
```

## 6. 为什么现代 React 使用函数而非类

### 6.1 历史背景

React 早期使用 class 组件来管理状态和生命周期：

```jsx
// 旧式 class 组件（不推荐新项目使用）
class Counter extends React.Component {
  constructor(props) {
    super(props);
    this.state = { count: 0 };
  }

  render() {
    return (
      <button onClick={() => this.setState({ count: this.state.count + 1 })}>
        Count: {this.state.count}
      </button>
    );
  }
}
```

### 6.2 函数组件的优势

| 方面 | 函数组件 | Class 组件 |
|------|---------|-----------|
| 代码量 | 更少，更简洁 | 样板代码多（constructor, this, bind） |
| 状态管理 | `useState` Hook | `this.state` + `setState` |
| 副作用 | `useEffect` | `componentDidMount/Update/Unmount` |
| 逻辑复用 | 自定义 Hook（简单函数） | HOC / Render Props（复杂模式） |
| `this` 问题 | 无 | 需要绑定 `this` |
| TypeScript | 自然 | 泛型写法繁琐 |
| 未来趋势 | 官方推荐 | 仅维护，不再推荐 |
| 性能优化 | `React.memo` | `PureComponent` / `shouldComponentUpdate` |

### 6.3 class 组件仍然存在的场景

- **Error Boundary**：目前只能用 class 组件实现（React 没有提供 Hook 等价物）
- **遗留项目**：大量历史代码可能仍是 class 组件

```jsx
// Error Boundary —— 目前只能用 class 组件
class ErrorBoundary extends React.Component {
  constructor(props) {
    super(props);
    this.state = { hasError: false };
  }

  static getDerivedStateFromError(error) {
    return { hasError: true };
  }

  componentDidCatch(error, errorInfo) {
    console.error('Error caught:', error, errorInfo);
  }

  render() {
    if (this.state.hasError) {
      return <h1>Something went wrong.</h1>;
    }
    return this.props.children;
  }
}
```

## 7. 常见模式与最佳实践

### 7.1 导出方式

```jsx
// 方式一：具名导出（推荐，便于 tree-shaking 和调试）
export function Button({ children }) {
  return <button>{children}</button>;
}

// 方式二：默认导出
export default function Button({ children }) {
  return <button>{children}</button>;
}

// 方式三：定义后导出
function Button({ children }) {
  return <button>{children}</button>;
}
export default Button;
```

> **建议**：尽量使用具名导出，便于 IDE 自动导入和重构。

### 7.2 组件文件结构

```
components/
  Button/
    Button.jsx          # 组件定义
    Button.css          # 组件样式
    Button.test.jsx     # 组件测试
    index.js            # 导出入口
```

`index.js` 内容：
```jsx
export { Button } from './Button';
```

### 7.3 Fragment 简写

当组件需要返回多个兄弟元素时，使用 `<>...</>`（Fragment 简写）替代额外的 `<div>` 包裹。

```jsx
// 不推荐：多余的 div
function NameSection() {
  return (
    <div>
      <h2>First Name</h2>
      <h2>Last Name</h2>
    </div>
  );
}

// 推荐：Fragment
function NameSection() {
  return (
    <>
      <h2>First Name</h2>
      <h2>Last Name</h2>
    </>
  );
}
```

## 8. 常见面试问题

### Q1：函数组件和 class 组件的本质区别是什么？

函数组件是普通函数，接收 props 返回 JSX。Class 组件是 ES6 class，继承 `React.Component`，通过 `this.state` 管理状态，通过生命周期方法处理副作用。现代 React 通过 Hook 赋予了函数组件与 class 组件相同的能力，但代码更简洁。

### Q2：为什么函数组件不能作为 class 组件使用？

函数组件没有 `this`，没有实例，没有生命周期方法。但通过 Hook，函数组件可以获得所有 class 组件的功能。

### Q3：函数组件每次渲染都会重新执行吗？

是的。每次状态或 props 变化时，React 会重新调用函数组件的函数体。但 React 通过 Virtual DOM diff 算法，只更新真实 DOM 中变化的部分。

```jsx
function Counter({ count }) {
  // 每次渲染，这个函数都会被重新调用
  console.log('Counter rendered');

  // 但 DOM 不会全部重建，React 只更新变化的部分
  return <div>Count: {count}</div>;
}
```
