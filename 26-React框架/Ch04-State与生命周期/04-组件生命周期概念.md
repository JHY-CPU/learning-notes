# 组件生命周期概念

## 1. 生命周期的三个阶段

每个 React 组件都会经历三个阶段：

```
Mounting（挂载） ──> Updating（更新） ──> Unmounting（卸载）
   组件首次渲染         状态/属性变化          组件从 DOM 移除
   DOM 创建完成        重新渲染计算             清理资源
```

---

## 2. Class 组件的生命周期方法

虽然现代 React 推荐函数组件和 Hook，但理解 Class 组件的生命周期有助于深入理解 React 的工作原理。

### 2.1 挂载阶段（Mounting）

组件实例被创建并插入 DOM 的过程。

```
constructor()
    ↓
static getDerivedStateFromProps()
    ↓
render()
    ↓
componentDidMount()
```

```jsx
class Example extends React.Component {
  constructor(props) {
    super(props);
    // 1. 初始化 state，绑定方法
    this.state = { data: null };
    console.log('constructor');
  }

  static getDerivedStateFromProps(props, state) {
    // 2. 在 render 之前调用，根据 props 更新 state（很少使用）
    console.log('getDerivedStateFromProps');
    return null;
  }

  componentDidMount() {
    // 4. 组件挂载完成后调用（DOM 已就绪）
    // 典型用途：API 请求、订阅、定时器、DOM 操作
    console.log('componentDidMount');
    fetch('/api/data')
      .then(res => res.json())
      .then(data => this.setState({ data }));
  }

  render() {
    // 3. 渲染 JSX
    console.log('render');
    return <div>{this.state.data}</div>;
  }
}
```

### 2.2 更新阶段（Updating）

Props 或 state 变化时触发。

```
static getDerivedStateFromProps()
    ↓
shouldComponentUpdate()  ──返回 false 则停止──┐
    ↓                                         │
render()                                      │
    ↓                                         │
getSnapshotBeforeUpdate()                     │
    ↓                                         │
componentDidUpdate()                          │
                                              ↓
                                          跳过本次更新
```

```jsx
class Example extends React.Component {
  static getDerivedStateFromProps(props, state) {
    // 1. 每次渲染前调用
    return null;
  }

  shouldComponentUpdate(nextProps, nextState) {
    // 2. 决定是否重新渲染（性能优化）
    console.log('shouldComponentUpdate');
    return nextProps.value !== this.props.value;
  }

  getSnapshotBeforeUpdate(prevProps, prevState) {
    // 4. 在 DOM 更新前调用，返回值传给 componentDidUpdate
    // 典型用途：获取滚动位置
    if (prevState.list.length < this.state.list.length) {
      const list = this.listRef.current;
      return list.scrollHeight - list.scrollTop;
    }
    return null;
  }

  componentDidUpdate(prevProps, prevState, snapshot) {
    // 5. DOM 更新后调用
    // 典型用途：根据变化执行副作用
    console.log('componentDidUpdate');
    if (snapshot !== null) {
      const list = this.listRef.current;
      list.scrollTop = list.scrollHeight - snapshot;
    }
  }

  render() {
    // 3. 渲染
    return <div>{this.props.value}</div>;
  }
}
```

### 2.3 卸载阶段（Unmounting）

组件从 DOM 中移除时调用。

```jsx
class Example extends React.Component {
  componentDidMount() {
    this.timer = setInterval(() => {
      this.setState({ count: this.state.count + 1 });
    }, 1000);

    window.addEventListener('resize', this.handleResize);
  }

  componentWillUnmount() {
    // 清理：定时器、订阅、事件监听器
    console.log('componentWillUnmount');
    clearInterval(this.timer);
    window.removeEventListener('resize', this.handleResize);
  }

  render() {
    return <div>{this.state.count}</div>;
  }
}
```

### 2.4 错误处理

```jsx
class ErrorBoundary extends React.Component {
  static getDerivedStateFromError(error) {
    // 子组件抛出错误时调用，返回新的 state
    return { hasError: true };
  }

  componentDidCatch(error, errorInfo) {
    // 记录错误信息
    console.error('Error:', error);
    console.error('Component Stack:', errorInfo.componentStack);
  }

  render() {
    if (this.state.hasError) {
      return <h1>Something went wrong.</h1>;
    }
    return this.props.children;
  }
}
```

## 3. Hook 如何映射到生命周期

函数组件没有生命周期方法，但 Hook 可以实现相同的功能。

### 3.1 挂载阶段

```jsx
// componentDidMount 等价
function Example() {
  React.useEffect(() => {
    // 组件挂载后执行（首次渲染后）
    console.log('Mounted');
    fetch('/api/data').then(/* ... */);

    // 返回清理函数（componentWillUnmount 等价）
    return () => {
      console.log('Will unmount');
    };
  }, []);  // 空依赖数组 = 只在挂载和卸载时执行

  return <div>Example</div>;
}
```

### 3.2 更新阶段

```jsx
// componentDidUpdate 等价
function Example({ userId }) {
  const [user, setUser] = React.useState(null);

  React.useEffect(() => {
    // userId 变化时执行（包括挂载时）
    fetchUser(userId).then(setUser);
  }, [userId]);  // 依赖数组指定何时重新执行

  return <div>{user?.name}</div>;
}

// 每次渲染后都执行（无依赖数组）
React.useEffect(() => {
  console.log('Updated');
});
```

### 3.3 卸载阶段

```jsx
// componentWillUnmount 等价
function Timer() {
  const [count, setCount] = React.useState(0);

  React.useEffect(() => {
    const timer = setInterval(() => {
      setCount(c => c + 1);
    }, 1000);

    // 清理函数在组件卸载时执行
    return () => clearInterval(timer);
  }, []);

  return <div>{count}</div>;
}
```

### 3.4 映射总览

| Class 生命周期 | Hook 等价 |
|---------------|----------|
| `constructor` | `useState` 的初始值 |
| `componentDidMount` | `useEffect(() => {}, [])` |
| `componentDidUpdate` | `useEffect(() => {}, [deps])` |
| `componentWillUnmount` | `useEffect` 的清理函数 |
| `shouldComponentUpdate` | `React.memo` |
| `getDerivedStateFromProps` | 在函数体中直接计算 |
| `getSnapshotBeforeUpdate` | `useLayoutEffect` + ref |
| `getDerivedStateFromError` | 目前无 Hook 等价 |
| `componentDidCatch` | 目前无 Hook 等价 |

## 4. React Strict Mode 双重渲染

### 4.1 什么是 Strict Mode

`<React.StrictMode>` 是一个开发工具，在开发模式下会**额外执行**一些操作来帮助发现问题。

```jsx
// index.js
import React from 'react';
import ReactDOM from 'react-dom/client';

const root = ReactDOM.createRoot(document.getElementById('root'));
root.render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
);
```

### 4.2 双重渲染

在开发模式下，Strict Mode 会故意让组件渲染两次：

```
// 开发模式下
Mount: constructor → render → render → componentDidMount

// 仅第一次 render 的结果会展示到 DOM，但两次 render 都执行
```

**目的**：发现意外的副作用。如果在 render 中有副作用（如修改全局变量、发起 API 请求），双重渲染会暴露问题。

### 4.3 Strict Mode 检测的问题

1. **不安全的生命周期**：使用了被弃用的 `componentWillMount` 等
2. **过时的 API**：使用了 `findDOMNode`、字符串 ref 等
3. **意外的副作用**：render 函数中有副作用
4. **过时的 Context API**：使用旧版 Context

### 4.4 应对双重渲染

```jsx
function App() {
  // Strict Mode 下会打印两次 "render"
  console.log('render');

  // useEffect 的清理函数会在挂载后立即调用一次（Strict Mode 行为）
  // 然后重新执行 effect
  React.useEffect(() => {
    console.log('effect');
    return () => console.log('cleanup');
  }, []);

  return <div>App</div>;
}

// Strict Mode 开发模式下的行为：
// 1. mount: render → cleanup → effect → cleanup → effect
//    (故意执行 mount → unmount → mount 来测试清理逻辑)
// 2. update: cleanup → effect
```

> **重要**：Strict Mode 的双重渲染**只在开发模式**发生，生产模式不受影响。

## 5. 完整的生命周期流程图

### 函数组件首次渲染

```
函数体执行（render）
    ↓
DOM 更新
    ↓
useLayoutEffect（同步，DOM 更新后立即执行）
    ↓
useEffect（异步，浏览器绘制后执行）
```

### 函数组件更新

```
函数体执行（render）
    ↓
DOM 更新
    ↓
useLayoutEffect 清理函数（旧的）
    ↓
useLayoutEffect（新的）
    ↓
useEffect 清理函数（旧的）
    ↓
useEffect（新的）
```

### 函数组件卸载

```
useLayoutEffect 清理函数
    ↓
useEffect 清理函数
    ↓
DOM 移除
```

## 6. useLayoutEffect vs useEffect

```jsx
// useEffect：浏览器绘制后异步执行，不阻塞渲染
React.useEffect(() => {
  // 用户可能看到闪烁（先看到旧内容，再看到新内容）
  document.title = `Count: ${count}`;
}, [count]);

// useLayoutEffect：DOM 更新后同步执行，阻塞浏览器绘制
React.useLayoutEffect(() => {
  // 用户不会看到中间状态
  const { height } = ref.current.getBoundingClientRect();
  setHeight(height);
}, []);
```

| 特性 | useEffect | useLayoutEffect |
|------|----------|----------------|
| 执行时机 | 浏览器绘制后 | DOM 更新后、绘制前 |
| 同步/异步 | 异步 | 同步 |
| 阻塞渲染 | 否 | 是 |
| 典型用途 | 数据获取、订阅、日志 | DOM 测量、同步更新样式 |
| SSR 兼容 | 是 | 否（会警告） |

> **经验法则**：默认用 `useEffect`。只有在需要在浏览器绘制前读取/修改 DOM 布局时才用 `useLayoutEffect`。

## 7. 组件生命周期与性能

### 7.1 避免在渲染期间执行副作用

```jsx
// 错误：render 中有副作用
function BadComponent() {
  // 每次渲染都会执行，且可能执行两次（Strict Mode）
  document.title = 'Updated';  // 副作用！
  data.push(newItem);          // 副作用！

  return <div>Content</div>;
}

// 正确：副作用放到 useEffect
function GoodComponent() {
  React.useEffect(() => {
    document.title = 'Updated';
  });

  return <div>Content</div>;
}
```

### 7.2 清理过期的异步操作

```jsx
function SearchResults({ query }) {
  const [results, setResults] = React.useState([]);

  React.useEffect(() => {
    let cancelled = false;  // 标记是否取消

    fetch(`/api/search?q=${query}`)
      .then(res => res.json())
      .then(data => {
        if (!cancelled) {  // 只在未取消时更新
          setResults(data);
        }
      });

    return () => {
      cancelled = true;  // 清理：标记为已取消
    };
  }, [query]);

  return results.map(r => <div key={r.id}>{r.name}</div>);
}
```

## 8. 常见面试问题

### Q1：useEffect 和 useLayoutEffect 的区别？

`useEffect` 在浏览器绘制后异步执行，不阻塞渲染。`useLayoutEffect` 在 DOM 更新后同步执行，阻塞浏览器绘制。大多数场景用 `useEffect`，需要同步 DOM 操作时用 `useLayoutEffect`。

### Q2：为什么 Strict Mode 要双重渲染？

帮助开发者发现不纯的渲染和未正确清理的副作用。如果组件是纯函数、副作用有正确的清理逻辑，双重渲染不会产生问题。

### Q3：什么时候用 useLayoutEffect？

当你需要在浏览器绘制前读取 DOM 布局信息（如元素大小、位置），并根据这些信息同步更新 UI 时。常见场景：防止闪烁、计算 tooltip 位置、同步动画。

### Q4：useEffect 的清理函数什么时候执行？

- 组件卸载时
- 下一次 effect 执行前（依赖变化导致重新执行）
- Strict Mode 开发模式下，挂载后也会立即执行一次清理再重新执行
