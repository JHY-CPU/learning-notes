# 状态提升与单向数据流

## 1. 单向数据流（One-Way Data Flow）

React 的数据流向是**自上而下**的：数据从父组件通过 props 流向子组件。子组件不能直接修改父组件的数据，必须通过回调函数通知父组件。

```
父组件 (state) ──props──> 子组件 ──props──> 孙组件
      ^                                    │
      └──────────回调函数────────────────────┘
```

这种单向数据流使应用的数据变化可预测、易于调试。

## 2. 状态提升（Lifting State Up）

当两个或多个组件需要共享同一份数据时，将 state 提升到它们**最近的共同父组件**，然后通过 props 向下传递。

### 2.1 经典案例：温度转换器

```jsx
// 提升前：各自管理自己的 state，不同步
function CelsiusInput({ value, onChange }) {
  return (
    <fieldset>
      <legend>Celsius</legend>
      <input value={value} onChange={(e) => onChange(e.target.value)} />
    </fieldset>
  );
}

function FahrenheitInput({ value, onChange }) {
  return (
    <fieldset>
      <legend>Fahrenheit</legend>
      <input value={value} onChange={(e) => onChange(e.target.value)} />
    </fieldset>
  );
}

// 提升后：共同的父组件管理 state
function TemperatureConverter() {
  const [temperature, setTemperature] = React.useState('');
  const [scale, setScale] = React.useState('c');  // 当前编辑的温度类型

  const celsius = scale === 'f'
    ? tryConvert(temperature, toCelsius)
    : temperature;
  const fahrenheit = scale === 'c'
    ? tryConvert(temperature, toFahrenheit)
    : temperature;

  return (
    <div>
      <CelsiusInput
        value={celsius}
        onChange={(val) => { setTemperature(val); setScale('c'); }}
      />
      <FahrenheitInput
        value={fahrenheit}
        onChange={(val) => { setTemperature(val); setScale('f'); }}
      />
      <BoilingVerdict celsius={parseFloat(celsius)} />
    </div>
  );
}
```

### 2.2 为什么要提升状态

- **单一数据源（Single Source of Truth）**：同一份数据只有一个来源，避免不同步
- **可预测的数据流**：数据从一个地方流出，变化路径清晰
- **便于调试**：可以追踪每次状态变化的来源

## 3. 单一数据源原则

应用中的每一份数据都应该有且只有一个"权威来源"。

### 3.1 违反单一数据源

```jsx
function App() {
  const [user, setUser] = React.useState({ name: 'Alice' });

  return (
    <>
      <Header user={user} />
      <Profile user={user} onUpdate={setUser} />
      {/* 如果 Header 也自己维护了一份 user，就违反了单一数据源 */}
    </>
  );
}
```

### 3.2 派生数据不是独立数据源

```jsx
function App() {
  const [items, setItems] = React.useState([
    { name: 'Apple', price: 3, quantity: 5 },
    { name: 'Banana', price: 2, quantity: 10 },
  ]);

  // 派生数据：从 items 计算而来，不是独立的 state
  const totalPrice = items.reduce((sum, item) => sum + item.price * item.quantity, 0);
  const itemCount = items.length;
  const hasExpensiveItem = items.some(item => item.price > 5);

  return (
    <ShoppingCart
      items={items}
      totalPrice={totalPrice}
      itemCount={itemCount}
      onQuantityChange={(name, qty) => {
        setItems(prev => prev.map(item =>
          item.name === name ? { ...item, quantity: qty } : item
        ));
      }}
    />
  );
}
```

## 4. 何时应该提升状态

问自己以下问题：

| 问题 | 行动 |
|------|------|
| 多个组件需要显示相同的数据？ | 提升 state |
| 多个组件需要修改同一份数据？ | 提升 state |
| 兄弟组件之间需要同步？ | 提升到共同父组件 |
| 数据变化时需要重新计算其他值？ | 提升 state，派生计算 |

### 4.1 示例：兄弟组件通信

```jsx
// 问题：LeftPanel 和 RightPanel 互不通信
function App() {
  return (
    <>
      <LeftPanel />
      <RightPanel />
    </>
  );
}

// 解决：提升搜索状态到父组件
function App() {
  const [searchQuery, setSearchQuery] = React.useState('');

  return (
    <>
      <SearchPanel query={searchQuery} onQueryChange={setSearchQuery} />
      <ResultsPanel query={searchQuery} />
    </>
  );
}
```

### 4.2 示例：表单与预览同步

```jsx
function EditorPage() {
  const [content, setContent] = React.useState('# Hello');

  return (
    <div className="editor-layout">
      <MarkdownEditor value={content} onChange={setContent} />
      <MarkdownPreview content={content} />
    </div>
  );
}

// 编辑器组件
function MarkdownEditor({ value, onChange }) {
  return (
    <textarea
      value={value}
      onChange={(e) => onChange(e.target.value)}
    />
  );
}

// 预览组件
function MarkdownPreview({ content }) {
  return (
    <div
      className="preview"
      dangerouslySetInnerHTML={{ __html: markdownToHtml(content) }}
    />
  );
}
```

## 5. Prop Drilling

当状态提升后需要穿过多个中间组件才能到达目标组件时，就产生了 prop drilling。

### 5.1 问题

```jsx
function App() {
  const [theme, setTheme] = React.useState('light');

  return <Layout theme={theme} setTheme={setTheme} />;
}

function Layout({ theme, setTheme }) {
  // Layout 不需要 theme，只是传递
  return (
    <div>
      <Header theme={theme} setTheme={setTheme} />
      <Main />
    </div>
  );
}

function Header({ theme, setTheme }) {
  // Header 也不直接需要 theme
  return (
    <header>
      <ThemeToggle theme={theme} setTheme={setTheme} />
    </header>
  );
}

function ThemeToggle({ theme, setTheme }) {
  // 终于用到了！但中间穿了两层
  return (
    <button onClick={() => setTheme(theme === 'light' ? 'dark' : 'light')}>
      Current: {theme}
    </button>
  );
}
```

### 5.2 解决方案

**方案一：组件组合**

```jsx
function App() {
  const [theme, setTheme] = React.useState('light');

  return (
    <Layout
      header={
        <Header>
          <ThemeToggle theme={theme} setTheme={setTheme} />
        </Header>
      }
    />
  );
}

function Layout({ header, children }) {
  return (
    <div>
      {header}
      <main>{children}</main>
    </div>
  );
}
```

**方案二：React Context**

```jsx
const ThemeContext = React.createContext();

function App() {
  const [theme, setTheme] = React.useState('light');

  return (
    <ThemeContext.Provider value={{ theme, setTheme }}>
      <Layout />
    </ThemeContext.Provider>
  );
}

function ThemeToggle() {
  const { theme, setTheme } = React.useContext(ThemeContext);
  return (
    <button onClick={() => setTheme(theme === 'light' ? 'dark' : 'light')}>
      {theme}
    </button>
  );
}
```

**方案三：状态管理库**

```jsx
// Zustand 示例
import { create } from 'zustand';

const useThemeStore = create((set) => ({
  theme: 'light',
  toggleTheme: () => set((state) => ({
    theme: state.theme === 'light' ? 'dark' : 'light',
  })),
}));

// 任何组件直接使用
function ThemeToggle() {
  const { theme, toggleTheme } = useThemeStore();
  return <button onClick={toggleTheme}>{theme}</button>;
}
```

## 6. 避免冗余状态

### 6.1 可计算值不存为 state

```jsx
// 不好
function PriceDisplay({ items }) {
  const [total, setTotal] = React.useState(0);

  React.useEffect(() => {
    setTotal(items.reduce((sum, item) => sum + item.price, 0));
  }, [items]);

  return <p>Total: ${total}</p>;
}

// 好：直接派生
function PriceDisplay({ items }) {
  const total = items.reduce((sum, item) => sum + item.price, 0);
  return <p>Total: ${total}</p>;
}
```

### 6.2 Props 的映射不存为 state

```jsx
// 不好
function FullName({ firstName, lastName }) {
  const [fullName, setFullName] = React.useState('');

  React.useEffect(() => {
    setFullName(`${firstName} ${lastName}`);
  }, [firstName, lastName]);
}

// 好
function FullName({ firstName, lastName }) {
  const fullName = `${firstName} ${lastName}`;
  return <span>{fullName}</span>;
}
```

### 6.3 表单值与提交

```jsx
// 不好：提交后还保存着表单值
function ContactForm() {
  const [name, setName] = React.useState('');
  const [submittedName, setSubmittedName] = React.useState('');  // 冗余

  const handleSubmit = () => {
    setSubmittedName(name);  // 没必要
  };
}

// 好：提交后清空表单
function ContactForm() {
  const [name, setName] = React.useState('');

  const handleSubmit = () => {
    sendToServer(name);
    setName('');  // 清空
  };
}
```

## 7. 受控组件模式中的状态提升

表单组件的状态管理是状态提升的典型场景。

### 7.1 受控表单

```jsx
// 状态在父组件
function UserForm() {
  const [formData, setFormData] = React.useState({
    name: '',
    email: '',
    bio: '',
  });

  const updateField = (field) => (e) => {
    setFormData(prev => ({ ...prev, [field]: e.target.value }));
  };

  const handleSubmit = () => {
    console.log(formData);
  };

  return (
    <form onSubmit={handleSubmit}>
      <TextField label="Name" value={formData.name} onChange={updateField('name')} />
      <TextField label="Email" value={formData.email} onChange={updateField('email')} />
      <TextArea label="Bio" value={formData.bio} onChange={updateField('bio')} />
      <button type="submit">Save</button>
    </form>
  );
}

// 表单子组件——不管理状态
function TextField({ label, value, onChange }) {
  return (
    <div>
      <label>{label}</label>
      <input value={value} onChange={onChange} />
    </div>
  );
}
```

### 7.2 验证与状态提升

```jsx
function ValidatedForm() {
  const [email, setEmail] = React.useState('');
  const [error, setError] = React.useState('');

  const handleChange = (value) => {
    setEmail(value);
    // 父组件统一处理验证逻辑
    if (value && !value.includes('@')) {
      setError('Invalid email');
    } else {
      setError('');
    }
  };

  return (
    <FormField
      label="Email"
      value={email}
      onChange={handleChange}
      error={error}
    />
  );
}
```

## 8. 状态提升的权衡

### 优点
- 单一数据源，数据一致
- 变化可追踪
- 便于添加日志和调试

### 缺点
- 可能产生 prop drilling
- 共同父组件可能变得庞大
- 任何子组件变化都会导致整个子树重渲染

### 何时不提升

- 状态只在一个组件内部使用（如 UI 状态：下拉菜单是否打开）
- 状态与其他组件无关（如输入框的输入法中间状态）
- 临时/瞬时状态（如 hover 状态、focus 状态）

```jsx
// 不需要提升：纯 UI 状态
function Dropdown({ options, onSelect }) {
  const [isOpen, setIsOpen] = React.useState(false);  // 不需要提升
  const [highlightedIndex, setHighlightedIndex] = React.useState(-1);  // 不需要提升

  return (
    <div>
      <button onClick={() => setIsOpen(!isOpen)}>Toggle</button>
      {isOpen && (
        <ul>
          {options.map((opt, i) => (
            <li
              key={i}
              className={i === highlightedIndex ? 'highlighted' : ''}
              onMouseEnter={() => setHighlightedIndex(i)}
              onClick={() => { onSelect(opt); setIsOpen(false); }}
            >
              {opt}
            </li>
          ))}
        </ul>
      )}
    </div>
  );
}
```

## 9. 常见面试问题

### Q1：什么是状态提升？为什么要这样做？

将多个组件需要共享的 state 提升到它们最近的共同父组件。目的是实现单一数据源，避免数据不同步。

### Q2：状态提升后遇到 prop drilling 怎么办？

三种方案：组件组合（将需要数据的组件直接放在拥有数据的位置）、React Context（跨层级共享）、状态管理库。

### Q3：什么状态不需要提升？

只在单个组件内部使用的 UI 状态（如开关、高亮索引）、瞬时状态（如 hover）不需要提升。
