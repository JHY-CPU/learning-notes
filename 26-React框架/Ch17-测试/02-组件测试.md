# 组件测试

## 一、render() 与 screen

### 1.1 render 基础

```tsx
import { render, screen, cleanup } from '@testing-library/react';
import { Button } from './Button';

// render 返回的 API
const result = render(<Button variant="primary">提交</Button>);

result.container;          // 根 DOM 元素 (<div>)
result.baseElement;        // document.body
result.getByRole(...);     // 查询方法
result.debug();            // 打印 DOM
result.rerender(<Button variant="secondary">取消</Button>);  // 重新渲染
result.unmount();          // 卸载组件（cleanup 自动调用）

// 推荐使用 screen 对象（从 document.body 查询，更接近用户视角）
render(<Button>提交</Button>);
screen.getByRole('button', { name: '提交' });
```

### 1.2 render 选项

```tsx
// 包装 Wrapper（提供 Context）
const wrapper = ({ children }) => (
  <ThemeProvider>
    <AuthProvider>{children}</AuthProvider>
  </ThemeProvider>
);

render(<UserProfile />, { wrapper });

// 自定义 container
const customContainer = document.createElement('div');
document.body.appendChild(customContainer);
render(<MyComponent />, { container: customContainer });

// hydrate 模式（用于 SSR 测试）
render(<MyComponent />, { hydrate: true });
```

## 二、Screen 查询方法

### 2.1 查询优先级（官方推荐）

```
getByRole          ← 首选（最接近用户感知）
  ↓ 找不到时
getByLabelText     ← 表单元素
  ↓ 找不到时
getByPlaceholderText
  ↓ 找不到时
getByText          ← 非交互元素的内容
  ↓ 找不到时
getByDisplayValue  ← 当前值
  ↓ 找不到时
getByTestId        ← 最后的手段（data-testid）
```

### 2.2 getBy 系列（同步，找不到则抛错）

```tsx
import { render, screen } from '@testing-library/react';

render(
  <form>
    <label htmlFor="email">邮箱</label>
    <input id="email" placeholder="请输入邮箱" defaultValue="test@example.com" />
    <button type="submit">提交</button>
    <span data-testid="error-msg">格式错误</span>
  </form>
);

// getByRole - 按 ARIA 角色查找
// 最推荐！因为用户（和屏幕阅读器）就是这样感知页面的
screen.getByRole('button', { name: '提交' });      // 按钮
screen.getByRole('textbox', { name: '邮箱' });      // 输入框（label 文本）
screen.getByRole('form');                           // 如果有 role="form"

// getByRole 的高级用法
screen.getByRole('button', {
  name: '提交',            // accessible name（aria-label / 文本内容）
  hidden: false,           // 是否包含隐藏元素
  pressed: true,           // aria-pressed 状态
  selected: true,          // aria-selected 状态
  expanded: true,          // aria-expanded 状态
});

// getByLabelText - 通过 label 查找表单元素
screen.getByLabelText('邮箱');  // 找到 htmlFor="email" 关联的 input

// getByPlaceholderText
screen.getByPlaceholderText('请输入邮箱');

// getByText - 按文本内容查找
screen.getByText('格式错误');
screen.getByText(/格式错误/i);   // 正则忽略大小写
screen.getByText((content, element) => {
  return content.startsWith('格式');  // 自定义匹配函数
});

// getByDisplayValue - 按当前值查找
screen.getByDisplayValue('test@example.com');

// getByTestId - 按 data-testid 查找（最后手段）
screen.getByTestId('error-msg');
```

### 2.3 queryBy 系列（同步，返回 null）

```tsx
// 用于断言元素不存在
render(<Alert type="error" visible={false} />);

// ❌ getBy 会抛异常
// screen.getByText('错误信息');  // Error: Unable to find...

// ✅ queryBy 返回 null
expect(screen.queryByText('错误信息')).not.toBeInTheDocument();

// queryAllBy 返回数组
const items = screen.queryAllByRole('listitem');
expect(items).toHaveLength(0);
```

### 2.4 findBy 系列（异步，返回 Promise）

```tsx
// 用于等待异步元素出现
render(<UserProfile userId="123" />);

// findBy 返回 Promise，等待元素出现（默认 1000ms）
const userName = await screen.findByText('张三');
expect(userName).toBeInTheDocument();

// 带超时配置
await screen.findByText('加载完成', {}, { timeout: 3000 });

// findAllBy 返回数组
const items = await screen.findAllByRole('listitem');
expect(items).toHaveLength(5);

// 典型使用场景：等待 loading 消失、数据渲染
it('should display user data after loading', async () => {
  render(<UserList />);

  // 等待数据加载完成
  await screen.findByRole('list');

  // 然后断言内容
  expect(screen.getAllByRole('listitem')).toHaveLength(3);
});
```

### 2.5 查询调试

```tsx
import { screen } from '@testing-library/react';

// 列出所有可用的 role
screen.getByRole('');  // 报错会列出所有可用 role

// 或使用 logRoles
import { logRoles } from '@testing-library/react';
const { container } = render(<MyComponent />);
logRoles(container);

// screen.logTestingPlaygroundURL()
// 生成一个链接，在浏览器中交互式调试查询
screen.logTestingPlaygroundURL();
```

## 三、用户事件 userEvent vs fireEvent

### 3.1 核心区别

```typescript
// fireEvent - 底层事件触发（简单但不完整）
fireEvent.click(button);           // 仅触发 click 事件

// userEvent - 模拟真实用户行为（推荐）
await userEvent.click(button);     // 触发：focus → mousedown → mouseup → click
```

**关键差异：**

| 行为 | fireEvent | userEvent |
|------|-----------|-----------|
| 鼠标移动 | ❌ | ✅ |
| 焦点管理 | ❌ | ✅ |
| 键盘事件序列 | ❌ | ✅ |
| 默认行为 | 部分 | 完整 |
| 异步 | 同步 | 异步 |

### 3.2 userEvent 完整用法

```tsx
import userEvent from '@testing-library/user-event';

// 推荐：先 setup（可复用配置）
const user = userEvent.setup();

// 或快速使用（每个调用自动 setup）
// await userEvent.click(element);

describe('用户交互测试', () => {
  it('完整输入流程', async () => {
    const user = userEvent.setup();
    render(<LoginForm />);

    // 1. 点击输入框（获取焦点）
    const emailInput = screen.getByLabelText('邮箱');
    await user.click(emailInput);

    // 2. 输入文本
    await user.type(emailInput, 'test@example.com');
    // type = 逐字符触发 keydown → keypress → input → keyup

    // 3. 清空并重新输入
    await user.clear(emailInput);
    await user.type(emailInput, 'new@example.com');

    // 4. Tab 键切换焦点
    await user.tab();
    expect(screen.getByLabelText('密码')).toHaveFocus();

    // 5. 输入密码
    await user.type(screen.getByLabelText('密码'), '123456');

    // 6. 按 Enter 提交
    await user.keyboard('{Enter}');

    // 7. 验证提交
    expect(mockSubmit).toHaveBeenCalledWith({
      email: 'new@example.com',
      password: '123456',
    });
  });

  it('键盘导航', async () => {
    const user = userEvent.setup();
    render(<DropdownMenu />);

    // Tab 键前进
    await user.tab();
    expect(screen.getByRole('button', { name: '菜单' })).toHaveFocus();

    // Shift+Tab 后退
    await user.tab({ shift: true });
    expect(screen.getByRole('button', { name: '返回' })).toHaveFocus();

    // 按键组合
    await user.keyboard('{Control>}a{/Control}');  // Ctrl+A
  });

  it('选择操作', async () => {
    const user = userEvent.setup();
    render(<SettingsForm />);

    // 选择下拉框
    await user.selectOptions(
      screen.getByRole('combobox', { name: '语言' }),
      'zh-CN'
    );

    // 勾选复选框
    await user.click(screen.getByRole('checkbox', { name: '接受协议' }));

    // 选择单选按钮
    await user.click(screen.getByRole('radio', { name: '男性' }));

    // 悬浮（触发 tooltip 等）
    await user.hover(screen.getByText('查看详情'));
    expect(screen.getByRole('tooltip')).toBeVisible();

    // 双击
    await user.dblClick(screen.getByText('编辑'));

    // 右键
    await user.pointer([
      { keys: '[MouseRight]', target: screen.getByText('更多') },
    ]);
  });

  // setup 选项
  it('setup options', async () => {
    const user = userEvent.setup({
      delay: 100,           // 每个事件间的延迟（模拟真实输入速度）
      document: customDoc,  // 自定义 document
      skipHover: true,      // 跳过 hover 事件
      skipClick: false,     // 不跳过 click
      writeToClipboard: true, // 允许写剪贴板
    });
  });
});
```

### 3.3 fireEvent 使用场景

```tsx
import { fireEvent } from '@testing-library/react';

// fireEvent 适合的场景：

// 1. 触发非用户交互事件
fireEvent.load(screen.getByAltText('头像'));
fireEvent.error(screen.getByAltText('头像'));
fireEvent.resize(window);
fireEvent.scroll(element, { target: { scrollY: 200 } });

// 2. 自定义事件数据
fireEvent.change(input, {
  target: { value: '自定义值' },  // 不支持 type 的特殊 input
});

// 3. 简单测试中替代 userEvent（不推荐但可用）
fireEvent.click(button);
```

## 四、异步查询与 waitFor

### 4.1 findBy - 等待元素出现

```tsx
// 场景：组件异步加载数据
it('should render user profile after fetch', async () => {
  vi.mocked(api.getUser).mockResolvedValue({ name: '张三', email: 'zhangsan@example.com' });

  render(<UserProfile userId="1" />);

  // 初始状态显示 loading
  expect(screen.getByText('加载中...')).toBeInTheDocument();

  // 等待数据渲染
  expect(await screen.findByText('张三')).toBeInTheDocument();

  // loading 应该消失
  expect(screen.queryByText('加载中...')).not.toBeInTheDocument();
});
```

### 4.2 waitFor - 等待条件满足

```tsx
import { waitFor } from '@testing-library/react';

// 基本用法
await waitFor(() => {
  expect(screen.getByText('保存成功')).toBeInTheDocument();
});

// 带选项
await waitFor(
  () => {
    expect(mockCallback).toHaveBeenCalled();
  },
  {
    timeout: 3000,      // 超时时间
    interval: 50,       // 轮询间隔
    onTimeout: (error) => {
      // 超时自定义处理
      console.log('当前 DOM:', screen.debug());
      return error;
    },
  }
);

// 等待元素状态变化
await waitFor(() => {
  expect(screen.getByRole('button')).not.toBeDisabled();
});

// 等待网络请求完成
await waitFor(() => {
  expect(screen.getByText('3 条评论')).toBeInTheDocument();
});
```

### 4.3 waitForElementToBeRemoved

```tsx
import { waitForElementToBeRemoved } from '@testing-library/react';

// 等待 loading 元素消失
it('should remove loading after data loaded', async () => {
  render(<DataLoader />);

  const loading = screen.getByText('加载中...');

  // 等待元素从 DOM 中移除
  await waitForElementToBeRemoved(loading);

  expect(screen.getByText('数据内容')).toBeInTheDocument();
});

// 也可以用回调
await waitForElementToBeRemoved(() => screen.queryByText('加载中...'));
```

### 4.4 常见异步测试模式

```tsx
// 模式1：等待内容 → 断言
it('displays data after loading', async () => {
  render(<UserList />);
  await screen.findByRole('list');
  expect(screen.getAllByRole('listitem')).toHaveLength(3);
});

// 模式2：触发操作 → 等待变化 → 断言
it('submits form and shows success', async () => {
  const user = userEvent.setup();
  vi.mocked(api.submit).mockResolvedValue({ success: true });

  render(<ContactForm />);
  await user.type(screen.getByLabelText('姓名'), '张三');
  await user.click(screen.getByRole('button', { name: '提交' }));

  expect(await screen.findByText('提交成功')).toBeInTheDocument();
});

// 模式3：等待 loading 消失 → 断言内容
it('shows content after loading', async () => {
  render(<Dashboard />);
  await waitForElementToBeRemoved(() => screen.getByTestId('skeleton'));
  expect(screen.getByText('欢迎回来')).toBeInTheDocument();
});

// 模式4：错误处理
it('shows error on fetch failure', async () => {
  vi.mocked(api.getUsers).mockRejectedValue(new Error('Network Error'));
  render(<UserList />);
  expect(await screen.findByText('加载失败，请重试')).toBeInTheDocument();
});
```

## 五、测试 Hooks

### 5.1 renderHook 基础

```tsx
import { renderHook, act } from '@testing-library/react';

// 基本用法
const { result } = renderHook(() => useCounter());
expect(result.current.count).toBe(0);

// 触发状态更新
act(() => {
  result.current.increment();
});
expect(result.current.count).toBe(1);

// 带初始值
const { result } = renderHook(() => useCounter(10));
expect(result.current.count).toBe(10);

// 带参数
const { result } = renderHook(
  ({ initialValue }) => useCounter(initialValue),
  { initialProps: { initialValue: 5 } }
);

// 更新 props
rerender({ initialValue: 10 });
```

### 5.2 卸载与清理

```tsx
const { result, unmount } = renderHook(() => useEventListener());

// 卸载（触发 cleanup）
unmount();

// 验证清理逻辑
const removeEventListenerSpy = vi.spyOn(window, 'removeEventListener');
const { unmount } = renderHook(() => useEventListener('click', handler));
unmount();
expect(removeEventListenerSpy).toHaveBeenCalledWith('click', handler);
```

## 六、测试 Context

### 6.1 包装 Context Provider

```tsx
import { render, screen } from '@testing-library/react';
import { ThemeProvider, useTheme } from './ThemeContext';
import { ThemeToggle } from './ThemeToggle';

// 方法1：使用 wrapper 选项
it('should toggle theme', async () => {
  const user = userEvent.setup();

  const wrapper = ({ children }) => (
    <ThemeProvider>{children}</ThemeProvider>
  );

  render(<ThemeToggle />, { wrapper });

  expect(screen.getByText('当前主题: light')).toBeInTheDocument();

  await user.click(screen.getByRole('button', { name: '切换主题' }));

  expect(screen.getByText('当前主题: dark')).toBeInTheDocument();
});

// 方法2：自定义 render 包装（复用）
function renderWithTheme(ui, { theme = 'light', ...options } = {}) {
  function Wrapper({ children }) {
    return <ThemeProvider initialTheme={theme}>{children}</ThemeProvider>;
  }
  return render(ui, { wrapper: Wrapper, ...options });
}

it('renders with dark theme', () => {
  renderWithTheme(<ThemeToggle />, { theme: 'dark' });
  expect(screen.getByText('当前主题: dark')).toBeInTheDocument();
});

// 方法3：测试 useTheme hook 直接
it('useTheme provides correct values', () => {
  const wrapper = ({ children }) => (
    <ThemeProvider>{children}</ThemeProvider>
  );

  const { result } = renderHook(() => useTheme(), { wrapper });

  expect(result.current.theme).toBe('light');
  expect(typeof result.current.toggleTheme).toBe('function');

  act(() => {
    result.current.toggleTheme();
  });

  expect(result.current.theme).toBe('dark');
});
```

### 6.2 测试多层嵌套 Context

## 八、Snapshot 测试

### 8.1 基本用法

```tsx
// 生成快照
it('should match snapshot', () => {
  const { container } = render(<UserCard user={mockUser} />);
  expect(container.firstChild).toMatchSnapshot();
});

// 内联快照（推荐，更易读）
it('should render user card', () => {
  render(<UserCard user={mockUser} />);
  expect(screen.getByRole('article')).toMatchInlineSnapshot(`
    <article class="user-card">
      <h3>张三</h3>
      <p>前端工程师</p>
    </article>
  `);
});
```

### 8.2 Snapshot 测试陷阱

```tsx
// ❌ 陷阱1：包含动态数据的快照
it('matches snapshot', () => {
  render(<UserCard user={mockUser} />);
  // 如果组件有 Date.now() 或 Math.random()，快照每次都会不同
  expect(container).toMatchSnapshot();
});

// ✅ 解决：mock 动态值
vi.spyOn(Date, 'now').mockReturnValue(1700000000000);
vi.spyOn(Math, 'random').mockReturnValue(0.5);

// ❌ 陷阱2：过大的快照（失去可读性）
it('matches whole page snapshot', () => {
  render(<App />);
  expect(container).toMatchSnapshot();  // 几千行快照，没人看
});

// ✅ 解决：针对特定组件生成小快照
it('matches header snapshot', () => {
  render(<Header />);
  expect(screen.getByRole('banner')).toMatchSnapshot();
});

// ❌ 陷阱3：快照过期不更新
// 快照只是记录了"当前"状态，不验证"正确"状态
// 当组件故意改变时，开发者往往会盲目更新快照

// ✅ 正确使用方式：
// 快照适合用于：
// 1. 稳定的、不常变化的 UI（如 logo 组件）
// 2. API 响应格式验证
// 3. AST 或配置文件的结构验证
// 不适合用于：
// 1. 频繁变化的 UI
// 2. 需要验证业务逻辑的组件
```

### 8.3 更新快照

```bash
# Jest
npx jest --updateSnapshot      # 更新所有快照
npx jest -u                    # 简写

# Vitest
npx vitest run --update        # 更新所有快照
# 在 watch 模式下按 u 键更新
```

## 九、综合示例：完整组件测试

```tsx
// UserForm.test.tsx
import { render, screen, waitFor } from '@/test/test-utils';
import userEvent from '@testing-library/user-event';
import { UserForm } from './UserForm';
import { createUser } from '@/api/users';
import type { User } from '@/types';

vi.mock('@/api/users');

const mockUser: Partial<User> = {
  name: '',
  email: '',
  role: 'viewer',
};

describe('UserForm', () => {
  const mockOnSuccess = vi.fn();
  const mockOnCancel = vi.fn();

  beforeEach(() => {
    vi.clearAllMocks();
  });

  it('renders all form fields', () => {
    render(<UserForm onSuccess={mockOnSuccess} onCancel={mockOnCancel} />);

    expect(screen.getByRole('form', { name: '用户表单' })).toBeInTheDocument();
    expect(screen.getByLabelText('姓名')).toBeInTheDocument();
    expect(screen.getByLabelText('邮箱')).toBeInTheDocument();
    expect(screen.getByRole('combobox', { name: '角色' })).toBeInTheDocument();
    expect(screen.getByRole('button', { name: '创建' })).toBeInTheDocument();
    expect(screen.getByRole('button', { name: '取消' })).toBeInTheDocument();
  });

  it('validates required fields', async () => {
    const user = userEvent.setup();
    render(<UserForm onSuccess={mockOnSuccess} />);

    await user.click(screen.getByRole('button', { name: '创建' }));

    expect(await screen.findByText('姓名不能为空')).toBeInTheDocument();
    expect(screen.getByText('邮箱不能为空')).toBeInTheDocument();
    expect(createUser).not.toHaveBeenCalled();
  });

  it('validates email format', async () => {
    const user = userEvent.setup();
    render(<UserForm onSuccess={mockOnSuccess} />);

    await user.type(screen.getByLabelText('姓名'), '张三');
    await user.type(screen.getByLabelText('邮箱'), 'invalid-email');
    await user.click(screen.getByRole('button', { name: '创建' }));

    expect(await screen.findByText('邮箱格式不正确')).toBeInTheDocument();
  });

  it('submits form with valid data', async () => {
    const user = userEvent.setup();
    const createdUser = { id: 1, name: '张三', email: 'zhangsan@example.com', role: 'viewer' };
    vi.mocked(createUser).mockResolvedValue(createdUser);

    render(<UserForm onSuccess={mockOnSuccess} />);

    await user.type(screen.getByLabelText('姓名'), '张三');
    await user.type(screen.getByLabelText('邮箱'), 'zhangsan@example.com');
    await user.selectOptions(screen.getByRole('combobox', { name: '角色' }), 'editor');
    await user.click(screen.getByRole('button', { name: '创建' }));

    await waitFor(() => {
      expect(createUser).toHaveBeenCalledWith({
        name: '张三',
        email: 'zhangsan@example.com',
        role: 'editor',
      });
    });

    expect(mockOnSuccess).toHaveBeenCalledWith(createdUser);
  });

  it('shows error when submission fails', async () => {
    const user = userEvent.setup();
    vi.mocked(createUser).mockRejectedValue(new Error('邮箱已存在'));

    render(<UserForm onSuccess={mockOnSuccess} />);

    await user.type(screen.getByLabelText('姓名'), '张三');
    await user.type(screen.getByLabelText('邮箱'), 'zhangsan@example.com');
    await user.click(screen.getByRole('button', { name: '创建' }));

    expect(await screen.findByText('创建失败: 邮箱已存在')).toBeInTheDocument();
    expect(mockOnSuccess).not.toHaveBeenCalled();
  });

  it('disables form during submission', async () => {
    const user = userEvent.setup();
    // 让 API 永不 resolve，模拟提交中
    vi.mocked(createUser).mockImplementation(() => new Promise(() => {}));

    render(<UserForm onSuccess={mockOnSuccess} />);

    await user.type(screen.getByLabelText('姓名'), '张三');
    await user.type(screen.getByLabelText('邮箱'), 'zhangsan@example.com');
    await user.click(screen.getByRole('button', { name: '创建' }));

    expect(screen.getByRole('button', { name: '创建中...' })).toBeDisabled();
    expect(screen.getByLabelText('姓名')).toBeDisabled();
  });

  it('calls onCancel when cancel button clicked', async () => {
    const user = userEvent.setup();
    render(<UserForm onSuccess={mockOnSuccess} onCancel={mockOnCancel} />);

    await user.click(screen.getByRole('button', { name: '取消' }));

    expect(mockOnCancel).toHaveBeenCalledTimes(1);
  });
});
```

## 总结

| 概念 | 要点 |
|------|------|
| **查询优先级** | getByRole > getByLabelText > getByText > getByTestId |
| **userEvent vs fireEvent** | 优先 userEvent，它模拟真实用户行为 |
| **异步查询** | findBy 等待出现，waitFor 等待条件 |
| **Mock** | vi.mock 模块级，vi.fn() 函数级 |
| **Context 测试** | 使用 wrapper 包装 Provider |
| **Snapshot** | 适合稳定 UI，不适合频繁变化的组件 |
