## 二、Vite

### 2.1 安装与配置

```bash
# 创建项目
npm create vite@latest my-app -- --template react-ts

# 安装依赖
cd my-app && npm install
```

```typescript
// vite.config.ts
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import path from 'path';

export default defineConfig({
  plugins: [
    react({
      // JSX 运行时
      jsxImportSource: '@emotion/react',  // 如果使用 Emotion
      babel: {
        plugins: [
          ['@babel/plugin-proposal-decorators', { legacy: true }],
        ],
      },
    }),
  ],

  // 路径别名
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src'),
      '@components': path.resolve(__dirname, './src/components'),
      '@utils': path.resolve(__dirname, './src/utils'),
    },
  },

  // 开发服务器
  server: {
    port: 3000,
    open: true,
    proxy: {
      '/api': {
        target: 'http://localhost:8080',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/api/, ''),
      },
    },
  },

  // 构建配置
  build: {
    outDir: 'dist',
    sourcemap: true,
    rollupOptions: {
      output: {
        manualChunks: {
          vendor: ['react', 'react-dom'],
          router: ['react-router-dom'],
          query: ['@tanstack/react-query'],
        },
      },
    },
    // 代码压缩
    minify: 'esbuild',
    // 资源内联阈值
    assetsInlineLimit: 4096,
  },

  // CSS 配置
  css: {
    modules: {
      localsConvention: 'camelCaseOnly',
    },
    preprocessorOptions: {
      scss: {
        additionalData: `@import "@/styles/variables.scss";`,
      },
    },
  },
});
```

### 2.2 Vite 工作原理

```
传统打包器（Webpack）：
1. 分析依赖图
2. 打包所有模块
3. 启动开发服务器
→ 项目越大，启动越慢

Vite（基于 ESM）：
1. 启动开发服务器（极快）
2. 浏览器请求模块时，按需编译
3. 未访问的模块不编译
→ 启动速度与项目大小无关

HMR 对比：
Webpack：修改模块 → 重新构建依赖图 → 更新（慢）
Vite：修改模块 → 只编译该模块 → 更新（毫秒级）
```

### 2.3 常用 Vite 插件

```typescript
// vite.config.ts
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import checker from 'vite-plugin-checker';
import svgr from 'vite-plugin-svgr';
import { ViteImageOptimizer } from 'vite-plugin-image-optimizer';
import { visualizer } from 'rollup-plugin-visualizer';

export default defineConfig({
  plugins: [
    react(),

    // TypeScript 类型检查（并行）
    checker({
      typescript: true,
      eslint: {
        lintCommand: 'eslint "./src/**/*.{ts,tsx}"',
      },
    }),

    // SVG 作为组件导入
    svgr({
      svgrOptions: { icon: true },
    }),

    // 图片压缩
    ViteImageOptimizer({
      jpeg: { quality: 80 },
      png: { quality: 80 },
      webp: { quality: 80 },
    }),

    // 构建分析
    visualizer({
      filename: 'dist/stats.html',
      open: true,
      gzipSize: true,
    }),
  ],
});
```

## 三、Webpack

### 3.1 基本配置

```javascript
// webpack.config.js
const path = require('path');
const HtmlWebpackPlugin = require('html-webpack-plugin');
const MiniCssExtractPlugin = require('mini-css-extract-plugin');
const { BundleAnalyzerPlugin } = require('webpack-bundle-analyzer');

module.exports = (env, argv) => {
  const isProd = argv.mode === 'production';

  return {
    entry: './src/index.tsx',

    output: {
      path: path.resolve(__dirname, 'dist'),
      filename: isProd ? 'js/[name].[contenthash:8].js' : 'js/[name].js',
      chunkFilename: isProd ? 'js/[name].[contenthash:8].chunk.js' : 'js/[name].chunk.js',
      clean: true,
      publicPath: '/',
    },

    resolve: {
      extensions: ['.ts', '.tsx', '.js', '.jsx'],
      alias: {
        '@': path.resolve(__dirname, 'src'),
      },
    },

    module: {
      rules: [
        // TypeScript/JSX
        {
          test: /\.(ts|tsx)$/,
          exclude: /node_modules/,
          use: {
            loader: 'babel-loader',
            options: {
              presets: [
                ['@babel/preset-env', { targets: 'defaults' }],
                ['@babel/preset-react', { runtime: 'automatic' }],
                '@babel/preset-typescript',
              ],
              plugins: [
                isProd && ['transform-react-remove-prop-types'],
              ].filter(Boolean),
            },
          },
        },

        // CSS Modules
        {
          test: /\.module\.css$/,
          use: [
            isProd ? MiniCssExtractPlugin.loader : 'style-loader',
            {
              loader: 'css-loader',
              options: {
                modules: {
                  localIdentName: isProd
                    ? '[hash:base64:8]'
                    : '[name]__[local]--[hash:base64:5]',
                },
              },
            },
            'postcss-loader',
          ],
        },

        // 普通 CSS
        {
          test: /\.css$/,
          exclude: /\.module\.css$/,
          use: [
            isProd ? MiniCssExtractPlugin.loader : 'style-loader',
            'css-loader',
            'postcss-loader',
          ],
        },

        // 图片
        {
          test: /\.(png|jpg|gif|svg)$/,
          type: 'asset',
          parser: {
            dataUrlCondition: { maxSize: 10 * 1024 },
          },
          generator: {
            filename: 'images/[name].[hash:8][ext]',
          },
        },
      ],
    },

    plugins: [
      new HtmlWebpackPlugin({
        template: './public/index.html',
        favicon: './public/favicon.ico',
      }),

      isProd && new MiniCssExtractPlugin({
        filename: 'css/[name].[contenthash:8].css',
      }),

      env.analyze && new BundleAnalyzerPlugin(),
    ].filter(Boolean),

    optimization: {
      splitChunks: {
        chunks: 'all',
        cacheGroups: {
          vendor: {
            test: /[\\/]node_modules[\\/]/,
            name: 'vendors',
            chunks: 'all',
          },
        },
      },
    },

    devServer: {
      port: 3000,
      historyApiFallback: true,
      hot: true,
      proxy: {
        '/api': 'http://localhost:8080',
      },
    },
  };
};
```

## 四、Turbopack

### 4.1 简介

```
Turbopack - Vercel 开发的 Rust 编写的打包器
- 用于替代 Webpack
- Next.js 13+ 内置支持
- 增量编译，比 Webpack 快 700 倍
```

```bash
# 在 Next.js 中使用
next dev --turbo  # 启用 Turbopack

# 目前仅支持开发模式
# 生产构建仍使用 Webpack
```


```javascript
        'chore',    // 杂务
```

## 九、Monorepo

### 9.1 pnpm Workspaces

```yaml
# pnpm-workspace.yaml
packages:
  - 'packages/*'
  - 'apps/*'
```

```
my-monorepo/
├── apps/
│   ├── web/           # React 前端
│   └── admin/         # 管理后台
├── packages/
│   ├── ui/            # 共享 UI 组件
│   ├── utils/         # 共享工具函数
│   ├── api-client/    # API 客户端
│   └── tsconfig/      # 共享 TS 配置
├── package.json
└── pnpm-workspace.yaml
```

```json
// packages/ui/package.json
{
  "name": "@myproject/ui",
  "version": "0.1.0",
  "main": "./src/index.ts",
  "types": "./src/index.ts",
  "exports": {
    ".": "./src/index.ts",
    "./styles": "./src/styles.css"
  },
  "dependencies": {
    "react": "^18.3.0"
  }
}

// apps/web/package.json
{
  "name": "@myproject/web",
  "dependencies": {
    "@myproject/ui": "workspace:*",
    "@myproject/utils": "workspace:*"
  }
}
```

### 9.2 Turborepo

```json
// turbo.json
{
  "$schema": "https://turbo.build/schema.json",
  "globalDependencies": ["**/.env.*local"],
  "pipeline": {
    "build": {
      "dependsOn": ["^build"],
      "outputs": ["dist/**", ".next/**"]
    },
    "dev": {
      "cache": false,
      "persistent": true
    },
    "lint": {},
    "test": {
      "dependsOn": ["build"],
      "outputs": ["coverage/**"]
    },
    "typecheck": {}
  }
}
```

```bash
# 运行所有包的 build
turbo run build

# 只运行修改过的包
turbo run build --filter=...[main]

# 并行运行开发服务器
turbo run dev --parallel
```

## 总结

| 工具 | 用途 | 推荐度 |
|------|------|--------|
| **Vite** | 构建工具 | 新项目首选 |
| **ESLint** | 代码质量 | 必须 |
| **Prettier** | 代码格式 | 必须 |
| **Husky + lint-staged** | Git hooks | 推荐 |
| **commitlint** | 提交规范 | 团队项目推荐 |
| **pnpm + Turborepo** | Monorepo | 多包项目 |
| **TypeScript** | 类型系统 | 强烈推荐 |
