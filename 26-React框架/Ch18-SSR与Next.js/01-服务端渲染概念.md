# 服务端渲染概念

## 一、渲染策略对比

### 1.1 四种渲染方式

```
CSR (Client-Side Rendering)  - 客户端渲染
SSR (Server-Side Rendering)  - 服务端渲染
SSG (Static Site Generation) - 静态站点生成
ISR (Incremental Static Regeneration) - 增量静态再生
```

### 1.2 详细对比

```
┌──────────┬────────────────┬────────────────┬────────────────┬────────────────┐
│          │      CSR       │      SSR       │      SSG       │      ISR       │
├──────────┼────────────────┼────────────────┼────────────────┼────────────────┤
│ 渲染时机 │ 浏览器运行时   │ 每次请求时     │ 构建时         │ 构建+定时再生   │
│ HTML内容 │ 空壳           │ 完整HTML       │ 完整HTML       │ 完整HTML       │
│ 数据新鲜│ 最新           │ 最新           │ 构建时数据     │ 定时更新       │
│ TTFB     │ 快(静态文件)   │ 慢(需服务计算) │ 快(预生成)     │ 快(预生成)     │
│ FCP      │ 慢(需JS加载)   │ 快(HTML直达)   │ 快(HTML直达)   │ 快(HTML直达)   │
│ SEO      │ 差(空HTML)     │ 好(完整HTML)   │ 好(完整HTML)   │ 好(完整HTML)   │
│ 服务端   │ 不需要         │ 需要           │ 不需要(可CDN)  │ 需要           │
│ 适用场景 │ 管理后台       │ 电商、新闻     │ 博客、文档     │ 新闻、商品     │
└──────────┴────────────────┴────────────────┴────────────────┴────────────────┘
```

### 1.3 请求流程对比

```
=== CSR 流程 ===
浏览器 → 请求HTML → 返回空HTML + JS → JS执行 → 请求数据 → 渲染页面
时间线: [===加载JS===][===请求数据===][===渲染===]

=== SSR 流程 ===
浏览器 → 请求HTML → 服务端请求数据 → 生成完整HTML → 返回HTML → JS hydrate
时间线: [========服务端处理========][===返回HTML===][=hydrate=]

=== SSG 流程 ===
构建时: 数据请求 → 生成HTML → 上传CDN
浏览器 → 请求HTML → CDN返回预生成HTML → JS hydrate
时间线: [===CDN返回===][=hydrate=]
```

## 二、为什么需要 SSR

### 2.1 SEO 优化

```html
<!-- CSR 返回的 HTML（搜索引擎看不到内容） -->
<html>
<body>
  <div id="root"></div>  <!-- 空的！ -->
  <script src="/bundle.js"></script>
</body>
</html>

<!-- SSR 返回的 HTML（搜索引擎可直接读取） -->
<html>
<head>
  <title>React 学习指南 - 前端进阶必备</title>
  <meta name="description" content="最全面的 React 学习教程..." />
</head>
<body>
  <div id="root">
    <h1>React 学习指南</h1>
    <article>
      <h2>第一章：组件基础</h2>
      <p>React 组件是...</p>
    </article>
  </div>
  <script src="/bundle.js"></script>
</body>
</html>
```

### 2.2 首屏性能

```
CSR 性能问题：
- 用户看到白屏时间长
- 需要等 JS 下载 + 执行 + API 请求 + 渲染
- 低端设备更慢

SSR 优势：
- 用户更快看到内容
- 浏览器可并行处理：HTML渲染 + CSS加载 + JS下载
- 渐进式展示：内容先到，交互后到
```

### 2.3 社交分享

```
当用户在微信/Twitter/Facebook分享链接时：
- 平台爬虫请求 URL 获取 meta 信息
- CSR 页面只有空 HTML → 分享卡片没有标题/图片/描述
- SSR 页面有完整 meta → 分享卡片显示正确信息

Open Graph 示例：
<meta property="og:title" content="React 学习指南" />
<meta property="og:description" content="最全面的 React 教程" />
<meta property="og:image" content="https://example.com/cover.jpg" />
<meta property="og:url" content="https://example.com/react-guide" />
```

## 三、Hydration（注水/激活）过程

### 3.1 什么是 Hydration

```
SSR 渲染的 HTML 是"静态的"——不能交互
Hydration = 让静态 HTML 变成可交互的 React 应用

过程：
1. 浏览器收到完整 HTML → 用户看到内容（FCP 完成）
2. 浏览器下载 JS bundle
3. React 在已有 DOM 上"附加"事件处理器
4. 应用变为可交互（TTI 完成）
```

```
Timeline:
[===SSR 返回HTML===][==用户看到内容==][===下载JS===][===Hydrate===][交互就绪]
                   ↑ FCP                            ↑ TTI
```

### 3.2 Hydration 的问题

```tsx
// 问题1：服务端和客户端 HTML 不匹配
// Warning: Text content did not match

function TimeDisplay() {
  // 服务端渲染时：2024-01-15 10:00:00
  // 客户端渲染时：2024-01-15 10:00:01（时间不同！）
  return <div>{new Date().toLocaleString()}</div>;
}

// 解决：只在客户端渲染动态内容
function TimeDisplay() {
  const [time, setTime] = useState('');

  useEffect(() => {
    setTime(new Date().toLocaleString());
  }, []);

  return <div>{time || '...'}</div>;
}

// 或使用 suppressHydrationWarning
function TimeDisplay() {
  return <div suppressHydrationWarning>{new Date().toLocaleString()}</div>;
}
```

```tsx
// 问题2：Hydration 过程阻塞主线程
// 大量组件需要 hydrate → 页面卡顿

// 解决方案：选择性注水（Selective Hydration）
// React 18 的 Suspense 边界
<Suspense fallback={<Skeleton />}>
  <Comments />  {/* 不阻塞主要内容的 hydrate */}
</Suspense>
```

### 3.3 流式 SSR（Streaming SSR）

```
传统 SSR：服务端一次性生成完整 HTML → 整体返回
流式 SSR：边生成边返回 → 浏览器逐步展示

=== 传统 SSR ===
Server: [==========生成完整HTML==========]
Client:                          [==接收==][==渲染==]
                                     ↑ 用户开始看到页面

=== 流式 SSR ===
Server: [==Shell==][==数据加载==][==填充数据==]
Client: [==Shell==]↑先展示     [==填充数据==]
                    ↑ 用户更快看到骨架
```

```tsx
// React 18 流式 SSR
// 使用 Suspense 实现流式渲染
function App() {
  return (
    <html>
      <body>
        <Layout>
          <Header />
          <main>
            {/* 主要内容立即发送 */}
            <ArticleContent />

            {/* 评论区异步加载，不影响整体 HTML 的发送 */}
            <Suspense fallback={<CommentsSkeleton />}>
              <Comments />
            </Suspense>
          </main>
          <Footer />
        </Layout>
      </body>
    </html>
  );
}
```

## 四、React DOM Server API

### 4.1 renderToString（传统）

```tsx
import { renderToString } from 'react-dom/server';

// 同步渲染为 HTML 字符串
const html = renderToString(<App />);

// 完整使用示例
import express from 'express';
import { renderToString } from 'react-dom/server';
import App from './App';

const app = express();

app.get('*', (req, res) => {
  const html = renderToString(<App url={req.url} />);

  res.send(`
    <!DOCTYPE html>
    <html>
      <head><title>My App</title></head>
      <body>
        <div id="root">${html}</div>
        <script src="/bundle.js"></script>
      </body>
    </html>
  `);
});

// 局限：
// ❌ 同步阻塞 - 所有数据必须在渲染前准备好
// ❌ 不支持 Suspense - 无法流式传输
// ❌ 不支持流 - 必须等整个树渲染完
```

### 4.2 renderToPipeableStream（推荐，React 18+）

```tsx
import { renderToPipeableStream } from 'react-dom/server';
import express from 'express';
import App from './App';

const app = express();

app.get('*', (req, res) => {
  const { pipe, abort } = renderToPipeableStream(<App url={req.url} />, {
    // Shell HTML 渲染完成后回调
    onShellReady() {
      // 如果是爬虫或需要完整 HTML
      if (isCrawler(req)) return;

      // 设置响应头
      res.statusCode = 200;
      res.setHeader('Content-Type', 'text/html');

      // 开始流式传输
      res.write('<!DOCTYPE html><html><head><title>My App</title></head><body><div id="root">');
      pipe(res);
      // 注意：需要在 Suspense fallback 后补上闭合标签
    },

    // 所有内容（包括 Suspense fallback）渲染完成
    onAllReady() {
      // 爬虫需要完整 HTML
      if (isCrawler(req)) {
        res.statusCode = 200;
        res.setHeader('Content-Type', 'text/html');
        res.write('<!DOCTYPE html><html><head><title>My App</title></head><body><div id="root">');
        pipe(res);
      }
    },

    // 错误处理
    onError(error) {
      console.error('SSR Error:', error);
      res.statusCode = 500;
      res.end('Internal Server Error');
    },

    // Bootstrap scripts（在 HTML 后加载的 JS）
    bootstrapScripts: ['/client.js'],
  });

  // 超时处理
  setTimeout(() => abort(), 10000);
});
```

### 4.3 renderToReadableStream（Web Streams API）

```tsx
import { renderToReadableStream } from 'react-dom/server';

// 用于支持 Web Streams 的环境（如 Cloudflare Workers、Deno）
async function handleRequest(request: Request) {
  const stream = await renderToReadableStream(<App />);

  return new Response(stream, {
    headers: { 'Content-Type': 'text/html' },
  });
}
```

### 4.4 API 对比

```
┌───────────────────────┬───────────┬────────────┬──────────────┐
│ API                   │ 流式      │ Suspense   │ 适用环境      │
├───────────────────────┼───────────┼────────────┼──────────────┤
│ renderToString        │ ❌        │ ❌         │ 旧项目兼容    │
│ renderToStaticMarkup  │ ❌        │ ❌         │ 静态邮件模板  │
│ renderToPipeableStream│ ✅        │ ✅         │ Node.js      │
│ renderToReadableStream│ ✅        │ ✅         │ Web Workers  │
└───────────────────────┴───────────┴────────────┴──────────────┘
```

## 六、各种渲染模式选择指南

### 6.1 决策树

```
你的页面需要实时数据吗？
├── 是 → 数据每次请求都不同吗？
│   ├── 是 → SSR（服务端渲染）
│   └── 否 → ISR（增量静态再生，定时更新）
└── 否 → 页面数量多吗？
    ├── 是 → ISR（按需生成）
    └── 否 → SSG（构建时全部生成）
```

### 6.2 实际项目中的混合使用

```
一个网站通常混合多种渲染策略：

/ (首页)              → SSG（内容相对固定）
/products/[id]        → ISR（每 60 秒重新验证）
/search?q=xxx         → SSR（搜索结果实时）
/dashboard            → CSR（登录后，不需要 SEO）
/blog/[slug]          → SSG（构建时生成所有文章）
/blog/new             → CSR（写文章页面，不需要 SEO）
/api/*                → API Routes（服务端逻辑）
```

### 6.3 性能对比数据

```
典型电商产品页性能数据：

CSR:
- TTFB: 50ms（返回空 HTML）
- FCP:  2800ms（等 JS 加载执行）
- TTI:  3500ms（可交互）
- LCP:  3200ms

SSR:
- TTFB: 400ms（服务端计算）
- FCP:  800ms（HTML 直达）
- TTI:  2500ms（等 hydrate）
- LCP:  900ms

SSG:
- TTFB: 50ms（CDN 返回）
- FCP:  600ms（HTML 从 CDN）
- TTI:  2200ms（等 hydrate）
- LCP:  700ms

ISR (revalidate: 60):
- TTFB: 50ms（CDN 返回缓存）
- FCP:  600ms
- TTI:  2200ms
- LCP:  700ms
- 后台每 60s 更新
```

## 七、常见 SSR 问题与解决方案

### 7.1 浏览器 API 不可用

```tsx
// ❌ 服务端没有 window/document
function MyComponent() {
  const width = window.innerWidth;  // Error: window is not defined
}

// ✅ 解决方案 1：条件判断
function MyComponent() {
  const [width, setWidth] = useState(
    typeof window !== 'undefined' ? window.innerWidth : 0
  );

  useEffect(() => {
    setWidth(window.innerWidth);
  }, []);
}

// ✅ 解决方案 2：动态导入
import dynamic from 'next/dynamic';

const Chart = dynamic(() => import('./Chart'), {
  ssr: false,  // 只在客户端加载
  loading: () => <Skeleton />,
});
```

### 7.2 数据获取时序

```tsx
// 问题：组件需要数据才能渲染，但服务端不知道需要什么数据

// 方案1：在页面级别获取数据（Next.js Server Components）
async function Page({ params }) {
  const data = await fetchData(params.id);  // 服务端执行
  return <Component data={data} />;
}

// 方案2：使用 Suspense 流式加载
function Page() {
  return (
    <Suspense fallback={<Loading />}>
      <DataComponent />  {/* 内部异步获取数据 */}
    </Suspense>
  );
}
```

### 7.3 CSS 处理

```tsx
// SSR 中的 CSS 方案：

// 1. CSS Modules - 构建时提取，无运行时问题
import styles from './Button.module.css';

// 2. Tailwind CSS - 构建时生成，无运行时问题
<button className="bg-blue-500 hover:bg-blue-700 text-white">

// 3. CSS-in-JS（styled-components）- 需要特殊处理
import { ServerStyleSheet } from 'styled-components';

function renderApp() {
  const sheet = new ServerStyleSheet();
  try {
    const html = renderToString(sheet.collectStyles(<App />));
    const styleTags = sheet.getStyleTags();
    return { html, styleTags };
  } finally {
    sheet.seal();
  }
}
```

## 总结

| 概念 | 要点 |
|------|------|
| **CSR vs SSR vs SSG vs ISR** | 各有适用场景，现代框架支持混合使用 |
| **Hydration** | 将静态 HTML 变为可交互应用的过程 |
| **流式 SSR** | 边生成边传输，用户更快看到内容 |
| **renderToPipeableStream** | React 18 推荐的服务端渲染 API |
| **Meta 管理** | SEO 必需，Next.js 有原生 Metadata API |
| **选择策略** | 根据数据新鲜度需求和 SEO 需求选择渲染方式 |
