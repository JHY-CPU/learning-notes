# 列表渲染深入

列表渲染是 React 中最频繁使用的功能之一。深入理解其底层机制对写出高性能、无 Bug 的应用至关重要。

---

## 一、Array.map() 基础

React 中渲染列表的标准方式是使用 `Array.map()`，将数据数组映射为 JSX 元素数组。

```jsx
function UserList({ users }) {
  return (
    <ul>
      {users.map(user => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  );
}
```

### map 与 filter 链式调用

```jsx
function ActiveUsers({ users }) {
  return (
    <ul>
      {users
        .filter(user => user.isActive)
        .map(user => (
          <li key={user.id}>{user.name}</li>
        ))
      }
    </ul>
  );
}
```

### 带索引的 map

```jsx
function NumberedList({ items }) {
  return (
    <ol>
      {items.map((item, index) => (
        <li key={item.id}>
          {index + 1}. {item.text}
        </li>
      ))}
    </ol>
  );
}
```

> **注意**：这里的 `index` 仅用于显示序号，**不应该用作 `key`**。

## 二、key 属性深入

### 2.1 key 的作用

`key` 是 React 用来识别列表中每个元素的唯一标识。React 使用 `key` 来判断哪些元素发生了变化、被添加或被移除，从而**最小化 DOM 操作**。

### 2.2 协调（Reconciliation）过程

当状态更新触发重新渲染时，React 会执行**协调算法**（也叫 Diff 算法）：

```
旧列表: [A, B, C, D]
新列表: [A, C, B, E]

React 通过 key 比对：
1. A(key=1) === A(key=1)  → 复用，无需操作
2. B(key=2) → 在新列表位置2找到，移动到位置2
3. C(key=3) → 在新列表位置1找到，移动到位置1
4. D(key=4) → 未找到，删除
5. E(key=5) → 新增
```

### 2.3 Fiber 树中的 key

React 16+ 使用 Fiber 架构。在 Fiber 树中，`key` 是识别 Fiber 节点的核心属性之一：

```
Fiber 节点结构（简化）:
{
  type: 'li',
  key: 'user-123',        // key 用于匹配
  props: { children: '...' },
  stateNode: <li>DOM节点,
  child: ...,
  sibling: ...,
  return: ...,
}
```

**协调过程详解**：

1. **单节点 vs 单节点**：如果 `key` 和 `type` 都相同，复用 Fiber 节点，只更新 props
2. **单节点 vs 多节点**：删除旧节点，创建新节点
3. **多节点 vs 多节点**（最常见）：
   - 第一轮：遍历新列表，通过 `key` 找到可复用的节点（遇到不可复用的就停止）
   - 第二轮：处理剩余节点（移动、新增、删除）

### 2.4 有 key vs 无 key 的区别

```jsx
// 无 key：React 默认按索引比对
<ul>
  {items.map(item => <li>{item.text}</li>)}
</ul>

// 有 key：React 按 key 比对
<ul>
  {items.map(item => <li key={item.id}>{item.text}</li>)}
</ul>
```

**无 key 时的行为**（等同于 `key={index}`）：

```
旧: [<li>A</li>, <li>B</li>, <li>C</li>]
新: [<li>B</li>, <li>A</li>, <li>C</li>]

React 按索引比对：
index 0: 期望A → 发现B → 替换内容为B（即使 DOM 节点复用）
index 1: 期望B → 发现A → 替换内容为A
index 2: 期望C → 发现C → 无变化
```

这会导致**不必要的 DOM 更新**和**组件状态错乱**。

## 三、为什么不能用 index 作为 key

### 3.1 列表重排 Bug

```jsx
// 用 index 作为 key 的问题演示
function TodoList() {
  const [todos, setTodos] = useState([
    { text: '学习 React', done: false },
    { text: '写项目',     done: false },
    { text: '复习笔记',   done: false },
  ]);

  // 反转列表
  const reverse = () => setTodos([...todos].reverse());

  return (
    <div>
      <button onClick={reverse}>反转</button>
      {todos.map((todo, index) => (
        <TodoItem key={index} todo={todo} />
        // index 作为 key: 反转后 key=0 仍对应第一个位置
      ))}
    </div>
  );
}

function TodoItem({ todo }) {
  const [isEditing, setIsEditing] = useState(false);

  return (
    <li>
      {isEditing ? (
        <input defaultValue={todo.text} />
      ) : (
        <span onClick={() => setIsEditing(true)}>{todo.text}</span>
      )}
    </li>
  );
}
```

**Bug 复现**：

1. 点击第一个 Todo "学习 React" 进入编辑模式
2. 点击"反转"按钮
3. **Bug**：编辑框出现在了 "复习笔记" 的位置，而不是 "学习 React"

**原因分析**：

```
反转前:
  index=0: "学习 React" (编辑状态)
  index=1: "写项目"
  index=2: "复习笔记"

反转后:
  index=0: "复习笔记"  → React 认为 key=0 的组件没变，复用旧状态（编辑状态）
  index=1: "写项目"    → 同上，复用
  index=2: "学习 React" → 同上，复用
```

React 按 index 比对时，认为每个位置的**组件类型没变**，因此**保留了组件的内部状态**。

### 3.2 列表插入 Bug

```jsx
// 在列表头部插入新元素
function InsertDemo() {
  const [items, setItems] = useState(['A', 'B', 'C']);
  const addFront = () => setItems(['X', ...items]);

  return (
    <div>
      <button onClick={addFront}>在头部添加</button>
      {items.map((item, index) => (
        <ListItem key={index} label={item} />
      ))}
    </div>
  );
}

function ListItem({ label }) {
  const [count, setCount] = useState(0);
  return (
    <div>
      {label}: {count}
      <button onClick={() => setCount(c => c + 1)}>+1</button>
    </div>
  );
}
```

**问题**：在头部插入 "X" 后，每个 index 对应的数据都变了，但组件状态没跟着变：

```
插入前: index=0:A(计数3), index=1:B(计数1), index=2:C(计数5)
插入后: index=0:X(计数3), index=1:A(计数1), index=2:B(计数5)
// X 获取了 A 的状态，A 获取了 B 的状态...
```

### 3.3 列表删除 Bug

```jsx
// 删除中间元素
function DeleteDemo() {
  const [items, setItems] = useState(['A', 'B', 'C', 'D']);

  return (
    <div>
      {items.map((item, index) => (
        <div key={index}>
          <input defaultValue={item} />
          <button onClick={() => {
            setItems(items.filter((_, i) => i !== index));
          }}>删除</button>
        </div>
      ))}
    </div>
  );
}
```

删除 B(index=1) 后：

```
旧: index=0:A, index=1:B, index=2:C, index=3:D
新: index=0:A, index=1:C, index=2:D

问题：在输入框中编辑了 C 的值后删除 B，C 的输入框内容丢失
因为 React 认为 index=1 还是同一个组件，只是 props 变了
```

### 3.4 什么时候可以用 index 作为 key

仅在**同时满足**以下条件时可以用 index：

1. 列表**静态不变**（不会增删、重排）
2. 列表项**没有内部状态**
3. 列表项的**顺序不会改变**

```jsx
// 可以接受用 index 的场景：纯展示的静态列表
function StaticList({ items }) {
  return (
    <ul>
      {items.map((item, index) => (
        <li key={index}>{item}</li>
        // 静态列表，没有交互状态，顺序不变
      ))}
    </ul>
  );
}
```

## 四、生成稳定的 key

### 4.1 使用数据中的唯一标识

```jsx
// 最佳：使用数据库 ID
{users.map(user => (
  <UserCard key={user.id} user={user} />
))}

// 次选：使用唯一业务标识
{products.map(product => (
  <ProductCard key={product.sku} product={product} />
))}
```

### 4.2 使用组合 key

当单一字段不够唯一时，组合多个字段：

```jsx
// 用多个字段拼接
{orders.map(order => (
  <OrderRow
    key={`${order.userId}-${order.productId}-${order.date}`}
    order={order}
  />
))}
```

### 4.3 生成唯一 ID

对于前端创建的数据（无数据库 ID），需要生成稳定 ID。

```jsx
import { useId } from 'react';

// 方案1：使用 useId（React 18+）
// useId 生成稳定的、SSR 安全的唯一 ID
function FormFields({ fields }) {
  const baseId = useId();
  return fields.map((field, index) => (
    <div key={`${baseId}-${index}`}>
      <label htmlFor={`${baseId}-${field.name}`}>{field.label}</label>
      <input id={`${baseId}-${field.name}`} name={field.name} />
    </div>
  ));
}

// 方案2：在创建数据时生成 ID
import { v4 as uuidv4 } from 'uuid';

function TodoApp() {
  const [todos, setTodos] = useState([]);

  const addTodo = (text) => {
    setTodos(prev => [...prev, {
      id: uuidv4(),  // 创建时就生成稳定 ID
      text,
      done: false,

  return (
    <ul>
      {todos.map(todo => (
        <TodoItem key={todo.id} todo={todo} />
      ))}
    </ul>
  );
}

// 方案3：简单自增 ID（仅客户端，不适合 SSR）
let nextId = 0;
const createItem = (text) => ({ id: nextId++, text });
```

## 五、过滤与映射组合

### 5.1 filter + map

```jsx
function ProductList({ products, category, inStock }) {
  return (
    <div>
      {products
        .filter(p => !category || p.category === category)
        .filter(p => !inStock || p.stock > 0)
        .map(p => (
          <ProductCard key={p.id} product={p} />
        ))
      }
    </div>
  );
}
```

### 5.2 提前计算过滤结果

```jsx
function ProductList({ products, category, inStock, sortBy }) {
  // 当数据量大时，使用 useMemo 缓存计算结果
  const filteredProducts = useMemo(() => {
    let result = products;

    if (category) {
      result = result.filter(p => p.category === category);
    }
    if (inStock) {
      result = result.filter(p => p.stock > 0);
    }
    if (sortBy === 'price') {
      result = [...result].sort((a, b) => a.price - b.price);
    } else if (sortBy === 'name') {
      result = [...result].sort((a, b) => a.name.localeCompare(b.name));
    }

    return result;
  }, [products, category, inStock, sortBy]);

  if (filteredProducts.length === 0) {
    return <EmptyState />;
  }

  return (
    <div>
      {filteredProducts.map(p => (
        <ProductCard key={p.id} product={p} />
      ))}
    </div>
  );
}
```

```jsx
function GroupedProducts({ products }) {
```

```jsx
// 支持泛型数据的通用树
function GenericTree({ data, renderNode, childrenKey = 'children', keyField = 'id' }) {
  const renderTree = (nodes, depth = 0) => {
    return nodes.map(node => (
      <React.Fragment key={node[keyField]}>
        {renderNode(node, depth)}
        {node[childrenKey]?.length > 0 && (
          renderTree(node[childrenKey], depth + 1)
        )}
      </React.Fragment>
    ));
  };

  return <div>{renderTree(data)}</div>;
}

/>
```

## 八、列表排序

### 8.1 基础排序

```jsx
function SortedList({ items, sortKey, sortDirection = 'asc' }) {
  const sorted = useMemo(() => {
    return [...items].sort((a, b) => {
      const aVal = a[sortKey];
      const bVal = b[sortKey];

      if (typeof aVal === 'string') {
        return sortDirection === 'asc'
          ? aVal.localeCompare(bVal)
          : bVal.localeCompare(aVal);
      }

      return sortDirection === 'asc' ? aVal - bVal : bVal - aVal;
    });
  }, [items, sortKey, sortDirection]);

  return (
    <ul>
      {sorted.map(item => (
        <li key={item.id}>{item.name}: {item[sortKey]}</li>
      ))}
    </ul>
  );
}
```

### 8.2 多列排序

```jsx
function MultiSort({ items, sortConfigs }) {
  // sortConfigs: [{ key: 'name', direction: 'asc' }, { key: 'age', direction: 'desc' }]
  const sorted = useMemo(() => {
    return [...items].sort((a, b) => {
      for (const { key, direction } of sortConfigs) {
        const aVal = a[key];
        const bVal = b[key];
        const modifier = direction === 'asc' ? 1 : -1;

        if (aVal < bVal) return -1 * modifier;
        if (aVal > bVal) return 1 * modifier;
      }
      return 0;
    });
  }, [items, sortConfigs]);

  return (
    <ul>
      {sorted.map(item => <li key={item.id}>{item.name}</li>)}
    </ul>
  );
}
```

```jsx
}
```

## 十、性能注意事项

### 10.1 避免在 map 中创建新函数

```jsx
// 不好：每次渲染都创建新函数
{items.map(item => (
  <Item
    key={item.id}
    onClick={() => handleClick(item.id)}  // 每次都是新函数
  />
/>
)}

// 更好：使用 useCallback 或事件委托
const handleItemClick = useCallback((id) => {
  handleClick(id);
}, [handleClick]);

{items.map(item => (
  <Item key={item.id} onClick={handleItemClick} id={item.id} />
))}
```

### 10.2 对列表项使用 React.memo

```jsx
const ListItem = React.memo(function ListItem({ item, onSelect }) {
```

### 10.3 大列表考虑虚拟化

当列表项超过 100+ 时，考虑使用虚拟列表（参见下一篇文章）。
