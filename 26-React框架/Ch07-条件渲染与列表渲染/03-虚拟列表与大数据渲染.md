# 虚拟列表与大数据渲染

当列表数据量达到成百上千甚至上万条时，将所有 DOM 节点一次性渲染会导致严重的性能问题。虚拟列表（Virtual Scrolling / Windowing）技术通过只渲染可视区域内的元素来解决这个问题。

---

## 一、为什么需要虚拟化

### 1.1 传统列表的性能问题

```jsx
// 渲染 10000 条数据 —— 灾难性性能
function HugeList({ items }) {
  return (
    <div>
      {items.map(item => (
        <div key={item.id} className="row">
          <img src={item.avatar} />
          <span>{item.name}</span>
          <span>{item.email}</span>
        </div>
      ))}
    </div>
  );
}
```

**性能瓶颈**：

| 数据量 | DOM 节点数 | 首次渲染时间 | 内存占用 |
|---|---|---|---|
| 100 | ~500 | ~15ms | ~2MB |
| 1,000 | ~5,000 | ~120ms | ~15MB |
| 10,000 | ~50,000 | ~1,500ms | ~120MB |
| 100,000 | ~500,000 | 浏览器卡死 | ~1GB+ |

### 1.2 虚拟化原理

虚拟列表的核心思想：**只渲染可视区域（viewport）内可见的元素**，用空白占位撑起滚动条。

```
┌─────────────────────────────────┐
│         滚动容器 (1000px)         │
│  ┌───────────────────────────┐  │
│  │  空白占位 (padding-top)    │  │  ← 上方不可见区域的占位
│  ├───────────────────────────┤  │
│  │  列表项 50 (可见)          │  │
│  │  列表项 51 (可见)          │  │  ← 只渲染可视区域内的元素
│  │  列表项 52 (可见)          │  │
│  │  列表项 53 (可见)          │  │
│  ├───────────────────────────┤  │
│  │  空白占位 (padding-bottom) │  │  ← 下方不可见区域的占位
│  └───────────────────────────┘  │
│                                 │
│  总数据: 10000 条               │
│  实际渲染: 4 条                  │
└─────────────────────────────────┘
```

### 2.2 固定高度列表 (FixedSizeList)

```jsx
import { FixedSizeList as List } from 'react-window';

function UserList({ users }) {
  const Row = ({ index, style }) => (
    <div style={style} className="row">
      <img src={users[index].avatar} alt="" />
      <span>{users[index].name}</span>
      <span>{users[index].email}</span>
    </div>
  );

  return (
    <List
      height={600}          // 容器可视高度
      itemCount={users.length} // 总数据条数
      itemSize={64}         // 每行高度（px）
      width="100%"          // 容器宽度
    >
      {Row}
    </List>
  );
}
```

### 2.3 可变高度列表 (VariableSizeList)

```jsx
import { VariableSizeList as List } from 'react-window';

function MessageList({ messages }) {
  // 估算每条消息的高度
  const getItemSize = (index) => {
    const msg = messages[index];
    const baseHeight = 48;
    const textLines = Math.ceil(msg.text.length / 50);
    return baseHeight + textLines * 20 + (msg.hasImage ? 200 : 0);
  };

  const Row = ({ index, style }) => (
    <div style={style} className="message">
      <strong>{messages[index].sender}</strong>
      <p>{messages[index].text}</p>
      {messages[index].hasImage && (
        <img src={messages[index].imageUrl} alt="" />
      )}
    </div>
  );

  return (
    <List
      height={600}
      itemCount={messages.length}
      itemSize={getItemSize}
      width="100%"
    >
      {Row}
    </List>
  );
}
```

### 2.4 固定大小网格 (FixedSizeGrid)

```jsx
import { FixedSizeGrid as Grid } from 'react-window';

function ProductGrid({ products, columnCount = 4 }) {
  const rowCount = Math.ceil(products.length / columnCount);

  const Cell = ({ columnIndex, rowIndex, style }) => {
    const index = rowIndex * columnCount + columnIndex;
    const product = products[index];

    if (!product) return null;

    return (
      <div style={style} className="grid-cell">
        <img src={product.image} alt={product.name} />
        <span>{product.name}</span>
        <span>¥{product.price}</span>
      </div>
    );
  };

  return (
    <Grid
      columnCount={columnCount}
      columnWidth={250}
      height={800}
      rowCount={rowCount}
      rowHeight={300}
      width={1000}
    >
      {Cell}
    </Grid>
  );
}
```

### 2.6 配合 react-window-infinite-loader 无限滚动

```bash
npm install react-window-infinite-loader
```

```jsx
import { FixedSizeList } from 'react-window';
import InfiniteLoader from 'react-window-infinite-loader';

function InfiniteUserList({ users, loadMore, hasNextPage, isLoading }) {
  const itemCount = hasNextPage ? users.length + 1 : users.length;

  const isItemLoaded = (index) => !hasNextPage || index < users.length;

  const Row = ({ index, style }) => {
    if (!isItemLoaded(index)) {
      return (
        <div style={style} className="loading-row">
          加载中...
        </div>
      );
    }

    return (
      <div style={style}>
        {users[index].name}
      </div>
    );
  };

  return (
    <InfiniteLoader
      isItemLoaded={isItemLoaded}
      itemCount={itemCount}
      loadMoreItems={isLoading ? () => {} : loadMore}
      threshold={5}  // 距离底部还有 5 项时触发加载
    >
      {({ onItemsRendered, ref }) => (
        <FixedSizeList
          height={600}
          itemCount={itemCount}
          itemSize={64}
          width="100%"
          onItemsRendered={onItemsRendered}
          ref={ref}
        >
          {Row}
        </FixedSizeList>
      )}
    </InfiniteLoader>
  );
}
```

### 3.4 无限滚动

```jsx
import { Virtuoso } from 'react-virtuoso';

function InfiniteList() {
  const [users, setUsers] = useState([]);
  const [loading, setLoading] = useState(false);
  const [hasMore, setHasMore] = useState(true);

  const loadMore = useCallback(() => {
    if (loading || !hasMore) return;
    setLoading(true);

    fetch(`/api/users?offset=${users.length}`)
      .then(res => res.json())
      .then(data => {
        setUsers(prev => [...prev, ...data.users]);

      style={{ height: 600 }}
      data={users}
      endReached={loadMore}
      overscan={200}  // 预渲染可视区域外 200px 的内容
      itemContent={(index, user) => (
        <div className="row">{user.name}</div>
      )}
      components={{
        Footer: () => loading ? <div className="spinner">加载中...</div> : null,
      }}
    />
  );
}
```

## 四、固定高度 vs 可变高度

### 对比

| 特性 | 固定高度 | 可变高度 |
|---|---|---|
| 性能 | 最优 | 略低（需要测量） |
| 实现复杂度 | 简单 | 中等 |
| 适用场景 | 表格行、统一卡片 | 聊天、动态内容 |
| 滚动精度 | 精确 | 近似（首次需要校正） |
| 库支持 | react-window FixedSizeList | react-virtuoso / VariableSizeList |

### 选择建议

```
数据量 < 100 ──────────────► 普通列表，无需虚拟化
统一高度 + 大数据 ──────────► react-window FixedSizeList
动态高度 + 大数据 ──────────► react-virtuoso
需要网格 ──────────────────► react-virtuosoGrid / FixedSizeGrid
需要无限加载 ──────────────► react-virtuoso (内置) / react-window + infinite-loader
```

## 六、自己实现简单虚拟列表

理解原理的最佳方式是自己实现一个。

```jsx
import { useState, useRef, useCallback, useMemo } from 'react';

function SimpleVirtualList({ items, itemHeight = 50, containerHeight = 600 }) {
  const containerRef = useRef(null);
  const [scrollTop, setScrollTop] = useState(0);

  // 计算可见范围
  const { startIndex, endIndex, visibleItems, totalHeight, offsetY } = useMemo(() => {
    const start = Math.floor(scrollTop / itemHeight);
    const visibleCount = Math.ceil(containerHeight / itemHeight);
    const buffer = 3; // 上下各多渲染 3 项，减少白屏
    const end = Math.min(start + visibleCount + buffer, items.length);
    const bufferedStart = Math.max(0, start - buffer);

    return {
      startIndex: bufferedStart,
      endIndex: end,
      visibleItems: items.slice(bufferedStart, end),
      totalHeight: items.length * itemHeight,
      offsetY: bufferedStart * itemHeight,
    };
  }, [items, itemHeight, containerHeight, scrollTop]);

  const handleScroll = useCallback((e) => {
    setScrollTop(e.currentTarget.scrollTop);
  }, []);

  return (
    <div
      ref={containerRef}
      onScroll={handleScroll}
      style={{
        height: containerHeight,
        overflow: 'auto',
        position: 'relative',
      }}
    >
      {/* 总高度占位，撑起滚动条 */}
      <div style={{ height: totalHeight, position: 'relative' }}>
        {/* 可视区域偏移 */}
        <div style={{ position: 'absolute', top: offsetY, left: 0, right: 0 }}>
          {visibleItems.map((item, i) => (
            <div
              key={item.id}
              style={{
                height: itemHeight,
                display: 'flex',
                alignItems: 'center',
                borderBottom: '1px solid #eee',
              }}
            >
              {startIndex + i}: {item.name}
            </div>
          ))}
        </div>
      </div>
    </div>
  );
}

// 使用
const items = Array.from({ length: 100000 }, (_, i) => ({
  id: i,
  name: `项目 ${i}`,
}));

<SimpleVirtualList items={items} itemHeight={50} containerHeight={600} />
```

## 七、与无限滚动结合

### 7.1 基础无限滚动（无虚拟化）

```jsx
function InfiniteScroll({ loadMore, hasMore, children }) {
  const observerRef = useRef(null);
  const sentinelRef = useRef(null);

  useEffect(() => {
    observerRef.current = new IntersectionObserver((entries) => {
      if (entries[0].isIntersecting && hasMore) {
        loadMore();
      }
    }, { rootMargin: '200px' });

    if (sentinelRef.current) {
      observerRef.current.observe(sentinelRef.current);
    }

    return () => observerRef.current?.disconnect();
  }, [loadMore, hasMore]);

  return (
    <div>
      {children}
      <div ref={sentinelRef} style={{ height: 1 }} />
      {hasMore && <div className="loading">加载中...</div>}
    </div>
  );
}
```

### 7.2 虚拟化 + 无限滚动完整示例

```jsx
import { useState, useCallback } from 'react';
import { Virtuoso } from 'react-virtuoso';

function PaginatedList() {
  const [data, setData] = useState([]);
  const [page, setPage] = useState(0);
  const [hasMore, setHasMore] = useState(true);
  const [loading, setLoading] = useState(false);

  const loadMore = useCallback(async () => {
    if (loading || !hasMore) return;
    setLoading(true);

    try {
      const res = await fetch(`/api/items?page=${page}&size=50`);
      const result = await res.json();

      setData(prev => [...prev, ...result.items]);
      setPage(prev => prev + 1);
      setHasMore(result.hasMore);
    } finally {
      setLoading(false);
    }
  }, [page, loading, hasMore]);

  // 初始加载
  useEffect(() => {
    loadMore();
  }, []);

  return (
    <Virtuoso
      style={{ height: '100vh' }}
      data={data}
      endReached={loadMore}
      overscan={300}
      itemContent={(index, item) => (
        <div style={{ padding: 16, borderBottom: '1px solid #eee' }}>
          <h3>{item.title}</h3>
          <p>{item.description}</p>
        </div>
      )}
      components={{
        Header: () => <h2 style={{ padding: 16 }}>数据列表</h2>,
        Footer: () =>
          loading ? (
            <div style={{ textAlign: 'center', padding: 16 }}>
              加载更多...
            </div>
          ) : !hasMore ? (
            <div style={{ textAlign: 'center', padding: 16, color: '#999' }}>
              已经到底了
            </div>
          ) : null,
      }}
    />
  );
}
```

## 九、何时不需要虚拟化

虚拟化不是万能的，在以下场景中**不需要**使用：

1. **数据量小**（< 100 条）：现代浏览器可以轻松处理
2. **列表项高度差异极大且不可预测**：虚拟化估算高度会导致跳动
3. **需要浏览器原生搜索（Ctrl+F）**：虚拟化会丢失不在视口中的文本
4. **需要打印整个列表**：虚拟化只渲染可见部分
5. **需要 `querySelector` 查找**：不可见元素不在 DOM 中
