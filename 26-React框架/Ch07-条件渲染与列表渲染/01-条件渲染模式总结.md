# 条件渲染模式总结

条件渲染是 React 中根据不同条件显示不同内容的核心技术。本文系统总结所有常见模式及其适用场景。

---

## 一、值的渲染行为（真值表）

React 在渲染时会自动处理以下值：

| 值 | 渲染结果 | 说明 |
|---|---|---|
| `true` | 不渲染 | 空白节点 |
| `false` | 不渲染 | 空白节点 |
| `null` | 不渲染 | 空白节点 |
| `undefined` | 不渲染 | 空白节点 |
| `0` | 渲染为 `"0"` | **注意陷阱！** |
| `""` (空字符串) | 不渲染 | 空白节点 |
| `"0"` | 渲染为 `"0"` | 字符串会渲染 |
| `NaN` | 渲染为 `"NaN"` | **注意陷阱！** |
| `[]` (空数组) | 不渲染 | React 忽略数组 |
| `[element]` | 渲染元素 | 数组中有效元素正常渲染 |

### 关键陷阱

```jsx
// 陷阱1: 0 会被渲染出来
const count = 0;
<div>{count && <p>有内容</p>}</div>  // 输出 "0" 而不是空白！

// 正确写法
<div>{count > 0 && <p>有内容</p>}</div>

// 陷阱2: NaN 被渲染
const value = parseInt("abc");
<div>{value}</div>  // 输出 "NaN"
```

## 二、if/else 方式

最基础的方式，在 `return` 之前使用条件判断。

```jsx
function Greeting({ isLoggedIn }) {
  if (isLoggedIn) {
    return <h1>欢迎回来！</h1>;
  }
  return <h1>请先登录</h1>;
}
```

### 多条件分支

```jsx
function StatusBadge({ status }) {
  if (status === 'loading') {
    return <span className="badge loading">加载中...</span>;
  } else if (status === 'success') {
    return <span className="badge success">成功</span>;
  } else if (status === 'error') {
    return <span className="badge error">失败</span>;
  }
  return <span className="badge">未知</span>;
}
```

**适用场景**：逻辑复杂、需要多个条件分支、返回的 JSX 结构差异较大时。

---

## 三、三元运算符

最常用的内联条件渲染方式。

```jsx
function Greeting({ isLoggedIn }) {
  return (
    <h1>
      {isLoggedIn ? '欢迎回来！' : '请先登录'}
    </h1>
  );
}
```

### 嵌套三元（不推荐，但有时不可避免）

```jsx
// 可读性差，但有时必要
function Price({ isVip, isMember, price }) {
  return (
    <span>
      {isVip
        ? price * 0.5
        : isMember
          ? price * 0.8
          : price
      }
    </span>
  );
}

// 更好的方式：提前计算
function Price({ isVip, isMember, price }) {
  const finalPrice = isVip
    ? price * 0.5
    : isMember
      ? price * 0.8
      : price;

  return <span>{finalPrice}</span>;
}
```

### 三元渲染不同组件

```jsx
function Page({ isAdmin }) {
  return (
    <div>
      {isAdmin ? <AdminPanel /> : <UserDashboard />}
    </div>
  );
}
```

**适用场景**：简单的二选一渲染，内联在 JSX 中。

---

## 四、逻辑与 (&&) 运算符

用于"条件成立时渲染，不成立时不渲染"。

```jsx
function Notifications({ count }) {
  return (
    <div>
      <span>通知</span>
      {count > 0 && <span className="badge">{count}</span>}
    </div>
  );
}
```

### 多条件组合

```jsx
function UserProfile({ user }) {
  return (
    <div>
      <h1>{user.name}</h1>
      {user.isAdmin && <AdminBadge />}
      {user.isVerified && <VerifiedBadge />}
      {user.bio && <p>{user.bio}</p>}
    </div>
  );
}
```

### 注意事项

```jsx
// 错误：左侧可能为 0
{items.length && <ItemList items={items} />}  // 当 length 为 0 时渲染 "0"！

// 正确
{items.length > 0 && <ItemList items={items} />}
{!!items.length && <ItemList items={items} />}
{items.length !== 0 && <ItemList items={items} />}
```

**适用场景**：只关心"有或无"的简单条件渲染。

---

## 五、switch/case 方式

适合多分支条件。

```jsx
function Alert({ type, message }) {
  const getAlertClass = () => {
    switch (type) {
      case 'success': return 'alert-success';
      case 'warning': return 'alert-warning';
      case 'error':   return 'alert-error';
      default:        return 'alert-info';
    }
  };

  // 可以同时用于样式和组件
  const getIcon = () => {
    switch (type) {
      case 'success': return <CheckIcon />;
      case 'warning': return <WarningIcon />;
      case 'error':   return <ErrorIcon />;
      default:        return <InfoIcon />;
    }
  };

  return (
    <div className={getAlertClass()}>
      {getIcon()}
      <span>{message}</span>
    </div>
  );
}
```

### 在 JSX 中直接使用 switch（用 IIFE 包装）

```jsx
function Dashboard({ role }) {
  return (
    <div>
      {(() => {
        switch (role) {
          case 'admin':  return <AdminDashboard />;
          case 'editor': return <EditorDashboard />;
          case 'viewer': return <ViewerDashboard />;
          default:       return <GuestDashboard />;
        }
      })()}
    </div>
  );
}
```

**适用场景**：三个以上的离散值分支。

---

## 六、IIFE（立即执行函数）

在 JSX 中执行复杂逻辑。

```jsx
function DataTable({ data, viewMode }) {
  return (
    <div>
      {(() => {
        if (!data || data.length === 0) {
          return <EmptyState />;
        }

        if (viewMode === 'grid') {
          return (
            <div className="grid">
              {data.map(item => <GridCard key={item.id} item={item} />)}
            </div>
          );
        }

        return (
          <table>
            <tbody>
              {data.map(item => <TableRow key={item.id} item={item} />)}
            </tbody>
          </table>
        );
      })()}
    </div>
  );
}
```

> **提示**：IIFE 可读性较差，现代 React 中更推荐将逻辑提取到单独的变量或组件中。箭头函数 IIFE `(() => ...)()` 可以简化为 `{(() => ...)()}` 或使用变量。

---

## 七、Early Return 模式

在函数顶部提前返回，减少嵌套。

```jsx
// 不好的写法：深层嵌套
function UserCard({ user }) {
  return (
    <div>
      {user ? (
        user.isActive ? (
          <ActiveUser user={user} />
        ) : (
          <InactiveUser user={user} />
        )
      ) : (
        <GuestView />
      )}
    </div>
  );
}

// 好的写法：early return
function UserCard({ user }) {
  if (!user) {
    return <GuestView />;
  }

  if (!user.isActive) {
    return <InactiveUser user={user} />;
  }

  return (
    <div>
      <ActiveUser user={user} />
    </div>
  );
}
```

### 组件内部的 early return

```jsx
function ProductPage({ product, loading, error }) {
  if (loading) return <Spinner />;
  if (error) return <ErrorMessage error={error} />;
  if (!product) return <NotFound />;

  return (
    <div>
      <h1>{product.name}</h1>
      <p>{product.description}</p>
      <Price amount={product.price} />
    </div>
  );
}
```

**适用场景**：处理加载、错误、空数据等前置条件。

## 八、Guard 模式

条件成立才渲染子内容，否则渲染 null 或替代内容。

```jsx
// 基础 Guard 组件
function Guard({ condition, fallback = null, children }) {
  return condition ? children : fallback;
}

// 使用
function Dashboard({ user }) {
  return (
    <div>
      <Guard condition={user.isAdmin} fallback={<p>无权限</p>}>
        <AdminPanel />
      </Guard>

      <Guard condition={user.isVerified}>
        <VerifiedFeatures />
      </Guard>
    </div>
  );
}
```

### 权限守卫

```jsx
function RequireAuth({ children, redirectTo = '/login' }) {
  const { user, loading } = useAuth();

  if (loading) return <Spinner />;
  if (!user) return <Navigate to={redirectTo} />;

  return children;
}

// 使用
<Route path="/dashboard" element={
  <RequireAuth>
    <Dashboard />
  </RequireAuth>
} />
```

## 九、枚举/映射模式

用对象映射代替 switch/case，更简洁可扩展。

```jsx
// 用对象映射替代 switch
const roleComponents = {
  admin:  <AdminDashboard />,
  editor: <EditorDashboard />,
  viewer: <ViewerDashboard />,
  guest:  <GuestDashboard />,
};

function Dashboard({ role }) {
  return roleComponents[role] || <GuestDashboard />;
}
```

### 映射函数（支持动态 props）

```jsx
const statusConfig = {
  active:   { color: 'green',  icon: CheckIcon,    label: '活跃' },
  inactive: { color: 'gray',   icon: MinusIcon,    label: '未激活' },
  banned:   { color: 'red',    icon: BanIcon,      label: '已封禁' },
};

function StatusBadge({ status }) {
  const config = statusConfig[status];
  if (!config) return null;

  const { color, icon: Icon, label } = config;
  return (
    <span className={`badge badge-${color}`}>
      <Icon />
      {label}
    </span>
  );
}
```

### 动态组件渲染

```jsx
// 注册表模式
const widgetMap = {
  chart:    ChartWidget,
  table:    TableWidget,
  calendar: CalendarWidget,
  map:      MapWidget,
};

function DynamicWidget({ type, props }) {
  const WidgetComponent = widgetMap[type];
  if (!WidgetComponent) return <div>未知组件: {type}</div>;
  return <WidgetComponent {...props} />;
}

// 使用
<DynamicWidget type="chart" props={{ data: chartData }} />
```

**适用场景**：组件类型由数据驱动、插件系统、CMS 内容块。

## 十、条件包装组件

只在满足条件时才渲染额外的包装元素。

```jsx
// 条件包装
function ConditionalWrapper({ condition, wrapper, children }) {
  return condition ? wrapper(children) : children;
}

// 使用：只在有链接时才包 <a> 标签
function Card({ title, link }) {
  return (
    <ConditionalWrapper
      condition={link}
      wrapper={(children) => <a href={link}>{children}</a>}
    >
      <div className="card">
        <h2>{title}</h2>
      </div>
    </ConditionalWrapper>
  );
}
```

### 条件添加布局

```jsx
function MaybeTooltip({ showTooltip, tooltipText, children }) {
  if (!showTooltip) return children;

  return (
    <Tooltip content={tooltipText}>
      {children}
    </Tooltip>
  );
}
```

---

## 十一、渲染 null 和 undefined

明确不渲染任何内容。

```jsx
function OptionalSection({ show, title, children }) {
  if (!show) return null;

  return (
    <section>
      <h2>{title}</h2>
      {children}
    </section>
  );
}
```

### null vs undefined vs false

```jsx
// 三者在 JSX 中行为一致：都不渲染
function Demo({ mode }) {
  if (mode === 'a') return null;
  if (mode === 'b') return undefined;
  if (mode === 'c') return false;
  return <div>内容</div>;
}
```

> **约定**：React 社区惯例是使用 `return null` 表示"不渲染"。

## 十二、自定义 Hook 封装条件逻辑

```jsx
function useConditionalRender({ data, loading, error }) {
  if (loading) return { status: 'loading', content: <Spinner /> };
  if (error) return { status: 'error', content: <ErrorMessage error={error} /> };
  if (!data) return { status: 'empty', content: <EmptyState /> };
  return { status: 'ready', content: null };
}

// 使用
function UserList() {
  const { data, loading, error } = useFetch('/api/users');
  const { status, content } = useConditionalRender({ data, loading, error });

  if (status !== 'ready') return content;

  return (
    <ul>
      {data.map(user => <li key={user.id}>{user.name}</li>)}
    </ul>
  );
}
```

## 十三、模式选择指南

```
简单二选一 ──────────► 三元运算符 ?:
是否显示某个元素 ────► 逻辑与 &&
多个离散分支 ────────► 枚举映射 / switch
复杂条件逻辑 ────────► 提前计算变量 / 提取子组件
加载/错误/空状态 ────► Early Return
权限/认证检查 ───────► Guard 组件
动态组件类型 ────────► 枚举映射（注册表模式）
条件包装 ───────────► ConditionalWrapper
```

### 综合示例

```jsx
function OrderPage({ orderId }) {
  const { data: order, loading, error } = useOrder(orderId);
  const { user } = useAuth();

  // Early return 模式
  if (loading) return <PageSpinner />;
  if (error) return <ErrorPage error={error} />;
  if (!order) return <NotFound />;

  // 提前计算条件
  const canEdit = user.isAdmin || order.userId === user.id;
  const isCompleted = order.status === 'completed';
  const showRefundButton = isCompleted && order.paidAt;

  return (
    <div>
      <h1>订单 #{order.id}</h1>

      {/* 逻辑与 */}
      {order.isUrgent && <UrgentBadge />}

      {/* 三元运算符 */}
      <OrderStatus status={order.status} />

      {/* 枚举映射 */}
      {orderStatusActions[order.status]}

      {/* 逻辑与 + 复杂条件 */}
      {canEdit && !isCompleted && (
        <EditOrderButton orderId={order.id} />
      )}

      {showRefundButton && (
        <RefundButton orderId={order.id} />
      )}

      {/* Guard 模式 */}
      <RequirePermission permission="view_financials">
        <FinancialDetails order={order} />
      </RequirePermission>
    </div>
  );
}
```

## 总结

| 模式 | 复杂度 | 可读性 | 适用场景 |
|---|---|---|---|
| if/else | 低 | 高 | 函数顶部、复杂分支 |
| 三元运算符 | 低 | 高 | 简单二选一 |
| 逻辑与 `&&` | 低 | 高 | 条件显示 |
| switch/case | 中 | 中 | 多分支离散值 |
| 枚举映射 | 中 | 高 | 多分支、可扩展 |
| IIFE | 高 | 低 | JSX 中复杂逻辑（尽量避免） |
| Early Return | 低 | 高 | 前置条件处理 |
| Guard 组件 | 中 | 高 | 权限、认证 |
