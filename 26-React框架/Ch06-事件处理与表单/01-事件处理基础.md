# React 事件处理基础

## 1. React 合成事件 (SyntheticEvent)

React 实现了自己的事件系统——**合成事件 (SyntheticEvent)**，它是浏览器原生事件的跨浏览器包装层。

### 1.1 为什么需要合成事件

- **跨浏览器兼容**：抹平 IE、Chrome、Firefox 等浏览器的事件差异
- **性能优化**：利用事件委托，所有事件绑定在根节点（React 17+ 绑定在 root 容器）
- **与 React 渲染机制集成**：可以与批量更新、优先级调度等机制配合

### 1.2 事件委托机制

React 不会把事件直接绑定到每个 DOM 元素上，而是采用**事件委托**：

```jsx
// React 内部做的事情（简化示意）
// 所有点击事件都委托到 root 容器
rootContainer.addEventListener('click', dispatchEvent);

// 你写的代码
<button onClick={handleClick}>点击</button>

// React 只在 root 上监听 click，事件触发时通过 fiber 树找到对应组件
```

React 17 之前：事件委托到 `document`
React 17+：事件委托到 React root 容器（支持多个 React 版本共存）

### 1.3 SyntheticEvent 对象

事件处理函数接收的参数是 `SyntheticEvent`，它拥有与原生事件相同的接口：

```jsx
function handleClick(event) {
  // SyntheticEvent 的常用属性和方法
  event.preventDefault();    // 阻止默认行为
  event.stopPropagation();  // 阻止冒泡
  event.target;             // 触发事件的元素
  event.currentTarget;      // 绑定事件的元素
  event.type;               // 事件类型，如 'click'
  event.timeStamp;          // 事件触发时间戳
  event.nativeEvent;        // 原生事件对象

  // 坐标相关
  event.clientX;            // 鼠标相对于视口的 X 坐标
  event.clientY;            // 鼠标相对于视口的 Y 坐标
  event.pageX;              // 鼠标相对于页面的 X 坐标
  event.pageY;              // 鼠标相对于页面的 Y 坐标
}
```

### 1.4 事件池化（React 17 已移除）

React 16 及更早版本中，SyntheticEvent 对象会被**池化**（复用），事件回调执行后属性会被清空：

```jsx
// React 16 中需要这样处理
function handleClick(event) {
  event.preventDefault();
  // 如果异步访问 event 属性会出错
  const capturedEvent = { ...event }; // 错误！展开运算符无法复制事件对象
  const capturedTarget = event.target; // 正确：立即取出需要的值
  setTimeout(() => {
    console.log(capturedTarget); // ✅
    console.log(event.target);   // ❌ React 16 中已被清空
  }, 100);
}

// React 17+：不再池化，可以异步访问
function handleClick(event) {
  setTimeout(() => {
    console.log(event.target); // ✅ 正常工作
  }, 100);
}
```

## 2. 常用事件类型

### 2.1 鼠标事件

```jsx
function MouseEvents() {
  const handleClick = (e) => {
    console.log('单击', e.clientX, e.clientY);
  };

  const handleDoubleClick = (e) => {
    console.log('双击');
  };

  const handleContextMenu = (e) => {
    e.preventDefault(); // 阻止默认右键菜单
    console.log('右键点击');
  };

  const handleMouseEnter = (e) => {
    // 不冒泡，鼠标进入元素时触发
    e.currentTarget.style.backgroundColor = '#f0f0f0';
  };

  const handleMouseLeave = (e) => {
    // 不冒泡，鼠标离开元素时触发
    e.currentTarget.style.backgroundColor = '';
  };

  const handleMouseMove = (e) => {
    // 鼠标在元素内移动时持续触发
    console.log(`位置: ${e.clientX}, ${e.clientY}`);
  };

  return (
    <div
      onClick={handleClick}
      onDoubleClick={handleDoubleClick}
      onContextMenu={handleContextMenu}
      onMouseEnter={handleMouseEnter}
      onMouseLeave={handleMouseLeave}
      onMouseMove={handleMouseMove}
      style={{ width: 200, height: 200, border: '1px solid #ccc' }}
    >
      鼠标交互区域
    </div>
  );
}
```

**mouseenter/mouseover 区别：**
- `mouseenter/mouseleave`：不冒泡，进入子元素不会重复触发
- `mouseover/mouseout`：冒泡，进入子元素会触发（先触发 leave 再触发 enter）

### 2.2 键盘事件

```jsx
function KeyboardEvents() {
  const handleKeyDown = (e) => {
    // 按键按下时触发（持续按住会重复触发）
    console.log(`键码: ${e.code}, 字符: ${e.key}`);

    // 组合键检测
    if (e.ctrlKey && e.key === 's') {
      e.preventDefault(); // 阻止浏览器默认保存
      console.log('Ctrl+S 被按下');
    }

    if (e.shiftKey && e.key === 'Enter') {
      console.log('Shift+Enter 换行');
    }

    // 常用快捷键
    switch (e.key) {
      case 'Enter':
        console.log('回车');
        break;
      case 'Escape':
        console.log('ESC');
        break;
      case 'Tab':
        e.preventDefault(); // 阻止 Tab 切换
        break;
      case 'ArrowUp':
      case 'ArrowDown':
      case 'ArrowLeft':
      case 'ArrowRight':
        console.log(`方向键: ${e.key}`);
        break;
    }
  };

  const handleKeyUp = (e) => {
    // 按键松开时触发
    console.log(`松开: ${e.key}`);
  };

  return <input onKeyDown={handleKeyDown} onKeyUp={handleKeyUp} placeholder="按键盘试试" />;
}
```

**e.key vs e.code：**
- `e.key`：按键的字符含义（会随 Shift 改变，如 'a' vs 'A'）
- `e.code`：物理按键位置（不变，如 'KeyA'），适合游戏/快捷键

### 2.3 焦点事件

```jsx
function FocusEvents() {
  const handleFocus = (e) => {
    // 获得焦点时触发
    e.currentTarget.parentElement.classList.add('focused');
  };

  const handleBlur = (e) => {
    // 失去焦点时触发
    e.currentTarget.parentElement.classList.remove('focused');
  };

  // focusin/focusout 会冒泡，focus/blur 不会
  const handleFocusIn = (e) => {
    console.log('focusin 冒泡', e.target);
  };

  return (
    <div onFocusIn={handleFocusIn}>
      <input onFocus={handleFocus} onBlur={handleBlur} placeholder="点击聚焦" />
    </div>
  );
}
```

### 2.4 表单事件

```jsx
function FormEvents() {
  const handleChange = (e) => {
    // input/textarea/select 值变化时触发
    const { name, value, type, checked } = e.target;
    console.log(`${name}: ${type === 'checkbox' ? checked : value}`);
  };

  const handleSubmit = (e) => {
    e.preventDefault(); // 阻止表单默认提交行为
    // 处理表单数据
    const formData = new FormData(e.target);
    console.log(Object.fromEntries(formData));
  };

  const handleReset = (e) => {
    console.log('表单重置');
  };

  return (
    <form onSubmit={handleSubmit} onReset={handleReset}>
      <input name="username" onChange={handleChange} />
      <input name="email" type="email" onChange={handleChange} />
      <button type="submit">提交</button>
      <button type="reset">重置</button>
    </form>
  );
}
```

### 2.5 触摸事件（移动端）

```jsx
function TouchEvents() {
  const handleTouchStart = (e) => {
    const touch = e.touches[0];
    console.log(`触摸开始: ${touch.clientX}, ${touch.clientY}`);
  };

  const handleTouchMove = (e) => {
    const touch = e.touches[0];
    console.log(`触摸移动: ${touch.clientX}, ${touch.clientY}`);
  };

  const handleTouchEnd = (e) => {
    const touch = e.changedTouches[0];
    console.log(`触摸结束: ${touch.clientX}, ${touch.clientY}`);
  };

  return (
    <div
      onTouchStart={handleTouchStart}
      onTouchMove={handleTouchMove}
      onTouchEnd={handleTouchEnd}
      style={{ width: 300, height: 300, background: '#eee' }}
    >
      触摸区域
    </div>
  );
}
```

### 2.6 滚动事件

```jsx
function ScrollEvents() {
  const handleScroll = (e) => {
    const { scrollTop, scrollLeft, scrollHeight, clientHeight } = e.currentTarget;
    const scrollPercent = (scrollTop / (scrollHeight - clientHeight)) * 100;
    console.log(`滚动进度: ${scrollPercent.toFixed(1)}%`);
  };

  return (
    <div
      onScroll={handleScroll}
      style={{ height: 200, overflow: 'auto', border: '1px solid #ccc' }}
    >
      <div style={{ height: 1000 }}>长内容...</div>
    </div>
  );
}
```

## 3. 事件绑定语法

### 3.1 直接绑定处理函数

```jsx
function App() {
  function handleClick() {
    console.log('clicked');
  }

  return <button onClick={handleClick}>点击</button>;
}
```

### 3.2 内联函数（注意性能）

```jsx
function App() {
  return (
    <button onClick={() => console.log('clicked')}>
      点击
    </button>
  );
  // ⚠️ 每次渲染都创建新函数，可能导致子组件不必要的重渲染
}
```

### 3.3 传递参数

```jsx
function App() {
  const handleDelete = (id, e) => {
    // 注意：事件对象作为最后一个参数（使用内联函数时）
    console.log('删除', id);
  };

  const items = [
    { id: 1, name: '项目A' },
    { id: 2, name: '项目B' },
  ];

  return (
    <ul>
      {items.map((item) => (
        <li key={item.id}>
          {item.name}
          {/* 方式一：内联箭头函数 */}
          <button onClick={(e) => handleDelete(item.id, e)}>删除</button>

          {/* 方式二：bind（不推荐，可读性差） */}
          {/* <button onClick={handleDelete.bind(null, item.id)}>删除</button> */}
        </li>
      ))}
    </ul>
  );
}
```

## 4. 事件冒泡与阻止

### 4.1 事件传播

React 事件传播遵循 DOM 标准：捕获 → 目标 → 冒泡

```jsx
function EventPropagation() {
  const handleOuterClick = () => {
    console.log('外层（冒泡阶段）');
  };

  const handleOuterClickCapture = () => {
    console.log('外层（捕获阶段）');
  };

  const handleInnerClick = (e) => {
    console.log('内层（目标阶段）');
    e.stopPropagation(); // 阻止继续传播
  };

  return (
    <div onClick={handleOuterClick} onClickCapture={handleOuterClickCapture}>
      <button onClick={handleInnerClick}>点击</button>
    </div>
  );
  // 输出顺序：外层（捕获阶段）→ 内层（目标阶段）
  // 因为 stopPropagation 阻止了冒泡
}
```

### 4.2 React 中的捕获事件

```jsx
// React 为部分事件提供了捕获阶段版本，命名加 Capture 后缀
<div
  onClick={handleBubble}          // 冒泡阶段
  onClickCapture={handleCapture}  // 捕获阶段
  onFocus={handleFocus}           // 冒泡阶段
  onFocusCapture={handleFocusCap} // 捕获阶段
>
  <button>按钮</button>
</div>
```

### 4.3 阻止默认行为

```jsx
function PreventDefault() {
  const handleSubmit = (e) => {
    e.preventDefault(); // 阻止表单提交跳转
    // 自定义提交逻辑
  };

  const handleLinkClick = (e) => {
    e.preventDefault(); // 阻止链接跳转
    // SPA 路由跳转
  };

  const handleDragOver = (e) => {
    e.preventDefault(); // 允许 drop
  };

  return (
    <>
      <form onSubmit={handleSubmit}>...</form>
      <a href="/about" onClick={handleLinkClick}>关于</a>
      <div onDragOver={handleDragOver} onDrop={handleDrop}>拖放区域</div>
    </>
  );
}
```

## 5. 事件处理中的 this 绑定

### 5.1 函数组件（不存在 this 问题）

函数组件中不存在 `this` 绑定问题，这是推荐使用函数组件的原因之一：

```jsx
function App() {
  const count = useRef(0);

  const handleClick = () => {
    // 箭头函数自动绑定外层 this（函数组件中没有 this，但可以正确捕获闭包）
    count.current++;
    console.log(count.current);
  };

  return <button onClick={handleClick}>计数</button>;
}
```

### 5.2 类组件中的 this（历史知识）

```jsx
// 类组件中 this 可能丢失，需要手动绑定
class App extends React.Component {
  constructor(props) {
    super(props);
    this.state = { count: 0 };
    // 方式一：构造函数中绑定
    this.handleClick = this.handleClick.bind(this);
  }

  // 方式二：使用类字段 + 箭头函数（推荐）
  handleClick = () => {
    this.setState({ count: this.state.count + 1 });
  };

  render() {
    return (
      // 方式三：render 中内联（每次渲染创建新函数，不推荐）
      <button onClick={this.handleClick}>
        计数: {this.state.count}
      </button>
    );
  }
}
```

> **提示**：现代 React 开发中应使用函数组件 + Hooks，类组件的 this 绑定只是历史遗留知识。

## 6. 性能注意事项

### 6.1 避免内联函数导致的重渲染

```jsx
// ❌ 每次渲染都创建新函数
function Parent() {
  const [count, setCount] = useState(0);
  return (
    <div>
      <span>{count}</span>
      <button onClick={() => setCount(count + 1)}>+1</button>
      {/* 每次都创建新的 onClick，导致 Child 重渲染 */}
      <Child onClick={() => console.log('hello')} />
    </div>
  );
}

// ✅ 使用 useCallback 稳定引用
function Parent() {
  const [count, setCount] = useState(0);
  const handleChildClick = useCallback(() => {
    console.log('hello');
  }, []);

  return (
    <div>
      <span>{count}</span>
      <button onClick={() => setCount(count + 1)}>+1</button>
      <Child onClick={handleChildClick} />
    </div>
  );
}

const Child = React.memo(({ onClick }) => {
  console.log('Child render');
  return <button onClick={onClick}>子组件按钮</button>;
});
```

### 6.2 事件处理函数的常见模式

```jsx
function TodoApp() {
  const [todos, setTodos] = useState([]);

  // 模式一：高阶函数返回事件处理函数
  const createDeleteHandler = (id) => (e) => {
    e.preventDefault();
    setTodos((prev) => prev.filter((t) => t.id !== id));
  };

  // 模式二：使用 data 属性
  const handleClick = (e) => {
    const { action, id } = e.currentTarget.dataset;
    if (action === 'delete') {
      setTodos((prev) => prev.filter((t) => t.id !== Number(id)));
    }
  };

  return (
    <ul>
      {todos.map((todo) => (
        <li key={todo.id}>
          {todo.text}
          {/* 模式一 */}
          <button onClick={createDeleteHandler(todo.id)}>删除</button>
          {/* 模式二 */}
          <button data-action="delete" data-id={todo.id} onClick={handleClick}>
            删除
          </button>
        </li>
      ))}
    </ul>
  );
}
```

## 总结

| 事件分类 | 常用事件 | 注意事项 |
|---------|---------|---------|
| 鼠标事件 | click, dblclick, mouseenter/leave | mouseenter 不冒泡，mouseover 冒泡 |
| 键盘事件 | keydown, keyup | e.code 适合快捷键，e.key 适合输入 |
| 焦点事件 | focus, blur, focusin, focusout | focus/blur 不冒泡，focusin/focusout 冒泡 |
| 表单事件 | change, submit, reset | 始终 preventDefault 阻止默认提交 |
| 触摸事件 | touchstart/move/end | 移动端注意 passive 选项 |
| 滚动事件 | scroll, wheel | 注意节流处理 |
