## 1. 纯函数与非纯函数

### 1.1 纯函数 (Pure Function)

纯函数是函数式编程的核心概念，它满足两个条件：

1. **确定性**：相同的输入始终产生相同的输出
2. **无副作用**：函数执行过程中不修改外部状态

```js
// ✅ 纯函数
function add(a, b) {
  return a + b;
}

// ✅ 纯函数
function double(arr) {
  return arr.map((n) => n * 2);
}

// ✅ 纯函数 - 不修改原数组
function addItem(arr, item) {
  return [...arr, item];
}
```

### 1.2 非纯函数 (Impure Function)

非纯函数存在副作用，可能修改外部状态或依赖外部变量：

```js
// ❌ 非纯函数 - 依赖外部变量
let total = 0;
function addToTotal(value) {
  total += value; // 修改了外部状态
  return total;
}

// ❌ 非纯函数 - 产生 I/O 副作用
function logMessage(msg) {
  console.log(msg); // 副作用：控制台输出
}

// ❌ 非纯函数 - 产生网络请求副作用
async function fetchUser(id) {
  const res = await fetch(`/api/users/${id}`); // 副作用：网络请求
  return res.json();
}

// ❌ 非纯函数 - 修改传入的参数
function addItemMutate(arr, item) {
  arr.push(item); // 修改了原数组
  return arr;
}
```

### 1.3 React 与纯函数

React 组件在理想情况下应该是纯函数——给定相同的 props，始终返回相同的 JSX：

```jsx
// ✅ 纯组件
function Greeting({ name }) {
  return <h1>Hello, {name}!</h1>;
}

// ❌ 非纯组件 - 包含副作用
function Timer() {
  // setInterval 是副作用
  useEffect(() => {
    const id = setInterval(() => console.log("tick"), 1000);
    return () => clearInterval(id);
  }, []);

  return <div>Timer running</div>;
}
```

> **核心思想**：React 将副作用从渲染逻辑中分离出来，让组件函数本身保持纯粹，副作用统一在 `useEffect` 中处理。

## 2. 副作用的分类

### 2.1 数据获取 (Data Fetching)

```jsx
function UserProfile({ userId }) {
  const [user, setUser] = useState(null);

  useEffect(() => {
    fetch(`/api/users/${userId}`)
      .then((res) => res.json())
      .then(setUser);
  }, [userId]);

  return user ? <div>{user.name}</div> : <div>Loading...</div>;
}
```

### 2.2 订阅 (Subscriptions)

```jsx
function ChatRoom({ roomId }) {
  const [messages, setMessages] = useState([]);

  useEffect(() => {
    const subscription = chatService.subscribe(roomId, (msg) => {
      setMessages((prev) => [...prev, msg]);
    });

    return () => subscription.unsubscribe();
  }, [roomId]);

  return <MessageList messages={messages} />;
}
```

### 2.3 DOM 操作

```jsx
function AutoFocusInput() {
  const inputRef = useRef(null);

  useEffect(() => {
    inputRef.current?.focus();
  }, []);

  return <input ref={inputRef} />;
}
```

### 2.4 定时器 (Timers)

```jsx
function Countdown({ seconds }) {
  const [remaining, setRemaining] = useState(seconds);

  useEffect(() => {
    const timer = setInterval(() => {
      setRemaining((prev) => {
        if (prev <= 1) {
          clearInterval(timer);
          return 0;
        }
        return prev - 1;
      });
    }, 1000);

    return () => clearInterval(timer);
  }, [seconds]);

  return <div>剩余 {remaining} 秒</div>;
}
```

### 2.5 日志记录与分析

```jsx
function PageView({ pageName }) {
  useEffect(() => {
    analytics.track("page_view", { page: pageName });
  }, [pageName]);

  return <div>当前页面：{pageName}</div>;
}
```

### 2.6 副作用分类总结

| 类型 | 示例 | 是否需要清理 |
|------|------|-------------|
| 数据获取 | fetch, axios | 是 (AbortController) |
| 订阅 | WebSocket, EventSource | 是 (unsubscribe) |
| DOM 操作 | focus, scroll, resize | 视情况而定 |
| 定时器 | setTimeout, setInterval | 是 (clearTimeout/clearInterval) |
| 事件监听 | window.addEventListener | 是 (removeEventListener) |
| 日志/分析 | console.log, analytics | 否 |
| 修改全局状态 | document.title | 通常不需要 |

## 3. useEffect 基础

### 3.1 基本语法

```jsx
useEffect(() => {
  // 副作用逻辑
  // 在组件渲染后执行

  return () => {
    // 清理函数（可选）
    // 在组件卸载或下次 effect 执行前调用
  };
}, [dependencies]); // 依赖数组
```

### 3.2 依赖数组的三种形态

```jsx
// 形态一：无依赖数组 - 每次渲染后都执行
useEffect(() => {
  console.log("每次渲染后都执行");
});

// 形态二：空依赖数组 - 仅在挂载时执行一次
useEffect(() => {
  console.log("仅在挂载时执行");

  return () => console.log("仅在卸载时执行");
}, []);

// 形态三：有依赖项 - 依赖变化时执行
useEffect(() => {
  console.log(`userId 变化了: ${userId}`);
}, [userId]);
```

### 3.3 Effect 执行时机

```
组件渲染流程：
  1. React 调用组件函数 → 生成新的 JSX
  2. React 对比 Virtual DOM (Reconciliation)
  3. React 提交 DOM 更新 (Commit)
  4. 浏览器绘制 (Paint)
  5. React 执行 useEffect ← 在浏览器绘制之后！
```

> 这意味着 `useEffect` 中的代码不会阻塞浏览器绘制，适合执行不需要同步反映在 UI 上的副作用。

## 4. Effect 清理机制

### 4.1 为什么需要清理

不清理副作用会导致：
- **内存泄漏**：未清除的订阅、定时器持续占用内存
- **状态更新错误**：在已卸载的组件上调用 setState
- **重复执行**：事件监听器被多次注册

### 4.2 清理函数的执行时机

```
清理函数执行时机：
  1. 组件卸载时 → 执行清理
  2. 依赖项变化时 → 先执行上一次的清理，再执行新的 effect
```

### 4.3 常见清理模式

```jsx
// 模式一：清除定时器
useEffect(() => {
  const timer = setInterval(() => {
    setCount((c) => c + 1);
  }, 1000);

  return () => clearInterval(timer);
}, []);

// 模式二：取消网络请求
useEffect(() => {
  const controller = new AbortController();

  fetch(`/api/users/${userId}`, { signal: controller.signal })
    .then((res) => res.json())
    .then(setUser)
    .catch((err) => {
      if (err.name !== "AbortError") {
        setError(err.message);
      }
    });

  return () => controller.abort();
}, [userId]);

// 模式三：移除事件监听
useEffect(() => {
  const handleResize = () => setWidth(window.innerWidth);
  window.addEventListener("resize", handleResize);

  return () => window.removeEventListener("resize", handleResize);
}, []);

// 模式四：取消订阅
useEffect(() => {
  const subscription = observable.subscribe((data) => {
    setData(data);
  });

  return () => subscription.unsubscribe();
}, []);

// 模式五：WebSocket 连接清理
useEffect(() => {
  const ws = new WebSocket("wss://example.com/chat");

  ws.onmessage = (event) => {
    setMessages((prev) => [...prev, JSON.parse(event.data)]);
  };

  return () => ws.close();
}, []);
```

### 4.4 自定义 Hook 封装清理逻辑

```jsx
// 封装：可复用的定时器 Hook
function useInterval(callback, delay) {
  const savedCallback = useRef(callback);

  useEffect(() => {
    savedCallback.current = callback;
  }, [callback]);

  useEffect(() => {
    if (delay === null) return;

    const id = setInterval(() => savedCallback.current(), delay);
    return () => clearInterval(id);
  }, [delay]);
}

// 使用
function Counter() {
  const [count, setCount] = useState(0);

  useInterval(() => {
    setCount((c) => c + 1);
  }, 1000);

  return <div>{count}</div>;
}
```

## 5. Effect 执行顺序

### 5.1 单个组件中的多个 Effect

多个 `useEffect` 按照代码中的声明顺序依次执行：

```jsx
function App() {
  useEffect(() => {
    console.log("Effect 1");
    return () => console.log("Cleanup 1");
  }, []);

  useEffect(() => {
    console.log("Effect 2");
    return () => console.log("Cleanup 2");
  }, []);

  useEffect(() => {
    console.log("Effect 3");
    return () => console.log("Cleanup 3");
  }, []);

  return <div>App</div>;
}

// 输出顺序（挂载时）：
// Effect 1
// Effect 2
// Effect 3

// 输出顺序（卸载时）：
// Cleanup 1
// Cleanup 2
// Cleanup 3
```

### 5.2 父子组件中的 Effect 顺序

```jsx
function Parent() {
  useEffect(() => {
    console.log("Parent Effect");
    return () => console.log("Parent Cleanup");
  }, []);

  return (
    <div>
      <Child />
    </div>
  );
}

function Child() {
  useEffect(() => {
    console.log("Child Effect");
    return () => console.log("Child Cleanup");
  }, []);

  return <div>Child</div>;
}

// 挂载时的输出：
// Child Effect    ← 子组件 effect 先执行
// Parent Effect   ← 父组件 effect 后执行

// 卸载时的输出：
// Parent Cleanup  ← 父组件 cleanup 先执行
// Child Cleanup   ← 子组件 cleanup 后执行
```

### 5.3 Strict Mode 下的双重执行

React 18 的 Strict Mode 在开发环境下会故意双重执行 effect 来帮助发现缺少清理的 bug：

```jsx
// StrictMode 下的执行顺序（仅开发环境）：
// 1. 组件渲染
// 2. 第一次 effect 执行
// 3. 第一次 cleanup 执行（模拟卸载）
// 4. 第二次 effect 执行
// 5. 组件挂载完成

// 这就是为什么 cleanup 函数非常重要：
// 如果不写清理函数，双重执行可能导致重复订阅、重复请求等问题
```

## 6. useLayoutEffect

### 6.1 执行时机对比

```
useEffect:
  DOM 更新 → 浏览器绘制 → useEffect 执行
  （异步，不阻塞绘制）

useLayoutEffect:
  DOM 更新 → useLayoutEffect 执行 → 浏览器绘制
  （同步，阻塞绘制）
```

### 6.2 适用场景

```jsx
// 场景一：需要在绘制前测量 DOM 元素
function Tooltip({ targetRef, text }) {
  const [position, setPosition] = useState({ top: 0, left: 0 });
  const tooltipRef = useRef(null);

  // 必须用 useLayoutEffect，否则会出现闪烁
  useLayoutEffect(() => {
    if (!targetRef.current || !tooltipRef.current) return;

    const targetRect = targetRef.current.getBoundingClientRect();
    const tooltipRect = tooltipRef.current.getBoundingClientRect();

    setPosition({
      top: targetRect.top - tooltipRect.height - 8,
      left: targetRect.left + (targetRect.width - tooltipRect.width) / 2,
    });
  }, [targetRef, text]);

  return (
    <div
      ref={tooltipRef}
      style={{
        position: "fixed",
        top: position.top,
        left: position.left,
      }}
    >
      {text}
    </div>
  );
}

// 场景二：同步读取 DOM 属性并更新
function SyncScroll({ containerRef }) {
  const [scrollTop, setScrollTop] = useState(0);

  useLayoutEffect(() => {
    const container = containerRef.current;
    if (!container) return;

    const handleScroll = () => {
      // 同步更新，避免视觉闪烁
      setScrollTop(container.scrollTop);
    };

    container.addEventListener("scroll", handleScroll);
    return () => container.removeEventListener("scroll", handleScroll);
  }, [containerRef]);

  return <div>滚动位置: {scrollTop}px</div>;
}
```

### 6.3 选择指南

| 场景 | 使用 |
|------|------|
| 数据获取、订阅、日志 | `useEffect` |
| 需要在浏览器绘制前读取/修改 DOM | `useLayoutEffect` |
| 需要同步触发重渲染（避免闪烁） | `useLayoutEffect` |
| 绝大多数副作用 | `useEffect` |

> **注意**：`useLayoutEffect` 在服务端渲染 (SSR) 中会触发警告，因为服务端没有 DOM。如果项目使用 SSR，可以使用条件导入：
>
> ```js
> const useLayoutEffect =
>   typeof window !== "undefined" ? React.useLayoutEffect : React.useEffect;
> ```

## 7. useInsertionEffect

### 7.1 用途

`useInsertionEffect` 是 React 18 引入的 Hook，**专门供 CSS-in-JS 库使用**，在 DOM 变更之后、`useLayoutEffect` 之前执行。

```jsx
// 仅供 CSS-in-JS 库开发者使用
useInsertionEffect(() => {
  // 在这里插入 <style> 标签
  const style = document.createElement("style");
  style.textContent = `.dynamic-class { color: red; }`;
  document.head.appendChild(style);

  return () => {
    document.head.removeChild(style);
  };
}, [dependencies]);
```

### 7.2 执行时机

```
完整执行顺序：
  1. React 调用组件函数（渲染阶段）
  2. React 提交 DOM 更新
  3. useInsertionEffect 执行   ← CSS-in-JS 插入样式
  4. useLayoutEffect 执行      ← 读取/修改 DOM 布局
  5. 浏览器绘制
  6. useEffect 执行            ← 常规副作用
```

### 7.3 使用限制

```jsx
useInsertionEffect(() => {
  // ❌ 不能在 useInsertionEffect 中读取 refs
  // ❌ 不能在 useInsertionEffect 中触发重新渲染
  // ✅ 只能用于插入和清理动态样式
  return () => { /* 清理样式 */ };
}, [deps]);
```

> **警告**：除非你在开发 CSS-in-JS 库，否则不应该使用 `useInsertionEffect`。

## 8. 最佳实践与常见陷阱

### 8.1 副作用应该放在 useEffect 中

```jsx
// ❌ 错误：副作用在渲染期间执行
function BadExample({ userId }) {
  const data = fetch(`/api/users/${userId}`); // 每次渲染都会请求！
  return <div>{data?.name}</div>;
}

// ✅ 正确：副作用在 useEffect 中
function GoodExample({ userId }) {
  const [data, setData] = useState(null);

  useEffect(() => {
    fetch(`/api/users/${userId}`)
      .then((res) => res.json())
      .then(setData);
  }, [userId]);

  return <div>{data?.name}</div>;
}
```

### 8.2 避免无限循环

```jsx
// ❌ 错误：对象作为依赖导致无限循环
function InfiniteLoop() {
  const [data, setData] = useState([]);

  useEffect(() => {
    fetch("/api/data")
      .then((res) => res.json())
      .then((result) => setData(result)); // setData 创建新数组
  }, [data]); // ← 每次 data 变化都重新执行 → 无限循环！

  return <div>{data.length}</div>;
}

// ✅ 正确：空依赖数组
function Fixed() {
  const [data, setData] = useState([]);

  useEffect(() => {
    fetch("/api/data")
      .then((res) => res.json())
      .then(setData);
  }, []); // 只执行一次

  return <div>{data.length}</div>;
}
```

### 8.3 依赖数组完整性

```jsx
// ⚠️ ESLint 的 react-hooks/exhaustive-deps 规则
// 会检测依赖数组是否包含了所有必要的依赖

function UserProfile({ userId }) {
  const [user, setUser] = useState(null);
  const [posts, setPosts] = useState([]);

  // ❌ 缺少依赖
  useEffect(() => {
    fetchUser(userId).then(setUser);
  }, []); // ← 警告：缺少 userId

  // ✅ 依赖完整
  useEffect(() => {
    fetchUser(userId).then(setUser);
  }, [userId]);

  // 处理"函数依赖"的几种方式：
  // 方式1：将函数移入 effect 内部
  useEffect(() => {
    async function loadUser() {
      const data = await fetchUser(userId);
      setUser(data);
    }
    loadUser();
  }, [userId]);

  // 方式2：使用 useCallback
  const fetchPosts = useCallback(async () => {
    const data = await fetch(`/api/users/${userId}/posts`);
    setPosts(data);
  }, [userId]);

  useEffect(() => {
    fetchPosts();
  }, [fetchPosts]);

  return (
    <div>
      <h1>{user?.name}</h1>
      <PostList posts={posts} />
    </div>
  );
}
```

### 8.4 数据获取推荐模式

```jsx
// 推荐使用 AbortController 处理竞态条件
function SearchResults({ query }) {
  const [results, setResults] = useState([]);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState(null);

  useEffect(() => {
    if (!query) {
      setResults([]);
      return;
    }

    const controller = new AbortController();
    setLoading(true);
    setError(null);

    fetch(`/api/search?q=${query}`, { signal: controller.signal })
      .then((res) => {
        if (!res.ok) throw new Error(`HTTP ${res.status}`);
        return res.json();
      })
      .then((data) => {
        setResults(data);
        setLoading(false);
      })
      .catch((err) => {
        if (err.name !== "AbortError") {
          setError(err.message);
          setLoading(false);
        }
      });

    return () => controller.abort();
  }, [query]);

  if (loading) return <div>搜索中...</div>;
  if (error) return <div>错误: {error}</div>;

  return (
    <ul>
      {results.map((item) => (
        <li key={item.id}>{item.name}</li>
      ))}
    </ul>
  );
}
```

### 8.5 三种 Effect Hook 对比总结

| Hook | 执行时机 | 阻塞绘制 | 使用场景 |
|------|---------|---------|---------|
| `useEffect` | 浏览器绘制后 | 否 | 绝大多数副作用 |
| `useLayoutEffect` | DOM 更新后，绘制前 | 是 | 需同步读取/修改 DOM |
| `useInsertionEffect` | DOM 更新后，LayoutEffect 前 | 是 | CSS-in-JS 库样式插入 |

---

## 小结

- React 将副作用从渲染逻辑中分离，让组件保持纯粹
- `useEffect` 是处理副作用的主要工具，根据依赖数组控制执行时机
- 清理函数对于避免内存泄漏至关重要
- `useLayoutEffect` 适用于需要在绘制前同步操作 DOM 的场景
- `useInsertionEffect` 仅供 CSS-in-JS 库使用
- 遵循依赖数组完整性规则，使用 AbortController 处理数据请求的竞态条件
