# 开发工具与调试

## React Developer Tools

React Developer Tools 是调试 React 应用最重要的工具，提供浏览器扩展形式。

### 安装

- **Chrome**: Chrome 网上应用店搜索 "React Developer Tools"
- **Firefox**: Firefox 附加组件商店搜索 "React Developer Tools"
- **Edge**: Edge 扩展商店同样可获取

安装后，打开开发者工具（F12），会看到两个新标签：

### Components 标签

Components 标签展示当前页面的组件树结构：

```
<App>
  <Header>
    <Logo />
    <Nav>
      <NavItem />
      <NavItem />
    </Nav>
  </Header>
  <Main>
    <ArticleList>
      <Article />
      <Article />
    </ArticleList>
  </Main>
</App>
```

**主要功能**：

- **查看组件树**：展开/折叠查看嵌套关系
- **查看 Props**：选中组件后查看传入的 props
- **查看 State**：查看和编辑组件的 state
- **查看 Hooks**：查看 useState、useEffect 等 Hook 的值
- **搜索组件**：按名称搜索组件
- **高亮更新**：勾选 "Highlight updates when components render" 查看哪些组件在重渲染

**实时编辑 Props 和 State**：

在右侧面板中，可以直接修改 props 或 state 的值，界面会实时更新，非常适合调试。

### Profiler 标签

Profiler 用于性能分析，记录组件的渲染时间：

1. 点击蓝色录制按钮开始记录
2. 操作页面（点击、输入等）
3. 再次点击停止录制
4. 查看渲染火焰图和排名图

**火焰图（Flame Chart）**：
- 显示组件渲染的时间层级
- 颜色越红表示渲染耗时越长
- 可以点击查看每次渲染的详细信息

**排名图（Ranked Chart）**：
- 按渲染时间从长到短排序
- 快速定位性能瓶颈组件

## VS Code 调试配置

### 启用 Source Maps

在 `vite.config.js` 中确保开发模式下生成 source map：

```js
export default defineConfig({
  // Vite 开发模式默认支持 source map
  // 如需调整：
  build: {
    sourcemap: true,  // 生产构建也生成 source map
  },
})
```

### 配置 launch.json

在 `.vscode/launch.json` 中配置调试：

```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "type": "chrome",
      "request": "launch",
      "name": "调试 React 应用",
      "url": "http://localhost:5173",
      "webRoot": "${workspaceFolder}/src",
      "sourceMapPathOverrides": {
        "webpack:///src/*": "${webRoot}/*"
      }
    }
  ]
}
```

### 使用断点调试

1. 在 VS Code 中打开需要调试的文件
2. 点击行号左侧添加断点（红色圆点）
3. 按 F5 启动调试
4. 触发相关操作，代码会在断点处暂停
5. 使用调试工具栏：
   - **继续（F5）**：继续执行到下一个断点
   - **单步跳过（F10）**：执行当前行，不进入函数
   - **单步进入（F11）**：进入函数内部
   - **单步跳出（Shift+F11）**：跳出当前函数

### 调试面板变量查看

暂停时可以在 VS Code 中查看：
- **变量面板**：查看当前作用域内所有变量
- **监视面板**：添加自定义表达式持续监视
- **调用堆栈**：查看函数调用链
- **控制台**：在暂停状态下执行表达式

## console.log 调试策略

虽然有强大的调试器，`console.log` 仍然是最常用的调试方式：

### 基本用法

```jsx
function UserProfile({ user }) {
  console.log('UserProfile 渲染', user)

  // 条件调试
  if (!user) {
    console.warn('用户数据为空')
  }

  return <div>{user?.name}</div>
}
```

### 带样式的日志

```jsx
console.log('%c调试信息', 'color: blue; font-size: 16px; font-weight: bold')
console.log('%c警告', 'color: orange; font-size: 14px', data)
```

### 分组日志

```jsx
function UserProfile({ user }) {
  console.group('UserProfile')
  console.log('props:', user)
  console.log('组件渲染时间:', new Date().toLocaleTimeString())
  console.groupEnd()

  return <div>{user?.name}</div>
}
```

### 表格形式

```jsx
const users = [
  { id: 1, name: '张三', age: 25 },
  { id: 2, name: '李四', age: 30 },
  { id: 3, name: '王五', age: 28 },
]

// 以表格形式展示数组或对象
console.table(users)
```

### useEffect 调试

```jsx
import { useEffect, useState } from 'react'

function DataLoader({ userId }) {
  const [data, setData] = useState(null)

  useEffect(() => {
    console.log('useEffect 触发, userId:', userId)

    fetch(`/api/user/${userId}`)
      .then((res) => res.json())
      .then((result) => {
        console.log('数据加载成功:', result)
        setData(result)
      })
      .catch((err) => {
        console.error('数据加载失败:', err)
      })

    return () => {
      console.log('useEffect 清理, userId:', userId)
    }
  }, [userId])

  return <div>{data?.name}</div>
}
```

## React StrictMode

StrictMode 是 React 提供的开发工具组件，帮助发现潜在问题：

```jsx
import { StrictMode } from 'react'
import { createRoot } from 'react-dom/client'
import App from './App'

createRoot(document.getElementById('root')).render(
  <StrictMode>
    <App />
  </StrictMode>,
)
```

### StrictMode 的检查内容

1. **检测不安全的生命周期**：警告已废弃的生命周期方法
2. **检测过时的 API**：如 `findDOMNode`、旧版 Context API
3. **检测意外的副作用**：组件函数会被调用两次来暴露副作用问题
4. **检测废弃的 `ref` 字符串用法**

> StrictMode 只在开发环境下生效，不会影响生产构建。

### 为什么 useEffect 会执行两次

在 StrictMode 下，`useEffect` 会在开发环境中执行两次（挂载→卸载→挂载），这是为了验证清理逻辑是否正确：

```jsx
useEffect(() => {
  const timer = setInterval(() => {
    console.log('定时器运行中')
  }, 1000)

  // StrictMode 会执行清理再重新执行，确保清理逻辑正确
  return () => {
    clearInterval(timer)
    console.log('定时器已清理')
  }
}, [])
```

## 常见调试技巧

### 组件不更新

```jsx
// 检查 key 是否正确
{items.map((item) => (
  <Item key={item.id} data={item} />  // 使用稳定且唯一的 key
))}

// 检查 state 是否不可变更新
// 错误：直接修改数组
items.push(newItem)
setItems(items)

// 正确：创建新数组
setItems([...items, newItem])
```

### Props 传递问题

```jsx
// 在子组件中打印 props 检查传递是否正确
function ChildComponent(props) {
  console.log('ChildComponent props:', props)
  return <div>{props.name}</div>
}
```

### 性能问题定位

```jsx
import { Profiler } from 'react'

function onRenderCallback(id, phase, actualDuration) {
  console.log(`组件 ${id} ${phase} 耗时: ${actualDuration}ms`)
}

function App() {
  return (
    <Profiler id="App" onRender={onRenderCallback}>
      <ExpensiveComponent />
    </Profiler>
  )
}
```

## 小结

- **React Developer Tools**：必备工具，查看组件树、编辑 props/state、性能分析
- **VS Code 调试**：配置 source maps 和断点，实现 IDE 内调试
- **console.log**：简单有效，使用分组、表格等方式提升可读性
- **StrictMode**：开发模式自动检测潜在问题，帮助写出更健壮的代码
- **Profiler**：精确定位性能瓶颈组件
