# 开发环境搭建

## Node.js 安装

React 开发依赖 Node.js 环境。Node.js 提供了 JavaScript 运行时和包管理器 npm。

### 安装步骤

1. 访问 [Node.js 官网](https://nodejs.org/)，下载 LTS 版本（推荐）
2. 运行安装程序，按照提示完成安装
3. 验证安装：

```bash
node -v    # 例如：v20.11.0
npm -v     # 例如：10.2.4
```

> **建议**：使用 [nvm](https://github.com/nvm-sh/nvm)（Windows 上用 nvm-windows）管理多个 Node.js 版本，方便在不同项目间切换。

```bash
# nvm 常用命令
nvm install 20         # 安装 Node 20
nvm use 20             # 切换到 Node 20
nvm list               # 列出已安装版本
```

## 包管理器对比

Node.js 自带 npm，但还有其他流行的选择：

| 特性 | npm | yarn | pnpm |
|------|-----|------|------|
| 速度 | 中等 | 较快 | 最快 |
| 磁盘占用 | 高 | 中等 | 低（硬链接） |
| 安装方式 | 内置 | `npm i -g yarn` | `npm i -g pnpm` |
| 锁文件 | package-lock.json | yarn.lock | pnpm-lock.yaml |
| 幽灵依赖 | 有 | 有 | 无（严格隔离） |
| 推荐场景 | 通用 | 旧项目 | 新项目推荐 |

### pnpm 安装与配置

```bash
npm install -g pnpm
pnpm -v    # 验证安装

# 设置国内镜像（加速下载）
pnpm config set registry https://registry.npmmirror.com
```

### 常用命令对比

```bash
# 初始化项目
npm init -y
yarn init -y
pnpm init

# 安装依赖
npm install react
yarn add react
pnpm add react

# 安装开发依赖
npm install -D vite
yarn add -D vite
pnpm add -D vite

# 全局安装
npm install -g create-react-app
yarn global add create-react-app
pnpm add -g create-react-app

# 运行脚本
npm run dev
yarn dev
pnpm dev
```

## VS Code 编辑器配置

### 推荐扩展

安装以下扩展来提升 React 开发体验：

**1. ES7+ React/Redux/React-Native snippets**
- 提供常用代码片段，输入 `rafce` 快速生成 React 函数组件
- 常用缩写：
  - `rafce` → React Arrow Function Component with Export
  - `rafcp` → React Arrow Function Component with PropTypes
  - `usf` → useState
  - `uef` → useEffect

**2. Prettier - Code formatter**
- 自动格式化代码，保持团队风格一致
- 在项目根目录创建 `.prettierrc`：

```json
{
  "semi": true,
  "singleQuote": true,
  "trailingComma": "all",
  "printWidth": 80,
  "tabWidth": 2,
  "arrowParens": "always"
}
```

**3. ESLint**
- 代码质量检查，捕获潜在错误
- VS Code 中安装 ESLint 扩展后，配合项目中的 ESLint 配置自动工作

**4. 其他推荐扩展**

| 扩展名 | 用途 |
|--------|------|
| Auto Rename Tag | 自动重命名配对的 HTML/JSX 标签 |
| Path Intellisense | 路径自动补全 |
| Import Cost | 显示导入包的大小 |
| GitLens | Git 增强 |
| Color Highlight | 颜色值高亮 |
| Error Lens | 行内显示错误和警告 |

### VS Code 设置

在项目根目录创建 `.vscode/settings.json`：

```json
{
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": "explicit"
  },
  "emmet.includeLanguages": {
    "javascript": "javascriptreact"
  },
  "files.associations": {
    "*.js": "javascriptreact"
  }
}
```

## Chrome DevTools React 扩展

React Developer Tools 是 Chrome 浏览器扩展，用于调试 React 应用：

1. 打开 Chrome 网上应用店，搜索 **React Developer Tools**
2. 安装后，打开 React 应用的页面
3. 按 F12 打开开发者工具，会看到新增的 **Components** 和 **Profiler** 标签

> 如果使用 Edge 浏览器，同样可以在 Edge 扩展商店中找到 React Developer Tools。

## 项目脚手架对比

### create-react-app（CRA）

CRA 是 React 官方曾经推荐的脚手架工具：

```bash
npx create-react-app my-app
cd my-app
npm start
```

**特点**：
- 基于 Webpack，零配置
- 开箱即用（包含测试、构建等）
- 配置复杂，难以自定义
- **已停止维护**（2023 年起不再推荐使用）

### Vite（推荐）

Vite 是新一代前端构建工具，由 Vue 作者 Evan You 创建：

```bash
npm create vite@latest my-app -- --template react
cd my-app
npm install
npm run dev
```

**特点**：
- 基于原生 ES Module，开发启动极快
- 使用 Rollup 打包，构建产物小
- 配置简洁，扩展性强
- 热更新（HMR）几乎即时
- 官方推荐的 React 项目创建方式

### 对比总结

| 特性 | CRA | Vite |
|------|-----|------|
| 开发启动速度 | 慢（打包所有文件） | 快（按需编译） |
| 热更新速度 | 中等 | 极快 |
| 构建工具 | Webpack | Rollup / esbuild |
| 配置灵活性 | 低（需 eject） | 高（vite.config.js） |
| 维护状态 | 停止维护 | 活跃开发中 |
| 生态 | 成熟 | 快速成长 |
| 推荐度 | 不推荐 | **强烈推荐** |

## 小结

推荐的开发环境配置：
- **Node.js**：使用 nvm 管理，选择 LTS 版本
- **包管理器**：推荐 pnpm（快速、节省磁盘空间）
- **编辑器**：VS Code + 推荐扩展（ES7 snippets、Prettier、ESLint）
- **脚手架**：Vite（快速、灵活、现代）
- **调试工具**：React Developer Tools
