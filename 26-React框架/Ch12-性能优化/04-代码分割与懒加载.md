## 1. React.lazy 基础

### 1.1 基本概念

`React.lazy` 允许你将组件渲染推迟到需要加载它时再加载其代码。它必须配合 `<Suspense>` 使用：

```jsx
import { lazy, Suspense } from "react";

// lazy 接受一个返回 Promise 的函数
// 该 Promise 必须 resolve 一个带有 default export 的模块
const LazyComponent = lazy(() => import("./MyComponent"));

function App() {
  return (
    <Suspense fallback={<div>Loading...</div>}>
      <LazyComponent />
    </Suspense>
  );
}
```

### 1.2 工作原理

```
传统方式（打包在一起）：
  main.js (500KB) → 包含所有组件代码

代码分割后：
  main.js (100KB) → 核心代码
  chunk-Home.js (80KB) → 首页组件
  chunk-About.js (50KB) → 关于页组件
  chunk-Admin.js (120KB) → 管理后台（按需加载）

首次加载：只下载 main.js + chunk-Home.js
访问 /about：按需下载 chunk-About.js
访问 /admin：按需下载 chunk-Admin.js
```

### 1.3 与普通 import 的区别

```js
// 普通 import - 打包时就包含在 bundle 中
import HomeComponent from "./Home";

// 动态 import - 按需加载
const HomeComponent = lazy(() => import("./Home"));
// 编译器会将 ./Home 分离为独立的 chunk
```

## 2. Suspense Fallback

### 2.1 基本 Fallback

```jsx
function App() {
  return (
    <Suspense fallback={<div>加载中...</div>}>
      <LazyComponent />
    </Suspense>
  );
}
```

### 2.2 骨架屏 Fallback

```jsx
function DashboardSkeleton() {
  return (
    <div className="skeleton">
      <div className="skeleton-header" />
      <div className="skeleton-sidebar" />
      <div className="skeleton-content">
        {Array.from({ length: 5 }).map((_, i) => (
          <div key={i} className="skeleton-row" />
        ))}
      </div>
    </div>
  );
}

const Dashboard = lazy(() => import("./Dashboard"));

function App() {
  return (
    <Suspense fallback={<DashboardSkeleton />}>
      <Dashboard />
    </Suspense>
  );
}
```

### 2.3 嵌套 Suspense

```jsx
function App() {
  return (
    <Suspense fallback={<AppShell />}>
      <Header />
      <main>
        <Suspense fallback={<ContentSkeleton />}>
          <LazyContent />
        </Suspense>
        <Suspense fallback={<SidebarSkeleton />}>
          <LazySidebar />
        </Suspense>
      </main>
    </Suspense>
  );
}
// 渐进式加载：
// 1. 先显示 AppShell
// 2. Header 加载完成，显示 Header
// 3. Content 和 Skeleton 各自独立加载
// 4. 最终全部显示
```

## 3. 路由级代码分割

### 3.1 React Router + lazy

```jsx
import { lazy, Suspense } from "react";
import { BrowserRouter, Routes, Route } from "react-router-dom";

// 路由级懒加载
const Home = lazy(() => import("./pages/Home"));
const About = lazy(() => import("./pages/About"));
const Dashboard = lazy(() => import("./pages/Dashboard"));
const Settings = lazy(() => import("./pages/Settings"));
const NotFound = lazy(() => import("./pages/NotFound"));

function App() {
  return (
    <BrowserRouter>
      <Suspense fallback={<PageSkeleton />}>
        <Routes>
          <Route path="/" element={<Home />} />
          <Route path="/about" element={<About />} />
          <Route path="/dashboard" element={<Dashboard />} />
          <Route path="/settings" element={<Settings />} />
          <Route path="*" element={<NotFound />} />
        </Routes>
      </Suspense>
    </BrowserRouter>
  );
}
```

### 3.2 React Router v6.4+ lazy 路由

```jsx
import { createBrowserRouter, RouterProvider } from "react-router-dom";

const router = createBrowserRouter([
  {
    path: "/",
    lazy: () => import("./pages/Home").then((m) => ({ Component: m.default })),
  },
  {
    path: "/dashboard",
    lazy: () =>
      import("./pages/Dashboard").then((m) => ({ Component: m.default })),
  },
  {
    path: "/settings",
    lazy: () =>
      import("./pages/Settings").then((m) => ({ Component: m.default })),
  },
]);

function App() {
  return <RouterProvider router={router} />;
}
```

## 4. 组件级代码分割

### 4.1 条件加载

```jsx
import { lazy, Suspense, useState } from "react";

const HeavyChart = lazy(() => import("./components/HeavyChart"));
const HeavyEditor = lazy(() => import("./components/HeavyEditor"));

function Dashboard() {
  const [showChart, setShowChart] = useState(false);
  const [showEditor, setShowEditor] = useState(false);

  return (
    <div>
      <button onClick={() => setShowChart(true)}>显示图表</button>
      <button onClick={() => setShowEditor(true)}>打开编辑器</button>

      {showChart && (
        <Suspense fallback={<ChartSkeleton />}>
          <HeavyChart />
        </Suspense>
      )}

      {showEditor && (
        <Suspense fallback={<EditorSkeleton />}>
          <HeavyEditor />
        </Suspense>
      )}
    </div>
  );
}
```

### 4.2 Tab 切换懒加载

```jsx
const TabA = lazy(() => import("./tabs/TabA"));
const TabB = lazy(() => import("./tabs/TabB"));
const TabC = lazy(() => import("./tabs/TabC"));

function TabContainer() {
  const [activeTab, setActiveTab] = useState("a");

  const tabComponents = {
    a: TabA,
    b: TabB,
    c: TabC,
  };

  const ActiveComponent = tabComponents[activeTab];

  return (
    <div>
      <nav>
        <button onClick={() => setActiveTab("a")}>Tab A</button>
        <button onClick={() => setActiveTab("b")}>Tab B</button>
        <button onClick={() => setActiveTab("c")}>Tab C</button>
      </nav>

      <Suspense fallback={<TabSkeleton />}>
        <ActiveComponent />
      </Suspense>
    </div>
  );
}
```

### 4.3 对话框懒加载

```jsx
// 只有打开对话框时才加载其代码
```

## 5. 命名导出的懒加载

### 5.1 处理命名导出

```jsx
// 如果模块使用命名导出：
// components/Chart.jsx
export function Chart() { ... }
export function ChartLegend() { ... }

// 需要这样懒加载：
const Chart = lazy(() =>
  import("./components/Chart").then((module) => ({
    default: module.Chart,
  }))
);

const ChartLegend = lazy(() =>
  import("./components/Chart").then((module) => ({
    default: module.ChartLegend,
  }))
);
```

### 5.2 封装辅助函数

```jsx
// 工具函数
function namedLazy(loadModule, name) {
  return lazy(() =>
    loadModule().then((module) => ({ default: module[name] }))
  );
}

// 使用
const Chart = namedLazy(() => import("./components/Chart"), "Chart");
const ChartLegend = namedLazy(
  () => import("./components/Chart"),
  "ChartLegend"
);

function App() {
  return (
    <Suspense fallback={<div>Loading...</div>}>
      <Chart />
      <ChartLegend />
    </Suspense>
  );
}
```

### 5.3 批量导入

```jsx
// 同时懒加载一个模块的多个导出
function lazyNamedExports(modulePath, exportNames) {
  const components = {};
  const modulePromise = import(modulePath);

  for (const name of exportNames) {
    components[name] = lazy(() =>
      modulePromise.then((mod) => ({ default: mod[name] }))
    );
  }

  return components;
}

const { Chart, ChartLegend, ChartTooltip } = lazyNamedExports(
  "./components/Chart",
  ["Chart", "ChartLegend", "ChartTooltip"]
);
```

## 6. 预取与预加载

### 6.1 预取 (Prefetch)

```jsx
// 在空闲时间预取可能需要的模块
function prefetchModule(importFn) {
  const link = document.createElement("link");
  link.rel = "prefetch";
  // 浏览器会在空闲时下载
}

// 使用 IntersectionObserver 在元素可见时预取
function usePrefetch(importFn, ref) {
  useEffect(() => {
    if (!ref.current) return;

    const observer = new IntersectionObserver(
      ([entry]) => {
        if (entry.isIntersecting) {
          importFn(); // 触发预加载
          observer.disconnect();
        }
      },
      { threshold: 0 }
    );

    observer.observe(ref.current);
    return () => observer.disconnect();
  }, [importFn, ref]);
}

// 组件中使用
function DashboardLink() {
  const ref = useRef(null);
  const prefetchDashboard = useCallback(
    () => import("./pages/Dashboard"),
    []
  );

  usePrefetch(prefetchDashboard, ref);

  return <Link ref={ref} to="/dashboard">Dashboard</Link>;
}
```

### 6.2 Hover 预加载

```jsx
const LazyDashboard = lazy(() => import("./pages/Dashboard"));

function NavLink({ to, children }) {
  const prefetch = useCallback(() => {
    // 鼠标悬停时预加载
    import("./pages/Dashboard");
  }, []);

  return (
    <Link to={to} onMouseEnter={prefetch}>
      {children}
    </Link>
  );
}

// 使用
<nav>
  <NavLink to="/dashboard">Dashboard</NavLink>
  <NavLink to="/settings">Settings</NavLink>
</nav>
```

### 6.3 预加载 Preload

```jsx
// 与 prefetch 不同，preload 是高优先级加载
// <link rel="preload"> 会立即开始下载

// 组件自带 preload 方法
const Dashboard = lazy(() => import("./pages/Dashboard"));
Dashboard.preload = () => import("./pages/Dashboard");

// 在路由转换前预加载
function App() {
  const location = useLocation();

  useEffect(() => {
    if (location.pathname === "/") {
      // 在首页时预加载 Dashboard
      Dashboard.preload();
    }
  }, [location]);

  return (
    <Suspense fallback={<PageSkeleton />}>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/dashboard" element={<Dashboard />} />
      </Routes>
    </Suspense>
  );
}
```

## 7. 动态 import()

### 7.1 基本用法

```jsx
// 动态导入可以在任何地方使用，不仅限于 lazy
async function loadComponent() {
  const module = await import("./MyComponent");
  return module.default;
}

// 条件导入
async function loadChart(type) {
  if (type === "bar") {
    const { BarChart } = await import("./charts/BarChart");
    return BarChart;
  } else if (type === "line") {
    const { LineChart } = await import("./charts/LineChart");
    return LineChart;
  }
}
```

### 7.2 命名 chunk

```jsx
// webpack 魔法注释 - 指定 chunk 名称
const Admin = lazy(() =>
  import(/* webpackChunkName: "admin" */ "./pages/Admin")
);

const Settings = lazy(() =>
  import(/* webpackChunkName: "settings" */ "./pages/Settings")
);

// Vite 也支持
const Dashboard = lazy(() =>
  import(/* viteChunkName: "dashboard" */ "./pages/Dashboard")
);
```

### 7.3 预加载模式

```jsx
// 带预加载的动态导入
function createLazyComponent(importFn) {
  const LazyComponent = lazy(importFn);
  LazyComponent.preload = importFn;
  return LazyComponent;
}

const Dashboard = createLazyComponent(() =>
  import("./pages/Dashboard")
);

// 可以在需要时触发预加载
function onRouteChange() {
  Dashboard.preload(); // 提前开始加载
}
```

## 8. Bundle 优化策略

### 8.1 共享依赖

```jsx
// Vite 配置 - 将大型共享依赖分离
// vite.config.js
export default {
  build: {
    rollupOptions: {
      output: {
        manualChunks: {
          // 将 React 分离到独立 chunk
          "react-vendor": [
            "react",
            "react-dom",
            "react-router-dom",
          ],
          // 将 UI 库分离
          "ui-vendor": [
            "@mui/material",
            "@emotion/react",
            "@emotion/styled",
          ],
          // 将工具库分离
          "utils-vendor": [
            "lodash-es",
            "date-fns",
            "axios",
          ],
        },
      },
    },
  },
};
```

### 8.2 Webpack SplitChunks

```js
// webpack.config.js
module.exports = {
  optimization: {
    splitChunks: {
      chunks: "all",
      cacheGroups: {
        vendor: {
          test: /[\\/]node_modules[\\/]/,
          name: "vendors",
          chunks: "all",
          priority: 10,
        },
        common: {
          minChunks: 2,
          priority: 5,
          reuseExistingChunk: true,
        },
      },
    },
  },
};
```

### 8.3 避免过度分割

```
过度分割的问题：
  - 每个 chunk 都有运行时开销
  - HTTP/1.1 下并行请求数受限
  - 小 chunk 的压缩效果差

最佳实践：
  - 首屏 chunk 控制在 200KB 以内（gzip 后）
  - 按路由分割是最常见的方式
  - 共享依赖单独打包
  - 使用 HTTP/2 后，小 chunk 的问题缓解
```

## 9. Tree Shaking

### 9.1 什么是 Tree Shaking

Tree Shaking 是一种通过移除 JavaScript 中未使用的代码来减小最终 bundle 大小的技术：

```js
// math.js
export function add(a, b) {
  return a + b;
}

export function subtract(a, b) {
  return a - b;
}

export function multiply(a, b) {
  return a * b;
}

// app.js
import { add } from "./math";
// subtract 和 multiply 会被 Tree Shaking 移除
console.log(add(1, 2));
```

### 9.2 Tree Shaking 的条件

```
必要条件：
  1. 使用 ES Modules (import/export)
     - CommonJS (require/module.exports) 无法被 Tree Shaking
  2. 构建工具支持（Webpack、Vite、Rollup）
  3. 代码中没有副作用
  4. package.json 中设置 "sideEffects": false

常见破坏 Tree Shaking 的模式：
  ❌ import "./polyfill"（副作用导入）
  ❌ module.exports（CommonJS）
  ❌ 动态 require()
  ❌ 全局 CSS 导入 import "./styles.css"（不是 JS 侧的问题，但会影响）
```

### 9.3 优化 Tree Shaking

```jsx
// ❌ 整个库导入 - 即使 Tree Shaking 也包含大量未使用代码
import _ from "lodash";
_.debounce(fn, 300);

// ✅ 按需导入
import debounce from "lodash/debounce";
// 或使用 lodash-es（ESM 版本）
import { debounce } from "lodash-es";

// ❌ 整个图标库导入
import { Icon } from "@ant-design/icons";
// 包含所有图标

// ✅ 只导入需要的图标
import { SearchOutlined, UserOutlined } from "@ant-design/icons";

// ❌ 导入整个 MUI
import { Button, TextField } from "@mui/material";

// ✅ 路径导入
import Button from "@mui/material/Button";
import TextField from "@mui/material/TextField";
```

## 10. sideEffects 配置

### 10.1 package.json 中的 sideEffects

```json
{
  "name": "my-library",
  "sideEffects": false
}
```

`"sideEffects": false` 告诉打包工具：这个包中的任何模块如果没有被导入使用，就可以安全地移除。

### 10.2 指定有副作用的文件

```json
{
  "sideEffects": [
    "*.css",
    "*.scss",
    "./src/polyfills.js",
    "./src/global-setup.js"
  ]
}
```

### 10.3 项目中的应用

```json
// 你的 React 项目 package.json
{
  "name": "my-react-app",
  "sideEffects": false
}
```

```js
// 这允许打包工具安全地移除未使用的导出
// utils.js
export function usedFunction() {
  return "used";
}

export function unusedFunction() {
  return "unused"; // 会被 Tree Shaking 移除
}

// 只有导入了 usedFunction 时，unusedFunction 才会被保留
```

### 10.4 CSS 和副作用

```jsx
// CSS 导入是有副作用的
import "./global.css"; // 副作用：注入 <style> 标签
import styles from "./Component.module.css"; // CSS Modules：无副作用

// 如果设置了 "sideEffects": false，
// 必须将 CSS 文件加入 sideEffects 列表：
{
  "sideEffects": ["*.css", "*.scss"]
}
```

## 小结

- React.lazy 配合 Suspense 实现组件级和路由级的代码分割
- 使用骨架屏作为 fallback 提供更好的加载体验
- 预取和预加载可以在用户交互前提前加载代码
- 动态 import() 支持条件加载和命名 chunk
- Tree Shaking 依赖 ES Modules 和 "sideEffects" 配置
- 合理的 bundle 分割策略平衡首屏加载和后续体验
- 避免过度分割，HTTP/2 环境下小 chunk 的开销较低
