# 客户端与服务端状态

## 1. 客户端状态 vs 服务端状态

### 本质区别

```
客户端状态 (Client State)          服务端状态 (Server State)
├── UI 状态：模态框开关、Tab 切换    ├── API 数据：用户列表、文章内容
├── 表单状态：输入值、验证错误       ├── 缓存数据：分页、搜索结果
├── 本地状态：主题、语言偏好        ├── 实时数据：WebSocket 消息
├── 临时状态：选中项、拖拽位置       └── 需要同步：可能过时、需要刷新
└── 不需要同步，纯本地
```

### 为什么需要不同的工具

| 特性 | 客户端状态 | 服务端状态 |
|------|-----------|-----------|
| 数据来源 | 用户交互、UI 逻辑 | 服务器 API |
| 时效性 | 实时 | 可能过时 |
| 同步需求 | 无 | 需要定期刷新 |
| 错误处理 | 简单 | 重试、缓存失效 |
| 加载状态 | 通常不需要 | 必需（loading/error） |
| 推荐工具 | useState/Context/Zustand | TanStack Query/SWR |

## 2. TanStack Query（React Query）

### 安装

```bash
npm install @tanstack/react-query
npm install -D @tanstack/react-query-devtools
```

### 基础配置

```tsx
// main.tsx
import { QueryClient, QueryClientProvider } from '@tanstack/react-query'
import { ReactQueryDevtools } from '@tanstack/react-query-devtools'

const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      staleTime: 5 * 60 * 1000,     // 5 分钟后数据变陈旧
      gcTime: 10 * 60 * 1000,       // 10 分钟后清除缓存（v5，旧名 cacheTime）
      retry: 2,                       // 失败重试次数
      refetchOnWindowFocus: true,     // 窗口聚焦时重新获取
    },
  },
})

function App() {
  return (
    <QueryClientProvider client={queryClient}>
      <MyApp />
      <ReactQueryDevtools initialIsOpen={false} />
    </QueryClientProvider>
  )
}
```

### 基本用法

```tsx
import { useQuery, useMutation, useQueryClient } from '@tanstack/react-query'

// GET 请求
function UserList() {
  const { data, isLoading, error, isFetching, refetch } = useQuery({
    queryKey: ['users'],                           // 缓存 key
    queryFn: async () => {
      const res = await fetch('/api/users')
      if (!res.ok) throw new Error('Failed to fetch')
      return res.json() as Promise<User[]>
    },
    staleTime: 5 * 60 * 1000,                     // 5 分钟内不会重新获取
  })

  if (isLoading) return <Spinner />
  if (error) return <div>Error: {error.message}</div>

  return (
    <div>
      {isFetching && <span>刷新中...</span>}
      <button onClick={() => refetch()}>刷新</button>
      <ul>
        {data?.map(user => <li key={user.id}>{user.name}</li>)}
      </ul>
    </div>
  )
}
```

### 带参数的查询

```tsx
// 根据 ID 获取
function UserDetail({ userId }: { userId: string }) {
  const { data: user, isLoading } = useQuery({
    queryKey: ['users', userId],          // queryKey 包含参数
    queryFn: () => fetchUser(userId),
    enabled: !!userId,                     // userId 存在时才执行
  })

  if (isLoading) return <Spinner />
  if (!user) return <div>Not found</div>
  return <div>{user.name}</div>
}

// 分页查询
function PaginatedUsers() {
  const [page, setPage] = useState(1)

  const { data, isLoading, isPreviousData } = useQuery({
    queryKey: ['users', page],
    queryFn: () => fetchUsers(page),
    keepPreviousData: true,    // 保留上一页数据，避免闪烁
  })

  return (
    <div style={{ opacity: isPreviousData ? 0.5 : 1 }}>
      <UserList users={data?.users ?? []} />
      <button disabled={page === 1} onClick={() => setPage(p => p - 1)}>上一页</button>
      <button disabled={!data?.hasMore} onClick={() => setPage(p => p + 1)}>下一页</button>
    </div>
  )
}
```

### Mutations（修改数据）

```tsx
function CreateUserForm() {
  const queryClient = useQueryClient()

  const mutation = useMutation({
    mutationFn: (newUser: CreateUserRequest) =>
      fetch('/api/users', {
        method: 'POST',
        body: JSON.stringify(newUser),
      }).then(res => res.json()),

    onSuccess: () => {
      // 使相关查询失效，触发重新获取
      queryClient.invalidateQueries({ queryKey: ['users'] })
    },

    onError: (error) => {
      console.error('Create failed:', error)
    },

    // 乐观更新
    onMutate: async (newUser) => {
      // 取消正在进行的查询
      await queryClient.cancelQueries({ queryKey: ['users'] })

      // 保存之前的数据（用于回滚）
      const previousUsers = queryClient.getQueryData<User[]>(['users'])

      // 乐观更新缓存
      queryClient.setQueryData<User[]>(['users'], (old) =>
        old ? [...old, { ...newUser, id: 'temp' }] : []
      )

      return { previousUsers }
    },

    onError: (err, newUser, context) => {
      // 回滚
      queryClient.setQueryData(['users'], context?.previousUsers)
    },
  })

  const handleSubmit = (e: React.FormEvent<HTMLFormElement>) => {
    e.preventDefault()
    const formData = new FormData(e.currentTarget)
    mutation.mutate({
      name: formData.get('name') as string,
      email: formData.get('email') as string,
    })
  }

  return (
    <form onSubmit={handleSubmit}>
      <input name="name" required />
      <input name="email" type="email" required />
      <button type="submit" disabled={mutation.isPending}>
        {mutation.isPending ? 'Creating...' : 'Create'}
      </button>
      {mutation.isError && <div>Error: {mutation.error.message}</div>}
      {mutation.isSuccess && <div>Created successfully!</div>}
    </form>
  )
}
```

### 无限滚动（Infinite Query）

```tsx
import { useInfiniteQuery } from '@tanstack/react-query'

function InfiniteUserList() {
  const {
    data,
    fetchNextPage,
    hasNextPage,
    isFetchingNextPage,
    isLoading,
  } = useInfiniteQuery({
    queryKey: ['users'],
    queryFn: ({ pageParam }) => fetchUsers(pageParam),
    initialPageParam: 1,
    getNextPageParam: (lastPage) => lastPage.nextCursor,
  })

  const allUsers = data?.pages.flatMap(page => page.users) ?? []

  return (
    <div>
      {allUsers.map(user => (
        <div key={user.id}>{user.name}</div>
      ))}
      <button
        onClick={() => fetchNextPage()}
        disabled={!hasNextPage || isFetchingNextPage}
      >
        {isFetchingNextPage ? 'Loading more...' : hasNextPage ? 'Load More' : 'Nothing more'}
      </button>
    </div>
  )
}
```

## 3. Zustand 用于客户端状态

```tsx
import { create } from 'zustand'

// UI 状态
interface UIState {
  sidebarOpen: boolean
  theme: 'light' | 'dark'
  modal: { type: string; data?: unknown } | null
  notifications: Notification[]

  toggleSidebar: () => void
  setTheme: (theme: 'light' | 'dark') => void
  openModal: (type: string, data?: unknown) => void
  closeModal: () => void
  addNotification: (notification: Notification) => void
  removeNotification: (id: string) => void
}

export const useUIStore = create<UIState>()((set) => ({
  sidebarOpen: true,
  theme: 'light',
  modal: null,
  notifications: [],

  toggleSidebar: () => set((state) => ({ sidebarOpen: !state.sidebarOpen })),
  setTheme: (theme) => set({ theme }),
  openModal: (type, data) => set({ modal: { type, data } }),
  closeModal: () => set({ modal: null }),
  addNotification: (notification) =>
    set((state) => ({ notifications: [...state.notifications, notification] })),
  removeNotification: (id) =>
    set((state) => ({
      notifications: state.notifications.filter((n) => n.id !== id),
    })),
}))

// 表单状态（也可以用 React Hook Form）
interface FormState {
  values: Record<string, unknown>
  errors: Record<string, string>
  touched: Record<string, boolean>

  setFieldValue: (field: string, value: unknown) => void
  setFieldError: (field: string, error: string) => void
  setFieldTouched: (field: string) => void
  reset: () => void
}
```

## 4. 表单状态管理

### React Hook Form

```bash
npm install react-hook-form @hookform/resolvers zod
```

```tsx
import { useForm } from 'react-hook-form'
import { zodResolver } from '@hookform/resolvers/zod'
import { z } from 'zod'

// Zod schema 定义
const userSchema = z.object({
  name: z.string().min(2, '至少 2 个字符'),
  email: z.string().email('邮箱格式不正确'),
  age: z.number().min(0, '年龄不能为负数').max(150),
  role: z.enum(['admin', 'user', 'guest']),
})

type UserFormData = z.infer<typeof userSchema>

function UserForm({ onSubmit }: { onSubmit: (data: UserFormData) => void }) {
  const {
    register,
    handleSubmit,
    formState: { errors, isSubmitting },
    watch,
    setValue,
  } = useForm<UserFormData>({
    resolver: zodResolver(userSchema),
    defaultValues: {
      name: '',
      email: '',
      age: 0,
      role: 'user',
    },
  })

  const watchedName = watch('name')  // 监听某个字段

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      <div>
        <label>Name</label>
        <input {...register('name')} />
        {errors.name && <span>{errors.name.message}</span>}
      </div>

      <div>
        <label>Email</label>
        <input type="email" {...register('email')} />
        {errors.email && <span>{errors.email.message}</span>}
      </div>

      <div>
        <label>Age</label>
        <input type="number" {...register('age', { valueAsNumber: true })} />
        {errors.age && <span>{errors.age.message}</span>}
      </div>

      <div>
        <label>Role</label>
        <select {...register('role')}>
          <option value="user">User</option>
          <option value="admin">Admin</option>
          <option value="guest">Guest</option>
        </select>
      </div>

      <button type="submit" disabled={isSubmitting}>
        {isSubmitting ? 'Submitting...' : 'Submit'}
      </button>
    </form>
  )
}
```

### Formik（备选方案）

```tsx
import { Formik, Form, Field, ErrorMessage } from 'formik'
import * as Yup from 'yup'

const schema = Yup.object({
  name: Yup.string().min(2).required(),
  email: Yup.string().email().required(),
})

function UserForm() {
  return (
    <Formik
      initialValues={{ name: '', email: '' }}
      validationSchema={schema}
      onSubmit={async (values, { setSubmitting }) => {
        await submitUser(values)
        setSubmitting(false)
      }}
    >
      {({ isSubmitting }) => (
        <Form>
          <Field name="name" placeholder="Name" />
          <ErrorMessage name="name" component="div" />
          <Field name="email" type="email" />
          <ErrorMessage name="email" component="div" />
          <button type="submit" disabled={isSubmitting}>Submit</button>
        </Form>
      )}
    </Formik>
  )
}
```

### React Hook Form vs Formik

| 特性 | React Hook Form | Formik |
|------|----------------|--------|
| 性能 | 更好（非受控） | 一般（受控） |
| 包大小 | ~8KB | ~44KB |
| 验证 | 集成 Zod/Yup | 集成 Yup |
| TypeScript | 原生支持 | 支持 |
| 学习曲线 | 低 | 中等 |

## 5. URL 作为状态管理器

URL 本身是一个优秀的状态管理器，尤其是对于需要持久化、可分享的状态。

### 使用搜索参数

```tsx
import { useSearchParams } from 'react-router-dom'

function SearchPage() {
  const [searchParams, setSearchParams] = useSearchParams()

  // 从 URL 读取状态
  const query = searchParams.get('q') ?? ''
  const page = Number(searchParams.get('page') ?? '1')
  const sort = searchParams.get('sort') ?? 'relevance'
  const filter = searchParams.get('filter') ?? 'all'

  // 更新 URL 状态
  const setQuery = (q: string) => {
    setSearchParams(prev => {
      prev.set('q', q)
      prev.set('page', '1')  // 搜索时重置页码
      return prev
    })
  }

  const setPage = (p: number) => {
    setSearchParams(prev => {
      prev.set('page', String(p))
      return prev
    })
  }

  // URL 状态变化自动触发 TanStack Query 重新获取
  const { data } = useQuery({
    queryKey: ['search', { query, page, sort, filter }],
    queryFn: () => searchAPI({ query, page, sort, filter }),
    enabled: !!query,
  })

  return (
    <div>
      <input value={query} onChange={e => setQuery(e.target.value)} />
      <select value={sort} onChange={e => setSearchParams(prev => {
        prev.set('sort', e.target.value)
        return prev
      })}>
        <option value="relevance">Relevance</option>
        <option value="date">Date</option>
      </select>
      <Results data={data} />
      <Pagination page={page} onPageChange={setPage} />
    </div>
  )
}
```

## 6. 架构决策指南

### 选择合适的工具

```
状态是什么？
  ├── 服务端数据 → TanStack Query / SWR
  ├── 全局 UI 状态 → Zustand / Context
  ├── 表单状态 → React Hook Form
  ├── URL 可表示的状态 → URL 搜索参数
  └── 组件局部状态 → useState / useReducer
```

### 完整架构示例

```
src/
├── stores/              # Zustand stores（客户端状态）
│   ├── uiStore.ts       # UI 状态（sidebar, theme, modal）
│   ├── authStore.ts     # 认证状态（token, user profile 缓存）
│   └── cartStore.ts     # 购物车状态
├── hooks/               # TanStack Query hooks（服务端状态）
│   ├── useUsers.ts
│   ├── usePosts.ts
│   └── useComments.ts
├── components/
│   ├── forms/           # React Hook Form 组件
│   └── ...
└── types/
    └── index.ts
```

### 混合使用


## 7. 性能优化

### TanStack Query 优化

```tsx
// 1. 设置合理的 staleTime
const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      staleTime: 5 * 60 * 1000,  // 5 分钟内不重新获取
    },
  },
})

// 2. 使用 select 只取需要的数据
const { data: userName } = useQuery({
  queryKey: ['user', userId],
  queryFn: () => fetchUser(userId),
  select: (user) => user.name,  // 只取 name，变化时才重新渲染
})

// 3. 预取数据
const prefetchUser = (id: string) => {
  queryClient.prefetchQuery({
    queryKey: ['user', id],
    queryFn: () => fetchUser(id),
  })
}

// 4. 使用 placeholderData 保持 UI 连贯
const { data } = useQuery({
  queryKey: ['users', page],
  queryFn: () => fetchUsers(page),
  placeholderData: keepPreviousData,  // v5
})
```

### 避免常见陷阱

```tsx
// ❌ 不要在 queryKey 中放对象引用
const { data } = useQuery({
  queryKey: ['users', { page, filter }],  // 如果对象每次都是新的，会导致无限请求
  queryFn: () => fetchUsers(page, filter),
})

// ✅ 确保 queryKey 中的值是稳定的
const { data } = useQuery({
  queryKey: ['users', page, filter],  // 基本类型值
  queryFn: () => fetchUsers(page, filter),
})

// ❌ 不要手动管理 loading 状态
const [loading, setLoading] = useState(false)
// TanStack Query 自动管理 isLoading, isFetching 等状态
```

## 8. 工具选型总结

| 工具 | 用途 | 何时选择 |
|------|------|---------|
| `useState` | 组件局部状态 | 简单的本地状态 |
| `useReducer` | 复杂局部状态 | 有多个子值的状态逻辑 |
| `useContext` | 跨组件共享 | 主题、认证等不频繁变化的状态 |
| **Zustand** | 全局客户端状态 | UI 状态、表单临时状态 |
| **Jotai** | 细粒度原子状态 | 状态分散，派生关系复杂 |
| **Redux Toolkit** | 大型项目全局状态 | 需要严格架构约束 |
| **TanStack Query** | 服务端状态 | API 数据获取、缓存 |
| **React Hook Form** | 表单状态 | 复杂表单、验证 |
| **URL 参数** | 可分享/持久化状态 | 搜索、筛选、分页 |

---

## 总结

- **客户端状态**用 Zustand/Redux 等，**服务端状态**用 TanStack Query
- TanStack Query 管理缓存、加载状态、错误处理、重新获取
- 表单用 React Hook Form + Zod，性能好且类型安全
- URL 参数适合需要持久化、可分享的状态
- 根据状态类型选择合适的工具，不要一刀切
