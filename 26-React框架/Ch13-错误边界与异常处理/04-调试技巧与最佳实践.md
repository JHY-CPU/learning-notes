### 1.2 Components 面板

```
Components 面板功能：
  - 查看组件树结构
  - 查看/编辑 props 和 state
  - 查看 hooks 的值
  - 查看组件源码位置
  - 搜索组件
  - 查看组件的渲染时间

使用技巧：
  - 点击组件树中的组件，右侧面板显示详细信息
  - 双击 state/prop 值可以直接修改
  - 使用搜索框按组件名/props 值搜索
  - 点击源码链接跳转到组件定义
```

### 1.3 Profiler 面板

```
Profiler 面板功能：
  - 录制组件渲染过程
  - 查看每次渲染的耗时
  - Flamegraph 视图和 Ranked 视图
  - 查看为什么组件重新渲染了
  - 对比多次录制结果

使用流程：
  1. 点击录制按钮
  2. 执行要分析的操作
  3. 停止录制
  4. 分析结果
```

## 2. 组件高亮与检查

### 2.1 高亮更新

```
开启方式：
1. 打开 React DevTools
2. 勾选 "Highlight updates when components render"
3. 操作页面，观察高亮

高亮含义：
  - 绿色边框闪烁 = 组件重新渲染
  - 频繁闪烁 = 可能存在不必要的渲染
  - 大范围闪烁 = 状态提升过高

进阶设置：
  - 可以设置只在特定组件上显示高亮
  - 可以记录渲染次数
```

### 2.2 组件选择器

```
使用组件选择器：
1. 点击 DevTools 左上角的选择器图标
2. 在页面上点击任意元素
3. DevTools 自动定位到对应的 React 组件

用途：
  - 快速找到某个 UI 元素对应的组件
  - 查看组件的 props 和 state
  - 定位组件在代码中的位置
```

### 2.3 组件搜索

```
搜索方式：
  - 按组件名搜索：Header, UserCard
  - 按 props 值搜索：userId=123
  - 按 CSS 选择器搜索：.container

快捷操作：
  - Enter 跳转到下一个匹配
  - Shift+Enter 跳转到上一个匹配
```

## 3. Strict Mode 警告

### 3.1 开启 Strict Mode

```jsx
import { StrictMode } from "react";

function App() {
  return (
    <StrictMode>
      <MyApplication />
    </StrictMode>
  );
}
```

### 3.2 Strict Mode 的作用

```jsx
// Strict Mode 在开发模式下会：

// 1. 双重执行渲染函数
//    - 帮助发现渲染中的副作用
function Component() {
  console.log("渲染"); // 会打印两次
  return <div>Hello</div>;
}

// 2. 双重执行 useEffect
//    - 帮助发现缺少清理的副作用
function Component() {
  useEffect(() => {
    console.log("Effect"); // 会执行两次
    // 如果没有清理函数，第二次执行可能导致问题
    return () => console.log("Cleanup");
  }, []);

  return <div>Hello</div>;
}

// 3. 检测过时的 API 使用
//    - findDOMNode
//    - 字符串 refs
//    - componentWillMount 等旧生命周期

// 4. 检测意外的副作用
//    - 在 render 阶段调用 setState
//    - 在 render 阶段修改 DOM
```

### 3.3 常见 Strict Mode 警告

```jsx
// 警告一：componentWillMount 已废弃
// ❌
class OldComponent extends React.Component {
  componentWillMount() { ... } // 警告
}
// ✅ 使用 constructor 或 componentDidMount

// 警告二：findDOMNode 已废弃
// ❌
class OldComponent extends React.Component {
  componentDidMount() {
    findDOMNode(this); // 警告
  }
}
// ✅ 使用 ref

// 警告三：字符串 ref 已废弃
// ❌
class OldComponent extends React.Component {
  render() {
    return <div ref="myRef">Hello</div>; // 警告
  }
}
// ✅ 使用 useRef 或 createRef

// 警告四：不安全的生命周期
// ❌
class OldComponent extends React.Component {
  componentWillReceiveProps(nextProps) { ... } // 警告
  componentWillUpdate(nextProps, nextState) { ... } // 警告
}
// ✅ 使用 getDerivedStateFromProps 和 getSnapshotBeforeUpdate
```

## 4. console 调试策略

### 4.1 条件日志

```jsx
function Component() {
  const DEBUG = process.env.NODE_ENV === "development";

  useEffect(() => {
    if (DEBUG) {
      console.log("[Component] mounted");
    }
    return () => {
      if (DEBUG) {
        console.log("[Component] unmounted");
      }
    };
  }, [DEBUG]);

  return <div>Hello</div>;
}
```

### 4.2 分组日志

```jsx
function Component() {
  const [data, setData] = useState(null);

  useEffect(() => {
    console.group("🔄 数据加载");
    console.log("开始加载数据...");
    console.time("加载耗时");

    loadData()
      .then((result) => {
        console.log("数据加载成功:", result);
        console.timeEnd("加载耗时");
        console.groupEnd();
        setData(result);
      })
      .catch((error) => {
        console.error("数据加载失败:", error);
        console.timeEnd("加载耗时");
        console.groupEnd();
      });
  }, []);

  return <div>{data?.name}</div>;
}
```

### 4.3 表格日志

```jsx
function UserList({ users }) {
  useEffect(() => {
    // 以表格形式显示数据，便于查看
    console.table(users.map((u) => ({
      id: u.id,
      name: u.name,
      email: u.email,
    })));
  }, [users]);

  return <div>...</div>;
}
```

### 4.4 状态变化追踪

```jsx
function useDebugState(initialValue, label) {
  const [value, setValue] = useState(initialValue);

  const debugSetValue = useCallback((newValue) => {
    const actualValue = typeof newValue === "function"
      ? newValue(value)
      : newValue;

    console.log(`[${label}] 状态变化:`, value, "→", actualValue);
    console.trace("调用栈:");

    setValue(newValue);
  }, [value, label]);

  return [value, debugSetValue];
}

// 使用
function Counter() {
  const [count, setCount] = useDebugState(0, "Counter");
  return <button onClick={() => setCount((c) => c + 1)}>{count}</button>;
}
```

## 6. Source Maps

### 6.1 配置 Source Maps

```js
// vite.config.js
export default {
  build: {
    sourcemap: true, // 生成 source map
    // 或者只在开发环境生成
    sourcemap: process.env.NODE_ENV === "development",
  },
};

// webpack.config.js
module.exports = {
  devtool: "source-map", // 开发环境
  // devtool: "hidden-source-map", // 生产环境（不暴露给浏览器但可用于错误监控）
};
```

### 6.2 Source Map 类型

```
Source Map 类型（从快到慢，从信息少到多）：

eval             - 最快，不生成独立文件
eval-source-map  - 快速，完整的 source map
cheap-source-map - 较快，只包含行信息
cheap-module-source-map - 较快，包含 loader 处理后的行信息
source-map       - 最慢，最完整

推荐配置：
  开发环境: eval-source-map 或 eval-cheap-module-source-map
  生产环境: source-map 或 hidden-source-map
```

### 6.3 在错误监控中使用 Source Map

```js
// Sentry 配置自动上传 source map
```

## 7. useEffect 断点调试

### 7.1 Chrome DevTools 断点

```jsx
function UserProfile({ userId }) {
  const [user, setUser] = useState(null);

  useEffect(() => {
    debugger; // 在此处暂停执行，检查上下文

    fetch(`/api/users/${userId}`)
      .then((res) => res.json())
      .then((data) => {
        debugger; // 检查返回的数据
        setUser(data);
      });
  }, [userId]);

  return <div>{user?.name}</div>;
}
```

### 7.2 条件断点

```jsx
function List({ items }) {
  useEffect(() => {
    // 只在特定条件下暂停
    if (items.length > 100) {
      debugger; // 只有列表长度超过 100 时才暂停
    }
  }, [items]);

  return <div>{items.length}</div>;
}
```

### 7.3 在 DevTools 中调试 Hook

```
调试 Hooks 的步骤：
1. 打开 Chrome DevTools → Sources
2. 找到组件文件（使用 Cmd+P / Ctrl+P 搜索）
3. 在 useEffect 回调中设置断点
4. 触发组件渲染（刷新页面或操作 UI）
5. 当断点命中时：
   - 查看 "Scope" 面板中的变量
   - 在 "Console" 中执行表达式
   - 使用 "Step Over" / "Step Into" 单步执行
```

### 7.4 Logpoints（日志断点）

```
Logpoints 不会暂停执行，只在控制台输出日志：

1. 在 Sources 面板中找到代码行
2. 右键点击行号 → "Add logpoint"
3. 输入要打印的表达式，如：
   "userId changed:", userId
4. 当执行到该行时，控制台会输出日志

优势：
  - 不会中断程序执行
  - 适合调试异步代码
  - 适合调试频繁触发的代码
```

## 9. 状态变化调试

### 9.1 useReducer 调试

```jsx
function useDebugReducer(reducer, initialState, name) {
  const enhancedReducer = (state, action) => {
    console.group(`🔄 ${name} - Action: ${action.type}`);
    console.log("Previous State:", state);
    console.log("Action:", action);

    const newState = reducer(state, action);

    console.log("Next State:", newState);
    console.groupEnd();

    return newState;
  };

  return useReducer(enhancedReducer, initialState);
}

// 使用
function TodoApp() {
  const [state, dispatch] = useDebugReducer(todoReducer, initialState, "TodoApp");
  // 每次 dispatch 都会在控制台记录状态变化
  return <div>...</div>;
}
```

### 9.2 状态快照

```jsx
function useStateWithHistory(initialValue, capacity = 10) {
  const [value, setValue] = useState(initialValue);
  const historyRef = useRef([initialValue]);

  const setValueWithHistory = useCallback((newValue) => {
    setValue((prev) => {
      const actualValue = typeof newValue === "function"
        ? newValue(prev)
        : newValue;

      // 保存历史
      historyRef.current = [
        ...historyRef.current.slice(-(capacity - 1)),
        actualValue,
      ];

      return actualValue;
    });
  }, [capacity]);

  return [value, setValueWithHistory, historyRef.current];
}

// 使用
function Counter() {
  const [count, setCount, history] = useStateWithHistory(0, 5);

  return (
    <div>
      <button onClick={() => setCount((c) => c + 1)}>+1</button>
      <span>{count}</span>
      <div>历史: {history.join(" → ")}</div>
    </div>
  );
}
```

### 9.3 Context 变化追踪

```jsx
function useContextDebug(context, contextName) {
  const value = useContext(context);
  const prevValue = useRef(value);

  useEffect(() => {
    if (prevValue.current !== value) {
      console.group(`🔄 ${contextName} 变化`);
      console.log("Previous:", prevValue.current);
      console.log("Current:", value);
      console.trace("变更调用栈:");
      console.groupEnd();
      prevValue.current = value;
    }
  }, [value, contextName]);

  return value;
}

// 使用
function ThemeButton() {
  const theme = useContextDebug(ThemeContext, "Theme");
  return <button style={{ color: theme.color }}>Button</button>;
}
```

## 10. 综合调试流程

### 10.1 性能问题调试

```
步骤：
1. 打开 React DevTools Profiler
2. 录制操作过程
3. 查看 Flamegraph，找出耗时最长的组件
4. 检查不必要的重新渲染（why-did-you-render）
5. 检查 props 稳定性（useMemo/useCallback）
6. 使用 Chrome Performance 确认是否是主线程阻塞
7. 优化后再次 Profiler 录制，对比结果
```

### 10.2 Bug 调试

```
步骤：
1. 复现问题，确认触发条件
2. 打开 React DevTools，检查组件的 props/state
3. 使用 Strict Mode 检查潜在问题
4. 添加 console.log / debugger 定位问题
5. 使用 ErrorBoundary 捕获并显示错误信息
6. 修复后确认问题是否解决
```

### 10.3 内存泄漏调试

```
步骤：
1. 打开 Chrome DevTools → Memory
2. 录制堆快照（Heap Snapshot）
3. 执行操作
4. 再次录制堆快照
5. 对比两次快照，查看增长的对象
6. 检查组件卸载后是否有未清理的订阅/定时器
7. 使用 "Allocation instrumentation on timeline" 追踪分配
```

## 小结

- React DevTools 是 React 应用调试的核心工具
- Strict Mode 帮助在开发阶段发现潜在问题
- 使用条件日志和分组日志提高调试效率
- Source Maps 让生产环境的错误可追踪到源码
- useEffect 中使用 debugger / Logpoints 调试副作用
- why-did-you-render 自动检测不必要的重新渲染
- useReducer + 自定义增强函数可以记录完整的状态变化历史
- 综合使用 Chrome DevTools 和 React DevTools 进行全面调试
