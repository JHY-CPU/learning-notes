# 代码分割与懒加载

## 一、概念说明

代码分割将应用代码拆分为多个小块（chunk），按需加载，减少首屏加载体积。Vite 内置支持动态 `import()` 实现代码分割。

```js
// router/index.js - 路由懒加载
const routes = [
  {
    path: '/',
    component: () => import('../views/Home.vue') // 首屏加载
  },
  {
    path: '/about',
    component: () => import('../views/About.vue') // 访问时才加载
  },
  {
    path: '/dashboard',
    component: () => import('../views/Dashboard.vue')
  }
]
```

```vue
<script setup>
import { defineAsyncComponent } from 'vue'

// 组件懒加载
const HeavyChart = defineAsyncComponent(() =>
  import('../components/HeavyChart.vue')
)
</script>

<template>
  <Suspense>
    <template #default>
      <HeavyChart />
    </template>
    <template #fallback>
      <div>图表加载中...</div>
    </template>
  </Suspense>
</template>
```

## 二、具体用法

### 2.1 动态 import 语法

```js
// 按条件加载
async function loadModule(type) {
  const module = type === 'admin'
    ? await import('./admin.js')
    : await import('./user.js')
  return module.default
}
```

### 2.2 Vite 分块命名

```js
// vite.config.js
export default defineConfig({
  build: {
    rollupOptions: {
      output: {
        manualChunks: {
          'vendor': ['vue', 'vue-router', 'pinia'],
          'chart': ['echarts'],
          'utils': ['lodash-es', 'dayjs']
        }
      }
    }
  }
})
```

### 2.3 预加载提示

```html
<!-- 预加载关键资源 -->
<link rel="preload" href="/assets/hero.webp" as="image">
<!-- 预获取下一页资源 -->
<link rel="prefetch" href="/assets/about-xxx.js">
```

## 三、注意事项与常见陷阱

- 首屏组件不应懒加载，否则反而增加请求
- 懒加载组件需要加载状态的 fallback UI
- 分块粒度不宜过细，过多小文件增加 HTTP 请求数
