# 代码分割与懒加载

## 一、概念说明

代码分割将应用代码拆分为多个小块（chunk），按需加载，减少首屏加载体积。Vite 内置支持动态 `import()` 实现代码分割。

```js
// router/index.js - 路由懒加载
const routes = [
  {
    path: '/',
    component: () => import('../views/Home.vue') // 首屏加载
  },
  {
    path: '/about',
    component: () => import('../views/About.vue') // 访问时才加载
  },
  {
    path: '/dashboard',
    component: () => import('../views/Dashboard.vue')
  }
]
```

```vue
<script setup>
import { defineAsyncComponent } from 'vue'

// 组件懒加载
const HeavyChart = defineAsyncComponent(() =>
  import('../components/HeavyChart.vue')
)
</script>

<template>
  <Suspense>
    <template #default>
      <HeavyChart />
    </template>
    <template #fallback>
      <div>图表加载中...</div>
    </template>
  </Suspense>
</template>
```

## 二、具体用法

### 2.1 动态 import 语法

```js
// 按条件加载
async function loadModule(type) {
  const module = type === 'admin'
    ? await import('./admin.js')
    : await import('./user.js')
  return module.default
}
```

### 2.2 Vite 分块命名

```js
// vite.config.js
export default defineConfig({
  build: {
    rollupOptions: {
      output: {
        manualChunks: {
          'vendor': ['vue', 'vue-router', 'pinia'],
          'chart': ['echarts'],
          'utils': ['lodash-es', 'dayjs']
        }
      }
    }
  }
})
```

### 2.3 预加载提示

```html
<!-- 预加载关键资源 -->
<link rel="preload" href="/assets/hero.webp" as="image">
<!-- 预获取下一页资源 -->
<link rel="prefetch" href="/assets/about-xxx.js">
```

## 四、路由懒加载进阶

```js
// router/index.js
const routes = [
  {
    path: '/',
    component: () => import('../views/Home.vue')
  },
  {
    path: '/dashboard',
    component: () => import('../views/Dashboard.vue'),
    // 预加载：鼠标悬停时开始加载
    meta: { prefetch: true }
  },
  {
    // 分组：同一 chunk
    path: '/user',
    children: [
      {
        path: 'profile',
        component: () => import(
          /* webpackChunkName: "user" */
          '../views/user/Profile.vue'
        )
      },
      {
        path: 'settings',
        component: () => import(
          /* webpackChunkName: "user" */
          '../views/user/Settings.vue'
        )
      }
    ]
  }
]

// 鼠标悬停预加载
router.beforeEach((to) => {
  if (to.meta.prefetch) {
    // 触发浏览器预加载
    const link = document.createElement('link')
    link.rel = 'prefetch'
    link.href = to.meta.chunkUrl
    document.head.appendChild(link)
  }
})
```

## 五、手动分块策略

```js
// vite.config.js
export default defineConfig({
  build: {
    rollupOptions: {
      output: {
        manualChunks(id) {
          // 按依赖分块
          if (id.includes('node_modules')) {
            if (id.includes('vue')) return 'vendor-vue'
            if (id.includes('echarts')) return 'vendor-chart'
            if (id.includes('lodash')) return 'vendor-utils'
            return 'vendor'
          }
          // 按功能模块分块
          if (id.includes('/views/admin/')) return 'admin'
          if (id.includes('/views/user/')) return 'user'
        }
      }
    },
    chunkSizeWarningLimit: 500  // KB
  }
})
```

## 六、加载策略对比

| 策略 | 首屏速度 | 交互延迟 | 实现复杂度 |
|------|---------|---------|-----------|
| 全量加载 | 慢 | 无 | 低 |
| 路由懒加载 | 快 | 路由切换时 | 低 |
| 组件懒加载 | 快 | 首次交互时 | 中 |
| 预加载 + 懒加载 | 快 | 几乎无 | 中 |
| 流式 SSR | 最快 | 无 | 高 |

## 三、注意事项与常见陷阱

- 首屏组件不应懒加载，否则反而增加请求
- 懒加载组件需要加载状态的 fallback UI
- 分块粒度不宜过细，过多小文件增加 HTTP 请求数
- `manualChunks` 的分块策略要结合业务路由设计
- 预加载（prefetch）会消耗带宽，移动端需谨慎使用
