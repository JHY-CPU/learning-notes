# 避免不必要渲染

## 一、概念说明

不必要的组件重新渲染是性能问题的常见原因。通过合理的 key 设计、组件拆分和响应式优化来减少无效渲染。

```vue
<script setup>
import { ref, shallowRef, markRaw } from 'vue'

// ❌ 深度响应式大对象（每次属性访问都创建代理）
const bigData = ref({ /* 1000 个属性 */ })

// ✅ 浅层响应式（只追踪顶层变化）
const bigList = shallowRef([{ /* 大量数据 */ }])

// ✅ 标记为非响应式
const staticConfig = markRaw({
  columns: [/* 列配置 */],
  formatters: { /* 格式化函数 */ }
})
</script>
```

## 二、具体用法

### 2.1 key 的正确使用

```vue
<!-- ❌ 用 index 作为 key，列表变化时导致错误复用 -->
<li v-for="(item, index) in list" :key="index">{{ item.name }}</li>

<!-- ✅ 用唯一 id 作为 key -->
<li v-for="item in list" :key="item.id">{{ item.name }}</li>
```

### 2.2 组件拆分

```vue
<!-- ❌ 整个页面是一个大组件，任何数据变化都重新渲染整个页面 -->
<template>
  <header>...</header>
  <Sidebar />
  <main>
    <table><!-- 100 行数据 --></table>
  </main>
  <Footer />
</template>

<!-- ✅ 拆分为独立组件，各自独立更新 -->
<template>
  <AppHeader />
  <Sidebar />
  <DataTable :data="items" />
  <AppFooter />
</template>
```

### 2.3 v-show vs v-if

```vue
<!-- 频繁切换用 v-show（只切换 CSS display） -->
<Tab v-show="activeTab === 'a'">内容 A</Tab>
<Tab v-show="activeTab === 'b'">内容 B</Tab>

<!-- 条件渲染用 v-if（不渲染就不创建 DOM） -->
<HeavyComponent v-if="needHeavy" />
```

### 2.4 函数式组件

```js
// 无状态、无副作用的纯展示组件
const Label = (props, { slots }) => (
  <span class="label">{slots.default()}</span>
)
```

## 三、注意事项与常见陷阱

- `shallowRef` 和 `shallowReactive` 适合大型数据结构
- `markRaw` 适合永远不会变的配置对象
- 不要用 `JSON.parse(JSON.stringify(obj))` 来"断开"响应式，性能极差
- 组件拆分不是越细越好，平衡可维护性和性能
