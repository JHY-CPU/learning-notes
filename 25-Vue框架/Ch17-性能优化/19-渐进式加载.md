# 渐进式加载

## 一、概念说明

渐进式加载指分批次获取和渲染数据，而非一次性加载全部内容。常见方案有分页加载和无限滚动（Infinite Scroll）。

```vue
<script setup>
import { ref, onMounted, onUnmounted } from 'vue'
import axios from 'axios'

const items = ref([])
const page = ref(1)
const loading = ref(false)
const hasMore = ref(true)

async function loadMore() {
  if (loading.value || !hasMore.value) return
  loading.value = true

  const { data } = await axios.get('/api/items', {
    params: { page: page.value, size: 20 }
  })

  items.value.push(...data.list)
  hasMore.value = data.list.length === 20
  page.value++
  loading.value = false
}

// 无限滚动
const sentinel = ref(null)

onMounted(() => {
  const observer = new IntersectionObserver(([entry]) => {
    if (entry.isIntersecting) loadMore()
  }, { rootMargin: '200px' })

  observer.observe(sentinel.value)
  onUnmounted(() => observer.disconnect())
})
</script>

<template>
  <div v-for="item in items" :key="item.id" class="item">
    {{ item.name }}
  </div>
  <div ref="sentinel" class="sentinel">
    <span v-if="loading">加载中...</span>
    <span v-else-if="!hasMore">没有更多了</span>
  </div>
</template>
```

## 二、具体用法

### 2.1 分页加载

```vue
<template>
  <div v-for="item in items" :key="item.id">{{ item.name }}</div>
  <Pagination
    :current="page"
    :total="total"
    @change="(p) => { page = p; fetchItems() }"
  />
</template>
```

### 2.2 无限滚动

```js
// 基于 scroll 事件
function onScroll() {
  const { scrollTop, scrollHeight, clientHeight } = document.documentElement
  if (scrollTop + clientHeight >= scrollHeight - 100) {
    loadMore()
  }
}
```

### 2.3 虚拟列表 + 无限滚动

```vue
<!-- 大量数据的最优方案 -->
<RecycleScroller
  :items="items"
  :item-size="60"
  @scroll-end="loadMore"
  v-slot="{ item }"
>
  <ItemCard :data="item" />
</RecycleScroller>
```

### 2.4 分页 vs 无限滚动

| 特性 | 分页 | 无限滚动 |
|------|------|---------|
| 适合场景 | 数据表格、搜索结果 | 信息流、社交媒体 |
| 跳转 | 支持跳转到指定页 | 不支持 |
| 内存 | 只保留当前页 | 累积所有加载的 |
| SEO | 友好 | 较差 |

## 三、注意事项与常见陷阱

- 无限滚动需要处理重复加载的竞态问题
- Intersection Observer 比 scroll 事件性能更好
- 长列表要配合虚拟滚动，否则 DOM 过多会卡顿
