# 计算属性缓存

## 一、概念说明

Vue 的 `computed` 具有缓存机制：只有当依赖的响应式数据变化时才会重新计算。相比方法调用，计算属性避免了不必要的重复计算。

```vue
<script setup>
import { ref, computed } from 'vue'

const items = ref([
  { name: '苹果', price: 8, count: 3 },
  { name: '香蕉', price: 5, count: 5 },
  { name: '橙子', price: 6, count: 2 },
])

// 计算属性：只在 items 变化时重新计算
const totalPrice = computed(() => {
  console.log('计算总价...') // 仅在 items 变化时输出
  return items.value.reduce((sum, item) => sum + item.price * item.count, 0)
})

const expensiveItems = computed(() => {
  return items.value.filter(item => item.price * item.count > 10)
})
</script>

<template>
  <ul>
    <li v-for="item in items" :key="item.name">
      {{ item.name }} - 单价 {{ item.price }} x {{ item.count }}
    </li>
  </ul>
  <p>总价: {{ totalPrice }}</p>  <!-- 读取缓存，不重新计算 -->
  <p>总价: {{ totalPrice }}</p>  <!-- 多次使用仍只计算一次 -->
  <p>高价商品: {{ expensiveItems.length }} 件</p>
</template>
```

## 二、具体用法

### 2.1 computed vs method

```js
// method：每次渲染都重新调用
function totalPrice() {
  return items.value.reduce((sum, item) => sum + item.price * item.count, 0)
}
// {{ totalPrice() }} 每次渲染调用
// {{ totalPrice() }} 同一渲染再次调用

// computed：依赖不变就不重新计算
const totalPrice = computed(() => {
  return items.value.reduce((sum, item) => sum + item.price * item.count, 0)
})
// {{ totalPrice }} 读取缓存
// {{ totalPrice }} 仍然读取缓存
```

### 2.2 可写计算属性

```js
const fullName = computed({
  get() { return `${firstName.value} ${lastName.value}` },
  set(value) {
    const [first, last] = value.split(' ')
    firstName.value = first
    lastName.value = last
  }
})
```

### 2.3 computed 链式依赖

```js
const filteredList = computed(() => list.value.filter(item => item.active))
const sortedList = computed(() => [...filteredList.value].sort((a, b) => a.name.localeCompare(b.name)))
// sortedList 依赖 filteredList，filteredList 变化时 sortedList 也会更新
```

## 三、注意事项与常见陷阱

- 计算属性应是**纯函数**，不要产生副作用
- 不要在计算属性中执行异步操作
- 复杂计算如果不需要缓存，使用 `method` 或 `watch` 更合适
- 计算属性的依赖必须是响应式的
