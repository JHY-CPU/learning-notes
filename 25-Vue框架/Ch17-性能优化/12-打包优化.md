# 打包优化

## 一、概念说明

打包优化旨在减小最终产物的体积，加速下载和解析。Vite 基于 Rollup 提供了强大的打包优化能力。

```js
// vite.config.js
import { defineConfig } from 'vite'
import { visualizer } from 'rollup-plugin-visualizer'
import compression from 'vite-plugin-compression'

export default defineConfig({
  build: {
    // 启用 CSS 代码分割
    cssCodeSplit: true,

    // 压缩配置
    minify: 'terser',
    terserOptions: {
      compress: {
        drop_console: true,  // 移除 console.log
        drop_debugger: true  // 移除 debugger
      }
    },

    // 分块策略
    rollupOptions: {
      output: {
        manualChunks: {
          'vue-vendor': ['vue', 'vue-router', 'pinia'],
          'ui-lib': ['element-plus']
        }
      }
    }
  },

  plugins: [
    // Gzip 压缩
    compression({ algorithm: 'gzip' }),
    // Brotli 压缩
    compression({ algorithm: 'brotliCompress' }),
    // 分析工具
    visualizer({ open: true })
  ]
})
```

## 二、具体用法

### 2.1 分析包大小

```bash
# 安装分析工具
npm install -D rollup-plugin-visualizer

# 构建后自动打开可视化报告
npm run build
```

### 2.2 分割策略

```js
rollupOptions: {
  output: {
    // 按库分割
    manualChunks(id) {
      if (id.includes('node_modules')) {
        if (id.includes('echarts')) return 'chart'
        if (id.includes('lodash')) return 'utils'
        return 'vendor'
      }
    }
  }
}
```


## 三、注意事项与常见陷阱

- `drop_console` 会移除所有 console，调试信息需用其他方式保留
- 分块粒度不宜过细，过多小文件增加 HTTP 请求数
- 生产环境使用 `terser` 压缩，开发环境不需要
- 分析报告只代表开发时的大小，gzip 后通常小 60-80%
