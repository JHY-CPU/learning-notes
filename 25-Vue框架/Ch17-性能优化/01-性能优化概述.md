# 性能优化概述

## 一、概念说明

Vue 应用的性能优化分为两大方向：**加载性能**（首屏速度、资源体积）和**运行时性能**（渲染效率、响应速度）。通过合理优化，可以显著提升用户体验。

```vue
<script setup>
// 性能指标关注点：
// - FCP（First Contentful Paint）：首次内容绘制
// - LCP（Largest Contentful Paint）：最大内容绘制
// - TTI（Time to Interactive）：可交互时间
// - CLS（Cumulative Layout Shift）：累积布局偏移
</script>
```

## 二、具体用法

### 2.1 优化方向总览

| 方向 | 优化手段 | 影响指标 |
|------|---------|---------|
| 代码体积 | Tree Shaking、代码分割 | FCP、LCP |
| 加载策略 | 懒加载、预加载 | FCP、TTI |
| 渲染效率 | 虚拟滚动、缓存 | TTI、FPS |
| 资源优化 | 图片压缩、CDN | LCP |
| 运行时 | 防抖节流、Web Worker | FPS、响应速度 |

### 2.2 性能检测工具

- **Chrome DevTools Performance**：运行时性能分析
- **Lighthouse**：综合性能评分
- **Vue DevTools**：组件渲染追踪
- **webpack-bundle-analyzer**：打包体积分析

### 2.3 关键原则

```js
// 1. 减少不必要的响应式
import { shallowRef, markRaw } from 'vue'
const largeList = shallowRef([]) // 浅层响应式
const immutableConfig = markRaw(config) // 完全跳过响应式

// 2. 计算属性缓存
const filtered = computed(() => list.value.filter(/* ... */))

// 3. 组件懒加载
const HeavyComponent = defineAsyncComponent(() => import('./Heavy.vue'))
```

## 三、注意事项与常见陷阱

- 优化前先测量，不要凭感觉优化
- 关注"关键路径"上的瓶颈，不要过度优化无关环节
- 用户可感知的优化（如骨架屏）比底层优化更重要
- 不同设备和网络条件下表现可能差异很大

## 四、加载性能优化清单

```
1. 代码分割（dynamic import）
2. 路由懒加载
3. 组件异步加载（defineAsyncComponent）
4. Tree Shaking（确保 ESM 导出）
5. 压缩（gzip/brotli）
6. CDN 加载第三方库
7. 图片优化（WebP、懒加载、响应式图片）
8. 字体子集化
9. 预加载关键资源（<link rel="preload">）
10. HTTP/2 多路复用
```

## 五、运行时性能优化清单

```
1. v-memo 缓存大列表渲染
2. v-once 缓存静态内容
3. computed 缓存派生数据
4. shallowRef/shallowReactive 避免深度代理
5. markRaw 跳过不可变数据的响应式
6. 虚拟滚动（大列表）
7. 防抖/节流（输入框、滚动事件）
8. Web Worker 处理耗时计算
9. keep-alive 缓存组件状态
10. 避免不必要的组件重新渲染
```

## 六、常见性能瓶颈

| 瓶颈 | 表现 | 解决方案 |
|------|------|---------|
| JS 包过大 | 首屏白屏时间长 | 代码分割、懒加载 |
| 大列表渲染 | 滚动卡顿 | 虚拟滚动 |
| 频繁 DOM 更新 | 交互卡顿 | v-memo、防抖 |
| 内存泄漏 | 页面越用越慢 | 及时清理事件监听、定时器 |
| 深度响应式 | 大数据操作慢 | shallowRef、markRaw |
