# 运行时性能

## 一、概念说明

运行时性能关注的是应用在用户设备上的渲染效率和响应速度。核心是减少不必要的 DOM 操作和计算。

```vue
<script setup>
import { ref, nextTick, shallowRef } from 'vue'

const list = shallowRef([]) // 大列表用 shallowRef

// 批量更新：将多次赋值合并为一次
async function batchUpdate() {
  // ❌ 每次赋值都触发渲染
  // list.value = [...list.value, item1]
  // list.value = [...list.value, item2]

  // ✅ 一次赋值
  list.value = [...list.value, item1, item2, item3]

  // ✅ 使用 nextTick 确保 DOM 更新完成
  await nextTick()
  // DOM 已更新，可以操作 DOM
}

// 减少不必要的 ref 解构
const { value: items } = ref([])
</script>
```

## 二、具体用法

### 2.1 减少 DOM 操作

```vue
<!-- ❌ 频繁切换导致 DOM 反复创建销毁 -->
<div v-if="tab === 'a'">A 内容</div>
<div v-if="tab === 'b'">B 内容</div>
<div v-if="tab === 'c'">C 内容</div>

<!-- ✅ 使用 v-show 避免 DOM 重建 -->
<div v-show="tab === 'a'">A 内容</div>
<div v-show="tab === 'b'">B 内容</div>
<div v-show="tab === 'c'">C 内容</div>
```

### 2.2 nextTick 时机控制

```js
// 数据更新后，DOM 可能还未更新
count.value++
// 此时 DOM 中还是旧值
console.log(el.textContent) // 旧值

await nextTick()
// 此时 DOM 已更新
console.log(el.textContent) // 新值
```

### 2.3 批量处理

```js
// 使用 shallowRef + 整体替换减少代理开销
const data = shallowRef([])

function addItems(newItems) {
  // 整体替换，只触发一次更新
  data.value = [...data.value, ...newItems]
}
```

## 四、虚拟滚动

```vue
<script setup>
import { ref, computed } from 'vue'

const items = ref(Array.from({ length: 10000 }, (_, i) => ({ id: i, text: `Item ${i}` })))
const containerRef = ref(null)
const scrollTop = ref(0)
const itemHeight = 40
const visibleCount = 20

const startIndex = computed(() => Math.floor(scrollTop.value / itemHeight))
const endIndex = computed(() => Math.min(startIndex.value + visibleCount, items.value.length))
const visibleItems = computed(() => items.value.slice(startIndex.value, endIndex.value))
const offsetY = computed(() => startIndex.value * itemHeight)

const onScroll = (e) => {
  scrollTop.value = e.target.scrollTop
}
</script>

<template>
  <div ref="containerRef" class="virtual-list" @scroll="onScroll" style="height: 800px; overflow-y: auto;">
    <div :style="{ height: items.length * itemHeight + 'px', position: 'relative' }">
      <div :style="{ transform: `translateY(${offsetY}px)` }">
        <div v-for="item in visibleItems" :key="item.id" class="list-item" :style="{ height: itemHeight + 'px' }">
          {{ item.text }}
        </div>
      </div>
    </div>
  </div>
</template>
```

## 五、Web Worker 处理重计算

```js
// worker.js
self.onmessage = (e) => {
  const { type, data } = e.data
  if (type === 'sort') {
    const sorted = data.sort((a, b) => a.value - b.value)
    self.postMessage({ type: 'sorted', data: sorted })
  }
}

// 组件中
const worker = new Worker(new URL('./worker.js', import.meta.url))
worker.postMessage({ type: 'sort', data: largeArray })
worker.onmessage = (e) => {
  result.value = e.data.data
}
```

## 六、性能优化检查清单

```
响应式层面：
  ✓ 大对象/数组使用 shallowRef / shallowReactive
  ✓ 不需要响应的数据使用 markRaw
  ✓ computed 中避免昂贵计算（考虑缓存）
  ✓ 批量更新合并为一次赋值

DOM 层面：
  ✓ 长列表使用虚拟滚动
  ✓ 频繁切换使用 v-show 而非 v-if
  ✓ 静态内容使用 v-once
  ✓ 条件缓存使用 v-memo

计算层面：
  ✓ 重计算放入 Web Worker
  ✓ 防抖/节流处理高频操作
  ✓ 使用 nextTick 确保 DOM 更新后再操作

内存层面：
  ✓ 组件卸载清理定时器、事件监听、Worker
  ✓ 事件监听使用 { passive: true } 提升滚动性能
  ✓ 避免闭包导致的内存泄漏
```

## 三、注意事项与常见陷阱

- `shallowRef` 只追踪 `.value` 的替换，不追踪内部属性变化
- `nextTick` 只保证 Vue 的 DOM 更新完成，不保证浏览器绘制完成
- 复杂计算放在 Web Worker 中，不阻塞主线程
- 虚拟滚动库（vue-virtual-scroller、vue-virtual-scroll-grid）比手写更稳定
- `{ passive: true }` 事件监听器不能调用 `preventDefault()`
