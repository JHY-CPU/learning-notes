# 性能优化清单

## 一、概念说明

这是一份 Vue 应用性能优化的完整检查清单，覆盖从开发到部署的各个环节。

## 二、优化检查清单

### 2.1 加载性能

- [ ] 路由和组件使用懒加载（`defineAsyncComponent`、动态 `import()`）
- [ ] 第三方库按需导入（`lodash-es`、Element Plus 按需引入）
- [ ] 图片使用 WebP/AVIF 格式，设置 `width`/`height`
- [ ] 首屏图片使用 `<link rel="preload">`
- [ ] 非关键 CSS 使用 `<link rel="preload" as="style">`
- [ ] 配置 CDN 分发静态资源
- [ ] 启用 Gzip / Brotli 压缩
- [ ] 使用 HTTP/2 协议
- [ ] 骨架屏或 loading 占位

### 2.2 打包优化

- [ ] 分析包大小，移除无用依赖
- [ ] 配置 `manualChunks` 分割大库
- [ ] 启用 Tree Shaking（使用 ESM 语法）
- [ ] 移除 `console.log` 和 `debugger`
- [ ] 配置 `sideEffects` 排除副作用文件
- [ ] 小资源内联为 base64（`assetsInlineLimit`）

### 2.3 运行时性能

- [ ] 大列表使用虚拟滚动
- [ ] 频繁变化的数据使用 `shallowRef`
- [ ] 不变的配置对象使用 `markRaw`
- [ ] 高频事件使用防抖/节流
- [ ] 计算密集任务使用 Web Worker
- [ ] 避免用 `index` 作为 `v-for` 的 `key`
- [ ] 频繁切换用 `v-show`，条件渲染用 `v-if`

### 2.4 缓存策略

- [ ] 合理使用 `<KeepAlive>` 缓存组件
- [ ] HTTP 资源配置 `Cache-Control`
- [ ] API 响应使用客户端缓存（SWR）
- [ ] Service Worker 离线缓存

### 2.5 内存管理

- [ ] 组件卸载清理定时器、事件监听器
- [ ] AbortController 取消未完成的请求
- [ ] 第三方库实例调用 `dispose()` 销毁
- [ ] 不再需要的 observer 调用 `disconnect()`

### 2.6 监控与度量

- [ ] 集成 Web Vitals 监控（FCP、LCP、CLS、FID）
- [ ] 建立性能基线，CI 中检查性能退化
- [ ] 使用 Lighthouse 定期审计
- [ ] 真实用户数据（RUM）收集

## 三、优先级建议

| 优先级 | 优化项 | 收益 |
|--------|-------|------|
| P0 | 代码分割、懒加载 | 大幅减少首屏体积 |
| P0 | 图片优化 | 大幅减少带宽消耗 |
| P1 | Tree Shaking、压缩 | 减少总体积 |
| P1 | 虚拟滚动大列表 | 解决卡顿问题 |
| P2 | 防抖节流 | 提升交互流畅度 |
| P2 | KeepAlive 缓存 | 避免重复渲染 |
| P3 | Web Worker | 处理极端计算场景 |

## 四、注意事项与常见陷阱

- 优化前先测量，用数据驱动决策
- 不要过度优化，关注用户可感知的瓶颈
- 不同设备和网络条件下表现差异很大，需覆盖测试
- 性能优化是持续过程，不是一次性工作
