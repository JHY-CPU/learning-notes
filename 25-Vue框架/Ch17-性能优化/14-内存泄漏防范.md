# 内存泄漏防范

## 一、概念说明

内存泄漏是指不再使用的内存没有被释放，长期累积会导致页面变慢甚至崩溃。Vue 应用中常见泄漏源包括：未清理的定时器、事件监听器、第三方库实例。

```vue
<script setup>
import { ref, onMounted, onUnmounted } from 'vue'

const time = ref('')
let timer = null

// ✅ 正确：组件卸载时清理定时器
onMounted(() => {
  timer = setInterval(() => {
    time.value = new Date().toLocaleTimeString()
  }, 1000)
})

onUnmounted(() => {
  clearInterval(timer)
})

// ✅ 正确：清理事件监听器
function handleResize() {
  console.log(window.innerWidth)
}

onMounted(() => {
  window.addEventListener('resize', handleResize)
})

onUnmounted(() => {
  window.removeEventListener('resize', handleResize)
})
</script>
```

## 二、具体用法

### 2.1 常见泄漏场景

```vue
<script setup>
import { onUnmounted } from 'vue'

// 定时器
const timer = setInterval(fn, 1000)
onUnmounted(() => clearInterval(timer))

// 事件监听器
window.addEventListener('scroll', handler)
onUnmounted(() => window.removeEventListener('scroll', handler))

// AbortController（用于 fetch）
const controller = new AbortController()
fetch('/api', { signal: controller.signal })
onUnmounted(() => controller.abort())

// 第三方库实例
const chart = echarts.init(el)
onUnmounted(() => chart.dispose())

// 观察者
const observer = new IntersectionObserver(callback)
observer.observe(el)
onUnmounted(() => observer.disconnect())
</script>
```

### 2.2 封装清理逻辑

```js
// composables/useInterval.js
import { ref, onUnmounted } from 'vue'

export function useInterval(fn, delay) {
  const timer = ref(null)
  function start() { timer.value = setInterval(fn, delay) }
  function stop() { clearInterval(timer.value) }
  onUnmounted(stop)
  return { start, stop }
}
```

## 三、注意事项与常见陷阱

- `onUnmounted` 中清理所有副作用是最重要的习惯
- 全局事件（`window`、`document`）最容易遗漏清理
- `watch` 和 `watchEffect` 返回停止函数，需要时手动调用
- Vue 3 中组件卸载会自动清理响应式副作用，但不会清理 DOM 事件
