# 组件懒加载

## 一、概念说明

使用 `defineAsyncComponent` 将非首屏组件延迟到需要时再加载，减少初始包体积。特别适合弹窗、复杂图表、富文本编辑器等重量级组件。

```vue
<script setup>
import { defineAsyncComponent, ref } from 'vue'

// 基本用法
const HeavyEditor = defineAsyncComponent(() =>
  import('../components/RichEditor.vue')
)

// 带加载和错误状态
const DataChart = defineAsyncComponent({
  loader: () => import('../components/DataChart.vue'),
  loadingComponent: { template: '<div>图表加载中...</div>' },
  errorComponent: { template: '<div>加载失败</div>' },
  delay: 200,       // 延迟显示 loading 组件（ms）
  timeout: 10000,   // 超时时间
  suspensible: true // 支持 <Suspense>
})

const showEditor = ref(false)
</script>

<template>
  <button @click="showEditor = true">打开编辑器</button>
  <HeavyEditor v-if="showEditor" />
</template>
```

## 二、具体用法

### 2.1 配合 Suspense

```vue
<Suspense>
  <template #default>
    <AsyncComponent />
  </template>
  <template #fallback>
    <SkeletonLoader />
  </template>
</Suspense>
```

### 2.2 条件懒加载

```vue
<script setup>
import { defineAsyncComponent, shallowRef } from 'vue'

const isAdmin = shallowRef(false)

// 只有管理员才加载管理面板
const AdminPanel = defineAsyncComponent(() =>
  import('../components/AdminPanel.vue')
)
</script>

<template>
  <AdminPanel v-if="isAdmin" />
</template>
```

### 2.3 按路由懒加载组件

```js
// 在路由 meta 中标记需要懒加载的侧边栏组件
{
  path: '/analytics',
  component: () => import('../views/Analytics.vue'),
  meta: { sidebar: () => import('../components/AnalyticsSidebar.vue') }
}
```

## 四、重试机制

```vue
<script setup>
import { defineAsyncComponent } from 'vue'

// 加载失败后自动重试
function loadWithRetry(url, retries = 3) {
  return import(url).catch((error) => {
    if (retries <= 0) throw error
    console.log(`加载失败，${retries} 次重试...`)
    return new Promise((resolve) => {
      setTimeout(() => {
        resolve(loadWithRetry(url, retries - 1))
      }, 1000)
    })
  })
}

const AsyncChart = defineAsyncComponent(() =>
  loadWithRetry('./components/Chart.vue')
)
</script>
```

## 五、按需加载与预加载结合

```vue
<script setup>
import { defineAsyncComponent, onMounted } from 'vue'

// 条件懒加载：用户悬停时预加载
const Editor = defineAsyncComponent(() =>
  import('./components/RichEditor.vue')
)

let editorLoaded = false
function preloadEditor() {
  if (!editorLoaded) {
    import('./components/RichEditor.vue')
    editorLoaded = true
  }
}
</script>

<template>
  <button @mouseenter="preloadEditor" @click="showEditor = true">
    打开编辑器
  </button>
  <Editor v-if="showEditor" />
</template>
```

## 六、懒加载状态管理

```js
// composables/useAsyncComponent.js
import { ref, defineAsyncComponent } from 'vue'

export function useAsyncComponent(loader) {
  const loading = ref(false)
  const error = ref(null)
  const component = ref(null)

  const load = async () => {
    loading.value = true
    error.value = null
    try {
      const mod = await loader()
      component.value = mod.default || mod
    } catch (e) {
      error.value = e
    } finally {
      loading.value = false
    }
  }

  const AsyncComp = defineAsyncComponent({
    loader: load,
    loadingComponent: { /* ... */ },
    errorComponent: { /* ... */ },
    delay: 200,
    timeout: 10000
  })

  return { component: AsyncComp, loading, error }
}
```

## 三、注意事项与常见陷阱

- 首屏必须展示的组件不要懒加载
- 懒加载组件需要考虑加载失败的降级处理
- `delay` 设得太短会导致加载状态闪烁
- `defineAsyncComponent` 的工厂函数返回 Promise
- 预加载策略可以平衡懒加载的首次交互延迟
- 重试机制避免用户因网络波动看到加载失败
