# 首屏加载优化

## 一、概念说明

首屏加载时间直接影响用户留存。优化策略包括：减少关键资源体积、延迟非关键资源、展示加载状态等。

```vue
<!-- index.html - 骨架屏 -->
<div id="app">
  <div class="skeleton">
    <div class="skeleton-header"></div>
    <div class="skeleton-content">
      <div class="skeleton-line"></div>
      <div class="skeleton-line short"></div>
    </div>
  </div>
</div>

<style>
.skeleton-header { height: 60px; background: #f0f0f0; }
.skeleton-line {
  height: 16px; margin: 8px 0; background: #f0f0f0;
  animation: pulse 1.5s infinite;
}
.skeleton-line.short { width: 60%; }
@keyframes pulse {
  0%, 100% { opacity: 1; }
  50% { opacity: 0.4; }
}
</style>
```

```js
// main.js - 关键资源预加载
import { createApp } from 'vue'
import App from './App.vue'

// 预加载关键路由
import('./views/Home.vue')

const app = createApp(App)
app.mount('#app')
```

## 二、具体用法

### 2.1 关键资源优化

```html
<!-- 预加载关键 CSS -->
<link rel="preload" href="/critical.css" as="style">
<!-- 预连接 API 服务器 -->
<link rel="preconnect" href="https://api.example.com">
<!-- DNS 预解析 -->
<link rel="dns-prefetch" href="https://cdn.example.com">
```

### 2.3 HTTP 优化

```
- HTTP/2 多路复用
- Brotli / Gzip 压缩
- CDN 分发静态资源
- Cache-Control 缓存策略
```

### 2.4 渐进式加载

```vue
<script setup>
import { ref, onMounted } from 'vue'
const content = ref(null)

onMounted(() => {
  // 先渲染骨架屏，空闲时加载真实内容
  requestIdleCallback(() => {
    loadContent()
  })
})
</script>
```

## 三、注意事项与常见陷阱

- 骨架屏布局要和真实内容尽量接近
- 首屏不需要的 JS 使用 `defer` 或 `async` 属性
- 关键渲染路径上的 CSS 要内联，不要外部引入
- 使用 `preconnect` 提前建立服务器连接
