# 性能监控

## 一、概念说明

性能监控帮助开发者持续跟踪应用性能指标，及时发现和定位性能问题。工具包括浏览器 DevTools、Lighthouse 和 Web Vitals API。

```vue
<script setup>
import { onMounted } from 'vue'

onMounted(() => {
  // 使用 Performance API 测量
  performance.mark('app-mount-start')

  // 测量关键操作耗时
  function measureRender(label, fn) {
    performance.mark(`${label}-start`)
    fn()
    performance.mark(`${label}-end`)
    performance.measure(label, `${label}-start`, `${label}-end`)
  }

  // 获取 Web Vitals
  new PerformanceObserver((list) => {
    for (const entry of list.getEntries()) {
      if (entry.entryType === 'largest-contentful-paint') {
        console.log('LCP:', entry.startTime)
      }
    }
  }).observe({ entryTypes: ['largest-contentful-paint'] })
})
</script>
```

## 二、具体用法

### 2.1 核心指标

| 指标 | 全称 | 目标值 | 说明 |
|------|------|-------|------|
| FCP | First Contentful Paint | < 1.8s | 首次内容绘制 |
| LCP | Largest Contentful Paint | < 2.5s | 最大内容绘制 |
| FID | First Input Delay | < 100ms | 首次输入延迟 |
| CLS | Cumulative Layout Shift | < 0.1 | 累积布局偏移 |

### 2.2 使用 web-vitals 库

```bash
npm install web-vitals
```

```js
import { onLCP, onFCP, onCLS, onINP } from 'web-vitals'

onFCP(console.log)
onLCP(console.log)
onCLS(console.log)
onINP(console.log)
```

### 2.3 Chrome DevTools Performance

1. 打开 DevTools → Performance 面板
2. 点击录制按钮
3. 操作页面
4. 停止录制，分析火焰图

### 2.4 Lighthouse 审计

```bash
# 命令行运行
npx lighthouse https://your-site.com --view

# 或在 Chrome DevTools → Lighthouse 面板
```

## 三、注意事项与常见陷阱

- 性能数据要在真实设备和网络条件下收集
- 不同浏览器的性能指标可能有差异
- 监控代码本身也有性能开销，不要过于频繁
- 建立性能基线，对比每次发布的性能变化
