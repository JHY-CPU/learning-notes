# 包管理器选择

## 一、概念说明

包管理器用于管理项目依赖（第三方库）。Node.js 生态主要有三个包管理器：**npm**（随 Node.js 自带）、**yarn**（Facebook 出品）、**pnpm**（高效、节省空间）。

推荐使用 **pnpm**，它通过硬链接和内容寻址存储，避免重复下载相同的包，安装速度更快，磁盘占用更小。

## 二、具体用法

### 2.1 常用命令对比

```bash
# 初始化项目
npm init -y          yarn init            pnpm init

# 安装依赖
npm install vue      yarn add vue         pnpm add vue

# 安装开发依赖
npm i -D vite        yarn add -D vite     pnpm add -D vite

# 安装全部依赖
npm install          yarn install         pnpm install

# 删除依赖
npm uninstall vue    yarn remove vue      pnpm remove vue

# 运行脚本
npm run dev          yarn dev             pnpm dev
```

### 2.2 lock 文件

```bash
# 各包管理器的 lock 文件（必须提交到 Git）
package-lock.json    # npm
yarn.lock            # yarn
pnpm-lock.yaml       # pnpm
```

### 2.3 pnpm 优势详解

```bash
# pnpm 使用内容寻址存储
# 相同版本的包只存储一份，通过硬链接共享
pnpm store status    # 查看存储状态
pnpm store prune     # 清理未使用的包
```

```
磁盘占用对比（以一个中型项目为例）：
npm:    ~500 MB
yarn:   ~480 MB
pnpm:   ~200 MB（硬链接共享）
```

### 2.4 pnpm 特殊配置

```ini
# .npmrc
# 某些包不兼容 pnpm 的严格依赖结构
shamefully-hoist=true

# 使用国内镜像加速
registry=https://registry.npmmirror.com
```

### 2.5 Corepack（Node.js 内置）

```bash
# Node.js 16.13+ 内置 Corepack，可统一包管理器版本
corepack enable
corepack prepare pnpm@latest --activate

# package.json 中指定包管理器
{
  "packageManager": "pnpm@8.15.0"
}
```

## 三、常见用例

### 3.1 选择建议

| 场景 | 推荐 |
|------|------|
| 个人/小项目 | pnpm |
| 企业/大型 monorepo | pnpm workspace |
| 遗留项目（已有 yarn.lock） | 保持 yarn |
| 纯前端学习 | npm（最简单） |

### 3.2 pnpm workspace（monorepo）

```yaml
# pnpm-workspace.yaml
packages:
  - 'packages/*'
  - 'apps/*'
```

```bash
# 在 monorepo 中安装依赖
pnpm add lodash --filter @my-app/core    # 指定子项目
pnpm add lodash -w                        # 安装到根目录
```

## 四、注意事项与常见陷阱

- lock 文件必须提交到 Git，确保团队依赖一致
- 不要混用不同包管理器，会导致 lock 文件冲突
- pnpm 的 `node_modules` 结构是扁平化的，某些包可能需要 `shamefully-hoist=true`
- yarn 1.x（Classic）和 yarn 3+（Berry）差异较大
- npm 的 v7+ 支持 `package-lock.json` 的 `lockfileVersion: 3`
- 使用 `corepack` 可确保团队成员使用相同版本的包管理器
- `pnpm install` 默认使用 `--frozen-lockfile` 行为（CI 环境中常用）
