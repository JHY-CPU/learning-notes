# 服务端渲染概念

## 一、概念说明

**服务端渲染**（Server-Side Rendering，SSR）指在服务器端将 Vue 组件渲染为 HTML 字符串，直接发送给浏览器。相比客户端渲染（CSR），SSR 能提供更快的首屏加载和更好的 SEO。

SSR 的流程：服务器接收请求 → 运行 Vue 渲染组件为 HTML → 发送完整 HTML 到浏览器 → 浏览器显示内容 → 客户端"激活"（hydration）使页面可交互。

## 二、具体用法

### 2.1 Nuxt.js 框架

Nuxt 是 Vue 生态中最成熟的 SSR 框架，开箱即用：

```bash
pnpm dlx nuxi@latest init my-nuxt-app
```

```vue
<!-- pages/index.vue - Nuxt 自动路由 -->
<script setup>
const { data } = await useFetch('/api/posts')
</script>

<template>
  <div>
    <h1>文章列表</h1>
    <article v-for="post in data" :key="post.id">
      <h2>{{ post.title }}</h2>
      <p>{{ post.summary }}</p>
    </article>
  </div>
</template>
```

### 2.2 CSR vs SSR vs SSG

| 渲染方式 | 全称 | 适用场景 | 首屏速度 | SEO |
|---------|------|---------|---------|-----|
| CSR | Client-Side Rendering | 后台管理系统 | 慢 | 差 |
| SSR | Server-Side Rendering | 需要 SEO 的动态页面 | 快 | 好 |
| SSG | Static Site Generation | 博客、文档站 | 最快 | 好 |

### 2.3 基础 SSR 示例

```js
// server.js - 简化的 SSR 服务
import { createSSRApp } from 'vue'
import { renderToString } from 'vue/server-renderer'
import App from './App.vue'

async function render(req, res) {
  const app = createSSRApp(App)
  const html = await renderToString(app)

  res.send(`
    <!DOCTYPE html>
    <html>
      <body>
        <div id="app">${html}</div>
        <script type="module" src="/src/entry-client.js"></script>
      </body>
    </html>
  `)
}
```

### 2.4 Hydration 过程

```js
// entry-client.js - 客户端激活
import { createApp } from 'vue'
import App from './App.vue'

// createApp 会自动"激活"已有的 SSR HTML
// 而非重新创建 DOM
createApp(App).mount('#app')
```

### 2.5 SSR 中的注意事项

```vue
<script setup>
import { ref, onMounted } from 'vue'

const count = ref(0)

// 浏览器 API 只能在 onMounted 中使用
onMounted(() => {
  // 此时已在客户端
  const stored = localStorage.getItem('count')
  if (stored) count.value = Number(stored)
})

// ❌ 错误：SSR 时 window 不存在
// localStorage.setItem('x', 'y')
</script>
```

## 三、常见用例

- 电商网站（SEO 友好的商品页面）
- 新闻/博客平台（搜索引擎索引文章内容）
- 需要社交分享预览的页面（Open Graph 元数据）

## 四、注意事项与常见陷阱

- SSR 代码不能使用浏览器 API（`window`、`document`、`localStorage`）
- 第三方库可能不兼容 SSR（需动态导入或条件判断）
- SSR 增加服务器负载和响应时间
- 需要处理数据预取和状态序列化
- hydration 不匹配会导致页面闪烁或交互异常
- 使用 `import.meta.env.SSR` 判断当前是否为 SSR 环境
- 推荐直接使用 Nuxt 而非手动搭建 SSR（降低复杂度）
