# 插件系统概述

## 一、概念说明

Vue 插件是为 Vue 应用添加全局功能的工具。插件通过 `app.use()` 安装，可以添加全局组件、指令、mixin、provide 值等。常见的插件有 Vue Router、Pinia、Vue I18n 等。

插件是一个对象或函数，必须暴露一个 `install` 方法。

## 二、具体用法

### 2.1 安装插件

```js
// main.js
import { createApp } from 'vue'
import App from './App.vue'
import router from './router'
import { createPinia } from 'pinia'

const app = createApp(App)
app.use(createPinia())
app.use(router)
app.mount('#app')
```

### 2.2 编写简单插件

```js
// plugins/myPlugin.js
export default {
  install(app, options) {
    // 添加全局方法
    app.config.globalProperties.$hello = (name) => {
      console.log(`Hello, ${name}!`)
    }

    // 添加全局指令
    app.directive('focus', {
      mounted(el) {
        el.focus()
      }
    })

    // 添加全局 provide
    app.provide('myOptions', options)

    // 添加全局组件
    app.component('MyButton', {
      template: '<button><slot /></button>'
    })
  }
}
```

```js
// 使用插件
import myPlugin from './plugins/myPlugin'
const app = createApp(App)
app.use(myPlugin, { theme: 'dark' })
```

### 2.3 函数式插件

```js
// 插件也可以是函数
export function themePlugin(app) {
  app.provide('theme', { primary: '#42b883', dark: false })
}
```

### 2.4 带有组合式函数的插件

```js
// plugins/toastPlugin.js
import { ref } from 'vue'

export default {
  install(app) {
    const toasts = ref([])

    const toast = {
      show(message, type = 'info', duration = 3000) {
        const id = Date.now()
        toasts.value.push({ id, message, type })
        setTimeout(() => {
          toasts.value = toasts.value.filter(t => t.id !== id)
        }, duration)
      },
      success(msg) { this.show(msg, 'success') },
      error(msg) { this.show(msg, 'error') },
      warning(msg) { this.show(msg, 'warning') }
    }

    app.config.globalProperties.$toast = toast
    app.provide('toast', toast)
  }
}
```

```vue
<!-- 组件中使用 -->
<script setup>
import { inject } from 'vue'
const toast = inject('toast')

function handleClick() {
  toast.success('操作成功！')
}
</script>
```

### 2.5 TypeScript 类型扩展

```ts
// plugins/myPlugin.d.ts
import { Toast } from './toastPlugin'

declare module 'vue' {
  interface ComponentCustomProperties {
    $toast: Toast
  }
}
```

## 三、常见用例

| 插件 | 用途 |
|------|------|
| Vue Router | 前端路由管理 |
| Pinia | 状态管理 |

| VueUse | 组合式工具集 |

## 四、注意事项与常见陷阱

- `app.use()` 必须在 `app.mount()` 之前调用
- 插件的 `install` 方法接收 `(app, options)` 两个参数
- 重复调用 `app.use()` 同一插件不会报错但也不必要
- TypeScript 项目中使用插件可能需要扩展全局类型
- 推荐使用 provide/inject 而非 globalProperties（更易于类型推断）
- 插件中创建的响应式数据应确保每个应用实例独立（避免 SSR 中的状态共享）
