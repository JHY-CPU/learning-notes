# 调试技巧

## 一、概念说明

Vue 应用的调试主要依赖 **Vue DevTools** 浏览器扩展和浏览器开发者工具。Vue DevTools 提供组件树查看、状态检查、事件追踪等专用功能，而 Chrome DevTools 提供断点调试、网络分析等通用能力。

```vue
<script setup>
import { ref, computed } from 'vue'

const items = ref([
  { id: 1, name: '苹果', price: 5 },
  { id: 2, name: '香蕉', price: 3 }
])

const total = computed(() =>
  items.value.reduce((sum, item) => sum + item.price, 0)
)

// 在 DevTools 中可以看到 items 和 total 的实时值
</script>

<template>
  <ul>
    <li v-for="item in items" :key="item.id">
      {{ item.name }} - ¥{{ item.price }}
    </li>
  </ul>
  <p>总计: ¥{{ total }}</p>
</template>
```

## 二、具体用法

### 2.1 Vue DevTools

安装 Chrome/Firefox 扩展后，打开浏览器 DevTools 会出现 "Vue" 面板：

- **Components**：查看组件树、props、data、computed
- **Pinia/Vuex**：查看状态管理数据和时间旅行调试
- **Timeline**：查看事件触发和性能时间线
- **Routes**：查看 Vue Router 路由状态

### 2.2 console 调试

```vue
<script setup>
import { ref, watch } from 'vue'

const count = ref(0)

// 使用 watch 监听数据变化
watch(count, (newVal, oldVal) => {
  console.log(`count: ${oldVal} → ${newVal}`)
})

// 调试响应式对象
import { toRaw } from 'vue'
const debugObj = () => console.log(toRaw(state))
</script>
```

### 2.3 断点调试

```vue
<script setup>
import { ref } from 'vue'

const data = ref(null)

async function fetchData() {
  debugger // 浏览器在此处暂停
  const res = await fetch('/api/data')
  data.value = await res.json()
}
</script>
```

### 2.4 源码映射（Source Maps）

```js
// vite.config.js
export default defineConfig({
  build: {
    sourcemap: true // 生产构建也生成 source map
  }
})
```

## 三、注意事项与常见陷阱

- Vue DevTools 在生产环境默认禁用
- 响应式数据是 Proxy 包裹的，打印原始值用 `toRaw()`
- 调试 `<script setup>` 时，变量名可能被编译器重命名
- 使用 `debugger` 语句后记得在生产构建前移除
- DevTools 连接不上时尝试刷新页面或检查扩展版本
