# 选项式 vs 组合式对比

## 一、概念说明

Vue 3 同时支持**选项式 API**（Options API）和**组合式 API**（Composition API）。两者是同一框架的两种编程风格，可以混用。

选项式 API 按选项（data/methods/computed/watch）分类组织代码，适合简单组件。组合式 API 按**逻辑关注点**组织代码，适合复杂组件和逻辑复用。

```vue
<!-- 选项式 API -->
<script>
export default {
  data() { return { count: 0 } },
  methods: { increment() { this.count++ } }
}
</script>

<!-- 组合式 API -->
<script setup>
import { ref } from 'vue'
const count = ref(0)
const increment = () => count.value++
</script>
```

## 二、具体用法

### 2.1 同一功能两种写法

**选项式 API：**
```vue
<script>
export default {
  data() {
    return { searchQuery: '', results: [], loading: false }
  },
  computed: {
    hasResults() { return this.results.length > 0 }
  },
  methods: {
    async search() {
      this.loading = true
      this.results = await fetchResults(this.searchQuery)
      this.loading = false
    }
  },
  watch: {
    searchQuery() { this.search() }
  }
}
</script>
```

**组合式 API：**
```vue
<script setup>
import { ref, computed, watch } from 'vue'

const searchQuery = ref('')
const results = ref([])
const loading = ref(false)

const hasResults = computed(() => results.value.length > 0)

async function search() {
  loading.value = true
  results.value = await fetchResults(searchQuery.value)
  loading.value = false
}

watch(searchQuery, search)
</script>
```

### 2.2 选择建议

| 场景 | 推荐 |
|------|------|
| 简单组件、快速原型 | 选项式 API |
| 复杂逻辑、逻辑复用 | 组合式 API |
| Vue 2 迁移项目 | 选项式 API |
| TypeScript 项目 | 组合式 API |
| 团队熟悉 Vue 2 | 选项式 API |

## 三、注意事项与常见陷阱

- 两种 API 可以混用，但建议统一风格
- `<script setup>` 中不能同时使用 `export default`
- 组合式 API 的代码在模板中自动可用，无需 return
- 选项式 API 的 `this` 在组合式 API 的 `setup()` 中不可用
- Vue 3 官方推荐新项目使用组合式 API + `<script setup>`
