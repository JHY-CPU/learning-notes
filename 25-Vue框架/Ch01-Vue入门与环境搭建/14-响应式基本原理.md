# 响应式基本原理

## 一、概念说明

Vue 的响应式系统是其核心特性，实现了**数据变化自动驱动视图更新**。Vue 3 使用 ES6 的 **Proxy** 对象拦截数据的读取和修改操作，建立数据与副作用（DOM 更新）之间的依赖关系。

当组件渲染时，会读取响应式数据（触发 getter），Vue 记录这个依赖关系。当数据被修改时（触发 setter），Vue 通知所有依赖的组件重新渲染。

```vue
<script setup>
import { ref } from 'vue'

const count = ref(0)

// 以下代码触发响应式：
// 读取 count.value → 触发 getter，收集依赖
// 修改 count.value → 触发 setter，通知更新
</script>

<template>
  <p>当前值: {{ count }}</p>
  <button @click="count++">+1</button>
</template>
```

## 二、具体用法

### 2.1 Proxy 基本原理

```js
// 简化版响应式实现
function reactive(obj) {
  return new Proxy(obj, {
    get(target, key) {
      track(target, key) // 收集依赖
      return Reflect.get(target, key)
    },
    set(target, key, value) {
      const result = Reflect.set(target, key, value)
      trigger(target, key) // 触发更新
      return result
    }
  })
}
```

### 2.2 ref 的实现原理

```js
// 简化版 ref 实现
function ref(value) {
  return {
    get value() {
      track(this, 'value') // 收集依赖
      return value
    },
    set value(newVal) {
      value = newVal
      trigger(this, 'value') // 触发更新
    }
  }
}
```

### 2.3 响应式追踪流程

```
组件渲染 → 读取数据 → Proxy getter → 收集依赖
                ↓
数据修改 → Proxy setter → 触发更新 → 组件重新渲染
```

## 三、注意事项与常见陷阱

- 只有在组件渲染或 watch/computed 计算时才会收集依赖
- 使用解构赋值会破坏 Proxy 引用，丢失响应式
- `reactive()` 只能用于对象，基本类型用 `ref()`
- 数组的索引赋值和 length 修改在 Vue 3 中是响应式的（Proxy 优势）
- 添加新属性到 reactive 对象是响应式的（Vue 2 需要 `Vue.set`）
