# 开发环境搭建

## 一、概念说明

Vue 3 开发需要准备以下工具链：**Node.js**（JavaScript 运行时）、**包管理器**（npm/pnpm）、**代码编辑器**（VS Code）以及 **Vue 相关插件**。

Node.js 提供了运行构建工具的环境，npm/pnpm 用于管理项目依赖。VS Code 配合 Volar 插件能提供出色的 Vue 开发体验，包括语法高亮、智能提示、类型检查等。

## 二、具体用法

### 2.1 安装 Node.js

前往 [Node.js 官网](https://nodejs.org/) 下载 LTS 版本（推荐 18.x 或更高）。安装后验证：

```bash
node -v    # 输出: v18.x.x 或更高
npm -v     # 输出: 9.x.x 或更高
```

### 2.2 安装 pnpm（推荐）

pnpm 比 npm 更快、更节省磁盘空间：

```bash
npm install -g pnpm
pnpm -v    # 验证安装成功
```

### 2.3 使用 nvm 管理 Node 版本

```bash
# 安装 nvm（Windows 使用 nvm-windows）
# https://github.com/coreybutler/nvm-windows/releases

nvm install 18        # 安装 Node 18
nvm install 20        # 安装 Node 20
nvm use 18            # 切换到 Node 18
nvm list              # 查看已安装版本
```

### 2.4 创建 Vue 项目

```bash
# 使用 create-vue（官方脚手架）
pnpm create vue@latest my-vue-app

# 交互式选择：
# ✓ TypeScript? Yes
# ✓ JSX? No
# ✓ Router? Yes
# ✓ Pinia? Yes
# ✓ Vitest? Yes
# ✓ ESLint? Yes
# ✓ Prettier? Yes

cd my-vue-app
pnpm install
pnpm dev
```

### 2.5 VS Code 插件安装

必装插件：
- **Vue - Official**（原 Volar）：Vue 3 官方插件，提供语法支持
- **ESLint**：代码规范检查
- **Prettier**：代码格式化

```bash
# 命令行安装（如果已安装 code 命令）
code --install-extension Vue.volar
code --install-extension dbaeumer.vscode-eslint
code --install-extension esbenp.prettier-vscode
```

### 2.6 配置 VS Code 设置

```json
// .vscode/settings.json
{
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "[vue]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  },
  "eslint.validate": ["javascript", "vue"]
}
```

## 三、常见用例

### 3.1 项目目录结构

```
my-vue-app/
├── src/
│   ├── assets/        # 静态资源
│   ├── components/    # 公共组件
│   ├── views/         # 页面组件
│   ├── router/        # 路由配置
│   ├── stores/        # Pinia 状态
│   ├── composables/   # 组合式函数
│   ├── App.vue        # 根组件
│   └── main.js        # 入口文件
├── public/            # 不经打包的静态文件
├── index.html         # HTML 入口
├── vite.config.js     # Vite 配置
└── package.json       # 项目配置
```

### 3.2 环境变量配置

```bash
# .env                # 所有环境
# .env.local          # 本地覆盖（git 忽略）
# .env.development    # 开发环境
# .env.production     # 生产环境

VITE_API_BASE=http://localhost:3000
VITE_APP_TITLE=我的Vue应用
```

```vue
<script setup>
// 使用环境变量（必须以 VITE_ 开头）
console.log(import.meta.env.VITE_API_BASE)
console.log(import.meta.env.MODE) // 'development' | 'production'
</script>
```

## 四、注意事项与常见陷阱

- 安装 Volar 前必须**禁用**旧的 Vetur 插件，两者冲突
- VS Code 需要 1.67+ 版本才能完全支持 Volar
- pnpm 的 `node_modules` 结构与 npm 不同，某些包可能需要 `.npmrc` 配置 `shamefully-hoop=true`
- 建议使用 nvm（Node Version Manager）管理多版本 Node.js
- Windows 用户注意终端编码设置为 UTF-8 避免中文乱码
- 环境变量必须以 `VITE_` 前缀开头才能在客户端代码中访问
- `pnpm dev` 默认端口是 5173，可通过 `--port` 修改
