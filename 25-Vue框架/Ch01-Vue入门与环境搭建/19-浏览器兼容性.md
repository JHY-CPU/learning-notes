# 浏览器兼容性

## 一、概念说明

Vue 3 使用了 ES2015+ 特性（如 Proxy、Reflect），**不再支持 IE11**。Vue 3 支持所有支持 ES2015 的现代浏览器。如果需要支持旧浏览器，可以使用 Vue 2 或配合 Babel/Polyfill。

```vue
<script setup>
// Vue 3 依赖以下现代特性：
// - Proxy（响应式系统核心）
// - Reflect（配合 Proxy 使用）
// - Map/Set/WeakMap/WeakSet
// - Promise
// - Symbol
// - 箭头函数、模板字符串等 ES6+ 语法
import { ref, reactive } from 'vue'

const data = reactive({ items: new Set() })
</script>

<template>
  <p>Vue 3 需要现代浏览器支持</p>
</template>
```

## 二、具体用法

### 2.1 浏览器支持范围

```
支持的最低版本：
- Chrome >= 87
- Firefox >= 78
- Safari >= 14 (macOS) / >= 13 (iOS)
- Edge >= 88
- 不支持 IE 任何版本
```

### 2.2 构建目标配置

```js
// vite.config.js
export default defineConfig({
  build: {
    target: 'es2015', // 构建目标
    // 'modules' - 支持原生 ESM 的浏览器（默认）
    // 'es2015' - ES2015 支持的浏览器
  }
})
```

### 2.3 Polyfill 策略

```js
// vite.config.js
import { defineConfig } from 'vite'
import legacy from '@vitejs/plugin-legacy'

export default defineConfig({
  plugins: [
    legacy({
      targets: ['defaults', 'not IE 11'],
      additionalLegacyPolyfills: ['regenerator-runtime/runtime']
    })
  ]
})
```

## 三、注意事项与常见陷阱

- Vue 3 的 Proxy 无法 polyfill，IE11 绝对不可用
- iOS Safari 13 以下可能有 CSS 变量兼容问题
- 使用 `@vitejs/plugin-legacy` 为旧浏览器提供降级支持
- `.mjs` 文件在某些旧服务器可能需要配置 MIME 类型
- CSS `v-bind()` 功能需要现代浏览器支持
