# 浏览器兼容性

## 一、概念说明

Vue 3 使用了 ES2015+ 特性（如 Proxy、Reflect），**不再支持 IE11**。Vue 3 支持所有支持 ES2015 的现代浏览器。如果需要支持旧浏览器，可以使用 Vue 2 或配合 Babel/Polyfill。

## 二、具体用法

### 2.1 浏览器支持范围

```
支持的最低版本：
- Chrome >= 87
- Firefox >= 78
- Safari >= 14 (macOS) / >= 13 (iOS)
- Edge >= 88
- 不支持 IE 任何版本
```

### 2.2 Vue 3 依赖的现代特性

```js
// Vue 3 响应式系统核心依赖 Proxy
const reactive = (obj) => new Proxy(obj, {
  get(target, key) {
    track(target, key) // 追踪依赖
    return Reflect.get(target, key)
  },
  set(target, key, value) {
    Reflect.set(target, key, value)
    trigger(target, key) // 触发更新
    return true
  }
})
```

Proxy 无法被 polyfill，因此 IE11 完全不可用。

### 2.3 构建目标配置

```js
// vite.config.js
export default defineConfig({
  build: {
    target: 'es2015', // 构建目标
    // 'modules' - 支持原生 ESM 的浏览器（默认）
    // 'es2015' - ES2015 支持的浏览器
    // 'esnext' - 最新特性
  }
})
```

### 2.4 Polyfill 策略

```js
// vite.config.js
import { defineConfig } from 'vite'
import legacy from '@vitejs/plugin-legacy'

export default defineConfig({
  plugins: [
    legacy({
      targets: ['defaults', 'not IE 11'],
      additionalLegacyPolyfills: ['regenerator-runtime/runtime']
    })
  ]
})
```

### 2.5 CSS 兼容处理

```js
// vite.config.js
import { defineConfig } from 'vite'

export default defineConfig({
  css: {
    // PostCSS 自动添加浏览器前缀
    // postcss.config.js
    // module.exports = {
    //   plugins: [require('autoprefixer')]
    // }
  }
})
```

## 三、常见用例

### 3.1 检测浏览器支持

```js
// 在应用入口检测
function checkBrowserSupport() {
  if (typeof Proxy === 'undefined') {
    document.body.innerHTML = `
      <h1>浏览器不支持</h1>
      <p>请使用 Chrome 87+、Firefox 78+ 或 Safari 14+</p>
    `
    return false
  }
  return true
}
```

### 3.2 渐进增强策略

```vue
<script setup>
import { ref, onMounted } from 'vue'
const supportsLazy = ref(false)

onMounted(() => {
  // 检测是否支持原生懒加载
  supportsLazy.value = 'loading' in HTMLImageElement.prototype
})
</script>

<template>
  <img
    :loading="supportsLazy ? 'lazy' : undefined"
    :src="imageUrl"
    alt="图片"
  />
</template>
```

## 四、注意事项与常见陷阱

- Vue 3 的 Proxy 无法 polyfill，IE11 绝对不可用
- iOS Safari 13 以下可能有 CSS 变量兼容问题
- 使用 `@vitejs/plugin-legacy` 为旧浏览器提供降级支持
- `.mjs` 文件在某些旧服务器可能需要配置 MIME 类型
- CSS `v-bind()` 功能需要现代浏览器支持
- `@vitejs/plugin-legacy` 会增大构建体积，仅在需要时使用
- 移动端浏览器通常更新较快，兼容性问题较少
