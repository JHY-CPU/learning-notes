# 生产部署

## 一、概念说明

将 Vue 应用部署到生产环境需要经过**构建**（生成优化后的静态文件）和**发布**（上传到服务器/CDN）两个步骤。Vite 构建会自动进行代码压缩、Tree-shaking、代码分割等优化。

## 二、具体用法

### 2.1 构建命令

```bash
pnpm build    # 生成 dist/ 目录
pnpm preview  # 本地预览生产构建
```

### 2.2 环境变量配置

```bash
# .env.production
VITE_API_BASE=https://api.example.com
VITE_APP_TITLE=我的生产应用
```

```vue
<script setup>
// 构建时环境变量被嵌入代码中
const apiBase = import.meta.env.VITE_API_BASE
const mode = import.meta.env.MODE // 'production'
</script>
```

### 2.3 Nginx 部署配置

```nginx
server {
    listen 80;
    server_name example.com;
    root /var/www/vue-app/dist;

    # SPA 路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 静态资源缓存
    location /assets/ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # 开启 gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript;
    gzip_min_length 1000;
}
```

### 2.4 Docker 部署

```dockerfile
# Dockerfile
FROM node:18-alpine AS build
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=build /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

```bash
# 构建并运行
docker build -t my-vue-app .
docker run -p 8080:80 my-vue-app
```

### 2.5 静态托管平台部署

```bash
# Vercel
npm i -g vercel
vercel

# Netlify
# 构建命令: pnpm build
# 发布目录: dist

# GitHub Pages
# 在 vite.config.js 中设置 base
export default defineConfig({
  base: '/your-repo-name/'
})
```

### 2.6 CI/CD 配置（GitHub Actions）

```yaml
# .github/workflows/deploy.yml
name: Deploy
on:
  push:
    branches: [main]

jobs:
  build-deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v2
        with: { version: 8 }
      - uses: actions/setup-node@v4
        with: { node-version: 18, cache: pnpm }
      - run: pnpm install
      - run: pnpm build
      - uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./dist
```

## 三、常见用例

### 3.1 Base URL 配置

```js
// vite.config.js
// 应用部署在子路径时需要配置 base
export default defineConfig({
  base: '/app/',  // 部署在 https://example.com/app/
})
```

## 四、注意事项与常见陷阱

- `dist/` 是构建产物，不要提交到 Git
- 环境变量在构建时嵌入，修改后需重新构建
- SPA 路由需要服务器配置 fallback 到 `index.html`
- 开启 gzip/brotli 压缩减少传输体积
- 使用 CDN 加速静态资源加载
- 部署在子路径时必须配置 `base` 选项
- 生产环境关闭 Vue DevTools（默认已关闭）
- 设置安全响应头（CSP、X-Frame-Options 等）
