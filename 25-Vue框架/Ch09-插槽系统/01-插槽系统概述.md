# 插槽系统概述

## 一、概念说明
**插槽（Slots）**是 Vue 的**内容分发机制**，允许父组件向子组件的模板中插入内容。插槽让组件变得更加灵活和可复用。

## 二、插槽类型

### 2.1 三种插槽
```
默认插槽    → 子组件 <slot> 中的默认内容区域
具名插槽    → 通过 name 属性区分多个插槽位置
作用域插槽  → 子组件向父组件传递数据
```

### 2.2 基本示例
```vue
<!-- 子组件 Card.vue -->
<template>
  <div class="card">
    <header><slot name="header" /></header>
    <main><slot /></main>
    <footer><slot name="footer" /></footer>
  </div>
</template>
```

```vue
<!-- 父组件使用 -->
<template>
  <Card>
    <template #header>标题</template>
    <p>主内容</p>
    <template #footer>页脚</template>
  </Card>
</template>
```

### 2.3 作用域插槽
```vue
<!-- 子组件传数据给父组件 -->
<template>
  <ul>
    <li v-for="item in items" :key="item.id">
      <slot :item="item" :index="index" />
    </li>
  </ul>
</template>
```

```vue
<!-- 父组件接收数据 -->
<template>
  <List :items="list">
    <template #default="{ item, index }">
      {{ index }}. {{ item.name }}
    </template>
  </List>
</template>
```

### 2.4 使用场景
- **布局组件**：header/content/footer 分区
- **列表组件**：自定义每一项的渲染
- **模态框**：自定义标题、内容、按钮
- **高阶组件**：控制渲染逻辑的封装

## 三、注意事项与常见陷阱
- 默认插槽的内容在 `<slot>` 标签的后备内容位置显示
- 父组件的模板内容会在编译时替换子组件的插槽
- 作用域插槽使子组件成为"渲染器"
- 缩写 `#` 只能用在 `<template>` 上

## 四、插槽的渲染机制

```
父组件编译阶段：
  模板内容 → 生成渲染函数 → 传递给子组件

子组件渲染阶段：
  <slot /> → 替换为父组件的渲染函数 → 输出 DOM
```

### 4.1 插槽的作用域
```vue
<!-- 子组件 Child.vue -->
<template>
  <div>
    <!-- 插槽内容在父组件作用域中编译 -->
    <slot />
    <!-- 无法访问子组件的 data -->
  </div>
</template>
<script setup>
const childData = '子组件数据'
</script>
```

```vue
<!-- 父组件 -->
<template>
  <Child>
    <!-- ✅ 可以访问父组件的 data -->
    <p>{{ parentData }}</p>
    <!-- ❌ 无法访问子组件的 childData -->
  </Child>
</template>
```

### 4.2 作用域插槽的数据流
```
子组件 → slot props → 父组件渲染函数 → DOM 输出
         (数据)       (渲染逻辑)       (最终结果)
```

## 五、插槽的常见设计模式

### 5.1 布局组件模式
```vue
<!-- Layout.vue -->
<template>
  <div class="layout">
    <header><slot name="header" /></header>
    <nav><slot name="sidebar" /></nav>
    <main><slot /></main>
    <footer><slot name="footer" /></footer>
  </div>
</template>
```

### 5.2 容器组件模式
```vue
<!-- Container.vue：提供数据和状态，渲染由父组件决定 -->
<template>
  <div>
    <div v-if="loading">加载中...</div>
    <div v-else-if="error">{{ error }}</div>
    <slot v-else :data="data" :refresh="fetchData" />
  </div>
</template>
```

### 5.3 高阶组件模式
```vue
<!-- WithAuth.vue：控制渲染条件 -->
<template>
  <slot v-if="isLoggedIn" :user="user" :logout="logout" />
  <slot v-else name="unauthorized">
    <p>请先登录</p>
  </slot>
</template>
```
