# 插槽设计模式与最佳实践

## 一、概念说明
良好的插槽设计可以极大提高组件的**灵活性和可复用性**。以下是经过实践验证的插槽设计模式和最佳实践。

## 二、设计模式

### 2.1 容器/展示模式
```vue
<!-- 容器组件：只管数据和逻辑 -->
<script setup>
const { data, loading, error } = useAsyncData()
</script>

<template>
  <slot :data="data" :loading="loading" :error="error" />
</template>
```

```vue
<!-- 展示组件：只管渲染 -->
<template>
  <DataProvider v-slot="{ data, loading }">
    <Spinner v-if="loading" />
    <UserCard v-else :user="data" />
  </DataProvider>
</template>
```

### 2.2 布局组件模式
```vue
<template>
  <div class="layout" :class="{ 'has-sidebar': hasSidebar }">
    <slot name="header" />
    <div class="body">
      <slot name="sidebar" />
      <main><slot /></main>
    </div>
    <slot name="footer" />
  </div>
</template>
```

### 2.3 复合组件模式
```vue
<!-- Dropdown.vue -->
<template>
  <div class="dropdown">
    <button @click="open = !open">
      <slot name="trigger">请选择</slot>
    </button>
    <div v-if="open" class="dropdown-menu">
      <slot />
    </div>
  </div>
</template>
```

### 2.4 最佳实践总结
```vue
<!-- ✅ 好的做法 -->
<DataTable :data="users">
  <template #cell-name="{ value }">
    <strong>{{ value }}</strong>
  </template>
  <template #empty>
    <p>暂无数据</p>
  </template>
</DataTable>

<!-- ❌ 不好的做法：暴露太多内部实现 -->
<DataTable :data="users">
  <template #row="{ row, rowIndex, columns, selection, sort }">
    <!-- 过度暴露内部细节 -->
  </template>
</DataTable>
```

### 2.5 命名约定
```
具名插槽:     kebab-case (#header, #user-info)
作用域变量:   camelCase ({ userData, totalCount })
后备内容:     语义化默认值
```

## 三、注意事项与常见陷阱
- 插槽暴露的数据应够用但不多，保持接口简洁
- 后备内容应提供合理的默认渲染
- 避免过度使用插槽，简单场景用 props 就够了
- 组件文档中说明每个插槽的作用和可用数据
- 使用 TypeScript 的 `defineSlots` 提供类型安全
