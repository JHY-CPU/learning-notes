# 插槽性能考虑

## 一、概念说明
插槽内容会在**每次父组件重新渲染时重新创建**。如果插槽内容包含复杂的子组件或大量 DOM，可能会导致不必要的性能开销。

## 二、具体用法

### 2.1 插槽内容的渲染时机
```vue
<!-- 子组件 -->
<template>
  <div>
    <slot />  <!-- 每次父组件渲染，这里的内容都会重新生成 -->
  </div>
</template>
```

```vue
<!-- 父组件 -->
<template>
  <Wrapper>
    <!-- 每次 count 变化，这个插槽内容都被重新创建 -->
    <ExpensiveComponent :data="items" />
  </Wrapper>
  <button @click="count++">+1</button>
</template>
```

### 2.2 使用 v-memo 优化
```vue
<template>
  <Wrapper>
    <!-- 只有 items 变化时才重新渲染 -->
    <div v-memo="[items]">
      <ExpensiveComponent :data="items" />
    </div>
  </Wrapper>
</template>
```

### 2.3 使用 computed 缓存数据
```vue
<script setup>
import { computed } from 'vue'
const processedData = computed(() => expensiveProcess(rawData.value))
</script>

<template>
  <Wrapper>
    <!-- 传入 computed 值，减少变化频率 -->
    <ExpensiveComponent :data="processedData" />
  </Wrapper>
</template>
```

### 2.4 拆分插槽减少重渲染
```vue
<!-- 将频繁变化的部分分离 -->
<template>
  <Layout>
    <template #static>
      <!-- 不变化的内容放这里 -->
      <Sidebar />
    </template>

    <template #default>
      <!-- 变化的内容放这里 -->
      <DynamicContent :count="count" />
    </template>
  </Layout>
</template>
```

### 2.5 性能对比
```
简单插槽（文本/图标）     → 性能影响可忽略
中等插槽（少量组件）      → 注意更新频率
复杂插槽（大量子组件）    → 使用 v-memo / computed 优化
```

## 三、注意事项与常见陷阱
- 插槽内容在父组件作用域编译，父组件变化会触发重新创建
- 不要将不需要变化的复杂内容放在插槽中
- 使用 `v-memo` 可以有效缓存插槽子树
- 高频更新场景下，考虑拆分插槽或用 computed
