# 插槽与组合式函数

## 一、概念说明
组合式函数（Composables）和插槽可以**配合使用**：组合式函数处理数据逻辑，插槽控制渲染。两者结合提供强大的组件复用模式。

## 二、具体用法

### 2.1 Composable 封装逻辑
```js
// composables/useSearch.js
import { ref, computed } from 'vue'

export function useSearch(items, searchKey = 'name') {
  const query = ref('')

  const results = computed(() => {
    if (!query.value) return items.value
    return items.value.filter(item =>
      item[searchKey].toLowerCase().includes(query.value.toLowerCase())
    )
  })

  return { query, results }
}
```

### 2.2 配合插槽使用
```vue
<!-- SearchList.vue -->
<script setup>
import { useSearch } from '@/composables/useSearch'

const props = defineProps<{ items: any[] }>()
const { query, results } = useSearch(toRef(props, 'items'))
</script>

<template>
  <input v-model="query" placeholder="搜索..." />
  <slot :results="results" :query="query" />
</template>
```

```vue
<!-- 父组件自定义搜索结果渲染 -->
<template>
  <SearchList :items="users" v-slot="{ results, query }">
    <p v-if="!results.length && query">未找到 "{{ query }}" 的结果</p>
    <ul v-else>
      <li v-for="item in results" :key="item.id">
        <slot name="item" :item="item" />
      </li>
    </ul>
  </SearchList>
</template>
```

### 2.3 Pagination Composable + 插槽
```vue
<script setup>
import { usePagination } from '@/composables/usePagination'

const props = defineProps<{ items: any[], pageSize?: number }>()
const { currentPage, totalPages, paginatedItems, next, prev } =
  usePagination(toRef(props, 'items'), { pageSize: props.pageSize || 10 })
</script>

<template>
  <slot :items="paginatedItems" />
  <div class="pagination">
    <button @click="prev">上一页</button>
    <span>{{ currentPage }} / {{ totalPages }}</span>
    <button @click="next">下一页</button>
  </div>
</template>
```

## 三、注意事项与常见陷阱
- Composable 负责逻辑，插槽负责渲染，职责分离
- 使用 `toRef` 将 props 转为 ref 传给 composable
- 不要在插槽模板中直接调用 composable（应由子组件调用）
- 这种模式让子组件成为"逻辑容器"，父组件控制外观
