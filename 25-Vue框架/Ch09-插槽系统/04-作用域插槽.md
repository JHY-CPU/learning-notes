# 作用域插槽

## 一、概念说明
作用域插槽允许**子组件向父组件传递数据**。子组件在 `<slot>` 上绑定属性（类似 props），父组件通过 `v-slot` 接收这些数据来渲染内容。

## 二、具体用法

### 2.1 子组件传递数据
```vue
<!-- UserList.vue -->
<template>
  <ul>
    <li v-for="(user, index) in users" :key="user.id">
      <!-- 向插槽传递 user 和 index -->
      <slot :user="user" :index="index" :isSelected="selectedId === user.id">
        {{ user.name }}  <!-- 默认渲染 -->
      </slot>
    </li>
  </ul>
</template>
<script setup>
defineProps<{
  users: Array<{ id: number; name: string }>
  selectedId?: number
}>()
</script>
```

### 2.2 父组件接收数据
```vue
<template>
  <UserList :users="list" :selectedId="selected">
    <template v-slot:default="slotProps">
      <span :class="{ active: slotProps.isSelected }">
        {{ slotProps.index + 1 }}. {{ slotProps.user.name }}
      </span>
    </template>
  </UserList>
</template>
```

### 2.3 解构语法
```vue
<template>
  <UserList :users="list">
    <template #default="{ user, index, isSelected }">
      <span :class="{ active: isSelected }">
        {{ index + 1 }}. {{ user.name }}
      </span>
    </template>
  </UserList>
</template>
```

### 2.4 独占默认插槽的简写
```vue
<!-- 只有默认插槽时，可以直接在组件上用 v-slot -->
<template>
  <UserList v-slot="{ user }" :users="list">
    <span>{{ user.name }}</span>
  </UserList>
</template>
```

## 三、注意事项与常见陷阱
- 作用域插槽的数据是子组件的，渲染逻辑是父组件的
- 解构语法中可以用别名：`v-slot="{ user: person }"`
- 独占默认插槽简写不能与具名插槽混用
- 作用域插槽使子组件成为"渲染器"模式

## 五、实际应用

### 5.1 通用列表组件
```vue
<!-- GenericList.vue -->
<script setup>
defineProps({
  items: { type: Array, required: true },
  itemKey: { type: String, default: 'id' }
})
</script>

<template>
  <ul class="generic-list">
    <li v-for="(item, index) in items" :key="item[itemKey]">
      <slot :item="item" :index="index" :isFirst="index === 0" :isLast="index === items.length - 1">
        <!-- 默认渲染 -->
        {{ item }}
      </slot>
    </li>
  </ul>
</template>
```

```vue
<!-- 使用 -->
<template>
  <GenericList :items="users" item-key="userId">
    <template #default="{ item, index, isFirst }">
      <div class="user-card" :class="{ highlight: isFirst }">
        <span class="rank">#{{ index + 1 }}</span>
        <span class="name">{{ item.name }}</span>
        <span class="score">{{ item.score }}</span>
      </div>
    </template>
  </GenericList>
</template>
```

### 5.2 表单字段组件
```vue
<!-- FormField.vue -->
<template>
  <div class="form-field">
    <label v-if="label">{{ label }}</label>
    <slot :value="modelValue" :error="error" :update="update" />
    <p v-if="error" class="error">{{ error }}</p>
  </div>
</template>
```

```vue
<!-- 使用 -->
<template>
  <FormField v-model="email" label="邮箱" :rules="emailRules">
    <template #default="{ value, error, update }">
      <input
        :value="value"
        :class="{ invalid: error }"
        @input="update($event.target.value)"
        type="email"
      />
    </template>
  </FormField>
</template>
```

### 5.3 数据提供组件
```vue
<!-- DataProvider.vue -->
<template>
  <slot
    :data="data"
    :loading="loading"
    :error="error"
    :refresh="refresh"
    :loadMore="loadMore"
  />
</template>
<script setup>
import { ref, onMounted } from 'vue'

const props = defineProps({ url: String })
const data = ref([])
const loading = ref(false)
const error = ref(null)

async function refresh() {
  loading.value = true
  try {
    data.value = await fetch(props.url).then(r => r.json())
  } catch (e) {
    error.value = e
  } finally {
    loading.value = false
  }
}

onMounted(refresh)
</script>
```

```vue
<!-- 使用 -->
<template>
  <DataProvider url="/api/users">
    <template #default="{ data, loading, error, refresh }">
      <div v-if="loading">加载中...</div>
      <div v-else-if="error">
        <button @click="refresh">重试</button>
      </div>
      <ul v-else>
        <li v-for="user in data" :key="user.id">{{ user.name }}</li>
      </ul>
    </template>
  </DataProvider>
</template>
```
