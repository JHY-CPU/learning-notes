# 作用域插槽

## 一、概念说明
作用域插槽允许**子组件向父组件传递数据**。子组件在 `<slot>` 上绑定属性（类似 props），父组件通过 `v-slot` 接收这些数据来渲染内容。

## 二、具体用法

### 2.1 子组件传递数据
```vue
<!-- UserList.vue -->
<template>
  <ul>
    <li v-for="(user, index) in users" :key="user.id">
      <!-- 向插槽传递 user 和 index -->
      <slot :user="user" :index="index" :isSelected="selectedId === user.id">
        {{ user.name }}  <!-- 默认渲染 -->
      </slot>
    </li>
  </ul>
</template>
<script setup>
defineProps<{
  users: Array<{ id: number; name: string }>
  selectedId?: number
}>()
</script>
```

### 2.2 父组件接收数据
```vue
<template>
  <UserList :users="list" :selectedId="selected">
    <template v-slot:default="slotProps">
      <span :class="{ active: slotProps.isSelected }">
        {{ slotProps.index + 1 }}. {{ slotProps.user.name }}
      </span>
    </template>
  </UserList>
</template>
```

### 2.3 解构语法
```vue
<template>
  <UserList :users="list">
    <template #default="{ user, index, isSelected }">
      <span :class="{ active: isSelected }">
        {{ index + 1 }}. {{ user.name }}
      </span>
    </template>
  </UserList>
</template>
```

### 2.4 独占默认插槽的简写
```vue
<!-- 只有默认插槽时，可以直接在组件上用 v-slot -->
<template>
  <UserList v-slot="{ user }" :users="list">
    <span>{{ user.name }}</span>
  </UserList>
</template>
```

## 三、注意事项与常见陷阱
- 作用域插槽的数据是子组件的，渲染逻辑是父组件的
- 解构语法中可以用别名：`v-slot="{ user: person }"`
- 独占默认插槽简写不能与具名插槽混用
- 作用域插槽使子组件成为"渲染器"模式
