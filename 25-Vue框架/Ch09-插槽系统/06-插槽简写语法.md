# 插槽简写语法

## 一、概念说明
Vue 提供了 `#` 符号作为 `v-slot:` 的**简写**。这使模板代码更简洁，但有一些使用限制需要注意。

## 二、具体用法

### 2.1 基本简写
```vue
<template>
  <!-- 完整写法 -->
  <template v-slot:header>标题</template>
  <template v-slot:default>内容</template>
  <template v-slot:footer>页脚</template>

  <!-- 简写 -->
  <template #header>标题</template>
  <template #default>内容</template>
  <template #footer>页脚</template>
</template>
```

### 2.2 独占默认插槽简写
```vue
<template>
  <!-- 当只有默认插槽时 -->
  <Card v-slot="{ item }">
    {{ item.name }}
  </Card>

  <!-- 不能对其他具名插槽使用此简写 -->
</template>
```

### 2.3 与动态插槽名组合
```vue
<template>
  <Layout>
    <template #[slotName]>动态插槽</template>
  </Layout>
</template>
<script setup>
import { ref } from 'vue'
const slotName = ref('header')
</script>
```

### 2.4 完整示例
```vue
<template>
  <Modal>
    <template #header="{ title }">
      <h2>{{ title }}</h2>
    </template>

    <template #default="{ data }">
      <p>{{ data.content }}</p>
    </template>

    <template #footer="{ onClose, onConfirm }">
      <button @click="onClose">取消</button>
      <button @click="onConfirm">确认</button>
    </template>
  </Modal>
</template>
```

### 2.5 对比写法
```vue
<!-- 冗长写法 -->
<template v-slot:header="{ title: t }">
  <h2>{{ t }}</h2>
</template>

<!-- 简写 -->
<template #header="{ title: t }">
  <h2>{{ t }}</h2>
</template>
```

## 三、注意事项与常见陷阱
- `#` 只能用在 `<template>` 上
- `#default` 可以省略，直接写内容
- `#` 后面不能有空格：`# header` 是错误的
- 独占默认插槽简写（在组件上 `v-slot`）不能与具名插槽混用

## 六、简写的适用场景

```vue
<!-- 场景一：模态框 -->
<template>
  <Modal v-model:visible="show">
    <template #header>
      <h3>确认删除</h3>
    </template>
    <p>确定要删除这条记录吗？</p>
    <template #footer="{ confirm, cancel }">
      <button @click="cancel">取消</button>
      <button @click="confirm">确认</button>
    </template>
  </Modal>
</template>

<!-- 场景二：表格自定义列 -->
<template>
  <Table :data="users">
    <template #cell-name="{ value }">
      <strong>{{ value }}</strong>
    </template>
    <template #cell-actions="{ row }">
      <button @click="edit(row)">编辑</button>
      <button @click="remove(row)">删除</button>
    </template>
  </Table>
</template>

<!-- 场景三：作用域插槽独占默认 -->
<template>
  <Select :options="options" v-slot="{ option, isSelected }">
    <span :class="{ selected: isSelected }">
      {{ option.label }}
    </span>
  </Select>
</template>
```

## 七、常见错误

```vue
<!-- ❌ 错误：同时在组件上用 v-slot 和具名插槽 -->
<template>
  <List v-slot="{ item }" :items="data">
    <span>{{ item.name }}</span>
    <template #footer>页脚</template>  <!-- 这里会报错 -->
  </List>
</template>

<!-- ✅ 正确：只用默认插槽时可以独占 -->
<template>
  <List v-slot="{ item }" :items="data">
    <span>{{ item.name }}</span>
  </List>
</template>

<!-- ✅ 正确：有多个插槽时全部用 template -->
<template>
  <List :items="data">
    <template #default="{ item }">
      <span>{{ item.name }}</span>
    </template>
    <template #footer>页脚</template>
  </List>
</template>
```
