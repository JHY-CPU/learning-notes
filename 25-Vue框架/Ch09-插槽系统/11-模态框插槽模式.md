# 模态框插槽模式

## 一、概念说明
模态框（Modal/Dialog）是插槽模式的经典应用。通过具名插槽将模态框拆分为**头部、内容、底部**三个区域，让父组件完全控制模态框的显示内容。

## 二、具体用法

### 2.1 基础模态框组件
```vue
<!-- Modal.vue -->
<template>
  <Teleport to="body">
    <div v-if="visible" class="modal-overlay" @click.self="$emit('close')">
      <div class="modal">
        <header class="modal-header">
          <slot name="header">
            <h3>提示</h3>
          </slot>
          <button class="close-btn" @click="$emit('close')">×</button>
        </header>

        <main class="modal-body">
          <slot />
        </main>

        <footer class="modal-footer">
          <slot name="footer">
            <button @click="$emit('close')">关闭</button>
          </slot>
        </footer>
      </div>
    </div>
  </Teleport>
</template>
<script setup>
defineProps<{ visible: boolean }>()
defineEmits(['close'])
</script>
```

### 2.2 父组件使用
```vue
<template>
  <Modal :visible="showModal" @close="showModal = false">
    <template #header>
      <h3>确认删除</h3>
    </template>

    <p>确定要删除这条记录吗？此操作不可撤销。</p>

    <template #footer>
      <button @click="showModal = false">取消</button>
      <button class="danger" @click="confirmDelete">确认删除</button>
    </template>
  </Modal>
</template>
<script setup>
import { ref } from 'vue'
const showModal = ref(false)
function confirmDelete() { /* ... */ }
</script>
```

### 2.3 确认对话框封装
```vue
<template>
  <ConfirmDialog
    :visible="showConfirm"
    title="操作确认"
    @confirm="onConfirm"
    @cancel="showConfirm = false"
  >
    <template #default="{ data }">
      <p>即将删除: {{ data?.name }}</p>
    </template>
  </ConfirmDialog>
</template>
```

## 三、注意事项与常见陷阱
- 使用 `<Teleport>` 将模态框挂载到 body 下，避免 z-index 问题
- 遮罩层点击关闭要用 `@click.self` 避免点击内容区也关闭
- 模态框打开时应禁止背景滚动
- 多个模态框嵌套时注意层级管理
