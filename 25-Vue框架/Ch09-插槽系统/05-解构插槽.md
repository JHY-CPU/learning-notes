# 解构插槽

## 一、概念说明
解构插槽是使用 ES6 **解构语法**来接收作用域插槽传递的数据，让代码更简洁。可以解构对象、设置默认值和别名。

## 二、具体用法

### 2.1 基本解构
```vue
<template>
  <!-- 不解构 -->
  <DataList v-slot="props">
    {{ props.item.name }}
  </DataList>

  <!-- 解构 -->
  <DataList v-slot="{ item }">
    {{ item.name }}
  </DataList>
</template>
```

### 2.2 完整解构语法
```vue
<template>
  <Table :data="rows">
    <template #cell="{ row, column, value }">
      <!-- row: 行数据, column: 列配置, value: 单元格值 -->
      <td>{{ value }}</td>
    </template>
  </Table>
</template>
```

### 2.3 设置默认值
```vue
<template>
  <UserCard v-slot="{ user, role = 'guest' }">
    <p>{{ user.name }} - {{ role }}</p>
    <!-- 如果子组件未传 role，使用 'guest' -->
  </UserCard>
</template>
```

### 2.4 别名
```vue
<template>
  <!-- 将 item 重命名为 product -->
  <ProductList v-slot="{ item: product, index: i }">
    <span>#{{ i + 1 }}</span>
    <span>{{ product.name }}</span>
    <span>{{ product.price }}</span>
  </ProductList>
</template>
```

### 2.5 嵌套解构
```vue
<template>
  <OrderList v-slot="{ order: { id, items, total } }">
    <p>订单号: {{ id }}</p>
    <p>商品数: {{ items.length }}</p>
    <p>总金额: {{ total }}</p>
  </OrderList>
</template>
```

### 2.6 解构多个具名插槽
```vue
<template>
  <Layout>
    <template #header="{ title, subtitle }">
      <h1>{{ title }}</h1>
      <p>{{ subtitle }}</p>
    </template>

    <template #default="{ content }">
      <div v-html="content" />
    </template>
  </Layout>
</template>
```

## 三、注意事项与常见陷阱
- 解构语法与 JavaScript 对象解构规则一致
- 别名语法：`v-slot="{ original: alias }"`
- 空插槽数据解构会得到 `undefined`，注意设置默认值
- 解构不影响子组件传递的其他未解构属性

## 七、解构与 TypeScript

```vue
<!-- 使用泛型的子组件 -->
<script setup lang="ts" generic="T">
interface Props {
  items: T[]
  selected?: number[]
}
defineProps<Props>()

defineSlots<{
  default(props: { item: T; index: number; isSelected: boolean }): any
}>()
</script>

<template>
  <ul>
    <li v-for="(item, index) in items" :key="index">
      <slot :item="item" :index="index" :isSelected="selected?.includes(index)" />
    </li>
  </ul>
</template>
```

```vue
<!-- 父组件中自动获得类型推断 -->
<template>
  <List :items="users" v-slot="{ item, index, isSelected }">
    <!-- item 自动推断为 User 类型 -->
    <!-- TypeScript 会检查 item.name 是否存在 -->
    <span :class="{ active: isSelected }">
      #{{ index + 1 }} {{ item.name }}
    </span>
  </List>
</template>
```

## 八、解构陷阱与解决方案

```vue
<!-- 陷阱：子组件未传某个属性时解构得到 undefined -->
<template>
  <UserCard v-slot="{ user, role }">
    <!-- 如果子组件没传 role，role 是 undefined -->
    <p>{{ user.name }} - {{ role }}</p>
  </UserCard>
</template>

<!-- 解决：设置默认值 -->
<template>
  <UserCard v-slot="{ user, role = 'guest' }">
    <p>{{ user.name }} - {{ role }}</p>
  </UserCard>
</template>

<!-- 解决：使用可选链 -->
<template>
  <UserCard v-slot="{ user, role }">
    <p>{{ user.name }} - {{ role ?? 'guest' }}</p>
  </UserCard>
</template>
```
