# 嵌套插槽

## 一、概念说明
插槽可以**多层嵌套传递**：父组件传给子组件的插槽内容中，可以再包含对更深层子组件的插槽使用。这在多层组件嵌套中非常有用。

## 二、具体用法

### 2.1 三层嵌套
```vue
<!-- 最外层: AppLayout.vue -->
<template>
  <div class="app">
    <header><slot name="header" /></header>
    <main><slot /></main>
  </div>
</template>
```

```vue
<!-- 中间层: PageLayout.vue -->
<template>
  <AppLayout>
    <template #header>
      <slot name="page-header" />  <!-- 透传插槽 -->
    </template>
    <slot />  <!-- 透传默认插槽 -->
  </AppLayout>
</template>
```

```vue
<!-- 使用 -->
<template>
  <PageLayout>
    <template #page-header>
      <h1>页面标题</h1>
    </template>
    <p>页面内容</p>
  </PageLayout>
</template>
```

### 2.2 透传作用域插槽
```vue
<!-- Wrapper.vue -->
<template>
  <DataTable :data="data">
    <!-- 透传所有插槽 -->
    <template v-for="(_, name) in $slots" #[name]="slotProps">
      <slot :name="name" v-bind="slotProps || {}" />
    </template>
  </DataTable>
</template>
```

### 2.3 手动透传特定插槽
```vue
<template>
  <div class="wrapper">
    <ChildComponent>
      <template #header>
        <slot name="child-header" />
      </template>
      <template #default="childSlotProps">
        <slot v-bind="childSlotProps" />
      </template>
    </ChildComponent>
  </div>
</template>
```

### 2.4 渲染函数中透传插槽
```js
import { h } from 'vue'

export default {
  setup(props, { slots }) {
    return () => h(ChildComponent, null, {
      // 透传所有插槽
      ...slots
    })
  }
}
```

## 三、注意事项与常见陷阱
- 多层嵌套的插槽内容在**最外层父组件**的作用域中编译
- 透传插槽时注意作用域数据的传递
- 过深的插槽嵌套会降低可读性，考虑扁平化
- 可以用 `v-for` 循环 `$slots` 自动透传所有插槽
