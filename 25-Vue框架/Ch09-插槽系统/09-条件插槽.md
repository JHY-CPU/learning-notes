# 条件插槽

## 一、概念说明
通过 `$slots`（选项式 API）或 `useSlots()`（组合式 API）可以**检查插槽是否存在**，从而根据是否提供了插槽内容来条件性地渲染不同结构。

## 二、具体用法

### 2.1 使用 useSlots 检查
```vue
<script setup>
import { useSlots } from 'vue'

const slots = useSlots()

// 检查插槽是否存在
const hasHeader = !!slots.header
const hasFooter = !!slots.footer
const hasDefault = !!slots.default
</script>

<template>
  <div class="card">
    <header v-if="hasHeader">
      <slot name="header" />
    </header>

    <main>
      <slot />
    </main>

    <footer v-if="hasFooter">
      <slot name="footer" />
    </footer>
  </div>
</template>
```

### 2.2 条件渲染不同布局
```vue
<script setup>
import { useSlots } from 'vue'
const slots = useSlots()
</script>

<template>
  <!-- 有侧边栏时用双栏布局，否则用单栏 -->
  <div :class="slots.sidebar ? 'two-column' : 'single-column'">
    <main>
      <slot />
    </main>
    <aside v-if="slots.sidebar">
      <slot name="sidebar" />
    </aside>
  </div>
</template>
```

### 2.3 检查作用域插槽
```vue
<script setup>
import { useSlots } from 'vue'
const slots = useSlots()
</script>

<template>
  <ul>
    <li v-for="item in items" :key="item.id">
      <!-- 如果父组件提供了自定义渲染，用插槽；否则用默认渲染 -->
      <slot v-if="slots.default" :item="item" />
      <span v-else>{{ item.name }}</span>
    </li>
  </ul>
</template>
```

### 2.4 在组合式函数中检查
```js
export function useSlotChecker() {
  const slots = useSlots()

  return {
    hasSlot: (name) => !!slots[name],
    hasDefaultSlot: () => !!slots.default,
    hasAnyNamedSlot: () => Object.keys(slots).some(k => k !== 'default')
  }
}
```

## 三、注意事项与常见陷阱
- `$slots` 只能检测插槽是否在**父组件模板中声明**，不检测是否为空
- `useSlots()` 必须在 `setup()` 中调用
- 检查结果不是响应式的：在渲染期间确定后不会变化
- 只有默认插槽需要特殊检查方式：`!!slots.default`
