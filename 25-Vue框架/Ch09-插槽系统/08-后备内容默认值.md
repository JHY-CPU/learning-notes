# 后备内容（默认值）

## 一、概念说明
`<slot>` 标签内部的内容是**后备内容（fallback content）**。当父组件没有提供对应插槽内容时，会显示这些默认值。

## 二、具体用法

### 2.1 默认插槽后备内容
```vue
<!-- 子组件 Button.vue -->
<template>
  <button class="btn">
    <slot>
      <!-- 父组件没传内容时显示 -->
      <span>点击我</span>
    </slot>
  </button>
</template>
```

```vue
<!-- 父组件 -->
<template>
  <Button />            <!-- 显示: 点击我 -->
  <Button>保存</Button> <!-- 显示: 保存 -->
</template>
```

### 2.2 具名插槽后备内容
```vue
<!-- 子组件 Card.vue -->
<template>
  <div class="card">
    <header>
      <slot name="header">默认标题</slot>
    </header>
    <main>
      <slot>默认内容</slot>
    </main>
    <footer>
      <slot name="footer">
        <button>默认按钮</button>
      </slot>
    </footer>
  </div>
</template>
```

### 2.3 作用域插槽的后备内容
```vue
<!-- 子组件 -->
<template>
  <ul>
    <li v-for="item in items" :key="item.id">
      <slot :item="item">
        <!-- 带数据的后备内容 -->
        {{ item.name }} - {{ item.price }}元
      </slot>
    </li>
  </ul>
</template>
```

```vue
<!-- 父组件只覆盖部分项目 -->
<template>
  <ProductList :items="products">
    <template #default="{ item }">
      <!-- 只有传了内容才覆盖默认渲染 -->
      <strong>{{ item.name }}</strong>
    </template>
  </ProductList>
</template>
```

### 2.4 条件性显示后备内容
```vue
<script setup>
// 子组件内部可以检查是否有插槽内容
import { useSlots } from 'vue'
const slots = useSlots()
const hasCustomContent = !!slots.default
</script>

<template>
  <div>
    <slot v-if="hasCustomContent" />
    <p v-else>默认显示内容</p>
  </div>
</template>
```

## 三、注意事项与常见陷阱
- 后备内容只在**父组件未提供任何内容**时显示
- 即使父组件传了空内容（如 `<Button></Button>`），也会覆盖后备
- 作用域插槽的后备内容也可以访问子组件传递的数据
- 后备内容可以是任意复杂的模板
