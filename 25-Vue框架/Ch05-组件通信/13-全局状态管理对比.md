# 全局状态管理对比

## 一、概念说明
Vue 生态中有多种全局状态管理方案，主要包括 Pinia、Vuex（已停止维护）、事件总线（mitt）和 provide/inject。选择合适的方案对项目可维护性至关重要。

## 二、方案对比

| 特性 | Pinia | Event Bus | Provide/Inject |
|------|-------|-----------|----------------|
| 官方推荐 | 是 | 否 | 是（有限场景）|
| DevTools | 完整支持 | 无 | 无 |
| 类型安全 | 优秀 | 差 | 一般 |
| 学习成本 | 低 | 低 | 低 |
| 适用规模 | 中大型 | 小型 | 小中型 |
| 数据追踪 | 可追踪 | 难追踪 | 隐式依赖 |
| SSR 支持 | 完整 | 无 | 部分 |

### 2.1 Pinia 示例
```js
import { defineStore } from 'pinia'
export const useAppStore = defineStore('app', {
  state: () => ({ theme: 'light' }),
  actions: {
    toggleTheme() {
      this.theme = this.theme === 'light' ? 'dark' : 'light'
    }
  }
})
```

### 2.2 Provide/Inject 示例
```vue
<script setup>
import { ref, provide } from 'vue'
const theme = ref('light')
provide('theme', theme)
</script>
```

### 2.3 Event Bus 示例
```js
import mitt from 'mitt'
export const bus = mitt()
bus.emit('theme-change', 'dark')
```

### 2.4 Composable 共享状态
```js
// 模块级 ref，所有组件共享
import { ref } from 'vue'
const sharedState = ref({ theme: 'light' })
export const useShared = () => ({ state: sharedState })
```

## 三、选择决策树

```
项目规模？
  → 小型（< 5 页面）: provide/inject 或 composable
  → 中型（5-20 页面）: Pinia
  → 大型（> 20 页面）: Pinia + 模块化

是否需要 DevTools？
  → 是: Pinia
  → 否: 均可

是否需要 SSR？
  → 是: Pinia
  → 否: 均可
```

## 四、注意事项与常见陷阱

- 项目超过 5 个页面建议直接使用 Pinia
- provide/inject 适合组件库开发，不适合业务状态管理
- 事件总线已被 Vue 官方明确不推荐，应尽早迁移
- Vuex 4 仍然可用但不再维护，新项目请使用 Pinia
- Composable 共享状态适合轻量场景，但缺乏 DevTools 支持
- 不要混用多种状态管理方案，保持一致性
