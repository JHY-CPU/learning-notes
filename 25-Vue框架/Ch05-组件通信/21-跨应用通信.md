# 跨应用通信

## 一、概念说明
在**微前端**架构或 iframe 嵌入场景中，需要在不同的 Vue 应用之间通信。常用方式包括 `postMessage`、共享存储和自定义事件。

## 二、具体用法

### 2.1 postMessage 基础
```vue
<!-- 主应用 -->
<script setup>
import { onMounted, onUnmounted } from 'vue'

function handleMessage(event) {
  if (event.origin !== 'https://sub-app.com') return
  console.log('收到子应用消息:', event.data)
}

onMounted(() => window.addEventListener('message', handleMessage))
onUnmounted(() => window.removeEventListener('message', handleMessage))

function sendToSubApp(iframe, data) {
  iframe.contentWindow.postMessage(data, 'https://sub-app.com')
}
</script>
```

### 2.2 子应用接收消息
```vue
<script setup>
import { onMounted, onUnmounted } from 'vue'

function handleMessage(event) {
  if (event.data.type === 'THEME_CHANGE') {
    document.body.classList.toggle('dark')
  }
}

onMounted(() => window.addEventListener('message', handleMessage))
onUnmounted(() => window.removeEventListener('message', handleMessage))

function notifyParent() {
  window.parent.postMessage({ type: 'READY' }, '*')
}
</script>
```

### 2.3 BroadcastChannel
```js
const channel = new BroadcastChannel('app-sync')

// 应用 A
channel.postMessage({ type: 'UPDATE', payload: { theme: 'dark' } })

// 应用 B
channel.onmessage = (event) => {
  console.log('同步数据:', event.data)
}
```

### 2.4 封装为组合式函数
```js
// composables/useCrossApp.js
import { onMounted, onUnmounted } from 'vue'

export function useCrossApp(handler) {
  onMounted(() => window.addEventListener('message', handler))
  onUnmounted(() => window.removeEventListener('message', handler))

  const send = (data, targetOrigin = '*') => {
    window.parent.postMessage(data, targetOrigin)
  }

  return { send }
}
```

## 三、常见用例

| 场景 | 方式 |
|------|------|
| iframe 通信 | postMessage |
| 同源多标签页 | BroadcastChannel |
| 微前端 | 框架提供的通信机制 |
| Web Worker | postMessage |

## 四、注意事项与常见陷阱

- **始终验证 `event.origin`**，防止 XSS 攻击
- `postMessage` 的数据会被结构化克隆，不能传递函数
- iframe 中的 Vue 应用是独立的，有自己的响应式系统
- 微前端框架（qiankun、micro-app）提供了更高级的通信机制
- 使用 `structuredClone` 处理复杂数据结构
- 避免传递大量数据，考虑使用 URL 参数或共享存储
