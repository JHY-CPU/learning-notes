# 通信调试技巧

## 一、概念说明
当组件通信出现问题时，需要借助工具和技巧来追踪数据流、事件触发和状态变化。Vue DevTools 是最主要的调试工具。

## 二、具体用法

### 2.1 Vue DevTools 追踪组件树
```
安装 Vue DevTools 浏览器扩展后:
1. 打开开发者工具 → Vue 标签页
2. 查看组件树结构
3. 点击组件查看 props、emit、data
4. 实时修改组件状态进行测试
```

### 2.2 Pinia 时间旅行调试
```js
// stores/counter.js
export const useCounterStore = defineStore('counter', () => {
  const count = ref(0)

  // 在 DevTools 中可以看到每次状态变化
  const increment = () => { count.value++ }

  return { count, increment }
})
```
DevTools 中 Pinia 面板可以查看每一次 mutation，支持**时间旅行**回到任意状态。

### 2.3 使用 console.trace 追踪 emit 来源
```vue
<script setup>
const emit = defineEmits(['change'])

function handleChange(value) {
  console.trace('change 事件触发')  // 打印完整调用栈
  emit('change', value)
}
</script>
```

### 2.4 自定义调试 Hook
```js
// composables/useDebugEmit.js
export function useDebugEmit(emit, componentName) {
  return (event, ...args) => {
    console.log(`[${componentName}] emit: ${event}`, args)
    return emit(event, ...args)
  }
}
```

## 三、注意事项与常见陷阱
- Vue DevTools 在生产环境会被自动禁用
- 使用 `__VUE_PROD_DEVTOOLS__` 环境变量强制启用生产调试
- 过度使用 console.log 会影响性能，调试完记得移除
- Pinia 的 `$subscribe` 可以作为轻量级的状态变化日志
