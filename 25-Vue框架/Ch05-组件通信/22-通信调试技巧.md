# 通信调试技巧

## 一、概念说明
当组件通信出现问题时，需要借助工具和技巧来追踪数据流、事件触发和状态变化。Vue DevTools 是最主要的调试工具。

## 二、具体用法

### 2.1 Vue DevTools 追踪
```
1. 打开开发者工具 → Vue 标签页
2. 查看组件树结构
3. 点击组件查看 props、emit、data
4. 实时修改组件状态进行测试
```

### 2.2 Pinia 时间旅行调试
```js
// 在 DevTools 中可以看到每次状态变化
// Pinia 面板支持时间旅行回到任意状态

store.$subscribe((mutation, state) => {
  console.log('状态变化:', mutation.type)
  console.log('事件:', mutation.events)
})
```

### 2.3 使用 console.trace 追踪 emit
```vue
<script setup>
const emit = defineEmits(['change'])

function handleChange(value) {
  console.trace('change 事件触发')
  emit('change', value)
}
</script>
```

### 2.4 自定义调试 Hook
```js
// composables/useDebugEmit.js
export function useDebugEmit(emit, componentName) {
  return (event, ...args) => {
    console.log(`[${componentName}] emit: ${event}`, args)
    return emit(event, ...args)
  }
}
```

```vue
<script setup>
import { useDebugEmit } from './composables/useDebugEmit'

const rawEmit = defineEmits(['change', 'submit'])
const emit = useDebugEmit(rawEmit, 'MyComponent')
// 现在所有 emit 调用都会被记录
</script>
```

### 2.5 Props 变化追踪
```vue
<script setup>
import { watch } from 'vue'
const props = defineProps({ data: Object })

watch(
  () => props.data,
  (newVal, oldVal) => {
    console.log('Props data 变化:', { oldVal, newVal })
  },
  { deep: true }
)
</script>
```

### 2.6 提供/注入调试
```vue
<script setup>
import { inject } from 'vue'

const theme = inject('theme', undefined)
if (!theme) {
  console.warn('theme 未找到提供者，请确认祖先组件是否 provide 了 theme')
}
</script>
```

## 三、常见用例

| 问题 | 调试方法 |
|------|---------|
| Props 不更新 | watch props 变化 |
| 事件未触发 | console.trace 追踪 |
| Pinia 状态异常 | DevTools 时间旅行 |
| provide 未找到 | 检查祖先组件 |

## 四、注意事项与常见陷阱

- Vue DevTools 在生产环境会被自动禁用
- 使用 `__VUE_PROD_DEVTOOLS__` 环境变量强制启用生产调试
- 过度使用 console.log 会影响性能，调试完记得移除
- Pinia 的 `$subscribe` 可以作为轻量级的状态变化日志
- 使用 `performance.mark` 和 `performance.measure` 进行性能调试
- Vue 3.4+ 的 DevTools 支持更详细的事件追踪
