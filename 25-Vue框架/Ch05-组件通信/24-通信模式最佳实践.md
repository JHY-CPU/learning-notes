# 通信模式最佳实践

## 一、概念说明
良好的通信模式可以提高代码的可读性、可维护性和可测试性。以下是经过实践验证的最佳通信模式。

## 二、最佳实践

### 2.1 单向数据流原则
```vue
<template>
  <FilterBar :filters="filters" @update:filters="handleFilterChange" />
  <ItemList :items="filteredItems" @select="handleSelect" />
  <DetailPanel :selected="selectedItem" @close="selectedItem = null" />
</template>
```

数据从父到子流动，事件从子到父冒泡。

### 2.2 命名规范
```vue
<!-- Props: camelCase -->
<script setup>
defineProps<{ userName: string; itemCount: number }>()
</script>

<!-- 事件名: kebab-case -->
<template>
  <Child @user-login="onLogin" @item-select="onSelect" />
</template>

<!-- v-model: camelCase -->
<template>
  <Input v-model:searchText="query" />
</template>
```

### 2.3 组合式函数封装通信逻辑
```js
// composables/useSelection.js
import { ref } from 'vue'

export function useSelection(items) {
  const selected = ref(null)
  const select = (item) => { selected.value = item }
  const clear = () => { selected.value = null }
  const isSelected = (item) => selected.value?.id === item.id
  return { selected, select, clear, isSelected }
}
```

### 2.4 合理使用 readonly 保护
```vue
<script setup>
import { provide, reactive, readonly } from 'vue'

const state = reactive({ count: 0 })
provide('state', readonly(state))  // 后代只读
provide('actions', { increment: () => state.count++ })  // 通过 action 修改
</script>
```

### 2.5 哑组件模式
```vue
<!-- 哑组件：只负责展示和触发事件 -->
<script setup>
defineProps({
  items: Array,
  selectedId: [Number, String]
})
const emit = defineEmits(['select', 'delete'])
</script>

<template>
  <div
    v-for="item in items"
    :key="item.id"
    :class="{ active: item.id === selectedId }"
    @click="emit('select', item)"
  >
    {{ item.name }}
    <button @click.stop="emit('delete', item.id)">删除</button>
  </div>
</template>
```

### 2.6 避免反模式

```vue
<!-- ❌ 反模式：直接修改 props -->
<script setup>
const props = defineProps({ count: Number })
// props.count++  // 错误
</script>

<!-- ❌ 反模式：在模板中创建对象 -->
<Child :config="{ show: true }" />  // 每次渲染新对象

<!-- ❌ 反模式：过度使用全局状态 -->
// 所有数据都放 Pinia

<!-- ✅ 正确做法 -->
<script setup>
import { computed } from 'vue'
const config = computed(() => ({ show: true }))  // 缓存
</script>
```

## 三、常见用例

| 模式 | 适用场景 |
|------|---------|
| Props + Emit | 父子直接交互 |
| v-model | 表单控件 |
| 作用域插槽 | 列表自定义渲染 |
| Provide/Inject | 组件库、主题系统 |
| Pinia | 全局业务状态 |

## 四、注意事项与常见陷阱

- 数据向下流动，事件向上冒泡，保持单向数据流
- 组件应尽量"哑"——只负责展示和触发事件
- 业务逻辑集中在 Store 或组合式函数中
- 命名应清晰表达意图，避免 `data1`、`handleStuff` 之类
- 使用 TypeScript 获得编译时类型安全
- 写组件时先定义 props 和 events 接口，再实现逻辑
