# 事件验证

## 一、概念说明
`defineEmits` 支持对触发的事件进行**参数验证**，类似于 props 的类型检查。当事件被触发时，如果参数不符合验证规则，Vue 会在控制台发出警告。

## 二、具体用法

### 2.1 对象形式的事件验证
```vue
<script setup>
const emit = defineEmits({
  submit: null,  // 无参数事件，不需要验证

  change: (value) => {
    if (typeof value !== 'string') {
      console.warn('change 事件的参数必须是字符串')
      return false
    }
    return true
  },

  updateUser: ({ name, age }) => {
    return typeof name === 'string' && typeof age === 'number'
  }
})
</script>
```

### 2.2 TypeScript 类型化验证
```vue
<script setup>
const emit = defineEmits<{
  change: [value: string]
  submit: [data: { name: string; age: number }]
  close: []
}>()

emit('change', 123)     // 编译错误
emit('change', 'hello')  // 正确
</script>
```

### 2.3 实际应用场景
```vue
<script setup>
const emit = defineEmits({
  'page-change': (page) => {
    const valid = Number.isInteger(page) && page > 0
    if (!valid) console.error('page 必须是正整数')
    return valid
  },

  'size-change': (size) => {
    return ['sm', 'md', 'lg'].includes(size)
  }
})
</script>
```

### 2.4 结合 props 验证的完整模式
```vue
<script setup>
defineProps({
  items: {
    type: Array,
    required: true,
    validator: (val) => val.every(item => item.id && item.name)
  }
})

const emit = defineEmits({
  select: (item) => {
    if (!item?.id) {
      console.warn('select 事件必须传递有效的 item')
      return false
    }
    return true
  }
})
</script>
```

## 三、常见用例

| 场景 | 验证内容 |
|------|---------|
| 分页事件 | 页码必须为正整数 |
| 排序事件 | 排序字段必须为允许值 |
| 选择事件 | 选中项必须有 id |
| 输入事件 | 值必须为字符串 |

## 四、注意事项与常见陷阱

- 验证函数返回 `true` 表示通过，`false` 表示失败
- TypeScript 用户建议优先使用类型声明形式
- 运行时验证与类型检查是互补的，不冲突
- 验证失败只产生警告，不会阻止事件触发
- 对象形式和类型声明形式不能同时使用
- 验证逻辑应保持简单，复杂验证在父组件处理
