# 事件验证

## 一、概念说明
`defineEmits` 支持对触发的事件进行**参数验证**，类似于 props 的类型检查。当事件被触发时，如果参数不符合验证规则，Vue 会在控制台发出警告。

## 二、具体用法

### 2.1 对象形式的事件验证
```vue
<script setup>
const emit = defineEmits({
  // 无参数事件
  submit: null,

  // 带验证的事件
  change: (value) => {
    if (typeof value !== 'string') {
      console.warn('change 事件的参数必须是字符串')
      return false
    }
    return true
  },

  // 验证多个参数
  updateUser: ({ name, age }) => {
    return typeof name === 'string' && typeof age === 'number'
  }
})
</script>
```

### 2.2 TypeScript 类型化验证
```vue
<script setup>
const emit = defineEmits<{
  change: [value: string]
  submit: [data: { name: string; age: number }]
  close: []
}>()

// TypeScript 编译时检查
emit('change', 123)  // 类型错误！
emit('change', 'hello')  // 正确
</script>
```

### 2.3 实际应用场景
```vue
<script setup>
const emit = defineEmits({
  'page-change': (page) => {
    const valid = Number.isInteger(page) && page > 0
    if (!valid) console.error('page 必须是正整数')
    return valid
  }
})

function goToPage(page) {
  emit('page-change', page)
}
</script>
```

## 三、注意事项与常见陷阱
- 验证函数返回 `true` 表示通过，`false` 表示失败
- TypeScript 用户建议优先使用类型声明形式
- 运行时验证与类型检查是互补的，不冲突
- 验证失败只产生警告，不会阻止事件触发
