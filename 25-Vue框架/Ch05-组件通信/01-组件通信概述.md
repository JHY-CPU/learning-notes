# 组件通信概述

## 一、概念说明
在 Vue 应用中，组件是独立的、可复用的 UI 单元。由于组件的封装性，它们之间需要通过特定的方式进行数据交换，这就是**组件通信**。

组件之间的关系可以分为三类：父子关系、兄弟关系、跨层级关系。不同的关系适用不同的通信方式。

## 二、通信方式总览

### 2.1 父子组件通信
| 方向 | 方式 | 说明 |
|------|------|------|
| 父 -> 子 | Props | 父组件通过属性传递数据 |
| 子 -> 父 | Emit | 子组件通过事件向父组件发送消息 |
| 双向 | v-model | 父子双向数据绑定 |

```vue
<template>
  <Child :msg="message" @update="handleUpdate" />
</template>
<script setup>
import { ref } from 'vue'
const message = ref('你好')
const handleUpdate = (val) => { message.value = val }
</script>
```

### 2.2 兄弟组件通信
通过共同的父组件中转，或使用 Pinia 状态管理。

### 2.3 跨层级通信
```vue
<script setup>
import { provide, ref } from 'vue'
const theme = ref('dark')
provide('theme', theme)
</script>
```

### 2.4 任意组件通信
- **Pinia 全局状态管理**：适用于中大型项目
- **事件总线（mitt）**：简单但不推荐用于大型项目
- **模板 ref**：直接访问子组件实例

## 三、通信方式选择指南

```
是否父子关系？
  → 是：Props / Emit / v-model
  → 否：是否跨 2-3 层？
    → 是：Provide / Inject
    → 否：是否全局共享？
      → 是：Pinia
      → 否：Composable 共享状态
```

## 四、注意事项与常见陷阱

- Vue 遵循**单向数据流**，不要直接修改 props
- 优先使用 Pinia 进行跨层级通信，而非层层传递
- 避免过度使用事件总线，会导致数据流向难以追踪
- 选择通信方式时，优先考虑 Props/Emit，再考虑 provide/inject 和 Pinia
- 组件间耦合度越低越好，避免直接访问其他组件的内部状态

## 五、各通信方式完整对比

| 方式 | 方向 | 适用关系 | 优点 | 缺点 |
| --- | --- | --- | --- | --- |
| Props | 父 -> 子 | 父子 | 简单直接、类型安全 | 层层传递繁琐 |
| Emit | 子 -> 父 | 父子 | 解耦、可验证 | 只能通知父组件 |
| v-model | 双向 | 父子 | 双向绑定 | 一个组件只能有一个 |
| provide/inject | 祖先 -> 后代 | 跨层 | 跳过中间层 | 依赖关系不明确 |
| Pinia | 任意 | 全局 | 集中式管理 | 引入额外依赖 |
| mitt | 任意 | 全局 | 轻量 | 难追踪、易内存泄漏 |
| 模板 ref | 父 -> 子实例 | 父子 | 直接访问方法 | 强耦合 |

## 六、完整示例：Pinia 状态管理

```vue
<!-- stores/user.js -->
import { defineStore } from 'pinia'
export const useUserStore = defineStore('user', {
  state: () => ({ name: '', isLoggedIn: false }),
  actions: {
    async login(credentials) {
      this.isLoggedIn = true
      this.name = credentials.username
    }
  }
})

<!-- 组件中使用 -->
<script setup>
import { useUserStore } from '@/stores/user'
const user = useUserStore()
</script>
<template>
  <span v-if="user.isLoggedIn">{{ user.name }}</span>
</template>
```

## 七、通信方式选择流程

```
1. 需要传数据给子组件？ -> Props
2. 子组件需要通知父组件？ -> Emit / v-model
3. 跨 2-3 层传递？ -> Provide/Inject
4. 全局共享状态？ -> Pinia
5. 兄弟组件通信？ -> 通过 Pinia 或共同父组件
6. 需要直接调用子组件方法？ -> 模板 ref（尽量避免）
```
