# 组件解耦设计

## 一、概念说明
组件解耦是指组件之间**减少直接依赖**，通过接口抽象实现通信。解耦的组件更易测试、复用和维护。

## 二、具体用法

### 2.1 接口抽象：Slots 替代 Props
```vue
<!-- 耦合：组件知道子组件结构 -->
<Table :columns="cols" :data="rows" />

<!-- 解耦：父组件控制渲染 -->
<Table :data="rows">
  <template #header="{ col }">
    <CustomHeader :col="col" />
  </template>
  <template #cell="{ row, col }">
    <CustomCell :value="row[col.key]" />
  </template>
</Table>
```

### 2.2 事件接口抽象
```vue
<!-- 耦合：子组件直接依赖具体业务 -->
<script setup>
const emit = defineEmits(['userLogin', 'userLogout', 'userRegister'])
</script>

<!-- 解耦：统一事件接口 -->
<script setup>
const emit = defineEmits<{
  action: [type: string, payload: Record<string, any>]
}>()

function login() { emit('action', 'login', { provider: 'oauth' }) }
function logout() { emit('action', 'logout', {}) }
</script>
```

### 2.3 组合式函数解耦
```js
// 逻辑与组件分离
export function useSearch(items, options = {}) {
  const query = ref('')
  const results = computed(() => {
    return items.value.filter(item =>
      item.name.includes(query.value)
    )
  })
  return { query, results }
}

// 任意组件都可以使用
const { query, results } = useSearch(myItems)
```

### 2.4 依赖注入接口化
```ts
// 定义接口
interface AppServices {
  notify: (msg: string) => void
  navigate: (path: string) => void
}

const SERVICES_KEY: InjectionKey<AppServices> = Symbol('services')

// 后代组件只依赖接口，不依赖具体实现
const services = inject(SERVICES_KEY)
services?.notify('操作成功')
```

## 三、注意事项与常见陷阱
- 过度解耦会增加复杂度，找到合适的平衡点
- 组件库应高度解耦，业务组件可适当耦合
- 使用 TypeScript 接口约束依赖关系
- 解耦的目标是提高可测试性，而非追求理论上的完美
