# 插槽通信

## 一、概念说明
**作用域插槽**（Scoped Slot）允许子组件将数据暴露给父组件的插槽内容。父组件在插槽中可以访问子组件传递的数据，实现子到父的数据传递，同时保持灵活的内容渲染控制。

## 二、具体用法

### 2.1 子组件通过插槽暴露数据
```vue
<!-- UserList.vue -->
<template>
  <ul>
    <li v-for="(user, index) in users" :key="user.id">
      <slot :user="user" :index="index">
        {{ user.name }}
      </slot>
    </li>
  </ul>
</template>
<script setup>
defineProps({
  users: Array
})
</script>
```

### 2.2 父组件使用插槽数据
```vue
<template>
  <UserList :users="list">
    <template v-slot="{ user, index }">
      <span>{{ index + 1 }}. {{ user.name }} ({{ user.age }}岁)</span>
    </template>
  </UserList>
</template>
```

### 2.3 具名作用域插槽
```vue
<!-- 子组件 -->
<template>
  <header>
    <slot name="header" :title="title" :count="count" />
  </header>
  <main>
    <slot :items="items" />
  </main>
</template>
```

```vue
<!-- 父组件 -->
<template>
  <MyLayout>
    <template #header="{ title, count }">
      <h1>{{ title }} ({{ count }})</h1>
    </template>
    <template #default="{ items }">
      <div v-for="item in items" :key="item.id">{{ item.name }}</div>
    </template>
  </MyLayout>
</template>
```

### 2.4 列表组件通用模式
```vue
<!-- DataTable.vue -->
<template>
  <table>
    <tr v-for="(row, i) in data" :key="i">
      <td v-for="col in columns" :key="col.key">
        <slot :name="col.key" :row="row" :value="row[col.key]">
          {{ row[col.key] }}
        </slot>
      </td>
    </tr>
  </table>
</template>
```

## 三、常见用例

| 场景 | 说明 |
|------|------|
| 列表渲染自定义 | 子组件提供数据，父控制渲染 |
| 表格列自定义 | 每列渲染不同格式 |
| 表单组件 | 暴露验证状态给插槽 |
| 通用容器组件 | 暴露内部状态给插槽内容 |

## 四、注意事项与常见陷阱

- 作用域插槽使子组件成为"渲染器"，父组件控制渲染内容
- 解构语法 `v-slot="{ user }"` 更简洁易读
- 默认插槽和具名插槽都可以传递数据
- 避免在插槽中传递过多数据，保持接口简洁
- 作用域插槽的数据在子组件重新渲染时更新
- 可以用 `#` 简写 `v-slot`（如 `#default` 或 `#="{ user }"`）
