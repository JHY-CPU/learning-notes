# 插槽通信

## 一、概念说明
**作用域插槽**（Scoped Slot）允许子组件将数据暴露给父组件的插槽内容。父组件在插槽中可以访问子组件传递的数据，实现子到父的数据传递，同时保持灵活的内容渲染控制。

## 二、具体用法

### 2.1 子组件通过插槽暴露数据
```vue
<!-- UserList.vue -->
<template>
  <ul>
    <li v-for="user in users" :key="user.id">
      <slot :user="user" :index="index">
        {{ user.name }}  <!-- 默认内容 -->
      </slot>
    </li>
  </ul>
</template>
<script setup>
defineProps<{
  users: Array<{ id: number; name: string; age: number }>
}>()
</script>
```

### 2.2 父组件使用插槽数据
```vue
<template>
  <UserList :users="list">
    <!-- 通过 v-slot 接收子组件传递的数据 -->
    <template v-slot="{ user, index }">
      <span>{{ index + 1 }}. {{ user.name }} ({{ user.age }}岁)</span>
    </template>
  </UserList>
</template>
```

### 2.3 列表组件通用模式
```vue
<!-- DataTable.vue -->
<template>
  <table>
    <tr v-for="(row, i) in data" :key="i">
      <td v-for="col in columns" :key="col.key">
        <slot :name="col.key" :row="row" :value="row[col.key]">
          {{ row[col.key] }}
        </slot>
      </td>
    </tr>
  </table>
</template>
```

## 三、注意事项与常见陷阱
- 作用域插槽使子组件成为"渲染器"，父组件控制渲染内容
- 解构语法 `v-slot="{ user }"` 更简洁易读
- 默认插槽和具名插槽都可以传递数据
- 避免在插槽中传递过多数据，保持接口简洁
