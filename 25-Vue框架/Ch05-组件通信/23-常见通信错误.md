# 常见通信错误

## 一、概念说明
组件通信中容易犯的错误往往源于对 Vue 响应式系统和单向数据流的误解。了解这些错误可以帮助我们写出更健壮的代码。

## 二、常见错误与修复

### 2.1 直接修改 Props
```vue
<!-- 错误写法 -->
<script setup>
const props = defineProps<{ count: number }>()
function increment() {
  props.count++  // ❌ 警告：不要直接修改 props
}
</script>

<!-- 正确写法 -->
<script setup>
import { ref, watch } from 'vue'
const props = defineProps<{ count: number }>()
const localCount = ref(props.count)
watch(() => props.count, (v) => { localCount.value = v })
</script>
```

### 2.2 循环更新
```vue
<!-- 错误：父改子 → 子改父 → 父改子 → 无限循环 -->
<script setup>
const props = defineProps<{ value: number }>()
const emit = defineEmits(['update:value'])

watch(() => props.value, (val) => {
  emit('update:value', val + 1)  // ❌ 每次 emit 又触发 watch
})
</script>

<!-- 正确：添加条件判断 -->
<script setup>
watch(() => props.value, (val) => {
  if (val < 100) {
    emit('update:value', val + 1)
  }
})
</script>
```

### 2.3 忘记清理事件监听
```vue
<script setup>
import { onMounted, onUnmounted } from 'vue'

// ❌ 内存泄漏
onMounted(() => {
  window.addEventListener('resize', handleResize)
})

// ✅ 正确清理
onUnmounted(() => {
  window.removeEventListener('resize', handleResize)
})
</script>
```

### 2.4 异步操作竞态条件
```vue
<script setup>
import { ref } from 'vue'

const data = ref(null)
let currentId = 0

async function fetchData(id) {
  currentId = id
  const result = await api.get(id)
  // ❌ 可能覆盖了更新的请求结果
  // ✅ 检查是否仍然是当前请求
  if (id === currentId) {
    data.value = result
  }
}
</script>
```

## 三、注意事项与常见陷阱
- Props 修改应该通过 emit 让父组件处理
- watch 中触发可能导致循环，务必添加终止条件
- 所有 addEventListener 都应有对应的 removeEventListener
- 异步操作要考虑请求的先后顺序和取消机制
