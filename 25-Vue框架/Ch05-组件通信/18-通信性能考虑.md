# 通信性能考虑

## 一、概念说明
不恰当的组件通信方式可能导致**不必要的重新渲染**，影响应用性能。了解每种通信方式的性能特性，可以帮助我们做出更好的设计决策。

## 二、具体用法

### 2.1 避免模板中创建新对象
```vue
<!-- 错误：每次渲染创建新对象 -->
<ChildComponent :config="{ show: true, size: 'large' }" />

<!-- 正确：使用 shallowRef 或外部变量 -->
<script setup>
import { shallowRef } from 'vue'
const config = shallowRef({ show: true, size: 'large' })
</script>
<ChildComponent :config="config" />
```

### 2.2 使用 shallowRef 减少深度响应
```vue
<script setup>
import { shallowRef } from 'vue'

const largeList = shallowRef(getLargeData())

function updateData() {
  largeList.value = getNewData()
}
</script>
```

### 2.3 按需传递 vs 全量传递
```vue
<!-- 错误：传递整个大对象 -->
<UserCard :user="fullUserData" />

<!-- 正确：只传递需要的字段 -->
<UserCard :name="user.name" :avatar="user.avatar" />
```

### 2.4 使用 computed 缓存派生数据
```vue
<script setup>
import { computed } from 'vue'
const props = defineProps({ items: Array })

const total = computed(() => props.items.reduce((a, b) => a + b, 0))
</script>
```

### 2.5 使用 v-once 静态内容
```vue
<template>
  <!-- 不会再变化的内容用 v-once -->
  <header v-once>{{ staticTitle }}</header>
</template>
```

## 三、性能优化清单

| 问题 | 解决方案 |
|------|---------|
| 模板内创建对象/数组 | 提取到 computed 或 ref |
| 大型数据深度响应 | shallowRef / shallowReactive |
| 传递不必要的数据 | 只传递需要的字段 |
| 子组件不必要渲染 | 使用 computed 缓存 |
| 静态内容重新渲染 | v-once |

## 四、注意事项与常见陷阱

- 避免在模板中创建新对象/数组作为 prop 值
- 大型数据集考虑使用 `shallowRef` 或 `shallowReactive`
- 超长列表考虑虚拟滚动而非全部传递
- props 变化会触发子组件重新渲染，即使子组件未使用该 prop
- 使用 Vue DevTools 的 Performance 面板检测不必要的渲染
- `v-memo`（Vue 3.2+）可以缓存子树避免重新渲染
