# 兄弟组件通信

## 一、概念说明
兄弟组件是拥有**共同父组件**的两个组件。它们之间没有直接的通信渠道，需要通过父组件中转，或者使用全局状态管理（Pinia）。

## 二、具体用法

### 2.1 通过父组件中转
```vue
<!-- 父组件 Parent.vue -->
<template>
  <ComponentA @send="handleData" />
  <ComponentB :data="sharedData" />
</template>
<script setup>
import { ref } from 'vue'
const sharedData = ref(null)
const handleData = (data) => { sharedData.value = data }
</script>
```

### 2.2 通过 Pinia 共享状态
```js
// stores/shared.js
import { defineStore } from 'pinia'
import { ref } from 'vue'

export const useSharedStore = defineStore('shared', () => {
  const selectedItem = ref(null)
  const setSelected = (item) => { selectedItem.value = item }
  return { selectedItem, setSelected }
})
```

```vue
<!-- ComponentA.vue -->
<script setup>
import { useSharedStore } from '@/stores/shared'
const store = useSharedStore()
</script>
<template>
  <button @click="store.setSelected({ id: 1 })">选择</button>
</template>
```

### 2.3 使用 provide/inject
```vue
<script setup>
import { ref, provide } from 'vue'
const sharedState = ref(null)
provide('sharedState', sharedState)
</script>
```

### 2.4 封装共享逻辑
```js
// composables/useSharedSelection.js
import { ref } from 'vue'

const selected = ref(null)

export function useSharedSelection() {
  const select = (item) => { selected.value = item }
  const clear = () => { selected.value = null }
  return { selected, select, clear }
}
```

## 三、常见用例

| 场景 | 推荐方式 |
|------|---------|
| 简单兄弟通信 | 父组件中转 |
| 复杂/多级兄弟通信 | Pinia |
| 跨模块兄弟通信 | Pinia |
| 临时调试 | provide/inject |

## 四、注意事项与常见陷阱

- 父组件中转适合简单场景，但会让父组件承担过多职责
- Pinia 适合复杂或跨多级的兄弟通信
- 避免使用事件总线处理兄弟通信，Pinia 更可维护
- 如果经常需要兄弟通信，考虑是否应该合并为一个组件
- 使用组合式函数共享状态是轻量级替代方案
