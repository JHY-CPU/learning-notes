# 组合式函数通信

## 一、概念说明
**组合式函数**（Composables）可以封装响应式状态和逻辑，多个组件共享同一个组合式函数实例即可实现通信。这是一种轻量级的共享状态方案，适合不需要 Pinia 的简单场景。

## 二、具体用法

### 2.1 模块级共享状态
```js
// composables/useCounter.js
import { ref, readonly } from 'vue'

const count = ref(0)

export function useCounter() {
  const increment = () => count.value++
  const decrement = () => count.value--
  const reset = () => { count.value = 0 }

  return {
    count: readonly(count),
    increment,
    decrement,
    reset
  }
}
```

### 2.2 多组件共享同一状态
```vue
<!-- ComponentA.vue -->
<script setup>
import { useCounter } from '@/composables/useCounter'
const { count, increment } = useCounter()
</script>
<template>
  <button @click="increment">计数: {{ count }}</button>
</template>
```

```vue
<!-- ComponentB.vue -->
<script setup>
import { useCounter } from '@/composables/useCounter'
const { count, reset } = useCounter()
</script>
<template>
  <p>当前计数: {{ count }}</p>
  <button @click="reset">重置</button>
</template>
```

### 2.3 私有状态版本（每次调用创建新实例）
```js
export function usePrivateCounter() {
  const count = ref(0)
  const increment = () => count.value++
  return { count, increment }
}
```

### 2.4 带事件的通知模式
```js
// composables/useEvent.js
const listeners = {}

export function useEvent() {
  const on = (event, fn) => {
    (listeners[event] = listeners[event] || []).push(fn)
  }
  const emit = (event, ...args) => {
    (listeners[event] || []).forEach(fn => fn(...args))
  }
  const off = (event, fn) => {
    if (listeners[event]) {
      listeners[event] = listeners[event].filter(f => f !== fn)
    }
  }
  return { on, emit, off }
}
```

## 三、常见用例

| 场景 | 是否共享 |
|------|---------|
| 全局计数器 | 是（模块级 ref） |
| 表单验证逻辑 | 否（每次调用独立） |
| 主题切换 | 是（模块级 ref） |
| 动画状态 | 否（每次调用独立） |

## 四、注意事项与常见陷阱

- 模块级 ref 是全局共享的，函数内创建的 ref 是每次调用私有的
- 使用 `readonly()` 保护暴露的状态，防止外部意外修改
- 适合简单场景，复杂状态建议直接使用 Pinia
- 组件卸载时共享状态仍然存在，注意是否需要清理
- 没有 DevTools 支持，调试不如 Pinia 方便
- SSR 中模块级状态可能导致请求间状态泄漏
