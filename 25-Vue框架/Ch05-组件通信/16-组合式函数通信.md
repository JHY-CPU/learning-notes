# 组合式函数通信

## 一、概念说明
**组合式函数**（Composables）可以封装响应式状态和逻辑，多个组件共享同一个组合式函数实例即可实现通信。这是一种轻量级的共享状态方案，适合不需要 Pinia 的简单场景。

## 二、具体用法

### 2.1 创建共享状态的 Composable
```js
// composables/useCounter.js
import { ref, readonly } from 'vue'

const count = ref(0)  // 模块级状态，所有组件共享

export function useCounter() {
  const increment = () => count.value++
  const decrement = () => count.value--
  const reset = () => { count.value = 0 }

  return {
    count: readonly(count),  // 只读暴露
    increment,
    decrement,
    reset
  }
}
```

### 2.2 组件 A 使用
```vue
<script setup>
import { useCounter } from '@/composables/useCounter'
const { count, increment } = useCounter()
</script>
<template>
  <button @click="increment">计数: {{ count }}</button>
</template>
```

### 2.3 组件 B 使用（共享同一状态）
```vue
<script setup>
import { useCounter } from '@/composables/useCounter'
const { count, reset } = useCounter()
</script>
<template>
  <p>当前计数: {{ count }}</p>
  <button @click="reset">重置</button>
</template>
```

### 2.4 带私有状态的版本
```js
export function usePrivateCounter() {
  const count = ref(0)  // 每次调用创建新的状态
  const increment = () => count.value++
  return { count, increment }
}
```

## 三、注意事项与常见陷阱
- 模块级 ref 是全局共享的，函数内创建的 ref 是每次调用私有的
- 使用 `readonly()` 保护暴露的状态，防止外部意外修改
- 适合简单场景，复杂状态建议直接使用 Pinia
- 组件卸载时共享状态仍然存在，注意是否需要清理
