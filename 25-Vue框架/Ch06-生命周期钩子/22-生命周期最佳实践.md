# 生命周期最佳实践

## 一、概念说明
在正确的生命周期钩子中做正确的事情，可以提高代码的可预测性和可维护性。以下是经过实践验证的最佳实践总结。

## 二、最佳实践

### 2.1 各钩子的推荐用途
```
setup()        -> 初始化响应式数据、定义方法
onBeforeMount  -> 很少使用，避免操作 DOM
onMounted      -> DOM 操作、API 请求、第三方库初始化
onBeforeUpdate -> 获取更新前的 DOM 状态（少用）
onUpdated      -> 获取更新后的 DOM 状态（少用）
onBeforeUnmount-> 清理定时器、事件监听、销毁实例
onUnmounted    -> 最终清理和日志
```

### 2.2 API 请求放在 onMounted
```vue
<script setup>
import { ref, onMounted } from 'vue'

const data = ref(null)

onMounted(async () => {
  data.value = await fetch('/api/data').then(r => r.json())
})
</script>
```

### 2.3 清理放在 onBeforeUnmount
```vue
<script setup>
import { onMounted, onBeforeUnmount } from 'vue'

let ws = null

onMounted(() => {
  ws = new WebSocket('ws://api.example.com')
})

onBeforeUnmount(() => {
  ws?.close()
})
</script>
```

### 2.4 使用 Composable 封装重复模式
```js
export function useEventListener(target, event, handler) {
  onMounted(() => target.addEventListener(event, handler))
  onBeforeUnmount(() => target.removeEventListener(event, handler))
}
```

## 三、注意事项与常见陷阱
- 每个 `addEventListener` 都应有对应的 `removeEventListener`
- 不要在生命周期中创建不必要的副作用
- `setup()` 中的同步代码在所有钩子之前执行
- Composable 比直接在组件中写生命周期更可复用

## 四、反模式

```vue
<!-- 反模式 1: 在 onUpdated 中修改数据 -->
<script setup>
import { ref, onUpdated } from 'vue'
const count = ref(0)
// 错误：会导致无限循环
onUpdated(() => { count.value++ })
</script>

<!-- 反模式 2: setup 中访问 $refs -->
<script setup>
import { ref } from 'vue'
const el = ref(null)
// 错误：el.value 此时为 null
// console.log(el.value) // null
</script>

<!-- 反模式 3: 忘记清理 -->
<script setup>
import { onMounted } from 'vue'
// 错误：没有对应的 removeEventListener
onMounted(() => {
  window.addEventListener('resize', handleResize)
})
</script>

<!-- 正确做法 -->
<script setup>
import { onMounted, onBeforeUnmount } from 'vue'
function handleResize() { console.log(window.innerWidth) }
onMounted(() => window.addEventListener('resize', handleResize))
onBeforeUnmount(() => window.removeEventListener('resize', handleResize))
</script>
```

## 五、生命周期选择速查表

| 需求 | 使用钩子 | 原因 |
| --- | --- | --- |
| API 请求 | onMounted | DOM 就绪，SSR 兼容 |
| DOM 操作 | onMounted | 保证 DOM 存在 |
| 清理定时器 | onBeforeUnmount | 组件仍可访问 |
| WebSocket 关闭 | onBeforeUnmount | 组件仍可访问 |
| 第三方库初始化 | onMounted | 需要 DOM |
| 第三方库销毁 | onBeforeUnmount | 组件仍可用 |
| 滚动位置恢复 | onActivated | keep-alive 恢复时 |
| 暂停动画 | onDeactivated | keep-alive 失活时 |
| 错误降级 UI | onErrorCaptured | 捕获子组件错误 |
| 性能测量 | onBeforeMount + onMounted | 区间计时 |

## 六、团队编码规范建议

1. 每个组件中的 `onMounted` 和 `onBeforeUnmount` 应成对出现
2. Composable 中使用 `onScopeDispose` 而非 `onUnmounted`
3. 异步请求放在 `onMounted` 中，使用 AbortController 管理
4. 避免在 `onBeforeUpdate` 和 `onUpdated` 中修改响应式数据
5. 调试钩子仅用于开发环境，不要加入业务逻辑
