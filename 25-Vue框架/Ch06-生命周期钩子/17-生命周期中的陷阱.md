# 生命周期中的陷阱

## 一、概念说明
在使用生命周期钩子时，有一些常见的陷阱和错误模式需要避免。了解这些陷阱可以帮助我们写出更稳定、更可预测的代码。

## 二、常见陷阱

### 2.1 在 onUpdated 中修改数据
```vue
<!-- ❌ 错误：导致无限循环 -->
<script setup>
import { ref, onUpdated } from 'vue'
const count = ref(0)

onUpdated(() => {
  count.value++  // 每次更新又触发 onUpdated → 无限循环
})
</script>

<!-- ✅ 正确：使用 watch + 条件 -->
<script setup>
import { ref, watch } from 'vue'
const count = ref(0)

watch(count, (val) => {
  if (val < 10) count.value++
})
</script>
```

### 2.2 访问未挂载的 ref
```vue
<!-- ❌ 错误：onMounted 之前 ref 为 null -->
<script setup>
import { ref } from 'vue'
const el = ref(null)
console.log(el.value)  // null！
</script>

<!-- ✅ 正确：在 onMounted 中访问 -->
<script setup>
import { ref, onMounted } from 'vue'
const el = ref(null)
onMounted(() => {
  console.log(el.value)  // DOM 元素
})
</script>
```

### 2.3 竞态条件
```vue
<script setup>
import { ref, onMounted } from 'vue'
const data = ref(null)

// ❌ 如果快速切换页面，旧请求可能覆盖新结果
onMounted(async () => {
  data.value = await fetchData()
})

// ✅ 使用 AbortController
let controller
onMounted(async () => {
  controller?.abort()
  controller = new AbortController()
  const res = await fetchData({ signal: controller.signal })
  data.value = res
})
</script>
```

### 2.4 忘记清理副作用
```vue
<script setup>
import { onMounted, onUnmounted } from 'vue'
let timer

onMounted(() => {
  timer = setInterval(() => {}, 1000)
})

// ❌ 忘记清理 → 内存泄漏
// ✅ 必须清理
onUnmounted(() => clearInterval(timer))
</script>
```

## 三、注意事项与常见陷阱
- 不要在 update 钩子中修改数据
- DOM ref 必须在 onMounted 之后访问
- 异步操作要考虑取消和竞态条件
- 所有副作用都需要对应的清理逻辑
