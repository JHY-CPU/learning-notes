# 组合式 vs 选项式生命周期

## 一、概念说明
Vue 3 同时支持组合式 API 和选项式 API 的生命周期。了解两者的**映射关系**可以帮助我们在迁移代码或阅读旧项目时快速理解。

## 二、映射关系对照表

| 选项式 API | 组合式 API | 说明 |
|-----------|-----------|------|
| beforeCreate | setup() | setup 本身替代 |
| created | setup() | setup 本身替代 |
| beforeMount | onBeforeMount | 挂载前 |
| mounted | onMounted | 挂载后 |
| beforeUpdate | onBeforeUpdate | 更新前 |
| updated | onUpdated | 更新后 |
| beforeUnmount | onBeforeUnmount | 卸载前 |
| unmounted | onUnmounted | 卸载后 |
| activated | onActivated | 缓存激活 |
| deactivated | onDeactivated | 缓存失活 |
| errorCaptured | onErrorCaptured | 错误捕获 |
| - | onRenderTracked | 调试（仅组合式）|
| - | onRenderTriggered | 调试（仅组合式）|

### 2.1 对照代码示例
```vue
<!-- 选项式 -->
<script>
export default {
  created() { console.log('数据初始化') },
  mounted() { console.log('DOM 可用') }
}
</script>

<!-- 组合式等价写法 -->
<script setup>
console.log('数据初始化')  // created 的等价
import { onMounted } from 'vue'
onMounted(() => console.log('DOM 可用'))
</script>
```

### 2.2 执行顺序差异
```
选项式: beforeCreate → created → beforeMount → mounted
组合式: setup() → onBeforeMount → onMounted

注意: setup() 中代码在 beforeCreate 之前执行
```

## 三、注意事项与常见陷阱
- `setup()` 替代了 `beforeCreate` + `created` 两个钩子
- 组合式 API 的 `onXxx` 钩子可以注册**多个**，按注册顺序执行
- 选项式 API 中每个钩子只能有一个
- 两种 API 的钩子在同一个组件中可以共存，但不推荐
