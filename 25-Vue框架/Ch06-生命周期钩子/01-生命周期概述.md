# 生命周期概述

## 一、概念说明
Vue 组件从创建到销毁会经历一系列阶段，**生命周期钩子**是在特定阶段自动调用的函数。Vue 3 组合式 API 中通过 `onXxx` 函数注册生命周期钩子。

## 二、生命周期全景图

```
创建阶段: setup → onBeforeMount → onMounted
更新阶段: onBeforeUpdate → onUpdated
卸载阶段: onBeforeUnmount → onUnmounted
缓存组件: onActivated / onDeactivated
错误处理: onErrorCaptured
调试: onRenderTracked / onRenderTriggered
```

### 2.1 组合式 API 钩子
```vue
<script setup>
import {
  onBeforeMount, onMounted,
  onBeforeUpdate, onUpdated,
  onBeforeUnmount, onUnmounted
} from 'vue'

onBeforeMount(() => console.log('挂载前：模板已编译'))
onMounted(() => console.log('挂载后：DOM 可用'))
onBeforeUpdate(() => console.log('更新前：数据已变，DOM 未更新'))
onUpdated(() => console.log('更新后：DOM 已更新'))
onBeforeUnmount(() => console.log('卸载前：清理副作用'))
onUnmounted(() => console.log('卸载后：完全销毁'))
</script>
```

### 2.2 执行顺序
```
1. setup()（最先执行）
2. onBeforeMount
3. onMounted
   ↓ 数据变化
4. onBeforeUpdate
5. onUpdated
   ↓ 组件销毁
6. onBeforeUnmount
7. onUnmounted
```

### 2.3 使用场景速查

| 钩子 | 典型用途 |
|------|---------|
| setup | 初始化数据、定义方法 |
| onBeforeMount | 不依赖 DOM 的准备工作 |
| onMounted | DOM 操作、API 请求、初始化第三方库 |
| onBeforeUpdate | 保存更新前的 DOM 状态 |
| onUpdated | 访问更新后的 DOM |
| onBeforeUnmount | 清理定时器、事件监听、取消请求 |
| onUnmounted | 最终清理、日志记录 |
| onActivated | 缓存组件恢复时重新获取数据 |
| onDeactivated | 缓存组件暂停时保存状态 |
| onErrorCaptured | 错误边界处理 |

## 三、注意事项与常见陷阱

- `setup()` 本身相当于 `onBeforeCreate` 和 `created` 的合并
- `onMounted` 中才能安全访问 DOM 元素
- 不要在 `onBeforeUpdate` 中修改响应式数据，会导致无限循环
- `onUnmounted` 是清理定时器、事件监听的最佳位置
- SSR 中只有 `setup` 和 `onBeforeMount` 会在服务端执行
- 所有生命周期钩子都必须在 setup 中同步调用（不能在 setTimeout 中调用）
