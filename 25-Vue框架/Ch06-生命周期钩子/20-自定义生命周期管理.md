# 自定义生命周期管理

## 一、概念说明
通过**组合式函数**（Composables）可以将生命周期相关的逻辑封装为可复用的函数，实现自定义的生命周期管理。这使得初始化、清理等逻辑可以在多个组件间共享。

## 二、具体用法

### 2.1 封装定时器 Composable
```js
// composables/useInterval.js
import { ref, onMounted, onUnmounted } from 'vue'

export function useInterval(callback, delay) {
  const isActive = ref(false)
  let timer = null

  function start() {
    if (isActive.value) return
    timer = setInterval(callback, delay)
    isActive.value = true
  }

  function stop() {
    clearInterval(timer)
    timer = null
    isActive.value = false
  }

  onMounted(start)
  onUnmounted(stop)

  return { isActive, start, stop }
}
```

### 2.2 封装数据请求 Composable
```js
// composables/useAsync.js
import { ref, onMounted } from 'vue'

export function useAsync(fn, options = {}) {
  const data = ref(null)
  const loading = ref(false)
  const error = ref(null)

  async function execute() {
    loading.value = true
    error.value = null
    try {
      data.value = await fn()
    } catch (e) {
      error.value = e
    } finally {
      loading.value = false
    }
  }

  if (options.immediate !== false) {
    onMounted(execute)
  }

  return { data, loading, error, execute }
}
```

### 2.3 使用 Composable
```vue
<script setup>
import { useInterval } from '@/composables/useInterval'
import { useAsync } from '@/composables/useAsync'

const { data: users } = useAsync(() => fetch('/api/users').then(r => r.json()))
const { isActive, start, stop } = useInterval(() => console.log('tick'), 1000)
</script>
```

## 三、注意事项与常见陷阱
- 组合式函数中可以安全使用 `onMounted`、`onUnmounted` 等钩子
- Composable 应在 `setup()` 的顶层调用，不要在条件判断或循环中使用
- 每个 Composable 调用都有独立的生命周期实例
- 清理逻辑应在 Composable 内部管理，不要依赖调用者
