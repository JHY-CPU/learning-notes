# 组件间生命周期顺序

## 一、概念说明
在包含父子关系的组件树中，不同组件的生命周期钩子有特定的**执行顺序**。理解这个顺序对于调试和编写正确的初始化逻辑非常重要。

## 二、执行顺序

### 2.1 挂载顺序（从外到内再到外）
```
父 onBeforeMount
  子 onBeforeMount
  子 onMounted
父 onMounted
```

### 2.2 完整示例
```vue
<!-- Parent.vue -->
<script setup>
import { onBeforeMount, onMounted } from 'vue'
import Child from './Child.vue'

onBeforeMount(() => console.log('1. 父 onBeforeMount'))
onMounted(() => console.log('4. 父 onMounted'))
</script>
```

```vue
<!-- Child.vue -->
<script setup>
import { onBeforeMount, onMounted } from 'vue'

onBeforeMount(() => console.log('2. 子 onBeforeMount'))
onMounted(() => console.log('3. 子 onMounted'))
</script>
```

### 2.3 更新顺序
```
父 onBeforeUpdate
  子 onBeforeUpdate
  子 onUpdated
父 onUpdated
```

### 2.4 卸载顺序
```
父 onBeforeUnmount
  子 onBeforeUnmount
  子 onUnmounted
父 onUnmounted
```

### 2.5 兄弟组件顺序
```vue
<!-- 按模板中的声明顺序 -->
<template>
  <CompA />  <!-- A 先挂载 -->
  <CompB />  <!-- B 后挂载 -->
</template>

<!-- 顺序: A onBeforeMount → A onMounted → B onBeforeMount → B onMounted -->
```

## 三、注意事项与常见陷阱
- 子组件的 `onMounted` 先于父组件的 `onMounted`（子先完成挂载）
- 子组件的 `onUnmounted` 先于父组件的 `onUnmounted`（子先卸载）
- 更新顺序也是从外到内再到外
- 兄弟组件按声明顺序执行，不要依赖这个顺序
