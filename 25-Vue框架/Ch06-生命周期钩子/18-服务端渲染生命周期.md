# 服务端渲染生命周期

## 一、概念说明
在**服务端渲染（SSR）**环境下，组件的生命周期与客户端不同。服务端只执行到 `onBeforeMount` 之前，`onMounted` 等依赖 DOM 的钩子不会在服务端执行。Vue 提供了 `onServerPrefetch` 专门用于 SSR 数据预取。

## 二、具体用法

### 2.1 SSR 中哪些钩子会被调用
```
服务端执行:
  setup() 中的同步代码
  onServerPrefetch()
  onBeforeMount()（部分框架会执行）

不会执行:
  onMounted, onBeforeUpdate, onUpdated, onBeforeUnmount, onUnmounted
  onActivated, onDeactivated, onRenderTracked, onRenderTriggered
```

### 2.2 onServerPrefetch
```vue
<script setup>
import { ref, onServerPrefetch } from 'vue'
import { useRoute } from 'vue-router'

const data = ref(null)

onServerPrefetch(async () => {
  const route = useRoute()
  data.value = await fetch(`/api/posts/${route.params.id}`)
    .then(r => r.json())
})
</script>

<template>
  <div v-if="data">
    <h1>{{ data.title }}</h1>
  </div>
</template>
```

### 2.3 同时支持 SSR 和客户端
```vue
<script setup>
import { ref, onMounted, onServerPrefetch } from 'vue'

const data = ref(null)

// 服务端预取
onServerPrefetch(async () => {
  data.value = await fetchData()
})

// 客户端：如果数据未预取则请求
onMounted(async () => {
  if (!data.value) {
    data.value = await fetchData()
  }
})
</script>
```

### 2.4 Pinia + SSR
```vue
<script setup>
import { useUserStore } from '@/stores/user'
import { onServerPrefetch } from 'vue'

const store = useUserStore()

onServerPrefetch(async () => {
  await store.fetchUser()
})
</script>
```

## 三、注意事项与常见陷阱
- 所有依赖 DOM 的钩子在 SSR 中都不会执行
- `onServerPrefetch` 必须返回 Promise
- 不要在 SSR 代码中使用 `window`、`document` 等浏览器 API
- 使用 `import.meta.env.SSR` 判断当前环境

## 四、SSR 安全的代码模式

```js
// 检查环境
if (typeof window !== 'undefined') {
  // 浏览器特有逻辑
}

// 或使用 Vite 环境变量
if (!import.meta.env.SSR) {
  // 客户端特有逻辑
}
```

## 五、完整 SSR 生命周期

```
服务端：
  setup() -> onServerPrefetch() -> render() -> 序列化 HTML

客户端（hydration）：
  setup() -> onBeforeMount() -> hydrate() -> onMounted()
  后续更新：onBeforeUpdate() -> patch() -> onUpdated()
```
