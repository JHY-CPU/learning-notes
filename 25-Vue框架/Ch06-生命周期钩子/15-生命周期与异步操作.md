# 生命周期与异步操作

## 一、概念说明
在生命周期钩子中执行异步操作（API 请求、文件读取等）是非常常见的需求。需要注意**竞态条件**（多次请求结果顺序不确定）和**组件卸载后的状态更新**问题。

## 二、具体用法

### 2.1 onMounted 中请求数据
```vue
<script setup>
import { ref, onMounted } from 'vue'

const data = ref(null)
const loading = ref(true)
const error = ref(null)

onMounted(async () => {
  try {
    const res = await fetch('/api/data')
    data.value = await res.json()
  } catch (e) {
    error.value = e.message
  } finally {
    loading.value = false
  }
})
</script>
```

### 2.2 处理竞态条件
```vue
<script setup>
import { ref, onMounted, onUnmounted } from 'vue'

const data = ref(null)
let abortController = null

async function fetchData(id) {
  abortController?.abort()
  abortController = new AbortController()

  try {
    const res = await fetch(`/api/data/${id}`, {
      signal: abortController.signal
    })
    data.value = await res.json()
  } catch (e) {
    if (e.name !== 'AbortError') throw e
  }
}

onMounted(() => fetchData(1))
onUnmounted(() => abortController?.abort())
</script>
```

### 2.3 异步守卫模式
```vue
<script setup>
import { ref, onMounted } from 'vue'

const ready = ref(false)

async function init() {
  await loadConfig()
  await loadUser()
  ready.value = true
}

onMounted(init)
</script>

<template>
  <div v-if="ready">内容</div>
  <div v-else>加载中...</div>
</template>
```

### 2.4 错误重试机制
```vue
<script setup>
import { ref, onMounted } from 'vue'

const data = ref(null)
const retries = ref(0)
const maxRetries = 3

async function fetchWithRetry() {
  try {
    data.value = await fetch('/api/data').then(r => r.json())
  } catch (e) {
    if (retries.value < maxRetries) {
      retries.value++
      setTimeout(fetchWithRetry, 1000 * retries.value)
    }
  }
}

onMounted(fetchWithRetry)
</script>
```

## 三、注意事项与常见陷阱
- 不要在异步操作完成后检查组件是否已卸载，使用 AbortController 取消请求
- 竞态条件是常见 bug：快速切换页面时旧请求可能覆盖新请求的结果
- 使用 `onUnmounted` 取消未完成的请求
- 考虑使用 VueUse 的 `useFetch` 等封装好的方案

## 四、VueUse 推荐

```vue
<script setup>
import { useFetch } from '@vueuse/core'

// 自动处理竞态、取消、loading/error 状态
const { data, error, isFetching, execute } = useFetch('/api/data', {
  immediate: false,
  refetch: true
}).json()

onMounted(() => execute())
</script>
```
