# 路由基础配置

## 一、概念说明

路由配置通过`routes`数组定义，每个路由对象包含`path`（路径）和`component`（组件）。还可以配置嵌套路由、重定向、守卫等。

```js
// router/index.js
import { createRouter, createWebHistory } from 'vue-router'

const routes = [
  {
    path: '/',
    name: 'Home',
    component: () => import('@/views/Home.vue')
  },
  {
    path: '/about',
    name: 'About',
    component: () => import('@/views/About.vue')
  },
  {
    path: '/user/:id',
    name: 'User',
    component: () => import('@/views/User.vue')
  }
]

const router = createRouter({
  history: createWebHistory(),
  routes
})

export default router
```

## 二、具体用法

### 路由对象属性

```js
{
  path: '/user/:id',       // 路径（支持动态参数）
  name: 'UserDetail',      // 命名路由
  component: UserDetail,   // 组件（或懒加载函数）
  children: [],            // 嵌套路由
  redirect: '/home',       // 重定向
  alias: '/profile',       // 别名
  meta: { requiresAuth: true },  // 元信息
  props: true,             // 将params作为props传入
  beforeEnter: (to, from) => {}  // 独享守卫
}
```

### 懒加载配置

```js
// 不推荐：全部立即加载
import Home from '@/views/Home.vue'

// 推荐：按需懒加载
const Home = () => import('@/views/Home.vue')

// 带webpack分组名
const Home = () => import(/* webpackChunkName: "home" */ '@/views/Home.vue')
```

## 三、注意事项与常见陷阱

1. `path`必须以`/`开头（根路径除外）
2. 通配符路由`/:pathMatch(.*)*`放在最后匹配404
3. 路由匹配按定义顺序，先定义先匹配
4. 懒加载组件需要代码分割支持（webpack/vite默认支持）
5. `routes`数组中不带`/`的路径会拼接父路径

## 四、路由配置最佳实践

```js
// router/modules/user.js
export default [
  {
    path: '/user',
    component: () => import('@/views/user/Layout.vue'),
    children: [
      { path: '', name: 'UserList', component: () => import('@/views/user/List.vue') },
      { path: ':id(\\d+)', name: 'UserDetail', component: () => import('@/views/user/Detail.vue') },
      { path: 'create', name: 'UserCreate', component: () => import('@/views/user/Create.vue') },
      { path: ':id/edit', name: 'UserEdit', component: () => import('@/views/user/Edit.vue') }
    ]
  }
]

// router/index.js
import userRoutes from './modules/user'
import adminRoutes from './modules/admin'

const routes = [
  { path: '/', name: 'Home', component: () => import('@/views/Home.vue') },
  ...userRoutes,
  ...adminRoutes,
  { path: '/:pathMatch(.*)*', name: 'NotFound', component: () => import('@/views/404.vue') }
]
```

## 五、路径匹配优先级

```
精确匹配:  /user/123       → 最高优先级
参数匹配:  /user/:id       → 次优先级
通配匹配:  /*              → 最低优先级
```
