# 路由守卫-独享守卫

## 一、概念说明

独享守卫直接定义在路由配置的`beforeEnter`中，只在进入该路由时触发，不影响其他路由。

```js
const routes = [
  {
    path: '/admin',
    component: Admin,
    beforeEnter: (to, from) => {
      if (!isAdmin()) {
        return '/403'  // 无权限
      }
    }
  },
  {
    path: '/public',
    component: Public
    // 无beforeEnter，不触发守卫
  }
]
```

## 二、具体用法

### 权限控制

```js
const routes = [
  {
    path: '/dashboard',
    component: Dashboard,
    beforeEnter: (to, from) => {
      const role = getUserRole()
      if (!role) return '/login'
      if (role !== 'admin') return '/403'
    }
  },
  {
    path: '/profile',
    component: Profile,
    beforeEnter: (to, from) => {
      if (!isLoggedIn()) return '/login'
    }
  }
]
```

### 函数数组形式

```js
// 多个守卫函数
{
  path: '/secure',
  component: Secure,
  beforeEnter: [authGuard, roleGuard, logGuard]
}

function authGuard(to, from) {
  if (!localStorage.getItem('token')) return '/login'
}

function roleGuard(to, from) {
  if (!hasPermission(to.path)) return '/403'
}
```

## 四、多条件守卫组合

```js
// guards/auth.js
export function requireAuth(to, from) {
  if (!localStorage.getItem('token')) {
    return { name: 'Login', query: { redirect: to.fullPath } }
  }
}

// guards/role.js
export function requireRole(role) {
  return (to, from) => {
    const userRole = localStorage.getItem('role')
    if (userRole !== role) {
      return '/403'
    }
  }
}

// guards/feature.js
export function requireFeature(featureName) {
  return (to, from) => {
    const features = JSON.parse(localStorage.getItem('features') || '[]')
    if (!features.includes(featureName)) {
      return '/upgrade'
    }
  }
}

// 路由配置
const routes = [
  {
    path: '/admin/users',
    component: AdminUsers,
    beforeEnter: [requireAuth, requireRole('admin')]
  },
  {
    path: '/premium/analytics',
    component: Analytics,
    beforeEnter: [requireAuth, requireFeature('analytics')]
  }
]
```

## 五、异步独享守卫

```js
{
  path: '/checkout',
  component: Checkout,
  beforeEnter: async (to, from) => {
    // 检查购物车是否为空
    const cart = await api.getCart()
    if (cart.items.length === 0) {
      return '/cart'
    }
    // 检查用户是否登录
    const user = await api.getCurrentUser()
    if (!user) {
      return { name: 'Login', query: { redirect: to.fullPath } }
    }
  }
}
```

## 三、注意事项与常见陷阱

1. `beforeEnter`只在**进入**路由时触发，不会在路由参数变化时触发
2. `beforeEnter`可以是函数或函数数组
3. 在组件内定义更灵活（可访问组件状态），独享守卫适合简单检查
4. 独享守卫优先级：全局`beforeEach` -> 独享`beforeEnter` -> 组件`beforeRouteEnter`
5. 嵌套路由中，独享守卫只在**当前层**触发
6. 守卫函数可复用，抽取为独立模块
7. 异步守卫返回 Promise，Vue Router 会等待完成
