# 命名视图

## 一、概念说明

命名视图允许在同一路由中渲染多个组件到不同的`<RouterView>`。适用于复杂布局（如侧边栏+主内容+底部）。

```vue
<!-- App.vue -->
<template>
  <RouterView />               <!-- 默认视图 -->
  <RouterView name="sidebar" /> <!-- 命名视图 -->
  <RouterView name="footer" />  <!-- 命名视图 -->
</template>
```

```js
const routes = [
  {
    path: '/dashboard',
    components: {
      default: DashboardMain,
      sidebar: DashboardSidebar,
      footer: DashboardFooter
    }
  }
]
```

## 二、具体用法

### 典型布局

```js
const routes = [
  {
    path: '/settings',
    components: {
      default: SettingsContent,
      nav: SettingsNav,
      helper: SettingsHelper
    }
  }
]
```

### 嵌套路由中使用命名视图

```js
{
  path: '/admin',
  component: AdminLayout,
  children: [
    {
      path: 'dashboard',
      components: {
        default: AdminDashboard,
        chart: AdminChart
      }
    }
  ]
}
```

## 四、为命名视图传参

```js
{
  path: '/dashboard',
  components: {
    default: DashboardMain,
    sidebar: DashboardSidebar
  },
  // 为不同视图传递 props
  props: {
    default: true,       // 路由参数作为 props 传给 default
    sidebar: { collapsed: false }  // 静态对象作为 props 传给 sidebar
  }
}
```

## 五、与嵌套路由的选择

| 方案 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| 命名视图 | 同一层级多个独立区域 | 灵活控制每个区域 | 配置复杂 |
| 嵌套路由 | 层级关系明确的布局 | 结构清晰 | 嵌套层级有限 |

```js
// 命名视图：侧边栏固定，主内容变化
{
  path: '/app',
  components: { default: AppMain, sidebar: AppSidebar },
  children: [
    { path: 'page1', components: { default: Page1, sidebar: AppSidebar } },
    { path: 'page2', components: { default: Page2, sidebar: AppSidebar } }
  ]
}

// 嵌套路由：更适合层级布局
{
  path: '/app',
  component: AppLayout,  // 内含 <RouterView />
  children: [
    { path: 'page1', component: Page1 },
    { path: 'page2', component: Page2 }
  ]
}
```

## 三、注意事项与常见陷阱

1. 默认视图名称为`default`，对应`<RouterView>`（不带name）
2. 路由配置用`components`（复数）而非`component`
3. 所有命名视图的`<RouterView>`必须在同一组件中
4. 命名视图可配合`props: { sidebar: true }`分别传参
5. 使用频率较低，大部分场景单个`<RouterView>`+嵌套路由即可
6. 命名视图的子路由需要为每个视图单独配置组件
7. 复杂布局可考虑使用 Layout 组件 + 嵌套路由替代命名视图
