# 命名路由

## 一、概念说明

命名路由给路由配置添加`name`属性，导航时可以用名称替代路径。好处是路径变更时只需修改路由配置，不用修改所有导航代码。

```js
const routes = [
  { path: '/', name: 'Home', component: Home },
  { path: '/user/:id', name: 'User', component: User },
  { path: '/about', name: 'About', component: About }
]
```

```vue
<template>
  <RouterLink :to="{ name: 'User', params: { id: 123 } }">用户123</RouterLink>
</template>
```

## 二、具体用法

### 导航使用

```js
// 编程式导航
router.push({ name: 'User', params: { id: 123 } })
router.push({ name: 'About' })

// 模板导航
<RouterLink :to="{ name: 'User', params: { id: 1 } }">用户</RouterLink>
```

### 拼接query参数

```js
router.push({
  name: 'Search',
  query: { q: 'vue', page: 1 }
})
// URL: /search?q=vue&page=1
```

### 命名路由的好处

```js
// ❌ 硬编码路径（路径变更需全局替换）
router.push('/user/' + id)

// ✅ 使用命名路由（路径变更只需改路由配置）
router.push({ name: 'User', params: { id } })
```

## 四、路由名称常量

```ts
// router/names.ts
export const RouteNames = {
  HOME: 'Home',
  ABOUT: 'About',
  USER: 'User',
  USER_PROFILE: 'UserProfile',
  USER_SETTINGS: 'UserSettings',
  NOT_FOUND: 'NotFound'
} as const

// 使用
import { RouteNames } from '@/router/names'

router.push({ name: RouteNames.USER, params: { id: '123' } })
```

## 五、与路径导航的对比

```js
// 路径导航（不推荐）
router.push('/user/123')
router.push('/search?q=vue&page=1')

// 命名路由导航（推荐）
router.push({ name: 'User', params: { id: '123' } })
router.push({ name: 'Search', query: { q: 'vue', page: 1 } })

// 优势：路径变更时只改路由配置
// { path: '/member/:id', name: 'User', component: User }
// 所有使用 { name: 'User' } 的代码都不需要修改
```

## 六、动态路由与命名路由

```js
// 添加动态路由
router.addRoute({
  path: '/new-page',
  name: 'NewPage',
  component: NewPage
})

// 检查路由是否存在
if (router.hasRoute('NewPage')) {
  router.push({ name: 'NewPage' })
}

// 移除路由
router.removeRoute('NewPage')
```

## 三、注意事项与常见陷阱

1. `name`必须**全局唯一**，否则后面的路由覆盖前面的
2. 使用`params`导航时必须搭配`name`，**不能搭配`path`**
3. `params`和`path`一起使用时，`params`会被忽略
4. 命名路由在大型项目中强烈推荐，便于维护
5. 可用`router.hasRoute(name)`检查路由是否存在
6. TypeScript项目建议将路由名称定义为常量，获得更好的类型检查
7. 动态添加的路由也可以使用命名路由导航
