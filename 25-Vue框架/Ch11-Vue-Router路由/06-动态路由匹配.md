# 动态路由匹配

## 一、概念说明

动态路由通过`:param`语法定义URL参数，同一个组件可以匹配不同的路径。参数通过`route.params`获取。

```js
// 路由配置
const routes = [
  { path: '/user/:id', component: User },
  { path: '/post/:postId/comment/:commentId', component: Comment }
]
```

```vue
<!-- User.vue -->
<template>
  <p>用户ID: {{ $route.params.id }}</p>
</template>

<script setup>
import { useRoute } from 'vue-router'
const route = useRoute()
console.log(route.params.id)  // 如 '123'
</script>
```

## 二、具体用法

### 参数模式

```js
// 单个参数
{ path: '/user/:id' }
// 匹配 /user/123 -> params: { id: '123' }

// 多个参数
{ path: '/user/:userId/post/:postId' }
// 匹配 /user/1/post/42 -> params: { userId: '1', postId: '42' }

// 可选参数 (Vue Router 4)
{ path: '/user/:id?' }
// 匹配 /user 和 /user/123

// 通配符
{ path: '/:pathMatch(.*)*' }  // 匹配所有路径
```

### props传参

```js
// 将params作为props传入组件
{ path: '/user/:id', component: User, props: true }

// 函数模式
{
  path: '/search',
  component: Search,
  props: (route) => ({ query: route.query.q })
}
```

```vue
<!-- User.vue -->
<script setup>
const props = defineProps({ id: String })
console.log(props.id)  // '123'
</script>
```

## 三、注意事项与常见陷阱

1. `route.params`的值始终是**字符串**，需要数字时要转换
2. 路由参数变化时**组件不会重建**（复用），需监听变化
3. 使用`props: true`将参数作为组件props，更符合Vue理念
4. 参数变化时`watch(route.params)`或`onBeforeRouteUpdate`捕获
5. 路径中的`/`会被编码，注意URL编码问题
