# 动态路由匹配

## 一、概念说明

动态路由通过`:param`语法定义URL参数，同一个组件可以匹配不同的路径。参数通过`route.params`获取。

```js
// 路由配置
const routes = [
  { path: '/user/:id', component: User },
  { path: '/post/:postId/comment/:commentId', component: Comment }
]
```

```vue
<!-- User.vue -->
<template>
  <p>用户ID: {{ $route.params.id }}</p>
</template>

<script setup>
import { useRoute } from 'vue-router'
const route = useRoute()
console.log(route.params.id)  // 如 '123'
</script>
```

## 二、具体用法

### 参数模式

```js
// 单个参数
{ path: '/user/:id' }
// 匹配 /user/123 -> params: { id: '123' }

// 多个参数
{ path: '/user/:userId/post/:postId' }
// 匹配 /user/1/post/42 -> params: { userId: '1', postId: '42' }

// 可选参数 (Vue Router 4)
{ path: '/user/:id?' }
// 匹配 /user 和 /user/123

// 通配符
{ path: '/:pathMatch(.*)*' }  // 匹配所有路径
```

### props传参

```js
// 将params作为props传入组件
{ path: '/user/:id', component: User, props: true }

// 函数模式
{
  path: '/search',
  component: Search,
  props: (route) => ({ query: route.query.q })
}
```

```vue
<!-- User.vue -->
<script setup>
const props = defineProps({ id: String })
console.log(props.id)  // '123'
</script>
```

## 三、注意事项与常见陷阱

1. `route.params`的值始终是**字符串**，需要数字时要转换
2. 路由参数变化时**组件不会重建**（复用），需监听变化
3. 使用`props: true`将参数作为组件props，更符合Vue理念
4. 参数变化时`watch(route.params)`或`onBeforeRouteUpdate`捕获
5. 路径中的`/`会被编码，注意URL编码问题

## 四、路由参数变化的处理

```vue
<script setup>
import { useRoute, onBeforeRouteUpdate } from 'vue-router'
import { watch } from 'vue'

const route = useRoute()

// 方式一：watch 监听参数变化
watch(
  () => route.params.id,
  async (newId) => {
    // 参数变化时重新获取数据
    await fetchUser(newId)
  }
)

// 方式二：路由守卫
onBeforeRouteUpdate(async (to, from) => {
  if (to.params.id !== from.params.id) {
    await fetchUser(to.params.id)
  }
})
</script>
```

## 五、高级路径匹配

```js
const routes = [
  // 正则约束：只匹配数字
  { path: '/user/:id(\\d+)', component: User },

  // 可选参数
  { path: '/user/:id?', component: User },

  // 重复参数
  { path: '/files/:path+', component: Files },  // /files/a, /files/a/b, /files/a/b/c

  // 零个或多个
  { path: '/files/:path*', component: Files },  // /files, /files/a, /files/a/b

  // 自定义正则
  { path: '/:id(\\d+)', component: NumberOnly },
  { path: '/:slug([a-z]+)', component: TextOnly }
]
```

## 六、使用 props 传递路由参数

```js
// 路由配置
const routes = [
  { path: '/user/:id', component: User, props: true },          // params 作为 props
  { path: '/search', component: Search, props: (route) => ({    // 函数模式
      query: route.query.q,
      page: Number(route.query.page) || 1
    })
  }
]
```

```vue
<!-- User.vue -->
<script setup>
const props = defineProps({
  id: { type: String, required: true }
})
// 不需要 useRoute，直接用 props.id
</script>
```
