# 路由守卫-组件内守卫

## 一、概念说明

组件内路由守卫在路由组件中直接定义，包括：`onBeforeRouteLeave`（离开前）、`onBeforeRouteUpdate`（更新前）。

```vue
<script setup>
import { onBeforeRouteLeave, onBeforeRouteUpdate } from 'vue-router'

// 离开当前路由前触发
onBeforeRouteLeave((to, from) => {
  const answer = window.confirm('表单未保存，确定离开？')
  if (!answer) return false  // 阻止导航
})

// 同路由参数变化时触发
onBeforeRouteUpdate(async (to, from) => {
  if (to.params.id !== from.params.id) {
    await fetchUser(to.params.id)
  }
})
</script>
```

## 二、具体用法

### onBeforeRouteLeave

```vue
<script setup>
import { ref } from 'vue'
import { onBeforeRouteLeave } from 'vue-router'

const isSaved = ref(false)

onBeforeRouteLeave((to, from) => {
  if (!isSaved.value) {
    const confirm = window.confirm('有未保存的内容，确定离开？')
    return confirm
  }
})
</script>
```

### onBeforeRouteUpdate

```vue
<script setup>
import { ref, watch } from 'vue'
import { onBeforeRouteUpdate, useRoute } from 'vue-router'

const route = useRoute()
const userData = ref(null)

// 当 /user/1 -> /user/2 时触发
onBeforeRouteUpdate(async (to) => {
  userData.value = await fetchUser(to.params.id)
})
</script>
```

### 选项式API写法

```vue
<script>
export default {
  beforeRouteEnter(to, from) {
    // 组件创建前，没有this
  },
  beforeRouteUpdate(to, from) {
    // 参数变化时
  },
  beforeRouteLeave(to, from) {
    // 离开时
  }
}
</script>
```

## 四、表单离开确认（完整示例）

```vue
<script setup>
import { ref, computed } from 'vue'
import { onBeforeRouteLeave } from 'vue-router'

const form = ref({ name: '', email: '', bio: '' })
const originalForm = JSON.stringify(form.value)

const isDirty = computed(() =>
  JSON.stringify(form.value) !== originalForm
)

onBeforeRouteLeave((to, from) => {
  if (!isDirty.value) return true  // 未修改，直接放行

  const answer = window.confirm('表单有未保存的修改，确定离开？')
  if (!answer) return false  // 阻止导航

  return true  // 放行
})
</script>
```

## 五、参数变化时重新获取数据

```vue
<script setup>
import { ref } from 'vue'
import { useRoute, onBeforeRouteUpdate } from 'vue-router'

const route = useRoute()
const user = ref(null)

async function fetchUser(id) {
  const res = await fetch(`/api/users/${id}`)
  user.value = await res.json()
}

// 初始加载
fetchUser(route.params.id)

// 参数变化时重新加载（如 /user/1 -> /user/2）
onBeforeRouteUpdate(async (to) => {
  await fetchUser(to.params.id)
})
</script>
```

## 六、守卫执行顺序

```
导航触发顺序：

1. 导航被触发
2. 失活组件的 beforeRouteLeave
3. 全局 beforeEach
4. 重用组件的 beforeRouteUpdate（参数变化时）
5. 路由配置的 beforeEnter
6. 解析异步组件
7. 激活组件的 beforeRouteEnter
8. 全局 beforeResolve
9. 导航被确认
10. 全局 afterEach
11. DOM 更新
12. beforeRouteEnter 的 next 回调
```

## 三、注意事项与常见陷阱

1. `onBeforeRouteUpdate`只在**同组件参数变化**时触发，不是每次导航都触发
2. `<script setup>`中使用`onBeforeRouteLeave`/`onBeforeRouteUpdate`
3. 选项式API中还有`beforeRouteEnter`，组合式API无对应钩子
4. 返回`false`取消导航，返回`true`或`undefined`放行
5. 适合组件级别的权限检查和表单离开确认
6. `onBeforeRouteLeave`在组件卸载前调用，可以访问组件状态
7. 异步守卫中可以返回 Promise，Vue Router 会等待解析
