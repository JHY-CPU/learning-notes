# 路由最佳实践

## 一、概念说明

良好的路由配置和管理策略能提升项目的可维护性。包括文件组织、命名规范、性能优化等方面。

```
src/
  router/
    index.js          # 路由器创建
    routes.js          # 路由配置
    guards.js          # 守卫逻辑
    modules/           # 按模块拆分
      admin.js
      user.js
  views/              # 路由页面组件
    Home.vue
    NotFound.vue
```

## 二、具体用法

### 模块化路由配置

```js
// router/modules/user.js
export default [
  { path: '/profile', name: 'Profile', component: () => import('@/views/user/Profile.vue') },
  { path: '/settings', name: 'Settings', component: () => import('@/views/user/Settings.vue') }
]

// router/index.js
import adminRoutes from './modules/admin'
import userRoutes from './modules/user'

const routes = [
  { path: '/', component: () => import('@/views/Home.vue') },
  {
    path: '/admin',
    component: () => import('@/layouts/AdminLayout.vue'),
    children: adminRoutes
  },
  ...userRoutes,
  { path: '/:pathMatch(.*)*', component: () => import('@/views/NotFound.vue') }
]
```

### 命名规范

```js
// ✅ 命名使用PascalCase
{ name: 'UserProfile', path: '/user/:id/profile' }
{ name: 'AdminDashboard', path: '/admin/dashboard' }

// ✅ 路径使用kebab-case
{ path: '/user-management' }
{ path: '/order-detail/:id' }

// ✅ 组件文件名与路由一致
// /user/:id/profile -> views/user/Profile.vue
```

### 守卫分离

```js
// router/guards.js
export function setupAuthGuard(router) { /* ... */ }
export function setupTitleGuard(router) { /* ... */ }
export function setupProgressGuard(router) { /* ... */ }

// router/index.js
import { setupAuthGuard, setupTitleGuard } from './guards'

const router = createRouter({ /* ... */ })
setupAuthGuard(router)
setupTitleGuard(router)
```

## 三、注意事项与常见陷阱

1. 路由配置按功能模块拆分，避免单个文件过大
2. 命名路由使用语义化名称，便于维护
3. 守卫逻辑分离到独立文件，保持router文件简洁
4. 懒加载使用webpack分组名，合理控制chunk数量
5. 定期审查路由配置，清理无用路由
