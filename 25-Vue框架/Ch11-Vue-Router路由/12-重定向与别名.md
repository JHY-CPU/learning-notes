# 重定向与别名

## 一、概念说明

**重定向**将一个路径自动跳转到另一个路径，URL会改变。**别名**让一个路径的内容在另一个路径也可访问，URL不变。

```js
const routes = [
  // 重定向：访问 /home 跳转到 /（URL变）
  { path: '/home', redirect: '/' },

  // 别名：访问 /main 显示 / 的内容（URL不变）
  { path: '/', component: Home, alias: '/main' },

  // 函数重定向
  {
    path: '/user/:id',
    redirect: (to) => ({ path: `/profile/${to.params.id}` })
  }
]
```

## 二、具体用法

### 重定向形式

```js
// 字符串
{ path: '/old', redirect: '/new' }

// 命名路由
{ path: '/old', redirect: { name: 'NewPage' } }

// 函数（动态重定向）
{
  path: '/user/:id',
  redirect: (to) => {
    if (to.params.id === '1') return '/admin'
    return `/profile/${to.params.id}`
  }
}
```

### 别名形式

```js
// 单个别名
{ path: '/user/:id', component: User, alias: '/member/:id' }

// 多个别名
{ path: '/about', component: About, alias: ['/info', '/company'] }

// 嵌套路由别名
{
  path: '/parent',
  component: Parent,
  alias: '/p',
  children: [
    { path: 'child', component: Child, alias: 'c' }
  ]
}
```

## 四、带参数的重定向

```js
const routes = [
  // 旧路径重定向到新路径（参数保留）
  {
    path: '/old-user/:id',
    redirect: to => {
      return { path: `/user/${to.params.id}`, query: to.query }
    }
  },

  // 根据角色重定向
  {
    path: '/dashboard',
    redirect: () => {
      const role = getUserRole()
      return role === 'admin' ? '/admin/dashboard' : '/user/dashboard'
    }
  }
]
```

## 五、SEO 友好的重定向

```js
{
  path: '/blog/:slug',
  redirect: to => {
    // 301 永久重定向（需要服务器配合设置状态码）
    return { path: `/articles/${to.params.slug}` }
  }
}
```

> 注意：前端路由重定向始终是客户端行为，真正的 HTTP 301/302 需要服务器端配置。

## 六、别名的实际用途

```js
const routes = [
  // 支持中英文路径
  { path: '/about', component: About, alias: '/guanyu' },

  // 兼容旧版 URL
  { path: '/products/:id', component: Product, alias: '/item/:id' },

  // 短链接
  { path: '/articles/:id', component: Article, alias: '/a/:id' }
]
```

## 三、注意事项与常见陷阱

1. 重定向：URL变化，是**服务器端跳转**行为
2. 别名：URL不变，内容相同，是**路由别名**
3. 重定向支持守卫触发，别名不触发
4. 常见用途：旧URL重定向到新URL、简写路径
5. 别名中的参数模式需与原路径匹配
6. 函数重定向可以访问路由参数和查询参数
7. 别名和原路径共享同一套守卫和元信息
