# 编程式导航

## 一、概念说明

编程式导航通过JavaScript代码进行页面跳转，替代点击`<RouterLink>`。使用`router.push()`、`router.replace()`等方法。

```vue
<script setup>
import { useRouter } from 'vue-router'

const router = useRouter()

const goToAbout = () => router.push('/about')
const goToUser = (id) => router.push(`/user/${id}`)
const goToUserObj = (id) => router.push({ name: 'User', params: { id } })
const login = () => router.replace('/login')  // 不保留历史记录
</script>
```

## 二、具体用法

### push方法

```js
// 字符串路径
router.push('/about')

// 对象形式
router.push({ path: '/about' })
router.push({ name: 'User', params: { id: 1 } })
router.push({ path: '/search', query: { q: 'vue' } })
router.push({ hash: '#section1' })

// 带返回值
router.push('/about').then(() => {
  console.log('导航完成')
})
```

### replace方法

```js
// 替换当前历史记录（不可后退）
router.replace('/login')
router.replace({ name: 'Login' })
```

### go / back / forward

```js
router.go(-1)      // 后退一步
router.go(1)       // 前进一步
router.go(-3)      // 后退三步
router.back()      // 后退（等同 go(-1)）
router.forward()   // 前进（等同 go(1)）
```

### 导航守卫中重定向

```js
router.beforeEach((to) => {
  if (to.path === '/admin' && !isAuthenticated) {
    return '/login'  // 重定向
  }
})
```

## 三、注意事项与常见陷阱

1. `push`和`replace`的区别：push添加历史记录，replace替换
2. 用`name`导航时，`params`必须搭配`name`使用，不能搭配`path`
3. 导航是异步的，返回Promise，可等待完成或捕获错误
4. 重复导航（跳转到当前路径）会触发NavigationFailure
5. 使用`router.resolve()`获取路由信息而不跳转
