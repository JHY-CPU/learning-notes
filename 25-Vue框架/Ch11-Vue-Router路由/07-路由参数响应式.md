# 路由参数响应式

## 一、概念说明

当路由参数变化时（如`/user/1`变为`/user/2`），Vue Router会复用同一组件实例。需要通过监听`route.params`来响应变化。

```vue
<template>
  <div>
    <p>用户ID: {{ route.params.id }}</p>
    <button @click="goNext">下一个用户</button>
  </div>
</template>

<script setup>
import { watch } from 'vue'
import { useRoute, useRouter } from 'vue-router'

const route = useRoute()
const router = useRouter()

// 监听参数变化
watch(
  () => route.params.id,
  (newId, oldId) => {
    console.log(`用户从 ${oldId} 变为 ${newId}`)
    fetchUserData(newId)
  }
)

const goNext = () => {
  const nextId = Number(route.params.id) + 1
  router.push(`/user/${nextId}`)
}
</script>
```

## 二、具体用法

### 使用computed获取参数

```vue
<script setup>
import { computed } from 'vue'
import { useRoute } from 'vue-router'

const route = useRoute()
const userId = computed(() => route.params.id)
const searchQuery = computed(() => route.query.q)

// userId是响应式的，参数变化自动更新
</script>
```

### 使用onBeforeRouteUpdate

```vue
<script setup>
import { onBeforeRouteUpdate } from 'vue-router'

onBeforeRouteUpdate(async (to, from) => {
  // 参数变化时调用
  if (to.params.id !== from.params.id) {
    await fetchUser(to.params.id)
  }
})
</script>
```

### 使用watchEffect自动追踪

```vue
<script setup>
import { watchEffect } from 'vue'
import { useRoute } from 'vue-router'

const route = useRoute()

watchEffect(async () => {
  // 自动追踪route.params.id
  const data = await fetchUser(route.params.id)
})
</script>
```

## 三、注意事项与常见陷阱

1. `route`对象是响应式的，但**解构会丢失响应式**
2. 不要用`const { params } = route`，用`route.params`或`computed`
3. 同路径不同参数的导航不会触发组件生命周期钩子（组件复用）
4. `watch(() => route.params.id, ...)`是正确的监听方式
5. 需要数据随参数变化重新加载时，使用watch或onBeforeRouteUpdate
