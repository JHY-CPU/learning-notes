# 路由懒加载

## 一、概念说明

路由懒加载使用动态`import()`将路由组件的代码分割，在访问时才加载。减少首屏加载的JS体积，提升应用性能。

```js
const routes = [
  // ❌ 立即加载：所有组件打包在一起
  // import Home from './views/Home.vue'
  // { path: '/', component: Home }

  // ✅ 懒加载：访问时才加载
  { path: '/', component: () => import('./views/Home.vue') },
  { path: '/about', component: () => import('./views/About.vue') },
  { path: '/user', component: () => import('./views/User.vue') }
]
```

## 二、具体用法

### Webpack分组名

```js
{
  path: '/dashboard',
  component: () => import(
    /* webpackChunkName: "dashboard" */
    './views/Dashboard.vue'
  )
}

// 相同chunkName的组件打包在一起
{
  path: '/settings',
  component: () => import(
    /* webpackChunkName: "dashboard" */
    './views/Settings.vue'
  )
}
```

### Vite分组名

```js
{
  path: '/admin',
  component: () => import(
    /* viteChunkName: "admin" */
    './views/Admin.vue'
  )
}
```

### 预加载

```js
// 通过webpackMagicComments预加载
{
  path: '/about',
  component: () => import(
    /* webpackPrefetch: true */
    './views/About.vue'
  )
}

// 或使用router-link的prefetch
<RouterLink to="/about" prefetch>关于</RouterLink>
```

## 三、注意事项与常见陷阱

1. 懒加载不会减少总JS体积，只是延迟加载
2. 首屏核心路由可使用立即加载，其余用懒加载
3. 分组名将多个路由打包在一个chunk中，减少HTTP请求数
4. 过度分割可能导致大量小文件请求
5. 配合loading组件和错误处理提升用户体验

## 四、懒加载最佳实践

### 4.1 按模块分组
```js
const routes = [
  // 首页直接加载（首屏核心）
  { path: '/', name: 'Home', component: Home },

  // 用户模块懒加载
  { path: '/user/:id', component: () => import('@/views/user/Profile.vue') },

  // 管理模块单独分组
  {
    path: '/admin',
    component: () => import(/* webpackChunkName: "admin" */ '@/views/admin/Layout.vue'),
    children: [
      { path: 'dashboard', component: () => import(/* webpackChunkName: "admin" */ '@/views/admin/Dashboard.vue') },
      { path: 'users', component: () => import(/* webpackChunkName: "admin" */ '@/views/admin/Users.vue') }
    ]
  },

  // 营销页面独立分组
  { path: '/campaign', component: () => import(/* webpackChunkName: "marketing" */ '@/views/Campaign.vue') }
]
```

### 4.2 配合 Suspense（Vue 3）
```vue
<template>
  <Suspense>
    <template #default>
      <RouterView />
    </template>
    <template #fallback>
      <div class="loading">加载中...</div>
    </template>
  </Suspense>
</template>
```

### 4.3 错误处理
```js
// 重试加载
const lazyLoad = (importFn, retries = 3) => {
  return new Promise((resolve, reject) => {
    importFn()
      .then(resolve)
      .catch((error) => {
        if (retries <= 0) return reject(error)
        setTimeout(() => {
          lazyLoad(importFn, retries - 1).then(resolve, reject)
        }, 1000)
      })
  })
}

const routes = [
  { path: '/about', component: () => lazyLoad(() => import('@/views/About.vue')) }
]
```

## 五、性能对比

| 方式 | 首屏加载 | 后续导航 | HTTP请求 |
|------|---------|---------|---------|
| 全部立即加载 | 慢 | 快 | 1个 |
| 逐页面懒加载 | 快 | 略慢 | 多个 |
| 分组懒加载 | 快 | 快 | 较少 |
