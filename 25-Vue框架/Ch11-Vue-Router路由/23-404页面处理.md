# 404页面处理

## 一、概念说明

使用通配符路由匹配所有未定义的路径，显示404页面。Vue Router 4使用`/:pathMatch(.*)*`作为通配符。

```js
const routes = [
  { path: '/', component: Home },
  { path: '/about', component: About },
  // 必须放在最后
  { path: '/:pathMatch(.*)*', name: 'NotFound', component: NotFound }
]
```

```vue
<!-- NotFound.vue -->
<template>
  <div class="not-found">
    <h1>404</h1>
    <p>页面不存在</p>
    <RouterLink to="/">返回首页</RouterLink>
  </div>
</template>
```

## 二、具体用法

### 不同的通配符语法

```js
// Vue Router 4 推荐
{ path: '/:pathMatch(.*)*', component: NotFound }

// 匹配 /xxx 但不匹配 /xxx/yyy
{ path: '/:pathMatch(.*)', component: NotFound }

// 旧版语法（Vue Router 3）
{ path: '*', component: NotFound }
```

### 嵌套路由的404

```js
const routes = [
  {
    path: '/admin',
    component: AdminLayout,
    children: [
      { path: '', component: AdminHome },
      { path: 'users', component: Users },
      // 管理模块的404
      { path: ':pathMatch(.*)*', component: AdminNotFound }
    ]
  },
  // 全局404
  { path: '/:pathMatch(.*)*', component: GlobalNotFound }
]
```

### 捕获路由参数

```vue
<!-- NotFound.vue -->
<script setup>
import { useRoute } from 'vue-router'
const route = useRoute()
console.log('未匹配的路径:', route.params.pathMatch)
</script>
```

## 四、分模块404处理

```js
const routes = [
  {
    path: '/admin',
    component: AdminLayout,
    children: [
      { path: '', component: AdminHome },
      { path: 'users', component: Users },
      { path: 'settings', component: Settings },
      // admin 模块的 404
      { path: ':pathMatch(.*)*', name: 'AdminNotFound', component: AdminNotFound }
    ]
  },
  {
    path: '/blog',
    component: BlogLayout,
    children: [
      { path: '', component: BlogHome },
      { path: ':slug', component: BlogPost },
      // blog 模块的 404
      { path: ':pathMatch(.*)*', name: 'BlogNotFound', component: BlogNotFound }
    ]
  },
  // 全局 404（必须放最后）
  { path: '/:pathMatch(.*)*', name: 'NotFound', component: NotFound }
]
```

## 六、404重定向 vs 404页面

```js
// 方式1：显示404页面（推荐）
{ path: '/:pathMatch(.*)*', component: NotFound }

// 方式2：重定向到首页
{ path: '/:pathMatch(.*)*', redirect: '/' }

// 方式3：重定向到404页面（带URL变化）
{ path: '/:pathMatch(.*)*', redirect: '/404' }
```

| 方式 | URL 变化 | 用户体验 | SEO |
|------|---------|---------|-----|
| 直接显示404 | 保持原URL | 好 | 好 |
| 重定向到首页 | 变为/ | 一般 | 差 |
| 重定向到/404 | 变为/404 | 好 | 好 |

## 三、注意事项与常见陷阱

1. 通配符路由必须放在路由配置的**最后**
2. `/:pathMatch(.*)*`中的`*`表示捕获所有子路径段
3. 嵌套路由中的404只匹配该层的未定义路径
4. 访问404页面时返回404状态码需服务器配合
5. 也可以将404页面做成重定向到首页
6. 分模块404能提供更精准的错误提示
7. 记录404日志有助于发现错误链接
