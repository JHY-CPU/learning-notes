# 404页面处理

## 一、概念说明

使用通配符路由匹配所有未定义的路径，显示404页面。Vue Router 4使用`/:pathMatch(.*)*`作为通配符。

```js
const routes = [
  { path: '/', component: Home },
  { path: '/about', component: About },
  // 必须放在最后
  { path: '/:pathMatch(.*)*', name: 'NotFound', component: NotFound }
]
```

```vue
<!-- NotFound.vue -->
<template>
  <div class="not-found">
    <h1>404</h1>
    <p>页面不存在</p>
    <RouterLink to="/">返回首页</RouterLink>
  </div>
</template>
```

## 二、具体用法

### 不同的通配符语法

```js
// Vue Router 4 推荐
{ path: '/:pathMatch(.*)*', component: NotFound }

// 匹配 /xxx 但不匹配 /xxx/yyy
{ path: '/:pathMatch(.*)', component: NotFound }

// 旧版语法（Vue Router 3）
{ path: '*', component: NotFound }
```

### 嵌套路由的404

```js
const routes = [
  {
    path: '/admin',
    component: AdminLayout,
    children: [
      { path: '', component: AdminHome },
      { path: 'users', component: Users },
      // 管理模块的404
      { path: ':pathMatch(.*)*', component: AdminNotFound }
    ]
  },
  // 全局404
  { path: '/:pathMatch(.*)*', component: GlobalNotFound }
]
```

### 捕获路由参数

```vue
<!-- NotFound.vue -->
<script setup>
import { useRoute } from 'vue-router'
const route = useRoute()
console.log('未匹配的路径:', route.params.pathMatch)
</script>
```

## 三、注意事项与常见陷阱

1. 通配符路由必须放在路由配置的**最后**
2. `/:pathMatch(.*)*`中的`*`表示捕获所有子路径段
3. 嵌套路由中的404只匹配该层的未定义路径
4. 访问404页面时返回404状态码需服务器配合
5. 也可以将404页面做成重定向到首页
