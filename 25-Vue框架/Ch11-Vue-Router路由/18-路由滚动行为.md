# 路由滚动行为

## 一、概念说明

`scrollBehavior`控制路由切换时页面的滚动位置。可以返回顶部、恢复之前的位置、滚动到锚点等。

```js
const router = createRouter({
  history: createWebHistory(),
  routes,
  scrollBehavior(to, from, savedPosition) {
    // 有保存的位置（浏览器前进后退）
    if (savedPosition) return savedPosition
    // 有hash锚点
    if (to.hash) return { el: to.hash, behavior: 'smooth' }
    // 默认返回顶部
    return { top: 0, behavior: 'smooth' }
  }
})
```

## 二、具体用法

### 常用配置

```js
scrollBehavior(to, from, savedPosition) {
  // 1. 浏览器前进后退：恢复位置
  if (savedPosition) return savedPosition

  // 2. 锚点滚动
  if (to.hash) {
    return {
      el: to.hash,
      behavior: 'smooth',
      top: 60  // 偏移量（如固定导航栏高度）
    }
  }

  // 3. 特定路由返回顶部
  if (to.meta.scrollToTop) return { top: 0 }

  // 4. 默认不滚动
  return false
}
```

### 异步滚动

```js
scrollBehavior(to, from, savedPosition) {
  return new Promise((resolve) => {
    setTimeout(() => {
      resolve({ top: 0 })
    }, 300)  // 等待页面内容加载
  })
}
```

## 三、注意事项与常见陷阱

1. `savedPosition`只在浏览器前进后退时有值
2. `el`可以是CSS选择器或DOM元素
3. `behavior: 'smooth'`需浏览器支持
4. 异步组件加载期间，滚动位置可能不准确，考虑延时滚动
5. hash模式下锚点功能与hash路由冲突，需要特殊处理
