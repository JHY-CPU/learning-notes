# 路由守卫-全局守卫

## 一、概念说明

全局路由守卫在**每次导航**时触发，用于统一的权限检查、登录验证等。三种守卫：`beforeEach`（前置）、`beforeResolve`（解析）、`afterEach`（后置）。

```js
// router/index.js
const router = createRouter({ /* ... */ })

// 全局前置守卫 - 导航前触发
router.beforeEach((to, from) => {
  if (to.meta.requiresAuth && !isAuthenticated) {
    return '/login'  // 重定向
  }
})

// 全局解析守卫 - 组件解析后触发
router.beforeResolve((to) => {
  console.log('即将进入:', to.path)
})

// 全局后置守卫 - 导航完成后触发
router.afterEach((to) => {
  document.title = to.meta.title || '默认标题'
})
```

## 二、具体用法

### 完整的守卫流程

```
导航触发
  -> beforeEach (全局)
  -> beforeRouteLeave (组件内)
  -> beforeEnter (独享)
  -> 组件解析
  -> beforeRouteUpdate (组件内, 复用时)
  -> beforeResolve (全局)
  -> 导航确认
  -> afterEach (全局)
  -> DOM更新
```

### 登录验证示例

```js
router.beforeEach((to, from) => {
  const token = localStorage.getItem('token')

  // 不需要认证的页面
  if (!to.meta.requiresAuth) return true

  // 需要认证但未登录
  if (!token) {
    return {
      path: '/login',
      query: { redirect: to.fullPath }
    }
  }

  return true  // 放行
})
```

### 返回值含义

| 返回值 | 行为 |
|--------|------|
| `true` / `undefined` | 放行 |
| `false` | 取消导航 |
| 路径字符串 `/login` | 重定向 |
| 路由对象 `{ name: 'Login' }` | 重定向 |
| `Error`实例 | 终止导航并报错 |

## 三、注意事项与常见陷阱

1. `beforeEach`中**必须返回**值（true/false/路径），不返回等同于`false`
2. 守卫是异步的，可以返回Promise
3. 不要在守卫中修改`to`和`from`（只读）
4. `afterEach`没有`next`参数，不能阻止导航
5. 守卫按注册顺序执行
