# 路由错误处理

## 一、概念说明

Vue Router提供导航错误处理机制，可以捕获导航失败并做出相应处理。导航失败通常是由于守卫返回`false`或重定向导致的。

```js
const router = createRouter({ /* ... */ })

// 全局错误处理
router.onError((error) => {
  console.error('路由错误:', error)
})

// push/replace返回Promise
router.push('/about').catch((error) => {
  if (isNavigationFailure(error)) {
    console.log('导航被取消:', error.type)
  }
})
```

## 二、具体用法

### 导航失败类型

```js
import { isNavigationFailure, NavigationFailureType } from 'vue-router'

router.push('/target').catch((failure) => {
  if (isNavigationFailure(failure, NavigationFailureType.aborted)) {
    // 导航被守卫取消
    console.log('导航被取消')
  } else if (isNavigationFailure(failure, NavigationFailureType.cancelled)) {
    // 导航被新导航取消（如快速点击）
    console.log('导航被覆盖')
  } else if (isNavigationFailure(failure, NavigationFailureType.duplicated)) {
    // 重复导航（跳转到当前路径）
    console.log('重复导航')
  }
})
```

### 区分错误类型

| 类型 | 说明 | 常见原因 |
|------|------|----------|
| `aborted` | 守卫返回false | 权限不足 |
| `cancelled` | 新的导航覆盖 | 快速切换路由 |
| `duplicated` | 跳转到当前路径 | 重复点击 |

### 全局错误兜底

```js
router.onError((error) => {
  // 组件加载失败（网络问题）
  if (error.message.includes('Loading chunk')) {
    window.location.reload()  // 刷新页面
  }
})
```

## 三、注意事项与常见陷阱

1. `NavigationFailure`是预期的导航取消，不是真正的错误
2. 普通`Error`才是真正的错误（如组件加载失败）
3. 使用`isNavigationFailure()`区分预期失败和真正错误
4. 快速切换路由会产生`cancelled`类型失败，通常可忽略
5. 组件懒加载失败时，可提示用户刷新页面
