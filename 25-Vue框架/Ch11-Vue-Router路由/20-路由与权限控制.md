# 路由与权限控制

## 一、概念说明

结合路由守卫和Pinia实现完整的权限控制系统：登录验证、角色权限、动态路由等。

```js
// router/guards.js
export function setupGuards(router) {
  // 白名单
  const whiteList = ['/login', '/404', '/403']

  router.beforeEach(async (to) => {
    const auth = useAuthStore()

    // 白名单直接放行
    if (whiteList.includes(to.path)) return true

    // 未登录跳登录
    if (!auth.isLoggedIn) {
      return { path: '/login', query: { redirect: to.fullPath } }
    }

    // 已登录访问登录页跳首页
    if (to.path === '/login') return '/'

    // 检查角色权限
    if (to.meta.roles && !to.meta.roles.includes(auth.role)) {
      return '/403'
    }

    return true
  })
}
```

## 二、具体用法

### 动态路由

```js
// 根据角色动态添加路由
const adminRoutes = [
  { path: 'users', component: UserManagement, meta: { roles: ['admin'] } },
  { path: 'logs', component: SystemLogs, meta: { roles: ['admin'] } }
]

const editorRoutes = [
  { path: 'articles', component: ArticleEditor, meta: { roles: ['admin', 'editor'] } }
]

router.beforeEach(async (to) => {
  const auth = useAuthStore()
  if (!auth.isRoutesAdded) {
    if (auth.role === 'admin') {
      adminRoutes.forEach(r => router.addRoute('Layout', r))
    }
    if (['admin', 'editor'].includes(auth.role)) {
      editorRoutes.forEach(r => router.addRoute('Layout', r))
    }
    auth.isRoutesAdded = true
    return to.fullPath  // 重新导航
  }
})
```

### 按钮级权限

```vue
<template>
  <button v-if="hasPermission('user:delete')">删除用户</button>
</template>

<script setup>
import { useAuthStore } from '@/stores/auth'
const auth = useAuthStore()
const hasPermission = (perm) => auth.permissions.includes(perm)
</script>
```

## 三、注意事项与常见陷阱

1. 动态路由添加后需**重新导航**（`return to.fullPath`）
2. 刷新页面后动态路由丢失，需重新添加
3. 404路由应在动态路由添加**之后**定义
4. 权限数据可从后端获取，存入store
5. 避免在前端硬编码所有路由，后端返回用户可访问的路由
