# 自定义指令实战 - 防抖

## 一、概念说明
`v-debounce` 指令用于给事件处理函数添加**防抖（debounce）**功能，防止用户频繁点击或输入导致重复请求。

## 二、具体用法

### 2.1 基础防抖指令
```js
// directives/debounce.js
export const debounceDirective = {
  mounted(el, binding) {
    const { value, arg = 'click', modifiers } = binding
    const delay = parseInt(Object.keys(modifiers)[0]) || 500

    let timer = null

    el.addEventListener(arg, (e) => {
      if (timer) clearTimeout(timer)
      timer = setTimeout(() => {
        value(e)
      }, delay)
    })

    el._debounceCleanup = () => {
      if (timer) clearTimeout(timer)
      el.removeEventListener(arg, value)
    }
  },
  unmounted(el) {
    el._debounceCleanup?.()
  }
}
```

### 2.2 使用防抖指令
```vue
<template>
  <!-- 点击防抖，500ms 内只执行一次 -->
  <button v-debounce="handleSubmit">提交</button>

  <!-- 输入防抖，300ms 后执行 -->
  <input v-debounce:input.300="onSearch" placeholder="搜索" />

  <!-- 自定义延迟 -->
  <button v-debounce:click.1000="handleSlowClick">慢速点击</button>
</template>
<script setup>
function handleSubmit() {
  console.log('提交表单')
}

function onSearch(e) {
  console.log('搜索:', e.target.value)
}

function handleSlowClick() {
  console.log('1秒内只触发一次')
}
</script>
```

### 2.3 带立即执行的防抖
```js
// leading debounce: 第一次立即执行，之后防抖
export const debounceDirective = {
  mounted(el, binding) {
    const { value, arg = 'click' } = binding
    const delay = 500
    let timer = null

    const handler = () => {
      if (timer) {
        clearTimeout(timer)
      } else {
        value()  // 第一次立即执行
      }
      timer = setTimeout(() => { timer = null }, delay)
    }

    el.addEventListener(arg, handler)
  }
}
```

### 2.4 实际场景：搜索框
```vue
<template>
  <input
    v-debounce:input.300="handleSearch"
    placeholder="输入关键词搜索..."
  />
</template>
<script setup>
function handleSearch(e) {
  fetch(`/api/search?q=${e.target.value}`)
}
</script>
```

## 三、注意事项与常见陷阱
- 防抖延迟要合理：按钮 300-500ms，输入框 300ms
- 在 `unmounted` 中清理定时器，防止内存泄漏
- 同一个元素上不要同时用 v-debounce 和 v-throttle
- 考虑使用 lodash 的 debounce 作为底层实现，更健壮
