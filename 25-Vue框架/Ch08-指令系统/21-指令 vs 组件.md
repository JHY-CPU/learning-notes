# 指令 vs 组件

## 一、概念说明
指令和组件都可以实现 DOM 行为的封装，但适用场景不同。选择合适的工具可以让代码更简洁和可维护。

## 二、对比分析

### 2.1 何时用指令
```
指令适合: 对单个 DOM 元素的底层操作
- 聚焦 (v-focus)
- 权限控制 (v-permission)
- 点击外部 (v-click-outside)
- 懒加载 (v-lazy)
- 防抖/节流
- 工具提示 (v-tooltip)
```

### 2.2 何时用组件
```
组件适合: 包含模板、样式、复杂逻辑的 UI 单元
- 模态框 (Modal)
- 下拉菜单 (Dropdown)
- 日期选择器 (DatePicker)
- 表格 (Table)
- 导航栏 (Navbar)
```

### 2.3 对比表
| 特性 | 指令 | 组件 |
|------|------|------|
| 操作 | 原生 DOM | Vue 响应式 |
| 模板 | 无 | 有 |
| 复用粒度 | 单个元素 | UI 单元 |
| 复杂度 | 低 | 中-高 |
| 样式 | 不封装 | 封装 |

### 2.4 示例对比
```vue
<!-- 指令：简单 DOM 操作 -->
<script setup>
const vFocus = {
  mounted: (el) => el.focus()
}
</script>
<template><input v-focus /></template>

<!-- 组件：复杂 UI 逻辑 -->
<template>
  <div class="modal" v-if="visible">
    <header><slot name="header" /></header>
    <main><slot /></main>
    <footer><slot name="footer" /></footer>
  </div>
</template>
```

### 2.5 决策流程
```
1. 需要模板/样式？ → 组件
2. 需要操作多个元素？ → 组件
3. 只操作单个 DOM 属性/事件？ → 指令
4. 逻辑很简短？ → 指令
5. 以上都不是？ → 考虑组合式函数
```

## 三、注意事项与常见陷阱
- 不要把复杂逻辑塞进指令，该用组件就用组件
- 指令不能做组件能做的事（模板、插槽、响应式状态）
- 组合式函数也可以替代某些指令的功能
- 两者并不互斥，可以配合使用
