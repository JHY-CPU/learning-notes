# 指令最佳实践

## 一、概念说明
自定义指令是强大的工具，但需要遵循最佳实践来确保代码的可读性、可维护性和性能。

## 二、最佳实践

### 2.1 单一职责
```js
// ✅ 每个指令只做一件事
const vFocus = {
  mounted: (el) => el.focus()
}

const vTooltip = {
  mounted: (el, binding) => { el.title = binding.value }
}

// ❌ 一个指令做太多事
const vDoEverything = {
  mounted(el, binding) {
    el.focus()
    el.title = binding.value
    el.style.color = 'red'
    el.addEventListener('click', handler)
  }
}
```

### 2.2 避免副作用和内存泄漏
```js
// ✅ 好的实践：清理所有副作用
export const vEventListener = {
  mounted(el, binding) {
    const handler = (e) => binding.value(e)
    el.addEventListener('click', handler)
    el._handler = handler  // 保存引用以便清理
  },
  unmounted(el) {
    el.removeEventListener('click', el._handler)
    delete el._handler
  }
}
```

### 2.3 使用有意义的参数和修饰符
```vue
<!-- ✅ 清晰的语义 -->
<button v-tooltip:top="'提示文字'">按钮</button>
<p v-format:uppercase>自动大写</p>
<img v-lazy.lazy="imageUrl" />

<!-- ❌ 不清晰 -->
<button v-x="'提示'">按钮</button>
```

### 2.4 提供默认行为和错误处理
```js
const vColor = {
  mounted(el, binding) {
    if (!binding.value) {
      console.warn('v-color 需要一个颜色值')
      return
    }
    el.style.color = binding.value
  }
}
```

### 2.5 文档和类型
```ts
/**

 * @param arg - 事件名（默认 click）
 * @param modifier - 延迟毫秒数（默认 500）
 * @param value - 回调函数
 */
export const vDebounce: Directive<HTMLElement, Function> = {
  // ...
}
```

## 三、注意事项与常见陷阱
- 始终在 `unmounted` 中清理事件监听和定时器
- 不要在指令中访问组件实例（`binding.instance` 应谨慎使用）
- 全局指令的命名要加前缀避免冲突
- 复杂逻辑应移至组合式函数，指令只做 DOM 层面的事情
