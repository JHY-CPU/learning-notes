# 自定义指令实战 - 权限控制

## 一、概念说明
`v-permission` 是常见的业务指令，用于根据用户权限**控制元素的显示/隐藏**。比 v-if 更优雅，比组件更轻量。

## 二、具体用法

### 2.1 基础权限指令
```js
// directives/permission.js
export const permissionDirective = {
  mounted(el, binding) {
    const { value } = binding  // 权限标识，如 'admin'、'edit'
    const userPermissions = getUserPermissions()  // 从 store 或 token 获取

    if (value && !userPermissions.includes(value)) {
      // 无权限：移除元素
      el.parentNode?.removeChild(el)
    }
  }
}

function getUserPermissions() {
  // 从 localStorage、Pinia store 或 JWT token 获取
  return JSON.parse(localStorage.getItem('permissions') || '[]')
}
```

### 2.2 在组件中使用
```vue
<template>
  <!-- 只有 admin 权限可见 -->
  <button v-permission="'admin'">删除用户</button>

  <!-- 只有 edit 权限可见 -->
  <button v-permission="'edit'">编辑</button>

  <!-- 无权限时隐藏（不移除） -->
  <button v-permission:hide="'admin'">管理</button>
</template>
```

### 2.3 支持多个权限（满足其一）
```js
export const permissionDirective = {
  mounted(el, binding) {
    const { value, arg } = binding
    const permissions = getUserPermissions()
    const requiredPerms = Array.isArray(value) ? value : [value]
    const hasPermission = requiredPerms.some(p => permissions.includes(p))

    if (!hasPermission) {
      if (arg === 'hide') {
        el.style.display = 'none'
      } else {
        el.parentNode?.removeChild(el)
      }
    }
  }
}
```

### 2.4 支持权限码数组
```vue
<template>
  <!-- 满足任一权限即可 -->
  <button v-permission="['admin', 'superAdmin']">高级操作</button>
</template>
```

## 三、注意事项与常见陷阱
- 权限数据应该来自后端，不要在前端硬编码
- 移除 DOM 元素是不可逆的，考虑用 v-show 代替
- 权限变化时指令不会重新执行，需要配合响应式数据
- 注意安全性：前端隐藏不等于安全，后端也要做权限校验
