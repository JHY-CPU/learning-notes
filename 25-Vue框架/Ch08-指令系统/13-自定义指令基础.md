# 自定义指令基础

## 一、概念说明
Vue 允许注册**自定义指令**来封装可复用的 DOM 操作逻辑。指令提供了一组生命周期钩子，在元素的不同阶段执行代码。

## 二、具体用法

### 2.1 在 script setup 中注册局部指令
```vue
<script setup>
// 在 <script setup> 中，任何以 v 开头的驼峰命名变量都会被注册为指令
const vFocus = {
  mounted: (el) => el.focus()
}
</script>

<template>
  <input v-focus placeholder="自动聚焦" />
</template>
```

### 2.2 指令钩子函数
```js
const myDirective = {
  // 绑定到元素前
  created(el, binding, vnode, prevVnode) {},
  // 插入 DOM 前
  beforeMount(el, binding, vnode, prevVnode) {},
  // 插入 DOM 后
  mounted(el, binding, vnode, prevVnode) {},
  // 更新前
  beforeUpdate(el, binding, vnode, prevVnode) {},
  // 更新后
  updated(el, binding, vnode, prevVnode) {},
  // 卸载前
  beforeUnmount(el, binding, vnode, prevVnode) {},
  // 卸载后
  unmounted(el, binding, vnode, prevVnode) {}
}
```

### 2.3 简写形式（函数）
```vue
<script setup>
// 如果只需要 mounted 和 updated，可以用函数简写
const vColor = (el, binding) => {
  el.style.color = binding.value
}
</script>

<template>
  <p v-color="'red'">红色文字</p>
</template>
```

### 2.4 全局注册
```js
// main.js
const app = createApp(App)

app.directive('focus', {
  mounted: (el) => el.focus()
})

app.mount('#app')
```

## 三、注意事项与常见陷阱
- 自定义指令操作的是原生 DOM，不是 Vue 组件
- 在 `<script setup>` 中使用驼峰命名：`vMyDirective`
- 函数简写等价于 `{ mounted: fn, updated: fn }`
- 指令不能访问组件实例，只能通过 binding 传递数据

## 四、binding 对象详解

```js
const myDirective = {
  mounted(el, binding) {
    console.log(binding.value)      // 指令的值 v-my="'hello'" → 'hello'
    console.log(binding.arg)        // 参数 v-my:foo → 'foo'
    console.log(binding.modifiers)  // 修饰符 v-my.foo.bar → { foo: true, bar: true }
    console.log(binding.instance)   // 使用指令的组件实例
    console.log(binding.dir)        // 指令定义对象
    console.log(binding.oldValue)   // 上一个值（updated 中可用）
  }
}
```

## 五、实用自定义指令示例

### 5.1 点击外部关闭
```vue
<script setup>
const vClickOutside = {
  mounted(el, binding) {
    el._clickOutsideHandler = (e) => {
      if (!el.contains(e.target) && !e.target.closest('[data-no-outside]')) {
        binding.value()
      }
    }
    setTimeout(() => {
      document.addEventListener('click', el._clickOutsideHandler)
    }, 0)
  },
  unmounted(el) {
    document.removeEventListener('click', el._clickOutsideHandler)
  }
}
</script>

<template>
  <div v-clickOutside="closeDropdown" class="dropdown">
    <!-- 下拉内容 -->
  </div>
</template>
```

### 5.2 拖拽指令
```vue
<script setup>
const vDraggable = {
  mounted(el) {
    el.style.position = 'relative'
    let startX, startY, initialX, initialY

    el.addEventListener('mousedown', (e) => {
      startX = e.clientX
      startY = e.clientY
      initialX = el.offsetLeft
      initialY = el.offsetTop

      const onMove = (e) => {
        el.style.left = initialX + e.clientX - startX + 'px'
        el.style.top = initialY + e.clientY - startY + 'px'
      }

      const onUp = () => {
        document.removeEventListener('mousemove', onMove)
        document.removeEventListener('mouseup', onUp)
      }

      document.addEventListener('mousemove', onMove)
      document.addEventListener('mouseup', onUp)
    })
  }
}
</script>
```

### 5.3 长按指令
```vue
<script setup>
const vLongPress = {
  mounted(el, binding) {
    let timer = null
    const handler = (e) => {
      timer = setTimeout(() => {
        binding.value(e)
      }, binding.arg || 1000)
    }
    const cancel = () => clearTimeout(timer)

    el.addEventListener('mousedown', handler)
    el.addEventListener('mouseup', cancel)
    el.addEventListener('mouseleave', cancel)
    el._longPressCleanup = () => {
      el.removeEventListener('mousedown', handler)
      el.removeEventListener('mouseup', cancel)
      el.removeEventListener('mouseleave', cancel)
    }
  },
  unmounted(el) {
    el._longPressCleanup?.()
  }
}
</script>

<template>
  <button v-longPress:500="handleLongPress">长按 500ms</button>
</template>
```
