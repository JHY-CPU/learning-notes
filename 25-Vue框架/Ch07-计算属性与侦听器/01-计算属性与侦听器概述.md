# 计算属性与侦听器概述

## 一、概念说明
Vue 提供了三种响应式派生数据的方式：`computed`（计算属性）、`watch`（侦听器）和 `watchEffect`（自动侦听副作用）。它们各有适用场景。

## 二、对比总览

| 特性 | computed | watch | watchEffect |
|------|----------|-------|-------------|
| 返回值 | 有（派生值） | 无（副作用） | 无（副作用）|
| 依赖追踪 | 自动 | 显式指定 | 自动 |
| 缓存 | 有 | 无 | 无 |
| 懒执行 | 是 | 默认是 | 否（立即执行）|
| 典型用例 | 派生数据 | 数据变化响应 | DOM 副作用 |

### 2.1 computed - 派生值
```vue
<script setup>
import { ref, computed } from 'vue'
const firstName = ref('张')
const lastName = ref('三')
const fullName = computed(() => `${firstName.value} ${lastName.value}`)
</script>
```

### 2.2 watch - 监听特定数据变化
```vue
<script setup>
import { ref, watch } from 'vue'
const query = ref('')

watch(query, (newVal, oldVal) => {
  console.log(`搜索词从 "${oldVal}" 变为 "${newVal}"`)
})
</script>
```

### 2.3 watchEffect - 自动追踪依赖
```vue
<script setup>
import { ref, watchEffect } from 'vue'
const width = ref(100)

watchEffect(() => {
  console.log('宽度为:', width.value)
})
</script>
```

## 三、如何选择
- 需要**返回一个值**且依赖自动追踪 → `computed`
- 需要**监听特定数据**变化做副作用 → `watch`
- 需要**自动追踪多个依赖**做副作用 → `watchEffect`
- 不要在 computed 中做副作用（如 API 请求）

## 四、实际场景对比

### 4.1 购物车计算：用 computed
```vue
<script setup>
import { ref, computed } from 'vue'

const cart = ref([
  { name: 'Vue教程', price: 68, quantity: 2 },
  { name: 'React指南', price: 58, quantity: 1 },
  { name: 'Node实战', price: 78, quantity: 3 }
])

// computed 适合纯计算，有缓存
const totalPrice = computed(() =>
  cart.value.reduce((sum, item) => sum + item.price * item.quantity, 0)
)

const itemCount = computed(() =>
  cart.value.reduce((sum, item) => sum + item.quantity, 0)
)

// 链式 computed
const discountPrice = computed(() =>
  totalPrice.value > 200 ? totalPrice.value * 0.9 : totalPrice.value
)
</script>
```

### 4.2 搜索防抖：用 watch
```vue
<script setup>
import { ref, watch } from 'vue'

const searchQuery = ref('')
const searchResults = ref([])
let debounceTimer = null

// watch 适合做副作用：API 请求
watch(searchQuery, (query) => {
  clearTimeout(debounceTimer)
  debounceTimer = setTimeout(async () => {
    if (query.length < 2) return
    const res = await fetch(`/api/search?q=${encodeURIComponent(query)}`)
    searchResults.value = await res.json()
  }, 300)
})
</script>
```

### 4.3 DOM 副作用：用 watchEffect
```vue
<script setup>
import { ref, watchEffect } from 'vue'

const theme = ref('light')
const fontSize = ref(16)

// watchEffect 自动追踪多个依赖，适合同步 DOM 副作用
watchEffect(() => {
  document.body.className = `theme-${theme.value}`
  document.documentElement.style.fontSize = `${fontSize.value}px`
})
</script>
```

## 五、性能考量

| 方面 | computed | watch | watchEffect |
|------|----------|-------|-------------|
| 缓存机制 | 依赖不变则返回缓存值 | 无缓存 | 无缓存 |
| 触发时机 | 懒求值（访问时） | 值变化后异步 | 值变化后异步 |
| 内存占用 | 低（缓存单值） | 中 | 中 |
| 适用数据量 | 大量数据派生 | 单个/少数数据 | 多个依赖 |

## 六、常见错误模式

```js
// ❌ 错误：在 computed 中做副作用
const bad = computed(() => {
  saveToServer(count.value)  // 不要在 computed 中发请求
  return count.value * 2
})

// ❌ 错误：用 watch 代替 computed 返回值
const doubled = ref(0)
watch(count, (val) => {
  doubled.value = val * 2  // 应该用 computed
})

// ✅ 正确：computed 返回值
const doubled = computed(() => count.value * 2)

// ❌ 错误：watchEffect 中访问非响应式数据
let localCount = 0
watchEffect(() => {
  console.log(localCount)  // 不会追踪变化！
})

// ✅ 正确：使用 ref
const localCount = ref(0)
watchEffect(() => {
  console.log(localCount.value)  // 正确追踪
})
```
