# 常见陷阱与最佳实践

## 一、概念说明
使用 computed、watch 和 watchEffect 时有一些常见错误和最佳实践。掌握这些可以避免 bug 并写出更高效的代码。

## 二、常见陷阱

### 2.1 computed 中的副作用
```vue
<!-- ❌ 错误：在 computed 中执行副作用 -->
<script setup>
const count = ref(0)
const doubled = computed(() => {
  count.value++  // 修改其他状态 → 无限循环！
  return count.value * 2
})
</script>

<!-- ✅ computed 应该是纯函数 -->
<script setup>
const doubled = computed(() => count.value * 2)
</script>
```

### 2.2 循环依赖
```vue
<!-- ❌ 错误：A 依赖 B，B 依赖 A -->
<script setup>
const a = computed(() => b.value + 1)
const b = computed(() => a.value + 1)  // 无限循环
</script>

<!-- ✅ 重新设计数据关系 -->
<script setup>
const base = ref(0)
const a = computed(() => base.value + 1)
const b = computed(() => base.value + 2)
</script>
```

### 2.3 过度侦听
```vue
<!-- ❌ 不好：深度侦听大对象 -->
watch(bigObject, callback, { deep: true })

<!-- ✅ 好：只侦听需要的属性 -->
watch(() => bigObject.value.specificProp, callback)
```

### 2.4 忘记处理异步竞态
```vue
<script setup>
const id = ref(1)

// ❌ 忽略竞态条件
watch(id, async (newId) => {
  data.value = await fetchData(newId)
})

// ✅ 使用 watchEffect + onInvalidate
watchEffect(async (onInvalidate) => {
  let cancelled = false
  onInvalidate(() => { cancelled = true })
  const result = await fetchData(id.value)
  if (!cancelled) data.value = result
})
</script>
```

### 2.5 computed 不更新
```vue
<!-- 常见原因：依赖的数据不是响应式的 -->
<script setup>
let count = 0  // ❌ 不是响应式
const doubled = computed(() => count * 2)  // 永远是 0

const countRef = ref(0)  // ✅ 响应式
const doubledOk = computed(() => countRef.value * 2)
</script>
```

## 三、最佳实践总结
1. 派生数据用 computed，副作用用 watch/watchEffect
2. computed 必须是纯函数，不能有副作用
3. 优先使用 getter 函数精确侦听属性
4. 异步操作要考虑竞态条件
5. 深度侦听大对象有性能开销，尽量精确侦听
