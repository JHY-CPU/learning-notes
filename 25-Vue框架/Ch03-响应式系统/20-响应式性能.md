# 响应式性能优化

## 一、概念说明

Vue 3 的响应式系统虽然高效，但在处理大型数据或复杂场景时仍需注意性能。主要优化策略包括：避免不必要的响应式转换、使用浅层响应式、合理使用 computed 缓存。

```vue
<script setup>
import { shallowRef, markRaw, ref } from 'vue'

const chartInstance = shallowRef(null)

const staticConfig = markRaw({
  apiUrl: 'https://api.example.com',
  timeout: 5000
})
</script>
```

## 二、具体用法

### 2.1 避免深度响应式

```vue
<script setup>
import { ref, shallowRef } from 'vue'

// 差: 大型数组深度响应式
const bigList = ref(new Array(10000).fill({ data: '...' }))

// 好: 浅层响应式 + 整体替换
const bigListOptimized = shallowRef(new Array(10000).fill({ data: '...' }))

function updateList(newList) {
  bigListOptimized.value = newList
}
</script>
```

### 2.2 markRaw 跳过响应式

```vue
<script setup>
import { reactive, markRaw } from 'vue'

const state = reactive({
  config: markRaw({ /* 大型不可变配置 */ }),
  data: [] // 这个需要响应式
})
</script>
```

### 2.3 computed 缓存

```vue
<script setup>
import { ref, computed } from 'vue'
const items = ref([])

// 好: 有缓存，items 不变时不重新计算
const expensiveResult = computed(() =>
  items.value.filter(i => i.active).map(i => i.name)
)
</script>
```

### 2.4 虚拟滚动

```vue
<script setup>
import { ref } from 'vue'

const items = ref(new Array(10000).fill(null).map((_, i) => ({ id: i, text: `Item ${i}` })))
</script>

<template>
  <!-- 使用虚拟滚动库，只渲染可见区域 -->
  <RecycleScroller :items="items" :item-size="50" key-field="id">
    <template #default="{ item }">
      <div>{{ item.text }}</div>
    </template>
  </RecycleScroller>
</template>
```

### 2.5 减少不必要的响应式

```vue
<script setup>
import { ref, shallowRef, markRaw } from 'vue'

// ✅ 常量不需要响应式
const API_URL = 'https://api.example.com'

// ✅ 配置对象不需要响应式
const config = markRaw({ timeout: 5000, retries: 3 })

// ✅ 第三方实例用 shallowRef
const chart = shallowRef(null)
</script>
```

## 三、常见用例

### 3.1 性能对比

```
ref + 深度响应式（10000 项数组）：
  首次创建: ~50ms
  每次修改: 遍历所有嵌套属性

shallowRef（10000 项数组）：
  首次创建: ~5ms
  每次修改: 仅检测 .value 替换
```

## 四、注意事项与常见陷阱

- 不需要响应式的数据不要用 `ref`/`reactive` 包装
- 大型列表考虑虚拟滚动（如 vue-virtual-scroller）
- `markRaw` 后的数据不会被 `reactive` 转换
- `Object.freeze()` 可以阻止修改但不阻止 Vue 的响应式转换
- 对于只读数据，使用 `readonly()` 而非 `reactive()`
- Vue DevTools 中可以查看响应式依赖数量
- 响应式数据的代理有内存开销，不需要时及时清理
- computed 的缓存可以避免重复计算，但本身也有开销
