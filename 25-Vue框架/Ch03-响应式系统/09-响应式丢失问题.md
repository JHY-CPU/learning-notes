# 响应式丢失问题

## 一、概念说明

响应式丢失是 Vue 3 开发中的常见问题。主要原因包括：解构 reactive 对象、直接赋值 ref、传递响应式数据时丢失代理等。理解这些陷阱能避免难以调试的 bug。

```vue
<script setup>
import { reactive, ref, toRefs } from 'vue'

// 问题1: 解构 reactive 丢失响应式
const state = reactive({ count: 0 })
let { count } = state  // count 不再是响应式的！

// 问题2: 直接赋值丢失 ref
const myRef = ref(0)
// myRef = ref(1) // 错误！丢失了响应式

// 正确做法
myRef.value = 1
</script>
```

## 二、具体用法

### 2.1 reactive 解构丢失

```vue
<script setup>
import { reactive, toRefs, toRef } from 'vue'

const state = reactive({ count: 0, name: 'Vue' })

// 错误: 解构后丢失响应式
// let { count, name } = state

// 正确: 使用 toRefs
const { count, name } = toRefs(state)
count.value++ // 保持响应式

// 或使用 toRef 取单个属性
const countOnly = toRef(state, 'count')
countOnly.value++
</script>
```

### 2.2 ref 重新赋值丢失

```vue
<script setup>
import { ref } from 'vue'

const myRef = ref(0)

// 错误: 直接重新赋值（在 const 声明下还会报错）
// myRef = ref(1)

// 正确: 修改 .value
myRef.value = 1

// 用 let 声明可以重新赋值 ref 本身
let count = ref(0)
count = ref(1) // 可以，但引用会变
</script>
```

### 2.3 函数传参丢失

```vue
<script setup>
import { ref } from 'vue'

const count = ref(0)

// 错误: 传递 .value 会丢失 ref
// someFunction(count.value)

// 正确: 传递 ref 本身
someFunction(count)

function someFunction(r) {
  r.value++
}
</script>
```

### 2.4 数组替换丢失

```vue
<script setup>
import { reactive } from 'vue'

const state = reactive({ items: [1, 2, 3] })

// ✅ 使用数组方法（保持响应式）
state.items.push(4)
state.items.splice(0, 1)

// ✅ 整体替换 reactive 的属性
state.items = [4, 5, 6]

// ❌ 替换整个 reactive 对象会丢失
// state = { items: [4, 5, 6] }  // 丢失响应式
</script>
```

### 2.5 解构 props 丢失

```vue
<script setup>
// ❌ 错误：解构 props 丢失响应式
// const { title } = defineProps({ title: String })

// ✅ 正确：使用 toRefs
import { toRefs } from 'vue'
const props = defineProps({ title: String })
const { title } = toRefs(props)
// 或直接使用 props.title（保持响应式）
</script>
```

## 三、常见用例

### 3.1 自定义 hook 中的响应式

```js
// composables/useCounter.js
import { ref } from 'vue'

export function useCounter() {
  const count = ref(0)
  const increment = () => count.value++
  const decrement = () => count.value--

  // ✅ 返回整个 ref，保持响应式
  return { count, increment, decrement }

  // ❌ 错误：返回 .value 丢失响应式
  // return { count: count.value, increment, decrement }
}
```

## 四、注意事项与常见陷阱

- 解构 reactive 时必须使用 `toRefs` 保持响应式
- ref 的 `.value` 是原始值，传递它会丢失响应式
- reactive 对象赋新值会丢失响应式（`state = newObj`）
- 将响应式数据传入 `JSON.stringify()` 会丢失代理（但得到正确结果）
- `readonly()` 包装后的数据修改不会生效但不报错
- 在 setup 函数中 `return` reactive 对象时，解构也会丢失响应式
- 使用 TypeScript 时，类型系统可以帮助检测部分丢失问题
