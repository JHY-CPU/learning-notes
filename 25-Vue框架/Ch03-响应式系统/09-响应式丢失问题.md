# 响应式丢失问题

## 一、概念说明

响应式丢失是 Vue 3 开发中的常见问题。主要原因包括：解构 reactive 对象、直接赋值 ref、传递响应式数据时丢失代理等。理解这些陷阱能避免难以调试的 bug。

```vue
<script setup>
import { reactive, ref, toRefs } from 'vue'

// 问题1: 解构 reactive 丢失响应式
const state = reactive({ count: 0 })
let { count } = state  // count 不再是响应式的！

// 问题2: 直接赋值丢失 ref
const myRef = ref(0)
myRef = ref(1) // 错误！丢失了响应式

// 正确做法
myRef.value = 1
</script>
```

## 二、具体用法

### 2.1 reactive 解构丢失

```vue
<script setup>
import { reactive, toRefs } from 'vue'

const state = reactive({ count: 0, name: 'Vue' })

// 错误: 解构后丢失响应式
// let { count, name } = state

// 正确: 使用 toRefs
let { count, name } = toRefs(state)
count.value++ // 保持响应式
</script>
```

### 2.2 ref 重新赋值丢失

```vue
<script setup>
import { ref } from 'vue'

const myRef = ref(0)

// 错误: 直接重新赋值
// myRef = ref(1) // myRef 不再指向原来的 ref

// 正确: 修改 .value
myRef.value = 1

// 或者用 let 声明（可以重新赋值 ref 本身）
let count = ref(0)
count = ref(1) // 可以，但引用会变
</script>
```

### 2.3 函数传参丢失

```vue
<script setup>
import { ref } from 'vue'

const count = ref(0)

// 错误: 传递 .value 会丢失 ref
// someFunction(count.value)

// 正确: 传递 ref 本身
someFunction(count)

function someFunction(ref) {
  ref.value++ // 在函数内通过 .value 修改
}
</script>
```

## 三、注意事项与常见陷阱

- 解构 reactive 时必须使用 `toRefs` 保持响应式
- ref 的 `.value` 是原始值，传递它会丢失响应式
- reactive 对象赋新值会丢失响应式（`state = newObj`）
- 将响应式数据传入 `JSON.stringify()` 会丢失代理
- `readonly()` 包装后的数据修改不会生效但不报错
