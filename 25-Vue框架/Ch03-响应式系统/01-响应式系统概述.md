# 响应式系统概述

## 一、概念说明

Vue 3 的响应式系统基于 ES6 **Proxy** 实现。当数据被读取时（getter），Vue 追踪哪些组件或副作用依赖了这个数据；当数据被修改时（setter），Vue 自动触发依赖更新，实现**数据驱动视图**。

相比 Vue 2 的 `Object.defineProperty`，Vue 3 的 Proxy 方案可以拦截更多操作（属性增删、数组索引、length 修改等），且性能更优。

```vue
<script setup>
import { ref, reactive } from 'vue'

const count = ref(0)
const state = reactive({ name: 'Vue', version: 3 })

function increment() {
  count.value++
  state.version++
}
</script>

<template>
  <p>{{ count }}</p>
  <p>{{ state.name }} {{ state.version }}</p>
  <button @click="increment">+1</button>
</template>
```

## 二、具体用法

### 2.1 响应式数据创建

```js
import { ref, reactive, readonly, shallowRef } from 'vue'

// ref: 适合基本类型
const count = ref(0)
const message = ref('hello')

// reactive: 适合对象
const user = reactive({ name: '张三', age: 25 })

// readonly: 只读代理
const config = readonly({ debug: true })

// shallowRef: 浅层响应
const largeData = shallowRef({ items: [] })
```

### 2.2 ref 与 reactive 选择

```js
// ref: 适用于所有类型，模板中自动解包
const count = ref(0)
const user = ref({ name: '张' })
count.value++           // JS 中需要 .value
console.log(count)      // 模板中自动解包，显示 1

// reactive: 仅适用于对象，不能解构
const state = reactive({ count: 0, name: '张' })
state.count++           // 不需要 .value
// const { count } = state  // ❌ 解构丢失响应式
```

### 2.3 响应式工作流程

```
1. 组件渲染 → 读取响应式数据 → 触发 getter → 收集依赖
2. 数据修改 → 触发 setter → 通知依赖 → 组件重新渲染
3. 组件重新渲染 → 重复步骤 1
```

### 2.4 Proxy 原理简析

```js
// Vue 3 响应式的简化实现
function reactive(obj) {
  return new Proxy(obj, {
    get(target, key, receiver) {
      track(target, key)  // 追踪依赖
      return Reflect.get(target, key, receiver)
    },
    set(target, key, value, receiver) {
      Reflect.set(target, key, value, receiver)
      trigger(target, key)  // 触发更新
      return true
    },
    deleteProperty(target, key) {
      Reflect.deleteProperty(target, key)
      trigger(target, key)
      return true
    }
  })
}
```

## 三、常见用例

### 3.1 API 返回数据处理

```vue
<script setup>
import { ref, reactive } from 'vue'

const loading = ref(false)
const user = reactive({ name: '', email: '' })

async function fetchUser(id) {
  loading.value = true
  const res = await fetch(`/api/users/${id}`)
  const data = await res.json()
  Object.assign(user, data)  // reactive 对象可以用 Object.assign 更新
  loading.value = false
}
</script>
```

## 四、注意事项与常见陷阱

- `reactive()` 只能用于对象类型（对象、数组、Map、Set）
- 基本类型（string、number、boolean）必须用 `ref()`
- `reactive()` 返回的代理与原始对象不相等（`===` 为 false）
- 解构 `reactive()` 会丢失响应式，使用 `toRefs()`
- 响应式追踪在组件渲染期间才生效
- 直接替换 reactive 对象会丢失响应式（应使用 `Object.assign`）
- Vue 3.5+ 对 reactive 有更多优化，建议查阅最新文档
