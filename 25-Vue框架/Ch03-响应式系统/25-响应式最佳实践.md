# 响应式最佳实践

## 一、概念说明

合理使用 Vue 3 的响应式 API 能编写出高效、可维护的代码。核心原则：选择合适的数据类型、避免响应式丢失、善用 computed 缓存、合理使用浅层响应式。

```vue
<script setup>
import { ref, reactive, computed, toRefs } from 'vue'

// 好: 基本类型用 ref
const count = ref(0)
const name = ref('Vue')

// 好: 复杂表单用 reactive
const form = reactive({
  username: '',
  email: '',
  password: ''
})

// 好: 复杂计算用 computed
const isValid = computed(() =>
  form.username.length > 0 && form.email.includes('@')
)
</script>
```

## 二、具体用法

### 2.1 何时用 ref vs reactive

```vue
<script setup>
import { ref, reactive } from 'vue'

// ref: 基本类型、需要整体替换的数据
const count = ref(0)
const selectedItem = ref(null)
const apiResponse = ref<ApiResponse | null>(null)

// reactive: 多属性表单状态、全局配置
const form = reactive({ name: '', email: '' })
const settings = reactive({ theme: 'dark', lang: 'zh' })
</script>
```

### 2.2 避免常见错误

```vue
<script setup>
import { ref, reactive, toRefs } from 'vue'

// 错误1: 解构 reactive
const state = reactive({ count: 0 })
// const { count } = state  // 丢失响应式

// 正确: 使用 toRefs
const { count } = toRefs(state)

// 错误2: 覆盖 ref
const data = ref([])
// data = ref([1, 2])  // 丢失引用

// 正确: 修改 .value
data.value = [1, 2]
</script>
```

### 2.3 组合式函数中的响应式

```js
// composables/useAuth.js
import { ref, computed } from 'vue'

export function useAuth() {
  const user = ref(null)
  const token = ref(localStorage.getItem('token'))

  const isLoggedIn = computed(() => !!token.value)

  function login(credentials) {
    // 登录逻辑
  }

  function logout() {
    user.value = null
    token.value = null
  }

  // 返回 ref，保持响应式
  return { user, token, isLoggedIn, login, logout }
}
```

## 三、注意事项与常见陷阱

- 团队应统一 ref 或 reactive 的使用风格
- 不需要响应式的数据不要用 ref/reactive 包装
- 大型对象使用 shallowRef/shallowReactive 优化
- 函数返回响应式数据时使用 toRefs
- computed 只用于纯计算，副作用用 watch/watchEffect
- 保持响应式数据的扁平化，减少深层嵌套
