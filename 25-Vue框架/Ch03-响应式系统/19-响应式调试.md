# 响应式调试

## 一、概念说明

Vue 3 的 `watch` 和 `watchEffect` 提供了 `onTrack` 和 `onTrigger` 选项，用于调试响应式系统的依赖收集和触发更新。这些回调仅在开发环境生效。

```vue
<script setup>
import { ref, watch } from 'vue'

const count = ref(0)

watch(
  count,
  () => {
    console.log('count 变化了')
  },
  {
    onTrack(e) {
      // 依赖被收集时触发
      console.log('依赖追踪:', e)
    },
    onTrigger(e) {
      // 依赖触发更新时触发
      console.log('触发更新:', e)
    }
  }
)
</script>

<template>
  <button @click="count++">+1</button>
</template>
```

## 二、具体用法

### 2.1 调试 watch

```vue
<script setup>
import { ref, reactive, watch } from 'vue'

const state = reactive({ count: 0 })

watch(
  () => state.count,
  () => console.log('更新'),
  {
    onTrack({ target, key, type }) {
      console.log(`追踪: ${String(key)} (${type})`)
    },
    onTrigger({ target, key, type, newValue, oldValue }) {
      console.log(`触发: ${String(key)} ${oldValue}→${newValue} (${type})`)
    }
  }
)
</script>
```

### 2.2 调试 watchEffect

```vue
<script setup>
import { ref, watchEffect } from 'vue'

const a = ref(1)
const b = ref(2)

watchEffect(
  () => {
    console.log(`${a.value} + ${b.value} = ${a.value + b.value}`)
  },
  {
    onTrack(e) {
      console.log('watchEffect 追踪了:', e.key)
    },
    onTrigger(e) {
      console.log('watchEffect 被触发:', e.key)
    }
  }
)
</script>
```

## 三、注意事项与常见陷阱

- `onTrack` 和 `onTrigger` 仅在开发环境生效
- 回调参数包含 `target`、`key`、`type`、`newValue`、`oldValue` 等信息
- 不要在生产环境中使用这些调试选项
- 响应式调试可以帮助理解依赖关系和更新时机
- 复杂组件中使用调试选项可能产生大量日志
