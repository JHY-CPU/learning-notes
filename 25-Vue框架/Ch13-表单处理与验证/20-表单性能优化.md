# 表单性能优化

## 一、概念说明

大型表单（50+字段）可能有性能问题。优化策略包括：减少响应式数据、防抖验证、虚拟化、按需渲染等。

```vue
<script setup>
import { ref, shallowRef } from 'vue'
import { useDebounceFn } from '@vueuse/core'

// 大对象使用shallowRef
const formData = shallowRef({})

// 防抖验证
const validateField = useDebounceFn((field) => {
  // 验证逻辑
}, 300)
</script>
```

## 二、具体用法

### 分步表单（避免一次渲染大量字段）

```vue
<template>
  <div v-show="step === 1">
    <h3>基本信息</h3>
    <!-- 第一步字段 -->
  </div>
  <div v-show="step === 2">
    <h3>详细信息</h3>
    <!-- 第二步字段 -->
  </div>
  <button @click="step--" v-if="step > 1">上一步</button>
  <button @click="step++" v-if="step < 3">下一步</button>
</template>
```

### 防抖输入

```js
import { useDebounceFn } from '@vueuse/core'

const debouncedValidate = useDebounceFn(() => {
  validateAll()
}, 500)
```

### 懒加载验证

```js
// 只在字段获得焦点后才启用验证
const touched = reactive({})

const handleBlur = (field) => {
  touched[field] = true
}

// 模板中
const showError = (field) => touched[field] && errors[field]
```

## 四、按需渲染与虚拟化

```vue
<script setup>
import { ref, computed } from 'vue'

const fields = ref(generateFields(100))  // 100个字段
const visibleRange = ref({ start: 0, end: 20 })

// 只渲染可见范围内的字段
const visibleFields = computed(() =>
  fields.value.slice(visibleRange.value.start, visibleRange.value.end)
)

const onScroll = (e) => {
  const scrollTop = e.target.scrollTop
  const itemHeight = 60
  const start = Math.floor(scrollTop / itemHeight)
  visibleRange.value = { start, end: start + 20 }
}
</script>

<template>
  <div class="form-container" @scroll="onScroll" style="height: 600px; overflow-y: auto;">
    <div :style="{ height: fields.length * 60 + 'px' }">
      <div
        v-for="field in visibleFields"
        :key="field.name"
        :style="{ position: 'absolute', top: fields.indexOf(field) * 60 + 'px' }"
      >
        <label>{{ field.label }}</label>
        <input v-model="field.value" />
      </div>
    </div>
  </div>
</template>
```

## 五、减少响应式代理开销

```vue
<script setup>
import { shallowRef, shallowReactive, markRaw, triggerRef } from 'vue'

// 大型只读配置使用markRaw
const fieldConfigs = markRaw([
  { name: 'field1', label: '字段1', type: 'text' },
  // ... 大量配置
])

// 大型表单数据使用shallowRef
const formData = shallowRef({
  field1: '',
  field2: '',
  // ... 大量字段
})

// 更新后手动触发
const updateField = (name, value) => {
  formData.value = { ...formData.value, [name]: value }
  triggerRef(formData)
}
</script>
```

## 六、验证性能对比

```
策略                  每次按键验证次数    用户体验    适用场景
─────────────────────────────────────────────────────────
input实时验证         非常高            最好        简单表单
blur失焦验证          低                好        中型表单
debounce验证         中                好        搜索等场景
提交时验证            0                 差        极简表单
touched+blur验证      最低              好        大型表单
```

## 三、注意事项与常见陷阱

1. 大表单使用`shallowRef`或`markRaw`减少代理开销
2. 实时验证加debounce，避免每次按键都验证
3. 分步表单减少同时渲染的字段数
4. 验证规则设lazy（blur时触发），而非input时
5. 避免在验证中进行昂贵的操作（如DOM查询）
6. `markRaw`标记不需要响应式的配置数据
7. 虚拟化适用于100+字段的超大表单
