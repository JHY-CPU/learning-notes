# 表单最佳实践

## 一、概念说明

良好的表单设计提升用户体验和数据质量。最佳实践涵盖设计原则、交互模式、验证策略等方面。

```vue
<template>
  <form @submit.prevent="handleSubmit" novalidate>
    <!-- 1. 清晰的标签 -->
    <label for="email">邮箱 <span class="required">*</span></label>
    <input id="email" v-model="form.email" type="email"
      aria-required="true" :aria-invalid="!!errors.email"
      @blur="validate('email')" />
    <span v-if="touched.email && errors.email" class="error">
      {{ errors.email }}
    </span>

    <!-- 2. 友好的提交 -->
    <button type="submit" :disabled="!isValid || submitting">
      {{ submitting ? '提交中...' : '提交' }}
    </button>
  </form>
</template>
```

## 二、具体用法

### 设计原则

| 原则 | 做法 |
|------|------|
| 清晰标签 | 每个字段有明确的label |
| 即时反馈 | blur时验证，不等提交 |
| 明确错误 | 错误消息具体、可操作 |
| 防止丢失 | 失焦前不验证，草稿自动保存 |
| 键盘友好 | Tab导航，Enter提交 |

### 表单状态管理

```js
const formState = {
  data: reactive({}),         // 表单数据
  errors: reactive({}),       // 验证错误
  touched: reactive({}),      // 是否触碰
  submitting: ref(false),     // 提交状态
  dirty: computed(() => {     // 是否修改
    return JSON.stringify(data) !== JSON.stringify(initialData)
  })
}
```

### 国际化错误消息

```js
const messages = {
  'zh-CN': { required: '此字段必填', email: '邮箱格式不正确' },
  'en-US': { required: 'This field is required', email: 'Invalid email' }
}
```

## 三、注意事项与常见陷阱

1. 不要只在提交时验证（用户体验差）
2. 不要在用户输入时实时验证（太频繁）
3. 正确做法：blur时验证，input时清除错误
4. 保存草稿防止用户数据丢失
5. 提交按钮禁用提供视觉反馈
6. 移动端使用合适的输入类型（tel、email、number）
