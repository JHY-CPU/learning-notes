# 表单数据序列化

## 一、概念说明

表单数据需要序列化为不同格式发送：JSON（最常见）、FormData（文件上传）、URLSearchParams等。

```vue
<script setup>
const form = reactive({
  name: '张三',
  age: 25,
  avatar: null  // File对象
})

// JSON提交
const submitJSON = async () => {
  await fetch('/api/user', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(form)
  })
}

// FormData提交（支持文件）
const submitFormData = async () => {
  const fd = new FormData()
  fd.append('name', form.name)
  fd.append('age', form.age)
  fd.append('avatar', form.avatar)  // 文件

  await fetch('/api/user', {
    method: 'POST',
    body: fd  // 不要设Content-Type，浏览器自动设
  })
}
</script>
```

## 二、具体用法

### 对象转FormData

```js
function toFormData(obj) {
  const fd = new FormData()
  Object.entries(obj).forEach(([key, val]) => {
    if (val instanceof File) {
      fd.append(key, val)
    } else if (Array.isArray(val)) {
      val.forEach(v => fd.append(`${key}[]`, v))
    } else if (val !== null && val !== undefined) {
      fd.append(key, String(val))
    }
  })
  return fd
}
```

### 文件上传

```vue
<template>
  <input type="file" @change="handleFile" accept="image/*" />
  <img v-if="preview" :src="preview" />
</template>

<script setup>
const handleFile = (e) => {
  const file = e.target.files[0]
  form.avatar = file
  preview.value = URL.createObjectURL(file)
}
</script>
```

## 三、注意事项与常见陷阱

1. JSON提交用`JSON.stringify`，设置`Content-Type: application/json`
2. FormData提交**不要**手动设置`Content-Type`（浏览器自动加boundary）
3. 文件上传使用FormData，JSON不支持文件
4. 数组数据需要特殊处理（`key[]`格式）
5. 敏感数据不要放在URL中（使用POST请求）
