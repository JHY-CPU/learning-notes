# 表单数据序列化

## 一、概念说明

表单数据需要序列化为不同格式发送：JSON（最常见）、FormData（文件上传）、URLSearchParams等。

```vue
<script setup>
const form = reactive({
  name: '张三',
  age: 25,
  avatar: null  // File对象
})

// JSON提交
const submitJSON = async () => {
  await fetch('/api/user', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(form)
  })
}

// FormData提交（支持文件）
const submitFormData = async () => {
  const fd = new FormData()
  fd.append('name', form.name)
  fd.append('age', form.age)
  fd.append('avatar', form.avatar)  // 文件

  await fetch('/api/user', {
    method: 'POST',
    body: fd  // 不要设Content-Type，浏览器自动设
  })
}
</script>
```

## 二、具体用法

### 对象转FormData

```js
function toFormData(obj) {
  const fd = new FormData()
  Object.entries(obj).forEach(([key, val]) => {
    if (val instanceof File) {
      fd.append(key, val)
    } else if (Array.isArray(val)) {
      val.forEach(v => fd.append(`${key}[]`, v))
    } else if (val !== null && val !== undefined) {
      fd.append(key, String(val))
    }
  })
  return fd
}
```

### 文件上传

```vue
<template>
  <input type="file" @change="handleFile" accept="image/*" />
  <img v-if="preview" :src="preview" />
</template>

<script setup>
const handleFile = (e) => {
  const file = e.target.files[0]
  form.avatar = file
  preview.value = URL.createObjectURL(file)
}
</script>
```

## 四、深层嵌套对象转FormData

```js
function objectToFormData(obj, prefix = '') {
  const fd = new FormData()

  for (const [key, value] of Object.entries(obj)) {
    const formKey = prefix ? `${prefix}[${key}]` : key

    if (value instanceof File || value instanceof Blob) {
      fd.append(formKey, value)
    } else if (Array.isArray(value)) {
      value.forEach((item, index) => {
        if (typeof item === 'object' && item !== null) {
          const nested = objectToFormData(item, `${formKey}[${index}]`)
          for (const [k, v] of nested.entries()) {
            fd.append(k, v)
          }
        } else {
          fd.append(`${formKey}[${index}]`, item)
        }
      })
    } else if (typeof value === 'object' && value !== null) {
      const nested = objectToFormData(value, formKey)
      for (const [k, v] of nested.entries()) {
        fd.append(k, v)
      }
    } else if (value !== null && value !== undefined) {
      fd.append(formKey, String(value))
    }
  }

  return fd
}

// 使用
const data = {
  user: { name: '张三', age: 25 },
  tags: ['前端', 'Vue'],
  avatar: fileObject
}
const formData = objectToFormData(data)
// 生成: user[name]=张三, user[age]=25, tags[0]=前端, tags[1]=Vue, avatar=[File]
```

## 五、URL查询参数序列化

```js
function toSearchParams(obj) {
  const params = new URLSearchParams()

  Object.entries(obj).forEach(([key, value]) => {
    if (Array.isArray(value)) {
      value.forEach(v => params.append(key, v))
    } else if (value !== null && value !== undefined) {
      params.append(key, String(value))
    }
  })

  return params
}

// 使用
const filters = { page: 1, pageSize: 20, category: 'tech', tags: ['vue', 'react'] }
const params = toSearchParams(filters)
// page=1&pageSize=20&category=tech&tags=vue&tags=react
const url = `/api/articles?${params.toString()}`
```

## 六、序列化对比与选择

| 格式 | 适用场景 | Content-Type | 文件支持 | 复杂度 |
|------|---------|-------------|---------|--------|
| JSON | API通用 | application/json | 不支持 | 低 |
| FormData | 文件上传 | multipart/form-data | 支持 | 中 |
| URLSearchParams | GET请求 | application/x-www-form-urlencoded | 不支持 | 低 |
| x-www-form-urlencoded | 简单表单提交 | application/x-www-form-urlencoded | 不支持 | 低 |

## 三、注意事项与常见陷阱

1. JSON提交用`JSON.stringify`，设置`Content-Type: application/json`
2. FormData提交**不要**手动设置`Content-Type`（浏览器自动加boundary）
3. 文件上传使用FormData，JSON不支持文件
4. 数组数据需要特殊处理（`key[]`格式）
5. 敏感数据不要放在URL中（使用POST请求）
6. 深层嵌套对象需要递归转换FormData
7. `JSON.stringify`会丢失`undefined`、`Function`、`Symbol`类型
