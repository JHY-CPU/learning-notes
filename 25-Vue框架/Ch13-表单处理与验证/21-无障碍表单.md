# 无障碍表单

## 一、概念说明

无障碍（Accessibility/a11y）确保表单对所有用户可用，包括使用屏幕阅读器和键盘导航的用户。关键包括语义化HTML、ARIA属性、键盘支持。

```vue
<template>
  <form @submit.prevent="handleSubmit" aria-label="登录表单">
    <div>
      <label for="email">邮箱</label>
      <input
        id="email"
        v-model="form.email"
        type="email"
        aria-required="true"
        :aria-invalid="!!errors.email"
        aria-describedby="email-error"
      />
      <span id="email-error" v-if="errors.email" role="alert">
        {{ errors.email }}
      </span>
    </div>

    <div>
      <label for="password">密码</label>
      <input
        id="password"
        v-model="form.password"
        type="password"
        aria-required="true"
        :aria-invalid="!!errors.password"
      />
    </div>

    <button type="submit">登录</button>
  </form>
</template>
```

## 二、具体用法

### 关键ARIA属性

| 属性 | 用途 | 示例 |
|------|------|------|
| `aria-label` | 为元素添加标签 | `aria-label="搜索"` |
| `aria-required` | 标记必填 | `aria-required="true"` |
| `aria-invalid` | 标记无效 | `:aria-invalid="hasError"` |
| `aria-describedby` | 关联描述 | `aria-describedby="error-msg"` |
| `role="alert"` | 错误通知 | `<span role="alert">` |

### 键盘导航

```vue
<template>
  <div @keydown.enter="handleSubmit" @keydown.escape="handleCancel">
    <input ref="firstInput" />
    <input />
    <button>提交</button>
  </div>
</template>
```

## 四、完整的无障碍表单组件

```vue
<template>
  <form @submit.prevent="handleSubmit" aria-label="用户注册表单" novalidate>
    <!-- 姓名字段 -->
    <div class="form-group" :class="{ 'has-error': errors.name }">
      <label id="name-label" for="name">
        姓名 <span aria-hidden="true">*</span>
        <span class="sr-only">（必填）</span>
      </label>
      <input
        id="name"
        v-model="form.name"
        type="text"
        aria-labelledby="name-label"
        :aria-required="true"
        :aria-invalid="!!errors.name"
        :aria-describedby="errors.name ? 'name-error' : undefined"
        autocomplete="name"
        @blur="validateField('name')"
      />
      <span
        v-if="errors.name"
        id="name-error"
        role="alert"
        aria-live="assertive"
        class="error-message"
      >
        {{ errors.name }}
      </span>
    </div>

    <!-- 提交状态播报 -->
    <div aria-live="polite" aria-atomic="true" class="sr-only">
      {{ submitAnnouncement }}
    </div>

    <button type="submit" :disabled="isSubmitting">
      <span v-if="isSubmitting" aria-hidden="true">...</span>
      {{ isSubmitting ? '提交中' : '注册' }}
    </button>
  </form>
</template>

<script setup>
import { reactive, ref } from 'vue'

const form = reactive({ name: '' })
const errors = reactive({})
const isSubmitting = ref(false)
const submitAnnouncement = ref('')

const validateField = (field) => {
  if (!form[field]) {
    errors[field] = `${field}不能为空`
  } else {
    delete errors[field]
  }
}

const handleSubmit = async () => {
  validateField('name')
  if (Object.keys(errors).length) {
    submitAnnouncement.value = '表单有错误，请修正后提交'
    return
  }
  isSubmitting.value = true
  submitAnnouncement.value = '正在提交表单...'
  try {
    // ... 提交逻辑
    submitAnnouncement.value = '表单提交成功'
  } catch {
    submitAnnouncement.value = '提交失败，请重试'
  } finally {
    isSubmitting.value = false
  }
}
</script>

<style>
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  border: 0;
}
</style>
```

## 五、自定义组件的无障碍

```vue
<!-- CustomCheckbox.vue -->
<template>
  <div
    role="checkbox"
    :aria-checked="modelValue"
    :aria-labelledby="labelId"
    tabindex="0"
    @click="toggle"
    @keydown.space.prevent="toggle"
    @keydown.enter.prevent="toggle"
  >
    <span class="checkbox-icon">{{ modelValue ? '✓' : '' }}</span>
    <span :id="labelId"><slot /></span>
  </div>
</template>

<script setup>
import { computed } from 'vue'

const modelValue = defineModel({ type: Boolean })
const props = defineProps({ id: String })
const labelId = computed(() => `label-${props.id}`)

const toggle = () => { modelValue.value = !modelValue.value }
</script>
```

## 六、焦点管理最佳实践

```vue
<script setup>
import { ref, nextTick } from 'vue'

const firstErrorRef = ref(null)

const handleSubmit = async () => {
  if (!validate()) {
    // 验证失败后聚焦到第一个错误字段
    await nextTick()
    firstErrorRef.value?.focus()
    return
  }
  // 提交逻辑
}
</script>
```

## 三、注意事项与常见陷阱

1. 每个输入必须有关联的`<label>`（for/id匹配）
2. 错误消息使用`aria-describedby`关联到输入框
3. 错误提示使用`role="alert"`触发屏幕阅读器
4. 支持Tab键在表单控件间导航
5. 提交按钮可通过Enter键触发
6. `aria-live`区域用于播报动态状态变化
7. 自定义组件需手动实现键盘交互（Space/Enter触发）
