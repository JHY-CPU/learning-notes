# 表单处理概述

## 一、概念说明

Vue的表单处理基于`v-model`双向绑定，将表单输入与数据自动同步。验证策略分为：手动验证、computed验证、第三方库验证。

```vue
<template>
  <form @submit.prevent="handleSubmit">
    <input v-model="form.name" placeholder="姓名" />
    <input v-model="form.email" type="email" placeholder="邮箱" />
    <button type="submit">提交</button>
  </form>
  <p>输入预览: {{ form.name }} - {{ form.email }}</p>
</template>

<script setup>
import { reactive } from 'vue'

const form = reactive({
  name: '',
  email: ''
})

const handleSubmit = () => {
  console.log('提交:', form)
}
</script>
```

## 二、具体用法

### 验证策略分类

| 策略 | 工具 | 适用场景 |
|------|------|----------|
| 手动验证 | computed + v-if | 简单表单 |
| Schema验证 | VeeValidate + Yup | 复杂表单 |
| UI库自带 | Element Plus表单 | 快速开发 |

### 表单数据绑定方式

```vue
<script setup>
// 文本: v-model="text"
const text = ref('')

// 多选: v-model="checked" (boolean)
const checked = ref(false)

// 多选组: v-model="selected" (array)
const selected = ref([])

// 单选: v-model="picked" (value)
const picked = ref('A')

// 下拉: v-model="option" (value)
const option = ref('')
</script>
```

## 三、注意事项与常见陷阱

1. `v-model`本质是`:value` + `@input`的语法糖
2. 表单元素的值始终是字符串（除非用.number修饰符）
3. 复杂表单使用`reactive`管理表单对象
4. 提交前进行验证，不要信任用户输入
5. 表单状态管理包括：初始值、修改值、验证状态、提交状态
