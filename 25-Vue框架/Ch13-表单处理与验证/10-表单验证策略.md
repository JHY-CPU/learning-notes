# 表单验证策略

## 一、概念说明

表单验证确保用户输入符合预期格式。有三种常见策略：手动验证、computed验证、第三方库验证。

```vue
<template>
  <form @submit.prevent="handleSubmit">
    <div>
      <input v-model="email" type="email" placeholder="邮箱" />
      <span v-if="emailError" class="error">{{ emailError }}</span>
    </div>
    <div>
      <input v-model="password" type="password" placeholder="密码" />
      <span v-if="passwordError" class="error">{{ passwordError }}</span>
    </div>
    <button :disabled="!isValid">提交</button>
  </form>
</template>

<script setup>
import { ref, computed } from 'vue'

const email = ref('')
const password = ref('')

const emailError = computed(() => {
  if (!email.value) return '邮箱不能为空'
  if (!/^\S+@\S+\.\S+$/.test(email.value)) return '邮箱格式不正确'
  return ''
})

const passwordError = computed(() => {
  if (!password.value) return '密码不能为空'
  if (password.value.length < 6) return '密码至少6位'
  return ''
})

const isValid = computed(() => !emailError.value && !passwordError.value)

const handleSubmit = () => {
  if (isValid.value) {
    console.log('提交:', { email: email.value, password: password.value })
  }
}
</script>
```

## 二、具体用法

### 封装验证函数

```js
const rules = {
  required: (val) => !!val || '不能为空',
  email: (val) => /^\S+@\S+\.\S+$/.test(val) || '邮箱格式不正确',
  minLength: (min) => (val) => val.length >= min || `至少${min}个字符`,
  maxLength: (max) => (val) => val.length <= max || `最多${max}个字符`
}

const validate = (value, ruleList) => {
  for (const rule of ruleList) {
    const result = rule(value)
    if (result !== true) return result
  }
  return ''
}
```

## 四、响应式验证状态

```vue
<script setup>
import { reactive, computed, watch } from 'vue'

const form = reactive({ username: '', email: '', password: '', confirmPwd: '' })
const touched = reactive({ username: false, email: false, password: false, confirmPwd: false })
const errors = reactive({})

// 只在字段被触摸后才显示错误
const showError = (field) => touched[field] && errors[field]

// 验证函数
const validators = {
  username: (v) => {
    if (!v) return '用户名不能为空'
    if (v.length < 3) return '至少3个字符'
    if (!/^[a-zA-Z0-9_]+$/.test(v)) return '只能包含字母、数字、下划线'
    return ''
  },
  email: (v) => !v ? '邮箱必填' : !/\S+@\S+\.\S+/.test(v) ? '格式不正确' : '',
  password: (v) => !v ? '密码必填' : v.length < 8 ? '至少8位' : '',
  confirmPwd: (v) => v !== form.password ? '两次密码不一致' : ''
}

// 实时验证
Object.keys(validators).forEach(field => {
  watch(() => form[field], (val) => {
    errors[field] = validators[field](val)
  })
})

const isValid = computed(() =>
  Object.values(errors).every(e => !e) &&
  Object.values(touched).some(t => t)
)
</script>
```

## 五、异步验证

```vue
const username = ref('')
const usernameStatus = ref('')  // '' | 'checking' | 'valid' | 'invalid'
let checkTimer = null

watch(username, (val) => {
  if (!val) { usernameStatus.value = ''; return }

  usernameStatus.value = 'checking'
  clearTimeout(checkTimer)

</script>

<template>
  <input v-model="username" placeholder="用户名" />
  <span v-if="usernameStatus === 'checking'">检查中...</span>

  <span v-else-if="usernameStatus === 'invalid'" class="error">已被占用</span>
</template>
```

## 六、验证策略选择指南

| 策略 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| computed | 小型表单、简单规则 | 响应式、代码简洁 | 频繁计算 |
| watch + touched | 中型表单 | 按需验证 | 需管理touched状态 |
| 提交时验证 | 简单场景 | 实现简单 | 用户体验差 |
| VeeValidate | 大型表单 | 功能完整 | 引入依赖 |
| Yup Schema | 复杂验证逻辑 | 规则可复用、可测试 | 学习成本 |

## 三、注意事项与常见陷阱

1. computed验证实时反馈，但可能过于频繁（考虑lazy验证）
2. 提交前统一验证所有字段
3. 验证规则和错误消息分离，便于国际化
4. 避免只在提交时验证（用户体验差）
5. 复杂表单推荐使用VeeValidate等专业库
6. 异步验证要加防抖，避免频繁请求
7. 验证状态（touched/dirty）与错误消息分开管理
