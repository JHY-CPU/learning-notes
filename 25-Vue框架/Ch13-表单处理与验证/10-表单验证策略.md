# 表单验证策略

## 一、概念说明

表单验证确保用户输入符合预期格式。有三种常见策略：手动验证、computed验证、第三方库验证。

```vue
<template>
  <form @submit.prevent="handleSubmit">
    <div>
      <input v-model="email" type="email" placeholder="邮箱" />
      <span v-if="emailError" class="error">{{ emailError }}</span>
    </div>
    <div>
      <input v-model="password" type="password" placeholder="密码" />
      <span v-if="passwordError" class="error">{{ passwordError }}</span>
    </div>
    <button :disabled="!isValid">提交</button>
  </form>
</template>

<script setup>
import { ref, computed } from 'vue'

const email = ref('')
const password = ref('')

const emailError = computed(() => {
  if (!email.value) return '邮箱不能为空'
  if (!/^\S+@\S+\.\S+$/.test(email.value)) return '邮箱格式不正确'
  return ''
})

const passwordError = computed(() => {
  if (!password.value) return '密码不能为空'
  if (password.value.length < 6) return '密码至少6位'
  return ''
})

const isValid = computed(() => !emailError.value && !passwordError.value)

const handleSubmit = () => {
  if (isValid.value) {
    console.log('提交:', { email: email.value, password: password.value })
  }
}
</script>
```

## 二、具体用法

### 封装验证函数

```js
const rules = {
  required: (val) => !!val || '不能为空',
  email: (val) => /^\S+@\S+\.\S+$/.test(val) || '邮箱格式不正确',
  minLength: (min) => (val) => val.length >= min || `至少${min}个字符`,
  maxLength: (max) => (val) => val.length <= max || `最多${max}个字符`
}

const validate = (value, ruleList) => {
  for (const rule of ruleList) {
    const result = rule(value)
    if (result !== true) return result
  }
  return ''
}
```

## 三、注意事项与常见陷阱

1. computed验证实时反馈，但可能过于频繁（考虑lazy验证）
2. 提交前统一验证所有字段
3. 验证规则和错误消息分离，便于国际化
4. 避免只在提交时验证（用户体验差）
5. 复杂表单推荐使用VeeValidate等专业库
