# 表单错误展示

## 一、概念说明

良好的错误展示提升用户体验：实时反馈、明确位置、清晰消息。Vue中可通过computed、VeeValidate或手动管理实现。

```vue
<template>
  <form @submit.prevent="handleSubmit">
    <div class="field" :class="{ error: errors.email }">
      <input v-model="form.email" @blur="validateEmail" placeholder="邮箱" />
      <span v-if="errors.email" class="error-msg">{{ errors.email }}</span>
    </div>

    <div class="field" :class="{ error: errors.password }">
      <input v-model="form.password" type="password" @blur="validatePassword" />
      <span v-if="errors.password" class="error-msg">{{ errors.password }}</span>
    </div>

    <button :disabled="hasErrors">提交</button>
  </form>
</template>

<script setup>
import { reactive, computed } from 'vue'

const form = reactive({ email: '', password: '' })
const errors = reactive({ email: '', password: '' })

const validateEmail = () => {
  errors.email = !form.email ? '邮箱必填' :
    !/^\S+@\S+\.\S+$/.test(form.email) ? '邮箱格式不正确' : ''
}
const validatePassword = () => {
  errors.password = !form.password ? '密码必填' :
    form.password.length < 6 ? '密码至少6位' : ''
}

const hasErrors = computed(() => errors.email || errors.password)
</script>
```

## 二、具体用法

### 使用VeeValidate

```vue
<template>
  <Field v-slot="{ field, errors }" name="email" rules="required|email">
    <input v-bind="field" :class="{ 'is-invalid': errors[0] }" />
    <div v-if="errors[0]" class="invalid-feedback">{{ errors[0] }}</div>
  </Field>
</template>
```

### 汇总错误展示

```vue
<template>
  <div v-if="errorList.length" class="error-summary">
    <p>请修正以下错误：</p>
    <ul>
      <li v-for="(err, i) in errorList" :key="i">{{ err }}</li>
    </ul>
  </div>
</template>
```

## 五、工具提示式错误

```vue
<template>
  <div class="field-wrapper">
    <input
      v-model="value"
      :class="{ invalid: !!error }"
      @blur="validate"
    />
    <Transition name="tooltip">
      <div v-if="error" class="tooltip-error">
        {{ error }}
        <div class="arrow"></div>
      </div>
    </Transition>
  </div>
</template>

<style scoped>
```

## 六、可访问性错误展示

```vue
<template>
  <div>
    <label for="email">邮箱</label>
    <input
      id="email"
      v-model="email"
      :aria-invalid="!!emailError"
      :aria-describedby="emailError ? 'email-error' : undefined"
      @blur="validateEmail"
    />
    <span
      v-if="emailError"
      id="email-error"
      role="alert"
      aria-live="polite"
      class="error-msg"
    >
      {{ emailError }}
    </span>
  </div>
</template>
```

- `aria-invalid` 标记输入框为无效状态
- `aria-describedby` 将错误消息关联到输入框
- `role="alert"` 让屏幕阅读器立即朗读错误
- `aria-live="polite"` 在错误变化时通知辅助设备

## 三、注意事项与常见陷阱

1. 错误消息应清晰、具体，告诉用户如何修正
2. 实时验证（blur时）比提交时验证体验更好
3. 多个错误时可汇总展示在表单顶部
4. 错误状态应有视觉反馈（颜色、图标）
5. 避免过于频繁的验证（考虑debounce）
6. 错误消息消失时使用过渡动画，避免页面跳动
7. 使用ARIA属性确保错误消息对屏幕阅读器可见
