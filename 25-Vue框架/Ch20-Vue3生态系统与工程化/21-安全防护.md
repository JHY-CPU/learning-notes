# 安全防护

## 一、概念说明

Vue 3 前端安全防护涵盖 **XSS（跨站脚本攻击）**、**CSRF（跨站请求伪造）**、**CSP（内容安全策略）** 三大方向。Vue 模板默认转义 HTML 防止 XSS，但仍需注意 `v-html`、用户输入、第三方内容等场景。

## 二、具体用法

### XSS 防护

```vue
<script setup lang="ts">
// Vue 模板自动转义，以下内容安全
const userInput = '<script>alert("xss")<\/script>'
// 输出：<script>alert("xss")</script>（纯文本显示）

// 危险：v-html 直接渲染 HTML
const dangerousHtml = '<img src=x onerror="alert(1)">'

// 安全：使用 DOMPurify 过滤
import DOMPurify from 'dompurify'
const safeHtml = DOMPurify.sanitize(dangerousHtml)
// safeHtml 输出：'<img src="x">'（onerror 被移除）
</script>

<template>
  <!-- 安全：自动转义 -->
  <p>{{ userInput }}</p>

  <!-- 危险：避免使用 v-html -->
  <!-- <div v-html="dangerousHtml"></div> -->

  <!-- 安全：经过滤后使用 -->
  <div v-html="safeHtml"></div>
</template>
```

### CSRF 防护

```ts
// utils/request.ts
import axios from 'axios'

const service = axios.create({
  baseURL: import.meta.env.VITE_API_URL,
  withCredentials: true  // 携带 cookie
})

// 方案1：Token 方案
service.interceptors.request.use(config => {
  const token = localStorage.getItem('token')
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})

// 方案2：CSRF Token（从 cookie 读取）
function getCsrfToken(): string {
  const match = document.cookie.match(/XSRF-TOKEN=([^;]+)/)
  return match ? decodeURIComponent(match[1]) : ''
}

service.interceptors.request.use(config => {
  config.headers['X-XSRF-TOKEN'] = getCsrfToken()
  return config
})
```

### CSP 配置

```nginx
# nginx.conf - Content Security Policy
server {
  add_header Content-Security-Policy "
    default-src 'self';
    script-src 'self' 'nonce-{随机值}';
    style-src 'self' 'unsafe-inline';
    img-src 'self' data: https:;
    connect-src 'self' https://api.example.com;
    font-src 'self';
    object-src 'none';
    base-uri 'self';
  ";
}
```

```ts
// Vite CSP nonce 支持
// vite.config.ts
export default defineConfig({

  }

```

### 输入验证与过滤

```ts
// utils/sanitize.ts
import DOMPurify from 'dompurify'

// 通用过滤函数
export function sanitizeHtml(dirty: string): string {
  return DOMPurify.sanitize(dirty, {
    ALLOWED_TAGS: ['p', 'br', 'strong', 'em', 'a'],
    ALLOWED_ATTR: ['href', 'target']
  })
}

// URL 验证
export function isValidUrl(url: string): boolean {
  try {
    const parsed = new URL(url)
    return ['http:', 'https:'].includes(parsed.protocol)
    // 阻止 javascript: 协议
  } catch {
    return false
  }
}

// 用户输入转义
export function escapeHtml(text: string): string {
  const map: Record<string, string> = {
    '&': '&amp;',
    '<': '&lt;',
    '>': '&gt;',
    '"': '&quot;',
    "'": '&#039;'
  }
  return text.replace(/[&<>"']/g, (c) => map[c])
}
```

## 三、注意事项与常见陷阱

1. **DOMPurify 不是万能的**：它过滤 HTML 但不防止所有攻击向量
2. **CSP 可能破坏功能**：配置太严格会导致第三方脚本无法加载
3. **Token 存 localStorage 有 XSS 风险**：更安全的做法是存 HttpOnly Cookie
4. **`npm audit` 要定期执行**：第三方依赖可能引入安全漏洞
5. **安全是后端的责任**：前端防护只是增加攻击难度，真正的安全靠后端
