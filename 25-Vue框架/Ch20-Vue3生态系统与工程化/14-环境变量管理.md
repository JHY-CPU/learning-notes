# 环境变量管理

## 一、概念说明

Vite 使用 `.env` 文件管理环境变量，只有以 `VITE_` 为前缀的变量才会暴露给客户端代码。不同环境使用不同文件（`.env.development`、`.env.production`），`import.meta.env` 对象在代码中访问。

## 二、具体用法

### 环境文件层级

```text
项目根目录下：
.env                  # 所有环境加载（基础配置）
.env.local            # 所有环境加载，git 忽略
.env.development      # 仅开发环境
.env.development.local # 仅开发环境，git 忽略
.env.production       # 仅生产环境
.env.staging          # 自定义环境（staging）

优先级：.env.[mode].local > .env.[mode] > .env.local > .env
```

### 变量定义

```bash
# .env
VITE_APP_TITLE=我的Vue应用
VITE_APP_VERSION=1.0.0

# .env.development
VITE_API_URL=http://localhost:3000/api
VITE_DEBUG=true

# .env.production
VITE_API_URL=https://api.example.com
VITE_DEBUG=false
VITE_CDN_URL=https://cdn.example.com
```

### 代码中使用

```ts
// 获取环境变量
const apiUrl = import.meta.env.VITE_API_URL
const isDebug = import.meta.env.VITE_DEBUG === 'true'
const appTitle = import.meta.env.VITE_APP_TITLE

// 生产环境：apiUrl = 'https://api.example.com'
// 开发环境：apiUrl = 'http://localhost:3000/api'

// Vite 内置变量
const isProd = import.meta.env.PROD     // boolean
const isDev = import.meta.env.DEV       // boolean
const mode = import.meta.env.MODE       // 'development' | 'production'
```

### TypeScript 类型声明

```ts
// src/env.d.ts
/// <reference types="vite/client" />

interface ImportMetaEnv {
  readonly VITE_APP_TITLE: string
  readonly VITE_API_URL: string
  readonly VITE_DEBUG: string
  readonly VITE_CDN_URL: string
}

interface ImportMeta {
  readonly env: ImportMetaEnv
}
// 这样 import.meta.env.VITE_API_URL 有类型提示
```

### Axios 中使用

```ts
// utils/request.ts
import axios from 'axios'

const service = axios.create({
  baseURL: import.meta.env.VITE_API_URL,
  timeout: 10000
})

// 请求拦截器
service.interceptors.request.use(config => {
  if (import.meta.env.VITE_DEBUG === 'true') {
    console.log('请求:', config.url)
  }
  return config
})
```

## 三、注意事项与常见陷阱

1. **不加 VITE_ 前缀的变量不会暴露给客户端**：这是安全机制
2. **.env 文件不要提交敏感信息**：密码、密钥等放 .env.local 或 CI 环境变量
3. **环境变量在构建时内联**：运行时无法修改，需要重新构建
4. **布尔值是字符串**：`VITE_DEBUG=false` 的值是 `"false"` 字符串，不是布尔值
5. **自定义模式通过 --mode 指定**：`vite build --mode staging` 加载 .env.staging
