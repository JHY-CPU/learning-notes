# 代码生成工具

## 一、概念说明

代码生成工具帮助快速创建标准化的组件、页面、Store 等模板文件。常用工具有 Plop（通用代码生成器）、Hygen（模板管理）和自定义 CLI 脚本。这些工具统一代码风格，减少重复劳动。

## 二、具体用法

### Plop 代码生成器

```bash
npm install -D plop
```

```js
// plopfile.js
export default function (plop) {
  plop.setGenerator('component', {
    description: '创建 Vue 3 组件',
    prompts: [
      { type: 'input', name: 'name', message: '组件名称（PascalCase）：' },
      { type: 'list', name: 'type', message: '组件类型：',
        choices: ['基础组件', '业务组件', '布局组件']
      }
    ],
    actions: [
      {
        type: 'add',
        path: 'src/components/{{type}}/{{name}}.vue',
        templateFile: 'plop-templates/Component.vue.hbs'
      },
      {
        type: 'add',
        path: 'src/components/{{type}}/{{name}}.test.ts',
        templateFile: 'plop-templates/Component.test.ts.hbs'
      }
    ]
  })
}
```

```handlebars
{{!-- plop-templates/Component.vue.hbs --}}
<script setup lang="ts">
interface Props {
  title: string
}

const props = defineProps<Props>()
</script>

<template>
  <div class="{{dashCase name}}">
    <h2>{{ title }}</h2>
    <slot />
  </div>
</template>

<style scoped>
.{{dashCase name}} {
  padding: 16px;
}
</style>
```

```bash
# 使用
npx plop
# ? 组件名称（PascalCase）: UserCard
# ? 组件类型: 业务组件
# → 生成 src/components/业务组件/UserCard.vue
# → 生成 src/components/业务组件/UserCard.test.ts
```

### Hygen 模板

```bash
npm install -g hygen
npx hygen generator new component
```

```handlebars
---
to: src/components/<%= name %>.vue
---
<script setup lang="ts">
// <%= name %> 组件
</script>

<template>
  <div class="<%= h.inflection.dasherize(name) %>">
    <%= name %>
  </div>
</template>
```

### 自定义 CLI 脚本

```ts
// scripts/create-component.ts
import { writeFileSync, mkdirSync, existsSync } from 'fs'
import { resolve } from 'path'

const name = process.argv[2]
if (!name) {
  console.error('请提供组件名称')
  process.exit(1)
}

const dir = resolve('src/components', name)
if (!existsSync(dir)) mkdirSync(dir, { recursive: true })

const template = `<script setup lang="ts">
interface Props {
  // 定义 props
}
defineProps<Props>()
</script>

<template>
  <div class="${name.toLowerCase()}">
    ${name}
  </div>
</template>
`

writeFileSync(resolve(dir, `${name}.vue`), template)
console.log(`已创建组件: src/components/${name}/${name}.vue`)
```

```json
// package.json
{
  "scripts": {
    "create:component": "tsx scripts/create-component.ts"
  }
}
```

## 三、注意事项与常见陷阱

1. **模板要覆盖团队的编码规范**：生成的代码应该可以直接使用
2. **Plop 需要 ESM 配置**：`plopfile.js` 中使用 `export default` 而非 `module.exports`
3. **模板变量要一致**：name、Name、NAME 等格式在 Handlebars 中用 helper 转换
4. **不要过度工程化**：简单的组件直接创建比使用生成器更快
5. **模板应纳入版本控制**：模板变更与代码变更一起 review
