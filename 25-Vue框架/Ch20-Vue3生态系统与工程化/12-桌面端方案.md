# 桌面端方案

## 一、概念说明

将 Vue 3 应用打包为桌面应用的两个主要方案：**Electron**（基于 Chromium，生态成熟）和 **Tauri**（基于系统 WebView，体积小性能好）。Electron 适合需要完整浏览器环境的场景，Tauri 适合追求轻量和原生体验的场景。

## 二、具体用法

### Electron 集成

```bash
npm install electron electron-builder --save-dev
npm install vite-plugin-electron --save-dev
```

```ts
// electron/main.ts
import { app, BrowserWindow, ipcMain } from 'electron'
import path from 'path'

let win: BrowserWindow | null

function createWindow() {
  win = new BrowserWindow({
    width: 1200,
    height: 800,
    webPreferences: {
      preload: path.join(__dirname, 'preload.js'),
      nodeIntegration: false,
      contextIsolation: true
    }
  })

  // 开发环境加载 Vite 服务器
  if (process.env.VITE_DEV_SERVER) {
    win.loadURL('http://localhost:5173')
  } else {
    win.loadFile(path.join(__dirname, '../dist/index.html'))
  }
}

app.whenReady().then(createWindow)

// IPC 通信
ipcMain.handle('get-app-version', () => {
  return app.getVersion()
})
```

```ts
// electron/preload.ts
import { contextBridge, ipcRenderer } from 'electron'

contextBridge.exposeInMainWorld('electronAPI', {
  getVersion: () => ipcRenderer.invoke('get-app-version'),
  // 渲染进程安全调用主进程功能
})
```

```vue
<!-- Vue 组件中调用 -->
<script setup lang="ts">
declare global {
  interface Window {
    electronAPI: {
      getVersion: () => Promise<string>
    }
  }
}

const version = ref('')

onMounted(async () => {
  version.value = await window.electronAPI.getVersion()
  // 输出：version.value = '1.0.0'
})
</script>
```

### Tauri 集成

```bash
# 安装 Tauri CLI
npm install --save-dev @tauri-apps/cli
# 初始化 Tauri
npx tauri init
```

```toml
# src-tauri/tauri.conf.json
{
  "build": {
    "beforeDevCommand": "npm run dev",
    "beforeBuildCommand": "npm run build",
    "devPath": "http://localhost:5173",
    "distDir": "../dist"
  },
  "app": {
    "title": "My Tauri App",
    "windows": [{ "title": "My App", "width": 1200, "height": 800 }]
  }
}
```

```rust
// src-tauri/src/main.rs - Rust 后端
#[tauri::command]
fn greet(name: &str) -> String {
    format!("Hello, {}! From Rust.", name)
}

fn main() {
    tauri::Builder::default()
        .invoke_handler(tauri::generate_handler![greet])
        .run(tauri::generate_context!())
        .expect("error running tauri app");
}
```

```vue
<script setup lang="ts">
import { invoke } from '@tauri-apps/api/tauri'

async function greet() {
  const result = await invoke('greet', { name: 'Vue' })
  console.log(result)
  // 输出：Hello, Vue! From Rust.
}
</script>
```

### 方案对比

```text
Electron vs Tauri：

Electron
├── 安装包大小：~150MB（含 Chromium）
├── 内存占用：较高（每个窗口一个 Chromium）
├── 生态：成熟，npm 包直接可用
├── 学习曲线：低（纯 Web 技术）
└── 适用：复杂应用，需要完整浏览器 API

Tauri
├── 安装包大小：~3-10MB（使用系统 WebView）
├── 内存占用：低
├── 生态：发展中，原生功能需 Rust
├── 学习曲线：中（需了解 Rust 基础）
└── 适用：轻量应用，追求性能
```

## 三、注意事项与常见陷阱

1. **Electron 包体积大**：每个应用都包含完整 Chromium
2. **Tauri 的系统 WebView 可能有兼容性问题**：不同 OS 的 WebView 能力不同
3. **前后端通信需通过 IPC**：不能直接在渲染进程调用 Node.js
4. **生产环境需要打包配置**：electron-builder 或 tauri build
5. **自动更新需要额外实现**：electron-updater 或 Tauri updater 插件
