# 监控与日志

## 一、概念说明

前端监控包括**错误监控**、**性能监控**和**用户行为追踪**。Sentry 是最流行的错误监控平台，支持 Vue 3 错误捕获和 Source Map。自建监控可使用 Web Vitals API 和 PerformanceObserver。

## 二、具体用法

### Sentry 集成

```bash
npm install @sentry/vue
```

```ts
// main.ts
import * as Sentry from '@sentry/vue'

Sentry.init({
  app,
  dsn: 'https://your-dsn@sentry.io/project-id',
  integrations: [
    Sentry.browserTracingIntegration({ router }),
    Sentry.replayIntegration()
  ],
  tracesSampleRate: 0.1,     // 性能采样率 10%
  replaysSessionSampleRate: 0.1,
  replaysOnErrorSampleRate: 1.0,  // 错误时 100% 录制
  environment: import.meta.env.MODE
})
```

### 全局错误捕获

```ts
// utils/error-handler.ts
export function setupErrorHandler(app: App) {
  // Vue 组件错误
  app.config.errorHandler = (error, instance, info) => {
    console.error('Vue 错误:', error)
    console.error('组件:', instance?.$options.name)
    console.error('信息:', info)

    // 上报到监控平台
    reportError({
      type: 'vue-error',
      message: (error as Error).message,
      stack: (error as Error).stack,
      component: instance?.$options.name,
      info
    })
  }

  // 全局未捕获 Promise 错误
  window.addEventListener('unhandledrejection', (event) => {
    console.error('未处理的 Promise 错误:', event.reason)
    reportError({
      type: 'unhandled-rejection',
      message: event.reason?.message || String(event.reason)
    })
  })

  // 全局 JS 错误
  window.addEventListener('error', (event) => {
    console.error('全局错误:', event.error)
    reportError({
      type: 'global-error',
      message: event.message,
      filename: event.filename,
      lineno: event.lineno
    })
  })
}
```

### 性能监控

```ts
// utils/performance.ts
export function reportWebVitals() {
  // 核心 Web 指标
  new PerformanceObserver((list) => {
    for (const entry of list.getEntries()) {
      const metric = {
        name: entry.name,           // 'LCP', 'FID', 'CLS'
        value: entry.startTime,
        rating: getRating(entry)
      }
      console.log('性能指标:', metric)
      // 上报：{ name: 'LCP', value: 1200, rating: 'good' }
    }
  }).observe({ type: 'largest-contentful-paint', buffered: true })

  // 首屏加载时间
  window.addEventListener('load', () => {
    const timing = performance.getEntriesByType('navigation')[0] as PerformanceNavigationTiming
    console.log('页面加载时间:', timing.loadEventEnd - timing.startTime)
    console.log('DOM 解析:', timing.domContentLoadedEventEnd - timing.startTime)
    console.log('白屏时间:', timing.responseStart - timing.startTime)
  })
}

function getRating(entry: PerformanceEntry): string {
  // LCP: good < 2500ms, poor < 4000ms
  if (entry.name === 'LCP') {
    return entry.startTime < 2500 ? 'good' : entry.startTime < 4000 ? 'needs-improvement' : 'poor'
  }
  return 'unknown'
}
```

### 用户行为追踪

```ts
// utils/tracker.ts
interface TrackEvent {
  action: string
  category: string
  label?: string
  value?: number
}

export function track(event: TrackEvent) {
  const data = {
    ...event,
    timestamp: Date.now(),
    page: window.location.pathname,
    userAgent: navigator.userAgent
  }
  // 发送到分析服务
  navigator.sendBeacon('/api/track', JSON.stringify(data))
  console.log('追踪事件:', data)
}
```

## 三、注意事项与常见陷阱

1. **采样率要合理**：100% 采样在高流量下成本很高
2. **Source Map 上传到 Sentry**：否则错误堆栈是压缩后的代码
3. **不要在错误处理中抛出新错误**：会导致无限循环
4. **性能监控在生产环境启用**：开发环境数据不准确
5. **用户隐私合规**：追踪需告知用户，提供 opt-out 选项
