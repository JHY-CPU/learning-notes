# 构建部署

## 一、概念说明

Vue 3 项目的部署流程包括：构建（`vite build`）、静态资源上传、CI/CD 自动化。常见部署目标有 Docker 容器、Vercel/Netlify 无服务器平台、Cloudflare Pages 等。Vite 构建产物是纯静态文件，部署灵活。

## 二、具体用法

### Vite 构建

```bash
npm run build
# 生成 dist/ 目录
# ├── assets/
# │   ├── index-a1b2c3.js
# │   └── index-d4e5f6.css
# ├── index.html
# └── favicon.ico
```

```ts
// vite.config.ts - 构建优化配置
export default defineConfig({
  build: {
    outDir: 'dist',
    assetsDir: 'assets',
    sourcemap: false,       // 生产环境不生成 sourcemap
    minify: 'esbuild',      // 使用 esbuild 压缩
    rollupOptions: {
      output: {
        manualChunks: {
          // 分包策略
          vendor: ['vue', 'vue-router', 'pinia'],
          elementPlus: ['element-plus']
        }
      }
    }
  }
})
```

### Docker 部署

```dockerfile
# Dockerfile
FROM node:18-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

```nginx
# nginx.conf
server {
    listen 80;
    root /usr/share/nginx/html;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
        # SPA 路由回退
    }

    location /assets/ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

### GitHub Actions CI/CD

```yaml
# .github/workflows/deploy.yml
name: Deploy
on:
  push:
    branches: [main]
jobs:
  build-deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 18
          cache: npm
      - run: npm ci
      - run: npm run build
      - run: npm run test  # 可选：运行测试

      # 部署到 GitHub Pages
      - uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./dist
```

### Vercel 部署

```json
// vercel.json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```

```bash
# 使用 Vercel CLI
npm i -g vercel
vercel --prod
```

### 环境变量

```bash
# .env.production
VITE_API_URL=https://api.example.com
VITE_APP_TITLE=我的生产应用

# .env.development
VITE_API_URL=http://localhost:3000
VITE_APP_TITLE=我的开发应用
```

```ts
// 代码中使用
const apiUrl = import.meta.env.VITE_API_URL
console.log('API 地址:', apiUrl)
// 生产环境：https://api.example.com
// 开发环境：http://localhost:3000
```

## 三、注意事项与常见陷阱

1. **SPA 路由需要服务器配置**：Nginx 需 `try_files` 回退到 index.html
2. **环境变量以 VITE_ 开头**：不加前缀的变量不会暴露给客户端
3. **构建产物不含 .env 文件**：变量在构建时被内联到代码中
4. **Docker 镜像优化**：多阶段构建减小最终镜像体积
5. **CDN 缓存需考虑更新**：文件名含 hash，但 index.html 需要短缓存
