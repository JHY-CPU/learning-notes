# 微前端方案

## 一、概念说明

微前端将大型前端应用拆分为多个独立开发、独立部署的子应用。Vue 3 可作为微前端的主应用或子应用。主流方案包括 qiankun（基于 single-spa）和 Webpack 5 Module Federation。qiankun 侵入性较低，Module Federation 更底层灵活。

## 二、具体用法

### qiankun 主应用

```bash
npm install qiankun
```

```ts
// main.ts - 主应用
import { registerMicroApps, start } from 'qiankun'

registerMicroApps([
  {
    name: 'vue-sub',
    entry: '//localhost:7101',
    container: '#sub-app',
    activeRule: '/vue-sub',
    // 访问 /vue-sub 时加载子应用
  },
  {
    name: 'react-sub',
    entry: '//localhost:7102',
    container: '#sub-app',
    activeRule: '/react-sub'
  }
])

start({
  prefetch: 'all',
  sandbox: { experimentalStyleIsolation: true }
})
```

```vue
<!-- 主应用布局 -->
<template>
  <div class="main-app">
    <nav>
      <router-link to="/">首页</router-link>
      <router-link to="/vue-sub">Vue子应用</router-link>
      <router-link to="/react-sub">React子应用</router-link>
    </nav>
    <div id="sub-app"></div>
  </div>
</template>
```

### qiankun 子应用（Vue 3）

```ts
// 子应用 main.ts
import { createApp } from 'vue'
import { createRouter, createWebHistory } from 'vue-router'
import App from './App.vue'

let app: ReturnType<typeof createApp> | null = null

// qiankun 生命周期钩子
function render(props: any = {}) {
  const { container } = props
  app = createApp(App)
  app.use(createRouter({
    history: createWebHistory('/vue-sub'),
    routes: [...]
  }))
  app.mount(container ? container.querySelector('#app') : '#app')
}

// 独立运行时
if (!(window as any).__POWERED_BY_QIANKUN__) {
  render()
}

// qiankun 导出生命周期
export async function bootstrap() { console.log('子应用启动') }
export async function mount(props: any) { render(props) }
export async function unmount() { app?.unmount(); app = null }
```

### Module Federation

```ts
// vite.config.ts - 主应用
import { federation } from '@module-federation/vite'

export default defineConfig({
  plugins: [
    federation({
      name: 'host',
      remotes: {
        remoteApp: 'remoteApp@http://localhost:5001/remoteEntry.js'
      },
      shared: ['vue', 'vue-router']
    })
  ]
})
```

```ts
// 动态加载远程组件
const RemoteButton = defineAsyncComponent(
  () => import('remoteApp/Button')
)
```

## 三、注意事项与常见陷阱

1. **qiankun 的 CSS 隔离不完美**：`experimentalStyleIsolation` 可能仍有样式冲突
2. **子应用需支持独立运行**：用 `__POWERED_BY_QIANKUN__` 判断运行环境
3. **共享依赖需手动配置**：Vue、Vue Router 等需设置为 externals 或 shared
4. **通信通过 props 或全局状态**：qiankun 提供 `initGlobalState` API
5. **Module Federation 需要 Webpack 5 或 Vite 插件**：不同构建工具配置不同
