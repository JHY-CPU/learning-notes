# 权限管理方案

## 一、概念说明

Vue 3 的权限管理涵盖**路由权限**、**按钮权限**和**接口权限**三个层面。常用方案包括：路由守卫控制页面访问、自定义指令控制按钮显示、Store 存储用户角色和权限列表。

## 二、具体用法

### 路由权限守卫

```ts
// router/guards.ts
import type { Router } from 'vue-router'

const whiteList = ['/login', '/404']

export function setupPermissionGuard(router: Router) {
  router.beforeEach((to, from, next) => {
    const token = localStorage.getItem('token')
    const userStore = useUserStore()

    if (token) {
      if (to.path === '/login') {
        next('/')
      } else if (userStore.roles.length > 0) {
        next()
      } else {
        // 动态生成路由
        userStore.generateRoutes().then(() => {
          next({ ...to, replace: true })
        })
      }
    } else {
      if (whiteList.includes(to.path)) {
        next()
      } else {
        next(`/login?redirect=${to.path}`)
        // 未登录访问 /admin → 重定向到 /login?redirect=/admin
      }
    }
  })
}
```

### 按钮级权限指令

```ts
// directives/permission.ts
import type { Directive } from 'vue'

export const permission: Directive = {
  mounted(el: HTMLElement, binding) {
    const { value } = binding  // 期望权限字符串，如 'user:delete'
    const userStore = useUserStore()
    const permissions = userStore.permissions  // ['user:list', 'user:add']

    if (value && !permissions.includes(value)) {
      el.parentNode?.removeChild(el)
      // 无权限时直接移除 DOM 元素
    }
  }
}
```

```ts
// main.ts
import { permission } from './directives/permission'
app.directive('permission', permission)
```

```vue
<template>
  <!-- 有 user:add 权限时才显示 -->
  <el-button v-permission="'user:add'">新增用户</el-button>

  <!-- 有 user:delete 权限时才显示 -->
  <el-button v-permission="'user:delete'" type="danger">删除用户</el-button>

  <!-- 无权限时按钮不渲染 -->
</template>
```

### 权限 Store

```ts
// stores/permission.ts
export const usePermissionStore = defineStore('permission', () => {
  const roles = ref<string[]>([])
  const permissions = ref<string[]>([])

  function setRoles(newRoles: string[]) {
    roles.value = newRoles
  }

  function hasPermission(perm: string): boolean {
    return permissions.value.includes(perm) || permissions.value.includes('*')
  }

  function hasRole(role: string): boolean {
    return roles.value.includes(role)
  }

  return { roles, permissions, setRoles, hasPermission, hasRole }
})
```

```vue
<script setup lang="ts">
const permissionStore = usePermissionStore()

// 条件渲染
const canEdit = computed(() => permissionStore.hasPermission('article:edit'))
</script>

<template>
  <button v-if="canEdit">编辑文章</button>
</template>
```

## 三、注意事项与常见陷阱

1. **权限数据必须来自服务端**：前端权限只是 UX 优化，真正的安全靠后端接口验证
2. **v-permission 是 DOM 级操作**：也可以用 v-show 或 v-if 配合 computed 实现
3. **白名单要包含 404 等公共页面**：避免未登录时跳转死循环
4. **动态路由需要刷新兼容**：页面刷新时要重新生成路由
5. **权限变更后需要刷新页面**：或清除缓存重新拉取权限数据
