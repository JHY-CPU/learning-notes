# 第三方库类型

## 一、概念说明

第三方 JavaScript 库可能自带类型定义（.d.ts 文件），也可能需要安装 `@types/` 包。没有类型定义的库可以通过声明文件手动提供类型。正确配置类型定义是 Vue + TypeScript 项目的基础工作。

## 二、具体用法

### 安装类型包

```bash
# 大部分流行库的类型在 @types 组织下
npm install -D @types/lodash
npm install -D @types/node
npm install -D @types/jest

# 检查库是否自带类型定义
# 查看库的 package.json 中是否有 "types" 字段
```

### 使用自带类型的库

```ts
// Axios 自带类型定义
import axios, { type AxiosResponse, type AxiosError } from 'axios'

// 完整类型推断
async function fetchUser(id: number): Promise<AxiosResponse<{ name: string }>> {
  const response = await axios.get(`/api/users/${id}`)
  // response.data 类型: { name: string }
  return response
}

// 错误处理带类型
fetchUser(1).catch((error: AxiosError) => {
  console.error('请求失败:', error.response?.status)
})
```

### 手动声明类型

```ts
// src/types/legacy-lib.d.ts
// 为没有类型的 JS 库声明类型
declare module 'legacy-lib' {
  export function init(config: { apiKey: string; debug?: boolean }): void
  export function track(event: string, data?: Record<string, unknown>): void
  export const version: string
}

// 使用时获得类型
import { init, track } from 'legacy-lib'
init({ apiKey: 'xxx', debug: true })
track('pageview', { path: '/' })
```

### 全局类型声明

```ts
// src/types/global.d.ts
// 扩展全局 Window 接口
declare global {
  interface Window {
    dataLayer: Array<Record<string, unknown>>
    myCustomLib: {
      init: () => void
      destroy: () => void
    }
  }
}

// 扩展 ImportMeta（Vite）
interface ImportMetaEnv {
  readonly VITE_API_URL: string
  readonly VITE_APP_TITLE: string
}

interface ImportMeta {
  readonly env: ImportMetaEnv
}

export {}
```

### 带泛型的库类型

```ts
// vue-toastification 自带类型
import { useToast, type ToastOptions } from 'vue-toastification'

const toast = useToast()

const options: ToastOptions = {
  position: 'top-right',
  timeout: 3000,
  closeOnClick: true
}

toast.success('操作成功！', options)
// TypeScript 检查 position 值是否合法
```

## 三、注意事项与常见陷阱

1. **优先选自带类型的库**：选择库时检查是否有 TypeScript 支持
2. **@types 版本需匹配库版本**：`@types/lodash@4` 应匹配 `lodash@4`
3. **声明文件不要导出**：.d.ts 中使用 `export {}` 避免污染全局
4. **skipLibCheck 可跳过库类型错误**：在 tsconfig.json 中设置为 true
5. **node_modules 中的 .d.ts 优先级最高**：手动声明不会覆盖已安装的类型
