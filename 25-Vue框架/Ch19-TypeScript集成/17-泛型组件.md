# 泛型组件

## 一、概念说明

Vue 3.3+ 支持泛型组件，允许组件接收泛型类型参数来处理不同数据类型。泛型组件在列表、表单、选择器等通用组件中非常有用，保持类型安全的同时提高复用性。

## 二、具体用法

### 基本泛型组件

```vue
<!-- GenericList.vue -->
<script setup lang="ts" generic="T">
// generic 属性声明泛型参数
interface Props {
  items: T[]
  keyExtractor: (item: T) => string | number
  renderItem: (item: T) => string
}

const props = defineProps<Props>()

const emit = defineEmits<{
  select: [item: T]
}>()
</script>

<template>
  <ul>
    <li
      v-for="item in items"
      :key="keyExtractor(item)"
      @click="emit('select', item)"
    >
      {{ renderItem(item) }}
    </li>
  </ul>
</template>
```

```vue
<!-- 使用泛型组件 -->
<script setup lang="ts">
interface User {
  id: number
  name: string
  email: string
}

const users: User[] = [
  { id: 1, name: '张三', email: 'zhang@example.com' },
  { id: 2, name: '李四', email: 'li@example.com' }
]

function handleSelect(user: User) {
  // TypeScript 知道 user 是 User 类型
  console.log('选中用户:', user.name, user.email)
  // 输出：选中用户: 张三 zhang@example.com
}
</script>

<template>
  <!-- items 传入 User[] 后，T 被推断为 User -->
  <GenericList
    :items="users"
    :key-extractor="(u) => u.id"
    :render-item="(u) => `${u.name} (${u.email})`"
    @select="handleSelect"
  />
</template>
```

### 带约束的泛型

```vue
<!-- GenericTable.vue -->
<script setup lang="ts" generic="T extends { id: number }">
// T 必须有 id 属性
interface Props {
  data: T[]
  columns: { key: keyof T; label: string }[]
}

const props = defineProps<Props>()
</script>

<template>
  <table>
    <thead>
      <tr>
        <th v-for="col in columns" :key="String(col.key)">{{ col.label }}</th>
      </tr>
    </thead>
    <tbody>
      <tr v-for="row in data" :key="row.id">
        <td v-for="col in columns" :key="String(col.key)">
          {{ row[col.key] }}
        </td>
      </tr>
    </tbody>
  </table>
</template>
```

```vue
<!-- 使用带约束的泛型组件 -->
<script setup lang="ts">
interface Order {
  id: number
  product: string
  amount: number
  status: string
}

const orders: Order[] = [
  { id: 1, product: 'Vue课程', amount: 99, status: '已支付' },
  { id: 2, product: 'TS手册', amount: 59, status: '待支付' }
]
</script>

<template>
  <GenericTable
    :data="orders"
    :columns="[
      { key: 'product', label: '商品' },
      { key: 'amount', label: '金额' },
      { key: 'status', label: '状态' }
    ]"
  />
</template>
```

### 泛型 Select 组件

```vue
<!-- GenericSelect.vue -->
<script setup lang="ts" generic="T">
const props = defineProps<{
  options: T[]
  modelValue: T | null
  getLabel: (option: T) => string
}>()

const emit = defineEmits<{
  'update:modelValue': [value: T]
}>()
</script>

<template>
  <select
    :value="modelValue ? getLabel(modelValue) : ''"
    @change="emit('update:modelValue', options[($event.target as HTMLSelectElement).selectedIndex])"
  >
    <option v-for="opt in options" :key="getLabel(opt)" :value="getLabel(opt)">
      {{ getLabel(opt) }}
    </option>
  </select>
</template>
```

## 三、注意事项与常见陷阱

1. **需要 Vue 3.3+**：泛型组件是较新的特性，确保版本支持
2. **`generic="T"` 必须写在 script 标签属性中**：不是普通的泛型定义
3. **泛型约束用 extends**：`generic="T extends BaseInterface"` 限制类型
4. **模板中也能使用泛型类型**：Vue 编译器会正确处理模板中的类型
5. **复杂泛型考虑提取到 .ts 文件**：过于复杂的类型定义放在单独文件中
