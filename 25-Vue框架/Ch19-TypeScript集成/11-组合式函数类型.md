# 组合式函数类型

## 一、概念说明

组合式函数（Composables）的类型定义决定了调用方的类型安全。返回值应使用明确的类型定义，支持泛型的组合式函数能适应多种数据场景。命名约定以 `use` 开头，返回值使用对象形式（非数组）以保持扩展性。

## 二、具体用法

### 基本 Composable 类型

```ts
// composables/useCounter.ts
import { ref, computed, type Ref, type ComputedRef } from 'vue'

interface UseCounterReturn {
  count: Ref<number>
  doubled: ComputedRef<number>
  increment: () => void
  decrement: () => void
  reset: () => void
}

export function useCounter(initialValue = 0): UseCounterReturn {
  const count = ref(initialValue)
  const doubled = computed(() => count.value * 2)

  function increment() { count.value++ }
  function decrement() { count.value-- }
  function reset() { count.value = initialValue }

  return { count, doubled, increment, decrement, reset }
}
```

```vue
<script setup lang="ts">
const { count, doubled, increment } = useCounter(10)
// count 类型: Ref<number>
// doubled 类型: ComputedRef<number>
// increment 类型: () => void
</script>
```

### 泛型 Composable

```ts
// composables/useFetch.ts
import { ref, type Ref } from 'vue'

interface UseFetchReturn<T> {
  data: Ref<T | null>
  error: Ref<Error | null>
  loading: Ref<boolean>
  fetch: () => Promise<void>
}

export function useFetch<T>(url: string): UseFetchReturn<T> {
  const data = ref<T | null>(null) as Ref<T | null>
  const error = ref<Error | null>(null)
  const loading = ref(false)

  async function fetch() {
    loading.value = true
    error.value = null
    try {
      const res = await window.fetch(url)
      data.value = await res.json()
    } catch (e) {
      error.value = e as Error
    } finally {
      loading.value = false
    }
  }

  return { data, error, loading, fetch }
}
```

```vue
<script setup lang="ts">
interface User {
  id: number
  name: string
  email: string
}

// 泛型参数指定返回数据类型
const { data: user, loading, fetch: fetchUser } = useFetch<User>('/api/user/1')
// user 类型: Ref<User | null>

onMounted(() => fetchUser())
</script>

<template>
  <div v-if="loading">加载中...</div>
  <div v-else-if="user">
    <h1>{{ user.name }}</h1>
    <p>{{ user.email }}</p>
  </div>
</template>
```

### 带选项的 Composable

```ts
// composables/useLocalStorage.ts
import { ref, watch, type Ref } from 'vue'

interface UseLocalStorageOptions<T> {
  serializer?: (value: T) => string
  deserializer?: (value: string) => T
}

export function useLocalStorage<T>(
  key: string,
  defaultValue: T,
  options: UseLocalStorageOptions<T> = {}
): Ref<T> {
  const { serializer = JSON.stringify, deserializer = JSON.parse } = options

  const stored = localStorage.getItem(key)
  const data = ref<T>(stored ? deserializer(stored) : defaultValue) as Ref<T>

  watch(data, (newVal) => {
    localStorage.setItem(key, serializer(newVal))
  }, { deep: true })

  return data
}
```

```ts
// 使用
const theme = useLocalStorage<'light' | 'dark'>('theme', 'light')
// theme 类型: Ref<'light' | 'dark'>

const config = useLocalStorage('app-config', {
  fontSize: 14,
  language: 'zh-CN'
})
// config 类型: Ref<{ fontSize: number; language: string }>
```

## 三、注意事项与常见陷阱

1. **返回对象而非数组**：对象便于扩展和解构，避免顺序依赖
2. **使用接口定义返回值**：让调用方获得明确的类型提示
3. **泛型参数约束数据类型**：使 composable 适应不同场景
4. **不要在 composable 中使用自动导入的 ref**：显式 import 保持可读性
5. **类型文件可提取到 types/ 目录**：复杂接口集中管理
