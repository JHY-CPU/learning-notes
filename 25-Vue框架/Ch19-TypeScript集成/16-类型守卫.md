# 类型守卫

## 一、概念说明

类型守卫（Type Guards）在运行时检查值的类型，让 TypeScript 在条件分支中自动收窄类型。在 Vue 中，模板和脚本中都可以使用类型守卫来安全地访问特定类型的属性。

## 二、具体用法

### typeof 守卫

```vue
<script setup lang="ts">
import { ref } from 'vue'

// 联合类型
const value = ref<string | number>('hello')

function displayValue() {
  if (typeof value.value === 'string') {
    // 此分支中 value.value 类型收窄为 string
    console.log(value.value.toUpperCase())
    // 输出：HELLO
  } else {
    // 此分支中 value.value 类型收窄为 number
    console.log(value.value.toFixed(2))
    // 输出：数字保留两位小数
  }
}

// API 响应联合类型
type ApiResponse = { success: true; data: string } | { success: false; error: string }

function handleResponse(res: ApiResponse) {
  if (res.success) {
    console.log('数据:', res.data)
    // res 类型收窄为 { success: true; data: string }
  } else {
    console.error('错误:', res.error)
    // res 类型收窄为 { success: false; error: string }
  }
}
</script>
```

### in 守卫

```vue
<script setup lang="ts">
// 区分不同对象类型
interface Cat {
  meow: () => void
  purr: () => void
}

interface Dog {
  bark: () => void
  fetch: () => void
}

type Pet = Cat | Dog

function playWith(pet: Pet) {
  if ('meow' in pet) {
    pet.meow()
    // pet 类型收窄为 Cat
    console.log('和猫咪玩耍')
  } else {
    pet.bark()
    // pet 类型收窄为 Dog
    console.log('和狗狗玩耍')
  }
}
</script>
```

### 自定义类型守卫

```ts
// 自定义类型守卫函数
interface Product {
  id: number
  name: string
  price: number
}

interface Service {
  id: number
  name: string
  hourlyRate: number
}

function isProduct(item: Product | Service): item is Product {
  return 'price' in item
}

function isService(item: Product | Service): item is Service {
  return 'hourlyRate' in item
}

function displayItem(item: Product | Service) {
  if (isProduct(item)) {
    console.log(`产品: ${item.name}, 价格: ¥${item.price}`)
    // item 类型收窄为 Product
  } else {
    console.log(`服务: ${item.name}, 时薪: ¥${item.hourlyRate}`)
    // item 类型收窄为 Service
  }
}
```

### 模板中的类型收窄

```vue
<script setup lang="ts">
const userData = ref<{ name: string } | null>(null)

// computed 中使用类型守卫
const hasUser = computed(() => userData.value !== null)

function loadUser() {
  userData.value = { name: '张三' }
}
</script>

<template>
  <div>
    <!-- v-if 自动收窄类型 -->
    <div v-if="userData">
      <h1>{{ userData.name }}</h1>
      <!-- userData 在此分支中类型为 { name: string } -->
    </div>
    <div v-else>
      <button @click="loadUser">加载用户</button>
    </div>
  </div>
</template>
```

## 三、注意事项与常见陷阱

1. **typeof 只能检查基本类型**：`typeof obj === 'object'` 只能确认是对象，不能确认具体接口
2. **`item is Type` 是类型谓词**：函数返回值的类型标注告诉 TS 如何收窄类型
3. **v-if 在模板中起类型守卫作用**：`v-if="user"` 后 user 自动排除 null
4. **类型守卫只在当前作用域有效**：异步回调中的类型不会自动收窄
5. **discriminated union 最佳实践**：使用字面量属性（如 `type`）区分联合类型
