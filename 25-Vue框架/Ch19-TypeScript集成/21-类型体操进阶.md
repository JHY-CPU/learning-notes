# 类型体操进阶

## 一、概念说明

TypeScript 的高级类型特性包括条件类型、映射类型、模板字面量类型、infer 关键字等。这些特性在 Vue 中用于构建精确的类型工具，如深度响应式类型、组件事件类型提取等。掌握这些能写出更精确的库和框架级代码。

## 二、具体用法

### 条件类型

```ts
// 根据条件选择类型
type IsString<T> = T extends string ? true : false

type A = IsString<string>  // true
type B = IsString<number>  // false

// Vue 中的应用：根据 props 类型选择行为
type ExtractProps<T> = T extends DefineComponent<infer P, any, any> ? P : never
```

### 映射类型

```ts
// 深度 Readonly
type DeepReadonly<T> = {
  readonly [K in keyof T]: T[K] extends object
    ? DeepReadonly<T[K]>
    : T[K]
}

interface Config {
  database: {
    host: string
    port: number
  }
  api: {
    url: string
    timeout: number
  }
}

type ReadonlyConfig = DeepReadonly<Config>
// 所有属性（包括嵌套）都变为 readonly

// Vue 应用：创建不可变的配置对象
const config: DeepReadonly<Config> = {
  database: { host: 'localhost', port: 5432 },
  api: { url: '/api', timeout: 5000 }
}
// config.database.host = 'other' → 只读错误
```

### 模板字面量类型

```ts
// 字符串模板类型
type EventName = `on${Capitalize<'click' | 'focus' | 'blur'>}`
// 'onClick' | 'onFocus' | 'onBlur'

// CSS 类名生成
type Size = 'sm' | 'md' | 'lg'
type Variant = 'primary' | 'secondary'
type ButtonClass = `btn-${Variant}-${Size}`
// 'btn-primary-sm' | 'btn-primary-md' | 'btn-primary-lg'
// | 'btn-secondary-sm' | 'btn-secondary-md' | 'btn-secondary-lg'

// Vue 路由路径类型
type RoutePath = `/${string}`
const path: RoutePath = '/users/123'  // 有效
// const bad: RoutePath = 'users' → 错误：不是 / 开头
```

### infer 关键字

```ts
// 提取函数返回类型
type ReturnType<T> = T extends (...args: any[]) => infer R ? R : never

type A = ReturnType<() => string>         // string
type B = ReturnType<(x: number) => boolean> // boolean

// 提取数组元素类型
type ElementOf<T> = T extends (infer E)[] ? E : never

type C = ElementOf<string[]>              // string
type D = ElementOf<[number, boolean]>     // number | boolean

// Vue 应用：提取 Store 返回类型
type StoreState<T> = T extends () => infer R ? R : never

const myStore = () => ({
  count: ref(0),
  user: ref({ name: 'test' })
})

type MyState = StoreState<typeof myStore>
// { count: Ref<number>; user: Ref<{ name: string }> }
```

### 联合类型转交叉类型

```ts
type UnionToIntersection<U> =
  (U extends any ? (k: U) => void : never) extends
  (k: infer I) => void ? I : never

type Result = UnionToIntersection<{ a: 1 } | { b: 2 }>
// { a: 1 } & { b: 2 }

// Vue 应用：合并多个插件的 provide 类型
type MergedProvides<T extends any[]> = UnionToIntersection<T[number]>
```

### 深度 Partial

```ts
type DeepPartial<T> = {
  [K in keyof T]?: T[K] extends object
    ? DeepPartial<T[K]>
    : T[K]
}

// 用于嵌套表单或配置的部分更新
function updateConfig(patch: DeepPartial<Config>) {
  // 只需要提供要更新的字段
}

updateConfig({
  database: { port: 3306 }  // 只更新端口，不用提供 host
})
```

## 三、注意事项与常见陷阱

1. **类型体操可能影响编译性能**：过于复杂的类型会导致 TS 语言服务变慢
2. **infer 只能在条件类型的 extends 子句中使用**：不能在其他位置使用
3. **映射类型会丢失可变性修饰符**：需要显式添加 readonly 或 -
4. **递归类型需要限制深度**：无限递归会导致 TS 报错
5. **项目中适度使用**：库和框架代码需要精密类型，业务代码保持简洁即可
