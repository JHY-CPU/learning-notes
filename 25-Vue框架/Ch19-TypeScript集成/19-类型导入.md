# 类型导入

## 一、概念说明

TypeScript 的 `import type` 语法仅导入类型信息，编译后被完全移除，不会增加运行时代码体积。Vue 3.3+ 的 `<script setup>` 支持内联类型导入。正确使用类型导入能优化构建产物大小和避免循环依赖。

## 二、具体用法

### 基本类型导入

```ts
// 方式1：import type（推荐）
import type { Ref, ComputedRef } from 'vue'
import type { RouteLocationRaw } from 'vue-router'
import type { User, Product } from '@/types/models'

// 这些导入在编译后完全消失，不会出现在 bundle 中

// 方式2：inline type import（混合导入）
import { ref, computed, type Ref } from 'vue'
// ref 和 computed 是运行时导入
// Ref 是类型导入
```

### SFC 中的类型导入

```vue
<script setup lang="ts">
// 类型导入
import type { User } from '@/types/models'
import type { ApiResponse } from '@/types/api'

// 运行时导入
import { useUserStore } from '@/stores/user'
import { formatDate } from '@/utils/date'

// 混合导入
import { ref, type Ref } from 'vue'

const user = ref<User | null>(null)
// user 类型: Ref<User | null>
</script>
```

### 重新导出类型

```ts
// types/index.ts
export type { User, Product, Order } from './models'
export type { ApiResponse, PaginatedResponse } from './api'
export type { ThemeConfig, AppConfig } from './config'

// 集中管理类型导出，其他文件只需从 types/index 导入
```

### const enum 与类型导入

```ts
// types/enums.ts
export const enum HttpStatus {
  OK = 200,
  NotFound = 404,
  ServerError = 500
}

// const enum 在编译时内联，不生成运行时对象
// 使用时直接替换为字面量值
```

```ts
// 使用
import { HttpStatus } from '@/types/enums'

if (response.status === HttpStatus.OK) {
  // 编译后：if (response.status === 200) { ... }
}
```

### PreserveValueImports 配置

```ts
// tsconfig.json
{
  "compilerOptions": {
    // Vue 3.3+ 推荐配置
    "verbatimModuleSyntax": true,
    // 强制区分 type import 和 regular import
    // 运行时导入不能用 import type
    // 类型导入必须用 import type
  }
}
```

## 三、注意事项与常见陷阱

1. **类型导入不能当值使用**：`import type { ref } from 'vue'` 后 `ref()` 会报错
2. **`verbatimModuleSyntax` 替代已废弃的选项**：`importsNotUsedAsValues` 和 `preserveValueImports` 已废弃
3. **Vue 编译器自动处理 SFC 中的类型导入**：不需要手动优化
4. **避免循环依赖**：类型导入不创建运行时依赖，可打破循环
5. **TypeScript 5.0+ 支持 `import { type X }` 语法**：在同一行混合导入中区分类型
