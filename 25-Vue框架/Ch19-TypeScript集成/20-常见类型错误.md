# 常见类型错误

## 一、概念说明

Vue + TypeScript 开发中常见的类型错误包括隐式 any、类型不兼容、null 未处理等。了解这些错误的原因和修复方法能显著提高开发效率。本节总结最常见问题和解决方案。

## 二、具体用法

### 隐式 any 错误

```ts
// 错误：参数隐式具有 any 类型
function formatUser(user) {
  // 严格模式下报错：参数 "user" 隐式具有 "any" 类型
  return user.name
}

// 修复：添加类型注释
function formatUser(user: { name: string }) {
  return user.name
}

// 或者定义接口
interface User { name: string }
function formatUser(user: User) {
  return user.name
}
```

### null 未处理

```ts
// 错误：Object is possibly 'null'
const inputRef = ref<HTMLInputElement | null>(null)

function focus() {
  inputRef.value.focus()
  // 报错：Object is possibly 'null'
}

// 修复1：可选链
function focus() {
  inputRef.value?.focus()
}

// 修复2：非空断言（需确认不为 null）
function focus() {
  inputRef.value!.focus()
}

// 修复3：if 守卫
function focus() {
  if (inputRef.value) {
    inputRef.value.focus()
  }
}
```

### 类型不兼容

```ts
// 错误：不能将 string 赋给 number
const count = ref(0)
count.value = 'hello'
// 类型错误：不能将类型 "string" 分配给类型 "number"

// 修复：正确的类型转换
count.value = parseInt('42', 10)
```

### Props 类型错误

```vue
<script setup lang="ts">
// 错误：withDefaults 中的值不匹配类型
interface Props {
  count: number
  items: string[]
}

// withDefaults 的值必须与类型兼容
const props = withDefaults(defineProps<Props>(), {
  count: 0,          // 正确：0 是 number
  items: () => []    // 正确：空数组是 string[]
  // items: 'hello'  // 错误：string 不是 string[]
})
</script>
```

### 事件类型错误

```ts
// 错误：事件参数类型不匹配
const emit = defineEmits<{
  submit: [data: { name: string }]
}>()

// emit('submit', { wrong: 'data' })
// 错误：类型 "{ wrong: string }" 的参数不能赋给类型 "{ name: string }" 的参数

// 修复：传入正确的对象
emit('submit', { name: '张三' })
```

### 泛型类型推断失败

```ts
// 错误：泛型推断不理想
const items = ref([])  // 类型: Ref<never[]>

// items.value.push('hello') → 报错：never 类型不能 push

// 修复：显式指定泛型
const items = ref<string[]>([])
items.value.push('hello')  // 正确
```

### 递归类型溢出

```ts
// 错误：类型实例化过深
type DeepPartial<T> = {
  [P in keyof T]?: DeepPartial<T[P]>
}

// 对嵌套过深的类型可能溢出

// 修复：限制递归深度
type DeepPartial<T, D extends number = 5> = D extends 0
  ? T
  : {
      [P in keyof T]?: DeepPartial<T[P], Prev[D]>
    }
```

## 三、注意事项与常见陷阱

1. **开启 strict: true**：大部分类型错误在非严格模式下被忽略
2. **不要频繁使用 `as` 断言**：断言绕过类型检查，应尽量用类型守卫
3. **`any` 是逃生舱**：遇到无法解决的类型问题可用 any，但要添加 TODO 注释
4. **IDE 的快速修复功能很有用**：`Ctrl+.` 或 `Cmd+.` 常能自动修复类型错误
5. **版本升级可能引入新的类型错误**：Vue 或 TS 升级后需要检查类型兼容性
