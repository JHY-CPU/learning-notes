# 响应拦截器

## 一、概念说明

响应拦截器在收到响应后自动执行，用于统一处理响应数据解包、错误状态码、Token 过期刷新等逻辑。

```js
// utils/request.js
request.interceptors.response.use(
  (response) => {
    const { data, config } = response

    // 直接返回数据，省去每次 .data
    if (config.raw) return response // 需要原始响应时
    return data
  },
  (error) => {
    if (error.response) {
      const { status } = error.response
      switch (status) {
        case 401: handleUnauthorized(); break
        case 403: console.error('无权限'); break
        case 500: console.error('服务器错误'); break
      }
    } else if (error.request) {
      console.error('网络异常')
    }
    return Promise.reject(error)
  }
)
```

## 二、具体用法

### 2.1 统一解包数据

```js
// 假设后端返回 { code: 0, data: ..., message: '' }
request.interceptors.response.use(res => {
  if (res.data.code !== 0) {
    return Promise.reject(new Error(res.data.message))
  }
  return res.data.data
})
```

### 2.2 Token 刷新

```js
let isRefreshing = false
let queue = []

request.interceptors.response.use(
  res => res,
  async error => {
    if (error.response?.status === 401 && !error.config._retry) {
      if (!isRefreshing) {
        isRefreshing = true
        const newToken = await refreshToken()
        queue.forEach(cb => cb(newToken))
        queue = []
        isRefreshing = false
      }
      return new Promise(resolve => {
        queue.push(token => {
          error.config.headers.Authorization = `Bearer ${token}`
          resolve(request(error.config))
        })
      })
    }
    return Promise.reject(error)
  }
)
```

### 2.3 错误分类

| 状态码 | 含义 | 处理方式 |
|--------|------|---------|
| 400 | 参数错误 | 提示用户修改 |
| 401 | 未认证 | 跳转登录页 |
| 403 | 无权限 | 提示无权访问 |
| 404 | 资源不存在 | 提示并返回 |
| 500 | 服务器错误 | 提示稍后重试 |

## 三、注意事项与常见陷阱

- 响应拦截器的错误处理函数捕获的是 HTTP 层错误
- 业务错误（code 不为 0）应在响应成功回调中处理
- Token 刷新要处理并发请求的队列问题
