# 请求取消

## 一、概念说明

在用户快速切换页面、重复点击或组件卸载时，需要取消未完成的请求，避免资源浪费和数据错乱。现代方案使用 `AbortController`，旧方案使用 `CancelToken`（已废弃）。

```vue
<script setup>
import { ref, onUnmounted } from 'vue'
import axios from 'axios'

const data = ref(null)
const controller = new AbortController()

async function fetchData() {
  try {
    const res = await axios.get('/api/data', {
      signal: controller.signal
    })
    data.value = res.data
  } catch (e) {
    if (axios.isCancel(e)) {
      console.log('请求已取消')
    } else {
      console.error('请求失败:', e)
    }
  }
}

onUnmounted(() => {
  controller.abort() // 组件卸载时取消请求
})
</script>
```

## 二、具体用法

### 2.1 AbortController

```js
const controller = new AbortController()

// 发起请求
axios.get('/api/data', { signal: controller.signal })

// 取消请求
controller.abort('用户取消')
```

### 2.2 页面离开取消所有请求

```js
// 在路由守卫中
const pendingControllers = []

function addRequest(controller) {
  pendingControllers.push(controller)
}

router.beforeEach(() => {
  pendingControllers.forEach(c => c.abort())
  pendingControllers.length = 0
})
```

### 2.3 防止重复请求

```js
const requestMap = new Map()

function requestWithDedup(url, config) {
  const key = `${config.method}:${url}`
  if (requestMap.has(key)) {
    requestMap.get(key).abort()
  }
  const controller = new AbortController()
  requestMap.set(key, controller)
  return axios({ ...config, url, signal: controller.signal })
    .finally(() => requestMap.delete(key))
}
```

## 四、组件级别取消管理

```js
// composables/useAbortableFetch.js
import { ref, onUnmounted } from 'vue'

export function useAbortableFetch() {
  const controller = ref(null)

  function abort() {
    controller.value?.abort()
    controller.value = null
  }

  async function fetch(url, options = {}) {
    abort()  // 取消上一次请求
    controller.value = new AbortController()

    try {
      const res = await axios.get(url, {
        ...options,
        signal: controller.value.signal
      })
      return res.data
    } catch (e) {
      if (!axios.isCancel(e)) throw e
      return null
    }
  }

  onUnmounted(abort)

  return { fetch, abort }
}

// 使用
const { fetch: fetchUser } = useAbortableFetch()
const data = await fetchUser('/api/users/1')
```

## 五、搜索防抖 + 取消

```js
import { ref, watch } from 'vue'

const keyword = ref('')
let timer = null
let controller = null

watch(keyword, (val) => {
  clearTimeout(timer)
  controller?.abort()

  timer = setTimeout(async () => {
    controller = new AbortController()
    try {
      const { data } = await axios.get('/api/search', {
        params: { q: val },
        signal: controller.signal
      })
      results.value = data
    } catch (e) {
      if (!axios.isCancel(e)) console.error(e)
    }
  }, 300)
})
```

## 六、竞态条件处理

```js
// 问题：快速切换页面时，旧请求可能晚于新请求返回
// 解决：使用请求 ID 判断响应是否仍然有效

let currentRequestId = 0

async function fetchData(id) {
  const requestId = ++currentRequestId
  const { data } = await axios.get(`/api/data/${id}`)

  // 如果有更新的请求已经发出，忽略这个响应
  if (requestId !== currentRequestId) return null

  return data
}
```

## 三、注意事项与常见陷阱

- `CancelToken` 已被 Axios 废弃，优先使用 `AbortController`
- 取消请求会触发错误回调，需用 `axios.isCancel()` 判断
- 取消后的请求不会发送到服务器，服务器无感知
- 组件卸载时取消请求可以避免更新已卸载组件的状态
- 竞态条件除了取消请求，也可以用请求 ID 方案解决
