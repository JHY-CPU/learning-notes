# 请求缓存策略

## 一、概念说明

缓存 API 响应可以减少重复请求，提升应用性能和用户体验。常见策略包括：简单缓存、SWR（Stale-While-Revalidate）和带过期时间的缓存。

```js
// composables/useCachedFetch.js
import { ref, shallowRef } from 'vue'

const cache = new Map()

export function useCachedFetch(key, fetcher, { maxAge = 60000 } = {}) {
  const data = shallowRef(null)
  const loading = ref(false)

  async function execute() {
    const cached = cache.get(key)
    const now = Date.now()

    // 缓存有效，直接返回
    if (cached && now - cached.time < maxAge) {
      data.value = cached.data
      return cached.data
    }

    loading.value = true
    try {
      const result = await fetcher()
      cache.set(key, { data: result, time: now })
      data.value = result
      return result
    } finally {
      loading.value = false
    }
  }

  execute()
  return { data, loading, refresh: execute }
}
```

## 二、具体用法

### 2.1 SWR 策略

```js
// 先返回缓存，后台更新
async function swrFetch(key, fetcher) {
  const cached = cache.get(key)
  if (cached) {
    // 立即返回旧数据
    // 后台刷新
    fetcher().then(data => cache.set(key, { data, time: Date.now() }))
    return cached.data
  }
  const data = await fetcher()
  cache.set(key, { data, time: Date.now() })
  return data
}
```

### 2.2 请求去重

```js
const pending = new Map()

function dedupFetch(key, fetcher) {
  if (pending.has(key)) {
    return pending.get(key) // 返回进行中的 Promise
  }
  const promise = fetcher().finally(() => pending.delete(key))
  pending.set(key, promise)
  return promise
}
```

### 2.3 缓存失效

```js
function invalidateCache(key) {
  cache.delete(key)
}

function invalidateByPrefix(prefix) {
  for (const key of cache.keys()) {
    if (key.startsWith(prefix)) cache.delete(key)
  }
}
```

## 三、注意事项与常见陷阱

- 缓存 key 要包含所有影响结果的参数
- 写操作（POST/PUT/DELETE）后要清除相关缓存
- 注意内存占用，设置合理的缓存条数上限
