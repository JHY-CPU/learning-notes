# 并发请求

## 一、概念说明

当需要同时发起多个请求并等待全部完成时，使用并发请求。常用 `Promise.all`、`Promise.allSettled` 或 Axios 提供的 `axios.all` 实现。

```vue
<script setup>
import { ref, onMounted } from 'vue'
import axios from 'axios'

const users = ref([])
const posts = ref([])
const loading = ref(false)

onMounted(async () => {
  loading.value = true
  try {
    const [usersRes, postsRes] = await Promise.all([
      axios.get('/api/users'),
      axios.get('/api/posts')
    ])
    users.value = usersRes.data
    posts.value = postsRes.data
  } finally {
    loading.value = false
  }
})
</script>

<template>
  <div v-if="loading">加载中...</div>
  <div v-else>
    <h3>用户 ({{ users.length }})</h3>
    <h3>文章 ({{ posts.length }})</h3>
  </div>
</template>
```

## 二、具体用法

### 2.1 Promise.all（全部成功才成功）

```js
const [res1, res2, res3] = await Promise.all([
  axios.get('/api/users'),
  axios.get('/api/posts'),
  axios.get('/api/comments')
])
// 任一请求失败，整体失败
```

### 2.2 Promise.allSettled（无论成功失败都返回）

```js
const results = await Promise.allSettled([
  axios.get('/api/users'),
  axios.get('/api/posts')
])
results.forEach(r => {
  if (r.status === 'fulfilled') console.log(r.value.data)
  else console.error(r.reason)
})
```

### 2.3 Promise.race（取最快的结果）

```js
// 从多个数据源取最快响应
const result = await Promise.race([
  axios.get('/api/primary/data'),
  axios.get('/api/backup/data')
])
```

### 2.4 限制并发数量

```js
async function limitConcurrency(tasks, max = 3) {
  const results = []
  const executing = new Set()

  for (const task of tasks) {
    const p = task().then(r => { executing.delete(p); return r })
    executing.add(p)
    results.push(p)
    if (executing.size >= max) {
      await Promise.race(executing)
    }
  }
  return Promise.all(results)
}
```

## 三、注意事项与常见陷阱

- `Promise.all` 有一个失败即全部失败，使用时需评估是否可接受
- 并发数过多可能触发浏览器同域连接数限制（通常 6 个）
- 需要部分失败时使用 `Promise.allSettled`
