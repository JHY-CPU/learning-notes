# 错误处理策略

## 一、概念说明

HTTP 请求中的错误分为三层：**网络错误**（断网、超时）、**HTTP 错误**（4xx/5xx）、**业务错误**（接口正常但业务逻辑失败）。需要不同策略分别处理。

```js
import axios from 'axios'
import { ElMessage } from 'element-plus'

const request = axios.create({ timeout: 10000 })

request.interceptors.response.use(
  (res) => {
    // 业务错误处理
    if (res.data.code !== 0) {
      ElMessage.error(res.data.message || '业务异常')
      return Promise.reject(new Error(res.data.message))
    }
    return res.data.data
  },
  (error) => {
    // 网络/HTTP 错误处理
    if (!error.response) {
      // 网络错误或超时
      ElMessage.error(error.code === 'ECONNABORTED' ? '请求超时' : '网络异常')
    } else {
      const msg = { 401: '请重新登录', 403: '无权限', 500: '服务器错误' }
      ElMessage.error(msg[error.response.status] || `错误: ${error.response.status}`)
    }
    return Promise.reject(error)
  }
)
```

## 二、具体用法

### 2.1 错误类型分类

```js
function handleError(error) {
  if (error.code === 'ECONNABORTED') {
    return '请求超时，请重试'
  }
  if (!error.response) {
    return '网络连接失败'
  }
  const { status, data } = error.response
  return data?.message || `HTTP 错误: ${status}`
}
```

### 2.2 重试机制

```js
async function requestWithRetry(config, retries = 3) {
  for (let i = 0; i < retries; i++) {
    try {
      return await axios(config)
    } catch (e) {
      if (i === retries - 1) throw e
      await new Promise(r => setTimeout(r, 1000 * (i + 1))) // 指数退避
    }
  }
}
```

### 2.3 统一错误提示

```js
// composables/useRequest.js
import { ref } from 'vue'

export function useRequest(fn) {
  const loading = ref(false)
  const error = ref(null)

  async function execute(...args) {
    loading.value = true
    error.value = null
    try {
      return await fn(...args)
    } catch (e) {
      error.value = e.message
      throw e
    } finally {
      loading.value = false
    }
  }

  return { loading, error, execute }
}
```

## 三、注意事项与常见陷阱

- 不要 `console.log` 后就吞掉错误，至少要 `return Promise.reject(error)`
- 超时时间要合理设置，太短容易误判，太长用户体验差
- 重试次数不宜过多，避免对服务器造成压力
