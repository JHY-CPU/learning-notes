# 请求拦截器

## 一、概念说明

请求拦截器在请求发出前自动执行，用于统一处理认证 Token、请求头、请求参数等。避免在每个请求中重复编写相同的逻辑。

```js
// utils/request.js
import axios from 'axios'

const request = axios.create({
  baseURL: import.meta.env.VITE_API_BASE,
  timeout: 10000
})

// 请求拦截器
request.interceptors.request.use(
  (config) => {
    // 自动添加 Token
    const token = localStorage.getItem('token')
    if (token) {
      config.headers.Authorization = `Bearer ${token}`
    }
    // 添加时间戳防止缓存
    if (config.method === 'get') {
      config.params = { ...config.params, _t: Date.now() }
    }
    return config
  },
  (error) => Promise.reject(error)
)

export default request
```

## 二、具体用法

### 2.1 常见用途

```js
request.interceptors.request.use(config => {
  // 1. 添加认证 Token
  config.headers.Authorization = `Bearer ${getToken()}`

  // 2. 添加统一请求头
  config.headers['X-Request-Id'] = generateId()

  // 3. 请求前显示 loading
  store.commit('setLoading', true)

  // 4. 参数预处理
  if (config.method === 'post') {
    config.data = formatData(config.data)
  }

  return config
})
```

### 2.2 多个拦截器

```js
// 可以添加多个拦截器，按注册顺序执行
request.interceptors.request.use(authInterceptor)
request.interceptors.request.use(logInterceptor)
```

## 三、注意事项与常见陷阱

- 拦截器中必须 `return config`，否则请求会被阻断
- 不要在拦截器中执行异步操作（如 refresh token）而不处理好竞态
- 错误处理函数中也要返回 `Promise.reject(error)`，否则错误被吞掉
