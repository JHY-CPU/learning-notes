# 请求拦截器

## 一、概念说明

请求拦截器在请求发出前自动执行，用于统一处理认证 Token、请求头、请求参数等。避免在每个请求中重复编写相同的逻辑。

```js
// utils/request.js
import axios from 'axios'

const request = axios.create({
  baseURL: import.meta.env.VITE_API_BASE,
  timeout: 10000
})

// 请求拦截器
request.interceptors.request.use(
  (config) => {
    // 自动添加 Token
    const token = localStorage.getItem('token')
    if (token) {
      config.headers.Authorization = `Bearer ${token}`
    }
    // 添加时间戳防止缓存
    if (config.method === 'get') {
      config.params = { ...config.params, _t: Date.now() }
    }
    return config
  },
  (error) => Promise.reject(error)
)

export default request
```

## 二、具体用法

### 2.1 常见用途

```js
request.interceptors.request.use(config => {
  // 1. 添加认证 Token
  config.headers.Authorization = `Bearer ${getToken()}`

  // 2. 添加统一请求头
  config.headers['X-Request-Id'] = generateId()

  // 3. 请求前显示 loading
  store.commit('setLoading', true)

  // 4. 参数预处理
  if (config.method === 'post') {
    config.data = formatData(config.data)
  }

  return config
})
```

### 2.2 多个拦截器

```js
// 可以添加多个拦截器，按注册顺序执行
request.interceptors.request.use(authInterceptor)
request.interceptors.request.use(logInterceptor)
```

## 四、响应拦截器

```js
// 响应拦截器
request.interceptors.response.use(
  (response) => {
    // 统一处理响应数据
    const { data, code, message } = response.data

    if (code === 0) {
      return data  // 直接返回业务数据
    }

    // 业务错误
    return Promise.reject(new Error(message || '请求失败'))
  },
  (error) => {
    // HTTP 错误
    const status = error.response?.status

    switch (status) {
      case 401:
        // Token 过期，尝试刷新
        return handleTokenRefresh(error)
      case 403:
        console.error('没有权限')
        break
      case 404:
        console.error('资源不存在')
        break
      case 500:
        console.error('服务器错误')
        break
      default:
        if (!error.response) {
          console.error('网络错误，请检查网络连接')
        }
    }

    return Promise.reject(error)
  }
)
```

## 五、Token 刷新与请求重试

```js
let isRefreshing = false
let pendingRequests = []

async function handleTokenRefresh(error) {
  const originalRequest = error.config

  if (isRefreshing) {
    // 正在刷新中，将请求加入队列
    return new Promise((resolve, reject) => {
      pendingRequests.push({ resolve, reject })
    }).then(token => {
      originalRequest.headers.Authorization = `Bearer ${token}`
      return request(originalRequest)
    })
  }

  isRefreshing = true
  try {
    const { data } = await axios.post('/api/auth/refresh', {
      refreshToken: localStorage.getItem('refreshToken')
    })
    const newToken = data.token
    localStorage.setItem('token', newToken)

    // 重试所有挂起的请求
    pendingRequests.forEach(p => p.resolve(newToken))
    pendingRequests = []

    originalRequest.headers.Authorization = `Bearer ${newToken}`
    return request(originalRequest)
  } catch (refreshError) {
    pendingRequests.forEach(p => p.reject(refreshError))
    pendingRequests = []
    // 刷新失败，跳转登录
    localStorage.removeItem('token')
    window.location.href = '/login'
    return Promise.reject(refreshError)
  } finally {
    isRefreshing = false
  }
}
```

## 六、请求日志拦截器

```js
request.interceptors.request.use(config => {
  config.metadata = { startTime: Date.now() }
  console.log(`[Request] ${config.method?.toUpperCase()} ${config.url}`)
  return config
})

request.interceptors.response.use(
  (response) => {
    const duration = Date.now() - response.config.metadata.startTime
    console.log(`[Response] ${response.config.url} - ${duration}ms`)
    return response
  },
  (error) => {
    const duration = Date.now() - error.config?.metadata?.startTime
    console.error(`[Error] ${error.config?.url} - ${duration}ms`, error)
    return Promise.reject(error)
  }
)
```

## 三、注意事项与常见陷阱

- 拦截器中必须 `return config`，否则请求会被阻断
- 不要在拦截器中执行异步操作（如 refresh token）而不处理好竞态
- 错误处理函数中也要返回 `Promise.reject(error)`，否则错误被吞掉
- Token 刷新需要处理并发请求的队列逻辑
- 响应拦截器中的业务错误和 HTTP 错误应分别处理
