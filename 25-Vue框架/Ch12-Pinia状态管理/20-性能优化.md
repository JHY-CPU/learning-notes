# 性能优化

## 一、概念说明

Pinia本身性能良好，但在大型应用中需要注意：避免过度订阅、合理使用computed缓存、批量修改状态等。

```js
// ❌ 多次单独修改（多次触发更新）
store.name = '新名字'
store.age = 25
store.email = 'new@example.com'

// ✅ $patch批量修改（一次更新）
store.$patch({
  name: '新名字',
  age: 25,
  email: 'new@example.com'
})
```

## 二、具体用法

### 使用computed缓存

```js
// ❌ 每次渲染都计算
const totalPrice = () => {
  return store.items.reduce((s, i) => s + i.price * i.quantity, 0)
}

// ✅ computed有缓存
const totalPrice = computed(() =>
  store.items.reduce((s, i) => s + i.price * i.quantity, 0)
)
```

### 使用shallowRef存储大对象

```js
export const useDataStore = defineStore('data', () => {
  // 大数据使用shallowRef，不深度代理
  const largeList = shallowRef([])

  const fetchData = async () => {
    largeList.value = await api.getLargeList()
  }

  return { largeList, fetchData }
})
```

### 按需订阅

```vue
<script setup>
// ❌ 订阅整个store
const store = useBigStore()

// ✅ 只解构需要的字段
const { name, age } = storeToRefs(useBigStore())
</script>
```

## 三、注意事项与常见陷阱

1. `$patch`比逐个修改性能更好（合并为一次更新）
2. 大数组/对象使用`shallowRef`避免深度响应式
3. 避免在循环中频繁读取store
4. getter有缓存，复杂计算放在getter中
5. 不需要的组件不要导入不必要的store
