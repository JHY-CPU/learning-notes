# 全局状态vs局部状态

## 一、概念说明

**全局状态**（Pinia Store）：多组件共享的数据。**局部状态**（组件内ref）：仅组件内部使用的数据。正确区分避免过度使用全局状态。

```vue
<script setup>
// 局部状态：仅在组件内使用
const inputValue = ref('')
const isExpanded = ref(false)

// 全局状态：多组件共享
import { useUserStore } from '@/stores/user'
import { useCartStore } from '@/stores/cart'

const userStore = useUserStore()
const cartStore = useCartStore()
</script>
```

## 二、具体用法

### 何时使用全局状态

| 场景 | 例子 | 使用 |
|------|------|------|
| 多组件共享 | 用户信息、登录状态 | Pinia |
| 跨页面保持 | 购物车、表单草稿 | Pinia |
| 需要持久化 | 用户偏好设置 | Pinia |
| 后端数据缓存 | 商品列表缓存 | Pinia |

### 何时使用局部状态

| 场景 | 例子 | 使用 |
|------|------|------|
| UI状态 | 弹窗开关、折叠状态 | ref |
| 表单输入 | 输入框内容 | ref |
| 组件私有 | 当前页码、加载状态 | ref |
| 临时计算 | 拖拽位置、动画状态 | ref |

### 判断原则

```
数据是否需要跨组件/跨页面共享？
  ├── 是 -> Pinia Store
  └── 否 -> 组件内 ref/computed
```

## 四、状态提升示例

```vue
<!-- 初始：弹窗状态在组件内 -->
<script setup>
const showModal = ref(false)
</script>

<!-- 后来：发现需要从其他组件打开弹窗 → 提升为 Store -->
// stores/modal.js
export const useModal = defineStore('modal', () => {
  const visible = ref(false)
  const type = ref('')
  const data = ref(null)

  const open = (modalType, modalData = null) => {
    type.value = modalType
    data.value = modalData
    visible.value = true
  }

  const close = () => {
    visible.value = false
    type.value = ''
    data.value = null
  }

  return { visible, type, data, open, close }
})
```

## 五、混合使用模式

```vue
<script setup>
import { useUserStore } from '@/stores/user'
import { ref, computed } from 'vue'

// 全局：用户信息
const userStore = useUserStore()

// 局部：表单状态
const form = ref({ name: '', email: '' })
const editing = ref(false)

// 局部：UI状态
const showPassword = ref(false)
const activeTab = ref('profile')

// 局部：计算属性（派生自全局 + 局部）
const hasChanges = computed(() =>
  form.value.name !== userStore.profile?.name ||
  form.value.email !== userStore.profile?.email
)
</script>
```

## 三、注意事项与常见陷阱

1. 不要所有状态都放Store，UI状态留组件内
2. 不要因为"怕以后可能用到"就提前放Store
3. Store是响应式的，但不必要的订阅影响性能
4. 可以将组件状态提升为Store（当发现需要共享时）
5. 页面级状态（如分页）可以不放全局Store
6. 表单状态通常保持局部，除非需要跨页面保持
7. Composable 可以封装局部逻辑，不需要放 Store
