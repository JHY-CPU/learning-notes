# 订阅状态变化

## 一、概念说明

Pinia提供两个订阅方法：`$subscribe`（监听状态变化）和`$onAction`（监听action调用）。用于日志记录、持久化、调试等。

```js
const store = useCounterStore()

// 监听状态变化
store.$subscribe((mutation, state) => {
  console.log('Store变化:', mutation.storeId)
  console.log('类型:', mutation.type)  // 'direct' | 'patch object' | 'patch function'
  console.log('新状态:', state)
})

// 监听action调用
store.$onAction(({ name, args, after, onError }) => {
  console.log(`调用 ${name}，参数:`, args)
  after((result) => console.log('完成:', result))
  onError((error) => console.error('失败:', error))
})
```

## 二、具体用法

### $subscribe详细用法

```js
store.$subscribe((mutation, state) => {
  // mutation.type: 修改类型
  // mutation.storeId: store ID
  // mutation.events: 具体事件（仅dev模式）

  // 自动保存到localStorage
  localStorage.setItem('store', JSON.stringify(state))
}, {
  detached: true,  // 组件卸载后仍监听
  deep: true,      // 深度监听
  flush: 'sync'    // 同步触发
})
```

### $onAction详细用法

```js
store.$onAction(({ name, store, args, after, onError }) => {
  const startTime = Date.now()

  after((result) => {
    console.log(`${name} 耗时: ${Date.now() - startTime}ms`)
    console.log('结果:', result)
  })

  onError((error) => {
    console.error(`${name} 失败:`, error)
    // 发送错误报告
    reportError(error)
  })
})
```

## 四、持久化存储

```js
const store = useUserStore()

// 自动保存到 localStorage
store.$subscribe((mutation, state) => {
  localStorage.setItem('user-store', JSON.stringify(state))
}, { detached: true })

// 页面加载时恢复
const saved = localStorage.getItem('user-store')
if (saved) {
  store.$patch(JSON.parse(saved))
}
```

## 五、性能监控

```js
const store = useProductStore()

store.$onAction(({ name, args, after, onError }) => {
  const startTime = performance.now()
  const startMemory = performance.memory?.usedJSHeapSize

  after((result) => {
    const duration = performance.now() - startTime
    const memoryDelta = performance.memory
      ? performance.memory.usedJSHeapSize - startMemory
      : null

    console.log(`[Perf] ${name}: ${duration.toFixed(2)}ms`, memoryDelta
      ? `| 内存变化: ${(memoryDelta / 1024).toFixed(1)}KB`
      : '')

    // 超时警告
    if (duration > 1000) {
      console.warn(`[Slow Action] ${name} 耗时 ${duration.toFixed(0)}ms`)
    }
  })
})
```

## 六、跨组件同步

```js
// Store A 中订阅，通知 Store B
const storeA = useAuthStore()
const storeB = useNotificationStore()

storeA.$subscribe((mutation, state) => {
  if (!state.isLoggedIn) {
    storeB.add({ type: 'warning', text: '会话已过期' })
  }
})
```

## 三、注意事项与常见陷阱

1. `$subscribe`在组件卸载时自动取消（除非`detached: true`）
2. `$onAction`的`after`回调在action成功后执行
3. `$subscribe`的`flush`选项控制触发时机（`sync`/`pre`/`post`）
4. 订阅返回取消函数：`const unsubscribe = store.$subscribe(...)`
5. 避免在订阅中修改状态（可能导致无限循环）
6. `$subscribe`的`detached: true`用于全局订阅（如持久化插件）
7. `$onAction`的`onError`可以捕获异步 action 的错误
