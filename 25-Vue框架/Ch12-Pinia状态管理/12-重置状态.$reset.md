# 重置状态.$reset

## 一、概念说明

`$reset()`方法将Store的状态恢复到初始值。仅在选项式Store中可用，Setup式需手动实现。

```js
export const useCounter = defineStore('counter', {
  state: () => ({ count: 0, name: '计数器' }),
  actions: {
    increment() { this.count++ }
  }
})

// 使用
const store = useCounter()
store.increment()
console.log(store.count) // 1
store.$reset()
console.log(store.count) // 0  // 恢复初始值
```

## 二、具体用法

### Setup式手动实现reset

```js
export const useCounter = defineStore('counter', () => {
  const getDefault = () => ({ count: 0, name: '计数器' })
  const state = reactive(getDefault())

  const reset = () => {
    Object.assign(state, getDefault())
  }

  return { ...toRefs(state), reset }
})
```

### 批量重置多个Store

```js
// utils/resetStores.js
export function resetAllStores(pinia) {
  // 遍历所有已注册的store
  Object.keys(pinia._s).forEach((key) => {
    const store = pinia._s.get(key)
    store?.$reset?.()
  })
}

// 使用
import { getActivePinia } from 'pinia'
resetAllStores(getActivePinia())
```

### 重置部分状态

```js
const store = useUserStore()

// 只重置部分字段
store.$patch((state) => {
  state.token = null
  state.error = null
  // 不重置user，保留
})
```

## 三、注意事项与常见陷阱

1. `$reset()`只在**选项式**Store中可用
2. Setup式Store需自己实现reset函数
3. `$reset()`是通过`$patch`实现的，会触发订阅
4. 重置时会丢失所有修改的状态
5. 深层嵌套对象会被完全替换（非合并）
