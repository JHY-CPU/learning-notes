# 性能优化动画

## 一、概念说明

动画性能直接影响用户体验。低效的动画会导致卡顿、掉帧。核心原则是：**优先使用 `transform` 和 `opacity`**，因为它们可以由 GPU 加速，不会触发重排（reflow）。

```vue
<script setup>
import { ref } from 'vue'
const expanded = ref(false)
</script>

<template>
  <button @click="expanded = !expanded">展开</button>
  <div
    class="optimised-box"
    :class="{ expanded }"
  >
    GPU 加速的动画
  </div>
</template>

<style>
.optimised-box {
  will-change: transform, opacity;
  transform: translateZ(0); /* 引发 GPU 加速 */
  transition: transform 0.3s ease, opacity 0.3s ease;
  opacity: 0.5;
}
.optimised-box.expanded {
  transform: translateZ(0) scale(1.2);
  opacity: 1;
}
</style>
```

## 二、具体用法

### 2.1 will-change 属性

提示浏览器哪些属性将要变化，让浏览器提前优化。

```css
.animated {
  will-change: transform, opacity;
}
```

### 2.2 避免动画属性

| 低性能属性 | 高性能替代 |
|-----------|-----------|
| `top` / `left` | `transform: translate()` |
| `width` / `height` | `transform: scale()` |
| `margin` / `padding` | `transform: translate()` |

### 2.3 GPU 加速技巧

```css
.hardware-accelerated {
  transform: translateZ(0);
  /* 或 */
  backface-visibility: hidden;
  perspective: 1000px;
}
```

## 四、动画性能分析

```
渲染层变化 (从快到慢):
┌──────────────────────────────────────────────────┐
│  Composite (合成)     ← transform, opacity       │  GPU加速
│  Paint (绘制)         ← color, box-shadow        │  部分GPU
│  Layout (布局)        ← width, height, margin    │  CPU计算
└──────────────────────────────────────────────────┘

目标：尽量只触发 Composite 层
```

| 属性 | 触发的层 | 性能 |
|------|---------|------|
| `transform` | Composite | 优秀 |
| `opacity` | Composite | 优秀 |
| `filter` | Composite | 良好 |
| `color` | Paint | 一般 |
| `background-color` | Paint | 一般 |
| `width` / `height` | Layout | 差 |
| `margin` / `padding` | Layout | 差 |
| `box-shadow` | Paint | 一般 |

## 五、动态 will-change 管理

```vue
<script setup>
import { ref } from 'vue'
const animating = ref(false)
const boxRef = ref(null)

function startAnimation() {
  // 动画前添加 will-change
  animating.value = true

  // 动画结束后移除
  boxRef.value?.addEventListener('transitionend', () => {
    animating.value = false
  }, { once: true })
}
</script>

<template>
  <div
    ref="boxRef"
    class="box"
    :class="{ animating }"
    :style="{ willChange: animating ? 'transform' : 'auto' }"
    @click="startAnimation"
  >
    点击动画
  </div>
</template>

<style>
.box {
  transition: transform 0.3s ease;
}
.box.animating {
  transform: scale(1.2);
}
</style>
```

## 六、减少动画帧数的技巧

```css
/* 1. 使用 contain 限制重排范围 */
.animated-container {
  contain: layout style;
}

/* 2. 使用 isolation 创建新的层叠上下文 */
.animated-element {
  isolation: isolate;
}

/* 3. 使用 content-visibility 跳过屏幕外元素 */
.offscreen-items {
  content-visibility: auto;
}

/* 4. 动画元素提升到独立层 */
.animated {
  will-change: transform;
  transform: translateZ(0);
}
```

## 三、注意事项与常见陷阱

- `will-change` 不要滥用，过多使用反而消耗内存
- 使用 `transform` 和 `opacity` 动画不会触发 layout 和 paint，性能最佳
- 动画完成后移除 `will-change`，释放 GPU 资源
- `contain` CSS 属性可以限制动画影响的渲染范围
- 移动端尤其要注意动画性能，低端设备 GPU 资源有限
