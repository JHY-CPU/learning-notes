# 交错列表过渡

## 一、概念说明

交错列表过渡是指列表中每个元素的动画**依次延迟执行**，产生波浪式或级联式的视觉效果。这需要结合 JavaScript 钩子来动态设置每个元素的延迟时间。

```vue
<script setup>
import { ref } from 'vue'
const items = ref([])
let nextId = 0

function addItem() {
  items.value.push({ id: nextId++, text: `项目 ${nextId}` })
}
function onBeforeEnter(el) {
  el.style.opacity = 0
  el.style.transform = 'translateY(20px)'
}
function onEnter(el, done) {
  const delay = el.dataset.index * 100
  setTimeout(() => {
    el.style.transition = 'all 0.3s ease'
    el.style.opacity = 1
    el.style.transform = 'translateY(0)'
    el.addEventListener('transitionend', done)
  }, delay)
}
</script>

<template>
  <button @click="addItem">添加项目</button>
  <TransitionGroup
    tag="ul"
    @before-enter="onBeforeEnter"
    @enter="onEnter"
  >
    <li v-for="(item, index) in items" :key="item.id" :data-index="index">
      {{ item.text }}
    </li>
  </TransitionGroup>
</template>
```

## 二、具体用法

### 2.1 使用 data 属性传递索引

通过 `:data-index` 将元素在列表中的索引传递给 JavaScript 钩子，用于计算延迟。

### 2.2 动态延迟计算

```js
function onEnter(el, done) {
  const delay = el.dataset.index * 100 // 每个元素延迟 100ms
  setTimeout(() => {
    // 执行动画后调用 done
    done()
  }, delay)
}
```

## 四、交错离开动画

```vue
<script setup>
import { ref } from 'vue'
const items = ref([
  { id: 1, text: '项目 1' },
  { id: 2, text: '项目 2' },
  { id: 3, text: '项目 3' }
])

function onBeforeLeave(el) {
  // 保持当前位置
}

function onLeave(el, done) {
  const delay = el.dataset.index * 80
  setTimeout(() => {
    el.style.transition = 'all 0.3s ease'
    el.style.opacity = 0
    el.style.transform = 'translateX(50px)'
    el.addEventListener('transitionend', done)
  }, delay)
}

function removeLast() {
  items.value.pop()
}
</script>

<template>
  <button @click="removeLast">删除最后一项</button>
  <TransitionGroup
    tag="ul"
    @before-leave="onBeforeLeave"
    @leave="onLeave"
  >
    <li v-for="(item, index) in items" :key="item.id" :data-index="index">
      {{ item.text }}
    </li>
  </TransitionGroup>
</template>
```

## 五、结合 GSAP 实现更流畅的交错动画

```vue
<script setup>
import { ref } from 'vue'
import gsap from 'gsap'

const items = ref([])
let nextId = 0

function addItem() {
  items.value.push({ id: nextId++, text: `项目 ${nextId}` })
}

function onEnter(el, done) {
  gsap.fromTo(el,
    { opacity: 0, y: 20, scale: 0.95 },
    {
      opacity: 1, y: 0, scale: 1,
      duration: 0.4,
      delay: el.dataset.index * 0.08,
      ease: 'power2.out',
      onComplete: done
    }
  )
}
</script>

<template>
  <button @click="addItem">添加</button>
  <TransitionGroup tag="div" @enter="onEnter">
    <div v-for="(item, index) in items" :key="item.id" :data-index="index">
      {{ item.text }}
    </div>
  </TransitionGroup>
</template>
```

## 三、注意事项与常见陷阱

- 必须调用 `done` 回调，否则 Vue 不知道动画何时结束
- 使用 `data-*` 属性传递数据给钩子函数
- 延迟时间不宜过长，否则列表较长时最后一个元素等待过久
- GSAP 的 `delay` 参数比 `setTimeout` 更精确，推荐使用
- 交错离开时要注意列表索引的变化（从后往前删除更安全）
