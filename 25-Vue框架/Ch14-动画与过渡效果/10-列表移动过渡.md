# 列表移动过渡

## 一、概念说明

`<TransitionGroup>` 除了处理进入/离开动画，还能为列表中**已有元素的位置变化**添加平滑的移动动画。Vue 使用 FLIP 技巧（First, Last, Invert, Play）实现高效的位移动画。

```vue
<script setup>
import { ref } from 'vue'
const items = ref([1, 2, 3, 4, 5])

function shuffle() {
  items.value.sort(() => Math.random() - 0.5)
}
</script>

<template>
  <button @click="shuffle">随机排序</button>
  <TransitionGroup name="flip" tag="div" class="grid">
    <div v-for="item in items" :key="item" class="card">{{ item }}</div>
  </TransitionGroup>
</template>

<style>
.grid { display: flex; flex-wrap: wrap; gap: 10px; }
.card {
  width: 60px; height: 60px;
  display: flex; align-items: center; justify-content: center;
  background: #42b883; color: white; border-radius: 8px;
}
.flip-move { transition: transform 0.5s ease; }
</style>
```

## 二、具体用法

### 2.1 v-move 类名

Vue 自动为位置变化的元素添加 `{name}-move` 类，定义 `transition: transform` 即可实现平滑移动。

```css
.flip-move {
  transition: transform 0.5s ease;
}
```

### 2.2 FLIP 动画原理

1. **First**：记录元素移动前的位置
2. **Last**：获取元素移动后的最终位置
3. **Invert**：计算差值，通过 `transform` 将元素移回原位
4. **Play**：移除 `transform`，触发 CSS 过渡回到正确位置

### 2.3 排序 + 增删结合

```vue
<TransitionGroup name="list" tag="ul">
  <li v-for="item in items" :key="item.id">
    {{ item.text }}
  </li>
</TransitionGroup>
```

## 四、完整排序动画示例

```vue
<script setup>
import { ref } from 'vue'

const items = ref([
  { id: 1, name: 'Alice', score: 85 },
  { id: 2, name: 'Bob', score: 92 },
  { id: 3, name: 'Charlie', score: 78 },
  { id: 4, name: 'Diana', score: 95 }
])

const sortBy = ref('name')

const sortedItems = computed(() => {
  return [...items.value].sort((a, b) => {
    if (sortBy.value === 'name') return a.name.localeCompare(b.name)
    return b.score - a.score
  })
})
</script>

<template>
  <button @click="sortBy = 'name'">按姓名排序</button>
  <button @click="sortBy = 'score'">按分数排序</button>

  <TransitionGroup name="rank" tag="ul" class="rank-list">
    <li v-for="item in sortedItems" :key="item.id" class="rank-item">
      <span>{{ item.name }}</span>
      <span>{{ item.score }}分</span>
    </li>
  </TransitionGroup>
</template>

<style>
.rank-list { list-style: none; padding: 0; }
.rank-item {
  display: flex;
  justify-content: space-between;
  padding: 12px;
  margin: 4px 0;
  background: #f0f9f4;
  border-radius: 6px;
}
.rank-move {
  transition: transform 0.5s cubic-bezier(0.22, 1, 0.36, 1);
}
</style>
```

## 五、解决离开元素的跳动问题

```css
/* 方案1：离开元素使用 absolute 定位 */
.list-leave-active {
  position: absolute;
}

/* 方案2：保持占位直到离开动画完成 */
.list-leave-active {
  transition: all 0.4s ease;
}

/* 方案3：同时添加移动和离开过渡 */
.list-move, .list-enter-active, .list-leave-active {
  transition: all 0.4s ease;
}
```

## 三、注意事项与常见陷阱

- 移动过渡依赖 `transform` 属性，必须在 `*-move` 类中定义 `transition: transform`
- 离开中的元素不参与移动计算，可能产生跳动。可通过 `position: absolute` 解决
- 动态高度列表的移动效果可能不如固定高度的好
- 离开动画期间其他元素的移动可能导致视觉跳动，`*-move` 类可以平滑过渡
- FLIP 技巧在大量元素时性能很好，因为只操作 `transform`
