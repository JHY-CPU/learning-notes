# 动画与过渡概述

## 一、概念说明

Vue 提供了强大的过渡（Transition）系统，用于在以下场景中为元素添加动画效果：

- **条件渲染**：`v-if` / `v-else` / `v-else-if`
- **条件展示**：`v-show`
- **动态组件**：`<component :is="...">`
- **路由切换**：Vue Router 的路由视图

Vue 的过渡系统支持三种实现方式：**CSS 过渡**、**CSS 动画**和 **JavaScript 钩子**，它们可以单独使用也可以组合使用。

```vue
<script setup>
import { ref } from 'vue'
const show = ref(true)
</script>

<template>
  <button @click="show = !show">切换</button>
  <Transition>
    <p v-if="show">Hello Vue 动画!</p>
  </Transition>
</template>

<style>
.v-enter-active, .v-leave-active {
  transition: opacity 0.5s ease;
}
.v-enter-from, .v-leave-to {
  opacity: 0;
}
</style>
```

## 二、具体用法

### 2.1 CSS 过渡（Transition）

使用 CSS `transition` 属性定义属性变化时的过渡效果，如 `opacity`、`transform`、`height` 等。

### 2.2 CSS 动画（Animation）

使用 CSS `@keyframes` 定义关键帧动画，适合需要多步骤的复杂动画序列。

### 2.3 JavaScript 钩子

通过 `@before-enter`、`@enter`、`@leave` 等事件钩子，使用 JavaScript（如 GSAP、anime.js）控制动画。

## 三、注意事项与常见陷阱

- `<Transition>` 只能包裹**单个**根元素，多元素请使用 `<TransitionGroup>`
- 动画元素必须有 `v-if` 或 `v-show` 控制才能触发过渡
- 使用 `mode="out-in"` 可以让离开动画先完成再开始进入动画
- 生产环境应优先使用 CSS 过渡，性能优于 JavaScript 动画

## 四、CSS 过渡的六个类名

```
v-enter-from      → 进入开始状态
v-enter-active    → 进入过程中
v-enter-to        → 进入结束状态
v-leave-from      → 离开开始状态
v-leave-active    → 离开过程中
v-leave-to        → 离开结束状态
```

设置 `name="fade"` 后：`fade-enter-from`、`fade-enter-active` 等。

## 五、选择动画实现方式

| 方式 | 性能 | 复杂度 | 适用场景 |
|------|------|--------|----------|
| CSS transition | 最优 | 低 | 简单过渡（淡入淡出、滑动）|
| CSS animation | 优 | 中 | 关键帧动画 |
| JS hooks | 略低 | 高 | 复杂动画、第三方库 |
| Web Animations API | 优 | 中 | 程序化动画 |

## 六、动画性能优化

```css
/* ✅ 使用 transform 和 opacity（GPU 加速）*/
.v-enter-active {
  transition: transform 0.3s, opacity 0.3s;
}

/* ❌ 避免触发 layout 的属性 */
.v-enter-active {
  transition: width 0.3s, height 0.3s, top 0.3s; /* 性能差 */
}
```

```vue
<!-- 使用 will-change 提示浏览器优化 -->
<style>
.animated {
  will-change: transform, opacity;
}
</style>
```
