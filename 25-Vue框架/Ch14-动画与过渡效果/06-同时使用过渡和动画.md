# 同时使用过渡和动画

## 一、概念说明

在某些场景中，CSS `transition` 和 `animation` 会同时作用于同一个元素。此时 Vue 需要判断动画何时结束，`type` 属性用于指定以哪个持续时间为准。

```vue
<script setup>
import { ref } from 'vue'
const show = ref(true)
</script>

<template>
  <button @click="show = !show">切换</button>
  <Transition name="combined" type="animation">
    <div v-if="show" class="box">同时过渡和动画</div>
  </Transition>
</template>

<style>
.combined-enter-active {
  animation: bounceIn 0.5s;
  transition: background-color 0.8s;
}
.combined-leave-active {
  animation: bounceOut 0.5s;
  transition: background-color 0.8s;
}
@keyframes bounceIn { 0% { transform: scale(0); } 100% { transform: scale(1); } }
@keyframes bounceOut { 0% { transform: scale(1); } 100% { transform: scale(0); } }
</style>
```

## 二、具体用法

### 2.1 type 属性

- `type="animation"`：以 `animationend` 事件为准，忽略 `transitionend`
- `type="transition"`：以 `transitionend` 事件为准，忽略 `animationend`

```vue
<Transition name="combined" type="animation">
  <div v-if="show">动画优先</div>
</Transition>
```

### 2.2 为什么需要 type

当 `transition: all 0.8s` 和 `animation: bounceIn 0.5s` 同时存在时，浏览器会触发两个结束事件。Vue 默认等待**两者都结束**才移除过渡类。`type` 让 Vue 只监听其中一个。

## 四、实际场景：入场动画 + 持续过渡

```vue
<script setup>
import { ref } from 'vue'
const show = ref(true)
</script>

<template>
  <Transition name="card" type="animation">
    <div v-if="show" class="card">
      <!-- 入场时有翻转动画，同时背景色有过渡 -->
      <h3>卡片标题</h3>
      <p>卡片内容</p>
    </div>
  </Transition>
</template>

<style>
.card-enter-active {
  animation: flipIn 0.5s ease;
  transition: background-color 0.8s ease, box-shadow 0.3s ease;
}
.card-leave-active {
  animation: flipOut 0.3s ease;
}
.card-enter-from { background-color: #f0f0f0; }
.card-enter-to { background-color: white; }

@keyframes flipIn {
  0% { transform: perspective(400px) rotateY(90deg); opacity: 0; }
  100% { transform: perspective(400px) rotateY(0); opacity: 1; }
}
@keyframes flipOut {
  0% { transform: scale(1); opacity: 1; }
  100% { transform: scale(0.8); opacity: 0; }
}
</style>
```

## 五、性能建议

| 场景 | 推荐方案 |
|------|---------|
| 简单位移/淡入 | 只用 `transition`，不用 `animation` |
| 复杂多阶段动画 | 只用 `animation`，避免同时用 `transition` |
| 入场动画 + 持续状态过渡 | 两者同时使用，指定 `type="animation"` |
| 需要精确时间控制 | 使用 `:duration` 属性配合 `type` |

## 三、注意事项与常见陷阱

- 不指定 `type` 时，Vue 等待过渡和动画**全部**完成
- 如果 `transition` 和 `animation` 时长差距很大，会出现"动画已结束但类未移除"的延迟
- 建议要么只用一种，要么显式指定 `type`
- 两者时长尽量接近，避免视觉上的不协调
- `type="animation"` 通常更安全，因为动画时间更精确
