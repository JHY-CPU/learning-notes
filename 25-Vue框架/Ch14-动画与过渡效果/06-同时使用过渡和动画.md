# 同时使用过渡和动画

## 一、概念说明

在某些场景中，CSS `transition` 和 `animation` 会同时作用于同一个元素。此时 Vue 需要判断动画何时结束，`type` 属性用于指定以哪个持续时间为准。

```vue
<script setup>
import { ref } from 'vue'
const show = ref(true)
</script>

<template>
  <button @click="show = !show">切换</button>
  <Transition name="combined" type="animation">
    <div v-if="show" class="box">同时过渡和动画</div>
  </Transition>
</template>

<style>
.combined-enter-active {
  animation: bounceIn 0.5s;
  transition: background-color 0.8s;
}
.combined-leave-active {
  animation: bounceOut 0.5s;
  transition: background-color 0.8s;
}
@keyframes bounceIn { 0% { transform: scale(0); } 100% { transform: scale(1); } }
@keyframes bounceOut { 0% { transform: scale(1); } 100% { transform: scale(0); } }
</style>
```

## 二、具体用法

### 2.1 type 属性

- `type="animation"`：以 `animationend` 事件为准，忽略 `transitionend`
- `type="transition"`：以 `transitionend` 事件为准，忽略 `animationend`

```vue
<Transition name="combined" type="animation">
  <div v-if="show">动画优先</div>
</Transition>
```

### 2.2 为什么需要 type

当 `transition: all 0.8s` 和 `animation: bounceIn 0.5s` 同时存在时，浏览器会触发两个结束事件。Vue 默认等待**两者都结束**才移除过渡类。`type` 让 Vue 只监听其中一个。

## 三、注意事项与常见陷阱

- 不指定 `type` 时，Vue 等待过渡和动画**全部**完成
- 如果 `transition` 和 `animation` 时长差距很大，会出现"动画已结束但类未移除"的延迟
- 建议要么只用一种，要么显式指定 `type`
