# 初始渲染动画

## 一、概念说明

默认情况下，`<Transition>` 只在元素的条件状态变化时触发动画。如果希望元素在**首次渲染**时也播放动画，需要添加 `appear` 属性。

```vue
<script setup>
import { ref } from 'vue'
const show = ref(true)
</script>

<template>
  <Transition name="fade" appear>
    <div v-if="show">首次渲染时也会淡入</div>
  </Transition>
</template>

<style>
.fade-enter-active { transition: opacity 0.5s ease; }
.fade-enter-from { opacity: 0; }
</style>
```

## 二、具体用法

### 2.1 appear 属性

```vue
<!-- 基本用法 -->
<Transition name="slide" appear>
  <div v-if="visible">页面加载时就会滑入</div>
</Transition>
```

### 2.2 appear 配合 mode

```vue
<!-- 首次渲染动画 + 离开/进入模式 -->
<Transition name="fade" mode="out-in" appear>
  <component :is="currentView" />
</Transition>
```

### 2.3 appear 使用独立类名

初始渲染时使用与进入相同的类名体系：`*-enter-from`、`*-enter-active`、`*-enter-to`。

## 四、页面加载入场动画

```vue
<!-- 首页加载时的元素依次入场 -->
<template>
  <TransitionGroup name="stagger" appear tag="div" class="hero">
    <h1 key="title">欢迎使用</h1>
    <p key="subtitle">这是副标题</p>
    <button key="cta" @click="go">开始使用</button>
  </TransitionGroup>
</template>

<style>
.stagger-enter-active {
  transition: all 0.6s ease;
}
.stagger-enter-from {
  opacity: 0;
  transform: translateY(30px);
}
/* 为不同元素设置不同延迟 */
.stagger-enter-active:nth-child(1) { transition-delay: 0s; }
.stagger-enter-active:nth-child(2) { transition-delay: 0.15s; }
.stagger-enter-active:nth-child(3) { transition-delay: 0.3s; }
</style>
```

## 五、appear 配合 JS 钩子

```vue
<template>
  <Transition
    appear
    @before-appear="onBeforeAppear"
    @appear="onAppear"
    @after-appear="onAfterAppear"
  >
    <div v-if="show">内容</div>
  </Transition>
</template>

<script setup>
import gsap from 'gsap'

function onBeforeAppear(el) {
  el.style.opacity = 0
}

function onAppear(el, done) {
  gsap.to(el, {
    opacity: 1,
    y: 0,
    duration: 0.8,
    ease: 'power2.out',
    onComplete: done
  })
}

function onAfterAppear(el) {
  console.log('初始动画完成')
}
</script>
```

> `appear` 的 JS 钩子命名格式为 `@before-appear`、`@appear`、`@after-appear`，与普通钩子类似但加了 `appear` 前缀。

## 三、注意事项与常见陷阱

- `appear` 只在**首次渲染**时生效，后续的 `v-if` 切换使用正常的进入/离开类名
- 如果组件一开始就满足 `v-if="true"`，不加 `appear` 则无任何动画
- `appear` 对 `v-show` 同样有效
- `<TransitionGroup>` 的 `appear` 会让列表中所有元素同时入场
- `appear` 也可配合 `mode="out-in"` 使用
