# 动画最佳实践

## 一、概念说明

动画是提升用户体验的重要手段，但过度或不当使用反而会让界面显得杂乱。以下是 Vue 项目中动画的核心原则和最佳实践。

```vue
<script setup>
import { ref } from 'vue'
const loading = ref(false)

async function fetchData() {
  loading.value = true
  await new Promise(r => setTimeout(r, 1000))
  loading.value = false
}
</script>

<template>
  <button @click="fetchData" :disabled="loading">
    <Transition mode="out-in">
      <span v-if="loading" key="load">加载中...</span>
      <span v-else key="ready">获取数据</span>
    </Transition>
  </button>
</template>
```

## 二、具体用法

### 2.1 核心原则

| 原则 | 说明 |
|------|------|
| **适度使用** | 只在有意义的地方使用动画，避免泛滥 |
| **一致性** | 全站动画风格、时长、缓动函数保持统一 |
| **快速响应** | 交互反馈动画 100ms 以内，过渡动画 200-500ms |
| **可跳过** | 尊重 `prefers-reduced-motion` 媒体查询 |

### 2.2 无障碍支持

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

### 2.3 统一动画变量

```css
:root {
  --transition-fast: 0.15s ease;
  --transition-normal: 0.3s ease;
  --transition-slow: 0.5s ease;
  --ease-bounce: cubic-bezier(0.34, 1.56, 0.64, 1);
}
```

### 2.4 动画清单

- 页面切换使用统一的路由过渡
- 按钮/链接有 hover 和 active 反馈
- 加载状态有占位动画（骨架屏、spinner）
- 列表操作有增删移动过渡
- 模态框/抽屉有进入退出动画

## 四、动画设计决策表

| 场景 | 推荐方案 | 时长 | 缓动函数 |
|------|---------|------|---------|
| 页面切换 | Transition + CSS | 300ms | ease-in-out |
| 模态框弹出 | CSS animation | 200ms | ease-out |
| 列表增删 | TransitionGroup | 300-400ms | ease |
| 按钮反馈 | CSS transition | 100-150ms | ease |
| 加载状态 | CSS animation | 1s循环 | linear |
| 数值变化 | JS rAF | 500ms | ease-out |
| 表单错误 | CSS animation | 300ms | ease |
| Toast 通知 | CSS animation | 200ms | ease-out |

## 五、Vue 动画代码组织

```
src/
  assets/
    animations/
      transitions.css     # 全局过渡类
      keyframes.css       # 全局关键帧
      variables.css       # CSS 变量（时长、缓动）
  components/
    MyComponent/
      MyComponent.vue
      MyComponent.css     # 组件特定动画
  composables/
    useAnimation.js       # 复用的动画逻辑
```

```css
/* animations/variables.css */
:root {
  --duration-fast: 150ms;
  --duration-normal: 300ms;
  --duration-slow: 500ms;
  --ease-default: cubic-bezier(0.4, 0, 0.2, 1);
  --ease-in: cubic-bezier(0.4, 0, 1, 1);
  --ease-out: cubic-bezier(0, 0, 0.2, 1);
  --ease-bounce: cubic-bezier(0.34, 1.56, 0.64, 1);
}
```

## 六、调试动画

```js
// 减慢所有动画便于调试
document.getAnimations().forEach(anim => {
  anim.playbackRate = 0.1
})

// 恢复正常速度
document.getAnimations().forEach(anim => {
  anim.playbackRate = 1
})

// 查看所有动画
console.log(document.getAnimations())
```

## 三、注意事项与常见陷阱

- 不要用动画替代好的 UI 设计，动画只是锦上添花
- 避免动画时长超过 500ms，用户会感到等待
- 性能敏感场景（大列表、频繁更新）优先用 CSS 动画
- 动画结束后移除 `will-change` 等优化属性，释放资源
- `prefers-reduced-motion` 不是可选的，而是无障碍必需的
- 使用 CSS 变量统一管理动画参数，便于主题切换和维护
