# 组件注册方式

## 一、概念说明

Vue 组件有两种注册方式：**局部注册**（在使用组件的文件中导入）和**全局注册**（在应用入口注册，所有组件可用）。`<script setup>` 中导入的组件自动局部注册。

```vue
<script setup>
// 局部注册：导入即注册
import MyButton from './MyButton.vue'
</script>

<template>
  <MyButton /> <!-- 可直接使用 -->
</template>
```

## 二、具体用法

### 2.1 局部注册

```vue
<script setup>
// 导入后自动注册，只在当前组件可用
import ComponentA from './ComponentA.vue'
import ComponentB from './ComponentB.vue'
</script>

<template>
  <ComponentA />
  <ComponentB />
</template>
```

### 2.2 全局注册

```js
// main.js
import { createApp } from 'vue'
import App from './App.vue'
import MyButton from './components/MyButton.vue'
import Icon from './components/Icon.vue'

const app = createApp(App)

// 全局注册：所有组件中可用
app.component('MyButton', MyButton)
app.component('Icon', Icon)

app.mount('#app')
```

### 2.3 批量全局注册

```js
// main.js
import { createApp } from 'vue'
import App from './App.vue'

const app = createApp(App)

// 自动导入 components/ 下所有组件
const modules = import.meta.glob('./components/*.vue', { eager: true })
for (const [path, module] of Object.entries(modules)) {
  const name = path.match(/\/(\w+)\.vue$/)[1]
  app.component(name, module.default)
}

app.mount('#app')
```

## 三、注意事项与常见陷阱

- 推荐使用局部注册，Tree-shaking 更友好
- 全局注册的组件无法被 Tree-shaking
- 全局组件应该限于真正需要全局使用的组件（如按钮、图标）
- 全局注册的组件名应该有统一前缀（如 `App`、`Base`）
- `<script setup>` 中 import 的组件不需要额外注册步骤
