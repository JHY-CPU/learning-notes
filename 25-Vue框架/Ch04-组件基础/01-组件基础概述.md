# 组件基础概述

## 一、概念说明

组件是 Vue 应用的基本构建块。每个组件封装了自己的模板、逻辑和样式，形成独立的、可复用的 UI 单元。组件化开发将复杂界面拆分为小的、可管理的部分。

Vue 3 的组件基于**单文件组件**（SFC，`.vue` 文件），使用 `<script setup>` 语法编写，是 Composition API 的最佳实践。

```vue
<script setup>
import { ref } from 'vue'

const count = ref(0)
</script>

<template>
  <div class="counter">
    <span>计数: {{ count }}</span>
    <button @click="count++">+1</button>
  </div>
</template>

<style scoped>
.counter {
  display: flex;
  gap: 10px;
  align-items: center;
}
</style>
```

## 二、具体用法

### 2.1 组件化思想

```
应用 (App)
├── 导航栏 (Navbar)
│   ├── Logo
│   └── 导航链接 (NavLink)
├── 侧边栏 (Sidebar)
└── 主内容 (MainContent)
    ├── 文章列表 (ArticleList)
    │   └── 文章卡片 (ArticleCard)
    └── 评论区 (Comments)
        └── 评论项 (CommentItem)
```

### 2.2 最小组件示例

```vue
<!-- ButtonCounter.vue -->
<script setup>
import { ref } from 'vue'
const count = ref(0)
</script>

<template>
  <button @click="count++">点击了 {{ count }} 次</button>
</template>
```

### 2.3 组件的三个核心

| 核心 | 说明 |
|------|------|
| **Props** | 父组件向子组件传递数据 |
| **Events** | 子组件向父组件发送消息 |
| **Slots** | 父组件向子组件传递内容 |

### 2.4 组件设计原则

- **单一职责**：每个组件只做一件事
- **高内聚**：相关的逻辑和模板放在同一组件中
- **低耦合**：组件间通过 props/events 通信，不直接访问内部状态
- **可测试**：组件应该可以独立测试

## 三、常见用例

### 3.1 拆分判断标准

| 情况 | 建议 |
|------|------|
| 组件超过 300 行 | 考虑拆分 |
| 同一功能多处使用 | 提取为组件 |
| 逻辑相对独立 | 提取为组件 |
| 需要不同的数据实例 | 提取为组件 |

## 四、注意事项与常见陷阱

- 每个组件应该是独立的，不依赖外部状态
- 组件名使用 PascalCase（如 `MyComponent.vue`）
- 组件应该有单一职责
- 避免组件过于庞大（超过 300 行考虑拆分）
- 复用性高的组件应尽量无状态
- 过度拆分也会增加复杂度，找到合适的粒度
