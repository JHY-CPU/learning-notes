# 组件调试技巧

## 一、概念说明

Vue 组件调试主要依赖 **Vue DevTools** 和浏览器开发者工具。DevTools 提供组件树查看、Props/State 检查、事件追踪等专用功能，是调试 Vue 应用的首选工具。

```vue
<script setup>
import { ref, computed, watch } from 'vue'

const items = ref([
  { id: 1, name: '项目 A', done: false },
  { id: 2, name: '项目 B', done: true }
])

const doneCount = computed(() =>
  items.value.filter(i => i.done).length
)

// 在 DevTools 中可以实时查看 items 和 doneCount
watch(items, (newVal) => {
  console.log('items 变化:', JSON.stringify(newVal))
}, { deep: true })
</script>

<template>
  <div>
    <p>已完成: {{ doneCount }} / {{ items.length }}</p>
    <div v-for="item in items" :key="item.id">
      <input type="checkbox" v-model="item.done" />
      <span>{{ item.name }}</span>
    </div>
  </div>
</template>
```

## 二、具体用法

### 2.1 Vue DevTools 面板

```
打开 DevTools → Vue 面板：

1. Components 面板:
   - 查看完整组件树
   - 查看每个组件的 Props、Data、Computed
   - 实时编辑 Props 和 Data
   - 定位组件在源码中的位置

2. Timeline 面板:
   - 查看事件触发时间线
   - 查看性能追踪

3. Pinia/Vuex 面板:
   - 查看状态管理数据
   - 时间旅行调试
```

### 2.2 console 调试

```vue
<script setup>
import { ref, watch, toRaw } from 'vue'

const data = ref({ nested: { value: 0 } })

// 打印原始对象（避免 Proxy 干扰）
function logRaw() {
  console.log(JSON.stringify(toRaw(data.value), null, 2))
}

// 监听特定变化
watch(() => data.value.nested.value, (newVal, oldVal) => {
  console.log(`nested.value: ${oldVal} → ${newVal}`)
})

// 调试 computed
const computedValue = computed(() => {
  const result = expensiveCalculation()
  console.log('computed 重新计算:', result)
  return result
})
</script>
```

### 2.3 模板调试

```vue
<template>

```

## 三、注意事项与常见陷阱

- Vue DevTools 在生产环境默认禁用
- 打印 Proxy 对象用 `toRaw()` 或 `JSON.stringify`
- 使用 `watch` 的 `onTrack/onTrigger` 调试依赖关系
- `<script setup>` 中的变量名可能被编译器重命名
- DevTools 无法连接时刷新页面或检查扩展版本
- 在生产环境使用 `app.config.performance = true` 进行性能追踪
