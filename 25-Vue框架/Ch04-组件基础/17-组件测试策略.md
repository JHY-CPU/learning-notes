# 组件测试策略

## 一、概念说明

Vue 组件测试分为三层：**单元测试**（测试组件逻辑）、**组件测试**（测试组件渲染和交互）、**端到端测试**（测试完整用户流程）。推荐使用 Vitest + Vue Test Utils。

```vue
<script setup>
import { ref, computed } from 'vue'

const count = ref(0)
const doubled = computed(() => count.value * 2)
const increment = () => count.value++
</script>

<template>
  <div>
    <span data-testid="count">{{ count }}</span>
    <span data-testid="doubled">{{ doubled }}</span>
    <button @click="increment">+1</button>
  </div>
</template>
```

```js
// Counter.test.js
import { mount } from '@vue/test-utils'
import Counter from './Counter.vue'

describe('Counter', () => {
  it('初始值为 0', () => {
    const wrapper = mount(Counter)
    expect(wrapper.find('[data-testid="count"]').text()).toBe('0')
  })

  it('点击后计数增加', async () => {
    const wrapper = mount(Counter)
    await wrapper.find('button').trigger('click')
    expect(wrapper.find('[data-testid="count"]').text()).toBe('1')
  })

  it('doubled 计算正确', async () => {
    const wrapper = mount(Counter)
    await wrapper.find('button').trigger('click')
    expect(wrapper.find('[data-testid="doubled"]').text()).toBe('2')
  })
})
```

## 二、具体用法

### 2.1 测试 Props

```js
it('接收并显示 props', () => {
  const wrapper = mount(UserCard, {
    props: { name: '张三', age: 25 }
  })
  expect(wrapper.text()).toContain('张三')
  expect(wrapper.text()).toContain('25')
})
```

### 2.2 测试事件

```js
it('触发 emit 事件', async () => {
  const wrapper = mount(MyButton)
  await wrapper.find('button').trigger('click')
  expect(wrapper.emitted('click')).toHaveLength(1)
})
```

### 2.3 测试 v-model

```js
it('v-model 双向绑定', async () => {
  const wrapper = mount(CustomInput, {
    props: { modelValue: '初始值' }
  })
  await wrapper.find('input').setValue('新值')
  expect(wrapper.emitted('update:modelValue')[0]).toEqual(['新值'])
})
```

## 三、注意事项与常见陷阱

- 使用 `data-testid` 属性定位元素，避免依赖类名
- 使用 `async/await` 和 `nextTick` 等待 DOM 更新
- 测试应该独立，不依赖外部状态
- 优先测试用户行为而非实现细节
- 使用 Vitest 获得与 Vite 的无缝集成
