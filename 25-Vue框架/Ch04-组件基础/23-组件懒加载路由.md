# 组件懒加载与路由

## 一、概念说明

路由级懒加载在用户访问对应路由时才下载组件代码。这是优化单页应用加载性能的关键策略。Vue Router 原生支持动态导入实现懒加载。

```vue
<!-- router/index.js -->
<script setup>
// 路由配置中使用动态导入
</script>
```

```js
// router/index.js
import { createRouter, createWebHistory } from 'vue-router'

const routes = [
  {
    path: '/',
    name: 'Home',
    component: () => import('@/views/Home.vue') // 懒加载
  },
  {
    path: '/about',
    name: 'About',
    component: () => import('@/views/About.vue') // 懒加载
  },
  {
    path: '/dashboard',
    name: 'Dashboard',
    component: () => import('@/views/Dashboard.vue'),
    children: [
      {
        path: 'settings',
        component: () => import('@/views/Settings.vue')
      }
    ]
  }
]

const router = createRouter({
  history: createWebHistory(),
  routes
})

export default router
```

## 二、具体用法

### 2.1 带 loading 的懒加载

```js
import { defineAsyncComponent } from 'vue'

const Dashboard = defineAsyncComponent({
  loader: () => import('@/views/Dashboard.vue'),
  loadingComponent: LoadingSpinner,
  delay: 200,
  timeout: 10000
})
```

### 2.2 命名 chunk

```js
// 使用魔法注释命名 chunk，便于调试
const Home = () => import(/* webpackChunkName: "home" */ '@/views/Home.vue')
const About = () => import(/* webpackChunkName: "about" */ '@/views/About.vue')
const Dashboard = () => import(/* webpackChunkName: "dashboard" */ '@/views/Dashboard.vue')
```

### 2.3 预加载

```js
// 预加载用户可能访问的路由
function preloadRoute(routePath) {
  const route = routes.find(r => r.path === routePath)
  if (route && route.component) {
    route.component() // 触发预加载
  }
}

// 用户 hover 导航链接时预加载
function onNavHover(path) {
  preloadRoute(path)
}
```

### 2.4 骨架屏配合

```vue
<script setup>
import { ref, defineAsyncComponent } from 'vue'

const PageComponent = defineAsyncComponent({
  loader: () => import('./HeavyPage.vue'),
  loadingComponent: {
    template: '<div class="skeleton">加载中...</div>'
  }
})
</script>

<template>
  <Suspense>
    <PageComponent />
    <template #fallback>
      <div class="skeleton">加载中...</div>
    </template>
  </Suspense>
</template>
```

## 三、注意事项与常见陷阱

- 每个懒加载组件都会生成独立的 chunk 文件
- 懒加载可能导致加载时的白屏，需要 loading 状态
- 过度拆分会增加 HTTP 请求数
- 首屏组件不建议懒加载
- 使用命名 chunk 便于在 DevTools 中识别
