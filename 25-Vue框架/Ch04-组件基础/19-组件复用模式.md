# 组件复用模式

## 一、概念说明

Vue 3 推荐的组件复用方式是**组合优于继承**。通过组合式函数（Composables）、插槽（Slots）和组件组合实现代码复用，而非 mixins 或 extends。

```vue
<script setup>
// 组合式函数: 复用逻辑
import { useMouse } from '@/composables/useMouse'
import { useFetch } from '@/composables/useFetch'

const { x, y } = useMouse()
const { data, loading } = useFetch('/api/users')
</script>

<template>
  <p>鼠标位置: {{ x }}, {{ y }}</p>
  <p v-if="loading">加载中...</p>
  <ul v-else>
    <li v-for="user in data" :key="user.id">{{ user.name }}</li>
  </ul>
</template>
```

## 二、具体用法

### 2.1 组合式函数

```js
// composables/useCounter.js
import { ref, computed } from 'vue'

export function useCounter(initialValue = 0) {
  const count = ref(initialValue)
  const doubled = computed(() => count.value * 2)
  const increment = () => count.value++
  const decrement = () => count.value--
  const reset = () => { count.value = initialValue }

  return { count, doubled, increment, decrement, reset }
}
```

### 2.2 插槽复用

```vue
<!-- 复用布局: Card.vue -->
<script setup>
defineProps({ title: String })
</script>

<template>
  <div class="card">
    <header v-if="$slots.header">
      <slot name="header" />
    </header>
    <main>
      <slot /> <!-- 默认插槽 -->
    </main>
    <footer v-if="$slots.footer">
      <slot name="footer" />
    </footer>
  </div>
</template>
```

### 2.3 为什么不推荐 mixins

```js
// 差: mixins 导入的来源不清晰
// 好: 组合式函数的返回值清晰可见
const { count, increment } = useCounter()
```

| 复用方式 | 推荐程度 | 说明 |
|---------|---------|------|
| 组合式函数 | 最推荐 | 来源清晰，类型友好 |
| 插槽 | 推荐 | UI 复用 |
| 组件组合 | 推荐 | 布局复用 |
| mixins | 不推荐 | 来源不明，命名冲突 |
| extends | 不推荐 | 继承关系复杂 |

## 三、注意事项与常见陷阱

- 组合式函数名以 `use` 开头
- 每次调用组合式函数应该创建独立的实例
- 不要用 mixins（逻辑来源不清晰，容易命名冲突）
- 组合式函数中避免使用 `getCurrentInstance`
- 保持组合式函数的单一职责
