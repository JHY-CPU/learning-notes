# 组件设计原则

## 一、概念说明

好的组件设计遵循**单一职责**、**接口最小化**、**可组合性**等原则。组件应该做一件事并做好，暴露简洁的 props/events/slots 接口，通过组合实现复杂功能。

```vue
<!-- 好的设计: 单一职责 -->
<!-- SearchInput.vue: 只负责搜索输入 -->
<script setup>
const emit = defineEmits(['search', 'clear'])

const props = defineProps({
  placeholder: { type: String, default: '搜索...' },
  modelValue: { type: String, default: '' }
})
</script>

<template>
  <div class="search-input">
    <input
      :value="modelValue"
      :placeholder="placeholder"
      @input="$emit('search', $event.target.value)"
    />
    <button v-if="modelValue" @click="$emit('clear')">清除</button>
  </div>
</template>
```

## 二、具体用法

### 2.1 单一职责

```vue
<!-- 差: 一个组件做太多事情 -->
<!-- UserManagement.vue (500行) -->
<!-- 包含: 列表、搜索、编辑、删除、权限... -->

<!-- 好: 拆分为多个小组件 -->
<!-- UserList.vue       - 用户列表展示 -->
<!-- UserSearch.vue     - 搜索功能 -->
<!-- UserForm.vue       - 用户表单 -->
<!-- UserCard.vue       - 用户卡片 -->
```

### 2.2 接口最小化

```vue
<!-- 好: 接口简洁 -->
<script setup>
defineProps({
  type: { type: String, default: 'button' },
  disabled: { type: Boolean, default: false },
  loading: { type: Boolean, default: false }
})
defineEmits(['click'])
</script>

<!-- 差: 接口过于复杂 -->
<script setup>
defineProps({
  type: String, variant: String, size: String,
  rounded: Boolean, outlined: Boolean, flat: Boolean,
  dense: Boolean, disableRipple: Boolean,
  // ... 20 个 props
})
</script>
```

### 2.3 可组合性

```vue
<!-- 通过插槽和组合实现灵活性 -->
<script setup>
defineProps({ title: String })
</script>

<template>
  <div class="card">
    <header v-if="$slots.header"><slot name="header" /></header>
    <main><slot /></main>
    <footer v-if="$slots.footer"><slot name="footer" /></footer>
  </div>
</template>
```

## 三、注意事项与常见陷阱

- 组件超过 300 行考虑拆分
- 优先组合而非继承
- props 应该有类型验证和默认值
- 组件应该在不同上下文中可复用
- 避免组件间直接调用方法，使用事件通信
- 保持组件的纯度（相同输入产生相同输出）
