# 动态组件

## 一、概念说明

Vue 的 `<component :is>` 允许动态切换组件。`:is` 绑定一个组件对象或组件名，实现条件性渲染不同组件。配合 `<keep-alive>` 可以缓存组件状态。

```vue
<script setup>
import { ref, shallowRef } from 'vue'
import TabHome from './TabHome.vue'
import TabAbout from './TabAbout.vue'
import TabContact from './TabContact.vue'

const tabs = { TabHome, TabAbout, TabContact }
const currentTab = shallowRef(TabHome)
</script>

<template>
  <div>
    <button @click="currentTab = TabHome">首页</button>
    <button @click="currentTab = TabAbout">关于</button>
    <button @click="currentTab = TabContact">联系</button>

    <!-- 动态组件 -->
    <component :is="currentTab" />
  </div>
</template>
```

## 二、具体用法

### 2.1 keep-alive 缓存

```vue
<template>
  <!-- 缓存组件状态，切换时不销毁 -->
  <keep-alive>
    <component :is="currentTab" />
  </keep-alive>
</template>
```

### 2.2 keep-alive 选项

```vue
<template>
  <keep-alive
    :include="['TabHome', 'TabAbout']"
    :exclude="['TabContact']"
    :max="10"
  >
    <component :is="currentTab" />
  </keep-alive>
</template>
```

### 2.3 字符串组件名

```vue
<script setup>
import { ref } from 'vue'

// 使用字符串引用全局注册的组件
const componentName = ref('MyComponent')
</script>

<template>
  <!-- 注意: 需要组件被全局注册 -->
  <component :is="componentName" />
</template>
```

## 三、注意事项与常见陷阱

- `shallowRef` 比 `ref` 更适合存储组件对象（避免深度响应式）
- `<keep-alive>` 要求组件有 `name` 选项或通过文件名推断
- 动态组件切换时会触发销毁和创建生命周期
- `<keep-alive>` 缓存的组件触发 `activated` / `deactivated` 而非 `mounted` / `unmounted`
- 不要在 `<keep-alive>` 中使用 v-if，改用 `include/exclude`
