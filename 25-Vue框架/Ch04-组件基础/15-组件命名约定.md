# 组件命名约定

## 一、概念说明

Vue 推荐组件使用 **PascalCase** 命名（如 `MyComponent.vue`），并在模板中也使用 PascalCase。多词命名避免与 HTML 元素冲突，提高可读性。

```vue
<!-- UserProfile.vue -->
<script setup>
// 文件名: UserProfile.vue（PascalCase）
defineOptions({
  name: 'UserProfile' // 组件名也用 PascalCase
})
</script>

<template>
  <!-- 模板中使用 PascalCase -->
  <div class="user-profile">
    <h3>用户信息</h3>
  </div>
</template>
```

## 二、具体用法

### 2.1 命名规则

| 位置 | 格式 | 示例 |
|------|------|------|
| 文件名 | PascalCase | `UserProfile.vue` |
| 组件名 | PascalCase | `UserProfile` |
| 模板标签 | PascalCase | `<UserProfile />` |
| 文件夹名 | kebab-case | `user-profile/` |

### 2.2 多词命名原因

```vue
<!-- 好: 多词命名，不会与 HTML 冲突 -->
<MyButton />
<SearchInput />
<UserProfile />

<!-- 差: 单词命名，可能与 HTML 冲突 -->
<button />    <!-- 冲突 -->
<input />     <!-- 冲突 -->
<table />     <!-- 冲突 -->
```

### 2.3 特殊组件命名

```vue
<!-- App: 应用根组件 -->
<!-- App.vue -->

<!-- 开头的组件用于不直接使用的抽象组件 -->
<!-- NoSearchResults.vue -->

<!-- 表单相关 -->
<!-- LoginForm.vue, RegistrationForm.vue -->

<!-- 布局组件 -->
<!-- MainLayout.vue, HeaderLayout.vue -->
```

## 三、注意事项与常见陷阱

- 始终使用多词命名，避免与 HTML 元素冲突
- 在 DOM 模板中必须使用 kebab-case（HTML 不区分大小写）
- 组件名应该描述组件的功能，而非实现细节
- 通用组件加前缀（如 `BaseButton`、`AppHeader`）
- 保持团队命名风格一致
