# 组件命名约定

## 一、概念说明

Vue 推荐组件使用 **PascalCase** 命名（如 `MyComponent.vue`），并在模板中也使用 PascalCase。多词命名避免与 HTML 元素冲突，提高可读性。

```vue
<!-- UserProfile.vue -->
<script setup>
defineOptions({
  name: 'UserProfile'
})
</script>

<template>
  <div class="user-profile">
    <h3>用户信息</h3>
  </div>
</template>
```

## 二、具体用法

### 2.1 命名规则

| 位置 | 格式 | 示例 |
|------|------|------|
| 文件名 | PascalCase | `UserProfile.vue` |
| 组件名 | PascalCase | `UserProfile` |
| 模板标签 | PascalCase | `<UserProfile />` |
| 文件夹名 | kebab-case | `user-profile/` |

### 2.2 多词命名原因

```vue
<!-- 好: 多词命名，不会与 HTML 冲突 -->
<MyButton />
<SearchInput />
<UserProfile />

<!-- 差: 单词命名，可能与 HTML 冲突 -->
<!-- <button />  冲突 -->
<!-- <input />   冲突 -->
<!-- <table />   冲突 -->
```

### 2.3 常见命名模式

```
基础组件:    BaseButton.vue, AppHeader.vue
页面组件:    HomePage.vue, UserProfile.vue
布局组件:    MainLayout.vue, AuthLayout.vue
业务组件:    OrderList.vue, PaymentForm.vue
工具组件:    LoadingSpinner.vue, ErrorBoundary.vue
```

### 2.4 组件名的推断

```vue
<script setup>
// <script setup> 中组件名由文件名推断
// MyComponent.vue → name: 'MyComponent'

// 如需自定义，使用 defineOptions（Vue 3.3+）
defineOptions({
  name: 'CustomName'
})
</script>
```

## 三、常见用例

### 3.1 项目目录命名

```
src/
├── components/
│   ├── base/           # 基础组件
│   │   ├── BaseButton.vue
│   │   └── BaseInput.vue
│   ├── layout/         # 布局组件
│   │   ├── AppHeader.vue
│   │   └── AppSidebar.vue
│   └── business/       # 业务组件
│       ├── OrderCard.vue
│       └── UserAvatar.vue
├── views/              # 页面组件
│   ├── HomeView.vue
│   └── AboutView.vue

    ├── useAuth.js
    └── useFetch.js
```

## 四、注意事项与常见陷阱

- 始终使用多词命名，避免与 HTML 元素冲突
- 在 DOM 模板中必须使用 kebab-case（HTML 不区分大小写）
- 组件名应该描述组件的功能，而非实现细节
- 通用组件加前缀（如 `BaseButton`、`AppHeader`）
- 保持团队命名风格一致
- 文件夹命名使用 kebab-case，文件名使用 PascalCase
- 组件名影响 `keep-alive` 的 `include`/`exclude` 匹配
