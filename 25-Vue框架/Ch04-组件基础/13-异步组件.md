# 异步组件

## 一、概念说明

异步组件允许延迟加载组件，只在需要时才从服务器下载。通过 `defineAsyncComponent` 实现，常用于路由懒加载和大型组件的按需加载，减小初始包体积。

```vue
<script setup>
import { defineAsyncComponent } from 'vue'

// 异步加载组件
const HeavyChart = defineAsyncComponent(() =>
  import('./HeavyChart.vue')
)

// 带加载和错误状态
const AsyncComponent = defineAsyncComponent({
  loader: () => import('./BigComponent.vue'),
  loadingComponent: LoadingSpinner,
  errorComponent: ErrorDisplay,
  delay: 200,
  timeout: 3000
})
</script>

<template>
  <Suspense>
    <template #default>
      <HeavyChart />
    </template>
    <template #fallback>
      <p>加载中...</p>
    </template>
  </Suspense>
</template>
```

## 二、具体用法

### 2.1 基本异步加载

```vue
<script setup>
import { defineAsyncComponent } from 'vue'

const AdminPanel = defineAsyncComponent(() =>
  import('./AdminPanel.vue')
)
</script>

<template>
  <AdminPanel v-if="showAdmin" />
</template>
```

### 2.2 完整配置

```vue
<script setup>
import { defineAsyncComponent } from 'vue'
import LoadingComponent from './LoadingComponent.vue'
import ErrorComponent from './ErrorComponent.vue'

const AsyncComp = defineAsyncComponent({
  loader: () => import('./MyComponent.vue'),

  // 加载中显示的组件
  loadingComponent: LoadingComponent,

  // 加载失败显示的组件
  errorComponent: ErrorComponent,

  // 延迟显示 loadingComponent（ms）
  delay: 200,

  // 超时时间（ms）
  timeout: 3000,

  // 失败重试
  onError(error, retry, fail) {
    if (error.message.includes('Network')) {
      retry() // 重试
    } else {
      fail() // 失败
    }
  }
})
</script>
```

### 2.3 路由懒加载

```js
// router/index.js
const routes = [
  {
    path: '/',
    component: () => import('@/views/Home.vue') // 异步加载
  },
  {
    path: '/about',
    component: () => import('@/views/About.vue')
  }
]
```

## 三、注意事项与常见陷阱

- 异步组件需要配合 `<Suspense>` 或 loading/error 组件使用
- 异步组件在加载完成前不渲染任何内容
- 网络慢时需要显示 loading 状态提升用户体验
- 路由懒加载是最常见的异步组件应用场景
- 不要对频繁使用的小组件使用异步加载
