# 模板语法概述

## 一、概念说明

Vue 使用基于 HTML 的模板语法，允许声明式地将 DOM 绑定到组件实例的数据。所有 Vue 模板都是合法的 HTML，能被遵循 HTML 规范的浏览器和解析器解析。

Vue 编译器将模板编译为虚拟 DOM 渲染函数，结合响应式系统实现高效更新。相比 JSX，模板语法更直观，对设计师和初学者更友好。

```vue
<script setup>
import { ref } from 'vue'

const title = ref('Vue 模板语法')
const visible = ref(true)
</script>

<template>
  <!-- 声式式: 描述"是什么"而非"怎么做" -->
  <h1>{{ title }}</h1>
  <p v-if="visible">内容可见</p>
</template>
```

## 二、具体用法

### 2.1 模板 vs JSX

```vue
<!-- 模板写法 -->
<template>
  <div class="card">
    <h2>{{ title }}</h2>
    <p>{{ description }}</p>
  </div>
</template>

<!-- JSX 写法（需要在 setup 中返回渲染函数） -->
<script setup>
import { h } from 'vue'

function render() {
  return h('div', { class: 'card' }, [
    h('h2', title.value),
    h('p', description.value)
  ])
}
</script>
```

### 2.2 模板中的 JavaScript

```vue
<template>
  <!-- 模板内可以使用 JavaScript 表达式 -->
  <p>{{ message.split('').reverse().join('') }}</p>
  <p>{{ number > 0 ? '正数' : '非正数' }}</p>
  <p>{{ items.filter(i => i.done).length }} 已完成</p>
</template>
```

## 三、注意事项与常见陷阱

- 模板中的表达式只能访问组件实例的属性和全局白名单对象（如 `Math`、`Date`）
- 不能在模板中使用语句（`if`、`for`、`try/catch`）
- 模板内表达式应保持简洁，复杂逻辑放在 computed 中
- Vue 3 支持多个根节点（Fragment）
- 模板引用和自定义指令只能在模板中使用
