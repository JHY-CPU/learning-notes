# 模板语法概述

## 一、概念说明

Vue 使用基于 HTML 的模板语法，允许声明式地将 DOM 绑定到组件实例的数据。所有 Vue 模板都是合法的 HTML，能被遵循 HTML 规范的浏览器和解析器解析。

Vue 编译器将模板编译为虚拟 DOM 渲染函数，结合响应式系统实现高效更新。相比 JSX，模板语法更直观，对设计师和初学者更友好。

```vue
<script setup>
import { ref } from 'vue'

const title = ref('Vue 模板语法')
const visible = ref(true)
</script>

<template>
  <h1>{{ title }}</h1>
  <p v-if="visible">内容可见</p>
</template>
```

## 二、具体用法

### 2.1 模板 vs JSX

```vue
<!-- 模板写法 -->
<template>
  <div class="card">
    <h2>{{ title }}</h2>
    <p>{{ description }}</p>
  </div>
</template>

<!-- JSX 写法（需要在 setup 中返回渲染函数） -->
<script setup>
import { h } from 'vue'

function render() {
  return h('div', { class: 'card' }, [
    h('h2', title.value),
    h('p', description.value)
  ])
}
</script>
```

### 2.2 模板中的 JavaScript 表达式

```vue
<template>
  <!-- 算术运算 -->
  <p>{{ price * quantity }}</p>

  <!-- 三元表达式 -->
  <p>{{ number > 0 ? '正数' : '非正数' }}</p>

  <!-- 方法调用 -->
  <p>{{ message.toUpperCase() }}</p>

  <!-- 数组操作 -->
  <p>{{ items.filter(i => i.done).length }} 已完成</p>

  <!-- 模板字面量 -->
  <p>{{ `你好，${name}！` }}</p>
</template>
```

### 2.3 受限的全局访问

模板中只能访问以下全局对象：
- `Infinity`、`undefined`、`NaN`
- `isNaN`、`parseFloat`、`parseInt`
- `decodeURI`、`decodeURIComponent`
- `encodeURI`、`encodeURIComponent`
- `Math`、`Number`、`Date`、`RegExp`、`Object`、`Array`

```vue
<template>
  <p>{{ Math.round(3.7) }}</p>
  <p>{{ new Date().getFullYear() }}</p>
  <!-- ❌ 不能访问 console、alert 等 -->
</template>
```

### 2.4 编译过程

```
模板 (.vue)
    ↓ 编译器解析
虚拟 DOM 渲染函数 (render)
    ↓ 响应式数据变化时
虚拟 DOM 树 (新)
    ↓ diff 对比
真实 DOM 最小化更新
```

## 三、常见用例

### 3.1 选择模板还是 JSX

| 场景 | 推荐 |
|------|------|
| 大多数项目 | 模板语法（更直观、工具支持好） |
| 动态渲染逻辑复杂 | JSX（更灵活） |
| 从 React 迁移 | JSX（更熟悉） |
| 组件库开发 | 两者结合 |

## 四、注意事项与常见陷阱

- 模板中的表达式只能访问组件实例的属性和全局白名单对象
- 不能在模板中使用语句（`if`、`for`、`try/catch`）
- 模板内表达式应保持简洁，复杂逻辑放在 computed 中
- Vue 3 支持多个根节点（Fragment）
- `<template>` 标签不会渲染为 DOM 元素
- 模板中 ref 会自动解包，不需要 `.value`
