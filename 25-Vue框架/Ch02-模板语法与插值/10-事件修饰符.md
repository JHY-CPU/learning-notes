# 事件修饰符

## 一、概念说明

Vue 提供事件修饰符来简化常见的事件处理模式。修饰符以 `.` 后缀的形式添加在事件名称后面，可以在不编写额外 JavaScript 的情况下处理事件冒泡、默认行为等。

```vue
<script setup>
import { ref } from 'vue'

function handleSubmit() {
  console.log('表单提交')
}

function handleClick() {
  console.log('点击事件，不会冒泡')
}
</script>

<template>
  <!-- .stop 阻止事件冒泡 -->
  <div @click="parentClick">
    <button @click.stop="handleClick">点击不冒泡</button>
  </div>

  <!-- .prevent 阻止默认行为 -->
  <form @submit.prevent="handleSubmit">
    <button type="submit">提交</button>
  </form>
</template>
```

## 二、具体用法

### 2.1 常用修饰符

```vue
<template>
  <!-- .stop: 阻止冒泡 (event.stopPropagation()) -->
  <button @click.stop="doThis">阻止冒泡</button>

  <!-- .prevent: 阻止默认行为 (event.preventDefault()) -->
  <a @click.prevent="doThat">阻止链接跳转</a>

  <!-- .once: 只触发一次 -->
  <button @click.once="doThis">只触发一次</button>

  <!-- .self: 只在事件目标是元素本身时触发 -->
  <div @click.self="doThis">
    <button>点击按钮不触发</button>
  </div>

  <!-- .capture: 使用捕获模式 -->
  <div @click.capture="doThis">捕获模式</div>
</template>
```

### 2.2 修饰符链式使用

```vue
<template>
  <!-- 顺序很重要: 先 .stop 再 .prevent -->
  <a @click.stop.prevent="doSomething">阻止冒泡和默认行为</a>

  <!-- 只有修饰符不带处理函数 -->
  <form @submit.prevent></form>
</template>
```

### 2.3 按键修饰符与鼠标修饰符

```vue
<template>
  <!-- .left / .right / .middle 鼠标按钮 -->
  <div @click.left="leftClick">左键</div>
  <div @click.right.prevent="rightClick">右键（禁用默认菜单）</div>
</template>
```

## 三、注意事项与常见陷阱

- 修饰符可以链式使用，顺序影响结果
- `@click.prevent.self` 和 `@click.self.prevent` 效果不同
- 不要过度使用修饰符，保持代码可读性
- `.once` 修饰符也可以用于组件事件
- `.exact` 修饰符可以精确控制组合键触发
