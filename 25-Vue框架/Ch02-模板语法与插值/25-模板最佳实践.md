# 模板最佳实践

## 一、概念说明

编写 Vue 模板时遵循最佳实践能提升代码的可读性、可维护性和性能。核心原则包括：保持模板简洁、避免副作用、合理使用指令、统一命名风格。

```vue
<script setup>
import { ref, computed } from 'vue'

const items = ref([
  { id: 1, name: 'Vue', done: true },
  { id: 2, name: 'React', done: false }
])

// 好: 复杂逻辑放在 computed 中
const completedCount = computed(() =>
  items.value.filter(i => i.done).length
)
</script>

<template>
  <!-- 好: 模板简洁，逻辑在 JS 中 -->
  <p>已完成: {{ completedCount }}</p>
  <ul>
    <li v-for="item in items" :key="item.id" :class="{ done: item.done }">
      {{ item.name }}
    </li>
  </ul>
</template>
```

## 二、具体用法

### 2.1 模板简洁性

```vue
<!-- 差: 模板中写复杂逻辑 -->
<template>
  <p>{{ items.filter(i => i.done).map(i => i.name).join(', ') }}</p>
</template>

<!-- 好: 用 computed 包装 -->
<script setup>
const completedNames = computed(() =>
  items.value.filter(i => i.done).map(i => i.name).join(', ')
)
</script>
<template>
  <p>{{ completedNames }}</p>
</template>
```

### 2.2 避免副作用

```vue
<!-- 差: 在模板中产生副作用 -->
<template>
  <p>{{ count++ }}</p>  <!-- 每次渲染都修改数据！ -->
</template>

<!-- 好: 只读取数据 -->
<template>
  <p>{{ count }}</p>
</template>
```

### 2.3 统一指令简写

```vue
<template>
  <!-- 推荐: 使用简写 -->
  <div :class="classes" @click="handleClick">
    <input v-model="value" />
  </div>

  <!-- 不推荐: 混用完整写法和简写 -->
  <div v-bind:class="classes" @click="handleClick">
    <input v-bind:value="value" v-on:input="value = $event.target.value" />
  </div>
</template>
```

### 2.4 组件 Props 顺序

```vue
<template>
  <!-- 推荐的属性顺序 -->
  <MyComponent
    v-if="show"
    v-show="visible"
    v-model="value"
    ref="myRef"
    :key="id"
    :prop-a="a"
    :prop-b="b"
    @event-a="handleA"
    @event-b="handleB"
    class="custom"
    style="color: red"
  />
</template>
```

## 三、注意事项与常见陷阱

- 模板表达式应保持简短，复杂逻辑放在 computed/methods 中
- 不要在模板中修改响应式数据（避免副作用）
- 始终为 `v-for` 添加 `:key`
- 避免 `v-if` 和 `v-for` 在同一元素上
- 使用 PascalCase 命名组件，kebab-case 使用组件
- 保持模板缩进和格式一致
