# 模板编译优化

## 一、概念说明

Vue 3 的编译器对模板进行了多项优化，显著提升渲染性能。主要优化包括：**静态提升**（hoistStatic）、**补丁标记**（patch flags）、**缓存事件处理器**（cacheHandlers）。这些优化对开发者透明，无需手动干预。

```vue
<script setup>
import { ref } from 'vue'

const count = ref(0)
</script>

<template>
  <!-- 静态节点: 编译时提升到渲染函数外部，不参与 diff -->
  <div class="static">
    <h1>标题</h1>
    <p>静态内容不会每次重新创建</p>
  </div>

  <!-- 动态节点: 带有 patch flag，只更新变化的部分 -->
  <div>
    <p>{{ count }}</p>
    <button @click="count++">+1</button>
  </div>
</template>
```

## 二、具体用法

### 2.1 静态提升（HoistStatic）

```vue
<!-- 编译前 -->
<template>
  <div>静态内容</div>
  <p>{{ dynamic }}</p>
</template>

<!-- 编译后（简化） -->
<!-- 静态节点提升到 render 函数外部 -->
// const hoisted = createVNode("div", null, "静态内容")
function render() {
  return [
    hoisted,              // 复用，不重新创建
    createVNode("p", null, dynamic)
  ]
}
```

### 2.2 补丁标记（Patch Flags）

```vue
<template>
  <!-- Vue 编译器标记动态部分的类型 -->
  <div>
    <p>{{ text }}</p>          <!-- TEXT flag: 只更新文本 -->
    <p :class="cls"></p>       <!-- CLASS flag: 只更新 class -->
    <p :style="sty"></p>       <!-- STYLE flag: 只更新 style -->
    <p :title="ttl"></p>       <!-- PROPS flag: 只更新 props -->
  </div>
</template>
```

### 2.3 缓存事件处理器

```vue
<template>
  <!-- 内联处理器会被缓存 -->
  <button @click="count++">+1</button>
  <!-- 编译器自动缓存此函数，不每次重建 -->
</template>
```

## 三、注意事项与常见陷阱

- 静态内容越多，编译优化效果越明显
- `v-once` 可以手动标记静态内容（Vue 3 中通常不需要）
- 复杂的动态表达式可能导致优化失效
- 使用 `<template v-for>` 时，子节点的静态提升更高效
- SSR 场景下还有跨请求的静态提升优化
