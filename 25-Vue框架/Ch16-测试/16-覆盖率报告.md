# 覆盖率报告

## 一、概念说明

代码覆盖率衡量测试用例执行了多少源代码。Vitest 内置支持覆盖率报告，使用 V8 引擎或 Istanbul 工具收集覆盖率数据。

```js
// vitest.config.ts
import { defineConfig } from 'vitest/config'

export default defineConfig({
  test: {
    coverage: {
      provider: 'v8',          // 或 'istanbul'
      reporter: ['text', 'html', 'lcov'],
      include: ['src/**/*.{js,ts,vue}'],
      exclude: ['src/main.ts', '**/*.d.ts'],
      thresholds: {
        lines: 80,
        functions: 80,
        branches: 70,
        statements: 80
      }
    }
  }
})
```

## 二、具体用法

### 2.1 运行覆盖率

```bash
npx vitest run --coverage
```

### 2.2 覆盖率指标

| 指标 | 说明 |
|------|------|
| **Statements** | 语句覆盖率 |
| **Branches** | 分支覆盖率（if/else） |
| **Functions** | 函数覆盖率 |
| **Lines** | 行覆盖率 |

### 2.3 报告格式

```js
coverage: {
  reporter: [
    'text',        // 终端输出
    'text-summary', // 终端摘要
    'html',        // HTML 可视化报告
    'lcov',        // CI/CD 集成
    'json'         // JSON 格式
  ]
}
```

### 2.4 门禁设置

```js
// 低于阈值时测试失败
coverage: {
  thresholds: {
    lines: 80,
    functions: 80,
    branches: 70,
    statements: 80,
    perFile: true // 每个文件都要达标
  }
}
```

## 三、注意事项与常见陷阱

- 不要盲目追求 100% 覆盖率，关注关键业务逻辑的覆盖
- V8 提供器性能优于 Istanbul，但 Istanbul 的报告更精确
- `exclude` 排除入口文件、类型定义、测试文件本身
- 覆盖率报告输出目录应加入 `.gitignore`
