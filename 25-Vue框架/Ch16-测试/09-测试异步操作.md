# 测试异步操作

## 一、概念说明

Vue 组件中常见异步操作：API 请求、定时器、DOM 更新等。测试异步代码需要正确等待操作完成再断言结果。

```vue
<!-- UserList.vue -->
<script setup>
import { ref, onMounted } from 'vue'
import axios from 'axios'

const users = ref([])
const loading = ref(false)
const error = ref(null)

async function fetchUsers() {
  loading.value = true
  error.value = null
  try {
    const { data } = await axios.get('/api/users')
    users.value = data
  } catch (e) {
    error.value = e.message
  } finally {
    loading.value = false
  }
}

onMounted(fetchUsers)
</script>

<template>
  <div v-if="loading">加载中...</div>
  <div v-else-if="error" class="error">{{ error }}</div>
  <ul v-else>
    <li v-for="u in users" :key="u.id">{{ u.name }}</li>
  </ul>
</template>
```

```js
import { mount, flushPromises } from '@vue/test-utils'
import { describe, it, expect, vi } from 'vitest'
import axios from 'axios'
import UserList from '../UserList.vue'

vi.mock('axios')

describe('UserList 异步', () => {
  it('加载成功显示用户列表', async () => {
    axios.get.mockResolvedValue({
      data: [{ id: 1, name: '张三' }]
    })

    const wrapper = mount(UserList)
    await flushPromises() // 等待所有 Promise 完成

    expect(wrapper.findAll('li')).toHaveLength(1)
    expect(wrapper.text()).toContain('张三')
  })

  it('加载失败显示错误信息', async () => {
    axios.get.mockRejectedValue(new Error('网络错误'))

    const wrapper = mount(UserList)
    await flushPromises()

    expect(wrapper.find('.error').text()).toBe('网络错误')
  })
})
```

## 二、具体用法

### 2.1 flushPromises

```js
import { flushPromises } from '@vue/test-utils'
// 等待所有微任务和宏任务完成
await flushPromises()
```

### 2.2 vi.useFakeTimers

```js
vi.useFakeTimers()

// 设置了 setTimeout 的组件
const wrapper = mount(DelayedComponent)
vi.advanceTimersByTime(1000) // 快进 1 秒
await wrapper.vm.$nextTick()
expect(wrapper.text()).toContain('1秒后')

vi.useRealTimers()
```

### 2.3 等待条件

```js
import { waitFor } from '@testing-library/vue'

await waitFor(() => {
  expect(wrapper.find('.result').exists()).toBe(true)
})
```

## 三、注意事项与常见陷阱

- `flushPromises` 比 `$nextTick` 更彻底，能等待所有 Promise
- 测试定时器使用 `vi.useFakeTimers()` 避免真实等待
- Mock 的 API 调用要在 `mount` 组件之前完成
