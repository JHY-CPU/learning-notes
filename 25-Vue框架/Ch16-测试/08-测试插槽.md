# 测试插槽

## 一、概念说明

测试组件的插槽（slot）是否正确渲染了传入的内容，包括默认插槽、具名插槽和作用域插槽。

```vue
<!-- Modal.vue -->
<script setup>
defineProps({ title: String })
defineEmits(['close'])
</script>

<template>
  <div class="modal">
    <header>
      <slot name="header">
        <h2>{{ title }}</h2>
      </slot>
      <button @click="$emit('close')">关闭</button>
    </header>
    <main>
      <slot />
    </main>
    <footer>
      <slot name="footer" :confirm="() => {}">
        <button>确定</button>
      </slot>
    </footer>
  </div>
</template>
```

```js
import { mount } from '@vue/test-utils'
import Modal from '../Modal.vue'

describe('Modal 插槽', () => {
  it('默认插槽内容', () => {
    const wrapper = mount(Modal, {
      slots: { default: '<p>弹窗内容</p>' }
    })
    expect(wrapper.find('main p').text()).toBe('弹窗内容')
  })

  it('具名插槽', () => {
    const wrapper = mount(Modal, {
      slots: {
        header: '<h1>自定义标题</h1>',
        footer: '<button>自定义按钮</button>'
      }
    })
    expect(wrapper.find('header h1').text()).toBe('自定义标题')
    expect(wrapper.find('footer button').text()).toBe('自定义按钮')
  })

  it('未传入插槽时使用默认内容', () => {
    const wrapper = mount(Modal, { props: { title: '默认标题' } })
    expect(wrapper.find('header h2').text()).toBe('默认标题')
  })

  it('作用域插槽', () => {
    const wrapper = mount(Modal, {
      slots: {
        footer: `<template #footer="{ confirm }">
          <button @click="confirm">确认</button>
        </template>`
      }
    })
    expect(wrapper.find('footer button').exists()).toBe(true)
  })
})
```

## 二、具体用法

### 2.1 传入插槽的多种形式

```js
// 字符串
slots: { default: '<p>内容</p>' }

// 组件
import Child from './Child.vue'
slots: { default: Child }

// 多个具名插槽
slots: {
  header: '<h1>标题</h1>',
  default: '<p>正文</p>',
  footer: '<button>按钮</button>'
}
```

## 三、注意事项与常见陷阱

- 插槽内容在 `mount` 时传入，不是组件内部定义的
- 测试作用域插槽需要使用 `<template #slotName>` 语法
- 默认插槽使用 `default` 作为键名
