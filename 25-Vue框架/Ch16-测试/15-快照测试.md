# 快照测试

## 一、概念说明

快照测试将组件的渲染输出保存为快照文件，后续测试运行时对比当前输出与快照是否一致。适合检测 UI 的意外变化。

```vue
<!-- Badge.vue -->
<script setup>
defineProps({
  count: { type: Number, default: 0 },
  variant: { type: String, default: 'primary' }
})
</script>

<template>
  <span :class="['badge', `badge-${variant}`]">
    <slot>{{ count }}</slot>
  </span>
</template>
```

```js
// Badge.test.js
import { describe, it, expect } from 'vitest'
import { mount } from '@vue/test-utils'
import Badge from '../Badge.vue'

describe('Badge 快照', () => {
  it('默认渲染匹配快照', () => {
    const wrapper = mount(Badge, { props: { count: 5 } })
    expect(wrapper.html()).toMatchSnapshot()
  })

  it('danger 变体匹配快照', () => {
    const wrapper = mount(Badge, {
      props: { count: 99, variant: 'danger' }
    })
    expect(wrapper.html()).toMatchSnapshot()
  })

  it('插槽内容匹配快照', () => {
    const wrapper = mount(Badge, {
      slots: { default: '新' }
    })
    expect(wrapper.html()).toMatchSnapshot()
  })
})
```

## 二、具体用法

### 2.1 内联快照

```js
// 快照内容直接嵌入测试文件，而非单独的 .snap 文件
expect(wrapper.html()).toMatchInlineSnapshot(`
  "<span class="badge badge-primary">5</span>"
`)
```

### 2.2 更新快照

```bash
# 首次运行生成快照，后续运行对比
npx vitest run

# UI 变化后更新快照
npx vitest run --update
```

### 2.3 局部快照

```js
// 只对特定部分做快照
expect(wrapper.find('.header').html()).toMatchSnapshot()
expect(wrapper.find('button').attributes()).toMatchSnapshot()
```

## 三、注意事项与常见陷阱

- 快照测试适合稳定的 UI 组件，频繁变化的组件不适合
- 快照文件（`.snap`）应该提交到版本控制
- 不要用快照测试替代具体的断言，快照只检测"是否变化"
- 时间戳、随机数等动态内容会导致快照频繁变化
