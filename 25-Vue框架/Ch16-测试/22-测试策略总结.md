# 测试策略总结

## 一、概念说明

测试策略决定了"测什么"和"不测什么"。合理的策略能以最低成本获得最高的质量保障。

```mermaid
测试金字塔:
  E2E (10%)     → 关键用户流程
  集成测试 (20%) → 组件协作
  单元测试 (70%) → 逻辑函数、Composable
```

## 二、具体用法

### 2.1 应该测试的内容

| 类别 | 示例 | 测试类型 |
|------|------|---------|
| 核心业务逻辑 | 计算函数、数据转换 | 单元 |
| 组件渲染 | Props、条件渲染、插槽 | 单元 |
| 用户交互 | 点击、表单提交 | 单元/集成 |
| API 交互 | 请求参数、响应处理 | 集成 |
| 路由导航 | 页面跳转、守卫 | 集成 |
| 关键流程 | 登录、下单、支付 | E2E |

### 2.2 不需要测试的内容

- Vue 框架本身的功能（响应式、生命周期）
- 第三方库的内部实现
- 简单的 getter/setter（不含逻辑）
- CSS 样式的具体值（视觉测试除外）
- TypeScript 类型检查（由编译器负责）

### 2.3 推荐的测试文件结构

```
src/
  components/
    UserCard.vue
    __tests__/
      UserCard.test.js   # 与组件同目录
  composables/
    useCounter.js
    __tests__/
      useCounter.test.js
```

### 2.4 工具选择指南

| 场景 | 推荐工具 |
|------|---------|
| 函数/Composable 测试 | Vitest |
| 组件测试 | Vitest + Vue Test Utils |
| 状态管理测试 | Vitest + @pinia/testing |
| E2E 测试 | Playwright（推荐）或 Cypress |
| API Mock | MSW 或 vi.mock |
| 覆盖率 | Vitest + V8 |

## 三、测试投资回报率

投入产出比从高到低排序：
1. 纯函数和工具函数的单元测试（成本最低，收益最高）
2. Composable 的单元测试
3. 关键组件的渲染和交互测试
4. API 交互的集成测试
5. 关键用户流程的 E2E 测试（成本最高，但保障最重要）

## 四、注意事项与常见陷阱

- 先写测试还是后写测试都可以，关键是写了
- 不要因为"没时间"就不写测试，bug 修复更耗时
- 测试覆盖率是指标，不是目标
- 持续维护测试，删除过时测试，更新快照
