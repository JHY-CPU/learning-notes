# 测试最佳实践

## 一、概念说明

良好的测试习惯能提高测试的可维护性和有效性。遵循约定和最佳实践，让测试成为开发的助力而非负担。

```js
// 好的测试结构示例
import { describe, it, expect, beforeEach } from 'vitest'
import { mount } from '@vue/test-utils'
import UserForm from '../UserForm.vue'

describe('UserForm', () => {
  let wrapper

  beforeEach(() => {
    wrapper = mount(UserForm)
  })

  describe('表单验证', () => {
    it('姓名为空时显示错误提示', async () => {
      await wrapper.find('form').trigger('submit')
      expect(wrapper.find('.error').text()).toContain('姓名不能为空')
    })

    it('邮箱格式错误时显示错误提示', async () => {
      await wrapper.find('[name="email"]').setValue('invalid')
      await wrapper.find('form').trigger('submit')
      expect(wrapper.find('.error').text()).toContain('邮箱格式错误')
    })
  })

  describe('表单提交', () => {
    it('有效数据时触发 submit 事件', async () => {
      await wrapper.find('[name="name"]').setValue('张三')
      await wrapper.find('[name="email"]').setValue('zhang@test.com')
      await wrapper.find('form').trigger('submit')

      expect(wrapper.emitted('submit')).toBeTruthy()
    })
  })
})
```

## 二、具体用法

### 2.1 命名规范

```js
// 推荐：描述行为
it('点击提交按钮后显示成功消息')

// 不推荐：描述实现
it('调用 submitForm 方法并设置 success 为 true')
```

### 2.2 AAA 模式

```js
it('测试描述', () => {
  // Arrange（准备）
  const wrapper = mount(Comp, { props: { count: 0 } })

  // Act（执行）
  wrapper.find('button').trigger('click')

  // Assert（断言）
  expect(wrapper.text()).toContain('1')
})
```

### 2.3 测试组织原则

- 每个 `describe` 对应一个组件或功能
- 按行为分组（验证、提交、渲染）
- 使用 `beforeEach` 共享 setup
- 每个测试只验证一件事

### 2.4 应该测试的内容

| 测试 | 说明 |
|------|------|
| 用户可见的输出 | 文本、样式、可见性 |
| 用户交互 | 点击、输入、提交 |
| 组件通信 | Props 接收、事件触发 |
| 边界情况 | 空数据、错误状态、极端值 |

## 三、注意事项与常见陷阱

- 测试行为，不测试实现细节（避免依赖内部变量名）
- 不要测试第三方库或 Vue 框架本身
- 每个测试应独立，可按任意顺序运行
- 避免在测试中使用 `setTimeout`，用 fake timers 替代
