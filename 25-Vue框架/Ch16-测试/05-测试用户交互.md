# 测试用户交互

## 一、概念说明

测试用户的点击、输入、键盘操作等交互行为，验证组件在用户操作后的状态变化是否正确。

```vue
<!-- SearchBox.vue -->
<script setup>
import { ref } from 'vue'
const emit = defineEmits(['search'])
const query = ref('')

function onSearch() {
  if (query.value.trim()) {
    emit('search', query.value)
  }
}
</script>

<template>
  <div>
    <input v-model="query" @keyup.enter="onSearch" data-testid="input" />
    <button @click="onSearch" :disabled="!query.trim()">搜索</button>
  </div>
</template>
```

```js
import { mount } from '@vue/test-utils'
import SearchBox from '../SearchBox.vue'

describe('SearchBox', () => {
  it('输入文字后按钮可用', async () => {
    const wrapper = mount(SearchBox)
    const input = wrapper.find('[data-testid="input"]')

    await input.setValue('Vue')
    expect(wrapper.find('button').attributes('disabled')).toBeUndefined()
  })

  it('点击搜索按钮触发事件', async () => {
    const wrapper = mount(SearchBox)
    await wrapper.find('input').setValue('Vue')
    await wrapper.find('button').trigger('click')

    expect(wrapper.emitted('search')).toBeTruthy()
    expect(wrapper.emitted('search')[0]).toEqual(['Vue'])
  })

  it('回车键触发搜索', async () => {
    const wrapper = mount(SearchBox)
    await wrapper.find('input').setValue('测试')
    await wrapper.find('input').trigger('keyup.enter')

    expect(wrapper.emitted('search')[0]).toEqual(['测试'])
  })
})
```

## 二、具体用法

### 2.1 常用交互方法

```js
await wrapper.find('button').trigger('click')         // 点击
await wrapper.find('input').setValue('hello')          // 输入
await wrapper.find('select').setValue('option1')       // 选择
await wrapper.find('input').trigger('keyup.enter')     // 键盘事件
await wrapper.find('form').trigger('submit.prevent')   // 表单提交
```

### 2.2 异步更新

```js
// 交互后必须 await DOM 更新
await wrapper.find('button').trigger('click')
await wrapper.vm.$nextTick()
expect(wrapper.text()).toContain('新内容')
```

## 三、注意事项与常见陷阱

- 所有交互操作必须使用 `await` 等待完成
- `trigger` 只触发事件，不会更新 `v-model` 绑定的值，需用 `setValue`
- 测试禁用状态时检查 `disabled` 属性是否存在，而非其值
