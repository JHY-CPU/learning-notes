# 测试组合式函数

## 一、概念说明

组合式函数（Composable）是 Vue 3 中逻辑复用的核心模式。测试 Composable 可以在不渲染组件的情况下，直接验证其逻辑正确性。

```js
// composables/useCounter.js
import { ref, computed } from 'vue'

export function useCounter(initialValue = 0) {
  const count = ref(initialValue)
  const doubled = computed(() => count.value * 2)

  function increment() { count.value++ }
  function decrement() { count.value-- }
  function reset() { count.value = initialValue }

  return { count, doubled, increment, decrement, reset }
}
```

```js
// tests/useCounter.test.js
import { describe, it, expect } from 'vitest'
import { useCounter } from '../composables/useCounter'

describe('useCounter', () => {
  it('默认初始值为 0', () => {
    const { count } = useCounter()
    expect(count.value).toBe(0)
  })

  it('自定义初始值', () => {
    const { count } = useCounter(10)
    expect(count.value).toBe(10)
  })

  it('increment 增加计数', () => {
    const { count, increment } = useCounter()
    increment()
    expect(count.value).toBe(1)
  })

  it('doubled 是 count 的两倍', () => {
    const { count, doubled, increment } = useCounter(5)
    increment()
    expect(doubled.value).toBe(12) // (5+1) * 2
  })

  it('reset 恢复初始值', () => {
    const { count, increment, reset } = useCounter(10)
    increment()
    increment()
    reset()
    expect(count.value).toBe(10)
  })
})
```

## 二、具体用法

### 2.1 测试需要生命周期的 Composable

```js
// 需要在 setup() 中使用的 composable，用测试组件包裹
import { defineComponent } from 'vue'
import { mount } from '@vue/test-utils'

function withSetup(composable) {
  let result
  const wrapper = mount(defineComponent({
    setup() {
      result = composable()
      return () => {}
    }
  }))
  return { result, wrapper }
}
```

### 2.2 测试 watch 和 watchEffect

```js
it('监听值变化', async () => {
  const { count, increment } = useCounter()
  const spy = vi.fn()
  watch(count, spy)
  increment()
  await nextTick()
  expect(spy).toHaveBeenCalled()
})
```

### 2.3 测试依赖注入的 Composable

```js
const wrapper = mount(TestComponent, {
  global: {
    provide: { theme: 'dark' }
  }
})
```

## 三、注意事项与常见陷阱

- 纯 Composable 无需 mount 组件，直接调用即可测试
- 使用了 `onMounted` 等生命周期的 Composable 需要包裹在组件中测试
- 每次测试创建新的 Composable 实例，确保测试隔离
