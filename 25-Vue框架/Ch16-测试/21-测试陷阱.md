# 测试陷阱

## 一、概念说明

测试中常见的陷阱和错误，了解这些坑能避免写出看似通过但实际无效的"假绿色"测试。

```js
// 陷阱一：忘记 await 异步操作
it('错误示范', () => {
  const wrapper = mount(Counter)
  wrapper.find('button').trigger('click') // 缺少 await！
  // 断言可能通过，但实际 DOM 还没更新
  expect(wrapper.text()).toContain('1') // 不可靠
})

// 正确写法
it('正确示范', async () => {
  const wrapper = mount(Counter)
  await wrapper.find('button').trigger('click')
  expect(wrapper.text()).toContain('1')
})

// 陷阱二：测试之间共享状态
describe('有陷阱的测试', () => {
  const items = []

  it('添加项目', () => {
    items.push('a')
    expect(items).toHaveLength(1) // 通过
  })

  it('检查项目数', () => {
    // 暗藏 bug：items 从上个测试继承了数据
    expect(items).toHaveLength(0) // 失败！
  })
})
```

## 二、常见陷阱

### 2.1 异步陷阱

```js
// 错误：未等待 Promise
it('异步陷阱', () => {
  fetchUsers().then(() => {
    expect(users.value).toHaveLength(3) // 永远不会执行到
  })
})

// 正确：使用 async/await
it('正确处理异步', async () => {
  await fetchUsers()
  expect(users.value).toHaveLength(3)
})
```

### 2.2 Mock 陷阱

```js
// 错误：mock 了但忘记断言
axios.get.mockResolvedValue({ data: [] })
// ... 调用了组件，但从未检查 axios.get 是否被调用

// 正确：验证 mock 被调用
expect(axios.get).toHaveBeenCalledWith('/api/users')
```

### 2.3 断言陷阱

```js
// 错误：toBe 用于对象比较
expect({ a: 1 }).toBe({ a: 1 }) // 失败，引用不同

// 正确：toEqual 用于深度比较
expect({ a: 1 }).toEqual({ a: 1 }) // 通过
```

### 2.4 组件清理陷阱

```js
// 每个测试后卸载组件
afterEach(() => {
  wrapper?.unmount()
})
```

## 三、调试技巧

```js
// 打印当前 DOM
console.log(wrapper.html())

// 使用 debug()
import { debug } from 'vitest/debug'

// 在浏览器中测试
it('调试测试', async () => {
  await page.pause() // Playwright 暂停
})
```

## 四、总结

- 永远 await 异步操作
- 每个测试独立，不共享可变状态
- Mock 后必须验证调用
- 用 `toEqual` 比较对象，`toBe` 比较原始值
