# 代码组织与复用策略

## 一、概念说明

组合式API允许按**逻辑关注点**而非选项类型组织代码。良好的代码组织提升可维护性和可读性。

```
src/
  composables/          # 组合式函数目录
    useMouse.js         # 鼠标追踪
    useFetch.js         # 数据请求
    useAuth.js          # 认证逻辑
    index.js            # 统一导出
  components/
    UserProfile.vue     # 使用composables的组件
```

## 二、具体用法

### 按逻辑关注点组织代码

```vue
<script setup>
// ❌ 按选项类型组织（旧方式思维）
import { ref, computed, watch, onMounted } from 'vue'
// 所有ref混在一起...

// ✅ 按逻辑关注点组织
// 用户相关逻辑
const userName = ref('')
const userAge = ref(0)
const userInfo = computed(() => `${userName.value}, ${userAge.value}岁`)

// 搜索相关逻辑
const keyword = ref('')
const results = ref([])
const searching = ref(false)
watch(keyword, async (val) => {
  searching.value = true
  results.value = await search(val)
  searching.value = false
})
</script>
```

### composables统一导出

```js
// composables/index.js
export { useMouse } from './useMouse'
export { useFetch } from './useFetch'
export { useCounter } from './useCounter'
export { useLocalStorage } from './useLocalStorage'
```

### 复用策略

```vue
<script setup>
// 1. 组合式函数复用（最常用）
import { useAuth } from '@/composables/useAuth'
import { useFetch } from '@/composables/useFetch'

// 2. 组合式函数嵌套组合
const { user, login } = useAuth()
const { data } = useFetch(() => `/api/users/${user.value?.id}`)

// 3. 在组件中组装不同composables
</script>
```

## 三、注意事项与常见陷阱

1. composables目录放**可复用**的逻辑，组件私有逻辑放组件内
2. 每个composable文件一个主要函数，文件名与函数名一致
3. 避免在composables中导入具体组件（防止循环依赖）
4. 组合式函数之间可以互相调用，但注意不要形成循环调用
5. 大型项目按功能模块分子目录：`composables/auth/`、`composables/ui/`
