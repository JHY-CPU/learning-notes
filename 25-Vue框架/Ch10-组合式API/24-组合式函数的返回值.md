# 组合式函数的返回值

## 一、概念说明

组合式函数的返回值设计是重要实践。推荐返回`ref`而不是`reactive`，因为解构reactive会丢失响应式，而ref解构后仍然是响应式的值。

```vue
<script setup>
import { useCounter } from './composables/useCounter'

// ref返回值：解构安全
const { count, doubled, increment } = useCounter()

// reactive返回值：解构会丢失响应式 ❌
// const { count, doubled } = useCounter()  // 不再响应式
</script>
```

## 二、具体用法

### 推荐：返回ref

```js
export function useCounter() {
  const count = ref(0)
  const doubled = computed(() => count.value * 2)
  const increment = () => count.value++

  return { count, doubled, increment }
  // 使用者可安全解构
}
```

### 返回reactive + toRefs

```js
export function useUser() {
  const state = reactive({
    name: '张三',
    age: 20,
    email: 'test@example.com'
  })

  return {
    ...toRefs(state),  // 解构后保持响应式
    getFullName: () => `${state.name} 先生`
  }
}
```

### 返回数组格式

```js
// 类似React Hooks风格
export function useToggle(initial = false) {
  const state = ref(initial)
  const toggle = () => { state.value = !state.value }
  return [state, toggle]
}

// 使用
const [isOpen, toggle] = useToggle()
```

### 返回函数（以函数为核心）

```js
export function useDebounce(fn, delay = 300) {
  let timer = null
  return (...args) => {
    clearTimeout(timer)
    timer = setTimeout(() => fn(...args), delay)
  }
}
```

## 三、注意事项与常见陷阱

1. **优先返回ref**：避免调用方解构丢失响应式
2. 返回对象比返回数组更语义化（对象有命名）
3. 返回computed也是只读ref，修改会警告
4. 不要返回组件内部创建的私有ref（暴露过多实现）
5. 返回的方法已绑定对应的ref，可直接作为事件处理函数

## 四、各种返回格式对比

### 4.1 返回对象（推荐）
```js
export function useFeature() {
  const count = ref(0)
  const increment = () => count.value++
  return { count, increment }
}
// 使用：const { count, increment } = useFeature()
// 优点：语义化、可选择性使用、解构安全
```

### 4.2 返回 reactive + toRefs
```js
export function useFeature() {
  const state = reactive({ count: 0, name: 'test' })
  return {
    ...toRefs(state),
    increment: () => state.count++
  }
}
// 使用：const { count, name, increment } = useFeature()
// 优点：适合大量相关状态
```

### 4.3 返回数组
```js
export function useToggle(initial = false) {
  const state = ref(initial)
  const toggle = () => { state.value = !state.value }
  return [state, toggle]
}
// 使用：const [isOpen, toggle] = useToggle()
// 优点：类似 React Hooks、可重命名
// 缺点：不语义化、顺序必须正确
```

### 4.4 返回函数
```js
export function useDebounce(fn, delay = 300) {
  let timer = null
  return (...args) => {
    clearTimeout(timer)
    timer = setTimeout(() => fn(...args), delay)
  }
}
// 使用：const debouncedSearch = useDebounce(search, 500)
// 适用：纯函数式逻辑
```

## 五、返回值设计原则

1. **优先返回 ref 对象**：解构安全，语义清晰
2. **大量相关状态用 reactive + toRefs**：减少 ref 数量
3. **避免返回原始 reactive**：解构会丢失响应式
4. **返回的方法应已绑定上下文**：可直接作为事件处理
5. **使用 computed 保护只读数据**：防止外部修改内部状态
