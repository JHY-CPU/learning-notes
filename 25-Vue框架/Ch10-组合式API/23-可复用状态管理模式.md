# 可复用状态管理模式

## 一、概念说明

使用组合式函数可以实现轻量级的全局状态管理模式，类似简化版的Pinia。适用于不想引入额外依赖的小型项目。

```vue
<template>
  <p>全局计数: {{ count }}</p>
  <button @click="increment">+1</button>
</template>

<script setup>
import { useStore } from './store'

const { count, increment } = useStore()
</script>
```

## 二、具体用法

### 模块化全局Store

```js
// store/index.js
import { reactive, readonly, computed } from 'vue'

// 私有状态
const state = reactive({
  user: null,
  theme: 'light',
  count: 0
})

// 只暴露readonly + 方法
export function useStore() {
  const user = computed(() => state.user)
  const theme = computed(() => state.theme)
  const count = computed(() => state.count)

  const setUser = (u) => { state.user = u }
  const toggleTheme = () => {
    state.theme = state.theme === 'light' ? 'dark' : 'light'
  }
  const increment = () => { state.count++ }

  return {
    user, theme, count,
    setUser, toggleTheme, increment
  }
}
```

### 分模块Store

```js
// store/modules/user.js
const userState = reactive({ profile: null, token: null })

export function useUserStore() {
  const login = async (creds) => { /* ... */ }
  const logout = () => { userState.token = null }
  return { ...toRefs(userState), login, logout }
}

// store/modules/cart.js
const cartState = reactive({ items: [], total: 0 })

export function useCartStore() {
  const addItem = (item) => { cartState.items.push(item) }
  return { ...toRefs(cartState), addItem }
}
```

## 三、注意事项与常见陷阱

1. 模块级变量天然单例，所有组件共享同一个状态
2. 使用`readonly`防止外部直接修改状态
3. 复杂项目建议使用Pinia，此模式适合简单场景
4. 返回computed保持响应式和只读性
5. SSR场景需注意服务端和客户端状态同步
