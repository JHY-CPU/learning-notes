# 可复用状态管理模式

## 一、概念说明

使用组合式函数可以实现轻量级的全局状态管理模式，类似简化版的Pinia。适用于不想引入额外依赖的小型项目。

```vue
<template>
  <p>全局计数: {{ count }}</p>
  <button @click="increment">+1</button>
</template>

<script setup>
import { useStore } from './store'

const { count, increment } = useStore()
</script>
```

## 二、具体用法

### 模块化全局Store

```js
// store/index.js
import { reactive, readonly, computed } from 'vue'

// 私有状态
const state = reactive({
  user: null,
  theme: 'light',
  count: 0
})

// 只暴露readonly + 方法
export function useStore() {
  const user = computed(() => state.user)
  const theme = computed(() => state.theme)
  const count = computed(() => state.count)

  const setUser = (u) => { state.user = u }
  const toggleTheme = () => {
    state.theme = state.theme === 'light' ? 'dark' : 'light'
  }
  const increment = () => { state.count++ }

  return {
    user, theme, count,
    setUser, toggleTheme, increment
  }
}
```

### 分模块Store

```js
// store/modules/user.js
const userState = reactive({ profile: null, token: null })

export function useUserStore() {
  const login = async (creds) => { /* ... */ }
  const logout = () => { userState.token = null }
  return { ...toRefs(userState), login, logout }
}

// store/modules/cart.js
const cartState = reactive({ items: [], total: 0 })

export function useCartStore() {
  const addItem = (item) => { cartState.items.push(item) }
  return { ...toRefs(cartState), addItem }
}
```

## 三、注意事项与常见陷阱

1. 模块级变量天然单例，所有组件共享同一个状态
2. 使用`readonly`防止外部直接修改状态
3. 复杂项目建议使用Pinia，此模式适合简单场景
4. 返回computed保持响应式和只读性
5. SSR场景需注意服务端和客户端状态同步

## 四、完整的 Store 模式示例

```js
// store/index.js
import { reactive, readonly, toRefs, computed } from 'vue'

// 私有状态（模块级变量，天然单例）
const state = reactive({
  user: null,
  token: localStorage.getItem('token'),
  theme: 'light',
  notifications: []
})

// 内部方法
function setUser(user) {
  state.user = user
}

function setToken(token) {
  state.token = token
  if (token) {
    localStorage.setItem('token', token)
  } else {
    localStorage.removeItem('token')
  }
}

// 公共 Store
export function useStore() {
  // 只读状态
  const isLoggedIn = computed(() => !!state.token)
  const userName = computed(() => state.user?.name || '游客')

  // 方法
  async function login(credentials) {
    const { token, user } = await api.login(credentials)
    setToken(token)
    setUser(user)
  }

  function logout() {
    setToken(null)
    setUser(null)
  }

  function toggleTheme() {
    state.theme = state.theme === 'light' ? 'dark' : 'light'
  }

  function addNotification(msg) {
    state.notifications.push({ id: Date.now(), message: msg })
  }

  return {
    // 只读状态
    user: computed(() => state.user),
    theme: computed(() => state.theme),
    token: computed(() => state.token),
    notifications: computed(() => state.notifications),
    isLoggedIn,
    userName,
    // 方法
    login,
    logout,
    toggleTheme,
    addNotification
  }
}
```

## 五、与 Pinia 的对比

| 特性 | Composable Store | Pinia |
|------|-----------------|-------|
| 依赖 | 无 | 需安装 |
| 体积 | 极小 | ~2KB |
| DevTools | 基础 | 完整 |
| 插件系统 | 无 | 有 |
| 持久化 | 手动 | 插件 |
| SSR | 手动处理 | 内置支持 |
| 适用 | 小型项目 | 中大型项目 |
