# 单元测试框架


## 一、Jest（JavaScript测试框架）


Jest是Facebook开发的JavaScript测试框架，零配置、内置断言、Mock和覆盖率，是前端测试的主流选择。


### 1.1 基本结构


```
// math.js - 被测模块
function add(a, b) { return a + b; }
function divide(a, b) {
    if (b === 0) throw new Error('除数不能为零');
    return a / b;
}
module.exports = { add, divide };

// math.test.js - 测试文件
const { add, divide } = require('./math');

describe('数学函数测试', () => {
    // 测试套件（test suite）
    describe('add', () => {
        test('两个正数相加', () => {
            expect(add(1, 2)).toBe(3);
        });
        test('负数相加', () => {
            expect(add(-1, -2)).toBe(-3);
        });
    });

    describe('divide', () => {
        test('正常除法', () => {
            expect(divide(6, 2)).toBe(3);
        });
        test('除以零抛出异常', () => {
            expect(() => divide(1, 0)).toThrow('除数不能为零');
        });
    });
});
```


### 1.2 常用断言（Matchers）


| 断言 | 说明 | 示例 |
| --- | --- | --- |
| toBe(value) | 严格相等（===） | expect(1+1).toBe(2) |
| toEqual(value) | 深度相等（对象/数组） | expect({a:1}).toEqual({a:1}) |
| toBeTruthy() | 真值 | expect(1).toBeTruthy() |
| toBeFalsy() | 假值 | expect(0).toBeFalsy() |
| toContain(item) | 数组/字符串包含 | expect([1,2]).toContain(2) |
| toThrow(error) | 抛出异常 | expect(fn).toThrow('msg') |
| toBeGreaterThan(n) | 大于 | expect(5).toBeGreaterThan(3) |
| toMatch(regexp) | 正则匹配 | expect('abc').toMatch(/a/) |
| toBeNull() | 为null | expect(null).toBeNull() |
| toHaveLength(n) | 长度为n | expect([1,2]).toHaveLength(2) |


## 二、Mock（模拟）


Mock用于隔离被测代码的依赖，模拟外部函数/模块的行为。


### 2.1 Jest Mock示例


```
// 被测函数：依赖API调用
async function fetchUser(id) {
    const response = await fetch(`/api/users/${id}`);
    return response.json();
}

// user.test.js
// 模拟全局fetch
global.fetch = jest.fn();

describe('fetchUser', () => {
    beforeEach(() => {
        fetch.mockClear(); // 每次测试前清除mock
    });

    test('获取用户成功', async () => {
        // 设置mock返回值
        fetch.mockResolvedValueOnce({
            json: () => Promise.resolve({ id: 1, name: '张三' })
        });

        const user = await fetchUser(1);

        expect(user.name).toBe('张三');
        expect(fetch).toHaveBeenCalledWith('/api/users/1');
        expect(fetch).toHaveBeenCalledTimes(1);
    });

    test('获取用户失败', async () => {
        fetch.mockRejectedValueOnce(new Error('网络错误'));
        await expect(fetchUser(999)).rejects.toThrow('网络错误');
    });
});
```


### 2.2 Mock类型


| 类型 | 说明 | Jest方法 |
| --- | --- | --- |
| Mock Function | 模拟函数调用 | jest.fn() |
| Spy | 监视但不替换原函数 | jest.spyOn() |
| Module Mock | 模拟整个模块 | jest.mock() |
| Timer Mock | 模拟定时器 | jest.useFakeTimers() |


## 三、其他测试框架简介


### 3.1 JUnit（Java）


```
// JUnit 5 示例
import org.junit.jupiter.api.*;
import static org.junit.jupiter.api.Assertions.*;

class CalculatorTest {
    private Calculator calc;

    @BeforeEach
    void setUp() {
        calc = new Calculator();
    }

    @Test
    @DisplayName("测试加法")
    void testAdd() {
        assertEquals(5, calc.add(2, 3));
    }

    @Test
    void testDivideByZero() {
        assertThrows(ArithmeticException.class, () -> {
            calc.divide(1, 0);
        });
    }

    @ParameterizedTest
    @ValueSource(ints = {1, 2, 3})
    void testPositive(int num) {
        assertTrue(num > 0);
    }
}
```


### 3.2 pytest（Python）


```
# pytest 示例
import pytest

def add(a, b):
    return a + b

def divide(a, b):
    if b == 0:
        raise ValueError("除数不能为零")
    return a / b

class TestMath:
    def test_add(self):
        assert add(1, 2) == 3

    def test_divide(self):
        assert divide(6, 2) == 3

    def test_divide_zero(self):
        with pytest.raises(ValueError):
            divide(1, 0)

    @pytest.mark.parametrize("a,b,expected", [
        (1, 2, 3), (-1, 1, 0), (0, 0, 0)
    ])
    def test_add_param(self, a, b, expected):
        assert add(a, b) == expected
```


### 3.3 框架对比


| 框架 | 语言 | 特点 |
| --- | --- | --- |
| Jest | JavaScript | 零配置、内置Mock、快照测试 |
| JUnit | Java | 注解驱动、参数化测试 |
| pytest | Python | 简洁语法、fixture、参数化 |
| Mocha | JavaScript | 灵活、需配合断言库(Chai) |


## 四、单元测试最佳实践（FIRST原则）


| 原则 | 说明 |
| --- | --- |
| **F**ast（快） | 测试应快速执行，毫秒级别 |
| **I**solated（隔离） | 测试之间相互独立，不依赖执行顺序 |
| **R**epeatable（可重复） | 每次运行结果一致 |
| **S**elf-validating（自验证） | 测试自动判断通过/失败 |
| **T**imely（及时） | 与代码同步编写，不拖延 |


> **Important:** **AAA模式：**
>
>
> Arrange（准备）：设置测试数据和环境
>
>
> Act（执行）：调用被测函数
>
>
> Assert（断言）：验证结果


## 五、知识要点总结


1. Jest：describe/test/expect，内置Mock和覆盖率
2. Mock：jest.fn()、jest.spyOn()、jest.mock()
3. JUnit：@Test注解、参数化测试
4. pytest：简洁assert语法、fixture、参数化装饰器
5. FIRST原则：Fast/Isolated/Repeatable/Self-validating/Timely
6. AAA模式：Arrange/Act/Assert
