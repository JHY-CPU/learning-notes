性能测试 (Performance Testing)

负载测试、压力测试、浸泡测试 — 全面评估系统性能与稳定性

1. 性能测试类型

类型目的负载模式持续时间关注指标
基准测试 (Baseline)建立性能基准线单用户/少量请求几分钟单次响应时间
负载测试 (Load)验证预期负载下的表现逐步升至预期值15-60 分钟TPS、RT、错误率
压力测试 (Stress)找到系统极限超预期持续加压30-120 分钟崩溃点、恢复时间
浸泡测试 (Soak)发现内存泄漏等问题中等负载长时间数小时~数天内存趋势、GC 表现
峰值测试 (Spike)验证突发流量处理瞬时大幅增长短时间响应降级、自动扩缩
容量测试 (Capacity)确定最大用户/数据量逐步增加数据量1-4 小时最大支持量

2. 核心性能指标

吞吐量 (TPS / QPS)

  • TPS = Transactions Per Second（事务/秒）
  • QPS = Queries Per Second（查询/秒）
  • RPS = Requests Per Second（请求/秒）
  • 衡量系统处理能力的核心指标

响应时间 (Response Time)

  • 平均响应时间 (Avg)
  • 中位数 (P50)：50% 请求在此时间内完成
  • 尾部延迟 (P95, P99)：95%/99% 的值
  • 最大响应时间 (Max)

错误率 (Error Rate)

  • 失败请求数 / 总请求数
  • 目标：通常低于 0.1%
  • 分类型统计：超时、5xx、连接拒绝
  • 错误率上升是系统过载的信号

系统资源指标

  • CPU 使用率 (CPU Usage)
  • 内存占用 (Memory Usage)
  • 磁盘 I/O (Disk I/O)
  • 网络带宽 (Network Throughput)
  • 数据库连接池使用率

性能指标关系

吞吐量-响应时间曲线：在低负载时，吞吐量随并发增加而上升，响应时间基本稳定；达到拐点后，吞吐量不再增长而响应时间急剧上升——这就是系统的"最优并发数"。

4. 工具使用示例

k6 脚本 (JavaScript)

import http from 'k6/http';
import { check, sleep } from 'k6';

// 负载配置
export let options = {
    stages: [
        { duration: '30s', target: 50 },   // 30秒内升至50用户
        { duration: '2m',  target: 200 },  // 2分钟内升至200用户
        { duration: '5m',  target: 200 },  // 保持200用户5分钟
        { duration: '30s', target: 0 },    // 30秒内降至0
    ],
    thresholds: {
        http_req_duration: ['p(95)<500'],  // 95% 请求在500ms内
        http_req_failed: ['rate<0.01'],   // 错误率低于1%
    },
};

export default function() {
    let res = http.get('https://api.example.com/products');
    check(res, {
        '状态码为200': (r) => r.status === 200,
        '响应时间 < 500ms': (r) => r.timings.duration < 500,
    });
    sleep(1); // 思考时间
}

// 运行: k6 run --out json=results.json script.js

5. 性能测试流程

准备阶段

  • 明确性能目标与 SLA
  • 识别关键业务场景
  • 准备测试数据（脱敏）
  • 搭建独立测试环境
  • 配置监控 (Prometheus/Grafana)

执行与分析

  • 执行基准测试建立基线
  • 逐步增加负载执行
  • 实时监控系统资源
  • 收集日志和错误信息
  • 生成报告，对比基线
常见性能瓶颈排查顺序：1) 数据库慢查询 → 2) 不合理的缓存策略 → 3) 序列化/反序列化开销 → 4) GC 停顿 → 5) 线程池配置不当 → 6) 网络延迟 → 7) 锁竞争
