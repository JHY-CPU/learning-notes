# 覆盖准则与路径测试


## 一、控制流图（Control Flow Graph）


控制流图是白盒测试的基础工具，用节点表示语句/语句块，用边表示控制转移。


### 1.1 控制流图的基本元素


| 元素 | 含义 | 对应代码 |
| --- | --- | --- |
| 节点（Node） | 基本块（无分支的语句序列） | 顺序执行的语句 |
| 边（Edge） | 控制转移 | 跳转、分支 |
| 判定节点 | 有多个出边的节点 | if、while、for |


### 1.2 示例代码的控制流图


```
// 示例函数
int foo(int a, int b, int x) {
    ① int y = 0;
    ② if (a > 0 && b > 0)
    ③     y = a + b;
    ④ if (x > 0)
    ⑤     y = y + x;
    ⑥ return y;
}

// 控制流图：
//   ①
//   ↓
//   ② ──T──→ ③ ──→ ④
//   │              │
//   F ──────────→  ④ ──T──→ ⑤ ──→ ⑥
//                     │           ↑
//                     F ──────────┘
```


$$
环路复杂度（Cyclomatic Complexity）：
                V(G) = E - N + 2P
                E = 边数，N = 节点数，P = 连通分量数（通常P=1）
                或 V(G) = 判定节点数 + 1
                或 V(G) = 区域数（闭合区域 + 外部区域）
                环路复杂度 = 独立路径数 = 最少测试用例数
$$


## 二、白盒测试覆盖准则


覆盖准则是衡量测试充分性的标准，覆盖要求从弱到强递进。


### 2.1 覆盖准则层次（从弱到强）


| 覆盖准则 | 要求 | 覆盖率公式 |
| --- | --- | --- |
| **语句覆盖** | 每条可执行语句至少执行一次 | 执行语句数 / 总语句数 |
| **判定覆盖**
（分支覆盖） | 每个判定的真假分支都至少执行一次 | 覆盖分支 / 总分支数 |
| **条件覆盖** | 每个条件的真假取值都至少出现一次 | 覆盖条件值 / 总条件值 |
| **判定/条件覆盖** | 同时满足判定覆盖和条件覆盖 | - |
| **条件组合覆盖** | 每个判定中条件的所有组合都至少执行一次 | 覆盖组合 / 总组合数 |
| **路径覆盖** | 所有可能的路径都至少执行一次 | 覆盖路径 / 总路径数 |


### 2.2 示例分析


```
// 被测代码
if (A > 0 && B > 0)
    C = C / (A + B);
if (A == 2 || C > 1)
    C = C + 1;
```


### 各覆盖准则的测试用例


| 覆盖准则 | 测试用例 | 覆盖内容 |
| --- | --- | --- |
| 语句覆盖 | A=2, B=1, C=3 | 所有4条语句都执行到 |
| 判定覆盖 | ① A=2, B=1, C=3（两个if都真）
② A=-1, B=-1, C=0（两个if都假） | 每个判定的T和F分支都覆盖 |
| 条件覆盖 | ① A=2, B=-1, C=0（A>0真，B>0假，A==2真，C>1假）
② A=-1, B=1, C=2（A>0假，B>0真，A==2假，C>1真） | 每个简单条件的T和F都出现 |


## 三、基路径测试（Basis Path Testing）


由McCabe提出，基于环路复杂度确定一组独立路径作为测试基础。


### 3.1 基路径测试步骤


1. 画出程序的控制流图
2. 计算环路复杂度 V(G)
3. 确定一组线性独立路径（基路径集）
4. 为每条独立路径设计测试用例


### 3.2 独立路径


独立路径是指至少包含一条在其他独立路径中未曾包含的边的路径。


$$
示例：控制流图有 4个节点(N=4)，5条边(E=5)
                V(G) = E - N + 2 = 5 - 4 + 2 = 3
                需要3条独立路径，即3个测试用例
$$


### 3.3 示例


```
// 代码
void sort(int a, int b, int c) {
    ① if (a > b)
    ②     swap(a, b);
    ③ if (a > c)
    ④     swap(a, c);
    ⑤ if (b > c)
    ⑥     swap(b, c);
    ⑦ return;
}

// 控制流图：7个节点，8条边（估算）
// 环路复杂度 V(G) = 3 + 1 = 4
// 4条独立路径：
// 路径1：①F → ③F → ⑤F → ⑦  （a≤b≤c，有序）
// 路径2：①T → ② → ③F → ⑤F → ⑦  （a>b≤c）
// 路径3：①F → ③T → ④ → ⑤F → ⑦  （a≤b，a>c，交换后b>c？）
// 路径4：①F → ③F → ⑤T → ⑥ → ⑦  （a≤b≤c不成立，b>c）
```


## 四、数据流测试


基于变量的定义（def）和使用（use）来设计测试用例。


### 4.1 关键概念


| 概念 | 含义 | 示例 |
| --- | --- | --- |
| 定义点（def） | 变量被赋值的位置 | x = 5; 的位置 |
| 使用点（use） | 变量被使用的节点 | y = x + 1; 中x的使用 |
| P-use | 谓词使用（在判定条件中） | if (x > 0) 中x的使用 |
| C-use | 计算使用（在赋值/表达式中） | y = x + 1 中x的使用 |


### 4.2 数据流覆盖准则


- **全定义覆盖**
   ：每个定义点至少有一个从定义到使用的路径被覆盖
- **全使用覆盖**
   ：每个定义点到每个使用点的路径都被覆盖
- **全定义使用路径覆盖**
   ：每个定义到使用的简单路径都被覆盖（最强）


## 五、知识要点总结


1. 控制流图：节点=基本块，边=控制转移
2. 环路复杂度 V(G) = E - N + 2 = 判定数 + 1
3. 覆盖准则：语句 < 判定 < 条件 < 判定/条件 < 条件组合 < 路径
4. 基路径测试：用V(G)确定最少测试用例数
5. 数据流测试：关注变量的定义和使用
