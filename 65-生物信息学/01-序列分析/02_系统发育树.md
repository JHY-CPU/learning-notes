# 系统发育树


## 系统发育树


生物信息学系统发育进化


系统发育树描述了生物序列或物种之间的进化关系，是分子进化研究的核心工具。


## 系统发育树基本概念


```
树的组成要素：
┌──────────────────────────────────────────────────┐
│                  进化树结构                        │
│                                                  │
│        ┌── 物种A                                  │
│     ┌──┤                                         │
│     │  └── 物种B    ← 叶节点（OTU, 外部分类单元）  │
│  ───┤                                           │
│     │  ┌── 物种C                                  │
│     └──┤                  ← 分支长度 = 进化距离    │
│        └── 物种D                                  │
│                                                  │
│  根节点：最近共同祖先                               │
│  内节点：推断的祖先节点                             │
│  叶节点：观测到的物种/序列                          │
│  分支：进化路径                                    │
│  分支长度：进化距离（替代数/时间）                   │
└──────────────────────────────────────────────────┘

树的类型：
┌──────────────┬──────────────────────────────────────┐
│ 有根树       │ 指定根节点，反映进化方向               │
│ (Rooted)    │ 根到叶的路径表示祖先到后代的进化       │
├──────────────┼──────────────────────────────────────┤
│ 无根树       │ 不指定根节点                          │
│ (Unrooted)  │ 只表示亲缘关系，不表示进化方向         │
├──────────────┼──────────────────────────────────────┤
│ 分支树       │ 分支长度有生物学意义                   │
│ (Phylogram) │ 通常按进化距离缩放                     │
├──────────────┼──────────────────────────────────────┤
│ 阶梯树       │ 分支长度无意义                        │
│ (Cladogram) │ 只表示拓扑结构                        │
└──────────────┴──────────────────────────────────────┘
```


## 距离法（Distance-Based Methods）


距离法首先计算所有序列对之间的进化距离，然后根据距离矩阵构建树。


```
距离计算模型：
┌──────────────────────────────────────────────────┐
│ Jukes-Cantor（JC69）：                            │
│ d = -3/4 * ln(1 - 4p/3)                          │
│ 其中 p = 序列间观测差异数 / 序列长度               │
│ 假设：所有替代等概率，碱基频率相等                  │
│                                                  │
│ Kimura 2-parameter（K2P）：                       │
│ d = -1/2 * ln(1-2P-Q) - 1/4 * ln(1-2Q)          │
│ P = 转换（transition）比例                        │
│ Q = 颠换（transversion）比例                      │
│ 区分了转换和颠换的不同速率                          │
│                                                  │
│ 更复杂模型：                                      │
│ - HKY85：考虑碱基频率不等                          │
│ - GTR：最通用的可逆替换模型                        │
│ - +Γ：加入位点间速率异质性（Gamma 分布）           │
│ - +I：加入不变位点比例                             │
└──────────────────────────────────────────────────┘

UPGMA（非加权配对算术平均法）：
- 假设分子钟（恒定进化速率）
- 算法：每步合并距离最近的两个类群
- 合并后距离取两组距离的加权平均
- 结果总是有根的、超度量树

Neighbor-Joining（NJ，邻接法）：
- 不假设分子钟
- 算法：
  1. 计算净分化距离 r(i) = Σ_j D(i,j) / (n-2)
  2. 计算修正距离 M(i,j) = D(i,j) - r(i) - r(j)
  3. 合并 M 最小的两个类群
  4. 更新距离矩阵
  5. 重复直到只剩两个类群
- 速度快 O(n³)，适合大数据集
```


## 最大简约法（Maximum Parsimony）


最大简约法寻找需要最少进化变化的树，基于奥卡姆剃刀原则。


```
基本思想：
找到一棵树，使得所有位点上的总替代数最少

算法步骤：
1. 对每棵候选树，计算每个位点的最小替代数
2. 对所有位点求和，得到树的总代价
3. 选择总代价最小的树（简约树）

位点分类：
┌──────────────┬────────────────────────────────────┐
│ 信息位点      │ 至少两种状态出现超过一次             │
│ (Informative)│ 对树的构建有贡献                    │
├──────────────┼────────────────────────────────────┤
│ 自裔位点      │ 只在一个分类单元中出现               │
│ (Autapomorphy)│ 不影响树的拓扑结构                  │
├──────────────┼────────────────────────────────────┤
│ 恒定位点     │ 所有分类单元状态相同                  │
│ (Constant)   │ 无信息                              │
└──────────────┴────────────────────────────────────┘

加权简约法：
- 不同等价替代给予不同权重
- 例：转换比颠换更频繁 → 转换权重低
- 避免长枝吸引（Long Branch Attraction）

树搜索方法：
- 穷举法：检查所有可能的树（仅适合少量序列）
- 分支定界法：剪枝策略减少搜索空间
- 启发式搜索：逐步加入序列，SPR/TBR/NNI 树重排
```


## 贝叶斯推断法（Bayesian Inference）


贝叶斯方法通过马尔可夫链蒙特卡洛（MCMC）采样来估计系统发育树的后验分布。


```
贝叶斯公式：
P(Tree, θ | Data) ∝ P(Data | Tree, θ) × P(Tree) × P(θ)

其中：
- P(Data | Tree, θ)：似然函数（同 ML）
- P(Tree)：树拓扑的先验（通常均匀）
- P(θ)：模型参数先验（分支长度、替换模型参数等）
- P(Tree, θ | Data)：后验分布（目标）

MCMC 采样（MrBayes, BEAST）：
┌─────────────────────────────────────────────────────┐
│  1. 初始化树和参数                                    │
│  2. 进行大量迭代（通常百万级）：                       │
│     a. 提议新的树拓扑（SPR / NNI）                    │
│     b. 提议新的分支长度                               │
│     c. 提议新的模型参数                               │
│     d. 计算接受概率 = min(1, 后验比)                  │
│     e. 以接受概率接受或拒绝提议                        │
│  3. 定期采样当前状态作为后验样本                       │
│  4. 丢弃 burn-in 期的样本                             │
│  5. 剩余样本近似后验分布                               │
└─────────────────────────────────────────────────────┘

MCMC 诊断：
- ESS（有效样本量）> 200
- 轨迹图（Trace Plot）平稳
- 分裂频率标准差（PSRF）接近 1
- 多条独立链结果一致

优势：
- 提供后验概率（支持度量）
- 可纳入先验知识
- 自然处理参数不确定性
- 适合分子钟定年（BEAST）
```


> **Note:** 贝叶斯方法和最大似然法在实际应用中通常给出相似的树拓扑。贝叶斯方法的优势在于提供后验概率和支持分子钟分析。


## 树的评估与可视化


```
支持度评估方法：
┌────────────────────────────────────────────────────┐
│ Bootstrap 自举法                                   │
│ 从原始数据中重采样列（有放回）                       │
│ 重新构建树，重复 100-1000 次                        │
│ 统计每个分支在重采样树中出现的频率                    │
│ >70% 通常认为可靠，>95% 认为强支持                  │
├────────────────────────────────────────────────────┤
│ 后验概率（贝叶斯）                                  │
│ MCMC 采样中某分支出现的频率                          │
│ >0.95 通常认为强支持                               │
├────────────────────────────────────────────────────┤
│ 似然比检验                                         │
│ 嵌套模型的比较                                     │
│ 用于检验特定假设（如分子钟假设）                     │
└────────────────────────────────────────────────────┘

常用软件：
┌──────────────┬────────────────────────────────────┐
│ MEGA         │ 综合工具箱，GUI 友好                │
│ RAxML        │ 最大似然法，大规模数据               │
│ IQ-TREE      │ 快速 ML，模型选择                   │
│ MrBayes      │ 贝叶斯推断                         │
│ BEAST        │ 贝叶斯，分子钟定年                  │
│ FigTree      │ 树可视化                           │
│ iTOL         │ 在线树可视化和注释                  │
└──────────────┴────────────────────────────────────┘
```
