# 序列比对算法


## 序列比对算法


生物信息学序列比对BLAST


序列比对是生物信息学的基础，用于发现DNA/蛋白质序列之间的相似性。


## Needleman-Wunsch (全局比对)


```
全局比对：找到两条序列的最优对齐方式

动态规划递推：
F(i,j) = max {
    F(i-1, j-1) + S(xi, yj)  // 匹配/错配
    F(i-1, j) + gap_penalty   // x有gap
    F(i, j-1) + gap_penalty   // y有gap
}

示例（序列ACGT vs AGT，BLOSUM62矩阵）：

      -    A    G    T
  -   0   -2   -4   -6
  A  -2    1   -1   -3
  C  -4   -1    0   -2
  G  -6   -3    2    0
  T  -8   -5    0    3

回溯得到最优比对：
A C G T
|   | |
A - G T

时间复杂度：O(m×n)
空间复杂度：O(m×n)（可优化到O(min(m,n))）
```


## Smith-Waterman (局部比对)


```
局部比对：找到两条序列中最相似的子序列

与全局比对的关键区别：
F(i,j) = max {
    0                    ← 新增：允许从0开始
    F(i-1, j-1) + S(xi, yj)
    F(i-1, j) + gap_penalty
    F(i, j-1) + gap_penalty
}

取最大值时如果小于0则设为0
回溯从最大值开始，到0结束

适用场景：
- 寻找蛋白质中的保守结构域
- 部分相似序列的发现
- 外显子比对
```


## BLAST (快速序列搜索)


```
BLAST (Basic Local Alignment Search Tool)：
启发式搜索，比动态规划快数千倍

核心思想（种子-扩展）：
1. 将查询序列切分为短词(w-mer，通常3-11)
2. 在数据库中查找相似的种子词（使用索引）
3. 以种子为中心，双向扩展高分片段对(HSP)
4. 统计评估HSP的显著性(E-value)

BLAST变体：
- blastn：核酸 vs 核酸数据库
- blastp：蛋白质 vs 蛋白质数据库
- blastx：核酸翻译后 vs 蛋白质库
- tblastn：蛋白质 vs 翻译的核酸库
- tblastx：翻译核酸 vs 翻译核酸库

E-value（期望值）：
E = K × m × n × e^(-λS)
- m, n：序列长度
- S：比对得分
- E < 1e-5 通常认为显著
- E < 1e-10 强烈相关

打分矩阵：
BLOSUM62：基于保守区域的替换频率
- 正分：保守替换（如I↔L）
- 负分：非保守替换（如A↔W）
- 数字62：序列相似度≤62%的组
```


> **Note:** BLOSUM矩阵用于蛋白质比对，PAM矩阵基于进化模型。实际中BLOSUM62最常用。
