🤝 TCP 三次握手与四次挥手

TCP 连接建立（三次握手）、连接释放（四次挥手）的详细流程和状态变化。

三次握手 (Three-Way Handshake)

TCP 在传输数据前必须建立连接，三次握手确保双方都具备收发能力。

// ========== 三次握手流程 ========== // // 客户端 服务器 // │ │ // │ 1. SYN (seq=x) │ // │ ────────────────────────────→ │ SYN-SENT SYN-RECEIVED // │ │ // │ 2. SYN+ACK (seq=y, ack=x+1)│ // │ ←──────────────────────────── │ ESTABLISHED SYN-RECEIVED // │ │ // │ 3. ACK (seq=x+1, ack=y+1) │ // │ ────────────────────────────→ │ ESTABLISHED ESTABLISHED // │ │ // │ 开始传输数据... │ // // ========== 每一步的作用 ========== // 第 1 步 (SYN): // 客户端 → 服务器: "我要连接你" // 客户端初始序列号 x // // 第 2 步 (SYN+ACK): // 服务器 → 客户端: "收到,我准备好了" // 服务器初始序列号 y // 确认收到客户的 seq=x // // 第 3 步 (ACK): // 客户端 → 服务器: "好的,开始通信" // 确认收到服务器的 seq=y
🔑 为什么是三次而不是两次? 防止已过期的 SYN 请求再次建立连接。如果客户端第一个 SYN 在网络中延迟,重发后连接建立,而延迟的 SYN 又到了——三次握手可以避免这种情况。

四次挥手 (Four-Way Wavehand)

TCP 连接释放需要四次交互，因为 TCP 连接是全双工的，双方需要独立关闭。

// ========== 四次挥手流程 ========== // // 客户端 服务器 // │ │ // │ 1. FIN (seq=u) │ FIN-WAIT-1 CLOSE-WAIT // │ ────────────────────────────→ │ // │ │ // │ 2. ACK (ack=u+1) │ FIN-WAIT-2 CLOSE-WAIT // │ ←──────────────────────────── │ // │ │ // │ (服务器可能还有数据发送) │ // │ │ // │ 3. FIN (seq=v) │ TIME-WAIT LAST-ACK // │ ←──────────────────────────── │ // │ │ // │ 4. ACK (ack=v+1) │ TIME-WAIT CLOSED // │ ────────────────────────────→ │ // │ │ // │ (2MSL 等待后关闭) │ CLOSED // // ========== 关键状态 ========== // FIN-WAIT-1: 客户端发送 FIN,等待确认 // FIN-WAIT-2: 收到 ACK,等待服务器 FIN // CLOSE-WAIT: 服务器等待应用关闭 // TIME-WAIT: 等待 2MSL 确保所有包消失 // LAST-ACK: 服务器发送 FIN,等待最终 ACK

2MSL 等待与状态详解

// ========== TIME-WAIT (2MSL) ========== // MSL (Maximum Segment Lifetime): 最大段生存时间 // 通常 30s - 2 分钟 // 2MSL = 1-4 分钟 // // TIME-WAIT 的目的: // 1. 确保最后一个 ACK 到达服务器 // (如果 ACK 丢失,服务器重发 FIN) // 2. 让旧连接的所有数据包在网络中消失 // (防止影响新连接) // // ========== 常见问题 ========== // 大量 TIME-WAIT 连接: // 高并发短连接场景,端口被占用 // 解决方案: SO_REUSEADDR 选项 // // SYN 洪水攻击: // 攻击者发送大量 SYN 不完成握手 // 服务器半开连接耗尽资源 // 解决方案: SYN Cookie, 缩短超时 // ========== TCP 状态机 ========== // CLOSED → SYN-SENT → ESTABLISHED → FIN-WAIT-1 // ↑ ↓ // └──────── TIME-WAIT ← FIN-WAIT-2 ←─┘ // ↓ // CLOSE-WAIT → LAST-ACK → CLOSED
