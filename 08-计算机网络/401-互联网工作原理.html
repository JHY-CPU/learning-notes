🌐 互联网工作原理

互联网的定义、历史、基本架构和数据流动方式。

什么是互联网

互联网是一个全球性的计算机网络系统，通过标准协议（TCP/IP）将全世界数十亿设备连接在一起。

// ========== 互联网的核心组成 ========== // // 用户设备 (客户端) // ↓ // 接入网 (ISP) // ↓ // 骨干网 (核心路由器) // ↓ // 服务器 (数据中心) // ========== 关键概念 ========== // 节点 (Node) : 任何连接到网络的设备 // 链路 (Link) : 连接节点的物理或无线通道 // 数据包 (Packet) : 数据传输的基本单位 // 协议 (Protocol) : 通信的规则和标准 // 带宽 (Bandwidth): 数据传输速率 (bps) // 延迟 (Latency) : 数据从源到目的地的时间
💡 互联网不是"云"——它由遍布全球的海底光缆、数据中心、路由器和交换机等物理基础设施组成。

数据如何传输

当你访问一个网站时，数据经历以下步骤：

// ========== HTTP 请求旅程 ========== // // 1. 浏览器缓存检查 → 有缓存则直接使用 // 2. DNS 解析 → 域名 → IP 地址 // 3. TCP 三次握手 → 建立连接 // 4. TLS 握手 (HTTPS) → 加密通信 // 5. HTTP 请求 → 发送到服务器 // 6. 服务器处理 → 生成响应 // 7. HTTP 响应 → 返回给客户端 // 8. 浏览器渲染 → 显示页面 // ========== 时间分布 (典型值) ========== // DNS 查询 : 10-50ms // TCP 连接 : 10-30ms // TLS 握手 : 20-100ms // 请求传输 : 5-20ms // 服务器处理 : 50-200ms // 响应传输 : 5-20ms // 合计 ≈ 100-420ms

网络拓扑

常见的网络拓扑结构及其特点：

// ========== 网络拓扑类型 ========== // // 星型拓扑 (Star): // 所有节点连接到中心交换机 // 优点: 易于管理,单点故障不影响其他 // 缺点: 中心节点是单点瓶颈 // // 总线拓扑 (Bus): // 所有节点共享一条通信线路 // 优点: 布线简单 // 缺点: 冲突多,故障难定位 // // 环形拓扑 (Ring): // 节点组成闭环 // 优点: 无冲突,传输公平 // 缺点: 单点故障断开整个环 // // 网状拓扑 (Mesh): // 节点间全连接 // 优点: 高可靠性 // 缺点: 布线成本极高

互联网的物理基础设施

// ========== 关键基础设施 ========== // // 海底光缆: // - 全球 400+ 条海底光缆 // - 承担 99% 的国际数据流量 // - 典型容量: 100Tbps+ // - 延迟: 纽约→伦敦 ≈ 30ms // // 数据中心: // - 全球约 6000+ 个数据中心 // - AWS/Azure/Google Cloud 占主导 // - 单数据中心可容纳 10万+ 服务器 // // IXP (互联网交换中心): // - ISP 之间交换流量的物理点 // - 降低延迟,减少带宽成本

分层模型

互联网通信通过分层模型组织，每层负责不同的功能：

// ========== OSI 七层模型 vs TCP/IP 模型 ========== // // OSI 模型 TCP/IP 模型 协议示例 // ─────────────────────────────────────────────── // 7. 应用层 ┐ // 6. 表示层 ├→ 应用层 HTTP/FTP/SMTP/DNS // 5. 会话层 ┘ // 4. 传输层 → 传输层 TCP/UDP // 3. 网络层 → 网络互连层 IP/ICMP // 2. 数据链路层 ┐ // 1. 物理层 ┘→ 网络接口层 Ethernet/WiFi // ========== 数据封装过程 ========== // 应用层: HTTP 请求数据 // 传输层: 数据 + TCP 头 (源/目的端口) // 网络层: 数据 + IP 头 (源/目的 IP) // 链路层: 数据 + 帧头 (源/目的 MAC) // 物理层: 比特流 (010101...)
📌 每个分层只与相邻层通信，这种"分层解耦"使得各层可以独立演化——比如应用层不必关心底层是光纤还是 5G。
