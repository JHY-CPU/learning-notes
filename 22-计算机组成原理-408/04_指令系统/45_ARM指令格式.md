# 45_ARM指令格式

## 核心概念

### ARM基本特征
- **32位定长指令**（ARM模式）
- **16位定长指令**（Thumb模式）
- **Load/Store结构**
- **16个通用寄存器**（R0\~R15）
- **条件执行域**：每条指令都可条件执行

### ARM指令格式（A32模式）

```
┌────────┬──────────────────────────────────────────────┐
│cond(4) │              其他字段                        │
│ 31~28  │              27~0                           │
└────────┴──────────────────────────────────────────────┘
```

**每条指令的高4位都是条件码**，这意味着**每条指令都可以条件执行**，无需单独的条件分支。

### 条件码域

| 条件码 | 助记符 | 含义 | 标志位条件 |
|:---:|:---|:---|:---|
| 0000 | EQ | 相等 | Z=1 |
| 0001 | NE | 不相等 | Z=0 |
| 0010 | CS/HS | 进位设置/高于等于 | C=1 |
| 0011 | CC/LO | 进位清除/低于 | C=0 |
| 0100 | MI | 负数 | N=1 |
| 0101 | PL | 正数或零 | N=0 |
| 0110 | VS | 溢出 | V=1 |
| 0111 | VC | 无溢出 | V=0 |
| 1000 | HI | 高于（无符号） | C=1 且 Z=0 |
| 1001 | LS | 低于等于 | C=0 或 Z=1 |
| 1010 | GE | 大于等于（有符号） | N=V |
| 1011 | LT | 小于 | N≠V |
| 1100 | GT | 大于 | Z=0 且 N=V |
| 1101 | LE | 小于等于 | Z=1 或 N≠V |
| 1110 | AL | 总是执行（默认） | 任何 |

### 数据处理指令格式

```
┌──────┬───┬────────┬─────┬─────┬─────────────────┐
│cond  │00 │opcode  │ S   │ Rn  │ Rd  │  operand2 │
│ 4位  │2位│ 4位    │ 1位 │ 4位 │ 4位 │   12位    │
└──────┴───┴────────┴─────┴─────┴─────┴─────────────┘
```

| 字段 | 含义 |
|:---|:---|
| **cond** | 条件码 |
| **opcode** | 操作类型（ADD/SUB/AND等） |
| **S** | 是否更新条件标志（S=1更新CPSR） |
| **Rn** | 第一源操作数寄存器 |
| **Rd** | 目的寄存器 |
| **operand2** | 第二操作数（立即数或移位寄存器） |

### Load/Store指令格式

```
┌──────┬───┬──┬──┬─────┬─────┬───────────────────┐
│cond  │01 │P │U │  B  │  W  │ L │ Rn │ Rd │offset│
│ 4位  │2位│1 │1 │ 1位│ 1位 │1位│4位 │4位 │ 12位 │
└──────┴───┴──┴──┴─────┴─────┴───┴─────┴─────┴──────┘
```

- **P**：前/后变址（Pre/Post indexing）
- **U**：加/减（Up/Down）
- **B**：字节/字
- **W**：写回基址寄存器
- **L**：Load/Store

### Thumb模式（16位指令）

- 为了提高代码密度（嵌入式系统）
- 16位定长指令
- 寄存器受限（只能访问R0\~R7）
- 功能比A32模式少

### ARM vs MIPS对比

| 对比项 | ARM | MIPS |
|:---|:---|:---|
| 指令长度 | 32位（A32）/ 16位（Thumb） | 32位 |
| 条件执行 | 每条指令都有条件码 | 需要单独条件分支 |
| 寄存器 | 16个 | 32个 |
| 操作数 | 三操作数（灵活移位） | 三操作数 |
| 立即数 | 12位（8位值+4位旋转） | 16位 |

## 原理分析

### ARM的条件执行优势

**MIPS的if语句**：
```
    BNE $t0, $t1, skip  ; 需要分支
    ADD $t2, $t3, $t4   ; 条件为真时执行
skip:
```

**ARM的if语句**：
```
    ADDEQ R2, R3, R4    ; 条件码直接在指令中，无需分支
```

ARM的方式**避免了分支指令**，减少了流水线冲突。

### ARM的桶形移位器

ARM的operand2字段支持**移位操作**：

$$operand2 = Rm \ll shift$$

一条指令可以完成"移位+运算"：
```
ADD R0, R1, R2, LSL #3   ; R0 = R1 + (R2 << 3)
```

在MIPS中这需要两条指令（先SLL再ADD）。

### 记忆技巧

### 易错陷阱
1. **ARM的条件码在每条指令中**：不是只在分支指令中
2. **Thumb和ARM是两种执行模式**：同一处理器可切换
3. **ARM的12位立即数不是简单的12位**：8位值+4位旋转编码
4. **S位控制是否更新标志**：默认不更新（与x86不同）
