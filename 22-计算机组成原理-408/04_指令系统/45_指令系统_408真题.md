# 46_指令系统_408真题

## 核心概念

### 408真题题型分布

指令系统在408中主要出题形式：
1. **选择题**：寻址方式判断、EA计算、操作码计算
2. **综合题**：扩展操作码方案设计、指令格式分析

## 真题详解

### 真题1：扩展操作码计算（选择题高频）

**题目（改编自2009年408）**：某机指令字长16位，操作码占4位，采用扩展操作码技术。三地址指令15条，二地址指令若干条，一地址指令若干条，零地址指令16条。问二地址和一地址指令最多各多少条？

**解答**：

三地址：OP 4位，$2^4 - 1 = 15$ 条（保留1111扩展）。地址各 $(16-4)/3 = 4$ 位。

二地址：前4位=1111，后4位OP。$2^4 - 1 = 15$ 条（保留1111继续扩展）。地址各 $(16-8)/2 = 4$ 位。

一地址：前8位=1111 1111，后4位OP。$2^4 - 1 = 15$ 条（保留1111继续扩展）。地址 $16-12 = 4$ 位。

零地址：前12位=1111 1111 1111，后4位OP。$2^4 = 16$ 条。

**答案**：二地址最多15条，一地址最多15条。

### 真题2：寻址方式判断与EA计算

**题目（改编自2010年408）**：某机按字节编址，BR=2000H，IX=0100H，PC=3000H。指令`MOV R1, [BR+IX+50H]`，求EA和操作数地址。

**解答**：

寻址方式：**基址+变址+偏移**（组合寻址）

$$EA = BR + IX + A = 2000H + 0100H + 50H = 2150H$$

操作数存放在地址2150H处。

访存次数（不含取指）= 1次。

### 真题3：指令格式分析

**题目（改编自2012年408）**：某机指令字长32位，操作码8位，三地址格式。主存容量1MB，按字节编址。

（1）每个地址字段至少需要多少位？
（2）若采用扩展操作码，三地址指令最多多少条？

**解答**：

（1）主存1MB = $2^{20}$ 字节，地址需要20位。

三地址：$3 \times 20 = 60$ 位地址字段，但指令字长32位，$32 - 8 = 24$ 位用于地址。

$24 / 3 = 8$ 位/地址字段。寻址范围 $2^8 = 256$，无法覆盖1MB。

**结论**：三地址格式在32位指令字长下无法直接寻址1MB空间。需要使用基址/变址等间接寻址方式。

（2）操作码8位，最多 $2^8 - 1 = 255$ 条三地址指令（保留全1扩展）。

### 真题4：访存次数计算

**题目（改编自2013年408）**：某指令的寻址方式为寄存器间接寻址，R1=1000H，M[1000H]=2000H。求获取操作数所需的访存次数。

**解答**：

寄存器间接寻址：
- $EA = R1 = 1000H$（从寄存器取，0次访存）
- 操作数 = M[1000H] = 2000H（1次访存）

$$访存次数（不含取指）= 1$$

### 真题5：相对寻址的PC值

**题目（改编自2015年408）**：某条件转移指令位于地址2000H，指令长度4字节，偏移量为+20H（补码）。求跳转目标地址。

**解答**：

关键：PC取指后已更新为下一条指令地址。

$$PC_{取指后} = 2000H + 4 = 2004H$$
$$EA = PC + A = 2004H + 20H = 2024H$$

跳转目标地址：2024H。

## 直观理解

### 真题解题要点

| 题型 | 关键点 |
|:---|:---|
| 扩展操作码 | 每级非末级保留1个码点（$2^n-1$） |
| EA计算 | 先识别寻址方式，再套公式 |
| 指令格式分析 | $指令字长 = OP + 地址数 \times 地址位数$ |
| 访存次数 | 不含取指，按寻址方式计算 |
| 相对寻址PC值 | PC是"下一条指令地址" |

## 知识关联

### 易错总结

1. **扩展操作码忘了保留码点**
2. **PC值是取指后的值**
3. **访存次数不含取指令**
4. **地址字段位数和寻址范围的换算**
5. **有符号偏移量用补码**
