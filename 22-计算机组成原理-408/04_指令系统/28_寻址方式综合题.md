# 28_寻址方式综合题

## 核心概念

### 综合分析的思路

分析一段程序中各指令的寻址方式时：
1. 逐条分析指令格式
2. 根据地址码字段和寻址方式标志确定寻址方式
3. 计算EA和操作数
4. 追踪寄存器值的变化

## 例题详解

### 综合例题1：程序段分析

**题目**：分析以下程序段中每条指令的寻址方式、EA和执行结果。

已知条件：
- R0 = 0, R1 = 1000H, R2 = 200H
- BR = 5000H（基址寄存器）
- M[1000H] = 0050H, M[1200H] = 0030H, M[5200H] = 0080H
- 指令长度均为4字节，起始地址2000H

```
地址    指令                    寻址方式
2000H: MOV R3, #20H           ① 立即寻址
2004H: MOV R4, [R1]           ② 寄存器间接寻址
2008H: ADD R4, [BR + 200H]    ③ 基址寻址
200CH: MOV [R1 + R2], R4      ④ 基址+变址组合
```

**逐条分析**：

**① `MOV R3, #20H`**
- 寻址方式：**立即寻址**
- 无EA，操作数 = 20H
- 执行后：R3 = 20H
- 访存次数（不含取指）：0

**② `MOV R4, [R1]`**
- 寻址方式：**寄存器间接寻址**
- $EA = R1 = 1000H$
- 操作数 = M[1000H] = 0050H
- 执行后：R4 = 0050H
- 访存次数：1

**③ `ADD R4, [BR + 200H]`**
- 寻址方式：**基址寻址**
- $EA = BR + 200H = 5000H + 200H = 5200H$
- 操作数 = M[5200H] = 0080H
- 执行后：R4 = 0050H + 0080H = 00D0H
- 访存次数：1

**④ `MOV [R1 + R2], R4`**
- 寻址方式：**寄存器间接+偏移**（基址+变址组合）
- 目标EA = R1 + R2 = 1000H + 200H = 1200H
- 执行后：M[1200H] = R4 = 00D0H
- 访存次数：1（写入存储器）

### 综合例题2：循环程序分析

**题目**：分析以下循环程序的寻址方式。

```
      MOV R0, #0        ; i = 0
      MOV R1, #10       ; 循环次数
      MOV R2, #1000H    ; 数组首地址
LOOP: LOAD R3, (R2)     ; 从R2指向的地址取数据
      ADD R4, R3        ; 累加到R4
      ADD R2, #4        ; 指针后移4字节
      INC R0            ; i++
      CMP R0, R1        ; 比较i和10
      JL LOOP           ; i < 10则跳转
```

**寻址方式分析**：

| 指令 | 寻址方式 | 说明 |
|:---|:---|:---|
| `MOV R0, #0` | 立即寻址 | 操作数=0 |
| `MOV R1, #10` | 立即寻址 | 操作数=10 |
| `MOV R2, #1000H` | 立即寻址 | 操作数=1000H |
| `LOAD R3, (R2)` | 寄存器间接寻址 | EA=R2 |
| `ADD R4, R3` | 寄存器寻址 | 操作数=R3 |
| `ADD R2, #4` | 立即寻址（源）、寄存器寻址（目的） | R2 ← R2 + 4 |
| `INC R0` | 寄存器寻址 | R0 ← R0 + 1 |
| `CMP R0, R1` | 寄存器寻址 | 比较两个寄存器 |
| `JL LOOP` | 相对寻址 | EA = PC + 偏移（LOOP标签） |

### 综合例题3：函数调用与堆栈

**题目**：分析函数调用过程中的寻址方式。

```
主程序:
    PUSH arg3          ; ① 堆栈寻址
    PUSH arg2          ; ② 堆栈寻址
    PUSH arg1          ; ③ 堆栈寻址
    CALL func          ; ④ 相对寻址+堆栈寻址

func:
    PUSH BP            ; ⑤ 堆栈寻址（保存旧基指针）
    MOV BP, SP         ; ⑥ 寄存器寻址
    MOV R1, [BP+8]     ; ⑦ 基址寻址（取参数）
    ...
    POP BP             ; ⑧ 堆栈寻址
    RET                ; ⑨ 堆栈寻址
```

**分析**：

- **①②③ PUSH**：堆栈寻址，EA = SP（递增或递减后）
- **④ CALL**：跳转用相对寻址，同时PUSH返回地址用堆栈寻址
- **⑤ PUSH BP**：堆栈寻址
- **⑥ MOV BP, SP**：寄存器间传送，两个都是寄存器寻址
- **⑦ MOV R1, [BP+8]**：基址寻址（BP是当前栈帧基址，+8是参数偏移）
- **⑧ POP BP**：堆栈寻址
- **⑨ RET**：从堆栈弹出返回地址，堆栈寻址

## 直观理解

### 综合分析的检查清单

- [ ] 每条指令确定了寻址方式吗？
- [ ] EA计算正确吗？（注意PC值、寄存器值）
- [ ] 访存次数统计对吗？（不含取指）
- [ ] 寄存器值的变化追踪了吗？
- [ ] 循环/跳转的目标地址算对了吗？

### 跨章节联系
- **高级语言编译**：表达式→指令序列→寻址方式选择
- **第5章（CPU）**：每种寻址方式的微操作实现
- **操作系统**：函数调用的栈帧布局

### 易错陷阱
1. **一条指令可能涉及多种寻址方式**：源和目的可能用不同方式
2. **"寄存器间传送"两个操作数都是寄存器寻址**
3. **PUSH/POP同时涉及堆栈寻址和另一种寻址**
4. **循环中寄存器值不断变化**：每次迭代EA可能不同
