# 19_基址寻址

## 核心概念

### 定义
- **基址寻址（Base Addressing）**：有效地址EA = **基址寄存器BR的内容** + **形式地址A**
- 基址寄存器的内容由**操作系统**设置，用户程序**不可修改**

### 格式与语义

$$[OP][寻址方式标志][形式地址A]$$

$$EA = BR + A$$

$$操作数 = M[BR + A]$$

其中 $BR$ 为基址寄存器（Base Register），$A$ 为偏移量（形式地址/位移量）。

### 基址寄存器的特点

| 特征 | 说明 |
|:---|:---|
| **设置者** | 操作系统（特权指令设置） |
| **用户可否修改** | 不可（用户态指令无法修改BR） |
| **作用** | 提供程序的**基地址**（起始地址） |
| **典型值** | 程序在主存中的起始地址 |

### 特点

| 优点 | 缺点 |
|:---|:---|
| **支持程序重定位**：程序可在主存任意位置运行 | **BR由OS管理**：灵活性受限 |
| **寻址范围大**：EA = BR + A，BR宽度=主存地址宽度 | 需要额外的BR寄存器 |
| **便于多道程序**：不同程序有不同的BR值 | 偏移量A的范围仍受限于其位数 |
| **保护机制基础**：可配合界限寄存器做越界检查 | |

## 原理分析

### 程序重定位的实现

**问题**：程序编译时不知道会被加载到主存的哪个位置。

**解决**：
1. 程序内部使用**相对地址**（从0开始编址）
2. OS将程序加载到主存后，将**起始地址**写入BR
3. 运行时：$EA = BR + 相对地址$

**例题**：程序从逻辑地址0开始编译，被加载到物理地址2000H。
- BR = 2000H
- 指令访问逻辑地址100H
- $EA = 2000H + 100H = 2100H$

### 寻址范围分析

设：
- BR占 $m$ 位（主存地址宽度）
- A占 $n$ 位（$n < m$）

$$EA范围 = BR_{min} + 0 \sim BR_{max} + (2^n - 1)$$

由于BR可设置为任意值，寻址范围覆盖**整个主存空间**。

### 访存次数

$$访存次数（不含取指）= 1$$

- BR在CPU寄存器中（0次访存）
- 从M[BR+A]取操作数（1次访存）

### 基址寻址与重定位寄存器

在分页/分段存储管理中：
- **基址寄存器** = 段基址 / 页表基址
- **偏移量** = 段内偏移 / 页内偏移

$$物理地址 = 基址 + 逻辑地址$$

### 计算题

**例题**：BR = 3000H，A = 0200H，主存按字节编址。求EA和操作数地址。

$$EA = 3000H + 0200H = 3200H$$

操作数存放在地址3200H处。

## 直观理解

### 生活类比

基址寻址就像**公司总部+楼层偏移**：
- 公司大楼的基地址（BR）由总部统一安排
- "3楼的会议室"中的"3楼"是偏移量（A）
- 实际位置 = 大楼基地址 + 3楼偏移
- 员工只需说"3楼"（相对地址），系统自动加上基地址

### 记忆技巧

- **"基址 = 基地 + 偏移 = OS管基地，用户填偏移"**
- **"BR是地基，A是楼层，EA是绝对位置"**

## 知识关联

### 跨章节联系
- **第3章（存储系统）**：基址寻址是分段存储管理的硬件基础
- **第5章（CPU）**：BR是专用寄存器，需特权指令修改
- **操作系统**：程序重定位、内存分配、地址映射

### 常考对比
- **基址寻址 vs 变址寻址**：基址BR由OS设（不变），变址IX由用户改（可变）
- **基址寻址 vs 直接寻址**：基址可重定位，直接寻址的地址固定

### 易错陷阱
1. **BR用户不可改**：这是基址寻址的核心特征，由OS用特权指令设置
2. **EA = BR + A，不是BR × A**：加法不是乘法
3. **BR和A的位数可能不同**：BR通常是完整的主存地址宽度
4. **基址寻址支持重定位但不等于分页**：基址是分段概念，分页另有机制
