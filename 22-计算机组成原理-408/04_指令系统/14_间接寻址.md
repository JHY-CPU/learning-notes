# 15_间接寻址

## 核心概念

### 定义
- **间接寻址（Indirect Addressing）**：指令地址码中的形式地址 $A$ 不是操作数的有效地址，而是**存放有效地址的存储单元的地址**
- 需要**两次或多次访存**才能获得操作数

### 格式与语义

**一次间接寻址**：

$$EA = M[A]$$

$$操作数 = M[EA] = M[M[A]]$$

形式地址 $A$ 指向的存储单元中存放的是**有效地址EA**，EA指向的存储单元中才是**操作数**。

**二次间接寻址**：

$$EA = M[M[A]]$$

$$操作数 = M[M[M[A]]]$$

形式地址 $A$ 指向的单元存放**第一次间接地址**，该地址指向的单元存放**有效地址EA**。

### 图示

```
一次间接：
指令: [OP][A]
         │
         ▼
      M[A] = EA     ← 第一次访存获得EA
         │
         ▼
      M[EA] = 操作数  ← 第二次访存获得操作数

二次间接：
指令: [OP][A]
         │
         ▼
      M[A] = A1     ← 第一次访存
         │
         ▼
      M[A1] = EA    ← 第二次访存获得EA
         │
         ▼
      M[EA] = 操作数  ← 第三次访存获得操作数
```

### 特点

| 优点 | 缺点 |
|:---|:---|
| **寻址范围大**：EA可以是完整地址 | **访存次数多**：一次间接需2次访存 |
| **地址灵活**：EA可动态改变 | **执行速度慢**：多次访存增加延迟 |
| **支持指针操作**：类似高级语言指针 | **实现复杂**：需区分直接/间接 |

## 原理分析

### 访存次数

| 间接次数 | 访存次数（不含取指） | 说明 |
|:---:|:---:|:---|
| 0次（直接） | 1 | 直接用A作EA |
| 1次间接 | 2 | 先取EA，再取操作数 |
| 2次间接 | 3 | 先取A1，再取EA，再取操作数 |
| $k$ 次间接 | $k+1$ | 递推 |

**实际上**：大多数机器只支持一次间接寻址，多次间接效率太低。

### 间接寻址标志

指令中需要一个标志位区分直接/间接寻址：

$$[OP][间接标志I][A]$$

- $I = 0$：直接寻址，$EA = A$
- $I = 1$：间接寻址，$EA = M[A]$

### 间接寻址的寻址范围

设形式地址 $A$ 占 $n$ 位，主存按字节编址，地址占 $m$ 位：
- **直接寻址范围**：$2^n$ 个单元
- **间接寻址范围**：$2^m$ 个单元（取决于间接地址的宽度，通常等于主存地址宽度）

间接寻址不受 $A$ 的位数限制，可访问全部主存。

### 间接寻址与指针的对应关系

高级语言中的**指针变量**在底层通常用间接寻址实现：

```c
int *p = &x;  // p中存放x的地址
int y = *p;   // 间接寻址：从p取地址，再从该地址取值
```

$p$ 对应形式地址 $A$ 指向的单元（存放EA），$*p$ 对应 $M[EA]$。

## 直观理解

### 生活类比

间接寻址就像**查电话簿找人**：
- 直接寻址：纸条写着"去3号房间找人"
- 间接寻址：纸条写着"去电话簿第20页"，翻到第20页写着"3号房间"——再去3号房间

二次间接：纸条→电话簿→索引卡→3号房间（多绕一层）。

### 记忆技巧

- **"间接 = 中间多一层 = 地址的地址"**
- **"一次间接访存两次，两次间接访存三次"**

## 知识关联

### 跨章节联系
- **第3章（存储系统）**：多次访存严重影响性能，Cache可缓解
- **第5章（CPU）**：间接寻址需要额外的微操作周期
- **高级语言**：指针、引用的底层实现

### 常考对比
- 间接寻址 vs 直接寻址：访存次数和寻址范围的权衡
- 间接寻址 vs 寄存器间接寻址：后者通过寄存器存放EA，只需1次访存

### 易错陷阱
1. **"间接寻址的EA不是A"**：$EA = M[A]$，A指向的单元内容才是EA
2. **访存次数 = 间接次数 + 1**：一次间接2次访存
3. **间接寻址不等于多次间接**：一般默认一次间接
4. **间接寻址和"取间址"是不同概念**：间址周期是CPU执行间接寻址的微操作阶段
