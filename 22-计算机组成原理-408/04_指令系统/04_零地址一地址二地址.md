# 04_零地址一地址二地址

## 核心概念

### 各地址格式的完整特点

#### 零地址指令

- **格式**：仅含操作码，无地址字段
- **操作数来源**：
  1. **堆栈型计算机**：操作数隐含在堆栈的栈顶和次栈顶
  2. **空操作**：`NOP`、`HLT`等不需要操作数的指令
  3. **隐含寄存器**：某些操作隐含使用特定寄存器（如PSW）
- **代表指令**：`NOP`（空操作）、`HLT`（停机）、`RET`（返回）、`PUSHF`（标志入栈）
- **堆栈运算示例**：
  ```
  栈顶 = B, 次栈顶 = A
  执行 ADD: 弹出B和A, 计算A+B, 压入结果
  ```

#### 一地址指令

- **格式**：OP A
- **单目运算**：$M[A] \leftarrow \text{OP}(M[A])$
  - `NEG A`（取反）、`INC A`（自增）、`NOT A`（逻辑非）
- **双目运算（隐含另一操作数在ACC）**：
  - $ACC \leftarrow ACC \text{ OP } M[A]$
  - `ADD A` 表示 $ACC \leftarrow ACC + M[A]$
- **传送类**：`LOAD A`（$ACC \leftarrow M[A]$）、`STORE A`（$M[A] \leftarrow ACC$）
- **特点**：指令短，节省存储空间，但表达复杂运算需多条指令

#### 二地址指令

- **格式**：OP A1, A2
- **语义**：$M[A1] \leftarrow M[A1] \text{ OP } M[A2]$
- **A1既是源也是目的**：运算结果覆盖A1的原值
- **最常见的格式**：x86的主要指令格式
- **示例**：
  - `ADD R1, R2`：$R1 \leftarrow R1 + R2$
  - `MOV R1, [1000H]`：$R1 \leftarrow M[1000H]$

### 按存储结构分类

| 分类 | 特点 | 典型指令格式 |
|:---|:---|:---|
| **寄存器-寄存器（RR型）** | 两个源和结果都在寄存器 | 三地址或二地址，所有地址都是寄存器 |
| **寄存器-存储器（RS型）** | 一个源在存储器 | 二地址，一个地址为存储器地址 |
| **存储器-存储器（SS型）** | 两个源和结果都在存储器 | 三地址，所有地址都是存储器地址 |

### RISC vs CISC 的地址格式倾向

| 特征 | RISC | CISC |
|:---|:---|:---|
| **主要格式** | 寄存器-寄存器（RR） | 寄存器-存储器（RS）或存储器-存储器（SS） |
| **地址格式** | Load/Store结构：只有LOAD/STORE访存 | 多数指令可直接访存 |
| **原因** | 访存慢，尽量用寄存器 | 指令功能强大，一条指令做更多事 |

## 原理分析

### 堆栈型零地址指令执行过程

以计算 $(A+B) \times C$ 为例：

```
初始堆栈: 空
PUSH A   → 栈: [A]
PUSH B   → 栈: [A, B]
ADD      → 栈: [A+B]        （弹出B和A，相加后压入）
PUSH C   → 栈: [A+B, C]
MUL      → 栈: [(A+B)*C]    （弹出C和A+B，相乘后压入）
```

全部是零地址指令（ADD/MUL/MUL不带地址），操作数全从栈中获取。

### 一地址指令的执行过程

以计算 $ACC \times M[A]$ 为例：

```
LOAD B    → ACC ← M[B]
MUL C     → ACC ← ACC × M[C]
ADD D     → ACC ← ACC + M[D]
STORE R   → M[R] ← ACC
```

需要4条一地址指令完成 $B \times C + D$ 的计算。

### 各格式效率对比

设完成 $Z = X + Y$ 运算：

| 格式 | 所需指令 | 访存次数（假设存储器操作数） |
|:---:|:---|:---|
| 三地址 | `ADD Z, X, Y` 一条 | 3次（读X,读Y,写Z） |
| 二地址 | `MOV Z, X; ADD Z, Y` 两条 | 4次 |
| 一地址 | `LOAD X; ADD Y; STORE Z` 三条 | 3次 |
| 零地址 | `PUSH X; PUSH Y; ADD; POP Z` 四条 | 4次 |

## 直观理解

### 生活类比

- **零地址**：像餐厅流水线——上一道菜已摆好，厨师直接操作栈顶两盘，不需指定
- **一地址**：像单手干活——一只手（ACC）已经拿着东西，每次只需告诉另一只手拿什么
- **二地址**：像双手操作——两只手各拿一样东西，告诉你"把右手的加到左手上"
- **三地址**：像有三只手——两只手拿操作数，第三只手放结果

### 记忆口诀

> **"零靠栈，一靠ACC，二最常见，三最自由"**

## 知识关联

### 跨章节联系
- **第5章（CPU）**：堆栈指令需要堆栈指针SP，影响微操作设计
- **第3章（存储系统）**：SS型指令多次访存影响Cache命中率和性能
- **高级语言编译**：表达式树→后序遍历→堆栈指令；中间代码→寄存器分配→RR型指令

### 常考组合
- 地址格式与操作码扩展结合（扩展操作码利用不同地址格式的地址字段位数不同）
- RISC用RR型，CISC用RS/SS型——与4.3节联合

### 易错陷阱
1. **零地址≠无操作数**：堆栈指令的操作数在栈中
2. **二地址的A1既读又写**：执行后A1原值被覆盖
3. **RR型不一定比SS型"高级"**：只是设计取舍不同
4. **一地址的"隐含"不只是ACC**：也可能是其他特殊寄存器
