# 38_转移指令

## 核心概念

### 转移指令的分类

```
转移指令
├── 无条件转移：JMP
├── 条件转移
│   ├── 基于标志位：Jcc
│   └── 基于寄存器值：JCXZ（CX=0则跳转）
└── 循环转移：LOOP、LOOPE、LOOPNE
```

### 条件码（条件转移的依据）

条件转移指令根据**标志位（条件码）**判断是否跳转：

| 条件码 | 涉及标志 | 判断含义 |
|:---|:---|:---|
| **Z（零）** | ZF | 结果是否为零 |
| **N（非零）** | ZF=0 | 结果是否非零 |
| **G（大于有符号）** | ZF=0, SF=OF | 有符号比较：大于 |
| **L（小于有符号）** | SF≠OF | 有符号比较：小于 |
| **A（高于无符号）** | CF=0, ZF=0 | 无符号比较：高于 |
| **B（低于无符号）** | CF=1 | 无符号比较：低于 |
| **O（溢出）** | OF=1 | 是否有溢出 |
| **C（有进位）** | CF=1 | 是否有进位 |

### 转移条件的完整判断

**有符号数比较 R1 与 R2**（执行 `CMP R1, R2` 即 $R1 - R2$ 后）：

| 转移条件 | 标志位判断 |
|:---|:---|
| $R1 = R2$ | $ZF = 1$ |
| $R1 \neq R2$ | $ZF = 0$ |
| $R1 > R2$ | $ZF = 0$ 且 $SF = OF$ |
| $R1 < R2$ | $SF \neq OF$ |
| $R1 \geq R2$ | $SF = OF$ |
| $R1 \leq R2$ | $ZF = 1$ 或 $SF \neq OF$ |

**无符号数比较**：

| 转移条件 | 标志位判断 |
|:---|:---|
| $R1 > R2$（高于） | $CF = 0$ 且 $ZF = 0$ |
| $R1 < R2$（低于） | $CF = 1$ |

## 原理分析

### 相对转移 vs 绝对转移

| 类型 | EA计算 | 转移范围 | 代码可重定位 |
|:---|:---|:---|:---|
| **相对转移** | $EA = PC + 偏移$ | 受限（±128B / ±32KB） | 支持 |
| **绝对转移** | $EA = $ 目标地址 | 全地址空间 | 不支持 |

现代编译器倾向于使用**相对转移**（位置无关）。

### 转移范围的计算

**8位偏移**（条件转移常用）：

$$范围 = PC_{下一条} + (-128) \sim PC_{下一条} + 127$$

**例题**：JMP在地址2000H，指令2字节，偏移=+30H。

$$PC_{下一条} = 2002H$$
$$EA = 2002H + 30H = 2032H$$

跳转范围：$2002H - 80H = 1F82H$ 到 $2002H + 7FH = 2081H$

### 编译器如何使用条件转移

**if-else结构**：

```c
if (R1 > R2) {
    // then块
} else {
    // else块
}
```

```
    CMP R1, R2
    JLE else      ; R1 <= R2 则跳到else
    ; then块代码
    JMP end
else:
    ; else块代码
end:
```

**for循环**：

```
    MOV CX, 10
loop_start:
    ; 循环体
    DEC CX
    JNZ loop_start   ; CX != 0 继续循环
```

### 易错陷阱
1. **CMP是减法但不保存结果**：只更新标志位
2. **有符号和无符号比较的条件码不同**：G vs A，L vs B
3. **8位条件转移范围有限**：±128字节
4. **转移指令本身不设标志位**：只读取已有的标志位
