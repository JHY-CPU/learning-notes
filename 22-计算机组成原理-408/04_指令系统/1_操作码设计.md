# 02_操作码设计

## 核心概念

### 操作码的定义
- **操作码（OP, Operation Code）**：指令中用于指定操作类型的编码字段
- 操作码告诉CPU"做什么"——是加法、减法、数据传送还是跳转
- 操作码经过**指令译码器**译码后，产生相应的微操作控制信号

### 操作码设计的两种基本方案

| 方案 | 特点 | 代表 |
|------|------|------|
| **定长操作码** | 所有指令OP字段位数相等 | RISC（MIPS、ARM） |
| **可变长操作码** | 不同指令OP字段位数不同 | CISC（x86、PDP-11） |

### 定长操作码

**定义**：所有指令的操作码字段位数相同，且位置固定（通常在指令的最高位）。

**优点**：
- **译码简单快速**：硬件只需固定位数即可译码
- **规整性好**：指令格式统一，便于流水线实现
- **设计和验证简单**

**缺点**：
- **编码空间浪费**：若OP占 $k$ 位，最多表示 $2^k$ 条指令，但实际可能用不满
- 灵活性差，难以在同一指令系统中混合不同地址格式的指令

**适用场景**：RISC架构，如MIPS（OP固定6位）、ARM

### 可变长操作码

**定义**：不同指令的操作码位数不同，通过"保留码点"进行扩展。

**优点**：
- **编码空间利用率高**：常用指令用短OP，不常用指令用长OP
- **灵活性强**：可在同一指令字长内支持多种地址格式

**缺点**：
- **译码复杂**：需要多位逐步判断
- **硬件设计困难**

**适用场景**：CISC架构，如x86、PDP-11

## 原理分析

### 操作码扩展技术

扩展操作码的核心思想：**用操作码中的某些特殊编码作为"前缀"，表示后续位也属于操作码。**

以16位指令字为例，假设最多三地址指令：

**第一步**：4位OP → 16条三地址指令（地址各4位）
```
0000 xxxx xxxx xxxx ~ 1110 xxxx xxxx xxxx  → 15条三地址指令
1111               → 扩展标志
```

**第二步**：1111 + 4位 → 扩展为8位OP → 15条二地址指令
```
1111 0000 xxxx xxxx ~ 1111 1110 xxxx xxxx  → 15条二地址指令（地址各4位）
1111 1111          → 扩展标志
```

**第三步**：1111 1111 + 4位 → 扩展为12位OP → 15条一地址指令
```
1111 1111 0000 xxxx ~ 1111 1111 1110 xxxx  → 15条一地址指令（地址4位）
1111 1111 1111      → 扩展标志
```

**第四步**：1111 1111 1111 + 4位 → 16位全OP → 16条零地址指令

### 扩展操作码的约束条件

1. **短码不能是长码的前缀**：保证译码无歧义
2. **保留码点**：每级扩展需保留至少一个编码作为"继续扩展"标志
3. **各地址段位数不必相等**：可灵活分配

### 与Huffman编码的关系

扩展操作码的本质与Huffman编码类似：
- **高频出现的指令（常用指令）** → 使用较短的操作码
- **低频出现的指令（不常用指令）** → 使用较长的操作码
- 目标都是：**在总位数有限的条件下，最大化编码效率**

$$平均操作码长度 = \sum_{i} p_i \times l_i$$

其中 $p_i$ 为第 $i$ 条指令的使用频率，$l_i$ 为其操作码位数。理想情况下趋近于信息熵：

$$H = -\sum_{i} p_i \log_2 p_i$$

### 计算题：求可扩展指令条数

**规则**：若某地址格式有 $m$ 位地址字段，则扩展时最多可表示 $2^m - 1$ 条指令（需保留1个码点作扩展标志）。

**例题**：16位指令，三地址各4位，求总共最多多少条指令？
- 三地址：$2^4 - 1 = 15$ 条，保留1111扩展
- 二地址：$2^4 - 1 = 15$ 条，保留1111扩展
- 一地址：$2^4 - 1 = 15$ 条，保留1111扩展
- 零地址：$2^4 = 16$ 条（无需再扩展）
- **总计**：$15 + 15 + 15 + 16 = 61$ 条

## 直观理解

### 生活类比

**邮编系统**：
- 定长操作码 = 固定用6位数字表示省市区，不管地址多简单都用6位
- 可变长操作码 = 先看前2位（省），如果需要再看中间2位（市），最后2位（区）

就像拨电话：市内拨7位，长途拨10位，国际拨13位——可变长编码更高效。

### 记忆技巧

- **定长**：整齐划一，像军队方阵——"好看但可能浪费空间"
- **可变长**：见缝插针，像拼图——"复杂但空间利用率高"
- 扩展操作码：**"用特殊编码做跳板，向下一级扩展"**

- RISC用定长、CISC用可变长——与CISC/RISC对比联合出题

### 易错陷阱
1. **扩展操作码的"保留码点"不可遗漏**：每级最多 $2^m - 1$ 条（不是 $2^m$ 条）
2. **定长操作码不等于指令条数固定**：可以用未使用的编码做其他用途
3. **操作码扩展 ≠ 操作码变长**：扩展是具体的技术方案，变长是设计思路
