# 20_变址寻址

## 核心概念

### 定义
- **变址寻址（Index Addressing）**：有效地址EA = **变址寄存器IX的内容** + **形式地址A**
- 变址寄存器的内容由**用户程序**修改

### 格式与语义

$$[OP][寻址方式标志][形式地址A]$$

$$EA = IX + A$$

$$操作数 = M[IX + A]$$

其中 $IX$ 为变址寄存器（Index Register），$A$ 为基准地址（基地址/首地址）。

### 变址寄存器的特点

| 特征 | 说明 |
|:---|:---|
| **设置者** | 用户程序（普通指令可修改） |
| **用户可否修改** | 可以 |
| **作用** | 提供**变化的偏移量**（下标） |
| **典型值** | 数组下标、循环计数器 |

### 特点

| 优点 | 缺点 |
|:---|:---|
| **适合数组和循环访问**：修改IX即可遍历 | 需要额外的IX寄存器 |
| **用户灵活控制**：可自由修改IX | 单独使用时寻址范围可能受限 |
| **指令简洁**：循环中IX自增即可 | |
| **与基址配合效果最佳**：EA = BR + IX + A | |

## 原理分析

### 变址寻址的典型应用：数组访问

设数组首地址 = A，数组元素大小 = 4字节，IX存放下标 $i$：

$$EA = IX \times 4 + A$$

或更一般地：

$$EA = IX + A$$

（若IX已预乘元素大小）

**例题**：A = 1000H（数组首地址），IX = 0, 4, 8, ...（字节偏移）

```
IX = 0  → EA = 1000H + 0  = 1000H  → arr[0]
IX = 4  → EA = 1000H + 4  = 1004H  → arr[1]
IX = 8  → EA = 1000H + 8  = 1008H  → arr[2]
...
```

循环中只需 `ADD IX, #4` 即可访问下一个元素。

### 与基址寻址的对比

| 对比项 | 基址寻址 | 变址寻址 |
|:---|:---|:---|
| **EA公式** | $EA = BR + A$ | $EA = IX + A$ |
| **BR/IX由谁设置** | OS（不可改） | 用户（可改） |
| **A的含义** | 偏移量 | 基地址 |
| **典型用途** | 程序重定位 | 数组/循环访问 |
| **谁变化** | A（程序内偏移） | IX（循环中递增） |

### 基址+变址组合寻址

$$EA = BR + IX + A$$

- BR提供程序基地址（OS设置）
- IX提供数组下标（用户设置）
- A提供数组内偏移

**例题**：BR = 5000H，IX = 200H，A = 10H

$$EA = 5000H + 200H + 10H = 5210H$$

这是x86架构中常用的寻址方式（如 `[EBX + ESI + disp]`）。

### 访存次数

$$访存次数（不含取指）= 1$$

IX在CPU寄存器中，从M[IX+A]取操作数。

## 直观理解

### 生活类比

变址寻址就像**图书馆找书**：
- 书架首地址 = A（形式地址，书架编号）
- 第几层 = IX（变址寄存器，你决定看第几层）
- 想看不同层的书？只需改变IX（上下移动）

### 记忆技巧

- **"变址 = 变化的下标 = 用户改IX"**
- **"基址管地基（OS定），变址管楼层（用户定）"**
- **基址变址区别口诀：基址OS管，变址用户改**

## 知识关联

### 跨章节联系
- **第3章（存储系统）**：数组在主存中的连续存储，Cache行预取
- **第5章（CPU）**：IX为通用寄存器，可被用户指令修改
- **高级语言编译**：`for`循环遍历数组 → 变址寻址

### 常考对比
- **基址 vs 变址**：谁设置BR/IX，A是偏移还是基址
- **变址 vs 相对寻址**：变址用IX，相对用PC

### 易错陷阱
1. **变址的IX由用户改**：这是与基址寻址的关键区别
2. **A是基地址不是偏移**：变址寻址中A是基准，IX是变化的偏移
3. **基址+变址组合时**：$EA = BR + IX + A$，三者都是加法
4. **变址寄存器可以是通用寄存器**：不一定是专用IX寄存器
