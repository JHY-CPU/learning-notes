# 11_指令字长与存储

## 核心概念

### 指令在存储器中的存放

- 指令和数据都存放在**主存储器**中
- 取指令时，CPU通过地址总线发出指令地址，从存储器读取指令
- 指令的存放位置由**程序计数器PC**决定

### 指令跨存储字边界存放

当指令字长 > 存储字长时：
- 一条指令需要占用**多个连续的存储字**
- 取指需要**多次访存**，分多次取出
- 这会增加取指时间，影响CPU效率

**例题**：存储字长8位（按字节编址），指令字长16位。

```
地址 0x00: [指令第1字节]  ← PC指向这里
地址 0x01: [指令第2字节]  ← 第二次访存取这里
地址 0x02: [下条指令第1字节] ← PC更新后
```

取一条16位指令需要两次访存。

### 边界对齐

**边界对齐**：指令和数据的起始地址满足一定的对齐规则。

常见的对齐方式：
- **字节对齐**：地址可以是任意值（无对齐要求）
- **半字对齐**：地址必须是2的倍数
- **字对齐**：地址必须是4的倍数（32位系统）
- **双字对齐**：地址必须是8的倍数（64位系统）

$$地址 \mod 对齐单位 = 0$$

### 边界对齐的好处

1. **一次访存取完整数据**：对齐的指令/数据不需要跨存储字边界
2. **硬件简化**：地址译码电路简单
3. **存储器带宽利用率高**：不需要为非对齐数据做额外处理

### 边界对齐的代价

- **存储空间浪费**：对齐填充（padding）导致空间利用率下降

**例题**：数据A占1字节，数据B占4字节。若要求4字节对齐：

```
无对齐:  [A][B0][B1][B2][B3]     → 5字节
4字节对齐: [A][pad][pad][pad][B0][B1][B2][B3] → 8字节
```

浪费了3字节的填充空间。

## 原理分析

### 取指过程中的访存次数分析

**情况1：指令字长 = 存储字长**
- 一次访存取出整条指令
- 最理想的情况

**情况2：指令字长 = 2 × 存储字长**
- 需要两次访存
- 第一次取指令的前半部分（含操作码）
- 第二次取指令的后半部分

**情况3：可变长指令**
- 需要先取操作码部分，译码确定指令长度
- 再根据长度决定是否继续取剩余部分
- 取指过程复杂，可能需要1\~多次访存

### 指令存放与取指的时序

```
时钟周期1: PC → MAR（地址寄存器）
           1 → R（读命令）
时钟周期2: M(MAR) → MDR（数据寄存器）
           MDR → IR（指令寄存器）
           如果指令字长>存储字长，继续取下一部分
时钟周期3: PC + 指令长度 → PC
           译码
```

### 计算题：存储器中指令的存放位置

**例题**：存储器按字节编址，指令字长32位（4字节），首条指令存放在地址0x0000。求第10条指令的起始地址。

**解答**：
- 每条指令占4字节
- 第10条指令（从0开始编号）起始地址：

$$地址 = 0x0000 + 10 \times 4 = 0x0028$$

若从1开始编号（第10条）：
$$地址 = 0x0000 + (10-1) \times 4 = 0x0024$$

### 指令字长、存储字长、机器字长三者关系

| 术语 | 含义 |
|:---|:---|
| **机器字长** | CPU一次能处理的二进制位数（ALU宽度、通用寄存器宽度） |
| **存储字长** | 存储器一次读写操作的数据位数（MDR宽度） |
| **指令字长** | 一条指令的二进制位数 |

三者关系：
- 通常 **存储字长 = 机器字长**（但不是必须的）
- **指令字长**是**存储字长**的整数倍（保证取指能在一个或整数个存储周期完成）
- 指令字长可以 $\neq$ 机器字长

## 直观理解

### 生活类比

**行李箱类比**：
- 存储字长 = 行李箱的容量
- 指令字长 = 你需要装的东西的总量
- 如果东西比一个箱子大，你需要两个箱子（两次访存）
- 边界对齐 = 把东西按箱子大小整齐打包，不跨箱

### 记忆技巧

- **"指令字长是存储字长的整数倍"**——保证取指规整
- **"对齐以空间换时间"**——对齐浪费空间但提升访存效率

### 常考组合
- 按字节编址 + 指令字长 + 存储字长 → 求指令地址和取指次数
- 对齐要求 + 数据大小 → 求存储空间利用率

### 易错陷阱
1. **"按字节编址"意味着存储字长 = 8位（1字节）**
2. **指令编号从0还是从1**：408通常从0开始
3. **对齐和非对齐的访存次数不同**：非对齐可能需要两次访存
4. **存储字长和机器字长可能不同**：不要混为一谈
