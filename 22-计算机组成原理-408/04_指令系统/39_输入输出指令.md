# 40_输入输出指令

## 核心概念

### I/O指令的定义
- **I/O指令**：CPU与外部设备之间进行数据交换的专用指令
- I/O操作涉及I/O端口（接口中的寄存器）

### I/O编址方式

#### 独立编址（I/O映射I/O）

**特点**：I/O端口有**独立的地址空间**，与存储器地址空间分开。

**专用I/O指令**：
| 指令 | 含义 |
|:---|:---|
| `IN AL, port` | $AL \leftarrow I/O[port]$（从端口读入） |
| `OUT port, AL` | $I/O[port] \leftarrow AL$（向端口写出） |

- x86使用独立编址
- I/O地址空间通常64KB（16位端口地址）
- 需要专门的`IN`/`OUT`指令访问

#### 统一编址（存储器映射I/O）

**特点**：I/O端口**占用存储器地址空间**的一部分，用普通存储器访问指令即可访问I/O。

**用MOV访问**：
| 指令 | 含义 |
|:---|:---|
| `MOV R1, [IO_addr]` | $R1 \leftarrow I/O[IO\_addr]$ |
| `MOV [IO_addr], R1` | $I/O[IO\_addr] \leftarrow R1$ |

- ARM、MIPS使用统一编址
- 不需要专门I/O指令
- I/O地址占用部分存储器空间

### 两种编址方式对比

| 对比项 | 独立编址 | 统一编址 |
|:---|:---|:---|
| **地址空间** | I/O和存储器独立 | I/O占用存储器空间 |
| **访问指令** | 专用IN/OUT | 通用MOV/LOAD/STORE |
| **地址线** | 需要区分I/O和存储器访问 | 无需区分 |
| **存储器空间** | 不受影响 | 被I/O占用一部分 |
| **指令功能** | I/O指令较简单 | 可用所有存储器指令访问I/O |
| **典型代表** | x86 | ARM、MIPS |

## 原理分析

### I/O指令的执行过程

**独立编址（x86 IN指令）**：

```
1. 取指：从PC取IN指令
2. 译码：识别为IN指令
3. 端口地址 → 地址总线的低16位
4. IOR（I/O读）控制信号有效
5. 外设将数据送到数据总线
6. 数据总线 → AL/AX/EAX
```

**统一编址（存储器映射I/O）**：

```
1. 取指：从PC取MOV指令
2. 译码：识别为普通MOV
3. I/O端口地址 → 地址总线
4. RD（读）控制信号有效（与存储器读相同）
5. 外设响应（不是存储器响应）
6. 数据 → 寄存器
```

### I/O指令的特点

1. **与外设速度匹配**：外设比CPU慢得多，可能需要等待或中断
2. **需要控制信号**：读写控制、地址译码、设备选择
3. **数据宽度可能不同**：8位外设、16位外设等
4. **同步/异步**：程序查询、中断、DMA

### I/O操作的三种方式

| 方式 | I/O指令的作用 | 说明 |
|:---|:---|:---|
| **程序查询** | CPU不断用IN指令查询状态 | 简单但浪费CPU |
| **中断驱动** | I/O完成触发中断 | CPU效率高 |
| **DMA** | DMA控制器直接传送 | 最快，不需CPU干预 |

## 直观理解

### 生活类比

- **独立编址**：I/O设备有自己的门牌号，要用专门的"拜访I/O"指令
- **统一编址**：I/O设备混在普通房间中，用普通"访问房间"指令即可

### 记忆技巧

- **"独立编址用IN/OUT，统一编址用MOV"**
- **"x86独立，ARM/MIPS统一"**

### 易错陷阱
1. **统一编址不需要I/O指令**：用普通存储器指令即可
2. **独立编址的I/O空间与存储器空间完全分开**：同一地址可以同时存在于两个空间
3. **IN/OUT不只是x86的**：很多CISC架构有独立I/O指令
4. **统一编址的缺点**：占用了存储器地址空间
