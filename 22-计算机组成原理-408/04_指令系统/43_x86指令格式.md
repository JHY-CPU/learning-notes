# 44_x86指令格式

## 核心概念

### x86指令格式概述

x86是**变长指令**，长度从1字节到15字节不等：

```
┌──────────┬──────────┬──────────┬──────────┬──────────┬──────────┐
│  前缀    │  操作码  │ ModR/M   │   SIB    │   偏移   │  立即数  │
│ 0~4字节  │ 1~3字节  │ 0~1字节  │ 0~1字节  │ 0~4字节  │ 0~4字节  │
└──────────┴──────────┴──────────┴──────────┴──────────┴──────────┘
```

各字段都是**可选的**（操作码除外），这使得x86指令长度极不固定。

### 各字段详解

#### 前缀（Prefix）0~4字节

| 前缀类型 | 编码 | 作用 |
|:---|:---|:---|
| 指令前缀 | 0xF3, 0xF2, 0xF0 | REP/REPN/LOCK |
| 段覆盖 | 0x2E, 0x36, 0x3E... | 指定段寄存器 |
| 操作数大小 | 0x66 | 切换16/32位操作数 |
| 地址大小 | 0x67 | 切换16/32位地址 |

#### 操作码（Opcode）1~3字节

- 主操作码：1字节（大多数指令）
- 2字节操作码：0x0F前缀 + 1字节
- 3字节操作码：0x0F + 0x38/0x3A + 1字节

#### ModR/M（0~1字节）

```
┌──────┬──────┬──────┐
│ Mod  │ Reg  │ R/M  │
│ 2位  │ 3位  │ 3位  │
└──────┴──────┴──────┘
```

- **Mod**（2位）：寻址模式
  - 00: 无偏移（特殊情况：R/M=110时为直接寻址）
  - 01: 8位偏移
  - 10: 32位偏移
  - 11: 寄存器寻址
- **Reg**（3位）：寄存器编号或操作码扩展
- **R/M**（3位）：寄存器或存储器寻址

#### SIB（Scale-Index-Base）0~1字节

```
┌──────┬──────┬──────┐
│Scale │Index │ Base │
│ 2位  │ 3位  │ 3位  │
└──────┴──────┴──────┘
```

$$EA = Base + Index \times Scale + 偏移$$

| Scale | 乘数 |
|:---:|:---|
| 00 | ×1 |
| 01 | ×2 |
| 10 | ×4 |
| 11 | ×8 |

#### 偏移（Displacement）0~4字节

- 0字节、1字节（符号扩展）或4字节

#### 立即数（Immediate）0~4字节

- 0字节、1字节、2字节或4字节

## 原理分析

### x86寻址方式（通过ModR/M + SIB）

$$EA = 基址寄存器 + 变址寄存器 \times 比例因子 + 偏移量$$

| 组合 | 示例 | EA计算 |
|:---|:---|:---|
| 寄存器 | `EAX` | 无EA（操作数在寄存器） |
| 直接寻址 | `[1234H]` | EA = 1234H |
| 寄存器间接 | `[EAX]` | EA = EAX |
| 基址+偏移 | `[EAX+8]` | EA = EAX + 8 |
| 基址+变址 | `[EAX+EBX]` | EA = EAX + EBX |
| 基址+变址+偏移 | `[EAX+EBX+4]` | EA = EAX + EBX + 4 |
| 比例变址 | `[EAX+EBX*4+8]` | EA = EAX + EBX×4 + 8 |

### x86指令编码示例

**`ADD EAX, EBX`**（$EAX \leftarrow EAX + EBX$）

```
01 D8
│  └── ModR/M: Mod=11(寄存器), Reg=011(EBX), R/M=000(EAX)
└──── 操作码: ADD (reg, r/m)
```

**`MOV EAX, [EBX+ECX*4+8]`**

```
8B 44 8B 08
│  │  │  └── 偏移: 08H
│  │  └───── SIB: Scale=10(×4), Index=001(ECX), Base=011(EBX)
│  └──────── ModR/M: Mod=01(8位偏移), Reg=000(EAX), R/M=100(SIB follows)
└─────────── 操作码: MOV r32, r/m32
```

## 直观理解

### x86 vs MIPS对比

| 对比项 | x86 | MIPS |
|:---|:---|:---|
| 指令长度 | 变长（1~15字节） | 定长（4字节） |
| 格式数量 | 无数种组合 | 3种（R/I/J） |
| 寻址方式 | 10+种 | 3~5种 |
| 设计哲学 | CISC | RISC |

### 易错陷阱
1. **x86指令长度可达15字节**：超过15字节会触发异常
2. **ModR/M的Mod=00且R/M=110时**：是直接寻址（不是[EAX+EBX]）
3. **0x66前缀切换操作数大小**：32位模式下加0x66变为16位
4. **指令各字段都是可选的**：最短指令仅1字节（如NOP = 0x90）
