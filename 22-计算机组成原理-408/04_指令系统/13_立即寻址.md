# 13_立即寻址

## 核心概念

### 定义
- **立即寻址（Immediate Addressing）**：指令的地址码部分**直接给出操作数本身**，而不是操作数的地址
- 操作数称为**立即数（Immediate）**
- 指令中不包含有效地址EA的概念

### 格式

$$[OP][寻址方式标志][立即数A]$$

- 操作数 = 形式地址 $A$（$A$ 本身就是操作数，不是地址）
- 没有有效地址EA，不需要访存取操作数

### 典型指令示例

| 指令 | 含义 | 说明 |
|:---|:---|:---|
| `MOV R1, #100` | $R1 \leftarrow 100$ | 把立即数100送入R1 |
| `ADD R1, #5` | $R1 \leftarrow R1 + 5$ | R1加立即数5 |
| `MOV R1, #0xFF` | $R1 \leftarrow 255$ | 十六进制立即数 |

### 特点

| 优点 | 缺点 |
|:---|:---|
| **速度最快**：取指令同时获得操作数 | **操作数范围受限**：受地址码位数限制 |
| **不需访存**：0次访存 | **每次使用需编码在指令中** |
| **适合常量**：固定值直接给出 | **不适合变量**：变量值运行时才知 |

## 原理分析

### 执行过程

```
1. 取指：从PC指向的存储单元取出整条指令
2. 译码：识别操作码和寻址方式（立即寻址）
3. 执行：直接从指令中提取立即数A作为操作数，送ALU运算
4. 结果写回
```

**关键**：步骤3不需要额外访存，立即数已随指令一起取出。

### 立即数的范围

设立即数字段占 $n$ 位：
- **无符号立即数**：范围 $0 \sim 2^n - 1$
- **有符号立即数（补码）**：范围 $-2^{n-1} \sim 2^{n-1} - 1$

**例题**：MIPS I型指令，立即数16位。
- 无符号：$0 \sim 65535$
- 补码：$-32768 \sim 32767$

### 立即寻址的扩展

当立即数范围不够时：
1. **符号扩展**：有符号数扩展到更宽位数（高位填充符号位）
2. **零扩展**：无符号数高位补0
3. **多条指令组合**：如MIPS的 `LUI` + `ORI` 组合加载32位立即数

### 访存次数

$$访存次数 = 0$$

操作数在指令中，取指令时一并取出，不需要额外访存。

### 计算题

**例题**：某指令 `MOV R1, #200`，若立即数占8位（无符号），是否能表示？

解答：$200 \leq 2^8 - 1 = 255$，**可以**。

**例题**：立即数占8位（补码），能表示的最大正数？

解答：$2^{8-1} - 1 = 127$。$200 > 127$，**不能**表示为8位补码。

## 直观理解

### 生活类比

立即寻址就像**直接给你钱**：
- "给你100块"——钱直接到手，不需要去银行取
- 对比直接寻址："去3号柜拿钱"——需要额外去一次银行（访存）

### 记忆技巧

- **"立即 = 马上就有 = 在指令里 = 不用去找"**
- **速度最快，但大小受限**

- **第3章（存储系统）**：0次访存 → 不涉及Cache和主存

### 常考组合
- 立即寻址 vs 直接寻址的区别（最常考的对比）

### 易错陷阱
1. **立即寻址没有EA**：不要试图找"地址"，立即数就是操作数本身
2. **"#"符号是立即寻址标志**：`#100` 表示立即数100，不是地址100
3. **立即数范围受字段位数限制**：不是任意大的数都能表示
4. **访存次数 = 0**：只算取操作数的访存，不含取指令
