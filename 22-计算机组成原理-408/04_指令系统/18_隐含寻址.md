# 18_隐含寻址

## 核心概念

### 定义
- **隐含寻址（Implicit Addressing）**：指令中**不显式给出操作数的地址**，操作数隐含在某个**约定的寄存器**中（通常是累加器ACC）
- 指令只需给出操作码和（可能的）一个地址

### 格式与语义

$$[OP][A]$$

- 操作数1：隐含在ACC中（指令不显式指定）
- 操作数2：$M[A]$（指令给出地址A）
- 结果：存回ACC

$$ACC \leftarrow ACC \text{ OP } M[A]$$

### 典型指令示例

| 指令 | 含义 |
|:---|:---|
| `ADD A` | $ACC \leftarrow ACC + M[A]$ |
| `SUB A` | $ACC \leftarrow ACC - M[A]$ |
| `LOAD A` | $ACC \leftarrow M[A]$ |
| `STORE A` | $M[A] \leftarrow ACC$ |
| `INC A` | $M[A] \leftarrow M[A] + 1$（隐含操作数=A单元本身） |
| `NOT A` | $M[A] \leftarrow \neg M[A]$ |

### 隐含寻址的两种形式

| 形式 | 说明 | 示例 |
|:---|:---|:---|
| **隐含源操作数** | 一个源操作数在ACC | `ADD A`: ACC + M[A] |
| **隐含目标地址** | 结果存入ACC | `LOAD A`: ACC ← M[A] |
| **两者都隐含** | 栈操作 | `PUSH`: M[SP] ← ACC, SP-- |

### 特点

| 优点 | 缺点 |
|:---|:---|
| **指令短**：少一个地址字段 | **灵活性差**：操作数位置固定 |
| **代码密度高**：适合单地址格式 | **ACC成为瓶颈**：频繁读写ACC |
| **硬件简单**：早期计算机常用 | **不适合现代流水线** |

## 原理分析

### 为什么隐含寻址能缩短指令

对比三地址和一地址（隐含）指令：

**三地址**：`ADD R1, R2, R3` → 需3个地址字段
**一地址（隐含）**：`ADD A` → 只需1个地址字段（ACC隐含）

$$节省的位数 = 2 \times 每个地址字段位数$$

### 隐含寻址与单地址计算机

早期计算机（如EDSAC）只有一个累加器ACC：
- 所有双目运算都以ACC为一个操作数
- 指令只需给出另一个操作数的地址
- 结果存回ACC

```
计算 Z = X + Y:
    LOAD X    ; ACC ← M[X]
    ADD Y     ; ACC ← ACC + M[Y]
    STORE Z   ; M[Z] ← ACC
```

### 访存次数

对于 `ADD A`（$ACC \leftarrow ACC + M[A]$）：

| 步骤 | 是否访存 |
|:---:|:---:|
| 从ACC取操作数1 | 否（寄存器） |
| 从M[A]取操作数2 | 是（1次） |
| **总计** | **1次** |

## 直观理解

### 生活类比

隐含寻址就像**单手操作的工人**：
- 工人一只手（ACC）始终拿着一个零件
- 每次只需告诉他从哪里拿另一个零件
- 完成操作后结果回到手里

### 记忆技巧

- **"隐含 = 不说也知道 = 约定在ACC"**
- **"指令短了，但ACC忙了"**

## 知识关联

### 跨章节联系
- **第5章（CPU）**：ACC是专用寄存器，隐含寻址简化了微操作
- **堆栈指令**：`PUSH`/`POP`也使用隐含寻址（SP隐含）
- **指令格式设计**：隐含寻址使一地址指令成为可能

### 常考对比
- 隐含寻址 vs 立即寻址：隐含操作数在寄存器 vs 在指令中
- 隐含寻址 vs 寄存器寻址：隐含寻址的寄存器是固定的（ACC），寄存器寻址可指定任意寄存器

### 易错陷阱
1. **隐含寻址的"隐含"不等于没有操作数**：操作数在约定的寄存器中
2. **隐含寻址和零地址指令不同**：零地址指令操作数在栈中（两个都隐含）
3. **隐含的寄存器不一定是ACC**：也可能是SP、PC等
4. **"隐含"只针对操作数**：操作码仍然是显式给出的
