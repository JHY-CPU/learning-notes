# 34_数据传送指令

## 核心概念

### 定义
- **数据传送指令**：将数据从一个位置复制到另一个位置
- 不改变数据的内容，只改变其位置

### 常见数据传送指令

| 指令 | 格式 | 含义 | 说明 |
|:---|:---|:---|:---|
| **MOV** | `MOV dst, src` | $dst \leftarrow src$ | 通用传送 |
| **LOAD** | `LOAD R, addr` | $R \leftarrow M[addr]$ | 存储器→寄存器 |
| **STORE** | `STORE addr, R` | $M[addr] \leftarrow R$ | 寄存器→存储器 |
| **PUSH** | `PUSH src` | $M[SP] \leftarrow src; SP \leftarrow SP \pm \Delta$ | 压栈 |
| **POP** | `POP dst` | $dst \leftarrow M[SP]; SP \leftarrow SP \pm \Delta$ | 弹栈 |
| **XCHG** | `XCHG a, b` | $a \leftrightarrow b$ | 交换 |
| **LEA** | `LEA R, addr` | $R \leftarrow addr$（地址而非内容） | 加载有效地址 |

### 传送方向

```
寄存器 ←→ 寄存器    MOV R1, R2
存储器 → 寄存器     LOAD R1, [addr] / MOV R1, [addr]
寄存器 → 存储器     STORE [addr], R1 / MOV [addr], R1
立即数 → 寄存器     MOV R1, #imm
立即数 → 存储器     MOV [addr], #imm
```

### 数据宽度

| 宽度 | x86指令后缀 | 示例 |
|:---:|:---|:---|
| 8位（字节） | B | `MOV AL, BL` |
| 16位（字） | W | `MOV AX, BX` |
| 32位（双字） | D | `MOV EAX, EBX` |
| 64位（四字） | Q | `MOV RAX, RBX` |

## 原理分析

### 数据传送指令的微操作

以 `MOV R1, [addr]` 为例（LOAD操作）：

```
1. 取指：PC → MAR → M → MDR → IR
2. 译码：识别为LOAD指令
3. 地址计算：EA = addr
4. 访存：EA → MAR → M → MDR（从存储器取数据）
5. 写回：MDR → R1
```

以 `PUSH R1` 为例：

```
1. SP ← SP - Δ（调整栈指针）
2. M[SP] ← R1（写入栈顶）
```

### XCHG的实现

交换两个寄存器的内容，可以用异或实现（不需要临时变量）：

```
XOR R1, R2    ; R1 ← R1 ⊕ R2
XOR R2, R1    ; R2 ← R2 ⊕ R1 = R2 ⊕ R1 ⊕ R2 = R1
XOR R1, R2    ; R1 ← R1 ⊕ R2 = (R1 ⊕ R2) ⊕ R1 = R2
```

## 直观理解

### 生活类比

数据传送就像**搬运**：
- MOV：把东西从A搬到B
- PUSH：往箱子里放东西
- POP：从箱子里取东西
- XCHG：两个人互换手中的东西

### 记忆技巧

- **"传送不改变值，只改变位置"**
- **"LOAD从内存到寄存器，STORE从寄存器到内存"**

### 易错陷阱
1. **MOV不是"移动"是"复制"**：源操作数不变
2. **不能MOV存储器到存储器**（x86可以，但RISC不行）
3. **PUSH/POP的SP变化方向**：递减栈PUSH时SP减
4. **LEA取的是地址不是内容**：`LEA R, [addr]` 把addr本身（不是M[addr]）给R
