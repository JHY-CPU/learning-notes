# 09_扩展操作码_408考点

## 核心概念

### 408扩展操作码考题类型

1. **验证型**：给定各级指令数量，验证扩展方案是否可行
2. **求解型**：给定部分条件，求各级最大指令数
3. **设计型**：给定需求，设计扩展方案
4. **综合型**：扩展操作码与寻址范围、存储容量结合

### 解题通用步骤

1. 确定指令字长 $L$ 和最大地址数 $m$
2. 确定每个地址字段的位数 $n$
3. 从最高地址数开始，逐级计算：
   - 当前级OP位数 = $L - \text{当前地址数} \times n$
   - 可表示指令数 = $2^{\text{OP位数}} - 1$（非末级）或 $2^{\text{OP位数}}$（末级）
   - 验证：需求量 $\leq$ 最大可表示数
4. 累加所有级的指令数

## 原理分析

### 经典题型一：给定三/二/一/零地址指令数求OP位数

**例题**：指令字长16位，每个地址字段4位，三地址指令15条，二地址指令10条，一地址指令8条，零地址指令5条。求各级OP位数，验证可行性。

**解答**：

**三地址**：需15条。
OP位数 = $16 - 3 \times 4 = 4$位。
最大可表示 = $2^4 - 1 = 15$。
$15 \leq 15$ ✓ 保留1111。

**二地址**：需10条。
OP位数 = $4 + 4 = 8$位（高4位=1111已固定，低4位可用）。
最大可表示 = $2^4 - 1 = 15$。
$10 \leq 15$ ✓ 保留1111（即1111 1111作为扩展标志）。

**一地址**：需8条。
OP位数 = $8 + 4 = 12$位。
最大可表示 = $2^4 - 1 = 15$。
$8 \leq 15$ ✓ 保留1111。

**零地址**：需5条。
OP位数 = $12 + 4 = 16$位。
最大可表示 = $2^4 = 16$。
$5 \leq 16$ ✓

**结论：方案可行。** 总共 $15 + 10 + 8 + 5 = 38$ 条指令。

### 经典题型二：求某级最大指令数

**例题**：指令字长16位，三地址指令15条，二地址指令若干条（最多？），一地址指令12条，零地址指令16条。地址各4位。

**解答**：

**三地址**：$2^4 - 1 = 15$ 条。保留1111。✓

**二地址**：OP扩展8位，可用低4位。但需给一/零地址留扩展码。
- 一地址需OP至少 $\lceil \log_2 13 \rceil = 4$ 位（12+1保留），所以二地址的扩展位必须保留1111。
- 最多 $2^4 - 1 = 15$ 条。

**一地址**：已保留1111给零地址。
- $2^4 - 1 = 15$，但题目说12条，实际有 $15 - 12 = 3$ 个空闲码点。
- 保留1111。✓

**零地址**：$2^4 = 16$ 条。✓

**答案：二地址最多15条。**

### 经典题型三：给定指令总数求各地址格式指令数

**例题**：指令字长16位，操作码4位，地址各4位，总共需要40条指令，其中必须包含三地址、二地址、一地址、零地址四种格式。问各格式至少几条？

**解答**：

每种至少1条，剩余 $40 - 4 = 36$ 条可自由分配。

设各级为 $n_3, n_2, n_1, n_0$，约束条件：
- $n_3 + n_2 + n_1 + n_0 = 40$
- $n_3 \leq 15$（三地址最多15条）
- $n_2 \leq 15$
- $n_1 \leq 15$
- $n_0 \leq 16$

由于各上限之和 $15+15+15+16 = 61 \geq 40$，方案可行。

**至少各1条**是可行的（最小分配）。

### 经典题型四：扩展方案不可行的判断

**例题**：指令字长16位，需60条三地址指令。地址各4位，是否可行？

**解答**：
三地址OP = 4位，最多 $2^4 - 1 = 15$ 条。
$60 > 15$，**不可行！**

需要扩大OP位数或扩大指令字长。若三地址各3位，OP = 7位，$2^7 - 1 = 127 \geq 60$，但寻址范围 $2^3 = 8$ 太小。

## 直观理解

### 解题检查清单

- [ ] 每级非末级保留1个码点了吗？
- [ ] 末级不需要保留码点吗？
- [ ] 地址字段位数整除了吗？（指令字长 = OP + 地址数 × 地址位数）
- [ ] 各级需求量 ≤ 最大可表示数吗？

### 记忆口诀

> **"先算三地址（OP最短），逐级往下推，每级减一留码，末级全用光"**

## 知识关联

### 与408其他章节联系
- **存储器容量计算**：地址位数 → 存储单元数
- **指令流水线**：可变长OP → 译码阶段延迟不确定

### 易错陷阱汇总
1. **非末级少用1个码点**：$2^n - 1$ 不是 $2^n$
2. **末级不需保留**：$2^n$ 全部可用
3. **扩展位的位数**：由地址字段决定，不是任意的
4. **总指令数 ≤ 各级最大之和**：不是简单的乘法
5. **指令字长必须整除**：$L = OP + m \times n$，各字段必须整数位
