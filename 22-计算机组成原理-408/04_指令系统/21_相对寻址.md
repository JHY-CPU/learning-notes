# 21_相对寻址

## 核心概念

### 定义
- **相对寻址（PC-Relative Addressing）**：有效地址EA = **PC的内容** + **形式地址A（偏移量）**
- 以当前指令的位置为基准，用偏移量指定目标地址

### 格式与语义

$$[OP][寻址方式标志][偏移量A]$$

$$EA = PC + A$$

其中 $A$ 为**有符号偏移量**（补码表示，可正可负）。

### 关于PC的值（408重点！）

**关键问题**：PC中是当前指令的地址还是下一条指令的地址？

**标准答案**：在取指阶段，CPU先将PC的内容送到MAR取指令，然后PC自动更新（$PC \leftarrow PC + 指令长度$）。因此，**相对寻址中使用的PC值是"当前指令的下一条指令地址"**（即PC已自增后的值）。

$$EA = (PC)_{取指后} + A = PC_{下一条} + A$$

### 偏移量A的范围

设A占 $n$ 位（补码）：

$$A \in [-2^{n-1}, 2^{n-1} - 1]$$

| A的位数 | 偏移范围 | 典型用途 |
|:---:|:---|:---|
| 8位 | -128 ~ +127字节 | 短跳转 |
| 16位 | -32768 ~ +32767字节 | 中距离跳转 |
| 32位 | ±2GB | 长跳转 |

### 特点

| 优点 | 缺点 |
|:---|:---|
| **代码可重定位**：不依赖绝对地址 | **跳转范围受限**：受偏移量位数限制 |
| **指令短**：只需偏移量，不需要完整地址 | 偏移量计算需要知道指令位置 |
| **位置无关代码(PIC)**：程序可加载到任意位置 | |

## 原理分析

### 相对寻址的应用

**1. 条件转移指令**

```
CMP R1, R2       ; 比较R1和R2
JE +8            ; 相等则跳转，偏移+8字节
...              ; 不跳转时执行这里
偏移8字节处:      ; 跳转目标
```

**2. 无条件转移（短跳转）**

```
JMP -16          ; 向前跳转16字节（循环回跳）
```

**3. 过程调用的返回地址**

CALL指令用相对寻址确定跳转目标，同时保存返回地址。

### 相对寻址的程序浮动

程序从地址0x1000搬到0x5000：
- **绝对寻址**：所有跳转目标需要重新计算 → 需要重定位
- **相对寻址**：偏移量不变，PC自动调整 → 无需重定位

```
原: PC=0x1004, JMP +8 → EA=0x100C
新: PC=0x5004, JMP +8 → EA=0x500C
偏移量始终是+8，不变！
```

### 访存次数

**对于数据寻址**（用EA取操作数）：

$$访存次数（不含取指）= 1$$

从M[PC+A]取操作数。

**对于指令寻址**（跳转指令）：

不额外访存——跳转只是修改PC的值，下一条取指自动用新PC。

### 计算题

**例题**：某指令在地址2000H处，指令长度4字节，偏移量A = 06H（补码，+6）。求跳转目标地址。

$$PC_{下一条} = 2000H + 4 = 2004H$$
$$EA = 2004H + 06H = 200AH$$

**例题**：同上，A = FAH（补码 = -6）。

$$EA = 2004H + (-6) = 2004H - 6 = 1FFEH$$

## 直观理解

### 生活类比

相对寻址就像**"往前走3个路口"**：
- 不说具体的街道名（绝对地址）
- 只说相对当前位置的偏移
- 不管你从哪里出发，"3个路口"的操作是一样的

### 记忆技巧

- **"相对 = PC + 偏移 = 以当前为基准"**
- **"PC是下一条指令地址"——408必记！**
- **"偏移量可正可负（补码），向前向后都能跳"**

## 知识关联

### 跨章节联系
- **第2章（数据表示）**：偏移量用补码表示
- **第3章（存储系统）**：位置无关代码便于动态加载
- **第5章（CPU）**：转移指令修改PC，需刷新流水线

### 常考对比
- **相对寻址 vs 基址寻址**：相对用PC，基址用BR；相对用于指令跳转，基址用于数据重定位
- **相对寻址 vs 直接寻址**：相对寻址的EA随PC变化，直接寻址的EA固定

### 易错陷阱
1. **PC值是"下一条指令地址"**：不是当前指令地址！
2. **偏移量是补码**：可正可负，注意符号扩展
3. **"相对"是对指令寻址而言**：转移目标相对于PC
4. **偏移量的单位**：通常按字节计算（不是按指令条数）
