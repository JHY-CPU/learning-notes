# 14_直接寻址

## 核心概念

### 定义
- **直接寻址（Direct Addressing）**：指令地址码中的形式地址 $A$ 就是操作数的**有效地址EA**
- 操作数存放在存储器中，地址 $A$ 直接指定位置

### 格式与语义

$$[OP][寻址方式标志][形式地址A]$$

$$EA = A$$

$$操作数 = M[EA] = M[A]$$

### 典型指令示例

| 指令 | 含义 |
|:---|:---|
| `MOV R1, [1000H]` | $R1 \leftarrow M[1000H]$，从地址1000H取操作数 |
| `ADD R1, [2000H]` | $R1 \leftarrow R1 + M[2000H]$ |
| `STORE [3000H], R1` | $M[3000H] \leftarrow R1$ |

### 特点

| 优点 | 缺点 |
|:---|:---|
| **简单直观**：地址直接给出 | **寻址范围受限**：受形式地址A的位数限制 |
| **译码快速**：不需要计算 | **地址固定**：难以处理动态地址 |
| **一次访存**：只需一次额外访存 | **指令较长**：需要完整地址位数 |

## 原理分析

### 执行过程

```
1. 取指：从PC指向的存储单元取出整条指令
2. 译码：识别操作码和寻址方式（直接寻址）
3. 取数：以形式地址A作为EA，访问存储器M[EA]，取出操作数
4. 执行：在ALU中进行运算
5. 结果写回
```

### 寻址范围

设形式地址A占 $n$ 位：

$$寻址范围 = 2^n \text{ 个存储单元}$$

**例题**：A占16位，按字节编址。

$$寻址范围 = 2^{16} = 64KB$$

若主存容量超过64KB，直接寻址无法访问全部存储空间。

### 访存次数分析

| 步骤 | 是否访存 | 说明 |
|:---:|:---:|:---|
| 取指 | 是 | 从PC取指令 |
| 取操作数 | 是 | 从A指向的地址取操作数 |
| **总计（不含取指）** | **1次** | 从EA取操作数 |

$$访存次数（不含取指）= 1$$

### 直接寻址的局限性

**问题**：形式地址A的位数有限，导致：
1. 若主存64MB，地址需要24位，但A可能只有16位——寻址范围不够
2. 解决方案：使用基址寻址、变址寻址等间接方式扩展寻址范围

### 计算题

**例题**：指令字长32位，操作码8位，采用直接寻址，问：
1. 形式地址A占多少位？
2. 寻址范围多大？

**解答**：
1. $A = 32 - 8 = 24$位
2. $2^{24} = 16MB$（按字节编址）

## 直观理解

### 生活类比

直接寻址就像**快递地址**：
- "请送到XX路XX号"——直接告诉快递员目的地
- 对比立即寻址："东西就在这里"（不用送）
- 对比间接寻址："去找张三问地址"（多一步）

### 记忆技巧

- **"直接 = 地址直接告诉你 = A就是EA"**
- **一次访存（从EA取数据），简单但范围受限**

## 知识关联

### 跨章节联系
- **第3章（存储系统）**：1次访存，涉及Cache命中/缺失
- **第5章（CPU）**：取指→译码→取数的微操作流程
- **高级语言**：全局变量的访问通常用直接寻址

### 常考对比
- **直接寻址 vs 立即寻址**：立即寻址的操作数在指令中（0次访存），直接寻址的操作数在存储器中（1次访存）
- **直接寻址 vs 间接寻址**：直接寻址1次访存，间接寻址2次访存但范围更大

### 易错陷阱
1. **A是地址不是数据**：和立即寻址的区别就在于A是地址还是数据
2. **寻址范围 = $2^{A位数}$**：不是A位数本身
3. **"直接"二字的含义**：不是"简单"，而是"直接用A作为地址"
4. **访存次数不含取指**：直接寻址取操作数需1次额外访存
