# 23_段寻址

## 核心概念

### 定义
- **段寻址（Segment Addressing）**：将主存划分为若干**逻辑段**，用**段基址 + 段内偏移**来确定有效地址
- 是x86架构的经典寻址方式

### 格式与语义

$$EA = 段基址 \times 16 + 段内偏移$$

或更一般地：

$$EA = 段基址 + 段内偏移$$

x86实模式下：

$$物理地址 = 段寄存器值 \times 16 + 偏移量$$

$$= 段寄存器值 \times 10H + 偏移量$$

### 段寄存器

x86架构有多个段寄存器：

| 段寄存器 | 用途 |
|:---|:---|
| CS（Code Segment） | 代码段基址 |
| DS（Data Segment） | 数据段基址 |
| SS（Stack Segment） | 堆栈段基址 |
| ES（Extra Segment） | 附加段基址 |
| FS, GS | 额外段寄存器（386+） |

### 实模式 vs 保护模式

| 特征 | 实模式 | 保护模式 |
|:---|:---|:---|
| **地址计算** | 段基址 × 16 + 偏移 | 段选择子 → 段描述符 → 段基址 + 偏移 |
| **寻址空间** | 1MB（20位地址） | 4GB（32位地址） |
| **保护机制** | 无 | 有（界限检查、权限检查） |
| **多任务** | 不支持 | 支持 |

## 原理分析

### 实模式下的地址计算

**公式**：$物理地址 = 段基址 \times 16 + 偏移量$

**例题**：CS = 1234H，IP = 5678H

$$物理地址 = 1234H \times 10H + 5678H = 12340H + 5678H = 179B8H$$

验证：$12340H + 5678H = 179B8H$

**注意**：
- 段基址 × 16 = 左移4位（乘以16 = $2^4$）
- 20位地址可寻址 $2^{20} = 1MB$

### 保护模式下的地址转换

```
逻辑地址(段选择子:偏移)
    │
    ▼
段选择子 → 查段描述符表 → 段描述符
    │                       │
    │                  段基址(32位)
    │                       │
    ▼                       ▼
        物理地址 = 段基址 + 偏移
```

### 段的属性

每个段有以下属性（保护模式下）：
- **段基址**：段在主存中的起始地址
- **段界限**：段的最大长度
- **访问权限**：可读/可写/可执行
- **特权级**：DPL（0\~3级）

$$偏移 \leq 段界限 \Rightarrow 访问合法$$
$$偏移 > 段界限 \Rightarrow 越界异常$$

### 访存次数

| 步骤 | 访存 |
|:---|:---:|
| 查段描述符表 | 1次（页表可能更多） |
| 从EA取操作数 | 1次 |
| **总计** | **2+次**（保护模式） |

实模式下：1次（直接计算EA后访存）。

## 直观理解

### 生活类比

段寻址就像**大型商场的楼层+柜台号**：
- 段基址 = 楼层的起始位置
- 段内偏移 = 柜台号
- 物理位置 = 楼层起始 + 柜台号

### 记忆技巧

- **"段 = 分区管理 = 基址+偏移"**
- **"实模式：段值×16+偏移，左移4位"**
- **"保护模式：段选择子→描述符→基址"**

### 易错陷阱
1. **实模式下是"×16"不是"×10"**：16是十进制，10H是十六进制
2. **同一物理地址可用不同段:偏移表示**：如 0100:0200 = 0120:0000 = 01200H
3. **保护模式比实模式多一次访存**：查段表
4. **段大小可以不同**：段可以是变长的（与分页不同）
