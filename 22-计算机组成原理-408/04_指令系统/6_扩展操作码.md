# 07_扩展操作码

## 核心概念

### 定义
- **扩展操作码**：利用地址字段中未使用的编码位来扩展操作码长度，使不同地址格式的指令具有不同长度的操作码
- 核心思想：**牺牲部分地址空间，换取更多的操作码编码空间**

### 基本原理

在**固定指令字长**的条件下：
- 三地址指令：地址字段多 → 操作码短 → 可表示指令数少
- 零地址指令：无地址字段 → 操作码最长 → 可表示指令数多

通过**逐级扩展**，在地址字段减少的指令中，用更多的位来表示操作码。

### 扩展操作码的设计规则

1. **保留规则**：每级扩展必须至少保留一个编码（通常是全1），用于标志"继续扩展"
2. **无前缀规则**：短操作码不能是长操作码的前缀（类似前缀码的要求）
3. **自顶向下设计**：先设计地址最多的指令，再逐级向零地址扩展

## 原理分析

### 扩展操作码的详细推导

**设定**：指令字长 $L$ 位，最多三地址格式，每个地址字段 $n$ 位，操作码初始 $k$ 位。

$$L = k + 3n \Rightarrow k = L - 3n$$

**第一级——三地址指令**：
- OP占 $k$ 位
- 编码范围：$00\cdots0 \sim 11\cdots10$（保留全1）
- 可表示 $2^k - 1$ 条三地址指令
- 全1编码表示"这不是三地址指令，继续看下一级"

**第二级——二地址指令**：
- OP = 前 $k$ 位全1 + 新 $n$ 位
- 总OP长度 = $k + n$ 位
- 地址共 $2n$ 位（两个地址各 $n$ 位）
- 可表示 $2^n - 1$ 条二地址指令

**第三级——一地址指令**：
- OP = 前 $k+n$ 位含全1 + 新 $n$ 位
- 总OP长度 = $k + 2n$ 位
- 地址共 $n$ 位
- 可表示 $2^n - 1$ 条一地址指令

**第四级——零地址指令**：
- OP = 全部 $L$ 位
- 可表示 $2^n$ 条零地址指令（无需保留扩展码点）

### 扩展操作码与Huffman编码的关系

Huffman编码的核心原则：
- 高频元素用短码，低频元素用长码
- 前缀码保证无歧义译码

扩展操作码完全遵循这些原则：
- **三地址指令（最常用）** → OP最短
- **零地址指令（最少用）** → OP最长
- **保留码点** → 保证前缀码特性（无歧义）

不同点：
- Huffman编码的位数可以任意分配，扩展操作码的位数受地址字段约束
- 扩展操作码是**等长地址字段内的均匀扩展**，不如Huffman编码灵活

### 最大指令数量计算

设指令字长 $L$，三地址各 $n$ 位，操作码 $k = L - 3n$ 位：

$$N_{max} = (2^k - 1) + (2^n - 1) + (2^n - 1) + 2^n = 2^k + 3 \times 2^n - 3$$

**例题**：$L = 16$，$n = 4$，$k = 16 - 12 = 4$：

$$N_{max} = 2^4 + 3 \times 2^4 - 3 = 16 + 48 - 3 = 61$$

## 直观理解

### 生活类比——电话号码系统

- **国际区号**（短，如+86）→ 对应三地址指令（OP短）
- **城市区号**（中等，如021）→ 对应二地址指令
- **本地号码**（长，7\~8位）→ 对应一地址指令
- **分机号**（最长）→ 对应零地址指令

越常用（市话）编码越短，越不常用（国际长途）编码越长。

### 易错陷阱
1. **每级保留1个码点**：不是 $2^n$ 而是 $2^n - 1$
2. **全1是保留码**：从最小编码（全0）开始数，到全1前一个
3. **零地址不需要保留**：最后一级可全部使用
4. **扩展不一定是均匀的**：各地址字段位数可以不同（408一般默认相等）
