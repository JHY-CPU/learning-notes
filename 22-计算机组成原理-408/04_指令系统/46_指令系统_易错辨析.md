# 47_指令系统_易错辨析

## 核心概念

### 本章易混淆知识点汇总

本章是408中概念密集、容易混淆的章节。以下逐一辨析最常见易错点。

## 原理分析

### 易错点1：立即寻址 vs 直接寻址

| 对比项 | 立即寻址 | 直接寻址 |
|:---|:---|:---|
| 地址码A的含义 | **操作数本身** | **操作数的地址** |
| EA | 无EA | EA = A |
| 操作数来源 | 指令中 | 存储器M[A] |
| 访存次数 | 0 | 1 |
| 典型写法 | `MOV R1, #100` | `MOV R1, [100]` |

**易错**：`#100`是立即数100，`[100]`是地址100处的内容。

### 易错点2：基址寻址 vs 变址寻址

| 对比项 | 基址寻址 | 变址寻址 |
|:---|:---|:---|
| EA公式 | $EA = BR + A$ | $EA = IX + A$ |
| BR/IX由谁设置 | **OS（特权指令）** | **用户程序** |
| A的含义 | **偏移量** | **基地址** |
| 典型用途 | 程序重定位 | 数组/循环访问 |
| 谁变化 | A（程序内偏移） | IX（循环中递增） |

**易错**：
- 基址寻址中BR用户不可改，变址寻址中IX用户可改
- 基址中A是偏移，变址中A是基地址（含义相反！）

### 易错点3：相对寻址的PC值

**关键问题**：相对寻址中的PC是当前指令地址还是下一条指令地址？

**正确答案**：PC中存放的是**下一条指令的地址**（取指后PC已自增）。

$$EA = PC_{取指后} + A = (当前指令地址 + 指令长度) + A$$

**易错**：以为PC是当前指令地址。实际上取指时PC已更新。

**例题**：指令在2000H，长度4字节，偏移+8H。

- 错误：$EA = 2000H + 8H = 2008H$
- 正确：$EA = 2004H + 8H = 200CH$

### 易错点4：间接寻址 vs 寄存器间接寻址

| 对比项 | 间接寻址 | 寄存器间接寻址 |
|:---|:---|:---|
| EA来源 | $EA = M[A]$ | $EA = R$ |
| 访存次数 | **2次** | **1次** |
| 间接地址在哪 | 存储器 | 寄存器 |

**易错**：间接寻址需要2次访存（先取EA，再取操作数），寄存器间接只需1次。

### 易错点5："访存次数"的定义

**定义**：获取操作数所需的访问存储器的次数。

- **不含**取指令的访存
- **含**间接寻址中取EA的访存

| 寻址方式 | 访存次数 |
|:---|:---:|
| 立即寻址 | 0 |
| 直接寻址 | 1 |
| 一次间接寻址 | 2 |
| 寄存器寻址 | 0 |
| 寄存器间接 | 1 |
| 基址/变址/相对 | 1 |

### 易错点6：操作码位数 vs 指令条数

| 公式 | 适用场景 |
|:---|:---|
| $指令条数 = 2^k$ | 定长操作码，$k$位，无保留 |
| $指令条数 \leq 2^k - 1$ | 定长操作码，需保留扩展码点 |
| 按扩展方案计算 | 可变长操作码 |

**易错**：$k$位操作码不是恰好$2^k$条指令（可能用不满）。

### 易错点7：指令字长 vs 机器字长 vs 存储字长

| 术语 | 含义 | 关系 |
|:---|:---|:---|
| 机器字长 | CPU一次处理的位数 | ALU宽度 |
| 存储字长 | 存储器一次读写的位数 | MDR宽度 |
| 指令字长 | 一条指令的位数 | 通常为存储字长整数倍 |

**易错**：三者不一定相等。指令字长是存储字长的整数倍，但不一定是机器字长。

### 易错点8：隐含寻址 vs 零地址指令

| 对比项 | 隐含寻址 | 零地址指令 |
|:---|:---|:---|
| 操作数 | 一个隐含在ACC | 两个都隐含在栈中 |
| 指令格式 | 一地址（OP + A） | 零地址（仅OP） |
| 典型指令 | `ADD A`（ACC + M[A]） | `ADD`（栈顶两元素相加） |

**易错**：隐含寻址是一地址指令（一个操作数隐含），零地址指令是两个操作数都隐含。

### 易错点9：CISC和RISC的绝对化说法

以下说法都是**错误的**：
- "CISC一定用微程序控制"——主流是，但不是绝对
- "RISC不能有复杂指令"——可以有，只是数量少
- "CISC一定比RISC慢"——现代x86内部已经RISC化
- "RISC不需要编译器优化"——恰恰相反，RISC非常依赖编译器

### 易错点10：条件转移的有符号/无符号

| 比较类型 | 用的条件码 | 大于指令 | 小于指令 |
|:---|:---|:---|:---|
| **有符号** | SF, OF | JG（大于） | JL（小于） |
| **无符号** | CF | JA（高于） | JB（低于） |

**易错**：有符号和无符号比较使用不同的条件码和跳转指令。

## 直观理解

### 速查辨析表

| 容易混淆的对 | 区分关键 |
|:---|:---|
| 立即 vs 直接 | A是数据 vs A是地址 |
| 基址 vs 变址 | BR由OS设 vs IX由用户改 |
| 间接 vs 寄存器间接 | M[A]取EA（2次访存） vs R取EA（1次访存） |
| 隐含 vs 零地址 | 一个隐含 vs 两个隐含 |
| 相对寻址PC值 | 取指后PC已更新 |
| CF vs OF | 无符号溢出 vs 有符号溢出 |

## 知识关联

### 总结记忆

> **"立即数在指令，直接地址在A；间接地址存存储，寄存间接R当家；基址BR不可改，变址IX用户来；相对PC是下条，堆栈SP指栈顶"**
