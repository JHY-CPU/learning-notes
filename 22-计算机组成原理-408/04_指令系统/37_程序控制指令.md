# 37_程序控制指令

## 核心概念

### 定义
- **程序控制指令**：改变程序执行顺序的指令
- 通过修改PC（程序计数器）来实现跳转

### 分类

| 类别 | 指令 | 功能 |
|:---|:---|:---|
| **无条件转移** | JMP | 无条件跳转到目标地址 |
| **条件转移** | Jcc（JE/JNE/JG/JL等） | 满足条件则跳转 |
| **循环控制** | LOOP | CX减1，非零则跳转 |
| **过程调用** | CALL | 调用子程序（保存返回地址） |
| **过程返回** | RET | 从子程序返回 |

### 无条件转移 JMP

```
JMP target
PC ← target（EA）
```

- 直接跳转：目标地址在指令中
- 间接跳转：目标地址在寄存器/存储器中

### 条件转移 Jcc

| 指令 | 条件 | 含义 |
|:---|:---|:---|
| **JE/JZ** | ZF = 1 | 相等/为零则跳转 |
| **JNE/JNZ** | ZF = 0 | 不相等/非零则跳转 |
| **JG/JNLE** | ZF=0 且 SF=OF | 大于（有符号）则跳转 |
| **JL/JNGE** | SF ≠ OF | 小于（有符号）则跳转 |
| **JA/JNBE** | CF=0 且 ZF=0 | 高于（无符号）则跳转 |
| **JB/JNAE** | CF = 1 | 低于（无符号）则跳转 |
| **JGE/JNL** | SF = OF | 大于等于（有符号） |
| **JLE/JNG** | ZF=1 或 SF≠OF | 小于等于（有符号） |

### 循环 LOOP

```
LOOP label
CX ← CX - 1
if CX ≠ 0 then PC ← label
```

### CALL和RET

```
CALL label:
    PUSH (PC)        ; 保存返回地址（下一条指令地址）
    PC ← label       ; 跳转到子程序

RET:
    POP PC            ; 弹出返回地址到PC
```

## 原理分析

### 条件转移的判断逻辑

条件转移基于**标志位**判断，而标志位由之前的算术/逻辑运算设置：

```
CMP R1, R2      ; R1 - R2，设标志位（结果不保存）
JG label        ; 若ZF=0且SF=OF（即R1 > R2有符号），则跳转
```

**有符号比较**：用SF和OF判断
- $R1 > R2$：$ZF = 0$ 且 $SF = OF$
- $R1 < R2$：$SF \neq OF$

**无符号比较**：用CF判断
- $R1 > R2$：$CF = 0$ 且 $ZF = 0$
- $R1 < R2$：$CF = 1$

### 转移范围

**相对转移**：$EA = PC + 偏移量$（补码）
- 偏移量8位：-128 ~ +127字节
- 偏移量16位：-32768 ~ +32767字节

**绝对转移**：$EA = $ 目标地址（直接给出）
- 范围取决于地址宽度

## 直观理解

### 生活类比

- **JMP**：走到另一个房间（不管怎样都走）
- **Jcc**：如果条件满足才走到另一个房间
- **LOOP**：重复做N次
- **CALL**：去另一个房间干活，记住回来的路
- **RET**：干完活按记的路回来

### 记忆技巧

- **"JMP无脑跳，Jcc看标志，LOOP减CX，CALL记住回，RET回到原"**
- **"CMP设标志，Jcc查标志"**

### 易错陷阱
1. **CMP不保存结果**：只设标志位
2. **有符号和无符号比较用不同条件码**：JG vs JA
3. **LOOP的退出条件是CX=0**：执行LOOP时先减再判断
4. **CALL保存的是"下一条指令"的地址**：不是当前指令
