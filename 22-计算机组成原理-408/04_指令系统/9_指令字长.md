# 10_指令字长

## 核心概念

### 指令字长的定义
- **指令字长**：一条指令所包含的二进制位数
- 指令字长决定了每条指令在存储器中占用的空间大小
- 指令字长 = 操作码位数 + 地址码位数（含各扩展位）

### 指令字长与机器字长的关系

| 关系 | 说明 | 典型例子 |
|:---:|:---|:---|
| 指令字长 = 机器字长 | 称为**单字长指令** | 早期计算机 |
| 指令字长 = 2 × 机器字长 | 称为**双字长指令** | 复杂指令 |
| 指令字长 = 半 × 机器字长 | 称为**半字长指令** | 短指令 |

- 现代计算机中，指令字长通常是**机器字长的整数倍**
- 也可能是**存储字长的整数倍**

### 定长指令 vs 可变长指令

| 特征 | 定长指令 | 可变长指令 |
|:---|:---|:---|
| **指令字长** | 所有指令等长 | 不同指令长度不同 |
| **代表** | MIPS（32位）、ARM（32位） | x86（1~15字节） |
| **取指** | 简单，固定取n字节 | 复杂，需先译码确定长度 |
| **对齐** | 天然对齐 | 可能跨存储字边界 |
| **空间效率** | 可能浪费 | 高效 |
| **流水线** | 友好 | 复杂 |

### 指令字长的选择考量

1. **指令功能需求**：功能越丰富，需要的编码空间越大
2. **寻址范围需求**：寻址范围越大，地址码位数越多
3. **存储空间效率**：指令字长越短，程序占用空间越小
4. **取指效率**：指令字长影响取指时间和访存次数

## 原理分析

### 指令字长与取指时间

**核心约束**：取指令需要访问主存。

- 若指令字长 $\leq$ 存储字长：一次访存可取出完整指令
- 若指令字长 > 存储字长：需要**多次访存**才能取出一条指令

$$取指时间 \propto \left\lceil \frac{指令字长}{存储字长} \right\rceil$$

**例题**：存储字长16位，指令字长32位，取一条指令需几次访存？

$$\left\lceil \frac{32}{16} \right\rceil = 2 \text{ 次}$$

### 指令字长对PC的影响

PC（程序计数器）需要能表示主存中所有指令的地址。

若按字节编址，指令可能为1字节、2字节、4字节等：
- 定长指令：PC每次增加固定值（如4）
- 可变长指令：PC每次增加当前指令的长度（需译码后确定）

$$PC_{新} = PC_{旧} + 指令长度$$

### 指令字长与存储字长的关系

**关键要求**：取指令操作必须在**一个存储周期**内完成。

因此：
- 指令字长一般 $\leq$ 存储字长（或整数倍）
- 这样才能保证每条指令在一个（或整数个）存储周期内取出
- 若不满足，需多次取指，严重影响性能

**408考试中的常见假设**：
- 指令字长 = 存储字长（最常见）
- 指令字长 = 机器字长（等价于存储字长）

## 直观理解

### 生活类比

指令字长就像**信封的大小**：
- **定长指令**：所有信封一样大，小卡片也要用大信封——浪费但整齐
- **可变长指令**：信封按需选择大小——灵活但分拣麻烦
- **指令字长 > 存储字长**：一封信装不下，需要两个信封——取信时要收两次

### 记忆技巧

- **"取指要一次搞定"**：指令字长不应超过存储字长太多
- **"定长好流水，变长省空间"**

### 常考组合
- 指令字长 + 存储字长 → 取指访存次数
- 指令字长 + 操作码位数 → 各地址字段位数

### 易错陷阱
1. **指令字长不一定是机器字长**：可以是整数倍
2. **取指访存次数向上取整**：$\lceil 指令字长 / 存储字长 \rceil$
3. **可变长指令的PC更新**：不是简单加固定值
4. **指令字长与数据字长可能不同**：指令和数据可能有不同的宽度要求
