# 03_地址码设计

## 核心概念

### 地址码的定义
- **地址码（Address Code）**：指令中用于指明操作数位置和结果存放位置的字段
- 地址码回答"操作数在哪里"和"结果放到哪里"的问题
- 地址码可以是**存储器地址**、**寄存器编号**或**隐含操作数**

### 地址格式分类

根据地址码中地址字段的个数，指令格式分为：

| 地址格式 | 地址字段数 | 典型形式 | 说明 |
|:---:|:---:|:---:|:---|
| **三地址** | 3个 | OP A1, A2, A3 | A1=结果, A2=操作数1, A3=操作数2 |
| **二地址** | 2个 | OP A1, A2 | A1=结果/操作数1, A2=操作数2 |
| **一地址** | 1个 | OP A1 | 单目运算或隐含另一操作数 |
| **零地址** | 0个 | OP | 操作数全部隐含（堆栈型/空操作） |

### 三地址指令

**格式**：`OP A1, A2, A3`

**语义**：$M[A1] \leftarrow M[A2] \text{ OP } M[A3]$

**特点**：
- 源操作数和结果分别在不同地址，**结果不破坏源操作数**
- 表达式编译直接，一条指令完成一次运算
- 指令字长需求最大，若指令字长固定，每个地址字段位数较少，寻址范围受限

**例子**：`ADD R1, R2, R3` → $R1 \leftarrow R2 + R3$

### 二地址指令

**格式**：`OP A1, A2`

**语义**：$M[A1] \leftarrow M[A1] \text{ OP } M[A2]$

**特点**：
- A1既是源操作数又是结果存放位置，**结果会覆盖A1的原值**
- 比三地址少一个字段，指令更短
- 是最常见的格式，x86主要使用二地址格式

**例子**：`ADD R1, R2` → $R1 \leftarrow R1 + R2$

### 一地址指令

**格式**：`OP A1`

**语义**（单目运算）：$M[A1] \leftarrow \text{OP } M[A1]$

**语义**（双目运算，另一操作数隐含）：$ACC \leftarrow ACC \text{ OP } M[A1]$

**特点**：
- 另一操作数隐含在累加器ACC中
- 指令很短，适合简单硬件
- 早期单累加器计算机常用

**例子**：`INC A1` → $M[A1] \leftarrow M[A1] + 1$；`ADD A1` → $ACC \leftarrow ACC + M[A1]$

### 零地址指令

**格式**：`OP`

**语义**：
- **堆栈型**：操作数隐含在栈顶，如 `ADD` → 弹出栈顶两个元素相加后压入
- **空操作**：`NOP`，不进行任何操作
- **特殊指令**：如停机 `HLT`、置标志等

**特点**：
- 指令最短，但功能受限
- 需要配合堆栈或其他隐含机制使用

## 原理分析

### 地址格式的选择权衡

设指令字长为 $L$ 位，操作码占 $k$ 位，有 $m$ 个地址字段，每个地址字段 $n$ 位：

$$L = k + m \times n$$

$$n = \frac{L - k}{m}$$

- 地址字段数 $m$ 越多 → 每个字段位数 $n$ 越少 → 寻址范围 $2^n$ 越小
- 需要在**指令功能丰富度**和**寻址范围**之间权衡

### 不同地址格式的指令条数对比

设指令字长16位，操作码4位：

| 地址格式 | 地址字段位数 | 每字段寻址范围 | 最大指令条数（无扩展） |
|:---:|:---:|:---:|:---:|
| 三地址 | $(16-4)/3 = 4$位 | 16 | $2^4 = 16$ |
| 二地址 | $(16-4)/2 = 6$位 | 64 | $2^4 = 16$ |
| 一地址 | $(16-4)/1 = 12$位 | 4096 | $2^4 = 16$ |
| 零地址 | 全部16位 | — | $2^{16} = 65536$ |

### 计算示例

**例题**：某机指令字长32位，操作码8位，二地址格式，每个地址可访问64个寄存器，问：
1. 每个地址字段需几位？
2. 若改为三地址格式，每字段几位？寻址范围？

**解答**：
1. 二地址：$地址字段 = (32 - 8) / 2 = 12$位。访问64个寄存器只需 $\log_2 64 = 6$位，剩余 $12-6=6$位可用于其他编码。
2. 三地址：$地址字段 = (32 - 8) / 3 = 8$位。每字段寻址范围 $2^8 = 256$。

## 直观理解

### 生活类比

- **三地址**：像数学公式 $z = x + y$，三个变量互不干扰
- **二地址**：像 $x = x + y$，结果覆盖了 $x$ 的旧值
- **一地址**：像计算器，一个数显示在屏幕上，输入另一个数按"+"，结果回到屏幕上（ACC隐含）
- **零地址**：像RPN计算器（如HP），操作数已经在栈上，按运算符自动弹出计算

### 记忆技巧

**地址数越多越好用，但每段越短**：
- "三地址最强大但最短的胳膊（寻址范围小）"
- "零地址最弱但胳膊最长（所有位都可用于寻址）"

### 易错陷阱
1. **二地址指令的"结果覆盖源"**：执行后A1的原值丢失
2. **零地址指令不等于没有操作数**：操作数通过堆栈等隐含方式提供
3. **"地址字段位数"不等于"可寻址单元数"**：$n$ 位最多寻址 $2^n$ 个单元
