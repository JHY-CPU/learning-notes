# 39_子程序调用与返回

## 核心概念

### CALL指令

**功能**：调用子程序（过程/函数）

**执行过程**：
1. **保存返回地址**：将PC（下一条指令地址）压入堆栈
2. **跳转**：PC ← 子程序入口地址

```
CALL label:
    SP ← SP - Δ          ; 堆栈指针调整
    M[SP] ← PC + len     ; 保存返回地址（下一条指令地址）
    PC ← label           ; 跳转到子程序
```

### RET指令

**功能**：从子程序返回

**执行过程**：
1. **恢复返回地址**：从堆栈弹出返回地址到PC
2. **跳转**：PC ← 返回地址

```
RET:
    PC ← M[SP]           ; 弹出返回地址
    SP ← SP + Δ          ; 恢复堆栈指针
```

### 调用和返回的堆栈变化

```
CALL前:          CALL后:           RET后:
┌────────┐      ┌────────┐       ┌────────┐
│  ...   │      │  ...   │       │  ...   │
│        │      │ 返回地址│←SP    │        │
│        │←SP   │        │       │        │←SP
└────────┘      └────────┘       └────────┘
```

### 参数传递方式

| 方式 | 说明 | 优点 | 缺点 |
|:---|:---|:---|:---|
| **寄存器传参** | 参数放在寄存器中 | 速度快 | 寄存器数量有限 |
| **堆栈传参** | 参数压入堆栈 | 支持任意多参数 | 需要访存，较慢 |
| **全局变量** | 参数放在固定内存位置 | 简单 | 不可重入 |
| **寄存器+堆栈混合** | 前几个参数用寄存器，其余用堆栈 | 平衡速度和灵活性 | 复杂 |

### 调用约定（Calling Convention）

定义了调用者和被调用者之间的约定：
- 哪些寄存器传参
- 谁负责清理堆栈（caller/callee cleanup）
- 返回值放在哪里
- 哪些寄存器被调用者需要保存

x86 cdecl约定：
- 参数从右到左压栈
- 调用者清理堆栈（`ADD SP, n`）
- 返回值在EAX中
- EAX、ECX、EDX由调用者保存

## 原理分析

### 嵌套调用的堆栈变化

```
main调用func1，func1调用func2：

main:  CALL func1  → 堆栈: [ret_main]
func1: CALL func2  → 堆栈: [ret_main, ret_func1]
func2: RET         → 堆栈: [ret_main]，PC←ret_func1
func1: RET         → 堆栈: []，PC←ret_main
```

### 栈帧（Stack Frame）

每个函数调用在堆栈中创建一个**栈帧**：

```
高地址
┌─────────────────┐
│   参数3          │  ← [BP+16]
│   参数2          │  ← [BP+12]
│   参数1          │  ← [BP+8]
│   返回地址       │  ← [BP+4]
│   旧BP          │  ← [BP]（当前BP指向这里）
│   局部变量1      │  ← [BP-4]
│   局部变量2      │  ← [BP-8]
└─────────────────┘
低地址（SP）
```

**ENTER指令**：`ENTER size, level` 创建栈帧
**LEAVE指令**：销毁栈帧

### 递归调用

递归函数每次调用都创建新的栈帧：

```
fact(3):
  调用fact(2):
    调用fact(1):
      调用fact(0): 返回1
    返回1×1=1
  返回2×1=2
返回3×2=6

堆栈: [fact(3)帧, fact(2)帧, fact(1)帧, fact(0)帧]
```

## 直观理解

### 生活类比

子程序调用就像**去分店办事**：
- CALL = 去分店，同时在便签上写下总店地址放进口袋（PUSH返回地址）
- RET = 办完事从口袋拿出便签，按地址回总店（POP返回地址）

### 记忆技巧

- **"CALL = PUSH返回地址 + JMP"**
- **"RET = POP到PC"**
- **"参数可以放寄存器（快）或堆栈（灵活）"**

### 易错陷阱
1. **CALL保存的是"下一条指令"的地址**：不是CALL本身的地址
2. **RET必须有对应的CALL**：否则堆栈不平衡
3. **参数传递方式因调用约定而异**：不是唯一的
4. **递归深度受堆栈大小限制**：栈溢出
