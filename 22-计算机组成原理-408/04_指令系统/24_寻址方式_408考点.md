## 原理分析

### 考点一：给定条件求EA

**例题1**：某机有R0\~R7共8个通用寄存器，BR为基址寄存器（BR=2000H），IX为变址寄存器（IX=0100H），PC=3000H。存储器按字节编址。求以下各指令的EA：

（1）`MOV R1, #50H`（立即寻址）
（2）`MOV R1, [1000H]`（直接寻址）
（3）`MOV R1, [R2]`（寄存器间接寻址，R2=500H）
（4）`MOV R1, [BR + 200H]`（基址寻址）
（5）`MOV R1, [IX + 300H]`（变址寻址）
（6）`JMP +30H`（相对寻址，当前指令长2字节）

**解答**：

（1）立即寻址：无EA，操作数 = 50H
（2）直接寻址：$EA = 1000H$
（3）寄存器间接：$EA = R2 = 500H$
（4）基址寻址：$EA = BR + 200H = 2000H + 200H = 2200H$
（5）变址寻址：$EA = IX + 300H = 0100H + 300H = 0400H$
（6）相对寻址：$PC_{取指后} = 3000H + 2 = 3002H$，$EA = 3002H + 30H = 3032H$

### 考点二：求访存次数

**例题2**：对例题1中的每种寻址方式，求获取操作数所需的访存次数。

**解答**：

| 寻址方式 | 访存次数 | 说明 |
|:---|:---:|:---|
| 立即寻址 | 0 | 操作数在指令中 |
| 直接寻址 | 1 | 从M[1000H]取 |
| 寄存器间接 | 1 | 从M[500H]取 |
| 基址寻址 | 1 | 从M[2200H]取 |
| 变址寻址 | 1 | 从M[0400H]取 |
| 相对寻址 | 0 | 跳转只修改PC，不取操作数 |

### 考点三：判断寻址方式

**例题3**：某指令 `LOAD R1, 100(R2)` 表示 $R1 \leftarrow M[R2 + 100]$，这是什么寻址方式？

**解答**：**寄存器间接+偏移**（也称为寄存器相对寻址）。
$$EA = R2 + 100$$

若R2是基址寄存器（由OS设置）→ 基址寻址
若R2是通用寄存器（用户设置）→ 变址寻址（或寄存器相对寻址）

### 考点四：间接寻址的EA计算

**例题4**：某指令使用间接寻址，形式地址A=1000H。M[1000H]=2000H，M[2000H]=3000H。

一次间接：$EA = M[1000H] = 2000H$，操作数 = M[2000H] = 3000H
二次间接：$EA = M[M[1000H]] = M[2000H] = 3000H$，操作数 = M[3000H]

## 直观理解

### 解题步骤

1. **识别寻址方式**：看指令格式和寻址方式标志
2. **写出EA公式**：套用对应方式的公式
3. **代入已知值**：BR、IX、PC、R等寄存器的当前值
4. **计算EA**：得到有效地址
5. **计算访存次数**：从EA取操作数需要几次额外访存

### 速查表

$$\boxed{
\begin{array}{ll}
\text{立即：操作数=A，0次访存} & \text{直接：EA=A，1次访存} \\
\text{间接：EA=M[A]，2次访存} & \text{寄存器：操作数=R，0次访存} \\
\text{寄存器间接：EA=R，1次访存} & \text{基址：EA=BR+A，1次访存} \\
\text{变址：EA=IX+A，1次访存} & \text{相对：EA=PC+A，0~1次} \\
\text{隐含：ACC等，0~1次} & \text{堆栈：EA=SP，1次}
\end{array}
}$$

## 知识关联

### 易错陷阱

1. **PC值**：取指后PC已更新为下一条指令地址
2. **访存次数不含取指**：只算取操作数
3. **间接寻址的"层数"**：一次间接2次访存，二次间接3次访存
4. **基址vs变址的A含义不同**：基址中A是偏移，变址中A是基地址
5. **立即寻址没有EA**：不要强行计算"地址"
