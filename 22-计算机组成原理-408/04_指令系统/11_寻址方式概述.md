# 12_寻址方式概述

## 核心概念

### 寻址方式的定义
- **寻址方式（Addressing Mode）**：确定指令中操作数所在位置的方法
- 解决的核心问题：**"操作数在哪里？"**
- 不同的寻址方式提供了不同的灵活性和效率

### 指令寻址 vs 数据寻址

| 类别 | 解决的问题 | 说明 |
|:---:|:---|:---|
| **指令寻址** | 下一条指令在哪里 | 由PC（程序计数器）决定 |
| **数据寻址** | 操作数在哪里 | 由指令的地址码和寻址方式决定 |

**指令寻址**：
- **顺序寻址**：$PC \leftarrow PC + 1$（或+指令长度）
- **跳跃寻址**：$PC \leftarrow 目标地址$（转移指令）

**数据寻址**：本章重点，多种方式。

### 数据寻址的基本框架

指令格式中，地址码部分通常包含：

$$[寻址方式特征位][形式地址A]$$

- **形式地址A（Formal Address）**：指令地址码中直接给出的编码值
- **有效地址EA（Effective Address）**：操作数在存储器中的真实地址
- 寻址方式决定了如何从形式地址A计算出有效地址EA

### 各种寻址方式一览

| 序号 | 寻址方式 | EA计算公式 | 访存次数 | 特点 |
|:---:|:---|:---:|:---:|:---|
| 1 | 立即寻址 | 操作数 = A（无EA） | 0 | 操作数在指令中 |
| 2 | 直接寻址 | EA = A | 1 | A就是地址 |
| 3 | 间接寻址 | EA = (A) | 2+ | A指向地址的地址 |
| 4 | 寄存器寻址 | EA = R（无访存） | 0 | 操作数在寄存器 |
| 5 | 寄存器间接寻址 | EA = (R) | 1 | 寄存器存地址 |
| 6 | 隐含寻址 | 操作数在ACC等 | 0 | 不显式给出 |
| 7 | 基址寻址 | EA = BR + A | 1 | 支持重定位 |
| 8 | 变址寻址 | EA = IX + A | 1 | 适合数组访问 |
| 9 | 相对寻址 | EA = PC + A | 1 | 程序浮动 |
| 10 | 堆栈寻址 | EA = (SP) | 1 | 栈顶操作 |

## 原理分析

### 形式地址到有效地址的转换

```
形式地址A ──→ [寻址方式计算] ──→ 有效地址EA ──→ 访存取操作数
                                    │
                              （某些方式不需访存）
                              （立即寻址：操作数=A本身）
                              （寄存器寻址：操作数在R中）
```

### 寻址方式的层次设计

```
第一层：操作数是否在指令中？
├── 是 → 立即寻址（操作数 = A）
└── 否 → 操作数在存储器还是寄存器？
    ├── 寄存器 → 寄存器寻址（EA = R编号）
    └── 存储器 → 如何计算存储器地址？
        ├── 直接：EA = A
        ├── 间接：EA = (A)
        ├── 基址：EA = BR + A
        ├── 变址：EA = IX + A
        ├── 相对：EA = PC + A
        └── 堆栈：EA = SP
```

### 访存次数的计算

**"访存次数"指的是获取操作数所需访问存储器的次数**（不含取指）：

- **立即寻址**：操作数在指令中，0次访存
- **直接寻址**：从EA取操作数，1次访存
- **一次间接寻址**：先从A取EA，再从EA取操作数，2次访存
- **寄存器寻址**：从寄存器取，0次访存
- **寄存器间接寻址**：从(EA)取操作数，1次访存

## 直观理解

### 生活类比

寻址方式就像**找东西的各种方法**：

1. **立即寻址**：东西就在你手里（直接给出）
2. **直接寻址**：纸条写着"在3号柜子里"（直接地址）
3. **间接寻址**：纸条写着"看另一张纸条"，那张纸条写着"3号柜"（地址的地址）
4. **寄存器寻址**：东西在你随身的口袋里（寄存器）
5. **寄存器间接寻址**：口袋里的纸条写着"3号柜"
6. **基址寻址**："基地"（BR）+ "偏移"（A）
7. **变址寻址**："基准"（A）+ "下标"（IX）
8. **相对寻址**："当前位置"（PC）+ "偏移"（A）
9. **堆栈寻址**：栈顶在哪就操作哪里

### 记忆口诀

> **"立即手里拿，直接告诉你地址，间接指两层，寄存器不访存，基址靠基地，变址靠下标，相对看PC，堆栈看SP"**

## 代码/模拟

### Python模拟各种寻址方式

```python
"""寻址方式模拟 - 适用于408考研复习"""

class AddressingModeSimulator:
    """模拟各种寻址方式的地址计算过程"""

    def __init__(self):
        # 模拟存储器和寄存器
        self.memory = {}    # 地址 → 值
        self.registers = {} # 寄存器名 → 值

    def setup(self, memory_dict, reg_dict):
        """设置初始状态"""
        self.memory.update(memory_dict)
        self.registers.update(reg_dict)

    def immediate(self, A):
        """立即寻址: 操作数 = 形式地址A"""
        print(f"  立即寻址: 操作数 = A = {A}")
        print(f"  访存次数: 0 (操作数在指令中)")
        return A

    def direct(self, A):
        """直接寻址: EA = A"""
        EA = A
        operand = self.memory.get(EA, '?')
        print(f"  直接寻址: EA = A = {EA}")
        print(f"  访存: M[{EA}] = {operand}")
        print(f"  访存次数: 1")
        return operand

    def indirect(self, A):
        """间接寻址: EA = (A)"""
        EA = self.memory.get(A, '?')
        operand = self.memory.get(EA, '?') if EA != '?' else '?'
        print(f"  间接寻址: 第1次访存 M[{A}] = {EA} (得到EA)")
        print(f"           第2次访存 M[{EA}] = {operand} (得到操作数)")
        print(f"  访存次数: 2")
        return operand

    def register(self, R):
        """寄存器寻址: 操作数在寄存器R中"""
        operand = self.registers.get(R, '?')
        print(f"  寄存器寻址: 操作数 = {R} = {operand}")
        print(f"  访存次数: 0")
        return operand

    def register_indirect(self, R):
        """寄存器间接寻址: EA = (R)"""
        EA = self.registers.get(R, '?')
        operand = self.memory.get(EA, '?')
        print(f"  寄存器间接寻址: EA = ({R}) = {EA}")
        print(f"  访存: M[{EA}] = {operand}")
        print(f"  访存次数: 1")
        return operand

    def base(self, A, BR_name='BR'):
        """基址寻址: EA = BR + A"""
        BR = self.registers.get(BR_name, 0)
        EA = BR + A
        operand = self.memory.get(EA, '?')
        print(f"  基址寻址: EA = BR({BR}) + A({A}) = {EA}")
        print(f"  访存: M[{EA}] = {operand}")
        print(f"  访存次数: 1 (用于程序重定位)")
        return operand

    def index(self, A, IX_name='IX'):
        """变址寻址: EA = IX + A"""
        IX = self.registers.get(IX_name, 0)
        EA = IX + A
        operand = self.memory.get(EA, '?')
        print(f"  变址寻址: EA = A({A}) + IX({IX}) = {EA}")
        print(f"  访存: M[{EA}] = {operand}")
        print(f"  访存次数: 1 (适合数组遍历)")
        return operand

    def relative(self, A, PC):
        """相对寻址: EA = PC + A"""
        EA = PC + A
        operand = self.memory.get(EA, '?')
        print(f"  相对寻址: EA = PC({PC}) + A({A}) = {EA}")
        print(f"  访存次数: 1 (用于程序浮动/分支跳转)")
        return EA

# 运行示例
sim = AddressingModeSimulator()
sim.setup(
    memory_dict={100: 200, 200: 999, 300: 500, 500: 1234},
    reg_dict={'R1': 300, 'BR': 100, 'IX': 200, 'PC': 400}
)

print("=== 各种寻址方式对比 ===")
print("初始状态: M[100]=200, M[200]=999, M[300]=500, M[500]=1234")
print("         R1=300, BR=100, IX=200, PC=400\n")

print("1. 形式地址A=100:")
print("---")
sim.immediate(100)
print()
sim.direct(100)
print()
sim.indirect(100)
print()
sim.register('R1')
print()
sim.register_indirect('R1')
print()
sim.base(200)          # EA = BR(100) + 200 = 300
print()
sim.index(300)         # EA = 300 + IX(200) = 500
print()
sim.relative(100, 400) # EA = PC(400) + 100 = 500
```

## 知识关联

### 跨章节联系
- **第3章（存储系统）**：访存次数影响Cache效率和整体性能
- **第5章（CPU）**：不同寻址方式需要不同的微操作序列
- **高级语言编译**：数组访问→变址寻址；函数调用→堆栈寻址；全局变量→直接寻址

### 常考组合
- 计算各寻址方式的EA和访存次数
- 判断给定指令使用了什么寻址方式

### 易错陷阱
1. **"访存次数"不含取指令的次数**：只算获取操作数的访存
2. **间接寻址可能多次**：一次间接2次访存，二次间接3次访存
3. **形式地址≠有效地址**：除了直接寻址和立即寻址，其他都需要计算
4. **立即寻址没有EA**：操作数就是形式地址本身，不存在"地址"的概念
