# 01_指令基本格式

## 核心概念

### 指令的定义
- **指令（Instruction）**：CPU能够识别和执行的最基本操作命令，是机器语言的基本单位
- **指令系统（Instruction Set）**：一台计算机所有指令的集合，决定了计算机的功能和性能
- **机器指令**：用二进制编码表示的指令，CPU可直接识别

### 指令的基本组成

$$指令 = 操作码(OP) + 地址码(A)$$

- **操作码（Operation Code, OP）**：指明指令要执行的操作类型，如加法、减法、传送、跳转等
- **地址码（Address Code, A）**：指明操作数的来源和结果的去向，包括操作数在内存中的地址或寄存器编号

### 指令字长

$$指令字长 = 操作码位数 + 地址码位数$$

- **指令字长**：一条指令所占用的二进制位数
- 指令字长可以是**定长**的（如MIPS固定32位），也可以是**变长**的（如x86从1到15字节）
- 指令字长通常是**机器字长的整数倍**，也可以是**存储字长的整数倍**

### 操作码的作用
- 操作码决定了指令的**操作类型**（做什么）
- 操作码的位数决定了可表示的指令种类数
- 若操作码占 $n$ 位，则最多可表示 $2^n$ 条不同指令

### 地址码的作用
- 地址码指明操作数的**位置**（在哪里）
- 地址码可以是：内存地址、寄存器编号、立即数本身、或隐含使用

## 原理分析

### 一条完整指令的格式展开

以三地址指令为例：

| OP（操作码） | A1（目标地址） | A2（源操作数1） | A3（源操作数2） |
|:---:|:---:|:---:|:---:|
| k位 | n位 | n位 | n位 |

执行语义：$M[A1] \leftarrow M[A2] \text{ OP } M[A3]$

### 指令执行的基本流程

1. **取指（Fetch）**：根据PC（程序计数器）从存储器中取出指令
2. **译码（Decode）**：分析操作码，确定操作类型；分析地址码，确定操作数位置
3. **取数（Fetch Operand）**：根据地址码从寄存器或存储器中获取操作数
4. **执行（Execute）**：在ALU中完成运算
5. **写回（Write Back）**：将结果写入目标地址
6. **更新PC**：PC指向下一条指令

### 指令格式设计的核心权衡

$$指令空间 = 操作码空间 + 地址码空间$$

- 操作码位数越多 → 支持更多指令种类，但留给地址码的位数越少
- 地址码位数越多 → 寻址范围越大，但留给操作码的位数越少
- **固定指令字长条件下，操作码与地址码是此消彼长的关系**

### 计算题：给定指令字长求各字段位数

**例题**：某机指令字长16位，操作码4位，采用二地址格式，求每个地址字段位数。

解：
$$地址字段位数 = \frac{指令字长 - 操作码位数}{地址个数} = \frac{16 - 4}{2} = 6位$$

每个地址字段6位，可寻址 $2^6 = 64$ 个不同的存储单元或寄存器。

## 直观理解

### 生活类比

把指令比作**菜谱中的一道步骤**：
- **操作码** = 动作（"切"、"炒"、"加"），告诉你做什么
- **地址码** = 对象（"萝卜"、"锅里"、"盐罐"），告诉你在哪里操作

菜谱步骤"把盐罐里的盐加到锅里"就类似一条二地址指令：OP=加，A1=盐罐，A2=锅。

### 记忆技巧

- **指令 = 命令 = 做什么 + 在哪做**（操作码管"做什么"，地址码管"在哪做"）
- **OP决定"动作"，A决定"位置"**
- 操作码像**动词**，地址码像**名词（宾语）**

### 常考组合
- 指令字长 + 操作码位数 + 地址格式 → 求可表示指令条数和寻址范围（选择题高频）
- 指令字长与存储字长的关系（是否必须相等、对齐问题）

### 易错陷阱
1. **"指令字长=机器字长"不一定成立**：指令字长可以是机器字长的整数倍
2. **"操作码位数固定=指令条数固定"不一定成立**：扩展操作码可增加指令条数
3. **地址码个数≠操作数个数**：零地址指令可能隐含操作数（如栈顶元素）
4. **指令字长与存储字长**：若指令字长>存储字长，取一条指令需多次访存
