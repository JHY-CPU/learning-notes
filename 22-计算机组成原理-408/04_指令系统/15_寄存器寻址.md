# 16_寄存器寻址

## 核心概念

### 定义
- **寄存器寻址（Register Addressing）**：操作数存放在**CPU的通用寄存器**中，指令地址码给出寄存器编号
- 不需要访问主存储器，直接从寄存器中获取操作数

### 格式与语义

$$[OP][寻址方式标志][寄存器编号R]$$

$$操作数 = R$$

指令中的地址字段给出的是**寄存器编号**，而不是存储器地址。

### 典型指令示例

| 指令 | 含义 |
|:---|:---|
| `MOV R1, R2` | $R1 \leftarrow R2$，寄存器间传送 |
| `ADD R1, R2` | $R1 \leftarrow R1 + R2$ |
| `SUB R3, R4` | $R3 \leftarrow R3 - R4$ |

### 特点

| 优点 | 缺点 |
|:---|:---|
| **速度最快**：不需要访存 | **寄存器数量有限**：通常8~32个 |
| **0次访存**：从寄存器直接读写 | **操作数范围受限于寄存器宽度** |
| **指令短**：只需寄存器编号（3\~5位） | **大量数据无法全放寄存器** |
| **RISC核心方式**：Load/Store架构基础 | 需配合LOAD/STORE访问存储器 |

## 原理分析

### 执行过程

```
1. 取指：从PC指向的存储单元取出指令
2. 译码：识别操作码和寻址方式（寄存器寻址）
3. 执行：直接从指定寄存器读取操作数，送ALU运算
4. 写回：结果写回目标寄存器
```

**关键**：步骤3不需要访存，直接从寄存器文件（Register File）中读取。

### 寄存器编号的位数

设有 $N$ 个通用寄存器：

$$寄存器编号位数 = \log_2 N$$

| 寄存器数 | 编号位数 | 典型架构 |
|:---:|:---:|:---|
| 8 | 3位 | x86早期 |
| 16 | 4位 | ARM（AArch32） |
| 32 | 5位 | MIPS、RISC-V |
| 64 | 6位 | AArch64 |

### 访存次数

$$访存次数（不含取指）= 0$$

操作数在寄存器中，不需要访问主存储器。

### 寄存器寻址在RISC中的核心地位

RISC架构（如MIPS）采用**Load/Store结构**：
- **只有LOAD和STORE指令可以访存**：$LOAD: R \leftarrow M[addr]$，$STORE: M[addr] \leftarrow R$
- **所有运算指令只使用寄存器操作数**：$ADD R1, R2, R3$
- 这意味着：RISC的运算指令**全部使用寄存器寻址**

$$RISC: LOAD → 寄存器运算 → STORE$$

## 直观理解

### 生活类比

寄存器寻址就像**从随身口袋拿东西**：
- 直接寻址 = 去仓库（存储器）找东西
- 寄存器寻址 = 从口袋（寄存器）里直接拿
- 口袋里的东西用得最快，但口袋数量有限

### 记忆技巧

- **"寄存器 = CPU的口袋 = 最快但最少"**
- **"0访存，速度第一"**

## 知识关联

### 跨章节联系
- **第3章（存储系统）**：寄存器是最快的存储层次，在Cache之上
- **第5章（CPU）**：寄存器堆（Register File）的读写操作
- **第2章（数据表示）**：寄存器宽度 = 机器字长

### 常考对比
- 寄存器寻址 vs 直接寻址：0次访存 vs 1次访存
- 寄存器寻址 vs 寄存器间接寻址：操作数在寄存器 vs 操作数在存储器

### 易错陷阱
1. **寄存器编号 ≠ 存储器地址**：地址码给出的是寄存器编号，不是内存地址
2. **"0次访存"**：不含取指令的访存
3. **寄存器数量有限**：不是所有操作都能放寄存器，需要LOAD/STORE配合
4. **寄存器寻址不需要计算EA**：操作数直接在指定寄存器中
