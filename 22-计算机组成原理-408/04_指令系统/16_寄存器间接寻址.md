# 17_寄存器间接寻址

## 核心概念

### 定义
- **寄存器间接寻址（Register Indirect Addressing）**：指令给出寄存器编号，寄存器中存放的是操作数的**有效地址EA**
- 结合了寄存器寻址的快速和间接寻址的灵活

### 格式与语义

$$[OP][寻址方式标志][寄存器编号R]$$

$$EA = R的内容$$

$$操作数 = M[EA] = M[R]$$

### 典型指令示例

| 指令 | 含义 |
|:---|:---|
| `MOV R1, (R2)` | $R1 \leftarrow M[R2]$，R2内容是地址 |
| `ADD R1, (R2)` | $R1 \leftarrow R1 + M[R2]$ |
| `MOV (R1), R2` | $M[R1] \leftarrow R2$ |

### 与直接寻址和寄存器寻址的对比

| 寻址方式 | EA | 操作数位置 | 访存次数 |
|:---|:---|:---|:---:|
| 直接寻址 | $EA = A$（形式地址） | 存储器 | 1 |
| 寄存器寻址 | 无EA（操作数=R） | 寄存器 | 0 |
| 寄存器间接寻址 | $EA = R$（寄存器内容） | 存储器 | 1 |

### 特点

| 优点 | 缺点 |
|:---|:---|
| **一次访存**：比存储器间接寻址快 | **寄存器数量限制**：间接地址数受限于寄存器数 |
| **寻址范围大**：EA = 寄存器宽度 | **需预加载地址到寄存器** |
| **适合指针和数组遍历**：修改寄存器值即可改变地址 | 比直接寻址多一步 |
| **指令短**：寄存器编号只需几位 | |

## 原理分析

### 执行过程

```
1. 取指：从PC指向的存储单元取出指令
2. 译码：识别操作码和寻址方式（寄存器间接寻址）
3. 取EA：从指定寄存器R中读取内容（这就是EA）
4. 取数：以EA访问存储器M[EA]，取出操作数
5. 执行：ALU运算
6. 写回
```

### 访存次数

$$访存次数（不含取指）= 1$$

- 从寄存器取EA（0次访存）
- 从M[EA]取操作数（1次访存）

对比存储器间接寻址（2次访存），寄存器间接寻址少了一次访存。

### 寄存器间接寻址的应用

**1. 指针操作**

```c
int *p = &arr[0];  // p → 寄存器R存放arr[0]的地址
int val = *p;      // MOV R_val, (R_p)
```

**2. 数组遍历**

```
LOOP:
    MOV R1, (R2)    ; 从R2指向的地址取数据
    ADD R2, #4      ; R2指针后移4字节
    DEC R3          ; 计数器减1
    JNZ LOOP        ; 未完继续
```

只需修改R2的值，同一指令可访问不同元素。

**3. 间接跳转**

```
JMP (R1)    ; 跳转到R1中存放的地址
```

## 直观理解

### 生活类比

寄存器间接寻址就像**你口袋里有张纸条，纸条上写着仓库的柜号**：
- 纸条在口袋里（寄存器）→ 不用去仓库查地址（快）
- 但东西还是在仓库里（存储器）→ 需要去仓库拿（1次访存）
- 换张纸条（修改寄存器值）→ 就能指向不同的柜子

### 记忆技巧

- **"寄存器里装的是地址（不是数据）"**
- **"R的内容=EA → 比存储器间接快一次访存"**

## 知识关联

### 跨章节联系
- **第3章（存储系统）**：1次访存，Cache可加速
- **第5章（CPU）**：寄存器读取（0延迟）+ 存储器访问（1次访存）
- **高级语言**：指针变量、数组下标访问的底层实现

### 常考对比
- 寄存器间接 vs 存储器间接：1次访存 vs 2次访存
- 寄存器间接 vs 直接：地址在寄存器 vs 地址在指令中
- 寄存器间接 vs 寄存器：操作数在存储器 vs 操作数在寄存器

### 易错陷阱
1. **R的内容是地址，不是操作数**：这是与寄存器寻址的关键区别
2. **"(R)"表示间接**：括号表示"取R指向的内容"
3. **访存次数 = 1**：从M[R]取操作数
4. **需要提前把地址加载到寄存器**：这是额外的指令开销
