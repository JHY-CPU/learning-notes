# 22_堆栈寻址

## 核心概念

### 定义
- **堆栈寻址（Stack Addressing）**：操作数在堆栈中，由**堆栈指针SP**指定栈顶位置
- 操作数的地址隐含在SP中，指令不需要显式给出地址

### 堆栈的基本概念

- **堆栈（Stack）**：一种**后进先出（LIFO）**的数据结构
- **栈顶（Top）**：最后压入的元素所在位置
- **栈底（Bottom）**：堆栈的起始位置
- **堆栈指针SP（Stack Pointer）**：始终指向栈顶的寄存器

### 堆栈操作

**PUSH（压栈/入栈）**：
```
SP ← SP - 1       （满递减堆栈）
M[SP] ← 数据
```

**POP（弹栈/出栈）**：
```
数据 ← M[SP]
SP ← SP + 1
```

### 堆栈的四种类型

| 类型 | SP指向 | PUSH操作 | POP操作 |
|:---|:---|:---|:---|
| **满递减（FD）** | 最后一个有效数据 | SP先减，再存 | 先取，SP再增 |
| **空递减（ED）** | 下一个空位置 | 先存，SP再减 | SP先增，再取 |
| **满递增（FA）** | 最后一个有效数据 | SP先增，再存 | 先取，SP再减 |
| **空递增（EA）** | 下一个空位置 | 先存，SP再增 | SP先减，再取 |

ARM常用**满递减（Full Descending）**堆栈。

### 特点

| 优点 | 缺点 |
|:---|:---|
| **零地址指令**：操作数地址全隐含 | **只能访问栈顶** |
| **表达式求值方便**：后缀表达式 | **寻址范围受限于堆栈大小** |
| **函数调用天然支持**：保存现场、传参 | **SP管理增加硬件复杂度** |

## 原理分析

### 堆栈寻址的执行过程

以 `ADD`（零地址，堆栈操作）为例：

```
栈状态: [A, B]  ← SP指向B（栈顶）

执行ADD:
1. POP B      → SP↑, 取B
2. POP A      → SP↑, 取A
3. A + B = C
4. PUSH C     → SP↓, 存C

结果: [C]  ← SP指向C
```

### 堆栈在函数调用中的应用

```
CALL func:
1. PUSH 返回地址  → 将PC（下一条指令地址）压栈
2. JMP func       → 跳转到func

RET:
1. POP PC         → 弹出返回地址到PC
```

函数参数传递：
```
PUSH arg3
PUSH arg2
PUSH arg1
CALL func
```

局部变量也在堆栈中分配（栈帧）。

### SP的自动修改

每次PUSH/POP后，SP自动修改：
- **递减堆栈**：PUSH时 $SP \leftarrow SP - \Delta$，POP时 $SP \leftarrow SP + \Delta$
- **递增堆栈**：PUSH时 $SP \leftarrow SP + \Delta$，POP时 $SP \leftarrow SP - \Delta$

其中 $\Delta$ 为数据宽度（如4字节）。

### 访存次数

| 操作 | 访存次数（不含取指） |
|:---:|:---:|
| PUSH | 1次（写入M[SP]） |
| POP | 1次（从M[SP]读取） |

$$访存次数 = 1$$

## 直观理解

### 生活类比

堆栈就像**一摞盘子**：
- 只能从顶部放盘子（PUSH）或拿盘子（POP）
- SP告诉你最上面的盘子在哪
- 不能直接拿中间的盘子（不像数组可以直接访问）

### 记忆技巧

- **"堆栈 = 只认栈顶 = SP指路"**
- **"满栈SP指最后元素，空栈SP指空位"**
- **"递减栈从高地址往低地址生长"**

### 易错陷阱
1. **SP指向栈顶还是空位**：取决于"满"还是"空"堆栈设计
2. **堆栈增长方向**：递减（高→低）还是递增（低→高）
3. **堆栈寻址是零地址指令**：操作数地址全部隐含
4. **PUSH/POP的SP修改顺序**：先减后存（满栈）还是先存后减（空栈）
