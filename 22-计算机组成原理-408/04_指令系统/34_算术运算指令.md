# 35_算术运算指令

## 核心概念

### 常见算术运算指令

| 指令 | 格式 | 含义 | 说明 |
|:---|:---|:---|:---|
| **ADD** | `ADD dst, src` | $dst \leftarrow dst + src$ | 加法 |
| **SUB** | `SUB dst, src` | $dst \leftarrow dst - src$ | 减法 |
| **MUL** | `MUL src` | $AX \leftarrow AL \times src$（字节） | 无符号乘 |
| **IMUL** | `IMUL src` | $AX \leftarrow AL \times src$（字节） | 有符号乘 |
| **DIV** | `DIV src` | $AL \leftarrow AX / src$ 的商 | 无符号除 |
| **INC** | `INC dst` | $dst \leftarrow dst + 1$ | 自增 |
| **DEC** | `DEC dst` | $dst \leftarrow dst - 1$ | 自减 |
| **NEG** | `NEG dst` | $dst \leftarrow 0 - dst$ | 取负（求补） |

### 算术运算对标志位的影响

执行算术运算后，CPU的状态标志位（PSW）会更新：

| 标志位 | 名称 | 含义 | 设置条件 |
|:---:|:---|:---|:---|
| **CF** | 进位标志 | 无符号运算溢出 | 最高位产生进位/借位 |
| **ZF** | 零标志 | 结果为零 | 结果 = 0 |
| **SF** | 符号标志 | 结果为负 | 结果最高位 = 1 |
| **OF** | 溢出标志 | 有符号运算溢出 | 符号位溢出 |

### CF和OF的区别（408重点！）

$$CF = C_n \oplus C_{n-1} \text{（最高位进位）}$$

$$OF = C_n \oplus C_{n-1} \text{（次高位到最高位的进位异或最高位进位）}$$

更准确：
- **CF**：无符号数的溢出。加法时 $CF = C_{out}$（最高位进位输出）；减法时 $CF = \overline{C_{out}}$
- **OF**：有符号数的溢出。$OF = C_n \oplus C_{n-1}$（最高位进位输入与输出的异或）

**例题**：8位加法，$127 + 1 = ?$

$$01111111 + 00000001 = 10000000$$

- 无符号：$127 + 1 = 128$，无溢出 → $CF = 0$
- 有符号：$127 + 1 = -128$（溢出！）→ $OF = 1$
- $SF = 1$（结果最高位为1）
- $ZF = 0$（结果非零）

## 原理分析

### 乘法和除法

**乘法**：
- 8位 × 8位 → 16位结果
- 16位 × 16位 → 32位结果
- 结果宽度 = 操作数宽度之和

**除法**：
- 16位 / 8位 → 8位商 + 8位余数
- 32位 / 16位 → 16位商 + 16位余数

### 溢出检测

**有符号溢出（OF）**：
- 两个正数相加结果为负 → 溢出
- 两个负数相加结果为正 → 溢出
- 一正一负相加 → 不会溢出

**无符号溢出（CF）**：
- 加法结果小于任一操作数 → 溢出（产生了进位）

## 直观理解

### 生活类比

- **ADD/SUB**：算盘的加减
- **CF**：算盘位数不够，需要进位到上一档
- **OF**：用有限位数表示有符号数时的溢出（符号变反）

### 记忆技巧

- **"CF管无符号，OF管有符号"**
- **"正+正=负，负+负=正 → OF=1"**

### 易错陷阱
1. **CF和OF可能不同**：同一个运算CF和OF可以一个为0一个为1
2. **减法的CF**：减法时CF表示借位（与加法逻辑相反）
3. **INC/DEC不影响CF**：只影响ZF、SF、OF
4. **乘法结果宽度翻倍**：不要丢失高位
