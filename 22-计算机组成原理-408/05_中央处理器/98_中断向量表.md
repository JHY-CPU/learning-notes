# 92_中断向量表

## 核心概念

- **中断向量**：中断服务程序的**入口地址**
- **中断向量表**：存放所有中断向量的**内存区域**
- **中断类型号**：标识不同中断源的编号，用于索引向量表

## 原理分析

### 一、中断向量

每个中断源对应一个**中断向量**：

$$\text{中断向量} = (\text{服务程序入口地址 CS:IP})$$

- 实模式（x86）：每个向量 4 字节（2 字节 CS + 2 字节 IP）
- 保护模式：每个向量 8 字节（段选择子 + 偏移）

### 二、中断向量表的组织

#### x86 实模式

- 向量表位于内存最低地址：**0x00000 - 0x003FF**
- 共 256 个中断向量（0-255）
- 每个向量 4 字节
- 总大小：$256 \times 4 = 1024$ 字节（1KB）

$$\text{向量地址} = \text{中断类型号} \times 4$$

例如：
| 类型号 | 向量地址 | 用途 |
|--------|---------|------|
| 0 | 0x0000 | 除法错误 |
| 1 | 0x0004 | 单步/断点 |
| 2 | 0x0008 | NMI |
| 3 | 0x000C | 断点（INT 3）|
| 4 | 0x0010 | 溢出（INTO）|
| 8 | 0x0020 | 定时器 |
| ... | ... | ... |

### 三、中断响应与向量表

**完整流程**：

```
1. 中断请求到达
2. CPU 响应中断（隐指令）
3. 获取中断类型号 n
4. 计算向量地址：Addr = n × 4
5. 从内存读取：CS ← Mem[Addr+2], IP ← Mem[Addr]
6. 跳转到 CS:IP 处执行中断服务程序
```

### 四、中断类型号的获取

#### 内部中断
- 类型号由 CPU **硬件直接给出**
- 如除法错误 = 0，NMI = 2

#### 外部中断（INTR）
- 通过**中断响应周期**从 8259A 获取
- CPU 发送 INTA 信号，8259A 返回类型号

### 五、向量表的设计考量

| 设计要素 | 说明 |
|---------|------|
| 位置固定 | 便于硬件直接访问 |
| 大小固定 | 256 个向量，覆盖所有可能中断 |
| 顺序排列 | 类型号直接计算地址，无需搜索 |
| 用户可修改 | 可以安装自定义中断处理程序 |

## 直观理解

**中断向量表**像**通讯录**：

- 每个人（中断源）有一个编号（中断类型号）
- 通讯录按编号排列（向量表按类型号排列）
- 查编号找到对应的电话号码（服务程序入口地址）
- 拨打电话开始处理事情（转到服务程序执行）

## 知识关联

- 与**中断响应过程**（第 90 节）的关系：响应后需要查向量表
- 与**中断优先级**（第 93 节）的关系：优先级决定先查哪个向量
- 与**存储器**（第 6 章）的关系：向量表就是内存中的一个区域
- 与**操作系统**的关系：OS 初始化时设置中断向量表
## 代码/模拟

### Python模拟中断向量表

```python
"""中断向量表模拟 - 适用于408考研复习"""

class InterruptVectorTable:
    """中断向量表"""

    def __init__(self, base_addr=0x0000, num_vectors=256):
        self.base_addr = base_addr
        self.num_vectors = num_vectors
        # 向量表: 类型号 → (中断服务程序入口地址, CS)
        self.table = {}

    def set_vector(self, type_num, isr_addr, cs=0):
        """设置中断向量"""
        self.table[type_num] = (cs, isr_addr)

    def get_vector_addr(self, type_num):
        """计算向量在表中的地址 (每个向量4字节: 2字节IP + 2字节CS)"""
        return self.base_addr + type_num * 4

    def lookup(self, type_num):
        """查表获取ISR入口地址"""
        vector_addr = self.get_vector_addr(type_num)
        if type_num in self.table:
            cs, ip = self.table[type_num]
            print(f"  中断类型号: {type_num} ({type_num:#04x})")
            print(f"  向量地址: {vector_addr:#06x} = {self.base_addr:#06x} + {type_num}×4")
            print(f"  向量内容: CS={cs:#06x}, IP={ip:#06x}")
            print(f"  ISR入口: {cs:#06x}:{ip:#06x}")
            return (cs, ip)
        else:
            print(f"  类型号 {type_num} 未设置!")
            return None

# 设置向量表
ivt = InterruptVectorTable(base_addr=0x0000)

# x86常见中断向量
ivt.set_vector(0, 0x1000)    # 除法错误
ivt.set_vector(1, 0x2000)    # 单步中断
ivt.set_vector(2, 0x3000)    # NMI
ivt.set_vector(3, 0x4000)    # 断点
ivt.set_vector(4, 0x5000)    # 溢出
ivt.set_vector(8, 0x6000)    # 时钟中断
ivt.set_vector(13, 0x7000)   # 一般保护错误
ivt.set_vector(14, 0x8000)   # 缺页中断

print("=== 中断向量表示例 ===\n")
print(f"{'类型号':<8} {'向量地址':<12} {'ISR入口':<12} {'说明':<15}")
print("-" * 50)
info = {0: "除法错误", 1: "单步", 2: "NMI", 3: "断点",
        4: "溢出", 8: "时钟", 13: "保护错误", 14: "缺页"}
for t in sorted(ivt.table.keys()):
    addr = ivt.get_vector_addr(t)
    cs, ip = ivt.table[t]
    print(f"{t:<8} {addr:#06x}{'':>5} {ip:#06x}{'':>5} {info.get(t, ''):<15}")

# 查询过程演示
print("\n=== 查询过程演示 ===")
for t in [0, 8, 14]:
    print(f"\n--- 中断类型号={t} ---")
    ivt.lookup(t)

# 向量地址计算
print("\n=== 向量地址计算 ===")
for t in [0, 1, 8, 13, 14]:
    addr = ivt.get_vector_addr(t)
    print(f"  类型号{t:>2}: 地址 = 0x0000 + {t}×4 = {addr:#06x}")
```
