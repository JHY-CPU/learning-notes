# 92_中断向量表

## 核心概念

- **中断向量**：中断服务程序的**入口地址**
- **中断向量表**：存放所有中断向量的**内存区域**
- **中断类型号**：标识不同中断源的编号，用于索引向量表

## 原理分析

### 一、中断向量

每个中断源对应一个**中断向量**：

$$\text{中断向量} = (\text{服务程序入口地址 CS:IP})$$

- 实模式（x86）：每个向量 4 字节（2 字节 CS + 2 字节 IP）
- 保护模式：每个向量 8 字节（段选择子 + 偏移）

### 二、中断向量表的组织

#### x86 实模式

- 向量表位于内存最低地址：**0x00000 - 0x003FF**
- 共 256 个中断向量（0-255）
- 每个向量 4 字节
- 总大小：$256 \times 4 = 1024$ 字节（1KB）

$$\text{向量地址} = \text{中断类型号} \times 4$$

例如：
| 类型号 | 向量地址 | 用途 |
|--------|---------|------|
| 0 | 0x0000 | 除法错误 |
| 1 | 0x0004 | 单步/断点 |
| 2 | 0x0008 | NMI |
| 3 | 0x000C | 断点（INT 3）|
| 4 | 0x0010 | 溢出（INTO）|
| 8 | 0x0020 | 定时器 |
| ... | ... | ... |

### 三、中断响应与向量表

**完整流程**：

```
1. 中断请求到达
2. CPU 响应中断（隐指令）
3. 获取中断类型号 n
4. 计算向量地址：Addr = n × 4
5. 从内存读取：CS ← Mem[Addr+2], IP ← Mem[Addr]
6. 跳转到 CS:IP 处执行中断服务程序
```

### 四、中断类型号的获取

#### 内部中断
- 类型号由 CPU **硬件直接给出**
- 如除法错误 = 0，NMI = 2

#### 外部中断（INTR）
- 通过**中断响应周期**从 8259A 获取
- CPU 发送 INTA 信号，8259A 返回类型号

### 五、向量表的设计考量

| 设计要素 | 说明 |
|---------|------|
| 位置固定 | 便于硬件直接访问 |
| 大小固定 | 256 个向量，覆盖所有可能中断 |
| 顺序排列 | 类型号直接计算地址，无需搜索 |
| 用户可修改 | 可以安装自定义中断处理程序 |

## 直观理解

**中断向量表**像**通讯录**：

- 每个人（中断源）有一个编号（中断类型号）
- 通讯录按编号排列（向量表按类型号排列）
- 查编号找到对应的电话号码（服务程序入口地址）
- 拨打电话开始处理事情（转到服务程序执行）

## 知识关联

- 与**中断响应过程**（第 90 节）的关系：响应后需要查向量表
- 与**中断优先级**（第 93 节）的关系：优先级决定先查哪个向量
- 与**存储器**（第 6 章）的关系：向量表就是内存中的一个区域
- 与**操作系统**的关系：OS 初始化时设置中断向量表
- 408 考点：向量表位置、向量地址计算、中断类型号来源
