# 43_控制存储器CM

## 核心概念

- **控制存储器（Control Memory, CM）**：存放微程序的存储器，是微程序控制器的核心部件。
- **特点**：只读（ROM/PROM/EPROM），按地址访问，位于CPU内部。
- **容量**：$CM容量 = 微指令条数 \times 微指令字长$

### CM的基本参数

| 参数 | 说明 | 举例 |
|------|------|------|
| 地址位数 | $n$ 位地址对应 $2^n$ 条微指令 | 7位地址 → 128条微指令 |
| 字长 | 每条微指令的位数 | 32位、48位、64位 |
| 总容量 | $2^n \times w$ bit | 128 × 32 bit = 512字节 |
| 读写方式 | 只读（ROM）或可写（WCM） | 通常用ROM/EPROM |

### CM的实现技术

| 类型 | 全称 | 特点 |
|------|------|------|
| ROM | 只读存储器 | 出厂固化，不可修改 |
| PROM | 可编程ROM | 一次性编程 |
| EPROM | 可擦除PROM | 紫外线擦除后重新编程 |
| EEPROM | 电可擦除ROM | 电信号擦除，可在线修改 |
| Flash | 闪存 | 现代CM常用实现 |

## 原理分析

### CM在CPU中的位置

```

│  │ └─────┘ │    └───────────────────────────┘               │
```

CM是CPU内部的部件，不通过系统总线访问，因此读取速度非常快。

### CM的地址空间分配

CM的地址空间通常分为几个区域：

| 地址范围 | 用途 | 说明 |
|---------|------|------|
| 000H ~ 00FH | 公共取指微程序 | 所有指令共享 |
| 010H ~ 01FH | 公共间址微程序 | 间址指令共享 |
| 020H ~ 02FH | 公共中断微程序 | 中断响应共享 |
| 030H ~ FFH | 各指令执行微程序 | 每条指令一段 |

### CM容量计算实例

**例1**：某计算机有 64 条机器指令，每条指令的微程序平均包含 8 条微指令，每条微指令 32 位。求CM容量。

$$CM容量 = 64 \times 8 \times 32 = 16384 \text{ bit} = 2 \text{ KB}$$

**例2**：某CM地址为10位，字长为40位，求CM容量。

$$CM容量 = 2^{10} \times 40 = 1024 \times 40 = 40960 \text{ bit} = 5 \text{ KB}$$

### CM与主存的区别

| 对比项 | CM（控制存储器） | 主存（Main Memory） |
|--------|-----------------|-------------------|
| 位置 | CPU内部 | CPU外部 |
| 存放内容 | 微程序（微指令） | 机器指令和数据 |
| 读写方式 | 只读 | 可读可写 |
| 访问者 | 微程序控制器 | CPU（通过MAR/MDR） |
| 容量 | 较小（几KB） | 较大（几GB） |
| 速度 | 非常快 | 较慢 |
| 实现 | ROM/EPROM | DRAM |

## 直观理解

### 生活类比

CM就像**自动钢琴的音乐滚筒**：
- 滚筒上的凸起（存储内容）代表微指令
- 滚筒转动（μPC递增）时，凸起触发琴键（控制信号）
- 不同的曲子（机器指令）使用滚筒的不同区段
- 滚筒一旦制作好，上面的曲子就不容易改了（ROM特性）

### 记忆技巧

- **CM = 控制器的"大脑"**：存放所有的控制知识（微程序）
- **CM是ROM**：只读，断电不丢失
- **CM在CPU内部**：不在系统总线上

### 易错陷阱

1. **陷阱**：CM的地址位数取决于微指令总条数，不是机器指令条数。例如有32条机器指令，但每条平均需要10条微指令（加上公共微程序），则CM需要至少320+条微指令的空间。
2. **陷阱**：CM是只读的，但微程序控制器中的CM可以通过特殊手段修改（如EPROM擦写），这与硬连线控制器的"不可修改"有本质区别。
3. **陷阱**：CM的访问不经过系统总线，没有MAR/MDR的问题，但有自己的μPC和μIR。
