# 30_微操作命令分析_取指

## 核心概念

- **微操作命令（Micro-operation Command）**：CPU在一个时钟周期内完成的最基本操作，由一个或多个微命令（控制信号）驱动。
- **取指微操作命令**：所有指令共有的微操作序列，完成从主存取出指令的过程。
- **取指是公共操作**：无论执行什么指令，都必须先完成取指。

### 取指微操作序列（标准模型）

| 节拍 | 微操作 | 控制信号 | 说明 |
|------|--------|---------|------|
| T0(Fe0) | PC → MAR | PC_out, MAR_in | 指令地址送地址寄存器 |
| T1(Fe1) | 1 → R | Read | 向主存发出读命令 |
| T1 | MAR → 地址总线 → 主存 | — | 地址选中存储单元 |
| T2(Fe2) | M(MAR) → MDR | MDR_in | 指令数据读入MDR |
| T2 | MDR → IR | MDR_out, IR_in | 指令送指令寄存器 |
| T2 | PC + 1 → PC | PC_inc | 程序计数器递增 |

## 原理分析

### 各微操作的详细分析

#### T0：PC → MAR

- **数据流**：PC(内容) → 内部总线 → MAR
- **控制信号**：
  - `PC_out`：使PC的内容送上内部总线
  - `MAR_in`：使MAR从总线接收数据
- **硬件动作**：PC的输出三态门打开，MAR的输入使能有效

#### T1：读主存

- **数据流**：MAR → 地址总线 → 主存
- **控制信号**：
  - `Read`：向主存发送读命令
- **硬件动作**：地址总线被MAR的内容驱动，主存根据地址读取存储单元

#### T2：取指令到IR

- **数据流**：主存 → 数据总线 → MDR → 内部总线 → IR
- **控制信号**：
  - `MDR_in`：MDR从数据总线接收主存读出的数据
  - `MDR_out`：MDR的内容送上内部总线
  - `IR_in`：IR从总线接收指令
  - `PC_inc`：PC自增（可以与上述操作并行，若PC有独立的自增电路）

### 取指阶段的状态机模型

```
          ┌─────────────────────────┐
          │                         │
          ▼                         │
     ┌─────────┐   T0    ┌─────────┐
     │  Fe0    │────────→│  Fe1    │
     │PC→MAR   │         │Read     │
     └─────────┘         └────┬────┘
                              │ T2
                              ▼
                         ┌─────────┐
                         │  Fe2    │
                         │MDR→IR   │
                         │PC+1     │
                         └────┬────┘
                              │
                              ▼
                         译码→进入间址/执行周期
```

### 取指结束后的分支

取指结束后，CPU根据IR中的信息决定下一步：

```
取指完成（Fe2结束）
     │
     ├──→ 间接寻址标志=1 → 进入间址周期(IND)
     │
     └──→ 间接寻址标志=0 → 进入执行周期(EX)
```

## 直观理解

### 生活类比

取指就像**从自动取款机取一张纸条**：
1. 你告诉机器你要第几号柜子的纸条（PC→MAR）
2. 机器去那个柜子取纸条（Read + 主存读取）
3. 纸条通过传送带送到你手上（MDR→IR）
4. 你把柜子计数器加一（PC+1）

### 记忆技巧

- **取指三步**：送地址（T0）→ 读主存（T1）→ 取数据到IR（T2）
- **取指口诀**：「P-M-R-M-I-加」（PC→MAR, Read, M(MAR)→MDR→IR, PC+1）

## 知识关联

- 取指微操作是所有指令公共的，是微程序设计中"取指微程序"的基础
- 与间址、执行、中断微操作构成完整的指令执行过程
- 控制信号产生逻辑中，取指部分的表达式与指令操作码无关

### 易错陷阱

1. **陷阱**：PC+1的"1"不是数字1，而是指令长度。
2. **陷阱**：取指阶段读的是指令，不是数据。地址来源是PC，不是IR(Ad)。
3. **陷阱**：T2中MDR→IR和PC+1可能需要分两个时钟周期（取决于总线结构）。
