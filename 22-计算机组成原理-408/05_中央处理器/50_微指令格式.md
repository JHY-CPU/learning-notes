# 44_微指令格式

## 核心概念

- **微指令格式**：一条微指令的二进制编码方式，决定了如何表示微命令和后继地址。
- **微指令 = 控制字段 + 判别测试字段 + 下址字段**
- **格式设计目标**：在并行性和微指令长度之间取得平衡。

### 微指令的基本结构

```
┌──────────────────────────────────────────────────────────────┐
│                      微指令（μIR）                            │
├──────────────────┬─────────────────┬─────────────────────────┤
│   控制字段        │  判别测试字段    │     下址字段            │
│  (微命令编码)     │  (条件判断)      │  (下条微指令地址)       │
│  n位             │  k位             │  m位                   │
└──────────────────┴─────────────────┴─────────────────────────┘
```

### 各字段功能

**控制字段**：
- 编码了本条微指令需要发出的所有微命令
- 位数最多，是微指令长度的主要决定因素
- 编码方式：直接编码、字段直接编码、字段间接编码

**判别测试字段**：
- 标志当前微指令是否需要进行条件判断
- 用于实现微程序的分支（如判断是否间址、是否有中断）
- 位数通常为 1~4 位

**下址字段**：
- 指明下一条微指令在CM中的地址
- 位数 = $\lceil \log_2(\text{CM微指令条数}) \rceil$
- 若CM有128条微指令，则下址字段需要 7 位

## 原理分析

### 控制字段的位数计算

假设系统需要 $N$ 个互斥的微命令组，每组有 $n_i$ 个微命令：

**直接编码**：
$$\text{控制字段位数} = N$$

**字段直接编码**（每组增加1位用于"无操作"）：
$$\text{控制字段位数} = \sum_{i=1}^{N} \lceil \log_2(n_i + 1) \rceil$$

### 微指令字长的计算

$$\text{微指令字长} = \text{控制字段位数} + \text{判别测试字段位数} + \text{下址字段位数}$$

**例题**：某计算机有32个微命令，分为4组（每组内互斥），组大小分别为8、8、8、8个。CM有64条微指令。求微指令字长。

字段直接编码：
- 每组：$\lceil \log_2(8+1) \rceil = \lceil 3.17 \rceil = 4$ 位
- 控制字段：$4 \times 4 = 16$ 位
- 判别测试字段：假设 2 位
- 下址字段：$\lceil \log_2 64 \rceil = 6$ 位
- 微指令字长：$16 + 2 + 6 = 24$ 位

### 三种格式的对比

| 特性 | 水平型微指令 | 垂直型微指令 | 混合型微指令 |
|------|------------|------------|------------|
| 控制字段 | 宽，直接/字段编码 | 窄，类似机器指令 | 中等宽度 |
| 微指令长度 | 长 | 短 | 中等 |
| 并行度 | 高（多个微命令并行） | 低（一次一个微操作） | 中等 |
| 微程序长度 | 短 | 长 | 中等 |
| 编码效率 | 低 | 高 | 中等 |
| 设计难度 | 较难 | 较易 | 中等 |

## 直观理解

### 生活类比

微指令格式就像**遥控器的按键布局**：
- **水平型** = 万能遥控器：每个设备一个独立按键，功能多但遥控器很大
- **垂直型** = 简易遥控器：用模式键+数字键组合，遥控器小但操作步骤多
- **控制字段** = 功能按键区
- **下址字段** = "下一步"指示
- **判别测试字段** = "如果...则..."的选择器

### 记忆技巧

- **三字段**：「控判下」（控制字段、判别测试字段、下址字段）
- **控制字段最宽**：因为需要编码大量微命令
- **下址字段位数 = $\log_2(CM条数)$**

### 易错陷阱

1. **陷阱**：下址字段的位数取决于CM的容量（微指令总条数），不是机器指令条数。
2. **陷阱**：判别测试字段不是所有微指令都需要。无条件微指令的判别测试字段可以为全0。
3. **陷阱**：控制字段的位数不是简单地等于微命令个数。需要考虑互斥性分组和编码方式。
