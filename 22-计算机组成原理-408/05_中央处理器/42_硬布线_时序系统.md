# 36_硬布线_时序系统

## 核心概念

- **时序系统**：CPU内部的时钟信号层次结构，从大到小依次为：指令周期 → 机器周期（CPU周期）→ 时钟周期（节拍）。
- **节拍发生器**：产生等间隔的时钟脉冲 $T_0, T_1, T_2, T_3, \ldots$，控制微操作的执行顺序。
- **时序信号是硬布线控制器的三大输入之一**：操作码决定做什么，时序决定何时做。

### 三级时序层次

$$\text{指令周期} \supset \text{机器周期} \supset \text{时钟周期（节拍）}$$

| 层次 | 名称 | 说明 | 举例 |
|------|------|------|------|
| 第1级 | 指令周期 | 执行一条指令的完整时间 | 取指+执行 = 2~4个机器周期 |
| 第2级 | 机器周期 | 完成一个基本操作的时间 | 取指周期、间址周期、执行周期 |
| 第3级 | 时钟周期 | 一个节拍，完成一个微操作 | $T_0$完成PC→MAR，$T_1$完成M(MAR)→MDR |

## 原理分析

### 节拍发生器的实现

节拍发生器可以用**环形计数器**或**扭环形计数器**实现：

**环形计数器（4位）**：
```
初始: 1000 → 0100 → 0010 → 0001 → 1000 (循环)
       T0      T1      T2      T3
```

- 优点：译码简单，直接输出
- 缺点：n位只能产生n个节拍

**扭环形计数器（4位）**：
```
初始: 0000 → 1000 → 1100 → 1110 → 1111 → 0111 → 0011 → 0001 → 0000
```

- 优点：n位产生 2n 个节拍
- 缺点：需要译码

### 机器周期的划分

典型指令的机器周期划分：

```
┌───────┬───────┬───────┬───────┐
│ 取指  │ 间址  │ 执行  │ 中断  │
│  FE   │  IND  │  EX   │  INT  │
└───────┴───────┴───────┴───────┘
   S1      S2      S3      S4     (状态触发器)
```

每个机器周期包含若干个时钟周期（节拍）：
- **取指周期（FE）**：通常 3~4 个节拍
- **间址周期（IND）**：通常 3 个节拍
- **执行周期（EX）**：视指令复杂度，1~6 个节拍
- **中断周期（INT）**：通常 3 个节拍

### 节拍电位与节拍脉冲

- **节拍电位**（$T_0, T_1, T_2$）：高电平持续一个时钟周期，控制数据通路的门控信号
- **节拍脉冲**（$P_1, P_2$）：在节拍电位内出现的窄脉冲，用于寄存器的打入（写入）

```
时钟 ───┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐
        └──┘  └──┘  └──┘  └──┘  └──┘

T0 ────────────────┐
                   └────────────────────
T1 ────────────────────────────┐
                               └────────
P1 ──┐  ┌──────────────────────┐  ┌────
     └──┘                      └──┘
```

### 状态触发器

用触发器记录当前处于哪个机器周期：

| $S_4$ (INT) | $S_3$ (EX) | $S_2$ (IND) | $S_1$ (FE) | 当前周期 |
|-------------|------------|-------------|------------|---------|
| 0 | 0 | 0 | 1 | 取指（FE） |
| 0 | 0 | 1 | 0 | 间址（IND） |
| 0 | 1 | 0 | 0 | 执行（EX） |
| 1 | 0 | 0 | 0 | 中断（INT） |

## 直观理解

### 生活类比

时序系统就像**工厂的生产流水线**：
- **指令周期** = 生产一个完整产品（如一辆汽车）
- **机器周期** = 一个生产工位（如组装底盘、安装发动机）
- **时钟周期** = 工位上的一个动作步骤（如拧一个螺丝）
- **节拍发生器** = 流水线的传送带定时器，每隔固定时间向前推进一步

### 记忆技巧

- **三级时序**：「指令机时」（指令周期 > 机器周期 > 时钟周期）
- **节拍**：就是"心跳"，CPU每"跳"一下完成一个微操作
- **状态触发器**：记住"我现在在哪一步"

## 知识关联

### 跨章节联系

- **第3章（存储系统）**：存储器的读写需要一个完整的机器周期
- **第5章（硬布线控制器）**：时序信号是控制信号产生的输入之一
- **第5章（微程序控制器）**：微程序控制器中，微指令的执行也有时序

### 408考点

1. **给定机器周期和节拍数，计算指令执行时间**
2. **判断某时刻的时序信号状态**（$T_i$ = ?，$S_j$ = ?）
3. **节拍发生器的设计**（环形计数器的状态转移）

### 易错陷阱

1. **陷阱**：不是所有指令都有间址周期和中断周期，但所有指令都有取指周期。
2. **陷阱**：不同指令的执行周期长度可能不同，硬布线设计需要处理"周期结束"的判断。
3. **陷阱**：节拍脉冲（P）和节拍电位（T）的作用不同，P用于寄存器打入，T用于门控。
