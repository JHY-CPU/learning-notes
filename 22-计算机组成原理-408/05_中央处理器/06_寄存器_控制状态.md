# 06_寄存器_控制状态

## 核心概念

- **控制和状态寄存器（Control and Status Registers）**：CPU内部控制用的寄存器，对用户（程序员）**透明**（不可见/不可直接访问）。
- **关键寄存器**：MAR（存储器地址寄存器）、MDR（存储器数据寄存器）、IR（指令寄存器）。
- **透明的含义**：用户无法通过任何指令直接读写这些寄存器的内容，它们完全由CPU硬件自动管理。

### 控制与状态寄存器一览

| 寄存器 | 英文全称 | 位数 | 功能 | 用户可见性 |
|--------|----------|------|------|-----------|
| MAR | Memory Address Register | 地址线宽度 | 存放待访问的主存地址 | 透明 |
| MDR | Memory Data Register | 数据线宽度 | 存放与主存交换的数据 | 透明 |
| IR | Instruction Register | 指令字长 | 存放当前正在执行的指令 | 透明 |
| 时序寄存器 | — | — | 存放当前时序状态 | 透明 |
| 中断请求寄存器 | IRR | — | 记录各中断源的请求 | 透明 |
| 中断屏蔽寄存器 | IMR | — | 屏蔽各中断源 | 部分透明 |

## 原理分析

### MAR（存储器地址寄存器）

- **功能**：存放CPU要访问的主存单元的地址。
- **位数**：等于地址总线的宽度，决定了可寻址的主存空间大小。若MAR为n位，则可寻址空间为 $2^n$ 个存储单元。
- **数据来源**：
  - 取指周期：PC → MAR
  - 间址周期：IR的地址码字段 → MAR
  - 执行周期：有效地址 → MAR
- **数据去向**：MAR的内容 → 地址总线 → 主存

### MDR（存储器数据寄存器）

- **功能**：存放CPU与主存之间交换的数据。
- **位数**：等于数据总线的宽度，通常等于机器字长。
- **数据来源**：
  - 读操作：主存数据 → MDR（通过数据总线）
  - 写操作：CPU内部数据 → MDR → 数据总线 → 主存
- **双向性**：MDR是双向寄存器，既可接收主存读出的数据，也可向主存发送待写入的数据。

### IR（指令寄存器）

- **功能**：存放当前正在执行的指令。
- **数据来源**：取指周期结束后，MDR → IR。
- **数据去向**：
  - 操作码字段 → 指令译码器 → CU
  - 地址码字段 → MAR（取操作数时）
- **指令格式示例**：

```
┌──────────────────────────────────────┐
│     操作码OP     │    地址码Ad       │
│   (送CU译码)     │  (送MAR取操作数)   │
└──────────────────────────────────────┘
```

### MAR/MDR的位置问题

**重要考点**：MAR和MDR在逻辑上属于CPU还是主存？

- **冯·诺依曼原始定义**：MAR和MDR属于存储器（主存的接口寄存器）。
- **现代计算机设计**：MAR和MDR集成在CPU芯片内部，作为CPU与主存之间的缓冲接口。
- **考试原则**：以题目给出的架构模型为准。若题目说"CPU内部"，则MAR/MDR在CPU内；若题目说"存储器接口"，则归为主存侧。

## 直观理解

### 生活类比

- **MAR**= 你要去图书馆借书，先在电脑上输入的**书架编号**（告诉图书馆找哪里）
- **MDR**= 图书馆找到书后递给你的**书本身**（读取时），或者你递给图书馆要还的**书**（写入时）
- **IR**= 你手头正在执行的**任务清单**（当前在做的事）
- **MAR/MDR透明**= 你不需要关心图书管理员怎么找到书、怎么传递书，你只需要说出书名（地址），拿到书（数据）就行

### 记忆技巧

- **MAR = 地址 = 找哪里**（Memory **Address** Register）
- **MDR = 数据 = 拿什么**（Memory **Data** Register）
- **IR = 指令 = 做什么**（Instruction Register）
- **透明**的意思：用户（程序员）看不见、管不了，硬件自动处理

### MAR位数与存储容量关系

$$\text{存储容量} = 2^{MAR位数} \times MDR位数$$

例如：MAR = 16位，MDR = 8位，则存储容量 = $2^{16} \times 8$ bit = 64KB。

### 常考组合

- 取指过程中PC→MAR→地址总线→主存→MDR→IR的完整数据流

- 用户可见寄存器 vs 透明寄存器的对比判断

### 易错陷阱

1. **陷阱**：MAR的位数不一定等于机器字长。MAR位数取决于地址空间大小，MDR位数通常等于机器字长。
2. **陷阱**：有些教材把IR也列为"用户可见"（因为用户可以看到IR存放的指令），但408考试中一般认为IR是透明的。
3. **陷阱**：MAR/MDR的位置问题——必须以题目描述为准，不要想当然。
