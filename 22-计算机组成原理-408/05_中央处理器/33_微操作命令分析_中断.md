# 33_微操作命令分析_中断

## 核心概念

- **中断微操作命令**：CPU在指令执行完毕后响应中断时发出的控制信号序列。
- **中断响应的条件**：有中断请求（INTR=1）且中断允许（EINT=1）。
- **中断周期的核心任务**：保存断点 → 关中断 → 转向中断服务程序。

### 中断微操作序列

| 节拍 | 微操作 | 控制信号 | 说明 |
|------|--------|---------|------|
| T0(Int0) | 0 → EINT | CLR_EINT | 关中断（清除中断允许标志） |
| T1(Int1) | SP - 1 → SP | SP_dec | 栈指针下移 |
| T1 | SP → MAR | SP_out, MAR_in | 栈顶地址送MAR |
| T2(Int2) | PC → MDR | PC_out, MDR_in | 断点（PC值）送MDR |
| T2 | 1 → W | Write | 发出写命令 |
| T3(Int3) | MDR → M(MAR) | — | 断点压入堆栈 |
| T3 | 中断向量地址 → PC | VecAddr_out, PC_in | PC指向中断服务程序入口 |

## 原理分析

### 中断响应的完整数据流

```
中断周期开始
     │
     ▼
┌─────────────────────────┐
│ 0 → EINT（关中断）       │  硬件自动完成
└──────────┬──────────────┘
           │
           ▼
┌─────────────────────────┐
│ SP - 1 → SP → MAR       │  栈指针下移，地址送MAR
└──────────┬──────────────┘
           │
           ▼
┌─────────────────────────┐
│ PC → MDR → M(MAR)       │  断点压栈
│ 1 → W                   │
└──────────┬──────────────┘
           │
           ▼
┌─────────────────────────┐
│ 中断向量地址 → PC        │  转向中断服务程序
└──────────┬──────────────┘
           │
           ▼
      取指（开始执行中断服务程序）
```

### 中断向量的获取方式

| 方式 | 说明 | 微操作 |
|------|------|--------|
| 向量中断 | 中断类型号×每项字节数+向量表基址 | 硬件计算向量地址 |
| 非向量中断 | 固定地址存放查询程序入口 | 软件查询中断源 |

**向量中断**（更高效）：

$$\text{向量地址} = \text{向量表基址} + \text{中断类型号} \times \text{项大小}$$

硬件直接计算并送入PC。

**非向量中断**（更灵活）：
- PC被设为固定的中断查询程序入口地址
- 查询程序通过软件逐一检查中断源
- 确定中断源后再跳转到对应的中断服务程序

### 中断周期的进入条件

$$\text{进入中断} = EX_{结束} \cdot INTR \cdot EINT$$

- $EX_{结束}$：执行周期结束
- $INTR$：有中断请求（中断请求寄存器某位=1）
- $EINT$：中断允许标志=1

### 中断返回（IRET指令执行阶段）

| 节拍 | 微操作 | 控制信号 |
|------|--------|---------|
| T0 | SP → MAR | SP_out, MAR_in |
| T1 | 1 → R | Read |
| T2 | M(MAR) → MDR | MDR_in |
| T3 | MDR → PC | MDR_out, PC_in |
| T4 | SP + 1 → SP | SP_inc |

## 直观理解

### 生活类比

中断就像**工作中被打断**：
1. **关中断**= 告诉同事"暂时不要打扰我处理中断"（防止中断嵌套）
2. **保存断点**= 在笔记本上记下"我刚才做到第几页了"
3. **转向服务程序**= 翻到紧急事务处理手册的第一页
4. **中断返回**= 紧急事务处理完，翻回笔记本，继续之前的工作

### 记忆技巧

- **中断四步口诀**：「关保写转」（关中断→保存断点→写栈→转服务程序）
- **中断周期触发条件**：「有请求 + 允许中断」（INTR AND EINT）

### 易错陷阱

1. **陷阱**：关中断是**硬件自动**完成的（0→EINT），不是软件实现的。
2. **陷阱**：中断周期保存的是PC（断点地址），通用寄存器由中断服务程序的软件保存（PUSH指令）。
3. **陷阱**：中断向量地址 ≠ 中断服务程序入口地址。向量地址是向量表中的某项地址，其中存放的内容才是服务程序入口。
