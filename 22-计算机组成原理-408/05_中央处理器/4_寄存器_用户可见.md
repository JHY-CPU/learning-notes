# 05_寄存器_用户可见

## 核心概念

- **用户可见寄存器（Programmer-Visible Registers）**：程序员可以通过机器指令或汇编指令直接访问的寄存器。
- **与透明寄存器的区分**：用户可见寄存器可被指令操作；对用户透明的寄存器（如MAR、MDR）不能被指令直接访问。

### 用户可见寄存器一览

| 寄存器 | 英文 | 位数 | 功能 |
|--------|------|------|------|
| **程序计数器PC** | Program Counter | 机器字长 | 存放下一条指令的地址 |
| **通用寄存器组** | General Purpose Registers | 机器字长 | 存放操作数、地址、中间结果 |
| **程序状态字PSW** | Program Status Word | 机器字长 | 存放状态标志和控制位 |
| **累加器ACC** | Accumulator | 机器字长 | 存放操作数和运算结果 |
| **变址寄存器** | Index Register | 机器字长 | 变址寻址时使用 |
| **基址寄存器** | Base Register | 机器字长 | 基址寻址时使用（注意与PSW中的基址寄存器区分） |
| **堆栈指针SP** | Stack Pointer | 机器字长 | 指向栈顶元素 |

## 原理分析

### PC（程序计数器）

$$\text{取指后：} PC \leftarrow PC + \Delta \quad (\Delta = \text{指令字长})$$

- 顺序执行时：PC每取一条指令就 + $\Delta$
- 转移执行时：$PC \leftarrow \text{目标地址}$（由转移指令执行阶段修改）
- PC的内容可以被指令读取（如CALL指令需要保存返回地址）

### 通用寄存器组

- 典型的x86架构：EAX, EBX, ECX, EDX, ESI, EDI, EBP, ESP（8个32位通用寄存器）
- RISC架构（如MIPS）：$0~$31（32个32位通用寄存器）
- 通用寄存器可以存放：
  - 操作数（参与ALU运算）
  - 地址（用于间接寻址）
  - 中间结果
  - 返回地址（CALL指令保存的PC值）

### PSW（程序状态字）

PSW包含**状态标志位**和**控制位**：

| 位 | 名称 | 含义 |
|----|------|------|
| CF | 进位标志 | 加法有进位或减法有借位时置1 |
| ZF | 零标志 | 运算结果为0时置1 |
| SF | 符号标志 | 运算结果最高位为1时置1（负数） |
| OF | 溢出标志 | 运算发生溢出时置1 |
| IF | 中断允许标志 | 1=允许中断，0=禁止中断 |
| TF | 陷阱标志 | 1=单步调试模式 |

- **PSW可通过指令访问**：如x86的PUSHF/POPF（压入/弹出标志寄存器），8086的LAHF/SAHF。
- **条件转移指令依赖PSW**：如JE（ZF=1则跳转）、JG（ZF=0且SF=OF则跳转）。

### SP（堆栈指针）

- SP指向栈顶元素（满递减栈）或栈顶下一个空位（空递减栈）。
- PUSH操作：$SP \leftarrow SP - 1$，然后数据写入 $(SP)$ 所指位置。
- POP操作：从 $(SP)$ 所指位置读取数据，然后 $SP \leftarrow SP + 1$。
- CALL指令：$SP \leftarrow SP - 1$，$(SP) \leftarrow PC$（保存返回地址）。
- RET指令：$PC \leftarrow (SP)$，$SP \leftarrow SP + 1$。

## 直观理解

### 生活类比

- **PC**= GPS导航中"下一个路口"的指示
- **通用寄存器**= 随身携带的便携文件夹，随时可以存取文件
- **PSW**= 驾驶仪表盘，显示车速（正负）、油量（是否为空）、发动机状态（是否异常）
- **SP**= 一摞盘子的顶部标记，放盘子时往上加，取盘子时从顶部取

### 记忆技巧

- **用户可见**的关键特征：**汇编指令可以直接操作这些寄存器**
- 「**P通P堆P**」：PC、通用寄存器（General）、PSW、SP 都是用户可见的
- PSW = 程序状态字 = "成绩单"，记录了上次运算的结果特征

### 易错陷阱

1. **陷阱**：PC是否"用户可见"？严格来说，PC可以通过转移指令间接修改，也可以通过某些特殊指令直接读取（如x86没有直接读取EIP的指令，但可通过CALL间接保存），考试中通常认为PC是用户可见的。
2. **陷阱**：MAR/MDR/IR是用户不可见（透明）的，但PC是用户可见的。这是一个高频对比考点。
3. **陷阱**：基址寄存器（Base Register）和变址寄存器（Index Register）在不同CPU架构中可能与通用寄存器合并，也可能独立存在。x86中EBP是基址指针，ESI/EDI是变址寄存器。
4. **陷阱**：PSW中的IF位（中断允许标志）影响中断响应，与第8章中断机制紧密关联。
