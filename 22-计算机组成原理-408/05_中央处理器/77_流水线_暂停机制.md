# 77_流水线_暂停机制

## 核心概念

- **暂停（Stall）**：流水线中某些段空闲等待，不推进指令
- **气泡（Bubble）**：插入的空操作（NOP），占用流水线但不做有用工作
- 暂停是解决流水线冲突的**通用手段**

## 原理分析

### 一、暂停的触发条件

1. **结构冲突**：资源被占用
2. **数据冲突**：无法通过旁路解决的 RAW
3. **控制冲突**：转移指令结果未确定

### 二、暂停的硬件实现

#### 1. 流水线寄存器冻结

- 当检测到需要暂停时，**保持当前流水线寄存器的值不变**
- 不写入新的指令

```
if (stall_condition) then
    IF/IR ← IF/IR      // 不更新
    PC ← PC             // PC 不推进
    // 流水线寄存器保持不变
```

#### 2. 插入气泡（NOP）

- 将流水线中某段的控制信号**全部置为无效**
- 该指令不产生任何微操作

```
if (bubble) then
    控制信号全0  // 相当于插入一条 NOP 指令
```

### 三、暂停的类型

#### 1. 数据暂停

**Load-Use 暂停**：
```
检测条件：
if (ID/EX.MemRead == 1) &&           // 前一条是 Load
   ((ID/EX.Rd == IF/ID.Rs1) ||       // 目的寄存器 = 下一条的源寄存器
    (ID/EX.Rd == IF/ID.Rs2)) then
    stall = 1                         // 暂停 1 周期
```

#### 2. 控制暂停

**转移暂停**：
```
if (Branch taken) then
    flush IF/ID, ID/EX     // 作废已取指令
    PC ← branch_target      // PC 转向目标地址
```

### 四、暂停对性能的影响

每暂停 1 个周期，CPI 增加 1/n（n 为指令数）：

$$CPI = 1 + \frac{\text{总暂停周期数}}{n}$$

### 五、时空图中的暂停表示

```
时间→    1    2    3    4    5    6    7
I1:     [IF] [ID] [EX][MEM][WB]
I2:          [IF] [ID] [--] [EX][MEM][WB]  ← 气泡
I3:               [IF] [--] [ID] [EX]...   ← 气泡
```

`[--]` 表示气泡：该周期该段不做工作。

## 直观理解

**暂停**就像**流水线工厂临时停工**：

- 某个工位发现零件还没到（数据未就绪），整条线暂停等待
- 气泡 = 空转的工位，不做任何操作
- 暂停越多，工厂效率越低

## 知识关联

- 与**数据冲突**（第 70-72 节）的关系：旁路不能解决时需要暂停
- 与**控制冲突**（第 74 节）的关系：转移结果不确定时暂停
- 与**结构冲突**（第 69 节）的关系：资源不足时暂停
- 暂停直接影响**流水线性能指标**（第 64-67 节）
