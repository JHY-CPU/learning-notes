# 101_多核处理器

## 核心概念

- **多核处理器**：在单个芯片上集成多个处理核心
- **共享资源**：通常共享最后一级 Cache（LLC）和内存
- **核间通信**：通过共享内存或消息传递

## 原理分析

### 一、多核架构

```
┌─────────────────────────────────────────┐
│               多核处理器芯片              │
│  ┌─────────┐              ┌─────────┐   │
│  │ Core 0  │              │ Core 1  │   │
│  │ L1 I$   │              │ L1 I$   │   │
│  │ L1 D$   │              │ L1 D$   │   │
│  │ L2 $    │              │ L2 $    │   │
│  └────┬────┘              └────┬────┘   │
│       │                        │        │
│  ┌────┴────────────────────────┴────┐   │
│  │        共享 L3 Cache (LLC)        │   │
│  └────────────────┬─────────────────┘   │
│                   │                      │
│  ┌────────────────┴─────────────────┐   │
│  │           内存控制器               │   │
│  └────────────────┬─────────────────┘   │
└───────────────────┼──────────────────────┘
                    │
              ┌─────┴─────┐
              │   主存     │
              └───────────┘
```

### 二、Cache 一致性

多核共享内存时，每个核心有自己的 Cache，可能出现**数据不一致**。

**解决方案**：
- **监听协议（Snooping）**：每个核心监听总线上的内存访问
- **目录协议（Directory）**：集中记录每个缓存块的状态
- **MESI 协议**：Modified, Exclusive, Shared, Invalid 四种状态

### 三、核间通信

| 方式 | 说明 | 优点 | 缺点 |
|------|------|------|------|
| 共享内存 | 通过共享变量通信 | 简单直观 | 需要同步机制 |
| 消息传递 | 发送/接收消息 | 无共享冲突 | 编程复杂 |

### 四、Amdahl 定律

程序中有一部分**不可并行**（串行部分），设串行比例为 $f$：

$$S_{max} = \frac{1}{f + \frac{1-f}{n}}$$

其中 $n$ 为处理器核数。

- $f = 0.1$（10% 串行），$n = 100$：$S_{max} = \frac{1}{0.1 + 0.009} \approx 9.17$
- **启示**：串行部分是瓶颈，增加核数的收益递减

### 五、多核的挑战

1. **并行编程困难**：需要将程序分解为并行任务
2. **负载均衡**：各核的工作量要尽量均匀
3. **同步开销**：锁、信号量等同步机制有开销
4. **Cache 一致性**：维护多核间数据一致性

## 直观理解

**多核处理器**像**多车道厨房**：

- 每个厨师（核心）有自己的小冰箱（L1 Cache）
- 大家共用一个大冰箱（共享 LLC/内存）
- 如果两个厨师同时拿同一种食材，需要协调（Cache 一致性）
- Amdahl 定律：如果有些菜只能一个人做（串行部分），加再多厨师也没用

## 知识关联

- 与**Cache**（第 6 章）的关系：多核 Cache 一致性
- 与**Amdahl 定律**的关系：多核加速比的上限
- 与**流水线**（第 62-85 节）的关系：每个核内部可以是流水线的
- 与**多线程**（第 102 节）的关系：多核 + 多线程是现代处理器的趋势
- 与**操作系统**的关系：多核调度、同步是 OS 重要内容
