# 07_PC程序计数器

## 核心概念

- **PC（Program Counter，程序计数器）**：又称**指令指针IP**（x86中），存放**下一条待取指令**的内存地址。
- **PC是用户可见寄存器**：可通过转移指令修改，也可通过特殊指令读取。
- **PC的自动更新**：每取一条指令后，PC自动递增，指向下一条指令。
- **PC的被动修改**：执行转移/跳转/调用/返回指令时，PC被修改为目标地址。

### PC的关键特性

| 特性 | 说明 |
|------|------|
| 位数 | 等于地址线宽度（与MAR位数一致） |
| 初始值 | 程序入口地址（由操作系统加载） |
| 自增规则 | PC ← PC + 指令长度（定长指令中为固定值） |
| 被修改方式 | 转移指令、调用指令、返回指令、中断响应 |
| 用户可见性 | 可见（可通过指令间接访问） |

## 原理分析

### PC的自增规则

#### 定长指令字长（如RISC）

若指令字长 = 存储字长 = $L$ 字节，按字节编址：

$$PC \leftarrow PC + L$$

例如：MIPS指令为32位（4字节），每取一条指令后 $PC \leftarrow PC + 4$。

#### 变长指令字长（如CISC）

取指阶段需先分析指令长度，PC增加相应的字节数：

$$PC \leftarrow PC + \text{当前指令长度}$$

### PC在指令执行各阶段的变化

#### 顺序执行

```
初始：PC = 1000
取指后：PC = 1004（假设指令长4字节）
执行：PC保持1004
取下一条：PC = 1008
...
```

#### 转移指令执行

以"JMP 2000"（无条件跳转到地址2000）为例：

```
取指阶段：PC = 1000 → 取出JMP指令 → PC ← 1004（自动+4）
执行阶段：PC ← 2000（被指令修改）
下一条取指：从地址2000处取指令
```

#### 条件转移

以"JZ 2000"（零标志为1则跳转到2000）为例：

```
若 ZF = 1：PC ← 2000（跳转）
若 ZF = 0：PC保持原值1004（顺序执行）
```

#### 子程序调用（CALL）

```
CALL 3000 的执行过程：
1. 取指：PC ← PC + 指令长度（假设为1004）
2. 保存返回地址：SP ← SP - 1, M(SP) ← 1004（将1004压栈）
3. PC ← 3000（跳转到子程序入口）
```

#### 子程序返回（RET）

```
RET 的执行过程：
1. PC ← M(SP)（从栈顶取出返回地址）
2. SP ← SP + 1
3. 从PC所指地址继续执行
```

#### 中断响应

```
中断响应过程：
1. 关中断
2. SP ← SP - 1, M(SP) ← PC（保存断点）
3. PC ← 中断向量地址（转向中断服务程序入口）
```

### PC与MAR的关系

- 在取指阶段：**PC → MAR**（PC的内容送到MAR，作为访问主存的地址）。
- 两者位数相同（都等于地址线宽度），但功能不同：
  - PC指向下一条指令
  - MAR指向当前要访问的存储单元（可能是指令，也可能是数据）

## 直观理解

### 生活类比

- **PC**= 阅读文章时，手指指着的"下一个要读的字"的位置
- **PC自增**= 读完一个字，手指自然向右移一个位置
- **转移指令修改PC**= 文章说"跳到第5段"，手指就直接指到第5段
- **CALL压栈保存PC**= "跳到第5段之前，先用书签记住当前位置"
- **RET恢复PC**= "做完第5段的事，翻回书签位置继续"

### 记忆技巧

- **PC的核心一句话**：永远指向"下一条"要取的指令
- **PC的变化**只有三种情况：
  1. 自增（取指后）
  2. 被转移指令修改
  3. 被中断响应修改

### 常考组合

- 取指过程中PC的数据流：PC → MAR → 地址总线 → 主存
- CALL/RET指令执行过程中PC和SP的变化
- 中断响应时PC的保存与恢复
- PC位数与MAR位数、地址总线宽度三者相等

### 易错陷阱

1. **陷阱**：PC存的是"下一条"指令地址，不是"当前正在执行的"指令地址。当前指令在IR中。
2. **陷阱**：取指过程中PC会"先自增"还是"后自增"？标准流程是取指后PC自动 + 指令长度。但有些题目定义PC在取指前就指向"当前指令"，取指同时PC+1，考试以题目描述为准。
3. **陷阱**：条件转移指令如果条件不满足，PC保持顺序递增的结果，继续取下一条指令。
4. **陷阱**：PC的位数与地址总线宽度一致，而不是与数据总线宽度一致。
