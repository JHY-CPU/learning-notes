# 54_微程序设计_中断微程序

## 核心概念

### 中断微程序的定义
中断微程序是微程序控制器中处理**中断请求**的专用微程序序列。当CPU检测到中断请求时，会暂停当前指令的执行，转而去执行中断微程序，完成中断响应的准备工作。

### 中断微程序与中断隐指令的关系
- **中断隐指令**：CPU在响应中断时自动执行的一系列硬件操作，不是真正的指令，而是由控制器产生的微操作序列
- **中断微程序**：实现中断隐指令功能的微程序代码，存储在**控制存储器（CM）**中
- 中断隐指令的功能：
  1. **关中断**：将中断允许标志IF置0
  2. **保存断点（PC）**：将当前PC值压入堆栈或写入特定存储单元
  3. **保存PSW**：将程序状态字压入堆栈
  4. **转中断服务程序**：将中断服务程序入口地址送入PC

### 关键术语
| 术语 | 含义 |
|------|------|
| 中断响应微程序 | 完成中断响应阶段工作的微程序 |
| 中断服务微程序 | 实际处理中断请求的微程序 |
| 中断向量 | 中断服务程序的入口地址 |
| 中断类型号 | 标识中断源的编号 |
| 断点 | 被中断的指令地址（即当前PC值） |
| 现场 | CPU中所有需要保护的信息（PC、PSW、通用寄存器等） |

## 原理分析

### 中断微程序的工作流程

```
中断请求到来
    ↓
当前指令执行完毕（取指/执行完检查中断）
    ↓
若中断允许位IF=1且有中断请求
    ↓
进入中断响应微程序
    ↓
┌─────────────────────────────┐
│ 微指令1: IF←0（关中断）      │
│ 微指令2: MAR←SP-1            │
│ 微指令3: M(MAR)←PC（压入PC） │
│ 微指令4: MAR←SP-1            │
│ 微指令5: M(MAR)←PSW（压入PSW）│
│ 微指令6: PC←中断向量地址     │
│ 微指令7: 取指（开始执行中断服务程序）│
└─────────────────────────────┘
```

### 中断隐指令的微操作序列

**关中断微操作**：
$$\text{IF} \leftarrow 0$$

**保存PC（断点）微操作**：
$$\text{MAR} \leftarrow \text{SP} - 1$$
$$\text{MDR} \leftarrow \text{PC}$$
$$\text{M}[\text{MAR}] \leftarrow \text{MDR}$$

**保存PSW微操作**：
$$\text{MAR} \leftarrow \text{SP} - 1$$
$$\text{MDR} \leftarrow \text{PSW}$$
$$\text{M}[\text{MAR}] \leftarrow \text{MDR}$$

**转中断服务程序微操作**：
$$\text{PC} \leftarrow \text{中断向量表}[\text{中断类型号}]$$

### 中断微程序在控制存储器中的位置

控制存储器（CM）的布局：
| 地址范围 | 内容 |
|---------|------|
| 0000H~00FFH | 取指微程序 |
| 0100H~01FFH | 指令1的微程序 |
| 0200H~02FFH | 指令2的微程序 |
| ... | ... |
| FF00H~FFFFH | **中断响应微程序** |

中断微程序一般放在控制存储器的**高地址区域**或**保留区域**。

## 直观理解

### 生活类比：快递代收
想象你正在认真看书（执行指令），突然门铃响了（中断请求）。你需要：
1. 先暂停看书（关中断，IF=0）
2. 在书上夹个书签（保存PC/断点）
3. 记住当前看到哪一页（保存PSW/现场）
4. 去开门处理快递（转中断服务程序）
5. 处理完回来，找到书签继续看（恢复现场，返回断点）

### 记忆技巧
**"关保保转"** —— 中断隐指令四步骤：
- **关**中断（IF=0）
- **保**护断点（PC入栈）
- **保**护现场（PSW入栈）
- **转**中断服务程序（PC←中断向量）

### 常考组合与易错陷阱

**易错陷阱1**：
- 关中断操作由**硬件自动完成**（中断隐指令的一部分），不需要程序员编写指令
- 而中断返回指令（如x86的IRET）是**软件指令**，属于中断服务程序的一部分

**易错陷阱2**：
- 中断隐指令的执行是在**中断响应阶段**，不是在中断服务阶段
- 中断响应阶段：关中断、保存断点、找入口
- 中断服务阶段：开中断（允许嵌套）、保存通用寄存器、执行服务、恢复通用寄存器、关中断、开中断返回

**易错陷阱3**：
- 在微程序控制器中，中断隐指令由微程序实现
- 在硬布线控制器中，中断隐指令由组合逻辑电路实现
- 两种方式实现的功能完全相同

**408常见考法**：
- 给出中断响应过程，要求写出微操作序列
- 判断"中断隐指令是否为一条真正的指令"（不是！）
- 中断响应微程序在CM中的地址位置分析
