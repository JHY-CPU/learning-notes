# 22_数据通路

## 核心概念

- **数据通路（Datapath）**：CPU内部数据从一个部件流向另一个部件的物理路径，包括寄存器、ALU、总线和它们之间的连接。
- **数据通路是控制器的"被控对象"**：CU产生的控制信号作用于数据通路，控制数据的流动方向和操作。
- **数据通路的两大组成**：**操作元件**（ALU、存储器）和**状态元件**（寄存器、触发器）。
- **数据通路的宽度**：等于机器字长。

### 数据通路的组成要素

| 类型 | 部件 | 特点 |
|------|------|------|
| 操作元件 | ALU | 组合逻辑，无记忆功能 |
| 操作元件 | 主存 | 可读可写 |
| 操作元件 | 符号扩展器 | 组合逻辑 |
| 状态元件 | 通用寄存器组 | 有记忆功能，时钟边沿更新 |
| 状态元件 | PC | 有记忆功能 |
| 状态元件 | IR | 有记忆功能 |
| 状态元件 | MAR, MDR | 有记忆功能 |
| 连接元件 | 内部总线 | 传输数据 |
| 连接元件 | 多路选择器（MUX） | 选择数据来源 |

## 原理分析

### 数据通路的基本结构

```
┌──────────────────────────────────────────────────┐
│                    数据通路                        │
│                                                  │
│  ┌─────┐  ┌─────┐  ┌─────┐  ┌─────┐            │
│  │ PC  │  │ R0  │  │ R1  │  │ R2  │  ...        │
│  └──┬──┘  └──┬──┘  └──┬──┘  └──┬──┘            │
│     │        │        │        │                 │
│  ═══╪════════╪════════╪════════╪══════════════   │ 内部总线
│     │        │        │        │                 │
│     └────────┴────┬───┴────────┘                 │
│                   │                              │
│            ┌──────┴──────┐                       │
│            │  MUX(A/B)   │  ← 选择操作数来源     │
│            └──────┬──────┘                       │
│                   │                              │

│            └──────┬──────┘                       │

│  ┌─────┐    ┌────┴────┐    ┌─────┐              │
```

### 数据通路上的典型数据流动

#### 取指操作的数据流

```
PC ──out──→ 内部总线 ──in──→ MAR
MAR ──→ 地址总线 ──→ 主存
主存 ──→ 数据总线 ──→ MDR
MDR ──out──→ 内部总线 ──in──→ IR
同时：PC + 1 ──→ PC
```

控制信号：PC_out, MAR_in, Read, MDR_in, MDR_out, IR_in, PC_inc

#### 加法操作的数据流（ADD R1, R2 → R1）

```
R1 ──out──→ 总线A ──→ ALU输入A
R2 ──out──→ 总线B ──→ ALU输入B
ALU: Add操作
ALU结果 ──→ 结果总线 ──in──→ R1
```

控制信号：R1_out_A, R2_out_B, ALU_Add, R1_in

### 微操作与控制信号的对应关系

| 微操作 | 需要的控制信号 |
|--------|---------------|
| PC → MAR | PC_out, MAR_in |
| Read | Read |
| M(MAR) → MDR | MDR_in（隐含主存数据输出） |
| MDR → IR | MDR_out, IR_in |
| PC + 1 | PC_inc |
| R1 → Y | R1_out, Y_in |
| Y + R2 → R1 | R2_out, ALU_Add, R1_in |

## 直观理解

### 生活类比

数据通路就像**工厂的传送带系统**：
- **寄存器**= 各个工位的物料架
- **总线**= 传送带
- **ALU**= 加工机器
- **控制信号**= 各个闸门/开关的开合指令
- **微操作**= "从A架取料放到传送带，传到加工机，加工后放到B架"

### 记忆技巧

- **数据通路 = "路" + "车" + "红绿灯"**：路是总线，车是数据，红绿灯是控制信号
- **控制器决定红绿灯的时序，数据通路是红绿灯控制的对象**

### 易错陷阱

1. **陷阱**：单总线结构中同一时刻只有一个数据在总线上传输，需要分时使用。
2. **陷阱**：ALU是组合逻辑，输入变输出立即变。需要输入锁存器（Y）保持第一个操作数稳定。
3. **陷阱**：控制信号不是随意发出的，必须按照时序节拍有序发出。
