# 52_微程序设计_间址微程序

## 核心概念

- **间址微程序**：实现间接寻址功能的微程序，将指令地址字段所指单元的内容（有效地址）取出。
- **公共性**：所有使用间接寻址的指令共享同一段间址微程序。
- **入口地址**：间址微程序在CM中的固定起始地址。

### 间址微程序的微操作序列

| 微指令地址 | 微操作 | 说明 |
|-----------|--------|------|
| 003 | $Ad(IR) \to MAR$ | 指令地址段送MAR |
| 004 | $M(MAR) \to MDR$ | 读存储器，取出有效地址 |
| 005 | $MDR \to Ad(IR)$ | 有效地址写回IR的地址段 |

### 间址周期与取指周期的对比

| 对比项 | 取指周期 | 间址周期 |
|--------|---------|---------|
| 功能 | 取出指令 | 取出有效地址 |
| 起始地址 | $PC \to MAR$ | $Ad(IR) \to MAR$ |
| 读操作 | $M(MAR) \to IR$ | $M(MAR) \to MDR$ |
| 写回 | $MDR \to IR$ | $MDR \to Ad(IR)$ |
| PC变化 | $PC+1$ | PC不变 |
| 共享性 | 所有指令共享 | 间址指令共享 |

## 原理分析

### 间址微程序的详细步骤

**第一次间址**（单级间址）：

```
003H: Ad(IR) → MAR     // IR中的地址段送MAR
004H: M(MAR) → MDR     // 从主存读出有效地址
005H: MDR → Ad(IR)     // 有效地址写回IR地址段
```

**多级间址**（如果支持）：

多级间址需要循环执行间址微程序，直到取出的数据不再是地址（用标志位判断）。

```
003H: Ad(IR) → MAR
004H: M(MAR) → MDR
005H: MDR → Ad(IR)
      │
      ├── 间址标志=1（还是地址）→ 回到003H
      └── 间址标志=0（是数据）→ 转执行微程序
```

### 间址微程序的控制信号

**003号微指令**（$Ad(IR) \to MAR$）：

| 微命令 | 值 |
|--------|---|
| $Ad(IR)_{\text{out}}$ | 1 |
| $MAR_{\text{in}}$ | 1 |

**004号微指令**（$M(MAR) \to MDR$）：

| 微命令 | 值 |
|--------|---|
| Read | 1 |
| $MDR_{\text{in}}$ | 1 |

**005号微指令**（$MDR \to Ad(IR)$）：

| 微命令 | 值 |
|--------|---|
| $MDR_{\text{out}}$ | 1 |
| $Ad(IR)_{\text{in}}$ | 1 |

### 间址的条件触发

不是所有指令都需要间址周期。在取指微程序结束时，需要判断：

```
取指结束
    │
    ├── 间接寻址标志 = 1 → 转间址微程序入口（003H）
    │                           │
    │                           ▼ (间址完成后)
    │                           │
    └── 间接寻址标志 = 0 → 直接转执行微程序入口
```

这个判断由微指令的**判别测试字段**完成。

### 完整的微程序流程（含间址）

```
取指微程序 (000H~002H)
        │
        ▼
   ┌─────────┐
   │间址标志? │
   └──┬───┬──┘
      │   │
     =1  =0
      │   │
      ▼   ▼
   间址   执行
   微程序  微程序
   (003H~ (010H~
    005H)  ...)
      │
      ▼
   执行微程序
```

## 直观理解

### 生活类比

间址就像**查地址簿**：
- 指令中给出的是"地址簿的页号"（不是直接地址）
- 需要先翻到那一页，找到真正的地址
- 再用真正的地址去访问目标数据

间址微程序就是"翻地址簿查地址"这个过程的标准化操作流程。

### 记忆技巧

- **间址三步**：「送址 → 读址 → 回填」（Ad(IR)→MAR → M(MAR)→MDR → MDR→Ad(IR)）
- **与取指的区别**：取指取出的是指令（送IR），间址取出的是地址（送回IR的地址段）
- **间址是可选的**：不是所有指令都需要间址周期

### 易错陷阱

1. **陷阱**：间址取出的内容写回IR的地址段，不是写回整个IR。这与取指不同（取指是整个MDR写入IR）。
2. **陷阱**：间址过程中PC不变。只有取指时PC才+1。
3. **陷阱**：间址标志通常在指令的寻址方式字段中，由操作码译码后产生。
