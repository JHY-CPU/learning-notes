# 58_微程序_408考点

## 核心概念

- **408高频考点**：微程序设计分析、微地址计算、微指令编码、微程序流程。
- **题型**：综合题（给定指令和数据通路，设计微程序并写出微指令编码）。
- **关键能力**：能将指令的微操作序列转化为微程序，并用特定编码方式写出微指令。

## 原理分析

### 考点一：微程序流程分析

**典型题**：某计算机有 ADD、LOAD、JMP 三条指令，CM容量为 128×32 位。取指微程序在 00H~02H，间址微程序在 03H~05H。ADD 入口 10H，LOAD 入口 20H，JMP 入口 30H。

**问题**：画出 ADD 指令（不涉及间址）的微程序流程，标出每条微指令的下址。

**答案**：

| 微指令地址 | 微操作 | 下址 | 说明 |
|-----------|--------|------|------|
| 00H | $PC \to MAR$ | 01H | 取指，顺序 |
| 01H | $M(MAR) \to MDR, PC+1$ | 02H | 取指，顺序 |
| 02H | $MDR \to IR$ | 判别 | 测试间址标志 |
| (判别) | - | 10H（ADD）或03H（间址） | 分支 |
| 10H | $R_d \to Y$ | 11H | ADD执行，顺序 |
| 11H | $(Y) + R_s \to Z$ | 12H | ADD执行，顺序 |
| 12H | $Z \to R_d$ | 00H | ADD执行，回取指 |

### 考点二：微地址计算

**典型题**：某微程序控制器采用断定法，下址字段7位，判别测试字段2位。当前微指令的下址字段为 0110100，判别测试字段为 01（测试间址标志），间址标志 IND=1。求后继微地址。

**分析**：
- 判别测试=01 表示测试间址标志
- IND=1 需要分支到间址微程序
- 分支方式：下址字段的某些位由条件值替换

假设分支方式为：下址字段的最低位由 IND 决定
- 下址字段：011010**0**
- IND=1，替换最低位：011010**1**
- 后继微地址 = 0110101 = 53H

### 考点三：微指令编码计算

**典型题**：某系统有 24 个微命令，分为 3 组（每组8个互斥），CM有64条微指令，判别测试2位。用字段直接编码，求微指令字长。

**分析**：
- 控制字段：每组 $\lceil\log_2(8+1)\rceil = 4$ 位，3组共 12 位
- 判别测试字段：2 位
- 下址字段：$\lceil\log_2 64\rceil = 6$ 位
- 微指令字长：$12 + 2 + 6 = 20$ 位

### 考点四：微程序控制器 vs 硬布线控制器

**典型题**：以下关于微程序控制器的描述，正确的是？
- (A) 微程序存放在主存中 → 错误，存放在CM中
- (B) 每条机器指令对应一段微程序 → 正确
- (C) 微指令和机器指令一一对应 → 错误，一条机器指令对应多条微指令
- (D) CM用DRAM实现 → 错误，通常用ROM

## 直观理解

### 解题方法总结

1. **先画流程图**：取指→（间址）→执行→（中断）→取指
2. **再标地址**：给每条微指令分配CM地址
3. **最后编码**：用指定的编码方式写出二进制编码

### 关键步骤

1. 确定指令的微操作序列（取自数据通路分析）
2. 分配CM地址空间
3. 确定每条微指令的下址字段和判别测试字段
4. 用字段直接/间接编码写出控制字段

## 知识关联

### 跨章节联系

- **第5章（微操作分析）**：微操作序列是微程序设计的输入
- **第5章（微指令编码）**：编码方式决定微指令的二进制表示
- **第5章（后继地址）**：下址字段和判别测试决定后继微地址

### 408真题特点

1. 综合题通常 10~15 分
2. 要求画微程序流程图
3. 要求写出关键微指令的编码
4. 可能涉及微地址计算
5. 注意区分单总线和专用通路结构

### 易错陷阱

1. **陷阱**：取指微程序结束后，不是直接跳到执行微程序入口，而是先判断间址标志。
2. **陷阱**：执行微程序结束后，不是直接回取指，而是先检查中断标志。
3. **陷阱**：字段直接编码中，"无操作"需要占用一个编码，不能忘记。
