# 21_多指令周期CPU

## 核心概念

- **多指令周期CPU（Multi-Cycle CPU）**：不同类型的指令使用不同数量的时钟周期，每条指令经历多个机器周期。
- **时钟周期固定**：等于最慢的机器周期（通常是一次主存访问时间）。
- **指令周期可变**：简单指令用较少的周期，复杂指令用较多的周期。
- **控制方式**：需要状态机（有限状态自动机）控制各机器周期之间的转换。

### 多指令周期 vs 单指令周期

| 对比项 | 单指令周期 | 多指令周期 |
|--------|-----------|-----------|
| 时钟周期 | 取最长指令时间 | 取最慢机器周期时间 |
| 指令周期长度 | 所有指令相同 | 不同指令不同 |
| CPU利用率 | 低（浪费多） | 高（按需分配） |
| 控制复杂度 | 简单 | 较复杂（需状态机） |
| 时钟频率 | 低（周期长） | 高（周期短） |

## 原理分析

### 多指令周期CPU的时钟周期

$$T_{时钟} = \max(T_{取指}, T_{间址}, T_{执行各步}, T_{中断})$$

通常 $T_{时钟}$ = 一次主存访问时间。

### 各指令的时钟周期数

| 指令 | 取指 | 间址 | 执行 | 中断 | 总计 |
|------|------|------|------|------|------|
| ADD R1,R2 | 3 | 0 | 2 | 0 | 5 |
| LOAD [addr] | 3 | 0 | 3 | 0 | 6 |
| LOAD [[addr]] | 3 | 3 | 3 | 0 | 9 |
| STORE [addr] | 3 | 0 | 3 | 0 | 6 |
| JMP addr | 3 | 0 | 1 | 0 | 4 |
| 有中断的ADD | 3 | 0 | 2 | 3 | 8 |

### 状态机控制

多指令周期CPU使用状态触发器控制当前处于哪个机器周期：

- **FE = 1**：取指周期
- **IND = 1**：间址周期
- **EX = 1**：执行周期
- **INT = 1**：中断周期

状态转移逻辑：
```
FE=1 → (间址标志?) → IND=1 → EX=1
                   ↘ EX=1 → (中断请求?) → INT=1 → FE=1
                                       ↘ FE=1
```

### 性能分析

执行 $n$ 条指令，第 $i$ 条指令需要 $c_i$ 个时钟周期：

$$T_{总} = \left(\sum_{i=1}^{n} c_i\right) \times T_{时钟}$$

$$\overline{CPI} = \frac{1}{n} \sum_{i=1}^{n} c_i$$

**示例**：1000条指令中，ADD 800条（CPI=5），LOAD 200条（CPI=6）。

$$\overline{CPI} = \frac{800 \times 5 + 200 \times 6}{1000} = \frac{4000 + 1200}{1000} = 5.2$$

若 $T_{时钟} = 2$ ns（假设单指令周期的时钟周期为12ns，多指令周期为2ns）：

$$T_{多指令} = 1000 \times 5.2 \times 2 = 10400 \text{ ns}$$

对比单指令周期：$T_{单指令} = 1000 \times 12 = 12000$ ns

多指令周期更快（节省了1600ns）。

## 直观理解

### 生活类比

多指令周期CPU就像**自助餐厅的计时收费**：
- 简单的汤品（简单指令）：拿一碗就走，2分钟
- 复杂的菜品（复杂指令）：需要排队、挑选、加热，8分钟
- 每个人按实际花费时间计费（按需分配时间）
- 管理比统一时间复杂（需要记录每个人进来和离开的时间）

### 记忆技巧

- **多指令周期 = "按需分配"**：简单指令快，复杂指令慢
- **时钟周期固定，指令周期可变**
- **需要状态机**：控制在四个周期（F/I/E/INT）之间切换

## 知识关联

### 跨章节联系

- 与**单指令周期**对比：多指令周期更高效
- 与**流水线**对比：多指令周期不重叠执行，流水线重叠执行
- **状态机模型**是指令周期四种周期对比（18节）的直接应用

### 常考组合

- 给定指令混合比例和各指令CPI，计算平均CPI
- 对比单指令周期和多指令周期的执行时间
- 状态转移图的设计

### 易错陷阱

1. **陷阱**：多指令周期CPU的时钟周期是固定的（取最慢机器周期），而不是每条指令自适应。
2. **陷阱**：多指令周期不等于流水线。多指令周期是串行执行（一条执行完再取下一条），流水线是重叠执行。
3. **陷阱**：CPI在多指令周期中不等于1（不同指令CPI不同），在单指令周期中CPI是常数。
