# 46_垂直型微指令

## 核心概念

- **垂直型微指令（Vertical Micro-instruction）**：类似机器指令的编码方式，包含微操作码字段和微地址字段，一条微指令通常只指定一个微操作。
- **特点**：微指令字长短，并行度低，微程序长。
- **本质**：把微指令当作一种"微机器"的指令，这种微机器的指令集很小且规整。

### 垂直型微指令的格式

```
┌──────────────────────────────────────────┐
│  微操作码  │  目的地  │  源1  │  源2  │ ... │
│   μOP      │   Dest   │  Src1 │  Src2 │     │
│  (4~6位)   │ (3~4位)  │(3~4位)│(3~4位)│     │
└──────────────────────────────────────────┘
```

### 典型微指令格式（垂直型）

| 字段 | 位数 | 说明 |
|------|------|------|
| 微操作码（μOP） | 5位 | 指定微操作类型（最多32种） |
| 目的地址（Dest） | 4位 | 目的寄存器/部件编号 |
| 源地址1（Src1） | 4位 | 第一源操作数 |
| 源地址2（Src2） | 4位 | 第二源操作数 |
| 其他 | 3位 | 条件控制等 |
| **总计** | **20位** | 远小于水平型 |

### 微操作码的编码示例

| μOP编码 | 微操作 | 含义 |
|---------|--------|------|
| 00000 | NOP | 空操作 |
| 00001 | MOV | 寄存器传送 |
| 00010 | ADD | 加法 |
| 00011 | SUB | 减法 |
| 00100 | READ | 读存储器 |
| 00101 | WRITE | 写存储器 |
| 00110 | INC | 自增 |
| 00111 | JMP | 转移（跳转到指定微地址） |
| ... | ... | ... |

## 原理分析

### 垂直型微指令的执行

以 $R_0 \to MAR$ 为例：

**水平型**（直接编码）：
```
PCout=0, MDRout=0, R0out=1, MARin=1, MDRin=0, IRin=0, ...
```
一条微指令，控制字段 30+ 位。

**垂直型**：
```
μOP=MOV, Dest=MAR, Src=R0
```
一条微指令，20 位。

### 垂直型的微程序长度

由于每条微指令只完成一个微操作，完成同样的功能需要更多条微指令：

**ADD $R_0, R_1$ 的微程序**：

| 地址 | 垂直型微指令 | 说明 |
|------|-------------|------|
| 000 | MOV Y, R0 | $R_0 \to Y$ |
| 001 | ADD Z, Y, R1 | $(Y) + R_1 \to Z$ |
| 010 | MOV R0, Z | $Z \to R_0$ |

对比水平型：同样的功能可能只需 1~2 条微指令（可以并行发出多个微命令）。

### 垂直型的编码效率

垂直型微指令编码效率高，因为：
1. 微操作码字段用 $\lceil \log_2 M \rceil$ 位编码 M 种微操作
2. 地址字段用 $\lceil \log_2 R \rceil$ 位编码 R 个寄存器
3. 没有大量闲置的 0 位

$$\text{编码效率} = \frac{\text{有效信息位}}{\text{微指令总位数}}$$

垂直型效率可达 80%~90%，水平型可能只有 20%~40%。

### 垂直型的优缺点

**优点**：
1. 微指令短，CM字长短
2. 编码效率高
3. 设计简单，类似编写汇编程序
4. 易于理解和调试

**缺点**：
1. 并行度低，一条微指令只能完成一个微操作
2. 微程序长，CM中需要更多微指令
3. 执行速度慢，需要更多CM访问次数
4. 微操作码译码需要额外时间

## 直观理解

### 生活类比

垂直型微指令就像**使用简单的按钮遥控器**：
- 每次只按一个按钮（一个微操作）
- 按钮上标有明确的功能（ADD、MOV、READ）
- 想做多件事就需要多按几次（多条微指令）
- 遥控器很小（微指令短）

而水平型就像**控制台上的拨码开关**：
- 可以同时拨动多个开关（并行微命令）
- 开关很多，控制台很大（微指令长）
- 但一次拨动就能完成复杂操作

### 记忆技巧

- **垂直 = "窄高"**：微指令窄（位数少），但"高"（条数多）
- **垂直 = 串行**：一条微指令做一件事
- **垂直 = 类机器指令**：有操作码、有地址字段

### 易错陷阱

1. **陷阱**：垂直型虽然微指令短，但微程序更长，CM总容量未必比水平型小。
2. **陷阱**：垂直型微指令需要译码才能产生微命令，增加了信号传播延迟。
3. **陷阱**：现代处理器通常采用混合型或水平型，因为性能是首要考虑。垂直型主要用于教学和简单系统。
