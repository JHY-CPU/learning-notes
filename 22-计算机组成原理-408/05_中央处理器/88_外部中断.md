# 88_外部中断

## 核心概念

- **外部中断**：由 CPU **外部**设备通过中断请求线发出的中断信号
- 分为**可屏蔽中断（INTR）**和**不可屏蔽中断（NMI）**
- 外部中断是 CPU 与外部设备**交互的核心机制**

## 原理分析

### 一、可屏蔽中断（INTR）

#### 基本特征
- 通过**中断标志位 IF** 控制是否响应
- $IF = 1$：允许中断（开中断，STI 指令设置）
- $IF = 0$：禁止中断（关中断，CLI 指令设置）

#### 常见来源
| 设备 | 中断原因 |
|------|---------|
| 键盘 | 按键输入 |
| 鼠标 | 移动/点击 |
| 硬盘 | 数据传输完成 |
| 网卡 | 数据包到达 |
| 定时器 | 时间片到 |
| 打印机 | 打印完成 |

#### 响应条件
$$\text{响应 INTR} = (IF = 1) \land (\text{无更高优先级中断})$$

### 二、不可屏蔽中断（NMI）

#### 基本特征
- **不能被 IF 位屏蔽**
- 上升沿触发（从低到高的跳变）
- 优先级高于 INTR

#### 常见来源
| 事件 | 说明 |
|------|------|
| 电源掉电预警 | 系统即将断电，需紧急保存 |
| 奇偶校验错误 | 内存数据出错 |
| 看门狗超时 | 系统死机检测 |

### 三、中断请求的传递

```
设备1 ──→ [中断控制器 8259A] ──→ INTR ──→ CPU
设备2 ──→                        ↑
设备3 ──→                        │
  ...                            │
NMI 源 ────────────────────────→ NMI ──→ CPU
```

#### 中断控制器（8259A）

- 接收多个设备的中断请求
- 进行**优先级排队**
- 向 CPU 发送单一的中断请求信号
- 提供**中断类型号**

### 四、中断响应时序

```
┌──────────────────────────────────────────┐
│  1. 外设发出中断请求（INTR 或 NMI）       │
│  2. CPU 在每条指令结束时检查中断请求       │
│  3. 满足响应条件则响应                    │
│  4. CPU 发送 INTA（中断应答）信号          │
│  5. 外设（8259A）送上中断类型号            │
│  6. CPU 进入中断响应周期                  │
└──────────────────────────────────────────┘```

### 五、中断响应周期（第 4 个机器周期）

$$\overline{INTA} \rightarrow \text{读中断类型号} \rightarrow \text{保存 PC/PSW} \rightarrow \text{转向服务程序}$$

## 直观理解

**外部中断**像**门铃**：

- **INTR**：普通的门铃，你可以选择不理（IF=0 关门铃）
- **NMI**：紧急火警铃，不管你在做什么都必须响应
- **中断控制器**：大楼前台，管理所有门铃的优先级

## 知识关联

- 与**异常**（第 87 节）对比：外部 vs 内部，异步 vs 同步
- 与**中断响应过程**（第 90 节）直接相关
- 与**中断优先级**（第 93 节）的关系：NMI > INTR
- 与**IO 接口**（第 7 章）的关系：中断是 IO 的主要控制方式之一
- 与**操作系统**的关系：时钟中断是进程调度的基础
