# 42_微指令与微程序

## 核心概念

- **微指令（Micro-instruction）**：控制存储器中一个存储单元的内容，一条微指令对应一个节拍（时钟周期）内所有需要同时发出的微命令的集合。
- **微程序（Micro-program）**：实现一条机器指令所需的所有微指令按顺序组成的序列。
- **对应关系**：一条机器指令 → 一段微程序 → 多条微指令 → 多个微命令 → 多个微操作。

### 层次关系

```
机器指令（1条）
  └── 微程序（1段）
        └── 微指令（多条）
              └── 微命令（多个）
                    └── 微操作（多个）
```

### 举例：ADD $R_0, R_1$ 指令的微程序

| 微指令地址 | 微指令（微命令集合） | 对应微操作 | 说明 |
|-----------|---------------------|-----------|------|
| 0000 | $PC_{\text{out}}, MAR_{\text{in}}$ | $PC \to MAR$ | 取指第1步 |
| 0001 | Read, $MDR_{\text{in}}, PC+1$ | $M(MAR) \to MDR, PC+1$ | 取指第2步 |
| 0010 | $MDR_{\text{out}}, IR_{\text{in}}$ | $MDR \to IR$ | 取指第3步 |
| 0100 | $R_0^{\text{out}}, Y_{\text{in}}$ | $R_0 \to Y$ | 执行第1步 |
| 0101 | $R_1^{\text{out}}, ALU_{\text{add}}, Z_{\text{in}}$ | $(Y)+R_1 \to Z$ | 执行第2步 |
| 0110 | $Z_{\text{out}}, R_0^{\text{in}}$ | $Z \to R_0$ | 执行第3步 |

0000~0010 是公共取指微程序（所有指令共享）。
0100~0110 是 ADD 指令的执行微程序。

## 原理分析

### 微程序与机器指令的关系

```
                机器指令集
                    │
    ┌───────┬───────┼───────┬───────┐
    │       │       │       │       │
  ADD     SUB    LOAD   STORE    JMP
    │       │       │       │       │
  微程序A 微程序B 微程序C 微程序D 微程序E
    │       │       │       │       │
  μI1     μI1     μI1     μI1     μI1
  μI2     μI2     μI2     μI2     μI2
  μI3     μI3     μI3     μI3     μI3
  ...     ...     ...     ...     ...
```

- 指令越多，微程序数量越多
- 指令越复杂，每段微程序包含的微指令条数越多

### 微程序的存储

微程序存放在**控制存储器（CM）**中：
- CM按地址组织，每个地址存放一条微指令
- 微指令地址（μPC）指向当前正在执行的微指令
- 执行完一条微指令后，μPC更新为下一条微指令的地址

### 微指令的执行过程

1. **取微指令**：根据μPC从CM中读出微指令，送入μIR
2. **译码**：对μIR中的控制字段译码，产生微命令
3. **执行**：微命令驱动数据通路完成微操作
4. **形成后继地址**：根据μIR的下址字段或操作码，确定下一条微指令地址
5. **更新μPC**：将后继地址送入μPC

### 微程序控制器的时序

```
┌──────────┐    ┌──────────┐    ┌──────────┐
│ 取微指令  │──→│ 译码执行  │──→│ 形成后继 │──→ (下一条)
│ CM→μIR   │    │ 产生微命令│    │ 更新μPC  │
└──────────┘    └──────────┘    └──────────┘
```

每个微指令周期 = 一个时钟周期（节拍）。

## 直观理解

### 生活类比

- **微程序** = 一份完整的操作手册
- **微指令** = 手册中的一个操作步骤
- **微命令** = 步骤中的具体动作指示

比如做一道宫保鸡丁：
- 微程序 = 整个菜谱（从备料到出锅）
- 微指令 = 每一步（"第3步：大火加热油锅"）
- 微命令 = 具体动作（"开大火"、"倒油"）

### 记忆技巧

- **"微"的层次**：微程序 > 微指令 > 微命令 > 微操作
- **一条微指令 = 一个节拍内并行的所有微命令**
- **一段微程序 = 一条机器指令的完整控制序列**

### 易错陷阱

1. **陷阱**：一条微指令对应的是一个节拍（时钟周期）内的控制信号，不是一条机器指令。
2. **陷阱**：公共取指微程序是所有指令共享的，不能在每条指令的微程序中重复编写。
3. **陷阱**：微指令的执行不需要再"取指"（从主存取），而是从CM中读取，速度比主存快得多。
