# 56_微程序后继地址

## 核心概念

- **后继微地址**：当前微指令执行完毕后，下一条微指令在CM中的地址。
- **后继地址来源**：顺序执行、条件分支、微程序转移、微程序入口。
- **微地址形成部件**：根据当前微指令的下址字段、判别测试字段和操作码，确定下一条微指令的地址。

### 后继微地址的四种来源

| 来源 | 触发条件 | 说明 |
|------|---------|------|
| **顺序执行** | 无条件 | μPC + 1 → μPC |
| **条件分支** | 判别测试字段 ≠ 0 + 条件满足 | 根据条件选择不同后继地址 |
| **微程序转移** | 微指令中指定了转移地址 | 下址字段 → μPC |
| **微程序入口** | 取指结束 | 操作码 → 微地址形成部件 → μPC |

## 原理分析

### 后继地址形成逻辑

```
                    ┌─────────────────────┐
                    │  微地址形成部件      │
                    │                     │
  μPC + 1 ────────→ │                     │
  (顺序)            │                     │
                    │                     ├──→ μPC（下条微指令地址）
  下址字段 ────────→ │    选择逻辑         │
  (转移)            │                     │
                    │                     │
  操作码 ─────────→ │                     │
  (入口)            │                     │
                    │                     │
  条件标志 ────────→ │                     │
  (分支)            └─────────────────────┘
```

### 顺序执行

最简单的情况，当前微指令执行完后，μPC直接+1：

$$\mu PC \leftarrow \mu PC + 1$$

适用于微程序中连续执行的微指令。

### 条件分支（判别测试）

微指令的判别测试字段指明需要测试的条件：

| 判别测试字段 | 测试内容 | 分支含义 |
|-------------|---------|---------|
| 00 | 无测试 | 顺序执行 |
| 01 | 测试间址标志 $IND$ | $IND=1$ 转间址，$IND=0$ 转执行 |
| 10 | 测试中断标志 $INT$ | $INT=1$ 转中断，$INT=0$ 回取指 |
| 11 | 测试条件标志 $ZF$ | 用于条件转移指令 |

分支时，后继地址 = 下址字段的某几位 + 条件值拼接。

### 微程序转移

无条件转移到指定微地址：

$$\mu PC \leftarrow \text{下址字段}$$

用于微程序之间的跳转（如间址完成后跳转到执行微程序入口）。

### 微程序入口形成

取指微程序结束时，根据操作码形成执行微程序入口：

$$\mu PC \leftarrow f(\text{操作码})$$

常见实现方式：
1. **直接拼接**：操作码直接作为μPC的高位
2. **查表法**：用PROM将操作码映射为入口地址
3. **加法**：入口基址 + 操作码 × 步长

## 直观理解

### 生活类比

后继地址就像**导航系统的下一个目的地**：
- **顺序执行** = 沿当前路线继续开（μPC+1）
- **条件分支** = 到了岔路口，根据红绿灯选择左转或右转
- **转移** = GPS说"前方调头，去另一个目的地"
- **入口形成** = 根据顾客订单（操作码），去对应的取货点

### 记忆技巧

- **四种来源**：「顺分转入」（顺序、分支、转移、入口）
- **μPC是核心**：所有来源最终都是更新μPC的值
- **判别测试是关键**：有判别测试才需要分支

### 408考点

1. **给定微指令的判别测试字段和条件，确定后继地址**（综合题）
2. **分析取指结束后的微地址形成**
3. **计算条件分支时的后继微地址**

### 易错陷阱

1. **陷阱**：后继地址的四种来源不是互斥的。一条微指令可能同时有"顺序执行"和"条件分支"两种可能。
2. **陷阱**：判别测试字段=00时不是"不执行"，而是"无条件顺序执行"。
3. **陷阱**：入口形成时，操作码到微地址的映射不一定简单拼接，可能需要经过转换逻辑。
