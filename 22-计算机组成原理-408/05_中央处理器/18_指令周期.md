# 12_指令周期

## 核心概念

- **指令周期（Instruction Cycle）**：CPU从主存取出一条指令并执行该指令所需的全部时间。
- **指令周期 = 取指周期 + 执行周期**（基本模型），还可能包含间址周期和中断周期。
- **机器周期（Machine Cycle）**：指令周期中的一个阶段，通常对应一次主存访问或一次ALU运算。
- **时钟周期（Clock Cycle / 节拍）**：CPU的基本定时单位，一个机器周期包含若干个时钟周期。
- **三者的层次关系**：指令周期 > 机器周期 > 时钟周期。

### 三级时序关系

```
指令周期
├── 机器周期1（取指）──┬── T0（时钟周期/节拍）
│                      ├── T1
│                      └── T2
├── 机器周期2（执行）──┬── T0
│                      ├── T1
│                      └── T3
├── 机器周期3（间址）──┬── T0（可选）
│                      └── T1
└── 机器周期4（中断）──┬── T0（可选）
                       └── T1
```

## 原理分析

### 指令周期的完整组成

一条指令的完整执行可能经历四个机器周期：

$$\text{指令周期} = \text{取指周期} + (\text{间址周期}) + \text{执行周期} + (\text{中断周期})$$

其中间址周期和中断周期不是每条指令都有的：
- **间址周期**：仅当指令使用间接寻址时才有
- **中断周期**：仅当有中断请求且允许中断时才有

### 各机器周期的典型时钟周期数

| 机器周期 | 典型时钟周期数 | 主要操作 |
|----------|---------------|----------|
| 取指周期 | 3个节拍 | PC→MAR, 读主存, MDR→IR, PC+1 |
| 间址周期 | 3个节拍 | Ad(IR)→MAR, 读主存, MDR→Ad(IR) |
| 执行周期 | 不定（1~多个） | 取操作数、运算、写回 |
| 中断周期 | 3~4个节拍 | 关中断、保存PC、转向中断服务程序 |

### 机器周期的确定原则

**以访存操作为基准**：在以主存为核心的系统中，一次主存读写的时间通常是最长的基本操作，因此一个机器周期的时间通常等于一次主存访问时间。

$$\text{机器周期时间} \geq \text{主存访问时间}$$

### 指令周期与指令类型的关系

不同指令的指令周期长度不同：

| 指令类型 | 指令周期组成 | 总时钟周期数（示例） |
|----------|-------------|---------------------|
| 寄存器运算（ADD R1,R2） | 取指 + 执行 | 3 + 2 = 5 |
| 直接寻址取数（LOAD [addr]） | 取指 + 执行（含访存） | 3 + 3 = 6 |
| 间接寻址取数（LOAD [[addr]]） | 取指 + 间址 + 执行 | 3 + 3 + 3 = 9 |
| 转移指令（JMP addr） | 取指 + 执行 | 3 + 1 = 4 |
| 有中断的任意指令 | 取指 + 执行 + 中断 | 3 + N + 3 |

### 时序系统的实现

```
时钟源（晶振）
    │
    ▼
┌──────────┐
│ 节拍发生器 │──→ T0, T1, T2, T3...（电位信号）
└──────────┘
    │
    ▼
┌──────────┐
│ 机器周期  │──→ M1(取指), M2(间址), M3(执行), M4(中断)
│ 标志触发器 │
└──────────┘
    │
    ▼
┌──────────┐
│ 工作触发器 │──→ FE(取指), IND(间址), EX(执行), INT(中断)
└──────────┘
```

## 直观理解

### 生活类比

- **指令周期**= 完成一个工作任务的完整流程（从领取任务到完成）
- **机器周期**= 工作流程中的一个大步骤（如"准备材料"、"加工"、"交付"）
- **时钟周期**= 每个大步骤中的小动作（如"拿起工具"、"放下工件"）
- **类比**：做一道菜 = 取菜谱（取指）→ 买菜（间址，可能不需要）→ 做菜（执行）→ 清理灶台（中断处理）

### 记忆技巧

- **层次口诀**：「指机时」（指令周期→机器周期→时钟周期，从大到小）
- **四大周期**：「取间执中」（取指、间址、执行、中断）
- **核心关系**：$T_{指令周期} \geq T_{机器周期} \geq T_{时钟周期}$

## 知识关联

### 跨章节联系

- **第6章（总线）**：取指和间址涉及总线传输（地址总线、数据总线）
- **第3章（存储）**：访存时间决定机器周期的最小长度

### 常考组合

- 给定指令类型和寻址方式，计算指令周期中包含的机器周期数
- 给定时钟频率和各机器周期的时钟周期数，计算指令执行时间（CPI）
- 中断响应时机：指令执行完后、下一条指令取指前

### 易错陷阱

1. **陷阱**：不是所有指令都有间址周期。只有间接寻址的指令才有。
2. **陷阱**：中断周期也不是每条指令都有，取决于是否有中断请求。
3. **陷阱**：转移指令的执行周期可能很短（只需修改PC），但取指周期与其他指令相同。
