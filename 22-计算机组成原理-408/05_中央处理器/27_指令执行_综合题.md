# 27_指令执行_综合题

## 核心概念

本文件提供408考研中指令执行过程的综合大题训练，涵盖完整的微操作序列分析。

### 综合题解题框架

1. **识别指令类型和寻址方式**
2. **确定经历的周期**：取指（必有）→ 间址（若有）→ 执行（必有）→ 中断（若有）
3. **分周期写出微操作序列**
4. **统计总时钟周期数**
5. **画出数据流图**（若要求）

## 综合题1：直接寻址加法

**题目**：某机单总线CPU结构，有ALU、Y锁存器、通用寄存器R0~R3、PC、IR、MAR、MDR。时钟频率1GHz。写出指令"ADD R1, [1000]"（直接寻址，M(1000)+R1→R1）的完整微操作序列，计算执行时间。

**解答**：

### 取指阶段（3个周期）

| 节拍 | 微操作 | 说明 |
|------|--------|------|
| T0 | PC → MAR | 指令地址送MAR |
| T1 | 1→R, M(MAR)→MDR | 读主存取指令 |
| T2 | MDR → IR, PC+1 | 指令送IR，PC递增 |

### 执行阶段（4个周期）

| 节拍 | 微操作 | 说明 |
|------|--------|------|
| T3 | Ad(IR) → MAR | 地址1000送MAR |
| T4 | 1→R, M(MAR)→MDR | 读M(1000)得操作数 |
| T5 | MDR → Y | 操作数送Y锁存器 |
| T6 | R1→ALU, Y+R1→R1 | ALU做加法，结果写回R1 |

### 总计

$$CPI = 3 + 4 = 7 \text{ 个时钟周期}$$

$$T = 7 \times \frac{1}{1 \times 10^9} = 7 \text{ ns}$$

## 综合题2：间接寻址存数

**题目**：写出指令"STORE R2, [[500]]"（二次间址存数，将R2的内容存入M(M(M(500)))指向的地址）的微操作序列。

**解答**：

### 取指阶段（3个周期）

| 节拍 | 微操作 |
|------|--------|
| T0 | PC → MAR |
| T1 | 1→R, M(MAR)→MDR |
| T2 | MDR → IR, PC+1 |

### 一次间址（3个周期）

| 节拍 | 微操作 | 说明 |
|------|--------|------|
| T3 | Ad(IR) → MAR | 形式地址500送MAR |
| T4 | 1→R, M(MAR)→MDR | 读M(500)得中间地址 |
| T5 | MDR → Ad(IR) | 中间地址写回IR的地址字段 |

### 二次间址（3个周期）

| 节拍 | 微操作 | 说明 |
|------|--------|------|
| T6 | Ad(IR) → MAR | 中间地址送MAR |
| T7 | 1→R, M(MAR)→MDR | 读出有效地址 |
| T8 | MDR → Ad(IR) | 有效地址写回IR |

### 执行阶段（3个周期）

| 节拍 | 微操作 | 说明 |
|------|--------|------|
| T9 | Ad(IR) → MAR | 有效地址送MAR |
| T10 | R2 → MDR | 待存数据送MDR |
| T11 | 1→W, MDR→M(MAR) | 写入主存 |

### 总计

$$CPI = 3 + 3 + 3 + 3 = 12 \text{ 个时钟周期}$$

## 综合题3：子程序调用

**题目**：写出"CALL 2000"（调用地址2000处的子程序）的微操作序列，假设返回地址为CALL指令的下一条指令地址。

**解答**：

### 取指阶段（3个周期）

| 节拍 | 微操作 |
|------|--------|
| T0 | PC → MAR |
| T1 | 1→R, M(MAR)→MDR |
| T2 | MDR → IR, **PC+1** |

注意：取指后PC已递增到下一条指令地址（即返回地址）。

### 执行阶段（4个周期）

| 节拍 | 微操作 | 说明 |
|------|--------|------|
| T3 | SP - 1 → SP | 栈指针下移 |
| T4 | SP → MAR | 栈顶地址送MAR |
| T5 | PC → MDR | 返回地址（即PC+1后的值）送MDR |
| T6 | 1→W, MDR→M(MAR), Ad(IR)→PC | 返回地址压栈，同时目标地址送PC |

**关键**：T6中两个操作可以并行吗？
- MDR→M(MAR) 用到数据总线
- Ad(IR)→PC 是CPU内部操作（IR→PC不经过总线）
- 在单总线结构中，T6的写主存和PC修改可能需要分开，变为2个周期。

**修正执行阶段**（5个周期）：

| 节拍 | 微操作 |
|------|--------|
| T3 | SP - 1 → SP |
| T4 | SP → MAR |
| T5 | PC → MDR |
| T6 | 1→W, MDR→M(MAR) |
| T7 | Ad(IR) → PC |

$$CPI = 3 + 5 = 8$$

## 综合题4：带中断的指令

**题目**：某指令"ADD R1, R2"（寄存器直接寻址）在执行完毕后检测到中断请求。写出完整的过程（含中断周期），假设关中断、保存PC、转向中断服务程序共需3个时钟周期。

**解答**：

### 取指（3个周期）

| 节拍 | 微操作 |
|------|--------|
| T0 | PC → MAR |
| T1 | 1→R, M(MAR)→MDR |
| T2 | MDR → IR, PC+1 |

### 执行（2个周期）

| 节拍 | 微操作 |
|------|--------|
| T3 | R1 → Y |
| T4 | R2→ALU, Y+R2→R1 |

### 中断周期（3个周期）

| 节拍 | 微操作 | 说明 |
|------|--------|------|
| T5 | 0→EINT | 关中断 |
| T6 | SP-1→SP→MAR, PC→MDR | 保存断点准备 |
| T7 | 1→W, MDR→M(MAR), 向量地址→PC | 断点入栈，转向中断服务程序 |

$$CPI = 3 + 2 + 3 = 8$$

---

## 直观理解

### 大题得分要点

1. **分阶段写**：取指、间址、执行、中断必须分开写
2. **每个节拍一个微操作**（单总线）：同一节拍内不能有两个总线操作
3. **控制信号可选**：题目要求写就写，没要求可以不写
4. **PC+1别忘**：取指阶段最后一步
5. **访存标记**：每次访存要写 Read 或 Write

### 易错汇总

| 错误 | 正确 |
|------|------|
| CALL只写跳转不写压栈 | CALL必须先保存返回地址 |
| 间址忘了写回IR(Ad) | 间址后有效地址要写回IR |
| 中断忘了关中断 | 中断第一步是0→EINT |
| 双总线按单总线写 | 注意并行操作可合并节拍 |
