# 16_执行周期

## 核心概念

- **执行周期（Execute Cycle）**：指令周期中真正"做事"的阶段，根据指令类型完成相应的操作。
- **执行周期的长度不固定**：不同指令的执行周期差异很大，取决于指令类型和寻址方式。
- **执行阶段的核心操作**：取操作数（若有）→ 运算/操作 → 写回结果（若有）。

### 典型指令的执行周期

| 指令类型 | 执行操作 | 访存次数 |
|----------|----------|----------|
| 寄存器运算（ADD R1,R2） | R1+R2→R1 | 0 |
| 取数（LOAD [addr]） | M(addr)→R | 1（读） |
| 存数（STORE R1,[addr]） | R1→M(addr) | 1（写） |
| 无条件跳转（JMP addr） | addr→PC | 0 |
| 条件跳转（JZ addr） | 若ZF=1则addr→PC | 0 |
| 调用（CALL addr） | PC入栈，addr→PC | 1（写栈） |
| 返回（RET） | 栈→PC | 1（读栈） |

## 原理分析

### 各类指令的执行微操作

#### 1. 寄存器-寄存器型加法：ADD R1, R2

```
执行阶段微操作：
T0: R1 → Y           （R1的内容送ALU输入锁存器Y）
T1: R2 → ALU, Y+R2   （R2送ALU另一输入，ALU做加法）
T2: ALU → R1         （结果写回R1）
```

#### 2. 取数指令：LOAD R1, [addr]（直接寻址）

```
执行阶段微操作：
T0: Ad(IR) → MAR      （IR中的地址码送MAR）
T1: 1 → R             （发出读命令）
T2: M(MAR) → MDR      （主存数据读入MDR）
T3: MDR → R1          （数据送R1）
```

#### 3. 存数指令：STORE R1, [addr]（直接寻址）

```
执行阶段微操作：
T0: Ad(IR) → MAR      （目标地址送MAR）
T1: R1 → MDR          （待存数据送MDR）
T2: 1 → W             （发出写命令）
T3: MDR → M(MAR)      （数据写入主存）
```

#### 4. 无条件跳转：JMP addr

```
执行阶段微操作：
T0: Ad(IR) → PC       （目标地址直接送PC）
```

仅需1个时钟周期（单总线结构下）。

#### 5. 条件跳转：JZ addr（零标志为1则跳转）

```
执行阶段微操作：
T0: if (ZF = 1) then Ad(IR) → PC
     else PC不变
```

#### 6. 加法（隐含寻址）：ADD [addr]（ACC + M(addr) → ACC）

```
执行阶段微操作：
T0: Ad(IR) → MAR
T1: 1 → R, M(MAR) → MDR
T2: ACC → Y
T3: MDR → ALU, Y + MDR
T4: ALU → ACC
```

### 执行阶段的控制逻辑

执行阶段的控制信号取决于：
1. **指令类型**（操作码）：决定执行什么操作
2. **寻址方式**：决定如何获取操作数
3. **当前节拍**：决定执行的哪一步

$$\text{控制信号} = f(\text{OP}, \text{寻址方式}, T_i, \text{状态标志})$$

### 执行阶段的访存分析

| 寻址方式 | 取操作数访存次数 | 存结果访存次数 | 总访存 |
|----------|-----------------|---------------|--------|
| 寄存器寻址 | 0 | 0 | 0 |
| 直接寻址 | 1 | 1 | 1 |
| 一次间址 | 1（间址周期已完成）+ 1（取数） | 1 | 2 |
| 寄存器间接寻址 | 1 | 1 | 1 |
| 立即寻址 | 0 | 0 | 0 |

## 直观理解

### 生活类比

执行周期就像**厨房做菜的"炒菜"环节**：
- 取指 = 从菜谱架上拿下菜谱
- 间址 = 根据菜谱上的提示找到真正的食材位置
- **执行 = 真正炒菜**（取食材→加工→装盘）
- 中断 = 炒完菜后收拾灶台

不同类型指令的执行就像不同的菜：
- ADD = 把两种食材混合搅拌（寄存器运算，不涉及冰箱）
- LOAD = 从冰箱拿食材（从主存取数据）
- STORE = 把做好的菜放回冰箱（数据写入主存）
- JMP = 换一本菜谱（改变执行流程）

### 记忆技巧

- **执行阶段的核心**：「**取→算→写**」（取操作数→运算/操作→写回结果）
- **关键区分**：执行阶段一定有"操作"（运算/传送/跳转），但不一定有"访存"

## 知识关联

### 跨章节联系

- **第4章（指令系统）**：各种寻址方式决定执行阶段如何获取操作数
- **第2章（运算）**：ALU的运算方法在执行阶段被使用
- **第3章（存储）**：执行阶段的访存操作依赖存储系统

### 常考组合

- 给定指令和寻址方式，写出执行阶段的微操作序列
- 计算执行阶段需要的时钟周期数
- 比较不同寻址方式下执行阶段的访存次数

### 易错陷阱

1. **陷阱**：间址周期已经将有效地址取到了IR(Ad)中，执行阶段不需要再做间址操作。
2. **陷阱**：跳转指令的执行阶段只需修改PC，不需要运算。
3. **陷阱**：STORE指令的执行阶段需要先将数据从寄存器送到MDR，再写入主存（两步）。
