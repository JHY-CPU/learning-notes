# 23_数据通路_单总线

## 核心概念

- **单总线数据通路**：CPU内部所有寄存器和ALU共享**一条内部总线**。
- **核心限制**：同一时刻总线上只有一个数据源，因此每一步数据传输需要一个独立的时钟周期。
- **优点**：结构最简单，硬件成本最低。
- **缺点**：性能最低，执行指令需要更多时钟周期。

### 单总线结构示意

```
┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐
│ PC  │ │ MAR │ │ MDR │ │ IR  │ │ R0  │ │ R1  │ │ ACC │
└──┬──┘ └──┬──┘ └──┬──┘ └──┬──┘ └──┬──┘ └──┬──┘ └──┬──┘
   │       │       │       │       │       │       │
═══╪═══════╪═══════╪═══════╪═══════╪═══════╪═══════╪══  内部总线
   │       │       │       │       │       │       │
   └───────┴───────┴───┬───┴───────┴───────┴───────┘
                       │
                  ┌────┴────┐
                  │    Y    │ ← ALU输入锁存器
                  └────┬────┘
                       │
                  ┌────┴────┐
                  │   ALU   │
                  └────┬────┘
                       │
                  结果送回总线
```

## 原理分析

### 典型指令的微操作序列（单总线）

#### 取指（所有指令共有）

| 节拍 | 微操作 | 控制信号 |
|------|--------|---------|
| T0 | PC → BUS → MAR | PC_out, MAR_in |
| T1 | Read, M(MAR) → MDR | Read, MDR_in |
| T2 | MDR → BUS → IR | MDR_out, IR_in |
| T2 | PC + 1 → PC | PC_inc |

共 **3个时钟周期**。

#### 加法指令：ADD R1, [addr]（直接寻址）

| 节拍 | 微操作 | 控制信号 |
|------|--------|---------|
| T0 | Ad(IR) → BUS → MAR | IR_out, MAR_in |
| T1 | Read, M(MAR) → MDR | Read, MDR_in |
| T2 | MDR → BUS → Y | MDR_out, Y_in |
| T3 | R1 → BUS → ALU, Y + BUS → Z | R1_out, ALU_add, Z_in |
| T4 | Z → BUS → R1 | Z_out, R1_in |

执行阶段共 **5个时钟周期**。

整条指令：取指3 + 执行5 = **8个时钟周期**。

> **注：** T3中不能同时使用R1_out和R1_in——单总线同一时刻总线只能被一个源驱动。ALU的结果需先存入中间寄存器Z，再在T4写回R1。这是单总线结构的关键约束。

#### 存数指令：STORE R1, [addr]（直接寻址）

| 节拍 | 微操作 | 控制信号 |
|------|--------|---------|
| T0 | Ad(IR) → BUS → MAR | IR_out, MAR_in |
| T1 | R1 → BUS → MDR | R1_out, MDR_in |
| T2 | Write | Write |

执行阶段共 **3个时钟周期**。

#### 寄存器运算：ADD R1, R2

| 节拍 | 微操作 | 控制信号 |
|------|--------|---------|
| T0 | R1 → BUS → Y | R1_out, Y_in |
| T1 | R2 → BUS → ALU, Y + BUS → R1 | R2_out, ALU_add, R1_in |

执行阶段共 **2个时钟周期**。

### 为什么单总线需要更多周期

**根本原因**：单总线同一时刻只能有一个数据传输。

执行 A + B → C 的最小步骤：
1. A → 总线 → Y（锁存第一个操作数）
2. B → 总线 → ALU另一输入，Y + B → 结果
3. 结果 → 总线 → C（写回）

**3步，不能更少**，因为总线在同一时刻只能被一个源占用。

## 直观理解

### 生活类比

单总线就像**单车道公路**：
- 所有车辆（数据）必须排队通过
- 同一时刻只能有一个方向的车通行
- 简单可靠但效率低

### 记忆技巧

- **单总线的核心约束**：一个时钟周期 = 一次总线传输（或一次ALU运算）
- **计算时钟周期数**：数"需要在总线上传输几次数据"就是最少周期数

## 知识关联

### 跨章节联系

- 与**双总线**和**专用通路**对比：并行度递增，速度递增，成本递增
- **微操作序列**直接用于硬布线/微程序控制器设计

### 常考组合

- 给定单总线结构，写出指令的微操作序列并计算时钟周期数
- 对比单总线和双总线执行同一指令的周期数差异

### 易错陷阱

1. **陷阱**：ALU运算也需要一个时钟周期（ALU是组合逻辑，但结果写入目标寄存器需要时钟沿）。
2. **陷阱**：Y锁存器是必要的，否则第一个操作数在总线被其他数据占用时会丢失。
## 代码/模拟

### Verilog实现单总线数据通路

```verilog
// ============================================
// 单总线数据通路模拟 - 适用于408考研理解微操作
// 核心约束: 同一时刻总线只有一个数据源
// ============================================

module single_bus_datapath(
    input         clk,
    input  [2:0]  alu_op,       // ALU操作选择
    // 控制信号（来自控制器）
    input         pc_out, pc_in, pc_inc,
    input         mar_in, mdr_in, mdr_out,
    input         ir_in,
    input         r0_out, r0_in,
    input         r1_out, r1_in,
    input         y_in,          // ALU输入锁存器
    input         z_out,         // ALU输出寄存器
    input         read, write,
    // 状态输出
    output [31:0] ir_out,
    output        zero_flag
);

    // 内部寄存器
    reg [31:0] PC, MAR, MDR, IR, R0, R1, Y, Z;
    reg [31:0] bus;              // 内部总线

    // 存储器（简化为数组）
    reg [31:0] memory [0:255];

    // ALU
    wire [31:0] alu_result;
    always @(*) begin
        case (alu_op)
            3'b000: alu_result = Y + bus;   // ADD
            3'b001: alu_result = Y - bus;   // SUB
            3'b010: alu_result = Y & bus;   // AND
            3'b011: alu_result = Y | bus;   // OR
            default: alu_result = 32'b0;
        endcase
    end

    // 总线仲裁：根据控制信号选择数据源
    always @(*) begin
        casex ({pc_out, mdr_out, r0_out, r1_out, z_out})
            5'b1xxxx: bus = PC;
            5'b01xxx: bus = MDR;
            5'b001xx: bus = R0;
            5'b0001x: bus = R1;
            5'b00001: bus = Z;
            default:  bus = 32'bz;  // 高阻
        endcase
    end

    // 时钟沿更新各寄存器
    always @(posedge clk) begin
        if (pc_in)   PC  <= bus;
        if (pc_inc)  PC  <= PC + 1;
        if (mar_in)  MAR <= bus;
        if (mdr_in)  MDR <= read ? memory[bus[7:0]] : bus;
        if (ir_in)   IR  <= bus;
        if (r0_in)   R0  <= bus;
        if (r1_in)   R1  <= bus;
        if (y_in)    Y   <= bus;
        if (write)   memory[MAR[7:0]] <= MDR;
        // Z寄存器在每个时钟沿更新ALU结果
        Z <= alu_result;
    end

    assign ir_out = IR;
    assign zero_flag = (alu_result == 0);

endmodule
```

### Python模拟单总线微操作序列

```python
"""单总线数据通路微操作模拟 - 适用于408考研复习"""

def simulate_single_bus():
    """模拟ADD R1, [addr]在单总线上的执行过程"""
    # 模拟寄存器状态
    PC = 0x1000
    IR = 0x00010050  # 假设的指令: ADD R1, [0x50]
    R1 = 10
    memory = {0x50: 20}  # M[0x50] = 20

    print("=== ADD R1, [0x50] 单总线微操作序列 ===")
    print(f"初始: R1={R1}, M[0x50]={memory[0x50]}\n")

    # 取指周期
    print("【取指周期】")
    print(f"  T0: PC(0x{PC:X}) → 总线 → MAR        控制: PC_out, MAR_in")
    print(f"  T1: Read, M(MAR) → MDR             控制: Read, MDR_in")
    print(f"  T2: MDR → 总线 → IR                控制: MDR_out, IR_in")
    print(f"      PC+1 → PC                      控制: PC_inc")

    # 执行周期
    print(f"\n【执行周期】(直接寻址)")
    addr = 0x50
    print(f"  T0: Ad(IR)=0x{addr:X} → 总线 → MAR    控制: IR_out, MAR_in")
    print(f"  T1: Read, M(MAR)={memory[addr]} → MDR  控制: Read, MDR_in")
    print(f"  T2: MDR({memory[addr]}) → 总线 → Y       控制: MDR_out, Y_in")
    print(f"  T3: R1({R1}) → 总线 → ALU               控制: R1_out, ALU_add")
    print(f"      Y({memory[addr]}) + BUS → Z           Z = {R1 + memory[addr]}")
    print(f"  T4: Z({R1 + memory[addr]}) → 总线 → R1    控制: Z_out, R1_in")

    print(f"\n共: 取指3周期 + 执行5周期 = 8周期")
    print(f"结果: R1 = {R1 + memory[addr]}")

    # 对比：寄存器运算
    print(f"\n=== 对比: ADD R1, R2 (寄存器运算) ===")
    print(f"  取指: T0, T1, T2 (3周期)")
    print(f"  执行: T0: R1 → Y (R1_out, Y_in)")
    print(f"        T1: R2 → ALU, Y+R2 → R1 (R2_out, ALU_add, R1_in)")
    print(f"  共: 取指3 + 执行2 = 5周期")

simulate_single_bus()
```

3. **陷阱**：PC+1可以与其他操作并行（在某些设计中PC是独立的，不经过内部总线自增）。
