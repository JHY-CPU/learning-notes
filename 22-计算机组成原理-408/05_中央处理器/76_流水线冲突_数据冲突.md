# 70_流水线冲突_数据冲突

## 核心概念

- **数据冲突（Data Hazard）**：后续指令需要用到前面指令尚未产生的数据
- 是流水线中**最常见**的冲突类型
- 分为三种：**RAW**（写后读）、**WAR**（读后写）、**WAW**（写后写）

## 原理分析

### 一、RAW（Read After Write）— 写后读

- **最常见**的数据冲突
- 后续指令需要**读取**前面指令尚未**写回**的数据
- 也称为**真数据依赖（True Dependency）**

**示例**：
```
I1: ADD  R1, R2, R3    ; R1 ← R2 + R3
I2: SUB  R4, R1, R5    ; R4 ← R1 - R5  （需要 R1 的新值）
```

I2 在 ID 阶段需要读 R1，但 I1 要到 WB 阶段才写入 R1。

```
时间→    1    2    3    4    5    6
I1:     [IF] [ID] [EX][MEM][WB]
I2:          [IF] [ID] [EX] [MEM] [WB]
                      ↑
                 I2 在 ID 读 R1，
                 但 I1 还没写回！
```

### 二、WAR（Read After Write 的反向）— 读后写

- 后面指令要**写入**的寄存器，前面指令还没**读完**
- 也称为**反依赖（Anti-Dependency）**

**示例**：
```
I1: ADD  R1, R2, R3    ; R1 ← R2 + R3（读 R2）
I2: SUB  R2, R4, R5    ; R2 ← R4 - R5（写 R2）
```

如果 I1 还没读完 R2，I2 就写入了新值，I1 会读到错误数据。

**注意**：在**按序执行**的流水线中，WAR 不会发生（前面的指令先读）。只有在**乱序执行**或**写回提前**时才会出现。

### 三、WAW（Write After Write）— 写后写

- 两条指令**写入同一个寄存器**
- 必须保证写入顺序正确

**示例**：
```
I1: MUL  R1, R2, R3    ; R1 ← R2 × R3（多周期）
I2: ADD  R1, R4, R5    ; R1 ← R4 + R5（单周期）
```

如果 I2 先写回，R1 的最终值错误。

**注意**：在**按序执行**的单发射流水线中，WAW 不会发生。只有在**多发射**或**乱序执行**中才出现。

### 四、三种冲突对比

| 类型 | 全称 | 依赖关系 | 何时出现 |
|------|------|---------|---------|
| RAW | Write After Write（先写后读） | 真数据依赖 | 按序流水线中**最常见** |
| WAR | Read After Write（先读后写） | 名字依赖（反依赖） | 乱序执行/提前写回 |
| WAW | Write After Write（先写后写） | 名字依赖（输出依赖） | 多发射/乱序执行 |

## 直观理解

- **RAW**：妈妈还没做好饭（写），孩子就要吃（读）——必须等
- **WAR**：爸爸还没读完报纸（读），孩子就要在报纸上画画（写）——必须等读完
- **WAW**：两个人都要在同一张纸上写最终答案——顺序必须正确

## 代码/模拟

### Python模拟流水线数据冲突检测

```python
"""流水线数据冲突检测与处理模拟 - 适用于408考研复习"""

class PipelineHazardDetector:
    """检测流水线中的RAW、WAR、WAW数据冲突"""

    def __init__(self):
        self.instructions = []

    def add_instruction(self, name, dest=None, src_regs=None):
        """
        添加指令
        :param name: 指令名称，如 'ADD R1, R2, R3'
        :param dest: 目的寄存器，如 'R1'
        :param src_regs: 源寄存器列表，如 ['R2', 'R3']
        """
        self.instructions.append({
            'name': name,
            'dest': dest,
            'src': src_regs or []
        })

    def detect_raw_hazards(self):
        """检测RAW（写后读）冲突 - 最常见"""
        print("=== RAW（写后读）冲突检测 ===")
        hazards = []
        for i in range(len(self.instructions)):
            for j in range(i + 1, min(i + 4, len(self.instructions))):
                # j指令的源寄存器 = i指令的目的寄存器
                if self.instructions[i]['dest'] in self.instructions[j]['src']:
                    distance = j - i
                    hazards.append((i, j))
                    stall_cycles = max(0, 2 - distance)  # EX后才有结果
                    print(f"  I{i+1}→I{j+1}: {self.instructions[i]['name']}")
                    print(f"    写 {self.instructions[i]['dest']}, "
                          f"I{j+1}需要读取 → 距离={distance}, "
                          f"{'需暂停'+str(stall_cycles)+'周期' if stall_cycles > 0 else '可用数据旁路'}")
        return hazards

    def detect_war_hazards(self):
        """检测WAR（读后写）冲突 - 乱序执行时出现"""
        print("\n=== WAR（读后写）冲突检测 ===")
        hazards = []
        for i in range(len(self.instructions)):
            for j in range(i + 1, min(i + 4, len(self.instructions))):
                # j指令的目的寄存器 = i指令的源寄存器
                if self.instructions[j]['dest'] in self.instructions[i]['src']:
                    hazards.append((i, j))
                    print(f"  I{i+1}→I{j+1}: {self.instructions[i]['name']}")
                    print(f"    I{i+1}读 {self.instructions[j]['dest']}, "
                          f"I{j+1}要写该寄存器 → 按序执行不会发生")
        if not hazards:
            print("  无WAR冲突（按序流水线中不会发生）")
        return hazards

    def simulate_with_stalls(self):
        """模拟带暂停的流水线执行"""
        print("\n=== 带暂停的流水线时空图 ===")
        stages = ['IF', 'ID', 'EX', 'MEM', 'WB']
        n = len(self.instructions)

        # 简化：在RAW距离<2时插入暂停
        stalls = [0] * n
        for i in range(n):
            for j in range(i + 1, min(i + 3, n)):
                if (self.instructions[i]['dest'] in self.instructions[j]['src']
                        and j - i < 2):
                    stalls[j] = max(stalls[j], 2 - (j - i))

        total = n + 4 + sum(stalls)
        # 打印表头
        print(f"{'':8}", end='')
        for c in range(total):
            print(f" T{c+1:>2}", end='')
        print()

        cycle = 0
        for i in range(n):
            start = i + sum(stalls[:i+1])
            row = f"I{i+1:<7}"
            for c in range(total):
                stage_idx = c - start
                if 0 <= stage_idx < 5:
                    print(f" {stages[stage_idx]:>3}", end='')
                else:
                    print(f"    ", end='')
            print()

# 检测笔记中的示例
detector = PipelineHazardDetector()

# RAW冲突示例
detector.add_instruction('ADD R1, R2, R3', dest='R1', src_regs=['R2', 'R3'])
detector.add_instruction('SUB R4, R1, R5', dest='R4', src_regs=['R1', 'R5'])
detector.add_instruction('AND R6, R1, R7', dest='R6', src_regs=['R1', 'R7'])
detector.add_instruction('OR  R8, R4, R9', dest='R8', src_regs=['R4', 'R9'])

detector.detect_raw_hazards()
detector.detect_war_hazards()
detector.simulate_with_stalls()
```

### 数据旁路（Forwarding）模拟

```python
def simulate_forwarding():
    """模拟数据旁路解决RAW冲突"""
    print("\n=== 数据旁路（Forwarding）模拟 ===")
    print("指令序列: ADD R1,R2,R3 | SUB R4,R1,R5 | AND R6,R7,R8\n")

    stages = ['IF', 'ID', 'EX', 'MEM', 'WB']
    # 无旁路：SUB需暂停2周期
    print("【无旁路】SUB需在ID阶段等待R1:")
    print(f"  ADD: IF  ID  EX  MEM  WB")
    print(f"  SUB:     IF  --  --  ID  EX  MEM  WB  (暂停2周期)")

    # 有旁路：从EX结果直接旁路到SUB的EX输入
    print("\n【有旁路】从EX/MEM寄存器旁路:")
    print(f"  ADD: IF  ID  EX  MEM  WB")
    print(f"  SUB:     IF  ID  EX  MEM  WB       (无需暂停)")
    print(f"                    ↑")
    print(f"              旁路: ADD的EX结果→SUB的EX输入")

    # 需要1周期暂停的情况
    print("\n【需1周期暂停】LD后紧跟使用该寄存器的指令:")
    print(f"  LD  R1,0(R2): IF  ID  EX  MEM  WB")
    print(f"  ADD R3,R1,R4:     IF  ID  --  EX  MEM  WB  (暂停1周期)")
    print(f"                           ↑")
    print(f"                     MEM结果无法及时旁路到EX")

simulate_forwarding()
```
