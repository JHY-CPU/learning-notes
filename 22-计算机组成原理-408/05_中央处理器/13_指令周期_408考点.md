# 13_指令周期_408考点

## 核心概念

本文件聚焦指令周期相关计算题和判断题，是408高频考点。

### 核心公式

**CPI（Cycles Per Instruction，每条指令的时钟周期数）**：

$$CPI = \frac{\text{指令执行所需的总时钟周期数}}{1}$$

**CPU执行时间**：

$$T_{CPU} = \text{指令数} \times CPI \times T_{时钟} = \frac{\text{指令数} \times CPI}{f_{时钟}}$$

**平均CPI**（多种指令混合）：

$$\overline{CPI} = \sum_{i=1}^{n} (CPI_i \times P_i)$$

其中 $P_i$ 为第 $i$ 类指令在程序中的比例。

**MIPS（百万条指令每秒）**：

$$MIPS = \frac{f_{时钟}}{CPI \times 10^6}$$

## 典型考题解析

### 题型1：给定指令求时钟周期数

**题目**：某机采用单总线结构，取指需3个时钟周期，间址需3个时钟周期，执行需2个时钟周期，中断需3个时钟周期。求以下各指令的时钟周期数：
(1) 寄存器直接寻址的加法指令 ADD R1, R2
(2) 直接寻址的取数指令 LOAD [addr]
(3) 间接寻址的取数指令 LOAD [[addr]]
(4) 无条件跳转指令 JMP addr

**解答**：
(1) ADD R1, R2（无间址，无中断）：
$$CPI = 3(\text{取指}) + 2(\text{执行}) = 5$$

(2) LOAD [addr]（直接寻址，无间址）：
$$CPI = 3(\text{取指}) + 2(\text{执行}) = 5$$

注意：直接寻址时执行阶段需要额外访存取数据，但这里执行周期已包含访存时间。若题目说执行周期不含访存，则：
$$CPI = 3(\text{取指}) + 2(\text{执行访址}) + 2(\text{执行读数}) = 7$$
具体以题目描述为准。

(3) LOAD [[addr]]（间接寻址）：
$$CPI = 3(\text{取指}) + 3(\text{间址}) + 2(\text{执行}) = 8$$

(4) JMP addr（无间址）：
$$CPI = 3(\text{取指}) + 2(\text{执行}) = 5$$

### 题型2：平均CPI和执行时间

**题目**：某程序包含三种指令，比例和CPI如下：
- 指令A：占50%，CPI = 2
- 指令B：占30%，CPI = 4
- 指令C：占20%，CPI = 3

时钟频率为2GHz。程序共100万条指令。求：
(1) 平均CPI
(2) 执行时间

**解答**：

(1) 平均CPI：
$$\overline{CPI} = 0.5 \times 2 + 0.3 \times 4 + 0.2 \times 3 = 1.0 + 1.2 + 0.6 = 2.8$$

(2) 执行时间：
$$T = \frac{10^6 \times 2.8}{2 \times 10^9} = \frac{2.8 \times 10^6}{2 \times 10^9} = 1.4 \times 10^{-3} \text{ s} = 1.4 \text{ ms}$$

### 题型3：中断对指令周期的影响

**题目**：CPU在每个指令周期结束时检查中断请求。若检测到中断请求，需要3个时钟周期处理中断响应（关中断、保存PC、转向中断服务程序）。某程序包含1000条指令，执行过程中发生了100次中断。每条指令的取指和执行共需5个时钟周期。求总时钟周期数。

**解答**：

$$\text{总周期数} = 1000 \times 5 + 100 \times 3 = 5000 + 300 = 5300$$

### 题型4：指令执行时间对比

**题目**：比较以下两种方案执行同一程序的时间。
- 方案A：单指令周期CPU，所有指令周期 = 8个时钟周期
- 方案B：多指令周期CPU，取指3 + 间址3 + 执行可变

程序包含1000条指令，其中：
- 50%取指+执行（各需3个时钟周期，共6个）
- 30%取指+间址+执行（各需3个时钟周期，共9个）
- 20%取指+执行+中断（各需3个时钟周期，共9个）

**解答**：

方案A：$T_A = 1000 \times 8 \times T_{时钟} = 8000 \times T_{时钟}$

方案B：$T_B = 1000 \times (0.5 \times 6 + 0.3 \times 9 + 0.2 \times 9) \times T_{时钟}$
$$= 1000 \times (3.0 + 2.7 + 1.8) \times T_{时钟} = 7500 \times T_{时钟}$$

方案B更快（节省了500个时钟周期）。

## 直观理解

### 答题技巧

1. **审题关键**：明确"取指/间址/执行/中断各需几个时钟周期"是题目给的。
2. **间址判断**：看寻址方式，间接寻址才有间址周期。
3. **中断判断**：看题目是否有"发生中断"的条件。
4. **平均CPI**：加权平均，权重是各类指令的比例。

### 常见陷阱

1. **陷阱**：直接寻址 ≠ 间址。直接寻址的地址码本身就是有效地址，不需要间址周期。间接寻址的地址码指向的存储单元中存放的才是有效地址。
2. **陷阱**：CPI是"每条指令的时钟周期数"，不是"每时钟周期的指令数"。
3. **陷阱**：平均CPI不能简单地将各指令CPI相加取平均，必须按比例加权。
