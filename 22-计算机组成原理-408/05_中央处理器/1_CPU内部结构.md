# 02_CPU内部结构

## 核心概念

- **CPU内部组成**：运算器 + 控制器 + 寄存器组 + 内部总线（四大部分）。
- **运算器（Data Processing Unit）**：负责数据的算术和逻辑运算。
- **控制器（Control Unit）**：负责指挥和协调CPU内外各部件的工作。
- **内部总线**：连接CPU内部各寄存器和功能部件的数据传输通道。

### 详细部件清单

| 部件 | 所属模块 | 功能 |
|------|----------|------|
| ALU（算术逻辑单元） | 运算器 | 执行算术/逻辑运算 |
| 累加器ACC | 运算器 | 存放操作数和运算结果 |
| 乘商寄存器MQ | 运算器 | 乘法存放低位结果，除法存放商 |
| 操作数寄存器X | 运算器 | ALU的输入寄存器 |
| 通用寄存器组R0~R(n-1) | 运算器 | 存放操作数、中间结果 |
| CU（控制单元） | 控制器 | 产生微命令（控制信号） |
| PC（程序计数器） | 控制器 | 存放下一条指令地址 |
| IR（指令寄存器） | 控制器 | 存放当前正在执行的指令 |
| 时序系统 | 控制器 | 产生时钟/节拍信号 |
| MAR（存储器地址寄存器） | 接口 | 存放要访问的主存地址 |
| MDR（存储器数据寄存器） | 接口 | 存放与主存交换的数据 |

## 原理分析

### 运算器内部结构

```
通用寄存器组        ACC      MQ
  R0~R(n-1)         │        │
    │               │        │
    ▼               ▼        ▼
 ┌─────────────────────────────────┐
 │          内部总线               │
 └──────────┬──────────────────────┘
            │
            ▼
    ┌───────────────┐
    │      ALU      │◄── X（操作数寄存器）
    │  算术逻辑单元  │
    └───────────────┘
            │
            ▼
        运算结果 → 写回总线 → 目标寄存器
```

- ACC（累加器）：加法运算时存放一个加数，运算后存放和。
- MQ（乘商寄存器）：乘法时存放乘数和低位积，除法时存放商。
- X：ALU的另一个输入寄存器，存放被操作数。
- 不同运算中各寄存器的分工：

| 运算 | 被加/被减/被乘/被除数 | 加/减/乘/除数 | 结果高位 | 结果低位 |
|------|----------------------|---------------|----------|----------|
| 加法 | ACC | X | ACC(和) | — |
| 减法 | ACC | X | ACC(差) | — |
| 乘法 | X | MQ | ACC(高位积) | MQ(低位积) |
| 除法 | ACC | X | ACC(余数) | MQ(商) |

### 控制器内部结构

```
         指令流
           │
           ▼
  ┌─────┐    ┌─────┐
  │ PC  │───→│MAR  │───→ 地址总线 → 主存
  └─────┘    └─────┘               │
                                   ▼
  ┌─────┐    ┌─────┐        数据总线
  │ IR  │◄───│MDR  │◄────── 主存数据
  └──┬──┘    └─────┘
     │  操作码
     ▼
  ┌─────┐
  │ CU  │───→ 控制信号 → 各执行部件
  │     │
  └──┬──┘
     │  时序
     ▼
  ┌──────────┐
  │ 时序系统  │──→ T0, T1, T2... 节拍信号
  └──────────┘
```

- CU根据IR中的操作码、时序信号、状态标志，产生相应的控制信号序列。
- PC指向指令流，每取一条指令PC自动递增（或被转移指令修改）。

### 内部总线连接

- **单总线**：所有寄存器共用一条总线，简单但同一时刻只能有一个数据传送。
- **双总线**：两条总线分别连接ALU的两个输入，可同时读取两个操作数。
- **三总线**：更进一步的并行，速度最快，硬件最复杂。

## 直观理解

### 生活类比

CPU内部结构就像一个**公司组织**：
- **运算器**= 业务部门（干活的），ALU是核心骨干
- **控制器**= 管理层（指挥的），CU是总经理
- **寄存器组**= 办公桌面/文件柜，存放手头材料
- **内部总线**= 公司内部的走廊/快递通道，运送文件
- **PC**= 日程表，写下一项任务
- **IR**= 当前正在处理的任务单

### 记忆技巧

- 运算器记住「**ALU+寄存器组**」，核心就是"算"
- 控制器记住「**CU+PC+IR+时序**」，核心就是"管"
- **MAR和MDR**是CPU与主存的"桥梁"，有时归在CPU内，有时归在CPU外（考试看题目说明）

### 常考组合

- 运算器中各寄存器在不同运算中的角色（加减乘除对比表是高频考点）
- 控制器产生控制信号的依据：操作码 + 时序 + 状态标志
- 内部总线类型与数据通路速度的关系

### 易错陷阱

1. **陷阱**：乘法运算中被乘数放X，乘数放MQ，结果高位在ACC、低位在MQ。不要记反。
2. **陷阱**：除法中被除数在ACC，除数在X（不是MQ），商在MQ，余数在ACC。
3. **陷阱**：PC属于控制器的一部分（因为它控制指令流），不属于运算器。
