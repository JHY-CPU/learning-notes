# 74_流水线冲突_控制冲突

## 核心概念

- **控制冲突（Control Hazard / Branch Hazard）**：转移指令改变了 PC，导致已取指令可能作废
- 本质：**下一条指令的地址不确定**
- 是流水线中**影响最大**的冲突类型

## 原理分析

### 一、控制冲突的产生

```
时间→    1    2    3    4    5    6    7
I1:     [IF] [ID] [EX][MEM][WB]        ← 正常指令
I2(BR):      [IF] [ID] [EX]...         ← 转移指令
I3:               [IF] [ID]...         ← 已取，但可能作废！
I4:                    [IF]...         ← 已取，可能作废！
```

- I2 是转移指令（如 BEQ R1, R2, offset）
- I2 在 **EX 阶段**才能确定是否转移、转移到哪
- 但 I3、I4 已经在 I2 之前被取出了
- 如果发生转移，I3、I4 需要**作废**，从转移目标地址重新取指令

### 二、转移指令的损失

假设转移指令在 EX 阶段确定转移目标：

- 流水线深度为 $k$，转移判断在第 $j$ 段
- 最坏情况需要作废 $j - 1$ 条已取指令
- 这 $j - 1$ 个周期称为**转移损失（Branch Penalty）**

### 三、控制冲突的解决方法

#### 方法 1：暂停（Stall）

- 检测到转移指令后，暂停流水线直到转移结果确定
- 简单但损失大：每个转移损失 $j-1$ 个周期

#### 方法 2：尽早判断

- 将转移判断**提前到 ID 阶段**
- 损失从 $j-1$ 减少到 1 个周期
- 需要在 ID 阶段增加比较器

#### 方法 3：分支预测

- **预测不转移**：继续取后续指令
  - 预测正确：无损失
  - 预测错误：作废已取指令，损失若干周期
- **分支预测**详见第 75 节

#### 方法 4：延迟槽（Delayed Branch）

- 转移指令后紧跟一条**延迟槽指令**
- 无论是否转移，延迟槽指令都执行
- 编译器将有用指令放入延迟槽

```
BEQ  R1, R2, target
ADD  R3, R4, R5      ← 延迟槽指令，始终执行
...
target:
SUB  R6, R7, R8
```

### 四、转移指令的比例和影响

典型程序中，转移指令约占 **15%-25%**。

若转移损失为 2 个周期，转移比例为 20%：
$$\text{CPI} = 1 + 0.20 \times 2 = 1.4$$

性能损失 = 40%！控制冲突的影响非常大。

## 直观理解

**控制冲突**就像**开车到岔路口**：

- 一直往前开（取后续指令）
- 到了路口才知道要左转还是直行（EX 阶段才知道转移结果）
- 如果猜错了（预测失败），要倒回去走另一条路
- **暂停**：到了路口停车想清楚再走
- **预测**：提前猜一个方向，猜对了不用停
