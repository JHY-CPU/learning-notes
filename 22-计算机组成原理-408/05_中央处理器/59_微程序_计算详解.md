# 59_微程序_计算详解

## 核心概念

- **CM容量计算**：微指令条数 × 微指令字长
- **微指令字长计算**：控制字段 + 判别测试字段 + 下址字段
- **微指令条数计算**：各指令微程序的微指令条数之和（含公共微程序）
- **408综合题常见计算类型**

## 原理分析

### 例题1：CM容量计算

**题目**：某计算机有 32 条机器指令，取指微程序 3 条，间址微程序 3 条，中断微程序 3 条。各指令的执行微程序平均包含 4 条微指令。微指令字长 32 位。求 CM 容量。

**解**：
- 公共微程序：$3 + 3 + 3 = 9$ 条
- 指令执行微程序：$32 \times 4 = 128$ 条
- 总微指令条数：$9 + 128 = 137$ 条
- 取大于137的2的幂：$2^8 = 256$（地址需要8位）
- CM容量：$256 \times 32 = 8192$ bit $= 1$ KB

**注意**：如果题目不要求地址对齐2的幂，则 CM = 137 × 32 = 4384 bit = 548 B。

### 例题2：微指令字长计算

**题目**：某系统有 40 个微命令，分为 5 组：
- A组：8个（寄存器输出选择）
- B组：12个（寄存器输入选择）
- C组：8个（ALU操作选择）
- D组：8个（存储器控制）
- E组：4个（其他控制）

CM有 128 条微指令，判别测试字段 3 位。用字段直接编码，求微指令字长。

**解**：
- A组位数：$\lceil\log_2(8+1)\rceil = 4$ 位
- B组位数：$\lceil\log_2(12+1)\rceil = 4$ 位
- C组位数：$\lceil\log_2(8+1)\rceil = 4$ 位
- D组位数：$\lceil\log_2(8+1)\rceil = 4$ 位
- E组位数：$\lceil\log_2(4+1)\rceil = 3$ 位
- 控制字段：$4+4+4+4+3 = 19$ 位
- 判别测试字段：3 位
- 下址字段：$\lceil\log_2 128\rceil = 7$ 位
- 微指令字长：$19 + 3 + 7 = 29$ 位

### 例题3：微指令条数与CM地址分配

**题目**：某计算机有 4 条指令（ADD, SUB, LOAD, STORE），取指 3 条，间址 3 条，中断 3 条。各指令执行微程序条数：
- ADD: 3 条
- SUB: 3 条
- LOAD: 4 条（需要间址）
- STORE: 4 条（需要间址）

求微指令总条数，并分配CM地址。

**解**：

| 区域 | 微指令条数 | 地址范围 |
|------|-----------|---------|
| 公共取指 | 3 | 00H ~ 02H |
| 公共间址 | 3 | 03H ~ 05H |
| 公共中断 | 3 | 06H ~ 08H |
| ADD执行 | 3 | 09H ~ 0BH |
| SUB执行 | 3 | 0CH ~ 0EH |
| LOAD执行 | 4 | 0FH ~ 12H |
| STORE执行 | 4 | 13H ~ 16H |

总计：$3 + 3 + 3 + 3 + 3 + 4 + 4 = 23$ 条

CM至少需要 23 个存储单元。地址需要 $\lceil\log_2 23\rceil = 5$ 位。

### 例题4：综合设计题

**题目**：某单总线CPU有 $R_0 \sim R_3$ 四个寄存器，ALU支持 ADD/SUB，CM有 64 条微指令，字长 28 位。字段直接编码如下：

| 组 | 微命令 | 位数 |
|----|--------|------|
| 1（输出选择） | $PC_{\text{out}}, MDR_{\text{out}}, R_0^{\text{out}} \sim R_3^{\text{out}}, Z_{\text{out}}, Ad(IR)_{\text{out}}$ | 4位（8+1） |
| 2（输入选择） | $MAR_{\text{in}}, MDR_{\text{in}}, IR_{\text{in}}, PC_{\text{in}}, R_0^{\text{in}} \sim R_3^{\text{in}}, Y_{\text{in}}, Z_{\text{in}}$ | 4位（10+1） |
| 3（ALU操作） | $ALU_{\text{add}}, ALU_{\text{sub}}$ | 2位（2+1） |
| 4（存储器） | Read, Write | 2位（2+1） |
| 5（其他） | $PC+1$, 中断允许 | 2位（直接编码2位） |
| 判别测试 | | 2位 |
| 下址 | 64条 | 6位 |

验证：$4 + 4 + 2 + 2 + 2 + 2 + 6 = 22$ 位 ≠ 28 位

说明还有额外控制信号未列出，或某些组更大。题目应保证各字段位数之和等于给定的微指令字长。

写出 ADD $R_0, R_1$ 取指 $T_0$ 的微指令编码（$PC \to MAR$）：
- 组1：$PC_{\text{out}}$ 编码 = 0001
- 组2：$MAR_{\text{in}}$ 编码 = 0001
- 组3：无操作 = 00
- 组4：无操作 = 00
- 组5：无操作 = 00
- 判别测试：00（无）
- 下址：000001（下一条 = 01H）

微指令 = `0001 0001 00 00 00 00 000001` = 22 位

## 直观理解

### 计算口诀

1. **控制字段**：分组求和，$\sum \lceil \log_2(n_i+1) \rceil$
2. **下址字段**：$\lceil \log_2(\text{CM微指令条数}) \rceil$
3. **字长**：控制字段 + 判别测试 + 下址
4. **CM容量**：微指令条数 × 字长

### 常见错误

1. 忘记每组 +1（为"无操作"留编码）
2. 下址字段位数算错（用微指令条数，不是机器指令条数）
3. 没有考虑公共微程序（取指、间址、中断）
4. 直接编码和字段编码混淆
