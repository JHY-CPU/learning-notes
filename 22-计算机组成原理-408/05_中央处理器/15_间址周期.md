# 15_间址周期

## 核心概念

- **间址周期（Indirect Cycle）**：当指令使用**间接寻址**时，在取指之后、执行之前需要额外的访存操作来获取有效地址。
- **间接寻址**：指令地址码字段中存放的不是操作数的地址，而是存放操作数地址的那个存储单元的地址。
- **一次间址**：需要一次额外访存来得到有效地址。
- **多次间址**：可能需要多次额外访存（直到标志位指示得到的是有效地址）。

### 间址的地址变换过程

$$\text{指令地址码} \xrightarrow{第一次访存} \text{有效地址} \xrightarrow{第二次访存(执行阶段)} \text{操作数}$$

## 原理分析

### 间址周期的微操作序列

| 节拍 | 微操作 | 说明 |
|------|--------|------|
| T0 | Ad(IR) → MAR | IR中的地址码字段（形式地址）送MAR |
| T1 | 1 → R | 向主存发出读命令 |
| T2 | M(MAR) → MDR | 主存将有效地址送MDR |
| T3 | MDR → Ad(IR) | 有效地址替换IR中的地址码字段 |

**注意**：最后一步将有效地址写回IR的地址码字段，使得执行阶段可以直接使用有效地址。

### 间址周期的数据流

$$IR(Ad) \to MAR \to \text{主存} \to MDR \to IR(Ad)$$

详细过程：
```
IR（指令寄存器）
├── OP字段 ──→ CU（操作码不变，保持译码结果）
└── Ad字段 ──→ MAR（形式地址送地址寄存器）
                    │
                    ▼
              地址总线 → 主存（根据形式地址读出存储单元的内容）
                    │
                    ▼
              数据总线 → MDR（读出的是有效地址）
                    │
                    ▼
              MDR → IR(Ad)（有效地址替换形式地址）
```

### 间址周期的触发条件

间址周期不是每条指令都有的，触发条件：
- IR中的**间址标志位**为1（或指令的寻址方式编码指示为间接寻址）

控制逻辑：
$$\text{间址周期触发} = \text{IND} \cdot \text{间址标志}$$

其中 IND 是"处于间址周期"的状态触发器。

### 一次间址 vs 多次间址

**一次间址**（单级间接）：

$$\text{指令}[Ad] \to \text{存储单元}[M[Ad]] \to \text{有效地址}$$

仅需一次间址访存。

**多次间址**（多级间接）：

$$\text{指令}[Ad] \to M[Ad_1] \to M[Ad_2] \to \cdots \to \text{有效地址}$$

每次间址访存后检查标志位，判断是否还需要继续间接。

**示例**：假设存储单元内容的最高位为间接标志（1=还需间接，0=有效地址）。

```
指令：OP | Ad=100 | 间接标志=1
           │
           ▼
     M[100] = 1 | 200  （最高位1，还需间接）
           │
           ▼
     M[200] = 0 | 300  （最高位0，300就是有效地址）
           │
           ▼
     执行阶段：从M[300]取操作数
```

## 直观理解

### 生活类比

- **直接寻址**= 通讯录直接写着"张三：138xxxx"（号码就是你要的）
- **一次间址**= 通讯录写着"张三的办公室：201室"，你去201室再查张三的电话号码
- **多次间址**= 通讯录写着"问201室的人"，201室的人说"问305室的人"，305室的人告诉你张三的号码

### 记忆技巧

- **间址 = 多走一步（或多步）才能找到目标**
- **间址周期 = 取指之后额外的一次"读主存"**
- **间址只增加访存次数，不改变操作码**

## 知识关联

### 跨章节联系

- **第4章（指令系统）**：间接寻址是寻址方式之一，与直接寻址、寄存器寻址等并列
- **第3章（存储系统）**：每次间址都需要一次主存访问，增加访存开销
- **第5章（后续）**：间址周期是控制器状态机的一个状态

### 易错陷阱

1. **陷阱**：直接寻址不需要间址周期！指令地址码本身就是有效地址。
2. **陷阱**：一次间址增加**一次**访存（不是两次）。执行阶段还需要一次访存取操作数，但那是执行周期的事。
3. **陷阱**：间址访存取出的是"地址"，不是"操作数"。这个地址在执行阶段才能用来取操作数。
