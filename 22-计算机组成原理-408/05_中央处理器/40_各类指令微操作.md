# 34_各类指令微操作

## 核心概念

本文件汇总各类指令的完整微操作序列（取指+间址+执行+中断），作为硬布线/微程序控制器设计的参考。

### 指令分类与微操作总览

| 指令大类 | 典型指令 | 执行阶段主要操作 | 执行周期时钟数 |
|----------|----------|----------------|-------------|
| 算术运算 | ADD, SUB, MUL, DIV | 取操作数→ALU运算→写回 | 2~多 |
| 逻辑运算 | AND, OR, NOT, XOR | 同算术 | 2 |
| 数据传送 | LOAD, STORE | 访存取/存数据 | 3~4 |
| 转移 | JMP, JZ, JNZ, JC | 修改PC | 1~2 |
| 子程序 | CALL, RET | 压栈/弹栈+修改PC | 4~5 |
| 堆栈 | PUSH, POP | 修改SP+访存 | 3~4 |
| I/O | IN, OUT | 通过I/O端口传送 | 3~4 |
| 系统 | NOP, HALT, 中断相关 | 特殊操作 | 1~3 |

## 各类指令详细微操作

### 1. 算术运算指令

#### ADD Rd, Rs（寄存器直接寻址）

完整微操作：
```
取指(3): PC→MAR, Read, M(MAR)→MDR→IR, PC+1
执行(2): Rd→Y, Rs→ALU, Y+Rs→Rd
总计: 5个时钟周期
```

#### ADD Rd, [addr]（直接寻址）

```
取指(3): PC→MAR, Read, M(MAR)→MDR→IR, PC+1
执行(4): Ad(IR)→MAR, Read, M(MAR)→MDR, MDR→Y, Rd+Y→Rd
总计: 7个时钟周期
```

#### MUL Rs（乘法，多位乘需要多个周期）

```
取指(3): PC→MAR, Read, M(MAR)→MDR→IR, PC+1
执行(n): Booth算法循环n次（n=操作数位数）
  每次循环: 检查乘数位→决定加/减被乘数→移位
总计: 3+n个时钟周期
```

### 2. 逻辑运算指令

#### AND Rd, Rs

```
取指(3): PC→MAR, Read, M(MAR)→MDR→IR, PC+1
执行(2): Rd→Y, Rs→ALU, Y AND Rs → Rd
总计: 5个时钟周期
```

#### NOT Rd

```
取指(3): PC→MAR, Read, M(MAR)→MDR→IR, PC+1
执行(1): Rd→ALU, NOT Rd → Rd
总计: 4个时钟周期
```

### 3. 数据传送指令

#### LOAD Rd, [Rs]（寄存器间接寻址）

```
取指(3): PC→MAR, Read, M(MAR)→MDR→IR, PC+1
执行(3): Rs→MAR, Read, M(MAR)→MDR, MDR→Rd
总计: 6个时钟周期
```

#### STORE Rs, [Rd]（寄存器间接寻址）

```
取指(3): PC→MAR, Read, M(MAR)→MDR→IR, PC+1
执行(3): Rd→MAR, Rs→MDR, Write
总计: 6个时钟周期
```

### 4. 转移指令

#### JMP addr（无条件跳转）

```
取指(3): PC→MAR, Read, M(MAR)→MDR→IR, PC+1
执行(1): Ad(IR)→PC
总计: 4个时钟周期
```

#### JZ addr（条件跳转）

```
取指(3): PC→MAR, Read, M(MAR)→MDR→IR, PC+1
执行(1): if(ZF=1) Ad(IR)→PC
总计: 4个时钟周期
```

### 5. 子程序调用与返回

#### CALL addr

```
取指(3): PC→MAR, Read, M(MAR)→MDR→IR, PC+1
执行(5): SP-1→SP, SP→MAR, PC→MDR, Write, Ad(IR)→PC
总计: 8个时钟周期
```

#### RET

```
取指(3): PC→MAR, Read, M(MAR)→MDR→IR, PC+1
执行(4): SP→MAR, Read, M(MAR)→MDR, MDR→PC, SP+1→SP
总计: 7个时钟周期
```

### 6. 堆栈指令

#### PUSH Rs

```
取指(3): PC→MAR, Read, M(MAR)→MDR→IR, PC+1
执行(3): SP-1→SP, SP→MAR, Rs→MDR, Write
总计: 6个时钟周期
```

#### POP Rd

```
取指(3): PC→MAR, Read, M(MAR)→MDR→IR, PC+1
执行(4): SP→MAR, Read, M(MAR)→MDR, MDR→Rd, SP+1→SP
总计: 7个时钟周期
```

### 7. 特殊指令

#### NOP（空操作）

```
取指(3): PC→MAR, Read, M(MAR)→MDR→IR, PC+1
执行(0): 无操作
总计: 3个时钟周期
```

#### HALT（停机）

```
取指(3): PC→MAR, Read, M(MAR)→MDR→IR, PC+1
执行(1): 0→RUN（清除运行触发器）
总计: 4个时钟周期
```

## 直观理解

### 微操作数汇总表

| 指令 | 取指 | 间址 | 执行 | 总计(无中断) |
|------|------|------|------|-------------|
| ADD R,R | 3 | 0 | 2 | 5 |
| ADD R,M | 3 | 0 | 4 | 7 |
| NOT R | 3 | 0 | 1 | 4 |
| LOAD R,[M] | 3 | 0 | 3 | 6 |
| STORE R,[M] | 3 | 0 | 3 | 6 |
| JMP | 3 | 0 | 1 | 4 |
| CALL | 3 | 0 | 5 | 8 |
| RET | 3 | 0 | 4 | 7 |
| PUSH | 3 | 0 | 3 | 6 |
| POP | 3 | 0 | 4 | 7 |
| NOP | 3 | 0 | 0 | 3 |
| 间址取数 | 3 | 3 | 3 | 9 |

### 记忆规律

- **取指固定3个周期**
- **执行最快1个周期**（JMP, NOT），**最慢多个周期**（乘除法）
- **访存指令3~4个执行周期**（取决于是否需要先取地址再取数据）
- **堆栈操作需要修改SP+访存**，共3~4个执行周期

## 知识关联

- 本文件是硬布线控制器设计（35节）和微程序控制器设计（40+节）的基础
- 每条指令的微操作序列决定了控制信号的逻辑表达式
- 微操作数的差异决定了CPI的差异
