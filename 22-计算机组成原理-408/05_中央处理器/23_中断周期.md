# 17_中断周期

## 核心概念

- **中断周期（Interrupt Cycle）**：在指令执行完毕后，若有中断请求且CPU允许中断，则进入中断周期。
- **中断响应的时机**：在当前指令执行完成之后、下一条指令取指之前。
- **中断周期的核心操作**：关中断 → 保存断点（PC和PSW入栈） → 转向中断服务程序入口。

### 中断周期的微操作序列

| 节拍 | 微操作 | 说明 |
|------|--------|------|
| T0 | 0 → EINT（关中断） | 关闭中断允许，防止嵌套中断 |
| T1 | SP - 1 → SP | 堆栈指针下移，准备压栈 |
| T1 | SP → MAR | 栈顶地址送MAR |
| T2 | PC → MDR | 保存断点（返回地址）送MDR |
| T2 | 1 → W | 发出写命令 |
| T3 | MDR → M(MAR) | 断点压入堆栈 |
| T3 | 中断向量地址 → PC | PC指向中断服务程序入口 |

## 原理分析

### 中断响应的完整流程

```
指令执行完毕
     │
     ▼
  有中断请求？ ──否──→ 取下一条指令
     │
    是
     ▼
  中断允许(EINT=1)? ──否──→ 取下一条指令
     │
    是
     ▼
  进入中断周期：
  ┌─────────────────────────────────┐
  │ 1. 关中断：0 → EINT             │
  │ 2. 保存断点：PC → 堆栈          │
  │    SP-1 → SP → MAR              │
  │    PC → MDR → M(MAR)            │
  │ 3. 转向服务程序：向量地址 → PC   │
  └─────────────────────────────────┘
     │
     ▼
  开始执行中断服务程序
```

### 中断向量与中断向量表

- **中断向量**：中断服务程序的入口地址。
- **中断向量表**：存放所有中断向量的内存区域，通常位于低地址区。
- **中断类型号**：中断源的编号，用来索引中断向量表。

$$\text{中断向量地址} = \text{中断向量表基地址} + \text{中断类型号} \times \text{每项字节数}$$

### 中断周期中保存的内容

| 保存内容 | 保存位置 | 用途 |
|----------|----------|------|
| PC（断点地址） | 堆栈 | 中断返回后继续执行 |
| PSW（程序状态字） | 堆栈 | 恢复中断前的处理器状态 |
| 通用寄存器 | 中断服务程序开头保存 | 保护现场 |

- **断点**：被打断的程序的下一条指令地址（即PC的当前值）。
- **现场**：PSW和通用寄存器的内容。PSW通常由CPU硬件自动保存，通用寄存器由软件（中断服务程序）保存。

### 中断返回

中断服务程序执行完毕后，通过**中断返回指令（IRET）**恢复：

```
中断返回微操作：
T0: M(SP) → MDR      （从栈顶取出保存的PC）
T1: MDR → PC          （恢复断点）
T2: SP + 1 → SP       （栈指针上移）
T3: M(SP) → MDR       （取出保存的PSW）
T4: MDR → PSW         （恢复程序状态）
T5: 1 → EINT          （开中断）
```

## 直观理解

### 生活类比

中断周期就像**工作中接到紧急电话**：
1. **关中断**= 把手机调成"请勿打扰"模式（不再接受新的打扰）
2. **保存断点**= 在正在读的书里夹上书签，记住读到哪一页
3. **转向中断服务**= 翻开电话本找到紧急联系人的号码，开始处理紧急事务
4. **中断返回**= 紧急事务处理完后，翻回书签位置继续读书

### 记忆技巧

- **中断周期口诀**：「关保转」（关中断→保存断点→转向服务程序）
- **中断的核心思想**：让CPU能"暂停当前任务，处理紧急事件，然后再回来"

- 中断响应时机（指令执行完后，不是指令执行中）
- 中断周期的微操作序列（大题常考）
- 中断与异常的区别（中断是外部的，异常是内部的）

### 易错陷阱

1. **陷阱**：中断响应发生在指令执行**完毕后**，不是指令执行中。当前指令必须完整执行完。
2. **陷阱**：关中断（0→EINT）是为了防止中断嵌套，先保存好断点再开中断。
3. **陷阱**：中断向量地址 ≠ 中断服务程序入口地址。中断向量地址是向量表中的某项地址，其中存放的才是服务程序入口地址。但某些简化模型中，中断类型号直接就是向量地址。
