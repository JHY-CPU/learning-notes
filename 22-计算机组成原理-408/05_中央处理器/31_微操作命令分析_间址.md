# 31_微操作命令分析_间址

## 核心概念

- **间址微操作命令**：间接寻址时，在取指之后、执行之前需要额外的访存操作来获取有效地址。
- **间址的核心**：从主存读出的不是操作数，而是操作数的地址（有效地址）。
- **间址结束标志**：有效地址已替换IR中的地址码字段。

### 间址微操作序列

| 节拍 | 微操作 | 控制信号 | 说明 |
|------|--------|---------|------|
| T0(Ind0) | Ad(IR) → MAR | IR(Ad)_out, MAR_in | IR中的形式地址送MAR |
| T1(Ind1) | 1 → R | Read | 向主存发出读命令 |
| T2(Ind2) | M(MAR) → MDR | MDR_in | 读出有效地址到MDR |
| T2 | MDR → Ad(IR) | MDR_out, IR(Ad)_in | 有效地址写回IR的地址字段 |

## 原理分析

### 间址的数据流

$$IR(Ad) \xrightarrow{总线} MAR \xrightarrow{地址总线} \text{主存} \xrightarrow{数据总线} MDR \xrightarrow{总线} IR(Ad)$$

### 间址周期的触发逻辑

$$\text{进入间址} = FE_{结束} \cdot \text{间址标志}$$

间址标志可以是：
- IR中的某一位（如最高位）
- 操作码译码结果中的一部分（某些操作码暗示间址）

### 一次间址 vs 多次间址

**一次间址**（以上序列执行一次即可）：

$$M[\text{形式地址}] = \text{有效地址}$$

**多次间址**（需要重复执行间址序列，直到标志位=0）：

```
Ind0: Ad(IR) → MAR
Ind1: Read
Ind2: M(MAR) → MDR
      MDR → Ad(IR)
      检查标志位
      ├── 标志=1 → 再次进入Ind0
      └── 标志=0 → 进入执行周期
```

### 间址与取指的对比

| 对比项 | 取指 | 间址 |
|--------|------|------|
| 地址来源 | PC | IR(Ad) |
| 读出内容 | 指令 | 有效地址 |
| 数据去向 | IR | IR(Ad) |
| 节拍数 | 3 | 3 |
| 是否所有指令都有 | 是 | 否（仅间接寻址） |

## 直观理解

### 生活类比

间址就像**查电话号码的"转接"过程**：
- 你拿到一张纸条写着"请拨打总机转301"（形式地址）
- 你先打总机（第一次访存：形式地址→MAR→主存）
- 总机告诉你"301的号码是138xxxx"（第二次访存得到有效地址）
- 现在你知道了真正的电话号码（有效地址），可以拨打了（执行阶段）

### 记忆技巧

- **间址三步**：送形式地址（T0）→ 读主存取有效地址（T1）→ 有效地址写回IR(Ad)（T2）
- **间址口诀**：「Ad出-M-回Ad」（IR(Ad)出→MAR→主存→MDR→回IR(Ad)）

## 知识关联

- 间址微操作与取指微操作结构相似（都是"送地址→读主存→取数据"）
- 间址增加了一次访存，是间接寻址"慢"的原因
- 在微程序控制器中，间址是一个独立的微程序段

### 易错陷阱

1. **陷阱**：间址读出的是**地址**，不是操作数。操作数在执行阶段才能取到。
2. **陷阱**：间址后有效地址写回IR(Ad)，覆盖了原来的形式地址。执行阶段使用的是新写入的有效地址。
3. **陷阱**：不是所有指令都有间址周期。直接寻址、寄存器寻址等不需要。
