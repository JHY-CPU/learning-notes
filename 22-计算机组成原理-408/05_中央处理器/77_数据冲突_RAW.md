# 71_数据冲突_RAW

## 核心概念

- **RAW（Read After Write）**：写后读冲突，是最常见的数据冲突
- 后续指令需要读取前面指令尚未写回的寄存器值
- 解决方案：**暂停等待**、**数据旁路（Forwarding）**

## 原理分析

### 一、RAW 的产生条件

设流水线为 5 段：IF - ID - EX - MEM - WB

在**ID 阶段读寄存器**，在**WB 阶段写寄存器**。

若两条连续指令存在寄存器依赖：

```
I1: ADD  R1, R2, R3    ; WB 阶段写 R1
I2: SUB  R4, R1, R5    ; ID 阶段读 R1
```

时间线：
- I1 在时刻 1 进入 ID，时刻 5 完成 WB（写 R1）
- I2 在时刻 2 进入 ID（读 R1），但此时 R1 还未被写入
- I2 读到的是 R1 的**旧值**，产生错误

### 二、数据冲突的间隔分析

设 I1 写寄存器 $R_d$，I2 读寄存器 $R_d$：

- I1 在 **WB 的前半周期**写寄存器
- I2 在 **ID 的后半周期**读寄存器
- 若 WB 和 ID 在**同一时钟周期**，可实现"先写后读"（不冲突）

**不产生冲突的最小间隔**：

| 间隔 | 是否冲突 | 说明 |
|------|---------|------|
| 0 条（相邻） | **冲突** | I2 的 ID 和 I1 的 WB 差 3 个周期 |
| 1 条（隔 1 条） | **冲突** | I2 的 ID 和 I1 的 WB 差 2 个周期 |
| 2 条（隔 2 条） | **可能不冲突** | WB 前半写，ID 后半读，可同周期 |
| 3 条及以上 | **不冲突** | I1 已完成 WB |

### 三、RAW 的解决方法

#### 方法 1：暂停等待（Stall）

- 插入气泡，延迟后续指令的读取
- I1 和 I2 相邻时，暂停 2 个周期
- I1 和 I2 隔 1 条时，暂停 1 个周期

```
时间→    1    2    3    4    5    6    7
I1:     [IF] [ID] [EX][MEM][WB]
I2:          [IF] [--] [--] [ID] [EX][MEM][WB]
                               ↑
                         等 I1 WB 完成后再读 R1
```

#### 方法 2：数据旁路（Forwarding）

- 将 ALU 的计算结果**直接旁路**到下一条指令的 ALU 输入端
- 无需等待 WB 阶段
- 详见第 73 节

### 四、经典例题

**题**：分析以下指令序列中的 RAW 冲突。

```
I1: ADD  R1, R2, R3
I2: SUB  R4, R1, R5
I3: AND  R6, R1, R7
I4: OR   R8, R4, R9
```

**分析**：
- I1→I2：R1 的 RAW 冲突
- I1→I3：R1 的 RAW 冲突
- I2→I4：R4 的 RAW 冲突

若无旁路，I2 需暂停 2 周期，I3 需暂停 1 周期（距 I1 隔了 1 条），I4 需暂停 2 周期。

若有旁路（EX→EX）：I2 不需暂停（I1 EX 结果直接旁路给 I2 EX），I3 不需暂停，I4 不需暂停。

## 直观理解

**RAW** 像**接力赛中的交接棒**：

- A 还没跑到（写数据），B 就要起跑（读数据）
- B 只能等 A 跑到交接区（暂停），或者 A 提前把棒子扔给 B（旁路）

## 知识关联

- 与**数据旁路**（第 73 节）：旁路是解决 RAW 最有效的硬件方法
- 与**暂停机制**（第 77 节）：暂停是通用但效率低的方法
- 与**编译器调度**：编译器可以重排指令减少 RAW 冲突
- 408 高频考题：给定指令序列，判断需要暂停几个周期
