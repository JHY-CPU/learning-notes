# 86_异常与中断概念

## 核心概念

- **异常（Exception）**：由 CPU **内部**事件引起，又称**内中断**
- **中断（Interrupt）**：由 CPU **外部**事件引起，又称**外中断**
- 两者都是**改变程序正常执行流程**的机制

## 原理分析

### 一、异常 vs 中断

| 特性 | 异常（内中断） | 中断（外中断） |
|------|-------------|-------------|
| 来源 | CPU 内部 | CPU 外部 |
| 时机 | 与指令执行**同步** | 与指令执行**异步** |
| 可预测性 | 可预测（除终止类） | 不可预测 |
| 示例 | 溢出、缺页、除零 | IO 完成、时钟中断 |
| 也称 | Fault, Trap, Abort | INTR, NMI |

### 二、异常的分类

#### 1. 故障（Fault）

- 由**可恢复的错误**引起
- 处理后**返回**当前指令**重新执行**
- 例：缺页异常（Page Fault）

$$\text{处理完} \rightarrow \text{PC 回到当前指令} \rightarrow \text{重新执行}$$

#### 2. 陷阱（Trap）

- 由**有意的指令执行**引起
- 处理后返回**下一条指令**继续执行
- 例：系统调用（INT/syscall）、断点调试

$$\text{处理完} \rightarrow \text{PC 指向下一条指令} \rightarrow \text{继续执行}$$

#### 3. 终止（Abort）

- 由**不可恢复的严重错误**引起
- 程序**无法继续执行**
- 例：硬件故障、非法指令

$$\text{程序终止}$$

### 三、中断的分类

#### 1. 可屏蔽中断（INTR）

- 通过**中断标志位 IF** 可以屏蔽
- IO 设备请求、时钟中断等
- 可以被 CLI 指令屏蔽

#### 2. 不可屏蔽中断（NMI）

- **不能被屏蔽**，必须响应
- 电源掉电、硬件严重错误等
- 优先级最高

### 四、统一处理

无论异常还是中断，CPU 的处理流程相似：
1. 检测到异常/中断请求
2. 保存当前状态（PC、PSW）
3. 转入相应的处理程序
4. 处理完成后恢复现场

## 直观理解

- **异常**像**自己开车发现故障**（爆胎、没油）——你停车处理
- **中断**像**手机突然来电**——外部事件打断你
- **故障**：修好车继续走（回到刚才的路）
- **陷阱**：主动靠边停车问路（有意识的行为）
- **终止**：车报废了，叫拖车

## 知识关联

- 与**中断周期**（第 17 节）的关系：中断周期是处理中断的硬件阶段
- 与**中断响应过程**（第 90 节）直接相关
- 与**流水线**（第 62-85 节）的关系：流水线中处理异常需要**精确异常**
- 与**操作系统**（第 20 章）的关系：中断是 OS 的基础
- 与**存储管理**的关系：缺页异常与虚拟内存管理联系紧密
