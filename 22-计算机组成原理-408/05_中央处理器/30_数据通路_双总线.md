# 24_数据通路_双总线

## 核心概念

- **双总线数据通路**：CPU内部有两条内部总线，ALU的两个输入端分别连接不同的总线。
- **核心优势**：可以同时从两条总线各读取一个操作数，减少了操作数准备的时钟周期数。
- **典型结构**：总线A连接一组寄存器，总线B连接另一组寄存器，ALU的两个输入分别接总线A和总线B。

### 双总线结构示意

```
┌─────┐ ┌─────┐ ┌─────┐         ┌─────┐ ┌─────┐ ┌─────┐
│ PC  │ │ R0  │ │ R1  │         │ R2  │ │ ACC │ │ MDR │
└──┬──┘ └──┬──┘ └──┬──┘         └──┬──┘ └──┬──┘ └──┬──┘
   │       │       │               │       │       │
═══╪═══════╪═══════╪═══════════════╪═══════╪═══════╪══  总线A
   │       │       │               │       │       │
═══╪═══════╪═══════╪═══════════════╪═══════╪═══════╪══  总线B
   │       │       │               │       │       │
   └───────┴───┐   │               │   ┌───┴───────┘
               │   │               │   │
               ▼   ▼               ▼   ▼
          ┌─────────────────────────────────┐
          │        ALU输入选择(MUX)         │
          └────────────┬────────────────────┘
                       │
                  ┌────┴────┐
                  │   ALU   │
                  └────┬────┘
                       │
                  结果 → 总线A或B → 目标寄存器
```

## 原理分析

### 双总线的优势分析

#### 执行 A + B → C 的对比

**单总线**（需要3个时钟周期）：
1. A → 总线 → Y
2. B → 总线 → ALU，运算
3. 结果 → 总线 → C

**双总线**（需要2个时钟周期）：
1. A → 总线A → ALU输入1，同时 B → 总线B → ALU输入2，ALU运算
2. 结果 → 总线A/B → C

节省了1个时钟周期。

### 典型指令的微操作序列（双总线）

#### 取指

| 节拍 | 微操作 | 说明 |
|------|--------|------|
| T0 | PC → BUS_A → MAR | PC送上总线A，同时送MAR |
| T0 | PC + 1 → PC | PC自增可与此并行 |
| T1 | Read, M(MAR) → MDR | 读主存 |

共 **2个时钟周期**（比单总线少1个）。

#### 加法：ADD R1, R2 → R1

| 节拍 | 微操作 | 说明 |
|------|--------|------|
| T0 | R1 → BUS_A, R2 → BUS_B | 两操作数同时就位 |
| T1 | ALU_Add, 结果 → R1 | 运算并写回 |

执行阶段共 **2个时钟周期**。

#### 取数：LOAD R1, [addr]

| 节拍 | 微操作 | 说明 |
|------|--------|------|
| T0 | Ad(IR) → BUS_A → MAR | 送地址 |
| T1 | Read, M(MAR) → MDR | 读主存 |
| T2 | MDR → BUS_A → R1 | 数据送寄存器 |

执行阶段共 **3个时钟周期**。

### 总线仲裁

双总线结构中仍需总线仲裁：
- 每条总线同一时刻只能有一个数据源
- ALU输出需要选择送到哪条总线
- 需要MUX（多路选择器）控制数据来源

## 直观理解

### 生活类比

- **单总线**= 单车道，所有车排队过
- **双总线**= 双车道，两个方向可以同时通行
- **双总线的代价**= 需要更多的红绿灯（控制逻辑）和道路规划

### 记忆技巧

- **双总线比单总线快的核心**：两个操作数可以同时送上总线
- **双总线取指更快**：PC送上总线A的同时PC+1可以并行（因为PC有独立的自增逻辑）

## 知识关联

### 跨章节联系

- 与单总线和专用通路的对比是常考内容
- 数据通路设计直接影响CPI

### 常考组合

- 对比单总线和双总线执行同一指令的时钟周期数
- 分析双总线结构中的总线冲突

### 易错陷阱

1. **陷阱**：双总线不是"两倍速度"。ALU运算仍需要一个时钟周期，只是操作数准备更快。
2. **陷阱**：双总线中每条总线仍需仲裁，同一时刻只能有一个源。
3. **陷阱**：结果写回也需要占用一条总线，不能与读操作数同时进行（除非有专用的结果通路）。
