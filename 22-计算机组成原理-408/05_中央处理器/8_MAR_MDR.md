# 09_MAR_MDR

## 核心概念

- **MAR（Memory Address Register，存储器地址寄存器）**：存放CPU要访问的主存单元的地址。
- **MDR（Memory Data Register，存储器数据寄存器）**：存放CPU与主存之间交换的数据。
- **MAR连接地址总线**，**MDR连接数据总线**，两者是CPU与主存之间的接口。
- **位数含义**：MAR的位数决定可寻址空间大小，MDR的位数决定一次可读写的数据宽度。

### MAR与MDR的对比

| 对比项 | MAR | MDR |
|--------|-----|-----|
| 全称 | Memory Address Register | Memory Data Register |
| 存放内容 | 主存地址 | 主存数据 |
| 位数 | 地址总线宽度（决定寻址空间） | 数据总线宽度（通常=机器字长） |
| 连接总线 | 地址总线 | 数据总线 |
| 方向 | 单向（CPU → 主存） | 双向（CPU ↔ 主存） |
| 用户可见性 | 透明 | 透明 |

## 原理分析

### MAR的详细分析

#### 位数与存储容量

$$\text{可寻址存储单元数} = 2^{n} \quad (n = MAR的位数)$$

$$\text{存储容量} = 2^{n} \times MDR位数 \quad \text{(bit)}$$

**例题**：MAR为20位，MDR为16位，求存储容量。

$$2^{20} \times 16 \text{ bit} = 1048576 \times 16 \text{ bit} = 2^{20} \times 2 \text{ B} = 2 \text{ MB}$$

#### MAR的数据来源

| 阶段 | 数据来源 | 说明 |
|------|----------|------|
| 取指周期 | PC → MAR | PC的内容（指令地址）送MAR |
| 间址周期 | IR(Ad) → MAR | IR中的地址码送MAR |
| 执行周期 | 有效地址 → MAR | 经过寻址计算得到的地址 |
| 堆栈操作 | SP → MAR | 栈顶地址送MAR |
| I/O操作 | I/O端口地址 → MAR | I/O指令中的端口地址 |

### MDR的详细分析

#### MDR的数据流动方向

**读操作**（CPU从主存读取数据）：

$$\text{MAR} \to \text{地址总线} \to \text{主存} \xrightarrow{1 \to R(读命令)} \text{数据总线} \to \text{MDR}$$

**写操作**（CPU向主存写入数据）：

$$\text{MDR} \to \text{数据总线} \to \text{主存} \xleftarrow{\text{MAR} \to \text{地址总线} \to \text{主存地址}} \xrightarrow{1 \to W(写命令)}$$

#### 取指过程中的MAR/MDR协同

完整的取指微操作序列：

```
1. PC → MAR              （指令地址送地址寄存器）
2. MAR → 地址总线 → 主存  （地址送主存）
3. 1 → R（发出读命令）    （控制信号：读主存）
4. 主存 → 数据总线 → MDR  （指令数据读入数据寄存器）
5. MDR → IR              （指令送指令寄存器）
6. PC + 1 → PC           （PC指向下一条指令）
```

### MAR/MDR的位置问题

**这是一个408高频考点**：

| 观点 | MAR/MDR归属 | 适用场景 |
|------|------------|----------|
| 经典冯·诺依曼 | 属于主存（存储器接口） | 理论模型 |
| 现代计算机 | 属于CPU（芯片内部） | 实际芯片设计 |
| 408考试 | **以题目描述为准** | 考试原则 |

- 有的题目说"CPU内包含MAR和MDR"，则算CPU内部寄存器。
- 有的题目把MAR和MDR画在CPU外面（连接总线的接口处），则算存储器接口。
- **关键原则**：看题目图示和文字描述。

## 直观理解

### 生活类比

- **MAR**= 你去银行存取款时，在单子上填写的**账户号码**（告诉银行找哪个账户）
- **MDR**= 银行实际给你的**钱数**（取款时），或者你递给银行的**钱**（存款时）
- **地址总线**= 银行内部传递账户号的管道
- **数据总线**= 银行内部传递钞票的管道
- **读命令R**= "请帮我查这个账户的余额"（读取操作）
- **写命令W**= "请帮我往这个账户存入XX元"（写入操作）

### 记忆技巧

- **MAR = Memory Address = 找地方**（地址→定位）
- **MDR = Memory Data = 拿东西**（数据→内容）
- **MAR是"指路牌"，MDR是"快递箱"**
- MAR只出不进（CPU → 主存），MDR可进可出（双向）

### 常考组合

- MAR位数 + MDR位数 → 计算存储容量（高频计算题）
- 取指过程中MAR/MDR的数据流（大题常考）
- MAR/MDR归属问题（选择判断题）

### 易错陷阱

1. **陷阱**：MAR位数不一定等于MDR位数。MAR位数决定地址空间，MDR位数决定数据宽度。两者可以不同（例如MAR=20位，MDR=16位）。
2. **陷阱**：MAR位数与PC位数相同（都等于地址总线宽度），但MAR位数不等于机器字长（机器字长通常等于数据总线宽度）。
3. **陷阱**：MDR是双向的（读操作时接收主存数据，写操作时发送数据给主存），MAR是单向的（只向主存发送地址）。
