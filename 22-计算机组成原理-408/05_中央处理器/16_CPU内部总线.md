# 10_CPU内部总线

## 核心概念

- **CPU内部总线**：连接CPU内部各寄存器和功能部件（ALU等）的数据传输通道。
- **与系统总线的区别**：内部总线在CPU芯片内部；系统总线连接CPU、主存、I/O设备。
- **内部总线的三种典型结构**：单总线、双总线、三总线。
- **总线带宽**：并行传输的位数 = 机器字长。

### 三种内部总线结构对比

| 结构 | 总线数量 | 并行度 | 速度 | 硬件复杂度 | 典型应用 |
|------|----------|--------|------|-----------|----------|
| 单总线 | 1条 | 低 | 最慢 | 最简单 | 简单CPU |
| 双总线 | 2条 | 中 | 较快 | 中等 | 通用CPU |
| 三总线 | 3条 | 高 | 最快 | 最复杂 | 高性能CPU |

## 原理分析

### 单总线结构

所有寄存器和ALU共享**一条内部总线**。

```
┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐
│ R0  │ │ R1  │ │ ACC │ │ MQ  │ │ X   │
└──┬──┘ └──┬──┘ └──┬──┘ └──┬──┘ └──┬──┘
   │       │       │       │       │
═══╪═══════╪═══════╪═══════╪═══════╪═══  ← 单总线
   │       │       │       │       │
   └───────┴───────┼───────┴───────┘
                   │
              ┌────┴────┐
              │   ALU   │
              └─────────┘
```

**单总线的特点**：
- **优点**：结构简单，硬件成本低，控制简单。
- **缺点**：同一时刻只能有一个数据在总线上传输，两个操作数需要分两个时钟周期传送。
- **微操作**：执行 A + B 需要至少3个时钟周期：
  1. T0：A → 总线 → Y（ALU输入锁存器）
  2. T1：B → 总线 → ALU的另一个输入端
  3. T2：ALU运算 → 总线 → 目标寄存器

### 双总线结构

两条内部总线，ALU的两个输入分别连接不同的总线。

```
┌─────┐ ┌─────┐         ┌─────┐ ┌─────┐
│ R0  │ │ R1  │ ...     │ ACC │ │ MQ  │
└──┬──┘ └──┬──┘         └──┬──┘ └──┬──┘
   │       │               │       │
═══╪═══════╪═══════════════╪═══════╪═══  ← 总线A
   │       │               │       │
═══╪═══════╪═══════════════╪═══════╪═══  ← 总线B
   │       │               │       │
   └───────┴───┐     ┌─────┴───────┘
               │     │
          ┌────┴─────┴────┐
          │      ALU      │
          └───────────────┘
```

**双总线的特点**：
- **优点**：可以同时从两条总线各读取一个操作数（A和B同时就位），减少了操作数准备时间。
- **缺点**：比单总线复杂，仍需要一个时钟周期输出结果。
- **微操作**：执行 A + B 可在2个时钟周期完成：
  1. T0：A → 总线A → ALU输入1，同时 B → 总线B → ALU输入2
  2. T1：ALU运算 → 结果 → 总线A/B → 目标寄存器

### 三总线结构（专用通路）

三条或更多总线，部分路径为专用通路，避免总线冲突。

```
┌─────┐ ┌─────┐
│ R0  │ │ R1  │ ...          ┌─────┐
└──┬──┘ └──┬──┘              │ Y   │
   │       │                 └──┬──┘
═══╪═══════╪════════════    ┌───┴───┐
   │       │  总线A         │       │
   │       │           ┌────┤  ALU  ├────┐
═══╪═══════╪══════════ │    │       │    │
   │       │  总线B     │    └───────┘    │
   │       │           │                 │
   └───────┴───┐       │    ┌───────┐    │
               │       └────┤  结果  ├────┘
               │            │  总线  │
═══════════════╪════════════└───┬───┘  ← 专用通路
         总线C                  │
                          ┌────┴────┐
                          │ 目标寄存器│
                          └─────────┘
```

**三总线的特点**：
- **优点**：ALU两个输入和输出各有独立通路，可在一个时钟周期内完成运算并输出。
- **缺点**：硬件最复杂，成本最高。

### 总线控制方式

- **同步总线**：由时钟信号统一控制数据传输，操作在时钟边沿触发。
- **异步总线**：使用握手信号（请求/应答）控制数据传输，不依赖统一时钟。
- **半同步总线**：混合方式，以时钟为基本定时，附加握手信号。

## 直观理解

### 生活类比

- **单总线**= 单车道公路，所有车辆（数据）只能排成一队通过，效率最低
- **双总线**= 双车道公路，可以同时双向行驶，效率提升
- **三总线**= 三车道高速公路，各有专用通道，效率最高
- **总线宽度**= 车道的宽度，越宽一次通过的车越多（数据位数越多）

### 记忆技巧

- **总线数量↑ = 并行度↑ = 速度↑ = 成本↑**
- **单总线需要"排队"，双总线可以"同时取两个数"，三总线可以"取-算-写同时进行"**

### 常考组合

- 单总线结构中执行一条指令需要多少个时钟周期（微操作数）
- 双总线/三总线与单总线的执行时间对比
- 总线宽度与机器字长、数据总线宽度的关系

### 易错陷阱

1. **陷阱**：单总线结构中，一个时钟周期内总线上只有一个有效数据，因此读两个操作数需要两个时钟周期。
2. **陷阱**：内部总线（CPU内部）和系统总线（CPU外部）是不同层次的概念，不要混淆。
3. **陷阱**：ALU本身不是寄存器，不能连接总线。需要通过输入锁存器（如Y寄存器）接收总线数据。
