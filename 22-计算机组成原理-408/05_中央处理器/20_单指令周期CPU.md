# 20_单指令周期CPU

## 核心概念

- **单指令周期CPU（Single-Cycle CPU）**：所有指令的指令周期长度相同，都等于最复杂指令所需的时间。
- **时钟周期 = 最长指令的执行时间**：由最慢的指令（如间接寻址的存数指令）决定。
- **优点**：控制简单，每条指令一个时钟周期完成。
- **缺点**：简单指令浪费大量时间，CPU利用率低。

### 单指令周期CPU的时钟周期确定

$$T_{时钟} = \max(T_{指令1}, T_{指令2}, \ldots, T_{指令n})$$

假设某机有以下指令：

| 指令类型 | 执行时间 |
|----------|----------|
| 寄存器运算（ADD R1,R2） | 5ns |
| 直接寻址取数（LOAD [addr]） | 8ns |
| 间接寻址取数（LOAD [[addr]]） | 12ns |
| 无条件跳转（JMP） | 4ns |
| 存数（STORE） | 10ns |

则单指令周期CPU的时钟周期 $T_{时钟} = 12$ ns（取最长的）。

## 原理分析

### 单指令周期CPU的数据通路

```
┌─────────────────────────────────────────────────┐
│                                                 │
│  PC ──→ MAR ──→ 地址总线 ──→ 主存              │
│                                  │              │
│                                  ▼              │
│  IR ◄── MDR ◄── 数据总线 ◄── 主存数据          │
│   │                                             │
│   ├──→ OP ──→ CU ──→ 控制信号                  │
│   └──→ Ad ──→ MAR (取操作数)                   │
│                                                 │
│  寄存器组 ═══ 内部总线 ═══ ALU                  │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 性能分析

执行 $n$ 条指令（其中简单指令 $n_1$ 条，复杂指令 $n_2$ 条）：

$$T_{总} = n \times T_{时钟} = n \times \max(T_{指令})$$

**示例**：1000条指令，其中：
- 800条是ADD（需5ns）
- 200条是间接取数（需12ns）

单指令周期：$T = 1000 \times 12 = 12000$ ns

实际需要的时间：$800 \times 5 + 200 \times 12 = 4000 + 2400 = 6400$ ns

浪费的时间：$12000 - 6400 = 5600$ ns（浪费率约46.7%）

### 控制信号

单指令周期CPU中，控制信号在每个时钟周期内保持不变（因为每个指令一个周期完成）：

$$\text{控制信号} = f(\text{指令操作码})$$

不需要时序信号（节拍），因为不需要分步执行。

## 直观理解

### 生活类比

单指令周期CPU就像**统一考试时间**：
- 无论题目简单（选择题）还是复杂（论述题），每道题都给同样的答题时间
- 简单题做完后只能干等（时间浪费）
- 好处是规则简单，监考老师好管理

### 记忆技巧

- **单指令周期 = "一刀切"**：所有指令等最慢的那个
- **适用场景**：指令系统非常简单、所有指令复杂度相近的CPU

## 知识关联

### 跨章节联系

- 本节是理解CPU设计三种方式的基础概念

### 易错陷阱

1. **陷阱**：单指令周期CPU不是"一条指令一个时钟周期就很快"的意思。恰恰相反，因为时钟周期取最慢的指令时间，简单指令会浪费时间。
2. **陷阱**：单指令周期CPU不使用流水线技术。流水线要求指令分阶段重叠执行，与"一条指令一个周期"矛盾。
