# 94_中断屏蔽

## 核心概念

- **中断屏蔽**：禁止某些中断被 CPU 响应
- **IF 位**：控制可屏蔽中断 INTR 的全局开关
- **屏蔽字**：细粒度控制每个中断源的屏蔽

## 原理分析

### 一、IF 位（中断标志位）

- IF 位于**标志寄存器 PSW** 中
- $IF = 1$：允许响应 INTR（开中断）
- $IF = 0$：禁止响应 INTR（关中断）

| 指令 | 功能 |
|------|------|
| STI | $IF \leftarrow 1$（开中断） |
| CLI | $IF \leftarrow 0$（关中断） |

**注意**：IF 位对 NMI 和异常**无效**（不可屏蔽）。

### 二、中断屏蔽字

- **屏蔽字**：一个位向量，每位对应一个中断源
- 屏蔽位 = 1：允许该中断
- 屏蔽位 = 0：屏蔽该中断

$$\text{屏蔽字} = (m_{n-1}, m_{n-2}, \ldots, m_1, m_0)$$

**示例**（4 个中断源）：

| 屏蔽字 | INTR3 | INTR2 | INTR1 | INTR0 |
|--------|-------|-------|-------|-------|
| 1111   | 允许  | 允许  | 允许  | 允许  |
| 1110   | 允许  | 允许  | 允许  | 屏蔽  |
| 1000   | 允许  | 屏蔽  | 屏蔽  | 屏蔽  |
| 0000   | 屏蔽  | 屏蔽  | 屏蔽  | 屏蔽  |

### 三、屏蔽字的实现

- 8259A 中断控制器中有**中断屏蔽寄存器 IMR**
- 通过 OUT 指令设置 IMR

```
; 屏蔽 INTR0 和 INTR2，允许 INTR1 和 INTR3
MOV AL, 1010b    ; 屏蔽字
OUT 21h, AL      ; 写入 IMR
```

### 四、中断处理中的屏蔽

在中断服务程序中，通常需要：
1. 进入服务程序后，设置**当前屏蔽字**
   - 屏蔽同级和低优先级中断
   - 允许更高优先级中断（支持嵌套）
2. 退出服务程序前，**恢复原屏蔽字**

### 五、屏蔽 vs 关中断

| 操作 | 范围 | 控制对象 |
|------|------|---------|
| 关中断（CLI） | **全部** INTR | IF 位 |
| 屏蔽字 | **特定**中断源 | IMR 寄存器 |

- 关中断 = "全部不响应"
- 屏蔽字 = "选择性不响应"

## 直观理解

- **IF 位**像**手机静音键**：一按全部不响
- **屏蔽字**像**手机通知设置**：可以设置微信响、电话不响
- **关中断**：在处理重要事情时不想被打扰
- **屏蔽字**：只允许紧急电话响

## 知识关联

- 与**中断优先级**（第 93 节）的关系：屏蔽字常与优先级配合使用
- 与**中断嵌套**（第 95 节）的关系：通过调整屏蔽字控制嵌套
- 与**中断响应条件**（第 90 节）的关系：$IF = 1$ 是 INTR 响应的必要条件
- 与**PSW**（第 6 节）的关系：IF 位在 PSW 中
- 408 考点：关中断的时机、IF 位的作用
