# 53_微程序设计_执行微程序

## 核心概念

- **执行微程序**：实现各机器指令具体功能的微程序，每条指令有自己独立的执行微程序。
- **入口地址**：由操作码经微地址形成部件产生的微程序起始地址。
- **执行微程序是各指令的"个性化"部分**：取指和间址是公共的，执行是指令特有的。

### 各指令执行微程序入口地址示例

| 指令 | 操作码 | 执行微程序入口 | 功能 |
|------|--------|---------------|------|
| ADD | 00 | 010H | $R_d \leftarrow R_d + R_s$ |
| SUB | 01 | 020H | $R_d \leftarrow R_d - R_s$ |
| LOAD | 10 | 030H | $R_d \leftarrow M[addr]$ |
| STORE | 11 | 040H | $M[addr] \leftarrow R_s$ |

入口地址的形成方式：将操作码映射为CM地址。

### ADD $R_0, R_1$ 的执行微程序

| 微指令地址 | 微操作 | 说明 |
|-----------|--------|------|
| 010 | $R_0 \to Y$ | 目的操作数送暂存器Y |
| 011 | $(Y) + R_1 \to Z$ | ALU加法，结果送Z |
| 012 | $Z \to R_0$ | 结果写回$R_0$ |

### LOAD $R_0, [addr]$ 的执行微程序

| 微指令地址 | 微操作 | 说明 |
|-----------|--------|------|
| 030 | $Ad(IR) \to MAR$ | 地址送MAR |
| 031 | $M(MAR) \to MDR$ | 读存储器 |
| 032 | $MDR \to R_0$ | 数据送目的寄存器 |

### STORE $R_0, [addr]$ 的执行微程序

| 微指令地址 | 微操作 | 说明 |
|-----------|--------|------|
| 040 | $Ad(IR) \to MAR$ | 地址送MAR |
| 041 | $R_0 \to MDR$ | 源数据送MDR |
| 042 | $MDR \to M(MAR)$ | 写存储器 |

### JMP addr（无条件转移）的执行微程序

| 微指令地址 | 微操作 | 说明 |
|-----------|--------|------|
| 050 | $Ad(IR) \to PC$ | 目标地址送PC |

### 条件转移 BZ addr（零标志转移）的执行微程序

| 微指令地址 | 微操作 | 条件 | 说明 |
|-----------|--------|------|------|
| 060 | 若 $ZF=1$，$Ad(IR) \to PC$；否则空操作 | $ZF$ | 根据零标志决定是否跳转 |

## 原理分析

### 执行微程序的入口形成

取指微程序结束后，需要根据操作码形成执行微程序的入口地址。

**简单方法（直接映射）**：
$$\text{入口地址} = \text{操作码} \times \text{每段微程序占用的地址空间}$$

例如：操作码4位，每段占16个地址
- ADD (0000) → 000H + 0 = 000H（但通常跳过取指和间址区域）
- 实际上：ADD → 010H, SUB → 020H, LOAD → 030H

**查表法**：
用一个小的映射表（PROM）将操作码映射为入口地址：

| 操作码 | 入口地址 |
|--------|---------|
| 0000 | 010H |
| 0001 | 020H |
| 0010 | 030H |
| 0011 | 040H |

### 执行微程序结束后的流向

```
执行微程序最后一条微指令
        │
        ▼
  ┌───────────┐
  │ 检查中断  │
  └──┬─────┬──┘
     │     │
   有中断 无中断
     │     │
     ▼     ▼
  中断微  回到取指
  程序    微程序(000H)
  (INT入口)
```

执行完一条指令后，总是先检查有无中断请求。若有，进入中断微程序；若无，回到取指开始下一条指令。

### 指令周期与微程序的对应

```
机器指令执行过程：

取指周期          间址周期(可选)      执行周期           中断周期(可选)
┌─────────┐      ┌─────────┐      ┌──────────┐       ┌─────────┐
│取指微程序│  →  │间址微程序│  →  │执行微程序 │   →   │中断微程序│
│(000~002)│      │(003~005)│      │(010~...) │       │(070~...)│
└─────────┘      └─────────┘      └──────────┘       └─────────┘
```

## 直观理解

### 生活类比

执行微程序就像**不同菜品的烹饪方法**：
- 取指 = 从菜单选菜（所有菜共用一个点菜流程）
- 间址 = 查找食材位置（如果菜谱写了"用2号冰箱的材料"）
- 执行 = 按照特定菜谱做菜（宫保鸡丁的做法与红烧肉不同）
- 中断 = 有紧急订单时的处理流程

### 记忆技巧

- **执行微程序 = 指令的"个性"**：每条指令不同
- **入口由操作码决定**：操作码→微地址
- **执行完必检查中断**：每条指令执行完后都要检查

### 易错陷阱

1. **陷阱**：执行微程序的入口地址不是固定的，由操作码决定。不同指令入口不同。
2. **陷阱**：执行微程序结束后不是直接回到取指，而是先检查中断。这是408常考的细节。
3. **陷阱**：条件转移指令的执行微程序中包含条件判断，需要根据状态标志选择不同的微操作（类似微程序的分支）。
