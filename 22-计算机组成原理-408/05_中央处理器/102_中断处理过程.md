# 96_中断处理过程

## 核心概念

- 中断处理的**完整流程**：关中断 → 保存现场 → 中断服务 → 恢复现场 → 开中断 → 返回
- 每一步都至关重要，缺一不可
- 408 必考知识点

## 原理分析

### 一、中断处理的完整步骤

#### 第一阶段：中断响应（硬件自动，隐指令）

1. **关中断**：$IF \leftarrow 0$
2. **保存断点**：$SP \leftarrow SP - 2$，$Mem[SP] \leftarrow PC$
3. **保存 PSW**：$SP \leftarrow SP - 2$，$Mem[SP] \leftarrow PSW$
4. **识别中断源**，获取中断类型号
5. **查中断向量表**，$PC \leftarrow$ 服务程序入口

#### 第二阶段：中断服务（软件完成）

6. **保存通用寄存器**（保护现场）

```
PUSH R0
PUSH R1
PUSH R2
...
```

7. **开中断**：$IF \leftarrow 1$（允许更高优先级中断嵌套）

8. **执行中断服务程序**（核心处理逻辑）

9. **关中断**：$IF \leftarrow 0$（准备恢复现场）

10. **恢复通用寄存器**（恢复现场）

```
POP R2
POP R1
POP R0
...
```

#### 第三阶段：中断返回（软件+硬件）

11. **恢复 PSW**：$PSW \leftarrow Mem[SP]$，$SP \leftarrow SP + 2$
12. **恢复 PC**：$PC \leftarrow Mem[SP]$，$SP \leftarrow SP + 2$
13. **开中断**：$IF \leftarrow 1$
14. **返回被中断程序**继续执行

### 二、为什么要两次关中断？

| 时机 | 原因 |
|------|------|
| 第一次（响应时） | 防止保存断点/PSW 时被打断 |
| 中间（服务中） | 开中断允许嵌套 |
| 第二次（恢复前） | 防止恢复现场时被打断 |

### 三、保存内容总结

| 保存内容 | 谁保存 | 何时保存 |
|---------|--------|---------|
| PC（断点） | **硬件（隐指令）** | 中断响应时 |
| PSW | **硬件（隐指令）** | 中断响应时 |
| 通用寄存器 | **软件（服务程序）** | 进入服务程序后 |

### 四、完整流程图

```
中断请求 ──→ 响应?
              │ 是
              ↓
         关中断 (IF←0)
              ↓
         保存 PC
              ↓
         保存 PSW
              ↓
         取中断类型号
              ↓
         查向量表
              ↓
    PC ← 服务程序入口
              ↓
         保存通用寄存器
              ↓
         开中断 (IF←1)
              ↓
         执行服务程序
              ↓
         关中断 (IF←0)
              ↓
         恢复通用寄存器
              ↓
         恢复 PSW
              ↓
         恢复 PC
              ↓
         开中断 (IF←1)
              ↓
         返回被中断程序
```

## 直观理解

中断处理像**接电话的过程**：

1. 听到铃响，先挂"请勿打扰"牌（关中断）
2. 写便签记下刚才在做什么（保存 PC/PSW）
3. 查看是谁打来的（查向量表）
4. 放下便签，接电话（开中断，执行服务）
5. 打完电话，取下便签（恢复现场）
6. 继续之前的工作（返回）

## 知识关联

- 与**中断响应**（第 90 节）的关系：响应是处理的第一阶段
- 与**中断向量表**（第 92 节）的关系：查表获取入口地址
- 与**中断嵌套**（第 95 节）的关系：中间开中断允许嵌套
- 408 高频考点：保存内容、两次关中断原因、PC 保存值
