# 51_微程序设计_取指微程序

## 核心概念

- **公共取指微程序**：所有机器指令共享的一段微程序，完成从主存取出指令的操作。
- **入口地址**：取指微程序的第一条微指令在CM中的固定地址，通常为000H。
- **结束标志**：取指微程序执行完毕后，根据操作码转向各指令的执行微程序入口。

### 取指微程序的微操作序列

| 微指令地址 | 微操作 | 说明 |
|-----------|--------|------|
| 000 | $PC \to MAR$ | 程序计数器送地址寄存器 |
| 001 | $M(MAR) \to MDR, PC+1$ | 读存储器，PC自增 |
| 010 | $MDR \to IR$ | 指令送指令寄存器 |

以上是基本的三步取指。如果使用微指令编码，每步用一条微指令表示。

### 取指微程序在CM中的位置

```
CM地址空间：
┌────────────────────────────────────┐
│ 000H: 取指 μI1 (PC→MAR)           │ ← 公共取指微程序
│ 001H: 取指 μI2 (M(MAR)→MDR, PC+1) │   （所有指令共享）
│ 002H: 取指 μI3 (MDR→IR)           │
├────────────────────────────────────┤
│ 003H: 间址 μI1 (Ad(IR)→MAR)       │ ← 公共间址微程序
│ 004H: 间址 μI2 (M(MAR)→MDR)       │
│ 005H: 间址 μI3 (MDR→Ad(IR))       │
├────────────────────────────────────┤
│ 010H: ADD 执行 μI1                 │ ← 各指令执行微程序
│ 011H: ADD 执行 μI2                 │
│ ...                                │
└────────────────────────────────────┘
```

## 原理分析

### 取指微程序的控制信号

**000号微指令**（$PC \to MAR$）：

| 微命令 | 编码值 |
|--------|--------|
| $PC_{\text{out}}$ | 1 |
| $MAR_{\text{in}}$ | 1 |
| 其他 | 0 |

**001号微指令**（$M(MAR) \to MDR, PC+1$）：

| 微命令 | 编码值 |
|--------|--------|
| Read | 1 |
| $MDR_{\text{in}}$ | 1 |
| $PC+1$ | 1 |
| 其他 | 0 |

**002号微指令**（$MDR \to IR$）：

| 微命令 | 编码值 |
|--------|--------|
| $MDR_{\text{out}}$ | 1 |
| $IR_{\text{in}}$ | 1 |
| 其他 | 0 |

### 取指后的分支

取指微程序执行完毕后，需要根据 IR 中的操作码跳转到不同指令的执行微程序入口：

```
取指 μI3 (002H)
    │
    ▼
┌──────────┐
│ 操作码   │──→ ADD: 跳转到 010H
│ 译码     │──→ SUB: 跳转到 020H
│          │──→ LOAD: 跳转到 030H
│          │──→ STORE: 跳转到 040H
└──────────┘
```

这个分支由微指令的**判别测试字段**和**微地址形成部件**完成。

### 取指微程序的流程图

```
   ┌─────────┐
   │  开始   │
   └────┬────┘
        ▼
  ┌───────────┐
  │ PC → MAR  │  (μI at 000H)
  └─────┬─────┘
        ▼
  ┌───────────────┐
  │ M(MAR)→MDR    │  (μI at 001H)
  │ PC+1          │
  └─────┬─────────┘
        ▼
  ┌───────────┐
  │ MDR → IR  │  (μI at 002H)
  └─────┬─────┘
        ▼
  ┌───────────────┐
  │ 操作码译码    │  (形成执行微程序入口)
  │ 转执行微程序  │
  └───────────────┘
```

## 直观理解

### 生活类比

取指微程序就像**收银员扫码的标准流程**：
1. 拿起商品放到扫码区（PC→MAR）
2. 扫码读取价格（M(MAR)→MDR, PC+1）
3. 价格显示在屏幕上（MDR→IR）

无论顾客买什么商品，扫码的流程都一样——这就是"公共"的含义。

### 记忆技巧

- **三步取指**：「送址 → 读存 → 送指」（PC→MAR → M(MAR)→MDR → MDR→IR）
- **公共**：所有指令共享，不区分指令类型
- **入口固定**：取指微程序总是在CM的固定地址（通常000H）

1. **写出取指微程序的微指令序列**（综合题）
2. **计算取指微程序包含几条微指令**
3. **取指微程序执行完毕后如何分支**

### 易错陷阱

1. **陷阱**：取指微程序虽然"所有指令共享"，但取指完毕后的分支是不同的——根据操作码转到不同入口。
2. **陷阱**：$PC+1$ 和 $M(MAR) \to MDR$ 可以在同一条微指令中完成，因为它们互不干扰（一个是PC内部操作，一个是存储器操作）。
3. **陷阱**：有些系统中取指可能需要4步（如果存储器读需要2个时钟周期）。
