# 75_分支预测

## 核心概念

- **分支预测**：在转移指令结果确定之前，预测是否转移并提前取指令
- 分为**静态预测**和**动态预测**
- 目标：减少控制冲突带来的流水线暂停

## 原理分析

### 一、静态预测

#### 1. 预测不转移（Predict Not Taken）

- 假设转移**不发生**，继续顺序取指令
- 预测正确：无损失
- 预测错误：作废已取指令，转向转移目标

$$\text{损失} = \begin{cases} 0 & \text{预测正确} \\ j-1 & \text{预测错误} \end{cases}$$

- 循环中此预测效果差（循环几乎总转移回循环头）

#### 2. 预测转移（Predict Taken）

- 假设转移**发生**，直接取转移目标指令
- 需要提前计算转移目标地址
- 对条件转移效果不佳（不知道条件是否满足）

#### 3. BTFN（Back Taken, Forward Not Taken）

- **向后转移预测为转移**（循环回跳）
- **向前转移预测为不转移**（if 语句）
- 效果比纯预测不转移好

### 二、动态预测

根据**历史信息**调整预测。

#### 1. 1-bit 预测器

- 记录上次该转移指令**是否转移**
- 下次预测同上次

```
状态机：
    ┌───────────┐        ┌───────────┐
    │ Predict T │──不转─→│ Predict NT│
    │           │←──转移─│           │
    └───────────┘        └───────────┘
```

- 问题：循环最后一次会预测错误两次

#### 2. 2-bit 预测器（饱和计数器）

- 四种状态：强不转(00)、弱不转(01)、弱转移(10)、强转移(11)
- 连续两次预测错误才改变预测

```
状态机：
强不转(00) ←──不转── 弱不转(01)
    │转移              │转移
    ↓                  ↓
弱转移(10) ──不转──→ 强转移(11)
```

- 更稳定，避免偶尔的行为变化导致预测翻转

#### 3. 转移目标缓冲器（BTB, Branch Target Buffer）

- 缓存**转移指令的 PC** 和对应的**转移目标地址**
- 取指时查 BTB：命中则直接取转移目标

| PC（标记） | 转移目标 | 预测位 |
|-----------|---------|-------|
| 0x1000    | 0x2000  | 11    |
| 0x1008    | 0x1050  | 10    |
| ...       | ...     | ...   |

#### 4. 全局历史预测

- 考虑**之前多条转移指令**的历史
- 使用**分支历史寄存器（BHR）**记录最近 $n$ 次转移的结果
- 预测准确率更高

### 三、预测准确率的影响

设预测准确率为 $p$，转移比例为 $f$，预测错误损失为 $m$ 个周期：

$$CPI = 1 + f \times [(1-p) \times m]$$

| 预测准确率 | $f=20\%$, $m=2$ | CPI |
|-----------|-----------------|-----|
| 50%       | $0.2 \times 1 \times 2$ | 1.40 |
| 90%       | $0.2 \times 0.1 \times 2$ | 1.04 |
| 95%       | $0.2 \times 0.05 \times 2$ | 1.02 |

## 直观理解

**分支预测**像**天气预报**：

- **静态预测**：总是预报不下雨（预测不转移）
- **1-bit 动态预测**：昨天是否下雨来预测今天
- **2-bit 动态预测**：连续观察两天，更稳定
- **BTB**：记录每个路口上次怎么走的
- **全局历史**：参考之前所有路口的走法综合判断

## 代码/模拟

### Python模拟分支预测器

```python
"""分支预测器模拟 - 适用于408考研复习"""

class TwoBitPredictor:
    """2-bit饱和计数器分支预测器"""

    STATES = {
        0b00: "强不转(00)",
        0b01: "弱不转(01)",
        0b10: "弱转移(10)",
        0b11: "强转移(11)"
    }

    def __init__(self):
        self.state = 0b00  # 初始状态: 强不转

    def predict(self):
        """预测是否转移"""
        return self.state >= 0b10  # 状态>=10预测转移

    def update(self, actual_taken):
        """根据实际结果更新状态"""
        predicted = self.predict()
        correct = (predicted == actual_taken)

        if actual_taken:
            # 实际转移: 状态+1 (不超过11)
            self.state = min(self.state + 1, 0b11)
        else:
            # 实际不转: 状态-1 (不低于00)
            self.state = max(self.state - 1, 0b00)

        return correct

    def get_state_str(self):
        return self.STATES[self.state]

def simulate_branch_prediction(branch_history, name="2-bit预测器"):
    """模拟分支预测过程"""
    predictor = TwoBitPredictor()
    correct = 0
    total = len(branch_history)

    print(f"\n=== {name}模拟 ===")
    print(f"分支历史(T=转移, N=不转): {''.join('T' if t else 'N' for t in branch_history)}")
    print(f"{'步骤':<4} {'实际':<6} {'预测':<6} {'状态':<12} {'正确?':<6}")
    print("-" * 40)

    for i, taken in enumerate(branch_history):
        predicted = predictor.predict()
        was_correct = predictor.update(taken)
        if was_correct:
            correct += 1

        actual_str = "转移" if taken else "不转"
        pred_str = "转移" if predicted else "不转"
        print(f" {i+1:<3} {actual_str:<6} {pred_str:<6} "
              f"{predictor.get_state_str():<12} {'✓' if was_correct else '✗':<6}")

    print(f"\n预测准确率: {correct}/{total} = {correct/total:.1%}")
    return correct / total

# 循环示例: for(i=0; i<10; i++) 内部的分支
# 前9次循环条件为真(转移), 第10次为假(不转)
loop_branch = [True]*9 + [False]
simulate_branch_prediction(loop_branch, "循环分支(2-bit)")

# if-else模式
if_else = [True, False, True, True, False, False, True, False]
simulate_branch_prediction(if_else, "if-else模式(2-bit)")

# CPI影响分析
print("\n=== 分支预测对CPI的影响 ===")
print(f"{'准确率':<8} {'CPI (f=20%, m=2)':<20}")
print("-" * 30)
for p in [0.5, 0.7, 0.9, 0.95, 0.99]:
    cpi = 1 + 0.2 * (1 - p) * 2
    print(f"  {p:<6} {cpi:.3f}")
```
