# 25_数据通路_专用通路

## 核心概念

- **专用通路（Dedicated Path）**：为CPU内关键路径（如PC→MAR、ALU输出→寄存器）设计独立的专用数据通路，不经过共享总线。
- **核心优势**：避免总线冲突，关键操作可以并行执行，速度最快。
- **代价**：硬件最复杂，布线最多，成本最高。
- **典型应用**：高性能RISC处理器。

### 专用通路结构示意

```
     专用通路1: PC → MAR
     专用通路2: ALU输出 → 寄存器组
     专用通路3: MDR → IR

┌─────┐  专用  ┌─────┐
│ PC  │──────→│ MAR │──→ 地址总线
└──┬──┘  通路  └─────┘
   │
   │ (PC也可送内部总线)
   │
┌──┴──┐ ┌─────┐ ┌─────┐
│ R0  │ │ R1  │ │ R2  │  ...
└──┬──┘ └──┬──┘ └──┬──┘
   │       │       │
═══╪═══════╪═══════╪════  内部总线
   │       │       │
   └───────┴───┐   │
               ▼   ▼
          ┌─────────────┐
          │  Y     ALU  │────── 专用通路 ────→ 寄存器组
          └─────────────┘                        (结果写回)
                    │
                    │ 专用通路
                    ▼
              ┌──────────┐
              │  结果寄存器│
              └──────────┘
                    │
                    ▼
              内部总线 → 目标寄存器
```

## 原理分析

### 专用通路与共享总线的对比

| 对比项 | 单总线 | 双总线 | 专用通路 |
|--------|--------|--------|---------|
| 并行度 | 最低 | 中等 | 最高 |
| 时钟周期数 | 最多 | 较少 | 最少 |
| 硬件复杂度 | 最简单 | 中等 | 最复杂 |
| 布线数量 | 最少 | 中等 | 最多 |
| 典型应用 | 教学CPU | 通用CPU | 高性能CPU |

### 专用通路的设计原则

1. **高频使用路径专用化**：PC→MAR、ALU→寄存器是每条指令都用的路径，专用化收益最大。
2. **避免总线冲突**：专用通路不经过共享总线，不影响其他数据传输。
3. **并行执行**：PC→MAR的同时，内部总线可以传输其他数据。

### 执行时间分析

以 ADD R1, R2 → R1 为例：

**单总线**：3个时钟周期
1. R1 → 总线 → Y
2. R2 → 总线 → ALU → 结果
3. 结果 → 总线 → R1

**专用通路**：1个时钟周期（理想情况）
1. R1 → ALU输入A（专用通路），R2 → ALU输入B（专用通路），ALU运算 → 结果 → R1（专用通路）

所有操作在一个时钟周期内完成，不需要占用共享总线。

### 专用通路的代价

- **面积增大**：每条专用通路需要独立的导线和驱动电路。
- **MUX增多**：寄存器的输入可能来自多条通路，需要多路选择器。
- **功耗增大**：更多的导线意味着更多的电容负载。
- **设计复杂度**：需要仔细规划哪些路径需要专用化。

## 直观理解

### 生活类比

- **单总线**= 所有人共用一部电梯
- **双总线**= 两部电梯
- **专用通路**= 除了电梯，还有专用的楼梯、滑梯、传送带，各走各的
- **代价**= 修更多的通道需要更多的钱和空间

### 记忆技巧

- **专用通路 = VIP通道**：关键路径不排队
- **速度排序**：专用通路 > 双总线 > 单总线
- **成本排序**：专用通路 > 双总线 > 单总线

### 易错陷阱

1. **陷阱**：专用通路不是"不需要总线"，而是"关键路径不需要经过共享总线"。共享总线仍然存在。
2. **陷阱**：即使有专用通路，一个时钟周期内也只能完成一个ALU运算（ALU是共享资源）。
3. **陷阱**：专用通路的设计需要权衡——不是所有路径都值得专用化，只有高频使用的路径才值得。
