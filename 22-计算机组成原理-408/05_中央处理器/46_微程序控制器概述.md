# 40_微程序控制器概述

## 核心概念

- **微程序控制器（Microprogrammed Control Unit）**：将每条机器指令的控制信号序列编写为一段**微程序**，存放在**控制存储器（CM）**中，执行指令时逐条读出微指令来产生控制信号。
- **核心思想**：用软件方法（编写微程序）实现硬件控制功能。
- **提出者**：M.V. Wilkes，1951年首次提出微程序控制的概念。

### 微程序控制器的基本原理

```
机器指令 ──→ 微程序地址 ──→ 控制存储器CM ──→ 微指令 ──→ 控制信号
                ↑                                    │
                └──────── 下一条微指令地址 ←──────────┘
```

每条机器指令对应一段微程序，微程序由若干条微指令组成，每条微指令产生一个或多个控制信号（微命令）。

### 与硬布线的本质区别

| 对比项 | 硬布线 | 微程序 |
|--------|--------|--------|
| 控制信号来源 | 组合逻辑电路计算 | 控制存储器查表 |
| 修改方式 | 改电路 | 改微程序（软件） |
| 核心硬件 | 门电路、译码器 | 控制存储器（CM） |

## 原理分析

### 微程序控制器的工作流程

1. **取指**：执行公共的取指微程序，将指令从主存取到IR
2. **译码**：根据IR中的操作码，确定该指令对应的微程序入口地址
3. **执行**：从入口地址开始，逐条执行微指令，产生控制信号
4. **后继地址**：每条微指令指明下一条微指令的地址

### 微程序控制器的结构

```
              ┌─────────────┐
              │  控制存储器  │
              │    (CM)     │
              │  ┌───────┐  │
   μPC ──地址─→  │ 微指令 │  ──→ 控制字段 ──→ 控制信号
              │  └───────┘  │
              │     ↑       │
              └─────┼───────┘
                    │
              ┌─────┴──────┐
              │ 微地址形成  │
              │   部件      │
              └────────────┘
                 ↑    ↑
            操作码  下址字段
```

主要部件：
- **CM（Control Memory）**：存放所有微程序，通常用ROM或EPROM实现
- **μPC（Micro-Program Counter）**：微程序计数器，指向当前微指令
- **μIR（Micro-Instruction Register）**：微指令寄存器，存放当前微指令
- **微地址形成部件**：根据操作码或微指令的下址字段，形成下一条微指令地址

### CM的容量计算

$$\text{CM容量} = \text{微指令条数} \times \text{每条微指令的位数}$$

例：64条微指令，每条32位 → CM = 64 × 32 bit = 256字节

### Wilkes模型

Wilkes在1951年提出的原始模型：
- 用二极管矩阵实现控制存储器
- 每一行是一条微指令
- 每列对应一个控制信号
- 交叉点有二极管表示该微指令产生该控制信号

## 直观理解

### 生活类比

微程序控制器就像**一本菜谱**：
- **菜谱（CM）**：存放所有菜肴的做法
- **一道菜（微程序）**：一条机器指令对应的一组步骤
- **一个步骤（微指令）**：如"大火翻炒3分钟"
- **具体动作（微命令）**：如"开大火"、"拿锅铲"、"翻炒"

每道菜的步骤都写在菜谱上，厨师（CPU）按菜谱一步步执行。

### 记忆技巧

- **微程序 = 软件控制硬件**：用编写微程序（软件）的方法实现硬件控制
- **CM = 微程序的家**：所有微程序都存放在CM中
- **Wilkes 1951**：微程序控制的思想比集成电路还古老

### 易错陷阱

1. **陷阱**：CM（控制存储器）是CPU内部的部件，不是主存。主存存放机器指令和数据，CM存放微指令。
2. **陷阱**：微程序控制器虽然灵活，但速度比硬布线慢，因为每次都要访问CM。
3. **陷阱**：现代CPU通常混合使用两种方式——简单指令用硬布线，复杂指令用微程序。
