# 57_微程序_顺序控制

## 核心概念

- **顺序控制**：微程序中控制微指令执行顺序的方法，包括顺序执行、分支和转移。
- **断定法（下址字段法）**：每条微指令显式指定下一条微指令的地址。
- **计数器法（μPC+1法）**：用μPC计数器自动递增产生顺序地址。

### 两种基本顺序控制方式

| 方式 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **断定法** | 微指令中包含下址字段 | 灵活，可任意跳转 | 微指令更长 |
| **计数器法** | μPC自动+1，仅分支时修改 | 微指令较短 | 不够灵活 |

## 原理分析

### 断定法（下址字段法）

每条微指令都包含一个完整的下址字段：

```
┌────────────────────────────────────────┐
│  控制字段  │ 判别测试 │  下址字段      │
│            │          │ (显式指定下址) │
└────────────────────────────────────────┘
```

- 顺序执行：下址字段 = 当前地址 + 1
- 分支/转移：下址字段 = 目标地址

**优点**：
- 可以跳转到CM中任意位置
- 微程序之间可以任意跳转
- 适合非线性微程序

**缺点**：
- 下址字段增加了微指令长度
- 即使顺序执行也要指定下址

### 计数器法（μPC+1法）

利用μPC计数器自动产生顺序地址，仅在需要分支/转移时修改μPC：

```
┌──────────────────────────────┐
│  控制字段  │ 判别测试 │ 转移 │
│            │          │ 地址 │
└──────────────────────────────┘
                              │
正常: μPC+1                   │
分支: μPC ← 转移地址 ─────────┘
```

**优点**：
- 微指令较短（不需要完整的下址字段）
- 顺序执行时自动+1

**缺点**：
- 转移地址通常较小（相对转移）
- 不够灵活

### 混合控制

实际系统中常采用混合方式：
- 顺序执行用计数器法（μPC+1）
- 分支和转移用断定法（下址字段）

```
如果 (判别测试=00 且 无转移):
    μPC ← μPC + 1          // 顺序
否则如果 (判别测试≠00):
    μPC ← 条件拼接的地址    // 分支
否则:
    μPC ← 下址字段          // 转移
```

### 微程序流程中的顺序控制

```
         μPC=000 (取指μI1)
              │ (顺序: μPC+1)
              ▼
         μPC=001 (取指μI2)
              │ (顺序: μPC+1)
              ▼
         μPC=002 (取指μI3)
              │ (判别测试: 间址?)
              ├── IND=0 ──→ μPC=操作码入口 (分支)
              └── IND=1 ──→ μPC=003 (间址μI1)
                              │ (顺序: μPC+1)
                              ▼
                         μPC=004 (间址μI2)
                              │ (顺序: μPC+1)
                              ▼
                         μPC=005 (间址μI3)
                              │ (转执行: μPC=操作码入口)
                              ▼
                         μPC=010 (执行μI1)
                              │ ...
```

## 直观理解

### 生活类比

- **断定法** = 快递员每送一件包裹，包裹上都写着下一件的地址（完全灵活但信息量大）
- **计数器法** = 快递员按门牌号顺序送（1号、2号、3号...），只有特殊情况才跳号
- **混合控制** = 大部分按顺序送，但有些包裹标明了特殊目的地

### 记忆技巧

- **断定 = "指定"**：每条都指定下址
- **计数 = "自动"**：μPC自动递增
- **混合 = "取长补短"**：顺序用计数，分支用断定

2. **给定微程序流程，用断定法写出各微指令的下址字段**（综合题）
3. **计算两种方法所需的微指令长度**

### 易错陷阱

1. **陷阱**：断定法的下址字段位数取决于CM的总容量（微指令条数），不是微程序段数。
2. **陷阱**：计数器法虽然μPC+1自动顺序，但分支时需要修改μPC，这需要额外的控制逻辑。
3. **陷阱**：实际中纯断定法和纯计数器法都很少用，混合方式最常见。
