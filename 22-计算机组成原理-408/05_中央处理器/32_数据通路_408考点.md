# 26_数据通路_408考点

## 核心概念

数据通路是408考研的重点和难点，常见题型包括：
1. 给定数据通路结构，分析指令执行的微操作序列
2. 画出数据流图（数据在通路中的流动路径）
3. 计算指令执行的时钟周期数
4. 判断微操作的并行性

### 考试大纲要求

- 理解数据通路的基本组成和工作原理
- 能够分析单总线/双总线数据通路中指令的执行过程
- 能够写出微操作序列并计算时钟周期数

## 典型考题解析

### 题型1：分析微操作序列

**题目**：某机采用单总线结构的CPU，ALU的一个输入端有锁存器Y。写出执行"ADD R1, [2000]"（直接寻址，R1 + M(2000) → R1）的微操作序列。取指周期需要3个时钟周期。

**解答**：

**取指阶段**（3个周期）：
| 节拍 | 微操作 | 控制信号 |
|------|--------|---------|
| T0 | PC → MAR | PC_out, MAR_in |
| T1 | 1→R, M(MAR)→MDR | Read, MDR_in |
| T2 | MDR → IR, PC+1 | MDR_out, IR_in, PC_inc |

**执行阶段**（4个周期）：
| 节拍 | 微操作 | 控制信号 |
|------|--------|---------|
| T3 | Ad(IR) → MAR | IR(Ad)_out, MAR_in |
| T4 | 1→R, M(MAR)→MDR | Read, MDR_in |
| T5 | MDR → Y | MDR_out, Y_in |
| T6 | R1 → ALU, Y+R1 → R1 | R1_out, ALU_add, R1_in |

**总时钟周期数**：3 + 4 = 7

### 题型2：画数据流图

**题目**：画出执行"LOAD R1, [addr]"（直接寻址取数）时，数据在单总线数据通路中的流动路径。

**解答**：

```
取指阶段数据流：
PC ──→ 总线 ──→ MAR ──→ 地址总线 ──→ 主存
                                      │
PC + 1 ← PC                          ▼
                                主存数据
                                      │
                                      ▼
                    数据总线 ──→ MDR ──→ 总线 ──→ IR

执行阶段数据流：
IR(Ad) ──→ 总线 ──→ MAR ──→ 地址总线 ──→ 主存
                                           │
                                           ▼
                                     主存数据
                                           │
                                           ▼
                         数据总线 ──→ MDR ──→ 总线 ──→ R1
```

### 题型3：计算执行时间

**题目**：某单总线CPU，时钟频率为500MHz。取指需3个时钟周期，直接寻址执行需3个时钟周期，间接寻址执行需6个时钟周期。求执行一条"LOAD R1, [[100]]"（二次间址）所需时间。

**解答**：

该指令经历：取指 + 一次间址 + 二次间址 + 执行

假设一次间址需3个时钟周期，则：
$$CPI = 3(\text{取指}) + 3(\text{一次间址}) + 3(\text{二次间址}) + 3(\text{执行}) = 12$$

$$T = 12 \times \frac{1}{500 \times 10^6} = 12 \times 2 \times 10^{-9} = 24 \text{ ns}$$

### 题型4：判断并行性

**题目**：在双总线结构中，以下哪些微操作可以并行执行？
A. R1 → BUS_A 和 R2 → BUS_B
B. R1 → BUS_A 和 R1 → BUS_B
C. BUS_A → MAR 和 BUS_A → MDR
D. Read 和 Write

**解答**：

A. **可以**：不同总线，无冲突
B. **不可以**：R1同时输出到两条总线可能可行（看硬件设计），但更关键的是这是同一个源
C. **不可以**：MAR和MDR同时从BUS_A读取——同一总线同一时刻只有一个数据，但两个接收方同时接收同一数据是可以的（如果硬件支持广播）。严格来说取决于题目设计。
D. **不可以**：主存不能同时读和写

正确答案：**A**

### 题型5：数据通路对比

**题目**：比较单总线和双总线结构执行"ADD R1, R2"指令所需的时钟周期数（取指均需3个周期，执行阶段单总线需Y锁存器）。

**解答**：

| 阶段 | 单总线 | 双总线 |
|------|--------|--------|
| 取指 | 3 | 3 |
| 执行 | 2（R1→Y, R2+Y→R1） | 1（R1+R2→R1，同时送上两条总线） |
| 总计 | 5 | 4 |

## 直观理解

### 答题模板

对于"写出微操作序列"的题目，按以下框架作答：

```
1. 确定指令的寻址方式和操作类型
2. 写出取指阶段微操作（通常是固定的3步）
3. 如有间址周期，写出间址微操作
4. 写出执行阶段微操作
   - 取操作数
   - 运算/操作
   - 写回结果
5. 统计总时钟周期数
```

### 常见陷阱

| 陷阱 | 正确做法 |
|------|---------|
| 忘记取指阶段 | 任何指令都要先取指 |
| 单总线操作数不先存Y | 单总线需先将一个操作数锁存在Y中 |
| 忘记PC+1 | 取指后必须更新PC |
| 间址和执行混淆 | 间址取的是地址，执行取的才是数据 |
| 读写命令遗漏 | 每次访存都要发出Read或Write |

## 知识关联

- 本节是第5章的大题核心，与硬布线/微程序控制器设计直接关联
- 数据通路的微操作序列是控制信号产生的基础
