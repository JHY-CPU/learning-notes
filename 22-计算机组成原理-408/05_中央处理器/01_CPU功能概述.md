# 01_CPU功能概述

## 核心概念

- **CPU（Central Processing Unit，中央处理器）**：计算机的核心部件，由**运算器**和**控制器**两大部件组成，负责执行指令、处理数据。
- **冯·诺依曼结构**：CPU = 运算器 + 控制器 + 寄存器组 + 内部总线（现代教材的扩展定义）。
- **CPU的五大基本功能**：

| 功能 | 说明 | 典型操作 |
|------|------|----------|
| **指令控制** | 控制程序的执行顺序 | 按PC取指、顺序/跳转执行 |
| **操作控制** | 产生控制信号驱动各部件 | CU译码产生微命令 |
| **时间控制** | 对各操作定时，保证有序进行 | 时序系统产生节拍信号 |
| **数据加工** | 对数据进行算术/逻辑运算 | ALU执行加减乘除与逻辑运算 |
| **中断处理** | 响应外部/内部中断请求 | 保存现场→转向中断服务程序 |

### 考试重点

- CPU = 运算器 + 控制器 是最经典的定义，408考试首选此定义。
- 注意区分"CPU"与"处理器"：现代SoC中"处理器"可能包含多个CPU核心。
- CPU不包含主存储器，主存是独立于CPU的部件。
- **易混淆**：Cache属于存储层次结构，不属于CPU基本组成；但在某些芯片封装中Cache集成在CPU芯片内。

## 原理分析

### CPU在计算机系统中的位置

```
┌─────────────────────────────────┐
│            主存储器              │
│         (Memory)                │
└──────────┬──────────────────────┘
           │ 数据总线/地址总线/控制总线
┌──────────┴──────────────────────┐
│              CPU                │
│  ┌──────────┐  ┌────────────┐  │
│  │  运算器   │  │  控制器    │  │
│  │ (ALU等)  │  │  (CU等)    │  │
│  └──────────┘  └────────────┘  │
│  ┌──────────────────────────┐  │
│  │   寄存器组  │ 内部总线    │  │
│  └──────────────────────────┘  │
└─────────────────────────────────┘
```

### 五大功能的硬件支撑

1. **指令控制**：由**PC（程序计数器）**和**控制器**配合完成。PC指出下一条指令地址，控制器控制取指流程。
2. **操作控制**：由**CU（控制单元）**根据指令操作码产生微命令（控制信号），驱动各部件协同工作。
3. **时间控制**：由**时序系统**（节拍发生器）产生时钟信号，将指令周期划分为机器周期、时钟周期。
4. **数据加工**：由**ALU（算术逻辑单元）**完成，配合寄存器组提供操作数、存放结果。
5. **中断处理**：由**中断系统**完成，包括中断请求寄存器、中断判优逻辑、中断响应机制。

### 指令执行的宏观流程

$$\text{取指} \rightarrow \text{译码} \rightarrow \text{执行} \rightarrow \text{（写回）} \rightarrow \text{取下一条指令...}$$

每个步骤由CU发出的控制信号驱动，微操作在时钟节拍的同步下有序进行。

## 直观理解

### 生活类比

可以把CPU想象成一个**工厂的厂长**：
- **运算器（ALU）**= 车间工人，负责实际的加工（计算）工作
- **控制器（CU）**= 厂长本人，负责调度、指挥，告诉工人下一步做什么
- **寄存器**= 办公桌上的文件夹，临时存放当前工作的材料
- **PC**= 工作日程表，记录"下一步该做什么"
- **指令**= 工作任务单，上面写着"做什么"和"用什么材料"
- **时钟**= 上下班铃声，保证所有人同步工作

### 记忆技巧

- **五功能口诀**：「指操时加断」（指令控制、操作控制、时间控制、数据加工、中断处理）
- **CPU组成口诀**：「运控寄总」（运算器、控制器、寄存器组、内部总线）
- **理解关键**：CPU不"记住"数据（那是存储器的事），CPU只"处理"数据

## 知识关联

### 跨章节联系

| 章节 | 与本节的联系 |
|------|-------------|
| 数据的表示与运算 | ALU完成定点/浮点运算，运算方法详见第2章 |
| 存储系统 | CPU通过MAR/MDR与主存交互，Cache加速访存 |
| 指令系统 | 指令格式决定CPU如何译码和执行 |
| 总线 | CPU通过系统总线与I/O设备、主存通信 |
| I/O系统 | 中断机制连接CPU与外部设备 |

### 常考组合

- **CPU功能 + 寄存器功能**：选择题常见，问"完成XX功能需要哪些部件"
- **CPU结构 + 数据通路**：大题常考，画数据流图、写微操作序列
- **CPU五大功能 + 指令执行过程**：分析具体指令需要哪些功能配合

### 易错陷阱

1. **陷阱**：问"CPU包含哪些部件"时，不要把主存储器算进去。
2. **陷阱**：MAR/MDR有时归为主存（冯·诺依曼原始定义），有时归为CPU内部寄存器（现代设计），考试中以题目给定的架构为准。
3. **陷阱**：控制器的功能不只是"译码"，还包括产生时序信号、控制数据通路、处理中断等。
