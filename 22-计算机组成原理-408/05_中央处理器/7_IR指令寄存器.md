# 08_IR指令寄存器

## 核心概念

- **IR（Instruction Register，指令寄存器）**：存放CPU当前正在执行的指令。
- **IR对用户透明**：程序员不能通过指令直接读写IR的内容。
- **IR的两个字段**：操作码（OP）和地址码（Ad），分别送往不同部件。

### IR的功能分工

| 字段 | 去向 | 功能 |
|------|------|------|
| 操作码OP | 指令译码器 → CU | 确定指令类型（加法？存数？转移？） |
| 地址码Ad | MAR（取操作数时） | 提供操作数的地址信息 |

## 原理分析

### IR在指令执行流程中的作用

```
取指周期结束时：
  MDR ──→ IR
         ├──→ OP字段 ──→ 指令译码器 ──→ CU（产生控制信号的依据之一）
         └──→ Ad字段 ──→ MAR（取操作数地址）
                          │
                          ▼
                       主存（读取操作数）
```

### 取指后IR的内容变化

假设取到的指令为"ADD R1, [2000]"（将地址2000处的数据加到R1）：

```
IR内容：
┌──────────┬──────────────────────┐
│  OP=ADD  │    Ad=2000           │
└──────────┴──────────────────────┘
     │              │
     ▼              ▼
   CU译码        MAR ← 2000
  (产生ADD      (取操作数时
   控制信号)     送往主存)
```

### 指令译码过程

1. IR的OP字段送入**指令译码器**。
2. 译码器是一个组合逻辑电路，将n位操作码译码为 $2^n$ 根输出线（其中只有一根有效）。
3. 有效的输出线指示当前指令的类型。
4. CU根据译码结果、时序节拍、状态标志，产生控制信号序列。

**示例**（3位操作码，8种指令）：

| OP编码 | 译码输出 | 指令类型 |
|--------|----------|----------|
| 000 | 线0有效 | LOAD（取数） |
| 001 | 线1有效 | STORE（存数） |
| 010 | 线2有效 | ADD（加法） |
| 011 | 线3有效 | SUB（减法） |
| 100 | 线4有效 | JMP（跳转） |
| 101 | 线5有效 | AND（逻辑与） |
| 110 | 线6有效 | OR（逻辑或） |
| 111 | 线7有效 | NOT（逻辑非） |

### IR与PC的对比

| 对比项 | IR | PC |
|--------|----|----|
| 存放内容 | 当前正在执行的指令 | 下一条待取指令的地址 |
| 用户可见性 | 透明 | 可见 |
| 修改方式 | 取指时由MDR自动写入 | 取指后自增或被指令修改 |
| 内容去向 | OP→CU, Ad→MAR | →MAR（取指时） |
| 生命周期 | 一条指令执行期间 | 整个程序运行期间 |

## 直观理解

### 生活类比

- **IR**= 工人手中正在执行的"工单"，上面写着"做什么"和"用什么材料"
- **操作码**= 工单上的"任务名称"（如"组装"、"焊接"、"检测"）
- **地址码**= 工单上的"材料位置"（如"A货架第3层"）
- **译码**= 主管看懂工单内容后，决定调派哪些工人和设备
- **IR透明**= 工人不需要知道自己拿的工单编号是什么，只需要照做

### 记忆技巧

- **IR = "当前正在做的活"**，PC = "下一个要做的活的地址"
- IR的两个去向：**OP→CU（管什么），Ad→MAR（用什么）**
- IR的内容来自MDR，因为取指时指令从主存通过MDR进入CPU

- 取指微操作序列：MDR → IR 是关键步骤

- IR和PC在指令执行过程中的各自职责

### 易错陷阱

1. **陷阱**：IR存放的是指令的**二进制编码**，不是助记符。"ADD R1, R2"在IR中是类似"0001 0001 0010"的机器码。
2. **陷阱**：IR的内容在整个指令执行过程中**保持不变**，直到下一次取指操作覆盖它。
3. **陷阱**：IR的操作码字段不一定是固定的前几位——变长操作码（扩展操作码）中，OP和Ad的边界需要根据前缀判断。
