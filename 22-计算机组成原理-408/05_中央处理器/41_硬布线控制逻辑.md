# 35_硬布线控制逻辑

## 核心概念

- **控制信号方程**：每个控制信号 $C_i$ 都是操作码 $I$、时序信号 $T$ 和状态标志 $F$ 的逻辑函数。
- **硬布线控制逻辑**：将控制信号的布尔表达式用组合逻辑门电路实现。
- **本质**：用硬件直接"算"出每一步需要发出哪些控制信号。

### 控制信号的数学表达

$$C_i = f(I_0, I_1, \ldots, I_n, T_0, T_1, \ldots, T_m, F_0, F_1, \ldots, F_k)$$

其中：
- $I_j$：指令译码后的各指令标志信号（如 ADD、SUB、LOAD 等）
- $T_k$：时序节拍信号（如 $T_0, T_1, T_2, T_3$）
- $F_l$：状态标志信号（如 $ZF, CF, OF, \text{间址标志}IND$ 等）

### 真值表方法

对于每个控制信号 $C_i$：

1. 列出所有使 $C_i = 1$ 的（指令, 时序, 状态）组合
2. 将这些组合用逻辑"或"连接
3. 用卡诺图化简
4. 得到最简与或表达式

## 原理分析

### 控制信号产生的逻辑框架

```
                    ┌──────────┐
 指令操作码 ──译码──→│ 各指令    │──→ I_ADD, I_SUB, I_LOAD, I_STORE...
                    │ 标志信号  │          │
                    └──────────┘          │
                                          ▼
                    ┌──────────┐    ┌──────────┐
 时序系统 ──────────→│ T0,T1,T2 │──→│ 组合逻辑 │──→ C0, C1, C2, ...
                    └──────────┘    │  门电路  │
                    ┌──────────┐    │          │
 条件标志 ──────────→│ ZF,CF,IND│──→│          │
                    └──────────┘    └──────────┘
```

### 真值表构建示例

以控制信号 $C_{\text{PCout}}$（将 PC 内容送总线）为例：

| ADD | LOAD | STORE | JMP | $T_0$ | $T_1$ | $T_2$ | IND | $C_{\text{PCout}}$ |
|-----|------|-------|-----|-------|-------|-------|-----|-------------------|
| 1   | 0    | 0     | 0   | 1     | 0     | 0     | 0   | 1                 |
| 0   | 1    | 0     | 0   | 1     | 0     | 0     | 0   | 1                 |
| 0   | 0    | 1     | 0   | 1     | 0     | 0     | 0   | 1                 |
| 0   | 0    | 0     | 1   | 1     | 0     | 0     | 0   | 1                 |
| ... | ...  | ...   | ... | ...   | ...   | ...   | ... | ...               |

从表中可见，所有指令在 $T_0$ 时都需要 $PC_{\text{out}}$，因此：

$$C_{\text{PCout}} = T_0$$

### 化简的意义

- 如果不化简，控制逻辑需要大量门电路
- 化简后能减少门的数量和级数，降低信号传播延迟
- 卡诺图、Quine-McCluskey 等方法都可使用

### 逻辑电路实现

化简后的表达式直接用**与门、或门、非门**实现：

$$C_{\text{MARin}} = T_0 + (\text{ADD} \cdot T_2) + (\text{LOAD} \cdot T_1 \cdot \overline{IND})$$

对应的电路：
```
T0 ─────────────────────┐
                         │
ADD ──┬── T2 ── AND ────┼── OR ──→ C_MARin
      │                  │
LOAD ─┼── T1 ──┬─ AND ──┘
      │        │
      └── IND'─┘
```

## 直观理解

### 生活类比

控制信号逻辑就像**交通信号灯的控制电路**：
- 输入：时间（白天/夜间）、传感器（是否有车）、路口类型（主干道/支路）
- 输出：红灯、绿灯、黄灯信号
- 每个输出信号都是输入条件的逻辑组合
- 硬布线就是用固定的电路来实现这个逻辑

### 设计思路记忆

1. **先建表**：列出所有指令在所有时序下的微操作
2. **再找规律**：哪些控制信号在哪些条件下有效
3. **然后化简**：布尔代数化简表达式
4. **最后搭电路**：用门电路实现化简后的表达式

### 易错陷阱

1. **陷阱**：化简时要确认哪些输入变量是"无关项"（某些指令不可能与某些条件同时出现），合理利用无关项可大幅化简。
2. **陷阱**：控制信号是**组合逻辑**输出，没有记忆功能。时序信息由专门的时序电路（触发器、计数器）提供。
3. **陷阱**：同一条指令在不同机器周期中，同一控制信号的表达式可能不同，不能简单合并。
