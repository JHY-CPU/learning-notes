# 11_程序查询_流程

## 核心概念

程序查询方式的数据传送流程是408考试的重点，需要理解完整的操作序列和每个步骤的作用。

## 原理分析

### 完整的单设备查询流程

```
步骤1: CPU向接口发出启动命令
       ↓
步骤2: CPU读取状态寄存器
       ↓
步骤3: 判断设备是否就绪(READY=1?)
       ├─ 否 → 回到步骤2(循环等待)
       └─ 是 → 步骤4
       ↓
步骤4: CPU执行数据传送
       ├─ 输入: CPU从数据输入寄存器(DIR)读数据
       └─ 输出: CPU向数据输出寄存器(DOR)写数据
       ↓
步骤5: 判断数据是否传送完毕
       ├─ 否 → 回到步骤2(准备下一个数据)
       └─ 是 → 步骤6
       ↓
步骤6: IO操作结束
```

### 汇编语言级流程(伪代码)

```assembly
; 输入一个数据块到主存缓冲区
; CX = 要传送的数据个数
; DI = 主存缓冲区起始地址

READ_LOOP:
    IN AL, STATUS_PORT    ; 读状态端口(步骤2)
    TEST AL, 01h          ; 测试READY位(D0)
    JZ READ_LOOP          ; READY=0，继续等待(步骤3)

    IN AL, DATA_PORT      ; 读数据端口(DIR)(步骤4)
    MOV [DI], AL          ; 存入主存缓冲区
    INC DI                ; 缓冲区地址+1
    DEC CX                ; 计数器-1
    JNZ READ_LOOP         ; 未完，继续循环(步骤5)

    ; CX=0，传送完毕(步骤6)
```

### 每次数据传送的详细时间线

```
时间轴:
├── t_cmd ──┤── t_poll × N ──┤── t_xfer ──┤
│           │                │            │
│ 发命令    │ 等待(N次轮询)  │ 数据传送   │
│ 1条指令   │ N条读状态指令  │ 1条IN/OUT  │
```

- $t_{\text{cmd}}$：发出启动命令的时间(1条OUT指令)
- $t_{\text{poll}}$：单次查询时间(1条IN指令+1条条件判断)
- $N$：查询次数(取决于设备就绪速度)
- $t_{\text{xfer}}$：数据传送时间(1条IN或OUT指令)

**每个数据的CPU开销**：

$$T_{\text{per\_data}} = t_{\text{cmd}} + N \times t_{\text{poll}} + t_{\text{xfer}}$$

### 多设备查询流程

```
轮询顺序: 设备1 → 设备2 → 设备3 → 回到设备1

对每个设备:
  读该设备状态 → 就绪则传送一个数据 → 继续下一个设备
```

**问题**：查询频率 = $\frac{1}{N \times t_{\text{poll}}}$（N为设备数）
- 设备数越多，每个设备被查询的间隔越长
- 设备2必须等CPU查完设备1后才能被轮到

**例**：3个设备，每次查询耗时2μs
- 轮询一圈耗时 = $3 \times 2 = 6\mu s$
- 每个设备最大等待时间 = 6μs
- 如果某个设备数据周期为5μs → 可能丢失数据！

### 程序查询的两种变体

**变体1：独占式查询**
- CPU只查询一个设备，不执行其他任务
- 设备数据周期内CPU全部用于等待
- 最简单，但CPU利用率最低

**变体2：定时查询**
- CPU在空闲时查询设备
- 可以在查询间隙执行其他计算任务
- 需要合理安排查询时间间隔

## 直观理解

**单设备查询类比**：取干洗的衣服
1. 把衣服送到干洗店(发命令)
2. 每隔一段时间打电话问"好了吗"(读状态)
3. 好了就去取(数据传送)
4. 洗一批取一批，直到全部洗完

**多设备查询类比**：同时等三个外卖
- 先问商家1 → 再问商家2 → 再问商家3 → 回到商家1
- 哪家先做好先取哪家的
- 但每个商家被问的间隔变长了

## 知识关联

### 常考计算

**例题**：某系统用程序查询方式从磁盘读取数据。磁盘数据传输率为4MB/s，每次查询耗时100ns。求：
1. 读取一个512字节扇区需要查询多少次？
2. CPU用于查询的开销占总时间的比例？

**解答**：
1. 磁盘就绪时间 = $512B / 4MB/s = 512 / (4 \times 10^6) = 128\mu s$
   查询次数 = $128\mu s / 100ns = 1280$ 次
2. 传送时间 = $512 / (4 \times 10^6) = 128\mu s$（数据传输率决定）
   查询开销 = $1280 \times 100ns = 128\mu s$
   CPU开销比例 = $128\mu s / (128\mu s + 128\mu s) = 50\%$

### 易错陷阱

1. **查询次数与设备速度有关**：设备越慢，查询次数越多
2. **"每次传送CPU全程参与"**：即使数据由接口缓冲，CPU仍然需要发起每次读写
3. **查询不等于检测中断**：查询是CPU主动读端口，中断是CPU被动响应
