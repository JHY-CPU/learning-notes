# 44_字符显示器

## 核心概念

- **字符显示模式**：显示器以字符为单位显示文本(不是逐像素)
- **典型规格**：80列 × 25行(一屏2000个字符)
- **字符发生器(Character Generator)**：将ASCII码转换为字符点阵的硬件

## 原理分析

### 字符显示的工作原理

```
显示过程:
CPU将字符编码(ASCII)写入显存(字符缓冲区)
    ↓
显示控制器按顺序读取字符缓冲区
    ↓
每个ASCII码 → 字符发生器 → 点阵数据(5×7或8×8)
    ↓
点阵数据逐行输出 → 显示器显示字符
```

### 显存组织(字符模式)

**字符缓冲区**：存储每个字符的ASCII码
- 80×25 = 2000个字符
- 每个字符1字节(ASCII码) → 字符缓冲区 = 2000字节

**属性缓冲区**：存储每个字符的显示属性
- 颜色、亮度、闪烁等
- 每个字符1字节属性 → 属性缓冲区 = 2000字节

```
字符缓冲区:  属性缓冲区:
┌──────┐    ┌──────┐
│'H'=72│    │白色  │
│'e'=69│    │白色  │
│'l'=6C│    │白色  │
│'l'=6C│    │白色  │
│'o'=6F│    │白色  │
└──────┘    └──────┘
```

**总显存** = 字符缓冲区 + 属性缓冲区 = 2000 + 2000 = 4000字节

### 字符发生器

- **ROM芯片**：存储所有可显示字符的点阵图案
- **输入**：ASCII码(字符编码)
- **输出**：字符的点阵数据(每个像素的亮/灭)

```
ASCII码 'A' (0x41) → 字符发生器ROM → 点阵:
  01110    ← 第0行
  10001    ← 第1行
  10001    ← 第2行
  11111    ← 第3行
  10001    ← 第4行
  10001    ← 第5行
  10001    ← 第6行
```

- 典型点阵大小：5×7(早期)、7×9、8×8、8×16
- ROM大小 = 字符数 × 每字符点阵字节数
  - 例：256个字符 × 8行 × 1字节/行 = 2KB

### 光标(Cursor)

- 光标位置记录在**光标寄存器**中
- 光标寄存器值 = 当前光标所在位置的字符缓冲区偏移
- 按键时，光标位置+1(或换行时跳到下一行行首)

## 直观理解

**字符显示类比**：电子公告牌
- 显存(字符缓冲区) = 公告内容的文字表
- 字符发生器 = 字体库(告诉系统每个字长什么样)
- 显示控制器 = 按表格查字体、排版、显示
- 光标 = 当前编辑位置的标记

## 知识关联

### 显存容量计算

**例**：80×25字符模式，每个字符用1字节ASCII码+1字节属性。字符发生器256个字符，8×16点阵。求显存容量。

- 字符缓冲区 = $80 \times 25 \times 1 = 2000$ 字节
- 属性缓冲区 = $80 \times 25 \times 1 = 2000$ 字节
- 字符发生器ROM = $256 \times 16 = 4096$ 字节(每个字符16字节点阵)
- 总容量 = $2000 + 2000 + 4096 = 8096$ 字节

### 408常考

1. **字符模式显存容量**：字符缓冲区 + 属性缓冲区
2. **字符发生器是ROM**：内容固定，不可修改
3. **80×25字符 = 2000个字符**：这是经典规格

### 易错陷阱

1. **字符模式显存远小于图形模式**：字符只需编码，图形需要每个像素
2. **字符发生器不在显存中**：它是独立的ROM芯片
3. **属性缓冲区是额外的**：不要漏算
