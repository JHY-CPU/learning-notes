# 35_DMA与中断对比

## 核心概念

DMA和中断是两种最重要的IO控制方式，408考试中经常作为对比考点出现。

## 原理分析

### 全面对比表(必背)

| 对比维度 | 程序中断方式 | DMA方式 |
|----------|-------------|---------|
| **数据传送主体** | CPU(软件) | DMA控制器(硬件) |
| **数据通路** | 设备→CPU→主存 | 设备→主存(直接) |
| **响应时机** | 每传一个数据就中断一次 | 传完一个数据块才中断(后处理) |
| **CPU参与程度** | 每个数据都要CPU搬运 | 仅预处理和后处理 |
| **中断次数** | N次(N为数据个数) | 1次(后处理) |
| **CPU开销** | 每次中断开销约10~50μs | 预处理+后处理，共约30μs |
| **适合数据量** | 小数据量(几个~几十个字) | 大数据量(数百~数百万字) |
| **适合速度** | 低速~中速设备 | 高速设备 |
| **硬件复杂度** | 中等(中断控制器) | 高(DMA控制器) |
| **编程复杂度** | 中等(需写ISR) | 较高(需设置DMA参数) |
| **典型设备** | 键盘、鼠标、串口 | 磁盘、网卡、显卡 |

### CPU开销对比分析

设传送 $N$ 个数据，每个数据的中断开销为 $t_i$，DMA预处理为 $t_p$，后处理为 $t_{po}$。

**中断方式CPU开销**：

$$T_{\text{中断}} = N \times t_i$$

**DMA方式CPU开销**：

$$T_{\text{DMA}} = t_p + t_{po}$$

**当 $T_{\text{中断}} > T_{\text{DMA}}$ 时**：

$$N > \frac{t_p + t_{po}}{t_i}$$

一般 $t_i \approx 20\mu s$，$t_p + t_{po} \approx 30\mu s$

**临界点**：$N > 2$ 时DMA就更高效！实际上只要超过几个数据，DMA就占优。

### 选择中断还是DMA的判断依据

| 条件 | 推荐方式 | 原因 |
|------|----------|------|
| 数据量 < 几十个字 | 中断 | DMA预处理开销不值得 |
| 数据量 > 几十个字 | DMA | 中断开销太大 |
| 设备速度 < 10KB/s | 中断 | 数据到达慢，中断频率低 |
| 设备速度 > 1MB/s | DMA | 中断太频繁会压垮CPU |
| 实时性要求高 | 中断 | 可以立即响应每个数据 |
| 批量传输 | DMA | 一次传完整个块 |

## 直观理解

**中断 vs DMA类比**：

**中断方式** = 你自己搬砖，搬一块歇一次
- 每块砖都要你自己动手(CPU参与每个字)
- 搬一块休息一下(中断响应+返回)
- 砖少还行，多了累死

**DMA方式** = 请搬家公司
- 你只管吩咐(预处理)和验收(后处理)
- 搬家公司自己搬(DMA硬件传送)
- 砖越多越省事

## 知识关联

### 408典型题

**题1**：以下关于中断和DMA的叙述，正确的是？
- A. 中断方式数据不经过CPU → 错(经过CPU)
- B. DMA方式不需要中断机制 → 错(后处理需要中断)
- C. DMA适合高速批量传输 → 对
- D. 中断方式适合高速设备 → 错(不适合)

**题2**：传送1000个数据，中断开销20μs/次，DMA预处理10μs、后处理20μs。比较CPU开销。
- 中断：$1000 \times 20 = 20000\mu s$
- DMA：$10 + 20 = 30\mu s$
- DMA节省 $20000 / 30 = 667$ 倍

### 易错陷阱

1. **DMA也需要中断**：后处理阶段用中断通知CPU
2. **中断不是比DMA慢**：中断的响应速度可能更快(每个数据立即处理)
3. **中断方式CPU利用率低≠中断方式差**：在低速设备上中断完全够用
4. **DMA不是万能的**：小数据量场景DMA反而更慢(预处理开销)
