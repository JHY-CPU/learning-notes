# 04_IO端口编址

## 核心概念

- **IO端口(Port)**：IO接口中的可被CPU寻址的寄存器，CPU通过端口地址访问接口中的寄存器
- **端口类型**：
  - **数据端口**：数据输入寄存器(DIR)和数据输出寄存器(DOR)各占一个端口地址
  - **状态端口**：状态寄存器对应一个端口地址(只读)
  - **控制端口**：控制寄存器对应一个端口地址(只写)
- **端口编址**：为每个端口分配唯一的地址，CPU通过地址访问不同端口

### 端口 vs 接口 vs 设备

```
一台IO设备
  └── 一个IO接口(适配器)
        └── 多个IO端口(寄存器)
              ├── 数据端口(地址: N)
              ├── 状态端口(地址: N+1)
              └── 控制端口(地址: N+2)
```

- **一个接口**包含**多个端口**，每个端口有独立的地址
- **一个设备**通常对应**一个接口**（也有例外，如一个USB控制器管理多个设备）

### IO端口编址空间

- 每个端口地址对应一个寄存器，CPU通过不同的地址访问不同的端口
- 端口地址的位数决定了可寻址端口的数量
  - 8位地址：$2^8 = 256$ 个端口
  - 16位地址：$2^{16} = 65536$ 个端口

## 原理分析

### 端口地址分配原则

- **数据端口**和**控制/状态端口**分开：数据端口可读可写，状态端口只读，控制端口只写
- 通过读/写信号区分：
  - 对同一地址，读操作 → 访问数据输入寄存器或状态寄存器
  - 对同一地址，写操作 → 访问数据输出寄存器或控制寄存器
- 典型的8255可编程接口芯片：
  - 端口A数据：地址0（读=读A口数据，写=写A口数据）
  - 端口B数据：地址1
  - 端口C数据：地址2
  - 控制寄存器：地址3

### CPU访问端口的操作序列

```
输入操作(IN指令，从设备读数据):
1. CPU将端口地址 → 地址总线
2. CPU发出 IOR(IO Read)信号
3. 接口译码后将端口数据 → 数据总线
4. CPU从数据总线读取数据

输出操作(OUT指令，向设备写数据):
1. CPU将端口地址 → 地址总线
2. CPU将数据 → 数据总线
3. CPU发出 IOW(IO Write)信号
4. 接口将数据总线上的数据锁存到端口寄存器
```

### 端口地址译码电路

```
地址总线 A15~A0
        ↓
    地址译码器
        ↓
    选中信号 CS
        ↓
    IO接口响应读/写操作
```

- 高位地址用于选择接口(片选)
- 低位地址用于选择接口内部的某个端口(片内寻址)
- 例：A15~A2 用于片选，A1A0 用于选择4个端口之一

## 直观理解

**端口编址就像图书馆的索书号**：
- 图书馆 = IO地址空间
- 每个书架 = 一个IO接口
- 每本书的位置 = 一个端口地址
- 图书管理员(CPU)根据索书号(端口地址)找到特定的书(端口寄存器)

**端口类型类比**：
- 数据端口 = 传送带(货物进出)
- 状态端口 = 信号灯(告诉CPU当前状态)
- 控制端口 = 开关面板(CPU下达指令)

### 易错陷阱

1. **一个接口有多个端口地址**：不要将接口和端口混淆
2. **端口地址位数不等于存储器地址位数**：独立编址中IO地址空间远小于存储器地址空间
3. **读写操作可能访问不同寄存器**：同一端口地址，读和写可能对应不同寄存器
