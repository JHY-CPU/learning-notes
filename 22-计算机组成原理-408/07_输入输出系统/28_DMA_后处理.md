# 28_DMA_后处理

## 核心概念

**DMA后处理**：DMA数据传送完毕后，通过中断通知CPU，CPU进行收尾工作的过程。

## 原理分析

### 后处理流程

```
DMA控制器:                        CPU:
  │                                │
  │ DC=0(所有数据传送完毕)         │
  │                                │
  │ 置"传送完成"状态位             │
  │                                │
  │ 发中断请求(INTR) ──────────→ │
  │                                │ ← CPU响应中断
  │                                │ 关中断
  │                                │ 保存PC/PSW
  │                                │ 转入中断服务程序
  │                                │
  │                                │ 保护现场
  │                                │ 开中断
  │                                │
  │                                │ 读DMA状态寄存器
  │                                │  ← 检查是否有错误
  │                                │
  │                                │ 处理错误(如有)
  │                                │ 清除DMA状态
  │                                │ 释放DMA控制器
  │                                │
  │                                │ 关中断
  │                                │ 恢复现场
  │                                │ IRET
```

### 后处理中CPU需要做的事

1. **检查传送状态**：
   - 读DMA状态寄存器
   - 判断传送是否成功完成
   - 如果有错误标志，进行错误处理

2. **错误处理**：
   - CRC校验错误 → 重传
   - 超时错误 → 记录日志
   - 数据完整性检查

3. **释放资源**：
   - 清除DMA控制寄存器
   - 释放DMA控制器(供其他IO使用)
   - 通知操作系统IO完成

4. **唤醒等待进程**：
   - 将等待此IO的进程从阻塞队列移到就绪队列
   - 进程可以继续使用已传送好的数据

### 后处理的时间开销

后处理本质上是一次中断处理：
- 中断响应：几个时钟周期
- 保护现场：若干PUSH指令
- 检查状态：1~2个IN指令
- 恢复现场+返回：若干POP + IRET
- **总计**：约几十到几百个时钟周期(微秒级)

## 直观理解

**后处理类比**：验收快递
- 快递员(DMA)通知你"货到了"(中断请求)
- 你下楼验收(CPU响应中断)
- 检查包裹是否完好(读状态寄存器)
- 有破损找快递公司索赔(错误处理)
- 签收(清除状态)、回去继续工作(IRET)

## 知识关联

### 与中断方式的对比

| 对比项 | DMA后处理 | 程序中断方式 |
|--------|----------|-------------|
| 中断次数 | 一个数据块完成后1次 | 每个数据1次 |
| 中断开销 | 一次中断处理 | N次中断处理 |
| CPU参与 | 仅检查状态+错误处理 | 每个数据的传送 |

### 易错陷阱

1. **后处理只有一次中断**：整个DMA传送过程只中断一次(不是每传一个字中断)
2. **中断发生在DC=0时**：不是传完第一个字就中断
3. **后处理不涉及数据传送**：数据已经在传送阶段全部到位
4. **DMA中断优先级通常较高**：因为可能有时间敏感的后续操作
