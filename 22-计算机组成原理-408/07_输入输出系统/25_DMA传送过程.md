# 25_DMA传送过程

## 核心概念

DMA方式的数据传送分为三个阶段：**预处理、数据传送、后处理**。CPU只参与预处理和后处理，数据传送由DMA控制器独立完成。

## 原理分析

### 三阶段完整流程

```
CPU(软件)                    DMA控制器(硬件)               IO设备
   │                              │                         │
   │ 1.预处理:                    │                         │
   │  设置传送方向                 │                         │
   │  设置主存起始地址             │                         │
   │  设置传送字数                 │                         │
   │  启动DMA ──────────────────→│                         │
   │                              │                         │
   │ (CPU去做其他任务)             │ 2.数据传送:              │
   │                              │ ←──── 发总线请求HRQ ───→│
   │ ←─── CPU让出总线HLDA ────── │                         │
   │                              │ 取得总线控制权            │
   │                              │ 送地址+发读写命令         │
   │                              │ ←──── 数据 ───────────→ │
   │                              │ (重复N次)                │
   │                              │ DC=0 → 传送完成          │
   │                              │                         │
   │ ←──── 3.后处理:中断请求 ─── │                         │
   │ 响应中断                     │                         │
   │ 检查状态/处理错误            │                         │
   │ IO操作结束                   │                         │
```

### 各阶段详细说明

#### 阶段1：预处理(Preprocessing)

CPU在启动DMA之前，需要设置以下参数：

| 参数 | 写入位置 | 说明 |
|------|----------|------|
| 传送方向 | DMA控制寄存器 | 读(设备→主存)或写(主存→设备) |
| 主存起始地址 | DMA地址寄存器AR | 数据在主存中的起始位置 |
| 传送字数 | DMA数据计数器DC | 要传送的数据总个数 |
| 设备地址 | DMA设备选择逻辑 | 选择哪个IO设备 |
| 传送模式 | DMA控制寄存器 | 单字/成组传送 |
| 启动命令 | DMA控制寄存器 | GO位，启动DMA传送 |

**预处理完成后**：CPU发送"启动DMA"命令，之后DMA控制器接管数据传送。

#### 阶段2：数据传送(Data Transfer)

每个数据的传送过程：
1. IO设备就绪，向DMA发DREQ(DMA请求)
2. DMA向CPU发HRQ(总线请求)
3. CPU让出总线，发HLDA(总线响应)
4. DMA取得总线控制权
5. DMA将AR中的地址放到地址总线
6. DMA发读/写命令
7. 数据在数据总线上传送(设备↔主存)
8. AR+1(或+N)，DC-1
9. DMA释放总线(单字模式)或继续(成组模式)
10. 重复1~9，直到DC=0

#### 阶段3：后处理(Postprocessing)

当DC=0时(所有数据传送完毕)：
1. DMA向CPU发中断请求
2. CPU响应中断
3. CPU检查DMA状态寄存器(有无错误)
4. 处理可能的错误
5. IO操作正式结束

## 直观理解

**三阶段类比**：公司项目外包
- **预处理**：你签合同，约定交货地址、数量、方向(设置参数)
- **数据传送**：外包团队自己执行，你去做其他事(CPU不参与)
- **后处理**：外包团队通知你"活干完了"(中断)，你验收检查(检查状态)

## 知识关联

### 408常考要点

1. **预处理由CPU完成**：是软件操作
2. **数据传送由DMA硬件完成**：不需要CPU干预
3. **后处理由CPU通过中断完成**：DMA发中断通知CPU
4. **AR和DC的更新**：每传一字，AR+1，DC-1(DMA硬件自动)

### 易错陷阱

1. **预处理不是"预先传送数据"**：只是设置参数
2. **后处理不是"再传送一遍数据"**：只是检查和结束
3. **数据传送阶段CPU不一定完全空闲**：取决于传送方式(停止CPU/周期挪用/交替)
4. **DC到0后DMA才发中断**：不是每传一个字都发中断(与中断方式的关键区别)
