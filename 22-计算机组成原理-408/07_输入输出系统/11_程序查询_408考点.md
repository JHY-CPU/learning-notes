# 12_程序查询_408考点

## 核心概念

程序查询方式在408中主要考查特征判断和CPU利用率计算，常以选择题形式出现。

## 原理分析

### 考点一：程序查询方式特点总结(必背)

| 特点 | 详细说明 |
|------|----------|
| 信息传送 | 以CPU为中心，数据经过CPU寄存器中转 |
| CPU角色 | 全程参与，从发命令到数据传送都需要CPU |
| 数据单位 | 每次传送一个数据(字节或字) |
| CPU利用率 | 极低，大量时间用于空转等待 |
| 硬件需求 | 最低，只需简单状态寄存器 |
| 并行性 | 无并行性，CPU和设备串行工作 |
| 适用场景 | 低速设备、简单嵌入式系统 |
| 编程复杂度 | 最简单 |

### 考点二：CPU利用率计算

**典型题型**：给定设备数据周期和CPU操作开销，求CPU利用率。

**公式推导**：

设每传送一个数据：
- 设备数据准备时间(周期)：$T_{\text{dev}}$
- CPU查询时间(每次)：$t_p$
- CPU数据传送时间：$t_t$
- 查询次数：$N = \lceil T_{\text{dev}} / t_p \rceil$

**CPU总开销**（每个数据）：

$$T_{\text{cpu}} = N \times t_p + t_t$$

**CPU利用率**（指CPU在IO操作上的有效工作比例）：

$$\eta = \frac{t_t}{N \times t_p + t_t}$$

若忽略传送时间，CPU几乎100%时间都在做无用功。

### 考点三：典型例题

**例1**：某设备每20ms输出一个字节数据，CPU每次查询需0.5μs，传送一个字节需1μs。求CPU利用率。

**解答**：
- 设备周期 = 20ms = 20000μs
- 每次查询 = 0.5μs
- 理想查询次数(刚好就绪)：$20000 / 0.5 = 40000$ 次
- CPU查询总时间：$40000 \times 0.5 = 20000\mu s = 20ms$
- 传送时间 = 1μs
- CPU利用率 = $\frac{1}{20000 + 1} \approx 0.005\%$

**例2**：某系统有4台设备，使用程序查询方式。每次查询需2μs。要求任何设备的最大等待时间不超过100μs，求系统能否满足要求？

**解答**：
- 轮询一圈 = $4 \times 2 = 8\mu s$
- 每个设备最大等待时间 = 8μs < 100μs
- 能满足要求

**例3**：某程序查询IO系统中，CPU时钟频率为1GHz，查询一条指令需3个时钟周期，数据传送需5个时钟周期。设备每10μs准备好一个数据。求：
(1) CPU每秒查询次数
(2) CPU利用率

**解答**：
- 时钟周期 = $1 / 1GHz = 1ns$
- 每次查询 = $3 \times 1ns = 3ns$
- 数据传送 = $5 \times 1ns = 5ns$
- 设备周期 = 10μs = 10000ns
- 查询次数/数据 = $10000 / 3 = 3334$ 次
- (1) 每秒查询次数 = $1 / 3ns = 333,333,333$ 次
- (2) CPU利用率(用于IO) = $(3334 \times 3 + 5) / 10000 \approx 100\%$
  - 注意：这表示CPU几乎全部时间都花在IO上，无法做其他工作

## 直观理解

**408常见选项判断**：
- "程序查询方式硬件最简单" → 对
- "程序查询方式CPU利用率最高" → 错(最低)
- "程序查询方式可以实现CPU与设备并行工作" → 错(串行)
- "程序查询方式适合高速设备" → 错(适合低速)

## 知识关联

### 与其他IO方式对比(高频考点)

| 对比项 | 程序查询 | 程序中断 | DMA |
|--------|----------|----------|-----|
| CPU参与程度 | 全程参与 | 仅中断时参与 | 仅预/后处理 |
| 数据传送 | 经CPU中转 | 经CPU中转 | 设备直接到主存 |
| CPU利用率 | 最低 | 中等 | 最高 |
| 硬件复杂度 | 最低 | 中等 | 最高 |
| 适用速度 | 低速 | 低速中速 | 高速 |

### 易错陷阱

1. **CPU利用率100%不等于高效**：如果CPU全部时间都在查询，反而说明IO拖累了CPU
2. **查询时间包括读端口+条件判断**：不仅是IN指令，还包括CMP/JMP等
3. **多设备轮询时每个设备的查询频率降低**：$f_{\text{poll}} = \frac{1}{N \times t_{\text{poll}}}$
4. **程序查询不产生中断**：不要混淆查询和中断的响应方式
