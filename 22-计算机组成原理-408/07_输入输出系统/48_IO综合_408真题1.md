# 49_IO综合_408真题1

## 核心概念

本节汇总IO控制方式综合真题，涵盖程序查询、中断、DMA的对比分析。

## 原理分析

### 真题1：IO控制方式综合判断

**题目**：以下关于IO控制方式的叙述中，正确的是（ ）。
A. 程序查询方式中CPU与外设可以并行工作
B. 程序中断方式中CPU与外设可以并行工作
C. DMA方式中CPU与外设可以并行工作
D. 以上三种方式CPU与外设都可以并行工作

**答案**：B、C

**解析**：
- 程序查询：CPU全程等待，不能并行 → A错
- 程序中断：CPU可以做其他事，设备准备好后中断 → B对
- DMA：CPU可以做其他事，DMA独立传送 → C对
- D说"都可以"→ 错

### 真题2：中断方式数据通路

**题目**：在程序中断方式中，IO设备与主机之间的数据传送（ ）。
A. 不经过CPU，直接传送到主存
B. 经过CPU的寄存器中转
C. 由DMA控制器直接完成
D. 由通道控制器完成

**答案**：B

**解析**：
- 中断方式的数据由CPU执行IN/OUT指令传送
- 数据路径：设备→CPU寄存器→主存
- A是DMA方式的特征
- C、D分别是DMA和通道的特征

### 真题3：DMA与中断的本质区别

**题目**：DMA方式与中断方式的本质区别是（ ）。
A. DMA使用硬件，中断使用软件
B. DMA响应速度快，中断响应慢
C. DMA方式的数据传送不经过CPU
D. DMA优先级高于中断

**答案**：C

**解析**：
- A不准确：中断也有硬件(中断控制器)
- B不准确：中断响应速度可能更快
- **C是本质区别**：DMA直接在设备和主存之间传送，不经过CPU；中断方式数据经过CPU寄存器
- D是特点，不是本质区别

### 真题4：IO系统各方式适用场景

**题目**：某系统需要从高速磁盘连续读取大量数据到主存。最合适的IO控制方式是（ ）。
A. 程序查询方式
B. 程序中断方式
C. DMA方式
D. 通道方式

**答案**：C

**解析**：
- 高速+大量数据 → 排除A(查询方式CPU利用率太低)
- 高速连续传输 → 排除B(中断开销太大)
- **DMA最适合高速批量传输** → C对
- D(通道)也可以，但一般中小型系统用DMA更常见
- 如果是大型机管理多个高速设备，D也对

### 真题5：周期挪用对CPU的影响

**题目**：采用周期挪用方式的DMA，对CPU的影响是（ ）。
A. CPU被完全暂停
B. CPU可能被延迟一个存取周期
C. CPU执行速度减半
D. 对CPU完全没有影响

**答案**：B

**解析**：
- A是停止CPU方式的特征
- 周期挪用：DMA在CPU不访存时挪用一个存取周期，可能延迟CPU一个周期
- C是交替访问方式(夸张说法)
- D错：周期挪用会影响CPU(延迟一个周期)

## 直观理解

**做题技巧**：
1. 看到"数据经过CPU" → 中断方式
2. 看到"数据直接到主存" → DMA方式
3. 看到"CPU全程参与" → 程序查询
4. 看到"高速批量" → 首选DMA
5. 看到"低速少量" → 中断或查询

## 知识关联

### IO控制方式选择决策树

```
需要IO操作
    │
    ├─ 数据量大(>几十个字)?
    │   ├─ 是 → DMA或通道
    │   │      ├─ 中小型系统 → DMA
    │   │      └─ 大型机 → 通道
    │   └─ 否 → 程序中断
    │
    └─ 设备极简单(如LED)?
        └─ 程序查询
```

### 易错陷阱

1. **"并行"的含义**：CPU和设备可以同时工作(不是同时传数据)
2. **中断方式也有硬件**：中断控制器是硬件
3. **DMA的"直接"是相对于CPU的**：仍然经过总线和主存
4. **通道和DMA的区别**：通道有指令系统，DMA没有
