# 26_DMA_预处理

## 核心概念

**DMA预处理**：CPU在启动DMA传送之前，对DMA控制器进行初始化设置的过程。

## 原理分析

### CPU设置的参数清单

```
CPU执行IO指令设置DMA:

1. 传送方向(读/写)
   CPU → DMA控制寄存器
   读: 设备数据 → 主存
   写: 主存 → 设备

2. 主存起始地址
   CPU → DMA地址寄存器(AR)
   数据在主存中的首地址

3. 传送字数(数据量)
   CPU → DMA数据计数器(DC)
   要传送多少个数据

4. 设备选择
   CPU → DMA设备选择逻辑
   选择哪个IO设备

5. 启动DMA
   CPU → DMA控制寄存器(GO位)
   DMA开始工作
```

### 预处理的伪代码

```c
// CPU执行的预处理代码
void DMA_PreSetup(int direction, int mem_addr, int count, int device_id) {
    OUT(DMA_CONTROL_PORT, direction);   // 设置传送方向
    OUT(DMA_ADDR_PORT, mem_addr);       // 设置主存起始地址
    OUT(DMA_COUNT_PORT, count);         // 设置传送字数
    OUT(DMA_DEVICE_PORT, device_id);    // 选择设备
    OUT(DMA_CONTROL_PORT, GO_BIT);      // 启动DMA
}
// 之后CPU可以去做其他事情
// DMA控制器自动完成数据传送
```

### 预处理的典型场景

**场景1：从磁盘读取一个扇区(512字节)到主存**
```
方向 = 读(磁盘→主存)
起始地址 = 0x100000 (主存中的缓冲区首地址)
传送字数 = 512
设备 = 磁盘控制器
```

**场景2：将主存中的一块数据发送到网卡**
```
方向 = 写(主存→网卡)
起始地址 = 0x200000 (数据包首地址)
传送字数 = 1500 (以太网MTU)
设备 = 网卡
```

### 预处理之后CPU的行为

预处理完成后，CPU可以：
- 继续执行原来的程序
- 处理其他IO请求
- 执行其他计算任务

**关键**：CPU不需要等待DMA传送完成。DMA在后台独立工作，完成后通过中断通知CPU。

## 直观理解

**预处理类比**：给快递下单
- 你在网上填写：寄到哪(地址)、寄多少(数量)、用哪个快递公司(设备)、寄还是取(方向)
- 点"下单"(启动DMA)
- 然后你该干嘛干嘛，等快递到了通知你

## 知识关联

### 408常考

1. **预处理是CPU唯一参与数据传送的阶段**（除了后处理）
2. **设置的参数在数据传送中自动使用**：AR每次+1，DC每次-1
3. **预处理时间一般很短**：几个OUT指令，微秒级

### 易错陷阱

1. **预处理不传送任何数据**：只是"准备"工作
2. **AR初值 = 起始地址**：不是0
3. **DC初值 = 传送字数**：不是0，每次-1直到0
4. **不同DMA传送需要重新预处理**：每次新的IO操作都需要
