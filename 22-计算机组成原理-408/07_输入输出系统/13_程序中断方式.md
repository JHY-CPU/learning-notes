# 13_程序中断方式

## 核心概念

- **程序中断方式(Interrupt-Driven I/O)**：IO设备准备好数据后主动向CPU发出中断请求，CPU响应中断后执行中断服务程序完成数据传送
- **核心思想**：CPU不用等待设备，设备就绪后"通知"CPU
- **对比程序查询**：查询是CPU主动问设备，中断是设备主动通知CPU

### 中断方式的基本特征

| 特征 | 说明 |
|------|------|
| CPU参与度 | 仅在中断时参与，其他时间可做其他工作 |
| 数据传送 | 仍然经过CPU中转(设备→CPU→主存) |
| CPU利用率 | 较程序查询大幅提升 |
| 硬件复杂度 | 中等，需要中断控制器(如8259) |
| 响应速度 | 较快，设备就绪立即通知 |
| 适用场景 | 低速到中速设备 |
| 并行性 | CPU与设备可部分并行工作 |

## 原理分析

### 中断方式工作流程

```
CPU侧:                          设备侧:
┌──────────────────┐            ┌──────────────────┐
│ CPU启动设备       │──命令──→│ 设备开始工作       │
└────────┬─────────┘            └────────┬─────────┘
         │                               │
┌────────▼─────────┐            ┌────────▼─────────┐
│ CPU去做其他任务   │            │ 设备准备数据       │
│ (不必等待)        │            │ (耗时较长)         │
└────────┬─────────┘            └────────┬─────────┘
         │                               │
         │            ┌──────────────────┘
         │            │ 中断请求(INTR)
         │            ↓
┌────────▼─────────────────────────┐
│ CPU响应中断                       │
│ 1. 关中断                        │
│ 2. 保存PC和PSW(硬件自动)         │
│ 3. 转入中断服务程序               │
│ 4. 保护现场(软件, PUSH寄存器)    │
│ 5. 执行数据传送(IN/OUT)          │
│ 6. 恢复现场(软件, POP寄存器)     │
│ 7. 开中断                        │
│ 8. 中断返回(IRET)                │
└────────┬─────────────────────────┘
         │
┌────────▼─────────┐
│ CPU继续做其他任务  │
└──────────────────┘
```

### 中断方式 vs 程序查询方式时间对比

假设设备每100ms准备好一个数据，传送需1μs。

**程序查询方式**：
- CPU每100ms都在循环查询 → 有效利用率极低

**中断方式**：
- 设备就绪前CPU执行其他任务
- 中断响应 + 现场保护 + 传送 + 恢复 + 返回 = 约10~20μs
- CPU利用率 = $(100ms - 20\mu s) / 100ms = 99.98\%$ 可用于其他任务

### 中断方式的局限性

- **每次中断只能传送一个数据**：每传一个数据就要中断一次
- **中断开销固定**：每次中断有响应、保护现场、恢复、返回的固定开销
- **不适合高速设备**：高速设备数据周期短，频繁中断的开销将超过有效工作

**例**：磁盘每1μs输出一个数据
- 每次中断开销约10μs
- 10μs中断开销只能传1个数据(1μs)
- 效率 = $1 / (10+1) \approx 9\%$
- 结论：高速设备不适合中断方式，应使用DMA

## 直观理解

**中断方式类比**：你在办公室等快递
- 程序查询：你每隔5分钟下楼看一次快递到了没 → 浪费时间
- 中断方式：快递到了快递员打电话通知你 → 你安心工作，接到电话才下楼

**中断开销类比**：
- 每次接电话：挂断手头工作(保护现场) → 下楼取快递(传送) → 回来继续工作(恢复)
- 如果快递每分钟来一个(高速设备)，你光接电话跑腿就忙不过来
- 这种情况应该请个助理专门收快递(DMA)

### 易错陷阱

1. **中断方式数据仍经过CPU**：与DMA的关键区别，中断方式的数据由CPU传送
2. **每次中断只传一个数据**：不要以为中断一次就能传一批数据
3. **中断有固定开销**：不是设备一中断就立即传数据，需要响应和现场保护
4. **中断方式不适合高速设备**：频繁中断的开销会压垮CPU
