# 36_DMA优先级

## 核心概念

- **DMA优先级高于中断**：因为DMA往往有严格的时间窗口限制
- **多个DMA之间的优先级**：通过总线仲裁器确定
- **优先级管理**：确保关键IO操作不被延误

## 原理分析

### DMA优先级高于中断的原因

| 原因 | 说明 |
|------|------|
| 时间窗口限制 | 磁盘等设备有固定的扇区窗口，错过就丢失数据 |
| 数据连续性 | 网络数据包必须连续接收，不能中断 |
| 带宽利用 | DMA传送期间暂停会降低传输效率 |
| 中断可延迟 | 中断推迟几微秒影响不大 |

**例**：磁盘以7200RPM旋转，每个扇区窗口约4μs。如果DMA因中断处理错过窗口，这个扇区要等一整圈(8.3ms)才能重新读取。

### 优先级层次结构

```
最高优先级
    │
    ├── 内部中断(如硬件错误、掉电)
    │
    ├── DMA请求(多个DMA按优先级排列)
    │   ├── 磁盘DMA(最高)
    │   ├── 网卡DMA
    │   └── 其他DMA(最低)
    │
    ├── 不可屏蔽中断(NMI)
    │
    ├── 可屏蔽中断(INTR)
    │   ├── 高优先级中断源
    │   ├── 中优先级中断源
    │   └── 低优先级中断源
    │
    └── 程序查询(最低)
```

### 多个DMA的优先级仲裁

当系统有多个DMA控制器时(如磁盘DMA、网卡DMA同时请求总线)：

**仲裁方式**：
1. **固定优先级**：每个DMA有固定的优先级，高优先级先获得总线
2. **轮转优先级**：轮流获得总线，保证公平性
3. **动态优先级**：根据等待时间动态调整优先级

```
           ┌──────────┐
DMA1请求 ──→│          │
DMA2请求 ──→│ 总线仲裁器 │──→ 总线授予信号
DMA3请求 ──→│          │
           └──────────┘
```

### DMA与CPU的总线仲裁

DMA和CPU共享总线时的仲裁：

```
DMA: HRQ(总线请求) → CPU
CPU: HLDA(总线响应) → DMA
```

**握手过程**：
1. DMA发出HRQ(请求使用总线)
2. CPU在当前总线周期结束时响应，发出HLDA
3. CPU释放总线(三态门高阻态)
4. DMA接管总线
5. DMA传送完毕后释放总线
6. CPU撤回HLDA，恢复总线控制

## 直观理解

**DMA优先级高于中断的类比**：

想象在医院急诊室：
- **DMA** = 心脏骤停病人(必须立即抢救，错过黄金4分钟就来不及)
- **中断** = 普通急诊(可以等几分钟不影响)
- 心脏骤停优先级高于普通急诊，这就是DMA优先级更高的原因

### 易错陷阱

1. **DMA优先级高于中断，但不是绝对的**：某些不可屏蔽中断(NMI如掉电)优先级最高
2. **多个DMA之间也有优先级差异**：磁盘DMA通常高于网卡DMA
3. **DMA优先级高不等于DMA永远先**：仲裁器可以使用公平策略
4. **总线仲裁不等于中断仲裁**：总线仲裁是硬件信号级的，中断仲裁是系统级的
