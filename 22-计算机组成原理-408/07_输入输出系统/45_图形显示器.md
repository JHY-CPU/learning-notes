# 45_图形显示器

## 核心概念

- **图形显示模式**：每个像素独立控制，可显示任意图形(不限于字符)
- **帧缓冲(Frame Buffer)**：存储每个像素颜色值的显存区域
- **像素映射**：帧缓冲中的每个单元对应屏幕上一个像素

## 原理分析

### 帧缓冲(显存)结构

```
帧缓冲:
┌─────────────────────────────┐
│ 像素(0,0) │ 像素(1,0) │ ... │  ← 第0行
├───────────┼───────────┼─────┤
│ 像素(0,1) │ 像素(1,1) │ ... │  ← 第1行
├───────────┼───────────┼─────┤
│ ...       │ ...       │ ... │
└─────────────────────────────┘

每个像素 = n位(颜色深度)
```

### 颜色表示

| 颜色深度 | 颜色数 | 说明 |
|----------|--------|------|
| 1位 | 2色 | 黑白(单色) |
| 4位 | 16色 | 基本彩色 |
| 8位 | 256色 | 索引颜色(使用颜色表) |
| 16位 | 65536色 | 高彩色 |
| 24位 | 1677万色 | 真彩色(RGB各8位) |
| 32位 | 1677万色+Alpha | 真彩色+透明度 |

### 颜色表(调色板)

- 8位及以下颜色深度通常使用**颜色表(Color Table/LUT)**
- 像素值作为索引，查表得到实际RGB颜色

```
像素值(8位) → 颜色表(256项) → RGB值(24位)
    0x00  →  [0x000000]  → 黑色
    0x01  →  [0xFF0000]  → 红色
    0x02  →  [0x00FF00]  → 绿色
    ...
```

- **颜色表大小** = $2^n \times 3$ 字节(n为颜色深度)
- 例：8位颜色表 = $256 \times 3 = 768$ 字节

### 显存容量计算

$$\text{显存容量} = \text{分辨率} \times \text{每像素位数} / 8$$

| 分辨率 | 颜色深度 | 显存容量 |
|--------|----------|----------|
| 640×480 | 8位 | 300KB |
| 640×480 | 24位 | 900KB |
| 1024×768 | 24位 | 2.25MB |
| 1920×1080 | 24位 | 6MB |
| 3840×2160 | 24位 | 24MB |

**考虑双缓冲**：

$$\text{实际显存} \approx 2 \times \text{帧缓冲容量}$$

### 图形加速

- **GPU(Graphics Processing Unit)**：专用图形处理器
- **功能**：2D/3D图形渲染、视频解码、并行计算
- **显存**：GPU专用的高速存储器(GDDR)
- GPU可以直接操作显存，不需要CPU干预

## 直观理解

**帧缓冲类比**：巨大的调色板画布
- 每个格子(像素)可以独立涂上不同颜色
- 帧缓冲 = 记录每个格子涂了什么颜色的表格
- 刷新 = 每秒60次照着表格重新画一遍

## 知识关联

### 与字符模式对比

| 对比项 | 字符模式 | 图形模式 |
|--------|----------|----------|
| 显示单位 | 字符 | 像素 |
| 显存内容 | ASCII码+属性 | 像素颜色值 |
| 显存大小 | 很小(几KB) | 很大(几MB~几GB) |
| 灵活性 | 只能显示字符 | 可显示任意图形 |
| 硬件需求 | 简单(字符发生器) | 复杂(GPU) |

### 408常考

1. **帧缓冲容量计算**：分辨率×颜色深度/8
2. **颜色表的作用**：用较少位数表示较多颜色
3. **双缓冲技术**：避免画面撕裂

### 易错陷阱

1. **显存容量需要对齐**：实际显存可能大于理论计算值
2. **颜色深度与颜色数的关系**：n位颜色深度最多$2^n$种颜色
3. **真彩色不需要颜色表**：24位直接存储RGB值
4. **双缓冲不等于容量翻倍**：也可能使用其他缓冲策略
