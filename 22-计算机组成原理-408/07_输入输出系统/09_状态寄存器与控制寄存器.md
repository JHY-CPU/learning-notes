# 09_状态寄存器与控制寄存器

## 核心概念

### 状态寄存器(Status Register, SR)

- **作用**：向CPU报告IO设备的当前工作状态
- **访问方式**：CPU**只读**(硬件自动更新状态位)
- **典型状态位**：

| 位 | 名称 | 值=1 | 值=0 |
|----|------|------|------|
| D0 | READY(就绪) | 设备数据已准备好 | 数据未准备好 |
| D1 | BUSY(忙碌) | 设备正在执行操作 | 设备空闲 |
| D2 | DONE(完成) | 上一次操作已完成 | 操作进行中 |
| D3 | ERROR(错误) | 操作出错 | 操作正常 |
| D4 | BUFFER FULL | 缓冲区满 | 缓冲区有空 |
| D5~D7 | 保留 | 扩展用 | 扩用用 |

- **状态更新机制**：由硬件(设备)自动设置，CPU只能读取
- **清零机制**：某些状态位在CPU读取后自动清零(如DONE)，某些需设备操作清零(如BUSY)

### 控制寄存器(Control Register, CR)

- **作用**：CPU向IO设备发出控制命令
- **访问方式**：CPU**只写**(设备根据命令执行操作)
- **典型控制位**：

| 位 | 名称 | 值=1 | 值=0 |
|----|------|------|------|
| D0 | START(启动) | 启动设备操作 | 停止操作 |
| D1 | IE(中断使能) | 允许设备发中断 | 禁止中断 |
| D2 | R/W(读写方向) | 输入(读设备) | 输出(写设备) |
| D3 | RESET(复位) | 设备复位 | 正常工作 |
| D4~D7 | 功能选择 | 特定功能 | 特定功能 |

- **写入机制**：CPU执行OUT指令将命令字写入控制寄存器，设备硬件电路根据命令位执行相应操作

## 原理分析

### CPU与IO接口的交互流程

**标准IO操作(读取设备数据)**：
```
1. CPU → 写控制寄存器：设置方向=输入，启动设备
2. 设备 → 开始工作，状态寄存器 BUSY=1
3. CPU ← 读状态寄存器：检查 BUSY 位
4. (循环检查直到)状态寄存器 DONE=1, BUSY=0
5. CPU ← 读数据输入寄存器：获取数据
6. CPU → 写控制寄存器：清除完成标志(或硬件自动清除)
```

**中断驱动的IO操作**：
```
1. CPU → 写控制寄存器：设置方向=输入，启动设备，IE=1(允许中断)
2. 设备 → 开始工作，CPU 去做其他任务
3. 设备完成 → 设置 DONE=1，发出中断请求 INTR
4. CPU 响应中断 → 转入中断服务程序
5. 中断服务程序：读状态寄存器确认完成 → 读数据 → 清标志 → 中断返回
```

### 读状态→发控制 的标准模式

这是IO控制中最基本的操作模式：

```c
// 伪代码：程序查询方式
do {
    status = IN(STATUS_PORT);    // 读状态寄存器
} while ((status & BUSY) != 0);  // 等待设备空闲
OUT(CONTROL_PORT, command);       // 写控制寄存器，发出命令
do {
    status = IN(STATUS_PORT);    // 再次读状态
} while ((status & DONE) == 0);  // 等待操作完成
data = IN(DATA_PORT);            // 读数据端口
```

### 可编程接口的概念

- **可编程**：CPU通过写控制寄存器来配置接口的工作方式
- 例：8255可编程并行接口芯片
  - 控制字写入控制端口(端口地址3)
  - 可设置A/B/C三个端口的工作模式
  - 一个控制字决定整个芯片的工作方式
- 这种"通过写控制字配置工作方式"的接口称为**可编程接口**

## 直观理解

**状态寄存器类比**：汽车仪表盘
- READY = 油灯亮(可以出发)
- BUSY = 转速表(发动机在工作)
- DONE = 到达目的地指示灯
- ERROR = 故障警告灯
- 驾驶员(CPU)看仪表盘获取信息，但不能直接改变仪表读数

**控制寄存器类比**：汽车操控面板
- START = 点火按钮
- IE = 是否允许鸣笛
- R/W = 前进/倒车档
- RESET = 重启按钮
- 驾驶员(CPU)操作控制面板指挥汽车工作

**"读状态→发控制"类比**：
- 去餐厅吃饭：先看是否营业(读状态) → 点餐(写控制) → 等菜(读状态) → 吃饭(读数据)

## 知识关联

### 跨章节联系

| 相关知识点 | 联系 |
|------------|------|
| **CPU状态寄存器(第4章)** | IO的状态寄存器类似CPU的PSW，记录当前状态 |
| **中断系统(第4章)** | 状态寄存器的DONE位可触发中断请求 |
| **程序查询/中断/DMA** | 状态寄存器是程序查询的核心，中断方式用状态触发中断 |

### 常考组合

- 状态寄存器 + 程序查询方式(CPU轮询状态位)
- 控制寄存器 + 中断使能(IE位决定是否允许中断)
- 读状态→发控制→读数据(标准IO操作三步曲)

### 易错陷阱

1. **状态寄存器是只读的**：CPU不能直接写入状态寄存器来改变设备状态
2. **控制寄存器是只写的**：CPU一般不能读回控制寄存器的值(某些芯片例外)
3. **DONE位的清除**：通常在CPU读数据后自动清除，而非读状态后
4. **中断使能位(IE)在控制寄存器中**：不是在状态寄存器中，这是常考的混淆点
5. **同一端口地址，读写方向不同**：读可能是读状态，写可能是写控制，需结合IOR/IOW信号判断
