# 24_DMA控制器结构

## 核心概念

**DMA控制器**是独立于CPU的专用硬件，负责在IO设备和主存之间直接传送数据。

### DMA控制器内部结构

```
┌─────────────────────────────────────┐
│           DMA控制器                  │
│                                     │
│  ┌───────────────┐                  │
│  │ 主存地址寄存器 │ ←AR(MAR)        │
│  │ (Memory Addr)  │  存放主存地址    │
│  ├───────────────┤                  │
│  │ 数据计数器     │ ←DC(WC)         │
│  │ (Data Count)   │  存放剩余字数    │
│  ├───────────────┤                  │
│  │ 数据缓冲寄存器 │ ←BR             │
│  │ (Buffer Reg)   │  暂存传送数据    │
│  ├───────────────┤                  │
│  │ 控制/状态逻辑  │                  │
│  │ ·控制寄存器    │  存放传送方向等  │
│  │ ·状态寄存器    │  传送完成/出错   │
│  └───────────────┘                  │
│                                     │
│  ┌───────────────┐                  │
│  │ DMA请求触发器  │  向CPU发DMA请求  │
│  │ DREQ          │                  │
│  ├───────────────┤                  │
│  │ DMA响应触发器  │  接收CPU响应     │
│  │ DACK          │                  │
│  └───────────────┘                  │
└──────────────┬──────────────────────┘
               │
    连接系统总线(地址/数据/控制)
```

### 各寄存器功能

| 寄存器 | 符号 | 功能 | 更新方式 |
|--------|------|------|----------|
| 主存地址寄存器 | AR(MAR) | 存放下一个数据的主存地址 | 每传送一个字+1(或+N) |
| 数据计数器 | DC(WC) | 存放剩余传送字数 | 每传送一个字-1 |
| 数据缓冲寄存器 | BR | 暂存正在传送的数据 | 硬件自动 |
| 控制寄存器 | CR | 传送方向、模式等 | CPU预处理设置 |
| 状态寄存器 | SR | 传送完成/错误标志 | 硬件自动 |

## 原理分析

### DMA的数据传送路径

**读主存(主存→设备)**：
```
主存 → 数据总线 → DMA(BR) → IO设备
         ↑                    ↑
     AR提供地址           设备地址
```

**写主存(设备→主存)**：
```
IO设备 → DMA(BR) → 数据总线 → 主存
                            ↑
                        AR提供地址
```

### DMA控制器的总线连接

```
        地址总线(A)       数据总线(D)       控制总线(C)
           ↑                 ↑                 ↑
    ┌──────┴─────────────────┴─────────────────┴──────┐
    │                    DMA控制器                     │
    │  AR ──→ 地址总线(输出主存地址)                    │
    │  BR ←→ 数据总线(双向数据传送)                     │
    │  控制逻辑 ──→ 控制总线(发读写命令)                │
    │                                                 │
    │  ←── DREQ(设备的DMA请求)                         │
    │  →── DACK(DMA响应给设备)                         │
    │  ←── HRQ(DMA向CPU发总线请求)                     │
    │  →── HLDA(CPU让出总线的响应)                     │
    └─────────────────────────────────────────────────┘
```

### DMA的两种工作模式

**单字传送模式**：
- 每次传送一个字后释放总线
- CPU可以在DMA传送间隙使用总线
- 适合IO设备速度相对较慢的情况

**成组(突发)传送模式**：
- 一次获得总线后连续传送多个字
- 直到数据块传送完毕才释放总线
- 适合高速IO设备(如磁盘DMA)

## 直观理解

**DMA控制器类比**：搬家公司
- **AR(地址寄存器)**：搬家的目的地地址(每搬一件更新)
- **DC(计数器)**：还需要搬几件(每搬一件减1)
- **BR(缓冲)**：搬运工手上的货物
- **CR(控制寄存器)**：搬家指令(从哪搬到哪)
- **SR(状态寄存器)**：搬家进度(完成/出错)

### 易错陷阱

1. **DMA控制器没有指令系统**：不能执行程序，只能通过寄存器设置参数
2. **AR存放的是主存地址**：不是设备地址(设备地址在设备控制器中)
3. **DC存放的是剩余字数**：不是总字数(预处理设初值，每传一字减1)
4. **数据缓冲BR是必需的**：因为总线宽度和设备数据宽度可能不同
