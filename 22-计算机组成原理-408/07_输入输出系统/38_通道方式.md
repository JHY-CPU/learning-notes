# 38_通道方式

## 核心概念

- **通道(Channel)**：一种简化的IO处理器(处理机)，有自己的指令系统(通道指令)，可以执行简单的通道程序来控制IO操作
- **本质**：通道 = DMA + 简单CPU，比DMA更智能，比完整CPU更简单
- **地位**：在大型机和服务器中广泛使用，是DMA的进一步发展

### 通道的基本特征

| 特征 | 说明 |
|------|------|
| 独立性 | 有自己的指令(通道指令)，可执行通道程序 |
| 灵活性 | 可处理复杂IO操作(如链式传输、条件判断) |
| CPU参与度 | 仅在IO开始和结束时参与(比DMA更少) |
| 适用系统 | 大型机、服务器(管理大量IO设备) |
| 硬件复杂度 | 高于DMA，低于完整CPU |

### 通道的工作原理

```
CPU:                    通道:                    IO设备:
  │                       │                        │
  │ 1.CPU启动通道         │                        │
  │  (发出SIO指令) ──────→│                        │
  │                       │                        │
  │ 2.CPU去执行其他程序   │ 通道执行通道程序        │
  │                       │  从主存取通道指令       │
  │                       │  译码并执行             │
  │                       │  控制IO设备传送数据     │
  │                       │  ←──────────────────→  │
  │                       │  (通道自己管理整个IO)    │
  │                       │                        │
  │ ←── 3.通道发中断 ─── │ (IO完成或出错)          │
  │ 4.CPU处理中断         │                        │
```

### 通道指令(Channel Command Word, CCW)

通道有自己的指令集，称为**通道指令**或**通道命令字(CCW)**：

| 指令类型 | 说明 |
|----------|------|
| 数据传送 | 读(设备→主存)、写(主存→设备) |
| 控制 | 启动、停止、复位设备 |
| 转移 | 跳转到其他通道指令(类似GOTO) |
| 条件判断 | 根据状态条件跳转 |

**CCW格式**：
```
┌────────┬────────┬──────────┬──────────┐
│ 操作码 │ 数据地址│ 标志位   │ 计数     │
│ (命令) │ (主存)  │ (链/跳)  │ (字节数) │
└────────┴────────┴──────────┴──────────┘
```

### 通道程序

- 多条CCW组成的程序，存放在主存中
- CPU设置好通道程序的起始地址后，通道自动执行
- 通道按顺序执行CCW，直到遇到"停止"或"结束"指令

## 直观理解

**通道类比**：部门经理
- **CPU** = 公司CEO，只在项目开始时下指令(启动通道)，项目结束时听取汇报(中断)
- **通道** = 部门经理，有自己的管理手册(通道指令)，可以自己组织团队完成项目(IO操作)
- **DMA** = 高级员工，只会按CEO的具体指示一步步搬数据

**通道 vs DMA**：
- DMA：CEO说"把这1000个字从A搬到B"，员工照搬
- 通道：CEO说"把这个文件处理一下"，经理自己决定怎么搬、搬多少、检查对错

### 易错陷阱

1. **通道不是完整CPU**：指令简单，只能做IO操作
2. **通道指令不等于CPU指令**：CCW是专门的通道指令
3. **通道程序在主存中**：CPU需要预先在主存中设置好通道程序
4. **通道可以管理多个设备**：一个通道可以连接多个IO设备
