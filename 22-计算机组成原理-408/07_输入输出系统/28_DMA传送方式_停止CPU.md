# 29_DMA传送方式_停止CPU

## 核心概念

**停止CPU访存方式**：DMA传送期间，CPU完全停止访问主存，让出总线给DMA独占使用。

## 原理分析

### 工作原理

```
时间轴:
├── CPU执行 ──┤───────── DMA独占总线 ─────────┤── CPU继续 ──→
              │  传送第1个  传送第2个 ... 传送第N个 │
              │←──────── DMA占用期间 ────────→│
              CPU暂停(CPU不访存)
```

- DMA请求到来时，CPU让出总线
- CPU暂停访存(不能读写主存)
- DMA独占总线，连续传送所有数据
- 传送完毕后，DMA释放总线，CPU恢复工作

### CPU暂停期间的行为

**如果CPU有Cache**：
- CPU可以继续从Cache取指令和数据(不用访存)
- Cache命中时CPU仍可工作
- Cache未命中时CPU必须等待(不能访存)

**如果CPU没有Cache**：
- CPU完全停止工作
- 等DMA释放总线后才能继续

### 停止CPU方式的特点

| 特征 | 说明 |
|------|------|
| 实现复杂度 | 最简单 |
| CPU暂停时间 | 最长(DMA传送整个数据块) |
| 数据传输率 | 最高(独占总线) |
| CPU利用率 | 最低(被长时间暂停) |
| 适用场景 | 数据量小、对延迟不敏感 |

### 带宽分析

设总线宽度 $W$ 位，总线频率 $f$ Hz，传送 $N$ 个数据。

$$T_{\text{DMA}} = N \times \frac{1}{f}$$

$$R_{\text{DMA}} = W \times f \quad (\text{bit/s，DMA独占})$$

$$\text{CPU暂停时间占比} = \frac{T_{\text{DMA}}}{T_{\text{DMA}} + T_{\text{CPU}}}$$

## 直观理解

**类比**：修路
- 修路机(DMA)需要占用整条路(总线)
- 所有车辆(CPU)必须停下等待
- 路修完后车辆恢复通行
- 优点是修得最快(独占道路)
- 缺是所有车都被堵住了(CPU暂停)

## 知识关联

### 与其他方式对比

| 方式 | CPU暂停 | 传输效率 | 实现复杂度 |
|------|---------|----------|-----------|
| 停止CPU | 最长 | 最高 | 最简单 |
| 周期挪用 | 短(每次一个周期) | 中等 | 中等 |
| 交替访问 | 不暂停 | 较低 | 最复杂 |

### 易错陷阱

1. **"停止CPU"不是关机**：CPU只是暂停访存，可能继续执行Cache中的指令
2. **适用于小数据量**：大数据量会使CPU暂停太久
3. **数据传输率最高**：因为没有中断DMA的开销
4. **与中断方式完全不同**：不要混淆"停止CPU"和"中断"
