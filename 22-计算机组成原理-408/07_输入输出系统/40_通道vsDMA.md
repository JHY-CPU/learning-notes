# 40_通道vsDMA

## 核心概念

通道和DMA都是IO控制方式，但通道比DMA更智能、更独立。

## 原理分析

### 全面对比表(必背)

| 对比维度 | DMA | 通道 |
|----------|-----|------|
| **本质** | 专用硬件控制器 | 简化的IO处理器 |
| **指令系统** | 无(只有寄存器) | 有(通道指令CCW) |
| **控制方式** | CPU预处理设置参数 | CPU启动通道程序 |
| **CPU参与程度** | 预处理+后处理 | 启动+中断(更少) |
| **数据传送控制** | DMA硬件逐字控制 | 通道执行通道程序控制 |
| **灵活性** | 低(参数固定) | 高(通道程序可变) |
| **设备管理** | 一个DMA管一个设备 | 一个通道管多个设备 |
| **独立性** | 需CPU预处理 | 可独立执行程序 |
| **硬件复杂度** | 中等 | 较高 |
| **适用系统** | 中小型系统 | 大型机、服务器 |

### 关键区别：智能化程度

**DMA的工作方式**：
1. CPU设置参数(地址、字数、方向)
2. DMA按参数逐字传送
3. 传送完毕通知CPU
4. **DMA没有判断能力**：不能根据情况改变传送策略

**通道的工作方式**：
1. CPU在主存中设置通道程序(多条CCW)
2. 通道执行通道程序
3. 通道可以**自己做判断**：如根据状态跳转、链式传送
4. 通道完成或出错通知CPU

**举例**：
- DMA：老板说"把这1000个字节从A搬到B" → DMA照搬
- 通道：老板说"把这个文件读出来，如果出错就重试3次，成功就发给打印机" → 通道自己执行整个流程

### 通道 = DMA + 简单CPU

| 组件 | DMA | 通道 |
|------|-----|------|
| 地址寄存器 | 有 | 有 |
| 计数器 | 有 | 有 |
| 数据缓冲 | 有 | 有 |
| 指令译码器 | 无 | **有** |
| 程序计数器 | 无 | **有** |
| 条件判断逻辑 | 无 | **有** |

通道在DMA的基础上增加了**指令译码、程序计数、条件判断**等功能，相当于一个简单的专用CPU。

## 直观理解

**DMA vs 通道类比**：

| 场景 | DMA | 通道 |
|------|-----|------|
| 搬家 | 搬家公司：你指定每件东西搬到哪，他们照做 | 高级管家：你只说"搬完这个家"，他自己规划怎么搬 |
| 独立性 | 员工：老板说一步做一步 | 经理：老板说目标，经理自己安排 |
| 灵活性 | 机器人：只能执行预设程序 | 智能助手：可以根据情况调整 |

## 知识关联

### IO控制方式演进路线

```
程序查询 → 程序中断 → DMA → 通道 → IO处理机
(最简单)                                   (最复杂)
(CPU参与最多)                              (CPU参与最少)
```

每个阶段的核心改进：
- **中断**：CPU不必等待设备
- **DMA**：数据传送不经过CPU
- **通道**：IO控制有独立的程序
- **IO处理机**：完整的IO计算机

### 408常考

1. **通道比DMA更独立** — 对
2. **通道有指令系统** — 对，DMA没有
3. **通道是DMA的进一步发展** — 对
4. **通道可以管理多个设备** — 对

### 易错陷阱

1. **通道不是完整CPU**：指令简单，只能做IO操作
2. **通道也需要CPU参与**：启动和中断处理阶段需要CPU
3. **DMA不能执行程序**：只有通道和IO处理机能执行程序
4. **通道程序在主存中**：不是在通道内部
