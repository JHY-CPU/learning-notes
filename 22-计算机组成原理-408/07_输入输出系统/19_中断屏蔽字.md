# 19_中断屏蔽字

## 核心概念

- **中断屏蔽字(Interrupt Mask Word)**：每个中断源对应一个屏蔽字，控制该中断源可以被哪些中断打断
- **屏蔽字的位数** = 中断源的个数
- **每位含义**：
  - 1 = 屏蔽(不允许该中断源打断当前处理)
  - 0 = 开放(允许该中断源打断当前处理)

### 屏蔽字的基本规则

- 每个中断源的屏蔽字是一个n位二进制数(n=中断源数量)
- 屏蔽字的第i位对应第i个中断源
- **自身的屏蔽位必须为1**：处理自己的中断时，不能被自己再次中断(避免重入)
- **优先级关系**：优先级高的中断源可以打断优先级低的处理

## 原理分析

### 屏蔽字的构造方法

**规则**：对于优先级为 $P_k$ 的中断源 $k$，其屏蔽字的构造原则为：
- 对于所有优先级 $\leq P_k$ 的中断源(包括自身)，屏蔽位设为1
- 对于所有优先级 $> P_k$ 的中断源，屏蔽位设为0

**直觉理解**：屏蔽字告诉你"我可以被谁打断"——只能被比我更高优先级的打断。

**举例**：5个中断源 $I_1, I_2, I_3, I_4, I_5$，优先级 $I_1 > I_2 > I_3 > I_4 > I_5$

| 中断源 | 优先级 | 屏蔽字(从高到低: I1 I2 I3 I4 I5) | 含义 |
|--------|--------|----------------------------------|------|
| $I_1$ | 最高 | 1 1 1 1 1 | 屏蔽所有(不能被打断) |
| $I_2$ | 次高 | 0 1 1 1 1 | 只允许$I_1$打断 |
| $I_3$ | 中等 | 0 0 1 1 1 | 允许$I_1, I_2$打断 |
| $I_4$ | 次低 | 0 0 0 1 1 | 允许$I_1, I_2, I_3$打断 |
| $I_5$ | 最低 | 0 0 0 0 1 | 允许所有其他中断打断 |

### 屏蔽字的使用

**在中断响应时**：
1. CPU正在处理中断源 $k$
2. 当前屏蔽字为中断源 $k$ 的屏蔽字 $M_k$
3. 新的中断请求向量为 $R$
4. 实际有效请求 = $R \text{ AND } (\text{NOT } M_k)$
5. 若结果非零，说明有更高优先级中断请求，响应之

**例**：正在处理$I_3$，屏蔽字 = 00111
- 设备$I_1$请求 → $R = 10000$，$\text{NOT }M_3 = 11000$
- 有效请求 = $10000 \text{ AND } 11000 = 10000 \neq 0$ → 响应$I_1$

### 动态改变优先级

通过修改屏蔽字，可以动态改变中断源的优先级：
- 屏蔽字的设置可以由操作系统在运行时修改
- 这提供了很大的灵活性

## 直观理解

**屏蔽字类比**：门禁卡系统
- 每个楼层(中断源)有一张门禁卡(屏蔽字)
- 1 = 此门关闭(不允许进入/不允许打断)
- 0 = 此门开放(允许进入/允许打断)
- 老总的卡(最高优先级)可以进入所有楼层，所有门都对他关闭(屏蔽字全1)
- 实习生的卡(最低优先级)只能进自己的楼层，其他楼层的门都对他开放

### 易错陷阱

1. **屏蔽位=1表示屏蔽(不允许打断)**：不是1=允许
2. **屏蔽字最高位对应最高优先级中断源**：位的排列顺序要明确
3. **中断响应次序 ≠ 中断处理次序**：响应次序由硬件优先级决定，处理次序受屏蔽字影响
4. **自身屏蔽位必须为1**：处理自己时不允许自己再次中断
