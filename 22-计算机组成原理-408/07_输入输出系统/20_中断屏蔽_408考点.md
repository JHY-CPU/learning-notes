# 21_中断屏蔽_408考点

## 核心概念

本节汇总408考研中关于中断屏蔽和中断处理次序的高频真题和典型考法。

## 原理分析

### 考点一：中断响应次序 vs 中断处理次序

| 概念 | 含义 | 决定因素 |
|------|------|----------|
| 中断响应次序 | 多个中断同时请求时，先响应哪个 | 硬件优先级(排队器) |
| 中断处理次序 | 多个中断实际被处理的顺序 | 屏蔽字(软件可设) |

**关键理解**：
- 响应次序是固定的，由硬件决定
- 处理次序可以通过屏蔽字调整
- **响应次序 ≠ 处理次序**

**例**：硬件优先级 $I_1 > I_2 > I_3 > I_4$
- 同时请求 → 先响应 $I_1$
- 但 $I_1$ 的屏蔽字若允许 $I_2$ 打断 → $I_1$ 处理被 $I_2$ 抢占 → 处理次序变为 $I_2$ 先完成

### 考点二：给定屏蔽字求执行序列

**408典型题**：

4个中断源 $I_1 \sim I_4$，硬件优先级 $I_1 > I_2 > I_3 > I_4$。屏蔽字设置如下：

| 中断源 | 屏蔽字(1 2 3 4) |
|--------|----------------|
| $I_1$ | 1 1 0 0 |
| $I_2$ | 0 1 1 0 |
| $I_3$ | 0 0 1 1 |
| $I_4$ | 0 0 0 1 |

时间序列：
- $t=0$：$I_1$ 请求
- $t=1$：$I_3$ 请求(此时CPU在处理 $I_1$)
- $t=2$：$I_2$ 请求(此时CPU在处理 $I_1$)
- $t=3$：$I_1$ 处理完毕

求中断处理的完整执行序列。

**解答过程**：

1. $t=0$：$I_1$ 请求，CPU响应 $I_1$，开始处理
2. $t=1$：$I_3$ 请求。$I_1$ 屏蔽字 = 1100，$I_3$ 对应位 = 0(开放)→ **$I_3$ 打断 $I_1$**
3. CPU响应 $I_3$，开始处理 $I_3$
4. $t=2$：$I_2$ 请求。$I_3$ 屏蔽字 = 0011，$I_2$ 对应位 = 0(开放)→ **$I_2$ 打断 $I_3$**
5. CPU响应 $I_2$，开始处理 $I_2$
6. $I_2$ 的屏蔽字 = 0110，检查是否有可打断的：$I_4$ 对应位=0，但无 $I_4$ 请求
7. $I_2$ 处理完毕，回到 $I_3$ 继续
8. $I_3$ 处理完毕，回到 $I_1$ 继续
9. $I_1$ 处理完毕，回到主程序

**执行序列**：$I_1 \to I_3 \to I_2 \to I_3 \to I_1$ → 主程序

### 考点三：中断处理次序与响应次序不同

**408典型题**：系统有4个中断源 $I_1 \sim I_4$，硬件响应次序为 $I_1 > I_2 > I_3 > I_4$。要求中断处理次序为 $I_4 \to I_3 \to I_2 \to I_1$。设置屏蔽字。

**解答**：

处理次序 $I_4 \to I_3 \to I_2 \to I_1$ 意味着：
- $I_4$ 先处理完，不被任何其他中断打断
- $I_3$ 在 $I_4$ 之后处理，可被 $I_4$ 打断
- $I_2$ 在 $I_3$ 之后，可被 $I_4, I_3$ 打断
- $I_1$ 最后，可被所有其他打断

| 中断源 | 处理顺序 | 屏蔽字(1 2 3 4) | 逻辑 |
|--------|----------|----------------|------|
| $I_1$ | 第4(最后) | 1 0 0 0 | 只屏蔽自身，允许$I_2, I_3, I_4$打断 |
| $I_2$ | 第3 | 1 1 0 0 | 屏蔽$I_1$和自身，允许$I_3, I_4$打断 |
| $I_3$ | 第2 | 1 1 1 0 | 屏蔽$I_1, I_2$和自身，允许$I_4$打断 |
| $I_4$ | 第1(最先) | 1 1 1 1 | 屏蔽所有，不被打断 |

## 直观理解

**核心记忆**：
- **响应次序看硬件**：谁的优先级排线靠前谁先被响应
- **处理次序看屏蔽字**：屏蔽字决定谁能打断谁
- **屏蔽字0位越多**：能被打断的中断源越多，处理越靠后

## 知识关联

### 408常见陷阱

1. **同时请求时按硬件优先级响应**：屏蔽字不影响响应顺序
2. **屏蔽字只影响嵌套**：即处理过程中能否被打断
3. **处理完回到被中断处继续**：不是回到主程序，是继续被中断的中断服务程序
4. **中断返回(IRET)恢复的是被打断点**：多层嵌套时逐层返回

### 易错辨析

| 概念 | 正确理解 | 常见错误 |
|------|----------|----------|
| 响应次序 | 硬件优先级决定 | 认为由屏蔽字决定 |
| 处理次序 | 屏蔽字决定 | 认为等于响应次序 |
| 屏蔽字=1 | 屏蔽(不允许打断) | 认为是开放 |
| IRET | 逐层返回 | 认为直接回主程序 |
