# 16_中断处理过程

## 核心概念

**中断处理过程**：CPU执行中断服务程序(ISR, Interrupt Service Routine)的过程，由**软件(程序员编写)**实现。

### 中断服务程序的完整流程

```
┌──────────────────┐
│ 保护现场         │ ← 软件(PUSH指令)
│ (保存通用寄存器)  │
└────────┬─────────┘
         ↓
┌──────────────────┐
│ 开中断           │ ← 软件(STI指令)
│ (允许更高优先级   │
│  中断嵌套)       │
└────────┬─────────┘
         ↓
┌──────────────────┐
│ 执行IO数据传送   │ ← 软件(IN/OUT指令)
│ (核心任务)       │
└────────┬─────────┘
         ↓
┌──────────────────┐
│ 关中断           │ ← 软件(CLI指令)
│ (保护恢复过程)    │
└────────┬─────────┘
         ↓
┌──────────────────┐
│ 恢复现场         │ ← 软件(POP指令)
│ (恢复通用寄存器)  │
└────────┬─────────┘
         ↓
┌──────────────────┐
│ 中断返回         │ ← 软件(IRET指令)
│ (恢复PC和PSW)    │
└──────────────────┘
```

## 原理分析

### 各步骤详解

#### 步骤1：保护现场

- **操作**：将中断服务程序中要用到的通用寄存器压入堆栈
- **指令**：`PUSH AX`、`PUSH BX`、...
- **目的**：防止中断服务程序破坏被中断程序的寄存器值
- **注意**：PC和PSW已经在中断响应阶段由硬件保存，这里只保存通用寄存器

```assembly
保护现场:
    PUSH AX
    PUSH BX
    PUSH CX
    PUSH DX
    PUSH SI
    PUSH DI
    ; ... 用到哪些就保存哪些
```

#### 步骤2：开中断

- **指令**：`STI` (IE←1)
- **目的**：允许更高优先级的中断打断当前中断处理(中断嵌套)
- **注意**：中断响应时硬件已关中断，这里重新打开
- **条件**：只有需要支持多重中断时才开中断

#### 步骤3：执行IO数据传送(核心)

- **操作**：读取状态、传送数据、清除标志
- **指令**：`IN`/`OUT` 指令

```assembly
    IN AL, STATUS_PORT    ; 读状态
    IN AL, DATA_PORT      ; 读数据
    MOV [BUFFER], AL      ; 存入缓冲区
```

#### 步骤4：关中断

- **指令**：`CLI` (IE←0)
- **目的**：保护恢复过程不被打断
- **如果在步骤2没有开中断，则此步可省略**

#### 步骤5：恢复现场

- **操作**：将堆栈中的通用寄存器值弹出
- **指令**：`POP` 指令，**顺序与PUSH相反**(后进先出)

```assembly
恢复现场:
    POP DI
    POP DI
    POP SI
    POP DX
    POP CX
    POP BX
    POP AX
    ; 注意: POP顺序与PUSH相反
```

#### 步骤6：中断返回

- **指令**：`IRET` (Interrupt Return)
- **操作**：从堆栈弹出PSW和PC，恢复被中断程序的执行
- **等效于**：`POP PSW` + `POP PC`

### 现场保存的分工

| 保存内容 | 保存方式 | 保存者 |
|----------|----------|--------|
| PC(程序计数器) | 硬件自动压栈 | 硬件 |
| PSW(程序状态字) | 硬件自动压栈 | 硬件 |
| 通用寄存器(AX,BX...) | 软件PUSH指令 | 软件 |

- **硬件保存**：PC和PSW(中断响应阶段)
- **软件保存**：通用寄存器(中断服务程序开始)

## 直观理解

**中断处理过程类比**：你正在看书，有人来问路
1. 在书中夹个书签(硬件保存PC/PSW) — 这是中断响应
2. 记住你桌上其他资料的摆放位置(软件保存通用寄存器) — 保护现场
3. 允许其他人再来打断你(开中断) — 如果你愿意被打断
4. 帮他指路(执行IO操作) — 核心任务
5. 再次集中注意力(关中断) — 准备恢复
6. 按记忆恢复桌上资料(软件恢复通用寄存器) — 恢复现场
7. 翻开书签继续看书(IRET恢复PC/PSW) — 中断返回

### 易错陷阱

1. **PC和PSW由硬件保存**：不要写在保护现场的软件代码中
2. **PUSH和POP顺序相反**：先压入的后弹出(堆栈后进先出)
3. **开中断的时机**：必须在保护现场之后开中断，否则现场保存不完整
4. **关中断的时机**：必须在恢复现场之前关中断
## 代码/模拟

### Python模拟中断处理完整过程

```python
"""中断处理过程模拟 - 适用于408考研复习"""

class CPU:
    """简化CPU，模拟中断处理过程"""

    def __init__(self):
        self.PC = 0          # 程序计数器
        self.PSW = 0xF000    # 程序状态字
        self.IE = 1          # 中断允许位
        self.SP = 0xFFFF     # 堆栈指针
        self.regs = {'AX': 0, 'BX': 0, 'CX': 0, 'DX': 0}
        self.stack = []       # 堆栈（简化）

    def push(self, value):
        """压栈操作"""
        self.stack.append(value)

    def pop(self):
        """出栈操作"""
        return self.stack.pop() if self.stack else 0

    def interrupt_response(self, interrupt_pc, isr_addr):
        """
        中断响应（硬件自动完成）
        :param interrupt_pc: 被中断的PC值
        :param isr_addr: 中断服务程序入口地址
        """
        print("【中断响应 - 硬件自动完成】")
        # Step 1: 关中断
        self.IE = 0
        print(f"  1. 关中断: IE ← 0")

        # Step 2: 保存断点（PC和PSW）
        self.push(self.PSW)
        self.push(interrupt_pc)
        print(f"  2. 保存断点: PUSH PSW({self.PSW:#06x}), PUSH PC({interrupt_pc:#06x})")

        # Step 3: 中断向量寻址，找到ISR入口
        self.PC = isr_addr
        print(f"  3. PC ← ISR入口地址 {isr_addr:#06x}")

    def protect_context(self):
        """保护现场（软件/PUSH指令完成）"""
        print("\n【保护现场 - 软件PUSH指令】")
        for reg_name in ['AX', 'BX', 'CX', 'DX']:
            self.push(self.regs[reg_name])
            print(f"  PUSH {reg_name}({self.regs[reg_name]})")

    def enable_interrupt(self):
        """开中断"""
        self.IE = 1
        print(f"  STI: IE ← 1 (允许中断嵌套)")

    def execute_io(self):
        """执行I/O数据传送（核心任务）"""
        print("\n【执行I/O数据传送】")
        data = 0x42  # 假设从设备读入的数据
        print(f"  IN AL, DATA_PORT  ; 读入数据 = {data:#04x}")
        print(f"  MOV [BUFFER], AL  ; 存入缓冲区")

    def disable_interrupt(self):
        """关中断"""
        self.IE = 0
        print(f"  CLI: IE ← 0 (保护恢复过程)")

    def restore_context(self):
        """恢复现场（软件/POP指令完成）"""
        print("\n【恢复现场 - 软件POP指令（与PUSH相反）】")
        for reg_name in reversed(['AX', 'BX', 'CX', 'DX']):
            self.regs[reg_name] = self.pop()
            print(f"  POP {reg_name}({self.regs[reg_name]})")

    def interrupt_return(self):
        """中断返回（IRET指令）"""
        print("\n【中断返回 - IRET指令】")
        old_pc = self.pop()
        old_psw = self.pop()
        self.PC = old_pc
        self.PSW = old_psw
        self.IE = 1
        print(f"  POP PC  → PC ← {old_pc:#06x}")
        print(f"  POP PSW → PSW ← {old_psw:#06x}")
        print(f"  恢复被中断程序的执行")

# 完整中断处理流程模拟
print("=" * 50)
print("中断处理完整流程模拟")
print("=" * 50)

cpu = CPU()
isr_entry = 0x5000  # ISR入口地址
interrupted_pc = 0x1234  # 被中断时的PC

# 硬件：中断响应
cpu.interrupt_response(interrupted_pc, isr_entry)

# 软件：中断服务程序
cpu.protect_context()
cpu.enable_interrupt()

# 核心任务
cpu.execute_io()

# 恢复
cpu.disable_interrupt()
cpu.restore_context()
cpu.interrupt_return()

# 总结
print("\n" + "=" * 50)
print("现场保存分工总结")
print("=" * 50)
print(f"{'保存内容':<12} {'保存方式':<15} {'保存者':<8}")
print("-" * 40)
print(f"{'PC':<12} {'硬件自动压栈':<15} {'硬件':<8}")
print(f"{'PSW':<12} {'硬件自动压栈':<15} {'硬件':<8}")
print(f"{'通用寄存器':<12} {'软件PUSH指令':<15} {'软件':<8}")
```

5. **IRET不是RET**：IRET额外恢复PSW，RET只恢复PC(子程序返回)
