# 16_中断处理过程

## 核心概念

**中断处理过程**：CPU执行中断服务程序(ISR, Interrupt Service Routine)的过程，由**软件(程序员编写)**实现。

### 中断服务程序的完整流程

```
┌──────────────────┐
│ 保护现场         │ ← 软件(PUSH指令)
│ (保存通用寄存器)  │
└────────┬─────────┘
         ↓
┌──────────────────┐
│ 开中断           │ ← 软件(STI指令)
│ (允许更高优先级   │
│  中断嵌套)       │
└────────┬─────────┘
         ↓
┌──────────────────┐
│ 执行IO数据传送   │ ← 软件(IN/OUT指令)
│ (核心任务)       │
└────────┬─────────┘
         ↓
┌──────────────────┐
│ 关中断           │ ← 软件(CLI指令)
│ (保护恢复过程)    │
└────────┬─────────┘
         ↓
┌──────────────────┐
│ 恢复现场         │ ← 软件(POP指令)
│ (恢复通用寄存器)  │
└────────┬─────────┘
         ↓
┌──────────────────┐
│ 中断返回         │ ← 软件(IRET指令)
│ (恢复PC和PSW)    │
└──────────────────┘
```

## 原理分析

### 各步骤详解

#### 步骤1：保护现场

- **操作**：将中断服务程序中要用到的通用寄存器压入堆栈
- **指令**：`PUSH AX`、`PUSH BX`、...
- **目的**：防止中断服务程序破坏被中断程序的寄存器值
- **注意**：PC和PSW已经在中断响应阶段由硬件保存，这里只保存通用寄存器

```assembly
保护现场:
    PUSH AX
    PUSH BX
    PUSH CX
    PUSH DX
    PUSH SI
    PUSH DI
    ; ... 用到哪些就保存哪些
```

#### 步骤2：开中断

- **指令**：`STI` (IE←1)
- **目的**：允许更高优先级的中断打断当前中断处理(中断嵌套)
- **注意**：中断响应时硬件已关中断，这里重新打开
- **条件**：只有需要支持多重中断时才开中断

#### 步骤3：执行IO数据传送(核心)

- **操作**：读取状态、传送数据、清除标志
- **指令**：`IN`/`OUT` 指令

```assembly
    IN AL, STATUS_PORT    ; 读状态
    IN AL, DATA_PORT      ; 读数据
    MOV [BUFFER], AL      ; 存入缓冲区
```

#### 步骤4：关中断

- **指令**：`CLI` (IE←0)
- **目的**：保护恢复过程不被打断
- **如果在步骤2没有开中断，则此步可省略**

#### 步骤5：恢复现场

- **操作**：将堆栈中的通用寄存器值弹出
- **指令**：`POP` 指令，**顺序与PUSH相反**(后进先出)

```assembly
恢复现场:
    POP DI
    POP DI
    POP SI
    POP DX
    POP CX
    POP BX
    POP AX
    ; 注意: POP顺序与PUSH相反
```

#### 步骤6：中断返回

- **指令**：`IRET` (Interrupt Return)
- **操作**：从堆栈弹出PSW和PC，恢复被中断程序的执行
- **等效于**：`POP PSW` + `POP PC`

### 现场保存的分工

| 保存内容 | 保存方式 | 保存者 |
|----------|----------|--------|
| PC(程序计数器) | 硬件自动压栈 | 硬件 |
| PSW(程序状态字) | 硬件自动压栈 | 硬件 |
| 通用寄存器(AX,BX...) | 软件PUSH指令 | 软件 |

- **硬件保存**：PC和PSW(中断响应阶段)
- **软件保存**：通用寄存器(中断服务程序开始)

## 直观理解

**中断处理过程类比**：你正在看书，有人来问路
1. 在书中夹个书签(硬件保存PC/PSW) — 这是中断响应
2. 记住你桌上其他资料的摆放位置(软件保存通用寄存器) — 保护现场
3. 允许其他人再来打断你(开中断) — 如果你愿意被打断
4. 帮他指路(执行IO操作) — 核心任务
5. 再次集中注意力(关中断) — 准备恢复
6. 按记忆恢复桌上资料(软件恢复通用寄存器) — 恢复现场
7. 翻开书签继续看书(IRET恢复PC/PSW) — 中断返回

### 易错陷阱

1. **PC和PSW由硬件保存**：不要写在保护现场的软件代码中
2. **PUSH和POP顺序相反**：先压入的后弹出(堆栈后进先出)
3. **开中断的时机**：必须在保护现场之后开中断，否则现场保存不完整
4. **关中断的时机**：必须在恢复现场之前关中断
5. **IRET不是RET**：IRET额外恢复PSW，RET只恢复PC(子程序返回)
