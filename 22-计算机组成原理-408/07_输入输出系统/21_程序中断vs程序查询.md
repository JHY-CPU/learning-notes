# 22_程序中断vs程序查询

## 核心概念

程序查询和程序中断是两种基本的IO控制方式，408考试中经常以对比选择题的形式出现。

## 原理分析

### 全面对比表

| 对比维度 | 程序查询方式 | 程序中断方式 |
|----------|-------------|-------------|
| **基本思想** | CPU主动轮询设备状态 | 设备就绪后主动通知CPU |
| **CPU角色** | 全程主动参与 | 被动响应，仅中断时参与 |
| **等待期间CPU** | 执行查询循环(空转) | 可执行其他任务 |
| **数据传送** | 经CPU寄存器中转 | 经CPU寄存器中转(同) |
| **每次传送数据量** | 1个数据 | 1个数据(同) |
| **CPU利用率** | 极低 | 显著提高 |
| **硬件复杂度** | 最低(只需状态寄存器) | 中等(需中断控制器) |
| **响应速度** | 取决于查询频率 | 设备就绪立即通知 |
| **中断开销** | 无 | 有(响应+保护+恢复+返回) |
| **适用设备** | 低速设备 | 低速~中速设备 |
| **编程复杂度** | 最简单 | 中等(需编写ISR) |
| **并行性** | 无(CPU和设备串行) | 部分并行(CPU可做其他事) |
| **实时性** | 差(最坏等一个查询周期) | 好(就绪立即响应) |

### CPU利用率对比分析

**场景**：设备每100ms准备好一个数据，传送需1μs，查询需0.5μs。

**程序查询方式**：
- CPU 100%时间在查询循环中
- 有效工作仅1μs/100ms = 0.001%
- CPU利用率(有效)极低

**程序中断方式**：
- 设备就绪前CPU执行其他任务
- 中断开销约20μs(响应5μs + 保护5μs + 传送1μs + 恢复5μs + 返回4μs)
- CPU用于IO的有效开销 = 20μs/100ms = 0.02%
- CPU利用率(可用于其他任务) = 99.98%

### 适用场景分析

| 场景 | 推荐方式 | 原因 |
|------|----------|------|
| 键盘输入 | 程序中断 | 速度慢，等待时间长 |
| 简单LED显示 | 程序查询 | 极简系统，无需中断 |
| 串口通信(低速) | 程序中断 | 数据到达时间不确定 |
| 嵌入式简单传感器 | 程序查询 | 硬件简单，成本低 |
| 磁盘高速传输 | DMA | 中断开销太大 |

## 直观理解

**类比对比**：

**程序查询** = 你每隔5分钟查看快递柜
- 不停跑下楼查看(空转)
- 发现没到再回来(继续查询)
- 终于到了取走(传送)
- 全程你都在关注这件事

**程序中断** = 快递到了发短信通知
- 你安心工作(执行其他任务)
- 短信来了(中断请求)
- 你下楼取件(中断服务程序)
- 取完回来继续工作(中断返回)

## 知识关联

### 408典型选择题

**题1**：以下关于程序查询方式和程序中断方式的叙述，正确的是？
- A. 两者CPU利用率相同 → 错
- B. 两者数据都经过CPU → 对(这是与DMA的关键区别)
- C. 程序中断不需要状态寄存器 → 错(仍需状态寄存器触发中断)
- D. 程序查询方式响应速度更快 → 错(中断方式更快)

**题2**：程序中断方式相比程序查询方式的优点不包括？
- A. CPU利用率更高 → 是优点
- B. 响应速度更快 → 是优点
- C. 每次传送的数据量更大 → **不是优点**(都是每次一个数据)
- D. CPU可以并行处理其他任务 → 是优点

### 易错陷阱

1. **两者数据都经过CPU**：程序中断方式的数据仍由CPU传送，与DMA的关键区别
2. **中断方式也有状态寄存器**：不是替代状态寄存器，而是用中断请求代替CPU轮询
3. **中断不等于更高速度**：中断方式的数据传送速率不一定更快，只是CPU利用率更高
4. **中断不适合高速设备**：频繁中断的开销会压垮CPU，高速设备应使用DMA
