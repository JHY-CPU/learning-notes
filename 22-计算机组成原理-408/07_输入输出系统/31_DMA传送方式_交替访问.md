# 31_DMA传送方式_交替访问

## 核心概念

- **交替访问方式**：将一个存取周期分为两半，前半部分给CPU，后半部分给DMA，交替使用
- **核心思想**：时间分割，CPU和DMA各用一半存取周期
- **要求**：需要双端口存储器或高速存储器支持

## 原理分析

### 工作原理

```
一个存取周期 T:
├──── T/2 ────┤──── T/2 ────┤
│  CPU访存     │  DMA访存     │
│  (前半周期)  │  (后半周期)  │
```

CPU和DMA在一个存取周期内各访问一次存储器，互不干扰。

### 交替访问的实现方式

**方式1：分时复用**
- 存储器速度足够快(是CPU的2倍)
- 前半周期响应CPU请求
- 后半周期响应DMA请求

**方式2：双端口存储器**
- 存储器有两个独立的访问端口
- CPU用端口A，DMA用端口B
- 两个端口可以同时访问(真正的并行)

```
        CPU端口(A)
            │
    ┌───────┴───────┐
    │               │
    │   双端口      │
    │   存储器      │
    │               │
    └───────┬───────┘
            │
        DMA端口(B)
```

### 交替访问的特点

| 特征 | 说明 |
|------|------|
| CPU暂停 | 不暂停(CPU和DMA同时工作) |
| 存储器要求 | 存储器速度需为CPU的2倍(分时)或双端口 |
| 传输效率 | 较低(每个设备只用一半带宽) |
| 数据传输率 | 较低(DMA只能用一半存储器带宽) |
| 硬件复杂度 | 最高(需要双端口或高速存储器) |
| CPU利用率 | 最高(完全不影响CPU) |

### 带宽分析

设存储器存取周期为 $T_m$，存储器数据宽度为 $W$ 位。

**存储器总带宽**：

$$B_{\text{total}} = \frac{W}{T_m}$$

**DMA可用带宽**：

$$B_{\text{DMA}} = \frac{W}{2 \times T_m} = \frac{1}{2} B_{\text{total}}$$

**CPU可用带宽**：

$$B_{\text{CPU}} = \frac{W}{2 \times T_m} = \frac{1}{2} B_{\text{total}}$$

各用一半，互不干扰。

## 直观理解

**交替访问类比**：单行道上两方向交替通行
- 道路(存储器)只有一个
- 正方向车流(CPU)和反方向车流(DMA)各走一半时间
- 互不干扰，但每方向的通行能力减半
- 需要信号灯(时序控制)协调

**双端口类比**：房子有两个门
- 前门给主人(CPU)进出
- 后门给快递(DMA)进出
- 两人可以同时进出，互不影响

## 知识关联

### 三种方式总结

| 方式 | CPU暂停 | DMA可用带宽 | 存储器利用率 | 实现成本 |
|------|---------|------------|-------------|---------|
| 停止CPU | 长 | 100% | 100%(DMA用) | 低 |
| 周期挪用 | 短(1个周期) | 看空闲情况 | 高 | 中 |
| 交替访问 | 无 | 50% | 100%(分时) | 高 |

### 易错陷阱

1. **交替访问≠DMA和CPU完全并行**：分时方式仍需串行访问，只是间隔极短
2. **存储器带宽是瓶颈**：交替方式下每个设备只用一半
3. **双端口是真正的并行**：分时复用是伪并行
4. **实际系统中交替方式较少用**：硬件成本高，周期挪用更实用
