# 05_IO端口_统一编址vs独立

## 核心概念

IO端口有两种编址方式，核心区别在于**IO端口地址是否占用主存地址空间**。

### 统一编址(存储器映射IO, Memory-Mapped IO)

- **定义**：IO端口和主存单元统一编址，共享同一个地址空间
- **地址分配**：主存地址空间的一部分分配给IO端口
  - 例：主存地址空间4GB，其中低3GB给主存，高1GB给IO端口
- **访问指令**：使用通用的数据传送指令(MOV、LOAD、STORE)访问IO端口
- **优点**：
  - 不需要专门的IO指令，指令系统简化
  - 可以用所有访存指令操作IO端口(加减、比较等)
  - 端口数量几乎不受限制
- **缺点**：
  - 占用主存地址空间，减少了可用主存容量
  - 地址译码电路需要区分存储器访问和IO访问
  - 不易从指令形式上区分是访存还是访IO

### 独立编址(Isolated IO / Port-Mapped IO)

- **定义**：IO端口有独立的地址空间，与主存地址空间分离
- **地址分配**：IO地址空间独立，通常较小(如64KB = $2^{16}$)
- **访问指令**：使用专门的IO指令(IN/OUT)
  - `IN AL, port` — 从端口读数据到AL
  - `OUT port, AL` — 将AL数据写到端口
- **优点**：
  - 不占用主存地址空间
  - IO指令短，执行快
  - 从指令形式上可以明确区分访存和访IO
  - 程序可读性好
- **缺点**：
  - 需要专用IO指令，增加了指令系统的复杂性
  - IO指令功能较简单，不如访存指令灵活
  - IO地址空间有限

## 原理分析

### 地址空间对比

```
统一编址(假设地址总线32位):
┌──────────────────────────────────────┐ 0xFFFFFFFF
│         IO端口地址空间                │
├──────────────────────────────────────┤
│         主存地址空间                  │
└──────────────────────────────────────┘ 0x00000000
两者共享4GB地址空间，无重叠

独立编址(两个独立空间):
主存空间(32位):                   IO空间(16位):
┌──────────────────┐             ┌──────────┐
│  主存 4GB        │             │ IO 64KB  │
│  0x00000000      │             │ 0x0000   │
│  ~               │             │ ~        │
│  0xFFFFFFFF      │             │ 0xFFFF   │
└──────────────────┘             └──────────┘
两个完全独立的地址空间
```

### 访问指令对比

| 操作 | 统一编址 | 独立编址 |
|------|----------|----------|
| 从IO端口读数据 | `MOV AX, [IO地址]` | `IN AX, port` |
| 向IO端口写数据 | `MOV [IO地址], AX` | `OUT port, AX` |
| 比较IO端口值 | `CMP [IO地址], 10` | `IN AL, port` + `CMP AL, 10` |
| 地址表示 | 主存地址的一部分 | 独立的IO地址 |

### 典型处理器的编址方式

| 处理器 | 编址方式 | 说明 |
|--------|----------|------|
| Intel x86 | **独立编址**为主 | 使用IN/OUT指令，IO空间64KB，但也支持存储器映射IO |
| ARM | **统一编址** | 没有IN/OUT指令，所有IO端口映射到存储器地址空间 |
| MIPS | **统一编址** | IO端口映射到特定的存储器地址段 |
| PowerPC | 两者都支持 | 可以配置使用哪种方式 |

## 直观理解

**统一编址的类比**：一个大仓库，一半放货物(主存)，一半放工具(IO端口)
- 进出仓库的规则相同，只是位置不同
- 使用同一套登记表(MOV指令)记录进出

**独立编址的类比**：两个仓库，一个放货物(主存)，一个放工具(IO端口)
- 两个仓库有各自的编号系统
- 进出货物用"存/取"指令(MOV)，进出工具用"领/还"指令(IN/OUT)

**记忆技巧**：
- **统一编址** = "统一管理" —— 同一地址空间、同一套指令
- **独立编址** = "分而治之" —— 两个地址空间、两套指令

### 易错陷阱

1. **统一编址不是把IO放在Cache中**：Cache通常只缓存主存数据，IO端口映射的区域一般标记为不可缓存(uncacheable)
2. **x86处理器两种方式都支持**：虽然以独立编址为主，但PCIe设备的MMIO也属于统一编址
3. **统一编址也需要地址译码**：需要区分是访问主存还是IO端口
4. **端口地址与存储器地址无关**：独立编址中，IO地址0x3F8和存储器地址0x3F8是两个完全不同的位置
