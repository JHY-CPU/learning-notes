# 42_键盘接口

## 核心概念

- **键盘**：最基本的输入设备，将按键操作转换为计算机可识别的编码
- **键盘接口**：连接键盘与主机的桥梁，负责按键检测和编码转换

### 键盘工作原理

**按键检测——行列扫描法**：
- 键盘矩阵：$m$ 行 $\times$ $n$ 列的按键矩阵
- 扫描过程：
  1. 依次给每行送低电平(扫描行)
  2. 检测每列的电平(检测列)
  3. 行列交叉点即为被按下的键
- 按键数 = $m \times n$，只需 $m + n$ 条线

```
        列0  列1  列2  列3
行0 ──── [0]  [1]  [2]  [3]
行1 ──── [4]  [5]  [6]  [7]
行2 ──── [8]  [9]  [A]  [B]
行3 ──── [C]  [D]  [E]  [F]

扫描: 行0送低电平 → 检测列0~3
      行1送低电平 → 检测列0~3
      ...
```

### 键盘控制器

- **键盘控制器(Keyboard Controller)**：独立的微处理器(如8048)
- **功能**：
  - 执行行列扫描
  - 去抖动(消除按键抖动，约5~20ms)
  - 生成扫描码(每个键唯一的编码)
  - 通过接口向主机发送扫描码
  - 检测按键释放

### 扫描码(Scan Code)

- 每个按键对应唯一的扫描码(按下码+释放码)
- 扫描码 ≠ ASCII码，需由操作系统转换
- **按下时**：发送按下扫描码
- **释放时**：发送释放扫描码(通常 = 按下码 + 0x80)

### 键盘中断

```
按键按下 → 键盘控制器检测 → 生成扫描码
    → 向主机发中断请求(IRQ1)
    → CPU响应中断
    → 中断服务程序读取扫描码(从端口0x60)
    → 转换为ASCII码
    → 存入键盘缓冲区
```

## 原理分析

### 键盘接口的端口

| 端口地址 | 读/写 | 功能 |
|----------|-------|------|
| 0x60 | 读 | 读取扫描码(数据端口) |
| 0x64 | 读 | 读取状态(状态寄存器) |
| 0x64 | 写 | 发送命令(控制寄存器) |

### 去抖动处理

按键机械触点会产生抖动(按下和释放瞬间多次通断)：
- 抖动时间：5~20ms
- 解决方法：
  - **硬件去抖**：RC滤波电路
  - **软件去抖**：检测到按下后延时20ms再确认

## 直观理解

**键盘工作类比**：电话键盘的自动拨号系统
- 行列扫描 = 逐行检查哪个键被按下
- 扫描码 = 每个数字的内部编码
- 去抖动 = 按键后等一下确认不是误触
- 中断 = 按键后通知CPU"有新输入"

### 408常考

1. **键盘用中断方式**：不是查询方式(408常考)
2. **扫描码≠ASCII码**：需要软件转换
3. **行列扫描法的线数**：$m \times n$ 键盘需要 $m+n$ 条线
4. **去抖动是必要的**：机械按键有抖动

### 易错陷阱

1. **键盘不用DMA**：数据量太小，每次只一个字节
2. **扫描码由键盘控制器生成**：不是由CPU生成
3. **中断请求线是IRQ1**：在PC中键盘固定使用IRQ1
