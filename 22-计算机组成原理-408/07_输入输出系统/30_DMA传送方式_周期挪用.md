# 30_DMA传送方式_周期挪用

## 核心概念

- **周期挪用(周期窃取, Cycle Stealing)**：DMA在CPU不访存的存取周期内"借用"一个存取周期来传送一个数据
- **核心思想**：利用CPU访存的空闲时间，不完全停止CPU
- **最常用的DMA传送方式**

## 原理分析

### 工作原理

```
CPU指令周期: 取指 → 执行 → 取指 → 执行 → ...
               ↓              ↓
           DMA挪用这一个周期   DMA挪用这一个周期
           传送一个数据        传送一个数据
```

**关键条件**：CPU在执行指令时不总是访问主存
- 取指阶段需要访存
- 执行阶段可能不需要访存(如纯运算指令)
- DMA利用CPU不访存的间隙挪用一个存取周期

### 周期挪用的过程

1. DMA检测到CPU当前不访存(或有空闲存取周期)
2. DMA向CPU发总线请求
3. CPU响应(让出一个存取周期)
4. DMA在一个存取周期内传送一个数据
5. DMA释放总线
6. CPU继续工作

```
时间轴:
├── CPU取指 ──┤── CPU执行(不访存) ──┤── CPU取指 ──┤
              │← DMA挪用一个周期 →│              │
              传1个数据            总线归还CPU
```

### 如果CPU正在访存怎么办

- DMA必须等待当前存取周期结束
- DMA请求可能被延迟几个时钟周期
- **DMA优先级通常高于CPU**：当DMA和CPU同时请求访存时，DMA优先

### 周期挪用的特点

| 特征 | 说明 |
|------|------|
| CPU暂停时间 | 极短(每次一个存取周期) |
| 数据传送 | 每次挪用传送1个字 |
| CPU影响 | 轻微(偶尔延迟一个存取周期) |
| 传输效率 | 中等(受CPU访存模式影响) |
| 实现复杂度 | 中等(需要检测CPU访存状态) |
| 适用场景 | 最通用，大多数系统采用 |

### 周期挪用次数计算

设每个指令周期中CPU有 $m$ 个存取周期(取指+访存)，其中 $k$ 个是空闲的(不访存)。

每条指令可以挪用 $k$ 个周期。

若每秒执行 $f_{\text{inst}}$ 条指令：

$$\text{每秒可挪用周期数} = f_{\text{inst}} \times k$$

$$\text{DMA可用带宽} = f_{\text{inst}} \times k \times W \quad (\text{bit/s})$$

其中 $W$ 为总线宽度。

## 直观理解

**周期挪用类比**：红灯时过马路
- 马路上车流(CPU指令流)不断
- 红灯时车停(CPU不访存的存取周期)
- 行人(DMA)趁红灯间隙快速通过(挪用一个周期)
- 绿灯一亮车流继续(CPU继续执行)
- 行人不影响正常交通(DMA不影响CPU执行)

## 知识关联

### 408常考计算

**例题**：某系统CPU主频500MHz，平均每条指令有2个存取周期(取指+访存)，每个存取周期1个时钟周期。每条指令平均有0.5个空闲存取周期(不访存)。总线宽度32位。求DMA的可用最大带宽。

**解答**：
- 每秒执行指令数 = $500MHz / 2 = 250M$ 条指令
- 每秒可挪用周期数 = $250M \times 0.5 = 125M$ 个
- DMA可用带宽 = $125M \times 4B = 500MB/s$

### 与其他方式对比

| 对比项 | 停止CPU | 周期挪用 | 交替访问 |
|--------|---------|----------|----------|
| CPU暂停时间 | 整个传送期间 | 每次一个周期 | 不暂停 |
| 传输效率 | 最高 | 中等 | 较低 |
| CPU利用率影响 | 大 | 小 | 无 |
| 硬件复杂度 | 简单 | 中等 | 复杂 |

### 易错陷阱

1. **周期挪用不等于完全不影响CPU**：CPU可能会被延迟一个存取周期
2. **"空闲周期"的定义**：CPU执行指令时不一定访存(如ALU运算)
3. **DMA优先级高于CPU**：同时请求时DMA优先
4. **每次挪用只传一个字**：不是挪用一次传一批
