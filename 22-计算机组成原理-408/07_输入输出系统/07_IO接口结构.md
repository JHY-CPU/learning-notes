# 07_IO接口结构

## 核心概念

- **IO接口**是连接系统总线与IO设备的桥梁，内部包含多个功能模块
- 接口结构 = **数据寄存器** + **状态寄存器** + **控制寄存器** + **地址译码电路** + **中断控制逻辑** + **数据缓冲器**

### 接口内部典型结构图

```
          系统总线

    │  数据总线DB  地址总线AB  控制总线CB │
```

## 原理分析

### 各功能模块详解

#### 1. 数据缓冲器

- **数据输入寄存器(DIR)**：
  - 方向：设备 → CPU
  - 设备将数据写入DIR → CPU从DIR读取数据
  - 读DIR时产生"数据已被取走"信号，设备可准备下一个数据
- **数据输出寄存器(DOR)**：
  - 方向：CPU → 设备
  - CPU将数据写入DOR → 设备从DOR取走数据
  - 写DOR后设备自动开始处理
- **三态缓冲器**：确保DIR在未被选中时输出高阻态，不干扰总线

#### 2. 地址译码电路

- **功能**：接收地址总线信号，判断本接口是否被选中
- **组成**：
  - 高位地址比较器(片选)：将地址总线高位与本接口基地址比较
  - 低位地址译码(片内)：从多个端口中选择一个
- **输出**：片选信号CS + 端口选择信号

```
地址总线 A15~A0
A15~A2 → 与基地址比较 → CS(片选)
A1, A0 → 2-4译码器 → 选择4个端口之一(00=DIR, 01=DOR, 10=SR, 11=CR)
```

#### 3. 状态寄存器(SR)

- CPU读取，了解设备当前状态
- 典型位定义：

| 位 | 名称 | 含义 |
|----|------|------|
| D0 | READY | 1=设备就绪 |
| D1 | BUSY | 1=设备忙 |
| D2 | DONE | 1=操作完成 |
| D3 | ERROR | 1=出错 |
| D7~D4 | 保留 | 扩展用 |

#### 4. 控制寄存器(CR)

- CPU写入，向设备发出控制命令
- 典型位定义：

| 位 | 名称 | 含义 |
|----|------|------|
| D0 | GO | 1=启动设备 |
| D1 | IE | 1=允许中断 |
| D2 | DIR | 1=输入, 0=输出 |
| D7~D3 | 保留 | 扩展用 |

#### 5. 中断控制逻辑

- **中断请求触发器**：当设备就绪或出错时置1，向CPU发出中断请求INTR
- **中断屏蔽触发器**：控制是否允许发出中断请求
  - 屏蔽触发器=1：即使设备就绪也不发中断(屏蔽)
  - 屏蔽触发器=0：设备就绪时发中断(开放)
- **中断判优逻辑**：多个中断源同时请求时的优先级仲裁

### 端口地址映射表(实例)

以某串口接口为例，基地址=0x3F8：

| 偏移 | 地址 | 端口 | 方向 | 说明 |
|------|------|------|------|------|
| +0 | 0x3F8 | 数据寄存器 | 读/写 | 发送/接收数据缓冲 |
| +1 | 0x3F9 | 中断使能寄存器 | 写 | 设置中断允许位 |
| +2 | 0x3FA | 中断标志寄存器 | 读 | 查看中断状态 |
| +3 | 0x3FB | 线路控制寄存器 | 写 | 设置数据格式 |
| +4 | 0x3FC | Modem控制寄存器 | 写 | 控制调制解调器 |
| +5 | 0x3FD | 线路状态寄存器 | 读 | 查看传输状态 |

## 直观理解

**接口内部结构就像一个"办事大厅"**：
- 数据寄存器 = 服务窗口(传递文件)
- 状态寄存器 = 电子公告牌(显示"正在办理/空闲")
- 控制寄存器 = 任务指示牌(领导下达的指令)
- 地址译码 = 门禁系统(确认来者是否找对房间)
- 中断控制逻辑 = 铃铛(事情办完了按铃通知)

### 跨章节联系

| 知识点 | 联系 |
|--------|------|
| **总线三态门(第6章)** | 数据缓冲器的三态输出确保不干扰总线 |

### 易错陷阱

1. **DIR和DOR是两个不同的寄存器**，即使它们可能共享同一端口地址(读DIR，写DOR)
2. **状态寄存器只读**：CPU不能写入状态寄存器(硬件自动更新状态)
3. **控制寄存器只写**：CPU不能读控制寄存器(硬件不提供读回通路，但某些芯片例外)
4. **中断屏蔽触发器在接口内**：与CPU的中断允许触发器(IE)不同，接口的屏蔽是局部的
