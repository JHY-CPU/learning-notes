# 25_存储器层次概述

## 核心概念

### 存储器的层次结构

现代计算机采用多级存储器层次结构，从速度快到慢、容量小到大依次排列：

```
       ┌──────────┐
       │ 寄存器   │  ← 速度最快，容量最小，价格最高
       ├──────────┤
       │ L1 Cache │
       ├──────────┤
       │ L2 Cache │
       ├──────────┤
       │ L3 Cache │  （部分CPU有）
       ├──────────┤
       │ 主存     │  ← DRAM，容量较大
       ├──────────┤
       │ 辅存     │  ← SSD/HDD，容量最大，速度最慢，价格最低
       └──────────┘
```

### 各层存储器特征对比

| 层次 | 速度 | 容量 | 价格(元/GB) | 介质 |
|------|------|------|------------|------|
| 寄存器 | ~0.3ns | 几十~几百B | 非常高 | SRAM |
| L1 Cache | ~1ns | 32-64KB | 非常高 | SRAM |
| L2 Cache | ~3-10ns | 256KB-1MB | 高 | SRAM |
| L3 Cache | ~10-20ns | 4-64MB | 较高 | SRAM |
| 主存 | ~50-100ns | 4-64GB | 中等 | DRAM |
| SSD | ~100us | 256GB-4TB | 较低 | 闪存 |
| HDD | ~5-10ms | 1-16TB | 最低 | 磁盘 |

### 速度-容量-价格的Trade-off

$$
\text{速度快} \Leftrightarrow \text{容量小} \Leftrightarrow \text{价格高}
$$

这是一个基本的工程权衡。更快的存储技术通常每个存储单元需要更多的晶体管，因此容量更小、价格更高。

## 原理分析

### 局部性原理

存储器层次结构之所以有效，基于程序执行的**局部性原理**：

**时间局部性（Temporal Locality）**：
- 最近被访问的数据很可能再次被访问
- 例如：循环中的变量、重复调用的函数

**空间局部性（Spatial Locality）**：
- 访问某个地址后，很可能访问其附近的地址
- 例如：数组的顺序遍历、指令的顺序执行

$$
\text{有效访问时间} = H \times T_1 + (1-H) \times T_2
$$

其中：
- $H$：命中率（在上层找到数据的概率）
- $T_1$：上层访问时间
- $T_2$：下层访问时间
- $(1-H)$：缺失率

### 平均访问时间计算

**例题**：某系统Cache命中率为95%，Cache访问时间1ns，主存访问时间50ns。求平均访问时间。

$$
T_{\text{avg}} = 0.95 \times 1 + 0.05 \times (1 + 50) = 0.95 + 2.55 = 3.5\text{ns}
$$

（注意：Cache缺失时，需要先访问Cache（发现缺失），再访问主存，所以是 $1+50=51$ns）

如果命中率降低到80%：
$$
T_{\text{avg}} = 0.80 \times 1 + 0.20 \times (1 + 50) = 0.8 + 10.2 = 11\text{ns}
$$

命中率从95%降到80%，平均访问时间从3.5ns增加到11ns，增加了3倍多！

## 直观理解

### 生活类比

存储器层次就像你的**书桌和书架**：
- **桌面（寄存器）**：当前正在看的书，随手可得
- **书架第一层（L1 Cache）**：经常用的参考书，转身就能拿到
- **书架其他层（L2/L3 Cache）**：偶尔用的书，需要找一下
- **书房（主存）**：不常用但需要的书，走几步去拿
- **图书馆（辅存）**：大部分书都在这里，需要专门去借

速度递减、容量递增、价格递减

### 易错陷阱

1. **Cache和主存的内容是重叠的**：Cache是主存的副本
2. **虚拟存储器的地址转换**：逻辑地址 → 物理地址
3. **辅存不属于主存层次**：辅存通过虚拟存储器技术与主存交互
4. **命中率对性能的影响是非线性的**：命中率从95%到99%的提升比从80%到85%更有价值
