# 12_指令执行过程概述

## 核心概念

### 指令格式

一条指令通常由两部分组成：

```
| 操作码（OP） | 地址码（A） |
|--------------|-------------|
| 指明操作类型 | 指明操作数的位置 |
```

- **操作码（Opcode）**：规定要执行的操作类型，如加法、减法、传送、跳转等
- **地址码（Address Code）**：给出操作数的地址或操作数本身

### 指令执行的四个周期

完整的指令执行过程可以分为四个机器周期：

| 周期 | 名称 | 说明 |
|------|------|------|
| 取指周期 | Fetch | 从主存取指令到IR |
| 间址周期 | Indirect | 如果有间接寻址，需额外访存取操作数地址 |
| 执行周期 | Execute | 执行指令规定的操作 |
| 中断周期 | Interrupt | 检查是否有中断请求，若有则响应 |

$$
\text{指令周期} = \text{取指周期} + \text{间址周期（可能为0）} + \text{执行周期} + \text{中断周期（可能为0）}
$$

## 原理分析

### 取指周期详解

```
时间节拍    微操作                    说明
T0         (PC) → MAR               PC的内容送MAR
T1         M(MAR) → MDR; (PC)+1→PC  读指令，PC自增
T2         (MDR) → IR               指令送IR
```

取指周期完成后，IR中存放着当前要执行的指令，PC已指向下一条指令。

### 间址周期详解

仅当指令采用**间接寻址**时才需要间址周期：

```
时间节拍    微操作                    说明
T0         Ad(IR) → MAR             指令中的地址码送MAR
T1         M(MAR) → MDR             从主存读出操作数的地址
T2         (MDR) → Ad(IR)           将有效地址写回指令的地址字段
```

间址可能需要多次（多次间接寻址），直到得到最终的有效地址。

### 执行周期详解

执行周期的操作取决于指令类型，以下是几种典型指令的执行过程：

**加法指令 ADD R0, R1**：
```
T0: (R1) → ALU的输入
T1: (R0) + (R1) → R0
```

**访存指令 LOAD R0, [addr]**：
```
T0: Ad(IR) → MAR
T1: M(MAR) → MDR
T2: (MDR) → R0
```

**跳转指令 JMP addr**：
```
T0: Ad(IR) → PC
```

### 中断周期详解

每条指令执行完后，CPU检查是否有中断请求：

```
时间节拍    微操作                      说明
T0         保存断点（SP）→ MAR          保存返回地址（PC的值）
T1         (PC) → MDR; MDR → M(MAR)    PC（断点）压入堆栈
T2         中断服务程序入口地址 → PC     PC指向中断处理程序
```

如果有中断请求，CPU暂停当前程序，转去执行中断服务程序。处理完中断后，再返回被中断的程序继续执行。

## 直观理解

### 指令执行过程类比

四周期就像一个完整的"任务循环"：
1. **取指令**：从笔记本上看下一步要做什么
2. **间址**：如果写的地址不够详细，再查地图找到确切位置
3. **执行**：实际完成这个任务
4. **中断处理**：完成任务后，看看有没有紧急事件需要处理

### 数据流图

```
┌──────┐    ┌──────┐    ┌──────┐
│ 取指令 │───→│ 分析 │───→│ 执行 │
└──────┘    └──────┘    └──────┘
                              │
                    ┌─────────┴─────────┐
                    │                    │
               有中断请求            无中断请求
                    │                    │
              ┌─────┴─────┐         ┌─────┴─────┐
              │ 保存断点   │         │ 取下一条   │
              │ 转中断处理 │         │ 指令       │
              └───────────┘         └───────────┘
```

### 常考组合

1. 四个周期的顺序和各自的工作
2. 间址周期只在间接寻址时存在
3. 中断周期在每条指令结束后检查
4. 取指周期中PC自增的时机

### 易错陷阱

1. **间址周期不是每个指令都有**：只有间接寻址指令才有
2. **中断周期不是每个指令都会触发**：只有有中断请求时才执行
3. **执行周期的操作因指令不同而不同**
4. **一条指令的执行可能涉及多次访存**：取指令、间址、取操作数、存结果
