# 28_固件概念

## 核心概念

### 固件的定义

**固件（Firmware）**：固化在只读存储器（ROM）中的软件。它是介于硬件和软件之间的一种形态。

$$
\text{固件} = \text{软件} + \text{ROM硬件}
$$

固件的特点：
- 以软件的形式存在（由程序代码组成）
- 以硬件的方式运行（固化在ROM芯片中）
- 掉电不丢失（非易失性）
- 一般不允许用户修改（只读或需特殊方式改写）

### 固件的层次

在计算机系统层次结构中，固件处于特殊位置：

```
应用软件
   ↓
系统软件（操作系统等）
   ↓
固件（BIOS/UEFI、微程序）  ← 介于软件和硬件之间
   ↓
硬件
```

### 典型固件

| 固件类型 | 功能 | 存储位置 |
|---------|------|---------|
| BIOS/UEFI | 系统启动、硬件自检、加载操作系统 | 主板上的ROM/Flash |
| 微程序 | 解释机器指令，控制CPU内部操作 | 控制存储器（CPU内部） |
| 设备固件 | 控制设备的底层操作 | 设备内部的ROM |
| 嵌入式程序 | 嵌入式系统的主控程序 | Flash/EEPROM |

## 原理分析

### BIOS的工作过程

**BIOS（Basic Input/Output System）**：基本输入输出系统。

计算机启动时的BIOS工作流程：
1. **POST（Power-On Self-Test）**：加电自检，检查硬件是否正常
2. **初始化硬件**：设置中断向量表、初始化各硬件设备
3. **加载引导程序**：从硬盘/SSD的引导扇区读取引导程序
4. **移交控制权**：将控制权交给引导程序，进而加载操作系统

### 微程序

**微程序**是一种用程序来实现机器指令的方法：
- 每条机器指令对应一段微程序
- 微程序由一系列微指令组成
- 微指令直接控制CPU内部的数据通路

```
机器指令（如ADD）
    ↓ 对应
微程序
    ↓ 包含
微指令1 → 微指令2 → 微指令3 → ...
    ↓
微操作（直接控制硬件的控制信号）
```

微程序存放在**控制存储器（CM）** 中，CM是一种ROM。

### 固件与软件、硬件的关系

| 对比维度 | 硬件 | 固件 | 软件 |
|---------|------|------|------|
| 存在形式 | 物理器件 | ROM中的代码 | 磁盘/内存中的代码 |
| 运行速度 | 最快 | 快 | 较慢 |
| 可修改性 | 不可修改 | 难修改（刷写） | 容易修改 |
| 掉电保存 | — | 保存 | 需要外存保存 |
| 开发成本 | 高 | 中 | 低 |

## 直观理解

### 生活类比

**固件就像"刻在石板上的法律"**：
- 普通法律（软件）可以随时修改、打印
- 宪法（固件）刻在石碑上，修改需要特殊程序
- 物理定律（硬件）是自然界的规则，无法改变

或者类比为**自动售货机的程序**：
- 售货机内部的控制程序（固件）出厂时就烧录好了
- 用户不能修改，但厂家可以升级
- 它不像手机App那样可以随意安装卸载

### 记忆技巧

固件：「固化的软件」
- 本质是软件
- 但像硬件一样"固"定在ROM中

### 易错陷阱

1. **固件不是硬件**：它是写入ROM的软件
2. **固件不是普通软件**：它运行在硬件之上，直接控制硬件
3. **微程序存放在控制存储器（CM）中**，不是主存
4. **BIOS/UEFI是固件**，但用户安装的操作系统不是固件
