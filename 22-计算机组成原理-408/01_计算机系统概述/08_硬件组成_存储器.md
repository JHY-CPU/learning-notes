# 08_硬件组成_存储器

## 核心概念

### 主存储器的基本结构

主存储器（主存）是计算机中存放当前运行程序和数据的存储部件。CPU可以直接访问主存。

| 组成部分 | 功能 |
|---------|------|
| 存储体 | 存放数据和指令的核心阵列，由大量存储单元组成 |
| MAR（存储地址寄存器） | 存放要访问的存储单元地址 |
| MDR（存储数据寄存器） | 存放从存储器读出或要写入存储器的数据 |
| 读写控制电路 | 根据控制信号完成读/写操作 |
| 地址译码器 | 将MAR中的地址译码，选中对应的存储单元 |

### 关键术语辨析

| 术语 | 定义 | 说明 |
|------|------|------|
| 存储单元 | 存放一个机器字的存储空间 | 每个存储单元有唯一的地址 |
| 存储字 | 存储单元中存放的二进制代码 | 一个存储字 = 一个存储单元的内容 |
| 存储字长 | 一个存储单元的二进制位数 | 如8位、16位、32位、64位 |
| 存储容量 | 存储器能存储的二进制信息总量 | = 存储单元数 × 存储字长 |
| 存储元 | 存放一位二进制信息的物理器件 | 最小的存储单位（1 bit） |
| 按字节编址 | 每个字节（8位）分配一个地址 | 现代计算机的主流方式 |
| 按字编址 | 每个存储字分配一个地址 | 早期计算机使用 |

## 原理分析

### MAR和MDR的位数

**MAR的位数决定了寻址空间**：
$$
\text{最大寻址范围} = 2^{\text{MAR位数}} \text{ 个存储单元}
$$

例如：MAR为16位 → 可寻址 $2^{16} = 65536$ 个存储单元

**MDR的位数 = 存储字长**：
- MDR位数决定了每次读写的数据宽度
- 如果存储字长为32位，则MDR也为32位

### 存储容量计算

$$
\text{存储容量} = \text{存储单元数} \times \text{存储字长} = 2^{\text{MAR位数}} \times \text{MDR位数}
$$

**例题**：某计算机MAR为20位，MDR为32位，求主存容量。
$$
\text{容量} = 2^{20} \times 32\text{位} = 1\text{M} \times 4\text{B} = 4\text{MB}
$$

### 按字节编址下的地址空间

现代计算机通常按字节编址：
- MAR的位数决定字节数
- 如果MAR为n位，可寻址 $2^n$ 个字节

**例题**：MAR为24位，按字节编址，存储字长32位，求：
1. 最大寻址空间 = $2^{24}$ 字节 = 16MB
2. 存储单元数 = $2^{24}/4 = 2^{22}$ 个字（因为每4字节为一个字）
3. 地址编号：0x000000 ~ 0xFFFFFF

## 直观理解

### 生活类比

**主存就像一个巨大的仓库**：
- **存储体** = 仓库中一排排的货架
- **每个存储单元** = 货架上的一个格子
- **地址** = 每个格子的编号
- **MAR** = 写着要找哪个格子的纸条
- **MDR** = 从格子里取出（或放进去）的东西
- **存储字长** = 每个格子能装多少东西

### 记忆技巧

主存三要素：「体址数」
- **体**：存储体（存东西的地方）
- **址**：MAR（地址在哪里）
- **数**：MDR（数据是什么）

存储术语三兄弟：「单元-字-字长」
- 存储单元 = 一个格子
- 存储字 = 格子里的东西
- 存储字长 = 格子的大小

### 常考组合

1. MAR位数 → 寻址空间
2. MDR位数 → 存储字长
3. 地址总线宽度 → 寻址能力
4. 数据总线宽度 → 一次能传送的数据量

### 易错陷阱

1. **MAR的位数不一定等于地址总线宽度**：在某些计算机中可能不同
2. **存储字长不一定等于数据总线宽度**：但考试中通常假设相等
3. **按字节编址 vs 按字编址**：地址空间计算方式不同
4. **注意单位**：$2^{10}$ = 1K, $2^{20}$ = 1M, $2^{30}$ = 1G
5. **存储容量和寻址空间**：寻址空间是理论最大值，实际容量可能小于寻址空间
