# 26_总线结构概述

## 核心概念

### 总线的定义

**总线（Bus）**：是计算机各部件之间进行信息传送的公共通道，是一组导线和相关的控制电路。

总线的本质是**共享传输介质**：多个设备通过同一组线路进行数据传送。

### 总线的分类

| 分类方式 | 类型 | 说明 |
|---------|------|------|
| 按位置 | 片内总线 | CPU内部各寄存器之间、ALU与寄存器之间的总线 |
| | 系统总线 | CPU、主存、IO接口之间的总线 |
| | 通信总线 | 计算机与外部设备或其他系统之间的总线 |
| 按数据传输方式 | 并行总线 | 多位数据同时传送（如PCI） |
| | 串行总线 | 数据逐位传送（如USB、PCIe） |
| 按功能 | 数据总线 | 传送数据（双向） |
| | 地址总线 | 传送地址（单向，CPU发出） |
| | 控制总线 | 传送控制信号（部分双向） |

### 系统总线的三类信号线

| 总线类型 | 方向 | 宽度决定 | 功能 |
|---------|------|---------|------|
| 数据总线（DB） | 双向 | 数据通路宽度 | 传送数据和指令 |
| 地址总线（AB） | 单向（CPU→外） | 寻址空间 | 传送存储器/IO地址 |
| 控制总线（CB） | 部分双向 | — | 传送读写命令、中断等控制信号 |

## 原理分析

### 数据总线

- **宽度**：决定了一次能传送的数据位数
- 常见宽度：8位、16位、32位、64位
- **双向**：数据可以从CPU传到存储器，也可以反过来
- 数据总线宽度 = 数据通路宽度 = 存储字长（通常情况下）

$$
\text{数据传送速率} = \text{总线宽度（字节）} \times \text{总线频率}
$$

### 地址总线

- **宽度**：决定了可寻址的存储空间大小
- **单向**：地址总是由CPU（或DMA控制器）发出
- n位地址总线可寻址 $2^n$ 个存储单元

$$
\text{最大寻址空间} = 2^n \text{ 字节（按字节编址时）}
$$

**例**：20位地址总线 → $2^{20}$ = 1M字节的寻址空间

### 控制总线

- **读信号（$\overline{RD}$）**：CPU从存储器或IO读数据
- **写信号（$\overline{WR}$）**：CPU向存储器或IO写数据
- **中断请求（IRQ）**：IO设备向CPU发出中断
- **总线请求（BR）/ 总线允许（BG）**：总线仲裁信号
- 有上划线的信号表示低电平有效

## 直观理解

### 生活类比

**总线就像城市道路系统**：
- **数据总线** = 公路（运货的通道，双向通行）
- **地址总线** = 门牌号系统（标识目的地，单向——由导航系统指定）
- **控制总线** = 交通信号灯（指挥交通，部分双向）
- **总线宽度** = 车道数量（车道越多，一次通过的车越多）
- **总线频率** = 限速（速度越高，通过越快）

### 记忆技巧

系统总线三条：「数地控」
- **数**据总线（双向，运数据）
- **地**址总线（单向，找地址）
- **控**制总线（部分双向，发命令）

### 常考组合

1. 地址总线宽度 → 最大寻址空间
2. 数据总线宽度 → 一次传送的数据量
3. 数据总线宽度 × 频率 = 总线带宽

### 易错陷阱

1. **数据总线是双向的，地址总线是单向的**
2. **地址总线宽度不一定等于MAR位数**（但考试中通常假设相等）
3. **控制总线的信号数量因系统而异**
4. **总线复用**：某些系统中地址总线和数据总线共享同一组线路（分时使用）
