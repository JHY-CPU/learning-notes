# 27_IO系统概述

## 核心概念

### IO系统的功能

IO系统（输入输出系统）负责管理计算机与外部世界之间的信息交换，包括数据的输入和输出。

### IO系统的基本组成

```
CPU ←→ 主存 ←→ IO接口 ←→ IO设备
          ↑
       总线（数据、地址、控制）
```

| 组成部分 | 功能 |
|---------|------|
| IO设备 | 信息的输入输出（键盘、显示器、硬盘等） |
| IO接口 | 连接IO设备与系统总线的桥梁 |
| IO软件 | 驱动程序、中断处理程序 |
| 总线 | 连接各部件的公共通道 |

### IO接口的功能

| 功能 | 说明 |
|------|------|
| 数据缓冲 | 解决CPU与IO设备速度不匹配 |
| 信号转换 | 电平转换、串并转换 |
| 设备寻址 | 识别不同的IO设备 |
| 控制逻辑 | 接收CPU命令，控制设备操作 |
| 状态报告 | 向CPU报告设备状态 |
| 中断管理 | 产生中断请求 |

### IO控制方式

| 控制方式 | 特点 | 适用场景 |
|---------|------|---------|
| 程序查询方式 | CPU不断查询设备状态，效率最低 | 低速、简单设备 |
| 程序中断方式 | 设备就绪后通知CPU | 中低速设备 |
| DMA方式 | DMA控制器直接在主存和设备之间传送数据 | 高速批量传送 |
| 通道方式 | 专用IO处理机管理IO操作 | 大型机 |

## 原理分析

### 四种IO控制方式的对比

| 对比维度 | 程序查询 | 程序中断 | DMA | 通道 |
|---------|---------|---------|-----|------|
| CPU参与程度 | 全程参与 | 仅处理中断 | 仅初始化和结束 | 仅发出IO指令 |
| 数据传送单位 | 字 | 字 | 数据块 | 数据块 |
| CPU与IO并行性 | 无 | 部分 | 高 | 最高 |
| 硬件复杂度 | 最低 | 低 | 中等 | 最高 |
| 适用速度 | 最低速 | 低中速 | 高速 | 最高速 |

### 程序查询方式

```
CPU → 查询设备状态 → 设备忙？ → 是 → 继续查询
                      ↓ 否
              传送一个数据 → 完成？ → 否 → 继续查询
                      ↓ 是
                   结束
```

**缺点**：CPU大部分时间在等待，效率极低。

### 程序中断方式

```
CPU → 启动设备 → CPU做其他工作
                    ↓
              设备就绪 → 发中断请求 → CPU响应中断 → 传送数据 → 返回继续
```

**优点**：CPU不用等待，可以做其他工作。
**缺点**：每次中断只能传送一个数据，中断开销较大。

### DMA方式

```
CPU → 设置DMA参数 → DMA控制器接管总线 → 直接在主存和设备之间传送
CPU做其他工作 ←—————————— 传送完成 ←—— DMA通知CPU
```

**优点**：数据传送不经过CPU，效率高。
**缺点**：需要额外的DMA控制器。

## 直观理解

### 生活类比

**程序查询**：你每隔5分钟去查看快递是否到了。
**程序中断**：快递员到了给你打电话（中断）。
**DMA**：快递员直接把包裹放在你家门口，不需要你去取。
**通道**：你雇了一个管家（通道），专门处理所有快递事务。

### 记忆技巧

四种IO方式：「查中D通」
- **查**询（CPU全参与）
- **中**断（CPU响应后参与）
- **D**MA（CPU只管开始和结束）
- **通**道（CPU只发指令）

效率从低到高：「查 < 中 < D < 通」

### 易错陷阱

1. **中断方式每次只能传送一个数据**，不是一批
2. **DMA方式也需要CPU参与**：初始化参数、结束处理
3. **通道是比DMA更高级的IO方式**，有自己的指令系统
4. **IO接口不等于IO设备**：接口是连接设备和总线的桥梁
