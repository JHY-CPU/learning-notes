# 34_易错辨析_体系结构术语

## 常考术语辨析

### 辨析1：透明性（Transparency）

**定义**：某事件或属性客观存在，但从某个角度来看"看不见"。

| 透明对象 | 对谁透明 | 说明 |
|---------|---------|------|
| Cache | 应用程序员 | 程序员不需要管理Cache |
| 虚拟存储器 | 应用程序员 | 程序员看到的是逻辑地址空间 |
| 流水线 | 程序员 | 程序员不需要关心流水线细节 |
| 寄存器 | 不透明 | 程序员需要使用寄存器名来编程 |

**易错点**：
- "透明"不等于"不存在"。Cache确实存在，只是程序员"看不见"。
- "透明"是相对的。对应用层透明的东西，对系统程序员可能不透明。
- 寄存器对程序员**不透明**，因为编程时需要使用寄存器。

### 辨析2：虚拟机（Virtual Machine）

**定义**：在某一层级之上构建的抽象计算机。

| 层级 | 虚拟机? | 说明 |
|------|---------|------|
| 高级语言级 | 是 | 由编译器/解释器构建 |
| 汇编语言级 | 是 | 由汇编器构建 |
| 操作系统级 | 是 | 由操作系统构建 |
| 机器级（ISA） | 否 | 真实硬件，不是虚拟机 |
| 微程序级 | 否 | 硬件层 |

**易错点**：
- **机器级不是虚拟机**：它是由硬件直接实现的，是"真机"。
- **微程序级不是虚拟机**：微程序虽然看起来像软件，但它直接控制硬件。
- **虚拟机 = 下层机器 + 解释/翻译程序**

### 辨析3：存储程序原理

**核心含义**：
1. 指令和数据都以二进制形式表示
2. 指令和数据都存储在存储器中
3. 计算机按程序规定的顺序自动执行

**易错点**：
- **存储程序 ≠ 把程序存在硬盘上**：是指存在主存（内存）中。
- **存储程序是冯诺依曼结构的核心**，不是可有可无的特点。
- **存储程序不意味着程序不能修改**：程序可以被操作系统调度和管理。

### 辨析4：指令和数据的区分

**问题**：计算机如何区分存储器中存的是指令还是数据？

**答案**：在冯诺依曼结构中，指令和数据以相同的形式存储在存储器中。计算机通过**指令周期的阶段**来区分：
- **取指阶段**取出的认为是指令
- **执行阶段**访问的认为是数据

**易错点**：
- 不能说"靠地址区分"——指令和数据可能在同一地址空间
- 不能说"靠格式区分"——二进制形式完全相同
- 正确答案是"靠指令周期的不同阶段区分"

### 辨析5：机器语言、汇编语言、高级语言

| 语言 | 机器可直接执行? | 需要翻译? | 程序员友好度 |
|------|----------------|---------|-------------|
| 机器语言 | 是 | 不需要 | 最差（二进制） |
| 汇编语言 | 否 | 需要汇编器 | 差（助记符） |
| 高级语言 | 否 | 需要编译器/解释器 | 好（接近自然语言） |

**易错点**：
- **汇编语言不是机器语言**：汇编语言需要汇编器翻译成机器语言
- **编译和解释不同**：编译产生目标文件，解释逐行执行
- **高级语言的执行速度通常比汇编慢**：因为编译器优化可能不如手工优化

### 辨析6：固件与软件

| 对比 | 固件 | 软件 |
|------|------|------|
| 存储介质 | ROM/Flash | 磁盘/内存 |
| 可修改性 | 较难修改 | 容易修改 |
| 运行速度 | 接近硬件 | 受OS调度影响 |
| 掉电保存 | 保存 | 需要外存 |

**易错点**：
- **固件是软件的一种**：本质是程序代码
- **但固件不是普通软件**：固化在ROM中，直接控制硬件
- **微程序是固件**：存放在CPU内部的控制存储器中

## 总结

408考试中关于体系结构术语的高频考点：

3. 存储程序原理的核心内容
4. 指令和数据的区分方式
5. 不同层次语言的翻译过程
