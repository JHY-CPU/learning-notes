# 11_计算机工作过程

## 核心概念

### 计算机的自动工作原理

计算机按照**存储程序原理**自动工作：程序和数据预先存入存储器，计算机从存储器中逐条取出指令并执行，直到程序执行完毕。

这个过程的核心是**指令驱动**：每条指令告诉计算机下一步做什么。

### 指令执行的三个基本阶段

| 阶段 | 名称 | 完成的工作 |
|------|------|-----------|
| 1 | 取指令（Fetch） | 从存储器中取出下一条指令 |
| 2 | 分析指令（Decode） | 解释指令的含义（操作码译码） |
| 3 | 执行指令（Execute） | 完成指令规定的操作 |

## 原理分析

### 完整的指令执行过程

以一条加法指令 `ADD (R1), R0`（将R1所指内存单元的内容加到R0）为例：

**取指令阶段**：
```
1. (PC) → MAR           // PC的值送入MAR，准备取指令
2. 1 → R                // 发出读命令
3. M(MAR) → MDR         // 从存储器读出指令，送入MDR
4. (MDR) → IR           // 指令从MDR送入IR
5. (PC) + 1 → PC        // PC自增，指向下一条指令
```

**分析指令阶段**：
```
6. OP(IR) → 译码器      // 操作码送入译码器
7. 译码器确定是ADD指令   // 分析出操作类型
8. 分析地址码(R1)       // 确定操作数来源
```

**执行指令阶段**：
```
9. (R1) → MAR           // R1的值作为地址送入MAR
10. 1 → R               // 发出读命令
11. M(MAR) → MDR        // 从存储器读出操作数
12. (R0) + (MDR) → R0   // ALU执行加法，结果存入R0
```

### PC自增的时机

PC的自增发生在**取指令阶段的最后**：
- 先取出当前指令
- 然后PC + 1，准备取下一条指令
- 不是在执行阶段结束才自增

### 指令周期

**指令周期** = 取指周期 + 执行周期

$$
T_{\text{指令}} = T_{\text{取指}} + T_{\text{执行}}
$$

- 取指周期通常对所有指令相同
- 执行周期因指令类型不同而不同
- 简单指令（如寄存器传送）执行周期很短
- 复杂指令（如乘法、访存）执行周期较长

## 直观理解

### 生活类比

**计算机的工作过程就像照着菜谱做菜**：

1. **取指令**：翻到菜谱的下一步（看下一步做什么）
2. **分析指令**：读懂这一步的含义（理解"切菜"是什么意思）
3. **执行指令**：执行这个动作（拿起刀切菜）
4. **PC自增**：把书签移到下一步
5. **重复**：继续做下一步

每一步就是一条"指令"，整本菜谱就是一个"程序"。

### 内存中的程序示意

```
地址    内容（指令）       含义
0x00    MOV R0, #5        R0 ← 5
0x04    MOV R1, #3        R1 ← 3
0x08    ADD R0, R1        R0 ← R0 + R1
0x0C    STORE R0, [0x100]  M[0x100] ← R0
0x10    HALT              停机
```

执行过程：
1. PC=0x00，取出MOV R0, #5，PC变为0x04，执行：R0=5
2. PC=0x04，取出MOV R1, #3，PC变为0x08，执行：R1=3
3. PC=0x08，取出ADD R0, R1，PC变为0x0C，执行：R0=8
4. PC=0x0C，取出STORE，PC变为0x10，执行：M[0x100]=8
5. PC=0x10，取出HALT，程序结束

### 易错陷阱

1. **PC自增不是在执行阶段结束才做**，而是在取指阶段完成
2. **不是所有指令的执行时间都相同**，但取指时间通常相同
3. **取指令一定涉及访存**（从主存读指令到IR）
4. **PC指向的是指令地址**，执行完当前指令后PC已经指向下一条了
