# 05_计算机系统层次结构

## 核心概念

### 计算机系统的层次模型

计算机系统从底层到高层可以划分为多个层次，每一层都在下层的基础上提供更高级的抽象。

```
第5级：高级语言级    ← 面向用户（应用程序员）
第4级：汇编语言级    ← 面向汇编程序员
第3级：操作系统级    ← 操作系统提供的虚拟机
第2级：机器级（ISA） ← 面向硬件/系统程序员（传统机器级边界）
第1级：微程序级      ← 面向硬件设计者
第0级：硬联逻辑级    ← 实际硬件电路
```

### 各层详细说明

| 层次 | 名称 | 面向对象 | 核心功能 | 执行方式 |
|------|------|---------|---------|---------|
| 5 | 高级语言级 | 应用程序员 | 用C/Java等高级语言编程 | 编译/解释执行 |
| 4 | 汇编语言级 | 系统程序员 | 用汇编语言编写程序 | 汇编器翻译为机器语言 |
| 3 | 操作系统级 | 系统程序员 | 提供系统调用、资源管理 | OS管理硬件资源 |
| 2 | 机器级(ISA) | 硬件设计者 | 机器语言指令集 | CPU直接执行 |
| 1 | 微程序级 | 硬件设计者 | 微指令控制硬件操作 | 微程序解释机器指令 |
| 0 | 硬联逻辑级 | 硬件 | 门电路、触发器等 | 电子信号直接工作 |

## 原理分析

### 虚拟机概念

**虚拟机（Virtual Machine）**：在某一层级之上构建的抽象计算机，它对上层提供统一的接口，隐藏了下层的实现细节。

- 第3、4、5层都是虚拟机
- 每一层虚拟机都运行在下一层虚拟机（或真实机器）之上
- 用户只看到自己所在层的抽象，不需要了解下层细节

$$
\text{虚拟机} = \text{下层硬件（或虚拟机）} + \text{本层解释/翻译程序}
$$

### 透明性（Transparency）

**定义**：某事件或属性客观存在，但从某一层的角度来看好像不存在，对该层"透明"。

**常见透明性概念**：

| 透明概念 | 含义 |
|---------|------|
| 寄存器对用户透明 | 用户不需要知道寄存器的存在和使用 |
| Cache对用户透明 | 程序员不需要管理Cache，硬件自动处理 |
| 虚拟存储器对用户透明 | 用户看到的是连续的逻辑地址空间 |
| 流水线对用户透明 | 程序员不感知流水线的执行过程 |

**关键区分**：
- "透明" = 对某层"看不见"，不是真的不存在
- "不透明" = 对某层"看得见"，程序员需要了解和管理

### 编译与解释

| 方式 | 过程 | 特点 | 典型应用 |
|------|------|------|---------|
| 编译 | 源程序→编译器→目标程序→执行 | 执行速度快，一次编译多次执行 | C/C++、Go |
| 解释 | 源程序→解释器→逐行解释执行 | 执行速度慢，每次都要解释 | Python、JavaScript |

## 直观理解

### 层次结构类比

计算机的层次结构就像**洋葱**：
- 最外层（高级语言）是用户接触的界面
- 逐层剥开，每一层都提供不同的抽象
- 最内层（硬联逻辑）是真正的硬件

或者类比为**翻译系统**：
- 高级语言 → 编译器 → 汇编语言 → 汇编器 → 机器语言 → 微程序 → 硬件

### 记忆技巧

从高到低记忆：「高汇操机微硬」
- 高级语言级
- 汇编语言级
- 操作系统级
- 机器级
- 微程序级
- 硬联逻辑级

**传统机器级边界**：第2级（ISA级）是软件和硬件的分界线。ISA之上是软件世界，ISA之下是硬件世界。

## 知识关联

### 透明性与各章联系

- **第2章 数据的表示**：补码运算对程序员是透明的
- **第3章 存储系统**：Cache对程序员透明，虚拟存储器对用户透明
- **第5章 CPU**：流水线对程序员透明
- **第6章 总线**：总线仲裁过程对程序员透明

### 常考组合

1. 层次划分 + 各层功能（选择题）
2. 虚拟机概念 + 哪些层是虚拟机（判断题）
3. 透明性概念 + 具体哪些是透明的（易混淆）

### 易错陷阱

1. **"透明"不是"不存在"**：透明是指对某层用户不可见，不是真的没有
2. **编译器和解释器不同**：编译器产生目标代码，解释器逐行执行不产生目标代码
3. **机器级不是虚拟机**：机器级是真实硬件直接执行，不是虚拟机
4. **操作系统级是虚拟机**：OS提供了系统调用接口，对上层相当于一台虚拟计算机
