# 32_总线写操作

## 核心概念

### 写操作的定义

**总线写操作**：主设备（通常是CPU）向从设备（如存储器或IO设备）写入数据。

### 写操作的时序（同步方式）

```
时钟CLK: ╲╱╲╱╲╱╲╱╲╱╲╱╲╱╲╱
         T0  T1  T2  T3  T4  T5  T6  T7

地址:    ══有效地址══
数据:    ══有效数据══
WR(写):  ══低有效══════
BS(忙):  ══高有效══════════════
```

**各阶段操作**：

| 时钟周期 | 操作 |
|---------|------|
| T0 | 主设备将地址送到地址总线，将数据送到数据总线 |
| T1 | 主设备发出写命令（WR低有效） |
| T2 | 从设备接收数据并写入指定地址 |
| T3 | 从设备确认写入完成 |
| T4 | 主设备撤销写命令和数据 |
| T5 | 释放总线 |

### 写操作与读操作的区别

| 对比维度 | 读操作 | 写操作 |
|---------|--------|--------|
| 数据方向 | 从设备→主设备 | 主设备→从设备 |
| 数据出现时机 | 从设备响应后 | 主设备同时发出 |
| 命令 | RD（读） | WR（写） |
| 从设备动作 | 读出数据放到总线 | 从总线接收数据并写入 |

## 原理分析

### 写操作的数据流

```
CPU ──数据──→ 数据总线 ──数据──→ 存储器/IO设备
CPU ──地址──→ 地址总线 ──地址──→ 存储器
CPU ──写命令──→ 控制总线 ──────→ 存储器
```

### 写操作的微操作序列

```
1. (数据) → MDR
2. (地址) → MAR
3. 1 → W        （发出写命令）
4. (MDR) → M(MAR) （将数据写入存储器）
```

## 直观理解

### 生活类比

写操作就像去银行存钱：
1. 告诉柜员你要存钱的账户（地址）
2. 把钱递给柜员（数据）
3. 说"请存入"（写命令）
4. 柜员确认存入完成（确认）
5. 离开柜台（释放总线）

### 记忆技巧

读：「地址→读命令→数据返回」
写：「地址+数据→写命令→确认」

### 易错陷阱

1. **WR低有效**：低电平表示写命令有效
2. **写操作中数据和地址可以同时出现**（在某些设计中）
3. **写操作中数据方向是主设备→从设备**
4. **写操作需要从设备确认**：确保数据已写入
