# 34_总线事务

## 核心概念

### 总线事务的定义

**总线事务（Bus Transaction）**：在总线上进行的一次完整的数据传送操作。包括总线的申请、仲裁、寻址、数据传送和释放。

### 总线事务的完整过程

```
┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐
│ 申请总线  │→│ 总线仲裁  │→│  寻址    │→│ 数据传送  │→│ 释放总线  │
└──────────┘  └──────────┘  └──────────┘  └──────────┘  └──────────┘
```

### 总线事务的类型

| 事务类型 | 说明 | 示例 |
|---------|------|------|
| 读事务 | 主设备从从设备读取数据 | CPU从内存读指令 |
| 写事务 | 主设备向从设备写入数据 | CPU向内存写数据 |
| 读-修改-写事务 | 先读后写（原子操作） | 信号量操作 |
| 块传输事务 | 一次传输多个数据 | DMA块传送 |

### 总线占用期

**总线占用期**：一个设备从获得总线控制权到释放总线的时间。

$$
T_{\text{占用}} = T_{\text{寻址}} + T_{\text{传输}} + T_{\text{释放}}
$$

在总线占用期内，其他设备不能使用总线。

## 原理分析

### 一次完整读事务的时间分析

| 阶段 | 时间 | 说明 |
|------|------|------|
| 申请 | $T_{\text{req}}$ | 设备发出请求 |
| 仲裁 | $T_{\text{arb}}$ | 仲裁器做出决策 |
| 寻址 | $T_{\text{addr}}$ | 发送地址 |
| 数据传输 | $T_{\text{data}}$ | 数据从从设备到主设备 |
| 释放 | $T_{\text{rel}}$ | 释放总线 |

$$
T_{\text{总}} = T_{\text{req}} + T_{\text{arb}} + T_{\text{addr}} + T_{\text{data}} + T_{\text{rel}}
$$

### 提高总线事务效率的方法

1. **突发传输**：减少地址传输开销
2. **分离事务**：在等待期间释放总线给其他设备
3. **流水线传输**：多个事务重叠进行
4. **DMA**：减少CPU参与

## 直观理解

### 生活类比

总线事务就像去餐厅吃饭：
1. 申请：排队等位
2. 仲裁：服务员叫号
3. 寻址：点菜
4. 传输：上菜
5. 释放：买单走人，让位给下一桌

### 易错陷阱

1. **申请和仲裁是分开的两个阶段**
2. **总线占用期不包括申请和仲裁时间**
3. **分离事务可以提高总线利用率**
