# 17_三种仲裁对比

## 核心概念

### 三种集中仲裁方式全面对比

| 对比维度 | 链式查询 | 计数器定时 | 独立请求 |
|---------|---------|-----------|---------|
| **信号线总数** | 3 | $2 + \lceil \log_2 n \rceil$ | $2n + 1$ |
| **BR线数量** | 1（共用） | 1（共用） | n（独立） |
| **BG线数量** | 1（串行） | 0（用地址线） | n（独立） |
| **地址线** | 0 | $\lceil \log_2 n \rceil$ | 0 |
| **BS线** | 1 | 1 | 1 |
| **仲裁速度** | 最慢（与位置有关） | 中等 | 最快（常数1周期） |
| **优先级** | 固定（近高远低） | 可变/可编程 | 最灵活 |
| **公平性** | 差（远设备易饿死） | 好（可轮询） | 可控 |
| **故障敏感性** | 高（BG链故障影响后续） | 低 | 低 |
| **扩展性** | 好 | 中等（需改地址线） | 差（需加线） |
| **硬件成本** | 最低 | 中等 | 最高 |
| **典型应用** | 简单嵌入式 | 一般总线 | 高性能总线 |

### 仲裁时间详细分析

假设n个设备，BG信号传递一个设备需要1个时钟周期。

**链式查询**：
- 最好：设备1请求 → 1周期
- 最坏：设备n请求 → n周期
- 平均：$(n+1)/2$ 周期

**计数器定时**：
- 最好：计数器正好指向请求设备 → 1周期
- 最坏：需要轮询一圈 → n周期
- 平均：$n/2$ 周期

**独立请求**：
- 无论哪个设备请求 → 1周期

## 直观理解

### 生活类比

**链式查询**：排队买东西，从第一个人开始问"你要买吗？"
**计数器定时**：叫号系统，按号码顺序叫号
**独立请求**：每个人都有独立的呼叫按钮，服务员直接看灯判断

### 选择建议

| 场景 | 推荐方式 | 理由 |
|------|---------|------|
| 设备少（<4个） | 链式查询 | 简单便宜 |
| 一般系统 | 计数器定时 | 平衡性能和成本 |
| 高性能系统 | 独立请求 | 速度最快 |
| 需要公平性 | 计数器定时 | 可实现轮询 |
| 设备经常变化 | 链式查询 | 扩展方便 |

## 知识关联

### 常考公式

链式查询信号线：$3$
计数器定时信号线：$2 + \lceil \log_2 n \rceil$
独立请求信号线：$2n + 1$

### 易错陷阱

1. **链式查询的3根信号线是固定的**：不随设备数变化
2. **计数器定时的地址线**：是仲裁器发给设备的，不是设备发给仲裁器
3. **独立请求线数最多但速度最快**：用空间换时间
4. **三种方式都需要BS线**：BS是共用的
