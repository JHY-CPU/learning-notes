# 20_总线定时概述

## 核心概念

### 总线定时的定义

**总线定时**：规定总线上各事件（地址、数据、控制信号）出现的时序关系，确保通信双方能正确收发数据。

总线定时解决的核心问题：**"什么时候传数据？"**

### 总线定时方式的分类

```
总线定时
├── 同步定时（Synchronous）
│   └── 统一时钟信号控制所有操作
├── 异步定时（Asynchronous）
│   ├── 不互锁
│   ├── 半互锁
│   └── 全互锁
└── 半同步定时（Semi-synchronous）
    └── 时钟+握手结合
```

### 三种定时方式对比

| 对比维度 | 同步定时 | 异步定时 | 半同步定时 |
|---------|---------|---------|-----------|
| 时钟信号 | 有统一时钟 | 无统一时钟 | 有时钟+握手 |
| 同步方式 | 时钟边沿同步 | 握手协议同步 | 两者结合 |
| 速度 | 快 | 较慢 | 中等 |
| 灵活性 | 低（设备速度需一致） | 高（兼容不同速度） | 中等 |
| 复杂度 | 简单 | 较复杂 | 中等 |
| 适用场景 | 高速同步设备 | 速度差异大的设备 | 通用系统 |

## 原理分析

### 同步定时的基本原理

所有设备共享一个统一的时钟信号。每个操作（地址传输、命令发出、数据传输）都在固定的时钟周期完成。

```
时钟:  ╲╱╲╱╲╱╲╱╲╱╲╱╲╱╲╱
        T0  T1  T2  T3  T4
        地址 命令 数据 释放
```

### 异步定时的基本原理

没有统一时钟，使用**握手协议**（Handshake）来同步：
- 主设备发出请求信号（Request）
- 从设备处理完成后发出应答信号（Acknowledge）
- 双方通过请求-应答的交互来同步

```
主设备:  ──请求──────────
从设备:  ────应答────
数据:    ────数据────
```

### 半同步定时的基本原理

在时钟信号的基础上增加握手信号：
- 大多数操作按同步方式进行
- 如果从设备速度不够，可以插入等待周期
- 兼顾了同步的速度和异步的灵活性

## 直观理解

### 生活类比

**同步定时**就像学校的统一作息时间表：
- 全校师生按同一张时间表行动
- 8:00上课，8:45下课，所有人同步
- 简单高效，但要求所有人节奏一致

**异步定时**就像自由市场：
- 没有统一的时间表
- 买卖双方通过"你要买吗？""我准备好了！"来协调
- 灵活，但交易时间不确定

**半同步定时**就像带弹性时间的课程表：
- 基本按课表走
- 但如果某个实验没做完，可以延长时间

### 记忆技巧

定时方式：「同步异步半同步」
- 同步：统一时钟，简单快速
- 异步：握手协议，灵活但慢
- 半同步：时钟+握手，折中方案

### 易错陷阱

1. **同步定时不是不需要控制信号**：只是控制信号按时钟同步
2. **异步定时不是随意传输**：仍然有严格的时序规则
3. **半同步定时以同步为主**：只在必要时转为异步
