# 43_总线_易错辨析

## 常见混淆点

### 混淆1：总线宽度 vs 数据通路宽度

| 概念 | 含义 |
|------|------|
| 总线宽度 | 数据总线的位数 |
| 数据通路宽度 | CPU内部数据通路的位数 |

**易错**：两者通常相等，但不总是相等。某些计算机的内部数据通路比外部总线宽。

**考试技巧**：题目未特别说明时，假设两者相等。

### 混淆2：同步 vs 异步

| 对比维度 | 同步 | 异步 |
|---------|------|------|
| 时钟 | 有 | 无 |
| 同步机制 | 时钟边沿 | 握手协议 |
| 速度 | 快 | 慢 |
| 灵活性 | 低 | 高 |

**常见错误**：认为"异步"比"同步"快。实际上异步因为有握手开销，通常更慢。

**正确理解**：
- 同步：按节拍走，效率高但要求设备速度匹配
- 异步：按需走，灵活但有握手开销

### 混淆3：集中仲裁 vs 分布式仲裁

| 对比维度 | 集中仲裁 | 分布式仲裁 |
|---------|---------|-----------|
| 仲裁器 | 有中央仲裁器 | 无中央仲裁器 |
| 决策方式 | 仲裁器集中决策 | 设备共同决策 |
| 可靠性 | 仲裁器是单点故障 | 无单点故障 |
| 典型应用 | PCI | 以太网 |

**常见错误**：认为分布式仲裁"没有仲裁"。实际上是仲裁逻辑分布在各设备中。

### 混淆4：总线宽度 vs 地址总线宽度

| 总线 | 决定 |
|------|------|
| 数据总线宽度 | 一次传送的数据量 |
| 地址总线宽度 | 可寻址的空间大小 |

**易错公式**：
- 数据总线宽度 → 传送带宽
- 地址总线宽度 → $2^n$ 的寻址空间

### 混淆5：PCI vs PCIe

| 对比维度 | PCI | PCIe |
|---------|-----|------|
| 传输方式 | 并行 | 串行 |
| 拓扑 | 共享总线 | 点对点 |
| 速度 | 较慢 | 快 |
| 状态 | 淘汰中 | 主流 |

**常见错误**：认为PCIe"串行"所以比PCI"并行"慢。实际上PCIe远快于PCI。

### 混淆6：AGP vs PCI

| 对比维度 | AGP | PCI |
|---------|-----|-----|
| 用途 | 显卡专用 | 通用 |
| 拓扑 | 点对点 | 共享总线 |
| 状态 | 已淘汰 | 逐渐淘汰 |

### 混淆7：总线带宽的理论值 vs 有效值

- **理论带宽**：宽度 × 频率
- **有效带宽**：考虑地址传输、仲裁等开销后的实际带宽

**有效带宽 < 理论带宽**

### 混淆8：仲裁信号线

| 方式 | BR | BG | 地址线 | 总数 |
|------|----|----|--------|------|
| 链式查询 | 1(共用) | 1(串行) | 0 | 3 |
| 计数器定时 | 1(共用) | 0 | $\lceil \log_2 n \rceil$ | $2 + \lceil \log_2 n \rceil$ |
| 独立请求 | n(独立) | n(独立) | 0 | $2n+1$ |

**易错**：计数器定时不需要BG线，用地址线代替。

## 总结

总线章节的高频易错点：
1. 带宽计算中bit和Byte的转换
2. 同步比异步快（正确）
3. 三种仲裁方式的信号线数量
4. PCIe比PCI快（虽然是串行）
5. USB是主从结构，不是对等
6. 突发传输的带宽计算
