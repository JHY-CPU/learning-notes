# 37_PCIe总线

## 核心概念

### PCIe总线概述

**PCIe（PCI Express）**：新一代高速串行总线标准，采用点对点串行连接，取代了传统的并行PCI总线。

### PCIe的核心特点

| 特点 | 说明 |
|------|------|
| 串行传输 | 逐位串行传送数据 |
| 点对点 | 每个设备独享连接，无竞争 |
| 通道（Lane） | 可聚合多个通道提高带宽 |
| 双工 | 全双工通信（同时收发） |
| 热插拔 | 支持热插拔 |
| 高带宽 | 远超传统PCI |

### PCIe的通道配置

| 配置 | 通道数 | PCIe 3.0带宽 | PCIe 4.0带宽 |
|------|--------|-------------|-------------|
| x1 | 1 | ~1GB/s | ~2GB/s |
| x4 | 4 | ~4GB/s | ~8GB/s |
| x8 | 8 | ~8GB/s | ~16GB/s |
| x16 | 16 | ~16GB/s | ~32GB/s |

### PCIe版本演进

| 版本 | 年份 | 单通道带宽 | 编码方式 |
|------|------|-----------|---------|
| PCIe 1.0 | 2003 | 250MB/s | 8b/10b |
| PCIe 2.0 | 2007 | 500MB/s | 8b/10b |
| PCIe 3.0 | 2010 | ~1GB/s | 128b/130b |
| PCIe 4.0 | 2017 | ~2GB/s | 128b/130b |
| PCIe 5.0 | 2019 | ~4GB/s | 128b/130b |
| PCIe 6.0 | 2022 | ~8GB/s | PAM4 |

## 原理分析

### PCIe的架构

```
CPU ←→ PCIe Root Complex ←→ PCIe Switch ←→ 设备1 (x16)
                              ├───────────→ 设备2 (x4)
                              └───────────→ 设备3 (x1)
```

- **Root Complex**：PCIe的根节点，通常集成在CPU或芯片组中
- **Switch**：PCIe交换器，扩展多个端口
- **Endpoint**：终端设备（显卡、SSD等）

### PCIe vs PCI的关键区别

| 对比维度 | PCI | PCIe |
|---------|-----|------|
| 传输方式 | 并行 | 串行 |
| 拓扑结构 | 共享总线 | 点对点 |
| 总线竞争 | 有 | 无 |
| 带宽共享 | 所有设备共享 | 每个设备独享 |
| 频率 | 33/66MHz | GHz级 |

## 直观理解

### 生活类比

**PCI**就像一条多车道公路，所有车辆共享——车多了就堵。
**PCIe**就像多条独立的高速公路，每辆车有专用车道——从不堵车。

### 记忆技巧

PCIe：「串行、点对点、通道聚合」
- 串行传送
- 点对点连接
- 多通道聚合带宽

### 易错陷阱

1. **PCIe虽然串行，但比并行的PCI快得多**
2. **x16不是16倍速，是16个通道**
3. **PCIe全双工**：同时收发，每个方向都有独立通道
