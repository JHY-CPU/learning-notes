# 04_总线结构_单总线

## 核心概念

### 单总线结构定义

**单总线结构**：CPU、主存、所有IO设备都连接到同一组总线上，通过这组总线进行数据传送。

```
CPU ───┬─── 主存 ───┬─── IO接口1 ───┬─── IO接口2 ───┬─── ...
       │            │               │               │
═══════╧════════════╧═══════════════╧═══════════════╧══════  系统总线
```

### 单总线结构的特点

| 优点 | 缺点 |
|------|------|
| 结构简单，成本低 | 所有设备共享总线，容易产生瓶颈 |
| 易于扩展，新设备直接挂接 | 同一时刻只能一对设备通信 |
| 控制逻辑简单 | 高速设备和低速设备争用总线 |
| 系统可靠性较高（总线少） | 不能满足高性能需求 |

## 原理分析

### 单总线的工作方式

在单总线结构中，任意两个设备之间的通信都要通过同一组总线。

**一次完整的读操作过程**：
1. 主设备（CPU）获得总线控制权
2. 主设备将从设备地址放到地址总线上
3. 主设备发出读命令
4. 从设备响应，将数据放到数据总线上
5. 主设备从数据总线读取数据
6. 释放总线

**性能限制**：

假设总线带宽为 $B$（MB/s），有 $n$ 个设备需要通信：

$$
\text{每个设备的平均带宽} = \frac{B}{n}
$$

设备越多，每个设备获得的带宽越低。

### 速度匹配问题

单总线结构中，高速设备和低速设备共享总线：
- CPU速度最快（ns级）
- 主存速度中等（几十ns级）
- IO设备速度最慢（us-ms级）

当低速IO设备占用总线时，CPU和主存之间的高速数据传送被迫等待，严重影响性能。

## 直观理解

### 生活类比

**单总线就像单车道公路**：
- 所有车辆都走同一条路
- 快车和慢车混行
- 车多了就堵车
- 好处是路只有一条，修路便宜

### 记忆技巧

单总线：「一总线连所有」
- 优点：简单、便宜、易扩展
- 缺点：瓶颈、竞争、慢

## 知识关联

### 与双总线、三总线的关系

单总线是最简单的总线结构，但性能最差：
- 双总线通过分离CPU-主存通道和IO通道来改善
- 三总线为高速IO设备提供专用通道

### 易错陷阱

1. **单总线不是只有一根线**：是一组导线（数据+地址+控制）
2. **单总线可以有多个设备**：但同一时刻只有一对设备通信
3. **单总线需要仲裁**：因为多设备共享总线
