# 13_总线仲裁概述

## 核心概念

### 为什么需要总线仲裁

在多设备共享总线的系统中，可能存在多个设备同时请求使用总线的情况。**总线仲裁**就是决定哪个设备获得总线使用权的过程。

**总线仲裁的目的**：
1. 解决总线使用权的竞争
2. 保证在任一时刻只有一个设备控制总线
3. 按照一定的优先级策略分配总线

### 仲裁的基本分类

```
总线仲裁
├── 集中式仲裁（Centralized）
│   ├── 链式查询（Daisy Chain）
│   ├── 计数器定时查询（Counter Polling）
│   └── 独立请求（Independent Request）
└── 分布式仲裁（Distributed）
    ├── 自举分布式
    └── 冲突检测分布式
```

### 仲裁涉及的信号线

| 信号 | 全称 | 方向 | 功能 |
|------|------|------|------|
| BR | Bus Request | 设备→仲裁器 | 设备请求使用总线 |
| BG | Bus Grant | 仲裁器→设备 | 仲裁器允许设备使用总线 |
| BS | Bus Busy | 设备→仲裁器 | 总线正在被使用 |

## 原理分析

### 仲裁的基本过程

1. **请求**：设备通过BR线向仲裁器发出总线请求
2. **仲裁**：仲裁器根据优先级策略选择一个设备
3. **授权**：仲裁器通过BG线向被选中的设备发出许可
4. **使用**：被授权设备使用总线进行数据传输
5. **释放**：传输完成后，设备释放总线

### 集中式仲裁的特点

| 方面 | 说明 |
|------|------|
| 仲裁器 | 有一个中央仲裁器负责仲裁 |
| 位置 | 通常在CPU中或总线控制器中 |
| 决策 | 仲裁器集中做出仲裁决策 |

### 分布式仲裁的特点

| 方面 | 说明 |
|------|------|
| 仲裁器 | 没有中央仲裁器 |
| 决策 | 各设备共同参与仲裁决策 |
| 复杂度 | 每个设备都需要仲裁逻辑 |

## 直观理解

### 生活类比

**总线仲裁就像会议室发言权的分配**：
- **链式查询**：主持人按顺序问"你要发言吗？"从第一个人开始
- **计数器查询**：主持人按编号叫号"几号要发言？"
- **独立请求**：每个人都有一个按钮，按下就申请发言，主持人根据优先级选
- **分布式仲裁**：没有主持人，大家自行协商谁先说

### 记忆技巧

仲裁三信号：「请许忙」
- **请**求（BR）：设备请求
- **许**可（BG）：仲裁器允许
- **忙**碌（BS）：总线正在使用

## 知识关联

### 与总线定时的关系

仲裁解决"谁来用总线"，定时解决"什么时候传数据"。两者共同保证总线的正确工作。

### 常考组合

1. 三种集中仲裁方式的特点和对比
2. 仲裁过程中的信号变化
3. 分布式仲裁与集中式仲裁的区别

### 易错陷阱

1. **仲裁不是速度匹配**：仲裁解决的是竞争问题
2. **同一时刻只能有一个设备控制总线**：这是仲裁的目的
3. **仲裁信号线的数量因仲裁方式不同而不同**
