# 27_同步vs异步

## 核心概念

### 同步与异步的全面对比

| 对比维度 | 同步定时 | 异步定时 |
|---------|---------|---------|
| **时钟信号** | 有统一时钟 | 无统一时钟 |
| **同步方式** | 时钟边沿 | 握手协议 |
| **传输速度** | 快 | 慢 |
| **灵活性** | 低 | 高 |
| **设备兼容性** | 设备速度需匹配 | 兼容不同速度设备 |
| **控制复杂度** | 简单 | 较复杂 |
| **时钟分布** | 需要时钟分布网络 | 不需要 |
| **典型应用** | 内存总线、CPU内部 | IO总线、设备接口 |

### 何时使用同步

- 设备速度相近
- 传输距离短
- 追求高速传输
- 系统规模小

**典型场景**：CPU与Cache之间的总线、内存总线

### 何时使用异步

- 设备速度差异大
- 传输距离长
- 设备种类多
- 灵活性要求高

**典型场景**：IO总线、USB、各种设备接口

## 原理分析

### 同步总线的带宽

$$
B_{\text{同步}} = \frac{W}{8} \times f
$$

### 异步总线的带宽

$$
B_{\text{异步}} = \frac{W}{8} \times \frac{1}{T_{\text{握手}} + T_{\text{数据}}}
$$

异步总线的带宽取决于握手时间和数据时间之和。

### 半同步定时——折中方案

半同步定时结合了两者的优点：

| 特征 | 说明 |
|------|------|
| 时钟信号 | 有，作为基本时间基准 |
| 握手信号 | 有，用于等待慢设备 |
| 正常操作 | 按同步方式（按时钟节拍） |
| 慢设备 | 插入等待周期（Wait信号） |

```
时钟: ╲╱╲╱╲╱╲╱╲╱╲╱╲╱╲╱
Wait: ────────┐  ┌──────  （高有效时插入等待周期）
              └──┘
```

半同步定时的工作方式：
1. 操作开始时按同步方式进行
2. 如果从设备来不及响应，发出Wait信号
3. 等待期间时钟继续，但数据保持不变
4. 从设备准备好后撤销Wait，继续同步操作

## 直观理解

### 生活类比

**同步**：军训齐步走——所有人按同一个口令走。
**异步**：自由市场——买卖双方自行协商。
**半同步**：有弹性的工作时间——8点上班，但迟到可以补时。

### 记忆技巧

「同步快但死板，异步慢但灵活，半同步折中」

### 易错陷阱

1. **同步总线不是不需要控制信号**
2. **异步总线不是没有时序约束**
3. **半同步定时以同步为主**：只有必要时才转异步
4. **PCI总线是半同步定时**
