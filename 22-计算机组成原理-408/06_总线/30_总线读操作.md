# 31_总线读操作

## 核心概念

### 读操作的定义

**总线读操作**：主设备（通常是CPU）从从设备（如存储器或IO设备）读取数据。

### 读操作的时序（同步方式）

```
时钟CLK: ╲╱╲╱╲╱╲╱╲╱╲╱╲╱╲╱
         T0  T1  T2  T3  T4  T5  T6  T7

地址:    ══有效地址══
RD(读):  ══低有效══════
数据:              ══有效数据══
BS(忙):  ══高有效══════════════
```

**各阶段操作**：

| 时钟周期 | 操作 |
|---------|------|
| T0 | 主设备将地址送到地址总线，发出读命令（RD低有效） |
| T1 | 从设备识别地址，准备数据 |
| T2 | 从设备将数据送到数据总线 |
| T3 | 主设备读取数据 |
| T4 | 主设备撤销读命令 |
| T5 | 释放总线 |

### 读操作的时序（异步全互锁方式）

```
Req: ──┐           ┌────────
       └───────────┘
ACK:   ────┐       ┌────
           └───────┘
数据:  ────有效数据────
```

**四步握手**：
1. 主设备发出Req（请求读取）
2. 从设备准备数据，发出ACK
3. 主设备读取数据，撤销Req
4. 从设备撤销ACK

## 原理分析

### 读操作的数据流

```
存储器/IO设备 ──数据──→ 数据总线 ──数据──→ CPU
                                          ↑
CPU ──地址──→ 地址总线 ──地址──→ 存储器
CPU ──读命令──→ 控制总线 ──────→ 存储器
```

### 读操作的微操作序列

```
1. (PC) → MAR    （取指时）或 (地址) → MAR
2. 1 → R         （发出读命令）
3. M(MAR) → MDR  （从存储器读出数据）
4. (MDR) → 目标寄存器
```

## 直观理解

### 生活类比

读操作就像去图书馆借书：
1. 告诉管理员书的编号（地址）
2. 说"我要借书"（读命令）
3. 管理员找到书，递给你（数据传输）
4. 你拿到书，离开柜台（释放总线）

### 易错陷阱

1. **RD低有效**：低电平表示读命令有效
2. **读操作中数据方向是从设备→主设备**
3. **先传地址，后传数据**：不是同时
