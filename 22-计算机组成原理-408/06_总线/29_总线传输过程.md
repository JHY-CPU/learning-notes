# 30_总线传输过程

## 核心概念

### 总线事务（Bus Transaction）

**总线事务**：一次完整的总线操作，从申请总线到释放总线的全过程。

### 总线事务的五个阶段

| 阶段 | 名称 | 工作内容 |
|------|------|---------|
| 1 | 申请总线 | 主设备向仲裁器发出总线请求 |
| 2 | 总线仲裁 | 仲裁器决定哪个设备获得总线 |
| 3 | 寻址 | 主设备将从设备地址送到地址总线 |
| 4 | 数据传输 | 主设备和从设备通过数据总线传送数据 |
| 5 | 释放总线 | 主设备释放总线控制权 |

### 完整的读操作时序

```
阶段：  申请 → 仲裁 → 寻址 → 读数据 → 释放
时间：  t0-t1  t1-t2  t2-t3  t3-t4   t4-t5

BR:     ──┐  ┌────
          └──┘
BG:     ────┐  ┌──
            └──┘
BS:     ──────┐        ┌──
              └────────┘
地址:         ┐  ┌────
              └──┘
RD:           ┐  ┌────
              └──┘
数据:              ┐  ┌──
                   └──┘
```

### 完整的写操作时序

```
阶段：  申请 → 仲裁 → 寻址 → 写数据 → 释放
```

写操作与读操作类似，只是数据方向相反：
- 读：从设备 → 数据总线 → 主设备
- 写：主设备 → 数据总线 → 从设备

## 原理分析

### 总线占用时间

一次总线事务占用总线的时间：

$$
T_{\text{占用}} = T_{\text{仲裁}} + T_{\text{寻址}} + T_{\text{传输}} + T_{\text{释放}}
$$

- $T_{\text{仲裁}}$：取决于仲裁方式（1-n个周期）
- $T_{\text{寻址}}$：通常1个时钟周期
- $T_{\text{传输}}$：取决于数据量和传输方式
- $T_{\text{释放}}$：通常1个时钟周期

### 影响总线效率的因素

1. **仲裁开销**：每次传输都需要仲裁
2. **地址传输开销**：每次传输都需要传地址
3. **总线释放开销**：释放总线需要时间
4. **设备响应时间**：从设备可能需要等待

## 直观理解

### 生活类比

总线事务就像**银行办理业务**：
1. **申请**：拿号排队（总线请求）
2. **仲裁**：叫号（总线仲裁）
3. **寻址**：告诉柜员你要办什么业务（地址）
4. **传输**：办理业务（数据传送）
5. **释放**：让位给下一个人（释放总线）

### 记忆技巧

总线事务：「申仲寻传释」
- 申：申请总线
- 仲：仲裁
- 寻：寻址
- 传：数据传输
- 释：释放总线

### 易错陷阱

1. **申请和仲裁是两个不同的阶段**
2. **读操作中数据方向是从设备→主设备**
3. **写操作中数据方向是主设备→从设备**
4. **释放总线是必须的**：不能一直占用总线
