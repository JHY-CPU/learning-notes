# 36_PCI总线

## 核心概念

### PCI总线概述

**PCI（Peripheral Component Interconnect）**：外围部件互连总线，由Intel在1991年提出，是曾经最广泛使用的系统总线标准。

### PCI总线的特点

| 特点 | 说明 |
|------|------|
| 即插即用（PnP） | 自动分配资源，无需手动配置 |
| 突发传输 | 支持突发传输，提高效率 |
| 独立于CPU | 不依赖特定CPU架构 |
| 集中仲裁 | 使用集中式仲裁 |
| 地址/数据复用 | AD线复用，减少引脚 |
| 32/64位 | 支持32位和64位数据宽度 |
| 33/66MHz | 支持33MHz和66MHz时钟 |

### PCI总线的带宽

| 配置 | 带宽 |
|------|------|
| 32位 @ 33MHz | 132MB/s |
| 32位 @ 66MHz | 264MB/s |
| 64位 @ 33MHz | 264MB/s |
| 64位 @ 66MHz | 528MB/s |

## 原理分析

### PCI总线的系统结构

```
CPU ←→ 主存
  ↕
北桥（Host Bridge）
  ↕
PCI总线
  ↕
南桥（South Bridge）
  ↕
ISA/USB/IDE...
```

**北桥**：连接CPU、内存和高速PCI设备
**南桥**：连接低速IO设备

### PCI总线的仲裁

PCI使用集中式独立请求仲裁：
- 每个PCI设备有独立的REQ#（请求）和GNT#（授权）线
- 仲裁器在北桥芯片中
- 仲裁可以在当前传输期间进行（隐藏仲裁延迟）

### PCI总线定时

PCI采用半同步定时：
- 有统一时钟（CLK）
- 有IRDY#（主设备就绪）和TRDY#（从设备就绪）
- 如果任一设备未就绪，插入等待周期

## 直观理解

### 生活类比

PCI总线就像城市的**标准高速公路**：
- 所有车辆（设备）都按统一标准行驶
- 有标准的入口（即插即用）
- 有收费站（桥接器）连接到其他道路

### 记忆技巧

PCI：「标准、即插即用、桥接」
- 标准化接口
- 即插即用
- 通过桥接器连接

## 知识关联

### PCI vs PCIe

| 对比维度 | PCI | PCIe |
|---------|-----|------|
| 架构 | 共享总线 | 点对点串行 |
| 带宽 | 最高528MB/s | 最高64GB/s (x16) |
| 竞争 | 设备竞争总线 | 无竞争 |
| 状态 | 逐渐淘汰 | 主流 |

1. **PCI是并行共享总线**
2. **PCI通过桥接器连接到CPU总线**
