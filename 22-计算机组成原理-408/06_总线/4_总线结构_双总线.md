# 05_总线结构_双总线

## 核心概念

### 双总线结构定义

**双总线结构**：以CPU为中心，设置两条独立的总线——主存总线和IO总线。

```
         ┌───── 主存总线 ───── 主存
         │
   CPU ──┤
         │
         └───── IO总线 ─────── IO接口1 ─── IO接口2 ─── ...
```

### 两种常见的双总线结构

**结构1：以CPU为中心的双总线**
```
主存 ── 主存总线 ── CPU ── IO总线 ── IO接口
```
- CPU通过主存总线直接访问主存
- CPU通过IO总线访问IO设备
- IO设备不能直接访问主存

**结构2：以存储器为中心的双总线**
```
CPU ── 总线1 ── 主存 ── 总线2 ── IO接口
```
- 总线1连接CPU和主存
- 总线2连接主存和IO设备
- IO设备可以通过总线2直接访问主存

## 原理分析

### 双总线结构的优势

相比单总线结构，双总线有以下优势：

| 方面 | 单总线 | 双总线 |
|------|--------|--------|
| 总线竞争 | 所有设备竞争同一总线 | CPU-主存和IO分开 |
| IO效率 | IO操作占用总线，影响CPU | IO走专用总线，互不干扰 |
| 并行性 | 不能并行 | CPU计算和IO可并行 |
| 复杂度 | 简单 | 较简单 |

### 双总线的性能分析

在双总线结构中，CPU访问主存和IO设备可以同时进行：

$$
\text{系统总吞吐量} = B_{\text{主存总线}} + B_{\text{IO总线}}
$$

其中 $B_{\text{主存总线}}$ 和 $B_{\text{IO总线}}$ 分别是两条总线的带宽。

## 直观理解

### 生活类比

**双总线就像城市有了两条独立的道路**：
- 一条是高速路（主存总线）：专门连接核心区域
- 一条是普通路（IO总线）：连接各个外围设施
- 两条路互不干扰，可以同时使用

### 记忆技巧

双总线：「一分为二」
- 主存总线：CPU和主存之间的高速通道
- IO总线：CPU和IO设备之间的通道

## 知识关联

### 与单总线和三总线的关系

- 双总线是单总线的改进：解决了IO和CPU争用总线的问题
- 三总线是双总线的进一步改进：为高速IO提供DMA总线

### 常考组合

1. 双总线结构的组成和特点
2. 双总线 vs 单总线的性能对比
3. 以CPU为中心 vs 以存储器为中心的区别

### 易错陷阱

1. **双总线的两条总线可以同时工作**：这是它比单总线快的原因
2. **以CPU为中心的双总线中，IO不能直接访问主存**
3. **双总线不一定比单总线总带宽高**：取决于每条总线的带宽
