# 29_半同步定时

## 核心概念

### 半同步定时定义

**半同步定时（Semi-synchronous Timing）**：结合了同步定时和异步定时的特点。系统有统一的时钟信号，同时增加了握手信号（如Wait），用于处理速度较慢的设备。

### 半同步定时的信号

| 信号 | 类型 | 功能 |
|------|------|------|
| CLK | 时钟 | 统一的时钟信号 |
| Addr/Data | 数据 | 地址和数据信号 |
| RD/WR | 控制 | 读写命令 |
| Wait | 握手 | 从设备请求插入等待周期 |

### 工作方式

```
正常操作（从设备够快）：
CLK:  ╲╱╲╱╲╱╲╱╲╱
       T0 T1 T2 T3
Wait: ──────────── （低，不需要等待）
操作： 地址 命令 数据 完成

慢设备需要等待：
CLK:  ╲╱╲╱╲╱╲╱╲╱╲╱╲╱
       T0 T1 T2 T3 T4 T5
Wait: ──────┐  ┌─────── （高，插入等待）
            └──┘
操作： 地址 命令 等待 数据 完成
```

## 原理分析

### 半同步定时的时序

一次读操作（假设需要2个等待周期）：

| 时钟周期 | 操作 | Wait信号 |
|---------|------|---------|
| T0 | 发出地址和读命令 | 0 |
| T1 | 从设备识别地址 | 1（从设备请求等待） |
| T2 | 等待 | 1 |
| T3 | 从设备准备好数据 | 0（等待结束） |
| T4 | 主设备读取数据 | 0 |
| T5 | 释放总线 | 0 |

### 半同步定时的带宽

$$
B = \frac{W}{8} \times \frac{f}{n + w}
$$

其中：
- $W$：数据宽度（bit）
- $f$：时钟频率
- $n$：基本操作周期数
- $w$：插入的等待周期数

## 直观理解

### 生活类比

半同步定时就像**带弹性时间的课表**：
- 基本上按课表走（同步）
- 如果某个实验没做完，可以延长下课时间（Wait）
- 但总的时间框架还在（时钟继续）

### 记忆技巧

半同步：「时钟为主，握手为辅」
- 主要按时钟同步
- 必要时通过Wait信号插入等待

## 知识关联

### PCI总线的定时

PCI总线采用半同步定时：
- 有统一的时钟（33MHz或66MHz）
- 有IRDY（Initiator Ready）和TRDY（Target Ready）信号
- 如果任一信号未就绪，插入等待周期

### 常考组合

1. 半同步定时的工作原理
2. Wait信号的作用
3. PCI总线的定时特点

### 易错陷阱

1. **半同步定时有时钟**：不是纯异步
2. **Wait信号是从设备发出的**：表示从设备没准备好
3. **等待期间时钟继续计数**：只是操作暂停
