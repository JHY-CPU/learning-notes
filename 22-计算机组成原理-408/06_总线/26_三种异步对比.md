# 26_三种异步对比

## 核心概念

### 三种异步定时方式全面对比

| 对比维度 | 不互锁 | 半互锁 | 全互锁 |
|---------|--------|--------|--------|
| **握手步骤** | 1步 | 2-3步 | 4步 |
| **需要ACK信号** | 否 | 是 | 是 |
| **主设备等ACK** | 否 | 是 | 是 |
| **从设备等Req撤销** | 否 | 否 | 是 |
| **传输速度** | 最快 | 中等 | 最慢 |
| **可靠性** | 最低 | 中等 | 最高 |
| **硬件复杂度** | 最低 | 中等 | 最高 |
| **适用场景** | 速度一致的设备 | 一般系统 | 高可靠性系统 |

### 时序对比图

**不互锁**：
```
Req: ──┐       ┌──
       └───────┘
Data:  ──有效数据──
```
（没有ACK信号）

**半互锁**：
```
Req: ──┐           ┌──
       └───────────┘
ACK:   ────┐   ┌────
           └───┘
Data: ──有效数据────
```

**全互锁**：
```
Req: ──┐           ┌────────
       └───────────┘
ACK:   ────┐       ┌────
           └───────┘
Data: ──有效数据────────────
```

### 详细工作过程对比

| 步骤 | 不互锁 | 半互锁 | 全互锁 |
|------|--------|--------|--------|
| 1 | 主发Req | 主发Req | 主发Req |
| 2 | 等固定时间 | 从发ACK | 从发ACK |
| 3 | 主读数据，撤Req | 主读数据，撤Req | 主读数据，撤Req |
| 4 | — | 从适时撤ACK | 从检到Req↓，撤ACK |

## 直观理解

### 生活类比

**不互锁**：广播通知——我通知了，不管你听没听到。
**半互锁**：发短信——你回复"收到了"就行，我不管你怎么想。
**全互锁**：正式会议——A发言→B确认→A说完→B确认A说完。

### 选择指南

| 场景 | 推荐方式 |
|------|---------|
| 设备速度已知且一致 | 不互锁 |
| 一般应用 | 半互锁 |
| 高可靠性要求 | 全互锁 |
| 速度差异大的设备 | 全互锁 |

## 知识关联

### 常考组合

1. 三种方式的时序图辨认
2. 速度和可靠性的权衡
3. ACK信号的作用

### 易错陷阱

1. **全互锁不是最快的**：是最慢的
2. **不互锁不是握手协议**：没有ACK
3. **半互锁只确认一个方向**：主确认从的ACK，但不反向
