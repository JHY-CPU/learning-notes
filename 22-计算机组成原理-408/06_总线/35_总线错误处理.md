# 35_总线错误处理

## 核心概念

### 总线错误的类型

| 错误类型 | 说明 |
|---------|------|
| 总线超时 | 从设备在规定时间内没有响应 |
| 地址无效 | 访问的地址不存在对应的设备 |
| 数据校验错误 | 传输的数据有误（如奇偶校验失败） |
| 总线冲突 | 多个设备同时向总线发送数据 |

### 总线超时

**定义**：主设备发出请求后，在规定的等待时间内没有收到从设备的响应。

**处理方式**：
1. 主设备发出超时信号
2. 记录错误日志
3. 终止当前操作
4. 释放总线
5. 向CPU报告错误（通常通过中断）

### 奇偶校验

**奇偶校验**：在数据中增加一位校验位，使得数据中1的个数为奇数（奇校验）或偶数（偶校验）。

**奇校验**：数据位+校验位中1的个数为奇数
**偶校验**：数据位+校验位中1的个数为偶数

**例**：数据 = 1011001
- 奇校验：1的个数=4（偶数），校验位=1，使总1个数=5（奇数）
- 偶校验：1的个数=4（偶数），校验位=0，保持1个数=4（偶数）

$$
\text{奇校验位} = d_0 \oplus d_1 \oplus d_2 \oplus \cdots \oplus d_{n-1} \oplus 1
$$

$$
\text{偶校验位} = d_0 \oplus d_1 \oplus d_2 \oplus \cdots \oplus d_{n-1}
$$

## 原理分析

### 总线错误检测机制

1. **超时检测**：计时器检测从设备响应时间
2. **奇偶校验**：检测数据传输是否出错
3. **地址译码**：检测地址是否有效
4. **总线监控**：检测是否有总线冲突

### 错误恢复策略

| 策略 | 说明 |
|------|------|
| 重试 | 重新执行失败的操作 |
| 报错 | 向CPU报告错误，由软件处理 |
| 忽略 | 对于非关键操作，忽略错误 |
| 复位 | 严重错误时复位设备 |

## 直观理解

### 生活类比

**总线超时**就像打电话没人接——等了30秒后自动挂断。
**奇偶校验**就像数手指——说"我传了5个苹果"，对方数一下是不是5个。

### 易错陷阱

1. **奇偶校验只能检测奇数个错误**：如果同时错两位，检测不出来
2. **奇偶校验不能纠正错误**：只能检测
3. **总线超时的"规定时间"是系统设定的**
