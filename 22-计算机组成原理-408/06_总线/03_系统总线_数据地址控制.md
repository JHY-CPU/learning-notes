# 03_系统总线_数据地址控制

## 核心概念

### 数据总线（Data Bus）

**功能**：在CPU、主存、IO设备之间传送数据和指令。

| 特征 | 说明 |
|------|------|
| 方向 | **双向**（CPU可读可写） |
| 宽度 | 决定了一次传送的数据位数 |
| 常见宽度 | 8位、16位、32位、64位 |
| 宽度意义 | 数据总线宽度 = 数据通路宽度（通常=存储字长） |

**数据总线宽度的影响**：
- 16位数据总线：一次传送2字节
- 32位数据总线：一次传送4字节
- 64位数据总线：一次传送8字节

### 地址总线（Address Bus）

**功能**：传送CPU要访问的存储器单元地址或IO端口地址。

| 特征 | 说明 |
|------|------|
| 方向 | **单向**（CPU→存储器/IO） |
| 宽度 | 决定了寻址空间的大小 |
| 计算 | n位地址总线可寻址 $2^n$ 个单元 |

**地址总线宽度与寻址空间**：

| 地址线宽度 | 最大寻址空间（按字节编址） |
|-----------|-------------------------|
| 16位 | $2^{16}$ = 64KB |
| 20位 | $2^{20}$ = 1MB |
| 32位 | $2^{32}$ = 4GB |
| 64位 | $2^{64}$ = 16EB |

### 控制总线（Control Bus）

**功能**：传送各种控制信号和状态信号。

| 信号 | 方向 | 含义 |
|------|------|------|
| 读信号 $\overline{RD}$ | CPU→外 | CPU从存储器/IO读数据 |
| 写信号 $\overline{WR}$ | CPU→外 | CPU向存储器/IO写数据 |
| 中断请求 IRQ | 外→CPU | IO设备请求中断 |
| 总线请求 BR | 外→CPU | 设备请求使用总线 |
| 总线允许 BG | CPU→外 | CPU允许设备使用总线 |
| 时钟 CLK | CPU→外 | 同步时钟信号 |
| 复位 RESET | CPU→外 | 系统复位 |

**控制总线的方向**：
- 部分信号是单向的（CPU发出或设备发出）
- 部分信号是双向的（如数据传输确认信号）
- 控制总线的信号数量决定了系统的控制能力

## 原理分析

### 三类总线的关系

```
        地址总线（单向: CPU → 存储器/IO）
              ┌───────────────────┐
              ↓                   ↓
    CPU ←──→ 数据总线（双向）←──→ 存储器/IO
              ↑                   ↑
        控制总线（部分双向）────────┘
```

一次完整的读操作：
1. 地址总线：CPU将地址送到地址总线上
2. 控制总线：CPU发出读信号 $\overline{RD}$
3. 数据总线：存储器将数据送到数据总线上
4. CPU从数据总线读取数据

### 总线宽度的匹配

- **数据总线宽度 = 存储字长**：一次读写一个存储字
- **地址总线宽度 = MAR位数**：决定寻址范围
- **控制总线**：宽度由控制信号的数量决定

**例题**：某计算机数据总线宽度为32位，地址总线宽度为24位，求：
1. 一次最多传送多少字节数据？
2. 最大寻址空间是多少？

**解答**：
1. 32位数据总线 → 一次传送4字节
2. 24位地址总线 → $2^{24}$ = 16MB

## 直观理解

### 生活类比

三类总线就像道路系统的三个组成部分：
- **地址总线** = 门牌号系统（告诉你去哪）
- **数据总线** = 公路（运货的通道）
- **控制总线** = 交通信号灯和交警（指挥交通）

### 记忆技巧

三类总线：「数地控」
- **数**据总线：双向，运数据
- **地**址总线：单向，找地址
- **控**制总线：部分双向，发信号

## 知识关联

### 常考公式

$$
\text{最大寻址空间} = 2^n \text{ 字节（n为地址线位数）}
$$

$$
\text{总线带宽} = \text{数据总线宽度（字节）} \times \text{总线频率}
$$

### 易错陷阱

1. **数据总线是双向的**
2. **地址总线是单向的**
3. **"地址总线宽度=20位"不等于"最大地址是20"**：是$2^{20}$-1
4. **存储字长可能不等于数据总线宽度**：但在考试中通常假设相等
