# 76_浮点数与定点数对比

## 核心概念

### 定点数与浮点数的本质区别

| 特征 | 定点数 | 浮点数 |
|------|--------|--------|
| 小数点位置 | 固定 | 浮动（由阶码决定） |
| 表示范围 | 小且固定 | 大且可变 |
| 精度 | 均匀分布 | 不均匀（接近0处精度高，远处精度低） |
| 硬件复杂度 | 简单（加法器即可） | 复杂（需要对阶、规格化等） |
| 运算速度 | 快 | 慢 |
| 存储效率 | 对特定范围高效 | 对大范围高效 |

### 精度对比

**定点数**：
- n 位定点小数，精度恒定为 $2^{-n}$
- 任意相邻两个可表示数的间距相同

**浮点数**：
- 精度取决于尾数位数
- 相邻两个可表示数的间距 = $2^E \times 2^{-m}$（与阶码有关）
- 阶码越大，间距越大，精度越低

### 范围对比

**定点数（n 位，含 1 位符号）**：
- 定点整数：$[-2^{n-1}, 2^{n-1}-1]$
- 定点小数：$[-1, 1-2^{-(n-1)}]$

**浮点数（n 位，m 位尾数，e 位阶码）**：
- 范围：$\pm(1-2^{-m}) \times 2^{2^{e-1}-1}$
- 远大于同等位数的定点数

### 硬件实现对比

| 组件 | 定点数 | 浮点数 |
|------|--------|--------|
| 加法 | 直接用加法器 | 对阶 + 加法 + 规格化 + 舍入 |
| 减法 | 补码取负后加法 | 同上 |
| 乘法 | Booth算法 | 阶码相加 + 尾数相乘 + 规格化 |
| 除法 | 恢复/加减交替 | 阶码相减 + 尾数相除 + 规格化 |

### 适用场景

**定点数适合**：
- 整数运算（计数、地址、循环控制）
- 范围已知且固定的数值
- 嵌入式系统（无浮点硬件时）
- 金融计算（定点精度可控）

**浮点数适合**：
- 科学计算（物理模拟、数值分析）
- 大范围数值（天文数据、微观数据）
- 图形处理（3D 坐标变换）
- 机器学习（训练中的梯度值）

### C 语言中的对应

| 类型 | 对应 | 位数 |
|------|------|------|
| `int` | 定点整数 | 32位 |
| `float` | 浮点数 | 32位（IEEE 754单精度） |
| `double` | 浮点数 | 64位（IEEE 754双精度） |

## 直观理解

### 类比

- **定点数**：像一把固定刻度的尺子，精度均匀但范围有限
- **浮点数**：像一把可伸缩的尺子，范围很大但刻度不均匀

### 记忆要点

- **定点简单快速，浮点范围大**
- **浮点精度不均匀**：接近0精度高，远离0精度低
- **浮点运算需要更多步骤**：对阶、规格化、舍入
- **408考试重点考浮点运算过程**
