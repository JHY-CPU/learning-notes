# 39_溢出检测_双符号位

## 核心概念

### 变形补码（模4补码）

**变形补码**（Deformed Complement / Two-Bit Signed）：使用**2位符号位**的补码表示。也称为**模4补码**。

### 符号位编码规则

| 双符号位 | 含义 |
|----------|------|
| `00` | 正数（正常） |
| `01` | **上溢**（正溢出） |
| `10` | **下溢**（负溢出） |
| `11` | 负数（正常） |

### 溢出判断方法

运算时**两位符号位都参与运算**，观察最终结果的双符号位：

- `00` → 正数，无溢出
- `11` → 负数，无溢出
- `01` → **上溢**（正溢出）
- `10` → **下溢**（负溢出）

**溢出判断公式**：$V = S_1 \oplus S_2$（两个符号位异或）

- $V = 1$：溢出（`01` 或 `10`）
- $V = 0$：无溢出（`00` 或 `11`）

## 原理分析

### 双符号位的工作原理

在模4补码中，2位符号位的权值不同：
- 第1位（高位符号位 $S_1$）：权值 $-2 \times 2^{n-2}$
- 第2位（低位符号位 $S_2$）：权值 $-1 \times 2^{n-2}$

**正常情况**：
- 正数：$S_1 = S_2 = 0$（双符号位 `00`）
- 负数：$S_1 = S_2 = 1$（双符号位 `11`）

**溢出时**：两位符号位不再一致，说明运算结果超出了表示范围。

### 示例分析

**上溢示例**：$85 + 60$（8位数据，用2位符号位 = 10位变形补码）

$$[85]_{\text{变形补码}} = 00\ 1010101$$
$$[60]_{\text{变形补码}} = 00\ 0111100$$

```
  00 1010101
+ 00 0111100
------------
  01 0010001
```

双符号位 `01` → **上溢** ✓

**下溢示例**：$(-85) + (-60)$

$$[-85]_{\text{变形补码}} = 11\ 0101011$$
$$[-60]_{\text{变形补码}} = 11\ 1000100$$

```
  11 0101011
+ 11 1000100
------------
 110 1101111
```

进位到第3位，但只保留10位：`10 1101111`

双符号位 `10` → **下溢** ✓

**无溢出示例**：$42 + 25$

$$[42]_{\text{变形补码}} = 00\ 0101010$$
$$[25]_{\text{变形补码}} = 00\ 0011001$$

```
  00 0101010
+ 00 0011001
------------
  00 1000011
```

双符号位 `00` → 正数，无溢出 ✓

### 无溢出示例（负数）：$(-42) + (-25)$

$$[-42]_{\text{变形补码}} = 11\ 1010110$$
$$[-25]_{\text{变形补码}} = 11\ 1100111$$

```
  11 1010110
+ 11 1100111
------------
 111 1011101
```

保留10位：`11 1011101`

双符号位 `11` → 负数，无溢出 ✓

## 直观理解

### 生活类比

双符号位就像温度计上的"过热/过冷"指示灯：
- `00`：温度正常（正数）
- `11`：温度正常（负数）
- `01`：温度过高（上溢）
- `10`：温度过低（下溢）

### 记忆技巧

1. **两位相同 = 正常**（`00` 正数，`11` 负数）
2. **两位不同 = 溢出**（`01` 上溢，`10` 下溢）
3. **$01$ = 0正1负 = 正数变负数 = 上溢**
4. **$10$ = 1负0正 = 负数变正数 = 下溢**

### 与单符号位的对比

| 方法 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| 单符号位 | 进位异或 | 节省1位 | 需要额外硬件 |
| 双符号位 | 符号位组合 | 直观 | 多占1位 |

### 易错陷阱
1. **双符号位是2位符号位**：不是1位符号位+1位什么别的
2. **两位相同是正常**：`00` 和 `11` 都表示无溢出
3. **运算时两位符号位都要参与**：不能只用一位
4. **`01` 是上溢不是正数**：`01` 说明本应是很大的正数，溢出了
