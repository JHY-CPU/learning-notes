# 17_CRC_408考点

## 核心概念

### CRC检错能力

设生成多项式 $G(x)$ 为 $r+1$ 位（$r$ 次多项式），CRC编码具有以下检错能力：

| 错误类型 | 检错能力 |
|----------|----------|
| 1位错误 | 全部能检测 |
| 2位错误 | 全部能检测（前提是 $G(x)$ 含至少3项） |
| 奇数位错误 | 全部能检测 |
| 所有长度 $\leq r$ 位的突发错误 | 全部能检测 |
| 长度 $= r+1$ 位的突发错误 | 检测概率 $1 - 2^{-r}$ |
| 长度 $> r+1$ 位的突发错误 | 检测概率 $1 - 2^{-r}$ |

### 与海明码的对比（408高频选择题）

| 特性 | CRC | 海明码 | 奇偶校验 |
|------|-----|--------|---------|
| 检错能力 | 所有 $\leq r$ 位突发错 | 2位错误 | 1位错误 |
| 纠错能力 | 无 | 1位错误 | 无 |
| 运算基础 | 多项式除法 | 异或/奇偶 | 异或 |
| 应用场景 | 数据链路层 | 内存ECC | 简单校验 |
| 实现复杂度 | 低 | 中等 | 最低 |

## 典型考题

### 考题1：检错能力判断

**题目**：CRC编码中，若生成多项式为 $r$ 次，则能检测（）

A. 所有1位错误
B. 所有 $\leq r$ 位的突发错误
C. 所有偶数位错误
D. 所有 $\leq r+1$ 位的突发错误

**分析**：
- A：正确，CRC能检测所有单bit错误
- B：正确，CRC能检测所有长度 $\leq r$ 的突发错误
- C：**不一定**，CRC并不保证检测所有偶数位错误（虽然大多数情况下可以）
- D：错误，只能保证检测 $\leq r$ 位的突发错误

**答案**：A、B

### 考题2：选择生成多项式

**题目**：下列生成多项式中，最适合用于检错的是（）

A. $G(x) = 1$
B. $G(x) = x + 1$
C. $G(x) = x^3 + x + 1$
D. $G(x) = x^2$

**分析**：
- A：$r = 0$，无检错能力
- B：$r = 1$，只能检测1位突发错误，能力太弱
- C：$r = 3$，3项多项式，检错能力强 ✓
- D：$G(x) = x^2$ 只有1项，无法检测某些偶数位错误

**答案**：C

好的生成多项式应满足：
1. 最高位和最低位系数为1（保证覆盖所有位）
2. 至少3项以上（保证检错能力）
3. 不能被 $x$ 整除（即不含 $x$ 因子以外的公因子）

### 考题3：编码效率

**题目**：1000位数据采用CRC-16校验，编码效率是多少？

**解**：CRC-16的 $r = 16$，校验位16位。

$$\text{编码效率} = \frac{1000}{1000 + 16} = \frac{1000}{1016} \approx 98.4\%$$

数据越长，CRC编码效率越高。

### 考题4：与海明码的选择

**题目**：某系统需要检测传输过程中可能出现的连续多位翻转，应选择（）

A. 奇偶校验
B. 海明码
C. CRC
D. 重复发送

**分析**：连续多位翻转是"突发错误"，CRC专门针对突发错误设计，能检测所有 $\leq r$ 位的突发错误。

**答案**：C

## 易错陷阱

### 陷阱1：CRC不能纠错
- CRC只能检错，不能纠错（除非结合ARQ重传机制）
- 如果题目要求"纠错"，应选择海明码

### 陷阱2：CRC不是万能的
- 长度超过 $r$ 的突发错误有一定概率检测不到
- 概率为 $2^{-r}$（CRC-16的漏检率为 $2^{-16} \approx 0.0015\%$）

### 陷阱3：模2除法 ≠ 普通除法
- 模2除法的"减法"是XOR（异或），不是十进制减法
- 没有借位的概念
- 这是最常见的计算错误

### 陷阱4：生成多项式的选择
- 并非任意多项式都适合作为生成多项式
- $G(x)$ 必须满足一定条件才能保证良好的检错能力
- 408中通常直接给出 $G(x)$，不需要自行选择

## 知识关联

### TCP/IP协议栈中的CRC
- **以太网帧**：使用CRC-32校验
- **PPP协议**：使用CRC-16或CRC-32
- **Wi-Fi**：使用CRC-32
- **USB**：使用CRC-5（令牌包）和CRC-16（数据包）

### 408考试规律
- CRC的模2除法计算是大题常见考点
- 检错能力是选择题常见考点
- 与海明码的对比是易混淆选择题
- 奇偶校验、海明码、CRC的适用场景对比需要掌握
