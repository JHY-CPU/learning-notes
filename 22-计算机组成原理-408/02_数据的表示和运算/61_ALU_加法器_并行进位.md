# 62_ALU_加法器_并行进位

## 核心概念

### 为什么要并行进位？

串行进位加法器的延迟为 $O(n)$，高位必须等低位进位到来后才能计算。**并行进位**（超前进位）的目标是让各位的进位**同时计算**，将延迟从 $O(n)$ 降到 $O(1)$ 或 $O(\log n)$。

### 关键定义：进位产生和进位传递

对于第 $i$ 位：

**进位产生（Generate）**：
$$G_i = A_i \cdot B_i$$

含义：当 $A_i$ 和 $B_i$ 都为 1 时，**必然**产生进位，不管低位的进位是什么。

**进位传递（Propagate）**：
$$P_i = A_i \oplus B_i$$

含义：当 $A_i$ 和 $B_i$ 恰好一个为 1 时，**如果**低位有进位，就传递上去。

> 注意：部分教材定义 $P_i = A_i + B_i$，含义略有不同。408考试以教材为准，唐朔飞版用 $P_i = A_i \oplus B_i$。

### 并行进位的基本思想

利用 $G_i$ 和 $P_i$，将进位 $C_i$ 展开为只与 $A$、$B$ 有关的表达式，从而各位可以同时计算。

### 并行进位公式推导

基本进位公式：
$$C_i = G_i + P_i \cdot C_{i-1}$$

展开：

**$C_0$**（假设最低位进位输入为 $C_{in}$）：
$$C_0 = G_0 + P_0 \cdot C_{in}$$

**$C_1$**：
$$C_1 = G_1 + P_1 \cdot C_0 = G_1 + P_1 \cdot (G_0 + P_0 \cdot C_{in}) = G_1 + P_1 G_0 + P_1 P_0 C_{in}$$

**$C_2$**：
$$C_2 = G_2 + P_2 \cdot C_1 = G_2 + P_2 G_1 + P_2 P_1 G_0 + P_2 P_1 P_0 C_{in}$$

**$C_3$**：
$$C_3 = G_3 + P_3 \cdot C_2 = G_3 + P_3 G_2 + P_3 P_2 G_1 + P_3 P_2 P_1 G_0 + P_3 P_2 P_1 P_0 C_{in}$$

### 重要观察

从展开式可以看出：
- 每个 $C_i$ 都**只与** $G_j$, $P_j$（$j \leq i$）和 $C_{in}$ 有关
- $G_j$ 和 $P_j$ 只需要 $A_j$ 和 $B_j$，可以**同时计算**
- 因此所有 $C_i$ 可以**同时产生**！

### 超前进位的延迟

- 产生所有 $G_i$, $P_i$：1 级门延迟
- 产生所有 $C_i$：$\lceil \log_2 n \rceil + 1$ 级门延迟（因为 $C_i$ 的表达式中有 $i+1$ 个乘积项相或）
- 产生 $S_i = P_i \oplus C_{i-1}$：1 级门延迟

**总延迟**：$O(\log n)$，远小于串行进位的 $O(n)$。

### 4位超前进位表达式汇总

$$G_i = A_i \cdot B_i, \quad P_i = A_i \oplus B_i, \quad i = 0,1,2,3$$

$$C_0 = G_0 + P_0 C_{in}$$

$$C_1 = G_1 + P_1 G_0 + P_1 P_0 C_{in}$$

$$C_2 = G_2 + P_2 G_1 + P_2 P_1 G_0 + P_2 P_1 P_0 C_{in}$$

$$C_3 = G_3 + P_3 G_2 + P_3 P_2 G_1 + P_3 P_2 P_1 G_0 + P_3 P_2 P_1 P_0 C_{in}$$

$$S_i = P_i \oplus C_{i-1}, \quad i = 0,1,2,3$$

其中 $C_{-1} = C_{in}$。

### 超前进位的问题

4位以上的超前进位，表达式中乘积项的数量急剧增加：
- $C_3$ 有 5 个乘积项
- $C_7$ 有 9 个乘积项
- $C_{15}$ 有 17 个乘积项

直接用超前进位实现 32 位或 64 位加法器不实际，因此需要**分组**处理。

## 原理分析

### 超前进位的硬件实现

```
A₃B₃  A₂B₂  A₁B₁  A₀B₀  C_in
 │     │     │     │     │
 ▼     ▼     ▼     ▼     │
 G₃P₃  G₂P₂  G₁P₁  G₀P₀  │
 │     │     │     │     │
 ▼     ▼     ▼     ▼     ▼
┌────────────────────────────┐
│    超前进位逻辑 (CLA)       │
│  C₀ = G₀ + P₀·C_in        │
│  C₁ = G₁ + P₁·G₀ + ...    │
│  C₂ = ...                  │
│  C₃ = ...                  │
└──┬────┬────┬────┬─────────┘
   │    │    │    │
   ▼    ▼    ▼    ▼
 S₃⊕C₂ S₂⊕C₁ S₁⊕C₀ S₀⊕C₋₁
```

### 超前进位与串行进位的对比

| 特征 | 串行进位 | 超前进位 |
|------|---------|---------|
| 延迟 | $O(n)$ | $O(\log n)$ |
| 硬件复杂度 | 简单 | 复杂 |
| 门数量 | 少 | 多 |
| 适用场景 | 低速、低成本 | 高速 |
| 扩展性 | 容易 | 需要分组 |

## 直观理解

### 类比

- **串行进位**：排队办事，必须一个一个来
- **超前进位**：所有人同时提交材料，系统同时处理

### 记忆要点

- **$G_i = A_i \cdot B_i$（产生）**
- **$P_i = A_i \oplus B_i$（传递）**
- **$C_i = G_i + P_i \cdot C_{i-1}$（基本公式）**
- **超前进位 = 展开 $C_i$，使其与 $C_{i-1}$ 无关**
- **延迟 $O(\log n)$ 但硬件复杂**

## 知识关联

- 串行进位（第61节）是超前进位的对比对象
- 分组超前进位（第63节）是超前进位的实用形式
- CLA（Carry Lookahead Adder）是超前进位加法器的缩写
- 408考试要求掌握 $C_i$ 的展开式推导
