# 25_移码_408考点

## 核心概念

### 408常考题型
1. 移码与补码的转换
2. 移码表示的数值比较
3. 浮点数阶码为什么用移码
4. 移码的表示范围和0的表示

---

## 典型考题

### 考题1：补码与移码转换

**题目**：$[X]_{\text{补}} = 10110110$，求 $[X]_{\text{移}}$。

**解**：移码 = 补码符号位取反

$$[X]_{\text{移}} = 00110110$$

**验证**：$[X]_{\text{补}} = 10110110$，是负数（符号位1）
- 补码求真值：取反+1 = `01001001` + 1 = `01001010` = 74，所以 $X = -74$
- 移码 = $128 + (-74) = 54 = 00110110$ ✓

### 考题2：移码大小比较

**题目**：以下移码中，最大的数是（）

A. `01100000`  B. `10000000`  C. `11111111`  D. `00000000`

**分析**：移码可以直接按二进制大小比较：
- A = 96（真值 = 96 - 128 = -32）
- B = 128（真值 = 0）
- C = 255（真值 = 127）
- D = 0（真值 = -128）

$255 > 128 > 96 > 0$

**答案**：C（真值最大，为+127）

### 考题3：移码0的表示

**题目**：8位移码中，0的表示是什么？

**解**：$[0]_{\text{移}} = 128 + 0 = 128 = 10000000$

与补码不同：
- 补码的0：`00000000`
- 移码的0：`10000000`

### 考题4：移码范围

**题目**：8位移码的表示范围是什么？

**解**：偏置值 = 128

$$[X]_{\text{移}} = 128 + X$$

当 $X = -128$：$[X]_{\text{移}} = 0 = 00000000$

当 $X = +127$：$[X]_{\text{移}} = 255 = 11111111$

**答案**：移码表示范围为 $00000000 \sim 11111111$（即0~255），对应真值范围 $-128 \sim +127$。

### 考题5：浮点数阶码

**题目**：IEEE 754单精度浮点数的阶码用8位移码表示，偏置值为127。阶码`10000010`表示的阶数是多少？

**解**：
$$\text{阶数} = [E]_{\text{移}} - 127 = 130 - 127 = +3$$

## 易错陷阱

### 陷阱1：移码和补码的关系
- 移码 = 补码**符号位取反**（不是全部取反！）
- 仅最高位不同，其余位完全相同

### 陷阱2：移码0
- 移码0 = $100\cdots0$（偏置值本身）
- 不要与补码0（$000\cdots0$）混淆

### 陷阱3：IEEE 754的偏置值
- 标准移码偏置值 = $2^{n-1}$
- IEEE 754偏置值 = $2^{n-1} - 1$（8位阶码用127而非128）
- 这是因为IEEE 754不使用移码的全0和全1编码（保留给0和无穷大）

### 陷阱4：移码比较
- 移码可以直接比大小，但**不是和0比较判断正负**
- 移码与偏置值比较：移码 < 偏置值 → 真值为负；移码 > 偏置值 → 真值为正；移码 = 偏置值 → 真值为0

## 知识关联

### 与浮点数的联系
- 浮点数章节会详细讲解IEEE 754格式
- 阶码用移码的原因：便于比较大小和排序
- 移码0唯一保证了浮点数0的表示唯一

### 408考试规律
- 移码单独出题较少，常与浮点数结合
- 选择题中常与补码、原码、反码一起考查
- 需要掌握移码与补码的快速转换
