# 18_无符号整数

## 核心概念

### 定义
**无符号整数**（Unsigned Integer）是一种不包含符号位的整数表示方法，所有位都用于表示数值的大小。只能表示非负整数（包括0）。

### 表示方法
$n$ 位无符号整数的每一位都是数值位，权值从低位到高位依次为 $2^0, 2^1, 2^2, \ldots, 2^{n-1}$。

$$X = \sum_{i=0}^{n-1} x_i \times 2^i$$

其中 $x_i \in \{0, 1\}$。

### 表示范围

$n$ 位无符号整数的表示范围：

$$0 \leq X \leq 2^n - 1$$

| 位数 $n$ | 最小值 | 最大值 | 可表示个数 |
|----------|--------|--------|-----------|
| 8 | 0 | 255 | 256 |
| 16 | 0 | 65535 | 65536 |
| 32 | 0 | 4294967295 | $2^{32}$ |
| $n$ | 0 | $2^n - 1$ | $2^n$ |

---

## 原理分析

### 无符号整数的加法

$n$ 位无符号整数加法：直接按二进制加法运算。

**溢出条件**：当结果超过 $2^n - 1$ 时，产生溢出。

**溢出检测**：最高位产生进位即为溢出。

**示例**（8位无符号）：

$200 + 100 = 300$，而 $300 > 255$，溢出！

$$11001000 + 01100100 = 1\ 00101100$$

最高位进位1，结果模 $2^8 = 256$ 后为 $00101100 = 44$（错误结果）。

### 无符号整数的减法

无符号整数减法：$A - B$（要求 $A \geq B$）

在计算机中，减法通过**补码加法**实现：
$$A - B = A + (2^n - B)$$

其中 $2^n - B$ 是 $B$ 对 $2^n$ 的补数。

**示例**（8位）：$100 - 42$

$$100 + (256 - 42) = 100 + 214 = 314$$

模 $256$ 后：$314 - 256 = 58$ ✓

### 自然溢出（模运算）

无符号整数运算本质上是**模 $2^n$ 运算**：

$$A + B \mod 2^n$$
$$A - B \mod 2^n$$

当结果超出范围时，自动对 $2^n$ 取模，这就是**自然溢出**或**模运算溢出**。

## 直观理解

### 生活类比

无符号整数就像一个**只显示正数的计数器**：
- 汽车里程表（假设从000000开始）：999999 + 1 = 000000（溢出/翻转）
- 时钟：12点过后是1点（模12运算）
- 无符号整数的溢出就像计数器"归零"

### 记忆要点
- 无符号整数**没有符号位**，所有位都是数值位
- 只能表示 **$\geq 0$** 的整数
- 运算是**模 $2^n$** 的循环运算
- 溢出判断很简单：看最高位是否有进位

## 知识关联

### 跨章节联系
- **C语言**：`unsigned int`、`unsigned char` 等类型对应无符号整数
- **地址**：内存地址本质上是无符号整数
- **浮点数**：无符号整数不能直接参与浮点运算，需要类型转换
- **逻辑运算**：位与、位或、位异或等操作通常在无符号整数上进行

### 408考试要点
- 无符号整数的表示范围
- 加法溢出的判断（最高位进位 = 溢出）
- 模运算特性
- 与有符号整数的对比

### 易错陷阱
1. **溢出不代表错误**：无符号整数溢出后结果是对 $2^n$ 取模的值，不是"错误"
2. **$A < B$ 时的减法**：如果 $A < B$，结果会是一个很大的正数（因为取模）
3. **与有符号整数混用**：同一个位串，无符号和有符号解释完全不同
