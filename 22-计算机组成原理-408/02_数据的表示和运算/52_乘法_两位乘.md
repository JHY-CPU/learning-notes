# 53_乘法_两位乘

## 核心概念

### 两位乘法的定义

两位乘法（也称为**基4乘法**）是每次检查乘数的**2位**，从而将迭代次数减半的乘法方法。

- 原码一位乘：每次看1位，n位乘数需要n轮
- 原码两位乘：每次看2位，n位乘数需要 **n/2 轮**

### 为什么需要两位乘？

一位乘法每次只处理1位，硬件利用率低。两位乘法将速度提高一倍，是现代高性能乘法器的基础。

### 两位乘的判断规则

每次检查乘数的2位 $(y_{i-1}, y_i)$，根据其值决定操作：

| $(y_{i-1}, y_i)$ | 操作 | 说明 |
|-------------------|------|------|
| $00$ | 不加，右移2位 | 相当于乘以0 |
| $01$ | 加 $[x]_补$，右移2位 | 相当于乘以1 |
| $10$ | 加 $2[x]_补$，右移2位 | 相当于乘以2 |
| $11$ | 加 $3[x]_补$，右移2位 | 相当于乘以3 |

### 改进的两位乘规则（常用方法）

将 $3x$ 分解为 $4x - x$，即先减 $x$，再移位两次。或者采用"部分积+乘数"联合移2位的方式：

| $(y_{i-1}, y_i)$ | 操作 |
|-------------------|------|
| $00$ | 不加，右移2位 |
| $01$ | 加 $x$，右移2位 |
| $10$ | 加 $2x$（左移1位后加），右移2位 |
| $11$ | 减 $x$（即加 $[-x]_补$），然后加 $4x$ ... |

### 原码两位乘

对于**原码**两位乘法，符号位仍然单独处理。每次看乘数的2位：

| $(y_{i-1}, y_i)$ | 操作 |
|-------------------|------|
| $00$ | 不加，右移2位 |
| $01$ | 加 $|x|$，右移2位 |
| $10$ | 加 $2|x|$，右移2位 |
| $11$ | 加 $3|x|$，右移2位 |

**执行方式**：需要一个辅助触发器C（记录是否需要额外加一次被乘数），以及保存 $2|x|$ 的寄存器。

### 补码两位乘（扩展Booth算法）

补码两位乘是将 Booth 算法扩展为每次检查**3位**（当前2位+前一位的附加位）：

| $(y_{i-1}, y_i, y_{i+1})$ | 操作 |
|---------------------------|------|
| $000$ 或 $111$ | 不加，右移2位 |
| $001$ 或 $010$ | 加 $[x]_补$，右移2位 |
| $011$ | 加 $2[x]_补$，右移2位 |
| $100$ | 加 $-2[x]_补$，右移2位 |
| $101$ 或 $110$ | 加 $[-x]_补$，右移2位 |

**需要预先准备**：
- $[x]_补$
- $[-x]_补$
- $2[x]_补$（$[x]_补$ 左移1位）
- $-2[x]_补$（$[-x]_补$ 左移1位）

## 原理分析

### 迭代次数对比

设乘数为 n 位：

| 方法 | 每次看几位 | 迭代次数 | 加法判断次数 |
|------|-----------|---------|------------|
| 一位乘 | 1 | n | n |
| 两位乘 | 2 | n/2 | n/2 |
| k位乘 | k | n/k | n/k |

### 硬件复杂度

两位乘需要：
- 更多的被加数选择（不只是 $x$ 和 $-x$，还有 $2x$ 和 $-2x$）
- 更复杂的控制逻辑
- 可能需要更多的加法器操作

速度和复杂度的权衡：两位乘在实际中非常常用，是速度和复杂度的最佳平衡点。

## 直观理解

### 类比

一位乘好比"一步一个台阶"，两位乘好比"一步两个台阶"——速度快了一倍，但每步的动作更复杂了。

- 补码两位乘与 Booth 算法（每次看2位 vs 看3位）有关联
- 现代处理器的乘法器通常使用 2-4 位的并行乘法
