# 24_移码

## 核心概念

### 定义
**移码**（Excess/Offset Binary）是在真值上加一个**偏置值**（Bias）得到的编码。

$$[X]_{\text{移}} = \text{偏置值} + X$$

通常偏置值取 $2^{n-1}$。

### 编码规则

$n$ 位移码（偏置值 $= 2^{n-1}$）：

$$[X]_{\text{移}} = 2^{n-1} + X$$

其中 $X$ 的范围为 $-2^{n-1} \leq X \leq +(2^{n-1} - 1)$。

### 移码与补码的关系

**移码 = 补码的符号位取反**

或者说：**移码和补码仅符号位相反，数值位完全相同**。

**推导**：
- 补码中正数的符号位为0，负数的符号位为1
- 移码 $[X]_{\text{移}} = 2^{n-1} + X$
- 补码 $[X]_{\text{补}} = 2^n + X$（负数时，模意义下等价于 $X$）

对于正数 $X$：
- $[X]_{\text{补}}$ 符号位为0
- $[X]_{\text{移}} = 2^{n-1} + X$，符号位为1（因为 $2^{n-1}$ 贡献了最高位的1）

对于负数 $X$（$X < 0$）：
- $[X]_{\text{补}} = 2^n + X$，符号位为1
- $[X]_{\text{移}} = 2^{n-1} + X$，符号位为0（因为 $X$ 是负数，$2^{n-1} + X < 2^{n-1}$）

所以：**移码符号位与补码相反**。

### 移码0的唯一性

移码中**只有一个零**：

$$[0]_{\text{移}} = 2^{n-1} + 0 = 2^{n-1} = 100\cdots0 \text{（n位）}$$

移码的 $-2^{n-1}$：
$$[-2^{n-1}]_{\text{移}} = 2^{n-1} + (-2^{n-1}) = 0 = 000\cdots0 \text{（n位）}$$

## 原理分析

### 移码示例（8位，偏置值=128）

| 真值 | 补码 | 移码 | 仅符号位不同 |
|------|------|------|------------|
| $+127$ | `01111111` | `11111111` | 0→1 |
| $+1$ | `00000001` | `10000001` | 0→1 |
| $+0$ | `00000000` | `10000000` | 0→1 |
| $-0$ | `00000000` | `10000000` | 同上 |
| $-1$ | `11111111` | `01111111` | 1→0 |
| $-127$ | `10000001` | `00000001` | 1→0 |
| $-128$ | `10000000` | `00000000` | 1→0 |

### 移码的核心优势：可直接比较大小

移码最重要的性质：**可以直接按二进制大小比较真值大小**，无需考虑符号位。

**原因**：移码将整个数轴平移了 $2^{n-1}$，使得所有值都变为非负数，二进制大小顺序与真值大小顺序一致。

**示例**：
- $+5$ 的移码：$128 + 5 = 133 = 10000101$
- $-3$ 的移码：$128 + (-3) = 125 = 01111101$
- $133 > 125$，对应 $+5 > -3$ ✓

对比补码：
- $[+5]_{\text{补}} = 00000101$
- $[-3]_{\text{补}} = 11111101$
- 补码直接比：$00000101 < 11111101$（但 $+5 > -3$）
- 补码不能直接比较大小！

### 移码的应用：浮点数阶码

IEEE 754浮点数的**阶码（Exponent）使用移码表示**：

- 单精度（32位）：8位阶码，偏置值 $= 127$
- 双精度（64位）：11位阶码，偏置值 $= 1023$

使用移码的原因：
1. 阶码需要比较大小（浮点数比较时先比阶码）
2. 移码方便排序和比较
3. 移码0唯一，不会出现歧义

## 直观理解

### 生活类比
移码就像考试成绩加上一个基础分（如50分）：
- 原始分 -20 分 → 移码 50 + (-20) = 30
- 原始分 +10 分 → 移码 50 + 10 = 60
- 30 < 60，对应 -20 < +10，大小关系直接成立

### 记忆技巧
1. **移码 = 补码符号位取反**（最实用的转换方法）
2. **移码0**：$100\cdots0$（偏置值本身）
3. **可直接比大小**：移码的二进制大小直接反映真值大小

## 知识关联

### 408考试要点
- 移码的定义和偏置值的选取
- 移码与补码的关系（仅符号位不同）
- 移码0唯一且为 $100\cdots0$
- 浮点数阶码使用移码的原因
- 移码的表示范围

### 易错陷阱
1. **移码的偏置值**：通常是 $2^{n-1}$，但IEEE 754中是 $2^{n-1}-1$（单精度是127不是128）
2. **移码和补码的转换**：只需要取反符号位，其他位不变
3. **移码0的值**：$100\cdots0$，不是 $000\cdots0$
