# 86_易错辨析_补码与移码

## 核心概念

### 补码与移码的关系

补码和移码是两种有符号整数的编码方式，它们的**数值部分完全相同，仅符号位不同**。

### 对比表（以 8 位为例）

| 真值 | 补码 | 移码 |
|------|------|------|
| +127 | 01111111 | 11111111 |
| +1 | 00000001 | 10000001 |
| 0 | 00000000 | 10000000 |
| -1 | 11111111 | 01111111 |
| -127 | 10000001 | 00000001 |
| -128 | 10000000 | 00000000 |

### 关键公式

$$[x]_{移} = [x]_{补} + 2^{n-1}$$

即：移码 = 补码的符号位取反（加上 $2^{n-1}$ 等价于最高位翻转）。

或者：$[x]_{移} = 2^{n-1} + x$（其中 $x$ 是真值）

### 补码的特点

- 最高位是符号位：0 正 1 负
- 零的表示唯一：$000...0$
- 负数比正数多一个：$-128$（8位时）
- 可以直接做加减法

### 移码的特点

- 最高位不是纯粹的符号位
- **全 0 表示最小值**，**全 1 表示最大值**
- 零的表示：$100...0$
- **比较大小方便**：可以直接用无符号比较器

### 为什么浮点数的阶码用移码？

1. **比较方便**：浮点数比较时，首先比较阶码。移码可以直接用无符号比较器，无需判断符号位。
2. **零值对齐**：移码的最小值全 0，使得浮点零值（阶码全 0、尾数全 0）的判断更简单。
3. **范围对称**：移码的表示范围关于 $2^{n-1}$ 对称。

### 示例：8 位移码比较

$[+5]_{移} = 10000101$，$[-3]_{移} = 01111101$

无符号比较：$10000101 = 133 > 01111101 = 125$

所以 $+5 > -3$，直接用无符号比较器即可。

### 补码与移码的转换

$$[x]_{移} = [x]_{补} \oplus 1000...0$$

即：**最高位取反**，其余位不变。

| 操作 | 补码 → 移码 | 移码 → 补码 |
|------|-----------|-----------|
| 方法 | 符号位取反 | 符号位取反 |

## 直观理解

### 类比

- **补码**：像温度计，$0$ 度在中间，正负对称
- **移码**：像把温度计整体上移，所有刻度都是正的，方便比较

### 记忆要点

- **补码和移码只有符号位不同**
- **移码全 0 最小，全 1 最大**
- **阶码用移码 = 方便比较**
- **408 常考：补码移码互转、移码的比较优势**

## 408 考点

### 考点1：补码与移码互转

**典型问法**：$[x]_{补} = 10101010$，则 $[x]_{移} =$？

**答案**：$00101010$（最高位取反）

### 考点2：移码的比较

**典型问法**：为什么浮点数阶码用移码？

**答案**：移码可以用无符号比较器直接比较大小，简化硬件。

### 考点3：移码的零值

**典型问法**：8 位移码中，零的表示是？

**答案**：$10000000$

### 考点4：补码 vs 移码的范围

8 位时：
- 补码：$[-128, +127]$
- 移码：$[-128, +127]$（范围相同，编码不同）

## 易错总结

1. **移码不是补码的按位取反**，只有符号位不同
2. **移码的零是 $100...0$**，不是 $000...0$
3. **移码全 0 对应最小真值**（不是零）
4. **移码和补码的表示范围相同**
5. **移码适合比较，补码适合运算**
