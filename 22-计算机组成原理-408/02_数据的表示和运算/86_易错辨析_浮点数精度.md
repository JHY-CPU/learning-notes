# 87_易错辨析_浮点数精度

## 核心概念

### 浮点数精度问题的本质

浮点数用**有限位数**表示**无限精度**的实数，必然存在精度损失。

### 精度损失的来源

**1. 表示误差**

不是所有十进制小数都能精确表示为有限位二进制小数。

$$0.1_{10} = 0.0001100110011..._2$$（无限循环）

IEEE 754 单精度只能存储 23 位尾数，必须截断，导致 $0.1$ 不精确。

**2. 舍入误差**

对阶右移、乘法截断等操作引入的误差。

**3. 运算误差**

两个不精确的数运算，误差可能累积或放大。

### 经典案例：$0.1 + 0.2 \neq 0.3$

在大多数编程语言中：

```python
>>> 0.1 + 0.2
0.30000000000000004
```

**原因**：
1. $0.1$ 和 $0.2$ 都不能精确表示为二进制浮点数
2. 它们的存储值是近似值
3. 近似值的和不是 $0.3$ 的近似值

### IEEE 754 中 $0.1$ 的表示

$0.1 \times 2 = 0.2 \to 0$
$0.2 \times 2 = 0.4 \to 0$
$0.4 \times 2 = 0.8 \to 0$
$0.8 \times 2 = 1.6 \to 1$
$0.6 \times 2 = 1.2 \to 1$
$0.2 \times 2 = 0.4 \to 0$
...循环

$0.1 = 0.00011001100110011..._2$

规格化：$1.1001100110011... \times 2^{-4}$

IEEE 754 存储（截断后）：$1.10011001100110011001100 \times 2^{-4}$

这个值实际上略大于 $0.1$。

### 精度与位数的关系

| 格式 | 尾数位数 | 十进制精度 | 最大相对误差 |
|------|---------|-----------|------------|
| Half | 10 (+1) | ~3.3 位 | $\approx 2^{-11} \approx 0.05\%$ |
| Float | 23 (+1) | ~7.2 位 | $\approx 2^{-24} \approx 0.000006\%$ |
| Double | 52 (+1) | ~15.9 位 | $\approx 2^{-53} \approx 0.00000000000001\%$ |

### 精度不均匀性

浮点数的精度不是均匀的：
- 阶码越小（数越接近零），相邻可表示数的间距越小，精度越高
- 阶码越大（数远离零），间距越大，精度越低

**示例**（float）：
- 在 $[1, 2)$ 区间，间距 $= 2^{-23}$
- 在 $[2, 4)$ 区间，间距 $= 2^{-22}$
- 在 $[2^{10}, 2^{11})$ 区间，间距 $= 2^{-13}$

### 如何避免精度问题

1. **比较时用误差范围**：`abs(a - b) < epsilon`
2. **金融计算用定点数或 decimal 类型**
3. **累加时用 Kahan 求和算法**
4. **避免对非常大和非常小的数直接加减**（大数吃小数）

### 大数吃小数

```python
>>> 1e16 + 1.0 - 1e16
0.0  # 1.0 被"吃掉"了
```

原因：$1.0$ 相对于 $10^{16}$ 太小，对阶后 $1.0$ 被右移 53 位以上，全部丢失。

## 直观理解

### 类比

浮点数就像一把刻度不均匀的尺子：
- 在零附近刻度很密（精度高）
- 远离零刻度很稀（精度低）
- 有些位置没有刻度（如 0.1）

### 记忆要点

- **$0.1 + 0.2 \neq 0.3$ 是浮点数精度问题的标志**
- **浮点数精度 = 尾数位数 $\times \log_{10}2$**
- **精度不均匀**：接近零精度高
- **大数吃小数**是严重的精度问题

## 408 考点

### 考点1：浮点数精度的决定因素

**典型问法**：浮点数的精度由什么决定？

**答案**：尾数位数（不是阶码位数）

### 考点2：浮点数不能精确表示所有十进制小数

**典型问法**：以下哪个数不能精确表示为 IEEE 754 单精度浮点数？

A. 0.5　　B. 0.25　　C. 0.1　　D. 0.125

**答案**：C（$0.1 = 0.000110011..._2$，无限循环）

$0.5 = 2^{-1}$、$0.25 = 2^{-2}$、$0.125 = 2^{-3}$ 都可以精确表示。

### 考点3：大数吃小数

**典型问法**：浮点数加法中，两个数量级相差很大的数相加，会发生什么？

**答案**：小数可能被完全忽略（对阶右移后低位全部丢失）。
