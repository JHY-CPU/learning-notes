# 26_原码、补码、反码、移码对比

## 核心概念

### 四种编码综合对比表（以8位为例）

| 真值 | 原码 | 反码 | 补码 | 移码 |
|------|------|------|------|------|
| +127 | `01111111` | `01111111` | `01111111` | `11111111` |
| +1 | `00000001` | `00000001` | `00000001` | `10000001` |
| +0 | `00000000` | `00000000` | `00000000` | `10000000` |
| -0 | `10000000` | `11111111` | `00000000` | `10000000` |
| -1 | `10000001` | `11111110` | `11111111` | `01111111` |
| -127 | `11111111` | `10000000` | `10000001` | `00000001` |
| -128 | 不存在 | 不存在 | `10000000` | `00000000` |

### 关键属性对比

| 属性 | 原码 | 反码 | 补码 | 移码 |
|------|------|------|------|------|
| **正数表示** | 同真值 | 同真值 | 同真值 | 真值+偏置 |
| **负数表示** | 符号+绝对值 | 数值位取反 | 取反+1 | 真值+偏置 |
| **零的个数** | 2个 | 2个 | 1个 | 1个 |
| **表示范围(8位)** | -127~+127 | -127~+127 | -128~+127 | 0~255(映射-128~+127) |
| **可表示个数** | 255 | 255 | 256 | 256 |
| **加减法** | 需判断符号 | 需循环进位 | 统一加法 | 需判断偏置 |
| **硬件复杂度** | 高 | 中 | 低 | 低 |
| **主要用途** | 浮点尾数 | 历史/校验和 | 整数运算 | 浮点阶码 |

### 编码转换关系

```
        取反         +1
原码 ----------→ 反码 ----------→ 补码
正数不变          正数不变         正数不变
负数:符号+绝对值   负数:取反        负数:取反+1
```

$$\text{补码} \xleftrightarrow{\text{符号位取反}} \text{移码}$$

## 原理分析

### 编码规则速查

#### 原码
$$[X]_{\text{原}} = \begin{cases} 0, X & X \geq 0 \\ 1, |X| & X < 0 \end{cases}$$

#### 反码
$$[X]_{\text{反}} = \begin{cases} X & X \geq 0 \\ \text{原码数值位取反} & X < 0 \end{cases}$$

#### 补码
$$[X]_{\text{补}} = \begin{cases} X & X \geq 0 \\ \text{反码} + 1 & X < 0 \end{cases}$$

等价定义：$[X]_{\text{补}} = 2^n + X$（模 $2^n$，$X < 0$ 时）

#### 移码
$$[X]_{\text{移}} = 2^{n-1} + X$$

等价于：补码符号位取反。

### 互相转换速查

| 转换 | 正数 | 负数 |
|------|------|------|
| 原码 → 反码 | 不变 | 数值位取反 |
| 反码 → 补码 | 不变 | 反码 + 1 |
| 原码 → 补码 | 不变 | 数值位取反 + 1 |
| 补码 → 原码 | 不变 | 数值位取反 + 1（可迭代） |
| 补码 → 移码 | 符号位取反 | 符号位取反 |
| 移码 → 补码 | 符号位取反 | 符号位取反 |

## 直观理解

### 记忆口诀

- **原码**："符号+绝对值"，最直观但运算复杂
- **反码**："取反"，是原码到补码的桥梁
- **补码**："取反+1"，计算机内部标准
- **移码**："补码符号取反"，用于浮点阶码

### 选择题快速判断技巧

1. 看到最高位1且其余位全0 → 如果是补码则 $-128$，如果是原码则 $-0$
2. 看到全1 → 如果是补码则 $-1$，如果是原码则 $-127$，如果是反码则 $-0$
3. 看到全0 → 如果是补码则 $0$，如果是移码则 $-128$

## 知识关联

### 使用场景

| 编码 | 使用场景 |
|------|---------|
| 原码 | 浮点数尾数（IEEE 754中有些实现）、人类直观理解 |
| 反码 | 网络校验和（IP/TCP/UDP checksum）、历史计算机 |
| 补码 | 现代计算机有符号整数（C语言的int等） |
| 移码 | 浮点数阶码（IEEE 754） |

### 408考试要点
- 四种编码的定义、范围、0的表示
- 编码之间的转换（特别是补码 ↔ 其他编码）
- 各编码的优缺点和适用场景
- 给定位数的表示范围计算
- 特殊值的识别（$-2^{n-1}$、$-0$、全0、全1）

### 易错陷阱
1. **正数的四种编码都相同**（除了移码），不要对正数做额外处理
2. **$-2^{n-1}$ 只有补码和移码能表示**，原码和反码不能
3. **补码和原码的转换不是简单的取反+1**（负数原码→补码是取反+1，但补码→原码也是取反+1，可以迭代使用直到结果为正数）
4. **移码0是 $100\cdots0$**，不是 $000\cdots0$
