# 56_除法_加减交替法

## 核心概念

### 加减交替法的定义

加减交替法（也称**不恢复余数法**）是对恢复余数法的改进。它的核心思想是：

**不需要显式恢复余数**，而是通过下一步的反向操作来隐式恢复。

### 与恢复余数法的区别

| 特征 | 恢复余数法 | 加减交替法 |
|------|-----------|-----------|
| 不够减时 | 加回除数（恢复），然后左移再减 | 直接左移，**改为加**除数 |
| 每轮操作数 | 固定为减法 | 减法和加法交替 |
| 硬件效率 | 较低（可能多一次加法） | 较高（操作数固定） |
| 最后一步 | 需要判断是否恢复 | 余数为负仍需恢复 |

### 加减交替法的核心思想

当试探减法结果为**负**时（不够减）：
- 恢复余数法：先加回除数（恢复），左移，再减除数
- 加减交替法：**直接左移，然后加除数**（省去恢复步骤）

数学等价性证明：

设当前余数 $R$，除数 $D$：
- 不够减时 $R - D < 0$
- 恢复余数法：$(R - D + D) \times 2 - D = 2R - D$
- 加减交替法：$(R - D) \times 2 + D = 2R - D$

两者数学等价！

### 加减交替法的算法步骤

**初始化**：
- ACC = $|$被除数$|$
- X = $|$除数$|$
- MQ = 0

**迭代过程**（共 n 轮）：

1. 执行操作（减法或加法）：
   - 第1轮：`ACC ← ACC - X`
   - 后续轮：根据上一轮结果决定
     - 上轮 ACC ≥ 0（够减）：本轮 `ACC ← ACC - X`
     - 上轮 ACC < 0（不够减）：本轮 `ACC ← ACC + X`

2. 判断 ACC 符号：
   - ACC ≥ 0：商 1
   - ACC < 0：商 0

3. 左移：ACC ← ACC << 1

4. 重复 n 轮

**最后一步特殊处理**：
- 若第 n 轮后 ACC < 0（余数为负），需要执行 `ACC ← ACC + X` 恢复余数
- 此时商已经确定，余数恢复不影响商

### 简化描述

```
第1步：ACC ← ACC - X
循环n次：
  若ACC ≥ 0：商1，左移，ACC ← ACC - X
  若ACC < 0：商0，左移，ACC ← ACC + X
最后：若ACC < 0，ACC ← ACC + X（恢复余数）
```

## 原理分析

### 为什么加减交替法有效？

关键在于"左移+反向操作"等价于"恢复+左移+原操作"：

**情况1：够减（R ≥ D）**
- 商 1，余数 $R - D$
- 左移得 $2(R - D) = 2R - 2D$
- 下轮减 D：$2R - 3D$
- 正确

**情况2：不够减（R < D）**
- 恢复余数法：加 D 恢复得 R，左移得 2R，减 D 得 $2R - D$
- 加减交替法：左移得 $2(R - D) = 2R - 2D$，加 D 得 $2R - D$
- 两者等价！

### 硬件实现优势

加减交替法的硬件逻辑更简单：
- 不需要判断"是否需要恢复"后"是否需要重新减"
- 每轮的操作类型（加/减）完全由上轮的符号决定
- 流水线更友好

## 直观理解

### 类比

想象在走迷宫：
- 恢复余数法：走错一步，退回来，再换方向走
- 加减交替法：走错一步，不退回来，直接反方向继续走

结果一样，但加减交替法省去了"退回"的动作。

### 记忆口诀

- **"够减减，不够加"**
- **"正商1减，负商0加"**
- **"最后余数若为负，加回除数来恢复"**
