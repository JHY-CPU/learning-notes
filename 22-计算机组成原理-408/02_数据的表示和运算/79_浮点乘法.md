# 80_浮点乘法

## 核心概念

### 浮点乘法的公式

设 $A = M_A \times 2^{E_A}$，$B = M_B \times 2^{E_B}$

$$A \times B = (M_A \times M_B) \times 2^{(E_A + E_B)}$$

### 浮点乘法的步骤

**第一步：阶码相加**

$$E_{结果} = E_A + E_B$$

注意：
- 阶码用**移码**表示时，相加需要修正（减去一个 Bias）
- 阶码用**补码**表示时，直接加法

**第二步：尾数相乘**

$$M_{结果} = M_A \times M_B$$

使用定点乘法（Booth 算法等）。

**第三步：规格化**

尾数乘积可能需要规格化：
- 两个规格化小数的乘积，绝对值范围 $[1/R^2, 1)$
- 如果 $|M_{结果}| \geq 1/R$（即 $1/2$），已规格化
- 如果 $|M_{结果}| < 1/R$，需要左规

实际上，两个 $[1/2, 1)$ 的数相乘，结果在 $[1/4, 1)$，可能需要左移 1 位。

**第四步：舍入**

尾数乘积的位数可能超出存储位数，需要舍入。

**第五步：溢出判断**

阶码相加可能产生溢出：
- 上溢：$E_{结果} > E_{max}$
- 下溢：$E_{结果} < E_{min}$

### 浮点乘法流程图

```
开始
  │
阶码相加：E = E_A + E_B
  │
尾数相乘：M = M_A × M_B
  │
规格化：尾数调整为规格化形式
  │
舍入
  │
溢出判断
  │
结束
```

### 示例

$A = 0.1100 \times 2^3$，$B = 0.1010 \times 2^2$

**阶码相加**：$E = 3 + 2 = 5$

**尾数相乘**：$0.1100 \times 0.1010$

用原码一位乘：
```
      0.1100
    × 0.1010
    ---------
     0000
    1100
   0000
  1100
  ---------
  0.01111000
```

**规格化**：$0.01111000$，最高数值位为 0，需要左规。

左移 1 位：$0.11110000$，阶码减 1：$5 - 1 = 4$

**结果**：$A \times B = 0.1111 \times 2^4$

验证：$A = 0.75 \times 8 = 6$，$B = 0.625 \times 4 = 2.5$，$A \times B = 15$

$0.1111 \times 2^4 = 0.9375 \times 16 = 15$ ✓

### 阶码用移码时的修正

如果阶码用移码（Bias = 127）：

$$E_{移码结果} = e_A + e_B - Bias = (E_A + Bias) + (E_B + Bias) - Bias$$

$$= E_A + E_B + Bias$$

即：先直接加存储的阶码值，再减去一次 Bias。

## 直观理解

### 记忆要点

- **浮点乘法 = 阶码相加 + 尾数相乘**
- **不需要对阶**（与加法的最大区别）
- **规格化最多左移 1 位**（两个 $[1/2,1)$ 相乘 $\geq 1/4$）
- **阶码相加可能溢出**

### 与浮点加法的区别

| 步骤 | 浮点加法 | 浮点乘法 |
|------|---------|---------|
| 阶码操作 | 对阶（比较大小） | 直接相加 |
| 尾数操作 | 加法 | 乘法 |
| 规格化 | 可能左规多次或右规 | 最多左移1位 |
