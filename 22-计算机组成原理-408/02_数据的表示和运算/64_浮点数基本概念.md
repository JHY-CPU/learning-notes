# 65_浮点数基本概念

## 核心概念

### 浮点数的定义

**浮点数**（Floating Point Number）是一种用**科学计数法**表示实数的方法。在计算机中，浮点数将一个数表示为**尾数**和**阶码**两部分。

### 浮点数的数学形式

$$N = M \times R^E$$

其中：
- $N$：要表示的数
- $M$：**尾数**（Mantissa/Significand），表示有效数字
- $R$：**基数/底**（Radix/Base），通常为 2
- $E$：**阶码/指数**（Exponent），表示数量级

### 类比科学计数法

十进制科学计数法：
$$123.45 = 1.2345 \times 10^2$$
- 尾数 $M = 1.2345$
- 基数 $R = 10$
- 阶码 $E = 2$

二进制科学计数法：
$$1011.01 = 0.101101 \times 2^4$$
- 尾数 $M = 0.101101$
- 基数 $R = 2$
- 阶码 $E = 4$

### 定点数的局限性

定点数的局限在于：
- **表示范围有限**：n 位定点小数只能表示 $[0, 1-2^{-n}]$
- **无法表示很大或很小的数**：如 $10^{30}$ 或 $10^{-30}$
- **精度与范围矛盾**：增加位数能扩展范围但浪费存储

### 浮点数的优势

| 特征 | 定点数 | 浮点数 |
|------|--------|--------|
| 表示范围 | 小（固定） | 大（随阶码扩展） |
| 精度 | 均匀 | 与尾数位数相关 |
| 硬件复杂度 | 简单 | 复杂 |
| 适用场景 | 整数、固定范围 | 科学计算、大范围数值 |

### 浮点数的组成

一个浮点数由以下部分组成：

```
┌───────┬────────────────────────────────────┐
│ 阶码E │              尾数M                  │
└───────┴────────────────────────────────────┘
  决定范围              决定精度
```

- **阶码**：决定了数的数量级（范围）
- **尾数**：决定了数的有效数字（精度）

### 浮点数的表示范围

假设尾数 $M$ 为纯小数（$1/2 \leq |M| < 1$），阶码 $E$ 为整数：

$$N_{max} = (1 - 2^{-m}) \times 2^{(2^n - 1)}$$

其中 $m$ 是尾数位数，$n$ 是阶码位数。

浮点数能表示的数的范围远大于同等位数的定点数。

## 原理分析

### 浮点数的规格化

为了保证精度，浮点数通常要求**规格化**（Normalize）：

- **原码尾数**：$M$ 的最高数值位为 1（即 $1/2 \leq |M| < 1$）
- **补码尾数**：$M$ 的符号位与最高数值位不同

规格化的目的是**充分利用每一位来表示精度**。

### 阶码的表示方法

阶码通常用以下方式表示：
- **移码**：便于比较大小（全0最小，全1最大）
- **补码**：也有使用

移码的优点：阶码可以直接用无符号数比较，简化了硬件。

### 浮点数的符号

浮点数的符号位在**尾数**中：
- 尾数的符号就是整个浮点数的符号
- 阶码没有符号位（用移码表示时）

## 直观理解

### 类比

浮点数就像"可调刻度的尺子"：
- **阶码**决定尺子的量程（毫米尺还是千米尺）
- **尾数**决定尺子上刻度的精细程度

不同量程用不同精度的刻度，这就是浮点数"范围大但精度不均匀"的原因。

### 记忆要点

- **浮点数 = $M \times R^E$**
- **阶码管范围，尾数管精度**
- **规格化 = 尾数最高位为有效值**
- **浮点数比定点数范围大得多**
