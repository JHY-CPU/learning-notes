# 78_浮点加减法_计算详解

## 核心概念

本文件通过完整例题，演示浮点加减法的五步过程。

### 设定

- 尾数用**补码**，含 1 位符号 + 4 位数值（共 5 位）
- 阶码用**补码**，含 1 位阶符 + 2 位阶码（共 3 位）
- 基数 $R = 2$
- 规格化形式：尾数符号位与最高数值位不同

---

## 例题 1：$A + B$，其中 $A = 0.1101 \times 2^{01}$，$B = 0.1011 \times 2^{11}$

### 初始状态

$A$：阶码 $01$（$+1$），尾数 $0.1101$
$B$：阶码 $11$（$-1$），尾数 $0.1011$

等等，让我重新设定更清楚的例子。

设阶码 4 位补码，尾数 6 位补码（1符号+5数值），R = 2。

### 例题重设

$A = +0.10110 \times 2^{+3}$，$B = +0.11010 \times 2^{+1}$

**编码**：
- $A$：阶码 $0011$，尾数 $0.10110$
- $B$：阶码 $0001$，尾数 $0.11010$

### 第一步：对阶

- $E_A = +3$，$E_B = +1$
- $E_B < E_A$，小阶向大阶看齐
- $B$ 的尾数右移 $3 - 1 = 2$ 位
- $B$ 的尾数：$0.11010 \to 0.001101$（右移2位后）
- $B$ 的阶码改为 $0011$

对阶后：
- $A$：阶码 $0011$，尾数 $0.10110$
- $B$：阶码 $0011$，尾数 $0.00111$（$0.00110$ 右移2位后，5位截断为 $0.00110$，考虑到舍入...）

> 注：右移时低位丢失，这里先不考虑舍入，用截断。

对阶后 $B$ 尾数：$0.00110$（5位，截断）

### 第二步：尾数加法

$$M_A + M_B = 0.10110 + 0.00110 = 0.11100$$

### 第三步：规格化

结果 $0.11100$：符号位 $0$，最高数值位 $1$，不同，已规格化！

### 第四步：舍入

本例未发生右移，无需舍入。

### 第五步：溢出判断

阶码 $0011 = +3$，在 $[-8, +7]$ 范围内，无溢出。

### 最终结果

$$A + B = 0.11100 \times 2^{+3}$$

验证：$A = 0.10110 \times 2^3 = 0.6875 \times 8 = 5.5$，$B = 0.11010 \times 2^1 = 0.8125 \times 2 = 1.625$

$A + B = 7.125$，$0.11100 \times 2^3 = 0.875 \times 8 = 7.0$

差异来自对阶时的截断误差。如使用更精确的对阶，结果一致。

## 例题 2：需要右规的情况

$A = 0.11010 \times 2^{+2}$，$B = 0.10110 \times 2^{+2}$

### 第一步：对阶

$E_A = E_B = +2$，不需要对阶。

### 第二步：尾数加法

$$0.11010 + 0.10110 = 1.10000$$

结果尾数 $1.10000$，绝对值 $\geq 1$（溢出），需要右规。

### 第三步：右规

尾数右移 1 位：$1.10000 \to 1.11000$

> 注意：补码右移高位补符号位。原 $1.10000$ 右移得 $1.11000$。

阶码加 1：$+2 \to +3$

结果：$1.11000 \times 2^{+3}$

### 验证

$A = 0.11010 \times 4 = 0.8125 \times 4 = 3.25$
$B = 0.10110 \times 4 = 0.6875 \times 4 = 2.75$
$A + B = 6.0$

$1.11000$（补码）$= -0.01000 = -1/32$... 这里有问题。

实际上尾数加法应该用补码：

$0.11010$（补码）$= +0.8125$
$0.10110$（补码）$= +0.6875$

$0.11010 + 0.10110 = 1.10000$（补码，溢出了正数范围）

$1.10000$（补码）$= -(0.01111 + 0.00001) = -(1 - 0.10000) = ...$

更直接：$1.10000$（补码）$= -0.10000$？不对。

补码 $1.10000$ 的求值：取反 $0.01111$，加1 $0.10000 = 0.5$，所以 $1.10000 = -0.5$？

不对，$0.8125 + 0.6875 = 1.5$，在5位补码小数中 $[-1, 1-2^{-4}]$，$1.5 > 0.9375$ 溢出了。

这说明 $1.10000$ 实际上是 $1.5$ 在溢出时的补码表现。右移1位（等价于除以2再加阶码加1）：

$1.10000$ 右移1位（算术右移）$= 1.11000$

$1.11000$（补码）$= -0.01000 = -0.0625$？不对...

实际上在浮点运算中，溢出后右移是**逻辑右移**还是保留正确性：

正确的做法是：$0.11010 + 0.10110 = 1.10000$（有进位到符号位）

由于加法溢出，需要右规：右移 1 位，进位的1成为最高数值位：

$1.10000 \to$ 右移：$0.11000$（进位1移到最高位），阶码 +1

结果：$0.11000 \times 2^3 = 0.75 \times 8 = 6.0$ ✓

> 注：这里的右移处理方式需要根据具体硬件和教材定义来确定。

## 例题 3：需要左规的情况

$A = 0.00110 \times 2^{+3}$，$B = (-0.01010) \times 2^{+3}$（即 $B$ 的尾数为 $1.10110$ 补码）

### 第一步：对阶

$E_A = E_B = +3$，不需要对阶。

### 第二步：尾数减法

$A - B$（假设做减法）：

$0.00110 - 0.01010 = 0.00110 + 1.10110 = 1.11100$（补码）

$1.11100$ 的真值 = $-0.00100 = -1/32$

或者 $A + B$：$0.00110 + 1.10110 = 1.11100$... 结果溢出需右规的情况更典型。

让我用更合适的例子：

$A = 0.10100 \times 2^{+2}$，$B = (-0.10010) \times 2^{+2}$

$B$ 的尾数补码 = $1.01110$

### 第二步：尾数加法

$0.10100 + 1.01110 = 0.00010$（溢出进位丢弃）

结果 $0.00010$，最高数值位为 0，未规格化。

### 第三步：左规

左移 3 位：$0.00010 \to 0.10000$

阶码减 3：$+2 - 3 = -1$

结果：$0.10000 \times 2^{-1} = 0.5 \times 0.5 = 0.25$

验证：$A = 0.101 \times 4 = 2.5$，$B = -0.1001 \times 4 = -2.25$，$A + B = 0.25$ ✓

## 例题 4：浮点减法

$A = 0.11010 \times 2^{+2}$，$B = 0.10010 \times 2^{+1}$

求 $A - B$。

### 第一步：对阶

$E_A = +2$，$E_B = +1$，$B$ 右移 1 位。

$B$ 尾数：$0.10010 \to 0.01001$，阶码改为 $+2$。

### 第二步：尾数减法

$0.11010 - 0.01001 = 0.11010 + 1.10111$（$-0.01001$ 的补码）

$= 0.10001$（溢出进位丢弃）

### 第三步：规格化

$0.10001$：符号位 0，最高数值位 1，已规格化。

### 结果

$A - B = 0.10001 \times 2^{+2} = 0.53125 \times 4 = 2.125$

验证：$A = 0.8125 \times 4 = 3.25$，$B = 0.5625 \times 2 = 1.125$，$A - B = 2.125$ ✓

## 知识关联

- 对阶是浮点加法的关键步骤，精度损失主要来源
- 左规可能多次，右规最多一次
- 舍入可能引起再次溢出
