# 32_补码特殊值

## 核心概念

### $-2^{n-1}$ 的补码表示

在 $n$ 位补码中，最小负数 $-2^{n-1}$ 的表示为：

$$[-2^{n-1}]_{\text{补}} = \underbrace{100\cdots0}_{n\text{位}}$$

例如8位：$[-128]_{\text{补}} = 10000000$

### 为什么 $-2^{n-1}$ 特殊？

1. **没有对应的原码**：原码只能表示到 $-(2^{n-1}-1)$
2. **没有对应的反码**：反码与原码范围相同
3. **无法通过"取反+1"得到它的相反数**：$[-(-128)]_{\text{补}} = [+128]_{\text{补}}$，但+128超出8位补码正数范围
4. **是补码独有的值**

---

## 原理分析

### $-128$ 补码的推导

**方法1：模运算定义**

$$[-128]_{\text{补}} = 2^8 + (-128) = 256 - 128 = 128 = 10000000_2$$

**方法2：利用 $-127$ 的补码**

$[-127]_{\text{补}} = 10000001$

$-128 = -127 - 1$

$$[-128]_{\text{补}} = [-127]_{\text{补}} + [-1]_{\text{补}} = 10000001 + 11111111 = 1\ 10000000$$

截断8位：$10000000$ ✓

### $-128$ 补码的特殊性质

**取反+1操作**：

对 $10000000$ 取反+1：
$$\overline{10000000} + 1 = 01111111 + 1 = 10000000$$

结果还是它自己！

这说明 $-128$ 的"相反数"（$+128$）在8位补码系统中无法表示，取反+1操作"溢出"回到了自己。

**验证**：

$[-(-128)]_{\text{补}} = [+128]_{\text{补}}$，但8位补码正数最大为 $+127$，所以 $+128$ 超出范围。

在模 $2^8 = 256$ 系统中：$+128 \equiv -128 \pmod{256}$

所以 $[-(-128)]_{\text{补}}$ 的"模等价值"还是 $[-128]_{\text{补}}$，即 $10000000$。

### 其他特殊值（8位补码）

| 编码 | 真值 | 特殊性质 |
|------|------|---------|
| `00000000` | $+0$ | 唯一的零 |
| `01111111` | $+127$ | 最大正数 |
| `10000000` | $-128$ | 最小负数，无对应原码 |
| `11111111` | $-1$ | 最大负数（绝对值最小的负数） |

### 边界值转换

**$+127$ 取反+1**：

$$\overline{01111111} + 1 = 10000000 + 1 = 10000001 = -127 \quad \text{（不是-128!）}$$

等等，$[+127]_{\text{补}} = 01111111$

取反+1：$\overline{01111111} + 1 = 10000000 + 1 = 10000001$

$10000001$ 是 $-127$ 的补码。但 $[-127]_{\text{补}}$ 应该是 $-[+127]$ = $-127$，不等于 $-128$。

这说明 $[+127]_{\text{补}}$ 取反+1 = $[-127]_{\text{补}}$ = $-127$，不是 $-128$。

## 直观理解

### 记忆要点

1. $-2^{n-1}$ 是补码的"特殊居民"，其他编码系统没有它的位置
2. `10000000`（8位）既不是 $-0$，也不是某个原码的补码，它是补码独有的
3. $-2^{n-1}$ 的相反数 $+2^{n-1}$ 超出了补码正数范围
4. 在补码系统中，$-2^{n-1}$ 是"不对称"的——负数比正数多一个

### 边界值速查（8位补码）

| 编码 | 读作 |
|------|------|
| `00000000` | 0 |
| `00000001` | +1 |
| `01111111` | +127（正数最大） |
| `10000000` | -128（负数最小，特殊值） |
| `10000001` | -127 |
| `11111111` | -1 |
| `11111110` | -2 |

## 知识关联

### 408考试要点
- $-2^{n-1}$ 的补码表示方法
- $-2^{n-1}$ 没有对应的原码和反码
- $-2^{n-1}$ 取反+1后得到自身
- 补码范围不对称的原因

### 易错陷阱
1. **不要试图求 $-2^{n-1}$ 的原码**——它不存在
2. **$-2^{n-1}$ 取反+1后不是0**——而是它自己
3. **`10000000` 是 $-128$ 不是 $-0$**——与原码不同
4. **补码正数最大值比负数最大值（绝对值）小1**——因为不对称
