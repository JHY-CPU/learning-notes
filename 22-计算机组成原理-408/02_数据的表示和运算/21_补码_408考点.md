# 22_补码_408考点

## 核心概念

### 408高频考点清单
1. 补码的表示范围（比原码多一个数）
2. 给定真值求补码，给定补码求真值
3. $[X]_{\text{补}}$ 求 $[-X]_{\text{补}}$
4. 补码加减法及溢出判断
5. $-2^{n-1}$ 的特殊补码表示

---

## 典型考题

### 考题1：补码求法

**题目**：写出 $-75$ 的8位补码。

**解法1（从真值直接求）**：
1. $75_{10} = 01001011_2$（8位）
2. $-75$ 的补码 $= 256 + (-75) = 181 = 10110101_2$

**解法2（从原码取反+1）**：
1. $-75$ 的原码：`11001011`
2. 数值位取反：`10110100`
3. 加1：`10110101`

**答案**：$[-75]_{\text{补}} = 10110101$

### 考题2：补码求真值

**题目**：$[X]_{\text{补}} = 10110110$，求 $X$。

**解**：
1. 最高位为1，是负数
2. 数值位取反：`01001001`
3. 加1：`01001010` $= 74_{10}$
4. $X = -74$

**答案**：$X = -74$

### 考题3：$[X]_{\text{补}}$ 求 $[-X]_{\text{补}}$

**题目**：$[X]_{\text{补}} = 01011010$，求 $[-X]_{\text{补}}$。

**解**：全部位取反再加1：

$$\overline{01011010} + 1 = 10100101 + 1 = 10100110$$

**答案**：$[-X]_{\text{补}} = 10100110$

**验证**：$[X]_{\text{补}} = 01011010 = +90$，$[-X]_{\text{补}} = 10100110 = -90$ ✓

### 考题4：补码加减法

**题目**：用8位补码计算 $53 + (-37)$。

**解**：
$$[53]_{\text{补}} = 00110101$$
$$[-37]_{\text{补}} = 11011011$$

$$00110101 + 11011011 = 1\ 00010000$$

舍弃进位，结果 $= 00010000 = +16$。

**验证**：$53 + (-37) = 16$ ✓

### 考题5：$-2^{n-1}$ 的特殊性

**题目**：8位补码中，$-128$ 的表示是什么？

**解**：$[-128]_{\text{补}} = 256 + (-128) = 128 = 10000000$

**注意**：$-128$ 没有对应的原码！原码最大只能表示 $-127$。

$[-128]_{\text{补}} = 10000000$

对它取反+1：$\overline{10000000} + 1 = 01111111 + 1 = 10000000$

还是它自己！说明 $-128$ 的补码取反+1后还是 $-128$，这与 $[-(-128)]_{\text{补}} = [+128]_{\text{补}}$ 矛盾（因为 $+128$ 超出8位补码范围）。

所以 $-128$ 的负数 $+128$ 超出了8位补码的正数范围，这是边界特殊情况。

## 常考陷阱

### 陷阱1：补码表示范围
- 8位补码范围：$-128 \sim +127$（不是 $-127 \sim +127$）
- 补码多表示一个 $-2^{n-1}$

### 陷阱2：$[X]_{\text{补}}$ 求 $[-X]_{\text{补}}$
- **正确**：全部取反+1（包括符号位）
- **错误**：只取反数值位（这是原码的求负方法）

### 陷阱3：补码加法的进位
- 超出字长的进位**必须舍弃**
- 如果不舍弃，结果会是错误的

### 陷阱4：溢出不等于进位
- 补码加法的进位不等于溢出
- 两个正数相加得负数 → 上溢
- 两个负数相加得正数 → 下溢
- 详细溢出判断见第38、39节

### 陷阱5：补码和原码的转换
- **正数**：补码 = 原码
- **负数**：补码 ≠ 原码（需要取反+1）
- 不要混淆两种编码

## 知识关联

### 与浮点数的关系
- IEEE 754中阶码用移码表示，尾数可以用原码或补码
- 补码的理解有助于理解移码（移码 = 补码 + $2^{n-1}$，即仅符号位不同）

### 408答题技巧
1. 做补码运算时，先写出两个操作数的补码
2. 按二进制加法相加
3. 舍弃最高位进位
4. 检查溢出（如果题目要求）
5. 最后将结果转回真值
