# 20_有符号整数_补码

## 核心概念

### 定义
**补码**（Two's Complement）是计算机内部表示有符号整数的标准编码方式。它将减法统一为加法，简化了运算器设计。

### 编码规则

设 $n$ 位机器字长：

$$[X]_{\text{补}} = \begin{cases} X & \text{当 } 0 \leq X < 2^{n-1} \quad \text{（正数：原样表示）} \\ 2^n + X & \text{当 } -2^{n-1} \leq X < 0 \quad \text{（负数：模+真值）} \end{cases}$$

### 实用求法

**正数的补码** = 原码（符号位0 + 数值的绝对值）

**负数的补码** = 原码的**数值位取反再加1**（符号位不变）

或者等价地：**负数的补码 = 反码 + 1**

### 表示范围

$n$ 位补码的表示范围：

$$-2^{n-1} \leq X \leq +(2^{n-1} - 1)$$

| 位数 | 范围 | 可表示个数 |
|------|------|-----------|
| 8位 | $-128 \sim +127$ | 256 |
| 16位 | $-32768 \sim +32767$ | 65536 |
| $n$位 | $-2^{n-1} \sim +(2^{n-1}-1)$ | $2^n$ |

**关键特点**：补码比原码**多表示一个数**（$-2^{n-1}$），因为只有一个0。

## 原理分析

### 补码示例（8位）

| 真值 | 原码 | 补码 | 求法 |
|------|------|------|------|
| $+0$ | `00000000` | `00000000` | 正数不变 |
| $-0$ | `10000000` | `00000000` | 与+0相同 |
| $+1$ | `00000001` | `00000001` | 正数不变 |
| $-1$ | `10000001` | `11111111` | 原码取反+1 |
| $+127$ | `01111111` | `01111111` | 正数不变 |
| $-127$ | `11111111` | `10000001` | 原码取反+1 |
| $-128$ | 不存在 | `10000000` | 补码特有 |
| $+42$ | `00101010` | `00101010` | 正数不变 |
| $-42$ | `10101010` | `11010110` | 原码取反+1 |

**求 $-42$ 的补码过程**：
1. $+42$ 的二进制：`00101010`
2. 数值位取反：`11010101`
3. 加1：`11010110`

### 补码的核心优势

**减法 = 加法**：$A - B = A + (-B)_{\text{补}}$

**示例**：$5 - 3$

$$[5]_{\text{补}} = 00000101$$
$$[-3]_{\text{补}} = 11111101$$

$$00000101 + 11111101 = 1\ 00000010$$

舍弃最高位进位，结果 $= 00000010 = +2$ ✓

### 补码求 $[-X]_{\text{补}}$

对 $[X]_{\text{补}}$ **所有位取反再加1**，即得 $[-X]_{\text{补}}$。

**示例**：$[+42]_{\text{补}} = 00101010$

$$[-42]_{\text{补}} = \overline{00101010} + 1 = 11010101 + 1 = 11010110$$

## 直观理解

### 时钟类比
补码就像时钟（模12）：
- 现在是3点，要往回拨5个小时
- 不需要"3 - 5 = -2"，而是 "3 + 7 = 10"（因为 $-5 \equiv 7 \pmod{12}$）
- 补码就是把负数转换成它的"模等价正数"

### 为什么补码只有一个0？
- $+0$ 的补码：`00000000`
- $-0$ 的补码：`00000000`（因为 $10000000$ 取反+1 = `01111111`+1 = `10000000`... 不对）

实际上 $-0$ 在补码中：
- 原码 $-0$ = `10000000`
- 取反 = `11111111`
- 加1 = `100000000`，截断8位 = `00000000`
- 与 $+0$ 相同！

所以补码中 `10000000` 不再是 $-0$，而是表示 $-128$。

## 知识关联

### 跨章节联系
- **CPU运算器**：ALU使用补码进行所有有符号整数运算
- **浮点数**：阶码用移码，尾数用原码或补码
- **指令系统**：有符号数比较、跳转指令依赖补码表示
- **C语言**：`int`、`short`、`long`等有符号类型使用补码

### 408考试要点
- 补码的定义和求法（原码取反+1）
- 补码表示范围（比原码多一个 $-2^{n-1}$）
- 补码加减法统一为加法
- $[X]_{\text{补}}$ 求 $[-X]_{\text{补}}$（全部取反+1）
- 补码溢出判断

### 易错陷阱
1. **正数补码 = 原码**，不要对正数做取反操作
2. **负数补码的求法**：先写原码，再取反+1（符号位保持1）
3. **$-128$ 的补码**：`10000000`，没有对应的原码和反码
4. **补码加法的进位要舍弃**：超出字长的进位直接丢弃
