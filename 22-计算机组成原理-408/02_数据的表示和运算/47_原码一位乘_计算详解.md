# 47_原码一位乘_计算详解

## 核心概念

本文件通过详细的例题，演示原码一位乘法的完整手算过程。重点追踪每一步 ACC 和 MQ 的状态变化。

### 运算规则回顾

1. 初始化：ACC = 0...0，MQ = |乘数|，X = |被乘数|
2. 对于第 i 轮（i = 1, 2, ..., n）：
   - 检查 MQ 最低位
   - 若为 1：ACC ← ACC + X
   - 若为 0：不操作
   - 右移：(ACC, MQ) 联合右移一位
3. 共进行 n 轮，得到 2n 位乘积

## 例题 1：0.1101 × 0.1011（小数，4位数值）

### 题目信息

- 被乘数 $x = 0.1101$，$[x]_{原} = 0.1101$
- 乘数 $y = 0.1011$，$[y]_{原} = 0.1011$
- 符号位：$0 \oplus 0 = 0$（正数）
- 数值部分：$|x| = 1101$，$|y| = 1011$

### 初始状态

| 寄存器 | 值 | 说明 |
|--------|-----|------|
| ACC | 0000 | 部分积初始为0（4位+1位扩展位=5位） |
| MQ | 1011 | 存放乘数 |
| X | 1101 | 存放被乘数 |

### 逐步运算

**第1轮**（看 $y_4 = 1$）：
- MQ 最低位 = 1，执行 `ACC ← ACC + X`
- `ACC = 0000 + 1101 = 01101`
- 右移：`(ACC, MQ) >> 1`
- ACC = 00110，MQ 最低位移出，MQ 最高位接收 ACC 最低位 0
- MQ = **0**101 → 实际为 0101

| 步骤 | 操作 | ACC | MQ |
|------|------|-----|-----|
| 初始 | — | 0000 | 1011 |
| 判断y₄=1 | ACC←ACC+X | 01101 | 1011 |
| 右移 | 联合右移 | 00110 | 1101 |

> 注：实际考试中，我们通常用 n+1 位 ACC，以下使用标准5位 ACC（1位扩展+4位数值）。

### 标准格式表格（5位 ACC + 4位 MQ）

初始：ACC = 00000, MQ = 1011, X = 01101（5位，含扩展位0）

| 轮次 | 操作 | ACC | MQ | 说明 |
|------|------|-----|-----|------|
| 初始 | — | 00000 | 1011 | |
| 1 | y₄=1, 加X | 01101 | 1011 | ACC+X |
| 1 | 右移 | 00110 | 1101 | ACC最低位→MQ最高位 |
| 2 | y₄=1, 加X | 10011 | 1101 | ACC+X |
| 2 | 右移 | 01001 | 1110 | |
| 3 | y₄=0, 不加 | 01001 | 1110 | |
| 3 | 右移 | 00100 | 1111 | |
| 4 | y₄=1, 加X | 10001 | 1111 | ACC+X |
| 4 | 右移 | 01000 | 1111 | |

### 最终结果

- ACC = 01000，MQ = 1111
- 乘积数值部分 = **010001111**（即 8 位：0.10001111）
- 符号位 = 0
- $[x \times y]_{原} = 0.10001111$

### 验证

$$0.1101_2 = \frac{13}{16}, \quad 0.1011_2 = \frac{11}{16}$$
$$\frac{13}{16} \times \frac{11}{16} = \frac{143}{256} = 0.10001111_2 \checkmark$$

## 例题 2：1101 × 1011（整数，4位）

### 题目信息

- 被乘数 $x = 1101$（13）
- 乘数 $y = 1011$（11）
- 均为正数，符号位均为 0
- X = 01101（5位），ACC = 00000，MQ = 1011

### 运算表格

| 轮次 | 操作 | ACC | MQ | 说明 |
|------|------|-----|-----|------|
| 初始 | — | 00000 | 1011 | |
| 1 | y₄=1, +X | 01101 | 1011 | |
| 1 | 右移 | 00110 | 1101 | |
| 2 | y₄=1, +X | 10011 | 1101 | |
| 2 | 右移 | 01001 | 1110 | |
| 3 | y₄=0, 不加 | 01001 | 1110 | |
| 3 | 右移 | 00100 | 1111 | |
| 4 | y₄=1, +X | 10001 | 1111 | |
| 4 | 右移 | 01000 | 1111 | |

### 最终结果

- ACC = 01000，MQ = 1111
- 连接起来：010001111 = **143**（$13 \times 11 = 143$）✓

## 例题 3：带符号位的原码一位乘 $(-0.1101) \times (+0.1011)$

### 题目信息

- 被乘数 $x = -0.1101$，$[x]_{原} = 1.1101$，$|x| = 0.1101$
- 乘数 $y = +0.1011$，$[y]_{原} = 0.1011$，$|y| = 0.1011$
- 符号位：$1 \oplus 0 = 1$（负数）
- 数值部分运算与例题1完全相同

### 运算过程

数值部分运算过程与例题1完全一致：

| 轮次 | 操作 | ACC | MQ |
|------|------|-----|-----|
| 初始 | — | 00000 | 1011 |
| 1 | y₄=1, +X, 右移 | 00110 | 1101 |
| 2 | y₄=1, +X, 右移 | 01001 | 1110 |
| 3 | y₄=0, 右移 | 00100 | 1111 |
| 4 | y₄=1, +X, 右移 | 01000 | 1111 |

### 最终结果

- 数值部分 = 0.10001111
- 符号位 = 1
- $[x \times y]_{原} = 1.10001111$
- 真值 = $-0.10001111_2 = -\frac{143}{256}$

## 例题 4（补充）：$0.1010 \times 0.0110$

### 运算表格

X = 01010, ACC = 00000, MQ = 0110

| 轮次 | 操作 | ACC | MQ | 说明 |
|------|------|-----|-----|------|
| 初始 | — | 00000 | 0110 | |
| 1 | y₄=0, 右移 | 00000 | 0011 | y₄=0不加 |
| 2 | y₄=1, +X | 01010 | 0011 | |
| 2 | 右移 | 00101 | 0001 | |
| 3 | y₄=1, +X | 01111 | 0001 | |
| 3 | 右移 | 00111 | 1000 | |
| 4 | y₄=0, 右移 | 00011 | 1100 | y₄=0不加 |

### 结果

- ACC = 00011, MQ = 1100
- 乘积 = 000111100
- $0.1010_2 \times 0.0110_2 = \frac{10}{16} \times \frac{6}{16} = \frac{60}{256} = 0.00111100_2$ ✓

## 直观理解

### 关键观察

1. **每次右移后，MQ 最低位变为新的待判断位** — 这就是为什么 MQ 要右移
2. **ACC 扩展位的存在**：加法可能产生进位，需要额外一位来存放
3. **n 位乘 n 位 = 2n 位结果**：4位×4位得到8位乘积
4. **符号位永远不参与加法运算**：整个过程中处理的都是数值部分

### 手算速记法

把每次的 ACC 看作"部分积高位"，MQ 看作"部分积低位 + 未处理的乘数"，右移操作使两者无缝衔接。

## 知识关联

- 原码一位乘是理解补码Booth算法的基础
- ACC+MQ 联合右移的硬件结构是运算器设计的核心
