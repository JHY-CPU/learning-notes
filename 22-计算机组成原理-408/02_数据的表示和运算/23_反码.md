# 23_反码

## 核心概念

### 定义
**反码**（One's Complement）是一种过渡性质的有符号数编码方式。

### 编码规则

$$[X]_{\text{反}} = \begin{cases} X & \text{当 } 0 \leq X < 2^{n-1} \quad \text{（正数：不变）} \\ (2^n - 1) + X & \text{当 } -2^{n-1} < X \leq 0 \quad \text{（负数：取反）} \end{cases}$$

**实用求法**：
- **正数**：反码 = 原码 = 补码
- **负数**：符号位保持1，数值位每位取反（0变1，1变0）

### 表示范围

$n$ 位反码的表示范围：$-(2^{n-1} - 1) \sim +(2^{n-1} - 1)$

与原码相同，比补码少一个数（$-2^{n-1}$无法表示）。

### 特殊值：+0 和 -0

与原码一样，反码也有两个零：
- $+0$：$00000000$（8位）
- $-0$：$11111111$（8位，反码中 $-0$ 的反码是所有位取反 = $00000000$ = $+0$）

## 原理分析

### 反码示例（8位）

| 真值 | 原码 | 反码 | 补码 |
|------|------|------|------|
| $+0$ | `00000000` | `00000000` | `00000000` |
| $-0$ | `10000000` | `11111111` | `00000000` |
| $+1$ | `00000001` | `00000001` | `00000001` |
| $-1$ | `10000001` | `11111110` | `11111111` |
| $+127$ | `01111111` | `01111111` | `01111111` |
| $-127$ | `11111111` | `10000000` | `10000001` |

**规律**：负数的反码 = 原码数值位取反；补码 = 反码 + 1。

### 反码加法

反码加法需要**循环进位**（end-around carry）：最高位的进位要加到最低位。

**示例**：$5 + (-3)$ 用8位反码

$$[5]_{\text{反}} = 00000101$$
$$[-3]_{\text{反}} = 11111100$$

$$00000101 + 11111100 = 1\ 00000001$$

最高位进位1，回加到最低位：$00000001 + 1 = 00000010 = +2$ ✓

**问题**：如果回加后又产生进位，需要继续回加（理论上可能多次循环），硬件实现复杂。

### 反码的历史地位

反码是补码的前身：
1. 早期计算机使用反码（如CDC 6600）
2. 后来发现补码更简洁（不需要循环进位）
3. 现代计算机全部使用补码

反码的主要价值在于**理解补码的推导过程**：补码 = 反码 + 1。

---

## 直观理解

### 记忆要点
- 反码 = 原码数值位取反
- 反码是原码到补码的中间步骤
- 反码有两个0，补码只有一个0
- 反码加法需要循环进位，补码不需要

### 反码的作用
- 历史上用于简化原码加法
- 现在主要用于理解补码的推导（补码 = 反码 + 1）
- 网络校验和计算中仍使用反码加法（one's complement addition）

---

## 知识关联

### 三种编码的关系

$$\text{正数：原码} = \text{反码} = \text{补码}$$

$$\text{负数：反码} = \text{原码取反}; \quad \text{补码} = \text{反码} + 1$$

$$\text{反码} \to \text{补码}：+1$$
$$\text{补码} \to \text{反码}：-1$$

### 408考试要点
- 反码的定义和求法
- 反码有两个零
- 反码的循环进位特性
- 反码是理解补码的桥梁
- 网络校验和使用反码加法

### 易错陷阱
1. 反码的 $-0$ 是 `11111111`，不是 `10000000`（那是原码的 $-0$）
2. 反码加法的循环进位不要忘记
3. 反码的表示范围与原码相同，不是与补码相同
