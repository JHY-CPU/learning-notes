# 43_移位运算_算术移位

## 核心概念

### 算术移位的定义

**算术移位**（Arithmetic Shift）是针对**有符号数**的移位操作，移位时需要保持**符号位不变**。

### 算术左移（ASL / SAL）

- 规则：**符号位不变**，所有数值位左移1位，最低位补0
- 效果：相当于 **$\times 2$**（在不溢出的前提下）
- 风险：可能导致溢出（符号位被改变）

### 算术右移（ASR / SAR）

- 规则：**符号位不变**，所有数值位右移1位，最高数值位补**符号位**的值
- 效果：相当于 **$\div 2$**（向下取整）
- 特点：负数右移后仍是负数（补符号位1）

## 原理分析

### 原码的算术移位

原码的符号位和数值位分离，移位只影响数值位。

**左移**：数值位左移，低位补0。符号位不变。

**右移**：数值位右移，高位补0。符号位不变。

**示例**：$[+42]_{\text{原}} = 00101010$

左移1位：$0\ 1010100 = +84$（$\times 2$）
右移1位：$0\ 0010101 = +21$（$\div 2$）

### 补码的算术移位

**补码左移**：所有位左移，低位补0（包括符号位... 但符号位被覆盖）

等等，正确的描述是：
- **补码左移**：所有位左移1位，低位补0。符号位由原来的次高位决定。
- **补码右移**：所有位右移1位，高位补**符号位**（因为符号位要保持不变）。

**示例1**：$[+42]_{\text{补}} = 00101010$

左移1位：$01010100 = +84$

右移1位：$00010101 = +21$

**示例2**：$[-42]_{\text{补}} = 11010110$

左移1位：$10101100$

验证：$10101100$（补码）→ 取反+1 = `01010011` + 1 = `01010100` = 84，所以 $= -84$。

$-42 \times 2 = -84$ ✓

右移1位：$11101011$

验证：$11101011$（补码）→ 取反+1 = `00010100` + 1 = `00010101` = 21，所以 $= -21$。

$-42 \div 2 = -21$ ✓

### 各编码的移位规则对比

| 编码 | 左移（低位补） | 右移（高位补） |
|------|-------------|-------------|
| 原码 | 补0 | 补0 |
| 补码 | 补0 | 补符号位 |
| 反码 | 补0（正数）/补1（负数） | 补符号位 |

### 溢出判断

**左移溢出**：
- 正数左移后符号位变为1 → 上溢
- 负数左移后符号位变为0 → 下溢
- 即：左移后符号位与原符号位不同 → 溢出

**右移不会溢出**（结果绝对值只会变小）

## 直观理解

### 生活类比

算术左移就像把数字放大2倍：
- 所有位权值增大（$2^i \to 2^{i+1}$）
- 如果最高有效位"撞"到符号位，就溢出了

算术右移就像把数字缩小2倍（取整）：
- 所有位权值缩小（$2^i \to 2^{i-1}$）
- 最低位被丢弃（相当于向下取整）

### 记忆技巧

1. **左移 = $\times 2$，右移 = $\div 2$**（近似关系）
2. **右移补符号位**：保证符号不变，正数补0，负数补1
3. **左移低位补0**：和逻辑左移一样
4. **左移可能溢出**：检查符号位是否变化

## 知识关联

### 与乘除法的关系
- 左移 $k$ 位 ≈ $\times 2^k$
- 右移 $k$ 位 ≈ $\div 2^k$（取整方向取决于实现）
- 编译器常将乘以2的幂优化为移位操作

### 408考试要点
- 算术移位的规则（特别是右移补符号位）
- 移位后的结果计算
- 左移溢出的判断
- 不同编码的移位规则差异

### 易错陷阱
1. **补码右移补符号位，不是补0**：补0会改变符号
2. **原码右移补0，不补符号位**：原码的数值位和符号位分开
3. **左移后的符号位检查**：如果变了就是溢出
4. **右移对负数是向下取整**：$-5 \div 2 = -3$（不是-2）
