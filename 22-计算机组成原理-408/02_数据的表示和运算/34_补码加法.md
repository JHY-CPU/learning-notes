# 35_补码加法

## 核心概念

### 补码加法公式

$$[X + Y]_{\text{补}} = [X]_{\text{补}} + [Y]_{\text{补}} \pmod{2^n}$$

**规则**：
1. 将两个操作数的补码按位相加
2. **符号位参与运算**
3. 最高位产生的进位**舍弃**（模 $2^n$ 的自然结果）
4. 结果仍为补码形式

### 补码加法的三种情况

| 情况 | 操作数 | 结果 | 是否溢出 |
|------|--------|------|---------|
| 正+正 | 两个正数 | 正数（可能溢出） | 结果为负则溢出 |
| 负+负 | 两个负数 | 负数（可能溢出） | 结果为正则溢出 |
| 正+负 | 一正一负 | 不会溢出 | — |

---

## 原理分析

### 例题1：正数 + 正数（无溢出）

计算 $42 + 25$（8位补码）

$$[42]_{\text{补}} = 00101010$$
$$[25]_{\text{补}} = 00011001$$

```
  00101010
+ 00011001
----------
  01000011  = +67
```

最高位无进位，符号位为0（正数），结果正确。$42 + 25 = 67$ ✓

### 例题2：正数 + 正数（上溢）

计算 $85 + 60$（8位补码）

$$[85]_{\text{补}} = 01010101$$
$$[60]_{\text{补}} = 00111100$$

```
  01010101
+ 00111100
----------
  10010001  = -111（补码解释）
```

符号位为1，结果为负数！但两个正数相加不可能为负——**上溢**。

实际结果应为 $85 + 60 = 145$，但8位补码最大正数为127，超出范围。

### 例题3：负数 + 负数（无溢出）

计算 $(-42) + (-25)$（8位补码）

$$[-42]_{\text{补}} = 11010110$$
$$[-25]_{\text{补}} = 11100111$$

```
  11010110
+ 11100111
----------
 110111101  （9位）
```

舍弃最高位进位：$10111101$

验证：$10111101$（补码）→ 取反+1 = `01000010` + 1 = `01000011` = 67，所以结果 = $-67$。

$(-42) + (-25) = -67$ ✓

### 例题4：负数 + 负数（下溢）

计算 $(-85) + (-60)$（8位补码）

$$[-85]_{\text{补}} = 10101011$$
$$[-60]_{\text{补}} = 11000100$$

```
  10101011
+ 11000100
----------
 101101111  （9位）
```

舍弃最高位进位：$01101111$，符号位为0（正数）！

两个负数相加得正数——**下溢**。

实际结果应为 $(-85) + (-60) = -145$，但8位补码最小负数为 $-128$，超出范围。

### 例题5：正数 + 负数

计算 $42 + (-25)$（8位补码）

$$[42]_{\text{补}} = 00101010$$
$$[-25]_{\text{补}} = 11100111$$

```
  00101010
+ 11100111
----------
 1000010001  （10位...不对，重算）
```

```
  00101010
+ 11100111
----------
 100010001  （9位）
```

舍弃进位：$00010001 = +17$

$42 + (-25) = 17$ ✓

符号位为0，正数，无溢出。

## 直观理解

### 记忆要点
1. **符号位参与运算**：不要单独处理符号位
2. **进位舍弃**：最高位的进位直接丢弃
3. **溢出条件**：同号相加结果异号则溢出
4. **异号相加不会溢出**：结果绝对值一定小于任一操作数

### 硬件实现
补码加法只需一个普通的二进制加法器，不需要判断符号——这就是补码的优势。
