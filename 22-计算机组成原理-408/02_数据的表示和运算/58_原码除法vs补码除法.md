# 59_原码除法vs补码除法

## 核心概念

### 原码除法与补码除法的主要区别

| 特征 | 原码除法 | 补码除法 |
|------|---------|---------|
| 符号位 | 单独处理（异或） | 参与运算 |
| 被除数/除数 | 用绝对值运算 | 用补码运算 |
| 商的形式 | 原码 | 补码 |
| 余数形式 | 原码 | 补码 |
| 初始比较 | 用减法试探 | 用加减法试探 |
| 最后一步 | 余数为负需恢复 | 余数为负需恢复 |

### 补码除法（加减交替法）的算法

补码除法同样采用加减交替的思想，但符号位参与运算：

**初始化**：
- ACC = $[x]_补$（被除数的补码，含符号位）
- X = $[y]_补$（除数的补码，含符号位）
- MQ = 0

**迭代过程**（共 n 轮）：

关键区别在于第一轮和后续轮的操作：

**第一轮**（特殊处理）：
- 若被除数与除数**同号**：`ACC ← ACC - [y]_补`
- 若被除数与除数**异号**：`ACC ← ACC + [y]_补`

**后续各轮**：
- 若 ACC 与除数**同号**：`ACC ← ACC - [y]_补`，商 1（若够减）或 0
- 若 ACC 与除数**异号**：`ACC ← ACC + [y]_补`，商 0 或 1

**商的确定规则**：
- 若 ACC 与除数**同号**：商 **1**
- 若 ACC 与除数**异号**：商 **0**

**最后一轮**：
- 商采用**末位恒置1法**：无论最后一轮结果如何，商的末位恒置为 1
- 引入了误差（最大误差 $2^{-n}$），但简化了硬件

### 补码除法的判断规则详解

| ACC与除数 | 操作 | 商 |
|-----------|------|-----|
| 同号 | 减去除数 | 1 |
| 异号 | 加上除数 | 0 |

这与原码除法不同：
- 原码看 ACC 正负（绝对值是否够减）
- 补码看 ACC 与除数是否同号（相对大小判断）

### 末位恒置1法

补码除法的最后一位商，不经过完整的判断，而是**强制置为1**。

原因：
- 简化硬件控制逻辑
- 引入的误差很小（$2^{-n}$）
- 误差在浮点运算中通常可以接受

## 原理分析

### 补码除法的数学基础

设 $[x]_补$ 和 $[y]_补$，需要找 $[q]_补$ 使得：

$$[x]_补 = [q]_补 \times [y]_补 + [r]_补 \times 2^{-n}$$

其中 $q$ 是商，$r$ 是余数。

关键约束：
- 余数与被除数**同号**
- 商的符号由补码运算自然得出

### 补码除法的流程

```
初始化：ACC ← [x]_补, X ← [y]_补, MQ ← 0

第1轮：
  若x与y同号：ACC ← ACC - X
  若x与y异号：ACC ← ACC + X

第2轮至第n-1轮：
  若ACC与y同号：ACC ← ACC - X, 商1
  若ACC与y异号：ACC ← ACC + X, 商0
  左移

第n轮（末轮）：
  执行与上述相同的操作
  商的末位恒置1

最后：
  若余数与被除数异号，需要修正余数
```

### 补码除法的特殊情况

1. **除数为0**：产生溢出中断
2. **被除数为最小负数除以-1**：结果溢出（$-2^{n-1} / (-1) = 2^{n-1}$ 超出补码范围）
3. **商的末位恒置1**：引入误差但简化硬件

## 直观理解

### 记忆要点

- **补码除法：符号位参与运算，商也是补码**
- **第一轮特殊**：同号做减法，异号做加法
- **判断看同号异号**：不是看正负
- **末位恒置1**：简化硬件

### 与原码除法的直观对比

原码除法：先把符号去掉，对绝对值做除法，最后拼符号
补码除法：直接对补码做除法，符号自然参与每一步

## 知识关联

- 补码加减法（第1章）是补码除法的基础
- 补码乘法（Booth算法，第50节）与补码除法在思想上一致
- 浮点运算中，尾数的除法可能用到补码除法
- 末位恒置1法与浮点运算的舍入方法（第82节）有关联

## 408考试要点

### 常考选择题

**题目**：补码除法中，商的确定依据是什么？

A. ACC 的正负
B. ACC 与被除数是否同号
C. ACC 与除数是否同号
D. ACC 的绝对值大小

**答案**：C

### 易错点

1. **补码除法判断的是"同号异号"，不是"正负"**
2. **第一轮和后续轮的操作规则不同**
3. **末位恒置1法是补码除法的特色**
4. **余数的修正**：最后需要检查余数与被除数是否同号
