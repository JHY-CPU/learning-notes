# 07_BCD码_408考点

## 核心概念

### 8421码加法修正规则（高频考点）

8421码的4位二进制只能表示0~9，但二进制加法会产生0~15的结果。当加法结果**大于9**或**产生进位**时，需要**+6修正**（即加上 $0110_2$）。

### 修正条件（满足任一即需修正）
1. 两个8421码数字相加后，结果**大于9**（即结果在 `1010` ~ `1111` 之间）
2. 两个8421码数字相加后，产生了**向高位的进位**

### 修正方法
结果 **加上 $0110_2$（即+6）**。

**为什么加6？** 因为8421码只用了0000~1001这10个编码，而4位二进制能表示0~15共16个值，跳过了6个非法编码（1010~1111）。加6就是跳过这6个非法编码，进入下一个十进制位。

## 原理分析

### 例题1：不需修正的情况

计算 $3 + 4$ 的8421码加法：

$$3_{8421} = 0011, \quad 4_{8421} = 0100$$

$$0011 + 0100 = 0111 = 7_{8421}$$

结果 $0111 \leq 1001$，无进位，**不需要修正**。

结果：$0111$，即 $7$。✓

### 例题2：结果大于9，需要修正

计算 $8 + 5$ 的8421码加法：

$$8_{8421} = 1000, \quad 5_{8421} = 0101$$

第一步加法：$1000 + 0101 = 1101$

$1101 > 1001$（结果 $> 9$），需要 +6 修正：

$$1101 + 0110 = 1\ 0011$$

产生进位1，低4位为 $0011$。

结果：进位 $1$，本位 $0011$，即 $13$（$0001\ 0011_{8421}$）。

验证：$8 + 5 = 13$ ✓

### 例题3：产生进位，需要修正

计算 $7 + 6$ 的8421码加法：

$$7_{8421} = 0111, \quad 6_{8421} = 0110$$

第一步加法：$0111 + 0110 = 1101$

$1101 > 1001$，需要 +6 修正：

$$1101 + 0110 = 1\ 0011$$

结果：进位 $1$，本位 $0011$，即 $13$。

验证：$7 + 6 = 13$ ✓

### 例题4：多位BCD码加法

计算 $59 + 48$ 的8421码加法：

$$59_{8421} = 0101\ 1001, \quad 48_{8421} = 0100\ 1000$$

**低位（个位）相加**：$1001 + 1000 = 1\ 0001$

产生进位，需要 +6 修正：$0001 + 0110 = 0111$

低位结果：$0111$（$= 7$），向高位进位1。

**高位（十位）相加**：$0101 + 0100 + 1(\text{进位}) = 1010$

$1010 > 1001$，需要 +6 修正：$1010 + 0110 = 1\ 0000$

高位结果：$0000$（$= 0$），向更高位进位1。

**最终结果**：进位 $1$，高位 $0000$，低位 $0111$，即 $107$。

验证：$59 + 48 = 107$ ✓

## 常考题型

### 题型1：判断是否需要修正

**题目**：下列8421码加法中，不需要修正的是（）

A. $5 + 3$  B. $6 + 7$  C. $8 + 4$  D. $9 + 2$

**分析**：
- A：$5 + 3 = 8$，$0101 + 0011 = 1000 \leq 1001$，不需修正 ✓
- B：$6 + 7 = 13$，$0110 + 0111 = 1101 > 1001$，需修正
- C：$8 + 4 = 12$，$1000 + 0100 = 1100 > 1001$，需修正
- D：$9 + 2 = 11$，$1001 + 0010 = 1011 > 1001$，需修正

**答案**：A

### 题型2：余3码加法

余3码加法的修正规则：**结果减3（即 -0011）**，但如果产生进位则还需额外调整。

简单规则：余3码相加后，若无进位则结果-3（-0011），若有进位则结果+3（+0011）。

## 易错陷阱

### 陷阱1：修正次数
- 每位BCD码独立判断是否需要修正，修正后可能向高位产生新的进位
- 如果高位因此也需要修正，则高位也要+6

### 陷阱2：修正后仍产生进位
- 加6后可能产生新的进位，这个进位要正常传递到下一个BCD位

### 陷阱3：混淆BCD加法和二进制加法
- BCD加法：按4位一组分别处理，可能需要修正
- 二进制加法：整体作为一个数相加，不存在修正问题

### 陷阱4：8421码合法范围
- 8421码每个4位组只能是 `0000` ~ `1001`（0~9）
- `1010` ~ `1111` 是非法编码

## 知识关联

### 与纯二进制运算的对比
- 二进制加法：直接相加，处理进位即可
- BCD加法：相加后需检查是否超过9或有进位，超则+6修正
- BCD乘法：更为复杂，通常需要专用硬件或微程序支持

### 408真题特征
- BCD码加法修正是常考计算题
- 通常给定两个十进制数，要求写出8421码加法的完整过程
- 关键得分点：识别修正条件、正确加6、处理进位
