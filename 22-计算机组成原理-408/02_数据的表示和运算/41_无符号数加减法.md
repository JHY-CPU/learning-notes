# 42_无符号数加减法

## 核心概念

### 无符号数加法

$n$ 位无符号数的加法：直接按二进制加法运算，结果模 $2^n$。

$$A + B \mod 2^n$$

**溢出（进位）检测**：最高位产生进位则结果超出 $2^n - 1$。

### 无符号数减法

$n$ 位无符号数的减法：$A - B$（要求 $A \geq B$）

在计算机中，减法通过**补码加法**实现：

$$A - B = A + (2^n - B)$$

其中 $2^n - B$ 是 $B$ 对 $2^n$ 的**补数**。

**注意**：这里的"补数"是针对模 $2^n$ 的，与有符号补码不同。

---

## 原理分析

### 无符号加法示例（8位）

**例1：无溢出**

$$42 + 25 = 67$$

$$00101010 + 00011001 = 01000011 = 67$$

最高位无进位，结果正确。

**例2：有进位（溢出）**

$$200 + 100 = 300$$

$$11001000 + 01100100 = 1\ 00101100$$

最高位进位1，模256后：$00101100 = 44$

结果不等于300（被截断），但在模256系统中，$300 \equiv 44 \pmod{256}$。

### 无符号减法示例（8位）

**例1：$A > B$**

$$100 - 42$$

方法1：直接减法
$$100 - 42 = 58$$

方法2：补数加法
$$100 + (256 - 42) = 100 + 214 = 314$$
$$314 \mod 256 = 314 - 256 = 58$$

二进制：
$$[100] = 01100100$$
$$[-42]_{\text{补数}} = 256 - 42 = 214 = 11010110$$

$$01100100 + 11010110 = 1\ 00111010$$

舍弃进位：$00111010 = 58$ ✓

**例2：$A < B$（结果为负数的情况）**

$$42 - 100$$

$$42 + (256 - 100) = 42 + 156 = 198$$

二进制：
$$00101010 + 10011100 = 11000110 = 198$$

最高位无进位。但在无符号数中，$198$ 就是结果（实际应为 $-58$）。

**无符号数减法中，如果 $A < B$，结果会是一个很大的正数**，这是因为模运算的特性。

**无符号数的"借位"**：当 $A < B$ 时，减法过程中最高位会向更高位"借位"，这个借位标志在CPU中记录为 **CF**（Carry Flag）。

- CF = 0：无借位（$A \geq B$，结果正确）
- CF = 1：有借位（$A < B$，结果为模值）

## 直观理解

### 生活类比

无符号数减法就像一个只显示正数的计数器：
- 从100倒数到0，再倒数1次就变成255（8位）
- 这个255实际上是 $-1$ 的模等价值

### 有符号 vs 无符号溢出

| 类型 | 溢出标志 | 含义 |
|------|---------|------|
| 无符号加法 | CF（Carry Flag） | 最高位进位 |
| 无符号减法 | CF | 最高位借位（$A < B$） |
| 有符号加法 | OF（Overflow Flag） | 结果超出有符号范围 |

**关键区别**：
- 同一次运算，CF和OF可能不同
- CF关注无符号结果是否正确
- OF关注有符号结果是否正确

### 示例：$CF \neq OF$

$$200 + 200 = 400$$（8位）

$$11001000 + 11001000 = 1\ 10010000$$

- CF = 1（有进位，无符号溢出——$400 > 255$）
- OF = 0（异号... 不对，200在有符号中是-56，两个-56相加=-112，不溢出）

实际上 $[200]_{\text{补}} = -56$，$-56 + (-56) = -112$，在范围 $-128 \sim +127$ 内，OF = 0。

结果 `10010000` 无符号解释 = 144（错误，应为400），有符号解释 = $-112$（正确）。

### 408考试要点

### 易错陷阱
1. **无符号数没有"溢出"概念**：CF表示进位/借位，不是溢出
2. **$A < B$ 时减法结果很大**：这是正常的模运算结果
3. **同一运算CF和OF可能不同**：取决于从无符号还是有符号角度解释
