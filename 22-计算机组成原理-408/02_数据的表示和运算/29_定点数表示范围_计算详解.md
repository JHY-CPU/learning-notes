# 29_定点数表示范围_计算详解

## 原理分析

### 例题1：给定n位求各编码范围

**题目**：设机器字长为16位，分别求原码、补码、反码定点整数的表示范围。

**解**：

**原码**：$-(2^{16-1} - 1) \sim +(2^{16-1} - 1) = -32767 \sim +32767$

**补码**：$-2^{16-1} \sim +(2^{16-1} - 1) = -32768 \sim +32767$

**反码**：$-(2^{16-1} - 1) \sim +(2^{16-1} - 1) = -32767 \sim +32767$

### 例题2：定点小数范围

**题目**：设8位字长表示定点小数（1位符号位），求补码的表示范围和最小精度。

**解**：

- **范围**：$-1 \leq X \leq +(1 - 2^{-7}) = -1 \sim +\frac{127}{128}$
- **最小精度**：$2^{-7} = \frac{1}{128} \approx 0.0078125$
- **最大正数**：$0.1111111 = \frac{127}{128}$
- **最小负数**：$1.0000000 = -1$

### 例题3：编码数计算

**题目**：8位字长，原码和补码分别能表示多少个不同的数值？

**解**：

**原码**：
- 总编码数：$2^8 = 256$
- +0 和 -0 表示同一个值，所以有效数值个数：$256 - 1 = 255$

**补码**：
- 总编码数：$2^8 = 256$
- 只有一个0，所有编码都有效
- 有效数值个数：$256$

### 例题4：$-2^{n-1}$ 的特殊性

**题目**：为什么8位补码能表示 $-128$，而原码和反码不能？

**解**：

$-128$ 的补码：$256 + (-128) = 128 = 10000000$

$-128$ 的原码：需要在8位内表示 $-128$，符号位1 + 数值位需要7位能表示128。但7位最大表示 $2^7-1 = 127 < 128$，所以**原码无法表示 $-128$**。

$-128$ 的反码：基于原码，原码无法表示则反码也无法表示。

**原因**：补码的 $-2^{n-1}$ 利用了原码和反码中 $-0$ 的编码位置（`10000000`）。在原码中 `10000000` 表示 $-0$，在补码中它被重新利用表示 $-128$。

### 例题5：给定范围求位数

**题目**：如果需要用补码表示 $-1000 \sim +999$ 范围内的整数，至少需要多少位？

**解**：

需要满足：$-2^{n-1} \leq -1000$ 且 $2^{n-1} - 1 \geq 999$

$2^{n-1} \geq 1000$ 且 $2^{n-1} \geq 1000$

$n-1 \geq \lceil \log_2 1000 \rceil = \lceil 9.97 \rceil = 10$

$n \geq 11$

验证：11位补码范围 = $-1024 \sim +1023$，包含 $-1000 \sim +999$ ✓

10位补码范围 = $-512 \sim +511$，不满足 ✓

**答案**：至少需要 **11位**。

## 直观理解

### 计算公式速查

| 求什么 | 公式 |
|--------|------|
| n位补码整数最大值 | $2^{n-1} - 1$ |
| n位补码整数最小值 | $-2^{n-1}$ |
| n位原码整数最大值 | $2^{n-1} - 1$ |
| n位原码整数最小值 | $-(2^{n-1} - 1)$ |
| n位补码小数最大值 | $1 - 2^{-(n-1)}$ |
| n位补码小数最小值 | $-1$ |
| n位能表示的数值个数 | 原码$2^n-1$，补码$2^n$ |
| 需要的最小位数 | $\lceil \log_2(\max(|min|, max)+1) \rceil + 1$ |

## 知识关联

### 408考试应用
- 给定位数求范围（选择题常考）
- 给定范围求位数（需要反向推导）
- 理解溢出（结果超出表示范围）
- 编码效率分析

### 易错陷阱
1. 不要把原码和补码的范围搞混（补码多一个 $-2^{n-1}$）
2. 计算位数时注意符号位
3. 定点小数的范围公式和定点整数不同
4. 无符号数范围是 $0 \sim 2^n-1$，不涉及符号位
