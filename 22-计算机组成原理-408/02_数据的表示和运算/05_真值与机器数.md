# 05_真值与机器数

## 核心概念

### 定义

**真值（True Value）**：带有正负号的数值，是人类日常书写和理解的数值形式。如 $+127$、$-42$、$+3.14$ 等。

**机器数（Machine Number）**：数值在计算机内部的二进制编码表示，是一个纯0/1的位串，不显式包含正负号信息（正负由编码规则隐含表示）。

### 核心区别

| 特征 | 真值 | 机器数 |
|------|------|--------|
| 形式 | 带正负号的数值 | 纯0/1位串 |
| 可读性 | 人类可直接理解 | 需要知道编码规则才能解读 |
| 正负表示 | 用符号"+"、"-" | 用最高位（符号位）或特定编码 |
| 例 | $+127$ | `01111111`（原码） |
| 例 | $-42$ | `10101010`（原码） |

### 符号位约定

在定点数的机器数表示中：
- **最高位为符号位**：0表示正数，1表示负数
- 其余位为数值位（具体含义取决于编码方式：原码、补码、反码、移码）

### 同一位串，不同解读

以8位二进制 `11001000` 为例：

| 编码方式 | 如果是原码 | 如果是补码 | 如果是移码 |
|----------|-----------|-----------|-----------|
| 解读 | $-72$（符号位1，数值1001000=72） | $-56$ | 待定 |

**同一位串在不同编码下的真值完全不同！** 这是理解机器数的关键。

## 原理分析

### 真值 → 机器数的转换流程

1. **确定数值**：给出的真值（如 $-42$）
2. **分离符号和绝对值**：符号为"-"，绝对值为 $42$
3. **绝对值转二进制**：$42_{10} = 101010_2$
4. **按编码规则编码**：
   - 原码：符号位1 + 绝对值 → `1 0101010`
   - 补码：负数的补码 = 反码 + 1 → 详见后续章节
   - 反码：负数的反码 = 符号位1 + 绝对值各位取反

### 机器数 → 真值的转换流程

1. **识别编码方式**：确定是原码、补码、反码还是移码
2. **提取符号位**：最高位
3. **根据编码规则还原数值**：
   - 原码：符号位 + 数值位绝对值
   - 补码：正数直接读取；负数需先取补码还原
4. **写出真值**：组合符号和数值

### 定点整数与定点小数

**定点整数**：小数点固定在最低数值位之后
```
符号位 | 数值位（整数部分）
```

**定点小数**：小数点固定在符号位之后（即纯小数）
```
符号位 | 数值位（小数部分）
小数点↑
```

对于 $n$ 位机器数：
- 定点整数：1位符号位 + $(n-1)$ 位数值位
- 定点小数：1位符号位 + $(n-1)$ 位小数位

## 直观理解

### 生活类比

真值就像我们用文字描述温度："零下5度"——直接告诉你正负和数值。

机器数就像温度计上的一串刻度标记——你需要知道怎么读这个刻度（编码规则），才能理解它表示的是什么温度。

**类比**：同一个人的身份证号是机器数，而姓名是真值。你需要一部"编码表"（编码规则）才能从身份证号还原出这个人是谁。

### 记忆要点

1. **真值是"人话"**，机器数是"机器话"，编码规则是"翻译字典"
2. **同一个真值在不同编码下有不同的机器数**
3. **同一个机器数在不同编码下解读出不同的真值**
4. **符号位只在有符号数中使用**，无符号数所有位都是数值位

## 知识关联

### 跨章节联系
- **第一章**：冯·诺依曼体系中，数据和指令都以二进制存储，需要编码规则来区分含义
- **浮点数章节**：浮点数的阶码和尾数分别编码，需要各自独立的编码规则
- **CPU运算器**：ALU进行运算时，操作数必须是机器数形式
- **存储器**：存储的都是机器数，如何解释取决于指令类型

### 常考组合
1. 给定真值求机器数（原码/补码/反码）
2. 给定机器数求真值
3. 混合运算中机器数的加减法

### 易错陷阱
1. **混淆真值和机器数**：写答案时务必看清题目要的是哪种
2. **忘记编码方式**：同一个位串，不知道编码就无法确定真值
3. **符号位位置**：有符号数最高位是符号位，无符号数没有符号位
4. **溢出问题**：机器数的位数有限，超出范围的真值无法正确表示
