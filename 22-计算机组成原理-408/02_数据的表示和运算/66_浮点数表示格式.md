# 66_浮点数表示格式

## 核心概念

### 浮点数的一般格式

```
┌──────┬────────┬──────┬──────────────────────┐
│ 阶符 │  阶码  │ 数符 │        尾数          │
│ E_s  │   E    │ M_s  │         M            │
└──────┴────────┴──────┴──────────────────────┘
  1位    n位      1位        m位
```

其中：
- **阶符**（$E_s$）：阶码的符号位（用移码时不需要单独的阶符）
- **阶码**（$E$）：表示数量级
- **数符**（$M_s$）：尾数的符号 = 整个浮点数的符号
- **尾数**（$M$）：有效数字部分

### 阶码的编码方式

**移码**（最常用）：
- 将阶码值加上一个偏置值（Bias）
- $E_{移码} = E_{真值} + Bias$
- $Bias = 2^{n-1} - 1$（n 位阶码时）
- 优点：可以用无符号数比较阶码大小

**补码**：
- 直接用补码表示阶码
- 阶符在最高位

### 尾数的编码方式

**原码**：
- 符号位 + 数值绝对值
- 规格化条件：最高数值位为 1

**补码**：
- 符号位参与编码
- 规格化条件：符号位与最高数值位不同

### 常见浮点数格式对比

| 格式 | 总位数 | 阶码位数 | 尾数位数 | 阶码编码 | 尾数编码 |
|------|--------|---------|---------|---------|---------|
| 短浮点数 | 16 | 5 | 10 | 移码 | 原码 |
| 长浮点数 | 32 | 8 | 23 | 移码 | 原码 |
| IEEE 754 float | 32 | 8 | 23 | 移码(127) | 原码+隐含1 |
| IEEE 754 double | 64 | 11 | 52 | 移码(1023) | 原码+隐含1 |

### 408考试常见格式

408 考试中常见的格式（以唐朔飞版为例）：

```
15  14-12    11      10-0
┌───┬───────┬───┬───────────┐
│Es │  E    │Ms │     M     │
└───┴───────┴───┴───────────┘
 1位  3位    1位    11位
```

总位数 16 位：1位阶符 + 3位阶码 + 1位数符 + 11位尾数

- 阶码用**补码**，含阶符共 4 位
- 尾数用**原码**，含数符共 12 位
- 阶码的底 $R = 2$

### 浮点数真值计算

给定格式：阶符1位 + 阶码3位 + 数符1位 + 尾数11位

设浮点数为：$0,010; 1,11000000000$

- 阶码部分：$0010$（补码），真值 = $+2$
- 尾数部分：$1.11000000000$（原码），真值 = $-0.11_2 = -0.75$
- 浮点数 $N = (-0.75) \times 2^2 = -3.0$

## 原理分析

### 阶码用移码的原因

1. **比较方便**：移码可以直接用无符号比较器比较大小
2. **全0表示最小阶**：移码的全0对应最小的阶码值
3. **与浮点数的零值表示兼容**：阶码全0 + 尾数全0 = 零

### 阶码偏置值的选择

对于 n 位阶码（移码）：
- $Bias = 2^{n-1} - 1$
- 阶码范围：$[-(2^{n-1}-1), +(2^{n-1}-1)]$

| n | Bias | 范围 |
|---|------|------|
| 3 | 3 | [-3, +4] |
| 4 | 7 | [-7, +8] |
| 8 | 127 | [-127, +128] |
| 11 | 1023 | [-1023, +1024] |

### 尾数用原码的好处

- 规格化判断简单：最高位为1即可
- 与 IEEE 754 标准兼容
- 原码的0对称性

## 直观理解

### 记忆要点

- **阶码管范围（指数），尾数管精度（有效数字）**
- **数符是浮点数的符号，不是阶码的符号**
- **移码 = 真值 + 偏置值**
- **408考浮点格式，要能从二进制编码读出真值**

### 浮点数的"三件套"

浮点数 = **阶码**（决定大小）+ **尾数**（决定精度）+ **符号**（决定正负）
