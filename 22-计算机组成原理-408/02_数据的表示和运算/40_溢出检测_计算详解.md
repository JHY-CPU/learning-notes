# 40_溢出检测_计算详解

## 原理分析

### 三种溢出检测方法汇总

| 方法 | 判断依据 | $V=1$表示溢出 |
|------|---------|-------------|
| 单符号位（同号判断） | 两操作数同号且结果异号 | 是 |
| 单符号位（进位异或） | $C_s \oplus C_1 = 1$ | 是 |
| 双符号位 | 符号位不同（`01`或`10`） | 是 |

---

### 例题1：上溢检测（三种方法）

**题目**：用8位补码计算 $100 + 50$，判断是否溢出。

$$[100]_{\text{补}} = 01100100$$
$$[50]_{\text{补}} = 00110010$$

```
  01100100
+ 00110010
----------
  10010110
```

**方法1：同号判断法**
- 两操作数符号位：0（正数），同号
- 结果符号位：1（负数）
- 同号相加结果异号 → **上溢** ✓

**方法2：进位异或法**
- 次高位（第6位）：$1+1 = 10$，$C_1 = 1$
- 符号位（第7位）：$0+0+1 = 1$，$C_s = 0$
- $V = 0 \oplus 1 = 1$ → **溢出** ✓

**方法3：双符号位法**

$$[100]_{\text{变形补码}} = 00\ 1100100$$
$$[50]_{\text{变形补码}} = 00\ 0110010$$

```
  00 1100100
+ 00 0110010
------------
  01 0010110
```

双符号位 `01` → **上溢** ✓

### 例题2：下溢检测

**题目**：用8位补码计算 $(-100) + (-50)$，判断是否溢出。

$$[-100]_{\text{补}} = 10011100$$
$$[-50]_{\text{补}} = 11001110$$

```
  10011100
+ 11001110
----------
 101101010  （9位）
```

舍弃进位：$01101010$

**方法1：同号判断法**
- 两操作数符号位：1（负数），同号
- 结果符号位：0（正数）
- 同号相加结果异号 → **下溢** ✓

**方法2：进位异或法**
- 次高位：$0+1+进位 = $ 需要仔细计算
- $C_1 = 0$（最高数值位相加无进位）
- $C_s = 1$（符号位相加有进位）
- $V = 1 \oplus 0 = 1$ → **溢出** ✓

**方法3：双符号位法**

$$[-100]_{\text{变形补码}} = 11\ 0011100$$
$$[-50]_{\text{变形补码}} = 11\ 1001110$$

```
  11 0011100
+ 11 1001110
------------
 110 1101010
```

保留10位：`10 1101010`

双符号位 `10` → **下溢** ✓

### 例题3：无溢出（正+负）

**题目**：用8位补码计算 $100 + (-50)$，判断是否溢出。

$$[100]_{\text{补}} = 01100100$$
$$[-50]_{\text{补}} = 11001110$$

```
  01100100
+ 11001110
----------
 100110010  （9位）
```

舍弃进位：$00110010 = +50$

**方法1**：操作数异号（0和1），异号相加不会溢出 ✓

**方法2**：$C_1 = 1$，$C_s = 1$，$V = 1 \oplus 1 = 0$，无溢出 ✓

**方法3**：

```
  00 1100100
+ 11 1001110
------------
 100 0110010
```

保留10位：`00 0110010`（进位舍弃），双符号位 `00`，无溢出 ✓

### 例题4：边界值运算

**题目**：用8位补码计算 $(-128) + (-1)$，判断是否溢出。

$$[-128]_{\text{补}} = 10000000$$
$$[-1]_{\text{补}} = 11111111$$

```
  10000000
+ 11111111
----------
 101111111  （9位）
```

舍弃进位：$01111111 = +127$

两个负数相加得正数 → **下溢**！

实际结果应为 $-129$，但8位补码最小值为 $-128$，超出范围。

**双符号位验证**：

```
  11 0000000
+ 11 1111111
------------
 110 1111111
```

保留10位：`10 1111111`，双符号位 `10` → 下溢 ✓

## 直观理解

### 快速判断技巧

1. **先看操作数符号**：
   - 异号 → 不会溢出
   - 同号 → 需要进一步判断

2. **同号时看结果符号**：
   - 结果同号 → 无溢出
   - 结果异号 → 溢出

3. **数值大小判断**（辅助验证）：
   - 正+正：结果应 $< 2^{n-1}$，否则上溢
   - 负+负：结果应 $\geq -2^{n-1}$，否则下溢
