# 46_乘法_原码一位乘

## 核心概念

### 原码一位乘的定义

原码一位乘法是利用**移位和加法**操作来实现乘法运算的方法。其核心思想是将乘法分解为一系列的加法和移位操作，每次处理乘数的**一位**。

### 关键思想

- **符号位单独处理**：结果的符号位 = 被乘数符号位 $\oplus$ 乘数符号位（异或）
- **数值部分用移位加法实现**：逐位判断乘数的每一位，若为1则加被乘数，若为0则不加，然后右移
- **不直接用乘法器**：只用加法器 + 移位器 + 寄存器即可完成

### 三个核心寄存器

| 寄存器 | 名称 | 功能 |
|--------|------|------|
| **ACC** | 累加器 | 存放部分积的高位，最终存放乘积的高位 |
| **MQ** | 乘商寄存器 | 存放乘数，运算过程中逐步移出，最终存放乘积的低位 |
| **X** | 操作数寄存器 | 存放被乘数 |

> **注意**：乘法开始前，部分积（ACC）清零。

### 乘法运算的核心公式

设被乘数 $[x]_{原} = x_s.x_1x_2\cdots x_n$，乘数 $[y]_{原} = y_s.y_1y_2\cdots y_n$

对于数值部分的无符号乘法：

$$|x| \times |y| = |x| \times \sum_{i=1}^{n} y_i \times 2^{-i} = \sum_{i=1}^{n} (y_i \times |x|) \times 2^{-i}$$

即每次取乘数最低位 $y_i$，若 $y_i=1$ 则累加 $|x| \times 2^{-i}$（即加后移位）。

### 一位乘的工作步骤（数值部分）

**共进行 n 轮操作**（n 为数值位数）：

1. **判断乘数最低位** $y_n$：
   - 若 $y_n = 1$：`ACC ← ACC + X`（部分积加被乘数）
   - 若 $y_n = 0$：不加（相当于加 0）
2. **右移一位**：`(ACC, MQ) ← (ACC, MQ) >> 1`
   - ACC 和 MQ 连接在一起整体右移
   - ACC 的最低位移入 MQ 的最高位
   - ACC 的最高位（符号位侧）补 0
3. **重复**步骤1-2，共 n 次

### 运算过程示意

```
初始：ACC = 0...0，MQ = |乘数|，X = |被乘数|

第1轮：看MQ最低位 → 加/不加X → 右移
第2轮：看MQ最低位 → 加/不加X → 右移
...
第n轮：看MQ最低位 → 加/不加X → 右移

最终：ACC存放乘积高位，MQ存放乘积低位
```

## 原理分析

### 运算过程的数学推导

设 $|x| = 0.1101$（被乘数），$|y| = 0.1011$（乘数）

$$|x| \times |y| = 0.1101 \times 0.1011$$

按手算竖式：
```
      0.1101
    × 0.1011
    ---------
      1101     ← y4=1, 即 0.1101 × 2⁻⁴
     1101      ← y3=1, 即 0.1101 × 2⁻³
    0000       ← y2=0
   1101        ← y1=1, 即 0.1101 × 2⁻¹
   ---------
   0.10001111
```

计算机的做法是从**最低位开始**，逐位处理，每步加完后右移（相当于乘以 $2^{-1}$）。

### 原码一位乘的流程图

```
开始
  |
初始化：ACC ← 0, MQ ← |乘数|, X ← |被乘数|, 计数器C ← n
  |
+--→ 判断 MQ最低位
|     |
|     +-- y_n=1: ACC ← ACC + X
|     |
|     +-- y_n=0: (不加)
|     |
|     右移: (ACC, MQ) >> 1
|     |
|     C ← C - 1
|     |
+----- C ≠ 0?
        |
        C = 0 → 结束
```

### 运算规则总结

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 判断乘数最低位 | 决定是否加被乘数 |
| 2 | 加法（可能） | $y_n=1$ 时执行 `ACC+X` |
| 3 | 右移 | ACC和MQ联合右移，等价于部分积乘以 $2^{-1}$ |
| 4 | 重复 n 次 | 每次处理乘数的一位 |

### 结果存放

- 乘积的**高 n 位**在 ACC 中
- 乘积的**低 n 位**在 MQ 中
- 结果共 **2n 位**（不含符号位）
- 符号位单独计算：$s = x_s \oplus y_s$

## 直观理解

### 生活类比：分批付钱

假设被乘数是单价 1101 元，乘数是数量 1011 个。从数量的最低位开始：

- 第1位是1：先收 1101 元（加一次），然后金额除以10（移位）
- 第2位是1：再收 1101 元，然后除以10
- 第3位是0：不收钱，直接除以10
- 第4位是1：收 1101 元，除以10

每次右移 = 等价于把当前位的权重从"个位"调整到正确位置。

### 记忆技巧

- **"看1加，看0跳，每次右移"**
- 符号位：**异或**（同号为正，异号为负）
- 右移 = 除以2 = 权重降低一级
- 共移位 **n 次**，加法最多 **n 次**

### 与手算的对应

手算乘法从**最高位**开始，逐位乘完后按位权对齐相加。

计算机从**最低位**开始，每处理一位就右移，逐步累加。两者数学等价，但计算机从低位开始更方便（右移不影响已处理的位）。

### 与其他乘法的对比

| 特征 | 原码一位乘 | 补码一位乘(Booth) | 原码两位乘 |
|------|-----------|------------------|-----------|
| 符号处理 | 单独计算 | 参与运算 | 单独计算 |
| 每次看几位 | 1位 | 2位（含附加位） | 2位 |
| 迭代次数 | n | n+1次右移 | n/2 |
| 加法次数 | 最多n次 | 最多n+1次 | 最多n/2次 |
| 速度 | 慢 | 中等 | 快 |

### 易错陷阱

1. **符号位不能参与数值运算**：先去掉符号位，数值部分运算完再拼符号
2. **右移时 ACC 高位补 0 而非补符号位**：这是无符号数的右移（逻辑右移）
3. **MQ 右移时最低位丢失**：被判断过就不需要了
4. **最终结果是 2n 位**：不要只看 ACC，MQ 中也有
5. **初始 ACC 必须清零**：部分积从 0 开始累加
