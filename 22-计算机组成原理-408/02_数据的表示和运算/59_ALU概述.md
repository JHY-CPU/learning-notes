# 60_ALU概述

## 核心概念

### ALU 的定义

**ALU（Arithmetic Logic Unit，算术逻辑单元）**是计算机CPU中负责执行算术运算和逻辑运算的核心部件。它是运算器的核心组成部分。

### ALU 的功能

ALU 可以执行两大类运算：

**算术运算**：
- 加法（ADD）
- 减法（SUB）
- 乘法（MUL）—— 部分ALU支持
- 除法（DIV）—— 部分ALU支持
- 加1/减1（INC/DEC）

**逻辑运算**：
- 与（AND）
- 或（OR）
- 非（NOT）
- 异或（XOR）
- 移位（SHL/SHR）

### ALU 的输入输出

```
          A (n位输入)
          │
          ▼
    ┌───────────┐
    │           │
    │   ALU     │──── F (n位输出)
    │           │
    └───────────┘
          │
          B (n位输入)
          
    控制信号 ALUop (决定执行哪种运算)
```

**输入**：
- **操作数 A**：n 位数据，来自寄存器或数据总线
- **操作数 B**：n 位数据，来自寄存器或数据总线
- **控制信号（ALUop）**：决定执行哪种运算

**输出**：
- **结果 F**：n 位运算结果
- **标志位**：零标志（ZF）、进位（CF）、溢出（OF）、符号（SF）等

### ALU 的标志位

| 标志位 | 名称 | 含义 |
|--------|------|------|
| ZF | Zero Flag | 结果是否为零 |
| CF | Carry Flag | 是否产生进位（无符号数溢出） |
| OF | Overflow Flag | 是否溢出（有符号数溢出） |
| SF | Sign Flag | 结果的符号位 |
| PF | Parity Flag | 结果中1的个数是否为偶数 |

### ALU 的控制信号编码

以4位ALU为例，控制信号可以编码为：

| ALUop | 功能 |
|-------|------|
| 000 | AND |
| 001 | OR |
| 010 | ADD |
| 011 | NOT |
| 110 | SUB |
| 111 | SLT（小于置位）|

## 原理分析

### ALU 的内部结构

ALU 的核心是一个**加法器**，因为：
- 减法 = 加法（通过补码取负）
- 乘法 = 多次加法 + 移位
- 除法 = 多次减法 + 移位
- 逻辑运算可以由门电路直接实现

```
        A    B
        │    │
        ▼    ▼
    ┌──────────────┐
    │  多路选择器   │ ← ALUop控制
    │  (MUX)       │
    └──────┬───────┘
           │
     ┌─────┼─────┬──────┐
     ▼     ▼     ▼      ▼
   加法器 AND   OR    NOT ...
     │     │     │      │
     └─────┴─────┴──────┘
           │
           ▼
      结果F + 标志位
```

### 74181 ALU 芯片

经典的 74181 是 4 位 ALU 芯片，可以执行 16 种算术运算和 16 种逻辑运算。

输入：4位A，4位B，模式选择M，功能选择S3S2S1S0
输出：4位F，进位Cn+4

### 现代 ALU 特点

现代处理器的 ALU 通常包含：
- 整数 ALU（定点运算）
- 浮点 ALU（浮点运算）
- SIMD ALU（单指令多数据，向量运算）
- 多个 ALU 并行工作

## 直观理解

### 类比

ALU 就像一个"万能计算器"：
- 输入两个数和一个操作指令
- 输出计算结果
- 标志位就像"计算状态灯"（结果为零？有进位？溢出？）

### 记忆要点

- **ALU = 算术 + 逻辑**，核心是加法器
- **控制信号决定功能**
- **标志位记录运算状态**
- **n 位 ALU 由 n 个 1 位 ALU 级联组成**

## 知识关联

- 加法器（第61-63节）是 ALU 的核心部件
