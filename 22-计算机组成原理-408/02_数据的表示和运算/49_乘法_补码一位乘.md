# 49_乘法_补码一位乘

## 核心概念

### 为什么要用补码乘法？

原码乘法的缺点：
- 需要单独处理符号位
- 计算机内部存储的是补码，用原码乘法需要额外的转换
- 不能直接处理负数的乘法

补码乘法的优势：
- **符号位参与运算**，无需单独处理
- 操作数本身就是补码形式，不需要转换
- 一次完成，效率更高

### 补码一位乘与原码一位乘的关键区别

| 特征 | 原码一位乘 | 补码一位乘 |
|------|-----------|-----------|
| 符号位 | 单独异或 | 参与运算 |
| 被加数 | 始终是 $[x]_原$ | 根据乘数位，可能是 $[x]_补$ 或 $[-x]_补$ |
| 右移方式 | 逻辑右移（补0） | 算术右移（补符号位） |
| 迭代次数 | n 次 | n 次 |
| 附加位 | 不需要 | 需要（$y_{n+1} = 0$） |
| 最后一步 | 正常右移 | **不右移** |

### 补码一位乘的核心公式

设 $[x]_补 = x_0.x_1x_2\cdots x_n$，$[y]_补 = y_0.y_1y_2\cdots y_n$

补码一位乘基于以下恒等式：

$$[x \times y]_补 = [x]_补 \times \sum_{i=1}^{n} (y_i - y_{i+1}) \times 2^{-i}$$

其中定义 $y_{n+1} = 0$（附加位）。

展开来看，每一步的修正量为 $(y_i - y_{i+1})$：
- $(y_i, y_{i+1}) = (0, 0)$ 或 $(1, 1)$：修正量为 0，只需右移
- $(y_i, y_{i+1}) = (0, 1)$：修正量为 +1，加 $[x]_补$ 后右移
- $(y_i, y_{i+1}) = (1, 0)$：修正量为 -1，加 $[-x]_补$ 后右移

### 补码一位乘的算法步骤

**准备**：
- ACC = 0（部分积初始值，补码形式）
- MQ = $[y]_补$（乘数的补码）
- X = $[x]_补$（被乘数的补码）
- $\bar{X} = [-x]_补$（被乘数负数的补码）
- 附加位 $y_{n+1} = 0$

**迭代过程**（共 n 轮）：

对 $i = n, n-1, \ldots, 1$（从最低位开始）：

1. **看 $(y_i, y_{n+1})$ 这两位**（当前最低位和附加位）：
   - $(0, 0)$ 或 $(1, 1)$：不加，直接右移
   - $(0, 1)$：`ACC ← ACC + [x]_补`，然后右移
   - $(1, 0)$：`ACC ← ACC + [-x]_补`，然后右移

2. **右移**：ACC 和 MQ 联合算术右移一位
   - ACC 高位补原来的符号位（算术右移）
   - MQ 最低位移入附加位 $y_{n+1}$

3. **注意**：第 n 轮（最后一轮）之后**不再右移**！

### 为什么最后一步不右移？

在补码乘法中，部分积本身就是补码形式，且符号位已经参与了运算。最后一步如果再右移，会导致符号位被破坏，结果不正确。

具体来说，共进行 **n 次加法判断** 和 **n-1 次右移**（最后一步判断后不移位）。也可以说共 **n+1 次判断**（含附加位的初始判断）。

> **不同教材的表述可能略有差异**：有的说"共 n 轮，最后一轮不移位"，有的说"共 n+1 次判断，n 次移位"。核心含义一致。

## 原理分析

### 补码一位乘的推导

设 $[x]_补 = x_0.x_1x_2\cdots x_n$，$[y]_补 = y_0.y_1y_2\cdots y_n$

$$[x \times y]_补 = [x]_补 \times (y_1 \cdot 2^{-1} + y_2 \cdot 2^{-2} + \cdots + y_n \cdot 2^{-n} - y_0)$$

但上述公式不便于逐位运算。引入附加位 $y_{n+1} = 0$ 后：

$$[x \times y]_补 = [x]_补 \times \sum_{i=1}^{n}(y_i - y_{i+1}) \times 2^{-i}$$

每一轮从低位到高位处理 $(y_i, y_{i+1})$ 这两位的组合，决定加什么。

### 算术右移的含义

补码乘法中的右移是**算术右移**：
- ACC（部分积）的符号位保持不变
- 右移时高位补符号位
- 这保证了补码的正确性（正数补0，负数补1）

## 直观理解

### 记忆技巧

- **"看相邻两位，00/11只移，01加正，10加负"**
- **"最后一步不移位"** — 这是补码乘法最容易忘的点
- 附加位 $y_{n+1} = 0$ 是判断的锚点，让第一轮也能正确判断

### 与 Booth 算法的关系

补码一位乘法就是 **Booth 算法** 的原始形式。Booth 算法是对补码一位乘的规范化表述。在实际考试中，两者经常等价使用。

## 知识关联

- 与原码一位乘对比：原码符号单独算，补码符号参与算
- 与 Booth 算法（第50节）本质上是同一算法
- 算术右移的概念在第1章补码右移中已学过
- 补码除法（加减交替法）与补码乘法有类似的思想
