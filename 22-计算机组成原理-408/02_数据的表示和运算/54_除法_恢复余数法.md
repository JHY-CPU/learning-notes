# 54_除法_恢复余数法

## 核心概念

### 原码恢复余数法的定义

原码恢复余数法是最基本的**定点除法算法**。其核心思想是：

1. **符号位单独处理**：商的符号位 = 被除数符号位 $\oplus$ 除数符号位
2. **数值部分做除法**：用 $|$被除数$|$ ÷ $|$除数$|$ 的无符号除法
3. **每次用减法试探**：余数 - 除数，若结果为正则商1，为负则商0并恢复余数

### 关键术语

- **被除数（Dividend）**：被除的数
- **除数（Divisor）**：用来除的数
- **商（Quotient）**：除法的结果
- **余数（Remainder）**：除法的剩余部分

### 三个核心寄存器

| 寄存器 | 功能 |
|--------|------|
| **ACC** | 存放被除数/余数 |
| **MQ** | 存放商 |
| **X** | 存放除数 |

### 恢复余数法的工作原理

**基本思想**：手算除法的计算机实现

手算除法：
```
        1 0 1 1     ← 商
      _________
 110 | 1 0 0 0 0    ← 被除数
       1 1 0        ← 减去除数(够减)
      __________
         1 0 0      ← 余数
         1 1 0      ← 减去除数(不够减)
        __________
         ...        ← 恢复余数
```

计算机的实现方式：
1. **初始**：ACC = $|$被除数$|$，X = $|$除数$|$
2. **每一轮**：
   - 执行 `ACC ← ACC - X`（试探性减法）
   - 检查 ACC 的符号：
     - **ACC ≥ 0（够减）**：商 1，余数保留
     - **ACC < 0（不够减）**：商 0，**恢复余数** `ACC ← ACC + X`
   - 左移 ACC（余数左移等价于乘以2，为下一位的减法做准备）
3. **重复** n 轮

### 为什么是左移？

除法中余数左移，等价于：
- 余数扩大2倍（对应十进制中"下一位"的权重）
- 与乘法中的右移形成对比（乘法是部分积缩小）

> 注意：也有的实现方式用右移（方向取决于具体硬件设计），但最常见的是**左移**。部分教材描述为右移（把除数往低位对齐）。考试以教材为准。

## 原理分析

### 算法流程

```
初始化：
  ACC ← |被除数|（数值部分）
  X ← |除数|
  MQ ← 0（商初始为0）
  C ← n（迭代计数器）

循环（共n轮）：
  1. ACC ← ACC - X（试探减法）
  2. 判断ACC符号位：
     - 若 ACC ≥ 0：MQ最低位置1，保留余数
     - 若 ACC < 0：MQ最低位置0，ACC ← ACC + X（恢复余数）
  3. 左移：ACC ← ACC << 1（余数左移1位）
     MQ ← MQ << 1（商左移，准备下一位）
  4. C ← C - 1，若 C ≠ 0 返回步骤1

特殊情况：若最后一轮余数为负，仍需恢复
```

### 为什么叫"恢复余数"？

当试探减法结果为负时（不够减），说明刚才的减法是错误的。需要**加回除数**来恢复原来的余数，然后再左移进行下一步。这就是"恢复余数"名称的由来。

### 数学原理

除法等价于：

$$\frac{|被除数|}{|除数|} = \sum_{i=1}^{n} q_i \times 2^{-i} + \frac{余数}{|除数| \times 2^{-n}}$$

其中 $q_i$ 是第 $i$ 位商，每次判断"当前余数是否够减除数"。

## 直观理解

### 生活类比：分苹果

有 100 个苹果，每次分 30 个：
- 第1次：100 - 30 = 70 ≥ 0，够分，商+1
- 第2次：70 - 30 = 40 ≥ 0，够分，商+1
- 第3次：40 - 30 = 10 ≥ 0，够分，商+1
- 第4次：10 - 30 = -20 < 0，不够分，恢复（加回30得10），商+0

结果：商 = 3，余数 = 10

计算机中的左移等价于"把余数看成下一位的数字"（10→100），继续除。

### 记忆要点

- **"减了够，商写1；减不够，商写0，加回来"**
- **"恢复余数 = 加回除数"**
- 符号位单独处理（异或）
