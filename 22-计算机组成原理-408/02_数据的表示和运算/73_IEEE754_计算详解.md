# 74_IEEE754_计算详解

## 核心概念

本文件详细演示十进制与 IEEE 754 之间的转换过程。

### 转换公式回顾

$$N = (-1)^S \times 1.M \times 2^{e - Bias}$$

其中 Bias = 127（float），1023（double）

---

## 十进制 → IEEE 754

### 例题 1：将 $13.625$ 转为 IEEE 754 单精度

**第一步**：将整数部分转为二进制
$$13 = 1101_2$$

**第二步**：将小数部分转为二进制
$$0.625 \times 2 = 1.25 \to 1$$
$$0.25 \times 2 = 0.5 \to 0$$
$$0.5 \times 2 = 1.0 \to 1$$
$$0.625 = 0.101_2$$

**第三步**：合并
$$13.625 = 1101.101_2$$

**第四步**：规格化
$$1101.101 = 1.101101 \times 2^3$$

- 尾数部分（隐含1后）：$101101$
- 阶码真值：$3$

**第五步**：计算存储的阶码
$$e = 3 + 127 = 130 = 10000010_2$$

**第六步**：组装
- $S = 0$
- 阶码：$10000010$
- 尾数：$10110100000000000000000$（23位，后面补0）

**IEEE 754 表示**：
```
0 10000010 10110100000000000000000
```

**十六进制**：`0x415A0000`

---

### 例题 2：将 $-0.75$ 转为 IEEE 754 单精度

**第一步**：转为二进制
$$0.75 = 0.11_2$$

**第二步**：规格化
$$0.11 = 1.1 \times 2^{-1}$$

**第三步**：计算阶码
$$e = -1 + 127 = 126 = 01111110_2$$

**第四步**：组装
- $S = 1$（负数）
- 阶码：$01111110$
- 尾数：$10000000000000000000000$

**IEEE 754 表示**：
```
1 01111110 10000000000000000000000
```

**十六进制**：`0xBF400000`

---

### 例题 3：将 $0.1$ 转为 IEEE 754 单精度

**第一步**：转为二进制（循环小数）
$$0.1 \times 2 = 0.2 \to 0$$
$$0.2 \times 2 = 0.4 \to 0$$
$$0.4 \times 2 = 0.8 \to 0$$
$$0.8 \times 2 = 1.6 \to 1$$
$$0.6 \times 2 = 1.2 \to 1$$
$$0.2 \times 2 = 0.4 \to 0$$
...循环

$$0.1 = 0.00011001100110011..._2$$

**第二步**：规格化
$$= 1.1001100110011... \times 2^{-4}$$

**第三步**：计算阶码
$$e = -4 + 127 = 123 = 01111011_2$$

**第四步**：尾数截断到23位
$10011001100110011001100$

**IEEE 754 表示**：
```
0 01111011 10011001100110011001100
```

**十六进制**：`0x3DCCCCCD`

> 注意：由于截断和舍入，IEEE 754 中 $0.1$ 不精确，这就是 $0.1 + 0.2 \neq 0.3$ 的原因。

## IEEE 754 → 十进制

### 例题 4：`0x42280000` 是什么数？

**第一步**：转为二进制
$$42280000_{16} = 0100\ 0010\ 0010\ 1000\ 0000\ 0000\ 0000\ 0000_2$$

**第二步**：划分字段
```
0 10000100 01010000000000000000000
S=0  e=10000100  M=0101000...0
```

**第三步**：计算各部分
- $S = 0$（正数）
- $e = 10000100_2 = 132$
- $E = 132 - 127 = 5$
- 尾数：$1.0101000...0_2 = 1 + 0.25 + 0.0625 = 1.3125$

**第四步**：计算真值
$$N = 1.3125 \times 2^5 = 1.3125 \times 32 = 42.0$$

---

### 例题 5：`0xC0E00000` 是什么数？

**第一步**：转为二进制
$$C0E00000_{16} = 1100\ 0000\ 1110\ 0000\ 0000\ 0000\ 0000\ 0000_2$$

**第二步**：划分字段
```
1 10000001 11000000000000000000000
S=1  e=10000001  M=110000...0
```

**第三步**：计算各部分
- $S = 1$（负数）
- $e = 10000001_2 = 129$
- $E = 129 - 127 = 2$
- 尾数：$1.11_2 = 1 + 0.5 + 0.25 = 1.75$

**第四步**：计算真值
$$N = -1.75 \times 2^2 = -1.75 \times 4 = -7.0$$

---

### 例题 6：判断 `0x7F800000` 是什么？

二进制：`0 11111111 00000000000000000000000`

- $S = 0$
- 阶码 = 全1（255）
- 尾数 = 全0

答案：**正无穷大**（$+\infty$）

---

### 例题 7：判断 `0x7FC00000` 是什么？

二进制：`0 11111111 10000000000000000000000`

- $S = 0$
- 阶码 = 全1（255）
- 尾数 ≠ 0

答案：**NaN**

## 转换技巧总结

### 十进制 → IEEE 754

1. 整数部分：除2取余
2. 小数部分：乘2取整
3. 合并后规格化：$1.xxx \times 2^E$
4. $e = E + Bias$
5. 组装 $S$, 阶码, 尾数

### IEEE 754 → 十进制

1. 拆分 $S$, $e$, $M$
2. 判断特殊值（阶码全0/全1）
3. $E = e - Bias$
4. 真值 $= (-1)^S \times 1.M \times 2^E$

## 知识关联

- 这些计算是 408 考试的**必考内容**
- 需要熟练掌握双向转换
- 注意舍入误差（如 0.1 的表示）
