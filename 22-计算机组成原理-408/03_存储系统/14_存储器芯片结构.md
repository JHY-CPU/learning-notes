# 14_存储器芯片结构

## 核心概念

### 存储器芯片的三大组成部分

```
                    地址总线
                       │
                ┌──────┴──────┐
                │  地址译码器  │
                └──────┬──────┘
                       │ 字线(行选择)
                ┌──────┴──────┐
   数据总线 ←──→ │  存储矩阵   │ ←── 行列交叉选择
                └──────┬──────┘
                       │
                ┌──────┴──────┐
                │ 读写控制电路 │ ←── 控制信号(CE, WE, OE)
                └─────────────┘
```

1. **存储矩阵（Memory Array）**：大量存储单元按行列排列
2. **地址译码器（Address Decoder）**：将地址信号翻译为选中某行/列的信号
3. **读写控制电路（Read/Write Control）**：控制读/写操作和数据流向

### 存储矩阵的两种组织方式

#### 字结构（Word-Organized）

- 每个存储单元存放**一个完整的字**
- 数据线宽度 = 字长
- 例：$1\text{K} \times 8$ 位芯片，有1024个单元，每单元8位
- 适合：字长需求明确的场合

```
地址 → 译码器 → 选中一行 → 8位数据同时读出/写入
```

#### 位结构（Bit-Organized）

- 每个存储单元只存**1位**
- 需要多片并联才能组成完整字长
- 例：$16\text{K} \times 1$ 位芯片，有16384个单元，每单元1位
- 适合：大容量存储器的位扩展

```
地址 → 译码器 → 选中一行 → 1位数据读出/写入
多片并联 → 拼成完整字长
```

### 地址译码方式

#### 线性译码（单译码器）

- 1个n位地址 → $2^n$ 条字线
- 每条字线选中一个存储单元
- 适用于小容量存储器
- 缺点：地址线多时，字线数量爆炸（n=20时需要100万条字线）

#### 复合译码（行列译码/双译码器）

- 将地址分为行地址和列地址
- 行译码器：$n/2$ 位 → $2^{n/2}$ 条行选择线
- 列译码器：$n/2$ 位 → $2^{n/2}$ 条列选择线
- 行列交叉处选中存储单元
- **字线总数 = $2^{n/2} + 2^{n/2} = 2 \times 2^{n/2}$**（远少于 $2^n$）

**例**：20位地址
- 线性译码：需要 $2^{20} = 1048576$ 条字线
- 复合译码（10+10）：需要 $2^{10} + 2^{10} = 2048$ 条选择线
- 大幅减少！

## 原理分析

### 存储单元的内部结构

每个存储单元（SRAM为例）包含：
- 6个MOS管构成双稳态触发器
- 2个访问管连接到位线
- 字线控制访问管的通断

### 典型芯片结构（以 $4\text{K} \times 8$ 位SRAM为例）

- 地址线：$\log_2(4096) = 12$ 根（A0~A11）
- 数据线：8根（D0~D7）
- 控制线：CE（片选）、WE（写使能）、OE（输出使能）
- 内部结构：
  - 复合译码：6位行地址（64行）+ 6位列地址（64列）
  - 存储矩阵：64 × 64 = 4096 个单元
  - 每个单元8位

### 存储芯片容量的表示

**标准格式**：$M \times N$
- $M$ = 存储单元数（地址数）
- $N$ = 每单元位数（字长/数据线宽度）

**例**：
- $1\text{K} \times 4$：1024个单元，每单元4位
- $8\text{K} \times 8$：8192个单元，每单元8位
- $256\text{K} \times 1$：262144个单元，每单元1位

$$\text{总容量} = M \times N \quad (\text{bit})$$

## 直观理解

**存储矩阵像教室座位表**：
- **行译码器** = 按排号找人（第几排）
- **列译码器** = 按列号找人（第几列）
- **行列交叉** = 确定是哪个座位（哪个存储单元）
- **线性译码** = 给每个座位编号，直接查编号（座位多了编号位数爆炸）
- **复合译码** = 先说第几排再说第几列（信息量一样，但需要的"路"少得多）

### 易错陷阱

1. **地址线数量 = $\log_2$(存储单元数)**，不是 $\log_2$(容量字节数)
2. **数据线数量 = 字长位数**
3. **复合译码的字线数 = $2 \times 2^{n/2}$**，不是 $2^{n/2}$
4. **存储芯片的容量标注 $M \times N$ 中，M是单元数不是字节数**
