# 70_虚拟存储_计算详解

## 例题1: 基本EAT计算

**题目**: 某系统主存访问时间100ns, 磁盘访问时间10ms。TLB命中率96%, 缺页率2%。不考虑Cache, 求有效访问时间。

**解**:

四种情况:
1. TLB命中且不缺页: 概率 = $0.96 \times 0.98$, 时间 = 100ns
2. TLB命中且缺页: 概率 = $0.96 \times 0.02$, 时间 = $100 + 10^7$ ns
3. TLB未命中且不缺页: 概率 = $0.04 \times 0.98$, 时间 = 200ns (查页表100 + 访数据100)
4. TLB未命中且缺页: 概率 = $0.04 \times 0.02$, 时间 = $200 + 10^7$ ns

$$EAT = 0.96 \times 0.98 \times 100 + 0.96 \times 0.02 \times 10^7 + 0.04 \times 0.98 \times 200 + 0.04 \times 0.02 \times (200 + 10^7)$$

$$= 94.08 + 192000 + 7.84 + 160$$

$$= 192261.92 \text{ ns} \approx 192.3 \text{ }\mu\text{s}$$

**关键结论**: 缺页处理时间(10ms = $10^7$ ns)远大于主存时间(100ns), 即使只有2%缺页率, EAT也被严重拉高。

## 例题2: 含Cache的完整EAT

**题目**: 某计算机:
- TLB命中率: 95%, 查TLB时间: 1ns
- Cache命中率: 98%, 查Cache时间: 2ns
- 主存访问时间: 80ns
- 缺页率: 1%, 缺页处理时间: 8ms
- 假设Cache未命中时才可能缺页

求有效访问时间。

**解**:

简化公式 (Cache未命中时查主存, 可能缺页):

$$EAT = h_t \times [h_c \times t_c + (1-h_c) \times t_m] + (1-h_t) \times [h_c \times t_c + (1-h_c) \times 2t_m] + f \times t_{\text{pf}}$$

更精确:
$$EAT_{\text{基本}} = h_t h_c (t_t + t_c) + h_t (1-h_c)(t_t + t_m) + (1-h_t)h_c(t_t + t_m + t_c) + (1-h_t)(1-h_c)(t_t + 2t_m)$$

代入:
- $h_t h_c (t_t + t_c) = 0.95 \times 0.98 \times 3 = 2.793$
- $h_t (1-h_c)(t_t + t_m) = 0.95 \times 0.02 \times 81 = 1.539$
- $(1-h_t)h_c(t_t + t_m + t_c) = 0.05 \times 0.98 \times 83 = 4.067$
- $(1-h_t)(1-h_c)(t_t + 2t_m) = 0.05 \times 0.02 \times 161 = 0.161$

基本EAT = $2.793 + 1.539 + 4.067 + 0.161 = 8.56$ ns

加上缺页代价:
$$EAT = 8.56 + 0.01 \times 8 \times 10^6 = 8.56 + 80000 = 80008.56 \text{ ns}$$

**结论**: 缺页代价主导了EAT!

## 例题3: 多级页表EAT

**题目**: 某系统采用二级页表, 每级页表4KB(与页面大小相同), 页目录常驻内存, 次级页表可能缺页。主存访问时间100ns, 磁盘访问8ms。TLB命中率98%。次级页表缺页率0.5%, 数据缺页率1%。

求不考虑Cache时的EAT。

**解**:

TLB命中时(98%): 访存1次 = 100ns

TLB未命中时(2%): 查二级页表
- 页目录常驻内存, 必定命中: 访存1次
- 次级页表:
  - 在内存(99.5%): 访存1次 → 共2次
  - 缺页(0.5%): 缺页处理8ms → 共 100 + 8×10⁶ + 100 ns

得到PPN后, 查数据:
- 不缺页(99%): 访存1次
- 缺页(1%): 缺页处理8ms

$$EAT = 0.98 \times 100 + 0.02 \times [100 + 0.995 \times 100 + 0.005 \times 8 \times 10^6 + 0.99 \times 100 + 0.01 \times 8 \times 10^6]$$

简化:
TLB未命中基本时间 ≈ $100 + 100 + 100 = 300$ ns
缺页代价 = $0.02 \times (0.005 + 0.01) \times 8 \times 10^6 = 0.02 \times 0.015 \times 8 \times 10^6 = 2400$ ns

$$EAT \approx 0.98 \times 100 + 0.02 \times 300 + 2400 = 98 + 6 + 2400 = 2504 \text{ ns}$$

## 例题4: 页表大小与多级页表设计

**题目**: 某系统虚拟地址40位, 页大小8KB, 物理地址36位, 页表项8B。

(1) 单级页表有多少项?
(2) 单级页表大小?
(3) 设计二级页表, 每级页表可存于一页中, 求各级索引位数。
(4) 二级页表需要常驻内存的部分大小?

**解**:

**(1)** 虚拟地址空间中页数:
$$\frac{2^{40}}{2^{13}} = 2^{27} \text{ 项}$$
(8KB = $2^{13}$B)

**(2)** 单级页表大小:
$$2^{27} \times 8B = 2^{30}B = 1\text{GB}$$

**(3)** 二级页表设计:
- 每页可存放页表项数: $2^{13} / 2^3 = 2^{10}$ = 1024项
- 每级索引位数: $\log_2 1024$ = 10位
- 二级共: 10 + 10 = 20位索引
- 页内偏移: $\log_2 8K$ = 13位
- 合计: 20 + 13 = 33位 < 40位

需要三级页表:
- $2^{27}$ 项, 每级1024项
- 三级: $2^{27} = 1024 \times 1024 \times 1024$
- 每级索引10位, 共30位 + 13位 = 43位 > 40位 ✓ (有余量)

实际划分(40位):
- 页内偏移: 13位
- 剩余: 27位分配给页表索引
- 三级各9位: 9 + 9 + 9 = 27位 ✓
- 每级 $2^9 = 512$ 项, 每级页表大小 = $512 \times 8B = 4KB$ = 一页 ✓

**(4)** 二级页表常驻部分:
- 一级页目录必须常驻: 4KB
- 二级和三级按需调入

## 例题5: 考虑写回策略的EAT

**题目**: 在例题2基础上, 假设缺页处理时, 30%的情况需要先将脏页写回磁盘(写回时间=8ms), 然后再调入新页(8ms)。

**解**:

缺页处理时间:
- 70%概率: 只需调入 = 8ms
- 30%概率: 写回+调入 = 16ms
- 平均缺页处理时间 = $0.7 \times 8 + 0.3 \times 16 = 5.6 + 4.8 = 10.4$ ms

$$EAT = 8.56 + 0.01 \times 10.4 \times 10^6 = 8.56 + 104000 = 104008.56 \text{ ns}$$

**结论**: 写回操作使缺页处理时间增加30%, EAT进一步恶化。
