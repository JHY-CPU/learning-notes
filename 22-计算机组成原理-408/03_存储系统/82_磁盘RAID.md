# 82_磁盘RAID

## 核心概念
- **RAID (Redundant Array of Independent Disks)**: 独立磁盘冗余阵列
- **目的**: 提高可靠性(冗余)和/或性能(并行)
- **基本思想**: 将多个磁盘组合起来, 对外表现为一个逻辑磁盘

### RAID级别总览

| 级别 | 名称 | 最少磁盘 | 容量 | 可靠性 | 性能 |
|------|------|----------|------|--------|------|
| RAID 0 | 条带化 | 2 | 100% | 无冗余 | 最高(读写) |
| RAID 1 | 镜像 | 2 | 50% | 最高 | 读快写慢 |
| RAID 5 | 分布式奇偶校验 | 3 | (N-1)/N | 容1盘故障 | 读快写慢 |
| RAID 6 | 双校验 | 4 | (N-2)/N | 容2盘故障 | 读快写更慢 |
| RAID 10 | 镜像+条带 | 4 | 50% | 高 | 高 |

## 原理分析

### RAID 0 (条带化 Striping)

```
数据块: A1 A2 A3 A4 A5 A6 A7 A8

磁盘0: A1 A3 A5 A7
磁盘1: A2 A4 A6 A8
```

**特点**:
- 数据被分割成条带, 分布到各磁盘
- **无冗余**: 任一磁盘损坏, 所有数据丢失
- **容量**: 所有磁盘容量之和 = $N \times C$
- **性能**: 读写都可并行, 速度最高
- **适用**: 对速度要求高, 不要求可靠性(如临时数据)

### RAID 1 (镜像 Mirroring)

```
数据块: A1 A2 A3 A4

磁盘0: A1 A2 A3 A4  ← 完整副本
磁盘1: A1 A2 A3 A4  ← 完整副本
```

**特点**:
- 每个数据块在两个磁盘上各存一份
- **可靠性最高**: 一块盘损坏, 数据仍在另一块
- **容量**: 总容量 = $N \times C / 2$ (利用率50%)
- **读性能**: 可从任一盘读, 读速度翻倍
- **写性能**: 需同时写两盘, 写速度不变
- **成本最高**: 用两倍磁盘存一份数据

### RAID 5 (分布式奇偶校验)

```
数据块: A1 A2 A3 A4

磁盘0: A1  A2  P1  ← P1 = A1 XOR A2
磁盘1: A3  P2  A5  ← P2 = A3 XOR A4
磁盘2: P3  A4  A6  ← P3 = A5 XOR A6

(P代表奇偶校验块, 分布在不同磁盘上)
```

**特点**:
- 奇偶校验块**分布式**存放(不在固定磁盘上)
- **容1盘故障**: 任何一块盘损坏, 可通过其他盘的奇偶校验恢复
- **容量**: $(N-1) \times C$
- **读性能**: 好(并行读)
- **写性能**: 较差(每次写需要读旧数据+旧校验, 计算新校验, 再写入)
  - **写惩罚**: 一次逻辑写 → 两次物理读 + 两次物理写
- **最常用**: 性能和可靠性的平衡

### RAID 6 (双校验)

```
磁盘0: A1  A2  P1  Q1
磁盘1: A3  P2  Q2  A5
磁盘2: P3  Q3  A4  A6
磁盘3: Q4  A7  A8  P4

(P和Q是两种不同的校验算法)
```

**特点**:
- 使用**两组**独立的校验算法
- **容2盘故障**: 同时损坏两块盘仍可恢复数据
- **容量**: $(N-2) \times C$
- **写惩罚更大**: 需更新两组校验
- **适用**: 对可靠性要求极高的场景

### 奇偶校验原理

**XOR(异或)校验**:
$$P = D_1 \oplus D_2 \oplus \cdots \oplus D_{n-1}$$

**恢复**: 若 $D_k$ 所在盘损坏:
$$D_k = P \oplus D_1 \oplus \cdots \oplus D_{k-1} \oplus D_{k+1} \oplus \cdots \oplus D_{n-1}$$

**例**: 3块盘, 数据 1010, 1100, 校验 = 0110
- 若第1盘损坏: 0110 XOR 1100 = 1010 ✓
- 若第2盘损坏: 0110 XOR 1010 = 1100 ✓

## 直观理解

**多人合作类比**:
- **RAID 0**: 把任务分成几份, 每人做一部分(快但一人出错全完)
- **RAID 1**: 每人都做全部任务(慢但一人出错不影响)
- **RAID 5**: 分工做, 但每人记一份汇总(校验), 一人出错可用汇总恢复
- **RAID 6**: 分工做, 记两份不同汇总, 容两人同时出错

### 记忆口诀
> **"0条带无冗余, 1镜像最安全; 5单校验容1盘, 6双校验容2盘"**

## 知识关联

### 408常考点
1. **RAID 0 vs RAID 1**: 容量、可靠性、性能对比
2. **RAID 5校验**: XOR计算, 数据恢复
3. **RAID级别选择**: 根据需求选择
4. **写惩罚**: RAID 5/6的写操作代价

### 各级适用场景
- **RAID 0**: 视频编辑、临时缓存(要速度, 不要安全)
- **RAID 1**: 操作系统盘、关键数据库(要安全, 不差钱)
- **RAID 5**: 文件服务器、一般数据存储(平衡选择)
- **RAID 6**: 大容量归档存储(大磁盘阵列, 可靠性要求高)
