# 26_低位交叉_计算详解

## 核心概念

本文件通过4道例题，详细展示低位交叉存储器的带宽和访问时间计算。

### 关键参数

- **$m$**：存储模块数
- **$T$**：每个模块的存取周期
- **$t$**：总线传输周期（数据从模块传到总线的时间）
- **理想条件**：$T = m \times t$（存取周期等于模块数乘以总线传输周期）

## 原理分析

### 例题1（基本带宽计算）

**题目**：4个存储模块低位交叉编址，每个模块存取周期 $T = 400\,\text{ns}$，总线传输周期 $t = 100\,\text{ns}$，数据宽度为64位。求：
(1) 理想情况下连续读取4个数据的时间
(2) 存储器带宽

**解答**：

(1) 检查条件：$T = 400\,\text{ns} = 4 \times 100\,\text{ns} = m \times t$，满足理想条件。

流水时间线（读取地址0、1、2、3）：
- 第0个$T$：模块0完成地址0的存取
- 第0个$T$ + $t$：模块0通过总线传输数据0，模块1完成地址1的存取
- ...
- 连续读取4个数据的总时间 = $T + (m-1) \times t = 400 + 3 \times 100 = 700\,\text{ns}$

或者理解为：启动流水需要 $T$ 时间，之后每个 $t$ 传一个数据，共 $m$ 个数据：

$$\text{总时间} = T + (m-1) \times t = 400 + 300 = 700\,\text{ns}$$

(2) 带宽：
$$B = \frac{m \times 64\,\text{bit}}{T + (m-1) \times t} = \frac{4 \times 64}{700 \times 10^{-9}} = \frac{256}{700 \times 10^{-9}} = 365.7\,\text{Mbit/s} \approx 45.7\,\text{MB/s}$$

**对比**：单模块带宽 = $64 / (400 \times 10^{-9}) = 160\,\text{Mbit/s} = 20\,\text{MB/s}$

提升倍数 ≈ $45.7 / 20 \approx 2.29$ 倍（针对4个数据的有限传输，不是理想的4倍）。

**对于大量连续数据传输**（流水线稳定后）：
$$B_{\text{稳定}} = \frac{64\,\text{bit}}{t} = \frac{64}{100 \times 10^{-9}} = 640\,\text{Mbit/s} = 80\,\text{MB/s}$$

这才是理想的4倍提升。

### 例题2（条件不满足的情况）

**题目**：4个模块低位交叉，$T = 300\,\text{ns}$，$t = 100\,\text{ns}$。求连续读取8个数据的时间。

**解答**：
- $T = 300\,\text{ns} < m \times t = 400\,\text{ns}$
- 条件不满足，流水线会断流

详细分析：
- 模块0在 $t=0$ 启动存取，$t=300$ 完成，$t=300\text{-}400$ 传输
- 模块1在 $t=100$ 启动存取，$t=400$ 完成，$t=400\text{-}500$ 传输
- 模块2在 $t=200$ 启动存取，$t=500$ 完成，$t=500\text{-}600$ 传输
- 模块3在 $t=300$ 启动存取，$t=600$ 完成，$t=600\text{-}700$ 传输
- 模块0在 $t=400$ 启动地址4的存取，$t=700$ 完成...

每个数据传输间隔为 $100\,\text{ns}$（$t$），所以8个数据的总时间 = $T + 7t = 300 + 700 = 1000\,\text{ns}$

即使 $T < mt$，流水仍然能工作（效率略有不同），因为模块启动可以重叠。

### 例题3（存取周期不等于模块数×传输周期）

**题目**：3个模块低位交叉，$T = 250\,\text{ns}$，$t = 100\,\text{ns}$。求连续读取10个数据的时间。

**解答**：
- 模块0：0~250ns存取，250~350ns传输
- 模块1：100~350ns存取，350~450ns传输
- 模块2：200~450ns存取，450~550ns传输
- 模块0（地址3）：300~550ns存取，550~650ns传输
- 模块1（地址4）：450~700ns存取，700~800ns传输

流水稳定后，传输间隔不完全是 $t$，因为 $T \neq m \times t$。

简化处理：10个数据分4轮（3+3+3+1），每轮间隔为 $T$（存取周期）。

$$\text{总时间} \approx T + (10-1) \times t = 250 + 900 = 1150\,\text{ns}$$

### 例题4（给定带宽求模块数）

**题目**：某存储器单模块存取周期 $T = 500\,\text{ns}$，总线传输周期 $t = 50\,\text{ns}$。若要使带宽提升为单模块的10倍，最少需要多少个模块？

**解答**：
- $T / t = 500 / 50 = 10$
- 理想条件下，带宽提升倍数 = 模块数
- 最少需要 **10个模块**

验证：$m = 10$，$T = 500 = 10 \times 50 = m \times t$，满足理想条件。

## 直观理解

### 带宽计算的两种情况

**情况1：传输大量数据（流水稳定后）**
$$B = \frac{\text{数据宽度}}{t}$$

**情况2：传输少量数据（含流水启动开销）**
$$B = \frac{m \times \text{数据宽度}}{T + (m-1) \times t}$$

考试通常考**情况1**（大量数据的稳定带宽）。

### 计算口诀

```
稳定带宽 = 数据宽度 ÷ 总线传输周期
启动时间 = 存取周期
流水间隔 = 总线传输周期
n个数据总时间 = T + (n-1)×t
```

## 知识关联

### 常考组合
- 带宽计算（计算题必考）
- 模块数的确定（给定带宽要求）
- 流水线启动时间的计算

### 易错陷阱

1. **$T$ 和 $t$ 不要搞混**：$T$ 是存取周期，$t$ 是总线传输周期，$T > t$
2. **带宽单位**：注意 bit/s 和 B/s 的换算
3. **稳定带宽和含启动开销的带宽不同**：题目要明确是哪种
4. **模块数 $m = T / t$ 是最优值**：大于或小于这个值都会降低效率
5. **$T = m \times t$ 是理想条件**：不满足时需要具体分析流水线时序
