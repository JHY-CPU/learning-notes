# 10_DRAM_刷新方式

## 核心概念

### 刷新的基本概念

- **刷新**：定期对DRAM的每一行进行重新读出并回写，防止电容漏电导致数据丢失
- **刷新周期**：完成所有行一轮刷新的最大允许间隔时间
  - 标准DRAM：**2ms**
  - 现代DRAM：**64ms**（取决于芯片规格）
- **刷新以行为单位**：一次刷新操作刷新一整行（所有列）
- **刷新由硬件自动完成**：刷新计数器+刷新控制器，不需要CPU参与

### 刷新的三种方式

| 刷新方式 | 死时间 | 速度影响 | 复杂度 |
|---------|--------|---------|--------|
| **集中刷新** | 有（集中一大段时间） | 刷新期间CPU不能访存 | 最简单 |
| **分散刷新** | 无（分散到每个周期） | 每个存取周期变长 | 简单 |
| **异步刷新** | 有（很小，分散在各处） | 影响最小 | 较复杂 |

## 原理分析

### 方式一：集中刷新（Burst Refresh）

**原理**：在刷新周期的**最后一段时间**，集中刷新所有行。

**举例**：
- DRAM有128行，存取周期为 $0.5\,\mu\text{s}$
- 刷新周期 = 2ms = 2000μs
- 2ms内可进行 $2000 / 0.5 = 4000$ 个存取周期
- 刷新128行需要 $128 \times 0.5 = 64\,\mu\text{s}$
- 死时间占比 = $64 / 2000 = 3.2\%$

**死时间（Dead Time）**：刷新期间CPU不能访问主存，这段时间叫"死时间"或"死区"。

**时间线**：

```
|←————— 正常读写（3872个周期） ——————→|←—— 刷新128行（128个周期） ——→|
|                                      |                              |
0                                    1936μs                       2000μs
```

**优缺点**：
- 优点：实现简单，大部分时间CPU可以正常访存
- 缺点：存在死时间，刷新期间CPU必须等待

### 方式二：分散刷新（Distributed Refresh）

**原理**：把刷新操作**分散到每个存取周期**中。

**关键**：每个存取周期被分为两部分：
- 前半部分：CPU读/写（正常访存）
- 后半部分：刷新一行

因此存取周期变为原来的 **2倍**。

**举例**：
- 原存取周期 = $0.5\,\mu\text{s}$
- 分散刷新后，每个存取周期 = $1.0\,\mu\text{s}$（0.5μs读写 + 0.5μs刷新）
- 128行 → 每 $128 \times 1.0 = 128\,\mu\text{s}$ 完成一轮刷新
- 远小于2ms的刷新周期要求

**优缺点**：
- 优点：**没有死时间**——刷新和读写交替进行，CPU总是能访问存储器
- 缺点：每个存取周期变长，**存储器速度变慢**

### 方式三：异步刷新（Asynchronous Refresh）

**原理**：在刷新周期内，将刷新操作**均匀分散**到各处，但不固定在每个存取周期。

**实现**：
- 刷新周期2ms，128行
- 每 $2\,\text{ms} / 128 = 15.625\,\mu\text{s}$ 刷新一行
- 可以利用CPU不访存的空闲时间进行刷新

**死时间**：每行刷新只需1个存取周期，死时间很小且分散。

**优缺点**：
- 优点：死时间小且分散，对CPU影响最小
- 缺点：实现较复杂

### 三种方式对比详解

**集中刷新**：死时间 = $n \times t_c$（n为行数，$t_c$为存取周期）
- 死时间集中，占比 = $n / (\text{刷新周期} / t_c)$

**分散刷新**：死时间 = 0
- 代价是存取周期翻倍

**异步刷新**：死时间 = $n \times t_c$（同集中刷新）
- 但分散在各处，每行只影响1个存取周期

## 直观理解

**打扫教室类比**：

- **集中刷新** = 白天所有人上课，下课后集中打扫所有教室。打扫期间不能进教室。
- **分散刷新** = 每节课后花一半时间打扫一间教室。没有"不让进教室"的时段，但每节课变长了。
- **异步刷新** = 每隔几分钟趁没人进出时打扫一间。影响最小，但调度最复杂。

### 刷新与访存冲突

刷新和CPU访存使用**同一组地址线和数据线**，所以两者不能同时进行：
- 集中刷新：冲突集中在最后一段时间
- 分散刷新：每半个周期冲突一次
- 异步刷新：冲突分散在小的时间片内

### 易错陷阱

1. **分散刷新没有死时间**——但代价是存取周期翻倍，整体变慢
2. **集中刷新的死时间是连续的一段时间**——期间CPU必须等待
3. **异步刷新也有死时间**——只是很小且分散
4. **刷新操作对CPU是透明的**——由硬件自动完成，但会影响CPU访存（冲突时CPU等待）
5. **刷新是按行进行的**——一次刷新操作刷新一整行的所有列
6. **刷新不需要CPU参与**——但可能占用CPU的访存时间
