# 76_磁盘调度算法_SCAN

## 核心概念
- **SCAN (扫描算法)**: 又称**电梯算法(Elevator Algorithm)**
- **原理**: 磁头沿一个方向移动, 依次服务沿途所有请求, 到达尽头后反向
- **特点**: 无饥饿, 效率高, 但两端的请求等待时间较长

## 原理分析

### 算法描述

```
1. 磁头从当前位置开始, 沿当前方向移动
2. 移动过程中, 服务所有经过的请求
3. 到达磁盘末端(最外道或最内道)
4. 反向移动, 继续服务沿途请求
5. 重复直到所有请求完成
```

### 扫描方向约定

通常假设有两种方向:
- **向外扫描**: 从小磁道号到大磁道号
- **向内扫描**: 从大磁道号到小磁道号

考试中需注意题目给定的初始方向!

### 示例

**题目**: 请求队列: 98, 183, 37, 122, 14, 124, 65, 67。磁头位置: 53, 当前方向: 向外(磁道号增大方向)。磁道范围: 0~199。

**解**:

向扫描(磁道号增大): 服务沿途 65, 67, 98, 122, 124, 183
到达199(或183后), 反向扫描(磁道号减小): 服务 37, 14

访问顺序: 53 → 65 → 67 → 98 → 122 → 124 → 183 → 37 → 14

```
磁道: 0    14 37     53  65 67   98     122 124    183  199
      |     |  |      |   |  |    |       |   |      |    |
      |     |  |      ●→  ●  ●    ●       ●   ●      ● ← |
      |     ●← ─────── |── |──|────|───────|───|────── |   |
      ●← ───|────────── |── |──|────|───────|───|────── |   |
```

**总移动距离**:
$$|65-53| + |67-65| + |98-67| + |122-98| + |124-122| + |183-124| + |183-37| + |37-14|$$
$$= 12 + 2 + 31 + 24 + 2 + 59 + 146 + 23 = 299$$

**注意**: 反转后从183到37, 需要经过199(磁盘末端):
$$\text{实际距离} = (199 - 183) + (199 - 37) + |37-14| = 16 + 162 + 23 = 201$$

更精确的计算:
向外: $53 \to 65 \to 67 \to 98 \to 122 \to 124 \to 183 \to 199$
距离 = $12+2+31+24+2+59+16 = 146$
向内: $199 \to 37 \to 14$
距离 = $162 + 23 = 185$

**总移动距离** = $146 + 185 = 331$ 磁道

**另一种理解** (扫描到最远请求即反向):
向外到183即反转, 不到199:
$53 \to 65 \to 67 \to 98 \to 122 \to 124 \to 183 \to 37 \to 14$
距离 = $12+2+31+24+2+59 + |183-37| + |37-14| = 130 + 146 + 23 = 299$

> **考试注意**: SCAN算法是否扫描到磁盘末端, 需要看题目具体要求。408考试通常假设扫描到最远请求即反向, 或扫描到末端。看清题意!

### SCAN的特点

**优点**:
- **无饥饿**: 所有请求最终都会被服务
- 效率优于FCFS
- 吞吐量大

**缺点**:
- **两端不公平**: 刚被磁头经过的请求(反方向的)等待时间很长
- 刚反向时, 刚被跳过的请求要等磁头走到另一端再回来

### SCAN vs SSTF

| 对比项 | SSTF | SCAN |
|--------|------|------|
| 策略 | 贪心(最近优先) | 单向扫描 |
| 饥饿 | 可能 | 不会 |
| 公平性 | 差(偏向中间) | 较好(但两端不公平) |
| 实现 | 简单 | 稍复杂 |
| 寻道距离 | 较短 | 稍长于SSTF |

## 直观理解

**电梯类比**:
- SCAN就是标准的电梯运行方式
- 电梯从1楼上到20楼, 中间各层都停; 到20楼后掉头往下
- 住在5楼的人: 电梯上行时经过了没停(因为你去的方向不对), 要等电梯到20楼再下来
- 这就是"两端等待时间长"的含义

### 记忆口诀
> **"SCAN像电梯, 单向服务到头反转; 无饥饿, 但两端等得久"**

### 常见陷阱
- 忘记确定初始扫描方向
- 是否扫描到物理末端(199)还是到最远请求(183)
- 反向后也要服务沿途的请求, 不能跳过
