# 56_页表结构

## 核心概念

- **页表基地址寄存器（PTBR, Page Table Base Register）**：存放当前进程页表在主存中的起始地址，CPU通过PTBR找到页表。
- **页表项内容**：物理页号 + 控制位（存在位、修改位、访问位、权限位等）。
- **多级页表**：将单级页表拆分成多级，减少页表占用的连续内存空间。
- **倒排页表**：以物理页号为索引，减少页表总大小，但查找复杂。
- **408考点**：页表大小计算、多级页表的原理和优缺点。

## 原理分析

### 页表基地址寄存器（PTBR）

- 每个进程有独立的页表，PTBR指向当前活跃进程的页表
- 进程切换时，OS将新进程的页表基地址写入PTBR
- PTBR是CPU中的特殊寄存器，访问速度极快

### 单级页表的问题

**示例**：虚拟地址32位，页面4KB，PTE占4字节

- 虚页数：$2^{32}/2^{12} = 2^{20} = 1\text{M}$ 页
- 页表大小：$2^{20} \times 4\text{B} = 4\text{MB}$

**问题**：
- 需要4MB**连续**内存存放页表
- 大多数虚拟地址空间是稀疏的（代码段+数据段+栈，中间大片空白）
- 浪费大量空间存储无效的页表项

### 多级页表

**思想**：将页表本身也分页，建立"页表的页表"。

**二级页表**：

```
虚拟地址:  [一级页号 | 二级页号 | 页内偏移]
```

**转换过程**：
1. 用一级页号查**一级页表**（页目录），得到二级页表的基地址
2. 用二级页号查**二级页表**，得到物理页号
3. 物理页号 + 偏移 = 物理地址

**示例**（32位系统，4KB页面）：

- 偏移：12位
- 将虚页号20位分为：一级10位 + 二级10位
- 一级页表（页目录）：$2^{10} = 1024$ 项，每项4B，共4KB（刚好一页）
- 二级页表：最多1024个，每个4KB
- **实际占用**：页目录4KB + 实际需要的二级页表（远小于4MB）

**优势**：
- 不需要连续4MB内存
- 只需为实际使用的虚拟地址空间分配二级页表
- 稀疏地址空间大幅节省内存

**64位系统的多级页表**：

通常使用4级页表（PGD→PUD→PMD→PTE）：
- 虚拟地址48位有效位，偏移12位
- 剩余36位分为4级，每级9位（$2^9 = 512$ 项）

### 倒排页表

**思想**：不以虚页号为索引，而以**物理页号**为索引。

| 物理页号 | 对应的虚页号 | 进程ID |
|---------|------------|--------|
| 0 | VPN: 100 | PID: 1 |
| 1 | VPN: 5 | PID: 2 |
| ... | ... | ... |

**优点**：
- 页表大小 = 物理页数（与虚拟空间大小无关）
- 节省空间，适合64位大虚拟空间

**缺点**：
- 查找时需要搜索整个表（或用Hash加速）
- 实现复杂

## 直观理解

**单级页表**：一本巨大的电话簿，从头到尾一页不落。

**多级页表**：先查"城市目录"（一级），找到城市后再查该城市的电话簿（二级）。空白城市不需要电话簿。

**倒排页表**：反向电话簿——以电话号码为索引，查是谁的号码。

## 知识关联

### 与Cache/TLB的联系

- TLB缓存页表项（PTE），减少查页表的次数
- 多级页表的每一级查表都可能产生TLB缺失
- 倒排页表需要Hash查找，不适合用TLB直接缓存

### 408考点

- 单级页表大小计算
- 多级页表的级数划分和每级大小
- PTBR的作用
- 多级页表的优点（减少连续内存占用）和缺点（增加查表次数）

### 易错陷阱

- 多级页表虽然减少了内存占用，但增加了访存次数（每级一次）
- TLB可以缓解多级页表的多次访存问题
- 64位系统不用单级页表（虚页数太大）
- 倒排页表在408中较少考查，了解概念即可
