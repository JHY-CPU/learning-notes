# 91_存储系统_408真题综合3

## 综合真题1: 存储器扩展 (2011年408)

**题目**: 用若干片 4K×8位 的SRAM芯片, 组成一个 16K×16位 的存储器。

(1) 需要多少片SRAM芯片?
(2) 如何连接(位扩展+字扩展)?
(3) 各芯片的地址线、数据线如何连接?

### 解答

**(1) 芯片数量**:
- 字扩展: $16K / 4K = 4$ 组
- 位扩展: $16 / 8 = 2$ 片/组
- 总芯片数 = $4 \times 2 = 8$ 片

**(2) 连接方式**:

**位扩展** (2片一组, 扩展数据位):
```
芯片A(低8位) + 芯片B(高8位) → 4K×16位
```
- 地址线并联(共享12位地址)
- 数据线各占8位(拼成16位)
- 片选信号并联

**字扩展** (4组, 扩展地址空间):
```
第0组: 0x0000~0x0FFF
第1组: 0x1000~0x1FFF
第2组: 0x2000~0x2FFF
第3组: 0x3000~0x3FFF
```
- 地址总线低12位: 4组共享(组内地址)
- 地址总线高2位: 2-4译码器选组(片选)

**(3) 引脚连接**:

| 信号 | 连接方式 |
|------|----------|
| 地址线A0~A11 | 8片共享(组内寻址) |
| 地址线A12~A13 | 经2-4译码器产生4个片选 |
| 数据线D0~D7 | 各组的低8位芯片 |
| 数据线D8~D15 | 各组的高8位芯片 |
| 读写信号WE | 8片共享 |
| 片选CS | 每组一片(译码器输出) |

## 综合真题2: 交叉存取 (2015年408改编)

**题目**: 某系统采用4体交叉存储器, 存储周期200ns, 总线传输周期50ns。

(1) 采用低位交叉编址, 连续读取16个存储字需要多长时间?
(2) 与单体存储器相比加速比是多少?
(3) 高位交叉编址和低位交叉编址有什么区别?

### 解答

**(1) 低位交叉, 16个存储字**:

低位交叉编址:
- 地址 mod 4 = 0 → 体0
- 地址 mod 4 = 1 → 体1
- 地址 mod 4 = 2 → 体2
- 地址 mod 4 = 3 → 体3

流水线启动:
```
时间:  0    50   100  150  200  250  ...
体0:  [第0字启动]          [第0字完成,第4字启动]
体1:       [第1字启动]          [第1字完成,第5字启动]
体2:            [第2字启动]          [第2字完成,第6字启动]
体3:                 [第3字启动]          [第3字完成,第7字启动]
```

- 启动间隔 = 50ns (总线传输周期)
- 第1个字: 从启动到完成 = 200ns (存储周期)
- 之后每50ns出一个字
- 16个字的时间 = $200 + (16-1) \times 50 = 200 + 750 = 950$ ns

**(2) 加速比**:
- 单体: $16 \times 200 = 3200$ ns
- 交叉: 950 ns
- 加速比 = $3200 / 950 = 3.37$

理论最大加速比 = 4 (4体交叉, 启动间隔 = 存储周期/4 = 200/50 = 4)

**(3) 高位 vs 低位交叉**:

| 对比项 | 高位交叉 | 低位交叉 |
|--------|----------|----------|
| 地址分配 | 高位选体, 低位选字 | 低位选体, 高位选字 |
| 连续地址 | 在同一存储体 | 分布在不同体 |
| 并行性 | 无(单体操作) | 有(多体并行流水) |
| 用途 | 扩展容量 | 提高带宽 |
| 别名 | 顺序存储器 | 交叉存储器 |

## 综合真题3: 存储器扩展+时序 (2017年408改编)

**题目**: 某8位CPU, 地址总线16位(A15~A0), 数据总线8位。现有以下芯片:
- 8K×8位 EPROM: 需要2片(地址0x0000~0x3FFF)
- 4K×8位 RAM: 需要2片(地址0x4000~0x4FFF各一片太小, 实际需要扩展)

简化题目: 用2片8K×8位EPROM和2片4K×8位RAM, 设计地址分配使:
- EPROM: 0x0000~0x3FFF (16KB)
- RAM: 0x4000~0x5FFF (8KB)

**解**:

EPROM(2片8K×8位, 字扩展):
- 每片8KB, 2片共16KB = 0x0000~0x3FFF
- 地址A12~A0: 片内寻址(13位=8K)
- A13: 选片(0→EPROM0, 1→EPROM1)
- A15~A14 = 00: EPROM区

RAM(2片4K×8位, 字扩展):
- 每片4KB, 2片共8KB = 0x4000~0x5FFF
- 地址A11~A0: 片内寻址(12位=4K)
- A12: 选片(0→RAM0, 1→RAM1)
- A15~A13 = 010: RAM区

片选逻辑:
- EPROM CS: $\overline{A_{15}} \cdot \overline{A_{14}}$
- RAM CS: $\overline{A_{15}} \cdot A_{14} \cdot \overline{A_{13}}$

## 综合真题4: DRAM刷新+访问冲突 (2020年408)

**题目**: 某DRAM芯片16K×1位, 内部排列为128×128矩阵, 存取周期0.5μs, 刷新周期2ms。

(1) 采用集中式刷新, 每次刷新一行, 刷新期间CPU不能访问, 刷新占用多少时间?
(2) 采用分散式刷新, 每个存取周期刷新一行, 刷新对CPU的影响是什么?

### 解答

**(1) 集中式刷新**:
- 需要刷新128行
- 每行刷新时间 = 0.5μs (占用一个存取周期)
- 总刷新时间 = $128 \times 0.5\mu s = 64\mu s$
- 刷新周期 = 2ms
- CPU不能访问时间比例 = $64 / 2000 = 3.2\%$

**(2) 分散式刷新**:
- 每个存取周期内嵌一个刷新操作
- 实际存取周期变为 $0.5 + 0.5 = 1.0\mu s$ (慢了一倍)
- 2ms内可完成 $2000 / 1.0 = 2000$ 次操作, 远超128行
- 对CPU影响: 每次访存变慢(存取周期翻倍), 但不存在集中不可访问期

**对比**:
| 方式 | 刷新时间 | CPU影响 |
|------|----------|---------|
| 集中 | 64μs/2ms集中 | 集中期完全不能访问(死区) |
| 分散 | 每个周期0.5μs | 每次访存变慢, 但无死区 |
