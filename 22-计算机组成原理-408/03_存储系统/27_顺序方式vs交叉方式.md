# 27_顺序方式vs交叉方式

## 核心概念

多模块存储器有**顺序方式**和**交叉方式**两种组织形式，对应高位交叉和低位交叉编址。

### 两种方式的对比

| 对比项 | 顺序方式（高位交叉） | 交叉方式（低位交叉） |
|--------|-------------------|-------------------|
| **编址方式** | 高位做模块号 | 低位做模块号 |
| **连续地址** | 在同一模块内 | 在不同模块间 |
| **并行访问** | 不支持 | 支持（流水访问） |
| **带宽** | = 单模块带宽 | ≈ $m$ × 单模块带宽 |
| **访问特点** | 模块独立工作 | 模块流水协同 |
| **主要目的** | 扩大容量 | 提高带宽 |
| **扩展难度** | 简单 | 较复杂 |

## 原理分析

### 顺序方式的工作过程

4个模块，每模块4K，顺序编址：

```
模块0：地址 0000~4095
模块1：地址 4096~8191
模块2：地址 8192~12287
模块3：地址 12288~16383
```

CPU顺序访问地址0、1、2、3...时，全部都在模块0中。模块1、2、3闲置。

**时间线**：
```
模块0：[访问0][访问1][访问2][访问3]...[访问4095][访问4096→模块1]
模块1：  [闲置 ][闲置 ][闲置 ][闲置 ]...[闲置    ][访问4096]
模块2：  [闲置 ]...
模块3：  [闲置 ]...
```

**带宽**：与单模块相同，因为任何时候只有一个模块在工作。

### 交叉方式的工作过程

4个模块，低位交叉编址：

```
模块0：地址 0, 4, 8, 12, ...
模块1：地址 1, 5, 9, 13, ...
模块2：地址 2, 6, 10, 14, ...
模块3：地址 3, 7, 11, 15, ...
```

CPU顺序访问地址0、1、2、3时，分别访问模块0、1、2、3。

**时间线**（$T=400\,\text{ns}$, $t=100\,\text{ns}$）：
```
         0    100  200  300  400  500  600  700 (ns)
模块0：  [存取0     ][传输0]
模块1：     [存取1     ][传输1]
模块2：        [存取2     ][传输2]
模块3：           [存取3     ][传输3]
总线：                     [D0  ][D1  ][D2  ][D3  ]
```

**带宽**：约为单模块的4倍。

### 为什么顺序方式不能并行？

因为局部性原理——程序倾向于访问连续地址。在顺序编址中，连续地址在同一模块，导致：
- 一个模块忙死，其他模块闲死
- 无法利用多模块的并行能力

### 什么时候用顺序方式？

- 主要目的是**扩大容量**而非提高带宽
- 访问模式**不连续**（随机访问为主）
- 系统设计简单优先
- 实际应用：主存储器的字扩展

### 什么时候用交叉方式？

- 主要目的是**提高带宽**
- 访问模式**连续**（顺序访问为主）
- 需要高吞吐率
- 实际应用：Cache与主存之间的数据传送、DMA传输

## 直观理解

**排队买票类比**：

**顺序方式** = 4个售票窗口各有自己的队列
- 买完票的人如果还想买下一场，必须排在同一条队
- 其他队窗口可能空闲

**交叉方式** = 4个售票窗口轮流服务同一条队
- 每个窗口处理一个顾客
- 顾客总是被最快空闲的窗口服务
- 效率最高

### 易错陷阱

1. **顺序方式不意味着"轮流使用"模块**——模块是独立的，由地址决定访问哪个
2. **交叉方式的并行性来自流水**——不是同时访问（总线只有一条）
3. **顺序方式扩展容量更简单**——类似字扩展
4. **交叉方式需要更复杂的控制逻辑**——地址译码和流水调度
5. **实际计算机中两种方式可能混合使用**——不同层次采用不同方式
