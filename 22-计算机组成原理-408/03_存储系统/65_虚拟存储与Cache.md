# 65_虚拟存储与Cache

## 核心概念
- **Cache-主存层次**: 解决CPU与主存的速度差距, 以块(Block)为单位
- **主存-辅存层次**: 解决主存容量不足, 以页(Page)为单位
- **TLB**: 页表的Cache, 加速地址转换
- 两套机制在结构上高度类比

### 三级存储体系

```
CPU ←→ TLB ←→ 页表(主存) ←→ 辅存(磁盘)
        ↓          ↓
      Cache ←→  主存 ←→ 辅存
```

## 原理分析

### 结构类比

| 对比项 | Cache-主存 | 主存-辅存(虚拟存储) |
|--------|-----------|-------------------|
| **管理对象** | 块(Block) | 页(Page) |
| **典型大小** | 64B~256B | 4KB~64KB |
| **管理主体** | 硬件 | OS + 硬件 |
| **对程序员** | 透明 | 透明(用户不可见) |
| **缺失代价** | 几十个周期 | 几百万个周期(磁盘I/O) |
| **目的** | 提高速度 | 扩大容量 |
| **映射方式** | 直接/全相联/组相联 | 全相联(页可放任意页框) |
| **替换算法** | LRU/FIFO/Random | OPT/FIFO/LRU/Clock |
| **写策略** | 写回/写直达 | 写回(只写辅存在换出时) |
| **命中率影响** | 性能 | 正确性(不命中必须处理) |

### TLB与Cache的类比

| TLB | Cache | 类比关系 |
|-----|-------|----------|
| TLB项 | Cache行 | 都是高速缓存项 |
| VPN(虚页号) | Tag(标记) | 用于查找匹配 |
| PPN(物理页号) | 块内数据 | 实际需要的内容 |
| TLB命中 | Cache命中 | 快速获取 |
| TLB未命中→查页表 | Cache未命中→访主存 | 回退到下一级 |
| 页表 | 主存 | 下一级存储 |

### 映射方式对比

**Cache映射** (前面章节):
- 直接映射: 每块只能放固定位置
- 全相联: 每块可放任意位置
- 组相联: 每块可放固定组内任意位置

**虚存映射**:
- 页式: **全相联**——每个虚页可映射到任意物理页框
  - 因为辅存缺失代价极高, 必须充分利用主存空间
  - 不能用直接映射(会浪费页框)

### 替换算法对比

| 算法 | Cache | 虚拟存储 |
|------|-------|----------|
| OPT | 理论最优 | 理论最优(不可实现) |
| FIFO | 简单实现 | Belady异常存在 |
| LRU | 常用(硬件实现) | 常用(近似实现) |
| Clock/NRU | 组相联常用 | 虚存最常用(近似LRU) |
| Random | 低开销方案 | - |

**区别**: Cache替换代价小, 可用精确LRU; 虚存替换代价大, 多用近似算法(Clock)

### 写策略对比

**Cache写策略**:
- **写直达(Write-Through)**: 同时写Cache和主存, 需要写缓冲
- **写回(Write-Back)**: 只写Cache, 被替换时才写回主存, 需脏位

**虚存写策略**:
- 始终采用**写回**: 写操作只修改主存中的页
- 辅存中的页在被替换时才写回(脏页)
- 因为辅存I/O代价极高, 写直达不可行

## 直观理解

### 快递仓库类比

**Cache-主存** = 快递点仓库:
- 块 = 小包裹
- Cache = 货架(取货快)
- 主存 = 仓库后仓(取货慢)
- 缺失 = 货架上没有, 去后仓找

**主存-辅存** = 城市快递网络:
- 页 = 大箱子
- 主存 = 本市仓库(小但快)
- 辅存 = 外地总仓(大但慢)
- 缺页 = 本市仓库没有, 需从外地调货(几天时间)

**TLB** = 你手机里的订单记录:
- 有记录(TLB命中)→ 直接知道在哪
- 没记录(TLB未命中)→ 得查系统(页表)

### 关键区别记忆

> **Cache管速度, 虚存管容量; Cache靠硬件, 虚存靠OS; Cache块小, 虚存页大; Cache缺失可忍, 缺页必须处理"**

## 知识关联

### 跨章节联系
- **操作系统**: 虚拟存储管理是OS核心功能之一
- **计组第2章**: Cache是硬件实现的透明缓存
- **计组第4章**: 缺页中断是异常处理的一种

### 两者的协同工作
1. CPU发出虚地址
2. TLB查VPN→PPN(类比Cache查Tag)
3. 若TLB未命中, 查页表(类比Cache未命中查主存)
4. 得到物理地址后, Cache用物理地址查数据
5. 若Cache未命中, 从主存取块到Cache

### 常考对比选择题

**Q: 以下哪些说法正确?**
- A. Cache和虚存都对程序员透明 → 错(虚存对应用程序透明, 但OS可见)
- B. Cache和虚存都使用全相联映射 → 错(Cache常用组相联, 虚存用全相联)
- C. Cache和虚存缺失都需要从下一级调入 → 对
- D. Cache和虚存都采用LRU替换 → 对(但虚存用近似LRU)

### 设计哲学对比
- **Cache**: 缺失代价相对低(几十ns vs 几ns), 可以容忍一定缺失率
- **虚存**: 缺失代价极高(磁盘ms级), 必须尽量减少缺页率
- 因此Cache追求命中率99%+, 虚存追求更低的缺页率
