# 30_地址映射_全相联映射

## 核心概念

### 全相联映射的定义

**全相联映射（Fully Associative Mapping）**：主存中的每个块可以放到Cache中的**任意一行**。

### 与直接映射的对比

| 对比项 | 直接映射 | 全相联映射 |
|--------|---------|-----------|
| **映射规则** | 块号 mod 行数 → 指定行 | 任意空行都行 |
| **地址结构** | 标记 + 行号 + 偏移 | 标记 + 偏移 |
| **查找方式** | 定位+比较（一次比较） | 所有行同时比较 |
| **冲突率** | 高 | **低** |
| **查找速度** | **快** | 慢（需并行比较） |
| **硬件开销** | 小 | **大**（需要多路比较器） |
| **替换策略** | 不需要 | **需要**（LRU等） |

### 地址结构

$$\text{主存地址} = \underbrace{\text{标记(Tag)}}_{\text{主存块号}} + \underbrace{\text{块内偏移(Offset)}}_{\text{块内位置}}$$

**注意**：全相联映射**没有行号字段**！因为块可以放在任意行，不需要指定行号。

**各字段位数**：
- 块内偏移 = $\log_2(\text{块大小})$ 位
- 标记 = 总地址位数 - 块内偏移位数 = $\log_2(\text{主存块数})$ 位

## 原理分析

### 地址结构计算示例

**条件**：主存64KB，Cache 2KB，块大小 64B，全相联映射。

- 主存地址：16位
- 块内偏移：$\log_2(64) = 6$ 位
- 标记：$16 - 6 = 10$ 位（等于主存块号的位数）
- 主存块数：$64\text{K} / 64 = 1024$ 块（$2^{10}$，10位块号）
- Cache行数：$2\text{K} / 64 = 32$ 行

$$\text{地址} = \underbrace{10\,\text{位}}_{\text{标记}} + \underbrace{6\,\text{位}}_{\text{偏移}}$$

### 查找算法

全相联映射的查找需要**同时比较所有行的标记**：

```
CPU地址：|  标记(10位)  |  偏移(6位)  |
                │              │
                │              │
    ┌───────────┤              │
    │           │              │
    ▼           │              │
  行0标记 =? ───┤              │
  行1标记 =? ───┤              │
  行2标记 =? ───┤   （同时比较） │
    ...         │              │
  行31标记 =? ──┤              │
    │           │              │
    ▼           │              │
  命中行号 ─────┘              │
    │                          │
    ▼                          ▼
  读取该行数据的指定偏移位置
```

**关键硬件**：需要 **32个比较器**（Cache行数个）**并行工作**，同时比较所有行的标记。

### 替换策略

全相联映射**必须有替换策略**，因为块可以放任意行，当Cache满了需要选择替换哪个：

| 策略 | 说明 | 实现复杂度 |
|------|------|-----------|
| **LRU**（最近最少使用） | 替换最久没被访问的行 | 中等 |
| **FIFO**（先进先出） | 替换最早调入的行 | 简单 |
| **LFU**（最不经常使用） | 替换访问次数最少的行 | 复杂 |
| **随机** | 随机选择一行替换 | 最简单 |

### 优点

- **冲突率最低**：块可以放任何空行，不会出现直接映射的"抖动"
- **Cache利用率高**：只要有空行就不会发生冲突
- **命中率通常最高**（给定容量下）

### 缺点

- **查找慢**：需要同时比较所有行的标记
- **硬件开销大**：需要大量比较器和优先编码器
- **替换策略复杂**：需要记录访问顺序/次数

### 实际应用

全相联映射因为查找开销大，**很少单独用于主Cache**。但广泛用于：
- **TLB（Translation Lookaside Buffer）**：页表缓存，通常只有64~256项
- **小容量Cache**：行数少时，全相联的查找开销可接受

## 直观理解

**停车场类比**：
- **直接映射** = 指定车位（车牌号 mod 32 = 车位号），车只能停指定车位
- **全相联** = 任意车位，哪里空停哪里

**全相联的代价**：
- 保安需要检查所有32个车位才能找到你的车（并行比较）
- 还要记录每辆车的停放时间以便替换（LRU）

**为什么全相联冲突率低？**
- 直接映射：块0和块32必须竞争Cache行0——即使其他31行都空着
- 全相联：块0和块32可以放在任意空行——根本不会冲突

| 特性 | 直接映射 | 全相联 | 组相联 |
|------|---------|--------|--------|
| 映射位置 | 唯一指定行 | 任意行 | 组内任意行 |
| 地址字段 | Tag+行号+偏移 | Tag+偏移 | Tag+组号+偏移 |
| 查找复杂度 | 低 | 高 | 中 |
| 冲突率 | 高 | 低 | 中 |
| 典型应用 | L1 Cache | TLB | L2/L3 Cache |

### 易错陷阱

1. **全相联地址中没有"行号"字段**——这是与直接映射的关键区别
2. **标记 = 主存块号**——标记的位数等于主存块号的位数
3. **全相联必须有替换策略**——直接映射不需要（映射关系固定）
4. **全相联的比较器数量 = Cache行数**——硬件开销与行数成正比
5. **全相联不是"万能的"**——查找开销限制了Cache的行数，大容量Cache不用全相联
6. **组相联映射是直接映射和全相联的折中**——考试常考三种方式的对比
