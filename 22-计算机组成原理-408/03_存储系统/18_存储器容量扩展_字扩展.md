# 18_存储器容量扩展_字扩展

## 核心概念

### 字扩展的定义

**字扩展（Word Expansion）**：用多片**字长相同但存储单元数较少**的芯片，通过**片选信号**区分，增加存储器的**存储单元数（地址空间）**。

### 字扩展的核心思想

- **字长不变**，存储单元数增加
- 每片芯片占据地址空间的不同区域
- 通过**高位地址译码**产生片选信号
- 任意时刻只有**一片芯片被选中**

### 适用场景

当目标存储单元数大于单片存储单元数时，需要字扩展。

$$\text{字扩展需要的芯片数} = \frac{\text{目标存储单元数}}{\text{单片存储单元数}}$$

## 原理分析

### 例题：$4\text{K} \times 8$ → $16\text{K} \times 8$

**需求**：用 $4\text{K} \times 8$ 位的芯片组成 $16\text{K} \times 8$ 位的存储器。

**分析**：
- 字长不变（8位），存储单元数从4K扩展到16K
- 需要 $16\text{K} / 4\text{K} = 4$ 片
- $4\text{K} = 2^{12}$，芯片需要12根地址线（A0~A11）
- $16\text{K} = 2^{14}$，系统需要14根地址线（A0~A13）

**地址分配**：

| 芯片 | 地址范围（二进制） | 地址范围（十六进制） | 片选条件 |
|------|-------------------|---------------------|---------|
| 芯片0 | 00 000000000000 ~ 00 111111111111 | 0000H ~ 0FFFH | A13A12 = 00 |
| 芯片1 | 01 000000000000 ~ 01 111111111111 | 1000H ~ 1FFFH | A13A12 = 01 |
| 芯片2 | 10 000000000000 ~ 10 111111111111 | 2000H ~ 2FFFH | A13A12 = 10 |
| 芯片3 | 11 000000000000 ~ 11 111111111111 | 3000H ~ 3FFFH | A13A12 = 11 |

**连接方式**：

```
                    地址总线（14位：A0~A13）
                         │
              A0~A11（低位地址）──┬──────┬──────┬──────┐
                                │      │      │      │
                           ┌────┴──┐┌──┴───┐┌─┴───┐┌─┴────┐
                           │芯片0  ││芯片1 ││芯片2││芯片3 │
                           │4K×8   ││4K×8  ││4K×8 ││4K×8  │
                           └────┬──┘└──┬───┘└──┬──┘└──┬───┘
                    D0~D7 ←─────┴──────┴───────┴──────┘
                           （数据线并联到数据总线）

A12,A13 ──→ 2:4译码器 ──→ CE0, CE1, CE2, CE3（片选信号）
```

**连接规则**：
1. **低位地址线并联**：A0~A11 接所有芯片的地址引脚
2. **高位地址译码产生片选**：A12、A13 送入2:4译码器，输出4个片选信号
3. **数据线并联**：所有芯片的数据线 D0~D7 连接到同一组数据总线
4. **读写控制并联**：WE信号接所有芯片

### 片选信号的产生方式

**方式1：线选法**（当芯片数较少时）
- 每根高位地址线直接控制一片芯片
- 简单但浪费地址空间

**方式2：译码法**（常用）
- 用译码器（如74LS138）将高位地址翻译为片选信号
- 地址空间利用率高

### 字扩展的关键特点

| 特征 | 说明 |
|------|------|
| 存储单元数 | 增加 |
| 字长 | 不变 |
| 地址线 | 低位共用，高位做片选 |
| 数据线 | 并联（所有芯片接同一组数据线） |
| 片选信号 | 各芯片独立（同一时刻只有一片被选中） |
| 芯片数量 | $= \text{目标存储单元数} / \text{单片存储单元数}$ |

## 直观理解

**多个储物柜类比**：
- 位扩展 = 把几个小柜子合并成一个大柜子（空间更大）
- 字扩展 = 增加一排柜子（数量更多）
- 每排柜子共用一个走廊（数据总线）
- 片选 = "请3号柜组的顾客来取件"（只有一组响应）

### 易错陷阱

1. **字扩展增加的是存储单元数，不是字长**
2. **数据线是并联的**（不是拼接）——与位扩展不同！
3. **片选信号是独立的**（不是共用）——与位扩展不同！
4. **同一时刻只有一片芯片工作**——通过片选保证
5. **高位地址做片选，低位地址接芯片**——位不要搞反
6. **地址空间是连续分配的**——每片芯片占据连续的地址范围
