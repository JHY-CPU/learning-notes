# 61_页式地址转换_408考点

## 核心概念
- **408大题经典模型**: TLB + Cache + 主存三级存储体系的综合访问流程
- **虚拟地址划分**: 虚页号(VPN) + 页内偏移(Offset)
- **物理地址划分**: 物理页号(PPN) + 页内偏移(Offset)
- **TLB**: Translation Lookaside Buffer, 页表的高速缓存, 存放VPN→PPN映射
- **Cache**: 存放主存数据块的副本, 标记(Tag) + 索引(Index) + 偏移

### 考试重点
- 给定虚拟地址, 求物理地址和Cache行号
- 三类命中情况的区分: TLB命中/缺页, Cache命中/缺失
- 访问次数的统计(主存实际访问几次)

### 地址转换完整流程

```
虚拟地址: | VPN(虚页号) | Offset(页内偏移) |
                |                    |
                v                    |
          查找TLB                    |
         /       \                   |
      命中       未命中              |
       |          |                  |
       v          v                  |
    得到PPN   查页表(访问主存1次)     |
       |       /     \               |
       |    有效     缺页             |
       |     |        |              |
       |     v        v              |
       |  得到PPN   缺页中断处理      |
       |     |                       |
       v     v                       |
  物理地址: | PPN | Offset | <--------+
                |
                v
          查找Cache
         /        \
      命中        未命中
       |           |
       v           v
    CPU获得数据   主存调块到Cache(访问主存)
```

## 原理分析

### 关键公式

**虚地址空间与页大小**:
$$\text{虚页号位数} = \log_2(\text{虚地址空间} / \text{页大小})$$
$$\text{页内偏移位数} = \log_2(\text{页大小})$$

**物理地址空间与页框**:
$$\text{物理页号位数} = \log_2(\text{物理地址空间} / \text{页大小})$$

**Cache索引与标记**:
$$\text{Cache行数} = \text{Cache总容量} / \text{块大小}$$
$$\text{直接映射索引位数} = \log_2(\text{行数})$$

### 综合访问时间计算

**情况1: TLB命中 + Cache命中** (最理想):
$$T_{\text{总}} = T_{\text{TLB}} + T_{\text{Cache}}$$

**情况2: TLB未命中 + Cache命中**:
$$T_{\text{总}} = T_{\text{TLB}} + T_{\text{页表}} + T_{\text{Cache}}$$
(查TLB未命中, 需要访问主存查页表)

**情况3: TLB命中 + Cache未命中**:
$$T_{\text{总}} = T_{\text{TLB}} + T_{\text{主存}}$$
(通过TLB得到物理地址, 从主存取数据)

**情况4: TLB未命中 + Cache未命中**:
$$T_{\text{总}} = T_{\text{TLB}} + T_{\text{页表}} + T_{\text{主存}}$$

**综合有效访问时间** (设TLB命中率$h_t$, Cache命中率$h_c$):
$$EAT = h_t \cdot h_c \cdot (t_{\text{TLB}} + t_c) + h_t(1-h_c)(t_{\text{TLB}} + t_m) + (1-h_t)h_c(t_{\text{TLB}} + t_m + t_c) + (1-h_t)(1-h_c)(t_{\text{TLB}} + 2t_m)$$

其中 $t_m$ 为主存访问时间, $t_c$ 为Cache访问时间, 页表存于主存。

### 典型例题1: 完整地址转换

**题目**: 某系统虚拟地址32位, 页大小4KB, 物理地址30位, TLB 64项全相联, Cache容量16KB, 块大小64B, 直接映射。给定虚拟地址 `0x00403018`, 求物理地址和Cache行号。

**解**:

**(1) 分析地址结构**:
- 页大小 = 4KB = $2^{12}$B, 页内偏移12位
- 虚页号 = 32 - 12 = 20位
- 物理页号 = 30 - 12 = 18位
- Cache行数 = 16KB / 64B = 256 = $2^8$, 索引8位
- Cache块内偏移 = $\log_2 64$ = 6位

**(2) 虚地址分解**:
虚拟地址 `0x00403018` = `0000 0000 0100 0000 0011 0000 0001 1000`
- 虚页号 VPN = `0x00403` (高20位)
- 页内偏移 Offset = `0x018` (低12位)

**(3) TLB查找**:
查TLB中VPN=`0x00403`对应的表项, 假设命中, PPN = `0x12345`

**(4) 构造物理地址**:
物理地址 = PPN + Offset = `0x12345018`
= `0001 0010 0011 0100 0101 0000 0001 1000`

**(5) Cache行号计算**:
物理地址低6位为块内偏移, 接下来8位为Cache索引:
- 块内偏移 = `0x018` & `0x3F` = `0x18` (低6位)
- Cache索引 = (`0x018` >> 6) & `0xFF` = `0x00`
- 或: 从完整物理地址, 位[11:6] = Cache索引

块地址 = 物理地址 / 块大小 = `0x12345018` / 64
Cache行号 = 块地址 mod Cache行数 = ... (需具体计算)

### 典型例题2: 访问次数统计

**题目**: TLB命中率95%, Cache命中率98%, TLB查表时间忽略, 主存访问时间100ns, Cache访问时间10ns。求平均访问时间。

**解**:
$$EAT = 0.95 \times 0.98 \times 10 + 0.95 \times 0.02 \times 100 + 0.05 \times 0.98 \times (100 + 10) + 0.05 \times 0.02 \times (100 + 100)$$
$$= 9.31 + 1.9 + 5.39 + 0.2 = 16.8 \text{ ns}$$

## 直观理解

**邮局取包裹类比**:
- TLB = 你口袋里的通讯录(最快)
- 页表 = 邮局总目录本(需要去邮局查)
- Cache = 邮局大厅的货架(取了就走)
- 主存 = 邮局仓库(需要工作人员去取)

取包裹流程: 先查口袋通讯录(TLB), 有地址直接去货架(Cache)拿; 通讯录没有就要翻目录本(页表), 目录本告诉你地址后去货架(Cache)或仓库(主存)取。

### 记忆口诀
> **"虚查TLB, 不中查页表; 物查Cache, 不中访主存"**

## 知识关联

### 跨章节联系
- **操作系统**: 虚拟存储管理, 页面置换算法(OPT/FIFO/LRU)
- **计组第2章**: Cache映射方式(直接/全相联/组相联)
- **计组第4章**: 缺页中断属于异常(内中断), 由CPU内部产生

### 408常考题型
1. **地址转换全路径**: 给虚地址, 求物理地址和Cache行号
2. **访问时间计算**: EAT公式, TLB命中率+Cache命中率
3. **页表容量计算**: 页表项数×每项大小
4. **缺页处理流程**: TLB未命中→查页表→缺页→调页→更新TLB

### 易错陷阱
- **页内偏移不变**: 虚拟地址和物理地址的低offset位完全相同
- **Cache用物理地址查**: 先完成地址转换, 再查Cache(物理Cache)
- **页表基址寄存器**: CR3(或叫PTBR), 存当前进程页表起始地址
- **多级页表**: 虚页号被分成多段, 每段索引一级页表, 最后得到PPN
