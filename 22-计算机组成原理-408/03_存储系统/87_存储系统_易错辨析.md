# 88_存储系统_易错辨析

## 易错点1: Cache命中 ≠ 虚存命中

### 常见混淆

很多同学认为: "Cache命中了, 就不涉及虚存了"——**错误!**

**正确理解**:
- Cache命中 = 在Cache中找到了物理地址对应的数据
- 虚存命中 = 要访问的虚拟页在主存中(页表有效位=1)
- **两者是独立的检查**, 同一条访存指令可能涉及两者

**完整流程**:
```
虚拟地址 → TLB查虚页号 → 得到物理地址 → Cache查物理地址 → 得到数据
   |            |                |               |
   |       可能命中/未命中    可能缺页       可能命中/未命中
```

**四种组合**:

| TLB | 页表 | Cache | 说明 |
|-----|------|-------|------|
| 命中 | (不必查) | 命中 | 最优, 两次都命中 |
| 命中 | (不必查) | 未命中 | TLB命中但Cache没有 |
| 未命中 | 命中 | 命中 | TLB没有但页表有, Cache也有 |
| 未命中 | 缺页 | - | 页不在主存, 必须先处理缺页 |

### 关键区分
- **TLB未命中 ≠ 缺页**: TLB未命中只是地址转换缓存没有, 页表中可能有
- **Cache未命中 ≠ 缺页**: Cache缺失只是数据缓存没有, 主存中有
- **缺页 = 必须处理的严重事件**: 页不在主存中

## 易错点2: DRAM刷新周期 vs 存取周期

### 常见混淆

- **刷新周期(Refresh Cycle)**: DRAM需要定期刷新所有存储单元的间隔, 通常2ms或64ms
- **存取周期(Access Cycle)**: 完成一次完整读写操作的时间, 通常几十ns

**区别**:
| 概念 | 典型值 | 含义 |
|------|--------|------|
| 刷新周期 | 2ms~64ms | 所有行必须被刷新一遍的时间 |
| 存取周期 | 50~100ns | 一次读/写操作的时间 |
| 刷新时间 | 几十ns | 刷新一行的时间 |

**易错**: 不要把刷新周期和存取周期搞混!

- 刷新周期 >> 存取周期
- 刷新会占用少量存取周期(但影响很小)

## 易错点3: 直接映射地址划分

### 常见混淆

直接映射Cache的地址划分:
$$\text{主存地址} = \underbrace{\text{Tag}}_{\text{标记}} + \underbrace{\text{Index}}_{\text{行索引}} + \underbrace{\text{Offset}}_{\text{块内偏移}}$$

**易错**: 行索引位数的计算

$$\text{行数} = \frac{\text{Cache容量}}{\text{块大小}}$$
$$\text{Index位数} = \log_2(\text{行数})$$

**例**: Cache 16KB, 块大小 64B
- 行数 = 16KB / 64B = 256 = $2^8$
- Index = 8位
- Offset = $\log_2 64$ = 6位
- Tag = 总位数 - 8 - 6

**易错点**: 组相联的索引位数
$$\text{组数} = \frac{\text{Cache容量}}{\text{路数} \times \text{块大小}}$$
$$\text{Index位数} = \log_2(\text{组数})$$

## 易错点4: 虚拟地址空间 vs 物理地址空间

### 常见混淆

- 虚拟地址空间: 程序看到的地址空间(通常很大, 如4GB)
- 物理地址空间: 实际内存大小(通常较小)
- **虚拟地址空间可以大于物理地址空间** (这就是虚存的意义!)

**页表项中的物理页号**: 指向物理主存中的页框
**辅存(磁盘)**: 存放不在主存中的页

**易错**: 虚拟地址空间满了 ≠ 物理内存满了
- 虚拟地址空间由位数决定(如32位 = 4GB)
- 物理内存是实际安装的内存大小

## 易错点5: 写策略的适用范围

### Cache写策略
- **写直达**: 同时写Cache和主存, 需要写缓冲
- **写回**: 只写Cache, 被替换时才写回主存, 需要脏位

### 虚存写策略
- **只用写回**: 辅存I/O代价极高, 写直达不可行
- 脏页(D=1)在被替换时写回辅存

**易错**: "虚存也可以用写直达"——**不行!** 因为每次写都写磁盘, 性能无法接受。

## 易错点6: 替换算法的适用场景

| 算法 | Cache | 虚存 |
|------|-------|------|
| OPT | 理论基准 | 理论基准 |
| FIFO | 简单, 少用 | 有Belady异常 |
| LRU | 常用(硬件实现) | 近似实现(Clock) |
| Clock/NRU | 组相联常用 | 最常用 |

**易错**: "虚存中用LRU"——精确LRU在虚存中难以实现, 实际用Clock近似

## 易错点7: 地址位数计算

### 常见错误

**错误**: 虚拟地址32位 → 虚拟地址空间32MB
**正确**: 虚拟地址32位 → 虚拟地址空间 $2^{32}$B = 4GB

**错误**: 页大小4KB → 页内偏移10位
**正确**: 页大小4KB = $2^{12}$B → 页内偏移12位

### 正确公式
$$\text{位数} = \log_2(\text{大小/字节})$$

- 4KB = $2^{12}$B → 12位
- 1KB = $2^{10}$B → 10位
- 64B = $2^6$B → 6位
