# 40_写策略_408考点

## 核心概念

- **408高频考点**：全写法vs写回法的对比判断、写缓冲器的作用、脏位的功能、策略组合关系。
- **常见题型**：选择题判断说法正误、综合题中写操作的处理。
- **易混淆点**：全写法需不需要脏位、写缓冲器属于哪一层、策略组合的搭配关系。

## 原理分析

### 全写法 vs 写回法 核心对比

| 对比维度 | 全写法（Write-Through） | 写回法（Write-Back） |
|---------|------------------------|---------------------|
| 写命中处理 | 同时写Cache和主存 | 只写Cache，置脏位=1 |
| 脏位 | **不需要** | **需要** |
| 替换时操作 | 直接替换 | 检查脏位，脏则写回 |
| 主存一致性 | **始终一致** | **可能不一致** |
| 写速度 | 慢 | 快 |
| 读操作影响 | 无 | 无 |
| 总线带宽 | 高 | 低 |
| 硬件复杂度 | 低（需写缓冲器） | 高（需脏位逻辑） |
| 适用场景 | 对一致性要求高 | 高性能场景 |

### 写缓冲器（Write Buffer）

**位置**：在Cache与主存之间。

**作用**：
- 全写法每次写都要写主存，速度慢
- 写缓冲器暂存写数据，CPU不必等待主存写入完成
- 写缓冲器在后台以主存速度异步写入主存

**特点**：
- 容量有限（通常4~8个字）
- 缓冲器满时CPU必须等待
- 仅用于全写法，写回法不需要（写回法只写Cache）
- 可能导致**写后读（RAW）**冲突：若读的地址在写缓冲器中还未写入主存

### 脏位（Dirty Bit）详解

**作用**：标记Cache行是否被修改过。

| 脏位值 | 含义 | 替换时 |
|--------|------|--------|
| 0 | 未被修改（与主存一致） | 直接替换 |
| 1 | 被修改过（主存是旧数据） | 先写回主存再替换 |

**初始化**：所有Cache行初始脏位=0（与主存一致）。

**只用于写回法**：全写法每次写都更新主存，不存在脏数据，不需要脏位。

### 组合搭配关系速记

$$\text{全写法} + \text{非写分配法} \quad \text{（"都直接访问主存"）}$$
$$\text{写回法} + \text{写分配法} \quad \text{（"都尽量在Cache中操作"）}$$

**原因**：
- 全写法时刻保持主存最新→不命中直接写主存→非写分配
- 写回法尽量减少主存访问→不命中调入Cache再写→写分配

## 直观理解

**四种组合的行为类比**：

| 组合 | 类比 |
|------|------|
| 全写+非写分配 | 严格管家：写什么都立即通知总部，不合作的东西不引进 |
| 写回+写分配 | 懒散员工：尽量在自己桌上处理，需要的资料先借来再说 |

**脏位类比**：
> 脏位就像草稿纸上的"已修改"印章。如果没盖章（脏位=0），可以扔掉重写；如果盖了章（脏位=1），必须先抄到正式文件上才能扔。

## 知识关联

### 408典型考题

**题型一：判断说法正误**

- "全写法需要脏位"——**错误**，全写法不需要脏位
- "写回法不需要脏位"——**错误**，写回法必须有脏位
- "全写法只写主存不写Cache"——**错误**，是同时写
- "写缓冲器用于写回法"——**错误**，写缓冲器用于全写法
- "非写分配法会将数据调入Cache"——**错误**，不调入

**题型二：综合计算**

给定含写操作的访问序列，在写回+写分配策略下，计算Cache状态、脏位变化和命中率。

**题型三：性能分析**

比较全写法和写回法在不同写操作比例下的性能差异。

### 与总线的联系

- 全写法每次写都产生总线事务，占用总线带宽
- 写回法只有替换脏行时才产生总线事务
- 多处理器系统中，全写法更易维护一致性

### 与存储器层次的联系

- L1 Cache通常使用写回法（追求速度）
- L2/L3 Cache可能使用全写法（简化一致性管理）
- 虚拟存储器的写策略类似：磁盘相当于主存，主存相当于Cache

## 易错陷阱

### 陷阱一：全写法"不需要写Cache"

全写法是**同时写Cache和主存**，不是只写主存。如果只写主存，Cache中的旧数据会导致后续读得到旧值。

### 陷阱二：写缓冲器与Cache的关系

写缓冲器在Cache**外部**，位于Cache和主存之间。它是独立的硬件模块，不是Cache的一部分。

### 陷阱三：脏位初始值

新调入的Cache行脏位=0（刚从主存调入，与主存一致）。只有经过写操作后才变为1。

### 陷阱四：替换时脏位的处理

写回法替换脏行时：
1. 将脏行数据写入主存
2. 然后才能装入新块
3. 新块的脏位初始为0

这不是并行操作，是**串行**的，会增加缺失惩罚。

### 陷阱五：408默认假设

408考试中，若未特别说明：
- 默认使用**写回法+写分配法**（现代处理器主流方案）
- Cache容量通常指数据容量
- 块大小通常指数据块大小
