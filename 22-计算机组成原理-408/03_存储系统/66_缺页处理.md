# 67_缺页处理

## 核心概念
- **缺页(Page Fault)**: 访问的页不在主存中, 需要从辅存调入
- **有效位(Valid Bit)**: 页表项中的标志, 1=在主存, 0=不在主存
- **缺页中断(Page Fault Interrupt)**: 也叫**缺页异常**, 是一种**内中断(异常)**
- **缺页处理程序**: OS内核中负责处理缺页的代码

### 重要区分
| 概念 | 说明 |
|------|------|
| TLB未命中 | TLB中没有VPN→PPN映射, 需查页表, **不一定缺页** |
| 缺页 | 页表中有效位=0, 页不在主存, **必须处理** |
| 缺页中断 | 由CPU内部检测到缺页产生的异常, 属于**故障(Fault)** |

## 原理分析

### 缺页检测过程

```
CPU发出虚拟地址(VPN, Offset)
        |
        v
   查找TLB
   /       \
命中      未命中
 |          |
 v          v
PPN      查页表(访主存)
 |       /       \
 |    有效位=1   有效位=0
 |     |          |
 |     v          v
 |   得到PPN   缺页异常!! ← 核心检测点
 |     |          |
 v     v          v
  构造物理地址   OS缺页处理程序接管
```

### 缺页处理完整流程

```
1. 检测到缺页(有效位=0)
        |
        v
2. CPU陷入操作系统(产生异常)
        |
        v
3. 保存当前进程上下文(PC, 寄存器...)
        |
        v
4. OS查找空闲页框
   /            \
有空闲页框     无空闲页框
   |              |
   |              v
   |         执行页面置换算法
   |         (OPT/FIFO/LRU/Clock)
   |              |
   |              v
   |         选中牺牲页
   |         /        \
   |     脏页(D=1)   干净页(D=0)
   |       |            |
   |       v            v
   |    写回辅存     直接覆盖
   |       |            |
   v       v            v
5. 将目标页从辅存读入页框
        |
        v
6. 更新页表项(有效位=1, 物理页号=页框号)
        |
        v
7. 更新TLB(若使用TLB)
        |
        v
8. 恢复进程上下文
        |
        v
9. 重新执行引起缺页的指令 ← 关键!
```

### 与一般中断的区别

| 对比项 | 缺页中断(异常/故障) | 一般I/O中断 |
|--------|---------------------|-------------|
| 产生原因 | CPU执行指令时内部检测 | 外部设备完成I/O |
| 发生时机 | 指令执行过程中 | 指令执行间(异步) |
| 返回方式 | **重新执行**引起缺页的指令 | 从下一条指令继续 |
| 类型 | 内中断(同步异常) | 外中断(异步中断) |
| 处理紧迫性 | 必须处理(否则指令无法继续) | 可延迟 |

### 关键: 重新执行指令

缺页中断处理完后, 必须**重新执行引起缺页的那条指令**, 因为该指令还没执行完。

**例**: `MOV R1, [0x12345678]` — 该指令要从虚地址0x12345678读数据
1. 地址转换时发现该页不在主存(缺页)
2. OS调入该页, 更新页表
3. 重新执行 `MOV R1, [0x12345678]`
4. 这次地址转换成功, 指令完成

## 直观理解

**图书馆找书类比**:
1. 你去书架找书(TLB查找)
2. 书架上没有(缺页!)
3. 图书管理员(OS)接管
4. 管理员去仓库找这本书(从辅存调页)
5. 如果书架满了, 管理员先拿走一本不常用的书(页面置换)
6. 新书放到书架上(更新页表)
7. 你重新去找这本书(重新执行指令)

### 记忆口诀
> **"缺页是故障, 指令重新做; 置换看脏页, 写回再调入"**

## 知识关联

### 跨章节联系
- **操作系统**: 虚拟存储管理核心机制
- **计组第4章**: 缺页属于**故障(Fault)**, 不是中断(Interrupt)
  - Fault: 可以恢复, 重新执行引起故障的指令
  - Interrupt: 异步事件, 从下一条指令继续

### 影响缺页率的因素
1. **页面大小**: 页越大, 局部性利用率越低, 缺页率可能高
2. **分配给进程的页框数**: 越多缺页率越低(但浪费内存)
3. **页面置换算法**: LRU优于FIFO
4. **程序局部性**: 时间局部性和空间局部性越好, 缺页率越低

### 工作集模型(补充)
- **工作集**: 进程在最近 $\Delta$ 时间内访问的页的集合
- 若分配的页框数 ≥ 工作集大小, 缺页率低
- 否则频繁缺页(抖动/颠簸)

### 常考陷阱
1. **缺页中断后执行哪条指令?** → 重新执行引起缺页的指令, 不是下一条
2. **缺页中断是否属于外中断?** → 不属于, 是内中断(异常)
3. **TLB未命中一定是缺页吗?** → 不一定, 页表中可能有映射
4. **修改页表后需要更新TLB吗?** → 需要(或使TLB对应项无效)
