# 84_SSD_磨损均衡

## 核心概念
- **磨损均衡(Wear Leveling)**: 均匀使用各闪存块, 避免某些块过早损坏
- **垃圾回收(Garbage Collection)**: 回收无效数据占用的空间
- **TRIM命令**: 操作系统通知SSD哪些数据已无效
- **FTL(Flash Translation Layer)**: 闪存转换层, 管理逻辑地址到物理地址的映射

### 为什么需要磨损均衡?

NAND Flash每个块有擦写次数上限:
- SLC: ~100,000次
- MLC: ~10,000次
- TLC: ~3,000次
- QLC: ~1,000次

**问题**: 如果某些块被频繁擦写(如文件分配表所在区域), 这些块会先于其他块损坏。

**解决**: 磨损均衡——将写操作分散到所有块, 使各块的擦写次数尽量均匀。

## 原理分析

### FTL映射

```
逻辑地址空间(操作系统看到的)     物理地址空间(闪存实际)
┌──────────┐                    ┌──────────┐
│  逻辑页0  │──映射──→          │  物理页5  │
│  逻辑页1  │──映射──→          │  物理页12 │
│  逻辑页2  │──映射──→          │  物理页3  │
│  ...     │                    │  ...     │
└──────────┘                    └──────────┘
```

**映射表存储**: DRAM(快速) + 闪存(持久化备份)

### 垃圾回收

SSD不能原地更新数据(必须先擦除整个块), 所以:

```
写入新数据流程:
1. 找到一个空闲页
2. 将新数据写入空闲页
3. 将旧页标记为无效(不立即擦除)
4. 更新FTL映射表
```

**垃圾回收过程**:
```
块0: [有效] [有效] [无效] [有效]  ← 3有效1无效
块1: [无效] [无效] [有效] [无效]  ← 1有效3无效

GC选择块1(有效数据最少):
1. 将块1中的有效数据复制到其他块
2. 擦除整个块1
3. 块1变为空闲块, 可供写入
```

### 磨损均衡策略

**动态磨损均衡**: 写入新数据时, 优先选择擦写次数少的块

**静态磨损均衡**: 定期将"冷数据"(长期不变)从低磨损块移动到高磨损块, 反之亦然

```
冷数据(很少修改, 低磨损)  ──移动──→ 高磨损块
热数据(频繁修改, 高磨损)  ──移动──→ 低磨损块
```

### TRIM命令

**问题**: 操作系统删除文件时, 只是标记文件系统元数据为删除, SSD不知道这些数据已无效。

**TRIM**: 操作系统主动告诉SSD"这些逻辑页不再使用了"

```
传统流程(OS不通知SSD):
OS: "删除文件A"
SSD: 不知道, 仍保留数据 → 垃圾回收效率低

使用TRIM后:
OS: "删除文件A" + 发送TRIM
SSD: 标记对应物理页为无效 → 垃圾回收更高效
```

### 写放大(Write Amplification)

**定义**: 实际写入闪存的数据量 / 主机请求写入的数据量

$$\text{写放大系数} = \frac{\text{实际闪存写入量}}{\text{主机写入量}}$$

**原因**:
- 垃圾回收需要复制有效数据
- 擦除前需要搬移数据
- 磨损均衡的数据移动

**例**: 主机写入4KB, 但GC需要搬移64KB有效数据 + 擦除256KB块
实际闪存写入: 4KB(新数据) + 64KB(搬移) = 68KB
写放大系数: 68/4 = 17

## 直观理解

**停车位类比**:
- 闪存块 = 停车位
- 页 = 车位中的一个位置
- 擦除 = 整排车位清空(必须一起)
- 垃圾回收 = 把有用的车挪到别处, 然后清空整排
- 磨损均衡 = 不要老停同一个车位, 分散使用

### 记忆口诀
> **"磨损均衡保寿命, 垃圾回收腾空间; TRIM通知减无效, 写放大概率要关注"**

### 与HDD对比(补充)
- HDD没有磨损均衡问题(磁介质理论上无限次读写)
- HDD可以直接覆写(不需要先擦除)
- SSD的随机写性能远优于HDD(无寻道), 但持续大量写入可能变慢
