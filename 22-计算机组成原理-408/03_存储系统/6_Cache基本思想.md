# 07_Cache基本思想

## 核心概念

### Cache的定义

**Cache（高速缓冲存储器）**：位于CPU和主存之间的小容量高速存储器，存放主存中当前最活跃的数据块的副本。

### Cache的基本目标

- 让CPU的**大部分访存请求**在Cache中完成
- 利用**局部性原理**，使Cache的命中率尽可能高（通常 > 95%）
- 从而使得**平均访存时间**接近Cache的访问时间

### 命中率

$$H = \frac{N_c}{N_c + N_m}$$

其中：
- $N_c$ = Cache命中次数
- $N_m$ = Cache未命中（访问主存）次数
- $H$ = 命中率（Hit Rate），$0 \leq H \leq 1$

**缺失率（未命中率）**：

$$M = 1 - H$$

### 平均访问时间

$$t_{\text{avg}} = H \times t_c + (1-H) \times t_m$$

其中：
- $t_c$ = 访问Cache的时间
- $t_m$ = 访问主存的时间

**注意**：此公式假设未命中时需要先访问Cache发现未命中，再访问主存。严格公式为：

$$t_{\text{avg}} = H \times t_c + (1-H) \times (t_c + t_m)$$

如果题目未特别说明，通常使用简化公式 $t_{\text{avg}} = H \times t_c + (1-H) \times t_m$。

### 访问效率

$$e = \frac{t_c}{t_{\text{avg}}}$$

$e$ 越接近1，说明层次结构效率越高。

**等效访问时间**：$t_{\text{avg}}$ 就是等效的存储器访问时间。

## 原理分析

### Cache工作的核心流程

**读操作**：
1. CPU给出地址，Cache控制器检查该地址是否在Cache中
2. 若在（命中），直接从Cache读取数据给CPU
3. 若不在（未命中），从主存读取数据块到Cache，再给CPU
4. Cache中若已满，需按替换策略替换某一块

**写操作**（更复杂）：
- **写命中**（要写的地址在Cache中）：
  - **写直达（Write-Through）**：同时写Cache和主存，保证一致性，但慢
  - **写回（Write-Back）**：只写Cache，设"脏位"，被替换时才写回主存，快
- **写不命中**（要写的地址不在Cache中）：
  - **写分配（Write-Allocate）**：先将块调入Cache，再写（配合写回）
  - **非写分配（No-Write-Allocate）**：直接写主存，不调入Cache（配合写直达）

### 命中率的计算示例

**例**：某程序执行期间，CPU共访问存储器10000次，其中Cache命中9500次。

$$H = \frac{9500}{10000} = 0.95$$

$$M = 1 - 0.95 = 0.05$$

### 平均访问时间的计算示例

**例**：$t_c = 10\,\text{ns}$，$t_m = 100\,\text{ns}$，$H = 0.95$。

简化公式：
$$t_{\text{avg}} = 0.95 \times 10 + 0.05 \times 100 = 9.5 + 5 = 14.5\,\text{ns}$$

严格公式（先查Cache再查主存）：
$$t_{\text{avg}} = 0.95 \times 10 + 0.05 \times (10 + 100) = 9.5 + 5.5 = 15\,\text{ns}$$

访问效率：
$$e = \frac{10}{15} = 0.667 \quad (\text{严格公式})$$

### Cache容量与命中率的关系

| Cache容量 | 典型命中率 |
|-----------|-----------|
| 8 KB | 85%~90% |
| 16 KB | 90%~93% |
| 32 KB | 93%~95% |
| 64 KB | 95%~97% |
| 128 KB+ | 97%~99% |

**规律**：容量越大命中率越高，但边际收益递减。

## 直观理解

**冰箱类比**：
- **主存** = 超市（东西全，但去一趟要花时间）
- **Cache** = 家里的冰箱（放常用食材，拿取快）
- **命中率** = "打开冰箱就找到想吃的"概率
- **未命中** = "冰箱没有，得去超市买"

**设计Cache就像设计冰箱**：
- 放什么进去？（替换策略）
- 放多少？（容量）
- 怎么分隔？（映射方式）
- 买回来怎么摆？（写策略）

### 命中率的直觉

- 命中率从90%提升到99%，看起来只提升了9个百分点
- 但未命中率从10%降到了1%——降了**10倍**
- 因为未命中代价远大于命中收益，这10倍的差距对平均访问时间影响巨大

## 知识关联

### 跨章节联系
- **第3章（存储系统）**：Cache是存储层次的核心，后续地址映射、替换策略都是围绕Cache展开
- **第5章（CPU）**：CPU访存操作由Cache控制器截获，CPU不直接面对主存
- **第6章（总线）**：Cache与主存之间的数据传送通过总线完成

### 常考组合
- 命中率 + 平均访问时间计算（计算题必考）
- 写直达 vs 写回策略对比（选择题）
- 影响命中率的因素（容量、块大小、映射方式、替换策略）

### 易错陷阱

1. **公式中的 $t_m$ 是否包含 $t_c$**：
   - 如果题目说"访问主存需要200ns"，通常指**从发出请求到数据返回的总时间**，即已包含查Cache的时间
   - 如果题目明确说"先查Cache再查主存"，用 $t_c + t_m$
   - 考试务必看清题意

2. **"命中率99%"不等于"99%的数据在Cache中"**：
   - 命中率是访问次数的比例，不是数据量的比例

3. **写操作比读操作复杂得多**：
   - 读操作只需考虑命中/未命中
   - 写操作还要考虑写策略（写直达/写回）和一致性问题

4. **Cache的存在对程序员是透明的**：
   - 程序员写代码时不需要知道Cache的存在
   - 但了解Cache原理有助于写出高效的程序（如循环遍历顺序影响命中率）
