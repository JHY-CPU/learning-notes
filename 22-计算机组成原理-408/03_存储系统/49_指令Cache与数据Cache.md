# 50_指令Cache与数据Cache

## 核心概念

- **分离Cache（Split Cache）**：L1 Cache分为**指令Cache（I-Cache）**和**数据Cache（D-Cache）**，分别存放指令和数据。
- **哈佛结构在Cache层的应用**：指令和数据分开存储，支持同时取指和访存，消除结构冒险。
- **统一Cache（Unified Cache）**：指令和数据共用同一个Cache，容量利用率高但可能存在访问冲突。
- **408考点**：分离Cache的优点、与哈佛结构的关系、I-Cache和D-Cache缺失率的分别计算。

## 原理分析

### 分离Cache的原因

**结构冒险问题**：

在流水线CPU中，取指（IF阶段）和访存（MEM阶段）可能同时发生：

```
周期1:  IF  | ID | EX | MEM | WB
周期2:      | IF | ID | EX  | MEM | WB
```

- 第1条指令在MEM阶段访问数据Cache
- 第2条指令在IF阶段访问指令Cache
- 如果用统一Cache，两者同时请求 → **冲突**，必须串行处理

**分离Cache解决冲突**：
- I-Cache处理取指请求
- D-Cache处理访存请求
- 两者可**并行工作**，无冲突

### 分离Cache vs 统一Cache

| 对比维度 | 分离Cache | 统一Cache |
|---------|----------|----------|
| 结构 | I-Cache + D-Cache 各自独立 | 指令和数据共用 |
| 并行性 | 取指和访存可并行 | 可能冲突 |
| 容量利用率 | 较低（可能一边满一边空） | 较高（灵活分配） |
| 控制复杂度 | 较高（需两套控制逻辑） | 较低 |
| 适用层级 | L1 | L2/L3 |
| 命中率 | 可能低于统一Cache（容量分半） | 可能更高 |

### I-Cache和D-Cache的特性差异

| 特性 | I-Cache | D-Cache |
|------|---------|---------|
| 写操作 | **极少**（自修改程序除外） | 频繁 |
| 替换算法 | 简单（如FIFO） | 复杂（通常LRU） |
| 写策略 | 可能不需要写回 | 需要写回/全写 |
| 缺失率 | 通常较低（指令局部性强） | 通常较高 |
| 大小 | 可以较小 | 需要较大 |

### 哈佛结构 vs 冯诺依曼结构

| | 哈佛结构 | 冯诺依曼结构 |
|---|---------|------------|
| 指令和数据 | 分开存储 | 混合存储 |
| 总线 | 指令总线+数据总线 | 共用总线 |
| 并行性 | 可同时取指和访存 | 不可同时 |
| 实现 | 成本高 | 成本低 |
| 现代应用 | L1 Cache采用 | L2/L3和主存采用 |

现代计算机是**混合结构**：Cache层采用哈佛结构（分离），主存层采用冯诺依曼结构（统一）。

## 直观理解

**分离Cache类比**：

就像银行的"VIP窗口"和"普通窗口"：
- VIP窗口（I-Cache）只处理固定业务（取指令），效率高
- 普通窗口（D-Cache）处理各种业务（读写数据），灵活但复杂
- 分开排队不冲突，合在一起可能互相等待

**记忆口诀**：
> "L1分I和D，哈佛在Cache；L2L3统一，冯氏在存储"

## 知识关联

### 与流水线的联系（第5章）

- 流水线的结构冒险需要分离Cache来避免
- IF阶段用I-Cache，MEM阶段用D-Cache

### 与AMAT计算的联系

分离Cache需要分别计算：

$$\text{指令AMAT} = t_{I\text{-Cache}} + (1-H_I) \times t_{\text{miss}}$$

$$\text{数据AMAT} = t_{D\text{-Cache}} + (1-H_D) \times t_{\text{miss}}$$

综合CPI中：
$$\text{缺失CPI} = \text{指令缺失率} + \text{访存比例} \times \text{数据缺失率}$$

## 易错陷阱

- 分离Cache的"容量"是各半还是独立？通常是各自独立设计（如I-Cache 32KB + D-Cache 32KB = L1共64KB）
- I-Cache也可能发生写操作（自修改程序），但408通常忽略
- 统一Cache的命中率可能高于分离Cache（相同总容量下），但没有并行优势
- 哈佛结构在主存层的实现成本太高，所以只在Cache层使用
