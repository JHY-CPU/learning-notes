# 17_存储器容量扩展_位扩展

## 核心概念

### 位扩展的定义

**位扩展（Bit Expansion）**：用多片**存储单元数相同但字长较短**的芯片并联，增加存储器的**字长（数据宽度）**。

### 位扩展的核心思想

- **存储单元数不变**，每个单元的位数增加
- 多片芯片**共享地址线**（所有芯片同时被选中）
- 各片的数据线**拼接**成更宽的数据总线
- **片选信号共用**（所有芯片同时工作）

### 适用场景

当目标字长大于单片字长时，需要位扩展。

$$\text{位扩展需要的芯片数} = \frac{\text{目标字长}}{\text{单片字长}}$$

## 原理分析

### 例题：$4\text{K} \times 4$ → $4\text{K} \times 8$

**需求**：用 $4\text{K} \times 4$ 位的芯片组成 $4\text{K} \times 8$ 位的存储器。

**分析**：
- 存储单元数不变（4K → 4K），字长从4位扩展到8位
- 需要 $8 / 4 = 2$ 片

**连接方式**：

```
          地址总线（12位：A0~A11）
               │
        ┌──────┼──────┐
        │             │
   ┌────┴────┐  ┌────┴────┐
   │ 芯片1   │  │ 芯片2   │
   │ 4K×4   │  │ 4K×4   │
   │(低4位)  │  │(高4位)  │
   └────┬────┘  └────┬────┘
        │             │
   D0~D3(低4位)  D4~D7(高4位)
        │             │
        └──────┬──────┘
               │
          数据总线（8位：D0~D7）
```

**具体连接规则**：
1. **地址线并联**：A0~A11 同时连接到两片芯片的地址引脚
2. **片选信号并联**：两片芯片的CE连在一起（同时选中或同时不选中）
3. **读写控制并联**：WE信号连在一起
4. **数据线分配**：
   - 芯片1的数据输出接 D0~D3（低4位）
   - 芯片2的数据输出接 D4~D7（高4位）

### 另一个例题：$1\text{K} \times 1$ → $1\text{K} \times 8$

- 需要 $8 / 1 = 8$ 片
- 每片接1位数据线：芯片i 接 Di
- 地址线全部并联（A0~A9）

### 位扩展的关键特点

| 特征 | 说明 |
|------|------|
| 存储单元数 | 不变 |
| 字长（数据宽度） | 增加 |
| 地址线 | 共用（所有芯片接同一组地址线） |
| 数据线 | 拼接（每片接不同位段） |
| 片选信号 | 共用（同时选中所有芯片） |
| 芯片数量 | $= \text{目标字长} / \text{单片字长}$ |

## 直观理解

**合写黑板报类比**：
- 一个人写4个字太慢 → 两个人各写4个字，拼起来就是8个字
- 两个人看同一份草稿（共享地址线）
- 各自负责不同部分（数据线分配）
- 老师说"开始"两人同时写（共用片选）

**乐高积木类比**：
- 你有两块4厘米宽的积木
- 横着拼起来就变成8厘米宽
- 宽度增加了（字长扩展），但层数没变（存储单元数不变）

### 易错陷阱

1. **位扩展只增加字长，不增加存储单元数**
2. **位扩展的芯片是"并联"的**——同时工作，数据拼接
3. **片选信号是共用的**——选中时所有芯片同时响应
4. **数据线分配是"各取几位"**——不是所有芯片数据线接同一条总线
5. **位扩展不影响地址空间大小**——地址空间由存储单元数决定
6. **位扩展芯片数 = 目标字长 / 单片字长**——必须整除
