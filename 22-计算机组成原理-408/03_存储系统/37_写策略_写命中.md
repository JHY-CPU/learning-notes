# 38_写策略_写命中

## 核心概念

- **写策略（Write Policy）**：CPU写操作时，Cache与主存之间数据一致性的维护策略。分为**写命中策略**和**写不命中策略**两大类。
- **写命中（Write Hit）**：CPU要写入的地址在Cache中找到了对应的有效行。
- **两种写命中策略**：全写法（Write-Through）和写回法（Write-Back）。
- **408必考**：全写法与写回法的对比、优缺点、适用场景。

## 原理分析

### 全写法（Write-Through）

**定义**：写命中时，**同时写Cache和主存**，保证两者数据一致。

**工作流程**：
1. CPU发出写请求，地址命中Cache
2. 数据同时写入Cache行和对应的主存单元
3. Cache和主存始终保持一致

**优点**：
- 实现简单，数据一致性容易维护
- 不需要脏位（dirty bit）
- 主存中的数据始终是最新的

**缺点**：
- 每次写都要访问主存，**写操作速度慢**
- 增加总线带宽占用
- 写操作频繁时性能瓶颈明显

**优化——写缓冲器（Write Buffer）**：
- 全写法通常配合**写缓冲器**使用
- CPU写入Cache的同时，将数据放入写缓冲器
- 写缓冲器在后台异步写入主存
- CPU不必等待主存写入完成，可继续执行
- 写缓冲器满时，CPU必须等待

### 写回法（Write-Back）

**定义**：写命中时，**只写Cache**，不写主存。只有当该行被替换时，才将脏数据写回主存。

**工作流程**：
1. CPU发出写请求，地址命中Cache
2. 数据**只写入Cache行**，不写主存
3. 将该行的**脏位（Dirty Bit）置1**，标记为"被修改过"
4. 当该行被替换时，检查脏位：脏位=1则写回主存，脏位=0则直接替换

**优点**：
- 写操作速度快（只写Cache）
- 减少总线带宽占用
- 多次写同一块只在替换时写一次主存
- 适合写操作频繁的场景

**缺点**：
- 需要**脏位**，硬件复杂度增加
- 数据一致性维护复杂
- 替换时可能需要额外的写回操作，增加替换延迟
- 系统崩溃时可能丢失未写回的数据

## 直观理解

**全写法类比**：
> 像写日记——每次写笔记都在两个本子（日记本和备份本）上同时抄写。安全但费时间。

**写回法类比**：
> 像草稿本——先在草稿本上随意修改，等定稿了才抄到正式本上。快速但草稿本丢了就麻烦了。

**记忆口诀**：
> "全写同步两处写，写回懒惰标记脏"

## 知识关联

### 全写法 vs 写回法对比表

| 对比项 | 全写法（Write-Through） | 写回法（Write-Back） |
|--------|------------------------|---------------------|
| 写命中时 | 同时写Cache和主存 | 只写Cache |
| 脏位 | **不需要** | **需要** |
| 主存一致性 | 始终一致 | 可能不一致 |
| 写速度 | 慢（受主存速度限制） | 快（只写Cache） |
| 总线带宽 | 占用多 | 占用少 |
| 替换时 | 无需写回 | 脏位=1需写回 |
| 硬件复杂度 | 低（需写缓冲器） | 高（需脏位） |
| 适用场景 | 对一致性要求高 | 高性能处理器 |
| 配合策略 | 写缓冲器 + 非写分配 | 写分配法 |

### 与其他Cache机制的联系

- **写缓冲器**：全写法的标配优化，缓解CPU等待主存写入的瓶颈
- **脏位（Dirty Bit）**：写回法的标志位，每个Cache行额外增加1位
- **替换算法**：写回法下，替换时需检查脏位决定是否写回

### 408考点

- 选择题：全写法是否需要脏位（不需要）
- 选择题：写回法替换时的操作（检查脏位，脏则写回）
- 对比两种写策略的优缺点
- 写缓冲器的作用和工作原理

## 易错陷阱

### 陷阱一：全写法也需要写Cache

全写法是**同时写**Cache和主存，不是只写主存。很多同学误以为全写法只写主存。

### 陷阱二：写回法的数据一致性问题

写回法下，主存中的数据可能是旧的。多处理器环境下需要额外的Cache一致性协议（如MESI），但408不深入考查。

### 陷阱三：写缓冲器溢出

写缓冲器容量有限。当写操作密集导致缓冲器满时，CPU必须**暂停等待**，这会影响性能。

### 陷阱四：脏位与有效位的区别

- **有效位（V）**：该Cache行是否有有效数据
- **脏位（D）**：该Cache行数据是否被修改过
- 两者独立：有效位=1且脏位=1表示"有有效数据且被修改过"
