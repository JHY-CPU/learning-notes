# 03_存储器性能指标_容量

## 核心概念

### 存储容量的基本公式

$$\text{存储容量} = \text{存储单元数} \times \text{存储字长}$$

- **存储单元数**：有多少个可寻址的位置，由**MAR**（存储器地址寄存器）的位数决定
- **存储字长**：每个存储单元能存多少位，由**MDR**（存储器数据寄存器）的位数决定

$$\text{存储单元数} = 2^{\text{MAR位数}}$$

$$\text{存储字长} = \text{MDR位数}$$

因此：

$$\text{存储容量} = 2^{\text{MAR位数}} \times \text{MDR位数} \quad (\text{bit})$$

### 举例

若MAR为16位，MDR为8位：
- 存储单元数 = $2^{16} = 65536$ 个单元
- 每单元 8 bit = 1 Byte
- 总容量 = $2^{16} \times 8 = 524288$ bit = 64 KB

### 编址方式（重要！）

**按字节编址**：每个字节（8bit）分配一个地址
- 现代计算机绝大多数采用按字节编址
- 若字长32位（4字节），每个字占4个地址

**按字编址**：每个字（word）分配一个地址
- 字长是多少位，每个存储单元就占多少位
- 若字长32位，每个地址对应4字节

> **考试重点**：题目给出MAR和MDR时，首先要判断是按字节编址还是按字编址，这直接影响地址空间大小的理解。

### 地址空间与实际容量的关系

若MAR为n位，按字节编址：
- 地址空间 = $2^n$ 字节
- 例如：MAR=20位，地址空间 = $2^{20}$ B = 1 MB

若MAR为n位，按字编址且字长为W位：
- 地址空间 = $2^n$ 个字 = $2^n \times \frac{W}{8}$ 字节

## 原理分析

### 容量计算的三种常见题型

**题型1：给MAR/MDR求容量**
> MAR = 16位，MDR = 32位，求存储容量

解答：
- 存储单元数 = $2^{16} = 64\text{K}$
- 字长 = 32 bit = 4 B
- 容量 = $2^{16} \times 32 = 2^{21}$ bit = 256 KB

**题型2：给容量反求MAR位数**
> 某机主存容量为 64KB，按字节编址，求MAR至少需要多少位？

解答：
- 64KB = $2^6 \times 2^{10} = 2^{16}$ B
- 按字节编址，需要 $2^{16}$ 个地址
- MAR至少需要 **16位**

**题型3：芯片容量与系统容量**
> 用 $4\text{K} \times 8$ 的芯片组成 $16\text{K} \times 16$ 的存储器，需要多少片？

解答：
- 字方向扩展：$16\text{K} / 4\text{K} = 4$ 组
- 位方向扩展：$16 / 8 = 2$ 片/组
- 总片数 = $4 \times 2 = 8$ 片

### 地址线与容量的换算

| 容量 | 地址线数量（按字节编址） |
|------|------------------------|
| 1 KB | 10位 ($2^{10} = 1024$) |
| 4 KB | 12位 |
| 8 KB | 13位 |
| 16 KB | 14位 |
| 64 KB | 16位 |
| 1 MB | 20位 |
| 4 MB | 22位 |
| 16 MB | 24位 |
| 4 GB | 32位 |

**记忆技巧**：$1\text{K} = 2^{10}$，所以每增加10位地址线，容量增大1024倍。

### 芯片引脚数的计算

对于一个存储芯片：
- 地址线：$\lceil \log_2(\text{存储单元数}) \rceil$ 根
- 数据线：等于字长位数 根
- 控制线：至少 CE（片选）和 WE（写使能）/ OE（输出使能）

$$\text{总引脚数} \geq \text{地址线} + \text{数据线} + \text{控制线}$$

## 直观理解

**储物柜类比**：
- **存储单元数** = 储物柜的个数（比如有100个柜子）
- **存储字长** = 每个柜子的大小（能放多少东西）
- **MAR** = 编号位数（001-999需要3位数编号，2位数只够编99个）
- **MDR** = 每个柜子一次能取出的数据量

**地址空间的直觉**：
- 如果你用2位十进制数编号储物柜，最多能编100个（00-99）
- 用3位，最多1000个（000-999）
- 同理，n位二进制最多表示 $2^n$ 个地址

## 知识关联

### 跨章节联系
- **第2章**：二进制与容量单位的换算（$1\text{KB} = 2^{10}\text{B}$, $1\text{MB} = 2^{20}\text{B}$）
- **第4章**：指令长度与存储字长的关系——指令必须能在存储字长范围内存放
- **第5章**：MAR和MDR在CPU内部，是CPU与主存之间的接口寄存器
- **第7章**：I/O接口中的数据缓冲寄存器也有类似的数据宽度概念

### 常考组合
- MAR/MDR → 容量（选择题必考）
- 容量 → 地址线数量 → 芯片扩展（大题必考）
- 按字节编址 vs 按字编址的地址空间差异

### 易错陷阱
1. **MAR位数和MDR位数不一定相等**！地址线和数据线是独立的
2. **"按字节编址"时，MAR位数决定的是字节数，不是字数**
3. **芯片标注 "K×8" 中的K是1024**，不是1000（计算机领域）
4. **地址从0开始编号**，n位地址有 $2^n$ 个地址（0到 $2^n-1$）
5. **容量单位**：B=字节，b=位，考试中经常混用设陷阱
