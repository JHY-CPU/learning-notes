# 33_地址映射_408考点

## 核心概念

- **408高频题型**：给定主存容量、Cache容量、块大小、映射方式，求地址字段划分（Tag/行号/组号/偏移各占几位）。
- **解题关键**：先确定块内偏移位数，再确定行号或组号位数，最后用总地址位数减去已知位数得到Tag位数。
- **易混淆点**：直接映射用"行号"，组相联用"组号"，全相联两者都不用。

## 原理分析

### 解题模板

**已知条件**：主存容量 $M$ 字节，Cache容量 $C$ 字节，块大小 $B$ 字节，映射方式。

**步骤**：

$$\text{Step 1: 主存地址位数} = \log_2 M$$

$$\text{Step 2: 块内偏移位数} = \log_2 B$$

$$\text{Step 3: Cache行数 } L = \frac{C}{B}$$

$$\text{Step 4: 根据映射方式确定中间字段}$$

- 直接映射：行号位数 $= \log_2 L$
- R路组相联：组数 $Q = L/R$，组号位数 $= \log_2 Q$
- 全相联：无行号/组号

$$\text{Step 5: Tag位数} = \text{地址位数} - \text{中间字段位数} - \text{偏移位数}$$

### 计算题一（直接映射）

**题目**：某计算机主存容量为64KB，Cache容量为4KB，块大小为64B，采用直接映射。求地址各字段位数。

**解答**：

| 计算项 | 公式 | 结果 |
|--------|------|------|
| 主存地址位数 | $\log_2 64\text{K} = \log_2 2^{16}$ | 16位 |
| 块内偏移 | $\log_2 64 = \log_2 2^6$ | 6位 |
| Cache行数 | $4\text{K} / 64 = 2^{12} / 2^6 = 2^6$ | 64行 |
| 行号位数 | $\log_2 64 = 6$ | 6位 |
| Tag位数 | $16 - 6 - 6$ | 4位 |

地址格式：`[Tag: 4位][行号: 6位][偏移: 6位]`

### 计算题二（组相联映射）

**题目**：主存64KB，Cache 4KB，块大小64B，采用4路组相联。求地址各字段位数。

**解答**：

| 计算项 | 公式 | 结果 |
|--------|------|------|
| 主存地址位数 | $\log_2 2^{16}$ | 16位 |
| 块内偏移 | $\log_2 2^6$ | 6位 |
| Cache行数 | $2^{12}/2^6$ | 64行 |
| 组数 | $64 / 4$ | 16组 |
| 组号位数 | $\log_2 16 = 4$ | 4位 |
| Tag位数 | $16 - 4 - 6$ | 6位 |

地址格式：`[Tag: 6位][组号: 4位][偏移: 6位]`

### 计算题三（全相联映射）

**题目**：主存64KB，Cache 4KB，块大小64B，采用全相联映射。求地址各字段位数。

**解答**：

| 计算项 | 公式 | 结果 |
|--------|------|------|
| 主存地址位数 | $\log_2 2^{16}$ | 16位 |
| 块内偏移 | $\log_2 2^6$ | 6位 |
| Tag位数 | $16 - 0 - 6$ | 10位 |

地址格式：`[Tag: 10位][偏移: 6位]`

验证：主存块数 $= 64\text{K}/64 = 1024$ 块，$\log_2 1024 = 10$ 位Tag，正确。

## 直观理解

**解题三步走**（记忆口诀）：
> "偏移看块大小，行号看Cache几行，Tag减一减"

- 块内偏移：只和**块大小**有关
- 行号/组号：只和**Cache组织**有关（行数或组数）
- Tag：剩下全归它

**万能公式**：

$$\text{Tag} + \text{行号(或组号)} + \text{块内偏移} = \text{主存地址总位数}$$

## 知识关联

### 与Cache行结构的联系

每个Cache行的实际大小 = 有效位(1bit) + Tag位数 + 脏位(1bit, 写回法) + 数据块(B字节)

Cache总容量（含开销）：

$$C_{\text{total}} = L \times (1 + \text{Tag位数} + 1 + B \times 8) \text{ bits}$$

### 与主存容量的联系

- 主存块数 = 主存容量 / 块大小
- Tag位数 = $\log_2(\text{主存块数}) - \text{行号(或组号)位数}$
- 主存地址总能被三段（或两段）完全划分，不会有剩余

### 408常见变化

1. **给Tag位数求主存容量**：反向计算
2. **给定相联度和Tag位数求Cache容量**
3. **多级Cache联合计算**：L1和L2可能采用不同映射方式

## 易错陷阱

### 陷阱一：混淆"行数"和"组数"

直接映射中行数 = 组数，但组相联中组数 = 行数 / 路数。

错误：4路组相联，64行Cache，算组号用 $\log_2 64$（应为 $\log_2 16$）

### 陷阱二：忽略字节编址 vs 字编址

题目说"字长32位"不等于按字节编址。若按字编址，偏移位数可能不同。408默认**按字节编址**。

### 陷阱三：Cache容量包含行开销吗？

题目若说"Cache容量为4KB"，一般指**数据存储容量**（不含Tag等开销）。但需看清题意。

### 陷阱四：主存地址位数取整

主存容量不一定是2的幂。若主存60KB，则地址需要 $\lceil \log_2 60\text{K} \rceil = 16$ 位（取整到最近的大于等于的2的幂，或按实际地址线算）。

### 练习题

**题目**：主存128KB，Cache 8KB，块大小128B，8路组相联。求地址各字段。

**解答**：
- 地址位数：$\log_2 128\text{K} = 17$ 位
- 偏移：$\log_2 128 = 7$ 位
- Cache行数：$8\text{K}/128 = 64$ 行
- 组数：$64/8 = 8$ 组
- 组号：$\log_2 8 = 3$ 位
- Tag：$17 - 3 - 7 = 7$ 位

答案：`[Tag: 7位][组号: 3位][偏移: 7位]`
