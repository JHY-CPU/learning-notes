# 57_TLB快表

## 核心概念

- **TLB（Translation Lookaside Buffer）**：页表的Cache，缓存最近使用的页表项（PTE），加速虚拟地址到物理地址的转换。
- **相联存储器实现**：TLB通常采用**全相联**或**高组相联度**实现，支持并行查找。
- **命中时**：直接从TLB获得物理页号，无需访问主存中的页表。
- **未命中时**：访问主存中的页表，取出PTE并更新TLB。
- **408重点**：TLB+Cache+主存三级访问时间计算。

## 原理分析

### TLB的基本结构

TLB的每一项包含：

| 字段 | 说明 |
|------|------|
| 虚页号（VPN） | 用于匹配比较 |
| 物理页号（PPN） | 命中时直接返回 |
| 有效位 | 该TLB项是否有效 |
| 保护位 | 读/写/执行权限 |
| ASID | 地址空间标识符（区分不同进程） |

### TLB的工作流程

```
虚拟地址: [VPN | Offset]
      |
      v
  查TLB（并行比较）
      |
  ┌───┴───┐
  |        |
命中      未命中
  |        |
  v        v
直接获得   查页表（主存）
PPN        |
  |        v
  |    找到PTE?
  |    ┌──┴──┐
  |   是     否
  |    |      |
  |    v      v
  |  更新   缺页中断
  |  TLB     (OS处理)
  |    |
  v    v
物理地址: [PPN | Offset]
```

### TLB的组织方式

**全相联TLB**：
- VPN与所有TLB项并行比较
- 查找速度最快但硬件复杂
- 适用于小容量TLB（如32~128项）

**组相联TLB**：
- 用VPN的一部分选择组，组内并行比较
- 容量更大时更实用
- 如4路组相联，64组TLB

### TLB与Cache的协作

完整的访存流程（带TLB和Cache）：

1. **查TLB**：VPN → PPN（或缺失）
2. **形成物理地址**：PPN + Offset
3. **查Cache**：用物理地址的Tag和Index查Cache
4. **命中则返回数据，缺失则访问主存**

**并行优化**：某些设计中，TLB查表和Cache索引可以并行进行（使用虚拟索引物理标记的VIPT Cache）。

### TLB的刷新

- **进程切换时**：需要刷新TLB（不同进程的VPN可能相同）
- **ASID方案**：每个TLB项携带ASID，进程切换时不必刷新，只需切换ASID
- **全局TLB项**：内核空间的页表项标记为全局，不随进程切换刷新

## 直观理解

**TLB像"记忆增强器"**：

查页表就像查字典：每次翻译一个词都要翻字典，很慢。TLB就像你记住了最近查过的词，下次直接想起来，不用翻字典。

**记忆口诀**：
> "TLB是页表的Cache，命中秒得PPN，未命中查页表再更新TLB"

## 知识关联

### 与Cache的关系

| 对比 | TLB | Cache |
|------|-----|-------|
| 缓存对象 | 页表项（PTE） | 数据/指令 |
| 目的 | 加速地址转换 | 加速数据访问 |
| 容量 | 小（几十~几百项） | 大（几KB~几MB） |
| 相联度 | 高（全相联或高路数） | 中等（4~8路） |

### 完整访存时间

$$T_{\text{total}} = T_{\text{TLB}} + (1-H_{\text{TLB}}) \times T_{\text{页表}} + T_{\text{Cache}} + (1-H_{\text{Cache}}) \times T_{\text{mem}}$$

### 408考点

- TLB的作用和原理
- TLB+Cache+主存三级访问时间计算（高频大题）
- TLB在进程切换时的处理
- TLB命中率对整体访存时间的影响

## 易错陷阱

- TLB和Cache是**独立的**两个结构，不是同一个
- 查TLB得到的是物理页号（PPN），不是完整的物理地址（还需拼接偏移）
- TLB缺失不一定缺页（页可能在主存中只是TLB没缓存）
- TLB缺失代价比Cache缺失代价小（查页表在主存中，查数据缺失要访问磁盘）
