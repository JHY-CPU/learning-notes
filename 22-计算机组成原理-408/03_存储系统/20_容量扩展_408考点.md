# 21_容量扩展_408考点

## 核心概念

容量扩展是408大题的高频考点，常与地址分配、片选逻辑、连接图一起考。

## 原理分析

### 考点一：芯片数计算

**例题1**（408真题改编）：某计算机主存容量为256KB，按字节编址，用 $32\text{K} \times 4$ 位的SRAM芯片构成。问需要多少片？

**解答**：
- 目标：$256\text{K} \times 8$ 位（256KB按字节编址）
- 单片：$32\text{K} \times 4$ 位
- 字方向：$256\text{K} / 32\text{K} = 8$ 组
- 位方向：$8 / 4 = 2$ 片/组
- 总片数 = $8 \times 2 = 16$ 片

**例题2**：某系统有地址线A0~A19（20位），数据线D0~D7（8位），用 $64\text{K} \times 1$ 位芯片组成。

**解答**：
- 地址空间 = $2^{20} = 1\text{M}$ 个字节 → 目标：$1\text{M} \times 8$ 位
- 单片：$64\text{K} \times 1$ 位
- 字方向：$1\text{M} / 64\text{K} = 1024\text{K} / 64\text{K} = 16$ 组
- 位方向：$8 / 1 = 8$ 片/组
- 总片数 = $16 \times 8 = 128$ 片

### 考点二：地址分配

**例题3**：用 $4\text{K} \times 8$ 位ROM芯片（1片）和 $4\text{K} \times 8$ 位RAM芯片（3片）组成存储器。ROM地址0000H~0FFFH，RAM依次排列。求RAM的地址范围。

**解答**：
- ROM：0000H~0FFFH（4K）
- RAM0：1000H~1FFFH（4K）
- RAM1：2000H~2FFFH（4K）
- RAM2：3000H~3FFFH（4K）

### 考点三：片选逻辑设计

**例题4**：用 $8\text{K} \times 8$ 位芯片组成 $32\text{K} \times 8$ 位存储器，画出片选逻辑。

**解答**：
- 4片芯片，每片8K
- 芯片内地址线：A0~A12（13位）
- 片选地址线：A13、A14（2位）
- 2:4译码器输入A13A14，输出4个片选信号

| A14 | A13 | 选中芯片 | 地址范围 |
|-----|-----|---------|---------|
| 0 | 0 | 芯片0 | 0000H~1FFFH |
| 0 | 1 | 芯片1 | 2000H~3FFFH |
| 1 | 0 | 芯片2 | 4000H~5FFFH |
| 1 | 1 | 芯片3 | 6000H~7FFFH |

### 考点四：ROM和RAM混合扩展

**例题5**（大题高频）：某系统地址总线A0~A15（16位），数据总线D0~D7（8位），要求：
- ROM区：0000H~3FFFH（16KB），用 $8\text{K} \times 8$ ROM芯片
- RAM区：4000H~BFFFH（32KB），用 $8\text{K} \times 8$ RAM芯片

**解答**：

ROM需要：$16\text{K} / 8\text{K} = 2$ 片
- ROM0：0000H~1FFFH（A15A14A13 = 000）
- ROM1：2000H~3FFFH（A15A14A13 = 001）

RAM需要：$32\text{K} / 8\text{K} = 4$ 片
- RAM0：4000H~5FFFH（A15A14A13 = 010）
- RAM1：6000H~7FFFH（A15A14A13 = 011）
- RAM2：8000H~9FFFH（A15A14A13 = 100）
- RAM3：A000H~BFFFH（A15A14A13 = 101）

片选逻辑：使用3:8译码器（74LS138），A13A14A15接译码器输入。

### 考点五：存储器扩展图的画法

**画图步骤**：
1. 画出CPU的地址总线和数据总线
2. 低位地址线连接到所有芯片的地址引脚
3. 高位地址线接译码器，产生片选信号
4. 数据线根据位扩展方式连接
5. 控制信号（WE、OE）并联

## 直观理解

### 快速判断题型

```
目标单元数 ÷ 单片单元数 = 1 ？→ 纯位扩展
目标字长 ÷ 单片字长 = 1 ？→ 纯字扩展
两者都 > 1 ？→ 字位同时扩展
```

### 地址范围速算法

- $n$ 位地址线 → $2^n$ 个地址
- 4K = $2^{12}$ → 12位地址
- 8K = $2^{13}$ → 13位地址
- 16K = $2^{14}$ → 14位地址
- 32K = $2^{15}$ → 15位地址
- 64K = $2^{16}$ → 16位地址

**地址范围记忆**：每8K占用地址范围为 $2000\text{H}$（十六进制）。

## 知识关联

### 常考陷阱

1. **ROM和RAM混合时**：ROM是只读的，不能向ROM地址写数据
2. **片选信号是低电平有效**：通常记为 $\overline{CE}$，注意译码器输出也可能是低电平有效
3. **地址空间可能不连续**：中间有空洞时，需要分析每个区域
4. **"按字节编址"决定了数据线为8位**：即使目标容量用KB表示，也要确认字长
5. **地址范围的计算**：末地址 = 首地址 + 容量 - 1（容量单位为字节）
6. **不要忘记控制线**：WE、OE也是芯片引脚的重要部分
