# 49_多级Cache

## 核心概念

- **多级Cache**：现代处理器通常采用L1、L2、L3三级Cache，形成多级存储层次。
- **各级特点**：L1最小最快，L3最大最慢。各级容量、速度、相联度不同。
- **工作方式**：CPU → L1 → L2 → L3 → 主存，逐级查找。
- **哈佛结构**：L1通常分为指令Cache（I-Cache）和数据Cache（D-Cache），实现取指和访存并行。

## 原理分析

### 多级Cache的层次结构

```
CPU → L1(I-Cache + D-Cache) → L2(统一Cache) → L3(统一Cache) → 主存
```

### 各级Cache典型参数（现代x86处理器）

| 层级 | 典型容量 | 典型延迟 | 相联度 | 共享性 |
|------|---------|---------|--------|--------|
| L1 | 32~64 KB | 1~4 周期 | 8路 | 每核私有 |
| L2 | 256 KB~1 MB | 10~20 周期 | 8~16路 | 每核私有 |
| L3 | 2~64 MB | 30~50 周期 | 12~16路 | 多核共享 |

### 为什么需要多级Cache？

**单一Cache的矛盾**：
- 容量大 → 命中率高，但命中时间长
- 容量小 → 命中时间短，但命中率低

**多级Cache解决矛盾**：
- L1小而快：大多数访问命中L1，快速返回
- L2/L3大而慢：处理L1缺失，提高整体命中率

### L1分为指令Cache和数据Cache

**原因**：
- 指令流水线中，**取指**和**访存**可能同时发生
- 统一Cache可能导致取指和访存冲突（结构冒险）
- 分离后可**并行访问**，提高流水线效率

**L2/L3通常为统一Cache**：
- L2/L3访问频率较低，不需要分离
- 统一Cache容量利用率更高（指令和数据共享）

### 包含性（Inclusion）与排他性（Exclusion）

**包含性层次**：L2中包含L1的所有数据（L1是L2的子集）
- 优点：一致性维护简单
- 缺点：有效容量 = L3容量（各级重复存储）

**排他性层次**：各级数据不重复
- 优点：有效容量 = L1 + L2 + L3
- 缺点：一致性维护复杂

**408一般假设包含性**。

## 直观理解

**多级Cache像"快递网络"**：
- L1 = 小区快递柜（最近最快，容量小）
- L2 = 区域快递站（中等距离，中等容量）
- L3 = 城市分拣中心（较远，大容量）
- 主存 = 总仓库（最远，最大）

**找快递流程**：先查小区柜 → 没有去区域站 → 没有去分拣中心 → 没有从总仓库调。

## 知识关联

### 与AMAT计算的联系

$$\text{AMAT} = t_{L1} + (1-H_{L1}) \times t_{L2} + (1-H_{L1})(1-H_{L2}) \times t_{L3} + \cdots$$

### 与虚拟存储器的联系

- TLB是页表的Cache
- Cache+TLB+主存+磁盘构成完整的存储层次
- 多级Cache的思想与多级页表类似

### 与总线的联系

- L1到L2通常通过专用内部总线
- L3到主存通过系统总线
- 多核共享L3时需要Cache一致性协议（MESI等）

### 408考点

- 多级Cache的AMAT计算（最常考）
- L1分I-Cache和D-Cache的原因
- 包含性vs排他性的概念
- 各级Cache的容量/速度趋势

## 易错陷阱

- L2命中率是在L1缺失条件下的条件概率
- L1分离但L2统一是常见配置，不要混淆
- 多级Cache增大的是"整体命中率"，不是单级命中率
