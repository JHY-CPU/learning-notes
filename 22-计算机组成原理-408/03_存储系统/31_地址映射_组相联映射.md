# 31_地址映射_组相联映射

## 核心概念

- **组相联映射（Set-Associative Mapping）**：将Cache分为Q个组（Set），每个组内有R个行（Line/Way），主存块按模Q映射到特定组，组内按全相联方式存放。是直接映射和全相联映射的**折中方案**。
- **分组公式**：主存第 $j$ 块 映射到 Cache 第 $j \mod Q$ 组，其中 $Q$ 为Cache的组数。
- **组数与路数关系**：若Cache有 $L$ 行，分为 $Q$ 组，每组 $R$ 行（R路组相联），则 $L = Q \times R$。
- **常见类型**：2路组相联（2-way）、4路组相联（4-way）、8路组相联（8-way）。
- **考试重点**：
  - 给定Cache容量、块大小、路数，求组数
  - 地址字段划分：标记（Tag）+ 组号（Set Index）+ 块内偏移（Block Offset）
  - 组相联的查找过程：先用组号定位到组，再在组内并行比较所有行的标记

## 原理分析

### 工作流程

1. CPU发出地址，分为三段：**Tag | 组号 | 块内偏移**
2. 用**组号**从Cache中选中一个组（该组有R行）
3. 将**Tag**与该组内**所有R行的标记**并行比较
4. 若某行标记匹配且有效位=1，则**命中**，根据块内偏移取出数据
5. 若都不匹配，则**缺失**，需从主存调入该块

### 地址字段划分

设：
- 主存容量 $M$ 字节，每块 $B$ 字节
- Cache共 $L$ 行，$Q$ 组，$R$ 路（$L = Q \times R$）

| 字段 | 位数 | 说明 |
|------|------|------|
| 块内偏移 | $\log_2 B$ 位 | 定位块内字节 |
| 组号 | $\log_2 Q$ 位 | 定位Cache中的组 |
| 标记Tag | $\log_2(M/B) - \log_2 Q$ 位 | 组内区分不同主存块 |

$$\text{Tag位数} = \log_2\frac{M}{B} - \log_2 Q$$

### 示例：2路组相联

假设：主存32KB，Cache 1KB，每块64B，2路组相联。

- 主存块数：$32\text{KB} / 64\text{B} = 512$ 块
- Cache行数：$1\text{KB} / 64\text{B} = 16$ 行
- 组数：$16 / 2 = 8$ 组
- 块内偏移：$\log_2 64 = 6$ 位
- 组号：$\log_2 8 = 3$ 位
- 标记：$\log_2 512 - 3 = 9 - 3 = 6$ 位

地址格式：`[Tag: 6位][组号: 3位][偏移: 6位]`

## 直观理解

组相联就像一个**图书馆的分馆制度**：
- 直接映射：每本书只能放在一个固定书架（位置唯一）
- 全相联：每本书可以放在图书馆任意位置（完全自由）
- 组相联：图书馆按楼层分组（如每层一个组），每本书按编号决定放在哪一层（组号），但在该层内可以放在任意书架上（组内全相联）

**记忆技巧**：组相联 = **"组内自由，组间固定"**
- 先用组号找到"楼层"（定位组）
- 再在该层所有书架中并行找"Tag"（组内比较）

## 知识关联

- 与**直接映射**的关系：当R=1时，组相联退化为直接映射（每组1行）
- 与**全相联映射**的关系：当Q=1时，组相联退化为全相联映射（只有1个组）
- 相联度（路数R）越大，冲突缺失越少，但比较器硬件开销越大
- 408考试中，组相联映射常与**替换算法**（LRU、FIFO）结合考查
- TLB本质上就是一个全相联或高组相联度的特殊Cache

### 常考组合

- 组相联 + LRU替换 + 写回策略 → 综合计算题
- 给定访问序列，求命中率（需画出Cache状态表）

### 易错陷阱

- 混淆"行数"和"组数"：组数 = 行数 / 路数
- 计算Tag时忘记减去组号位数
- 将主存地址位数直接当作Tag位数（应减去组号和偏移位数）
- 组号和行号混淆：组号用于定位组，组内还需比较Tag
