# 69_虚拟存储_性能分析

## 核心概念
- **有效访问时间(EAT)**: 考虑TLB命中率和缺页率后的平均访存时间
- **命中率**: TLB命中率, Cache命中率
- **缺页率**: 访问时发生缺页的概率
- **性能瓶颈**: 缺页处理时间 >> 正常访存时间, 缺页率对性能影响极大

## 原理分析

### 基本EAT公式

设:
- $t_m$ = 主存访问时间
- $h$ = TLB命中率
- $f$ = 缺页率(每次访存发生缺页的概率)
- $t_{\text{TLB}}$ = 查TLB时间(通常忽略)
- $t_{\text{pf}}$ = 缺页处理时间(含磁盘I/O)

**不含Cache的情况**:

$$EAT = h \times t_m + (1-h) \times [(1-f) \times 2t_m + f \times (2t_m + t_{\text{pf}})]$$

简化理解:
- TLB命中: 访存1次
- TLB未命中但不缺页: 访存2次(查页表+查数据)
- TLB未命中且缺页: 访存2次+缺页处理时间

若忽略TLB(或TLB命中率h=1):
$$EAT = (1-f) \times t_m + f \times (t_m + t_{\text{pf}})$$

### 完整EAT公式(含TLB+Cache+缺页)

设:
- $h_t$ = TLB命中率
- $h_c$ = Cache命中率
- $f$ = 缺页率
- $t_t$ = 查TLB时间
- $t_c$ = 查Cache时间
- $t_m$ = 主存访问时间
- $t_{\text{pf}}$ = 缺页处理时间

**情况分析**:

| TLB | Cache | 缺页 | 概率 | 访问时间 |
|-----|-------|------|------|----------|
| 命中 | 命中 | - | $h_t h_c$ | $t_t + t_c$ |
| 命中 | 未命中 | 否 | $h_t(1-h_c)(1-f)$ | $t_t + t_m$ |
| 命中 | 未命中 | 是 | $h_t(1-h_c)f$ | $t_t + t_m + t_{\text{pf}}$ |
| 未命中 | 命中 | 否 | $(1-h_t)h_c(1-f)$ | $t_t + t_m + t_c$ |
| 未命中 | 命中 | 是 | $(1-h_t)h_c f$ | $t_t + t_m + t_c + t_{\text{pf}}$ |
| 未命中 | 未命中 | 否 | $(1-h_t)(1-h_c)(1-f)$ | $t_t + 2t_m$ |
| 未命中 | 未命中 | 是 | $(1-h_t)(1-h_c)f$ | $t_t + 2t_m + t_{\text{pf}}$ |

**综合EAT**:
$$EAT = \sum_{i} p_i \times t_i$$

### 简化计算(常用)

**近似公式** (忽略TLB时间):

$$EAT \approx (1-f) \times [h_c t_c + (1-h_c) t_m] + f \times [h_c t_c + (1-h_c) t_m + t_{\text{pf}}]$$

$$= [h_c t_c + (1-h_c) t_m] + f \times t_{\text{pf}}$$

**关键结论**: 缺页率 $f$ 和缺页处理时间 $t_{\text{pf}}$ 对EAT影响极大。

### 抖动(Thrashing)

当缺页过于频繁时:
- 进程大部分时间在等待页面调入调出
- CPU利用率急剧下降
- 系统吞吐量崩溃

**发生条件**: 分配的页框数 < 工作集大小

**对策**:
- 增加物理内存
- 减少并发进程数
- 使用工作集模型动态调整

## 直观理解

### 超市购物类比

- **TLB命中+Cache命中**: 你记得要买什么, 货架上也有 → 秒拿
- **TLB命中+Cache未命中**: 记得要买, 但货架没有, 去仓库找
- **缺页**: 仓库也没有, 得从外地调货 → 等好几天
- **抖动**: 你一直在等调货, 什么也买不了

### 性能数字感受

假设:
- $t_m = 100$ ns
- $t_{\text{pf}} = 10$ ms = $10^7$ ns (磁盘I/O)
- Cache命中率 = 99%

| 缺页率 | EAT | 影响 |
|--------|-----|------|
| 0% | ~101 ns | 正常 |
| 0.1% | ~101 + 10000 = 10101 ns | 慢100倍 |
| 1% | ~101 + 100000 = 100101 ns | 慢1000倍 |

**结论**: 即使1%的缺页率, 性能也可能下降3个数量级!

### 记忆口诀
> **"EAT = 命中时间 + 缺页率 × 缺页代价; 缺页代价巨大, 缺页率必须压低"**

### 常见陷阱
- 缺页率通常以**百分比**给出, 代入公式时要转换为小数
- 缺页处理时间通常比主存访问时间大**4~5个数量级**
- TLB命中率和缺页率是独立的(TLB命中≠不缺页, 但实际中两者强相关)
