# 64_三种虚拟存储对比

## 核心概念
- 三种虚拟存储管理方式: 页式、段式、段页式
- 核心区别: 划分方式、地址结构、碎片类型、共享/保护粒度

### 总览对比表

| 对比项 | 页式 | 段式 | 段页式 |
|--------|------|------|--------|
| 划分依据 | 物理等分(固定大小) | 逻辑功能(大小可变) | 先逻辑分段,段内等分页 |
| 地址维度 | 一维(页号+偏移) | 一维(段号+偏移) | 三维(段号+页号+偏移) |
| 地址空间 | 一维线性 | 二维 | 二维(段内线性) |
| 单位大小 | 固定(4KB等) | 不等 | 页固定,段不等 |
| 内部碎片 | 有(最后一页) | 无 | 有(段最后一页) |
| 外部碎片 | 无 | 有 | 无 |
| 主存分配 | 页框(连续与否无关) | 需连续空间 | 页框(连续与否无关) |
| 共享保护 | 不便(页无语义) | 方便(以段为单位) | 方便(以段为单位) |
| 查表次数 | 1次(页表) | 1次(段表) | 2次(段表+页表) |
| 硬件支持 | 页表寄存器 | 段表寄存器 | 段表+页表寄存器 |
| 实现复杂度 | 简单 | 中等 | 复杂 |

## 原理分析

### 页式虚拟存储

**地址结构**:
$$\text{虚地址} = \underbrace{\text{VPN}}_{\text{虚页号}} + \underbrace{\text{Offset}}_{\text{页内偏移}}$$

**工作方式**:
- 逻辑地址空间被等分为固定大小的页
- 物理内存被等分为相同大小的页框
- 通过页表建立VPN→PPN的映射
- 页可以存放在任意页框(非连续)

**碎片分析**:
- **内部碎片**: 最后一页可能未满, 平均浪费半页
- **外部碎片**: 无(页框大小固定, 随时可分配)

### 段式虚拟存储

**地址结构**:
$$\text{逻辑地址} = \underbrace{\text{段号}}_{S} + \underbrace{\text{段内偏移}}_{W}$$

**工作方式**:
- 按程序逻辑功能划分为大小可变的段
- 段表记录每段的基址和段长
- 地址转换: 物理地址 = 段基址 + 段内偏移
- 每段必须连续存放

**碎片分析**:
- **内部碎片**: 无(段大小按需设定)
- **外部碎片**: 有(段大小不等, 频繁分配回收后出现小空闲块)

### 段页式虚拟存储

**地址结构**:
$$\text{逻辑地址} = \underbrace{S}_{\text{段号}} + \underbrace{P}_{\text{段内页号}} + \underbrace{W}_{\text{页内偏移}}$$

**工作方式**:
- 先按逻辑分段, 段内再等分页
- 段表指向页表, 页表指向物理页框
- 兼顾段式的逻辑性和页式的内存利用率

**碎片分析**:
- **内部碎片**: 有(段内最后一页可能未满)
- **外部碎片**: 无(最终分配单位是页框)

## 直观理解

### 碎片辨析

**内部碎片**: 分配给你的空间你没用完, 浪费在"内部"
- 页式: 你申请100字节, 给你一页4KB, 浪费3996字节
- 段页式: 最后一页也有类似问题

**外部碎片**: 内存中有足够总空间, 但没有足够大的连续块
- 段式: 你申请200KB, 内存有3个100KB的空闲块分散各处, 没有一个够200KB
- 页式/段页式: 页框可以不连续, 不存在此问题

### 共享与保护

**页式不便共享的原因**:
- 一页可能包含多个逻辑单位(半页代码+半页数据)
- 共享一页意味着代码和数据一起共享, 不安全
- 不同进程的共享逻辑可能跨页边界

**段式方便共享的原因**:
- 段有语义(代码段可共享, 数据段不共享)
- 共享粒度恰好是逻辑单位
- 权限控制自然(代码段只读可执行)

**例**: 两个进程运行同一程序
- 段式: 各自代码段段表项指向同一物理段 → 共享
- 页式: 需要精确到页级的共享管理, 复杂

### 记忆口诀
> **"页式简单有内碎, 段式逻辑有外碎; 段页结合两不误, 查表次数最多累"**

## 知识关联

### 实际系统选择
| 系统 | 方式 | 原因 |
|------|------|------|
| 早期Windows | 段页式 | x86硬件支持段机制 |
| Linux | 纯页式 | 简化设计, 段基址设0 |
| 现代x86 | 页式为主 | 段机制被弱化(兼容模式) |

### 408考试要点
1. **碎片类型判断**: 页式内部碎片, 段式外部碎片
2. **查表次数**: 页式1次, 段式1次, 段页式2次(不考虑TLB)
3. **共享粒度**: 页式以页为单位(不便), 段式以段为单位(方便)
4. **地址空间维度**: 页式一维, 段式二维, 段页式三维
5. **最常用方式**: 现代系统普遍采用页式(或段页式中段基址设0退化为页式)

### 常见考法

**选择题**: "以下哪种虚拟存储管理方式会产生外部碎片?"
- 答案: 只有段式会产生外部碎片

**判断题**: "页式虚拟存储便于实现共享和保护。"
- 答案: 错误, 段式更便于共享和保护

**分析题**: "比较页式和段式, 说明各自适用场景。"
- 页式: 通用操作系统, 追求简单高效
- 段式: 有特殊共享/保护需求的系统
