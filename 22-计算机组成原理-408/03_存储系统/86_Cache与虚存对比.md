# 87_Cache与虚存对比

## 核心概念
- **Cache-主存层次**: 解决CPU与主存的速度差距, 硬件管理
- **主存-辅存层次**: 解决主存容量不足, OS+硬件共同管理
- 两者都是基于**局部性原理**的存储层次管理

### 两大存储层次对比

| 对比项 | Cache-主存 | 主存-辅存(虚拟存储) |
|--------|-----------|-------------------|
| **管理对象** | 块(Block) | 页(Page)/段(Segment) |
| **典型大小** | 64B~256B | 4KB~64KB |
| **管理主体** | 纯硬件 | OS + 硬件 |
| **对程序员** | 完全透明 | 对应用程序透明, OS可见 |
| **缺失名称** | Cache缺失(Miss) | 缺页(Page Fault) |
| **缺失代价** | 几十ns~几百ns | 几ms~十几ms(磁盘I/O) |
| **缺失处理** | 硬件自动处理 | OS中断处理(缺页中断) |
| **目的** | 提高访问速度 | 扩大存储容量 |
| **映射方式** | 直接/全相联/组相联 | 全相联(虚页可放任意页框) |
| **替换算法** | LRU/FIFO/Random | OPT/FIFO/LRU/Clock |
| **写策略** | 写回/写直达 | 写回(只写辅存在换出时) |
| **命中率影响** | 性能下降 | 严重性能下降(差4~5个数量级) |

## 原理分析

### 对象差异

**Cache管理的"块"**:
- 大小: 64B~256B
- 由硬件自动管理
- 块内数据连续存放

**虚存管理的"页"**:
- 大小: 4KB~64KB
- 由OS和硬件共同管理
- 一页内的数据在辅存中连续

### 管理主体差异

**Cache**: 完全由硬件(控制器)管理
- 程序员无法干预
- CPU自动完成块的调入、替换
- 块的划分对软件不可见

**虚拟存储**: OS管理, 硬件辅助
- OS负责: 页面置换策略、缺页处理、页表维护
- 硬件负责: 地址转换(TLB)、缺页检测(有效位)
- 程序员可以看到虚拟地址空间

### 缺失处理差异

**Cache缺失**:
```
CPU访存 → Cache未命中
→ 暂停流水线
→ 从主存取块到Cache(硬件完成)
→ 继续执行
总耗时: 几十~几百ns
```

**缺页**:
```
CPU访存 → TLB未命中 → 查页表 → 有效位=0
→ 产生缺页异常
→ OS接管 → 选择牺牲页 → 写回脏页(如有)
→ 从磁盘调入目标页 → 更新页表
→ 重新执行指令
总耗时: 几ms~十几ms
```

### 透明性差异

**Cache**: 完全透明
- 程序员写 `LOAD R1, [0x1000]`
- 不知道数据在Cache还是主存
- Cache的存在不影响程序正确性

**虚存**: 对应用层透明, OS层可见
- 程序员写 `LOAD R1, [VA]`
- 不知道物理地址
- 但OS可以干预(如 `mmap`, `mprotect`)

## 直观理解

### 仓库类比

**Cache-主存** = 快递点-仓库:
- 快递点(Cache)放热门商品(热数据)
- 仓库(主存)放所有商品
- 快递点缺货时, 快递员(硬件)自动去仓库取
- 你(程序员)根本不知道这回事

**主存-辅存** = 本市仓库-外地总仓:
- 本市仓库(主存)小但快
- 外地总仓(辅存/磁盘)大但慢
- 本市仓库缺货时, 需要打电话给外地(OS处理), 让货车运过来
- 你知道(虚拟地址), 但不需要关心具体怎么运

### 两者的联系

```
虚拟地址(VA) → TLB(页表Cache) → 物理地址(PA) → Cache(主存Cache) → 数据
                |                      |
           页表在主存中              主存
                |                      |
           辅存(磁盘)               (主存即Cache的下级)
```

TLB和Cache本质相同(都是高速缓存):
- TLB缓存页表项(地址映射)
- Cache缓存数据块(实际数据)

## 知识关联

### 408常考对比题

**Q: Cache缺失和缺页, 哪个代价更大?**
A: 缺页代价大得多(Cache缺失~100ns, 缺页~10ms, 差5个数量级)

**Q: 为什么Cache用组相联, 虚存用全相联?**
A: Cache缺失代价低, 可以接受一定冲突; 虚存缺失代价极高, 必须充分利用主存

**Q: 为什么虚存用写回, Cache可用写直达?**
A: 辅存I/O代价极高, 写直达不可行; Cache写直达配合写缓冲可行

### 设计哲学对比
- Cache: 用小的硬件开销换大的速度提升, 缺失可容忍
- 虚存: 用大的磁盘空间扩展主存容量, 缺页必须尽量避免
