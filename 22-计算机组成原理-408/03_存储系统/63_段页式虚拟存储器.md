# 63_段页式虚拟存储器

## 核心概念
- **段页式虚拟存储**: 先按逻辑分段, 段内再按固定大小分页
- **二维地址结构**: 段号 + 段内页号 + 页内偏移
- **两级查表**: 先查段表得到页表基址, 再查页表得到物理页号
- **综合了段式和页式的优点**: 逻辑分段便于共享保护, 分页消除外部碎片

### 逻辑地址结构
$$\text{逻辑地址} = \underbrace{S}_{\text{段号}} + \underbrace{P}_{\text{段内页号}} + \underbrace{W}_{\text{页内偏移}}$$

| 字段 | 作用 | 位数计算 |
|------|------|----------|
| 段号S | 索引段表 | $\log_2(\text{最大段数})$ |
| 页号P | 索引页表 | $\log_2(\text{段最大页数})$ |
| 偏移W | 页内偏移 | $\log_2(\text{页大小})$ |

### 数据结构
- **段表**: 每个进程一张, 每项指向一张页表
- **页表**: 每个段一张, 每项指向一个物理页框

```
段表                    页表(段0)              页表(段1)
┌──────────┐           ┌──────────┐          ┌──────────┐
│段0:页表基址│──────────>│页0:物理页号│          │页0:物理页号│
├──────────┤           ├──────────┤          ├──────────┤
│段1:页表基址│─────┐    │页1:物理页号│          │页1:物理页号│
├──────────┤     │    ├──────────┤          └──────────┘
│段2:页表基址│     │    │页2:物理页号│
└──────────┘     │    └──────────┘
                 └──-> (页表)
```

## 原理分析

### 地址转换流程

```
逻辑地址: | S | P | W |
            |   |   |
            v   |   |
        查段表   |   |
        第S项    |   |
         / \     |   |
      有效 缺段  |   |
       |    |    |   |
       v    v    |   |
    页表基址     |   |
    + P×项大小   |   |
       |         |   |
       v         |   |
    查页表第P项   |   |
       / \       |   |
    有效  缺页   |   |
     |     |     |   |
     v     v     |   |
  物理页号PPN    |   |
     |           |   |
     v           v   v
  物理地址 = PPN×页大小 + W
```

**三次访存问题**:
1. 查段表: 访问主存1次
2. 查页表: 访问主存1次
3. 访问目标数据: 访问主存1次

**共需3次主存访问!** (不考虑TLB和Cache)

**引入TLB后**:
- TLB命中: 只需1次(直接访问数据)
- TLB未命中: 最多3次(段表+页表+数据)

### 有效访问时间计算

设:
- TLB命中率 $h$
- TLB查表时间 $t_{\text{TLB}}$
- 主存访问时间 $t_m$

$$EAT = h \times (t_{\text{TLB}} + t_m) + (1-h) \times (t_{\text{TLB}} + 3t_m)$$

若 $t_{\text{TLB}} \approx 0$, 则:
$$EAT = h \cdot t_m + (1-h) \cdot 3t_m = (1 + 2 - 2h) \cdot t_m = (3 - 2h) \cdot t_m$$

**例**: $h = 0.98$, $t_m = 100$ ns
$$EAT = (3 - 2 \times 0.98) \times 100 = 1.04 \times 100 = 104 \text{ ns}$$

### 与纯页式的对比

段页式比纯页式多一次查表(段表), 所以:
- **纯页式**: 最多2次访存(页表+数据)
- **段页式**: 最多3次访存(段表+页表+数据)

## 直观理解

**大图书馆类比**:
- **段号** = 图书馆楼层号(1楼=文学, 2楼=科技...)
- **页号** = 该楼层的书架号
- **页内偏移** = 书架上的第几本书
- **段表** = 一楼大厅的楼层索引牌
- **页表** = 每层楼的书架分布图

找书流程: 大厅查楼层索引(段表) → 到楼层查书架图(页表) → 到书架拿书(数据)

### 记忆要点
> **"先段后页, 两级查表; 段表在前, 页表在后; 三次访存, TLB优化"**

## 知识关联

### 实际系统中的应用
- **x86架构**: 支持段页式管理
  - 逻辑地址 → (段选择子, 偏移) → 线性地址 → 页式转换 → 物理地址
  - 现代OS通常将段基址设为0, 实质上等同于纯页式
- **Linux**: 主要使用纯页式, 段机制被弱化

### 段页式的优缺点

**优点**:
- 兼具段式的逻辑清晰和页式的内存利用率
- 便于实现共享(多个进程共享同一个代码段, 各自页表指向同一物理页)
- 便于保护(段级权限控制)
- 无外部碎片(页是等大的)

**缺点**:
- 需要更多存储空间(段表+多张页表)
- 地址转换更复杂, 访问次数更多(最多3次)
- 实现成本高

### 常考问题

1. **给定逻辑地址求物理地址**: 按S查段表, 再按P查页表, 最后+W
2. **计算表大小**: 段表大小 + 所有页表大小之和
3. **共享实现**: 不同进程的段表项指向同一张页表

### 易错点
- 段页式的逻辑地址是**三维**的(S, P, W), 不是二维
- 段表项存的是**页表基址**, 不是物理基址(与纯段式的区别)
- 共享以**段**为单位, 保护也以**段**为单位
