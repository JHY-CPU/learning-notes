# 06_存储系统_层次结构原理

## 核心概念

存储系统采用层次结构的根本目的是：在有限的成本下，同时获得**大容量**和**高速度**。

### 两大层次

| 特征 | Cache-主存层次 | 主存-辅存层次 |
|------|--------------|--------------|
| **解决的问题** | 速度不匹配 | 容量不够 |
| **实现方式** | 硬件（Cache控制器） | 操作系统+硬件（虚存管理） |
| **CPU可见性** | CPU可以直接访问Cache | CPU不能直接访问辅存 |
| **程序员透明性** | 对**程序员**透明（硬件自动管理） | 对**应用程序员**透明（OS管理） |
| **传送单位** | 块（Block） | 页（Page）或段（Segment） |
| **访问速度比** | Cache快5~10倍 | 主存快 $10^5$ 倍 |

> **考试重点**：两个层次的对比是408高频选择题/简答题。

### 层次结构的理论基础

**局部性原理**使得层次结构可行：

1. **时间局部性**：刚访问过的内容很快会被再次访问
   - Cache保留最近使用的数据块
   - 虚存中"热页面"留在主存

2. **空间局部性**：访问某个位置后，附近位置很快也会被访问
   - Cache按"块"为单位从主存调入数据（一个块包含多个字）
   - 虚存按"页"为单位从辅存调入数据

### 命中与不命中

**命中（Hit）**：CPU要访问的数据在上层存储器中已存在
- Cache命中：数据在Cache中，直接读取，速度很快

**未命中/缺失（Miss）**：CPU要访问的数据不在上层存储器中
- Cache未命中：需要从主存调入数据块到Cache，然后再读取

**命中率**：

$$H = \frac{\text{命中次数}}{\text{总访问次数}}$$

**平均访问时间**：

$$t = H \times t_{\text{上层}} + (1-H) \times (t_{\text{上层}} + t_{\text{下层}})$$

或等价写法：

$$t = H \times t_{\text{上层}} + (1-H) \times t_{\text{下层}}$$

（取决于 $t_{\text{下层}}$ 是否包含 $t_{\text{上层}}$，题目中需明确）

## 原理分析

### Cache-主存层次的工作流程

```
CPU发出地址
    │
    ▼
查Cache（硬件并行查找）
    │
   ┌┴┐
  命中？──是──→ 直接从Cache读取/写入（快！）
    │
    否
    │
    ▼
访问主存，将数据块调入Cache
    │
    ▼
CPU从Cache获取数据
```

### 主存-辅存层次的工作流程

```
CPU发出虚地址
    │
    ▼
地址变换（MMU/TLB）
    │
   ┌┴┐
  主存中有？──是──→ 从主存读取
    │
    否（缺页）
    │
    ▼
OS调度：从辅存调页到主存（可能替换某页）
    │
    ▼
CPU从主存获取数据
```

### 块/页大小的选择

**块/页越大**：
- 优点：空间局部性利用更好，命中率可能提高
- 缺点：调入调出时间增加，内部碎片增加

**块/页越小**：
- 优点：调入调出灵活，减少内部碎片
- 缺点：管理开销增大，空间局部性利用不充分

Cache块大小通常为 32B~128B，虚存页大小通常为 4KB。

### 替换策略（简介）

当Cache/主存满了，需要决定替换哪个块/页：

| 策略 | Cache常用 | 虚存常用 |
|------|----------|---------|
| **LRU**（最近最少使用） | 是 | 是 |
| **FIFO**（先进先出） | 少用 | 少用 |
| **LFU**（最不经常使用） | 少用 | 是 |
| **随机** | 是（简单） | 少用 |

## 直观理解

**衣柜类比**：
- **辅存** = 大仓库（存所有衣服，但去仓库拿衣服要走很远）
- **主存** = 衣柜（当季衣服放这里，拿取方便）
- **Cache** = 床头小架子（今天要穿的几件，伸手就拿）

**层次结构的智慧**：
- 你不需要把仓库里所有衣服都搬到床头
- 你只需要确保"最近要穿的"在床头，"最近不穿但常用的"在衣柜
- 仓库虽然大且远，但只有极少数时候才需要去翻找

### 易错陷阱

1. **"对CPU透明"的准确含义**：
   - Cache-主存：对**程序员**透明，硬件自动管理
   - 主存-辅存：对**应用程序员**透明，OS和硬件共同管理
   - 不能说"对CPU透明"——CPU是执行者，它"知道"Cache

2. **虚存中辅存不是主存的"扩展"**：
   - 辅存中的数据不能被CPU直接寻址
   - 必须先调入主存，由CPU按主存地址访问

3. **Cache缺失的代价远小于虚存缺页**：
   - Cache缺失：从主存调块，几十到几百ns
   - 虚存缺页：从磁盘调页，几到几十ms（差 $10^5$ 倍）

4. **命中率的重要性**：
   - $H = 0.9$ 和 $H = 0.99$ 的性能差距可能非常大
   - 命中率每提高一点，平均访问时间的改善是指数级的
