# 60_页式地址转换_计算详解

## 核心概念

- **408计算重点**：给定虚拟地址、页表内容、TLB内容，求物理地址的完整过程。
- **解题关键**：正确划分虚页号和偏移，查TLB/页表得到物理页号，拼接成物理地址。
- **常见变式**：带TLB的转换、缺页处理、多级页表转换。

## 原理分析

### 例题一：基本页式地址转换

**题目**：某系统虚拟地址16位，物理地址14位，页面大小1KB。当前进程页表如下：

| 虚页号 | 物理页号 | 存在位 |
|--------|---------|--------|
| 0 | 5 | 1 |
| 1 | 3 | 1 |
| 2 | - | 0 |
| 3 | 7 | 1 |
| 4 | 2 | 1 |
| 5 | - | 0 |
| 6 | 1 | 1 |
| 7 | 4 | 1 |

给定虚拟地址 0x3A5C，求物理地址。

**解答**：

**Step 1：划分虚拟地址**
- 页面1KB = $2^{10}$，偏移 = 10位
- VPN = 16 - 10 = 6位
- 0x3A5C = 0011 1010 0101 1100

VPN = 高6位 = 001110 = 0x0E = 14（十进制）
Offset = 低10位 = 10 0101 1100 = 0x25C

**Step 2：查页表**

VPN = 14，但页表只列出0~7。按6位VPN，最大虚页数 = $2^6 = 64$。

（题目简化，假设VPN = 0x3A的高6位。让我重新划分。）

0x3A5C = 0011 1010 0101 1100
- 低10位（Offset）：10 0101 1100 = 0x15C（取低10位 = 348十进制）
  实际：01 0101 1100 = 0x15C = 348

等一下，让我仔细算：0x3A5C 的二进制 = 0011 1010 0101 1100

低10位 = 10 0101 1100 = 确切是：
取 bits[9:0]：从右数10位 = 10 0101 1100 = 0x15C = 348

不对，让我重新来：
0x3A5C = 3×4096 + 10×256 + 5×16 + 12 = 14940
14940 的二进制 = 0011 1010 0101 1100（16位）

低10位 = 10 0101 1100... 

让我直接计算：
14940 mod 1024 = 14940 mod 1024 = 860（偏移）
14940 / 1024 = 14（虚页号）

**Step 2**：查页表，VPN = 14

但页表只列了0~7项。假设题目给的页表覆盖VPN 0~7，VPN=14需要更多项。这里假设题目的页表给出了VPN=14的映射：PPN=6, 存在位=1。

**Step 3**：形成物理地址
- PPN = 6
- PA = 6 × 1024 + 860 = 6144 + 860 = 7004 = 0x1B5C

物理地址 = 0x1B5C

### 例题二：带TLB的地址转换

**题目**：虚拟地址18位，物理地址16位，页面大小4KB。TLB为4路组相联，8组。TLB当前内容（部分）：

| 组 | 路 | VPN | PPN | 有效 |
|----|-----|-----|-----|------|
| 2 | 0 | 0x05 | 0x1A | 1 |
| 2 | 1 | 0x12 | 0x03 | 1 |
| 5 | 0 | 0x0B | 0x2F | 1 |

页表中 VPN = 0x0B 对应 PPN = 0x2F。

给定虚拟地址 0x0B3A7，求物理地址。

**解答**：

**Step 1**：页面4KB = $2^{12}$，偏移12位
- VPN = 18 - 12 = 6位
- 0x0B3A7 = 00 1011 0011 1010 0111（18位）
- VPN = 高6位 = 00 1011 = 0x0B
- Offset = 低12位 = 0011 1010 0111 = 0x3A7

**Step 2**：查TLB

TLB组数 = 8，组号 = VPN mod 8 = 0x0B mod 8 = 11 mod 8 = 3

查TLB组3...假设组3为空（TLB未命中）。

**Step 3**：查页表

VPN = 0x0B，页表给出 PPN = 0x2F，存在位=1。

**Step 4**：更新TLB，将 VPN=0x0B, PPN=0x2F 加入TLB组3。

**Step 5**：形成物理地址
- PPN = 0x2F = 0010 1111（8位）
- Offset = 0x3A7
- 物理地址 = 0x2F3A7

（物理地址16位：0x2F = 47，47×4096 + 0x3A7 = 192512 + 935 = 193447 = 0x2F3A7）

### 例题三：缺页处理

**题目**：同上设置，虚拟地址 0x058C2。

**解答**：

**Step 1**：
- VPN = 0x05，Offset = 0x8C2

**Step 2**：查TLB

组号 = 5 mod 8 = 5。TLB组5中有 VPN=0x0B 的项，但VPN=0x05不在组5中...（TLB未命中，假设）

**Step 3**：查页表

VPN = 0x05，假设页表中存在位=0（该页不在主存中）。

**Step 4**：触发缺页中断
- OS将该页从磁盘调入主存
- 假设分配物理页号 = 0x33
- 更新页表：VPN=0x05 → PPN=0x33，存在位=1
- 更新TLB
- 重新执行指令

**Step 5**：形成物理地址
- PPN = 0x33
- PA = 0x338C2

### 例题四：完整访存时间计算

**题目**：虚拟地址32位，物理地址30位，页面4KB。TLB命中率95%，TLB访问1ns。Cache命中率98%，Cache访问2ns。主存访问60ns。页表在主存中。求平均访存时间。

**解答**：

$$T = t_{\text{TLB}} + (1-H_t) \times t_{\text{mem}} + t_{\text{Cache}} + (1-H_c) \times t_{\text{mem}}$$

$$= 1 + 0.05 \times 60 + 2 + 0.02 \times 60$$

$$= 1 + 3 + 2 + 1.2 = 7.2 \text{ ns}$$

**各阶段贡献**：
- TLB查找：1ns（每次）
- 页表查找：3ns（5%的情况）
- Cache查找：2ns（每次）
- 主存访问：1.2ns（2%的情况）
- 总计：7.2ns

## 直观理解

**解题步骤总结**：

1. **分地址**：确定VPN位数和Offset位数
2. **查TLB**：先看TLB有没有缓存
3. **查页表**：TLB没有就查页表
4. **拼地址**：PPN和Offset拼成物理地址
5. **算时间**：各级命中率 × 访问时间求和

## 知识关联

### 与Cache地址映射的类比

| 步骤 | 虚拟存储器 | Cache |
|------|----------|-------|
| 分段 | VPN + Offset | Tag + Index + Offset |
| 查找 | 查TLB/页表 | 查Cache |
| 命中 | 获得PPN | 获得数据 |
| 缺失 | 查页表/缺页中断 | 访问主存 |
| 地址 | PPN + Offset = PA | 物理地址 |

### 408考试建议

- 地址转换题一定要画流程图
- 先算VPN和Offset，不要跳步
- TLB缺失不等于缺页，注意区分
- 页表在主存中，查页表需要一次主存访问
