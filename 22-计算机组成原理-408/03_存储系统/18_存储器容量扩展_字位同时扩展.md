# 19_存储器容量扩展_字位同时扩展

## 核心概念

### 字位同时扩展的定义

**字位同时扩展**：同时进行字扩展（增加存储单元数）和位扩展（增加字长），是最通用的容量扩展方式。

### 适用场景

当目标容量的**存储单元数**和**字长**都大于单片芯片时，需要字位同时扩展。

### 计算公式

$$\text{字方向组数} = \frac{\text{目标存储单元数}}{\text{单片存储单元数}}$$

$$\text{位方向每组片数} = \frac{\text{目标字长}}{\text{单片字长}}$$

$$\text{总芯片数} = \text{字方向组数} \times \text{位方向每组片数}$$

## 原理分析

### 例题：$1\text{K} \times 4$ → $4\text{K} \times 8$

**需求**：用 $1\text{K} \times 4$ 位的芯片组成 $4\text{K} \times 8$ 位的存储器。

**计算**：
- 字方向：$4\text{K} / 1\text{K} = 4$ 组
- 位方向：$8 / 4 = 2$ 片/组
- 总片数 = $4 \times 2 = 8$ 片

**地址分析**：
- $1\text{K} = 2^{10}$，芯片地址线：10位（A0~A9）
- $4\text{K} = 2^{12}$，系统地址线：12位（A0~A11）
- 高2位（A10、A11）用于片选

**连接方式**：

```
地址总线 A0~A11
    │
    ├─ A0~A9（低位10位）──┬──────┬──────┬──────┐
    │                      │      │      │      │
    │                 ┌────┴──┐┌──┴───┐│      │
    │                 │芯片0a ││芯片0b││  ...  │  ← 第0组（位扩展）
    │                 │1K×4   ││1K×4  ││      │
    │                 │(低4位) ││(高4) ││      │
    │                 └────┬──┘└──┬───┘│      │
    │                      │      │     │      │
    │                 D0~D3  D4~D7     │      │
    │                                  │      │
    │                 ┌────┴──┐┌──┴───┐│      │
    │                 │芯片1a ││芯片1b││  ...  │  ← 第1组（位扩展）
    │                 │1K×4   ││1K×4  ││      │
    │                 └────┬──┘└──┬───┘│      │
    │                      │      │     │      │
    │                     ...    ...   ...     │
    │                                          │
    └─ A10,A11 → 2:4译码器 → CE0,CE1,CE2,CE3   │
                                              │
                              （每组的片选信号并联）
```

**详细连接规则**：

**组内（位扩展）**：
- 同一组的芯片：地址线共用、数据线拼接、片选共用
- 第i组：芯片i-a接D0~D3，芯片i-b接D4~D7

**组间（字扩展）**：
- 不同组的芯片：低位地址共用、数据线并联到总线、片选独立
- A10、A11通过2:4译码器产生4个片选信号

**地址分配**：

| 组 | 芯片 | 地址范围 | 片选条件 |
|---|------|---------|---------|
| 第0组 | 0-a, 0-b | 0000H ~ 03FFH | A11A10 = 00 |
| 第1组 | 1-a, 1-b | 0400H ~ 07FFH | A11A10 = 01 |
| 第2组 | 2-a, 2-b | 0800H ~ 0BFFH | A11A10 = 10 |
| 第3组 | 3-a, 3-b | 0C00H ~ 0FFFH | A11A10 = 11 |

### 解题步骤模板

1. **计算所需芯片数**：
   - 字方向组数 = 目标单元数 / 单片单元数
   - 位方向每组片数 = 目标字长 / 单片字长
   - 总片数 = 字方向组数 × 位方向每组片数

2. **确定地址线分配**：
   - 芯片内地址线 = $\log_2$(单片存储单元数)
   - 片选地址线 = $\log_2$(字方向组数)
   - 总地址线 = 芯片内地址线 + 片选地址线

3. **画连接图**：
   - 低位地址线：所有芯片共用
   - 高位地址线：经过译码器产生片选
   - 数据线：组内拼接，组间并联到总线

## 直观理解

**图书馆类比**：
- 单片芯片 = 一个小书架（容量小，格子窄）
- 位扩展 = 把两个书架拼起来（格子变宽，放大书）
- 字扩展 = 增加书架数量（更多格子）
- 字位同时扩展 = 既增加书架数量，又把书架拼宽

**用表格理解**：

```
          位方向（数据宽度）
          ←── 4位 ──→←── 4位 ──→
         ┌──────────┬──────────┐
  字  0  │  芯片0a  │  芯片0b  │  ← 第0组
  方     ├──────────┼──────────┤
  向  1  │  芯片1a  │  芯片1b  │  ← 第1组
  (地     ├──────────┼──────────┤
  址) 2  │  芯片2a  │  芯片2b  │  ← 第2组
         ├──────────┼──────────┤
      3  │  芯片3a  │  芯片3b  │  ← 第3组
         └──────────┴──────────┘
```

每一行是一个字扩展组（同一时刻只有一行被选中），每一行内的芯片做位扩展（数据拼接）。

### 易错陷阱

1. **先位扩展后字扩展**——先在组内拼数据位，再在组间扩展地址
2. **组内芯片共用片选，组间芯片独立片选**
3. **地址线分配**：低位接芯片，高位做片选——不要搞反
4. **总片数 = 字方向组数 × 位方向每组片数**——不要遗漏任何一维
5. **数据线总宽度 = 目标字长**——组内拼接后必须等于数据总线宽度
