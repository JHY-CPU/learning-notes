# 29_地址映射_直接映射

## 核心概念

### 直接映射的定义

**直接映射（Direct Mapping）**：主存中的每个块只能映射到Cache中**唯一指定**的一行。

### 映射规则

$$\text{Cache行号} = \text{主存块号} \mod \text{Cache行数}$$

即：主存块 $j$ 只能放到 Cache 行 $j \mod m$（$m$ 为Cache总行数）。

### 地址结构

$$\text{主存地址} = \underbrace{\text{标记(Tag)}}_{\text{区分同位置不同块}} + \underbrace{\text{行号(Line)}}_{\text{Cache中的位置}} + \underbrace{\text{块内偏移(Offset)}}_{\text{块内位置}}$$

**各字段位数**：
- 块内偏移 = $\log_2(\text{块大小})$ 位
- 行号 = $\log_2(\text{Cache行数})$ 位
- 标记 = 总地址位数 - 行号位数 - 块内偏移位数

## 原理分析

### 映射过程详解

**例**：主存64KB，Cache 2KB，块大小 64B，直接映射。

- 主存地址：16位
- 块内偏移：$\log_2(64) = 6$ 位
- Cache行数：$2\text{K} / 64 = 32$ 行
- 行号：$\log_2(32) = 5$ 位
- 标记：$16 - 5 - 6 = 5$ 位

$$\text{地址} = \underbrace{5\,\text{位}}_{\text{Tag}} + \underbrace{5\,\text{位}}_{\text{行号}} + \underbrace{6\,\text{位}}_{\text{偏移}}$$

**主存块与Cache行的映射关系**：
- 主存块号0 → Cache行0（$0 \mod 32 = 0$）
- 主存块号1 → Cache行1
- ...
- 主存块号31 → Cache行31
- 主存块号32 → Cache行0（$32 \mod 32 = 0$）—— 冲突！
- 主存块号33 → Cache行1—— 冲突！

**关键**：主存块号 0、32、64、96... 都映射到 Cache行0。它们的**行号相同**，靠**标记**区分。

### 查找算法

1. 从地址中提取行号
2. 用行号定位Cache中的行
3. 检查有效位
4. 比较标记
5. 命中则用偏移读取数据；未命中则替换

### 优点

- **简单**：映射关系固定，只需比较一个标记
- **速度快**：只需一次标记比较，查找时间短
- **硬件开销小**：不需要比较器阵列

### 缺点

- **冲突率高**：多个主存块竞争同一Cache行
- **Cache利用率可能低**：即使Cache其他行空闲，冲突块也不能放

**极端情况**：如果程序交替访问映射到同一Cache行的两个块，命中率为0（抖动现象）。

### 替换策略

直接映射**不需要选择替换哪一行**——映射关系固定，只能替换该行的当前内容。

## 直观理解

**储物柜类比**：
- 你有编号0~31的32个柜子
- 规则：你的员工号 mod 32 = 你的专属柜子
- 员工0和员工32都用柜子0——只能轮流用
- 员工100用柜子4（$100 \mod 32 = 4$）

**停车场类比**：
- 32个停车位
- 车牌号 mod 32 = 指定停车位
- 牌照为0和32的车必须停同一个车位——先到先得，后来的必须替换

## 代码/模拟

### Python实现直接映射Cache

```python
"""直接映射Cache模拟 - 适用于408考研复习"""

class DirectMappedCache:
    """
    直接映射Cache模拟
    地址结构: [Tag | 行号(Line) | 块内偏移(Offset)]
    """
    def __init__(self, cache_size, block_size, main_mem_size):
        """
        :param cache_size: Cache总容量（字节）
        :param block_size: 块大小（字节）
        :param main_mem_size: 主存容量（字节）
        """
        self.num_lines = cache_size // block_size       # Cache行数
        self.offset_bits = block_size.bit_length() - 1  # 偏移位数
        self.line_bits = self.num_lines.bit_length() - 1  # 行号位数
        self.total_bits = main_mem_size.bit_length() - 1  # 总地址位数
        self.tag_bits = self.total_bits - self.line_bits - self.offset_bits

        # Cache行：每行包含 (valid, tag, data)
        self.cache = [{'valid': False, 'tag': 0} for _ in range(self.num_lines)]
        self.hits = 0
        self.misses = 0

        print(f"Cache参数: {self.num_lines}行, 偏移{self.offset_bits}位, "
              f"行号{self.line_bits}位, 标记{self.tag_bits}位")
        print(f"地址结构: [{self.tag_bits}位Tag][{self.line_bits}位行号][{self.offset_bits}位偏移]")

    def _parse_address(self, addr):
        """解析地址为Tag、行号、偏移"""
        offset = addr & ((1 << self.offset_bits) - 1)
        line = (addr >> self.offset_bits) & ((1 << self.line_bits) - 1)
        tag = addr >> (self.offset_bits + self.line_bits)
        return tag, line, offset

    def access(self, addr):
        """访问一个地址"""
        tag, line, offset = self._parse_address(addr)

        if self.cache[line]['valid'] and self.cache[line]['tag'] == tag:
            self.hits += 1
            return "命中"
        else:
            self.misses += 1
            self.cache[line] = {'valid': True, 'tag': tag}
            return "未命中(已调入)"

    def get_stats(self):
        total = self.hits + self.misses
        return f"命中{self.hits}次, 未命中{self.misses}次, 命中率={self.hits/total:.2%}"

# 示例：主存64KB，Cache 2KB，块大小64B（与笔记中例子一致）
cache = DirectMappedCache(cache_size=2048, block_size=64, main_mem_size=65536)

# 模拟访问
test_addresses = [0, 64, 128, 0, 2048, 64, 4096, 0]
for addr in test_addresses:
    tag, line, offset = cache._parse_address(addr)
    result = cache.access(addr)
    print(f"地址{addr:>5}(0x{addr:04X}): Tag={tag}, 行号={line}, 偏移={offset} → {result}")

print(f"\n统计: {cache.get_stats()}")
```

**关键点验证**：
- 地址0和地址2048都映射到行0（$2048 \div 64 \mod 32 = 0$），但Tag不同
- 第二次访问地址0时命中
- 访问2048会替换地址0所在的行

### 地址字段计算工具

```python
def calc_direct_mapping(cache_size, block_size, main_mem_size):
    """计算直接映射地址各字段位数"""
    num_lines = cache_size // block_size
    offset_bits = (block_size - 1).bit_length()
    line_bits = (num_lines - 1).bit_length()
    total_bits = (main_mem_size - 1).bit_length()
    tag_bits = total_bits - line_bits - offset_bits

    print(f"主存={main_mem_size}B, Cache={cache_size}B, 块大小={block_size}B")
    print(f"Cache行数 = {cache_size}/{block_size} = {num_lines}")
    print(f"总地址位数 = log2({main_mem_size}) = {total_bits}")
    print(f"偏移位数 = log2({block_size}) = {offset_bits}")
    print(f"行号位数 = log2({num_lines}) = {line_bits}")
    print(f"标记位数 = {total_bits} - {line_bits} - {offset_bits} = {tag_bits}")
    return tag_bits, line_bits, offset_bits

print("=== 笔记示例验算 ===")
calc_direct_mapping(2048, 64, 65536)
```

### 易错陷阱

1. **标记位数 = 总地址位数 - 行号位数 - 偏移位数**：不要算错
2. **行号 = 块号 mod 行数**：是取模不是整除
3. **直接映射不需要替换算法**——映射关系是固定的
4. **有效位为0时，即使标记匹配也是未命中**
5. **标记比较是确定命中的关键步骤**——行号只是定位，标记才是确认
6. **主存块号和Cache行号是不同概念**——块号是全局的，行号是Cache内的
