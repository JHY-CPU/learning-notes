# 59_页式地址转换

## 核心概念

- **地址转换**：将CPU发出的虚拟地址（VA）转换为主存的物理地址（PA）的过程。
- **转换流程**：虚拟地址 → (查TLB → 物理页号) 或 (查页表 → 物理页号) → 物理地址 = 物理页号 + 页内偏移。
- **TLB优先**：先查TLB，命中则快速得到PPN；TLB未命中才查页表，查完后更新TLB。
- **408重点**：地址转换的完整流程、各级访问的时序、给定虚拟地址求物理地址。

## 原理分析

### 虚拟地址格式

$$\text{虚拟地址} = \underbrace{\text{虚页号(VPN)}}_{高位} | \underbrace{\text{页内偏移(Offset)}}_{低位}$$

设虚拟地址 $n$ 位，页面大小 $P = 2^k$ 字节：
- 偏移：$k$ 位
- VPN：$n - k$ 位

### 物理地址格式

$$\text{物理地址} = \underbrace{\text{物理页号(PPN)}}_{高位} | \underbrace{\text{页内偏移(Offset)}}_{低位}$$

设物理地址 $m$ 位：
- 偏移：$k$ 位（与虚拟地址相同）
- PPN：$m - k$ 位

### 完整转换流程

```
Step 1: CPU发出虚拟地址 [VPN | Offset]
            |
            v
Step 2: 用VPN查TLB
        ┌──────┴──────┐
        |              |
    TLB命中         TLB未命中
        |              |
        v              v
Step 3a: 获得PPN    Step 3b: 用PTBR找到页表基址
                         |
                         v
                    用VPN作为索引查页表
                         |
                    ┌────┴────┐
                    |          |
                存在位=1    存在位=0
                    |          |
                    v          v
              获得PPN      缺页中断
              更新TLB     (OS处理后重试)
                    |
                    v
Step 4: 物理地址 = [PPN | Offset]
            |
            v
Step 5: 用物理地址查Cache
        ┌──────┴──────┐
        |              |
    Cache命中       Cache缺失
        |              |
        v              v
Step 6a: 返回数据  Step 6b: 访问主存获得数据
                      更新Cache
                      返回数据
```

### 详细示例

**题目**：虚拟地址32位，物理地址28位，页面大小4KB。TLB全相联8项。某时刻TLB内容：

| 项 | VPN | PPN | 有效 |
|----|-----|-----|------|
| 0 | 0x00351 | 0x0A7 | 1 |
| 1 | 0x00352 | 0x0B3 | 1 |
| ... | ... | ... | ... |

给定虚拟地址 0x00351234，求物理地址。

**解答**：

1. **划分虚拟地址**：
   - 页面4KB = $2^{12}$，偏移12位
   - Offset = 低12位 = 0x234
   - VPN = 高20位 = 0x00351

2. **查TLB**：VPN = 0x00351，TLB项0匹配，PPN = 0x0A7

3. **形成物理地址**：
   - PPN = 0x0A7 = 0000 1010 0111（高16位，实际物理地址28位，PPN占16位）
   - Offset = 0x234
   - 物理地址 = 0x0A7234

（具体位拼接取决于地址位数分配）

### 时序分析

| 阶段 | 操作 | 时间 |
|------|------|------|
| TLB命中 | TLB查找 + PPN获得 | 1周期 |
| TLB缺失 | TLB查找 + 页表访问 | 1 + 50周期 |
| Cache命中 | Cache查找 + 数据返回 | 2周期 |
| Cache缺失 | Cache查找 + 主存访问 | 2 + 50周期 |

## 直观理解

**地址转换像"翻译官"**：

- 虚拟地址是"外语地址"
- TLB是"常用词汇表"——先查它，找到了直接翻译
- 页表是"完整词典"——TLB没找到就查词典
- 物理地址是"翻译后的本地地址"
- Cache是"快递柜"——用本地地址去取数据

## 知识关联

### 与Cache地址映射的对比

| 对比 | Cache映射 | 虚拟存储器映射 |
|------|----------|-------------|
| 映射对象 | 主存块 → Cache行 | 虚拟页 → 物理页 |
| 查找加速 | Cache硬件 | TLB |
| 块/页大小 | 64B~256B | 4KB~64KB |
| 缺失代价 | 主存访问 | 磁盘访问 |

## 代码/模拟

### Python模拟页式地址转换过程

```python
"""页式地址转换模拟 - 适用于408考研复习"""

class PageTableEntry:
    """页表项"""
    def __init__(self, ppn, present=True):
        self.ppn = ppn       # 物理页号
        self.present = present  # 存在位（有效位）

class AddressTranslator:
    """虚拟地址到物理地址的转换器"""

    def __init__(self, virtual_bits=32, physical_bits=28, page_size=4096):
        """
        :param virtual_bits: 虚拟地址位数
        :param physical_bits: 物理地址位数
        :param page_size: 页面大小（字节）
        """
        self.page_size = page_size
        self.offset_bits = page_size.bit_length() - 1  # 页内偏移位数
        self.vpn_bits = virtual_bits - self.offset_bits
        self.ppn_bits = physical_bits - self.offset_bits

        # TLB: 全相联快表 {vpn: ppn}
        self.tlb = {}
        # 页表: {vpn: PageTableEntry}
        self.page_table = {}

        print(f"虚拟地址: {virtual_bits}位, 物理地址: {physical_bits}位")
        print(f"页面大小: {page_size}B = 2^{self.offset_bits}B")
        print(f"VPN: {self.vpn_bits}位, PPN: {self.ppn_bits}位, Offset: {self.offset_bits}位")

    def setup_page_table(self, entries):
        """设置页表内容: [(vpn, ppn, present), ...]"""
        for vpn, ppn, present in entries:
            self.page_table[vpn] = PageTableEntry(ppn, present)
            if present:
                self.tlb[vpn] = ppn  # 同时加入TLB

    def translate(self, virtual_addr):
        """执行地址转换"""
        offset_mask = (1 << self.offset_bits) - 1
        offset = virtual_addr & offset_mask
        vpn = virtual_addr >> self.offset_bits

        print(f"\n虚拟地址: 0x{virtual_addr:08X}")
        print(f"  划分: VPN=0x{vpn:X}, Offset=0x{offset:X}")

        # Step 1: 查TLB
        if vpn in self.tlb:
            ppn = self.tlb[vpn]
            print(f"  TLB命中: VPN=0x{vpn:X} → PPN=0x{ppn:X}")
        else:
            print(f"  TLB未命中，查页表...")
            # Step 2: 查页表
            if vpn in self.page_table and self.page_table[vpn].present:
                ppn = self.page_table[vpn].ppn
                self.tlb[vpn] = ppn  # 更新TLB
                print(f"  页表命中: VPN=0x{vpn:X} → PPN=0x{ppn:X} (已更新TLB)")
            else:
                print(f"  缺页中断! VPN=0x{vpn:X} 不在主存中")
                return None

        # Step 3: 拼接物理地址
        physical_addr = (ppn << self.offset_bits) | offset
        print(f"  物理地址: (PPN={ppn:X} << {self.offset_bits}) | Offset={offset:X}")
        print(f"           = 0x{physical_addr:08X}")
        return physical_addr

# 与笔记中示例一致的模拟
print("=== 页式地址转换模拟 ===")
trans = AddressTranslator(virtual_bits=32, physical_bits=28, page_size=4096)

# 设置页表
trans.setup_page_table([
    (0x00351, 0x0A7, True),   # VPN → PPN
    (0x00352, 0x0B3, True),
    (0x00353, 0x0C1, True),
    (0x00400, 0x000, False),  # 缺页
])

# 测试转换
trans.translate(0x00351234)  # TLB命中
trans.translate(0x00352567)  # TLB命中
trans.translate(0x00400000)  # 缺页

# 页表项位数分析
print("\n=== 页表项位数分析 ===")
print(f"  PPN位数 = {28} - {12} = {16}位")
print(f"  页表项 ≈ PPN + 存在位 + 修改位 + ... ≈ 20位")
print(f"  虚拟空间: 2^20个页 = 1M个页表项")
print(f"  页表大小 ≈ 1M × 20bit ≈ 2.5MB")
```

### 易错陷阱

- 偏移位在虚地址和物理地址中相同
- TLB用VPN匹配，不是用完整虚拟地址
- 物理地址 = PPN拼接Offset，不是PPN + Offset（加法）
- 查页表本身需要访问主存（页表存储在主存中）
