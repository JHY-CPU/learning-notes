# 28_Cache工作原理

## 核心概念

### 主存与Cache的分块

**主存分块（Block）**：
- 主存被划分为大小相等的**块**
- 每个块包含若干连续的存储单元
- 块是主存与Cache之间数据传送的**最小单位**

**Cache分行（Line / Slot）**：
- Cache被划分为大小相等的**行**
- 每行可以存放主存的一个块
- 行的大小 = 块的大小

### 地址结构

主存地址被分为三部分：

$$\text{主存地址} = \underbrace{\text{块号}}_{\text{标记+行号}} + \underbrace{\text{块内偏移}}_{\text{行内地址}}$$

具体地：

$$\text{主存地址} = \text{标记(Tag)} + \text{行号(Line)} + \text{块内偏移(Offset)}$$

各部分的位数：
- **块内偏移**：$\log_2(\text{块大小})$ 位
- **行号**：$\log_2(\text{Cache行数})$ 位（直接映射）
- **标记**：剩余位数 = 总地址位数 - 行号位数 - 块内偏移位数

### 命中与未命中

**命中（Hit）**：CPU要访问的数据所在的块已经在Cache中
- 直接从Cache读取
- 速度快

**未命中（Miss）**：CPU要访问的数据所在的块不在Cache中
- 需要从主存读取整个块到Cache
- 然后再从Cache读取
- 速度慢

## 原理分析

### Cache查找过程

**以直接映射为例**：

CPU发出地址 $A$：
1. 用**行号**找到Cache中对应的行
2. 比较该行的**标记**与地址中的标记
3. 若**标记匹配**且**有效位为1** → 命中
4. 用**块内偏移**从命中行中取出数据

```
CPU地址：|  标记  |  行号  | 块内偏移 |
             │        │         │
             │        ▼         │
             │   选中Cache行     │
             │        │         │
             ▼        │         │
        标记比较      │         │
             │        │         │
        ┌────┴────┐   │         │
        │ 标记匹配？│   │         │
        └────┬────┘   │         │
            是        │         │
             │        │         │
             ▼        ▼         ▼
           命中 → 从该行指定偏移处读取数据
```

### Cache行的结构

每行包含：
- **有效位（Valid bit）**：该行是否有有效数据
- **标记（Tag）**：用于区分映射到同一行的不同主存块
- **数据（Data）**：主存块的内容
- **脏位/修改位（Dirty bit）**：写回策略下，标记数据是否被修改

```
Cache行结构：
┌───────┬───────┬─────────────────┐
│ Valid │ Dirty │ Tag │   Data     │
│ (1bit)│ (1bit)│     │ (块大小)    │
└───────┴───────┴─────┴────────────┘
```

### 地址各部分的计算示例

**条件**：主存64KB，Cache 4KB，块大小 64B，直接映射。

- 主存地址位数：$\log_2(64\text{K}) = 16$ 位
- 块内偏移位数：$\log_2(64) = 6$ 位
- Cache行数：$4\text{K} / 64 = 64$ 行
- 行号位数：$\log_2(64) = 6$ 位
- 标记位数：$16 - 6 - 6 = 4$ 位

$$\text{地址} = \underbrace{4\,\text{位}}_{\text{标记}} + \underbrace{6\,\text{位}}_{\text{行号}} + \underbrace{6\,\text{位}}_{\text{块内偏移}}$$

### 查找过程实例

**CPU访问地址 0x3A7C**（二进制：0011 1010 0111 1100）

- 标记 = 0011（4位）
- 行号 = 101001（6位 = 第41行）
- 块内偏移 = 111100（6位 = 第60字节）

查找步骤：
1. 用行号41找到Cache第41行
2. 检查有效位：若为0 → 未命中
3. 若有效位为1，比较标记：若0011 = 0011 → 命中
4. 从第41行的第60字节处读取数据

## 直观理解

**图书馆索引类比**：
- **主存** = 大图书馆的所有书
- **Cache** = 你桌上的几本书
- **块** = 一个书架上的一个区域
- **行** = 桌上的一个位置
- **标记** = 书的编号（区分不同书）
- **行号** = 桌上位置的编号
- **块内偏移** = 翻到书的第几页

**命中** = 你想看的书正好在桌上 → 拿起来就看
**未命中** = 书不在桌上 → 去图书馆取整个区域的书放到桌上

### 易错陷阱

1. **块大小 = 行大小**：两者大小一致，只是称呼不同（主存侧叫块，Cache侧叫行）
2. **标记位数不是固定的**：取决于主存容量、Cache容量和块大小
3. **有效位很重要**：即使标记匹配，有效位为0也是未命中
4. **块内偏移位数 = $\log_2$(块大小)**，不是 $\log_2$(字节数/字数)
5. **主存块数 = 主存容量 / 块大小**，Cache行数 = Cache容量 / 块大小
6. **直接映射的行号由地址决定**：主存块 $j$ 映射到 Cache 行 $j \mod m$
