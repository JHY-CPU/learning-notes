# 78_磁盘调度_408考点

## 核心考点
- 给定磁道序列, 画出调度轨迹
- 计算总寻道距离(各算法)
- 算法对比选择题
- 饥饿与公平性分析

## 四种算法速查

| 算法 | 核心规则 | 是否有饥饿 |
|------|----------|-----------|
| FCFS | 按到达顺序 | 无 |
| SSTF | 每次找最近的 | 有 |
| SCAN | 电梯, 到头反转 | 无 |
| C-SCAN | 单向, 跳回不服务 | 无 |

## 408真题精选

### 真题1: 多算法对比 (2012年408改编)

**题目**: 磁盘请求: 55, 58, 39, 18, 90, 160, 150, 38, 184。磁头位置: 100, 方向: 向外(磁道号增大)。磁道范围: 0~199。分别用FCFS、SSTF、SCAN、C-SCAN调度, 求总寻道距离。

**FCFS**:
访问顺序: 100 → 55 → 58 → 39 → 18 → 90 → 160 → 150 → 38 → 184
$$\text{距离} = |55-100|+|58-55|+|39-58|+|18-39|+|90-18|+|160-90|+|150-160|+|38-150|+|184-38|$$
$$= 45+3+19+21+72+70+10+112+146 = 498$$

**SSTF**:
当前位置100, 最近: 90 → 58 → 55 → 39 → 38 → 18 → 150 → 160 → 184
$$\text{距离} = 10+32+3+16+1+20+132+10+24 = 250$$

**SCAN** (向外到199, 再向内):
向外: 100 → 150 → 160 → 184 → 199
向内: 199 → 90 → 58 → 55 → 39 → 38 → 18
$$\text{距离} = 50+10+24+15 + 109+32+3+16+1+20 = 280$$

**C-SCAN** (向外到199, 跳回0, 继续向外):
向外: 100 → 150 → 160 → 184 → 199
跳回: 199 → 0
继续向外: 0 → 18 → 38 → 39 → 55 → 58 → 90
$$\text{距离} = 50+10+24+15 + 18+20+1+16+3+32 = 189$$
(跳回199→0不计入)

### 真题2: SCAN方向判断 (2015年408)

**题目**: 某磁盘有200个磁道(0~199), 当前磁头在63号磁道, 正在向磁道号增加的方向移动。请求队列: 98, 183, 37, 122, 14, 124, 65, 67。用SCAN算法, 求服务序列和总移动距离。

**解**:

向外扫描(63到199): 65, 67, 98, 122, 124, 183, 199
向内扫描(199到0): 37, 14

服务序列: 63 → 65 → 67 → 98 → 122 → 124 → 183 → 199 → 37 → 14

距离:
- 向外: 2+2+31+24+2+59+16 = 136
- 向内: (199-37)+(37-14) = 162+23 = 185
- 总计: 136+185 = **321**

### 真题3: SSTF饥饿分析 (2018年408)

**题目**: 磁头在50号磁道, 请求队列: 51, 52, 195, 196, 197。用SSTF调度, 说明磁道195~197可能发生什么。

**解**:

SSTF调度:
50 → 51 (距离1) → 52 (距离1) → ...
如果持续有50~55附近的请求到来, 195/196/197永远不会被服务!

**结论**: SSTF可能导致远端请求**饥饿**。

### 真题4: 算法选择 (选择题)

**题目**: 以下哪种磁盘调度算法可能导致某些请求永远得不到服务?
A. FCFS  B. SSTF  C. SCAN  D. C-SCAN

**答案**: B (SSTF)

**分析**:
- FCFS: 严格按顺序, 无饥饿
- SSTF: 贪心, 可能饥饿
- SCAN: 电梯扫描, 无饥饿
- C-SCAN: 循环扫描, 无饥饿

---

### 真题5: 最优算法判断

**题目**: 在什么情况下FCFS的性能反而优于SSTF?

**分析**:
- 当请求很少且分散时, SSTF的优势不明显
- 当请求量很少时, FCFS的公平性没有实际性能损失
- 特殊情况: 如果请求恰好按到达顺序就是由近到远, FCFS和SSTF等价

一般情况下, SSTF的寻道距离远小于FCFS。

## 易错总结

1. **磁头初始方向**: SCAN和C-SCAN必须知道初始方向才能确定访问顺序
2. **是否到末端**: SCAN/C-SCAN扫描到物理末端还是最远请求? 看题目
3. **跳回距离**: C-SCAN的跳回是否计入总距离? 看题目
4. **SSTF的"最近"**: 是距离最近, 不是时间最近
5. **总距离计算**: 第一步从磁头当前位置到第一个请求, 最后一个请求后不需要返回
