# 53_虚拟存储器概念

## 核心概念

- **虚拟存储器（Virtual Memory）**：将主存和辅存（磁盘）统一管理，为程序员提供一个**大于实际主存容量**的逻辑地址空间。
- **核心思想**：程序不用全部装入主存即可运行，按需将用到的部分调入主存。
- **地址空间**：程序员看到的是**虚拟地址空间**（逻辑地址），实际访问的是**物理地址空间**（物理地址），两者通过**地址转换**映射。
- **对程序员透明**：虚拟存储器的管理由硬件和操作系统自动完成，程序员无需关心数据在主存还是磁盘。

## 原理分析

### 虚拟存储器的基本组成

$$\text{虚拟存储器} = \text{主存} + \text{辅存（磁盘）}$$

| 组件 | 角色 | 类比Cache |
|------|------|----------|
| 主存 | "Cache" | Cache存储器 |
| 辅存（磁盘） | "主存" | 主存储器 |
| 虚拟地址 | 程序员看到的地址 | - |
| 物理地址 | 实际主存地址 | Cache中的地址 |

### 地址空间关系

```
虚拟地址空间（大）          物理地址空间（小）
┌──────────────┐          ┌──────────┐
│  程序代码     │  ──映射──→│ 主存页0   │
│  数据         │          │ 主存页1   │
│  栈          │    ┌────→│ 主存页2   │
│  ...        │    │     │ ...      │
│  (可能比主存 │    │     └──────────┘
│   大很多)    │──映射│
│             │    │     ┌──────────┐
└──────────────┘    └────→│ 磁盘（辅存）│
                        └──────────┘
```

### 虚拟存储器解决的问题

1. **程序大于主存**：大型程序可能超过主存容量，虚拟存储器只调入需要的部分
2. **多道程序**：多个程序共享有限的主存，每个程序都认为自己独占完整的地址空间
3. **内存保护**：每个程序有独立的虚拟地址空间，互不干扰
4. **简化编程**：程序员无需关心内存布局和物理容量限制

### 与Cache的对比

| 对比维度 | Cache | 虚拟存储器 |
|---------|-------|----------|
| 管理层次 | CPU与主存之间 | 主存与磁盘之间 |
| 块/页 | 块（Block） | 页（Page） |
| 缺失 | Cache缺失 | 缺页（Page Fault） |
| 块大小 | 64B~256B | 4KB~64KB |
| 命中时间 | 纳秒级 | 纳秒级 |
| 缺失代价 | 几十~几百周期 | 几百万周期（磁盘访问） |
| 管理者 | 硬件为主 | 硬件+操作系统 |
| 替换算法 | LRU/FIFO/随机 | LRU/改进LRU |
| 映射方式 | 直接/全相联/组相联 | 页式/段式/段页式 |

### 虚拟存储器的工作过程

1. CPU发出**虚拟地址**
2. 硬件检查该虚拟地址对应的页是否在主存中（查页表）
3. **在主存中**（命中）：转换为物理地址，访问主存
4. **不在主存中**（缺页）：触发**缺页中断**，OS将该页从磁盘调入主存

## 直观理解

**虚拟存储器类比**：

- 你的**书桌**（主存）只能放5本书
- 你的**书架**（磁盘）可以放500本书
- **虚拟存储器**让你感觉书桌上能放500本书——需要哪本就从书架上拿，书桌满了就把不常用的放回书架

**对程序员的意义**：
> "我只需要写程序，不用管内存够不够。系统会自动把我需要的数据放进来。"

### 与Cache的层次关系

```
CPU → TLB → Cache → 主存 → 磁盘
```

### 与操作系统的关系

- 页表管理由操作系统负责
- 缺页中断处理由操作系统负责
- 页面置换算法部分由硬件支持
