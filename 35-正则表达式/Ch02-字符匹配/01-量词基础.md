# 量词基础

## 一、概念说明

量词（Quantifier）指定前面的字符或组应该匹配多少次。量词是正则表达式中最强大的特性之一。

```javascript
// 基本量词
/*
*     0次或多次
+     1次或多次
?     0次或1次
{n}   恰好 n 次
{n,}  至少 n 次
{n,m} n 到 m 次
*/
```

## 二、基本量词

```javascript
// * 零次或多次
/a*/.test('');               // true - 匹配 0 次
/a*/.test('aaa');            // true - 匹配多次
/a*/.test('bbb');            // true - 匹配 0 次

// + 一次或多次
/a+/.test('aaa');            // true
/a+/.test('bbb');            // false - 至少需要 1 次

// ? 零次或一次
/colou?r/.test('color');     // true - u 可有可无
/colou?r/.test('colour');    // true

// 精确次数
/\d{3}/.test('123');         // true - 恰好 3 位数字
/\d{3}/.test('12');          // false
/\d{3,}/.test('12345');      // true - 至少 3 位
/\d{3,5}/.test('12345');     // true - 3 到 5 位
```

## 三、贪婪与懒惰

```javascript
// 默认贪婪匹配（尽可能多）
const html = '<b>bold</b> and <i>italic</i>';

// 贪婪
html.match(/<.+>/);
// ['<b>bold</b> and <i>italic</i>'] - 匹配整个字符串

// 懒惰（加 ?）
html.match(/<.+?>/);
// ['<b>', '</b>', '<i>', '</i>'] - 尽可能少匹配

// 懒惰量词
/*
*?    0次或多次（懒惰）
+?    1次或多次（懒惰）
??    0次或1次（懒惰）
{n}?  恰好 n 次
{n,}? 至少 n 次（懒惰）
{n,m}? n 到 m 次（懒惰）
*/

// 提取引号内容
const text = 'he said "hello" and "world"';
text.match(/".+?"/g);  // ['"hello"', '"world"']
```

## 四、实践示例

```javascript
// 验证手机号
const isPhone = (str) => /^1[3-9]\d{9}$/.test(str);

// 验证邮箱（简化版）
const isEmail = (str) => /^\w+@\w+\.\w{2,}$/.test(str);

// 验证身份证号（18位）
const isIdCard = (str) => /^\d{17}[\dXx]$/.test(str);

// 提取 URL
const extractUrls = (text) =>
  text.match(/https?:\/\/[^\s]+/g) || [];

// 提取价格
const extractPrices = (text) =>
  text.match(/¥\d+(\.\d{2})?/g) || [];

// 验证 IP 地址（简化）
const isIPv4 = (str) =>
  /^(\d{1,3}\.){3}\d{1,3}$/.test(str);

// 提取标签内容
const extractTag = (html, tag) => {
  const regex = new RegExp(`<${tag}[^>]*>(.*?)</${tag}>`, 'g');
  return [...html.matchAll(regex)].map(m => m[1]);
};
```

## 五、注意事项

1. **贪婪导致匹配过多**：使用懒惰量词 `?` 限制匹配范围
2. **量词位置**：量词只修饰前面的一个字符或组
3. **回溯问题**：嵌套量词可能导致灾难性回溯
4. **零宽断言配合**：量词常与锚点配合使用
5. **性能影响**：懒惰量词可能比贪婪量词慢，因为需要反复尝试
