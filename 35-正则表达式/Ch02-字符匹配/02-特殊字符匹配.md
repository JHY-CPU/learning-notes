# 特殊字符匹配

## 一、概念说明

在正则表达式中，某些字符具有特殊含义（称为元字符），当需要匹配这些字符本身时需要进行转义。

```javascript
// 需要转义的特殊字符
/*
. * + ? ^ $ { } [ ] ( ) | \ /
*/
```

## 二、字符转义

```javascript
// 转义特殊字符
/\./.test('.');              // true - 匹配点号
/\*/.test('*');              // true - 匹配星号
/\+/.test('+');              // true - 匹配加号
/\?/.test('?');              // true - 匹配问号
/\^/.test('^');              // true - 匹配脱字符
/\$/.test('$');              // true - 匹配美元符
/\//.test('/');              // true - 匹配斜杠
/\\/.test('\\');             // true - 匹配反斜杠

// 匹配方括号
/\[/.test('[');              // true
/\]/.test(']');              // true

// 匹配花括号
/\{/.test('{');              // true
/\}/.test('}');              // true

// 匹配圆括号
/\(/.test('(');              // true
/\)/.test(')');              // true
```

## 三、动态构建正则

```javascript
// 转义函数（用于动态构建正则）
function escapeRegex(str) {
  return str.replace(/[.*+?^${}()|[\]\\]/g, '\\$&');
}

// 使用示例
const userInput = 'hello.world';
const escaped = escapeRegex(userInput); // 'hello\\.world'
const regex = new RegExp(escaped);
regex.test('hello.world');  // true
regex.test('helloXworld');  // false - 点号被转义了

// 在文本中搜索用户输入的关键词
function highlightText(text, keyword) {
  const escaped = escapeRegex(keyword);
  return text.replace(new RegExp(escaped, 'gi'), '<mark>$&</mark>');
}
highlightText('Price is $99.99', '$99');
// 'Price is <mark>$99</mark>.99'
```

## 四、Unicode 字符

```javascript
// Unicode 字符匹配
/中/.test('中');         // true - 使用 Unicode 编码
/中/.test('中');              // true - 直接使用汉字

// Unicode 属性（需要 u 标志）
/\p{Letter}/u.test('中');    // true
/\p{Number}/u.test('一');    // false - 中文数字不是 Number
/\p{Script=Han}/u.test('中'); // true - 汉字
/\p{Emoji}/u.test('😀');     // true

// 匹配中文字符
const isChinese = (str) => /^[一-龥]+$/.test(str);
isChinese('你好世界');        // true
isChinese('hello你好');       // false
```

## 五、实践示例

```javascript
// 匹配文件路径中的特殊字符
const path = 'C:\\Users\\test\\file.txt';
const escaped = escapeRegex(path);
// 'C:\\\\Users\\\\test\\\\file\\.txt'

// 匹配数学表达式
const expr = '2 + 3 * (4 - 1)';
const numbers = expr.match(/\d+/g);
// ['2', '3', '4', '1']

// 匹配 Markdown 链接
const md = '[link text](https://example.com)';
const links = md.match(/\[([^\]]+)\]\(([^)]+)\)/);
// ['[link text](https://example.com)', 'link text', 'https://example.com']

// 匹配 CSS 颜色值
const css = 'color: #ff0000; background: #333;';
const colors = css.match(/#[0-9a-fA-F]{3,8}/g);
// ['#ff0000', '#333']
```

## 六、注意事项

1. **遗漏转义**：动态构建正则时容易遗漏特殊字符
2. **字符类内**：在 `[]` 内，`]` `-` `^` `\` 需要转义
3. **Unicode 边界**：emoji 可能由多个 Unicode 码点组成
4. **性能**：转义后的正则仍然需要高效构建
5. **测试**：使用特殊字符的正则需要充分测试
