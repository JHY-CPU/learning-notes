# 正则性能优化

## 一、概念说明

正则表达式的性能差异可以很大，从毫秒级到灾难性的回溯（导致程序卡死）。了解正则的执行原理有助于编写高效的表达式。

```javascript
// 性能问题类型
/*
1. 灾难性回溯 - 嵌套量词导致指数级回溯
2. 过度匹配 - 匹配范围过大
3. 低效分支 - 分支顺序不合理
4. 不必要的捕获 - 创建不需要的捕获组
*/
```

## 二、避免灾难性回溯

```javascript
// 危险：嵌套量词
// /(a+)+$/ 匹配 'aaaaaaaaaaaaaaaaX' 会超时
const dangerous = /(a+)+$/;

// 安全：消除嵌套
const safe = /^a+$/;

// 危险：重叠分支
// /(a|a)+$/ 同样危险
const overlap = /(a|a)+$/;

// 安全：合并分支
const merged = /^a+$/;

// 危险：可选字符 + 量词
// /^(a?){20}a{20}$/ 匹配 'aaaaaaaaaaaaaaaaaaaa' 会卡死

// 安全：使用原子组或占有量词（部分语言支持）
// (?>a?)+  或 a?+
```

## 三、优化策略

### 3.1 具体化匹配范围

```javascript
// 低效 - 使用 . 匹配
/<.+?>/.test('<div>content</div>');

// 高效 - 使用具体字符类
/<[^>]+>/.test('<div>content</div>');

// 低效
/\w+@\w+\.\w+/.test('test@example.com');

// 高效 - 限制字符范围
/[\w.+-]+@[\w-]+\.[\w.]+/.test('test@example.com');
```

### 3.2 使用锚点

```javascript
// 低效 - 需要扫描整个字符串
/\d+/.test('hello world 123');

// 高效 - 从特定位置开始
/^\d+/.test('123 hello');

// 使用锚点加速匹配
const hasNumber = (str) => /\d/.test(str);    // 需要搜索
const startsWithNumber = (str) => /^\d/.test(str); // 直接检查
```

### 3.3 优化分支顺序

```javascript
// 将最可能匹配的分支放在前面
// 低效
/(uncommon|common)/.test('common text');

// 高效
/(common|uncommon)/.test('common text');

// 使用字符类替代简单分支
// 低效
/(a|b|c|d|e)/.test('c');

// 高效
/[abcde]/.test('c');
```

### 3.4 避免不必要捕获

```javascript
// 低效 - 创建捕获组
const matches = text.match(/(\d{4})-(\d{2})-(\d{2})/);

// 高效 - 如果不需要捕获，使用非捕获组
const matches = text.match(/(?:\d{4})-(?:\d{2})-(?:\d{2})/);

// 或使用 matchAll 只获取需要的组
const matches = [...text.matchAll(/(\d{4})-(\d{2})-(\d{2})/g)];
```

## 四、性能测试

```javascript
// 正则性能测试
function testPerformance(regex, text, iterations = 10000) {
  const start = performance.now();
  for (let i = 0; i < iterations; i++) {
    regex.test(text);
  }
  const end = performance.now();
  return (end - start).toFixed(2) + 'ms';
}

// 测试两个正则的性能
const text = 'a'.repeat(100) + 'b';
console.log(testPerformance(/a+b/, text));        // 通常较快
console.log(testPerformance(/(a+)+b/, text));     // 可能很慢

// 回溯检测
function detectBacktracking(regex, text, timeout = 1000) {
  const start = Date.now();
  const result = regex.test(text);
  const elapsed = Date.now() - start;
  if (elapsed > timeout) {
    console.warn(`正则执行时间过长: ${elapsed}ms`);
  }
  return result;
}
```

## 五、实践示例

```javascript
// 优化后的邮箱验证
const isValidEmail = (email) =>
  /^[a-zA-Z0-9.!#$%&'*+/=?^_`{|}~-]+@[a-zA-Z0-9-]+(?:\.[a-zA-Z0-9-]+)*$/.test(email);

// 优化后的 URL 提取
const extractUrls = (text) =>
  text.match(/https?:\/\/[^\s<>"{}|\\^`[\]]+/g) || [];

// 预编译正则（避免重复创建）
const EMAIL_REGEX = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
const PHONE_REGEX = /^1[3-9]\d{9}$/;

// 批量测试时重用正则对象
const testEmail = (email) => EMAIL_REGEX.test(email);
```

## 六、注意事项

1. **测试充分性**：对复杂正则进行边界测试和性能测试
2. **避免灾难性回溯**：不要嵌套量词
3. **使用具体字符类**：`[^>]` 比 `.` 更高效
4. **预编译正则**：重复使用的正则应该预编译
5. **考虑替代方案**：某些场景用字符串方法比正则更高效
