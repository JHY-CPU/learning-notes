# 断言进阶

## 一、概念说明

断言（Assertion）用于匹配位置而非字符。零宽断言检查某个位置的前后是否满足条件，但不消耗字符。

```javascript
// 断言类型
/*
(?=pattern)  正向先行断言（后面是）
(?!pattern)  负向先行断言（后面不是）
(?<=pattern) 正向后行断言（前面是）
(?<!pattern) 负向后行断言（前面不是）
*/
```

## 二、先行断言

```javascript
// 正向先行断言 (?=pattern)
// 匹配后面是 'world' 的 'hello'
/hello(?= world)/.test('hello world');  // true
/hello(?= world)/.test('hello dart');   // false

// 提取后面是数字的字母
const text = 'a1 b2 c3 d';
text.match(/\w(?=\d)/g);  // ['a', 'b', 'c']

// 负向先行断言 (?!pattern)
// 匹配后面不是 'world' 的 'hello'
/hello(?! world)/.test('hello dart');   // true
/hello(?! world)/.test('hello world');  // false

// 提取后面不是数字的字母
text.match(/\w(?!\d)/g);  // ['b', 'c', 'd']
```

## 三、后行断言

```javascript
// 正向后行断言 (?<=pattern)
// 匹配前面是 $ 的数字
/(?<=\$)\d+/.exec('价格: $100');  // ['100']

// 提取前面是 @ 的用户名
const email = 'user@example.com';
email.match(/(?<=@)\w+/);  // ['example']

// 负向后行断言 (?<!pattern)
// 匹配前面不是 $ 的数字
/(?<!\$)\d+/.exec('价格: $100 和 200');  // ['00'] (从第二个数字开始)

// 匹配前面不是数字的字母
const text = '1a2b c';
text.match(/(?<!\d)\w/g);  // ['a', 'b', ' ', 'c']
```

## 四、组合使用

```javascript
// 匹配不在 HTML 标签中的文本
const html = '<b>bold</b> and <i>italic</i>';
// 这需要更复杂的逻辑，断言可以辅助

// 密码强度检查
const strongPassword = (pwd) => {
  // 至少一个小写、一个大写、一个数字、一个特殊字符，8位以上
  return /^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)(?=.*[!@#$%^&*]).{8,}$/.test(pwd);
};
strongPassword('Abc123!@#');  // true
strongPassword('abc12345');    // false

// 提取不包含在引号中的内容
const text = 'hello "world" and \'dart\'';
// 使用断言提取未引用的内容
```

## 五、实践示例

```javascript
// 格式化数字（千分位）
const formatNumber = (num) => {
  return num.toString().replace(/\B(?=(\d{3})+(?!\d))/g, ',');
};
formatNumber(1234567);  // '1,234,567'
formatNumber(1234567890);  // '1,234,567,890'

// 提取不包含特定前缀的内容
const text = 'prefix_value other_value';
const values = text.match(/(?<!prefix_)\w+_value/g);
// ['other_value']

// 匹配后面跟指定字符的内容
const code = 'get(name) set(name) get(age)';
const getters = code.match(/get(?=\(\w+\))/g);
// ['get']

// 替换不在引号中的文本
const replaceOutsideQuotes = (text, from, to) => {
  const regex = new RegExp(`${from}(?=(?:[^"]*"[^"]*")*[^"]*$)`, 'g');
  return text.replace(regex, to);
};
replaceOutsideQuotes('foo "foo" foo', 'foo', 'bar');
// 'bar "foo" bar'
```

## 六、注意事项

1. **零宽特性**：断言不消耗字符，只是检查位置
2. **后行断言限制**：某些语言的后行断言要求固定长度
3. **性能**：复杂断言可能影响匹配性能
4. **可读性**：过多嵌套断言降低可读性，应考虑拆分
5. **浏览器兼容**：后行断言在旧版 JavaScript 中不支持
