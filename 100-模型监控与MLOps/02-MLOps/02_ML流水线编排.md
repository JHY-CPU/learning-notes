# ML流水线编排 - 模型监控与MLOps


## 1. 流水线的意义与核心价值


ML流水线将机器学习的各个步骤（数据处理、特征工程、模型训练、评估、部署）串联为可执行的有向无环图（DAG），实现自动化、可复现、可调度的ML工作流。


### 核心价值


- **可复现性：**
   相同输入产生相同输出，每次运行的环境、数据、参数完全一致
- **可调度性：**
   定时触发、事件驱动（数据更新、漂移告警）
- **可观测性：**
   每步的执行状态、耗时、资源占用可追踪
- **模块化：**
   步骤可独立测试、替换、重用
- **容错性：**
   支持失败重试、从断点恢复


> **Example:** **典型ML流水线DAG：**
>
>
> 数据获取 → 数据验证 → 数据预处理 → 特征工程 → 模型训练 → 模型评估 → 条件判断（指标达标？） → 模型注册/告警

## 3. Kubeflow Pipelines


Kubeflow Pipelines是Kubeflow的核心组件，在Kubernetes上运行ML工作流，每个步骤运行在独立的Docker容器中。


```
from kfp import dsl
from kfp.compiler import Compiler

@dsl.component(
    base_image='python:3.9',
    packages_to_install=['pandas', 'scikit-learn']
)
def data_load_op() -> str:
    """数据加载组件"""
    import pandas as pd
    from sklearn.datasets import load_iris
    X, y = load_iris(return_X_y=True)
    df = pd.DataFrame(X, columns=['f1','f2','f3','f4'])
    df['target'] = y
    df.to_csv('/tmp/data.csv', index=False)
    return '/tmp/data.csv'

@dsl.component(
    base_image='python:3.9',
    packages_to_install=['pandas', 'scikit-learn']
)
def train_op(data_path: str, n_estimators: int = 100) -> float:
    """模型训练组件"""
    import pandas as pd
    from sklearn.ensemble import RandomForestClassifier
    from sklearn.model_selection import train_test_split
    from sklearn.metrics import accuracy_score

    df = pd.read_csv(data_path)
    X = df.drop('target', axis=1).values
    y = df['target'].values
    X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

    model = RandomForestClassifier(n_estimators=n_estimators)
    model.fit(X_train, y_train)
    acc = accuracy_score(y_test, model.predict(X_test))
    return acc

@dsl.pipeline(
    name='iris-training-pipeline',
    description='鸢尾花分类训练流水线'
)
def iris_pipeline(n_estimators: int = 100):
    # 步骤1：数据加载
    load_task = data_load_op()

    # 步骤2：模型训练（依赖数据加载）
    train_task = train_op(
        data_path=load_task.output,
        n_estimators=n_estimators
    )

# 编译流水线
compiler = Compiler()
compiler.compile(iris_pipeline, 'iris_pipeline.yaml')
print("流水线编译完成，上传到Kubeflow运行")
```

## 5. 容器化部署：Docker + Kubernetes


| 层级 | 工具 | 作用 |
| --- | --- | --- |
| 容器化 | Docker | 将模型和依赖打包为标准化镜像 |
| 编排 | Kubernetes | 管理容器的部署、扩缩容、负载均衡 |
| 服务网格 | Istio / Linkerd | 流量管理、灰度发布、监控 |
| 配置管理 | Helm / Kustomize | K8s资源配置模板化 |
| 自动扩缩容 | HPA / KEDA | 根据负载自动增减Pod数量 |


### 模型服务Dockerfile示例


```
# Dockerfile for ML Model Serving
FROM python:3.9-slim

WORKDIR /app

# 安装依赖
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 复制模型和代码
COPY model/ ./model/
COPY serve.py .

# 暴露服务端口
EXPOSE 8080

# 健康检查
HEALTHCHECK --interval=30s --timeout=5s \
    CMD curl -f http://localhost:8080/health || exit 1

# 启动推理服务
CMD ["uvicorn", "serve:app", "--host", "0.0.0.0", "--port", "8080"]
```


> **Note:** **Kubernetes部署要点：**
> (1) 使用HPA根据CPU/GPU利用率自动扩缩容；
>                 (2) 设置资源请求和限制（requests/limits）避免资源争抢；
>                 (3) 使用Readiness Probe确保Pod就绪后才接收流量；
>                 (4) 模型文件使用PersistentVolume或Init Container预加载。


## 总结


- ML流水线将各步骤串联为DAG，实现自动化、可复现、可调度的ML工作流
- Kubeflow Pipelines适合K8s环境，组件化设计，每个步骤独立容器运行
- Apache Airflow是通用工作流调度器，社区庞大，适合数据和ML混合管道
- Prefect和Dagster是更现代化的替代方案，Python原生、开发体验更好
- Docker容器化保证环境一致性，Kubernetes提供部署、扩缩容和负载均衡
- 生产级ML流水线需要包含数据验证、模型评估、自动部署决策等环节
