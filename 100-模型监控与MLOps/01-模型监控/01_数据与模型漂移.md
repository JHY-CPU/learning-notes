# 数据与模型漂移 - 模型监控与MLOps


## 1. 数据漂移与概念漂移


模型上线后，由于数据分布的变化，模型性能会随时间逐渐衰退。理解漂移类型是模型监控的基础。


| 漂移类型 | 定义 | 表现 | 常见原因 |
| --- | --- | --- | --- |
| 数据漂移
（Data Drift） | P(X)发生变化，输入特征分布改变 | 特征均值、方差变化，新的特征值出现 | 用户行为变化、季节性、数据采集方式改变 |
| 概念漂移
（Concept Drift） | P(Y\|X)发生变化，输入与输出的关系改变 | 相同输入产生不同输出 | 市场环境变化、政策调整、用户偏好转移 |
| 标签漂移
（Label Drift） | P(Y)发生变化，标签分布改变 | 类别比例变化 | 业务场景变化、样本采样偏差 |
| 协变量漂移
（Covariate Shift） | 训练和推理时P(X)不同，但P(Y\|X)不变 | 特征分布偏移但模型仍可正确推理 | 数据采集端变化 |


> **Note:** **漂移的速率：**
> 漂移可以是突然的（abrupt，如节假日活动）、渐进的（gradual，如用户偏好缓慢变化）、周期性的（seasonal，如每年双11）、或增量的（incremental，新类别逐渐出现）。


## 2. 漂移检测方法


### KS检验（Kolmogorov-Smirnov Test）


非参数检验，比较两个分布是否有显著差异。适用于数值型特征。


$$
D = sup|F_ref(x) - F_cur(x)|
                其中 F_ref 为参考分布的CDF，F_cur 为当前分布的CDF
                若D > D_critical，则认为存在显著漂移
$$


### PSI（Population Stability Index）


群体稳定性指标，衡量两个分布之间的差异程度，广泛用于金融风控。


$$
PSI = Σ (p_cur_i - p_ref_i) × ln(p_cur_i / p_ref_i)
                PSI < 0.1：无显著变化
                0.1 ≤ PSI < 0.25：需要关注
                PSI ≥ 0.25：显著漂移，需要处理
$$


### 卡方检验（Chi-Square Test）


适用于类别型特征，检验观测频数与期望频数的差异。


| 检测方法 | 适用数据类型 | 优势 | 局限 |
| --- | --- | --- | --- |
| KS检验 | 数值型 | 非参数，无需假设分布 | 对样本量敏感 |
| PSI | 数值/类别（需分箱） | 直观，有明确阈值 | 依赖分箱方式 |
| 卡方检验 | 类别型 | 统计理论扎实 | 需要足够样本量 |
| Jensen-Shannon散度 | 数值/类别 | 对称，有界[0,1] | 需要概率密度估计 |
| Page-Hinkley Test | 时间序列 | 在线检测，实时性好 | 需要调参数 |


## 3. 模型性能衰退监控


除了直接监控数据分布变化，监控模型的业务指标是更直接的方式。


### 监控指标


- **预测分布：**
   模型输出概率分布的变化（如平均预测概率偏移）
- **准确性指标：**
   当有延迟标签时，监控AUC、F1、准确率等
- **延迟指标：**
   模型推理延迟的变化可能暗示计算图退化
- **异常预测比例：**
   低置信度预测的比例增加
- **特征重要性变化：**
   通过SHAP值监控特征贡献度的变化


> **Example:** **实践建议：**
> 建立基线（baseline）监控窗口，通常使用模型上线初期的稳定数据作为参考分布。比较当前窗口（如最近1天）与基线窗口的差异。当漂移检测触发时，先检查是否为已知的周期性变化（如节假日），再决定是否需要重训练。


## 4. 预警机制与自动重训练


### 预警分级


| 级别 | 触发条件 | 响应动作 |
| --- | --- | --- |
| Info（信息） | 轻微变化，PSI < 0.1 | 记录日志，继续观察 |
| Warning（警告） | 中等变化，0.1 ≤ PSI < 0.25 | 发送通知，人工审查 |
| Critical（严重） | 显著漂移，PSI ≥ 0.25 | 自动触发重训练流程 |


### 自动重训练触发条件


1. 漂移检测结果持续超过阈值（如连续3天）
2. 模型性能指标下降超过容忍范围（如AUC下降5%）
3. 积累了足够多的新标注数据
4. 定时触发（如每周/每月定期重训练）


> **Important:** 自动重训练需要谨慎：必须确保新数据的质量，设置自动验证环节（如新模型在验证集上必须优于旧模型），并考虑渐进式替换（灰度发布）而非直接切换。


## 5. Python代码：用evidently监控数据漂移


```
import pandas as pd
import numpy as np
from evidently.report import Report
from evidently.metric_preset import DataDriftPreset, DataQualityPreset
from evidently.metrics import DatasetDriftMetric

# 准备数据：参考数据（训练集）和当前数据（线上数据）
np.random.seed(42)
n_samples = 1000

# 参考数据集
reference_data = pd.DataFrame({
    'age': np.random.normal(35, 10, n_samples),
    'income': np.random.normal(50000, 15000, n_samples),
    'credit_score': np.random.normal(700, 50, n_samples),
    'category': np.random.choice(['A', 'B', 'C'], n_samples)
})

# 当前数据集（模拟数据漂移：收入均值偏移）
current_data = pd.DataFrame({
    'age': np.random.normal(35, 10, n_samples),
    'income': np.random.normal(55000, 15000, n_samples),  # 均值偏移
    'credit_score': np.random.normal(700, 50, n_samples),
    'category': np.random.choice(['A', 'B', 'C', 'D'], n_samples)  # 新类别
})

# 生成漂移报告
report = Report(metrics=[
    DataDriftPreset(),
    DataQualityPreset(),
])
report.run(reference_data=reference_data, current_data=current_data)

# 保存HTML报告
report.save_html("drift_report.html")
print("漂移报告已生成：drift_report.html")

# 编程式获取检测结果
from evidently.metric_preset import DataDriftPreset
drift_report = Report(metrics=[DataDriftPreset()])
drift_report.run(reference_data=reference_data, current_data=current_data)
result = drift_report.as_dict()

# 检查各特征是否漂移
for feature, info in result['metrics'][1]['result']['drift_by_columns'].items():
    is_drifted = info['drift_detected']
    print(f"特征 {feature}: 漂移={'是' if is_drifted else '否'}, "
          f"统计量={info.get('stattest_value', 'N/A'):.4f}")
```


## 总结


- 数据漂移（P(X)变化）和概念漂移（P(Y|X)变化）是模型性能衰退的主要原因
- 漂移检测方法包括KS检验、PSI、卡方检验等，需根据数据类型选择
- PSI是最直观的指标：小于0.1无漂移，大于0.25需立即处理
- 模型监控应结合业务指标（AUC、转化率等）和数据分布指标
- 预警分级处理：Info→Warning→Critical，对应不同响应级别
- 自动重训练需谨慎，必须有验证环节和灰度发布机制
- evidently库提供了完整的漂移检测和可视化工具
