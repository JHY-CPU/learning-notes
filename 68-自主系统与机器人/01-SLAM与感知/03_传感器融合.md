# 传感器融合


## 传感器融合


IMULiDAR卡尔曼滤波传感器标定


传感器融合结合多种传感器的优势，提供更鲁棒、更精确的位姿估计和环境感知能力。


## IMU+相机融合 (Visual-Inertial)


```
传感器互补性：
┌─────────────────────────────────────────────────────┐
│  相机 (Camera)：                                     │
│  + 丰富的环境信息                                   │
│  + 无累积漂移（有回环时）                           │
│  - 受光照、纹理影响                                 │
│  - 高频运动下模糊                                   │
│  - 单目尺度不可观                                   │
│                                                     │
│  IMU (Inertial Measurement Unit)：                  │
│  + 高频数据 (100~1000Hz)                            │
│  + 不受环境影响                                     │
│  + 提供尺度信息                                     │
│  + 快速运动时仍可用                                 │
│  - 累积漂移（加速度计偏置、陀螺仪偏置）             │
│  - 纯惯性导航不可行                                 │
└─────────────────────────────────────────────────────┘

IMU运动模型：
┌─────────────────────────────────────────────────────┐
│  加速度计测量：                                      │
│  a_m = R^T(a - g) + ba + na                         │
│  a_m：测量值，a：真实加速度，g：重力                 │
│  ba：加速度计偏置，na：高斯噪声                      │
│                                                     │
│  陀螺仪测量：                                        │
│  ω_m = ω + bg + ng                                  │
│  ω_m：测量值，ω：真实角速度                         │
│  bg：陀螺仪偏置，ng：高斯噪声                       │
│                                                     │
│  偏置模型（随机游走）：                               │
│  ba(t+1) = ba(t) + nba                              │
│  bg(t+1) = bg(t) + nbg                              │
└─────────────────────────────────────────────────────┘

紧耦合 vs 松耦合：
┌─────────────────────────────────────────────────────┐
│  松耦合 (Loosely Coupled)：                         │
│  - 各传感器独立估计位姿                             │
│  - 在状态估计层融合                                 │
│  - 实现简单，但信息利用不充分                       │
│                                                     │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐     │
│  │ 视觉里程计│    │ IMU预积分 │    │ 融合滤波  │     │
│  │ →位姿估计│    │ →位姿估计│    │ →最终位姿│     │
│  └──────────┘    └──────────┘    └──────────┘     │
│                                                     │
│  紧耦合 (Tightly Coupled)：                         │
│  - 在特征/残差层面融合                               │
│  - 构建统一的优化问题                               │
│  - 信息利用更充分，精度更高                         │
│                                                     │
│  ┌──────────────────────────────────────────┐      │
│  │  统一优化问题：                            │      │
│  │  min Σ ‖视觉残差‖² + Σ ‖IMU残差‖²       │      │
│  │  联合优化位姿、速度、偏置、地图点          │      │
│  └──────────────────────────────────────────┘      │
└─────────────────────────────────────────────────────┘

IMU预积分 (Pre-integration)：
┌─────────────────────────────────────────────────────┐
│  问题：IMU积分依赖初始状态，优化时初始状态变化       │
│  → 需要重新积分，计算量大                           │
│                                                     │
│  预积分：将积分结果表示为与初始状态无关的形式        │
│  ΔRij = ∏_{k=i}^{j-1} Exp((ωk - bg)·Δt)          │
│  Δvij = Σ ΔRik·(ak - ba)·Δt                       │
│  Δpij = Σ [Δvik·Δt + ½ΔRik·(ak - ba)·Δt²]       │
│                                                     │
│  通过一阶修正处理偏置变化：                          │
│  ΔRij(bg) ≈ ΔRij(bg0) · Exp(∂ΔR/∂bg · δbg)       │
│  → 偏置变化时只需线性修正，无需重新积分             │
│                                                     │
│  状态向量：                                          │
│  x = [R, p, v, ba, bg] (旋转、位置、速度、偏置)     │
│  维度：15维                                         │
└─────────────────────────────────────────────────────┘

VINS-Mono 流程：
1. IMU预积分：在关键帧之间预积分IMU测量
2. 视觉处理：特征提取、跟踪、三角化
3. 初始化：视觉SFM + IMU对齐（尺度、重力、偏置）
4. 滑窗优化：联合优化视觉和IMU残差
5. 回环检测+全局优化：消除累积漂移
```


## LiDAR+相机融合


```
融合动机：
┌─────────────────────────────────────────────────────┐
│  LiDAR：精确的距离测量，但缺少颜色/语义信息          │
│  相机：丰富的颜色/纹理/语义信息，但深度精度低       │
│  融合：获得精确的3D几何 + 丰富的语义信息            │
└─────────────────────────────────────────────────────┘

数据融合层级：
┌─────────────────────────────────────────────────────┐
│  1. 数据级融合 (Early Fusion)                       │
│     - 将LiDAR点云投影到图像平面                     │
│     - 在像素级别关联颜色和深度                      │
│     - 信息保留最完整，但数据对齐要求高              │
│                                                     │
│     p_image = K × [R|t] × p_lidar                  │
│     K：相机内参矩阵                                  │
│     [R|t]：LiDAR到相机的外参                        │
│                                                     │
│  2. 特征级融合 (Mid Fusion)                         │
│     - 分别提取LiDAR和图像特征                       │
│     - 在特征空间进行融合                            │
│     - 灵活性好，是当前主流方法                      │
│                                                     │
│     - PointPainting：用图像语义标注点云颜色         │
│     - PointAugmenting：用图像特征增强点云特征       │
│     - 3D proposals + 2D ROI 特征融合               │
│                                                     │
│  3. 决策级融合 (Late Fusion)                        │
│     - 各传感器独立检测/跟踪                         │
│     - 在结果层面关联和融合                          │
│     - 实现简单，但信息损失最大                      │
│                                                     │
│     - LiDAR检测框 + 相机检测框                     │
│     - 使用匈牙利算法进行关联                        │
│     - 融合置信度加权                                │
└─────────────────────────────────────────────────────┘

典型融合架构（自动驾驶）：
┌─────────────────────────────────────────────────────┐
│  LiDAR ─→ 3D点云处理 ─→ 3D候选区域 ─┐              │
│                                     ├→ 3D检测+跟踪 │
│  Camera ─→ 2D图像处理 ─→ 2D候选区域 ─┘              │
│                                                     │
│  融合策略：                                          │
│  - BEV (Bird's Eye View)融合：将LiDAR和图像特征    │
│    都投影到BEV空间进行融合                          │
│  - BEVFusion：当前SOTA方法                          │
│  - 在BEV空间中进行特征对齐和融合                    │
└─────────────────────────────────────────────────────┘

时序对齐：
- LiDAR和相机的采集频率不同（10Hz vs 30Hz）
- 需要通过时间戳进行插值对齐
- 运动补偿：利用IMU补偿扫描期间的运动畸变
```


## 卡尔曼滤波与扩展卡尔曼滤波


```
卡尔曼滤波 (Kalman Filter)：
最优线性状态估计器

系统模型：
┌─────────────────────────────────────────────────────┐
│  状态方程（预测模型）：                               │
│  x_k = F × x_{k-1} + B × u_k + w_k                │
│  w_k ~ N(0, Q)  过程噪声                            │
│                                                     │
│  观测方程（观测模型）：                               │
│  z_k = H × x_k + v_k                               │
│  v_k ~ N(0, R)  观测噪声                            │
│                                                     │
│  其中：                                              │
│  x：状态向量，F：状态转移矩阵                       │
│  u：控制输入，B：控制矩阵                           │
│  z：观测值，H：观测矩阵                             │
│  Q：过程噪声协方差，R：观测噪声协方差              │
└─────────────────────────────────────────────────────┘

KF算法（两步循环）：
┌─────────────────────────────────────────────────────┐
│  预测步 (Predict)：                                  │
│  x̂_k = F × x̂_{k-1} + B × u_k                     │
│  P̂_k = F × P_{k-1} × F^T + Q                     │
│                                                     │
│  更新步 (Update)：                                   │
│  K_k = P̂_k × H^T × (H × P̂_k × H^T + R)^(-1)    │
│  x̂_k = x̂_k + K_k × (z_k - H × x̂_k)              │
│  P_k = (I - K_k × H) × P̂_k                        │
│                                                     │
│  K_k：卡尔曼增益                                     │
│  - 预测不确定大 → K大 → 更信观测                   │
│  - 观测噪声大 → K小 → 更信预测                     │
└─────────────────────────────────────────────────────┘

扩展卡尔曼滤波 (EKF)：
处理非线性系统：
┌─────────────────────────────────────────────────────┐
│  非线性系统模型：                                     │
│  x_k = f(x_{k-1}, u_k) + w_k                       │
│  z_k = h(x_k) + v_k                                │
│                                                     │
│  EKF处理：在当前估计点进行一阶泰勒展开               │
│  F_k = ∂f/∂x |_{x=x̂_{k-1}}                        │
│  H_k = ∂h/∂x |_{x=x̂_k}                            │
│                                                     │
│  预测步：                                            │
│  x̂_k = f(x̂_{k-1}, u_k)                            │
│  P̂_k = F_k × P_{k-1} × F_k^T + Q                  │
│                                                     │
│  更新步：                                            │
│  K_k = P̂_k × H_k^T × (H_k × P̂_k × H_k^T + R)^(-1)│
│  x̂_k = x̂_k + K_k × (z_k - h(x̂_k))                │
│  P_k = (I - K_k × H_k) × P̂_k                      │
│                                                     │
│  注意：一阶近似可能引入误差                         │
│  高度非线性时考虑UKF或粒子滤波                       │
└─────────────────────────────────────────────────────┘

在SLAM中的应用：
┌─────────────────────────────────────────────────────┐
│  EKF-SLAM：                                         │
│  状态：x = [px, py, θ, l1x, l1y, l2x, l2y, ...]   │
│  预测：根据IMU/里程计运动模型                       │
│  更新：根据LiDAR/相机观测到的路标                   │
│  复杂度：O(n²)，n=路标数                            │
│                                                     │
│  IMU+EKF融合：                                       │
│  状态：x = [R, p, v, ba, bg] (15维)                │
│  预测：IMU高频预积分                                │
│  更新：视觉/GPS/低频LiDAR观测                       │
│  维护不确定性估计                                   │
└─────────────────────────────────────────────────────┘

EKF变体：
┌─────────────────────────────────────────────────────┐
│  UKF (Unscented Kalman Filter)：                    │
│  - 使用Sigma点捕获非线性                            │
│  - 不需要计算雅可比矩阵                            │
│  - 精度高于EKF                                     │
│                                                     │
│  ESKF (Error-State Kalman Filter)：                 │
│  - 在误差状态上进行滤波                             │
│  - 误差状态接近零，线性化更准确                     │
│  - 广泛用于IMU融合                                  │
│                                                     │
│  多状态约束卡尔曼滤波 (MSCKF)：                      │
│  - 视觉惯性里程计的经典方法                         │
│  - 维护滑窗内的多个相机状态                         │
│  - 将特征点从状态中消去（EKF-SLAM改进）            │
└─────────────────────────────────────────────────────┘
```


## 传感器标定


```
标定目的：确定传感器之间的相对位置和方向关系
标定精度直接影响融合系统的性能

相机内参标定：
┌─────────────────────────────────────────────────────┐
│  相机模型（针孔）：                                   │
│  p_pixel = K × [R|t] × P_world                     │
│                                                     │
│  内参矩阵K：                                         │
│  K = [fx  0  cx]                                    │
│      [0  fy  cy]                                    │
│      [0   0   1]                                    │
│  fx, fy：焦距（像素单位）                            │
│  cx, cy：主点坐标                                    │
│                                                     │
│  畸变模型：                                          │
│  径向畸变：k1, k2, k3                               │
│  切向畸变：p1, p2                                   │
│                                                     │
│  标定方法：                                          │
│  - 棋盘格标定（张正友标定法）                       │
│  - 使用多张不同角度的棋盘格图像                      │
│  - Zhang标定：15+张图像，亚像素精度                │
│  - 工具：OpenCV calibrateCamera, MATLAB Camera     │
│          Calibrator                                 │
└─────────────────────────────────────────────────────┘

LiDAR-相机外参标定：
┌─────────────────────────────────────────────────────┐
│  目标：求解 [R|t] 将LiDAR坐标系变换到相机坐标系     │
│                                                     │
│  方法一：基于标定板                                  │
│  1. 在视野中放置棋盘格/圆形标定板                   │
│  2. 相机检测角点，LiDAR检测标定板平面               │
│  3. 建立3D-3D点对应                                 │
│  4. 求解PnP或ICP得到[R|t]                          │
│                                                     │
│  方法二：无靶标自动标定                              │
│  1. 使用边缘特征对齐                                │
│     - 相机提取边缘                                  │
│     - LiDAR点云投影到图像平面，提取深度边缘         │
│     - 优化边缘对齐                                  │
│                                                     │
│  2. 基于互信息的标定                                │
│     - 最大化LiDAR强度和图像灰度的互信息             │
│     - 无需靶标，但需要丰富的纹理                    │
│                                                     │
│  方法三：在线标定                                    │
│  - 将外参作为优化变量的一部分                        │
│  - 在SLAM/里程计中同时优化外参                      │
│  - 需要足够的运动和场景变化                         │
└─────────────────────────────────────────────────────┘

IMU-相机标定：
┌─────────────────────────────────────────────────────┐
│  目标：确定IMU和相机之间的 [R|t] 和时间偏移         │
│                                                     │
│  Kalibr工具：                                        │
│  1. 准备AprilGrid标定板                             │
│  2. 手持传感器做充分激励运动（旋转+平移）           │
│  3. 收集相机图像和IMU数据                           │
│  4. 运行kalibr calibrate                            │
│  5. 输出：外参、时间偏移、IMU噪声参数              │
│                                                     │
│  关键要求：                                          │
│  - 充分的旋转运动（激励陀螺仪偏置估计）             │
│  - 充分的平移运动（激励加速度计偏置估计）           │
│  - 时间同步精度                                     │
│  - 标定板在图像中覆盖足够区域                       │
└─────────────────────────────────────────────────────┘

多传感器时间同步：
┌─────────────────────────────────────────────────────┐
│  硬件同步：                                          │
│  - PTP (IEEE 1588)：精确时间协议，微秒级同步        │
│  - GPS PPS：GPS秒脉冲同步                           │
│  - 触发信号：统一硬件触发采集                       │
│                                                     │
│  软件同步：                                          │
│  - 时间戳插值：根据时间戳对齐不同频率的数据          │
│  - 缓冲区管理：维护最近数据用于插值                 │
│  - 离线标定：测量固定时间偏移                       │
│                                                     │
│  时间偏移标定：                                       │
│  Δt = t_imu - t_camera                              │
│  影响：高速运动时导致位置误差                        │
│  典型要求：< 5ms 对齐精度                           │
└─────────────────────────────────────────────────────┘
```


> **Note:** 传感器融合的核心是互补：IMU提供高频运动信息，相机提供环境特征，LiDAR提供精确深度。卡尔曼滤波及其变体是经典融合方法，优化方法（因子图）是现代主流。标定精度是系统性能的基础。


## 融合系统选型与对比


```
典型融合架构对比：
┌──────────────┬──────────┬──────────┬──────────┬──────────┐
│ 方法          │ 传感器    │ 融合方式  │ 精度     │ 实时性    │
├──────────────┼──────────┼──────────┼──────────┼──────────┤
│ VINS-Mono    │ 单目+IMU  │ 紧耦合   │ 高       │ 实时      │
│ VINS-Fusion  │ 双目+IMU  │ 紧耦合   │ 很高     │ 实时      │
│ LIO-SAM      │ LiDAR+IMU│ 紧耦合   │ 极高     │ 实时      │
│ R3LIVE       │ LiDAR+视觉+IMU│ 紧耦合│ 极高   │ 接近实时  │
│ FAST-LIO2    │ LiDAR+IMU│ 紧耦合   │ 高       │ 实时      │
│ Cartographer │ 2D/3D LiDAR│ 回环优化│ 中等     │ 实时      │

选型建议：
┌─────────────────────────────────────────────────────┐
│  室内机器人（低成本）：                               │
│  单目/双目 + IMU → VINS系列                         │
│  理由：成本低，精度足够                              │
│                                                     │
│  室外自动驾驶：                                      │
│  LiDAR + IMU + GPS → LIO-SAM + GPS因子             │
│  理由：精度和鲁棒性要求高                            │
│                                                     │
│  无人机：                                            │
│  单目 + IMU → VINS-Mono                             │
│  理由：轻量级，速度和精度平衡                        │
│                                                     │
│  工业巡检：                                          │
│  LiDAR + 相机 → R3LIVE / FAST-LIO2                 │
│  理由：需要3D重建和精确里程计                        │
│                                                     │
│  大规模建图：                                        │
│  LiDAR + GPS + IMU → LIO-SAM                        │
│  理由：大范围不漂移                                  │
└─────────────────────────────────────────────────────┘

前沿方向：
- 4D毫米波雷达 + 相机融合
- 事件相机 + 传统相机融合（高动态范围）
- 学习型传感器融合（端到端训练）
- 多智能体协同感知
```
