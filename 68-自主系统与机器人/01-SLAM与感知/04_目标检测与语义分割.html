目标检测与语义分割

YOLOFaster R-CNNU-Net3D检测

目标检测定位和分类图像中的物体，语义分割对每个像素进行分类，是机器人感知的核心技术。

YOLO系列

YOLO (You Only Look Once)：单阶段检测器，一次前向传播完成检测 核心思想： ┌─────────────────────────────────────────────────────┐ │ 将目标检测视为回归问题： │ │ 输入图像 → CNN → 直接输出边界框和类别概率 │ │ 不需要候选区域生成（Region Proposal） │ │ 速度快，适合实时应用 │ └─────────────────────────────────────────────────────┘ YOLOv1 (2016)： ┌─────────────────────────────────────────────────────┐ │ 将图像划分为 S×S 网格 │ │ 每个网格预测 B个边界框 + C个类别概率 │ │ 输出张量：S × S × (B×5 + C) │ │ 每个边界框：[x, y, w, h, confidence] │ │ 缺点：每个网格只能预测固定数量物体，定位不精确 │ └─────────────────────────────────────────────────────┘ YOLOv3 改进： ┌─────────────────────────────────────────────────────┐ │ 1. 多尺度预测 │ │ 三个尺度的特征图：13×13, 26×26, 52×52 │ │ 检测不同大小的物体 │ │ │ │ 2. Darknet-53 backbone │ │ 残差连接的深度网络 │ │ 更强的特征提取能力 │ │ │ │ 3. 多标签分类 │ │ 使用独立的logistic分类器 │ │ 替代softmax（支持多标签） │ │ │ │ 4. K-means anchor boxes │ │ 在数据集上聚类得到先验框尺寸 │ │ 9个anchor，3个尺度各3个 │ └─────────────────────────────────────────────────────┘ YOLOv5/v8 (Ultralytics)： ┌─────────────────────────────────────────────────────┐ │ 工程化改进为主： │ │ - Focus/CSP模块：计算效率优化 │ │ - 自适应anchor计算 │ │ - Mosaic数据增强 │ │ - 自适应图片缩放 │ │ - 解耦检测头 │ │ - Anchor-free选项 │ │ │ │ YOLOv8特性： │ │ - C2f模块替代C3模块 │ │ - Anchor-free检测头 │ │ - TaskAlignedAssigner标签分配 │ │ - Distribution Focal Loss │ │ - 支持目标检测、实例分割、关键点检测 │ │ │ │ 性能 (COCO, V100)： │ │ YOLOv8n：37.3% mAP, ~1.2ms │ │ YOLOv8s：44.9% mAP, ~2.3ms │ │ YOLOv8m：50.2% mAP, ~5.3ms │ │ YOLOv8l：52.9% mAP, ~8.5ms │ │ YOLOv8x：53.9% mAP, ~12.8ms │ └─────────────────────────────────────────────────────┘ YOLOv9 / v10 / v11 最新进展： ┌─────────────────────────────────────────────────────┐ │ YOLOv9 (2024)： │ │ - PGI (Programmable Gradient Information) │ │ - GELAN (Generalized Efficient Layer Aggregation) │ │ - 解决信息瓶颈问题 │ │ │ │ YOLOv10 (2024)： │ │ - NMS-free 设计 │ │ - 双重标签分配 │ │ - 效率-精度平衡优化 │ │ │ │ YOLO11 (2024)： │ │ - C3k2模块改进 │ │ - 注意力机制集成 │ │ - 多任务统一框架 │ └─────────────────────────────────────────────────────┘

Faster R-CNN与两阶段检测器

两阶段检测器：先生成候选区域，再分类和回归 R-CNN系列演进： ┌─────────────────────────────────────────────────────┐ │ R-CNN (2014)： │ │ Selective Search → ~2000候选区域 │ │ → CNN提取特征 → SVM分类 → 边界框回归 │ │ 问题：每个候选区域独立提特征，极慢 │ │ │ │ Fast R-CNN (2015)： │ │ 整图CNN → RoI Pooling → 分类+回归 │ │ 共享卷积特征，但候选区域生成仍是独立步骤 │ │ │ │ Faster R-CNN (2016)： │ │ 用RPN (Region Proposal Network) 替代Selective Search│ │ 端到端训练，成为两阶段检测器的标杆 │ └─────────────────────────────────────────────────────┘ Faster R-CNN架构： ┌─────────────────────────────────────────────────────┐ │ 输入图像 │ │ │ │ │ ▼ │ │ ┌──────────────┐ │ │ │ Backbone │ (ResNet/VGG) │ │ │ 特征提取 │ │ │ └──────┬───────┘ │ │ │ 特征图 │ │ ├──────────────────────┐ │ │ ▼ ▼ │ │ ┌──────────────┐ ┌──────────────┐ │ │ │ RPN │ │ RoI Pooling │ │ │ │ 候选区域生成 │───→│ 特征裁剪 │ │ │ │ 前景/背景 │ └──────┬───────┘ │ │ │ anchor回归 │ │ │ │ └──────────────┘ ▼ │ │ ┌──────────────┐ │ │ │ 分类+回归头 │ │ │ │ 类别+边界框 │ │ │ └──────────────┘ │ │ │ │ RPN (Region Proposal Network)： │ │ - 在特征图每个位置生成k个anchor │ │ - 预测每个anchor的前景/背景分数 │ │ - 预测anchor到目标的偏移量 │ │ - 使用NMS筛选候选区域 │ │ - 输出Top-N候选区域（如300个） │ │ │ │ Anchor设计： │ │ - 3种尺度 × 3种比例 = 9个anchor │ │ - 尺度：128², 256², 512² 像素 │ │ - 比例：1:1, 1:2, 2:1 │ └─────────────────────────────────────────────────────┘ 损失函数： ┌─────────────────────────────────────────────────────┐ │ RPN损失： │ │ L_rpn = L_cls(anchor前景/背景) + L_reg(anchor回归) │ │ │ │ 检测头损失： │ │ L_det = L_cls(类别分类) + L_reg(边界框回归) │ │ │ │ 分类损失：交叉熵 (Cross Entropy) │ │ 回归损失：Smooth L1 Loss │ │ Smooth L1(x) = 0.5x² if |x|<1 │ │ = |x| - 0.5 otherwise │ │ │ │ 总损失：L = L_rpn + L_det │ └─────────────────────────────────────────────────────┘ 改进的两阶段检测器： - Cascade R-CNN：级联回归提高定位精度 - Mask R-CNN：增加分割分支（实例分割） - HTC (Hybrid Task Cascade)：多任务级联 - Detectron2：Facebook的检测框架

语义分割

3D目标检测

3D检测：在3D空间中定位和分类物体（自动驾驶核心） 输入模态： ┌─────────────────────────────────────────────────────┐ │ 纯LiDAR： │ │ - Point-based：PointNet++直接处理点云 │ │ - Voxel-based：体素化后3D卷积 │ │ - Pillar-based：将点云展平为伪图像 │ │ │ │ 纯相机： │ │ - 单目3D检测：从单张图像估计深度和3D框 │ │ - BEV视角：将图像特征转到鸟瞰图 │ │ │ │ 多模态融合： │ │ - LiDAR + 相机特征融合 │ │ - BEV空间融合 │ └─────────────────────────────────────────────────────┘ PointPillars (2019)： ┌─────────────────────────────────────────────────────┐ │ 将3D点云转换为2D伪图像进行处理 │ │ │ │ 流程： │ │ 1. Pillar化：将点云沿Z轴划分为柱子（Pillar） │ │ 每个pillar内的点组成一个集合 │ │ │ │ 2. 特征提取： │ │ 对每个pillar用PointNet提取特征 │ │ 输出：(C, H, W) 的伪图像 │ │ │ │ 3. 2D骨干网络： │ │ 类似于2D检测的CNN骨干 │ │ 特征金字塔（FPN）进行多尺度检测 │ │ │ │ 4. SSD检测头： │ │ 在BEV空间检测3D边界框 │ │ 输出：[x, y, z, w, l, h, θ] + 类别分数 │ │ │ │ 优势： │ │ - 速度极快（~20ms，适合实时） │ │ - 简单高效 │ │ - 可在嵌入式平台部署 │ └─────────────────────────────────────────────────────┘ CenterPoint (2021)： ┌─────────────────────────────────────────────────────┐ │ 基于中心点的3D检测 │ │ │ │ 核心思想：用热图预测物体中心，而非anchor-based │ │ │ │ 流程： │ │ 1. Backbone：提取BEV特征 │ │ 2. Center Head：预测中心点热图 │ │ 每个峰值对应一个物体 │ │ 3. 回归头：从中心点回归3D框属性 │ │ 4. （可选）第二阶段：利用3D框内点精调 │ │ │ │ 优势： │ │ - 无需anchor设计 │ │ - NMS简单（3×3最大池化） │ │ - 高精度 + 高速度 │ │ │ │ BEVFusion (2022)： │ │ - 在BEV空间融合LiDAR和图像特征 │ │ - 当前多模态3D检测SOTA │ │ - nuScenes: 72.9% NDS │ └─────────────────────────────────────────────────────┘ 3D检测输出格式： ┌─────────────────────────────────────────────────────┐ │ 3D边界框表示： │ │ - 中心坐标：(x, y, z) │ │ - 尺寸：(length, width, height) │ │ - 朝向角：θ (yaw angle) │ │ - 类别：car, pedestrian, cyclist, ... │ │ - 置信度：confidence score │ │ │ │ 评估指标： │ │ - AP (Average Precision)：不同IoU阈值下的精度 │ │ - 3D IoU：在3D空间计算交并比 │ │ - ATE (Average Translation Error) │ │ - ASE (Average Scale Error) │ │ - AOE (Average Orientation Error) │ │ - NDS (nuScenes Detection Score)：综合指标 │ └─────────────────────────────────────────────────────┘ 部署与推理优化： - TensorRT：NVIDIA推理加速 - ONNX Runtime：跨平台推理 - OpenVINO：Intel推理优化 - 模型量化：INT8量化减少计算量 - TensorRT FP16/INT8：混合精度推理
YOLO系列是实时检测的首选，Faster R-CNN精度更高。语义分割中U-Net适合小数据，DeepLab系列精度最高。3D检测中PointPillars和CenterPoint是自动驾驶的主流方案。

语义SLAM与场景理解
