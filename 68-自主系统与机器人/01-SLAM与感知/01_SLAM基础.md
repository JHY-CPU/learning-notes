# SLAM基础


## SLAM基础


机器人SLAM定位


SLAM实现机器人在未知环境中同时定位和构建地图。


## SLAM问题定义


```
SLAM (Simultaneous Localization and Mapping)：
机器人在未知环境中，边移动边构建地图，同时确定自身位置

核心问题：
- 我在哪里？（定位 Localization）
- 环境长什么样？（建图 Mapping）
- 两者互相依赖，需同时求解

传感器：
┌─────────────────────────────────────────────┐
│  里程计(Odometry)：轮式编码器/IMU            │
│  - 提供短时间精确的位姿估计                   │
│  - 存在累积漂移                               │
│                                             │
│  激光雷达(LiDAR)：2D/3D点云                  │
│  - 精确的距离测量                             │
│  - 不受光照影响                               │
│  - 成本较高                                   │
│                                             │
│  相机(Camera)：单目/双目/RGB-D/鱼眼          │
│  - 信息丰富，成本低                           │
│  - 受光照、纹理影响                           │
│  - 计算量大                                   │
│                                             │
│  GPS/RTK：全局定位                            │
│  - 室外可用，提供全局位置                     │
│  - 室内不可用，精度有限                       │
└─────────────────────────────────────────────┘
```


## EKF-SLAM


```
扩展卡尔曼滤波SLAM：
状态向量：x = [机器人位姿, 路标1, 路标2, ...]

状态空间：
x_t = [px, py, θ, l1x, l1y, l2x, l2y, ...]

EKF步骤：
1. 预测(Prediction)：
   根据运动模型预测下一状态
   x̂_t = f(x_t-1, u_t)  非线性运动模型
   P̂_t = F × P_t-1 × F^T + Q

2. 更新(Update)：
   根据观测修正状态
   K = P̂ × H^T × (H × P̂ × H^T + R)^(-1)
   x_t = x̂_t + K × (z - h(x̂_t))
   P_t = (I - K × H) × P̂

问题：
- 状态维度随路标增长（O(n²)复杂度）
- 线性化误差
- 不适合大规模环境
```


## 图优化SLAM


```
现代主流方法：将SLAM转化为图优化问题

因子图(Factor Graph)：
- 节点：机器人位姿 x1, x2, ..., xn
- 边(因子)：约束条件
  - 里程计因子：xi ↔ xi+1（运动约束）
  - 观测因子：xi ↔ lj（观测约束）
  - 回环因子：xi ↔ xj（回环检测）

优化目标：
min Σ ||h(xi, xj) - zij||²_Ω
对所有约束求残差的加权平方和

求解：Gauss-Newton / Levenberg-Marquardt
利用稀疏性高效求解

回环检测(Loop Closure)：
- 检测机器人是否回到之前的位置
- 消除累积漂移的关键
- 方法：词袋模型(BoW)、点云匹配
- 工具：DBoW2(视觉)、Scan Context(LiDAR)

工具库：
- g2o：图优化库
- GTSAM：因子图优化
- Ceres Solver：通用非线性优化
```


## 视觉与LiDAR SLAM


```
视觉SLAM (Visual SLAM)：
┌─────────────────────────────────────────────┐
│  特征法：ORB-SLAM                            │
│  - 提取ORB特征点，特征匹配                   │
│  - 三角化恢复3D点                            │
│  - BA(Bundle Adjustment)优化                 │
│                                             │
│  直接法：DSO/LSD-SLAM                        │
│  - 利用像素灰度信息                          │
│  - 最小化光度误差                            │
│  - 不需要特征提取                            │
│                                             │
│  半直接法：SVO                               │
│  - 结合特征和直接法的优点                     │
└─────────────────────────────────────────────┘

LiDAR SLAM：
┌─────────────────────────────────────────────┐
│  LOAM / LIO-SAM                              │
│  - 提取边缘和平面特征                         │
│  - 点云配准(ICP/NDT)                         │
│  - 与IMU融合(LIO)                           │
│                                             │
│  Cartographer (Google)                       │
│  - 子图(submap)概念                         │
│  - 回环检测 + 图优化                          │
│  - 2D/3D支持                                │
└─────────────────────────────────────────────┘

实际应用：
- 自动驾驶：LIO-SAM + GPS
- 室内机器人：ORB-SLAM / Cartographer
- 无人机：VINS-Mono (视觉惯性)
- AR/VR：Apple ARKit / Google ARCore
```


> **Note:** 现代SLAM系统通常使用图优化+EKF的混合方法。LiDAR精度高但贵，视觉成本低但受环境影响。
