视觉SLAM使用相机作为主要传感器，通过图像信息实现同时定位和建图。

视觉里程计 (Visual Odometry)

视觉里程计：通过连续图像帧估计相机运动 分类： ┌─────────────────────────────────────────────────────┐ │ 按使用的相机类型： │ │ - 单目 (Monocular)：单个相机，尺度不可观 │ │ - 双目 (Stereo)：两个相机，可恢复尺度 │ │ - RGB-D：深度相机，直接获取深度 │ │ - 全向相机/鱼眼：大视场角 │ │ │ │ 按处理方法： │ │ - 特征法 (Feature-based) │ │ - 直接法 (Direct) │ │ - 半直接法 (Semi-direct) │ └─────────────────────────────────────────────────────┘ 特征法VO流程： 1. 特征提取：检测图像中的特征点 2. 特征匹配：在连续帧间匹配特征点 3. 运动估计：根据匹配关系估计相机位姿 4. 地图点三角化：利用多视角恢复3D点 对极约束： ┌─────────────────────────────────────────────────────┐ │ 两帧图像中对应点满足对极约束： │ │ p2^T × E × p1 = 0 │ │ │ │ 其中： │ │ - p1, p2：归一化坐标 │ │ - E = t×R：本质矩阵 │ │ - R, t：两帧间的旋转和平移 │ │ │ │ 求解方法： │ │ - 八点法 (Eight-point Algorithm) │ │ - 五点法 (Five-point Algorithm，单目) │ │ - RANSAC：鲁棒估计，剔除外点 │ │ │ │ PnP (Perspective-n-Point)： │ │ 已知3D-2D对应，估计相机位姿 │ │ 最少需要3对点（P3P），通常使用EPnP/DLS │ └─────────────────────────────────────────────────────┘ 直接法VO： ┌─────────────────────────────────────────────────────┐ │ 不提取特征点，直接利用像素灰度信息 │ │ │ │ 光度误差 (Photometric Error)： │ │ E(R,t) = Σ |I1(p1) - I2(p2)|² │ │ 其中 p2 = π(R × π^(-1)(p1, d1) + t) │ │ │ │ 优化目标：最小化光度误差 │ │ min_{R,t} Σ ||I1(p1) - I2(warp(p1, d1, R, t))||² │ │ │ │ 优势： │ │ - 无需特征提取和匹配 │ │ - 可利用所有像素信息（更鲁棒于弱纹理） │ │ - 计算效率可以很高 │ │ │ │ 劣势： │ │ - 对光照变化敏感 │ │ - 需要好的初始值 │ │ - 非凸优化，容易陷入局部极小 │ └─────────────────────────────────────────────────────┘

LSD-SLAM与直接法SLAM

LSD-SLAM (Large-Scale Direct SLAM)： 大规模直接法单目SLAM 核心思想： ┌─────────────────────────────────────────────────────┐ │ 不使用特征点，直接在图像上进行像素级跟踪 │ │ 利用半稠密深度估计构建环境地图 │ │ │ │ 深度估计方式： │ │ - 对每个像素估计深度值和不确定性 │ │ - 通过极线搜索找到对应点 │ │ - 使用概率模型维护深度估计 │ │ - 只在梯度大的区域估计深度（半稠密） │ └─────────────────────────────────────────────────────┘ LSD-SLAM 流程： 1. 跟踪 (Tracking) - 使用光度误差最小化估计位姿 - E = Σ |Iref(warp(p, d)) - Icur(p)|² / σ² - 高斯-牛顿法求解 2. 深度图估计 (Depth Map Estimation) - 极线搜索：在当前帧极线上找最佳匹配 - 深度滤波：用高斯分布维护深度估计 - 只在梯度大的像素点（边缘、纹理）估计深度 3. 地图维护 (Map Maintenance) - 关键帧选择：位移/旋转超过阈值 - 地图更新：融合新关键帧的深度信息 - 尺度漂移检测和校正 4. 回环检测 - 基于相似度的回环候选检测 - 7-DoF位姿图优化（含尺度校正） DSO (Direct Sparse Odometry)： ┌─────────────────────────────────────────────────────┐ │ 改进的直接法SLAM │ │ │ │ 特点： │ │ - 稀疏直接法：只使用少量像素点 │ │ - 联合优化位姿和结构 │ │ - 光度标定（相机响应函数、暗角） │ │ - 窗口优化（滑窗内的帧联合优化） │ │ │ │ 与LSD的区别： │ │ - LSD半稠密 vs DSO稀疏 │ │ - LSD有回环 vs DSO纯VO（无回环） │ │ - DSO更精确但需要标定 │ │ │ │ 光度误差模型： │ │ E = Σ ||(Ij[warp(p)] - bj) / aj │ │ - exp(bi-aj) × (Ii[p] - bi)||²_γ │ │ 考虑曝光时间(a)和亮度偏移(b) │ └─────────────────────────────────────────────────────┘ 特征法 vs 直接法对比： ┌──────────────┬──────────────────┬──────────────────┐ │ │ 特征法 │ 直接法 │ ├──────────────┼──────────────────┼──────────────────┤ │ 信息利用 │ 特征点(稀疏) │ 像素(半稠密/稀疏) │ │ 计算效率 │ 较高 │ 中等~高 │ │ 弱纹理环境 │ 差 │ 好 │ │ 光照变化 │ 好(描述子鲁棒) │ 差(需光度标定) │ │ 运动模糊 │ 中等 │ 中等 │ │ 地图密度 │ 稀疏点云 │ 半稠密深度图 │ │ 回环检测 │ 容易(词袋) │ 困难 │ │ 代表算法 │ ORB-SLAM │ LSD/DSO │ └──────────────┴──────────────────┴──────────────────┘

SVO与混合方法

SVO (Semi-direct Visual Odometry)： 结合特征法和直接法的优点 流程： ┌─────────────────────────────────────────────────────┐ │ 1. 直接法跟踪 (Semi-direct) │ │ - 不提取描述子，直接用像素灰度跟踪 │ │ - 利用稀疏特征点的位置进行光度误差最小化 │ │ - 跟踪速度快（不需要描述子匹配） │ │ │ │ 2. 特征对齐 (Feature Alignment) │ │ - 在关键帧中提取FAST特征点 │ │ - 使用直接法对齐特征点到当前帧 │ │ - 建立帧间特征对应关系 │ │ │ │ 3. 深度估计 (Depth Filter) │ │ - 对每个特征点维护深度概率分布 │ │ - 使用高斯+均匀混合模型 ││ - 融合多帧观测更新深度估计 │ │ │ │ 4. 全局优化 │ │ - 关键帧间进行BA优化 │ │ - 相对轻量的后端 │ └─────────────────────────────────────────────────────┘ SVO2.0改进： - 支持多种相机模型（针孔、广角、鱼眼） - IMU预积分融合 - 支持多摄像头系统 - 深度估计改进 视觉SLAM系统选型： ┌─────────────────────────────────────────────────────┐ │ 应用场景 推荐系统 │ │ ───────────────────────────────────────────── │ │ 通用机器人 ORB-SLAM3（最成熟全面） │ │ 高精度要求 DSO（精度最高） │ │ 无人机 SVO（速度最快） │ │ AR/VR ARKit/ARCore（移动端优化） │ │ 室内导航 ORB-SLAM + RGB-D │ │ 自动驾驶 VINS-Mono/Fusion（VI融合） │ │ 大规模环境 LSD-SLAM（半稠密地图） │ └─────────────────────────────────────────────────────┘ 开源工具链： - OpenCV：图像处理和特征提取 - Eigen：线性代数库 - g2o / Ceres：非线性优化 - DBoW2：词袋模型回环检测 - Pangolin：可视化
特征法（ORB-SLAM）适合通用场景，直接法（DSO）精度高但需标定，半直接法（SVO）速度优势明显。实际选型需根据精度、速度和环境特点综合考虑。

视觉SLAM的挑战与前沿
