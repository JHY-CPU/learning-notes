运动控制

PIDMPCPure PursuitStanley

运动控制使机器人精确跟踪规划路径，PID、MPC、Pure Pursuit和Stanley是常用的路径跟踪算法。

PID控制

PID控制器：比例-积分-微分控制，工业控制的基石 控制律： ┌─────────────────────────────────────────────────────┐ │ u(t) = Kp × e(t) + Ki × ∫e(τ)dτ + Kd × de(t)/dt │ │ │ │ 各项作用： │ │ 比例项 (P)：Kp × e(t) │ │ - 反应当前误差 │ │ - Kp大：响应快但可能超调振荡 │ │ - Kp小：响应慢但稳定 │ │ │ │ 积分项 (I)：Ki × ∫e(τ)dτ │ │ - 消除稳态误差 │ │ - 积累历史误差 │ │ - Ki大：积分饱和(windup)风险 │ │ - Ki小：稳态误差消除慢 │ │ │ │ 微分项 (D)：Kd × de(t)/dt │ │ - 预测误差趋势 │ │ - 抑制振荡 │ │ - 对噪声敏感 │ │ - Kd大：响应超前但放大噪声 │ └─────────────────────────────────────────────────────┘ 离散化PID： ┌─────────────────────────────────────────────────────┐ │ 位置式PID： │ │ u[k] = Kp × e[k] + Ki × Σe[j] + Kd × (e[k]-e[k-1])│ │ │ │ 增量式PID： │ │ Δu[k] = Kp×(e[k]-e[k-1]) + Ki×e[k] │ │ + Kd×(e[k]-2e[k-1]+e[k-2]) │ │ u[k] = u[k-1] + Δu[k] │ │ 避免积分累积，更稳定 │ │ │ │ 采样周期选择： │ │ - 太大：控制精度低，响应慢 │ │ - 太小：计算负担大，数值问题 │ │ - 经验值：系统时间常数的1/10 ~ 1/20 │ └─────────────────────────────────────────────────────┘ PID调参方法： ┌─────────────────────────────────────────────────────┐ │ Ziegler-Nichols方法： │ │ 1. 设置Ki=Kd=0，逐渐增大Kp │ │ 2. 直到系统持续振荡，记录临界增益Ku和振荡周期Tu │ │ 3. 按公式计算参数： │ │ P控制：Kp = 0.5 × Ku │ │ PI控制：Kp = 0.45 × Ku, Ki = 1.2 × Kp/Tu │ │ PID控制：Kp = 0.6 × Ku, Ki = 2 × Kp/Tu, │ │ Kd = Kp × Tu / 8 │ │ │ │ 经验调参法： │ │ 1. 先调P：增大Kp直到响应快且不振荡 │ │ 2. 再调I：加入Ki消除稳态误差 │ │ 3. 最后调D：加入Kd减少超调和振荡 │ │ │ │ 抗积分饱和 (Anti-windup)： │ │ - 积分限幅：限制积分项范围 │ │ - 积分分离：误差大时禁用积分项 │ │ - 条件积分：只在可控范围内积分 │ └─────────────────────────────────────────────────────┘ 在机器人中的应用： ┌─────────────────────────────────────────────────────┐ │ 速度控制： │ │ 期望速度 - 实际速度 → PID → 电机PWM │ │ │ │ 航向控制： │ │ 期望航向 - 实际航向 → PID → 转向角 │ │ │ │ 位置控制： │ │ 期望位置 - 实际位置 → PID → 速度命令 │ │ │ │ 双环控制（速度+位置）： │ │ 位置PID → 速度设定值 → 速度PID → 电机PWM │ │ 外环（位置）控制内环（速度） │ └─────────────────────────────────────────────────────┘

自行车运动学模型

自行车模型 (Bicycle Model)： 将四轮车辆简化为前后各一个轮子的模型 运动学方程： ┌─────────────────────────────────────────────────────┐ │ 状态变量： │ │ (x, y, θ, v) │ │ x, y：位置 │ │ θ：航向角 │ │ v：速度 │ │ │ │ 运动学方程： │ │ ẋ = v × cos(θ) │ │ ẏ = v × sin(θ) │ │ θ̇ = v × tan(δ) / L │ │ │ │ 其中： │ │ δ：前轮转角（控制输入） │ │ L：轴距（前后轮距离） │ │ │ │ 曲率： │ │ κ = tan(δ) / L = 1 / R │ │ R：转弯半径 │ └─────────────────────────────────────────────────────┘ 模型假设： ┌─────────────────────────────────────────────────────┐ │ 1. 车辆在平面上运动 │ │ 2. 无侧滑（纯滚动） │ │ 3. 前后轮在同一轴线上 │ │ 4. 低速运动（忽略动力学效应） │ │ 5. 刚体运动 │ │ │ │ 适用场景： │ │ - 低速运动（< 5m/s） │ │ - 平坦路面 │ │ - 路径规划和跟踪 │ │ │ │ 局限性： │ │ - 高速时不准确（需动力学模型） │ │ - 不考虑轮胎侧偏 │ │ - 不考虑车身姿态 │ └─────────────────────────────────────────────────────┘ 阿克曼转向： ┌─────────────────────────────────────────────────────┐ │ 内外轮转角不同，使所有轮子绕同一瞬时中心转动 │ │ │ │ tan(δ_o) = L / (R + d/2) │ │ tan(δ_i) = L / (R - d/2) │ │ │ │ 其中 d = 轮距 │ │ │ │ 阿克曼率： │ │ 阿克曼% = (δ_i - δ_实际内) / (δ_i - δ_o) × 100% │ │ 100% = 完美阿克曼转向 │ │ │ │ 在低速跟踪中通常使用等效转角δ（自行车模型） │ └─────────────────────────────────────────────────────┘

Pure Pursuit路径跟踪

Pure Pursuit：几何路径跟踪算法 基于当前位姿和路径上的预瞄点计算转向角 算法原理： ┌─────────────────────────────────────────────────────┐ │ 1. 在路径上找到预瞄点 (Look-ahead Point) │ │ 距离当前位置 ld 处的路径点 │ │ │ │ 2. 计算预瞄点在车辆坐标系中的位置 │ │ (x_l, y_l) = R(-θ) × (x_goal - x, y_goal - y) │ │ │ │ 3. 计算曲率： │ │ κ = 2 × y_l / ld² │ │ 或 κ = 2 × sin(α) / ld │ │ α = 预瞄点与车辆航向的夹角 │ │ │ │ 4. 转向角： │ │ δ = arctan(κ × L) │ │ L = 轴距 │ │ │ │ 5. 速度控制： │ │ v = v_ref （恒速或根据曲率调整） │ │ 高曲率时减速 │ └─────────────────────────────────────────────────────┘ 预瞄距离选择： ┌─────────────────────────────────────────────────────┐ │ ld过大：切弯严重，跟踪不精确 │ │ ld过小：振荡，不稳定 │ │ │ │ 自适应预瞄距离： │ │ ld = ld_min + (ld_max - ld_min) × v / v_max │ │ 或 ld = k × v （与速度成正比） │ │ │ │ 典型值： │ │ - 低速（<1m/s）：ld = 0.5~1m │ │ - 中速（1~3m/s）：ld = 1~3m │ │ - 高速（>3m/s）：ld = 3~6m │ │ │ │ 前瞻点选择： │ │ - 在路径上找到距离当前位置 ld 的最近点 │ │ - 如果路径曲率大，可能需要多步搜索 │ └─────────────────────────────────────────────────────┘ Pure Pursuit特性： ┌─────────────────────────────────────────────────────┐ │ 优势： │ │ - 实现简单，计算量小 │ │ - 对路径噪声鲁棒 │ │ - 广泛用于自动驾驶和机器人 │ │ - 参数调节直观 │ │ │ │ 劣势： │ │ - 存在稳态跟踪误差（切弯） │ │ - 高速时可能不稳定 │ │ - 不考虑车辆动力学约束 │ │ - 对急转弯跟踪效果差 │ │ │ │ 改进方向： │ │ - 自适应预瞄距离 │ │ - 结合Stanley补偿切弯 │ │ - 速度-曲率联动控制 │ └─────────────────────────────────────────────────────┘ Python实现要点： ┌─────────────────────────────────────────────────────┐ │ def pure_pursuit(x, y, theta, path, ld): │ │ # 1. 找到最近路径点 │ │ idx = find_nearest_point(path, x, y) │ │ # 2. 找到预瞄点 │ │ goal = find_lookahead(path, idx, ld) │ │ # 3. 转换到车辆坐标系 │ │ dx = goal.x - x │ │ dy = goal.y - y │ │ local_x = dx * cos(theta) + dy * sin(theta) │ │ local_y = -dx * sin(theta) + dy * cos(theta) │ │ # 4. 计算曲率和转向角 │ │ kappa = 2 * local_y / (local_x**2 + local_y**2)│ │ delta = atan(kappa * wheelbase) │ │ return delta │ └─────────────────────────────────────────────────────┘

Stanley控制器

Stanley控制器：斯坦福大学DARPA竞赛的路径跟踪算法 基于横向误差和航向误差的几何控制 控制律： ┌─────────────────────────────────────────────────────┐ │ 前轮转角 = 航向误差 + 横向误差引起的转角 │ │ │ │ δ = θ_e + arctan(k × e / v) │ │ │ │ 其中： │ │ δ：前轮转角（控制输出） │ │ θ_e：航向误差 = θ_path - θ_vehicle │ │ e：横向误差（前轴中心到最近路径点的距离） │ │ k：增益参数（控制横向误差响应速度） │ │ v：车速 │ │ │ │ 物理意义： │ │ - 航向误差项：修正车辆朝向与路径方向的偏差 │ │ - 横向误差项：修正车辆与路径的距离偏差 │ │ - 速度v在分母：高速时减小横向修正，避免振荡 │ └─────────────────────────────────────────────────────┘ 误差计算： ┌─────────────────────────────────────────────────────┐ │ 横向误差 e： │ │ 1. 找到路径上距离车辆前轴最近的点 (x_ref, y_ref) │ │ 2. 计算该点处路径的切向量方向 θ_path │ │ 3. 横向误差 = 车辆前轴到路径的有符号距离 │ │ e = sign × √((x_front-x_ref)² + (y_front-y_ref)²)│ │ 符号由车辆在路径的左侧还是右侧决定 │ │ │ │ 航向误差 θ_e： │ │ θ_e = θ_path - θ_vehicle │ │ 归一化到 [-π, π] │ │ │ │ 增益 k 选择： │ │ - k大：横向误差响应快，但可能过冲 │ │ - k小：响应慢，跟踪精度低 │ │ - 典型值：k = 1~5 │ └─────────────────────────────────────────────────────┘ Stanley vs Pure Pursuit： ┌──────────────┬──────────────────┬──────────────────┐ │ │ Pure Pursuit │ Stanley │ ├──────────────┼──────────────────┼──────────────────┤ │ 误差定义 │ 预瞄点距离 │ 最近点横向误差 │ │ 控制方式 │ 曲率控制 │ 角度叠加 │ │ 切弯问题 │ 存在 │ 较小 │ │ 高速稳定性 │ 一般 │ 较好 │ │ 参数调节 │ ld（预瞄距离） │ k（增益） │ │ 适用速度 │ 低速为主 │ 中低速 │ │ 实现复杂度 │ 简单 │ 简单 │ └──────────────┴──────────────────┴──────────────────┘ 横向误差补偿的几何解释： ┌─────────────────────────────────────────────────────┐ │ 车辆前轴到路径的距离为 e │ │ 通过一个与 e 成正比的转向角 δ_lat = arctan(k×e/v) │ │ 使车辆向路径方向"收敛" │ │ 速度v越大，需要的转向角越小（避免过冲） │ │ │ │ 路径 │ │ ─────────── │ │ ↑ e (横向误差) │ │ │ │ │ 🚗 → 车辆前轴 │ │ \ δ (转向角) │ │ \→ │ └─────────────────────────────────────────────────────┘

模型预测控制 (MPC)

MPC：基于模型的最优控制方法 在每个时间步求解有限时域的优化问题 核心思想： ┌─────────────────────────────────────────────────────┐ │ 1. 建立系统预测模型 │ │ 2. 在每个控制周期求解优化问题 │ │ 3. 只执行第一步的控制输入 │ │ 4. 滚动时域，重复优化 │ │ │ │ 与PID的区别： │ │ - PID：只利用当前和过去的误差 │ │ - MPC：预测未来的状态，提前规划控制 │ │ - MPC：能处理约束，多变量耦合 │ │ - MPC：计算量大，需要模型 │ └─────────────────────────────────────────────────────┘ MPC优化问题： ┌─────────────────────────────────────────────────────┐ │ 目标函数： │ │ min Σ_{k=0}^{N-1} [‖x_k - x_ref‖²_Q + ‖u_k‖²_R] │ │ + ‖x_N - x_ref‖²_P │ │ │ │ 约束条件： │ │ x_{k+1} = f(x_k, u_k) 状态方程（预测模型） │ │ u_min ≤ u_k ≤ u_max 控制输入约束 │ │ x_min ≤ x_k ≤ x_max 状态约束 │ │ Δu_min ≤ Δu_k ≤ Δu_max 控制增量约束 │ │ │ │ 其中： │ │ N：预测时域长度 │ │ Q：状态权重矩阵（跟踪精度） │ │ R：控制权重矩阵（控制能量） │ │ P：终端状态权重矩阵 │ │ x_ref：参考轨迹 │ └─────────────────────────────────────────────────────┘ 车辆MPC控制： ┌─────────────────────────────────────────────────────┐ │ 状态：x = [X, Y, θ, v] │ │ 控制：u = [a, δ] (加速度, 转向角) │ │ │ │ 预测模型（离散化自行车模型）： │ │ X_{k+1} = X_k + v_k × cos(θ_k) × dt │ │ Y_{k+1} = Y_k + v_k × sin(θ_k) × dt │ │ θ_{k+1} = θ_k + v_k × tan(δ_k) / L × dt │ │ v_{k+1} = v_k + a_k × dt │ │ │ │ 约束： │ │ |δ| ≤ δ_max 最大转向角 │ │ |a| ≤ a_max 最大加速度 │ │ |Δδ| ≤ Δδ_max 转向角速率限制 │ │ v_min ≤ v ≤ v_max 速度范围 │ │ │ │ 参考轨迹： │ │ 从全局路径规划器获取的路径点序列 │ └─────────────────────────────────────────────────────┘ 求解方法： ┌─────────────────────────────────────────────────────┐ │ 线性MPC： │ │ - 线性化模型 → 二次规划(QP)问题 │ │ - 求解速度快（<1ms） │ │ - 工具：OSQP, qpOASES, HPIPM │ │ │ │ 非线性MPC： │ │ - 直接使用非线性模型 │ │ - 序列二次规划(SQP)或直接法 │ │ - 精度高但计算慢 │ │ - 工具：ACADO, CasADi, FORCES Pro │ │ │ │ 代码生成MPC： │ │ - 离线生成C代码 │ │ - 运行时高效执行 │ │ - 工具：ACADO code generation, Forces Pro │ └─────────────────────────────────────────────────────┘ MPC vs 几何跟踪对比： ┌──────────────┬──────────────────┬──────────────────┐ │ │ 几何法 │ MPC │ ├──────────────┼──────────────────┼──────────────────┤ │ 模型需求 │ 简单几何 │ 需精确模型 │ │ 约束处理 │ 有限 │ 自然支持 │ │ 预测能力 │ 无 │ 有 │ │ 计算量 │ 极小 │ 较大 │ │ 跟踪精度 │ 中等 │ 高 │ │ 调参难度 │ 简单 │ 较复杂 │ │ 适用场景 │ 低速/简单场景 │ 高速/复杂场景 │ │ 实时性 │ 毫秒级 │ 5~50ms │ └──────────────┴──────────────────┴──────────────────┘
PID适合简单控制回路，几何方法（Pure Pursuit/Stanley）适合低速路径跟踪，MPC适合高速和复杂约束场景。实际系统常组合使用：MPC做上层规划，PID做底层执行。

机器人控制系统架构
