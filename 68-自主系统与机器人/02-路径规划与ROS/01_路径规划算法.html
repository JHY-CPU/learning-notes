路径规划算法与ROS

机器人路径规划ROS

路径规划使机器人找到从起点到终点的可行路径，ROS是机器人开发的标准框架。

A*搜索算法

A* = Dijkstra + 启发式 评估函数：f(n) = g(n) + h(n) - g(n)：从起点到节点n的实际代价 - h(n)：从节点n到终点的估计代价（启发函数） - f(n)：总估计代价 常用启发函数（栅格地图）： - 曼哈顿距离：h = |dx| + |dy|（4方向移动） - 欧几里得距离：h = sqrt(dx² + dy²) - 切比雪夫距离：h = max(|dx|, |dy|)（8方向） A*保证最优的条件：h(n)是可采纳的（admissible） → h(n) ≤ h*(n)，不大于真实代价 算法流程： 1. 将起点加入Open列表 2. 取f值最小的节点current 3. 如果current是终点，回溯路径 4. 将current移入Closed列表 5. 对current的每个邻居： 如果在Closed中，跳过 计算g值，如果更优则更新 加入Open列表 6. 重复步骤2 变体： - D*：增量搜索，动态环境 - D* Lite：更高效的增量版本 - Theta*：允许任意角度移动（非栅格） - Jump Point Search：跳过对称节点，加速

RRT与采样方法

RRT (Rapidly-exploring Random Tree)： 快速探索随机树，适合高维空间 算法流程： 1. 初始化：树T = {x_start} 2. 重复N次： a. 随机采样 x_rand b. 找到T中离x_rand最近的节点 x_near c. 从x_near向x_rand方向扩展步长δ得到x_new d. 如果路径无碰撞，将x_new加入T 3. 如果x_goal在T中，回溯路径 RRT*：渐近最优版本 - 新节点加入时，重选父节点(rewire) - 随采样次数增加趋向最优路径 - 使用邻居半径进行局部优化 采样方法对比： ┌──────────┬────────────┬────────────┬──────────┐ │ 算法 │ 最优性 │ 完备性 │ 适用空间 │ ├──────────┼────────────┼────────────┼──────────┤ │ A* │ 最优 │ 完备 │ 低维离散 │ │ RRT │ 非最优 │ 概率完备 │ 高维连续 │ │ RRT* │ 渐近最优 │ 概率完备 │ 高维连续 │ │ PRM │ 非最优 │ 概率完备 │ 高维连续 │ └──────────┴────────────┴────────────┴──────────┘ 势场法 (Potential Field)： - 目标产生吸引力，障碍物产生排斥力 - 简单快速，适合实时避障 - 问题：局部极小值 - 常作为局部规划器使用

ROS架构

ROS (Robot Operating System)： 机器人软件开发的元操作系统 核心概念： ┌─────────────────────────────────────────────┐ │ 节点 (Node)： │ │ - 独立运行的进程 │ │ - 完成特定功能（感知/控制/规划） │ │ │ │ 话题 (Topic)： │ │ - 发布-订阅通信模式 │ │ - 异步数据流（如传感器数据） │ │ - 多对多通信 │ │ │ │ 服务 (Service)： │ │ - 请求-响应通信模式 │ │ - 同步调用 │ │ - 一对一通信 │ │ │ │ 动作 (Action)： │ │ - 支持取消的长时间任务 │ │ - 目标-反馈-结果 │ │ │ │ 参数 (Parameter)： │ │ - 全局共享的配置参数 │ │ - 运行时可修改 │ └─────────────────────────────────────────────┘ ROS2改进： - 去中心化（不需要ROS Master） - 支持实时系统 - DDS通信中间件 - 安全机制（SROS） - 多平台支持 导航栈 (Navigation Stack)： AMCL定位 + 全局规划(A*/Dijkstra) + 局部规划(DWA/TEB) + 控制
A*/Dijkstra用于全局路径规划，RRT适合高维配置空间。ROS2是当前推荐版本，DDS提供了可靠的通信。
