# 02-路径规划与ROS

## 1. 路径规划概述

路径规划是自主系统根据环境信息和任务目标，计算出从起点到终点的安全、可行轨迹的过程。路径规划通常分为三个层次：

- **全局路径规划**：在已知地图上规划从起点到终点的粗略路径
- **局部路径规划**：根据实时感知信息，在全局路径的引导下生成避障轨迹
- **行为规划与决策**：在更高层面做出驾驶行为决策（换道、路口通行、让行等）

## 2. 全局路径规划

### 2.1 A* 算法及其变体

**A\* 算法**：结合 Dijkstra 的代价搜索和启发式搜索的图搜索算法。

- 评估函数：$f(n) = g(n) + h(n)$
  - $g(n)$：从起点到节点 $n$ 的实际代价
  - $h(n)$：从节点 $n$ 到终点的启发式估计代价
- 启发式函数需满足**可采纳性**（admissibility）：$h(n) \leq h^*(n)$（不超过真实代价），保证找到最优解
- 常用启发式：欧几里得距离、曼哈顿距离、切比雪夫距离

**变体**：

- **D\*（Dynamic A\*）**：适用于环境信息逐步获取的场景，当发现新的障碍物时，只重新计算受影响的区域，而非全局重规划
- **LPA\*（Lifelong Planning A\*）**：在增量式环境变化中复用之前的搜索信息，减少重复计算
- **Weighted A\***：使用 $f(n) = g(n) + \omega \cdot h(n)$（$\omega > 1$），加快搜索速度但可能牺牲最优性
- **Hybrid A\***：考虑车辆运动学约束（阿克曼转向），搜索连续状态空间，适用于自动驾驶车辆的路径规划

### 2.2 RRT（快速随机树）与 RRT*

**RRT（Rapidly-exploring Random Tree）**：

- 在配置空间中随机采样，从最近的树节点向采样点扩展
- 概率完备（有解时，采样次数趋于无穷必能找到解）
- 不保证路径最优

**RRT\***：

- 在 RRT 的基础上增加"重新布线（rewiring）"步骤
- 每次扩展新节点后，检查邻近节点是否可以通过新节点获得更短路径
- 渐近最优（随着采样数增加，路径趋于最优）

**Informed RRT\***：在找到初始解后，将采样范围限制在以起点和终点为焦点的椭球内，加速收敛。

**采样方法 vs 搜索方法**：RRT 系列基于采样，适合高维空间（如机械臂关节空间），无需网格化但路径需后处理平滑；A* 系列基于搜索，需离散化环境，可保证最优解但受网格分辨率制约。

## 3. 局部路径规划

局部路径规划在全局路径的引导下，根据实时传感器信息生成短期的避障轨迹。

### 3.1 DWA（动态窗口法）

在速度空间（$v, \omega$）中采样速度对，根据运动学模型向前模拟轨迹，综合考虑朝向、速度和障碍物距离选择最优速度指令。计算效率高可实时运行，但只做短时域优化。

### 3.2 TEB（时间弹性带）

将路径规划建模为优化问题：路径表示为带时间信息的位姿点序列，约束包含路径跟随、障碍物避让、速度/加速度限制和运动学约束，使用 g2o 框架求解。轨迹平滑、约束灵活，是 ROS Navigation 主流局部规划器。

### 3.3 人工势场法

将机器人在势场中运动：

- **引力场**：目标点产生吸引力，引导机器人前进
- **斥力场**：障碍物产生排斥力，使机器人远离

$$F_{total} = F_{att} + F_{rep}$$

**局限**：局部最小值问题（合力为零时机器人停滞）、在狭窄通道中震荡、目标点附近有障碍物时不可达。

### 3.4 MPC（模型预测控制）

基于系统动力学模型预测未来 $N$ 步状态，求解有限时域优化问题最小化跟踪误差，只执行第一个控制输入后滚动优化。可显式处理速度、加速度等约束，广泛用于自动驾驶的横向和纵向控制。

## 4. 行为规划与决策

### 4.1 有限状态机（FSM）

将驾驶任务分解为有限个离散状态（如巡航、跟车、换道、路口等待），通过事件触发状态转移。

- **优点**：逻辑清晰、易于实现和调试
- **局限**：状态数量随场景复杂度爆炸式增长，难以处理复杂的交互场景

### 4.2 行为树（Behavior Tree）

以树状结构组织行为逻辑，节点类型包括：

- **选择节点（Selector）**：按优先级依次尝试子节点
- **序列节点（Sequence）**：按顺序执行所有子节点
- **条件节点**：检查环境条件
- **动作节点**：执行具体行为

**相比 FSM 的优势**：模块化更强、可复用性好、更易扩展复杂行为。

### 4.3 强化学习决策

使用深度强化学习（DRL）直接从传感器数据学习驾驶策略：

- **端到端方法**：输入原始传感器数据，输出控制指令（转向、油门）
- **混合方法**：在高层决策中使用 DRL（如换道决策），底层控制仍用传统方法
- **挑战**：样本效率低、安全性难以保证、sim-to-real 迁移差距
- **仿真训练**：CARLA、AirSim 等仿真环境提供安全的训练平台

## 5. ROS（机器人操作系统）

ROS 是机器人软件开发的事实标准框架，提供了一套通信中间件、工具和库。

### 5.1 ROS1 vs ROS2

| 特性 | ROS1 | ROS2 |
|------|------|------|
| 通信 | 自定义 TCPROS/UDPROS | DDS（数据分发服务） |
| 发现机制 | 集中式 ROS Master | 去中心化，基于 DDS 发现 |
| 实时性 | 无实时支持 | 支持实时调度 |
| 安全性 | 无内置安全机制 | DDS-Security 加密认证 |
| 多平台 | 主要支持 Linux | 支持 Linux、Windows、macOS、RTOS |
| 生态 | 成熟丰富 | 逐步完善 |

### 5.2 核心概念

- **节点（Node）**：执行计算的最小单元
- **话题（Topic）**：发布/订阅模式的异步通信通道
- **服务（Service）**：请求/响应模式的同步通信
- **动作（Action）**：支持长时间任务的通信模式，包含目标、反馈和结果
- **参数服务器**：全局共享的参数存储

### 5.3 TF 坐标变换

TF 系统管理机器人各部件之间的坐标变换关系，维护坐标变换树，提供任意时刻任意两个坐标系之间的变换查询。典型坐标系：`map` → `odom` → `base_link` → `sensor_link`。TF2 是当前推荐实现。

### 5.4 常用 ROS 包

- **Navigation Stack**：2D 导航框架（全局/局部规划、代价地图、AMCL 定位）
- **MoveIt**：机械臂运动规划框架，集成 OMPL、CHOMP 等规划库
- **gmapping / cartographer**：SLAM 建图
- **robot_localization**：多传感器融合定位（EKF/UKF）

### 5.5 ROS 工具

- **RViz**：3D 可视化（传感器数据、路径、TF）
- **rqt**：GUI 工具集（绘图、话题查看、参数调整）
- **rosbag**：消息录制与回放，用于数据记录和离线分析
- **Launch 文件**：批量启动多个节点的配置

### 5.6 ROS2 改进

- **DDS 通信**：工业级 DDS 标准（Fast DDS、Cyclone DDS），去中心化发现，支持实时调度
- **安全性**：DDS-Security 插件实现身份认证、访问控制和加密
- **Component 构架**：共享库加载，支持零拷贝 Intra-process 通信
- **生命周期管理**：Managed Node 状态机（Unconfigured → Inactive → Active → Finalized）

## 6. 运动学基础

### 6.1 差速驱动（Differential Drive）

- 两个独立驱动的轮子 + 万向轮（或球轮）支撑
- 通过左右轮速差实现转向

$$v = \frac{v_R + v_L}{2}, \quad \omega = \frac{v_R - v_L}{d}$$

其中 $d$ 为两轮间距。

- **特点**：结构简单，控制容易，但在原地旋转时轨迹不平滑
- **应用**：室内移动机器人、扫地机器人

### 6.2 阿克曼转向（Ackermann Steering）

- 模仿汽车的前轮转向机构，内侧轮和外侧轮的转向角度不同
- 满足阿克曼几何关系，使所有轮子沿同心圆滚动，避免侧滑
- **约束**：有最小转弯半径，不能原地转向
- **运动学模型**：

$$\dot{x} = v\cos\theta, \quad \dot{y} = v\sin\theta, \quad \dot{\theta} = \frac{v}{L}\tan\delta$$

其中 $L$ 为轴距，$\delta$ 为前轮转角。

- **应用**：自动驾驶汽车

## 7. 仿真环境

仿真环境在安全、低成本的条件下验证算法，是自主系统开发不可或缺的工具。

| 仿真器 | 开发方 | 特点与适用场景 |
|--------|--------|----------------|
| **Gazebo** | Open Robotics | ROS 官方仿真器，支持多种物理引擎，适合室内机器人和机械臂 |
| **CARLA** | Intel/Toyota | 自动驾驶仿真平台，逼真城市场景、天气变化，支持多传感器模拟和 ROS 桥接 |
| **AirSim** | Microsoft | 基于 Unreal Engine，适合无人机自主飞行和视觉导航研究 |
| **Isaac Sim** | NVIDIA | GPU 加速大规模并行仿真，支持 sim-to-real 迁移 |
| **Webots** | Cyberbotics | 开源机器人仿真器，支持多种机器人模型 |
