# 组合与桥接模式


## 一、组合模式（Composite）


将对象组合成树形结构以表示"部分-整体"的层次结构，使单个对象和组合对象的使用具有一致性。


### 1.1 适用场景


- 需要表示对象的部分-整体层次结构（树形结构）
- 希望用户忽略组合对象和单个对象的不同
- 文件系统、组织架构、UI组件树、菜单系统


### 1.2 JavaScript实现：文件系统


```
// 组件接口（统一接口）
class FileSystemComponent {
    getName() { throw new Error('子类必须实现'); }
    getSize() { throw new Error('子类必须实现'); }
    print(indent = '') { throw new Error('子类必须实现'); }
}

// 叶子节点：文件
class File extends FileSystemComponent {
    constructor(name, size) {
        super();
        this._name = name;
        this._size = size;
    }
    getName() { return this._name; }
    getSize() { return this._size; }
    print(indent = '') {
        console.log(`${indent}📄 ${this._name} (${this._size}KB)`);
    }
}

// 组合节点：文件夹
class Folder extends FileSystemComponent {
    constructor(name) {
        super();
        this._name = name;
        this._children = [];
    }

    add(component) {
        this._children.push(component);
        return this;
    }

    remove(component) {
        const index = this._children.indexOf(component);
        if (index > -1) this._children.splice(index, 1);
    }

    getName() { return this._name; }

    getSize() {
        return this._children.reduce((sum, child) => sum + child.getSize(), 0);
    }

    print(indent = '') {
        console.log(`${indent}📁 ${this._name}/ (${this.getSize()}KB)`);
        this._children.forEach(child => child.print(indent + '  '));
    }
}

// 使用
const root = new Folder('root');
const src = new Folder('src');
src.add(new File('index.js', 10));
src.add(new File('app.js', 25));
root.add(src);
root.add(new File('package.json', 2));
root.print();
// 📁 root/ (37KB)
//   📁 src/ (35KB)
//     📄 index.js (10KB)
//     📄 app.js (25KB)
//   📄 package.json (2KB)
```


> **Note:** **透明组合 vs 安全组合：**
>
>
> 透明：叶子和组合节点实现相同接口（add/remove对叶子无意义）
>
>
> 安全：组合节点独有的add/remove在组合类中定义（类型安全但失去统一性）


## 二、桥接模式（Bridge）


将抽象部分与其实现部分分离，使它们都可以独立变化。用组合代替继承来解决多维度变化的问题。


### 2.1 问题：多层继承导致类爆炸


|  | 圆形 | 矩形 | 三角形 |
| --- | --- | --- | --- |
| 红色 | RedCircle | RedRectangle | RedTriangle |
| 蓝色 | BlueCircle | BlueRectangle | BlueTriangle |
| 绿色 | GreenCircle | GreenRectangle | GreenTriangle |


3种形状 × 3种颜色 = 9个类！新增一种颜色或形状需要增加大量类。


### 2.2 桥接模式解决方案


```
// 实现接口：颜色
class Color {
    fill() { throw new Error('子类必须实现'); }
}
class Red extends Color {
    fill() { return '红色'; }
}
class Blue extends Color {
    fill() { return '蓝色'; }
}

// 抽象部分：形状（通过组合持有颜色）
class Shape {
    constructor(color) {
        this.color = color;  // 桥接：组合而非继承
    }
    draw() { throw new Error('子类必须实现'); }
}

class Circle extends Shape {
    draw() {
        console.log(`绘制${this.color.fill()}圆形`);
    }
}

class Rectangle extends Shape {
    draw() {
        console.log(`绘制${this.color.fill()}矩形`);
    }
}

// 使用：自由组合
new Circle(new Red()).draw();    // 绘制红色圆形
new Rectangle(new Blue()).draw(); // 绘制蓝色矩形
```


### 2.3 桥接 vs 策略


| 对比 | 桥接模式 | 策略模式 |
| --- | --- | --- |
| 目的 | 分离抽象和实现 | 封装可互换的算法 |
| 使用时机 | 设计阶段 | 运行时切换 |
| 维度 | 多维度独立变化 | 单维度算法替换 |


## 三、享元模式（Flyweight）简介


运用共享技术有效地支持大量细粒度的对象，减少内存使用。


```
// 享元工厂：缓存共享对象
class TreeTypeFactory {
    static #types = {};

    static getTreeType(name, color, texture) {
        const key = `${name}_${color}_${texture}`;
        if (!this.#types[key]) {
            this.#types[key] = new TreeType(name, color, texture);
        }
        return this.#types[key];
    }
}

// 上下文状态（外部状态，不共享）
class Tree {
    constructor(x, y, type) {
        this.x = x;          // 外部状态
        this.y = y;          // 外部状态
        this.type = type;    // 内部状态（共享）
    }
}

// 1000棵树只需少量TreeType实例
const type = TreeTypeFactory.getTreeType('松树', '绿色', '粗糙');
const trees = Array.from({ length: 1000 }, (_, i) =>
    new Tree(Math.random() * 100, Math.random() * 100, type)
);
```


> **Important:** **享元模式要点：**
> 区分内部状态（可共享，存储在享元对象中）和外部状态（不可共享，由客户端维护）。


## 四、知识要点总结


1. 组合模式：树形结构，统一叶子和组合的接口
2. 桥接模式：用组合代替继承，解决多维度变化问题
3. 享元模式：共享细粒度对象，减少内存消耗
4. 结构型模式的核心思想：通过组合和委托来组织类和对象
