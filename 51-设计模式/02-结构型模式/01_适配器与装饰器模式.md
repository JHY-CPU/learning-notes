# 适配器与装饰器模式


## 一、适配器模式（Adapter）


将一个类的接口转换为客户期望的另一个接口，使不兼容的类可以一起工作。


### 1.1 类适配器 vs 对象适配器


| 类型 | 实现方式 | JavaScript实现 |
| --- | --- | --- |
| 类适配器 | 多重继承 | 不直接支持（可用Mixin模拟） |
| 对象适配器 | 组合 | 通过持有被适配对象的引用实现 |


### 1.2 JavaScript实现


```
// 已有的旧接口
class OldPaymentSystem {
    makePayment(amount, cardNumber) {
        console.log(`旧系统：从卡号 ${cardNumber} 扣款 ${amount} 元`);
        return { success: true, transactionId: 'OLD_' + Date.now() };
    }
}

// 新系统期望的接口
class NewPaymentAPI {
    process({ amount, currency, token }) {
        console.log(`新系统：处理 ${currency} ${amount}，token=${token}`);
        return { status: 'completed', ref: 'NEW_' + Date.now() };
    }
}

// 适配器：让旧系统适配新接口
class PaymentAdapter {
    constructor(oldSystem) {
        this.oldSystem = oldSystem;
    }

    // 新接口方法
    process({ amount, currency, token }) {
        // 将新接口参数转换为旧接口参数
        const cardNumber = this.tokenToCard(token);
        const result = this.oldSystem.makePayment(amount, cardNumber);
        // 将旧接口返回值转换为新接口格式
        return {
            status: result.success ? 'completed' : 'failed',
            ref: result.transactionId
        };
    }

    tokenToCard(token) {
        return '****-****-****-' + token.slice(-4);
    }
}

// 使用
const oldSystem = new OldPaymentSystem();
const adapter = new PaymentAdapter(oldSystem);

// 客户端统一使用新接口
adapter.process({ amount: 100, currency: 'CNY', token: 'tok_abc1234' });
```


### 1.3 适用场景


- 集成第三方库/遗留系统
- 统一多个类的不同接口
- 数据格式转换（如XML到JSON）
- 跨平台兼容层


## 二、装饰器模式（Decorator）


动态地给对象添加额外的职责，比继承更灵活。装饰器和被装饰对象有相同的接口。


### 2.1 JavaScript实现


```
// 基础组件
class Coffee {
    cost() { return 10; }
    description() { return '普通咖啡'; }
}

// 装饰器基类
class CoffeeDecorator {
    constructor(coffee) {
        this.coffee = coffee;
    }
    cost() { return this.coffee.cost(); }
    description() { return this.coffee.description(); }
}

// 具体装饰器
class MilkDecorator extends CoffeeDecorator {
    cost() { return this.coffee.cost() + 3; }
    description() { return this.coffee.description() + ' + 牛奶'; }
}

class SugarDecorator extends CoffeeDecorator {
    cost() { return this.coffee.cost() + 1; }
    description() { return this.coffee.description() + ' + 糖'; }
}

class WhipDecorator extends CoffeeDecorator {
    cost() { return this.coffee.cost() + 5; }
    description() { return this.coffee.description() + ' + 奶油'; }
}

// 使用：动态叠加装饰器
let myCoffee = new Coffee();
myCoffee = new MilkDecorator(myCoffee);
myCoffee = new SugarDecorator(myCoffee);
myCoffee = new WhipDecorator(myCoffee);

console.log(myCoffee.description()); // 普通咖啡 + 牛奶 + 糖 + 奶油
console.log(myCoffee.cost());        // 19
```


### 2.2 ES装饰器语法


```
// TypeScript/ES装饰器（类装饰器）
function log(target) {
    return class extends target {
        constructor(...args) {
            super(...args);
            console.log(`创建了 ${target.name} 实例`);
        }
    };
}

// 方法装饰器
function deprecated(msg) {
    return function(target, key, descriptor) {
        const original = descriptor.value;
        descriptor.value = function(...args) {
            console.warn(`警告：${key} 已弃用。${msg}`);
            return original.apply(this, args);
        };
        return descriptor;
    };
}

@log
class MyClass {
    @deprecated('请使用 newMethod()')
    oldMethod() { return '旧方法'; }
}
```


> **Important:** **装饰器 vs 继承：**
>
>
> 继承：编译时确定，静态扩展，类爆炸问题
>
>
> 装饰器：运行时组合，动态扩展，更灵活
>
>
> 原则：优先使用组合而非继承（Favor composition over inheritance）


## 三、外观模式（Facade）简介


为子系统中的一组接口提供一个统一的高层接口，使子系统更容易使用。


```
// 外观：封装复杂的子系统调用
class VideoConverter {
    convert(filename, format) {
        const file = new VideoFile(filename);
        const sourceCodec = new CodecFactory().extract(file);
        const destinationCodec = format === "mp4"
            ? new MPEG4Codec() : new OggCodec();
        const buffer = new VideoReader().read(file, sourceCodec);
        const result = new VideoWriter().write(buffer, destinationCodec);
        return result;
    }
}
// 客户端只需调用：converter.convert("video.avi", "mp4")
```


## 四、代理模式（Proxy）简介


为另一个对象提供一个替身或占位符以控制对该对象的访问。


```
// ES6 Proxy 是代理模式的原生实现
const target = { name: '张三', age: 25 };

const handler = {
    get(obj, prop) {
        console.log(`访问属性: ${prop}`);
        return obj[prop];
    },
    set(obj, prop, value) {
        if (prop === 'age' && typeof value !== 'number') {
            throw new TypeError('age必须是数字');
        }
        obj[prop] = value;
        return true;
    }
};

const proxy = new Proxy(target, handler);
proxy.name;     // 触发get陷阱
proxy.age = 30; // 触发set验证
```


## 五、知识要点总结


1. 适配器模式：转换接口，让不兼容的类协同工作
2. 装饰器模式：动态添加职责，比继承更灵活
3. 外观模式：简化复杂子系统的使用
4. 代理模式：控制对象访问，ES6 Proxy原生支持
5. 结构型模式都涉及对象之间的组合关系
