观察者与策略模式

一、观察者模式（Observer）

定义对象之间的一对多依赖关系，当一个对象状态改变时，所有依赖它的对象都会收到通知并自动更新。

1.1 JavaScript实现

// 发布-订阅（观察者模式的变体）
class EventEmitter {
    constructor() {
        this.events = {};
    }

    // 订阅
    on(event, callback) {
        if (!this.events[event]) {
            this.events[event] = [];
        }
        this.events[event].push(callback);
        return this;
    }

    // 发布
    emit(event, ...args) {
        if (this.events[event]) {
            this.events[event].forEach(cb => cb(...args));
        }
        return this;
    }

    // 取消订阅
    off(event, callback) {
        if (this.events[event]) {
            this.events[event] = this.events[event]
                .filter(cb => cb !== callback);
        }
        return this;
    }

    // 一次性订阅
    once(event, callback) {
        const wrapper = (...args) => {
            callback(...args);
            this.off(event, wrapper);
        };
        this.on(event, wrapper);
        return this;
    }
}

// 使用
const bus = new EventEmitter();

// 订阅者
const logHandler = (data) => console.log(`日志：${data}`);
const alertHandler = (data) => console.log(`告警：${data}`);

bus.on('order', logHandler);
bus.on('order', alertHandler);

// 发布
bus.emit('order', '新订单 #12345');
// 日志：新订单 #12345
// 告警：新订单 #12345

// 取消订阅
bus.off('order', alertHandler);

1.2 观察者 vs 发布订阅

对比观察者模式发布-订阅模式
耦合Subject直接通知Observer通过事件中心解耦
关系Subject维护Observer列表发布者和订阅者互不知道
典型应用DOM事件EventEmitter、消息队列
JavaScript中的观察者：
- DOM事件：element.addEventListener('click', handler)
- Node.js：EventEmitter类
- Vue响应式：data变化触发视图更新
- RxJS：基于观察者模式的响应式编程库

二、策略模式（Strategy）

定义一系列算法，将每个算法封装起来，并使它们可以互相替换。策略模式让算法独立于使用它的客户。

2.1 JavaScript实现

// 策略对象
const strategies = {
    // 表单验证策略
    isNotEmpty(value, errMsg) {
        if (value === '') return errMsg;
    },
    minLength(value, length, errMsg) {
        if (value.length < length) return errMsg;
    },
    isEmail(value, errMsg) {
        if (!/^\w+@\w+\.\w+$/.test(value)) return errMsg;
    },
    isMobile(value, errMsg) {
        if (!/^1\d{10}$/.test(value)) return errMsg;
    }
};

// 验证器
class Validator {
    constructor() {
        this.rules = [];
    }

    add(value, rule, ...args) {
        this.rules.push(() =>
            strategies[rule](value, ...args)
        );
    }

    validate() {
        for (const rule of this.rules) {
            const errMsg = rule();
            if (errMsg) return errMsg;
        }
    }
}

// 使用
const validator = new Validator();
validator.add('test@', 'isEmail', '邮箱格式不正确');
validator.add('138', 'isMobile', '手机号格式不正确');
validator.add('abc', 'minLength', 6, '密码至少6位');

console.log(validator.validate()); // 邮箱格式不正确

2.2 策略模式示例：折扣计算

const discountStrategies = {
    normal(price) { return price; },
    member(price) { return price * 0.9; },
    vip(price) { return price * 0.8; },
    superVip(price) { return price * 0.7; }
};

function calculatePrice(price, level) {
    const strategy = discountStrategies[level] || discountStrategies.normal;
    return strategy(price);
}

console.log(calculatePrice(100, 'vip')); // 80
策略模式 vs 状态模式：
策略：客户端主动选择策略，策略之间不互相知道
状态：状态自动切换，每个状态知道其他状态

三、命令模式（Command）简介

将请求封装成对象，支持撤销/重做操作、队列请求、日志请求等。

class Calculator {
    constructor() { this.value = 0; this.history = []; }

    execute(command) {
        this.value = command.execute(this.value);
        this.history.push(command);
    }

    undo() {
        const cmd = this.history.pop();
        if (cmd) this.value = cmd.undo(this.value);
    }
}

const add = (n) => ({
    execute: (val) => val + n,
    undo: (val) => val - n
});

const calc = new Calculator();
calc.execute(add(10));  // 10
calc.execute(add(5));   // 15
calc.undo();            // 10

四、知识要点总结

  1. 观察者模式：一对多依赖，事件驱动的核心
  2. 发布-订阅：通过事件中心解耦，比观察者更松耦合
  3. 策略模式：封装算法，运行时切换，消除if-else
  4. 命令模式：封装请求，支持撤销/重做
  5. 行为型模式关注对象之间的通信方式
