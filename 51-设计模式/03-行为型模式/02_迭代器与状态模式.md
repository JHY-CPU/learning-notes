# 迭代器与状态模式


## 一、迭代器模式（Iterator）


提供一种方法顺序访问聚合对象中的各个元素，而又不暴露该对象的内部表示。


### 1.1 JavaScript迭代器协议


ES6原生支持迭代器协议（Iterator Protocol）和可迭代协议（Iterable Protocol）。


```
// 迭代器协议：对象必须有 next() 方法
// next() 返回 { value, done }

// 自定义可迭代对象
class Range {
    constructor(start, end, step = 1) {
        this.start = start;
        this.end = end;
        this.step = step;
    }

    // 实现可迭代协议
    [Symbol.iterator]() {
        let current = this.start;
        const end = this.end;
        const step = this.step;

        return {
            next() {
                if (current <= end) {
                    const value = current;
                    current += step;
                    return { value, done: false };
                }
                return { value: undefined, done: true };
            }
        };
    }
}

// 使用：for...of自动调用[Symbol.iterator]
for (const num of new Range(1, 10, 2)) {
    console.log(num); // 1, 3, 5, 7, 9
}

// 展开运算符也利用迭代器
console.log([..new Range(1, 5)]); // [1, 2, 3, 4, 5]
```


### 1.2 生成器（Generator）


生成器是创建迭代器的更简洁方式。


```
function* range(start, end, step = 1) {
    for (let i = start; i <= end; i += step) {
        yield i;  // 暂停并返回值
    }
}

// 生成器自动实现迭代器协议
const iter = range(1, 5);
console.log(iter.next()); // { value: 1, done: false }
console.log(iter.next()); // { value: 2, done: false }

// 惰性求值：按需生成，节省内存
function* fibonacci() {
    let [a, b] = [0, 1];
    while (true) {
        yield a;
        [a, b] = [b, a + b];
    }
}

const fib = fibonacci();
for (let i = 0; i < 10; i++) {
    console.log(fib.next().value);
}
```


## 二、状态模式（State）


允许对象在内部状态改变时改变它的行为，对象看起来好像修改了它的类。


### 2.1 JavaScript实现：订单状态机


```
// 状态类
class OrderState {
    constructor(order) { this.order = order; }
    pay() { console.log('当前状态不能支付'); }
    ship() { console.log('当前状态不能发货'); }
    deliver() { console.log('当前状态不能签收'); }
    cancel() { console.log('当前状态不能取消'); }
}

// 待支付状态
class PendingState extends OrderState {
    pay() {
        console.log('支付成功！');
        this.order.setState(new PaidState(this.order));
    }
    cancel() {
        console.log('订单已取消');
        this.order.setState(new CancelledState(this.order));
    }
}

// 已支付状态
class PaidState extends OrderState {
    ship() {
        console.log('已发货！');
        this.order.setState(new ShippedState(this.order));
    }
    cancel() {
        console.log('已退款取消');
        this.order.setState(new CancelledState(this.order));
    }
}

// 已发货状态
class ShippedState extends OrderState {
    deliver() {
        console.log('已签收！');
        this.order.setState(new DeliveredState(this.order));
    }
}

// 已签收 / 已取消
class DeliveredState extends OrderState {}
class CancelledState extends OrderState {}

// 上下文
class Order {
    constructor(id) {
        this.id = id;
        this.state = new PendingState(this);
    }
    setState(state) { this.state = state; }
    pay() { this.state.pay(); }
    ship() { this.state.ship(); }
    deliver() { this.state.deliver(); }
    cancel() { this.state.cancel(); }
}

// 使用
const order = new Order('ORD-001');
order.pay();    // 支付成功！
order.ship();   // 已发货！
order.cancel(); // 当前状态不能取消
order.deliver();// 已签收！
```


### 2.2 状态模式 vs 策略模式


| 对比 | 状态模式 | 策略模式 |
| --- | --- | --- |
| 目的 | 封装基于状态的行为变化 | 封装可互换的算法 |
| 状态切换 | 状态对象自己控制切换 | 客户端选择策略 |
| 上下文知晓 | 状态知道上下文和其他状态 | 策略不知道上下文 |


## 三、模板方法与职责链模式简介


### 3.1 模板方法（Template Method）


定义算法的骨架，将某些步骤延迟到子类实现。


```
class DataMiner {
    // 模板方法
    mine(path) {
        this.openFile(path);
        this.extractData();
        this.parseData();    // 抽象方法：子类实现
        this.analyzeData();
        this.sendReport();
    }
    openFile(path) { console.log(`打开文件: ${path}`); }
    extractData() { console.log('提取数据'); }
    parseData() { throw new Error('子类必须实现'); }
    analyzeData() { console.log('分析数据'); }
    sendReport() { console.log('发送报告'); }
}

class CSVMiner extends DataMiner {
    parseData() { console.log('解析CSV数据'); }
}
class JSONMiner extends DataMiner {
    parseData() { console.log('解析JSON数据'); }
}
```


### 3.2 职责链模式（Chain of Responsibility）


将请求沿着处理者链传递，直到有一个处理者处理它。


```
class Handler {
    constructor(limit) {
        this.limit = limit;
        this.next = null;
    }
    setNext(handler) {
        this.next = handler;
        return handler;
    }
    handle(amount) {
        if (amount <= this.limit) {
            console.log(`${this.constructor.name} 审批了 ${amount} 元`);
        } else if (this.next) {
            this.next.handle(amount);
        } else {
            console.log('无人能审批此金额');
        }
    }
}

class TeamLead extends Handler { constructor() { super(1000); } }
class Manager extends Handler { constructor() { super(5000); } }
class Director extends Handler { constructor() { super(20000); } }

const lead = new TeamLead();
lead.setNext(new Manager()).setNext(new Director());

lead.handle(800);   // TeamLead 审批了 800 元
lead.handle(3000);  // Manager 审批了 3000 元
lead.handle(15000); // Director 审批了 15000 元
```


## 四、知识要点总结


1. 迭代器模式：ES6原生支持，Symbol.iterator协议
2. 生成器：惰性求值，按需生成数据
3. 状态模式：状态对象控制切换，消除条件分支
4. 模板方法：定义算法骨架，子类实现具体步骤
5. 职责链：请求沿链传递，解耦发送者和接收者
