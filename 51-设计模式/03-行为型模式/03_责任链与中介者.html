责任链模式与中介者模式

Chain of Responsibility & Mediator — 解耦请求发送者与处理者，集中管理对象交互

1. 责任链模式 (Chain of Responsibility)

核心思想：将请求沿着处理者链传递，每个处理者决定是否处理该请求或传递给下一个处理者。发送者无需知道请求最终由谁处理。
请求→ Handler A→ Handler B→ Handler C→ 结束

经典实现

// 抽象处理者
public abstract class Handler {
    protected Handler _next;
    public Handler SetNext(Handler next) {
        _next = next;
        return next; // 支持链式调用
    }
    public abstract void Handle(Request request);
}

// 认证处理者
public class AuthHandler : Handler {
    public override void Handle(Request req) {
        if (string.IsNullOrEmpty(req.Token)) {
            Console.WriteLine("认证失败：缺少Token");
            return; // 中断链
        }
        Console.WriteLine("认证通过");
        _next?.Handle(req); // 传递给下一个
    }
}

// 权限处理者
public class AuthzHandler : Handler {
    public override void Handle(Request req) {
        if (!HasPermission(req.User, req.Resource)) {
            Console.WriteLine("权限不足");
            return;
        }
        Console.WriteLine("权限验证通过");
        _next?.Handle(req);
    }
}

// 日志处理者
public class LoggingHandler : Handler {
    public override void Handle(Request req) {
        Console.WriteLine($"记录日志: {req.Path} at {DateTime.Now}");
        _next?.Handle(req);
    }
}

// 构建链条
var auth = new AuthHandler();
var authz = new AuthzHandler();
var log = new LoggingHandler();
auth.SetNext(authz).SetNext(log); // Auth → Authz → Log
auth.Handle(new Request { Token = "abc123", Path = "/api/data" });

中间件管道 (Middleware Pipeline)

Web 框架中的中间件是责任链的变体，ASP.NET Core 和 Express 都采用此模式。

// ASP.NET Core 中间件管道
app.UseAuthentication();   // 认证中间件
app.UseAuthorization();    // 授权中间件
app.UseRateLimiter();      // 限流中间件
app.UseExceptionHandler(); // 异常处理中间件

// 自定义中间件
public class RequestTimingMiddleware {
    private readonly RequestDelegate _next;
    public async Task Invoke(HttpContext context) {
        var sw = Stopwatch.StartNew();
        await _next(context); // 调用下一个中间件
        sw.Stop();
        Console.WriteLine($"请求耗时: {sw.ElapsedMilliseconds}ms");
    }
}

2. 中介者模式 (Mediator)

核心思想：用一个中介对象封装一系列对象的交互，使各对象不需要显式地相互引用，从而使其耦合松散。

问题：对象间直接通信

  • N 个对象两两通信需要 N(N-1)/2 条连接
  • 每个对象都了解其他所有对象
  • 添加新对象需要修改多个现有对象

解决：引入中介者

  • 所有对象只与中介者通信
  • 连接数降为 N 条
  • 添加新对象只修改中介者
  • 符合迪米特法则

聊天室示例

// 中介者接口
public interface IChatMediator {
    void SendMessage(string message, User sender);
    void SendPrivate(string message, User sender, User target);
    void Register(User user);
}

// 具体中介者：聊天室
public class ChatRoom : IChatMediator {
    private Dictionary<string, User> _users = new();

    public void Register(User user) {
        _users[user.Name] = user;
        user.SetMediator(this);
    }

    public void SendMessage(string message, User sender) {
        foreach (var user in _users.Values)
            if (user != sender)
                user.Receive($"[群聊]{sender.Name}: {message}");
    }

    public void SendPrivate(string message, User sender, User target) {
        if (_users.ContainsValue(target))
            target.Receive($"[私聊]{sender.Name}: {message}");
    }
}

// 同事类
public class User {
    public string Name { get; }
    private IChatMediator _mediator;

    public User(string name) { Name = name; }
    public void SetMediator(IChatMediator m) { _mediator = m; }

    public void Send(string msg) => _mediator.SendMessage(msg, this);
    public void Receive(string msg) => Console.WriteLine($"{Name} 收到: {msg}");
}

3. 责任链 vs 中介者 对比

维度责任链中介者
结构链式（单向）星型（中心化）
对象关系每个处理者只知下一个所有对象都知中介者
请求传递沿链传递，可中途截断由中介者决定分发
典型应用中间件、审批流程、异常处理聊天室、AirTrafficControl、GUI对话框
设计原则单一职责迪米特法则

实际框架中的应用

责任链应用

  • Servlet Filter / Express 中间件
  • Java Logger 的 Level 过滤
  • 审批流引擎 (Activiti/Camunda)
  • 异常处理链 (try-catch-finally)

中介者应用

  • Spring ApplicationContext
  • MVVM 框架的 ViewModel
  • 消息代理 (RabbitMQ/Kafka)
  • GUI 框架的对话框协调
