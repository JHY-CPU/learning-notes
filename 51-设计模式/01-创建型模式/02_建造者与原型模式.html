建造者与原型模式

一、建造者模式（Builder）

将一个复杂对象的构建过程与其表示分离，使得同样的构建过程可以创建不同的表示。

1.1 意图与适用场景

  • 对象有多个可选参数，构造函数参数过多
  • 对象的构建过程需要分步骤完成
  • 需要创建不同表示的同一类型的对象

1.2 JavaScript实现

// 产品类
class Computer {
    constructor() {
        this.cpu = '';
        this.ram = '';
        this.storage = '';
        this.gpu = '';
        this.screen = '';
    }

    toString() {
        return `电脑配置: CPU=${this.cpu}, RAM=${this.ram}, ` +
               `存储=${this.storage}, GPU=${this.gpu}`;
    }
}

// 建造者
class ComputerBuilder {
    constructor() {
        this.computer = new Computer();
    }

    setCPU(cpu) {
        this.computer.cpu = cpu;
        return this;  // 返回this实现链式调用
    }

    setRAM(ram) {
        this.computer.ram = ram;
        return this;
    }

    setStorage(storage) {
        this.computer.storage = storage;
        return this;
    }

    setGPU(gpu) {
        this.computer.gpu = gpu;
        return this;
    }

    build() {
        return this.computer;
    }
}

// 使用：链式调用
const gamingPC = new ComputerBuilder()
    .setCPU('i9-13900K')
    .setRAM('32GB DDR5')
    .setStorage('2TB NVMe SSD')
    .setGPU('RTX 4090')
    .build();

console.log(gamingPC.toString());

// Director：封装常用的构建流程
class ComputerDirector {
    static buildGamingPC(builder) {
        return builder
            .setCPU('i9-13900K')
            .setRAM('32GB DDR5')
            .setStorage('2TB NVMe SSD')
            .setGPU('RTX 4090')
            .build();
    }

    static buildOfficePC(builder) {
        return builder
            .setCPU('i5-13400')
            .setRAM('16GB DDR4')
            .setStorage('512GB SSD')
            .build();
    }
}
建造者 vs 工厂：
工厂模式：一步创建整个对象
建造者模式：分步创建复杂对象，关注构建过程

二、原型模式（Prototype）

通过复制现有对象来创建新对象，而不是通过构造函数从头创建。

2.1 意图

  • 创建对象的成本很高（如需要复杂计算或数据库查询）
  • 需要创建大量相似对象
  • 避免与具体类耦合

2.2 JavaScript实现

JavaScript本身就是基于原型继承的语言，原型模式是其天然特性。

// 方式一：Object.create()
const carPrototype = {
    type: '汽车',
    wheels: 4,
    drive() {
        console.log(`${this.brand} ${this.model} 正在行驶`);
    },
    clone() {
        return Object.create(this);
    }
};

const car1 = carPrototype.clone();
car1.brand = '宝马';
car1.model = 'X5';

const car2 = carPrototype.clone();
car2.brand = '奔驰';
car2.model = 'GLC';

car1.drive(); // 宝马 X5 正在行驶
car2.drive(); // 奔驰 GLC 正在行驶
// 方式二：深拷贝实现
class Shape {
    constructor(color, x, y) {
        this.color = color;
        this.x = x;
        this.y = y;
        this.metadata = { created: Date.now() };
    }

    // 深拷贝方法
    clone() {
        const cloned = Object.create(Object.getPrototypeOf(this));
        const keys = Object.keys(this);
        keys.forEach(key => {
            if (typeof this[key] === 'object' && this[key] !== null) {
                cloned[key] = JSON.parse(JSON.stringify(this[key]));
            } else {
                cloned[key] = this[key];
            }
        });
        return cloned;
    }
}

class Circle extends Shape {
    constructor(color, x, y, radius) {
        super(color, x, y);
        this.radius = radius;
    }

    draw() {
        console.log(`绘制圆形: 颜色=${this.color}, 半径=${this.radius}`);
    }
}

// 使用
const redCircle = new Circle('red', 10, 20, 50);
const blueCircle = redCircle.clone();
blueCircle.color = 'blue';
blueCircle.radius = 30;

2.3 浅拷贝 vs 深拷贝

方式方法特点
浅拷贝Object.assign(), 展开运算符 {...obj}只拷贝第一层，嵌套对象仍共享引用
深拷贝JSON.parse(JSON.stringify()), structuredClone()完全独立的副本，互不影响
structuredClone()：现代浏览器和Node.js提供的原生深拷贝API，支持循环引用、Date、Map、Set等类型，比JSON方式更强大。
const clone = structuredClone(originalObj);

三、知识要点总结

  1. 建造者模式：分步构建复杂对象，支持链式调用
  2. Director封装常用构建流程，进一步解耦
  3. 原型模式：通过克隆创建对象，JavaScript天然支持
  4. 浅拷贝只复制第一层，深拷贝完全独立
  5. 建造者关注"怎么建"，原型关注"怎么复制"
