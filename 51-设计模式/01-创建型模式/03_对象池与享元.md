# 对象池模式与享元模式


## 1. 对象池模式 (Object Pool)

**核心思想：**
预先创建一组对象放入"池"中，需要时借用、用完归还，避免频繁创建/销毁的开销。
客户端
→
借出 (Acquire)
→
使用对象
→
归还 (Release)
→
对象池

#### 适用场景


- 对象创建代价高（数据库连接、线程）
- 对象数量有限（如连接数限制）
- 高频创建/销毁的场景
- 需要控制资源并发数


#### 关键参数


- 初始大小 (Initial Size)
- 最大容量 (Max Size)
- 借出超时 (Borrow Timeout)
- 空闲回收 (Idle Eviction)
- 健康检查 (Validation)


```
// 通用对象池实现
public class ObjectPool<T> where T : new() {
    private readonly ConcurrentBag<T> _pool = new();
    private readonly int _maxSize;
    private int _currentCount = 0;

    public ObjectPool(int maxSize, int preFill = 0) {
        _maxSize = maxSize;
        for (int i = 0; i < preFill; i++) {
            _pool.Add(Factory());
            _currentCount++;
        }
    }

    public T Acquire() {
        if (_pool.TryTake(out T item)) return item;
        if (_currentCount < _maxSize) {
            Interlocked.Increment(ref _currentCount);
            return Factory();
        }
        throw new InvalidOperationException("池已满");
    }

    public void Release(T item) {
        if (Validate(item)) _pool.Add(item);
        else Interlocked.Decrement(ref _currentCount);
    }

    protected virtual T Factory() => new T();
    protected virtual bool Validate(T item) => true;
}

// 数据库连接池示例
public class DbConnectionPool : ObjectPool<DbConnection> {
    private readonly string _connStr;
    protected override DbConnection Factory() {
        var conn = new SqlConnection(_connStr);
        conn.Open();
        return conn;
    }
    protected override bool Validate(DbConnection conn)
        => conn.State == ConnectionState.Open;
}
```


## 2. 享元模式 (Flyweight)

**核心思想：**
将对象的状态分为"内在状态"（可共享、不变）和"外在状态"（不可共享、由客户端传入），只共享内在状态以减少内存。

#### 内在状态 (Intrinsic)


- 存储在享元对象内部
- 独立于上下文、可共享
- 不可变 (Immutable)
- 例：字符的字形数据、棋子类型


#### 外在状态 (Extrinsic)


- 由客户端上下文决定
- 每次使用时传入
- 不可共享
- 例：字符在文档中的位置(x,y)


```
// 享元对象 — 只包含内在状态
public class TreeType {
    public string Name { get; }       // 内在状态
    public string Color { get; }      // 内在状态
    public byte[] Texture { get; }   // 内在状态（共享的大数据）

    public TreeType(string name, string color, byte[] tex) {
        Name = name; Color = color; Texture = tex;
    }

    // 外在状态由调用方传入
    public void Draw(Canvas canvas, int x, int y) {
        canvas.DrawImage(Texture, x, y);
        canvas.DrawText(Name, x, y + 10);
    }
}

// 享元工厂 — 管理共享对象
public class TreeFactory {
    private static readonly Dictionary<string, TreeType> _cache = new();

    public static TreeType GetTreeType(string name, string color, byte[] tex) {
        string key = $"{name}_{color}";
        if (!_cache.ContainsKey(key))
            _cache[key] = new TreeType(name, color, tex);
        return _cache[key];
    }
}

// 客户端 — 森林中 100 万棵树，但只共享几种类型
public class Forest {
    private List<(TreeType type, int x, int y)> _trees = new();

    public void PlantTree(string name, string color, byte[] tex, int x, int y) {
        var type = TreeFactory.GetTreeType(name, color, tex);
        _trees.Add((type, x, y)); // 只存储引用 + 坐标
    }

    public void Draw(Canvas canvas) {
        foreach (var (type, x, y) in _trees)
            type.Draw(canvas, x, y);
    }
}
```


## 3. 对象池 vs 享元模式 对比


| 维度 | 对象池 (Object Pool) | 享元 (Flyweight) |
| --- | --- | --- |
| 目标 | 减少创建/销毁开销 | 减少内存占用 |
| 对象数量 | 有限（池容量） | 大量重复对象 |
| 复用方式 | 借出 → 使用 → 归还 | 多个上下文共享同一实例 |
| 状态 | 对象有状态，用完需重置 | 内在状态不可变 |
| 典型例子 | 数据库连接池、线程池 | Java String Pool、字体缓存 |
| 外在状态 | 不涉及 | 由客户端传入 |


### 连接池实战配置 (HikariCP)


```
// HikariCP 高性能连接池配置
HikariConfig config = new HikariConfig();
config.setJdbcUrl("jdbc:mysql://localhost:3306/mydb");
config.setMaximumPoolSize(20);        // 最大连接数
config.setMinimumIdle(5);             // 最小空闲连接
config.setIdleTimeout(300000);        // 空闲超时 5分钟
config.setConnectionTimeout(30000);    // 获取连接超时 30秒
config.setMaxLifetime(1800000);       // 连接最大存活 30分钟
config.setValidationTimeout(5000);     // 验证超时
HikariDataSource ds = new HikariDataSource(config);
```
