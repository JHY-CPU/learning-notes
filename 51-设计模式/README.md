## 设计模式概述

设计模式是软件工程中经过验证的、用于解决特定场景下常见设计问题的方案。GoF（Gang of Four）在1994年将23种模式分为三大类：

- **创建型模式**：关注对象的创建机制，将对象创建与使用分离
- **结构型模式**：关注类和对象的组合，形成更大的结构
- **行为型模式**：关注对象之间的职责分配和通信

此外，本笔记还收录了常见的**架构模式**，用于指导整个系统的高层设计。

## 学习建议

1. **先理解意图**：每个模式解决什么问题，比记住实现细节更重要
2. **对比学习**：相似模式（如策略与状态）需要对比理解差异
3. **实践驱动**：在实际项目中识别模式的应用场景
4. **避免过度设计**：不是所有场景都需要模式，简单优先
