# 键管理命令

## 一、概念说明

Redis的键管理命令用于管理数据库中的键，包括查看、删除、重命名、查看过期时间等操作。合理使用这些命令可以有效管理Redis中的数据。

## 二、具体用法

### KEYS与SCAN

```bash
# KEYS - 查找所有匹配的键（危险：生产环境禁止使用）
KEYS *
# 输出: 1) "user:1" 2) "user:2" 3) "session:abc"

KEYS user:*
# 输出: 1) "user:1" 2) "user:2"

# SCAN - 安全的迭代遍历（推荐使用）
SCAN 0 MATCH user:* COUNT 10
# 输出: 1) "17"（游标位置）
#        2) 1) "user:1" 2) "user:2"（匹配的键）

# 继续迭代直到游标为0
SCAN 17 MATCH user:* COUNT 10
# 输出: 1) "0"（迭代完成）
#        2) 1) "user:3"（剩余的键）

# HSCAN - 遍历Hash字段
HSCAN myhash 0 MATCH field:* COUNT 10

# SSCAN - 遍历Set元素
SSCAN myset 0 MATCH "prefix:*" COUNT 10

# ZSCAN - 遍历ZSet元素
ZSCAN myzset 0 MATCH "member:*" COUNT 10
```

### RENAME与RENAMENX

```bash
# 重命名键
SET oldkey "value"
RENAME oldkey newkey
# 输出: OK
GET newkey
# 输出: "value"

# 仅当新键不存在时重命名
SET source "data"
SET target "existing"
RENAMENX source target
# 输出: (integer) 0（失败，target已存在）

DEL target
RENAMENX source target
# 输出: (integer) 1（成功）
```

### RANDOMKEY与TYPE

```bash
# 随机返回一个键
RANDOMKEY
# 输出: "user:1"

# 查看键的数据类型
TYPE user:1
# 输出: "hash"

TYPE mylist
# 输出: "list"

TYPE myzset
# 输出: "zset"
```

### OBJECT命令

```bash
# 查看键的编码方式
OBJECT ENCODING user:1
# 输出: "ziplist" 或 "hashtable"

# 查看键的空闲时间（秒）
OBJECT IDLETIME user:1
# 输出: (integer) 120

# 查看键的引用计数
OBJECT REFCOUNT user:1
# 输出: (integer) 1

# 查看键的序列化长度
OBJECT FREQ user:1
# 输出: (integer) 1
```

### UNLINK（异步删除）

```bash
# 异步删除大键（不阻塞主线程）
UNLINK largekey
# 输出: (integer) 1

# DEL vs UNLINK
# DEL - 同步删除，阻塞主线程
# UNLINK - 异步删除，后台线程处理（Redis 4.0+）
```

### DUMP与RESTORE

```bash
# 序列化键值
SET mykey "hello"
DUMP mykey
# 输出: "\x00\x05hello\t\x00\xf5\xf7\xae\x19\xf4\xac\xd5\xcd"

# 反序列化（在其他Redis实例恢复）
RESTORE newkey 0 "\x00\x05hello\t\x00\xf5\xf7\xae\x19\xf4\xac\xd5\xcd"
# 输出: OK
GET newkey
# 输出: "hello"
```

## 三、注意事项与常见陷阱

1. **绝对禁止KEYS命令**：O(N)复杂度会阻塞Redis，应使用SCAN替代
2. **SCAN非原子**：迭代过程中键可能变化，结果可能重复或遗漏
3. **COUNT只是建议**：SCAN的COUNT参数不保证返回指定数量的结果
4. **RENAME会覆盖**：RENAME会直接覆盖已存在的同名键
5. **UNLINK不保证立即释放**：异步删除需要时间，内存不会立即释放
6. **大键删除**：删除大键使用UNLINK，避免阻塞
