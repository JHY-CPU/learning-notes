# 多数据库概念

## 一、概念说明

Redis默认支持16个数据库（编号0-15），每个数据库之间数据隔离。在实际生产环境中，通常建议只使用一个数据库（db0），因为Redis Cluster模式不支持多数据库。

## 二、具体用法

### 多数据库操作

```bash
# 查看默认数据库数量
CONFIG GET databases
# 输出: 1) "databases" 2) "16"

# 切换数据库
SELECT 0
SET key "database0"
SELECT 1
SET key "database1"
SELECT 0
GET key
# 输出: "database0"

# 每个数据库独立的Key空间
SELECT 0
KEYS *
# 输出: 1) "key"

SELECT 1
KEYS *
# 输出: 1) "key"（不同的key）

# 查看当前数据库键数量
DBSIZE
# 输出: (integer) 1
```

### 修改数据库数量

```bash
# redis.conf
databases 32

# 重启Redis生效
# 注意：生产环境不建议修改，默认16个足够
```

### 多数据库的局限性

```bash
# Redis Cluster不支持多数据库
# 集群模式下只能使用数据库0

# 主从复制支持多数据库
# 但实际复制的是所有数据库

# FLUSHALL清空所有数据库
# FLUSHDB只清空当前数据库
```

## 三、多数据库使用建议

```bash
# 场景1：开发环境隔离
# db0: 开发数据
# db1: 测试数据
# db2: 缓存数据

# 场景2：业务模块隔离
# db0: 用户会话
# db1: 业务数据
# db2: 消息队列

# 生产环境建议：
# 1. 只使用db0，通过Key前缀区分
# 2. 使用命名空间：user:session:123
# 3. 便于集群扩展和数据迁移
```

## 四、注意事项与常见陷阱

1. **集群模式限制**：Redis Cluster只支持db0
2. **SELECT命令开销**：频繁切换数据库有额外开销
3. **FLUSHALL危险**：清空所有数据库，包括db0-db15
4. **内存共享**：所有数据库共享同一内存空间
5. **命名规范**：通过Key前缀（如`业务:类型:id`）优于多数据库
6. **监控困难**：多数据库增加监控和运维复杂度

## 五、多数据库与Key命名空间对比

```bash
# 方案A：使用多数据库（不推荐）
SELECT 0
SET session:user:1001 "token_abc"
SELECT 1
SET order:info:1001 '{"id":1001}'

# 问题：
# 1. 集群模式不支持
# 2. 需要频繁切换
# 3. 监控和备份复杂

# 方案B：使用Key前缀（推荐）
SET session:user:1001 "token_abc"
SET order:info:1001 '{"id":1001}'

# 优点：
# 1. 集群模式兼容
# 2. 统一管理
# 3. SCAN按前缀遍历
SCAN 0 MATCH session:* COUNT 100
SCAN 0 MATCH order:* COUNT 100
```

### 命名空间设计规范

```bash
# 格式：业务:模块:标识
# 示例
user:profile:1001       # 用户资料
user:session:abc123     # 用户会话
order:detail:1001       # 订单详情
order:status:1001       # 订单状态
product:cache:2001      # 商品缓存
product:stock:2001      # 商品库存
config:app:timeout      # 应用配置
stats:daily:20240101    # 日统计
lock:resource:order     # 分布式锁
queue:email             # 消息队列

# 注意事项
# 1. 使用冒号分隔（业界标准）
# 2. 全小写，避免大小写混乱
# 3. 层级不超过3-4层
# 4. Key总长度不宜过长
```

### 数据隔离策略

```bash
# 开发/测试/生产环境隔离
# 方案1：不同Redis实例
# 开发: dev-redis:6379
# 测试: test-redis:6379
# 生产: prod-redis:6379

# 方案2：不同数据库（仅限非集群）
SELECT 0    # 生产
SELECT 1    # 测试
SELECT 2    # 开发

# 方案3：Key前缀区分
prod:user:1001
test:user:1001
dev:user:1001

# 推荐方案1，完全隔离
```

## 六、Redis Cluster的数据库限制

```bash
# Redis Cluster只支持db0
# 连接集群后执行SELECT会报错
redis-cli -c -h 192.168.1.100 -p 7000
SELECT 1
# (error) ERR SELECT is not allowed in cluster mode

# 如果要从多数据库迁移到集群
# 1. 导出各数据库数据
# 2. 统一导入到db0
# 3. 通过Key前缀区分数据
# 4. 使用redis-shake等工具迁移
```
