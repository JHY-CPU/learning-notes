# Redis概述

## 一、概念说明

Redis（Remote Dictionary Server）是一个开源的、基于内存的高性能键值存储系统。它支持多种数据结构，包括字符串（String）、哈希（Hash）、列表（List）、集合（Set）、有序集合（ZSet）等。Redis既可以作为数据库、缓存，也可以作为消息中间件使用。

### 核心特点

- **基于内存**：数据存储在内存中，读写速度极快，读取速度可达10万+ QPS
- **持久化支持**：支持RDB快照和AOF日志两种持久化方式，防止数据丢失
- **丰富的数据类型**：不止是简单的键值对，还支持复杂的数据结构
- **原子性操作**：所有操作都是原子性的，支持事务（MULTI/EXEC）
- **支持主从复制**：数据可以从主节点复制到多个从节点
- **高可用与分布式**：通过Sentinel提供高可用，通过Cluster提供分布式

### 使用场景

```bash
# 典型使用场景
# 1. 缓存 - 减少数据库压力，提升响应速度
# 2. 会话存储 - 分布式Session共享
# 3. 排行榜 - 利用有序集合实现实时排名
# 4. 消息队列 - 利用List或Stream实现
# 5. 分布式锁 - 利用SET NX EX实现
# 6. 计数器 - 利用INCR命令实现
# 7. 限流 - 利用过期时间+计数器
# 8. 地理位置 - 利用Geo命令实现附近的人
```

## 二、Redis与其他存储对比

| 特性 | Redis | Memcached | MySQL |
|------|-------|-----------|-------|
| 数据类型 | 丰富（5种+） | 仅字符串 | 关系型 |
| 持久化 | 支持 | 不支持 | 支持 |
| 集群 | 原生支持 | 客户端分片 | 主从复制 |
| 内存管理 | 多种策略 | LRU | 磁盘为主 |
| 事务 | 支持 | 不支持 | 支持 |

```bash
# Redis适用场景
# - 需要高速读写的场景
# - 数据结构复杂的场景
# - 需要持久化但对持久化要求不极高的场景
# - 需要发布订阅、分布式锁等高级功能的场景

# 不适用场景
# - 数据量远超内存容量
# - 需要复杂SQL查询
# - 对数据一致性要求极高的金融场景
```

## 四、注意事项与常见陷阱

1. **Redis不是万能的**：不适合存储超大数据集，内存成本较高
2. **单线程模型**：Redis命令执行是单线程的，避免使用耗时命令（如KEYS *）
3. **持久化不是实时的**：RDB快照有间隔，极端情况可能丢失部分数据
4. **内存管理**：需要合理设置maxmemory和淘汰策略，避免OOM
5. **网络瓶颈**：高并发场景下网络可能成为瓶颈，需考虑Pipeline批量操作
6. **数据一致性**：Redis是最终一致性，不适合强一致性场景
7. **版本选择**：生产环境建议使用稳定版（如6.2.x或7.0.x+）
