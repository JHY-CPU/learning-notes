# 数据库操作

## 一、概念说明

Redis默认提供16个数据库（编号0-15），可以通过SELECT命令切换。数据库之间是隔离的，但共享同一个Redis实例的内存和CPU资源。

## 二、具体用法

### SELECT切换数据库

```bash
# 切换到数据库0（默认）
SELECT 0
# 输出: OK

# 切换到数据库3
SELECT 3
# 输出: OK

# 在不同数据库设置数据
SELECT 0
SET key "db0-value"

SELECT 1
SET key "db1-value"

# 验证隔离性
SELECT 0
GET key
# 输出: "db0-value"

SELECT 1
GET key
# 输出: "db1-value"
```

### FLUSHDB与FLUSHALL

```bash
# 清空当前数据库
SELECT 2
FLUSHDB
# 输出: OK

# 异步清空（不阻塞）
FLUSHDB ASYNC
# 输出: OK

# 清空所有数据库（危险操作）
FLUSHALL
# 输出: OK

# 异步清空所有数据库
FLUSHALL ASYNC
# 输出: OK

# 生产环境建议重命名这些命令
# redis.conf
rename-command FLUSHDB ""
rename-command FLUSHALL ""
```

### SWAPDB交换数据库

```bash
# 交换数据库0和数据库1的内容
SWAPDB 0 1
# 输出: OK

# 常用于在线切换数据库
# 先将新数据写入db1，然后SWAPDB 0 1
```

### 数据库数量配置

```bash
# 修改数据库数量（redis.conf）
databases 32

# 重启后生效
# 注意：设置过大会增加内存开销
```

## 三、注意事项与常见陷阱

1. **数据库隔离但共享资源**：所有数据库共享同一内存和CPU
2. **生产环境只用一个数据库**：多数据库不利于集群扩展
3. **FLUSHALL极其危险**：会清空所有数据库的数据
4. **SELECT在集群模式下不可用**：Redis Cluster只支持数据库0
5. **数据库编号从0开始**：SELECT 16会报错
6. **多数据库不利于备份**：RDB文件包含所有数据库的数据

## 五、数据库操作实战

### KEYS与SCAN遍历

```bash
# KEYS命令（生产环境禁止使用）
SELECT 0
KEYS user:*     # 返回所有user:开头的key
KEYS *          # 返回所有key（危险！）

# SCAN替代KEYS（安全遍历）
SCAN 0 MATCH user:* COUNT 100
# 输出: 1) "17" 2) 1) "user:1001" 2) "user:1002"
# 第一个返回值是游标，0表示遍历完成

# 分页遍历
SCAN 0 MATCH order:* COUNT 10
SCAN 17 MATCH order:* COUNT 10
SCAN 42 MATCH order:* COUNT 10
# 直到游标返回0
```

### 数据库迁移

```bash
# 场景：将db1的数据迁移到db0
# 方法1：使用MOVE命令
SELECT 1
MOVE user:1001 0    # 将key从db1移动到db0
MOVE user:1002 0

# 方法2：使用DUMP+RESTORE
SELECT 1
DUMP user:1001
# 输出: "\x00\x05hello\t\x00\xf5\xf7\xae\x19\xf4\xac\xd5\xcd"

SELECT 0
RESTORE user:1001 0 "\x00\x05hello\t\x00\xf5\xf7\xae\x19\xf4\xac\xd5\xcd"

# 方法3：使用redis-shake工具批量迁移
# 适合大规模数据迁移
```

### OBJECT命令查看Key信息

```bash
# 查看Key的内部编码
OBJECT ENCODING user:1001
# 输出: "embstr" / "raw" / "ziplist" / "hashtable" 等

# 查看Key的引用计数
OBJECT REFCOUNT user:1001
# 输出: (integer) 1

# 查看Key的空闲时间（秒）
OBJECT IDLETIME user:1001
# 输出: (integer) 120

# 查看Key的频率（需要LFU淘汰策略）
OBJECT FREQ user:1001
```

### RANDOMKEY与TYPE

```bash
# 随机返回一个Key
RANDOMKEY
# 输出: "user:1001"

# 查看Key的数据类型
TYPE user:1001
# 输出: "string" / "hash" / "list" / "set" / "zset"

# 查看Key是否存在
EXISTS user:1001
# 输出: (integer) 1（存在）或 0（不存在）

# 批量检查
EXISTS user:1001 user:1002 user:9999
# 输出: (integer) 2（存在2个）
```

## 六、生产环境建议

```bash
# 1. 只使用db0
# 通过Key前缀区分业务
user:session:1001    # 用户会话
order:info:1001      # 订单信息
cache:product:2001   # 商品缓存

# 2. 重命名危险命令
rename-command FLUSHDB "FLUSHDB_xK9m2"
rename-command FLUSHALL "FLUSHALL_xK9m2"
rename-command KEYS "KEYS_xK9m2"
rename-command DEBUG ""

# 3. 设置数据库数量
databases 1    # 生产环境只开1个

# 4. 监控数据库使用情况
INFO keyspace
# db0:keys=10000,expires=5000,avg_ttl=3600000
```
