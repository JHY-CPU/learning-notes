# 数据库操作

## 一、概念说明

Redis默认提供16个数据库（编号0-15），可以通过SELECT命令切换。数据库之间是隔离的，但共享同一个Redis实例的内存和CPU资源。

## 二、具体用法

### SELECT切换数据库

```bash
# 切换到数据库0（默认）
SELECT 0
# 输出: OK

# 切换到数据库3
SELECT 3
# 输出: OK

# 在不同数据库设置数据
SELECT 0
SET key "db0-value"

SELECT 1
SET key "db1-value"

# 验证隔离性
SELECT 0
GET key
# 输出: "db0-value"

SELECT 1
GET key
# 输出: "db1-value"
```

### FLUSHDB与FLUSHALL

```bash
# 清空当前数据库
SELECT 2
FLUSHDB
# 输出: OK

# 异步清空（不阻塞）
FLUSHDB ASYNC
# 输出: OK

# 清空所有数据库（危险操作）
FLUSHALL
# 输出: OK

# 异步清空所有数据库
FLUSHALL ASYNC
# 输出: OK

# 生产环境建议重命名这些命令
# redis.conf
rename-command FLUSHDB ""
rename-command FLUSHALL ""
```

### SWAPDB交换数据库

```bash
# 交换数据库0和数据库1的内容
SWAPDB 0 1
# 输出: OK

# 常用于在线切换数据库
# 先将新数据写入db1，然后SWAPDB 0 1
```

### 数据库数量配置

```bash
# 修改数据库数量（redis.conf）
databases 32

# 重启后生效
# 注意：设置过大会增加内存开销
```

## 三、注意事项与常见陷阱

1. **数据库隔离但共享资源**：所有数据库共享同一内存和CPU
2. **生产环境只用一个数据库**：多数据库不利于集群扩展
3. **FLUSHALL极其危险**：会清空所有数据库的数据
4. **SELECT在集群模式下不可用**：Redis Cluster只支持数据库0
5. **数据库编号从0开始**：SELECT 16会报错
6. **多数据库不利于备份**：RDB文件包含所有数据库的数据
