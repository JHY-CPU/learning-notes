# 延迟队列

## 一、概念说明

延迟队列用于处理需要延迟执行的任务，如订单超时取消、延迟通知等。Redis的ZSet可以用分数存储执行时间戳来实现延迟队列。

## 二、ZSet实现延迟队列

```python
import redis
import json
import time

r = redis.Redis()

def add_delay_task(queue, task, delay_seconds):
    """添加延迟任务"""
    execute_time = time.time() + delay_seconds
    r.zadd(queue, {json.dumps(task): execute_time})

def get_ready_tasks(queue, count=10):
    """获取已到期的任务"""
    now = time.time()
    tasks = r.zrangebyscore(queue, 0, now, start=0, num=count)
    return [json.loads(t) for t in tasks]

def remove_task(queue, task):
    """删除已处理的任务"""
    r.zrem(queue, json.dumps(task))

def process_tasks(queue):
    """处理到期任务"""
    while True:
        tasks = get_ready_tasks(queue)
        for task in tasks:
            try:
                # 处理任务
                handle_task(task)
                remove_task(queue, task)
            except Exception as e:
                print(f"处理失败: {e}")
        time.sleep(1)

# 使用
add_delay_task("delay:orders", {"order_id": 1001, "action": "cancel"}, 1800)
# 30分钟后执行订单取消
```

## 三、Redisson实现

```java
// Spring Boot + Redisson
RBlockingQueue<Task> queue = redisson.getBlockingQueue("delay:queue");
RDelayedQueue<Task> delayedQueue = redisson.getDelayedQueue(queue);

// 添加延迟任务
delayedQueue.offer(new Task("cancel", 1001), 30, TimeUnit.SECONDS);

// 消费任务
Task task = queue.take();
```

## 四、注意事项

1. **时间精度**：秒级或毫秒级时间戳
2. **轮询间隔**：合理设置检查频率
3. **任务幂等**：防止重复执行
4. **内存控制**：及时清理已处理任务
5. **多消费者**：使用分布式锁防止重复消费
