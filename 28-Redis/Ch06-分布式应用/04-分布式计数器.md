# 分布式计数器

## 一、概念说明

Redis的INCR/DECR命令是原子性的，非常适合实现分布式计数器，用于限流、统计、排行榜等场景。

## 二、具体用法

### 基本计数器

```bash
# 递增
INCR counter
# 输出: (integer) 1

INCR counter
# 输出: (integer) 2

# 指定步长
INCRBY counter 10
# 输出: (integer) 12

# 递减
DECR counter
# 输出: (integer) 11

# 浮点数
INCRBYFLOAT price 0.5
```

### 限流计数器

```python
def is_rate_limited(user_id, limit=100, window=60):
    key = f"rate:limit:{user_id}"
    current = r.incr(key)
    
    if current == 1:
        # 第一次请求，设置过期时间
        r.expire(key, window)
    
    return current > limit

# 使用
if is_rate_limited("user:1001", 100, 60):
    return "请求过于频繁"
```

### 统计计数器

```bash
# 文章阅读量
INCR article:1001:views

# 用户登录次数
INCR user:1001:login_count

# 每日统计
INCR stats:daily:20240101:visits
EXPIRE stats:daily:20240101:visits 86400

# 获取当前值
GET stats:daily:20240101:visits
```

### 滑动窗口限流

```python
def sliding_window_limit(user_id, limit=100, window=60):
    key = f"sw:limit:{user_id}"
    now = time.time()
    
    pipe = r.pipeline()
    pipe.zremrangebyscore(key, 0, now - window)
    pipe.zadd(key, {f"{now}": now})
    pipe.zcard(key)
    pipe.expire(key, window)
    results = pipe.execute()
    
    count = results[2]
    return count > limit
```

## 三、注意事项

1. **原子性**：INCR是原子操作
2. **过期时间**：首次INCR时设置过期
3. **溢出**：超过64位有符号整数会报错
4. **限流策略**：固定窗口/滑动窗口/令牌桶
5. **分布式安全**：多个应用实例共享计数器
