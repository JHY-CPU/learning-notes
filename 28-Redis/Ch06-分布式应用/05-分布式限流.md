# 分布式限流

## 一、概念说明

分布式限流用于控制分布式系统中的请求速率，防止系统过载。Redis提供多种限流实现方式。

## 二、限流算法

### 固定窗口限流

```python
def fixed_window_limit(key, limit, window):
    current = r.incr(key)
    if current == 1:
        r.expire(key, window)
    return current > limit
```

### 滑动窗口限流

```python
def sliding_window_limit(key, limit, window):
    now = time.time() * 1000
    pipe = r.pipeline()
    pipe.zremrangebyscore(key, 0, now - window * 1000)
    pipe.zadd(key, {str(now): now})
    pipe.zcard(key)
    pipe.expire(key, window)
    _, _, count, _ = pipe.execute()
    return count > limit
```

### 令牌桶限流

```python
def token_bucket(key, rate, capacity):
    lua = """
    local tokens = redis.call('hget', KEYS[1], 'tokens') or capacity
    local last = redis.call('hget', KEYS[1], 'last') or 0
    local now = tonumber(ARGV[3])
    local elapsed = math.max(0, now - last)
    tokens = math.min(capacity, tokens + elapsed * rate / 1000)
    if tokens >= 1 then
        redis.call('hmset', KEYS[1], 'tokens', tokens-1, 'last', now)
        redis.call('expire', KEYS[1], ARGV[4])
        return 1
    end
    return 0
    """
    return r.eval(lua, 1, key, rate, capacity, int(time.time()*1000), 60)
```


## 四、注意事项

1. **原子性**：使用Lua脚本保证原子操作
2. **精度**：滑动窗口比固定窗口更精确
3. **性能**：避免限流逻辑成为瓶颈
4. **分布式**：所有实例共享同一个Redis
5. **降级策略**：限流触发时的降级处理
