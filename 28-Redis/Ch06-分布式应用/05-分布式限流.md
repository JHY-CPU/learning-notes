# 分布式限流

## 一、概念说明

分布式限流用于控制分布式系统中的请求速率，防止系统过载。Redis提供多种限流实现方式。

## 二、限流算法

### 固定窗口限流

```python
def fixed_window_limit(key, limit, window):
    current = r.incr(key)
    if current == 1:
        r.expire(key, window)
    return current > limit
```

### 滑动窗口限流

```python
def sliding_window_limit(key, limit, window):
    now = time.time() * 1000
    pipe = r.pipeline()
    pipe.zremrangebyscore(key, 0, now - window * 1000)
    pipe.zadd(key, {str(now): now})
    pipe.zcard(key)
    pipe.expire(key, window)
    _, _, count, _ = pipe.execute()
    return count > limit
```

### 令牌桶限流

```python
def token_bucket(key, rate, capacity):
    lua = """
    local tokens = redis.call('hget', KEYS[1], 'tokens') or capacity
    local last = redis.call('hget', KEYS[1], 'last') or 0
    local now = tonumber(ARGV[3])
    local elapsed = math.max(0, now - last)
    tokens = math.min(capacity, tokens + elapsed * rate / 1000)
    if tokens >= 1 then
        redis.call('hmset', KEYS[1], 'tokens', tokens-1, 'last', now)
        redis.call('expire', KEYS[1], ARGV[4])
        return 1
    end
    return 0
    """
    return r.eval(lua, 1, key, rate, capacity, int(time.time()*1000), 60)
```

## 四、注意事项

1. **原子性**：使用Lua脚本保证原子操作
2. **精度**：滑动窗口比固定窗口更精确
3. **性能**：避免限流逻辑成为瓶颈
4. **分布式**：所有实例共享同一个Redis
5. **降级策略**：限流触发时的降级处理

## 五、漏桶算法

```python
class LeakyBucket:
    def __init__(self, key, rate, capacity):
        """
        rate: 每秒流出速率
        capacity: 桶容量
        """
        self.key = f"leaky:{key}"
        self.rate = rate
        self.capacity = capacity
    
    def try_acquire(self):
        lua = """
        local key = KEYS[1]
        local rate = tonumber(ARGV[1])
        local capacity = tonumber(ARGV[2])
        local now = tonumber(ARGV[3])
        
        local data = redis.call('HMGET', key, 'water', 'last')
        local water = tonumber(data[1]) or 0
        local last = tonumber(data[2]) or now
        
        -- 计算流出量
        local elapsed = now - last
        water = math.max(0, water - elapsed * rate)
        
        if water < capacity then
            water = water + 1
            redis.call('HMSET', key, 'water', water, 'last', now)
            redis.call('EXPIRE', key, 60)
            return 1
        else
            redis.call('HMSET', key, 'water', water, 'last', now)
            return 0
        end
        """
        return r.eval(lua, 1, self.key, self.rate, self.capacity, time.time())
```

## 六、分布式限流注解（Spring Boot）

```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface RateLimit {
    String key() default "";
    int limit() default 100;
    int window() default 60;
    String message() default "请求过于频繁";
}

@Aspect
@Component
public class RateLimitAspect {
    @Autowired
    private StringRedisTemplate redisTemplate;
    
    @Around("@annotation(rateLimit)")
    public Object around(ProceedingJoinPoint point, RateLimit rateLimit) throws Throwable {
        String key = "rate:" + rateLimit.key();
        long now = System.currentTimeMillis();
        
        // 滑动窗口限流
        Long count = redisTemplate.execute(new RedisCallback<Long>() {
            public Long doInRedis(RedisConnection connection) {
                // 清理过期数据
                connection.zRemRangeByScore(key.getBytes(), 0, now - rateLimit.window() * 1000);
                // 添加当前请求
                connection.zAdd(key.getBytes(), now, String.valueOf(now).getBytes());
                // 设置过期
                connection.expire(key, rateLimit.window());
                // 返回计数
                return connection.zCard(key.getBytes());
            }
        });
        
        if (count > rateLimit.limit()) {
            throw new RuntimeException(rateLimit.message());
        }
        
        return point.proceed();
    }
}

// 使用
@GetMapping("/api/users")
@RateLimit(key = "getUsers", limit = 100, window = 60)
public List<User> getUsers() {
    return userService.findAll();
}
```

## 七、限流策略对比

```bash
# 固定窗口
# 实现：INCR + EXPIRE
# 优点：简单高效
# 缺点：窗口边界突发流量
# 适用：粗粒度限流

# 滑动窗口
# 实现：ZSet + 时间戳
# 优点：精确控制
# 缺点：内存占用较多
# 适用：精确限流

# 令牌桶
# 实现：Lua脚本 + Hash
# 优点：允许一定突发
# 缺点：实现复杂
# 适用：允许突发的场景

# 漏桶
# 实现：Lua脚本 + Hash
# 优点：输出平滑
# 缺点：不允许突发
# 适用：平滑限流

# 选择建议
# 简单场景 → 固定窗口
# 精确要求 → 滑动窗口
# 允许突发 → 令牌桶
# 平滑输出 → 漏桶
```
