# 分布式ID生成

## 一、概念说明

分布式系统中需要全局唯一的ID。Redis的INCR命令可以生成连续的数字ID，结合其他方案可以生成更复杂的ID。

## 二、Redis INCR方案

```bash
# 简单递增ID
INCR global:id
# 输出: (integer) 1

# 带前缀的ID
INCR order:id
# 输出: 1001
# 生成: ORDER202401011001
```

## 三、Snowflake + Redis

```python
# 雪花算法结合Redis获取worker_id
def get_worker_id():
    return r.incr("snowflake:worker_id")

class Snowflake:
    def __init__(self, worker_id):
        self.worker_id = worker_id
        self.sequence = 0
        self.last_timestamp = 0
    
    def generate(self):
        timestamp = int(time.time() * 1000)
        if timestamp == self.last_timestamp:
            self.sequence = (self.sequence + 1) & 0xFFF
        else:
            self.sequence = 0
        self.last_timestamp = timestamp
        
        return ((timestamp - 1609459200000) << 22 |
                self.worker_id << 12 |
                self.sequence)
```

## 四、号段模式

```python
# 从Redis批量获取ID段
def get_id_segment(key, step=1000):
    max_id = r.incrby(key, step)
    min_id = max_id - step + 1
    return min_id, max_id

# 本地消耗ID段
class IdGenerator:
    def __init__(self):
        self.current = 0
        self.max = 0
    
    def next_id(self):
        if self.current >= self.max:
            self.current, self.max = get_id_segment("order:id")
        self.current += 1
        return self.current
```

## 五、注意事项

1. **ID连续性**：INCR生成连续ID，有规律可循
2. **性能**：号段模式减少Redis访问
3. **时钟问题**：Snowflake依赖系统时钟
4. **Worker ID**：确保每个节点唯一
5. **ID格式**：根据业务需求设计ID格式

## 六、完整雪花算法实现

```python
import time
import redis

r = redis.Redis()

class Snowflake:
    # 各部分位数
    WORKER_BITS = 10
    SEQUENCE_BITS = 12
    
    # 最大值
    MAX_WORKER_ID = (1 << WORKER_BITS) - 1  # 1023
    MAX_SEQUENCE = (1 << SEQUENCE_BITS) - 1  # 4095
    
    # 时间戳左移位数
    WORKER_SHIFT = SEQUENCE_BITS
    TIMESTAMP_SHIFT = WORKER_BITS + SEQUENCE_BITS
    
    # 起始时间戳 (2020-01-01 00:00:00)
    EPOCH = 1577808000000
    
    def __init__(self, worker_id=None):
        if worker_id is None:
            worker_id = self._get_worker_id()
        
        if worker_id < 0 or worker_id > self.MAX_WORKER_ID:
            raise ValueError(f"Worker ID必须在0-{self.MAX_WORKER_ID}之间")
        
        self.worker_id = worker_id
        self.sequence = 0
        self.last_timestamp = -1
    
    def _get_worker_id(self):
        """从Redis获取Worker ID"""
        return r.incr("snowflake:worker_id") % (self.MAX_WORKER_ID + 1)
    
    def _current_timestamp(self):
        """获取当前时间戳（毫秒）"""
        return int(time.time() * 1000)
    
    def generate(self):
        """生成ID"""
        timestamp = self._current_timestamp()
        
        if timestamp < self.last_timestamp:
            raise Exception("时钟回拨，拒绝生成ID")
        
        if timestamp == self.last_timestamp:
            # 同一毫秒内，序列号递增
            self.sequence = (self.sequence + 1) & self.MAX_SEQUENCE
            if self.sequence == 0:
                # 序列号溢出，等待下一毫秒
                while timestamp <= self.last_timestamp:
                    timestamp = self._current_timestamp()
        else:
            # 不同毫秒，序列号重置
            self.sequence = 0
        
        self.last_timestamp = timestamp
        
        # 组合ID
        id = ((timestamp - self.EPOCH) << self.TIMESTAMP_SHIFT) | \
             (self.worker_id << self.WORKER_SHIFT) | \
             self.sequence
        
        return id
    
    def parse(self, id):
        """解析ID"""
        timestamp = (id >> self.TIMESTAMP_SHIFT) + self.EPOCH
        worker_id = (id >> self.WORKER_SHIFT) & self.MAX_WORKER_ID
        sequence = id & self.MAX_SEQUENCE
        
        return {
            'timestamp': timestamp,
            'worker_id': worker_id,
            'sequence': sequence,
            'datetime': time.strftime('%Y-%m-%d %H:%M:%S', time.localtime(timestamp / 1000))
        }

# 使用
snowflake = Snowflake(worker_id=1)
id = snowflake.generate()
print(f"生成ID: {id}")
print(f"解析: {snowflake.parse(id)}")
```

## 七、号段模式详细实现

```python
class SegmentIdGenerator:
    def __init__(self, biz_tag, step=1000):
        self.biz_tag = biz_tag
        self.step = step
        self.current_id = 0
        self.max_id = 0
        self.loaded = False
    
    def _load_segment(self):
        """从Redis加载号段"""
        key = f"id:segment:{self.biz_tag}"
        max_id = r.incrby(key, self.step)
        self.current_id = max_id - self.step + 1
        self.max_id = max_id
        self.loaded = True
        print(f"加载号段: {self.current_id}-{self.max_id}")
    
    def next_id(self):
        """获取下一个ID"""
        if not self.loaded or self.current_id > self.max_id:
            self._load_segment()
        
        current = self.current_id
        self.current_id += 1
        return current
    
    def next_id_with_prefix(self, prefix="", date_format="%Y%m%d"):
        """获取带前缀的ID"""
        date_str = time.strftime(date_format)
        id = self.next_id()
        return f"{prefix}{date_str}{id:08d}"

# 使用
generator = SegmentIdGenerator("order", step=1000)

# 批量生成ID（只需访问Redis一次）
for _ in range(1000):
    id = generator.next_id()
    # print(id)

# 带业务前缀
order_id = generator.next_id_with_prefix(prefix="ORD")
print(f"订单ID: {order_id}")  # ORD2024010100000001
```

## 八、ID方案对比

```bash
# 方案对比表
# | 方案         | 唯一性 | 有序性 | 性能 | 复杂度 | 适用场景 |
# | INCR         | 是     | 是     | 高   | 低     | 简单场景 |
# | 雪花算法     | 是     | 是     | 高   | 中     | 分布式系统 |
# | 号段模式     | 是     | 是     | 极高 | 中     | 高并发场景 |
# | UUID         | 是     | 否     | 高   | 低     | 不要求有序 |
# | Redis+日期   | 是     | 是     | 中   | 低     | 业务ID |

# 选择建议
# 简单业务 → INCR
# 高性能要求 → 号段模式
# 分布式节点多 → 雪花算法
# 不需要有序 → UUID
# 需要可读性 → Redis+日期

# 注意事项
# 1. ID不应暴露业务信息
# 2. 考虑ID的可排序性
# 3. 评估ID生成的QPS需求
# 4. 准备故障降级方案
```
