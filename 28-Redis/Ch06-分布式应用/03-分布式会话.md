# 分布式会话

## 一、概念说明

分布式会话将用户Session存储在Redis中，实现多服务器间的Session共享。用户在任何服务器上都能获取到相同的Session数据。

## 二、Spring Session + Redis

```xml
<!-- pom.xml -->
<dependency>
    <groupId>org.springframework.session</groupId>
    <artifactId>spring-session-data-redis</artifactId>
</dependency>
```

```java
// application.yml
spring:
  session:
    store-type: redis
    redis:
      namespace: spring:session
    timeout: 1800

// 使用Session
@GetMapping("/user")
public User getUser(HttpSession session) {
    User user = (User) session.getAttribute("user");
    if (user == null) {
        user = loadUser();
        session.setAttribute("user", user);
    }
    return user;
}
```

## 四、Express + Redis

```javascript
const session = require('express-session');
const RedisStore = require('connect-redis').default;
const redis = require('redis');

const client = redis.createClient({ host: 'localhost', port: 6379 });
client.connect();

app.use(session({
    store: new RedisStore({ client }),
    secret: 'your-secret',
    resave: false,
    saveUninitialized: false,
    cookie: { maxAge: 1800000 }
}));
```

## 五、注意事项

1. **Session序列化**：选择合适的序列化方式
2. **过期时间**：合理设置Session过期时间
3. **安全性**：使用HTTPS传输Session ID
4. **性能**：Session操作有网络开销
5. **清理过期Session**：Redis自动清理过期Key

## 六、Session存储结构详解

```bash
# Spring Session存储结构
# 1. Session数据Hash

HSET spring:session:sessions:{sessionId} lastAccessedTime 1704067200
HSET spring:session:sessions:{sessionId} maxInactiveInterval 1800
HSET spring:session:sessions:{sessionId} sessionAttr:user '{"name":"张三","role":"admin"}'

# 2. 过期索引（用于批量清理）
ZADD spring:session:sessions:expires 1704069000 {sessionId}

# 3. Session ID索引
SET spring:session:expirations:1704069000 {sessionId}

# 4. 删除Session
DEL spring:session:sessions:{sessionId}
ZREM spring:session:sessions:expires {sessionId}
```

## 七、自定义Session管理

```python
import redis
import json
import time
from uuid import uuid4

r = redis.Redis()

class RedisSession:
    def __init__(self, session_id=None, expire=1800):
        self.expire = expire
        if session_id:
            self.session_id = session_id
        else:
            self.session_id = str(uuid4())
        self.key = f"session:{self.session_id}"
        self._load()
    
    def _load(self):
        """加载Session数据"""
        data = r.get(self.key)
        if data:
            self.data = json.loads(data)
        else:
            self.data = {}
            self.data['_created'] = time.time()
        
        # 更新访问时间
        self.data['_last_accessed'] = time.time()
        r.expire(self.key, self.expire)
    
    def get(self, key, default=None):
        return self.data.get(key, default)
    
    def set(self, key, value):
        self.data[key] = value
        self._save()
    
    def delete(self, key):
        if key in self.data:
            del self.data[key]
            self._save()
    
    def _save(self):
        """保存Session数据"""
        r.setex(self.key, self.expire, json.dumps(self.data))
    
    def destroy(self):
        """销毁Session"""
        r.delete(self.key)

# 使用示例
session = RedisSession()
session.set('user_id', 1001)
session.set('username', '张三')
print(session.get('user_id'))  # 1001

# 重新加载
session2 = RedisSession(session_id=session.session_id)
print(session2.get('username'))  # 张三
```

## 八、Session安全最佳实践

```bash
# 1. 使用安全的Session ID
# - 使用UUID v4或加密随机数
# - 长度至少128位
# - 不要在URL中传递Session ID

# 2. 设置安全Cookie
Set-Cookie: SESSIONID=abc123; HttpOnly; Secure; SameSite=Strict

# 3. Session固定攻击防护
# 登录成功后重新生成Session ID

# 4. Session劫持防护
# 绑定IP地址或User-Agent
# 检测异常IP变化时要求重新登录

# 5. 并发Session控制
# 限制同一用户的同时登录数
# 后登录踢掉先登录 / 允许多设备同时登录

# 6. Session过期策略
# 绝对过期：登录后固定时间过期
# 滑动过期：每次操作延长过期时间
# 空闲超时：无操作固定时间过期
```
