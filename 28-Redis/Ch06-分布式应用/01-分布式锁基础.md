# 分布式锁基础

## 一、概念说明

分布式锁用于在分布式系统中控制对共享资源的并发访问。Redis通过SET NX EX命令实现简单高效的分布式锁。

## 二、具体用法

### 基本实现

```bash
# 获取锁
SET lock:order:1001 "uuid-value" NX EX 30
# NX: 不存在时才设置
# EX 30: 30秒自动过期
# 返回OK表示获取成功
# 返回(nil)表示获取失败

# 释放锁（原子性）
EVAL "
if redis.call('GET', KEYS[1]) == ARGV[1] then
    return redis.call('DEL', KEYS[1])
else
    return 0
end
" 1 lock:order:1001 "uuid-value"
```

### Python实现

```python
import redis
import uuid

r = redis.Redis()

def acquire_lock(key, timeout=30):
    uid = str(uuid.uuid4())
    result = r.set(key, uid, nx=True, ex=timeout)
    if result:
        return uid
    return None

def release_lock(key, uid):
    lua = """
    if redis.call('get', KEYS[1]) == ARGV[1] then
        return redis.call('del', KEYS[1])
    else
        return 0
    end
    """
    return r.eval(lua, 1, key, uid)

# 使用
lock_key = "lock:resource:1001"
uid = acquire_lock(lock_key)
if uid:
    try:
        # 临界区操作
        print("获取锁成功")
    finally:
        release_lock(lock_key, uid)
```

## 三、关键要素

```bash
# 1. 原子性获取
SET key value NX EX timeout

# 2. 唯一标识
# 使用UUID防止误解锁

# 3. 自动过期
# 防止死锁

# 4. 原子性释放
# Lua脚本保证
```

## 四、注意事项

1. **必须设置过期时间**：防止死锁
2. **使用UUID**：防止误解锁
3. **Lua脚本释放**：保证原子性
4. **锁粒度**：不要使用大粒度锁
5. **重试机制**：获取失败时的处理
