# 分布式队列

## 一、概念说明

Redis可以用作分布式消息队列，支持点对点和发布订阅模式。常用List和Stream实现。

## 二、List实现队列

```bash
# 生产者
LPUSH queue:orders '{"id":1001,"data":"order1"}'

# 消费者（阻塞式）
BRPOP queue:orders 0
# 阻塞等待直到有消息

# 多队列
BRPOP queue:high queue:normal queue:low 0
# 按优先级消费
```

## 三、Stream实现队列

```bash
# 生产者
XADD orders * order_id 1001 amount 99.9

# 创建消费者组
XGROUP CREATE orders group1 0

# 消费者
XREADGROUP GROUP group1 consumer1 COUNT 10 STREAMS orders >

# ACK确认
XACK orders group1 1704067200000-0
```

## 四、Python实现

```python
# List队列
class RedisQueue:
    def __init__(self, name, redis_client):
        self.name = name
        self.redis = redis_client
    
    def put(self, item):
        self.redis.lpush(self.name, json.dumps(item))
    
    def get(self, timeout=0):
        result = self.redis.brpop(self.name, timeout)
        if result:
            return json.loads(result[1])
        return None

# Stream队列
class RedisStreamQueue:
    def __init__(self, stream, group, consumer):
        self.stream = stream
        self.group = group
        self.consumer = consumer
        self.redis = redis.Redis()
    
    def put(self, data):
        return self.redis.xadd(self.stream, data)
    
    def get(self, count=10):
        return self.redis.xreadgroup(
            self.group, self.consumer,
            {self.stream: '>'}, count=count
        )
```

## 五、注意事项

1. **消息持久化**：Stream支持持久化，List不支持
2. **消息确认**：Stream支持ACK，List不支持
3. **消费者组**：Stream支持消费者组
4. **消息丢失**：List队列消费者崩溃可能丢失消息
5. **队列积压**：监控队列长度，防止内存溢出
