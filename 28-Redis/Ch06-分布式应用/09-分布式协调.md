# 分布式协调

## 一、概念说明

Redis可用于分布式系统的协调任务，如服务注册发现、配置同步、分布式通知等。

## 二、服务注册与发现

```bash
# 服务注册
HSET service:api:node1 host 192.168.1.1
HSET service:api:node1 port 8080
HSET service:api:node1 status healthy
EXPIRE service:api:node1 30

# 心跳续期
# 每10秒续期
EXPIRE service:api:node1 30

# 服务发现
HKEYS service:api:*
# 获取所有api服务节点

# 获取健康节点
HSCAN service:api: 0 MATCH * COUNT 100
```

## 三、配置同步

```bash
# 存储配置
HSET config:app max_connections 100
HSET config:app timeout 30

# 配置变更通知
PUBLISH config:changed "max_connections"

# 客户端订阅
SUBSCRIBE config:changed
# 收到通知后重新加载配置
```

## 四、分布式通知

```bash
# 简单通知
LPUSH notifications:user:1001 "有新消息"
EXPIRE notifications:user:1001 86400

# 批量通知
PUBLISH channel:all "系统维护通知"

# 事件通知
XADD events * type "deploy" service "api" version "1.0.0"
```

## 五、分布式Leader选举

```bash
# 竞选Leader
SET leader "node1" NX EX 30
# 获取成功成为Leader

# Leader续期
EXPIRE leader 30

# 其他节点检查
GET leader
# 返回当前Leader
```

## 六、注意事项

1. **过期时间**：服务注册必须设置过期
2. **心跳机制**：定期续期防止服务过期
3. **一致性**：最终一致性，不保证强一致
4. **性能**：大量服务注册时的性能考虑
5. **替代方案**：ZooKeeper/etcd更适合协调任务
