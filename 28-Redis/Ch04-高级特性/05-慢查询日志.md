# 慢查询日志

## 一、概念说明

Redis慢查询日志记录执行时间超过阈值的命令，帮助发现和优化性能问题。类似于MySQL的慢查询日志。

## 二、具体用法

### 配置慢查询

```bash
# redis.conf 配置

# 慢查询阈值（微秒）
slowlog-log-slower-than 10000
# 超过10000微秒（10毫秒）的命令记录

# 慢查询日志最大条数
slowlog-max-len 128
# 最多记录128条

# 禁用慢查询日志
slowlog-log-slower-than 0

# 记录所有命令
slowlog-log-slower-than -1
```

### 查看慢查询日志

```bash
# 获取慢查询日志
SLOWLOG GET 10
# 输出:
# 1) 1) (integer) 10     # 日志ID
#    2) (integer) 1704067200  # 时间戳
#    3) (integer) 15000  # 执行时间（微秒）
#    4) 1) "KEYS"        # 命令
#       2) "*"
#    5) "192.168.1.1:6379"  # 客户端地址
#    6) ""              # 客户端名称

# 获取慢查询数量
SLOWLOG LEN
# 输出: (integer) 5

# 清空慢查询日志
SLOWLOG RESET
```

## 三、分析慢查询

```bash
# 常见慢查询原因
# 1. KEYS * 命令
# 2. 大集合操作（SMEMBERS大Set）
# 3. 大Hash的HGETALL
# 4. SORT命令无LIMIT
# 5. Lua脚本执行时间过长

# 优化建议
# 使用SCAN替代KEYS
# 使用SSCAN/HSCAN/ZSCAN
# 分批处理大数据
```

## 四、注意事项

1. **阈值设置**：生产环境建议10000微秒（10毫秒）
2. **日志大小**：slowlog-max-len不宜过大
3. **内存中存储**：慢查询日志存在内存中
4. **定期检查**：定期分析慢查询日志
5. **不记录阻塞命令**：BLPOP等阻塞命令不记录
