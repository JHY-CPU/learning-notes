# 键过期通知

## 一、概念说明

Redis的键空间通知（Keyspace Notifications）允许客户端订阅键的事件，包括键过期、删除、修改等。基于Redis的Pub/Sub实现。

## 二、具体用法

### 开启通知

```bash
# redis.conf 配置
notify-keyspace-events Ex

# 参数说明
# K - 键空间通知
# E - 键事件通知
# $ - 字符串命令
# l - 列表命令
# s - 集合命令
# h - 哈希命令
# z - 有序集合命令
# x - 过期事件
# e - 驱逐事件
# A - 所有事件

# 示例配置
notify-keyspace-events Ex    # 只通知过期事件
notify-keyspace-events KEx   # 键空间+过期事件
notify-keyspace-events A     # 所有事件
```

### 订阅过期事件

```bash
# 订阅键过期事件
SUBSCRIBE __keyevent@0__:expired
# @0表示数据库0

# 设置一个有过期时间的键
SET mykey "value" EX 5

# 5秒后键过期，订阅者收到通知
# 1) "message"
# 2) "__keyevent@0__:expired"
# 3) "mykey"
```

### 其他事件

```bash
# 订阅删除事件
SUBSCRIBE __keyevent@0__:del

# 订阅所有事件
SUBSCRIBE __keyevent@0__:*

# 订阅特定键的事件
SUBSCRIBE __keyspace@0__:mykey:*
```

## 三、应用场景

```bash
# 1. Session过期处理
# 用户Session过期时清理相关数据

# 2. 缓存失效处理
# 缓存过期时重新加载

# 3. 定时任务触发
# 键过期触发业务逻辑

# 4. 资源清理
# 临时资源过期时清理
```

## 四、注意事项

1. **通知不保证送达**：Redis不保证通知一定送达
2. **过期精度**：过期事件有0-1毫秒延迟
3. **性能影响**：开启通知有额外开销
4. **仅通知已过期的键**：TTL到期后才通知
5. **集群模式**：每个节点独立发布事件

## 五、Python订阅过期事件

```python
import redis
import threading

r = redis.Redis()

def handle_expired_events():
    """处理键过期事件"""
    pubsub = r.pubsub()
    pubsub.subscribe('__keyevent@0__:expired')
    
    for message in pubsub.listen():
        if message['type'] == 'message':
            expired_key = message['data'].decode()
            print(f"Key过期: {expired_key}")
            
            # 根据Key类型处理不同业务
            if expired_key.startswith('session:'):
                # Session过期清理
                cleanup_user_session(expired_key)
            elif expired_key.startswith('cache:'):
                # 缓存过期重新加载
                reload_cache(expired_key)
            elif expired_key.startswith('lock:'):
                # 分布式锁过期告警
                alert_lock_expired(expired_key)

# 启动监听线程
thread = threading.Thread(target=handle_expired_events, daemon=True)
thread.start()
```

## 六、事件类型完整列表

```bash
# 配置参数格式
# K - 键空间通知
# E - 键事件通知
# g - 通用命令（DEL, EXPIRE, RENAME等）
# $ - 字符串命令
# l - 列表命令
# s - 集合命令
# h - 哈希命令
# z - 有序集合命令
# x - 过期事件
# e - 驱逐事件
# t - Stream命令
# m - 键缺失事件

# 常用配置组合
notify-keyspace-events Ex     # 过期事件
notify-keyspace-events Kx     # 键空间+过期
notify-keyspace-events A      # 所有事件（性能影响大）
notify-keyspace-events ""     # 禁用通知
notify-keyspace-events Egx    # 通用命令+驱逐事件
```

## 七、过期事件精度问题

```bash
# Redis的过期删除是惰性+定期两种方式
# 惰性删除：访问时发现过期才删除
# 定期删除：后台任务定期扫描过期Key

# 因此过期事件可能有延迟
# 实际过期时间 = TTL + 删除延迟（0-1000ms）

# 如果Key在TTL到期后未被访问
# 可能延迟到下次定期删除时才触发事件

# 解决方案
# 1. 定期访问Key（但不推荐，浪费资源）
# 2. 接受延迟，在业务逻辑中处理
# 3. 使用Lua脚本在TTL到期前主动删除
```
