# 内存淘汰策略

## 一、概念说明

当Redis内存达到maxmemory限制时，根据淘汰策略删除部分key以腾出空间。选择合适的淘汰策略对应用性能至关重要。

## 二、淘汰策略

```bash
# 八种淘汰策略

# 1. noeviction（默认）
#    内存满时返回错误，不删除任何key
maxmemory-policy noeviction

# 2. allkeys-lru
#    从所有key中淘汰最近最少使用的
maxmemory-policy allkeys-lru

# 3. volatile-lru
#    从有过期时间的key中淘汰LRU
maxmemory-policy volatile-lru

# 4. allkeys-random
#    从所有key中随机淘汰
maxmemory-policy allkeys-random

# 5. volatile-random
#    从有过期时间的key中随机淘汰
maxmemory-policy volatile-random

# 6. volatile-ttl
#    从有过期时间的key中淘汰TTL最小的
maxmemory-policy volatile-ttl

# 7. allkeys-lfu（Redis 4.0+）
#    从所有key中淘汰最不常用的
maxmemory-policy allkeys-lfu

# 8. volatile-lfu（Redis 4.0+）
#    从有过期时间的key中淘汰LFU
maxmemory-policy volatile-lfu
```

### 策略选择

```bash
# 缓存场景推荐
# allkeys-lru：所有key都可能被淘汰
# allkeys-lfu：基于访问频率淘汰

# 混合场景推荐
# volatile-lru：只淘汰设了过期时间的key
# volatile-lfu：基于频率淘汰过期key

# 严格场景
# noeviction：不自动淘汰，内存满报错
```

## 三、配置与监控

```bash
# 设置最大内存
maxmemory 1gb

# 设置淘汰策略
maxmemory-policy allkeys-lru

# 查看内存使用
INFO memory
# used_memory_human:1.00G
# maxmemory_human:1.00G

# 查看淘汰数量
INFO stats
# evicted_keys:1000
```

## 四、注意事项

1. **必须设置maxmemory**：否则可能耗尽系统内存
2. **LRU是近似算法**：采样方式，不是精确LRU
3. **allkeys vs volatile**：volatile只淘汰有过期时间的key
4. **LFU需要4.0+**：旧版本不支持LFU策略
5. **淘汰有性能开销**：大量淘汰会影响性能

## 五、LRU与LFU深入

```bash
# LRU（Least Recently Used）
# 基于访问时间淘汰
# 近似LRU：随机采样5个Key，淘汰最久未访问的
# 采样数量可通过maxmemory-samples调整
maxmemory-samples 5    # 默认5，增加到10更精确但更耗CPU

# LFU（Least Frequently Used）- Redis 4.0+
# 基于访问频率淘汰
# 每个Key有一个8位的访问计数器
# 计数器随时间衰减

# LFU调优
lfu-decay-time 1      # 计数器衰减时间（分钟）
lfu-log-factor 10     # 计数器递增因子

# 查看Key的LFU频率
OBJECT FREQ key_name
# 返回0-255之间的值，越大越频繁
```

## 六、策略选择指南

```bash
# 场景1：纯缓存（数据不重要）
maxmemory-policy allkeys-lru
# 所有Key都可以被淘汰

# 场景2：混合存储（部分数据重要）
maxmemory-policy volatile-lru
# 只淘汰设置了过期时间的Key
# 重要的Key不设过期时间

# 场景3：访问频率不均匀
maxmemory-policy allkeys-lfu
# 高频访问的Key保留更久

# 场景4：需要精确控制
maxmemory-policy noeviction
# 内存满时返回错误
# 应用层处理内存不足

# 场景5：有过期时间的Key为主
maxmemory-policy volatile-ttl
# 优先淘汰快过期的Key
```

## 七、淘汰监控

```bash
# 监控淘汰数量
INFO stats | grep evicted_keys
# evicted_keys:1000（表示淘汰了1000个Key）

# 监控内存使用
INFO memory | grep used_memory_human
INFO memory | grep maxmemory_human

# 监控淘汰频率
# 如果evicted_keys快速增长，说明内存不足
# 需要增加内存或优化数据结构

# Python监控脚本
import redis
import time

r = redis.Redis()
last_evicted = 0

while True:
    info = r.info('stats')
    evicted = info['evicted_keys']
    delta = evicted - last_evicted
    if delta > 0:
        print(f"警告: 过去1秒淘汰了{delta}个Key")
    last_evicted = evicted
    time.sleep(1)
```
