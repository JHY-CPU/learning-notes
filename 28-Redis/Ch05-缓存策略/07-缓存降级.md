# 缓存降级

## 一、概念说明

缓存降级是在缓存服务不可用或压力过大时，返回降级数据或默认值，保证系统的可用性。

## 二、降级策略

### 默认值降级

```bash
# 缓存不可用时返回默认值
def get_product(id):
    try:
        data = redis.get(f"product:{id}")
        if data:
            return data
        data = db.query(id)
        redis.set(f"product:{id}", data, ex=3600)
        return data
    except RedisError:
        return {"name": "默认商品", "price": 0}
```

### 本地缓存降级

```bash
# Redis不可用时使用本地缓存
from functools import lru_cache

@lru_cache(maxsize=1000)
def get_product(id):
    try:
        return redis.get(f"product:{id}")
    except:
        return local_cache.get(id)
```

### 熔断降级

```bash
# 使用熔断器
from circuitbreaker import circuit

@circuit(failure_threshold=5, recovery_timeout=30)
def get_from_redis(key):
    return redis.get(key)

# 连续失败5次后熔断
# 30秒后尝试恢复
```

## 三、降级级别

```bash
# L1 - 一级降级
# Redis不可用，使用本地缓存

# L2 - 二级降级
# 本地缓存也不可用，使用默认值

# L3 - 三级降级
# 返回空数据或错误页面
```

## 四、配置示例

```yaml
# 降级配置
degradation:
  enabled: true
  fallback:
    product: '{"name":"商品暂缺","available":false}'
    user: '{"name":"用户","vip":false}'
  timeout: 1000  # 超时时间
  max-failures: 5  # 最大失败次数
```

## 五、注意事项

1. **降级预案**：提前准备降级数据
2. **分级降级**：不同服务不同降级策略
3. **监控告警**：触发降级时发送告警
4. **自动恢复**：缓存恢复后自动切换回正常模式
5. **用户体验**：降级时保持基本功能可用
