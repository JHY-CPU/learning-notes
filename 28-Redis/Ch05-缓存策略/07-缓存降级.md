# 缓存降级

## 一、概念说明

缓存降级是在缓存服务不可用或压力过大时，返回降级数据或默认值，保证系统的可用性。

## 二、降级策略

### 默认值降级

```bash
# 缓存不可用时返回默认值
def get_product(id):
    try:
        data = redis.get(f"product:{id}")
        if data:
            return data
        data = db.query(id)
        redis.set(f"product:{id}", data, ex=3600)
        return data
    except RedisError:
        return {"name": "默认商品", "price": 0}
```

### 本地缓存降级

```bash
# Redis不可用时使用本地缓存
from functools import lru_cache

@lru_cache(maxsize=1000)
def get_product(id):
    try:
        return redis.get(f"product:{id}")
    except:
        return local_cache.get(id)
```

### 熔断降级

```bash
# 使用熔断器
from circuitbreaker import circuit

@circuit(failure_threshold=5, recovery_timeout=30)
def get_from_redis(key):
    return redis.get(key)

# 连续失败5次后熔断
# 30秒后尝试恢复
```

## 三、降级级别

```bash
# L1 - 一级降级
# Redis不可用，使用本地缓存

# L2 - 二级降级
# 本地缓存也不可用，使用默认值

# L3 - 三级降级
# 返回空数据或错误页面
```

## 四、配置示例

```yaml
# 降级配置
degradation:
  enabled: true
  fallback:
    product: '{"name":"商品暂缺","available":false}'
    user: '{"name":"用户","vip":false}'
  timeout: 1000  # 超时时间
  max-failures: 5  # 最大失败次数
```

## 五、注意事项

1. **降级预案**：提前准备降级数据
2. **分级降级**：不同服务不同降级策略
3. **监控告警**：触发降级时发送告警
4. **自动恢复**：缓存恢复后自动切换回正常模式
5. **用户体验**：降级时保持基本功能可用

## 六、Python完整实现

```python
import redis
import json
import time
from functools import lru_cache
from enum import Enum

class DegradationLevel(Enum):
    NORMAL = 0       # 正常
    LOCAL_CACHE = 1  # 使用本地缓存
    DEFAULT = 2      # 使用默认值
    EMPTY = 3        # 返回空

class CacheDegradation:
    def __init__(self):
        self.r = redis.Redis()
        self.local_cache = {}
        self.failure_count = 0
        self.degradation_level = DegradationLevel.NORMAL
        self.last_failure_time = 0
    
    def get(self, key, db_query_func, default_value=None):
        # 根据降级级别处理
        if self.degradation_level == DegradationLevel.EMPTY:
            return None
        
        if self.degradation_level == DegradationLevel.DEFAULT:
            return default_value
        
        if self.degradation_level == DegradationLevel.LOCAL_CACHE:
            return self.local_cache.get(key, default_value)
        
        # 正常模式
        try:
            # 查缓存
            cached = self.r.get(key)
            if cached:
                if cached == b'NULL':
                    return default_value
                value = json.loads(cached)
                self.local_cache[key] = value  # 同步本地缓存
                return value
            
            # 查数据库
            value = db_query_func()
            if value:
                self.r.setex(key, 3600, json.dumps(value))
                self.local_cache[key] = value
            else:
                self.r.setex(key, 300, 'NULL')
            
            return value or default_value
            
        except redis.RedisError as e:
            self._handle_failure(e)
            return self.local_cache.get(key, default_value)
    
    def _handle_failure(self, error):
        self.failure_count += 1
        self.last_failure_time = time.time()
        
        if self.failure_count >= 3:
            self.degradation_level = DegradationLevel.LOCAL_CACHE
        if self.failure_count >= 10:
            self.degradation_level = DegradationLevel.DEFAULT
        if self.failure_count >= 20:
            self.degradation_level = DegradationLevel.EMPTY
        
        print(f"降级级别: {self.degradation_level}, 失败次数: {self.failure_count}")
    
    def reset(self):
        """恢复到正常模式"""
        self.failure_count = 0
        self.degradation_level = DegradationLevel.NORMAL
        print("已恢复到正常模式")

# 使用
degradation = CacheDegradation()
product = degradation.get("product:1001", query_product, {"name": "默认商品"})
```

## 七、降级预案设计

```yaml
# 降级配置文件 degradation.yml
degradation:
  levels:
    - name: L1-本地缓存
      condition: redis_failures >= 3
      action: use_local_cache
      ttl: 60
    
    - name: L2-默认值
      condition: redis_failures >= 10
      action: use_default
      defaults:
        product: '{"name":"商品暂缺","available":false}'
        user: '{"name":"用户","vip":false}'
    
    - name: L3-空响应
      condition: redis_failures >= 20
      action: return_empty
  
  recovery:
    check_interval: 30  # 每30秒检查恢复
    success_threshold: 3  # 连续3次成功则恢复
  
  alerting:
    enabled: true
    channels:
      - dingtalk
      - email
```

## 八、降级效果监控

```python
def monitor_degradation():
    """监控降级效果"""
    while True:
        info = {
            'level': degradation.degradation_level.name,
            'failure_count': degradation.failure_count,
            'local_cache_size': len(degradation.local_cache),
            'last_failure': degradation.last_failure_time
        }
        
        print(f"降级状态: {info}")
        
        # 尝试恢复
        if degradation.degradation_level != DegradationLevel.NORMAL:
            try:
                degradation.r.ping()
                degradation.reset()
            except:
                pass
        
        time.sleep(30)
```
