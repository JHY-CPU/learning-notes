# 缓存容量规划

## 一、概念说明

缓存容量规划是估算Redis所需的内存空间，选择合适的实例规格和淘汰策略。

## 二、内存估算

```bash
# 单个Key内存估算
# String类型
SET user:1001 '{"name":"张三","age":25}'
# Key: user:1001 ≈ 11 bytes
# Value: JSON ≈ 35 bytes
# Redis对象头 ≈ 40 bytes
# 总计 ≈ 86 bytes

# 100万用户估算
100万 * 86 bytes ≈ 86 MB

# Hash类型更省内存
HSET user:1001 name "张三" age 25
# ziplist编码更紧凑
```

### 详细估算公式

```bash
# String
内存 = Key长度 + Value长度 + 对象头(约40bytes)

# Hash (ziplist)
内存 ≈ 字段数 * (字段名 + 值 + 开销)

# List (quicklist)
内存 ≈ 元素数 * (元素值 + 指针开销)

# Set/ZSet
内存 ≈ 元素数 * (元素值 + 额外开销)
```

## 三、容量规划

```bash
# 预估数据量
用户数: 100万
每个用户缓存: 1 KB
总内存需求: 1 GB

# 考虑开销
实际内存 ≈ 预估 * 1.5（预留50%余量）
实际内存 ≈ 1.5 GB

# 选择实例
# 推荐实例内存 = 实际内存 * 2
# 给持久化、复制等留出空间
```

## 四、淘汰策略配置

```bash
# 根据业务需求选择
# 缓存场景：allkeys-lru
maxmemory 4gb
maxmemory-policy allkeys-lru

# 混合场景：volatile-lru
maxmemory 4gb
maxmemory-policy volatile-lru

# 监控内存使用
INFO memory
# used_memory_human:2.50G
# maxmemory_human:4.00G
```

## 五、注意事项

1. **预留空间**：实际内存需求的2倍
2. **考虑过期Key**：过期Key也占用内存
3. **碎片率**：mem_fragmentation_ratio > 1.5需要关注
4. **扩容计划**：提前规划扩容方案
5. **监控告警**：内存使用超过80%告警
