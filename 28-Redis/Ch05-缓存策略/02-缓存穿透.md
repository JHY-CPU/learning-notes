# 缓存穿透

## 一、概念说明

缓存穿透是指查询一个不存在的数据，缓存和数据库都没有，每次请求都会打到数据库。大量此类请求可能导致数据库压力过大。

## 二、问题分析

```bash
# 正常流程
GET user:9999 → 缓存nil → 直接返回

# 穿透问题
GET user:9999 → 缓存没有 → 查数据库 → 数据库也没有 → 不写缓存
# 下次请求重复上述过程

# 恶意攻击
# 攻击者使用随机ID大量查询不存在的数据
# 所有请求都穿透到数据库
```

## 三、解决方案

### 方案一：空值缓存

```bash
# 查询不存在的数据时，缓存空值
GET user:9999
# 返回nil
# 查询数据库，不存在
SET user:9999 "NULL" EX 300
# 设置较短的过期时间

# 下次查询
GET user:9999
# 返回"NULL"，不再查数据库
```

### 方案二：布隆过滤器

```bash
# 添加所有存在的ID到布隆过滤器
BF.ADD user:ids 1001
BF.ADD user:ids 1002
BF.ADD user:ids 1003

# 查询前先检查布隆过滤器
BF.EXISTS user:ids 9999
# 返回0，一定不存在，直接返回
# 返回1，可能存在，继续查缓存/数据库
```

### 方案三：参数校验

```bash
# 接口层校验参数
# ID必须是正整数
# ID必须在有效范围内
# 非法参数直接返回

if id <= 0 or id > max_id:
    return "参数无效"
```

## 四、方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 空值缓存 | 实现简单 | 占用内存 | 通用 |
| 布隆过滤器 | 内存效率高 | 有误判率 | 数据量大 |
| 参数校验 | 彻底解决 | 需要业务配合 | 有规则的ID |

## 五、注意事项

1. **空值过期时间**：设置较短的过期时间（如5分钟）
2. **布隆过滤器容量**：预估准确的数据量
3. **多层防御**：组合使用多种方案
4. **监控告警**：监控穿透率

r = redis.Redis()

class CachePenetrationGuard:
    def __init__(self):

        self.valid_ids = set()
    
    def register_id(self, user_id):
        """注册有效ID到布隆过滤器"""
        self.valid_ids.add(user_id)
        # 生产环境：bf.add('user:ids', user_id)
    
    def get_user(self, user_id):
        key = f"user:{user_id}"
        
        # 方案1：布隆过滤器检查
        if user_id not in self.valid_ids:
            return None  # 一定不存在
        
        # 方案2：查缓存
        cached = r.get(key)
        if cached:
            if cached == b'NULL':
                return None  # 空值缓存
            return json.loads(cached)
        
        # 方案3：查数据库
        user = self._query_db(user_id)
        
        if user:
            # 写入缓存
            r.setex(key, 3600, json.dumps(user))
        else:
            # 缓存空值（短TTL）
            r.setex(key, 300, 'NULL')
        
        return user
    
    def _query_db(self, user_id):

## 七、生产环境监控

def monitor_penetration():
    """监控缓存穿透率"""
    info = r.info('stats')
    hits = info['keyspace_hits']
    misses = info['keyspace_misses']
    
    total = hits + misses
    if total == 0:
        return
    
    hit_rate = hits / total * 100
    miss_rate = misses / total * 100
    
    print(f"命中率: {hit_rate:.2f}%")
    print(f"穿透率: {miss_rate:.2f}%")
    
    if miss_rate > 20:
        print("警告：缓存穿透率过高")
