# 缓存最佳实践

## 一、设计原则

```bash
# 1. 缓存粒度适当
# 不要缓存整个大对象，只缓存需要的字段

# 2. 过期时间必须设置
# 避免内存泄漏
SET key value EX 3600

# 3. Key命名规范
# 业务:类型:标识
user:1001:profile
product:2001:detail
```

## 二、常见问题与解决

```bash
# 缓存穿透 → 布隆过滤器/空值缓存
# 缓存击穿 → 互斥锁/逻辑过期
# 缓存雪崩 → 随机过期/多级缓存
# 数据一致性 → 先更新DB再删缓存
```

## 三、性能优化

```bash
# 1. 使用Pipeline批量操作
pipe = redis.pipeline()
for key in keys:
    pipe.get(key)
results = pipe.execute()

# 2. 避免大Key
# 单个Key不超过10MB

# 3. 使用合适的数据类型
# 对象用Hash（ziplist编码）
# 简单值用String

# 4. 连接池配置
# 合理设置连接池大小
```

## 四、监控与运维

```bash
# 关键指标
# 1. 命中率 > 90%
# 2. 内存使用 < 80%
# 3. 连接数 < 80%最大值
# 4. 慢查询 < 10条/分钟

# 告警规则
# 命中率 < 80% 告警
# 内存 > 80% 告警
# 连接数异常 告警
```
