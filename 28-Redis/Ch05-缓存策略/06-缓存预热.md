# 缓存预热

## 一、概念说明

缓存预热是在系统启动或缓存失效前，主动将热点数据加载到缓存中，避免冷启动时大量请求穿透到数据库。

## 二、预热策略

### 启动时预热

```bash
# 应用启动时加载热点数据
# 从配置文件读取热点Key列表

def warmup_cache():
    hot_keys = ["product:1001", "product:1002", "product:1003"]
    for key in hot_keys:
        data = db.query(key)
        redis.set(key, data, ex=3600)
    print("缓存预热完成")

# Spring Boot中使用@PostConstruct
@PostConstruct
public void init() {
    warmupCache();
}
```

### 定时预热

```bash
# 定时任务提前刷新即将过期的Key
# 使用定时任务每小时检查一次

# Lua脚本检查过期时间
EVAL "
  local ttl = redis.call('TTL', KES[1])
  if ttl < 600 then
    return 1  -- 需要刷新
  end
  return 0
" 1 hot:key
```

## 四、预热脚本示例

```python
import redis
import mysql.connector

def cache_warmup():
    r = redis.Redis(host='localhost', port=6379)
    db = mysql.connector.connect(host='localhost', database='app')
    cursor = db.cursor()
    
    # 预热热门商品
    cursor.execute("SELECT id, data FROM products WHERE hot=1 LIMIT 100")
    for id, data in cursor:
        r.set(f"product:{id}", data, ex=7200)
    
    # 预热热门配置
    cursor.execute("SELECT key, value FROM configs WHERE active=1")
    for key, value in cursor:
        r.set(f"config:{key}", value)
    
    print("预热完成")
```

## 五、注意事项

1. **预热时机**：启动时或低峰期
2. **数据量控制**：避免预热过多数据
3. **超时设置**：合理设置过期时间
4. **监控效果**：监控预热后的命中率
5. **动态调整**：根据访问模式调整预热列表
