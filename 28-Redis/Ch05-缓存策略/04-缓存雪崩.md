# 缓存雪崩

## 一、概念说明

缓存雪崩是指大量Key在同一时间过期或Redis集群宕机，导致所有请求都打到数据库，造成数据库压力骤增甚至宕机。

## 二、问题分析

```bash
# 场景1：大量Key同时过期
SET product:1 data EX 3600
SET product:2 data EX 3600
# 1小时后这些Key同时过期
# 所有请求同时穿透到数据库

# 场景2：Redis集群宕机
# 所有缓存不可用
# 所有请求直接打到数据库
```

## 三、解决方案

### 方案一：随机过期时间

```bash
# 在基础过期时间上增加随机值
import random

expire = 3600 + random.randint(0, 600)
redis.set("product:1001", data, ex=expire)
# 过期时间：3600-4200秒随机
```

### 方案二：多级缓存

```bash
# 本地缓存 → Redis缓存 → 数据库
# 使用Guava/Caffeine做本地缓存
# Redis作为二级缓存
# 数据库作为最后手段

# 架构
# 请求 → 本地缓存 → Redis → 数据库
```

### 方案三：Redis高可用

```bash
# 主从 + 哨兵
# 集群模式
# 避免单点故障

# 降级策略
# Redis不可用时返回默认值或降级数据
```

### 方案四：限流与熔断

```bash
# 接口限流
# 使用令牌桶或漏桶算法
# 限制QPS

# 熔断机制
# 数据库压力过大时熔断
# 返回降级数据
```

## 四、方案对比

| 方案 | 适用场景 | 复杂度 |
|------|----------|--------|
| 随机过期 | 同时过期问题 | 低 |
| 多级缓存 | 高并发场景 | 中 |
| 高可用 | 容灾场景 | 高 |
| 限流熔断 | 极端场景 | 中 |

## 五、注意事项

1. **过期时间设计**：避免大批量Key同时过期
2. **Redis高可用**：主从/哨兵/集群
3. **降级预案**：提前准备降级方案
4. **监控告警**：监控Redis状态和数据库压力
