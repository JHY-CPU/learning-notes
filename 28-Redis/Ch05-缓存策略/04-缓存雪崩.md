# 缓存雪崩

## 一、概念说明

缓存雪崩是指大量Key在同一时间过期或Redis集群宕机，导致所有请求都打到数据库，造成数据库压力骤增甚至宕机。

## 二、问题分析

```bash
# 场景1：大量Key同时过期
SET product:1 data EX 3600
SET product:2 data EX 3600
# 1小时后这些Key同时过期
# 所有请求同时穿透到数据库

# 场景2：Redis集群宕机
# 所有缓存不可用
# 所有请求直接打到数据库
```

## 三、解决方案

### 方案一：随机过期时间

```bash
# 在基础过期时间上增加随机值
import random

expire = 3600 + random.randint(0, 600)
redis.set("product:1001", data, ex=expire)
# 过期时间：3600-4200秒随机
```

### 方案二：多级缓存

```bash
# 本地缓存 → Redis缓存 → 数据库
# 使用Guava/Caffeine做本地缓存
# Redis作为二级缓存
# 数据库作为最后手段

# 架构
# 请求 → 本地缓存 → Redis → 数据库
```

### 方案三：Redis高可用

```bash
# 主从 + 哨兵
# 集群模式
# 避免单点故障

# 降级策略
# Redis不可用时返回默认值或降级数据
```

### 方案四：限流与熔断

```bash
# 接口限流
# 使用令牌桶或漏桶算法
# 限制QPS

# 熔断机制
# 数据库压力过大时熔断
# 返回降级数据
```

## 四、方案对比

| 方案 | 适用场景 | 复杂度 |
|------|----------|--------|
| 随机过期 | 同时过期问题 | 低 |
| 多级缓存 | 高并发场景 | 中 |
| 高可用 | 容灾场景 | 高 |
| 限流熔断 | 极端场景 | 中 |

## 五、注意事项

1. **过期时间设计**：避免大批量Key同时过期
2. **Redis高可用**：主从/哨兵/集群
3. **降级预案**：提前准备降级方案
4. **监控告警**：监控Redis状态和数据库压力

## 六、Python实现防护方案

```python
import redis
import json
import random
import time
from functools import lru_cache

r = redis.Redis()

class CacheAvalancheGuard:
    def __init__(self):
        self.local_cache = {}
        self.local_cache_ttl = {}
    
    def set_with_random_expire(self, key, value, base_ttl=3600, jitter=600):
        """随机过期时间"""
        ttl = base_ttl + random.randint(0, jitter)
        r.setex(key, ttl, json.dumps(value))
        return ttl
    
    def get_with_local_cache(self, key, db_query_func, ttl=3600):
        """多级缓存方案"""
        # 1. 本地缓存
        if key in self.local_cache:
            if time.time() < self.local_cache_ttl.get(key, 0):
                return self.local_cache[key]
            else:
                del self.local_cache[key]
        
        # 2. Redis缓存
        cached = r.get(key)
        if cached:
            value = json.loads(cached)
            # 更新本地缓存
            self.local_cache[key] = value
            self.local_cache_ttl[key] = time.time() + 60  # 本地缓存60秒
            return value
        
        # 3. 数据库
        try:
            value = db_query_func()
            if value:
                self.set_with_random_expire(key, value, ttl)
                self.local_cache[key] = value
                self.local_cache_ttl[key] = time.time() + 60
            return value
        except Exception as e:
            # 降级：返回本地缓存或默认值
            if key in self.local_cache:
                return self.local_cache[key]
            return None

# 使用示例
guard = CacheAvalancheGuard()

def query_products():
    return [{"id": 1, "name": "商品1"}, {"id": 2, "name": "商品2"}]

# 设置随机过期
guard.set_with_random_expire("products:all", query_products(), 3600, 600)

# 多级缓存获取
products = guard.get_with_local_cache("products:all", query_products)
```

## 七、Redis高可用架构

```bash
# 主从复制
# Master处理写操作
# Slave处理读操作
# 避免单点故障

# 哨兵模式
# 自动故障转移
# Master故障时自动提升Slave
# 应用自动切换连接

# 集群模式
# 数据分片到多个节点
# 每个分片有主从
# 支持水平扩展

# 高可用配置
# min-replicas-to-write 1
# 至少1个从节点才能写入
# 避免主节点故障时数据丢失
```

## 八、熔断降级策略

```python
class CircuitBreaker:
    def __init__(self, failure_threshold=5, recovery_timeout=30):
        self.failure_count = 0
        self.failure_threshold = failure_threshold
        self.recovery_timeout = recovery_timeout
        self.last_failure_time = 0
        self.state = "CLOSED"  # CLOSED, OPEN, HALF_OPEN
    
    def call(self, func, fallback=None):
        if self.state == "OPEN":
            # 检查是否可以恢复
            if time.time() - self.last_failure_time > self.recovery_timeout:
                self.state = "HALF_OPEN"
            else:
                return fallback() if fallback else None
        
        try:
            result = func()
            if self.state == "HALF_OPEN":
                self.state = "CLOSED"
                self.failure_count = 0
            return result
        except Exception as e:
            self.failure_count += 1
            self.last_failure_time = time.time()
            
            if self.failure_count >= self.failure_threshold:
                self.state = "OPEN"
                print(f"熔断器开启：{self.failure_count}次失败")
            
            return fallback() if fallback else None

# 使用
breaker = CircuitBreaker(failure_threshold=5, recovery_timeout=30)
result = breaker.call(
    lambda: r.get("key"),
    fallback=lambda: "默认值"
)
```
