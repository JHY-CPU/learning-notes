# 多级缓存架构

## 一、概念说明

多级缓存架构结合本地缓存和分布式缓存，形成多层防护。本地缓存速度快但容量有限，分布式缓存容量大且可共享。

## 二、架构设计

```
请求
  │
  ▼
┌──────────────┐
│  CDN缓存     │ ← 静态资源
├──────────────┤
│  网关缓存    │ ← API响应
├──────────────┤
│  本地缓存    │ ← Guava/Caffeine
├──────────────┤
│  Redis缓存   │ ← 分布式缓存
├──────────────┤
│  数据库      │ ← 持久化存储
└──────────────┘
```

## 三、实现示例

### 本地缓存（Caffeine）

```java
// Spring Boot + Caffeine
Cache<String, Object> cache = Caffeine.newBuilder()
    .maximumSize(10000)
    .expireAfterWrite(5, TimeUnit.MINUTES)
    .build();

// 使用
public Object get(String key) {
    // 1. 查本地缓存
    Object value = cache.getIfPresent(key);
    if (value != null) return value;
    
    // 2. 查Redis
    value = redisTemplate.opsForValue().get(key);
    if (value != null) {
        cache.put(key, value);
        return value;
    }
    
    // 3. 查数据库
    value = db.query(key);
    if (value != null) {
        redisTemplate.opsForValue().set(key, value, 30, TimeUnit.MINUTES);
        cache.put(key, value);
    }
    return value;
}
```

### Redis + 本地缓存

```python
import redis
from cachetools import TTLCache

# 本地缓存：1000个Key，5分钟过期
local_cache = TTLCache(maxsize=1000, ttl=300)
r = redis.Redis()

def get_data(key):
    # 1. 本地缓存
    if key in local_cache:
        return local_cache[key]
    
    # 2. Redis
    data = r.get(key)
    if data:
        local_cache[key] = data
        return data
    
    # 3. 数据库
    data = db_query(key)
    if data:
        r.set(key, data, ex=3600)
        local_cache[key] = data
    return data
```

## 四、缓存同步

```bash
# 本地缓存同步策略
# 1. TTL过期 - 定期失效
# 2. 消息通知 - Redis Pub/Sub通知
# 3. 版本号 - 每次写入增加版本号

# Pub/Sub同步
SUBSCRIBE cache:invalidation
# 当缓存更新时发布消息
PUBLISH cache:invalidation "user:1001"
```

## 五、注意事项

1. **缓存一致性**：多级缓存的一致性更难保证
2. **内存管理**：本地缓存占用JVM内存
3. **同步机制**：选择合适的同步策略
4. **监控各层**：监控各级缓存的命中率
5. **失效策略**：各层使用不同的失效策略
