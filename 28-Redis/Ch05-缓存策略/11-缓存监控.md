# 缓存监控

## 一、概念说明

缓存监控是跟踪Redis缓存的性能和健康状态，包括命中率、内存使用、连接数等关键指标。

## 二、关键指标

### 命中率

```bash
# 查看命中率
INFO stats
# keyspace_hits:100000
# keyspace_misses:10000

# 计算命中率
# 命中率 = hits / (hits + misses)
# 100000 / 110000 ≈ 90.9%

# 健康标准
# > 90% 优秀
# 80-90% 正常
# < 80% 需要优化
```

### 内存使用

```bash
INFO memory
# used_memory_human:2.50GB
# maxmemory_human:4.00GB
# mem_fragmentation_ratio:1.20

# 内存使用率 = used / max
# 2.50 / 4.00 = 62.5%

# 碎片率
# 1.0-1.5 正常
# > 1.5 需要关注
# < 1.0 可能使用了swap
```

### 连接数

```bash
INFO clients
# connected_clients:100
# blocked_clients:5

# 设置最大连接数
maxclients 10000

# 连接数过多可能是连接池泄漏
```

## 三、监控命令

```bash
# 实时监控命令
redis-cli MONITOR  # 生产环境禁用

# 慢查询
SLOWLOG GET 10

# 客户端列表
CLIENT LIST

# 内存分析
MEMORY DOCTOR
MEMORY USAGE key_name

# 信息汇总
INFO ALL
```

## 四、Prometheus + Grafana

```yaml
# prometheus配置
scrape_configs:
  - job_name: 'redis'
    static_configs:
      - targets: ['redis-exporter:9121']

# 关键监控面板
# 1. 命中率趋势
# 2. 内存使用趋势
# 3. 连接数趋势
# 4. 命令执行频率
# 5. 慢查询数量
```

## 五、告警规则

```bash
# 命中率低于80%
keysace_hits / (keyspace_hits + keypace_misses) < 0.8

# 内存使用超过80%
used_memory / maxmemory > 0.8

# 连接数超过80%
connected_clients / maxclients > 0.8

# 慢查询突增
slowlog_len > 100
```

## 六、注意事项

1. **定期检查**：每天检查关键指标
2. **趋势分析**：关注指标变化趋势
3. **基线对比**：建立性能基线
4. **告警响应**：及时响应告警
5. **容量预警**：提前预警容量不足
