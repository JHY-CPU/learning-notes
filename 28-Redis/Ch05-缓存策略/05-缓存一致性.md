# 缓存一致性

## 一、概念说明

缓存一致性是指缓存数据与数据库数据保持一致。由于缓存和数据库是两个独立的系统，保持一致性是一个挑战。

## 二、常见策略

### Cache-Aside（旁路缓存）

```bash
# 读操作
1. 读缓存
2. 命中则返回
3. 未命中则读DB，写缓存

# 写操作
1. 更新DB
2. 删除缓存（推荐）

# 为什么不更新缓存？
# 并发写可能导致脏数据
# 删除更简单可靠
```

### Write-Through（写穿透）

```bash
# 写操作同时更新缓存和DB
1. 更新缓存
2. 更新DB
3. 两个都成功才返回

# 优点：缓存始终最新
# 缺点：写入延迟增加
```

### Write-Behind（写回）

```bash
# 写操作只更新缓存
1. 更新缓存
2. 异步更新DB

# 优点：写入性能高
# 缺点：可能丢失数据
```

## 三、延迟双删

```bash
# 解决并发读写不一致
1. 删除缓存
2. 更新DB
3. 延迟一段时间后再次删除缓存

# 伪代码
redis.del("user:1001")
db.update(user)
sleep(500)  # 等待读请求完成
redis.del("user:1001")  # 再次删除
```

## 四、基于消息队列

```bash
# 更新DB后发送消息
1. 更新DB
2. 发送MQ消息
3. 消费者删除缓存

# 保证最终一致性
```

## 五、注意事项

1. **最终一致性**：大多数场景接受最终一致
2. **删除优于更新**：写操作优先删除缓存
3. **并发控制**：考虑并发读写的顺序
4. **异常处理**：缓存操作失败的重试机制
5. **监控不一致**：定期对比缓存和DB数据
