# 缓存击穿

## 一、概念说明

缓存击穿是指热点Key过期的瞬间，大量并发请求同时打到数据库，导致数据库压力骤增。

## 二、问题分析

```bash
# 热点Key过期
GET hot:product:1001
# Key刚过期，缓存nil
# 1000个并发请求同时查数据库
# 数据库瞬间压力增大

# 对比缓存穿透：查询不存在的数据
# 缓存击穿：数据存在但缓存过期
```

## 三、解决方案

### 方案一：互斥锁

```bash
# 使用Redis分布式锁
SET lock:product:1001 "uuid" NX EX 10
# 获取锁成功，查询数据库并写缓存
# 获取锁失败，等待或返回默认值

# 伪代码
if redis.get("product:1001"):
    return redis.get("product:1001")
if redis.set("lock:product:1001", uuid, NX=True, EX=10):
    data = db.query("product:1001")
    redis.set("product:1001", data, EX=3600)
    return data
else:
    sleep(0.1)
    retry()
```

### 方案二：逻辑过期

```bash
# 不设置物理过期时间
# 在Value中存储逻辑过期时间

SET product:1001 '{"data":"商品信息","expire":1704067200}'

# 读取时检查逻辑过期
value = redis.get("product:1001")
if value.expire < now():
    # 异步更新缓存
    async_update_cache()
return value.data
```

### 方案三：永不过期+异步更新

```bash
# Key永不过期
SET product:1001 data

# 后台任务定期更新
# 每隔一段时间更新热点数据
```

## 四、方案对比

| 方案 | 优点 | 缺点 |
|------|------|------|
| 互斥锁 | 强一致 | 有延迟 |
| 逻辑过期 | 高性能 | 数据可能不一致 |
| 永不过期 | 简单 | 占用内存 |

## 五、注意事项

1. **锁超时时间**：避免死锁
2. **重试机制**：获取锁失败时的处理
3. **热点识别**：识别并保护热点Key
4. **降级策略**：极端情况下的降级

## 六、Python完整实现

```python
import redis
import json
import time
import threading
from uuid import uuid4

r = redis.Redis()

class CacheBreakdownGuard:
    def get_with_mutex(self, key, db_query_func, ttl=3600):
        """互斥锁方案"""
        # 1. 查缓存
        cached = r.get(key)
        if cached:
            return json.loads(cached)
        
        # 2. 获取分布式锁
        lock_key = f"lock:{key}"
        lock_value = str(uuid4())
        acquired = r.set(lock_key, lock_value, nx=True, ex=10)
        
        if acquired:
            try:
                # 双重检查
                cached = r.get(key)
                if cached:
                    return json.loads(cached)
                
                # 3. 查数据库
                data = db_query_func()
                
                # 4. 写缓存
                r.setex(key, ttl, json.dumps(data))
                
                return data
            finally:
                # 5. 释放锁
                self._release_lock(lock_key, lock_value)
        else:
            # 等待后重试
            time.sleep(0.1)
            return self.get_with_mutex(key, db_query_func, ttl)
    
    def _release_lock(self, lock_key, lock_value):
        """原子性释放锁"""
        lua = """
        if redis.call('get', KEYS[1]) == ARGV[1] then
            return redis.call('del', KEYS[1])
        else
            return 0
        end
        """
        r.eval(lua, 1, lock_key, lock_value)
    
    def get_with_logical_expire(self, key, db_query_func, expire_seconds=3600):
        """逻辑过期方案"""
        cached = r.get(key)
        if cached:
            data = json.loads(cached)
            # 检查逻辑过期时间
            if data.get('expire', 0) > time.time():
                return data['data']
            
            # 已过期，异步更新
            threading.Thread(
                target=self._async_update,
                args=(key, db_query_func, expire_seconds)
            ).start()
            
            # 返回旧数据
            return data['data']
        
        # 缓存不存在，同步加载
        data = db_query_func()
        self._set_with_logical_expire(key, data, expire_seconds)
        return data
    
    def _set_with_logical_expire(self, key, data, expire_seconds):
        """设置带逻辑过期的缓存"""
        cache_data = {
            'data': data,
            'expire': time.time() + expire_seconds
        }
        r.set(key, json.dumps(cache_data))
    
    def _async_update(self, key, db_query_func, expire_seconds):
        """异步更新缓存"""
        lock_key = f"lock:{key}"
        lock_value = str(uuid4())
        
        if r.set(lock_key, lock_value, nx=True, ex=10):
            try:
                data = db_query_func()
                self._set_with_logical_expire(key, data, expire_seconds)
            finally:
                self._release_lock(lock_key, lock_value)

# 使用示例
guard = CacheBreakdownGuard()

def query_product():
    return {"id": 1001, "name": "商品A", "price": 99.9}

# 互斥锁方案
product = guard.get_with_mutex("product:1001", query_product)

# 逻辑过期方案
product = guard.get_with_logical_expire("product:1001", query_product)
```

## 七、热点Key识别与保护

```bash
# 识别热点Key
# 方法1：redis-cli --hotkeys
redis-cli --hotkeys

# 方法2：客户端统计
# 在应用层统计Key访问频率
# 超过阈值标记为热点

# 方法3：monitor分析
redis-cli MONITOR | head -n 10000 | awk '{print $4}' | sort | uniq -c | sort -rn

# 保护策略
# 1. 热点Key永不过期 + 异步更新
# 2. 热点Key使用互斥锁
# 3. 热点Key分散到多个Key
# 4. 热点Key使用本地缓存
```
