# 缓存策略概述

## 一、概念说明

缓存策略决定如何在Redis中存储和管理缓存数据。好的缓存策略可以提升性能、减少数据库压力，同时避免缓存穿透、击穿、雪崩等问题。

## 二、核心策略

### 读策略（Cache-Aside）

```bash
# 旁路缓存模式（最常用）
# 1. 先读缓存
# 2. 缓存命中，直接返回
# 3. 缓存未命中，读数据库
# 4. 写入缓存，设置过期时间

GET user:1001
# 返回nil（缓存未命中）
# → 查询数据库
# → SET user:1001 data EX 3600
```

### 写策略

```bash
# 1. 先更新数据库，再更新缓存（常用）
# 2. 先删除缓存，再更新数据库
# 3. 延迟双删

# 先更新DB再删缓存（推荐）
UPDATE users SET name='新名字' WHERE id=1001
DEL user:1001
```

### 失效策略

```bash
# 1. 定时过期 - 设置TTL
SET key value EX 3600

# 2. 惰性删除 - 访问时检查过期
# 3. 定期删除 - 后台定期清理
# 4. 内存淘汰 - maxmemory-policy
```

## 三、常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 缓存穿透 | 查询不存在的数据 | 布隆过滤器/空值缓存 |
| 缓存击穿 | 热点Key过期 | 互utex锁/永不过期 |
| 缓存雪崩 | 大量Key同时过期 | 随机过期时间 |

## 四、注意事项

1. **过期时间必须设置**：避免内存泄漏
2. **热点数据预加载**：启动时加载热门数据
3. **缓存与数据库一致性**：选择合适的同步策略
4. **监控命中率**：低于80%需要优化

## 五、缓存策略详细分析

### Cache-Aside（旁路缓存）详解

```python
def get_user(user_id):
    key = f"user:{user_id}"
    
    # 1. 先查缓存
    cached = redis.get(key)
    if cached:
        return json.loads(cached)
    
    # 2. 缓存未命中，查数据库
    user = db.query("SELECT * FROM users WHERE id = %s", user_id)
    
    # 3. 写入缓存
    if user:
        redis.setex(key, 3600, json.dumps(user))
    
    return user

def update_user(user_id, data):
    # 1. 更新数据库
    db.execute("UPDATE users SET name=%s WHERE id=%s", data['name'], user_id)
    
    # 2. 删除缓存（不是更新）
    redis.delete(f"user:{user_id}")
    
    # 为什么删除而不是更新？
    # 避免并发写导致的脏数据问题
```

### Read-Through（读穿透）

```python
# 缓存层自动处理数据库查询
class ReadThroughCache:
    def __init__(self, redis_client, db_client):
        self.redis = redis_client
        self.db = db_client
    
    def get(self, key):
        # 尝试从缓存获取
        value = self.redis.get(key)
        if value:
            return value
        
        # 缓存未命中，从数据库加载
        value = self.db.load(key)
        if value:
            self.redis.setex(key, 3600, value)
        return value
```

### Write-Through（写穿透）

```python
def write_through(key, value):
    # 同时更新缓存和数据库
    redis.setex(key, 3600, value)
    db.save(key, value)
    # 两个操作都成功才返回
```

## 六、缓存粒度设计

```bash
# 整个对象缓存
SET user:1001 '{"id":1001,"name":"张三","age":25,"email":"zhangsan@example.com"}'
# 优点：一次获取所有数据
# 缺点：更新部分字段时需重新缓存整个对象

# 字段级缓存
HSET user:1001 name "张三"
HSET user:1001 age 25
HSET user:1001 email "zhangsan@example.com"
# 优点：可单独更新字段
# 缺点：多次命令获取所有字段

# 选择建议
# 对象小（<1KB）→ String整体缓存
# 对象大（>1KB）→ Hash字段级缓存
# 只读场景 → String整体缓存
# 频繁更新 → Hash字段级缓存
```

## 七、缓存命中率计算与分析

```bash
# 查看命中统计
INFO stats
# keyspace_hits:95000
# keyspace_misses:5000

# 计算命中率
命中率 = hits / (hits + misses) = 95000 / 100000 = 95%

# 命中率标准
# > 95% 优秀
# 90-95% 良好
# 80-90% 需要优化
# < 80% 严重问题

# 低命中率原因
# 1. 过期时间太短
# 2. 缓存容量不足
# 3. 冷数据太多
# 4. 热点Key变化太快
# 5. 缓存击穿/穿透/雪崩
```
