# 缓存策略概述

## 一、概念说明

缓存策略决定如何在Redis中存储和管理缓存数据。好的缓存策略可以提升性能、减少数据库压力，同时避免缓存穿透、击穿、雪崩等问题。

## 二、核心策略

### 读策略（Cache-Aside）

```bash
# 旁路缓存模式（最常用）
# 1. 先读缓存
# 2. 缓存命中，直接返回
# 3. 缓存未命中，读数据库
# 4. 写入缓存，设置过期时间

GET user:1001
# 返回nil（缓存未命中）
# → 查询数据库
# → SET user:1001 data EX 3600
```

### 写策略

```bash
# 1. 先更新数据库，再更新缓存（常用）
# 2. 先删除缓存，再更新数据库
# 3. 延迟双删

# 先更新DB再删缓存（推荐）
UPDATE users SET name='新名字' WHERE id=1001
DEL user:1001
```

### 失效策略

```bash
# 1. 定时过期 - 设置TTL
SET key value EX 3600

# 2. 惰性删除 - 访问时检查过期
# 3. 定期删除 - 后台定期清理
# 4. 内存淘汰 - maxmemory-policy
```

## 三、常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 缓存穿透 | 查询不存在的数据 | 布隆过滤器/空值缓存 |
| 缓存击穿 | 热点Key过期 | 互utex锁/永不过期 |
| 缓存雪崩 | 大量Key同时过期 | 随机过期时间 |

## 四、注意事项

1. **过期时间必须设置**：避免内存泄漏
2. **热点数据预加载**：启动时加载热门数据
3. **缓存与数据库一致性**：选择合适的同步策略
4. **监控命中率**：低于80%需要优化
