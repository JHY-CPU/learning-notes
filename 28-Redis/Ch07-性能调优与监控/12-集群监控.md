# 集群监控

## 一、概念说明

监控Redis Cluster的状态，包括节点状态、槽位分布、数据迁移等。

## 二、集群状态查询

```bash
# 集群信息
redis-cli -c CLUSTER INFO
# cluster_state:ok
# cluster_slots_assigned:16384
# cluster_known_nodes:6
# cluster_size:3

# 节点信息
redis-cli -c CLUSTER NODES

# 槽位信息
redis-cli -c CLUSTER SLOTS
```

## 三、关键指标

```bash
# 集群状态
cluster_state:ok  # 正常
cluster_state:fail  # 异常

# 槽位覆盖
cluster_slots_assigned:16384  # 所有槽位已分配

# 节点状态
# master/fail?/fail/握手状态

# 数据迁移状态
cluster_slots_ok:16384
cluster_slots_pfail:0
cluster_slots_fail:0
```

## 四、监控脚本

```python
def check_cluster():
    r = redis.Redis(host='192.168.1.100', port=7000)
    
    info = r.cluster_info()
    
    if info['cluster_state'] != 'ok':
        alert("集群状态异常")
    
    if info['cluster_slots_assigned'] != 16384:
        alert("槽位未完全分配")
    
    if info['cluster_slots_pfail'] > 0:
        alert("有节点疑似下线")
```

## 五、注意事项

1. **槽位均衡**：监控槽位是否均匀分布
2. **节点状态**：监控所有节点状态
3. **数据迁移**：迁移期间监控性能
4. **脑裂检测**：监控网络分区
