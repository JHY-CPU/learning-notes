# 慢查询优化

## 一、概念说明

慢查询是Redis性能问题的主要来源。通过分析慢查询日志可以发现和优化性能瓶颈。

## 二、配置慢查询

```bash
# redis.conf
slowlog-log-slower-than 10000  # 10毫秒
slowlog-max-len 128

# 查看慢查询
SLOWLOG GET 10
# 输出: ID, 时间戳, 耗时(微秒), 命令, 客户端IP
```

## 三、常见慢查询

```bash
# 1. KEYS * - O(N)复杂度
# 替代: SCAN 0 MATCH * COUNT 100

# 2. SMEMBERS大Set
# 替代: SSCAN key 0 COUNT 100

# 3. HGETALL大Hash
# 替代: HSCAN key 0 COUNT 100

# 4. SORT无LIMIT
# 替代: SORT key LIMIT 0 10

# 5. 大Key删除
# 替代: UNLINK key（异步删除）
```

## 四、优化建议

```bash
# 1. 使用SCAN替代KEYS
# 2. 分批获取大数据
# 3. 避免大Key
# 4. 使用Pipeline批量操作
# 5. 复杂操作使用Lua脚本
```

## 五、注意事项

1. **阈值设置**：根据业务需求设置合理阈值
2. **定期清理**：SLOWLOG RESET清理日志
3. **监控告警**：慢查询突增时告警
## 六、常见慢查询命令对比

```bash
# KEYS vs SCAN
# KEYS *: O(N)复杂度，阻塞所有命令
# SCAN: O(1)每次迭代，不阻塞

# SMEMBERS vs SSCAN
# SMEMBERS: O(N)，大Set会很慢
# SSCAN: O(1)每次迭代

# HGETALL vs HSCAN
# HGETALL: O(N)，大Hash会很慢
# HSCAN: O(1)每次迭代

# ZRANGE vs ZSCAN
# ZRANGE: O(log(N)+M)，M为返回数量
# ZSCAN: O(1)每次迭代

# SORT vs SORT LIMIT
# SORT key: O(N+M*log(M))
# SORT key LIMIT 0 10: 只返回前10个

# DEL vs UNLINK
# DEL: 同步删除，大Key会阻塞
# UNLINK: 异步删除，不阻塞主线程
```

## 七、慢查询自动化分析

```python
import redis
import time
from collections import Counter

r = redis.Redis()

def analyze_slow_queries(limit=100):
    """分析慢查询"""
    logs = r.slowlog_get(limit)
    
    if not logs:
        print("没有慢查询")
        return
    
    print(f"慢查询分析 (共{len(logs)}条)")
    print("=" * 60)
    
    # 统计命令频率
    cmd_counter = Counter()
    duration_sum = {}
    max_duration = {}
    
    for entry in logs:
        cmd = entry['command'][0] if entry['command'] else 'UNKNOWN'
        duration = entry['duration']
        
        cmd_counter[cmd] += 1
        duration_sum[cmd] = duration_sum.get(cmd, 0) + duration
        max_duration[cmd] = max(max_duration.get(cmd, 0), duration)
    
    # 输出统计
    print("\n命令频率统计:")
    for cmd, count in cmd_counter.most_common(10):
        avg_dur = duration_sum[cmd] / count
        max_dur = max_duration[cmd]
        print(f"  {cmd}: {count}次, 平均{avg_dur:.0f}us, 最大{max_dur}us")
    
    # 最慢的查询
    print("\n最慢的5个查询:")
    sorted_logs = sorted(logs, key=lambda x: x['duration'], reverse=True)
    for i, entry in enumerate(sorted_logs[:5]):
        cmd = ' '.join(entry['command'][:5])
        print(f"  {i+1}. {entry['duration']}us: {cmd}")
    
    # 按时间段分布
    print("\n时间分布:")
    time_buckets = {}
    for entry in logs:
        hour = time.strftime('%H', time.localtime(entry['start_time']))
        time_buckets[hour] = time_buckets.get(hour, 0) + 1
    
    for hour, count in sorted(time_buckets.items()):
        print(f"  {hour}:00 - {count}次")

# 执行分析
analyze_slow_queries(200)
```

## 八、慢查询告警

```python
def monitor_slow_queries(threshold_us=10000, check_interval=60):
    """监控慢查询并告警"""
    last_id = 0
    
    while True:
        logs = r.slowlog_get(10)
        
        for entry in logs:
            if entry['id'] <= last_id:
                continue
            
            if entry['duration'] > threshold_us:
                cmd = ' '.join(entry['command'][:5])
                print(f"告警：慢查询 {entry['duration']}us - {cmd}")
                # 发送告警通知
                # send_alert(...)
        
        if logs:
            last_id = logs[0]['id']
        
        time.sleep(check_interval)
```
