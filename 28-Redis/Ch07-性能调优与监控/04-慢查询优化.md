# 慢查询优化

## 一、概念说明

慢查询是Redis性能问题的主要来源。通过分析慢查询日志可以发现和优化性能瓶颈。

## 二、配置慢查询

```bash
# redis.conf
slowlog-log-slower-than 10000  # 10毫秒
slowlog-max-len 128

# 查看慢查询
SLOWLOG GET 10
# 输出: ID, 时间戳, 耗时(微秒), 命令, 客户端IP
```

## 三、常见慢查询

```bash
# 1. KEYS * - O(N)复杂度
# 替代: SCAN 0 MATCH * COUNT 100

# 2. SMEMBERS大Set
# 替代: SSCAN key 0 COUNT 100

# 3. HGETALL大Hash
# 替代: HSCAN key 0 COUNT 100

# 4. SORT无LIMIT
# 替代: SORT key LIMIT 0 10

# 5. 大Key删除
# 替代: UNLINK key（异步删除）
```

## 四、优化建议

```bash
# 1. 使用SCAN替代KEYS
# 2. 分批获取大数据
# 3. 避免大Key
# 4. 使用Pipeline批量操作
# 5. 复杂操作使用Lua脚本
```

## 五、注意事项

1. **阈值设置**：根据业务需求设置合理阈值
2. **定期清理**：SLOWLOG RESET清理日志
3. **监控告警**：慢查询突增时告警
4. **大Key识别**：使用redis-cli --bigkeys发现大Key
