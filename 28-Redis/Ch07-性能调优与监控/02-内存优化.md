# 内存优化

## 一、概念说明

Redis内存优化包括选择合适的数据结构、控制Key大小、减少内存碎片等。

## 二、数据结构选择

```bash
# Hash使用ziplist编码（字段<128，值<64字节）
hash-max-ziplist-entries 128
hash-max-ziplist-value 64

# Set使用intset编码（元素都是整数且<512个）
set-max-intset-entries 512

# ZSet使用ziplist编码
zset-max-ziplist-entries 128
zset-max-ziplist-value 64

# 查看编码
OBJECT ENCODING key_name
```

## 三、内存优化技巧

```bash
# 1. 缩短Key名称
# 不好: user_profile_information_1001
# 好: u:1001:p

# 2. 使用Hash代替多个String
# 不好: SET user:1001:name "张三"
#        SET user:1001:age 25
# 好: HSET user:1001 name "张三" age 25

# 3. 设置过期时间
SET cache:key value EX 3600

# 4. 压缩Value
# 使用gzip/msgpack压缩大Value
```

## 四、内存碎片处理

```bash
# 查看碎片率
INFO memory | grep mem_fragmentation_ratio
# 正常: 1.0 - 1.5
# 过高: > 1.5（需要整理）
# 过低: < 1.0（可能使用了swap）

# 碎片整理（Redis 4.0+）
CONFIG SET activedefrag yes
# 自动整理内存碎片

# 手动触发
MEMORY PURGE
```

## 五、注意事项

1. **ziplist自动转换**：超过阈值自动转为hashtable
2. **碎片率监控**：定期检查碎片率
3. **swap检查**：确保Redis不使用swap
4. **maxmemory设置**：必须设置最大内存限制
