# 内存碎片

## 一、概念说明

内存碎片是Redis分配的内存与实际使用的内存之间的差值。碎片率过高表示内存利用效率低。

## 二、检查碎片

```bash
# 查看碎片率
INFO memory | grep mem_fragmentation_ratio
# 正常: 1.0 - 1.5
# 过高: > 1.5（需要整理）
# 过低: < 1.0（使用了swap，危险）

# 查看详细内存信息
MEMORY STATS
# 0. peak.allocated
# 1. total.allocated
# 2. startup.allocated
# 3. replication.backlog
# 4. clients.slaves
# 5. clients.normal
# 6. aof.buffer
# 7. lua.caches
# 8. db.0

# 内存诊断
MEMORY DOCTOR
```

## 三、碎片整理

```bash
# Redis 4.0+ 支持自动碎片整理
# redis.conf
activedefrag yes
active-defrag-ignore-bytes 100mb
active-defrag-threshold-lower 10
active-defrag-threshold-upper 100
active-defrag-cycle-min 5
active-defrag-cycle-max 75

# 手动触发碎片整理
MEMORY PURGE

# 关闭自动碎片整理
CONFIG SET activedefrag no
```

## 四、预防碎片

```bash
# 1. 合理的maxmemory设置
maxmemory 4gb

# 2. 使用合适的内存分配器
# jemalloc比glibc更高效

# 3. 避免频繁创建删除大Key

# 4. 使用Hash代替多个String
# ziplist编码更紧凑
```

## 五、注意事项

1. **碎片率监控**：定期检查碎片率
2. **碎片整理开销**：整理期间有CPU开销
3. **swap检查**：碎片率<1.0表示使用了swap
4. **jemalloc**：推荐使用jemalloc分配器
