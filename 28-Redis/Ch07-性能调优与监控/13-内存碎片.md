# 内存碎片

## 一、概念说明

内存碎片是Redis分配的内存与实际使用的内存之间的差值。碎片率过高表示内存利用效率低。

## 二、检查碎片

```bash
# 查看碎片率
INFO memory | grep mem_fragmentation_ratio
# 正常: 1.0 - 1.5
# 过高: > 1.5（需要整理）
# 过低: < 1.0（使用了swap，危险）

# 查看详细内存信息
MEMORY STATS
# 0. peak.allocated
# 1. total.allocated
# 2. startup.allocated
# 3. replication.backlog
# 4. clients.slaves
# 5. clients.normal
# 6. aof.buffer
# 7. lua.caches
# 8. db.0

# 内存诊断
MEMORY DOCTOR
```

## 三、碎片整理

```bash
# Redis 4.0+ 支持自动碎片整理
# redis.conf
activedefrag yes
active-defrag-ignore-bytes 100mb
active-defrag-threshold-lower 10
active-defrag-threshold-upper 100
active-defrag-cycle-min 5
active-defrag-cycle-max 75

# 手动触发碎片整理
MEMORY PURGE

# 关闭自动碎片整理
CONFIG SET activedefrag no
```

## 四、预防碎片

```bash
# 1. 合理的maxmemory设置
maxmemory 4gb

# 2. 使用合适的内存分配器
# jemalloc比glibc更高效

# 3. 避免频繁创建删除大Key

# 4. 使用Hash代替多个String
# ziplist编码更紧凑
```

## 五、注意事项

1. **碎片率监控**：定期检查碎片率
2. **碎片整理开销**：整理期间有CPU开销
3. **swap检查**：碎片率<1.0表示使用了swap
## 六、碎片率详细分析

```bash
# 碎片率计算
frag_ratio = used_memory_rss / used_memory

# 碎片率含义
# 1.0 - 完美，无碎片
# 1.0-1.5 - 正常范围
# > 1.5 - 碎片严重，需要整理
# < 1.0 - 使用了swap，危险

# 查看详细内存信息
INFO memory
# used_memory:1073741824        # Redis实际使用的内存
# used_memory_rss:1288490189    # 操作系统分配的内存（RSS）
# used_memory_peak:1073741824   # 峰值内存
# used_memory_lua:37888         # Lua脚本占用的内存
# mem_fragmentation_ratio:1.20  # 碎片率

# 碎片产生的原因
# 1. 频繁创建删除Key
# 2. Value大小不一致
# 3. 使用了jemalloc分配器的特性
# 4. 数据过期后空间未及时回收
```

## 七、自动碎片整理配置

```bash
# redis.conf 完整配置
# 开启自动碎片整理
activedefrag yes

# 碎片整理触发条件
active-defrag-ignore-bytes 100mb    # 碎片超过100MB时触发
active-defrag-threshold-lower 10    # 碎片率超过10%时触发
active-defrag-threshold-upper 100   # 碎片率超过100%时强制整理

# 碎片整理资源控制
active-defrag-cycle-min 5           # 最小CPU使用率（%）
active-defrag-cycle-max 75          # 最大CPU使用率（%）

# 监控碎片整理状态
INFO memory
# active_defrag_running:0          # 是否正在整理
# active_defrag_hits:1000          # 整理命中次数
# active_defrag_misses:100         # 整理未命中次数
# active_defrag_key_hits:500       # Key整理命中
# active_defrag_key_misses:50      # Key整理未命中
```

## 八、碎片整理性能影响

```python
import redis
import time

r = redis.Redis()

def monitor_defrag_impact():
    """监控碎片整理对性能的影响"""
    # 开始前性能
    print("碎片整理前性能测试...")
    start = time.time()
    for i in range(10000):
        r.set(f"test:{i}", f"value:{i}")
    before_qps = 10000 / (time.time() - start)
    
    # 开启碎片整理
    r.config_set('activedefrag', 'yes')
    time.sleep(5)  # 等待整理开始
    
    # 整理中性能
    print("碎片整理中性能测试...")
    start = time.time()
    for i in range(10000):
        r.set(f"test:{i}", f"value:{i}")
    during_qps = 10000 / (time.time() - start)
    
    # 关闭碎片整理
    r.config_set('activedefrag', 'no')
    time.sleep(2)
    
    # 整理后性能
    print("碎片整理后性能测试...")
    start = time.time()
    for i in range(10000):
        r.set(f"test:{i}", f"value:{i}")
    after_qps = 10000 / (time.time() - start)
    
    print(f"整理前: {before_qps:.0f} QPS")
    print(f"整理中: {during_qps:.0f} QPS (下降 {(1-during_qps/before_qps)*100:.1f}%)")
    print(f"整理后: {after_qps:.0f} QPS")
```

## 九、碎片预防策略

```bash
# 1. 合理设置maxmemory
# 避免内存用尽导致的碎片
maxmemory 4gb

# 2. 使用合适的数据类型
# Hash用ziplist编码（更紧凑）
hash-max-ziplist-entries 128
hash-max-ziplist-value 64

# 3. 避免频繁创建删除大Key
# 大Key删除后留下的碎片难以复用
# 使用UNLINK异步删除

# 4. 定期整理
# 低峰期手动触发
MEMORY PURGE

# 5. 使用jemalloc
# jemalloc比glibc malloc更高效
# 减少内存碎片
# Redis默认使用jemalloc

# 6. 监控碎片率
# 定期检查，超过1.5时告警
INFO memory | grep mem_fragmentation_ratio
```
