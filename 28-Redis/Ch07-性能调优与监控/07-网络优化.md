# 网络优化

## 一、概念说明

网络是Redis性能的重要因素。优化网络配置和减少网络往返可以显著提升性能。

## 二、优化策略

### Pipeline批量操作

```bash
# 减少RTT
cat commands.txt | redis-cli --pipe

# Python
pipe = r.pipeline()
for i in range(1000):
    pipe.set(f"key:{i}", f"value:{i}")
pipe.execute()
```

### TCP配置

```bash
# redis.conf
tcp-backlog 511          # TCP连接队列
tcp-keepalive 300        # TCP保活
timeout 0                # 客户端超时

# 系统参数
sysctl -w net.core.somaxconn=65535
sysctl -w net.ipv4.tcp_max_syn_backlog=65535
```

### 内网部署

```bash
# Redis部署在应用服务器同一内网
# 避免跨机房延迟

# 延迟对比
# 同机房: < 1ms
# 同城: 1-5ms
# 跨城: 20-100ms
```

## 三、多线程IO（Redis 6.0+）

```bash
# redis.conf
io-threads 4
io-threads-do-reads yes

# 多线程处理网络IO
# 命令执行仍是单线程
```

## 四、注意事项

1. **Pipeline大小**：建议100-1000个命令
2. **网络延迟**：监控网络延迟
3. **带宽限制**：监控网络带宽使用
4. **多线程IO**：CPU密集型场景有帮助
