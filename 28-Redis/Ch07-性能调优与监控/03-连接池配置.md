# 连接池配置

## 一、概念说明

合理配置Redis连接池可以减少连接创建开销，提高性能。

## 二、Jedis连接池

```xml
<!-- Maven -->
<dependency>
    <groupId>redis.clients</groupId>
    <artifactId>jedis</artifactId>
</dependency>
```

```java
JedisPoolConfig config = new JedisPoolConfig();
config.setMaxTotal(20);        // 最大连接数
config.setMaxIdle(10);         // 最大空闲连接
config.setMinIdle(5);          // 最小空闲连接
config.setMaxWaitMillis(3000); // 最大等待时间
config.setTestOnBorrow(true);  // 取连接时测试

JedisPool pool = new JedisPool(config, "192.168.1.100", 6379);
```

## 四、Python redis-py

```python
import redis

pool = redis.ConnectionPool(
    host='192.168.1.100',
    port=6379,
    max_connections=20,
    socket_timeout=5,
    socket_connect_timeout=5
)

r = redis.Redis(connection_pool=pool)
```

## 六、配置建议

```bash
# 连接池大小 = 2 * CPU核数 + 磁盘数
# 示例: 4核CPU, 1磁盘 → 2*4+1=9

# 避免过大连接池
# 每个连接占用Redis资源
# 监控连接数
INFO clients | grep connected_clients
```

## 七、注意事项

1. **连接池大小**：根据并发量和Redis容量设置
2. **超时设置**：避免长时间等待
3. **连接测试**：取连接前验证有效性

## 九、连接池监控

```python
import redis
import time

class ConnectionPoolMonitor:
    """连接池监控"""
    
    def __init__(self, host='localhost', port=6379):
        self.r = redis.Redis(host=host, port=port)
    
    def get_pool_stats(self):
        """获取连接池统计"""
        info = self.r.info('clients')
        
        return {
            'connected_clients': info['connected_clients'],
            'blocked_clients': info['blocked_clients'],
            'maxclients': info['maxclients'],
            'tracking_clients': info.get('tracking_clients', 0),
            'clients_in_timeout_table': info.get('clients_in_timeout_table', 0)
        }
    
    def get_client_list(self):
        """获取客户端列表"""
        clients = self.r.client_list()
        
        result = []
        for client in clients:
            result.append({
                'id': client.get('id'),
                'addr': client.get('addr'),
                'fd': client.get('fd'),
                'name': client.get('name', ''),
                'age': client.get('age'),
                'idle': client.get('idle'),
                'cmd': client.get('cmd')
            })
        
        return result
    
    def kill_idle_clients(self, max_idle_seconds=3600):
        """杀死空闲连接"""
        clients = self.r.client_list()
        killed = 0
        
        for client in clients:
            idle = int(client.get('idle', 0))
            if idle > max_idle_seconds:
                client_id = client.get('id')
                self.r.client_kill(client_id)
                killed += 1
        
        return killed

# 使用
monitor = ConnectionPoolMonitor()
stats = monitor.get_pool_stats()
print(f"连接数: {stats['connected_clients']}/{stats['maxclients']}")
```

## 十、连接泄漏检测

```python
def detect_connection_leak(host='localhost', port=6379):
    """检测连接泄漏"""
    r = redis.Redis(host=host, port=port)
    info = r.info('clients')
    
    connected = info['connected_clients']
    maxclients = info['maxclients']
    
    usage_rate = connected / maxclients * 100
    
    print(f"连接使用率: {usage_rate:.1f}%")
    
    if usage_rate > 80:
        print("警告：连接使用率过高，可能存在泄漏")
        
        # 获取客户端详情
        clients = r.client_list()
        
        # 按命令分组统计
        cmd_stats = {}
        for client in clients:
            cmd = client.get('cmd', 'unknown')
            cmd_stats[cmd] = cmd_stats.get(cmd, 0) + 1
        
        print("命令分布:")
        for cmd, count in sorted(cmd_stats.items(), key=lambda x: x[1], reverse=True):
            print(f"  {cmd}: {count}")
        
        # 查找长时间空闲的连接
        idle_clients = [c for c in clients if int(c.get('idle', 0)) > 300]
        if idle_clients:
            print(f"发现 {len(idle_clients)} 个空闲超过5分钟的连接")
    
    return usage_rate
```
