# 故障排查

## 一、概念说明

Redis故障排查需要系统地检查各个方面，快速定位问题根因。

## 二、常见问题

### 连接问题

```bash
# 无法连接
# 检查1：服务是否运行
ps aux | grep redis

# 检查2：端口是否监听
netstat -tlnp | grep 6379

# 检查3：防火墙
iptables -L -n | grep 6379

# 检查4：绑定地址
CONFIG GET bind
```

### 性能问题

```bash
# 慢查询
SLOWLOG GET 10

# 大Key
redis-cli --bigkeys

# 热Key
redis-cli --hotkeys

# 内存碎片
INFO memory | grep mem_fragmentation_ratio
```

### 数据问题

```bash
# 数据丢失
# 检查持久化状态
INFO persistence

# 检查RDB状态
rdb_last_bgsave_status:ok

# 检查AOF状态
aof_last_bgrewrite_status:ok
```

## 三、排查流程

```bash
1. 确认问题现象
   │
2. 检查服务状态
   │
3. 检查日志文件
   │
4. 检查关键指标
   │
5. 分析慢查询
   │
6. 检查内存/CPU/网络
   │
7. 定位根因并解决
```

## 四、日志分析

```bash
# 日志文件位置
logfile /var/log/redis/redis-server.log

# 常见错误
# OOM - 内存不足
# Can't save in background - fork失败
# Client closed connection - 客户端断开
# MASTER aborted replication - 复制中断
```

## 五、注意事项

1. **日志级别**：loglevel设置为notice
2. **定期检查**：定期检查日志和指标
3. **监控告警**：关键问题及时告警
4. **备份恢复**：定期备份，故障时快速恢复
