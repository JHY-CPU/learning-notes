# 故障排查

## 一、概念说明

Redis故障排查需要系统地检查各个方面，快速定位问题根因。

## 二、常见问题

### 连接问题

```bash
# 无法连接
# 检查1：服务是否运行
ps aux | grep redis

# 检查2：端口是否监听
netstat -tlnp | grep 6379

# 检查3：防火墙
iptables -L -n | grep 6379

# 检查4：绑定地址
CONFIG GET bind
```

### 性能问题

```bash
# 慢查询
SLOWLOG GET 10

# 大Key
redis-cli --bigkeys

# 热Key
redis-cli --hotkeys

# 内存碎片
INFO memory | grep mem_fragmentation_ratio
```

### 数据问题

```bash
# 数据丢失
# 检查持久化状态
INFO persistence

# 检查RDB状态
rdb_last_bgsave_status:ok

# 检查AOF状态
aof_last_bgrewrite_status:ok
```

## 三、排查流程

```bash
1. 确认问题现象
   │
2. 检查服务状态
   │
3. 检查日志文件
   │
4. 检查关键指标
   │
5. 分析慢查询
   │
6. 检查内存/CPU/网络
   │
7. 定位根因并解决
```

## 四、日志分析

```bash
# 日志文件位置
logfile /var/log/redis/redis-server.log

# 常见错误
# OOM - 内存不足
# Can't save in background - fork失败
# Client closed connection - 客户端断开
# MASTER aborted replication - 复制中断
```

## 五、注意事项

1. **日志级别**：loglevel设置为notice
2. **定期检查**：定期检查日志和指标
3. **监控告警**：关键问题及时告警
## 六、常见错误及解决

```bash
# 错误1: OOM (Out of Memory)
# 原因：内存不足
# 解决：
# 1. 增加maxmemory
# 2. 设置淘汰策略
# 3. 清理大Key
# 4. 优化数据结构

# 错误2: Can't save in background
# 原因：fork失败，内存不足
# 解决：
# 1. 检查系统内存
# 2. 增加swap空间
# 3. 减少Redis内存使用

# 错误3: Client closed connection
# 原因：客户端超时断开
# 解决：
# 1. 增加timeout配置
# 2. 检查网络稳定性
# 3. 配置TCP keepalive

# 错误4: MASTER aborted replication
# 原因：复制中断
# 解决：
# 1. 检查网络连通性
# 2. 检查主从密码配置
# 3. 检查复制积压缓冲区大小

# 错误5: MISCONF Redis is configured to save RDB snapshots
# 厌因：RDB持久化失败
# 解决：
# 1. 检查磁盘空间
# 2. 检查目录权限
# 3. 临时禁用stop-writes-on-bgsave-error

# 错误6: LOADING Redis is loading the dataset in memory
# 原因：正在加载数据
# 解决：等待加载完成
```

## 七、故障排查流程

```bash
# 1. 确认问题现象
# - 连接失败？
# - 性能下降？
# - 数据丢失？

# 2. 检查服务状态
ps aux | grep redis
systemctl status redis

# 3. 检查日志
tail -f /var/log/redis/redis-server.log

# 4. 检查关键指标
redis-cli INFO memory
redis-cli INFO stats
redis-cli INFO replication

# 5. 检查系统资源
free -h            # 内存
df -h              # 磁盘
top -p $(pgrep redis-server)  # CPU

# 6. 检查网络
ping redis-host
telnet redis-host 6379

# 7. 检查慢查询
redis-cli SLOWLOG GET 10

# 8. 检查大Key
redis-cli --bigkeys

# 9. 定位根因并解决

# 10. 验证修复
redis-cli PING
```

## 八、故障恢复工具

```python
import redis
import time

r = redis.Redis()

def diagnose_redis():
    """Redis诊断工具"""
    print("Redis故障诊断")
    print("=" * 50)
    
    # 连接检查
    try:
        r.ping()
        print("✓ 连接正常")
    except Exception as e:
        print(f"✗ 连接失败: {e}")
        return
    
    # 内存检查
    info = r.info('memory')
    used = info['used_memory']
    maxmem = info.get('maxmemory', 0)
    if maxmem > 0:
        usage = used / maxmem * 100
        if usage > 90:
            print(f"✗ 内存使用过高: {usage:.1f}%")
        else:
            print(f"✓ 内存使用: {usage:.1f}%")
    
    # 碎片率检查
    frag = info['mem_fragmentation_ratio']
    if frag > 1.5:
        print(f"⚠ 内存碎片过高: {frag:.2f}")
    elif frag < 1.0:
        print(f"✗ 使用了swap: {frag:.2f}")
    else:
        print(f"✓ 碎片率: {frag:.2f}")
    
    # 命中率检查
    stats = r.info('stats')
    hits = stats['keyspace_hits']
    misses = stats['keyspace_misses']
    total = hits + misses
    if total > 0:
        hit_rate = hits / total * 100
        if hit_rate < 80:
            print(f"⚠ 命中率过低: {hit_rate:.1f}%")
        else:
            print(f"✓ 命中率: {hit_rate:.1f}%")
    
    # 慢查询检查
    slow_logs = r.slowlog_get(10)
    if slow_logs:
        print(f"⚠ 发现 {len(slow_logs)} 条慢查询")
        for log in slow_logs[:3]:
            cmd = ' '.join(log['command'][:3])
            print(f"  {log['duration']}us: {cmd}")
    else:
        print("✓ 无慢查询")
    
    # 主从检查
    repl = r.info('replication')
    role = repl.get('role', 'unknown')
    if role == 'master':
        slaves = repl.get('connected_slaves', 0)
        if slaves == 0:
            print("⚠ 无从节点连接")
        else:
            print(f"✓ 从节点: {slaves}")
    else:
        link_status = repl.get('master_link_status', 'unknown')
        if link_status != 'up':
            print(f"✗ 主节点连接异常: {link_status}")
        else:
            print(f"✓ 主节点连接正常")

diagnose_redis()
```
