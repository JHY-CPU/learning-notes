# 哨兵配置详解

## 一、概念说明

Sentinel配置包括监控参数、故障转移参数、通知参数等。合理配置这些参数可以确保高可用性和快速故障恢复。

## 二、完整配置示例

```bash
# sentinel.conf 完整配置

# Sentinel端口
port 26379

# 工作目录
dir /var/lib/redis/sentinel

# 日志文件
logfile /var/log/redis/sentinel.log

# 监控主节点
sentinel monitor mymaster 192.168.1.100 6379 2

# 主节点密码
sentinel auth-pass mymaster yourpassword

# 主观下线时间（毫秒）
sentinel down-after-milliseconds mymaster 5000

# 故障转移超时（毫秒）
sentinel failover-timeout mymaster 60000

# 并行同步从节点数
sentinel parallel-syncs mymaster 1

# 通知脚本
sentinel notification-script mymaster /var/redis/notify.sh

# 客户端重配置脚本
sentinel client-reconfig-script mymaster /var/redis/reconfig.sh
```

## 三、关键参数详解

### 监控参数

```bash
# 格式：sentinel monitor <name> <ip> <port> <quorum>
# quorum：客观下线需要的Sentinel同意数

# 3个Sentinel的推荐配置
sentinel monitor mymaster 192.168.1.100 6379 2
# 2个Sentinel同意即可确认下线

# 5个Sentinel的推荐配置
sentinel monitor mymaster 192.168.1.100 6379 3
# 3个Sentinel同意即可确认下线
```

### 故障检测

```bash
# 主观下线时间
sentinel down-after-milliseconds mymaster 5000
# 5秒内没有响应则认为主观下线

# 调整建议
# 太短：网络抖动导致频繁误判
# 太长：故障恢复时间过长
# 一般设置：5000-30000毫秒
```

### 故障转移

```bash
# 故障转移超时
sentinel failover-timeout mymaster 60000
# 60秒内未完成则放弃

# 并行同步数
sentinel parallel-syncs mymaster 1
# 1：每次只同步一个从节点（稳定）
# 大于1：更快但更耗资源

# 从节点优先级
# 在从节点配置文件中设置
replica-priority 100
# 数值越小优先级越高
# 0表示永不提升为主节点
```

## 四、通知配置

```bash
# 事件通知脚本
sentinel notification-script mymaster /var/redis/notify.sh
# 脚本接收两个参数：事件类型、事件描述

# 事件类型
# +reset-master
# +slave
# +sdown（主观下线）
# +odown（客观下线）
# +failover-triggered
# +failover-state-reconf-slaves
# +failover-end

# 客户端重配置脚本
sentinel client-reconfig-script mymaster /var/redis/reconfig.sh
# 故障转移后执行，通知客户端更新配置
```

## 五、注意事项与常见陷阱

1. **quorum不是选举人数**：quorum是客观下线的判断阈值
2. **Sentinel至少3个**：避免脑裂问题
3. **故障转移时间**：down-after-milliseconds + failover-timeout
4. **密码同步**：Sentinel需要配置主节点密码
5. **配置自动更新**：Sentinel会自动更新sentinel.conf
6. **级联故障**：多个主节点同时故障的处理
