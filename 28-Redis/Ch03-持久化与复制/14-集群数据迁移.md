# 集群数据迁移

## 一、概念说明

Redis Cluster支持在线数据迁移，可以在不停服的情况下添加或删除节点，自动进行槽位（slot）迁移。迁移过程对客户端透明。

## 二、添加新节点

```bash
# 启动新节点
redis-server /etc/redis/cluster/redis-7006.conf

# 将新节点加入集群
redis-cli --cluster add-node \
  192.168.1.100:7006 \
  192.168.1.100:7000 \
  -a yourpassword

# 添加从节点
redis-cli --cluster add-node \
  192.168.1.100:7007 \
  192.168.1.100:7000 \
  --cluster-slave \
  --cluster-master-id <master-node-id> \
  -a yourpassword
```

## 三、槽位迁移（Reshard）

```bash
# 重新分配槽位
redis-cli --cluster reshard \
  192.168.1.100:7000 \
  -a yourpassword

# 交互式迁移
# How many slots do you want to move: 4096
# What is the receiving node ID: <new-node-id>
# Source node #1: all    # 从所有节点迁移
# 或
# Source node #1: <source-node-id>  # 从指定节点迁移

# 非交互式迁移
redis-cli --cluster reshard \
  192.168.1.100:7000 \
  --cluster-from <source-node-id> \
  --cluster-to <target-node-id> \
  --cluster-slots 4096 \
  --cluster-yes \
  -a yourpassword
```

### 迁移过程

```
1. 目标节点准备接收槽位
    │
2. 源节点设置槽位为迁移中状态
    │
3. 源节点逐个迁移key到目标节点
    │
4. 迁移完成后更新槽位映射
    │
5. 所有节点更新槽位信息
```

## 四、删除节点

```bash
# 先迁移该节点的槽位
redis-cli --cluster reshard \
  192.168.1.100:7000 \
  --cluster-from <node-to-remove-id> \
  --cluster-to <other-node-id> \
  --cluster-slots 16384 \
  --cluster-yes \
  -a yourpassword

# 删除空节点
redis-cli --cluster del-node \
  192.168.1.100:7000 \
  <node-to-remove-id> \
  -a yourpassword
```

## 五、集群均衡

```bash
# 自动均衡槽位
redis-cli --cluster rebalance \
  192.168.1.100:7000 \
  -a yourpassword

# 均衡并设置权重
redis-cli --cluster rebalance \
  192.168.1.100:7000 \
  --cluster-weight <node-id>=<weight> \
  -a yourpassword
```

## 六、注意事项与常见陷阱

1. **迁移期间性能**：大量key迁移会影响性能
2. **大Key迁移**：大Key迁移会阻塞相关槽位
3. **客户端重定向**：迁移期间客户端可能收到ASK/MOVED重定向
4. **数据一致性**：迁移过程中短暂的数据不一致
5. **超时设置**：大Key迁移可能超时，需要调整cluster-node-timeout
6. **备份**：迁移前建议备份数据
