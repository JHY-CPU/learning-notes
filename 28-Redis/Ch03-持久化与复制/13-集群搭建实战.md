# 集群搭建实战

## 一、概念说明

Redis Cluster搭建需要准备多个Redis实例，配置集群模式，然后使用redis-cli创建集群。本节介绍3主3从的最小集群搭建方法。

## 二、环境准备

```bash
# 创建6个节点配置文件
# 节点1：192.168.1.100:7000
# 节点2：192.168.1.100:7001
# 节点3：192.168.1.100:7002
# 节点4：192.168.1.100:7003
# 节点5：192.168.1.100:7004
# 节点6：192.168.1.100:7005

# 创建配置目录
mkdir -p /etc/redis/cluster/{7000,7001,7002,7003,7004,7005}
```

### 节点配置文件

```bash
# redis-7000.conf
port 7000
bind 0.0.0.0
daemonize yes
pidfile /var/run/redis-7000.pid
logfile /var/log/redis/redis-7000.log
dir /var/lib/redis/7000

# 开启集群模式
cluster-enabled yes
cluster-config-file nodes-7000.conf
cluster-node-timeout 5000

# 持久化
appendonly yes
aof-use-rdb-preamble yes

# 安全
requirepass "yourpassword"
masterauth "yourpassword"
```

```bash
# 其他节点类似，修改端口号即可
# redis-7001.conf - port 7001
# redis-7002.conf - port 7002
# ...
```

### 启动所有节点

```bash
# 启动6个Redis实例
redis-server /etc/redis/cluster/redis-7000.conf
redis-server /etc/redis/cluster/redis-7001.conf
redis-server /etc/redis/cluster/redis-7002.conf
redis-server /etc/redis/cluster/redis-7003.conf
redis-server /etc/redis/cluster/redis-7004.conf
redis-server /etc/redis/cluster/redis-7005.conf

# 验证启动
ps aux | grep redis
# 输出: 6个redis-server进程
```

## 三、创建集群

```bash
# 使用redis-cli创建集群
redis-cli --cluster create \
  192.168.1.100:7000 \
  192.168.1.100:7001 \
  192.168.1.100:7002 \
  192.168.1.100:7003 \
  192.168.1.100:7004 \
  192.168.1.100:7005 \
  --cluster-replicas 1 \
  -a yourpassword

# --cluster-replicas 1 表示每个主节点有1个从节点

# 输出示例
# Master[0] -> Slots 0 - 5460
# Master[1] -> Slots 5461 - 10922
# Master[2] -> Slots 10923 - 16383
# Adding replica 192.168.1.100:7004 to 192.168.1.100:7000
# ...

# 确认创建
# 输入 yes 确认
```

## 四、验证集群

```bash
# 连接到集群
redis-cli -c -p 7000 -a yourpassword

# 查看集群信息
CLUSTER INFO
# 输出: cluster_state:ok
#       cluster_slots_assigned:16384
#       cluster_known_nodes:6

# 查看节点信息
CLUSTER NODES

# 测试数据分片
SET key1 "value1"
# 输出: -> Redirected to slot [9189] located at ...

SET key2 "value2"
# 输出: -> Redirected to slot [4998] located at ...
```

## 五、注意事项与常见陷阱

1. **密码一致性**：所有节点密码必须一致
2. **网络连通**：所有节点之间必须能互相通信
3. **时间同步**：所有节点时间必须同步
4. **防火墙**：开放集群总线端口（端口+10000）
5. **数据目录**：确保每个节点的数据目录独立
6. **IP地址**：使用真实IP，不要用127.0.0.1
