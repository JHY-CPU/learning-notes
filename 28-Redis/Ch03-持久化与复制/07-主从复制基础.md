# 主从复制基础

## 一、概念说明

主从复制是将一个Redis服务器（主节点Master）的数据复制到一个或多个Redis服务器（从节点Slave/Replica）的过程。主节点负责写操作，从节点负责读操作，实现读写分离和数据备份。

## 二、具体用法

### 配置主从复制

```bash
# 方式1：配置文件
# 在从节点配置文件中添加
replicaof 192.168.1.100 6379

# 如果主节点有密码
masterauth "master_password"

# 方式2：命令行配置
# 在从节点执行
REPLICAOF 192.168.1.100 6379

# 取消复制（提升为主节点）
REPLICAOF NO ONE
```

### 查看复制状态

```bash
# 在主节点查看
INFO replication
# 输出:
# role:master
# connected_slaves:2
# slave0:ip=192.168.1.101,port=6379,state=online,offset=1000,lag=0
# slave1:ip=192.168.1.102,port=6379,state=online,offset=1000,lag=0

# 在从节点查看
INFO replication
# 输出:
# role:slave
# master_host:192.168.1.100
# master_port:6379
# master_link_status:up
# master_last_io_seconds_ago:1
# master_sync_in_progress:0
```

### 读写分离

```bash
# 主节点处理写操作
redis-cli -h 192.168.1.100 -p 6379
SET key "value"
# 输出: OK

# 从节点处理读操作
redis-cli -h 192.168.1.101 -p 6379
GET key
# 输出: "value"

# 从节点不能写入（默认）
SET key "new_value"
# 输出: (error) READONLY You can't write against a read only replica
```

## 三、复制原理简述

```
从节点连接主节点
    │
    ▼
全量同步（首次连接）
    ├── 主节点BGSAVE生成RDB
    ├── 发送RDB文件到从节点
    └── 从节点加载RDB
    │
    ▼
增量同步（持续）
    ├── 主节点将写命令发送到从节点
    └── 从节点执行写命令
```

## 四、配置参数

```bash
# 主节点配置
# 允许从节点最大延迟秒数
min-replicas-max-lag 10

# 允许的最少从节点数
min-replicas-to-write 1

# 从节点配置
# 从节点只读（默认开启）
replica-read-only yes

# 从节点是否使用磁盘同步
repl-diskless-sync no
# no: 先写磁盘再发送
# yes: 直接通过网络发送
```

## 五、注意事项与常见陷阱

1. **从节点是只读的**：默认不能写入数据
2. **复制延迟**：异步复制，从节点数据可能有延迟
3. **主节点故障**：需要手动提升从节点或使用哨兵
4. **级联复制**：从节点可以作为其他从节点的主节点
5. **复制缓冲区**：主节点维护复制缓冲区，从节点断线后可增量同步
6. **密码认证**：从节点需要配置masterauth密码
