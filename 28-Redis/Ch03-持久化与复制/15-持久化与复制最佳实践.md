# 持久化与复制最佳实践

## 一、概念说明

生产环境中，合理配置持久化和复制策略对数据安全和系统性能至关重要。本节总结最佳实践配置。

## 二、持久化最佳实践

### 推荐配置

```bash
# 推荐：混合持久化
appendonly yes
aof-use-rdb-preamble yes
appendfsync everysec

# RDB配置
save 900 1
save 300 10
save 60 10000

# AOF重写配置
auto-aof-rewrite-percentage 100
auto-aof-rewrite-min-size 64mb
no-appendfsync-on-rewrite yes
```

### 备份策略

```bash
# 每日备份RDB文件
0 2 * * * cp /var/lib/redis/dump.rdb /backup/redis/dump-$(date +\%Y\%m\%d).rdb

# 异地备份
0 3 * * * rsync -avz /backup/redis/ backup-server:/redis-backup/

# 定期验证备份
0 4 * * 0 redis-check-rdb /backup/redis/latest.rdb
```

## 三、主从复制最佳实践

```bash
# 主节点配置
min-replicas-to-write 1
min-replicas-max-lag 10

# 从节点配置
replica-read-only yes
repl-backlog-size 100mb

# 网络优化
repl-disable-tcp-nodelay no
```

## 四、Sentinel最佳实践

```bash
# 3个Sentinel配置
port 26379
sentinel monitor mymaster 192.168.1.100 6379 2
sentinel down-after-milliseconds mymaster 5000
sentinel failover-timeout mymaster 60000
sentinel parallel-syncs mymaster 1

# 部署建议
# Sentinel分布在不同机器
# 不要和Redis实例在同一台机器
```

## 六、监控检查清单

```bash
# 每日检查
INFO persistence
INFO replication
INFO memory
INFO clients

# 关键指标
# 1. 内存使用率 < 75%
# 2. 主从延迟 < 1秒
# 3. 持久化状态正常
# 4. 连接数正常
# 5. 慢查询数量可控
```

## 七、灾难恢复

```bash
# 主节点故障恢复
# 1. 检查Sentinel是否自动故障转移
# 2. 验证新主节点数据完整性
# 3. 修复故障节点并加入集群

# 数据恢复
# 1. 停止Redis
# 2. 使用最近的RDB/AOF文件恢复
# 3. 启动Redis验证数据

# 集群恢复
# 1. 检查CLUSTER INFO
# 2. 修复故障节点
# 3. 重新分配槽位
```

## 八、注意事项

1. **定期测试恢复**：定期模拟故障，验证恢复流程
2. **监控告警**：设置关键指标的告警阈值
3. **容量规划**：提前规划存储容量和扩展方案
4. **版本升级**：滚动升级，避免同时升级所有节点
5. **安全加固**：定期更新密码，限制网络访问
