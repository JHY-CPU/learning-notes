# 主从复制配置

## 一、概念说明

合理配置主从复制参数可以提高复制效率和数据安全性。需要根据业务需求调整同步策略、超时时间和网络参数。

## 二、关键配置参数

### 主节点配置

```bash
# 最少从节点数
min-replicas-to-write 1
# 至少有1个从节点连接时才接受写操作

# 从节点最大延迟
min-replicas-max-lag 10
# 从节点延迟超过10秒时拒绝写操作

# 复制积压缓冲区大小
repl-backlog-size 100mb
# 建议设置为：写入速度 * 断线容忍时间

# 缓冲区超时时间
repl-backlog-ttl 3600
# 从节点断线超过1小时后释放缓冲区
```

### 从节点配置

```bash
# 主节点地址
replicaof 192.168.1.100 6379

# 主节点密码
masterauth "password"

# 从节点只读
replica-read-only yes

# 无磁盘复制
repl-diskless-sync no
repl-diskless-sync-delay 5

# 从节点过期策略
replica-ignore-maxmemory-policy no

# 复制超时时间
repl-timeout 60
```

### 网络优化

```bash
# TCP_NODELAY
repl-disable-tcp-nodelay no
# no: 实时性好，但小包多
# yes: 批量发送，减少网络开销

# 发送缓冲区大小
client-output-buffer-limit replica 256mb 64mb 60
# 硬限制：256MB
# 软限制：64MB
# 软限制时间：60秒
```

```bash
# 刚写入的数据可能从节点还未同步

```

## 四、监控指标

```bash
# 关键监控指标
INFO replication

# 复制偏移量差值
master_repl_offset - slave_repl_offset
# 差值越大，延迟越大

# 连接状态
master_link_status:up  # 正常
master_link_status:down  # 异常

# 上次IO时间
master_last_io_seconds_ago:1
# 越大表示越长时间没有通信
```

## 五、注意事项与常见陷阱

1. **repl-backlog-size**：根据写入量和可容忍断线时间设置
2. **min-replicas-to-write**：防止所有从节点断线时数据丢失
3. **从节点过期**：从节点也会执行过期删除，但有延迟
4. **客户端缓冲区**：设置过小会导致从节点断开
5. **无磁盘复制**：主节点磁盘慢时建议开启
6. **级联复制**：减少主节点的复制压力
