# 持久化概述

## 一、概念说明

Redis持久化是将内存中的数据保存到磁盘的过程，防止服务器重启导致数据丢失。Redis提供三种持久化方式：RDB快照、AOF日志、混合持久化。

## 二、持久化方式对比

| 特性 | RDB | AOF | 混合 |
|------|-----|-----|------|
| 持久化方式 | 定期快照 | 追加日志 | RDB+AOF |
| 数据安全性 | 较低 | 较高 | 高 |
| 文件大小 | 较小 | 较大 | 中等 |
| 恢复速度 | 快 | 慢 | 快 |
| 性能影响 | 有（fork） | 有（fsync） | 有 |
| 适用场景 | 备份/容灾 | 数据安全 | 推荐 |

## 三、RDB快照

```bash
# 定期将内存数据快照保存到磁盘
# 触发方式：手动SAVE/BGSAVE 或 配置自动保存

# 配置文件示例
save 900 1       # 900秒内有1个key变化
save 300 10      # 300秒内有10个key变化
save 60 10000    # 60秒内有10000个key变化

# 优点：文件小、恢复快
# 缺点：可能丢失最后一次快照后的数据
```

## 四、AOF日志

```bash
# 将每个写操作追加到日志文件
# 重启时重放日志恢复数据

# 配置文件示例
appendonly yes
appendfsync everysec  # 每秒同步

# 同步策略
# always - 每个命令都同步（最安全，最慢）
# everysec - 每秒同步（推荐）
# no - 操作系统决定（最快，最不安全）

# 优点：数据安全性高
# 缺点：文件大、恢复慢
```

## 五、混合持久化

```bash
# Redis 4.0+支持
# AOF重写时使用RDB格式记录全量数据
# 后续增量数据使用AOF格式

# 配置
aof-use-rdb-preamble yes

# 优点：兼顾安全性和恢复速度
# 推荐生产环境使用
```

## 六、注意事项与常见陷阱

1. **RDB可能丢失数据**：两次快照之间的数据会丢失
2. **AOF文件膨胀**：需要定期重写
3. **fork开销**：RDB和AOF重写都需要fork子进程
4. **磁盘IO**：持久化会产生磁盘IO，影响性能
5. **生产环境推荐混合持久化**：兼顾安全和性能
6. **备份策略**：定期将持久化文件备份到远程存储
