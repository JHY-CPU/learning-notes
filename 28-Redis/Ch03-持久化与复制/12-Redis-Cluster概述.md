# Redis Cluster概述

## 一、概念说明

Redis Cluster是Redis的分布式解决方案，提供数据分片、高可用性和水平扩展能力。数据通过哈希槽（Hash Slot）分布到多个节点，每个节点负责一部分槽位。

## 二、核心概念

### 哈希槽

```bash
# Redis Cluster使用16384个哈希槽
# 每个key通过CRC16(key) % 16384确定所属槽位

# 槽位分配示例
# 节点A：0-5460
# 节点B：5461-10922
# 节点C：10923-16383

# 查看槽位信息
redis-cli -c CLUSTER SLOTS
# 输出: 槽位范围和对应的节点信息
```

### 集群节点

```bash
# 每个节点是一个Redis实例
# 至少需要3个主节点
# 从节点用于故障转移

# 查看集群节点
redis-cli -c CLUSTER NODES
# 输出: 所有节点信息、槽位分配、主从关系
```

### 数据分片

```bash
# 自动分片
SET key1 "value1"
# CRC16("key1") % 16384 = slot 9189
# 自动路由到负责该槽位的节点

# 手动指定槽位
redis-cli -c -h 192.168.1.100
SET key1 "value1"
# 输出: -> Redirected to slot [9189] located at 192.168.1.102:6379
#       OK
```

## 三、集群架构

```
客户端
  │
  ▼
┌─────────────────────────────────────┐
│           Redis Cluster             │
│                                     │
│  Master A ─── Master B ─── Master C │
│  (0-5460)     (5461-10922)  (10923-16383) │
│     │            │              │    │
│  Slave A1    Slave B1       Slave C1 │
└─────────────────────────────────────┘
```

## 四、集群特性

```bash
# 1. 数据分片
# 16384个槽位分布到多个节点
# 支持水平扩展

# 2. 高可用
# 主节点故障时自动故障转移
# 从节点提升为主节点

# 3. 无中心化
# 所有节点平等
# 客户端可连接任意节点

# 4. 在线扩容
# 支持在线添加/删除节点
# 槽位自动迁移
```

## 五、限制

```bash
# 1. 多Key操作限制
# 多Key操作必须在同一个槽位
# 使用{}指定哈希标签
SET {user:1001}:name "张三"
SET {user:1001}:age 25
MGET {user:1001}:name {user:1001}:age

# 2. 不支持多数据库
# 只能使用数据库0

# 3. 不支持Lua脚本跨槽位
# Lua脚本中的key必须在同一个槽位

# 4. 部分命令不支持
# KEYS、FLUSHALL等命令受限
```

## 六、注意事项与常见陷阱

1. **最少3主3从**：生产环境需要至少6个节点
2. **哈希标签**：使用{tag}确保相关key在同一槽位
3. **客户端支持**：需要支持Cluster的客户端库
4. **槽位迁移**：数据迁移期间可能有短暂不可用
5. **脑裂问题**：网络分区可能导致数据不一致
6. **大Key影响**：大Key迁移会阻塞整个槽位
