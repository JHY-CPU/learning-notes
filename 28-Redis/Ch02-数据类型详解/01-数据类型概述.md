# 数据类型概述

## 一、概念说明

Redis支持丰富的数据类型，包括五大基本类型（String、List、Hash、Set、ZSet）和特殊类型（HyperLogLog、Geo、Stream）。了解每种数据类型的特点和底层实现，是高效使用Redis的基础。

## 二、五大基本类型

### 数据类型一览

| 类型 | 简介 | 底层编码 | 最大容量 | 典型场景 |
|------|------|----------|----------|----------|
| String | 字符串或整数 | SDS/int | 512MB | 缓存、计数器 |
| List | 有序双端列表 | QuickList | 2^32-1 | 消息队列、时间线 |
| Hash | 字段-值映射 | HashTable/Ziplist | 2^32-1 | 对象存储 |
| Set | 无序不重复集合 | HashTable/IntSet | 2^32-1 | 标签、去重 |
| ZSet | 有序不重复集合 | SkipList/Ziplist | 2^32-1 | 排行榜 |

### 查看数据类型

```bash
# 设置不同类型的数据
SET mystring "hello"
LPUSH mylist "a" "b" "c"
HADD myhash field1 "value1"
SADD myset "a" "b" "c"
ZADD myzset 1 "one" 2 "two"

# 查看类型
TYPE mystring    # 输出: string
TYPE mylist      # 输出: list
TYPE myhash      # 输出: hash
TYPE myset       # 输出: set
TYPE myzset      # 输出: zset
```

## 三、特殊数据类型

```bash
# HyperLogLog - 基数统计
PFADD visitors user1 user2 user3
PFCOUNT visitors    # 输出: 3

# Geo - 地理位置
GEOADD cities 116.40 39.90 Beijing
GEODIST cities Beijing Shanghai km

# Stream - 消息流（Redis 5.0+）
XADD mystream * field value
XREAD COUNT 1 STREAMS mystream 0
```

## 四、编码转换机制

```bash
# Redis会根据数据量自动选择编码
# 小数据量使用紧凑编码（ziplist/intset）
# 大数据量使用高效编码（hashtable/skiplist）

# 查看编码方式
OBJECT ENCODING mylist
# 输出: "quicklist" 或 "ziplist"

# 阈值配置（redis.conf）
hash-max-ziplist-entries 128
hash-max-ziplist-value 64
list-max-ziplist-size -2
zset-max-ziplist-entries 128
zset-max-ziplist-value 64
```

## 五、注意事项与常见陷阱

1. **编码自动转换**：数据量超过阈值时自动转换编码
2. **内存与性能权衡**：ziplist省内存但操作慢，hashtable占内存但操作快
3. **避免大Key**：单个Key存储过多数据会影响性能
4. **选择合适类型**：根据业务场景选择最适合的数据类型
5. **二进制安全**：所有类型都支持二进制数据
