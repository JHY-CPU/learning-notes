# 算法公平性与可解释性 - 计算机伦理与法律笔记


## 算法公平性与可解释性


## 一、机器学习中的偏见


### 1. 偏见来源


| 来源 | 说明 | 案例 |
| --- | --- | --- |
| 历史偏见 | 训练数据反映了社会历史中的不平等 | 招聘数据中男性比例远高于女性，模型学到了性别偏见 |
| 表示偏见 | 某些群体在数据中代表性不足 | 人脸识别训练集缺乏深肤色样本，识别准确率下降 |
| 测量偏见 | 特征本身带有系统性偏差 | 用逮捕率代替犯罪率，不同社区的执法力度差异 |
| 聚合偏见 | 用一个模型服务所有群体 | 医疗模型在不同种族群体上表现差异大 |
| 部署偏见 | 模型在与训练不同的环境中使用 | 城市 A 训练的犯罪预测模型用于城市 B |


### 2. 经典案例


- **COMPAS 累犯预测：**
   美国法院使用的累犯风险评估系统，被发现对黑人被告的假阳性率显著高于白人（ProPublica 调查，2016）
- **Amazon 招聘工具：**
   对包含"女性"关键词的简历自动降分，因为训练数据中历史录用者多为男性
- **信用卡审批算法：**
   Apple Card 被指控给予女性更低的信用额度


## 二、公平性度量


### 1. 主要公平性定义


```
群体公平性（Group Fairness）：
┌─────────────────────────────────────────────────┐
│  人口均等 (Demographic Parity)                    │
│  P(Ŷ=1|A=0) = P(Ŷ=1|A=1)                        │
│  不同群体获得正预测的比例相同                        │
├─────────────────────────────────────────────────┤
│  机会均等 (Equalized Odds)                        │
│  P(Ŷ=1|A=0,Y=y) = P(Ŷ=1|A=1,Y=y) for y∈{0,1}   │
│  不同群体的 TPR 和 FPR 分别相同                    │
├─────────────────────────────────────────────────┤
│  预测均等 (Predictive Parity)                     │
│  P(Y=1|Ŷ=1,A=0) = P(Y=1|Ŷ=1,A=1)               │
│  不同群体的精确率相同                              │
└─────────────────────────────────────────────────┘
```


### 2. 公平性的不可能三角


> **Warning:** **Impossibility Theorem（Chouldechova, 2017）：**
> 当基率（base rate）在不同群体间不同时，不可能同时满足以下三个条件：1）预测均等 2）机会均等 3）整体校准。必须做出权衡取舍。


### 3. 个体公平性


```
// 个体公平性定义（Dwork et al., 2012）
// 相似的个体应获得相似的预测结果
d(f(x), f(y)) ≤ L · d(x, y)

// 问题：如何定义"相似"？
// 不同度量标准导致不同的公平性判断
```


## 三、偏见缓解方法


### 1. 预处理（Pre-processing）


- **重采样：**
   对少数群体过采样或多数群体欠采样
- **重加权：**
   给少数群体样本更高权重
- **数据增强：**
   生成合成数据平衡群体表示
- **去偏嵌入：**
   在词嵌入中移除性别/种族方向


### 2. 处理中（In-processing）


- **公平约束优化：**
   在损失函数中加入公平性约束项
- **对抗去偏：**
   使用对抗网络去除敏感属性信息
- **正则化：**
   惩罚模型对敏感属性的依赖


### 3. 后处理（Post-processing）


- **阈值调整：**
   对不同群体使用不同的决策阈值
- **校准：**
   确保不同群体的预测概率具有相同含义


## 四、可解释性（Explainable AI / XAI）


### 1. 为什么需要可解释性


- **信任：**
   用户需要理解 AI 的决策依据
- **合规：**
   GDPR 第 22 条——数据主体有权获得关于自动化决策的解释
- **调试：**
   发现模型的错误推理和偏见
- **公平性审计：**
   检查模型是否使用了不当特征


### 2. 模型可解释性分类


```
┌──────────────────┬──────────────────┐
│  本质可解释模型    │  事后解释方法      │
│  (Intrinsically  │  (Post-hoc       │
│   Interpretable) │   Explanation)   │
├──────────────────┼──────────────────┤
│  线性回归         │  SHAP            │
│  决策树           │  LIME            │
│  逻辑回归         │  注意力可视化      │
│  规则列表         │  梯度分析         │
│  GAM             │  反事实解释       │
└──────────────────┴──────────────────┘
```


### 3. SHAP（SHapley Additive exPlanations）


```
// 基于博弈论 Shapley 值
// 每个特征的贡献 = 该特征在所有特征子集中的边际贡献的加权平均

φ_i = Σ [|S|!(M-|S|-1)!/M!] × [f(S∪{i}) - f(S)]

// 直观理解：
// 将每个特征视为"玩家"，预测值视为"总收益"
// SHAP 值公平分配每个特征对预测的贡献

// Python 使用
import shap
explainer = shap.TreeExplainer(model)
shap_values = explainer.shap_values(X_test)
shap.summary_plot(shap_values, X_test)  # 全局特征重要性
shap.force_plot(explainer.expected_value, shap_values[0], X_test.iloc[0])  # 单条预测解释
```


- **优点：**
   理论上唯一满足公平分配公理（效率性、对称性、虚拟性、可加性）的方法
- **缺点：**
   计算成本高（指数级子集数量），需要近似算法


### 4. LIME（Local Interpretable Model-agnostic Explanations）


```
// 核心思想：在局部用简单模型近似复杂模型
// 对单条预测，在其邻域内生成扰动样本
// 用可解释模型（如线性回归）拟合这些样本

解释 = argmin_g L(f, g, π_x) + Ω(g)
// f: 原始模型, g: 简单模型, π_x: 邻域权重, Ω: 复杂度惩罚

// Python 使用
import lime
explainer = lime.lime_tabular.LimeTabularExplainer(X_train)
exp = explainer.explain_instance(X_test[0], model.predict_proba)
exp.show_in_notebook()
```


- **优点：**
   模型无关（任何模型都能用），局部解释直观
- **缺点：**
   不稳定（随机采样导致结果波动），只解释局部


## 五、算法审计


### 1. 审计流程


```
1. 定义审计目标 → 确定保护群体和公平性指标
2. 数据审计   → 检查训练数据的代表性和偏差
3. 模型审计   → 测试不同群体的性能差异
4. 结果审计   → 分析决策结果的分布差异
5. 报告与建议 → 提出改进方案和监控机制
```


### 2. 法律框架


| 法规 | 地区 | 相关条款 |
| --- | --- | --- |
| GDPR | 欧盟 | 第22条：自动化决策的解释权 |
| AI Act | 欧盟 | 高风险AI系统的透明度和可审计性要求 |
| EEOC 指南 | 美国 | 雇佣中AI工具的歧视风险 |
| 个人信息保护法 | 中国 | 自动化决策的透明度和拒绝权 |


> **Note:** **面试要点：**
> 算法偏见的五大来源（历史/表示/测量/聚合/部署）；SHAP 基于博弈论 Shapley 值保证公平分配；LIME 用局部线性模型近似；公平性不可能三角（预测均等、机会均等、校准无法同时满足）；GDPR 第 22 条的"解释权"。

总结：ML偏见来源包括历史偏见、表示偏见等五类；公平性度量有群体公平性（人口均等、机会均等、预测均等）和个体公平性，但存在不可能三角；SHAP基于Shapley值是最理论完备的可解释性方法，LIME是模型无关的局部解释；算法审计流程五步走；GDPR和AI Act是主要法律框架
