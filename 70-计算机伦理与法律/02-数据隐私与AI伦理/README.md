# 02-数据隐私与AI伦理

## 1. 数据隐私法规

### 1.1 GDPR（欧盟通用数据保护条例）

GDPR于2018年5月25日生效，是全球最具影响力的数据保护法规。

#### 数据主体权利

- **访问权（Right of Access）**：数据主体有权了解其个人数据是否被处理，以及获取数据副本
- **删除权（Right to Erasure / Right to be Forgotten）**：在特定条件下要求删除个人数据
- **可携带权（Right to Data Portability）**：以结构化、机器可读格式接收并转移数据
- **更正权**：要求更正不准确的个人数据
- **限制处理权**：限制对个人数据的处理
- **反对权**：反对基于合法利益的数据处理
- **拒绝自动化决策权**：不受仅基于自动化处理的决策约束

#### 数据处理原则

1. **合法性、公平性、透明性**：处理须有合法基础
2. **目的限制**：仅为明确、合法的目的收集数据
3. **数据最小化**：仅收集必要数据
4. **准确性**：保持数据准确和最新
5. **存储限制**：不超过必要期限保存
6. **完整性与保密性**：采取适当安全措施
7. **问责制**：数据控制者需能证明合规

合法处理基础包括：同意、合同履行、法律义务、重大利益、公共利益、合法利益。

#### DPIA（数据保护影响评估）

- 当数据处理可能对个人权利产生高风险时必须进行
- 评估内容：处理的必要性、比例性、风险、缓解措施
- 适用场景：大规模数据处理、自动化画像、公共区域监控、敏感数据处理

### 1.2 中国《个人信息保护法》

- 2021年11月1日生效，共8章74条
- **核心原则**：知情同意、目的明确、最小必要、公开透明
- **个人信息**：以电子或其他方式记录的与已识别或可识别自然人有关的各种信息
- **敏感个人信息**：生物识别、宗教信仰、特定身份、金融账户、行踪轨迹、14岁以下未成年人信息
- **跨境传输**：安全评估、标准合同、认证、国际条约
- **个人信息处理者义务**：指定负责人、影响评估、安全保护
- **处罚**：违法所得1-10倍罚款，最高5000万元或上一年度营业额5%

### 1.3 中国《数据安全法》

- 2021年9月1日生效
- 建立**数据分类分级保护**制度
- **核心数据**：关系国家安全、国民经济命脉、重要民生、重大公共利益的数据
- 数据安全审查制度和出口管制
- 重要数据出境需安全评估

### 1.4 CCPA（加州消费者隐私法）

- 2020年1月1日生效（CPRA修正案2023年生效）
- 适用：年收入超2500万美元、处理10万以上消费者数据、数据销售收入占50%以上
- 消费者权利：知情权、删除权、拒绝出售权、不受歧视权
- 与GDPR的主要差异：更侧重消费者对数据**出售**的控制

## 2. 数据治理框架

### 2.1 数据分类分级

| 级别 | 名称 | 说明 | 示例 |
|------|------|------|------|
| 第一级 | 公开数据 | 可自由公开的数据 | 公司官网信息 |
| 第二级 | 内部数据 | 仅限内部使用 | 内部通知、流程文档 |
| 第三级 | 敏感数据 | 泄露可能造成损害 | 客户信息、财务数据 |
| 第四级 | 机密数据 | 泄露会造成严重损害 | 核心算法、商业机密 |

### 2.2 数据生命周期管理

```
采集 → 存储 → 使用 → 共享 → 归档 → 销毁
```

- **采集**：最小化原则、知情同意
- **存储**：加密存储、访问控制、备份策略
- **使用**：目的限制、审计日志
- **共享**：脱敏处理、传输加密、合同约束
- **归档**：长期保存策略、格式兼容
- **销毁**：彻底删除、可验证销毁

### 2.3 跨境数据传输

- **GDPR机制**：充分性认定、标准合同条款（SCC）、约束性企业规则（BCR）
- **中国机制**：安全评估（CAC）、标准合同备案、个人信息保护认证
- 关键考量：接收国数据保护水平、传输量、数据类型

## 3. AI伦理

### 3.1 算法公平性与偏见

#### 偏见来源

- **数据偏见**：训练数据不具代表性、历史数据反映系统性不平等、标签偏见
- **算法偏见**：模型设计选择、特征工程偏差、正反馈循环
- **评估偏见**：评估指标选择不当、测试群体不具代表性
- **部署偏见**：使用场景与训练场景不匹配

#### 公平性度量

- **Demographic Parity（人口统计均等）**：不同群体获得有利结果的比例相同
  - P(Y=1|G=A) = P(Y=1|G=B)
- **Equalized Odds（均等机会）**：不同群体的真正率和假正率相同
  - P(Ŷ=1|Y=y,G=A) = P(Ŷ=1|Y=y,G=B)
- **Calibration（校准）**：预测概率在不同群体中含义一致
  - P(Y=1|Ŷ=p,G=A) = P(Y=1|Ŷ=p,G=B)
- 注意：不同公平性度量之间通常存在**不可能定理**，无法同时满足

### 3.2 AI透明性与可解释性

#### LIME（Local Interpretable Model-agnostic Explanations）

- 对单个预测生成局部可解释的近似模型
- 原理：在预测点附近扰动生成样本，用简单模型拟合局部行为
- 适用于任何黑盒模型的解释

#### SHAP（SHapley Additive exPlanations）

- 基于博弈论Shapley值的特征归因方法
- 满足一致性、局部准确性、缺失性等公理
- 提供全局和局部解释
- 变体：KernelSHAP（模型无关）、TreeSHAP（树模型专用，高效）

#### 注意力可视化

- 将Transformer等模型的注意力权重可视化
- 展示模型在预测时"关注"了输入的哪些部分
- 注意：注意力不等同于解释（相关研究存在争议）

### 3.3 深度伪造（Deepfake）与对策

**技术原理**：基于GAN或自编码器的人脸替换、表情迁移、语音合成

**风险**：
- 政治操纵和虚假信息传播
- 名誉损害和身份欺诈
- 虚假证据制造

**对策**：
- **检测技术**：帧间不一致性分析、频域检测、生理信号检测
- **数字水印**：内容来源标识（C2PA标准）
- **区块链验证**：内容溯源和真实性验证
- **法律法规**：深度伪造标识义务、传播虚假信息的法律责任

### 3.4 AI安全与对齐问题

- **对齐（Alignment）**：确保AI系统的目标与人类价值观一致
- **外对齐**：目标与设计者意图一致
- **内对齐**：模型内部表征与目标一致
- **涌现能力**：大模型的不可预测行为
- **Constitutional AI**：通过规则约束和人类反馈训练更安全的模型
- **红队测试（Red Teaming）**：对抗性测试发现安全漏洞

### 3.5 自动化决策的问责

- GDPR第22条：数据主体有权不受仅基于自动化处理的决策约束
- 责任链条：数据提供者 → 模型开发者 → 部署者 → 使用者
- 透明度要求：解释决策逻辑、重要性和预期后果
- 人工审查机制：高风险决策需保留人工介入环节

## 4. AI治理

### 4.1 AI监管趋势（EU AI Act）

EU AI Act是全球首部综合性AI法规（2024年通过）：

| 风险等级 | 说明 | 监管要求 |
|----------|------|----------|
| 不可接受风险 | 社会评分、实时远程生物识别（公共场所） | 禁止 |
| 高风险 | 生物识别、关键基础设施、教育、就业、执法 | 严格合规义务 |
| 有限风险 | 聊天机器人、深度伪造 | 透明度义务 |
| 最低风险 | 垃圾邮件过滤、游戏AI | 基本无特别要求 |

### 4.2 AI风险分级

- **技术风险**：模型故障、对抗攻击、数据投毒
- **社会风险**：就业替代、不平等加剧、隐私侵犯
- **存在风险**：失控AI、武器化AI
- 风险评估矩阵：可能性 × 影响程度

### 4.3 负责任AI框架

1. **公平性**：消除偏见，确保公平对待
2. **可解释性**：决策过程透明可理解
3. **隐私**：保护个人数据
4. **安全性**：防止恶意攻击和意外行为
5. **问责制**：明确责任归属
6. **人类控制**：保留人类决策权

代表性框架：Microsoft RAI、Google AI Principles、IBM AI Fairness 360

## 5. 计算机伦理原则

### 5.1 ACM/IEEE道德准则

ACM/IEEE计算机协会道德与职业行为准则核心原则：

1. 以公众利益为首要考量
2. 避免伤害他人
3. 诚实可信
4. 公平对待和不歧视
5. 尊重财产权（包括知识产权）
6. 尊重知识产权
7. 给予适当的信用认可
8. 尊重隐私
9. 保密

### 5.2 隐私设计（Privacy by Design）

Ann Cavoukian提出的七大基本原则：

1. **主动预防而非事后补救**：在设计阶段嵌入隐私保护
2. **默认隐私保护**：隐私作为默认设置
3. **隐私嵌入设计**：隐私是架构的组成部分
4. **全功能性**：安全与功能并行不矛盾
5. **全生命周期保护**：从数据采集到销毁全程保护
6. **可见性与透明性**：保持开放和可验证
7. **用户中心**：以用户隐私利益为核心

实践要点：
- 数据最小化收集
- 端到端加密
- 去标识化与匿名化
- 用户知情同意机制
- 隐私影响评估（PIA）
