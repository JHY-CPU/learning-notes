AWS 核心服务

// ========== EC2 (虚拟机) ========== // 创建实例: // aws ec2 run-instances \ // --image-id ami-0c55b159cbfafe1f0 \ // --instance-type t3.medium \ // --key-name my-key \ // --security-group-ids sg-xxx \ // --user-data file://setup.sh // 安全组: // 入站: 80 (HTTP), 443 (HTTPS), 22 (SSH) // 出站: 全部放行 // SSH 连接: // chmod 600 my-key.pem // ssh -i my-key.pem ec2-user@ // ========== S3 (对象存储) ========== // 存储桶 → 对象 (文件) // 适用: 图片/视频/备份/静态网站 // # 上传文件 // aws s3 cp file.png s3://my-bucket/uploads/ // // # 公开访问 // aws s3api put-public-access-block --bucket my-bucket --public // // # 预签名 URL (临时访问) // aws s3 presign s3://my-bucket/private.pdf --expires-in 3600 // S3 存储类: // Standard: 频繁访问 // Infrequent Access: 低频, 便宜 // Glacier: 归档 (取回需几分钟) // ========== RDS (托管数据库) ========== // 支持: PostgreSQL, MySQL, MariaDB, SQL Server // 特性: // 自动备份 (快照) // 多可用区部署 (高可用) // 只读副本 (读扩展) // 自动扩缩容 // # 创建 RDS // aws rds create-db-instance \ // --engine postgres \ // --db-instance-class db.t3.medium \ // --master-username app \ // --master-user-password secret \ // --allocated-storage 100 \ // --multi-az // ========== ELB (负载均衡) ========== // ALB (HTTP/HTTPS): 路径/主机路由 // NLB (TCP/UDP): 高性能 // 特性: // 健康检查 // SSL 终止 // 跨可用区 // 自动扩展集成

Serverless 与 CI/CD 部署

// ========== AWS Lambda ========== // 函数即服务, 按调用计费 // exports.handler = async (event) => { // const { userId } = JSON.parse(event.body); // const user = await db.findUser(userId); // // return { // statusCode: 200, // headers: { 'Content-Type': 'application/json' }, // body: JSON.stringify(user), // }; // }; // Lambda 触发: // API Gateway → Lambda (REST API) // S3 事件 → Lambda (图片处理) // SQS → Lambda (消息处理) // CloudWatch → Lambda (定时任务) // 冷启动: // 首次调用/闲置后启动慢 (~1s) // 优化: 减少包体积, 预留并发 // ========== Serverless 框架 ========== // serverless.yml: // service: my-api // // provider: // name: aws // runtime: nodejs20.x // region: ap-northeast-1 // // functions: // hello: // handler: handler.hello // events: // - http: // path: /hello // method: get // 替代平台: // Vercel: 前端 + API, Next.js 友好 // Netlify: 前端 + 函数, Jamstack // Railway: 简单部署, 无需配置 // ========== CI/CD 部署流水线 ========== // GitHub Actions → 服务器/Docker // name: Deploy // on: // push: // branches: [main] // // jobs: // deploy: // runs-on: ubuntu-latest // steps: // - uses: actions/checkout@v4 // // - name: Build Docker image // run: docker build -t myapp . // // - name: Push to registry // run: | // docker tag myapp registry.example.com/myapp:${{ github.sha }} // docker push registry.example.com/myapp:${{ github.sha }} // // - name: Deploy to server // uses: appleboy/ssh-action@v1 // with: // host: ${{ secrets.HOST }} // username: ${{ secrets.USER }} // key: ${{ secrets.SSH_KEY }} // script: | // cd /opt/myapp // docker-compose pull // docker-compose up -d // docker system prune -f // 多环境晋升: // dev (自动部署) → staging (手动) → prod (手动+审批) // 环境变量: .env.dev / .env.staging / .env.prod // 数据库: 迁移脚本版本管理
💡 云部署要点: IaaS 自管OS, PaaS 只管代码, FaaS 只管函数; AWS EC2 + S3 + RDS 核心服务; Nginx 反向代理 + SSL; Let's Encrypt 免费证书自动续期; PM2/systemd 进程管理; Lambda 函数按调用付费; CI/CD 构建→推送→部署流水线; 多环境 dev/staging/prod 晋升。
