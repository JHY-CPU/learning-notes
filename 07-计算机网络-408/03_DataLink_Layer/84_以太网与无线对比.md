# 84_以太网与无线对比

## 核心概念

- **以太网（802.3）**：有线局域网，CSMA/CD
- **WiFi（802.11）**：无线局域网，CSMA/CA
- **核心差异**：有线可以边发边收检测冲突，无线不能

### 全面对比

| 特性 | 以太网 | WiFi |
|------|--------|------|
| 标准 | IEEE 802.3 | IEEE 802.11 |
| 介质 | 双绞线/光纤 | 无线电波 |
| MAC协议 | CSMA/CD | CSMA/CA |
| 冲突检测 | 能 | 不能 |
| 确认机制 | 无ACK | 需要ACK |
| 速率 | 10M-100G | 2M-9.6G |
| 帧格式 | 单一格式 | 复杂（4地址） |
| 安全 | 物理安全 | 需加密(WPA2) |
| 覆盖范围 | 受线缆限制 | 灵活（无线） |

### 为什么无线用CA不用CD？

1. **硬件限制**：无线网卡同时收发时，自身发送功率远大于接收功率
2. **信号衰减**：远处站点的信号可能太弱检测不到冲突
3. **隐藏终端**：互相听不到的站点可能同时发送

### 帧格式差异

以太网帧：6+6+2+数据+4 = 18字节开销 + 数据

802.11帧：2+2+6+6+6+2+6+数据+4 = 最多34字节开销 + 数据

WiFi帧开销更大（因为支持AP转发，需要更多地址字段）。

### 考试重点

- CSMA/CD vs CSMA/CA的对比
- 为什么无线不能使用CSMA/CD
- 确认机制的差异

## 原理分析

### 全双工以太网 vs WiFi

**全双工以太网**：
- 点对点连接（通过交换机）
- 不需要CSMA/CD
- 不需要ACK
- 全速双向通信

**WiFi**：
- 半双工（同一频率不能同时收发）
- 需要CSMA/CA
- 需要ACK
- 共享信道

### 效率对比

以太网（交换式全双工）：接近100%利用率

WiFi：由于CSMA/CA的退避、DIFS、ACK等开销，实际利用率约50-70%

## 直观理解

- **有线 = 固定电话**：专线，质量稳定，不需要确认
- **无线 = 对讲机**：共享频率，容易干扰，需要确认

## 协议关联

- 以太网和WiFi在现代网络中互补：有线用于骨干和桌面，无线用于移动接入
- AP完成以太网帧和WiFi帧之间的转换
- 408考试中以太网 vs WiFi是经典对比题
