# 06_奇偶校验

## 核心概念

- **奇偶校验（Parity Check）**：最简单的差错检测方法，在数据末尾添加一位校验位
- **奇校验**：数据+校验位中1的个数为**奇数**
- **偶校验**：数据+校验位中1的个数为**偶数**
- **校验位**：额外添加1个比特，使整个码字中1的个数满足奇/偶要求

### 奇偶校验的基本规则

| 类型 | 规则 | 示例（数据1011001） |
|------|------|---------------------|
| 奇校验 | 1的个数为奇数 | 数据有4个1，校验位=1，总5个1 |
| 偶校验 | 1的个数为偶数 | 数据有4个1，校验位=0，总4个1 |

### 二维奇偶校验（方阵校验）

- 将数据排列成矩阵，分别对**行和列**进行奇偶校验
- 优点：不仅能检错，还能**定位错误位置**，实现**单比特纠错**
- 408考试偶尔会考到

### 奇偶校验的检错能力

- 能检测**奇数个**比特错误
- **不能检测偶数个**比特错误
- 不能纠错（一维奇偶校验）

### 检错能力总结

| 错误数 | 能否检测 |
|--------|----------|
| 1位错误 | 能 |
| 2位错误 | 不能 |
| 3位错误 | 能 |
| 4位错误 | 不能 |

规律：奇数个错误能检测，偶数个错误不能检测。

## 原理分析

### 奇校验的数学表达

$$P = d_1 \oplus d_2 \oplus d_3 \oplus \cdots \oplus d_n \oplus 1$$

其中 $\oplus$ 表示异或运算，$d_i$ 为数据位，$P$ 为校验位。

### 偶校验的数学表达

$$P = d_1 \oplus d_2 \oplus d_3 \oplus \cdots \oplus d_n$$

### 二维奇偶校验示例

数据：10110（5位），分成2行（每行若干位）

```
行1: 1 0 1 | P_row1
行2: 1 0 _ | P_row2
     -------
列校验: P_col
```

假如某一位出错，行列校验都会指示出错，从而**定位错误位置**。

### 奇偶校验的检错概率

假设每位出错概率为 $p$，且相互独立：

- 检测到错误的概率 = 奇数个错误发生的概率之和

$$P_{\text{检测}} = \sum_{k=1,3,5,...}^{n} \binom{n}{k} p^k (1-p)^{n-k}$$

当 $p$ 很小时，近似为：

$$P_{\text{检测}} \approx np = \text{误码率} \times \text{数据长度}$$

## 直观理解

- **奇偶校验就像数手指**：数一下有几个"竖起来的手指"（1），看看是奇数还是偶数
- **类比**：老师让学生报数，如果发现总人数的奇偶性与预期不符，就知道有人报错了
- **二维奇偶校验**就像**Excel表格的求和校验**：每行每列都有合计，如果某个数据改了，对应的行列合计都会变化

### 奇偶校验的记忆方法

- 奇校验：希望1的个数是**奇数** → "奇"就对了
- 偶校验：希望1的个数是**偶数** → "偶"就对了
- 校验时：数一数，看是否满足条件

## 协议关联

### 奇偶校验的应用场景

- **串口通信**：常用奇偶校验
- **内存ECC**：使用更复杂的校验
- **现代网络**：数据链路层使用CRC，不用奇偶校验

### 与其他差错检测方法的对比

| 方法 | 冗余位数 | 检错能力 | 纠错能力 | 应用 |
|------|---------|---------|---------|------|
| 奇偶校验 | 1位 | 弱（仅奇数个错误） | 无 | 串口通信 |
| CRC | 多位（取决于生成多项式） | 强 | 无 | 数据链路层 |
| 海明码 | 多位 | 强 | 有（单比特） | 内存 |

### 考试常见题型

1. 给定数据，计算奇偶校验位
2. 给定接收数据，判断是否有错（用奇偶校验）
3. 分析奇偶校验的检错能力
4. 二维奇偶校验的定位和纠错

### 真题示例

**题目**：数据为1001101，采用奇校验，校验位是多少？

解答：
- 数1的个数：1+0+0+1+1+0+1 = 4个1
- 奇校验要求总1个数为奇数，当前是4（偶数）
- 校验位 = 1（使总1个数变为5，奇数）

**注意**：408中奇偶校验一般不会单独出大题，但可能在选择题中出现，需要掌握基本概念和计算。
