# 35_CSMA协议

## 核心概念

- **CSMA（Carrier Sense Multiple Access）**：载波侦听多路访问，在发送前先**监听信道**
- **核心改进**：相比ALOHA增加了"先听后说"（Listen Before Talk）
- **三种CSMA变体**：1-坚持、非坚持、p-坚持

### CSMA的核心思想

```
ALOHA:   想发 → 发送 → 冲突 → 随机重传
CSMA:    想发 → 监听 → 信道忙? → 等待 → 再监听 → 信道空闲 → 发送
```

### 三种CSMA协议对比

| 协议 | 信道忙时 | 信道闲时 | 特点 |
|------|---------|---------|------|
| 1-坚持CSMA | 持续监听直到空闲 | 立即发送 | 传播时延小时冲突多 |
| 非坚持CSMA | 随机等待一段时间再监听 | 立即发送 | 减少冲突，但可能浪费空闲时间 |
| p-坚持CSMA | 持续监听直到空闲 | 以概率p发送 | 折中方案 |

### 1-坚持CSMA

- 信道忙：**持续监听**，直到信道空闲
- 信道空闲：**立即发送**（概率1，所以叫"1-坚持"）
- 缺点：两个节点同时监听到空闲，同时发送，必然冲突

### 非坚持CSMA

- 信道忙：**随机等待**一段时间后重新监听
- 信道空闲：立即发送
- 优点：减少了同时发送的概率
- 缺点：可能在信道空闲时没有发送（白白等待）

### p-坚持CSMA（用于时隙CSMA）

- 信道忙：持续监听
- 信道空闲：以概率 $p$ 发送，以概率 $1-p$ 推迟到下一时隙
- 优点：平衡了冲突和空闲浪费

### CSMA vs ALOHA

| 特性 | ALOHA | CSMA |
|------|-------|------|
| 发送前监听 | 无 | 有 |
| 冲突窗口 | $T$ 或 $2T$ | $T_p$（传播时延） |
| 最大吞吐率 | 18.4%/36.8% | 更高 |
| 实现复杂度 | 简单 | 较复杂 |

### 考试重点

- CSMA的核心改进是**载波侦听**
- 三种CSMA协议的区别（408选择题常考）
- CSMA仍然有冲突（因为传播时延的存在）

## 原理分析

### 为什么CSMA仍然有冲突？

由于**传播时延**，节点A开始发送后，信号需要时间传播到节点B。在此期间，B监听到信道空闲，也开始发送，导致冲突。

$$\text{冲突窗口} = T_p \text{（传播时延）}$$

### CSMA的吞吐率

1-坚持CSMA（假设 $a = T_p / T_f$）：

$$S \approx \frac{1}{1 + a(1+1/a)} = \frac{1}{2+a} \cdot \frac{G}{...}$$

（具体公式复杂，408不需要掌握推导，了解结论即可）

一般结论：CSMA的吞吐率 > ALOHA

### 最大吞吐率对比

| 协议 | 最大吞吐率 |
|------|-----------|
| 纯ALOHA | 18.4% |
| 时隙ALOHA | 36.8% |
| 非坚持CSMA | ≈ 80-90% |
| 1-坚持CSMA | ≈ 50-53% |
| p-坚持CSMA | ≈ 依赖p的选择 |

## 直观理解

- **ALOHA**就像"闭着眼睛说话"：不管别人在不在说
- **CSMA**就像"先竖起耳朵听"：没人说话才开口
- **仍然冲突**的原因：声音传播有延迟，你听到的"安静"可能是对方刚开始说但声音还没传到

## 协议关联

- CSMA是CSMA/CD和CSMA/CA的**基础**
- CSMA/CD = CSMA + 冲突检测（以太网使用）
- CSMA/CA = CSMA + 冲突避免（WiFi使用）
- CSMA解决了ALOHA的"盲发"问题，但没有解决冲突问题
