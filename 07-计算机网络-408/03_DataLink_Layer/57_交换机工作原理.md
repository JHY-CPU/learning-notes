# 57_交换机工作原理

## 核心概念

- **交换机（Switch）**：多端口网桥，工作在数据链路层，基于MAC地址转发帧
- **核心优势**：每个端口独立冲突域、独享带宽、支持全双工
- **本质**：交换机 = 多端口网桥 + 硬件转发（ASIC）

### 交换机的工作原理

1. **MAC地址学习**：从源MAC学习端口映射
2. **帧转发/过滤**：查MAC地址表决定转发端口
3. **环路避免**：运行STP协议

### 交换方式

| 方式 | 说明 | 时延 | 错误检测 |
|------|------|------|---------|
| 存储转发 | 收完整帧后查表转发 | 较大 | 有（检查FCS） |
| 直通转发 | 收到目的MAC就开始转发 | 最小 | 无 |
| 无碎片转发 | 收到前64字节后转发 | 中等 | 检查前64字节 |

### 交换机的MAC地址表

- **学习**：从入帧的源MAC地址学习
- **老化**：表项有TTL（默认300秒），超时删除
- **动态学习**：自动学习，也可手动配置静态表项

### 交换机的端口

| 端口类型 | 说明 |
|---------|------|
| Access端口 | 连接终端设备，属于一个VLAN |
| Trunk端口 | 连接其他交换机，承载多个VLAN |
| Hybrid端口 | 灵活端口，可同时承载多个VLAN |

### 交换机 vs 集线器

| 特性 | 集线器 | 交换机 |
|------|--------|--------|
| 工作层次 | 物理层 | 数据链路层 |
| 冲突域 | 所有端口共享1个 | 每端口1个 |
| 带宽 | 所有端口共享 | 每端口独立 |
| 转发方式 | 比特复制 | 帧转发 |
| 地址识别 | 不识别 | 基于MAC地址 |

### 考试重点

- 交换机的三种交换方式
- MAC地址表的学习过程
- 交换机每个端口独立冲突域

## 原理分析

### 存储转发详细流程

```
帧到达 → 缓存帧 → 检查FCS → 错误? → 丢弃
                    ↓ 无错误
              查目的MAC → 在表中? → 从对应端口转发
                    ↓ 不在
                   泛洪
```

### 直通转发详细流程

```
帧到达 → 收到目的MAC(6字节) → 立即查表 → 转发（不等帧收完）
```

优点：时延最小
缺点：会转发错误帧（还没校验FCS就开始转发了）

### 交换矩阵

高端交换机使用**交换矩阵（Crossbar）**实现多端口并行转发：

```
端口1 ──┬── 矩阵 ──┬── 端口1
端口2 ──┤          ├── 端口2
端口3 ──┤          ├── 端口3
端口4 ──┴──     ──┴── 端口4
```

支持多个端口对同时通信（非阻塞交换）。

## 直观理解

- **交换机就像电话交换台**：每个通话是独立的，不互相干扰
- **集线器就像大喇叭**：所有人共享一个频道，只能一个人说话

## 协议关联

- 交换机是现代局域网的**核心设备**
- 二层交换（MAC）vs 三层交换（IP）的区别
- VLAN在交换机上实现，分割广播域
- STP在交换机上运行，防止环路
