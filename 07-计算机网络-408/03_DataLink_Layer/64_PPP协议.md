# 64_PPP协议

## 核心概念

- **PPP（Point-to-Point Protocol）**：点对点协议，最常用的数据链路层协议之一
- **应用**：拨号上网、广域网专线、路由器串口连接
- **面向字节**的协议，使用字节填充的透明传输

### PPP的三个组成部分

1. **成帧方法**：将IP数据报封装到串行链路
2. **LCP（链路控制协议）**：建立、配置和测试数据链路连接
3. **NCP（网络控制协议）**：针对不同网络层协议（如IPCP用于IP）

### PPP的工作流程

```
[链路静止] --建立连接--> [链路建立(LCP)] --认证--> [认证(LCP)] 
    --> [网络层协议(NCP)] --> [链路打开] --> [链路终止]
```

### PPP的特点

| 特性 | PPP |
|------|-----|
| 拓扑 | 点对点 |
| 链路类型 | 串行链路 |
| 透明传输 | 字节填充 |
| 差错检测 | CRC |
| 确认机制 | 无ACK（不可靠） |
| 流量控制 | 无（靠上层） |
| 多协议支持 | 有（通过协议字段） |

### PPP不提供的功能

- **不提供可靠传输**：没有序号和确认
- **不提供流量控制**
- **不支持多点链路**：只能点对点
- **不使用CSMA/CD**：点对点没有冲突

### PPP vs 以太网

| 特性 | PPP | 以太网 |
|------|-----|--------|
| 拓扑 | 点对点 | 多点（广播） |
| MAC地址 | 不需要 | 需要 |
| CSMA/CD | 不需要 | 需要（半双工） |
| 链路建立 | 需要（LCP） | 不需要 |

### 考试重点

- PPP的三个组成部分
- PPP不提供可靠传输
- PPP是面向字节的协议
- PPP与以太网的对比

## 原理分析

### PPP帧格式

```
| 标志(1B) | 地址(1B) | 控制(1B) | 协议(2B) | 信息(可变) | FCS(2或4B) | 标志(1B) |
```

- **标志**：0x7E（01111110），帧定界符
- **地址**：0xFF（广播地址，PPP点对点无意义）
- **控制**：0x03（无编号帧）
- **协议**：标识信息字段的内容（0x0021=IP, 0xC021=LCP, 0x8021=NCP）
- **FCS**：CRC校验（2字节或4字节）

### 字节填充

- 标志字节 0x7E 转义为 0x7D 0x5E
- 转义字节 0x7D 转义为 0x7D 0x5D
- ASCII控制字符前加 0x7D

### LCP协商内容

- 最大接收单元（MRU）
- 认证协议（PAP/CHAP）
- 压缩协议
- Magic Number（检测环路）

## 直观理解

- **PPP就像"电话拨号"**：先拨号建立连接（LCP），然后验证身份（认证），最后开始通信（NCP）
- PPP是"轻量级"协议——没有MAC地址、没有冲突检测，因为点对点不需要

## 协议关联

- PPP是广域网最常用的数据链路层协议
- PPPoE（PPP over Ethernet）= PPP + 以太网，用于宽带接入
- PPP不提供可靠传输，靠TCP提供端到端的可靠传输
- PPP与HDLC是408考试中点对点协议的两大代表
