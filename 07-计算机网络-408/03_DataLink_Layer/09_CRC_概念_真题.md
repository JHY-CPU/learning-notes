# 09_CRC_概念_真题

## 核心概念

- **408高频考点**：CRC模2除法计算、FCS的求解、生成多项式的含义
- **常见变体**：给发送帧验证是否有错、给接收帧判断传输是否正确
- **易错点**：模2除法的规则、生成多项式的位数与余数位数的关系

### CRC检错能力总结

| 能检测的情况 | 不能检测的情况 |
|-------------|---------------|
| 所有单比特错误 | 余数恰好为0的错误模式 |
| 所有奇数个比特错误 | 特定的多位错误模式 |
| 所有长度≤r的突发差错 | — |
| 长度>r的突发差错（概率$1-2^{-r}$） | — |

## 原理分析

### 真题1（典型计算题）

**2013年408改编**：设生成多项式 $G(x) = x^3 + x^2 + 1$，数据为 $101001$，求：

(1) 发送帧中的FCS
(2) 发送帧

**解答**：

(1) $G(x) = x^3 + x^2 + 1$，对应二进制 `1101`，4位，需添加3个0

$M' = 101001000$

模2除法：

```
          110100
    _____________
1101|101001000
     1101
     -----
      1110
      1101
      ----
       1111
       1101
       ----
        0100
        0000
        ----
         1000
         1101
         ----
          101  ← 余数 R = 101
```

FCS = **101**（3位）

(2) 发送帧 = 101001 + 101 = **101001101**

### 真题2（接收验证题）

**题目**：接收端收到帧 `11010110110011`，生成多项式 $G(x) = x^4 + x + 1$，判断帧是否出错。

**解答**：

$G(x) = 10011$

用收到的帧除以 $G(x)$：

```
              1100001100
        ___________________
10011 | 11010110110011
        10011
        -----
         10011
         10011
         -----
          00001
          00000
          -----
           00011
           00000
           -----
            00110
            00000
            -----
             01101
             00000
             -----
              11011
              10011
              -----
               10001
               10011
               -----
                010 ← 余数 R = 010
```

余数 **≠ 0**，帧**出错了**！

### 真题3（多项式理解题）

**题目**：生成多项式 $G(x) = x^4 + x^3 + 1$，其对应的二进制串和除数分别是多少？

**解答**：
- $G(x) = x^4 + x^3 + 0 \cdot x^2 + 0 \cdot x + 1$
- 对应二进制：**11001**
- 除数 = 11001
- 余数位数 = 4位（最高次幂）
- 添加0的个数 = 4

### 真题4（对比理解题）

**题目**：以下关于CRC校验的描述，正确的是（ ）

A. CRC能纠正传输中的错误
B. CRC-32能检测所有长度不超过32的突发差错
C. CRC能检测所有偶数个比特的错误
D. CRC的检错率与生成多项式的阶数无关

**答案**：B

- A错：CRC只能检错，不能纠错
- B对：CRC-r能检测所有长度≤r的突发差错
- C错：CRC不能保证检测所有偶数个错误
- D错：阶数越高，检错能力越强

### 常见陷阱

1. **添加0个数**：看$G(x)$的最高次幂，不是位数。位数 = 最高次幂 + 1
2. **模2除法**：**没有减法**，只有异或（XOR）
3. **余数前补0**：如果余数不足r位，前面补0
4. **验证时**：用整个帧（含FCS）除以G(x)，余数应为0

## 直观理解

- CRC就像**"数字指纹"**：每段数据都有独特的余数，任何篡改都会改变这个指纹
- 为什么CRC比校验和强？因为它有**数学结构**（多项式环），能检测更多类型的错误
- **生成多项式的阶数**就像指纹的"精细度"：阶数越高，能区分的错误模式越多

## 协议关联

### CRC在408考试中的定位

- CRC是数据链路层**最核心的检错方法**
- 常与**海明码**对比出题（检错 vs 纠错）
- 常与**帧结构**结合出题（FCS在帧中的位置）

### 答题模板

1. 先确定生成多项式的二进制形式和位数
2. 计算需要添加的0个数 = 最高次幂
3. 在数据后添加0，做模2除法
4. 余数即为FCS
5. 验证时：整个帧除以G(x)，余数应为0

### 分值预估

- CRC计算题通常占 **4-8分**（选择题或综合题的一部分）
- 必须熟练掌握模2除法，考试时间紧张，不能现场学
