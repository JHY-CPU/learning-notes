# 41_CSMA-CD与CSMA-CA对比

## 核心概念

- **CSMA/CD**：冲突检测，用于有线以太网（802.3）
- **CSMA/CA**：冲突避免，用于无线局域网（802.11）
- **核心区别**：CD能边发边收检测冲突，CA不能（无线特性决定）

### 全面对比表

| 特性 | CSMA/CD | CSMA/CA |
|------|---------|---------|
| 全称 | 带冲突检测的CSMA | 带冲突避免的CSMA |
| 应用 | 有线以太网 | 无线WiFi |
| 载波侦听 | 有 | 有 |
| 冲突检测 | 有（边发边收） | 无（无法同时收发） |
| 冲突处理 | 检测到后停止+退避 | 退避避免 |
| 确认机制 | 不需要ACK | 需要ACK |
| RTS/CTS | 无 | 可选 |
| 帧间间隔 | IPG（96 bit） | DIFS/SIFS |
| 最小帧长 | 64字节（检测冲突） | 无此要求 |
| 退避算法 | 二进制指数退避 | 二进制指数退避 |

### 为什么无线用CA不用CD？

1. **硬件限制**：无线网卡发送时功率远大于接收功率，无法同时收发
2. **信号衰减**：无线信号衰减大，远处节点的冲突信号可能太弱检测不到
3. **隐藏终端**：互相听不到的节点可能同时发送，CD无法检测

### 共同点

- 都基于CSMA（载波侦听）
- 都使用二进制指数退避
- 都需要解决多节点共享信道的问题

## 原理分析

### 协议流程对比

**CSMA/CD**：
```
监听 → 空闲 → 发送 → 检测冲突 → 有冲突? → 停发+退避重传
                                    ↓ 无冲突
                                 发送完成
```

**CSMA/CA**：
```
监听 → 空闲 → 等DIFS → 发送 → 等ACK → 收到ACK? → 完成
              ↓ 忙                   ↓ 未收到
         退避等待                    退避重传
```

### 效率对比

CSMA/CD效率更高（不需要等待DIFS、不需要ACK、能检测冲突），但只适用于有线。

CSMA/CA效率较低（有DIFS开销、需要ACK、退避更保守），但能工作在无线环境。

## 直观理解

- **CSMA/CD = 面对面交谈**：你能同时说和听，发现对方也在说就立刻停
- **CSMA/CA = 对讲机通话**：只能收或发，必须按规则切换，每次都要等对方确认

## 协议关联

- 两者是MAC层协议的两大代表
- 408考试中几乎必考两者的对比
- 全双工以太网不需要CSMA/CD，802.11ax仍然使用CSMA/CA的变体
