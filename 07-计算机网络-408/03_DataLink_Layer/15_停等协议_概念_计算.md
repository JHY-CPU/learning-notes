# 15_停等协议_概念_计算

## 核心概念

- **停等协议的信道利用率**：$U = \frac{1}{1+2a}$，其中 $a = T_{prop}/T_{frame}$
- **关键参数**：$a$ 值越大（传播时延越长或帧越短），利用率越低
- **序号位数**：停等协议只需1位序号（0和1交替）

### 信道利用率的决定因素

| 因素 | 变化 | 对利用率的影响 |
|------|------|---------------|
| 帧长增大 | $T_{frame}$ 增大 | 利用率提高 |
| 传播时延增大 | $T_{prop}$ 增大 | 利用率降低 |
| 带宽增大 | $T_{frame}$ 减小 | 利用率降低 |

### 吞吐量计算

$$\text{吞吐量} = \text{带宽} \times U = \frac{\text{帧长}}{T_{frame} + T_{ACK} + 2T_{prop}}$$

## 原理分析

### 计算示例1（基础）

**题目**：数据传输速率 $R = 1$ Mbps，帧长 $L = 1000$ bit，单向传播时延 $T_p = 20$ ms，ACK长度忽略不计，求停等协议的信道利用率。

**解答**：

$$T_f = \frac{L}{R} = \frac{1000}{10^6} = 1 \text{ ms}$$

$$a = \frac{T_p}{T_f} = \frac{20}{1} = 20$$

$$U = \frac{1}{1 + 2a} = \frac{1}{1 + 40} = \frac{1}{41} \approx 2.44\%$$

**结论**：当传播时延远大于发送时延时，信道利用率极低。

### 计算示例2（反推帧长）

**题目**：要使停等协议的信道利用率不低于50%，$a=0.01$，求帧长至少多长？

**解答**：

$$U = \frac{1}{1+2a} \geq 0.5$$

$$1+2a \leq 2$$

$$a \leq 0.5$$

当 $a = 0.01$ 时，$U = \frac{1}{1.02} \approx 98\% > 50\%$，满足要求。

### 计算示例3（综合）

**题目**：链路长400km，传播速度 $2 \times 10^5$ km/s，数据率100kbps，帧长200bit，ACK长40bit，求：

(1) 单向传播时延
(2) 发送一帧的时间
(3) 信道利用率

**解答**：

(1) $T_p = \frac{400}{2 \times 10^5} = 2 \times 10^{-3}$ s = **2 ms**

(2) $T_f = \frac{200}{100000} = 2 \times 10^{-3}$ s = **2 ms**

(3) $T_{ACK} = \frac{40}{100000} = 0.4$ ms

$$U = \frac{T_f}{T_f + T_{ACK} + 2T_p} = \frac{2}{2 + 0.4 + 4} = \frac{2}{6.4} = 31.25\%$$

### 计算示例4（吞吐量）

**题目**：在上题基础上，求有效数据传输速率（吞吐量）。

**解答**：

$$\text{吞吐量} = R \times U = 100 \times 0.3125 = 31.25 \text{ kbps}$$

或直接计算：

$$\text{吞吐量} = \frac{L}{T_f + T_{ACK} + 2T_p} = \frac{200}{6.4 \times 10^{-3}} = 31250 \text{ bps} = 31.25 \text{ kbps}$$

## 直观理解

- **$a$ 值的含义**：帧在"路上"花的时间 vs "发送"花的时间。$a$ 越大，说明大部分时间都在等
- **形象比喻**：就像发短信——你用0.1秒打完一条短信（$T_f$），等了10秒才收到回复（$2T_p$）。大部分时间都在等
- **解决方案**：增大帧长（让$T_f$更大）或使用滑动窗口协议（连续发送多帧）

## 协议关联

- 停等协议的信道利用率公式是所有滑动窗口协议计算的**基础**
- GBN和SR的利用率公式是在停等协议基础上乘以窗口大小
- 408考试中，信道利用率的计算**必考**，必须熟练掌握

### 考试技巧

1. 先把所有时延算出来（$T_f$、$T_p$、$T_{ACK}$）
2. 直接套公式 $U = \frac{T_f}{T_f + T_{ACK} + 2T_p}$
3. 注意单位统一（ms、s、bit、bps）
4. 如果问吞吐量，用 $U \times R$ 或直接用帧长除以总时间
