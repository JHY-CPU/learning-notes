# 29_信道利用率_概念_计算

## 核心概念

- **计算步骤**：确定 $T_f$ → 确定 $T_p$ → 计算 $a$ → 套公式
- **注意约束**：$U$ 不可能超过100%
- **帧长包括**：数据载荷 + 帧头帧尾（但不影响大部分题目）

### 公式速查

| 协议 | 利用率公式 | 条件 |
|------|-----------|------|
| 停等 | $U = \frac{1}{1+2a}$ | 总是 |
| GBN/SR | $U = \frac{W_s}{1+2a}$ | $W_s \leq 1+2a$ |
| GBN/SR | $U = 1$ | $W_s > 1+2a$ |

## 原理分析

### 例题1

**题目**：数据率2Mbps，帧长2000bit，单向传播时延25ms，ACK长度忽略。

(1) 停等利用率
(2) GBN $W_s=7$ 的利用率
(3) 使利用率100%的最小GBN窗口

**解答**：

$T_f = 2000/(2 \times 10^6) = 1$ ms

$T_p = 25$ ms，$2T_p = 50$ ms

$a = 25/1 = 25$

(1) 停等：$U = \frac{1}{1+50} = \frac{1}{51} \approx 1.96\%$

(2) GBN：$W_s \times T_f = 7$ ms $< 51$ ms

$U = \frac{7}{51} \approx 13.73\%$

(3) $W_s \times 1 \geq 51$，$W_s \geq 51$

最小窗口 = **51**

### 例题2

**题目**：以太网，数据率100Mbps，最大帧长1518字节（含帧头帧尾18字节），单向传播时延 $25.6\mu s$，求以太网的最大信道利用率。

**解答**：

$T_f = 1518 \times 8 / (100 \times 10^6) = 121.44\ \mu s$

$T_p = 25.6\ \mu s$

$2T_p = 51.2\ \mu s$

以太网等效于停等（CSMA/CD需要等待确认）：

$U = \frac{121.44}{121.44 + 51.2} = \frac{121.44}{172.64} \approx 70.3\%$

### 例题3（包含ACK）

**题目**：停等协议，帧长2000bit，ACK长40bit，数据率1Mbps，单向传播时延20ms。

**解答**：

$T_f = 2$ ms，$T_{ACK} = 40/10^6 = 0.04$ ms

$U = \frac{2}{2 + 0.04 + 40} = \frac{2}{42.04} \approx 4.76\%$

### 例题4（帧长含开销）

**题目**：数据载荷1000字节，帧头帧尾共26字节，数据率10Mbps，单向传播时延5ms。

求发送一帧数据（含开销）的利用率。

**解答**：

总帧长 = 1000 + 26 = 1026 字节 = 8208 bit

$T_f = 8208/10^7 = 0.8208$ ms

$U = \frac{0.8208}{0.8208 + 10} \approx 7.6\%$

## 直观理解

- **计算信道利用率就像计算"有效工作时间占比"**：真正传数据的时间 /（传数据 + 等待的时间）
- **记住**：先算 $T_f$ 和 $2T_p$，再比较大小决定用哪个公式

## 协议关联

- 信道利用率是**数据链路层计算题的核心**
- 常与窗口大小约束结合出题
- 注意区分"发送时延"和"传播时延"——考试最容易混淆的两个概念
