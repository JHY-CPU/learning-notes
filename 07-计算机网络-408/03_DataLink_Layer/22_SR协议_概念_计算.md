# 22_SR协议_概念_计算

## 核心概念

- **SR窗口约束**：$W_s \leq 2^{n-1}$，$W_r \leq 2^{n-1}$，$W_s + W_r \leq 2^n$
- **SR的信道利用率**：与GBN相同公式，但实际效率更高（只重传出错帧）
- **逐一确认**：每个帧都有独立的ACK

### 关键公式

$$W_{s,\max} = W_{r,\max} = 2^{n-1}$$

$$U_{SR} = \frac{W_s \times T_f}{T_f + 2T_p} \quad (\text{当 } W_s \times T_f \leq T_f + 2T_p)$$

## 原理分析

### 例题1（窗口大小）

**题目**：SR协议使用4位序号，发送窗口和接收窗口最大各是多少？

**解答**：

$$W_s = W_r = 2^{4-1} = 8$$

验证：$W_s + W_r = 8 + 8 = 16 = 2^4$ ✓

### 例题2（序号位数推导）

**题目**：SR协议要求发送窗口至少为8，接收窗口至少为8，至少需要几位序号？

**解答**：

$$2^{n-1} \geq 8$$
$$n-1 \geq 3$$
$$n \geq 4$$

需要**4位序号**。

验证：$2^4 = 16$，$W_s + W_r = 16 \leq 2^4$ ✓

### 例题3（窗口约束判断）

**题目**：3位序号的SR协议，以下窗口组合哪些合法？

A. $W_s=4, W_r=4$
B. $W_s=5, W_r=3$
C. $W_s=3, W_r=3$
D. $W_s=4, W_r=3$

**解答**：

$n=3$，$2^{n-1}=4$，$2^n=8$

- A: $W_s=4\leq4$ ✓，$W_r=4\leq4$ ✓，$W_s+W_r=8\leq8$ ✓ **合法**
- B: $W_s=5>4$ ✗ **不合法**
- C: $W_s=3\leq4$ ✓，$W_r=3\leq4$ ✓，$W_s+W_r=6\leq8$ ✓ **合法**
- D: $W_s=4\leq4$ ✓，$W_r=3\leq4$ ✓，$W_s+W_r=7\leq8$ ✓ **合法**

答案：A、C、D

### 例题4（SR帧传输）

**题目**：SR协议，$W_s=4, W_r=4$，序号4位。发送方发送帧0,1,2,3，帧1和帧3丢失。

分析接收方的行为和最终确认过程：

1. 帧0到达，序号0在接收窗口[0,1,2,3]内，接收缓存，发送ACK0
2. 帧1丢失
3. 帧2到达，序号2在窗口内，接收缓存，发送ACK2
4. 帧3丢失
5. 发送方收到ACK0，帧0标记确认；收到ACK2，帧2标记确认
6. 帧1超时，只重传帧1
7. 帧1到达，接收缓存，发送ACK1
8. 此时帧0,1,2都收到了，帧3还缺
9. 帧3超时，只重传帧3
10. 帧3到达，接收缓存，发送ACK3
11. 接收窗口内帧0-3全部到齐，一起交付上层，窗口滑到[4,5,6,7]

### 例题5（信道利用率）

**题目**：SR协议，$W_s=4$，帧长2000bit，数据率1Mbps，单向传播时延30ms。

**解答**：

$T_f = 2000 / 10^6 = 2$ ms

$W_s \times T_f = 4 \times 2 = 8$ ms

$T_f + 2T_p = 2 + 60 = 62$ ms

$W_s \times T_f < T_f + 2T_p$，所以：

$$U = \frac{W_s \times T_f}{T_f + 2T_p} = \frac{8}{62} \approx 12.9\%$$

## 直观理解

- **SR的窗口约束**比GBN更严格：GBN是 $2^n-1$，SR是 $2^{n-1}$。因为SR的接收窗口也滑动，必须防止新旧序号混淆
- **逐一确认**就像"每件快递单独签收"，适合高误码率环境（避免不必要的重传）

## 协议关联

- SR vs GBN：序号位数相同时，SR的最大窗口比GBN小
  - GBN: $n=3$ → $W_s=7$
  - SR: $n=3$ → $W_s=4$
- SR适合高误码率信道，GBN适合低误码率信道
- TCP的SACK机制借鉴了SR的思想
