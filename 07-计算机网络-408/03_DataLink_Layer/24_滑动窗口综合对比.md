# 24_滑动窗口综合对比

## 核心概念

- **三种ARQ协议的对比**：停等、GBN、SR都是滑动窗口协议的特例
- **核心参数对比**：窗口大小、确认方式、重传策略、适用场景
- **408必考**：三种协议的区别和联系

### 三种协议对比表

| 特性 | 停等协议 | GBN | SR |
|------|---------|-----|-----|
| 发送窗口 $W_s$ | 1 | $\leq 2^n-1$ | $\leq 2^{n-1}$ |
| 接收窗口 $W_r$ | 1 | 1 | $\leq 2^{n-1}$ |
| 确认方式 | 单独ACK | 累积确认 | 逐一确认 |
| 失序帧处理 | 不会出现 | 丢弃 | 缓存 |
| 重传出错帧 | 重传该帧 | 重传后续所有 | 只重传该帧 |
| 实现复杂度 | 最简单 | 较简单 | 较复杂 |
| 适用场景 | 教学/简单链路 | 低误码率 | 高误码率 |

### 序号位数约束

| 协议 | 约束条件 | n=3时最大窗口 |
|------|---------|-------------|
| 停等 | $n \geq 1$ | $W_s=1$ |
| GBN | $W_s \leq 2^n-1$ | $W_s=7$ |
| SR | $W_s \leq 2^{n-1}$, $W_s+W_r \leq 2^n$ | $W_s=4$ |

### 信道利用率对比

$$U_{\text{停等}} = \frac{T_f}{T_f + T_{ACK} + 2T_p}$$

$$U_{\text{GBN}} = \frac{W_s \times T_f}{T_f + 2T_p} \quad (\text{当 } W_s T_f \leq T_f + 2T_p)$$

$$U_{\text{SR}} = \frac{W_s \times T_f}{T_f + 2T_p} \quad (\text{当 } W_s T_f \leq T_f + 2T_p)$$

注意：GBN和SR的利用率公式形式相同，但SR实际效率更高（重传更少）。

## 原理分析

### 关系图

```
停等协议 = GBN(Ws=1) = SR(Ws=1, Wr=1)
     ↓
GBN: Ws>1, Wr=1 (累积确认)
     ↓
SR:  Ws>1, Wr>1 (逐一确认)
```

### 窗口约束的推导

**GBN**: 接收窗口=1，只需防止发送窗口中的帧序号与待接收帧的序号混淆

$$W_s + 1 \leq 2^n \Rightarrow W_s \leq 2^n - 1$$

**SR**: 发送窗口和接收窗口都可能滑动，需要防止两边的序号重叠

$$W_s + W_r \leq 2^n$$

当 $W_s = W_r$ 时：$W_s = W_r \leq 2^{n-1}$

### 适用场景分析

| 场景 | 最佳选择 | 原因 |
|------|---------|------|
| 低误码率、短距离 | GBN | 累积确认高效，出错少 |
| 高误码率、长距离 | SR | 减少不必要的重传 |
| 简单嵌入式系统 | 停等 | 实现最简单 |
| TCP可靠传输 | GBN+SR混合 | 累积确认+SACK |

## 直观理解

- **停等**：一个人搬砖，一次一块
- **GBN**：一个人推车，一次搬N块，但路上掉了一块就得把整车推回去重来
- **SR**：一个人推车，一次搬N块，路上掉了哪块就回去拿哪块

### 窗口大小的记忆口诀

> "GBN减1（$2^n-1$），SR减半（$2^{n-1}$）"

## 协议关联

- 三种协议都是**可靠传输**的实现方式
- 都基于**确认+超时重传+序号**三个要素
- TCP综合使用了这些机制：滑动窗口 + 累积确认 + SACK
- 408考试中常以对比表格形式出选择题
