# 68_HDLC帧格式与控制字段

## 核心概念

- **HDLC控制字段**：1字节（基本模式）或2字节（扩展模式），决定帧类型
- **三种帧类型**：I帧（数据）、S帧（控制）、U帧（管理）
- **HDLC支持可靠传输**：通过序号和确认机制

### 控制字段详解（基本模式，1字节）

| 位 | I帧 | S帧 | U帧 |
|----|-----|-----|-----|
| 1 | 0 | 1 | 1 |
| 2 | N(S) bit0 | 0 | M1 bit0 |
| 3 | N(S) bit1 | 0 | M1 bit1 |
| 4 | N(S) bit2 | Type bit0 | P/F |
| 5 | P/F | Type bit1 | M2 bit0 |
| 6 | N(R) bit0 | N(R) bit0 | M2 bit1 |
| 7 | N(R) bit1 | N(R) bit1 | M2 bit2 |
| 8 | N(R) bit2 | N(R) bit2 | M2 bit3 |

### S帧类型

| Type值 | 名称 | 功能 |
|--------|------|------|
| 00 | RR | 接收准备好，确认N(R)之前的帧 |
| 01 | REJ | 拒绝，要求从N(R)开始重传（GBN） |
| 10 | RNR | 接收未准备好，暂停发送 |
| 11 | SREJ | 选择拒绝，只要求重传N(R)帧（SR） |

### 常见U帧

| M1M2 | 命令/响应 | 说明 |
|------|----------|------|
| 00000 | SNRM | 设置正常响应模式 |
| 11000 | SABM | 设置异步平衡模式 |
| 00010 | DISC | 断开连接 |
| 00110 | UA | 无编号确认 |
| 10001 | DM | 断开模式 |
| 10000 | FRMR | 帧拒绝 |

### 考试重点

- 通过控制字段第1位判断帧类型
- S帧的四种类型及其功能
- HDLC支持GBN和SR两种重传方式

## 原理分析

### I帧的确认机制

HDLC使用**滑动窗口**和**累积确认**：

- I帧的 $N(R)$ 字段表示"我期望接收的下一个帧序号"
- 等价于确认了 $N(R)-1$ 及之前的所有帧

例如：收到I帧 $N(R)=5$，表示帧0,1,2,3,4都已正确接收，期望帧5。

### HDLC的流量控制

- **RR帧**：接收准备好，允许发送方继续发送
- **RNR帧**：接收未准备好，要求发送方暂停
- 通过RR和RNR实现流量控制

### 示例流程

```
主站                    从站
 |  I帧, N(S)=0, N(R)=0  -->   |
 |                       <--  I帧, N(S)=0, N(R)=1  (确认帧0)
 |  I帧, N(S)=1, N(R)=1  -->   |
 |                       <--  S帧(RR), N(R)=2       (确认帧1)
 |  I帧, N(S)=2, N(R)=1  -->   |
 |                       <--  S帧(REJ), N(R)=2      (要求重传帧2)
 |  I帧, N(S)=2, N(R)=1  -->   (重传)
```

## 直观理解

- **控制字段就像"指令编码"**：第1位是"指令类型标志"，后面是具体参数
- **HDLC的I帧**既能传输数据又能确认（类似TCP的DATA+ACK）

## 协议关联

- HDLC的控制字段设计影响了后来的PPP和帧中继
- PPP简化了HDLC：PPP帧没有序号，不提供可靠传输
- HDLC的S帧与TCP的ACK/SACK有类似思想
