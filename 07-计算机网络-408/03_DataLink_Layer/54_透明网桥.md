# 54_透明网桥

## 核心概念

- **透明网桥（Transparent Bridge）**：即插即用，站点不需要知道网桥的存在
- **"透明"**的含义：网桥对站点是不可见的，站点感觉不到网桥的存在
- **IEEE 802.1D**标准定义了透明网桥的规范

### 透明网桥的工作特点

1. **即插即用**：自动学习MAC地址，无需配置
2. **逆向学习**：从帧的源MAC地址学习端口映射
3. **自适应**：自动适应网络拓扑变化
4. **使用生成树协议**：避免环路

### 逆向学习法

- 网桥收到帧，检查**源MAC地址**
- 如果源MAC不在转发表中，记录：源MAC → 入端口
- 如果源MAC已在表中但对应不同端口，更新端口映射
- 这叫"逆向学习"：因为从某端口收到A发的帧，说明去A应该从这个端口走

### 环路问题

多个网桥互联形成环路时：

```
网桥A ---- 网桥B
  |          |
  +-- 网桥C -+
```

- 广播帧会在环中**无限循环**（广播风暴）
- 转发表不断被错误更新
- 解决方案：**生成树协议（STP）**

### 生成树协议（STP）简介

- 通过交换BPDU报文，构建无环的生成树拓扑
- 选择根网桥（Bridge ID最小的）
- 阻塞冗余端口
- 故障时自动启用备份端口

## 原理分析

### 转发表的构建过程

初始：转发表为空

1. A从端口1发帧给B（目的MAC = B）
   - 源MAC = A → 记录：A → 端口1
   - 目的MAC = B → 不在表中 → 泛洪

2. B从端口2回复帧给A
   - 源MAC = B → 记录：B → 端口2
   - 目的MAC = A → 在表中，端口1 → 转发到端口1

3. 后续A→B的帧：查表直接转发，不再泛洪

### 老化时间

- 默认300秒（5分钟）
- 如果某MAC地址300秒内没有帧从对应端口到达，删除该表项
- 目的：适应网络变化（设备移动、断线等）

## 直观理解

- **"透明"**就像空气：设备通信时感觉不到网桥的存在
- **逆向学习**就像"通过快递反推地址"：收到来自张三的快递，就知道张三住在哪个方向

## 协议关联

- 透明网桥是**最常用**的网桥类型（以太网使用）
- 源路由网桥用于令牌环网（现在很少见）
- STP是透明网桥解决环路问题的核心协议
- 现代交换机本质上就是多端口的透明网桥
