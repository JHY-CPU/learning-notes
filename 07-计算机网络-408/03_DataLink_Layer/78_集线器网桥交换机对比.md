# 78_集线器网桥交换机对比

## 核心概念

- **集线器（Hub）**：物理层，共享带宽，所有端口同一冲突域
- **网桥（Bridge）**：数据链路层，基于MAC转发，隔离冲突域
- **交换机（Switch）**：多端口网桥，每个端口独享带宽，硬件转发

### 三者对比表

| 特性 | 集线器 | 网桥 | 交换机 |
|------|--------|------|--------|
| 工作层次 | 物理层 | 数据链路层 | 数据链路层 |
| 转发依据 | 无（比特复制） | MAC地址 | MAC地址 |
| 冲突域 | 所有端口共享1个 | 每端口1个 | 每端口1个 |
| 广播域 | 1个 | 1个 | 1个（除非VLAN） |
| 带宽 | 所有端口共享 | 端口独立 | 端口独立 |
| 转发方式 | 比特复制 | 存储转发 | 存储/直通/无碎片 |
| 转发表 | 无 | 有（学习） | 有（学习） |
| 速度 | 快（不处理） | 较慢（软件） | 快（ASIC硬件） |
| 端口数 | 4-24 | 2-16 | 8-48 |

### 集线器的工作方式

```
A发送帧 → 集线器 → 复制到所有端口 → B、C、D都收到
```

- 不识别帧，不知道目的地址
- 所有设备共享带宽（10M集线器，4台设备各分2.5M）
- 同一时刻只能一个设备发送（CSMA/CD）

### 网桥的工作方式

```
A发送帧 → 网桥 → 查MAC表 → 转发到目的端口 → B收到
```

- 识别帧，查MAC地址表
- 过滤：同端口的帧不转发
- 泛洪：未知目的时向所有端口广播

### 交换机的工作方式

```
A发送帧 → 交换机 → 查MAC表 → 转发到目的端口 → B收到
        （每个端口独立，可以多对同时通信）
```

- 与网桥原理相同
- 关键区别：硬件转发、端口数多、每个端口独立带宽

### 考试重点

- 集线器是物理层设备，交换机是数据链路层设备
- 集线器共享冲突域，交换机隔离冲突域
- 交换机的每个端口是独立冲突域但同一广播域

## 原理分析

### 带宽计算

**集线器场景**：10Mbps集线器，连接4台设备

每台设备可用带宽 = 10Mbps / 4 = 2.5Mbps（共享）

**交换机场景**：10Mbps交换机，连接4台设备

每台设备可用带宽 = 10Mbps（独享）
最大总带宽 = 10Mbps × 4 = 40Mbps（全双工时更高）

### 冲突域分析

**例题**：1台集线器连接2台交换机，每台交换机连接4台PC。

- 集线器：1个冲突域
- 每台交换机：4个冲突域（每端口1个）
- 总冲突域 = 1 + 4 + 4 = 9个

但注意：集线器连接的交换机端口在集线器的冲突域中。

更准确：集线器的所有端口（包括连接交换机的端口）共享1个冲突域。

如果每台交换机有8个端口，其中1个端口连集线器，7个端口连PC：
- 集线器：1个冲突域
- 交换机A：7个独立冲突域（7个PC端口）+ 1个共享冲突域（连集线器的端口）
- 交换机B：同上

## 直观理解

- **集线器 = 大房间**：所有人在一起，只能一个人说话
- **交换机 = 办公楼**：每间办公室独立，多人可同时通话

## 协议关联

- 现代网络中，集线器基本被淘汰，交换机成为标准
- 三层交换机 = 交换 + 路由
- 交换机 + VLAN 可以替代部分路由器的功能
