# 01_物理层基本概念

## 核心概念

- **物理层**是OSI参考模型的最底层（第1层），负责在传输介质上透明地传输比特流（0和1序列）
- 物理层的主要任务不是指具体的传输介质，而是确定与传输介质的**接口特性**
- 物理层关注的是如何在连接各种计算机的传输介质上传输数据比特流，而不是指具体的传输介质
- 408常考：物理层研究的是**接口特性**，不是传输介质本身

### 四大接口特性（必考）

| 特性 | 含义 | 举例 |
|------|------|------|
| **机械特性** | 接口的物理形状、尺寸、引脚数目 | RJ-45插头8根引脚 |
| **电气特性** | 电压范围、阻抗、信号速率 | 电压范围 -15V ~ +15V |
| **功能特性** | 每根引脚的功能定义 | 某引脚发送数据、某引脚接地 |
| **规程特性** | 工作规程和时序关系 | 建立连接时先做什么、后做什么 |

### 常见术语辨析

- **数据（Data）**：运送消息的实体
- **信号（Signal）**：数据的电气或电磁表现
- **模拟信号**：连续变化的信号
- **数字信号**：离散取值的信号
- **码元（Symbol）**：在使用时间域的波形表示数字信号时，代表不同离散数值的基本波形
- **单工/半双工/全双工**：通信双方的工作方式

### 考试关注点

- 物理层不负责差错控制（这是数据链路层的事）
- 物理层不负责流量控制
- 物理层协议常被称为**规程（Procedure）**
- 信源、信道、信宿的概念区分

## 原理分析

### 物理层的核心职责

```
应用层 → 表示层 → 会话层 → 传输层 → 网络层 → 数据链路层 → [物理层]
                                                              ↓
                                                       传输介质（比特流）
```

物理层只关心比特流如何从一个设备传到另一个设备，不关心比特流的含义。

### 数据通信系统基本模型

```
信源 → 发送器 → 信道 → 接收器 → 信宿
              ↑
            噪声源
```

- **信源**：产生数据的设备（如计算机）
- **发送器**：将数据转换为适合传输的信号（如调制解调器）
- **信道**：信号传输的通路
- **接收器**：将信号还原为数据
- **信宿**：接收数据的设备
- **噪声源**：信道中的干扰

## 直观理解

### 类比：物流系统

- 物理层好比**高速公路系统**：它不关心车上装的是什么货物（数据内容），只关心道路的通行规则（接口特性）
- 数据链路层好比**运输公司的管理**：关心货物是否完整送达
- 物理层的四大接口特性好比公路的**路面宽度（机械）、限速标志（电气）、车道功能（功能）、交通信号灯时序（规程）**

### 记忆口诀

> 物理层，定接口，四特性，记心间
> 机（机械）电（电气）功（功能）规（规程），顺序不会乱

## 协议关联

### 与上层的关系

- **数据链路层**依赖物理层提供的比特流传输服务
- 物理层向数据链路层提供**透明的比特流传输**服务
- "透明"意味着数据链路层不需要关心物理层如何实现传输

### 典型考题模式

- 问：物理层的功能是什么？答：定义接口特性，在介质上传输比特流
- 问：物理层传输的单位是什么？答：**比特（bit）**
- 易混淆：数据链路层传输单位是**帧**，网络层是**数据报**，传输层是**报文段**

### 与协议标准的关系

- 物理层的典型标准：EIA-232、RS-449、X.21、V.24 等
- 这些标准规定了接口的四大特性
- 408常考：这些标准不涉及传输介质本身的特性
