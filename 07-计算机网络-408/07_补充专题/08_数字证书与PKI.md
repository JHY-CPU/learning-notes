# 08_数字证书与PKI

## 核心概念

- **数字证书定义**：由CA签发的、绑定公钥与身份的电子文档
- **证书内容**（X.509格式）：
  - 版本号、序列号
  - 签名算法（如SHA256withRSA）
  - 颁发者（CA）名称
  - 有效期（起止日期）
  - 主体（持有者）名称
  - 主体公钥信息
  - CA的数字签名
- **CA（证书颁发机构）**：受信任的第三方，负责签发和管理证书
- **PKI（公钥基础设施）**：包括CA、RA（注册机构）、证书库、CRL的完整体系
- **证书链**：Root CA → Intermediate CA → 用户证书，逐级签名信任
- **CRL（证书吊销列表）**：被吊销的证书序列号列表
- **OCSP（在线证书状态协议）**：实时查询证书是否有效
- **408重点**：证书内容、证书验证过程、信任链

## 原理分析

### 证书签发流程

1. 用户生成密钥对 $(PK, SK)$
2. 用户向CA提交身份证明和公钥 $PK$
3. CA验证用户身份
4. CA用自己的私钥 $SK_{CA}$ 对用户信息和公钥签名：
   $$\text{Cert} = \{用户信息, PK_{用户}\} + D_{SK_{CA}}(Hash\{用户信息, PK_{用户}\})$$
5. CA颁发证书给用户

### 证书验证流程

1. 检查证书有效期
2. 检查证书是否在CRL中
3. 用颁发者CA的公钥验证证书签名
4. 沿证书链向上验证，直到根CA
5. 检查根CA是否在信任列表中

### 证书信任链

```
Root CA (自签名，预装在浏览器/OS中)
  ↓ 用Root CA私钥签名
Intermediate CA
  ↓ 用Intermediate CA私钥签名
用户证书 (www.example.com)
```

验证时从下往上：用户证书→Intermediate CA→Root CA

## 直观理解

- **证书类比**：数字证书就像身份证
  - CA = 公安局（权威签发机构）
  - 证书 = 身份证（绑定身份和照片/公钥）
  - 证书链 = 村委会→派出所→公安局的逐级盖章
  - CRL = 失效身份证公告栏
- **为什么需要PKI**：公钥谁都能生成，如何确认"这个公钥真的是张三的"？→ 需要CA来担保
- **证书链类比**：就像推荐信——我不认识你，但我认识推荐你的人，我信任推荐人，所以我信任你
- **记忆口诀**："证书=公钥+身份+CA签名，链式信任到Root"

## 协议关联

- **与HTTPS/TLS的关联**：
  - 浏览器访问HTTPS网站时，服务器发送其数字证书
  - 浏览器验证证书链（从服务器证书到根CA）
  - 验证通过后，从证书中提取服务器公钥用于密钥交换
- **与IPsec的关联**：IKE可用数字证书进行身份认证
- **与电子邮件的关联**：S/MIME用数字证书加密和签名邮件
- **408常见陷阱**：
  - "证书中有私钥"——错，证书只有公钥，私钥由用户自己保管
  - "CA用自己的公钥签名证书"——错，CA用私钥签名
  - "根CA的证书是别人签发的"——错，根CA是自签名证书
- **真题方向**：证书验证过程、信任链机制、证书与PKI的关系
