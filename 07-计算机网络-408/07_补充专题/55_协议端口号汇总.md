# 55_协议端口号汇总

## 核心概念

- **端口号范围**：
  - 知名端口（Well-Known）：0-1023（IANA分配）
  - 注册端口（Registered）：1024-49151
  - 动态端口（Dynamic/Private）：49152-65535（临时端口）
- **端口号作用**：标识主机上的进程，与IP地址共同构成套接字

## 原理分析

### 传输层协议与端口

| 协议 | 端口 | 传输层 | 功能 |
|------|------|-------|------|
| HTTP | 80 | TCP | 超文本传输 |
| HTTPS | 443 | TCP | 安全超文本传输 |
| FTP-控制 | 21 | TCP | 文件传输控制连接 |
| FTP-数据 | 20 | TCP | 文件传输数据连接 |
| SSH | 22 | TCP | 安全远程登录 |
| Telnet | 23 | TCP | 远程登录（不安全） |
| SMTP | 25 | TCP | 发送邮件 |
| DNS | 53 | UDP/TCP | 域名解析 |
| DHCP-服务器 | 67 | UDP | 动态主机配置 |
| DHCP-客户端 | 68 | UDP | 动态主机配置 |
| TFTP | 69 | UDP | 简单文件传输 |
| POP3 | 110 | TCP | 接收邮件 |
| IMAP | 143 | TCP | 邮件访问 |
| SNMP | 161 | UDP | 网络管理查询 |
| SNMP Trap | 162 | UDP | 网络管理告警 |
| BGP | 179 | TCP | 边界网关协议 |

### 常见组合记忆

**邮件相关**：
- SMTP(25)发邮件 → POP3(110)/IMAP(143)收邮件

**文件传输**：
- FTP(21控制+20数据) vs TFTP(69，简单版)

**远程登录**：
- SSH(22，安全) vs Telnet(23，不安全)

**网络管理**：
- SNMP(161查询+162告警)

**安全协议**：
- HTTPS(443) = HTTP(80) + TLS

## 直观理解

- **端口记忆法**：
  - "80看网页，443安全看，21传文件，25发邮件，53查域名"
  - "22安全登录，23不安全登录"
  - "67/68分IP地址，69简单传文件"
- **TCP vs UDP**：
  - TCP端口多（需要可靠的）
  - UDP端口少（DNS查询、DHCP、SNMP）
- **易错**：
  - DNS通常用UDP(53)，但区域传输用TCP(53)
  - FTP有2个端口：21控制，20数据
  - DHCP有2个端口：67服务器，68客户端

## 协议关联

- **端口与Socket**：Socket = IP + 端口号
- **端口与进程**：端口号标识主机上的具体进程
- **NAT与端口**：NAT/PAT使用端口号区分不同内网主机
- **408考法**：端口号与协议对应、TCP/UDP端口区分
