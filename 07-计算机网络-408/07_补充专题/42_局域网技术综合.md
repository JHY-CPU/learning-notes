# 42_局域网技术综合

## 核心概念

- **局域网（LAN）定义**：地理范围有限（一栋楼、一个校园）的计算机网络
- **以太网（Ethernet）**：最主流的LAN技术
  - 标准：IEEE 802.3
  - 介质访问控制：CSMA/CD（半双工）/ 全双工（交换式）
  - 帧格式：Ethernet II（DIX）/ IEEE 802.3
- **以太网帧格式对比**：

| 字段 | Ethernet II | IEEE 802.3 |
|------|------------|------------|
| 前导码 | 8B | 8B |
| 目的MAC | 6B | 6B |
| 源MAC | 6B | 6B |
| 类型/长度 | 类型(2B) | 长度(2B) |
| 数据 | 46-1500B | LLC+数据 |
| FCS | 4B | 4B |

- **最小帧长**：以太网最小帧64字节（数据部分最少46字节），保证冲突检测
- **交换机 vs 集线器**：
  - 集线器（Hub）：物理层，所有端口一个冲突域
  - 交换机（Switch）：数据链路层，每个端口一个冲突域
- **VLAN（虚拟局域网）**：在交换机上逻辑划分广播域
  - IEEE 802.1Q：在以太网帧中插入4字节VLAN标签
- **无线局域网（WLAN）**：IEEE 802.11系列
  - CSMA/CA（避免冲突，不是检测）
  - 为什么不用CSMA/CD：无线无法同时收发（冲突检测需要）
- **408重点**：以太网帧格式、CSMA/CD、交换机与集线器的区别

## 原理分析

### CSMA/CD工作流程

1. **载波侦听（Carrier Sense）**：发送前监听信道
2. **多路访问（Multiple Access）**：多台主机共享信道
3. **冲突检测（Collision Detection）**：边发边检测冲突
   - 检测到冲突：立即停止，发送Jam信号
   - 等待随机时间后重试（二进制指数退避）

**最小帧长的由来**：
$$L_{min} \geq 2 \times \tau \times R$$
- $\tau$：最大传播延迟
- $R$：数据速率
- 10Mbps以太网：$L_{min} = 2 \times 25\mu s \times 10Mbps = 512 bits = 64 bytes$

### CSMA/CD vs CSMA/CA

| 特性 | CSMA/CD | CSMA/CA |
|------|---------|---------|
| 应用 | 有线以太网 | 无线WLAN |
| 冲突处理 | 检测后停止 | 尽量避免冲突 |
| 隐藏站问题 | 无 | 有（使用RTS/CTS解决） |
| 全双工 | 支持（无冲突） | 不支持 |

### VLAN

**802.1Q标签插入**：
```
原始帧：[DMAC | SMAC | Type | Data | FCS]
VLAN帧：[DMAC | SMAC | VLAN Tag(4B) | Type | Data | FCS]
```

VLAN Tag = TPID(2B) + PRI(3bit) + CFI(1bit) + VID(12bit)

- VID：VLAN ID，0-4095
- 不同VLAN之间的通信需要三层设备（路由器或三层交换机）

## 直观理解

- **集线器 vs 交换机**：
  - 集线器：像一个大房间，所有人说话都能听到（共享冲突域）
  - 交换机：像多间办公室，每间独立隔音（每个端口独立冲突域）
- **VLAN**：在交换机上"虚拟"划分多个独立网络
- **CSMA/CD**：像开会——先听有没有人说话，说了发现同时有人在说就闭嘴等
- **CSMA/CA**：像打电话——先问"有人在用这条线吗？"再打

## 协议关联

- **以太网与ARP**：IP包需要ARP获取MAC地址才能封装成以太网帧
- **以太网与IP**：以太网帧承载IP包
- **VLAN与子网**：VLAN是链路层概念，子网是网络层概念，通常一一对应
- **408考法**：最小帧长计算、CSMA/CD流程、交换机vs集线器、VLAN概念
