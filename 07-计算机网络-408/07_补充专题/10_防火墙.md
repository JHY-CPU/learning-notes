# 10_防火墙

## 核心概念

- **防火墙定义**：部署在网络边界，根据安全策略控制进出流量的设备/软件
- **防火墙分类**（三代）：
  - **包过滤防火墙（Packet Filter）**：检查IP/端口/协议，无状态
  - **状态检测防火墙（Stateful Inspection）**：跟踪连接状态，基于连接表
  - **应用代理防火墙（Application Proxy）**：深度检查应用层内容
- **包过滤规则五元组**：源IP、目的IP、源端口、目的端口、协议类型
- **默认策略**：
  - 默认拒绝（Default Deny）：未明确允许的全部拒绝（更安全）
  - 默认允许（Default Permit）：未明确禁止的全部允许（不安全）
- **DMZ（非军事区）**：防火墙内部的特殊区域，放置对外服务器
- **408重点**：三种防火墙的特点对比、包过滤规则设计

## 原理分析

### 包过滤防火墙

工作在网络层和传输层，检查每个数据包的：
- 源/目的IP地址
- 源/目的端口号
- 协议类型（TCP/UDP/ICMP）

**优点**：速度快、对应用透明
**缺点**：无状态，无法检测IP欺骗和端口扫描

**规则示例**：
```
允许 内网 → 外网 HTTP (端口80)
允许 内网 → 外网 HTTPS(端口443)
拒绝 外网 → 内网 所有入站连接
```

### 状态检测防火墙

维护连接状态表，跟踪TCP连接状态：
```
(源IP, 源端口, 目的IP, 目的端口, 状态, 超时)
```

- 只有从内网发起的连接的返回流量才允许通过
- 能自动识别SYN/ACK/FIN/RST等TCP状态
- 防止SYN Flood：限制半连接数

### 应用代理防火墙

在应用层代理所有通信：
```
客户端 → 代理防火墙 → 服务器
       (连接1)     (连接2)
```

- 客户端与代理建立连接，代理与服务器建立另一连接
- 可检查应用层内容（URL、HTTP头、邮件附件等）
- **优点**：最深层检查，能识别应用层攻击
- **缺点**：性能开销大，需要为每种应用编写代理

### 三种防火墙对比

| 特性 | 包过滤 | 状态检测 | 应用代理 |
|------|-------|---------|---------|
| 工作层 | 网络/传输层 | 网络/传输层 | 应用层 |
| 检查深度 | 包头 | 包头+状态 | 完整内容 |
| 速度 | 最快 | 快 | 慢 |
| 安全性 | 低 | 中 | 高 |
| 应用感知 | 无 | 部分 | 完全 |

## 直观理解

- **包过滤**：像门卫只看你的证件（IP/端口），有证件就放行
- **状态检测**：像门卫不仅看证件，还记住谁进来了，只有"认识"的返回者才放行
- **应用代理**：像门卫替你进去办事，检查一切内容后再决定是否转交给你
- **DMZ类比**：访客区——外人可以进访客区见接待员，但不能进入内部办公区
- **记忆口诀**："包过滤看证件，状态检测记人脸，应用代理当替身"

## 协议关联

- **与网络层**：包过滤检查IP地址和协议号
- **与传输层**：状态检测跟踪TCP连接状态
- **与应用层**：应用代理检查HTTP/FTP/SMTP内容
- **与NAT的关联**：NAT设备通常也具备包过滤功能
- **408常见陷阱**：
  - "防火墙能防病毒"——包过滤和状态检测不能，应用代理部分能
  - "防火墙能防内部攻击"——错，防火墙在边界，只防外部
  - "应用代理防火墙最安全所以应该用它"——实际中需要权衡性能
- **真题方向**：三种防火墙的对比、包过滤规则设计
