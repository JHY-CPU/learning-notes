# 26_拥塞控制跨层_考试

## 核心概念

- **408考纲要点**：TCP拥塞控制算法、拥塞窗口变化
- **必背公式**：
  - 慢启动：每RTT，$cwnd$ 翻倍（$cwnd = 2^n$）
  - 拥塞避免：每RTT，$cwnd = cwnd + 1$
  - 超时：$ssthresh = cwnd/2$，$cwnd = 1$，进入慢启动
  - 3个重复ACK：$ssthresh = cwnd/2$，$cwnd = ssthresh + 3$，进入拥塞避免
- **拥塞控制 vs 流量控制**：
  - 流量控制：端到端，接收窗口 $rwnd$
  - 拥塞控制：全局性，拥塞窗口 $cwnd$
  - 实际发送窗口：$swnd = \min(rwnd, cwnd)$

## 原理分析

### 典型真题：拥塞窗口计算

**题目**：TCP连接初始 $ssthresh = 16$ MSS，发生超时后 $ssthresh$ 和 $cwnd$ 分别变为多少？（）
A. 8, 8 B. 8, 1 C. 16, 1 D. 16, 8

**解析**：答案B。超时后：$ssthresh = cwnd/2 = 16/2 = 8$，$cwnd = 1$，回到慢启动。

### 典型真题：cwnd变化过程

**题目**：TCP初始 $cwnd = 1$，$ssthresh = 8$。经过多少个RTT后 $cwnd$ 达到 $ssthresh$？

**解析**：慢启动阶段每RTT翻倍：
- RTT 1：$cwnd = 1 \to 2$
- RTT 2：$cwnd = 2 \to 4$
- RTT 3：$cwnd = 4 \to 8$（到达 $ssthresh$，转为拥塞避免）

答案：3个RTT。

### 典型真题：快重传触发

**题目**：TCP发送方收到3个重复ACK时，应该（）
A. 重传计时器超时再重传
B. 立即重传，cwnd减半后继续拥塞避免
C. 立即重传，cwnd设为1，回到慢启动
D. 忽略重复ACK

**解析**：答案B。快重传：收到3个重复ACK立即重传。快恢复：$ssthresh = cwnd/2$，$cwnd = ssthresh + 3$，继续拥塞避免（不回慢启动）。

### 典型真题：拥塞控制vs流量控制

**题目**：TCP实际发送窗口的大小取决于（）
A. 仅接收窗口 $rwnd$ B. 仅拥塞窗口 $cwnd$ C. $\min(rwnd, cwnd)$ D. $\max(rwnd, cwnd)$

**解析**：答案C。$swnd = \min(rwnd, cwnd)$，既要考虑接收方能力，也要考虑网络承载能力。

## 直观理解

- **答题技巧**：
  - 超时→回慢启动（$cwnd = 1$）
  - 3个重复ACK→快重传+快恢复（$cwnd = ssthresh + 3$）
  - 慢启动→指数增长
  - 拥塞避免→线性增长
- **cwnd变化图记忆**：像锯齿波，上升（拥塞避免线性增长）→下降（拥塞发生减半）→上升...
- **易错点**：
  - 快恢复不是回到慢启动（这是最大区别）
  - $swnd = \min(rwnd, cwnd)$ 不是 $\max$

## 协议关联

- **与TCP**：TCP拥塞控制是传输层核心
- **与IP层**：ECN由路由器设置，TCP响应ECN降速
- **与应用层**：应用层也可实施限流（如令牌桶）
- **408高频考法**：cwnd变化计算题、超时vs重复ACK的处理区别
