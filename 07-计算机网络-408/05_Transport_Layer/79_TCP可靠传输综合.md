# 79_TCP可靠传输综合

## 核心概念

- **TCP可靠传输四大机制**：序号、确认、超时重传、差错检测
- **累积确认**：确认号之前的所有字节都已正确收到
- **快速重传**：3个冗余ACK触发
- **SACK**：选择确认，精确告知丢失范围

### 可靠传输机制总结

| 机制 | 功能 | 实现方式 |
|------|------|---------|
| 序号 | 检测乱序和重复 | TCP首部32位SEQ字段 |
| 确认 | 告知发送方哪些数据已到 | TCP首部32位ACK字段 |
| 超时重传 | 数据丢失时重传 | RTO计时器 |
| 差错检测 | 检测数据损坏 | 校验和（含伪首部） |
| 快速重传 | 3冗余ACK时快速重传 | 冗余ACK计数 |
| SACK | 精确报告丢失范围 | TCP选项字段 |

## 原理分析

### 可靠传输完整流程

```
发送方:
1. 对数据按字节编号
2. 发送报文段，启动重传计时器
3. 等待ACK
4a. 收到ACK → 停止计时器，窗口右移
4b. 超时 → 重传，RTO加倍，重启计时器
4c. 收到3个冗余ACK → 快速重传

接收方:
1. 检查校验和
2. 校验失败 → 丢弃
3. 校验成功 → 检查序号
4a. 序号正确（按序） → 交付应用层，发送ACK
4b. 序号乱序 → 缓存，发送冗余ACK
```

### 与其他层可靠传输的对比

| 特征 | 数据链路层 | TCP |
|------|-----------|-----|
| 范围 | 点到点（相邻节点） | 端到端（源到目的） |
| 协议 | 停等、GBN、SR | 滑动窗口+累积确认 |
| 出错原因 | 噪声、干扰 | 拥塞、路由器丢弃 |

## 直观理解

- **可靠传输像考试收卷**：按编号收（序号），缺了补交（重传），坏了重写（差错检测）

## 协议关联

- **与流量控制**：可靠传输是基础，流量控制限制速率
- **与拥塞控制**：拥塞控制根据丢包信号调整cwnd
- **408综合考法**：结合序号、确认号、窗口、拥塞控制
