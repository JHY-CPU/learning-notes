# 82_传输层知识框架

## 核心概念

- **传输层知识体系总览**
- **从概述到细节的完整学习路径**

### 传输层知识结构

```
传输层
├── 概述
│   ├── 传输层功能（复用/分用、差错检测）
│   ├── 端口号（熟知/注册/动态）
│   ├── 套接字（Socket = IP + 端口）
│   └── 服务类型（面向连接/无连接）
│
├── UDP
│   ├── 概述（无连接、不可靠、面向报文）
│   ├── 报文格式（8字节首部：源端口、目的端口、长度、校验和）
│   ├── 校验和（伪首部、反码求和）
│   └── 应用场景（DNS、DHCP、实时音视频）
│
├── TCP
│   ├── 报文段格式（20~60字节，10+字段）
│   ├── 序号与确认号（32位，累积确认）
│   ├── 连接管理
│   │   ├── 三次握手（SYN → SYN+ACK → ACK）
│   │   ├── 四次挥手（FIN → ACK → FIN → ACK）
│   │   └── 状态转换图（11个状态）
│   ├── 可靠传输
│   │   ├── 确认机制（累积确认）
│   │   ├── 超时重传（RTO计算）
│   │   ├── 快速重传（3个冗余ACK）
│   │   └── SACK（选择确认）
│   ├── 流量控制
│   │   ├── 滑动窗口
│   │   ├── rwnd（接收窗口）
│   │   └── 持续计时器（零窗口探测）
│   ├── 拥塞控制
│   │   ├── 慢开始（cwnd指数增长）
│   │   ├── 拥塞避免（cwnd线性增长）
│   │   ├── 快重传/快恢复
│   │   └── AIMD（加法增大乘法减小）
│   ├── 窗口管理（发送窗口 = min(rwnd, cwnd)）
│   ├── 计时器（重传、持续、保活、时间等待）
│   └── 性能优化（Nagle、延迟确认、糊涂窗口综合症）
│
└── TCP vs UDP 对比与应用选择
```

## 原理分析

### 408考试重点排序

| 优先级 | 知识点 | 出题频率 |
|--------|--------|---------|
| ⭐⭐⭐ | 拥塞控制（cwnd变化计算） | 极高 |
| ⭐⭐⭐ | 三次握手/四次挥手 | 极高 |
| ⭐⭐⭐ | TCP状态转换 | 高 |
| ⭐⭐⭐ | 序号确认号计算 | 高 |
| ⭐⭐ | 流量控制（rwnd/cwnd） | 中高 |
| ⭐⭐ | TCP vs UDP对比 | 中高 |
| ⭐⭐ | RTO计算 | 中 |
| ⭐ | UDP校验和 | 中 |
| ⭐ | Nagle/糊涂窗口 | 低 |

## 直观理解

- **学习路径**：先整体后细节。理解传输层的位置和功能 → UDP（简单）→ TCP各机制（重点）→ 综合练习

## 协议关联

- **向上**：为应用层提供服务（端口号）
- **向下**：依赖网络层IP（主机到主机）
- **408考试**：传输层通常占15-20分，是重点章节
