# 78_TCP序号确认号综合_真题

## 核心概念

- **序号确认号综合计算是408常见题型**
- **关键**：记住SYN和FIN各消耗1序号

### 408真题精选

**题目**：主机甲和乙之间建立TCP连接，ISN甲=100，ISN乙=200。甲向乙发送数据：第一个报文段包含100字节，第二个报文段包含200字节。求：
1. 甲第一个数据报文段的SEQ
2. 乙对第一个数据报文段的ACK
3. 甲第二个数据报文段的SEQ
4. 乙对第二个数据报文段的ACK

**解析**：
1. 三次握手消耗：SYN(100) + 第三次握手(101) → 数据从**102**开始。SEQ=**102**
2. 收到100字节(102~201)，ACK=**202**
3. 第二个报文段SEQ=**202**
4. 收到200字节(202~401)，ACK=**402**

**题目**：TCP双方的ISN分别为x和y。第一个数据报文段的SEQ和ACK各是多少？

**解析**：
- 客户端发送的第一个数据报文段：SEQ=**x+2**（SYN消耗x+1，第三次握手ACK=x+1不消耗但占用x+1...）

等等，重新分析：
- ① SYN：SEQ=x（消耗x，下一个可用x+1）
- ② SYN+ACK：ACK=x+1
- ③ ACK：SEQ=x+1（不消耗序号），ACK=y+1
- 数据报文段：SEQ=x+1... 

不对，第三次握手ACK报文段的SEQ=x+1但这是ACK报文段（纯ACK不消耗序号）。所以数据报文段的SEQ应该还是**x+1**。

再仔细想：①SYN消耗了序号x，③ACK报文段的SEQ=x+1但不消耗序号。所以第一个数据报文段SEQ=**x+1**。

但很多教材说数据从x+2开始...这是因为有的把第三次握手也视为消耗了1个序号。

**408标准答案**：第三次握手ACK不消耗序号。所以第一个数据报文段的SEQ = **x+1**，ACK = **y+1**。

不过有些题目可能有不同约定，做题时注意题目的约定。

**题目**：TCP连接中，甲发送了序号为201、长度为200的报文段。乙收到后发送的确认号是多少？

**解析**：ACK = 201 + 200 - 1 + 1 = **401**

## 直观理解

- **做题口诀**："ACK = 收到的最后一个字节序号 + 1"

## 协议关联

- **与滑动窗口**：确认号推动窗口右移
- **408常见陷阱**：SYN消耗1序号、第三次握手ACK是否消耗序号
