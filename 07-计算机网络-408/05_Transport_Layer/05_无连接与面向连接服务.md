# 05_无连接与面向连接服务

## 核心概念

- **面向连接服务（Connection-Oriented）**：通信前需建立连接，通信中维护连接状态，通信后释放连接。TCP是典型的面向连接协议
- **无连接服务（Connectionless）**：无需事先建立连接，每个数据报独立发送和路由。UDP是典型的无连接协议
- **可靠传输（Reliable）**：保证数据无差错、不丢失、不重复、按序到达
- **不可靠传输（Unreliable）**：尽最大努力交付，不保证上述任何特性

### 两种服务对比

| 特征 | 面向连接（TCP） | 无连接（UDP） |
|------|----------------|--------------|
| 连接建立 | 三次握手 | 无 |
| 连接状态 | 维护 | 不维护 |
| 数据单元 | 字节流（报文段） | 数据报 |
| 传输保证 | 可靠 | 不可靠 |
| 按序到达 | 保证 | 不保证 |
| 流量控制 | 有 | 无 |
| 拥塞控制 | 有 | 无 |
| 传输延迟 | 较高 | 低 |
| 开销 | 大 | 小 |

## 原理分析

### 面向连接服务的工作流程

```
建立连接 → 数据传输 → 释放连接
  (三次握手)  (可靠传输)  (四次挥手)
```

1. **建立连接**：协商参数（如初始序号、窗口大小），分配资源（缓冲区、变量）
2. **数据传输**：使用序号、确认、重传等机制保证可靠
3. **释放连接**：归还资源，通知对方结束通信

### 无连接服务的工作流程

```
直接发送 → 可能丢失/乱序 → 接收或丢失
```

- 每个数据报独立处理，无前后关联
- 不需要维护任何连接状态
- 延迟低，开销小

### 可靠传输的实现机制（TCP）

1. **序号**：对每个字节编号，接收方据此排序
2. **确认（ACK）**：接收方告诉发送方哪些数据已正确收到
3. **超时重传**：发送方超时未收到确认则重传
4. **差错检测**：校验和检测数据是否损坏
5. **流量控制**：防止发送方发送过快淹没接收方
6. **拥塞控制**：防止过多数据注入网络导致网络拥塞

## 直观理解

- **打电话 vs 发短信**：TCP像打电话——先拨号建立连接，通话中有确认（"嗯""好的"），结束时说再见挂断；UDP像发短信——直接发，不管对方看没看
- **挂号信 vs 平信**：TCP是挂号信，有签收机制，丢失了会补发；UDP是平信，投进邮箱就完事

## 协议关联

- **与应用层协议的对应**：
  - TCP：HTTP、HTTPS、FTP、SMTP、Telnet
  - UDP：DNS、DHCP、SNMP、TFTP、RIP
- **与网络层的关系**：IP是无连接的。TCP在无连接的IP上构建了面向连接的服务；UDP则直接继承了IP的无连接特性
- **408常见考法**：
  - 判断某应用使用TCP还是UDP
  - 面向连接不等于可靠传输（概念上要区分），但TCP两者兼有
