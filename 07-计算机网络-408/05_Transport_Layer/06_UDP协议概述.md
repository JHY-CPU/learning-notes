# 06_UDP协议概述

## 核心概念

- **UDP（User Datagram Protocol，用户数据报协议）**：RFC 768定义，提供无连接、不可靠的数据报服务
- **UDP特点**：简单、高效、低延迟，只在IP之上增加了**复用/分用**和**差错检测**功能
- **UDP数据单元**：称为**UDP用户数据报（UDP Datagram）**
- **适用场景**：对实时性要求高、对少量丢包不敏感的应用

### UDP与IP的关系

- UDP几乎是IP的"薄包装"，仅增加了端口号和校验和
- UDP不提供：可靠传输、流量控制、拥塞控制、连接管理
- UDP提供：复用/分用、差错检测

## 原理分析

### UDP的工作方式

```
应用层 → UDP首部 + 数据 → IP数据报 → 发送
```

发送方：
1. 应用进程将数据交给UDP
2. UDP加上8字节首部（源端口、目的端口、长度、校验和）
3. 交给IP层发送

接收方：
1. IP层将数据交给UDP
2. UDP检查目的端口号
3. 通过端口号将数据交给对应的应用进程
4. 若校验和出错，直接丢弃（不通知发送方）

### 使用UDP的典型应用

| 应用 | 原因 |
|------|------|
| DNS | 查询简单、响应快、用TCP会太慢 |
| DHCP | 初始没有IP，无法建立TCP连接 |
| SNMP | 简单网络管理，偶尔丢包可接受 |
| TFTP | 简单文件传输，自行实现可靠性 |
| VoIP/视频会议 | 实时性要求高，旧数据无意义 |
| 在线游戏 | 低延迟要求 |

## 直观理解

- **明信片类比**：UDP就像寄明信片——写上地址直接投递，没有签收机制，可能会丢。但优点是快、简单、便宜
- **广播类比**：UDP像在广场上喊话，谁听到算谁的，不确认不重传

## 协议关联

- **与TCP的对比**：TCP提供全部可靠性机制，UDP几乎不提供。选择依据是应用需求
- **与应用层的关系**：如果应用需要可靠性但选择使用UDP（如DNS区域传送、TFTP、QUIC），则需要在应用层自行实现可靠机制
- **与网络层的关系**：UDP直接使用IP服务，增加的功能极少
- **408常见考法**：
  - 判断某应用为什么选择UDP
  - UDP不提供可靠传输、不保证按序到达
  - UDP支持一对一、一对多、多对一、多对多通信（TCP只支持一对一）
