# 77_TCP序号确认号综合

## 核心概念

- **序号（SEQ）**：本报文段数据部分第一个字节的编号
- **确认号（ACK）**：期望收到的下一个字节的序号
- **SYN和FIN各消耗1个序号**
- **ISN是随机的，不是从0开始**

### 序号确认号规则总结

| 规则 | 说明 |
|------|------|
| SEQ = 数据第一个字节的序号 | 报文段序号字段的含义 |
| ACK = 已收到的最后一个字节序号 + 1 | 确认号字段的含义 |
| SYN消耗1个序号 | SYN报文段占用1个序号 |
| FIN消耗1个序号 | FIN报文段占用1个序号 |
| 纯ACK不消耗序号 | 仅确认的报文段不占序号 |
| 数据消耗 数据长度 个序号 | 每个字节1个序号 |

## 原理分析

### 计算示例

**场景**：ISN_C=1000, ISN_S=5000

**三次握手**：
| 步骤 | 方向 | SEQ | ACK |
|------|------|-----|-----|
| ① SYN | C→S | 1000 | 无效 |
| ② SYN+ACK | S→C | 5000 | 1001 |
| ③ ACK | C→S | 1001 | 5001 |

**数据传输**（客户端发送300字节，服务器发送500字节）：
| 步骤 | 方向 | SEQ | ACK | 数据 |
|------|------|-----|-----|------|
| C→S | 数据 | 1001 | 5001 | 300字节 |
| S→C | ACK | 5001 | 1301 | 无 |
| S→C | 数据 | 5001 | 1301 | 500字节 |
| C→S | ACK | 1301 | 5501 | 无 |

**四次挥手**（客户端发起关闭）：
| 步骤 | 方向 | SEQ | ACK |
|------|------|-----|-----|
| ① FIN | C→S | 1301 | 5501 |
| ② ACK | S→C | 5501 | 1302 |
| ③ FIN | S→C | 5501 | 1302 |
| ④ ACK | C→S | 1302 | 5502 |

## 直观理解

- **SEQ像页码**：每一页（字节）都有编号
- **ACK像回执**："我已收到第N页以前的所有内容，请从第N+1页开始发"

## 协议关联

- **与滑动窗口**：SEQ和ACK定义了窗口边界
- **与可靠传输**：确认号是累积确认的基础
- **408常考**：给定ISN和数据量计算SEQ和ACK
