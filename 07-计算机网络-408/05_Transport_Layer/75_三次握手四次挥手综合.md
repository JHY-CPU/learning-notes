# 75_三次握手四次挥手综合

## 核心概念

- **三次握手 + 四次挥手 = TCP连接管理的全部内容**
- **三次握手**：SYN → SYN+ACK → ACK
- **四次挥手**：FIN → ACK → FIN → ACK
- **所有报文段的标志位和序号变化必须牢记**

### 完整TCP连接生命周期

```
         客户端                          服务器
           |                               |
    CLOSED |                          LISTEN|
           |  ① SYN=1, SEQ=x              |
           |  ──────────────────────────→  |
   SYN_SENT|                          SYN_RCVD|
           |  ② SYN=1,ACK=1,SEQ=y,ACK=x+1|
           |  ←──────────────────────────  |
 ESTABLISHED|                         ESTABLISHED|
           |  ③ ACK=1, SEQ=x+1, ACK=y+1 |
           |  ──────────────────────────→  |
           |                               |
           |     ←双向数据传输→             |
           |                               |
 FIN_WAIT_1|  ④ FIN=1, SEQ=u              |
           |  ──────────────────────────→  |
           |                         CLOSE_WAIT|
 FIN_WAIT_2|  ⑤ ACK=1, ACK=u+1           |
           |  ←──────────────────────────  |
           |                               |
   TIME_WAIT|  ⑥ FIN=1,ACK=1,SEQ=w,ACK=u+1|
           |  ←──────────────────────────  |
           |                         LAST_ACK|
           |  ⑦ ACK=1, SEQ=u+1, ACK=w+1  |
           |  ──────────────────────────→  |
           |                          CLOSED|
    CLOSED |  (2MSL后)                     |
```

## 原理分析

### 序号消耗总结

| 报文段 | 消耗序号 |
|--------|---------|
| SYN | 1 |
| FIN | 1 |
| 数据 | 数据长度 |
| 纯ACK | 0 |

### 状态转换总结

**客户端**：CLOSED → SYN_SENT → ESTABLISHED → FIN_WAIT_1 → FIN_WAIT_2 → TIME_WAIT → CLOSED

**服务器**：CLOSED → LISTEN → SYN_RCVD → ESTABLISHED → CLOSE_WAIT → LAST_ACK → CLOSED

## 直观理解

- **完整流程像一次完整的会议**：预约（握手）→ 开会（数据传输）→ 告别（挥手）

## 协议关联

- **与状态转换图**：三次握手和四次挥手是状态转换图的主体
- **与拥塞控制**：连接建立后开始慢开始
- **408必考**：给定事件判断状态、计算SEQ/ACK、分析异常情况
