# 36_TCP流量控制_概念_真题

## 核心概念

- **流量控制通过接收窗口（rwnd）实现**
- **发送窗口 = min(rwnd, cwnd)**
- **零窗口探测解决死锁问题**
- **流量控制是端到端的机制**

### 408真题精选

**题目**：主机甲向主机乙发送数据，TCP连接的接收窗口rwnd=3000字节，拥塞窗口cwnd=2000字节，最大报文段长度MSS=1000字节。甲最多还能向乙发送多少字节？

**解析**：
$$\text{发送窗口} = \min(\text{rwnd}, \text{cwnd}) = \min(3000, 2000) = 2000\text{字节}$$
甲最多能发送**2000字节**。

**题目**：TCP接收方通告rwnd=0后，发送方完全停止发送数据。这种做法正确吗？

**解析**：不完全正确。虽然发送方应该停止发送数据，但需要启动**持续计时器**，超时后发送**零窗口探测报文段**（1字节数据），以防止双方死锁。如果完全停止且不探测，当接收方有空间但通知报文段丢失时，双方将永远等待。

**题目**：主机甲和乙的TCP连接的MSS=1KB，RTT=10ms，接收窗口rwnd=10KB。甲向乙持续发送数据，最大吞吐量是多少？

**解析**：
$$\text{最大吞吐量} = \frac{\text{rwnd}}{\text{RTT}} = \frac{10\text{KB}}{10\text{ms}} = \frac{10 \times 1024 \text{B}}{0.01\text{s}} \approx 1\text{MB/s}$$

**题目**：TCP流量控制和拥塞控制有什么区别？

**解析**：
| 特征 | 流量控制 | 拥塞控制 |
|------|---------|---------|
| 目的 | 防止淹没接收方 | 防止网络拥塞 |
| 控制者 | 接收方 | 发送方（根据网络反馈） |
| 机制 | rwnd | cwnd |
| 范围 | 端到端 | 全局 |
| 变量 | 接收方通告 | 发送方自行调整 |

## 直观理解

- **rwnd像容量报告**：接收方定期告诉发送方"我的缓冲区还能装多少"

## 协议关联

- **与滑动窗口**：rwnd限制窗口大小
- **与拥塞控制**：实际窗口取rwnd和cwnd的最小值
- **408常见陷阱**：
  - "流量控制由发送方实现"——错误，由接收方控制
  - "rwnd=0时发送方完全停止"——错误，需要零窗口探测
