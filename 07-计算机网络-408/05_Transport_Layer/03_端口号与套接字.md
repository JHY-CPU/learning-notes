# 03_端口号与套接字

## 核心概念

- **端口号（Port Number）**：16位整数（0~65535），用于标识传输层上不同应用进程的逻辑地址
- **套接字（Socket）**：Socket = (IP地址, 端口号)，是传输层与应用层之间的接口，也称为**网络编程接口**
- **端口号分类**：

| 类别 | 范围 | 说明 |
|------|------|------|
| 熟知端口号（Well-Known） | 0~1023 | 分配给重要服务，如HTTP(80)、FTP(21)、SMTP(25)、DNS(53) |
| 注册端口号（Registered） | 1024~49151 | 分配给用户进程或应用程序 |
| 动态/临时端口号（Dynamic） | 49152~65535 | 客户端程序临时分配使用 |

### 常见熟知端口号（408必记）

| 服务 | 协议 | 端口号 |
|------|------|--------|
| FTP数据 | TCP | 20 |
| FTP控制 | TCP | 21 |
| SSH | TCP | 22 |
| Telnet | TCP | 23 |
| SMTP | TCP | 25 |
| DNS | UDP/TCP | 53 |
| HTTP | TCP | 80 |
| HTTPS | TCP | 443 |

## 原理分析

### 套接字的作用

- **服务器端套接字**：绑定在熟知端口号上，被动等待客户端连接
- **客户端套接字**：操作系统临时分配一个动态端口号
- **TCP连接标识**：一个TCP连接由**四元组**唯一标识——（源IP, 源端口, 目的IP, 目的端口）

### 端口号的复用原理

- 多个客户端可以同时连接到服务器的同一个熟知端口（如80）
- 因为每个连接的四元组不同，服务器可以同时维护多个连接
- 同一个客户端的不同进程使用不同的临时端口号

## 直观理解

- **学校类比**：IP地址像学校地址（定位到哪台主机），端口号像教室编号（定位到哪个进程）。套接字就是"某某学校某某教室"，完整定位到一个服务
- **超市收银台类比**：熟知端口号就像超市固定的收银台（1号收银台=结账），动态端口号像临时开放的快速通道

## 协议关联

- **与应用层的关系**：应用层协议规定了使用哪个端口号。DNS使用53，HTTP使用80
- **与网络层的关系**：IP地址+端口号构成完整的网络通信地址
- **408常见陷阱**：
  - 端口号是传输层概念，不是网络层概念
  - 套接字=(IP, 端口号)是408标准定义
  - TCP连接由四元组唯一确定，不只是端口号对
  - DNS主要使用UDP 53端口，但区域传送使用TCP 53端口
