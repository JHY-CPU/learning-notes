# 02_传输层功能与服务

## 核心概念

- **传输层核心功能**：复用与分用、差错检测、可靠传输（TCP）、流量控制、拥塞控制
- **服务原语**：传输层向应用层提供服务原语，包括 LISTEN、CONNECT、SEND、RECEIVE、DISCONNECT
- **面向连接服务特征**：建立连接、有序交付、无差错、无丢失、流量控制
- **无连接服务特征**：无需建立连接、独立路由、可能丢失乱序

### 传输层服务类型对比

| 特征 | TCP | UDP |
|------|-----|-----|
| 连接建立 | 需要（三次握手） | 不需要 |
| 可靠性 | 可靠（确认重传） | 不可靠 |
| 有序性 | 保证按序到达 | 不保证 |
| 流量控制 | 有（滑动窗口） | 无 |
| 拥塞控制 | 有 | 无 |
| 字节流/数据报 | 字节流 | 数据报 |
| 开销 | 大（20字节首部） | 小（8字节首部） |

## 原理分析

### 复用与分用机制

**分用（Demultiplexing）**的实现：

接收端传输层收到IP数据报后：
1. 检查传输层首部中的**目的端口号**
2. 根据端口号找到对应的**套接字（socket）**
3. 将数据通过套接字传递给相应的应用进程

**复用（Multiplexing）**的实现：

发送端多个应用进程的数据：
1. 各自通过不同的套接字进入传输层
2. 传输层加上端口号等首部信息
3. 交给网络层统一发送

### 传输层与网络层的服务关系

```
应用进程 ←——套接字——→ 传输层 ←——端口号复用/分用——→ 网络层
```

- 网络层提供**尽力而为**的主机到主机通信
- 传输层将网络层的不可靠服务转化为（TCP）可靠服务
- 传输层直接为应用进程服务，屏蔽底层网络细节

## 直观理解

- **火车站类比**：网络层像火车站，负责把人从一个城市运到另一个城市（主机到主机）。传输像出站口的检票员，根据车票上的座位号（端口号）把乘客引导到正确的出口（应用进程）
- **邮箱类比**：UDP像投递平信——投进邮箱就完事了，不管对方收没收到；TCP像挂号信——需要签收确认，丢失了还会补发

## 协议关联

- **与网络层IP协议的关系**：IP协议提供不可靠、无连接的数据报服务，传输层在其上构建。TCP增加了可靠性机制，UDP几乎不增加额外功能
- **与数据链路层的关系**：数据链路层的可靠传输是点到点的（相邻节点），传输层的可靠传输是端到端的（源主机到目的主机的进程）
- **408常见考法**：
  - 区分"复用"的层次：传输层复用（端口号）、网络层复用（IP地址）、数据链路层复用（MAC地址）
  - 判断哪些功能属于传输层：流量控制和拥塞控制是传输层功能
