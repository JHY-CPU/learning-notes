# 09_UDP校验_概念_计算

## 核心概念

- **校验和本质**：差错检测码，检测数据在传输中是否被修改
- **反码求和（One's Complement Sum）**：将16位字相加，若有进位则回卷（加到最低位）
- **伪首部参与计算**：确保IP层信息正确（源/目的IP、协议号）
- **UDP校验和字段位置**：UDP首部第7~8字节

### 校验和计算涉及的所有数据

```
|← 伪首部(12字节) →|← UDP首部(8字节) →|← UDP数据 →|
```

## 原理分析

### 详细计算步骤

**步骤1：构造计算对象**
- 伪首部（12字节）+ UDP首部（8字节，校验和字段临时填0）+ UDP数据
- 如果总长度非偶数，末尾补一个全0字节

**步骤2：反码二进制求和**
```
规则：两个16位数相加
a = 0110 0110 0110 0000
b = 0101 0101 0101 0101
a+b = 1011 1011 1011 0101
如果有进位（最高位产生进位）→ 将进位加到最低位
```

**步骤3：取反码得到校验和**
```
求和结果：1100 1010 1100 0101
取反：    0011 0101 0011 1010 = 校验和
```

**步骤4：填入校验和字段**

### 手算示例

假设有伪首部信息：源IP=192.168.1.1，目的IP=192.168.1.2，协议=17，UDP长度=12

UDP首部：源端口=1024，目的端口=80，长度=12，校验和=0（待算）

UDP数据：4字节 `0x00 0x01 0x02 0x03`

将所有内容按16位分组后反码求和，再取反即可。

### 接收方验证

接收方将所有16位字（校验和字段保留原值）进行反码求和。若结果为 `1111111111111111`（全1），则无差错。

证明：设校验和为 $C$，数据和为 $S$，则 $C = \sim S$（取反），所以 $S + C = S + \sim S = \text{全1}$。

## 直观理解

- **计算器类比**：就像把所有数字放进一个16位的计算器，溢出了就加回来，最后取反。接收方把结果和校验和再加一遍，应该是全1才对
- **记账类比**：把所有金额加起来算个总数（校验和），接收方重新算一遍，对上了就说明没算错

## 协议关联

- **与TCP校验和**：计算方法完全相同（反码求和+伪首部）
- **与IP校验和**：IP校验和只校验IP首部，不使用伪首部；UDP/TCP校验和使用伪首部跨层校验
- **408常见考法**：手算校验和（给定数据，按步骤计算）
