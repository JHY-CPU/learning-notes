# 44_TCP拥塞控制综合

## 核心概念

- **TCP拥塞控制四算法**：慢开始、拥塞避免、快重传、快恢复
- **AIMD（Additive Increase Multiplicative Decrease）**：加法增大、乘法减小
- **发送窗口 = min(rwnd, cwnd)**
- **cwnd以MSS为单位**

### 拥塞控制完整流程图

```
初始: cwnd=1, ssthresh=初始值
        │
   ┌────┴────┐
   │ 慢开始   │ cwnd < ssthresh
   │ 指数增长 │ 每RTT翻倍
   └────┬────┘
        │ cwnd ≥ ssthresh
   ┌────┴────┐
   │ 拥塞避免 │ 
   │ 线性增长 │ 每RTT +1 MSS
   └────┬────┘
        │
   ┌────┴─────────────────┐
   │                      │
超时                    3个冗余ACK
   │                      │
ssthresh=cwnd/2       ssthresh=cwnd/2
cwnd=1                cwnd=ssthresh+3
→ 慢开始              → 拥塞避免
```

## 原理分析

### AIMD的收敛性

- **加法增大（AI）**：每RTT cwnd += 1
- **乘法减小（MD）**：丢包时 cwnd /= 2
- 所有TCP连接使用相同的AI和MD规则，最终会收敛到公平共享带宽

### 全局拥塞控制视角

- 所有TCP连接独立执行拥塞控制
- 丢包是网络拥塞的信号
- 所有连接同时检测到丢包，同时减半cwnd
- 网络拥塞缓解，所有连接再慢慢增大

## 直观理解

- **拥塞控制像交通管理**：每辆车（TCP连接）都遵守相同规则——慢慢加速，看到堵车就减速。所有车都这么做，交通就自动流畅了
- **锯齿形图案**：cwnd变化图像锯齿——上升（增长）→ 丢包骤降 → 再上升

## 协议关联

- **与流量控制**：实际窗口 = min(rwnd, cwnd)，两者共同限制发送速率
- **与路由器AQM**：路由器的主动队列管理（如RED）可以提前通知端系统拥塞
- **与TCP版本**：Tahoe（仅慢开始）、Reno（加快恢复）、NewReno（改进快恢复）、CUBIC（Linux默认）
- **408综合考法**：画cwnd变化图、计算特定轮次的cwnd/ssthresh
