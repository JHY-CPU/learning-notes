# 39_拥塞避免算法

## 核心概念

- **拥塞避免（Congestion Avoidance）**：当cwnd ≥ ssthresh时，从指数增长转为**线性增长**
- **增长方式**：每收到一个ACK，cwnd增加 $\frac{\text{MSS}^2}{\text{cwnd}}$
- **效果**：每个RTT，cwnd增加**1个MSS**（加法增大，AI）
- **与慢开始合称为AIMD（Additive Increase Multiplicative Decrease）**

### 拥塞避免的增长规律

| RTT | cwnd（MSS） | 说明 |
|-----|-------------|------|
| n | s | 进入拥塞避免 |
| n+1 | s+1 | 每RTT +1 |
| n+2 | s+2 | |
| n+3 | s+3 | |
| ... | ... | |

**规律**：每个RTT，cwnd增加1 MSS（线性增长）

## 原理分析

### 拥塞避免的实现

每收到一个ACK：
$$\text{cwnd} += \frac{\text{MSS}^2}{\text{cwnd}}$$

推导：一个RTT内收到cwnd个ACK，所以cwnd总共增加：
$$\text{cwnd} \times \frac{\text{MSS}^2}{\text{cwnd}} = \text{MSS}^2$$

但由于cwnd的单位是MSS，所以实际每RTT增加约1 MSS。

### 丢包时的处理

**情况1：超时（严重拥塞）**
$$\text{ssthresh} = \frac{\text{cwnd}}{2}$$
$$\text{cwnd} = 1 \text{ MSS}$$
→ 重新进入**慢开始**

**情况2：3个冗余ACK（轻微拥塞）**
$$\text{ssthresh} = \frac{\text{cwnd}}{2}$$
$$\text{cwnd} = \text{ssthresh} + 3 \text{ MSS}$$
→ 进入**拥塞避免**（快恢复）

### AIMD总结

- **AI（Additive Increase）**：拥塞避免阶段，每RTT +1 MSS
- **MD（Multiplicative Decrease）**：丢包时，cwnd减半

## 直观理解

- **拥塞避免像慢慢加油门**：不像慢开始那样猛踩油门（指数增长），而是小心翼翼地每轮多加一点（线性增长）。一旦出了问题（丢包），立刻减半
- **AIMD像博弈**：所有人都线性增长、遇丢包减半，最终趋向公平共享带宽

## 协议关联

- **与慢开始的关系**：慢开始是快速"探测"可用带宽，拥塞避免是精细"逼近"可用带宽
- **与快恢复的关系**：3个冗余ACK触发快恢复，直接跳到拥塞避免（跳过慢开始）
