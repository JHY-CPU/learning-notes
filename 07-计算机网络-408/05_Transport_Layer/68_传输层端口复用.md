# 68_传输层端口复用

## 核心概念

- **复用（Multiplexing）**：多个应用进程的数据通过同一个传输层协议发送
- **分用（Demultiplexing）**：传输层根据端口号将数据交给正确的应用进程
- **端口号是传输层复用/分用的依据**
- **四元组唯一标识一个TCP连接**：(源IP, 源端口, 目的IP, 目的端口)

### 各层的复用/分用

| 层次 | 标识 | 复用/分用依据 |
|------|------|--------------|
| 数据链路层 | MAC地址 | 根据MAC地址分发帧 |
| 网络层 | IP地址 | 根据IP地址路由数据报 |
| 传输层 | 端口号 | 根据端口号分发数据 |
| 应用层 | 进程/线程 | 应用自行处理 |

## 原理分析

### 端口复用的工作过程

```
进程A(端口50001) ──┐
进程B(端口50002) ──┼──→ 传输层(TCP) ──→ 网络层 ──→ 目的主机
进程C(端口50003) ──┘
```

传输层通过源端口号将不同进程的数据区分开，交给IP层统一发送。

### 分用的工作过程

```
网络层 ──→ 传输层 ──→ 根据目的端口号 ──→ 对应的套接字 ──→ 应用进程
```

接收方传输层检查目的端口号，找到对应的套接字，将数据传递给相应的进程。

### TCP vs UDP的分用

**UDP**：根据(目的IP, 目的端口)找到对应的套接字。每个UDP套接字用(目的IP, 目的端口)标识。

**TCP**：根据四元组(源IP, 源端口, 目的IP, 目的端口)找到对应的套接字。服务器可为每个连接创建不同的套接字。

## 直观理解

- **端口复用像邮箱**：很多信件通过同一个邮箱入口（传输层），但每个信封上写着不同的房间号（端口号），邮递员（传输层）根据房间号送到正确的住户

## 协议关联

- **与应用层**：应用层协议定义了使用的端口号
- **与网络层**：网络层的IP地址+传输层的端口号=完整的通信地址
- **408常见考法**：
  - 端口号是传输层复用/分用的依据
  - TCP连接由四元组唯一标识
  - 复用/分用的概念
