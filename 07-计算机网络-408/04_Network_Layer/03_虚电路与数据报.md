# 03_虚电路与数据报

## 核心概念

- **虚电路（Virtual Circuit, VC）**：
  - 通信前建立逻辑连接
  - 所有分组沿同一路径传输
  - 每个分组携带**虚电路号**（而非完整目的地址）
  - 代表：X.25、帧中继、ATM

- **数据报（Datagram）**：
  - 无需预先建立连接
  - 每个分组独立路由
  - 每个分组携带**完整的目的地址**
  - 代表：IP网络（Internet）

- **虚电路号（VCI, Virtual Circuit Identifier）**：
  - 只在每段链路上有意义（局部有效）
  - 不同链路段的VCI可以不同
  - 路由器根据VCI查表转发

- **408重点**：
  - 虚电路是**网络层**的连接，不是传输层的连接
  - 虚电路与TCP连接的区别（层次不同！）
  - 数据报网络中每个分组可以走不同路径

## 原理分析

### 虚电路建立过程

```
步骤1: 源主机发送"呼叫请求"分组
步骤2: 路由器R1收到，分配VC号，查表确定下一跳
步骤3: 沿路径逐跳传递，每个路由器分配本地VC号
步骤4: 目的主机收到"呼叫请求"，回复"呼叫接受"
步骤5: "呼叫接受"沿原路返回，虚电路建立完成
步骤6: 数据传输，每个分组携带VC号
步骤7: 通信结束后发送"释放请求"，拆除虚电路
```

### 虚电路表（每个路由器维护）

| 入接口 | 入VC号 | 出接口 | 出VC号 |
|--------|--------|--------|--------|
| 1 | 5 | 3 | 11 |
| 3 | 7 | 2 | 4 |

- VC号在每段链路上独立分配
- 转发时用（入接口, 入VC号）查表，得到（出接口, 出VC号）

### 数据报转发过程

```
每个分组到达路由器：
1. 提取目的IP地址
2. 用目的地址查路由表（最长前缀匹配）
3. 确定输出端口
4. 转发分组
```

## 直观理解

**类比**：

- **虚电路** = 电话系统
  - 先拨号建立连接（呼叫建立）
  - 通话期间线路固定
  - 挂断后释放线路（呼叫释放）
  - 后续通话可能走不同线路

- **数据报** = 邮政系统
  - 每封信独立寄送
  - 邮局根据地址逐站转发
  - 不同信件可能走不同路线
  - 不用提前和邮局预约

## 协议关联

### 与其他协议的关系

- **虚电路与TCP连接**：
  - 虚电路在网络层，TCP连接在传输层
  - 虚电路对端系统透明，TCP连接是端到端的
  - 一条TCP连接的分组在数据报网络中可能走不同路径

- **数据报与IP**：
  - IP协议天然支持数据报模式
  - 每个IP数据报独立路由

### 考试陷阱
- "虚电路类似于TCP连接" → **错误**！虚电路在网络层，TCP连接在传输层
- "虚电路保证可靠传输" → **错误**！虚电路只保证按序到达和同一路径，不保证可靠（可靠性仍需传输层）
