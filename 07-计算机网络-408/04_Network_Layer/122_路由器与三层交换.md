# 路由器与三层交换

## 核心概念

### 路由器（Router）
- 工作在网络层（第三层），基于IP地址进行转发
- 主要功能：路由选择、分组转发、网络互联、隔离广播域
- 路由器的每个接口连接不同的广播域（不同子网）

### 三层交换机（Layer 3 Switch）
- 融合了二层交换和三层路由功能
- 基于硬件ASIC实现高速路由转发
- 适用于局域网内部的快速路由

### 路由器 vs 三层交换机
| 特性 | 路由器 | 三层交换机 |
|------|--------|-----------|
| 转发方式 | 基于软件 | 基于硬件ASIC |
| 接口数量 | 少（几个到几十个） | 多（几十到几百个） |
| 功能 | 丰富（NAT、VPN、ACL等） | 相对简单 |
| 速度 | 相对慢 | 非常快 |
| 适用场景 | WAN互联、复杂路由 | LAN内部快速路由 |
| 价格 | 较高 | 适中 |

### 路由器的基本结构
- **控制平面**：运行路由协议，维护路由表
- **数据平面**：根据转发表进行数据报转发
- **接口**：连接不同网络的物理端口
- **转发引擎**：执行查表和转发操作

## 原理分析

### 路由器转发过程
1. 收到帧，检查目的MAC是否是自己（否则丢弃）
2. 去掉帧头，提取IP数据报
3. 检查IP首部TTL，减1，重新计算首部校验和
4. 查路由表（最长前缀匹配），确定下一跳和出接口
5. 查ARP表，获取下一跳的MAC地址
6. 重新封装帧（新的源MAC和目的MAC），发送

### 三层交换的"一次路由，多次交换"
1. 第一个数据报走完整路由过程（CPU处理，建立ARP表）
2. 后续数据报通过硬件ASIC直接查转发表转发（高速）
3. 这就是"一次路由，多次交换"的原理

## 直观理解
- 路由器像"邮政分拣中心"——每个包裹都要检查地址、重新打包
- 三层交换机像"高速公路ETC"——第一次过要登记，后续直接快速通过

## 协议关联
- 路由器运行路由协议（OSPF、RIP、BGP）维护路由表
- 三层交换机通常只支持简单的路由协议
- 路由器的每个接口有独立的IP地址和MAC地址
- 路由器隔离广播域，交换机隔离冲突域
