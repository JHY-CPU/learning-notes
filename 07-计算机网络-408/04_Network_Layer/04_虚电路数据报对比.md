# 04_虚电路数据报对比

## 核心概念

- **两种网络层服务模式的系统性对比**

- **关键区别维度**：
  1. 是否需要建立连接
  2. 分组是否携带完整地址
  3. 分组是否走同一路径
  4. 路由器是否维护状态信息
  5. 对网络故障的适应性

- **408考试要求**：能清晰区分虚电路和数据报的特点，选择题常考

## 原理分析

### 详细对比表

| 对比维度 | 虚电路 | 数据报 |
|---------|--------|--------|
| 连接建立 | 需要（呼叫建立阶段） | 不需要 |
| 地址信息 | 每个分组携带虚电路号（短） | 每个分组携带完整目的地址（长） |
| 路由选择 | 建立时确定，所有分组走同一路 | 每个分组独立选择路由 |
| 分组顺序 | 保证按序到达 | 不保证按序到达 |
| 路由器状态 | 维护虚电路表（有状态） | 不维护连接状态（无状态） |
| 故障影响 | VC经过的节点故障，所有VC中断 | 节点故障可绕路，影响较小 |
| 分组开销 | 头部开销小（VC号短） | 头部开销大（完整地址长） |
| 适用场景 | 面向连接的应用（如语音） | 突发性强的应用（如数据传输） |
| 代表网络 | X.25、ATM、帧中继 | Internet（IP） |

### 虚电路的子类型

- **永久虚电路（PVC, Permanent Virtual Circuit）**：运营商预先配置，长期存在
- **交换虚电路（SVC, Switched Virtual Circuit）**：按需建立，用完释放

### Internet选择数据报的原因

1. **鲁棒性**：网络中某节点故障，分组可重新路由
2. **可扩展性**：路由器无需维护每条连接的状态
3. **适应突发流量**：数据通信具有突发性，数据报更灵活
4. **端到端原则**：将智能放在端系统，网络保持简单

## 直观理解

**生活类比**：

- **虚电路** = 预定的旅行团路线
  - 出发前确定好所有行程
  - 全程走同一条路线
  - 但一旦某景点关闭，整个行程受影响

- **数据报** = 自由行
  - 不用提前预约
  - 随时可以根据情况调整路线
  - 某条路不通就换一条

**记忆口诀**：
- "虚电路：先建后传，同路同序，有状态"
- "数据报：即传即走，各走各路，无状态"

## 协议关联

### 与TCP/IP体系的对应

- Internet采用数据报服务
- 网络层的不可靠由传输层TCP来弥补
- 这种设计使Internet具有极强的鲁棒性和适应性

### 考试常见组合
- 虚电路 vs 数据报 对比选择题
- "以下哪个不是数据报网络的特点" —— 注意"保证按序到达"是虚电路的特点
- 判断题：虚电路建立在网络层（对），虚电路等同于TCP连接（错）
