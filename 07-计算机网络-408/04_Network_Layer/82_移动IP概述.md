# 移动IP概述

## 核心概念
- 移动IP允许主机在不同网络之间移动时保持IP地址不变，实现不间断通信
- 核心组件：
  - **家乡代理（Home Agent, HA）**：移动节点家乡网络中的路由器
  - **外地代理（Foreign Agent, FA）**：移动节点当前所在网络中的路由器
  - **转交地址（Care-of Address, CoA）**：移动节点在外地网络获得的临时地址
  - **家乡地址（Home Address）**：移动节点的永久IP地址

### 工作流程
1. 移动节点离开家乡网络，到达外地网络
2. 移动节点获得转交地址（CoA）
3. 移动节点向家乡代理注册转交地址
4. 家乡代理截获发往移动节点的数据报
5. 家乡代理通过隧道将数据报转发到转交地址
6. 移动节点直接发送响应数据（三角路由）

### 移动IPv4 vs 移动IPv6
| 特性 | 移动IPv4 | 移动IPv6 |
|------|----------|----------|
| 外地代理 | 需要 | 不需要（使用CoA） |
| 三角路由 | 必须 | 支持路由优化 |
| IPsec | 可选 | 内置支持 |

## 原理分析

### 三角路由问题
- 数据报路径：通信对端 → 家乡代理 → 转交地址 → 移动节点
- 响应路径：移动节点 → 通信对端（直接发送）
- 移动IPv6支持路由优化，可直接通信避免三角路由

## 直观理解
- 移动IP像"快递改址"——搬家后原地址保留（家乡代理代收），快递员把包裹转寄到新地址
- 三角路由像"中转站"——信先寄到老家邮局，再转发到你的新住处

## 协议关联
- 移动IP工作在网络层，依赖IP隧道技术
- 家乡代理使用IP-in-IP隧道封装数据报
- 移动IPv6依赖ICMPv6的NDP实现邻居发现
