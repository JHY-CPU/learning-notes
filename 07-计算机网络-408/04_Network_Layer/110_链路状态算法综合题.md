# 链路状态算法综合题

## 核心概念
- 链路状态算法：每个路由器构建完整的网络拓扑图
- Dijkstra算法：从源点出发，逐步扩展计算最短路径树
- OSPF是典型的链路状态协议

### Dijkstra算法步骤
1. 初始化：S={源点}，dist[源点]=0，其他dist=∞
2. 在V-S中选择dist最小的节点u
3. 将u加入S
4. 更新u的邻居v：dist[v] = min(dist[v], dist[u] + cost(u,v))
5. 重复2-4直到所有节点在S中

### 算法复杂度
- 时间复杂度：$O(V^2)$（朴素实现）或 $O((V+E)\log V)$（优先队列实现）
- V = 节点数，E = 链路数

## 原理分析

### 综合计算题
网络拓扑（节点间数字为链路代价）：
```
    2
 A --- B
 |     |
 3     1
 |     |
 C --- D
   4
```

从A出发的Dijkstra计算过程：

| 步骤 | S | dist[A] | dist[B] | dist[C] | dist[D] |
|------|---|---------|---------|---------|---------|
| 初始 | {A} | 0 | ∞ | ∞ | ∞ |
| 1 | {A} | 0 | 2 | 3 | ∞ |
| 2 | {A,B} | 0 | 2 | 3 | 3 |
| 3 | {A,B,D} | 0 | 2 | 3 | 3 |
| 4 | {A,B,D,C} | 0 | 2 | 3 | 3 |

最短路径树：
- A→B：直接，代价2
- A→C：直接，代价3
- A→D：A→B→D，代价3

## 直观理解
- Dijkstra算法像"GPS寻路"——从当前位置开始，每次选择最近的未访问路口
- 链路状态算法像"所有人共享地图"——每人都有完整的网络拓扑，各自独立计算

## 协议关联
- OSPF使用Dijkstra算法计算最短路径
- 链路状态算法避免了距离向量的无穷计数问题
- 每个路由器独立计算，不依赖邻居的路由表
- LSA泛洪保证所有路由器的LSDB一致
