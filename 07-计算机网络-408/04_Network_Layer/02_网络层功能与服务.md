# 02_网络层功能与服务

## 核心概念

- **网络层提供的服务**：
  - **数据报服务（Datagram Service）**：Internet采用，无连接、不可靠
  - **虚电路服务（Virtual Circuit Service）**：ATM、X.25采用，面向连接、较可靠

- **尽最大努力交付（Best-effort Delivery）**的含义：
  - 不保证分组不丢失
  - 不保证分组不重复
  - 不保证分组按序到达
  - 不保证端到端时延
  - 不保证最小带宽

- **网络层核心任务**：
  1. **异构网络互连**：不同物理网络（以太网、WiFi、PPP）通过路由器互连
  2. **路由与转发**：确定路径+逐跳转发
  3. **拥塞控制**：网络层的全局性流量控制

- **408考点**：
  - TCP/IP体系的网络层 = 网际层（Internet Layer）
  - IP协议是TCP/IP网络层的核心协议
  - 网络层向上只提供简单灵活的、无连接的、尽最大努力交付的数据报服务

## 原理分析

### 数据报服务的工作方式

```
源主机H1 → [分组1] → 路由器R1 → R2 → 目的主机H2
源主机H1 → [分组2] → 路由器R1 → R3 → R4 → 目的主机H2
```

- 同一报文的不同分组可以走不同路径
- 每个分组独立进行路由选择
- 路由器不需要维持连接状态

### 虚电路服务的工作方式

```
呼叫建立：H1 → R1 → R2 → H2（建立虚电路VC）
数据传输：所有分组沿VC转发，带虚电路号
呼叫释放：拆除虚电路
```

- 三个阶段：建立 → 数据传送 → 释放
- 所有分组走同一路径，保证按序到达
- 路由器需要维护虚电路表

### Internet选择数据报服务的原因

1. **健壮性**：节点故障时分组可绕路
2. **灵活性**：适应各种应用需求
3. **简单性**：网络层协议简单，复杂功能交给端系统
4. **端到端原则**：网络核心保持简单，智能放在网络边缘

## 直观理解

**类比**：

- **数据报服务** = 寄普通信件
  - 不用先打电话通知对方
  - 每封信独立投递
  - 邮局不保证一定能送到
  - 丢了自己负责（上层TCP来保证可靠）

- **虚电路服务** = 约定好的快递专车
  - 先预约车辆和路线
  - 所有包裹走同一条路
  - 按序到达，更可靠
  - 但一旦车坏了，所有包裹都延误

**记忆口诀**：
- "IP尽力而为不可靠，TCP端到端保证可靠性"
- "网络层傻瓜式服务，传输层聪明补漏洞"

## 协议关联

### 层间分工

| 功能 | 网络层（IP） | 传输层（TCP） |
|------|------------|-------------|
| 可靠传输 | 不保证 | 保证（确认+重传） |
| 按序到达 | 不保证 | 保证（序列号） |
| 流量控制 | 不保证 | 保证（滑动窗口） |
| 拥塞控制 | 部分支持 | 保证（拥塞窗口） |

### 考试常见问题
- **Q：为什么TCP/IP把复杂功能放在端系统？**
  - A：端到端原则（End-to-End Principle），网络核心保持简单高效，复杂功能由端系统实现
- **Q：网络层与数据链路层的区别？**
  - A：数据链路层在相邻节点间传输帧（hop-to-hop）；网络层在源到目的间传输分组（end-to-end路径上逐跳）
