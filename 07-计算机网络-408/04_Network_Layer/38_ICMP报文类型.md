# 38_ICMP报文类型

## 核心概念

- **ICMP报文格式**：

```
| 类型(8位) | 代码(8位) | 校验和(16位) |
|          其他字段（取决于报文类型）         |
|       出错IP数据报首部+前8字节（差错报文）    |
```

- **详细报文类型**：

### 差错报告报文

**1. 目的不可达（Type=3）**
- 代码0：网络不可达
- 代码1：主机不可达
- 代码2：协议不可达
- 代码3：端口不可达
- 代码4：需要分片但DF置位
- 代码5：源路由失败
- 代码6：目的网络未知
- 代码7：目的主机未知

**2. 时间超过（Type=11）**
- 代码0：TTL超时（路由器丢弃TTL=0的分组）
- 代码1：分片重组超时（目的主机未在规定时间内收到所有分片）

**3. 参数问题（Type=12）**
- 代码0：IP首部字段有差错
- 代码2：缺少必要选项

**4. 重定向（Type=5）**
- 代码0：对网络的重定向
- 代码1：对主机的重定向
- 代码2：对服务类型和网络的重定向
- 代码3：对服务类型和主机的重定向

### 询问报文

**5. 回显请求（Type=8）/ 回显应答（Type=0）**
- 用于Ping命令
- 请求和应答的数据部分相同

**6. 时间戳请求（Type=13）/ 时间戳应答（Type=14）**
- 用于时钟同步
- 包含：发起时间戳、接收时间戳、发送时间戳

## 原理分析

### ICMP重定向的工作过程

```
步骤1: 主机H将发往网络N的分组发给默认路由器R1
步骤2: R1发现去网络N更优的路由是通过R2
步骤3: R1将分组转发给R2，同时向H发送ICMP重定向报文
步骤4: H收到重定向报文，更新路由表，以后直接发给R2
```

### ICMP差错报文的限制

- 每个连接只发送**一个**差错报文（防止拥塞）
- 不对ICMP差错报文再产生ICMP差错报文
- 不对分片中非第一个分片产生差错报文
- 不对多播地址产生差错报文
- 不对特殊地址（127, 0.0.0.0等）产生差错报文

## 直观理解

**类型代码速记**：

| 类型 | 名称 | 记忆 |
|------|------|------|
| 0 | 回显应答 | "0号是回答" |
| 3 | 目的不可达 | "3号送达不了" |
| 5 | 重定向 | "5号指方向" |
| 8 | 回显请求 | "8号是提问" |
| 11 | 时间超过 | "11号超时了" |

**口诀**："0回答3不可达5重定向8问11超时"

## 协议关联

### ICMP与网络层协议
- ICMP报文封装在IP数据报中
- ICMP差错报文中的IP首部包含源主机IP，用于回送差错信息

### ICMP与路由协议
- ICMP重定向帮助主机优化路由表
- ICMP路由器发现协议帮助主机找到默认网关

### ICMP与应用层
- Ping命令使用ICMP回显请求/应答
- Traceroute使用ICMP时间超过和目的不可达
- 考试常考Ping和Traceroute的ICMP类型
