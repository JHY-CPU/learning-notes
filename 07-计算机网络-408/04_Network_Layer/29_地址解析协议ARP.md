# 29_地址解析协议ARP

## 核心概念

- **ARP（Address Resolution Protocol）**：地址解析协议
  - 功能：将IP地址解析为MAC地址（硬件地址）
  - 工作在网络层（有些教材归为数据链路层）
  - 只在**同一局域网（同一网段）**内工作

- **ARP的必要性**：
  - 数据链路层封装需要目的MAC地址
  - 已知目的IP，但不知道其MAC地址
  - ARP完成 IP → MAC 的映射

- **ARP缓存（ARP Cache）**：
  - 每台主机维护ARP缓存表
  - 存储 IP-MAC 映射条目
  - 每个条目有生存时间（通常10~20分钟）
  - 先查缓存，有则直接用，无则发ARP请求

- **408考点**：
  - ARP的工作原理（请求/应答）
  - ARP只在同一网段内工作
  - 跨网段通信时ARP的使用场景
  - ARP是网络层协议

## 原理分析

### ARP工作流程

```
主机A要向主机B发送数据（已知B的IP，不知B的MAC）

步骤1: A查看ARP缓存，是否有B的IP-MAC映射？
  有 → 直接使用，结束
  无 → 继续步骤2

步骤2: A广播发送ARP请求（Request）
  源MAC = A的MAC
  目的MAC = FF:FF:FF:FF:FF:FF（广播地址）
  内容："谁的IP是IP_B？请告诉IP_A，你的MAC是什么？"

步骤3: 局域网内所有主机都收到该ARP请求
  非B主机：检查目的IP不是自己，丢弃
  主机B：目的IP是自己，准备应答

步骤4: B单播发送ARP应答（Reply）
  源MAC = B的MAC
  目的MAC = A的MAC（A的请求中包含A的MAC）
  内容："我是IP_B，我的MAC是MAC_B"

步骤5: A收到应答，更新ARP缓存
  添加条目：IP_B → MAC_B
```

### ARP报文格式（关键字段）

| 字段 | 内容 |
|------|------|
| 硬件类型 | 1（以太网） |
| 协议类型 | 0x0800（IPv4） |
| 硬件地址长度 | 6（MAC地址48位） |
| 协议地址长度 | 4（IPv4地址32位） |
| 操作 | 1=请求，2=应答 |
| 发送方MAC | 发送方硬件地址 |
| 发送方IP | 发送方协议地址 |
| 目标MAC | 请求时为全0，应答时为目标MAC |
| 目标IP | 目标协议地址 |

## 直观理解

**类比**：ARP像在教室里喊话
- 你在教室里喊："张三在吗？我是李四，我的座位在第三排"
- 整个教室的人都听到了（广播）
- 只有张三回应："我在第五排"（单播回复）
- 你记住了张三的位置（更新ARP缓存）

**记忆口诀**：
- "ARP广播问，单播答"
- "同网段直接问，跨网段问网关"

## 协议关联

### ARP与不同网段通信

当A（192.168.1.1）要发数据给B（10.0.0.1）时：

1. A判断B不在同一子网（用子网掩码判断）
2. A将数据帧的目的MAC设为**默认网关的MAC**（不是B的MAC）
3. A发ARP请求获取网关的MAC（如果缓存中没有）
4. 路由器（网关）收到帧，解封装，查路由表
5. 路由器向B所在网段转发，发ARP请求获取B的MAC
6. 路由器将帧发给B

**关键**：跨网段时，每一跳都要ARP获取下一跳的MAC，但IP地址始终不变（源IP=A，目的IP=B）。

### ARP与RARP
- ARP：IP → MAC
- RARP：MAC → IP（已基本淘汰，被DHCP替代）

### 考试陷阱
- ARP请求是广播，ARP应答是单播
- ARP只能解析同一网段内的MAC
- 每经过一个路由器，MAC地址改变，IP地址不变
