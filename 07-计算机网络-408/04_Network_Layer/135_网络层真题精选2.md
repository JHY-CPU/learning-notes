# 网络层真题精选（二）

## 核心概念

### 真题一：RIP路由更新
路由器R1的初始路由表：
| 目的 | 距离 | 下一跳 |
|------|------|--------|
| N1 | 0 | 直连 |
| N2 | 0 | 直连 |

收到邻居R2的路由表：N3距离2，N4距离3

R1更新后的路由表：
| 目的 | 距离 | 下一跳 |
|------|------|--------|
| N1 | 0 | 直连 |
| N2 | 0 | 直连 |
| N3 | 3 | R2 |
| N4 | 4 | R2 |

## 原理分析

### 真题二：ARP过程
主机A（IP=10.0.0.1）要发送IP数据报到主机B（IP=10.0.0.2），同一子网。请描述完整的ARP和数据发送过程。

**解**：
1. A检查ARP缓存，未找到10.0.0.2的MAC
2. A发送ARP请求（广播，目的MAC=FF:FF:FF:FF:FF:FF）
3. 同一子网所有主机收到，只有B识别自己的IP地址
4. B发送ARP响应（单播，包含自己的MAC地址）
5. A更新ARP缓存，封装以太网帧发送

### 真题三：DHCP工作过程
新主机加入网络，获取IP地址的四步交互：
1. **Discover**：广播（源0.0.0.0，目的255.255.255.255）
2. **Offer**：服务器响应，提供IP地址
3. **Request**：广播请求使用该IP
4. **ACK**：服务器确认分配

### 真题四：ICMP报文
ping命令使用ICMP回显请求（类型8）和回显应答（类型0）
traceroute原理：
1. 发送TTL=1的UDP数据报，第一跳路由器回ICMP超时
2. 发送TTL=2的UDP数据报，第二跳路由器回ICMP超时
3. 直到到达目的主机（回ICMP端口不可达）

### 真题五：跨子网传输
主机A（10.0.0.1/24）发送TCP段到主机B（10.0.1.2/24），经过路由器R
- A→R帧：源MAC_A，目的MAC_R，源IP_A，目的IP_B
- R→B帧：源MAC_R，目的MAC_B，源IP_A，目的IP_B

关键：IP地址不变（端到端），MAC地址每跳改变（逐跳）

## 直观理解
- 这些真题涵盖了网络层的核心操作过程，理解流程比记忆细节更重要

## 协议关联
- ARP、DHCP、ICMP是网络层最常见的辅助协议
- 跨子网传输涉及路由表查询、ARP解析、帧重新封装
- 这些过程是理解网络层整体运作的关键
