# IPv6与IPv4对比

## 核心概念

### 地址空间对比
| 项目 | IPv4 | IPv6 |
|------|------|------|
| 地址长度 | 32位 | 128位 |
| 地址数量 | $2^{32} \approx 43$亿 | $2^{128} \approx 3.4 \times 10^{38}$ |
| 表示格式 | 点分十进制 | 冒号分隔十六进制 |

### 首部对比
| 特性 | IPv4 | IPv6 |
|------|------|------|
| 首部长度 | 20-60字节可变 | 固定40字节 |
| 首部校验和 | 有 | 无 |
| 分片处理 | 路由器可分片 | 仅源端分片 |
| 可选字段 | 首部直接包含 | 扩展首部链 |
| TTL | TTL字段 | 跳数限制字段 |
| 服务类型 | TOS/DS字段 | 流量类别+流标签 |

### 功能对比
| 功能 | IPv4 | IPv6 |
|------|------|------|
| 地址解析 | ARP（广播） | NDP NS/NA（多播） |
| 差错报告 | ICMPv4 | ICMPv6 |
| 组管理 | IGMP | ICMPv6 MLD |
| 地址配置 | DHCP | SLAAC或DHCPv6 |
| 广播 | 支持 | 不支持（用多播替代） |

## 原理分析
- IPv6取消校验和的原因：数据链路层以太网CRC和传输层TCP/UDP校验和已足够，取消后减少路由器处理开销
- IPv6取消路由器分片的原因：分片和重组消耗路由器大量资源，IPv6使用路径MTU发现让源端直接发送合适大小的数据报
- IPv6取消首部长度字段原因：首部固定40字节，无需长度字段

## 直观理解
- IPv4像"老城区"——地址紧张、规划混乱
- IPv6像"新开发区"——地址充裕、规划整齐
- 共存阶段像"中英双语"并行，最终逐步过渡到IPv6

## 协议关联
- 过渡技术是408考试重点：双栈、隧道、NAT64
- IPv6对应用层透明：HTTP/FTP无需修改，DNS需扩展AAAA记录
- ICMPv6整合了ICMPv4+ARP+IGMP三个协议的功能
