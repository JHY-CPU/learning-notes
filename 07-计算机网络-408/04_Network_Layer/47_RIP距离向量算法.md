# 47_RIP距离向量算法

## 核心概念

- **距离向量算法（Bellman-Ford）**：
  - 核心方程：$d_x(y) = \min_v \{c(x,v) + d_v(y)\}$
  - $d_x(y)$：从x到y的最短距离
  - $c(x,v)$：x到邻居v的链路代价
  - $d_v(y)$：从v到y的最短距离（v告知x的）

- **RIP的距离就是跳数**：
  - 直连网络距离 = 1
  - 经过n个路由器的距离 = n+1
  - 不可达距离 = 16

- **RIP更新规则**：
  - 收到邻居的路由表后，每条距离加1
  - 与自己的路由表比较，更新更优的路由

## 原理分析

### Bellman-Ford方程在RIP中的应用

$$d_x(y) = \min_{v \in 邻居} \{1 + d_v(y)\}$$

RIP中所有链路代价为1（跳数），所以：

$$d_x(y) = \min_{v \in 邻居} \{1 + d_v(y)\}$$

### 更新示例

初始状态：
```
路由器A的直连网络：N1(距离1), N2(距离1)
路由器B的直连网络：N2(距离1), N3(距离1)
路由器C的直连网络：N3(距离1), N4(距离1)
A和B相邻，B和C相邻
```

第一轮更新后：
```
A的路由表：
  N1 → 距离1（直连）
  N2 → 距离1（直连）
  N3 → 距离2（从B学到：1+1=2）
  N4 → 距离3（从B学到：B到N4距离2，A到B距离1，1+2=3）

B的路由表：
  N1 → 距离2（从A学到）
  N2 → 距离1（直连）
  N3 → 距离1（直连）
  N4 → 距离2（从C学到）

C的路由表：
  N1 → 距离3（从B学到）
  N2 → 距离2（从B学到）
  N3 → 距离1（直连）
  N4 → 距离1（直连）
```

### 距离向量算法的伪代码

```
收到邻居v的路由表D_v:
  for 每个目的网络N:
    新距离 = D_v[N] + 1
    if N 不在我的路由表中:
      if 新距离 < 16:
        添加 (N, 新距离, 下一跳=v)
    else:
      if 当前下一跳 == v:  # 从同一邻居学的
        更新距离为 min(新距离, 16)  # 必须跟踪邻居的变化
      else:
        if 新距离 < 当前距离:
          更新距离为 新距离，下一跳=v
```

## 直观理解

**类比**：距离向量算法像"打听路线"
- 你问邻居："到XX地方多远？"
- 邻居说："我走3步"
- 你想："我到你1步 + 你到XX3步 = 4步"
- 如果比你已知的路更短，就更新

**口诀**："邻居告诉你距离，加1就是我的距离，取最小的"

## 协议关联

### 距离向量 vs 链路状态

| 特性 | 距离向量（RIP） | 链路状态（OSPF） |
|------|---------------|----------------|
| 信息量 | 整个路由表 | 只有链路状态 |
| 交换范围 | 仅相邻邻居 | 泛洪到整个AS |
| 算法 | Bellman-Ford | Dijkstra |
| 收敛 | 慢 | 快 |
| 环路风险 | 高 | 低 |

### 考试要点
- Bellman-Ford方程是距离向量算法的数学基础
- RIP中链路代价均为1（跳数）
- 收到邻居的路由表后距离加1
- 计数到无穷是距离向量算法的经典问题
