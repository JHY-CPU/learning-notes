# 双栈技术与隧道技术详解

## 核心概念

### 双栈技术详解
- **定义**：网络节点同时运行IPv4和IPv6两套完整协议栈
- **工作方式**：根据目的地址类型自动选择IPv4或IPv6协议
- **DNS机制**：查询A记录得到IPv4地址，查询AAAA记录得到IPv6地址
- **实现要求**：所有路由器和主机都必须支持双栈（部署成本较高）
- **DS-Lite**：运营商侧NAT+IPv4-in-IPv6隧道，减少IPv4地址消耗

### 隧道技术详解
- **核心操作**：
  - 入口（Ingress）：添加IPv4首部，将IPv6数据报作为IPv4数据部分
  - 出口（Egress）：去除IPv4首部，恢复原始IPv6数据报
- **隧道类型**：
  - **手动隧道**：管理员手动配置隧道端点IP地址
  - **自动隧道**：根据目的地址自动确定隧道端点
  - **6to4**：使用嵌入IPv4地址的2002::前缀自动建立
  - **ISATAP**：站点内自动隧道寻址协议
  - **Teredo**：通过NAT设备的UDP封装隧道

## 原理分析

### 隧道封装示意
```
[IPv4首部 20字节][IPv6首部 40字节][IPv6数据 变长]
```
- IPv4首部中的协议字段设为41（表示数据部分为IPv6）
- IPv4源地址 = 隧道入口节点的IPv4地址
- IPv4目的地址 = 隧道出口节点的IPv4地址

### 双栈选择流程
1. 应用程序请求DNS解析域名
2. DNS返回A记录（IPv4）和/或AAAA记录（IPv6）
3. 协议栈根据返回结果选择IPv4或IPv6
4. 若同时有AAAA和A记录，通常优先使用IPv6

## 直观理解
- 双栈像"两车道并行"——IPv4和IPv6各走各的路
- 隧道像"穿山隧道"——IPv6的车走IPv4的隧道通过

## 协议关联
- 隧道封装涉及IPv4协议字段=41
- 双栈节点需要同时维护ARP邻居表和NDP邻居缓存
- 实际网络中双栈+隧道常配合使用
- 6to4中继路由器连接6to4网络和原生IPv6网络
