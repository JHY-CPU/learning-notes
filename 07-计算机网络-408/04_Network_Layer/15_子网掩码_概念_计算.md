# 15_子网掩码_概念_计算

## 核心概念

- **子网掩码计算的三大任务**：
  1. 给定IP和掩码，求网络地址（AND运算）
  2. 给定IP和掩码，求广播地址和可用范围
  3. 给定需求（子网数/主机数），确定子网掩码

- **核心公式**：
  - 网络地址 = IP AND 子网掩码
  - 广播地址 = 网络地址 OR (~子网掩码)
  - 可用范围 = 网络地址+1 ~ 广播地址-1
  - 可用主机数 = $2^{\text{主机位数}} - 2$

## 原理分析

### 例1：AND运算

IP: 172.16.78.100，掩码: 255.255.240.0

```
78  = 01001110
240 = 11110000
AND = 01000000 = 64
```

网络地址 = 172.16.64.0

广播地址：主机位全1
```
~240 = 00001111
64 OR 15 = 01001111 = 79
```
广播地址 = 172.16.79.255

可用范围 = 172.16.64.1 ~ 172.16.79.254
可用主机数 = $2^{12} - 2 = 4094$

### 例2：反向计算——确定掩码

需求：某公司需要30个子网，使用C类网络200.10.1.0/24

步骤：
- 需要30个子网：$2^n \geq 30$，$n = 5$
- 借5位作子网号，剩余3位作主机号
- 新前缀 = 24 + 5 = 29
- 每子网主机数 = $2^3 - 2 = 6$
- 子网掩码 = 255.255.255.248

**问题**：每子网只能6台主机，如果需要更多就不行！
**解法**：可能需要更大的地址块或使用CIDR

### 例3：复杂掩码计算

IP: 10.217.128.0/20，求该子网的广播地址

- /20 掩码 = 255.255.240.0
- 第三字节：128 AND 240 = 128（128 = 10000000, 240 = 11110000, AND = 10000000 = 128）
- 广播：第三字节 128 + 15 = 143，第四字节 255
- 广播地址 = 10.217.143.255

## 直观理解

**速算技巧**：
- 掩码最后一个非255字节值 $m$
- $m$ 对应的借位数：查表（128→1, 192→2, 224→3, 240→4, 248→5, 252→6, 254→7）
- 子网间隔 = 256 - $m$
- 第三字节值 ÷ 间隔 = 商...余数 → 网络地址第三字节 = 商 × 间隔

**记忆**："256减掩码得间隔，除法定子网"

## 协议关联

- 路由器用子网掩码+IP做最长前缀匹配
- 主机用子网掩码判断是否同网段（决定ARP还是发网关）
- 考试中子网掩码经常与CIDR、路由表联合出题
