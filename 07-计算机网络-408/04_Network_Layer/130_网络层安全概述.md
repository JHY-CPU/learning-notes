# 网络层安全概述

## 核心概念
- 网络层安全主要解决：数据机密性、数据完整性、身份认证、抗重放攻击
- IPsec（IP Security）是网络层安全的核心协议
- 网络层安全与应用层安全（HTTPS）、传输层安全（TLS）不同

### 网络层面临的安全威胁
| 威胁 | 说明 | 应对 |
|------|------|------|
| 窃听 | 截获数据报 | 加密（ESP） |
| 篡改 | 修改数据报内容 | 完整性校验（AH/ESP） |
| 伪装 | 伪造源地址 | 身份认证（AH/ESP） |
| 重放 | 重发旧数据报 | 序列号（IPsec） |
| 拒绝服务 | 耗尽资源 | 限流、过滤 |

### IPsec两大组件
- **AH（Authentication Header）**：认证头部，提供完整性认证和抗重放，不加密
- **ESP（Encapsulating Security Payload）**：封装安全载荷，提供加密、完整性认证和抗重放

### IPsec两种模式
| 模式 | 封装方式 | 适用场景 |
|------|----------|----------|
| 传输模式 | AH/ESP插在IP首部和数据之间 | 端到端安全 |
| 隧道模式 | 整个IP数据报被封装在新的IP数据报中 | VPN（站点到站点）|

## 原理分析

### AH vs ESP对比
| 特性 | AH | ESP |
|------|-----|-----|
| 加密 | 否 | 是 |
| 完整性认证 | 是 | 是 |
| 抗重放 | 是 | 是 |
| NAT穿越 | 不支持（保护IP首部）| 支持 |

### SA（Security Association）
- IPsec的核心概念，定义安全参数
- 包含：加密算法、认证算法、密钥、SPI（安全参数索引）
- 每个方向需要一个SA（单向）

## 直观理解
- IPsec像"保险箱"——数据放入保险箱（加密），贴上防拆封条（认证）
- 传输模式像"信封加密"——只加密信的内容
- 隧道模式像"快递箱加密"——整个信封放入加密的快递箱

## 协议关联
- IPsec工作在网络层，对上层协议透明
- IPsec与NAT存在兼容性问题（AH保护IP首部）
- IKE（Internet Key Exchange）协议用于自动协商SA
- VPN利用IPsec隧道模式实现站点间安全通信
