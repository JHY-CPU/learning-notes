# SDN软件定义网络

## 核心概念
- SDN（Software Defined Networking）将网络的控制平面和数据平面分离
- 核心思想：网络控制逻辑集中到控制器，底层转发设备只负责执行

### SDN三层架构
- **应用层（Application Layer）**：网络应用程序（负载均衡、防火墙、QoS等）
- **控制层（Control Layer）**：SDN控制器，集中管理网络状态
- **数据层（Data Layer）**：转发设备（交换机），负责数据转发

### SDN vs 传统网络
| 特性 | 传统网络 | SDN |
|------|----------|-----|
| 控制方式 | 分布式（每台设备独立决策） | 集中式（控制器统一决策） |
| 转发表来源 | 路由协议计算 | 控制器下发 |
| 编程性 | 低（设备厂商锁定） | 高（软件可编程） |
| 灵活性 | 低 | 高 |

### OpenFlow协议
- SDN最著名的南向接口协议
- 控制器通过OpenFlow与交换机通信
- 交换机根据流表（Flow Table）执行转发动作
- 流表项 = 匹配字段 + 优先级 + 计数器 + 动作集合

## 原理分析

### SDN控制平面与数据平面分离
- **控制平面**：运行在控制器上，负责路由计算、策略制定
- **数据平面**：运行在交换机上，根据流表匹配和执行动作
- 分离后，网络管理员可以通过编程方式控制网络行为

### SDN的优势
- 集中控制：全局网络视图，更优的路由决策
- 可编程性：通过软件定义网络行为
- 虚拟化：网络资源灵活分配
- 快速创新：新功能通过软件更新实现

## 直观理解
- 传统网络像"每个路口都有交警自己判断"——SDN像"所有路口由一个指挥中心统一调度"
- OpenFlow像"遥控器指令"——控制器按下按钮，交换机执行动作

## 协议关联
- SDN控制器替代了传统路由协议（OSPF、BGP等）的部分功能
- SDN与NFV（网络功能虚拟化）结合，实现网络服务的软件化
- SDN控制器之间的东西向接口用于多控制器协同
