# OSPF综合真题

## 核心概念

### OSPF高频考点
- OSPF基于链路状态算法，使用Dijkstra计算最短路径
- OSPF直接封装在IP中（协议号89）
- OSPF使用链路代价（Cost）作为度量值
- OSPF支持多区域设计，Area 0为骨干区域

## 原理分析

### 真题一（选择题）
**Q1**: OSPF使用什么协议进行传输？
A. TCP B. UDP C. IP D. ICMP

**答**：C。OSPF直接封装在IP数据报中，协议号89。

**Q2**: OSPF使用的路由算法是？
A. 距离向量 B. 链路状态 C. 路径向量 D. 泛洪

**答**：B。OSPF使用链路状态算法，每个路由器构建完整的网络拓扑。

**Q3**: OSPF的链路代价通常与什么成反比？
A. 跳数 B. 带宽 C. 延迟 D. 距离

**答**：B。Cost = 参考带宽/接口带宽，带宽越大Cost越小。

### 真题二：Dijkstra算法
网络拓扑：
```
A --1-- B --2-- C
|             |
3             1
D -----------+
```
A到B链路代价1，B到C代价2，A到D代价3，D到C代价1

从A出发计算最短路径树：
1. 访问A（距离0），更新B=1，D=3
2. 访问B（距离1），更新C=min(1+2, 无穷)=3
3. 访问C（距离3），更新D=min(3+1, 3)=3
4. 访问D（距离3），无更新

最短路径：A→D=3，A→B→C=3，A→B=1

### 真题三：OSPF与RIP对比
| 特性 | OSPF | RIP |
|------|------|-----|
| 算法 | 链路状态 | 距离向量 |
| 度量值 | 链路代价 | 跳数 |
| 收敛速度 | 快 | 慢 |
| 适用范围 | 大型网络 | 小型网络 |
| 封装方式 | IP直接封装 | UDP |

## 直观理解
- OSPF像"每个人画完整地图"——开销大但决策准确
- RIP像"每个人只问邻居方向"——简单但可能走弯路

## 协议关联
- OSPF与RIP是内部网关协议（IGP），BGP是外部网关协议（EGP）
- OSPF LSA泛洪机制保证所有路由器的LSDB一致
- OSPF支持认证（明文和MD5）
