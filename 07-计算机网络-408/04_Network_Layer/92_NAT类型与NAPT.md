# NAT类型与NAPT

## 核心概念

### NAT三种类型详解
| 类型 | 映射方式 | 公有地址需求 | 端口使用 |
|------|----------|-------------|----------|
| 静态NAT | 一对一固定 | 每用户一个 | 不使用端口 |
| 动态NAT | 一对一双射 | 从地址池分配 | 不使用端口 |
| NAPT/PAT | 多对一复用 | 多用户共享一个 | 使用端口号区分 |

### NAPT（Network Address Port Translation）
- 也称PAT（Port Address Translation）或地址端口复用
- 多个私有IP共享一个公有IP，用端口号区分不同连接
- 是目前最常用的NAT类型

### NAPT转换表示例
| 内部地址:端口 | 外部地址:端口 |
|---------------|--------------|
| 10.0.0.1:3000 | 202.100.1.1:5001 |
| 10.0.0.2:3000 | 202.100.1.1:5002 |
| 10.0.0.1:3001 | 202.100.1.1:5003 |

## 原理分析

### NAPT工作流程
1. 主机A(10.0.0.1:3000)发送TCP连接请求
2. NAT路由器分配外部端口5001，记录映射
3. 外出数据报：源地址改为202.100.1.1:5001
4. 返回数据报：目的地址202.100.1.1:5001 → 查表 → 10.0.0.1:3000

### NAPT端口分配策略
- 端口范围：1024~65535（排除熟知端口0~1023）
- 理论最大并发连接数：约64512个
- 实际受NAT设备内存限制

### NAT穿越问题
- 外部主机无法主动发起与内部主机的连接
- 解决方案：
  - **静态NAT/端口映射**：手动配置端口转发
  - **STUN**：NAT会话穿透工具
  - **TURN**：通过中继服务器转发
  - **ICE**：综合STUN和TURN的框架

## 直观理解
- 静态NAT像"专属车位"——固定分配
- 动态NAT像"停车场临时车位"——来了分配一个
- NAPT像"合租房"——多人共用一个门牌号，用房间号区分

## 协议关联
- NAPT需要查看传输层端口号（TCP/UDP）
- FTP主动模式在NAT下有问题（PORT命令携带内部地址）
- IPsec与NAT存在兼容性问题（IPsec保护的IP首部无法被NAT修改）
