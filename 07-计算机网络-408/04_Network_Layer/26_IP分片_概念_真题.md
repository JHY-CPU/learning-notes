# 26_IP分片_概念_真题

## 核心概念

- **408 IP分片真题类型**：
  1. 给MTU和原数据报，求分片各字段值
  2. 给分片信息，求原数据报长度
  3. 多跳分片（路径上不同MTU）
  4. 判断某分片是否为第一个/最后一个

## 原理分析

### 真题1

**题**：一个IP数据报总长度为3820字节（固定首部20字节），标识为6666。现通过一个MTU=1500字节的网络，求分片后各片的总长度、标识、MF和片偏移。

**解**：
- 数据 = 3820 - 20 = 3800字节
- 每片最大数据 = $\lfloor(1500-20)/8\rfloor \times 8 = 1480$
- 分片数 = $\lceil3800/1480\rceil = 3$

| 分片 | 总长度 | 标识 | MF | 片偏移 |
|------|--------|------|-----|--------|
| 1 | 1500 | 6666 | 1 | 0 |
| 2 | 1500 | 6666 | 1 | 185 |
| 3 | 860 | 6666 | 0 | 370 |

验证：偏移185×8=1480，偏移370×8=2960，1480+1480+840=3800 ✓

### 真题2

**题**：收到以下3个IP分片（首部均20字节）：
- 分片1：总长1480，偏移0，MF=1
- 分片2：总长1480，偏移148，MF=1
- 分片3：总长380，偏移296，MF=0

（1）原数据报的数据部分长度？
（2）原数据报的总长度？

**解**：
- 分片1：偏移0，数据1480-20=1460...等等，题目总长1480不是1500
  - 数据1 = 1480 - 20 = 1460，偏移0
  - 数据2 = 1480 - 20 = 1460，偏移148×8 = 1184...这不对

重新计算（假设题目偏移单位无误）：
- 偏移148 → 实际位置 148×8 = 1184
- 偏移296 → 实际位置 296×8 = 2368
- 数据1：1460字节（位置0~1459）
- 数据2：1460字节（位置1184~2643）→ 与分片1重叠？

修正理解（合理假设偏移值正确）：
- 偏移值直接：0, 148, 296
- 偏移单位为字节时：数据=1480-20=1460
- 第一片到1460，第二片从148到148+1460=1608，第三片从296到296+360=656

采用标准8字节单位：
- 分片1：偏移0，数据1460
- 分片2：偏移185，数据1460（若总长1500则数据1480）
- 分片3：偏移370，数据360（若总长380）

假设合理数据：原数据 = 1460+1460+360 = 3280字节，总长=3300字节

### 真题3：标准真题

**题**：IP数据报数据部分4000字节，标识12345，经过MTU=1400的网络，写出各分片的字段值。

**解**：
- 每片最大数据 = $\lfloor(1400-20)/8\rfloor \times 8 = 1376$
- 分片1：总长1396，标识12345，MF=1，偏移0
- 分片2：总长1396，标识12345，MF=1，偏移172（1376/8=172）
- 分片3：总长1396，标识12345，MF=1，偏移344（2752/8=344）
- 分片4：总长872，标识12345，MF=0，偏移516（4128/8=516...）

验证：1376×3 = 4128 > 4000，所以只需要3片不够
- 4000/1376 = 2.9...，需要3片
- 分片3数据 = 4000 - 1376×2 = 1248
- 分片3总长 = 1248+20 = 1268
- 分片3偏移 = 2752/8 = 344，MF=0

### 真题4：判断题

**题**：IP分片可以在中间路由器上重组。判断正误。

**解**：**错误**。IP分片只能在目的主机重组，中间路由器只负责转发分片。

## 直观理解

**解题模板**：
1. 数据 = 总长度 - 首部(20)
2. 每片数据 = $\lfloor(MTU-20)/8\rfloor \times 8$
3. 分片数 = $\lceil$数据/每片数据$\rceil$
4. 每片偏移 = 起始位置/8
5. 最后一片MF=0，其余MF=1
6. 所有分片标识相同

## 协议关联

- IP分片与TCP分段不同：IP分片在网络层，TCP分段在传输层
- 重组只在目的主机进行
- 考试中分片计算是高频考点，必须掌握
