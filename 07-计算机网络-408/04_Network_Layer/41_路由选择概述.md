# 41_路由选择概述

## 核心概念

- **路由选择（Routing）**：确定分组从源到目的的传输路径
  - 路由选择 = 路由算法 + 路由协议
  - 目标：找到"最佳"路径（最短、最快、最便宜等）

- **路由表（Routing Table）**：
  - 路由器的核心数据结构
  - 存储目的网络 → 下一跳/出接口的映射
  - 转发时查路由表，确定下一跳

- **路由选择的两大功能**：
  1. **路由选择**（Routing）：用路由算法计算路由表
  2. **分组转发**（Forwarding）：根据路由表转发分组

- **路由协议分类**：

| 分类 | 协议 | 特点 |
|------|------|------|
| 内部网关协议IGP | RIP, OSPF | 自治系统内部 |
| 外部网关协议EGP | BGP | 自治系统之间 |

- **自治系统（AS, Autonomous System）**：
  - 一个AS由一个组织管理的网络
  - 有唯一的AS号
  - AS内部使用IGP，AS之间使用BGP

- **408考点**：
  - 路由选择 vs 分组转发
  - IGP vs EGP
  - 静态路由 vs 动态路由
  - 距离向量 vs 链路状态

## 原理分析

### 路由选择的过程

```
1. 各路由器运行路由协议（RIP/OSPF/BGP）
2. 通过路由协议交换路由信息
3. 运行路由算法（距离向量/链路状态）计算最佳路径
4. 生成/更新路由表
5. 分组到达时，查路由表确定下一跳
6. 转发分组
```

### 路由协议的层次

```
Internet
  ├── AS 1 (IGP: OSPF)
  │     ├── 网络1
  │     └── 网络2
  ├── AS 2 (IGP: RIP)
  │     ├── 网络3
  │     └── 网络4
  └── AS之间: BGP连接
```

### 最佳路径的度量

- **跳数（Hop Count）**：经过的路由器数量（RIP使用）
- **带宽（Bandwidth）**：链路带宽（OSPF使用）
- **延迟（Delay）**：端到端延迟
- **负载（Load）**：链路负载
- **可靠性（Reliability）**：链路可靠性
- **代价（Cost）**：综合度量

## 直观理解

**类比**：路由选择像GPS导航
- **路由选择** = GPS计算最佳路线
- **路由表** = 导航的地图数据
- **分组转发** = 按照导航指示在每个路口转向
- **路由协议** = 不同的地图公司如何共享路况信息

**记忆口诀**：
- "AS内部用IGP（RIP/OSPF），AS之间用BGP"
- "RIP数跳数，OSPF算代价，BGP看路径"

## 协议关联

### 路由与分层

| 层次 | 协议 | 范围 |
|------|------|------|
| AS内部 | RIP, OSPF | IGP |
| AS之间 | BGP | EGP |

### 路由与IP
- 路由选择确定分组的下一跳
- IP协议根据路由表转发分组
- 最长前缀匹配是查路由表的算法

### 考试重点
- RIP是距离向量协议，OSPF是链路状态协议，BGP是路径向量协议
- RIP使用UDP，OSPF直接封装在IP中（协议号89），BGP使用TCP
- AS内部关注最优路径，AS之间关注策略（政治、经济因素）
