# 网络层协议对比

## 核心概念

### 网络层核心协议
| 协议 | 功能 | 位置 |
|------|------|------|
| IP | 数据报传输、寻址、路由 | 核心 |
| ICMP | 差错报告、网络诊断 | 辅助IP |
| ARP | IP→MAC地址解析 | 辅助IP |
| DHCP | 动态IP地址分配 | 应用层辅助 |
| OSPF/RIP/BGP | 路由选择 | 路由器之间 |
| IGMP | 多播组管理 | 辅助IP |

### 协议关系图
```
应用层 ← DHCP
网络层 ← IP（核心）
         ├── ICMP（差错报告、诊断）
         ├── IGMP（多播组管理）
         ├── ARP（地址解析）
         ├── OSPF/RIP（AS内路由）
         └── BGP（AS间路由）
```

### IP与各协议的关系
- ICMP：封装在IP中（协议号1），向IP提供差错报告服务
- ARP：为IP提供MAC地址解析服务
- DHCP：为IP提供地址配置服务
- OSPF/RIP：为IP路由表提供路由信息
- IGMP：管理多播组，辅助IP多播

## 原理分析

### 数据报转发涉及的协议
1. DHCP获取IP地址、网关、DNS（应用层）
2. DNS解析目的域名到IP地址（应用层）
3. IP确定目的地址和下一跳
4. ARP解析下一跳的MAC地址
5. 数据链路层封装帧发送
6. 如果出错，ICMP差错报告

## 直观理解
- IP像"快递单"——记录源地址和目的地址
- ICMP像"异常通知"——快递送不到时通知发件人
- ARP像"查号台"——根据地址查门牌号
- DHCP像"分配工位"——新人自动获得位置
- 路由协议像"导航系统"——规划最佳路线

## 协议关联
- 所有协议都服务于IP协议
- ICMP、IGMP、OSPF、RIP都封装在IP数据报中
- ARP虽然服务IP但工作在数据链路层和网络层之间
