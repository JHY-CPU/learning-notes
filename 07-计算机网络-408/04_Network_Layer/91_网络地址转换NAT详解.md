# 网络地址转换NAT详解

## 核心概念
- NAT（Network Address Translation）：将私有IP地址转换为公有IP地址
- 核心目的：解决IPv4地址短缺问题，同时隐藏内部网络结构

### NAT类型
- **静态NAT**：一对一映射，一个私有地址固定对应一个公有地址
- **动态NAT**：从地址池中动态分配公有地址给私有地址
- **NAPT/PAT（Network Address Port Translation）**：
  - 多个私有地址共享一个公有地址
  - 使用传输层端口号区分不同连接
  - 也称为端口地址转换或地址端口复用

### NAT转换表
| 私有地址:端口 | 公有地址:端口 | 协议 |
|---------------|---------------|------|
| 192.168.1.10:1024 | 203.0.113.1:4001 | TCP |
| 192.168.1.20:1025 | 203.0.113.1:4002 | TCP |
| 192.168.1.30:1026 | 203.0.113.1:4003 | UDP |

## 原理分析

### NAT工作过程
1. 内部主机发送数据报（源地址=私有地址，源端口=临时端口）
2. NAT路由器查转换表，替换源地址为公有地址，分配新端口
3. 外部服务器响应（目的地址=公有地址，目的端口=新端口）
4. NAT路由器查转换表，替换目的地址为私有地址，恢复原端口

### NAPT地址复用计算
假设公有地址有1个，端口号16位：
- 理论最大连接数：$2^{16} = 65536$个（每个端口一个连接）
- 实际可用：约60000个（排除保留端口）

### NAT的问题
- 破坏了端到端原则
- 与IPsec、SIP等协议存在兼容性问题
- P2P应用需要打洞（NAT Traversal）

## 直观理解
- NAT像"公司前台"——公司内部员工用分机号（私有地址），对外统一用公司总机号（公有地址）
- NAPT像"共享手机号"——多个人共用一个号码，通过不同分机号区分

## 协议关联
- NAT工作在网络层和传输层之间（需查看端口号）
- 静态NAT和动态NAT只看IP地址，NAPT还需看端口号
- NAT与防火墙常集成在一起
- NAT穿越技术：STUN、TURN、ICE
