# 44_路由转发过程

## 核心概念

- **分组转发的完整流程**：
  1. 物理层接收比特流
  2. 数据链路层组装帧，检查目的MAC
  3. 网络层提取目的IP，查路由表
  4. 确定下一跳，用ARP获取下一跳MAC
  5. 重新封装帧，从出接口发送

- **路由器处理分组的关键步骤**：
  1. 接收帧 → 检查FCS → 提取IP数据报
  2. 检查TTL → TTL-1 → 若为0则丢弃
  3. 重新计算首部检验和
  4. 查路由表（最长前缀匹配）
  5. 确定出接口和下一跳
  6. ARP获取下一跳MAC（若缓存中没有）
  7. 重新封装以太网帧
  8. 从出接口发送

- **408考点**：
  - 路由器的完整转发过程
  - 每跳中IP地址不变，MAC地址变化
  - TTL减1和检验和重新计算
  - 路由器不改变IP数据报的数据部分

## 原理分析

### 转发过程详解

```
入接口（物理层/数据链路层）：
  1. 接收电信号/光信号 → 比特流
  2. 组装以太网帧
  3. 检查FCS（帧检验序列），若出错则丢弃
  4. 检查目的MAC是否是本接口的MAC
     不是 → 不是发给我的，丢弃（除非是混杂模式）
  5. 剥离以太网帧头，提取IP数据报

网络层处理：
  6. 检查IP首部检验和，若出错则丢弃
  7. TTL = TTL - 1
     若TTL = 0 → 丢弃，发送ICMP时间超时报文
  8. 重新计算首部检验和（因为TTL变了）
  9. 提取目的IP地址
  10. 查路由表（最长前缀匹配）
  11. 确定出接口和下一跳IP地址
  12. 若IP数据报总长度 > 出接口MTU
      DF=1 → 丢弃，发送ICMP目的不可达
      DF=0 → 分片

出接口（数据链路层/物理层）：
  13. 用ARP获取下一跳的MAC地址
  14. 封装新的以太网帧：
      源MAC = 出接口的MAC
      目的MAC = 下一跳的MAC
  15. 计算FCS
  16. 发送帧
```

### MAC地址变化示例

```
A(192.168.1.100) → R1 → R2 → B(192.168.3.50)

A→R1: 源MAC=MAC_A, 目的MAC=MAC_R1_eth0
       源IP=192.168.1.100, 目的IP=192.168.3.50

R1→R2: 源MAC=MAC_R1_eth1, 目的MAC=MAC_R2_eth0
        源IP=192.168.1.100, 目的IP=192.168.3.50（不变！）

R2→B: 源MAC=MAC_R2_eth1, 目的MAC=MAC_B
       源IP=192.168.1.100, 目的IP=192.168.3.50（不变！）
```

## 直观理解

**类比**：路由器像快递中转站
- 收到包裹 → 检查外包装（FCS）
- 看收件地址（目的IP） → 查分拣表（路由表）
- 换新的快递单（新MAC地址，IP不变）
- 从对应的门送出去（出接口）

**记忆口诀**：
- "收到先查错，TTL要减1，查表找下一跳，换MAC再出发"
- "IP端到端不变，MAC逐跳变化"

## 协议关联

- 转发过程涉及物理层、数据链路层、网络层
- ARP是转发过程中获取MAC地址的关键步骤
- TTL和检验和的修改是路由器的必要操作
- 分片可能在转发过程中发生（MTU限制）

### 考试要点
- 路由器转发时IP地址不变，MAC地址改变
- TTL每跳减1，检验和需重新计算
- 分片只在路由器发生，重组只在目的主机
- 路由器可能丢弃分组（TTL=0、FCS错误等）
