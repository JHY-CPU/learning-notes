# CIDR综合真题

## 核心概念

### CIDR真题考点
- 路由聚合：将多个连续子网合并为一个大网络
- 最长前缀匹配：路由器转发时选择掩码最长的匹配条目
- CIDR记法和地址块计算

## 原理分析

### 真题一：路由聚合
某路由器连接4个子网：
- 192.168.4.0/24
- 192.168.5.0/24
- 192.168.6.0/24
- 192.168.7.0/24

问：可以聚合为什么地址？

**解**：
第三个字节转二进制：
- 4 = 00000100
- 5 = 00000101
- 6 = 00000110
- 7 = 00000111

前5位相同（00000），共8+8+5=21位相同
聚合结果：192.168.4.0/21

### 真题二：最长前缀匹配
路由表：
```
目的网络             下一跳
10.0.0.0/8           A
10.1.0.0/16          B
10.1.1.0/24          C
0.0.0.0/0            D（默认路由）
```

目的地址10.1.2.3匹配哪些？选择哪条？
**答**：匹配10.0.0.0/8和默认路由，选择/8（更精确）→ 下一跳A

目的地址10.1.1.5匹配哪些？选择哪条？
**答**：匹配/8、/16、/24和默认路由，选择/24（最长）→ 下一跳C

### 真题三：CIDR地址块计算
已知地址块202.120.80.0/20，求：
1. 地址块的最小和最大地址
2. 可容纳多少台主机

**解**：
- /20：前20位固定，后12位可变
- 最小地址：202.120.80.0（全0）
- 最大地址：202.120.95.255（80=0101 0000，95=0101 1111）
- 可用主机数：$2^{12} - 2 = 4094$ 台

## 直观理解
- 路由聚合像"打包发货"——把几个小包裹合在一个大箱子里寄
- 最长前缀匹配像"查字典时选最接近的词"——越精确的匹配越优先

## 协议关联
- BGP协议利用CIDR进行路由聚合，减少全球路由表规模
- CIDR是IPv4地址管理的核心技术
- CIDR与NAT配合使用缓解IPv4地址耗尽
