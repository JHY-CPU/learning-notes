# ARP综合练习

## 核心概念
- ARP（Address Resolution Protocol）：地址解析协议，将IP地址解析为MAC地址
- ARP工作在数据链路层和网络层之间
- ARP使用广播请求和单播响应

### ARP工作流程
1. 发送方检查ARP缓存表是否有目的IP对应的MAC地址
2. 如果没有，发送ARP请求（广播，目的MAC = FF:FF:FF:FF:FF:FF）
3. 同一子网的所有主机收到ARP请求
4. 目的主机回复ARP响应（单播，包含自己的MAC地址）
5. 发送方更新ARP缓存表

### ARP报文格式
| 字段 | 长度 | 说明 |
|------|------|------|
| 硬件类型 | 2字节 | 1=以太网 |
| 协议类型 | 2字节 | 0x0800=IPv4 |
| 硬件地址长度 | 1字节 | 6（以太网MAC） |
| 协议地址长度 | 1字节 | 4（IPv4） |
| 操作码 | 2字节 | 1=请求，2=响应 |
| 发送方MAC | 6字节 | 发送方硬件地址 |
| 发送方IP | 4字节 | 发送方协议地址 |
| 目的MAC | 6字节 | 目的硬件地址（请求时全0） |
| 目的IP | 4字节 | 目的协议地址 |

## 原理分析

### 跨子网ARP
- 主机A（子网1）要发送数据给主机B（子网2）
- A的ARP请求的是网关路由器的MAC地址（不是B的MAC）
- 路由器收到后，用自己的MAC替换源MAC，查找B的MAC并转发

### ARP缓存表
- 条目有超时时间（通常2-20分钟）
- 包含动态条目（ARP学习）和静态条目（手动配置）

## 直观理解
- ARP像"查电话号码"——你知道名字（IP地址），通过广播喊"谁是X？"，X回答"我是，这是我的号码（MAC地址）"
- 跨子网ARP像"找前台"——你不知道目标在哪，先找网关（路由器），由网关转发

## 协议关联
- ARP依赖广播，只在同一条数据链路上有效
- ARP欺骗（ARP Spoofing）：攻击者发送伪造的ARP响应
- IPv6使用NDP的NS/NA消息替代ARP
- 免费ARP（Gratuitous ARP）：主机广播自己的IP-MAC映射，用于检测IP冲突
