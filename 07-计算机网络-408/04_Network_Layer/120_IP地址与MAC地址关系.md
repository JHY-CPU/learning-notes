# IP地址与MAC地址关系

## 核心概念
- IP地址：网络层地址，逻辑地址，标识网络中的主机，可随位置变化
- MAC地址：数据链路层地址，物理地址，固化在网卡中，全球唯一

### IP地址与MAC地址对比
| 特性 | IP地址 | MAC地址 |
|------|--------|---------|
| 层次 | 网络层 | 数据链路层 |
| 长度 | 32位（IPv4） | 48位 |
| 分配 | 网络管理员/DHCP | 网卡制造商 |
| 可变性 | 可变（随网络改变） | 通常不变 |
| 层次结构 | 有（网络+主机） | 无 |
| 功能 | 端到端寻址 | 逐跳寻址 |

### 两者缺一不可的原因
- 数据最终需要在数据链路上传输，必须使用MAC地址
- IP地址决定"去哪"，MAC地址决定"下一跳怎么到"
- 跨子网通信时MAC地址每跳改变，IP地址不变

### 地址解析
- ARP：IP地址 → MAC地址
- RARP：MAC地址 → IP地址（已被DHCP替代）

## 原理分析

### 数据报传输中的地址变化
源主机A → 路由器R → 目的主机B（不同子网）

| 跳 | 源MAC | 目的MAC | 源IP | 目的IP |
|----|-------|---------|------|--------|
| A→R | A的MAC | R的MAC | A的IP | B的IP |
| R→B | R的MAC | B的MAC | A的IP | B的IP |

关键结论：
- IP地址始终不变（端到端）
- MAC地址每跳改变（逐跳）

## 直观理解
- IP地址像"收件人地址"——最终送到哪，全程不变
- MAC地址像"每段运输的车辆"——飞机运到机场，卡车运到仓库，快递送到家，每段不同

## 协议关联
- ARP解析下一跳的MAC地址（不是最终目的的MAC）
- 路由器转发时修改帧的MAC地址但保持IP地址
- 数据链路层封装使用MAC地址，网络层使用IP地址
- IPv6使用NDP替代ARP进行地址解析
