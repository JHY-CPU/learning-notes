# IP分片综合计算

## 核心概念
- IP分片：当数据报长度超过链路MTU时，需要分片传输
- 分片相关字段：标识（16位）、标志（3位）、片偏移（13位）
- MTU（Maximum Transmission Unit）：最大传输单元（以太网MTU=1500字节）

### 分片三要素
| 字段 | 作用 |
|------|------|
| 标识（Identification） | 同一数据报的所有分片标识相同 |
| 标志（Flags） | MF位=1表示后面还有分片，DF位=1表示禁止分片 |
| 片偏移（Fragment Offset） | 以8字节为单位，表示该片在原数据报中的位置 |

### 分片计算公式
- 每个分片的数据部分必须是8字节的整数倍（片偏移以8为单位）
- 每个分片的总长度 = IP首部（20字节）+ 数据部分
- 最大数据部分 = MTU - 20（IP首部长度）

## 原理分析

### 计算示例
原始数据报：总长度3820字节，IP首部20字节 → 数据部分3800字节
链路MTU = 1500字节

最大数据部分 = 1500 - 20 = 1480字节
1480不是8的倍数？1480 / 8 = 185，是整数，OK

分片1：偏移=0，数据=1480，MF=1，总长度=1500
分片2：偏移=1480/8=185，数据=1480，MF=1，总长度=1500
分片3：偏移=(1480+1480)/8=370，数据=3800-1480-1480=840，MF=0，总长度=860

验证：1480 + 1480 + 840 = 3800 = 原始数据 ✓

### 重组判断
- 到达目的主机后，根据标识相同+偏移顺序重组
- 重组条件：所有MF=0的分片已到达（最后一个分片MF=0）

## 直观理解
- IP分片像"寄大件快递被拆成几个小包裹"——每个包裹上有编号和位置标记
- 收件人（目的主机）按编号和位置重新组装

## 协议关联
- TCP在传输层避免分片：MSS协商确保每个TCP段不超过MTU-40
- UDP不提供分片避免机制，大UDP数据报可能导致IP分片
- 路径MTU发现（PMTUD）：源端通过ICMP"数据报过大"消息动态发现最小MTU
