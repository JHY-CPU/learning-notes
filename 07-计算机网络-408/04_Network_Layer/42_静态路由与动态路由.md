# 42_静态路由与动态路由

## 核心概念

- **静态路由（Static Routing）**：
  - 管理员手动配置路由表条目
  - 适用于小型、稳定的网络
  - 不需要路由协议开销
  - 网络变化时需手动更新

- **动态路由（Dynamic Routing）**：
  - 路由器通过路由协议自动学习和更新路由
  - 适用于大型、变化的网络
  - 自动适应网络拓扑变化
  - 占用带宽和CPU资源

- **默认路由（Default Route）**：
  - 目的网络：0.0.0.0/0
  - 匹配所有目的地址
  - 优先级最低（最长前缀匹配中/0最短）
  - 通常用在边界路由器上，指向ISP

- **408考点**：
  - 静态路由和动态路由的特点和适用场景
  - 默认路由的配置和作用

## 原理分析

### 路由表条目类型

| 类型 | 目的网络 | 优先级 | 用途 |
|------|---------|--------|------|
| 特定主机路由 | x.x.x.x/32 | 最高 | 到特定主机的路由 |
| 特定网络路由 | x.x.x.x/n | 中等 | 到特定网络的路由 |
| 默认路由 | 0.0.0.0/0 | 最低 | 兜底路由 |

### 静态路由配置示例

```
ip route 192.168.2.0 255.255.255.0 10.0.0.2
（目的网络）（子网掩码）（下一跳）
```

### 动态路由协议对比

| 特性 | RIP | OSPF | BGP |
|------|-----|------|-----|
| 类型 | 距离向量 | 链路状态 | 路径向量 |
| 范围 | IGP | IGP | EGP |
| 度量 | 跳数 | 代价（带宽） | 策略 |
| 收敛速度 | 慢 | 快 | 中等 |
| 适用规模 | 小型网络 | 大型网络 | ISP之间 |
| 协议 | UDP 520 | IP 89 | TCP 179 |

### 静态 vs 动态选择

**使用静态路由的场景**：
- 小型办公室/家庭网络
- 末梢网络（stub network，只有一个出口）
- 默认路由指向ISP
- 需要精确控制路由的场景

**使用动态路由的场景**：
- 大型企业网络
- 有多条冗余路径
- 网络拓扑经常变化
- 需要自动故障切换

## 直观理解

**类比**：
- **静态路由** = 提前打印好的固定路线图
  - 简单直接，但不能应对变化
  - 适合熟悉的短途路线

- **动态路由** = 实时GPS导航
  - 自动更新路况，自动重新规划
  - 适合复杂多变的长途路线

**记忆口诀**：
- "小网用静态，大网用动态"
- "末梢默认指ISP，冗余拓扑靠协议"

## 协议关联

### 静态路由与默认路由
- 默认路由是特殊的静态路由（0.0.0.0/0）
- 末梢网络的出口路由器通常只配默认路由
- 可以减少路由表条目数量

### 动态路由协议分类
- 按范围：IGP（RIP, OSPF） vs EGP（BGP）
- 按算法：距离向量（RIP） vs 链路状态（OSPF） vs 路径向量（BGP）
- 按使用场景：AS内（IGP） vs AS间（BGP）

### 考试要点
- 静态路由不随网络变化自动更新
- 动态路由协议会占用网络带宽和路由器CPU
- 默认路由用于简化路由表
- 路由表中同时存在静态和动态条目时，按优先级选择
