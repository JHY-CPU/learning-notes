# 04_成帧_概念_考试

## 核心概念

- **成帧的关键考点**：四种成帧方法的对比、零比特填充的计算、透明传输的理解
- **帧定界方法的演变**：字符计数法(不可靠) -> 字符填充法(复杂) -> 零比特填充法(最常用)
- **408真题常考形式**：给数据求填充结果、判断帧边界、MTU与分片

### 零比特填充关键数据

| 项目 | 值 |
|------|-----|
| 标志字段 | 01111110（0x7E） |
| 规则 | 数据中每连续5个1插入1个0 |
| 应用协议 | PPP、HDLC |

### 透明传输的本质

- 数据中可以包含**任意比特组合**，包括与标志相同的01111110
- 通过填充技术确保标志字段只出现在帧的边界
- 透明 = 对上层透明，上层无需关心成帧细节

### 帧长度约束

| 协议 | 最大载荷 | 最小载荷 | 最小帧长 |
|------|---------|---------|---------|
| 以太网 | 1500B | 46B | 64B |
| PPP | 1500B | 无限制 | 不确定 |

## 原理分析

### 典型计算题：零比特填充

**例题1**：数据部分为 `11010111110111111010`，求零比特填充后的结果。

分析：
1. 扫描数据找到连续5个1的位置
2. `11010` **11111** `0` `11111` `0` `10` → 找到两处连续5个1
3. 在每处5个1后面插入0

填充后：`110101111100111110010`

**例题2**：接收方收到 `01111110 0001111101111101100000 01111110`，还原数据载荷。

分析：
1. 去掉首尾标志：`0001111101111101100000`
2. 遇连续5个1，删除后面的0：
   - `00011111` 删除0 → `00011111`
   - `011111` 删除0 → `011111`
3. 原始数据：`000111110111111100000`

### 以太网前导码

- **前导码（Preamble）**：7字节，每字节为 `10101010`，用于时钟同步
- **帧起始定界符（SFD）**：1字节，值为 `10101011`，标识帧的开始
- 以太网帧**没有结束标志**，靠帧间间隔来区分

$$\text{以太网帧} = \text{前导码(7B)} + \text{SFD(1B)} + \text{目的地址(6B)} + \text{源地址(6B)} + \text{类型(2B)} + \text{数据(46-1500B)} + \text{FCS(4B)}$$

### MTU与IP分片

- IP数据报最大长度：65535字节
- 以太网MTU：1500字节
- 每个分片的数据部分必须是**8字节的倍数**（除最后一个分片）

$$\text{分片数} = \lceil \frac{\text{IP数据报总长度} - \text{IP首部长度}}{\text{MTU} - \text{IP首部长度}} \rceil$$

## 直观理解

- **零比特填充**的比喻：在一串珠子里，每看到5颗红珠子就插1颗透明珠子，收方把透明珠子去掉
- **字符计数法的缺陷**：就像用数字标注每段话的字数，一旦某个数字错了，后面所有段落全乱了
- **MTU**就像高速公路的限高杆：超过限制高度的车辆不能直接通过，需要分成多辆车运输

## 协议关联

### 成帧方法与协议的对应

```
字符计数法 ──> 基本不用（不可靠）
字符填充法 ──> BSC协议（早期IBM）
零比特填充 ──> PPP、HDLC（点对点链路）
违规编码法 ──> 以太网（曼彻斯特编码）
```

### 408常见陷阱

1. **陷阱1**："帧"是数据链路层的PDU，不是网络层的
2. **陷阱2**：零比特填充是**在数据中**插入0，不是在标志字段中
3. **陷阱3**：以太网帧的前导码**不属于帧的一部分**，物理层添加
4. **陷阱4**：MTU限制的是**数据载荷**，不是整个帧的长度

### 真题回顾

**2015年408真题**：在数据链路层，若数据中出现与帧定界符相同的比特组合，为了实现透明传输，可采用的方法是（ ）

A. 字符计数法  B. 字符填充法  C. 零比特填充法  D. 以上都可以

答案：B和C（取决于使用的编码方式）
