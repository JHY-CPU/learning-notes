# 26_可靠传输机制总结

## 核心概念

- **可靠传输**：保证数据无差错、不丢失、不重复、按序到达的传输服务
- **三大机制**：确认（ACK）、超时重传、序号（Sequence Number）
- **实现位置**：数据链路层和传输层都提供可靠传输

### 可靠传输的三个基本要素

| 要素 | 作用 | 实现 |
|------|------|------|
| 序号 | 防止重复、检测失序 | 每帧/每字节编号 |
| 确认 | 告知发送方已正确接收 | ACK帧/TCP段 |
| 超时重传 | 处理帧/段丢失 | 定时器 |

### 可靠传输的实现方式对比

| 方式 | 检错方法 | 纠错方法 | 适用场景 |
|------|---------|---------|---------|
| CRC + ARQ | CRC检错 | 重传 | 数据链路层 |
| 海明码 | 海明校验 | 直接纠错 | 内存ECC |
| TCP校验和 + ARQ | 校验和 | 重传 | 传输层 |

### ARQ的三种实现

1. **停等ARQ**：窗口=1，最简单
2. **后退N帧ARQ（GBN）**：累积确认，重传所有
3. **选择重传ARQ（SR）**：逐一确认，只重传出错的

### 确认的类型

| 类型 | 说明 | 应用 |
|------|------|------|
| 逐一确认 | 每帧一个ACK | SR |
| 累积确认 | ACKn表示n及之前都收到了 | GBN、TCP |
| 延迟确认 | 收到后等一段时间再ACK | TCP |
| 否定确认（NAK） | 通知发送方某帧出错 | 选择性使用 |

### 无差错情况下的效率

- 停等：$U = \frac{1}{1+2a}$
- GBN/SR：$U = \frac{W_s}{1+2a}$（以$T_f$为单位）

其中 $a = T_p / T_f$

## 原理分析

### 可靠传输的状态机模型

**发送方**：

```
[空闲] --数据到达--> [发送帧,启动定时器]
  ↑                       |
  |                  收到ACK?
  |                 /       \
  |          超时/NAK     正确ACK
  |             |            |
  |      [重传帧]      [停止定时器]
  |             |            |
  +-------------+            +---> [空闲]
```

### 滑动窗口的统一模型

$$\text{停等} = \text{GBN}(W_s=1) = \text{SR}(W_s=1, W_r=1)$$

$$\text{GBN} = \text{滑动窗口}(W_r=1)$$

$$\text{SR} = \text{滑动窗口}(W_r>1)$$

### 有差错时的吞吐量分析

假设帧出错概率为 $p$：

**停等协议**：
$$\text{期望传输次数} = \frac{1}{1-p}$$

$$U_{\text{有差错}} = U_{\text{无差错}} \times (1-p)$$

**GBN协议**：

当帧i出错时，后续所有帧都需要重传。帧出错导致的额外开销更大。

## 直观理解

- **可靠传输就像可靠的快递服务**：每件快递有单号（序号），收件人签收（确认），丢了就补发（超时重传）
- **三种ARQ就像三种补发策略**：停等是"等你签收了再发下一件"；GBN是"一件丢了，从那件开始后面全重发"；SR是"只补发丢的那件"

## 协议关联

- 数据链路层：CRC + ARQ（停等/GBN/SR）
- 传输层（TCP）：校验和 + 滑动窗口 + 累积确认 + SACK
- 可靠传输和流量控制经常**联合考察**
- 408考试中，可靠传输机制常作为综合题的**一部分**
