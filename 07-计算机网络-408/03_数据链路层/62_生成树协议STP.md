# 62_生成树协议STP

## 核心概念

- **STP（Spanning Tree Protocol）**：生成树协议，IEEE 802.1D标准
- **目的**：在有冗余链路的交换网络中**消除环路**
- **核心思想**：通过阻塞某些端口，将物理拓扑变为逻辑上的树形无环拓扑

### STP解决的问题

交换网络中的环路会导致：
1. **广播风暴**：广播帧在环中无限循环
2. **MAC地址表震荡**：同一MAC从不同端口学来
3. **帧的重复接收**：同一帧被多次收到

### STP的工作原理

1. **选举根网桥（Root Bridge）**：Bridge ID最小的交换机
2. **选举根端口（Root Port）**：每个非根网桥到根网桥的最优端口
3. **选举指定端口（Designated Port）**：每个网段到根网桥的最优端口
4. **阻塞非指定端口**：其余端口进入阻塞状态

### Bridge ID

$$\text{Bridge ID} = \text{优先级(2字节)} + \text{MAC地址(6字节)}$$

优先级默认32768，越小越优先。

### 端口状态

| 状态 | 说明 | 时长 |
|------|------|------|
| 阻塞（Blocking） | 不转发帧，只接收BPDU | — |
| 侦听（Listening） | 不转发帧，参与计算 | 15秒 |
| 学习（Learning） | 学习MAC地址，不转发 | 15秒 |
| 转发（Forwarding） | 正常转发帧 | — |
| 禁用（Disabled） | 端口关闭 | — |

### 收敛时间

- 传统STP收敛时间：30-50秒
- RSTP（快速STP）：1-2秒

### BPDU（Bridge Protocol Data Unit）

- STP的控制报文
- 包含：Root ID、Root Path Cost、Bridge ID、Port ID
- 每2秒发送一次

### 考试重点

- STP的目的和工作原理
- 根网桥、根端口、指定端口的选举
- 端口状态和收敛时间

## 原理分析

### 根网桥选举

所有交换机比较Bridge ID，最小的成为根网桥。

如果优先级相同，比较MAC地址。

### 根端口选举

每个非根网桥选择一个根端口（到根网桥最优路径的端口）：

1. 最小Root Path Cost
2. 最小上游Bridge ID
3. 最小上游Port ID

### 指定端口选举

每个网段选择一个指定端口：

1. 到根网桥路径开销最小的端口
2. Bridge ID较小的端口

### 路径开销

| 链路速率 | 开销（802.1D-1998） |
|---------|-------------------|
| 10 Mbps | 2,000,000 |
| 100 Mbps | 200,000 |
| 1 Gbps | 20,000 |
| 10 Gbps | 2,000 |

## 直观理解

- **STP就像"修剪树枝"**：有环路的网络就像一棵疯长的树，STP剪掉一些树枝（阻塞端口），让它变成一棵规整的树
- **根网桥**就像树的"树根"，所有数据沿着树枝流向树根再分发

## 协议关联

- STP是透明网桥（交换机）解决环路的核心协议
- RSTP（802.1w）和MSTP（802.1s）是STP的改进版本
- STP在每个VLAN上独立运行（PVST+）
- 408考试中STP一般考概念，不会深入计算
