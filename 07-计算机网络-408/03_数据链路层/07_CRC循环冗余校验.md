# 07_CRC循环冗余校验

## 核心概念

- **CRC（Cyclic Redundancy Check）**：循环冗余校验，数据链路层**最常用的检错方法**
- **原理**：基于多项式除法，将数据视为二进制多项式，用生成多项式去除，余数作为校验码
- **FCS（Frame Check Sequence）**：帧校验序列，即CRC计算得到的余数，附加在帧尾

### CRC的核心术语

| 术语 | 含义 | 长度 |
|------|------|------|
| 信息码 $M$ | 要发送的原始数据 | $k$ 位 |
| 生成多项式 $G(x)$ | 发送方和接收方约定的除数 | $r+1$ 位 |
| 余数 $R$ | CRC校验码（FCS） | $r$ 位 |
| 发送帧 | $M$ 后接 $R$ | $k+r$ 位 |

### 常用生成多项式

| 名称 | 多项式 | 二进制 | 用途 |
|------|--------|--------|------|
| CRC-8 | $x^8+x^2+x+1$ | 100000111 | 一般用途 |
| CRC-16 | $x^{16}+x^{15}+x^2+1$ | 11000000000000101 | 通信 |
| **CRC-32** | 多项式较长 | 33位 | **以太网、PPP** |

### CRC的关键性质

1. CRC只能**检错**，不能**纠错**
2. 能检测所有**奇数个**比特错误
3. 能检测所有**长度≤r**的突发差错
4. 能以 $1-2^{-r}$ 的概率检测长度 $>r$ 的突发差错

### 考试重点

- CRC计算是**408必考内容**
- 掌握**模2除法**（异或除法）
- 生成多项式的位数比余数位数多1

## 原理分析

### CRC计算步骤

**步骤1**：确定生成多项式 $G(x)$，设其为 $r+1$ 位

**步骤2**：在信息码 $M$ 后面添加 $r$ 个0，得到 $M'$

**步骤3**：用 $M'$ 除以 $G(x)$（模2除法），得到余数 $R$

**步骤4**：发送帧 = $M$ + $R$

### 模2除法的规则

- 不借位、不进位
- **相同得0，不同得1**（等价于异或运算）
- 每次用被除数的最高位与除数的最高位对齐

### 详细计算示例

**例题**：信息码 $M = 101001$，生成多项式 $G(x) = x^3 + x^2 + 1$

1. $G(x) = x^3 + x^2 + 1$ → 二进制为 `1101`（4位，$r=3$）
2. 在 $M$ 后加3个0：$M' = 101001000$
3. 模2除法：

```
          110100
    _____________
1101|101001000
     1101
     -----
      1110
      1101
      ----
       1110
       1101
       ----
        0110
        0000
        ----
         1100
         1101
         ----
          001  ← 余数 R = 001
```

4. 发送帧 = `101001` + `001` = `101001001`

### 接收方验证

接收方用收到的帧（含FCS）除以 $G(x)$：
- 余数为0 → 无差错
- 余数不为0 → 有差错

$$\text{验证}: M' \div G(x) = Q \cdots R$$

若 $R = 0$，则接收正确。

### 发送帧的数学表达

$$T(x) = M(x) \cdot x^r + R(x)$$

其中 $T(x)$ 能被 $G(x)$ 整除。

## 直观理解

- CRC就像**除法验算**：把数据除以一个约定的数，记下余数；接收方再除一次，看余数是否为0
- 类比：老师布置10道题，学生做完后把"题号之和"附在作业后面。老师重新加一遍，如果结果不同，说明作业被改过
- **模2除法**就像"异或版的长除法"：不进位不借位，相同为0不同为1

## 协议关联

### CRC在协议中的应用

| 协议 | CRC类型 | 校验范围 |
|------|---------|----------|
| 以太网 | CRC-32 | 除前导码和SFD外的整个帧 |
| PPP | CRC-32（FCS字段4字节） | 地址+控制+信息 |
| HDLC | CRC-16或CRC-32 | 地址+控制+信息 |

### CRC与校验和的区别

| 特性 | CRC | 校验和 |
|------|-----|--------|
| 计算方法 | 多项式除法 | 求和取反 |
| 检错能力 | 强 | 较弱 |
| 计算速度 | 硬件快 | 软件快 |
| 应用层 | 数据链路层 | 传输层（TCP/UDP） |

### 408常见陷阱

1. 生成多项式的位数 = 余数位数 + 1
2. 模2除法**没有借位和进位**，只做异或
3. 余数的位数必须等于 $r$，不足前面补0
4. CRC检错时，**整个帧**（含FCS）除以G(x)，余数应为0
