# 72_令牌环网

## 核心概念

- **令牌环网（Token Ring）**：IEEE 802.5标准，使用令牌传递的介质访问控制方法
- **核心思想**：只有一个令牌在网络中传递，持有令牌的站点才能发送数据
- **物理拓扑**：星形（实际）| **逻辑拓扑**：环形

### 令牌传递的工作原理

1. 令牌（特殊的帧）在环中依次传递
2. 某站有数据要发送时，**捕获令牌**
3. 将数据帧发送到环上
4. 数据帧沿环传递，目的站**复制**数据并设置确认位
5. 数据帧回到发送站，发送站**移除**数据帧
6. 释放令牌，下一个站可以使用

### 令牌环网的特点

| 特性 | 说明 |
|------|------|
| MAC协议 | 令牌传递 |
| 逻辑拓扑 | 环形 |
| 物理拓扑 | 星形（通过MAU） |
| 速率 | 4/16 Mbps |
| 介质 | 屏蔽/非屏蔽双绞线 |
| 优先级 | 支持（优先级高的站先获令牌） |

### 令牌的格式

令牌 = 3字节：
```
| SD(1B) | AC(1B) | ED(1B) |
```

- SD：起始定界符
- AC：访问控制（包含优先级和令牌位）
- ED：结束定界符

### 令牌环网 vs 以太网

| 特性 | 令牌环网 | 以太网 |
|------|---------|--------|
| MAC协议 | 令牌传递 | CSMA/CD |
| 逻辑拓扑 | 环形 | 总线/星形 |
| 冲突 | 无 | 有 |
| 确定性 | 是（最坏情况可计算） | 否 |
| 实现复杂度 | 高 | 低 |
| 现状 | 淘汰 | 主流 |

- 令牌环网已被以太网取代

## 原理分析

### 令牌的捕获与释放

**空闲令牌**：AC字段的令牌位=0
**忙令牌（数据帧）**：AC字段的令牌位=1

捕获过程：
1. 站点收到空闲令牌
2. 将令牌位改为1（变为数据帧）
3. 在帧后附加数据
4. 发送到环上

释放过程：
1. 数据帧回到发送站
2. 发送站检查确认位
3. 从环上移除数据帧
4. 释放空闲令牌

### 优先级机制

- 令牌有优先级字段（3位，0-7）
- 只有优先级 ≥ 令牌优先级的站才能捕获令牌
- 站点可以预约下一个令牌的优先级

### 令牌维护

- **监控站（Active Monitor）**：负责维护令牌
- 如果令牌丢失，监控站负责生成新令牌
- 如果帧在环上循环（发送站故障），监控站负责移除

## 直观理解

- **令牌环网就像"传话游戏中的发言权"**：只有拿到"发言棒"的人才能说话，说完后传给下一个人
- **优势**：不会冲突（每个人都必须等发言棒）
- **劣势**：如果发言棒丢了或某人拿着不放，整个系统瘫痪

## 协议关联

- 令牌环网已被以太网全面取代
- 令牌传递的思想在光纤通道（Fibre Channel）中仍有应用
- FDDI使用类似的令牌传递机制
- 408考试中令牌环网一般只考概念，不会深入
