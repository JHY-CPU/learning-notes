# 36_CSMA-CD协议

## 核心概念

- **CSMA/CD（Carrier Sense Multiple Access with Collision Detection）**：带冲突检测的载波侦听多路访问
- **以太网的核心MAC协议**：用于有线局域网
- **核心思想**：先听后说，边说边听（Listen Before Talk, Listen While Talk）

### CSMA/CD的工作流程

1. **载波侦听**：发送前监听信道，如果忙则等待
2. **发送数据**：信道空闲时开始发送
3. **冲突检测**：发送过程中持续检测冲突
4. **冲突处理**：检测到冲突立即停止发送，发送干扰信号（Jam Signal）
5. **二进制指数退避**：随机等待后重新尝试

### 关键参数：争用期（Contention Slot）

$$\tau = \text{单向传播时延}$$

$$2\tau = \text{往返传播时延} = \text{争用期} = \text{冲突窗口}$$

- 发送方在 $2\tau$ 时间内没有检测到冲突，就认为发送成功
- **最迟多久能检测到冲突**？$2\tau$（信号到达最远点并返回冲突信号）

### 以太网的 $2\tau$ 规定

| 以太网类型 | 数据率 | 最大距离 | $2\tau$ |
|-----------|--------|---------|---------|
| 10BASE5 | 10 Mbps | 2500m | 51.2μs |
| 10BASE-T | 10 Mbps | 100m | 5.12μs |
| 100BASE-TX | 100 Mbps | 100m | 0.512μs |

### 最小帧长约束

为了保证在检测到冲突之前帧还没有发完：

$$L_{\min} = 2\tau \times R$$

其中 $R$ 为数据率。

以太网（10Mbps，2500m）：$L_{\min} = 51.2\mu s \times 10^7 = 512$ bit = 64 byte

### 二进制指数退避

- 第 $k$ 次重传时，从 $\{0, 1, 2, ..., 2^k - 1\}$ 中随机选择一个值 $r$
- 等待 $r \times 51.2\mu s$ 后重传
- $k$ 最大为10（之后 $2^k$ 不再增大，最大1023）
- 超过16次重传则放弃

### 考试重点

- $2\tau$ 的含义和计算是**必考内容**
- 最小帧长的推导是**高频考点**
- 二进制指数退避的过程

## 原理分析

### 冲突检测的时序

```
节点A                    最远点                    节点B
 |                         |                         |
 |---数据--->              |                         |
 |                    传播中...                       |
 t=0                    t=τ                        t=τ
 |                         | <---B也开始发送！       |
 |                         |---冲突!--->             |
 | <---冲突信号            |                    t=2τ
 t=2τ
 
A在t=2τ时检测到冲突
```

### 为什么最小帧长 = $2\tau \times R$？

- 如果帧太短，在 $2\tau$ 之前就发完了
- 发完后就不监听了，无法检测到延迟到达的冲突信号
- 所以帧的发送时间必须 $\geq 2\tau$

$$L_{\min} \geq 2\tau \times R$$

### 二进制指数退避示例

第1次重传：随机选 $\{0, 1\}$ → 等待 0 或 1 个争用期
第2次重传：随机选 $\{0, 1, 2, 3\}$ → 等待 0-3 个争用期
第3次重传：随机选 $\{0, 1, ..., 7\}$ → 等待 0-7 个争用期
...
第10次重传：随机选 $\{0, 1, ..., 1023\}$
第11-16次重传：随机选 $\{0, 1, ..., 1023\}$（不再增大）
超过16次：放弃，报告上层

## 直观理解

- **CSMA/CD就像对讲机通话**：先听有没有人说话（载波侦听），说话时如果听到别人也在说（冲突检测），立即停止说"喂喂喂"（干扰信号），等随机时间再试
- **$2\tau$**就像"回声时间"：你说的话传到最远的墙再反射回来的时间，超过这个时间没听到回声说明没冲突
- **最小帧长**就像"最短说话时间"：必须说够一定时间，否则远处的冲突信号传回来时你已经闭嘴了，听不到

## 协议关联

- CSMA/CD是**以太网**（IEEE 802.3）的核心协议
- 全双工以太网不需要CSMA/CD（无冲突）
- CSMA/CD不适合无线网络（无法边发边收）
- 与CSMA/CA是"兄弟"协议，分别用于有线和无线
