# 43_以太网概述

## 核心概念

- **以太网（Ethernet）**：IEEE 802.3标准，最常用的局域网技术
- **发明者**：Bob Metcalfe（1973年，Xerox PARC）
- **核心特征**：总线拓扑（早期）→ 星形拓扑（现代），使用CSMA/CD协议

### 以太网的发展历程

| 年代 | 标准 | 速率 | 特点 |
|------|------|------|------|
| 1980 | DIX Ethernet | 10 Mbps | 粗同轴电缆 |
| 1983 | IEEE 802.3 | 10 Mbps | 标准化 |
| 1995 | 802.3u | 100 Mbps | 快速以太网 |
| 1998 | 802.3z/ab | 1 Gbps | 千兆以太网 |
| 2002 | 802.3ae | 10 Gbps | 万兆以太网 |
| 2010+ | 802.3ba | 40/100 Gbps | 超高速 |

### 以太网的命名规则

| 名称 | 含义 |
|------|------|
| 10BASE5 | 10Mbps，基带传输，最大500m |
| 10BASE2 | 10Mbps，基带传输，最大185m（约200m） |
| 10BASE-T | 10Mbps，基带传输，双绞线 |
| 100BASE-TX | 100Mbps，基带传输，双绞线 |
| 1000BASE-T | 1Gbps，基带传输，双绞线 |

### 以太网的拓扑演变

```
总线拓扑(10BASE5/2) → 集线器星形 → 交换机星形
  共享介质              共享冲突域      独立冲突域
```

### 以太网的特性

- **无连接**：发送前不建立连接
- **不可靠**：不提供确认，帧可能丢失（靠上层处理）
- **CSMA/CD**：共享介质时使用（全双工不需要）
- **曼彻斯特编码**：每个比特中间都有跳变

### 考试重点

- 以太网的命名规则
- 以太网是**无连接不可靠**的服务
- 从共享式到交换式的演进

## 原理分析

### 以太网提供的服务

以太网向网络层提供的是**无连接不可靠的尽最大努力交付**服务：

- **无连接**：不建立连接，直接发送
- **不可靠**：不发ACK，不保证帧到达
- **差错检测**：使用CRC-32，但只丢弃错误帧，不重传
- **尽最大努力**：尽量把帧送到，不行就算了

### 曼彻斯特编码

```
数据:    1    0    1    0    0    1
编码:   ──┐  ┌──┐  ──┐  ┌──┐  ┌──┐  ──
         └──┘  └──┘  └──┘  └──┘  └──┘
```

每个比特中间有一次电平跳变：
- 1：从高到低跳变
- 0：从低到高跳变

优点：自带时钟信号，便于同步。

## 直观理解

- **以太网就像"公共广播系统"**：早期所有人都在同一条线路上（总线），谁想说先听有没有人说。现在有了交换机，每个人有独立的线路
- **无连接不可靠**就像"扔飞镖"：不管靶子在不在，直接扔过去，没扔中拉倒

## 协议关联

- 以太网是数据链路层和物理层的协议
- 向网络层提供简单的帧传输服务
- 交换机的发展使以太网从半双工变为全双工
- 以太网的帧格式是408考试的重点
