# 86_随机访问协议总结

## 核心概念

- **随机访问协议**：多个节点竞争共享信道，不保证无冲突
- **代表协议**：ALOHA、CSMA、CSMA/CD、CSMA/CA
- **演进方向**：从"盲发"到"先听后说"到"边说边听"到"避免冲突"

### 演进路线

```
纯ALOHA(想发就发, 18.4%)
    ↓ 改进
时隙ALOHA(定时发, 36.8%)
    ↓ 加载波侦听
CSMA(先听后说, ~50-80%)
    ↓ 加冲突检测
CSMA/CD(边说边听, ~90%+)
    ↓ 无线场景无法CD
CSMA/CA(避免冲突, ~50-70%)
```

### 各协议最大吞吐率

| 协议 | 最大吞吐率 | 冲突窗口 |
|------|-----------|---------|
| 纯ALOHA | 18.4% | $2T$ |
| 时隙ALOHA | 36.8% | $T$ |
| 非坚持CSMA | ~80-90% | $T_p$ |
| 1-坚持CSMA | ~53% | $T_p$ |
| CSMA/CD | ~90%+ | $2\tau$ |
| CSMA/CA | ~50-70% | — |

### 各协议的特点

| 协议 | 载波侦听 | 冲突检测 | 退避 | 确认 |
|------|---------|---------|------|------|
| ALOHA | 无 | 无 | 随机 | 超时 |
| CSMA | 有 | 无 | 随机 | 超时 |
| CSMA/CD | 有 | 有 | 二进制指数 | 不需要 |
| CSMA/CA | 有 | 无 | 二进制指数 | 需要ACK |

### 考试重点

- 各协议的演进关系
- 最大吞吐率（必背数字）
- CSMA/CD和CSMA/CA的区别

## 原理分析

### 为什么CSMA/CD效率高于ALOHA？

1. **载波侦听**：减少了"盲发"，降低了冲突概率
2. **冲突检测**：冲突后立即停止，减少了信道浪费时间
3. **冲突窗口缩小**：从 $T$ 或 $2T$ 缩小到 $2\tau$（通常远小于 $T$）

### 随机访问 vs 受控访问

| 特性 | 随机访问 | 受控访问 |
|------|---------|---------|
| 冲突 | 有 | 无 |
| 轻负载效率 | 高 | 低（令牌传递） |
| 重负载效率 | 低 | 高 |
| 实现复杂度 | 低 | 高 |
| 代表 | CSMA/CD | 令牌传递 |

## 直观理解

- **随机访问协议的演进**就像"人类文明的进步"：从原始社会的"想说就说"（ALOHA），到文明社会的"先听后说"（CSMA），到现代社会的"遵守规则"（CSMA/CD/CA）

## 协议关联

- 随机访问协议是数据链路层MAC子层的核心
- 以太网和WiFi分别使用CSMA/CD和CSMA/CA
- 408考试中随机访问协议是**数据链路层的重点**
