# 39_CSMA-CA协议

## 核心概念

- **CSMA/CA（Carrier Sense Multiple Access with Collision Avoidance）**：带冲突避免的载波侦听多路访问
- **用于无线局域网（WiFi/802.11）**
- **核心思想**：先听后说，**尽量避免冲突**（而不是检测冲突）

### 为什么无线网不用CSMA/CD？

1. **无法边发边收**：无线信号的能量差异大（发送信号>>接收信号），硬件无法同时收发
2. **隐藏终端问题**：A和C都在B的范围内，但A和C互相听不到，CSMA/CD无法检测到冲突
3. **暴露终端问题**：B向A发送，C听到B在发送就不敢向D发送，但实际上C→D不会与B→A冲突

### CSMA/CA的工作流程

1. **载波侦听**：监听信道是否空闲
2. **信道空闲**：等待一个**DIFS**时间后发送
3. **信道忙**：执行**二进制指数退避**
4. **可选确认**：接收方发送ACK（SIFS后）

### 帧间间隔（IFS）

| IFS类型 | 长度 | 用途 |
|---------|------|------|
| SIFS | 最短 | ACK、CTS |
| PIFS | 中等 | 轮询 |
| DIFS | 最长 | 数据帧 |

优先级：SIFS < PIFS < DIFS

### RTS/CTS机制（可选）

1. 发送方先发送**RTS**（Request To Send）
2. 接收方回复**CTS**（Clear To Send）
3. 收到CTS后发送数据
4. 接收方发送ACK

**NAV（Network Allocation Vector）**：RTS/CTS中携带的持续时间，其他节点据此推迟发送。

### CSMA/CA vs CSMA/CD

| 特性 | CSMA/CD | CSMA/CA |
|------|---------|---------|
| 应用 | 有线以太网 | 无线WiFi |
| 冲突处理 | 检测后处理 | 尽量避免 |
| 边发边收 | 能 | 不能 |
| 确认机制 | 不需要ACK | 需要ACK |
| RTS/CTS | 无 | 可选 |

### 考试重点

- CSMA/CA与CSMA/CD的**区别**是必考内容
- 隐藏终端和暴露终端问题的理解
- RTS/CTS机制的目的

## 原理分析

### 隐藏终端问题

```
A ---------> B <--------- C
A和C互相听不到，都以为信道空闲，同时向B发送，B处发生冲突
```

解决方案：RTS/CTS

- A发送RTS给B
- B回复CTS（A和C都能收到）
- C收到CTS后知道B正忙，推迟发送

### 暴露终端问题

```
B ---------> A     C ---------> D
C听到B在发送，以为不能发送给D。但实际上B→A不会干扰C→D
```

CSMA/CA通过RTS/CTS解决：C听到B的RTS但没听到A的CTS（C不在A范围内），知道可以向D发送。

### 退避机制

- 初始竞争窗口：CW = $CW_{\min}$（通常31）
- 随机退避时间：$\text{Random}(0, CW) \times \text{SlotTime}$
- 冲突后窗口翻倍：$CW = \min(2 \times CW + 1, CW_{\max})$
- 最大重传次数：短帧7次，长帧4次

## 直观理解

- **CSMA/CD就像面对面交谈**：能同时说和听，发现冲突马上停
- **CSMA/CA就像对讲机**：不能同时收发，只能先确认"可以讲了"再说
- **RTS/CTS就像"借用信道公告"**：先广播"我要用了"，别人都知道并让路

## 协议关联

- CSMA/CA是**802.11 WiFi**的MAC协议
- 与CSMA/CD是"同族兄弟"，但适用场景不同
- WiFi的确认机制（ACK）比以太网更必要（无线信道更不可靠）
- 408考试中CSMA/CD和CSMA/CA的对比是**经典考点**
