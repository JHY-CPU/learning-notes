# 02_链路与数据链路

## 核心概念

- **链路（Link）**：从一个节点到相邻节点的**物理线路**（有线或无线），中间没有任何其他交换节点，也称**物理链路**
- **数据链路（Data Link）**：在物理链路的基础上，加上**通信协议**和必要的**控制规程**，形成逻辑上的数据通路
- **关键区别**：链路是硬件概念，数据链路是硬件+软件的综合体

### 链路的类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 点对点链路 | 两个节点之间的专用链路 | PPP拨号、广域网专线 |
| 广播链路 | 多个节点共享同一信道 | 以太网、WiFi |

### 点对点链路特点

- 只有**两个节点**，不存在介质访问控制问题
- 需要解决：成帧、差错控制、流量控制
- 典型协议：**PPP**、**HDLC**

### 广播链路特点

- **多个节点**共享一条信道
- **核心问题**：谁在什么时候有权发送？（MAC问题）
- 典型协议：**CSMA/CD**（以太网）、**CSMA/CA**（WiFi）

### 帧（Frame）

- 数据链路层的**协议数据单元（PDU）**称为帧
- 帧 = 帧头 + 数据载荷 + 帧尾
- 帧头通常包含：目的地址、源地址、控制信息
- 帧尾通常包含：校验码（FCS）

### 考试重点

- 408常考"链路"与"数据链路"的**概念辨析**
- 数据链路 = 链路 + 协议 + 控制规程
- 点对点链路和广播链路对应的**协议不同**

## 原理分析

### 数据链路层的通信模型

```
节点A                                        节点B
┌──────────────┐                    ┌──────────────┐
│   网络层     │                    │   网络层     │
│  IP数据报    │                    │  IP数据报    │
├──────────────┤                    ├──────────────┤
│  数据链路层  │ ====== 帧 ======> │  数据链路层  │
│  成帧/检错   │                    │  解帧/检错   │
├──────────────┤                    ├──────────────┤
│   物理层     │ ====== 比特流 ==> │   物理层     │
└──────────────┘                    └──────────────┘
```

### 数据链路层协议的基本要素

1. **帧定界方法**：确定帧的开始和结束
2. **差错控制编码**：CRC校验码的生成多项式
3. **流量控制机制**：窗口大小、确认机制
4. **寻址方式**：MAC地址格式和长度

### 数据链路层向网络层提供的服务

$$\text{数据链路层服务} = \text{成帧} + \text{差错检测} + \text{流量控制} + \text{寻址}$$

### 可靠传输的条件

在数据链路层实现可靠传输需要：

1. **确认（ACK）**：接收方告知发送方已正确收到
2. **超时重传**：发送方在超时后重发未被确认的帧
3. **序号机制**：防止帧的重复或失序

$$\text{可靠传输} = \text{确认} + \text{超时重传} + \text{序号}$$

## 直观理解

- **链路**就像一条公路（物理存在的道路）
- **数据链路**就像公路上有了交通规则、红绿灯、交通标志后的**公路系统**
- 没有协议的链路就像没有交通规则的公路——虽然能走车，但无法有序通行

### 打电话的类比

- 链路 = 电话线（物理连接）
- 数据链路 = 电话线 + 通话规则（"听到请回答"、确认机制）
- 单有电话线不能保证有效通信，必须有通话协议

## 协议关联

### 与其他层的协作

| 层次 | 提供给上层的服务 | 数据链路层的角色 |
|------|-----------------|-----------------|
| 网络层 | IP数据报的路由和转发 | 提供相邻节点间可靠传输 |
| 物理层 | 比特流传输 | 将比特流组织成帧 |

### 常见考试陷阱

- 问"数据链路层的功能"时，**不要漏掉**：成帧、差错检测、流量控制、访问控制
- "链路"和"数据链路"是**不同概念**，考试常考区分
- 数据链路层的差错控制主要是**检错**（CRC），不是纠错（海明码也可用但较少）

### 点对点 vs 广播的协议选择

```
点对点链路 ──> PPP/HDLC ──> 无MAC问题
广播链路   ──> CSMA/CD ──> 需要MAC协议
           ──> CSMA/CA
```
