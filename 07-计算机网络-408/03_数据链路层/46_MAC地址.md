# 46_MAC地址.md

## 核心概念

- **MAC地址（Media Access Control Address）**：也叫物理地址、硬件地址，是网卡的唯一标识
- **长度**：48位（6字节），通常表示为12个十六进制数
- **全球唯一**：由IEEE统一分配，前3字节为OUI（厂商标识），后3字节由厂商分配

### MAC地址格式

```
XX:XX:XX:XX:XX:XX
如：00:1A:2B:3C:4D:5E
```

- 前3字节（24位）：**OUI（Organizationally Unique Identifier）**，IEEE分配给厂商
- 后3字节（24位）：由厂商自行分配

### MAC地址类型

| 类型 | 特征（第1字节最低位） | 说明 |
|------|---------------------|------|
| 单播 | 0 | 发往一个特定网卡 |
| 多播 | 1 | 发往一组网卡 |
| 广播 | 全1（FF:FF:FF:FF:FF:FF） | 发往所有网卡 |

### MAC地址 vs IP地址

| 特性 | MAC地址 | IP地址 |
|------|---------|--------|
| 层次 | 数据链路层 | 网络层 |
| 长度 | 48位 | 32位(IPv4)/128位(IPv6) |
| 分配 | 硬件出厂固化 | 可动态分配 |
| 可变性 | 理论上不变（可软件修改） | 可变 |
| 范围 | 局域网内有效 | 全局有效 |
| 作用 | 标识网卡 | 标识网络中的主机 |

### 广播地址

- 全F：`FF:FF:FF:FF:FF:FF`
- 交换机收到广播帧会**泛洪**（向除接收端口外的所有端口转发）
- ARP请求使用广播地址

### 考试重点

- MAC地址48位，6字节
- 区分单播、多播、广播地址
- MAC地址和IP地址的对比
- ARP协议通过IP地址获取MAC地址

## 原理分析

### MAC地址的帧发送

| 目的地址类型 | 交换机行为 |
|-------------|----------|
| 单播 | 查MAC表，从对应端口转发 |
| 广播 | 泛洪（所有端口） |
| 多播 | 泛洪或精确转发（取决于配置） |

### ARP协议（地址解析）

```
主机A想知道主机B的MAC地址：
A发送ARP请求（广播）：谁的IP是192.168.1.2？请告诉我你的MAC地址！
B收到后发送ARP回复（单播）：我的MAC是00:1A:2B:3C:4D:5E
```

### MAC地址表的学习

交换机通过**源MAC地址学习**：

1. 收到帧，读取源MAC地址和入端口
2. 记录：MAC地址 → 端口 的映射
3. 查目的MAC地址，决定转发端口

## 直观理解

- **MAC地址就像身份证号**：出厂时确定，全球唯一
- **IP地址就像手机号**：可以换，但在当前"网络圈子"里用来找你
- **ARP就像问路**：知道名字（IP），问周围的人"他住在哪里"（MAC）

## 协议关联

- MAC地址是数据链路层的寻址方式
- IP地址是网络层的寻址方式
- ARP是两者的桥梁（网络层协议，但工作在数据链路层）
- 交换机基于MAC地址转发，路由器基于IP地址转发
