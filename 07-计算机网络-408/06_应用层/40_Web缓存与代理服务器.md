# 40_Web缓存与代理服务器

## 核心概念

- **Web缓存（Web Cache）**：存储Web对象的副本，减少重复请求
- **代理服务器（Proxy Server）**：位于客户端和服务器之间的中间服务器
  - **正向代理**：代理客户端请求（客户端配置代理）
  - **反向代理**：代理服务器响应（如负载均衡、CDN）
  - **透明代理**：客户端不知道代理的存在
- **缓存验证**：使用条件GET验证缓存是否过期
  - `If-Modified-Since`：基于时间的验证
  - `If-None-Match`：基于ETag的验证
- **408考试重点**：Web缓存的作用、条件GET、代理服务器类型

## 原理分析

### Web缓存的工作原理

1. **缓存命中（Cache Hit）**：
   - 客户端请求的对象在缓存中
   - 缓存直接返回对象
   - 不需要访问源服务器

2. **缓存未命中（Cache Miss）**：
   - 客户端请求的对象不在缓存中
   - 缓存向源服务器请求对象
   - 缓存存储对象副本
   - 缓存返回对象给客户端

3. **缓存验证（Conditional GET）**：
   - 缓存中的对象可能已过期
   - 客户端使用条件GET验证缓存
   - 服务器返回304（未修改）或200（已修改）

### 条件GET

**基于时间的验证**：
```
GET /index.html HTTP/1.1
If-Modified-Since: Wed, 01 Jan 2025 00:00:00 GMT
```

**服务器响应**：
- 如果未修改：`HTTP/1.1 304 Not Modified`（无正文）
- 如果已修改：`HTTP/1.1 200 OK`（返回新内容）

**基于ETag的验证**：
```
GET /index.html HTTP/1.1
If-None-Match: "abc123"
```

**服务器响应**：
- ETag匹配：`HTTP/1.1 304 Not Modified`
- ETag不匹配：`HTTP/1.1 200 OK`（返回新内容和新ETag）

### 代理服务器类型

| 类型 | 客户端配置 | 服务器感知 | 用途 |
|------|-----------|-----------|------|
| 正向代理 | 需要配置 | 不知道 | 翻墙、缓存 |
| 反向代理 | 不需要配置 | 配置代理 | 负载均衡、CDN |
| 透明代理 | 不需要配置 | 不知道 | 内容过滤 |

### 缓存带来的好处

1. **减少延迟**：缓存命中直接返回，延迟低
2. **减少带宽**：减少网络传输的数据量
3. **减轻服务器负担**：减少服务器的请求量
4. **提高可用性**：源服务器故障时，缓存仍可提供服务

## 直观理解

**Web缓存就像图书馆的热门书架**：
- 热门书放在门口的书架上（缓存）
- 读者先去热门书架找（缓存查询）
- 找到了直接借走（缓存命中）
- 没找到再去书库找（缓存未命中）
- 找到后放到热门书架（缓存存储）

**记忆技巧**：
- 缓存命中 = "直接返回"
- 缓存未命中 = "去服务器拿"
- 条件GET = "验证缓存是否过期"
- 304 = "没修改，用缓存"

## 协议关联

- **Web缓存与HTTP**：条件GET是HTTP缓存验证机制
- **Web缓存与CDN**：CDN是分布式Web缓存系统
- **Web缓存与代理服务器**：代理服务器通常包含缓存功能
- **408考点**：
  - Web缓存的作用
  - 条件GET的工作机制
  - 304状态码的含义
  - 正向代理与反向代理的区别
- **陷阱**：条件GET不减少首次请求的延迟，只减少后续请求的延迟
