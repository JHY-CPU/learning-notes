# 30_HTTP报文格式_考试

## 核心概念

- **408常考题型**：选择题为主，涉及HTTP报文格式、请求/响应结构
- **关键考点**：
  - HTTP报文四部分：起始行、头部、空行、正文
  - 请求起始行：方法 + URL + 版本
  - 响应起始行：版本 + 状态码 + 原因短语
  - CRLF（\r\n）作为行结束符
  - 空行（\r\n\r\n）分隔头部和正文
- **易混淆点**：
  - 请求报文和响应报文起始行格式不同
  - HTTP头部与MIME格式的关系

## 原理分析

### 典型考题1：报文格式

**题目**：HTTP请求报文的起始行格式是（  ）
A. 版本 状态码 原因短语
B. 方法 URL 版本
C. URL 方法 版本
D. 版本 方法 URL

**答案**：B

**解析**：
- 请求起始行：`GET /index.html HTTP/1.1`
- 响应起始行：`HTTP/1.1 200 OK`
- 顺序不能颠倒

### 典型考题2：头部与正文分隔

**题目**：HTTP报文中头部与正文的分隔使用（  ）
A. 单个换行符
B. 空行（两个连续的CRLF）
C. 特殊字符
D. 没有分隔符

**答案**：B

**解析**：
- HTTP报文使用CRLF（\r\n）作为行结束
- 头部与正文之间用空行（\r\n\r\n）分隔
- 这是HTTP报文的标准格式

### 典型考题3：HTTP头部

**题目**：HTTP/1.1请求报文中必须包含的头部字段是（  ）
A. User-Agent
B. Accept
C. Host
D. Cookie

**答案**：C

**解析**：
- HTTP/1.1要求请求中必须包含Host头部
- User-Agent、Accept、Cookie都是可选的
- Host头部用于虚拟主机支持

### 典型考题4：响应报文分析

**题目**：HTTP响应报文`HTTP/1.1 200 OK`中，200表示（  ）
A. 请求方法
B. 状态码
C. 端口号
D. 版本号

**答案**：B

**解析**：
- 响应起始行格式：版本 状态码 原因短语
- HTTP/1.1是版本，200是状态码，OK是原因短语
- 状态码表示请求的处理结果

## 直观理解

**做题技巧**：
- 请求起始行 = 方法 + URL + 版本（记住"方法URL版本"）
- 响应起始行 = 版本 + 状态码 + 原因（记住"版本状态原因"）
- 空行分隔头部和正文（两个CRLF）
- Host头部在HTTP/1.1中是必需的

**常见错误**：
- 混淆请求和响应起始行的顺序
- 忘记空行分隔
- 不知道Host头部在HTTP/1.1中是必需的

## 协议关联

- **HTTP报文与TCP**：HTTP报文通过TCP传输
- **HTTP报文与MIME**：HTTP使用MIME格式的Content-Type
- **HTTP报文与Cookie**：Cookie通过HTTP头部传递
- **408常见组合**：HTTP报文格式 + TCP传输 = 综合分析
