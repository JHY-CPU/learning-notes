# 72_应用层安全协议

## 核心概念

- **应用层安全协议**：保护应用层数据的安全
- **常见安全协议**：
  - **HTTPS**：安全HTTP（HTTP + SSL/TLS）
  - **SFTP/FTPS**：安全文件传输
  - **SSH**：安全远程登录
  - **S/MIME**：安全邮件
  - **PGP**：邮件加密
- **安全三要素**：
  - **机密性**：数据加密，防止窃听
  - **完整性**：数据不被篡改
  - **身份认证**：验证通信方身份
- **408考试重点**：HTTPS、SSH的基本概念和端口号

## 原理分析

### 应用层安全需求

| 应用 | 安全需求 | 安全协议 |
|------|---------|---------|
| Web | 机密性、完整性、认证 | HTTPS |
| 文件传输 | 机密性、完整性 | SFTP/FTPS |
| 远程登录 | 机密性、认证 | SSH |
| 邮件 | 机密性、完整性、认证 | S/MIME、PGP |
| DNS | 完整性 | DNSSEC |

### 加密技术

**对称加密**：
- 加密和解密使用相同密钥
- 算法：AES、DES、3DES
- 优点：速度快
- 缺点：密钥分发困难

**非对称加密**：
- 公钥加密，私钥解密
- 算法：RSA、ECC
- 优点：密钥分发方便
- 缺点：速度慢

**哈希函数**：
- 任意长度输入 → 固定长度输出
- 算法：MD5、SHA-1、SHA-256
- 用途：完整性验证、数字签名

**数字签名**：
- 使用私钥签名，公钥验证
- 提供完整性、认证、不可否认

### SSL/TLS协议

- **作用**：在传输层提供安全服务
- **位置**：应用层和传输层之间
- **功能**：加密、完整性、认证
- **应用**：HTTPS、FTPS等

## 直观理解

**安全协议就像保镖**：
- 机密性 = 用密码箱装东西（加密）
- 完整性 = 贴封条防篡改（哈希/签名）
- 身份认证 = 验证身份证（数字证书）
- 对称加密 = 用同一把钥匙开关
- 非对称加密 = 用公钥锁，私钥开

## 协议关联

- **HTTPS与SSL/TLS**：HTTPS = HTTP + SSL/TLS
- **SSH与TELNET**：SSH是TELNET的安全替代
- **S/MIME与SMTP**：S/MIME扩展SMTP支持安全邮件
- **408考点**：HTTPS端口443，SSH端口22，安全三要素
