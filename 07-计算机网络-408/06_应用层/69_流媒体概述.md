# 69_流媒体概述

## 核心概念

- **流媒体（Streaming Media）**：边下载边播放的多媒体传输方式
- **流媒体特点**：
  - 实时性：数据连续到达
  - 时序性：数据有严格的时序要求
  - 容忍丢失：少量丢失不影响观看
- **流媒体协议**：
  - **RTP（Real-time Transport Protocol）**：实时传输协议
  - **RTCP（RTP Control Protocol）**：RTP控制协议
  - **RTSP（Real-Time Streaming Protocol）**：实时流协议
  - **HLS（HTTP Live Streaming）**：基于HTTP的流媒体
- **408考试重点**：流媒体基本概念、RTP的作用

## 原理分析

### 流媒体传输方式

1. **UDP流**：
   - 使用RTP/UDP传输
   - 低延迟
   - 可能丢包
   - 适合实时应用

2. **HTTP流**：
   - 使用HTTP/TCP传输
   - 延迟较高
   - 可靠传输
   - 适合点播

3. **自适应流**：
   - HLS、DASH等
   - 根据带宽自动调整质量
   - 分片传输

### RTP协议

- **RTP**：实时传输协议
- 使用**UDP**传输
- 端口：偶数端口（如5004）
- RTP头部包含：
  - 序列号：检测丢包
  - 时间戳：同步
  - 负载类型：媒体类型
  - SSRC：同步源标识

### RTSP协议

- **RTSP**：实时流协议
- 类似HTTP的控制协议
- 端口：554
- 支持的操作：DESCRIBE、SETUP、PLAY、PAUSE、TEARDOWN
- 控制RTP流的播放

### HLS（HTTP Live Streaming）

- Apple开发的流媒体协议
- 基于HTTP
- 将视频分成小片段（.ts文件）
- 使用.m3u8索引文件
- 支持自适应码率

## 直观理解

**流媒体就像水管流水**：
- 传统下载 = 先把水灌满桶，再用水（下载完再播放）
- 流媒体 = 水管一直流水，边流边用（边下载边播放）
- RTP = 水管规格（传输协议）
- RTSP = 水龙头开关（控制协议）

## 协议关联

- **RTP与UDP**：RTP使用UDP传输
- **RTSP与RTP**：RTSP控制RTP流
- **HLS与HTTP**：HLS基于HTTP
- **408考点**：流媒体基本概念、RTP的作用
