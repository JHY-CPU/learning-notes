# 42_HTTPS安全超文本

## 核心概念

- **HTTPS（HTTP Secure）**：安全的HTTP，HTTP + SSL/TLS
- HTTPS使用**TCP**协议，端口号**443**
- HTTPS提供：
  - **机密性**：数据加密传输
  - **完整性**：数据不被篡改
  - **身份认证**：验证服务器身份
- **SSL/TLS**：安全套接层/传输层安全协议
  - SSL 3.0 → TLS 1.0 → TLS 1.2 → TLS 1.3
  - 位于应用层和传输层之间
- **408考试重点**：HTTPS端口、SSL/TLS的作用、与HTTP的区别

## 原理分析

### HTTPS工作过程

1. **TCP三次握手**：建立TCP连接（端口443）

2. **SSL/TLS握手**：
   - 客户端发送支持的加密算法列表
   - 服务器选择算法，发送数字证书
   - 客户端验证证书，生成会话密钥
   - 双方使用会话密钥加密通信

3. **HTTP通信**：
   - 使用加密的HTTP请求和响应
   - 数据在SSL/TLS层加密后传输

4. **关闭连接**：
   - 关闭SSL/TLS连接
   - 关闭TCP连接

### SSL/TLS握手过程

```
Client                          Server
  |------- ClientHello --------->|
  |                               |
  |<------ ServerHello ----------|
  |<------ Certificate ----------|
  |<------ ServerHelloDone ------|
  |                               |
  |------- ClientKeyExchange --->|
  |------- ChangeCipherSpec ---->|
  |------- Finished ------------>|
  |                               |
  |<------ ChangeCipherSpec -----|
  |<------ Finished -------------|
```

### HTTP vs HTTPS

| 特性 | HTTP | HTTPS |
|------|------|-------|
| 端口 | 80 | 443 |
| 加密 | 无 | SSL/TLS加密 |
| 证书 | 不需要 | 需要CA证书 |
| 性能 | 快 | 较慢（加密开销） |
| 安全性 | 低 | 高 |
| URL前缀 | http:// | https:// |

### 数字证书

- **CA（Certificate Authority）**：证书颁发机构
- **证书内容**：
  - 服务器公钥
  - 服务器域名
  - CA签名
  - 有效期
- **验证过程**：
  - 客户端使用CA的公钥验证证书签名
  - 验证域名是否匹配
  - 验证证书是否过期

### 加密方式

- **对称加密**：加密和解密使用相同密钥（AES、DES）
  - 优点：速度快
  - 缺点：密钥分发困难

- **非对称加密**：使用公钥加密，私钥解密（RSA）
  - 优点：密钥分发方便
  - 缺点：速度慢

- **HTTPS的混合加密**：
  - 用非对称加密交换对称密钥
  - 用对称加密传输数据

## 直观理解

**HTTPS就像带保镖的快递**：
- 普通快递（HTTP）：明文传输，任何人都能看
- 带保镖的快递（HTTPS）：
  - 保镖验证对方身份（数字证书）
  - 用密码箱加密包裹（SSL/TLS加密）
  - 只有收件人能打开（会话密钥）

**记忆技巧**：
- HTTPS = HTTP + SSL/TLS
- 端口443 = "安全的HTTP"
- SSL/TLS提供：机密性 + 完整性 + 认证
- 混合加密：非对称交换密钥，对称传输数据

## 协议关联

- **HTTPS与HTTP**：HTTPS是HTTP的安全版本
- **HTTPS与TCP**：HTTPS使用TCP，端口443
- **HTTPS与SSL/TLS**：SSL/TLS提供安全层
- **HTTPS与DNS**：DNS解析过程不加密（需要DNS over HTTPS）
- **408考点**：
  - HTTPS端口443
  - SSL/TLS的作用
  - HTTPS与HTTP的区别
  - 数字证书的作用
- **陷阱**：HTTPS的"安全"仅指传输安全，不保护服务器或客户端本身
