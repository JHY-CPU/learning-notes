# 24_数字信号编码方式

## 核心概念

- **数字信号编码**：将数字数据转换为数字信号的过程
- 编码方式决定了信号在传输介质上的波形
- 不同编码方式有不同的特性（自同步能力、直流分量、带宽效率等）

### 常见编码方式分类

```
数字信号编码
├─ 不归零编码（NRZ）
│  ├─ NRZ-L（不归零电平）
│  └─ NRZI（不归零反转）
├─ 归零编码（RZ）
├─ 曼彻斯特编码
├─ 差分曼彻斯特编码
├─ 双极性编码（AMI）
└─ 其他（mBnB、4B/5B等）
```

### 各编码方式对比

| 编码方式 | 特点 | 自同步 | 直流分量 | 典型应用 |
|----------|------|--------|----------|----------|
| NRZ-L | 高电平=1，低电平=0 | 差 | 有 | 早期串口 |
| NRZI | 跳变=1（或0），无跳变=反 | 差 | 有 | USB 1.0 |
| 曼彻斯特 | 每比特中间必有跳变 | 好 | 无 | 10M以太网 |
| 差分曼彻斯特 | 每比特开始有跳变=0，无跳变=1 | 好 | 无 | 令牌环网 |

### NRZ编码详解

**NRZ-L（Non-Return-to-Zero Level）**：
- 1 → 高电平
- 0 → 低电平
- 缺点：连续的0或1会导致长时间无跳变，接收端难以同步

**NRZI（Non-Return-to-Zero Inverted）**：
- 1 → 电平跳变（从高到低或从低到高）
- 0 → 电平不变
- 比NRZ-L稍好，但连续0时仍然无跳变

### 归零编码（RZ）

- 每个比特中间回到零电平
- 1 → 先高后归零
- 0 → 先低后归零
- 优点：有自同步能力
- 缺点：带宽需求大（每个比特需要两次变化）

## 原理分析

### 编码效率

编码效率 = 数据比特数 / 信号变化数

- NRZ：1 bit 对应最多 1 次变化，效率最高
- 曼彻斯特：1 bit 对应 2 次变化（每个比特中间必有跳变），效率为 50%
- 这就是为什么10M以太网用曼彻斯特编码时，线路速率达到 20 Mbaud

### 各编码的波形示例

假设数据为 **10110010**：

```
NRZ-L:
    ┌───┐       ┌───────┐
────┘   └───────┘       └──────────

NRZI（规则：1跳变）:
    ┌───┐   ┌───┐       ┌──────────
────┘   │   │   └───────┘
        └───┘

曼彻斯特（规则：1=高→低，0=低→高）:
  ┌─┐ ┌─┐   ┌─┐     ┌─┐   ┌─┐
──┘ └─┘ └───┘ └─────┘ └───┘ └────

差分曼彻斯特（规则：比特开始无跳变=1）:
  ┌─┐   ┌─┐ ┌─┐     ┌─┐ ┌─┐   ┌─┐
──┘ └───┘ └─┘ └─────┘ └─┘ └───┘ └─
```

## 直观理解

### 编码方式选择类比

- **NRZ**：好比用明码通信，简单但容易出错
- **曼彻斯特**：好比用摩尔斯电码，每个符号都有固定节奏，容易同步

### 为什么需要自同步

- 接收端需要知道每个比特的开始和结束时间
- 如果信号长时间不变（如NRZ的连续0），接收端无法知道"过了多少个比特"
- 曼彻斯特编码每个比特中间都有跳变，接收端可以根据跳变来同步时钟

## 协议关联

### 与以太网的关系

- **10BASE-T**（10M以太网）使用**曼彻斯特编码**
- **100BASE-TX**使用**4B/5B编码** + MLT-3
- **1000BASE-T**使用**PAM-5编码**

### 与USB的关系

- USB 1.0/1.1使用**NRZI编码** + 位填充
- USB 2.0使用NRZI
- USB 3.0使用更复杂的编码

### 考试要点

1. 曼彻斯特编码每个比特中间有跳变（自同步能力）
2. 曼彻斯特编码效率为50%（线路速率是数据速率的2倍）
3. NRZ编码没有自同步能力
4. 10M以太网使用曼彻斯特编码
