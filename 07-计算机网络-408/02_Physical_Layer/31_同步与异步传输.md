# 31_同步与异步传输

## 核心概念

### 同步传输（Synchronous Transmission）

- **收发双方使用同一个时钟信号**来同步数据传输
- 数据以**数据块（帧）**为单位连续发送
- 每个数据块前加**同步字符（SYN）**或**帧头**
- 数据块之间**没有间隙**
- 效率高，适合高速大批量传输

### 异步传输（Asynchronous Transmission）

- **每个字符独立传输**，字符之间可以有任意间隔
- 每个字符前加**起始位**，后加**停止位**
- 不需要收发双方有统一的时钟
- 实现简单，效率较低

### 异步传输的帧格式

```
空闲 ──────┐  ┌─── 起始位（1 bit，低电平）
           │  │
           │  │ ┌───────────────────┐
           │  │ │ 数据位（5~8 bit）│
           │  │ └───────────────────┘
           │  │    ┌─── 奇偶校验位（可选）
           │  │    │
           └──┴────┴── 停止位（1/1.5/2 bit，高电平）
```

### 两者对比

| 特性 | 同步传输 | 异步传输 |
|------|----------|----------|
| 传输单位 | 数据块/帧 | 单个字符 |
| 时钟方式 | 统一时钟 | 各字符独立 |
| 开销 | 低（帧头开销） | 高（每个字符都有起止位） |
| 效率 | 高 | 低 |
| 复杂度 | 高 | 低 |
| 速率 | 高速 | 低速 |
| 典型应用 | 网络通信、硬盘 | 键盘、串口终端 |

## 原理分析

### 异步传输效率计算

设数据位为 8 bit，起始位 1 bit，停止位 1 bit：

$$\text{效率} = \frac{8}{8+1+1} = 80\%$$

如果还有奇偶校验位：

$$\text{效率} = \frac{8}{8+1+1+1} = 72.7\%$$

### 同步传输效率计算

设帧长为 1000 bit，帧头开销为 48 bit（6字节同步字符）：

$$\text{效率} = \frac{1000}{1000+48} = 95.4\%$$

帧越长，效率越高。

### 同步传输的同步方式

1. **字符同步**：使用特殊的同步字符（如SYN=0x16）
2. **位同步**：使用时钟信号线（如SPI、I2C）
3. **自同步编码**：使用曼彻斯特编码等自带时钟的编码

## 直观理解

### 类比：送快递

- **同步传输**：快递公司统一安排时间，整卡车货物一次送到（高效但需要协调）
- **异步传输**：每件快递单独送，随时可以发（灵活但效率低）

### 类比：上课

- **同步传输**：老师按课表统一上课，全班一起听（按时钟同步）
- **异步传输**：学生自学，有疑问随时来问老师（各走各的节奏）

### 异步传输为什么需要起止位

- 起始位：告诉接收端"注意，有数据来了"
- 停止位：告诉接收端"这个字符传完了"
- 没有起止位，接收端不知道什么时候开始采样、什么时候结束

## 协议关联

### 与数据链路层的关系

- 同步传输的"帧"概念与数据链路层的帧有关
- 异步传输的字符框架是物理层的
- 数据链路层使用同步传输方式（帧是连续的）

### 与实际协议的关系

- **串口（RS-232）**：异步传输，每个字符有起止位
- **HDLC**：同步传输，使用标志字段（01111110）做帧同步
- **以太网**：同步传输，使用前导码和帧起始定界符

### 考试要点

1. 异步传输每个字符有**起始位**和**停止位**
2. 异步传输的效率低于同步传输
3. 同步传输适合高速大批量数据传输
4. 异步传输不需要统一时钟，实现简单
5. 注意"异步传输"和"异步通信"的区别
