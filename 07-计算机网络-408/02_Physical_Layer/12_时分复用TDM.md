# 12_时分复用TDM

## 核心概念

- **时分复用（Time Division Multiplexing, TDM）**：将时间划分为若干个时隙（Time Slot），每个用户轮流占用信道
- 所有用户使用**相同的频率**，但在**不同的时间**使用信道
- TDM 是**数字**复用技术

### TDM帧结构

```
|← ─────────── 一个TDM帧 ─────────── →|
| TS0 | TS1 | TS2 | TS3 | TS0 | TS1 | ...
  U1    U2    U3    U4    U1    U2
```

- **帧周期** $T_f$：一个完整TDM帧的时间长度
- **时隙长度** $T_s = T_f / N$（$N$ 为用户数）
- 每个用户在每帧中固定占用一个时隙

### TDM的关键参数

- **帧速率**：$f_f = 1/T_f$
- **每帧时隙数**：$N$
- **每个时隙比特数**：$n$
- **总数据速率**：$R = N \times n \times f_f$

### E1标准（欧洲标准，408常考）

- **帧长**：256 bit（32个时隙，每个时隙8 bit）
- **帧周期**：$125\mu s$
- **帧速率**：8000 帧/s
- **总速率**：$32 \times 8 \times 8000 = 2.048 \text{ Mbit/s}$
- TS0：用于帧同步
- TS16：用于信令
- 其余30个时隙：用于语音数据

### T1标准（美国标准）

- 24个时隙 + 1 bit帧同步 = 193 bit/帧
- 帧速率：8000 帧/s
- 总速率：$193 \times 8000 = 1.544 \text{ Mbit/s}$

## 原理分析

### TDM的同步要求

- TDM 严格依赖**时间同步**
- 发送端和接收端必须知道每个时隙的起始和结束时间
- 如果同步丢失，所有用户的数据都会出错
- 因此 TS0 通常用于**帧同步**

### 统计时分复用（STDM）

**普通TDM的问题**：
- 用户没有数据发送时，时隙空闲
- 信道利用率低

**STDM的改进**：
- 动态分配时隙
- 用户有数据才分配时隙
- 需要在数据中加入用户标识（地址字段）
- 大幅提高信道利用率

```
普通TDM（固定分配）：
| U1 | U2 | U3 | U4 | U1 | U2 | U3 | U4 |
| 数据|空闲|数据|数据|空闲|数据|空闲|数据|

STDM（动态分配）：
|U1数据|U3数据|U4数据|U2数据|U3数据|U1数据|...
```

## 直观理解

### 类比：轮流发言

- TDM 好比**开会时大家轮流发言**
- 主持人规定每人每次发言1分钟（时隙）
- 大家按固定顺序轮流（帧结构）
- 如果某人不想发言，这1分钟就浪费了（普通TDM的缺点）
- STDM 好比谁有话谁举手发言，不浪费时间

### 类比：共享打印机

- 办公室有一台打印机（信道）
- 普通TDM：规定每个人每天10:00-10:10可以打印，不管有没有文件要打印
- STDM：谁有文件要打印就去排队，按顺序打印

## 协议关联

### 与数字传输的关系

- TDM 是数字复用技术，适合数字信号
- PCM + TDM = 数字电话系统的基础
- E1/T1 是电信网络的基石

### 与数据链路层的关系

- TDM 工作在物理层
- STDM 的思想在数据链路层的 MAC 协议中有类似应用（如令牌传递）
