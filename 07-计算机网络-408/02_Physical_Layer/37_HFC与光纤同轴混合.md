# 37_HFC与光纤同轴混合

## 核心概念

- **HFC（Hybrid Fiber-Coaxial，光纤同轴混合网）**：有线电视网络的升级版
- 主干使用**光纤**，用户接入使用**同轴电缆**
- 既可传输有线电视信号，也可提供宽带上网服务

### HFC的网络结构

```
头端(Headend) ──光纤── 分光器 ──光纤── 光节点 ──同轴── 用户
     │                       │
     │                   ┌───┴───┐
     │                   │ 光节点 │（光电转换）
     │                   └───┬───┘
     │                       │
     │              ┌──同轴电缆──┐
     │              │            │
   互联网        用户A        用户B
```

### HFC的特点

| 特性 | 说明 |
|------|------|
| 主干 | 光纤（树形拓扑） |
| 接入 | 同轴电缆 |
| 频带划分 | 上行5~42MHz，下行50~750MHz |
| 速率 | 下行30~40 Mbit/s（共享） |
| 双向通信 | 需要改造原有单向网络 |

### HFC的频带划分

```
频率(MHz)
↑
│  ┌─────────────────────────────────┐
│  │       下行数据+电视信号         │  50~750 MHz
│  │  (每个电视频道6~8MHz)            │
│  └─────────────────────────────────┘
│  ┌──────────┐
│  │ 上行数据  │                       5~42 MHz
│  └──────────┘
└────────────────────────────────────→ 频率
```

## 原理分析

### Cable Modem的工作原理

- **Cable Modem（电缆调制解调器）**：在HFC网络中提供双向数据通信
- 下行：从头端到用户，使用电视信号频带中的若干频道
- 上行：从用户到头端，使用5~42MHz频带
- 下行使用**广播**方式，上行使用**TDMA**方式

### HFC的共享特性

- 同一光节点下的所有用户**共享**下行带宽
- 好比一个小区的用户共享一条宽带
- 用户越多，每个用户分到的带宽越少
- 这与ADSL的独占带宽不同

### HFC vs ADSL对比

| 特性 | HFC | ADSL |
|------|-----|------|
| 介质 | 光纤+同轴 | 电话双绞线 |
| 拓扑 | 树形（共享） | 点对点（独占） |
| 下行速率 | 30~40 Mbit/s（共享） | 最高8 Mbit/s（独占） |
| 上行速率 | 较低（共享） | 1 Mbit/s（独占） |
| 电视服务 | 支持 | 不支持 |

## 直观理解

### 类比：供水系统

- HFC好比**小区供水系统**
- 主管道用粗水管（光纤），到每栋楼后分细水管（同轴电缆）
- 所有住户共享主管道的水量
- 高峰期水压下降（用户多时速率下降）

### 类比：公交vs出租车

- **HFC**像公交：固定路线，所有乘客共享，到站上下
- **ADSL**像出租车：专车专用，独占资源

## 协议关联

### 与DOCSIS标准

- **DOCSIS（Data Over Cable Service Interface Specification）**：Cable Modem的标准
- DOCSIS 3.0：下行最高 1 Gbit/s，上行 200 Mbit/s
- DOCSIS 3.1：支持更高频带，速率更高

### 与物理层的关系

- HFC使用频分复用（FDM）划分频带
- Cable Modem使用调制技术（QAM等）
- 属于物理层的宽带接入技术

### 考试要点

1. HFC = 光纤主干 + 同轴接入
2. 下行带宽共享，与ADSL的独占不同
3. 需要Cable Modem进行信号调制解调
4. 可以同时提供电视和上网服务
5. HFC是有线电视网络的宽带化改造
