# 21_EIA-232接口标准

## 核心概念

- **EIA-232**（原名RS-232）：美国电子工业协会（EIA）制定的串行通信接口标准
- 是最经典的物理层接口标准之一
- 广泛用于计算机串口（COM口）、调制解调器连接

### EIA-232的四大接口特性

**机械特性**：
- 使用**DB-25**或**DB-9**连接器
- DB-25：25针，功能最全
- DB-9：9针，常用（简化版）

**电气特性**：
- 采用**单端信号**传输（非差分）
- 逻辑"0"（空号）：**+3V ~ +15V**
- 逻辑"1"（传号）：**-3V ~ -15V**
- 过渡区：-3V ~ +3V（不确定区域）
- 最大电缆长度：约**15m**
- 最大数据速率：约**20 kbit/s**（标准规定）

**功能特性**：
- 定义了各引脚的功能
- 主要引脚：TxD（发送数据）、RxD（接收数据）、RTS（请求发送）、CTS（清除发送）、DSR（数据设备就绪）、DTR（数据终端就绪）、GND（信号地）

**规程特性**：
- 规定了信号交互的时序
- 典型流程：DTR→DSR→RTS→CTS→数据传输

### 常用引脚（DB-9）

| 引脚 | 名称 | 方向 | 功能 |
|------|------|------|------|
| 2 | RxD | 输入 | 接收数据 |
| 3 | TxD | 输出 | 发送数据 |
| 4 | DTR | 输出 | 数据终端就绪 |
| 5 | GND | — | 信号地 |
| 6 | DSR | 输入 | 数据设备就绪 |
| 7 | RTS | 输出 | 请求发送 |
| 8 | CTS | 输入 | 清除发送 |
| 9 | RI | 输入 | 振铃指示 |

## 原理分析

### 串口通信的工作流程

1. **建立连接**：
   - 终端设备置高DTR → 表示终端就绪
   - 调制解调器置高DSR → 表示设备就绪
   - 终端置高RTS → 请求发送
   - 调制解调器置高CTS → 允许发送

2. **数据传输**：
   - TxD发送数据，RxD接收数据
   - 全双工通信（TxD和RxD独立）

3. **释放连接**：
   - 按相反顺序释放各信号线

### EIA-232与RS-485对比

| 特性 | EIA-232 | RS-485 |
|------|---------|--------|
| 信号方式 | 单端 | 差分 |
| 最大距离 | 15m | 1200m |
| 最大速率 | 20 kbit/s | 10 Mbit/s |
| 节点数 | 点对点 | 多点（最多32个） |
| 抗干扰 | 差 | 好 |

## 直观理解

### 类比：打电话

- DTR/DSR = "喂，你在吗？"（确认双方在线）
- RTS/CTS = "我要说话了" "好的，你说吧"（请求和许可）
- TxD/RxD = 实际通话（发送/接收）
- 挂电话 = 按反序释放信号

### 记忆要点

- EIA-232电气特性：正电压=0，负电压=1（注意！和直觉相反）
- 记忆：正为零（"正"和"零"押韵）
- 实际考试中容易出"逻辑0对应正电压"的判断题

## 协议关联

### 与OSI各层的关系

- EIA-232是纯粹的**物理层**标准
- 只定义了电气、机械、功能和规程特性
- 不涉及数据格式、帧结构、差错控制（那是数据链路层的事）

### 与调制解调器的关系

- EIA-232最初就是为连接DTE（数据终端设备）和DCE（数据通信设备/调制解调器）设计的
- DTE = 计算机
- DCE = 调制解调器

### 考试要点

1. EIA-232电气特性：逻辑0为正电压，逻辑1为负电压
2. 最大距离约15m
3. 采用单端信号（不是差分）
4. 可以考与其他标准（RS-485、V.24、X.21）的对比
