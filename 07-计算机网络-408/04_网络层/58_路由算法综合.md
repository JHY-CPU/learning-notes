# 58_路由算法综合

## 核心概念

- **路由算法分类**：

| 分类标准 | 类型 | 代表 |
|---------|------|------|
| 按信息范围 | 全局式 | 链路状态（OSPF） |
| | 分布式 | 距离向量（RIP） |
| 按度量 | 静态 | 管理员配置 |
| | 动态 | 自适应路由 |
| 按更新时机 | 定期更新 | RIP（30秒） |
| | 触发更新 | OSPF（变化时） |

- **两大经典算法**：
  1. **Bellman-Ford算法**（距离向量）：RIP使用
  2. **Dijkstra算法**（链路状态）：OSPF使用

- **408考点**：
  - 两种算法的原理和步骤
  - 两种算法的优缺点
  - 两种算法在具体协议中的应用
  - 手动模拟算法计算路由表

## 原理分析

### Bellman-Ford算法

**核心方程**：
$$d_x(y) = \min_v \{c(x,v) + d_v(y)\}$$

- 每个节点只和邻居交换距离向量
- 收到邻居的距离向量后更新自己的距离
- 重复直到收敛

**特点**：
- 分布式，每个节点只依赖邻居信息
- 收敛慢，可能有环路
- 实现简单，资源开销小

### Dijkstra算法

**核心思想**：
- 每个节点有完整的网络拓扑
- 以自己为根，逐步扩展最短路径树
- 每次选择距离最小的未访问节点

**特点**：
- 全局式，需要完整拓扑
- 收敛快，保证无环路
- 资源开销大（内存、CPU）

### 算法对比

| 对比 | Bellman-Ford | Dijkstra |
|------|-------------|----------|
| 信息需求 | 只需邻居信息 | 需要全网拓扑 |
| 计算方式 | 分布式 | 集中式（每个节点独立计算） |
| 收敛速度 | 慢 | 快 |
| 环路风险 | 有 | 无 |
| 复杂度 | O(VE) | O(V²) 或 O(VlogV+ E) |
| 对应协议 | RIP | OSPF |
| 负权边 | 可以处理 | 不能处理 |

## 直观理解

**类比**：
- **Bellman-Ford** = 传话游戏
  - 每个人只问邻居"到XX多远"
  - 根据邻居的回答更新自己的信息
  - 可能传错（环路）

- **Dijkstra** = 看地图找路
  - 有完整地图
  - 一步步扩展最近的点
  - 保证找到最短路（无环路）

**选择策略**：
- 小型网络 → RIP（Bellman-Ford，简单够用）
- 大型网络 → OSPF（Dijkstra，快速无环）

## 协议关联

- Bellman-Ford与RIP：RIP中所有链路代价为1（跳数）
- Dijkstra与OSPF：OSPF中链路代价与带宽成反比
- 考试中经常要求手动模拟两种算法的执行过程
- 两种算法都可以计算单源最短路径

### 考试技巧
- Bellman-Ford：画距离向量表，逐轮更新
- Dijkstra：画表，每轮选最小距离节点，更新邻居
