# 51_OSPF链路状态路由

## 核心概念

- **链路状态路由的核心思想**：
  - 每个路由器了解**全网拓扑**
  - 每个路由器独立运行**Dijkstra算法**计算最短路径
  - 不依赖邻居的路由表（与距离向量不同）

- **链路状态协议的五个步骤**：
  1. **发现邻居**：发送Hello分组，建立邻居关系
  2. **测量链路代价**：确定到每个邻居的链路代价
  3. **构造LSA**：生成链路状态通告
  4. **泛洪LSA**：将LSA发送到区域内的所有路由器
  5. **计算最短路径**：运行Dijkstra算法

- **链路状态数据库（LSDB）**：
  - 存储全网所有路由器的LSA
  - 所有路由器的LSDB应该一致
  - LSDB构成完整的网络拓扑图

- **408考点**：
  - 链路状态协议与距离向量协议的区别
  - 泛洪机制的可靠性保证
  - Dijkstra算法的步骤

## 原理分析

### Dijkstra算法步骤

```
初始化：
  N' = {源节点u}  # 已确定最短路径的节点集合
  for 所有节点v:
    if v是u的邻居:
      D(v) = c(u,v), p(v) = u  # 距离和前驱
    else:
      D(v) = ∞, p(v) = undefined

循环（直到N'包含所有节点）：
  1. 在N'之外的节点中，选择D(w)最小的w
  2. 将w加入N'
  3. 对w的所有邻居v（不在N'中）:
     if D(w) + c(w,v) < D(v):
       D(v) = D(w) + c(w,v)
       p(v) = w  # 更新前驱
```

### 链路状态泛洪的可靠性

- 每个LSA有序列号和年龄
- 收到新LSA后泛洪到除来源外的所有接口
- 确认机制：收到LSA后发送LSAck
- 序列号防止旧的LSA覆盖新的

### 链路状态 vs 距离向量

| 特性 | 链路状态（OSPF） | 距离向量（RIP） |
|------|---------------|---------------|
| 信息 | 链路状态（邻居和代价） | 整个路由表 |
| 传播范围 | 泛洪到整个区域 | 仅相邻邻居 |
| 拓扑知识 | 每个路由器知道全网拓扑 | 只知道下一跳 |
| 算法 | Dijkstra | Bellman-Ford |
| 收敛速度 | 快（触发更新） | 慢（定期更新） |
| 环路 | 不会形成环路 | 可能形成环路 |
| 资源消耗 | CPU和内存较高 | 较低 |

## 直观理解

**类比**：
- **链路状态** = 每个人都有一张完整地图，自己规划最优路线
- **距离向量** = 每个人只问邻居"到XX多远"，根据回答更新自己的路线

**记忆口诀**：
- "链路状态泛洪全网，每个路由器自己算"
- "距离向量只传邻居，传来传去慢慢收敛"

## 协议关联

- OSPF使用链路状态算法
- IS-IS也是链路状态协议（常用于ISP）
- 链路状态协议的泛洪需要确认机制保证可靠性
- 区域划分减少了泛洪的范围，降低了LSDB的大小
