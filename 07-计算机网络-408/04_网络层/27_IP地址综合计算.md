# 27_IP地址综合计算

## 核心概念

- **IP地址综合计算涵盖**：
  1. 地址类别判断
  2. 网络地址/广播地址/可用范围计算
  3. 子网划分
  4. CIDR地址块计算
  5. 路由聚合
  6. IP分片计算

- **核心公式汇总**：
  - 网络地址 = IP AND 子网掩码
  - 广播地址 = 网络地址 OR (NOT 子网掩码)
  - 可用主机数 = $2^h - 2$（h为主机号位数）
  - 子网间隔 = $2^{\text{主机位数}}$
  - CIDR地址块大小 = $2^{(32-n)}$
  - 每片最大数据 = $\lfloor(MTU-20)/8\rfloor \times 8$

## 原理分析

### 综合例题1：地址分类+子网划分

**题**：IP地址200.1.2.3，子网掩码255.255.255.192
（1）该IP属于哪类地址？
（2）求网络地址、广播地址
（3）可用IP范围和主机数
（4）该子网是C类网络200.1.2.0的第几个子网？

**解**：
（1）200在192~223之间，属于**C类地址**
（2）192 = 11000000，借2位
- 3 AND 192 = 0 → 网络地址 = 200.1.2.0
- 广播地址 = 200.1.2.63
（3）可用范围 = 200.1.2.1 ~ 200.1.2.62，主机数 = 62
（4）间隔 = 64，子网号索引 = 0/64 = 0，第**1个子网**（索引0）

### 综合例题2：CIDR+路由聚合

**题**：ISP分配到192.168.0.0/20，需分配给4个子网，每个子网至少500台主机。如何分配？

**解**：
- $2^m - 2 \geq 500$，$m \geq 9$
- 每个子网需要/23（510台主机）
- 4个/23需要聚合为 /21（$4 \times 2^{11} = 2^{13}$，前缀减少2位）
- 但/20只有 $2^{12} = 4096$ 地址，4个/23需 $4 \times 2^{11} = 8192$ 地址

**结论**：/20不够分配4个/23！需要更大的地址块或减少主机数。

若改为4个/24（254台主机）：
- 每子网/24，共4个，够用
- 分配：192.168.0.0/24, 192.168.1.0/24, 192.168.2.0/24, 192.168.3.0/24

### 综合例题3：IP分片+地址判断

**题**：数据报总长4000字节（首部20字节），从192.168.1.1发往10.0.0.1，经过MTU=1400的网络。
（1）目的地址属于哪类？
（2）需要几个分片？
（3）写出各分片的偏移和MF。

**解**：
（1）10.x.x.x 属于 **A类地址**
（2）每片最大数据 = $\lfloor1380/8\rfloor \times 8 = 1376$
分片数 = $\lceil3980/1376\rceil = 3$
（3）

| 分片 | 总长 | 偏移 | MF |
|------|------|------|-----|
| 1 | 1396 | 0 | 1 |
| 2 | 1396 | 172 | 1 |
| 3 | 1228 | 344 | 0 |

验证：1376+1376+1228 = 3980 ✓

## 直观理解

**计算流程图**：
```
拿到IP和掩码
  ↓
判断类别（看第一字节）
  ↓
AND运算求网络地址
  ↓
NOT掩码 OR 得广播地址
  ↓
确定可用范围和主机数
  ↓
如涉及分片：MTU限制 → 分片计算
```

## 协议关联

- 地址计算是路由转发的基础
- 分片计算与MTU相关，与TCP MSS配合避免不必要的分片
- 考试中地址计算与路由表查表经常联合出题
