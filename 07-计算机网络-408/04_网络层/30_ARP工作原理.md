# 30_ARP工作原理

## 核心概念

- **ARP协议的四种操作**：
  1. ARP请求（Request）：广播，"谁拥有这个IP？"
  2. ARP应答（Reply）：单播，"我拥有这个IP，这是我的MAC"
  3. 免费ARP（Gratuitous ARP）：主机启动时广播自己的IP-MAC映射
  4. 代理ARP（Proxy ARP）：路由器代替其他网段的主机应答ARP

- **ARP缓存管理**：
  - 缓存条目有TTL（生存时间），超时删除
  - 收到ARP应答时更新/添加条目
  - 收到ARP请求时也可更新发送方的条目（被动学习）

- **ARP报文封装**：
  - ARP报文直接封装在以太网帧的数据部分
  - 以太网帧类型 = 0x0806（ARP）
  - ARP不是封装在IP数据报中的！

## 原理分析

### ARP请求的以太网帧

```
以太网帧头：
  目的MAC = FF:FF:FF:FF:FF:FF（广播）
  源MAC = 发送方MAC
  类型 = 0x0806

ARP请求内容：
  发送方MAC = A的MAC
  发送方IP = A的IP
  目标MAC = 00:00:00:00:00:00（未知）
  目标IP = B的IP
```

### ARP应答的以太网帧

```
以太网帧头：
  目的MAC = A的MAC（单播）
  源MAC = B的MAC
  类型 = 0x0806

ARP应答内容：
  发送方MAC = B的MAC
  发送方IP = B的IP
  目标MAC = A的MAC
  目标IP = A的IP
```

### 跨网段ARP详解

场景：A(192.168.1.100/24) → B(10.0.0.50/8)，默认网关192.168.1.1

```
步骤1: A判断B不在同一子网
       192.168.1.100 AND 255.255.255.0 = 192.168.1.0
       10.0.0.50 AND 255.255.255.0 = 10.0.0.0 ≠ 192.168.1.0

步骤2: A需要将帧发给网关
       A发ARP请求获取网关MAC（如果缓存没有）
       目的IP写网关IP（192.168.1.1），不是B的IP！

步骤3: 以太网帧
       源MAC = A的MAC
       目的MAC = 网关的MAC
       IP数据报：源IP=A，目的IP=B（不变！）

步骤4: 路由器收到帧，解封装
       查路由表，确定转发端口
       向B所在网段发ARP请求获取B的MAC

步骤5: 路由器重新封装帧
       源MAC = 路由器B端口的MAC
       目的MAC = B的MAC
       IP数据报：源IP=A，目的IP=B（仍然不变！）
```

**核心规律**：IP地址端到端不变，MAC地址逐跳改变。

## 直观理解

**类比**：
- ARP请求 = 在邮局喊"谁是张三？"（广播，所有人听到）
- ARP应答 = 张三举手说"我是"（单播，只回给问的人）
- 免费ARP = 新人自我介绍（广播告诉所有人自己的信息）

## 协议关联

- ARP与IP：ARP为IP提供MAC地址映射
- ARP与以太网：ARP报文封装在以太网帧中
- ARP与路由：每跳路由器都需要ARP获取下一跳MAC
- ARP攻击：伪造ARP应答，实现中间人攻击（ARP欺骗）

### 考试重点
- ARP请求是广播，应答是单播
- 跨网段时，A获取的是网关MAC，不是目的主机MAC
- IP地址在端到端传输中不变，MAC地址每跳都变
- ARP是网络层协议（408教材观点）
