# 21_最长前缀匹配_概念_考试

## 核心概念

- **408最长前缀匹配考点**：
  1. 给路由表和目的IP，选择转发端口
  2. 路由聚合后的转发判断
  3. 默认路由的作用和优先级
  4. 最长前缀匹配与子网划分的关系

- **关键规则**：
  - 前缀越长，匹配越精确，优先级越高
  - /0（默认路由）优先级最低
  - 路由表中可能有多个匹配，必须选最长的

## 原理分析

### 真题1：多条目匹配

**题**：目的地址172.16.60.5，路由表如下：

| 目的网络 | 掩码 | 下一跳 |
|---------|------|--------|
| 172.16.0.0 | 255.255.0.0(/16) | R1 |
| 172.16.48.0 | 255.255.240.0(/20) | R2 |
| 172.16.60.0 | 255.255.255.0(/24) | R3 |
| 0.0.0.0 | 0.0.0.0(/0) | R4 |

问：分组从哪个端口转发？

**解**：
1. 匹配/16: 172.16.60.5 AND 255.255.0.0 = 172.16.0.0 ✓
2. 匹配/20: 172.16.60.5 AND 255.255.240.0 = 172.16.48.0（60在48~63之间）✓
3. 匹配/24: 172.16.60.5 AND 255.255.255.0 = 172.16.60.0 ✓
4. 匹配/0: 默认 ✓

最长匹配：/24 → **R3**

### 真题2：聚合后的转发

**题**：路由器R1向R2通告了聚合路由192.168.0.0/22。R2收到目的地址为192.168.3.200的分组，路由表中无更具体的条目，应如何处理？

**解**：
- /22 匹配：192.168.0.0/22 包含 192.168.0.0~192.168.3.255
- 192.168.3.200 在范围内，匹配
- 转发给R1
- 如果192.168.3.0/24实际不存在，则R1收到后无法转发 → 路由黑洞

### 真题3：默认路由

**题**：企业网络边界路由器的路由表：
- 192.168.1.0/24 → 内部（接口1）
- 192.168.2.0/24 → 内部（接口2）
- 0.0.0.0/0 → ISP（接口3）

访问外部网站203.0.113.5时走哪个接口？

**解**：
- 不匹配/24的任何条目
- 匹配/0（默认路由）
- 走 **接口3（ISP）**

### 真题4：陷阱题

**题**：路由表中有两条：
- 10.0.0.0/8 → R1
- 10.1.0.0/16 → R2

目的地址10.2.0.5从哪个端口转发？

**解**：
- 匹配/8：10.2.0.5 AND 255.0.0.0 = 10.0.0.0 ✓
- 匹配/16：10.2.0.5 AND 255.255.0.0 = 10.2.0.0 ≠ 10.1.0.0 ✗
- 只匹配/8 → **R1**
- 注意：/16虽然前缀更长，但不匹配！

## 直观理解

**解题三步法**：
1. 对每个路由条目，用目的IP AND 掩码，看是否等于网络地址
2. 记录所有匹配条目
3. 选前缀长度最长的

**口诀**："先找匹配，再选最长"

## 协议关联

- 最长前缀匹配是CIDR路由转发的核心
- 路由器使用TCAM硬件实现高速匹配
- 考试中经常与CIDR聚合、子网划分联合考查
