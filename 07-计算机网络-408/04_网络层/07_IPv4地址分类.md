# 07_IPv4地址分类

## 核心概念

- **A类地址**：最高位为0
  - 网络号范围：1.0.0.0 ~ 126.0.0.0
  - 第一个字节范围：**1 ~ 126**
  - 默认子网掩码：255.0.0.0（/8）
  - 每网络最大主机数：$2^{24} - 2 = 16777214$

- **B类地址**：最高两位为10
  - 网络号范围：128.0.0.0 ~ 191.255.0.0
  - 第一个字节范围：**128 ~ 191**
  - 默认子网掩码：255.255.0.0（/16）
  - 每网络最大主机数：$2^{16} - 2 = 65534$

- **C类地址**：最高三位为110
  - 网络号范围：192.0.0.0 ~ 223.255.255.0
  - 第一个字节范围：**192 ~ 223**
  - 默认子网掩码：255.255.255.0（/24）
  - 每网络最大主机数：$2^8 - 2 = 254$

- **D类地址**：最高四位为1110
  - 第一个字节范围：224 ~ 239
  - 用于组播（multicast）

- **E类地址**：最高四位为1111
  - 第一个字节范围：240 ~ 255
  - 保留地址

## 原理分析

### 地址分类速查

| 类别 | 第一字节范围 | 前缀位 | 默认掩码 | 可用网络数 | 每网主机数 |
|------|------------|--------|---------|-----------|-----------|
| A | 1~126 | 0xxx xxxx | /8 | 126 | 16777214 |
| B | 128~191 | 10xx xxxx | /16 | 16384 | 65534 |
| C | 192~223 | 110x xxxx | /24 | 2097152 | 254 |
| D | 224~239 | 1110 xxxx | — | — | — |
| E | 240~255 | 1111 xxxx | — | — | — |

### 主机数计算公式

$$\text{可用主机数} = 2^{n} - 2$$

其中 $n$ 是主机号位数。减2是因为：
- 全0：表示网络地址（本网络）
- 全1：表示广播地址（本网络所有主机）

### 快速判断类别

- 第一字节 ≤ 126：A类
- 第一字节 = 127：回环地址（特殊）
- 128 ≤ 第一字节 ≤ 191：B类
- 192 ≤ 第一字节 ≤ 223：C类
- 224 ≤ 第一字节 ≤ 239：D类
- 240 ≤ 第一字节 ≤ 255：E类

## 直观理解

**记忆技巧**：

```
A类：0开头  →  0xxxxxxx  →  0~127  →  1~126（去掉0和127）
B类：10开头 →  10xxxxxx  →  128~191
C类：110开头 → 110xxxxx  →  192~223
D类：1110开头 → 1110xxxx  →  224~239
E类：1111开头 → 1111xxxx  →  240~255
```

**形象记忆**：
- A类地址像"一栋大房子（少数网络，每栋住很多人）"
- C类地址像"很多小公寓（大量网络，每间住很少人）"
- B类在中间，平衡型

## 协议关联

### 分类编址的缺陷与后续发展

- **问题**：地址浪费严重
  - 一个A类网络1600万+主机，很少有组织需要这么大的网络
  - 一个C类网络只有254个主机，很多组织不够用
- **解决方案**：
  1. **子网划分（Subnetting）**：在分类基础上进一步划分
  2. **CIDR（无分类编址）**：彻底抛弃分类，用前缀长度表示

### 考试要点
- 快速根据IP地址第一字节判断类别
- 计算每类网络的可用主机数
- 127.x.x.x 是回环测试地址，不属于任何类
