# 22_IPv4数据报格式

## 核心概念

- **IPv4数据报（Datagram）结构** = 首部（20~60字节）+ 数据部分

- **固定首部字段（20字节）**：

| 字段 | 长度 | 说明 |
|------|------|------|
| 版本（Version） | 4位 | IPv4 = 0100 |
| 首部长度（IHL） | 4位 | 单位4字节，最小5（20字节），最大15（60字节） |
| 区分服务（DS/ToS） | 8位 | 服务质量（以前叫服务类型） |
| 总长度（Total Length） | 16位 | 首部+数据，最大65535字节 |
| 标识（Identification） | 16位 | 分片时同一数据报的所有片标识相同 |
| 标志（Flags） | 3位 | DF（禁止分片）、MF（更多分片）、保留位 |
| 片偏移（Fragment Offset） | 13位 | 以8字节为单位 |
| 生存时间（TTL） | 8位 | 最大跳数，每经过一跳减1 |
| 协议（Protocol） | 8位 | 上层协议：TCP=6, UDP=17, ICMP=1 |
| 首部检验和（Header Checksum） | 16位 | 仅校验首部 |
| 源IP地址 | 32位 | 发送方IP |
| 目的IP地址 | 32位 | 接收方IP |
| 可选字段 | 可变 | 最长40字节 |

- **408必考字段**：
  - 标识、标志、片偏移（分片三件套）
  - TTL（防止环路）
  - 协议号（TCP=6, UDP=17, ICMP=1）
  - 首部长度（单位4字节）
  - 总长度（单位字节）

## 原理分析

### 首部长度的计算

IHL字段值 × 4 = 实际首部字节数
- IHL = 5 → 5 × 4 = 20字节（无选项，最常用）
- IHL = 15 → 15 × 4 = 60字节（最大首部）

### 标志字段（3位）

```
位0：保留（必须为0）
位1：DF（Don't Fragment）
    DF=1：禁止分片（若需分片则丢弃，返回ICMP错误）
    DF=0：允许分片
位2：MF（More Fragments）
    MF=1：后面还有分片
    MF=0：最后一个分片（或未分片）
```

### 片偏移的计算

片偏移字段以 **8字节** 为单位。
实际偏移字节 = 片偏移值 × 8

### 首部检验和

- 仅对IP首部进行检验（不包含数据）
- 发送方：将首部每16位求反码算术和，再取反码
- 接收方：同样计算，结果应全1（否则出错）
- 每经过一跳路由器，TTL减1，检验和需重新计算

## 直观理解

**类比**：IP数据报像一封信
- 版本 = 邮件格式版本
- 总长度 = 信封+信纸的总厚度
- TTL = 最多经过几个邮局（防止死循环）
- 标识 = 同一封拆开的多封信的编号
- MF = "我后面还有"
- 片偏移 = "我是第几部分"

**记忆口诀**：
- "版本IHL服务类，总长标识标片偏，TTL协议检验和，源目IP在后面"

## 协议关联

- **与传输层**：协议字段标识上层是TCP还是UDP
- **与数据链路层**：总长度不能超过MTU（以太网MTU=1500字节）
- **与ICMP**：TTL=0时发送ICMP时间超时报文
- **分片与重组**：分片可能在中间路由器发生，但重组只在目的主机
