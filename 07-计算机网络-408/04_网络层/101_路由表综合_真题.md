# 路由表综合真题

## 核心概念

### 路由表真题要点
- 最长前缀匹配是路由转发的核心原则
- 默认路由（0.0.0.0/0）匹配所有目的地址
- 直连路由无需下一跳，直接通过接口发送
- 非直连路由需要下一跳地址

## 原理分析

### 真题一
路由表如下：
```
目的网络/掩码          下一跳
10.0.0.0/8             接口1（直连）
10.1.0.0/16            接口2
10.1.1.0/24            接口3
0.0.0.0/0              10.0.0.1（默认）
```

目的地址10.1.1.5 → 最长匹配/24 → 接口3
目的地址10.1.2.5 → 最长匹配/16 → 接口2
目的地址10.2.1.5 → 最长匹配/8 → 接口1
目的地址192.168.1.1 → 匹配默认路由 → 10.0.0.1

### 真题二（选择题）
路由器收到目的地址为202.115.78.123的数据报，路由表中有：
- 202.115.78.0/24 → 下一跳A
- 202.115.0.0/16 → 下一跳B
- 0.0.0.0/0 → 下一跳C

该数据报应转发到：
A. 下一跳A B. 下一跳B C. 下一跳C D. 丢弃

**答**：A。最长前缀匹配/24。

### 真题三
某路由器接口eth0的IP为192.168.1.1/24，接口eth1的IP为10.0.0.1/8。当收到目的地址为10.1.2.3的数据报时，如何转发？

**解**：
10.1.2.3 AND 255.0.0.0 = 10.0.0.0 = 直连网络
→ 通过eth1直接发送（需要ARP解析下一跳MAC）

### 真题四：路由表构建
给定网络拓扑，写出每个路由器的路由表条目

**步骤**：
1. 添加直连路由（自动）
2. 根据路由协议（RIP/OSPF）添加动态路由
3. 添加默认路由（如有）

## 直观理解
- 最长前缀匹配像"查字典时选最精确的定义"
- 默认路由像"不确定走哪条路就走主干道"

## 协议关联
- 路由表由路由协议（RIP/OSPF/BGP）维护
- 转发表是路由表的简化版本，用于快速转发
- 静态路由和动态路由可以共存（管理距离不同）
