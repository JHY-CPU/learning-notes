# 12_子网划分_概念_计算

## 核心概念

- **子网划分计算的核心公式**：
  - 借位数 $n$：$2^n \geq$ 所需子网数
  - 主机位数 $m$：$32 - \text{网络前缀长度}$
  - 每子网可用主机数：$2^m - 2$
  - 子网间隔（块大小）：$2^m$

- **子网地址的规律**：
  - 子网号按借用位的二进制递增
  - 子网间隔 = $2^{(\text{主机号位数})}$

- **408必考计算类型**：
  1. 给定网络和子网数，求子网掩码
  2. 给定IP和掩码，求网络地址/广播地址/可用范围
  3. 给定需求，判断IP属于哪个子网

## 原理分析

### 计算题详解

**例1：B类网络172.16.0.0，需要10个子网**

步骤1：$2^n \geq 10$，$n = 4$（借4位，可分16个子网）
步骤2：新前缀长度 = $16 + 4 = 20$
步骤3：子网掩码 = 255.255.240.0（/20）
步骤4：每子网主机数 = $2^{12} - 2 = 4094$
步骤5：子网间隔 = $2^{12} = 4096$（在第三字节为16）

子网列表（部分）：
| 子网 | 网络地址 |
|------|---------|
| 0 | 172.16.0.0/20 |
| 1 | 172.16.16.0/20 |
| 2 | 172.16.32.0/20 |
| ... | ... |
| 15 | 172.16.240.0/20 |

**例2：IP地址192.168.5.100，子网掩码255.255.255.192**

步骤1：192 = 11000000，所以借了2位，/26
步骤2：子网间隔 = $2^6 = 64$
步骤3：第三字节5，第四字节100
步骤4：100 ÷ 64 = 1 余 36
步骤5：网络地址 = 192.168.5.64（1 × 64 = 64）
步骤6：广播地址 = 192.168.5.127（64 + 63 = 127）
步骤7：可用范围 = 192.168.5.65 ~ 192.168.5.126

**例3：判断172.16.80.20属于哪个/20子网**

步骤1：/20意味着子网间隔 = 16（第三字节）
步骤2：80 ÷ 16 = 5
步骤3：所属子网 = 172.16.80.0/20
步骤4：广播 = 172.16.95.255
步骤5：可用范围 = 172.16.80.1 ~ 172.16.95.254

### 掩码与位数对照

| 十进制 | 二进制 | 1的个数 |
|--------|--------|--------|
| 0 | 00000000 | 0 |
| 128 | 10000000 | 1 |
| 192 | 11000000 | 2 |
| 224 | 11100000 | 3 |
| 240 | 11110000 | 4 |
| 248 | 11111000 | 5 |
| 252 | 11111100 | 6 |
| 254 | 11111110 | 7 |
| 255 | 11111111 | 8 |

## 直观理解

**快速计算法**：
- 问自己：掩码最后一个非255字节是多少？
- 查上表知道借了几位
- $2^{\text{剩余位数}}$ = 子网间隔
- 用目标数字 ÷ 间隔 确定子网

**口诀**："掩码算借位，间隔2的剩余次方，除法定子网"

## 协议关联

- 子网划分直接影响路由器的转发决策
- 不同子网间的通信必须经过路由器（三层转发）
- 同一子网内的通信通过ARP获取MAC后直接以太网帧传输
