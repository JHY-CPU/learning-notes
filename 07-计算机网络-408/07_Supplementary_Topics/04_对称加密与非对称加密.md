# 04_对称加密与非对称加密

## 核心概念

- **对称加密**：加密和解密使用同一把密钥 $K$
  - 发送方：$C = E_K(M)$，接收方：$M = D_K(C)$
  - 代表算法：DES、3DES、AES、IDEA、RC4
- **非对称加密（公钥加密）**：使用一对密钥——公钥（PK）和私钥（SK）
  - 加密：$C = E_{PK}(M)$，解密：$M = D_{SK}(C)$
  - 代表算法：RSA、ECC、ElGamal
- **DES（数据加密标准）**：密钥56位（+8位校验），分组64位，16轮Feistel结构
- **AES（高级加密标准）**：密钥128/192/256位，分组128位，替代DES
- **RSA**：基于大数分解难题，密钥长度通常1024/2048位
- **408重点**：对称vs非对称的优缺点对比、RSA算法步骤

## 原理分析

### DES算法流程

1. 初始置换 IP（64位明文重新排列）
2. 16轮Feistel迭代：
   - 每轮将64位分为左右各32位 $L_i, R_i$
   - $L_i = R_{i-1}$
   - $R_i = L_{i-1} \oplus f(R_{i-1}, K_i)$
3. 逆初始置换 $IP^{-1}$

**安全性**：56位密钥已被暴力破解（1999年DES挑战赛被破解），因此有了3DES和AES。

### RSA算法步骤

**密钥生成**：
1. 选择两个大素数 $p, q$
2. 计算 $n = p \times q$，$\phi(n) = (p-1)(q-1)$
3. 选择 $e$，满足 $1 < e < \phi(n)$ 且 $\gcd(e, \phi(n)) = 1$
4. 计算 $d$，满足 $e \times d \equiv 1 \pmod{\phi(n)}$
5. 公钥 $(e, n)$，私钥 $(d, n)$

**加解密**：
- 加密：$C = M^e \mod n$
- 解密：$M = C^d \mod n$

**示例**：$p=3, q=11$，则 $n=33$，$\phi(n)=20$
- 选 $e=3$（$\gcd(3,20)=1$）
- 求 $d$：$3d \equiv 1 \pmod{20}$，得 $d=7$
- 加密 $M=4$：$C = 4^3 \mod 33 = 64 \mod 33 = 31$
- 解密：$M = 31^7 \mod 33 = 4$ ✓

### 对称 vs 非对称对比

| 特性 | 对称加密 | 非对称加密 |
|------|---------|-----------|
| 密钥数量 | 1把（共享） | 2把（公钥+私钥） |
| 速度 | 快（适合大数据） | 慢（约慢100-1000倍） |
| 密钥分发 | 困难（需安全通道） | 容易（公钥公开） |
| 典型用途 | 数据加密 | 密钥交换、数字签名 |
| 代表算法 | AES, DES | RSA, ECC |

## 直观理解

- **对称加密**：像一把普通钥匙，你和朋友各有一把相同的钥匙，开同一把锁。问题是：你如何安全地把钥匙给朋友？
- **非对称加密**：像一个带锁的信箱，公钥是投信口（谁都能投），私钥是取信的钥匙（只有你有）
- **混合加密**：现实中用非对称加密传送"钥匙"（会话密钥），用对称加密传送"信件"（大量数据）
- **记忆口诀**："对称快但分发难，非对称慢但公开便"

## 协议关联

- **实际应用的混合方案**：
  - TLS/SSL：用RSA/DH交换会话密钥，用AES加密数据
  - IPsec：IKE用非对称/DH建立SA，ESP用AES加密
  - PGP：用RSA加密会话密钥，用IDEA/AES加密邮件内容
- **408常见陷阱**：
  - "非对称加密比对称加密更安全"——错，安全强度取决于密钥长度和算法
  - "RSA可以直接加密大文件"——实际中不这样做，速度太慢
  - "公钥可以用来解密"——公钥加密→私钥解密（加密场景），私钥签名→公钥验签（签名场景）
- **真题方向**：RSA计算、对称/非对称混合使用场景分析
