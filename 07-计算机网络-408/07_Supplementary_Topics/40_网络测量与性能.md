# 40_网络测量与性能

## 核心概念

- **网络性能指标**：
  - **带宽（Bandwidth）**：最大数据传输速率，单位bps
  - **吞吐量（Throughput）**：实际成功传输的数据速率
  - **延迟（Latency）**：数据从源到目的的总时间
  - **抖动（Jitter）**：延迟的变化量
  - **丢包率（Packet Loss Ratio）**：丢失包数/总发送包数
  - **利用率（Utilization）**：实际使用量/总容量
- **带宽 vs 吞吐量**：
  - 带宽 = 理论最大值（高速公路限速120km/h）
  - 吞吐量 = 实际传输量（实际车流量）
  - 吞吐量 < 带宽（受协议开销、拥塞等影响）
- **延迟测量工具**：
  - **ping**：测量RTT（往返时间），使用ICMP Echo
  - **traceroute/tracert**：测量到目的的路径和每跳延迟
- **带宽测量**：
  - **iperf**：测量TCP/UDP吞吐量
  - **pathchar/pathrate**：测量端到端可用带宽
- **408重点**：性能指标定义、ping/traceroute原理

## 原理分析

### ping工作原理

```
主机A → ICMP Echo Request → 主机B
主机A ← ICMP Echo Reply ← 主机B
```

- 发送ICMP Echo Request（类型8，代码0）
- 接收ICMP Echo Reply（类型0，代码0）
- 计算RTT = 收到Reply时间 - 发出Request时间

### traceroute工作原理

利用IP头的TTL字段：

```
第1跳：TTL=1，第一跳路由器收到后TTL减为0，丢弃，返回ICMP超时
第2跳：TTL=2，第一跳减为1转发，第二跳TTL减为0，返回ICMP超时
...
第n跳：TTL=n，到达目的，返回ICMP端口不可达（UDP方式）或Echo Reply（ICMP方式）
```

- **UDP方式**（Unix默认）：发送UDP包到高端口号，目的返回ICMP端口不可达
- **ICMP方式**（Windows默认）：发送ICMP Echo Request

### 吞吐量计算

**理想吞吐量**：
$$Throughput = \frac{L}{RTT + L/R}$$
- $L$：数据大小
- $RTT$：往返时间
- $R$：链路带宽

**带宽时延积（BDP）**：
$$BDP = R \times RTT$$
表示在"管道"中同时存在的最大数据量。BDP越大，需要的缓冲区越大。

### 利用率与延迟关系

$$D = \frac{D_0}{1 - U}$$
- $D_0$：空闲时延迟
- $U$：利用率
- 当 $U \to 1$ 时，$D \to \infty$（拥塞）

## 直观理解

- **带宽 vs 吞吐量**：水管直径（带宽）vs 实际水流量（吞吐量）
- **延迟组成**：处理（看地址）+ 排队（等红灯）+ 传输（过收费站）+ 传播（跑高速）
- **BDP**：管道里同时能装多少水（在途数据量）
- **利用率公式**：像餐厅——空座时上菜很快（$D_0$），满座时等很久（$D_0/(1-U)$）
- **traceroute**：像接力赛，每跑一棒（一跳）就报一次到

## 协议关联

- **ping与ICMP**：ping使用ICMP Echo Request/Reply
- **traceroute与ICMP/TTL**：利用TTL超时触发ICMP超时消息
- **与传输层**：TCP吞吐量受拥塞控制、BDP影响
- **与应用层**：视频应用根据带宽自适应码率
- **408考法**：ping/traceroute原理、带宽时延积计算、利用率与延迟关系
