# 31_流量整形与漏桶算法

## 核心概念

- **流量整形（Traffic Shaping）定义**：调整数据包的发送速率和模式，使输出流量平滑
- **流量整形 vs 流量监管（Policing）**：
  - 整形：缓存超速包，延迟发送（有缓冲区）
  - 监管：超速包直接丢弃或标记（无缓冲区）
- **漏桶算法（Leaky Bucket）**：
  - 固定速率输出，平滑所有突发
  - 桶满则丢弃新包
  - 输出速率恒定 = $r$（令牌/秒）
- **令牌桶算法（Token Bucket）**：
  - 以固定速率产生令牌
  - 发送包需要消耗等量令牌
  - 允许一定突发（桶容量范围内）
- **408重点**：漏桶和令牌桶的工作原理和区别

## 原理分析

### 漏桶算法

**工作原理**：
- 水（数据包）以可变速率流入桶
- 桶底部以恒定速率 $r$ 流出
- 桶容量为 $B$，满则溢出（丢包）

**数学模型**：
- 输出速率：恒定 $r$ bits/s
- 最大突发容忍：$B$ bits（桶容量）
- 允许的最大输入速率：无限制（但超速会丢包）

**特点**：
- 强制输出速率恒定
- 不允许任何突发
- 可能丢包（桶满时）

### 令牌桶算法

**工作原理**：
- 系统以速率 $r$ 产生令牌，放入容量为 $B$ 的桶
- 每发送1 bit消耗1个令牌
- 桶满则丢弃新令牌
- 桶空则停止发送（等待令牌积累）

**数学模型**：
- 最大突发长度（桶满后一次能发的最大数据量）：$M = B + r \times T$
  - $B$：桶容量
  - $r$：令牌产生速率
  - $T$：从上次发送到现在的时间
- 长期平均速率 $\leq r$
- 允许最大突发：$B$ bits（瞬间能发出的最大量）

**示例**：
桶容量 $B = 250$ KB，令牌速率 $r = 2$ MB/s，系统空闲了 $T = 0.5$ s：
$$M = 250KB + 2MB/s \times 0.5s = 250KB + 1MB = 1250KB$$

### 漏桶 vs 令牌桶对比

| 特性 | 漏桶 | 令牌桶 |
|------|------|--------|
| 输出速率 | 恒定 | 允许突发 |
| 是否丢包 | 桶满丢包 | 令牌不够等 |
| 缓冲区 | 需要 | 不需要 |
| 突发容忍 | 不允许 | 允许（桶容量内） |
| 应用场景 | 严格平滑流量 | 允许一定突发 |

## 直观理解

- **漏桶类比**：水龙头（不管水怎么倒进来，出水速率恒定）
- **令牌桶类比**：游乐园入场券（积攒了几张券就可以一次进几个人，但不能超过券的数量）
- **记忆口诀**："漏桶死板只出不突，令牌灵活攒了能突"

## 协议关联

- **与DiffServ**：DiffServ边缘路由器使用令牌桶进行流量整形和监管
- **与ATM**：ATM网络广泛使用漏桶算法
- **与TCP**：TCP的拥塞窗口也是一种速率控制机制
- **408常见考法**：漏桶vs令牌桶对比、令牌桶最大突发计算
