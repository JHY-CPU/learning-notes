# 02_网络攻击类型

## 核心概念

- **DoS攻击（拒绝服务）**：耗尽目标资源，使其无法正常服务
  - SYN Flood：发送大量SYN包但不完成三次握手，占满半连接队列
  - UDP Flood：发送大量UDP包淹没目标带宽
  - ICMP Flood（Ping of Death）：发送超大ICMP包使目标崩溃
- **DDoS攻击（分布式拒绝服务）**：利用多台"僵尸机"同时攻击
- **中间人攻击（MITM）**：攻击者截获并篡改通信双方的数据
- **重放攻击**：截获合法数据包后重复发送，欺骗接收方
- **IP欺骗**：伪造源IP地址，绕过基于IP的认证
- **DNS劫持**：篡改DNS解析结果，将用户引导至恶意网站
- **ARP欺骗**：伪造ARP响应，将攻击者MAC地址与网关IP绑定
- **408重点**：SYN Flood原理、ARP欺骗原理、重放攻击

## 原理分析

### SYN Flood攻击原理

正常三次握手：
1. 客户端发送 SYN（seq=x）
2. 服务器回复 SYN+ACK（seq=y, ack=x+1），服务器分配资源等待
3. 客户端发送 ACK（ack=y+1），连接建立

攻击过程：
1. 攻击者发送大量 SYN 包，伪造不同源IP
2. 服务器为每个 SYN 分配 TCB（传输控制块），回复 SYN+ACK
3. 由于源IP伪造，ACK永远不会到达
4. 服务器半连接队列（SYN Queue）被占满，拒绝正常连接

**防御方案**：SYN Cookie——服务器不分配资源，将状态编码在序列号中：
$$\text{ISN} = \text{Hash}(源IP, 源端口, 目的IP, 目的端口, 秘密值) + t$$
其中 $t$ 为时间戳，保证序列号随时间变化。

### ARP欺骗原理

1. 正常ARP：主机A广播"谁有IP_B？", B回复"我有，MAC是MAC_B"
2. 攻击：主机C主动发送ARP响应"IP_B对应的MAC是MAC_C"
3. 主机A更新ARP缓存，发给B的数据实际发给了C
4. C可选择转发给B（中间人）或丢弃（DoS）

**防御**：静态ARP绑定、ARP防火墙、802.1X认证

### 重放攻击与防护

$$\text{Nonce} = \text{随机数}, \quad \text{Timestamp} = \text{时间戳}$$

- Nonce方案：每次认证使用不同的随机数，服务器验证Nonce是否已使用
- Timestamp方案：验证时间戳是否在合理窗口内
- 序列号方案：维护计数器，拒绝旧序列号的报文

## 直观理解

- **SYN Flood**：就像很多人打电话给餐厅订座但都不确认，餐厅的预订本写满了，真正想订座的人订不到
- **DDoS**：一个人捣乱容易抓，一万个人同时捣乱就抓不过来了
- **ARP欺骗**：邮递员问"谁是张三？"，骗子抢先喊"我是张三"，于是张三的信都寄给了骗子
- **重放攻击**：有人录下了你的"芝麻开门"，下次在你不在时播放录音打开你的门
- **记忆口诀**："SYN不回、ARP乱答、重放要防时"

## 协议关联

- **SYN Flood与TCP**：利用TCP三次握手机制的缺陷，属于传输层攻击
- **ARP欺骗与链路层**：ARP工作在链路层，只能在局域网内实施
- **DNS劫持与应用层**：DNS是应用层协议，劫持需要篡改DNS响应
- **408常见组合题**：
  - 问"哪种攻击利用了TCP特性"→SYN Flood
  - 问"哪种攻击工作在数据链路层"→ARP欺骗
  - 问"如何防御重放攻击"→Nonce/时间戳/序列号
- **跨层关联**：ARP欺骗可导致中间人攻击，进而实施SSL剥离，实现从传输层到应用层的全面攻击
