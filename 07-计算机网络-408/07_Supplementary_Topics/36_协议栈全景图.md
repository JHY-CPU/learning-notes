# 36_协议栈全景图

## 核心概念

- **TCP/IP四层模型全景**：

```
┌─────────────────────────────────────────┐
│ 应用层                                   │
│ HTTP  HTTPS  FTP  SMTP  DNS  SNMP  SSH  │
├─────────────────────────────────────────┤
│ 传输层                                   │
│ TCP                        UDP           │
├─────────────────────────────────────────┤
│ 网络层                                   │
│ IP    ICMP    IGMP    ARP    RARP       │
├─────────────────────────────────────────┤
│ 网络接口层                                │
│ Ethernet   PPP   Wi-Fi   802.11         │
└─────────────────────────────────────────┘
```

- **OSI七层与TCP/IP对应**：

| OSI | TCP/IP | 功能 |
|-----|--------|------|
| 应用层 | 应用层 | 用户接口和应用服务 |
| 表示层 | (合并到应用层) | 数据格式转换、加密 |
| 会话层 | (合并到应用层) | 会话管理 |
| 传输层 | 传输层 | 端到端可靠/不可靠传输 |
| 网络层 | 网络层 | 路由和寻址 |
| 数据链路层 | 网络接口层 | 帧传输 |
| 物理层 | 网络接口层 | 比特传输 |

- **关键协议端口映射**：

| 协议 | 端口 | 传输层 |
|------|------|-------|
| HTTP | 80 | TCP |
| HTTPS | 443 | TCP |
| FTP | 21(控制)/20(数据) | TCP |
| SSH | 22 | TCP |
| Telnet | 23 | TCP |
| SMTP | 25 | TCP |
| DNS | 53 | UDP/TCP |
| DHCP | 67/68 | UDP |
| TFTP | 69 | UDP |
| SNMP | 161/162 | UDP |
| BGP | 179 | TCP |

## 原理分析

### 各层典型设备

- **物理层**：中继器、集线器
- **数据链路层**：网桥、交换机
- **网络层**：路由器
- **传输层**：防火墙（状态检测）、负载均衡器
- **应用层**：代理服务器、应用防火墙

### 协议间调用关系

```
应用层协议调用传输层：
  HTTP → TCP（可靠传输）
  DNS  → UDP（快速查询）/ TCP（区域传输）
  DHCP → UDP（广播发现）

传输层调用网络层：
  TCP/UDP → IP（寻址和路由）

网络层调用链路层：
  IP → ARP → Ethernet（获取MAC后封装帧）
```

## 直观理解

- **全景记忆法**：
  - 应用层：HTTP看网页、FTP传文件、DNS查域名、SMTP发邮件、SNMP管网络
  - 传输层：TCP打电话（先建立连接），UDP发短信（直接发）
  - 网络层：IP像邮政系统（路由信件），ICMP像投诉热线（报告错误）
  - 链路层：Ethernet像局域网快递（MAC地址找邻居）
- **端口记忆**：80（HTTP）、443（HTTPS）、21（FTP）、25（SMTP）、53（DNS）、22（SSH）

## 协议关联

- **垂直关联**：HTTP→TCP→IP→Ethernet（完整封装链）
- **水平关联**：对等层逻辑通信（HTTP↔HTTP, TCP↔TCP, IP↔IP）
- **ARP**：连接IP层和Ethernet层的桥梁
- **ICMP**：IP层的错误报告机制
- **408考法**：协议层次归属、端口号、OSI与TCP/IP对比
