# 22_网络管理概述

## 核心概念

- **网络管理定义**：对网络进行监控、配置、故障排除、性能优化的综合过程
- **网络管理五大功能域（FCAPS模型）**：
  - **F**ault Management（故障管理）：检测、隔离、修复故障
  - **C**onfiguration Management（配置管理）：设备配置的管理和变更
  - **A**ccounting Management（计费管理）：资源使用统计和计费
  - **P**erformance Management（性能管理）：监控和优化网络性能
  - **S**ecurity Management（安全管理）：安全策略实施和监控
- **网络管理模型**：
  - **管理者-代理模型（Manager-Agent）**：
    - 管理者（NMS）：运行管理软件的中心站
    - 代理（Agent）：运行在被管设备上的软件
- **管理协议**：SNMP（简单网络管理协议）是TCP/IP网络的标准管理协议
- **管理信息库（MIB）**：被管对象的数据库，使用树形结构
- **SMI（管理信息结构）**：定义MIB中对象的命名和数据类型
- **408重点**：SNMP协议、MIB概念、管理者-代理模型

## 原理分析

### SNMP体系结构

```
NMS（管理者）
  |
  | SNMP协议（UDP 161/162）
  |
  +--- Agent1（路由器）--- MIB1
  +--- Agent2（交换机）--- MIB2
  +--- Agent3（服务器）--- MIB3
```

### SNMP操作

- **GetRequest**：管理者查询代理的MIB变量值
- **GetNextRequest**：管理者查询下一个MIB变量
- **SetRequest**：管理者修改代理的MIB变量值
- **Response**：代理回复查询/设置结果
- **Trap**：代理主动向管理者发送告警（异步通知）

### MIB树形结构

```
iso(1)
  └── org(3)
       └── dod(6)
            └── internet(1)
                 ├── directory(1)
                 ├── mgmt(2)
                 │    └── mib-2(1)
                 │         ├── system(1)    -- 系统信息
                 │         ├── interfaces(2) -- 接口信息
                 │         ├── ip(4)         -- IP信息
                 │         └── tcp(6)        -- TCP信息
                 ├── experimental(3)
                 └── private(4)
```

每个对象有唯一OID（对象标识符），如 `1.3.6.1.2.1.1.1.0` = sysDescr

## 直观理解

- **管理者-代理模型**：像公司管理
  - 管理者（NMS）= 总部经理
  - 代理（Agent）= 各部门主管
  - MIB = 各部门的报表数据
  - GetRequest = 经理打电话问"这个月销售额多少？"
  - Trap = 部门主管主动打电话汇报"出大事了！"
- **MIB**：像一个巨大的数据库表，每个被管对象一行
- **SNMP操作记忆**：Get查、Set改、Trap报

## 协议关联

- **SNMP与UDP**：SNMP使用UDP传输（端口161管理，162 Trap）
- **SNMP与MIB**：SNMP操作的对象是MIB中定义的变量
- **SNMP与SMI**：SMI定义MIB的结构和数据类型
- **与网络层**：MIB中有IP路由表、接口统计等信息
- **408常见考法**：SNMP操作类型、MIB概念、管理者-代理模型
