# 25_拥塞控制跨层分析

## 核心概念

- **拥塞定义**：网络中分组过多，导致性能下降的现象
  - 吞吐量随负载增加先升后降
  - 延迟随负载增加急剧上升
- **拥塞控制 vs 流量控制**：
  - 流量控制：点对点，发送方速度匹配接收方处理能力
  - 拥塞控制：全局性，发送方速度适应网络承载能力
- **拥塞控制层次**：
  - **网络层**：路由器队列管理、RED、ECN
  - **传输层**：TCP拥塞控制（慢启动、拥塞避免、快重传、快恢复）
  - **应用层**：应用级限流（如HTTP/2的流控）
- **开环控制 vs 闭环控制**：
  - 开环：设计时预防（如接纳控制、流量整形）
  - 闭环：运行时调整（如TCP拥塞控制）
- **408重点**：拥塞控制与流量控制的区别、TCP拥塞控制算法

## 原理分析

### TCP拥塞控制四阶段

**1. 慢启动（Slow Start）**
- 初始 $cwnd = 1$ MSS
- 每收到一个ACK，$cwnd = cwnd + 1$ MSS
- 效果：每个RTT，$cwnd$ 翻倍（指数增长）
  $$cwnd = 2^{n} \text{ MSS（第n个RTT后）}$$
- 到达 $ssthresh$（慢启动阈值）时转为拥塞避免

**2. 拥塞避免（Congestion Avoidance）**
- 每个RTT，$cwnd = cwnd + 1$ MSS（线性增长）
  $$cwnd = cwnd + \frac{1}{cwnd} \text{ MSS（每收到一个ACK）}$$
- 即每个RTT增加1个MSS

**3. 快重传（Fast Retransmit）**
- 收到3个重复ACK时，立即重传丢失的报文段
- 不等超时计时器

**4. 快恢复（Fast Recovery）**
- 收到3个重复ACK后：
  - $ssthresh = cwnd / 2$
  - $cwnd = ssthresh + 3$ MSS
  - 进入拥塞避免（线性增长）
- **不回到慢启动**

### 拥塞状态转换

```
         收到3个重复ACK                超时
拥塞避免 ──────────────> 快恢复         ──────> 慢启动
   ↑                         │                       │
   │          新ACK           │      cwnd达到ssthresh  │
   └─────────────────────────┘                       │
   ↑                                                 │
   └─────────────────────────────────────────────────┘
```

### 网络层拥塞机制

- **RED（随机早期检测）**：
  - 队列未满时就开始随机丢包
  - 平均队列长度 $avg$，设最小阈值 $min_{th}$ 和最大阈值 $max_{th}$
  - $avg < min_{th}$：不丢包
  - $min_{th} \leq avg \leq max_{th}$：以概率 $p$ 丢包
  - $avg > max_{th}$：全部丢弃

- **ECN（显式拥塞通知）**：
  - 路由器不丢包，而是在IP头打标记
  - 接收方在ACK中通知发送方
  - 发送方降低发送速率

## 直观理解

- **拥塞类比**：高速公路堵车
  - 慢启动 = 匝道汇入，逐渐加速
  - 拥塞避免 = 正常行驶，缓慢提速
  - 拥塞 = 堵车，急刹车（cwnd减半）
  - 快重传 = 前车提醒"你掉东西了"，立即捡回来
- **拥塞控制 vs 流量控制**：
  - 流量控制：别给对方灌太多（对方处理不过来）
  - 拥塞控制：别给网络灌太多（网络承受不住）
- **记忆口诀**："慢启动翻倍找阈值，拥塞避免加一往上爬，三个重复快重传，超时回退慢启动"

## 协议关联

- **与TCP**：TCP拥塞控制是传输层的核心机制
- **与IP**：路由器的RED/ECN在网络层辅助拥塞控制
- **与应用层**：HTTP/2的流控制在应用层实现
- **跨层协作**：IP层ECN标记→TCP层调整cwnd→应用层调整发送速率
- **408高频考点**：TCP拥塞控制四阶段、cwnd变化计算、慢启动vs拥塞避免
