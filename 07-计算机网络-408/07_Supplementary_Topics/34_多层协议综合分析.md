# 34_多层协议综合分析

## 核心概念

- **协议栈层次回顾**：

| 层次 | 协议 | PDU | 主要功能 |
|------|------|-----|---------|
| 应用层 | HTTP, FTP, DNS, SMTP | 报文(Message) | 用户服务 |
| 传输层 | TCP, UDP | 段(Segment)/数据报 | 端到端通信 |
| 网络层 | IP, ICMP, ARP, IGMP | 包(Packet) | 路由转发 |
| 数据链路层 | Ethernet, PPP, HDLC | 帧(Frame) | 帧传输 |
| 物理层 | - | 比特(Bit) | 信号传输 |

- **封装与解封装**：
  - 发送：每层加头部（有时加尾部）
  - 接收：每层剥去头部
- **跨层信息交互**：
  - ICMP通知IP层错误（如目的不可达）
  - TCP根据ICMP调整拥塞窗口
  - ARP为IP提供MAC地址映射
  - DNS为应用层提供域名解析
- **408重点**：封装过程、各层PDU名称、跨层交互

## 原理分析

### 完整封装过程

以HTTP请求为例：

```
应用层：[HTTP请求数据]
传输层：[TCP头 | HTTP请求数据]          ← 段(Segment)
网络层：[IP头 | TCP头 | HTTP请求数据]    ← 包(Packet)
链路层：[帧头 | IP头 | TCP头 | HTTP请求数据 | 帧尾] ← 帧(Frame)
物理层：01010101...                       ← 比特流
```

### 各层头部关键字段

**以太网帧头**（14字节）：
- 目的MAC（6B）+ 源MAC（6B）+ 类型（2B）

**IP头**（20字节基本）：
- 版本、首部长度、TTL、协议、源IP、目的IP

**TCP头**（20字节基本）：
- 源端口、目的端口、序列号、确认号、窗口大小、标志位

### 常见协议组合

**Web浏览**：
```
HTTP → TCP → IP → Ethernet
```

**Ping**：
```
ICMP → IP → Ethernet
```

**DNS查询**：
```
DNS → UDP → IP → Ethernet
```

**ARP请求**：
```
ARP → Ethernet（直接在链路层，不经过IP）
```

## 直观理解

- **封装类比**：像快递包装
  - HTTP数据 = 你要寄的东西
  - TCP头 = 快递单（发件端口、收件端口、快递单号）
  - IP头 = 物流标签（发件地址、收件地址）
  - 以太网帧头 = 货车标签（这站、下一站）
- **解封装**：像拆快递，从外到内一层层拆
- **协议栈全景**：每层解决不同问题，相互配合完成通信

## 协议关联

- **ARP与IP**：IP包需要ARP解析下一跳MAC地址
- **ICMP与IP**：ICMP封装在IP中，通知IP层错误
- **TCP与IP**：TCP依赖IP的尽力而为服务，自己实现可靠传输
- **DNS与UDP/TCP**：小查询用UDP（53端口），大响应用TCP
- **408常见考法**：封装顺序、各层PDU名称、协议间调用关系
