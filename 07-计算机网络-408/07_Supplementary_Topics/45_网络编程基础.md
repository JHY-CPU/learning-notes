# 45_网络编程基础

## 核心概念

- **网络编程定义**：编写能在网络上相互通信的程序
- **核心概念——套接字（Socket）**：
  - 套接字 = IP地址 + 端口号
  - 是应用层与传输层之间的编程接口
  - 类型：流式套接字（SOCK_STREAM, TCP）、数据报套接字（SOCK_DGRAM, UDP）
- **C/S模型（客户端-服务器）**：
  - 服务器：被动打开，监听端口，等待连接
  - 客户端：主动发起连接
- **B/S模型（浏览器-服务器）**：
  - 特殊的C/S模型，客户端是浏览器
- **TCP编程流程**：
  ```
  服务器：                          客户端：
  socket()创建套接字               socket()创建套接字
  bind()绑定地址                   connect()发起连接
  listen()监听                     ↓
  accept()接受连接 ←───────────────┘
  read()/write()通信 ←──────────────→ read()/write()通信
  close()关闭                       close()关闭
  ```
- **UDP编程流程**：
  ```
  服务器：                          客户端：
  socket()创建套接字               socket()创建套接字
  bind()绑定地址                   sendto()发送数据
  recvfrom()接收数据 ←──────────────┘
  sendto()回复数据 ────────────────→ recvfrom()接收数据
  close()关闭                       close()关闭
  ```
- **408重点**：TCP和UDP编程模型的区别、Socket概念

## 原理分析

### TCP服务器关键函数

1. **socket()**：创建套接字
   - `socket(AF_INET, SOCK_STREAM, 0)` → 创建IPv4 TCP套接字

2. **bind()**：绑定IP和端口
   - `bind(sockfd, (struct sockaddr*)&addr, sizeof(addr))`

3. **listen()**：开始监听
   - `listen(sockfd, backlog)`，backlog指定连接队列长度

4. **accept()**：接受连接
   - 阻塞等待客户端连接，返回新的套接字用于通信

5. **read()/write()**：数据收发
   - TCP是流式的，数据无边界

### UDP vs TCP编程对比

| 特性 | TCP | UDP |
|------|-----|-----|
| 套接字类型 | SOCK_STREAM | SOCK_DGRAM |
| 连接 | 需要（三次握手） | 不需要 |
| 发送函数 | write()/send() | sendto() |
| 接收函数 | read()/recv() | recvfrom() |
| 数据边界 | 无（流式） | 有（数据报） |
| 可靠性 | 可靠 | 不可靠 |

## 直观理解

- **Socket类比**：Socket像电话机
  - socket() = 买电话机
  - bind() = 分配电话号码
  - listen() = 等电话响
  - accept() = 接电话
  - read/write = 通话
  - close() = 挂电话
- **TCP像打电话**：先拨号建立连接（三次握手），通话完挂断（四次挥手）
- **UDP像发短信**：不需要先打电话确认，直接发就行

## 协议关联

- **Socket与传输层**：Socket是应用层访问TCP/UDP的接口
- **TCP与Socket**：每个TCP连接对应一对Socket（源IP:端口, 目的IP:端口）
- **UDP与Socket**：UDP不需要连接，但Socket仍用于指定发送/接收地址
- **408考法**：TCP/UDP编程流程对比、Socket概念理解
