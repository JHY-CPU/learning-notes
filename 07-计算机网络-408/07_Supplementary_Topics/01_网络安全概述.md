# 01_网络安全概述

## 核心概念

- **网络安全目标（CIA三元组）**：机密性（Confidentiality）、完整性（Integrity）、可用性（Availability）
- **扩展安全目标**：认证性（Authentication）、不可否认性（Non-repudiation）、访问控制（Access Control）
- **安全机制**：加密、数字签名、访问控制、数据完整性、认证交换、流量填充、路由控制、公证
- **安全攻击分类**：
  - 被动攻击：截获/窃听（eavesdropping），难以检测，重在预防
  - 主动攻击：篡改、伪造、重放、拒绝服务（DoS），可检测，重在恢复
- **攻击目标**：中断（Integrity）、截获（Confidentiality）、篡改（Integrity）、伪造（Authenticity）
- **408常考**：CIA三元组对应关系、主动/被动攻击的区分

## 原理分析

### 网络安全体系结构

网络安全涉及通信的每个层次：

- **物理层**：防窃听、防物理破坏
- **数据链路层**：链路加密、MAC地址过滤
- **网络层**：IPsec、防火墙、IP过滤
- **传输层**：SSL/TLS、端到端加密
- **应用层**：PGP、S-HTTP、应用认证

### 安全服务与机制对应

| 安全服务 | 实现机制 |
|---------|---------|
| 数据机密性 | 对称加密、非对称加密 |
| 数据完整性 | 散列函数、MAC |
| 身份认证 | 数字证书、口令、生物识别 |
| 不可否认性 | 数字签名 |
| 访问控制 | 防火墙、ACL |

### 安全攻击的数学模型

设攻击者截获密文 $C = E_K(M)$，在已知明文攻击中，攻击者获得 $(M, C)$ 对，需解方程：
$$K = D(C, M)$$
即从密文和明文反推密钥。

## 直观理解

- **CIA三元组比喻**：机密性=信封（不让别人看内容），完整性=蜡封（不让别人篡改），可用性=邮局正常营业（服务不中断）
- **被动攻击**：像在旁边偷听别人说话，说话的人不知道有人在听
- **主动攻击**：像冒充别人说话、打断别人说话、往信里塞假内容
- **记忆口诀**："主改被窃"——主动攻击能改数据，被动攻击只能偷看

## 协议关联

- **与TCP/IP的关系**：TCP/IP原始设计未考虑安全，安全是后加的"补丁"
- **层次叠加**：应用层安全（PGP）、传输层安全（TLS）、网络层安全（IPsec）可同时使用
- **408常见陷阱**：
  - "加密一定能保证安全"——错，密钥管理不当仍不安全
  - "防火墙能防所有攻击"——错，防火墙无法防御内部攻击和新型攻击
  - 被动攻击"无法检测"是相对的，流量分析仍可能被发现
- **真题高频**：安全服务与攻击类型的对应关系、CIA三元组的具体含义
