# 46_套接字编程详解

## 核心概念

- **套接字地址结构**：
  - IPv4：`struct sockaddr_in`（包含sin_family、sin_port、sin_addr）
  - 通用：`struct sockaddr`（所有地址类型的统一接口）
- **套接字类型**：
  - **流式套接字（SOCK_STREAM）**：面向连接、可靠、双向，基于TCP
  - **数据报套接字（SOCK_DGRAM）**：无连接、不可靠，基于UDP
  - **原始套接字（SOCK_RAW）**：直接访问IP层，用于ping/traceroute等
- **阻塞 vs 非阻塞**：
  - 阻塞：调用后等待结果（默认模式）
  - 非阻塞：调用后立即返回（需轮询检查结果）
- **I/O多路复用**：
  - select()：监控多个套接字，有事件时返回
  - poll()：类似select，无fd数量限制
  - epoll()：Linux高效方案，适合大量连接
- **408重点**：Socket基本操作、TCP/UDP编程区别

## 原理分析

### TCP Socket编程示例流程

**服务器端**：
```c
int sockfd = socket(AF_INET, SOCK_STREAM, 0);  // 创建TCP套接字
struct sockaddr_in server_addr;
server_addr.sin_family = AF_INET;
server_addr.sin_port = htons(8080);              // 端口8080
server_addr.sin_addr.s_addr = INADDR_ANY;        // 监听所有接口
bind(sockfd, (struct sockaddr*)&server_addr, sizeof(server_addr));
listen(sockfd, 5);                                // 监听，队列长度5
int client_fd = accept(sockfd, NULL, NULL);       // 阻塞等待连接
char buf[1024];
int n = read(client_fd, buf, sizeof(buf));        // 读取数据
write(client_fd, "Hello", 5);                     // 发送响应
close(client_fd);
close(sockfd);
```

**客户端**：
```c
int sockfd = socket(AF_INET, SOCK_STREAM, 0);
struct sockaddr_in server_addr;
server_addr.sin_family = AF_INET;
server_addr.sin_port = htons(8080);
inet_pton(AF_INET, "192.168.1.1", &server_addr.sin_addr);
connect(sockfd, (struct sockaddr*)&server_addr, sizeof(server_addr)); // 三次握手
write(sockfd, "Hello", 5);                        // 发送数据
char buf[1024];
int n = read(sockfd, buf, sizeof(buf));            // 读取响应
close(sockfd);                                     // 四次挥手
```

### TCP连接的五元组

$$\text{连接} = (源IP, 源端口, 目的IP, 目的端口, 协议)$$

- 每个TCP连接由唯一五元组标识
- 服务器accept()返回的新套接字与客户端一对一通信
- 监听套接字持续接受新连接

### accept()的本质

- listen套接字：监听端口，接受SYN，完成三次握手
- accept()：从已完成连接队列中取出一个连接，返回新套接字
- **关键**：新套接字≠监听套接字，新套接字用于数据通信

## 直观理解

- **Socket五元组**：像快递单上的完整信息——谁寄的（源IP+端口）、寄给谁（目的IP+端口）、走什么快递（协议）
- **listen + accept**：listen是开门营业，accept是接待每一个进来的客人
- **服务器能同时服务多个客户端吗**：可以，通过多进程/多线程/IO多路复用

## 协议关联

- **Socket与TCP**：TCP连接通过Socket API操作
- **Socket与端口号**：端口号标识主机上的进程
- **Socket与IP**：IP地址标识主机
- **应用层与传输层**：Socket是应用层访问传输层的桥梁
- **408考法**：Socket概念、TCP/UDP编程流程对比
