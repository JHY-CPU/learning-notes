# 24_服务原语与服务访问点

## 核心概念

### 服务访问点（SAP）

- **定义**：同一系统中相邻两层实体之间交换信息的逻辑接口
- 每一层通过SAP向上层提供服务
- SAP是服务的"入口"和"出口"

**各层SAP示例**：

| 层次 | SAP |
|------|------|
| 应用层 | 用户接口（API） |
| 传输层 | 端口号（Port） |
| 网络层 | 协议号（Protocol） |
| 数据链路层 | 类型字段（EtherType） |

### 服务原语

- **定义**：上层调用下层服务时使用的基本操作命令
- 四种基本服务原语：

| 原语 | 方向 | 含义 |
|------|------|------|
| **请求（Request）** | 上层 → 下层 | 请求下层提供服务 |
| **指示（Indication）** | 下层 → 上层 | 下层通知上层某个事件 |
| **响应（Response）** | 上层 → 下层 | 上层对指示的响应 |
| **证实（Confirm）** | 下层 → 上层 | 下层对请求的确认 |

## 原理分析

### 有证实服务与无证实服务

**有证实服务**（Confirmed Service）：
```
实体A ──Request──→ 实体A的N层
                    ↓
               对等N层通信
                    ↓
实体B的N层 ──Indication──→ 实体B
实体B ──Response──→ 实体B的N层
                    ↓
               对等N层通信
                    ↓
实体A的N层 ──Confirm──→ 实体A
```

**无证实服务**（Unconfirmed Service）：
```
实体A ──Request──→ 实体A的N层 → 对等N层通信 → 实体B的N层 ──Indication──→ 实体B
```
（没有Response和Confirm）

### 例子：TCP连接建立

```
客户端                                      服务器
  │                                           │
  │── CONNECT.request ──→                     │
  │                                           │
  │                    ── CONNECT.indication ──→
  │                                           │
  │                    ←── CONNECT.response ───
  │                                           │
  │←── CONNECT.confirm ──                     │
```

## 直观理解

- **SAP**像银行的服务窗口：不同业务（不同服务）到不同窗口办理
- **服务原语**像办理业务的步骤：
  - Request = 你递上申请表
  - Indication = 银行通知对方有人要转账
  - Response = 对方确认同意
  - Confirm = 银行告诉你"转账成功"
- **端口号**就是传输层的SAP：应用层通过指定端口号来请求传输层的服务

## 协议关联

- 端口号是传输层SAP，这是理解Socket编程的基础
- 协议号（IP首部中的Protocol字段）是网络层SAP，标识上层使用的是TCP还是UDP
- SAP是理解分层通信的关键：每一层通过SAP与相邻层交互，通过协议与对等层通信
- 408考试中服务原语的考查通常以选择题形式出现
