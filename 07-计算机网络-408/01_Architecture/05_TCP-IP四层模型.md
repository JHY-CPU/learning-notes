# 05_TCP/IP四层模型

## 核心概念

- **TCP/IP 协议族**是互联网事实上的标准，先有实践后有理论模型
- 四层从下到上：**网络接口层、网际层（网络层）、传输层、应用层**
- TCP/IP 模型与 TCP/IP 协议族是两个不同的概念：
  - **TCP/IP 协议族**：所有与互联网相关的协议的集合
  - **TCP/IP 模型**：描述这些协议的四层参考模型

### 各层对应协议

| 层次 | 名称 | 核心协议 |
|------|------|---------|
| 4 | 应用层 | HTTP、FTP、SMTP、DNS、DHCP |
| 3 | 传输层 | TCP、UDP |
| 2 | 网际层（网络层） | IP、ICMP、IGMP、ARP（有争议） |
| 1 | 网络接口层 | 没有明确定义，与底层网络相关 |

### TCP/IP协议族的核心协议

**应用层**：
- HTTP：超文本传输协议
- FTP：文件传输协议
- SMTP：简单邮件传输协议
- DNS：域名系统
- DHCP：动态主机配置协议

**传输层**：
- TCP：传输控制协议，面向连接、可靠传输
- UDP：用户数据报协议，无连接、不可靠

**网际层**：
- IP：网际协议，核心中的核心
- ICMP：互联网控制报文协议
- IGMP：互联网组管理协议

## 原理分析

### TCP/IP模型的设计理念

- **端到端原则**：复杂的传输控制放在端系统（主机），网络核心尽量简单
- **尽力而为（Best Effort）**：网络层不保证可靠传输，由传输层的TCP来保证
- IP层提供不可靠、无连接的数据报服务

### TCP/IP模型的数据流

```
应用层数据
    ↓ （加TCP/UDP首部）
传输层报文段
    ↓ （加IP首部）
IP数据报
    ↓ （加帧头帧尾）
帧 → 物理层比特流
```

## 直观理解

- **TCP/IP四层模型**像快递系统简化版：
  - 应用层 = 你写的信/要寄的东西
  - 传输层 = 决定用顺丰（TCP可靠）还是普通邮寄（UDP快但不可靠）
  - 网际层 = 地址系统，负责路由
  - 网络接口层 = 具体用卡车/飞机/轮船运送
- **端到端原则**：网络像高速公路，只管运输不管包裹里装什么；包裹的安全、顺序由收发双方（端系统）自己负责

## 协议关联

- TCP/IP 模型是实际运行的标准，408考试重点
- 与 OSI 模型的对比是必考题型（详见文件06）
- IP 协议（网际层）和 TCP/UDP 协议（传输层）是整个408网络部分的核心
- ARP 协议的归属有争议：有时归网际层，有时归网络接口层——408考试一般归网络层
- 应用层协议（HTTP、DNS等）直接构建在传输层之上，考试中常考查它们使用 TCP 还是 UDP
