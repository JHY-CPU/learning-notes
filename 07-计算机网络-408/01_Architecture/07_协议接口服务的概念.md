# 07_协议接口服务的概念

## 核心概念

### 三个核心概念的定义

- **协议（Protocol）**：控制两个对等实体之间通信的规则的集合。协议是"水平的"。
- **接口/服务访问点（Interface/SAP）**：同一系统中相邻两层实体之间交换信息的地方
- **服务（Service）**：下层向上层提供的功能。服务是"垂直的"。

### 协议的三要素

1. **语法**：数据与控制信息的结构或格式（怎么写）
2. **语义**：需要发出何种控制信息，完成何种动作以及做出何种响应（什么意思）
3. **同步**：事件实现顺序的详细说明（什么时候做）

### 服务与协议的关系

| 维度 | 服务 | 协议 |
|------|------|------|
| 方向 | 垂直（上下层之间） | 水平（对等层之间） |
| 提供者 | 下层实体 | 同层对等实体 |
| 使用者 | 上层实体 | 对等实体本身 |
| 关系 | 实现细节对上层透明 | 服务的实现手段 |

## 原理分析

### 服务原语

上层实体使用下层服务时，需要通过**服务原语**进行交互：

| 原语类型 | 说明 |
|---------|------|
| **请求（Request）** | 上层向下层请求服务 |
| **指示（Indication）** | 下层通知上层有事件发生 |
| **响应（Response）** | 上层对指示的回应 |
| **证实（Confirm）** | 下层对上层请求的确认回应 |

```
用户A                                  用户B
  │                                      │
  │—— Request ——→                        │
  │                   —— Indication ——→  │
  │                   ←—— Response ————  │
  │←—— Confirm ———                       │
```

### 面向连接与无连接服务

- **面向连接服务**：通信前先建立连接，数据传送完毕后释放连接（如TCP）
- **无连接服务**：通信前不需要建立连接，每个分组独立传输（如UDP）

## 直观理解

- **协议**像两个国家之间的外交礼仪规定：双方必须遵守同一套规则才能沟通
- **接口/SAP**像公司的部门交接窗口：销售部（上层）向生产部（下层）下订单，交接窗口就是接口
- **服务原语**像打电话的过程：
  - Request：你拨号
  - Indication：对方电话响了
  - Response：对方接听
  - Confirm：你听到"嘟"声确认接通
- **三要素类比**：
  - 语法 = 文章格式（用中文写还是英文写）
  - 语义 = 文章含义（表达什么内容）
  - 同步 = 写作流程（先写摘要还是先写正文）

## 协议关联

- 协议三要素是408高频选择题考点
- 服务与协议的区别是常见陷阱题（"服务就是协议"——错误！）
- 面向连接/无连接服务的辨析贯穿传输层（TCP vs UDP）
- SAP 在 TCP/IP 中表现为端口号（传输层SAP）和协议号（网络层SAP）
