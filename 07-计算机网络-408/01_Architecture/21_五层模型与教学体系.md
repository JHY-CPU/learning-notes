# 21_五层模型与教学体系

## 核心概念

### 五层教学模型

408考研和谢希仁教材采用的**五层模型**是OSI和TCP/IP的折中：

| 层次 | 名称 | 功能 |
|------|------|------|
| 5 | 应用层 | 直接为用户应用提供服务 |
| 4 | 传输层 | 提供端到端的通信 |
| 3 | 网络层 | 路由选择与分组转发 |
| 2 | 数据链路层 | 相邻结点间的帧传输 |
| 1 | 物理层 | 比特流的透明传输 |

### 五层模型与OSI、TCP/IP的映射

```
    OSI(7层)      五层(5层)      TCP/IP(4层)
    应用层 ─┐
    表示层 ─┼──→ 应用层 ────→  应用层
    会话层 ─┘
    传输层 ────→ 传输层 ────→  传输层
    网络层 ────→ 网络层 ────→  网际层
  数据链路层 ─┐
    物理层 ──┼──→ 数据链路层
             └──→ 物理层 ────→ 网络接口层
```

### 五层模型各层的核心任务

1. **应用层**：通过应用进程间的交互来完成特定网络应用
2. **传输层**：负责主机中两个进程之间的通信
3. **网络层**：负责为分组交换网上的不同主机提供通信
4. **数据链路层**：将网络层交下来的IP数据报组装成帧
5. **物理层**：透明地传送比特流

## 原理分析

### 各层的数据处理

```
发送端：
应用数据 → [H5|数据] → [H4|H5|数据] → [H3|H4|H5|数据] → [H2|H3|H4|H5|数据|T2] → 比特流

接收端：
比特流 → [H2|H3|H4|H5|数据|T2] → [H3|H4|H5|数据] → [H4|H5|数据] → [H5|数据] → 应用数据
```

### 各层地址映射

| 层次 | 地址类型 | 作用范围 | 长度 |
|------|---------|---------|------|
| 应用层 | URL/域名 | 全局 | 不固定 |
| 传输层 | 端口号 | 主机内 | 16 bit |
| 网络层 | IP地址 | 全局 | 32/128 bit |
| 数据链路层 | MAC地址 | 局域网内 | 48 bit |

## 直观理解

- **五层模型**是教学和考试的最佳折中——比OSI简洁，比TCP/IP详细
- 五层模型像一栋五层办公楼：
  - 1楼（物理层）：水电管道工
  - 2楼（数据链路层）：前台收发室
  - 3楼（网络层）：物流调度部
  - 4楼（传输层）：客户服务部
  - 5楼（应用层）：业务经营部

## 协议关联

- 408考试以五层模型为基准，但也会涉及OSI和TCP/IP的对比
- 五层模型中没有会话层和表示层，它们的功能由应用层承担
- 各层协议的归属以五层模型为准进行复习
