# 27_网络标准化概述

## 核心概念

### 标准化的重要性

- 不同厂商的设备需要互连互通
- 标准确保了互操作性和兼容性
- 没有标准，网络就是一个个信息孤岛

### 主要标准化组织

| 组织 | 全称 | 主要职责 |
|------|------|---------|
| **ISO** | 国际标准化组织 | 制定OSI参考模型等国际标准 |
| **ITU** | 国际电信联盟 | 电信标准（前身CCITT） |
| **IEEE** | 电气和电子工程师协会 | 局域网标准（802系列） |
| **IAB** | 互联网体系结构委员会 | 互联网标准的最终裁决 |
| **IETF** | 互联网工程任务组 | 制定互联网协议标准 |
| **W3C** | 万维网联盟 | Web标准 |

### IEEE 802标准系列

| 标准 | 内容 |
|------|------|
| 802.3 | CSMA/CD（以太网） |
| 802.4 | 令牌总线 |
| 802.5 | 令牌环 |
| 802.11 | 无线局域网（WiFi） |

## 原理分析

### RFC的成熟度级别

```
互联网草案（Internet Draft）
      |
      v
建议标准（Proposed Standard）
      |
      v
草案标准（Draft Standard）—— 已废弃
      |
      v
互联网标准（Internet Standard）—— 有STD编号
```

### 标准的制定过程

1. 个人或团体提出草案
2. 在IETF会议上讨论
3. 经过多次修订和同行评审
4. 通过IESG审核后成为RFC
5. 经过广泛实施和测试后成为标准

## 直观理解

- **标准化**像交通规则：不同国家的人开车都需要遵守统一的规则，否则会出事故
- **RFC**像学术论文：任何人都可以提交，经过同行评审后可能成为正式标准
- **IEEE 802**像建筑规范：规定了局域网如何搭建

## 协议关联

- IEEE 802.3标准（以太网）在数据链路层章节详细展开
- IETF制定的TCP/IP协议族是互联网的基础
- OSI虽然由ISO制定但未被广泛采用，TCP/IP虽然不是正式标准却是事实标准
- 408考试中标准化组织的考查一般以选择题形式出现
