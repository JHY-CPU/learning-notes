# 37_环形与网状拓扑

## 核心概念

### 环形拓扑（Ring Topology）

- **结构**：结点首尾相连形成一个闭合的环
- **数据传输**：数据沿环单向传输，每个结点接收并转发
- **介质访问**：通常使用令牌传递（Token Passing）

**优点**：
- 无冲突：令牌控制，同一时刻只有一个结点发送
- 公平性好：每个结点获得令牌的机会均等
- 负载重时性能稳定

**缺点**：
- 任意结点或链路故障影响全环
- 扩展困难（添加结点需断开环）
- 延迟与结点数成正比

### 令牌传递机制

```
结点A → 结点B → 结点C → 结点D → 结点A（回到起点）
```

1. 令牌沿环传递
2. 有数据要发送的结点捕获令牌
3. 发送数据帧
4. 数据绕环一周后被发送方收回
5. 释放令牌

### 网状拓扑（Mesh Topology）

- **结构**：结点之间存在多条冗余路径
- **全连接**：$n$ 个结点需要 $\frac{n(n-1)}{2}$ 条链路
- **部分连接**：只在关键结点之间建立多条路径

**优点**：
- 可靠性极高：一条路径故障可走另一条
- 可选择最优路径
- 适合高可靠性要求的场景

**缺点**：
- 线缆和端口成本高
- 管理复杂
- 适用于核心网络，不适合接入层

## 原理分析

### 网状拓扑的冗余计算

全连接网状拓扑的链路数：

$$
L = \frac{n(n-1)}{2}
$$

例如：$n=5$ 时，$L = 10$ 条链路。

### 环形拓扑的典型应用

| 应用 | 说明 |
|------|------|
| 令牌环（802.5） | IBM的4/16 Mbps局域网 |
| FDDI | 100 Mbps光纤分布式数据接口 |
| SONET/SDH | 广域网光纤环，自愈能力强 |

## 直观理解

- **环形拓扑**像接力赛跑：每个人把接力棒传给下一个人
- **网状拓扑**像蜘蛛网：任意两点之间有多条路径
- **全连接 vs 部分连接**：全连接像每两座城市之间都有直飞航班，部分连接像只有大城市之间有直飞

## 协议关联

- 令牌传递与CSMA/CD是数据链路层的两种MAC方式
- 网状拓扑与路由选择算法密切相关
- 环形拓扑的自愈机制在SONET/SDH中有实际应用
