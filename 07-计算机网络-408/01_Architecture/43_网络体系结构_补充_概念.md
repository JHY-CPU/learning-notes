# 43_网络体系结构_补充_概念

## 核心概念

### 补充知识点

**1. 网络的分类补充**

按交换技术分类：
- **电路交换网络**：如传统电话网
- **报文交换网络**：如电报网
- **分组交换网络**：如互联网

按拓扑结构分类：
- 总线形、星形、环形、网状、树形

**2. 协议的补充理解**

- 协议是"水平的"：对等实体之间
- 协议的实现：用软件（应用层、传输层、网络层）或硬件（数据链路层、物理层）
- 协议是分层的：每一层都有自己的协议

**3. 网络性能指标汇总**

| 指标 | 定义 | 单位 |
|------|------|------|
| 速率 | 数据传送速率 | b/s |
| 带宽 | 最高数据率 | b/s |
| 吞吐量 | 实际数据量/时间 | b/s |
| 时延 | 数据传输时间 | s |
| 时延带宽积 | 传播时延 x 带宽 | bit |
| RTT | 往返时间 | s |
| 利用率 | 信道忙的时间比例 | % |

**4. 互联网的核心协议**

- IP：网际协议（网络层）
- TCP：传输控制协议（传输层）
- UDP：用户数据报协议（传输层）
- HTTP：超文本传输协议（应用层）

## 原理分析

### 网络体系结构的总结图

```
应用层：HTTP、FTP、SMTP、DNS
    |
传输层：TCP、UDP
    |
网络层：IP、ICMP、ARP
    |
数据链路层：Ethernet、PPP、WiFi
    |
物理层：光纤、铜线、无线
```

### 交换技术对比汇总

| 维度 | 电路交换 | 报文交换 | 分组交换 |
|------|---------|---------|---------|
| 建立连接 | 需要 | 不需要 | 不需要 |
| 资源占用 | 独占 | 共享 | 共享 |
| 存储转发 | 否 | 是 | 是 |
| 适用场景 | 实时通信 | 非实时 | 通用 |

## 直观理解

- 网络体系结构像一栋大楼的管理规范：每一层都有自己的职责和规则
- 理解体系结构有助于从全局角度把握网络的工作原理
