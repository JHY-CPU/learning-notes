# 06_OSI与TCP/IP模型对比

## 核心概念

### 层次对应关系

```
        OSI模型                    TCP/IP模型
┌──────────────────┐      ┌──────────────────┐
│    应用层 (7)     │      │                  │
├──────────────────┤      │   应用层         │
│    表示层 (6)     │      │                  │
├──────────────────┤      │                  │
│    会话层 (5)     │      │                  │
├──────────────────┤      ├──────────────────┤
│    传输层 (4)     │      │   传输层         │
├──────────────────┤      ├──────────────────┤
│    网络层 (3)     │      │   网际层         │
├──────────────────┤      ├──────────────────┤
│  数据链路层 (2)   │      │                  │
├──────────────────┤      │  网络接口层      │
│    物理层 (1)     │      │                  │
└──────────────────┘      └──────────────────┘
```

### 核心差异对比

| 对比维度 | OSI 模型 | TCP/IP 模型 |
|---------|---------|------------|
| 层次数 | 7层 | 4层 |
| 设计理念 | 先有模型，后有协议 | 先有协议，后有模型 |
| 网络层服务 | 提供面向连接和无连接服务 | 仅提供无连接服务（IP） |
| 传输层服务 | 仅提供面向连接服务 | 提供面向连接(TCP)和无连接(UDP) |
| 会话层/表示层 | 独立存在 | 合并到应用层 |
| 标准性质 | 理论标准 | 事实标准 |
| 通用性 | 通用，不偏向任何协议族 | 以 TCP/IP 协议族为核心 |

## 原理分析

### 网络层服务的差异（408高频考点）

- **OSI 网络层**：同时支持面向连接（虚电路）和无连接（数据报）服务
- **TCP/IP 网络层**：仅支持**无连接**服务（IP协议是数据报方式）

### 传输层服务的差异

- **OSI 传输层**：仅面向连接
- **TCP/IP 传输层**：同时提供 TCP（面向连接）和 UDP（无连接）

### 两种模型的共同点

1. 都采用分层的体系结构
2. 都有传输层和网络层，且功能相似
3. 都有应用层，尽管层数不同
4. 都有独立的网络接口概念

## 直观理解

- **OSI模型**像一个精心设计的蓝图，但最终没有大规模推广——"理想很丰满，现实很骨感"
- **TCP/IP模型**像实际跑在路上的车，虽然设计不够精美，但实用性极强
- 一个类比：OSI像教科书上的标准食谱，TCP/IP像餐厅实际操作的菜谱

### 408常见陷阱

1. **问**：OSI模型在哪一层实现端到端的可靠传输？
   **答**：传输层（第4层）
2. **问**：TCP/IP网络层提供什么服务？
   **答**：仅提供无连接的数据报服务
3. **问**：OSI模型中，哪一层既有面向连接又有无连接？
   **答**：网络层（传输层只有面向连接）

## 协议关联

- OSI的七层与TCP/IP四层的映射关系是408选择题常考内容
- 五层教学模型（详见文件21）是两者的折中，408考试通常以五层模型为准
- TCP/IP 应用层合并了 OSI 的高三层，实际开发中这些功能由应用程序自行实现
- ARP 的归属：OSI中属于数据链路层还是网络层有争议；TCP/IP中一般归网际层
