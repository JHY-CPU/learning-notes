# 45_计算机网络性能综合

## 核心概念

### 性能指标总览

| 指标 | 符号 | 公式 | 单位 |
|------|------|------|------|
| 速率 | $R$ | —— | b/s |
| 带宽 | $B$ | 最高数据率 | b/s |
| 吞吐量 | $T$ | $T \leq B$ | b/s |
| 发送时延 | $t_s$ | $L/R$ | s |
| 传播时延 | $t_p$ | $d/s$ | s |
| 总时延 | $t_d$ | $t_s + t_p + t_{proc} + t_q$ | s |
| 时延带宽积 | $G$ | $t_p \times B$ | bit |
| 往返时延 | RTT | $2t_p + t_{proc} + t_q$ | s |
| 利用率 | $U$ | —— | % |

### 关键公式汇总

**香农定理**：

$$
C = W \log_2(1 + \text{SNR})
$$

**奈奎斯特准则**：

$$
C = 2W \log_2 V
$$

**时延-利用率关系**：

$$
D = \frac{D_0}{1 - U}
$$

**信道利用率**：

$$
U = \frac{t_s}{t_s + \text{RTT}}
$$

## 原理分析

### 综合计算框架

对于一条端到端通信路径：

1. **计算各段链路的发送时延**：$t_{s,i} = L / R_i$
2. **计算各段链路的传播时延**：$t_{p,i} = d_i / v_i$
3. **计算总时延**：$t_d = \sum t_{s,i} + \sum t_{p,i} + (n-1) \times t_{\text{转发}}$
4. **计算信道利用率**：$U = t_s / (t_s + \text{RTT})$

### 性能优化方向

| 瓶颈 | 解决方向 |
|------|---------|
| 发送时延大 | 提高带宽（链路升级） |
| 传播时延大 | 减少距离（CDN缓存） |
| 排队时延大 | 拥塞控制、流量工程 |
| 利用率低 | 增大发送窗口、流水线 |

## 直观理解

- 网络性能像水管系统：水压（带宽）、水管长度（传播距离）、水龙头开关速度（发送速率）、水管堵塞程度（拥塞）都影响最终效果
- 没有单一指标能完全描述网络性能，需要综合考量

## 协议关联

- 速率和带宽与物理层和数据链路层相关
- 时延的各组成部分分布在各层
- RTT和利用率与传输层的TCP窗口大小密切相关
- 香农定理和奈奎斯特准则是物理层的理论基础
