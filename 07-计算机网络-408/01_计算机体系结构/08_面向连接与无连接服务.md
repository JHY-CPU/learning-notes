# 08_面向连接与无连接服务

## 核心概念

### 面向连接服务（Connection-Oriented）

- 通信之前必须先建立连接
- 数据传输过程中保持连接
- 通信结束后释放连接
- 典型代表：TCP（传输层）、虚电路（网络层）

### 无连接服务（Connectionless）

- 不需要预先建立连接
- 每个分组独立携带完整的目的地址
- 各分组可能走不同路径
- 典型代表：UDP（传输层）、IP数据报（网络层）

### 面向连接服务的三种实现方式

| 实现 | 说明 | 代表 |
|------|------|------|
| **电路交换** | 物理通路独占 | 电话网 |
| **虚电路** | 逻辑连接，资源共享 | X.25、帧中继 |
| **TCP连接** | 软件层面的逻辑连接 | TCP |

## 原理分析

### 虚电路与数据报对比

| 对比项 | 虚电路 | 数据报 |
|--------|--------|--------|
| 建立连接 | 需要（建立虚电路） | 不需要 |
| 目的地址 | 仅建立时使用 | 每个分组都携带 |
| 路由选择 | 建立时确定，所有分组走同一路由 | 每个分组独立选择 |
| 分组顺序 | 保证按序到达 | 不保证按序到达 |
| 差错控制 | 网络层负责 | 端系统负责 |
| 故障影响 | 虚电路经过的结点故障则全部失效 | 单个分组受影响 |

### 连接建立的三次握手

以TCP为例（传输层面向连接服务）：

```
客户端                         服务器
  │                              │
  │—— SYN, seq=x ——→            │
  │                              │
  │       ←—— SYN+ACK, seq=y,ack=x+1 ——│
  │                              │
  │—— ACK, ack=y+1 ——→          │
  │                              │
  │     （连接建立，开始传输）      │
```

## 直观理解

- **面向连接**像打电话：先拨号等对方接通，通话完毕说"再见"挂断
- **无连接**像寄明信片：每张明信片独立寄出，可能走不同邮路，可能丢失也可能乱序到达
- **虚电路**像一条专用的虚拟高速公路：虽然路上的车辆（带宽）不是独占的，但走的路线固定
- **TCP vs UDP**：
  - TCP = 挂号信/快递，有回执，保证送达
  - UDP = 平信，便宜快速，但不保证送达

## 协议关联

- 面向连接与无连接的对比是408**必考**内容
- 网络层：OSI同时支持虚电路和数据报，TCP/IP仅支持数据报
- 传输层：TCP面向连接，UDP无连接
- 面向连接不等于"可靠"：虚电路提供面向连接服务，但可靠性还取决于具体实现
- 考试陷阱：电路交换和虚电路都是面向连接的，但电路交换是物理独占，虚电路是逻辑共享
