# 08_UDP校验和计算

## 核心概念

- **UDP校验和**：用于检测UDP数据报在传输中是否出错
- **伪首部（Pseudo Header）**：UDP校验和计算时需要加入IP层的一些信息（伪首部不实际传输，仅用于计算）
- **校验和计算方法**：将数据按16位分组，求反码算术运算和，再取反码
- **IPv4中校验和可选**（填0表示不计算），IPv6中强制

### UDP伪首部格式（12字节，不实际传输）

```
+--------+--------+--------+--------+
|          源IP地址（32位）          |
+--------+--------+--------+--------+
|         目的IP地址（32位）         |
+--------+--------+--------+--------+
| 全0(8位) |协议号(8位)|  UDP长度(16位) |
+--------+--------+--------+--------+
```

| 字段 | 长度 | 值 |
|------|------|------|
| 源IP地址 | 32位 | 来自IP首部 |
| 目的IP地址 | 32位 | 来自IP首部 |
| 全零 | 8位 | 00000000 |
| 协议号 | 8位 | UDP = **17** |
| UDP长度 | 16位 | UDP首部+数据的总长度 |

## 原理分析

### 校验和计算步骤

1. **构造校验对象**：伪首部 + UDP首部 + UDP数据
2. **填充**：如果总长度不是16位的整数倍，末尾补0字节
3. **按16位分组**，进行**二进制反码求和**
   - 反码求和：两个16位数相加，如果有进位则加到结果的最低位
4. **取反码**：将求和结果取反，即为校验和

### 计算示例

假设有3个16位字：`0110011001100000`、`0101010101010101`、`0000111100001111`

```
  0110011001100000
+ 0101010101010101
  ----------------
  1011101110110101
+ 0000111100001111
  ----------------
  1100101011000100  （有进位1，加到最低位）
+         1
  ----------------
  1100101011000101
取反：0011010100111010 = 校验和
```

### 接收方校验

接收方将所有16位字（包括校验和字段）进行反码求和，如果结果为**全1**（即16个1），则说明无差错。

## 直观理解

- **考卷类比**：伪首部就像把学生姓名、学号也纳入计算范围——虽然这些信息在"考卷"（UDP报文）上没有，但计算校验和时需要用到，确保数据送对了人
- **校验和就像验证码**：把所有数字加起来取反，接收方再算一遍，结果为全1就说明数据完整

## 协议关联

- **与IP校验和的区别**：IP校验和只校验IP首部，UDP校验和校验伪首部+UDP首部+数据
- **与TCP校验和的相似**：TCP也使用伪首部计算校验和，方法相同
- **408常见考法**：
  - 手算UDP校验和（需要掌握反码求和方法）
  - 伪首部包含哪些信息（源IP、目的IP、协议号、UDP长度）
  - 伪首部的作用：确保数据报到达正确的目的主机和正确的协议
