# 74_拥塞窗口变化图_真题

## 核心概念

- **408对cwnd变化图的考查非常频繁**
- **通常是给定图让你分析，或给定条件让你画图**

### 408真题精选

**题目**：TCP的拥塞窗口cwnd在传输过程中的变化如图所示（典型的锯齿形）。请分析：
1. 哪些阶段是慢开始？
2. 哪些阶段是拥塞避免？
3. ssthresh的初始值是多少？
4. 发生了什么事件导致cwnd下降？

**解析**：
1. cwnd指数增长的阶段是**慢开始**
2. cwnd线性增长（每RTT +1）的阶段是**拥塞避免**
3. ssthresh = cwnd从指数增长转为线性增长时的值（即第一个锯齿的顶点/拐点）
4. cwnd骤降到1 → **超时**；cwnd减半 → **3个冗余ACK**

**题目**：设TCP的ssthresh=8 MSS。传输过程中发生以下事件：cwnd达到12时超时，cwnd达到10时收到3个冗余ACK。画出cwnd变化图。

**解析**：

阶段1（1→8→12）：
- 1→2→4→8（SS）→9→10→11→12（CA）
- 超时：ssthresh=6, cwnd=1

阶段2（1→6→...→10）：
- 1→2→4（SS）→...注意4<6继续SS→8（8≥6，转CA）→9→10
- 3冗余ACK：ssthresh=5, cwnd=8

阶段3：
- 8→9→10→...（CA）

**题目**：TCP拥塞控制中，当cwnd = ssthresh时，应该使用慢开始还是拥塞避免？

**解析**：使用**拥塞避免**。RFC 5681规定当cwnd ≥ ssthresh时使用拥塞避免。注意这是**≥**不是**>**。

## 直观理解

- **读图技巧**：指数增长=慢开始，线性增长=拥塞避免，骤降到1=超时，减半=冗余ACK

## 协议关联

- **与流量控制**：rwnd也可能在图中体现（如果rwnd < cwnd）
- **408做题技巧**：仔细区分拐点（ssthresh）和丢包点
