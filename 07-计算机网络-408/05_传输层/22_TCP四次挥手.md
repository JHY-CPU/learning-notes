# 22_TCP四次挥手

## 核心概念

- **四次挥手（Four-Way Handshake）**：TCP连接释放的过程，也称为**四报文挥手**
- **目的**：双方分别确认自己没有数据要发送了，依次关闭连接
- **为什么需要四次**：TCP是全双工的，每个方向的连接需要单独关闭
- **FIN消耗1个序号**

### 四次挥手详细过程

```
        客户端                                服务器
          |                                     |
          |  ① FIN=1, SEQ=u                    |
          |  ─────────────────────────────────→ |
          |                                     |  服务器状态: CLOSE_WAIT
          |  ② ACK=1, SEQ=v, ACK=u+1           |
          |  ←───────────────────────────────── |
          |  客户端状态: FIN_WAIT_2              |
          |                                     |
          |  ③ FIN=1, ACK=1, SEQ=w, ACK=u+1    |
          |  ←───────────────────────────────── |
          |                                     |
          |  ④ ACK=1, SEQ=u+1, ACK=w+1         |
          |  ─────────────────────────────────→ |
          |  客户端状态: TIME_WAIT              |
          |  等待2MSL后 → CLOSED               |
          |                                     |  服务器状态: CLOSED
```

### 各步骤详解

**第①次挥手（客户端→服务器）**：
- FIN=1, SEQ=u
- 客户端不再发送数据，但仍可接收数据
- 客户端状态：FIN_WAIT_1
- 服务器状态：CLOSE_WAIT

**第②次挥手（服务器→客户端）**：
- ACK=1, SEQ=v, ACK=u+1
- 确认收到客户端的FIN
- 此时服务器可能还有数据要发送
- 客户端收到后状态：FIN_WAIT_2

**第③次挥手（服务器→客户端）**：
- FIN=1, ACK=1, SEQ=w, ACK=u+1
- 服务器数据发送完毕，请求关闭
- 服务器状态：LAST_ACK

**第④次挥手（客户端→服务器）**：
- ACK=1, SEQ=u+1, ACK=w+1
- 确认收到服务器的FIN
- 客户端状态：TIME_WAIT
- 等待2MSL后进入CLOSED

## 原理分析

### TIME_WAIT状态（2MSL等待）

**为什么要等待2MSL（Maximum Segment Lifetime）**：
1. **确保最后一个ACK到达服务器**：如果④丢失，服务器会重传FIN，客户端在TIME_WAIT期间可以接收并重新确认
2. **确保本次连接的所有报文段从网络中消失**：防止旧连接的延迟报文段被新连接误收

$$2\text{MSL} = 2 \times \text{最大报文段生存时间}$$

通常MSL=2分钟，所以2MSL=4分钟。实际实现中MSL通常设为30秒~2分钟。

### 为什么需要四次而不是两次

TCP是全双工的：
- 客户端关闭发送：我发完了 → 服务器确认
- 服务器关闭发送：我发完了 → 客户端确认
- 服务器可能在收到FIN后还有数据要发，所以②和③不能合并（特殊情况除外）

## 直观理解

- **挂电话类比**：
  - A："我说完了"（FIN）
  - B："好的我知道"（ACK）
  - B："我也说完了"（FIN）
  - A："好的再见"（ACK）
  - A等一会儿确认B没再说什么 → 挂断

## 协议关联

- **与TCP状态转换**：FIN_WAIT_1 → FIN_WAIT_2 → TIME_WAIT → CLOSED（客户端）；CLOSE_WAIT → LAST_ACK → CLOSED（服务器）
- **408常见考法**：四次挥手过程、TIME_WAIT的意义、MSL的概念
