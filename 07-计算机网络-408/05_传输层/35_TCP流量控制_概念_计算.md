# 35_TCP流量控制_概念_计算

## 核心概念

- **rwnd在TCP首部中为16位字段**，最大65535字节
- **窗口扩大选项（Window Scale）**：TCP选项中可将窗口扩大至 $2^{30}-1$ 字节
- **发送窗口 = min(rwnd, cwnd)**
- **零窗口探测机制**：防止死锁

### 408计算题型

**题型1：给定rwnd和MSS，计算最大发送速率**

$$\text{最大发送速率} = \frac{\text{rwnd}}{\text{RTT}}$$

## 原理分析

### 计算示例1：基本窗口计算

**题目**：接收方通告rwnd=4000字节，MSS=1000字节。发送方最多能连续发送多少个报文段？

**解析**：
$$\text{最大报文段数} = \frac{\text{rwnd}}{\text{MSS}} = \frac{4000}{1000} = 4\text{个}$$

### 计算示例2：窗口与确认的配合

**题目**：发送窗口大小=4000字节，已发送了序号1~4000的数据。收到ACK=2001后，发送方还能发送多少新数据？

**解析**：
- ACK=2001确认了字节1~2000（2000字节）
- 窗口右移2000字节
- 发送窗口覆盖2001~6000
- 已发送未确认：2001~4000
- 可发送新数据：4001~6000，共**2000字节**

### 计算示例3：零窗口探测

**题目**：接收方发送了rwnd=0，之后有1000字节空间，但通知报文段丢失了。发送方如何恢复？

**解析**：
1. 发送方启动持续计时器
2. 超时后发送1字节的零窗口探测报文段
3. 接收方回复当前rwnd=1000
4. 发送方恢复发送

### 窗口大小约束总结

$$\text{实际发送窗口} = \min(\text{rwnd}, \text{cwnd})$$

- 在流量控制阶段（无拥塞），cwnd很大，窗口主要由rwnd限制
- 在拥塞阶段，cwnd较小，窗口主要由cwnd限制

## 直观理解

- **窗口像容量限额**：rwnd是对方告诉你"我还能收多少"，cwnd是你自己估计"网络能承受多少"，取较小值确保不会出问题

## 协议关联

- **与拥塞控制**：发送窗口 = min(rwnd, cwnd)
- **与应用层**：如果应用层读取数据慢，rwnd就小，发送方就被限速
- **408常见计算**：给定rwnd、cwnd、MSS计算最大发送量
