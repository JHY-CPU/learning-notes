# 69_套接字编程基础

## 核心概念

- **套接字（Socket）**：应用层与传输层之间的编程接口
- **Socket = (IP地址, 端口号)**
- **TCP套接字**：基于流的（SOCK_STREAM），面向连接
- **UDP套接字**：基于数据报的（SOCK_DGRAM），无连接

### TCP套接字编程流程

```
服务器:                              客户端:
socket() 创建                       socket() 创建
    ↓                                  ↓
bind() 绑定端口                     connect() 连接（触发三次握手）
    ↓                                  ↓
listen() 监听                       send()/write() 发送数据
    ↓                                  ↓
accept() 接受连接                   recv()/read() 接收数据
    ↓                                  ↓
recv()/send() 读写数据              close() 关闭（触发四次挥手）
    ↓
close() 关闭
```

### UDP套接字编程流程

```
服务器:                              客户端:
socket() 创建                       socket() 创建
    ↓                                  ↓
bind() 绑定端口                     sendto() 发送数据报
    ↓                                  ↓
recvfrom() 接收数据报               recvfrom() 接收响应
    ↓                                  ↓
sendto() 发送响应                   close()
    ↓
close()
```

## 原理分析

### 套接字与TCP状态

| socket调用 | TCP状态变化 |
|-----------|------------|
| listen() | CLOSED → LISTEN |
| connect() | CLOSED → SYN_SENT → ESTABLISHED |
| accept() | 收到SYN后创建新套接字 |
| close() | ESTABLISHED → FIN_WAIT_1 |

### 套接字缓冲区

- **发送缓冲区**：应用层write()的数据先放入发送缓冲区，TCP按需发送
- **接收缓冲区**：TCP接收的数据先放入接收缓冲区，应用层read()从中取出
- **缓冲区大小影响rwnd**：rwnd = 接收缓冲区剩余空间

## 直观理解

- **套接字像插座**：不同电器（应用进程）使用不同的插座（端口号），通过电线（网络）连接。插座就是应用和网络之间的接口

## 协议关联

- **与TCP状态转换**：socket API调用触发TCP状态转换
- **与流量控制**：套接字接收缓冲区大小决定rwnd
- **408常见考法**：socket = (IP, 端口号)的定义、TCP和UDP套接字的区别
