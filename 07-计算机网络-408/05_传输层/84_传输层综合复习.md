# 84_传输层综合复习

## 核心概念

- **传输层核心考点汇总**
- **考前快速复习清单**

### 必背知识点清单

**一、UDP**
- 首部8字节，4个字段：源端口、目的端口、长度、校验和
- 无连接、不可靠、面向报文
- 校验和使用伪首部（源IP、目的IP、协议号17、UDP长度）
- IPv4校验和可选，IPv6必须

**二、TCP首部**
- 最少20字节，最多60字节
- 6个标志位：URG ACK PSH RST SYN FIN
- 序号32位、确认号32位、窗口16位
- 数据偏移字段以4字节为单位

**三、三次握手**
- ①SYN=1, SEQ=x ②SYN=1,ACK=1,SEQ=y,ACK=x+1 ③ACK=1,SEQ=x+1,ACK=y+1
- SYN消耗1序号
- 防止历史连接建立

**四、四次挥手**
- ①FIN ②ACK ③FIN ④ACK
- FIN消耗1序号
- TIME_WAIT等待2MSL

**五、可靠传输**
- 序号、确认、超时重传、差错检测
- 累积确认：ACK=N表示0~N-1已收到
- 快速重传：3个冗余ACK
- RTO = SRTT + 4 × RTTVAR

**六、流量控制**
- rwnd（接收窗口），接收方通告
- 发送窗口 = min(rwnd, cwnd)
- 持续计时器解决零窗口死锁

**七、拥塞控制**
- 慢开始：cwnd指数增长，cwnd < ssthresh
- 拥塞避免：cwnd线性增长，cwnd ≥ ssthresh
- 超时：ssthresh = cwnd/2, cwnd = 1
- 3冗余ACK：ssthresh = cwnd/2, cwnd = ssthresh + 3

**八、TCP vs UDP**
- TCP：可靠、面向连接、字节流、点对点
- UDP：不可靠、无连接、报文、支持组播

## 原理分析

### 408计算题速查

| 计算类型 | 公式 |
|---------|------|
| 发送窗口 | min(rwnd, cwnd) |
| 最大吞吐量 | 发送窗口 / RTT |
| RTO | SRTT + 4 × RTTVAR |
| 慢开始 | cwnd × 2 / RTT |
| 拥塞避免 | cwnd + 1 / RTT |
| 超时后 | ssthresh = cwnd/2, cwnd = 1 |
| 冗余ACK后 | ssthresh = cwnd/2, cwnd = ssthresh+3 |
| ACK号 | 收到的最后一个字节序号 + 1 |

### 常见陷阱

1. SYN和FIN各消耗1个序号
2. 发送窗口取min不是max
3. UDP有校验和（不是没有差错检测）
4. TIME_WAIT是客户端状态
5. DNS区域传送用TCP
6. cwnd = ssthresh时用拥塞避免（≥ 不是 >）
7. 端口号标识进程不是主机
8. 伪首部不实际传输
