# 38_慢开始算法

## 核心概念

- **慢开始（Slow Start）**：TCP拥塞控制的初始阶段，cwnd从1 MSS开始，每收到一个ACK，cwnd增加1 MSS
- **效果**：cwnd**指数增长**——每个RTT翻倍
- **"慢"的含义**：不是增长慢，而是相对于一开始就发送大量数据来说是"慢"的起步
- **慢开始门限（ssthresh）**：当cwnd增长到ssthresh时，转入拥塞避免

### 慢开始的增长规律

| RTT | cwnd（MSS） | 说明 |
|-----|-------------|------|
| 1 | 1 | 发送1个MSS |
| 2 | 2 | 收到1个ACK，cwnd+=1，发送2个MSS |
| 3 | 4 | 收到2个ACK，cwnd+=2，发送4个MSS |
| 4 | 8 | 收到4个ACK，cwnd+=4，发送8个MSS |
| 5 | 16 | 收到8个ACK，cwnd+=8 |

**规律**：每个RTT，cwnd翻倍（指数增长）

## 原理分析

### 慢开始的实现

1. 初始化：cwnd = 1 MSS，ssthresh = 初始值
2. 每收到一个ACK：cwnd += 1 MSS
3. 一个RTT内能收到cwnd个ACK，所以cwnd在一个RTT内翻倍
4. 当cwnd ≥ ssthresh时，转入拥塞避免

### 何时使用慢开始

- 连接刚建立时
- 超时重传后（cwnd重置为1，ssthresh减半）
- cwnd < ssthresh时

### 慢开始与发送窗口

$$\text{实际发送窗口} = \min(\text{rwnd}, \text{cwnd})$$

在慢开始阶段，cwnd通常远小于rwnd，所以窗口主要由cwnd限制。

## 直观理解

- **慢起步像学骑车**：刚开始小心地骑（1 MSS），越骑越稳越骑越快（指数增长），直到接近安全速度上限（ssthresh），然后平稳骑行（拥塞避免）
- **指数增长的直觉**：1→2→4→8→16→32→64→128...增长非常快，所以叫"慢"开始只是相对而言

## 协议关联

- **与拥塞避免的转换**：cwnd ≥ ssthresh时，从慢开始切换到拥塞避免
- **与超时的关系**：超时 → ssthresh = cwnd/2, cwnd = 1 → 重新慢开始
- **与快恢复的关系**：3个冗余ACK → ssthresh = cwnd/2, cwnd = ssthresh + 3 → 拥塞避免（跳过慢开始）
- **408常见考法**：画出cwnd变化曲线、计算每一轮的cwnd值
