# 09_DNS_概念_真题

## 核心概念

- **真题来源**：408统考及各校自命题中涉及DNS的选择题和简答题
- **高频考点**：DNS记录类型、查询过程、缓存机制、端口号
- **出题规律**：通常结合具体场景考查DNS的实际应用

## 原理分析

### 真题1（选择题）

**题目**：下列关于DNS的说法正确的是（  ）
A. DNS使用TCP协议，端口号53
B. DNS只能将域名解析为IPv4地址
C. DNS采用层次化的域名空间
D. DNS服务器之间只能使用递归查询

**答案**：C

**解析**：
- A错误：DNS主要使用UDP协议，端口53，大报文才用TCP
- B错误：DNS可以解析IPv4（A记录）和IPv6（AAAA记录）
- C正确：DNS采用树状层次结构
- D错误：服务器之间通常使用迭代查询

### 真题2（选择题）

**题目**：某公司域名为`company.com`，邮件服务器域名为`mail.company.com`（IP: 200.1.1.1），则DNS中至少需要（  ）
A. 1条A记录和1条MX记录
B. 1条A记录和1条NS记录
C. 2条A记录
D. 1条MX记录

**答案**：A

**解析**：
- 需要A记录：`(mail.company.com, 200.1.1.1, A)` - 给邮件服务器一个IP
- 需要MX记录：`(company.com, mail.company.com, MX)` - 告诉别人邮件发到哪里
- 共2条记录，1条A记录+1条MX记录

### 真题3（选择题）

**题目**：在DNS的递归查询中，如果本地DNS服务器没有缓存，则（  ）
A. 本地DNS服务器直接返回错误
B. 本地DNS服务器代替主机完成查询
C. 本地DNS将查询转发给主机
D. 本地DNS只查询根服务器

**答案**：B

**解析**：
- 递归查询中，被查询的服务器必须返回最终结果
- 本地DNS收到主机的递归查询请求后，会代替主机完成所有查询步骤
- 本地DNS会依次查询根DNS、TLD、权威DNS，直到获得答案

### 真题4（综合题）

**题目**：描述用户访问`www.example.com`时，DNS的查询过程（假设本地无缓存）。

**参考答案**：
1. 浏览器调用解析器，向本地DNS服务器发送递归查询请求
2. 本地DNS向根DNS服务器发送迭代查询 → 返回`.com`TLD地址
3. 本地DNS向`.com`TLD查询 → 返回`example.com`权威DNS地址
4. 本地DNS向`example.com`权威DNS查询 → 返回`www.example.com`的IP
5. 本地DNS缓存结果并返回给主机
6. 主机获得IP后，发起HTTP连接

## 直观理解

**真题做题技巧**：
- DNS端口号必然是53，不是53的选项可以直接排除
- "只能"、"必须"等绝对化表述通常是错误选项
- DNS缓存是考试重点，要理解缓存的工作机制
- 递归查询 = "我帮你查到底"，迭代查询 = "我告诉你找谁"

**常见错误**：
- 混淆递归查询和迭代查询的定义
- 忘记DNS也可以使用TCP协议
- 误认为MX记录直接包含IP地址

## 协议关联

- **DNS与HTTP**：DNS解析是HTTP请求的前置步骤
- **DNS与SMTP**：MX记录在邮件系统中必不可少
- **DNS与DHCP**：DHCP配置客户端的DNS服务器地址
- **408综合题常见组合**：DNS + HTTP + TCP + 路由 → 跨层综合分析
