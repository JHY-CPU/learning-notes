# 58_TCP连接与HTTP关系

## 核心概念

- **HTTP与TCP的关系**：HTTP基于TCP，使用TCP的可靠传输
- **TCP连接建立**：三次握手（SYN→SYN+ACK→ACK）
- **TCP连接释放**：四次挥手（FIN→ACK→FIN→ACK）
- **HTTP连接模式**：
  - HTTP/1.0：每个请求建立新TCP连接
  - HTTP/1.1：TCP连接可以复用
  - HTTP/2：单个TCP连接上多路复用
- **408考试重点**：TCP三次握手对HTTP延迟的影响

## 原理分析

### TCP三次握手与HTTP

1. **第一次握手**：客户端 → 服务器 SYN
2. **第二次握手**：服务器 → 客户端 SYN+ACK
3. **第三次握手**：客户端 → 服务器 ACK
4. **HTTP请求**：客户端 → 服务器 GET/POST
5. **HTTP响应**：服务器 → 客户端 200 OK

**时间线**：
- TCP握手：1 RTT
- HTTP请求响应：1 RTT
- 首次HTTP请求总时间：2 RTT

### TCP四次挥手与HTTP

1. **第一次挥手**：客户端 → 服务器 FIN
2. **第二次挥手**：服务器 → 客户端 ACK
3. **第三次挥手**：服务器 → 客户端 FIN
4. **第四次挥手**：客户端 → 服务器 ACK

### HTTP连接模式对比

| 模式 | TCP连接数 | 首次延迟 | 后续延迟 |
|------|----------|---------|---------|
| 非持久 | 每个对象1个 | 2 RTT | 2 RTT |
| 持久非流水线 | 1个 | 2 RTT | 1 RTT |
| 持久流水线 | 1个 | 2 RTT | 1 RTT（连续） |

### TCP对HTTP的影响

1. **延迟**：TCP三次握手增加1 RTT延迟
2. **队头阻塞**：TCP层的丢包影响所有HTTP流
3. **连接开销**：每次建立/释放TCP连接有开销
4. **HTTP/2多路复用**：单个TCP连接上并行传输

## 直观理解

**TCP三次握手就像打电话确认**：
1. 你："喂，能听到吗？"（SYN）
2. 对方："能听到，你能听到我吗？"（SYN+ACK）
3. 你："能听到，开始说正事吧"（ACK）
4. 你说正事（HTTP请求）
5. 对方回答（HTTP响应）

**记忆技巧**：
- TCP握手 = 1 RTT
- HTTP请求响应 = 1 RTT
- 首次HTTP总延迟 = 2 RTT
- 持久连接复用TCP，减少握手次数

## 协议关联

- **TCP与HTTP**：HTTP基于TCP
- **TCP与UDP**：HTTP用TCP，DNS用UDP
- **TCP与HTTP/2**：HTTP/2多路复用但仍受TCP队头阻塞影响
- **TCP与HTTP/3**：HTTP/3用QUIC（UDP）避免TCP问题
- **408考点**：TCP三次握手对HTTP延迟的影响，RTT计算
