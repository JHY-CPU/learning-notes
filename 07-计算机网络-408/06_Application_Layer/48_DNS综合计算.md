# 48_DNS综合计算

## 核心概念

- **DNS综合计算题型**：涉及DNS查询时间、缓存命中率、记录配置
- **关键公式**：
  - DNS查询时间 = 各级RTT之和
  - 缓存命中时查询时间 = 0（或本地RTT）
  - 平均查询时间 = 命中率 × 0 + (1-命中率) × 查询时间
- **408考试重点**：DNS查询过程的时间计算、缓存对查询的影响

## 原理分析

### 典型计算1：DNS查询时间

**题目**：主机到本地DNS的RTT=2ms，本地DNS到根DNS的RTT=10ms，到TLD的RTT=20ms，到权威DNS的RTT=30ms。无缓存时查询需要多长时间？

**解答**（迭代查询）：
- 主机→本地DNS：2ms（请求）
- 本地DNS→根DNS：10ms（请求）+ 10ms（响应）= 20ms
- 本地DNS→TLD：20ms（请求）+ 20ms（响应）= 40ms
- 本地DNS→权威DNS：30ms（请求）+ 30ms（响应）= 60ms
- 本地DNS→主机：2ms（响应）

**总时间**：2 + 20 + 40 + 60 + 2 = 124ms

简化模型（忽略响应时间）：
$$T = RTT_1 + RTT_2 + RTT_3 + RTT_4 = 2 + 10 + 20 + 30 = 62ms$$

### 典型计算2：缓存影响

**题目**：本地DNS缓存了.com TLD地址（命中率60%），查询`www.abc.com`的平均时间是多少？

**解答**：
- 缓存命中（60%）：跳过根DNS查询，只需查询TLD和权威DNS
  - 时间 = 2 + 20 + 30 = 52ms
- 缓存未命中（40%）：完整查询
  - 时间 = 2 + 10 + 20 + 30 = 62ms
- 平均时间 = 0.6 × 52 + 0.4 × 62 = 31.2 + 24.8 = 56ms

### 典型计算3：多级缓存

**题目**：本地DNS缓存命中率为80%，缓存命中时响应时间为1ms，未命中时查询时间为100ms。求平均响应时间。

**解答**：
$$T_{avg} = P_{hit} \times T_{hit} + (1 - P_{hit}) \times T_{miss}$$
$$T_{avg} = 0.8 \times 1 + 0.2 \times 100 = 0.8 + 20 = 20.8ms$$

### 典型计算4：DNS记录配置

**题目**：某公司有以下信息：
- Web服务器：www.company.com → 200.1.1.1
- 邮件服务器：mail.company.com → 200.1.1.2
- 权威DNS：ns.company.com → 200.1.1.3

需要配置哪些DNS记录？

**解答**：
1. A记录：`(www.company.com, 200.1.1.1, A)`
2. A记录：`(mail.company.com, 200.1.1.2, A)`
3. A记录：`(ns.company.com, 200.1.1.3, A)`
4. MX记录：`(company.com, mail.company.com, MX, 10)`
5. NS记录：`(company.com, ns.company.com, NS)`

## 直观理解

**做题技巧**：
- DNS查询时间 = 各级RTT之和
- 缓存命中 = 跳过已缓存的查询步骤
- 平均时间 = 命中率 × 命中时间 + 未命中率 × 未命中时间
- DNS记录配置：A记录+MX记录+NS记录

## 协议关联

- **DNS与UDP**：DNS查询使用UDP
- **DNS与HTTP**：HTTP请求前需要DNS解析
- **DNS与缓存**：缓存显著减少DNS查询时间
- **408常见组合**：DNS查询时间 + 缓存命中率 = 综合计算题
