# 18_电子邮件系统概述

## 核心概念

- **电子邮件（E-mail）**：互联网上最广泛的应用之一
- **电子邮件系统三个主要组成**：
  - **用户代理（User Agent, UA）**：邮件客户端（如Outlook、Foxmail）
  - **邮件服务器（Mail Server）**：存储和转发邮件
  - **邮件传输协议**：SMTP（发送）、POP3/IMAP（接收）
- **邮件传输过程**：
  - 发送方UA → 发送方邮件服务器（SMTP）
  - 发送方邮件服务器 → 接收方邮件服务器（SMTP）
  - 接收方邮件服务器 → 接收方UA（POP3或IMAP）
- **408考试重点**：邮件系统组成、协议选择（SMTP/POP3/IMAP）、端口号

## 原理分析

### 邮件系统工作流程

1. **发送邮件**：
   - 用户在UA中撰写邮件
   - UA使用SMTP将邮件发送到发送方邮件服务器
   - 发送方邮件服务器使用SMTP将邮件转发到接收方邮件服务器

2. **接收邮件**：
   - 接收方邮件服务器存储邮件
   - 用户使用UA通过POP3或IMAP从服务器获取邮件

3. **邮件格式**：
   - **信封（Envelope）**：SMTP协议使用，包含发件人和收件人地址
   - **首部（Header）**：邮件头部，包含主题、发件人、收件人、日期等
   - **正文（Body）**：邮件内容

### SMTP协议（发送邮件）

- SMTP（Simple Mail Transfer Protocol）：简单邮件传输协议
- 端口号：**25**
- 使用TCP协议
- 采用C/S模式
- SMTP是"推"协议：发送方将邮件推送到服务器

### POP3协议（接收邮件）

- POP3（Post Office Protocol v3）：邮局协议第3版
- 端口号：**110**
- 使用TCP协议
- POP3是"拉"协议：客户端从服务器拉取邮件
- 特点：下载后默认删除服务器上的邮件

### IMAP协议（接收邮件）

- IMAP（Internet Message Access Protocol）：互联网邮件访问协议
- 端口号：**143**
- 使用TCP协议
- IMAP是"拉"协议：客户端从服务器获取邮件
- 特点：邮件保留在服务器上，支持多设备同步

## 直观理解

**电子邮件系统就像邮政系统**：
- **用户代理** = 你家的邮箱和写信工具
- **SMTP** = 邮递员送信（推：从你家送到邮局，再送到对方邮局）
- **邮件服务器** = 各级邮局（存储和转发）
- **POP3** = 你去邮局取信（取完就拿走，邮局不保留）
- **IMAP** = 你在邮局看信（信还在邮局，可以在多处看）

**记忆技巧**：
- "SMTP发25，POP3取110，IMAP看143"
- SMTP是推（Push），POP3/IMAP是拉（Pull）
- POP3下载后删除，IMAP保留服务器副本

## 协议关联

- **SMTP与TCP**：SMTP使用TCP的可靠传输，端口25
- **POP3与TCP**：POP3使用TCP，端口110
- **IMAP与TCP**：IMAP使用TCP，端口143
- **SMTP与DNS**：发送邮件前先查MX记录找到接收方邮件服务器
- **408常见组合**：
  - SMTP + POP3 = 传统邮件系统
  - SMTP + IMAP = 现代邮件系统（支持多设备）
- **陷阱**：发送邮件用SMTP（25），接收邮件用POP3（110）或IMAP（143）
