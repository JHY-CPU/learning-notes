# 64_P2P协议详解

## 核心概念

- **P2P（Peer-to-Peer）**：对等网络，节点之间直接通信
- P2P没有永远在线的中心服务器
- 每个节点既是客户也是服务器（对等方）
- **自扩展性（Self-scalability）**：新节点加入带来服务能力
- **408考试重点**：P2P特点、文件分发时间计算、与C/S对比

## 原理分析

### P2P工作原理

1. **节点发现**：
   - 集中式：Tracker服务器（如BitTorrent）
   - 分布式：DHT（分布式哈希表）
   - 混合式：两者结合

2. **文件分发**：
   - 文件被分成多个块（chunk）
   - 节点同时从多个对等方下载不同块
   - 节点下载后立即成为上传源

3. **文件共享**：
   - 节点可以同时上传和下载
   - 节点可以下载多个文件的不同块

### P2P vs C/S文件分发时间

**C/S模式**：
$$T_{CS} = \max\left\{\frac{NF}{u_s}, \frac{F}{d_{min}}\right\}$$

**P2P模式**：
$$T_{P2P} = \max\left\{\frac{F}{u_s}, \frac{F}{d_{min}}, \frac{NF}{u_s + \sum_{i=1}^{N}u_i}\right\}$$

其中：
- $N$：对等方数量
- $F$：文件大小
- $u_s$：服务器上传速率
- $d_{min}$：最小下载速率
- $u_i$：第i个对等方的上传速率

### BitTorrent协议

**核心概念**：
- **Torrent文件**：包含Tracker地址和文件哈希
- **Tracker**：跟踪参与下载的节点
- **Chunk**：文件被分成的小块（通常256KB）
- **Tit-for-tat**：激励机制，上传多的优先下载

**工作过程**：
1. 客户端下载Torrent文件
2. 连接Tracker获取对等方列表
3. 与多个对等方建立连接
4. 同时从多个对等方下载不同块
5. 下载的块立即可被其他节点下载

### DHT（分布式哈希表）

- **作用**：去中心化的节点发现
- **原理**：每个节点负责一部分哈希空间
- **优点**：无中心服务器，更健壮
- **实现**：Kademlia协议

## 直观理解

**P2P就像百家宴**：
- 每个人带一道菜来（提供资源）
- 每个人也可以吃别人的菜（获取资源）
- 人越多，菜越丰富（自扩展性）
- C/S就像餐厅：厨师做菜，顾客吃（单向服务）

**记忆技巧**：
- P2P = "人人都是服务员和顾客"
- 自扩展性 = "人越多，服务越好"
- BitTorrent = "人人为我，我为人人"

## 协议关联

- **P2P与C/S**：对比是常考内容
- **P2P与文件分发时间计算**：408重点
- **P2P与DHT**：DHT是去中心化的P2P发现机制
- **408考点**：P2P自扩展性、文件分发时间计算
