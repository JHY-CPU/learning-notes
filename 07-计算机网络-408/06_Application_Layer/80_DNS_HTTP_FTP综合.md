# 80_DNS_HTTP_FTP综合

## 核心概念

- **DNS + HTTP + FTP综合**：涉及三个应用层协议的综合分析
- **关键考点**：
  - DNS解析是HTTP和FTP的前置步骤
  - HTTP单连接无状态，FTP双连接有状态
  - 各协议端口号和传输层选择
- **408考试重点**：协议对比和综合分析

## 原理分析

### 综合题1：协议对比

**题目**：比较DNS、HTTP、FTP的特点。

**参考答案**：
| 特性 | DNS | HTTP | FTP |
|------|-----|------|-----|
| 端口 | 53 | 80 | 20/21 |
| 传输层 | UDP/TCP | TCP | TCP |
| 连接数 | - | 1个 | 2个 |
| 状态 | - | 无状态 | 有状态 |
| 用途 | 域名解析 | Web | 文件传输 |

### 综合题2：访问流程分析

**题目**：用户通过FTP下载文件，描述涉及的协议和端口。

**参考答案**：
1. DNS解析FTP服务器域名（UDP 53）
2. TCP三次握手建立控制连接（端口21）
3. 用户认证（USER/PASS命令）
4. TCP三次握手建立数据连接（端口20，主动模式）
5. 数据传输（RETR命令）
6. 关闭数据连接
7. 控制连接保持或关闭

### 综合题3：端口汇总

**题目**：列出DNS、HTTP、FTP使用的端口号和传输层协议。

**参考答案**：
| 协议 | 端口 | 传输层 |
|------|------|--------|
| DNS | 53 | UDP/TCP |
| HTTP | 80 | TCP |
| FTP控制 | 21 | TCP |
| FTP数据 | 20 | TCP |

### 综合题4：Web vs FTP

**题目**：从连接方式、状态维护、用途比较HTTP和FTP。

**参考答案**：
- HTTP：单连接、无状态、Web文档
- FTP：双连接（控制+数据）、有状态、文件传输

## 直观理解

**记忆技巧**：
- DNS 53查域名，HTTP 80看网页，FTP 20/21传文件
- HTTP单连接无状态，FTP双连接有状态
- DNS用UDP，HTTP/FTP用TCP

## 协议关联

- **DNS → HTTP/FTP**：DNS解析域名
- **HTTP vs FTP**：对比是常考内容
- **端口号**：408必考
- **408常见组合**：DNS + HTTP + FTP = 应用层综合对比
