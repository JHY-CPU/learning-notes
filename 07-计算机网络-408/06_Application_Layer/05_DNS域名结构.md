# 05_DNS域名结构

## 核心概念

- **域名（Domain Name）**：用点分隔的字符串，表示互联网上的一个名称
- **域名空间（Domain Name Space）**：DNS的层次化命名结构，呈树状
- **FQDN（Fully Qualified Domain Name）**：完全限定域名，从叶节点到根的完整路径
- **DNS层次结构**：
  - **根域（Root）**：用`.`表示，全球13组根服务器（a.root-servers.net到m.root-servers.net）
  - **顶级域（TLD）**：`.com`、`.org`、`.edu`、`.cn`、`.net`等
  - **二级域（SLD）**：`baidu.com`、`google.com`等
  - **子域（Subdomain）**：`www.baidu.com`、`mail.baidu.com`
- **408考试重点**：域名的层次结构、域名解析过程、各层服务器的功能

## 原理分析

### 域名树结构

```
                    . (根)
                   / | \
                com  org  cn  edu  net ...
               /    |    \
          google  baidu  tsinghua
           / \     |        \
        www  mail  www      www
```

- 从右到左读：`www.baidu.com.`（末尾的`.`代表根）
- 域名不区分大小写
- 每个节点的标签最长63个字符，完整域名最长255个字符

### 顶级域名分类

| 类型 | 示例 | 说明 |
|------|------|------|
| 通用顶级域 | `.com` `.org` `.net` `.edu` `.gov` | 通用类别 |
| 国家代码顶级域 | `.cn` `.us` `.uk` `.jp` | 国家/地区 |
| 新通用顶级域 | `.top` `.xyz` `.tech` | 新增类别 |

### 域名服务器类型

1. **根域名服务器**：知道所有顶级域名服务器的IP地址
   - 全球13组根服务器（实际有镜像，数量超过13台）
   - 使用任播（Anycast）技术

2. **顶级域名服务器**：管理该顶级域下的所有二级域名
   - 如`.com`服务器知道所有`.com`域名的权威服务器

3. **权威域名服务器**：管理特定组织的域名
   - 存放该域名的DNS记录
   - 可以由组织自己维护，也可以委托给服务商

4. **本地域名服务器（递归服务器/解析器）**：
   - 不属于DNS层次结构
   - 每个ISP或企业都有自己的本地DNS服务器
   - 代替主机进行递归查询

## 直观理解

**类比：邮政系统**
- 根域 = 国家邮政总局（知道各省的邮局）
- 顶级域 = 省级邮局（知道各市的邮局）
- 二级域 = 市级邮局（知道各小区的邮箱）
- 子域 = 小区邮箱（最终地址）
- 本地DNS = 你楼下的快递柜（缓存常用地址）

## 协议关联

- **根服务器与TLD服务器**：根服务器不直接解析域名，只告诉你去哪个TLD服务器
- **权威服务器与本地DNS**：本地DNS代替客户端向权威服务器查询
- **DNS与HTTP**：浏览器访问`www.example.com`时，需要先通过DNS获取IP地址
- **408常见问题**：
  - 根服务器只有13组，为什么不增加？（UDP报文512字节限制，每个根服务器地址需要16字节）
  - 域名服务器之间是什么关系？（层次关系，上级知道下级的地址）
