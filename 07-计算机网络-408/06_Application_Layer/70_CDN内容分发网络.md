# 70_CDN内容分发网络

## 核心概念

- **CDN（Content Delivery Network）**：内容分发网络
- CDN将内容缓存到**边缘服务器**，就近为用户提供服务
- CDN目的是**减少延迟、减轻源服务器负担**
- CDN是**分布式Web缓存**系统
- **408考试重点**：CDN的作用、工作原理、与DNS的关系

## 原理分析

### CDN工作原理

1. **内容分发**：
   - 源服务器将内容分发到CDN边缘服务器
   - 边缘服务器分布在全球各地

2. **用户请求**：
   - 用户请求内容
   - DNS解析将用户导向最近的CDN服务器
   - CDN服务器返回缓存的内容

3. **缓存管理**：
   - CDN服务器缓存热门内容
   - 缓存过期后从源服务器更新
   - 使用TTL控制缓存有效期

### CDN与DNS

1. 用户请求`www.example.com`
2. DNS解析`www.example.com` → CNAME → CDN DNS
3. CDN DNS根据用户位置返回最近的CDN服务器IP
4. 用户连接CDN服务器获取内容

### CDN的优势

1. **减少延迟**：用户从最近的服务器获取内容
2. **减轻源服务器负担**：大部分请求由CDN处理
3. **提高可用性**：CDN有冗余，单点故障不影响服务
4. **DDoS防护**：CDN可以吸收DDoS攻击

### CDN的实现方式

1. **DNS重定向**：通过CNAME将用户导向CDN
2. **Anycast**：多个服务器共享同一IP
3. **HTTP重定向**：服务器返回302重定向

### CDN与Web缓存对比

| 特性 | Web缓存 | CDN |
|------|---------|-----|
| 范围 | 单个代理 | 全球分布 |
| 管理 | 用户配置 | 专业运营 |
| 性能 | 局部优化 | 全局优化 |
| 成本 | 低 | 高 |

## 直观理解

**CDN就像连锁超市**：
- 源服务器 = 总仓库
- CDN边缘服务器 = 各地超市
- 用户 = 顾客
- 顾客去最近的超市买东西（就近访问CDN）
- 超库存不够了去总仓库补货（CDN回源）

**记忆技巧**：
- CDN = "内容分发网络"
- CDN = 分布式缓存 + DNS重定向
- CDN优势 = 减少延迟 + 减轻源服务器 + 提高可用性

## 协议关联

- **CDN与DNS**：CDN使用DNS将用户导向最近的服务器
- **CDN与HTTP**：CDN缓存HTTP内容
- **CDN与Web缓存**：CDN是分布式Web缓存
