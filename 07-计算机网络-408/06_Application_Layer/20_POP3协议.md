# 20_POP3协议

## 核心概念

- **POP3（Post Office Protocol version 3）**：邮局协议第3版
- POP3使用**TCP**协议，端口号**110**
- POP3是**C/S模式**的应用协议
- POP3是"**拉（Pull）**"协议：客户端从服务器拉取邮件
- POP3下载邮件后**默认删除**服务器上的邮件
- **408考试重点**：POP3端口号、下载并删除模式、与IMAP的区别

## 原理分析

### POP3工作过程

1. **认证阶段（Authorization）**：
   - 客户端连接服务器110端口
   - 服务器响应：`+OK POP3 server ready`
   - 客户端发送：`USER username`
   - 客户端发送：`PASS password`
   - 服务器验证通过：`+OK`

2. **事务阶段（Transaction）**：
   - `LIST`：列出邮件列表
   - `RETR n`：获取第n封邮件
   - `DELE n`：标记删除第n封邮件
   - `STAT`：返回邮件数量和总大小

3. **更新阶段（Update）**：
   - `QUIT`：退出
   - 服务器执行标记的删除操作
   - 关闭TCP连接

### POP3的工作模式

1. **下载并删除（Download and Delete）**：
   - 邮件下载到客户端后从服务器删除
   - 问题：换一台设备就看不到以前的邮件了
   - 这是POP3的默认模式

2. **下载并保留（Download and Keep）**：
   - 邮件下载后保留在服务器上
   - 问题：服务器上邮件越来越多，需要手动管理
   - 可通过配置选择此模式

### POP3的特点

- **简单**：协议简单，易于实现
- **离线访问**：邮件下载后可以离线阅读
- **单向同步**：只能从服务器下载，不能同步客户端的修改
- **有限功能**：只支持基本的下载和删除操作
- **不支持文件夹**：不能在服务器上管理邮件文件夹

## 直观理解

**POP3就像去邮局取信**：
- 你（客户端）去邮局（服务器）取信
- 告诉邮局你的身份（USER/PASS）
- 邮局把信给你（RETR）
- 你取走信，邮局把这封信扔掉（DELE + QUIT执行删除）
- 换一个邮局就取不到之前的信了

**记忆技巧**：
- POP3 = "拉"（Pull），端口110
- "POP3取110"（从110端口取邮件）
- POP3下载后删除 = "取信后邮局不留底"
- 三个阶段：认证 → 事务 → 更新

**POP3与IMAP对比**：
- POP3简单但功能有限，IMAP功能丰富但复杂
- POP3下载后删除，IMAP保留服务器副本
- POP3适合单设备，IMAP适合多设备同步

## 协议关联

- **POP3与TCP**：POP3使用TCP的可靠传输，端口110
- **POP3与SMTP**：
  - SMTP推邮件（25端口），POP3拉邮件（110端口）
  - 发送用SMTP，接收用POP3
- **POP3与IMAP**：
  - 都是邮件接收协议
  - POP3简单但不支持多设备同步
  - IMAP功能更丰富，支持服务器端管理
- **408考点**：
  - POP3端口110
  - POP3是拉协议
  - POP3下载后删除的特点
  - POP3与IMAP的区别
- **陷阱**：POP3是接收协议，不是发送协议；发送用SMTP
