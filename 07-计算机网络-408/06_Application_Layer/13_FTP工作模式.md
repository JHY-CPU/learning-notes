# 13_FTP工作模式

## 核心概念

- **FTP两种工作模式**：
  - **主动模式（Active Mode / PORT模式）**：服务器主动连接客户端
  - **被动模式（Passive Mode / PASV模式）**：客户端主动连接服务器
- **控制连接**：始终是客户端主动连接服务器的21端口
- **数据连接**：根据模式不同，连接方向不同
- **408考试重点**：区分主动模式和被动模式的连接过程，理解各自的端口使用

## 原理分析

### 主动模式（Active Mode）

**工作流程**：
1. 客户端随机端口N → 服务器21端口：建立控制连接（三次握手）
2. 客户端发送`PORT N+1`命令，告诉服务器"我的N+1端口等着收数据"
3. 服务器从20端口 → 客户端N+1端口：主动发起数据连接（三次握手）
4. 通过数据连接传输数据
5. 数据传输完毕，关闭数据连接

**端口分配**：
- 控制连接：客户端随机端口 → 服务器21
- 数据连接：服务器20 → 客户端指定端口

**问题**：服务器主动连接客户端，如果客户端有防火墙，可能被阻挡

### 被动模式（Passive Mode）

**工作流程**：
1. 客户端随机端口N → 服务器21端口：建立控制连接
2. 客户端发送`PASV`命令
3. 服务器响应：进入被动模式，开放端口P（随机高端口）
4. 客户端从N+1端口 → 服务器P端口：主动发起数据连接
5. 通过数据连接传输数据
6. 数据完毕，关闭数据连接

**端口分配**：
- 控制连接：客户端随机端口 → 服务器21
- 数据连接：客户端随机端口 → 服务器随机高端口

**优点**：解决客户端防火墙问题，因为连接都是由客户端发起

### 主动模式与被动模式对比

| 特性 | 主动模式 | 被动模式 |
|------|---------|---------|
| 数据连接发起方 | 服务器 | 客户端 |
| 服务器数据端口 | 固定20 | 随机高端口 |
| 客户端防火墙 | 可能被阻挡 | 不会被阻挡 |
| 服务器防火墙 | 不需要额外配置 | 需要开放高端口 |
| 命令 | PORT | PASV |
| 使用场景 | 早期默认 | 现代常用 |

## 直观理解

**主动模式 = 上门送货**：
- 你（客户端）打电话给商店（服务器控制端口21）说"我要买什么东西"
- 商店派人（数据端口20）送货上门到你家
- 问题：你家门卫（防火墙）不让陌生人进来

**被动模式 = 上门取货**：
- 你打电话给商店说"我要买东西"
- 商店说"好，我开了个门（随机端口），你来取"
- 你自己去商店取货
- 问题：商店需要多开很多门（高端口）

**记忆技巧**：
- 主动 = 服务器主动连客户端（"我来找你"）
- 被动 = 服务器被动等客户端（"你来找我"）
- 现代互联网多用被动模式（NAT/防火墙普遍）

## 协议关联

- **主动模式与NAT**：服务器主动连接客户端时，NAT可能无法正确转换地址
- **被动模式与防火墙**：服务器需要开放高端口范围
- **FTP与网络安全**：FTP明文传输密码，不安全；应使用SFTP或FTPS
- **408常见问题**：
  - 为什么现代多用被动模式？（防火墙和NAT的普及）
  - 控制连接和数据连接哪个先关闭？（数据连接先关闭）
- **陷阱**：主动模式下数据连接的源端口是20，目的端口是客户端指定的端口
