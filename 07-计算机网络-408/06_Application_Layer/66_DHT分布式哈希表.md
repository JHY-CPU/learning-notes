# 66_DHT分布式哈希表

## 核心概念

- **DHT（Distributed Hash Table）**：分布式哈希表
- DHT是P2P网络中**去中心化**的节点发现机制
- DHT将数据分布存储在多个节点上
- **Kademlia**：最常用的DHT实现协议
- **408考试重点**：DHT的基本概念、去中心化特性

## 原理分析

### DHT工作原理

1. **哈希空间**：
   - 将数据和节点映射到同一个哈希空间
   - 通常使用160位哈希（如SHA-1）

2. **节点负责范围**：
   - 每个节点负责一部分哈希空间
   - 数据存储在哈希值最接近的节点上

3. **路由查找**：
   - 使用迭代或递归查找
   - 每次查找将范围缩小一半
   - 查找复杂度为O(log N)

### Kademlia协议

- **节点ID**：160位随机数
- **距离**：XOR距离（异或）
- **K桶**：每个节点维护多个路由表
- **查找过程**：每次查询距离更近的节点

### DHT vs 集中式

| 特性 | 集中式 | DHT |
|------|--------|-----|
| 中心服务器 | 有 | 无 |
| 健壮性 | 低 | 高 |
| 查找效率 | O(1) | O(log N) |
| 维护成本 | 低 | 高 |

### DHT应用场景

1. **BitTorrent**：Tracker的替代方案
2. **IPFS**：分布式文件系统
3. **区块链**：节点发现
4. **CDN**：内容定位

## 直观理解

**DHT就像分布式图书馆**：
- 没有中心图书馆（无中心服务器）
- 每个分馆负责一部分书（节点负责哈希空间）
- 查找一本书时，先问最近的分馆，再问更近的分馆（迭代查找）
- 分馆越多，查找越快（对数复杂度）

**记忆技巧**：
- DHT = "去中心化的节点发现"
- Kademlia = "XOR距离 + K桶"
- 查找复杂度 = O(log N)
