# 77_应用层综合计算

## 核心概念

- **应用层综合计算题型**：涉及DNS、HTTP、TCP、FTP的综合计算
- **关键公式**：
  - DNS查询时间 = 各级RTT之和
  - HTTP RTT = TCP握手 + HTTP请求响应
  - 非持久连接：每个对象2 RTT
  - 持久连接：首个2 RTT，后续1 RTT
  - P2P文件分发时间公式
- **408考试重点**：综合计算是大题常考内容

## 原理分析

### 综合题1：Web访问完整时间

**题目**：用户访问`http://www.example.com/index.html`，DNS解析30ms，TCP握手20ms，HTTP请求响应15ms。页面含3个图片（HTTP/1.1持久非流水线）。求总时间。

**解答**：
- DNS解析：30ms（一次性）
- TCP握手：20ms（一次性）
- HTML请求响应：15ms
- 3个图片：3 × 15ms = 45ms
- 总时间：30 + 20 + 15 + 45 = 110ms

### 综合题2：DNS + HTTP + TCP

**题目**：DNS查询RTT=40ms，TCP握手RTT=30ms，HTTP请求RTT=20ms。HTTP/1.0非持久连接，访问含1个HTML和2个图片的页面。求总时间。

**解答**：
- DNS：40ms
- HTML：TCP握手(30ms) + HTTP请求(20ms) = 50ms
- 图片1：TCP握手(30ms) + HTTP请求(20ms) = 50ms
- 图片2：TCP握手(30ms) + HTTP请求(20ms) = 50ms
- 总时间：40 + 50 + 50 + 50 = 190ms

### 综合题3：P2P文件分发

**题目**：文件大小F=100MB，服务器上传速率us=10Mbps，5个对等方各上传ui=2Mbps，最小下载速率dmin=5Mbps。求P2P最小分发时间。

**解答**：
$$T_{P2P} = \max\left\{\frac{F}{u_s}, \frac{F}{d_{min}}, \frac{NF}{u_s + \sum u_i}\right\}$$
$$T_{P2P} = \max\left\{\frac{100}{10}, \frac{100}{5}, \frac{5 \times 100}{10 + 5 \times 2}\right\}$$
$$T_{P2P} = \max\{10, 20, 25\} = 25 \text{ 秒}$$

### 综合题4：DNS缓存影响

**题目**：DNS查询RTT=50ms，缓存命中率70%，命中时查询时间0ms。HTTP请求RTT=20ms，TCP握手RTT=30ms。求平均访问时间（HTTP/1.1持久）。

**解答**：
- 平均DNS时间：0.7 × 0 + 0.3 × 50 = 15ms
- TCP握手：30ms
- HTTP请求：20ms
- 总时间：15 + 30 + 20 = 65ms

## 直观理解

**做题技巧**：
- 先画流程图，标注每段时间
- 注意哪些是一次性的（DNS、TCP握手）
- 注意持久/非持久连接的区别
- 注意缓存的影响

## 协议关联

- **DNS + TCP + HTTP** = Web访问综合计算
- **P2P + 文件分发** = 分布式计算
- **缓存 + 查询时间** = 性能优化
- **408常见组合**：DNS + TCP + HTTP = 综合大题
