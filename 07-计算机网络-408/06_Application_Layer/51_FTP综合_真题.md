# 51_FTP综合_真题

## 核心概念

- **真题来源**：408统考及各校自命题中涉及FTP的综合题
- **高频考点**：FTP端口号、双连接、主动/被动模式
- **出题规律**：通常结合HTTP对比考查

## 原理分析

### 真题1（选择题）

**题目**：FTP控制连接的端口号是（  ），数据连接的端口号是（  ）（主动模式）。

**答案**：21；20

**解析**：
- 控制连接：客户端端口 → 服务器21
- 数据连接（主动模式）：服务器20 → 客户端指定端口

### 真题2（选择题）

**题目**：关于FTP，以下说法正确的是（  ）
A. FTP使用UDP协议
B. FTP是无状态协议
C. FTP使用两个TCP连接
D. FTP只有一个端口号

**答案**：C

**解析**：
- FTP使用TCP（A错误）
- FTP是有状态的（B错误）
- FTP使用两个TCP连接：控制+数据（C正确）
- FTP使用两个端口：20和21（D错误）

### 真题3（简答题）

**题目**：比较FTP与HTTP的主要区别。

**参考答案**：
1. 连接方式：FTP使用两个TCP连接（控制+数据），HTTP使用一个
2. 状态维护：FTP有状态，HTTP无状态
3. 端口号：FTP使用20和21，HTTP使用80
4. 用途：FTP专用于文件传输，HTTP用于Web文档传输

### 真题4（综合题）

**题目**：描述FTP主动模式下下载文件的完整过程。

**参考答案**：
1. 客户端 → 服务器21：TCP三次握手建立控制连接
2. 客户端发送USER/PASS认证
3. 客户端发送PORT命令指定数据端口
4. 服务器20 → 客户端指定端口：TCP三次握手建立数据连接
5. 客户端发送RETR命令
6. 服务器通过数据连接发送文件
7. 数据传输完毕，关闭数据连接
8. 控制连接保持，客户端可继续其他操作

## 直观理解

**真题做题技巧**：
- FTP端口"21控制，20数据"必记
- FTP双连接有状态是核心特点
- 主动模式：服务器20连客户端
- 对比HTTP：单连接、无状态

## 协议关联

- **FTP与TCP**：FTP完全依赖TCP
- **FTP与HTTP**：对比考查是常见题型
- **408常见组合**：FTP + TCP + HTTP = 应用层综合对比
