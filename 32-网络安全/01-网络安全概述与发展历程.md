# 01-网络安全概述与发展历程

## 一、网络安全的定义

**网络安全（Cybersecurity / Network Security）** 是指通过技术手段、管理措施和法律手段，保护计算机网络系统中的硬件、软件及数据不因偶然或恶意原因而遭受破坏、更改、泄露，确保系统连续可靠正常运行，网络服务不中断。

### 1.1 网络安全的核心目标（CIA三元组）

| 目标 | 英文 | 含义 |
|------|------|------|
| **机密性** | Confidentiality | 信息不被未授权的个人、实体或过程获取 |
| **完整性** | Integrity | 信息在存储或传输过程中保持不被修改、不被破坏和丢失 |
| **可用性** | Availability | 授权实体在需要时可以访问和使用信息及资源 |

### 1.2 扩展的安全属性

| 属性 | 说明 |
|------|------|
| **可认证性（Authenticity）** | 验证通信双方身份的真实性 |
| **不可否认性（Non-repudiation）** | 发送方不能事后否认其发送行为，接收方也不能否认其接收行为 |
| **可审计性（Accountability）** | 能够追溯和记录所有的安全相关操作 |
| **可控性（Controllability）** | 对信息传播范围和内容具有控制能力 |
| **可靠性（Reliability）** | 系统在规定条件下和规定时间内完成规定功能的概率 |

## 二、网络安全威胁分类

### 2.1 按威胁来源分类

#### 内部威胁
- **恶意内部人员**：有意泄露敏感数据、破坏系统
- **疏忽内部人员**：因安全意识薄弱导致误操作
- **权限滥用**：超越权限访问或操作

#### 外部威胁
- **黑客攻击**：利用技术手段入侵系统
- **APT攻击**：高级持续性威胁，长期潜伏窃取情报
- **供应链攻击**：通过第三方组件植入后门
- **国家行为体**：国家级网络攻击力量

### 2.2 按攻击手段分类

```
网络攻击
├── 被动攻击（Passive Attack）
│   ├── 流量嗅探（Sniffing）
│   ├── 流量分析（Traffic Analysis）
│   └── 窃听（Eavesdropping）
│
├── 主动攻击（Active Attack）
│   ├── 篡改（Modification）
│   ├── 伪造（Fabrication）
│   ├── 重放攻击（Replay）
│   ├── 拒绝服务（DoS/DDoS）
│   └── 中间人攻击（MITM）
│
└── 混合攻击
    ├── 社会工程学 + 技术攻击
    └── APT多阶段攻击
```

### 2.3 常见攻击类型详解

| 攻击类型 | 描述 | 危害等级 |
|----------|------|----------|
| DDoS攻击 | 利用大量僵尸网络耗尽目标资源 | 高 |
| SQL注入 | 通过输入恶意SQL语句操纵数据库 | 严重 |
| XSS攻击 | 在网页中注入恶意脚本 | 高 |
| CSRF攻击 | 伪造用户请求执行非预期操作 | 中高 |
| 钓鱼攻击 | 伪装成可信实体骗取敏感信息 | 高 |
| 零日攻击 | 利用尚未公开的漏洞 | 严重 |
| 勒索软件 | 加密用户数据并索要赎金 | 严重 |
| APT攻击 | 长期定向渗透窃取核心数据 | 严重 |
| 供应链攻击 | 通过软件供应链植入恶意代码 | 严重 |
| 暴力破解 | 穷举密码或密钥 | 中 |

## 三、网络安全发展历程

### 3.1 发展阶段

#### 第一阶段：通信安全阶段（1940s-1970s）
- 核心关注：通信保密
- 主要技术：对称加密（DES）、一次性密码本
- 背景：冷战时期军事通信需求
- 标志事件：1949年Shannon发表《保密系统的通信理论》

#### 第二阶段：计算机安全阶段（1970s-1990s）
- 核心关注：计算机系统安全
- 主要技术：访问控制、身份认证、安全操作系统
- 标志事件：
  - 1977年 RSA算法提出
  - 1985年 TCSEC（橘皮书）发布
  - 1988年 Morris蠕虫事件

#### 第三阶段：网络安全阶段（1990s-2010s）
- 核心关注：网络互连环境下的安全
- 主要技术：防火墙、IDS/IPS、VPN、PKI
- 标志事件：
  - 1995年 SSL协议发布
  - 1999年 IPSec标准制定
  - 2000年 大规模DDoS攻击

#### 第四阶段：信息安全阶段（2010s-至今）
- 核心关注：数据安全、隐私保护、云安全
- 主要技术：零信任架构、AI安全、隐私计算
- 标志事件：
  - 2013年 斯诺登事件
  - 2017年 WannaCry勒索病毒
  - 2020年 SolarWinds供应链攻击
  - 2021年 Log4j漏洞

## 四、网络安全体系架构

### 4.1 PDRR模型

```
Protection（防护）→ Detection（检测）→ Response（响应）→ Recovery（恢复）
```

- **防护**：访问控制、加密、防火墙等
- **检测**：入侵检测、漏洞扫描、日志审计
- **响应**：事件处置、取证分析、通报协调
- **恢复**：数据备份、系统还原、业务连续性

### 4.2 WPDRRC模型

```
预警(W) → 保护(P) → 检测(D) → 响应(R) → 恢复(R) → 反击(C)
```

中国提出的网络安全模型，在PDRR基础上增加了**预警**和**反击**两个环节。

### 4.3 纵深防御（Defense in Depth）

```
                    ┌─────────────────┐
                    │   物理安全层    │
                    │ ┌─────────────┐ │
                    │ │ 网络安全层  │ │
                    │ │ ┌─────────┐ │ │
                    │ │ │主机安全层│ │ │
                    │ │ │┌───────┐│ │ │
                    │ │ ││应用安全││ │ │
                    │ │ ││┌─────┐││ │ │
                    │ │ │││数据层│││ │ │
                    │ │ ││└─────┘││ │ │
                    │ │ │└───────┘│ │ │
                    │ │ └─────────┘ │ │
                    │ └─────────────┘ │
                    └─────────────────┘
```

每一层都有独立的安全措施，即使某一层被突破，其他层仍可提供保护。

### 4.4 零信任架构（Zero Trust）

**核心原则**："永不信任，始终验证"（Never Trust, Always Verify）

| 传统模型 | 零信任模型 |
|----------|------------|
| 基于边界防护 | 无边界，内网外网同等对待 |
| 信任内网 | 不默认信任任何网络 |
| 一次认证持续访问 | 持续验证，最小权限 |
| 基于IP/位置 | 基于身份和上下文 |

关键组件：
- **身份认证**：多因素认证（MFA）
- **设备验证**：终端安全状态检查
- **微隔离**：网络细粒度分段
- **持续监控**：行为分析和异常检测
- **最小权限**：按需授权，即时回收

## 五、网络安全法律法规

| 法律/法规 | 生效时间 | 核心内容 |

| 《网络安全法》 | 2017.06.01 | 网络运行安全、网络信息安全、监测预警与应急处置 |
| 《数据安全法》 | 2021.09.01 | 数据分类分级、数据安全保护义务、数据安全审查 |
| 《个人信息保护法》 | 2021.11.01 | 个人信息处理规则、跨境提供规则、个人权利 |
| 《关键信息基础设施安全保护条例》 | 2021.09.01 | 关基识别认定、安全保护义务、安全审查 |
| 《密码法》 | 2020.01.01 | 核心密码、普通密码、商用密码管理 |
| 《网络安全审查办法》 | 2022.02.15 | 网络安全审查范围、程序和要求 |
| 《数据出境安全评估办法》 | 2022.09.01 | 数据出境安全评估条件和程序 |
| 《网络数据安全管理条例》 | 2025.01.01 | 网络数据处理活动的规范 |

| 标准/法规 | 适用范围 | 核心内容 |

| GDPR | 欧盟 | 个人数据保护，数据处理者义务，被遗忘权 |

五个安全等级：

| 等级 | 名称 | 适用对象 |
|------|------|----------|
| 第一级 | 用户自主保护 | 一般信息系统 |
| 第二级 | 系统审计保护 | 重要信息系统 |
| 第三级 | 安全标记保护 | 重要网络和信息系统 |
| 第四级 | 结构化保护 | 重要领域核心系统 |
| 第五级 | 访问验证保护 | 国家重要领域核心系统 |
