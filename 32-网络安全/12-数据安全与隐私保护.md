# 12-数据安全与隐私保护

## 一、数据安全概述

### 1.1 数据生命周期安全

```
数据生命周期安全：
┌────────────────────────────────────────────┐
│  采集 → 传输 → 存储 → 使用 → 共享 → 销毁   │
│  ↓      ↓      ↓      ↓      ↓      ↓     │
│  合规   加密   加密   脱敏   审批   彻底   │
│  告知   完整性 备份   审计   授权   删除   │
│  最小化 认证   访控   防泄露 日志   证明   │
└────────────────────────────────────────────┘
```

### 1.2 数据分类分级

| 级别 | 名称 | 示例 | 保护要求 |
|------|------|------|----------|
| L1 | 公开数据 | 公开网站内容 | 基本保护 |
| L2 | 内部数据 | 内部通知、政策文件 | 访问控制 |
| L3 | 敏感数据 | 客户信息、财务数据 | 加密、审计 |
| L4 | 高度敏感 | 核心商业机密、密钥 | 最高保护、最小知悉 |

## 二、数据加密

### 2.1 数据加密层次

```
加密层次：
├── 应用层加密
│   ├── 字段级加密（敏感字段单独加密）
│   ├── 应用层端到端加密
│   └── 信封加密
├── 数据库层加密
│   ├── TDE（透明数据加密）
│   ├── 列级加密
│   └── 本地加密
├── 存储层加密
│   ├── 磁盘加密（BitLocker/LUKS）
│   ├── 存储阵列加密
│   └── 对象存储加密
└── 传输层加密
    ├── TLS
    └── IPSec
```

### 2.2 信封加密（Envelope Encryption）

```
┌───────────────┐
│  明文数据      │
├───────────────┤
│  DEK加密数据  │ ← 数据加密密钥（Data Encryption Key）
├───────────────┤
│  KEK加密DEK  │ ← 密钥加密密钥（Key Encryption Key）
├───────────────┤
│  主密钥加密KEK│ ← 主密钥（Master Key）
└───────────────┘

优势：
- 主密钥使用频率低，更安全
- 密钥轮换只需重新加密DEK
- 支持分布式密钥管理
```

### 2.3 密钥管理

```
密钥管理生命周期：
生成 → 存储 → 分发 → 使用 → 轮换 → 归档 → 销毁

最佳实践：
✅ 使用HSM（硬件安全模块）保护主密钥
✅ 定期轮换密钥
✅ 密钥分离（不同用途不同密钥）
✅ 审计密钥访问
✅ 安全销毁（彻底删除）
✅ 密钥备份（安全存储）
```

## 三、数据脱敏

### 3.1 脱敏方法

| 方法 | 说明 | 示例 |
|------|------|------|
| 姓名脱敏 | 部分隐藏 | 张*三 |
| 手机号脱敏 | 中间隐藏 | 138****5678 |
| 身份证脱敏 | 部分隐藏 | 110***********1234 |
| 银行卡脱敏 | 中间隐藏 | 6222 **** **** 1234 |
| 邮箱脱敏 | 用户名部分隐藏 | a***@example.com |
| 地址脱敏 | 详细地址隐藏 | 北京市朝阳区*** |
| 哈希脱敏 | 不可逆哈希 | SHA-256(原始值) |
| 泛化 | 降低精度 | 年龄25→20-30岁 |
| 替换 | 随机值替换 | 姓名→随机姓名 |
| 扰动 | 添加噪声 | 工资±5% |

### 3.2 脱敏场景

```
场景         方法        要求
─────────────────────────────────────
测试环境     全量脱敏    不可逆
数据分析     去标识化    可关联性控制
对外共享     匿名化     不可重标识
开发调试     部分脱敏    保留格式
日志打印     实时脱敏    自动化
报表展示     动态脱敏    按角色显示
```

### 3.3 动态数据脱敏

```sql
-- 数据库动态脱敏示例（SQL Server）
ALTER TABLE Customers
ALTER COLUMN Phone ADD MASKED WITH (FUNCTION = 'partial(3,"****",4)');

-- 不同角色看到不同数据
-- 管理员：13812345678
-- 普通用户：138****5678
```

```
│   ├── 文件操作监控

│   ├── 剪贴板管控

├── 网络DLP

│   ├── 邮件内容检测
│   ├── FTP文件检测
│   └── 即时通讯监控
└── 云DLP
    ├── SaaS应用监控
    ├── 云存储监控
    └── API调用监控
```

### 4.2 DLP检测技术

| 技术 | 说明 |
|------|------|
| 正则表达式 | 匹配身份证号、手机号等格式 |
| 关键词匹配 | 检测敏感关键词 |
| 文档指纹 | 提取文档特征指纹 |
| 机器学习 | 识别敏感内容类别 |
| 数据标识 | 检测数据分类标签 |
| 精确数据匹配 | 比对已知数据源 |

### 4.3 DLP策略示例

```
策略1：身份证号泄露防护
├── 检测规则：身份证号正则
├── 监控通道：邮件、USB、云盘
├── 响应动作：阻断+告警+记录

策略2：源代码泄露防护
├── 检测规则：文件类型(.java,.py,.go) + 关键词
├── 监控通道：USB、邮件、网盘
├── 响应动作：阻断+审批

策略3：客户数据泄露防护
├── 检测规则：数据标识 + 数量阈值
├── 监控通道：全通道
├── 响应动作：告警+记录
```

## 五、隐私保护

### 5.1 隐私保护技术

#### K-匿名

```
原理：每条记录至少与K-1条其他记录在准标识符上不可区分

原始数据：
| 年龄 | 性别 | 邮编 | 疾病 |
|------|------|------|------|
| 25   | 男   | 10001| 感冒 |
| 27   | 男   | 10002| 肺炎 |
| 28   | 女   | 10003| 感冒 |

2-匿名泛化后：
| 年龄  | 性别 | 邮编  | 疾病 |
|-------|------|-------|------|
| 20-30 | 男   | 1000* | 感冒 |
| 20-30 | 男   | 1000* | 肺炎 |
| 20-30 | 女   | 1000* | 感冒 |
```

#### 差分隐私（Differential Privacy）

```
原理：在查询结果中添加噪声，使得单条记录的增减不影响输出分布

应用场景：
├── 统计查询 - 添加拉普拉斯噪声
├── 机器学习 - DP-SGD（差分隐私随机梯度下降）
└── 数据发布 - 满足(ε,δ)-差分隐私

Apple、Google等公司广泛使用差分隐私收集用户统计数据。
```

#### 同态加密

```
支持在密文上直接计算：
- 加法同态：E(a) ⊕ E(b) = E(a+b)
- 乘法同态：E(a) ⊗ E(b) = E(a*b)
- 全同态：同时支持加法和乘法

应用场景：
├── 加密数据库查询
├── 安全多方计算
├── 隐私保护机器学习
└── 加密搜索
```

#### 安全多方计算（MPC）

```
多个参与方在不泄露各自输入的情况下，共同计算一个函数。

应用：
├── 联合数据分析
├── 隐私保护的机器学习
├── 安全投票
├── 联邦学习
└── 匿名拍卖
```

#### 联邦学习

```
数据不出域，模型到数据端训练：
┌────────┐  ┌────────┐  ┌────────┐
│ 参与方A │  │ 参与方B │  │ 参与方C │
│ 本地数据 │  │ 本地数据 │  │ 本地数据 │
│ 本地训练 │  │ 本地训练 │  │ 本地训练 │
└───┬────┘  └───┬────┘  └───┬────┘
    │            │            │
    ▼            ▼            ▼
┌──────────────────────────────────┐
│        聚合服务器                  │
│    模型聚合（FedAvg等）           │
│    更新全局模型分发               │
└──────────────────────────────────┘
```

### 5.2 隐私增强技术对比

| 技术 | 安全性 | 性能 | 实用性 | 主要应用 |
|------|--------|------|--------|----------|
| K-匿名 | 中 | 高 | 高 | 数据发布 |
| 差分隐私 | 高 | 高 | 高 | 统计查询 |

## 七、数据安全合规

### 7.1 个人信息保护

```
个人信息处理原则（《个人信息保护法》）：
├── 合法、正当、必要、诚信
├── 明确、合理的目的
├── 最小必要原则
├── 公开透明
├── 准确、完整
├── 安全保障
└── 保存期限限制

个人敏感信息：
├── 生物识别信息
├── 宗教信仰
├── 特定身份
├── 医疗健康
├── 金融账户
├── 行踪轨迹
├── 不满14周岁未成年人信息
└── 其他敏感信息
```

### 7.2 跨境数据传输

```
数据出境安全评估条件：
├── 关键信息基础设施运营者向境外提供数据
├── 向境外提供重要数据
├── 处理100万人以上个人信息的数据处理者
├── 累计向境外提供10万人个人信息
└── 累计向境外提供1万人敏感个人信息

评估要求：
├── 数据出境安全评估申报
├── 与境外接收方签订合同
├── 评估数据出境风险
└── 接受网信部门监督
```
