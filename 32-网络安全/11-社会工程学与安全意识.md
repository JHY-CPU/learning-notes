# 11-社会工程学与安全意识

## 一、社会工程学概述

### 1.1 定义

社会工程学（Social Engineering）是利用心理操纵手段，诱使人们泄露机密信息或执行不安全操作的攻击技术。它是网络攻击中"最薄弱环节"——人类——的主要攻击面。

> "永远不要低估一个带着善意微笑走进大楼的人所能造成的破坏。" — Kevin Mitnick

### 1.2 攻击原理

```
社会工程学利用的心理因素：
├── 权威 - 冒充上级或权威机构
├── 紧迫感 - 制造时间压力
├── 信任 - 建立虚假信任关系
├── 稀缺 - 制造机会稀缺感
├── 社会认同 - 利用从众心理
├── 互惠 - 先给予后索取
├── 好奇心 - 诱导点击/打开
└── 恐惧 - 制造恐慌和紧迫
```

## 二、常见攻击手法

### 2.1 钓鱼攻击（Phishing）

#### 类型细分

| 类型 | 描述 | 目标 |
|------|------|------|
| 批量钓鱼（Phishing） | 群发欺骗性邮件 | 广泛目标 |
| 鱼叉式钓鱼（Spear Phishing） | 针对特定个人/组织 | 特定目标 |
| 鲸钓（Whaling） | 针对高管 | CEO/CFO |
| 短信钓鱼（Smishing） | 通过短信 | 手机用户 |
| 语音钓鱼（Vishing） | 通过电话 | 电话用户 |
| 克隆钓鱼 | 复制合法邮件修改链接 | 已有通信关系 |

#### 钓鱼邮件特征

```
识别要点：
├── 发件人地址异常（域名拼写错误）
├── 紧急/威胁性语言
├── 要求提供敏感信息
├── 链接指向非官方域名
├── 附件类型异常（.exe, .scr, .js, .vbs）
├── 邮件中有拼写/语法错误
├── 好得不真实的优惠
└── 不寻常的请求（从未见过的流程）
```

#### 高级钓鱼技术

```
1. 鱼叉式钓鱼示例
   目标：公司CFO
   手段：
   - 研究目标的LinkedIn、社交媒体
   - 了解公司组织架构和内部术语
   - 伪造CEO邮件，要求紧急转账
   - 邮件风格模仿CEO习惯用语

2. 水坑攻击（Watering Hole）
   - 分析目标常用网站
   - 入侵该网站植入恶意代码
   - 等待目标访问

3. 域名仿冒
   - typosquatting：g00gle.com, rnicrosoft.com
   - 同形异义字攻击：аррӏе.com（使用Unicode字符）
   - 子域名：login.apple.com.evil.com
```

### 2.2 电话诈骗（Vishing）

```
常见场景：
├── 冒充IT支持 - "您的电脑感染了病毒"
├── 冒充银行 - "您的账户存在异常"
├── 冒充执法部门 - "您的身份被盗用"
├── 冒充供应商 - "我们需要更新付款信息"
├── 冒充CEO - "立即给这个账户转款"
└── 技术支持诈骗 - 假弹窗要求拨打技术支持电话
```

### 2.3 物理社会工程学

```
手法：
├── 尾随进入（Tailgating）- 跟随授权人员进入
├── 肩窥（Shoulder Surfing）- 偷看密码输入
├── USB投毒 - 遗留含有恶意代码的USB
├── 垃圾搜索（Dumpster Diving）- 翻找丢弃的文件
├── 假冒维修人员 - 冒充IT维修进入机房
└── 伪装成快递/外卖 - 混入办公场所
```

### 2.4 社交媒体攻击

```
信息收集：
├── LinkedIn - 组织架构、人员关系
├── 微博/朋友圈 - 个人习惯、行程
├── GitHub - 代码泄露、配置信息
├── 脉脉 - 公司内部信息
└── 技术论坛 - 技术栈、问题讨论

攻击利用：
├── 建立虚假社交关系
├── 利用公开信息构造精准钓鱼
├── 利用社交媒体进行CEO欺诈
└── 通过社交媒体传播恶意链接
```

## 三、高级持续性威胁（APT）

### 3.1 APT定义

APT（Advanced Persistent Threat）是指由具有丰富资源的威胁行为体发起的、针对特定目标的长期持续性网络攻击。

### 3.2 APT攻击阶段

```
APT攻击生命周期：
1. 侦察 - 收集目标信息（组织架构、技术栈、人员信息）
2. 武器化 - 准备攻击工具（恶意软件、漏洞利用）
3. 投递 - 将武器投送到目标（钓鱼邮件、水坑攻击）
4. 利用 - 触发漏洞获取初始访问
5. 安装 - 安装后门和持久化机制
6. 命令控制 - 建立C2通信
7. 目标行动 - 横向移动、数据收集、数据渗出
```

### 3.3 知名APT组织

| 组织 | 代号 | 主要目标 | 知名事件 |
|------|------|----------|----------|
| APT28 | Fancy Bear | 政府、军事 | 美国大选干预 |
| APT29 | Cozy Bear | 政府、外交 | SolarWinds |
| APT41 | Winnti | 科技、游戏 | 供应链攻击 |
| Lazarus | Hidden Cobra | 金融、加密货币 | WannaCry |
| APT34 | Oilrig | 中东 | 针对能源行业 |
| APT1 | Comment Crew | 商业机密 | 大规模知识产权窃取 |

### 3.4 APT防御

```
防御策略：
├── 威胁情报 - 跟踪APT组织活动和TTP
├── 高级检测 - 沙箱、行为分析、AI检测
├── 网络分段 - 限制横向移动
├── 终端防护 - EDR覆盖
├── 安全意识 - 员工防钓鱼培训
├── 零信任 - 持续验证
├── 威胁猎捕 - 主动搜索威胁
└── 红队演练 - 模拟APT攻击
```

```
    ├── 公司安全政策
    └── 违规后果
```

### 4.2 钓鱼模拟演练

```
钓鱼模拟流程：
1. 制定演练计划
2. 设计钓鱼模板（不同难度级别）
3. 发送模拟钓鱼邮件
4. 收集数据（打开率、点击率、报告率）
5. 对上当人员进行即时培训
6. 分析结果并改进
7. 定期重复演练

效果指标：
- 钓鱼邮件点击率应<5%
- 报告率应>60%
- 随着培训次数增加，指标应持续改善
```

```
安全文化建设要素：
├── 高层支持 - 领导层以身作则
├── 持续教育 - 定期培训和提醒
├── 正向激励 - 表扬安全行为
├── 容错机制 - 鼓励报告安全事件
├── 安全大使 - 各部门安全代表
├── 安全通讯 - 定期安全简报
├── 实战演练 - 定期钓鱼模拟
└── 反馈闭环 - 持续改进
```

## 五、内部威胁

### 5.1 内部威胁类型

| 类型 | 动机 | 特征 |
|------|------|------|
| 恶意内部人员 | 经济利益、报复 | 有意窃取/破坏 |
| 疏忽内部人员 | 无意识 | 误操作、安全意识差 |
| 被渗透内部人员 | 被外部胁迫/收买 | 协助外部攻击 |
| 离职人员 | 离职前报复 | 窃取数据、留后门 |

### 5.2 内部威胁指标

```
技术指标：
├── 异常的数据访问模式
├── 大量数据下载/拷贝
├── 非工作时间的系统访问
├── 访问非授权资源
├── 安装未授权软件
├── 使用未授权存储设备
└── 网络流量异常

行为指标：
├── 对公司不满的言论
├── 与同事关系紧张
├── 财务困难
├── 违反公司政策
├── 频繁加班/非工作时间在公司
└── 突然辞职
```

### 5.3 内部威胁防护

```
防护措施：
├── DLP（数据防泄露）
│   ├── 终端DLP
│   ├── 网络DLP
│   └── 云DLP
├── UEBA（用户行为分析）
│   ├── 建立行为基线
│   ├── 异常检测
│   └── 风险评分
├── 权限管理
│   ├── 最小权限
│   ├── 定期审查
│   └── 离职即回收
├── 监控审计
│   ├── 数据访问日志
│   ├── 终端监控
│   └── 网络流量监控
└── 管理措施
    ├── 入职背景调查
    ├── 保密协议
    ├── 离职流程
    └── 举报机制
```
