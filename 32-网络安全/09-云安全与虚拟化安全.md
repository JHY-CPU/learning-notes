## 二、AWS安全

### 2.1 IAM安全

```json
// 最小权限策略示例
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:GetObject",
        "s3:PutObject"
      ],
      "Resource": "arn:aws:s3:::my-bucket/*",
      "Condition": {
        "IpAddress": {
          "aws:SourceIp": "203.0.113.0/24"
        }
      }
    }
  ]
}
```

**IAM安全最佳实践：**
- 启用MFA（所有IAM用户和root账户）
- 使用IAM角色而非长期凭据
- 定期轮换访问密钥
- 使用IAM Access Analyzer
- 启用CloudTrail审计
- 删除未使用的用户和角色
- 使用服务控制策略（SCP）限制组织单位

### 2.2 S3安全

```
常见配置错误：
├── 存储桶公开访问
├── ACL配置不当
├── 服务器访问日志未启用
├── 未启用加密
└── 版本控制未开启

安全加固：
1. 阻止公共访问（Block Public Access）
2. 启用默认加密（SSE-S3或SSE-KMS）
3. 启用访问日志
4. 使用VPC端点访问
5. 配置存储桶策略限制访问来源
6. 启用对象锁定（防删除）
```

### 2.3 EC2安全

```
安全要点：
├── 安全组（Security Group）- 最小化开放端口
├── 网络ACL（NACL）- 子网级别过滤
├── 系统管理器（SSM）- 代替SSH管理
├── EBS加密 - 启用默认加密
├── AMI安全 - 使用加固的AMI
├── 实例元数据 - 使用IMDSv2防SSRF
└── 密钥对管理 - 定期轮换SSH密钥
```

## 四、容器安全

### 4.1 Docker安全

```
镜像安全：
├── 使用最小基础镜像（Alpine、Distroless）
├── 固定镜像版本标签（不用latest）
├── 扫描镜像漏洞（Trivy, Grype）
├── 多阶段构建减少攻击面
├── 不在镜像中存储密钥
└── 使用官方或可信的基础镜像

运行时安全：
├── 非root用户运行
├── 只读文件系统
├── 限制能力（--cap-drop=ALL）
├── 资源限制（CPU/内存）
├── 健康检查
├── 网络隔离
└── 秘密管理（Docker Secrets）
```

### 4.2 Kubernetes安全

```
K8s安全层次：

1. 集群安全
   ├── API Server
   │   ├── 匿名认证禁用
   │   ├── RBAC授权
   │   ├── 准入控制
   │   └── TLS加密
   ├── etcd
   │   ├── 加密存储
   │   ├── 访问控制
   │   └── 备份加密
   └── Kubelet
       ├── 匿名认证禁用
       ├── 授权限制
       └── 证书轮换

2. 命名空间安全
   ├── 资源配额（ResourceQuota）
   ├── 限制范围（LimitRange）
   └── 网络策略（NetworkPolicy）

3. Pod安全
   ├── Pod安全标准（PSS）
   │   ├── Privileged（受限）
   │   ├── Baseline（基线）
   │   └── Restricted（限制）
   ├── 安全上下文（SecurityContext）
   │   ├── runAsNonRoot
   │   ├── readOnlyRootFilesystem
   │   ├── allowPrivilegeEscalation: false
   │   └── capabilities.drop: ALL
   └── 服务账户（ServiceAccount）
       ├── 自动挂载禁用
       └── 最小权限绑定

4. 镜像安全
   ├── 镜像准入控制（OPA/Gatekeeper）
   ├── 镜像签名验证（Cosign/Notary）
   ├── 运行时扫描
   └── 私有镜像仓库

5. 运行时安全
   ├── Falco（异常检测）
   ├── Seccomp（系统调用过滤）
   ├── AppArmor/SELinux
   └── 运行时镜像扫描
```

### 4.3 Kubernetes安全策略示例

```yaml
# NetworkPolicy - 限制Pod网络访问
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: deny-all-ingress
spec:
  podSelector: {}
  policyTypes:
  - Ingress
  # 默认拒绝所有入站流量

---
# Pod安全上下文
apiVersion: v1
kind: Pod
metadata:
  name: secure-pod
spec:
  securityContext:
    runAsNonRoot: true
    runAsUser: 1000
    fsGroup: 2000
  containers:
  - name: app
    image: myapp:v1.2.3
    securityContext:
      allowPrivilegeEscalation: false
      readOnlyRootFilesystem: true
      capabilities:
        drop:
        - ALL
    resources:
      limits:
        memory: "256Mi"
        cpu: "500m"
```

## 五、云原生安全（CNAPP）

### 5.1 CNAPP架构

```
CNAPP = CSPM + CWPP + CIEM + CI/CD安全

├── CSPM（云安全态势管理）
│   ├── 配置审计
│   ├── 合规性检查
│   └── 漏洞管理
├── CWPP（云工作负载保护平台）
│   ├── 主机安全
│   ├── 容器安全
│   └── 无服务器安全
├── CIEM（云基础设施授权管理）
│   ├── 权限分析
│   ├── 过度授权检测
│   └── JIT授权
└── CI/CD安全
    ├── 代码扫描
    ├── 镜像扫描
    └── IaC扫描
```

### 5.2 IaC安全

```
基础设施即代码（IaC）安全扫描：
```

## 六、Serverless安全

### 6.1 安全挑战

| 挑战 | 说明 |
|------|------|
| 函数级权限 | 每个函数需要独立的最小权限 |
| 事件注入 | 不可信的事件数据 |
| 依赖漏洞 | 第三方库安全 |
| 冷启动延迟 | 安全检查延迟 |
| 日志分散 | 难以进行统一审计 |
| DDoS/经济DoS | 按调用计费的滥用风险 |

### 6.2 Serverless安全最佳实践

```
✅ 每个函数最小权限IAM角色
✅ 输入验证和参数化
✅ 依赖扫描（OWASP Dependency-Check）
✅ 环境变量加密
✅ VPC部署（访问内网资源）
✅ 并发限制（防止经济DoS）
✅ API网关认证和限流
✅ 函数代码签名
✅ 冷启动安全检查
```
