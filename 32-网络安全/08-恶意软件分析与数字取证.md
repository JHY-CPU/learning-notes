# 08-恶意软件分析与数字取证

## 一、恶意软件概述

### 1.1 恶意软件分类

| 类型 | 特点 | 典型代表 |
|------|------|----------|
| **病毒（Virus）** | 依附于宿主程序，需要用户执行 | CIH、熊猫烧香 |
| **蠕虫（Worm）** | 自我复制和传播，无需宿主 | WannaCry、Conficker |
| **木马（Trojan）** | 伪装为合法程序 | 远程控制木马（RAT） |
| **勒索软件（Ransomware）** | 加密文件索要赎金 | WannaCry、LockBit |
| **间谍软件（Spyware）** | 窃取用户信息 | 键盘记录器 |
| **Rootkit** | 隐藏自身和恶意行为 | 内核级Rootkit |
| **僵尸程序（Botnet）** | 远程控制大量主机 | Mirai |
| **广告软件（Adware）** | 强制推送广告 | 浏览器劫持 |
| **挖矿程序（Cryptominer）** | 利用主机资源挖矿 | CoinHive |
| **无文件恶意软件** | 仅在内存中运行 | PowerShell Empire |

### 1.2 传播途径

```
传播方式：
├── 钓鱼邮件附件/链接
├── 恶意网站/水坑攻击
├── USB/移动设备
├── 软件供应链（植入后门）
├── 漏洞利用（远程代码执行）
├── 社会工程学
├── P2P/种子下载
└── 破解软件/盗版软件
```

## 三、静态分析

### 3.1 基础信息收集

```bash
# 文件哈希
md5sum malware.exe
sha256sum malware.exe

# 文件类型
file malware.exe

# 字符串提取
strings malware.exe | grep -i "http\|ftp\|cmd\|powershell"
strings -e l malware.exe  # Unicode字符串

# 在线查杀
# 上传到VirusTotal查看多引擎检测结果
```

### 3.2 PE文件分析

```
PE结构：
┌──────────────────┐
│  DOS Header      │  MZ签名
├──────────────────┤
│  PE Header       │  PE签名、时间戳、机器类型
├──────────────────┤
│  Optional Header │  入口点、镜像基址、子系统
├──────────────────┤
│  Section Headers │  .text(代码)、.data(数据)
│                  │  .rsrc(资源)、.idata(导入)
├──────────────────┤
│  Sections        │  实际数据
└──────────────────┘

关键分析点：
- 导入表（Import Table）- 调用的API
- 导出表（Export Table）- 导出的函数
- 资源段 - 嵌入的文件/图标
- 数字签名 - 是否有合法签名
- 编译时间 - 推测攻击时间
- 加壳检测 - UPX、ASPack等
```

## 五、逆向工程

### 5.1 基本流程

```
1. 确定文件类型和架构
2. 识别编译器和库
3. 定位main/入口点
4. 识别关键函数
5. 分析控制流
6. 理解加密/编码算法
7. 提取配置信息（C2地址、加密密钥）
8. 编写检测规则
```

### 5.2 汇编基础

```asm
; x86-64关键寄存器
RAX, RBX, RCX, RDX  ; 通用寄存器
RSI, RDI            ; 源/目的索引
RBP                 ; 基址指针
RSP                 ; 栈指针
RIP                 ; 指令指针

; 常见指令
MOV  dst, src       ; 数据传输
PUSH src            ; 压栈
POP  dst            ; 弹栈
CALL addr           ; 调用函数
RET                 ; 返回
JMP  addr           ; 无条件跳转
JE/JNE/JG/JL        ; 条件跳转
CMP  a, b           ; 比较
TEST a, b           ; 位测试
XOR  a, b           ; 异或（常用于加密/清零）
LEA  dst, [src]     ; 加载有效地址
```

## 七、YARA规则

### 7.1 YARA基础语法

```yara
rule Malware_Example {
    meta:
        description = "示例恶意软件检测规则"
        author = "安全分析员"
        date = "2024-01-15"
        severity = "high"

    strings:
        $str1 = "malicious_function" ascii
        $str2 = { 48 8B 45 F0 48 89 45 E8 }  // 十六进制模式
        $str3 = /http:\/\/[a-z]+\.evil\.com/  // 正则表达式
        $hex1 = { 4D 5A 90 00 }  // MZ头

    condition:
        uint16(0) == 0x5A4D and  // MZ签名
        filesize < 1MB and
        2 of ($str*)
}
```

### 7.2 YARA条件语法

```yara
condition:
    all of them              // 所有字符串都匹配
    any of them              // 任意字符串匹配
    2 of ($str*)             // 至少2个$str变量匹配
    $str1 at 100             // $str1在偏移100处
    $str1 in (0..100)        // $str1在0-100偏移范围内
    filesize > 10KB          // 文件大小条件
    uint32(0) == 0x464C457F  // ELF签名
    pe.number_of_sections > 5 // PE文件节区数量
```

## 八、数字取证

### 8.1 取证流程

```
数字取证原则（ACPO原则）：
1. 不得采取任何可能改变数据的行动
2. 必须采取行动时，记录变更内容和影响
3. 审计日志记录所有取证过程
4. 案件负责人对取证过程负责

取证流程：
识别 → 收集 → 保全 → 分析 → 报告 → 出示
```

### 8.2 磁盘取证

```bash
# 证据保全 - 位对位复制
dd if=/dev/sda of=/evidence/disk.img bs=4M
dc3dd if=/dev/sda of=/evidence/disk.img hash=sha256

# 磁盘镜像分析
# Autopsy - 开源数字取证平台
# FTK Imager - 磁盘镜像工具
# Sleuth Kit - 命令行取证工具

# 文件恢复
foremost -i disk.img -o /output/
photorec disk.img

# 时间线分析
fls -r -m / disk.img > bodyfile
mactime -d -b bodyfile > timeline.csv
```

### 8.3 内存取证

```bash
# 内存转储
# Windows: DumpIt, WinPmem
# Linux: LiME (Linux Memory Extractor)
sudo insmod lime.ko "path=/evidence/mem.lime format=lime"

# Volatility内存分析框架
# 基本信息
volatility -f memory.raw imageinfo

# 进程列表
volatility -f memory.raw --profile=Win10x64 pslist
volatility -f memory.raw --profile=Win10x64 psscan  # 隐藏进程

# 网络连接
volatility -f memory.raw --profile=Win10x64 netscan

# 文件提取
volatility -f memory.raw --profile=Win10x64 filescan
volatility -f memory.raw --profile=Win10x64 dumpfiles -D /output/

# 注册表
volatility -f memory.raw --profile=Win10x64 hivelist

# 恶意进程检测
volatility -f memory.raw --profile=Win10x64 malfind
```

### 8.4 网络取证

```
分析内容：
├── 流量捕获（pcap分析）
├── 连接日志
├── DNS日志
├── 代理日志
├── 防火墙日志
└── IDS/IPS告警

Wireshark分析技巧：
- 统计 → 会话：查看流量分布
```

### 8.6 证据链（Chain of Custody）

```
证据链记录：
├── 证据标识（唯一编号）
├── 收集时间、地点
├── 收集人
├── 证据哈希值（MD5+SHA256）
├── 存储位置
├── 每次访问/转移记录
└── 签名确认

法律要求：
- 证据必须真实、完整
- 取证过程必须可重复
- 必须保持证据完整性
- 取证人员资质要求
```
