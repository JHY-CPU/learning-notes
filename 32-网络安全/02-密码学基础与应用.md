# 02-密码学基础与应用

## 一、密码学概述

### 1.1 密码学的定义与目标

密码学（Cryptography）是研究信息加密、解密以及安全通信的学科，是网络安全的核心基石。

**三大目标：**
- **机密性**：防止信息被未授权方获取
- **完整性**：确保信息未被篡改
- **认证性**：验证通信双方的身份

### 1.2 密码学发展简史

| 时期 | 代表技术 | 特点 |
|------|----------|------|
| 古典密码 | 凯撒密码、维吉尼亚密码 | 手工加密，简单替换 |
| 近代密码 | Enigma机 | 机械加密 |
| 现代密码 | DES、AES、RSA | 基于数学理论 |
| 当代密码 | 后量子密码、同态加密 | 应对量子计算威胁 |

### 1.3 密码系统基本模型

```
明文(M) → [加密算法E + 密钥K] → 密文(C) → [解密算法D + 密钥K'] → 明文(M)

          加密：C = E(K, M)
          解密：M = D(K', C)
```

**柯克霍夫原则（Kerckhoffs' Principle）：**
密码系统的安全性应完全依赖于密钥的保密性，即使加密算法被公开，系统仍应是安全的。

## 二、对称加密（Symmetric Encryption）

### 2.1 基本概念

加密和解密使用**同一个密钥**。加密速度快，适合大量数据加密。

### 2.2 分组密码（Block Cipher）

将明文分成固定大小的块，逐块加密。

#### DES（Data Encryption Standard）
- 密钥长度：56位（64位密钥中有8位校验位）
- 分组长度：64位
- 轮数：16轮Feistel结构
- 安全性：已不安全，1999年被暴力破解
- 过程：初始置换 → 16轮Feistel轮函数 → 逆初始置换

#### 3DES（Triple DES）
- 使用3次DES加密
- 密钥选项：3个独立密钥（168位）、2个密钥（112位）
- 速度较慢，逐步被AES替代

#### AES（Advanced Encryption Standard）
- 密钥长度：128/192/256位
- 分组长度：128位
- 结构：SPN（代换-置换网络）
- 轮数：10/12/14轮（取决于密钥长度）
- 当前主流对称加密标准，广泛使用

**AES加密过程：**
```
┌────────────────────────────┐
│        明文状态矩阵         │
├────────────────────────────┤
│ 第1轮：轮密钥加            │
│ 第2~(N-1)轮：              │
│   字节代换(SubBytes)       │
│   行移位(ShiftRows)       │
│   列混合(MixColumns)      │
│   轮密钥加(AddRoundKey)   │
│ 最后一轮：                  │
│   字节代换 → 行移位 → 轮密钥加 │
├────────────────────────────┤
│        密文状态矩阵         │
└────────────────────────────┘
```

#### SM4国密算法
- 中国国家密码管理局发布的分组密码标准
- 密钥长度：128位
- 分组长度：128位
- 轮数：32轮
- 结构：非平衡Feistel网络

### 2.3 流密码（Stream Cipher）

逐比特/字节加密，通过密钥流生成器产生伪随机密钥序列。

#### 常见流密码
| 算法 | 说明 | 状态 |
|------|------|------|
| RC4 | 曾广泛用于SSL/WEP | 已不安全 |
| ChaCha20 | Google推广的现代流密码 | 安全 |
| A5/1 | GSM语音加密 | 已被攻破 |
| SNOW 3G | 4G通信加密 | 在用 |

#### 流密码 vs 分组密码

| 特性 | 流密码 | 分组密码 |
|------|--------|----------|
| 加密单位 | 比特/字节 | 固定分组 |
| 速度 | 通常更快 | 相对较慢 |
| 错误传播 | 不传播 | CBC模式会传播 |
| 典型应用 | 实时通信 | 文件加密 |
| 代表算法 | ChaCha20 | AES |

### 2.4 分组密码工作模式

| 模式 | 全称 | 特点 |
|------|------|------|
| ECB | 电子密码本 | 并行，但相同明文块产生相同密文，不安全 |
| CBC | 密码分组链接 | 需要IV，串行，错误传播 |
| CTR | 计数器模式 | 并行，支持预计算，将分组密码变为流密码 |
| CFB | 密文反馈 | 将分组密码变为自同步流密码 |
| OFB | 输出反馈 | 将分组密码变为同步流密码 |
| GCM | Galois/计数器模式 | 提供认证加密（AEAD），效率高 |
| CCM | 计数器+CBC-MAC | 提供认证加密，用于WLAN |

**推荐使用：GCM模式（同时提供加密和认证）**

## 三、非对称加密（Asymmetric Encryption）

### 3.1 基本概念

使用**一对密钥**：公钥（Public Key）和私钥（Private Key）。公钥公开，私钥保密。
- 加密：用公钥加密，私钥解密
- 签名：用私钥签名，公钥验证

### 3.2 RSA算法

**数学基础：** 大整数分解问题

**密钥生成：**
1. 选择两个大素数 p, q
2. 计算 n = p × q
3. 计算 φ(n) = (p-1)(q-1)
4. 选择 e，使得 gcd(e, φ(n)) = 1
5. 计算 d，使得 e × d ≡ 1 (mod φ(n))
6. 公钥：(n, e)，私钥：(n, d)

**加密：** C = M^e mod n
**解密：** M = C^d mod n

**安全性：** 基于大整数分解的困难性
**推荐密钥长度：** 2048位以上

### 3.3 椭圆曲线密码（ECC）

**数学基础：** 椭圆曲线离散对数问题（ECDLP）

**优势：**
- 相同安全强度下，密钥更短
- 256位ECC ≈ 3072位RSA
- 计算效率更高，适合移动设备和IoT

**常用曲线：**
| 曲线 | 应用 |
|------|------|
| P-256 (secp256r1) | TLS、数字签名 |
| P-384 (secp384r1) | 高安全等级 |
| Curve25519 | Signal协议、WireGuard |
| secp256k1 | 比特币 |

### 3.4 SM2国密算法

- 中国国家密码管理局发布的椭圆曲线公钥密码算法
- 支持数字签名、密钥交换、公钥加密
- 推荐256位素数域上的椭圆曲线

### 3.5 Diffie-Hellman密钥交换

**目的：** 在不安全信道上协商共享密钥

```
公开参数：大素数p，原根g

Alice：                          Bob：
选择私钥a                       选择私钥b
计算 A = g^a mod p              计算 B = g^b mod p
发送A →                          ← 发送B
计算 K = B^a mod p              计算 K = A^b mod p

共享密钥 K = g^(ab) mod p
```

**安全性：** 基于离散对数问题的困难性

## 四、哈希函数（Hash Function）

### 4.1 基本概念

将任意长度的输入映射为固定长度的输出（哈希值/摘要/指纹）。

**安全哈希函数的性质：**
- **抗原像性（Pre-image Resistance）**：给定H(x)，难以找到x
- **抗第二原像性（Second Pre-image Resistance）**：给定x，难以找到x'≠x使H(x)=H(x')
- **抗碰撞性（Collision Resistance）**：难以找到任意两个不同的x≠x'使H(x)=H(x')

### 4.2 常见哈希算法

| 算法 | 输出长度 | 状态 |
|------|----------|------|
| MD5 | 128位 | 已不安全，2004年被攻破 |
| SHA-1 | 160位 | 已不安全，2017年SHAttered攻击 |
| SHA-256 | 256位 | 安全，广泛使用 |
| SHA-384 | 384位 | 安全 |
| SHA-512 | 512位 | 安全 |
| SHA-3 (Keccak) | 224/256/384/512位 | 安全，不同于SHA-2的结构 |
| SM3 | 256位 | 中国国密标准 |

### 4.3 哈希函数的应用

```
哈希函数应用
├── 数据完整性校验（文件校验和）
├── 数字签名（对消息摘要签名）
├── 密码存储（加盐哈希）
├── 区块链（工作量证明、区块链接）
├── 消息认证码（HMAC）
├── 伪随机数生成
├── 布隆过滤器
└── 数据结构（哈希表）
```

### 4.4 HMAC（基于哈希的消息认证码）

```
HMAC(K, M) = H((K' ⊕ opad) || H((K' ⊕ ipad) || M))
```

- 提供消息完整性和认证
- 广泛用于API认证、TLS、JWT
- 支持任意哈希函数（HMAC-SHA256最常见）

## 五、数字签名

### 5.1 原理

```
签名：Sig = Sign(PrivKey, H(M))
验证：Verify(PubKey, Sig, H(M)) → True/False
```

**提供：**
- **认证性**：确认签名者身份
- **完整性**：消息未被篡改
- **不可否认性**：签名者不能否认签名

### 5.2 常见签名算法

| 算法 | 基础 | 说明 |
|------|------|------|
| RSA签名 | RSA | 基于大整数分解 |
| DSA | 离散对数 | 美国数字签名标准 |
| ECDSA | 椭圆曲线 | 广泛用于TLS、区块链 |
| EdDSA | 椭圆曲线 | Ed25519，高性能 |
| SM2签名 | 椭圆曲线 | 中国国密标准 |

## 六、公钥基础设施（PKI）

### 6.1 PKI组成

```
┌─────────────────────────────────────────────┐
│                    PKI                        │
├──────────┬──────────┬──────────┬─────────────┤
│   CA     │   RA     │ 证书库   │   终端实体   │
│证书颁发   │注册审批   │   CRL   │  用户/服务器 │
│   机构   │   机构   │         │             │
└──────────┴──────────┴──────────┴─────────────┘
```

### 6.2 数字证书（X.509）

证书包含：
- 版本号
- 序列号
- 签名算法
- 颁发者（CA）
- 有效期
- 主体（证书持有者）
- 主体公钥
- 扩展信息
- CA数字签名

### 6.3 证书链验证

```
根CA证书（自签名）
  └── 中间CA证书（根CA签名）
        └── 终端实体证书（中间CA签名）
```

验证过程：从终端证书开始，逐级验证签名直到根CA，同时检查CRL/OCSP。

### 6.4 中国商用密码体系

| 类型 | 密码算法 | 用途 |
|------|----------|------|
| 对称加密 | SM4（分组）、ZUC（流） | 数据加密 |
| 非对称加密 | SM2（椭圆曲线）、SM9（标识密码） | 密钥交换、数字签名 |
| 哈希算法 | SM3 | 完整性校验 |

### 7.1 TLS/SSL协议

详见《06-网络安全协议》。

### 7.2 IPSec

详见《06-网络安全协议》。

### 7.3 Kerberos协议

**目标：** 在不安全网络上实现安全的身份认证

**基本流程：**
1. 客户端向KDC（密钥分发中心）请求TGT
2. 使用TGT向TGS请求服务票据
3. 使用服务票据访问目标服务

### 7.4 Signal协议

- 用于端到端加密通信
- 核心技术：双棘轮算法（Double Ratchet）
- 应用：WhatsApp、Signal、Facebook Messenger
- 特点：前向保密、后向保密

## 八、密码分析与攻击

### 8.1 密码分析方法

| 方法 | 描述 |
|------|------|
| 唯密文攻击 | 仅知道密文 |
| 已知明文攻击 | 知道部分明文-密文对 |
| 选择明文攻击 | 可以选择明文并获得对应密文 |
| 选择密文攻击 | 可以选择密文并获得对应明文 |
| 侧信道攻击 | 通过功耗、时间、电磁泄漏等信息分析 |
| 中间相遇攻击 | 攻击双重加密 |

### 8.2 侧信道攻击

| 类型 | 利用的信息 |
|------|------------|
| 时间攻击 | 加密操作的执行时间 |
| 功耗分析 | 设备的功耗曲线 |
| 电磁分析 | 设备的电磁辐射 |
| 缓存攻击 | CPU缓存的命中/缺失模式 |
| 声学攻击 | 设备发出的声音 |

### 8.3 常见密码学错误

- 自行设计加密算法（不要"发明自己的加密"）
- 使用ECB模式加密结构化数据
- 密钥重用（如流密码的nonce重用）
- 使用弱随机数生成器
- 密钥管理不当
- 过时算法（MD5、SHA-1、DES、RC4）

## 九、后量子密码学

### 9.1 量子计算威胁

量子计算机可高效解决：
- 大整数分解（Shor算法）→ 威胁RSA、ECC
- 离散对数问题 → 威胁DH、DSA

**不受威胁：** 对称加密（AES-256足够）、哈希函数（增加输出长度）

### 9.2 后量子密码候选算法

| 类型 | 代表算法 | 基础问题 |
|------|----------|----------|
| 格密码 | CRYSTALS-Kyber, Dilithium | 格上困难问题 |
| 基于编码 | Classic McEliece | 纠错码解码 |
| 多变量 | Rainbow | 多变量方程组 |
| 基于哈希 | SPHINCS+ | 哈希函数安全性 |

NIST后量子标准化（2024年）：
- **CRYSTALS-Kyber**（ML-KEM）：密钥封装
- **CRYSTALS-Dilithium**（ML-DSA）：数字签名
- **SPHINCS+**（SLH-DSA）：基于哈希的签名

## 十、国密算法体系详解

### 10.1 SM2（椭圆曲线公钥密码）

**用途：** 数字签名、密钥交换、公钥加密
**曲线参数：** 256位素数域上的椭圆曲线
**签名算法：** 基于SM3哈希的ECDSA类签名

### 10.2 SM3（密码杂凑算法）

- 输出长度：256位
- 结构：Merkle-Damgård结构
- 消息分组：512位
- 迭代压缩：64轮

### 10.3 SM4（分组密码算法）

- 密钥长度：128位
- 分组长度：128位
- 轮数：32轮
- 结构：32轮非平衡Feistel结构

### 10.4 SM9（基于标识的密码）

- 不需要数字证书
- 直接使用用户标识（如邮箱地址）作为公钥
- 简化了PKI的管理
