# 类型兼容性矩阵

## 一、概念说明

TypeScript 版本之间的类型兼容性涉及语言特性、`lib` 类型定义和编译输出。了解兼容性矩阵有助于规划版本升级、确保代码在不同版本下的可移植性。关键维度包括：TS 版本与语法特性、`target` 与运行时环境、`@types/*` 与库版本。

## 二、具体用法

### 2.1 版本与语法特性对照

| 特性 | 最低 TS 版本 | 说明 |
|------|------------|------|
| `const` 类型参数 | 5.0 | `function f<const T>()` |
| 装饰器（新标准） | 5.0 | TC39 Stage 3 |
| `satisfies` | 4.9 | 类型验证不改变类型 |
| `import type` 内联 | 4.5 | `import { type X }` |
| 模板字面量类型 | 4.1 | `` `${A}-${B}` `` |
| 映射类型 `as` | 4.1 | 键重映射 |
| `NoInfer` | 5.4 | 阻止推断 |
| `using` 声明 | 5.2 | 资源管理 |
| Import 属性 `with` | 5.3 | `with { type: "json" }` |

### 2.2 lib 兼容性

```json
{
  "compilerOptions": {
    // 包含 ES2022 API 类型：Array.at(), Object.hasOwn(), Error.cause
    "lib": ["ES2022", "DOM", "DOM.Iterable"],

    // ESNext — 始终包含最新 ECMAScript 特性
    "lib": ["ESNext", "DOM"],

    // 注意：lib 不影响编译输出，只影响类型检查
    "target": "ES2020",  // 输出 ES2020 语法
    "lib": ["ES2022"]    // 但可以使用 ES2022 API 类型
  }
}
```

### 2.3 @types 版本匹配

```bash
# @types/node 版本应与 Node.js 版本对应
npm install -D @types/node@18  # Node.js 18
npm install -D @types/node@20  # Node.js 20
npm install -D @types/node@22  # Node.js 22

# @types/react 版本应与 React 版本对应
npm install -D @types/react@18  # React 18
npm install -D @types/react@19  # React 19
```

### 2.4 target 与运行时

```json
{
  "compilerOptions": {
    "target": "ES2020",   // Node.js 14+, 现代浏览器
    "target": "ES2022",   // Node.js 18+, 最新浏览器
    "target": "ESNext",   // 输出最新 JS 语法
    "target": "ES5"       // 兼容 IE11（不推荐）
  }
}
```

### 2.6 升级策略

```typescript
// 1. 检查 breaking changes
//    - TS 5.0：装饰器语法变更
//    - TS 5.0：moduleResolution: bundler
//    - TS 4.9：satisfies 关键字

// 2. 使用 --strict 逐步启用
// tsconfig.json
{
  "compilerOptions": {
    "strict": true,             // 全部严格
    // 或逐个启用
    "strictNullChecks": true,
    "noImplicitAny": true,
    "strictFunctionTypes": true
  }
}

// 3. 使用 @ts-expect-error 处理已知问题
// @ts-expect-error — TS 5.x 升级后需要修复

```

```javascript
// 1. 类型系统：新语法特性（satisfies, NoInfer）
// 2. 编译输出：target 决定生成的 JS 版本
```

## 三、注意事项与常见陷阱

1. **`target` 和 `lib` 要匹配运行时**：Node.js 18 支持 ES2022，不要用 ES5 target
2. **`@types/*` 版本锁定**：避免类型不匹配导致编译错误
3. **新特性需要对应的 TS 版本**：检查项目中是否使用了过新语法
4. **编译输出的 JS 版本要与目标环境匹配**：浏览器兼容性、Node.js 版本
5. **`skipLibCheck` 加快编译**：但可能隐藏第三方库的类型问题
6. **装饰器是重大变更**：TS 5.0 的装饰器与旧版不兼容，升级需修改代码
