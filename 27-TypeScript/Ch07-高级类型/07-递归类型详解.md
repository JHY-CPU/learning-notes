# 递归类型详解

## 一、概念说明

递归类型引用自身，用于描述无限深度的数据结构。结合条件类型和 `infer`，可以实现深度类型操作。

## 二、具体用法

### 2.1 JSON 类型

```typescript
type Json =
  | string | number | boolean | null
  | Json[]
  | { [key: string]: Json };
```

### 2.2 深度 Partial

```typescript
type DeepPartial<T> = T extends object
  ? { [K in keyof T]?: DeepPartial<T[K]> }
  : T;

interface Config {
  server: { host: string; port: number; };
  db: { url: string; pool: number; };
}

type PartialConfig = DeepPartial<Config>;
// 嵌套属性也变为可选
```

### 2.3 路径类型

```typescript
type Paths<T> = T extends object
  ? { [K in keyof T]: K extends string
      ? T[K] extends object
        ? K | `${K}.${Paths<T[K]>}`
        : K
      : never
    }[keyof T]
  : never;

type UserPaths = Paths<{ name: string; address: { city: string } }>;
// "name" | "address" | "address.city"
```

## 三、注意事项与常见陷阱

1. **递归深度限制**：TypeScript 约 1000 层
2. **性能影响**：深度递归显著影响编译速度
3. **必须有终止条件**：基本情况防止无限递归
4. **测试边界情况**：确保空对象、数组等正确处理
