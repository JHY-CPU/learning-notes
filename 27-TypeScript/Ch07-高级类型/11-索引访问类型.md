# 索引访问类型

## 一、概念说明

索引访问类型 `T[K]` 获取对象类型 T 中键 K 对应的**值类型**。它类似于 JavaScript 对象的属性访问操作 `obj[key]`，但在类型层面工作。K 可以是单个键、键的联合，甚至用 `keyof T` 获取所有值类型的联合。索引访问类型是类型编程中"读取"类型信息的基础操作。

## 二、具体用法

### 2.1 基本索引访问

```typescript
interface User {
  id: number;
  name: string;
  metadata: { age: number; city: string };
}

type UserId = User["id"];          // number
type UserName = User["name"];      // string
type Meta = User["metadata"];      // { age: number; city: string }

// 嵌套访问
type Age = User["metadata"]["age"]; // number
type City = User["metadata"]["city"]; // string
```

### 2.2 联合键访问

```typescript
// 一次获取多个属性类型的联合
type IdOrName = User["id" | "name"];  // number | string

// 获取所有值类型
type AllValues = User[keyof User];
// number | string | { age: number; city: string }

// 实际应用：通用的值类型提取
type ValueOf<T> = T[keyof T];
type UserValues = ValueOf<User>; // number | string | { age: number; city: string }
```

### 2.3 数组和元组的索引访问

```typescript
// 数组元素类型
type StrArr = string[];
type Elem = StrArr[number]; // string

// 元组：用数字字面量索引
type Pair = [string, number];
type First = Pair[0];  // string
type Second = Pair[1]; // number
type Either = Pair[number]; // string | number

// 提取长度类型
type TupleLen = Pair["length"]; // 2（字面量类型）
type ArrLen = StrArr["length"]; // number（数组长度不确定）
```

### 2.4 配合映射类型过滤

```typescript
// 提取特定类型的所有属性
type PickByType<T, U> = {
  [K in keyof T as T[K] extends U ? K : never]: T[K];
};

type StringProps = PickByType<User, string>;
// { name: string }

// 提取值类型为特定类型的键名
type KeysByType<T, U> = keyof PickByType<T, U>;
type StringKeys = KeysByType<User, string>; // "name"
```

### 2.5 实际应用：类型安全的 ORM 查询

```typescript
interface DB {
  users: { id: number; name: string; email: string };
  posts: { id: number; title: string; authorId: number };
  comments: { id: number; text: string; postId: number };
}

// 根据表名和字段名获取字段类型
type FieldType<T extends keyof DB, K extends keyof DB[T]> = DB[T][K];

type UserName = FieldType<"users", "name">;   // string
type PostTitle = FieldType<"posts", "title">; // string

// 获取表的所有字段名
type UserFields = keyof DB["users"]; // "id" | "name" | "email"
```

### 2.6 与 JavaScript 的对比

```javascript
// JavaScript：运行时访问属性值
const user = { id: 1, name: "Alice", metadata: { age: 25, city: "Beijing" } };
const age = user.metadata.age; // 25（运行时值）

// TypeScript：编译时获取属性类型
// type Age = User["metadata"]["age"]; // number（编译时类型）
// 两者语法相似，但操作层面完全不同
```

## 三、注意事项与常见陷阱

1. **必须用方括号**：`T[K]` 而非 `T.K`，后者是非法语法
2. **K 必须是 T 的键**：除非 T 有索引签名，否则 `T["nonExistent"]` 报错
3. **支持嵌套访问**：`T["a"]["b"]["c"]` 可以访问多层属性
4. **配合 `keyof`**：`T[keyof T]` 获取所有值类型的联合，是最常见的用法之一
5. **联合键返回联合类型**：`T["a" | "b"]` 返回 `T["a"] | T["b"]`
6. **数字索引**：数组用 `T[number]` 获取元素类型，元组可以用字面量数字索引
