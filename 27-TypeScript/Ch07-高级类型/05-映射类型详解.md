# 映射类型详解

## 一、概念说明

映射类型 `[K in keyof T]` 遍历类型 T 的所有键，对每个键的值类型进行变换。它是 TypeScript 类型编程的核心工具，结合属性修饰符（`readonly`、`?`）、键重映射（`as`）和条件类型，可以实现灵活的批量类型变换。所有内置工具类型（`Partial`、`Required`、`Readonly` 等）都基于映射类型实现。

## 二、具体用法

### 2.1 属性修饰符操作

```typescript
interface User {
  name: string;
  age: number;
  email: string;
}

// 添加可选属性
type MyPartial<T> = { [K in keyof T]?: T[K] };
type PartialUser = MyPartial<User>;
// { name?: string; age?: number; email?: string; }

// 移除可选属性
type MyRequired<T> = { [K in keyof T]-?: T[K] };
type RequiredUser = MyRequired<PartialUser>;
// { name: string; age: number; email: string; }

// 添加 readonly
type MyReadonly<T> = { readonly [K in keyof T]: T[K] };

// 移除 readonly
type MyMutable<T> = { -readonly [K in keyof T]: T[K] };

interface Frozen { readonly id: number; readonly name: string; }
type Thawed = MyMutable<Frozen>;
// { id: number; name: string; }
```

### 2.2 键重映射（as，TS 4.1+）

```typescript
// 生成 Getter 方法
type Getters<T> = {
  [K in keyof T as `get${Capitalize<string & K>}`]: () => T[K];
};
type UserGetters = Getters<User>;
// { getName: () => string; getAge: () => number; getEmail: () => string; }

// 生成 Setter 方法
type Setters<T> = {
  [K in keyof T as `set${Capitalize<string & K>}`]: (value: T[K]) => void;
};
type UserSetters = Setters<User>;
// { setName: (value: string) => void; ... }
```

### 2.3 条件过滤键

```typescript
// 按值类型过滤属性
type PickByType<T, U> = {
  [K in keyof T as T[K] extends U ? K : never]: T[K];
};

interface Mixed {
  id: number;
  name: string;
  age: number;
  active: boolean;
  email: string;
}

type StringProps = PickByType<Mixed, string>;
// { name: string; email: string; }

type NumberProps = PickByType<Mixed, number>;
// { id: number; age: number; }

// 过滤出可选属性
type OptionalKeys<T> = {
  [K in keyof T as {} extends Pick<T, K> ? K : never]: T[K];
};
```

### 2.4 值类型变换

```typescript
// 所有值转为 Promise
type Promisify<T> = {
  [K in keyof T]: Promise<T[K]>;
};
type AsyncUser = Promisify<User>;
// { name: Promise<string>; age: Promise<number>; email: Promise<string>; }

// 包装为响应式
type Reactive<T> = {
  [K in keyof T]: { value: T[K]; subscribe: (fn: (v: T[K]) => void) => void };
};
```

### 2.5 实际应用：API 响应包装

```typescript
// 每个字段独立的加载状态
type ApiState<T> = {
  [K in keyof T]: {
    data: T[K];
    loading: boolean;
    error: string | null;
  };
};

type UserApiState = ApiState<User>;
// {
//   name: { data: string; loading: boolean; error: string | null };
//   age: { data: number; loading: boolean; error: string | null };
//   email: { data: string; loading: boolean; error: string | null };
// }
```

### 2.6 与 JavaScript 的对比

```javascript
// JavaScript：运行时遍历对象属性转换
const user = { name: "Alice", age: 25, email: "a@b.com" };
const partial = {};
for (const key in user) {
  partial[key] = user[key]; // 手动处理
}

// TypeScript 映射类型：编译时批量变换类型
// type PartialUser = { [K in keyof User]?: User[K] };
// 不影响运行时，纯粹的类型操作
```

## 三、注意事项与常见陷阱

1. **`+` 和 `-` 修饰符**：`+` 添加（默认可省略），`-` 移除。如 `-?` 移除可选，`-readonly` 移除只读
2. **`as never` 过滤键**：条件返回 `never` 的键会被排除
3. **映射类型默认保留可选和 readonly**：除非显式用 `-?` 或 `-readonly` 修改
4. **键重映射需要 TS 4.1+**：`as` 语法不支持更早版本
5. **`string & K` 技巧**：确保 K 可以用于 `Capitalize`（需要 `string` 类型）
6. **性能注意**：大量属性的复杂映射（尤其嵌套递归）可能显著影响编译速度
