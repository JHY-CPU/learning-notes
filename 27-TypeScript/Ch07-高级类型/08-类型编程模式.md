# 类型编程模式

## 一、概念说明

类型编程（Type-Level Programming）是在编译时使用类型系统进行计算的技巧。核心模式包括：模式匹配、递归、组合。

## 二、具体用法

### 2.1 模式匹配

```typescript
// 匹配并提取
type Match<T, Pattern> = T extends Pattern ? T : never;
type R = Match<string | number, string>; // string
```

### 2.2 递归累加

```typescript
type Length<T extends readonly any[]> = T["length"];
type L = Length<[1, 2, 3]>; // 3

// 构造元组
type BuildTuple<N extends number, T extends any[] = []> =
  T["length"] extends N ? T : BuildTuple<N, [...T, any]>;

type T3 = BuildTuple<3>; // [any, any, any]
```

### 2.3 组合模式

```typescript
// Pipe 类型操作
type Pipe<Fns extends readonly ((arg: any) => any)[]> =
  Fns extends [(arg: infer A) => any, ...infer Rest]
    ? Rest extends ((arg: any) => any)[]
      ? (arg: A) => ReturnType<Fns[0]>
      : never
    : (arg: any) => any;
```

## 三、注意事项与常见陷阱

1. **类型编程是编译时的**：不影响运行时性能
2. **可读性优先**：复杂类型应有注释说明
3. **限制递归深度**：TypeScript 有递归深度限制
4. **测试类型**：确保类型在各种输入下正确
