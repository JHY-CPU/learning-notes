# 类型编程模式

## 一、概念说明

类型编程（Type-Level Programming）是在编译时使用 TypeScript 类型系统进行计算的技术。核心模式包括：**模式匹配**（条件类型 + infer）、**递归遍历**（递归条件类型）、**组合变换**（映射类型 + 工具类型）。掌握这些模式可以编写出强大的泛型工具类型。

## 二、具体用法

### 2.1 模式匹配（Pattern Matching）

```typescript
// 匹配并提取：从类型中"解构"出需要的部分
type UnwrapPromise<T> = T extends Promise<infer U> ? U : T;
type R1 = UnwrapPromise<Promise<string>>; // string

// 多分支匹配
type TypeName<T> =
  T extends string ? "string" :
  T extends number ? "number" :
  T extends boolean ? "boolean" :
  T extends undefined ? "undefined" :
  T extends Function ? "function" :
  "object";

type R2 = TypeName<() => void>; // "function"
type R3 = TypeName<[1, 2]>;     // "object"
```

### 2.2 递归遍历（Recursive Traversal）

```typescript
// 递归构造元组（类型级计数）
type BuildTuple<N extends number, T extends any[] = []> =
  T["length"] extends N ? T : BuildTuple<N, [...T, any]>;

type T3 = BuildTuple<3>; // [any, any, any]

// 递归累加
type Add<A extends number, B extends number> =
  [...BuildTuple<A>, ...BuildTuple<B>]["length"];

type Sum = Add<3, 4>; // 7（类型层面的计算）
```

### 2.3 组合变换（Composition）

```typescript
// 管道式类型变换
type Pipe<T> = T;

// 实际组合：先 Partial 再 Pick
type PartialPick<T, K extends keyof T> = Partial<Pick<T, K>> & Omit<T, K>;

interface User { id: number; name: string; email: string; age: number; }
type EditableUser = PartialPick<User, "name" | "email">;
// { id: number; age: number; name?: string; email?: string; }
```

### 2.4 验证模式（Validation）

```typescript
// 编译时类型断言
type Assert<T, U> = T extends U ? T : never;

// 类型级测试
type Test1 = Assert<{ a: string; b: number }, { a: string }>; // OK
// type Test2 = Assert<{ a: string }, { a: string; b: number }>; // never（失败）

// 确保两个类型完全匹配
type Equals<X, Y> =
  (<T>() => T extends X ? 1 : 2) extends
  (<T>() => T extends Y ? 1 : 2) ? true : false;

type E1 = Equals<string, string>; // true
type E2 = Equals<string, number>; // false
```

### 2.5 类型级状态机

```typescript
type State = "idle" | "loading" | "success" | "error";
type Event = "FETCH" | "RESOLVE" | "REJECT" | "RESET";

type Transition<S extends State, E extends Event> =
  S extends "idle"
    ? E extends "FETCH" ? "loading" : never
    : S extends "loading"
      ? E extends "RESOLVE" ? "success" : E extends "REJECT" ? "error" : never
      : S extends "success" | "error"
        ? E extends "RESET" ? "idle" : never
        : never;

type T1 = Transition<"idle", "FETCH">;     // "loading"
type T2 = Transition<"loading", "RESOLVE">; // "success"
// type T3 = Transition<"idle", "RESOLVE">;  // never（非法转移）
```

### 2.6 与 JavaScript 的对比

```javascript
// JavaScript：运行时组合函数
const pipe = (...fns) => (x) => fns.reduce((v, f) => f(v), x);
const process = pipe(trim, toLowerCase, escapeHtml);

// TypeScript：编译时组合类型
// type Process<T> = EscapeHtml<ToLowercase<Trim<T>>>;
// 编译时就确定最终类型，零运行时开销
```

## 三、注意事项与常见陷阱

1. **类型编程只在编译时运行**：不影响运行时性能，但复杂类型可能拖慢编译
2. **可读性优先**：复杂类型操作应添加注释，避免"类型体操"过度
3. **限制递归深度**：TypeScript 有递归限制，过度递归会导致编译超时
4. **组合优于单体**：将复杂类型操作拆分为简单可复用的工具类型
5. **测试类型正确性**：使用 `@ts-expect-error` 和类型断言验证边界情况
6. **渐进采用**：从简单模式（条件类型、映射类型）开始，逐步学习高级技巧
