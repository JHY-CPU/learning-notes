# 条件类型详解

## 一、概念说明

条件类型 `T extends U ? X : Y` 根据类型关系选择不同类型。它是类型编程的核心，结合 `infer` 可以提取、变换类型。条件类型在编译时计算，类似于 JavaScript 的三元运算符。

## 二、具体用法

### 2.1 基本条件类型

```typescript
type IsString<T> = T extends string ? true : false;

type A = IsString<"hello">; // true
type B = IsString<42>;      // false
```

### 2.2 嵌套条件类型

```typescript
type TypeName<T> =
  T extends string ? "string" :
  T extends number ? "number" :
  T extends boolean ? "boolean" :
  "object";

type T1 = TypeName<string>;  // "string"
type T2 = TypeName<true>;    // "boolean"
type T3 = TypeName<{}>;      // "object"
```

### 2.3 配合 infer

```typescript
type ReturnType<T> = T extends (...args: any[]) => infer R ? R : never;
type Params<T> = T extends (...args: infer P) => any ? P : never;

type R = ReturnType<() => string>; // string
type P = Params<(x: number, y: string) => void>; // [number, string]
```

### 2.4 实际应用

```typescript
// 提取函数返回类型
function getUser() { return { id: 1, name: "Alice" }; }
type User = ReturnType<typeof getUser>; // { id: number; name: string; }

// API 响应类型分支
type ApiResponse<T> = T extends void
  ? { success: boolean }
  : { success: boolean; data: T };
```

## 三、注意事项与常见陷阱

1. **分布式条件类型**：联合类型被逐个分发，用 `[T]` 包裹可阻止
2. **`never` 特殊情况**：`never extends T` 结果为 `never`
3. **`infer` 只能在条件类型中使用**：不能单独使用
4. **避免过度嵌套**：影响可读性和编译性能
5. **条件类型是类型级 if-else**：编译时计算，不影响运行时
6. **联合类型的分发**：`A | B extends T ? X : Y` 等于 `(A extends T ? X : Y) | (B extends T ? X : Y)`

## 四、高级 infer 技巧

```typescript
// 提取 Promise 的值类型
type UnwrapPromise<T> = T extends Promise<infer U> ? UnwrapPromise<U> : T;
type A = UnwrapPromise<Promise<Promise<string>>>; // string

// 提取数组元素类型
type ElementOf<T> = T extends (infer E)[] ? E : never;
type B = ElementOf<string[]>; // string

// 提取函数 this 类型
type ThisType<T> = T extends (this: infer U, ...args: any[]) => any ? U : never;

// 提取对象 value 类型
type ValueOf<T> = T[keyof T];

// 条件类型递归：深度 Readonly
type DeepReadonly<T> = T extends object
  ? { readonly [K in keyof T]: DeepReadonly<T[K]> }
  : T;

type C = DeepReadonly<{ a: { b: { c: string } } }>;
// { readonly a: { readonly b: { readonly c: string } } }
```

## 五、阻止联合类型分发

```typescript
// 分发行为：每个联合成员独立判断
type Dist<T> = T extends string ? 'str' : 'other';
type D1 = Dist<string | number>; // 'str' | 'other'

// 用元组包裹阻止分发
type NoDist<T> = [T] extends [string] ? 'str' : 'other';
type D2 = NoDist<string | number>; // 'other'

// 实际应用：判断联合类型是否整体满足条件
type IsUnion<T> = [T] extends [infer U]
  ? U extends T ? ([T] extends [U] ? false : true) : false
  : false;
```

## 六、实用类型工具

```typescript
// 深度合并两个对象类型
type DeepMerge<T, U> = {
  [K in keyof T | keyof U]:
    K extends keyof U
      ? K extends keyof T
        ? T[K] extends object ? U[K] extends object ? DeepMerge<T[K], U[K]> : U[K] : U[K]
        : U[K]
      : K extends keyof T ? T[K] : never;
};

// 提取函数参数类型（元组）
type Params<T> = T extends (...args: infer P) => any ? P : never;
type R = Params<(a: string, b: number) => void>; // [string, number]

// 提取构造函数参数
type ConstructorParams<T> = T extends new (...args: infer P) => any ? P : never;

// 字符串字面量操作
type StartsWith<T extends string, U extends string> = T extends `${U}${string}` ? true : false;
type S = StartsWith<'hello', 'hel'>; // true

// 递归 Required
type DeepRequired<T> = T extends object
  ? { [K in keyof T]-?: DeepRequired<T[K]> }
  : T;
```

## 七、条件类型在框架中的应用

```typescript
// Vue-like 响应式类型
type Reactive<T> = {
  [K in keyof T]: { value: T[K] };
};

// API 响应类型推断
type ApiResult<T> = T extends { success: true; data: infer D } ? D : never;

// 事件类型推断
type EventHandler<T> = T extends `on${infer Event}`
  ? Event extends Capitalize<infer E> ? E : never
  : never;
type E = EventHandler<'onClick'>; // 'click'
```
