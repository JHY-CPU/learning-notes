# 高级类型最佳实践

## 一、概念说明

高级类型（条件类型、映射类型、递归类型等）非常强大，但过度使用会导致代码难以理解和维护。遵循最佳实践，在类型安全和可读性之间找到平衡。

## 二、具体用法

### 2.1 何时使用高级类型

```typescript
// ✅ 适合：通用工具类型
type DeepReadonly<T> = T extends object
  ? { readonly [K in keyof T]: DeepReadonly<T[K]> }
  : T;

// ❌ 不适合：一次性使用的简单类型
// 简单场景用 interface/type 直接定义即可
```

### 2.2 命名和注释

```typescript
/**
 * 递归地将 T 的所有属性变为只读
 * @example DeepReadonly<{ a: { b: string } }>
 * // { readonly a: { readonly b: string } }
 */
type DeepReadonly<T> = T extends object
  ? { readonly [K in keyof T]: DeepReadonly<T[K]> }
  : T;
```

### 2.3 类型测试

```typescript
// 使用类型断言测试类型正确性
type Test1 = DeepReadonly<{ a: string }>;
type Assert1 = Test1 extends { readonly a: string } ? true : false;
// const _check: Assert1 = true; // 编译通过表示类型正确
```

### 2.4 性能优化

```typescript
// ❌ 避免深度递归
// type TooDeep<T> = T extends object ? { [K in keyof T]: TooDeep<T[K]> } : T;
// 嵌套 100+ 层时编译极慢

// ✅ 限制递归深度
type DeepPartial<T, D extends number = 5> = D extends 0
  ? T
  : T extends object
    ? { [K in keyof T]?: DeepPartial<T[K], Prev<D>> }
    : T;

type Prev<D extends number> = D extends 5 ? 4 : D extends 4 ? 3 : D extends 3 ? 2 : D extends 2 ? 1 : 0;
```

## 三、注意事项与常见陷阱

1. **可读性优先**：代码是给人读的，类型也是
2. **添加注释**：复杂类型必须有文档说明
3. **类型测试**：确保类型在各种输入下正确
4. **性能监控**：编译变慢时检查类型复杂度
5. **渐进采用**：从简单类型开始，逐步使用高级特性
