# 模板字面量类型详解

## 一、概念说明

模板字面量类型使用模板字符串语法在**类型层面**进行字符串操作，支持字符串拼接、联合类型笛卡尔展开和内置大小写变换工具类型。它是 TypeScript 4.1 引入的重要特性，使得类型系统能够精确描述字符串模式，广泛用于 CSS 类名生成、路由路径定义、API 端点类型等场景。

## 二、具体用法

### 2.1 类型级字符串拼接

```typescript
// 精确的 CSS 尺寸类型
type CSSSize = `${number}px` | `${number}%` | `${number}rem`;

const size1: CSSSize = "16px";   // OK
const size2: CSSSize = "100%";   // OK
const size3: CSSSize = "2rem";   // OK
// const bad: CSSSize = "16em";  // 错误：不匹配模式

// 拼接类型参数
type EventName<T extends string> = `on${Capitalize<T>}`;
type ClickHandler = EventName<"click">; // "onClick"
type FocusHandler = EventName<"focus">; // "onFocus"
```

### 2.2 联合类型笛卡尔展开

```typescript
type Color = "red" | "green" | "blue";
type Size = "sm" | "md" | "lg";

// 产生 3×3 = 9 种组合
type ButtonVariant = `${Color}-${Size}`;
// "red-sm" | "red-md" | "red-lg" | "green-sm" | ... | "blue-lg"

// 注意：组合数爆炸
type A = "a" | "b" | "c" | "d" | "e";
type B = "1" | "2" | "3" | "4" | "5";
type Combo = `${A}-${B}`; // 25 种组合
```

### 2.3 内置工具类型

```typescript
type Upper = Uppercase<"hello">;     // "HELLO"
type Lower = Lowercase<"HELLO">;     // "hello"
type Cap = Capitalize<"hello">;      // "Hello"
type Uncap = Uncapitalize<"Hello">;  // "hello"

// 实际应用：生成 CSS BEM 类名
type Theme = "light" | "dark";
type Component = "btn" | "card" | "nav";
type State = "active" | "disabled" | "hidden";

type CSSClass = `${Theme}__${Component}--${State}`;
const cls: CSSClass = "light__btn--active";
```

### 2.4 配合 infer 提取子串

```typescript
// 提取协议
type ExtractProtocol<T extends string> =
  T extends `${infer Protocol}://${string}` ? Protocol : never;
type Proto = ExtractProtocol<"https://example.com">; // "https"

// 提取域名
type ExtractDomain<T extends string> =
  T extends `${string}://${infer Domain}/${string}` ? Domain :
  T extends `${string}://${infer D}` ? D : never;
type Domain = ExtractProtocol<"https://api.example.com/v1">; // 不适用，用下面的
type D2 = ExtractDomain<"https://api.example.com/v1">; // "api.example.com"

// 字符串替换
type Replace<S extends string, From extends string, To extends string> =
  S extends `${infer L}${From}${infer R}` ? `${L}${To}${Replace<R, From, To>}` : S;
type Replaced = Replace<"hello-world", "-", "_">; // "hello_world"
```

### 2.5 实际应用：类型安全的 API 路由

```typescript
type HTTPMethod = "GET" | "POST" | "PUT" | "DELETE";
type ApiPath = "/users" | "/users/:id" | "/posts" | "/posts/:id/comments";

type ApiEndpoint = `${HTTPMethod} ${ApiPath}`;
// "GET /users" | "GET /users/:id" | ... | "DELETE /posts/:id/comments"

// 构建类型安全的 API 客户端
function createEndpoint<M extends HTTPMethod, P extends ApiPath>(
  method: M,
  path: P
): `${M} ${P}` {
  return `${method} ${path}`;
}

const ep = createEndpoint("GET", "/users");
// 类型为 "GET /users"
```

### 2.6 与 JavaScript 的对比

```javascript
// JavaScript：运行时模板字符串拼接
const color = "red";
const size = "lg";
const variant = `${color}-${size}`; // "red-lg"（运行时值）

// TypeScript：编译时类型拼接
// type Variant = `${Color}-${Size}`;
// 产生联合类型 "red-sm" | "red-md" | ... | "blue-lg"
// 编译时就确定所有可能的字符串组合
```

## 三、注意事项与常见陷阱

1. **联合类型笛卡尔展开**：`A(5种) | B(5种)` 的模板会产生 25 种组合，过多时影响编译性能
2. **类型参数必须 `extends string`**：非 string 类型不能用于模板
3. **与 `infer` 配合提取子串**：可以从模板字面量中反向提取部分
4. **`${number}` 匹配数字**：不限制范围，`"999999px"` 也合法
5. **递归替换可能超限**：复杂字符串操作可能触发递归深度限制
6. **TS 4.1+ 特性**：旧版本不支持
