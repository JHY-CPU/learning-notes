# 分布式条件类型

## 一、概念说明

当条件类型作用于联合类型时，会**逐个分发**（Distribute）联合成员分别计算，再将结果联合。例如 `T extends U ? X : Y` 中 T 为 `A | B` 时，等价于 `(A extends U ? X : Y) | (B extends U ? X : Y)`。

## 二、具体用法

### 2.1 基本分发

```typescript
type ToArray<T> = T extends unknown ? T[] : never;

type Result = ToArray<string | number>;
// 等价于 string[] | number[]（不是 (string | number)[]）

// 阻止分发：用方括号包裹
type ToArrayNonDist<T> = [T] extends [unknown] ? T[] : never;
type Result2 = ToArrayNonDist<string | number>;
// (string | number)[]
```

### 2.2 Exclude 的分发原理

```typescript
type MyExclude<T, U> = T extends U ? never : T;

type Result = MyExclude<"a" | "b" | "c", "a">;
// "a" extends "a" ? never : "a" = never
// "b" extends "a" ? never : "b" = "b"
// "c" extends "a" ? never : "c" = "c"
// 结果: "b" | "c"
```

### 2.3 实际应用

```typescript
type Event = "click" | "focus" | "blur" | "scroll";
type MouseEvents = Extract<Event, "click">; // "click"
type NonMouseEvents = Exclude<Event, "click">; // "focus" | "blur" | "scroll"
```

## 三、注意事项与常见陷阱

1. **只对联合类型分发**：非联合类型不触发分发
2. **阻止分发**：用 `[T]` 包裹即可
3. **`never` 的自动消除**：联合类型中的 `never` 被自动移除
4. **分发是 TypeScript 独有特性**：其他语言的条件类型可能不支持
