# 分布式条件类型

## 一、概念说明

当条件类型作用于**裸类型参数**（未被包装的联合类型）时，TypeScript 会将联合类型的每个成员**逐个分发**（Distribute）到条件类型中分别计算，再将结果合并为联合类型。这一行为是 TypeScript 类型系统的核心特性之一，使得 `Exclude`、`Extract` 等内置工具类型得以实现。

## 二、具体用法

### 2.1 基本分发行为

```typescript
// T 是裸类型参数，联合类型会分发
type ToArray<T> = T extends unknown ? T[] : never;

type Result = ToArray<string | number>;
// 等价于: (string extends unknown ? string[] : never)
//       | (number extends unknown ? number[] : never)
// 结果: string[] | number[]
// 注意: 不是 (string | number)[]
```

### 2.2 阻止分发（用元组包裹）

```typescript
// 用 [T] 包裹裸类型参数，阻止分发
type ToArrayNonDist<T> = [T] extends [unknown] ? T[] : never;

type Result1 = ToArray<string | number>;       // string[] | number[]
type Result2 = ToArrayNonDist<string | number>; // (string | number)[]

// 实际应用：判断联合类型整体是否满足条件
type IsUnion<T> = [T] extends [infer U]
  ? U extends T
    ? [T] extends [U] ? false : true
    : false
  : false;

type Check1 = IsUnion<string | number>; // true
type Check2 = IsUnion<string>;          // false
```

### 2.3 Exclude 与 Extract 的分发原理

```typescript
// Exclude: 分发时将匹配的成员替换为 never（自动从联合中移除）
type MyExclude<T, U> = T extends U ? never : T;

type R1 = MyExclude<"a" | "b" | "c", "a">;
// "a" extends "a" ? never : "a" = never
// "b" extends "a" ? never : "b" = "b"
// "c" extends "a" ? never : "c" = "c"
// never 自动从联合中移除 => "b" | "c"

// Extract: 分发时保留匹配的成员
type MyExtract<T, U> = T extends U ? T : never;

type R2 = MyExtract<"a" | "b" | "c", "a" | "b">;
// => "a" | "b"
```

### 2.4 实际应用：事件过滤

```typescript
type AppEvent =
  | { type: "click"; x: number; y: number }
  | { type: "keydown"; key: string }
  | { type: "scroll"; delta: number }
  | { type: "resize"; width: number; height: number };

// 提取鼠标相关事件
type MouseEvent = Extract<AppEvent, { type: "click" }>;
// { type: "click"; x: number; y: number }

// 排除键盘事件
type NonKeyboardEvent = Exclude<AppEvent, { type: "keydown" }>;
// { type: "click"; ... } | { type: "scroll"; ... } | { type: "resize"; ... }
```

### 2.5 与 JavaScript 的对比

```javascript
// JavaScript：运行时过滤数组
const events = ["click", "keydown", "scroll", "resize"];
const mouseRelated = events.filter(e => e === "click");
// ["click"]

// TypeScript 分布式条件类型：编译时过滤类型
// type MouseEvents = Extract<Event, "click">; // "click"
// 这是纯粹的类型操作，不影响运行时代码
```

## 三、注意事项与常见陷阱

1. **只对裸类型参数分发**：`T extends U ? X : Y` 中 T 必须是未包装的类型参数，直接写联合类型 `("a" | "b") extends U ? X : Y` 不会分发
2. **阻止分发用 `[T]`**：元组包裹是最常用的阻止方式，也可用 `T & {}` 等
3. **`never` 的自动消除**：分发结果中 `never` 会被自动从联合类型中移除
4. **分发是编译时行为**：不产生任何运行时代码
5. **数组类型不会分发**：`string[] | number[]` 不会分发到每个元素
6. **嵌套分发**：多层条件类型嵌套时，每层都会独立分发，可能导致类型爆炸
