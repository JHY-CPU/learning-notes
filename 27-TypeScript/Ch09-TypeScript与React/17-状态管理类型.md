# 状态管理类型

## 一、概念说明

TypeScript 与现代状态管理库（Redux Toolkit、Zustand、Jotai 等）的集成非常成熟。类型化状态管理的核心是定义好 State 类型、Action 类型和 Selector 类型，确保整个数据流的类型安全。

## 二、具体用法

### 2.1 Redux Toolkit 类型

```tsx
import { configureStore, createSlice, PayloadAction } from '@reduxjs/toolkit';
import { useSelector, useDispatch, TypedUseSelectorHook } from 'react-redux';

// 定义 State 类型
interface CounterState {
  value: number;
  status: 'idle' | 'loading';
}

const initialState: CounterState = { value: 0, status: 'idle' };

// createSlice 自动推断 action 类型
const counterSlice = createSlice({
  name: 'counter',
  initialState,
  reducers: {
    // PayloadAction<T> 定义 payload 类型
    increment(state) {
      state.value += 1;
    },
    incrementByAmount(state, action: PayloadAction<number>) {
      state.value += action.payload;
    },
  },
});

// Store 配置
const store = configureStore({
  reducer: {
    counter: counterSlice.reducer,
  },
});

// 类型导出
type RootState = ReturnType<typeof store.getState>;
type AppDispatch = typeof store.dispatch;

// 类型化的 Hooks — 替代 useSelector/useDispatch
const useAppSelector: TypedUseSelectorHook<RootState> = useSelector;
const useAppDispatch = () => useDispatch<AppDispatch>();

// 使用
function Counter() {
  const count = useAppSelector(state => state.counter.value); // 类型安全
  const dispatch = useAppDispatch();

  return (
    <button onClick={() => dispatch(counterSlice.actions.incrementByAmount(5))}>
      {count}
    </button>
  );
}
```

### 2.2 Zustand 类型

```tsx
import { create } from 'zustand';

// 定义 Store 接口
interface BearState {
  bears: number;
  fish: number;
  addBear: () => void;
  addFish: () => void;
  reset: () => void;
}

// create 自动推断类型
const useBearStore = create<BearState>((set) => ({
  bears: 0,
  fish: 0,
  addBear: () => set(state => ({ bears: state.bears + 1 })),
  addFish: () => set(state => ({ fish: state.fish + 1 })),
  reset: () => set({ bears: 0, fish: 0 }),
}));

// 使用 — 完整类型推断
function BearCounter() {
  const bears = useBearStore(state => state.bears); // number
  const addBear = useBearStore(state => state.addBear); // () => void
  return <button onClick={addBear}>{bears} 熊</button>;
}
```

### 2.3 Jotai 原子类型

```tsx
import { atom, useAtom } from 'jotai';

// 原子自动推断类型
const countAtom = atom(0);                  // WritableAtom<number>
const doubleCountAtom = atom(get => get(countAtom) * 2); // Atom<number> 只读

// 派生原子
const countHistoryAtom = atom<number[]>([]);

function Counter() {
  const [count, setCount] = useAtom(countAtom); // [number, (update: number) => void]
  return <button onClick={() => setCount(c => c + 1)}>{count}</button>;
}
```

### 2.4 Selector 类型安全

```tsx
// 复杂 Selector 的类型
const selectUserFullName = (state: RootState) => {
  const user = state.users.currentUser;
  return user ? `${user.firstName} ${user.lastName}` : '';
};

// 参数化 Selector（工厂模式）
const selectUserById = (userId: number) => (state: RootState) =>
  state.users.list.find(u => u.id === userId);

// 使用
function UserCard({ userId }: { userId: number }) {
  const user = useAppSelector(selectUserById(userId));
  return <div>{user?.name}</div>;
}
```

## 三、注意事项与常见陷阱

1. **始终创建类型化的 `useSelector` 和 `useDispatch`**：避免每次手动标注类型
2. **不要用 `any` 作为 State 类型**：失去了状态管理的全部类型价值
3. **Zustand 的 `create` 能自动推断类型**：不需要显式标注
4. **Redux Toolkit 的 `PayloadAction<T>`**：确保 action payload 有正确类型
5. **Selector 应导出为独立函数**：便于测试和复用
