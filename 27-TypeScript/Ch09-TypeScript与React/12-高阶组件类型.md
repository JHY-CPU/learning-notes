# 高阶组件类型

## 一、概念说明

高阶组件（HOC）是接收组件并返回新组件的函数。TypeScript 中 HOC 的类型设计需要同时处理输入组件的 Props 和输出组件的 Props，使用泛型可以精确表达这种类型变换关系。

## 二、具体用法

### 2.1 基本 HOC 类型

```tsx
import { ComponentType } from 'react';

// HOC：为组件注入 loading 状态
function withLoading<P extends object>(
  WrappedComponent: ComponentType<P>
): ComponentType<P & { isLoading: boolean }> {
  return function WithLoading(props: P & { isLoading: boolean }) {
    const { isLoading, ...rest } = props;

    if (isLoading) return <div>加载中...</div>;

    // 类型断言：rest 的类型是 P
    return <WrappedComponent {...(rest as P)} />;
  };
}

// 使用
interface UserCardProps {
  name: string;
  email: string;
}

function UserCard({ name, email }: UserCardProps) {
  return <div>{name} - {email}</div>;
}

const UserCardWithLoading = withLoading(UserCard);

// 使用时 Props 是 UserCardProps & { isLoading: boolean }
<UserCardWithLoading name="张三" email="test@example.com" isLoading={false} />;
```

### 2.2 消费 Props 的 HOC

```tsx
// HOC：注入用户信息
function withUser<P extends { userId?: number }>(
  Component: ComponentType<P>
): ComponentType<Omit<P, 'userId'>> {
  return function WithUser(props: Omit<P, 'userId'>) {
    const user = useUser(); // 假设从 Context 获取用户
    // userId 被 HOC 自动提供，不需要外部传入
    return <Component {...(props as P)} userId={user.id} />;
  };
}

interface ProfileProps {
  userId: number; // 由 HOC 注入
  name: string;
}

function Profile({ userId, name }: ProfileProps) {
  return <div>{name} (ID: {userId})</div>;
}

const EnhancedProfile = withUser(Profile);
// EnhancedProfile 的 Props 只需要 { name: string }
<EnhancedProfile name="张三" />;
```

### 2.3 泛型 HOC

```tsx
// 泛型 HOC：添加 ref 转发
function withForwardRef<P extends object, R>(
  Component: React.ForwardRefRenderFunction<R, P>
): React.ForwardRefExoticComponent<React.PropsWithoutRef<P> & React.RefAttributes<R>> {
  return React.forwardRef(Component);
}
```

### 2.4 组合多个 HOC

```tsx
// 类型安全的 compose 函数
function compose<P>(
  ...fns: Array<(arg: ComponentType<any>) => ComponentType<any>>
): (component: ComponentType<P>) => ComponentType<P> {
  return (component) => fns.reduce((acc, fn) => fn(acc), component);
}

// 使用
const Enhanced = compose(
  withLoading,
  withUser
)(UserCard);
```

## 三、注意事项与常见陷阱

1. **泛型约束用 `P extends object`**：确保 Props 是对象类型
2. **使用 `Omit<P, K>` 移除被 HOC 注入的 props**：避免外部传入已注入的属性
3. **展开 props 时需要类型断言**：解构后 `rest` 的类型可能不完整
4. **HOC 的 displayName**：方便 React DevTools 调试
5. **React 推荐用 Hooks 替代 HOC**：大多数 HOC 场景可以用自定义 Hook 更简洁地实现
