# 测试中的类型

## 一、概念说明

React 组件测试（使用 Vitest 或 Jest）需要正确的类型配置，包括模拟函数类型、渲染结果类型、用户交互类型等。TypeScript 确保测试代码与生产代码保持类型一致。

## 二、具体用法

### 2.1 Vitest 类型配置

```tsx
// vitest.config.ts
import { defineConfig } from 'vitest/config';

export default defineConfig({
  test: {
    globals: true,
    environment: 'jsdom',
    setupFiles: ['./src/test/setup.ts'],
  },
});

// 测试文件中不需要额外 import，globals 模式下自动可用
```

### 2.2 渲染与查询类型

```tsx
import { render, screen } from '@testing-library/react';
import { expect, it, describe } from 'vitest';
import { Button } from './Button';

describe('Button', () => {
  it('应该正确渲染文本', () => {
    render(<Button onClick={() => {}}>点击我</Button>);

    // screen.getByText 返回 HTMLElement — 自动类型推断
    const button = screen.getByText('点击我');
    expect(button).toBeInTheDocument();
  });

  it('应该支持按角色查询', () => {
    render(<Button onClick={() => {}}>提交</Button>);

    // getByRole 支持角色类型约束
    const button = screen.getByRole('button', { name: '提交' });
    expect(button).toBeEnabled();
  });

  it('应该支持异步查询', async () => {
    render(<Button onClick={() => {}}>加载</Button>);

    // findByText 返回 Promise<HTMLElement>
    const button = await screen.findByText('加载');
    expect(button).toBeDefined();
  });
});
```

### 2.3 模拟函数类型

```tsx
import { vi, Mock } from 'vitest';

// 方式一：vi.fn() 自动推断类型
const handleClick = vi.fn(); // Mock<[], void>

// 方式二：显式标注 mock 函数类型
const fetchData: Mock<[url: string], Promise<Data>> = vi.fn();

// 方式三：带实现的 mock
const mockHandler = vi.fn<(id: number) => void>((id) => {
  console.log(id);
});

// 使用
it('点击应该调用回调', async () => {
  const user = userEvent.setup();
  render(<Button onClick={handleClick}>点击</Button>);

  await user.click(screen.getByRole('button'));

  expect(handleClick).toHaveBeenCalledTimes(1);
  expect(handleClick).toHaveBeenCalledWith(expect.any(Object)); // MouseEvent
});
```

### 2.4 Mock 模块类型

```tsx
import { vi } from 'vitest';

// Mock 整个模块
vi.mock('./api', () => ({
  fetchUser: vi.fn<() => Promise<User>>(),
}));

// 导入后类型仍然正确
import { fetchUser } from './api';

it('应该显示用户信息', async () => {
  // 设置 mock 返回值 — 类型安全
  vi.mocked(fetchUser).mockResolvedValue({
    id: 1,
    name: '张三',
  });

  render(<UserProfile userId={1} />);

  expect(await screen.findByText('张三')).toBeInTheDocument();
});
```

### 2.5 自定义 Render 包装器

```tsx
import { render, RenderOptions } from '@testing-library/react';
import { ReactElement, ReactNode } from 'react';

// 自定义 wrapper 类型
interface WrapperProps {
  children: ReactNode;
}

function AllProviders({ children }: WrapperProps) {
  return (
    <ThemeProvider>
      <BrowserRouter>
        {children}
      </BrowserRouter>
    </ThemeProvider>
  );
}

// 自定义 render 函数
function customRender(
  ui: ReactElement,
  options?: Omit<RenderOptions, 'wrapper'>
) {
  return render(ui, { wrapper: AllProviders, ...options });
}

// 导出覆盖
export { customRender as render };
```

## 三、注意事项与常见陷阱

1. **使用 `vi.mocked()` 获得正确的 mock 类型**：比手动类型断言更安全
2. **`vi.fn()` 可以指定泛型**：`vi.fn<(arg: string) => number>()`
3. **`@testing-library/react` 已自带类型**：不需要额外安装 `@types`
4. **`userEvent` 比 `fireEvent` 类型更好**：返回 Promise，支持更真实的交互模拟
5. **Mock 函数的参数类型应与原函数一致**：保持类型契约
