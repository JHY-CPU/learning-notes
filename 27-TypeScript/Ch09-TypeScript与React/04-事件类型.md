# 事件类型

## 一、概念说明

React 的事件系统与原生 DOM 事件略有不同，所有事件类型都以 `React.` 命名空间开头。正确使用事件类型可以确保事件处理函数能安全地访问事件对象的属性和方法。

React 合成事件（SyntheticEvent）是对原生事件的包装，提供了跨浏览器一致的 API。

## 二、具体用法

### 2.1 常用事件类型

```tsx
import React from 'react';

function EventExamples() {
  // 鼠标事件 — 最常用的事件类型
  const handleClick = (e: React.MouseEvent<HTMLButtonElement>) => {
    console.log(e.currentTarget);  // HTMLButtonElement
    console.log(e.clientX, e.clientY); // 鼠标坐标
  };

  // 输入框变化事件
  const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    console.log(e.target.value); // string — 自动类型推断
  };

  // 键盘事件
  const handleKeyDown = (e: React.KeyboardEvent<HTMLInputElement>) => {
    if (e.key === 'Enter') {
      console.log('回车键被按下');
    }
  };

  // 表单提交事件
  const handleSubmit = (e: React.FormEvent<HTMLFormElement>) => {
    e.preventDefault();
    const formData = new FormData(e.currentTarget);
    console.log(formData.get('username'));
  };

  // 焦点事件
  const handleFocus = (e: React.FocusEvent<HTMLInputElement>) => {
    console.log('获得焦点');
  };

  return (
    <form onSubmit={handleSubmit}>
      <input
        onChange={handleChange}
        onKeyDown={handleKeyDown}
        onFocus={handleFocus}
        name="username"
      />
      <button type="submit" onClick={handleClick}>提交</button>
    </form>
  );
}
```

### 2.2 事件类型速查表

```tsx
// 鼠标事件
React.MouseEvent<T>          // click, dblclick, mouseenter 等

// 键盘事件
React.KeyboardEvent<T>       // keydown, keyup, keypress

// 表单事件
React.FormEvent<T>           // submit, reset
React.ChangeEvent<T>         // input, select, textarea 变化

// 焦点事件
React.FocusEvent<T>          // focus, blur

// 触摸事件（移动端）
React.TouchEvent<T>          // touchstart, touchmove, touchend

// 拖拽事件
React.DragEvent<T>           // drag, dragstart, drop

// 剪贴板事件
React.ClipboardEvent<T>      // copy, paste, cut

// 其中 T 是目标元素类型：
// HTMLButtonElement, HTMLInputElement, HTMLDivElement 等
```

### 2.3 事件处理函数类型

```tsx
// 方式一：内联类型
type ButtonProps = {
  onClick: (e: React.MouseEvent<HTMLButtonElement>) => void;
};

// 方式二：使用 React.EventHandler
type InputProps = {
  onChange: React.ChangeEventHandler<HTMLInputElement>;
};

// 方式三：通用事件处理器
type Handler<T extends HTMLElement> = (e: React.MouseEvent<T>) => void;
```

## 三、注意事项与常见陷阱

1. **泛型参数必须是 HTML 元素类型**：`React.MouseEvent<HTMLButtonElement>` 而非 `React.MouseEvent`
2. **`e.target` vs `e.currentTarget`**：`target` 是实际触发的元素（可能是子元素），`currentTarget` 是绑定了事件的元素
3. **异步访问事件属性**：React 事件是池化的，异步访问时需调用 `e.persist()`（React 17 之前）
4. **不要用 `any` 作为事件类型**：会失去对 `e.target.value` 等属性的类型推断
5. **`onChange` 在 React 中等同于原生 `onInput`**：每次输入都会触发
