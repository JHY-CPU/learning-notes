# 样式类型

## 一、概念说明

React 中的样式有内联样式、CSS Modules、CSS-in-JS 等多种方案。TypeScript 为每种方案提供了不同的类型支持，其中 `React.CSSProperties` 是内联样式的核心类型。

## 二、具体用法

### 2.1 内联样式类型

```tsx
import React from 'react';

// CSSProperties — 内联样式的类型
const styles: React.CSSProperties = {
  display: 'flex',
  flexDirection: 'column',
  padding: '16px',
  backgroundColor: '#f5f5f5',
  borderRadius: 8, // 数字会被转为 px
};

// 组件中使用
type BoxProps = {
  style?: React.CSSProperties;
  children: React.ReactNode;
};

function Box({ style, children }: BoxProps) {
  return <div style={{ ...styles, ...style }}>{children}</div>;
}

// 使用
<Box style={{ color: 'red', fontSize: 14 }}>
  <span>内容</span>
</Box>;
```

### 2.2 CSS Modules 类型

```tsx
// 方法一：手动声明类型
// styles.module.css.d.ts
declare const styles: {
  readonly container: string;
  readonly header: string;
  readonly footer: string;
};
export default styles;

// 使用
import styles from './styles.module.css';
<div className={styles.container}>内容</div>;

// 方法二：使用 vite-plugin-css-modules-types 或开启 CSS Module 类型推断
// 方法三：tsconfig 中配置 plugins
```

### 2.3 styled-components 类型

```tsx
import styled from 'styled-components';

// 基本使用 — 自动推断类型
const Button = styled.button`
  background: ${props => props.theme.primary};
  color: white;
  padding: 8px 16px;
`;

// 带泛型 props 的组件
interface CardProps {
  variant: 'elevated' | 'outlined';
  padding?: number;
}

const Card = styled.div<CardProps>`
  background: ${props => props.variant === 'elevated' ? 'white' : 'transparent'};
  box-shadow: ${props => props.variant === 'elevated' ? '0 2px 8px rgba(0,0,0,0.1)' : 'none'};
  padding: ${props => props.padding ?? 16}px;
`;

// 使用 — 自动类型检查
<Card variant="elevated" padding={24}>内容</Card>;

// 主题类型
interface Theme {
  primary: string;
  secondary: string;
  background: string;
}

const theme: Theme = {
  primary: '#007bff',
  secondary: '#6c757d',
  background: '#ffffff',
};
```

### 2.4 Tailwind CSS 类型

```tsx
// Tailwind 本身不需要特殊类型
// 但可以用联合类型约束 class 名称
type ButtonVariant = 'primary' | 'secondary' | 'danger';
type ButtonSize = 'sm' | 'md' | 'lg';

const variantClasses: Record<ButtonVariant, string> = {
  primary: 'bg-blue-500 text-white',
  secondary: 'bg-gray-200 text-gray-800',
  danger: 'bg-red-500 text-white',
};

type ButtonProps = {
  variant: ButtonVariant;
  size?: ButtonSize;
  className?: string;
  children: React.ReactNode;
  onClick?: () => void;
};

function Button({ variant, size = 'md', className, children, onClick }: ButtonProps) {
  const sizeClasses = { sm: 'px-2 py-1 text-sm', md: 'px-4 py-2', lg: 'px-6 py-3 text-lg' };
  return (
    <button
      className={`${variantClasses[variant]} ${sizeClasses[size]} ${className ?? ''}`}
      onClick={onClick}
    >
      {children}
    </button>
  );
}
```

## 三、注意事项与常见陷阱

1. **CSSProperties 的属性名用 camelCase**：`backgroundColor` 而非 `background-color`
2. **CSS Modules 需要类型声明文件**：否则 TS 不认识模块导入
3. **styled-components 的 props 在模板字符串中有类型**：能自动推断
4. **`className` 使用 `string | undefined`**：处理可选的类名合并
5. **Tailwind 没有内置类型约束**：需要自己维护 class 名的类型安全
