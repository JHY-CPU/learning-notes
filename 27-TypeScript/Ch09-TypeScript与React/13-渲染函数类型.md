# 渲染函数类型

## 一、概念说明

React 中有多种表示"可渲染内容"的类型，理解它们的区别是正确使用 TS + React 的基础。`ReactNode` 是最通用的类型，`JSX.Element` 更严格，`ReactElement` 是具体的元素类型。

## 二、具体用法

### 2.1 类型层次关系

```tsx
import React, { ReactNode, ReactElement, JSX } from 'react';

// 类型范围从小到大：
// ReactElement < JSX.Element < ReactNode

// ReactElement — JSX 元素的类型
const element: ReactElement = <div>Hello</div>;

// JSX.Element — 等同于 ReactElement<any, any>
const jsx: JSX.Element = <div>Hello</div>;

// ReactNode — 最通用的可渲染类型
// string | number | boolean | null | undefined | ReactElement | ReactFragment | ReactPortal
const node: ReactNode = 'Hello';    // OK
const node2: ReactNode = 42;        // OK
const node3: ReactNode = null;      // OK
const node4: ReactNode = <div />;   // OK
```

### 2.2 各场景推荐类型

```tsx
// children — 用 ReactNode（最通用）
type LayoutProps = {
  children: React.ReactNode;
  sidebar?: React.ReactNode;
};

function Layout({ children, sidebar }: LayoutProps) {
  return (
    <div>
      <aside>{sidebar}</aside>
      <main>{children}</main>
    </div>
  );
}

// 只接受单个元素 — 用 ReactElement
type WrapperProps = {
  child: React.ReactElement;
};

// Render Props — 用函数返回 ReactNode
type DataGridProps<T> = {
  data: T[];
  renderItem: (item: T, index: number) => React.ReactNode;
};

function DataGrid<T>({ data, renderItem }: DataGridProps<T>) {
  return <ul>{data.map((item, i) => renderItem(item, i))}</ul>;
}

// 条件渲染 — 返回 ReactNode | null
function Conditional({ show, children }: { show: boolean; children: React.ReactNode }) {
  return show ? <>{children}</> : null;
}
```

### 2.3 渲染函数 vs children

```tsx
// 方式一：children 作为函数（Render Props 模式）
type MouseTrackerProps = {
  children: (position: { x: number; y: number }) => React.ReactNode;
};

function MouseTracker({ children }: MouseTrackerProps) {
  const [pos, setPos] = useState({ x: 0, y: 0 });
  return (
    <div onMouseMove={e => setPos({ x: e.clientX, y: e.clientY })}>
      {children(pos)}
    </div>
  );
}

// 使用
<MouseTracker>
  {({ x, y }) => <p>鼠标位置: {x}, {y}</p>}
</MouseTracker>

// 方式二：render 属性
type ListProps<T> = {
  items: T[];
  render: (item: T) => React.ReactNode;
};

function List<T>({ items, render }: ListProps<T>) {
  return <>{items.map(render)}</>;
}
```

### 2.4 组件返回类型

```tsx
// 函数组件的返回类型
function Greeting(): React.ReactNode {
  return <div>Hello</div>; // OK
}

// 更精确的返回类型
function Header(): JSX.Element {
  return <header>标题</header>;
}

// 异步组件（React.lazy）返回 Promise<ReactElement>
```

## 三、注意事项与常见陷阱

1. **children 优先用 `ReactNode`**：可以接受任何可渲染内容
2. **Render Props 用 `ReactNode` 返回值**：而不是 `ReactElement`，因为可能返回 `null`、字符串等
3. **不要用 `JSX.Element`**：它是全局类型，建议用 `React.ReactElement`
4. **`ReactNode` 包含 `undefined` 和 `null`**：适合可选的渲染内容
5. **组件返回类型通常不需要标注**：TypeScript 自动推断
