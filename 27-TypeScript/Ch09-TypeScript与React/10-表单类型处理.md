# 表单类型处理

## 一、概念说明

表单处理是 React 应用中最常见的场景之一。TypeScript 提供了精确的类型来处理表单事件、表单元素和表单数据。正确使用类型可以避免 `e.target.value` 的类型错误，并确保表单数据的完整性。

## 二、具体用法

### 2.1 表单元素事件类型

```tsx
import { useState, FormEvent, ChangeEvent } from 'react';

function LoginForm() {
  const [username, setUsername] = useState('');
  const [password, setPassword] = useState('');
  const [role, setRole] = useState<'admin' | 'user'>('user');

  // input 变化事件
  const handleUsernameChange = (e: ChangeEvent<HTMLInputElement>) => {
    setUsername(e.target.value); // e.target.value 类型是 string
  };

  // select 变化事件
  const handleRoleChange = (e: ChangeEvent<HTMLSelectElement>) => {
    setRole(e.target.value as 'admin' | 'user');
  };

  // textarea 变化事件
  const handleTextareaChange = (e: ChangeEvent<HTMLTextAreaElement>) => {
    console.log(e.target.value);
  };

  // 表单提交
  const handleSubmit = (e: FormEvent<HTMLFormElement>) => {
    e.preventDefault();
    console.log({ username, password, role });
  };

  return (
    <form onSubmit={handleSubmit}>
      <input onChange={handleUsernameChange} />
      <select onChange={handleRoleChange}>
        <option value="user">用户</option>
        <option value="admin">管理员</option>
      </select>
      <button type="submit">登录</button>
    </form>
  );
}
```

### 2.2 受控组件类型

```tsx
// 通用的受控输入组件
type InputValue = string | number | boolean;

interface FormFieldProps {
  label: string;
  name: string;
  type?: 'text' | 'number' | 'email' | 'password' | 'checkbox';
  value: InputValue;
  onChange: (name: string, value: InputValue) => void;
  error?: string;
}

function FormField({ label, name, type = 'text', value, onChange, error }: FormFieldProps) {
  if (type === 'checkbox') {
    return (
      <div>
        <label>
          <input
            type="checkbox"
            checked={value as boolean}
            onChange={e => onChange(name, e.target.checked)}
          />
          {label}
        </label>
      </div>
    );
  }

  return (
    <div>
      <label>{label}</label>
      <input
        type={type}
        value={value as string | number}
        onChange={e => onChange(name, type === 'number' ? Number(e.target.value) : e.target.value)}
      />
      {error && <span className="error">{error}</span>}
    </div>
  );
}
```

### 2.3 FormData 类型处理

```tsx
// 使用 FormData API
function ContactForm() {
  const handleSubmit = (e: FormEvent<HTMLFormElement>) => {
    e.preventDefault();
    const formData = new FormData(e.currentTarget);

    // formData.get() 返回 FormDataEntryValue | null
    const name = formData.get('name') as string;
    const email = formData.get('email') as string;
    const message = formData.get('message') as string;

    // 类型安全的表单数据提取
    const data: Record<string, string> = {};
    formData.forEach((value, key) => {
      data[key] = value.toString();
    });

    console.log(data);
  };

  return (
    <form onSubmit={handleSubmit}>
      <input name="name" required />
      <input name="email" type="email" required />
      <textarea name="message" required />
      <button type="submit">发送</button>
    </form>
  );
}
```

### 2.4 泛型表单 Hook

```tsx
function useForm<T extends Record<string, unknown>>(initialValues: T) {
  const [values, setValues] = useState<T>(initialValues);
  const [errors, setErrors] = useState<Partial<Record<keyof T, string>>>({});

  const handleChange = (e: ChangeEvent<HTMLInputElement | HTMLSelectElement>) => {
    const { name, value, type } = e.target;
    setValues(prev => ({
      ...prev,
      [name]: type === 'checkbox'
        ? (e.target as HTMLInputElement).checked
        : value,
    }));
  };

  const reset = () => setValues(initialValues);

  return { values, errors, setErrors, handleChange, reset, setValues };
}

// 使用
function RegistrationForm() {
  const { values, handleChange } = useForm({
    username: '',
    email: '',
    password: '',
    agree: false,
  });
  // values 的类型被推断为 { username: string; email: string; ... }
}
```

## 三、注意事项与常见陷阱

1. **`e.target` vs `e.currentTarget`**：用 `e.currentTarget` 获取绑定了事件的元素
2. **checkbox 的值用 `e.target.checked`**：不是 `.value`
3. **`FormData.get()` 返回 `string | null`**：需要做 null 检查或类型断言
4. **受控组件的 `value` 不能为 `undefined`**：用空字符串 `''` 代替
5. **select 的 value 需要类型断言**：`e.target.value` 是 `string`，需要断言为联合类型
