# 常见类型错误

## 一、概念说明

React + TypeScript 开发中常见的类型错误包括隐式 children 属性、事件类型不匹配、条件渲染类型收窄失败等。了解这些错误及其解决方案可以大幅提高开发效率。

## 二、常见错误与解决

### 2.1 隐式 children 属性（React 18 变更）

```tsx
// React 18 错误 — children 不再隐式包含在 FC 类型中
// 错误写法
const Wrapper: React.FC = ({ children }) => <div>{children}</div>;
// 在 React 18 中如果没传 children，不会报错但会渲染空

// 正确写法 — 显式声明 children
interface WrapperProps {
  children: React.ReactNode;
}

function Wrapper({ children }: WrapperProps) {
  return <div>{children}</div>;
}

// 或者使用 PropsWithChildren
function Box({ children }: React.PropsWithChildren<{}>) {
  return <div>{children}</div>;
}
```

### 2.2 事件类型不匹配

```tsx
// 错误 — onClick 事件类型不匹配
const handleClick = (e: React.MouseEvent) => { /* ... */ };

// Button 组件的 onClick 期望的类型
// React.MouseEventHandler<HTMLButtonElement>

// 正确写法
const handleClick: React.MouseEventHandler<HTMLButtonElement> = (e) => {
  console.log(e.currentTarget); // 类型安全
};

// 或内联标注
<button onClick={(e: React.MouseEvent<HTMLButtonElement>) => {}} />;
```

### 2.3 条件渲染中的类型收窄

```tsx
// 错误 — TypeScript 无法跨越 JSX 边界收窄类型
interface User {
  name: string;
  email: string;
}

function Greeting({ user }: { user: User | null }) {
  // 错误：TypeScript 不知道这里的 user 已经是 User
  // return user && <div>Hello, {user.name}</div>;

  // 正确写法 — 先收窄
  if (!user) return null;
  return <div>Hello, {user.name}</div>;
}
```

### 2.4 状态更新函数类型

```tsx
// 错误 — 函数式更新的类型
const [items, setItems] = useState<string[]>([]);

// 错误：参数类型不匹配
// setItems([...items, 'new']);

// 正确：使用函数式更新
setItems(prev => [...prev, 'new']); // prev 类型自动推断为 string[]
```

### 2.5 Ref 类型错误

```tsx
// 错误 — 忘记 null 初始值
const ref = useRef<HTMLDivElement>(); // 类型是 MutableRefObject<HTMLDivElement | undefined>

// 正确
const ref = useRef<HTMLDivElement>(null); // 类型是 MutableRefObject<HTMLDivElement | null>

// 访问时需要 null 检查
ref.current?.focus(); // 可选链
```

### 2.6 Key 属性类型

```tsx
// 错误 — key 必须是 string | number
const items = [{ id: Symbol(), name: 'Item' }];

// 正确 — key 应该是 string 或 number
const items = [
  { id: 1, name: 'Item 1' },
  { id: 'abc', name: 'Item 2' },
];

items.map(item => <div key={item.id}>{item.name}</div>);
```

### 2.7 组件作为 JSX 元素

```tsx
// 错误 — 把组件类型用作值
function Container({ header }: { header: React.FC }) {
  return <div>{header()}</div>; // header 是组件函数，需要调用
}

// 正确 — 传入 ReactElement 而非组件类型
function Container({ header }: { header: React.ReactNode }) {
  return <div>{header}</div>;
}

// 使用
<Container header={<h1>标题</h1>} />;
```

### 2.8 受控组件的 value 类型

```tsx
// 错误 — value 不能是 undefined
const [value, setValue] = useState<string | undefined>(undefined);

// 正确 — 受控组件的 value 应该有值
const [value, setValue] = useState('');

<input value={value} onChange={e => setValue(e.target.value)} />;
```

### 2.9 dangerouslySetInnerHTML 类型

```tsx
// 正确的类型
<div dangerouslySetInnerHTML={{ __html: '<p>HTML内容</p>' }} />

// 类型定义
interface DangerousHTML {
  __html: string;
}
```

## 三、注意事项与常见陷阱

1. **React 18 不再隐式包含 children**：必须显式声明
2. **事件对象永远存在**：不需要做 null 检查
3. **`key` 不是 props 的一部分**：不会传递给组件
4. **`ref` 在 `forwardRef` 组件中是第二个参数**：不是 props
5. **保持 TypeScript 版本与 `@types/react` 版本同步**
