# 模块扩展

## 一、概念说明

模块扩展允许为已有模块添加新的类型声明，而无需修改原始代码。这在使用第三方库时特别有用，可以为其添加自定义属性、方法或事件类型。

## 二、具体用法

### 2.1 扩展 Express Request

```typescript
// types/express.d.ts
import 'express';

declare module 'express' {
  interface Request {
    user?: {
      id: number;
      email: string;
      role: 'admin' | 'user';
    };
    requestId?: string;
    startTime?: number;
  }

  interface Response {
    success<T>(data: T, message?: string): void;
    error(message: string, statusCode?: number): void;
  }
}
```

### 2.2 扩展 Vue 组件属性

```typescript
// types/vue.d.ts
import 'vue';

declare module 'vue' {
  interface ComponentCustomProperties {
    $http: HttpClient;
    $translate: (key: string, params?: Record<string, string>) => string;
    $formatDate: (date: Date | string) => string;
  }

  interface ComponentCustomProps {
    'data-testid'?: string;
  }
}
```

### 2.3 扩展第三方库类型

```typescript
// 扩展 lodash
import 'lodash';

declare module 'lodash' {
  interface LoDashStatic {
    myCustomMethod<T>(array: T[]): T[];
  }
}

// 扩展 axios
import 'axios';

declare module 'axios' {
  interface AxiosRequestConfig {
    metadata?: {
      startTime: number;
    };
  }

  interface AxiosResponse<T = unknown> {
    duration?: number;
  }
}
```

### 2.4 扩展 Pinia Store

```typescript
// types/pinia.d.ts
import 'pinia';

declare module 'pinia' {
  export interface PiniaCustomProperties {
    $reset: () => void;
    $clearPersist: () => void;
  }

  export interface DefineStoreOptionsBase<S, Store> {
    persist?: boolean | PersistOptions;
  }
}

interface PersistOptions {
  key?: string;
  storage?: Storage;
  paths?: string[];
}
```

### 2.5 扩展全局对象

```typescript
// 扩展 Window
declare global {
  interface Window {
    gtag: (...args: unknown[]) => void;
    dataLayer: unknown[];
    __APP_VERSION__: string;
  }
}

// 扩展 Array
declare global {
  interface Array<T> {
    toUnique(): T[];
    groupBy<K extends keyof T>(key: K): Map<T[K], T[]>;
  }
}

export {}; // 确保文件是模块
```

## 三、注意事项与常见陷阱

1. **扩展必须在模块文件中**：需要有 `import` 或 `export`
2. **扩展的类型会全局生效**：所有文件都能看到
3. **不要扩展不相关的模块**：可能导致类型冲突
4. **`export {}` 确保文件是模块**：在需要时使用
5. **扩展的属性应标记为可选**：`?: type`
