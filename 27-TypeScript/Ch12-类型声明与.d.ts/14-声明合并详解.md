# 声明合并详解

## 一、概念说明

声明合并是 TypeScript 的独特特性，允许将多个同名声明合并为一个。最常见的应用是接口合并和命名空间合并，用于扩展已有类型而不修改原始代码。

## 二、具体用法

### 2.1 接口合并

```typescript
// 同名接口自动合并
interface Box {
  width: number;
  height: number;
}

// 后续声明的属性被添加到同一接口
interface Box {
  depth: number;
  color: string;
}

// Box 现在有 4 个属性
const box: Box = {
  width: 10,
  height: 20,
  depth: 30,
  color: 'red',
};

// 合并规则：
// 1. 非函数成员必须唯一（否则类型必须相同）
// 2. 函数成员会被重载
interface Document {
  createElement(tagName: 'div'): HTMLDivElement;
  createElement(tagName: 'span'): HTMLSpanElement;
}

interface Document {
  createElement(tagName: 'canvas'): HTMLCanvasElement;
}

// Document.createElement 现在有 3 个重载
```

### 2.2 命名空间合并

```typescript
namespace Validation {
  export interface StringValidator {
    validate(value: string): boolean;
  }
}

namespace Validation {
  // 合并到同一命名空间
  export class LettersOnly implements StringValidator {
    validate(value: string): boolean {
      return /^[a-zA-Z]+$/.test(value);
    }
  }
}

// 可以使用所有合并后的成员
const validator: Validation.StringValidator = new Validation.LettersOnly();
```

### 2.3 命名空间与类合并

```typescript
class Greeter {
  greeting: string;
  constructor(message: string) {
    this.greeting = message;
  }
}

// 为类添加静态成员
namespace Greeter {
  export const DEFAULT_GREETING = 'Hello';
  export function createDefault(): Greeter {
    return new Greeter(Greeter.DEFAULT_GREETING);
  }
}

// 使用
const g = Greeter.createDefault();
Greeter.DEFAULT_GREETING;
```

### 2.4 命名空间与函数合并

```typescript
function createPerson(name: string): Person {
  return { name, age: 0 };
}

namespace createPerson {
  export interface Person {
    name: string;
    age: number;
  }
}

// 可以调用，也可以访问命名空间类型
const p = createPerson('张三'); // 返回类型是 Person
type PersonType = createPerson.Person;
```

### 2.5 命名空间与枚举合并

```typescript
enum Color {
  Red = 1,
  Green = 2,
  Blue = 3,
}

namespace Color {
  // 为枚举添加方法
  export function fromString(value: string): Color | undefined {
    const map: Record<string, Color> = {
      red: Color.Red,
      green: Color.Green,
      blue: Color.Blue,
    };
    return map[value.toLowerCase()];
  }

  export function toString(color: Color): string {
    return Color[color];
  }
}

// 使用
const c = Color.fromString('red'); // Color | undefined
```

### 2.6 模块扩展合并

```typescript
// 扩展 Express 类型
import 'express';

declare module 'express-serve-static-core' {
  interface Request {
    user?: { id: number; role: string };
  }
}
```

## 三、注意事项与常见陷阱

1. **接口合并的同名属性类型必须兼容**：否则编译报错
2. **函数成员会创建重载**：后声明的签名优先
3. **命名空间合并要求同名**：不同名的命名空间不会合并
4. **`declare` 声明的合并**：用于 `.d.ts` 文件
5. **不要滥用声明合并**：可能导致类型难以理解和维护
