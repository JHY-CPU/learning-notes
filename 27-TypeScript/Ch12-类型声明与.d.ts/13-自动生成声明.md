# 自动生成声明

## 一、概念说明

TypeScript 编译器可以自动为 `.ts` 文件生成 `.d.ts` 声明文件。这对于发布 npm 包非常有用，让用户在使用你的库时获得完整的类型提示。

## 二、具体用法

### 2.1 使用 tsc 生成

```bash
# 基本生成
npx tsc --declaration --emitDeclarationOnly

# 带 source map
npx tsc --declaration --declarationMap

# 指定输出目录
npx tsc --declaration --outDir dist
```

### 2.2 tsconfig 配置

```json
{
  "compilerOptions": {
    "declaration": true,         // 生成 .d.ts 文件
    "declarationMap": true,      // 生成 .d.ts.map（IDE 可跳转到源码）
    "declarationDir": "./dist",  // 声明文件输出目录
    "emitDeclarationOnly": true, // 只生成声明文件，不生成 .js
    "outDir": "./dist",
    "rootDir": "./src",
    "strict": true
  },
  "include": ["src/**/*"]
}
```

### 2.3 库发布配置

```json
// package.json
{
  "name": "my-library",
  "version": "1.0.0",
  "main": "dist/index.js",
  "types": "dist/index.d.ts",      // 声明文件入口
  "exports": {
    ".": {
      "import": "./dist/index.mjs",
      "require": "./dist/index.js",
      "types": "./dist/index.d.ts"
    }
  },
  "files": ["dist"],
  "scripts": {
    "build": "tsc",
    "build:types": "tsc --declaration --emitDeclarationOnly"
  }
}
```

### 2.4 生成结果示例

```typescript
// src/index.ts
export interface User {
  id: number;
  name: string;
}

export function greet(user: User): string {
  return `Hello, ${user.name}`;
}

export class UserService {
  private users: User[] = [];

  addUser(user: User): void {
    this.users.push(user);
  }

  findUser(id: number): User | undefined {
    return this.users.find(u => u.id === id);
  }
}

// 自动生成的 dist/index.d.ts
export interface User {
  id: number;
  name: string;
}
export declare function greet(user: User): string;
export declare class UserService {
  private users;
  addUser(user: User): void;
  findUser(id: number): User | undefined;
}
```

### 2.5 Rollup/Vite 生成声明

```typescript
// vite.config.ts
import { defineConfig } from 'vite';
import dts from 'vite-plugin-dts';

export default defineConfig({
  plugins: [
    dts({
      include: ['src/**/*'],
      outDir: 'dist',
      rollupTypes: true, // 合并声明文件
    }),
  ],
  build: {
    lib: {
      entry: 'src/index.ts',
      formats: ['es', 'cjs'],
    },
  },
});
```

## 三、注意事项与常见陷阱

1. **`declaration: true` 必须搭配 `outDir`**：否则声明文件与源文件混在一起
2. **私有类型不会被导出**：只有被公共 API 使用的类型才会出现在声明中
3. **`declarationMap` 让 IDE 跳转到源码**：调试体验更好
4. **检查生成的声明文件**：确保导出的类型正确
5. **`emitDeclarationOnly` 用于类型和实现分离的场景**
