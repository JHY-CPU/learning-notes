# 全局类型扩展

## 一、概念说明

`declare global` 允许在模块文件中声明全局类型，使这些类型在所有 TypeScript 文件中可用。这在需要扩展内置对象（如 `Window`、`Array`、`Promise`）或声明全局变量时非常有用。

## 二、具体用法

### 2.1 扩展 Window 对象

```typescript
// src/types/global.d.ts
export {};

declare global {
  interface Window {
    // 第三方库
    gtag: (...args: unknown[]) => void;
    fbq: (...args: unknown[]) => void;

    // 应用全局
    __APP_CONFIG__: {
      apiUrl: string;
      version: string;
      debug: boolean;
    };

    // Electron 环境
    electronAPI?: {
      openFile: () => Promise<string>;
      saveFile: (data: string) => Promise<void>;
    };
  }
}

// 使用
window.__APP_CONFIG__?.apiUrl;
window.gtag?.('event', 'click');
```

### 2.2 扩展内置类型

```typescript
export {};

declare global {
  // 扩展 Array
  interface Array<T> {
    /**
     * 数组去重
     */
    toUnique(): T[];

    /**
     * 按条件分组
     */
    groupBy<K extends keyof T>(key: K): Map<T[K], T[]>;

    /**
     * 安全获取元素
     */
    at(index: number): T | undefined;
  }

  // 扩展 String
  interface String {
    toCapitalized(): string;
    truncate(maxLength: number): string;
  }
}

// 实现（在 .ts 文件中）
Array.prototype.toUnique = function () {
  return [...new Set(this)];
};
```

### 2.3 全局常量

```typescript
export {};

declare global {
  // Webpack 注入
  const __webpack_public_path__: string;
  const __VERSION__: string;

  // Vite 注入
  const __DEV__: boolean;
  const __BUILD_TIME__: string;

  // 自定义
  const API_TIMEOUT: number;
  const FEATURE_FLAGS: Record<string, boolean>;
}
```

### 2.4 全局类型别名

```typescript
export {};

declare global {
  // 通用响应类型
  type ApiResponse<T> = {
    success: boolean;
    data: T;
    error?: string;
  };

  // ID 类型
  type ID = string | number;

  // 可空类型
  type Nullable<T> = T | null | undefined;

  // 时间戳类型
  type Timestamp = number;
}
```

### 2.5 条件全局扩展

```typescript
// 根据环境变量扩展类型
export {};

declare global {
  namespace NodeJS {
    interface ProcessEnv {
      NODE_ENV: 'development' | 'production' | 'test';
      API_URL: string;
      DATABASE_URL: string;
      JWT_SECRET: string;
      REDIS_URL?: string;
    }
  }
}
```

## 三、注意事项与常见陷阱

1. **必须在模块文件中**：需要 `export {}` 或其他 `import/export`
2. **不要过度使用全局类型**：会导致命名冲突
3. **全局常量只做类型声明**：运行时值由构建工具注入
4. **扩展 `Array.prototype` 需要同时扩展类型**：否则 TS 不知道新方法存在
5. **全局类型扩展是不可逆的**：在整个项目中生效
