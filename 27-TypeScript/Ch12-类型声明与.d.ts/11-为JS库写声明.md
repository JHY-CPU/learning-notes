# 为JS库写声明

## 一、概念说明

为没有类型的 JavaScript 库编写类型声明是一项常见任务。需要分析库的 API，定义模块导出、函数签名、类结构和配置选项的类型。

## 二、具体用法

### 2.1 分析库 API

```typescript
// 假设有一个 JS 库 my-chart-lib
// API: Chart.create(container, options)
//      Chart.prototype.update(data)
//      Chart.prototype.destroy()

// types/my-chart-lib.d.ts
declare module 'my-chart-lib' {
  interface ChartOptions {
    type: 'bar' | 'line' | 'pie';
    width?: number;
    height?: number;
    animate?: boolean;
    colors?: string[];
  }

  interface DataPoint {
    label: string;
    value: number;
  }

  class Chart {
    constructor(container: HTMLElement, options: ChartOptions);

    static create(container: HTMLElement, options: ChartOptions): Chart;

    update(data: DataPoint[]): void;
    destroy(): void;

    readonly element: HTMLElement;
    readonly options: ChartOptions;
  }

  export = Chart;
}
```

### 2.2 回调风格 API

```typescript
declare module 'legacy-ajax' {
  interface AjaxOptions {
    url: string;
    method?: 'GET' | 'POST' | 'PUT' | 'DELETE';
    data?: Record<string, unknown>;
    headers?: Record<string, string>;

    onSuccess?: (response: unknown) => void;
    onError?: (error: { status: number; message: string }) => void;
    onComplete?: () => void;
  }

  function ajax(options: AjaxOptions): void;

  // 工具函数
  namespace ajax {
    function get(url: string, callback: (data: unknown) => void): void;
    function post(url: string, data: unknown, callback: (data: unknown) => void): void;
    function serialize(obj: Record<string, unknown>): string;
  }

  export = ajax;
}
```

### 2.3 插件风格 API

```typescript
declare module 'my-plugin' {
  interface PluginOptions {
    enabled: boolean;
    config?: Record<string, unknown>;
  }

  interface Plugin {
    name: string;
    version: string;
    install(app: unknown, options?: PluginOptions): void;
    uninstall?(app: unknown): void;
  }

  // 默认导出插件对象
  const plugin: Plugin;
  export default plugin;

  // 命名导出
  export function usePlugin(plugin: Plugin): void;
  export function getInstalledPlugins(): Plugin[];
}
```

### 2.4 工具类型辅助

```typescript
declare module 'my-utils' {
  // 通用类型
  type Callback<T = void> = (result: T) => void;
  type AsyncResult<T> = Promise<{ success: boolean; data?: T; error?: string }>;

  // 函数声明
  function debounce<T extends (...args: any[]) => any>(
    fn: T,
    delay: number
  ): (...args: Parameters<T>) => void;

  function throttle<T extends (...args: any[]) => any>(
    fn: T,
    limit: number
  ): (...args: Parameters<T>) => void;

  function deepMerge<T extends object>(target: T, ...sources: Partial<T>[]): T;

  // 类型守卫
  function isString(value: unknown): value is string;
  function isNumber(value: unknown): value is number;
  function isObject(value: unknown): value is object;
}
```

### 2.5 测试声明文件

```typescript
// 在项目中测试声明是否正确
import { Chart, type ChartOptions, type DataPoint } from 'my-chart-lib';

const options: ChartOptions = {
  type: 'bar',
  width: 800,
  height: 400,
};

const chart = new Chart(document.getElementById('chart')!, options);

const data: DataPoint[] = [
  { label: 'A', value: 10 },
  { label: 'B', value: 20 },
];

chart.update(data);
chart.destroy();
```

## 三、注意事项与常见陷阱

1. **先阅读库的文档和源码**：理解完整的 API
2. **`export =` 用于 CommonJS 模块**：`export default` 用于 ES 模块
3. **不要用 `any`**：尽量精确描述类型
4. **提交到 DefinitelyTyped**：为社区贡献类型声明
5. **测试声明文件的正确性**：在实际项目中验证
