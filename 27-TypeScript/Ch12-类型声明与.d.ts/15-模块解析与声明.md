# 模块解析与声明

## 一、概念说明

TypeScript 的模块解析决定了编译器如何查找类型声明文件。`typeRoots`、`types` 和 `paths` 等配置选项控制类型包的搜索路径和模块解析行为。

## 二、具体用法

### 2.1 类型包解析

```json
// tsconfig.json
{
  "compilerOptions": {
    // typeRoots — 类型包搜索路径
    // 默认: ["node_modules/@types"]
    "typeRoots": ["./node_modules/@types", "./custom-types"],

    // types — 只包含指定的类型包
    // 默认: 包含所有 @types 包
    "types": ["node", "jest"],

    // 空数组 = 不自动包含任何 @types
    "types": []
  }
}
```

### 2.2 模块路径映射

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"],               // @/utils → src/utils
      "@components/*": ["src/components/*"],
      "@shared/*": ["../shared/*"],    // 跨包引用
      "my-lib": ["./custom-types/my-lib"] // 自定义类型覆盖
    }
  }
}
```

### 2.3 模块解析策略

```json
{
  "compilerOptions": {
    // Node — 经典 Node.js 解析
    "moduleResolution": "Node",

    // Node16/NodeNext — 匹配 Node.js ESM/CJS
    "moduleResolution": "NodeNext",

    // Bundler — 匹配 Vite/Webpack 行为
    "moduleResolution": "Bundler",

    // 经典 — 旧式解析（不推荐）
    "moduleResolution": "Classic"
  }
}
```

### 2.4 自定义类型目录

```
project/
├── custom-types/
│   ├── my-lib/
│   │   └── index.d.ts
│   └── global.d.ts
├── src/
├── tsconfig.json
```

```json
{
  "compilerOptions": {
    "typeRoots": ["./custom-types", "./node_modules/@types"]
  }
}
```

### 2.5 包导出类型

```json
// my-package/package.json
{
  "name": "my-package",
  "exports": {
    ".": {
      "types": "./dist/index.d.ts",  // types 条件必须在最前面
      "import": "./dist/index.mjs",
      "require": "./dist/index.js"
    },
    "./utils": {
      "types": "./dist/utils.d.ts",
      "import": "./dist/utils.mjs",
      "default": "./dist/utils.js"
    }
  }
}
```

### 2.6 路径映射与 monorepo

```json
// 根 tsconfig.json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@myorg/shared": ["packages/shared/src/index.ts"],
      "@myorg/api": ["packages/api/src/index.ts"],
      "@myorg/web": ["packages/web/src/index.ts"]
    }
  },
  "references": [
    { "path": "./packages/shared" },
    { "path": "./packages/api" }
  ]
}
```

## 三、注意事项与常见陷阱

1. **`paths` 需要配合 `baseUrl` 使用**：路径相对于 `baseUrl`
2. **`types` 为空数组时需要手动引入**：`/// <reference types="node" />`
3. **`moduleResolution: "Bundler"` 最宽松**：匹配打包工具的行为
4. **`exports` 中 `types` 条件必须在最前面**：确保类型优先解析
5. **路径映射只影响 TypeScript 编译**：运行时需要构建工具同步配置
