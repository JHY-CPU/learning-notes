# 条件类型声明

## 一、概念说明

条件类型声明允许根据不同的条件导出不同的类型。这在构建库时非常有用，可以根据环境或参数提供不同的类型定义。

## 二、具体用法

### 2.1 条件导出类型

```typescript
// types/index.d.ts
export type { Config } from './config';
export type { User, CreateUserDto } from './user';
export type { ApiResponse, PaginatedResponse } from './api';

// 根据环境条件导出
type EnvConfig = typeof process.env extends { NODE_ENV: 'test' }
  ? { mockData: boolean }
  : { apiUrl: string };
```

### 2.2 条件类型工具

```typescript
// 条件类型 — 根据输入类型返回不同类型
type IsString<T> = T extends string ? true : false;

type A = IsString<string>;  // true
type B = IsString<number>;  // false

// 条件类型与 infer
type UnwrapPromise<T> = T extends Promise<infer U> ? U : T;

type X = UnwrapPromise<Promise<string>>; // string
type Y = UnwrapPromise<number>;           // number
```

### 2.3 分发条件类型

```typescript
// 联合类型在条件类型中分发
type ToArray<T> = T extends any ? T[] : never;

type StrOrNumArray = ToArray<string | number>;
// 等价于 string[] | number[]（而非 (string | number)[]）

// 阻止分发
type ToArrayNonDist<T> = [T] extends [any] ? T[] : never;

type NonDistArray = ToArrayNonDist<string | number>;
// (string | number)[]
```

### 2.4 条件类型与函数重载

```typescript
// 根据参数类型返回不同结果
function process<T extends string | number>(
  input: T
): T extends string ? string[] : number {
  if (typeof input === 'string') {
    return input.split('') as any;
  }
  return (input * 2) as any;
}

const r1 = process('hello'); // string[]
const r2 = process(42);      // number
```

### 2.5 条件类型与模块声明

```typescript
// 根据导入方式提供不同类型
declare module 'my-lib' {
  export function create<T extends 'node' | 'browser'>(
    target: T
  ): T extends 'node' ? NodeInstance : BrowserInstance;

  interface NodeInstance {
    type: 'node';
    fs: typeof import('fs');
  }

  interface BrowserInstance {
    type: 'browser';
    window: Window;
  }
}
```

## 三、注意事项与常见陷阱

1. **条件类型在联合类型中分发**：`T extends U ? A : B` 中 T 是联合类型时会分发
2. **`infer` 只能在条件类型的 `extends` 子句中使用**
3. **嵌套条件类型可能导致性能问题**：避免过深的嵌套
4. **条件类型与类型守卫配合**：缩小运行时类型范围
5. **分发可以用 `[T] extends [U]` 阻止**
