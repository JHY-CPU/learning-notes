# 声明文件基础

## 一、概念说明

声明文件使用 `declare` 关键字声明各种 TypeScript 实体（变量、函数、类、模块等）的类型。这些声明不产生运行时代码，只提供编译时的类型信息。

## 二、具体用法

### 2.1 declare 关键字一览

```typescript
// 全局变量
declare const VERSION: string;

// 全局函数
declare function log(message: string): void;

// 全局类
declare class EventEmitter {
  on(event: string, handler: Function): void;
}

// 全局枚举
declare enum LogLevel {
  Debug = 0,
  Info = 1,
  Warn = 2,
  Error = 3,
}

// 全局接口
declare interface AppConfig {
  debug: boolean;
  version: string;
}

// 模块声明
declare module 'my-module' {
  export function doWork(): void;
}

// 命名空间
declare namespace MyLib {
  function init(): void;
  interface Options {
    verbose: boolean;
  }
}
```

### 2.2 声明合并

```typescript
// 接口声明合并
interface Window {
  myApp: { version: string };
}

// 后续声明会合并
interface Window {
  analytics: { track: (event: string) => void };
}

// Window 现在有 myApp 和 analytics 两个属性
```

### 2.3 三斜线指令

```typescript
/// <reference types="node" />
/// <reference path="./other.d.ts" />
/// <reference lib="es2022" />

// 引用类型 — 引入 @types 包
// 引用路径 — 引入其他声明文件
// 引用 lib — 引入标准库
```

### 2.4 export 与 export declare

```typescript
// 在 .d.ts 文件中
export declare function add(a: number, b: number): number;

// 等价于
export function add(a: number, b: number): number;

// 默认导出
export declare class MyService {
  doWork(): void;
}
export default MyService;
```

### 2.5 声明文件示例

```typescript
// types/mylib.d.ts
declare module 'mylib' {
  export interface Config {
    url: string;
    timeout?: number;
  }

  export function init(config: Config): void;
  export function request<T>(path: string): Promise<T>;

  export class Client {
    constructor(config: Config);
    get<T>(path: string): Promise<T>;
    post<T>(path: string, data: unknown): Promise<T>;
  }
}
```

## 三、注意事项与常见陷阱

1. **`declare` 不能有初始值或实现**：`declare const x = 1` 是错误的
2. **声明合并会导致意外的类型扩展**：注意同名接口的合并
3. **三斜线指令只在文件顶部有效**：放在其他代码之后会被忽略
4. **`.d.ts` 文件的 `export` 使其成为模块**：没有 `export` 是全局脚本
5. **`declare` 声明的类型在所有文件中可用**（全局声明文件中）
