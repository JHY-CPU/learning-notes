# 类型声明概述

## 一、概念说明

类型声明文件（`.d.ts`）是 TypeScript 中专门用于描述已有代码类型的文件。它们不包含实现代码，只包含类型信息。`.d.ts` 文件使得 TypeScript 能够理解 JavaScript 库、全局变量、模块导出等的类型。

**核心作用**：为没有类型的代码提供类型信息，让 TypeScript 编译器和 IDE 能正确理解和检查代码。

## 二、具体用法

### 2.1 声明文件 vs 普通 TS 文件

```typescript
// 普通 .ts 文件 — 包含类型和实现
function greet(name: string): string {
  return `Hello, ${name}`;
}

// .d.ts 文件 — 只包含类型声明
declare function greet(name: string): string;
// 没有函数体，没有 = { ... }
```

### 2.2 声明文件的来源

```bash
# 1. 内置声明 — TypeScript 自带
# lib.dom.d.ts — DOM API 类型
# lib.es2022.d.ts — ECMAScript 类型

# 2. DefinitelyTyped — 社区维护
npm install -D @types/node    # Node.js 类型
npm install -D @types/lodash  # Lodash 类型
npm install -D @types/express # Express 类型

# 3. 库自带 — 现代库自带 .d.ts
# 如 Vue 3、React 18+、Prisma 等

# 4. 手写 — 为没有类型的库编写
```

### 2.3 什么时候需要声明文件

```typescript
// 场景一：使用没有类型的 JavaScript 库
declare module 'legacy-lib' {
  export function doSomething(x: number): string;
}

// 场景二：全局变量（如 CDN 引入的脚本）
declare const jQuery: JQueryStatic;

// 场景三：Node.js 环境变量
declare namespace NodeJS {
  interface ProcessEnv {
    DATABASE_URL: string;
  }
}

// 场景四：CSS Modules
declare module '*.module.css' {
  const classes: Record<string, string>;
  export default classes;
}
```

### 2.4 声明文件的命名规则

```
# 文件命名
mylib.d.ts          # 与库同名
index.d.ts          # 包入口的声明
global.d.ts         # 全局类型声明
env.d.ts            # 环境变量类型

# 目录结构
@types/
  mylib/
    index.d.ts      # npm 包 @types/mylib 的结构
    package.json
```

## 三、注意事项与常见陷阱

1. **`.d.ts` 文件必须被 tsconfig 的 `include` 覆盖**
2. **声明文件不能有实现代码**：只有类型声明，没有函数体
3. **使用 `/// <reference>` 引用其他声明文件**
4. **全局声明用 `declare global`**：在模块文件中扩展全局类型
5. **不要手动编辑 `@types/` 包**：更新时会覆盖
