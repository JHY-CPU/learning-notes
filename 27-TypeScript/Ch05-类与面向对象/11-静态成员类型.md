# 静态成员类型

## 一、概念说明

静态成员（`static`）属于类本身而非实例，通过类名直接访问。静态属性和方法可以有独立的类型注解，常用于工具方法、常量定义和工厂模式。静态成员不参与类的类型兼容性检查。

## 二、具体用法

### 2.1 静态属性和方法

```typescript
class MathUtils {
  static readonly PI = 3.14159265;
  static readonly E = 2.71828182;

  static circleArea(radius: number): number {
    return MathUtils.PI * radius ** 2;
  }

  static factorial(n: number): number {
    if (n <= 1) return 1;
    return n * MathUtils.factorial(n - 1);
  }
}

console.log(MathUtils.PI);             // 输出: 3.14159265
console.log(MathUtils.circleArea(5));  // 输出: 78.53981625
console.log(MathUtils.factorial(5));   // 输出: 120
```

### 2.2 单例模式

```typescript
class Database {
  private static instance: Database;
  private connection: string = "";

  private constructor() {}

  static getInstance(): Database {
    if (!Database.instance) {
      Database.instance = new Database();
    }
    return Database.instance;
  }

  connect(url: string): void {
    this.connection = url;
    console.log(`已连接: ${url}`);
  }
}

const db1 = Database.getInstance();
const db2 = Database.getInstance();
db1.connect("postgres://localhost/mydb");
console.log(db1 === db2); // 输出: true
// 输出: 已连接: postgres://localhost/mydb
```

### 2.3 静态工厂方法

```typescript
class Color {
  private constructor(
    public readonly r: number,
    public readonly g: number,
    public readonly b: number
  ) {}

  static fromHex(hex: string): Color {
    const r = parseInt(hex.slice(1, 3), 16);
    const g = parseInt(hex.slice(3, 5), 16);
    const b = parseInt(hex.slice(5, 7), 16);
    return new Color(r, g, b);
  }

  static fromRGB(r: number, g: number, b: number): Color {
    return new Color(r, g, b);
  }

  toString(): string {
    return `rgb(${this.r}, ${this.g}, ${this.b})`;
  }
}

const red = Color.fromHex("#FF0000");
const blue = Color.fromRGB(0, 0, 255);
console.log(red.toString());  // 输出: rgb(255, 0, 0)
console.log(blue.toString()); // 输出: rgb(0, 0, 255)
```

### 2.4 静态计数器

```typescript
class UniqueId {
  private static counter = 0;

  readonly id: number;

  constructor() {
    this.id = ++UniqueId.counter;
  }

  static getCount(): number {
    return UniqueId.counter;
  }
}

const a = new UniqueId();
const b = new UniqueId();
console.log(a.id, b.id, UniqueId.getCount());
// 输出: 1 2 2
```

## 三、注意事项与常见陷阱

1. **静态成员不能用泛型**：类的类型参数 `T` 在静态上下文中不可用
2. **`this` 在静态方法中**：指向类本身，不是实例
3. **接口不能约束静态成员**：`implements` 只约束实例成员
4. **静态初始化块**（ES2022）：`static { ... }` 用于复杂初始化逻辑
5. **静态成员可被继承**：子类可以访问和重写（非 readonly）父类的静态成员
6. **静态成员参与结构类型检查**：有静态成员的类型需要结构兼容
