# 静态成员类型

## 一、概念说明

静态成员（`static`）属于类本身而非实例，通过类名直接访问。静态属性和方法可以有独立的类型注解，常用于工具方法、常量定义和工厂模式。

## 二、具体用法

### 2.1 静态属性和方法

```typescript
class MathUtils {
  static readonly PI = 3.14159265;
  static readonly E = 2.71828182;

  static circleArea(radius: number): number {
    return MathUtils.PI * radius ** 2;
  }

  static factorial(n: number): number {
    if (n <= 1) return 1;
    return n * MathUtils.factorial(n - 1);
  }
}

console.log(MathUtils.PI);             // 输出: 3.14159265
console.log(MathUtils.circleArea(5));  // 输出: 78.53981625
console.log(MathUtils.factorial(5));   // 输出: 120
```

### 2.2 单例模式

```typescript
class Database {
  private static instance: Database;
  private connection: string = "";

  private constructor() {}

  static getInstance(): Database {
    if (!Database.instance) {
      Database.instance = new Database();
    }
    return Database.instance;
  }

  connect(url: string): void {
    this.connection = url;
    console.log(`已连接: ${url}`);
  }
}

const db1 = Database.getInstance();
const db2 = Database.getInstance();
db1.connect("postgres://localhost/mydb");
console.log(db1 === db2); // 输出: true
// 输出: 已连接: postgres://localhost/mydb
```

## 三、注意事项与常见陷阱

1. **静态成员不能用泛型**：类的类型参数在静态上下文中不可用
2. **`this` 在静态方法中**：指向类本身，不是实例
3. **接口不能约束静态成员**：`implements` 只约束实例成员
4. **静态初始化块**（ES2022）：`static { ... }` 用于复杂初始化
