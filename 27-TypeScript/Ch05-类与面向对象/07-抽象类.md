# 抽象类

## 一、概念说明

抽象类（Abstract Class）使用 `abstract` 关键字声明，**不能被直接实例化**，只能作为其他类的基类。抽象类可以包含抽象方法（只有签名没有实现）和具体方法，强制子类实现特定接口。

## 二、具体用法

### 2.1 基本抽象类

```typescript
abstract class Shape {
  abstract color: string;
  abstract area(): number;

  // 具体方法
  describe(): string {
    return `这是一个 ${this.color} 的图形，面积为 ${this.area().toFixed(2)}`;
  }
}

class Circle extends Shape {
  color = "红色";
  constructor(public radius: number) {
    super();
  }

  area(): number {
    return Math.PI * this.radius ** 2;
  }
}

class Rectangle extends Shape {
  color = "蓝色";
  constructor(public width: number, public height: number) {
    super();
  }

  area(): number {
    return this.width * this.height;
  }
}

// const shape = new Shape(); // ❌ 不能实例化抽象类
const circle = new Circle(5);
console.log(circle.describe());
// 输出: 这是一个 红色 的图形，面积为 78.54
```

### 2.2 模板方法模式

```typescript
abstract class DataProcessor {
  // 模板方法：定义算法骨架
  process(data: string): string {
    const parsed = this.parse(data);
    const validated = this.validate(parsed);
    return this.format(validated);
  }

  abstract parse(data: string): unknown;
  abstract validate(data: unknown): unknown;
  abstract format(data: unknown): string;
}

class JsonProcessor extends DataProcessor {
  parse(data: string) { return JSON.parse(data); }
  validate(data: unknown) { return data; }
  format(data: unknown) { return JSON.stringify(data, null, 2); }
}

const processor = new JsonProcessor();
const result = processor.process('{"name":"Alice"}');
console.log(result);
// 输出: {\n  "name": "Alice"\n}
```

## 三、注意事项与常见陷阱

1. **不能实例化**：`new Shape()` 会编译报错
2. **子类必须实现所有抽象成员**：否则子类也必须声明为 `abstract`
3. **抽象构造函数**：可以用 `abstract new()` 约束构造函数类型
4. **抽象类可有构造函数**：子类通过 `super()` 调用
