# 类类型最佳实践

## 一、概念说明

在 TypeScript 中使用类时，遵循最佳实践可以提升代码质量和类型安全。核心原则包括：优先使用 `readonly`、合理使用访问修饰符、避免过度继承等。

## 二、具体用法

### 2.1 优先使用 readonly

```typescript
// ✅ 不可变属性更安全
class Config {
  readonly host: string;
  readonly port: number;

  constructor(host: string, port: number) {
    this.host = host;
    this.port = port;
  }
}

const config = new Config("localhost", 3000);
// config.host = "other"; // ❌ readonly 防止意外修改
```

### 2.2 组合优于继承

```typescript
// ❌ 不好：继承链过长
// class Animal -> Mammal -> Dog -> GoldenRetriever

// ✅ 好：使用组合
class Logger {
  log(msg: string): void { console.log(`[LOG] ${msg}`); }
}

class UserService {
  constructor(private logger: Logger) {}

  createUser(name: string): void {
    this.logger.log(`创建用户: ${name}`);
  }
}

const service = new UserService(new Logger());
service.createUser("Alice");
// 输出: [LOG] 创建用户: Alice
```

### 2.3 使用接口约束

```typescript
// ✅ 接口定义契约，类实现接口
interface Cache<T> {
  get(key: string): T | undefined;
  set(key: string, value: T): void;
}

class MemoryCache<T> implements Cache<T> {
  private store = new Map<string, T>();

  get(key: string): T | undefined {
    return this.store.get(key);
  }

  set(key: string, value: T): void {
    this.store.set(key, value);
  }
}

const cache = new MemoryCache<number>();
cache.set("count", 42);
console.log(cache.get("count"));
// 输出: 42
```

### 2.4 避免使用 any

```typescript
// ❌ 不好
// class Container { value: any; }

// ✅ 好：使用泛型
class Container<T> {
  constructor(public value: T) {}
  getValue(): T { return this.value; }
}

const box = new Container<string>("hello");
console.log(box.getValue());
// 输出: hello
```

## 三、注意事项与常见陷阱

1. **`readonly` 优先**：不可变属性减少 bug
2. **组合优于继承**：避免深层次继承链
3. **接口约束类行为**：用接口定义契约
4. **泛型替代 `any`**：保持类型安全
5. **最小权限原则**：默认 `private`，需要时再开放
