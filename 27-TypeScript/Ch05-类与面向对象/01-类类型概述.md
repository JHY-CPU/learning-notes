# 类类型概述

## 一、概念说明

TypeScript 的类在 JavaScript ES6 类的基础上增加了**类型注解**、**访问修饰符**、**接口实现**等特性。类定义了对象的蓝图——包含属性的类型声明和方法的签名，让面向对象编程更安全。TypeScript 类在编译后生成标准 ES6 类代码，类型信息被完全擦除。

## 二、具体用法

### 2.1 基本类定义

```typescript
class User {
  // 属性必须声明类型
  name: string;
  age: number;

  constructor(name: string, age: number) {
    this.name = name;
    this.age = age;
  }

  greet(): string {
    return `你好，我是 ${this.name}，今年 ${this.age} 岁`;
  }
}

const user = new User("张三", 25);
console.log(user.greet());
// 输出: 你好，我是 张三，今年 25 岁
```

### 2.2 属性类型声明

```typescript
class Product {
  id: number;
  name: string;
  price: number;
  inStock: boolean = true; // 默认值

  constructor(id: number, name: string, price: number) {
    this.id = id;
    this.name = name;
    this.price = price;
  }
}

const product = new Product(1, "TypeScript 书籍", 59.9);
console.log(`${product.name}: ¥${product.price}`);
// 输出: TypeScript 书籍: ¥59.9
```

### 2.3 与 JavaScript 的差异

```javascript
// JavaScript —— 无类型约束
class User {
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }
}
const user = new User("Alice", "二十五"); // 运行时不会报错
```

```typescript
// TypeScript —— 编译时检查类型
class User {
  name: string;
  age: number;
  constructor(name: string, age: number) {
    this.name = name;
    this.age = age;
  }
}
// new User("Alice", "二十五"); // ❌ 编译错误: string 不兼容 number
```

### 2.4 类作为类型和值

```typescript
// 类同时是类型和值
class Point {
  constructor(public x: number, public y: number) {}
}

// 作为值使用：创建实例
const p1 = new Point(1, 2);

// 作为类型使用：注解变量
const p2: Point = { x: 3, y: 4 }; // 结构兼容

// typeof 获取构造函数类型
type PointConstructor = typeof Point;
```

## 三、注意事项与常见陷阱

1. **属性必须初始化**：开启 `strictPropertyInitialization` 后，属性需在构造函数中赋值或有默认值
2. **类也是类型**：`const u: User = new User(...)` 中 `User` 既是值也是类型
3. **结构兼容**：TypeScript 是结构类型的，只要结构相同就兼容，不需要 `instanceof`
4. **类的实例类型**：`typeof User` 是构造函数类型，`User` 是实例类型
5. **私有成员打破结构兼容**：使用 `private` 的类只能与同源类兼容
6. **`--useDefineForClassFields`**：ES2022 标准行为，影响属性初始化顺序
