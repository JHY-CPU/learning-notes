# 类类型概述

## 一、概念说明

TypeScript 的类在 JavaScript ES6 类的基础上增加了**类型注解**、**访问修饰符**、**接口实现**等特性。类定义了对象的蓝图——包含属性的类型声明和方法的签名，让面向对象编程更安全。

## 二、具体用法

### 2.1 基本类定义

```typescript
class User {
  // 属性必须声明类型
  name: string;
  age: number;

  constructor(name: string, age: number) {
    this.name = name;
    this.age = age;
  }

  greet(): string {
    return `你好，我是 ${this.name}，今年 ${this.age} 岁`;
  }
}

const user = new User("张三", 25);
console.log(user.greet());
// 输出: 你好，我是 张三，今年 25 岁
```

### 2.2 属性类型声明

```typescript
class Product {
  id: number;
  name: string;
  price: number;
  inStock: boolean = true; // 默认值

  constructor(id: number, name: string, price: number) {
    this.id = id;
    this.name = name;
    this.price = price;
  }
}

const product = new Product(1, "TypeScript 书籍", 59.9);
console.log(`${product.name}: ¥${product.price}`);
// 输出: TypeScript 书籍: ¥59.9
```

## 三、注意事项与常见陷阱

1. **属性必须初始化**：开启 `strictPropertyInitialization` 后，属性需在构造函数中赋值或有默认值
2. **类也是类型**：`const u: User = new User(...)` 中 `User` 既是值也是类型
3. **结构兼容**：TypeScript 是结构类型的，只要结构相同就兼容
4. **类的实例类型**：`typeof User` 是构造函数类型，`User` 是实例类型
