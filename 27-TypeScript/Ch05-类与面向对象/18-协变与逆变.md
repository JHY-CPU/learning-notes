# 协变与逆变

## 一、概念说明

协变（Covariance）和逆变（Contravariance）描述了类型之间的赋值兼容关系。**协变**：子类型可以赋给父类型（如返回值类型）。**逆变**：父类型可以赋给子类型（如函数参数类型）。理解协变和逆变是深入理解 TypeScript 类型系统的关键。

## 二、具体用法

### 2.1 协变（返回值类型）

```typescript
class Animal { name = "animal"; }
class Dog extends Animal { breed = "金毛"; }

// 返回值类型是协变的
type AnimalFactory = () => Animal;
type DogFactory = () => Dog;

const getDog: DogFactory = () => new Dog();
const getAnimal: AnimalFactory = getDog; // ✅ 协变：Dog 是 Animal 的子类型

console.log(getAnimal().name); // 输出: animal
```

### 2.2 逆变（参数类型）

```typescript
// 参数类型是逆变的
type AnimalHandler = (animal: Animal) => void;
type DogHandler = (dog: Dog) => void;

const handleAnimal: AnimalHandler = (animal) => {
  console.log(animal.name);
};

const handleDog: DogHandler = handleAnimal; // ✅ 逆变：AnimalHandler 兼容 DogHandler

handleDog(new Dog());
// 输出: animal
```

### 2.3 strictFunctionTypes

```typescript
// 开启 strictFunctionTypes 后，函数参数检查更严格（逆变）
// 不开启时，参数类型是双向协变的（不够安全）

// 推荐开启 strictFunctionTypes
function process(handler: (animal: Animal) => void): void {
  handler(new Dog());
}
```

### 2.4 实际应用

```typescript
// React 事件处理中的协变
class BaseEvent { type = "base"; }
class ClickEvent extends BaseEvent { type = "click"; x = 0; y = 0; }

// ClickHandler 可以赋给 BaseHandler（参数逆变）
type BaseHandler = (event: BaseEvent) => void;
type ClickHandler = (event: ClickEvent) => void;

const handleBase: BaseHandler = (event) => {
  console.log(event.type);
};

// const handleClick: ClickHandler = handleBase; // ❌ 逆变方向
const handleBase2: BaseHandler = ((event: ClickEvent) => {
  console.log(event.x, event.y);
}) as BaseHandler; // ✅ 需要手动处理

// 数组协变
const dogs: Dog[] = [new Dog()];
const animals: Animal[] = dogs; // ✅ 数组协变（有潜在风险）
```

### 2.5 函数类型兼容规则

```typescript
// 返回值：协变（子类型兼容父类型）
type F1 = () => Dog;
type F2 = () => Animal;
const f1: F1 = () => new Dog();
const f2: F2 = f1; // ✅ Dog 兼容 Animal

// 参数：逆变（父类型兼容子类型）
type G1 = (a: Animal) => void;
type G2 = (a: Dog) => void;
const g1: G1 = (a) => console.log(a.name);
const g2: G2 = g1; // ✅ 能处理 Animal 的函数也能处理 Dog
```

## 三、注意事项与常见陷阱

1. **返回值协变**：子类方法可以返回更具体的类型
2. **参数逆变**：子类方法参数可以接受更宽泛的类型（但 TypeScript 默认双向协变）
3. **`strictFunctionTypes`**：开启后函数参数严格逆变检查，推荐开启
4. **数组协变**：`Dog[]` 可赋给 `Animal[]`（有潜在风险，写入时可能出错）
5. **方法参数不检查逆变**：类的方法参数是双向协变的（与独立函数不同）
6. **逆变的直觉理解**：能处理"动物"的函数，一定能处理"狗"（因为狗是动物）
