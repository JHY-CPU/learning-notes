# 参数属性

## 一、概念说明

参数属性是 TypeScript 的语法糖，在构造函数参数前添加访问修饰符（`public`、`private`、`protected`、`readonly`），自动将参数声明为类的同名属性并赋值，减少样板代码。这是 TypeScript 特有的语法，在实际项目中广泛使用。

## 二、具体用法

### 2.1 基本参数属性

```typescript
// 不使用参数属性
class UserOld {
  name: string;
  age: number;
  constructor(name: string, age: number) {
    this.name = name;
    this.age = age;
  }
}

// 使用参数属性（简洁得多）
class User {
  constructor(
    public name: string,
    public age: number,
    private email: string,
    readonly id: number
  ) {}
}

const user = new User("Alice", 25, "a@b.com", 1);
console.log(user.name, user.age); // 输出: Alice 25
console.log(user.id);             // 输出: 1
// user.email;                     // ❌ private
// user.id = 2;                    // ❌ readonly
```

### 2.2 配合默认值

```typescript
class Server {
  constructor(
    public host: string = "localhost",
    public port: number = 3000,
    public ssl: boolean = false
  ) {}
}

const server = new Server();
console.log(`${server.ssl ? "https" : "http"}://${server.host}:${server.port}`);
// 输出: http://localhost:3000
```

### 2.3 结合多种修饰符

```typescript
class ApiResponse<T> {
  constructor(
    public readonly status: number,
    public readonly data: T,
    private readonly _timestamp: number = Date.now()
  ) {}

  get timestamp(): Date {
    return new Date(this._timestamp);
  }
}

const response = new ApiResponse(200, { id: 1, name: "Alice" });
console.log(response.status); // 输出: 200
console.log(response.data);   // 输出: { id: 1, name: "Alice" }
// response.status = 500;     // ❌ readonly
// response._timestamp;       // ❌ private
```

### 2.4 与 JavaScript 的对比

```javascript
// JavaScript —— 需要手动赋值每个属性
class User {
  constructor(name, age, email) {
    this.name = name;
    this.age = age;
    this.email = email; // 没有私有概念
  }
}
```

```typescript
// TypeScript —— 参数属性一行搞定
class User {
  constructor(
    public name: string,
    public age: number,
    private email: string
  ) {}
}
// 自动生成 this.name = name; this.age = age; this.email = email;
```

## 三、注意事项与常见陷阱

1. **语法糖**：参数属性编译后等同于手动声明属性和赋值
2. **只能用于构造函数**：不能用于普通方法
3. **结合 readonly**：`readonly public name` 表示公共只读属性
4. **简洁但牺牲清晰**：大型类中可能难以看清属性全貌，可考虑拆分
5. **IDE 支持良好**：参数属性在 IDE 中显示为类属性，类型提示完整
6. **可选属性配合**：`public debug?: boolean` 表示可选的公共属性
