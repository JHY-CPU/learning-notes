# 访问修饰符

## 一、概念说明

TypeScript 提供三种访问修饰符控制类成员的可见性：`public`（公共，默认）、`private`（私有，仅类内部）、`protected`（受保护，类内部和子类）。

## 二、具体用法

### 2.1 public

```typescript
class User {
  public name: string; // public 是默认值，可省略

  constructor(name: string) {
    this.name = name;
  }
}

const user = new User("Alice");
console.log(user.name); // ✅ 公共属性可访问
// 输出: Alice
```

### 2.2 private

```typescript
class BankAccount {
  private balance: number;

  constructor(initialBalance: number) {
    this.balance = initialBalance;
  }

  deposit(amount: number): void {
    this.balance += amount;
  }

  getBalance(): number {
    return this.balance;
  }
}

const account = new BankAccount(1000);
account.deposit(500);
console.log(account.getBalance()); // 输出: 1500
// account.balance; // ❌ 编译错误: private
```

### 2.3 protected

```typescript
class Animal {
  protected name: string;

  constructor(name: string) {
    this.name = name;
  }
}

class Dog extends Animal {
  bark(): void {
    console.log(`${this.name} 汪汪!`); // ✅ 子类可访问 protected
  }
}

const dog = new Dog("旺财");
dog.bark();
// 输出: 旺财 汪汪!
// dog.name; // ❌ 外部不可访问 protected
```

## 三、注意事项与常见陷阱

1. **`private` 是编译时检查**：运行时仍可通过 `[key]` 访问
2. **ES2022 `#field`**：真正的运行时私有字段
3. **`protected` 子类可访问**：但不能通过实例从外部访问
4. **接口不支持修饰符**：接口的所有成员都是公共的
