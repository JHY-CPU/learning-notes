# 访问修饰符

## 一、概念说明

TypeScript 提供三种访问修饰符控制类成员的可见性：`public`（公共，默认）、`private`（私有，仅类内部）、`protected`（受保护，类内部和子类）。访问修饰符是编译时检查，不会影响运行时行为（除非使用 `#` 私有字段）。

## 二、具体用法

### 2.1 public

```typescript
class User {
  public name: string; // public 是默认值，可省略

  constructor(name: string) {
    this.name = name;
  }
}

const user = new User("Alice");
console.log(user.name); // ✅ 公共属性可访问
// 输出: Alice
```

### 2.2 private

```typescript
class BankAccount {
  private balance: number;

  constructor(initialBalance: number) {
    this.balance = initialBalance;
  }

  deposit(amount: number): void {
    this.balance += amount;
  }

  getBalance(): number {
    return this.balance;
  }
}

const account = new BankAccount(1000);
account.deposit(500);
console.log(account.getBalance()); // 输出: 1500
// account.balance; // ❌ 编译错误: private
```

### 2.3 protected

```typescript
class Animal {
  protected name: string;

  constructor(name: string) {
    this.name = name;
  }
}

class Dog extends Animal {
  bark(): void {
    console.log(`${this.name} 汪汪!`); // ✅ 子类可访问 protected
  }
}

const dog = new Dog("旺财");
dog.bark();
// 输出: 旺财 汪汪!
// dog.name; // ❌ 外部不可访问 protected
```

### 2.4 实际应用：封装

```typescript
class EventEmitter {
  private listeners: Map<string, Function[]> = new Map();

  on(event: string, handler: Function): void {
    const handlers = this.listeners.get(event) ?? [];
    handlers.push(handler);
    this.listeners.set(event, handlers);
  }

  emit(event: string, ...args: unknown[]): void {
    const handlers = this.listeners.get(event) ?? [];
    handlers.forEach(h => h(...args));
  }

  // private 方法不暴露给外部
  private validateEvent(event: string): boolean {
    return typeof event === "string" && event.length > 0;
  }
}
```

### 2.5 与 JavaScript 的对比

```javascript
// JavaScript —— 无法真正隐藏属性（约定以 _ 开头）
class Account {
  constructor(balance) {
    this._balance = balance; // 约定私有，但仍可访问
  }
}
```

```typescript
// TypeScript —— 编译时保护（更好的选择是 # 字段）
class Account {
  private balance: number; // 编译时阻止访问
  constructor(balance: number) {
    this.balance = balance;
  }
}
```

## 三、注意事项与常见陷阱

1. **`private` 是编译时检查**：运行时仍可通过 `[key]` 或类型断言访问，`#` 字段提供真正的私有
2. **ES2022 `#field`**：真正的运行时私有字段，推荐在新项目中使用
3. **`protected` 子类可访问**：但不能通过实例从外部访问
4. **接口不支持修饰符**：接口的所有成员都是公共的
5. **`private` 打破结构兼容**：有 `private` 成员的类只能与同源类实例兼容
6. **构造函数私有化**：`private constructor()` 可用于单例模式
