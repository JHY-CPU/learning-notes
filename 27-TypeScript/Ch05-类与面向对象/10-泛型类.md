# 泛型类

## 一、概念说明

泛型类在类定义时引入类型参数，使类可以存储和操作任意类型的值，同时保持类型安全。常见于集合类（Stack、Queue、Map 等）和容器类。泛型类在实例化时确定具体类型。

## 二、具体用法

### 2.1 基本泛型类

```typescript
class Stack<T> {
  private items: T[] = [];

  push(item: T): void {
    this.items.push(item);
  }

  pop(): T | undefined {
    return this.items.pop();
  }

  peek(): T | undefined {
    return this.items[this.items.length - 1];
  }

  size(): number {
    return this.items.length;
  }
}

const numStack = new Stack<number>();
numStack.push(1);
numStack.push(2);
numStack.push(3);
console.log(numStack.pop());  // 输出: 3
console.log(numStack.peek()); // 输出: 2

const strStack = new Stack<string>();
strStack.push("hello");
strStack.push("world");
console.log(strStack.pop()); // 输出: world
```

### 2.2 泛型约束

```typescript
class SortedList<T extends { compareTo(other: T): number }> {
  private items: T[] = [];

  add(item: T): void {
    this.items.push(item);
    this.items.sort((a, b) => a.compareTo(b));
  }

  getAll(): T[] {
    return [...this.items];
  }
}

class Number implements { compareTo(other: Number): number } {
  constructor(public value: number) {}
  compareTo(other: Number): number {
    return this.value - other.value;
  }
}

const list = new SortedList<Number>();
list.add(new Number(3));
list.add(new Number(1));
list.add(new Number(2));
console.log(list.getAll().map(n => n.value));
// 输出: [1, 2, 3]
```

### 2.3 多类型参数

```typescript
class Pair<K, V> {
  constructor(public key: K, public value: V) {}

  toString(): string {
    return `(${this.key}, ${this.value})`;
  }

  mapKey<T>(fn: (key: K) => T): Pair<T, V> {
    return new Pair(fn(this.key), this.value);
  }
}

const pair = new Pair<string, number>("age", 25);
console.log(pair.toString());  // 输出: (age, 25)

const mapped = pair.mapKey(k => k.toUpperCase());
console.log(mapped.toString()); // 输出: (AGE, 25)
```

### 2.4 泛型类型推断

```typescript
class Container<T> {
  constructor(public value: T) {}
}

// 从初始值推断类型
const c1 = new Container("hello");  // Container<string>
const c2 = new Container(42);       // Container<number>
const c3 = new Container({ a: 1 }); // Container<{ a: number }>

// 需要时显式指定
const c4 = new Container<string | number>("hello");
```

### 2.5 实际应用：响应包装

```typescript
class ApiResponse<T> {
  constructor(
    public data: T,
    public status: number = 200,
    public message: string = "success"
  ) {}

  map<U>(fn: (data: T) => U): ApiResponse<U> {
    return new ApiResponse(fn(this.data), this.status, this.message);
  }

  static error<T>(message: string, status: number = 500): ApiResponse<T> {
    return new ApiResponse(null as T, status, message);
  }
}

const userResponse = new ApiResponse({ id: 1, name: "Alice" });
const nameResponse = userResponse.map(u => u.name);
console.log(nameResponse.data); // 输出: Alice
```

## 三、注意事项与常见陷阱

1. **类级别泛型**：类型参数在整个类中共享，所有方法都能使用
2. **静态成员不能用泛型**：静态方法/属性不能引用类的类型参数 `T`
3. **实例化时指定类型**：`new Stack<number>()`
4. **类型推断**：`new Stack()` 从初始数据推断类型，或从使用方式推断
5. **泛型类与继承**：子类可以继承泛型类并指定具体类型或继续传递泛型参数
6. **运行时擦除**：泛型参数在编译后消失，不能用于 `typeof T` 等操作
