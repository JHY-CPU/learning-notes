# 类实现接口

## 一、概念说明

类通过 `implements` 关键字实现接口，承诺提供接口定义的所有成员。一个类可以实现多个接口，这是 TypeScript 实现多态的方式。`implements` 确保类满足接口的契约，但不强制使用继承。

## 二、具体用法

### 2.1 单接口实现

```typescript
interface Printable {
  print(): void;
}

interface Loggable {
  log(message: string): void;
}

class ConsolePrinter implements Printable, Loggable {
  print(): void {
    console.log("打印内容");
  }

  log(message: string): void {
    console.log(`[LOG] ${message}`);
  }
}

const printer = new ConsolePrinter();
printer.print();
printer.log("操作完成");
// 输出:
// 打印内容
// [LOG] 操作完成
```

### 2.2 泛型接口实现

```typescript
interface Repository<T> {
  findById(id: number): T | undefined;
  findAll(): T[];
  save(entity: T): void;
}

interface User {
  id: number;
  name: string;
}

class UserRepository implements Repository<User> {
  private users: User[] = [];

  findById(id: number): User | undefined {
    return this.users.find((u) => u.id === id);
  }

  findAll(): User[] {
    return [...this.users];
  }

  save(user: User): void {
    this.users.push(user);
  }
}

const repo = new UserRepository();
repo.save({ id: 1, name: "Alice" });
console.log(repo.findById(1));
// 输出: { id: 1, name: "Alice" }
```

### 2.3 接口作为依赖

```typescript
// 面向接口编程
interface Logger {
  info(message: string): void;
  error(message: string): void;
}

class ConsoleLogger implements Logger {
  info(msg: string) { console.log(`[INFO] ${msg}`); }
  error(msg: string) { console.error(`[ERROR] ${msg}`); }
}

class FileLogger implements Logger {
  info(msg: string) { /* 写入文件 */ }
  error(msg: string) { /* 写入文件 */ }
}

// 依赖接口而非具体实现
class App {
  constructor(private logger: Logger) {}

  run() {
    this.logger.info("应用启动");
  }
}

// 可以注入不同实现
const app = new App(new ConsoleLogger());
app.run(); // 输出: [INFO] 应用启动
```

### 2.4 接口约束类的形状

```typescript
interface Timestamped {
  createdAt: Date;
  updatedAt: Date;
}

// 不需要显式 implements，结构兼容即可
class LogEntry {
  constructor(
    public message: string,
    public createdAt: Date = new Date(),
    public updatedAt: Date = new Date()
  ) {}
}

function printTimestamped(item: Timestamped) {
  console.log(`创建于: ${item.createdAt.toISOString()}`);
}

printTimestamped(new LogEntry("操作记录")); // ✅ 结构兼容
```

## 三、注意事项与常见陷阱

1. **必须实现所有成员**：遗漏会编译报错
2. **接口只约束实例成员**：不约束静态成员和构造函数
3. **多接口实现**：用逗号分隔，如 `implements A, B`
4. **结构兼容即可**：不需要显式 `implements`，结构匹配就兼容（这是 TypeScript 的特性）
5. **`implements` 不产生运行时代码**：纯粹是编译时检查
6. **接口继承 vs 类实现**：接口用 `extends` 继承，类用 `implements` 实现
