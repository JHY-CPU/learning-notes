# 装饰器

## 一、概念说明

装饰器（Decorator）是一种特殊声明，使用 `@` 语法附加到类、方法、属性或参数上，用于修改或增强其行为。TypeScript 5.0+ 支持 ECMAScript 标准装饰器。

## 二、具体用法

### 2.1 类装饰器

```typescript
function sealed(constructor: Function) {
  Object.seal(constructor);
  Object.seal(constructor.prototype);
  console.log(`${constructor.name} 已被密封`);
}

@sealed
class MyClass {
  name = "MyClass";
}

const obj = new MyClass();
console.log(obj.name);
// 输出:
// MyClass 已被密封
// MyClass
```

### 2.2 方法装饰器

```typescript
function log(
  target: any,
  context: ClassMethodDecoratorContext
) {
  const methodName = String(context.name);
  return function (this: any, ...args: any[]) {
    console.log(`调用 ${methodName}(${args.join(", ")})`);
    const result = target.apply(this, args);
    console.log(`${methodName} 返回: ${result}`);
    return result;
  };
}

class Calculator {
  @log
  add(a: number, b: number): number {
    return a + b;
  }
}

const calc = new Calculator();
calc.add(3, 5);
// 输出:
// 调用 add(3, 5)
// add 返回: 8
```

### 2.3 自动绑定装饰器

```typescript
function autoBind(
  target: any,
  context: ClassMethodDecoratorContext
) {
  return function (this: any) {
    return target.bind(this);
  };
}

class Handler {
  message = "Hello";

  @autoBound
  greet() {
    return this.message;
  }
}
```

## 三、注意事项与常见陷阱

1. **TS 5.0+ 新装饰器**：与旧版实验装饰器语法不同
2. **`experimentalDecorators`**：旧项目可能仍在使用旧语法
3. **执行顺序**：装饰器从上到下执行，方法装饰器的返回值替换原方法
4. **装饰器不能用于函数声明**：只能用于类成员和类本身
