# 类的类型注解

## 一、概念说明

类中的属性和方法都需要明确的类型注解。属性声明类型、构造函数参数类型、方法参数和返回值类型共同构成了类的完整类型契约。正确的类型注解让类的使用者获得准确的类型提示。

## 二、具体用法

### 2.1 属性和方法类型

```typescript
class Calculator {
  result: number = 0;

  add(value: number): this {
    this.result += value;
    return this;
  }

  subtract(value: number): this {
    this.result -= value;
    return this;
  }

  getResult(): number {
    return this.result;
  }
}

const calc = new Calculator();
const result = calc.add(10).subtract(3).add(5).getResult();
console.log(result);
// 输出: 12
```

### 2.2 可选属性

```typescript
class Config {
  host: string;
  port: number;
  debug?: boolean;

  constructor(host: string, port: number, debug?: boolean) {
    this.host = host;
    this.port = port;
    this.debug = debug;
  }
}

const config = new Config("localhost", 3000);
console.log(config.debug); // 输出: undefined
```

### 2.3 只读属性

```typescript
class User {
  readonly id: number;
  name: string;

  constructor(id: number, name: string) {
    this.id = id;
    this.name = name;
  }
}

const user = new User(1, "Alice");
user.name = "Bob";     // ✅
// user.id = 2;        // ❌ 编译错误: readonly

console.log(user.name);
// 输出: Bob
```

### 2.4 方法重载

```typescript
class Formatter {
  format(value: string): string;
  format(value: number): string;
  format(value: string | number): string {
    if (typeof value === "string") {
      return value.trim().toUpperCase();
    }
    return value.toFixed(2);
  }
}

const fmt = new Formatter();
console.log(fmt.format("  hello ")); // 输出: HELLO
console.log(fmt.format(3.14159));    // 输出: 3.14
```

### 2.5 this 返回类型

```typescript
class Builder {
  private value = "";

  append(str: string): this {
    this.value += str;
    return this;
  }

  build(): string {
    return this.value;
  }
}

const result = new Builder()
  .append("Hello")
  .append(", ")
  .append("World")
  .build();
console.log(result); // 输出: Hello, World
```

## 三、注意事项与常见陷阱

1. **`this` 返回类型**：方法返回 `this` 支持链式调用，子类也能正确继承链式调用
2. **`strictPropertyInitialization`**：强制属性必须初始化，防止遗漏赋值
3. **`readonly` vs `const`**：`readonly` 用于属性，`const` 用于变量
4. **类型守卫与类**：`instanceof` 可以收窄类类型，是类场景最常用的类型保护
5. **方法类型推断**：TypeScript 通常能推断返回类型，但复杂方法建议显式标注
6. **`override` 关键字**：TypeScript 4.3+ 支持 `override` 显式标记重写方法
