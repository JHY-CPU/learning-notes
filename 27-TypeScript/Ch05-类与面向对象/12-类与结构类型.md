# 类与结构类型

## 一、概念说明

TypeScript 的类型系统是**结构化**的（Structural），类的兼容性取决于结构而非名称。这意味着不需要显式 `implements`，只要对象的结构与类实例兼容，就可以互相赋值。

## 二、具体用法

### 2.1 结构兼容

```typescript
class Cat {
  name: string;
  meow(): void { console.log("喵"); }
  constructor(name: string) { this.name = name; }
}

// 不继承 Cat，但结构兼容
const duck = {
  name: "鸭子",
  meow() { console.log("嘎嘎"); },
};

const cat: Cat = duck; // ✅ 结构兼容
cat.meow();
// 输出: 嘎嘎
```

### 2.2 接口兼容类

```typescript
class User {
  constructor(public name: string, public age: number) {}
}

interface HasName {
  name: string;
}

function greet(entity: HasName): void {
  console.log(`你好，${entity.name}`);
}

const user = new User("Alice", 25);
greet(user); // ✅ User 结构兼容 HasName
// 输出: 你好，Alice

// 普通对象也兼容
greet({ name: "Bob" });
// 输出: 你好，Bob
```

### 2.3 私有成员的影响

```typescript
class Safe {
  private secret: string = "hidden";
}

class Unsafe {
  private secret: string = "exposed";
}

// 有 private 成员时，必须是同一类的实例
const safe = new Safe();
// const unsafe: Safe = new Unsafe(); // ❌ private 成员来源不同

console.log("有 private 成员时需同一声明来源");
// 输出: 有 private 成员时需同一声明来源
```

## 三、注意事项与常见陷阱

1. **鸭子类型**：只要"看起来像"就可以互换
2. **`private` 打破结构兼容**：有 `private`/`protected` 时要求同一声明
3. **多余属性检查**：字面量赋值时严格检查，变量赋值时不检查
4. **有利有弊**：灵活但可能导致意外兼容
