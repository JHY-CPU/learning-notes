# 类的继承

## 一、概念说明

类通过 `extends` 继承其他类，获得父类的属性和方法。子类可以**重写**（Override）父类方法、添加新成员。TypeScript 还支持继承的类型兼容性检查。继承是代码复用和多态的基础。

## 二、具体用法

### 2.1 基本继承

```typescript
class Animal {
  constructor(public name: string) {}

  speak(): string {
    return `${this.name} 发出声音`;
  }
}

class Dog extends Animal {
  constructor(name: string, public breed: string) {
    super(name);
  }

  // 重写父类方法
  speak(): string {
    return `${this.name} 汪汪叫`;
  }

  fetch(): string {
    return `${this.name} 捡球`;
  }
}

const dog = new Dog("旺财", "金毛");
console.log(dog.speak());  // 输出: 旺财 汪汪叫
console.log(dog.fetch());  // 输出: 旺财 捡球
```

### 2.2 super 调用

```typescript
class Vehicle {
  constructor(public brand: string) {}

  describe(): string {
    return `品牌: ${this.brand}`;
  }
}

class Car extends Vehicle {
  constructor(brand: string, public doors: number) {
    super(brand);
  }

  describe(): string {
    return `${super.describe()}, ${this.doors}门`;
  }
}

const car = new Car("Toyota", 4);
console.log(car.describe());
// 输出: 品牌: Toyota, 4门
```

### 2.3 继承链类型兼容

```typescript
function makeAnimalSpeak(animal: Animal): void {
  console.log(animal.speak());
}

makeAnimalSpeak(new Animal("动物"));  // 输出: 动物 发出声音
makeAnimalSpeak(new Dog("旺财", "金毛")); // 输出: 旺财 汪汪叫
// 子类兼容父类类型
```

### 2.4 抽象类

```typescript
abstract class Shape {
  abstract area(): number;  // 子类必须实现

  describe(): string {
    return `面积: ${this.area().toFixed(2)}`;
  }
}

class Circle extends Shape {
  constructor(private radius: number) {
    super();
  }

  area(): number {
    return Math.PI * this.radius ** 2;
  }
}

class Rectangle extends Shape {
  constructor(private width: number, private height: number) {
    super();
  }

  area(): number {
    return this.width * this.height;
  }
}

// const s = new Shape(); // ❌ 抽象类不能实例化
const shapes: Shape[] = [new Circle(5), new Rectangle(4, 6)];
shapes.forEach(s => console.log(s.describe()));
// 输出:
// 面积: 78.54
// 面积: 24.00
```

### 2.5 override 关键字

```typescript
// TypeScript 4.3+ 支持 override
class Base {
  greet() { return "Hello"; }
}

class Derived extends Base {
  override greet() { return "Hi"; } // ✅ 显式标记重写
  // override foo() {} // ❌ 编译错误: 父类没有 foo 方法
}
```

## 三、注意事项与常见陷阱

1. **必须调用 `super()`**：子类构造函数必须先调用 `super`，且在访问 `this` 之前
2. **重写方法签名要兼容**：参数和返回类型应兼容父类（协变返回值，逆变参数）
3. **多继承不支持**：类只能继承一个父类，但可实现多个接口
4. **类型兼容**：子类实例可以赋值给父类类型（里氏替换原则）
5. **抽象类**：不能实例化，只能被继承，可包含抽象方法和具体方法
6. **`override` 关键字**：推荐使用，防止意外重写或遗漏重写
