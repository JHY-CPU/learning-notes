# 装饰器工厂

## 一、概念说明

装饰器工厂是返回装饰器的函数，允许向装饰器传递参数。通过工厂模式，可以创建可配置的装饰器，根据参数调整装饰行为。

## 二、具体用法

### 2.1 带参数的装饰器工厂

```typescript
// 装饰器工厂：返回装饰器函数
function log(prefix: string) {
  return function (
    target: any,
    context: ClassMethodDecoratorContext
  ) {
    return function (this: any, ...args: any[]) {
      console.log(`${prefix} 调用 ${String(context.name)}`);
      return (target as Function).apply(this, args);
    };
  };
}

class Service {
  @log("[INFO]")
  fetch(): string {
    return "数据";
  }

  @log("[DEBUG]")
  process(): void {
    // 处理逻辑
  }
}

const service = new Service();
service.fetch();
// 输出: [INFO] 调用 fetch
```

### 2.2 属性验证装饰器

```typescript
function min(minValue: number) {
  return function (value: any, context: ClassFieldDecoratorContext) {
    return function (initialValue: number) {
      if (initialValue < minValue) {
        throw new Error(`${String(context.name)} 必须 >= ${minValue}`);
      }
      return initialValue;
    };
  };
}

class Product {
  @min(0)
  price = 99.9;

  @min(1)
  quantity = 10;
}

const product = new Product();
console.log(product.price, product.quantity);
// 输出: 99.9 10
```

## 三、注意事项与常见陷阱

1. **三层函数嵌套**：工厂返回装饰器，装饰器返回替换函数
2. **参数在工厂调用时确定**：`@log("[INFO]")` 中的参数
3. **装饰器工厂可链式调用**：`@a @b class C {}`
4. **元数据传递**：可以配合 `Symbol.metadata` 传递元信息
