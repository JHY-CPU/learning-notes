# 数组类型

## 一、概念说明

TypeScript 中数组有两种等价的类型注解语法：`T[]` 和 `Array<T>`。数组是同质的——所有元素应为同一类型。对于固定长度、不同类型元素的集合，应使用元组类型。TypeScript 的数组类型系统提供了编译时的元素类型安全。

## 二、具体用法

### 2.1 数组类型注解

```typescript
// 两种等价写法
let nums1: number[] = [1, 2, 3];
let nums2: Array<number> = [1, 2, 3];

// 字符串数组
let names: string[] = ["Alice", "Bob", "Charlie"];

// 多维数组
let matrix: number[][] = [[1, 2], [3, 4]];
let cube: number[][][] = [[[1]]];

console.log(nums1, names);
// 输出: [1, 2, 3] ["Alice", "Bob", "Charlie"]
console.log(matrix[0][1]);
// 输出: 2
```

### 2.2 只读数组

```typescript
// readonly 数组不能修改
const readonlyArr: readonly number[] = [1, 2, 3];
// readonlyArr.push(4); // ❌ 编译错误
// readonlyArr[0] = 10; // ❌ 编译错误

// ReadonlyArray 等价写法
const arr2: ReadonlyArray<string> = ["a", "b"];

console.log(readonlyArr);
// 输出: [1, 2, 3]
```

### 2.3 常用数组方法类型

```typescript
const items: number[] = [3, 1, 4, 1, 5];

// 推断返回类型
const sorted = items.sort((a, b) => a - b); // number[]
const found = items.find(x => x > 3);       // number | undefined
const mapped = items.map(x => String(x));   // string[]
const filtered = items.filter(x => x > 2);  // number[]

console.log(sorted);   // 输出: [1, 1, 3, 4, 5]
console.log(found);    // 输出: 4
console.log(mapped);   // 输出: ["3", "1", "4", "1", "5"]
console.log(filtered); // 输出: [3, 4, 5]
```

### 2.4 与 JavaScript 的对比

```javascript
// JavaScript —— 没有元素类型约束
const mixed = [1, "two", true, null]; // 运行时才可能出错
```

```typescript
// TypeScript —— 编译时约束元素类型
const numbers: number[] = [1, 2, 3];
// numbers.push("hello"); // ❌ 编译错误: string 不兼容 number

// 需要混合类型时使用联合类型或元组
const mixed: (number | string)[] = [1, "two", 3];
const tuple: [number, string, boolean] = [1, "two", true];
```

### 2.5 空数组的类型推断

```typescript
// 空数组推断为 never[]
const empty = [];
// empty.push(1); // ❌ 编译错误: number 不兼容 never

// 解决方案：显式注解类型
const nums: number[] = [];
nums.push(1); // ✅
nums.push(2); // ✅
```

## 三、注意事项与常见陷阱

1. **`T[]` vs `Array<T>`**：功能相同，团队统一一种风格即可；泛型参数中建议用 `Array<T>`
2. **`readonly` 防止修改**：需要不可变数组时使用 `readonly T[]`，编译时阻止 `push`、`pop` 等操作
3. **空数组推断为 `never[]`**：`const arr = []` 需要显式注解类型
4. **数组方法的返回类型**：`find` 返回 `T | undefined`，`pop` 也是 `T | undefined`
5. **`filter` 的类型收窄**：使用类型谓词 `is` 可以让 `filter` 返回更精确的类型
6. **`as const` 数组**：使用 `as const` 后数组变为只读元组，类型最精确
