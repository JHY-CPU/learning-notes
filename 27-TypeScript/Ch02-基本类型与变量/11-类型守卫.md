# 类型守卫

## 一、概念说明

类型守卫（Type Guard）是在运行时检查类型的表达式，帮助 TypeScript 在特定代码分支中将类型收窄（Narrowing）为更具体的类型。主要有 `typeof`、`instanceof`、`in` 操作符和自定义类型谓词。

## 二、具体用法

### 2.1 typeof 守卫

```typescript
function format(value: string | number): string {
  if (typeof value === "string") {
    return value.toUpperCase(); // 收窄为 string
  }
  return value.toFixed(2); // 收窄为 number
}

console.log(format("hello")); // 输出: HELLO
console.log(format(3.14159)); // 输出: 3.14
```

### 2.2 instanceof 守卫

```typescript
function handleError(error: string | Error): void {
  if (error instanceof Error) {
    console.log(`错误堆栈: ${error.stack}`);
  } else {
    console.log(`错误消息: ${error}`);
  }
}

handleError(new Error("数据库连接失败"));
// 输出: 错误堆栈: Error: 数据库连接失败...
handleError("网络超时");
// 输出: 错误消息: 网络超时
```

### 2.3 in 守卫

```typescript
interface Fish {
  swim(): void;
}

interface Bird {
  fly(): void;
}

function move(animal: Fish | Bird): void {
  if ("swim" in animal) {
    animal.swim(); // 收窄为 Fish
  } else {
    animal.fly(); // 收窄为 Bird
  }
}

const fish: Fish = { swim: () => console.log("游泳中") };
move(fish);
// 输出: 游泳中
```

### 2.4 自定义类型守卫（类型谓词）

```typescript
interface Cat {
  meow(): void;
}

interface Dog {
  bark(): void;
}

function isCat(animal: Cat | Dog): animal is Cat {
  return "meow" in animal;
}

function interact(animal: Cat | Dog): void {
  if (isCat(animal)) {
    animal.meow(); // 收窄为 Cat
  } else {
    animal.bark(); // 收窄为 Dog
  }
}

const cat: Cat = { meow: () => console.log("喵~") };
interact(cat);
// 输出: 喵~
```

## 三、注意事项与常见陷阱

1. **`typeof` 只能区分原始类型**：`typeof []` 是 `"object"`，`typeof null` 也是 `"object"`
2. **`instanceof` 用于类实例**：不能用于检查普通对象或接口
3. **`in` 检查属性存在**：包括原型链上的属性
4. **类型谓词的实现必须准确**：返回 `false` 时不一定是另一种类型
