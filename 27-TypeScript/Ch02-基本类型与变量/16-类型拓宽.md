# 类型拓宽

## 一、概念说明

类型拓宽（Type Widening）是指 TypeScript 在变量赋值时将字面量类型自动提升为更宽泛的基础类型的过程。`let` 变量会触发拓宽，`const` 变量则保持字面量类型。理解拓宽有助于避免意外的类型推断，尤其是在需要精确字面量类型的场景中。

## 二、具体用法

### 2.1 let vs const 拓宽

```typescript
// let 导致类型拓宽
let str = "hello";      // 拓宽为 string
let num = 42;           // 拓宽为 number
let flag = true;        // 拓宽为 boolean

// const 保持字面量类型
const constStr = "hello"; // 类型: "hello"
const constNum = 42;      // 类型: 42

// 验证
const a: "hello" = constStr; // ✅
// const b: "hello" = str;   // ❌ string 不兼容 "hello"
```

### 2.2 对象属性拓宽

```typescript
// 对象属性使用 let 时会拓宽
const obj = {
  name: "Alice",  // 拓宽为 string（因为对象是 const 但属性不是）
};

obj.name = "Bob"; // ✅ string 允许赋任何字符串

// 使用 as const 阻止拓宽
const frozenObj = {
  name: "Alice",
} as const;

// frozenObj.name = "Bob"; // ❌ 只读
console.log(frozenObj.name);
// 输出: Alice
```

### 2.3 const 断言阻止拓宽

```typescript
// 没有 as const：拓宽为 string[]
const arr1 = ["red", "green", "blue"];

// 使用 as const：推断为元组
const arr2 = ["red", "green", "blue"] as const;
// 类型: readonly ["red", "green", "blue"]

function useColor(color: "red" | "green" | "blue") {
  console.log(color);
}

// useColor(arr1[0]); // ❌ string 不兼容
useColor(arr2[0]); // ✅ "red" 兼容
// 输出: red
```

### 2.4 函数参数的拓宽

```typescript
// 函数参数默认允许类型拓宽（以便灵活性）
function greet(name: string) {
  console.log(`Hello, ${name}`);
}

let userName = "Alice"; // 拓宽为 string
greet(userName); // ✅

// 但如果参数声明为字面量类型
function setAlignment(align: "left" | "center" | "right") {
  console.log(`对齐方式: ${align}`);
}

let alignment = "left"; // 拓宽为 string
// setAlignment(alignment); // ❌ string 不兼容 "left" | "center" | "right"

// 解决方案：使用 const
const alignConst = "left"; // 类型: "left"
setAlignment(alignConst); // ✅
```

### 2.5 最佳实践

```typescript
// ✅ 使用 as const 保持精确类型
const ROUTES = {
  HOME: "/",
  ABOUT: "/about",
  CONTACT: "/contact",
} as const;

type Route = (typeof ROUTES)[keyof typeof ROUTES];
// "/" | "/about" | "/contact"

// ✅ 配置对象使用 as const
const DEFAULT_CONFIG = {
  timeout: 5000,
  retries: 3,
  baseUrl: "https://api.example.com",
} as const;

function initConfig(config: Partial<typeof DEFAULT_CONFIG>) {
  console.log(config);
}
```

## 三、注意事项与常见陷阱

1. **`const` 不完全阻止拓宽**：对象的属性值仍会拓宽（属性可以被重新赋值）
2. **`as const` 是终极方案**：深度阻止所有层级的拓宽，使整个值变为 `readonly`
3. **拓宽是合理的默认行为**：大多数情况下拓宽是期望的行为，只在需要精确类型时阻止
4. **注意数组推断**：`let arr = []` 推断为 `never[]`，需显式注解 `let arr: number[] = []`
5. **拓宽与可赋值性**：拓宽后的类型是更宽泛的类型，子类型可以赋值给父类型
6. **`let` 与联合类型**：`let x: "a" | "b" = "a"` 不会拓宽，因为有显式类型注解
