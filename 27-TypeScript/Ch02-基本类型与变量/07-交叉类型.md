# 交叉类型

## 一、概念说明

交叉类型（Intersection Type）用 `&` 将多个类型合并为一个类型。结果类型同时具有所有参与类型的属性。常用于组合多个接口或类型别名。交叉类型在功能上类似于 `interface` 的 `extends`，但可以用于任何类型，不限于对象类型。

## 二、具体用法

### 2.1 基本交叉类型

```typescript
interface HasName {
  name: string;
}

interface HasAge {
  age: number;
}

// 合并两个接口
type Person = HasName & HasAge;

const person: Person = {
  name: "张三",
  age: 25,
};

console.log(person);
// 输出: { name: "张三", age: 25 }
```

### 2.2 交叉类型与函数

```typescript
interface Loggable {
  log(): void;
}

interface Serializable {
  toJSON(): string;
}

// 对象必须同时实现 log 和 toJSON
function process(obj: Loggable & Serializable): void {
  obj.log();
  console.log(obj.toJSON());
}

const logger: Loggable & Serializable = {
  log() { console.log("记录日志"); },
  toJSON() { return '{"logged": true}'; },
};

process(logger);
// 输出:
// 记录日志
// {"logged": true}
```

### 2.3 基本类型的交叉

```typescript
// 基本类型交叉结果为 never
type Impossible = string & number; // never

// 对象类型交叉会合并属性
type Combined = { a: string } & { b: number };
// 等价于 { a: string; b: number }

const obj: Combined = { a: "hello", b: 42 };
console.log(obj);
// 输出: { a: "hello", b: 42 }
```

### 2.4 在实际项目中的应用

```typescript
// API 响应类型组合
interface ApiResponse<T> {
  data: T;
  status: number;
  message: string;
}

interface Pagination {
  page: number;
  pageSize: number;
  total: number;
}

// 分页 API 响应
type PaginatedResponse<T> = ApiResponse<T> & Pagination;

// 使用
const response: PaginatedResponse<{ id: number; name: string }[]> = {
  data: [{ id: 1, name: "Alice" }],
  status: 200,
  message: "success",
  page: 1,
  pageSize: 10,
  total: 100,
};
```

### 2.5 交叉类型与联合类型配合

```typescript
// 混合使用 & 和 |
type BaseProps = {
  id: string;
  className?: string;
};

type ButtonProps = BaseProps & (
  | { variant: "primary"; onClick: () => void }
  | { variant: "link"; href: string }
);

const btn1: ButtonProps = {
  id: "btn1",
  variant: "primary",
  onClick: () => console.log("点击"),
};

const btn2: ButtonProps = {
  id: "btn2",
  variant: "link",
  href: "https://example.com",
};
```

## 三、注意事项与常见陷阱

1. **同名属性交叉**：如果两个类型有同名但类型不同的属性，交叉后为 `never`
2. **与联合类型的区别**：`&` 是"且"关系（必须同时满足），`|` 是"或"关系（满足其一）
3. **基本类型交叉无意义**：`string & number` 为 `never`，因为不存在既是字符串又是数字的值
4. **深层合并**：交叉只合并顶层属性，不会递归合并嵌套对象
5. **性能影响**：复杂交叉类型（超过 50 个属性）可能影响 IDE 的类型检查性能
6. **优先用 interface extends**：纯对象合并时，`interface extends` 更易读
