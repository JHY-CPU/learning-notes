# 交叉类型

## 一、概念说明

交叉类型（Intersection Type）用 `&` 将多个类型合并为一个类型。结果类型同时具有所有参与类型的属性。常用于组合多个接口或类型别名。

## 二、具体用法

### 2.1 基本交叉类型

```typescript
interface HasName {
  name: string;
}

interface HasAge {
  age: number;
}

// 合并两个接口
type Person = HasName & HasAge;

const person: Person = {
  name: "张三",
  age: 25,
};

console.log(person);
// 输出: { name: "张三", age: 25 }
```

### 2.2 交叉类型与函数

```typescript
interface Loggable {
  log(): void;
}

interface Serializable {
  toJSON(): string;
}

// 对象必须同时实现 log 和 toJSON
function process(obj: Loggable & Serializable): void {
  obj.log();
  console.log(obj.toJSON());
}

const logger: Loggable & Serializable = {
  log() { console.log("记录日志"); },
  toJSON() { return '{"logged": true}'; },
};

process(logger);
// 输出:
// 记录日志
// {"logged": true}
```

### 2.3 基本类型的交叉

```typescript
// 基本类型交叉结果为 never
type Impossible = string & number; // never

// 对象类型交叉会合并属性
type Combined = { a: string } & { b: number };
// 等价于 { a: string; b: number }

const obj: Combined = { a: "hello", b: 42 };
console.log(obj);
// 输出: { a: "hello", b: 42 }
```

## 三、注意事项与常见陷阱

1. **同名属性交叉**：如果两个类型有同名但类型不同的属性，交叉后为 `never`
2. **与联合类型的区别**：`&` 是"且"关系，`|` 是"或"关系
3. **基本类型交叉无意义**：`string & number` 为 `never`
4. **深层合并**：交叉只合并顶层属性，不会递归合并嵌套对象
