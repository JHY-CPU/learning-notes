# 对象类型

## 一、概念说明

对象类型用于描述具有特定属性结构的对象。可以定义属性的名称、类型、是否可选、是否只读。对象类型可以匿名定义，也可以用 `interface` 或 `type` 命名。

## 二、具体用法

### 2.1 匿名对象类型

```typescript
// 直接在参数中定义对象类型
function printUser(user: { name: string; age: number }): void {
  console.log(`${user.name}, ${user.age}岁`);
}

printUser({ name: "李四", age: 30 });
// 输出: 李四, 30岁
```

### 2.2 可选属性

```typescript
interface Config {
  host: string;
  port: number;
  debug?: boolean; // 可选属性
}

const config1: Config = { host: "localhost", port: 3000 };                    // ✅
const config2: Config = { host: "localhost", port: 3000, debug: true };      // ✅

console.log(config1.debug); // 输出: undefined
console.log(config2.debug); // 输出: true
```

### 2.3 只读属性

```typescript
interface User {
  readonly id: number;
  name: string;
}

const user: User = { id: 1, name: "Alice" };
user.name = "Bob";    // ✅ 可修改
// user.id = 2;       // ❌ 编译错误: 只读属性不可修改

console.log(user);
// 输出: { id: 1, name: "Bob" }
```

### 2.4 索引签名

```typescript
// 允许任意数量的字符串键
interface StringMap {
  [key: string]: string;
}

const headers: StringMap = {
  "Content-Type": "application/json",
  "Authorization": "Bearer token123",
};

console.log(headers["Content-Type"]);
// 输出: application/json
```

## 三、注意事项与常见陷阱

1. **可选属性访问**：使用前应检查 `undefined` 或用可选链 `?.`
2. **`readonly` 只防修改**：不防嵌套对象内部修改，需深冻结
3. **对象字面量赋值**：字面量会检查多余属性，变量赋值不会
4. **索引签名返回类型**：所有属性必须兼容索引签名的值类型
