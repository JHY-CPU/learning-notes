# 对象类型

## 一、概念说明

对象类型用于描述具有特定属性结构的对象。可以定义属性的名称、类型、是否可选、是否只读。对象类型可以匿名定义，也可以用 `interface` 或 `type` 命名。TypeScript 的对象类型系统基于结构类型（鸭子类型），只关心对象有什么属性，不关心它的类名或原型。

## 二、具体用法

### 2.1 匿名对象类型

```typescript
// 直接在参数中定义对象类型
function printUser(user: { name: string; age: number }): void {
  console.log(`${user.name}, ${user.age}岁`);
}

printUser({ name: "李四", age: 30 });
// 输出: 李四, 30岁
```

### 2.2 可选属性

```typescript
interface Config {
  host: string;
  port: number;
  debug?: boolean; // 可选属性
}

const config1: Config = { host: "localhost", port: 3000 };                    // ✅
const config2: Config = { host: "localhost", port: 3000, debug: true };      // ✅

console.log(config1.debug); // 输出: undefined
console.log(config2.debug); // 输出: true
```

### 2.3 只读属性

```typescript
interface User {
  readonly id: number;
  name: string;
}

const user: User = { id: 1, name: "Alice" };
user.name = "Bob";    // ✅ 可修改
// user.id = 2;       // ❌ 编译错误: 只读属性不可修改

console.log(user);
// 输出: { id: 1, name: "Bob" }
```

### 2.4 索引签名

```typescript
// 允许任意数量的字符串键
interface StringMap {
  [key: string]: string;
}

const headers: StringMap = {
  "Content-Type": "application/json",
  "Authorization": "Bearer token123",
};

console.log(headers["Content-Type"]);
// 输出: application/json
```

### 2.5 与 JavaScript 的对比

```javascript
// JavaScript —— 对象属性无类型约束
function createUser(name, age) {
  return { name, age };
}
const user = createUser("Alice", "三十"); // 运行时才可能发现问题
```

```typescript
// TypeScript —— 编译时约束属性类型
function createUser(name: string, age: number) {
  return { name, age };
}
// createUser("Alice", "三十"); // ❌ 编译错误: string 不兼容 number
createUser("Alice", 30);       // ✅
```

### 2.6 对象解构与类型

```typescript
interface ApiResponse {
  status: number;
  data: { id: number; name: string }[];
  message?: string;
}

function handleResponse({ status, data, message }: ApiResponse): void {
  if (status === 200) {
    console.log(`获取到 ${data.length} 条数据`);
  } else {
    console.log(`错误: ${message ?? "未知错误"}`);
  }
}
```

## 三、注意事项与常见陷阱

1. **可选属性访问**：使用前应检查 `undefined` 或用可选链 `?.`
2. **`readonly` 只防修改**：不防嵌套对象内部修改，需要深冻结时使用 `as const`
3. **对象字面量赋值**：字面量会检查多余属性（Excess Property Check），变量赋值不会
4. **索引签名返回类型**：所有属性必须兼容索引签名的值类型
5. **`in` 操作符收窄**：`if ("name" in obj)` 可以收窄对象类型
6. **`Object.keys` 返回 `string[]`**：不会返回联合字面量类型，使用时需注意
