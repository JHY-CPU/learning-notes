# 模板字面量类型

## 一、概念说明

模板字面量类型（Template Literal Types）是 TypeScript 4.1 引入的特性，允许使用模板字符串语法创建字符串类型。可以结合联合类型生成所有可能的字符串组合，实现类型级别的字符串操作。

## 二、具体用法

### 2.1 基本模板字面量类型

```typescript
type Greeting = `Hello, ${string}!`;

const greet1: Greeting = "Hello, World!";   // ✅
const greet2: Greeting = "Hello, Alice!";   // ✅
// const greet3: Greeting = "Hi, Bob!";     // ❌ 不匹配模式

console.log(greet1);
// 输出: Hello, World!
```

### 2.2 联合类型展开

```typescript
type Color = "red" | "green" | "blue";
type Size = "sm" | "md" | "lg";

// 生成所有组合：9 种可能
type ClassName = `${Color}-${Size}`;
// "red-sm" | "red-md" | "red-lg" | "green-sm" | ...共 9 种

const cls: ClassName = "red-md";
console.log(cls);
// 输出: red-md
```

### 2.3 内置字符串工具类型

```typescript
// TypeScript 内置的模板工具类型
type Upper = Uppercase<"hello">;         // "HELLO"
type Lower = Lowercase<"HELLO">;         // "hello"
type Cap = Capitalize<"hello">;          // "Hello"
type Uncap = Uncapitalize<"Hello">;      // "hello"

// 实际应用：事件处理器命名
type EventName = "click" | "focus" | "blur";
type HandlerName = `on${Capitalize<EventName>}`;
// "onClick" | "onFocus" | "onBlur"

const handler: HandlerName = "onClick";
console.log(handler);
// 输出: onClick
```

### 2.4 类型推断提取

```typescript
type ExtractRoute<T extends string> =
  T extends `/${infer Rest}` ? Rest : never;

type Path = ExtractRoute<"/api/users">;  // "api/users"
type Path2 = ExtractRoute<"api/users">;  // never

const path: Path = "api/users";
console.log(path);
// 输出: api/users
```

## 三、注意事项与常见陷阱

1. **联合类型会"笛卡尔展开"**：`A | B` 与 `C | D` 组合生成 4 种
2. **类型推断结合**：配合 `infer` 可以提取模板字符串的部分
3. **性能注意**：过度展开联合类型可能导致类型计算复杂度爆炸
4. **运行时不可用**：模板字面量类型只存在于编译时
