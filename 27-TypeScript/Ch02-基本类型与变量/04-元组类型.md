# 元组类型

## 一、概念说明

元组（Tuple）是**固定长度**、**每个位置有确定类型**的数组。与普通数组不同，元组中每个元素可以是不同类型。使用方括号语法定义，如 `[string, number, boolean]`。元组常用于函数返回多个不同类型的值，或表示具有固定结构的数据。

## 二、具体用法

### 2.1 基本元组

```typescript
// 固定长度，每个位置类型确定
let person: [string, number] = ["张三", 25];

const name = person[0]; // string
const age = person[1];  // number

console.log(`${name} 今年 ${age} 岁`);
// 输出: 张三 今年 25 岁

// 解构元组
const [userName, userAge] = person;
console.log(userName, userAge);
// 输出: 张三 25
```

### 2.2 可选元组元素

```typescript
// 可选元素放在最后
type NameAge = [string, number, string?];

const user1: NameAge = ["Alice", 30];          // ✅
const user2: NameAge = ["Bob", 25, "Engineer"]; // ✅

console.log(user1, user2);
// 输出: ["Alice", 30] ["Bob", 25, "Engineer"]
```

### 2.3 剩余元素元组

```typescript
// Rest 元素：不定长部分
type StringNumberBooleans = [string, number, ...boolean[]];

const data: StringNumberBooleans = ["hello", 42, true, false, true];
console.log(data);
// 输出: ["hello", 42, true, false, true]

// 带标签的元组（提升可读性）
type Labeled = [name: string, age: number, ...hobbies: string[]];
const user: Labeled = ["张三", 28, "阅读", "编程"];
console.log(user);
// 输出: ["张三", 28, "阅读", "编程"]
```

### 2.4 元组在函数返回中的应用

```typescript
// 函数返回多个不同类型的值
function divide(a: number, b: number): [number, number] {
  const quotient = Math.floor(a / b);
  const remainder = a % b;
  return [quotient, remainder];
}

const [q, r] = divide(17, 5);
console.log(`商: ${q}, 余数: ${r}`);
// 输出: 商: 3, 余数: 2

// React useState 的简化版本
function useState<T>(initial: T): [T, (value: T) => void] {
  let state = initial;
  const setState = (value: T) => { state = value; };
  return [state, setState];
}

const [count, setCount] = useState(0);
```

### 2.5 与 JavaScript 的对比

```javascript
// JavaScript —— 没有元组概念，用数组代替
const user = ["Alice", 30, "admin"];
// 无法在编译时约束长度和类型
user.push(true);     // 不报错
const age = user[5]; // 运行时为 undefined
```

```typescript
// TypeScript —— 编译时检查长度和类型
const user: [string, number, string] = ["Alice", 30, "admin"];
// user.push(true); // ❌ 编译错误
// user[5];         // ❌ 越界访问编译错误
```

## 三、注意事项与常见陷阱

1. **越界访问**：访问越界索引会报编译错误（但 `push` 在某些版本中允许添加越界元素）
2. **赋值兼容性**：`number[]` 不能赋值给 `[number, number]`，固定长度与可变长度不兼容
3. **解构赋值类型**：解构后的变量会自动推断为对应位置的类型
4. **`push`/`pop` 限制**：向固定长度元组 push 超出定义的类型会报错
5. **只读元组**：使用 `readonly [string, number]` 防止修改元组内容
6. **比较用 as const**：`as const` 推断的只读元组类型最精确，常用于配置对象
