# 类型系统概述

## 一、概念说明

TypeScript 的类型系统是**结构化的**、**静态的**、**渐进式的**。它在编译时检查类型错误，而非运行时。类型系统的核心目标是：在代码运行之前发现潜在的错误，同时提供良好的开发体验。

### 1.1 类型层次结构

```
        any / unknown
       /     |      \
    string  number  boolean  ...
       \     |      /
        never (底层类型)
```

- **`any` / `unknown`**：最顶层类型，可赋值给所有类型
- **`never`**：最底层类型，是所有类型的子类型
- **具体类型**：如 `string`、`number` 等

## 二、具体用法

### 2.1 静态类型检查

```typescript
// 编译时捕获错误
function greet(name: string): string {
  return `Hello, ${name}`;
}

greet("TypeScript"); // ✅
// greet(123);       // ❌ 编译错误: number 不能赋给 string
```

### 2.2 类型层次应用

```typescript
// never 是所有类型的子类型
function assertNever(x: never): never {
  throw new Error(`Unexpected: ${x}`);
}

type Shape = "circle" | "square";

function area(s: Shape): number {
  switch (s) {
    case "circle": return Math.PI * 100;
    case "square": return 100;
    default: return assertNever(s as never);
  }
}

console.log(area("circle"));
// 输出: 314.1592653589793
```

### 2.3 结构类型系统

```typescript
// TypeScript 只看结构，不看名称
interface Point2D {
  x: number;
  y: number;
}

interface Point3D {
  x: number;
  y: number;
  z: number;
}

const point3d: Point3D = { x: 1, y: 2, z: 3 };
const point2d: Point2D = point3d; // ✅ Point3D 包含 Point2D 的所有属性

console.log(point2d); // 输出: { x: 1, y: 2, z: 3 }
```

### 2.4 类型推断

```typescript
// TypeScript 能自动推断大多数类型
let message = "Hello";        // 推断为 string
let count = 42;               // 推断为 number
let items = [1, 2, 3];       // 推断为 number[]

// 函数返回类型推断
function add(a: number, b: number) {
  return a + b; // 推断返回类型为 number
}

// 上下文推断
const names = ["Alice", "Bob"];
names.forEach((name) => {
  // name 被推断为 string
  console.log(name.toUpperCase());
});
```

## 三、注意事项与常见陷阱

1. **类型在编译后消失**：运行时没有类型信息，`instanceof` 是运行时检查
2. **结构类型兼容**：只要结构相同，不同名称的类型互相兼容
3. **类型推断减少注解**：不必处处写类型，但显式注解提升可读性
4. **`any` 破坏类型安全**：滥用 `any` 等于放弃类型系统，应优先使用 `unknown`
5. **类型不是性能工具**：TypeScript 的类型系统与运行时性能无关
6. **渐进式采用**：可以从 `strict: false` 开始，逐步收紧类型检查
