# 类型最佳实践

## 一、概念说明

在 TypeScript 项目中遵循类型最佳实践，可以在获得最大类型安全的同时保持代码可读性和可维护性。核心原则是：避免 `any`、优先使用 `interface` 定义对象、善用类型推断。

## 二、具体用法

### 2.1 避免 any，优先 unknown

```typescript
// ❌ 不好：使用 any 丧失类型安全
function processDataBad(data: any): any {
  return data.value;
}

// ✅ 好：使用 unknown + 类型守卫
function processDataGood(data: unknown): string {
  if (typeof data === "object" && data !== null && "value" in data) {
    return String((data as { value: unknown }).value);
  }
  return "无数据";
}

console.log(processDataGood({ value: 42 }));
// 输出: 42
```

### 2.2 优先 interface 定义对象

```typescript
// ✅ 推荐：用 interface 定义对象形状
interface User {
  id: number;
  name: string;
}

// ✅ 用 type 定义联合类型、函数类型等
type ID = string | number;
type Callback = (result: string) => void;
type Result<T> = { data: T } | { error: string };

const user: User = { id: 1, name: "Alice" };
const id: ID = "abc";
console.log(user, id);
// 输出: { id: 1, name: "Alice" } abc
```

### 2.3 善用类型推断

```typescript
// ❌ 不必要：返回类型可以推断
function add(a: number, b: number): number {
  return a + b;
}

// ✅ 简洁：让编译器推断返回类型
function addClean(a: number, b: number) {
  return a + b;
}

// ✅ 公共 API 显式标注：提升可读性和文档性
export function publicApi(input: string): { result: string } {
  return { result: input.toUpperCase() };
}
```

### 2.4 严格模式配置

```json
// tsconfig.json - 推荐的严格配置
{
  "compilerOptions": {
    "strict": true,
    "noUncheckedIndexedAccess": true,
    "exactOptionalPropertyTypes": true
  }
}
```

## 三、注意事项与常见陷阱

1. **`any` 是最后手段**：只有在确实无法确定类型时才使用
2. **接口优于类型别名**（对象场景）：接口支持声明合并和 `extends`
3. **导出函数应显式标注返回类型**：避免重构时返回类型意外变化
4. **`noUncheckedIndexedAccess`**：索引访问可能返回 `undefined`，强制检查
5. **不要过度注解**：让编译器推断简单类型，复杂类型才显式标注
