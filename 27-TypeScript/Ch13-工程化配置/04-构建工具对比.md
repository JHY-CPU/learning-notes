# 构建工具对比

## 一、概念说明

TypeScript 的构建工具有多种选择，从官方的 `tsc` 到高性能的 `esbuild`、`swc`、`sucrase`。选择取决于项目需求：是否需要类型检查、构建速度要求、配置复杂度等。

## 二、具体用法

### 2.1 tsc（官方编译器）

```bash
# 完整编译（类型检查 + 代码转换）
npx tsc

# 优点：完全兼容 TS 特性
# 缺点：速度慢，不适合大型项目实时编译
```

### 2.2 esbuild（推荐）

```bash
npm install -D esbuild

# 基本编译
npx esbuild src/index.ts --bundle --outfile=dist/index.js --platform=node

# 构建脚本
```

```typescript
// build.mjs
import * as esbuild from 'esbuild';

await esbuild.build({
  entryPoints: ['src/index.ts'],
  bundle: true,
  minify: true,
  sourcemap: true,
  target: ['es2020'],
  outdir: 'dist',
  format: 'esm',
});
```

### 2.3 swc

```bash
npm install -D @swc/core @swc/cli

# 编译
npx swc src -d dist
```

```json
// .swcrc
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "jsc": {
    "parser": { "syntax": "typescript" },
    "target": "es2020"
  },
  "module": { "type": "es6" }
}
```

### 2.4 sucrase

```typescript
// 仅类型擦除，不做类型检查
import { transform } from 'sucrase';

const result = transform(code, {
  transforms: ['typescript', 'jsx'],
});
```

### 2.5 工具对比

| 工具 | 速度 | 类型检查 | Bundle | 配置 |
|------|------|---------|--------|------|
| tsc | 慢 | 是 | 否 | tsconfig |
| esbuild | 极快 | 否 | 是 | API/CLI |
| swc | 极快 | 否 | 否 | .swcrc |
| sucrase | 快 | 否 | 否 | API |
| Vite | 快 | 是 | 是 | vite.config |

### 2.6 推荐方案

```bash
# 应用开发
# 开发: Vite (esbuild) + vue-tsc / tsc --noEmit
# 生产: Vite build

# 库开发
# 开发: tsx watch
# 生产: tsc (类型声明) + esbuild (编译)
```

## 三、注意事项与常见陷阱

1. **esbuild 不支持所有 TS 特性**：如 `const enum`、`namespace`
2. **esbuild 不生成 `.d.ts`**：需要 tsc 单独生成
3. **swc 可以替代 Babel**：用于 JSX/装饰器等转换
4. **开发用 esbuild，类型检查用 tsc**：两者并行
5. **Vite 底层使用 esbuild**：所以有相同的限制
