# 库打包配置

## 一、概念说明

发布 npm 库时需要同时输出 ESM 和 CJS 两种格式，并生成类型声明文件。正确的打包配置确保库在不同环境下都能正确使用。

## 二、具体用法

### 2.1 双格式输出配置

```typescript
// rollup.config.mjs
import typescript from '@rollup/plugin-typescript';
import dts from 'rollup-plugin-dts';

export default [
  {
    input: 'src/index.ts',
    output: [
      { file: 'dist/index.cjs', format: 'cjs', sourcemap: true },
      { file: 'dist/index.mjs', format: 'esm', sourcemap: true },
    ],
    plugins: [typescript()],
    external: [/node_modules/],
  },
  {
    input: 'dist/types/index.d.ts',
    output: [{ file: 'dist/index.d.ts', format: 'esm' }],
    plugins: [dts()],
  },
];
```

### 2.2 package.json 完整配置

```json
{
  "name": "my-library",
  "version": "1.0.0",
  "type": "module",
  "main": "./dist/index.cjs",
  "module": "./dist/index.mjs",
  "types": "./dist/index.d.ts",
  "exports": {
    ".": {
      "types": "./dist/index.d.ts",
      "import": "./dist/index.mjs",
      "require": "./dist/index.cjs"
    }
  },
  "files": ["dist"],
  "sideEffects": false,
  "scripts": {
    "build": "rollup -c",
    "type-check": "tsc --noEmit",
    "prepublishOnly": "npm run type-check && npm run build"
  },
  "peerDependencies": {
    "react": ">=18.0.0"
  },
  "devDependencies": {
    "typescript": "^5.4.0",
    "rollup": "^4.0.0",
    "@rollup/plugin-typescript": "^11.0.0"
  }
}
```

### 2.3 多入口导出

```json
{
  "exports": {
    ".": {
      "types": "./dist/index.d.ts",
      "import": "./dist/index.mjs",
      "require": "./dist/index.cjs"
    },
    "./utils": {
      "types": "./dist/utils.d.ts",
      "import": "./dist/utils.mjs",
      "require": "./dist/utils.cjs"
    }
  }
}
```

## 三、注意事项与常见陷阱

1. **`types` 条件必须放在 `exports` 最前面**
2. **`files` 只包含 `dist`**：不发布源码
3. **`peerDependencies` 不打包**：由使用者提供
4. **`sideEffects: false` 支持 tree-shaking**
5. **发布前检查**：`npm pack --dry-run` 查看包含的文件
