# TS工程化概述

## 一、概念说明

TypeScript 工程化涉及项目结构、构建工具、代码规范、测试配置等多个方面。合理的工程化配置能显著提升开发效率、代码质量和团队协作体验。

## 二、具体用法

### 2.1 推荐项目结构

```
project/
├── src/                    # 源码
│   ├── components/         # 组件
│   ├── utils/              # 工具函数
│   ├── types/              # 类型定义
│   ├── services/           # 服务层
│   └── index.ts            # 入口
├── tests/                  # 测试
├── dist/                   # 编译输出
├── .eslintrc.cjs           # ESLint 配置
├── .prettierrc             # Prettier 配置
├── tsconfig.json           # TS 主配置
├── tsconfig.node.json      # Node 工具配置
├── vite.config.ts          # Vite 配置
├── package.json
└── README.md
```

### 2.2 核心配置文件

```json
// tsconfig.json
{
  "compilerOptions": {
    "strict": true,
    "target": "ES2022",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "jsx": "react-jsx",
    "noEmit": true,
    "isolatedModules": true,
    "skipLibCheck": true
  },
  "include": ["src"],
  "references": [{ "path": "./tsconfig.node.json" }]
}
```

### 2.3 构建工具选择

| 工具 | 速度 | 功能 | 适用场景 |
|------|------|------|----------|
| tsc | 慢 | 类型检查 + 编译 | 库开发 |
| Vite | 快 | 开发服务器 + 构建 | 应用开发 |
| esbuild | 极快 | 只编译，无类型检查 | 生产构建 |
| swc | 极快 | 类型擦除编译 | Babel 替代 |

### 2.4 代码规范工具链

```bash
# ESLint + TypeScript
npm install -D eslint @typescript-eslint/parser @typescript-eslint/eslint-plugin

# Prettier
npm install -D prettier eslint-config-prettier
```

### 2.5 CI/CD 集成

```yaml
# .github/workflows/ci.yml
name: CI
on: [push, pull_request]
jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
      - run: npm ci
      - run: npm run type-check  # tsc --noEmit
      - run: npm run lint        # eslint
      - run: npm run test        # vitest
      - run: npm run build       # 构建
```

## 三、注意事项与常见陷阱

1. **`strict: true` 从项目开始就开启**：后期开启成本极高
2. **`noEmit: true` 用于只做类型检查的项目**：如 React/Vue 应用
3. **分离 tsconfig**：应用代码和工具代码分开配置
4. **CI 中必须包含类型检查步骤**：`tsc --noEmit`
5. **版本锁定**：TypeScript 版本应精确锁定，避免类型不一致
