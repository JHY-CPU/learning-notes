# 环境变量类型

## 一、概念说明

环境变量在 TypeScript 中需要类型声明才能获得类型安全。Vite 使用 `import.meta.env`，Node.js 使用 `process.env`。通过 `.d.ts` 文件声明环境变量的类型，可以避免拼写错误、获得自动补全，并确保必填变量在编译时检查。

## 二、具体用法

### 2.1 Vite 环境变量

```bash
# .env（所有环境）
VITE_API_URL=https://api.example.com
VITE_APP_TITLE=我的应用

# .env.development（开发环境覆盖）
VITE_API_URL=http://localhost:8080
VITE_DEBUG=true

# .env.production（生产环境覆盖）
VITE_API_URL=https://api.prod.example.com
```

```typescript
// src/vite-env.d.ts
/// <reference types="vite/client" />

interface ImportMetaEnv {
  readonly VITE_API_URL: string;
  readonly VITE_APP_TITLE: string;
  readonly VITE_DEBUG: string;
  readonly DEV: boolean;
  readonly PROD: boolean;
  readonly MODE: "development" | "production" | "test";
}

interface ImportMeta {
  readonly env: ImportMetaEnv;
}
```

```typescript
// 使用（类型安全）
const apiUrl = import.meta.env.VITE_API_URL; // string
const isDev = import.meta.env.DEV;           // boolean
// import.meta.env.VITE_UNKNOWN; // 编译错误
```

### 2.2 Node.js 环境变量

```typescript
// src/types/env.d.ts
declare namespace NodeJS {
  interface ProcessEnv {
    NODE_ENV: "development" | "production" | "test";
    PORT: string;
    DATABASE_URL: string;
    JWT_SECRET: string;
    API_KEY?: string; // 可选
  }
}

export {};
```

```typescript
// 使用
const port = parseInt(process.env.PORT, 10); // string -> number
const dbUrl: string = process.env.DATABASE_URL;
const env: "development" | "production" | "test" = process.env.NODE_ENV;
```

### 2.3 强制必填环境变量

```typescript
// src/config/env.ts
function requireEnv(key: keyof NodeJS.ProcessEnv): string {
  const value = process.env[key];
  if (value === undefined) {
    throw new Error(`缺少环境变量: ${key}`);
  }
  return value;
}

const dbUrl = requireEnv("DATABASE_URL"); // string（非 undefined）
const jwtSecret = requireEnv("JWT_SECRET");
```

### 2.4 类型安全的配置对象

```typescript
// src/config/index.ts
interface AppConfig {
  apiUrl: string;
  debug: boolean;
  port: number;
  database: {
    url: string;
    poolSize: number;
  };
}

function loadConfig(): AppConfig {
  return {
    apiUrl: requireEnv("API_URL"),
    debug: process.env.NODE_ENV === "development",
    port: parseInt(process.env.PORT || "3000", 10),
    database: {
      url: requireEnv("DATABASE_URL"),
      poolSize: parseInt(process.env.DB_POOL_SIZE || "10", 10),
    },
  };
}

// 启动时加载，之后所有代码使用 config 对象（类型安全）
export const config = loadConfig();
```

### 2.5 环境变量验证（Zod）

```typescript
import { z } from "zod";

const envSchema = z.object({
  DATABASE_URL: z.string().url(),
  PORT: z.string().default("3000"),
  NODE_ENV: z.enum(["development", "production", "test"]),
  JWT_SECRET: z.string().min(32),
  API_KEY: z.string().optional(),
});

// 启动时验证所有环境变量
const env = envSchema.parse(process.env);

// env.DATABASE_URL: string (validated URL)
// env.PORT: string (default "3000")
// env.NODE_ENV: "development" | "production" | "test"
```

### 2.6 与 JavaScript 的对比

```javascript
// JavaScript：无类型，运行时才发现问题
const dbUrl = process.env.DATABASE_URL; // string | undefined
dbUrl.connect(); // 运行时 TypeError

// TypeScript：编译时类型检查
const dbUrl: string = process.env.DATABASE_URL;
// 如果 strictNullChecks 开启，提示可能是 undefined
// 引导开发者正确处理
```

## 三、注意事项与常见陷阱

1. **`VITE_` 前缀必需**：Vite 只暴露 `VITE_` 开头的环境变量给客户端
2. **环境变量类型始终是 `string`**：布尔值和数字需要手动转换（`parseInt`、`=== "true"`）
3. **`.env` 文件不要提交**：加入 `.gitignore`，提供 `.env.example` 作为模板
4. **`/// <reference types="vite/client" />`**：引入 Vite 内置类型
5. **`readonly` 防止修改**：环境变量不应在运行时被修改
6. **启动时验证**：使用 Zod 等库在应用启动时验证所有必要环境变量
