# 环境变量类型

## 一、概念说明

Vite 使用 `import.meta.env` 读取环境变量，需要在 `vite-env.d.ts` 中定义类型。`.env` 文件中以 `VITE_` 开头的变量才会暴露给客户端代码。

## 二、具体用法

### 2.1 环境变量文件

```bash
# .env（所有环境）
VITE_API_URL=https://api.example.com
VITE_APP_TITLE=我的应用

# .env.development（开发环境）
VITE_API_URL=http://localhost:8080
VITE_DEBUG=true

# .env.production（生产环境）
VITE_API_URL=https://api.prod.example.com
VITE_DEBUG=false
```

### 2.2 类型声明

```typescript
// src/vite-env.d.ts
/// <reference types="vite/client" />

interface ImportMetaEnv {
  readonly VITE_API_URL: string;
  readonly VITE_APP_TITLE: string;
  readonly VITE_DEBUG: string;
  readonly DEV: boolean;
  readonly PROD: boolean;
  readonly MODE: string;
}

interface ImportMeta {
  readonly env: ImportMetaEnv;
}
```

### 2.3 使用

```typescript
// 类型安全的环境变量
const apiUrl = import.meta.env.VITE_API_URL; // string
const isDev = import.meta.env.DEV;           // boolean

if (import.meta.env.PROD) {
  console.log('生产环境');
}
```

### 2.4 Node.js 环境变量

```typescript
// src/types/env.d.ts
declare namespace NodeJS {
  interface ProcessEnv {
    NODE_ENV: 'development' | 'production' | 'test';
    PORT: string;
    DATABASE_URL: string;
  }
}
```

## 三、注意事项与常见陷阱

1. **`VITE_` 前缀是必需的**：否则不会暴露给客户端
2. **环境变量类型始终是 `string`**：布尔值需要手动转换
3. **`.env` 文件不要提交到版本控制**：加入 `.gitignore`
4. **`/// <reference types="vite/client" />`**：引入 Vite 内置类型
5. **类型声明中的 `readonly`**：防止意外修改
