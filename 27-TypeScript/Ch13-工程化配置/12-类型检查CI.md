# 类型检查CI

## 一、概念说明

在 CI/CD 流程中加入 TypeScript 类型检查步骤，确保提交的代码没有类型错误。`tsc --noEmit` 只做类型检查不生成文件，是 CI 中的常用命令。

## 二、具体用法

### 2.1 CI 类型检查脚本

```json
{
  "scripts": {
    "type-check": "tsc --noEmit",
    "type-check:watch": "tsc --noEmit --watch"
  }
}
```

### 2.2 GitHub Actions 配置

```yaml
# .github/workflows/ci.yml
name: CI
on: [push, pull_request]

jobs:
  type-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: 'npm'

      - run: npm ci
      - run: npm run type-check

  lint:
    runs-on: ubuntu-latest
    needs: type-check
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: 'npm'

      - run: npm ci
      - run: npm run lint

  test:
    runs-on: ubuntu-latest
    needs: type-check
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: 'npm'

      - run: npm ci
      - run: npm run test

  build:
    runs-on: ubuntu-latest
    needs: [type-check, lint, test]
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: 'npm'

      - run: npm ci
      - run: npm run build
```

### 2.3 Monorepo CI

```yaml
jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v2
        with:
          version: 9
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: 'pnpm'

      - run: pnpm install --frozen-lockfile
      - run: pnpm turbo type-check
      - run: pnpm turbo lint
      - run: pnpm turbo test
      - run: pnpm turbo build
```

### 2.4 检查所有 tsconfig

```bash
# 检查所有引用的项目
npx tsc --build --noEmit
```

## 三、注意事项与常见陷阱

1. **CI 中使用 `--noEmit`**：不生成文件，只做类型检查
2. **类型检查应放在 CI 第一步**：类型错误比 lint 错误更严重
3. **`--build` 模式适合 monorepo**：按依赖顺序检查
4. **缓存 node_modules**：加快 CI 速度
5. **并行执行 lint 和 test**：它们互不依赖
