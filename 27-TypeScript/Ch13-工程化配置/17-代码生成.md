# 代码生成

## 一、概念说明

代码生成工具可以根据 API 定义（OpenAPI、GraphQL）自动生成 TypeScript 类型和客户端代码，确保前后端类型一致。这是实现端到端类型安全的重要手段。

## 二、具体用法

### 2.1 OpenAPI 类型生成

```bash
# openapi-typescript
npm install -D openapi-typescript

# 从 OpenAPI spec 生成类型
npx openapi-typescript api-spec.yaml -o src/types/api.d.ts
```

```typescript
// 生成的类型
export interface paths {
  '/users': {
    get: {
      responses: {
        200: {
          content: {
            'application/json': components['schemas']['User'][];
          };
        };
      };
    };
    post: {
      requestBody: {
        content: {
          'application/json': components['schemas']['CreateUserDto'];
        };
      };
      responses: {
        201: {
          content: {
            'application/json': components['schemas']['User'];
          };
        };
      };
    };
  };
}
```

### 2.2 GraphQL 代码生成

```bash
npm install -D @graphql-codegen/cli @graphql-codegen/typescript
```

```yaml
# codegen.yml
schema: 'src/graphql/schema.graphql'
generates:
  src/graphql/generated.ts:
    plugins:
      - typescript
      - typescript-operations
    config:
      withHooks: true
```

```typescript
// 生成的类型和 hooks
export type User = {
  id: number;
  name: string;
  email: string;
};

export type GetUserQuery = {
  user: User;
};

export function useGetUserQuery(options?: UseQueryOptions<GetUserQuery>) {
  // 自动生成的 hook
}
```

### 2.3 Zod 类型推断

```typescript
import { z } from 'zod';

// Schema 定义
const userSchema = z.object({
  name: z.string().min(2),
  email: z.string().email(),
  age: z.number().min(0).max(150),
});

// 自动推断类型
type User = z.infer<typeof userSchema>;
// { name: string; email: string; age: number }
```

### 2.4 Package.json 脚本

```json
{
  "scripts": {
    "generate:api": "openapi-typescript api-spec.yaml -o src/types/api.d.ts",
    "generate:graphql": "graphql-codegen --config codegen.yml",
    "generate": "npm run generate:api && npm run generate:graphql",
    "prebuild": "npm run generate"
  }
}
```

## 三、注意事项与常见陷阱

1. **生成的类型文件不要手动编辑**：每次生成会覆盖
2. **API spec 要保持更新**：过时的 spec 生成过时的类型
3. **生成的文件加入 `.gitignore` 或提交**：取决于项目策略
4. **CI 中重新生成类型**：确保类型与 API 一致
5. **Zod schema 可以同时用于验证和类型推断**
