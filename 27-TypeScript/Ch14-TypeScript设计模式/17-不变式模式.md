# 不变式模式

## 一、概念说明

不变式模式确保对象创建后不能被修改。TypeScript 通过 `readonly` 修饰符和品牌类型（Branded Types）实现编译时的不可变性和约束。

## 二、具体用法

### 2.1 readonly 修饰符

```typescript
// 只读接口
interface Config {
  readonly apiUrl: string;
  readonly timeout: number;
  readonly retries: number;
}

const config: Config = {
  apiUrl: 'https://api.example.com',
  timeout: 5000,
  retries: 3,
};

// config.apiUrl = 'new'; // 编译错误

// Readonly 工具类型
interface MutableUser {
  name: string;
  age: number;
}

const user: Readonly<MutableUser> = { name: '张三', age: 25 };
// user.name = '李四'; // 编译错误
```

### 2.2 深度只读

```typescript
type DeepReadonly<T> = {
  readonly [K in keyof T]: T[K] extends object
    ? T[K] extends Function
      ? T[K]
      : DeepReadonly<T[K]>
    : T[K];
};

interface State {
  users: { id: number; name: string }[];
  settings: { theme: string };
}

type ReadonlyState = DeepReadonly<State>;
// 所有属性都是只读的，包括嵌套对象
```

### 2.3 品牌类型（Branded Types）

```typescript
// 品牌类型 — 区分相同基础类型的不同语义
type Brand<T, B> = T & { __brand: B };

type PositiveNumber = Brand<number, 'Positive'>;
type NonEmptyString = Brand<string, 'NonEmpty'>;
type Email = Brand<string, 'Email'>;

// 构造函数
function positive(n: number): PositiveNumber {
  if (n <= 0) throw new Error('必须为正数');
  return n as PositiveNumber;
}

function nonEmpty(s: string): NonEmptyString {
  if (s.length === 0) throw new Error('不能为空');
  return s as NonEmptyString;
}

function email(s: string): Email {
  if (!s.includes('@')) throw new Error('无效的邮箱');
  return s as Email;
}

// 使用 — 类型不同，不能混用
function calculateDiscount(price: PositiveNumber): PositiveNumber {
  return positive(price * 0.9);
}

calculateDiscount(positive(100)); // OK
// calculateDiscount(100); // 编译错误：number 不能赋给 PositiveNumber
```

### 2.4 不可变数据操作

```typescript
// 使用展开运算符创建新对象
function updateUser(user: Readonly<User>, updates: Partial<User>): User {
  return { ...user, ...updates };
}

// 不可变数组操作
function addItem<T>(arr: readonly T[], item: T): T[] {
  return [...arr, item];
}

function removeItem<T>(arr: readonly T[], index: number): T[] {
  return arr.filter((_, i) => i !== index);
}
```

## 三、注意事项与常见陷阱

1. **`readonly` 只在编译时有效**：运行时仍可修改
2. **品牌类型需要构造函数**：确保值经过验证
3. **`Object.freeze` 提供运行时不可变**：但只冻结一层
4. **`as const` 创建只读字面量**：整个对象及其属性都是 `readonly`
5. **不可变操作创建新对象**：不修改原对象
