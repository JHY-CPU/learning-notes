# 单例模式

## 一、概念说明

单例模式确保一个类只有一个实例，并提供全局访问点。TypeScript 通过类型系统可以确保单例的类型安全，避免重复创建实例。

## 二、具体用法

### 2.1 基本单例

```typescript
class Database {
  private static instance: Database;
  private connection: string;

  private constructor(connectionString: string) {
    this.connection = connectionString;
  }

  // 类型安全的静态方法
  static getInstance(connectionString?: string): Database {
    if (!Database.instance) {
      if (!connectionString) {
        throw new Error('首次创建需要提供连接字符串');
      }
      Database.instance = new Database(connectionString);
    }
    return Database.instance;
  }

  query<T>(sql: string): Promise<T[]> {
    console.log(`执行: ${sql}`);
    return Promise.resolve([]);
  }
}

// 使用
const db1 = Database.getInstance('postgres://localhost/mydb');
const db2 = Database.getInstance(); // 返回同一实例
// db1 === db2 → true
```

### 2.2 泛型单例

```typescript
// 泛型单例管理器
class SingletonManager {
  private static instances = new Map<string, unknown>();

  static get<T>(key: string, factory: () => T): T {
    if (!this.instances.has(key)) {
      this.instances.set(key, factory());
    }
    return this.instances.get(key) as T;
  }
}

// 使用
interface Config {
  apiUrl: string;
  debug: boolean;
}

const config = SingletonManager.get<Config>('config', () => ({
  apiUrl: 'https://api.example.com',
  debug: true,
}));
```

### 2.3 使用 Symbol 实现

```typescript
const INSTANCE = Symbol('instance');

class Logger {
  private static [INSTANCE]: Logger;

  private constructor() {}

  static getInstance(): Logger {
    if (!Logger[INSTANCE]) {
      Logger[INSTANCE] = new Logger();
    }
    return Logger[INSTANCE];
  }

  log(message: string): void {
    console.log(`[LOG] ${message}`);
  }
}
```

## 三、注意事项与常见陷阱

1. **构造函数必须是 `private`**：防止外部 `new`
2. **单例难以测试**：考虑使用依赖注入替代
3. **多线程环境需要考虑同步**：Node.js 是单线程的
4. **单例会引入全局状态**：不利于代码解耦
5. **用 `Map` 管理多个单例实例**：比多个类更灵活
