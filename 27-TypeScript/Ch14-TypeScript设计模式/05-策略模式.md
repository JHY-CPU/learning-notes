# 策略模式

## 一、概念说明

策略模式定义一系列算法，将每个算法封装起来，并使它们可互换。TypeScript 通过接口和泛型确保所有策略遵循相同的契约。

## 二、具体用法

### 2.1 基本策略模式

```typescript
// 策略接口
interface PaymentStrategy {
  pay(amount: number): boolean;
  name: string;
}

// 具体策略
class CreditCardPayment implements PaymentStrategy {
  name = '信用卡';

  constructor(private cardNumber: string) {}

  pay(amount: number): boolean {
    console.log(`使用信用卡 ${this.cardNumber} 支付 ${amount} 元`);
    return true;
  }
}

class AlipayPayment implements PaymentStrategy {
  name = '支付宝';

  constructor(private account: string) {}

  pay(amount: number): boolean {
    console.log(`使用支付宝 ${this.account} 支付 ${amount} 元`);
    return true;
  }
}

class WechatPayment implements PaymentStrategy {
  name = '微信';

  pay(amount: number): boolean {
    console.log(`使用微信支付 ${amount} 元`);
    return true;
  }
}

// 上下文
class PaymentContext {
  private strategy: PaymentStrategy;

  constructor(strategy: PaymentStrategy) {
    this.strategy = strategy;
  }

  setStrategy(strategy: PaymentStrategy): void {
    this.strategy = strategy;
  }

  executePayment(amount: number): boolean {
    return this.strategy.pay(amount);
  }
}

// 使用
const context = new PaymentContext(new CreditCardPayment('1234-5678'));
context.executePayment(100);
context.setStrategy(new AlipayPayment('user@example.com'));
context.executePayment(200);
```

### 2.2 函数式策略

```typescript
// 策略作为函数类型
type SortStrategy<T> = (a: T, b: T) => number;

// 预定义策略
const sortStrategies = {
  byName: <T extends { name: string }>(a: T, b: T) => a.name.localeCompare(b.name),
  byAge: <T extends { age: number }>(a: T, b: T) => a.age - b.age,
  byDate: (a: { date: Date }, b: { date: Date }) => a.date.getTime() - b.date.getTime(),
} satisfies Record<string, SortStrategy<any>>;

// 使用
function sort<T>(items: T[], strategy: SortStrategy<T>): T[] {
  return [...items].sort(strategy);
}

const users = [{ name: '张三', age: 25 }, { name: '李四', age: 30 }];
const sorted = sort(users, sortStrategies.byAge);
```

### 2.3 策略注册表

```typescript
type StrategyType = 'compress' | 'encrypt' | 'encode';

interface DataStrategy {
  execute(data: string): string;
}

class StrategyRegistry {
  private static strategies = new Map<StrategyType, DataStrategy>();

  static register(type: StrategyType, strategy: DataStrategy): void {
    this.strategies.set(type, strategy);
  }

  static get(type: StrategyType): DataStrategy {
    const strategy = this.strategies.get(type);
    if (!strategy) throw new Error(`未注册的策略: ${type}`);
    return strategy;
  }
}

// 注册
StrategyRegistry.register('compress', { execute: (data) => `compressed(${data})` });
StrategyRegistry.register('encrypt', { execute: (data) => `encrypted(${data})` });

// 使用
const result = StrategyRegistry.get('compress').execute('hello');
```

## 三、注意事项与常见陷阱

1. **策略接口要精确定义**：参数和返回值类型
2. **策略可以是函数**：比类更简洁
3. **使用 `satisfies` 确保策略对象符合接口**
4. **策略注册表提供运行时动态切换**
5. **不要在策略中持有状态**：策略应是无状态的
