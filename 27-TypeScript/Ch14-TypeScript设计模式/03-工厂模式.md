# 工厂模式

## 一、概念说明

工厂模式将对象创建逻辑封装起来，让调用者不需要知道具体创建细节。TypeScript 的泛型可以让工厂方法返回精确的类型，确保类型安全。

## 二、具体用法

### 2.1 简单工厂

```typescript
// 产品类型
interface Logger {
  log(message: string): void;
}

type LogLevel = 'info' | 'warn' | 'error';

// 工厂
class LoggerFactory {
  static create(level: LogLevel): Logger {
    switch (level) {
      case 'info':
        return { log: (msg) => console.log(`[INFO] ${msg}`) };
      case 'warn':
        return { log: (msg) => console.warn(`[WARN] ${msg}`) };
      case 'error':
        return { log: (msg) => console.error(`[ERROR] ${msg}`) };
    }
  }
}

// 使用
const logger = LoggerFactory.create('info'); // 类型: Logger
logger.log('Hello');
```

### 2.2 泛型工厂

```typescript
// 抽象产品
interface Serializer {
  serialize(data: unknown): string;
  deserialize<T>(input: string): T;
}

// 具体产品
class JsonSerializer implements Serializer {
  serialize(data: unknown): string {
    return JSON.stringify(data);
  }
  deserialize<T>(input: string): T {
    return JSON.parse(input) as T;
  }
}

class YamlSerializer implements Serializer {
  serialize(data: unknown): string {
    return `yaml: ${JSON.stringify(data)}`;
  }
  deserialize<T>(input: string): T {
    return JSON.parse(input.replace('yaml: ', '')) as T;
  }
}

// 工厂注册
type SerializerType = 'json' | 'yaml';

class SerializerFactory {
  private static registry = new Map<SerializerType, () => Serializer>([
    ['json', () => new JsonSerializer()],
    ['yaml', () => new YamlSerializer()],
  ]);

  static create(type: SerializerType): Serializer {
    const factory = this.registry.get(type);
    if (!factory) throw new Error(`未知的序列化器: ${type}`);
    return factory();
  }

  static register(type: SerializerType, factory: () => Serializer): void {
    this.registry.set(type, factory);
  }
}
```

### 2.3 抽象工厂

```typescript
// 抽象产品接口
interface Button {
  render(): string;
}

interface Dialog {
  render(): string;
}

// 抽象工厂
interface UIFactory {
  createButton(): Button;
  createDialog(): Dialog;
}

// 具体工厂
class MaterialUIFactory implements UIFactory {
  createButton(): Button {
    return { render: () => '<button class="material">Material Button</button>' };
  }
  createDialog(): Dialog {
    return { render: () => '<div class="material-dialog">Material Dialog</div>' };
  }
}

// 使用 — 类型安全
function createUI(factory: UIFactory) {
  const button = factory.createButton(); // Button
  const dialog = factory.createDialog(); // Dialog
  return { button, dialog };
}
```

## 三、注意事项与常见陷阱

1. **工厂方法返回接口类型**：隐藏具体实现
2. **使用注册表模式**：支持运行时动态注册
3. **泛型确保产品类型一致**：`Factory<T>`
4. **Discriminated Union 可替代简单工厂**：更类型安全
5. **不要过度使用工厂**：简单场景直接创建即可
