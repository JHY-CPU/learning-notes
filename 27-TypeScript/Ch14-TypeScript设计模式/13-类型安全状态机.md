# 类型安全状态机

## 一、概念说明

TypeScript 的 Discriminated Union 可以实现编译时类型安全的状态机，确保所有状态和事件的组合都有效。这是 TypeScript 最强大的模式之一。

## 二、具体用法

### 2.1 状态与事件类型

```typescript
// 状态定义
type OrderState =
  | { status: 'pending' }
  | { status: 'paid'; paidAt: Date }
  | { status: 'shipped'; trackingNumber: string }
  | { status: 'delivered'; deliveredAt: Date }
  | { status: 'cancelled'; reason: string };

// 事件定义
type OrderEvent =
  | { type: 'PAY'; paidAt: Date }
  | { type: 'SHIP'; trackingNumber: string }
  | { type: 'DELIVER' }
  | { type: 'CANCEL'; reason: string };
```

### 2.2 状态转移

```typescript
function transition(state: OrderState, event: OrderEvent): OrderState {
  switch (state.status) {
    case 'pending':
      switch (event.type) {
        case 'PAY':
          return { status: 'paid', paidAt: event.paidAt };
        case 'CANCEL':
          return { status: 'cancelled', reason: event.reason };
        default:
          throw new Error(`pending 状态不支持事件: ${event.type}`);
      }

    case 'paid':
      switch (event.type) {
        case 'SHIP':
          return { status: 'shipped', trackingNumber: event.trackingNumber };
        case 'CANCEL':
          return { status: 'cancelled', reason: event.reason };
        default:
          throw new Error(`paid 状态不支持事件: ${event.type}`);
      }

    case 'shipped':
      switch (event.type) {
        case 'DELIVER':
          return { status: 'delivered', deliveredAt: new Date() };
        default:
          throw new Error(`shipped 状态不支持事件: ${event.type}`);
      }

    case 'delivered':
    case 'cancelled':
      throw new Error(`${state.status} 是终态，不接受任何事件`);
  }
}
```

### 2.3 状态机类

```typescript
class OrderStateMachine {
  private state: OrderState = { status: 'pending' };

  send(event: OrderEvent): void {
    this.state = transition(this.state, event);
    console.log(`状态变为: ${this.state.status}`);
  }

  getState(): OrderState {
    return this.state;
  }

  // 类型守卫
  isPaid(): this is { state: OrderState & { status: 'paid' } } {
    return this.state.status === 'paid';
  }
}

// 使用
const order = new OrderStateMachine();
order.send({ type: 'PAY', paidAt: new Date() });
order.send({ type: 'SHIP', trackingNumber: 'SF123456' });
console.log(order.getState().status); // 'shipped'
```

### 2.4 穷尽性检查

```typescript
function getOrderSummary(state: OrderState): string {
  switch (state.status) {
    case 'pending': return '等待支付';
    case 'paid': return `已支付于 ${state.paidAt.toLocaleDateString()}`;
    case 'shipped': return `已发货，单号: ${state.trackingNumber}`;
    case 'delivered': return '已送达';
    case 'cancelled': return `已取消: ${state.reason}`;
    default:
      // 编译时检查：如果添加了新状态但忘记处理，这里会报错
      const _exhaustive: never = state;
      return _exhaustive;
  }
}
```

## 三、注意事项与常见陷阱

1. **Discriminated Union 是核心**：每个状态都有唯一的判别属性
2. **穷尽性检查用 `never`**：确保所有状态都被处理
3. **状态转移函数是纯函数**：不修改原状态
4. **终态不应接受事件**：在默认分支抛异常
5. **复杂状态机考虑使用库**：如 XState
