# TS设计模式概述

## 一、概念说明

TypeScript 的类型系统让设计模式更加**安全和可维护**。类型化的模式能在编译时捕获接口不匹配、参数类型错误等问题，减少运行时异常。TypeScript 特有的模式（Branded Types、Discriminated Union、Result/Option 类型）结合了函数式编程理念，提供了比传统 OOP 模式更简洁的方案。

## 二、具体用法

### 2.1 类型安全的核心价值

```typescript
// 没有类型的设计模式 — 运行时才能发现错误
const strategyMap = {
  save: () => {},
  delete: () => {},
};
// strategyMap["unknown"]; // 运行时 undefined，无报错

// 有类型的设计模式 — 编译时就发现错误
type Action = "save" | "delete" | "update";
const handler: Record<Action, () => void> = {
  save: () => console.log("保存"),
  delete: () => console.log("删除"),
  update: () => console.log("更新"),
};
// handler["unknown"]; // 编译错误
```

### 2.2 常用模式分类

| 类型 | 模式 | TypeScript 特色 |
|------|------|----------------|
| 创建型 | 单例、工厂、建造者 | 泛型约束产品类型 |
| 结构型 | 适配器、装饰器、代理 | 类型转换与接口约束 |
| 行为型 | 策略、观察者、状态机 | Discriminated Union |

### 2.3 TypeScript 特有模式

```typescript
// 1. 品牌类型 — 模拟名义类型
type PositiveNumber = number & { __brand: "Positive" };
function createPositive(n: number): PositiveNumber {
  if (n <= 0) throw new Error("必须为正数");
  return n as PositiveNumber;
}

// 2. Result 类型 — 替代异常
type Result<T, E = Error> =
  | { ok: true; value: T }
  | { ok: false; error: E };

function divide(a: number, b: number): Result<number, string> {
  if (b === 0) return { ok: false, error: "除数不能为零" };
  return { ok: true, value: a / b };
}

// 3. Option 类型 — 替代 null
type Option<T> = { some: true; value: T } | { some: false };

function findUser(id: number): Option<{ name: string }> {
  const user = id === 1 ? { name: "Alice" } : undefined;
  return user ? { some: true, value: user } : { some: false };
}

// 4. Discriminated Union — 类型安全的分支
type Shape =
  | { kind: "circle"; radius: number }
  | { kind: "rect"; width: number; height: number };

function area(shape: Shape): number {
  switch (shape.kind) {
    case "circle": return Math.PI * shape.radius ** 2;
    case "rect": return shape.width * shape.height;
  }
}
```

### 2.4 策略模式（泛型版）

```typescript
interface SortStrategy<T> {
  compare(a: T, b: T): number;
}

class NumberAsc implements SortStrategy<number> {
  compare(a: number, b: number): number { return a - b; }
}

class NumberDesc implements SortStrategy<number> {
  compare(a: number, b: number): number { return b - a; }
}

function sort<T>(arr: T[], strategy: SortStrategy<T>): T[] {
  return [...arr].sort((a, b) => strategy.compare(a, b));
}

sort([3, 1, 2], new NumberAsc());  // [1, 2, 3]
sort([3, 1, 2], new NumberDesc()); // [3, 2, 1]
```

### 2.5 与 JavaScript 的对比

```javascript
// JavaScript：设计模式无类型保护
const strategy = strategies[action]; // 可能是 undefined
strategy.execute(data); // 运行时才检查

// TypeScript：编译时保证模式正确
// 策略接口约束、泛型参数约束、Discriminated Union
// 编译时就确保所有分支都正确处理
```

## 三、注意事项与常见陷阱

1. **不要为了用模式而用模式**：简单场景用简单方案，避免过度设计
2. **类型是编译时的**：运行时仍需要验证外部数据（API 响应、用户输入）
3. **泛型让模式更通用**：但不要过度抽象，保持可读性
4. **Discriminated Union 是 TS 最强大的模式之一**：替代继承层次，类型安全地处理分支
5. **优先考虑函数式模式**：Result/Option 比 try/catch 更明确，比 null 检查更安全
6. **组合优于继承**：TypeScript 的接口和类型组合比 class 继承更灵活
