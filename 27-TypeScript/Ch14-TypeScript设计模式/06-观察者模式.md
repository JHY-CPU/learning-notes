# 观察者模式

## 一、概念说明

观察者模式定义对象间**一对多**的依赖关系：当被观察者（Subject）状态变化时，所有观察者（Observer）自动收到通知。TypeScript 通过泛型确保事件类型和回调参数的**类型安全**，编译时就能发现事件名拼写错误、参数类型不匹配等问题。常用于事件系统、状态管理、UI 组件通信。

## 二、具体用法

### 2.1 泛型事件发射器

```typescript
type EventMap = {
  login: (userId: number, timestamp: number) => void;
  logout: () => void;
  error: (error: Error) => void;
};

class TypedEventEmitter<
  TEvents extends Record<string, (...args: any[]) => void>
> {
  private listeners = new Map<keyof TEvents, Set<Function>>();

  on<K extends keyof TEvents>(event: K, listener: TEvents[K]): () => void {
    if (!this.listeners.has(event)) {
      this.listeners.set(event, new Set());
    }
    this.listeners.get(event)!.add(listener);

    // 返回取消订阅函数（防止内存泄漏）
    return () => this.off(event, listener);
  }

  off<K extends keyof TEvents>(event: K, listener: TEvents[K]): void {
    this.listeners.get(event)?.delete(listener);
  }

  emit<K extends keyof TEvents>(
    event: K,
    ...args: Parameters<TEvents[K]>
  ): void {
    this.listeners.get(event)?.forEach(fn => fn(...args));
  }
}

// 使用
const emitter = new TypedEventEmitter<EventMap>();

const unsubscribe = emitter.on("login", (userId, timestamp) => {
  console.log(`用户 ${userId} 登录，时间: ${timestamp}`);
});

emitter.emit("login", 123, Date.now()); // 类型安全
// emitter.emit("login", "abc");         // 编译错误：string 不能赋给 number
// emitter.emit("unknown", {});          // 编译错误：unknown 不在事件中

unsubscribe(); // 取消订阅
```

### 2.2 发布订阅模式（类版本）

```typescript
interface Subscriber<T> {
  update(data: T): void;
}

class Publisher<T> {
  private subscribers = new Set<Subscriber<T>>();

  subscribe(subscriber: Subscriber<T>): () => void {
    this.subscribers.add(subscriber);
    return () => this.subscribers.delete(subscriber);
  }

  publish(data: T): void {
    this.subscribers.forEach(sub => sub.update(data));
  }
}

// 使用
const publisher = new Publisher<{ type: string; payload: unknown }>();

const logger: Subscriber<{ type: string; payload: unknown }> = {
  update(data) {
    console.log(`事件: ${data.type}`, data.payload);
  },
};

publisher.subscribe(logger);
publisher.publish({ type: "user_login", payload: { id: 1 } });
```

### 2.3 响应式状态（简化版）

```typescript
function reactive<T>(initial: T): {
  get: () => T;
  set: (value: T) => void;
  subscribe: (fn: (value: T) => void) => () => void;
} {
  let value = initial;
  const watchers = new Set<(value: T) => void>();

  return {
    get: () => value,
    set: (newValue: T) => {
      value = newValue;
      watchers.forEach(fn => fn(value));
    },
    subscribe: (fn: (value: T) => void) => {
      watchers.add(fn);
      return () => watchers.delete(fn);
    },
  };
}

// 使用
const count = reactive(0);
const unsub = count.subscribe((val) => console.log("计数变化:", val));
count.set(1); // 输出: 计数变化: 1
count.set(2); // 输出: 计数变化: 2
unsub();
```

### 2.4 实际应用：组件通信

```typescript
// 表单组件事件
interface FormEvents {
  submit: { values: Record<string, unknown>; timestamp: number };
  reset: undefined;
  fieldChange: { field: string; value: unknown; oldValue: unknown };
  validationError: { field: string; error: string };
}

const formEvents = new TypedEventEmitter<FormEvents>();

formEvents.on("submit", ({ values, timestamp }) => {
  console.log("表单提交", values);
});

formEvents.on("fieldChange", ({ field, value, oldValue }) => {
  console.log(`${field}: ${oldValue} -> ${value}`);
});
```

## 三、注意事项与常见陷阱

1. **使用泛型约束事件参数**：确保 `emit` 和 `on` 的参数类型匹配
2. **返回取消订阅函数**：防止内存泄漏，这是关键的 API 设计
3. **`Set` 存储监听器**：自动去重，同一函数不会被注册多次
4. **事件名用字面量联合类型**：IDE 自动补全，编译时检查拼写
5. **无数据事件用 `undefined`**：不要用空对象 `{}`，`undefined` 更准确
6. **大量监听器的性能**：频繁触发事件时注意回调函数的性能
