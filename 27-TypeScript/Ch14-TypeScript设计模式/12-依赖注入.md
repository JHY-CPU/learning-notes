# 依赖注入

## 一、概念说明

依赖注入（DI）是将依赖从外部传入而非内部创建的模式。TypeScript 的接口和泛型让 DI 容器有完整的类型安全。

## 二、具体用法

### 2.1 简单 DI 容器

```typescript
class DIContainer {
  private services = new Map<string, unknown>();

  register<T>(token: string, instance: T): void {
    this.services.set(token, instance);
  }

  resolve<T>(token: string): T {
    const service = this.services.get(token);
    if (!service) {
      throw new Error(`未注册的服务: ${token}`);
    }
    return service as T;
  }
}

// 使用
interface Logger {
  log(msg: string): void;
}

const container = new DIContainer();
container.register<Logger>('logger', {
  log: (msg) => console.log(msg),
});

const logger = container.resolve<Logger>('logger');
logger.log('Hello DI');
```

### 2.2 类型安全 DI

```typescript
// 使用 Symbol 作为 token
const TOKENS = {
  Logger: Symbol('Logger'),
  Database: Symbol('Database'),
  Config: Symbol('Config'),
} as const;

interface ILogger {
  log(msg: string): void;
}

interface IDatabase {
  query<T>(sql: string): Promise<T[]>;
}

interface IConfig {
  get<T>(key: string): T;
}

class TypedContainer {
  private services = new Map<symbol, unknown>();

  register<T>(token: symbol, instance: T): void {
    this.services.set(token, instance);
  }

  resolve<T>(token: symbol): T {
    const service = this.services.get(token);
    if (!service) throw new Error(`未注册: ${token.toString()}`);
    return service as T;
  }
}

// 注册
const container = new TypedContainer();
container.register<ILogger>(TOKENS.Logger, { log: console.log });
container.register<IDatabase>(TOKENS.Database, { query: async () => [] });

// 解析 — 类型安全
const db = container.resolve<IDatabase>(TOKENS.Database);
```

### 2.3 自动注入

```typescript
// 工厂函数注册
class SmartContainer {
  private factories = new Map<symbol, () => unknown>();
  private instances = new Map<symbol, unknown>();

  register<T>(token: symbol, factory: () => T): void {
    this.factories.set(token, factory);
  }

  singleton<T>(token: symbol, factory: () => T): void {
    this.register(token, () => {
      if (!this.instances.has(token)) {
        this.instances.set(token, factory());
      }
      return this.instances.get(token);
    });
  }

  resolve<T>(token: symbol): T {
    const factory = this.factories.get(token);
    if (!factory) throw new Error(`未注册: ${token.toString()}`);
    return factory() as T;
  }
}
```

## 三、注意事项与常见陷阱

1. **使用 Symbol 避免 token 冲突**：比字符串更安全
2. **注册时指定类型参数**：`register<Interface>(token, impl)`
3. **单例 vs 瞬态**：根据需求选择生命周期
4. **DI 容器本身是类型安全的**：利用泛型约束
5. **不要过度使用 DI**：简单场景直接实例化
