# 命令模式

## 一、概念说明

命令模式将请求封装为对象，支持撤销、重做和队列操作。TypeScript 泛型确保命令的参数和返回值类型安全。

## 二、具体用法

### 2.1 基本命令模式

```typescript
// 命令接口
interface Command<T> {
  execute(): T;
  undo(): void;
}

// 具体命令
class AddItemCommand implements Command<void> {
  private index: number = -1;

  constructor(
    private list: string[],
    private item: string
  ) {}

  execute(): void {
    this.list.push(this.item);
    this.index = this.list.length - 1;
  }

  undo(): void {
    if (this.index >= 0) {
      this.list.splice(this.index, 1);
    }
  }
}

class RemoveItemCommand implements Command<string | undefined> {
  private removed?: string;
  private index: number = -1;

  constructor(
    private list: string[],
    private item: string
  ) {}

  execute(): string | undefined {
    this.index = this.list.indexOf(this.item);
    if (this.index >= 0) {
      this.removed = this.list.splice(this.index, 1)[0];
    }
    return this.removed;
  }

  undo(): void {
    if (this.removed && this.index >= 0) {
      this.list.splice(this.index, 0, this.removed);
    }
  }
}
```

### 2.2 命令管理器（撤销/重做）

```typescript
class CommandManager {
  private history: Command<any>[] = [];
  private undone: Command<any>[] = [];

  execute<T>(command: Command<T>): T {
    const result = command.execute();
    this.history.push(command);
    this.undone = []; // 清空重做栈
    return result;
  }

  undo(): void {
    const command = this.history.pop();
    if (command) {
      command.undo();
      this.undone.push(command);
    }
  }

  redo(): void {
    const command = this.undone.pop();
    if (command) {
      command.execute();
      this.history.push(command);
    }
  }

  canUndo(): boolean {
    return this.history.length > 0;
  }

  canRedo(): boolean {
    return this.undone.length > 0;
  }
}
```

### 2.3 使用

```typescript
const items: string[] = [];
const manager = new CommandManager();

manager.execute(new AddItemCommand(items, '任务1'));
manager.execute(new AddItemCommand(items, '任务2'));
console.log(items); // ['任务1', '任务2']

manager.undo();
console.log(items); // ['任务1']

manager.redo();
console.log(items); // ['任务1', '任务2']
```

## 三、注意事项与常见陷阱

1. **命令接口需要泛型**：支持不同返回值类型
2. **undo 必须精确还原状态**：保存修改前的数据
3. **命令队列支持异步执行**
4. **宏命令组合多个命令**：批量执行
