# 适配器模式

## 一、概念说明

适配器模式将一个类的**接口**转换为调用者期望的另一个接口，使不兼容的类可以协同工作。TypeScript 通过 `implements` 约束确保适配器正确实现目标接口，编译时就能发现接口不匹配。常见应用：封装第三方 SDK、转换数据格式、统一不同服务的 API。

## 二、具体用法

### 2.1 类适配器（implements 约束）

```typescript
// 目标接口
interface PaymentGateway {
  charge(amount: number, currency: string): Promise<boolean>;
  refund(transactionId: string): Promise<void>;
}

// 被适配的类（第三方 SDK，接口不匹配）
class StripeSDK {
  async makeCharge(cents: number, curr: string): Promise<{ status: string }> {
    console.log(`Stripe: ${cents} ${curr}`);
    return { status: "success" };
  }

  async createRefund(chargeId: string): Promise<void> {
    console.log(`Stripe 退款: ${chargeId}`);
  }
}

// 适配器
class StripeAdapter implements PaymentGateway {
  constructor(private stripe: StripeSDK) {}

  async charge(amount: number, currency: string): Promise<boolean> {
    const result = await this.stripe.makeCharge(amount * 100, currency);
    return result.status === "success";
  }

  async refund(transactionId: string): Promise<void> {
    await this.stripe.createRefund(transactionId);
  }
}

// 使用（统一接口）
async function processPayment(gateway: PaymentGateway) {
  const success = await gateway.charge(99.99, "USD");
  if (!success) {
    await gateway.refund("tx_123");
  }
}

processPayment(new StripeAdapter(new StripeSDK()));
```

### 2.2 对象适配器（类型转换）

```typescript
// 旧 API 数据格式
interface LegacyUser {
  user_name: string;
  user_email: string;
  user_age: number;
  created_at: string;
}

// 新 API 数据格式
interface ModernUser {
  name: string;
  email: string;
  age: number;
  createdAt: Date;
}

// 适配函数
function adaptUser(legacy: LegacyUser): ModernUser {
  return {
    name: legacy.user_name,
    email: legacy.user_email,
    age: legacy.user_age,
    createdAt: new Date(legacy.created_at),
  };
}

function adaptUsers(legacyUsers: LegacyUser[]): ModernUser[] {
  return legacyUsers.map(adaptUser);
}

// 使用
const legacyData: LegacyUser[] = [
  { user_name: "Alice", user_email: "a@b.com", user_age: 25, created_at: "2024-01-01" },
];
const modernUsers = adaptUsers(legacyData);
console.log(modernUsers[0].name); // Alice
```

### 2.3 双向适配器

```typescript
interface Logger {
  info(message: string): void;
  error(message: string, error?: Error): void;
}

interface ThirdPartyLogger {
  log(level: string, msg: string, metadata?: object): void;
}

// 将标准 Logger 适配为第三方接口
class LoggerToThirdParty implements ThirdPartyLogger {
  constructor(private logger: Logger) {}

  log(level: string, msg: string, metadata?: object): void {
    if (level === "error") {
      this.logger.error(msg, metadata as Error);
    } else {
      this.logger.info(msg);
    }
  }
}

// 将第三方接口适配为标准 Logger
class ThirdPartyToLogger implements Logger {
  constructor(private tp: ThirdPartyLogger) {}

  info(message: string): void {
    this.tp.log("info", message);
  }

  error(message: string, error?: Error): void {
    this.tp.log("error", message, error);
  }
}
```

### 2.4 泛型适配器

```typescript
// 通用数据转换适配器
interface Adapter<TSource, TTarget> {
  adapt(source: TSource): TTarget;
}

function createAdapter<TSource, TTarget>(
  transform: (source: TSource) => TTarget
): Adapter<TSource, TTarget> {
  return { adapt: transform };
}

// 使用
const userAdapter = createAdapter<LegacyUser, ModernUser>(adaptUser);
const modernUser = userAdapter.adapt({
  user_name: "Bob",
  user_email: "b@b.com",
  user_age: 30,
  created_at: "2024-06-01",
});
```

## 三、注意事项与常见陷阱

1. **适配器实现目标接口**：使用 `implements` 确保编译时检查所有方法
2. **适配器不应修改被适配类**：只做包装和转换，不修改原始实现
3. **类型转换是适配器的常见用途**：旧格式到新格式的转换
4. **多个适配器可实现同一接口**：`StripeAdapter`、`PayPalAdapter` 都实现 `PaymentGateway`
5. **组合使用**：适配器 + 工厂模式 = 运行时切换适配器
6. **注意空值处理**：适配器要做好 null/undefined 的防御性处理
