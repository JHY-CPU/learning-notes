# 状态模式

## 一、概念说明

状态模式将状态相关的行为封装在独立的类中。TypeScript 通过接口约束确保状态类实现所有必要的方法，Discriminated Union 可以用于类型安全的状态表示。

## 二、具体用法

### 2.1 类型安全状态机

```typescript
type State = 'idle' | 'loading' | 'success' | 'error';
type Event = 'FETCH' | 'RESOLVE' | 'REJECT' | 'RETRY';

const transitions: Record<State, Partial<Record<Event, State>>> = {
  idle: { FETCH: 'loading' },
  loading: { RESOLVE: 'success', REJECT: 'error' },
  success: { FETCH: 'loading' },
  error: { RETRY: 'loading', FETCH: 'loading' },
};

class StateMachine {
  private state: State = 'idle';

  transition(event: Event): State {
    const nextState = transitions[this.state]?.[event];
    if (!nextState) {
      throw new Error(`无效的状态转换: ${this.state} -> ${event}`);
    }
    this.state = nextState;
    return this.state;
  }

  getState(): State {
    return this.state;
  }
}
```

### 2.2 状态类模式

```typescript
interface StateContext {
  setState(state: AppState): void;
}

abstract class AppState {
  constructor(protected context: StateContext) {}
  abstract handle(): void;
}

class IdleState extends AppState {
  handle(): void {
    console.log('空闲状态，开始获取数据');
    this.context.setState(new LoadingState(this.context));
  }
}

class LoadingState extends AppState {
  handle(): void {
    console.log('加载中...');
    // 模拟成功
    this.context.setState(new SuccessState(this.context));
  }
}

class SuccessState extends AppState {
  handle(): void {
    console.log('加载成功');
  }
}

class App implements StateContext {
  private state: AppState = new IdleState(this);

  setState(state: AppState): void {
    this.state = state;
  }

  run(): void {
    this.state.handle();
  }
}
```

## 三、注意事项与常见陷阱

1. **Discriminated Union 替代类**：更简洁的类型安全
2. **转移表确保所有状态转换有效**：编译时检查
3. **状态类模式适合复杂逻辑**：简单状态用联合类型
4. **`never` 实现穷尽性检查**：确保处理所有状态
