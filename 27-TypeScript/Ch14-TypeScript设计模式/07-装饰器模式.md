# 装饰器模式

## 一、概念说明

TypeScript 装饰器是一种特殊声明，可以修改类、方法、属性和参数的行为。TC39 Stage 3 装饰器是 TypeScript 5.0+ 的标准实现。

## 二、具体用法

### 2.1 类装饰器

```typescript
// 类装饰器 — 添加日志功能
function withLogger<T extends new (...args: any[]) => any>(Base: T) {
  return class extends Base {
    constructor(...args: any[]) {
      super(...args);
      console.log(`创建实例: ${Base.name}`);
    }
  };
}

@withLogger
class UserService {
  name = 'UserService';
}

// new UserService() 会打印日志
```

### 2.2 方法装饰器

```typescript
// 方法装饰器 — 性能监控
function measure(
  target: (this: any, ...args: any[]) => any,
  context: ClassMethodDecoratorContext
) {
  const name = String(context.name);

  return function (this: any, ...args: any[]) {
    const start = performance.now();
    const result = target.apply(this, args);
    const duration = performance.now() - start;
    console.log(`${name} 执行时间: ${duration.toFixed(2)}ms`);
    return result;
  };
}

class Calculator {
  @measure
  add(a: number, b: number): number {
    return a + b;
  }
}
```

### 2.3 自动绑定装饰器

```typescript
function autoBind(
  target: (this: any, ...args: any[]) => any,
  context: ClassMethodDecoratorContext
) {
  const name = context.name;
  context.addInitializer(function (this: any) {
    this[name] = this[name].bind(this);
  });
  return target;
}

class Button {
  label: string;
  constructor(label: string) {
    this.label = label;
  }

  @autoBind
  handleClick() {
    console.log(`${this.label} 被点击`);
  }
}
```

### 2.4 属性装饰器

```typescript
// 属性装饰器 — 验证
function minLength(min: number) {
  return function (target: any, context: ClassFieldDecoratorContext) {
    return function (this: any, value: string) {
      if (value.length < min) {
        throw new Error(`${String(context.name)} 至少需要 ${min} 个字符`);
      }
      return value;
    };
  };
}

class User {
  @minLength(2)
  name: string = '';

  constructor(name: string) {
    this.name = name;
  }
}
```

## 三、注意事项与常见陷阱

1. **Stage 3 装饰器与旧版不同**：不使用实验性装饰器选项
2. **装饰器是编译时元编程**：不影响运行时逻辑
3. **装饰器顺序从下往上执行**：多个装饰器的执行顺序
4. **`context` 参数提供元信息**：名称、静态/实例等
5. **不要过度使用装饰器**：增加代码复杂度
