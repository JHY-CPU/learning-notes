# 建造者模式

## 一、概念说明

建造者模式将复杂对象的构建过程与表示分离。TypeScript 的泛型和方法链（Fluent API）可以让建造者有精确的类型约束，确保必填属性不被遗漏。

## 二、具体用法

### 2.1 基本建造者

```typescript
class QueryBuilder {
  private table: string = '';
  private conditions: string[] = [];
  private orderBy: string = '';
  private limitValue?: number;

  from(table: string): this {
    this.table = table;
    return this;
  }

  where(condition: string): this {
    this.conditions.push(condition);
    return this;
  }

  sort(field: string, direction: 'ASC' | 'DESC' = 'ASC'): this {
    this.orderBy = `${field} ${direction}`;
    return this;
  }

  limit(count: number): this {
    this.limitValue = count;
    return this;
  }

  build(): string {
    let query = `SELECT * FROM ${this.table}`;
    if (this.conditions.length > 0) {
      query += ` WHERE ${this.conditions.join(' AND ')}`;
    }
    if (this.orderBy) {
      query += ` ORDER BY ${this.orderBy}`;
    }
    if (this.limitValue) {
      query += ` LIMIT ${this.limitValue}`;
    }
    return query;
  }
}

// 使用 — 链式调用
const query = new QueryBuilder()
  .from('users')
  .where('age > 18')
  .where('active = true')
  .sort('name')
  .limit(10)
  .build();

// SELECT * FROM users WHERE age > 18 AND active = true ORDER BY name ASC LIMIT 10
```

### 2.2 类型安全建造者

```typescript
// 使用泛型确保必填属性
interface User {
  id: number;
  name: string;
  email: string;
  age?: number;
  role?: 'admin' | 'user';
}

class UserBuilder {
  private data: Partial<User> = {};

  setId(id: number): this {
    this.data.id = id;
    return this;
  }

  setName(name: string): this {
    this.data.name = name;
    return this;
  }

  setEmail(email: string): this {
    this.data.email = email;
    return this;
  }

  setAge(age: number): this {
    this.data.age = age;
    return this;
  }

  build(): User {
    const { id, name, email } = this.data;
    if (id === undefined || !name || !email) {
      throw new Error('缺少必填字段: id, name, email');
    }
    return { id, name, email, ...this.data };
  }
}
```

### 2.3 泛型类型安全建造者

```typescript
// 编译时检查必填属性
type BuildState = {
  id?: number;
  name?: string;
  email?: string;
};

class TypedUserBuilder<T extends BuildState = {}> {
  private data: Partial<User> = {};

  setId(id: number): TypedUserBuilder<T & { id: number }> {
    this.data.id = id;
    return this as any;
  }

  setName(name: string): TypedUserBuilder<T & { name: string }> {
    this.data.name = name;
    return this as any;
  }

  setEmail(email: string): TypedUserBuilder<T & { email: string }> {
    this.data.email = email;
    return this as any;
  }

  build(this: TypedUserBuilder<{ id: number; name: string; email: string }>): User {
    return this.data as User;
  }
}

// 编译时检查：缺少必填字段会报错
// new TypedUserBuilder().setName('张三').build(); // 编译错误：缺少 id 和 email
new TypedUserBuilder().setId(1).setName('张三'). setEmail('test@example.com').build(); // OK
```

## 三、注意事项与常见陷阱

1. **`return this` 支持链式调用**：返回类型必须是 `this`
2. **建造者适合复杂对象**：简单对象不需要
3. **运行时验证必填属性**：编译时 + 运行时双重保障
4. **不可变对象用 `Object.freeze`**：build 后冻结
5. **不要滥用泛型状态追踪**：复杂度过高时考虑简化
