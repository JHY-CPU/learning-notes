# 日志类型

## 一、概念说明

日志是 Node.js 应用调试和监控的核心。主流日志库（Winston、Pino）都提供了完整的 TypeScript 支持，可以类型化日志级别、元数据和日志格式。

## 二、具体用法

### 2.1 Winston 类型配置

```typescript
import winston from 'winston';

// 日志级别类型
type LogLevel = 'error' | 'warn' | 'info' | 'http' | 'verbose' | 'debug' | 'silly';

// 自定义日志格式
interface LogMeta {
  userId?: number;
  requestId?: string;
  method?: string;
  path?: string;
  statusCode?: number;
  duration?: number;
}

// 创建 logger
const logger = winston.createLogger({
  level: (process.env.LOG_LEVEL ?? 'info') as LogLevel,
  format: winston.format.combine(
    winston.format.timestamp(),
    winston.format.json()
  ),
  defaultMeta: { service: 'api' },
  transports: [
    new winston.transports.Console({
      format: winston.format.combine(
        winston.format.colorize(),
        winston.format.simple()
      ),
    }),
    new winston.transports.File({ filename: 'logs/error.log', level: 'error' }),
    new winston.transports.File({ filename: 'logs/combined.log' }),
  ],
});

// 使用
logger.info('用户登录成功', { userId: 123 } satisfies LogMeta);
logger.error('数据库连接失败', { error: err.message });
```

### 2.2 Pino 类型配置

```typescript
import pino from 'pino';

// Pino 推荐用于高性能日志
const logger = pino({
  level: process.env.LOG_LEVEL ?? 'info',
  transport: {
    target: 'pino-pretty',
    options: { colorize: true },
  },
});

// 子日志器（带固定上下文）
const requestLogger = logger.child({ module: 'http' });

// 使用
logger.info({ userId: 123, action: 'login' }, '用户登录成功');

requestLogger.info({
  method: 'GET',
  url: '/api/users',
  statusCode: 200,
  responseTime: 45,
}, '请求完成');
```

### 2.3 请求日志中间件

```typescript
import { RequestHandler } from 'express';

function createRequestLogger(logger: pino.Logger): RequestHandler {
  return (req, res, next) => {
    const start = Date.now();

    // 响应完成时记录日志
    res.on('finish', () => {
      const duration = Date.now() - start;

      logger.info({
        method: req.method,
        path: req.path,
        statusCode: res.statusCode,
        duration,
        userAgent: req.headers['user-agent'],
        ip: req.ip,
      });
    });

    next();
  };
}

app.use(createRequestLogger(logger));
```

### 2.4 日志级别工具类型

```typescript
// 从 logger 实例提取可用的日志方法
type LoggerMethods = 'fatal' | 'error' | 'warn' | 'info' | 'debug' | 'trace';

// 日志上下文类型
interface LogContext {
  requestId: string;
  userId?: number;
  [key: string]: unknown;
}

// 创建类型安全的日志器
function createContextLogger(logger: pino.Logger, context: LogContext) {
  const child = logger.child(context);

  return {
    fatal: (msg: string, meta?: Record<string, unknown>) => child.fatal(meta, msg),
    error: (msg: string, meta?: Record<string, unknown>) => child.error(meta, msg),
    warn: (msg: string, meta?: Record<string, unknown>) => child.warn(meta, msg),
    info: (msg: string, meta?: Record<string, unknown>) => child.info(meta, msg),
    debug: (msg: string, meta?: Record<string, unknown>) => child.debug(meta, msg),
  };
}
```

## 三、注意事项与常见陷阱

1. **Pino 性能优于 Winston**：高吞吐场景首选 Pino
2. **日志级别在生产环境用 `info`**：开发环境可用 `debug`
3. **不要在日志中记录敏感信息**：密码、token、个人身份证号等
4. **使用 `child` 创建上下文日志器**：避免重复传递公共字段
5. **日志异步写入**：避免阻塞主线程
