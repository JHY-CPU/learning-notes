# 中间件链类型

## 一、概念说明

中间件链是 Express/Koa 等框架的核心模式。TypeScript 可以类型化中间件的组合、条件执行和管道式处理，确保中间件链的类型安全。

## 二、具体用法

### 2.1 中间件组合函数

```typescript
import { RequestHandler, Request, Response, NextFunction } from 'express';

// 组合多个中间件为一个
function compose(...middlewares: RequestHandler[]): RequestHandler {
  return (req: Request, res: Response, next: NextFunction) => {
    let index = 0;

    function dispatch(err?: any) {
      if (err) return next(err);

      if (index >= middlewares.length) return next();

      const middleware = middlewares[index++];
      try {
        middleware(req, res, dispatch);
      } catch (error) {
        next(error);
      }
    }

    dispatch();
  };
}

// 使用
const authAndLog = compose(
  logger,
  authenticate,
  rateLimiter
);

app.get('/api/data', authAndLog, dataHandler);
```

### 2.2 条件中间件

```typescript
// 条件执行中间件
function conditionalMiddleware(
  condition: (req: Request) => boolean,
  middleware: RequestHandler
): RequestHandler {
  return (req, res, next) => {
    if (condition(req)) {
      middleware(req, res, next);
    } else {
      next();
    }
  };
}

// 使用 — 只对 /api 路径执行认证
app.use(conditionalMiddleware(
  req => req.path.startsWith('/api'),
  authenticate
));
```

### 2.3 中间件管道类型

```typescript
// 类型安全的中间件管道
type MiddlewarePipe = {
  use(middleware: RequestHandler): MiddlewarePipe;
  handle(handler: RequestHandler): void;
};

function createPipe(app: ReturnType<typeof express>): MiddlewarePipe {
  const middlewares: RequestHandler[] = [];

  return {
    use(middleware) {
      middlewares.push(middleware);
      return this; // 支持链式调用
    },
    handle(handler) {
      // 注册所有中间件 + 最终处理器
      app.use(...middlewares, handler);
    },
  };
}

// 使用
createPipe(app)
  .use(cors())
  .use(helmet())
  .use(rateLimiter)
  .use(authenticate)
  .handle(userHandler);
```

### 2.4 路由级中间件类型

```typescript
import { Router } from 'express';

// 路由中间件配置
interface RouteConfig {
  method: 'get' | 'post' | 'put' | 'delete';
  path: string;
  middleware?: RequestHandler[];
  handler: RequestHandler;
}

// 创建路由
function createRoutes(configs: RouteConfig[]): Router {
  const router = Router();

  for (const config of configs) {
    const handlers = [...(config.middleware ?? []), config.handler];
    router[config.method](config.path, ...handlers);
  }

  return router;
}

// 使用
const userRoutes = createRoutes([
  { method: 'get', path: '/', handler: getUsers },
  {
    method: 'post',
    path: '/',
    middleware: [authenticate, validateBody(createUserSchema)],
    handler: createUser,
  },
]);
```

### 2.5 中间件错误传播

```typescript
// 带类型的异步中间件包装
function withErrorHandling(
  middleware: RequestHandler
): RequestHandler {
  return async (req, res, next) => {
    try {
      await middleware(req, res, next);
    } catch (err) {
      next(err); // 错误传递给下一个中间件
    }
  };
}

// 批量包装
function withErrorHandlingAll(...middlewares: RequestHandler[]): RequestHandler[] {
  return middlewares.map(withErrorHandling);
}
```

## 三、注意事项与常见陷阱

1. **中间件顺序很重要**：认证、日志等中间件应在路由之前
2. **`compose` 要处理异步中间件**：使用 Promise 或 async/await
3. **不要在中间件中多次调用 `next()`**：会导致未定义行为
4. **条件中间件的类型**：`condition` 函数的返回值影响中间件是否执行
5. **链式调用需要返回 `this`**：保持类型推断
