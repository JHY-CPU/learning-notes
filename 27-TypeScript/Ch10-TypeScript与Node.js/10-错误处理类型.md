# 错误处理类型

## 一、概念说明

TypeScript 中的错误处理需要区分不同类型的错误（HTTP 错误、验证错误、业务错误等）。自定义错误类可以携带更丰富的类型信息，让错误处理逻辑更精确。

## 二、具体用法

### 2.1 自定义错误类

```typescript
// 基础自定义错误
class AppError extends Error {
  public readonly statusCode: number;
  public readonly isOperational: boolean;

  constructor(message: string, statusCode = 500, isOperational = true) {
    super(message);
    this.name = this.constructor.name;
    this.statusCode = statusCode;
    this.isOperational = isOperational;

    // 确保 instanceof 正常工作
    Object.setPrototypeOf(this, new.target.prototype);
  }
}

// 具体错误类型
class NotFoundError extends AppError {
  constructor(resource: string, id?: string | number) {
    const message = id ? `${resource} (ID: ${id}) 不存在` : `${resource} 不存在`;
    super(message, 404);
  }
}

class ValidationError extends AppError {
  public readonly fields: Record<string, string>;

  constructor(fields: Record<string, string>) {
    super('数据验证失败', 400);
    this.fields = fields;
  }
}

class UnauthorizedError extends AppError {
  constructor(message = '未授权访问') {
    super(message, 401);
  }
}

class ConflictError extends AppError {
  constructor(message: string) {
    super(message, 409);
  }
}
```

### 2.2 类型安全的错误守卫

```typescript
// 错误类型守卫
function isAppError(error: unknown): error is AppError {
  return error instanceof AppError;
}

function isError(error: unknown): error is Error {
  return error instanceof Error;
}

// 统一错误处理
function handleError(error: unknown): { message: string; statusCode: number } {
  if (isAppError(error)) {
    return { message: error.message, statusCode: error.statusCode };
  }

  if (isError(error)) {
    return { message: error.message, statusCode: 500 };
  }

  return { message: '未知错误', statusCode: 500 };
}
```

### 2.3 Result 模式（替代异常）

```typescript
// Result 类型 — 避免异常，使用返回值表示错误
type Result<T, E = AppError> =
  | { success: true; data: T }
  | { success: false; error: E };

// 使用 Result 的函数
async function findUser(id: number): Promise<Result<User, NotFoundError>> {
  const user = await db.users.findById(id);

  if (!user) {
    return { success: false, error: new NotFoundError('用户', id) };
  }

  return { success: true, data: user };
}

// 消费 Result
async function handleRequest(userId: number) {
  const result = await findUser(userId);

  if (!result.success) {
    return { error: result.error.message, code: result.error.statusCode };
  }

  return { user: result.data }; // data 类型是 User
}
```

### 2.4 Express 错误中间件

```typescript
import { ErrorRequestHandler } from 'express';

const globalErrorHandler: ErrorRequestHandler = (err, req, res, next) => {
  if (isAppError(err)) {
    res.status(err.statusCode).json({
      success: false,
      error: err.message,
      ...(err instanceof ValidationError && { fields: err.fields }),
    });
    return;
  }

  console.error('未处理的错误:', err);
  res.status(500).json({
    success: false,
    error: '服务器内部错误',
  });
};

// 注册
app.use(globalErrorHandler);
```

### 2.5 异步错误包装器

```typescript
// 包装异步函数，返回 Result
function tryCatch<T>(
  fn: () => Promise<T>
): Promise<Result<T>> {
  return fn()
    .then(data => ({ success: true as const, data }))
    .catch(err => ({
      success: false as const,
      error: isAppError(err) ? err : new AppError(String(err)),
    }));
}

// 使用
const result = await tryCatch(() => fetchUser(1));
if (result.success) {
  console.log(result.data);
}
```

## 三、注意事项与常见陷阱

1. **自定义错误类需要 `Object.setPrototypeOf`**：否则 `instanceof` 不生效
2. **`unknown` 类型的错误**：catch 捕获的错误类型是 `unknown`
3. **不要抛出非 Error 对象**：`throw new Error()` 而非 `throw 'message'`
4. **Result 模式适合预期的失败**：异常用于意外错误
5. **Express 异步错误不会被捕获**：需要手动包装或使用 `express-async-errors`
