# 构建与部署

## 一、概念说明

TypeScript 项目的构建需要将 `.ts` 文件编译为 `.js` 文件，同时生成类型声明文件。不同的构建工具（tsc、esbuild、swc）有不同的速度和功能特点，选择合适的工具对开发效率至关重要。

## 二、具体用法

### 2.1 使用 tsc 编译

```bash
# 基本编译
npx tsc

# 带 watch 模式
npx tsc --watch

# 编译到指定目录
npx tsc --outDir dist

# 只做类型检查不生成文件
npx tsc --noEmit
```

```json
// tsconfig.json — 生产编译配置
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "NodeNext",
    "moduleResolution": "NodeNext",
    "outDir": "./dist",
    "rootDir": "./src",
    "declaration": true,        // 生成 .d.ts 类型声明
    "declarationMap": true,     // 类型声明的 source map
    "sourceMap": true,          // JS 的 source map
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "removeComments": false,    // 保留注释
    "noUnusedLocals": true
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "dist", "**/*.test.ts"]
}
```

### 2.2 使用 esbuild（推荐，极快）

```bash
npm install -D esbuild
```

```typescript
// build.mjs — esbuild 构建脚本
import * as esbuild from 'esbuild';

// 开发构建
await esbuild.build({
  entryPoints: ['src/server.ts'],
  bundle: true,
  platform: 'node',
  target: 'node20',
  outdir: 'dist',
  format: 'esm',
  sourcemap: true,
  packages: 'external',  // 不打包 node_modules
});

// 生产构建
await esbuild.build({
  entryPoints: ['src/server.ts'],
  bundle: true,
  platform: 'node',
  target: 'node20',
  outdir: 'dist',
  format: 'esm',
  minify: true,
  treeShaking: true,
  packages: 'external',
});
```

### 2.3 使用 swc

```json
// .swcrc
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "jsc": {
    "parser": {
      "syntax": "typescript",
      "decorators": true
    },
    "target": "es2022"
  },
  "module": {
    "type": "es6"
  }
}
```

### 2.4 Package.json 配置

```json
{
  "name": "my-node-app",
  "type": "module",
  "main": "dist/server.js",
  "types": "dist/server.d.ts",
  "scripts": {
    "dev": "tsx watch src/server.ts",
    "build": "tsc",
    "build:fast": "node build.mjs",
    "start": "node dist/server.js",
    "type-check": "tsc --noEmit"
  },
  "files": ["dist"],
  "exports": {
    ".": {
      "import": "./dist/server.js",
      "types": "./dist/server.d.ts"
    }
  }
}
```

### 2.5 Docker 部署

```dockerfile
# Dockerfile
FROM node:20-slim AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM node:20-slim
WORKDIR /app
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/node_modules ./node_modules
COPY package*.json ./
EXPOSE 3000
CMD ["node", "dist/server.js"]
```

## 三、注意事项与常见陷阱

1. **tsc 是参考实现**：其他工具可能不支持所有 TS 特性（如 `const enum`）
2. **esbuild 不生成 `.d.ts`**：类型声明需要 tsc 单独生成
3. **`type: "module"` 要求 ESM**：导入需要 `.js` 扩展名
4. **生产部署不要包含 `src/`**：只部署 `dist/` 和 `node_modules`
5. **source map 用于调试**：生产环境可以不生成以减小体积
