# 中间件类型

## 一、概念说明

Express 中间件是处理请求/响应管道的函数。TypeScript 中中间件有专门的类型定义 `RequestHandler`，理解中间件的类型签名对于编写类型安全的中间件至关重要。

## 二、具体用法

### 2.1 中间件基本类型

```typescript
import { Request, Response, NextFunction, RequestHandler } from 'express';

// 中间件的完整类型签名
type RequestHandler<
  P = ParamsDictionary,
  ResBody = any,
  ReqBody = any,
  ReqQuery = qs.ParsedQs,
  Locals extends Record<string, any> = Record<string, any>
> = (
  req: Request<P, ResBody, ReqBody, ReqQuery, Locals>,
  res: Response<ResBody, Locals>,
  next: NextFunction
) => void | Promise<void>;

// 基本中间件
const logger: RequestHandler = (req, res, next) => {
  console.log(`${req.method} ${req.path}`);
  next();
};

app.use(logger);
```

### 2.2 带配置的中间件工厂

```typescript
// 中间件工厂函数
function rateLimit(options: { maxRequests: number; windowMs: number }): RequestHandler {
  const requests = new Map<string, number[]>();

  return (req, res, next) => {
    const ip = req.ip ?? 'unknown';
    const now = Date.now();
    const windowStart = now - options.windowMs;

    const userRequests = requests.get(ip)?.filter(t => t > windowStart) ?? [];

    if (userRequests.length >= options.maxRequests) {
      res.status(429).json({ error: '请求过于频繁' });
      return;
    }

    userRequests.push(now);
    requests.set(ip, userRequests);
    next();
  };
}

app.use(rateLimit({ maxRequests: 100, windowMs: 60000 }));
```

### 2.3 修改 Request 的中间件

```typescript
// 扩展 Request 类型
declare global {
  namespace Express {
    interface Request {
      userId?: number;
      startTime?: number;
    }
  }
}

// 认证中间件
const authenticate: RequestHandler = (req, res, next) => {
  const token = req.headers.authorization?.split(' ')[1];

  if (!token) {
    res.status(401).json({ error: '未提供 token' });
    return;
  }

  try {
    const payload = jwt.verify(token, process.env.JWT_SECRET);
    req.userId = (payload as { userId: number }).userId;
    next();
  } catch {
    res.status(401).json({ error: 'token 无效' });
  }
};

// 使用
app.get('/profile', authenticate, (req, res) => {
  // req.userId 类型是 number | undefined
  res.json({ userId: req.userId });
});
```

### 2.4 错误处理中间件

```typescript
import { ErrorRequestHandler } from 'express';

// 错误中间件有 4 个参数
const errorHandler: ErrorRequestHandler = (err, req, res, next) => {
  console.error(err.stack);

  if (err instanceof ValidationError) {
    res.status(400).json({ error: err.message });
    return;
  }

  res.status(500).json({ error: '服务器内部错误' });
};

// 注册在所有路由之后
app.use(errorHandler);
```

### 2.5 异步中间件

```typescript
// 异步中间件包装器
const asyncHandler = (
  fn: (req: Request, res: Response, next: NextFunction) => Promise<void>
): RequestHandler => {
  return (req, res, next) => {
    fn(req, res, next).catch(next);
  };
};

// 使用
app.get('/data', asyncHandler(async (req, res) => {
  const data = await fetchData();
  res.json(data);
}));
```

## 三、注意事项与常见陷阱

1. **错误中间件必须有 4 个参数**：`(err, req, res, next)` 否则 Express 不识别
2. **中间件中不要忘记调用 `next()`**：否则请求会挂起
3. **修改 Request 类型需要 `declare global`**：在 `.d.ts` 文件中声明
4. **异步中间件需要手动捕获错误**：Express 不自动处理 Promise rejection
5. **`res.json()` 后不要再调用 `next()`**：响应已发送
