# 文件系统类型

## 一、概念说明

Node.js 的 `fs` 模块提供了完整的文件系统操作 API，TypeScript 通过 `@types/node` 为这些 API 提供了类型定义。`fs/promises` 是推荐的异步 API，返回 Promise 并有更好的类型推断。

## 二、具体用法

### 2.1 基本文件操作

```typescript
import fs from 'node:fs/promises';
import path from 'node:path';

// 读取文件 — 返回 string
async function readTextFile(filePath: string): Promise<string> {
  return fs.readFile(filePath, 'utf-8');
}

// 写入文件
async function writeTextFile(filePath: string, content: string): Promise<void> {
  await fs.writeFile(filePath, content, 'utf-8');
}

// 读取 JSON 文件
async function readJsonFile<T>(filePath: string): Promise<T> {
  const content = await fs.readFile(filePath, 'utf-8');
  return JSON.parse(content) as T;
}

// 使用
interface Config {
  port: number;
  database: string;
}

const config = await readJsonFile<Config>('./config.json');
```

### 2.2 目录操作

```typescript
import fs from 'node:fs/promises';

// 列出目录内容 — 返回 Dirent[]（文件详细信息）
async function listFiles(dir: string): Promise<string[]> {
  const entries = await fs.readdir(dir, { withFileTypes: true });
  return entries
    .filter(entry => entry.isFile())
    .map(entry => entry.name);
}

// 递归获取所有文件
async function getAllFiles(dir: string): Promise<string[]> {
  const entries = await fs.readdir(dir, { withFileTypes: true });
  const files: string[] = [];

  for (const entry of entries) {
    const fullPath = path.join(dir, entry.name);
    if (entry.isDirectory()) {
      files.push(...await getAllFiles(fullPath));
    } else {
      files.push(fullPath);
    }
  }

  return files;
}

// 创建目录（递归）
await fs.mkdir('./data/output', { recursive: true });
```

### 2.3 文件信息

```typescript
import fs from 'node:fs/promises';

// 获取文件信息 — 返回 Stats
async function getFileInfo(filePath: string) {
  const stats = await fs.stat(filePath);

  return {
    size: stats.size,            // 文件大小（字节）
    isFile: stats.isFile(),      // 是否是文件
    isDirectory: stats.isDirectory(), // 是否是目录
    createdAt: stats.birthtime,  // 创建时间
    modifiedAt: stats.mtime,     // 修改时间
  };
}

// 检查文件是否存在
async function fileExists(filePath: string): Promise<boolean> {
  try {
    await fs.access(filePath);
    return true;
  } catch {
    return false;
  }
}
```

### 2.4 监听文件变化

```typescript
import fs from 'node:fs';
import { watch } from 'node:fs/promises';

// 使用 fs.watch（同步 API）
fs.watch('./config.json', (eventType: string, filename: string | null) => {
  console.log(`文件 ${filename} 发生 ${eventType} 事件`);
});

// 使用异步迭代器（推荐）
async function watchFile(filePath: string) {
  const watcher = watch(filePath);
  for await (const event of watcher) {
    console.log(event.eventType, event.filename);
  }
}
```

### 2.5 流式文件操作

```typescript
import fs from 'node:fs';
import { pipeline } from 'node:stream/promises';

// 流式复制大文件
async function copyFile(src: string, dest: string): Promise<void> {
  const readStream = fs.createReadStream(src);
  const writeStream = fs.createWriteStream(dest);
  await pipeline(readStream, writeStream);
}

// 逐行读取
import { createInterface } from 'node:readline';

async function readLines(filePath: string): Promise<string[]> {
  const lines: string[] = [];
  const rl = createInterface({
    input: fs.createReadStream(filePath),
    crlfDelay: Infinity,
  });

  for await (const line of rl) {
    lines.push(line);
  }

  return lines;
}
```

## 三、注意事项与常见陷阱

1. **优先使用 `fs/promises`**：异步 API 返回 Promise，避免回调地狱
2. **`fs.readFile` 的编码参数**：不传编码返回 `Buffer`，传 `'utf-8'` 返回 `string`
3. **`fs.stat` vs `fs.lstat`**：`lstat` 不跟随符号链接
4. **`fs.access` 用于检查存在性**：不要用 `fs.stat` 的 try-catch
5. **大文件用流**：`fs.createReadStream/createWriteStream` 避免内存溢出
