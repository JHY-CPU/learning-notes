# 环境变量类型

## 一、概念说明

`process.env` 的类型默认是 `Record<string, string | undefined>`，所有值都是 `string | undefined`。通过类型扩展，可以让环境变量有精确的类型定义，避免运行时因环境变量缺失导致的错误。

## 二、具体用法

### 2.1 扩展 process.env 类型

```typescript
// src/types/env.d.ts
declare namespace NodeJS {
  interface ProcessEnv {
    NODE_ENV: 'development' | 'production' | 'test';
    PORT: string;
    DATABASE_URL: string;
    JWT_SECRET: string;
    REDIS_URL?: string;           // 可选
    LOG_LEVEL: 'debug' | 'info' | 'warn' | 'error';
  }
}

// 使用
const port = parseInt(process.env.PORT, 10);     // PORT: string
const dbUrl = process.env.DATABASE_URL;           // DATABASE_URL: string
const redis = process.env.REDIS_URL;              // REDIS_URL: string | undefined
```

### 2.2 类型安全的配置加载器

```typescript
// 读取并验证环境变量
interface Config {
  nodeEnv: 'development' | 'production' | 'test';
  port: number;
  databaseUrl: string;
  jwtSecret: string;
  logLevel: 'debug' | 'info' | 'warn' | 'error';
}

function loadConfig(): Config {
  // 辅助函数：读取必需的环境变量
  const required = (name: string): string => {
    const value = process.env[name];
    if (!value) {
      throw new Error(`缺少必需的环境变量: ${name}`);
    }
    return value;
  };

  return {
    nodeEnv: (process.env.NODE_ENV ?? 'development') as Config['nodeEnv'],
    port: parseInt(process.env.PORT ?? '3000', 10),
    databaseUrl: required('DATABASE_URL'),
    jwtSecret: required('JWT_SECRET'),
    logLevel: (process.env.LOG_LEVEL ?? 'info') as Config['logLevel'],
  };
}

// 启动时加载
const config = loadConfig();
// config 的类型是 Config，所有属性都有明确类型
```

### 2.3 使用 zod 验证环境变量

```typescript
import { z } from 'zod';

// 定义 schema
const envSchema = z.object({
  NODE_ENV: z.enum(['development', 'production', 'test']),
  PORT: z.string().transform(Number),
  DATABASE_URL: z.string().url(),
  JWT_SECRET: z.string().min(32),
  REDIS_URL: z.string().url().optional(),
  LOG_LEVEL: z.enum(['debug', 'info', 'warn', 'error']).default('info'),
});

// 验证并获得类型安全的配置
type EnvConfig = z.infer<typeof envSchema>;
// 类型: { NODE_ENV: 'development' | 'production' | 'test'; PORT: number; ... }

function validateEnv(): EnvConfig {
  const result = envSchema.safeParse(process.env);

  if (!result.success) {
    console.error('环境变量验证失败:');
    result.error.issues.forEach(issue => {
      console.error(`  ${issue.path.join('.')}: ${issue.message}`);
    });
    process.exit(1);
  }

  return result.data;
}
```

### 2.4 Vite 环境变量类型

```typescript
// vite-env.d.ts
interface ImportMetaEnv {
  readonly VITE_API_BASE_URL: string;
  readonly VITE_APP_TITLE: string;
  readonly VITE_DEBUG: string;
}

interface ImportMeta {
  readonly env: ImportMetaEnv;
}

// 使用
const apiUrl = import.meta.env.VITE_API_BASE_URL; // 类型安全
```

## 三、注意事项与常见陷阱

1. **环境变量值始终是 `string`**：数字需要手动转换
2. **`undefined` vs 缺失**：`process.env.MISSING` 返回 `undefined`，不会报错
3. **`.env` 文件不自动加载**：需要 `dotenv` 或构建工具支持
4. **类型声明放在 `.d.ts` 文件中**：确保被 tsconfig 的 `include` 覆盖
5. **生产环境启动时验证所有环境变量**：避免运行时才发现缺失
