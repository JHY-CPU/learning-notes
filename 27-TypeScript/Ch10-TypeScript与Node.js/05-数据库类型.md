# 数据库类型

## 一、概念说明

TypeScript 数据库 ORM（Prisma、TypeORM、Drizzle）都提供了出色的类型支持。选择合适的 ORM 可以实现从数据库 schema 到应用代码的端到端类型安全。

**推荐排序**：Prisma > Drizzle > TypeORM（基于类型推断能力和开发体验）。

## 二、具体用法

### 2.1 Prisma（推荐）

```typescript
// schema.prisma
datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

generator client {
  provider = "prisma-client-js"
}

model User {
  id        Int      @id @default(autoincrement())
  name      String
  email     String   @unique
  posts     Post[]
  createdAt DateTime @default(now())
}

model Post {
  id        Int      @id @default(autoincrement())
  title     String
  content   String?
  author    User     @relation(fields: [authorId], references: [id])
  authorId  Int
}

// 代码中使用 — 所有类型自动生成
import { PrismaClient } from '@prisma/client';

const prisma = new PrismaClient();

// 查询 — 返回值类型是 Prisma.UserGetPayload<{}>
async function getUsers() {
  return prisma.user.findMany({
    where: { name: { contains: '张' } },
    include: { posts: true }, // 自动推断包含 posts 的类型
  });
}

// 创建 — data 参数类型自动生成
async function createUser(data: { name: string; email: string }) {
  return prisma.user.create({ data });
}
```

### 2.2 Drizzle ORM

```typescript
import { pgTable, serial, text, integer } from 'drizzle-orm/pg-core';
import { drizzle } from 'drizzle-orm/node-postgres';

// Schema 定义
export const users = pgTable('users', {
  id: serial('id').primaryKey(),
  name: text('name').notNull(),
  email: text('email').notNull().unique(),
});

export const posts = pgTable('posts', {
  id: serial('id').primaryKey(),
  title: text('title').notNull(),
  authorId: integer('author_id').references(() => users.id),
});

// 使用 — 类型从 schema 自动推断
const db = drizzle(pool);

async function getUsers() {
  return db.select().from(users); // 类型: { id: number; name: string; email: string }[]
}

// Insert 类型
type NewUser = typeof users.$inferInsert;
type User = typeof users.$inferSelect;
```

### 2.3 TypeORM

```typescript
import { Entity, PrimaryGeneratedColumn, Column, OneToMany } from 'typeorm';

@Entity()
export class User {
  @PrimaryGeneratedColumn()
  id: number;

  @Column()
  name: string;

  @Column({ unique: true })
  email: string;

  @OneToMany(() => Post, post => post.author)
  posts: Post[];
}

// Repository 使用
const userRepo = dataSource.getRepository(User);

async function findUser(id: number): Promise<User | null> {
  return userRepo.findOneBy({ id });
}
```

### 2.4 原生 SQL 查询类型

```typescript
import { Pool } from 'pg';

const pool = new Pool();

// 使用泛型标注查询结果
interface UserRow {
  id: number;
  name: string;
  email: string;
}

async function getUserById(id: number): Promise<UserRow | null> {
  const result = await pool.query<UserRow>(
    'SELECT id, name, email FROM users WHERE id = $1',
    [id]
  );
  return result.rows[0] ?? null;
}
```

## 三、注意事项与常见陷阱

1. **Prisma 的类型是自动生成的**：修改 schema 后必须运行 `npx prisma generate`
2. **Drizzle 的类型是推断的**：不需要代码生成，但需要正确导入 schema
3. **TypeORM 使用装饰器**：需要开启 `experimentalDecorators`
4. **原生 SQL 查询结果类型需要手动标注**：数据库驱动不会自动推断
5. **数据库连接池的类型**：使用 `pg.Pool` 或 ORM 提供的连接管理
