# 剩余参数类型

## 一、概念说明

剩余参数（Rest Parameters）使用 `...` 语法将不定数量的参数收集到一个数组中。在 TypeScript 中，剩余参数需要标注数组类型，如 `...args: number[]`。

## 二、具体用法

### 2.1 基本剩余参数

```typescript
function sum(...numbers: number[]): number {
  return numbers.reduce((total, n) => total + n, 0);
}

console.log(sum(1, 2, 3));       // 输出: 6
console.log(sum(10, 20, 30, 40)); // 输出: 100
console.log(sum());               // 输出: 0
```

### 2.2 剩余参数与必选参数混用

```typescript
function log(prefix: string, ...messages: string[]): void {
  messages.forEach((msg) => console.log(`[${prefix}] ${msg}`));
}

log("INFO", "服务器启动", "监听端口 3000");
// 输出:
// [INFO] 服务器启动
// [INFO] 监听端口 3000
```

### 2.3 元组剩余参数

```typescript
// 使用元组类型精确描述参数结构
function tupleArgs(...args: [string, number, boolean?]): void {
  const [name, age, active] = args;
  console.log(`${name}, ${age}, ${active ?? "未指定"}`);
}

tupleArgs("Alice", 25, true);
// 输出: Alice, 25, true
tupleArgs("Bob", 30);
// 输出: Bob, 30, 未指定
```

### 2.4 剩余参数类型推断

```typescript
// 剩余参数在类型推断中的应用
type Callback = (...args: any[]) => void;

function debounce(fn: Callback, delay: number): Callback {
  let timer: ReturnType<typeof setTimeout>;
  return (...args) => {
    clearTimeout(timer);
    timer = setTimeout(() => fn(...args), delay);
  };
}

const log2 = debounce((...args) => console.log(...args), 100);
log2("防抖输出", 42);
// 输出: 防抖输出 42（延迟100ms后）
```

## 三、注意事项与常见陷阱

1. **剩余参数必须是最后一个参数**：`f(...args: number[], x: string)` 不合法
2. **剩余参数是数组**：在函数体内是 `T[]` 类型
3. **元组剩余参数**：可以精确描述"前几个参数固定，后面不定"的模式
4. **`arguments` 不推荐**：在 TypeScript 中使用剩余参数代替 `arguments`
