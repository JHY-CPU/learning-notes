# 构造函数类型

## 一、概念说明

构造函数类型用于描述可以被 `new` 调用的函数（类的构造函数）。语法为 `new (...args) => T`。常用于泛型约束、工厂函数等场景，确保传入的值是一个可以实例化的类。构造函数类型在依赖注入和工厂模式中非常有用。

## 二、具体用法

### 2.1 基本构造函数类型

```typescript
class User {
  name: string;
  constructor(name: string) {
    this.name = name;
  }
}

// 构造函数类型
type UserConstructor = new (name: string) => User;

// 工厂函数接受构造函数
function createInstance(Constructor: UserConstructor, name: string): User {
  return new Constructor(name);
}

const user = createInstance(User, "Alice");
console.log(user.name);
// 输出: Alice
```

### 2.2 泛型构造函数约束

```typescript
// T 必须是可以 new 的构造函数
function instantiate<T extends new (...args: any[]) => any>(
  Constructor: T,
  ...args: ConstructorParameters<T>
): InstanceType<T> {
  return new Constructor(...args);
}

class Point {
  constructor(public x: number, public y: number) {}
}

const p = instantiate(Point, 10, 20);
console.log(p.x, p.y);
// 输出: 10 20
```

### 2.3 abstract 构造函数

```typescript
abstract class Animal {
  abstract speak(): void;
}

class Dog extends Animal {
  speak() { console.log("汪汪"); }
}

// abstract new 表示抽象构造函数
type AnimalConstructor = abstract new () => Animal;

function createAnimal(Ctor: AnimalConstructor): Animal {
  // return new Ctor(); // ❌ 抽象类不能直接实例化
  return new (Ctor as any)(); // 需要具体子类
}
```

### 2.4 工厂模式

```typescript
// 注册多个构造函数，按名称创建实例
class ServiceA {
  name = "ServiceA";
  run() { console.log(`${this.name} 运行`); }
}

class ServiceB {
  name = "ServiceB";
  run() { console.log(`${this.name} 运行`); }
}

type ServiceConstructor = new () => { name: string; run(): void };

const registry: Record<string, ServiceConstructor> = {
  a: ServiceA,
  b: ServiceB,
};

function createService(name: string) {
  const Ctor = registry[name];
  if (!Ctor) throw new Error(`未知服务: ${name}`);
  return new Ctor();
}

createService("a").run(); // 输出: ServiceA 运行
```

### 2.5 内置工具类型

```typescript
// ConstructorParameters<T> —— 提取构造函数参数类型
class Server {
  constructor(public host: string, public port: number) {}
}

type ServerParams = ConstructorParameters<typeof Server>;
// [string, number]

// InstanceType<T> —— 提取实例类型
type ServerInstance = InstanceType<typeof Server>;
// Server

// 应用
function createServer(...args: ConstructorParameters<typeof Server>): InstanceType<typeof Server> {
  return new Server(...args);
}
```

## 三、注意事项与常见陷阱

1. **`new` 是必须的**：构造函数类型必须有 `new` 关键字，区别于普通函数类型
2. **`ConstructorParameters<T>`**：提取构造函数参数类型，避免手动重复
3. **`InstanceType<T>`**：提取构造函数实例类型，用于泛型返回值
4. **接口不能 new**：只有类有构造函数类型，接口没有
5. **抽象类用 `abstract new`**：表示不能直接实例化的构造函数
6. **`typeof` 获取构造函数类型**：`typeof MyClass` 返回类的构造函数类型
