# 回调函数类型

## 一、概念说明

回调函数是作为参数传递给另一个函数的函数。在 TypeScript 中，需要为回调参数定义精确的类型签名，包括参数类型和返回值类型。正确类型化的回调可以捕获参数不匹配等常见错误。回调类型是异步编程、事件处理和高阶函数的基础。

## 二、具体用法

### 2.1 基本回调类型

```typescript
// 回调函数类型注解
function fetchData(url: string, callback: (data: string) => void): void {
  // 模拟异步获取数据
  const data = `来自 ${url} 的数据`;
  callback(data);
}

fetchData("/api/users", (data) => {
  console.log(`收到: ${data}`);
});
// 输出: 收到: 来自 /api/users 的数据
```

### 2.2 使用类型别名

```typescript
type SuccessCallback<T> = (data: T) => void;
type ErrorCallback = (error: Error) => void;

function apiCall<T>(
  endpoint: string,
  onSuccess: SuccessCallback<T>,
  onError: ErrorCallback
): void {
  try {
    const data = { id: 1, name: "Alice" } as T;
    onSuccess(data);
  } catch (e) {
    onError(e as Error);
  }
}

apiCall<{ id: number; name: string }>(
  "/users/1",
  (user) => console.log(`用户: ${user.name}`),
  (err) => console.log(`错误: ${err.message}`)
);
// 输出: 用户: Alice
```

### 2.3 高阶函数类型

```typescript
type Transform<T, U> = (input: T) => U;

function pipe<A, B, C>(
  value: A,
  fn1: Transform<A, B>,
  fn2: Transform<B, C>
): C {
  return fn2(fn1(value));
}

const result = pipe(
  "hello",
  (s) => s.length,     // string -> number
  (n) => n * 2         // number -> number
);

console.log(result);
// 输出: 10
```

### 2.4 事件回调模式

```typescript
// 事件回调接口
interface EventHandlers {
  onClick?: (x: number, y: number) => void;
  onHover?: (element: string) => void;
  onFocus?: () => void;
  onBlur?: () => void;
}

function createComponent(handlers: EventHandlers) {
  // 模拟触发事件
  handlers.onClick?.(10, 20);
  handlers.onHover?.("button");
  handlers.onFocus?.();
}

createComponent({
  onClick: (x, y) => console.log(`点击位置: ${x}, ${y}`),
  onFocus: () => console.log("获得焦点"),
});
// 输出:
// 点击位置: 10, 20
// 获得焦点
```

### 2.5 Node.js 风格回调

```typescript
// Node.js 错误优先回调
type NodeCallback<T> = (err: Error | null, result?: T) => void;

function readFile(path: string, callback: NodeCallback<string>): void {
  // 模拟读取文件
  if (path === "/exists") {
    callback(null, "文件内容");
  } else {
    callback(new Error("文件不存在"));
  }
}

readFile("/exists", (err, data) => {
  if (err) {
    console.error(`错误: ${err.message}`);
  } else {
    console.log(`内容: ${data}`);
  }
});
// 输出: 内容: 文件内容
```

## 三、注意事项与常见陷阱

1. **回调的 `void` 返回类型**：声明 `() => void` 不阻止回调返回值，只是忽略它
2. **可选回调**：`callback?: (data: T) => void` 表示回调可选，调用前需检查
3. **回调中的 this**：注意 `this` 在回调中的指向问题，优先使用箭头函数
4. **异步回调 vs Promise**：现代代码优先使用 `async/await`，而非嵌套回调
5. **回调地狱**：深层嵌套回调难以维护，考虑用 Promise 或 async/await 替代
6. **类型推断**：回调参数的类型通常能从上下文推断，不需要显式注解
