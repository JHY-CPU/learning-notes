# 回调函数类型

## 一、概念说明

回调函数是作为参数传递给另一个函数的函数。在 TypeScript 中，需要为回调参数定义精确的类型签名，包括参数类型和返回值类型。正确类型化的回调可以捕获参数不匹配等常见错误。

## 二、具体用法

### 2.1 基本回调类型

```typescript
// 回调函数类型注解
function fetchData(url: string, callback: (data: string) => void): void {
  // 模拟异步获取数据
  const data = `来自 ${url} 的数据`;
  callback(data);
}

fetchData("/api/users", (data) => {
  console.log(`收到: ${data}`);
});
// 输出: 收到: 来自 /api/users 的数据
```

### 2.2 使用类型别名

```typescript
type SuccessCallback<T> = (data: T) => void;
type ErrorCallback = (error: Error) => void;

function apiCall<T>(
  endpoint: string,
  onSuccess: SuccessCallback<T>,
  onError: ErrorCallback
): void {
  try {
    const data = { id: 1, name: "Alice" } as T;
    onSuccess(data);
  } catch (e) {
    onError(e as Error);
  }
}

apiCall<{ id: number; name: string }>(
  "/users/1",
  (user) => console.log(`用户: ${user.name}`),
  (err) => console.log(`错误: ${err.message}`)
);
// 输出: 用户: Alice
```

### 2.3 高阶函数类型

```typescript
type Transform<T, U> = (input: T) => U;

function pipe<A, B, C>(
  value: A,
  fn1: Transform<A, B>,
  fn2: Transform<B, C>
): C {
  return fn2(fn1(value));
}

const result = pipe(
  "hello",
  (s) => s.length,     // string -> number
  (n) => n * 2         // number -> number
);

console.log(result);
// 输出: 10
```

## 三、注意事项与常见陷阱

1. **回调的 `void` 返回类型**：声明 `() => void` 不阻止回调返回值
2. **可选回调**：`callback?: (data: T) => void` 表示回调可选
3. **回调中的 this**：注意 `this` 在回调中的指向问题
4. **异步回调**：异步回调应返回 `Promise<void>`
