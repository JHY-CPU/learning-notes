# 柯里化函数类型

## 一、概念说明

柯里化（Currying）是将多参数函数转换为一系列单参数函数的技术。在 TypeScript 中，柯里化函数的类型需要逐层描述每个参数的返回函数类型，形成链式函数类型。柯里化在函数式编程中常用于创建可复用的偏函数。

## 二、具体用法

### 2.1 基本柯里化

```typescript
// 手动柯里化
function add(a: number): (b: number) => number {
  return (b) => a + b;
}

const add5 = add(5);
console.log(add5(3));  // 输出: 8
console.log(add5(10)); // 输出: 15
console.log(add(2)(7)); // 输出: 9
```

### 2.2 泛型柯里化类型

```typescript
// 柯里化工具类型
type Curried<A, B, R> = (a: A) => (b: B) => R;

function curry2<A, B, R>(
  fn: (a: A, b: B) => R
): Curried<A, B, R> {
  return (a) => (b) => fn(a, b);
}

function multiply(a: number, b: number): number {
  return a * b;
}

const curriedMultiply = curry2(multiply);
const double = curriedMultiply(2);

console.log(double(5));   // 输出: 10
console.log(double(10));  // 输出: 20
```

### 2.3 部分应用

```typescript
type PartialApply<F, A> = F extends (a: A, ...rest: infer R) => infer Ret
  ? (...rest: R) => Ret
  : never;

function partial<A, R>(fn: (a: A) => R, arg: A): () => R {
  return () => fn(arg);
}

function greet(name: string, greeting: string): string {
  return `${greeting}, ${name}!`;
}

// 绑定第一个参数
const greetAlice = (greeting: string) => greet("Alice", greeting);
console.log(greetAlice("Hi"));
// 输出: Hi, Alice!
```

### 2.4 实际应用：配置函数

```typescript
// 日志柯里化
function createLogger(prefix: string) {
  return (level: "info" | "warn" | "error") => {
    return (message: string) => {
      console.log(`[${prefix}] [${level}] ${message}`);
    };
  };
}

const appLogger = createLogger("APP");
const info = appLogger("info");
const error = appLogger("error");

info("启动成功");      // 输出: [APP] [info] 启动成功
error("连接失败");     // 输出: [APP] [error] 连接失败

// 一行调用
createLogger("DB")("warn")("查询超时"); // 输出: [DB] [warn] 查询超时
```

### 2.5 自动柯里化（高级）

```typescript
// 通用柯里化工具类型（简化版）
type Curry<A, B, C, R> = (a: A) => (b: B) => (c: C) => R;

function curry3<A, B, C, R>(
  fn: (a: A, b: B, c: C) => R
): Curry<A, B, C, R> {
  return (a) => (b) => (c) => fn(a, b, c);
}

function formatPrice(currency: string, symbol: string, amount: number): string {
  return `${currency} ${symbol}${amount}`;
}

const formatCNY = curry3(formatPrice)("CNY")("¥");
console.log(formatCNY(99.99));  // 输出: CNY ¥99.99
console.log(formatCNY(199.50)); // 输出: CNY ¥199.5
```

## 三、注意事项与常见陷阱

1. **类型复杂度**：多参数柯里化类型签名会变得复杂，影响可读性
2. **推断可能失败**：泛型柯里化时类型推断可能不如预期，需要显式注解
3. **实际使用较少**：TypeScript 中柯里化不如 Haskell 等函数式语言常见
4. **`bind` 方法**：JavaScript 的 `bind` 也是一种部分应用，但类型支持有限
5. **性能考虑**：每次柯里化都创建新闭包，频繁调用可能有性能开销
6. **可读性权衡**：过度柯里化可能降低代码可读性，应平衡简洁和清晰
