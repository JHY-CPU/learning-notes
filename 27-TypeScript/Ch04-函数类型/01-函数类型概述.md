# 函数类型概述

## 一、概念说明

TypeScript 中的函数类型描述了函数的**参数类型**和**返回值类型**。可以通过类型注解、类型别名、接口等方式定义函数类型。正确使用函数类型可以让编译器捕获参数错误、返回值错误等常见 bug。

## 二、具体用法

### 2.1 函数类型注解

```typescript
// 完整的函数类型注解
function add(a: number, b: number): number {
  return a + b;
}

// 箭头函数类型注解
const multiply = (a: number, b: number): number => a * b;

console.log(add(3, 5));      // 输出: 8
console.log(multiply(4, 6)); // 输出: 24
```

### 2.2 函数类型表达式

```typescript
// 使用函数类型表达式定义类型
type BinaryFn = (a: number, b: number) => number;

const add: BinaryFn = (a, b) => a + b;
const subtract: BinaryFn = (a, b) => a - b;

console.log(add(10, 3));      // 输出: 13
console.log(subtract(10, 3)); // 输出: 7
```

### 2.3 Contextual Typing（上下文类型推断）

```typescript
// 回调函数的参数类型从上下文推断
const numbers = [1, 2, 3, 4, 5];

// item 和 index 的类型自动推断
const doubled = numbers.map((item, index) => {
  return `${index}: ${item * 2}`;
});

console.log(doubled);
// 输出: ["0: 2", "1: 4", "2: 6", "3: 8", "4: 10"]
```

## 三、注意事项与常见陷阱

1. **返回值类型可推断**：大多数情况下不必显式标注返回类型
2. **公共 API 建议显式标注**：导出的函数应明确标注返回类型
3. **`void` vs `undefined`**：`void` 表示不使用返回值，`undefined` 是具体值
4. **函数类型是结构化的**：只要签名匹配，不同定义的函数类型互相兼容
