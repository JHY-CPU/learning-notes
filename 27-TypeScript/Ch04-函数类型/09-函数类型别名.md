# 函数类型别名

## 一、概念说明

使用 `type` 关键字为函数类型创建别名，可以提高代码可读性和复用性。函数类型别名常用于定义回调、事件处理器、中间件等。类型别名比接口定义函数类型更简洁，且支持联合类型、交叉类型等更灵活的组合方式。

## 二、具体用法

### 2.1 基本函数类型别名

```typescript
type MathFn = (a: number, b: number) => number;

const add: MathFn = (a, b) => a + b;
const multiply: MathFn = (a, b) => a * b;
const power: MathFn = (a, b) => a ** b;

console.log(add(3, 5));       // 输出: 8
console.log(multiply(4, 6));  // 输出: 24
console.log(power(2, 10));    // 输出: 1024
```

### 2.2 泛型函数类型别名

```typescript
type Mapper<T, U> = (item: T, index: number) => U;
type Predicate<T> = (item: T, index: number) => boolean;
type Reducer<T, U> = (acc: U, current: T, index: number) => U;

function customMap<T, U>(arr: T[], mapper: Mapper<T, U>): U[] {
  return arr.map(mapper);
}

function customFilter<T>(arr: T[], predicate: Predicate<T>): T[] {
  return arr.filter(predicate);
}

const doubled = customMap([1, 2, 3], (n) => n * 2);
const evens = customFilter([1, 2, 3, 4], (n) => n % 2 === 0);

console.log(doubled); // 输出: [2, 4, 6]
console.log(evens);   // 输出: [2, 4]
```

### 2.3 异步函数类型别名

```typescript
type AsyncHandler<T, R> = (input: T) => Promise<R>;
type Middleware<T> = (context: T, next: () => Promise<void>) => Promise<void>;

const fetchUser: AsyncHandler<number, { id: number; name: string }> = async (id) => {
  return { id, name: "Alice" };
};

(async () => {
  const user = await fetchUser(1);
  console.log(user.name);
})();
// 输出: Alice
```

### 2.4 函数联合类型

```typescript
// type 的独特能力：函数联合类型
type StringOrNumberHandler =
  | ((input: string) => string)
  | ((input: number) => number);

// 条件函数类型
type ConditionalHandler<T> = T extends string
  ? (input: string) => string
  : (input: number) => number;

// 重载函数类型
type OverloadedFn = {
  (x: string): string;
  (x: number): number;
};
```

### 2.5 中间件模式

```typescript
// Express/Koa 风格中间件类型
type Context = {
  request: { url: string; method: string };
  response: { status: number; body: unknown };
};

type Next = () => Promise<void>;

type MiddlewareFn = (ctx: Context, next: Next) => Promise<void>;

const logging: MiddlewareFn = async (ctx, next) => {
  console.log(`→ ${ctx.request.method} ${ctx.request.url}`);
  await next();
  console.log(`← ${ctx.response.status}`);
};

const auth: MiddlewareFn = async (ctx, next) => {
  // 检查认证...
  await next();
};

// 组合中间件
const middlewares: MiddlewareFn[] = [logging, auth];
```

## 三、注意事项与常见陷阱

1. **与 interface 函数类型的选择**：两者功能相同，`type` 更简洁，推荐使用 `type`
2. **类型别名不支持声明合并**：同名 `type` 会报错，这是与 interface 的主要区别
3. **泛型参数可推断**：大多数情况下不需要显式指定泛型参数
4. **命名应有描述性**：`AsyncHandler` 比 `Fn` 更具可读性
5. **类型别名支持联合**：`type Fn = A | B` 是 type 的独特优势
6. **性能**：复杂函数类型别名对类型检查性能影响极小
