# 函数类型推断

## 一、概念说明

TypeScript 可以根据**上下文**和**参数类型**自动推断函数的返回值类型、泛型参数类型和回调函数参数类型。良好的类型推断减少显式注解，同时保持类型安全。理解推断机制有助于在正确的地方添加注解。

## 二、具体用法

### 2.1 返回类型推断

```typescript
// TypeScript 自动推断返回类型为 number
function add(a: number, b: number) {
  return a + b;
}
// 推断: (a: number, b: number) => number

const result = add(3, 5);
console.log(result);
// 输出: 8
```

### 2.2 泛型推断

```typescript
// T 从参数自动推断
function identity<T>(arg: T): T {
  return arg;
}

const s = identity("hello");  // T = string
const n = identity(42);       // T = number

console.log(typeof s, typeof n);
// 输出: string number
```

### 2.3 上下文类型推断

```typescript
// 回调函数参数类型从上下文推断
type Handler = (event: { type: string; data: unknown }) => void;

const handler: Handler = (event) => {
  // event 类型自动推断为 { type: string; data: unknown }
  console.log(`${event.type}: ${event.data}`);
};

handler({ type: "click", data: { x: 10 } });
// 输出: click: [object Object]
```

### 2.4 最佳通用类型

```typescript
// TypeScript 找到所有元素的最通用类型
const arr = [1, "hello", true];
// 推断为 (number | string | boolean)[]

// 有共同基类时使用基类
class Animal {}
class Dog extends Animal {}
class Cat extends Animal {}

const animals = [new Dog(), new Cat()];
// 推断为 Animal[]

console.log(arr.length, animals.length);
// 输出: 3 2
```

### 2.5 推断失败与显式注解

```typescript
// 空数组推断为 never[]
const empty = [];
// empty.push(1); // ❌ 需要显式注解

// 解决方案
const nums: number[] = [];
nums.push(1); // ✅

// 复杂泛型推断失败时需要显式指定
function map<T, U>(arr: T[], fn: (item: T) => U): U[] {
  return arr.map(fn);
}

// 推断成功
const lengths = map(["hello", "world"], (s) => s.length); // number[]

// 推断可能不理想时显式注解
const result = map<string, { value: string }>(
  ["a", "b"],
  (s) => ({ value: s })
);
```

### 2.6 返回类型推断的陷阱

```typescript
// ❌ 返回类型推断过宽
function createNumber() {
  return 42; // 推断为 number
}

// ✅ 使用 as const 或显式注解
function createExactNumber() {
  return 42 as const; // 推断为 42（字面量类型）
}

// ❌ 函数重载推断可能不精确
function process(input: string | number) {
  if (typeof input === "string") {
    return input.toUpperCase(); // 推断为 string
  }
  return input.toFixed(2); // 推断为 string
}
// 返回类型是 string，丢失了重载信息
```

## 三、注意事项与常见陷阱

1. **推断不如注解精确**：某些情况下推断的类型过于宽泛（如返回 `string` 而非字面量）
2. **泛型推断失败时回退**：推断失败会回退到约束类型或 `{}`
3. **上下文类型传播**：赋值和参数传递时类型信息会"向下"传播到回调函数
4. **显式优于隐式**：复杂场景下显式注解更安全，尤其是公共 API
5. **`const` 上下文**：使用 `as const` 可以让推断更加精确
6. **类型推断的顺序**：从左到右推断参数，但返回类型在函数体执行后才确定
