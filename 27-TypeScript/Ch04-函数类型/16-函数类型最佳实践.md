# 函数类型最佳实践

## 一、概念说明

编写函数类型时遵循最佳实践，可以提高代码的类型安全性和可维护性。核心原则包括：显式标注公共 API 的返回类型、避免 `any` 参数、使用泛型而非联合类型等。

## 二、具体用法

### 2.1 显式返回类型

```typescript
// ❌ 不好：导出函数没有返回类型
export function calculateTotal(items: { price: number }[]) {
  return items.reduce((sum, item) => sum + item.price, 0);
}

// ✅ 好：显式标注返回类型
export function calculateTotal2(items: { price: number }[]): number {
  return items.reduce((sum, item) => sum + item.price, 0);
}

console.log(calculateTotal2([{ price: 10 }, { price: 20 }]));
// 输出: 30
```

### 2.2 避免 any 参数

```typescript
// ❌ 不好
// function process(data: any): any { return data; }

// ✅ 好：使用 unknown + 类型守卫
function process(data: unknown): string {
  if (typeof data === "string") return data;
  if (typeof data === "number") return data.toString();
  return JSON.stringify(data);
}

console.log(process("hello")); // 输出: hello
console.log(process(42));      // 输出: 42
```

### 2.3 使用泛型而非联合

```typescript
// ❌ 不好：返回类型不确定
function first(arr: (string | number)[]): string | number | undefined {
  return arr[0];
}

// ✅ 好：泛型保持类型精确
function first<T>(arr: T[]): T | undefined {
  return arr[0];
}

const n = first([1, 2, 3]);    // number | undefined
const s = first(["a", "b"]);   // string | undefined

console.log(n, s);
// 输出: 1 "a"
```

### 2.4 参数对象优于长参数列表

```typescript
// ❌ 不好：参数太多
// function create(host: string, port: number, ssl: boolean, timeout: number, retries: number) {}

// ✅ 好：使用选项对象
interface CreateOptions {
  host: string;
  port: number;
  ssl?: boolean;
  timeout?: number;
}

function create(options: CreateOptions): string {
  return `${options.ssl ? "https" : "http"}://${options.host}:${options.port}`;
}

console.log(create({ host: "api.com", port: 443, ssl: true }));
// 输出: https://api.com:443
```

## 三、注意事项与常见陷阱

1. **导出函数必须显式返回类型**：防止重构时返回类型意外变化
2. **`unknown` 优于 `any`**：保持类型安全
3. **泛型优于联合**：保持类型精确
4. **选项对象优于长参数列表**：提升可读性和可扩展性
