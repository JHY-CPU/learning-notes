# 可调用接口

## 一、概念说明

可调用接口（Callable Interface）是包含**调用签名**的接口，使接口描述的对象可以像函数一样被调用。同时可以拥有属性和方法，创建"增强函数"。

## 二、具体用法

### 2.1 基本调用签名

```typescript
interface Greeter {
  (name: string): string;
}

const hello: Greeter = (name) => `Hello, ${name}!`;
console.log(hello("World"));
// 输出: Hello, World!
```

### 2.2 带属性的可调用接口

```typescript
interface Logger {
  // 调用签名
  (message: string): void;
  // 属性
  level: "debug" | "info" | "warn" | "error";
  // 方法
  setLevel(level: "debug" | "info" | "warn" | "error"): void;
}

function createLogger(): Logger {
  const logger = ((message: string) => {
    console.log(`[${logger.level}] ${message}`);
  }) as Logger;

  logger.level = "info";
  logger.setLevel = (level) => { logger.level = level; };

  return logger;
}

const log = createLogger();
log("普通消息");
log.setLevel("debug");
log("调试消息");
// 输出:
// [info] 普通消息
// [debug] 调试消息
```

### 2.3 构造签名

```typescript
interface Constructable {
  new (name: string): { name: string };
}

class User {
  constructor(public name: string) {}
}

const factory: Constructable = User;
const user = new factory("Alice");
console.log(user.name);
// 输出: Alice
```

## 三、注意事项与常见陷阱

1. **调用签名语法**：`(参数) => 返回值`，与函数类型表达式略有不同
2. **可调用接口可有多个重载签名**
3. **构造签名用 `new`**：区分普通调用和构造调用
4. **与函数类型别名功能相似**：选择哪种取决于是否需要属性
