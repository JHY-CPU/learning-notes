# 泛型约束

## 一、概念说明

泛型约束使用 `extends` 关键字限制类型参数必须满足特定条件。这允许在泛型函数内部安全地访问约束类型的属性和方法，而不必担心类型错误。

## 二、具体用法

### 2.1 基本泛型约束

```typescript
// T 必须有 length 属性
interface HasLength {
  length: number;
}

function logLength<T extends HasLength>(arg: T): T {
  console.log(`长度: ${arg.length}`);
  return arg;
}

logLength("hello");      // string 有 length
logLength([1, 2, 3]);    // 数组有 length
logLength({ length: 5, name: "test" }); // 对象有 length
// logLength(42);        // ❌ number 没有 length

// 输出: 长度: 5
// 输出: 长度: 3
// 输出: 长度: 5
```

### 2.2 keyof 约束

```typescript
// K 必须是 T 的键
function getProperty<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}

const user = { id: 1, name: "Alice", age: 25 };

console.log(getProperty(user, "name")); // 输出: Alice
console.log(getProperty(user, "age"));  // 输出: 25
// getProperty(user, "email");           // ❌ "email" 不是 User 的键
```

### 2.3 多重约束

```typescript
interface Printable {
  print(): void;
}

interface Loggable {
  log(): void;
}

// T 必须同时满足 Printable 和 Loggable
function process<T extends Printable & Loggable>(item: T): void {
  item.print();
  item.log();
}

const logger: Printable & Loggable = {
  print() { console.log("打印"); },
  log() { console.log("日志"); },
};

process(logger);
// 输出:
// 打印
// 日志
```

### 2.4 约束中的默认类型

```typescript
function createPair<T extends string | number = string>(
  value: T
): [T, T] {
  return [value, value];
}

const p1 = createPair("hello");   // T = string
const p2 = createPair(42);        // T = number
const p3 = createPair();          // T = string（默认值）

console.log(p1, p2, p3);
// 输出: ["hello", "hello"] [42, 42] ["", ""]
```

## 三、注意事项与常见陷阱

1. **约束限制灵活性**：过于严格的约束限制了泛型的通用性
2. **`extends` 不是继承**：这里是类型约束，不是类继承
3. **约束后可访问属性**：`T extends HasLength` 后可访问 `T.length`
4. **`keyof` 约束最常用**：确保 key 参数是对象的有效键
