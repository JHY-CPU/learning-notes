# 类型声明文件 .d.ts

## 一、概念说明

`.d.ts` 文件是 TypeScript 的**类型声明文件**，它只包含类型信息，不包含实现代码。主要用于：为纯 JavaScript 库提供类型定义、声明全局变量和模块的类型。通过 `@types` npm 包可以为大量 JS 库获取社区维护的类型声明。

## 二、具体用法

### 2.1 声明文件的基本结构

```typescript
// types/global.d.ts
// 声明全局变量
declare const API_URL: string;

// 声明全局函数
declare function greet(name: string): string;

// 声明全局接口
interface AppConfig {
  debug: boolean;
  version: string;
}
```

### 2.2 为 JS 模块声明类型

```typescript
// types/lodash.d.ts
declare module "lodash" {
  export function debounce<T extends (...args: any[]) => any>(
    func: T,
    wait?: number
  ): T & { cancel(): void };

  export function uniq<T>(array: T[]): T[];
}
```

### 2.3 安装 @types 包

```bash
# 安装社区类型声明（以 lodash 为例）
npm install -D @types/lodash

# 安装后即可获得类型提示
```

```typescript
import { debounce } from "lodash";

// 现在有完整的类型提示
const debouncedLog = debounce((msg: string) => {
  console.log(msg);
}, 300);

debouncedLog("Hello");
// 输出: Hello（延迟300ms后输出）
```

## 三、注意事项与常见陷阱

1. **声明文件不生成 JS 代码**：`.d.ts` 仅用于类型检查，编译时被忽略
2. **`@types` 包自动包含**：安装后 TypeScript 自动识别，无需额外配置
3. **自定义声明优先**：项目内的 `.d.ts` 可覆盖 `@types` 中的定义
4. **避免在声明文件中写实现**：声明文件只应包含 `declare` 语句
