# 开发环境搭建

## 一、概念说明

搭建 TypeScript 开发环境需要三个核心组件：**Node.js**（运行时环境）、**TypeScript 编译器**（`tsc`）和 **VS Code**（推荐编辑器）。本节详细介绍从零开始配置完整的 TypeScript 开发环境。

## 二、具体用法

### 2.1 安装 Node.js 与 TypeScript

```bash
# 1. 安装 Node.js（访问 nodejs.org 下载 LTS 版本）
# 验证安装
node -v   # v20.x.x
npm -v    # 10.x.x

# 2. 全局安装 TypeScript 编译器
npm install -g typescript

# 验证安装
tsc -v    # Version 5.x.x
```

### 2.2 项目初始化

```bash
# 创建项目目录并初始化
mkdir my-ts-project && cd my-ts-project
npm init -y

# 安装 TypeScript 为项目依赖（推荐）
npm install -D typescript

# 生成 tsconfig.json 配置文件
npx tsc --init
```

### 2.3 编写并编译第一个文件

```typescript
// src/hello.ts
const greeting: string = "Hello, TypeScript!";
const year: number = 2024;

console.log(`${greeting} 欢迎来到 ${year} 年`);
```

```bash
# 编译 TypeScript 文件
tsc src/hello.ts

# 运行编译后的 JavaScript
node src/hello.js
```

**输出：**
```
Hello, TypeScript! 欢迎来到 2024 年
```

### 2.4 VS Code 插件推荐

```json
// VS Code 扩展推荐
{
  "extensions": [
    "dbaeumer.vscode-eslint",        // ESLint
    "esbenp.prettier-vscode",         // Prettier
    "christian-kohler.path-intellisense", // 路径补全
    "formulahendry.auto-rename-tag"   // HTML标签同步
  ]
}
```

**输出：**
```
安装后在 VS Code 中打开 .ts 文件即可获得类型检查和智能提示
```

## 三、注意事项与常见陷阱

1. **全局 vs 局部安装**：推荐将 TypeScript 安装为项目依赖，避免版本冲突
2. **Node.js 版本**：确保使用 LTS 版本，TypeScript 5.x 要求 Node.js 12+
3. **编译输出目录**：在 `tsconfig.json` 中配置 `outDir` 指定输出目录
4. **`npx tsc` 优先级**：项目本地安装的 `tsc` 优先于全局版本
5. **VS Code 内置 TypeScript**：VS Code 自带 TypeScript 支持，但版本可能滞后
