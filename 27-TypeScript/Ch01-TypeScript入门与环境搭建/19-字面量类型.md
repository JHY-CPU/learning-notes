# 字面量类型

## 一、概念说明

字面量类型（Literal Types）允许将变量类型限制为**特定的字面量值**，而非宽泛的基础类型。可以是字符串字面量、数字字面量或布尔字面量。常与联合类型结合，创建精确的类型约束。字面量类型是 TypeScript 类型系统表达力的重要体现。

## 二、具体用法

### 2.1 字符串字面量类型

```typescript
// 只能是特定字符串
type HttpMethod = "GET" | "POST" | "PUT" | "DELETE";

function request(url: string, method: HttpMethod): void {
  console.log(`${method} ${url}`);
}

request("/api/users", "GET");   // ✅
request("/api/users", "POST");  // ✅
// request("/api/users", "PATCH"); // ❌ 编译错误
```

**输出：**
```
GET /api/users
POST /api/users
```

### 2.2 数字字面量类型

```typescript
type DiceRoll = 1 | 2 | 3 | 4 | 5 | 6;

function rollDice(): DiceRoll {
  return Math.ceil(Math.random() * 6) as DiceRoll;
}

const result = rollDice();
console.log(`掷出了: ${result}`);
// 输出: 掷出了: 4（随机值）
```

### 2.3 const 断言产生字面量类型

```typescript
// let 推断为宽泛类型
let mutable = "hello"; // 类型: string

// const 推断为字面量类型
const literal = "hello"; // 类型: "hello"

// as const 强制字面量推断
const config = {
  port: 3000,
  host: "localhost",
} as const;
// 类型: { readonly port: 3000; readonly host: "localhost" }

console.log(config.port);
// 输出: 3000
```

### 2.4 布尔字面量类型

```typescript
// 极少使用，但技术上可行
type True = true;
type False = false;

// 更实用的场景：精确的配置类型
type Config = {
  enableCache: true;
  cacheTTL: number;
} | {
  enableCache: false;
};

function setup(config: Config) {
  if (config.enableCache) {
    console.log(`缓存已启用，TTL: ${config.cacheTTL}`);
  } else {
    console.log("缓存已禁用");
  }
}
```

### 2.5 模板字面量类型

```typescript
// TypeScript 4.1+ 支持模板字面量类型
type Color = "red" | "blue" | "green";
type Size = "sm" | "md" | "lg";

// 组合生成所有可能的类名
type ClassName = `${Color}-${Size}`;
// "red-sm" | "red-md" | "red-lg" | "blue-sm" | ...

function applyClass(cls: ClassName): void {
  console.log(`应用样式: ${cls}`);
}

applyClass("red-md"); // ✅
// applyClass("yellow-sm"); // ❌ 编译错误
```

### 2.6 字面量类型在 API 设计中的应用

```typescript
// 事件类型精确定义
type AppEvent =
  | { type: "click"; x: number; y: number }
  | { type: "keypress"; key: string }
  | { type: "scroll"; delta: number };

function handleEvent(event: AppEvent) {
  switch (event.type) {
    case "click":
      console.log(`点击位置: (${event.x}, ${event.y})`);
      break;
    case "keypress":
      console.log(`按键: ${event.key}`);
      break;
    case "scroll":
      console.log(`滚动偏移: ${event.delta}`);
      break;
  }
}
```

## 三、注意事项与常见陷阱

1. **`const` vs `let`**：`const` 自动推断为字面量类型，`let` 推断为宽泛类型
2. **`as const` 的影响**：会使整个对象变为 `readonly`，属性不可修改
3. **字面量类型是基础类型的子类型**：`"hello"` 是 `string` 的子类型
4. **联合字面量类型**：可用于创建枚举的替代方案，不生成额外代码
5. **类型收窄产生字面量类型**：通过 `===` 比较可以收窄到字面量类型
6. **数字字面量的注意点**：浮点数不能作为字面量类型的成员（如 `1.5 | 2.5` 不实用）
