# 类型检查原理

## 一、概念说明

TypeScript 使用**结构类型**（Structural Typing）系统，也称为**鸭子类型**（Duck Typing）。这意味着类型的兼容性取决于其"结构"（有哪些属性和方法），而不是类名或显式声明。只要两个类型的结构一致，它们就是兼容的。这与 Java/C# 的名义类型系统（Nominal Typing）有本质区别。

## 二、具体用法

### 2.1 结构类型兼容

```typescript
interface Cat {
  name: string;
  meow(): void;
}

interface Robot {
  name: string;
  meow(): void;
}

// Robot 结构与 Cat 兼容，可以赋值
const robot: Robot = { name: "机器猫", meow: () => console.log("哔哔") };
const cat: Cat = robot; // ✅ 合法！结构兼容

cat.meow();
// 输出: 哔哔
```

### 2.2 鸭子类型示例

```typescript
// "如果它走起来像鸭子，叫起来像鸭子，那它就是鸭子"
interface Quackable {
  quack(): string;
}

function makeQuack(thing: Quackable): void {
  console.log(thing.quack());
}

// 不需要 implements Quackable，只要有 quack 方法即可
const duck = { quack: () => "嘎嘎嘎" };
const toyDuck = { quack: () => "吱吱吱", color: "yellow" };

makeQuack(duck);    // 输出: 嘎嘎嘎
makeQuack(toyDuck); // 输出: 吱吱吱
```

### 2.3 多余属性检查的例外

```typescript
interface Point {
  x: number;
  y: number;
}

// 直接字面量：严格检查多余属性
// const p: Point = { x: 1, y: 2, z: 3 }; // ❌ 错误

// 通过变量赋值：只检查必要属性
const obj = { x: 1, y: 2, z: 3 };
const p: Point = obj; // ✅ 合法，结构包含 x 和 y 即可

console.log(p.x, p.y);
// 输出: 1 2
```

### 2.4 与名义类型的区别

```typescript
// TypeScript：结构类型 —— 只看形状
interface UserId {
  value: string;
}

interface ProductId {
  value: string;
}

const userId: UserId = { value: "user-123" };
const productId: ProductId = userId; // ✅ 合法，结构相同

// 问题：可能不小心混用 ID 类型
// 解决方案：使用 Brand 类型区分
type BrandedUserId = string & { __brand: "UserId" };
type BrandedProductId = string & { __brand: "ProductId" };
```

### 2.5 类的结构比较

```typescript
class Animal {
  name: string = "";
}

class Robot {
  name: string = "";
  serialNumber: string = "";
}

// Robot 比 Animal 多了属性，但结构兼容
const robot = new Robot();
robot.name = "机器狗";
robot.serialNumber = "SN-001";

const animal: Animal = robot; // ✅ 合法，Robot 包含 Animal 的所有属性
console.log(animal.name); // 输出: 机器狗
```

## 三、注意事项与常见陷阱

1. **结构类型 ≠ 任何类型兼容**：缺少必要属性仍会报错
2. **函数参数是逆变的**：参数类型更宽泛的函数可以赋值给更具体的
3. **类的结构比较**：类的实例比较看结构，不看 `instanceof`
4. **多余属性检查**：字面量赋值会严格检查多余属性，变量赋值不会
5. **可选属性影响兼容性**：接口中的可选属性不影响赋值兼容性
6. **私有成员打破结构兼容**：使用 `private` 或 `#` 的类只能与同类型的实例兼容
