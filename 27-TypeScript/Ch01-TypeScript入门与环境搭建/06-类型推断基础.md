# 类型推断基础

## 一、概念说明

TypeScript 的类型推断（Type Inference）是指编译器在没有显式类型注解的情况下，自动推导出变量、参数或返回值的类型。这让代码更简洁——不必每个地方都写类型注解，同时仍保持类型安全。

## 二、具体用法

### 2.1 基本类型推断

```typescript
// 变量初始化时自动推断
let count = 42;        // 推断为 number
let name = "TypeScript"; // 推断为 string
let isActive = true;    // 推断为 boolean

// 函数返回值推断
function multiply(a: number, b: number) {
  return a * b; // 推断返回类型为 number
}

const result = multiply(6, 7);
console.log(result);
// 输出: 42
```

### 2.2 最佳通用类型

```typescript
// TypeScript 会找到所有元素的"最佳通用类型"
let arr = [1, 2, 3];           // 推断为 number[]
let mixed = [1, "hello", true]; // 推断为 (number | string | boolean)[]

// 数组字面量的推断
const coords = [[0, 0], [1, 1]]; // 推断为 number[][]
console.log(coords[0][1]);
// 输出: 0
```

### 2.3 上下文类型推断

```typescript
// 从上下文推断回调函数参数类型
const numbers = [1, 2, 3, 4, 5];

// item 和 index 的类型从 Array<number>.map 推断
const doubled = numbers.map((item, index) => {
  return `${index}: ${item * 2}`;
});

console.log(doubled);
// 输出: ["0: 2", "1: 4", "2: 6", "3: 8", "4: 10"]
```

### 2.4 类型收窄（Narrowing）

```typescript
// 通过条件判断收窄类型
function printValue(value: string | number): void {
  if (typeof value === "string") {
    // 这里 value 被收窄为 string
    console.log(value.toUpperCase());
  } else {
    // 这里 value 被收窄为 number
    console.log(value.toFixed(2));
  }
}

printValue("hello");
// 输出: HELLO
printValue(3.14159);
// 输出: 3.14
```

## 三、注意事项与常见陷阱

1. **`const` vs `let`**：`const` 推断为字面量类型，`let` 推断为宽泛类型
2. **数组推断**：字面量数组推断为具体类型，不会推断为元组
3. **不能推断时回退到 `any`**：没有上下文且未初始化的变量会变成 `any`
4. **推断不等于弱类型**：推断出的类型同样受到严格检查
