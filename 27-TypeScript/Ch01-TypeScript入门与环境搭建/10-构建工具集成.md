# 构建工具集成

## 一、概念说明

现代前端项目通常使用构建工具来处理 TypeScript。主流构建工具（Vite、Webpack、Rollup）都原生支持 TypeScript。构建工具负责将 TypeScript 编译为 JavaScript，同时处理模块打包、代码压缩、Source Map 等工作。

## 二、具体用法

### 2.1 Vite（推荐）

```bash
# 创建 Vite + TypeScript 项目
npm create vite@latest my-app -- --template react-ts

cd my-app
npm install
npm run dev
```

```typescript
// vite.config.ts
import { defineConfig } from "vite";

export default defineConfig({
  esbuild: {
    target: "es2020",
  },
  build: {
    target: "es2020",
    sourcemap: true,
  },
});
```

### 2.2 Webpack 配置

```bash
npm install -D webpack webpack-cli ts-loader typescript
```

```javascript
// webpack.config.js
const path = require("path");

module.exports = {
  entry: "./src/index.ts",
  module: {
    rules: [
      {
        test: /\.tsx?$/,
        use: "ts-loader",
        exclude: /node_modules/,
      },
    ],
  },
  resolve: {
    extensions: [".tsx", ".ts", ".js"],
  },
  output: {
    filename: "bundle.js",
    path: path.resolve(__dirname, "dist"),
  },
};
```

### 2.3 swc/esbuild 加速编译

```bash
# 使用 esbuild 进行极快的 TypeScript 编译
npm install -D esbuild

# 编译 TypeScript
npx esbuild src/index.ts --bundle --outfile=dist/bundle.js --platform=node
```

**输出：**
```
esbuild 编译速度通常是 tsc 的 10-100 倍
但不执行类型检查，需单独运行 tsc --noEmit 检查类型
```

## 三、注意事项与常见陷阱

1. **Vite 使用 esbuild**：开发时用 esbuild 编译 TS，速度极快
2. **类型检查分离**：esbuild/swc 不做类型检查，CI 中需单独运行 `tsc --noEmit`
3. **Source Map**：调试时必须开启 source map 才能在浏览器中调试 TS 源码
4. **ts-loader vs esbuild-loader**：Webpack 中 esbuild-loader 比 ts-loader 快很多
