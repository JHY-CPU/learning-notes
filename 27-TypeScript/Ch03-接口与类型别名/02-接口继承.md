# 接口继承

## 一、概念说明

接口通过 `extends` 关键字继承其他接口的属性，形成层次结构。一个接口可以继承一个或多个接口，继承后拥有父接口的所有属性。这支持代码复用和接口的组合设计。接口继承在语义上比交叉类型更清晰。

## 二、具体用法

### 2.1 单继承

```typescript
interface Animal {
  name: string;
  age: number;
}

interface Dog extends Animal {
  breed: string;
  bark(): void;
}

const dog: Dog = {
  name: "旺财",
  age: 3,
  breed: "金毛",
  bark() {
    console.log("汪汪!");
  },
};

dog.bark();
// 输出: 汪汪!
console.log(dog.name, dog.breed);
// 输出: 旺财 金毛
```

### 2.2 多继承

```typescript
interface Flyable {
  fly(): void;
}

interface Swimmable {
  swim(): void;
}

// 继承多个接口
interface Duck extends Flyable, Swimmable {
  quack(): void;
}

const duck: Duck = {
  fly() { console.log("飞"); },
  swim() { console.log("游"); },
  quack() { console.log("嘎嘎"); },
};

duck.fly();
duck.swim();
duck.quack();
// 输出:
// 飞
// 游
// 嘎嘎
```

### 2.3 继承与类型兼容

```typescript
interface HasId {
  id: number;
}

interface HasName {
  name: string;
}

interface User extends HasId, HasName {
  email: string;
}

// 子接口兼容父接口
function needsId(obj: HasId): void {
  console.log(obj.id);
}

const user: User = { id: 1, name: "Alice", email: "a@b.com" };
needsId(user); // ✅ User 兼容 HasId
// 输出: 1
```

### 2.4 接口继承类型别名

```typescript
// 接口可以继承 type 定义的对象类型
type Coordinates = {
  x: number;
  y: number;
};

interface Point3D extends Coordinates {
  z: number;
}

const point: Point3D = { x: 1, y: 2, z: 3 };
console.log(point);
// 输出: { x: 1, y: 2, z: 3 }
```

### 2.5 实际应用：分层设计

```typescript
// 基础实体
interface BaseEntity {
  id: string;
  createdAt: Date;
  updatedAt: Date;
}

// 用户相关
interface UserEntity extends BaseEntity {
  name: string;
  email: string;
  role: "admin" | "user";
}

// 文章相关
interface PostEntity extends BaseEntity {
  title: string;
  content: string;
  authorId: string;
}

// 通用 CRUD 函数
function findById<T extends BaseEntity>(items: T[], id: string): T | undefined {
  return items.find(item => item.id === id);
}
```

## 三、注意事项与常见陷阱

1. **多继承冲突**：继承的多个接口不能有同名但类型不兼容的属性
2. **继承 vs 组合**：简单场景用继承，复杂场景考虑交叉类型 `&`
3. **接口继承类型别名**：接口可以继承 `type` 定义的对象类型，但不能继承联合类型
4. **循环继承**：不能形成接口继承的循环依赖
5. **继承保持可选性**：如果父接口有可选属性，继承后仍然是可选的
6. **重写属性类型**：子接口可以用更具体的类型重写父接口的属性（协变）
