# 函数类型接口

## 一、概念说明

接口不仅可以描述对象，还可以描述**函数的形状**——包括参数类型和返回值类型。函数类型接口和 `type` 定义的函数类型功能相同，但接口支持声明合并。函数类型接口在框架设计中常用于定义回调、中间件、处理器等。

## 二、具体用法

### 2.1 函数类型接口

```typescript
// 使用调用签名定义函数类型
interface Add {
  (a: number, b: number): number;
}

const add: Add = (a, b) => a + b;
console.log(add(3, 5));
// 输出: 8
```

### 2.2 混合对象与函数

```typescript
interface EventHandler {
  // 调用签名：函数本身
  (event: string): void;
  // 属性：函数上的属性
  count: number;
  reset(): void;
}

function createHandler(): EventHandler {
  const handler = ((event: string) => {
    handler.count++;
    console.log(`处理事件: ${event}`);
  }) as EventHandler;

  handler.count = 0;
  handler.reset = () => { handler.count = 0; };

  return handler;
}

const handler = createHandler();
handler("click");
handler("scroll");
console.log(`事件计数: ${handler.count}`);
handler.reset();
// 输出:
// 处理事件: click
// 处理事件: scroll
// 事件计数: 2
```

### 2.3 泛型函数接口

```typescript
interface Converter<T, U> {
  (input: T): U;
}

const toString: Converter<number, string> = (n) => n.toString();
const toNumber: Converter<string, number> = (s) => parseInt(s, 10);

console.log(toString(42));   // 输出: "42"
console.log(toNumber("123")); // 输出: 123
```

### 2.4 与 type 定义函数类型的对比

```typescript
// interface 写法
interface MathOp {
  (a: number, b: number): number;
}

// type 写法
type MathOpType = (a: number, b: number) => number;

// 功能完全相同，但 interface 支持声明合并
interface Logger {
  (msg: string): void;
}
interface Logger {
  level: number; // 自动合并到上面的 Logger
}

const logger: Logger = Object.assign(
  (msg: string) => console.log(msg),
  { level: 1 }
);
```

### 2.5 回调函数接口

```typescript
// 定义回调函数类型
interface SuccessCallback<T> {
  (data: T): void;
}

interface ErrorCallback {
  (error: Error): void;
}

function fetchData(
  url: string,
  onSuccess: SuccessCallback<unknown>,
  onError: ErrorCallback
): void {
  try {
    const data = { id: 1, name: "Alice" };
    onSuccess(data);
  } catch (err) {
    onError(err as Error);
  }
}

fetchData(
  "/api/user",
  (data) => console.log("成功:", data),
  (err) => console.error("失败:", err.message)
);
```

## 三、注意事项与常见陷阱

1. **调用签名语法**：`(参数) => 返回值` 在接口内部不带 `=>`，用 `:` 分隔
2. **函数类型接口 vs type**：两者功能相同，interface 支持声明合并
3. **可调用对象**：同时有调用签名和属性的接口可以创建"增强函数"
4. **重载**：函数类型接口支持多个调用签名实现重载
5. **this 类型**：调用签名中可以声明 `this` 参数类型
6. **构造签名**：使用 `new` 关键字定义构造函数类型接口
