# 条件类型

## 一、概念说明

条件类型（Conditional Type）使用 `extends` 和三元运算符语法 `T extends U ? X : Y`，根据类型关系选择不同的类型。它让类型系统具备了"条件判断"能力，是类型编程的核心。

## 二、具体用法

### 2.1 基本条件类型

```typescript
// 如果 T 是 string 的子类型，则结果为 string，否则为 number
type IsString<T> = T extends string ? string : number;

type A = IsString<"hello">; // string（字面量是 string 的子类型）
type B = IsString<42>;      // number

const a: A = "world";
const b: B = 100;

console.log(a, b);
// 输出: world 100
```

### 2.2 条件类型推断 infer

```typescript
// infer 在条件类型中推断类型变量
type UnpackPromise<T> = T extends Promise<infer U> ? U : T;

type Result1 = UnpackPromise<Promise<string>>; // string
type Result2 = UnpackPromise<Promise<number[]>>; // number[]
type Result3 = UnpackPromise<boolean>;          // boolean

const r1: Result1 = "hello";
console.log(r1);
// 输出: hello
```

### 2.3 多层条件类型

```typescript
type TypeName<T> =
  T extends string ? "string" :
  T extends number ? "number" :
  T extends boolean ? "boolean" :
  T extends undefined ? "undefined" :
  T extends Function ? "function" :
  "object";

type T1 = TypeName<string>;    // "string"
type T2 = TypeName<() => void>; // "function"
type T3 = TypeName<{}>;        // "object"

console.log("条件类型在编译时选择分支");
// 输出: 条件类型在编译时选择分支
```

### 2.4 实际应用

```typescript
// 根据输入类型返回不同的输出类型
type ApiResponse<T> =
  T extends "user" ? { id: number; name: string } :
  T extends "post" ? { id: number; title: string } :
  never;

type UserResponse = ApiResponse<"user">;
type PostResponse = ApiResponse<"post">;

const user: UserResponse = { id: 1, name: "Alice" };
const post: PostResponse = { id: 1, title: "TypeScript 入门" };

console.log(user, post);
// 输出: { id: 1, name: "Alice" } { id: 1, title: "TypeScript 入门" }
```

## 三、注意事项与常见陷阱

1. **分布式条件类型**：联合类型会逐个成员分发计算
2. **`infer` 只能在条件类型的 `extends` 子句中使用**
3. **条件类型可以嵌套**：但过多嵌套影响可读性
4. **`never` 的分发**：`T extends U ? X : Y` 中 T 为 never 时结果为 never
