# 索引签名

## 一、概念说明

索引签名（Index Signature）用于描述对象的键值模式，当对象的属性名称不确定但值类型确定时使用。语法为 `[key: K]: V`，其中 K 可以是 `string` 或 `number`，V 是值的类型。索引签名适用于字典、哈希表等键不确定的场景。

## 二、具体用法

### 2.1 字符串索引签名

```typescript
interface StringMap {
  [key: string]: string;
}

const headers: StringMap = {
  "Content-Type": "application/json",
  "Authorization": "Bearer token123",
  "X-Custom-Header": "value",
};

console.log(headers["Content-Type"]);
// 输出: application/json
```

### 2.2 数字索引签名

```typescript
interface NumberArray {
  [index: number]: string;
}

const names: NumberArray = ["Alice", "Bob", "Charlie"];
console.log(names[0]);
// 输出: Alice
```

### 2.3 混合索引签名

```typescript
// 数字索引签名的值类型必须是字符串索引签名的子类型
interface Mixed {
  [key: string]: string | number;
  [index: number]: string; // ✅ string 是 string | number 的子类型
  name: string;            // ✅ 固定属性必须兼容索引签名
  // count: boolean;       // ❌ boolean 不兼容 string | number
}

const data: Mixed = {
  name: "Alice",
  0: "first",
  extra: 42,
};

console.log(data.name, data[0], data.extra);
// 输出: Alice first 42
```

### 2.4 Record 工具类型

```typescript
// Record<K, V> 是索引签名的类型安全版本
type PageInfo = { title: string; url: string };

const pages: Record<string, PageInfo> = {
  home: { title: "首页", url: "/" },
  about: { title: "关于", url: "/about" },
};

console.log(pages.home.title);
// 输出: 首页
```

### 2.5 noUncheckedIndexedAccess

```json
// tsconfig.json
{
  "compilerOptions": {
    "noUncheckedIndexedAccess": true
  }
}
```

```typescript
// 开启后，索引访问返回 T | undefined
interface Cache {
  [key: string]: string;
}

const cache: Cache = { token: "abc123" };
// 未开启: cache["token"] 类型为 string
// 开启后: cache["token"] 类型为 string | undefined

const token = cache["token"];
if (token !== undefined) {
  console.log(token.toUpperCase()); // ✅ 安全使用
}
```

### 2.6 实际应用

```typescript
// 事件监听器注册表
type EventMap = {
  click: { x: number; y: number };
  scroll: { delta: number };
  resize: { width: number; height: number };
};

type Listeners = {
  [K in keyof EventMap]?: ((event: EventMap[K]) => void)[];
};

const listeners: Listeners = {
  click: [(e) => console.log(`点击: ${e.x}, ${e.y}`)],
  scroll: [(e) => console.log(`滚动: ${e.delta}`)],
};
```

## 三、注意事项与常见陷阱

1. **索引签名值类型必须一致**：所有属性的值类型必须兼容索引签名
2. **`number` 索引更严格**：必须是 `string` 索引的子类型（因为 JS 中 `obj[0]` 等价于 `obj["0"]`）
3. **`noUncheckedIndexedAccess`**：强烈建议开启，使索引访问返回 `T | undefined`
4. **优先用 `Record`**：更类型安全的替代方案，键类型可以是联合类型
5. **索引签名与 `keyof`**：`keyof` 索引签名类型返回 `string | number`
6. **`Object.keys` 返回 `string[]`**：不会返回索引签名的键类型
