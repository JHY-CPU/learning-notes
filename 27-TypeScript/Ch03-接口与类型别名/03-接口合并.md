# 接口合并

## 一、概念说明

接口合并（Declaration Merging）是 TypeScript 的独特特性：当同名接口被多次声明时，它们的成员会**自动合并**。这在扩展第三方库类型或为模块添加全局声明时非常有用。

## 二、具体用法

### 2.1 基本合并

```typescript
interface User {
  name: string;
}

// 同名接口自动合并
interface User {
  age: number;
}

// 合并后 User 同时有 name 和 age
const user: User = {
  name: "Alice",
  age: 25,
};

console.log(user);
// 输出: { name: "Alice", age: 25 }
```

### 2.2 方法合并

```typescript
interface Logger {
  log(message: string): void;
}

interface Logger {
  error(message: string): void;
  warn(message: string): void;
}

const logger: Logger = {
  log(msg) { console.log(`[LOG] ${msg}`); },
  error(msg) { console.log(`[ERROR] ${msg}`); },
  warn(msg) { console.log(`[WARN] ${msg}`); },
};

logger.log("正常");
logger.error("出错");
// 输出:
// [LOG] 正常
// [ERROR] 出错
```

### 2.3 扩展第三方模块

```typescript
// 扩展全局 Window 接口
declare global {
  interface Window {
    myAppConfig: {
      version: string;
      debug: boolean;
    };
  }
}

// 现在可以安全访问 window.myAppConfig
export {};
```

### 2.4 合并规则

```typescript
interface Box {
  height: number;
  width: number;
}

interface Box {
  depth: number;
  // height: string; // ❌ 合并冲突：同名属性类型必须相同
}

const box: Box = { height: 10, width: 20, depth: 30 };
console.log(box);
// 输出: { height: 10, width: 20, depth: 30 }
```

## 三、注意事项与常见陷阱

1. **非函数成员必须唯一**：同名非函数属性类型必须完全相同
2. **函数成员会重载**：同名方法会形成重载列表
3. **`type` 不能合并**：同名 `type` 会报重复定义错误
4. **谨慎使用**：接口合并可能导致意外的类型变化
