# 泛型接口

## 一、概念说明

泛型接口允许接口在定义时引入**类型参数**，使接口在使用时可以指定具体的类型。这增加了接口的灵活性和复用性，是创建通用数据结构和 API 的基础。泛型接口在库和框架设计中极为常见（如 `Promise<T>`、`Array<T>`）。

## 二、具体用法

### 2.1 基本泛型接口

```typescript
interface Container<T> {
  value: T;
  getValue(): T;
}

const numberBox: Container<number> = {
  value: 42,
  getValue() { return this.value; },
};

const stringBox: Container<string> = {
  value: "hello",
  getValue() { return this.value; },
};

console.log(numberBox.getValue()); // 输出: 42
console.log(stringBox.getValue()); // 输出: hello
```

### 2.2 泛型约束

```typescript
interface Repository<T extends { id: number }> {
  getById(id: number): T | undefined;
  getAll(): T[];
  save(item: T): void;
}

interface User {
  id: number;
  name: string;
}

const userRepo: Repository<User> = {
  getById(id) { return { id, name: "Alice" }; },
  getAll() { return [{ id: 1, name: "Alice" }]; },
  save(user) { console.log(`保存用户: ${user.name}`); },
};

userRepo.save({ id: 1, name: "Bob" });
// 输出: 保存用户: Bob
```

### 2.3 默认类型参数

```typescript
// TypeScript 2.3+ 支持泛型默认类型
interface ApiResponse<T = unknown> {
  data: T;
  status: number;
  message: string;
}

// 不指定类型参数时使用默认值 unknown
const response1: ApiResponse = {
  data: "anything",
  status: 200,
  message: "OK",
};

// 指定类型参数
const response2: ApiResponse<User[]> = {
  data: [{ id: 1, name: "Alice" }],
  status: 200,
  message: "OK",
};

console.log(response2.data[0].name);
// 输出: Alice
```

### 2.4 多类型参数

```typescript
interface Map<K, V> {
  get(key: K): V | undefined;
  set(key: K, value: V): void;
  has(key: K): boolean;
}

// 实现一个简单的 Map
function createMap<K, V>(): Map<K, V> {
  const store = new globalThis.Map<K, V>();
  return {
    get: (key) => store.get(key),
    set: (key, value) => store.set(key, value),
    has: (key) => store.has(key),
  };
}

const userMap = createMap<string, User>();
userMap.set("u1", { id: 1, name: "Alice" });
console.log(userMap.get("u1")?.name); // 输出: Alice
```

### 2.5 泛型接口与继承

```typescript
interface BaseResponse<T> {
  data: T;
  status: number;
}

// 继承泛型接口
interface PaginatedResponse<T> extends BaseResponse<T[]> {
  page: number;
  pageSize: number;
  total: number;
}

const list: PaginatedResponse<User> = {
  data: [{ id: 1, name: "Alice" }],
  status: 200,
  page: 1,
  pageSize: 10,
  total: 100,
};
```

## 三、注意事项与常见陷阱

1. **类型参数不是运行时存在**：`T` 编译后消失，不能用于 `typeof` 或 `instanceof`
2. **约束限制了泛型的能力**：约束越严格，泛型的灵活性越低，应适度使用
3. **默认类型参数**：减少重复类型声明，但不应过度使用导致类型推断失效
4. **嵌套泛型**：`Container<Promise<User[]>>` 是合法的，但应保持可读性
5. **类型推断优先**：大多数场景 TypeScript 能自动推断泛型参数，不需要显式指定
6. **接口泛型 vs type 泛型**：两者功能相同，选择依据与普通接口/type 相同
