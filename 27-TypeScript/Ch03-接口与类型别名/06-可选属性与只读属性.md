# 可选属性与只读属性

## 一、概念说明

可选属性（`?`）表示该属性可以不存在，只读属性（`readonly`）表示该属性不能被重新赋值。两者是接口和类型定义中最常用的属性修饰符。

## 二、具体用法

### 2.1 可选属性 `?`

```typescript
interface Config {
  host: string;        // 必选
  port: number;        // 必选
  ssl?: boolean;       // 可选
  timeout?: number;    // 可选
}

const minimal: Config = { host: "localhost", port: 3000 };         // ✅
const full: Config = { host: "localhost", port: 3000, ssl: true, timeout: 5000 }; // ✅

// 可选属性的类型等价于 T | undefined
function getConfig(config: Config): void {
  const ssl = config.ssl ?? false; // 使用默认值
  console.log(`SSL: ${ssl}`);
}

getConfig(minimal);
// 输出: SSL: false
getConfig(full);
// 输出: SSL: true
```

### 2.2 只读属性 `readonly`

```typescript
interface User {
  readonly id: number;
  readonly createdAt: Date;
  name: string;
}

const user: User = {
  id: 1,
  createdAt: new Date(),
  name: "Alice",
};

user.name = "Bob";     // ✅ 可修改
// user.id = 2;        // ❌ 编译错误: readonly
// user.createdAt = new Date(); // ❌ 编译错误: readonly

console.log(user.name);
// 输出: Bob
```

### 2.3 Readonly 工具类型

```typescript
// 将所有属性变为只读
interface Mutable {
  x: number;
  y: number;
}

const frozen: Readonly<Mutable> = { x: 1, y: 2 };
// frozen.x = 3; // ❌ 编译错误

// ReadonlyArray
const readonlyArr: ReadonlyArray<number> = [1, 2, 3];
// readonlyArr.push(4); // ❌ 编译错误

console.log(frozen);
// 输出: { x: 1, y: 2 }
```

## 三、注意事项与常见陷阱

1. **可选属性类型**：等价于 `T | undefined`，访问时需注意
2. **`readonly` 是浅层的**：不防嵌套对象的属性修改
3. **`readonly` vs `const`**：`readonly` 用于属性，`const` 用于变量
4. **`Readonly<T>` 是浅冻结**：需要深冻结需自定义工具类型
