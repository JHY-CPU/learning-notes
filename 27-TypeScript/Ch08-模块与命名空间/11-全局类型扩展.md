# 全局类型扩展

## 一、概念说明

使用 `declare global` 可以在模块文件中扩展全局类型（如 `Window`、`globalThis`、`NodeJS` 命名空间）。这在向全局添加自定义变量、方法或扩展第三方库的全局接口时非常有用。扩展必须在模块文件中（有 `import` 或 `export` 的文件），用 `export {}` 确保文件被当作模块。

## 二、具体用法

### 2.1 扩展 Window

```typescript
// types/global.d.ts
declare global {
  interface Window {
    analytics: {
      track(event: string, data?: Record<string, unknown>): void;
      identify(userId: string): void;
    };
    __APP_VERSION__: string;
  }
}

export {};

// 使用
window.analytics.track("page_view", { page: "/home" });
window.analytics.identify("user-123");
console.log(window.__APP_VERSION__);
```

### 2.2 扩展 globalThis

```typescript
declare global {
  var __DEV__: boolean;
  var __CONFIG__: { apiUrl: string; debug: boolean };
}

// 现在可以全局访问（不需要 window. 前缀）
if (__DEV__) {
  console.log("开发模式", __CONFIG__.apiUrl);
}

export {};
```

### 2.3 扩展 NodeJS 全局

```typescript
declare global {
  namespace NodeJS {
    interface ProcessEnv {
      DATABASE_URL: string;
      API_KEY: string;
      NODE_ENV: "development" | "production" | "test";
      PORT: string;
    }

    interface Process {
      customMethod(): void;
    }
  }
}

// 类型安全的环境变量
const dbUrl: string = process.env.DATABASE_URL; // 类型: string
const env: "development" | "production" | "test" = process.env.NODE_ENV;
// process.env.UNKNOWN; // 如果 strictNullChecks 开启，会提示可能 undefined

export {};
```

### 2.4 扩展内置原型

```typescript
declare global {
  interface Array<T> {
    toMap<K>(keyFn: (item: T) => K): Map<K, T>;
    groupBy<K>(keyFn: (item: T) => K): Map<K, T[]>;
  }
}

// 实现
Array.prototype.toMap = function <T, K>(keyFn: (item: T) => K): Map<K, T> {
  const map = new Map<K, T>();
  for (const item of this) {
    map.set(keyFn(item), item);
  }
  return map;
};

const users = [{ id: 1, name: "Alice" }, { id: 2, name: "Bob" }];
const userMap = users.toMap((u) => u.id);

export {};
```

### 2.5 在声明文件中扩展

```typescript
// global.d.ts（不需要 export）
interface Window {
  fbAsyncInit?: () => void;
}

// 或使用 declare global（在模块文件中）
declare global {
  const API_KEY: string;
  function fetchConfig(): Promise<Config>;
}

export {};
```

### 2.6 与 JavaScript 的对比

```javascript
// JavaScript：运行时直接添加全局属性
window.myGlobal = { version: "1.0" };
globalThis.__DEBUG__ = true;

// 无类型提示，容易拼写错误

// TypeScript declare global：编译时声明类型
// 不影响运行时，但提供类型安全和自动补全
```

## 三、注意事项与常见陷阱

1. **必须在模块中**：文件需有 `import` 或 `export` 语句，否则 TypeScript 视为全局脚本
2. **`export {}` 技巧**：在 `.d.ts` 文件末尾加 `export {}` 确保文件被当作模块
3. **全局污染**：谨慎扩展全局类型，避免与第三方库冲突
4. **影响全局**：`declare global` 的声明在所有文件中可见
5. **优先使用模块导出**：能用模块导出解决的不要用全局扩展
6. **类型声明文件**：全局扩展通常放在 `types/global.d.ts` 或 `src/global.d.ts` 中
