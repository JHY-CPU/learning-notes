# 全局类型扩展

## 一、概念说明

使用 `declare global` 可以在模块文件中扩展全局类型（如 `Window`、`globalThis`）。这在向全局添加自定义变量、方法时很有用。

## 二、具体用法

### 2.1 扩展 Window

```typescript
// types/global.d.ts
declare global {
  interface Window {
    analytics: {
      track(event: string, data?: unknown): void;
    };
  }
}

export {};

// 使用
window.analytics.track("page_view", { page: "/home" });
```

### 2.2 扩展 globalThis

```typescript
declare global {
  var __DEV__: boolean;
}

// 现在可以全局访问
console.log(__DEV__);
// 输出: true 或 false
```

### 2.3 扩展 NodeJS 全局

```typescript
declare global {
  namespace NodeJS {
    interface ProcessEnv {
      DATABASE_URL: string;
      API_KEY: string;
      NODE_ENV: "development" | "production" | "test";
    }
  }
}

// 类型安全的环境变量
const dbUrl: string = process.env.DATABASE_URL;
console.log(`环境: ${process.env.NODE_ENV}`);
```

## 三、注意事项与常见陷阱

1. **必须在模块中**：文件需有 `import` 或 `export` 语句
2. **`export {}`**：确保文件被当作模块
3. **全局污染**：谨慎扩展全局类型
4. **类型声明文件**：全局扩展通常放在 `.d.ts` 文件中
