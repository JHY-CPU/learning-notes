# 模块打包配置

## 一、概念说明

发布 npm 包时，通常需要同时输出 **ESM**（`import/export`）和 **CJS**（`require/module.exports`）两种格式，以兼容不同的使用场景。工具如 `tsup`、`unbuild` 简化了双格式输出。

## 二、具体用法

### 2.1 tsup 配置

```bash
npm install -D tsup
```

```typescript
// tsup.config.ts
import { defineConfig } from "tsup";

export default defineConfig({
  entry: ["src/index.ts"],
  format: ["cjs", "esm"],
  dts: true,        // 生成类型声明
  splitting: false,
  sourcemap: true,
  clean: true,
});
```

### 2.2 package.json 配置

```json
{
  "name": "my-package",
  "type": "module",
  "main": "./dist/index.cjs",
  "module": "./dist/index.js",
  "types": "./dist/index.d.ts",
  "exports": {
    ".": {
      "import": {
        "types": "./dist/index.d.ts",
        "default": "./dist/index.js"
      },
      "require": {
        "types": "./dist/index.d.cts",
        "default": "./dist/index.cjs"
      }
    }
  },
  "files": ["dist"],
  "scripts": {
    "build": "tsup"
  }
}
```

### 2.3 运行构建

```bash
npm run build
# 生成:
# dist/index.js    (ESM)
# dist/index.cjs   (CJS)
# dist/index.d.ts  (类型声明)
```

## 三、注意事项与常见陷阱

1. **`exports` 字段优先**：现代 Node.js 使用 `exports` 解析模块
2. **类型声明需对应格式**：ESM 用 `.d.ts`，CJS 用 `.d.cts`
3. **`type: "module"`**：声明包为 ESM
4. **`files` 字段**：只发布 `dist` 目录
