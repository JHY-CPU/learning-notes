# 模块打包配置

## 一、概念说明

发布 npm 包时，通常需要同时输出 **ESM**（`import/export`）和 **CJS**（`require/module.exports`）两种格式，以兼容不同的使用场景。使用 `tsup`、`unbuild` 等工具可以简化双格式输出。`package.json` 的 `exports` 字段是现代 Node.js 模块解析的关键配置。

## 二、具体用法

### 2.1 tsup 配置

```bash
npm install -D tsup
```

```typescript
// tsup.config.ts
import { defineConfig } from "tsup";

export default defineConfig({
  entry: ["src/index.ts"],
  format: ["cjs", "esm"],   // 双格式输出
  dts: true,                // 生成类型声明
  splitting: false,         // 不分割代码
  sourcemap: true,          // 生成 sourcemap
  clean: true,              // 构建前清理 dist
  minify: false,            // 不压缩（库通常不压缩）
  target: "node16",         // 目标运行时
});
```

### 2.2 package.json 配置

```json
{
  "name": "my-package",
  "version": "1.0.0",
  "type": "module",
  "main": "./dist/index.cjs",
  "module": "./dist/index.js",
  "types": "./dist/index.d.ts",
  "exports": {
    ".": {
      "import": {
        "types": "./dist/index.d.ts",
        "default": "./dist/index.js"
      },
      "require": {
        "types": "./dist/index.d.cts",
        "default": "./dist/index.cjs"
      }
    }
  },
  "files": ["dist"],
  "scripts": {
    "build": "tsup",
    "dev": "tsup --watch"
  }
}
```

### 2.3 运行构建

```bash
npm run build
# 生成:
# dist/index.js    (ESM)
# dist/index.mjs   (ESM, 如果配置)
# dist/index.cjs   (CJS)
# dist/index.d.ts  (ESM 类型声明)
# dist/index.d.cts (CJS 类型声明)
```

### 2.4 unbuild 配置（替代方案）

```typescript
// build.config.ts
import { defineBuildConfig } from "unbuild";

export default defineBuildConfig({
  entries: ["src/index"],
  declaration: true,
  rollup: {
    emitCJS: true,
  },
});
```

### 2.5 验证包结构

```bash
# 使用 attw 检查包的模块格式是否正确
npx @arethetypeswrong/cli --pack .

# 检查 exports 字段是否正确
npx publint
```

### 2.6 与 JavaScript 的对比

```javascript
// JavaScript 单格式输出
// package.json: { "main": "index.js" }

// TypeScript 双格式 + 类型声明
// package.json: { "exports": { "import": ..., "require": ..., "types": ... } }
// 确保所有用户都能以正确格式和类型使用你的包
```

## 三、注意事项与常见陷阱

1. **`exports` 字段优先**：现代 Node.js 优先使用 `exports` 而非 `main`/`module`
2. **类型声明需对应格式**：ESM 用 `.d.ts`，CJS 用 `.d.cts`，确保 `exports` 中 `types` 条件在最前面
3. **`type: "module"`**：声明包为 ESM，CJS 文件需用 `.cjs` 扩展名
4. **`files` 字段**：只发布 `dist` 目录，避免将源码和配置文件发布到 npm
5. **`sideEffects`**：声明包无副作用，帮助打包工具进行 tree-shaking
6. **构建顺序**：`dts: true` 时 tsup 会先编译类型声明再打包，确保类型正确
