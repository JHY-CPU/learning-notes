# 三斜线指令

## 一、概念说明

三斜线指令（Triple-Slash Directives）以 `///` 开头，用于声明文件间的依赖关系。在现代 TypeScript 项目中已基本被 ES Module 的 `import` 取代，仅在极少数场景使用。

## 二、具体用法

### 2.1 引用类型声明

```typescript
/// <reference types="node" />
// 等价于安装 @types/node，现代项目直接用 npm install -D @types/node

/// <reference path="./types/global.d.ts" />
// 引用相对路径的类型声明文件
```

### 2.2 引用其他声明文件

```typescript
/// <reference path="jquery.d.ts" />
// 已过时，现代项目使用 import
```

## 三、仍需使用的场景

```typescript
// 场景1：在全局.d.ts文件中引用@types
/// <reference types="node" />
// 声明文件中的process、Buffer等全局类型需要此引用

// 场景2：monorepo中声明文件互相引用
/// <reference path="../../packages/core/src/types.d.ts" />
// 当tsconfig的paths无法解决时

// 场景3：构建指令（极少使用）
/// <reference no-default-lib="true"/>
// 声明此文件不包含默认库引用，用于自定义运行时环境
```

## 四、注意事项与常见陷阱

1. **已过时**：99%的场景应使用 ES Module `import`，三斜线指令仅用于声明文件
2. **`types` 属性**：在tsconfig.json的`compilerOptions.types`数组中配置更清晰
3. **`path` 属性**：路径相对于当前文件，tsconfig的`paths`和`references`是更好的替代
4. **仅在 `.d.ts` 文件中有意义**：普通 `.ts` 文件中使用会被编译器忽略
5. **顺序敏感**：三斜线指令必须出现在文件最顶部（注释和空行之前）

## 五、现代替代方案

```json
// tsconfig.json 中用 types 替代三斜线指令
{
  "compilerOptions": {
    "types": ["node", "jest"],        // 替代 /// <reference types="..." />
    "paths": { "@types/*": ["types/*"] }  // 替代 /// <reference path="..." />
  },
  "references": [{ "path": "../core" }]   // 项目间依赖
}
```
