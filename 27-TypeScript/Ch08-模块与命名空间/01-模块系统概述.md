# 模块系统概述

## 一、概念说明

TypeScript 支持两种模块系统：**ES Modules**（ESM，推荐）和 **CommonJS**（CJS）。ESM 使用 `import/export` 语法，CJS 使用 `require/module.exports`。TypeScript 根据 `tsconfig.json` 的 `module` 和 `moduleResolution` 选项决定模块行为。现代项目应优先使用 ESM，它是 JavaScript 语言标准的模块方案。

## 二、具体用法

### 2.1 ES Modules（推荐）

```typescript
// math.ts — 导出模块
export function add(a: number, b: number): number {
  return a + b;
}

export const PI = 3.14159;

export interface Calculator {
  calculate(a: number, b: number): number;
}

// main.ts — 导入模块
import { add, PI, type Calculator } from "./math.js";

console.log(add(PI, 1));
// 输出: 4.14159
```

### 2.2 CommonJS

```typescript
// math.js
module.exports = {
  add(a, b) { return a + b; },
  PI: 3.14159,
};

// main.js
const { add, PI } = require("./math");
console.log(add(PI, 1));
// 输出: 4.14159
```

### 2.3 模块配置

```json
// tsconfig.json — 推荐配置
{
  "compilerOptions": {
    "module": "ESNext",
    "moduleResolution": "Bundler",
    "target": "ES2020",
    "esModuleInterop": true,
    "isolatedModules": true
  }
}
```

### 2.4 ESM 与 CJS 的差异

```typescript
// ESM：顶层 await 支持
const config = await import("./config.js");

// ESM：`this` 在模块顶层是 undefined
console.log(this); // undefined

// ESM：严格模式默认开启
// 不需要 "use strict"

// CJS：动态 require
const mod = require(`./plugins/${name}.js`);

// CJS：`this` 在模块顶层是 module.exports
console.log(this); // module.exports 对象
```

### 2.5 混合使用（interop）

```typescript
// tsconfig.json
{
  "compilerOptions": {
    "esModuleInterop": true,     // 允许 import CJS 模块
    "allowSyntheticDefaultImports": true
  }
}

// 使用 ESM 导入 CJS 模块
import express from "express"; // 需要 esModuleInterop
import { Router } from "express";
```

### 2.6 与 JavaScript 的对比

```javascript
// JavaScript（浏览器）：原生 ESM
import { add } from "./math.js";

// JavaScript（Node.js CJS）：传统方式
const { add } = require("./math");

// TypeScript：统一使用 ESM 语法
// 编译时根据 module 配置转换为目标格式
import { add } from "./math.js";
```

## 三、注意事项与常见陷阱

1. **ESM 是未来**：Node.js 和浏览器都推荐 ESM，新项目不要用 CJS
2. **导入需写 `.js` 扩展名**：即使源文件是 `.ts`，ESM 模式下导入要写 `.js`
3. **`module` 与 `moduleResolution` 配合使用**：`ESNext` + `Bundler` 是最常见的现代组合
4. **`esModuleInterop`**：导入 CJS 模块时需要开启，产生额外的帮助代码
5. **`isolatedModules`**：推荐开启，确保每个文件可以被独立转译
6. **`type: "module"` in package.json**：Node.js 中使用 ESM 需要在 package.json 中声明
