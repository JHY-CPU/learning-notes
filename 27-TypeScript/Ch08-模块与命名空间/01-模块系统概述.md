# 模块系统概述

## 一、概念说明

TypeScript 支持两种模块系统：**ES Modules**（ESM，推荐）和 **CommonJS**（CJS）。ESM 使用 `import/export` 语法，CJS 使用 `require/module.exports`。TypeScript 根据 `tsconfig.json` 的 `module` 选项决定使用哪种。

## 二、具体用法

### 2.1 ES Modules（推荐）

```typescript
// math.ts
export function add(a: number, b: number): number {
  return a + b;
}

export const PI = 3.14159;

// main.ts
import { add, PI } from "./math.js";

console.log(add(PI, 1));
// 输出: 4.14159
```

### 2.2 CommonJS

```typescript
// math.js
module.exports = {
  add(a, b) { return a + b; },
  PI: 3.14159,
};

// main.js
const { add, PI } = require("./math");
console.log(add(PI, 1));
// 输出: 4.14159
```

### 2.3 模块配置

```json
// tsconfig.json
{
  "compilerOptions": {
    "module": "ESNext",    // 或 "CommonJS", "Node16", "Bundler"
    "moduleResolution": "Bundler" // 或 "Node", "Node16"
  }
}
```

## 三、注意事项与常见陷阱

1. **ESM 是未来**：Node.js 和浏览器都推荐 ESM
2. **导入需写 `.js` 扩展名**：即使源文件是 `.ts`
3. **`module` 与 `moduleResolution` 配合使用**
4. **混合 CJS/ESM**：需额外配置 `esModuleInterop`
