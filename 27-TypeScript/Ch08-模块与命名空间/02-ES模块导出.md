# ES 模块导出

## 一、概念说明

ES Module 使用 `export` 导出模块成员，支持**命名导出**（Named Export）和**默认导出**（Default Export）。命名导出可以导出多个成员，默认导出每个模块只能有一个。

## 二、具体用法

### 2.1 命名导出

```typescript
// 直接导出
export const API_URL = "https://api.example.com";

export function greet(name: string): string {
  return `Hello, ${name}!`;
}

export interface User {
  id: number;
  name: string;
}

// 先声明后导出
const VERSION = "1.0.0";
export { VERSION };

// 重命名导出
export { greet as sayHello };
```

### 2.2 默认导出

```typescript
// 默认导出
export default class UserService {
  getUsers(): string[] {
    return ["Alice", "Bob"];
  }
}

// 每个模块只能有一个默认导出
```

### 2.3 混合导出

```typescript
// 同时有默认导出和命名导出
export default function main(): void {
  console.log("主函数");
}

export const helper = () => console.log("辅助函数");
```

## 三、注意事项与常见陷阱

1. **命名导出推荐**：更明确、支持 tree-shaking
2. **默认导出可省略花括号**：`import X from "./module"`
3. **一个模块一个默认导出**：不能有两个 `export default`
4. **导出的是引用**：导出值变化后导入方可见
