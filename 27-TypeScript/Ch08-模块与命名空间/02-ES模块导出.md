# ES 模块导出

## 一、概念说明

ES Module 使用 `export` 导出模块成员，支持**命名导出**（Named Export）和**默认导出**（Default Export）。命名导出可以导出多个成员，默认导出每个模块只能有一个。TypeScript 额外支持 `export type` 只导出类型信息。

## 二、具体用法

### 2.1 命名导出

```typescript
// 直接导出
export const API_URL = "https://api.example.com";

export function greet(name: string): string {
  return `Hello, ${name}!`;
}

export interface User {
  id: number;
  name: string;
}

// 先声明后导出（更清晰的风格）
const VERSION = "1.0.0";
const MAX_RETRIES = 3;

export { VERSION, MAX_RETRIES };

// 重命名导出
export { greet as sayHello };
```

### 2.2 默认导出

```typescript
// 默认导出类
export default class UserService {
  private users: string[] = [];

  addUser(name: string): void {
    this.users.push(name);
  }

  getUsers(): string[] {
    return [...this.users];
  }
}

// 默认导出函数
export default function main(config: { debug: boolean }): void {
  console.log("启动应用", config);
}

// 默认导出值
const config = { host: "localhost", port: 3000 };
export default config;
```

### 2.3 混合导出

```typescript
// 同时有默认导出和命名导出
export default class App {
  run(): void { console.log("运行中"); }
}

export const VERSION = "1.0.0";
export function helper(): void { console.log("辅助函数"); }
export type Config = { debug: boolean };
```

### 2.4 类型导出

```typescript
// 只导出类型（编译后完全消除）
export type { User, Config } from "./types.js";

// 重命名类型导出
export type { User as UserType } from "./types.js";

// 混合值和类型导出
export { UserService, API_URL, type User, type Config };
```

### 2.5 Re-export（重新导出）

```typescript
// 从其他模块重新导出
export { add, multiply } from "./math.js";
export { capitalize, slugify } from "./strings.js";

// 重命名重新导出
export { add as sum } from "./math.js";

// 全部重新导出（barrel 文件）
export * from "./math.js";
export * from "./strings.js";
export * from "./types.js";
```

### 2.6 与 JavaScript 的对比

```javascript
// JavaScript：没有类型导出
export { helper, VERSION };
export default class Service {}

// TypeScript：额外支持类型导出
export { helper, VERSION, type User, type Config };
// type 导出编译后完全消除，不增加 bundle 体积
```

## 三、注意事项与常见陷阱

1. **命名导出推荐**：更明确、支持 tree-shaking、IDE 补全更好
2. **默认导出可省略花括号**：`import X from "./module"` 但名称由导入方自定
3. **一个模块一个默认导出**：不能有两个 `export default`
4. **导出的是引用（live binding）**：导出值变化后导入方可见变化
5. **`export type` 不导出值**：只能用于类型检查，不能当值使用
6. **重新导出 `export *` 不包含默认导出**：需显式 `export { default } from "./mod"`
