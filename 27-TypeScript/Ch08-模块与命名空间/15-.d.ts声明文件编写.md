# .d.ts 声明文件编写

## 一、概念说明

`.d.ts` 文件为纯 JavaScript 库提供类型定义，让 TypeScript 项目能安全地使用无类型的 JS 库。社区通过 `@types/*` 包维护大量库的类型声明。

## 二、具体用法

### 2.1 基本声明文件

```typescript
// my-library.d.ts
declare module "my-library" {
  export function greet(name: string): string;
  export interface Config {
    debug: boolean;
    version: string;
  }
  export default function init(config: Config): void;
}
```

### 2.2 为全局库声明

```typescript
// global.d.ts
declare const APP_VERSION: string;
declare function analytics(event: string): void;
```

### 2.3 使用声明文件

```typescript
// main.ts
import init, { greet, Config } from "my-library";

const config: Config = { debug: true, version: "1.0" };
init(config);
console.log(greet("World"));
// 输出: Hello, World!
```

### 2.4 发布声明

```json
// package.json
{
  "types": "./dist/index.d.ts",
  "exports": {
    ".": {
      "types": "./dist/index.d.ts",
      "import": "./dist/index.mjs",
      "require": "./dist/index.js"
    }
  }
}
```

## 三、注意事项与常见陷阱

1. **声明文件不生成 JS 代码**：仅提供类型信息
2. **`@types` 包优先**：安装 `@types/*` 可能覆盖自定义声明
3. **`declare` vs `export`**：`declare` 声明全局，`export` 声明模块
4. **导出类型用 `export type`**：确保只导出类型信息
