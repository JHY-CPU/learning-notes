# .d.ts 声明文件编写

## 一、概念说明

`.d.ts` 文件是 TypeScript 的**类型声明文件**，为纯 JavaScript 库提供类型定义，让 TypeScript 项目能安全地使用无类型的 JS 库。社区通过 `@types/*` 包（DefinitelyTyped）维护大量库的类型声明。声明文件不包含实现代码，只有类型信息。

## 二、具体用法

### 2.1 为 JS 库编写声明文件

```typescript
// my-library.d.ts
declare module "my-library" {
  export function greet(name: string): string;
  export function add(a: number, b: number): number;

  export interface Config {
    debug: boolean;
    version: string;
    timeout?: number;
  }

  export default function init(config: Config): void;
}

// 使用
import init, { greet, add, type Config } from "my-library";
const config: Config = { debug: true, version: "1.0" };
init(config);
console.log(greet("World")); // Hello, World!
```

### 2.2 为全局库声明

```typescript
// global.d.ts
declare const APP_VERSION: string;
declare const API_BASE_URL: string;
declare function analytics(event: string, data?: unknown): void;

// 不需要导入，全局可用
console.log(APP_VERSION);
analytics("page_view");
```

### 2.3 声明模块结构

```typescript
// 为复杂库声明
declare module "complex-lib" {
  // 子模块
  namespace utils {
    function formatDate(d: Date): string;
    function parseDate(s: string): Date;
  }

  namespace models {
    interface User {
      id: number;
      name: string;
    }

    interface Post {
      id: number;
      title: string;
      author: User;
    }
  }

  // 主导出
  export function createApp(config: object): App;
  export { utils, models };
}
```

### 2.4 使用 `@types/*` 包

```bash
# 安装社区维护的类型声明
npm install -D @types/node
npm install -D @types/express
npm install -D @types/lodash

# TypeScript 自动查找 node_modules/@types/ 下的声明
```

### 2.5 发布声明

```json
// package.json
{
  "name": "my-package",
  "types": "./dist/index.d.ts",
  "exports": {
    ".": {
      "types": "./dist/index.d.ts",
      "import": "./dist/index.mjs",
      "require": "./dist/index.js"
    }
  },
  "files": ["dist"]
}
```

```typescript
// src/index.ts — 自动生成声明
export function add(a: number, b: number): number {
  return a + b;
}

export interface Config {
  debug: boolean;
}

// 编译后 dist/index.d.ts 自动生成上述类型声明
```

### 2.6 与 JavaScript 的对比

```javascript
// JavaScript：无类型，IDE 无法提供补全
const lib = require("my-library");
lib.greet("World"); // 不知道参数类型

// TypeScript + .d.ts：完整类型支持
import { greet } from "my-library";
greet("World"); // IDE 显示: greet(name: string): string
```

## 三、注意事项与常见陷阱

1. **声明文件不生成 JS 代码**：`.d.ts` 仅提供类型信息，不参与编译输出
2. **`@types` 包优先**：如果安装了 `@types/*` 包，可能覆盖自定义声明
3. **`declare` vs `export`**：`declare` 声明 ambient（全局/模块级），`export` 声明模块成员
4. **导出类型用 `export type`**：确保只导出类型信息，不产生运行时引用
5. **`types` 和 `typings` 字段**：`package.json` 中二选一，指向入口声明文件
6. **声明文件要包含在 `include` 中**：确保 `tsconfig.json` 的 `include` 包含 `.d.ts` 文件所在目录
