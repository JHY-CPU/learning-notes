# 声明合并

## 一、概念说明

声明合并（Declaration Merging）是 TypeScript 的独特特性：同名的接口会自动合并属性，同名的命名空间会合并成员。这在扩展第三方库类型时非常有用。

## 二、具体用法

### 2.1 接口合并

```typescript
interface User {
  name: string;
}

interface User {
  age: number;
}

// 合并后: { name: string; age: number; }
const user: User = { name: "Alice", age: 25 };
```

### 2.2 命名空间合并

```typescript
namespace Utils {
  export function log(msg: string): void {
    console.log(`[LOG] ${msg}`);
  }
}

namespace Utils {
  export function warn(msg: string): void {
    console.log(`[WARN] ${msg}`);
  }
}

Utils.log("正常");   // 输出: [LOG] 正常
Utils.warn("警告");  // 输出: [WARN] 警告
```

### 2.3 扩展全局类型

```typescript
declare global {
  interface Window {
    myApp: {
      version: string;
      init(): void;
    };
  }
}

export {};
```

## 三、注意事项与常见陷阱

1. **非函数成员类型必须相同**：同名属性冲突会报错
2. **函数成员形成重载**：同名方法会创建重载列表
3. **`type` 不能合并**：同名 `type` 会报重复定义
4. **谨慎使用**：声明合并可能导致意外的类型变化
