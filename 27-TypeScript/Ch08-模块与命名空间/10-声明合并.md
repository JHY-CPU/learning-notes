# 声明合并

## 一、概念说明

声明合并（Declaration Merging）是 TypeScript 的独特特性：同名的**接口**会自动合并属性，同名的**命名空间**会合并成员。这在扩展第三方库类型、增强全局对象时非常有用。`type` 别名**不支持**声明合并，同名 type 会报重复定义错误。

## 二、具体用法

### 2.1 接口合并

```typescript
// 分散定义同一接口，属性自动合并
interface User {
  name: string;
}

interface User {
  age: number;
}

interface User {
  email: string;
}

// 合并后: { name: string; age: number; email: string; }
const user: User = { name: "Alice", age: 25, email: "a@b.com" };
```

### 2.2 合并规则

```typescript
// 非函数成员：类型必须相同，否则报错
interface A { x: number; }
interface A { x: string; } // 错误：类型冲突

// 函数成员：形成重载列表
interface Logger {
  log(msg: string): void;
}
interface Logger {
  log(msg: string, level: number): void;
}
// 合并后 log 有两个重载

// 后声明的接口优先级更高（重载顺序）
```

### 2.3 命名空间合并

```typescript
namespace Validation {
  export function isString(v: unknown): v is string {
    return typeof v === "string";
  }
}

namespace Validation {
  export function isNumber(v: unknown): v is number {
    return typeof v === "number";
  }
}

// 合并后两个函数都可用
Validation.isString("hello");
Validation.isNumber(42);
```

### 2.4 接口与命名空间合并

```typescript
// 接口可以和同名命名空间合并（用于添加静态成员）
interface JQuery {
  (selector: string): JQuery;
}

namespace JQuery {
  export function ajax(url: string): void {}
  export const version = "3.6.0";
}

// 使用
const $ = JQuery as any;
JQuery.ajax("/api/data");
console.log(JQuery.version); // "3.6.0"
```

### 2.5 扩展全局类型

```typescript
// 扩展 Window 对象
declare global {
  interface Window {
    myApp: {
      version: string;
      init(): void;
    };
  }
}

// 现在可以安全使用
window.myApp.init();
console.log(window.myApp.version);

export {}; // 确保文件是模块
```

### 2.6 与 JavaScript 的对比

```javascript
// JavaScript：没有声明合并的概念
// 同名变量会覆盖，不会合并
const config = { a: 1 };
const config = { b: 2 }; // 严格模式报错

// TypeScript 声明合并：只在类型层面合并
// 编译后的 JS 没有这个行为
```

## 三、注意事项与常见陷阱

1. **非函数成员类型必须相同**：同名属性类型冲突会报错，不会自动合并类型
2. **函数成员形成重载**：同名方法会创建重载列表，后声明的排在前面
3. **`type` 不能合并**：同名 `type A = ...` 会报重复定义错误，用 interface 代替
4. **谨慎使用**：声明合并可能导致意外的类型变化，大型项目中可能产生冲突
5. **合并是全局性的**：所有文件中同名声明都会合并，注意命名空间隔离
6. **`module augmentation` 也是声明合并**：`declare module "express"` 扩展模块类型
