# ES 模块导入

## 一、概念说明

ES Module 使用 `import` 导入模块成员，支持命名导入（Named Import）、默认导入（Default Import）、命名空间导入（Namespace Import）和动态导入（Dynamic Import）。TypeScript 还支持 `import type` 只导入类型。

## 二、具体用法

### 2.1 命名导入

```typescript
// 导入指定成员
import { add, multiply } from "./math.js";

console.log(add(2, 3));      // 输出: 5
console.log(multiply(4, 5)); // 输出: 20

// 重命名导入
import { add as sum, multiply as product } from "./math.js";
console.log(sum(1, 2)); // 输出: 3
```

### 2.2 默认导入

```typescript
// 导入默认导出（名称由导入方自定）
import UserService from "./user-service.js";

const service = new UserService();
service.addUser("Alice");
console.log(service.getUsers());
// 输出: ["Alice"]
```

### 2.3 命名空间导入

```typescript
// 导入所有成员到一个命名空间对象
import * as MathUtils from "./math.js";

console.log(MathUtils.add(1, 2));      // 输出: 3
console.log(MathUtils.multiply(3, 4)); // 输出: 12
console.log(MathUtils.PI);             // 输出: 3.14159
```

### 2.4 混合导入

```typescript
// 默认导入 + 命名导入
import DefaultService, { helper, VERSION } from "./module.js";

// 默认导入 + 命名空间导入（少见但合法）
import DefaultExport, * as AllExports from "./module.js";
```

### 2.5 类型导入

```typescript
// 只导入类型（编译后消除）
import type { User, Config } from "./types.js";

// 内联类型导入（混合使用）
import { UserService, type User } from "./services.js";

// 导入整个模块的类型
import type * as Types from "./types.js";
```

### 2.6 与 JavaScript 的对比

```javascript
// JavaScript（浏览器 ESM）
import { add } from "./math.js"; // 必须写扩展名

// JavaScript（Node.js CJS）
const { add } = require("./math"); // 不需要扩展名

// TypeScript：根据 moduleResolution 策略决定
// bundler 策略：import { add } from "./math";（可省略扩展名）
// node16 策略：import { add } from "./math.js";（必须写扩展名）
```

## 三、注意事项与常见陷阱

1. **`.js` 扩展名**：ESM 模式下 TypeScript 文件导入通常需要写 `.js` 扩展名
2. **命名导入必须匹配**：名称需与导出完全一致（包括大小写）
3. **默认导入名称自定**：`import Anything from "./mod"` 可以用任意名称
4. **导入的是只读引用（live binding）**：不能修改导入的变量，但可以看到导出方的变化
5. **`import type` 只能用于类型**：不能用 `import type` 导入的值调用函数
6. **Side-effect 导入**：`import "./polyfill.js"` 只执行模块代码，不导入任何成员
