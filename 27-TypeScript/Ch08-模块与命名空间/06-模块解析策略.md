# 模块解析策略

## 一、概念说明

模块解析策略决定 TypeScript 如何查找导入的模块。`moduleResolution` 选项控制解析行为：`node`、`node16`、`bundler`、`classic`。

## 二、具体用法

### 2.1 解析策略对比

```json
// tsconfig.json
{
  "compilerOptions": {
    "module": "ESNext",
    "moduleResolution": "Bundler"
  }
}
```

| 策略 | 适用场景 | 特点 |
|------|----------|------|
| `node` | Node.js 项目 | 经典 Node 解析算法 |
| `node16` | Node.js ESM | 要求 `.js` 扩展名 |
| `bundler` | Vite/Webpack 等 | 灵活，不强制扩展名 |
| `classic` | AMD/SystemJS | 旧式解析 |

### 2.2 Bundler 策略

```typescript
// 不需要写扩展名
import { add } from "./math";      // ✅ 自动解析
import { config } from "./config"; // ✅ 自动查找 .ts/.tsx/.js

// 支持路径映射
import { User } from "@/types/user"; // ✅ 如果配置了 paths
```

### 2.3 Node16 策略

```typescript
// 必须写扩展名
import { add } from "./math.js";  // ✅
// import { add } from "./math";  // ❌ Node16 要求扩展名
```

## 三、注意事项与常见陷阱

1. **`bundler` 最灵活**：Vite、Webpack 项目推荐
2. **`node16` 最严格**：Node.js 原生 ESM 项目使用
3. **路径映射需配套**：`paths` 需要 `baseUrl`
4. **包导出**：`node16` 支持 `package.json` 的 `exports` 字段
