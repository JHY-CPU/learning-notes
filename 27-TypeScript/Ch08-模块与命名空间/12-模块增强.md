# 模块增强

## 一、概念说明

模块增强（Module Augmentation）允许为**已存在**的模块添加新的类型声明。使用 `declare module "module-name"` 语法扩展第三方库的接口或为已有模块补充类型。常见应用：扩展 Express 的 `Request` 类型、为 Vuex/Pinia store 添加类型、增强 Jest 的 matcher 等。

## 二、具体用法

### 2.1 增强第三方模块

```typescript
// types/express.d.ts
import "express";

declare module "express" {
  interface Request {
    user?: {
      id: number;
      name: string;
      roles: string[];
    };
    requestId?: string;
  }
}

// 使用（类型安全）
// app.use((req, res, next) => {
//   req.user = { id: 1, name: "Alice", roles: ["admin"] };
//   req.requestId = crypto.randomUUID();
//   next();
// });
```

### 2.2 增强 Vue/Pinia Store

```typescript
// pinia.d.ts
import "pinia";

declare module "pinia" {
  export interface PiniaCustomProperties {
    $myPlugin: {
      version: string;
      log(message: string): void;
    };
  }
}
```

### 2.3 增强 Express Response

```typescript
declare module "express" {
  interface Response {
    success(data: unknown, message?: string): void;
    error(message: string, code?: number): void;
  }
}

// 实现中间件
// app.use((req, res, next) => {
//   res.success = (data, msg) => res.json({ success: true, data, message: msg });
//   res.error = (msg, code = 500) => res.status(code).json({ success: false, message: msg });
//   next();
// });
```

### 2.4 增强环境变量

```typescript
declare global {
  namespace NodeJS {
    interface ProcessEnv {
      NODE_ENV: "development" | "production" | "test";
      DATABASE_URL: string;
      JWT_SECRET: string;
    }
  }
}

export {};
```

### 2.5 增强 Jest Matcher

```typescript
// jest.d.ts
declare module "expect" {
  interface Matchers<R> {
    toBeWithinRange(floor: number, ceiling: number): R;
    toBeValidDate(): R;
  }
}
```

### 2.6 与 JavaScript 的对比

```javascript
// JavaScript：运行时 monkey-patching
const express = require("express");
express.request.fullName = function() {
  return `${this.firstName} ${this.lastName}`;
};
// 无类型提示，容易出错

// TypeScript 模块增强：编译时类型声明
// 不修改运行时代码，只提供类型信息
// declare module "express" { interface Request { fullName(): string; } }
```

## 三、注意事项与常见陷阱

1. **影响全局**：模块增强会影响所有使用该模块的地方，可能与他人增强冲突
2. **必须匹配原模块路径**：增强声明需使用与导入相同的模块名
3. **`declare module` 不创建新模块**：只是扩展现有模块的类型
4. **增强需要在项目中可见**：确保增强文件被包含在 `tsconfig.json` 的 `include` 中
5. **属性类型不能冲突**：如果增强的属性已存在且类型不同，会报错
6. **建议放在单独文件中**：`types/` 目录下，便于维护
