# 模块增强

## 一、概念说明

模块增强（Module Augmentation）允许为已存在的模块添加新的类型声明。常用于为第三方库补充类型或扩展已有模块的功能。

## 二、具体用法

### 2.1 增强第三方模块

```typescript
// 扩展 express 的 Request 类型
import "express";

declare module "express" {
  interface Request {
    user?: {
      id: number;
      name: string;
    };
  }
}

// 现在 req.user 有类型提示
```

### 2.2 增强内置模块

```typescript
// 扩展 Array
declare global {
  interface Array<T> {
    toMap<K>(keyFn: (item: T) => K): Map<K, T>;
  }
}

// 实现
Array.prototype.toMap = function <T, K>(keyFn: (item: T) => K): Map<K, T> {
  const map = new Map<K, T>();
  for (const item of this) {
    map.set(keyFn(item), item);
  }
  return map;
};

const users = [{ id: 1, name: "Alice" }, { id: 2, name: "Bob" }];
const userMap = users.toMap((u) => u.id);
console.log(userMap.get(1));
// 输出: { id: 1, name: "Alice" }
```

## 三、注意事项与常见陷阱

1. **影响全局**：模块增强会影响所有使用该模块的地方
2. **必须匹配原模块路径**：增强声明需使用相同的模块名
3. **`declare module` 不创建新模块**：只是扩展现有模块
4. **谨慎使用**：可能与其他增强冲突
