# 模块最佳实践

## 一、概念说明

遵循模块最佳实践可以确保项目结构清晰、类型安全且性能良好。核心原则：ESM 优先、避免 namespace、合理组织文件结构。

## 二、具体用法

### 2.1 ESM 优先

```json
// tsconfig.json
{
  "compilerOptions": {
    "module": "ESNext",
    "moduleResolution": "Bundler",
    "target": "ES2020"
  }
}
```

### 2.2 合理的文件组织

```typescript
// ✅ 清晰的模块结构
src/
├── types/          # 类型定义
│   ├── user.ts
│   └── api.ts
├── utils/          # 工具函数
│   ├── math.ts
│   └── string.ts
├── services/       # 业务逻辑
│   └── user-service.ts
└── index.ts        # 入口文件

// ✅ 使用 barrel 文件聚合导出
export { UserService } from "./services/user-service.js";
export type { User } from "./types/user.js";
```

### 2.3 类型导入

```typescript
// ✅ 类型导入用 import type
import type { User, Config } from "./types.js";

// ✅ 值和类型混合导入
import { UserService, type User } from "./services/user-service.js";
```

### 2.4 避免 namespace

```typescript
// ❌ 不要使用 namespace
namespace Utils {
  export function add(a: number, b: number) { return a + b; }
}

// ✅ 使用 ES Module
export function add(a: number, b: number): number {
  return a + b;
}
```

## 三、注意事项与常见陷阱

1. **ESM 优先**：不要用 namespace，用 ES Module
2. **`import type`**：只导入类型时使用 `import type`
3. **避免循环依赖**：A -> B -> A 的导入链
4. **barrel 文件谨慎使用**：大型项目可能影响 tree-shaking
5. **一个模块一个职责**：每个文件只做一件事
6. **路径别名适度**：2-3 个别名即可，如 `@/`、`@types/`
