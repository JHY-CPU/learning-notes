# 组件类型导入

## 一、概念说明

Vue 3 中组件的类型导入需要注意 `import type` 的使用，确保类型信息在编译时可用，同时不影响运行时行为。在 `isolatedModules` 模式下，类型导入必须使用 `import type` 语法。

## 二、具体用法

### 2.1 类型导入语法

```typescript
// 使用 import type 导入类型
import type { Ref, ComputedRef } from 'vue';
import type { RouteRecordRaw } from 'vue-router';
import type { User, CreateUserDto } from './types';

// 组件类型导入
import type { Component } from 'vue';
import type MyComponent from './MyComponent.vue';

// 混合导入 — 值和类型
import { ref, computed } from 'vue';
import type { Ref, ComputedRef } from 'vue';
```

### 2.2 组件 Props 类型导出

```typescript
// types/components.ts
export interface ButtonProps {
  variant?: 'primary' | 'secondary';
  size?: 'sm' | 'md' | 'lg';
  disabled?: boolean;
}

export interface ModalProps {
  title: string;
  visible: boolean;
  width?: number | string;
}

export interface TableProps<T> {
  data: T[];
  columns: ColumnDef<T>[];
  loading?: boolean;
}

// 组件中导入
// import type { ButtonProps } from '../types/components';
```

### 2.3 从 .vue 文件提取类型

```typescript
import type { ComponentExposed } from 'vue-component-type-helpers';
import type MyModal from './MyModal.vue';

// 提取组件暴露的方法/属性类型
type ModalExposed = ComponentExposed<typeof MyModal>;

// 使用
const modalRef = ref<ModalExposed>();
// modalRef.value?.open() 有类型提示
```

### 2.4 TypeScript 配置

```json
// tsconfig.json
{
  "compilerOptions": {
    "isolatedModules": true,    // 强制使用 import type
    "verbatimModuleSyntax": true // 更严格的导入规则
  }
}
```

### 2.5 类型导出最佳实践

```typescript
// types/index.ts — 统一导出

// 值导出
export { default as MyButton } from '../components/MyButton.vue';

// 类型导出
export type { ButtonProps } from '../components/MyButton.vue';
export type { User, UserRole, CreateUserDto } from './user';
export type { ApiResponse, PaginatedResponse } from './api';

// 混合导出
export { AppError, NotFoundError } from './errors';
export type { ErrorDetails } from './errors';
```

### 2.6 自动类型导入

```json
// VS Code 设置自动使用 import type
// settings.json
{
  "typescript.preferences.autoImportFileExcludePatterns": [],
  "typescript.preferences.useAliasesForRenames": true
}
```

## 三、注意事项与常见陷阱

1. **`isolatedModules: true` 强制使用 `import type`**：确保代码兼容 Babel/esbuild
2. **`import type` 不会出现在编译输出中**：只用于类型检查
3. **不要用 `import type` 导入值**：如组件、函数等运行时需要的内容
4. **类型导出用 `export type`**：明确区分值和类型
5. **`.vue` 文件的类型导入**：使用 `import type` 避免运行时问题
