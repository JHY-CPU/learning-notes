# 模板类型推断

## 一、概念说明

Vue 3 的模板中，TypeScript 类型推断是自动的。在 `<script setup>` 中定义的 ref、computed、props 等在模板中都能获得正确的类型提示。Volar 插件负责提供 IDE 中的类型推断。

## 二、具体用法

### 2.1 模板中的自动推断

```vue
<script setup lang="ts">
import { ref, computed } from 'vue';

const count = ref(0);              // Ref<number>
const items = ref<string[]>([]);   // Ref<string[]>
const doubled = computed(() => count.value * 2);
</script>

<template>
  <!-- 模板中自动推断类型 -->
  <p>{{ count }}</p>           <!-- number -->
  <p>{{ doubled }}</p>         <!-- number -->
  <p>{{ items.length }}</p>    <!-- number，有类型提示 -->

  <!-- v-for 中的 item 类型 -->
  <ul>
    <li v-for="(item, index) in items" :key="index">
      {{ item }}  <!-- item: string, index: number -->
    </li>
  </ul>

  <!-- v-if 类型收窄 -->
  <div v-if="items.length > 0">
    {{ items[0].toUpperCase() }}  <!-- items[0]: string -->
  </div>
</template>
```

### 2.2 Props 类型在模板中

```vue
<script setup lang="ts">
interface Props {
  user: { name: string; age: number };
  title?: string;
}

const props = defineProps<Props>();
</script>

<template>
  <!-- props 自动有类型 -->
  <h1>{{ props.title ?? '默认标题' }}</h1>
  <p>{{ props.user.name }} - {{ props.user.age }}岁</p>

  <!-- 解构后也有类型 -->
  <div>
    <span>{{ user.name }}</span>
  </div>
</template>
```

### 2.3 事件处理类型

```vue
<script setup lang="ts">
const emit = defineEmits<{
  change: [value: string];
}>();

const handleInput = (event: Event) => {
  const value = (event.target as HTMLInputElement).value;
  emit('change', value);
};
</script>

<template>
  <!-- 内联事件有类型推断 -->
  <input @input="handleInput" />

  <!-- $event 类型 -->
  <button @click="(e: MouseEvent) => console.log(e.clientX)">
    点击
  </button>
</template>
```

### 2.4 模板 ref 类型

```vue
<script setup lang="ts">
import { ref } from 'vue';

// 模板 ref — 与模板中 ref 属性同名
const inputRef = ref<HTMLInputElement>();
const modalRef = ref<InstanceType<typeof MyModal>>();
</script>

<template>
  <!-- ref 名称匹配变量名时自动关联类型 -->
  <input ref="inputRef" />
  <!-- inputRef.value?.focus() 有类型提示 -->

  <MyModal ref="modalRef" />
  <!-- modalRef.value?.open() 有类型提示 -->
</template>
```

### 2.5 模板中的类型限制

```vue
<script setup lang="ts">
// 模板中不能使用 import 的类型
// import type { User } from './types'; // 在模板中不可用

// 但可以使用 import 的值
import { formatDate, formatCurrency } from './utils';
const date = new Date();
const price = 99.9;
</script>

<template>
  <!-- 可以调用导入的函数 -->
  <p>{{ formatDate(date) }}</p>
  <p>{{ formatCurrency(price) }}</p>

  <!-- 不能在模板中使用 TypeScript 语法 -->
  <!-- <p>{{ (price as number).toFixed(2) }}</p> --> <!-- 错误 -->
</template>
```

## 三、注意事项与常见陷阱

1. **Volar 插件必须安装**：Vetur 不支持 Vue 3 模板类型推断
2. **模板中的类型推断依赖 `<script setup>`**：Options API 推断较弱
3. **模板中不能使用类型断言**：`as` 语法在模板中不可用
4. **`$refs` 在模板中自动关联**：ref 名与变量名匹配时
5. **`v-for` 的解构有类型**：`v-for="item in items"` 中 `item` 有正确类型
