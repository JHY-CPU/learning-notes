# 自定义指令类型

## 一、概念说明

Vue 3 自定义指令的类型通过 `Directive` 泛型定义，可以约束指令应用到的元素类型和传入的值类型。正确类型化自定义指令能确保指令在使用时有类型安全。

## 二、具体用法

### 2.1 基本指令类型

```typescript
import type { Directive, DirectiveBinding } from 'vue';

// Directive<T, V> 泛型参数：
// T — 指令绑定到的元素类型
// V — 指令的值类型

// 聚焦指令
const vFocus: Directive<HTMLInputElement, boolean> = {
  mounted(el, binding) {
    // el: HTMLInputElement
    // binding.value: boolean
    if (binding.value) {
      el.focus();
    }
  },
};

// 注册
app.directive('focus', vFocus);
```

### 2.2 带配置的指令

```typescript
// Tooltip 指令
interface TooltipValue {
  text: string;
  position?: 'top' | 'bottom' | 'left' | 'right';
}

const vTooltip: Directive<HTMLElement, TooltipValue> = {
  mounted(el, binding) {
    const { text, position = 'top' } = binding.value;
    el.setAttribute('data-tooltip', text);
    el.setAttribute('data-tooltip-position', position);
  },

  updated(el, binding) {
    if (binding.value.text !== binding.oldValue?.text) {
      el.setAttribute('data-tooltip', binding.value.text);
    }
  },

  unmounted(el) {
    el.removeAttribute('data-tooltip');
  },
};
```

### 2.3 权限指令

```typescript
// 权限控制指令
type Role = 'admin' | 'editor' | 'viewer';

const vPermission: Directive<HTMLElement, Role[]> = {
  mounted(el, binding) {
    const userRole = getCurrentUserRole();
    const allowedRoles = binding.value; // Role[]

    if (!allowedRoles.includes(userRole)) {
      // 隐藏或移除元素
      el.style.display = 'none';
    }
  },
};

// 使用
// <button v-permission="['admin', 'editor']">编辑</button>
```

### 2.4 拖拽指令

```typescript
interface DragValue {
  onDragStart?: (event: DragEvent) => void;
  onDragEnd?: (event: DragEvent) => void;
}

const vDraggable: Directive<HTMLElement, DragValue> = {
  mounted(el, binding) {
    el.draggable = true;

    el.addEventListener('dragstart', (e) => {
      binding.value.onDragStart?.(e);
    });

    el.addEventListener('dragend', (e) => {
      binding.value.onDragEnd?.(e);
    });
  },
};
```

### 2.5 在模板中使用

```vue
<script setup lang="ts">
// 局部注册指令
const vHighlight: Directive<HTMLElement, string> = {
  mounted(el, binding) {
    el.style.backgroundColor = binding.value;
  },
};
</script>

<template>
  <div v-highlight="'yellow'">高亮显示</div>

  <!-- 值类型错误会报错 -->
  <!-- <div v-highlight="123">错误</div> -->
</template>
```

## 三、注意事项与常见陷阱

1. **`Directive<T, V>` 的泛型参数**：T 是元素类型，V 是值类型
2. **`binding.value` 的类型自动推断**：与泛型参数一致
3. **`binding.oldValue` 可能是 `undefined`**：首次调用时
4. **`vFocus` 等简写指令**：只有 `mounted` 钩子时可以用函数简化
5. **指令的命名**：在模板中用 `v-xxx`，在代码中用 `vXxx`（camelCase）
