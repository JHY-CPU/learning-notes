# 全局属性类型

## 一、概念说明

Vue 3 允许通过 `app.config.globalProperties` 添加全局属性（如 `$http`、`$translate`），这些属性在所有组件中可用。需要通过模块扩展声明类型，才能在模板和组件中获得类型提示。

## 二、具体用法

### 2.1 定义全局属性

```typescript
// plugins/http.ts
import type { App } from 'vue';

interface HttpClient {
  get<T>(url: string): Promise<T>;
  post<T>(url: string, data: unknown): Promise<T>;
}

export const httpClient: HttpClient = {
  async get<T>(url: string): Promise<T> {
    const res = await fetch(url);
    return res.json();
  },

  async post<T>(url: string, data: unknown): Promise<T> {
    const res = await fetch(url, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(data),
    });
    return res.json();
  },
};

export default {
  install(app: App) {
    app.config.globalProperties.$http = httpClient;
  },
};
```

### 2.2 类型扩展

```typescript
// src/types/global-properties.d.ts
export {};

declare module 'vue' {
  interface ComponentCustomProperties {
    $http: import('../plugins/http').HttpClient;
    $translate: (key: string) => string;
    $formatDate: (date: Date) => string;
    $user: { name: string; role: string } | null;
  }
}
```

### 2.3 在组件中使用

```vue
<script setup lang="ts">
import { getCurrentInstance } from 'vue';

// 方式一：通过 getCurrentInstance
const { appContext } = getCurrentInstance()!;
const $http = appContext.config.globalProperties.$http;

// 方式二：使用 useHttp composable（推荐）
import { useHttp } from '../composables/useHttp';
const http = useHttp();
</script>

<template>
  <!-- 模板中可直接使用全局属性 -->
  <!-- $http, $translate, $formatDate 可用且有类型提示 -->
  <div>{{ $translate('greeting') }}</div>
</template>
```

### 2.4 环境变量全局属性

```typescript
// types/global.d.ts
export {};

declare module 'vue' {
  interface ComponentCustomProperties {
    $env: {
      API_URL: string;
      APP_VERSION: string;
      IS_DEV: boolean;
    };
  }
}

// 注册
app.config.globalProperties.$env = {
  API_URL: import.meta.env.VITE_API_URL,
  APP_VERSION: import.meta.env.VITE_APP_VERSION,
  IS_DEV: import.meta.env.DEV,
};
```

### 2.5 TypeScript 配置

```json
// tsconfig.json — 确保类型声明被包含
{
  "include": [
    "src/**/*.ts",
    "src/**/*.d.ts",
    "src/**/*.vue"
  ]
}
```

## 三、注意事项与常见陷阱

1. **模块扩展文件必须被 tsconfig 包含**：放在 `src/types/` 目录
2. **`export {}` 确保文件是模块**：否则 `declare module` 不会生效
3. **优先使用 Composable 而非全局属性**：类型更安全，更容易测试
4. **全局属性在 `<script setup>` 中不自动可用**：需要通过模板或 `getCurrentInstance`
5. **`ComponentCustomProperties` 是 Vue 3 的扩展接口**：不是自定义的
