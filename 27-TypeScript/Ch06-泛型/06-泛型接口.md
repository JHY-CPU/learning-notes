# 泛型接口

## 一、概念说明

泛型接口在接口定义时引入类型参数，使接口可以描述多种数据结构。常用于定义通用的存储库、API 响应、集合等。

## 二、具体用法

### 2.1 基本泛型接口

```typescript
interface Repository<T> {
  getById(id: number): T | undefined;
  getAll(): T[];
  save(item: T): void;
  delete(id: number): boolean;
}

interface User {
  id: number;
  name: string;
}

class UserRepo implements Repository<User> {
  private users: User[] = [];

  getById(id: number): User | undefined {
    return this.users.find(u => u.id === id);
  }

  getAll(): User[] {
    return [...this.users];
  }

  save(user: User): void {
    this.users.push(user);
  }

  delete(id: number): boolean {
    const idx = this.users.findIndex(u => u.id === id);
    if (idx >= 0) { this.users.splice(idx, 1); return true; }
    return false;
  }
}

const repo = new UserRepo();
repo.save({ id: 1, name: "Alice" });
console.log(repo.getById(1));
// 输出: { id: 1, name: "Alice" }
```

### 2.2 多类型参数

```typescript
interface Cache<K, V> {
  get(key: K): V | undefined;
  set(key: K, value: V): void;
  has(key: K): boolean;
}

const map = new Map<string, number>();
const cache: Cache<string, number> = {
  get: (k) => map.get(k),
  set: (k, v) => map.set(k, v),
  has: (k) => map.has(k),
};

cache.set("age", 25);
console.log(cache.get("age"));
// 输出: 25
```

## 三、注意事项与常见陷阱

1. **泛型接口的实现类需指定类型**：`implements Repository<User>`
2. **也可保留泛型**：`class Repo<T> implements Repository<T>`
3. **接口默认类型**：`interface ApiResponse<T = unknown>`
4. **泛型约束**：`interface Repo<T extends { id: number }>`
