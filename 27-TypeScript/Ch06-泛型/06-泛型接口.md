# 泛型接口

## 一、概念说明

泛型接口在接口定义时引入类型参数，使接口可以描述多种数据结构。常用于定义通用的存储库、API 响应、集合等。泛型接口是框架和库设计的核心工具。

## 二、具体用法

### 2.1 基本泛型接口

```typescript
interface Repository<T> {
  getById(id: number): T | undefined;
  getAll(): T[];
  save(item: T): void;
  delete(id: number): boolean;
}

interface User {
  id: number;
  name: string;
}

class UserRepo implements Repository<User> {
  private users: User[] = [];

  getById(id: number): User | undefined {
    return this.users.find(u => u.id === id);
  }

  getAll(): User[] {
    return [...this.users];
  }

  save(user: User): void {
    this.users.push(user);
  }

  delete(id: number): boolean {
    const idx = this.users.findIndex(u => u.id === id);
    if (idx >= 0) { this.users.splice(idx, 1); return true; }
    return false;
  }
}

const repo = new UserRepo();
repo.save({ id: 1, name: "Alice" });
console.log(repo.getById(1));
// 输出: { id: 1, name: "Alice" }
```

### 2.2 多类型参数

```typescript
interface Cache<K, V> {
  get(key: K): V | undefined;
  set(key: K, value: V): void;
  has(key: K): boolean;
}

const map = new Map<string, number>();
const cache: Cache<string, number> = {
  get: (k) => map.get(k),
  set: (k, v) => map.set(k, v),
  has: (k) => map.has(k),
};

cache.set("age", 25);
console.log(cache.get("age"));
// 输出: 25
```

### 2.3 泛型接口与函数

```typescript
interface Comparator<T> {
  compare(a: T, b: T): number;
}

interface Sorter<T> {
  sort(items: T[], comparator: Comparator<T>): T[];
}

// 使用泛型接口定义排序策略
const numberComparator: Comparator<number> = {
  compare: (a, b) => a - b,
};
```

### 2.4 接口默认类型参数

```typescript
interface ApiResponse<T = unknown> {
  data: T;
  status: number;
  message: string;
}

// 不指定类型使用默认
const res1: ApiResponse = { data: {}, status: 200, message: "OK" };

// 指定类型
const res2: ApiResponse<User[]> = {
  data: [{ id: 1, name: "Alice" }],
  status: 200,
  message: "OK",
};
```

## 三、注意事项与常见陷阱

1. **泛型接口的实现类需指定类型**：`implements Repository<User>`
2. **也可保留泛型**：`class Repo<T> implements Repository<T>`（传递泛型参数）
3. **接口默认类型**：`interface ApiResponse<T = unknown>` 减少重复
4. **泛型约束**：`interface Repo<T extends { id: number }>` 约束类型参数
5. **接口继承泛型接口**：`interface UserRepo extends Repository<User>` 具体化泛型
6. **运行时擦除**：泛型参数编译后消失，不影响接口的使用
