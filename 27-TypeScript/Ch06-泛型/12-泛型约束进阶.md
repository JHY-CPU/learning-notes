# 泛型约束进阶

## 一、概念说明

进阶泛型约束包括多重约束、约束中的约束、条件约束等高级技巧。这些技术用于编写更精确、更类型安全的泛型代码，在库和框架设计中非常有用。

## 二、具体用法

### 2.1 多重约束（交叉类型）

```typescript
interface Loggable { log(): void; }
interface Serializable { toJSON(): string; }

// T 必须同时满足两个接口
function process<T extends Loggable & Serializable>(item: T): void {
  item.log();
  console.log(item.toJSON());
}
```

### 2.2 条件约束

```typescript
// K 只能是 T 中值为 string 类型的键
function getStringKeys<T extends object>(
  obj: T
): { [K in keyof T]: T[K] extends string ? K : never }[keyof T] {
  return Object.keys(obj).filter(
    k => typeof (obj as any)[k] === "string"
  ) as any;
}

const user = { name: "Alice", age: 25, email: "a@b.com" };
const keys = getStringKeys(user); // "name" | "email"
console.log(keys);
// 输出: ["name", "email"]
```

### 2.3 约束中的泛型

```typescript
function createWithDefault<T extends { new (): any } = { new (): object }>(
  Constructor: T
): InstanceType<T> {
  return new Constructor();
}
```

### 2.4 约束与映射类型

```typescript
// 只允许修改特定类型的属性
function updateStringProps<T extends object>(
  obj: T,
  updates: { [K in keyof T as T[K] extends string ? K : never]?: T[K] }
): void {
  Object.assign(obj, updates);
}

const user = { name: "Alice", age: 25, email: "a@b.com" };
updateStringProps(user, { name: "Bob", email: "b@c.com" }); // ✅
// updateStringProps(user, { age: 30 }); // ❌ age 不是 string 类型
```

### 2.5 可变参数约束

```typescript
// 限制参数数量和类型
function tuple<T extends unknown[]>(...args: T): T {
  return args;
}

const t = tuple(1, "hello", true); // [number, string, boolean]

// 限制尾部参数
function withLogger<F extends (...args: any[]) => any>(fn: F) {
  return (...args: Parameters<F>): ReturnType<F> => {
    console.log("调用:", args);
    return fn(...args);
  };
}
```

## 三、注意事项与常见陷阱

1. **交叉约束用 `&`**：联合约束不支持（`T extends A | B` 是另一种语义）
2. **条件约束过滤键**：如只取值为特定类型的键，用于精确 API 设计
3. **复杂约束影响推断**：过于复杂的约束可能导致推断失败，需要显式指定
4. **测试约束行为**：确保约束在各种场景下正确工作
5. **约束不影响运行时**：所有约束在编译后消失
6. **约束应有文档**：复杂的泛型约束应有注释说明意图
