# 泛型默认类型

## 一、概念说明

泛型默认类型允许为类型参数指定默认值 `<T = string>`，当调用时不显式指定类型参数，自动使用默认类型。这减少了冗余的类型声明，让泛型更易用。

## 二、具体用法

### 2.1 基本默认类型

```typescript
interface ApiResponse<T = unknown> {
  data: T;
  status: number;
}

// 不指定 T，使用默认 unknown
const res1: ApiResponse = { data: "anything", status: 200 };

// 指定 T
const res2: ApiResponse<number[]> = { data: [1, 2, 3], status: 200 };

console.log(res2.data);
// 输出: [1, 2, 3]
```

### 2.2 与约束结合

```typescript
function createArray<T = string>(length: number, fill: T): T[] {
  return Array(length).fill(fill);
}

const arr1 = createArray(3, "x");    // string[]
const arr2 = createArray(3, 0);      // number[]
const arr3 = createArray<string>(3, "a"); // 显式指定

console.log(arr1); // 输出: ["x", "x", "x"]
console.log(arr2); // 输出: [0, 0, 0]
```

### 2.3 常见内置默认类型

```typescript
// Promise 默认 void
async function logAsync(): Promise<void> {
  console.log("done");
}

// Map 默认 any
const map = new Map<string, number>();

// Fetch API
type RequestInit = {
  method?: string;
  headers?: Record<string, string>;
  body?: BodyInit | null;
};

// 泛型库常用默认
type Options<T = Record<string, unknown>> = {
  data?: T;
  timeout?: number;
};
```

### 2.4 多参数默认

```typescript
interface Cache<K = string, V = unknown> {
  get(key: K): V | undefined;
  set(key: K, value: V): void;
}

// 使用默认类型
const cache1: Cache = { get: () => undefined, set: () => {} };

// 指定部分类型
const cache2: Cache<number> = { get: () => undefined, set: () => {} };

// 全部指定
const cache3: Cache<string, User> = { get: () => undefined, set: () => {} };
```

## 三、注意事项与常见陷阱

1. **默认类型在推断失败时使用**：编译器能推断就不使用默认值
2. **约束优先**：`<T extends U = V>` 中 V 必须兼容 U
3. **接口常用默认类型**：如 `Partial<T>`、`Promise<T = void>`
4. **类和函数都支持**：默认类型不限于接口
5. **默认类型顺序**：有默认值的参数应放在无默认值的参数之后
6. **`unknown` 比 `any` 好**：默认类型用 `unknown` 更安全
