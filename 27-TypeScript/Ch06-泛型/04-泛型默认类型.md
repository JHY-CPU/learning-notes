# 泛型默认类型

## 一、概念说明

泛型默认类型允许为类型参数指定默认值 `<T = string>`，当调用时不显式指定类型参数，自动使用默认类型。这减少了冗余的类型声明。

## 二、具体用法

### 2.1 基本默认类型

```typescript
interface ApiResponse<T = unknown> {
  data: T;
  status: number;
}

// 不指定 T，使用默认 unknown
const res1: ApiResponse = { data: "anything", status: 200 };

// 指定 T
const res2: ApiResponse<number[]> = { data: [1, 2, 3], status: 200 };

console.log(res2.data);
// 输出: [1, 2, 3]
```

### 2.2 与约束结合

```typescript
function createArray<T = string>(length: number, fill: T): T[] {
  return Array(length).fill(fill);
}

const arr1 = createArray(3, "x");    // string[]
const arr2 = createArray(3, 0);      // number[]
const arr3 = createArray<string>(3, "a"); // 显式指定

console.log(arr1); // 输出: ["x", "x", "x"]
console.log(arr2); // 输出: [0, 0, 0]
```

## 三、注意事项与常见陷阱

1. **默认类型在推断失败时使用**：编译器能推断就不使用默认值
2. **约束优先**：`<T extends U = V>` 中 V 必须兼容 U
3. **接口常用默认类型**：如 `Partial<T>`、`Promise<T = void>`
4. **类和函数都支持**：默认类型不限于接口
