# 泛型类型推断

## 一、概念说明

TypeScript 可以从函数参数、返回值上下文等自动推断泛型参数的类型，避免手动指定。理解推断规则有助于编写无需显式类型参数的泛型代码。

## 二、具体用法

### 2.1 从参数推断

```typescript
function wrap<T>(value: T): { wrapped: T } {
  return { wrapped: value };
}

// T 自动推断为 string
const result = wrap("hello");
console.log(result.wrapped);
// 输出: hello
```

### 2.2 从多个参数推断

```typescript
function merge<T, U>(first: T, second: U): T & U {
  return { ...first, ...second };
}

// T = { name: string }, U = { age: number }
const merged = merge({ name: "Alice" }, { age: 25 });
console.log(merged);
// 输出: { name: "Alice", age: 25 }
```

### 2.3 最佳通用类型

```typescript
function first<T>(arr: T[]): T | undefined {
  return arr[0];
}

const num = first([1, 2, 3]);       // number | undefined
const mixed = first([1, "a", true]); // number | string | boolean | undefined
console.log(num);
// 输出: 1
```

## 三、注意事项与常见陷阱

1. **推断从左到右**：参数顺序影响推断结果
2. **推断失败时回退**：回退到约束类型或 `{}`
3. **显式优于隐式**：复杂场景显式指定更安全
4. **上下文类型传播**：赋值时类型信息双向传播
