# 泛型概述

## 一、概念说明

泛型（Generics）是 TypeScript 中编写**通用且类型安全**代码的核心机制。通过类型参数（如 `<T>`），函数、类、接口可以在定义时不指定具体类型，在使用时再确定。泛型避免了 `any` 带来的类型丢失问题。

## 二、具体用法

### 2.1 为什么需要泛型

```typescript
// ❌ 没有泛型：丢失类型信息
function identityAny(arg: any): any {
  return arg;
}
const result1 = identityAny("hello"); // 类型: any（丢失了 string 信息）

// ✅ 有泛型：保留类型信息
function identity<T>(arg: T): T {
  return arg;
}
const result2 = identity("hello"); // 类型: string（精确保留）

console.log(result1, result2);
// 输出: hello hello
```

### 2.2 泛型的核心价值

```typescript
// 类型安全的容器
class Box<T> {
  constructor(public value: T) {}
  getValue(): T { return this.value; }
}

const numBox = new Box(42);         // Box<number>
const strBox = new Box("hello");    // Box<string>

console.log(numBox.getValue()); // 输出: 42
console.log(strBox.getValue()); // 输出: hello
```

### 2.3 泛型的位置

```typescript
// 泛型函数
function wrap<T>(value: T): { wrapped: T } {
  return { wrapped: value };
}

// 泛型接口
interface ApiResponse<T> {
  data: T;
  status: number;
}

// 泛型类
class Stack<T> {
  private items: T[] = [];
  push(item: T): void { this.items.push(item); }
  pop(): T | undefined { return this.items.pop(); }
}

// 泛型类型别名
type Result<T, E> = { ok: true; value: T } | { ok: false; error: E };

console.log(wrap(42).wrapped); // 输出: 42
```

### 2.4 与 JavaScript 的对比

```javascript
// JavaScript —— 没有泛型，用 any 或不做约束
function first(arr) {
  return arr[0];
}
const item = first([1, 2, 3]); // item 的类型是 any
```

```typescript
// TypeScript —— 泛型保留类型
function first<T>(arr: T[]): T | undefined {
  return arr[0];
}
const item = first([1, 2, 3]); // item 的类型是 number | undefined
console.log(item); // 输出: 1
```

### 2.5 泛型的命名惯例

```typescript
// 常见泛型参数命名
// T — Type（通用类型）
// K — Key（键类型）
// V — Value（值类型）
// E — Element（元素类型）
// R — Result（返回类型）

function map<T, R>(arr: T[], fn: (item: T) => R): R[] {
  return arr.map(fn);
}

const lengths = map(["hello", "world"], (s) => s.length);
console.log(lengths); // 输出: [5, 5]
```

## 三、注意事项与常见陷阱

1. **类型参数运行时不存在**：`T` 编译后被擦除，不能 `typeof T` 或 `new T()`
2. **尽量让编译器推断**：不必显式指定类型参数，让 TypeScript 自动推断
3. **单字母命名惯例**：`T`、`U`、`K`、`V` 是通用约定，复杂场景用描述性名称
4. **不要过度泛型**：简单场景不需要泛型，`function greet(name: string)` 就够了
5. **泛型约束需要时再加**：先不加 `extends` 约束，需要时再添加
6. **泛型是编译时概念**：不影响运行时性能
