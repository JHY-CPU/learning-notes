# 泛型与类型擦除

## 一、概念说明

TypeScript 的泛型在编译后会被**擦除**（Type Erasure），运行时不存在类型参数信息。这意味着不能在运行时检查 `T` 的类型、不能 `new T()`、不能用 `T` 做 `instanceof`。这是 TypeScript 作为编译时类型系统的核心特征。

## 二、具体用法

### 2.1 类型擦除的表现

```typescript
function identity<T>(arg: T): T {
  return arg;
}

// 编译后变为:
// function identity(arg) { return arg; }

// 运行时无法获取 T 的信息
console.log(typeof identity("hello"));
// 输出: string（值的类型，不是泛型参数）
```

### 2.2 运行时的限制

```typescript
class Container<T> {
  private value: T;

  constructor(value: T) {
    this.value = value;
  }

  // ❌ 不能在运行时检查 T 的类型
  // isString(): boolean {
  //   return typeof this.value === "string"; // 无法引用 T
  // }

  // ✅ 手动传入类型检查函数
  isOfType(check: (v: unknown) => boolean): boolean {
    return check(this.value);
  }
}

const container = new Container("hello");
console.log(container.isOfType(v => typeof v === "string"));
// 输出: true
```

### 2.3 传递类型信息的模式

```typescript
// 通过构造函数传递类型信息
function createInstance<T>(type: new () => T): T {
  return new type();
}

class User {
  name = "Alice";
}

const user = createInstance(User);
console.log(user.name);
// 输出: Alice

// 通过标签传递类型信息
type TypedValue<T> = {
  value: unknown;
  __type: T;
};

function typed<T>(value: unknown): TypedValue<T> {
  return { value, __type: null as unknown as T };
}
```

## 三、注意事项与常见陷阱

1. **运行时无泛型**：`T` 在编译后完全消失，无法在运行时引用
2. **不能 `typeof T`、`new T()`、`T instanceof`**：这些都是运行时操作
3. **传递构造函数**：通过 `new () => T` 保留运行时类型信息
4. **JSON 序列化**：泛型信息不会保留到序列化结果中
5. **`interface` 也无运行时存在**：只有 `class` 在运行时存在
6. **类型守卫替代运行时泛型**：用 `typeof`、`instanceof`、`in` 等代替
