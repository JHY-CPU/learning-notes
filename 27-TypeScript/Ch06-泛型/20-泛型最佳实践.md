# 泛型最佳实践

## 一、概念说明

合理使用泛型可以编写通用且类型安全的代码，但过度使用会导致类型复杂、难以维护。遵循最佳实践，在灵活性和可读性之间找到平衡。

## 二、具体用法

### 2.1 何时使用泛型

```typescript
// ✅ 需要泛型：函数/类处理多种类型但保持类型关联
function first<T>(arr: T[]): T | undefined {
  return arr[0];
}

// ✅ 需要泛型：类型参数与返回类型关联
function identity<T>(value: T): T {
  return value;
}

// ❌ 不需要泛型：参数类型已足够
function greet(name: string): string {
  return `Hello, ${name}`;
}
```

### 2.2 命名约定

```typescript
// ✅ 标准命名
// T - 通用类型
// K - 键类型
// V - 值类型
// E - 元素类型
// R - 返回类型

// ✅ 有意义的命名（复杂场景）
type EventHandler<TEvent, TResult> = (event: TEvent) => TResult;
type Repository<TEntity, TId> = {
  findById(id: TId): TEntity | undefined;
};
```

### 2.3 约束的最佳实践

```typescript
// ✅ 只在需要时添加约束
function logLength<T extends { length: number }>(arg: T): T {
  console.log(arg.length);
  return arg;
}

// ❌ 过早约束
// function process<T extends object>(arg: T) { return arg; }
// 如果不需要访问属性，不需要约束
```

### 2.4 默认类型

```typescript
// ✅ 提供合理默认值
interface ApiResponse<T = unknown> {
  data: T;
  status: number;
}
```

## 三、注意事项与常见陷阱

1. **不要过度泛型**：能用具体类型就用具体类型
2. **有意义的命名**：单字母 `T` 用于简单场景，复杂场景用描述性名称
3. **约束不要过早**：先不加约束，需要时再添加
4. **导出函数的泛型**：确保用户能正确推断或指定类型参数
