# 泛型约束详解

## 一、概念说明

泛型约束使用 `extends` 关键字限制类型参数的范围，使其必须满足特定条件。这允许在泛型函数内部安全地访问约束类型的属性。约束让泛型从"完全未知"变为"至少满足某些条件"。

## 二、具体用法

### 2.1 基本约束

```typescript
interface HasLength {
  length: number;
}

function logLength<T extends HasLength>(arg: T): T {
  console.log(`长度: ${arg.length}`);
  return arg;
}

logLength("hello");     // string 有 length
logLength([1, 2, 3]);   // 数组有 length
logLength({ length: 5, name: "x" }); // 对象有 length
// logLength(42);       // ❌ number 没有 length
// 输出: 长度: 5
```

### 2.2 keyof 约束

```typescript
function getProperty<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}

const user = { name: "Alice", age: 25 };
console.log(getProperty(user, "name")); // 输出: Alice
console.log(getProperty(user, "age"));  // 输出: 25
// getProperty(user, "email"); // ❌ "email" 不在 keyof User 中
```

### 2.3 多重约束

```typescript
interface Printable { print(): void; }
interface Serializable { serialize(): string; }

function process<T extends Printable & Serializable>(item: T): string {
  item.print();
  return item.serialize();
}
```

### 2.4 约束与默认值

```typescript
// 约束结合默认类型
function createPair<T extends string | number = string>(
  value: T
): [T, T] {
  return [value, value];
}

const p1 = createPair("hello"); // [string, string]
const p2 = createPair(42);      // [number, number]
const p3 = createPair<string>("test"); // [string, string]
```

### 2.5 实际应用

```typescript
// 确保对象有 id 属性
function findById<T extends { id: string }>(items: T[], id: string): T | undefined {
  return items.find(item => item.id === id);
}

const users = [
  { id: "1", name: "Alice", email: "a@b.com" },
  { id: "2", name: "Bob", email: "b@c.com" },
];

const user = findById(users, "1");
console.log(user?.name); // 输出: Alice
```

## 三、注意事项与常见陷阱

1. **约束限制灵活性**：过于严格的约束限制泛型的通用性
2. **`extends` 在泛型中不是继承**：是类型约束，表示"至少满足"
3. **`keyof` 约束最常用**：确保 key 是对象的有效键
4. **默认约束**：`<T extends any = string>` 提供默认类型
5. **约束中的 `keyof` 不会遗漏属性**：只能传入已知的键名
6. **约束与推断**：约束帮助编译器推断更精确的类型
