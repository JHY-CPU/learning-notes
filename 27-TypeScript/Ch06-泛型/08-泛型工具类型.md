# 泛型工具类型

## 一、概念说明

TypeScript 内置了多个泛型工具类型（Utility Types），如 `Partial`、`Required`、`Pick`、`Omit`、`Record`、`ReturnType` 等。理解它们的实现原理有助于编写自定义工具类型。

## 二、具体用法

### 2.1 核心工具类型实现

```typescript
// Partial<T> - 所有属性可选
type MyPartial<T> = { [K in keyof T]?: T[K] };

// Required<T> - 所有属性必选
type MyRequired<T> = { [K in keyof T]-?: T[K] };

// Readonly<T> - 所有属性只读
type MyReadonly<T> = { readonly [K in keyof T]: T[K] };

// Pick<T, K> - 选取属性
type MyPick<T, K extends keyof T> = { [P in K]: T[P] };

// Record<K, V> - 键值类型
type MyRecord<K extends keyof any, V> = { [P in K]: V };
```

### 2.2 使用示例

```typescript
interface User {
  id: number;
  name: string;
  email: string;
}

type PartialUser = MyPartial<User>;
type ReadonlyUser = MyReadonly<User>;
type UserPreview = MyPick<User, "id" | "name">;

const partial: PartialUser = { name: "Alice" };
const preview: UserPreview = { id: 1, name: "Alice" };

console.log(partial, preview);
// 输出: { name: "Alice" } { id: 1, name: "Alice" }
```

### 2.3 Omit 和 Exclude

```typescript
// Omit<T, K> - 排除属性
type MyOmit<T, K extends keyof any> = Pick<T, Exclude<keyof T, K>>;

type UserWithoutEmail = Omit<User, "email">;
// { id: number; name: string; }

// Exclude<T, U> - 从联合类型中排除
type MyExclude<T, U> = T extends U ? never : T;

type AB = MyExclude<"a" | "b" | "c", "a">;
// "b" | "c"
```

### 2.4 ReturnType 和 Parameters

```typescript
// ReturnType<T> - 获取函数返回类型
type MyReturnType<T> = T extends (...args: any[]) => infer R ? R : never;

function getUser() {
  return { id: 1, name: "Alice", email: "a@b.com" };
}

type UserReturn = ReturnType<typeof getUser>;
// { id: number; name: string; email: string; }

// Parameters<T> - 获取函数参数类型
type MyParameters<T> = T extends (...args: infer P) => any ? P : never;

type GetUserParams = Parameters<typeof getUser>;
// []
```

### 2.5 自定义工具类型

```typescript
// 只读深度类型
type DeepPartial<T> = T extends object ? {
  [K in keyof T]?: DeepPartial<T[K]>;
} : T;

// 非空类型
type NonNullableProps<T> = {
  [K in keyof T]: NonNullable<T[K]>;
};

// 可写（移除 readonly）
type Mutable<T> = {
  -readonly [K in keyof T]: T[K];
};
```

## 三、注意事项与常见陷阱

1. **工具类型都是泛型**：需要传入类型参数
2. **浅层操作**：`Partial`、`Readonly` 等是浅层的，深层需递归实现
3. **可组合使用**：`Partial<Pick<T, K>>` 等组合提供更精确的类型操作
4. **了解原理有助自定义**：遇到特殊需求可自行实现工具类型
5. **`Exclude` vs `Omit`**：`Exclude` 操作联合类型值，`Omit` 操作对象类型键
6. **`Extract` vs `Exclude`**：`Extract` 取交集，`Exclude` 取差集
