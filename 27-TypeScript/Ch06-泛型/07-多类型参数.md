# 多类型参数

## 一、概念说明

泛型可以有多个类型参数 `<T, U, V>`，用于描述涉及多种类型的函数、类或接口。每个类型参数独立运作，可以有各自的约束和默认值。

## 二、具体用法

### 2.1 基本多参数

```typescript
function swap<T, U>(first: T, second: U): [U, T] {
  return [second, first];
}

const result = swap("hello", 42);
console.log(result);
// 输出: [42, "hello"]
```

### 2.2 映射函数

```typescript
function mapArray<T, U>(arr: T[], fn: (item: T, index: number) => U): U[] {
  return arr.map(fn);
}

const lengths = mapArray(["hello", "world", "ts"], (s) => s.length);
console.log(lengths);
// 输出: [5, 5, 2]
```

### 2.3 多参数约束

```typescript
function pick<T extends object, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}

const value = pick({ name: "Alice", age: 25 }, "name");
console.log(value);
// 输出: Alice
```

## 三、注意事项与常见陷阱

1. **命名惯例**：`T`、`U`、`V`、`K`、`R` 等
2. **每个参数独立约束**：`<T extends A, U extends B>`
3. **推断顺序**：从左到右推断
4. **过多参数降低可读性**：一般不超过 3-4 个
