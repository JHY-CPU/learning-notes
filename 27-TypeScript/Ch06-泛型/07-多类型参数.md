# 多类型参数

## 一、概念说明

泛型可以有多个类型参数 `<T, U, V>`，用于描述涉及多种类型的函数、类或接口。每个类型参数独立运作，可以有各自的约束和默认值。多类型参数常见于映射、转换、配对等场景。

## 二、具体用法

### 2.1 基本多参数

```typescript
function swap<T, U>(first: T, second: U): [U, T] {
  return [second, first];
}

const result = swap("hello", 42);
console.log(result);
// 输出: [42, "hello"]
```

### 2.2 映射函数

```typescript
function mapArray<T, U>(arr: T[], fn: (item: T, index: number) => U): U[] {
  return arr.map(fn);
}

const lengths = mapArray(["hello", "world", "ts"], (s) => s.length);
console.log(lengths);
// 输出: [5, 5, 2]
```

### 2.3 多参数约束

```typescript
function pick<T extends object, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}

const value = pick({ name: "Alice", age: 25 }, "name");
console.log(value);
// 输出: Alice
```

### 2.4 三个以上参数

```typescript
function reduce<T, U, R>(
  arr: T[],
  initial: U,
  fn: (acc: U, item: T, index: number) => R
): R {
  return arr.reduce((acc, item, idx) => fn(acc as U, item, idx), initial as unknown as R);
}

const sum = reduce([1, 2, 3, 4, 5], 0, (acc, n) => acc + n);
console.log(sum); // 输出: 15
```

### 2.5 实际应用

```typescript
// API 请求类型
type ApiRequest<TBody, TQuery, TParams> = {
  body: TBody;
  query: TQuery;
  params: TParams;
};

type CreateUserRequest = ApiRequest<
  { name: string; email: string },  // body
  { sendWelcome?: boolean },         // query
  {}                                  // params
>;

// 响应类型
type ApiResult<T, E> =
  | { success: true; data: T }
  | { success: false; error: E };

function handleResult<T, E>(result: ApiResult<T, E>): T | null {
  if (result.success) {
    return result.data;
  }
  console.error(result.error);
  return null;
}
```

## 三、注意事项与常见陷阱

1. **命名惯例**：`T`、`U`、`V`、`K`、`R` 等有约定含义
2. **每个参数独立约束**：`<T extends A, U extends B>` 分别约束
3. **推断顺序**：从左到右推断类型参数
4. **过多参数降低可读性**：一般不超过 3-4 个，超过考虑拆分
5. **类型参数可以互相引用**：`<T, U extends T>` 表示 U 是 T 的子类型
6. **默认参数顺序**：有默认值的参数应放在最后
