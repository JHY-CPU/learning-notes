# 递归泛型

## 一、概念说明

递归泛型是引用自身的泛型类型，用于描述深度嵌套的数据结构（如 JSON、树、链表）或递归处理类型。

## 二、具体用法

### 2.1 JSON 类型

```typescript
type Json =
  | string
  | number
  | boolean
  | null
  | Json[]
  | { [key: string]: Json };

const data: Json = {
  name: "Alice",
  age: 25,
  hobbies: ["reading", "coding"],
  address: { city: "Beijing", zip: "100000" },
};

console.log(data);
// 输出: { name: "Alice", age: 25, hobbies: [...], address: {...} }
```

### 2.2 深度只读

```typescript
type DeepReadonly<T> = T extends object
  ? { readonly [K in keyof T]: DeepReadonly<T[K]> }
  : T;

interface Config {
  server: {
    host: string;
    port: number;
    ssl: {
      enabled: boolean;
    };
  };
}

type FrozenConfig = DeepReadonly<Config>;
// 所有层级都变为 readonly
```

### 2.3 递归展开 Promise

```typescript
type DeepAwaited<T> = T extends Promise<infer U> ? DeepAwaited<U> : T;

type R = DeepAwaited<Promise<Promise<Promise<string>>>>;
// string
```

## 三、注意事项与常见陷阱

1. **递归深度限制**：TypeScript 对递归有深度限制（约 1000 层）
2. **性能影响**：深度递归可能显著降低类型检查速度
3. **必须有终止条件**：泛型递归也需要基本情况
4. **测试复杂递归**：确保类型在各种输入下正确计算
