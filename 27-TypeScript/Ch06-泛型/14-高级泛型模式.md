# 高级泛型模式

## 一、概念说明

高级泛型模式包括 Builder 模式、Fluent 接口、类型安全的状态机等。这些模式利用泛型实现编译时约束，确保 API 的正确使用，减少运行时错误。

## 二、具体用法

### 2.1 类型安全的 Builder

```typescript
class QueryBuilder<Selected = false, Ordered = false> {
  private query: string[] = [];

  select(fields: string): QueryBuilder<true, Ordered> {
    this.query.push(`SELECT ${fields}`);
    return this as any;
  }

  orderBy(field: string): QueryBuilder<Selected, true> {
    this.query.push(`ORDER BY ${field}`);
    return this as any;
  }

  // 必须先 select 才能 build
  build(this: QueryBuilder<true, Ordered>): string {
    return this.query.join(" ");
  }
}

const query = new QueryBuilder()
  .select("name, age")
  .orderBy("name")
  .build();

console.log(query);
// 输出: SELECT name, age ORDER BY name
```

### 2.2 类型安全的链式调用

```typescript
class FluentArray<T> {
  constructor(private items: T[]) {}

  map<U>(fn: (item: T) => U): FluentArray<U> {
    return new FluentArray(this.items.map(fn));
  }

  filter(fn: (item: T) => boolean): FluentArray<T> {
    return new FluentArray(this.items.filter(fn));
  }

  toArray(): T[] {
    return [...this.items];
  }
}

const result = new FluentArray([1, 2, 3, 4, 5])
  .filter(n => n % 2 === 0)
  .map(n => n * 10)
  .toArray();

console.log(result);
// 输出: [20, 40]
```

### 2.3 类型安全的状态机

```typescript
type State = "idle" | "loading" | "success" | "error";

interface StateMachine<S extends State> {
  state: S;
  transition: S extends "idle"
    ? () => StateMachine<"loading">
    : S extends "loading"
    ? () => StateMachine<"success" | "error">
    : () => StateMachine<"idle">;
}
```

### 2.4 类型安全的事件发射器

```typescript
interface EventMap {
  click: { x: number; y: number };
  scroll: { delta: number };
  resize: { width: number; height: number };
}

class TypedEmitter<T extends Record<string, unknown>> {
  private handlers = new Map<string, Function[]>();

  on<K extends keyof T>(event: K, handler: (data: T[K]) => void): void {
    const list = this.handlers.get(event as string) ?? [];
    list.push(handler);
    this.handlers.set(event as string, list);
  }

  emit<K extends keyof T>(event: K, data: T[K]): void {
    const list = this.handlers.get(event as string) ?? [];
    list.forEach(h => h(data));
  }
}

const emitter = new TypedEmitter<EventMap>();
emitter.on("click", (data) => {
  console.log(`点击: ${data.x}, ${data.y}`); // data 自动推断为 { x: number; y: number }
});
```

## 三、注意事项与常见陷阱

1. **类型参数追踪状态**：如 Builder 的 `Selected`、`Ordered` 追踪构建步骤
2. **Fluent 接口返回新实例**：或返回 `this` 以支持链式调用
3. **编译时检查**：利用泛型约束确保 API 正确使用
4. **平衡复杂度**：过于复杂的泛型降低可读性和维护性
5. **类型推断辅助**：好的泛型设计应尽量减少显式类型参数的需要
6. **文档很重要**：复杂泛型模式需要详细的使用文档
