# 泛型函数基础

## 一、概念说明

泛型函数在函数名后添加类型参数 `<T>`，使函数可以处理任意类型。类型参数在调用时被推断或显式指定，确保返回类型与输入类型一致。泛型函数是 TypeScript 中最常用的泛型形式。

## 二、具体用法

### 2.1 基本泛型函数

```typescript
function identity<T>(arg: T): T {
  return arg;
}

// 类型推断
const str = identity("hello"); // T = string
const num = identity(42);      // T = number
const bool = identity(true);   // T = boolean

console.log(str, num, bool);
// 输出: hello 42 true
```

### 2.2 多类型参数

```typescript
function pair<T, U>(first: T, second: U): [T, U] {
  return [first, second];
}

const p = pair("age", 25); // [string, number]
console.log(p);
// 输出: ["age", 25]
```

### 2.3 泛型数组函数

```typescript
function toArray<T>(...items: T[]): T[] {
  return items;
}

const numbers = toArray(1, 2, 3);       // number[]
const strings = toArray("a", "b", "c"); // string[]

console.log(numbers); // 输出: [1, 2, 3]
console.log(strings); // 输出: ["a", "b", "c"]
```

### 2.4 显式指定类型参数

```typescript
function parse<T>(input: string): T {
  return JSON.parse(input) as T;
}

const data = parse<{ name: string }>('{"name":"Alice"}');
console.log(data.name);
// 输出: Alice
```

### 2.5 实际应用：类型安全的工具函数

```typescript
// 数组去重（保持类型）
function unique<T>(arr: T[]): T[] {
  return [...new Set(arr)];
}

const nums = unique([1, 2, 2, 3, 3]); // number[]
const strs = unique(["a", "b", "a"]); // string[]
console.log(nums); // 输出: [1, 2, 3]
console.log(strs); // 输出: ["a", "b"]

// 数组分组
function groupBy<T, K extends string | number>(arr: T[], fn: (item: T) => K): Record<K, T[]> {
  const result = {} as Record<K, T[]>;
  for (const item of arr) {
    const key = fn(item);
    (result[key] ??= []).push(item);
  }
  return result;
}

const grouped = groupBy([1, 2, 3, 4, 5], (n) => (n % 2 === 0 ? "even" : "odd"));
console.log(grouped);
// 输出: { odd: [1, 3, 5], even: [2, 4] }
```

## 三、注意事项与常见陷阱

1. **编译器能推断就不指定**：显式类型参数仅在推断失败时使用
2. **类型参数不是值**：不能 `new T()` 或 `T.toString()`，因为编译后消失
3. **约束需要时再加**：不要过早 `extends` 约束，保持泛型的通用性
4. **返回值类型由推断决定**：不必总是显式标注返回类型
5. **泛型函数的类型推断顺序**：从左到右，从参数推断类型参数
6. **泛型与回调结合**：回调函数的类型参数也能推断（如 `Array.map`）
