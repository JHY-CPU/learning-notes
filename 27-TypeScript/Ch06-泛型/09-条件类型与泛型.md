# 条件类型与泛型

## 一、概念说明

条件类型结合泛型，可以根据类型参数的关系选择不同的类型结果。语法 `T extends U ? X : Y` 在泛型中实现了类型级别的"条件分支"。条件类型是 TypeScript 类型编程的核心工具。

## 二、具体用法

### 2.1 基本条件类型

```typescript
type IsString<T> = T extends string ? "yes" : "no";

type A = IsString<"hello">; // "yes"
type B = IsString<42>;      // "no"

console.log("条件类型在编译时计算");
// 输出: 条件类型在编译时计算
```

### 2.2 配合 infer

```typescript
type UnpackPromise<T> = T extends Promise<infer U> ? U : T;

type R1 = UnpackPromise<Promise<string>>; // string
type R2 = UnpackPromise<number>;           // number
```

### 2.3 分布式条件类型

```typescript
type ToArray<T> = T extends unknown ? T[] : never;
type Result = ToArray<string | number>; // string[] | number[]
// 联合类型逐个分发
```

### 2.4 实际应用

```typescript
// API 响应类型分支
type ApiResult<T> = T extends void
  ? { success: boolean }
  : { success: boolean; data: T };

type VoidResult = ApiResult<void>;       // { success: boolean }
type DataResult = ApiResult<User>;       // { success: boolean; data: User }

// 属性类型过滤
type StringProps<T> = {
  [K in keyof T]: T[K] extends string ? K : never;
}[keyof T];

interface User {
  id: number;
  name: string;
  email: string;
}

type UserStrings = StringProps<User>; // "name" | "email"
```

### 2.5 条件类型与 never

```typescript
// never 的特殊行为：never extends T ? A : B 结果为 never
type Test = never extends string ? "yes" : "no"; // never

// 过滤联合类型
type NonString<T> = T extends string ? never : T;
type Filtered = NonString<string | number | boolean>;
// number | boolean（string 被过滤为 never 并消除）
```

## 三、注意事项与常见陷阱

1. **分布式特性**：联合类型会被逐个分发处理，如 `T extends U ? X : Y` 对 `A | B` 分别处理
2. **`infer` 只能用在条件类型中**：不能单独使用
3. **`never` 的特殊行为**：`never extends T ? A : B` 结果为 `never`
4. **条件类型可嵌套**：但过多嵌套影响可读性
5. **用 `[]` 包裹防止分布式**：`[T] extends [U] ? X : Y` 不会分发
6. **条件类型是类型级编程**：编译时计算，不影响运行时
