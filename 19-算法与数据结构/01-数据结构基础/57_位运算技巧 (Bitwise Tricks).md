# Bitwise Tricks

### 什么是位运算技巧

位运算直接操作二进制位，比算术运算更快。常见技巧包括利用与、或、异或、移位等操作实现特定功能。

### 关键特性

- **判断奇偶**：n & 1，结果为 1 是奇数，0 是偶数
- **判断 2 的幂**：n > 0 && (n & (n-1)) === 0
- **清除最低位 1**：n & (n-1)，常用于统计 1 的个数
- **取最低位 1**：n & (-n)，利用补码特性
- **交换变量**：a ^= b; b ^= a; a ^= b（无需临时变量）

### 时间与空间复杂度

| 操作 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 判断奇偶 | O(1) | O(1) |
| 判断 2 的幂 | O(1) | O(1) |
| 统计 1 的个数 | O(k) k 为 1 的个数 | O(1) |
| 交换变量 | O(1) | O(1) |

### 适用场景 vs 替代方案

- **性能敏感代码**：位运算比模除、取余更快
- **嵌入式/底层编程**：寄存器操作必须用位运算
- **算法竞赛**：很多技巧可减少代码量
- **替代**：可读性优先时用普通算术运算

### 常见陷阱

- 负数的位运算行为与正数不同（补码表示）
- 移位超过位数的行为在不同语言中未定义
- 异或交换两个相同变量会得到 0（x ^ x = 0）

```
// 常用位运算技巧
console.log(`奇数? 7&1: ${7&1}`);  // 1
console.log(`偶数? 8&1: ${8&1}`);  // 0
console.log(`2的幂? 16&(16-1): ${16&(16-1)}`); // 0
// 清除最低位的1
const x = 0b1011000;
console.log(`清除最低位1: ${x & (x-1)}`); // 1010000
// 取最低位1
console.log(`取最低位1: ${x & (-x)}`); // 1000
// 交换变量
let p=3, q=5; p^=q; q^=p; p^=q;
console.log(`交换后: p=${p}, q=${q}`); // p=5, q=3
```


### 实际应用

- **权限系统**：用位标志组合多个权限（读=1, 写=2, 执行=4）
- **图形渲染**：快速计算奇偶像素实现棋盘格效果
- **哈希函数**：用异或和移位混合多个哈希值
- **网络协议**：用位运算提取和设置标志位
