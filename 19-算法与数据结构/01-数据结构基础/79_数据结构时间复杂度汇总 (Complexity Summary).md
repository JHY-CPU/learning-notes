# Complexity Summary

### 数据结构时间复杂度全景

掌握各数据结构的增删查改时间复杂度，是选择合适数据结构和分析算法效率的基础。

### 关键特性

- **线性结构**：数组随机访问 O(1)，链表插入删除 O(1)
- **哈希结构**：平均 O(1) 但无序，最坏 O(n)
- **树形结构**：有序且平衡时 O(log n)，退化时 O(n)
- **堆结构**：取最值 O(1)，插入删除 O(log n)

### 时间复杂度汇总

| 数据结构 | 查找 | 插入 | 删除 | 访问 |
|---------|------|------|------|------|
| 数组 | O(n) | O(n) | O(n) | O(1) |
| 链表 | O(n) | O(1) | O(1) | O(n) |
| 栈/队列 | O(n) | O(1) | O(1) | O(n) |
| 哈希表 | O(1) | O(1) | O(1) | - |
| BST（平衡） | O(log n) | O(log n) | O(log n) | - |
| 堆 | O(n) | O(log n) | O(log n) | O(1) 取最值 |
| 跳表 | O(log n) | O(log n) | O(log n) | - |

### 适用场景 vs 替代方案

- 需要随机访问：数组
- 频繁插入删除：链表或跳表
- 快速查找：哈希表（无序）或 BST（有序）
- 最值查询：堆

### 常见陷阱

- 忽略最坏情况只看平均复杂度
- 忘记考虑均摊复杂度（如动态数组扩容）
- 混淆"按值查找"和"按索引访问"的区别

```
// 常见数据结构操作时间复杂度

// 队列: enqueue O(1), dequeue O(1)
// 哈希表: 插入 O(1), 删除 O(1), 查找 O(1)
// 二叉搜索树: 插入 O(log n), 删除 O(log n), 查找 O(log n)
// 堆: 插入 O(log n), 取最值 O(1), 删除最值 O(log n)
// 跳表: 插入 O(log n), 删除 O(log n), 查找 O(log n)
```


### 实际应用

- **面试准备**：快速回忆各种数据结构的复杂度差异
- **性能优化**：识别代码中的性能瓶颈并选择更优结构
- **系统设计**：根据读写比例选择合适的底层数据结构
- **算法分析**：判断算法是否满足时间和空间约束
