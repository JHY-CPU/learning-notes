# Bitwise Sets

### 位运算表示集合

用整数的每一位表示一个元素是否在集合中，第 i 位为 1 表示第 i 个元素存在。适用于元素数量不超过 32/64 的小集合。

### 关键特性

- **并集**：A | B
- **交集**：A & B
- **差集**：A & ~B
- **对称差集**：A ^ B
- **子集判断**：(A & B) === A 表示 A 是 B 的子集

### 时间与空间复杂度

| 操作 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 添加元素 | O(1) | O(1) 整数 |
| 删除元素 | O(1) | O(1) |
| 判断存在 | O(1) | O(1) |
| 并/交/差 | O(1) | O(1) |
| 遍历集合 | O(σ) | O(1) |

σ 为全集大小（最多 32 或 64）。

### 适用场景 vs 替代方案

- **小集合运算**：位运算集合比 HashSet 更快更省内存
- **状态压缩 DP**：用整数表示子集状态
- **枚举子集**：利用位运算高效遍历所有子集
- **替代**：元素超过 64 个时必须用 HashSet

### 常见陷阱

- 元素数量超过整数位数时无法使用
- JavaScript 位运算将操作数视为 32 位有符号整数
- 遍历子集时的边界条件容易出错

```
// 用整数表示集合（最多32个元素）
const universe = ['A','B','C','D','E'];
let set = 0;
function add(s, i) { return s | (1 << i); }
function remove(s, i) { return s & ~(1 << i); }
function has(s, i) { return (s & (1 << i)) !== 0; }
function toArray(s) { return universe.filter((_,i) => has(s,i)); }
set = add(set, 0); // A
set = add(set, 2); // C
set = add(set, 4); // E
console.log(toArray(set)); // ['A','C','E']
set = remove(set, 2);
console.log(toArray(set)); // ['A','E']
```


### 实际应用

- **状态压缩 DP**：旅行商问题中用位掩码表示已访问城市集合
- **权限管理**：Linux 文件权限 rwx 用 3 位表示
- **游戏开发**：用位掩码表示角色拥有的道具集合
- **编译器**：用位向量表示变量的活跃集合进行数据流分析
