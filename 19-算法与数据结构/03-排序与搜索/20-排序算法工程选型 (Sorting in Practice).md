## 排序算法工程选型 (Sorting in Practice)

  在实际工程中，选择合适的排序算法不仅要考虑时间复杂度，还需要考虑数据特征、稳定性、内存限制等多方面因素。


>
    **黄金法则：**没有完美的排序算法，只有最适合当前场景的排序算法。


  ## 各语言内置排序











  | 语言/平台 | 使用算法 | 特点 |
| --- | --- | --- |
| JavaScript (V8) | TimSort | 归并+插入的混合算法 |
| Python | Timsort | 稳定，最坏 O(n log n) |
| Java (对象) | TimSort | 稳定排序 |
| Java (原始类型) | Dual-Pivot QuickSort | 双轴快排 |
| C++ std::sort | IntroSort | 快排+堆排+插入混合 |
| Rust | 迭代归并排序 | O(n log n)，稳定 |
| Go | pdqsort | 模式感知快排 |
| .NET | IntroSort | 内省排序 |

  ## 选型决策树


```

开始选型:
├─ 数据能否全部加载到内存?
│  ├─ 否 → 外部排序（多路归并）
│  └─ 是 →
│     ├─ 数据范围很小（如整数 0-100）?
│     │  └─ 是 → 计数排序
│     ├─ 数据分布均匀?
│     │  └─ 是 → 桶排序
│     ├─ 需要稳定排序?
│     │  ├─ 是 → 归并排序 / TimSort
│     │  └─ 否 → 快速排序 / 堆排序
│     ├─ 内存非常有限?
│     │  └─ 是 → 堆排序（O(1) 空间）
│     ├─ 数据接近有序?
│     │  └─ 是 → 插入排序
│     └─ 大规模随机数据?
│        └─ 快速排序（通常最快）```
