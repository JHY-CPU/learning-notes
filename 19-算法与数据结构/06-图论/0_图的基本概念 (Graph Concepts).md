# 图的基本概念 (Graph Concepts)

  图 G = (V, E) 由顶点集 V 和边集 E 组成。分为有向图和无向图。


> 关键术语：顶点、边、度、路径、环、连通分量、子图

## 图的分类


    - **无向图：**边无方向，如朋友关系网络

    - **有向图：**边有方向，如网页链接关系

    - **加权图：**边有权重，如地图中的距离

    - **稀疏图 vs 稠密图：**比较 |E| 与 |V|² 的关系

    - **简单图 vs 多重图：**是否存在平行边和自环



## 相关术语


    - **度 (Degree)：**顶点关联的边数。有向图中分为入度和出度

    - **路径 (Path)：**顶点序列，相邻顶点之间有边相连

    - **环/圈 (Cycle)：**起点和终点相同的路径

    - **连通分量：**无向图中的极大连通子图

    - **完全图：**任意两个顶点之间都有边



## 图的存储方式

  ### 邻接矩阵
  使用二维数组 `matrix[i][j]` 表示顶点 i 和 j 之间是否有边。适合稠密图，查询边 O(1)，空间 O(V^2)。

  ### 邻接表
  使用数组+链表（或哈希表）存储每个顶点的邻居列表。适合稀疏图，空间 O(V+E)，遍历邻居 O(degree)。

  ### 边列表
  直接存储所有边的列表 `[u, v, weight]`。适合 Kruskal 等按边处理的算法，空间 O(E)。

```javascript
// 邻接表表示（JavaScript）
const graph = {
  0: [{to: 1, w: 5}, {to: 2, w: 3}],
  1: [{to: 3, w: 2}],
  2: [{to: 1, w: 1}, {to: 3, w: 6}],
  3: []
};

// 邻接矩阵表示
const matrix = [
  [0, 5, 3, 0],
  [0, 0, 0, 2],
  [0, 1, 0, 6],
  [0, 0, 0, 0]
];
```

```cpp
// C++ 邻接表
#include <vector>
using namespace std;

struct Edge { int to; int weight; };
vector<vector<Edge>> adj(n);  // n 个顶点

// 添加边 (u -> v, 权重 w)
void addEdge(int u, int v, int w) {
    adj[u].push_back({v, w});
}
```

## 图的遍历框架

  所有图算法的基础是两种遍历：BFS（广度优先）和 DFS（深度优先）。理解遍历是掌握图论的关键。

## 实际应用

  - **社交网络：**用户是顶点，关注/好友关系是边。无向图表示互关，有向图表示单向关注
  - **交通网络：**城市是顶点，道路是边，边权可以是距离或时间
  - **网页图：**网页是顶点，超链接是有向边，PageRank 算法基于此
  - **依赖管理：**软件包是顶点，依赖关系是有向边，拓扑排序决定安装顺序

## 常见陷阱

  - 有向图和无向图的性质差异很大，建模时需注意
  - 邻接矩阵在稀疏图中浪费大量空间
  - 图中可能存在自环（边连接自身）和多重边（两点间多条边）
  - 编号从 0 还是 1 开始需保持一致
