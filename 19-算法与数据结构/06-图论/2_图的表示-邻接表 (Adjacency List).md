# 图的表示 - 邻接表 (Adjacency List)

  为每个顶点维护一个邻接顶点列表（或数组）。适合稀疏图，空间复杂度 O(V+E)。

## 什么是邻接表

  邻接表使用数组（或哈希表）存储图中每个顶点的邻接关系。对于顶点 i，`graph[i]` 是一个列表，包含所有与 i 直接相邻的顶点。如果是有权图，列表中存储 `(neighbor, weight)` 对。这是实际工程中最常用的图表示方式，因为它在稀疏图上空间效率极高。

## 关键性质

    - 空间与边数成正比：O(V+E)，而非 O(V²)
    - 遍历顶点 u 的所有邻居只需 O(degree(u)) 时间
    - 无向图中每条边被存储两次（两端各一次）
    - 顶点可以用连续整数编号，也可用字符串等作为键

## 复杂度分析

    - **空间：**O(V+E)，优于邻接矩阵
    - **查询边 (u,v)：**O(degree(u))，需要遍历邻接列表
    - **添加边：**O(1)，直接追加到列表
    - **删除边：**O(degree(u))，需要查找并移除
    - **遍历所有邻边：**O(degree(u))，最优

## 适用场景 vs 替代方案

    - 绝大多数算法竞赛和工程场景的首选表示
    - 稀疏图（E << V²）时空间优势明显
    - 需要频繁遍历邻居时（如 BFS、DFS、Dijkstra）
    - 需要频繁判断边存在性时，邻接矩阵更优
    - 边列表（edge list）适合按边遍历的算法如 Kruskal

## 常见陷阱

    - 无向图添加边时，两端都要添加，否则遍历会遗漏
    - 使用对象键时，数字键会被转为字符串，需注意类型转换
    - 删除边操作不常见，但实现时容易漏掉一端

```javascript
// 无向图邻接表
const graph = {
  A: ['B', 'C', 'D'],
  B: ['A', 'C'],
  C: ['A', 'B', 'D'],
  D: ['A', 'C']
};
```

## 实际应用

  社交网络中，每个用户是一个顶点，好友关系是边。使用邻接表可以高效地查找某用户的所有好友（推荐算法的基础）。搜索引擎的网页链接图也是天然的邻接表结构，每个网页的出链列表即为其邻接表。

## 优缺点


    - **优点：**空间省，O(V+E)，适合稀疏图

    - **优点：**遍历某顶点的所有邻边快速

    - **缺点：**判断两顶点是否相邻需要 O(degree)

    - **缺点：**删除边操作较慢
