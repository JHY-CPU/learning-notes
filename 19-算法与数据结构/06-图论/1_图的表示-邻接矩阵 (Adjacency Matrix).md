# 图的表示 - 邻接矩阵 (Adjacency Matrix)

  使用 N x N 的二维数组表示图。matrix[i][j] = 1 表示存在从 i 到 j 的边（无权图），或存储权重值。

## 什么是邻接矩阵

  邻接矩阵是图的最基本表示方式之一。对于 n 个顶点的图，使用 n x n 的二维数组 `matrix` 来表示顶点之间的关系。若 `matrix[i][j] = 1`（无权图）或 `matrix[i][j] = w`（有权图，w 为权重），则表示顶点 i 到 j 之间存在一条边。对于无向图，矩阵沿主对角线对称；对于有向图，矩阵通常不对称。

## 关键性质

    - 对角线元素 `matrix[i][i]` 通常为 0（无自环）或自环权重
    - 无向图的矩阵是对称的，即 `matrix[i][j] === matrix[j][i]`
    - 有权图中，不存在的边通常用 `Infinity` 或 -1 表示
    - 空间开销与边的数量无关，固定为 O(V²)

## 复杂度分析

    - **空间：**O(V²)，无论图有多少条边
    - **查询边：**O(1)，直接通过下标访问
    - **添加/删除边：**O(1)
    - **遍历所有邻边：**O(V)，需要扫描整行
    - **遍历所有边：**O(V²)，需扫描整个矩阵

## 适用场景 vs 替代方案

    - 顶点数较少（V < 10000）或图比较稠密时首选
    - 需要频繁判断两顶点是否相邻时，比邻接表高效
    - 稀疏图（E 远小于 V²）应使用邻接表节省空间
    - Floyd-Warshall 算法天然适合邻接矩阵表示

## 常见陷阱

    - 顶点数很大时（如 10^5），O(V²) 空间会导致内存超限
    - 无向图只需存储上三角或下三角，但代码复杂度增加
    - 注意区分有向图和无向图的存储差异

```javascript
// 无向无权图（4个顶点）
const matrix = [
  [0, 1, 1, 0],  // A
  [1, 0, 1, 0],  // B
  [1, 1, 0, 1],  // C
  [0, 0, 1, 0]   // D
];
```

## 实际应用

  在竞赛编程中，Floyd-Warshall 全源最短路算法直接在邻接矩阵上进行 DP。在城市交通网络中，当城市数量不超过几百个时，邻接矩阵可以快速查询任意两个城市之间是否有直达道路，并存储通行时间作为权重。

## 优缺点


    - **优点：**判断边是否存在 O(1)，添加/删除边 O(1)

    - **优点：**实现简单直观

    - **缺点：**空间复杂度 O(V²)，不适合稀疏图

    - **缺点：**遍历所有边需要 O(V²)
