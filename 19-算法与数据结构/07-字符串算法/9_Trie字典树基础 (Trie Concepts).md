# 10 - Trie 字典树基础 (Trie Concepts)

  ## 什么是 Trie？

  Trie（又称字典树、前缀树）是一种**多叉树结构**，用于高效地存储和检索字符串集合中的键。Trie 的核心思想是利用字符串的**公共前缀**来减少查询时间，最大限度地减少不必要的字符比较。


>
    **词源：** "Trie" 来自 "retrieval"（检索），发音为"try"（与 "tree" 区分）。

    **基本性质：**

      - 根节点不包含字符，根节点外的每个节点包含一个字符

      - 从根节点到某个节点的路径上的字符连接起来就是该节点对应的字符串

      - 每个节点的子节点对应的字符通常不同

      - 通常用**标记位**表示某个节点是否是一个完整单词的结尾




  ## Trie 的结构示例

  插入单词: **"and", "ant", "do", "dad", "ball"**


```

        (根)
       /  |  \
      a   b   d
     /    |    |\
    n     a    o  a
   / \    |       |
  d  t    l       d
  ↑  ↑    ↑       ↑
 and ant  ball    dad
  ```

  ↑ 标记表示单词结尾。

  ## 基本操作



    ****
    ****
    ****
    ****
  | 操作 | 描述 | 时间复杂度 |
| --- | --- | --- |
| 插入 | (Insert) | 将一个字符串插入 Trie 中 |
| 查找 | (Search) | 在 Trie 中查找一个字符串是否存在 |
| 前缀查询 | (StartsWith) | 判断是否有以给定前缀开头的字符串 |
| 删除 | (Delete) | 从 Trie 中删除一个字符串 |


>
    Trie 的插入和查找时间复杂度为 O(m)，与集合中已有多少元素**无关**。相比之下，平衡 BST 需要 O(m·log n)，哈希表需要 O(m) 平均但存在哈希冲突风险。


  ## 优缺点








  | 优点 | 缺点 |
| --- | --- |
| 前缀查询极其高效（O(m)） | 空间消耗大（每个节点存多个指针） |
| 查找时间与数据集大小无关 | 对长字符串且无公共前缀的情况浪费空间 |
| 支持自动补全、拼写检查 | 实现比哈希表复杂 |
| 字符串有序存储（字典序） | 对大量短字符串效率不如哈希表 |
| 无哈希冲突 | 指针开销可能很大（尤其 JS 对象） |

  ## 两种实现方式

  ### 1. 数组实现（固定字符集）


```javascript

// 每个节点维护一个大小为26的数组（仅限小写字母）
// 速度快，但可能浪费空间
class TrieNode {
  constructor() {
    this.children = new Array(26);  // 26个小写字母
    this.isEnd = false;
  }
}
// childIndex = charCode - 'a'.charCodeAt(0)
  ```

  ### 2. 哈希表/对象实现（通用）


```javascript

// 每个节点维护一个哈希表（支持任意字符）
// 空间高效，但略慢
class TrieNode {
  constructor() {
    this.children = {};   // { char: TrieNode }
    this.isEnd = false;
  }
}
  ```
