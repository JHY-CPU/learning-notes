## 概念说明

字符串算法是算法面试和工程实践中的高频考点。不同场景需要选择不同算法：单模式匹配用 KMP，多模式匹配用 AC 自动机，回文问题用 Manacher，子串统计用后缀数组/后缀自动机。

## 核心对比

| 算法 | 时间复杂度 | 空间 | 最佳场景 |
|------|-----------|------|---------|
| 朴素匹配 | O(nm) | O(1) | 短模式串 |
| KMP | O(n+m) | O(m) | 单模式匹配 |
| Boyer-Moore | O(n/m) 平均 | O(字符集) | 实际文本搜索 |
| Rabin-Karp | O(n+m) 平均 | O(1) | 多模式哈希匹配 |
| Trie | O(len) 查询 | O(字符集*节点) | 前缀匹配/字典 |
| AC自动机 | O(n+m) | O(字符集*节点) | 多模式同时匹配 |
| 后缀数组 | O(n log n) 构建 | O(n) | 子串搜索、重复串 |
| 后缀自动机 | O(n) 构建 | O(n) | 子串统计、LCS |
| Manacher | O(n) | O(n) | 最长回文子串 |
| Z算法 | O(n) | O(n) | 模式匹配预处理 |

## 适用场景

- 面试中根据模式串数量和文本长度选择合适算法
- 工程中搜索引擎、文本编辑器、日志分析等场景
- 生物信息学中的基因序列匹配

## 常见变体与技巧

- **面试选择策略：** 问清模式串数量——单模式用 KMP，多模式用 AC 自动机或 Rabin-Karp。
- **内存敏感场景：** 后缀自动机空间最优（O(n)），Trie 最耗内存但查询最快。
- **工程实现：** 标准库多用 Boyer-Moore 变体（如 grep），兼顾理论和实践效率。
