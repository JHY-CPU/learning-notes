# 04 - KMP 算法 — 前缀函数 (KMP: Prefix Function)

  ## 什么是前缀函数？

  前缀函数 π[i]（也称 failure function / next 数组）是 KMP 算法的核心。它定义为：**字符串 P[0..i] 中，既是真前缀又是真后缀的最长子串的长度**。


>
    **形式化定义：**

    π[i] = max{ k | 0 ≤ k < i 且 P[0..k-1] = P[i-k+1..i] }

    如果不存在这样的 k，则 π[i] = 0

    **直观理解：** 当匹配到位置 i 失败时，π[i-1] 告诉我们模式串应该回退到哪个位置继续匹配。


  ## 示例

  模式串 P = **"ABABACA"** 的前缀函数：










  | i | P[i] | 子串 P[0..i] | 最长公共前后缀 | π[i] |
| --- | --- | --- | --- | --- |
| 0 | A | A | 无 | 0 |
| 1 | B | AB | 无 | 0 |
| 2 | A | ABA | "A" | 1 |
| 3 | B | ABAB | "AB" | 2 |
| 4 | A | ABABA | "ABA" | 3 |
| 5 | C | ABABAC | 无 | 0 |
| 6 | A | ABABACA | "A" | 1 |

  ## 前缀函数计算算法

  高效计算前缀函数的核心思想是**利用已计算的前缀函数值**，避免重复比较。算法在 O(m) 时间内完成。


```

function computePrefixFunction(P):
  m = P.length
  π = new Array(m).fill(0)
  k = 0  // 当前匹配的前缀长度
  for i = 1 to m - 1:
    // 当字符不匹配时，回退到之前的前缀
    while k > 0 and P[i] !== P[k]:
      k = π[k - 1]
    // 如果字符匹配，前缀长度加 1
    if P[i] === P[k]:
      k++
    π[i] = k
  return π
  ```


>
    **关键理解：** while 循环中的 `k = π[k-1]` 是最精妙的部分。当 P[i] !== P[k] 时，我们不想直接从头开始，而是尝试**更短的前缀**，利用之前算好的 π 值来找到下一个可能的前缀长度。


  ## 前缀函数可视化

  从下图可以看出前缀函数的本质：π[i] 记录了子串 P[0..i] 中前后缀重叠的长度。


```

示例: P = "ABABACA"

i=2 (P[0..2]="ABA"):
  "A" B "A"   → 前缀"A" = 后缀"A" → π[2]=1

i=3 (P[0..3]="ABAB"):
  "AB" A "AB" → 前缀"AB" = 后缀"AB" → π[3]=2

i=4 (P[0..4]="ABABA"):
  "ABA" B "ABA" → 前缀"ABA" = 后缀"ABA" → π[4]=3
  ```
