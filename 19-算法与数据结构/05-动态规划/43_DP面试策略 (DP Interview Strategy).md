# DP面试策略 (DP Interview Strategy)

## 1. 面试中的DP思路

面试中遇到DP问题时，按照以下步骤思考：

```
步骤1：判断是否是DP问题
步骤2：暴力解法（递归）
步骤3：记忆化搜索
步骤4：转为表格DP
步骤5：空间优化
```

## 2. 判断是否是DP

### 2.1 DP问题的特征

- 求**最优值**（最大/最小/最长/最短）
- 求**方案数**
- 求**可行性**（能否达到某状态）
- 存在**重叠子问题**
- 具有**最优子结构**

### 2.2 不是DP的情况

- 需要输出所有方案（回溯/DFS）
- 数据规模太小（直接暴力）
- 可以贪心解决（贪心更简单）
- 无最优子结构（图的最短路径有负权环）

## 3. 解题框架

### 3.1 四问法

面试时向面试官确认以下四个问题：

```
1. 状态定义：dp[i] 代表什么？
2. 转移方程：dp[i] 如何推导？
3. 边界条件：初始值是什么？
4. 答案形式：返回 dp[n] 还是 max(dp)？
```

### 3.2 分析模板

```python
def solve_dp(arr):
    n = len(arr)
    # 1. 特殊情况
    if n == 0: return 0
    if n == 1: return arr[0]

    # 2. 定义状态
    dp = [0] * n

    # 3. 初始化
    dp[0] = arr[0]
    dp[1] = max(arr[0], arr[1])

    # 4. 状态转移
    for i in range(2, n):
        dp[i] = max(dp[i-1], dp[i-2] + arr[i])

    # 5. 返回答案
    return dp[n-1]
```

## 4. 常见题型套路

### 4.1 一维线性DP

```
状态：dp[i] = 以i结尾/前i个的最优值
转移：dp[i] = f(dp[i-1], dp[i-2], ...)
例子：打家劫舍、最大子数组和
```

### 4.2 二维网格DP

```
状态：dp[i][j] = 到达(i,j)的最优值
转移：dp[i][j] = f(dp[i-1][j], dp[i][j-1])
例子：不同路径、最小路径和
```

### 4.3 背包DP

```
0-1背包：逆序遍历
完全背包：正序遍历
例子：零钱兑换、目标和
```

### 4.4 区间DP

```
状态：dp[i][j] = 区间[i,j]的最优值
转移：枚举分割点k
例子：戳气球、石子合并
```

### 4.5 字符串DP

```
状态：dp[i][j] = 两个串前i/j个字符的最优值
转移：根据字符是否相同
例子：LCS、编辑距离
```

## 5. 面试沟通技巧

### 5.1 正确的沟通流程

```
1. 确认题意：确认输入输出格式和约束
2. 暴力思路：先说O(2^n)的暴力解法
3. 识别重叠子问题：画递归树，指出重复计算
4. 记忆化：提出加缓存的方案
5. 优化：转为迭代DP，考虑空间优化
6. 复杂度分析：说出时间和空间复杂度
```

### 5.2 常见对话模式

```
面试官：这道题你会怎么做？
你：首先这是一个XX问题，我可以想到一个暴力的递归解法...
（分析递归树，指出重复子问题）
...这里有很多重复计算，我们可以用记忆化搜索来优化...
（写记忆化代码）
...如果需要进一步优化空间，可以改成自底向上的DP...
（写表格DP代码）
...由于dp[i]只依赖dp[i-1]，可以用两个变量优化到O(1)空间...
```

## 7. 时间复杂度速查

```
O(n)：一维线性DP
O(n²)：二维DP / 区间DP
O(n³)：区间DP（枚举分割点）
O(nW)：背包DP
O(nlogn)：LIS二分优化
O(2^n)：状态压缩DP
```

## 8. 常见错误

### 8.1 代码错误

1. 索引偏移：dp[i] 对应 nums[i-1]
2. 遍历方向：0-1背包必须逆序
3. 边界遗漏：忘记初始化 dp[0]
4. 整数溢出：大数需要取模

### 8.2 思路错误

1. 试图一步到位写出最优DP，不如先写暴力再优化
2. 状态定义不清就开始写代码
3. 不和面试官确认题意就开始写
4. 不分析复杂度

## 9. 面试准备建议

```
1. 熟练掌握DP五步法
2. 背诵常见DP模板
3. 每种类型至少做5道题
4. 练习空间优化
5. 能清楚解释转移方程的含义
6. 注意代码规范和边界处理
```
