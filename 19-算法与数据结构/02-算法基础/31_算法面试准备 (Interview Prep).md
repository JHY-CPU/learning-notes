# 32-算法面试准备 (Interview Prep)

算法面试考察系统化思维和沟通能力，掌握解题框架和高频题型至关重要。

## 解题五步法

1. **澄清题意**：确认输入输出格式、边界条件、特殊数据
2. **举例分析**：用 2-3 个例子理解问题
3. **设计算法**：从暴力解法出发，逐步优化
4. **编码实现**：清晰的命名、合理的模块划分
5. **测试验证**：正常用例、边界用例、异常用例

## C++ 面试常用模板

```cpp
// 快排模板
void quickSort(vector<int>& arr, int l, int r) {
    if (l >= r) return;
    int pivot = arr[l + rand() % (r - l + 1)];
    int i = l, j = r;
    while (i <= j) {
        while (arr[i] < pivot) i++;
        while (arr[j] > pivot) j--;
        if (i <= j) swap(arr[i++], arr[j--]);
    }
    quickSort(arr, l, j);
    quickSort(arr, i, r);
}

// BFS 模板
void bfs(vector<vector<int>>& graph, int start) {
    queue<int> q;
    unordered_set<int> visited;
    q.push(start);
    visited.insert(start);
    while (!q.empty()) {
        int node = q.front(); q.pop();
        for (int nei : graph[node]) {
            if (!visited.count(nei)) {
                visited.insert(nei);
                q.push(nei);
            }
        }
    }
}
```

## 沟通技巧

- **先说思路再写代码**：让面试官了解你的思考过程
- **主动说明复杂度**：写完代码后立即分析时间空间复杂度
- **遇到困难时讨论**：说出卡在哪里，尝试什么方向
- **边写边解释**：不要沉默编码
- **主动测试**：写完后用例子验证

## 常见失败原因

1. **没澄清题意就开始写**：浪费时间在错误理解上
2. **死磕暴力解**：先说思路，确认方向再写
3. **不讨论复杂度**：面试官想知道你是否理解效率
4. **边界条件遗漏**：空数组、单元素、最大值溢出
5. **代码不可读**：变量命名随意，缺少注释
