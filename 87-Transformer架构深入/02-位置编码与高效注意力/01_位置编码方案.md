# 位置编码方案


## 位置编码方案


#### 为什么需要位置编码？


Transformer 的自注意力机制本身是置换不变的（permutation invariant）——打乱输入序列的顺序，输出也会被打乱但注意力权重不变。这意味着 Transformer 无法区分 token 的位置顺序。位置编码（Positional Encoding）通过显式注入位置信息来弥补这一缺陷，使模型能够区分"我在第1位"和"我在第100位"。


## 1. Sinusoidal 位置编码（正弦位置编码）


### 1.1 公式


原始 Transformer（Vaswani et al., 2017）使用固定的正弦和余弦函数生成位置编码：


$$
PE(pos, 2i) = sin(pos / 100002i/dmodel)
        PE(pos, 2i+1) = cos(pos / 100002i/dmodel)
$$


其中 pos 是位置索引（0, 1, 2, ...），i 是维度索引（0, 1, ..., d~model~/2 - 1）。偶数维度使用 sin，奇数维度使用 cos。


### 1.2 设计动机


- **有界性**
   ：sin/cos 的值域为 [-1, 1]，不会因位置增大而发散
- **唯一性**
   ：不同位置有不同编码，每个频率分量提供独立信息
- **泛化性**
   ：可泛化到训练中未见过的更长序列（理论上）
- **相对位置**
   ：PE(pos+k) 可以表示为 PE(pos) 的线性函数，使模型能学习相对位置关系


### 1.3 相对位置的线性性质


正弦编码的一个关键性质是：对于固定偏移 k，PE(pos+k) 可以通过 PE(pos) 的线性变换得到：


$$
[PE(pos+k, 2i)]   =   [cos(θik)   -sin(θik)] [PE(pos, 2i)]
        [PE(pos+k, 2i+1)]     [sin(θik)    cos(θik)] [PE(pos, 2i+1)]
$$


其中 θ~i~ = 1/10000^2i/d^。这个旋转矩阵的性质后来启发了 RoPE 的设计。


```
import torch
import math

def sinusoidal_encoding(max_len, d_model):
    """生成正弦位置编码矩阵"""
    pe = torch.zeros(max_len, d_model)
    position = torch.arange(0, max_len, dtype=torch.float).unsqueeze(1)
    div_term = torch.exp(
        torch.arange(0, d_model, 2).float() * (-math.log(10000.0) / d_model)
    )

    pe[:, 0::2] = torch.sin(position * div_term)  # 偶数维度
    pe[:, 1::2] = torch.cos(position * div_term)  # 奇数维度

    return pe  # shape: (max_len, d_model)

# 使用方式: 直接加到 token embedding 上
# x = token_embedding + pe[:seq_len]
```


## 2. Learned Positional Embedding（学习型位置编码）


### 2.1 原理


直接将位置编码作为可学习参数，通过训练自动学习每个位置的表示。BERT、GPT-1/2 等模型采用此方案：


```
class LearnedPositionalEmbedding(nn.Module):
    def __init__(self, max_len, d_model):
        super().__init__()
        # 可学习参数表: (max_len, d_model)
        self.embedding = nn.Embedding(max_len, d_model)

    def forward(self, x):
        """
        x: (batch, seq_len, d_model) - token embeddings
        """
        seq_len = x.size(1)
        positions = torch.arange(seq_len, device=x.device)
        return x + self.embedding(positions)  # 直接相加
```


### 2.2 优缺点


| 优点 | 缺点 |
| --- | --- |
| 实现简单，无需手工设计 | 受限于 max_len，无法泛化到更长序列 |
| 在训练长度内效果通常很好 | 增加少量参数（max_len × d~model~） |
| 每个位置有独特的表示 | 位置之间的关系未显式编码 |


## 3. RoPE（旋转位置编码，Rotary Position Embedding）


### 3.1 核心思想


RoPE（Su et al., 2021）是目前最主流的位置编码方案（LLaMA、Mistral、Qwen、Gemma 等均采用）。其核心思想是：不在 embedding 上加位置编码，而是将位置信息融入 QK 的点积计算中，通过旋转矩阵使得注意力分数自然地包含相对位置信息。


### 3.2 数学推导


对于 d 维的 query/key 向量，RoPE 将其分成 d/2 个二维子空间，对每个子空间施加旋转：


$$
f(q, m) = Rm · q,     f(k, n) = Rn · k
        其中 Rm = diag(R1(m), R2(m), ..., Rd/2(m))
        Ri(m) = [cos(mθi)   -sin(mθi)
                    sin(mθi)    cos(mθi)]
        θi = 10000-2i/d
$$


关键性质：q~m~^T^ · k~n~ 只与 (n-m) 有关，即注意力分数自然地编码了**相对位置**：


$$
(Rmq)T(Rnk) = qTRmTRnk = qTRn-mk
$$


### 3.3 高效实现


```
def precompute_rope_frequencies(d_model, max_len, base=10000.0):
    """预计算 RoPE 的旋转频率"""
    # θ_i = base^(-2i/d) for i = 0, 1, ..., d/2-1
    freqs = 1.0 / (base ** (torch.arange(0, d_model, 2).float() / d_model))
    # positions: 0, 1, ..., max_len-1
    positions = torch.arange(max_len)
    # 外积: (max_len,) × (d/2,) → (max_len, d/2)
    angles = torch.outer(positions, freqs)
    return angles  # shape: (max_len, d_model // 2)

def apply_rope(x, freqs):
    """
    x: (batch, heads, seq_len, head_dim)
    freqs: (seq_len, head_dim // 2)
    """
    # 将 x 分成两半并旋转
    x_r = x.float().reshape(*x.shape[:-1], -1, 2)  # (..., d/2, 2)
    x1, x2 = x_r[..., 0], x_r[..., 1]  # 各 (..., d/2)

    # freqs 扩展到 (1, 1, seq_len, d/2)
    cos_f = torch.cos(freqs).unsqueeze(0).unsqueeze(0)
    sin_f = torch.sin(freqs).unsqueeze(0).unsqueeze(0)

    # 旋转: [x1*cos - x2*sin, x1*sin + x2*cos]
    out1 = x1 * cos_f - x2 * sin_f
    out2 = x1 * sin_f + x2 * cos_f

    # 交错拼接回原始形状
    output = torch.stack([out1, out2], dim=-1).reshape(x.shape)
    return output.type_as(x)

# 高效实现（避免 reshape，直接用乘法）
def apply_rope_efficient(x, cos, sin):
    """更高效的 RoPE 实现"""
    # x: (batch, heads, seq_len, head_dim)
    # cos, sin: (seq_len, head_dim)
    x1 = x[..., :x.shape[-1]//2]
    x2 = x[..., x.shape[-1]//2:]
    rotated = torch.cat([-x2, x1], dim=-1)
    return x * cos + rotated * sin
```


### 3.4 RoPE 的优势


- **相对位置编码**
   ：注意力分数自然依赖于相对距离，而非绝对位置
- **线性衰减**
   ：距离越远，注意力分数自然降低（旋转导致的内积衰减）
- **远程衰减特性**
   ：较长的波长（低频）分量保持较慢的旋转，保持长距离信息
- **无额外参数**
   ：频率是固定的，不增加可学习参数
- **与 KV Cache 兼容**
   ：旋转操作可以缓存，不会破坏 KV Cache 的增量计算


## 4. ALiBi（Attention with Linear Biases）


### 4.1 原理


ALiBi（Press et al., 2022）不向 embedding 添加位置编码，而是在注意力分数上直接加上一个与距离成比例的线性偏置：


$$
Attention(Q, K, V) = softmax(QKT/√dk + m · bias)
        bias[i][j] = -(i - j)   (i ≥ j, 因果掩码场景)
        m ∈ ℝh 是每头固定的斜率（预设，非学习参数）
$$


```
def alibi_bias(num_heads, seq_len):
    """生成 ALiBi 线性偏置矩阵"""
    # 斜率: 等比数列，从 2^(-8/h) 到 2^(-8/h * h)
    slopes = torch.pow(2, -8.0 / num_heads * torch.arange(1, num_heads + 1))

    # 距离矩阵: pos[i][j] = j - i
    positions = torch.arange(seq_len)
    distance = positions.unsqueeze(0) - positions.unsqueeze(1)  # (seq_len, seq_len)

    # 因果掩码: 只看过去的 token
    causal_mask = torch.triu(torch.ones(seq_len, seq_len), diagonal=1).bool()
    distance = distance.masked_fill(causal_mask, 0)

    # bias: (num_heads, seq_len, seq_len)
    bias = slopes.unsqueeze(-1).unsqueeze(-1) * distance.unsqueeze(0)

    # 将因果掩码位置设为 -inf
    bias = bias.masked_fill(causal_mask.unsqueeze(0), float('-inf'))
    return bias
```


### 4.2 ALiBi 的特性


- **零额外参数**
   ：斜率是预设的等比数列，不需要学习
- **零额外计算**
   ：只需在注意力分数上加偏置，开销极小
- **天然外推**
   ：线性偏置在训练长度之外仍然有定义，具有较好的长度外推能力
- **近距偏好**
   ：线性惩罚使得模型天然偏好近距离 token，符合语言的局部性


## 5. YaRN（Yet another RoPE extensioN）


### 5.1 动机


RoPE 在训练长度之外的外推能力有限，因为频率基底在未见过的位置上会退化。YaRN 提出了一种精细的 RoPE 外推方法，通过区分不同频率分量来调整外推策略。


### 5.2 核心思想


YaRN 将 RoPE 的频率分为三个区间，对不同区间采用不同的缩放策略：


| 频率区间 | 特点 | 处理方式 |
| --- | --- | --- |
| 高频（低维） | 波长短，捕获局部模式 | 不缩放（高频信息已充分学习） |
| 中频 | 介于两者之间 | 线性插值缩放 |
| 低频（高维） | 波长长，捕获全局依赖 | NTK-aware 缩放 |


```
def yarn_scaling(d_model, original_len, target_len, base=10000.0,
                 alpha=1.0, beta=32.0):
    """
    YaRN 缩放: 对不同频率分量使用不同缩放比
    alpha: NTK-aware 缩放因子
    beta: 高频不缩放的阈值
    """
    freqs = 1.0 / (base ** (torch.arange(0, d_model, 2).float() / d_model))
    scale = target_len / original_len

    # 计算每个频率的"重要性"
    # 低频分量（高维索引）需要更多缩放
    # 高频分量（低维索引）保持不变
    ramp = torch.linspace(0, 1, d_model // 2)
    # 0 到 1 的斜坡，0 表示高频（不缩放），1 表示低频（完全缩放）
    smooth_scaling = 1.0 / scale * ramp + (1 - ramp)  # 线性插值

    adjusted_freqs = freqs * smooth_scaling
    return adjusted_freqs
```


## 6. NTK-aware Scaling


### 6.1 原理


NTK-aware Scaling 受神经正切核（Neural Tangent Kernel）理论启发，通过调整 RoPE 的 base 参数而非简单的位置插值来实现长度外推：


$$
basenew = baseold × scaled/(d-2)
        相当于将 θi = base-2i/d 调整为 θi = (base × scaled/(d-2))-2i/d
$$


核心思想：通过增大 base，使低频分量的波长更长，从而在更长序列上保持稳定的注意力模式。与直接线性插值（PI）相比，NTK-aware 保留了高频分量的分辨率。


### 6.2 与其他外推方法的对比


| 方法 | 策略 | 是否需要微调 | 外推效果 |
| --- | --- | --- | --- |
| 直接外推 | 不做任何处理 | 否 | 差（迅速退化） |
| 线性插值（PI） | 位置索引除以 scale | 少量微调 | 中等 |
| NTK-aware | 调整 base 参数 | 否或少量微调 | 较好 |
| YaRN | 分频段混合缩放 | 少量微调 | 最佳 |
| Dynamic NTK | 推理时动态调整 base | 否 | 较好 |


## 7. 位置编码外推（Extrapolation）


### 7.1 外推问题的本质


位置编码外推是指：模型在较短序列上训练后，能否在更长的序列上正常工作？这是长上下文 LLM 的核心挑战之一。


> **Warning:** #### 为什么外推困难？
>
>
> - **训练-推理不一致**
>    ：训练时见过的位置范围是 [0, L
>    ~train~
>    ]，推理时需要处理 [0, L
>    ~test~
>    ] 其中 L
>    ~test~
>    > L
>    ~train~
> - **高频分量退化**
>    ：对于 RoPE，未见过的位置可能产生与训练时截然不同的旋转角度
> - **注意力分布偏移**
>    ：外推位置上的注意力分布可能与训练时学到的分布不同


### 7.2 外推方案总结


| 方案 | 适用编码 | 核心思路 | 代表模型 |
| --- | --- | --- | --- |
| PI (Position Interpolation) | RoPE | 将长序列线性压缩到训练长度内 | Code LLaMA |
| NTK-aware | RoPE | 调整 base 来改变频率分布 | YaRN, LongLoRA |
| YaRN | RoPE | 分频段混合策略 | Yi, Mistral |
| ALiBi | 线性偏置 | 天然支持外推 | BLOOM, MPT |
| Dynamic NTK | RoPE | 推理时根据实际长度动态调整 | 多种开源模型 |


## 8. 各方案综合对比


| 方案 | 额外参数 | 外推能力 | 相对位置 | 代表模型 |
| --- | --- | --- | --- | --- |
| Sinusoidal | 无 | 弱 | 间接支持 | 原始 Transformer |
| Learned Embedding | max_len × d | 无 | 无 | BERT, GPT-2 |
| RoPE | 无 | 需外推技术 | 天然支持 | LLaMA, Qwen, Mistral |
| ALiBi | 无 | 天然支持 | 天然支持 | BLOOM, MPT |
| YaRN | 无 | 优秀 | 天然支持 | Yi, 长上下文模型 |


#### 当前趋势


RoPE + YaRN/NTK-aware 是当前最主流的方案组合。RoPE 提供了优秀的相对位置编码和与 KV Cache 的兼容性，而 YaRN 等外推技术解决了长度泛化问题。ALiBi 虽然简洁但在超长序列（128K+）上的表现不如 RoPE 系方案。新模型（2024-2025）几乎都选择 RoPE 作为基础位置编码。
