# 编码解码器架构


#### 编码解码器架构概览


编码解码器（Encoder-Decoder）架构保留了完整的 Transformer 结构：编码器负责理解输入序列（双向注意力），解码器负责生成输出序列（因果注意力 + 对编码器输出的交叉注意力）。这类架构天然适合序列到序列（seq2seq）任务，如机器翻译、摘要生成、问答等。虽然当前 LLM 主流采用解码器架构，但编码解码器在某些任务上仍有独特优势。


## 1. T5（Text-to-Text Transfer Transformer）


### 1.1 核心思想


T5（Raffel et al., 2020）将所有 NLP 任务统一为"文本到文本"格式：输入和输出都是文本字符串，通过不同的文本前缀来区分任务类型。


| 任务 | 输入格式 | 输出格式 |
| --- | --- | --- |
| 翻译 | "translate English to German: That is good." | "Das ist gut." |
| 摘要 | "summarize: 长文章..." | "文章摘要..." |
| 问答 | "question: 答案在上下文中? context: 上下文" | "答案" |
| 情感分析 | "sentiment: 这个电影很好看" | "positive" |
| 自然语言推理 | "mnli premise: 前提 hypothesis: 假设" | "entailment" |


### 1.2 模型架构


| 规格 | T5-Small | T5-Base | T5-Large | T5-XL | T5-XXL |
| --- | --- | --- | --- | --- | --- |
| 层数 | 6 | 12 | 24 | 24 | 24 |
| d~model~ | 512 | 768 | 1024 | 2048 | 4096 |
| d~ff~ | 2048 | 3072 | 4096 | 5120 | 10240 |
| 头数 | 8 | 12 | 16 | 32 | 64 |
| 参数量 | 60M | 220M | 770M | 3B | 11B |


### 1.3 编码器-解码器注意力


这是编码解码器架构的核心组件。解码器的每一层除了自注意力和 FFN 外，还有一层**交叉注意力**（Cross-Attention）：


```
# 编码器-解码器注意力详解
# 在解码器的每一层中:

# 1. 自注意力（Causal Self-Attention）
#    Q, K, V 都来自解码器前一层的输出
#    使用因果掩码，防止看到未来 token

# 2. 交叉注意力（Encoder-Decoder Attention）
#    Q 来自解码器自注意力的输出
#    K, V 来自编码器最后一层的输出
#    无因果掩码（解码器可以关注编码器的所有位置）
#    使用填充掩码屏蔽编码器中的 padding 位置

# 3. 前馈网络（FFN）
#    与普通 Transformer 相同

# 编码器输出被所有解码器层共享
# 每层的交叉注意力独立学习不同的关注模式
```


### 1.4 Span Corruption 预训练


T5 使用 span corruption（跨度损坏）作为预训练任务，与 BERT 的 MLM 类似但有所不同：


$$
输入: "The quick brown [X] fox jumps over [Y] the lazy dog"
        目标: "[X] fox jumps over [Y] dog [Z]"
        规则: 将连续的 token span 替换为唯一的 sentinel token（[X], [Y], ...）
        目标是预测被替换的 span，用 sentinel 分隔
$$


```
# Span Corruption 示例
# 原文: "The quick brown fox jumps over the lazy dog"
# 随机选择要替换的 span（如 "fox jumps over" 和 "the"）

# 输入:  "The quick brown   lazy dog"
# 目标:  " fox jumps over  the "

# 与 MLM 的关键区别:
# 1. 替换的是连续的 span（多个 token），而非单个 token
# 2. 目标序列的长度可变，需要解码器逐步生成
# 3. sentinel token 是唯一的，不与真实 token 混淆
```


> **Warning:** #### Span Corruption vs MLM 的比较
>
>
> - **Span Corruption 优势**
>    ：解码器需要生成完整的 span，更接近实际的生成任务；每个 span 有独立的 sentinel，模型可以区分不同的被遮盖区域
> - **MLM 优势**
>    ：更简单，不需要解码器；BERT 风格的模型在某些 NLU 任务上微调更方便
> - **T5 的发现**
>    ：span corruption 在 T5 的实验中效果最好（与 prefix LM、deshuffling 等对比）


## 2. BART（Bidirectional and Auto-Regressive Transformers）


### 2.1 模型架构


BART（Lewis et al., 2020）结合了 BERT 的双向编码器和 GPT 的自回归解码器：


- **编码器**
   ：类似 BERT，使用双向注意力
- **解码器**
   ：类似 GPT，使用因果注意力 + 交叉注意力
- **预训练**
   ：通过各种噪声函数破坏文本，然后用解码器重建原始文本


### 2.2 噪声函数


| 噪声类型 | 描述 | 示例 |
| --- | --- | --- |
| Token Masking | 随机 token 替换为 [MASK] | "The cat [MASK] on the mat" |
| Token Deletion | 随机删除 token | "The cat on the mat" |
| Token Infilling | 用单个 [MASK] 替换随机 span | "The [MASK] the mat" |
| Sentence Permutation | 打乱句子顺序 | 句子 2 + 句子 1 |
| Document Rotation | 随机选择起点旋转文档 | "cat sat on the mat. The" |


#### BART 的关键发现


BART 通过消融实验发现：**Token Infilling（用单个 [MASK] 替换随机 span）**和 **Sentence Permutation（打乱句子顺序）**的组合效果最好。这与 T5 的 span corruption 思路相似——都需要模型理解和重构被破坏的文本结构。


### 2.3 BART 与 T5 的对比


| 对比项 | T5 | BART |
| --- | --- | --- |
| 编码器 | 双向注意力 | 双向注意力 |
| 解码器 | 因果注意力 + 交叉注意力 | 因果注意力 + 交叉注意力 |
| 预训练 | Span Corruption | 多种噪声组合 |
| 任务统一 | 文本到文本（前缀区分） | 通常每个任务单独微调 |
| 相对位置编码 | 相对位置偏置（非 RoPE） | 学习型位置嵌入 |
| 规模 | 最大 11B | 最大 400M |


## 3. mT5（Multilingual T5）


### 3.1 多语言扩展


mT5（Xue et al., 2021）将 T5 扩展到 101 种语言，使用 mC4 数据集（Common Crawl 的多语言版本）：


- **词表**
   ：SentencePiece 250K 词表（覆盖 101 种语言）
- **预训练**
   ：多语言 span corruption，使用混合语言采样策略
- **翻译**
   ：跨语言迁移能力强，可以进行零样本跨语言翻译


### 3.2 多语言训练挑战


> **Warning:** #### 多语言模型的困境
>
>
> - **容量竞争**
>    ：有限的模型容量需要分配给多种语言，每种语言获得的容量减少
> - **数据不平衡**
>    ：英语数据远多于其他语言，可能导致模型对低资源语言表现不佳
> - **词表浪费**
>    ：大词表虽然覆盖多语言，但对单语言任务来说很多 token 用不到
> - **采样策略**
>    ：通常使用温度采样（temperature sampling）来平衡不同语言的数据量


## 4. FLAN-T5（Instruction-Tuned T5）


### 4.1 指令微调


FLAN-T5（Chung et al., 2022）在 T5 基础上进行了大规模指令微调（Instruction Tuning）：


- **训练数据**
   ：1800+ 个 NLP 任务的指令-输入-输出格式数据
- **关键发现**
   ：指令微调大幅提升了模型的 zero-shot 和 few-shot 能力
- **任务组合**
   ：训练时包含多种任务类型（分类、生成、推理、翻译等），推理时泛化到未见过的任务


### 4.2 FLAN-T5 的效果


| 模型 | 参数量 | 是否指令微调 | Zero-shot 性能 | Few-shot 性能 |
| --- | --- | --- | --- | --- |
| T5-Large | 770M | 否 | 弱 | 中等 |
| FLAN-T5-Large | 770M | 是 | 强 | 强 |
| T5-XL | 3B | 否 | 中等 | 强 |
| FLAN-T5-XL | 3B | 是 | 强 | 非常强 |


#### 指令微调的意义


FLAN 系列工作证明了一个重要结论：**在足够多样的任务上进行指令微调，可以使模型获得强大的泛化能力**。即使模型规模不是最大的（如 FLAN-T5-XL 3B），指令微调后的效果也能匹敌甚至超过未指令微调的大模型。这一发现深刻影响了后续的模型训练范式，几乎所有现代 LLM 都包含指令微调阶段。


## 5. 统一序列到序列框架


### 5.1 编码解码器架构的设计模式


| 设计维度 | 选择 | 代表模型 |
| --- | --- | --- |
| 编码器-解码器参数共享 | 完全独立 / 部分共享 / 完全共享 | T5(独立) / BART(独立) |
| 编码器层数 vs 解码器层数 | 等深 / 解码器更深 / 编码器更深 | T5(等深) |
| 预训练任务 | Span Corruption / 噪声去噪 / 前缀语言模型 | T5(span) / BART(denoise) |
| 位置编码 | 相对偏置 / 学习型 / RoPE | T5(相对) / BART(学习) |


### 5.2 三种 Transformer 架构的统一视角


```
# 三种架构可以看作同一个框架的特例

# 1. 编码器 (BERT)
#    - 注意力: 双向（无掩码）
#    - 预训练: MLM（判别式）
#    - 适用: 理解任务（分类、NER、嵌入）

# 2. 解码器 (GPT)
#    - 注意力: 因果（下三角掩码）
#    - 预训练: 自回归语言模型（生成式）
#    - 适用: 生成任务（对话、写作、代码）

# 3. 编码解码器 (T5)
#    - 编码器: 双向注意力（理解输入）
#    - 解码器: 因果注意力 + 交叉注意力（基于理解生成输出）
#    - 预训练: Span Corruption / 噪声去噪
#    - 适用: 序列到序列任务（翻译、摘要、问答）

# 统一视角:
# - 编码器可以看作"没有解码器的编码解码器"
# - 解码器可以看作"编码器和解码器参数共享的编码解码器"
# - Prefix-LM 可以看作"前缀部分用双向，后续用因果的解码器"
```


## 6. 编码解码器 vs 纯解码器：当前趋势


### 6.1 任务对比


| 任务类型 | 编码解码器 | 纯解码器 | 推荐 |
| --- | --- | --- | --- |
| 机器翻译 | 天然适配 | 可以但非最优 | 编码解码器 |
| 文本摘要 | 天然适配 | In-context learning 效果好 | 均可 |
| 开放域问答 | 适合 | 适合 | 均可 |
| 文本生成（对话等） | 可以但复杂 | 天然适配 | 纯解码器 |
| 分类任务 | 用编码器输出 | 用生成方式 | 均可 |
| 多任务泛化 | 需要文本到文本统一 | In-context learning 原生支持 | 纯解码器 |


### 6.2 为什么纯解码器成为主流


#### 当前趋势分析（2023-2025）


- **Scaling 优势**
   ：纯解码器在大规模参数下的 scaling 表现更好
- **统一性**
   ：所有任务都可以用自回归生成解决，无需区分编码器/解码器
- **推理简单**
   ：只需 KV Cache，不需要编码器-解码器的复杂推理流程
- **In-Context Learning**
   ：纯解码器的 ICL 能力使得无需微调就能适应新任务
- **训练效率**
   ：在相同的 token 预算下，纯解码器的训练效率通常更高（每个 token 都有训练信号）
- **生态成熟**
   ：RLHF、SFT 等对齐技术主要围绕解码器架构开发


### 6.3 编码解码器仍有优势的场景


- **精确的序列到序列任务**
   ：翻译、代码到文本等输入输出严格对齐的任务
- **资源受限场景**
   ：小规模模型（<1B）中，编码解码器可能效果更好（T5 vs GPT-2 small）
- **多语言任务**
   ：mT5 等编码解码器在多语言翻译任务上仍有优势
- **信息抽取**
   ：编码器的双向理解能力在某些 NLU 任务上有优势


## 7. 总结


#### 编码解码器架构核心要点


1. **T5**
   开创性地将所有 NLP 任务统一为文本到文本格式，span corruption 预训练高效实用
2. **BART**
   证明了多种噪声组合的去噪预训练在编码解码器上有效
3. **交叉注意力**
   是编码解码器的核心：连接编码器的理解能力和解码器的生成能力
4. **FLAN-T5**
   证明了指令微调可以在不增加模型规模的情况下大幅提升泛化能力
5. 虽然纯解码器成为当前 LLM 主流，但编码解码器在特定 seq2seq 任务和小规模模型上仍有价值
6. 三种架构（编码器/解码器/编码解码器）可以看作
   **同一框架的不同特例**
   ，核心都是 Transformer 自注意力机制
