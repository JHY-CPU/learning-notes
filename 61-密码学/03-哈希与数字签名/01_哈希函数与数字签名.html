哈希函数与数字签名

密码学SHA-256数字签名

哈希函数将任意长度数据映射为固定长度摘要，是数字签名和区块链的基础。

密码学哈希函数

哈希函数性质： 1. 确定性：相同输入始终产生相同输出 2. 快速计算：给定输入，快速计算哈希值 3. 抗原像性(Pre-image Resistance)： 给定H(x)，难以找到x（单向性） 4. 抗第二原像性(Second Pre-image Resistance)： 给定x1，难以找到x2≠x1使得H(x1)=H(x2) 5. 抗碰撞性(Collision Resistance)： 难以找到任意x1≠x2使得H(x1)=H(x2) 常用哈希算法： ┌──────────┬──────────┬──────────┬──────────┐ │ 算法 │ 输出长度 │ 安全状态 │ 备注 │ ├──────────┼──────────┼──────────┼──────────┤ │ MD5 │ 128位 │ 不安全 │ 已被破解 │ │ SHA-1 │ 160位 │ 不安全 │ 2017碰撞 │ │ SHA-256 │ 256位 │ 安全 │ 最广泛使用 │ │ SHA-512 │ 512位 │ 安全 │ 64位优化 │ │ SHA-3 │ 可变 │ 安全 │ Keccak算法│ │ BLAKE3 │ 可变 │ 安全 │ 最快 │ └──────────┴──────────┴──────────┴──────────┘ 哈希应用： - 密码存储：存hash(密码+盐)，不存明文 - 数据完整性校验：文件下载后比对hash - 区块链：每个区块包含前一区块的hash - HMAC：基于哈希的消息认证码 - 数字签名：对消息的hash进行签名

数字签名

数字签名 = 确认身份 + 数据完整性 + 不可否认性 签名过程： 消息 M → Hash → H(M) → Sign(PrivKey) → 签名 Sig ↓ 发送：M + Sig 验签过程： 收到 M + Sig 1. 计算 H'(M) = Hash(M) 2. 计算 Verify(PubKey, Sig) → H(M) 3. 比较 H'(M) == H(M)，一致则验签通过 常见数字签名算法： ┌──────────┬──────────┬──────────────────────┐ │ 算法 │ 基于 │ 特点 │ ├──────────┼──────────┼──────────────────────┤ │ RSA-SHA │ RSA │ 最广泛，密钥较大 │ │ DSA │ 离散对数 │ 只用于签名，不加密 │ │ ECDSA │ 椭圆曲线 │ 密钥短，性能好 │ │ Ed25519 │ EdDSA │ 最新标准，速度快 │ └──────────┴──────────┴──────────────────────┘ ECDSA 签名过程： 1. 选择随机数 k 2. 计算 R = k × G，取 r = R.x mod n 3. 计算 s = k^(-1) × (H(M) + r × privKey) mod n 4. 签名 = (r, s) 证书链 (Certificate Chain)： 根CA证书 (Root CA) └── 中间CA证书 (Intermediate CA) └── 终端实体证书 (Server/Client) 验证时从终端证书逐级向上验证签名
推荐使用Ed25519（SSH密钥）或ECDSA（TLS证书）替代RSA签名，性能更好且密钥更短。

SHA-256 算法结构

SHA-256 处理流程： ┌─────────────────────────────────────────────────────────┐ │ 1. 消息填充： │ │ - 附加 bit "1" │ │ - 附加 k 个 "0"，使得长度 ≡ 448 (mod 512) │ │ - 附加 64 位原始消息长度 │ │ │ │ 2. 消息分组（每 512 位一块）： │ │ W[0..15] = 消息块的 16 个 32 位字 │ │ W[t] = σ₁(W[t-2]) + W[t-7] + σ₀(W[t-15]) + W[t-16] │ │ σ₀(x) = (x >>> 7) ⊕ (x >>> 18) ⊕ (x >> 3) │ │ σ₁(x) = (x >>> 17) ⊕ (x >>> 19) ⊕ (x >> 10) │ │ │ │ 3. 压缩函数（64 轮迭代）： │ │ T₁ = h + Σ₁(e) + Ch(e,f,g) + K[t] + W[t] │ │ T₂ = Σ₀(a) + Maj(a,b,c) │ │ (a,b,c,d,e,f,g,h) = (T₁+T₂, a, b, c, d+T₁, e, f, g)│ │ │ │ 4. 输出：8 个 32 位字拼接 = 256 位哈希值 │ │ │ │ 常量 K[0..63]：取前 64 个素数立方根的小数部分的前 32 位 │ │ 初始值 H[0..7]：取前 8 个素数平方根的小数部分的前 32 位 │ └─────────────────────────────────────────────────────────┘ Merkle-Damgard 结构： ┌─────────────────────────────────────────────────────────┐ │ IV → H(M₁) → H(M₂) → ... → H(Mₙ) → 输出 │ │ │ │ 压缩函数：h_i = f(h_{i-1}, M_i) │ │ 每次处理一个 512 位块，更新 256 位状态 │ │ │ │ 长度扩展攻击： │ │ 已知 H(M) 和 len(M)，可计算 H(M || padding || M') │ │ → 不要直接用 Hash(key || message) 做 MAC │ │ → 使用 HMAC 防止此类攻击 │ │ │ │ SHA-3 使用不同的结构（海绵结构 Sponge），避免此问题 │ └─────────────────────────────────────────────────────────┘

Ed25519 签名算法详解

Ed25519 (Daniel Bernstein, 2011)： ┌─────────────────────────────────────────────────────────┐ │ 基于 Edwards 曲线：-x² + y² = 1 + dx²y² │ │ 其中 d = -121665/121666 │ │ 基点阶 L = 2²⁵² + 27742317777372353535851937790883648493│ │ │ │ 密钥生成： │ │ 1. 生成 32 字节随机种子 │ │ 2. k = SHA-512(seed)（64 字节） │ │ 3. 私钥标量 a = clamp(k[0..31]) │ │ clamp: 清除最低 3 位，清除最高位，设置次高位 │ │ 4. 前缀 = k[32..63] │ │ 5. 公钥 A = a × B（32 字节点编码） │ │ │ │ 签名过程： │ │ 1. r = SHA-512(prefix || M) mod L │ │ 2. R = r × B（签名的 R 部分） │ │ 3. S = (r + SHA-512(R || A || M) × a) mod L │ │ 4. 签名 = encode(R) || encode(S)（64 字节） │ │ │ │ 验证过程： │ │ 检查 S × B == R + SHA-512(R || A || M) × A │ │ │ │ 优势： │ │ - 确定性签名（相同消息和密钥产生相同签名） │ │ - 批量验证（可同时验证多个签名，更快） │ │ - 常数时间实现（抵抗侧信道攻击） │ │ - 不需要随机数生成器（签名中） │ └─────────────────────────────────────────────────────────┘
推荐使用Ed25519（SSH密钥）或ECDSA（TLS证书）替代RSA签名，性能更好且密钥更短。Ed25519 签名只需 64 字节，验证速度比 RSA-2048 快约 10 倍。
