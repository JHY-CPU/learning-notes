# AES与DES


## AES与DES对称加密


密码学AES分组密码


对称加密使用同一个密钥进行加密和解密，是密码学的基础。


## 分组密码基础


```
分组密码 (Block Cipher)：
- 将明文分成固定大小的块，逐块加密
- 每块大小：DES=64bit, AES=128bit
- 密钥长度：DES=56bit, AES=128/192/256bit

对称加密 vs 非对称加密：
┌──────────┬──────────────┬──────────────┐
│ 特性      │ 对称加密       │ 非对称加密    │
├──────────┼──────────────┼──────────────┤
│ 密钥      │ 同一密钥       │ 公钥+私钥     │
│ 速度      │ 快(100-1000x) │ 慢            │
│ 密钥分发  │ 困难           │ 简单(公开公钥) │
│ 适用场景  │ 大数据加密      │ 密钥交换/签名  │
└──────────┴──────────────┴──────────────┘

实际应用：混合加密
1. 用非对称加密(RSA)交换对称密钥
2. 用对称加密(AES)加密实际数据
（HTTPS的TLS协议就是这样工作的）
```


## DES (数据加密标准)


```
DES 算法参数：
- 分组大小：64位
- 密钥长度：64位（有效56位，8位校验）
- 轮数：16轮Feistel结构
- 已不安全：1999年被暴力破解（22小时）

Feistel 网络结构：
┌──────────────────────────────────────────┐
│  L0 (32位)          R0 (32位)            │
│   │                   │                  │
│   │              ┌────┴────┐             │
│   │              │ F函数    │             │
│   │              └────┬────┘             │
│   │    ⊕ ←────────────┘                  │
│   │                                       │
│  R0(=L1)        L0⊕F(R0,K0)(=R1)        │
│   │                   │                  │
│  ...               ... (16轮)            │
│                                           │
│  最后一轮不做交叉（保持加解密对称）         │
└──────────────────────────────────────────┘

Feistel优势：
- 加解密结构相同（只需反转子密钥顺序）
- F函数不需要可逆
- 容易实现

3DES (Triple DES)：
- 对数据加密3次：C = E_K3(D_K2(E_K1(P)))
- 密钥长度168位（实际安全性约80位）
- 仍被用于金融系统，但速度慢
```


## AES (高级加密标准)


```
AES 算法参数：
┌──────────┬──────────┬──────────┬──────────┐
│          │ AES-128  │ AES-192  │ AES-256  │
├──────────┼──────────┼──────────┼──────────┤
│ 密钥长度  │ 128位    │ 192位    │ 256位    │
│ 分组大小  │ 128位    │ 128位    │ 128位    │
│ 轮数      │ 10轮     │ 12轮     │ 14轮     │
└──────────┴──────────┴──────────┴──────────┘

AES 每轮操作：
1. SubBytes：字节替换（S盒非线性变换）
2. ShiftRows：行移位（打乱行顺序）
3. MixColumns：列混合（列内线性变换）
4. AddRoundKey：轮密钥加（XOR轮密钥）

最后一轮跳过MixColumns。

AES vs DES对比：
- AES分组128位 > DES分组64位（更安全）
- AES不是Feistel结构，而是SPN结构
- AES硬件实现更快（AES-NI指令集）
- AES是当前最广泛使用的对称加密算法
```


## 分组密码工作模式


```
ECB (电子密码本)：
P1 → Enc(K) → C1
P2 → Enc(K) → C2
问题：相同明文产生相同密文，不安全！
适用：加密密钥或随机数据

CBC (密码分组链接)：
IV → ⊕ → P1 → Enc(K) → C1 → ⊕ → P2 → Enc(K) → C2
每个明文块先与前一个密文块XOR
特点：需要IV(初始化向量)，串行处理

CTR (计数器模式)：
Nonce+Counter → Enc(K) → Keystream → ⊕ → C
将分组密码变为流密码
特点：可并行，支持随机访问

GCM (Galois/Counter Mode)：
- 基于CTR模式 + GMAC认证
- 提供机密性 + 完整性 + 认证
- 是目前推荐的工作模式
- TLS 1.3默认使用AES-GCM

推荐选择：
- 首选：AES-256-GCM（机密+认证）
- 备选：AES-256-CBC + HMAC（两步法）
- 禁止：ECB模式
```


> **Note:** 永远不要使用ECB模式！相同的明文块会产生相同的密文块，泄露数据模式。首选GCM模式。


## AES 内部结构详解


```
AES 状态矩阵 (State)：
┌─────────────────────────────────────────────────────────┐
│  128 位明文组织为 4×4 字节矩阵：                          │
│  ┌──────┬──────┬──────┬──────┐                          │
│  │ s₀₀  │ s₀₁  │ s₀₂  │ s₀₃  │  ← 第 0 行              │
│  ├──────┼──────┼──────┼──────┤                          │
│  │ s₁₀  │ s₁₁  │ s₁₂  │ s₁₃  │  ← 第 1 行              │
│  ├──────┼──────┼──────┼──────┤                          │
│  │ s₂₀  │ s₂₁  │ s₂₂  │ s₂₃  │  ← 第 2 行              │
│  ├──────┼──────┼──────┼──────┤                          │
│  │ s₃₀  │ s₃₁  │ s₃₂  │ s₃₃  │  ← 第 3 行              │
│  └──────┴──────┴──────┴──────┘                          │
│  每轮操作在此矩阵上进行                                   │
└─────────────────────────────────────────────────────────┘

SubBytes（S 盒替换）：
┌─────────────────────────────────────────────────────────┐
│  每个字节独立进行非线性替换                                │
│  S 盒构造：                                              │
│  1. 在 GF(2⁸) 中求逆元：b → b⁻¹（0 映射到 0）          │
│  2. 仿射变换：b' = Ab + c                                │
│     A 是 8×8 二进制矩阵，c 是常量向量                    │
│                                                         │
│  S 盒提供 AES 的非线性，抵抗线性分析                      │
│  设计保证了高代数次数和良好的扩散性                       │
│                                                         │
│  S 盒的前几行（16进制）：                                  │
│  63 7C 77 7B F2 6B 6F C5 30 01 67 2B FE D7 AB 76       │
│  CA 82 C9 7D FA 59 47 F0 AD D4 A2 AF 9C A4 72 C0       │
│  B7 FD 93 26 36 3F F7 CC 34 A5 E5 F1 71 D8 31 15       │
└─────────────────────────────────────────────────────────┘

ShiftRows（行移位）：
┌─────────────────────────────────────────────────────────┐
│  第 0 行：不移位                                          │
│  第 1 行：循环左移 1 字节                                 │
│  第 2 行：循环左移 2 字节                                 │
│  第 3 行：循环左移 3 字节                                 │
│                                                         │
│  ┌──────┬──────┬──────┬──────┐     ┌──────┬──────┬──────┬──────┐ │
│  │ a    │ b    │ c    │ d    │     │ a    │ b    │ c    │ d    │ │
│  │ e    │ f    │ g    │ h    │ →   │ f    │ g    │ h    │ e    │ │
│  │ i    │ j    │ k    │ l    │     │ k    │ l    │ i    │ j    │ │
│  │ m    │ n    │ o    │ p    │     │ p    │ m    │ n    │ o    │ │
│  └──────┴──────┴──────┴──────┘     └──────┴──────┴──────┴──────┘ │
│  提供列之间的扩散                                         │
└─────────────────────────────────────────────────────────┘

MixColumns（列混合）：
┌─────────────────────────────────────────────────────────┐
│  对每一列进行 GF(2⁸) 上的矩阵乘法：                       │
│  ┌ s'₀ᵢ ┐   ┌ 02 03 01 01 ┐ ┌ s₀ᵢ ┐                    │
│  │ s'₁ᵢ │ = │ 01 02 03 01 │ │ s₁ᵢ │                    │
│  │ s'₂ᵢ │   │ 01 01 02 03 │ │ s₂ᵢ │                    │
│  └ s'₃ᵢ ┘   └ 03 01 01 02 ┘ └ s₃ᵢ ┘                    │
│                                                         │
│  其中乘法和加法都在 GF(2⁸) 上：                           │
│  加法 = XOR                                              │
│  乘法 = 不可约多项式 x⁸+x⁴+x³+x+1 下的模乘              │
│                                                         │
│  提供字节之间的充分混合                                    │
│  最后一轮跳过 MixColumns（已足够安全）                     │
└─────────────────────────────────────────────────────────┘

密钥扩展 (Key Expansion)：
┌─────────────────────────────────────────────────────────┐
│  将 128/192/256 位密钥扩展为 11/13/15 个轮密钥           │
│  AES-128 示例：                                          │
│  初始密钥 W[0] W[1] W[2] W[3]（每个 32 位）              │
│  W[i] = W[i-4] ⊕ g(W[i-1])  （i ≡ 0 mod 4 时用 g）     │
│  W[i] = W[i-4] ⊕ W[i-1]     （其他情况）                 │
│                                                         │
│  g 函数：                                                │
│  1. 字循环：[a,b,c,d] → [b,c,d,a]                       │
│  2. S 盒替换：4 个字节分别过 S 盒                         │
│  3. 轮常量异或：[b,c,d,a] ⊕ [RC,0,0,0]                  │
│  RC 值：01, 02, 04, 08, 10, 20, 40, 80, 1B, 36         │
│                                                         │
│  AES-256 额外使用 h 函数（S 盒替换）来增加非线性          │
└─────────────────────────────────────────────────────────┘
```


> **Note:** AES 的安全性来自 S 盒的非线性和多轮迭代的扩散。SubBytes 提供非线性，ShiftRows 和 MixColumns 提供扩散，AddRoundKey 提供密钥混合。AES-NI 硬件指令可在单个时钟周期完成一轮 AES 操作。


## AES 安全性与攻击进展


```
已知攻击：
┌─────────────────────────────────────────────────────────┐
│  1. 暴力破解：                                           │
│     AES-128：2¹²⁸ ≈ 3.4×10³⁸ 次尝试（不可行）            │
│     AES-256：2²⁵⁶ 次尝试（完全不可行）                    │
│                                                         │
│  2. 侧信道攻击（Side-Channel）：                          │
│     - 功耗分析（SPA/DPA）：通过芯片功耗推断密钥           │
│     - 缓存时序攻击：利用 AES 查表的缓存行为              │
│     防御：常数时间实现、AES-NI 硬件指令                   │
│                                                         │
│  3. 相关密钥攻击（Biclique）：                            │
│     - 对 AES-128：复杂度 2¹²⁶·¹（理论，不实用）          │
│     - 对 AES-192：复杂度 2¹⁸⁹·⁷                          │
│     - 对 AES-256：复杂度 2²⁵⁴·⁴                          │
│     - 仅适用于相关密钥场景，不构成实际威胁                │
│                                                         │
│  4. 量子攻击（Grover）：                                  │
│     - AES-128 → 等效 64 位安全性（不安全）               │
│     - AES-256 → 等效 128 位安全性（仍安全）              │
│     → 建议使用 AES-256 以抵御量子攻击                    │
│                                                         │
│  结论：AES 至今未被发现实用攻击，是最安全的对称加密算法   │
└─────────────────────────────────────────────────────────┘
```
