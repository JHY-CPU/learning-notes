# 国密算法SM系列


## 国密算法 SM 系列


密码学国密SM2/SM3/SM4


国密算法是中国国家密码管理局发布的商用密码算法标准，包括 SM2（非对称）、SM3（哈希）、SM4（对称）和 SM9（标识密码）。


## SM2 椭圆曲线公钥密码算法


SM2 基于椭圆曲线密码学（ECC），用于数字签名、密钥交换和公钥加密，是中国对 RSA/ECC 的替代方案。


```
SM2 算法概述：
┌─────────────────────────────────────────────────────────┐
│  标准：GM/T 0003-2012                                   │
│  基于：椭圆曲线离散对数问题 (ECDLP)                       │
│  替代：RSA-2048、ECDSA (NIST P-256)                     │
│  密钥长度：256 位（等效 RSA-3072 的安全强度）              │
└─────────────────────────────────────────────────────────┘

推荐椭圆曲线参数 (sm2p256v1)：
- 素数域 F_p，p 为 256 位素数
- 曲线方程：y² = x³ + ax + b
- 基点 G 的阶 n 为大素数
- 余因子 h = 1

密钥生成：
1. 选择随机数 d ∈ [1, n-2] 作为私钥
2. 计算公钥 P = d × G（椭圆曲线标量乘法）
3. 公钥 P 是曲线上的一个点 (x_P, y_P)

SM2 数字签名：
1. 计算 e = Hash(Z_A || M)，其中 Z_A 为用户标识
2. 选择随机数 k ∈ [1, n-1]
3. 计算 (x_1, y_1) = k × G
4. r = (e + x_1) mod n；若 r=0 或 r+k=n 则重选 k
5. s = (1+d)^(-1) × (k - r×d) mod n
6. 签名 = (r, s)

SM2 密钥交换（3步握手）：
┌─────────────────────────────────────────────────────────┐
│  Alice (私钥 d_A, 公钥 P_A)     Bob (私钥 d_B, 公钥 P_B)│
│                                                         │
│  Step 1: A → B: R_A = r_A × G                           │
│  Step 2: B → A: R_B = r_B × G                           │
│  Step 3: 双方计算共享密钥：                               │
│    Alice: x = (x_A + r_A) mod n                         │
│           K_A = KDF(x × y × (R_B + x × P_B))            │
│    Bob:   x = (x_B + r_B) mod n                         │
│           K_B = KDF(x × y × (R_A + x × P_A))            │
│    验证：双方交换 Hash 值确认密钥一致                      │
└─────────────────────────────────────────────────────────┘

SM2 公钥加密：
1. 选择随机数 k ∈ [1, n-1]
2. 计算 C_1 = k × G（椭圆曲线点）
3. 计算 k × P_B = (x_2, y_2)
4. C_2 = M ⊕ KDF(x_2 || y_2)
5. C_3 = Hash(x_2 || M || y_2)
6. 密文 = C_1 || C_3 || C_2

与国际标准对比：
┌──────────┬──────────┬──────────────┬──────────────┐
│ 特性      │ SM2      │ ECDSA (P-256)│ RSA-2048     │
├──────────┼──────────┼──────────────┼──────────────┤
│ 安全基础  │ ECDLP    │ ECDLP        │ 大整数分解   │
│ 密钥长度  │ 256位    │ 256位        │ 2048位       │
│ 签名速度  │ 较快     │ 较快         │ 较慢         │
│ 签名长度  │ 64字节   │ 64字节       │ 256字节      │
│ 标识绑定  │ 支持     │ 不支持       │ 不支持       │
│ 国内合规  │ 必须     │ 可选         │ 可选         │
└──────────┴──────────┴──────────────┴──────────────┘
```


## SM3 密码杂凑算法


SM3 是中国国家密码标准的哈希算法，输出 256 位摘要，安全性与 SHA-256 相当。


```
SM3 算法概述：
┌─────────────────────────────────────────────────────────┐
│  标准：GM/T 0004-2012                                   │
│  输出：256 位（32 字节）                                  │
│  结构：Merkle-Damgard 结构，类似 SHA-256                 │
│  消息分组：512 位（64 字节）                               │
│  安全强度：128 位碰撞抗性                                 │
└─────────────────────────────────────────────────────────┘

SM3 处理流程：
1. 消息填充：
   - 附加 bit "1"
   - 附加 k 个 "0"，使得长度 ≡ 448 (mod 512)
   - 附加 64 位消息长度（大端序）

2. 消息扩展（每 512 位块）：
   W_0, W_1, ..., W_15 ← 消息块的 16 个 32 位字
   W_j = P_1(W_{j-16} ⊕ W_{j-9} ⊕ (W_{j-3} <<< 15))
         ⊕ (W_{j-13} <<< 7) ⊕ W_{j-6}
   W'_j = W_j ⊕ W_{j+4}

3. 压缩函数（64 轮迭代）：
   SS_1 = ((A <<< 12) + E + (T_j <<< j)) <<< 7
   SS_2 = SS_1 ⊕ (A <<< 12)
   TT_1 = FF_j(A,B,C,D) + D + SS_2 + W'_j
   TT_2 = GG_j(E,F,G,H) + H + SS_1 + W_j
   其中：
   - 前 16 轮：FF_j = A ⊕ B ⊕ C，GG_j = E ⊕ F ⊕ G
   - 后 48 轮：FF_j = (A∧B)∨(A∧C)∨(B∧C)

4. 输出：A || B || C || D || E || F || G || H（256 位）

SM3 vs SHA-256 对比：
┌──────────┬──────────┬──────────┐
│ 特性      │ SM3      │ SHA-256  │
├──────────┼──────────┼──────────┤
│ 输出长度  │ 256位    │ 256位    │
│ 结构      │ M-D      │ M-D      │
│ 轮数      │ 64轮     │ 64轮     │
│ 常量      │ 自定义   │ 立方根   │
│ 安全性    │ 128位    │ 128位    │
│ 国内合规  │ 必须     │ 可选     │
└──────────┴──────────┴──────────┘
```


## SM4 分组密码算法


SM4 是中国的对称加密标准，用于替代 AES 在国内敏感领域的使用。


```
SM4 算法概述：
┌─────────────────────────────────────────────────────────┐
│  标准：GM/T 0002-2012                                   │
│  类型：分组密码（SPN 结构）                               │
│  分组长度：128 位                                        │
│  密钥长度：128 位                                        │
│  轮数：32 轮                                             │
│  替代：AES-128                                           │
└─────────────────────────────────────────────────────────┘

SM4 加密过程：
明文 (X_0, X_1, X_2, X_3)，每部分 32 位

32 轮迭代：
X_{i+4} = F(X_i, X_{i+1}, X_{i+2}, X_{i+3}, rk_i)
        = X_i ⊕ T(X_{i+1} ⊕ X_{i+2} ⊕ X_{i+3} ⊕ rk_i)

T 变换 = L 变换 ∘ τ 变换：
τ 变换：4 个并行的 S 盒替换（8→8 位非线性变换）
L 变换：B ⊕ B <<< 2 ⊕ B <<< 10 ⊕ B <<< 18 ⊕ B <<< 24

密文输出：(X_35, X_34, X_33, X_32) ← 最后 4 轮输出逆序

轮密钥生成：
MK = (MK_0, MK_1, MK_2, MK_3) ← 128 位主密钥
K_0 = MK_0 ⊕ FK_0，K_1 = MK_1 ⊕ FK_1，...
rk_i = K_{i+4} = K_i ⊕ T'(K_{i+1} ⊕ K_{i+2} ⊕ K_{i+3} ⊕ CK_i)

其中 FK 和 CK 为固定常量。

SM4 vs AES 对比：
┌──────────┬──────────┬──────────────┐
│ 特性      │ SM4      │ AES-128      │
├──────────┼──────────┼──────────────┤
│ 分组长度  │ 128位    │ 128位        │
│ 密钥长度  │ 128位    │ 128位        │
│ 轮数      │ 32轮     │ 10轮         │
│ 结构      │ SPN      │ SPN          │
│ S盒设计  │ 自主设计  │ 代数结构     │
│ 差分分析  │ 安全     │ 安全         │
│ 硬件加速  │ 部分支持 │ AES-NI 广泛  │
│ 国内合规  │ 必须     │ 禁止         │
└──────────┴──────────┴──────────────┘
```


## SM9 标识密码算法


SM9 是基于标识的密码算法（IBC），用户的公钥直接从其身份标识（如邮箱地址）派生，无需数字证书。


```
SM9 IBC (Identity-Based Cryptography)：
┌─────────────────────────────────────────────────────────┐
│  核心思想：公钥 = 用户身份标识（如 email@example.com）     │
│  优势：无需 CA、无需证书、简化密钥管理                      │
│  标准：GM/T 0044-2016                                   │
└─────────────────────────────────────────────────────────┘

系统架构：
┌─────────────────────────────────────────────────────────┐
│  密钥生成中心 (KGC / Key Generation Center)              │
│  ┌─────────────────────────────────────────┐            │
│  │  主密钥对：                               │            │
│  │    主私钥：s（秘密保管）                   │            │
│  │    主公钥：P_pub = s × P_1（公开）         │            │
│  │                                         │            │
│  │  用户密钥生成：                           │            │
│  │    输入：用户身份 ID                      │            │
│  │    输出：私钥 d_ID = s × Q_ID             │            │
│  │    其中 Q_ID = H_1(ID)（身份哈希到曲线点） │            │
│  └─────────────────────────────────────────┘
│  用户公钥 = ID（无需分发！）                             │
└─────────────────────────────────────────────────────────┘

SM9 加密：
1. 接收方公钥 = 接收方 ID
2. 发送方计算 Q_B = H_1(ID_B)
3. 选择随机数 r，计算 g = e(P_pub, Q_B)
4. C_1 = r × P_2（椭圆曲线点）
5. C_2 = M ⊕ KDF(g^r)
6. 密文 = (C_1, C_2)

SM9 数字签名：
1. 签名者私钥 d_A = s × H_1(ID_A)
2. 选择随机数 r，计算 w = e(P_1, P_pub)^r
3. h = H_2(M || w)
4. l = (r - h) mod n
5. S = l × d_A
6. 签名 = (h, S)

使用双线性配对 (Bilinear Pairing)：
e: G_1 × G_2 → G_T
- e(a × P, b × Q) = e(P, Q)^(a × b)
- 基于超奇异椭圆曲线上的 Weil 配对或 Tate 配对

SM9 vs SM2+PKI 对比：
┌──────────────┬──────────────┬────────────────────┐
│ 特性          │ SM9 (IBC)    │ SM2 + PKI          │
├──────────────┼──────────────┼────────────────────┤
│ 公钥形式      │ 身份字符串   │ 证书中的公钥        │
│ 证书          │ 不需要       │ 必需（X.509）       │
│ 密钥管理      │ 集中式(KGC)  │ 分布式(CA层次)      │
│ 新用户部署    │ 简单         │ 需申请证书           │
│ 密钥撤销      │ 较复杂       │ CRL/OCSP 支持好     │
│ 计算开销      │ 较大(配对)   │ 较小                │
│ 适用场景      │ 物联网/邮件  │ Web/TLS            │
└──────────────┴──────────────┴────────────────────┘
```


> **Note:** 国密算法在国内金融、政务、通信等领域有强制使用要求。SM2 用于签名和密钥交换，SM3 用于哈希，SM4 用于对称加密，SM9 用于无证书场景。OpenSSL 3.0+ 已支持国密算法。

```
国密算法在各领域的应用：
┌─────────────────────────────────────────────────────────┐
│  金融领域：                                               │

│  - 移动支付：SM2 签名交易数据                              │
│  - 网银：国密 SSL 证书 (SM2+SM3+SM4)                      │
│                                                         │
│  政务领域：                                               │
│  - 电子政务外网：SM2 证书认证                              │
│  - 公文传输：SM4 加密 + SM3 完整性校验                     │
│  - 电子签章：SM2 数字签名                                 │
│                                                         │
│  通信领域：                                               │
│  - 5G 安全：SM4 加密通信数据                              │
│  - VPN：国密 IPSec / SSL VPN                              │
│                                                         │
│  物联网：                                                 │
│  - 设备认证：SM9 无证书认证                                │
│  - 固件更新：SM2 签名 + SM3 校验                           │
└─────────────────────────────────────────────────────────┘

国密 TLS (GMTLS) 握手流程：
Client → Server: ClientHello (支持国密密码套件)
Server → Client: ServerHello (选择国密套件)
                  + SM2 证书
                  + SM2 签名的 ServerKeyExchange
Client → Server: SM2 密钥交换 + Finished
双方使用 SM4-GCM 加密通信

国密密码套件：
┌─────────────────────────────────────────────────────────┐
│  ECDHE_SM4_CBC_SM3   ← ECDHE + SM4-CBC + SM3 HMAC      │
│  ECDHE_SM4_GCM_SM3   ← ECDHE + SM4-GCM + SM3           │
│  SM2_SM4_CBC_SM3     ← SM2 + SM4-CBC + SM3             │
│  SM2_SM4_GCM_SM3     ← SM2 + SM4-GCM + SM3             │
└─────────────────────────────────────────────────────────┘

开源支持：
- OpenSSL 3.0+：provider 模式支持国密
- GmSSL：国密 OpenSSL 分支，功能最全
- BouncyCastle：Java/C# 国密实现
- Tongsuo（铜锁）：蚂蚁集团 OpenSSL 分支
```
