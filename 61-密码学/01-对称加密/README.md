# 01-对称加密

## 1. 密码学概述

密码学（Cryptography）的核心目标是保障信息的**四大安全属性**：

| 属性 | 含义 | 实现手段 |
|------|------|----------|
| **保密性**（Confidentiality） | 信息仅被授权方访问 | 加密算法 |
| **完整性**（Integrity） | 信息未被篡改 | 哈希函数、MAC |
| **认证性**（Authentication） | 确认通信方身份 | 数字签名、证书 |
| **不可否认性**（Non-repudiation） | 发送方无法否认已发送的消息 | 数字签名 |

密码学分为两大分支：
- **对称加密**（Symmetric Cryptography）：加密与解密使用相同密钥
- **非对称加密**（Asymmetric Cryptography）：使用一对公钥/私钥

本章聚焦对称加密。

---

## 2. 对称加密基本概念

对称加密的核心思想：通信双方共享一个**秘密密钥**（Secret Key），发送方用密钥加密明文得到密文，接收方用同一密钥解密密文恢复明文。

```
明文 P --[加密算法 E, 密钥 K]--> 密文 C
密文 C --[解密算法 D, 密钥 K]--> 明文 P
```

**优点**：加解密速度快，适合大数据量加密。

**缺点**：密钥分发困难——如何安全地将密钥传递给通信对端？

对称加密分为两类：
- **分组密码**（Block Cipher）：将明文分成固定长度的块（如128位），逐块加密
- **流密码**（Stream Cipher）：逐比特/逐字节加密，常使用密钥流与明文异或

---

## 3. 分组密码

### 3.1 Feistel结构

Feistel结构由Horst Feistel提出，是许多经典分组密码的基础框架。其核心思想是将加密和解密使用相同的结构，仅密钥顺序相反。

**一轮Feistel轮函数**：
```
输入：左半部分 L(i-1)，右半部分 R(i-1)，子密钥 Ki
L(i) = R(i-1)
R(i) = L(i-1) XOR F(R(i-1), Ki)
```

其中 F 是轮函数（Round Function），包含代换、置换、密钥混合等操作。

**特点**：
- 加密和解密结构对称，只需反转子密钥顺序
- 轮函数 F 不需要可逆
- 多轮迭代增加安全性（通常16轮以上）

---

### 3.2 DES（数据加密标准）

DES（Data Encryption Standard）由IBM设计，1977年被NIST采纳为联邦标准。

**基本参数**：
- 分组长度：64位
- 密钥长度：64位（含8位校验位，实际有效56位）
- 轮数：16轮Feistel结构

**DES加密流程**：初始置换IP -> 16轮Feistel迭代（每轮含扩展置换、S盒代换、P盒置换，使用48位子密钥） -> 末置换IP⁻¹。

**安全性**：56位密钥空间仅2⁵⁶，1999年已被暴力破解，已被AES取代。

---

### 3.3 3DES（三重DES）

3DES通过对数据执行三次DES来增强安全性：

```
C = E_K3( D_K2( E_K1(P) ) )
```

- 当 K1 = K2 = K3 时，退化为普通DES（向后兼容）
- 有效密钥长度：112位（双密钥3DES）或168位（三密钥3DES）
- 缺点：速度是DES的三分之一，分组仍为64位（存在生日攻击边界问题）

---

### 3.4 AES（高级加密标准）

AES（Advanced Encryption Standard）由比利时密码学家Joan Daemen和Vincent Rijmen设计的Rijndael算法，2001年被NIST选为DES的继任者。

**基本参数**：
- 分组长度：128位（固定）
- 密钥长度：128/192/256位
- 轮数：AES-128为10轮，AES-192为12轮，AES-256为14轮
- 结构：**SPN**（代换-置换网络），非Feistel结构

**AES加密的四个步骤（每轮）**：

#### ① SubBytes（字节代换）
对状态矩阵中每个字节，通过S盒进行非线性代换，提供混淆（Confusion）效果。

#### ② ShiftRows（行移位）
对状态矩阵每一行循环左移不同字节数（0/1/2/3），提供扩散效果。

#### ③ MixColumns（列混合）
对每一列做GF(2⁸)上的矩阵乘法，进一步扩散每个字节的影响（最后一轮省略）。

#### ④ AddRoundKey（轮密钥加）
将当前状态与该轮子密钥逐字节异或。

**密钥扩展**：将初始密钥扩展为 Nr+1 个轮密钥（每个轮密钥128位）。

## 4. 分组密码工作模式

分组密码本身只能加密固定长度的数据块，**工作模式**（Block Cipher Modes of Operation）定义了如何处理长于一个分组的消息。

### 4.1 ECB（电子密码本）

```
C_i = E_K(P_i)
```

每个分组独立加密，相同明文块产生相同密文块。

- **优点**：可并行，简单
- **缺点**：无法隐藏数据模式，**不推荐使用**

### 4.2 CBC（密码分组链接）

```
C_0 = E_K(P_0 XOR IV)
C_i = E_K(P_i XOR C_(i-1))
```

每个明文块先与前一个密文块异OR后再加密，第一个块使用初始化向量（IV）。

- **优点**：隐藏了明文模式，安全性好
- **缺点**：加密不可并行（解密可并行），需要随机IV，一个比特错误会影响后续块

### 4.3 CFB（密文反馈）

```
C_i = P_i XOR E_K(C_(i-1))
```

将分组密码当作流密码使用，密文反馈到下一加密操作。

- 可将分组密码转化为自同步流密码
- 解密不可并行（因为需要前一个密文）

### 4.4 OFB（输出反馈）

```
S_i = E_K(S_(i-1))    # 密钥流
C_i = P_i XOR S_i
```

分组密码的输出作为密钥流，与明文异或。

- 加密和解密都使用加密操作
- 比特翻转攻击可能导致对应位置的明文被篡改（无完整性保护）

### 4.5 CTR（计数器模式）

```
C_i = P_i XOR E_K(Nonce || Counter_i)
```

使用Nonce和计数器的组合作为分组密码的输入，生成密钥流。

- **优点**：加密和解密均可并行，支持预计算，效率高
- 缺点：Nonce绝不能重复使用

### 4.6 GCM（Galois/计数器模式）

GCM = CTR模式加密 + GMAC认证。同时提供**保密性**和**完整性**。

- 基于CTR模式加密数据
- 使用GF(2¹²⁸)上的乘法计算认证标签（GHASH）
- 被广泛用于TLS 1.3、IPSec等协议
- 高效且支持硬件加速（AES-NI指令集）

## 5. 流密码

流密码逐比特或逐字节加密，核心是生成伪随机密钥流（Keystream），与明文异或。

```
C_i = P_i XOR KS_i
```

### RC4
- 由Ron Rivest于1987年设计
- 密钥长度1-256字节，状态向量256字节
- 曾广泛用于SSL/TLS和WEP
- **已不安全**：存在偏差攻击，如Fluhrer-Mantin-Shamir攻击

### ChaCha20 / Salsa20
- 由Daniel J. Bernstein设计
- ChaCha20是Salsa20的改进版本
- 256位密钥，96位Nonce
- 软件实现效率极高，安全分析表现优秀
- 被TLS 1.3、WireGuard等现代协议采用
- 常与Poly1305消息认证码配合使用（ChaCha20-Poly1305）

## 6. 填充方案（PKCS#7）

分组密码要求明文长度为分组长度的整数倍。当明文不满足此要求时，需要填充。

**PKCS#7填充规则**：
- 在明文末尾填充 N 个字节，每个字节的值均为 N
- N 的范围：1 到 分组长度（如AES为16字节）
- 如果明文正好是分组长度的整数倍，则填充一整块（16个值为0x10的字节）

**示例**（AES-128，分组16字节）：
```
原始数据：[0x48 0x65 0x6C 0x6C 0x6F]  (5字节 "Hello")
填充后：  [0x48 0x65 0x6C 0x6C 0x6F 0x0B 0x0B ... 0x0B]  (16字节，填充11个0x0B)
```

注意：CTR模式和GCM模式不需要填充（它们是流密码模式）。

## 7. 对称密钥管理与分发

对称加密的核心挑战在于**密钥管理**：

### 密钥分发问题
- **预共享密钥（PSK）**：双方事先通过安全信道交换密钥，适合少量用户
- **密钥分发中心（KDC）**：由可信第三方为每对用户生成会话密钥（如Kerberos协议）
- **Diffie-Hellman交换**：通过非对称方式协商对称密钥（详见第02章）

### 密钥管理要点
- **密钥生命周期**：定期更换密钥，限制密钥使用期限和加密数据量
- **密钥派生函数（KDF）**：如HKDF，从一个主密钥派生出多个用途不同的子密钥
- **密钥存储**：密钥应存储在安全区域（如HSM、TPM），绝不能以明文存储
- **密钥层级**：使用密钥加密密钥（KEK）保护数据加密密钥（DEK），形成层级结构
