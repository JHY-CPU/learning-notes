# 后量子密码学


## 后量子密码学 PQC


密码学PQC抗量子


后量子密码学研究能够抵抗量子计算机攻击的密码算法，以替代 RSA 和 ECC 等将被 Shor 算法破解的方案。


## 量子计算机对密码学的威胁


```
量子算法对现有密码的影响：
┌─────────────────────────────────────────────────────────┐
│  Shor 算法（多项式时间）：                                │
│  - 大整数分解 → RSA 被破解                               │
│  - 离散对数 → DH/ECDH/ECDSA 被破解                      │
│  - 影响：公钥加密、密钥交换、数字签名全部失效             │
│                                                         │
│  Grover 算法（平方加速）：                                │
│  - 对称加密：密钥强度减半                                │
│    AES-128 → 等效 64 位安全性（不安全）                  │
│    AES-256 → 等效 128 位安全性（仍安全）                 │
│  - 哈希函数：碰撞抗性减半                                │
│    SHA-256 → 等效 128 位碰撞抗性（仍安全）               │
│  - 影响：对称密码和哈希可通过增加密钥/输出长度应对        │
└─────────────────────────────────────────────────────────┘

"Harvest Now, Decrypt Later" 攻击：
┌─────────────────────────────────────────────────────────┐
│  攻击者现在录制加密通信数据                               │
│  等量子计算机成熟后解密                                  │
│                                                         │
│  对长期机密数据的威胁：                                   │
│  - 政府机密（保密期可能 50+ 年）                         │
│  - 医疗记录（终身保密）                                  │
│  - 金融数据（长期合规要求）                               │
│  - 基础设施控制指令                                      │
│                                                         │
│  NIST 建议：2030 年前完成 PQC 迁移                       │
│  NSA 要求：2035 年全面使用 PQC                           │
└─────────────────────────────────────────────────────────┘
```


## 格密码 (Lattice-based Cryptography)


```
格密码基础：
┌─────────────────────────────────────────────────────────┐
│  格 (Lattice)：n 维空间中的一组离散点，由一组基向量生成   │
│  L = {Σ aᵢbᵢ : aᵢ ∈ Z}，其中 {b₁, ..., bₙ} 为基       │
│                                                         │
│  困难问题：                                              │
│  - 最短向量问题 (SVP)：找格中最短非零向量                │
│  - 最近向量问题 (CVP)：找离给定点最近的格点              │
│  - 学习带错误问题 (LWE)：从 Ax + e 中恢复 x             │
│  - 小整数解问题 (SIS)：找小范数的非零格向量              │
│                                                         │
│  优势：                                                  │
│  - 抗量子：已知量子算法无法多项式时间解决                 │
│  - 最坏情况到平均情况的归约（安全性可证明）               │
│  - 支持同态加密和属性基加密                              │
└─────────────────────────────────────────────────────────┘

CRYSTALS-Kyber（NIST 标准 KEM）：
┌─────────────────────────────────────────────────────────┐
│  标准编号：FIPS 203 (ML-KEM)                             │
│  类型：密钥封装机制 (KEM)                                 │
│  基于：模格上的 LWE 问题 (Module-LWE)                    │
│                                                         │
│  Kyber 参数：                                            │
│  ┌────────────┬──────────┬──────────┬──────────┐        │
│  │ 参数        │ Kyber-512│ Kyber-768│Kyber-1024│        │
│  ├────────────┼──────────┼──────────┼──────────┤        │
│  │ 安全级别    │ AES-128  │ AES-192  │ AES-256  │        │
│  │ 公钥大小    │ 800B     │ 1184B    │ 1568B    │        │
│  │ 密文大小    │ 768B     │ 1088B    │ 1568B    │
│  │ 共享密钥    │ 32B      │ 32B      │ 32B      │        │
│  └────────────┴──────────┴──────────┴──────────┘        │
│                                                         │
│  密钥生成：                                              │
│  1. 生成矩阵 A ∈ R_q^(k×k)（公开）                       │
│  2. 采样私钥 s, 错误向量 e ← χ（中心二项分布）           │
│  3. 公钥 pk = (A, t = As + e)                            │
│  4. 私钥 sk = s                                          │
│                                                         │
│  封装（加密）：                                           │
│  1. 采样随机向量 r ← χ                                   │
│  2. 计算 u = A^T r + e₁                                  │
│  3. 计算 v = t^T r + e₂ + ⌈q/2⌋ × m（编码消息 m）       │
│  4. 密文 ct = (u, v)                                     │
│  5. 共享密钥 = KDF(H(ct))                                │
│                                                         │
│  解封装（解密）：                                         │
│  m' = v - s^T u = (t^T r + e₂ + ⌈q/2⌋m) - s^T(A^T r + e₁)
│     ≈ ⌈q/2⌋ × m  （误差很小，可正确解码）               │
│                                                         │
│  选择 Kyber-768 作为推荐参数（等效 AES-192 安全）        │
└─────────────────────────────────────────────────────────┘

CRYSTALS-Dilithium（NIST 标准签名）：
┌─────────────────────────────────────────────────────────┐
│  标准编号：FIPS 204 (ML-DSA)                             │
│  类型：数字签名方案                                       │
│  基于：模格上的 LWE 和 SIS 问题                          │
│                                                         │
│  Dilithium 参数：                                        │
│  ┌────────────┬──────────┬──────────┬──────────┐        │
│  │ 参数        │ ML-DSA-44│ ML-DSA-65│ML-DSA-87 │        │
│  ├────────────┼──────────┼──────────┼──────────┤        │
│  │ 安全级别    │ AES-128  │ AES-192  │ AES-256  │        │
│  │ 公钥大小    │ 1312B    │ 1952B    │ 2592B    │        │
│  │ 签名大小    │ 2420B    │ 3293B    │ 4595B    │        │
│  │ 推荐        │          │ ✅       │          │        │
│  └────────────┴──────────┴──────────┴──────────┘        │
│                                                         │
│  与 ECDSA 对比：                                         │
│  - ECDSA-256 公钥：33B，签名：64B                        │
│  - Dilithium 公钥和签名大 ~40-70 倍                     │
│  - 但抗量子且安全证明更强                                │
└─────────────────────────────────────────────────────────┘

NTRU（另一种格密码 KEM）：
┌─────────────────────────────────────────────────────────┐
│  历史最悠久的格密码方案（1996 年提出）                    │
│  基于多项式环上的困难问题                                 │
│                                                         │
│  NTRU-HPS（NIST 第四轮候选）：                            │
│  - 公钥：~699B（ntru-hps-509）                           │
│  - 密文：~699B                                           │
│  - 比 Kyber 更紧凑，但缺乏模块化结构                     │
│  - 未被选为 NIST 标准（Kyber 更优）                      │
└─────────────────────────────────────────────────────────┘
```


## 哈希签名方案 (Hash-based Signatures)


```
哈希签名的安全性：
┌─────────────────────────────────────────────────────────┐
│  仅依赖哈希函数的安全性（最保守的假设）                   │
│  即使量子计算机发展出更强的算法，只要哈希函数安全，签名就安全│
│  缺点：签名较大，密钥管理复杂                             │
└─────────────────────────────────────────────────────────┘

XMSS (eXtended Merkle Signature Scheme)：
┌─────────────────────────────────────────────────────────┐
│  标准：RFC 8391 / NIST SP 800-208                       │
│  类型：有状态签名方案（需跟踪已使用的一次性密钥对）       │
│                                                         │
│  结构：                                                  │
│  - 底层：WOTS+ (Winternitz One-Time Signature)           │
│  - 组织：Merkle 树（高度 h，支持 2^h 次签名）            │
│                                                         │
│  WOTS+ 一次性签名：                                      │
│  - 私钥：随机向量 (sk₁, ..., skₙ)                        │
│  - 公钥：对每个分量做 w 次哈希                            │
│    pkᵢ = H^w(skᵢ)                                      │
│  - 签名消息 m 的哈希值：                                  │
│    将哈希值分为 n 个 w 位的分量                           │
│    签名 σᵢ = H^(w - mᵢ)(skᵢ)                           │
│  - 验证：对 σᵢ 做 mᵢ 次哈希，应等于 pkᵢ                 │
│                                                         │
│  XMSS Merkle 树组织：                                    │
│  - 每个叶子节点是一个 WOTS+ 公钥                         │
│  - 根节点作为 XMSS 公钥                                  │
│  - 签名时使用下一个未使用的叶子节点                       │
│  - 必须跟踪使用状态（有状态！）                           │
│                                                         │
│  参数示例：                                               │
│  - XMSS-SHA2_10_256：h=10（1024 次签名）                │
│    公钥：64B，签名：2500B                                │
│  - 签名较大，但安全性极强                                │
└─────────────────────────────────────────────────────────┘

SPHINCS+（无状态哈希签名）：
┌─────────────────────────────────────────────────────────┐
│  标准编号：FIPS 205 (SLH-DSA)                            │
│  NIST 标准化的哈希签名方案                                │
│  核心创新：无状态（不需要跟踪使用过的密钥）               │
│                                                         │
│  架构：                                                  │
│  - 超树 (Hypertree)：多层 Merkle 树                      │
│  - FORS (Forest of Random Subsets)：底层少量签名方案     │
│  - WOTS+：中间层一次性签名                               │
│                                                         │
│  SPHINCS+ 参数对比：                                      │
│  ┌────────────────┬─────────┬─────────┬──────────┐      │
│  │ 参数            │ -128s   │ -128f   │ -256s    │      │
│  ├────────────────┼─────────┼─────────┼──────────┤      │
│  │ 安全级别        │ 128位   │ 128位   │ 256位    │      │
│  │ 公钥大小        │ 32B     │ 32B     │ 64B      │      │
│  │ 签名大小        │ 7856B   │ 17088B  │ 29792B   │      │
│  │ 签名速度        │ 慢      │ 快      │ 慢       │      │
│  │ 验证速度        │ 快      │ 快      │ 快       │      │
│  └────────────────┴─────────┴─────────┴──────────┘      │
│                                                         │
│  签名大小比格签名大很多，但：                              │
│  - 安全假设更保守（仅依赖哈希）                           │
│  - 实现简单，侧信道风险低                                 │
│  - 适合作为后量子签名的备份方案                           │
└─────────────────────────────────────────────────────────┘

有状态 vs 无状态签名：
┌─────────────────────────────────────────────────────────┐
│  有状态方案（XMSS, LMS）：                                │
│  - 签名更小                                              │
│  - 必须可靠跟踪使用状态                                   │
│  - 状态丢失 → 私钥泄露风险                               │
│  - 适合固件签名、证书颁发                                │
│                                                         │
│  无状态方案（SPHINCS+）：                                 │
│  - 不需要跟踪状态                                        │
│  - 签名更大                                              │
│  - 适合通用签名场景                                      │
│  - 推荐作为 PQC 签名的主要选择                           │
└─────────────────────────────────────────────────────────┘
```


## 码密码与其他 PQC 方向


```
码密码 (Code-based Cryptography)：
┌─────────────────────────────────────────────────────────┐
│  基于纠错码的困难问题                                     │
│                                                         │
│  McEliece 加密方案（1978）：                              │
│  - 历史最悠久的后量子方案之一                             │
│  - 公钥：G' = S × G × P（伪装后的生成矩阵）             │
│  - 私钥：(S, G, P)（随机矩阵、Goppa 码生成矩阵、置换矩阵）│
│  - 加密：c = m × G' + e（编码 + 加噪声）                 │
│  - 解密：使用 G 的高效译码算法去除噪声                    │
│                                                         │
│  Classic McEliece（NIST 第四轮候选）：                    │
│  - 公钥极大（~261KB 到 ~1MB）                            │
│  - 密文很小（~128B 到 ~240B）                            │
│  - 安全性经过 40+ 年检验                                 │
│  - 未被选为 NIST 标准（公钥太大）                         │
│                                                         │
│  BIKE (Bit Flipping Key Encapsulation)：                  │
│  - 基于 QC-MDPC 码（准循环中等密度奇偶校验码）            │
│  - 公钥较小（~1541B）                                    │
│  - NIST 第四轮候选                                       │
└─────────────────────────────────────────────────────────┘

多变量密码 (Multivariate Cryptography)：
┌─────────────────────────────────────────────────────────┐
│  基于求解多变量多项式方程组的困难性                       │
│                                                         │
│  核心思想：                                              │
│  - 公钥：多变量多项式映射 P: F^n → F^m                  │
│  - 私钥：P 的分解（容易求逆的映射组合）                  │
│  - 签名：利用私钥计算 P^(-1)(y)                          │
│                                                         │
│  Rainbow（NIST 第三轮候选，后被攻击）：                    │
│  - 签名很小（~66B）                                      │
│  - 2022 年被 Ward Beullens 攻破                          │
│  - 展示了多变量密码的安全风险                            │
│                                                         │
│  现状：多变量签名方案研究活跃但风险较高                   │
└─────────────────────────────────────────────────────────┘

同态签名与基于格的高级方案：
┌─────────────────────────────────────────────────────────┐
│  基于格的高级功能：                                       │
│  - 全同态加密 (FHE)：CKKS, TFHE 等方案均基于格            │
│  - 属性基加密 (ABE)：格上的 ABE 方案                     │
│  - 环签名 / 群签名：格上的匿名签名方案                   │
│                                                         │
│  量子密钥分发 (QKD)：                                     │
│  - 信息论安全的密钥分发                                  │
│  - 基于量子力学原理（窃听必然被检测）                     │
│  - 需要专用光纤或卫星信道                                │
│  - 不是密码学算法，而是物理层方案                         │
│  - 与 PQC 互补而非替代                                   │
└─────────────────────────────────────────────────────────┘
```


## NIST PQC 标准化进程


```
NIST PQC 标准化时间线：
┌─────────────────────────────────────────────────────────┐
│  2016：NIST 启动 PQC 标准化征集                          │
│  2017：69 个方案提交                                     │
│  2019：第二轮，26 个候选                                 │
│  2020：第三轮，15 个候选（7 个决赛 + 8 个备选）         │
│  2022.07：宣布首批标准候选                               │
│  2024.08：发布首批 FIPS 标准                             │
│  ─────────────────────────────────────                   │
│  FIPS 203: ML-KEM (CRYSTALS-Kyber) — 密钥封装           │
│  FIPS 204: ML-DSA (CRYSTALS-Dilithium) — 数字签名       │
│  FIPS 205: SLH-DSA (SPHINCS+) — 哈希签名（备用）       │
│  ─────────────────────────────────────                   │
│  2024+：第四轮候选（HQC, BIKE, Classic McEliece 等）    │
│  计划：签名方案补充（FN-DSA / Falcon）                   │
└─────────────────────────────────────────────────────────┘

Falcon（基于 NTRU 格的签名）：
┌─────────────────────────────────────────────────────────┐
│  NIST 选择的补充签名方案（FN-DSA）                        │
│  基于：NTRU 格上的 GPV 签名框架                          │
│                                                         │
│  优势：                                                  │
│  - 签名很小（~666B for Falcon-512）                     │
│  - 比 Dilithium 小 ~4 倍                                │
│  - 公钥较小（~897B）                                     │
│                                                         │
│  缺点：                                                  │
│  - 实现复杂（需要浮点运算或定点近似）                     │
│  - 侧信道攻击风险更高                                    │
│  - 需要仔细实现常数时间运算                              │
│                                                         │
│  Falcon-512 参数：                                       │
│  - 安全级别：AES-128                                     │
│  - 公钥：897B                                            │
│  - 签名：666B                                            │
└─────────────────────────────────────────────────────────┘

迁移策略：
┌─────────────────────────────────────────────────────────┐
│  阶段 1：混合模式（现在 - 2030）                          │
│  - X25519 + ML-KEM-768（密钥交换）                       │
│  - ECDSA-P256 + ML-DSA-65（签名）                        │
│  - 两层保护：经典安全 + 后量子安全                        │
│  - Signal 协议已部署 X25519 + ML-KEM-768                 │
│  - Chrome 已启用 X25519Kyber768 混合密钥交换             │
│                                                         │
│  阶段 2：纯 PQC（2030+）                                 │
│  - 移除经典算法                                          │
│  - 全面使用 ML-KEM 和 ML-DSA                             │
│                                                         │
│  迁移挑战：                                               │
│  - PQC 密钥和签名较大 → 协议需要调整                    │
│  - 性能差异 → 需要硬件加速                               │
│  - 兼容性 → 老系统需要更新                               │
│  - 合规要求 → 政府和金融系统需要优先迁移                  │
└─────────────────────────────────────────────────────────┘
```


> **Note:** 2024 年 8 月 NIST 正式发布了首批后量子密码标准（FIPS 203/204/205），标志着后量子密码时代的正式开始。建议立即开始混合模式迁移，特别是保护长期敏感数据。
