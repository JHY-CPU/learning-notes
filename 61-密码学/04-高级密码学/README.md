# 04-高级密码学

## 1. 零知识证明

零知识证明（Zero-Knowledge Proof, ZKP）允许证明者（Prover）向验证者（Verifier）证明某个陈述为真，而**不泄露任何额外信息**。

### 1.1 零知识性的定义

一个零知识证明系统必须满足三个性质：

| 性质 | 含义 |
|------|------|
| **完备性**（Completeness） | 如果陈述为真，诚实的验证者总能被诚实的证明者说服 |
| **可靠性**（Soundness） | 如果陈述为假，任何不诚实的证明者都无法说服诚实的验证者（只有极小概率） |
| **零知识性**（Zero-Knowledge） | 验证者在交互结束后，除了知道陈述为真之外，没有学到任何其他信息 |

### 1.2 交互式零知识证明

经典示例：**图同构的零知识证明**

- Alice想向Bob证明图G₁和G₂同构，但不揭示同构映射
- Bob生成一个随机同构的图H
- Alice证明H是G₁还是G₂的同构副本
- 重复多次，每次Bob都随机选择要挑战的内容
- 如果Alice每次都能正确回答，则以高概率证明她确实知道同构映射

### 1.3 非交互式零知识证明（NIZK）

交互式证明需要多轮通信。NIZK使用**公共参考串**（CRS）或**Fiat-Shamir启发式**将交互式证明转化为非交互式。

Fiat-Shamir变换：用哈希函数替代验证者的随机挑战值：
```
挑战值 c = H(公共输入, 承诺值)
```

### 1.4 zk-SNARKs基础

zk-SNARK（Zero-Knowledge Succinct Non-interactive Argument of Knowledge）：
- **Succinct**：证明体积小，验证时间短
- **Non-interactive**：无需交互
- **Argument**：计算可靠性（假设证明者算力有界）
- **of Knowledge**：证明者确实知道某个见证（Witness）

**基本框架**：
1. 将待证明的陈述转化为**算术电路**
2. 将算术电路转化为**R1CS**（Rank-1 Constraint System）
3. 将R1CS转化为**QAP**（Quadratic Arithmetic Program）
4. 使用多项式承诺和双线性配对生成简洁证明

**特点**：
- 证明体积仅几百字节，验证仅需几毫秒
- 需要可信设置（Trusted Setup）生成CRS
- 基于双线性配对（椭圆曲线上的特殊运算）

### 1.5 应用

- **Zcash**：使用zk-SNARKs隐藏交易的发送方、接收方和金额
- **身份验证**：证明自己满足某些条件（如年龄>18岁）而不泄露具体信息
- **区块链扩容**：zk-Rollup将大量交易压缩为单个证明
- **投票系统**：证明投票合法性而不泄露投票内容

## 2. 同态加密

同态加密允许在密文上直接执行运算，结果解密后与在明文上执行相同运算的结果一致。

### 2.1 部分同态加密

**加法同态**：D(E(m₁) ⊕ E(m₂)) = m₁ + m₂
- Paillier加密方案

**乘法同态**：D(E(m₁) ⊗ E(m₂)) = m₁ × m₂
- 原始RSA：E(m₁) × E(m₂) = m₁ᵉ × m₂ᵉ = (m₁·m₂)ᵉ = E(m₁·m₂)
- ElGamal加密

**部分同态加密**只能支持一种运算（加法或乘法），不能同时支持两者。

### 2.2 全同态加密

全同态加密（Fully Homomorphic Encryption, FHE）同时支持任意加法和乘法运算，理论上可计算任意函数。

**Gentry方案的思想**（2009年）：
1. 构造一个**浅层同态加密方案**（支持有限次运算）
2. 每次运算会产生"噪声"，噪声增长到一定程度会破坏正确性
3. **自举**（Bootstrapping）：用加密方案自身对密钥进行加密，然后在密文上"解密"自己的密文，从而刷新噪声
4. 经过自举后，可继续运算，实现无限深度计算

**现代FHE方案**：
- **BGV/BFV**：基于格的方案，支持整数运算
- **CKKS**：支持近似算术运算（适合机器学习推理）
- **TFHE**：基于布尔电路，支持逐比特运算

**主要挑战**：计算开销大（比明文运算慢数万倍到数十万倍），密文膨胀（明文扩展为数千倍的密文）。

### 2.3 应用

- **隐私计算**：云端处理加密数据而不获取明文
- **外包计算**：委托第三方计算敏感数据
- **联邦学习**：多方协作训练模型而不共享原始数据
- **医疗数据分析**：医院间联合分析而不泄露患者隐私

## 3. 秘密共享

### 3.1 Shamir秘密共享方案

Shamir秘密共享（Shamir's Secret Sharing）由Adi Shamir于1979年提出，基于多项式插值。

**核心思想**：
- 将秘密 S 编码为一个 t-1 次多项式 f(x) 的常数项：f(0) = S
- 给 n 个参与者每人分发一个点 (xᵢ, f(xᵢ))
- 任意 t 个点可以唯一确定多项式（拉格朗日插值），从而恢复 S
- 任意少于 t 个点无法获取关于 S 的任何信息

**构造**：
```
f(x) = S + a₁x + a₂x² + ... + a_(t-1)x^(t-1)  (mod p)
```
其中 a₁, ..., a_(t-1) 为随机系数，p 为大素数。

### 3.2 门限方案

**门限方案**（(t, n)-threshold scheme）：
- n 个参与者持有秘密份额
- 任意 t 个参与者可以恢复秘密
- 少于 t 个参与者得不到任何信息

**应用**：
- 密钥备份：防止密钥丢失（如企业CA根密钥）
- 多方授权：需要多个管理员共同同意才能执行敏感操作
- 安全投票：多方计票

## 4. 安全多方计算（MPC）基础

安全多方计算（Secure Multi-Party Computation）允许多个参与方联合计算一个函数，各方仅获得自己的输出，而不会泄露各自的私密输入。

**问题设定**：n 个参与方 P₁, ..., Pₙ 各有私密输入 x₁, ..., xₙ，希望共同计算 f(x₁, ..., xₙ)，且每个参与方只获得自己的输出。

**核心方法**：
- **不经意传输**（Oblivious Transfer, OT）：发送方有两条消息，接收方选择其中一条，发送方不知道接收方选择了哪条
- **秘密共享**：将数据拆分为份额，在份额上执行计算
- **混淆电路**（Garbled Circuit）：将计算转化为布尔电路，对电路进行加密（混淆）

**安全性模型**：
- **半诚实模型**（Semi-honest）：参与方诚实执行协议但可能尝试从通信中推断他人信息
- **恶意模型**（Malicious）：参与方可能任意偏离协议

**应用**：
- 隐私集合求交（PSI）：两方找出共同的数据元素而不暴露各自的完整数据
- 联合统计：多个机构在不共享原始数据的情况下联合计算统计量
- 电子投票：多方计票而不暴露单个选票
- 匿名拍卖

## 5. 格密码学基础

格密码学（Lattice-based Cryptography）是后量子密码学的主要方向之一。

### 5.1 LWE问题

LWE（Learning With Errors）由Oded Regev于2005年提出。

**定义**：给定矩阵 A ∈ Z_q^(m×n) 和向量 b = A·s + e (mod q)，其中 s 是秘密向量，e 是小误差向量，求解 s。

**困难性**：LWE问题可归约到格上最坏情况困难问题（如GapSVP、SIVP），具有很强的理论安全保障。

**RLWE（Ring-LWE）**：在多项式环上定义LWE，利用环结构提高效率。

### 5.2 NTRU

NTRU是1996年提出的基于格的公钥加密方案：
- 基于多项式环上的卷积运算
- 加解密速度快（比RSA快一个数量级）
- 密钥和密文尺寸小
- 历经多年密码分析，未发现有效攻击

**基本操作**：
- 密钥生成：在多项式环中生成私钥多项式 f、g，公钥 h = f⁻¹ · g
- 加密：用公钥和随机多项式对消息加密
- 解密：用私钥多项式恢复消息

## 6. 区块链密码学

### 6.1 哈希链与Merkle树

**哈希链**：
```
Block_n: [交易数据, 前一区块哈希 H(Block_(n-1)), Nonce, 时间戳]
```
每个区块包含前一个区块的哈希值，形成不可篡改的链。

**Merkle树**（Hash Tree）：
- 将交易两两哈希，逐层向上，最终得到根哈希（Merkle Root）
- 支持高效的存在性证明（SPV：仅需 log(n) 个哈希值即可验证某笔交易是否包含在区块中）

```
          Root Hash
         /         \
    H(AB)           H(CD)
   /     \         /     \
 H(A)   H(B)    H(C)   H(D)
  |       |       |       |
 Tx1    Tx2     Tx3     Tx4
```

### 6.2 工作量证明（PoW）

PoW要求矿工找到一个Nonce，使得区块头的哈希值小于目标值：

```
H(区块头 || Nonce) < Target
```

- 目标值越小，难度越大
- 调整难度保证出块速率稳定（如比特币约10分钟一个块）
- 安全性：篡改区块需要重新计算所有后续区块的PoW（51%攻击门槛）

### 6.3 数字签名在交易验证中的作用

每笔区块链交易需要发送方的数字签名：
1. 发送方用私钥对交易数据签名
2. 矿工/验证者用发送方的公钥验证签名
3. 验证通过后，交易才被纳入区块

**比特币**：使用ECDSA（secp256k1曲线）
**以太坊**：同样使用ECDSA，但地址由公钥经Keccak-256哈希生成

签名确保了：
- **认证性**：只有私钥持有者才能发起交易
- **完整性**：交易数据一旦签名不可更改
- **不可否认性**：发送方无法否认已签名的交易
