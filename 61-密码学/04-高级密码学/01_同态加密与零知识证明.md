# 同态加密与零知识证明


## 同态加密与零知识证明


密码学FHEZKP


同态加密允许在密文上计算，零知识证明允许证明而不泄露信息。


## 同态加密 (Homomorphic Encryption)


```
同态加密：在加密数据上直接计算，结果解密后等价于明文计算

分类：
┌─────────────────────────────────────────────┐
│  部分同态加密 (PHE)                          │
│  - 支持单一运算（加法或乘法）                 │
│  - RSA：E(m1) × E(m2) = E(m1 × m2)（乘法） │
│  - Paillier：E(m1) × E(m2) = E(m1+m2)（加法）│
│  - 效率高，实用性强                          │
│                                             │
│  些许同态加密 (SHE)                          │
│  - 支持有限次加法和乘法                       │
│  - BGV/BFV方案                              │
│                                             │
│  全同态加密 (FHE)                            │
│  - 支持任意次加法和乘法                       │
│  - 可以计算任意函数                           │
│  - Craig Gentry 2009年首次实现               │
│  - 性能极差：比明文计算慢10^3~10^6倍          │
│  - 方案：BGV, BFV, CKKS, TFHE               │
└─────────────────────────────────────────────┘

应用场景：
- 云端计算隐私数据（医疗、金融）
- 多方安全计算
- 机器学习中的隐私保护推理
- 区块链隐私交易

FHE挑战：
- 计算开销极大
- 密文膨胀（密文远大于明文）
- 噪声管理（每次运算增加噪声）
- Bootstrapping：重置噪声，但非常昂贵
```


## 零知识证明 (Zero-Knowledge Proof)


```
零知识证明：证明者能在不泄露任何额外信息的情况下，向验证者证明某个陈述为真

三个性质：
1. 完备性 (Completeness)：
   如果陈述为真，诚实的验证者会被说服
2. 可靠性 (Soundness)：
   如果陈述为假，欺骗者无法说服验证者
3. 零知识性 (Zero-Knowledge)：
   验证者除了知道陈述为真，得不到任何其他信息

经典例子 - 阿里巴巴洞穴：
- 洞穴有两个入口A和B，中间有道只有知道咒语才能打开的门
- 证明者：在验证者看不到时随机选择A或B进入
- 验证者：随机喊A或B，证明者从指定入口出来
- 重复多次，每次正确的概率=1/2
- 重复n次，欺骗概率=1/2^n → 趋近于0
- 验证者始终不知道咒语是什么

ZK-SNARKs (简洁非交互式零知识证明)：
- Succinct：证明很小（几百字节）
- Non-interactive：不需要交互，一次证明
- Arguments：计算可靠性（非信息论可靠性）
- of Knowledge：证明者确实知道见证
- 基于椭圆曲线配对和QAP/R1CS
- 应用：Zcash、以太坊隐私交易

ZK-STARKs：
- 可扩展(Scalable)：证明者时间亚线性
- 透明(Transparent)：无可信设置
- 抗量子：基于哈希而非椭圆曲线
- 证明体积比SNARKs大
```


## 安全多方计算 (MPC)


```
安全多方计算：多方在不暴露各自输入的情况下联合计算一个函数

示例：薪资比较
- Alice 和 Bob 想知道谁的工资更高
- 但都不想透露自己的具体数字
- MPC协议可以在不泄露输入的情况下完成比较

技术方案：
┌─────────────────────────────────────────────┐
│  1. 秘密分享 (Secret Sharing)                │
│     将秘密分成多份，任意k份可恢复              │
│     少于k份得不到任何信息                      │
│                                             │
│  2. 混淆电路 (Garbled Circuit)               │
│     将计算电路加密，双方依次求值               │
│     姚期智1986年提出                         │
│                                             │
│  3. 同态加密方案                              │
│     利用同态性质在密文上计算                   │
│                                             │
│  4. OT (不经意传输)                          │
│     接收方从多个消息中选一个                   │
│     发送方不知道选了哪个                       │
└─────────────────────────────────────────────┘

应用：
- 隐私集合求交(PSI)：找出双方数据的交集
- 联合机器学习：多方联合训练模型
- 电子投票：保护选票隐私
- 区块链隐私：门限签名、隐私交易
```


> **Note:** TEE(可信执行环境，如Intel SGX)是MPC的硬件替代方案，性能更好但需信任硬件厂商。


## 全同态加密方案详解


```
CKKS 方案（Cheon-Kim-Kim-Song, 2017）：
┌─────────────────────────────────────────────────────────┐
│  特点：支持近似算术运算（适用于机器学习）                  │
│  明文：复数向量（浮点数近似）                             │
│  安全性：基于 Ring-LWE 问题                               │
│                                                         │
│  核心操作：                                              │
│  - 编码：将实数向量编码为环上的多项式                     │
│  - 加密：添加噪声保证安全性                               │
│  - 同态加法：密文相加，噪声略有增加                       │
│  - 同态乘法：密文相乘，噪声显著增加                       │
│  - 重缩放 (Rescaling)：缩小密文以控制噪声                │
│                                                         │
│  应用：                                                   │
│  - 加密机器学习推理：在加密数据上运行神经网络             │
│  - 加密统计分析：计算加密数据的均值、方差                │
│  - 隐私保护云计算：将数据加密后外包计算                   │
└─────────────────────────────────────────────────────────┘

TFHE 方案（Torus FHE）：
┌─────────────────────────────────────────────────────────┐
│  特点：支持精确布尔运算，引导速度快                        │
│  基于：Ring-GGW / LWE 问题                               │
│                                                         │
│  核心操作：                                              │
│  - 支持任意布尔电路（NAND, XOR 等）                       │
│  - 每次布尔运算后执行引导（Bootstrapping）                │
│  - 引导时间约 10-15 毫秒（每次）                          │
│                                                         │
│  应用：                                                   │
│  - 加密数据库查询：在加密数据库上执行 SQL                 │
│  - 加密布尔电路：任意逻辑函数的加密计算                   │
│  - 隐私集合交集：多方安全计算的底层工具                   │
└─────────────────────────────────────────────────────────┘

FHE 性能现状（2024）：
┌─────────────────────────────────────────────────────────┐
│  CKKS 同态乘法：比明文慢 ~1000-10000 倍                   │
│  TFHE 引导：~10-15 ms / 次                               │
│  密文膨胀：~10-20 倍（明文 1KB → 密文 10-20KB）          │
│                                                         │
│  优化方向：                                               │
│  - GPU 加速：NVIDIA cuFHE，性能提升 10-50 倍             │
│  - 硬件加速：专用 FPGA / ASIC                            │
│  - 方案优化：更紧凑的参数，更快的引导                     │
│  - Microsoft SEAL, OpenFHE, TFHE-rs 等开源库             │
└─────────────────────────────────────────────────────────┘
```


## 零知识证明电路与应用


```
ZK 电路系统：
┌─────────────────────────────────────────────────────────┐
│  R1CS (Rank-1 Constraint System)：                       │
│  将计算表示为线性约束：(A·s) × (B·s) = C·s              │
│  s = [1, 公开输入, 私有见证, 中间值]                     │
│  每个约束 = 一次乘法门                                    │
│  SNARKs 的标准输入格式                                    │
│                                                         │
│  PLONK（2019）：                                          │
│  - 通用可信设置（只需一次）                               │
│  - 门约束 + 置换约束                                      │
│  - 证明更小，验证更快                                     │
│  - 以太坊 zk-rollups（zkSync, Polygon zkEVM）使用       │
│                                                         │
│  Halo2（Zcash）：                                         │
│  - 无可信设置（基于 IPA 承诺）                            │
│  - 递归证明（证明的证明）                                 │
│  - 用于 Zcash Orchard 交易                               │
└─────────────────────────────────────────────────────────┘

ZK 在区块链中的应用：
┌─────────────────────────────────────────────────────────┐
│  Zcash：                                                 │
│  - 使用 Groth16 (zk-SNARK)                               │
│  - 隐藏交易发送方、接收方、金额                           │
│  - 可选隐私（透明+屏蔽地址）                              │
│                                                         │
│  Ethereum zk-Rollups：                                    │
│  - zkSync, StarkNet, Polygon zkEVM, Scroll              │
│  - 将大量交易压缩为单个 ZK 证明                           │
│  - 主链只需验证证明（O(1) 成本）                          │
│  - L2 扩容的核心技术                                     │
│                                                         │
│  身份证明：                                               │
│  - 证明年龄 > 18 而不泄露出生日期                        │
│  - 证明在白名单中而不泄露身份                            │
│  - 世界币 (Worldcoin)：ZK 证明人类身份                   │
│                                                         │
│  ZK 桥：                                                 │
│  - 用 ZK 证明源链的状态转换                               │
│  - 目标链只需验证证明                                     │
│  - 无需信任第三方验证者                                   │
└─────────────────────────────────────────────────────────┘
```


> **Note:** ZK-SNARKs 证明大小仅几百字节但需要可信设置，ZK-STARKs 无需可信设置且抗量子但证明更大。PLONK 和 Halo2 是当前最活跃的 ZK 证明系统，zk-rollups 是以太坊扩容的核心方案。
