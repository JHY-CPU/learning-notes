# 02-非对称加密

## 1. 非对称加密的概念与动机

对称加密要求通信双方事先共享密钥，但**密钥分发**本身是个难题：在没有安全信道的情况下，如何把密钥安全地传给对方？

**非对称加密**（公钥密码学）由Diffie和Hellman于1976年提出，使用**一对密钥**：
- **公钥**（Public Key）：公开发布，任何人可用它加密消息或验证签名
- **私钥**（Private Key）：仅所有者持有，用于解密或签名

```
加密：C = E_公钥(P)
解密：P = D_私钥(C)
```

**核心思想**：基于某些数学上的**单向陷门函数**——正向计算容易，逆向计算困难，除非拥有陷门信息（私钥）。

---

## 2. 数论基础

### 2.1 模运算

模运算（Modular Arithmetic）是公钥密码学的基础。

- **同余**：a ≡ b (mod n) 表示 a 和 b 除以 n 的余数相同
- **模加法**：(a + b) mod n
- **模乘法**：(a × b) mod n
- **模幂**：a^b mod n（RSA和DH的核心运算）

**模逆元**：对于 a 和模数 n，若存在 b 使得 a·b ≡ 1 (mod n)，则 b 为 a 的模逆元。可通过扩展欧几里得算法求解。

### 2.2 欧拉函数

欧拉函数 φ(n) 定义为 1 到 n-1 中与 n 互质的正整数个数。

- 若 p 为素数：φ(p) = p - 1
- 若 n = p × q（p、q 为不同素数）：φ(n) = (p-1)(q-1)

### 2.3 费马小定理

若 p 为素数，a 不是 p 的倍数，则：
```
a^(p-1) ≡ 1 (mod p)
```

推广：**欧拉定理**——若 gcd(a, n) = 1，则：
```
a^φ(n) ≡ 1 (mod n)
```

### 2.4 中国剩余定理（CRT）

给定一组同余方程：
```
x ≡ a1 (mod m1)
x ≡ a2 (mod m2)
...
```
若 m1, m2, ... 两两互质，则存在唯一解 x（模 M = m1×m2×...）。

CRT可用于加速RSA解密：分别对 p 和 q 做模幂运算，再合并结果，可提速约4倍。

## 3. RSA算法

RSA由Rivest、Shamir和Adleman于1977年发明，是最著名的公钥算法。

### 3.1 密钥生成过程

1. 选择两个大素数 p 和 q（通常各1024位以上）
2. 计算 n = p × q（RSA模数）
3. 计算 φ(n) = (p-1)(q-1)
4. 选择公钥指数 e，满足 1 < e < φ(n) 且 gcd(e, φ(n)) = 1
   - 常用值：e = 65537 (2¹⁶+1)，兼顾安全性和计算效率
5. 计算私钥指数 d，使得 e·d ≡ 1 (mod φ(n))，即 d = e⁻¹ mod φ(n)
6. **公钥**：(n, e)；**私钥**：(n, d)（或含 p, q）

### 3.2 加密与解密

```
加密：C = M^e mod n
解密：M = C^d mod n
```

**正确性证明**（基于欧拉定理）：
```
C^d = (M^e)^d = M^(ed) ≡ M^(1 + kφ(n)) ≡ M · (M^φ(n))^k ≡ M · 1^k ≡ M (mod n)
```

### 3.3 安全性分析

RSA的安全性基于**大整数分解问题**：已知 n = p × q，从 n 推导出 p 和 q 在计算上是不可行的（当 n 足够大时）。

- 如果能分解 n，则可以计算 φ(n)，进而由 e 推出 d
- 当前推荐RSA密钥长度：2048位或以上
- 最佳已知攻击：通用数域筛法（GNFS），复杂度为亚指数级
- 量子计算机上的Shor算法可在多项式时间内分解大整数，威胁RSA

### 3.4 RSA签名

签名 = 用私钥"加密"，验证 = 用公钥"解密"：

```
签名：S = M^d mod n
验证：M = S^e mod n
```

实践中先对消息哈希，再对哈希值签名（避免消息长度限制，并加入随机化填充如PSS）。

## 4. Diffie-Hellman密钥交换

DH密钥交换由Diffie和Hellman于1976年提出，允许双方在不安全信道上协商出共享密钥。

### 4.1 离散对数问题

给定素数 p、生成元 g 和 y = g^x mod p，由 y 求 x 称为**离散对数问题**（DLP）。当 p 足够大时，DLP在计算上是困难的。

### 4.2 DH协议流程

公开参数：大素数 p，生成元 g

```
Alice                               Bob
------                              -----
选择私钥 a                           选择私钥 b
计算 A = g^a mod p                   计算 B = g^b mod p
发送 A  -------------------------->  发送 B
                                    <--------------------------
计算共享密钥 K = B^a mod p           计算共享密钥 K = A^b mod p

K = (g^b)^a = g^(ab) = (g^a)^b
```

### 4.3 中间人攻击（MITM）

DH协议本身不认证通信方身份，攻击者可以拦截双方通信：

```
Alice <--[K1]-- Mallory --[K2]--> Bob
```

Mallory分别与Alice和Bob建立密钥，截获并解密所有通信。

**防御**：结合数字签名或证书进行身份认证（如TLS中的DH认证版本DHE）。

---

## 5. ElGamal加密与签名

ElGamal基于离散对数问题，由Taher ElGamal于1985年提出。

### 加密过程
- 公钥：(p, g, y)，其中 y = g^x mod p，x 为私钥
- 加密时选择随机数 k，计算：
  ```
  c1 = g^k mod p
  c2 = m · y^k mod p
  密文 = (c1, c2)
  ```
- 解密：m = c2 · c1^(-x) mod p

**特点**：密文是明文的两倍长度（概率加密，密文膨胀）；加密时需要随机数k，同一明文可产生不同密文。

## 6. 椭圆曲线密码学（ECC）

### 6.1 椭圆曲线上的运算

椭圆曲线方程（简化Weierstrass形式）：
```
y² = x³ + ax + b
```
满足判别式 4a³ + 27b² ≠ 0（无奇点）。

定义在有限域 GF(p) 上（p 为大素数）。

**点加法**：曲线上的点构成一个阿贝尔群：
- 给定 P、Q，P + Q 的几何含义：过 P、Q 的直线与曲线交于第三点，取其关于x轴的对称点
- **倍点**：P + P = 2P，用切线求交点
- **标量乘法**：kP = P + P + ... + P（k 次）

### 6.2 ECDH密钥交换

将DH中的群运算从模幂替换为椭圆曲线标量乘法：

```
Alice：私钥 a，公钥 A = aG
Bob：  私钥 b，公钥 B = bG
共享密钥：K = aB = bA = abG
```

安全基础：**椭圆曲线离散对数问题**（ECDLP）——给定 P、Q 和 Q = kP，求 k 是困难的。

### 6.3 ECDSA签名

ECDSA（Elliptic Curve Digital Signature Algorithm）是DSA的椭圆曲线版本。

**签名过程**：
1. 选择随机数 k，计算 (x1, y1) = kG
2. r = x1 mod n
3. s = k⁻¹(hash + r·私钥) mod n
4. 签名为 (r, s)

**验证过程**：用公钥、消息哈希和签名 (r, s) 进行验证。

### 6.4 为什么ECC可用更短密钥达到同等安全

| 安全级别 | RSA密钥长度 | ECC密钥长度 |
|----------|-------------|-------------|
| 80位 | 1024位 | 160位 |
| 128位 | 3072位 | 256位 |
| 192位 | 7680位 | 384位 |
| 256位 | 15360位 | 512位 |

ECC基于**指数级难度**的ECDLP，而RSA基于**亚指数级难度**的整数分解。因此ECC在同等安全级别下密钥更短、计算更快、带宽消耗更低，特别适合资源受限环境（IoT、移动设备）。

## 7. 混合加密

实际系统中，**非对称加密不直接加密大量数据**（速度慢），而是采用混合加密：

```
1. 发送方生成随机对称密钥 K（会话密钥）
2. 用接收方的公钥加密 K：C_key = RSA_Encrypt(K, 公钥)
3. 用 K 加密数据：C_data = AES_Encrypt(明文, K)
4. 发送 (C_key, C_data)
5. 接收方用私钥解密 K，再用 K 解密数据
```

**TLS协议**就是混合加密的典型应用：握手阶段用RSA/ECDH交换密钥，数据传输阶段用AES-GCM加密。

---

## 8. 后量子密码学简介

量子计算机上的Shor算法可以多项式时间破解RSA、ECC和DH，因此需要**抗量子密码学**（Post-Quantum Cryptography, PQC）。

**主要方向**：格密码（CRYSTALS-Kyber, NTRU）、编码密码（McEliece）、哈希签名（SPHINCS+）。

**NIST后量子标准**（2024年）：CRYSTALS-Kyber（KEM）、CRYSTALS-Dilithium / FALCON（签名）、SPHINCS+（哈希签名）
