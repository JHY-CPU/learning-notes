非对称加密使用公钥/私钥对，解决了密钥分发问题。

RSA算法

RSA 基于大整数分解的困难性 密钥生成： 1. 选择两个大素数 p, q（各1024位以上） 2. 计算 n = p × q 3. 计算 φ(n) = (p-1)(q-1)（欧拉函数） 4. 选择 e，满足 1 < e < φ(n) 且 gcd(e, φ(n)) = 1 （常用 e = 65537 = 2^16+1） 5. 计算 d = e^(-1) mod φ(n)（e的模逆元） 公钥：(n, e) 私钥：(n, d) 加密：C = M^e mod n 解密：M = C^d mod n 签名：S = H(M)^d mod n（用私钥签名） 验签：H(M) = S^e mod n（用公钥验签） 安全性基础： - 已知 n 和 e，求 d 需要分解 n = p × q - 大整数分解是计算困难问题 - 当前2048位RSA密钥被认为安全 - 量子计算机(Shor算法)可破解RSA

椭圆曲线密码学 (ECC)

ECC 基于椭圆曲线离散对数问题(ECDLP)的困难性 椭圆曲线方程：y² = x³ + ax + b（在有限域上） 曲线上的点加法： - P + Q = R（几何上：过P、Q的直线与曲线的第三个交点关于x轴的对称点） - 2P = P + P（切线方式） 标量乘法：Q = k × P（点P自加k次） - 已知 k 和 P，容易计算 Q - 已知 Q 和 P，求 k 非常困难 → ECDLP ECC密钥交换 (ECDH)： Alice: 私钥 a, 公钥 A = a × G Bob: 私钥 b, 公钥 B = b × G 共享密钥： Alice 计算：a × B = a × b × G Bob 计算：b × A = b × a × G → 两者得到相同的共享密钥！ ECC vs RSA 密钥长度对比： ┌──────────┬──────────┬──────────┐ │ 安全级别 │ RSA密钥 │ ECC密钥 │ ├──────────┼──────────┼──────────┤ │ 80位 │ 1024位 │ 160位 │ │ 128位 │ 3072位 │ 256位 │ │ 192位 │ 7680位 │ 384位 │ │ 256位 │ 15360位 │ 512位 │ └──────────┴──────────┴──────────┘ → ECC密钥更短，计算更快，更适合移动设备和IoT

Diffie-Hellman密钥交换

DH 密钥交换协议（1976年，最早的公钥密码学）： 公开参数：大素数 p，生成元 g Alice：私钥 a，公钥 g^a mod p Bob： 私钥 b，公钥 g^b mod p 交换过程： 1. Alice 发送 A = g^a mod p 给 Bob 2. Bob 发送 B = g^b mod p 给 Alice 3. Alice 计算：B^a mod p = g^(ab) mod p 4. Bob 计算： A^b mod p = g^(ab) mod p → 双方得到相同的共享密钥 g^(ab) mod p 安全性：窃听者只能看到 g^a 和 g^b，无法计算 g^(ab) （离散对数问题的困难性） DH vs RSA 密钥交换： - DH：只用于密钥交换，不用于加密/签名 - RSA：可用于加密、签名、密钥交换 - 现代协议(TLS 1.3)优先使用(ECDH)进行密钥交换 TLS 1.3 密钥交换流程： Client → Server: ClientHello + ECDH公钥 Server → Client: ServerHello + ECDH公钥 + 签名 双方通过ECDH计算共享密钥，开始对称加密通信
TLS 1.3 已移除RSA密钥交换，只支持(ECDH)，提供前向安全性(Forward Secrecy)。

实际应用场景

HTTPS (TLS) 中的密码学： ┌─────────────────────────────────────────────┐ │ 1. 密钥交换：ECDHE（前向安全） │ │ 2. 身份认证：RSA/ECDSA 签名证书 │ │ 3. 对称加密：AES-256-GCM │ │ 4. 完整性校验：SHA-256（HMAC/签名） │ └─────────────────────────────────────────────┘ SSH 中的密码学： - 密钥交换：ECDH / Curve25519 - 主机认证：RSA / Ed25519 签名 - 会话加密：ChaCha20-Poly1305 / AES-GCM 数字证书 (X.509)： - CA用自己的私钥签发证书 - 浏览器用CA的公钥验证证书 - 证书链：根CA → 中间CA → 服务器证书

RSA 数学原理详解

RSA 正确性证明（欧拉定理）： ┌─────────────────────────────────────────────────────────┐ │ 欧拉定理：若 gcd(a, n) = 1，则 a^φ(n) ≡ 1 (mod n) │ │ │ │ RSA 加密：C = M^e mod n │ │ RSA 解密：C^d mod n = M^(ed) mod n │ │ │ │ 由于 ed ≡ 1 (mod φ(n))： │ │ ed = 1 + k×φ(n) 对某个整数 k 成立 │ │ │ │ M^(ed) = M^(1+k×φ(n)) = M × (M^φ(n))^k │ │ 若 gcd(M, n) = 1，则 M^φ(n) ≡ 1 (mod n) │ │ → M^(ed) ≡ M × 1^k ≡ M (mod n) ✓ │ │ │ │ 注意：当 M 是 p 或 q 的倍数时需用中国剩余定理证明 │ └─────────────────────────────────────────────────────────┘ RSA 填充方案（实际使用中的关键）： ┌─────────────────────────────────────────────────────────┐ │ 原始 RSA（教科书 RSA）不安全： │ │ - 确定性加密：相同明文产生相同密文 │ │ - 小明文攻击：M^e < n 时可直接开方 │ │ - 共模攻击：同 n 不同 e 加密同一消息 │ │ │ │ PKCS#1 v1.5 填充（旧标准）： │ │ EM = 0x00 || 0x02 || PS || 0x00 || M │ │ PS = 随机填充（至少 8 字节，非零） │ │ 注意：存在 Bleichenbacher 攻击（选择密文攻击） │ │ │ │ OAEP (Optimal Asymmetric Encryption Padding)： │ │ 现代推荐填充方案（PKCS#1 v2.x） │ │ M → 填充 → MGF1 掩码 → 加密 │ │ 安全性：在随机预言模型下可证明 IND-CCA2 安全 │ │ │ │ PSS (Probabilistic Signature Scheme)： │ │ RSA 数字签名的推荐填充方案 │ │ 随机性使得相同消息的签名不同 │ └─────────────────────────────────────────────────────────┘ 中国剩余定理 (CRT) 优化 RSA 解密： ┌─────────────────────────────────────────────────────────┐ │ 标准解密：M = C^d mod n（需要模 n 的幂运算，n 很大） │ │ │ │ CRT 优化： │ │ 已知 n = p × q │ │ 计算 d_p = d mod (p-1)，d_q = d mod (q-1) │ │ 解密时： │ │ M_p = C^(d_p) mod p │ │ M_q = C^(d_q) mod q │ │ M = CRT(M_p, M_q, p, q) = (M_p × q × q⁻¹_mod_p │ │ + M_q × p × p⁻¹_mod_q) mod n │ │ │ │ 性能提升：约 4 倍加速 │ │ 因为 p 和 q 各为 n 的一半长度 │ │ 幂运算从 O(k³) 降到 2×O((k/2)³) = O(k³/4) │ │ │ │ 现代 RSA 实现（如 OpenSSL）默认使用 CRT 优化 │ └─────────────────────────────────────────────────────────┘
实际使用 RSA 时必须使用填充方案（OAEP 加密，PSS 签名），教科书 RSA 是不安全的。CRT 优化可将 RSA 解密速度提升约 4 倍。现代应用推荐使用 ECC 替代 RSA。
