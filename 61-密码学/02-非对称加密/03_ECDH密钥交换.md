# ECDH密钥交换


## ECDH 密钥交换


密码学ECDH密钥交换


椭圆曲线 Diffie-Hellman（ECDH）是基于椭圆曲线离散对数问题的密钥协商协议，是现代互联网安全通信的核心。


## 经典 DH 与 ECDH 对比


```
经典 Diffie-Hellman (DH)：
┌─────────────────────────────────────────────────────────┐
│  公开参数：大素数 p，生成元 g                              │
│  Alice：私钥 a，公钥 A = g^a mod p                       │
│  Bob：  私钥 b，公钥 B = g^b mod p                       │
│  共享密钥：S = g^(ab) mod p                              │
│  安全基础：离散对数问题 (DLP)                              │
│  密钥长度：2048-4096 位（安全强度 112-128 位）             │
└─────────────────────────────────────────────────────────┘

ECDH（椭圆曲线版本）：
┌─────────────────────────────────────────────────────────┐
│  公开参数：椭圆曲线 E，基点 G，阶 n                       │
│  Alice：私钥 a ∈ [1, n-1]，公钥 A = a × G               │
│  Bob：  私钥 b ∈ [1, n-1]，公钥 B = b × G               │
│  共享密钥：S = a × B = b × A = a × b × G               │
│  安全基础：椭圆曲线离散对数问题 (ECDLP)                    │
│  密钥长度：256-384 位（等效安全强度 128-192 位）           │
└─────────────────────────────────────────────────────────┘

关键优势：ECDH 用更短的密钥达到同等安全性
- DH-2048 ≈ ECDH-224（112 位安全强度）
- DH-3072 ≈ ECDH-256（128 位安全强度）
- 更短密钥 → 更快计算、更少带宽
```


## ECDH 密钥协商协议详解


```
ECDH 完整协议流程：
┌─────────────────────────────────────────────────────────┐
│  公共参数（公开）：                                       │
│    椭圆曲线 E: y² = x³ + ax + b over F_p               │
│    基点 G ∈ E(F_p)，阶 n                                │
│    余因子 h = |E(F_p)| / n                               │
│                                                         │
│  Alice                              Bob                 │
│  ┌──────────────┐                  ┌──────────────┐     │
│  │ 私钥 a       │                  │ 私钥 b       │     │
│  │ a ∈ [1,n-1]  │                  │ b ∈ [1,n-1]  │     │
│  │ 公钥 A=a×G   │                  │ 公钥 B=b×G   │     │
│  └──────┬───────┘                  └──────┬───────┘     │
│         │                                 │             │
│         │ ──── A = a×G ──────────────────→│             │
│         │←──── B = b×G ──────────────────│             │
│         │                                 │             │
│  共享密钥 S = a×B                   共享密钥 S = b×A    │
│           = a×(b×G)                        = b×(a×G)    │
│           = ab×G                            = ab×G      │
│  → 两者相等！                                           │
│                                                         │
│  窃听者看到：E, G, A, B                                 │
│  需要从 A=a×G 中求 a → ECDLP，计算不可行                │
└─────────────────────────────────────────────────────────┘

密钥派生 (KDF)：
共享点 S = (x_S, y_S) → KDF(x_S) → 会话密钥
- 不直接用点坐标作为密钥（格式不适合）
- 使用 HKDF 或 ANSI X9.63 KDF 派生对称密钥
- 输出：加密密钥 + MAC 密钥 + IV

余因子处理（安全性加强）：
- 理想曲线 h = 1（素数阶群）
- 实际曲线可能 h > 1（如 Curve25519 的 h = 8）
- 小子群攻击：攻击者发送阶为小因子的点
- 防御：验证收到的公钥点在正确阶的子群上
  或在私钥中乘以余因子
```


## 静态 DH、临时 DH 与 ECDHE


```
DH 密钥交换的三种变体：
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  1. 静态 DH (Static DH)                                 │
│     ────────────────────                                │
│     Alice 和 Bob 长期持有固定的密钥对                     │
│     每次通信使用相同的私钥/公钥                           │
│                                                         │
│     缺点：                                               │
│     - 无前向安全性：长期私钥泄露 → 所有历史会话被破解     │
│     - 无身份保护：公钥可追踪                             │
│     - 需要验证公钥真实性（通常用证书）                    │
│                                                         │
│  2. 临时 DH (Ephemeral DH)                              │
│     ────────────────────                                │
│     每次握手生成新的临时密钥对                            │
│     握手完成后丢弃临时私钥                                │
│                                                         │
│     优势：                                               │
│     - 前向安全性 (Forward Secrecy)                       │
│     - 即使长期私钥泄露，历史会话仍安全                    │
│     - 每次会话使用不同密钥                               │
│                                                         │
│  3. 临时-静态 DH (ECDH-ES)                              │
│     ────────────────────                                │
│     一方临时，一方静态                                    │
│     用于加密（如 JWE 中的 ECDH-ES）                      │
│     每次加密使用新的临时密钥，前向安全                     │
│                                                         │
└─────────────────────────────────────────────────────────┘

前向安全性 (Forward Secrecy) 详解：
┌─────────────────────────────────────────────────────────┐
│  场景：今天的安全通信，十年后的计算能力                    │
│                                                         │
│  无前向安全（静态 RSA 密钥交换）：                        │
│  - 攻击者录制今天的 TLS 流量                             │
│  - 十年后私钥泄露或被量子计算机破解                       │
│  - 攻击者解密所有录制的流量                               │
│                                                         │
│  有前向安全（ECDHE）：                                    │
│  - 攻击者录制今天的 TLS 流量                             │
│  - 十年后长期私钥泄露                                    │
│  - 临时私钥已被丢弃，无法解密历史流量                     │
│  - 历史会话仍然安全！                                    │
│                                                         │
│  TLS 1.3 强制使用 ECDHE：                                │
│  - 移除了 RSA 密钥交换（无前向安全）                      │
│  - 移除了静态 DH（无前向安全）                            │
│  - 所有密码套件都支持前向安全                             │
└─────────────────────────────────────────────────────────┘
```


> **Note:** TLS 1.3 强制使用临时 ECDHE（Ephemeral ECDH），确保所有连接都具有前向安全性。即使服务器长期私钥未来被破解，过去的通信仍然安全。


## ECDH 常用椭圆曲线


```
TLS/SSH 中常用的椭圆曲线：
┌─────────────────────────────────────────────────────────┐
│  Curve25519 / X25519 (最常用)                            │
│  ─────────────────────────                              │
│  设计者：Daniel Bernstein (2006)                         │
│  基于：Montgomery 曲线 y² = x³ + 486662x² + x           │
│  域：F_p, p = 2²⁵⁵ - 19 (Mersenne 素数)                │
│  安全强度：128 位                                        │
│  特点：                                                   │
│    - 常数时间实现容易（防侧信道攻击）                      │
│    - 无需验证公钥点（x 坐标即可）                         │
│    - 速度极快（约 30 万次/秒 / 单核）                     │
│    - SSH、WireGuard、Noise 协议首选                      │
│                                                         │
│  Ed25519（签名用，X25519 的姐妹曲线）                    │
│  基于：Edwards 曲线 -x² + y² = 1 + dx²y²               │
│  用途：EdDSA 数字签名（不是密钥交换）                     │
│                                                         │
│  NIST P-256 (secp256r1)                                 │
│  ────────────────────                                   │
│  标准：NIST SP 800-56A                                  │
│  安全强度：128 位                                        │
│  特点：                                                   │
│    - TLS 最广泛使用的曲线                                │
│    - 与 SM2 曲线结构类似                                 │
│    - 部分参数来源不明引发信任问题                         │
│                                                         │
│  NIST P-384 (secp384r1)                                 │
│  ────────────────────                                   │
│  安全强度：192 位                                        │
│  用于：高安全级别场景、政府系统                           │
│                                                         │
│  Brainpool P-256r1                                      │
│  ─────────────────                                      │
│  设计者：德国 BSI                                        │
│  特点：参数生成过程完全透明                               │
│  用于：欧洲政府、金融系统                                │
└─────────────────────────────────────────────────────────┘

Curve25519 密钥交换实现（示意）：
function X25519(scalar, u_coordinate):
    // Montgomery ladder 标量乘法
    // 输入：标量 k（私钥，256 位）
    //       u 坐标（对方公钥的 x 坐标）
    // 输出：共享密钥的 u 坐标
    return k × point(u_coordinate)

私钥处理：
    k[0] &= 248   // 清除最低 3 位（余因子为 8）
    k[31] &= 127  // 清除最高位
    k[31] |= 64   // 设置第 6 位（确保在 [2²⁵⁴, 2²⁵⁵) 范围内）

曲线对比：
┌──────────┬──────────┬──────────┬──────────┬──────────┐
│ 曲线      │ 密钥长度  │ 安全强度  │ 速度      │ 用途      │
├──────────┼──────────┼──────────┼──────────┼──────────┤
│ X25519   │ 256位    │ 128位    │ 最快      │ 密钥交换  │
│ P-256    │ 256位    │ 128位    │ 快        │ TLS通用  │
│ P-384    │ 384位    │ 192位    │ 中等      │ 高安全级  │
│ P-521    │ 521位    │ 256位    │ 较慢      │ 极高安全  │
│ secp256k1│ 256位    │ 128位    │ 快        │ 比特币    │
└──────────┴──────────┴──────────┴──────────┴──────────┘
```


## ECDH 在实际协议中的应用


```
TLS 1.3 中的 ECDHE（1-RTT 握手）：
┌─────────────────────────────────────────────────────────┐
│  Client                                     Server     │
│  ┌────────────────────────────┐                        │
│  │ ClientHello                │                        │
│  │   key_share: {X25519: A}   │ ──────────────→        │
│  │   supported_groups: [X25519]                        │
│  └────────────────────────────┘                        │
│                                                        │
│  ┌────────────────────────────┐                        │
│  │ ServerHello                │                        │
│  │   key_share: {X25519: B}   │ ←──────────────        │
│  │ EncryptedExtensions        │                        │
│  │ Certificate                │                        │
│  │ CertificateVerify          │                        │
│  │ Finished                   │                        │
│  └────────────────────────────┘                        │
│                                                        │
│  Shared Secret = X25519(a, B) = X25519(b, A)          │
│  → HKDF-Expand → 会话密钥（加密 + MAC）                  │
│                                                        │
│  握手只需 1 个往返（1-RTT），0-RTT 模式下更快           │
└─────────────────────────────────────────────────────────┘

SSH 中的 ECDH：
┌─────────────────────────────────────────────────────────┐
│  密钥交换算法：                                           │
│  - curve25519-sha256（首选）                             │
│  - ecdh-sha2-nistp256                                   │
│  - ecdh-sha2-nistp384                                   │
│                                                         │
│  流程：                                                  │
│  1. 双方交换 ECDH 公钥                                  │
│  2. 计算共享密钥                                        │
│  3. 用共享密钥 + 会话参数派生会话密钥                     │
│  4. 交换 NEWKEYS 确认，开始加密通信                      │
└─────────────────────────────────────────────────────────┘

Noise 协议框架（WireGuard 使用）：
┌─────────────────────────────────────────────────────────┐
│  Noise_XX 握手（双向认证）：                              │
│  → e                                                      │
│  ← e, ee, s, es                                         │
│  → s, se                                                │
│                                                         │
│  其中：e = 临时 ECDH，s = 静态 ECDH                      │
│  ee = 临时-临时，es = 临时-静态                          │
│  最终密钥由所有 ECDH 结果混合派生                         │
│                                                         │
│  WireGuard 使用 Noise_IKpsk2：                           │
│  - 服务器公钥预配置在客户端                               │
│  - 2-RTT 完整握手                                        │
│  - 极简设计，仅约 4000 行代码                             │
└─────────────────────────────────────────────────────────┘
```


## ECDH 安全注意事项


```
ECDH 实现中的安全陷阱：
┌─────────────────────────────────────────────────────────┐
│  1. 公钥验证 (Key Validation)                            │
│     ─────────────────────────                           │
│     攻击者可能发送无效点（小阶点）来泄露私钥信息           │
│     防御：                                               │
│     - 验证点在曲线上（曲线方程检查）                      │
│     - 验证点的阶正确（乘以 n 后得到无穷远点）             │
│     - 使用安全曲线（h=1 时不需要阶验证）                  │
│     - X25519 设计上避免了大部分验证问题                   │
│                                                         │
│  2. 侧信道攻击 (Side-Channel Attacks)                    │
│     ─────────────────────────                           │
│     - 定时攻击：标量乘法时间泄露密钥位                    │
│     - 功耗分析：SPA/DPA 通过功耗曲线推断密钥              │
│     - 电磁辐射：EM 泄露计算过程                          │
│     防御：                                               │
│     - 常数时间实现（Montgomery ladder）                   │
│     - 标量随机化（blinding）                              │
│     - Curve25519 天然支持常数时间实现                     │
│                                                         │
│  3. 量子威胁 (Quantum Threat)                            │
│     ───────────────────────                             │
│     Shor 算法可在多项式时间解决 ECDLP                     │
│     后量子密钥交换：                                     │
│     - 混合方案：X25519 + Kyber768（过渡期）              │
│     - 纯后量子：CRYSTALS-Kyber（NIST 标准）              │
│     Signal 协议已部署 X25519+Kyber 混合密钥交换          │
│                                                         │
│  4. 曲线选择不当                                         │
│     ───────────────                                     │
│     - 避免使用有弱点的曲线（如某些自定义曲线）             │
│     - 优先选择 X25519 或 NIST P-256                     │
│     - 避免 secp256k1 用于非比特币场景                    │
└─────────────────────────────────────────────────────────┘

推荐配置：
┌─────────────────────────────────────────────────────────┐
│  Web/TLS：X25519（首选）或 P-256                         │
│  SSH：curve25519-sha256                                  │
│  VPN (WireGuard)：Curve25519                            │
│  IoT/嵌入式：X25519（计算量小）                          │
│  高安全：P-384 或 X448                                   │
│  后量子过渡：X25519 + Kyber768（混合）                   │
└─────────────────────────────────────────────────────────┘
```


> **Note:** 实际实现中应优先使用成熟库（如 libsodium、BoringSSL），避免自行实现椭圆曲线运算。Curve25519/X25519 是当前推荐的首选曲线。
