# 时序分析基础

## 建立时间与保持时间

时序分析是确保数字电路在目标时钟频率下正确工作的关键步骤。建立时间（Setup Time）和保持时间（Hold Time）是触发器最基本的时序参数。

- **建立时间（Tsu）**：在时钟有效沿到来之前，数据必须保持稳定的最短时间
- **保持时间（Th）**：在时钟有效沿到来之后，数据必须继续保持稳定的最短时间

```
          ┌──────────────────────┐
  CLK     │                      │
  ────────┘                      └──────────
              ↑ 时钟上升沿
          ┌──┐
  DATA    │  │←Tsu→│←Th→│
  ────────┘  └─────────────
```

时序路径分析的基本公式：

```
时钟周期 T = Tclk_q (触发器传播延迟) + Tlogic (组合逻辑延迟)
           + Troute (布线延迟) + Tsu (建立时间)

建立时间裕量 = Tclk - (Tclk_q + Tlogic + Troute + Tsu)
保持时间裕量 = Tclk_q + Tlogic + Troute - Th
```

```verilog
// 时序路径的基本结构
module timing_path (
    input  wire       clk,
    input  wire [7:0] din,
    output reg  [7:0] dout
);
    reg [7:0] stage1;
    wire [7:0] logic_out;

    // 第一级触发器（Tclk_q 延迟）
    always @(posedge clk) begin
        stage1 <= din;
    end

    // 组合逻辑路径（Tlogic 延迟）
    assign logic_out = stage1 ^ (stage1 >> 1) ^ (stage1 << 1);

    // 第二级触发器（需要满足 Tsu 要求）
    always @(posedge clk) begin
        dout <= logic_out;
    end
endmodule
```

## 时钟域交叉

当信号从一个时钟域传递到另一个时钟域时，就会发生时钟域交叉（Clock Domain Crossing, CDC）。由于两个时钟之间没有确定的相位关系，直接传递信号会导致亚稳态（metastability）问题。

### 单比特信号同步

单比特控制信号使用两级触发器同步器进行同步：

```verilog
// 两级同步器：解决单比特 CDC 问题
module sync_2ff (
    input  wire clk_dst,    // 目标时钟域
    input  wire rst_n,
    input  wire din_src,    // 源时钟域信号
    output wire dout_dst    // 同步后的信号
);
    reg sync_ff1, sync_ff2;

    always @(posedge clk_dst or negedge rst_n) begin
        if (!rst_n) begin
            sync_ff1 <= 1'b0;
            sync_ff2 <= 1'b0;
        end else begin
            sync_ff1 <= din_src;    // 第一级：可能进入亚稳态
            sync_ff2 <= sync_ff1;   // 第二级：大概率稳定
        end
    end

    assign dout_dst = sync_ff2;
endmodule
```

### 多比特信号同步

多比特信号不能直接用两级同步器，因为不同的位可能在不同时钟周期被采样。常用的方法是**格雷码**和**异步 FIFO**。

```verilog
// 格雷码编码器（用于跨时钟域传递计数器/指针）
module bin_to_gray #(
    parameter WIDTH = 4
) (
    input  wire [WIDTH-1:0] bin,
    output wire [WIDTH-1:0] gray
);
    assign gray = bin ^ (bin >> 1);
endmodule

// 格雷码解码器
module gray_to_bin #(
    parameter WIDTH = 4
) (
    input  wire [WIDTH-1:0] gray,
    output reg  [WIDTH-1:0] bin
);
    integer i;
    always @(*) begin
        bin[WIDTH-1] = gray[WIDTH-1];
        for (i = WIDTH-2; i >= 0; i = i - 1)
            bin[i] = bin[i+1] ^ gray[i];
    end
endmodule
```

### 异步 FIFO

异步 FIFO 是在不同时钟域之间传递数据的标准方法：

```verilog
module async_fifo #(
    parameter DATA_WIDTH = 8,
    parameter ADDR_WIDTH = 4   // FIFO 深度 = 2^ADDR_WIDTH
) (
    // 写端口（源时钟域）
    input  wire                   wr_clk,
    input  wire                   wr_rst_n,
    input  wire                   wr_en,
    input  wire [DATA_WIDTH-1:0]  wr_data,
    output wire                   full,
    // 读端口（目标时钟域）
    input  wire                   rd_clk,
    input  wire                   rd_rst_n,
    input  wire                   rd_en,
    output wire [DATA_WIDTH-1:0]  rd_data,
    output wire                   empty
);
    // 存储器
    reg [DATA_WIDTH-1:0] mem [0:(1<<ADDR_WIDTH)-1];

    // 写指针（二进制 + 格雷码）
    reg [ADDR_WIDTH:0] wr_ptr_bin, wr_ptr_gray;
    wire [ADDR_WIDTH:0] wr_ptr_bin_next = wr_ptr_bin + (wr_en & ~full);
    wire [ADDR_WIDTH:0] wr_ptr_gray_next = wr_ptr_bin_next ^ (wr_ptr_bin_next >> 1);

    // 读指针（二进制 + 格雷码）
    reg [ADDR_WIDTH:0] rd_ptr_bin, rd_ptr_gray;
    wire [ADDR_WIDTH:0] rd_ptr_bin_next = rd_ptr_bin + (rd_en & ~empty);
    wire [ADDR_WIDTH:0] rd_ptr_gray_next = rd_ptr_bin_next ^ (rd_ptr_bin_next >> 1);

    // 跨时钟域同步的读/写指针（格雷码）
    reg [ADDR_WIDTH:0] rd_ptr_gray_sync1, rd_ptr_gray_sync2;  // 同步到写时钟域
    reg [ADDR_WIDTH:0] wr_ptr_gray_sync1, wr_ptr_gray_sync2;  // 同步到读时钟域

    // 写端口逻辑
    always @(posedge wr_clk or negedge wr_rst_n) begin
        if (!wr_rst_n) begin
            wr_ptr_bin  <= 0;
            wr_ptr_gray <= 0;
        end else begin
            wr_ptr_bin  <= wr_ptr_bin_next;
            wr_ptr_gray <= wr_ptr_gray_next;
        end
    end

    // 写存储器
    always @(posedge wr_clk) begin
        if (wr_en && !full)
            mem[wr_ptr_bin[ADDR_WIDTH-1:0]] <= wr_data;
    end

    // 读端口逻辑
    always @(posedge rd_clk or negedge rd_rst_n) begin
        if (!rd_rst_n) begin
            rd_ptr_bin  <= 0;
            rd_ptr_gray <= 0;
        end else begin
            rd_ptr_bin  <= rd_ptr_bin_next;
            rd_ptr_gray <= rd_ptr_gray_next;
        end
    end

    // 同步读指针到写时钟域
    always @(posedge wr_clk or negedge wr_rst_n) begin
        if (!wr_rst_n) begin
            rd_ptr_gray_sync1 <= 0;
            rd_ptr_gray_sync2 <= 0;
        end else begin
            rd_ptr_gray_sync1 <= rd_ptr_gray;
            rd_ptr_gray_sync2 <= rd_ptr_gray_sync1;
        end
    end

    // 同步写指针到读时钟域
    always @(posedge rd_clk or negedge rd_rst_n) begin
        if (!rd_rst_n) begin
            wr_ptr_gray_sync1 <= 0;
            wr_ptr_gray_sync2 <= 0;
        end else begin
            wr_ptr_gray_sync1 <= wr_ptr_gray;
            wr_ptr_gray_sync2 <= wr_ptr_gray_sync1;
        end
    end

    // 满标志：比较写指针和同步后的读指针
    assign full = (wr_ptr_gray == {~rd_ptr_gray_sync2[ADDR_WIDTH:ADDR_WIDTH-1],
                                    rd_ptr_gray_sync2[ADDR_WIDTH-2:0]});

    // 空标志：比较读指针和同步后的写指针
    assign empty = (rd_ptr_gray == wr_ptr_gray_sync2);

    // 读数据
    assign rd_data = mem[rd_ptr_bin[ADDR_WIDTH-1:0]];
endmodule
```

## 亚稳态与同步失败概率

当触发器的建立时间或保持时间要求不满足时，触发器可能进入亚稳态，输出在 0 和 1 之间振荡一段时间后才稳定。

亚稳态的平均故障间隔时间（MTBF）计算公式：

```
MTBF = e^(Tmet / τ) / (fclock × fdata × T0)

其中：
- Tmet：可用的稳定时间（时钟周期减去其他延迟）
- τ：触发器的时间常数（与工艺相关）
- fclock：采样时钟频率
- fdata：数据变化频率
- T0：与工艺相关的常数
```

增加同步级数可以指数级提高 MTBF。两级同步器的 MTBF 通常已足够高（数十万年），三级同步器用于极高可靠性要求的场景。

时序分析要点：
1. 建立时间违反通过降低时钟频率或优化逻辑解决
2. 保持时间违反通常通过添加缓冲器（buffer）解决
3. CDC 信号必须经过正确的同步处理
4. 异步复位信号也需要考虑恢复时间（Recovery）和移除时间（Removal）
