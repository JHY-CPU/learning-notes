# FPGA 与可编程器件

第一部分：可编程器件概述

## 一、可编程逻辑器件发展史


可编程逻辑器件（Programmable Logic Device, PLD）允许用户通过编程来实现自定义的数字电路，无需定制芯片制造。


### 1.1 发展历程


| 器件 | 年代 | 特点 | 可编程性 |
| --- | --- | --- | --- |
| **PROM** | 1970s | 可编程只读存储器，与阵列固定 | 或阵列可编程 |
| **PLA** | 1970s | 可编程逻辑阵列，与阵列和或阵列均可编程 | 两者皆可 |
| **PAL** | 1978 | 可编程阵列逻辑，与阵列可编程，或阵列固定 | 与阵列可编程 |
| **GAL** | 1985 | 通用阵列逻辑，可擦除重编程 | 可重复编程 |
| **CPLD** | 1990s | 复杂可编程逻辑器件，多个宏单元 | 非易失性 |
| **FPGA** | 1985+ | 现场可编程门阵列，海量逻辑资源 | 易失性（SRAM） |

### 2.2 核心组件


| 组件 | 全称 | 功能 | 典型数量 |
| --- | --- | --- | --- |
| **CLB** | Configurable Logic Block | 实现组合/时序逻辑的基本单元 | 数千~数十万 |
| **IOB** | I/O Block | 芯片引脚的接口逻辑 | 数百~数千 |
| **BRAM** | Block RAM | 片上存储器块（双端口） | 数百~数千块 |
| **DSP** | DSP Slice | 硬件乘法器、累加器 | 数百~数千 |
| **PLL/MMCM** | Phase-Locked Loop | 时钟管理和频率合成 | 数个~数十个 |
| **互连矩阵** | Routing | 连接所有逻辑资源 | 全局资源 |

### 3.2 查找表（LUT）工作原理


LUT 本质上是一个小型的 SRAM，输入作为地址，输出为存储的内容。一个 N 输入的 LUT 需要 2^N 位存储。


```
6 输入 LUT（6-LUT）的结构：

                    6 位输入
                ┌──────────────┐
  I0 ─────────▶│              │
  I1 ─────────▶│   64×1 SRAM  │────────▶ O6（输出）
  I2 ─────────▶│   (查找表)    │
  I3 ─────────▶│              │
  I4 ─────────▶│              │
  I5 ─────────▶│              │
                └──────────────┘

示例：实现 F = I0 AND I1
SRAM 内容：
  地址 I5:I0    值
  000010 (2)     0
  000011 (3)     1   ← I0=1, I1=1 时输出 1
  其他地址        0

LUT 可以实现任意 6 输入布尔函数！
```


### 3.3 LUT 的模式


- **6 输入 LUT 模式**
   ：一个 6 输入布尔函数
- **双 5 输入 LUT 模式**
   ：两个共享输入的 5 输入函数
- **分布式 RAM 模式**
   ：作为小型 SRAM（64 位）
- **移位寄存器模式**
   ：32 级移位寄存器

第四部分：互连与布线

## 四、FPGA 互连与布线


### 4.1 互连资源类型


| 类型 | 长度 | 用途 |
| --- | --- | --- |
| **Local Lines** | 单个 CLB 内 | CLB 内部连接 |
| **Direct Connect** | 相邻 CLB | 高速相邻连接 |
| **Single Lines** | 1-2 个 CLB | 局部信号 |
| **Double Lines** | 2-4 个 CLB | 中距离信号 |
| **Hex Lines** | 6 个 CLB | 中长距离信号 |
| **Long Lines** | 整个芯片 | 全局信号（时钟、复位） |


### 4.2 开关矩阵（Switch Matrix）


```
开关矩阵实现交叉连接：

  来自上方的线 ──────┐
```

第五部分：设计流程

## 五、FPGA 设计流程（Synthesis Flow）


```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  HDL     │    │ 综合     │    │ 实现     │    │ 生成     │
│  源代码  │───▶│Synthesis │───▶│Implement │───▶│ 比特流   │
│(.v/.vhd) │    │          │    │          │    │(.bit)    │
└──────────┘    └────┬─────┘    └────┬─────┘    └──────────┘
                     │               │
                     ▼               ▼
              ┌──────────┐    ┌──────────┐
              │ 网表     │    │ P&R      │
              │ Netlist  │    │ 布局布线 │
              │(门级描述)│    │(位置+路径)│
              └──────────┘    └──────────┘
```


### 5.1 各阶段详解


| 阶段 | 输入 | 输出 | 关键任务 |
| --- | --- | --- | --- |
| **综合（Synthesis）** | HDL 代码 | 门级网表 | 逻辑优化、LUT 映射、资源共享 |
| **翻译（Translate）** | 网表 + 约束 | 统一数据库 | 合并多个网表文件 |
| **映射（Map）** | 统一数据库 | 映射后网表 | 将逻辑映射到 CLB/LUT/BRAM |
| **布局（Place）** | 映射后网表 | 位置分配 | 将逻辑单元分配到物理位置 |
| **布线（Route）** | 布局结果 | 完整设计 | 连接所有信号路径 |
| **比特流生成** | 布线结果 | .bit 文件 | 生成 FPGA 配置数据 |


### 5.2 设计约束


```
// XDC 约束文件示例（Xilinx Design Constraints）

// 时钟约束
create_clock -period 10 -name clk [get_ports clk]    // 100MHz 时钟
set_clock_uncertainty 0.5 [get_clocks clk]

// I/O 约束
set_property PACKAGE_PIN W5 [get_ports clk]           // 引脚分配
set_property IOSTANDARD LVCMOS33 [get_ports clk]      // 电平标准

// 时序例外
set_false_path -from [get_ports reset]                 // 异步复位无时序要求
set_max_delay 5 -from [get_pins reg1/Q] -to [get_pins reg2/D]

// 物理约束
set_property LOC SLICE_X0Y0 [get_cells my_ff]          // 位置约束
```

第六部分：FPGA vs ASIC

## 六、FPGA vs ASIC 对比


#### FPGA 优势


- 开发周期短（数周~数月）
- 可重复编程，灵活修改
- NRE 成本低（无需掩模）
- 适合中小批量生产
- 快速原型验证
- 现场可升级


#### ASIC 优势


- 更高的性能（无可编程开销）
- 更低的功耗
- 更小的面积
- 大批量成本低
- 可集成模拟电路
- 专用优化空间大


### 6.1 详细对比表


| 特性 | FPGA | ASIC |
| --- | --- | --- |
| NRE 成本 | 低（$0 ~ 数万$） | 高（数百万$ ~ 数千万$） |
| 单位成本 | 高 | 低（大批量时） |
| 开发周期 | 3-12 个月 | 12-36 个月 |
| 性能 | 中等 | 高 |
| 功耗 | 较高 | 低 |
| 灵活性 | 可重编程 | 固定（设计后不可更改） |
| 适合产量 | 1 ~ 100K | > 100K |
| 典型厂商 | Xilinx(AMD), Intel(Altera), Lattice | TSMC, Samsung, GlobalFoundries |

总结

## 总结


FPGA 是数字逻辑设计的灵活实现平台，通过 CLB（含 LUT）、IOB、BRAM、DSP 和可编程互连实现任意数字电路。其设计流程包括综合、布局布线和比特流生成。相比 ASIC，FPGA 以更低的 NRE 成本和更快的迭代速度，适合中小批量、原型验证和需要现场升级的应用。选择合适的 FPGA 平台和工具链是成功设计的关键。
