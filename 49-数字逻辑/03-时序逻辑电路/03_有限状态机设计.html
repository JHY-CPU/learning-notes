有限状态机（FSM）设计

一、有限状态机概述

有限状态机（Finite State Machine, FSM）是一种计算模型，由有限个状态、状态之间的转换规则和输入/输出组成。它是时序逻辑电路设计的核心方法论。

1.1 FSM 的五要素

  1. 状态集合 S：有限个状态的集合
  2. 输入集合 I：所有可能的输入值
  3. 输出集合 O：所有可能的输出值
  4. 状态转移函数：S x I → S（次态函数）
  5. 输出函数：S → O（Moore）或 S x I → O（Mealy）

1.2 FSM 的硬件结构

二、Mealy 机与 Moore 机

Moore 机

  • 输出仅取决于当前状态
  • 输出与输入无关
  • 输出在时钟边沿后稳定
  • 状态数通常较多
  • 输出无毛刺

Mealy 机

  • 输出取决于当前状态和输入
  • 输入变化立即影响输出
  • 输出可能提前一个周期
  • 状态数通常较少
  • 输出可能有毛刺

三、FSM 设计方法论

3.1 设计步骤

  1. 问题分析：确定输入、输出、需要几个状态
  2. 状态定义：为每个状态命名，定义含义
  3. 状态转移图/表：画出状态转换关系
  4. 状态编码：为每个状态分配二进制编码
  5. 状态转移方程：推导次态逻辑表达式
  6. 输出方程：推导输出逻辑表达式
  7. 电路实现：选择触发器类型，画出电路图

3.2 设计实例：序列检测器

设计一个检测输入序列 "1101" 的 Moore 型 FSM，当检测到连续的 1101 时输出 1。

3.3 状态转移表

当前状态 输入 X=0 输入 X=1 输出 Z
S0 (初始) S0 S1 0
S1 (检测到 1) S0 S2 0
S2 (检测到 11) S3 S2 0
S3 (检测到 110) S0 S4 0
S4 (检测到 1101) S0 S1 1

四、状态编码策略

状态编码直接影响电路的复杂度、功耗和速度。不同的编码方式产生不同的次态逻辑和输出逻辑。

4.1 常见编码方式

编码方式 S0 S1 S2 S3 S4 特点
二进制编码 000 001 010 011 100 触发器最少，组合逻辑可能复杂
格雷码 000 001 011 010 110 相邻状态只变 1 位，功耗低
独热码（One-Hot） 00001 00010 00100 01000 10000 触发器多但组合逻辑简单，适合 FPGA
Johnson 编码 0000 0001 0011 0111 1111 折中方案
选择建议：
  • 状态数 ≤ 4：二进制编码即可
  • 状态数 5-24 且使用 FPGA：独热码（FPGA 触发器资源丰富）
  • 状态数 5-24 且使用 ASIC：二进制或格雷码
  • 低功耗场景：格雷码（减少位翻转）

五、HDL 实现

5.1 Verilog 实现（序列检测器 1101）

// Moore 型 FSM - 序列检测器 "1101" // Verilog 实现 module seq_detector_1101_moore ( input wire clk, input wire rst_n, input wire din, output reg dout ); // 状态编码（二进制） parameter S0 = 3'b000; // 初始状态 parameter S1 = 3'b001; // 检测到 1 parameter S2 = 3'b010; // 检测到 11 parameter S3 = 3'b011; // 检测到 110 parameter S4 = 3'b100; // 检测到 1101（输出有效） reg [2:0] state, next_state; // 状态寄存器（时序逻辑） always @(posedge clk or negedge rst_n) begin if (!rst_n) state <= S0; else state <= next_state; end // 次态逻辑（组合逻辑） always @(*) begin case (state) S0: next_state = din ? S1 : S0; S1: next_state = din ? S2 : S0; S2: next_state = din ? S2 : S3; S3: next_state = din ? S4 : S0; S4: next_state = din ? S1 : S0; default: next_state = S0; endcase end // 输出逻辑（Moore：仅取决于状态） always @(*) begin dout = (state == S4) ? 1'b1 : 1'b0; end endmodule

5.2 Verilog 实现（Mealy 型对比）

// Mealy 型 FSM - 同样的序列检测器 // 输出取决于状态和输入 module seq_detector_1101_mealy ( input wire clk, input wire rst_n, input wire din, output reg dout ); parameter S0 = 2'b00; parameter S1 = 2'b01; parameter S2 = 2'b10; parameter S3 = 2'b11; reg [1:0] state, next_state; // 状态寄存器 always @(posedge clk or negedge rst_n) begin if (!rst_n) state <= S0; else state <= next_state; end // 次态逻辑 + 输出逻辑（合并，Mealy 特点） always @(*) begin dout = 1'b0; // 默认无输出 case (state) S0: next_state = din ? S1 : S0; S1: next_state = din ? S2 : S0; S2: next_state = din ? S2 : S3; S3: begin if (din) begin next_state = S1; dout = 1'b1; // Mealy：在状态转换时输出 end else begin next_state = S0; end end default: next_state = S0; endcase end endmodule

5.3 VHDL 实现

-- VHDL 实现 Moore 型序列检测器 library IEEE; use IEEE.STD_LOGIC_1164.ALL; entity seq_detector is Port ( clk : in STD_LOGIC; rst_n : in STD_LOGIC; din : in STD_LOGIC; dout : out STD_LOGIC ); end seq_detector; architecture Behavioral of seq_detector is type state_type is (S0, S1, S2, S3, S4); signal state, next_state : state_type; begin -- 状态寄存器 process(clk, rst_n) begin if rst_n = '0' then state <= S0; elsif rising_edge(clk) then state <= next_state; end if; end process; -- 次态逻辑 process(state, din) begin case state is when S0 => if din = '1' then next_state <= S1; else next_state <= S0; end if; when S1 => if din = '1' then next_state <= S2; else next_state <= S0; end if; when S2 => if din = '1' then next_state <= S2; else next_state <= S3; end if; when S3 => if din = '1' then next_state <= S4; else next_state <= S0; end if; when S4 => if din = '1' then next_state <= S1; else next_state <= S0; end if; end case; end process; -- 输出逻辑（Moore） dout <= '1' when state = S4 else '0'; end Behavioral;

六、FSM 设计注意事项

6.1 常见问题与解决方案

问题 原因 解决方案
无效状态 未使用的编码值（如独热码中未使用的状态） 添加 default 分支，将无效状态导向 S0
输出毛刺 Mealy 机的组合输出路径延迟不一致 在输出端添加寄存器（Registered Output）
亚稳态 异步输入未同步 两级同步器（双触发器）
状态机死锁 某些状态下无法转移 确保所有状态都有完整的转移分支
综合后不匹配 不完整的 case/if 语句 使用完整的 case 和 default 分支

6.2 设计最佳实践

编码风格建议：
  1. 将 FSM 拆分为 3 个 always 块：状态寄存器、次态逻辑、输出逻辑
  2. 使用参数或枚举定义状态，避免魔法数字
  3. 总是包含 default 分支处理无效状态
  4. 使用非阻塞赋值（<=）描述时序逻辑
  5. 使用阻塞赋值（=）描述组合逻辑
  6. 避免在多个 always 块中驱动同一信号

总结

有限状态机是数字系统设计的核心工具。Mealy 机输出与输入相关，状态少但可能有毛刺；Moore 机输出仅取决于状态，更稳定但状态更多。设计时需关注状态编码、完整性和输出同步。HDL 实现应遵循三段式风格，确保综合结果与设计意图一致。
