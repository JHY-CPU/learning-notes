JavaScript 引入方式

JS 有三种引入方式：内部、外部、内联（不推荐）

方式一：内部 script 标签

<script> console.log('Hello from internal JS!'); </script>

推荐放在 </body> 前，确保DOM已加载。

方式二：外部 JS 文件

<script src="script.js"></script>

通过 src 属性引入外部 .js 文件。

注意：带 src 时标签内的代码会被忽略。

方式三：内联事件处理（不推荐）

<button onclick="alert('click')">点我</button>

HTML 和 JS 耦合，不利于维护，仅简单测试时使用。

script 标签属性

<!-- 普通 script - 阻塞解析 --> <script src="app.js"></script> <!-- defer - 下载不阻塞，解析完再执行（按顺序） --> <script defer src="app.js"></script> <!-- async - 下载不阻塞，下载完立即执行（不按顺序） --> <script async src="app.js"></script>

对比：

• 普通：遇到 script 暂停 HTML 解析 → 下载 JS → 执行 JS → 继续解析

• defer：HTML 解析同时下载 JS → HTML 解析完按顺序执行 JS

• async：HTML 解析同时下载 JS → JS 下载完立即执行（暂停解析）

重要：script 位置建议

<!DOCTYPE html> <html> <head> <!-- CSS 放 head --> <link rel="stylesheet" href="style.css"> </head> <body> <!-- HTML 内容 --> <h1>页面内容</h1> <!-- JS 放 body 底部 --> <script src="app.js"></script> </body> </html>

• 把 JS 放在 body 底部，确保操作 DOM 时元素已存在

• 或者使用 defer 属性让 script 延迟执行
