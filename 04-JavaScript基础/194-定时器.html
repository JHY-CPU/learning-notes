定时器

setTimeout / setInterval / clearTimeout / clearInterval。

setTimeout — 延时执行

// setTimeout(函数, 延迟毫秒数, 参数...) // 在指定延迟后执行一次函数 const timerId = setTimeout(() => { console.log('1秒后执行'); }, 1000); // 带参数 setTimeout((a, b) => { console.log('参数:', a, b); }, 500, 'Hello', 'World'); // 字符串形式（不推荐，类似 eval） setTimeout('console.log("不推荐")', 1000); // clearTimeout — 取消定时器 clearTimeout(timerId); // ⚠️ 延迟时间是最小延迟，不是精确时间 // 如果主线程繁忙，实际执行可能更晚 // ⚠️ this 指向问题 // setTimeout 中的函数 this 指向 window（严格模式 undefined） // 箭头函数可以保持 this

setInterval — 间隔执行

// setInterval(函数, 间隔毫秒数, 参数...) // 每隔指定时间执行一次 const intervalId = setInterval(() => { console.log('每秒执行'); }, 1000); // clearInterval — 取消间隔 clearInterval(intervalId); // ✅ 推荐：用递归 setTimeout 替代 setInterval // 因为 setInterval 不管前一次是否完成，可能累积 function recursiveTimeout() { setTimeout(() => { // 执行操作 recursiveTimeout(); // 完成后再次调度 }, 1000); } // 递归 setTimeout 保证每次执行之间有固定间隔 // setInterval 不保证（回调执行时间也计算在内） // ⚠️ 最小间隔：浏览器通常限制最小 4ms // ⚠️ 页面隐藏时（切换标签），浏览器会降低定时器频率

clearTimeout / clearInterval

// setTimeout 和 setInterval 都返回一个数字 ID // 可以通过这个 ID 取消 // setTimeout const timeoutId = setTimeout(() => { console.log('不会执行'); }, 1000); clearTimeout(timeoutId); // setInterval const intervalId = setInterval(() => { console.log('不会执行'); }, 1000); clearInterval(intervalId); // 最佳实践 // 1. 总是保存返回值（timer ID） // 2. 在组件卸载/页面离开时清理定时器 // 3. 用递归 setTimeout 替代 setInterval // Node.js 额外 API // setImmediate — 在当前事件循环之后立即执行 // process.nextTick — 在当前操作完成后立即执行（微任务）

实际应用：倒计时
