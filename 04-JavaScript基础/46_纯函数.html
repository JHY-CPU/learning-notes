纯函数（Pure Function）

相同的输入永远得到相同的输出，且没有副作用。

纯函数的两个条件

// 1. 相同输入 → 相同输出（不依赖外部状态） // 2. 没有副作用（不修改外部变量、不IO、不DOM操作） // ✅ 纯函数 function add(a, b) { return a + b; } // 输入(1,2)永远返回3，不修改任何东西 // ❌ 非纯函数（依赖外部变量） let taxRate = 0.1; function calcPrice(price) { return price * (1 + taxRate); // taxRate 可能变化 } // ❌ 非纯函数（修改外部变量） let total = 0; function addToTotal(price) { total += price; // 副作用：修改外部变量 return total; }

副作用示例

// 常见的副作用： // 1. 修改全局变量/外部变量 // 2. console.log() // 3. DOM 操作 // 4. 网络请求 // 5. 文件读写 // 6. Math.random()（相同输入不同输出） // ❌ 不纯 function greet(name) { console.log('Hello, ' + name); // 副作用：控制台输出 } function getRandom() { return Math.random(); // 相同输入不同输出 }

纯函数的优点

// 1. 可预测 — 容易 debug // 2. 可测试 — 不需要 mock 外部状态 // 3. 可缓存 — 可以记住结果（memoization） // 4. 可并行 — 没有共享状态问题 // 缓存纯函数的结果 function memoize(fn) { const cache = new Map(); return function(arg) { if (cache.has(arg)) { return cache.get(arg); // 直接返回缓存 } const result = fn(arg); cache.set(arg, result); return result; }; } const factorial = memoize(n => { return n <= 1 ? 1 : n * factorial(n - 1); });

实践：尽量写纯函数

// ❌ 不纯：修改输入参数 function addItem(arr, item) { arr.push(item); // 修改原数组！ return arr; } // ✅ 纯：返回新数组 function addItemPure(arr, item) { return [...arr, item]; // 不修改原数组 } let original = [1, 2, 3]; addItem(original, 4); // original 被改了！ addItemPure(original, 4); // original 不变
💡 纯函数是函数式编程的基石。尽量把业务逻辑写成纯函数。
