引用与拷贝

原始类型按值传递，对象类型按引用传递。

值类型 vs 引用类型

// 值类型（存储在栈中） let a = 10; let b = a; // 复制值 a = 20; console.log(b); // 10（互不影响） // 引用类型（存储在堆中，变量存的是地址） let obj1 = { name: 'Alice' }; let obj2 = obj1; // 复制引用（地址） obj1.name = 'Bob'; console.log(obj2.name); // 'Bob'（都被改了！）

浅拷贝

// 只复制第一层，嵌套对象还是共享引用 // 方式一：展开运算符（推荐） const original = { name: 'Alice', address: { city: 'Beijing' } }; const copy1 = { ...original }; // 方式二：Object.assign const copy2 = Object.assign({}, original); // 问题：嵌套对象仍是引用 copy1.address.city = 'Shanghai'; console.log(original.address.city); // 'Shanghai'（也被改了！）

深拷贝

// 完全复制所有层级，互不影响 // 方式一：JSON 法（最简单，但有局限） const deepCopy = JSON.parse(JSON.stringify(original)); // 局限： // - 不支持函数、undefined、Symbol // - 不支持循环引用 // - Date → 字符串 // - RegExp → {} // 方式二：structuredClone（现代浏览器，推荐） // const deepCopy = structuredClone(original); // 局限： // - 不支持函数、Symbol // - 支持循环引用、Date、RegExp、Map、Set

对比示例

const original = { num: 0, str: 'hello', bool: true, arr: [1, 2, { nested: 'object' }], obj: { x: 10 }, fn: function() { return 42; }, date: new Date(), undef: undefined }; // 浅拷贝 const shallow = { ...original }; // JSON 深拷贝 const jsonDeep = JSON.parse(JSON.stringify(original)); // 丢失: fn, undef, date变成字符串 // structuredClone // const clone = structuredClone(original); // 保留: date, arr, obj, 循环引用 // 丢失: fn
