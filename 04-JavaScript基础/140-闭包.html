闭包（Closure）

函数 + 其创建时对外部变量的引用 = 闭包。函数能"记住"它出生时的环境。

什么是闭包

// 当一个函数可以记住并访问它所在词法作用域时就形成了闭包 function outer() { let count = 0; // outer 的局部变量 function inner() { count++; // inner 引用了 outer 的变量 console.log(count); } return inner; } const counter = outer(); // outer 执行完毕了... counter(); // 1 但 inner 还能访问 count！ counter(); // 2 counter(); // 3 // ❓ outer 执行完了，count 不是应该被销毁吗？ // → 闭包让 count 继续存活在内存中 // → inner 持有对 count 的引用
闭包 = 函数 + 函数能访问的外部变量。即使外部函数执行完，这些变量也不会被销毁。

闭包的作用

// 1. 数据私有化 function createCounter() { let count = 0; // 私有变量，外部不能直接访问 return { increment: () => ++count, decrement: () => --count, getCount: () => count }; } const c = createCounter(); c.increment(); // 1 c.increment(); // 2 // c.count; // undefined（不能直接访问！） console.log(c.getCount()); // 2

闭包应用：函数工厂

function makeMultiplier(x) { return function(y) { return x * y; }; } const double = makeMultiplier(2); const triple = makeMultiplier(3); double(5); // 10 triple(5); // 15 // 每个函数都有自己的 x // double 记住 x=2 // triple 记住 x=3

闭包经典问题：循环中的 var

// 问题：var i 是全局的 for (var i = 0; i < 3; i++) { setTimeout(function() { console.log(i); // 3, 3, 3（都是3！） }, 100); } // 修复1：用 let（块级作用域） for (let i = 0; i < 3; i++) { setTimeout(function() { console.log(i); // 0, 1, 2 ✅ }, 100); } // 修复2：用闭包（IIFE） for (var i = 0; i < 3; i++) { (function(j) { setTimeout(function() { console.log(j); // 0, 1, 2 ✅ }, 100); })(i); }

闭包注意事项

// 1. 内存占用：闭包引用的变量不会被垃圾回收 // 不用闭包时解除引用：counter = null; // 2. 闭包只能获取外部变量的"最新值" function buildFunctions() { const arr = []; for (var i = 0; i < 3; i++) { arr.push(function() { return i; }); } return arr; } // buildFunctions()[0]() → 3（不是0！）
💡 闭包很强大，但不要滥用。每次闭包都有内存开销。
