let — 变量声明

let 是 ES6（2015）引入的变量声明关键字，现在最常用。

基本用法

let name = 'Alice'; // 声明并赋值 let age; // 声明但不赋值（值为 undefined） age = 25; // 后续赋值 let x = 10, y = 20; // 一次声明多个变量

let 的特点

// 1. 块级作用域 { let a = 10; console.log(a); // 10 } // console.log(a); // ❌ 报错！a is not defined // 2. 不能重复声明 let b = 1; // let b = 2; // ❌ 报错！Identifier 'b' has already been declared // 3. 可以先声明后赋值 let c; c = 3; // 4. 可以重新赋值 let d = 'hello'; d = 'world'; // ✅ 可以
let 的块级作用域是它和 var 最大的区别！

块级作用域演示

if (true) { let msg = '我在块内'; console.log(msg); // ✅ 可以 } // console.log(msg); // ❌ 报错 for (let i = 0; i < 3; i++) { // 每次循环 i 都是新的变量 console.log(i); // 0, 1, 2 } // console.log(i); // ❌ 报错

暂时性死区 (TDZ)

// let 不存在变量提升（和 var 不同） // console.log(e); // ❌ 报错！Cannot access 'e' before initialization let e = 5; // 从块开始到声明之间是"暂时性死区" { // console.log(f); // ❌ 报错！TDZ let f = 10; console.log(f); // ✅ 10 }
在 let 声明之前访问变量会报 ReferenceError，这叫"暂时性死区"（Temporal Dead Zone, TDZ）

命名规则

// ✅ 合法命名 let userName; // 驼峰命名（推荐） let user_name; // 下划线命名 let $name; // 可以 $ 开头 let _age; // 可以 _ 开头 let 变量1; // 可以用中文（不推荐） let data2; // 可以包含数字 // ❌ 非法命名 // let 2data; // 不能数字开头 // let my-name; // 不能包含连字符 // let let; // 不能是保留字 // let const; // 不能是保留字 // let class; // 不能是保留字

推荐命名风格：驼峰命名法（camelCase）
