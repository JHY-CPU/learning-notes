sessionStorage 与 Cookie

会话存储（标签页关闭即失）和 Cookie（HTTP 自动携带）。

sessionStorage

// sessionStorage — 会话级存储 // 和 localStorage 的 API 完全相同 // 区别：标签页关闭后数据自动清除 // 基本操作（同 localStorage） sessionStorage.setItem('key', 'value'); sessionStorage.getItem('key'); sessionStorage.removeItem('key'); sessionStorage.clear(); sessionStorage.length; sessionStorage.key(0); // ⚠️ 生命周期：标签页关闭即删除 // ⚠️ 不同标签页之间不共享（即使是同一站点） // ⚠️ 页面刷新不会清除数据 // 使用场景： // 1. 表单临时数据（防止刷新丢失） // 2. 一次性登录 token // 3. 多步骤表单的中间状态 // 4. 页面间临时数据传递

Cookie

// Cookie — 服务器存储在浏览器的小数据（<4KB） // 每次 HTTP 请求会自动携带 Cookie // 读取所有 Cookie（返回字符串） document.cookie; // 'key1=value1; key2=value2' // 设置 Cookie document.cookie = 'username=Alice'; document.cookie = 'theme=dark'; // 设置过期时间 document.cookie = 'token=abc123; max-age=3600'; // 1小时后过期 document.cookie = 'session=xyz; expires=' + new Date('2025-12-31').toUTCString(); // 设置路径 document.cookie = 'lang=zh-CN; path=/'; // 设置域名 document.cookie = 'site=example; domain=.example.com'; // 安全标志 document.cookie = 'secureToken=xxx; secure'; // 仅 HTTPS document.cookie = 'httpOnlyToken=yyy; httponly'; // JS 不可读（服务器设置） document.cookie = 'sameSiteToken=zzz; samesite=strict'; // SameSite // 删除 Cookie（设置过期时间为过去） document.cookie = 'username=; max-age=0'; // ⚠️ Cookie 大小限制约 4KB // ⚠️ 每个域名 Cookie 数量有限（约 50 个） // ⚠️ 每次请求都会携带，影响性能 // Cookie vs localStorage： // Cookie: 自动携带，适合登录态 // localStorage: 不自动携带，适合大量数据

三种存储对比

// | 特性 | localStorage | sessionStorage | Cookie | // |--------------|-------------|----------------|-----------------| // | 生命周期 | 永不过期 | 标签关闭即删 | 可设置过期时间 | // | 大小限制 | ~5MB | ~5MB | ~4KB | // | 是否自动发送 | ❌ | ❌ | ✅（HTTP 请求） | // | API 易用性 | ✅ 简洁 | ✅ 简洁 | ❌ 原始字符串 | // | 同源共享 | ✅ 同源共享 | ❌ 仅当前标签 | ✅ 同源配置 | // | 作用域 | 同源 | 当前标签 | path/domain | // 选择建议： // 大量数据 → localStorage // 临时数据 → sessionStorage // 登录态/服务器需要的数据 → Cookie
