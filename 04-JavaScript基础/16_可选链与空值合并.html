可选链 ?. 和 空值合并 ??

ES2020 引入的两个语法糖，用于安全访问深层属性和设置默认值。

可选链 ?. — 问题背景

// 嵌套对象访问经常要判断每一层是否存在 let user = { profile: { name: 'Alice' } }; // 旧写法：每一层都要判断 let name = user && user.profile && user.profile.name; // 或更繁琐的： let name2 = user ? (user.profile ? user.profile.name : undefined) : undefined; // 可选链写法： let name3 = user?.profile?.name; // "Alice" ✅
?. 前面的值是 null 或 undefined 时，立即返回 undefined，不继续访问。

可选链的三种形式

// 1. 属性访问 obj?.prop // 如果 obj 是 null/undefined → undefined obj?.prop?.sub // 链式访问 // 2. 方法调用 obj?.method?.() // 如果 method 不存在就不调用，不报错 // 3. 索引访问 arr?.[index] // 如果 arr 是 null/undefined → undefined

可选链示例

let user1 = { name: 'Alice', getInfo() { return 'info'; } }; let user2 = null; let user3 = { name: 'Bob' }; user1?.name; // "Alice" user2?.name; // undefined（不报错！） user3?.getInfo?.(); // undefined（方法不存在，不报错） user1?.getInfo?.(); // "info" // 数组 let arr = [1, 2, 3]; let arr2 = null; arr?.[0]; // 1 arr2?.[0]; // undefined

空值合并 ?? — 问题背景

// 设置默认值，常见做法是用 || let count = 0; let result = count || 10; // 10 ❌ 因为 0 是 falsy！ // 但 0 是合法值，不应该被替换为默认值 // ?? 只检测 null 和 undefined，不检测其他 falsy 值 let result2 = count ?? 10; // 0 ✅ 只有 null/undefined 才用默认值
💡 ?? 和 || 的区别：?? 只对 null/undefined 使用默认值，|| 对所有 falsy 值使用默认值。

?? 使用场景对比

let score = 0; let username = ''; let isReady = false; // || 的问题：所有 falsy 值都被替换 score || 100; // 100（0被误替换） username || '游客'; // '游客'（空字符串被误替换） // ?? 更精确：只有 null/undefined 才替换 score ?? 100; // 0 ✅ username ?? '游客'; // '' ✅ isReady ?? true; // false ✅ // 真正需要默认值的场景 let data = null; data ?? '默认数据'; // '默认数据' ✅

?? 和 ?. 配合使用

// 安全访问 + 默认值 let user = null; let displayName = user?.name ?? '游客'; // user?.name → undefined // undefined ?? '游客' → '游客' console.log(displayName); // '游客' // 更复杂的场景 let config = { theme: { // colors: { primary: '#3498db' } } }; let primaryColor = config?.theme?.colors?.primary ?? '#default'; // 如果任意一层不存在，?? 就提供默认值
?? 和 ?. 是黄金搭档，一起用让代码更安全简洁。

⚠️ 注意事项

// 1. ?. 不能用于赋值 // obj?.prop = 'value'; // ❌ 报错 // 2. ?? 不能和 && 或 || 直接混用（需要括号） // null || undefined ?? 'default'; // ❌ 报错 (null || undefined) ?? 'default'; // ✅ // 3. 可选链的返回值类型是 undefined // 如果访问的路径不存在，结果是 undefined
