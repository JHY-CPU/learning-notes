数字类型 number

JavaScript 的 number 是 64 位双精度浮点数（IEEE 754）。

数字的写法

let int = 42; // 整数 let float = 3.14; // 浮点数 let negative = -10; // 负数 let scientific = 1.5e6; // 科学计数法 = 1500000 let scientific2 = 1e-3; // = 0.001 // 不同进制 let hex = 0xFF; // 十六进制 = 255 let octal = 0o77; // 八进制 = 63 let binary = 0b1010; // 二进制 = 10 // 数字分隔符（ES2021，提高可读性） let bigNum = 1_000_000; // = 1000000 let price = 99_99; // = 9999

特殊数字值

// NaN - Not a Number（非数字） let notNum = 0 / 0; // NaN let nan2 = parseInt('abc'); // NaN console.log(NaN === NaN); // false ← NaN 不等于自身！ // Infinity - 无穷大 let posInf = 1 / 0; // Infinity let negInf = -1 / 0; // -Infinity // 判断方法 isNaN(NaN); // true isNaN('123'); // false（会先转数字） Number.isNaN(NaN); // true（推荐，不强制转换） isFinite(Infinity); // false isFinite(42); // true Number.isFinite(42); // true
isNaN('123abc') → true （因为 '123abc' 转数字是 NaN）
Number.isNaN('abc') → false（不转换类型，更严格）

浮点数精度问题

// 经典问题 console.log(0.1 + 0.2); // 0.30000000000000004 console.log(0.1 + 0.2 === 0.3); // false！ // 为什么？IEEE 754 二进制无法精确表示 0.1 // 解决方法：用 toFixed 或乘以整数 let sum = (0.1 * 10 + 0.2 * 10) / 10; // 0.3 console.log(sum.toFixed(1)); // "0.3"（注意返回字符串）
⚠️ 做金额计算时注意精度，建议使用整数（分）计算，或使用专门的库。

Number 方法

let num = 123.456; num.toFixed(2); // "123.46"（四舍五入，返回字符串） num.toPrecision(4); // "123.5"（指定总位数） num.toString(16); // "7b.74..."（转16进制字符串） Number.isInteger(42); // true Number.isInteger(3.14); // false Number.isNaN(NaN); // true（推荐） Number.isFinite(Infinity); // false Number.parseFloat('3.14px'); // 3.14 Number.parseInt('100px', 10); // 100

Number.EPSILON 与安全整数

// 最小精度 Number.EPSILON; // 2.220446049250313e-16 // 安全整数范围 Number.MAX_SAFE_INTEGER; // 9007199254740991 (2^53 - 1) Number.MIN_SAFE_INTEGER; // -9007199254740991 Number.MAX_VALUE; // 1.7976931348623157e+308 Number.MIN_VALUE; // 5e-324（最小的正数） // 超出安全整数的运算会不精确 console.log(9999999999999999); // 10000000000000000
