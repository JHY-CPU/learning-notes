变量提升与暂存死区

Hoisting（提升）和 TDZ（Temporal Dead Zone，暂存死区）是 JS 变量声明的特殊行为。

var 的提升（Hoisting）

// 你写的代码： console.log(a); // undefined（不是报错！） var a = 10; // JS 实际执行的代码： var a; // 声明提升到顶部，初始化为 undefined console.log(a); // undefined a = 10; // 赋值留在原地
var 的提升是 bug 的来源——访问未声明的变量不会报错，只是 undefined。

let/const 的暂存死区（TDZ）

// let 也会被提升，但不会初始化 console.log(b); // ❌ ReferenceError: Cannot access 'b' before initialization let b = 20; // 从块开始到声明之间的区域称为"暂存死区" { // TDZ 开始（不能访问 c） // console.log(c); // ❌ 报错 let c = 30; // TDZ 结束（可以访问 c） console.log(c); // ✅ 30 } // const 同样有 TDZ // console.log(d); // ❌ 报错 const d = 40;

var vs let 提升对比

// var：声明提升并初始化为 undefined console.log(v1); // undefined var v1 = 1; // let：声明提升但不初始化（TDZ） // console.log(l1); // ❌ 报错 let l1 = 1; // 函数声明：整体提升（定义+赋值） foo(); // ✅ 可以正常调用 function foo() { return 'foo'; } // 函数表达式：取决于 var/let // bar(); // ❌ 报错（let 不提升） const bar = function() { return 'bar'; };

typeof 和 TDZ

// typeof 对未声明变量不会报错 typeof undeclaredVar; // "undefined"（不报错！） // 但 typeof 在 TDZ 中会报错！ // typeof x; // ❌ 报错 let x = 10; // 对 var 提升的变量 typeof 是安全的 console.log(typeof y); // "undefined" var y = 20;

最佳实践

// 1. 始终在作用域顶部声明变量 // ❌ function bad() { // ... 很多代码 ... let x = 1; // ... 更多代码 ... } // ✅ function good() { let x; // ... 很多代码 ... x = 1; // ... 更多代码 ... } // 2. 始终用 const/let，不用 var // 3. 声明顺序：const > let > (不用 var)
