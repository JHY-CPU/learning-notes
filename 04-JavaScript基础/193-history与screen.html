history 与 screen

浏览历史与屏幕信息。

history — 浏览历史

// history 对象管理浏览器会话历史 // 基本属性 history.length // 历史记录数量 // 导航方法 history.back() // 后退（等价于 history.go(-1)） history.forward() // 前进（等价于 history.go(1)） history.go(-2) // 后退两步 history.go(2) // 前进两步 history.go(0) // 刷新当前页 // ⚠️ 不能读取具体的历史 URL（隐私限制） // ⚠️ 只能通过 length 知道有多少条记录

pushState / replaceState — 无刷新改 URL

// pushState — 添加历史记录（不刷新页面） history.pushState(state, title, url); // state — 与记录关联的数据（可以是任何可序列化数据） // title — 标题（目前大部分浏览器忽略此参数） // url — 新 URL（可选，必须同源） // 示例： history.pushState({ page: 1 }, '', '?page=1'); history.pushState({ page: 2 }, '', '?page=2'); // replaceState — 替换当前历史记录（不新增） history.replaceState({ page: 3 }, '', '?page=3'); // 应用：单页应用（SPA）路由 function navigate(path) { history.pushState({ path }, '', path); renderPage(path); // 根据路径渲染内容 } // popstate 事件 — 当用户点击前进/后退时触发 window.addEventListener('popstate', (event) => { console.log('状态变化:', event.state); // event.state 就是 pushState 传入的 state }); // ⚠️ pushState/replaceState 不触发 popstate // ⚠️ 只有用户点击前进/后退时才触发 popstate

screen — 屏幕信息

// screen 对象包含显示屏幕的信息 screen.width // 屏幕宽度（物理像素） screen.height // 屏幕高度（物理像素） screen.availWidth // 可用宽度（排除任务栏） screen.availHeight // 可用高度（排除任务栏） screen.colorDepth // 色彩深度（通常 24 或 32） screen.pixelDepth // 像素深度（通常等于 colorDepth） screen.orientation // 屏幕方向 // screen 和 window 的区别： // screen — 整个物理屏幕的尺寸 // window.inner — 浏览器视口尺寸 // window.outer — 浏览器窗口尺寸 // 示例： // 1920x1080 屏幕，浏览器全屏： screen.width // 1920 screen.height // 1080 window.innerWidth // ~1920（有边框则略小） window.outerWidth // 1920 // 检查屏幕方向 screen.orientation?.type // 'landscape-primary' 或 'portrait-primary' screen.orientation?.angle // 旋转角度 // 锁定屏幕方向（需要用户手势触发） // screen.orientation?.lock('portrait');
