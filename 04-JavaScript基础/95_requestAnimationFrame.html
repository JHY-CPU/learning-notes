requestAnimationFrame

浏览器优化的动画帧调度，与屏幕刷新率同步（通常 60fps）。

requestAnimationFrame 基本用法

// requestAnimationFrame(callback) // 在下一次浏览器重绘之前调用 callback // 返回请求 ID（用于取消） const animId = requestAnimationFrame(function(timestamp) { // timestamp — 页面加载以来的毫秒数（高精度时间） console.log('动画帧触发, 时间:', timestamp); }); // 取消动画帧 cancelAnimationFrame(animId); // 常见模式：递归调用实现连续动画 function animate(timestamp) { // 更新动画状态 updatePosition(); // 请求下一帧 requestAnimationFrame(animate); } // 启动动画 requestAnimationFrame(animate);

rAF vs setTimeout / setInterval

// requestAnimationFrame 的优点： // 1. 与屏幕刷新率同步 // 60Hz 屏幕 → 约 16.67ms 执行一次 // 120Hz 屏幕 → 约 8.33ms 执行一次 // setTimeout 无法精确同步 // 2. 浏览器优化 // 页面不可见时（切换到后台标签），rAF 暂停 // 省电、省 CPU // 3. 更平滑的动画 // rAF 在重绘之前执行，不会丢帧 // setTimeout 可能错过帧同步 // 4. 自带时间戳 // callback 接收高精度时间戳 // 无需自己计算时间差 // 什么时候用 rAF： // ✅ 视觉动画（移动、缩放、旋转） // ✅ Canvas/SVG 渲染 // ✅ 滚动同步 // ❌ 网络请求（用 setTimeout 或 setInterval） // ❌ 精确计时（rAF 不适合） // rAF vs setInterval： // rAF: 浏览器控制执行时机，平滑高效 // setInterval: 固定间隔，可能丢帧或过度绘制

实用的帧率控制

// 限制 rAF 执行频率（比如每秒 30 帧） let lastTime = 0; const fps = 30; const interval = 1000 / fps; function animate(timestamp) { const delta = timestamp - lastTime; if (delta >= interval) { // 更新逻辑 update(); lastTime = timestamp - (delta % interval); } requestAnimationFrame(animate); } // 计算 FPS let frameCount = 0; let lastFpsTime = 0; function showFps(timestamp) { frameCount++; if (timestamp - lastFpsTime >= 1000) { console.log('FPS:', frameCount); frameCount = 0; lastFpsTime = timestamp; } requestAnimationFrame(showFps); } // 停止动画 // cancelAnimationFrame(rafId);
