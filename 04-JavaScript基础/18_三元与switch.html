三元运算符 ?: 与 switch 语句

两种替代 if...else 的条件判断方式。

三元运算符（条件 ? 值1 : 值2）

根据条件返回两个值之一，适合简单的二选一。

let age = 20; let status = age >= 18 ? '成年' : '未成年'; console.log(status); // '成年' // 语法：条件 ? 表达式1 : 表达式2 // 条件为 true → 返回 表达式1 // 条件为 false → 返回 表达式2

三元 vs if...else

// if...else 写法（5行） let price; if (isMember) { price = 100; } else { price = 200; } // 三元写法（1行） let price = isMember ? 100 : 200;
💡 三元运算符适合简单的赋值判断，逻辑复杂时还是用 if。

三元嵌套（不推荐）

// 嵌套三元（可读性差） let level = score >= 90 ? '优秀' : score >= 80 ? '良好' : score >= 70 ? '中等' : score >= 60 ? '及格' : '不及格'; // 如果条件复杂，建议用 if...else 或 switch
⚠️ 嵌套三元虽然简洁，但可读性差。条件和结果一目了然时才使用。

switch 语句

判断一个值与多个值是否相等，用 === 比较。

switch (表达式) { case 值1: // 匹配值1时执行 break; case 值2: // 匹配值2时执行 break; case 值3: case 值4: // 匹配值3或值4时执行（合并case） break; default: // 都没匹配时执行 break; }

switch 示例

let day = 3; let dayName; switch (day) { case 1: dayName = '星期一'; break; case 2: dayName = '星期二'; break; case 3: dayName = '星期三'; break; case 4: dayName = '星期四'; break; case 5: dayName = '星期五'; break; case 6: dayName = '星期六'; break; case 7: dayName = '星期日'; break; default: dayName = '无效的日期'; } console.log(dayName); // '星期三'

break 的重要性

// 没有 break 会"穿透"继续执行下一个 case let fruit = 'apple'; switch (fruit) { case 'apple': console.log('苹果'); // 没有 break！ case 'banana': console.log('香蕉'); // 也会执行！ break; default: console.log('其他'); } // 输出：'苹果' '香蕉'（穿透效应） // 有时故意利用穿透合并条件 let grade = 'B'; switch (grade) { case 'A': case 'B': case 'C': console.log('及格'); break; case 'D': case 'F': console.log('不及格'); break; }
每个 case 后面记得加 break，除非有意利用穿透。

何时用 switch？

// 适合：判断一个变量的多个具体值 switch (role) { case 'admin': // ... case 'editor': // ... case 'viewer': // ... } // 不适合：范围判断、复杂条件 // if 更合适： if (score >= 90) { // ... } if (age >= 18 && hasLicense) { // ... }
