var — 旧的变量声明

var 是 ES6 之前的声明方式，现在不推荐使用，理解它是为了看懂老代码。

⚠️ 新代码请使用 let 和 const，不要使用 var！

var 的特点（也是它的缺点）

// 1. 函数作用域（不是块级作用域） if (true) { var x = 10; } console.log(x); // ✅ 10（块外仍可访问！） // 2. 可以重复声明 var y = 1; var y = 2; // ✅ 不报错（后面覆盖前面） // 3. 变量提升（hoisting） console.log(z); // undefined（不报错！） var z = 5; // 相当于： // var z; // console.log(z); // undefined // z = 5;

变量提升（Hoisting）

var 声明的变量会被"提升"到作用域顶部，值初始化为 undefined。

// 代码这样写： console.log(a); // undefined var a = 10; // 实际执行时相当于： var a; console.log(a); // undefined a = 10;
这是 var 最容易导致 bug 的特性——访问未声明变量不会报错，而是得到 undefined。

函数作用域 vs 块级作用域

// var 是函数作用域：在函数内可见 function test() { var inside = '函数内'; console.log(inside); // ✅ } // console.log(inside); // ❌ 报错 // var 无视块级作用域 for (var i = 0; i < 3; i++) { // ... } console.log(i); // ✅ 3（循环结束还能访问！这是 bug 来源）

经典问题：var 在循环中

// var 在 for 循环中的问题 for (var j = 0; j < 3; j++) { setTimeout(function() { console.log(j); // 输出 3, 3, 3（不是 0, 1, 2） }, 100); } // 因为 var j 是全局的，循环结束时 j=3 // 所有回调共享同一个 j // 用 let 修复 for (let k = 0; k < 3; k++) { setTimeout(function() { console.log(k); // 输出 0, 1, 2 ✅ }, 100); } // 每次循环 k 都是新的变量（块级作用域）
