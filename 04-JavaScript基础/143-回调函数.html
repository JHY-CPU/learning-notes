回调函数

作为参数传递给另一个函数，在特定时机被调用的函数。

什么是回调

// 函数作为参数 function greet(name, callback) { const message = `你好, ${name}!`; callback(message); // 在函数内部调用回调 } // 使用 greet('Alice', function(msg) { console.log(msg); }); // 用箭头函数 greet('Bob', msg => console.log(msg));

同步回调

// 回调立即执行（同步） function process(arr, callback) { for (let i = 0; i < arr.length; i++) { callback(arr[i], i); } } process([1, 2, 3], (item, index) => { console.log(`索引${index}: ${item}`); }); // 索引0: 1, 索引1: 2, 索引2: 3 // forEach、map、filter 等都是同步回调

异步回调

// 回调不立即执行，在将来某个时间点执行 console.log('开始'); setTimeout(function() { console.log('2秒后执行（异步回调）'); }, 2000); console.log('结束'); // 输出顺序： // 开始 // 结束 // 2秒后执行（异步回调） // 其他异步回调例子： // - 按钮点击事件 // - AJAX 请求完成 // - 文件读取完成

回调地狱

// 多层嵌套的回调导致可读性差 // 称为"回调地狱"（Callback Hell） getUser(id, function(user) { getPosts(user.id, function(posts) { getComments(posts[0].id, function(comments) { getReplies(comments[0].id, function(replies) { console.log(replies); // 嵌套太深了！ }); }); }); }); // 解决方式： // 1. 命名函数代替匿名函数 // 2. Promise // 3. async/await
