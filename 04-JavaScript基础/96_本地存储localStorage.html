本地存储 localStorage

持久化存储，数据保存在浏览器中，不会过期。

localStorage 基本操作

// localStorage — 持久存储（除非手动清除，否则永不过期） // 存储键值对（值必须是字符串） localStorage.setItem('key', 'value'); // 获取值 const value = localStorage.getItem('key'); // 'value' 或 null // 删除某个键 localStorage.removeItem('key'); // 清空所有 localStorage.clear(); // 获取键名 localStorage.key(0); // 第0个键名 // 获取数量 localStorage.length; // 存储项数量 // ⚠️ 存储的值会被自动转为字符串 localStorage.setItem('num', 42); typeof localStorage.getItem('num'); // 'string'（不是 number） // ⚠️ 存储对象/数组需要 JSON 序列化 const user = { name: 'Alice', age: 25 }; localStorage.setItem('user', JSON.stringify(user)); const saved = JSON.parse(localStorage.getItem('user')); console.log(saved.name); // 'Alice' // ⚠️ 存储大小限制约 5MB（不同浏览器略有差异） // ⚠️ 同源策略（协议 + 域名 + 端口）共享

storage 事件（跨标签通信）

// 当 localStorage 发生变化时，其他标签页会收到 storage 事件 // （⚠️ 只有其他标签页才会触发，当前页面不会） window.addEventListener('storage', (event) => { console.log('键:', event.key); // 修改的键名 console.log('旧值:', event.oldValue); // 旧值 console.log('新值:', event.newValue); // 新值 console.log('URL:', event.url); // 修改的页面 URL console.log('存储区:', event.storageArea); // localStorage 对象 }); // 用途：跨标签页同步数据 // 标签A 修改了数据 → 标签B 收到 storage 事件 → 标签B 更新界面
