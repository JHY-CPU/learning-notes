const — 常量声明

const 用于声明不可重新赋值的变量（常量）。

基本用法

const PI = 3.14159; // 数学常量 const MAX_SIZE = 100; // 配置常量 const COLORS = ['red', 'green', 'blue']; // 数组 const USER = { name: 'Alice' }; // 对象

const 声明时必须初始化，不能先声明后赋值。

const 的特点

// 1. 必须初始化 // const a; // ❌ 报错！Missing initializer in const declaration // 2. 不能重新赋值 const TAX_RATE = 0.08; // TAX_RATE = 0.1; // ❌ 报错！Assignment to constant variable // 3. 块级作用域（和 let 一样） { const BLOCK_CONST = '块内常量'; console.log(BLOCK_CONST); // ✅ } // console.log(BLOCK_CONST); // ❌ 报错 // 4. 不能重复声明 // const TAX_RATE = 0.09; // ❌ 报错

重要：const 的对象和数组可以修改内容

const USER = { name: 'Alice', age: 25 }; // ❌ 不能重新赋值 // USER = { name: 'Bob' }; // 报错 // ✅ 可以修改属性 USER.name = 'Bob'; USER.age = 30; // ✅ 可以添加新属性 USER.email = 'bob@example.com'; // 原理：const 保护的是"引用"不变，不是内容不变
const 保证的是变量的绑定不可变，而不是值不可变。

数组用 const

const FRUITS = ['apple', 'banana']; // ✅ 可以修改内容 FRUITS.push('cherry'); FRUITS[0] = 'avocado'; // ❌ 不能重新赋值 // FRUITS = ['new']; // 报错

什么时候用 const？

尽可能使用 const，少用 let。

// ✅ 应该用 const 的场景 const BASE_URL = 'https://api.example.com'; const MAX_RETRIES = 3; const config = { theme: 'dark' }; const items = [1, 2, 3]; // ✅ 需要重新赋值时才用 let let count = 0; count++; // 需要修改值 let temp = getValue(); temp = process(temp); // 需要重新赋值
💡 最佳实践：默认都用 const，只有确定需要重新赋值时才用 let。

let vs const 对比

/* === let 用：值会改变 === */ let counter = 0; counter++; // ✅ /* === const 用：值不变 === */ const MIN_AGE = 18; // MIN_AGE = 20; // ❌ /* === const 对象：引用不变，内容可变 === */ const CART = []; CART.push('item'); // ✅
