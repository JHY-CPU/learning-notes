# Speculative Decoding - 模型推理与Serving

*通过小模型"投机"生成候选token、大模型并行验证的方式，在不牺牲输出质量的前提下实现2-3倍推理加速*

关键洞察

- **LLM推理是内存带宽瓶颈**
   : 生成每个token需要加载全部模型参数（如70B模型需要140GB），实际计算量很小，GPU大部分时间在等内存读取
- **验证比生成快**
   : 大模型验证K个token（并行计算）的时间只比生成1个token略多，因为瓶颈在内存读取而非计算
- **小模型预测大部分token正确**
   : 对于常见文本模式，小模型和大模型的输出分布高度重叠，大部分投机token会被接受
- **零质量损失**
   : 通过精心设计的拒绝采样机制，最终输出分布与大模型完全一致（无任何质量损失）

接受率影响因素

| 因素 | 影响 | 优化方向 |
| --- | --- | --- |
| 模型大小差距 | 差距越大，分布差异越大，接受率越低 | 选择中等大小的draft模型 |
| 训练数据一致性 | 数据差异导致分布不匹配 | 用相同数据训练draft模型 |
| 文本类型 | 常见模式接受率高，创造性文本接受率低 | 根据场景调整K值 |
| 温度参数 | 温度越高，分布越随机，接受率越低 | 高温时减小K值 |
| 语言 | 英文通常比中文接受率高（tokenizer影响） | 针对语言优化draft模型 |


**典型接受率**: 7B draft + 70B target，英文约70-80%，中文约60-70%。

EAGLE vs MEDUSA 对比

| 维度 | MEDUSA | EAGLE |
| --- | --- | --- |
| Draft方式 | 多个独立Head并行预测 | 自回归小模型逐个预测 |
| 输入特征 | 仅用最后一层hidden state | 使用多层hidden states + token embedding |
| 接受率 | 中等（Head独立预测） | 高（自回归+特征级draft） |
| 额外参数 | 几MB（线性层） | 几百MB（小型Transformer） |
| 训练成本 | 几小时 | 1-2天 |
| 加速比 | 2.5x-3.5x | 3x-4x |

加速比公式

设每轮投机K个token，平均接受率为 alpha（0-1之间），则：


```
# 每轮平均接受的token数 = alpha + alpha^2 + alpha^3 + ... + alpha^K
#                      = alpha * (1 - alpha^K) / (1 - alpha)

# 加速比 ≈ (平均接受token数 + 1) / 1
#        = 1 + alpha*(1-alpha^K)/(1-alpha)

# 例：K=5, alpha=0.8
# 平均每轮接受 = 0.8 + 0.64 + 0.512 + 0.4096 + 0.32768 = 2.69
# 加速比 ≈ 3.69 / 1 = 3.69x

# 注意：实际加速比还受draft模型推理时间影响
# 实际加速比 = (大模型1步时间) / (小模型K步时间 + 大模型1步验证时间/K轮平均token数)
```
