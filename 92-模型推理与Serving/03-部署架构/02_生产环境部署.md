# 生产环境部署 - 模型推理与Serving

*K8s部署、HPA自动扩缩容、Canary发布、全链路监控与SLA保障的LLM推理生产部署完整指南*

关键扩缩指标

| 指标 | 来源 | 扩容阈值 | 缩容阈值 |
| --- | --- | --- | --- |
| GPU利用率 | DCGM Exporter | >70% | <30% |
| 显存使用率 | DCGM Exporter | >85% | <50% |
| 请求队列深度 | vLLM metrics | >50 | <10 |
| P99推理延迟 | Prometheus | >500ms | <200ms |
| 吞吐量(tok/s) | vLLM metrics | 接近上限 | <50%上限 |

LLM推理核心监控指标

| 分类 | 指标 | 说明 | 告警阈值 |
| --- | --- | --- | --- |
| **延迟** | TTFT (Time to First Token) | 首token延迟 | P99 > 1s |
| TPOT (Time per Output Token) | 每个输出token延迟 | P99 > 100ms |
| E2E延迟 | 端到端请求延迟 | P99 > 10s |
| Queue等待时间 | 请求在队列中的等待时间 | P99 > 5s |
| **吞吐** | 请求吞吐 (req/s) | 每秒处理请求数 | 低于预期50% |
| Token吞吐 (tok/s) | 每秒生成token数 | 持续下降 |
| 并发请求数 | 同时处理的请求数 | > max_batch的80% |
| **显存** | GPU显存使用率 | 显存占用百分比 | > 95% |
| KV Cache使用率 | KV Cache分配比例 | > 90% |
| GPU利用率 | GPU计算单元使用率 | < 20%（空闲） |
| **质量** | 错误率 | 5xx / 超时比例 | > 1% |
| OOM次数 | 显存溢出次数 | > 0 |

LLM推理服务 SLO 示例

| SLI (指标) | SLO (目标) | 测量方式 |
| --- | --- | --- |
| 可用性 | 99.9% (月度) | 成功请求数 / 总请求数 |
| 首Token延迟 P99 | < 1秒 | TTFT第99百分位 |
| 端到端延迟 P95 | < 10秒 (长文本除外) | E2E第95百分位 |
| Token吞吐 | > 1000 tok/s (集群) | 总输出token / 时间 |
| 错误率 | < 0.1% | 5xx请求数 / 总请求数 |

LLM推理成本构成

| 成本项 | 占比 | 优化方向 |
| --- | --- | --- |
| GPU计算 | 60-70% | 量化、高效引擎、批处理 |
| GPU显存 | 15-20% | 模型量化、显存复用 |
| 网络传输 | 5-10% | 减少跨节点通信 |
| 存储 | 3-5% | 模型压缩、按需加载 |
| 人力运维 | 5-10% | 自动化运维 |

上线前检查清单

1. **性能基线**
   : 建立延迟、吞吐、错误率的性能基线
2. **负载测试**
   : 模拟峰值流量的压测，确认容量上限
3. **故障注入**
   : 模拟GPU故障、网络分区等异常，验证容灾能力
4. **监控就绪**
   : 核心指标、告警规则、Dashboard全部就绪
5. **日志收集**
   : 结构化日志接入ELK/Loki，支持请求链路追踪
6. **回滚方案**
   : 模型版本回滚流程文档化，演练通过
7. **限流熔断**
   : 配置合理的限流阈值和熔断规则
8. **安全审计**
   : API鉴权、输入过滤、输出安全检查就绪
9. **文档更新**
   : API文档、运维手册、应急预案更新完毕
10. **值班安排**
   : 确认on-call人员和升级路径
