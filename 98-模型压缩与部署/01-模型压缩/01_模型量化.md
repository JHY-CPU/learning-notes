# 模型量化 - 模型压缩与部署


## 1. 量化基本概念


模型量化是将模型中的浮点数权重和激活值从高精度（如FP32）映射到低精度（如INT8、INT4）的过程，目的是减少模型存储空间和加速推理计算。


$$
量化公式： Q(x) = round((x - zero_point) / scale)
反量化公式： x' = Q(x) × scale + zero_point
$$


- **Scale（缩放因子）：**
   浮点范围与整数范围的比值
- **Zero Point（零点）：**
   浮点数0对应的整数值，保证精确表示零
- **FP32 → INT8：**
   模型大小减少4倍，推理速度提升2-4倍
- **FP32 → INT4：**
   模型大小减少8倍，适用于大语言模型部署


> **Note:** 量化的本质是一种有损压缩，通过牺牲少量精度换取存储和计算效率。在大多数CV和NLP任务中，INT8量化带来的精度损失通常小于1%。


## 2. 训练后量化 vs 量化感知训练


| 对比维度 | 训练后量化（PTQ） | 量化感知训练（QAT） |
| --- | --- | --- |
| 是否需要训练 | 不需要，直接在训练好的模型上量化 | 需要，在训练中模拟量化效果 |
| 精度损失 | 较大（尤其低比特量化） | 较小，模型学习适应量化噪声 |
| 计算成本 | 低，仅需少量校准数据 | 高，需要重新训练或微调 |
| 适用场景 | 快速部署、资源有限 | 精度要求高、生产级部署 |
| 典型方法 | 动态量化、静态量化 | 伪量化训练、LSQ |


### PTQ流程


1. 准备代表性校准数据集（100-1000个样本）
2. 前向传播，统计每层激活值的分布范围
3. 计算scale和zero_point
4. 将权重和激活量化为低精度


### QAT原理


在训练过程中插入伪量化节点（Fake Quantization），模拟量化引入的误差，使模型权重在训练阶段就适应量化后的数值精度。


$$
FakeQuant(x) = Dequant(Quant(x)) = round((clamp(x, min, max) - min) / scale) × scale + min
$$


## 3. 对称量化 vs 非对称量化


| 对比维度 | 对称量化 | 非对称量化 |
| --- | --- | --- |
| 零点（zero_point） | 固定为0 | 可变，需计算 |
| 量化范围 | [-max(\|x\|), max(\|x\|)] | [min(x), max(x)] |
| 计算复杂度 | 低，仅需scale | 略高，需要scale和zero_point |
| 适用场景 | 权重分布对称（如ReLU激活后） | 分布不对称（如有正负值的激活） |
| 精度 | 分布不对称时精度较低 | 更充分利用量化范围 |


$$
对称量化： scale = max(|x|) / 127, zero_point = 0
非对称量化： scale = (max - min) / 255, zero_point = -min / scale
$$


> **Important:** 在LLM推理中，权重通常使用对称量化（W8A8），因为权重分布接近对称且可以跳过zero_point计算，提升推理速度。


## 4. 逐层量化 vs 逐通道量化


| 对比维度 | 逐层量化（Per-Tensor） | 逐通道量化（Per-Channel） |
| --- | --- | --- |
| 量化粒度 | 整个权重张量共用一个scale | 每个输出通道独立一个scale |
| 精度 | 各通道范围不一致时精度差 | 精度更高，适应不同通道的分布 |
| 硬件友好度 | 高，计算简单统一 | 略低，需额外存储各通道scale |
| 存储开销 | 1个scale/张量 | C个scale（C为通道数） |


> **Note:** 实践中，权重通常使用逐通道量化，激活值使用逐层量化。因为激活值在推理时动态计算，逐通道量化会显著增加计算开销。


## 5. INT8量化精度损失分析


INT8量化引入的精度损失主要来源于以下几个方面：


- **截断误差：**
   超出量化范围的值被截断到边界值
- **舍入误差：**
   浮点数映射到离散整数时的精度丢失
- **离群值影响：**
   极端值导致有效量化范围被拉大，正常值的精度降低


> **Important:** **离群值问题（Outlier Problem）：**
> 当激活值中存在离群值时，量化范围被拉大，导致绝大多数正常值的有效位数减少。SmoothQuant方法通过将激活的量化难度转移到权重上，有效缓解这一问题。


### 精度损失缓解策略


1. 使用KL散度校准选择最优量化阈值
2. 混合精度量化：敏感层保留FP16
3. SmoothQuant、AWQ等针对LLM的改进方法
4. 逐组量化（Group Quantization）：更细粒度的量化


## 6. LLM量化方法对比


| 方法 | 量化类型 | 特点 | 适用模型 |
| --- | --- | --- | --- |
| GPTQ | 逐层INT4/INT3量化 | 基于Hessian矩阵的最优量化，误差补偿 | LLaMA、OPT等 |
| AWQ | 激活感知权重量化 | 保护重要权重通道，per-channel scaling | 各类LLM |
| GGUF | 混合精度量化 | CPU友好，支持内存映射，llama.cpp生态 | GGML生态模型 |
| SmoothQuant | W8A8量化 | 将激活量化难度转移到权重 | OPT、BLOOM等 |
| QuIP# | INT2/INT4量化 | 随机Hadamard变换+格编码 | 超低比特LLM |


> **Example:** **选择建议：**
>
>
> - GPU推理追求速度：AWQ（4-bit）+ vLLM
>
>
> - CPU推理/边缘部署：GGUF格式 + llama.cpp
>
>
> - 需要最佳精度：GPTQ 4-bit + GroupSize=128
>
>
> - 超大模型超低比特：QuIP#或AQLM


## 7. PyTorch量化代码实践


### 动态量化（Dynamic Quantization）


```
import torch
import torch.nn as nn

# 动态量化：推理时动态计算激活值的量化参数
model = nn.Sequential(
    nn.Linear(512, 256),
    nn.ReLU(),
    nn.Linear(256, 10)
)

# 仅对Linear和LSTM层做动态量化
quantized_model = torch.quantization.quantize_dynamic(
    model,
    {nn.Linear, nn.LSTM},  # 指定要量化的层类型
    dtype=torch.qint8       # 目标数据类型
)

# 查看模型大小对比
import os
torch.save(model.state_dict(), 'original.pt')
torch.save(quantized_model.state_dict(), 'quantized.pt')
print(f"原始模型: {os.path.getsize('original.pt') / 1e6:.2f} MB")
print(f"量化模型: {os.path.getsize('quantized.pt') / 1e6:.2f} MB")
```


### 静态量化（Static Quantization）


```
import torch
import torch.nn as nn
from torch.quantization import quantize_fx, QConfigMapping
import torch.ao.quantization as quant

# 定义模型
class SimpleModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.conv = nn.Conv2d(3, 16, 3, padding=1)
        self.relu = nn.ReLU()
        self.pool = nn.AdaptiveAvgPool2d(1)
        self.fc = nn.Linear(16, 10)

    def forward(self, x):
        x = self.pool(self.relu(self.conv(x)))
        x = x.view(x.size(0), -1)
        return self.fc(x)

model = SimpleModel()
model.eval()

# 配置量化参数
qconfig = quant.get_default_qconfig('fbgemm')
qconfig_mapping = QConfigMapping().set_global(qconfig)

# 准备校准数据
calibration_data = [torch.randn(1, 3, 32, 32) for _ in range(100)]

# FX图模式量化（推荐方式）
prepared_model = quantize_fx.prepare_fx(model, qconfig_mapping, calibration_data[0])

# 校准：用代表性数据统计激活分布
with torch.no_grad():
    for data in calibration_data[:100]:
        prepared_model(data)

# 转换为量化模型
quantized_model = quantize_fx.convert_fx(prepared_model)

# 推理
input_tensor = torch.randn(1, 3, 32, 32)
with torch.no_grad():
    output = quantized_model(input_tensor)
    print(f"输出形状: {output.shape}")
```


### 使用bitsandbytes进行LLM量化


```
from transformers import AutoModelForCausalLM, AutoTokenizer, BitsAndBytesConfig

# 4-bit量化配置
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_quant_type="nf4",           # NF4量化类型
    bnb_4bit_compute_dtype=torch.float16, # 计算精度
    bnb_4bit_use_double_quant=True        # 双重量化进一步压缩
)

model = AutoModelForCausalLM.from_pretrained(
    "meta-llama/Llama-2-7b-hf",
    quantization_config=bnb_config,
    device_map="auto"
)

tokenizer = AutoTokenizer.from_pretrained("meta-llama/Llama-2-7b-hf")

# 检查量化后的内存占用
print(f"模型内存: {model.get_memory_footprint() / 1e9:.2f} GB")
```


## 总结


- 量化是模型压缩中最成熟、应用最广泛的技术，通过降低数值精度减少存储和计算
- PTQ适合快速部署，QAT精度更高但需要额外训练成本
- 对称量化和非对称量化的选择取决于数据分布特性
- 逐通道量化精度优于逐层量化，权重推荐逐通道、激活推荐逐层
- LLM量化中GPTQ、AWQ、GGUF各有优势，根据部署环境选择
- PyTorch提供了完整的量化工具链，从动态量化到FX图模式量化
