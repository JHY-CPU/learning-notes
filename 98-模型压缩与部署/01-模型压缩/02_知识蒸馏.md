# 知识蒸馏 - 模型压缩与部署


## 1. 知识蒸馏框架（Hinton 2015）


知识蒸馏（Knowledge Distillation）由Hinton在2015年提出，核心思想是用一个大型的Teacher模型指导一个小型的Student模型学习，使Student模型在保持较小规模的同时，尽可能逼近Teacher的性能。


### 核心原理


- **Teacher模型：**
   大型、高性能的预训练模型，提供"暗知识"（Dark Knowledge）
- **Student模型：**
   小型、高效的模型，学习模仿Teacher的输出分布
- **软标签（Soft Label）：**
   Teacher模型经过softmax温度变换后的输出概率分布
- **硬标签（Hard Label）：**
   原始数据集中的one-hot标签


> **Note:** 软标签包含类别间的相似性信息。例如，在手写数字识别中，数字"7"可能与"1"有一定相似性，Teacher模型的软标签会反映这种关系，而硬标签则无法表达。


## 2. 温度参数与软标签


$$
Softmax with Temperature：
                q_i = exp(z_i / T) / Σ_j exp(z_j / T)
                其中 z_i 为logits，T为温度参数
$$


| 温度T | 效果 | 适用场景 |
| --- | --- | --- |
| T = 1 | 标准softmax，分布尖锐 | 推理阶段（硬预测） |
| T > 1 | 分布变平滑，揭示类别间关系 | 蒸馏训练（推荐T=3~20） |
| T → ∞ | 均匀分布，丢失信息 | 不实用 |
| T → 0 | 退化为one-hot | 等价于硬标签 |


> **Example:** **直观理解：**
> 温度参数就像"放大镜的反向"——温度越高，Teacher输出的概率分布越平滑，暴露更多类别间的关系信息。例如，logits = [5, 1, 0]，T=1时softmax为[0.93, 0.05, 0.02]；T=5时变为[0.55, 0.27, 0.18]，类别间的关系更明显。


## 3. 蒸馏损失函数


$$
总损失： L = α · L_hard + (1 - α) · T² · L_soft
硬标签损失： L_hard = CE(y, Student(x))
软标签损失： L_soft = KL(Teacher_T(x) || Student_T(x))
$$


- **α：**
   平衡系数，通常取0.1~0.5，控制硬标签和软标签的权重
- **T²因子：**
   因为温度T使梯度缩小了T²倍，需要乘回来保持梯度量级一致
- **KL散度：**
   衡量Teacher和Student输出分布的差异


> **Important:** **α的选择：**
> 较小的α（如0.1）意味着更多依赖Teacher的软标签，适合Teacher和Student差异较大时；较大的α（如0.5）保留更多真实标签信息，适合数据噪声较少的场景。


## 4. 特征蒸馏方法


| 方法 | 蒸馏位置 | 核心思想 | 代表论文 |
| --- | --- | --- | --- |
| FitNets | 中间层特征 | Student学习Teacher中间层的表示 | ICLR 2015 |
| Attention Transfer | 注意力图 | Student学习Teacher的注意力分布 | ICLR 2017 |
| NST | 风格特征 | 神经风格迁移思想，对齐特征统计量 | NeurIPS 2017 |
| CRD | 表示空间 | 对比学习，拉近同一样本的师生表示 | ICML 2020 |
| ReviewKD | 多层次特征 | 构建特征金字塔进行多尺度蒸馏 | CVPR 2021 |


$$
Attention Transfer损失：
                L_AT = ||F_s / ||F_s||_2 - F_t / ||F_t||_2||_2
                其中 F_s, F_t 分别为Student和Teacher的特征图在空间维度的均值
$$


## 5. 自蒸馏与高级蒸馏


### 自蒸馏（Self-Distillation）


自蒸馏不依赖额外的Teacher模型，而是使用模型自身作为Teacher，常见方式包括：


- **深层→浅层：**
   用深层特征监督浅层特征（如Born-Again Networks）
- **数据增强蒸馏：**
   用原始样本的预测作为增强样本的Teacher
- **在线蒸馏：**
   训练过程中多个分支互相蒸馏（如BYOT）
- **Better Teacher：**
   使用训练后期更好的checkpoint蒸馏早期checkpoint


> **Note:** 自蒸馏的优势在于不需要维护额外的大模型，计算开销可控，且能持续提升模型性能。特别适合模型迭代优化阶段。


### 在NLP和CV中的应用


| 领域 | 应用 | 典型方法 |
| --- | --- | --- |
| NLP | BERT蒸馏为小模型 | DistilBERT、TinyBERT、MobileBERT |
| NLP | LLM蒸馏到小模型 | Alpaca、Vicuna（用GPT-4蒸馏） |
| CV | 大模型蒸馏为移动端模型 | ResNet→MobileNet |
| CV | 检测器蒸馏 | Feature Mimicking、GD |


## 6. PyTorch知识蒸馏实现


```
import torch
import torch.nn as nn
import torch.nn.functional as F

class DistillationLoss(nn.Module):
    """知识蒸馏损失函数"""
    def __init__(self, temperature=4.0, alpha=0.3):
        super().__init__()
        self.temperature = temperature
        self.alpha = alpha
        self.ce_loss = nn.CrossEntropyLoss()

    def forward(self, student_logits, teacher_logits, labels):
        # 硬标签损失
        hard_loss = self.ce_loss(student_logits, labels)

        # 软标签损失（KL散度）
        soft_student = F.log_softmax(student_logits / self.temperature, dim=1)
        soft_teacher = F.softmax(teacher_logits / self.temperature, dim=1)
        soft_loss = F.kl_div(
            soft_student, soft_teacher,
            reduction='batchmean'
        ) * (self.temperature ** 2)

        # 总损失
        total_loss = self.alpha * hard_loss + (1 - self.alpha) * soft_loss
        return total_loss, hard_loss, soft_loss


def train_with_distillation(teacher, student, train_loader, optimizer,
                             epochs=10, temperature=4.0, alpha=0.3):
    """蒸馏训练循环"""
    device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
    teacher.to(device).eval()   # Teacher设为评估模式
    student.to(device).train()

    criterion = DistillationLoss(temperature, alpha)

    for epoch in range(epochs):
        total_loss = 0
        correct = 0
        total = 0

        for inputs, labels in train_loader:
            inputs, labels = inputs.to(device), labels.to(device)

            # Teacher前向传播（不计算梯度）
            with torch.no_grad():
                teacher_logits = teacher(inputs)

            # Student前向传播
            student_logits = student(inputs)

            # 计算蒸馏损失
            loss, hard_loss, soft_loss = criterion(
                student_logits, teacher_logits, labels
            )

            # 反向传播与优化
            optimizer.zero_grad()
            loss.backward()
            optimizer.step()

            total_loss += loss.item()
            _, predicted = student_logits.max(1)
            total += labels.size(0)
            correct += predicted.eq(labels).sum().item()

        acc = 100. * correct / total
        print(f"Epoch {epoch+1}/{epochs} | "
              f"Loss: {total_loss/len(train_loader):.4f} | "
              f"Acc: {acc:.2f}%")

    return student
```

## 总结


- 知识蒸馏通过Teacher-Student框架，将大模型知识转移到小模型
- 温度参数T控制软标签的平滑程度，T越大揭示的暗知识越多
- 损失函数由硬标签损失和软标签损失加权组成，α控制两者的平衡
- 特征蒸馏（FitNets、Attention Transfer）从中间层提取知识
- 自蒸馏不需要额外Teacher模型，适合迭代优化
- NLP中DistilBERT、TinyBERT是蒸馏的经典应用；LLM时代，蒸馏用于训练指令跟随小模型
