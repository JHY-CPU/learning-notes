# 模型剪枝 - 模型压缩与部署


## 1. 剪枝概述与分类


模型剪枝（Pruning）通过移除神经网络中不重要的权重或结构，在保持模型性能的同时减少参数量和计算量。根据剪枝的粒度，可分为结构化剪枝和非结构化剪枝两大类。


| 对比维度 | 非结构化剪枝 | 结构化剪枝 |
| --- | --- | --- |
| 剪枝粒度 | 单个权重/神经元 | 整个通道/注意力头/层 |
| 稀疏模式 | 不规则稀疏矩阵 | 规则的结构化稀疏 |
| 硬件加速 | 需要专用稀疏硬件（如NVIDIA A100） | 通用硬件即可加速 |
| 压缩率上限 | 可达到90%+稀疏度 | 通常50-70% |
| 实现难度 | 简单，置零即可 | 需要重新设计网络结构 |
| 推理加速 | 不一定有加速（依赖硬件/库） | 直接减少FLOPs，有实际加速 |


## 2. 权重剪枝（Magnitude Pruning）


最简单直接的剪枝方法，根据权重绝对值大小决定是否剪枝。


$$
剪枝规则：
                w_i = 0, if |w_i| < threshold
                其中 threshold 通常取权重绝对值的某个百分位数
$$


### 剪枝策略


- **One-Shot剪枝：**
   一次性剪枝到目标稀疏度，简单但可能造成较大精度损失
- **迭代剪枝（Iterative Pruning）：**
   逐步增加稀疏度，每轮剪枝后微调恢复精度
- **逐渐稀疏（Gradual Sparsity）：**
   训练过程中按照余弦调度逐步增加稀疏度


> **Note:** **迭代剪枝流程：**
> 训练→剪枝10%→微调→剪枝10%→微调→...→达到目标稀疏度。每次只剪少量，通过微调恢复精度，最终精度损失远小于One-Shot剪枝。


$$
逐渐稀疏调度（Cubic Schedule）：
                s_t = s_f + (s_i - s_f) × (1 - t/T)³
                其中 s_i=0, s_f=目标稀疏度, t为当前步数, T为总步数
$$


## 3. 通道剪枝


通道剪枝移除整个卷积核或注意力头，直接减少模型的宽度，是结构化剪枝中最常用的方法。


### 通道重要性评估指标


| 评估指标 | 计算方式 | 优点 |
| --- | --- | --- |
| L1范数 | Σ\|w\| (通道所有权重绝对值之和) | 简单高效 |
| L2范数 | √(Σw²) (通道权重的欧几里得范数) | 对大权重更敏感 |
| BN缩放因子 | BatchNorm层的γ参数绝对值 | 无需额外计算 |
| 梯度信息 | 基于Taylor展开的一阶/二阶项 | 考虑任务相关性 |
| FLOPs/延迟 | 实际移除后的计算量减少 | 直接优化目标指标 |


> **Example:** **BN通道剪枝（Network Slimming）：**
> 利用BN层的缩放因子γ作为通道重要性的代理指标。γ值小说明该通道的特征响应弱，可以安全移除。通过L1正则化在训练时约束γ稀疏化。


## 4. 彩票假说（Lottery Ticket Hypothesis）


由Frankle和Carlin在2019年ICLR最佳论文中提出，是剪枝领域最重要的理论发现之一。


$$
彩票假说核心论断：
                随机初始化的密集网络包含一个子网络（中奖彩票），
                若以相同的初始化权重从头训练，该子网络
                在相同迭代次数内可以达到与原始网络相当的精度。
$$


### 发现中奖彩票的算法


1. 随机初始化网络权重 w_0
2. 训练网络至收敛，得到权重 w_t
3. 按权重绝对值剪枝 p% 的最小权重
4. 记录剩余权重对应的初始权重索引（即中奖彩票 mask）
5. 用原始初始化值 w_0 重置剩余权重
6. 回到步骤2迭代，直到达到目标稀疏度


> **Important:** **彩票假说的意义：**
> 它证明了密集网络中存在可训练的稀疏子结构，且初始化方式对稀疏网络的训练至关重要。随机初始化打乱权重后重新训练，精度会大幅下降。


### 后续发展


- **IMP（Iterative Magnitude Pruning）：**
   标准迭代剪枝流程
- **Lottery Ticket在LLM中：**
   大模型的剪枝通常结合权重共享和量化
- **Supermasks：**
   不改变权重，仅通过mask实现稀疏网络


## 5. 稀疏化训练与微调恢复


### 稀疏化训练


在训练过程中同时学习权重和稀疏模式，代表方法包括SET（Sparse Evolutionary Training）和RigL。


- **SET：**
   定期剪枝最小权重，随机生长新连接
- **RigL：**
   基于梯度信息选择要生长的连接，更高效
- **STE（Straight-Through Estimator）：**
   前向传播使用离散mask，反向传播时允许梯度通过被剪枝的权重


### 剪枝后微调策略


| 微调策略 | 描述 | 适用场景 |
| --- | --- | --- |
| 全局微调 | 所有未剪枝权重一起微调 | 通用场景 |
| 逐层微调 | 逐层解冻微调 | 迁移学习场景 |
| 被剪枝权重恢复 | 允许被置零的权重重新非零 | 过度剪枝时 |
| 知识蒸馏辅助 | 用原始模型做Teacher蒸馏剪枝后的模型 | 高精度要求 |


## 6. PyTorch剪枝代码实践


```
import torch
import torch.nn as nn
import torch.nn.utils.prune as prune

# 定义模型
class SimpleCNN(nn.Module):
    def __init__(self):
        super().__init__()
        self.conv1 = nn.Conv2d(3, 64, 3, padding=1)
        self.conv2 = nn.Conv2d(64, 128, 3, padding=1)
        self.fc1 = nn.Linear(128 * 8 * 8, 256)
        self.fc2 = nn.Linear(256, 10)
        self.relu = nn.ReLU()
        self.pool = nn.MaxPool2d(2)

    def forward(self, x):
        x = self.pool(self.relu(self.conv1(x)))
        x = self.pool(self.relu(self.conv2(x)))
        x = x.view(x.size(0), -1)
        x = self.relu(self.fc1(x))
        return self.fc2(x)

model = SimpleCNN()

# ========== 1. 非结构化剪枝（权重级别） ==========
# 对conv1层应用L1非结构化剪枝，剪掉30%的权重
prune.l1_unstructured(model.conv1, name='weight', amount=0.3)

# 查看稀疏度
weight = model.conv1.weight
sparsity = 100. * float(torch.sum(weight == 0)) / float(weight.nelement())
print(f"conv1权重稀疏度: {sparsity:.1f}%")

# ========== 2. 结构化剪枝（通道级别） ==========
# 对conv2层按L2范数剪枝25%的输出通道
prune.ln_structured(
    model.conv2, name='weight', amount=0.25,
    n=2, dim=0  # 按L2范数，沿输出通道维度
)

# ========== 3. 全局剪枝 ==========
# 全局剪枝：对全网络所有层统一剪枝40%
parameters_to_prune = [
    (model.conv1, 'weight'),
    (model.conv2, 'weight'),
    (model.fc1, 'weight'),
    (model.fc2, 'weight'),
]
prune.global_unstructured(
    parameters_to_prune,
    pruning_method=prune.L1Unstructured,
    amount=0.4,
)

# ========== 4. 逐渐稀疏训练 ==========
def gradual_sparsity_train(model, train_loader, optimizer, criterion,
                           target_sparsity=0.9, total_epochs=100):
    """逐渐稀疏训练"""
    for epoch in range(total_epochs):
        # 计算当前目标稀疏度（余弦调度）
        current_sparsity = target_sparsity * (
            1 - (1 - epoch / total_epochs) ** 3
        )

        # 应用全局剪枝
        prune.global_unstructured(
            parameters_to_prune,
            pruning_method=prune.L1Unstructured,
            amount=current_sparsity,
        )

        # 正常训练一个epoch
        model.train()
        for inputs, targets in train_loader:
            optimizer.zero_grad()
            outputs = model(inputs)
            loss = criterion(outputs, targets)
            loss.backward()
            optimizer.step()

        if (epoch + 1) % 10 == 0:
            total_zeros = sum(
                torch.sum(p == 0).item()
                for _, p in parameters_to_prune
            )
            total_params = sum(
                p.nelement() for _, p in parameters_to_prune
            )
            print(f"Epoch {epoch+1} | Sparsity: "
                  f"{100*total_zeros/total_params:.1f}%")

# ========== 5. 永久移除剪枝mask ==========
# 使剪枝永久生效（移除forward hook，将mask应用到权重）
for module, name in parameters_to_prune:
    prune.remove(module, name)

# 保存剪枝后的模型
# torch.save(model.state_dict(), 'pruned_model.pth')

print("剪枝完成！")
```


## 总结


- 模型剪枝通过移除不重要的权重或结构来压缩模型，分为非结构化和结构化两大类
- 权重剪枝（Magnitude Pruning）是最简单的方法，迭代剪枝优于一次性剪枝
- 通道剪枝可直接减少FLOPs，BN缩放因子是有效的通道重要性指标
- 彩票假说证明密集网络中存在可训练的稀疏子网络，初始化方式至关重要
- 稀疏化训练（SET、RigL）在训练过程中自动学习稀疏模式
- 剪枝后的微调对恢复精度至关重要，可结合知识蒸馏辅助恢复
- PyTorch的torch.nn.utils.prune模块提供了完整的剪枝工具链
