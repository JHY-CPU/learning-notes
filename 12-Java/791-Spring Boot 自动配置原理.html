🔧 Spring Boot 自动配置原理

@EnableAutoConfiguration 加载机制、@Conditional 条件装配、AutoConfiguration.imports、自定义 Starter、运行原理全链路。

自动配置机制

// ========== Spring Boot 自动配置 ========== // 核心: 根据 classpath 依赖、配置属性、条件, 自动创建 Bean // 无需手动 @Bean 声明 // ========== 启动流程 ========== @SpringBootApplication // → @EnableAutoConfiguration // → AutoConfigurationImportSelector // → 加载 META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports // → 解析所有自动配置类 // → 按 @Conditional 条件匹配 // → 匹配成功 → 创建 Bean // → 匹配失败 → 跳过 // ========== AutoConfiguration.imports ========== // 在 spring-boot-autoconfigure jar 中 // META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports // 列出所有自动配置类 (约 150+) // 内容示例: org.springframework.boot.autoconfigure.web.servlet.DispatcherServletAutoConfiguration org.springframework.boot.autoconfigure.orm.jpa.HibernateJpaAutoConfiguration org.springframework.boot.autoconfigure.data.redis.RedisAutoConfiguration org.springframework.boot.autoconfigure.security.servlet.SecurityAutoConfiguration // ========== 条件生效过程 ========== // 1. 加载所有 AutoConfiguration 类 // 2. 按 @AutoConfigureOrder / @AutoConfigureAfter / @AutoConfigureBefore 排序 // 3. 逐条判断 @Conditional 条件 // 4. 条件满足 → 执行 @Bean 方法, 注册 Bean // 5. 条件不满足 → 跳过 // ========== 调试自动配置 ========== // 查看哪些自动配置生效/未生效: application.yml: debug: true // 或 --debug // 输出: // ============================ // AUTO-CONFIGURATION REPORT // ============================ // Positive matches: (已生效) // DataSourceAutoConfiguration matched: // - @ConditionalOnClass found required class 'javax.sql.DataSource' // // Negative matches: (未生效) // RedisAutoConfiguration did not match: // - @ConditionalOnClass did not find required class 'redis.clients.jedis.Jedis'

@Conditional 家族

// ========== @Conditional 条件注解 ========== // 控制 Bean 是否注册的核心机制 // ========== 常用条件注解 ========== // @ConditionalOnClass — classpath 存在某类时才生效 @Configuration @ConditionalOnClass(DataSource.class) // 存在 DataSource 才加载 public class DataSourceAutoConfiguration { ... } // @ConditionalOnMissingClass — classpath 不存在某类 @Configuration @ConditionalOnMissingClass("org.springframework.web.servlet.DispatcherServlet") public class NonWebConfiguration { ... } // @ConditionalOnBean — 存在某 Bean 时才生效 @Bean @ConditionalOnBean(JdbcTemplate.class) // 已有 JdbcTemplate Bean public JdbcTemplateQuerier querier(JdbcTemplate jdbcTemplate) { ... } // @ConditionalOnMissingBean — 不存在某 Bean 才生效 @Bean @ConditionalOnMissingBean // 用户未自定义时才创建默认 public DataSource dataSource() { return new HikariDataSource(); // 默认数据源 } // @ConditionalOnProperty — 配置属性匹配时才生效 @Configuration @ConditionalOnProperty( prefix = "myapp.feature", name = "enabled", havingValue = "true", matchIfMissing = false ) public class FeatureAutoConfiguration { ... } // @ConditionalOnExpression — SpEL 表达式 @Bean @ConditionalOnExpression("${myapp.cache.enabled:true} and '${spring.profiles.active}' != 'dev'") public CacheManager cacheManager() { ... } // @ConditionalOnResource — 存在资源文件 @ConditionalOnResource(resources = "classpath:custom-config.yml") // @ConditionalOnWebApplication — 是 Web 应用 @ConditionalOnWebApplication(type = ConditionalOnWebApplication.Type.SERVLET) // @ConditionalOnSingleCandidate — 只有一个候选 Bean // ========== 组合示例 ========== // RedisAutoConfiguration 实际源码简化版: @Configuration @ConditionalOnClass(RedisOperations.class) // classpath 有 Redis 依赖 @EnableConfigurationProperties(RedisProperties.class) // 绑定 redis.* 配置 public class RedisAutoConfiguration { @Bean @ConditionalOnMissingBean(name = "redisTemplate") // 用户未自定义 @ConditionalOnSingleCandidate(RedisConnectionFactory.class) public RedisTemplate<Object, Object> redisTemplate( RedisConnectionFactory redisConnectionFactory) { RedisTemplate<Object, Object> template = new RedisTemplate<>(); template.setConnectionFactory(redisConnectionFactory); return template; } }

创建自定义 Starter

// ========== 自定义 Starter ========== // 封装可复用的自动配置库 // 命名: xxx-spring-boot-starter // ========== 项目结构 ========== my-email-starter/ ├── pom.xml └── src/main/java/com/example/email/ ├── EmailProperties.java // 配置属性 ├── EmailService.java // 服务类 └── EmailAutoConfiguration.java // 自动配置 // resources/META-INF/spring/ // └── org.springframework.boot.autoconfigure.AutoConfiguration.imports // ========== 1. 配置属性 ========== @ConfigurationProperties(prefix = "myapp.email") @Data public class EmailProperties { private String host = "localhost"; private int port = 25; private String username; private String password; private boolean enabled = true; } // ========== 2. 服务类 ========== @Slf4j public class EmailService { private final EmailProperties properties; public EmailService(EmailProperties properties) { this.properties = properties; } public void send(String to, String subject, String body) { if (!properties.isEnabled()) { log.warn("邮件服务已禁用"); return; } log.info("发送邮件到 {}: {}", to, subject); // 实际发送逻辑... } } // ========== 3. 自动配置类 ========== @Configuration @ConditionalOnClass(EmailService.class) @EnableConfigurationProperties(EmailProperties.class) public class EmailAutoConfiguration { @Bean @ConditionalOnMissingBean public EmailService emailService(EmailProperties properties) { return new EmailService(properties); } } // ========== 4. AutoConfiguration.imports ========== // META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports // 内容: com.example.email.EmailAutoConfiguration // ========== 5. spring.factories (旧版) ========== // Spring Boot 2.x 及之前使用: // META-INF/spring.factories // org.springframework.boot.autoconfigure.EnableAutoConfiguration=\ // com.example.email.EmailAutoConfiguration // ========== 6. 使用 Starter ========== // 引入依赖即可: // <dependency> // <groupId>com.example</groupId> // <artifactId>my-email-starter</artifactId> // </dependency> // application.yml 配置: myapp: email: host: smtp.gmail.com port: 587 username: ${EMAIL_USER} password: ${EMAIL_PASS}

自动配置全链路分析

// ========== 自动配置全链路 (以 Web 为例) ========== // 1. @SpringBootApplication @SpringBootApplication // → @EnableAutoConfiguration // 2. @EnableAutoConfiguration // 导入 AutoConfigurationImportSelector // 从 META-INF/spring/...AutoConfiguration.imports 加载类名 // 3. AutoConfigurationImportSelector // - 读取 AutoConfiguration.imports // - 过滤: 根据 @Conditional 条件 // - 排序: @AutoConfigureOrder / @AutoConfigureAfter // - 去重 // 4. ServletWebServerFactoryAutoConfiguration // @ConditionalOnClass(ServletRequest.class) // → 存在 Servlet API → 匹配 // 5. TomcatServletWebServerFactoryCustomizer // @ConditionalOnClass(Tomcat.class) // → 存在 Tomcat (spring-boot-starter-web 包含) → 创建内嵌 Tomcat // 6. DispatcherServletAutoConfiguration // @ConditionalOnClass(DispatcherServlet.class) // → 自动配置 DispatcherServlet + Spring MVC // 7. 最终效果: // 只引入 spring-boot-starter-web // → 自动配置: 内嵌 Tomcat + DispatcherServlet + Jackson + 错误处理 + ... // ========== 关键自动配置类 ========== // ┌────────────────────────────┬────────────────────────────────┐ // │ 自动配置类 │ 条件 │ // ├────────────────────────────┼────────────────────────────────┤ // │ DataSourceAutoConfig │ DataSource + HikariCP │ // │ HibernateJpaAutoConfig │ EntityManager + Hibernate │ // │ RedisAutoConfiguration │ RedisOperations │ // │ MongoAutoConfiguration │ MongoClient │ // │ SecurityAutoConfiguration │ SecurityFilterChain │ // │ RabbitAutoConfiguration │ ConnectionFactory │ // │ KafkaAutoConfiguration │ KafkaTemplate │ // │ MailSenderAutoConfig │ JavaMailSender │ // └────────────────────────────┴────────────────────────────────┘ // ========== 如何禁用自动配置 ========== // 方式 1: exclude @SpringBootApplication(exclude = { DataSourceAutoConfiguration.class, SecurityAutoConfiguration.class }) // 方式 2: 配置文件 spring.autoconfigure.exclude=org.springframework.boot.autoconfigure.jdbc.DataSourceAutoConfiguration // 方式 3: 条件属性 spring.datasource.url=jdbc:... // 设置数据源时自动配置生效

条件评估与最佳实践

// ========== 条件评估报告 ========== // 启用调试: debug: true // 或在 application.yml 中: logging: level: org.springframework.boot.autoconfigure: TRACE // 查看输出: // Positive matches: 已生效的自动配置 // Negative matches: 未生效的自动配置 (含原因) // Exclusions: 被排除的自动配置 // Unconditional classes: 无条件直接注册的类 // ========== 自动配置设计原则 ========== // 1. 优先使用 @ConditionalOnMissingBean // 让用户自定义 Bean 覆盖自动配置 @Bean @ConditionalOnMissingBean public MyService myService() { ... } // 用户可自定义覆盖 // 2. 提供配置属性开关 @ConditionalOnProperty(prefix = "myapp.feature", name = "enabled", matchIfMissing = true) // 3. 合理排序 @AutoConfigureAfter(DataSourceAutoConfiguration.class) @AutoConfigureBefore(WebMvcAutoConfiguration.class) // 4. 不要自动配置过多 // 只自动配置框架级的 Bean, 业务 Bean 留给用户 // 5. 使用 @ConfigurationProperties 绑定属性 // ========== 常见问题 ========== // Q: 自动配置未生效? // A: 检查 AutoConfiguration.imports 是否正确 // 检查 @Conditional 条件是否满足 // Q: 循环依赖? // A: 使用 @Lazy 或重新设计 Bean 依赖关系 // Q: 覆盖自动配置? // A: 定义同类型的 @Bean, @ConditionalOnMissingBean 会跳过自动配置
💡 自动配置要点: @EnableAutoConfiguration 加载 AutoConfiguration.imports 中类; @ConditionalOnClass/Bean/Property/MissingBean 条件控制; AutoConfigurationImportSelector 排序+过滤; 自定义 Starter 封装可复用功能; debug:true 查看生效报告; @ConditionalOnMissingBean 允许用户覆盖; 条件不满足时自动跳过。
