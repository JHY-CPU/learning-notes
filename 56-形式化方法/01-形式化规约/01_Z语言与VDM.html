Z语言与VDM形式化规约

一、形式化方法概述

形式化方法是基于数学的技术，用于软件系统的规约、开发和验证。它使用严格的数学符号和推理来描述系统行为，从而提高软件的可靠性。

1.1 形式化方法的分类

类别方法特点
形式化规约Z、VDM、B方法、Alloy描述系统应该做什么
形式化验证模型检验、定理证明验证系统满足规约
精化方法B方法、Event-B从抽象规约到具体实现
形式化方法的优势：消除歧义、支持推理和证明、减少缺陷、提高安全性关键系统的可靠性。
挑战：学习曲线陡峭、工具支持有限、不适合所有项目。

二、Z语言（Z Notation）

Z语言是由牛津大学开发的形式化规约语言，使用集合论和一阶谓词逻辑来描述系统状态和操作。

2.1 核心概念

  • 集合（Set）：无序、不重复元素的集合
  • 关系（Relation）：两个集合之间的映射关系
  • 函数（Function）：特殊的关系，每个输入对应唯一输出
  • 序列（Sequence）：有序的元素列表
  • 包（Bag）：允许重复元素的无序集合

2.2 模式（Schema）

模式是Z语言的核心结构，用于描述系统的状态和操作。

状态模式

示例：图书馆系统状态模式
┌─ Library ─────────────────────┐
│ books : P BOOK                │
│ members : P MEMBER            │
│ loaned : BOOK ↔ MEMBER       │
├───────────────────────────────┤
│ books ≠ ∅                    │
│ members ≠ ∅                  │
│ dom loaned ⊆ books           │
│ ran loaned ⊆ members         │
└───────────────────────────────┘
上半部分声明状态变量，下半部分定义状态不变式。

操作模式

示例：借书操作
┌─ BorrowBook ──────────────────┐
│ ΔLibrary                      │
│ b? : BOOK                     │
│ m? : MEMBER                   │
├───────────────────────────────┤
│ b? ∈ books                    │
│ m? ∈ members                  │
│ b? ∉ dom loaned               │
│ loaned' = loaned ∪ {b? ↦ m?} │
└───────────────────────────────┘
ΔLibrary表示此操作会修改Library状态。?后缀表示输入，'后缀表示操作后的新值。

2.3 Z语言的记号约定

符号含义
P S集合S的幂集
S ↔ TS到T的关系（S × T的子集）
dom R关系R的定义域
ran R关系R的值域
x?输入变量
x!输出变量
x'操作后的变量值
ΔS状态S可能发生改变
ΞS状态S不变（Xi记号）

三、VDM（Vienna Development Method）

VDM是由IBM维也纳实验室开发的形式化开发方法，强调从抽象模型逐步精化到可执行代码。

3.1 VDM-SL类型系统

类型语法示例
自然数nat / nat1age: nat
整数intbalance: int
实数realprice: real
布尔boolactive: bool
字符charinitial: char
字符串seq of charname: seq of char
集合set of Tids: set of nat
序列seq of Titems: seq of Item
映射map K to Vdb: map Id to Record

3.2 前置条件与后置条件

VDM操作通过前置条件（pre-）和后置条件（post-）来定义行为。

示例：栈的压入操作
push: Stack * Elem ==> Stack
push(st, e) == st ^ [e]
pre len st < MAX_SIZE
post st' = st ^ [e]
前置条件：栈未满
后置条件：新栈是原栈加上新元素

3.3 数据精化（Data Refinement）

数据精化是从抽象数据类型到具体数据类型的转换过程，保持操作行为的等价性。

  1. 定义抽象模型（如用集合表示）
  2. 定义具体模型（如用数组实现）
  3. 建立抽象与具体之间的映射关系（Retrieve Function）
  4. 证明具体操作与抽象操作的等价性
