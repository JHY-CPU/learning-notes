# 时序逻辑与模型检验


## 时序逻辑与模型检验

一、模型检验概述

## 一、模型检验（Model Checking）概述


模型检验是一种自动化的形式化验证技术，通过穷尽搜索系统状态空间来验证系统是否满足给定的规约。


> **Note:** **核心思想：**
> 将系统建模为状态转移系统，将属性表达为时序逻辑公式，通过算法检查模型是否满足公式。


### 1.1 模型检验流程


1. **建模：**
   用形式化语言描述系统行为（如Kripke结构、自动机）
2. **规约：**
   用时序逻辑表达期望的系统属性
3. **验证：**
   运行模型检验器，搜索状态空间
4. **分析：**
   如果不满足，分析反例（Counterexample）


### 1.2 优缺点


| 优点 | 缺点 |
| --- | --- |
| 完全自动化 | 状态空间爆炸问题 |
| 能提供反例 | 只能验证有限状态系统 |
| 穷尽搜索所有可能 | 建模需要专业知识 |
| 适合并发系统验证 | 规约编写有学习曲线 |

二、时序逻辑

## 二、时序逻辑（Temporal Logic）


时序逻辑是描述系统随时间变化行为的逻辑系统，是模型检验中表达属性的主要工具。


### 2.1 LTL（线性时序逻辑）


LTL将时间视为线性的路径（每一步只有一个后继），描述沿单一执行路径的属性。


| 算子 | 符号 | 含义 | 示例 |
| --- | --- | --- | --- |
| Next | `X p` | 下一时刻p为真 | `X green` |
| Finally/Eventually | `F p` | 将来某个时刻p为真 | `F served` |
| Globally/Always | `G p` | 所有时刻p为真 | `G (req → F ack)` |
| Until | `p U q` | p一直为真直到q为真 | `wait U ready` |
| Release | `p R q` | q为真直到p和q同时为真 | `error R safe` |

**常见LTL属性模式：**


- 活性（Liveness）：
`F p`
（某事最终会发生）


- 安全性（Safety）：
`G p`
（坏事永远不会发生）


- 响应性（Response）：
`G (request → F response)`
（每个请求最终得到响应）


- 公平性（Fairness）：
`G F p`
（p无限经常发生）

### 2.2 CTL（计算树时序逻辑）


CTL将时间视为分支的树结构（每一步可能有多个后继），使用路径量词结合时序算子。


| 路径量词 | 含义 | 时序算子 | 含义 |
| --- | --- | --- | --- |
| `A` | 所有路径（All paths） | `X` | 下一状态 |
| `E` | 存在路径（Exists path） | `F` | 将来某状态 |
|  |  | `G` | 所有未来状态 |
|  |  | `U` | Until |


### 2.3 常见CTL公式


| 公式 | 含义 | 属性类型 |
| --- | --- | --- |
| `AG p` | 所有路径上p始终为真 | 安全性 |
| `AF p` | 所有路径上p最终为真 | 活性 |
| `EG p` | 存在一条路径p始终为真 | 不变式可能性 |
| `EF p` | 存在一条路径p最终为真 | 可达性 |
| `AG(req → AF ack)` | 每个请求最终得到确认 | 响应性 |
| `EF deadlock` | 可能到达死锁状态 | 死锁检查 |


### 2.4 LTL vs CTL


| 维度 | LTL | CTL |
| --- | --- | --- |
| 时间结构 | 线性路径 | 分支树 |
| 表达能力 | 不可比较 | 不可比较 |
| 验证算法 | 基于自动机 | 基于标记算法 |
| 工具支持 | SPIN | NuSMV |


> **Note:** **CTL*：**
> CTL*统一了LTL和CTL，允许路径量词和时序算子自由组合。LTL和CTL都是CTL*的真子集，但彼此不可比较。

三、SPIN模型检验器

## 三、SPIN模型检验器


SPIN是贝尔实验室开发的LTL模型检验器，使用Promela语言建模并发系统。


### 3.1 Promela建模要素


- **进程（proctype）：**
   定义并发执行的进程
- **通道（chan）：**
   进程间通信的消息通道
- **变量：**
   全局和局部状态变量
- **选择（select）：**
   非确定性选择


### 3.2 验证方式


- **断言检查（Assertion Checking）：**
   验证assert语句是否永远为真
- **死锁检测：**
   检查系统是否可能进入死锁
- **LTL验证：**
   验证LTL公式是否满足
- **不变式检查：**
   验证全局不变式

四、状态空间爆炸

## 四、状态空间爆炸问题


状态空间爆炸是模型检验面临的主要挑战，系统状态数随组件数指数增长。


### 4.1 解决方案


| 技术 | 原理 |
| --- | --- |
| 符号模型检验 | 使用BDD（二元决策图）表示状态集 |
| 有界模型检验（BMC） | 限制搜索深度，使用SAT求解器 |
| 偏序归约 | 利用并发独立性减少需探索的状态数 |
| 对称归约 | 利用系统对称性合并等价状态 |
| 抽象 | 构建系统的抽象模型，减少状态数 |
| 组合验证 | 逐组件验证，使用假设-保证推理 |

**假设-保证推理（Assume-Guarantee）：**


形式：
`[Assumption] Component [Guarantee]`


例如：假设环境满足请求间隔>10ms，组件保证响应时间<5ms。
