# 观察者模式与事件系统

## 核心概念

观察者模式定义一对多的依赖关系，当一个对象状态变化时，所有依赖它的对象自动收到通知。在游戏开发中，这一模式演化为事件系统（Event System / Event Bus），实现模块间的解耦通信，是游戏架构中最重要的解耦手段之一。

### 为什么需要事件系统？

没有事件系统时，模块间会产生强耦合：

```csharp
// 紧耦合：战斗系统直接调用UI和音效
class CombatSystem
{
    void DealDamage(Character target, float damage)
    {
        target.Health -= damage;
        uiManager.UpdateHealthBar(target);           // 战斗系统依赖UI
        audioManager.PlayHitSound(target.Position);  // 战斗系统依赖音效
        achievementSystem.CheckDamageDealt(damage);  // 战斗系统依赖成就
        vfxManager.SpawnHitEffect(target.Position);  // 战斗系统依赖特效
        networkManager.SyncDamage(target, damage);   // 战斗系统依赖网络
        // 新增功能？继续加调用...这就是意大利面条代码
    }
}
```

解耦后——战斗系统只需要发布事件：

```csharp
class CombatSystem
{
    void DealDamage(Character target, float damage)
    {
        target.Health -= damage;
        EventBus.Publish(new DamageEvent { Target = target, Amount = damage });
        // 完毕。不关心谁在监听。
    }
}
```

### 完整事件总线实现

```csharp
using System;
using System.Collections.Generic;
using System.Linq;

// ========== 事件基类 ==========
public class GameEvent
{
    public float Timestamp;
    public GameEvent()
    {
        Timestamp = UnityEngine.Time.time;
    }
}

// ========== 具体事件定义 ==========
public class DamageEvent : GameEvent
{
    public GameObject Attacker;
    public GameObject Target;
    public float Amount;
    public DamageType Type;
}

public class PlayerDeathEvent : GameEvent
{
    public GameObject Player;
    public GameObject Killer;
}

public class ItemPickupEvent : GameEvent
{
    public GameObject Player;
    public ItemData Item;
    public int Quantity;
}

public class ScoreChangedEvent : GameEvent
{
    public int NewScore;
    public int Delta;
}

public class LevelUpEvent : GameEvent
{
    public GameObject Character;
    public int NewLevel;
}

public enum DamageType { Physical, Fire, Ice, Poison }

// ========== 优先级事件总线 ==========
public class EventBus
{
    // 每种事件类型 → 按优先级排序的订阅者列表
    private static Dictionary<Type, List<EventSubscription>> listeners = new();

    // 一次性订阅（触发一次后自动取消）
    private static HashSet<EventSubscription> onceListeners = new();

    // 事件队列（支持延迟处理，避免在回调中修改集合）
    private static Queue<GameEvent> eventQueue = new();
    private static bool isProcessing = false;

    /// <summary>
    /// 订阅事件
    /// </summary>
    /// <param name="handler">回调函数</param>
    /// <param name="priority">优先级，数值越小越先执行</param>
    /// <typeparam name="T">事件类型</typeparam>
    public static EventSubscription Subscribe<T>(
        Action<T> handler,
        int priority = 0) where T : GameEvent
    {
        var type = typeof(T);
        if (!listeners.ContainsKey(type))
            listeners[type] = new List<EventSubscription>();

        var subscription = new EventSubscription
        {
            Handler = evt => handler((T)evt),
            Priority = priority,
            EventType = type
        };

        listeners[type].Add(subscription);
        // 保持按优先级排序
        listeners[type].Sort((a, b) => a.Priority.CompareTo(b.Priority));

        return subscription;
    }

    /// <summary>
    /// 一次性订阅：触发后自动取消
    /// </summary>
    public static void SubscribeOnce<T>(Action<T> handler) where T : GameEvent
    {
        var sub = Subscribe<T>(evt =>
        {
            handler(evt);
            Unsubscribe(sub);
        });
    }

    /// <summary>
    /// 发布事件（同步立即执行）
    /// </summary>
    public static void Publish<T>(T evt) where T : GameEvent
    {
        var type = typeof(T);
        if (!listeners.ContainsKey(type)) return;

        // 复制一份列表，防止回调中修改集合导致异常
        var snapshot = listeners[type].ToList();

        foreach (var sub in snapshot)
        {
            if (sub.IsDisposed) continue;
            try
            {
                sub.Handler.Invoke(evt);
            }
            catch (Exception e)
            {
                UnityEngine.Debug.LogError($"EventBus callback error: {e}");
            }
        }
    }

    /// <summary>
    /// 延迟发布：将事件加入队列，在下一帧处理
    /// </summary>
    public static void PublishDeferred<T>(T evt) where T : GameEvent
    {
        eventQueue.Enqueue(evt);
    }

    /// <summary>
    /// 处理延迟事件队列（每帧调用一次）
    /// </summary>
    public static void ProcessQueue()
    {
        isProcessing = true;
        while (eventQueue.Count > 0)
        {
            var evt = eventQueue.Dequeue();
            Publish(evt);
        }
        isProcessing = false;
    }

    /// <summary>
    /// 取消订阅
    /// </summary>
    public static void Unsubscribe(EventSubscription subscription)
    {
        subscription.IsDisposed = true;
        // 延迟清理：在下次遍历时跳过已标记的订阅者
        // 实际项目中可以定期清理
    }

    /// <summary>
    /// 移除某个对象的所有订阅（用于对象销毁时防止内存泄漏）
    /// </summary>
    public static void UnsubscribeAll(object owner)
    {
        foreach (var subs in listeners.Values)
        {
            subs.RemoveAll(s => s.Owner == owner);
        }
    }

    /// <summary>
    /// 清除所有订阅（场景切换时使用）
    /// </summary>
    public static void ClearAll()
    {
        listeners.Clear();
        eventQueue.Clear();
    }
}

// ========== 订阅信息 ==========
public class EventSubscription
{
    public Action<GameEvent> Handler;
    public int Priority;
    public Type EventType;
    public object Owner; // 订阅者所属对象，用于批量取消
    public bool IsDisposed = false;
}
```

### C# delegate 与 event 关键字

C# 原生的 delegate 和 event 是观察者模式的内置实现，适合单个对象的事件通知：

```csharp
public class Player
{
    // 声明事件（只能在类内部触发，外部只能 += 和 -=）
    public event Action<float, float> OnHealthChanged; // (current, max)
    public event Action OnPlayerDeath;
    public event Action<int> OnLevelUp;
    public event Action<ItemData> OnItemCollected;

    private float health;
    private int level;

    public float Health
    {
        get => health;
        set
        {
            float oldHealth = health;
            health = Mathf.Clamp(value, 0, maxHealth);
            OnHealthChanged?.Invoke(health, maxHealth);

            if (health <= 0 && oldHealth > 0)
                OnPlayerDeath?.Invoke();
        }
    }

    public void AddExperience(int amount)
    {
        experience += amount;
        while (experience >= expToNextLevel)
        {
            experience -= expToNextLevel;
            level++;
            OnLevelUp?.Invoke(level);
        }
    }
}

// ===== 订阅者示例 =====
public class HealthBarUI : MonoBehaviour
{
    [SerializeField] private Player player;
    [SerializeField] private Image healthBarFill;

    void OnEnable()
    {
        player.OnHealthChanged += UpdateHealthBar;
        player.OnPlayerDeath += OnPlayerDied;
    }

    void OnDisable()
    {
        // 必须取消订阅，否则导致内存泄漏！
        player.OnHealthChanged -= UpdateHealthBar;
        player.OnPlayerDeath -= OnPlayerDied;
    }

    void UpdateHealthBar(float current, float max)
    {
        healthBarFill.fillAmount = current / max;
    }

    void OnPlayerDied()
    {
        ShowDeathScreen();
    }
}

public class AchievementTracker : MonoBehaviour
{
    void OnEnable()
    {
        // 监听全局事件
        EventBus.Subscribe<DamageEvent>(OnDamageDealt);
        EventBus.Subscribe<ItemPickupEvent>(OnItemPickedUp);
    }

    void OnDisable()
    {
        EventBus.UnsubscribeAll(this);
    }

    void OnDamageDealt(DamageEvent evt)
    {
        if (evt.Amount >= 100)
            Achievement.Unlock("Heavy_Hitter");
    }

    void OnItemPickedUp(ItemPickupEvent evt)
    {
        if (evt.Item.Rarity == ItemRarity.Legendary)
            Achievement.Unlock("Lucky_Find");
    }
}
```

### 信号系统（Signal / Typed Event）

某些框架使用轻量级的信号类替代全局事件总线：

```csharp
// 泛型信号类——每个事件是独立的对象，而非全局总线
public class Signal<T> where T : struct
{
    private List<Action<T>> listeners = new();

    public void AddListener(Action<T> handler) => listeners.Add(handler);
    public void RemoveListener(Action<T> handler) => listeners.Remove(handler);

    public void Fire(T data)
    {
        foreach (var listener in listeners.ToList()) // 安全复制
            listener.Invoke(data);
    }

    public void Clear() => listeners.Clear();
}

// 使用示例
public class EventBus_Signals
{
    public static readonly Signal<DamageEvent> OnDamage = new();
    public static readonly Signal<PlayerDeathEvent> OnPlayerDeath = new();
    public static readonly Signal<ItemPickupEvent> OnItemPickup = new();
}

// 发布
EventBus_Signals.OnDamage.Fire(new DamageEvent { Amount = 50 });

// 订阅
EventBus_Signals.OnDamage.AddListener(evt => ShowDamage(evt.Amount));
```

### 事件系统在游戏中的典型使用场景

```csharp
// 场景1：伤害通知系统
public class CombatSystem : MonoBehaviour
{
    void Attack(Character target, float damage)
    {
        target.Health -= damage;

        // 一个事件，多个系统同时响应
        EventBus.Publish(new DamageEvent
        {
            Target = target.gameObject,
            Amount = damage,
            Type = DamageType.Physical
        });
    }
}

// 以下各系统独立监听 DamageEvent：
// - DamageNumberUI：显示飘字
// - HitSoundEffect：播放受击音效
// - ScreenShake：屏幕震动
// - ComboSystem：连击计数
// - AchievementTracker：成就判定
// - NetworkSync：同步伤害到客户端
// - Analytics：记录伤害统计

// 场景2：成就系统（全局事件监听）
public class AchievementSystem : MonoBehaviour
{
    void OnEnable()
    {
        EventBus.Subscribe<DamageEvent>(CheckDamageAchievements);
        EventBus.Subscribe<PlayerDeathEvent>(CheckDeathAchievements);
        EventBus.Subscribe<ItemPickupEvent>(CheckCollectionAchievements);
        EventBus.Subscribe<LevelUpEvent>(CheckLevelAchievements);
    }

    void CheckDamageAchievements(DamageEvent evt)
    {
        if (evt.Amount >= 500) Unlock("One_Shot_Kill");
        totalDamageDealt += evt.Amount;
        if (totalDamageDealt >= 10000) Unlock("Veteran_Warrior");
    }
}

// 场景3：任务/引导系统
public class QuestTracker : MonoBehaviour
{
    void OnEnable()
    {
        EventBus.Subscribe<DamageEvent>(OnCombatEvent);
        EventBus.Subscribe<ItemPickupEvent>(OnItemEvent);
    }

    void OnCombatEvent(DamageEvent evt)
    {
        if (activeQuest?.Objective == QuestObjective.DealDamage)
        {
            activeQuest.Progress += (int)evt.Amount;
            if (activeQuest.IsComplete)
                EventBus.Publish(new QuestCompleteEvent { Quest = activeQuest });
        }
    }
}
```

## 方案对比

| 方案 | 解耦程度 | 类型安全 | 性能 | 调试难度 | 适用场景 |
|------|---------|---------|------|---------|---------|
| 直接调用 | 无 | 高 | 最高 | 低 | 小型项目 |
| C# event/delegate | 中 | 高 | 高 | 低 | 对象间事件 |
| 全局事件总线 | 高 | 中 | 中 | 中 | 跨系统事件 |
| 信号系统 | 高 | 高 | 中 | 中 | 中型项目 |
| 消息队列 | 极高 | 低 | 低 | 高 | 网络/分布式 |

## 常见陷阱与解决方案

1. **内存泄漏**：订阅者销毁前忘记取消订阅，事件持有其引用。解决方案：在 OnDestroy / OnDisable 中取消订阅；使用 WeakReference
2. **事件地狱**：事件链过长导致调试困难。解决方案：限制事件层级（最多2-3层），使用事件日志
3. **回调中修改集合**：在事件回调中添加/移除订阅者导致异常。解决方案：遍历时复制快照
4. **事件顺序依赖**：A系统的回调必须在B之前执行。解决方案：使用优先级参数
5. **性能问题**：高频事件（每帧多次）使用事件总线有开销。解决方案：对于热点事件直接调用，或使用结构体事件避免装箱

## Unity 实现

```csharp
// Unity 提供了多种事件机制

// 1. UnityEvent（可在Inspector中配置）
public class Weapon : MonoBehaviour
{
    public UnityEvent<GameObject> OnHit;  // 拖拽绑定
    public UnityEvent OnFire;

    void Fire()
    {
        OnFire?.Invoke();
        // 检测命中
        if (HitTarget(out var target))
            OnHit?.Invoke(target);
    }
}

// 2. ScriptableObject 作为事件通道
[CreateAssetMenu(menuName = "Events/Damage Event Channel")]
public class DamageEventChannel : ScriptableObject
{
    public event Action<DamageEvent> OnEvent;
    public void Raise(DamageEvent evt) => OnEvent?.Invoke(evt);
}
```

## 实际使用案例

- **Unity 的 C# Event 和 UnityEvent** 是观察者模式的标准实现
- **《魔兽世界》** 的插件系统基于事件钩子（Event Hook），允许玩家自定义UI
- **Unreal Engine 的 Delegate 系统**（单播/多播）支持类型安全的事件通信
- **手游的成就系统、引导系统** 通常全局监听游戏事件
- **《原神》** 的元素反应系统通过事件链触发连锁反应
