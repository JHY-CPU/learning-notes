# 108-游戏设计模式

本模块系统讲解游戏开发中常用的设计模式，涵盖架构模式、行为模式和优化模式。

## 目录

| 编号 | 主题 | 说明 |
|------|------|------|
| 01 | 游戏循环(Game Loop) | 固定步长vs可变步长、Update/Render分离 |
| 02 | ECS架构 | Entity-Component-System、数据驱动、Unity DOTS |
| 03 | 组件模式(Component) | 游戏对象组合、动态挂载、与继承的取舍 |
| 04 | 状态模式(State) | 角色状态机、动画状态、UI状态管理 |
| 05 | 观察者模式与事件系统 | 事件总线、解耦通信、C#委托/事件 |
| 06 | 对象池(Pool) | 子弹/敌人/特效池化、预热、动态扩容 |
| 07 | 命令模式(Command) | 输入重放、撤销/重做、录像回放 |
| 08 | 原型模式(Prototype) | Prefab本质、深拷贝、配置驱动生成 |
| 09 | 单例模式在游戏中的应用 | GameManager/AudioManager、场景持久化 |
| 10 | 装饰器与数据驱动 | Buff/Debuff系统、装备属性叠加 |
| 11 | 空间分区(Spatial Partitioning) | 四叉树/八叉树/BVH、碰撞检测、可见性剔除 |
| 12 | 更新方法(Update Method) | Tick驱动逻辑、优先级排序、批量更新 |

## 学习建议

建议先掌握游戏循环和ECS架构等基础架构模式，再学习具体的设计模式。每个模式都附带游戏开发中的实际应用案例。
