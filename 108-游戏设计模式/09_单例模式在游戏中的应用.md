# 单例模式在游戏中的应用

## 核心概念

单例模式确保一个类只有一个全局实例，并提供全局访问点。在游戏中常用于管理器类（Manager），如 GameManager、AudioManager、InputManager 等。单例是最简单也最具争议的设计模式——它方便但容易被滥用。

### 为什么单例在游戏中如此常见？

```
游戏需要大量全局服务：
- 音频管理器：任何地方都可能需要播放音效
- 游戏状态管理：分数、关卡进度全局可访问
- 输入管理器：任何系统都可能需要读取输入
- 场景管理器：切换场景是全局操作
- 网络管理器：全局通信通道

这些服务天然"全局唯一"，单例模式刚好满足需求。
```

### 标准单例实现

```csharp
// ===== 方式1：最简单的单例（Unity MonoBehaviour） =====
public class GameManager : MonoBehaviour
{
    // 静态实例引用
    public static GameManager Instance { get; private set; }

    // 游戏状态
    public GameState CurrentState { get; private set; }
    public int Score { get; private set; }
    public int Level { get; private set; }

    void Awake()
    {
        // 单例检查：防止场景中存在多个实例
        if (Instance != null && Instance != this)
        {
            Debug.LogWarning($"重复的 GameManager 实例，销毁 {gameObject.name}");
            Destroy(gameObject);
            return;
        }
        Instance = this;
    }

    void OnDestroy()
    {
        if (Instance == this)
            Instance = null; // 防止销毁后引用悬空
    }

    public void ChangeState(GameState newState)
    {
        var oldState = CurrentState;
        CurrentState = newState;
        EventBus.Publish(new GameStateChangedEvent { OldState = oldState, NewState = newState });
    }

    public void AddScore(int points)
    {
        Score += points;
        EventBus.Publish(new ScoreChangedEvent { NewScore = Score });
    }
}
```

```csharp
    public void PlaySFX(string clipName, Vector3 position = default)
    {
        if (clips.TryGetValue(clipName, out var clip))
        {
            AudioSource.PlayClipAtPoint(clip, position, masterVolume * sfxVolume);
        }
    }

    public void StopBGM() { /* ... */ }
```

### 场景间持久化（DontDestroyOnLoad）

```csharp
/// <summary>
/// 跨场景持久化单例——场景切换时不销毁
/// </summary>
public class PersistentManager : MonoBehaviour
{
    public static PersistentManager Instance { get; private set; }

    // 使用场景隔离避免冲突
    private static string singletonSceneName;

    void Awake()
    {
        if (Instance != null && Instance != this)
        {
            Destroy(gameObject);
            return;
        }

        Instance = this;
        singletonSceneName = gameObject.scene.name;
        DontDestroyOnLoad(gameObject);

        // 监听场景加载事件
        SceneManager.sceneLoaded += OnSceneLoaded;
        SceneManager.sceneUnloaded += OnSceneUnloaded;
    }

    void OnDestroy()
    {
        if (Instance == this)
        {
            Instance = null;
            SceneManager.sceneLoaded -= OnSceneLoaded;
            SceneManager.sceneUnloaded -= OnSceneUnloaded;
        }
    }

    void OnSceneLoaded(Scene scene, LoadSceneMode mode)
    {
        Debug.Log($"场景加载: {scene.name}");
        // 根据场景执行对应初始化
        HandleSceneEnter(scene.name);
    }

    void OnSceneUnloaded(Scene scene)
    {
        Debug.Log($"场景卸载: {scene.name}");
    }

    private void HandleSceneEnter(string sceneName)
    {
        switch (sceneName)
        {
            case "MainMenu":
                AudioManager.Instance.PlayBGM("menu_theme");
                break;
            case "Level1":
                AudioManager.Instance.PlayBGM("level1_theme");
                break;
        }
    }
}
```

### 泛型单例基类

```csharp
/// <summary>
/// 泛型单例基类——所有管理器继承此类即可获得单例能力
/// </summary>
public class Singleton<T> : MonoBehaviour where T : MonoBehaviour
{
    private static T instance;
    private static readonly object lockObj = new();
    private static bool applicationIsQuitting = false;

    public static T Instance
    {
        get
        {
            if (applicationIsQuitting)
            {
                Debug.LogWarning($"[Singleton] 应用正在退出，访问 {typeof(T).Name} 已销毁的实例");
                return null;
            }

            if (instance == null)
            {
                lock (lockObj)
                {
                    if (instance == null)
                    {
                        // 先尝试在场景中查找
                        instance = FindObjectOfType<T>();

                        if (instance == null)
                        {
                            // 场景中没有，自动创建
                            var go = new GameObject($"[Singleton] {typeof(T).Name}");
                            instance = go.AddComponent<T>();
                        }
                    }
                }
            }
            return instance;
        }
    }

    protected virtual void Awake()
    {
        if (instance == null)
        {
            instance = this as T;
        }
        else if (instance != this)
        {
            Debug.LogWarning($"[Singleton] 重复实例 {typeof(T).Name}，已销毁");
            Destroy(gameObject);
        }
    }

    protected virtual void OnDestroy()
    {
        if (instance == this)
            instance = null;
    }

    protected virtual void OnApplicationQuit()
    {
        applicationIsQuitting = true;
    }
}

// ===== 使用示例 =====
public class UIManager : Singleton<UIManager>
{
    // 直接拥有单例能力
    public void ShowPanel(string panelName) { /* ... */ }
    public void HidePanel(string panelName) { /* ... */ }
    public void ShowToast(string message) { /* ... */ }
}

public class InputManager : Singleton<InputManager>
{
    public Vector3 Movement => new Vector3(
        UnityEngine.Input.GetAxis("Horizontal"),
        0,
        UnityEngine.Input.GetAxis("Vertical")
    );

    public bool JumpPressed => UnityEngine.Input.GetButtonDown("Jump");
    public bool AttackPressed => UnityEngine.Input.GetButtonDown("Fire1");
}

public class NetworkManager : Singleton<NetworkManager>
{
    public bool IsConnected { get; private set; }

    public void Connect(string serverAddress) { /* ... */ }
    public void Disconnect() { /* ... */ }
    public void Send(byte[] data) { /* ... */ }
}

// 调用方式
void SomeMethod()
{
    UIManager.Instance.ShowPanel("Inventory");
    InputManager.Instance.Movement;
    NetworkManager.Instance.Send(data);
}
```

### 单例 + 场景管理 + 生命周期管理

```csharp
/// <summary>
/// 游戏管理器——管理整个游戏的生命周期
/// </summary>
public class GameManager : SingletonPersistent<GameManager>
{
    // 使用继承持久化单例基类
    public enum GamePhase { Boot, MainMenu, Loading, Playing, Paused, GameOver }

    public GamePhase CurrentPhase { get; private set; }
    public int CurrentLevel { get; private set; }
    public float PlayTime { get; private set; }

    public event Action<GamePhase, GamePhase> OnPhaseChanged;

    protected override void Awake()
    {
        base.Awake();
        Application.targetFrameRate = 60;
        QualitySettings.vSyncCount = 1;
    }

    void Update()
    {
        if (CurrentPhase == GamePhase.Playing)
            PlayTime += Time.deltaTime;
    }

    public void ChangePhase(GamePhase newPhase)
    {
        var oldPhase = CurrentPhase;
        CurrentPhase = newPhase;
        OnPhaseChanged?.Invoke(oldPhase, newPhase);

        switch (newPhase)
        {
            case GamePhase.MainMenu:
                Time.timeScale = 1f;
                SceneManager.LoadScene("MainMenu");
                break;
            case GamePhase.Loading:
                StartCoroutine(LoadLevelAsync(CurrentLevel));
                break;
            case GamePhase.Playing:
                Time.timeScale = 1f;
                break;
            case GamePhase.Paused:
                Time.timeScale = 0f;
                break;
        }
    }

    private IEnumerator LoadLevelAsync(int level)
    {
        var op = SceneManager.LoadSceneAsync($"Level_{level}");
        while (!op.isDone)
            yield return null;

        ChangePhase(GamePhase.Playing);
    }
}

/// <summary>
/// 持久化单例基类（组合 Singleton + DontDestroyOnLoad）
/// </summary>
public class SingletonPersistent<T> : MonoBehaviour where T : MonoBehaviour
{
    private static T instance;

    public static T Instance
    {
        get
        {
            if (instance == null)
            {
                instance = FindObjectOfType<T>();
                if (instance == null)
                {
                    var go = new GameObject($"[{typeof(T).Name}]");
                    instance = go.AddComponent<T>();
                }
            }
            return instance;
        }
    }

    protected virtual void Awake()
    {
        if (instance != null && instance != this)
        {
            Destroy(gameObject);
            return;
        }
        instance = this as T;
        DontDestroyOnLoad(gameObject);
    }
}
```

### 服务定位器（Service Locator）——单例的替代方案

```csharp
/// <summary>
/// 服务定位器——注册和获取全局服务，比单例更灵活
/// </summary>
public static class ServiceLocator
{
    private static Dictionary<Type, object> services = new();

    /// <summary>
    /// 注册服务
    /// </summary>
    public static void Register<T>(T service) where T : class
    {
        services[typeof(T)] = service;
    }

    /// <summary>
    /// 获取服务（找不到时抛异常）
    /// </summary>
    public static T Get<T>() where T : class
    {
        if (services.TryGetValue(typeof(T), out var service))
            return (T)service;

        throw new InvalidOperationException($"服务 {typeof(T).Name} 未注册");
    }

    /// <summary>
    /// 尝试获取服务（找不到时返回null）
    /// </summary>
    public static bool TryGet<T>(out T service) where T : class
    {
        if (services.TryGetValue(typeof(T), out var obj))
        {
            service = (T)obj;
            return true;
        }
        service = null;
        return false;
    }

    /// <summary>
    /// 移除服务
    /// </summary>
    public static void Unregister<T>() where T : class
    {
        services.Remove(typeof(T));
    }

    /// <summary>
    /// 清空所有服务（场景切换时使用）
    /// </summary>
    public static void Clear() => services.Clear();
}

// 使用示例
public class GameBootstrap : MonoBehaviour
{
    void Awake()
    {
        // 注册服务（在游戏启动时一次性配置）
        ServiceLocator.Register(new AudioManager());
        ServiceLocator.Register(new InputManager());
        ServiceLocator.Register(new NetworkManager());
    }
}

// 任何地方使用
void SomeClass()
{
    var audio = ServiceLocator.Get<AudioManager>();
    audio.PlaySFX("click");

    // 不再依赖单例的静态 Instance 属性
    // 更容易测试：可以注册 Mock 对象
}
```

## 方案对比

| 特性 | 单例模式 | 服务定位器 | 依赖注入 |
|------|---------|-----------|---------|
| 全局访问 | 静态属性 | Get<T>() | 构造函数注入 |
| 可测试性 | 差 | 中 | 优秀 |
| 耦合度 | 高（静态依赖） | 中 | 低 |
| 实现复杂度 | 极低 | 低 | 高 |
| 生命周期控制 | 弱 | 中 | 强 |
| 适用项目 | 小型 | 中型 | 大型 |

## 常见陷阱与解决方案

1. **测试困难**：单例的静态引用导致无法注入 Mock。解决方案：使用服务定位器或接口抽象
2. **DontDestroyOnLoad 滥用**：大量持久化对象导致场景切换后残留。解决方案：限制数量，使用场景卸载清理
3. **初始化顺序**：A单例的Awake依赖B单例，但执行顺序不确定。解决方案：使用 Lazy 延迟初始化，或显式初始化顺序
4. **多线程竞争**：多个线程同时访问单例。解决方案：加锁或使用 Lazy<T>（线程安全）
5. **隐式依赖**：代码中到处使用 `XxxManager.Instance`，看不出依赖关系。解决方案：限制使用范围，关键依赖通过构造函数传入

## 实际使用案例

- **Unity 的 Time、Input、SceneManager** 都是引擎内置的单例模式
- **《王者荣耀》** 的全局管理器包括 GameManager、MatchManager、NetworkManager
- **《原神》** 的场景管理通过单例管理器协调大世界和副本的切换
- **大型项目倾向** 用依赖注入（如 Zenject/VContainer）替代单例，降低耦合度
- **Unreal Engine** 的 Subsystem 系统提供了更结构化的全局服务管理
