排序算法工程选型 (Sorting in Practice)

在实际工程中，选择合适的排序算法不仅要考虑时间复杂度，还需要考虑数据特征、稳定性、内存限制等多方面因素。

黄金法则：没有完美的排序算法，只有最适合当前场景的排序算法。

选型决策树

开始选型:
├─ 数据能否全部加载到内存?
│  ├─ 否 → 外部排序（多路归并）
│  └─ 是 →
│     ├─ 数据范围很小（如整数 0-100）?
│     │  └─ 是 → 计数排序
│     ├─ 数据分布均匀?
│     │  └─ 是 → 桶排序
│     ├─ 需要稳定排序?
│     │  ├─ 是 → 归并排序 / TimSort
│     │  └─ 否 → 快速排序 / 堆排序
│     ├─ 内存非常有限?
│     │  └─ 是 → 堆排序（O(1) 空间）
│     ├─ 数据接近有序?
│     │  └─ 是 → 插入排序
│     └─ 大规模随机数据?
│        └─ 快速排序（通常最快）
