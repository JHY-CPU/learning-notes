剪枝优化

什么是剪枝

剪枝（Pruning）是在回溯搜索过程中，提前终止对不可能产生有效解的路径的探索。剪枝可以大幅减少搜索空间，提高算法效率。

剪枝的核心思想是：在递归之前判断当前选择是否有可能得到有效解，如果不可能，就跳过该选择。

常见剪枝策略

可行性剪枝：判断当前路径是否可能达到有效解，若不可能则剪断

最优性剪枝：判断当前路径是否可能比已知最优解更好，若不可能则剪断

重复性剪枝：通过排序和跳过相同元素，避免生成重复解

对称性剪枝：利用问题的对称性，减少搜索空间

剪枝示例：组合总和 II

给定数组 candidates 和目标值 target，找出所有和为 target 的组合。每个元素只能使用一次，解集不能包含重复组合。

function combinationSum2(candidates, target) { candidates.sort((a,b) => a - b); // 排序以支持剪枝 const result = []; function backtrack(start, target, path) { if (target === 0) { result.push([...path]); return; } for (let i = start; i < candidates.length; i++) { // 剪枝 1：跳过重复元素 if (i > start && candidates[i] === candidates[i-1]) continue; // 剪枝 2：当前元素已超过剩余目标 if (candidates[i] > target) break; // 剪枝 3：后续元素都 >= 当前，如果当前太大则后续都不行（已通过排序保证） path.push(candidates[i]); backtrack(i + 1, target - candidates[i], path); path.pop(); } } backtrack(0, target, []); return result; }
