递归三要素

递归三要素概述

编写正确的递归函数需要把握三个关键要素：

要素一：基本情况 (Base Case)

要素二：递归步骤 (Recursive Step)

要素三：组合结果 (Combine Results)

要素一：基本情况（Base Case）

递归必须包含一个或多个基本情况，用于直接返回结果而不进行递归调用。基本情况是递归的终止条件。

// 斐波那契数列的基本情况 function fib(n) { if (n === 0) return 0; // 基本情况 1 if (n === 1) return 1; // 基本情况 2 return fib(n-1) + fib(n-2); // 递归步骤 }
没有基本情况的递归会导致无限递归，最终造成栈溢出。

要素二：递归步骤（Recursive Step）

将原问题分解为更小的子问题，并调用自身来解决这些子问题。每次递归调用必须向着基本情况靠近。

关键：每次递归调用都必须改变状态（通常是减小问题规模），并逐步逼近基本情况。

要素三：将子问题的结果组合

将子问题的结果通过某种方式组合起来，形成原问题的答案。

// 计算数组和：组合子问题结果 function sum(arr, n) { if (n <= 0) return 0; // 基本情况 return sum(arr, n-1) + arr[n-1]; // 递归 + 组合 }
