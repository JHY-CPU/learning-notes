# 多模态大模型


## 多模态大模型


LLM多模态视觉语言


多模态大模型融合视觉、语言等多种模态的信息理解与生成能力，是通向通用人工智能的重要方向。


## CLIP 架构与原理


```
CLIP (Contrastive Language-Image Pre-training)：
OpenAI, 2021 - 连接视觉与语言的桥梁模型

核心思想：学习图像和文本的联合嵌入空间
┌─────────────────────────────────────────────────────┐
│                                                     │
│  图像编码器           文本编码器                      │
│  (ViT/ResNet)        (Transformer)                  │
│       ↓                   ↓                         │
│  image_embed          text_embed                    │
│       ↓                   ↓                         │
│       └──── 对比学习 ────┘                          │
│                                                     │
│  训练目标：对比损失 (InfoNCE)                        │
│  L = -log(exp(sim(I,T+)/τ) / Σ exp(sim(I,T)/τ))   │
│                                                     │
│  正样本：配对的(图像,文本)                            │
│  负样本：batch中其他不配对的(图像,文本)               │
│                                                     │
│  训练数据：4亿(图像,文本)对，来自互联网               │
└─────────────────────────────────────────────────────┘

CLIP 的关键能力：
┌─────────────────────────────────────────────────────┐
│  1. 零样本图像分类                                    │
│     将类别名转化为文本描述                             │
│     "a photo of a cat"                               │
│     "a photo of a dog"                               │
│     计算图像与每个文本描述的相似度 → 分类              │
│     不需要任何训练数据，即开即用                        │
│                                                     │
│  2. 开放词汇识别                                      │
│     不限于预定义的类别集合                             │
│     可以识别训练时没见过的物体类别                      │
│                                                     │
│  3. 图文检索                                          │
│     以文搜图：给定文本描述找到最匹配的图像             │
│     以图搜文：给定图像找到最匹配的文本描述             │
│                                                     │
│  4. 视觉-语言对齐                                     │
│     判断图像和文本是否语义匹配                          │
│     用于多模态推理和验证                                │
└─────────────────────────────────────────────────────┘

CLIP 变体与改进：
┌─────────────────────────────────────────────────────┐
│  SigLIP (Google, 2023)：                             │
│  使用Sigmoid替代Softmax的对比学习                    │
│  不需要跨GPU同步，训练更高效                          │
│  在更大规模数据上训练                                  │
│                                                     │
│  EVA-CLIP：                                          │
│  更高效的训练策略                                     │
│  使用EVA-ViT作为图像编码器                           │
│  在更少计算量下达到更好的性能                          │
│                                                     │
│  OpenCLIP：                                          │
│  开源复现版本                                         │
│  支持多种模型规模和训练配置                            │
│                                                     │
│  Chinese-CLIP：                                      │
│  中文版本的CLIP                                      │
│  文本编码器使用中文预训练模型                          │
│  适用于中文图文检索场景                                │
└─────────────────────────────────────────────────────┘
```


## 视觉语言大模型 (VLM)


```
LLaVA (Large Language and Vision Assistant)：
┌─────────────────────────────────────────────────────┐
│  架构：                                              │
│  图像 → Vision Encoder (CLIP ViT) → 图像特征         │
│  图像特征 → Projection Layer → 视觉Token             │
│  视觉Token + 文本Token → LLM (Vicuna/LLaMA)         │
│  → 生成文本回答                                       │
│                                                     │
│  训练阶段：                                          │
│  Stage 1: 特征对齐预训练                              │
│  冻结CLIP和LLM，只训练Projection Layer               │
│  用大量(图像,描述)对训练对齐                          │
│                                                     │
│  Stage 2: 端到端微调                                  │
│  解冻LLM，用视觉问答数据端到端微调                    │
│  学习理解和回答关于图像的问题                          │
│                                                     │
│  LLaVA-1.5 改进：                                    │
│  - 两层MLP投影层                                     │
│  - 更高分辨率输入(336×336)                            │
│  - 增加更多训练数据                                    │
│  - 简单架构达到SOTA                                   │
│                                                     │
│  LLaVA-NeXT (LLaVA-1.6)：                           │
│  - 动态高分辨率：支持多种分辨率输入                    │
│  - 更好的数据混合策略                                  │
│  - 支持更多语言                                        │
└─────────────────────────────────────────────────────┘

InternVL 系列：
┌─────────────────────────────────────────────────────┐
│  InternVL (上海AI Lab, 2024)：                       │
│  大规模视觉语言基础模型                                │
│                                                     │
│  架构特点：                                          │
│  - InternViT-6B：60亿参数的视觉编码器                │
│  - InternLM2-20B：200亿参数的语言模型                │
│  - 动态高分辨率方案                                    │
│                                                     │
│  InternVL 1.5：                                      │
│  - 动态分辨率：将图像切分为448×448的子图              │
│  - 通过pixel shuffle降低视觉token数                  │
│  - 支持超高分辨率图像理解                              │
│                                                     │
│  InternVL 2：                                        │
│  - 支持更多模态：图像/视频/文档                       │
│  - 多任务统一：VQA/OCR/文档理解/视频理解             │
│  - 多语言支持                                         │
└─────────────────────────────────────────────────────┘

其他重要VLM：
┌─────────────────────────────────────────────────────┐
│  Qwen-VL (阿里)：                                    │
│  - 支持多图、多轮对话                                  │
│  - 细粒度视觉理解（定位、OCR）                        │
│  - 中英文双语支持                                     │
│                                                     │
│  Yi-VL (零一万物)：                                  │
│  - 基于Yi语言模型                                     │
│  - 高分辨率图像理解                                   │
│                                                     │
│  CogVLM (智谱AI)：                                   │
│  - 视觉专家模块                                       │
│  - 深度融合视觉和语言特征                             │
│                                                     │
│  DeepSeek-VL：                                       │
│  - 基于DeepSeek LLM                                  │
│  - 高效的视觉token压缩                               │
│                                                     │
│  MiniCPM-V：                                         │
│  - 端侧部署的VLM                                     │
│  - 仅2B参数，移动端可用                               │
└─────────────────────────────────────────────────────┘
```


## 图文理解能力分析


```
视觉理解维度：
┌─────────────────────────────────────────────────────┐
│  1. 场景理解                                          │
│     识别图像中的场景、物体、属性                        │
│     "图片中有一个红色的沙发，旁边有一盏落地灯"          │
│                                                     │
│  2. 空间关系理解                                      │
│     理解物体之间的位置关系                              │
│     "猫在桌子上面，狗在桌子下面"                       │
│                                                     │
│  3. 文字识别 (OCR)                                   │
│     识别并理解图像中的文字                              │
│     包括手写文字、印刷文字、场景文字                   │
│                                                     │
│  4. 图表理解                                          │
│     理解柱状图、折线图、饼图等数据可视化                │
│     从图表中提取数据和趋势                              │
│                                                     │
│  5. 文档理解                                          │
│     理解文档的版面结构                                  │
│     表格、标题、段落的识别和语义理解                    │
│                                                     │
│  6. 细粒度识别                                        │
│     识别特定区域或物体                                  │
│     "左上角的那个物体是什么？"                         │
│                                                     │
│  7. 推理能力                                          │
│     基于图像内容进行逻辑推理                            │
│     "图片中的天气如何？为什么这个人要打伞？"           │
└─────────────────────────────────────────────────────┘

视觉定位 (Visual Grounding)：
┌─────────────────────────────────────────────────────┐
│  给定文本描述，在图像中定位对应的区域                   │
│                                                     │
│  任务类型：                                          │
│  - Referring Expression：找到"红色的那个杯子"         │
│  - Grounded Captioning：描述图中指定区域              │
│  - Visual QA：回答关于特定区域的问题                  │
│                                                     │
│  技术方案：                                          │
│  - 坐标回归：输出bounding box坐标                     │
│  - 区域选择：从候选区域中选择                          │
│  - 分割：输出像素级mask                               │
│  - 文本坐标：将坐标编码为文本token                    │
│    "物体位于"                       │
└─────────────────────────────────────────────────────┘
```


## 多模态 RAG


```
多模态RAG架构：
┌─────────────────────────────────────────────────────┐
│  传统RAG：文本检索 + 文本生成                         │
│  多模态RAG：支持图像/视频/文档等多种模态的检索和生成   │
│                                                     │
│  架构一：图文检索 + 文本生成                          │
│  用户问题(含图像) → 图文检索 → 相关图文对             │
│  → VLM融合检索结果生成回答                            │
│                                                     │
│  架构二：统一多模态索引                               │
│  将文本和图像嵌入到统一向量空间                        │
│  CLIP/SigLIP作为编码器                                │
│  统一检索，VLM生成                                     │
│                                                     │
│  架构三：分层多模态RAG                                │
│  第一层：文本检索（关键词/语义）                       │
│  第二层：图像检索（以文搜图）                          │
│  第三层：表格/图表检索                                │
│  VLM融合多层结果生成回答                              │
└─────────────────────────────────────────────────────┘

ColPali / ColQwen：文档检索新范式
┌─────────────────────────────────────────────────────┐
│  问题：传统文档检索需要先OCR → 文本检索               │
│  OCR错误会传播到检索环节                               │
│                                                     │
│  方案：直接对文档页面图像进行嵌入和检索                 │
│  使用VLM编码器直接理解文档版面                        │
│  查询和文档页面在同一嵌入空间匹配                      │
│                                                     │
│  ColPali (2024)：                                    │
│  - 基于PaliGemma VLM                                 │
│  - 文档页面 → 多个patch向量                          │
│  - 查询 → 多个patch向量                              │
│  - MaxSim匹配：所有patch对的最大相似度之和            │
│                                                     │
│  优势：跳过OCR，直接理解文档视觉信息                   │
│  版面信息（表格、图表、公式）自然保留                   │
└─────────────────────────────────────────────────────┘

多模态RAG应用场景：
┌─────────────────────────────────────────────────────┐
│  1. 文档问答                                          │
│     包含表格、图表、公式的文档理解                      │
│     学术论文、财务报告、技术手册                       │
│                                                     │
│  2. 产品图文问答                                      │
│     电商产品图+描述的联合检索和问答                     │
│     "这件衣服的袖子是什么款式？"                       │
│                                                     │
│  3. 医学影像问答                                      │
│     X光片、CT、MRI等医学影像的理解                    │
│     需要专业医学知识                                    │
│                                                     │
│  4. 视频理解问答                                      │
│     将视频切分为关键帧                                  │
│     对关键帧进行多模态检索和理解                        │
└─────────────────────────────────────────────────────┘
```


## GPT-4V / GPT-4o 能力分析


```
GPT-4V (Vision) 能力：
┌─────────────────────────────────────────────────────┐
│  2023年9月发布，OpenAI的首个商用多模态大模型           │
│                                                     │
│  核心能力：                                          │
│  - 细粒度图像理解：识别物体、场景、属性                │
│  - 文字/手写识别：OCR能力强大                         │
│  - 图表/文档理解：解析复杂版面                        │
│  - 推理能力：基于图像内容进行逻辑推理                  │
│  - 创意任务：图像描述、创意写作                       │
│  - 多语言：支持多种语言的图像理解                      │
│                                                     │
│  局限性：                                            │
│  - 空间关系理解不够精确                                │
│  - 数数能力有限（精确计数物体数量）                    │
│  - 无法处理视频（只能处理静态图像）                    │
│  - 幻觉：可能描述图像中不存在的内容                    │
│  - 安全限制：拒绝处理某些敏感图像                      │
└─────────────────────────────────────────────────────┘

GPT-4o (Omni) 进化：
┌─────────────────────────────────────────────────────┐
│  2024年5月发布，原生多模态模型                         │
│                                                     │
│  关键改进：                                          │
│  - 原生多模态：文本/图像/音频统一处理                  │
│  不是拼接模型，而是端到端训练                          │
│  - 实时对话：支持语音+图像的实时交互                   │
│  - 响应速度：比GPT-4V快2倍                            │
│  - 成本降低：API价格降低50%                           │
│  - 视觉能力提升：更好的OCR和空间理解                   │
│  - 情感理解：可感知语音中的情感                        │
│                                                     │
│  与GPT-4V对比：                                      │
│  ┌──────────┬──────────┬──────────┐                  │
│  │ 能力      │ GPT-4V   │ GPT-4o   │                  │
│  ├──────────┼──────────┼──────────┤                  │
│  │ 图像理解  │ 强        │ 更强     │                  │
│  │ 音频理解  │ 无        │ 原生支持  │                  │
│  │ 响应速度  │ 较慢      │ 快2倍    │                  │
│  │ 成本      │ 高        │ 降低50%  │                  │
│  │ 实时交互  │ 不支持    │ 支持     │                  │
│  └──────────┴──────────┴──────────┘                  │
└─────────────────────────────────────────────────────┘

Claude 3.5 Sonnet 多模态能力：
┌─────────────────────────────────────────────────────┐
│  Anthropic 的多模态大模型                             │
│  强大的视觉理解能力                                   │
│  在文档理解和图表分析上表现出色                        │
│  支持长上下文中的多张图像                              │
│  代码生成能力结合图像理解                              │
│  "根据这个UI截图生成对应的HTML代码"                    │
└─────────────────────────────────────────────────────┘

Gemini (Google) 多模态：
┌─────────────────────────────────────────────────────┐
│  原生多模态架构                                       │
│  从训练阶段就融合文本/图像/音频/视频/代码              │
│  Gemini 1.5 Pro：支持100万token上下文                 │
│  可以处理长达1小时的视频                               │
│  可以处理大型代码库                                    │
│  在多模态基准测试中领先                                │
└─────────────────────────────────────────────────────┘
```


## 多模态前沿方向


```
视觉生成 (Image Generation)：
┌─────────────────────────────────────────────────────┐
│  DALL-E 3 / Midjourney / Stable Diffusion 3         │
│  文本到图像生成                                       │
│                                                     │
│  Flux (Black Forest Labs)：                          │
│  新一代图像生成模型                                   │
│  更好的文本理解和遵循能力                              │
│                                                     │
│  视频生成：                                          │
│  Sora (OpenAI)：文本到视频生成                       │
│  可生成长达60秒的高质量视频                            │
│  理解物理世界的运动规律                                │
│                                                     │
│  Kling (快手) / Gen-3 (Runway)：                    │
│  商用视频生成模型                                     │
└─────────────────────────────────────────────────────┘

多模态Agent：
┌─────────────────────────────────────────────────────┐
│  将视觉理解与工具使用结合                              │
│                                                     │
│  屏幕理解Agent：                                     │
│  理解屏幕截图，执行GUI操作                            │
│  如：自动操作手机/电脑完成任务                         │
│                                                     │
│  具身智能 (Embodied AI)：                            │
│  将多模态理解与机器人控制结合                          │
│  理解环境 → 规划动作 → 执行操作                      │
│                                                     │
│  GUI Agent：                                         │
│  CogAgent / OmniParser                              │
│  理解软件界面，自动完成用户任务                        │
└─────────────────────────────────────────────────────┘

端侧多模态模型：
┌─────────────────────────────────────────────────────┐
│  趋势：在手机/边缘设备上运行多模态模型                 │
│                                                     │
│  代表模型：                                          │
│  - MiniCPM-V (2B)：手机可运行                       │
│  - Phi-3-Vision (4.2B)：微软轻量级VLM               │
│  - moondream (1.8B)：极小VLM                        │
│  - Qwen2-VL (2B)：阿里轻量版                        │
│                                                     │
│  优化技术：                                          │
│  - 量化：INT4/INT8降低内存需求                       │
│  - 视觉token压缩：减少视觉token数量                  │
│  - 知识蒸馏：大模型蒸馏到小模型                       │
│                                                     │
│  应用场景：                                          │
│  - 实时翻译（拍照翻译）                               │
│  - 智能相册（搜索和分类）                              │
│  - 辅助视觉（帮助视障人士）                            │
└─────────────────────────────────────────────────────┘
```


> **Note:** 多模态大模型从CLIP的对比学习到LLaVA/InternVL的视觉语言理解，再到GPT-4o的原生多模态，技术快速演进。多模态RAG将检索增强扩展到图文领域，为文档理解和知识问答提供了新范式。端侧部署和多模态Agent是重要的应用方向。
