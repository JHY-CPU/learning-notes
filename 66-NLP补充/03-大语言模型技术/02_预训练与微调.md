# 预训练与微调


## 预训练与微调


LLM预训练RLHF


预训练-微调范式是现代NLP的核心，大语言模型通过大规模预训练获得通用能力。


## 预训练目标


```
BERT式预训练（自编码）：
┌─────────────────────────────────────────────┐
│  MLM (Masked Language Modeling)              │
│  随机遮蔽15%的token，预测被遮蔽的token        │
│  "猫 [MASK] 在 [MASK] 子 上"                 │
│  → 预测"坐"和"垫"                            │
│  双向上下文理解                               │
│                                             │
│  NSP (Next Sentence Prediction)              │
│  判断两个句子是否连续                         │
│  （后续研究发现NSP帮助不大，RoBERTa去掉了）   │
└─────────────────────────────────────────────┘

GPT式预训练（自回归）：
┌─────────────────────────────────────────────┐
│  CLM (Causal Language Modeling)              │
│  根据前文预测下一个token                      │
│  P(xt | x1, x2, ..., xt-1)                 │
│  单向（只能看到左边的上下文）                  │
│  天然适合文本生成                             │
└─────────────────────────────────────────────┘

T5式预训练（Seq2Seq）：
┌─────────────────────────────────────────────┐
│  Span Corruption                              │
│  随机遮蔽连续的文本片段                        │
│  "猫 [MASK] [MASK] 上" → "坐在垫子"          │
│  编码器处理输入，解码器生成被遮蔽的片段        │
└─────────────────────────────────────────────┘

Scaling Laws（缩放定律）：
Loss ∝ N^(-α) × D^(-β) × C^(-γ)
- N：模型参数量
- D：数据量
- C：计算量
- 三者对损失的贡献是幂律关系
- 更大模型+更多数据=更好性能（尚未饱和）
```


## 微调策略


```
全量微调 (Full Fine-tuning)：
- 更新所有参数
- 需要大量GPU内存
- 效果最好但成本高

参数高效微调 (PEFT)：
┌─────────────────────────────────────────────┐
│  LoRA (Low-Rank Adaptation)                  │
│  冻结原始权重W，添加低秩分解 ΔW = A × B      │
│  A: d×r, B: r×d, r<
```

## RLHF与对齐


```
RLHF (Reinforcement Learning from Human Feedback)：

三阶段训练：
┌─────────────────────────────────────────────┐
│  Stage 1: SFT (监督微调)                     │
│  用高质量的(指令,回答)对微调                  │
│  得到SFT模型                                  │
│                                             │
│  Stage 2: Reward Model训练                   │
│  收集人类偏好数据：对同一问题的多个回答排序    │
│  训练奖励模型预测人类偏好得分                  │
│                                             │
│  Stage 3: PPO优化                            │
│  用PPO算法优化SFT模型                        │
│  最大化奖励模型的得分                         │
│  加入KL散度约束防止偏离SFT模型太远            │
└─────────────────────────────────────────────┘

DPO (Direct Preference Optimization)：
- 跳过奖励模型训练
- 直接从偏好数据优化策略
- 简化了RLHF流程
- 效果与RLHF相当

Prompt Engineering：
- Zero-shot：直接给指令
- Few-shot：给几个示例
- Chain-of-Thought (CoT)："让我们一步步思考"
- Self-Consistency：多次采样取多数
- Tree-of-Thought：探索多条推理路径

RAG (Retrieval-Augmented Generation)：
- 检索相关文档 + 生成回答
- 解决LLM知识过时和幻觉问题
- 检索器(Retriever)：向量数据库
- 生成器(Generator)：LLM
```


> **Note:** RLHF/DPO对齐使LLM变得有用、无害、诚实(HHH)。LoRA/QLoRA使微调变得平民化。

总结：预训练通过大规模无标注数据学习通用语言表示，微调将通用能力适配到具体任务。
    RLHF通过人类反馈对齐模型行为，DPO简化了这一过程。LoRA等PEFT方法大幅降低了微调成本。
    RAG结合检索和生成，解决了LLM的知识局限性。
