# 文本分类与情感分析


## 文本分类与情感分析


NLP文本分类情感分析


文本分类将文本分配到预定义类别，情感分析是其最重要的子任务之一，广泛应用于舆情监控、推荐系统等场景。


## 文本分类概述


```
文本分类任务类型：
┌─────────────────────────────────────────────────────┐
│  二分类：垃圾邮件检测（spam/ham）                     │
│  多分类：新闻分类（体育/科技/娱乐/财经/...）          │
│  多标签：一篇文章同时属于"科技"和"财经"               │
│  层次分类：类目具有层级结构（如电商类目树）            │
│                                                     │
│  应用场景：                                          │
│  - 情感分析：正面/负面/中性                          │
│  - 意图识别：查询/投诉/咨询/购买                     │
│  - 主题分类：新闻分类、论文分类                       │
│  - 垃圾检测：垃圾邮件、违规内容检测                   │
│  - 语言识别：判断文本的语言种类                       │
└─────────────────────────────────────────────────────┘

传统方法演进：
┌─────────────────────────────────────────────────────┐
│  1. 朴素贝叶斯 (Naive Bayes)                        │
│     P(class|doc) ∝ P(class) × Π P(word_i|class)    │
│     假设特征条件独立，简单高效                         │
│     适合高维稀疏文本数据                               │
│                                                     │
│  2. SVM + TF-IDF                                    │
│     将文本表示为TF-IDF向量                           │
│     使用线性SVM进行分类                               │
│     长期是文本分类的主流方法                           │
│     在小数据集上仍然有竞争力                           │
│                                                     │
│  3. 集成方法                                         │
│     XGBoost/LightGBM + 特征工程                      │
│     结合n-gram、TF-IDF、主题特征                     │
│     在表格化特征上效果好                               │
└─────────────────────────────────────────────────────┘
```


## 深度学习文本分类方法


```
1. TextCNN（Kim, 2014）：
┌─────────────────────────────────────────────────────┐
│  使用不同大小的卷积核提取n-gram特征                    │
│                                                     │
│  输入：词嵌入矩阵 (n × d)                            │
│  卷积核：宽度为 2, 3, 4, 5 的多组卷积核               │
│  池化：Max-over-time Pooling                         │
│  输出：拼接各卷积核的最大池化结果 → 分类               │
│                                                     │
│  优点：简单高效，训练快，适合短文本                    │
│  缺点：无法捕获长距离依赖，对长文本效果有限            │
└─────────────────────────────────────────────────────┘

2. TextRNN（BiLSTM + Attention）：
┌─────────────────────────────────────────────────────┐
│  BiLSTM编码：前向+后向捕获上下文                      │
│  Attention：加权聚合各位置的隐状态                    │
│  输出：attention加权表示 → 分类                      │
│                                                     │
│  优点：能捕获序列依赖关系                             │
│  缺点：训练速度慢于CNN，对长序列有梯度问题            │
└─────────────────────────────────────────────────────┘

3. TextRCNN（CNN + RNN结合）：
┌─────────────────────────────────────────────────────┐
│  BiLSTM获取上下文表示                                 │
│  CNN提取局部特征                                      │
│  MaxPooling得到最终表示 → 分类                       │
│  结合了CNN和RNN的优点                                 │
└─────────────────────────────────────────────────────┘

4. Transformer-based（当前主流）：
┌─────────────────────────────────────────────────────┐
│  BERT分类：                                          │
│  [CLS] token → BERT编码 → h_cls → Linear → Softmax │
│  利用[CLS]的表示做句子级分类                          │
│                                                     │
│  优势：                                              │
│  - 预训练语言模型提供强大的上下文理解                  │
│  - 微调即可获得优秀效果                               │
│  - 支持多语言（mBERT, XLM-R）                        │
│                                                     │
│  变体：                                              │
│  - RoBERTa-wwm-ext：中文优化版本                     │
│  - MacBERT：改进的MLM策略                            │
│  - DeBERTa：解耦注意力机制                           │
│  - ALBERT：参数共享，轻量级                          │
└─────────────────────────────────────────────────────┘

模型选择建议：
┌──────────────┬──────────────┬────────────────────────┐
│ 场景          │ 推荐模型      │ 说明                    │
├──────────────┼──────────────┼────────────────────────┤
│ 短文本+大数据  │ TextCNN      │ 快速训练，效果不错       │
│ 长文本分类     │ BERT+Attention│ 上下文理解能力强        │
│ 低延迟要求     │ DistilBERT   │ 蒸馏模型，速度快         │
│ 少样本场景     │ GPT-4 Few-shot│ 利用大模型的泛化能力   │
│ 最佳效果       │ RoBERTa/DeBERTa│ 预训练+微调          │
└──────────────┴──────────────┴────────────────────────┘
```


## 情感分析 (Sentiment Analysis)


```
情感分析粒度：
┌─────────────────────────────────────────────────────┐
│  1. 文档级情感分析                                    │
│     对整篇文档判断情感极性                             │
│     "这部电影太棒了，剧情紧凑，演员演技在线" → 正面   │
│                                                     │
│  2. 句子级情感分析                                    │
│     对每个句子判断情感                                 │
│     "画面很美但剧情很无聊"                            │
│     句1(正面) + 句2(负面)                             │
│                                                     │
│  3. 方面级情感分析 (ABSA)                            │
│     对特定方面/属性进行情感判断                        │
│     "这家餐厅菜很好吃但服务很差"                      │
│     菜品: 正面  服务: 负面                            │
│                                                     │
│  4. 观点挖掘                                         │
│     识别持有者(Holder)、目标(Target)、情感(Sentiment)│
│     "[我]Holder觉得[这款手机]Target的[拍照]Aspect     │
│      效果[很好]Sentiment"                            │
└─────────────────────────────────────────────────────┘

情感标签体系：
┌──────────────┬──────────────────────────────────────┐
│ 二分类        │ 正面 / 负面                          │
│ 三分类        │ 正面 / 中性 / 负面                   │
│ 五级分类      │ 非常正面 / 正面 / 中性 / 负面 / 非常负面 │
│ 连续评分      │ 1-5 星级评分                         │
│ 细粒度情感    │ 喜悦、愤怒、悲伤、恐惧、惊讶、厌恶   │
└──────────────┴──────────────────────────────────────┘
```


## 方面级情感分析 (ABSA)


```
ABSA 四个子任务：
┌─────────────────────────────────────────────────────┐
│  1. 方面词提取 (ATE: Aspect Term Extraction)         │
│     从句子中提取评价对象                               │
│     "这家[餐厅]的[菜品]很丰富，[服务]态度好"         │
│     → 方面词：餐厅、菜品、服务                        │
│                                                     │
│  2. 方面情感分类 (ATSC: Aspect-level Sentiment)     │
│     对给定方面判断情感极性                             │
│     "屏幕清晰但电池不耐用"                            │
│     屏幕→正面   电池→负面                             │
│                                                     │
│  3. 方面-意见对抽取                                   │
│     同时提取方面词和对应的情感表达                      │
│     "酒店的[位置]aspect很[方便]opinion"               │
│                                                     │
│  4. 细粒度意见挖掘                                    │
│     方面词 + 情感词 + 持有者 + 情感极性               │
│     完整四元组抽取                                    │
└─────────────────────────────────────────────────────┘

ABSA 模型方法：
┌─────────────────────────────────────────────────────┐
│  方法一：基于注意力                                  │
│  AEL / IAN：方面词和上下文交互注意力                  │
│  将方面词作为attention的query                         │
│  聚焦与方面相关的上下文信息                            │
│                                                     │
│  方法二：基于图神经网络                               │
│  ASGCN / Sentic-GCN                                 │
│  构建依存树，通过GNN传播方面-意见信息                  │
│  利用句法结构确定方面与情感词的关联                    │
│                                                     │
│  方法三：基于BERT                                    │
│  BERT-pair / BERT-ADA                               │
│  将方面词和句子拼接输入BERT                           │
│  [CLS] 方面 [SEP] 句子 [SEP]                        │
│  利用BERT的深层语义理解能力                            │
│                                                     │
│  方法四：统一生成式框架                               │
│  GAS / Para-ABSA                                     │
│  将ABSA转化为Seq2Seq生成任务                         │
│  输入：句子                                          │
│  输出：(aspect, opinion, sentiment) 三元组           │
│  用BART/T5等模型统一处理                             │
└─────────────────────────────────────────────────────┘
```


## 多标签文本分类


```
多标签 vs 多类分类：
┌─────────────────────────────────────────────────────┐
│  多类分类（Multi-class）：                            │
│  每个样本只属于一个类别                                │
│  "这篇新闻属于体育"                                   │
│  输出层：Softmax（概率和为1）                         │
│                                                     │
│  多标签分类（Multi-label）：                          │
│  每个样本可以同时属于多个类别                          │
│  "这篇新闻同时属于科技和财经"                          │
│  输出层：Sigmoid（每个类别独立判断）                   │
│  标签之间可能存在相关性                                │
└─────────────────────────────────────────────────────┘

多标签分类方法：
┌─────────────────────────────────────────────────────┐
│  1. Binary Relevance（二元关联）                     │
│     为每个标签训练独立的二分类器                       │
│     忽略标签间相关性                                   │
│     简单高效，适合标签数不多的情况                      │
│                                                     │
│  2. Classifier Chains（分类器链）                    │
│     将前一个分类器的输出作为下一个的输入特征           │
│     捕获标签间依赖关系                                │
│     顺序敏感，使用多条链取平均                         │
│                                                     │
│  3. Label Powerset（标签幂集）                       │
│     将每个标签组合视为独立类别                        │
│     可能的类别数 = 2^L（L为标签数）                   │
│     标签数多时不可行                                   │
│                                                     │
│  4. 基于注意力的标签特定表示                          │
│     为每个标签生成独立的attention权重                  │
│     标签特定的文本表示                                │
│     可学习标签间的语义关系                             │
│                                                     │
│  5. 基于Transformer                                  │
│     BERT + 多标签分类头                               │
│     每个标签对应一个Sigmoid输出                       │
│     或使用生成式方法：逐个生成标签                    │
└─────────────────────────────────────────────────────┘

标签相关性建模：
┌─────────────────────────────────────────────────────┐
│  例：新闻分类标签                                    │
│  "科技"和"财经"经常共现                              │
│  "体育"和"科技"很少共现                              │
│                                                     │
│  利用标签相关性的方法：                               │
│  - 标签嵌入：学习标签的向量表示                       │
│  - 图神经网络：在标签共现图上传播信息                 │
│  - 层次标签结构：利用标签的层次关系                   │
│  - 条件概率：P(label_j | label_i) 建模共现关系       │
└─────────────────────────────────────────────────────┘
```


## 少样本与零样本文本分类


```
问题定义：
- 零样本(Zero-shot)：测试类别在训练时未见过
- 少样本(Few-shot)：每个类别只有K个标注样本（通常K=1,5,10）

方法一：基于提示的大语言模型 (LLM)：
┌─────────────────────────────────────────────────────┐
│  Zero-shot Prompt：                                  │
│  "请判断以下评论的情感：                              │
│   评论：这个产品太好用了                              │
│   情感选项：正面 / 负面 / 中性                        │
│   答案："                                          │
│                                                     │
│  Few-shot Prompt (In-Context Learning)：             │
│  "判断情感：                                        │
│   评论：质量很差 → 负面                              │
│   评论：还行吧 → 中性                                │
│   评论：非常好用 → 正面                              │
│   评论：这个产品太好用了 →"                          │
│                                                     │
│  优势：无需训练，灵活适应新类别                       │
│  劣势：推理成本高，效果不稳定                         │
└─────────────────────────────────────────────────────┘

方法二：基于度量学习 (Metric Learning)：
┌─────────────────────────────────────────────────────┐
│  Prototypical Networks：                             │
│  1. 编码器将样本映射到嵌入空间                        │
│  2. 计算每个类别的原型（prototype）                   │
│     proto_c = mean(encoding(x_i)), x_i ∈ class_c   │
│  3. 新样本分类：找最近的原型                          │
│     pred = argmin_c ||encoding(x) - proto_c||       │
│                                                     │
│  训练策略：                                          │
│  - Episode训练：每次采样N-way K-shot                │
│  - 学习一个好的嵌入空间                               │
│  - 使同类样本靠近，异类样本远离                       │
│                                                     │
│  代表模型：                                          │
│  - BERT-PAIR：样本对匹配                             │
│  - DNNC (Deep Nearest Neighbor Classification)      │
│  - Entailment-based：蕴含关系判断                    │
└─────────────────────────────────────────────────────┘

方法三：数据增强 + 半监督：
┌─────────────────────────────────────────────────────┐
│  数据增强策略：                                       │
│  - 同义词替换 (EDA)                                  │
│  - 回译 (Back-translation)                          │
│  - 上下文替换 (BERT-based Augmentation)             │
│  - Mixup：嵌入空间的样本混合                          │
│                                                     │
│  半监督方法：                                         │
│  - 自训练 (Self-training)：用模型预测未标注数据      │
│  - 一致性正则化：对扰动输入保持一致输出               │
│  - 伪标签 (Pseudo Labeling)：高置信度预测作为标注    │
│                                                     │
│  SetFit (Sentence Transformers Fine-tuning)：       │
│  1. 用对比学习微调句子嵌入                            │
│  2. 在嵌入上训练简单分类器                            │
│  少样本效果好，训练成本低                             │
└─────────────────────────────────────────────────────┘

评估指标：
┌─────────────────────────────────────────────────────┐
│  分类指标：                                          │
│  - Accuracy：整体准确率                              │
│  - Precision / Recall / F1（宏平均/微平均）          │
│  - AUC-ROC：适合不平衡数据                           │
│                                                     │
│  多标签指标：                                         │
│  - Hamming Loss：标签级别的错误率                    │
│  - Subset Accuracy：完全匹配率（严格）               │
│  - Example-based F1：每个样本的F1取平均              │
│                                                     │
│  少样本指标：                                         │
│  - N-way K-shot 准确率                               │
│  - Cross-domain 泛化能力                             │
└─────────────────────────────────────────────────────┘
```


> **Note:** 文本分类从传统机器学习到BERT微调，再到LLM零样本提示，方法不断演进。情感分析从粗粒度发展到方面级的细粒度分析。实际应用中需根据数据量、类别数、延迟要求选择合适方案。
