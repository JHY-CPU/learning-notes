# 知识图谱构建与推理


## 知识图谱构建与推理


NLP知识图谱知识推理


知识图谱将世界知识组织为结构化的图表示，通过实体对齐、知识融合和图谱补全构建大规模知识库，并通过知识推理发现隐含知识。


## 实体对齐 (Entity Alignment)


```
实体对齐目标：将不同知识图谱中表示同一现实实体的节点匹配
┌─────────────────────────────────────────────────────┐
│  KG-A: (爱因斯坦, born_in, 德国)                     │
│  KG-B: (Albert_Einstein, place_of_birth, Germany)   │
│  → 对齐结果：爱因斯坦 ≡ Albert_Einstein              │
│                                                     │
│  应用场景：                                          │
│  - 融合多源知识图谱（维基百科+百度百科+领域知识库）   │
│  - 多语言知识图谱融合                                 │
│  - 数据集成和数据治理                                 │
└─────────────────────────────────────────────────────┘

传统方法：
┌─────────────────────────────────────────────────────┐
│  1. 基于字符串相似度                                  │
│     编辑距离、Jaccard相似度、Jaro-Winkler            │
│     简单但无法处理不同语言/别名                       │
│                                                     │
│  2. 基于属性值匹配                                    │
│     比较实体的属性值（名称、描述、日期等）             │
│     TF-IDF/嵌入相似度                                │
│                                                     │
│  3. 基于结构相似度                                    │
│     比较实体的邻域结构（类似图同构检测）               │
│     对孤立实体效果差                                  │
└─────────────────────────────────────────────────────┘

基于图嵌入的方法（当前主流）：
┌─────────────────────────────────────────────────────┐
│  核心思想：将两个KG嵌入到同一向量空间，对齐=最近邻    │
│                                                     │
│  MTransE (2017)：                                    │
│  分别嵌入两个KG，学习跨KG的转换矩阵                  │
│  e_A ≈ T × e_B                                      │
│                                                     │
│  JAPE (2017)：                                       │
│  联合嵌入，同时利用结构和属性信息                     │
│  L = L_structure + α × L_attribute                  │
│                                                     │
│  GCN-Align (2018)：                                  │
│  使用GCN编码KG结构                                   │
│  已知对齐作为监督信号                                 │
│  最小化对齐实体间的距离                               │
│                                                     │
│  AliNet (2020)：                                     │
│  多跳邻域聚合，处理邻域结构差异                       │
│  使用注意力机制选择相关邻域                           │
│                                                     │
│  BERT-INT (2020)：                                   │
│  使用BERT编码实体描述文本                             │
│  结合结构嵌入和文本嵌入                               │
│  对文本丰富的KG效果好                                 │
└─────────────────────────────────────────────────────┘

挑战与前沿：
┌─────────────────────────────────────────────────────┐
│  挑战：                                              │
│  - 不对称对齐：一个KG有而另一个没有的实体             │
│  - 一对多/多对多对齐                                  │
│  - 部分对齐：只有少量种子对齐对                       │
│  - 大规模KG的效率问题                                 │
│                                                     │
│  前沿方向：                                          │
│  - 基于LLM的实体对齐：利用语言知识理解实体            │
│  - 自监督对比学习：无需种子对齐对                      │
│  - 增量对齐：处理KG的动态更新                          │
└─────────────────────────────────────────────────────┘
```


## 知识融合 (Knowledge Fusion)


```
知识融合流程：
┌─────────────────────────────────────────────────────┐
│  多源数据 → 实体对齐 → 属性融合 → 冲突消解 → 质量评估 │
│                                                     │
│  1. 多源知识获取                                      │
│     结构化：数据库、已有KG                            │
│     半结构化：百科、表格                              │
│     非结构化：新闻、论文、网页                        │
│                                                     │
│  2. 实体链接与对齐                                    │
│     将不同来源的同一实体关联                          │
│     "北京" = "Beijing" = "Peking"                   │
│                                                     │
│  3. 属性融合                                          │
│     合并同一实体的不同属性值                          │
│     爱因斯坦的出生日期：来源A说1879.3.14             │
│     来源B说March 14, 1879 → 合并                      │
│                                                     │
│  4. 冲突消解                                          │
│     不同来源对同一事实的描述矛盾                      │
│     策略：投票、可信度加权、时间新鲜度                │
└─────────────────────────────────────────────────────┘

冲突消解方法：
┌─────────────────────────────────────────────────────┐
│  1. 真值发现 (Truth Discovery)                       │
│     同时估计数据源可信度和事实真值                    │
│     P(true) = f(source_trustworthiness, claim)      │
│     迭代优化：可信度高的来源权重更大                   │
│                                                     │
│  2. 基于投票                                         │
│     多数投票：选择最多来源支持的值                     │
│     加权投票：根据来源可信度加权                       │
│                                                     │
│  3. 基于外部知识验证                                  │
│     利用权威数据源（政府数据、学术数据库）验证         │
│     使用预训练模型判断语义一致性                       │
│                                                     │
│  4. 时间感知融合                                      │
│     优先选择最新数据                                   │
│     保留历史值的版本信息                               │
│     "CEO of Apple" → 乔布斯(1976-2011) → 库克(2011-) │
└─────────────────────────────────────────────────────┘

知识质量评估：
┌─────────────────────────────────────────────────────┐
│  完整性 (Completeness)：覆盖了多少知识               │
│  准确性 (Accuracy)：知识是否正确                     │
│  一致性 (Consistency)：知识之间是否矛盾              │
│  时效性 (Timeliness)：知识是否过时                   │
│  精细度 (Granularity)：知识的详细程度                │
│                                                     │
│  评估方法：                                          │
│  - 人工抽样评估                                      │
│  - 与权威知识库对比                                  │
│  - 逻辑一致性检查                                    │
│  - 自动化错误检测（异常值检测）                       │
└─────────────────────────────────────────────────────┘
```


## 知识图谱补全 (KGC)


```
知识图谱补全目标：预测缺失的三元组
┌─────────────────────────────────────────────────────┐
│  已知：(爱因斯坦, 出生于, 德国)                       │
│  已知：(爱因斯坦, 提出, 相对论)                       │
│  预测：(爱因斯坦, 职业, ?) → 物理学家                 │
│  预测：(?, 获得, 诺贝尔物理学奖) → 爱因斯坦           │
│                                                     │
│  链接预测任务：                                      │
│  给定 (h, r, ?) 预测 t（尾实体预测）                 │
│  给定 (?, r, t) 预测 h（头实体预测）                 │
└─────────────────────────────────────────────────────┘

基于平移的方法 (TransE系列)：
┌─────────────────────────────────────────────────────┐
│  TransE (Bordes et al., 2013)：                      │
│  核心假设：h + r ≈ t                                 │
│  得分函数：f(h,r,t) = -||h + r - t||                │
│  三元组满足 h + r = t 则得分高                        │
│                                                     │
│  示例：                                              │
│  爱因斯坦 + 出生于 ≈ 德国                            │
│  北京 + 首都 ≈ 中国                                  │
│                                                     │
│  训练：使用负采样的 margin-based ranking loss        │
│  L = max(0, γ + f(h,r,t) - f(h',r',t'))            │
│                                                     │
│  局限：无法建模 1-N, N-1, N-N 关系                   │
│  "出生于"关系：一人出生在一地，但一地出生很多人       │
└─────────────────────────────────────────────────────┐

TransE改进系列：
┌─────────────────────────────────────────────────────┐
│  TransR (2015)：                                     │
│  不同关系映射到不同空间                               │
│  h_r = M_r × h,  t_r = M_r × t                     │
│  f(h,r,t) = -||h_r + r - t_r||                     │
│  解决了TransE在不同关系类型上的局限                    │
│                                                     │
│  TransH (2014)：                                     │
│  将实体投影到关系超平面                               │
│  h_perp = h - w_r^T h × w_r                         │
│  同一实体在不同关系下有不同表示                       │
│                                                     │
│  RotatE (Sun et al., 2019)：                         │
│  将关系建模为复数空间中的旋转                          │
│  t = h ∘ r (Hadamard积), |r_i| = 1                 │
│  可建模对称/反对称/逆/组合关系                        │
│  对称：r_i ∈ {1, -1}                                │
│  逆：r_j = r_i* (共轭)                               │
│                                                     │
│  PairRE (2021)：                                     │
│  头尾实体分别投影                                     │
│  t = h ∘ r_t, r_h = r_t*                            │
│  解决自反关系和对称关系                               │
│                                                     │
│  ComplEx (2016)：                                    │
│  复数空间嵌入                                         │
│  score = Re(Σ h_i × r_i × conj(t_i))               │
│  可处理非对称关系                                     │
└─────────────────────────────────────────────────────┘

基于GNN的补全方法：
┌─────────────────────────────────────────────────────┐
│  R-GCN (Relational GCN)：                            │
│  不同关系类型使用不同的权重矩阵                       │
│  h_i = σ(Σ_r Σ_j W_r × h_j + W_0 × h_i)           │
│  编码KG的图结构信息                                   │
│                                                     │
│  CompGCN：                                           │
│  组合操作替代简单的线性变换                           │
│  支持多种组合方式：乘法、循环卷积等                   │
│  在链接预测任务上效果优秀                             │
│                                                     │
│  SACN (Weighted GCN)：                               │
│  根据关系类型加权邻居聚合                             │
│  结合GCN编码器和ConvE解码器                          │
└─────────────────────────────────────────────────────┘

评估指标：
┌─────────────────────────────────────────────────────┐
│  MRR (Mean Reciprocal Rank)：排名倒数的平均值       │
│  Hits@K：正确实体排在前K名的比例                      │
│  MR (Mean Rank)：正确实体的平均排名                  │
│                                                     │
│  例：预测 (?, 出生于, 德国)                          │
│  正确答案排在第3名 → MRR贡献 = 1/3                   │
│  正确答案在前10 → Hits@10 = 1                        │
└─────────────────────────────────────────────────────┘
```


## 知识推理 (Knowledge Reasoning)


```
知识推理类型：
┌─────────────────────────────────────────────────────┐
│  演绎推理 (Deductive)：从一般到特殊                  │
│  规则：所有的鸟都会飞                                 │
│  事实：企鹅是鸟                                       │
│  结论：企鹅会飞（规则有例外）                          │
│                                                     │
│  归纳推理 (Inductive)：从特殊到一般                  │
│  观察：A是B的父亲，B是C的父亲                         │
│  归纳：X的父亲的父亲 = X的祖父                        │
│                                                     │
│  类比推理 (Analogical)：基于相似性推断               │
│  已知：北京-中国，东京-日本                           │
│  类比：堪培拉-? → 澳大利亚                           │
└─────────────────────────────────────────────────────┘

基于规则的推理：
┌─────────────────────────────────────────────────────┐
│  规则形式：body → head                               │
│  father(X,Y) ∧ father(Y,Z) → grandfather(X,Z)      │
│                                                     │
│  规则挖掘方法：                                      │
│  1. AMIE/AMIE+：从KG中挖掘一阶逻辑规则              │
│     使用支持度、置信度、PCA置信度评估规则质量          │
│                                                     │
│  2. NeuralLP：                                       │
│     可微分的规则学习                                  │
│     将规则学习转化为连续优化问题                      │
│     自动发现软逻辑规则                                │
│                                                     │
│  3. DRUM (2020)：                                    │
│     可微分规则挖掘                                    │
│     使用LSTM生成规则的推理路径                        │
│     端到端学习规则和推理                              │
│                                                     │
│  规则示例：                                          │
│  born_in(X,Y) ∧ located_in(Y,Z) → nationality(X,Z) │
│  works_at(X,Y) ∧ located_in(Y,Z) → work_location(X,Z)│
└─────────────────────────────────────────────────────┘

基于嵌入的推理：
┌─────────────────────────────────────────────────────┐
│  评分函数推理：                                       │
│  使用TransE/RotatE等模型的评分函数                    │
│  score(h,r,t)高的三元组更可能为真                     │
│                                                     │
│  路径推理：                                          │
│  找到连接两个实体的关系路径                            │
│  X --born_in--> 德国 --located_in--> 欧洲            │
│  → X 的国籍在欧洲                                    │
│                                                     │
│  图神经网络推理：                                     │
│  在KG上进行消息传递                                   │
│  聚合邻居信息进行预测                                 │
│  可学习多跳推理模式                                   │
└─────────────────────────────────────────────────────┘

神经符号推理：
┌─────────────────────────────────────────────────────┐
│  结合神经网络的感知能力和符号推理的可解释性             │
│                                                     │
│  DeepProbLog：                                       │
│  将深度学习嵌入ProbLog概率逻辑编程                    │
│  神经网络提供感知（如图像识别）                        │
│  逻辑规则提供推理                                     │
│                                                     │
│  Neural Theorem Prover：                             │
│  可微分的定理证明                                     │
│  将规则推理过程连续化                                  │
│                                                     │
│  应用：                                              │
│  - 需要推理链解释的场景（金融风控、医疗诊断）          │
│  - 结合外部知识的推理                                  │
└─────────────────────────────────────────────────────┘
```


## Neo4j 知识图谱实践


```
Neo4j 简介：
┌─────────────────────────────────────────────────────┐
│  最流行的图数据库之一                                  │
│  原生图存储和处理引擎                                  │
│  查询语言：Cypher（声明式图查询语言）                  │
│  支持ACID事务                                        │
│  提供免费社区版和企业版                                │
└─────────────────────────────────────────────────────┘

Cypher 查询语言基础：
┌─────────────────────────────────────────────────────┐
│  创建节点：                                          │
│  CREATE (e:Person {name: "爱因斯坦", year: 1879})   │
│  CREATE (c:Country {name: "德国"})                   │
│                                                     │
│  创建关系：                                          │
│  MATCH (e:Person {name: "爱因斯坦"})                │
│  MATCH (c:Country {name: "德国"})                   │
│  CREATE (e)-[:BORN_IN]->(c)                         │
│                                                     │
│  查询：                                              │
│  // 查找爱因斯坦出生于哪个国家                         │
│  MATCH (p:Person {name: "爱因斯坦"})-[:BORN_IN]->(c) │
│  RETURN c.name                                      │
│                                                     │
│  // 多跳查询：谁是爱因斯坦出生城市的市长               │
│  MATCH (p:Person {name:"爱因斯坦"})-[:BORN_IN]->     │
│        (city:City)<-[:MAYOR_OF]-(m:Person)          │
│  RETURN m.name                                      │
│                                                     │
│  // 路径查询：找到两个实体之间的最短路径                │
│  MATCH path = shortestPath(                          │
│    (a:Person {name:"爱因斯坦"})-[*]-(b:Person       │
│    {name:"牛顿"})                                   │
│  )                                                   │
│  RETURN path                                         │
└─────────────────────────────────────────────────────┘

Python 操作 Neo4j：
┌─────────────────────────────────────────────────────┐
│  from neo4j import GraphDatabase                    │
│                                                     │
│  # 连接数据库                                        │
│  driver = GraphDatabase.driver(                     │
│      "bolt://localhost:7687",                        │
│      auth=("neo4j", "password")                     │
│  )                                                   │
│                                                     │
│  # 批量导入三元组                                    │
│  def import_triples(tx, triples):                   │
│      for h, r, t in triples:                        │
│          tx.run(                                     │
│              "MERGE (a:Entity {name: $h}) "         │
│              "MERGE (b:Entity {name: $t}) "         │
│              "MERGE (a)-[:RELATION {type: $r}]->(b)",│
│              h=h, r=r, t=t                          │
│          )                                           │
│                                                     │
│  # 执行查询                                          │
│  with driver.session() as session:                  │
│      result = session.run(                          │
│          "MATCH (n)-[r]->(m) RETURN n,r,m LIMIT 10"│
│      )                                               │
│      for record in result:                          │
│          print(record["n"], record["r"], record["m"])│
│                                                     │
│  # 图算法（需要APOC/GDS插件）                        │
│  # PageRank                                          │
│  CALL gds.pageRank.stream('myGraph')                │
│  YIELD nodeId, score                                │
│  RETURN gds.util.asNode(nodeId).name AS name, score │
│  ORDER BY score DESC                                │
│                                                     │
│  # 社区检测                                          │
│  CALL gds.louvain.stream('myGraph')                 │
│  YIELD nodeId, communityId                          │
│  RETURN gds.util.asNode(nodeId).name, communityId   │
└─────────────────────────────────────────────────────┘

Neo4j 在RAG中的应用：
┌─────────────────────────────────────────────────────┐
│  GraphRAG 架构：                                     │
│  用户问题 → 图查询 → 子图提取 → LLM生成回答          │
│                                                     │
│  优势：                                              │

│  - 两者互补，构建混合RAG系统                         │
│                                                     │
│  Microsoft GraphRAG：                                │
```
