NLPSRL知识图谱

语义分析理解句子含义，知识图谱组织结构化知识。

语义角色标注 (SRL)

语义角色标注：识别"谁对谁做了什么" 核心语义角色： - Agent (A0)：施事者（谁做的） - Patient (A1)：受事者（对谁做的） - Instrument (A2)：工具 - Location (AM-LOC)：地点 - Time (AM-TMP)：时间 - Manner (AM-MNR)：方式 示例： "张三 昨天 在图书馆 用电脑 写了 一篇论文" A0 TMP LOC A2 V A1 谓词：写了（V） A0(施事)：张三 A1(受事)：一篇论文 AM-TMP(时间)：昨天 AM-LOC(地点)：在图书馆 AM-MNR(工具)：用电脑 方法演进： - 特征工程 + SVM/CRF - BiLSTM + Attention - BERT-based SRL - 大语言模型直接理解

知识图谱 (Knowledge Graph)

知识图谱：结构化知识的图表示 三元组：(头实体, 关系, 尾实体) (爱因斯坦, 出生于, 德国) (爱因斯坦, 提出, 相对论) (相对论, 属于, 物理学) 知识图谱构建流程： ┌─────────────────────────────────────────────┐ │ 1. 知识抽取 │ │ 实体抽取(NER)：识别实体 │ │ 关系抽取(RE)：识别实体间关系 │ │ 事件抽取：识别事件及其要素 │ │ │ │ 2. 知识融合 │ │ 实体对齐/链接：不同来源的同一实体 │ │ 属性融合：合并同一实体的属性 │ │ 冲突消解：处理矛盾信息 │ │ │ │ 3. 知识存储 │ │ 图数据库：Neo4j, NebulaGraph │ │ RDF三元组库：Jena, Virtuoso │ │ │ │ 4. 知识推理 │ │ 基于规则的推理 │ │ 基于表示学习的推理(TransE/RotatE) │ │ 基于图神经网络的推理 │ └─────────────────────────────────────────────┘ 关系抽取方法： - 远程监督(Distant Supervision)：用知识库自动标注 - Pipeline：先NER后RE - Joint：联合抽取实体和关系 - GPT-based：通过提示提取三元组

实体链接与应用

实体链接 (Entity Linking)： 将文本中的实体提及链接到知识库中的标准实体 "苹果发布了新手机" → 苹果(公司) vs 苹果(水果) 流程： 1. 候选实体生成：查找知识库中的可能实体 2. 实体消歧：根据上下文选择正确实体 使用：上下文相似度、实体流行度、实体间关联 知识图谱应用： ┌─────────────────────────────────────────────┐ │ 智能问答(QA) │ │ "谁提出了相对论？" → 图谱查询 → 爱因斯坦 │ │ │ │ 推荐系统 │ │ 利用实体关系进行推荐 │ │ 用户→喜欢→导演→执导→电影 │ │ │ │ 搜索增强 │ │ Google Knowledge Graph：右侧知识卡片 │ │ 百度知识图谱：搜索结果增强 │ │ │ │ RAG增强 │ │ 结合知识图谱和向量检索的混合RAG │ │ 图谱提供结构化知识，向量检索提供语义匹配 │ └─────────────────────────────────────────────┘ 大规模知识图谱： - Wikidata：维基百科结构化知识 - Freebase（已关闭）：Google知识图谱数据源 - DBpedia：从维基百科抽取 - CN-DBpedia：中文知识图谱
知识图谱与大语言模型的结合是当前热点：LLM提供理解能力，KG提供精确知识，两者互补。
