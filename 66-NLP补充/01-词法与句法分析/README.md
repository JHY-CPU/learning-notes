# 01-词法与句法分析

词法与句法分析是自然语言处理的基础环节，负责将原始文本转化为结构化的语言表示。本章涵盖中文分词、词性标注、命名实体识别和句法分析等核心技术。

---

## 一、中文分词（Chinese Word Segmentation）

中文文本不像英文以空格分隔单词，因此分词是中文NLP的第一步。分词的质量直接影响后续所有任务的性能。

### 1.1 基于规则的方法

基于规则的分词依赖于预先构建的词典，通过字符串匹配进行切分：

- **最大匹配法（Maximum Matching）**：从句子的一端开始，取词典中最长的匹配词。
  - 正向最大匹配（Forward MM）：从左到右扫描
  - 逆向最大匹配（Backward MM）：从右到左扫描
  - 双向最大匹配：正向和逆向结果不一致时，选取词数较少或单字较少的切分
- **优点**：实现简单，速度快
- **缺点**：无法处理未登录词，对词典依赖严重

### 1.2 基于统计的方法

将分词视为序列标注问题，为每个字符标注B（词首）、M（词中）、E（词尾）、S（单字词）标签：

- **N-gram语言模型**：利用相邻字符的共现概率判断是否应该切分
- **HMM（隐马尔可夫模型）**：将字符序列视为观测值，切分标签视为隐状态，通过Viterbi算法求最优标注序列
- **CRF（条件随机场）**：相比HMM，CRF可以利用任意特征函数，且避免了独立性假设，在分词任务上表现更优

### 1.3 基于深度学习的方法

- **BiLSTM-CRF**：双向LSTM捕获上下文信息，CRF层保证标签序列的合法性
- **BERT分词**：利用预训练模型的上下文表示进行序列标注
- **优势**：能够学习到丰富的语义特征，对歧义和未登录词处理更好

### 1.4 歧义切分

中文中存在大量歧义字段，分为以下两种类型：

- **交集型歧义**：字符串ABC中，AB和BC都是词。例如"乒乓球拍卖了"可切分为"乒乓球/拍卖/了"或"乒乓球拍/卖/了"
- **组合型歧义**：字符串AB中，A、B和AB都是词。例如"这个人手不够"中的"人手"是否应切开
- **消歧方法**：利用上下文信息、统计模型概率、以及最大熵模型等进行消歧

### 1.5 未登录词识别

未登录词（OOV，Out-of-Vocabulary）指词典中不存在的词，是分词的主要难点之一：

- **新词发现**：利用统计量（互信息、左右熵）从语料中发现新词
- **命名实体**：人名、地名、机构名等通常为未登录词
- **网络用语**：缩写、谐音、流行语等不断涌现的新表达

### 1.6 分词工具

| 工具 | 特点 | 算法 |
|------|------|------|
| jieba | 简单易用，支持自定义词典 | 基于前缀词典+HMM+Viterbi |
| HanLP | 功能全面，支持多语言 | 规则+统计+深度学习 |
| LTP（哈工大） | 学术界广泛使用 | 深度学习框架 |
| PKUNTH | 北大开发 | 层叠隐马模型 |
| SnowNLP | 轻量级 | 基于朴素贝叶斯 |

## 二、词性标注（POS Tagging）

词性标注是为句子中的每个词赋予一个语法类别标签（名词、动词、形容词等）。

### 2.1 HMM词性标注

将词性标注建模为隐马尔可夫模型：

- **隐状态**：词性标签（如N、V、ADJ等）
- **观测值**：句子中的词语
- **参数**：
  - 转移概率 $P(t_i | t_{i-1})$：前一个词性到当前词性的转移
  - 发射概率 $P(w_i | t_i)$：给定词性产生某个词的概率
  - 初始概率 $P(t_1)$：句子起始位置的词性概率
- **解码算法**：Viterbi算法求最优词性序列
- **参数估计**：利用大规模标注语料，通过最大似然估计（MLE）统计频率

### 2.2 CRF词性标注

CRF相比HMM的优势：

- **判别式模型**：直接建模 $P(Y|X)$，不需要对观测序列建模
- **特征丰富**：可以使用当前词的前后文、词形特征（前后缀、是否包含数字等）
- **无独立性假设**：克服了HMM的观测独立性假设
- **常见特征模板**：当前词、前/后一个词、当前词的首尾字符、是否为数字等

### 2.3 BiLSTM-CRF词性标注

结合深度学习的端到端方法：

- **Embedding层**：将词映射为稠密向量（预训练词向量或字符级CNN）
- **BiLSTM层**：双向捕获上下文信息，生成每个词的表示
- **CRF层**：在BiLSTM输出上添加CRF，利用标签间转移约束保证输出的合理性
- **训练方式**：端到端训练，损失函数为CRF的负对数似然

## 三、命名实体识别（NER）

命名实体识别旨在从非结构化文本中识别出具有特定意义的实体，如人名、地名、机构名等。

### 3.1 基于规则的方法

- **词典匹配**：维护实体词典，通过字符串匹配识别
- **正则表达式**：针对时间、货币、电话号码等格式化实体
- **优点**：精确率高，可解释性强
- **缺点**：召回率低，覆盖面有限，维护成本高

### 3.2 基于序列标注的方法

- **CRF**：利用丰富的手工特征（词形、词性、上下文窗口等）进行序列标注
- **BiLSTM-CRF**：端到端学习特征表示，自动捕获上下文信息
- **标注方案**：BIO（Begin/Inside/Outside）或BIOES（Begin/Inside/End/Single/Outside）

### 3.3 基于BERT的方法

- **BERT-CRF**：BERT编码器替代BiLSTM，利用预训练语言模型的强大表示能力
- **BERT-MRC**：将NER转化为阅读理解问题，为每种实体类型设计问题模板
- **优势**：在少样本和跨领域场景下表现优异

### 3.4 实体类型

| 实体类型 | 英文缩写 | 示例 |
|----------|----------|------|
| 人名 | PER | 张三、李四 |
| 地名 | LOC | 北京、上海、太平洋 |
| 机构名 | ORG | 清华大学、华为公司 |
| 时间 | TIME | 2024年5月、上周三 |
| 货币 | MONEY | 100万元、$500 |
| 数量 | QUANTITY | 100公里、3.5吨 |
| 百分比 | PERCENT | 95%、百分之三十 |

## 四、句法分析

句法分析（Syntactic Parsing）的目标是分析句子的语法结构，揭示词语之间的语法关系。

### 4.1 成分句法分析（Constituency Parsing）

成分句法分析将句子组织成嵌套的短语结构树：

- **短语结构树**：每个非终结符代表一个短语类别（NP、VP、PP等）
- **终结符**：句子中的实际词语
- **示例**：`S -> NP VP`，`VP -> V NP`

#### CKY算法

Cocke-Kasami-Younger（CKY）算法是一种动态规划算法，用于上下文无关文法的解析：

- **前提**：文法需转换为乔姆斯基范式（CNF），即产生式形式为 `A -> BC` 或 `A -> a`
- **数据结构**：三角矩阵 `dp[i][j]` 存储从第i个词到第j个词的可能非终结符
- **时间复杂度**：$O(n^3 \cdot |G|)$，其中n为句子长度，|G|为文法规则数
- **过程**：
  1. 初始化：对每个词，根据词典填充矩阵对角线
  2. 填充：逐步扩大跨度，枚举所有分割点，检查是否存在合法的产生式规则
  3. 回溯：从根节点S回溯得到完整的句法树

#### PCFG（概率上下文无关文法）

PCFG在CFG的基础上为每条产生式规则附加概率：

- **规则概率**：$P(A \to \alpha)$，表示从非终结符A产生 $\alpha$ 的概率
- **树的概率**：所有使用规则概率的乘积
- **参数估计**：从树库中统计各规则的频率，使用MLE估计
- **解析目标**：寻找最大概率句法树（Most Probable Parse）
- **改进CKY**：在动态规划过程中维护最大概率及其来源

### 4.2 依存句法分析（Dependency Parsing）

依存句法分析不关注短语结构，而是直接建立词与词之间的依存关系：

- **核心概念**：
  - 每个词（除根节点）有且仅有一个修饰词（head）
  - 依存关系形成以ROOT为根的树结构
  - 关系类型标注（如主谓关系SBV、动宾关系VOB等）

#### Arc-Standard转移系统

Arc-Standard是一种常用的转移系统，包含三种操作：

1. **SHIFT**：将缓冲区中的下一个词移入栈顶
2. **LEFT-ARC(r)**：将栈顶两个元素中的第二个设为第一个的子节点，标注关系r，然后移除第二个元素
3. **RIGHT-ARC(r)**：将栈顶两个元素中的第一个设为第二个的子节点，标注关系r

- **特点**：每条弧在构建过程中一次性完成，不支持延迟决策

#### Arc-Eager转移系统

Arc-Eager系统支持更早地建立依存关系：

1. **SHIFT**：同Arc-Standard
2. **LEFT-ARC(r)**：将缓冲区顶部的词设为栈顶词的head，标注关系r
3. **RIGHT-ARC(r)**：将栈顶词设为缓冲区顶部词的head，标注关系r
4. **REDUCE**：弹出栈顶词（当其head已确定时）

- **特点**：每条弧可能在其两端的词还未完全处理完时就建立

#### 基于图的方法（MST Parser）

- **核心思想**：将依存分析视为在完全有向图上寻找最大生成树的问题
- **MST算法**：使用Chu-Liu/Edmonds算法求解最大生成树
- **评分函数**：为每对词之间的可能弧打分，学习弧的权重参数
- **全局优化**：直接优化整棵树的分数，考虑全局结构

### 4.3 语义角色标注（SRL）

语义角色标注识别句子中谓词（动词）与其论元之间的语义关系：

- **核心任务**：
  - 谓词识别（Predicate Identification）：确定句子中的谓词
  - 论元识别（Argument Identification）：找出谓词的论元
  - 论元分类（Argument Classification）：标注论元的语义角色
- **常见语义角色**：
  - A0（施事者）：动作的发出者
  - A1（受事者）：动作的承受者
  - A2（间接宾语）：间接受影响者
  - AM-LOC（地点）、AM-TMP（时间）、AM-MNR（方式）等附加角色
- **方法演进**：
  - 早期：基于句法分析结果 + 分类器
  - 深度学习：端到端的BiLSTM-CRF方法
  - 当前：基于BERT的序列标注或阅读理解式方法

## 五、总结

词法与句法分析构成了NLP的底层基础。从分词到句法分析，每一步都为上层的语义理解提供结构化信息。随着深度学习和预训练模型的发展，这些传统任务的性能得到了显著提升，但其核心思想和问题定义仍然是理解现代NLP系统的重要基础。
