# 命名实体识别深入


## 命名实体识别深入


NLPNER序列标注


命名实体识别(NER)是从非结构化文本中识别和分类命名实体的关键NLP任务，是信息抽取、知识图谱构建等下游任务的基础。


## 标注体系详解


```
BIO 标注体系（最常用）：
┌─────────────────────────────────────────────────────┐
│  B-X  (Begin)   实体X的开始                         │
│  I-X  (Inside)  实体X的内部                         │
│  O    (Outside) 非实体部分                          │
│                                                     │
│  示例：小明在北京大学学习                              │
│  小/B-PER  明/I-PER  在/O  北/B-ORG  京/I-ORG       │
│  大/I-ORG  学/I-ORG  学/O  习/O                     │
└─────────────────────────────────────────────────────┘

BIOES 标注体系（更精细）：
┌─────────────────────────────────────────────────────┐
│  B-X  (Begin)   实体X的开始                         │
│  I-X  (Inside)  实体X的内部                         │
│  E-X  (End)     实体X的结束                         │
│  S-X  (Single)  单字实体                            │
│  O    (Outside) 非实体部分                          │
│                                                     │
│  示例：张三去了北京                                   │
│  张/B-PER  三/E-PER  去/O  了/O  北/S-LOC  京/S-LOC │
│  （北京也可以标注为：北/B-LOC  京/E-LOC）             │
└─────────────────────────────────────────────────────┘

BIOES vs BIO 对比：
┌──────────┬──────────────────┬─────────────────────────┐
│ 体系      │ 优势              │ 劣势                     │
├──────────┼──────────────────┼─────────────────────────┤
│ BIO      │ 简单，标注成本低   │ 无法区分单字/多字实体边界 │
│ BIOES    │ 边界信息更精确     │ 标注规则更复杂           │
│          │ 通常效果略好       │ 标注一致性要求更高       │
└──────────┴──────────────────┴─────────────────────────┘

标注一致性挑战：
- 嵌套实体："北京大学"既是ORG也是LOC？
- 边界模糊："苹果公司"还是"苹果"？
- 类型歧义："华盛顿"是PER还是LOC？
- 长实体边界："中华人民共和国国务院"
```


## BiLSTM-CRF 模型


```
BiLSTM-CRF 架构（Lample et al., 2016）：
┌─────────────────────────────────────────────────────┐
│  输入层                                              │
│  ├── 词嵌入（预训练词向量 / 字符级CNN / BPE嵌入）    │
│  └── 字符嵌入（Character-level LSTM / CNN）          │
│       每个字符通过LSTM/CNN得到字符级词表示            │
│       捕获形态学信息（前后缀、词根等）                 │
│                                                     │
│  编码层                                              │
│  └── BiLSTM：前向+后向LSTM编码上下文                 │
│       h_t = [h_forward; h_backward]                 │
│       每个位置得到双向上下文表示                       │
│                                                     │
│  解码层                                              │
│  └── CRF层：学习标签间的转移约束                      │
│       score(y|x) = Σ(emission_t[y_t])              │
│                    + Σ(transition[y_{t-1}][y_t])    │
│       利用Viterbi算法找到全局最优标注序列              │
└─────────────────────────────────────────────────────┘

CRF 的关键作用：
- 学习标签转移规则：B-PER 后面更可能是 I-PER 而非 I-LOC
- 避免非法序列：I-LOC 不可能出现在 O 之后（必须先有 B-LOC）
- 全局最优：不是逐token贪心分类，而是找最优序列

转移矩阵示例（学习到的偏好）：
┌────────────┬─────────┬─────────┬─────────┬─────────┐
│ From\To    │ B-PER   │ I-PER   │ B-LOC   │ O       │
├────────────┼─────────┼─────────┼─────────┼─────────┤
│ START      │ 高分    │ 低分    │ 高分    │ 高分    │
│ B-PER      │ 低分    │ 高分    │ 低分    │ 高分    │
│ I-PER      │ 低分    │ 中等    │ 低分    │ 高分    │
│ B-LOC      │ 低分    │ 低分    │ 低分    │ 高分    │
│ O          │ 高分    │ 低分    │ 高分    │ 高分    │
└────────────┴─────────┴─────────┴─────────┴─────────┘
高分表示该转移是合理的，低分表示不太可能出现的转移
```


## BERT-based NER


```
BERT + CRF/Softmax 微调方案：
┌─────────────────────────────────────────────────────┐
│  [CLS] 小 明 在 北 京 大 学 学 习 [SEP]             │
│    ↓    ↓  ↓  ↓  ↓  ↓  ↓  ↓  ↓  ↓                 │
│  BERT Encoder (12层 Transformer)                    │
│    ↓    ↓  ↓  ↓  ↓  ↓  ↓  ↓  ↓  ↓                 │
│  h_cls h1 h2 h3 h4 h5 h6 h7 h8 h9                  │
│         ↓  ↓  ↓  ↓  ↓  ↓  ↓  ↓  ↓                 │
│       Linear / CRF 分类层                           │
│         ↓  ↓  ↓  ↓  ↓  ↓  ↓  ↓  ↓                 │
│       B-PER I-PER O B-ORG I-ORG I-ORG I-ORG O O    │
└─────────────────────────────────────────────────────┘

两种分类方案：
1. BERT + Softmax
   - 每个token独立分类，简单高效
   - 无法捕获标签间依赖关系
   - 实现：BertForTokenClassification

2. BERT + CRF
   - 在BERT编码后加CRF层
   - 利用转移矩阵约束标签序列
   - 效果通常更好，尤其在短实体上
   - 需要额外的CRF参数训练

训练技巧：
- 学习率：通常比BERT预训练时小（2e-5 ~ 5e-5）
- 最大序列长度：512 tokens（中文约300-400字）
- 数据增强：实体替换、回译
- 对抗训练：FGM/PGD 提升鲁棒性

性能对比（CoNLL-2003英文数据集）：
┌──────────────────────┬────────────┐
│ 模型                  │ F1 Score   │
├──────────────────────┼────────────┤
│ BiLSTM-CRF            │ 90.94      │
│ BERT-Large            │ 92.80      │
│ BERT + CRF            │ 93.10      │
│ RoBERTa + CRF         │ 93.50      │
│ LUKE (实体感知)        │ 94.30      │
└──────────────────────┴────────────┘
```


## 词汇增强 (Lexicon Augmentation)


```
中文NER的特殊挑战：词边界问题
┌─────────────────────────────────────────────────────┐
│  问题：字符级模型丢失词汇信息，词级模型受分词错误影响  │
│                                                     │
│  "南京市长江大桥"                                    │
│  正确分词：南京市/长江/大桥  → NER结果正确            │
│  错误分词：南京/市长/江大桥  → NER结果错误            │
│                                                     │
│  解决方案：词汇增强（融合词典信息）                    │
└─────────────────────────────────────────────────────┘

Lattice LSTM（Zhang & Yang, 2018）：
┌─────────────────────────────────────────────────────┐
│  将词典匹配到的词汇融入字符级LSTM                      │
│                                                     │
│  "发展中国家"                                        │
│  字符序列：发 展 中 国 家                             │
│  匹配词汇：发展、中国、国家、发展中国家                │
│                                                     │
│  每个字符位置：                                       │
│  ├── 字符级LSTM编码                                  │
│  └── 词汇门控：选择性融合匹配到的词汇信息              │
│       gate = σ(W[char; word] + b)                   │
│                                                     │
│  优点：无需分词，利用词典提升效果                      │
│  缺点：词汇间依赖关系不完整，推理速度慢                │
└─────────────────────────────────────────────────────┘

FLAT（Flat-Lattice Transformer, 2020）：
┌─────────────────────────────────────────────────────┐
│  将字符和词汇展平为一个序列，用Transformer编码         │
│                                                     │
│  "发展中国家"                                        │
│  序列：[发, 展, 中, 国, 家, 发展, 中国, 国家,        │
│         发展中国家]                                  │
│                                                     │
│  相对位置编码：                                       │
│  - 字符-字符：普通距离                                │
│  - 字符-词汇：头/尾字符的位置关系                     │
│  - 词汇-词汇：头/尾字符的位置关系                     │
│                                                     │
│  优点：全连接，任意字符/词汇间可直接交互               │
│  效果：在多个中文NER数据集上SOTA                      │
└─────────────────────────────────────────────────────┘

LEBERT / BERT+word information：
- 将词汇信息通过适配层注入BERT
- 利用外部词典（如jieba词典）增强
- 不改变BERT主干结构
```


## 嵌套实体识别 (Nested NER)


```
嵌套实体问题：
┌─────────────────────────────────────────────────────┐
│  一个实体片段可能同时属于多个类型                      │
│                                                     │
│  示例：                                              │
│  "[北京]LOC 大学 [计算机科学]学科"                    │
│  "[北京大学]ORG 的 [计算机科学]学科"                  │
│                                                     │
│  "[Steve Jobs]PER founded [Apple]ORG in [California]LOC"
│  "[Apple]PRODUCT released the new iPhone"            │
│                                                     │
│  传统BIO/BIOES标注无法处理这种重叠！                  │
└─────────────────────────────────────────────────────┘

方法一：片段排列 (Span-based)：
┌─────────────────────────────────────────────────────┐
│  枚举所有可能的文本片段，对每个片段进行分类            │
│                                                     │
│  对于长度为n的句子：                                  │
│  片段数 = n(n+1)/2                                   │
│  每个片段 (i, j)：                                   │
│  - 开头表示：h_i                                     │
│  - 结尾表示：h_j                                     │
│  - span表示：[h_i; h_j; h_i∘h_j]                    │
│  - 分类：PER/LOC/ORG/O + 非实体                     │
│                                                     │
│  优点：天然支持嵌套，一个片段可以被多个分类器标记      │
│  代表：SpERT, PL-Marker                             │
└─────────────────────────────────────────────────────┘

方法二：序列到序列 (Seq2Seq)：
┌─────────────────────────────────────────────────────┐
│  将NER转化为生成任务                                  │
│                                                     │
│  输入："小明在北京大学工作"                            │
│  输出：[PER]小明[/PER] 在[ORG]北京大学[/ORG] 工作     │
│                                                     │
│  或使用指针生成：                                     │
│  输出：(1,2,PER) (4,7,ORG)                           │
│                                                     │
│  代表：BART-NER, T5-NER, UIE (统一信息抽取)          │
│  优点：统一处理各种抽取任务，天然支持嵌套             │
│  缺点：推理速度较慢，可能存在幻觉                     │
└─────────────────────────────────────────────────────┘

方法三：二维标注 (2D NER)：
┌─────────────────────────────────────────────────────┐
│  构建二维表格，(i,j)位置标注span(i,j)的类型           │
│                                                     │
│       起始位置  1    2    3    4                     │
│  结束位置                                            │
│     1         O    -    -    -                       │
│     2         PER  O    -    -                       │
│     3         O    O    O    -                       │
│     4         ORG  O    LOC  O                       │
│                                                     │
│  用CNN/Transformer对表格编码和分类                   │
│  代表：Biaffine NER, Table-Grid NER                 │
└─────────────────────────────────────────────────────┘
```


## 命名实体消歧 (Entity Disambiguation)


```
实体消歧 vs 实体链接：
┌─────────────────────────────────────────────────────┐
│  实体消歧：确定实体的正确含义（不链接到知识库）         │
│  实体链接：将实体提及链接到知识库中的具体条目           │
│                                                     │
│  "苹果发布了新手机"                                   │
│  → 苹果(水果) vs 苹果(公司)                          │
│  实体消歧：选择"公司"                                 │
│  实体链接：链接到 Wikidata Q312 (Apple Inc.)          │
└─────────────────────────────────────────────────────┘

实体消歧方法：

1. 基于上下文相似度：
   ┌─────────────────────────────────────────────────┐
   │  候选实体：c1(水果), c2(公司), c3(唱片公司)       │
   │  上下文："发布 新 手机 CEO"                       │
   │  计算 context_embedding 与每个 candidate 的相似度 │
   │  sim(c2) > sim(c1) > sim(c3) → 选择 c2          │
   └─────────────────────────────────────────────────┘

2. 基于实体流行度：
   ┌─────────────────────────────────────────────────┐
   │  实体在知识库中的出现频率                          │
   │  "苹果" → 苹果(公司) 的流行度远高于苹果(水果)     │
   │  作为先验概率：P(entity)                          │
   │  结合上下文：P(entity|context) ∝ P(c|e) × P(e)  │
   └─────────────────────────────────────────────────┘

3. 基于实体关联图：
   ┌─────────────────────────────────────────────────┐
   │  句子中其他已消歧的实体提供上下文线索               │
   │  "乔布斯创立的苹果公司发布了新iPhone"              │
   │  乔布斯 → Apple Inc. (强关联)                     │
   │  iPhone → Apple Inc. (强关联)                     │
   │  利用知识图谱中实体间的共现和关联关系              │
   └─────────────────────────────────────────────────┘

深度学习消歧模型：
- BLINK：双编码器模型，分别编码提及和实体描述
- GENRE：基于Seq2Seq的生成式实体链接
- REL：轻量级端到端实体链接
```


## 零样本与少样本NER


```
问题定义：
- 零样本NER：不使用目标类型标注数据，识别新类型实体
- 少样本NER：仅使用少量（K-shot）标注样本

范式一：基于元学习 (Meta-learning)：
┌─────────────────────────────────────────────────────┐
│  学习"如何快速适应新类型"                             │
│                                                     │
│  训练阶段：                                          │
│  从训练类型中采样多个episode                         │
│  每个episode：支持集(support) + 查询集(query)        │
│  模型学习从支持集快速迁移到查询集                     │
│                                                     │
│  测试阶段：                                          │
│  给定新类型的少量示例（支持集）                       │
│  模型快速适应并识别新类型实体                         │
│                                                     │
│  代表：ProtoBERT, Pairwise-CRF                      │
└─────────────────────────────────────────────────────┘

范式二：基于提示 (Prompt-based)：
┌─────────────────────────────────────────────────────┐
│  通过自然语言描述定义新类型                           │
│                                                     │
│  输入："找出以下文本中的[MEDICATION]：               │
│         患者服用阿莫西林500mg，每日三次"              │
│                                                     │
│  MEDICATION的描述："药物名称，如抗生素、止痛药等"    │
│                                                     │
│  模型通过理解描述来识别未见过的实体类型               │
│  代表：InstructUIE, UniversalNER                    │
└─────────────────────────────────────────────────────┘

范式三：基于远程监督 (Distant Supervision)：
┌─────────────────────────────────────────────────────┐
│  利用外部知识库自动生成标注数据                       │
│                                                     │
│  1. 构建实体词典（从Wikidata/知识库）                │
│  2. 在大规模文本中进行字典匹配                        │
│  3. 使用匹配结果作为弱监督信号                        │
│  4. 噪声学习：处理远程监督的噪声标签                  │
│                                                     │
│  优点：无需人工标注，可大规模获取数据                  │
│  缺点：噪声大，覆盖率有限                            │
└─────────────────────────────────────────────────────┘

LLM时代的NER：
┌─────────────────────────────────────────────────────┐
│  GPT-4/Claude等大语言模型直接做NER                   │
│                                                     │
│  Zero-shot Prompt：                                 │
│  "请从以下文本中识别所有人名和地名：                   │
│   小明昨天去了北京的清华大学拜访李教授"                │
│                                                     │
│  优势：灵活，无需训练，支持任意类型                   │
│  劣势：速度慢、成本高、边界不够精确、可能存在幻觉     │
│                                                     │
│  最佳实践：                                          │
│  - 结构化输出（JSON Schema约束）                     │
│  - 自洽性验证（多次采样取交集）                       │
│  - 精调小模型+LLM验证的混合方案                      │
└─────────────────────────────────────────────────────┘
```


> **Note:** NER技术从规则到统计再到深度学习不断演进，大语言模型带来了新的零样本NER范式，但传统序列标注方法在精确性和速度上仍有优势。实际应用中常采用混合方案。
