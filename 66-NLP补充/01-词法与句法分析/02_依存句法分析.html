依存句法分析

NLP句法分析依存语法

依存句法分析识别句子中词与词之间的依存关系，是理解句子结构的重要技术。

依存语法基本概念

依存语法（Dependency Grammar）核心思想： ┌─────────────────────────────────────────────────────┐ │ 句子中的每个词（除根节点外）都依赖于另一个词 │ │ 这种依赖关系是有方向的：从核心词（head）到依存词（dependent）│ │ │ │ 例句：小明 在 教室里 认真地 看 了 一本 书 │ │ │ │ 看 (ROOT) │ │ ╱ │ ╲╲ │ │ 小明 在 了 书 │ │ │ ╱ │ │ 教室里 一本 │ │ │ │ │ 认真地 │ └─────────────────────────────────────────────────────┘ 依存关系三要素： 1. 核心词（Head）：被依赖的词 2. 依存词（Dependent）：依赖于核心词的词 3. 关系标签（Relation Label）：依存关系的类型 依存树性质： - 单一核心：每个词有且仅有一个核心词 - 无环：不能出现循环依赖 - 连通：所有词通过依存关系连接 - 有根：存在一个根节点（ROOT）统领全句

依存关系类型（Universal Dependencies）

Universal Dependencies（UD）提供了跨语言统一的依存关系标签集。

核心依存关系： ┌─────────────────┬───────────┬──────────────────────────┐ │ 关系标签 │ 缩写 │ 说明 │ ├─────────────────┼───────────┼──────────────────────────┤ │ nsubj │ 名词主语 │ 主语（名词性） │ │ dobj │ 直接宾语 │ 动作的直接承受者 │ │ iobj │ 间接宾语 │ 动作的间接承受者 │ │ amod │ 形容词修饰 │ 形容词修饰名词 │ │ advmod │ 副词修饰 │ 副词修饰动词/形容词 │ │ det │ 限定词 │ 冠词、指示词 │ │ nummod │ 数量修饰 │ 数词修饰名词 │ │ nmod │ 名词修饰 │ 名词修饰名词（领属等） │ │ case │ 格标记 │ 介词/格助词 │ │ conj │ 并列 │ 并列连词连接的成分 │ │ cc │ 并列连词 │ 并列连词（和、或） │ │ punct │ 标点 │ 标点符号 │ │ root │ 根 │ 句子的核心谓语 │ │ compound │ 复合词 │ 复合名词等 │ │ mark │ 从属标记 │ 引导从句的连词 │ │ aux │ 助动词 │ 助动词 │ │ xcomp │ 开放从句 │ 省略主语的补语从句 │ │ ccomp │ 从句补语 │ 带主语的补语从句 │ │ acl │ 从句修饰 │ 关系从句修饰名词 │ │ appos │ 同位语 │ 同位语 │ └─────────────────┴───────────┴──────────────────────────┘

依存分析方法

1. 基于转移的分析（Transition-Based） ┌──────────────────────────────────────────────────┐ │ 使用移进-归约（Shift-Reduce）策略 │ │ │ │ 数据结构： │ │ - Stack（栈）：已处理的部分 │ │ - Buffer（缓冲区）：待处理的词 │ │ - Arc Set（弧集合）：已建立的依存关系 │ │ │ │ 动作类型（Arc-Standard）： │ │ - SHIFT：将 buffer 首词移入 stack │ │ - LEFT-ARC：stack 顶两个元素建立左依存关系 │ │ - RIGHT-ARC：stack 顶两个元素建立右依存关系 │ │ │ │ 特点： │ │ - 线性时间复杂度 O(n) │ │ - 贪心或 beam search │ │ - 代表：MaltParser, Stanford CoreNLP │ └──────────────────────────────────────────────────┘ 2. 基于图的分析（Graph-Based） ┌──────────────────────────────────────────────────┐ │ 将依存分析建模为最大生成树（MST）问题 │ │ │ │ 步骤： │ │ 1. 对每对词 (i,j) 计算依存分数 score(i,j) │ │ 2. 构建完全有向图，边权为分数 │ │ 3. 找到最高分的有根树 │ │ │ │ 特点： │ │ - 全局最优（非贪心） │ │ - 使用 Eisner 算法（项目化）或 Chu-Liu/Edmonds │ │ - 时间复杂度 O(n³) 或 O(n²) │ │ - 代表：MSTParser, Biaffine Parser │ └──────────────────────────────────────────────────┘ 3. 基于神经网络的分析 ┌──────────────────────────────────────────────────┐ │ Biaffine Parser（Dozat & Manning, 2017） │ │ │ │ 编码器：BiLSTM / Transformer │ │ → 词表示 h_i │ │ │ │ Arc Scoring： │ │ score(i→j) = h_i^T W h_j + U^T h_i + V^T h_j │ │ 或使用 Biaffine 变换 │ │ │ │ Label Scoring： │ │ 对每条弧预测依存关系标签 │ │ │ │ 特点： │ │ - 端到端学习 │ │ - 准确率高（SOTA） │ │ - 可与预训练模型结合（BERT + Biaffine） │ └──────────────────────────────────────────────────┘

Universal Dependencies（UD）框架

UD 是跨语言统一的依存句法标注框架，支持 100+ 种语言。

UD 标注格式（CoNLL-U）： ┌─────────────────────────────────────────────────────┐ │ ID FORM LEMMA UPOS XPOS FEATS HEAD DEPREL│ │ 1 小明 小明 PROPN _ _ 5 nsubj │ │ 2 在 在 ADP _ _ 5 mark │ │ 3 教室里 教室里 NOUN _ _ 2 nmod │ │ 4 认真地 认真地 ADV _ _ 5 advmod │ │ 5 看 看 VERB _ _ 0 root │ │ 6 了 了 PART _ _ 5 aux │ │ 7 一 一 NUM _ _ 8 nummod │ │ 8 本 本 CLA _ _ 9 clf │ │ 9 书 书 NOUN _ _ 5 dobj │ 字段说明： - ID：词在句中的序号 - FORM：词形 - LEMMA：词的原形 - UPOS：通用词性标签（跨语言统一） - XPOS：语言特有词性标签 - FEATS：形态特征（性、数、格等） - HEAD：核心词的 ID - DEPREL：依存关系类型 └─────────────────────────────────────────────────────┘ UD 的优势： - 跨语言一致的标注规范 - 丰富的语言学资源和工具 - 支持迁移学习和多语言模型 - 持续扩展的语言覆盖
UD 框架使得多语言 NLP 模型成为可能。一个在 UD 上训练的模型可以迁移到其他语言，大大减少了低资源语言的标注需求。

依存句法分析应用

┌─────────────────────────────────────────────────────┐ │ 信息抽取 │ │ 识别主谓宾关系 → 提取事件和事实 │ │ 例："苹果公司发布了iPhone 15" │ │ → (苹果公司, 发布, iPhone 15) │ │ │ │ 关系抽取 │ │ 基于依存路径判断实体间关系 │ │ 例："位于北京的清华大学" │ │ → 依存路径：清华大学 --nmod--> 位于 --case--> 北京│ │ → 关系：located_in │ │ │ │ 情感分析 │ │ 基于依存树定位评价对象和情感词 │ │ 例："这家餐厅的服务很好但菜很贵" │ │ → 服务-好(正面)，菜-贵(负面) │ │ │ │ 问答系统 │ │ 理解问题的句法结构 │ │ 识别问题的核心成分（谁、什么、哪里） │ │ │ │ 机器翻译 │ │ 源语言依存结构指导目标语言词序调整 │ │ 编码长距离依赖关系 │ │ │ │ 语义角色标注（SRL） │ │ 依存分析为 SRL 提供结构化特征 │ │ 识别"谁对谁做了什么" │ └─────────────────────────────────────────────────────┘
