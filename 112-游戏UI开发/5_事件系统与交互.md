# 事件系统与交互

## 核心概念

Unity的EventSystem是UGUI输入事件的核心框架，统一处理鼠标、触摸、键盘、手柄等多种输入源，并将它们转化为UI事件分发给对应的UI元素。理解EventSystem的内部工作机制是处理复杂UI交互的前提。

### EventSystem架构详解

EventSystem的工作流程分为四个阶段：

```
输入源（Input Module）
    ↓ 原始输入数据
EventSystem（事件管理器）
    ↓ 射线检测
GraphicRaycaster（确定目标UI）
    ↓ 事件分发
UI元素（实现事件接口的组件）
```

- **EventSystem**：事件系统管理组件，每场景只需一个。管理当前选中对象、处理输入模块的优先级。核心属性`currentSelectedGameObject`记录当前焦点对象。
- **StandaloneInputModule**：处理PC输入（鼠标、键盘）。在新Input System中被InputSystemUIInputModule替代。
- **InputSystemUIInputModule**：新Input System的UI输入模块，支持更灵活的输入绑定和多设备支持。
- **GraphicRaycaster**：挂在Canvas上，负责射线检测确定事件目标。遍历Canvas下所有开启`raycastTarget`的Graphic组件，按深度排序返回命中结果。

### 核心事件接口完整列表

| 接口 | 触发时机 | 典型用途 | 注意事项 |
|------|---------|---------|---------|
| IPointerEnterHandler | 指针进入元素范围 | 高亮效果、Tooltip显示 | 鼠标静止时也会持续触发 |
| IPointerExitHandler | 指针离开元素范围 | 取消高亮、隐藏Tooltip | - |
| IPointerClickHandler | 点击完成（按下+松开在同一元素） | 按钮响应 | 区分于PointerDown |
| IPointerDownHandler | 按下瞬间 | 长按判断开始、按钮按下效果 | 只触发一次 |
| IPointerUpHandler | 松开瞬间 | 按钮弹起效果、长按结束 | 在任意位置松开都会触发 |
| IBeginDragHandler | 开始拖拽（移动超过阈值） | 拖拽初始化 | 需要先Down |
| IDragHandler | 拖拽中（持续移动） | 物品拖动跟随鼠标 | 每帧触发 |
| IEndDragHandler | 结束拖拽（松开） | 拖拽放置判断 | - |
| IDropHandler | 放置目标接收 | 背包格子接收物品 | 仅在放置目标上触发 |
| IScrollHandler | 滚轮滚动 | 缩放UI、滚动列表 | PC和移动端手势 |
| ISelectHandler | 被选中（Tab/手柄导航） | 焦点状态显示 | 需要StandaloneInputModule |
| IDeselectHandler | 取消选中 | 取消焦点状态 | - |
| ISubmitHandler | 确认键（Enter/手柄A键） | 表单提交 | 仅键盘/手柄 |
| IMoveHandler | 导航移动（方向键/摇杆） | 菜单导航 | - |

## 具体实现方法

### EventTrigger组件方式（简单但性能差）

EventTrigger是最简单的方式，但每个EventTrigger组件会增加一定的内存和CPU开销：

```csharp
// 代码方式添加EventTrigger（适合动态创建的UI）
public static void AddEventTrigger(GameObject target,
    EventTriggerType type, UnityEngine.Events.UnityAction<BaseEventData> action)
{
    EventTrigger trigger = target.GetComponent<EventTrigger>();
    if (trigger == null)
        trigger = target.AddComponent<EventTrigger>();

    EventTrigger.Entry entry = new EventTrigger.Entry();
    entry.eventID = type;
    entry.callback.AddListener(action);
    trigger.triggers.Add(entry);
}

// 使用示例
AddEventTrigger(button.gameObject, EventTriggerType.PointerEnter,
    (data) => { button.transform.DOScale(1.1f, 0.1f); });
AddEventTrigger(button.gameObject, EventTriggerType.PointerExit,
    (data) => { button.transform.DOScale(1f, 0.1f); });
```

### 接口实现方式（推荐，性能最优）

直接在MonoBehaviour上实现事件接口，避免额外组件的开销：

```csharp
/// <summary>
/// 完整的可拖拽物品组件
/// 支持拖拽、放置检测、视觉反馈
/// </summary>
public class DraggableItem : MonoBehaviour,
    IBeginDragHandler, IDragHandler, IEndDragHandler, IDropHandler,
    IPointerEnterHandler, IPointerExitHandler, IPointerClickHandler
{
    public InventorySlot SourceSlot { get; set; }
    public ItemData ItemData { get; set; }

    private RectTransform rectTransform;
    private CanvasGroup canvasGroup;
    private Canvas rootCanvas;
    private Transform originalParent;
    private Vector3 originalPosition;
    private int originalSiblingIndex;

    void Awake()
    {
        rectTransform = GetComponent<RectTransform>();
        canvasGroup = GetComponent<CanvasGroup>();
        rootCanvas = GetComponentInParent<Canvas>().rootCanvas;
    }

    // 开始拖拽
    public void OnBeginDrag(PointerEventData eventData)
    {
        originalParent = transform.parent;
        originalPosition = transform.position;
        originalSiblingIndex = transform.GetSiblingIndex();

        // 移动到Canvas根节点以确保在最上层渲染
        transform.SetParent(rootCanvas.transform);
        transform.SetAsLastSibling();

        // 半透明 + 允许穿透（让下方的格子接收OnDrop）
        canvasGroup.alpha = 0.6f;
        canvasGroup.blocksRaycasts = false;
    }

    // 拖拽中
    public void OnDrag(PointerEventData eventData)
    {
        // 跟随鼠标/手指移动
        if (RectTransformUtility.ScreenPointToWorldPointInRectangle(
            rootCanvas.transform as RectTransform,
            eventData.position, eventData.pressEventCamera, out Vector3 worldPos))
        {
            rectTransform.position = worldPos;
        }
    }

    // 结束拖拽
    public void OnEndDrag(PointerEventData eventData)
    {
        canvasGroup.alpha = 1f;
        canvasGroup.blocksRaycasts = true;

        // 如果没有被OnDrop接收，回到原位
        transform.SetParent(originalParent);
        transform.position = originalPosition;
        transform.SetSiblingIndex(originalSiblingIndex);
    }

    // 接收放置（在目标格子上触发）
    public void OnDrop(PointerEventData eventData)
    {
        DraggableItem dragged = eventData.pointerDrag?.GetComponent<DraggableItem>();
        if (dragged == null || dragged == this) return;

        // 交换物品
        if (SourceSlot != null && dragged.SourceSlot != null)
        {
            ItemData temp = SourceSlot.CurrentItem;
            SourceSlot.SetItem(dragged.ItemData);
            dragged.SourceSlot.SetItem(temp);
        }
    }

    // 悬停高亮
    public void OnPointerEnter(PointerEventData eventData)
    {
        if (ItemData != null)
            TooltipManager.Instance.Show(ItemData, rectTransform);
    }

    public void OnPointerExit(PointerEventData eventData)
    {
        TooltipManager.Instance.Hide();
    }

    // 点击
    public void OnPointerClick(PointerEventData eventData)
    {
        if (eventData.button == PointerEventData.InputButton.Right)
        {
            // 右键使用物品
            if (ItemData != null)
                InventoryManager.Instance.UseItem(ItemData);
        }
    }
}
```

### 输入穿透（Raycast Target优化）

所有带有`Graphic`组件的UI元素默认开启`Raycast Target`，会产生不必要的射线检测开销。射线检测的复杂度是O(n)，n为所有开启raycastTarget的Graphic组件数量：

```csharp
/// <summary>
/// 批量优化Raycast Target
/// 在UI预制体初始化时调用，关闭不需要接收事件的元素
/// </summary>
public static class RaycastOptimizer
{
    public static void OptimizeAll(Transform root)
    {
        var graphics = root.GetComponentsInChildren<Graphic>(true);
        int disabled = 0;

        foreach (var graphic in graphics)
        {
            // 装饰性Image（纯背景、分割线）
            if (graphic is Image img && img.sprite == null)
            {
                graphic.raycastTarget = false;
                disabled++;
                continue;
            }

            // 纯显示文本（非输入框、非按钮文字）
            if (graphic is TextMeshProUGUI tmp
                && graphic.GetComponentInParent<TMP_InputField>() == null
                && graphic.GetComponentInParent<Button>() == null)
            {
                graphic.raycastTarget = false;
                disabled++;
                continue;
            }

            // 带有特定标签的装饰元素
            if (graphic.CompareTag("Decoration"))
            {
                graphic.raycastTarget = false;
                disabled++;
            }
        }

        Debug.Log($"Raycast优化: 禁用了 {disabled}/{graphics.Length} 个Graphic的Raycast Target");
    }
}
```

### 自定义Raycast过滤器

```csharp
/// <summary>
/// 圆形射线检测过滤器
/// 替代默认的矩形检测区域，用于圆形按钮等UI
/// </summary>
public class CircleRaycastFilter : MonoBehaviour, ICanvasRaycastFilter
{
    [SerializeField] private float radiusMultiplier = 1f;

    public bool IsRaycastLocationValid(Vector2 screenPoint, Camera eventCamera)
    {
        RectTransform rt = GetComponent<RectTransform>();

        // 将屏幕坐标转换为本地坐标
        if (!RectTransformUtility.ScreenPointToLocalPointInRectangle(
            rt, screenPoint, eventCamera, out Vector2 localPoint))
            return false;

        // 圆形检测
        float radius = Mathf.Min(rt.rect.width, rt.rect.height) * 0.5f * radiusMultiplier;
        return localPoint.magnitude <= radius;
    }
}

/// <summary>
/// 自定义形状射线检测（基于Texture透明度）
/// 只有非透明像素区域才能接收点击
/// </summary>
public class AlphaRaycastFilter : MonoBehaviour, ICanvasRaycastFilter
{
    [SerializeField] private Image targetImage;
    [SerializeField] private float alphaThreshold = 0.1f;

    public bool IsRaycastLocationValid(Vector2 screenPoint, Camera eventCamera)
    {
        if (targetImage == null || targetImage.sprite == null) return false;

        RectTransform rt = targetImage.rectTransform;
        if (!RectTransformUtility.ScreenPointToLocalPointInRectangle(
            rt, screenPoint, eventCamera, out Vector2 localPoint))
            return false;

        // 转换为UV坐标
        Rect rect = rt.rect;
        float u = (localPoint.x - rect.x) / rect.width;
        float v = (localPoint.y - rect.y) / rect.height;

        // 采样纹理alpha值
        Texture2D tex = targetImage.sprite.texture;
        if (u < 0 || u > 1 || v < 0 || v > 1) return false;

        Color pixel = tex.GetPixelBilinear(u, v);
        return pixel.a > alphaThreshold;
    }
}
```

### 复合交互：长按检测

```csharp
/// <summary>
/// 长按检测组件
/// 支持自定义长按阈值和回调
/// </summary>
public class LongPressHandler : MonoBehaviour,
    IPointerDownHandler, IPointerUpHandler, IPointerExitHandler
{
    [SerializeField] private float longPressThreshold = 0.5f;
    [SerializeField] private UnityEngine.Events.UnityEvent onLongPress;

    private float pointerDownTime;
    private bool isPointerDown;
    private bool hasTriggeredLongPress;

    public void OnPointerDown(PointerEventData eventData)
    {
        pointerDownTime = Time.unscaledTime;
        isPointerDown = true;
        hasTriggeredLongPress = false;
    }

    public void OnPointerUp(PointerEventData eventData)
    {
        isPointerDown = false;

        float heldDuration = Time.unscaledTime - pointerDownTime;
        if (heldDuration < longPressThreshold && !hasTriggeredLongPress)
        {
            // 短按（正常点击）
            Debug.Log("短按");
        }
    }

    public void OnPointerExit(PointerEventData eventData)
    {
        isPointerDown = false;
    }

    void Update()
    {
        if (!isPointerDown || hasTriggeredLongPress) return;

        if (Time.unscaledTime - pointerDownTime >= longPressThreshold)
        {
            hasTriggeredLongPress = true;
            onLongPress?.Invoke();
            Debug.Log("长按触发");
        }
    }
}
```

## 最佳实践

- **纯装饰性UI元素关闭`raycastTarget`**以减少射线检测开销（可减少50-80%的检测量）
- **使用接口实现代替EventTrigger组件**，性能更优且更灵活（无额外GameObject组件开销）
- **拖拽时设置`blocksRaycasts = false`**让射线穿透到下方接收区域
- **对复杂不规则UI区域使用自定义RaycastFilter**（圆形、椭圆、基于透明度）
- **始终确保场景中存在EventSystem对象**，否则所有UI事件都无法工作
- **移动端注意触摸事件与手势识别的冲突**，避免滑动屏幕误触UI
- **使用`EventSystem.current.IsPointerOverGameObject()`判断是否在UI上**，防止点击穿透UI触发游戏逻辑

## 常见陷阱与修复

**陷阱1：场景中存在多个EventSystem**
- 症状：输入事件异常，按钮偶尔不响应，事件触发多次
- 修复：确保每场景只有一个EventSystem，跨场景使用DontDestroyOnLoad管理

**陷阱2：UI元素未设置Graphic组件导致无法接收事件**
- 症状：空的GameObject挂了Button但无法点击
- 修复：Button组件需要依赖Selectable，而Selectable需要Graphic组件（Image等）来初始化

**陷阱3：拖拽过程中目标UI的blocksRaycasts未关闭**
- 症状：拖拽的物品挡住了下方格子，无法触发OnDrop
- 修复：在OnBeginDrag中设置`canvasGroup.blocksRaycasts = false`

**陷阱4：忘记在Canvas上添加GraphicRaycaster**
- 症状：World Space Canvas上的按钮点击无反应
- 修复：World Space Canvas必须手动添加GraphicRaycaster组件

**陷阱5：新旧Input System混用导致事件冲突**
- 症状：同时存在StandaloneInputModule和InputSystemUIInputModule
- 修复：只使用其中一个Input Module，推荐迁移到新Input System
