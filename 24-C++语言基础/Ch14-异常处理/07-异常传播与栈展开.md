# 异常传播与栈展开

## 一、概念说明

当异常被抛出后，程序沿调用栈向上查找匹配的`catch`块。在查找过程中，每个被离开的函数的局部对象会被析构，这个过程称为**栈展开**（Stack Unwinding）。栈展开确保了资源的正确释放。

## 二、具体用法

### 2.1 栈展开过程

```cpp
#include <iostream>
#include <stdexcept>

class Resource {
    std::string name;
public:
    Resource(const std::string& n) : name(n) {
        std::cout << "获取资源: " << name << std::endl;
    }
    ~Resource() {
        std::cout << "释放资源: " << name << std::endl;
    }
};

void level3() {
    Resource r3("Level3资源");
    throw std::runtime_error("level3中的异常");
    // r3在这里被析构（栈展开）
}

void level2() {
    Resource r2("Level2资源");
    level3();  // 异常从这里传播
    // r2在这里被析构
}

void level1() {
    Resource r1("Level1资源");
    level2();  // 异常从这里传播
    // r1在这里被析构
}

int main() {
    try {
        level1();
    } catch (const std::exception& e) {
        std::cout << "捕获: " << e.what() << std::endl;
    }
}
```

```output
获取资源: Level1资源
获取资源: Level2资源
获取资源: Level3资源
释放资源: Level3资源
释放资源: Level2资源
释放资源: Level1资源
捕获: level3中的异常
```

### 2.3 构造函数中的栈展开

```cpp
class Container {
    Resource* r1;
    Resource* r2;
public:
    Container() : r1(new Resource("资源1")), r2(nullptr) {
        // 如果这里抛异常，r1已分配但不会自动释放
        r2 = new Resource("资源2");
        // 如果r2分配失败，r1会泄漏！
    }
    ~Container() {
        delete r1;
        delete r2;
    }
};

// 更好的做法：使用智能指针
class SafeContainer {
    std::unique_ptr<Resource> r1;
    std::unique_ptr<Resource> r2;
public:
    SafeContainer()
        : r1(std::make_unique<Resource>("安全资源1")),
          r2(std::make_unique<Resource>("安全资源2")) {}
    // 异常安全：即使r2构造失败，r1也会被自动释放
};
```

## 三、注意事项与常见陷阱

- 栈展开保证局部对象的析构函数被调用
- 如果析构函数在栈展开中抛出第二个异常，会调用`std::terminate`
- 使用智能指针和RAII确保异常时资源释放
- 构造函数中抛异常时，已构造的成员会被析构，但对象本身不会
- 栈展开的性能开销只在异常实际抛出时产生
- `noexcept`函数中抛异常会直接终止程序，不会进行栈展开
