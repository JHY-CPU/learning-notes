# 异常规范演进

## 一、概念说明

C++异常规范经历了从`throw()`到`noexcept`的演进。旧式的动态异常规范（C++98）在C++11中被废弃，C++17中移除了大部分支持，最终被`noexcept`完全替代。

## 二、具体用法

### 2.1 异常规范的历史

```cpp
// C++98 动态异常规范（已废弃）
// void old_style() throw(int, double) {
//     throw 42;  // OK，int在列表中
// }

// C++98 空异常规范（已废弃）
// void never_throw() throw() {
//     // 不抛异常，否则调用std::unexpected
// }

// C++11 noexcept（现代写法）
void modern_never_throw() noexcept {
    // 保证不抛异常，否则调用std::terminate
}
```

### 2.2 noexcept vs throw()

```cpp
#include <iostream>

// noexcept和throw()的区别
void using_noexcept() noexcept {
    // 如果抛异常，直接调用std::terminate
    // 编译器可以更好地优化
}

void using_throw() throw() {
    // C++17中等价于noexcept(true)
    // 但旧标准中有不同的行为（std::unexpected）
}

// 实际对比
int main() {
    std::cout << "noexcept: " << noexcept(using_noexcept()) << std::endl;  // true
    std::cout << "throw(): " << noexcept(using_throw()) << std::endl;      // true
    // 两者在C++17后基本等价
}
```

### 2.3 为什么废弃动态异常规范

```cpp
/*
原因：
1. 运行时检查代价高：每次调用都要验证异常类型
2. 与模板不兼容：模板函数的异常类型在编译时未知
3. 违反时行为不一致：std::unexpected难以使用
4. 维护困难：异常规范需要与实现同步更新
*/

// C++17移除了动态异常规范，只保留noexcept
// 但C++17保留了 throw() 作为 noexcept(true) 的别名
// C++20中 throw() 被完全移除
```

### 2.4 迁移指南

```cpp
// 旧代码迁移
// 从 throw(std::exception) 迁移到 noexcept(false)
// 从 throw() 迁移到 noexcept
// 从无规范 迁移到考虑是否可以添加noexcept

class OldClass {
public:
    // 旧风格
    // void method() throw() { }

    // 新风格
    void method() noexcept { }

    // 析构函数自动noexcept
    ~OldClass() = default;  // 隐式noexcept
};
```

## 三、注意事项与常见陷阱

- C++17中`throw()`被等价为`noexcept(true)`
- C++20完全移除了`throw()`语法
- 现代代码中只使用`noexcept`（带或不带条件）
- `noexcept`提供更强的优化保证（编译器可以省略异常处理代码）
- 迁移旧代码时，将`throw()`替换为`noexcept`
- 动态异常规范`throw(Type)`已完全不可用
