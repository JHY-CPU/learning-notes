# 标准异常类

## 一、概念说明

C++标准库提供了丰富的异常类，每种对应特定的错误类型。正确选择异常类可以使错误语义更清晰，帮助调用者针对性地处理。

## 二、具体用法

### 2.1 logic_error系列

```cpp
#include <stdexcept>
#include <vector>
#include <iostream>

void demo_logic_errors() {
    // invalid_argument：参数无效
    try {
        auto parse = [](const std::string& s) {
            if (s.empty()) throw std::invalid_argument("空字符串");
            return std::stoi(s);
        };
        parse("");
    } catch (const std::invalid_argument& e) {
        std::cout << "invalid_argument: " << e.what() << std::endl;
    }

    // out_of_range：越界访问
    try {
        std::vector<int> v{1, 2, 3};
        if (v.size() <= 5) throw std::out_of_range("索引5超出范围");
    } catch (const std::out_of_range& e) {
        std::cout << "out_of_range: " << e.what() << std::endl;
    }

    // length_error：长度超出限制
    try {
        std::string s;
        s.reserve(std::string::max_size() + 1);
    } catch (const std::length_error& e) {
        std::cout << "length_error: " << e.what() << std::endl;
    }
}
```

### 2.2 runtime_error系列

```cpp
void demo_runtime_errors() {
    // runtime_error：通用运行时错误
    try {
        int result = -1;
        if (result < 0) throw std::runtime_error("操作失败");
    } catch (const std::runtime_error& e) {
        std::cout << "runtime_error: " << e.what() << std::endl;
    }

    // overflow_error：算术溢出
    try {
        int a = INT_MAX;
        if (a > INT_MAX - 1) throw std::overflow_error("整数加法溢出");
    } catch (const std::overflow_error& e) {
        std::cout << "overflow_error: " << e.what() << std::endl;
    }

    // range_error：值域错误
    try {
        double result = 1.0 / 0.0;
        if (std::isinf(result)) throw std::range_error("计算结果无穷大");
    } catch (const std::range_error& e) {
        std::cout << "range_error: " << e.what() << std::endl;
    }
}
```

### 2.3 内存相关异常

```cpp
void demo_memory_errors() {
    // bad_alloc：内存分配失败
    try {
        // 尝试分配超大内存
        size_t huge = static_cast<size_t>(-1);
        int* p = new int[huge];
        delete[] p;
    } catch (const std::bad_alloc& e) {
        std::cout << "bad_alloc: " << e.what() << std::endl;
    }
}
```

### 2.4 异常选择指南

```cpp
/*
| 场景                    | 推荐异常类                |
|------------------------|--------------------------|
| 参数无效               | std::invalid_argument     |
| 索引越界               | std::out_of_range        |
| 字符串/容器长度超限     | std::length_error        |
| 运行时环境错误         | std::runtime_error       |
| 算术溢出               | std::overflow_error      |
| 算术下溢               | std::underflow_error     |
| 值域超出               | std::range_error         |
| 内存分配失败           | std::bad_alloc           |
*/
```

## 三、注意事项与常见陷阱

- `invalid_argument`用于参数本身有问题，`runtime_error`用于操作执行失败
- `std::stoi`等转换函数可能抛出`invalid_argument`或`out_of_range`
- `bad_alloc`在现代系统中很少触发（操作系统通常不会拒绝分配）
- `what()`返回的消息因编译器实现而异
- 标准异常类的构造函数接受`const char*`或`const std::string&`
