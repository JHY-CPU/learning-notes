# 异常与智能指针

## 一、概念说明

智能指针（`unique_ptr`、`shared_ptr`）是RAII的核心工具，在异常发生时自动释放管理的资源。正确使用智能指针可以避免异常安全中的资源泄漏问题。

## 二、具体用法

### 2.1 unique_ptr的异常安全

```cpp
#include <memory>
#include <iostream>
#include <stdexcept>

class Connection {
    std::string name;
public:
    Connection(const std::string& n) : name(n) {
        std::cout << "连接: " << name << std::endl;
    }
    ~Connection() { std::cout << "断开: " << name << std::endl; }
    void query() { std::cout << "查询: " << name << std::endl; }
};

void process() {
    auto conn1 = std::make_unique<Connection>("DB1");
    auto conn2 = std::make_unique<Connection>("DB2");

    conn1->query();
    throw std::runtime_error("查询失败");

    // conn1和conn2都会被自动释放
    conn2->query();
}

int main() {
    try {
        process();
    } catch (const std::exception& e) {
        std::cout << "异常: " << e.what() << std::endl;
    }
}
```

```output
连接: DB1
连接: DB2
查询: DB1
断开: DB2
断开: DB1
异常: 查询失败
```

### 2.2 shared_ptr的引用计数

```cpp
#include <vector>

void shared_ptr_safety() {
    std::vector<std::shared_ptr<Connection>> connections;

    connections.push_back(std::make_shared<Connection>("共享连接1"));
    connections.push_back(std::make_shared<Connection>("共享连接2"));

    // 复制shared_ptr（引用计数增加）
    auto backup = connections[0];

    connections.clear();  // 减少引用计数但不释放
    // backup仍持有连接1

    throw std::runtime_error("处理中断");
    // backup析构时释放连接1
}
```

### 2.3 构造函数中的异常安全

```cpp
class Server {
    std::unique_ptr<Connection> primary;
    std::unique_ptr<Connection> backup;
public:
    Server()
        : primary(std::make_unique<Connection>("主服务器"))
        , backup(std::make_unique<Connection>("备用服务器"))
    {
        // 如果这里抛异常
        // primary和backup都会被安全释放
        if (!primary) throw std::runtime_error("主服务器连接失败");
    }
};
```

### 2.4 自定义删除器

```cpp
// 文件句柄的自定义删除器
auto file_deleter = [](std::FILE* f) {
    if (f) {
        std::fclose(f);
        std::cout << "文件已关闭" << std::endl;
    }
};

void safe_file_operation() {
    std::unique_ptr<std::FILE, decltype(file_deleter)>(
        std::fopen("test.txt", "w"), file_deleter);

    // 即使抛异常，文件也会被关闭
    throw std::runtime_error("写入失败");
}
```

## 三、注意事项与常见陷阱

- 优先使用`make_unique`和`make_shared`（异常安全且高效）
- `make_shared`在构造异常时可能有内存泄漏风险（已知问题）
- 循环引用会导致`shared_ptr`无法释放（用`weak_ptr`解决）
- `unique_ptr`的自定义删除器是类型的一部分
- `shared_ptr`的自定义删除器不是类型的一部分
- 在异常传播路径上使用智能指针管理所有动态资源
