# 异常与析构函数

## 一、概念说明

析构函数中不应抛出异常。如果析构函数在栈展开过程中被调用（即已有异常在传播），此时再抛出第二个异常会导致`std::terminate`被调用，程序立即终止。

## 二、具体用法

### 2.1 析构函数抛异常的危险

```cpp
#include <iostream>
#include <stdexcept>

class Dangerous {
public:
    ~Dangerous() noexcept(false) {  // 允许抛异常（危险！）
        throw std::runtime_error("析构中异常");
    }
};

int main() {
    try {
        Dangerous d;
        throw std::runtime_error("主异常");
        // d析构时抛出第二个异常 → std::terminate
    } catch (...) {
        // 永远不会到这里
    }
}
// 输出：terminate called without an active exception
```

### 2.2 安全的析构函数

```cpp
class SafeResource {
    std::FILE* file;
public:
    SafeResource(const char* filename)
        : file(std::fopen(filename, "r")) {}

    ~SafeResource() noexcept {
        if (file) {
            // 如果关闭失败，记录错误但不抛异常
            if (std::fclose(file) != 0) {
                std::cerr << "警告: 文件关闭失败" << std::endl;
            }
        }
    }
};

class SafeDB {
public:
    ~SafeResource() noexcept {
        try {
            // 将可能抛异常的代码放在try中
            // close_connection();  // 可能抛异常
        } catch (const std::exception& e) {
            // 记录但不重新抛出
            std::cerr << "析构时异常: " << e.what() << std::endl;
        } catch (...) {
            std::cerr << "析构时未知异常" << std::endl;
        }
    }
};
```

### 2.3 默认析构函数

```cpp
// 编译器生成的析构函数默认为noexcept
class DefaultDestructor {
    std::vector<int> data;  // vector的析构是noexcept
public:
    ~DefaultDestructor() = default;  // 隐式noexcept
};

// 如果成员的析构不是noexcept，类的析构也不是
class NonNoexceptMember {
    // 如果某个成员的析构可能抛异常
    // 类的析构也会继承这个属性
};
```

### 2.4 显式标记noexcept

```cpp
class ExplicitNoexcept {
public:
    // 显式标记noexcept
    ~ExplicitNoexcept() noexcept {
        // 确保不会抛异常
        // 可能的操作...
    }
};

// 使用std::terminate的处理器
void my_terminate() {
    std::cerr << "自定义terminate处理器" << std::endl;
    std::abort();
}

int main() {
    std::set_terminate(my_terminate);
    // 如果析构函数抛异常，会调用my_terminate
}
```

## 三、注意事项与常见陷阱

- 析构函数应始终标记为`noexcept`
- 如果析构中可能发生异常，用try-catch包裹并记录
- 编译器生成的析构函数默认`noexcept`
- C++11后析构函数隐式`noexcept`（除非基类或成员不是）
- 在析构函数中做日志记录而非抛异常
- 资源释放的失败通常应记录而非传播
