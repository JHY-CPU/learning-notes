# 异常类型

## 一、概念说明

C++标准库定义了一套异常类层次结构，以`std::exception`为根。了解异常类型层次有助于正确捕获和处理异常，同时为自定义异常提供设计基础。

## 二、标准异常类层次

```
std::exception
├── std::logic_error
│   ├── std::invalid_argument
│   ├── std::domain_error
│   ├── std::length_error
│   ├── std::out_of_range
│   └── std::future_error
├── std::runtime_error
│   ├── std::range_error
│   ├── std::overflow_error
│   ├── std::underflow_error
│   └── std::system_error
│       └── std::filesystem::filesystem_error
├── std::bad_alloc
├── std::bad_cast
├── std::bad_typeid
└── std::bad_exception
```

## 二、具体用法

### 2.1 基类std::exception

```cpp
#include <exception>
#include <iostream>

int main() {
    try {
        throw std::exception();
    } catch (const std::exception& e) {
        std::cout << "异常: " << e.what() << std::endl;
    }
}
```

```output
异常: std::exception
```

### 2.2 捕获不同层次的异常

```cpp
#include <stdexcept>

void test(int choice) {
    switch (choice) {
        case 1: throw std::logic_error("逻辑错误");
        case 2: throw std::invalid_argument("无效参数");
        case 3: throw std::runtime_error("运行时错误");
        case 4: throw std::out_of_range("越界访问");
    }
}

int main() {
    for (int i = 1; i <= 4; ++i) {
        try {
            test(i);
        } catch (const std::invalid_argument& e) {
            std::cout << "[invalid_argument] " << e.what() << std::endl;
        } catch (const std::out_of_range& e) {
            std::cout << "[out_of_range] " << e.what() << std::endl;
        } catch (const std::logic_error& e) {
            std::cout << "[logic_error] " << e.what() << std::endl;
        } catch (const std::runtime_error& e) {
            std::cout << "[runtime_error] " << e.what() << std::endl;
        }
    }
}
```

```output
[logic_error] 逻辑错误
[invalid_argument] 无效参数
[runtime_error] 运行时错误
[out_of_range] 越界访问
```

### 2.3 常见异常使用场景

```cpp
#include <vector>

void validate_age(int age) {
    if (age < 0 || age > 150)
        throw std::invalid_argument("年龄超出合理范围");
}

void access_element(const std::vector<int>& v, size_t index) {
    if (index >= v.size())
        throw std::out_of_range("索引越界");
}

void check_overflow(int a, int b) {
    if (a > 0 && b > 0 && a > INT_MAX - b)
        throw std::overflow_error("整数溢出");
}
```

## 三、注意事项与常见陷阱

- `logic_error`表示程序逻辑错误（应在编码时避免）
- `runtime_error`表示运行时环境错误（无法在编码时预见）
- `what()`返回异常描述的C字符串
- 捕获顺序应从派生类到基类
- 自定义异常应继承`std::exception`或其派生类
- `std::exception::what()`返回的字符串是只读的
