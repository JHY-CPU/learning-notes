# 异常处理概述

## 一、概念说明

C++异常处理机制通过`try`、`catch`和`throw`三个关键字实现错误的检测、传播和处理。异常提供了一种从错误发生点跳转到处理代码的结构化方式，比返回错误码更安全和灵活。

## 二、异常处理基本流程

### 2.1 基本架构

```cpp
#include <iostream>
#include <stdexcept>

double divide(double a, double b) {
    if (b == 0) {
        throw std::runtime_error("除数不能为零");
    }
    return a / b;
}

int main() {
    try {
        double result = divide(10, 0);  // 抛出异常
        std::cout << "结果: " << result << std::endl;
    } catch (const std::runtime_error& e) {
        std::cerr << "捕获异常: " << e.what() << std::endl;
    }

    std::cout << "程序继续执行" << std::endl;
    return 0;
}
```

```output
捕获异常: 除数不能为零
程序继续执行
```

### 2.2 异常传播机制

当异常被抛出后，程序会沿调用栈向上查找匹配的`catch`块。如果在当前函数找不到，异常会传播到调用者，直至找到合适的处理器或程序终止。

```cpp
void inner() {
    throw std::logic_error("内部错误");
}

void middle() {
    inner();  // 异常从这里传播
}

void outer() {
    try {
        middle();
    } catch (const std::exception& e) {
        std::cout << "outer捕获: " << e.what() << std::endl;
    }
}

int main() {
    outer();  // outer捕获: 内部错误
}
```

## 三、异常处理的三个关键元素

1. **throw**：抛出异常对象，中断正常执行流
2. **try**：包裹可能抛出异常的代码块
3. **catch**：捕获并处理特定类型的异常

## 四、注意事项与常见陷阱

- 异常对象会被复制（按值抛出），建议抛出小型对象
- 异常安全是重要的设计考量（资源不会因异常而泄漏）
- 析构函数中不应抛出异常（可能导致`std::terminate`）
- 异常不是控制流工具，应用于真正的异常情况
- C++没有`finally`块，使用RAII替代
- 异常处理有性能开销（零开销模型：不抛异常时无开销）
