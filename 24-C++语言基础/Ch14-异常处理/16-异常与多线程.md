# 异常与多线程

## 一、概念说明

C++中异常不能跨线程传播。如果一个线程抛出异常且未捕获，该线程会终止。`std::exception_ptr`允许在线程间传递异常对象，实现跨线程的异常处理。

## 二、具体用法

### 2.1 线程中的异常处理

```cpp
#include <iostream>
#include <thread>
#include <stdexcept>

void thread_function() {
    try {
        throw std::runtime_error("线程中异常");
    } catch (const std::exception& e) {
        std::cerr << "线程捕获: " << e.what() << std::endl;
    }
}

int main() {
    std::thread t(thread_function);
    t.join();
}
```

```output
线程捕获: 线程中异常
```

### 2.2 exception_ptr传递异常

```cpp
#include <exception>

std::exception_ptr global_exception = nullptr;

void worker_thread() {
    try {
        throw std::runtime_error("工作线程错误");
    } catch (...) {
        global_exception = std::current_exception();  // 捕获当前异常
    }
}

int main() {
    std::thread t(worker_thread);
    t.join();

    if (global_exception) {
        try {
            std::rethrow_exception(global_exception);
        } catch (const std::exception& e) {
            std::cerr << "主线程捕获: " << e.what() << std::endl;
        }
    }
}
```

```output
主线程捕获: 工作线程错误
```

### 2.3 packaged_task和future

```cpp
#include <future>

int async_task() {
    throw std::runtime_error("异步任务失败");
    return 42;
}

int main() {
    std::packaged_task<int()> task(async_task);
    std::future<int> future = task.get_future();

    std::thread t(std::move(task));

    try {
        int result = future.get();  // 在此重新抛出异常
        std::cout << "结果: " << result << std::endl;
    } catch (const std::exception& e) {
        std::cerr << "捕获异步异常: " << e.what() << std::endl;
    }

    t.join();
}
```

### 2.4 async与异常

```cpp
#include <future>

std::string fetch_data() {
    throw std::runtime_error("网络超时");
    return "数据";
}

int main() {
    auto future = std::async(std::launch::async, fetch_data);

    try {
        std::string data = future.get();
        std::cout << "数据: " << data << std::endl;
    } catch (const std::exception& e) {
        std::cerr << "异步异常: " << e.what() << std::endl;
    }
}
```

## 三、注意事项与常见陷阱

- 未捕获的线程异常会导致`std::terminate`（整个程序终止）
- `std::current_exception()`捕获当前正在传播的异常
- `std::rethrow_exception()`重新抛出存储的异常
- `std::future::get()`会重新抛出异步任务中的异常
- 每个线程应有自己的异常处理
- `std::exception_ptr`可以为空（`nullptr`）
