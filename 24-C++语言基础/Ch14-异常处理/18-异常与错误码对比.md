# 异常与错误码对比

## 一、概念说明

C++中错误处理的两种主要方式是异常和错误码（返回值）。两者各有适用场景：异常适合不可恢复的错误和跨层传播，错误码适合高频错误和性能关键路径。

## 二、具体用法

### 2.1 错误码方式

```cpp
#include <iostream>
#include <system_error>

enum class ErrorCode {
    Success = 0,
    InvalidInput,
    NotFound,
    IOError
};

// 返回错误码
ErrorCode read_config(const std::string& path, int& value) {
    if (path.empty()) return ErrorCode::InvalidInput;
    // 模拟读取
    if (path == "missing.cfg") return ErrorCode::NotFound;
    value = 42;
    return ErrorCode::Success;
}

int main() {
    int config_value;
    ErrorCode err = read_config("missing.cfg", config_value);

    if (err != ErrorCode::Success) {
        std::cerr << "错误码: " << static_cast<int>(err) << std::endl;
    }
}
```

### 2.2 异常方式

```cpp
int read_config_ex(const std::string& path) {
    if (path.empty()) throw std::invalid_argument("路径为空");
    if (path == "missing.cfg") throw std::runtime_error("配置文件未找到");
    return 42;
}

int main() {
    try {
        int value = read_config_ex("config.cfg");
        std::cout << "值: " << value << std::endl;
    } catch (const std::exception& e) {
        std::cerr << "异常: " << e.what() << std::endl;
    }
}
```

### 2.3 选择指南

```cpp
/*
| 场景                    | 推荐方式        | 原因                    |
|------------------------|---------------|-----------------------|
| 文件未找到              | 异常           | 不是正常流程             |
| 解析失败               | 异常           | 需要传播到处理层          |
| 数组边界检查            | 异常或断言      | 不应频繁发生             |
| 用户输入验证            | 错误码         | 频繁发生，是正常流程       |
| 网络超时               | 异常           | 需要跨层传播             |
| 资源不足               | 异常           | 难以预测且需要处理        |
| 性能关键循环中的错误     | 错误码         | 异常开销太大             |
*/
```

### 2.4 std::expected（C++23）

```cpp
// expected结合了两者优点（C++23）
// template <typename T, typename E>
// class expected;

// C++17中可用optional模拟
#include <optional>
#include <variant>

std::optional<int> safe_divide(int a, int b) {
    if (b == 0) return std::nullopt;
    return a / b;
}

// variant方式：返回结果或错误信息
std::variant<int, std::string> safe_parse(const std::string& s) {
    try {
        return std::stoi(s);
    } catch (...) {
        return std::string("解析失败: ") + s;
    }
}

int main() {
    auto result = safe_divide(10, 0);
    if (result) {
        std::cout << *result << std::endl;
    } else {
        std::cout << "除零错误" << std::endl;
    }
}
```

## 三、注意事项与常见陷阱

- 不要混用异常和错误码（选择一种并坚持）
- 异常不应作为控制流的一部分
- 错误码容易被忽略（需要调用者主动检查）
- 异常可以强制处理（不捕获则程序终止）
- 在库的公共接口中，考虑提供两者的选择
- `std::system_error`和`std::error_code`是标准的错误码机制
