# 自定义异常类

## 一、概念说明

自定义异常类应继承`std::exception`或其派生类，重写`what()`方法提供错误描述。良好的自定义异常设计包含错误码、上下文信息和清晰的异常层次。

## 二、具体用法

### 2.1 基本自定义异常

```cpp
#include <exception>
#include <string>
#include <iostream>

class DatabaseError : public std::exception {
    std::string msg;
public:
    explicit DatabaseError(const std::string& message) : msg(message) {}

    const char* what() const noexcept override {
        return msg.c_str();
    }
};

void query_db(const std::string& sql) {
    if (sql.empty()) {
        throw DatabaseError("SQL查询为空");
    }
    // 模拟数据库操作
    throw DatabaseError("连接超时: " + sql);
}

int main() {
    try {
        query_db("SELECT * FROM users");
    } catch (const DatabaseError& e) {
        std::cerr << "数据库错误: " << e.what() << std::endl;
    }
}
```

```output
数据库错误: 连接超时: SELECT * FROM users
```

### 2.2 带错误码的异常

```cpp
class AppError : public std::runtime_error {
    int error_code;
public:
    AppError(int code, const std::string& msg)
        : std::runtime_error(msg), error_code(code) {}

    int code() const noexcept { return error_code; }
};

void process_request(int req_type) {
    switch (req_type) {
        case 1: throw AppError(400, "请求格式错误");
        case 2: throw AppError(404, "资源未找到");
        case 3: throw AppError(500, "服务器内部错误");
    }
}

int main() {
    try {
        process_request(2);
    } catch (const AppError& e) {
        std::cout << "错误码: " << e.code()
                  << ", 消息: " << e.what() << std::endl;
    }
}
```

```output
错误码: 404, 消息: 资源未找到
```

### 2.3 异常层次设计

```cpp
// 基础异常
class BaseError : public std::runtime_error {
public:
    using std::runtime_error::runtime_error;
};

// 文件相关异常
class FileError : public BaseError {
    std::string filename;
public:
    FileError(const std::string& file, const std::string& msg)
        : BaseError(msg), filename(file) {}
    const std::string& file() const { return filename; }
};

class FileNotFoundError : public FileError {
public:
    FileNotFoundError(const std::string& file)
        : FileError(file, "文件未找到: " + file) {}
};

void read_file(const std::string& path) {
    if (path.find(".txt") == std::string::npos) {
        throw FileNotFoundError(path);
    }
}

int main() {
    try {
        read_file("data.csv");
    } catch (const FileNotFoundError& e) {
        std::cerr << e.what() << std::endl;
    } catch (const FileError& e) {
        std::cerr << "文件错误: " << e.file() << std::endl;
    }
}
```

```output
文件未找到: data.csv
```

## 三、注意事项与常见陷阱

- `what()`方法必须声明为`noexcept`
- 自定义异常应提供有意义的错误描述
- 避免过于深层的异常继承体系（通常2-3层足够）
- 考虑存储错误码和上下文信息便于调试
- 赋值运算符和拷贝构造应正确处理（通常使用默认）
- 异常类应该轻量，避免在构造函数中做复杂操作
