# 嵌套异常

## 一、概念说明

C++11引入了`std::nested_exception`和`std::throw_with_nested`，支持嵌套异常的抛出和捕获。这允许保留完整的异常调用链，便于调试和错误追踪。

## 二、具体用法

### 2.1 throw_with_nested

```cpp
#include <iostream>
#include <stdexcept>
#include <exception>

void inner_function() {
    throw std::runtime_error("数据库连接失败");
}

void middle_function() {
    try {
        inner_function();
    } catch (...) {
        // 包装当前异常，嵌套存储
        std::throw_with_nested(std::runtime_error("middle_function执行失败"));
    }
}

void outer_function() {
    try {
        middle_function();
    } catch (...) {
        std::throw_with_nested(std::runtime_error("outer_function执行失败"));
    }
}
```

### 2.2 打印嵌套异常链

```cpp
// 递归打印嵌套异常
void print_nested(const std::exception& e, int level = 0) {
    std::cerr << std::string(level * 2, ' ') << e.what() << std::endl;
    try {
        std::rethrow_if_nested(e);
    } catch (const std::exception& nested) {
        print_nested(nested, level + 1);
    } catch (...) {
        std::cerr << std::string((level + 1) * 2, ' ') << "未知异常" << std::endl;
    }
}

int main() {
    try {
        outer_function();
    } catch (const std::exception& e) {
        print_nested(e);
    }
}
```

```output
outer_function执行失败
  middle_function执行失败
    数据库连接失败
```

### 2.3 检测嵌套异常

```cpp
#include <typeinfo>

void check_nested() {
    try {
        throw std::runtime_error("测试");
    } catch (const std::exception& e) {
        // 检查是否有嵌套异常
        try {
            std::rethrow_if_nested(e);
            std::cout << "无嵌套异常" << std::endl;
        } catch (const std::exception& nested) {
            std::cout << "有嵌套异常: " << nested.what() << std::endl;
        }
    }
}
```

### 2.4 nested_exception类

```cpp
// 自定义异常继承nested_exception
class MyError : public std::runtime_error, public std::nested_exception {
public:
    MyError(const std::string& msg) : std::runtime_error(msg) {}
};

void demo_custom_nested() {
    try {
        throw std::logic_error("原始逻辑错误");
    } catch (...) {
        // 用当前异常构造MyError
        throw MyError("包装后的错误");
    }
}
```

## 三、注意事项与常见陷阱

- `std::throw_with_nested`总是抛出异常（不返回）
- 嵌套异常链可以通过递归`rethrow_if_nested`遍历
- 嵌套异常会增加对象大小和复杂度
- `std::nested_exception`可以独立使用（不一定要继承）
- 异常链可能很深，注意递归深度
- 并非所有编译器都完整支持嵌套异常
