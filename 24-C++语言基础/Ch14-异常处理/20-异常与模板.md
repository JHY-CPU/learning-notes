# 异常与模板

## 一、概念说明

模板中的异常处理需要考虑类型相关的异常规范、SFINAE与异常的关系、以及`noexcept`在模板中的传播。C++17后异常规范已成为类型的一部分。

## 二、具体用法

### 2.1 模板中的noexcept

```cpp
#include <iostream>
#include <type_traits>

// 条件noexcept：根据类型决定
template <typename T>
void safe_swap(T& a, T& b) noexcept(std::is_nothrow_move_constructible_v<T> &&
                                     std::is_nothrow_move_assignable_v<T>) {
    T temp = std::move(a);
    a = std::move(b);
    b = std::move(temp);
}

int main() {
    int a = 1, b = 2;
    std::cout << "int swap noexcept: "
              << noexcept(safe_swap(a, b)) << std::endl;  // true

    std::string s1, s2;
    std::cout << "string swap noexcept: "
              << noexcept(safe_swap(s1, s2)) << std::endl;  // true (C++11后)
}
```

### 2.2 SFINAE与异常

```cpp
// 检测函数是否noexcept
template <typename F, typename... Args>
constexpr bool is_nothrow_callable_v =
    noexcept(std::declval<F>()(std::declval<Args>()...));

// 使用enable_if基于异常规范选择实现
template <typename T>
std::enable_if_t<std::is_nothrow_move_constructible_v<T>>
fast_move(T& a, T& b) {
    std::cout << "快速移动" << std::endl;
    T temp = std::move(a);
    a = std::move(b);
    b = std::move(temp);
}

template <typename T>
std::enable_if_t<!std::is_nothrow_move_constructible_v<T>>
fast_move(T& a, T& b) {
    std::cout << "安全拷贝" << std::endl;
    T temp = a;
    a = b;
    b = temp;
}
```

### 2.3 异常规范传播

```cpp
// 如果模板参数的操作是noexcept，整个函数也是noexcept
template <typename Container>
auto get_front(Container& c) noexcept(noexcept(c.front()))
    -> decltype(c.front()) {
    return c.front();
}

// 检查
int main() {
    std::vector<int> v{1, 2, 3};
    std::cout << "vector::front noexcept: "
              << noexcept(get_front(v)) << std::endl;  // true (C++11后)
}
```

### 2.4 异常安全的模板容器

```cpp
template <typename T>
class SafeVector {
    std::vector<T> data;
public:
    // 强异常安全：要么成功要么不变
    void push_back_safe(const T& val) {
        std::vector<T> backup = data;
        try {
            data.push_back(val);
        } catch (...) {
            data = std::move(backup);  // 回滚
            throw;
        }
    }

    // 利用移动语义
    void push_back_safe(T&& val) {
        std::vector<T> backup = std::move(data);
        try {
            data = std::move(backup);
            data.push_back(std::move(val));
        } catch (...) {
            data = std::move(backup);
            throw;
        }
    }
};
```

## 三、注意事项与常见陷阱

- `noexcept`在模板中应使用条件形式`noexcept(expr)`
- 异常规范已成为函数类型的一部分
- 模板中的异常安全级别应明确文档化
- 移动操作的noexcept状态影响容器行为
- `std::is_nothrow_*`系列trait用于编译期检查
