# 异常与构造函数

## 一、概念说明

构造函数中抛出异常会导致对象构造不完整。此时，已构造的基类和成员会被析构，但对象本身的析构函数不会被调用。理解这一机制对正确管理资源至关重要。

## 二、具体用法

### 2.1 构造函数中的异常

```cpp
#include <iostream>
#include <stdexcept>

class Resource {
    std::string name;
public:
    Resource(const std::string& n) : name(n) {
        std::cout << "获取: " << name << std::endl;
    }
    ~Resource() {
        std::cout << "释放: " << name << std::endl;
    }
};

class Manager {
    Resource r1;
    Resource r2;
public:
    Manager()
        : r1("资源1")
        , r2("")  // 假设这里需要额外初始化
    {
        // 如果r2的构造成功但这里抛异常
        // r1和r2都会被析构（成员析构函数被调用）
        if (true) throw std::runtime_error("构造失败");
    }
    ~Manager() { std::cout << "Manager析构" << std::endl; }
};

int main() {
    try {
        Manager m;
    } catch (const std::exception& e) {
        std::cout << "异常: " << e.what() << std::endl;
    }
}
```

```output
获取: 资源1
获取:
释放:
释放: 资源1
异常: 构造失败
```

### 2.2 部分构造与资源泄漏

```cpp
class Dangerous {
    int* data1;
    int* data2;
public:
    Dangerous() : data1(new int(1)), data2(nullptr) {
        data2 = new int(2);  // 如果这里抛异常，data1泄漏！
        throw std::bad_alloc();  // 模拟异常
    }
    ~Dangerous() {
        delete data1;
        delete data2;
    }
};

// 正确做法：使用智能指针
class Safe {
    std::unique_ptr<int> data1;
    std::unique_ptr<int> data2;
public:
    Safe()
        : data1(std::make_unique<int>(1))
        , data2(std::make_unique<int>(2))
    {
        // 即使这里抛异常，data1也会被正确释放
        throw std::runtime_error("安全的异常");
    }
};
```

### 2.3 函数try块处理构造异常

```cpp
class FileProcessor {
    std::FILE* file;
public:
    FileProcessor(const char* filename)
    try : file(std::fopen(filename, "r")) {
        if (!file) {
            throw std::runtime_error("打开文件失败");
        }
    } catch (const std::exception& e) {
        std::cerr << "构造失败: " << e.what() << std::endl;
        throw;  // 重新抛出，对象不会被构造
    }
};
```

### 2.4 委托构造中的异常

```cpp
class Config {
    int value;
public:
    explicit Config(int v) : value(v) {
        if (v < 0) throw std::invalid_argument("值不能为负");
    }

    // 委托构造：异常从被委托的构造函数传播
    Config() : Config(0) {}  // OK
    // Config() : Config(-1) {}  // 会抛异常
};
```

## 三、注意事项与常见陷阱

- 构造函数抛异常时，对象析构函数不会被调用
- 已构造的成员和基类的析构函数会被调用
- 使用智能指针避免构造函数中的资源泄漏
- 函数try块可以捕获成员初始化时的异常
- 委托构造中异常从被委托构造传播
- 构造函数不应是`noexcept`除非真的不会失败
