# 异常性能开销

## 一、概念说明

C++采用"零开销"异常模型：不抛异常时没有运行时开销，但抛出和捕获异常时有显著的性能代价。理解异常的性能特征有助于在性能关键代码中做出正确决策。

## 二、具体用法

### 2.1 零开销模型

```cpp
#include <iostream>
#include <chrono>

// 不抛异常时：零运行时开销
int no_exception(int a, int b) {
    return a + b;  // 没有额外的异常处理代码
}

// 抛异常时：需要展开栈、查找catch、调用析构
int with_exception(int a, int b) {
    if (b == 0) throw std::runtime_error("除零");
    return a / b;
}
```

### 2.2 性能对比

```cpp
void benchmark() {
    const int N = 1000000;

    // 测试1：无异常情况
    auto start = std::chrono::high_resolution_clock::now();
    for (int i = 0; i < N; ++i) {
        try {
            no_exception(i, i + 1);
        } catch (...) {}
    }
    auto end = std::chrono::high_resolution_clock::now();
    auto duration1 = std::chrono::duration_cast<std::chrono::microseconds>(end - start);

    // 测试2：频繁抛异常（非常慢）
    start = std::chrono::high_resolution_clock::now();
    for (int i = 0; i < N / 100; ++i) {  // 减少次数
        try {
            with_exception(i, 0);
        } catch (...) {}
    }
    end = std::chrono::high_resolution_clock::now();
    auto duration2 = std::chrono::duration_cast<std::chrono::microseconds>(end - start);

    std::cout << "无异常: " << duration1.count() << "μs" << std::endl;
    std::cout << "抛异常: " << duration2.count() << "μs" << std::endl;
    // 异常抛出通常比无异常慢数百到数千倍
}
```

### 2.4 替代方案对比

```cpp

// 返回错误码
std::optional<int> divide_opt(int a, int b) {
    if (b == 0) return std::nullopt;
    return a / b;
}

// 返回expected（C++23）
// std::expected<int, std::string> divide_exp(int a, int b);

void compare_approaches() {
    const int N = 1000000;

    // optional方式：稳定且快速

    for (int i = 0; i < N; ++i) {
```

## 三、注意事项与常见陷阱

- 异常不应用于常规控制流（性能代价过高）
- 零开销意味着不抛异常时无性能损失
- 抛出异常比返回错误码慢数百到数千倍
- 表驱动方式增加二进制文件大小
- 在热循环中避免使用异常
- 考虑`std::optional`、`std::expected`作为轻量替代
