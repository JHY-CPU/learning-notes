# 异常安全保证

## 一、概念说明

异常安全保证定义了函数在面对异常时的行为。David Abrahams提出了三个级别的异常安全保证：**基本保证**（无泄漏）、**强保证**（事务语义）和**不抛异常保证**（`noexcept`）。

## 二、异常安全级别

### 2.1 基本保证（Basic Guarantee）

```cpp
#include <vector>
#include <iostream>

class Container {
    std::vector<int> data;
    int* extra_resource;
public:
    Container() : extra_resource(new int(0)) {}

    // 基本保证：即使抛异常，不会泄漏资源
    void add_element(int val) {
        int* temp = new int(val);       // 可能抛出bad_alloc
        data.push_back(val);            // 可能抛出bad_alloc（扩容）
        // 如果push_back失败，temp会泄漏——不满足基本保证
        delete extra_resource;
        extra_resource = temp;          // 此时data已修改，无法回滚
    }
};
```

### 2.2 强保证（Strong Guarantee）

```cpp
class SafeContainer {
    std::vector<int> data;
public:
    // 强保证：要么完全成功，要么状态不变
    void add_element(int val) {
        std::vector<int> backup = data;  // 先备份
        data.push_back(val);             // 修改副本
        // 如果成功，用新数据替换（swap不会抛异常）
        backup.swap(data);
    }
    // 异常安全的交换
    void swap(SafeContainer& other) noexcept {
        data.swap(other.data);
    }
};
```

### 2.3 不抛异常保证（No-Throw Guarantee）

```cpp
class NoThrowExample {
    int value;
public:
    // 不抛异常保证：承诺绝不抛出异常
    NoThrowExample(int v) noexcept : value(v) {}

    int get() const noexcept { return value; }

    void swap(NoThrowExample& other) noexcept {
        std::swap(value, other.value);
    }

    // 移动构造通常应标记为noexcept
    NoThrowExample(NoThrowExample&& other) noexcept
        : value(std::exchange(other.value, 0)) {}
};
```

### 2.4 安全级别对比

```cpp
/*
| 级别         | 保证                     | 适用场景            |
|-------------|--------------------------|-------------------|
| 基本保证     | 无资源泄漏，对象可析构    | 最低要求           |
| 强保证       | 操作成功或状态完全不变    | 事务性操作         |
| 不抛异常保证 | 绝不抛出异常             | 析构函数、swap、移动 |
*/
```

```cpp
// 实际示例：copy-and-swap实现强保证
class StrongSafeBuffer {
    int* data;
    size_t size;
public:
    StrongSafeBuffer(size_t n) : data(new int[n]), size(n) {}

    // copy-and-swap赋值：强保证
    StrongSafeBuffer& operator=(StrongSafeBuffer other) noexcept {
        swap(other);  // swap必须不抛异常
        return *this;
    }

    void swap(StrongSafeBuffer& other) noexcept {
        std::swap(data, other.data);
        std::swap(size, other.size);
    }

    ~StrongSafeBuffer() { delete[] data; }
};
```

## 三、注意事项与常见陷阱

- 应为每个函数明确其异常安全级别
- 析构函数、`swap`、`move`操作应提供不抛异常保证
- copy-and-swap是实现强保证的经典模式
- 强保证需要额外的备份成本，需权衡性能
- 容器操作的异常安全：`push_back`提供基本保证
- C++11的移动语义使实现强保证更高效
