# 捕获异常的最佳实践

## 一、概念说明

正确捕获异常需要注意引用捕获、catch顺序、异常信息保留等方面。不当的异常捕获会导致对象切片、信息丢失或程序行为异常。

## 二、具体用法

### 2.1 引用捕获

```cpp
#include <iostream>
#include <stdexcept>

// 推荐：const引用捕获
void good_catch() {
    try {
        throw std::runtime_error("测试");
    } catch (const std::exception& e) {
        // 引用捕获避免对象切片
        std::cout << e.what() << std::endl;
    }
}

// 不推荐：值捕获（对象切片）
void bad_catch() {
    try {
        throw std::runtime_error("测试");
    } catch (std::exception e) {
        // e是std::exception的切片，丢失派生类信息
        std::cout << e.what() << std::endl;  // 信息可能不完整
    }
}
```

### 2.2 catch顺序

```cpp
void correct_order() {
    try {
        throw std::out_of_range("越界");
    } catch (const std::out_of_range& e) {  // 最具体
        std::cout << "out_of_range: " << e.what() << std::endl;
    } catch (const std::logic_error& e) {   // 中间层
        std::cout << "logic_error: " << e.what() << std::endl;
    } catch (const std::exception& e) {     // 最通用
        std::cout << "exception: " << e.what() << std::endl;
    } catch (...) {                          // 兜底
        std::cout << "未知异常" << std::endl;
    }
}
```

### 2.3 日志与重新抛出

```cpp
#include <exception>

void log_and_rethrow() {
    try {
        throw std::runtime_error("数据库错误");
    } catch (const std::exception& e) {
        // 记录日志但不处理
        std::cerr << "[LOG] 捕获异常: " << e.what() << std::endl;
        throw;  // 重新抛出原始异常（保持类型和信息）
    }
}

// 包装异常以添加上下文
void wrap_exception() {
    try {
        throw std::runtime_error("原始错误");
    } catch (const std::exception& e) {
        throw std::runtime_error(
            std::string("处理失败: ") + e.what());
    }
}
```

### 2.4 捕获所有异常

```cpp
void handle_all() {
    try {
        // 可能抛出任何类型的异常
        throw 42;
    } catch (const std::exception& e) {
        std::cerr << "标准异常: " << e.what() << std::endl;
    } catch (int code) {
        std::cerr << "错误码: " << code << std::endl;
    } catch (...) {
        // 记录但通常需要重新抛出或终止
        std::cerr << "未知异常，记录后重新抛出" << std::endl;
        throw;
    }
}
```

### 2.5 异常转换

```cpp
// 在库边界转换异常
class LibraryError : public std::runtime_error {
public:
    using std::runtime_error::runtime_error;
};

void library_function() {
    try {
        // 内部实现可能抛出各种异常
        throw std::bad_alloc();
    } catch (...) {
        // 转换为库的公共异常类型
        throw LibraryError("库内部错误");
    }
}
```

## 三、注意事项与常见陷阱

- 始终用`const`引用捕获异常，避免对象切片
- catch块顺序从具体到通用
- `throw;`重新抛出原始异常，`throw e;`会切片
- 在异常边界处记录日志并考虑是否重新抛出
- 不要静默吞掉异常（空catch块）
- 考虑在库边界处转换异常类型
