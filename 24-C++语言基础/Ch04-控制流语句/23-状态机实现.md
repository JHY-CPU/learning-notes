# 状态机实现

## 一、概念说明

有限状态机（FSM）用switch实现是一种经典模式：每个case代表一个状态，状态转换通过改变状态变量实现。

## 二、具体用法

### 2.1 用switch实现状态机

```cpp
#include <iostream>
using namespace std;

enum class State {
    Idle,
    Running,
    Paused,
    Stopped
};

int main() {
    State state = State::Idle;
    // 模拟事件序列
    string events[] = {"start", "pause", "resume", "stop", "start"};
    int eventIdx = 0;

    while (eventIdx < 5) {
        string event = events[eventIdx++];
        cout << "事件: " << event << " -> ";

        switch (state) {
            case State::Idle:
                if (event == "start") {
                    state = State::Running;
                    cout << "开始运行";
                } else {
                    cout << "（忽略）";
                }
                break;

            case State::Running:
                if (event == "pause") {
                    state = State::Paused;
                    cout << "暂停";
                } else if (event == "stop") {
                    state = State::Stopped;
                    cout << "停止";
                } else {
                    cout << "（忽略）";
                }
                break;

            case State::Paused:
                if (event == "resume") {
                    state = State::Running;
                    cout << "恢复运行";
                } else if (event == "stop") {
                    state = State::Stopped;
                    cout << "停止";
                } else {
                    cout << "（忽略）";
                }
                break;

            case State::Stopped:
                if (event == "start") {
                    state = State::Running;
                    cout << "重新开始";
                } else {
                    cout << "（忽略）";
                }
                break;
        }
        cout << endl;
    }

    return 0;
}
```

输出：
```
事件: start -> 开始运行
事件: pause -> 暂停
事件: resume -> 恢复运行
事件: stop -> 停止
事件: start -> 重新开始
```

## 三、注意事项与常见陷阱

1. **状态必须有default**：处理未预期的状态
2. **事件必须处理**：每个状态对每个事件都应有明确的响应（包括忽略）
3. **复杂状态机考虑类**：用类封装状态和转换逻辑
4. **状态表模式**：大量状态和事件时，用表驱动替代switch
5. **线程安全**：多线程访问状态机需要加锁
