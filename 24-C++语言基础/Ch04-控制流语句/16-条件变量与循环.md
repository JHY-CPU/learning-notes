# 条件变量与循环

## 一、概念说明

在多线程编程中，线程需要在**某个条件满足时**才继续执行。经典的模式是在循环中等待条件变量（condition variable），避免**虚假唤醒**。

## 二、具体用法

### 2.1 经典的等待循环模式

```cpp
#include <iostream>
#include <thread>
#include <mutex>
#include <condition_variable>
using namespace std;

mutex mtx;
condition_variable cv;
bool ready = false;
int data = 0;

void producer() {
    this_thread::sleep_for(chrono::milliseconds(100));
    {
        lock_guard<mutex> lock(mtx);
        data = 42;
        ready = true;
        cout << "生产者: 数据就绪" << endl;
    }
    cv.notify_one();
}

void consumer() {
    unique_lock<mutex> lock(mtx);
    // 经典模式：在循环中等待
    while (!ready) {  // 必须用循环，防止虚假唤醒
        cv.wait(lock);
    }
    cout << "消费者: 收到数据 " << data << endl;
}

int main() {
    thread t1(consumer);
    thread t2(producer);
    t1.join();
    t2.join();

    return 0;
}
```

输出：
```
生产者: 数据就绪
消费者: 收到数据 42
```

### 2.2 使用lambda的简洁写法

```cpp
#include <iostream>
#include <thread>
#include <mutex>
#include <condition_variable>
using namespace std;

mutex mtx;
condition_variable cv;
queue<int> tasks;

void worker(int id) {
    while (true) {
        unique_lock<mutex> lock(mtx);
        // lambda版本：等价于 while(!pred) wait();
        cv.wait(lock, []{ return !tasks.empty(); });

        int task = tasks.front();
        tasks.pop();
        lock.unlock();

        if (task == -1) break;  // 退出信号
        cout << "工作者" << id << ": 处理任务 " << task << endl;
    }
}

int main() {
    thread t1(worker, 1);

    {
        lock_guard<mutex> lock(mtx);
        tasks.push(1);
        tasks.push(2);
        tasks.push(-1);  // 退出信号
    }
    cv.notify_all();

    t1.join();

    return 0;
}
```

输出：
```
工作者1: 处理任务 1
工作者1: 处理任务 2
```

## 三、注意事项与常见陷阱

1. **必须用循环等待**：`while(!cond)`或`cv.wait(lock, pred)`，不能用`if`
2. **虚假唤醒**：条件变量可能在没有notify时返回，循环可以重新检查条件
3. **持有锁等待**：wait会自动释放锁，被唤醒后重新获取锁
4. **notify_all vs notify_one**：多个线程等待同一条件用notify_all
5. **退出条件**：提供明确的退出机制（如特殊值-1）
