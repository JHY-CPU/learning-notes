# 多条件分支优化

## 一、概念说明

当有多个条件分支时，不同方案的性能和可读性不同。`if-else链`适合少数分支，`switch`适合整型多分支，`查表法`适合大量分支。

## 二、具体用法

### 2.1 三种方式对比

```cpp
#include <iostream>
#include <unordered_map>
#include <functional>
using namespace std;

// 方式1：if-else链
string ifElseMethod(int code) {
    if (code == 200) return "OK";
    else if (code == 404) return "Not Found";
    else if (code == 500) return "Server Error";
    else if (code == 301) return "Moved";
    else if (code == 403) return "Forbidden";
    else return "Unknown";
}

// 方式2：switch
string switchMethod(int code) {
    switch (code) {
        case 200: return "OK";
        case 404: return "Not Found";
        case 500: return "Server Error";
        case 301: return "Moved";
        case 403: return "Forbidden";
        default: return "Unknown";
    }
}

// 方式3：查表法
string lookupMethod(int code) {
    static const unordered_map<int, string> table = {
        {200, "OK"}, {404, "Not Found"}, {500, "Server Error"},
        {301, "Moved"}, {403, "Forbidden"}
    };
    auto it = table.find(code);
    return (it != table.end()) ? it->second : "Unknown";
}

int main() {
    cout << ifElseMethod(200) << endl;
    cout << switchMethod(404) << endl;
    cout << lookupMethod(500) << endl;

    return 0;
}
```

输出：
```
OK
Not Found
Server Error
```

### 2.2 选择指南

| 条件 | 推荐方式 | 原因 |
|------|----------|------|
| 2-3个分支 | if-else | 简洁 |
| 整型/枚举、4-10分支 | switch | 编译器优化 |
| 大量分支、运行时值 | 查表 | O(1)查找 |
| 字符串匹配 | unordered_map | switch不支持字符串 |
| 需要动态修改 | 函数表 | 灵活性 |
| 复杂条件 | if-else | switch只支持等值比较 |

## 三、注意事项与常见陷阱

1. **switch比if-else快**：编译器可能将switch优化为跳转表
2. **查表有初始化开销**：static表只初始化一次
3. **字符串匹配用map**：switch不支持string类型的case
4. **复杂条件只能用if**：switch只支持等值比较
5. **性能差异可能很小**：可读性应优先于微优化
