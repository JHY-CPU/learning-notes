# 控制流与性能

## 一、概念说明

控制流语句（特别是分支语句）影响CPU的**分支预测**效率。现代CPU通过预测分支走向来预取指令，预测失败会导致流水线清空，造成性能损失。

## 二、具体用法

### 2.1 分支预测

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
#include <chrono>
using namespace std;

int main() {
    const int N = 10000000;
    vector<int> data(N);
    for (int i = 0; i < N; i++) data[i] = rand() % 100;

    // 无序数据：分支预测频繁失败
    auto start = chrono::high_resolution_clock::now();
    int sum1 = 0;
    for (int x : data) {
        if (x >= 50) sum1 += x;
    }
    auto mid = chrono::high_resolution_clock::now();

    // 排序后：分支更容易预测
    sort(data.begin(), data.end());
    int sum2 = 0;
    for (int x : data) {
        if (x >= 50) sum2 += x;
    }
    auto end = chrono::high_resolution_clock::now();

    auto t1 = chrono::duration_cast<chrono::microseconds>(mid - start).count();
    auto t2 = chrono::duration_cast<chrono::microseconds>(end - mid).count();

    cout << "无序: " << t1 << " 微秒, sum=" << sum1 << endl;
    cout << "排序: " << t2 << " 微秒, sum=" << sum2 << endl;

    return 0;
}
```

输出（示例）：
```
无序: 25000 微秒, sum=247536845
排序: 8000 微秒, sum=247536845
```

### 2.2 避免分支的技术

```cpp
#include <iostream>
#include <vector>
using namespace std;

int main() {
    vector<int> data = {5, 3, 8, 1, 9, 2, 7};

    // 使用分支
    int sum1 = 0;
    for (int x : data) {
        if (x > 4) sum1 += x;
    }

    // 避免分支：使用算术运算
    int sum2 = 0;
    for (int x : data) {
        sum2 += x * (x > 4);  // bool转int，无分支
    }

    // 使用无分支max
    auto branchlessMax = [](int a, int b) {
        return a ^ ((a ^ b) & -(a < b));
    };

    cout << "分支求和: " << sum1 << endl;
    cout << "无分支求和: " << sum2 << endl;
    cout << "max(5,3) = " << branchlessMax(5, 3) << endl;

    return 0;
}
```

输出：
```
分支求和: 29
无分支求和: 29
max(5,3) = 5
```

### 2.3 C++20 [[likely]]和[[unlikely]]

```cpp
#include <iostream>
using namespace std;

int main() {
    int x = 42;

    // C++20：提示编译器分支概率
    if (x > 0) [[likely]] {
        cout << "大概率执行" << endl;
    } else [[unlikely]] {
        cout << "小概率执行" << endl;
    }

    // 适用于错误检查
    // if (ptr == nullptr) [[unlikely]] {
    //     throw runtime_error("空指针");
    // }

    // 编译器可能根据这些属性优化指令布局

    return 0;
}
```

输出：
```
大概率执行
```

## 三、注意事项与常见陷阱

1. **先确保正确性**：不要为了性能牺牲代码可读性和正确性
2. **分支预测器很强大**：现代CPU的分支预测准确率超过95%
3. **无分支不总是更快**：有些无分支代码反而更慢（增加指令依赖）
4. **[[likely]]是提示**：编译器可以忽略这些属性
5. **数据特性影响性能**：有序数据分支预测准确率更高
