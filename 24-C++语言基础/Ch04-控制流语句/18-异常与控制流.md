# 异常与控制流

## 一、概念说明

异常处理机制使用`try`、`catch`和`throw`改变了正常的控制流。抛出异常时，程序立即跳出当前调用栈，寻找匹配的catch处理器。

## 二、具体用法

### 2.1 基本异常处理

```cpp
#include <iostream>
#include <stdexcept>
using namespace std;

double divide(double a, double b) {
    if (b == 0) {
        throw invalid_argument("除数不能为零");
    }
    return a / b;
}

int main() {
    try {
        cout << "10 / 3 = " << divide(10, 3) << endl;
        cout << "10 / 0 = " << divide(10, 0) << endl;  // 抛出异常
        cout << "这行不会执行" << endl;
    } catch (const invalid_argument& e) {
        cerr << "错误: " << e.what() << endl;
    } catch (const exception& e) {
        cerr << "其他异常: " << e.what() << endl;
    }

    cout << "程序继续执行" << endl;

    return 0;
}
```

输出：
```
10 / 3 = 3.33333
错误: 除数不能为零
程序继续执行
```

### 2.2 异常改变控制流

```cpp
#include <iostream>
using namespace std;

void level3() {
    cout << "进入level3" << endl;
    throw runtime_error("level3的错误");
    cout << "level3结束（不会执行）" << endl;
}

void level2() {
    cout << "进入level2" << endl;
    level3();
    cout << "level2结束（不会执行）" << endl;
}

void level1() {
    cout << "进入level1" << endl;
    try {
        level2();
    } catch (const exception& e) {
        cout << "level1捕获: " << e.what() << endl;
    }
    cout << "level1结束" << endl;
}

int main() {
    level1();
    cout << "main结束" << endl;
    return 0;
}
```

输出：
```
进入level1
进入level2
进入level3
level1捕获: level3的错误
level1结束
main结束
```

## 三、注意事项与常见陷阱

1. **异常会跳过中间代码**：抛出点到catch之间所有代码被跳过
2. **析构函数中不要抛异常**：可能导致`std::terminate`
3. **资源泄漏风险**：异常发生时可能跳过释放资源的代码，用RAII解决
4. **catch顺序**：先捕获派生类，再捕获基类
5. **异常安全保证**：了解基本保证、强保证和不抛异常保证
