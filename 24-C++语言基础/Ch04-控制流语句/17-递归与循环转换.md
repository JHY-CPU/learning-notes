# 递归与循环转换

## 一、概念说明

递归和循环是**等价的**：任何递归都可以转换为循环，反之亦然。递归更简洁直观但有栈溢出风险，循环效率更高但可能不够直观。

## 二、具体用法

### 2.1 递归转循环

```cpp
#include <iostream>
using namespace std;

// 递归版本
int factorialRecursive(int n) {
    if (n <= 1) return 1;
    return n * factorialRecursive(n - 1);
}

// 循环版本
int factorialIterative(int n) {
    int result = 1;
    for (int i = 2; i <= n; i++) {
        result *= i;
    }
    return result;
}

// 递归版本（斐波那契）
int fibRecursive(int n) {
    if (n <= 1) return n;
    return fibRecursive(n - 1) + fibRecursive(n - 2);
}

// 循环版本（O(n)时间）
int fibIterative(int n) {
    if (n <= 1) return n;
    int a = 0, b = 1;
    for (int i = 2; i <= n; i++) {
        int temp = a + b;
        a = b;
        b = temp;
    }
    return b;
}

int main() {
    cout << "阶乘 5: " << factorialRecursive(5) << " / "
         << factorialIterative(5) << endl;
    cout << "斐波那契 10: " << fibRecursive(10) << " / "
         << fibIterative(10) << endl;

    return 0;
}
```

输出：
```
阶乘 5: 120 / 120
斐波那契 10: 55 / 55
```

### 2.2 尾递归优化

```cpp
#include <iostream>
using namespace std;

// 普通递归：需要栈空间保存中间结果
int sumNormal(int n) {
    if (n <= 0) return 0;
    return n + sumNormal(n - 1);  // 递归调用后还要加n
}

// 尾递归：递归调用是最后一步操作
// 编译器可能优化为循环（但C++不保证）
int sumTail(int n, int acc = 0) {
    if (n <= 0) return acc;
    return sumTail(n - 1, acc + n);  // 递归调用是最后操作
}

// 循环版本（确定无栈溢出）
int sumLoop(int n) {
    int acc = 0;
    while (n > 0) {
        acc += n;
        n--;
    }
    return acc;
}

int main() {
    cout << "sum 100: " << sumTail(100) << " / " << sumLoop(100) << endl;

    // 大数测试（普通递归可能栈溢出）
    int large = 10000;
    cout << "sum " << large << ": " << sumLoop(large) << endl;

    return 0;
}
```

输出：
```
sum 100: 5050 / 5050
sum 10000: 50005000
```

## 三、注意事项与常见陷阱

1. **递归深度限制**：默认栈空间有限，深度约10000层可能栈溢出
2. **C++不保证尾递归优化**：依赖编译器实现
3. **循环更高效**：没有函数调用开销，不会栈溢出
4. **递归更直观**：树遍历、分治算法用递归更自然
5. **递归转循环用栈**：手动用`std::stack`模拟递归调用栈
