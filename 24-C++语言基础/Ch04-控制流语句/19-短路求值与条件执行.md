# 短路求值与条件执行

## 一、概念说明

`&&`和`||`的短路特性可以用来**控制表达式的执行**，替代简单的if语句。这是一种函数式编程风格，但要避免过度使用影响可读性。

## 二、具体用法

### 2.1 利用短路控制执行

```cpp
#include <iostream>
#include <string>
using namespace std;

int main() {
    // &&短路：左侧为true时才执行右侧
    int* ptr = new int(42);
    (ptr != nullptr) && (cout << "值: " << *ptr << endl);
    delete ptr;

    ptr = nullptr;
    (ptr != nullptr) && (cout << "这不会执行" << endl);

    // ||短路：左侧为false时才执行右侧
    bool processed = false;
    processed || (cout << "处理中..." << endl, processed = true);

    // 条件执行链
    int x = 10;
    (x > 0) && (cout << "正数" << endl);
    (x > 5) && (cout << "大于5" << endl);
    (x > 20) && (cout << "大于20" << endl);  // 不执行

    return 0;
}
```

输出：
```
值: 42
处理中...
正数
大于5
```

### 2.2 实际应用场景

```cpp
#include <iostream>
using namespace std;

int main() {
    // 场景：配置检查
    bool configLoaded = true;
    bool databaseReady = true;
    bool networkOK = false;

    // 使用短路的启动序列
    configLoaded
        && databaseReady
        && networkOK
        && (cout << "系统启动成功" << endl);

    if (!networkOK) {
        cout << "网络不可用，跳过启动" << endl;
    }

    // 但大多数情况下，if更清晰
    if (configLoaded && databaseReady && networkOK) {
        cout << "系统启动成功" << endl;
    }

    return 0;
}
```

输出：
```
网络不可用，跳过启动
网络不可用，跳过启动
```

## 三、注意事项与常见陷阱

1. **可读性优先**：过度使用短路控制执行会使代码难以理解
2. **副作用风险**：短路可能跳过重要的副作用操作
3. **逗号运算符配合**：`&&`右侧用逗号运算符执行多条语句
4. **不推荐日常使用**：这种技巧在代码审查中可能被要求改为if
5. **性能无差异**：编译器会优化if和短路表达式为相同代码
