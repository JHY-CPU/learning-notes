# 循环优化技巧

## 一、概念说明

循环是程序性能的关键热点。常见的优化技巧包括减少循环内计算、循环展开、减少分支和缓存友好访问等。

## 二、具体用法

### 2.1 减少循环内计算

```cpp
#include <iostream>
#include <vector>
#include <cmath>
using namespace std;

int main() {
    vector<double> data(1000000, 1.0);

    // 优化前：每次迭代都计算size()
    // for (size_t i = 0; i < data.size(); i++) { ... }

    // 优化后：缓存size
    size_t n = data.size();
    for (size_t i = 0; i < n; i++) {
        data[i] *= 2.0;
    }

    // 提取不变量
    double factor = sqrt(2.0) * 3.14;  // 不在循环内计算
    for (size_t i = 0; i < n; i++) {
        data[i] *= factor;
    }

    cout << "处理完成" << endl;

    return 0;
}
```

输出：
```
处理完成
```

### 2.2 循环展开

```cpp
#include <iostream>
using namespace std;

int main() {
    const int N = 1000;
    int arr[N];
    for (int i = 0; i < N; i++) arr[i] = i;

    // 未展开
    int sum1 = 0;
    for (int i = 0; i < N; i++) {
        sum1 += arr[i];
    }

    // 手动展开4倍（减少循环开销）
    int sum2 = 0;
    int i = 0;
    for (; i + 3 < N; i += 4) {
        sum2 += arr[i] + arr[i+1] + arr[i+2] + arr[i+3];
    }
    for (; i < N; i++) {
        sum2 += arr[i];
    }

    cout << "sum1 = " << sum1 << endl;
    cout << "sum2 = " << sum2 << endl;

    // 注意：现代编译器会自动展开简单的循环

    return 0;
}
```

输出：
```
sum1 = 499500
sum2 = 499500
```

### 2.3 缓存友好

```cpp
#include <iostream>
using namespace std;

const int SIZE = 1024;
int matrix[SIZE][SIZE];

int main() {
    // 初始化
    for (int i = 0; i < SIZE; i++)
        for (int j = 0; j < SIZE; j++)
            matrix[i][j] = i + j;

    // 缓存友好：按行访问（内层遍历列）
    long sum1 = 0;
    for (int i = 0; i < SIZE; i++)
        for (int j = 0; j < SIZE; j++)
            sum1 += matrix[i][j];

    // 缓存不友好：按列访问（内层遍历行）
    long sum2 = 0;
    for (int j = 0; j < SIZE; j++)
        for (int i = 0; i < SIZE; i++)
            sum2 += matrix[i][j];

    cout << "sum1 = " << sum1 << endl;
    cout << "sum2 = " << sum2 << endl;

    return 0;
}
```

输出：
```
sum1 = 1047552
sum2 = 1047552
```

## 三、注意事项与常见陷阱

1. **先测量再优化**：用性能分析工具找到真正的瓶颈
2. **编译器已经很聪明**：很多微优化编译器会自动做
3. **可读性优先**：不要为了微小性能提升牺牲可读性
4. **循环展开过度可能更慢**：展开太多导致指令缓存未命中
5. **数据布局影响更大**：缓存友好的数据布局比循环优化更重要
