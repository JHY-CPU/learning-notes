# 移动迭代器

## 一、概念说明

`std::make_move_iterator`（C++标准 §24.5.3.3，C++11）将普通迭代器转换为移动迭代器，使算法使用移动语义而非拷贝。适用于将元素从一个容器转移到另一个容器，特别是对不可拷贝但可移动的类型（如`unique_ptr`）。

### 1.1 移动 vs 拷贝

| 操作 | 源对象状态 | 性能 | 要求 |
|------|-----------|------|------|
| 拷贝 | 不变 | 慢 | 可拷贝 |
| 移动 | 有效但未指定 | 快 | 可移动 |

## 二、具体用法

### 2.1 基本用法

```cpp
#include <iterator>
#include <vector>
#include <string>
#include <algorithm>
#include <iostream>

int main() {
    std::vector<std::string> src = {"hello", "world", "foo", "bar"};
    std::vector<std::string> dst;

    // 移动而非拷贝
    dst.reserve(src.size());
    std::copy(std::make_move_iterator(src.begin()),
              std::make_move_iterator(src.end()),
              std::back_inserter(dst));

    // src中的字符串已被移动（可能为空）
    std::cout << "src[0]: \"" << src[0] << "\"" << std::endl;  // 空

    // dst包含移动后的字符串
    for (const auto& s : dst) std::cout << s << " ";
    // hello world foo bar
}
```

### 2.2 移动不可拷贝类型

```cpp
#include <memory>

void move_unique_ptr() {
    std::vector<std::unique_ptr<int>> src;
    src.push_back(std::make_unique<int>(1));
    src.push_back(std::make_unique<int>(2));

    // unique_ptr不能拷贝，只能移动
    std::vector<std::unique_ptr<int>> dst(
        std::make_move_iterator(src.begin()),
        std::make_move_iterator(src.end())
    );

    // src中的unique_ptr已移出
    std::cout << "dst[0]: " << *dst[0] << std::endl;  // 1
    std::cout << "src[0] is null: " << (src[0] == nullptr) << std::endl;  // true
}
```

### 2.3 移动构造容器

```cpp
void move_elements() {
    std::vector<std::string> src = {"a", "b", "c"};

    // 通过迭代器范围构造（移动）
    std::vector<std::string> dst(
        std::make_move_iterator(src.begin()),
        std::make_move_iterator(src.end())
    );

    // 等价于逐个移动构造
    // 对大字符串有显著性能提升
}
```

## 三、注意事项

1. **移动后源对象处于有效但未指定状态**：不应再依赖其值
2. **只对支持移动的类型有意义**：trivial类型移动等同于拷贝
3. **原始迭代器转换为移动迭代器后不能混用**
4. **配合`std::move`和`back_inserter`使用**
5. **移动迭代器解引用返回右值引用**：`*move_iterator`返回`T&&`
6. **对`unique_ptr`等不可拷贝类型必须使用移动迭代器**
