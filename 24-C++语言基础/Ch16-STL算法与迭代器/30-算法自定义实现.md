# 算法自定义实现

## 一、概念说明

虽然STL提供了丰富的算法，但有时需要自定义实现特定的算法逻辑。了解何时扩展STL、如何设计通用的自定义算法是高级编程技能。

## 二、具体用法

### 2.1 何时自定义

```cpp
/*
需要自定义算法的场景：
1. STL没有提供需要的操作（如分组、zip）
2. 特定数据结构需要优化实现
3. 组合多个STL算法效率低
4. 需要特殊的迭代模式
*/
```

### 2.2 自定义算法示例

```cpp
#include <algorithm>
#include <vector>
#include <iostream>

// 自定义：对每个相邻元素对执行操作
template <typename ForwardIt, typename BinaryOp>
void for_each_adjacent(ForwardIt first, ForwardIt last, BinaryOp op) {
    if (first == last) return;
    ForwardIt next = first;
    ++next;
    while (next != last) {
        op(*first, *next);
        first = next;
        ++next;
    }
}

// 自定义：分组
template <typename InputIt, typename KeyFunc>
auto group_by(InputIt first, InputIt last, KeyFunc key_func) {
    std::map<decltype(key_func(*first)), std::vector<typename InputIt::value_type>> groups;
    for (; first != last; ++first) {
        groups[key_func(*first)].push_back(*first);
    }
    return groups;
}

int main() {
    std::vector<int> v = {1, 2, 3, 4, 5};

    // 使用自定义算法
    for_each_adjacent(v.begin(), v.end(), [](int a, int b) {
        std::cout << "(" << a << "," << b << ") ";
    });
    // (1,2) (2,3) (3,4) (4,5)

    // 分组示例
    std::vector<int> nums = {1, 2, 3, 4, 5, 6, 7, 8};
    auto groups = group_by(nums.begin(), nums.end(),
                            [](int x) { return x % 2 == 0 ? "偶" : "奇"; });
}
```

### 2.3 扩展STL的正确方式

```cpp
// 不要修改std命名空间
// 应该在自己的命名空间中定义

namespace mylib {
    template <typename Iter, typename Pred>
    Iter find_last_if(Iter first, Iter last, Pred pred) {
        Iter result = last;
        for (; first != last; ++first) {
            if (pred(*first)) result = first;
        }
        return result;
    }
}

void use_custom() {
    std::vector<int> v = {1, 2, 3, 4, 5, 2, 1};
    auto it = mylib::find_last_if(v.begin(), v.end(),
                                    [](int x) { return x == 2; });
    if (it != v.end()) {
        std::cout << "最后的2在位置: " << std::distance(v.begin(), it) << std::endl;
    }
}
```

## 三、注意事项

- 优先使用STL算法，只在必要时自定义
- 自定义算法应遵循STL的接口风格（迭代器参数）
- 考虑异常安全和迭代器类别要求
- 文档化算法的前置条件和复杂度
- 可以组合STL算法实现更复杂的操作
- 自定义算法应放在自己的命名空间中
