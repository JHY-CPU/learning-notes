# 算法自定义实现

## 一、概念说明

虽然STL提供了丰富的算法，但有时需要自定义实现特定的算法逻辑（C++标准 §25-26）。了解何时扩展STL、如何设计通用的自定义算法是高级编程技能。自定义算法应遵循STL的接口风格（迭代器参数），放在自己的命名空间中。

### 1.1 何时自定义

```
需要自定义算法的场景：
1. STL没有提供需要的操作（如分组、zip、滑动窗口）
2. 特定数据结构需要优化实现
3. 组合多个STL算法效率低
4. 需要特殊的迭代模式
5. 单次遍历解决多个问题
```

## 二、具体用法

### 2.1 自定义相邻元素操作

```cpp
#include <algorithm>
#include <vector>
#include <iostream>

// 对每个相邻元素对执行操作
template <typename ForwardIt, typename BinaryOp>
void for_each_adjacent(ForwardIt first, ForwardIt last, BinaryOp op) {
    if (first == last) return;
    ForwardIt next = first;
    ++next;
    while (next != last) {
        op(*first, *next);
        first = next;
        ++next;
    }
}

int main() {
    std::vector<int> v = {1, 2, 3, 4, 5};

    for_each_adjacent(v.begin(), v.end(), [](int a, int b) {
        std::cout << "(" << a << "," << b << ") ";
    });
    // (1,2) (2,3) (3,4) (4,5)
}
```

### 2.2 自定义分组

```cpp
#include <map>

template <typename InputIt, typename KeyFunc>
auto group_by(InputIt first, InputIt last, KeyFunc key_func) {
    std::map<decltype(key_func(*first)),
             std::vector<typename std::iterator_traits<InputIt>::value_type>> groups;
    for (; first != last; ++first) {
        groups[key_func(*first)].push_back(*first);
    }
    return groups;
}

void group_demo() {
    std::vector<int> nums = {1, 2, 3, 4, 5, 6, 7, 8};
    auto groups = group_by(nums.begin(), nums.end(),
                            [](int x) { return x % 2 == 0 ? "偶" : "奇"; });
    // groups: {"奇": {1,3,5,7}, "偶": {2,4,6,8}}
}
```

### 2.3 自定义查找

```cpp
namespace mylib {
    // 查找最后一个满足条件的元素
    template <typename Iter, typename Pred>
    Iter find_last_if(Iter first, Iter last, Pred pred) {
        Iter result = last;
        for (; first != last; ++first) {
            if (pred(*first)) result = first;
        }
        return result;
    }
}

void use_custom() {
    std::vector<int> v = {1, 2, 3, 4, 5, 2, 1};
    auto it = mylib::find_last_if(v.begin(), v.end(),
                                    [](int x) { return x == 2; });
    if (it != v.end())
        std::cout << "最后的2在位置: " << std::distance(v.begin(), it)
                  << std::endl;  // 5
}
```

### 2.4 滑动窗口

```cpp
template <typename ForwardIt, typename BinaryOp>
void sliding_window(ForwardIt first, ForwardIt last, size_t window_size, BinaryOp op) {
    if (std::distance(first, last) < (int)window_size) return;

    auto window_end = first;
    std::advance(window_end, window_size);

    auto window_start = first;
    while (window_end != last) {
        op(window_start, window_end);
        ++window_start;
        ++window_end;
    }
}

void sliding_demo() {
    std::vector<int> v = {1, 2, 3, 4, 5};
    sliding_window(v.begin(), v.end(), 3,
        [](auto first, auto last) {
            int sum = std::accumulate(first, last, 0);
            std::cout << "窗口和: " << sum << std::endl;
        });
    // 6, 9, 12
}
```

## 三、扩展STL的最佳实践

```cpp
/*
1. 不要修改std命名空间
   - 在自己的命名空间中定义

2. 遵循STL的接口风格
   - 迭代器参数
   - 返回迭代器

3. 考虑异常安全和迭代器类别要求

4. 文档化算法的前置条件和复杂度

5. 可以组合STL算法实现更复杂的操作

6. 使用C++20 concepts约束模板参数
*/
```

## 四、注意事项

1. **优先使用STL算法**：只在必要时自定义
2. **自定义算法应遵循STL的接口风格**：迭代器参数
3. **考虑异常安全和迭代器类别要求**
4. **文档化算法的前置条件和复杂度**
5. **可以组合STL算法实现更复杂的操作**
6. **自定义算法应放在自己的命名空间中**
