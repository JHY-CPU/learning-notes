# 流迭代器

## 一、概念说明

流迭代器（C++标准 §24.6）将流（文件、控制台、字符串）与STL算法连接，允许用算法读写流数据。`istream_iterator`用于读取流，`ostream_iterator`用于写入流。这使得流操作可以无缝集成到STL算法管道中。

### 1.1 两种流迭代器

| 类型 | 方向 | 终止条件 |
|------|------|---------|
| `istream_iterator<T>` | 读取 | 默认构造的迭代器（流结束） |
| `ostream_iterator<T>` | 写入 | 无终止（输出到流） |

## 二、具体用法

### 2.1 ostream_iterator

```cpp
#include <iterator>
#include <vector>
#include <algorithm>
#include <iostream>

int main() {
    std::vector<int> v = {1, 2, 3, 4, 5};

    // 输出到cout，以空格分隔
    std::copy(v.begin(), v.end(), std::ostream_iterator<int>(std::cout, " "));
    // 1 2 3 4 5

    // 输出到字符串流
    std::ostringstream oss;
    std::copy(v.begin(), v.end(), std::ostream_iterator<int>(oss, ", "));
    std::cout << oss.str() << std::endl;  // 1, 2, 3, 4, 5,

    // 输出字符串
    std::vector<std::string> words = {"hello", "world"};
    std::copy(words.begin(), words.end(),
              std::ostream_iterator<std::string>(std::cout, "\n"));
}
```

### 2.2 istream_iterator

```cpp
#include <sstream>

void istream_demo() {
    // 从字符串流读取
    std::istringstream iss("10 20 30 40 50");
    std::vector<int> nums;

    std::copy(std::istream_iterator<int>(iss),
              std::istream_iterator<int>(),   // 流结束
              std::back_inserter(nums));
    // nums: {10, 20, 30, 40, 50}

    // 从cin读取
    // std::vector<int> input;
    // std::copy(std::istream_iterator<int>(std::cin),
    //           std::istream_iterator<int>(),
    //           std::back_inserter(input));
    // 按Ctrl+D(Linux)或Ctrl+Z(Windows)结束
}
```

### 2.3 字符串解析

```cpp
#include <string>

void parse_string() {
    std::string data = "hello world foo bar";
    std::istringstream iss(data);
    std::vector<std::string> words;

    // 按空格分割
    std::copy(std::istream_iterator<std::string>(iss),
              std::istream_iterator<std::string>(),
              std::back_inserter(words));
    // words: {"hello", "world", "foo", "bar"}
}
```

## 三、注意事项

1. **默认构造的`istream_iterator`表示流结束**：作为算法的end参数
2. **`ostream_iterator`的第二个参数是分隔符**：可选
3. **流迭代器是单遍的**：不能多次读取同一个位置
4. **适用于简单的流输入输出场景**
5. **读取失败时`istream_iterator`变为结束迭代器**：类型不匹配等
6. **性能不如直接操作流**：但代码更简洁
