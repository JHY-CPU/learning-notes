# 继承与组合对比

## 一、概念说明

**组合**（Composition）通过在类中包含另一个类的对象实现复用，表达"has-a"关系。**继承**表达"is-a"关系。设计原则是**优先使用组合**（Effective C++条款32）。两者都是代码复用机制，但语义和耦合度截然不同。

### 1.1 核心对比

| 考量 | 继承 | 组合 |
|------|------|------|
| 关系 | is-a | has-a |
| 耦合度 | 紧密耦合 | 松散耦合 |
| 复用方式 | 白盒复用（可见内部） | 黑盒复用（只用接口） |
| 灵活性 | 编译时确定 | 运行时可替换 |
| 访问protected | 可以 | 不可以 |
| 重写虚函数 | 可以 | 不可以 |
| 对象切片风险 | 有 | 无 |

## 二、具体用法

### 2.1 组合优于继承的场景

```cpp
#include <iostream>
#include <string>

// 不好的设计：Car不是Engine的一种！
class BadEngine {
public:
    int horsepower;
    void start() { std::cout << "引擎启动" << std::endl; }
};

class BadCar : public BadEngine {  // 违反is-a语义
    std::string brand;
};

// 好的设计：Car有一个Engine
class Engine {
    int horsepower;
    std::string type;
public:
    Engine(int hp, const std::string& t) : horsepower(hp), type(t) {}
    void start() { std::cout << type << "引擎启动: " << horsepower << "马力" << std::endl; }
    int getHP() const { return horsepower; }
};

class Car {
    Engine engine;       // 组合：Car有一个Engine
    std::string brand;
public:
    Car(const std::string& b, int hp, const std::string& engineType)
        : brand(b), engine(hp, engineType) {}

    void start() {
        std::cout << brand << " ";
        engine.start();  // 委托给engine
    }

    void drive() { std::cout << brand << "在行驶" << std::endl; }
};

int main() {
    Car car("宝马", 300, "V8");
    car.start();
    car.drive();
    return 0;
}
```

### 2.2 继承的正确使用场景

```cpp
#include <iostream>
#include <vector>
#include <memory>

// 接口继承：Shape定义了抽象契约
class Shape {
public:
    virtual double area() const = 0;
    virtual void draw() const = 0;
    virtual ~Shape() = default;
};

// is-a关系：Circle是一种Shape
class Circle : public Shape {
    double radius;
public:
    Circle(double r) : radius(r) {}
    double area() const override { return 3.14159 * radius * radius; }
    void draw() const override {
        std::cout << "绘制圆(r=" << radius << ")" << std::endl;
    }
};

// is-a关系：Rectangle是一种Shape
class Rectangle : public Shape {
    double w, h;
public:
    Rectangle(double w, double h) : w(w), h(h) {}
    double area() const override { return w * h; }
    void draw() const override {
        std::cout << "绘制矩形(" << w << "x" << h << ")" << std::endl;
    }
};

int main() {
    std::vector<std::unique_ptr<Shape>> shapes;
    shapes.push_back(std::make_unique<Circle>(5));
    shapes.push_back(std::make_unique<Rectangle>(3, 4));

    for (const auto& s : shapes) {
        s->draw();
        std::cout << "  面积: " << s->area() << std::endl;
    }
    return 0;
}
```

### 2.3 使用组合实现策略模式

```cpp
#include <iostream>
#include <memory>

// 策略接口
class SortStrategy {
public:
    virtual void sort(int* data, int size) = 0;
    virtual ~SortStrategy() = default;
};

class BubbleSort : public SortStrategy {
public:
    void sort(int* data, int size) override {
        std::cout << "冒泡排序(" << size << "个元素)" << std::endl;
    }
};

class QuickSort : public SortStrategy {
public:
    void sort(int* data, int size) override {
        std::cout << "快速排序(" << size << "个元素)" << std::endl;
    }
};

// 使用组合：运行时可替换策略
class Sorter {
    std::unique_ptr<SortStrategy> strategy;
public:
    Sorter(std::unique_ptr<SortStrategy> s) : strategy(std::move(s)) {}

    // 运行时更换策略
    void setStrategy(std::unique_ptr<SortStrategy> s) {
        strategy = std::move(s);
    }

    void doSort(int* data, int size) {
        strategy->sort(data, size);
    }
};

int main() {
    int data[] = {3, 1, 4, 1, 5};

    Sorter sorter(std::make_unique<BubbleSort>());
    sorter.doSort(data, 5);

    sorter.setStrategy(std::make_unique<QuickSort>());
    sorter.doSort(data, 5);

    return 0;
}
```

## 三、注意事项与常见陷阱

1. **问自己"派生类是基类的一种吗？"**：如果不是，用组合
2. **继承会暴露实现细节**：白盒复用，基类变化影响所有派生类
3. **组合更灵活**：可以在运行时替换内部组件
4. **过深的继承层次增加维护难度**：建议不超过3层
5. **接口继承（纯虚函数）是继承的好用途**：定义行为契约
6. **组合的代价是额外的代码**：需要编写委托函数转发调用
7. **私有继承可替代组合**：但组合更直观，优先选择组合
