# 继承与组合对比

## 一、概念说明

**组合**（Composition）通过在类中包含另一个类的对象实现复用，表达"has-a"关系。**继承**表达"is-a"关系。设计原则是**优先使用组合**。

### 1.1 选择依据

| 考量 | 继承 | 组合 |
|------|------|------|
| 关系 | is-a | has-a |
| 耦合度 | 紧密耦合 | 松散耦合 |
| 复用方式 | 白盒复用 | 黑盒复用 |
| 灵活性 | 运行时多态 | 编译时组合 |

## 二、具体用法

### 2.1 组合优于继承

```cpp
#include <iostream>
#include <string>

// 不好的设计：用继承
class BadEngine {
public:
    int horsepower;
};

class BadCar : public BadEngine {  // Car不是Engine!
    std::string brand;
};

// 好的设计：用组合
class Engine {
public:
    int horsepower;
    Engine(int hp) : horsepower(hp) {}
    void start() { std::cout << "引擎启动: " << horsepower << "马力" << std::endl; }
};

class Car {
    Engine engine;       // Car有一个Engine
    std::string brand;
public:
    Car(const std::string& b, int hp) : brand(b), engine(hp) {}
    void start() { engine.start(); }
    void drive() { std::cout << brand << "在行驶" << std::endl; }
};

int main() {
    Car car("宝马", 300);
    car.start();
    car.drive();
    return 0;
}
```

**输出：**
```
引擎启动: 300马力
宝马在行驶
```

## 三、注意事项与常见陷阱

- 问自己"派生类**是**基类的一种吗？"如果不是，用组合
- 继承会暴露实现细节（白盒复用）
- 组合更灵活，可以在运行时替换内部组件
- 过深的继承层次增加维护难度
- 接口继承（纯虚函数）是继承的好用途
