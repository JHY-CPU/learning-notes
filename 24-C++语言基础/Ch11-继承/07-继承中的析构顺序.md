# 继承中的析构顺序

## 一、概念说明

析构顺序是构造顺序的完全逆序（C++标准 §12.4）：**派生类析构函数体 → 非静态成员（声明逆序） → 非虚基类（声明逆序） → 虚基类**。这保证了对象在析构过程中，所依赖的基类和成员仍然有效。

### 1.1 析构与构造的对应

| 阶段 | 构造顺序 | 析构顺序 |
|------|---------|---------|
| 1 | 虚基类 | 派生类析构体 |
| 2 | 非虚基类 | 成员（声明逆序） |
| 3 | 成员（声明顺序） | 非虚基类（声明逆序） |
| 4 | 派生类构造体 | 虚基类 |

## 二、具体用法

### 2.1 析构顺序验证

```cpp
#include <iostream>

class Base {
public:
    virtual ~Base() { std::cout << "基类析构" << std::endl; }
};

class Member {
    std::string name;
public:
    Member(const std::string& n) : name(n) {}
    ~Member() { std::cout << "成员 " << name << " 析构" << std::endl; }
};

class Derived : public Base {
    Member m1, m2;  // 声明顺序：m1先，m2后
public:
    Derived() : m1("A"), m2("B") {}
    ~Derived() override { std::cout << "派生类析构" << std::endl; }
};

int main() {
    std::cout << "=== 创建对象 ===" << std::endl;
    {
        Derived d;
        std::cout << "=== 即将析构 ===" << std::endl;
    }
    std::cout << "=== 析构完成 ===" << std::endl;
    return 0;
}
```

**输出：**
```
=== 创建对象 ===
=== 即将析构 ===
派生类析构
成员 B 析构    ← 后构造的先析构
成员 A 析构    ← 先构造的后析构
基类析构
=== 析构完成 ===
```

### 2.2 虚析构函数的重要性

```cpp
#include <iostream>
#include <memory>

class Base {
    int* data;
public:
    Base() : data(new int[100]) {
        std::cout << "Base分配资源" << std::endl;
    }
    virtual ~Base() {  // 虚析构：确保多态删除正确
        delete[] data;
        std::cout << "Base释放资源" << std::endl;
    }
};

class Derived : public Base {
    int* extra;
public:
    Derived() : extra(new int[200]) {
        std::cout << "Derived分配资源" << std::endl;
    }
    ~Derived() override {
        delete[] extra;
        std::cout << "Derived释放资源" << std::endl;
    }
};

int main() {
    // 多态删除：通过基类指针删除派生类对象
    Base* p = new Derived();
    delete p;
    // 如果Base析构不是virtual，只会调用Base::~Base()
    // Derived的extra会泄漏！

    // 使用智能指针（推荐）
    std::unique_ptr<Base> ptr = std::make_unique<Derived>();
    // 自动调用正确的析构函数

    return 0;
}
```

### 2.3 析构中虚函数的绑定

```cpp
#include <iostream>

class Base {
public:
    virtual void cleanup() { std::cout << "Base::cleanup()" << std::endl; }
    virtual ~Base() {
        std::cout << "~Base()开始" << std::endl;
        cleanup();  // 注意：此时绑定到Base::cleanup()，非多态
        std::cout << "~Base()结束" << std::endl;
    }
};

class Derived : public Base {
    int* resource;
public:
    Derived() : resource(new int(42)) {}
    void cleanup() override {
        std::cout << "Derived::cleanup() - 释放resource" << std::endl;
        delete resource;
        resource = nullptr;
    }
    ~Derived() override {
        std::cout << "~Derived()开始" << std::endl;
        cleanup();  // 调用Derived::cleanup()
        std::cout << "~Derived()结束" << std::endl;
    }
};

int main() {
    Base* p = new Derived();
    delete p;
    // 输出：
    // ~Derived()开始
    // Derived::cleanup() - 释放resource
    // ~Derived()结束
    // ~Base()开始
    // Base::cleanup()    ← 非多态！Derived部分已析构
    // ~Base()结束
    return 0;
}
```

## 三、注意事项与常见陷阱

1. **析构顺序与构造顺序完全相反**：后构造的先析构
2. **基类析构应声明为`virtual`**：否则通过基类指针删除派生类对象是未定义行为（§10.4）
3. **析构中虚函数调用绑定到当前类版本**：派生类析构后，其虚函数表指针已回退到基类
4. **异常从析构函数中抛出会导致`std::terminate`**：析构函数应标记`noexcept`
5. **`delete this`在析构中是未定义行为**：析构中对象部分已销毁
6. **智能指针自动管理多态对象的析构**：`std::unique_ptr<Base>`保证调用正确的析构函数
