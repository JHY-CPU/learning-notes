# 继承中的析构顺序

## 一、概念说明

析构顺序是构造顺序的完全逆序：**派生类析构函数体 → 成员（声明逆序） → 基类**。这保证了对象在析构过程中，所依赖的基类仍然有效。

## 二、具体用法

### 2.1 析构顺序验证

```cpp
#include <iostream>

class Base {
public:
    virtual ~Base() { std::cout << "基类析构" << std::endl; }
};

class Member {
    std::string name;
public:
    Member(const std::string& n) : name(n) {}
    ~Member() { std::cout << "成员 " << name << " 析构" << std::endl; }
};

class Derived : public Base {
    Member m1, m2;
public:
    Derived() : m1("A"), m2("B") {}
    ~Derived() override { std::cout << "派生类析构" << std::endl; }
};

int main() {
    Derived d;
    std::cout << "--- 即将析构 ---" << std::endl;
    return 0;
}
```

**输出：**
```
--- 即将析构 ---
派生类析构
成员 B 析构
成员 A 析构
基类析构
```

## 三、注意事项与常见陷阱

- 析构顺序与构造顺序完全相反
- 基类析构应声明为`virtual`以保证多态删除正确
- 通过基类指针删除派生类对象时，虚析构保证派生类部分被清理
- 析构中虚函数调用绑定到当前类版本（非多态）
