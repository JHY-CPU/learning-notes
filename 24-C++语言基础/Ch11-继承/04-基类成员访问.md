# 基类成员访问

## 一、概念说明

派生类访问基类成员有多种方式（C++标准 §10.2）：直接访问（`protected`/`public`成员）、通过基类作用域`Base::`、以及使用`using`声明将基类成员引入派生类作用域。理解作用域遮蔽（name hiding）与虚函数覆盖（overriding）的区别至关重要。

### 1.1 名称查找规则

C++中的名称查找遵循**由内向外**原则：先在当前类作用域查找，找到即停止；如果没找到，再向上到基类作用域查找。这意味着派生类的同名成员会**遮蔽**基类的所有同名成员（包括重载版本）。

## 二、具体用法

### 2.1 using声明继承基类重载

```cpp
#include <iostream>

class Base {
public:
    void foo(int x) { std::cout << "Base::foo(int " << x << ")" << std::endl; }
    void foo(double x) { std::cout << "Base::foo(double " << x << ")" << std::endl; }
    void foo(const std::string& s) { std::cout << "Base::foo(string " << s << ")" << std::endl; }
};

class Derived : public Base {
public:
    // 使用using将Base的所有foo重载引入Derived作用域
    using Base::foo;
    void foo(const char* s) { std::cout << "Derived::foo(" << s << ")" << std::endl; }
};

int main() {
    Derived d;
    d.foo(42);        // Base::foo(int) - using使之可见
    d.foo(3.14);      // Base::foo(double) - using使之可见
    d.foo("hello");   // Derived::foo(const char*) - 派生类版本

    // 没有using的话，d.foo(42)会编译错误（被Derived::foo(const char*)遮蔽）
    return 0;
}
```

### 2.2 作用域遮蔽

```cpp
#include <iostream>

class Base {
public:
    int value = 10;
    void show() { std::cout << "Base::show()" << std::endl; }
    void show(int x) { std::cout << "Base::show(int " << x << ")" << std::endl; }
};

class Derived : public Base {
public:
    int value = 20;  // 遮蔽Base::value（名称相同即可）
    void show() { std::cout << "Derived::show()" << std::endl; }
    // show(int)也被遮蔽！即使签名不同

    void test() {
        // 名称查找先在Derived中找到value，停止
        std::cout << "Derived::value=" << value << std::endl;     // 20
        std::cout << "Base::value=" << Base::value << std::endl;  // 10

        show();       // Derived::show()
        Base::show(); // Base::show()
        // show(42);  // 错误！被Derived::show()遮蔽
        Base::show(42);  // 需要显式指定
    }
};

int main() {
    Derived d;
    d.test();

    // 通过基类引用访问
    Base& ref = d;
    std::cout << "ref.value=" << ref.value << std::endl;  // 10（静态类型）
    ref.show();  // 如果show是virtual则调用Derived::show()

    return 0;
}
```

### 2.3 using声明的应用场景

```cpp
#include <iostream>
#include <vector>

// 场景1：在私有继承中暴露特定接口
class Stack : private std::vector<int> {
public:
    using std::vector<int>::push_back;
    using std::vector<int>::pop_back;
    using std::vector<int>::back;
    using std::vector<int>::empty;
    using std::vector<int>::size;
    // 不暴露insert、erase等不适用于栈的操作
};

// 场景2：统一继承体系中的接口
class Logger {
public:
    void log(const std::string& msg) { std::cout << "[LOG] " << msg << std::endl; }
    void log(int level, const std::string& msg) {
        std::cout << "[LOG" << level << "] " << msg << std::endl;
    }
};

class FileLogger : public Logger {
public:
    using Logger::log;  // 引入所有log重载
    void log(const std::string& msg, const std::string& file) {
        std::cout << "[FILE:" << file << "] " << msg << std::endl;
    }
};

int main() {
    Stack s;
    s.push_back(1);
    s.push_back(2);
    std::cout << "top: " << s.back() << std::endl;  // 2

    FileLogger fl;
    fl.log("系统启动");              // Logger::log(string)
    fl.log(1, "警告信息");           // Logger::log(int, string)
    fl.log("写入完成", "app.log");   // FileLogger自己的版本

    return 0;
}
```

## 三、注意事项与常见陷阱

1. **派生类同名成员会遮蔽基类所有同名成员**：即使参数列表不同，这是名称遮蔽而非覆盖
2. **使用`Base::member`显式访问被遮蔽的基类成员**：避免二义性
3. **`using Base::func;`可将基类的重载函数全部引入**：解决遮蔽问题的标准方法
4. **作用域遮蔽是静态绑定**：与虚函数的动态多态覆盖不同
5. **成员变量也会被遮蔽**：`int x`在派生类中定义会遮蔽基类的`int x`
6. **名称查找在重载决议之前**：如果名称被遮蔽，根本不会进入重载决议阶段
