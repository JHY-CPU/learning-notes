# 虚继承的构造顺序

## 一、概念说明

虚继承中，**虚基类由最远派生类**（most derived class）负责构造。无论继承层次多深，虚基类只构造一次，且在所有非虚基类之前构造。

## 二、具体用法

### 2.1 虚基类构造顺序

```cpp
#include <iostream>

class A {
public:
    A(int v) { std::cout << "A(" << v << ")" << std::endl; }
};

class B : virtual public A {
public:
    B() : A(1) { std::cout << "B()" << std::endl; }
};

class C : virtual public A {
public:
    C() : A(2) { std::cout << "C()" << std::endl; }
};

class D : public B, public C {
public:
    // D是最远派生类，负责构造虚基类A
    // B()和C()中的A(1)、A(2)都被忽略
    D() : A(99), B(), C() {
        std::cout << "D()" << std::endl;
    }
};

int main() {
    D d;
    return 0;
}
```

**输出：**
```
A(99)
B()
C()
D()
```

## 三、注意事项与常见陷阱

- 最远派生类的构造参数决定虚基类的初始化
- 中间类对虚基类的初始化被忽略
- 构造顺序：虚基类 → 非虚基类（声明顺序） → 成员 → 自身
- 单独创建中间类（如`B b;`）时，由该类负责构造虚基类
