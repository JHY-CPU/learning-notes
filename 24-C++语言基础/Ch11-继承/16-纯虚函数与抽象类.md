# 纯虚函数与抽象类

## 一、概念说明

**纯虚函数**（C++标准 §10.4）用`= 0`声明，表示该函数没有默认实现（但C++允许纯虚函数有函数体）。包含纯虚函数的类称为**抽象类**（abstract class），不能直接实例化，只能作为基类被继承。派生类必须实现所有纯虚函数才能实例化。

### 1.1 纯虚函数 vs 普通虚函数

| 特性 | 普通虚函数 | 纯虚函数 |
|------|----------|---------|
| 声明 | `virtual void f();` | `virtual void f() = 0;` |
| 默认实现 | 必须有 | 可以有（但不自动继承） |
| 类实例化 | 可以 | 不可以（抽象类） |
| 派生类实现 | 可选 | 必须实现 |

## 二、具体用法

### 2.1 定义抽象类

```cpp
#include <iostream>
#include <cmath>
#include <vector>
#include <memory>

class Shape {
protected:
    std::string name;
public:
    Shape(const std::string& n) : name(n) {}

    // 纯虚函数
    virtual double area() const = 0;
    virtual double perimeter() const = 0;

    // 非纯虚函数（有默认实现）
    virtual void describe() const {
        std::cout << name << ": 面积=" << area()
                  << ", 周长=" << perimeter() << std::endl;
    }

    virtual ~Shape() = default;
};

class Circle : public Shape {
    double radius;
public:
    Circle(double r) : Shape("圆"), radius(r) {}
    double area() const override { return M_PI * radius * radius; }
    double perimeter() const override { return 2 * M_PI * radius; }
};

class Rectangle : public Shape {
    double w, h;
public:
    Rectangle(double w, double h) : Shape("矩形"), w(w), h(h) {}
    double area() const override { return w * h; }
    double perimeter() const override { return 2 * (w + h); }
};

class Triangle : public Shape {
    double a, b, c;  // 三边
public:
    Triangle(double a, double b, double c) : Shape("三角形"), a(a), b(b), c(c) {}
    double area() const override {
        double s = (a + b + c) / 2;
        return std::sqrt(s * (s-a) * (s-b) * (s-c));
    }
    double perimeter() const override { return a + b + c; }
};

int main() {
    // Shape s("test");  // 编译错误！抽象类不能实例化

    std::vector<std::unique_ptr<Shape>> shapes;
    shapes.push_back(std::make_unique<Circle>(5));
    shapes.push_back(std::make_unique<Rectangle>(3, 4));
    shapes.push_back(std::make_unique<Triangle>(3, 4, 5));

    for (const auto& s : shapes)
        s->describe();

    return 0;
}
```

### 2.2 纯虚函数可以有函数体

```cpp
#include <iostream>

class Abstract {
public:
    // 纯虚函数声明
    virtual void pure() = 0;

    // 纯虚函数可以有实现！
    // 用于提供公共逻辑，但不自动继承
    virtual void commonLogic() = 0;
};

// 纯虚函数的定义（必须在类外）
void Abstract::commonLogic() {
    std::cout << "Abstract::commonLogic() - 公共逻辑" << std::endl;
}

class Concrete : public Abstract {
public:
    void pure() override {
        std::cout << "Concrete::pure()" << std::endl;
    }
    void commonLogic() override {
        Abstract::commonLogic();  // 显式调用基类实现
        std::cout << "Concrete的附加逻辑" << std::endl;
    }
};

int main() {
    Concrete c;
    c.pure();
    c.commonLogic();
    return 0;
}
```

### 2.3 纯虚析构函数

```cpp
#include <iostream>

class Interface {
public:
    // 纯虚析构：类变为抽象类，但必须提供实现
    virtual ~Interface() = 0;
};

// 纯虚析构必须有定义（因为派生类析构会调用它）
Interface::~Interface() {
    std::cout << "Interface::~Interface()" << std::endl;
}

class Impl : public Interface {
    int* data;
public:
    Impl() : data(new int(42)) {}
    ~Impl() override {
        delete data;
        std::cout << "Impl::~Impl()" << std::endl;
    }
};

int main() {
    // Interface i;  // 编译错误！抽象类
    Interface* p = new Impl();
    delete p;  // 正确：Impl::~Impl() → Interface::~Interface()
    return 0;
}
```

## 三、注意事项与常见陷阱

1. **抽象类可以有数据成员和非纯虚函数**：并非只有纯虚函数
2. **纯虚函数可以有函数体**：`= 0`只表示"必须重写"，不禁止提供实现
3. **派生类必须实现所有纯虚函数才能实例化**：否则仍是抽象类
4. **纯虚析构必须提供实现**：`= 0`后加函数体，因为析构链需要调用
5. **抽象类不能作为参数或返回类型**：但可以使用引用/指针
6. **从抽象类派生的中间类可以继续不实现纯虚函数**：将责任传递给下一层
