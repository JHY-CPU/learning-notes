# 虚继承

## 一、概念说明

**虚继承**（Virtual Inheritance，C++标准 §10.2）通过`virtual`关键字解决菱形继承（Diamond Problem），确保共享基类在继承层次中只有一个实例。被虚继承的类称为**虚基类**。

### 1.1 菱形问题示意

```
      Animal          ← 虚基类
     /      \
  Mammal  WingedAnimal   ← 虚继承Animal
     \      /
       Bat              ← 最远派生类，负责构造Animal
```

没有虚继承：Bat包含两份Animal。使用虚继承：Bat只包含一份Animal。

## 二、具体用法

### 2.1 解决菱形问题

```cpp
#include <iostream>

class Animal {
public:
    int legs;
    Animal(int l = 0) : legs(l) {
        std::cout << "Animal(" << legs << ")" << std::endl;
    }
    void breathe() { std::cout << "呼吸中..." << std::endl; }
};

// 关键：virtual继承
class Mammal : virtual public Animal {
public:
    Mammal() { std::cout << "Mammal构造" << std::endl; }
    void nurse() { std::cout << "哺乳中..." << std::endl; }
};

class WingedAnimal : virtual public Animal {
public:
    WingedAnimal() { std::cout << "WingedAnimal构造" << std::endl; }
    void fly() { std::cout << "飞行中..." << std::endl; }
};

class Bat : public Mammal, public WingedAnimal {
public:
    // 最远派生类负责构造虚基类
    Bat() : Animal(2) {
        std::cout << "Bat构造" << std::endl;
    }
};

int main() {
    Bat bat;
    bat.legs = 2;         // OK！只有一份Animal，无二义性
    bat.breathe();         // OK
    bat.nurse();           // OK
    bat.fly();             // OK

    std::cout << "bat.legs=" << bat.legs << std::endl;
    std::cout << "sizeof(Bat)=" << sizeof(Bat) << std::endl;  // 比普通继承大

    return 0;
}
```

**输出：**
```
Animal(2)
Mammal构造
WingedAnimal构造
Bat构造
bat.legs=2
sizeof(Bat)=...  (包含虚基类指针，比预期大)
```

### 2.2 多层虚继承

```cpp
#include <iostream>

class Root {
public:
    Root() { std::cout << "Root" << std::endl; }
};

class A : virtual public Root {
public:
    A() { std::cout << "A" << std::endl; }
};

class B : virtual public Root {
public:
    B() { std::cout << "B" << std::endl; }
};

class C : public A, public B {
public:
    C() : Root() {  // C是最远派生类，构造Root
        std::cout << "C" << std::endl;
    }
};

int main() {
    C c;
    // Root只构造一次
    // 输出：Root → A → B → C
    return 0;
}
```

### 2.3 虚继承中的接口设计

```cpp
#include <iostream>
#include <string>

// 使用虚继承实现接口共享
class Logger {
public:
    virtual void log(const std::string& msg) {
        std::cout << "[LOG] " << msg << std::endl;
    }
    virtual ~Logger() = default;
};

class NetworkService : virtual public Logger {
public:
    void send(const std::string& data) {
        log("发送数据: " + data);
    }
};

class DatabaseService : virtual public Logger {
public:
    void query(const std::string& sql) {
        log("执行查询: " + sql);
    }
};

class App : public NetworkService, public DatabaseService {
public:
    void run() {
        send("Hello");
        query("SELECT * FROM users");
        log("应用运行中");  // 只有一份Logger
    }
};

int main() {
    App app;
    app.run();
    return 0;
}
```

## 三、注意事项与常见陷阱

1. **虚基类由最远派生类负责构造**：中间类的构造参数被忽略
2. **虚继承有性能开销**：通过间接指针（vbtable）访问虚基类成员
3. **不是所有中间类都需要虚继承**：只需直接继承虚基类的那层加`virtual`
4. **虚继承的内存布局更复杂**：使用vtable/vptr间接访问
5. **除非解决菱形问题，否则不需要虚继承**：不要滥用
6. **单独创建中间类时由该类负责虚基类**：如`Mammal m;`时Mammal构造Animal
