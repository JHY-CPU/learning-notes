# 虚继承的实现

## 一、概念说明

虚继承通过**虚基类表**（vbtable）和间接指针实现。每个虚继承的类对象包含一个指向虚基类的偏移量表，运行时通过该表定位虚基类子对象。

### 1.1 内存布局

- 普通继承：基类子对象直接嵌入派生类
- 虚继承：通过指针/vbtable间接访问虚基类

## 二、具体用法

### 2.1 查看虚继承的内存布局

```cpp
#include <iostream>

class A {
public:
    int a_val = 1;
};

class B : virtual public A {
public:
    int b_val = 2;
};

class C : virtual public A {
public:
    int c_val = 3;
};

class D : public B, public C {
public:
    int d_val = 4;
};

int main() {
    D d;
    d.a_val = 100;  // 只有一份A

    std::cout << "sizeof(A) = " << sizeof(A) << std::endl;
    std::cout << "sizeof(B) = " << sizeof(B) << std::endl;  // 包含vptr
    std::cout << "sizeof(D) = " << sizeof(D) << std::endl;

    std::cout << "d.a_val = " << d.a_val << std::endl;
    std::cout << "d.B::a_val = " << d.B::a_val << std::endl;
    std::cout << "d.C::a_val = " << d.C::a_val << std::endl;
    return 0;
}
```

**输出（示例，具体值因编译器而异）：**
```
sizeof(A) = 4
sizeof(B) = 16
sizeof(D) = 32
d.a_val = 100
d.B::a_val = 100
d.C::a_val = 100
```

## 三、注意事项与常见陷阱

- 虚继承通过间接访问实现，有性能开销
- 虚基类子对象位于对象的末尾
- `sizeof`会因虚继承增大（额外指针/vbtable）
- 不同编译器的虚继承实现细节不同
- 尽量避免在性能关键路径上使用虚继承
