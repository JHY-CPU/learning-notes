# 接口类

## 一、概念说明

**接口类**（Interface Class，C++标准 §10.4）是只包含纯虚函数（和虚析构）的抽象类，没有任何数据成员和非纯虚函数实现。它定义了一组行为契约，类似Java/C#中的`interface`关键字。C++没有专门的`interface`语法，通过约定实现。

### 1.1 接口类的特征

| 特征 | 要求 |
|------|------|
| 纯虚函数 | 所有函数都是`virtual ... = 0` |
| 数据成员 | 无（约定） |
| 析构函数 | `virtual ~Interface() = default;` |
| 构造函数 | 通常只有默认构造 |
| 多继承 | 接口多继承无菱形问题 |

## 二、具体用法

### 2.1 定义和实现接口

```cpp
#include <iostream>
#include <vector>
#include <memory>

// 接口类：定义绘制行为
class Drawable {
public:
    virtual void draw() const = 0;
    virtual ~Drawable() = default;
};

// 接口类：定义可点击行为
class Clickable {
public:
    virtual void onClick() = 0;
    virtual ~Clickable() = default;
};

// 接口类：定义可调整大小行为
class Resizable {
public:
    virtual void resize(double factor) = 0;
    virtual double getSize() const = 0;
    virtual ~Resizable() = default;
};

// 实现多个接口
class Button : public Drawable, public Clickable {
    std::string label;
public:
    Button(const std::string& l) : label(l) {}
    void draw() const override {
        std::cout << "绘制按钮: [" << label << "]" << std::endl;
    }
    void onClick() override {
        std::cout << "按钮 [" << label << "] 被点击" << std::endl;
    }
};

class Image : public Drawable, public Resizable {
    std::string path;
    double scale = 1.0;
public:
    Image(const std::string& p) : path(p) {}
    void draw() const override {
        std::cout << "绘制图片: " << path << " (缩放=" << scale << ")" << std::endl;
    }
    void resize(double factor) override { scale *= factor; }
    double getSize() const override { return scale; }
};

int main() {
    std::vector<std::unique_ptr<Drawable>> drawables;
    drawables.push_back(std::make_unique<Button>("确定"));
    drawables.push_back(std::make_unique<Image>("logo.png"));

    for (auto& d : drawables) d->draw();

    // 按接口处理
    Image img("photo.jpg");
    img.resize(2.0);
    img.draw();

    return 0;
}
```

### 2.2 接口与依赖注入

```cpp
#include <iostream>
#include <memory>

// 数据库接口
class IDatabase {
public:
    virtual bool connect(const std::string& connStr) = 0;
    virtual bool execute(const std::string& sql) = 0;
    virtual ~IDatabase() = default;
};

// 日志接口
class ILogger {
public:
    virtual void info(const std::string& msg) = 0;
    virtual void error(const std::string& msg) = 0;
    virtual ~ILogger() = default;
};

// 具体实现
class MySQLDatabase : public IDatabase {
public:
    bool connect(const std::string& connStr) override {
        std::cout << "MySQL连接: " << connStr << std::endl;
        return true;
    }
    bool execute(const std::string& sql) override {
        std::cout << "MySQL执行: " << sql << std::endl;
        return true;
    }
};

class ConsoleLogger : public ILogger {
public:
    void info(const std::string& msg) override {
        std::cout << "[INFO] " << msg << std::endl;
    }
    void error(const std::string& msg) override {
        std::cerr << "[ERROR] " << msg << std::endl;
    }
};

// 业务逻辑只依赖接口
class UserService {
    IDatabase& db;
    ILogger& logger;
public:
    UserService(IDatabase& d, ILogger& l) : db(d), logger(l) {}

    void createUser(const std::string& name) {
        logger.info("创建用户: " + name);
        if (!db.execute("INSERT INTO users VALUES('" + name + "')")) {
            logger.error("创建用户失败");
        }
    }
};

int main() {
    MySQLDatabase db;
    ConsoleLogger logger;
    UserService service(db, logger);  // 依赖注入

    service.createUser("张三");
    return 0;
}
```

### 2.3 接口版本控制

```cpp
#include <iostream>

class IService {
public:
    virtual void process() = 0;
    virtual ~IService() = default;
};

// 扩展接口：添加新功能
class IServiceV2 : public IService {
public:
    virtual void processAsync() = 0;
    virtual int version() const { return 2; }
};

class MyService : public IServiceV2 {
public:
    void process() override { std::cout << "同步处理" << std::endl; }
    void processAsync() override { std::cout << "异步处理" << std::endl; }
};

int main() {
    MyService svc;
    IService* v1 = &svc;
    v1->process();

    IServiceV2* v2 = &svc;
    v2->processAsync();
    std::cout << "版本: " << v2->version() << std::endl;

    return 0;
}
```

## 三、注意事项与常见陷阱

1. **接口类不应有数据成员**：Java/C#强制，C++靠约定
2. **虚析构函数必须有实现**：`= default`即可
3. **多接口继承不会产生菱形问题**：没有数据成员，无二义性
4. **接口是解耦和实现依赖注入的基础**：使代码可测试、可替换
5. **接口命名常以`I`前缀**：如`IDatabase`、`ILogger`（约定非强制）
6. **接口升级通过继承扩展**：`IServiceV2 : public IService`
