# 继承与友元

## 一、概念说明

**友元关系不可继承**（C++标准 §11.3）：如果类A是类B的友元，A的派生类不会自动成为B的友元。友元关系也不可传递：A是B的友元，B是C的友元，A不是C的友元。友元关系是单向的：A是B的友元不代表B是A的友元。

### 1.1 友元关系特性

| 特性 | 是否支持 |
|------|---------|
| 继承 | 否（不自动传递给派生类） |
| 传递 | 否（A友元B，B友元C，A不友元C） |
| 对称 | 否（A友元B，B不友元A） |

## 二、具体用法

### 2.1 友元不继承

```cpp
#include <iostream>

class Secret {
    int value;
    friend class Friend;  // 只有Friend是友元
public:
    Secret(int v) : value(v) {}
};

class Friend {
public:
    void show(const Secret& s) {
        std::cout << "Friend访问: " << s.value << std::endl;
    }
};

class ChildOfFriend : public Friend {
public:
    void show(const Secret& s) {
        // std::cout << s.value;  // 编译错误！ChildOfFriend不是友元
        std::cout << "不能直接访问私有成员" << std::endl;
    }
};

// 解决方案：在Secret中显式声明ChildOfFriend为友元
class SecretV2 {
    int value = 42;
    friend class FriendV2;
    friend class ChildOfFriendV2;  // 显式声明
public:
    SecretV2(int v) : value(v) {}
};

class FriendV2 {
public:
    virtual void show(const SecretV2& s) {
        std::cout << "FriendV2访问: " << s.value << std::endl;
    }
};

class ChildOfFriendV2 : public FriendV2 {
public:
    void show(const SecretV2& s) override {
        std::cout << "ChildOfFriendV2访问: " << s.value << std::endl;  // OK
    }
};

int main() {
    Secret s(42);
    Friend f;
    f.show(s);              // OK

    ChildOfFriend cf;
    cf.show(s);             // 不能访问

    SecretV2 s2(99);
    ChildOfFriendV2 cf2;
    cf2.show(s2);           // OK

    return 0;
}
```

### 2.2 友元函数模板与继承

```cpp
#include <iostream>

class Data {
    int value;
public:
    Data(int v) : value(v) {}

    // 友元函数模板
    template<typename T>
    friend void inspect(const T& obj);

    friend std::ostream& operator<<(std::ostream& os, const Data& d);
};

template<typename T>
void inspect(const T& obj) {
    std::cout << "inspect: value=" << obj.value << std::endl;
}

std::ostream& operator<<(std::ostream& os, const Data& d) {
    return os << "Data(" << d.value << ")";
}

class ExtendedData : public Data {
    int extra;
public:
    ExtendedData(int v, int e) : Data(v), extra(e) {}
    // ExtendedData不自动继承Data的友元关系
    // 但inspect<Data>可以访问Data的私有成员（不能访问ExtendedData::extra）
};

int main() {
    Data d(42);
    inspect(d);        // OK
    std::cout << d << std::endl;

    ExtendedData ed(10, 20);
    // inspect(ed) 调用inspect<ExtendedData>，但ExtendedData没有声明友元
    // 如果inspect被特化或重载可能出问题

    return 0;
}
```

### 2.3 友元与设计模式

```cpp
#include <iostream>

// Builder模式中使用友元
class Config {
    std::string host;
    int port;
    bool ssl;

    // 只允许ConfigBuilder访问私有构造
    friend class ConfigBuilder;
    Config(const std::string& h, int p, bool s) : host(h), port(p), ssl(s) {}

public:
    void show() const {
        std::cout << host << ":" << port << (ssl ? " (SSL)" : "") << std::endl;
    }
};

class ConfigBuilder {
    std::string host = "localhost";
    int port = 80;
    bool ssl = false;
public:
    ConfigBuilder& setHost(const std::string& h) { host = h; return *this; }
    ConfigBuilder& setPort(int p) { port = p; return *this; }
    ConfigBuilder& setSSL(bool s) { ssl = s; return *this; }
    Config build() { return Config(host, port, ssl); }
};

// ConfigBuilder的派生类不是Config的友元
class ExtendedBuilder : public ConfigBuilder {
public:
    // 无法直接调用Config的私有构造函数
};

int main() {
    auto cfg = ConfigBuilder()
        .setHost("api.example.com")
        .setPort(443)
        .setSSL(true)
        .build();
    cfg.show();

    return 0;
}
```

## 三、注意事项与常见陷阱

1. **友元关系不能被继承**：需要在目标类中显式声明每个友元
2. **友元关系不能传递**：避免形成隐式的访问依赖链
3. **可以将友元函数声明为模板函数**：但模板友元需要前置声明
4. **过度使用友元破坏封装性**：优先使用public接口
5. **友元关系是授予方控制的**：类自己决定谁是友元，而非友元自行声明
