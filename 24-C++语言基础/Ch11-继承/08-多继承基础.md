# 多继承基础

## 一、概念说明

**多继承**允许一个类同时从多个基类继承。C++支持多继承，但需要谨慎使用以避免**菱形继承问题**（Diamond Problem）。

## 二、具体用法

### 2.1 基本多继承

```cpp
#include <iostream>

class Printable {
public:
    virtual void print() const { std::cout << "Printable" << std::endl; }
    virtual ~Printable() = default;
};

class Serializable {
public:
    virtual void serialize() const { std::cout << "Serializable" << std::endl; }
    virtual ~Serializable() = default;
};

class Document : public Printable, public Serializable {
    std::string content;
public:
    Document(const std::string& c) : content(c) {}
    void print() const override {
        std::cout << "内容: " << content << std::endl;
    }
    void serialize() const override {
        std::cout << "序列化: " << content << std::endl;
    }
};

int main() {
    Document doc("Hello");
    doc.print();
    doc.serialize();

    // 两个基类指针
    Printable* p = &doc;
    p->print();
    return 0;
}
```

**输出：**
```
内容: Hello
序列化: Hello
内容: Hello
```

### 2.2 菱形问题

```cpp
#include <iostream>

class Animal {
public:
    int legs;
    Animal() : legs(0) {}
};

class Mammal : public Animal {};
class WingedAnimal : public Animal {};
// Bat同时继承Mammal和WingedAnimal
class Bat : public Mammal, public WingedAnimal {};

int main() {
    Bat bat;
    // bat.legs = 2;  // 编译错误！二义性：Animal::legs有两份
    // 解决方案：虚继承（见09-虚继承.md）
    return 0;
}
```

## 三、注意事项与常见陷阱

- 多继承会带来二义性问题，需用作用域解析
- 菱形继承导致基类有多个副本
- 优先使用单继承+组合替代多继承
- 接口类的多继承是安全的（纯虚函数，无状态）
