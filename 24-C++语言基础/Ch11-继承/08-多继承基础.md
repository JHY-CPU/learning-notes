# 多继承基础

## 一、概念说明

**多继承**（Multiple Inheritance，C++标准 §10.1）允许一个类同时从多个基类继承。C++是少数支持多继承的主流语言（Java/C#不支持类的多继承，只支持接口多继承）。多继承可以灵活组合功能，但也带来**菱形继承问题**（Diamond Problem）和**二义性**等复杂性。

### 1.1 多继承 vs 单继承

| 特性 | 单继承 | 多继承 |
|------|--------|--------|
| 基类数量 | 1个 | 多个 |
| 复杂度 | 低 | 高 |
| 菱形问题 | 不存在 | 需要虚继承解决 |
| 二义性 | 少见 | 需要作用域解析 |
| 设计模式 | 简单层次 | Mixin/接口组合 |

## 二、具体用法

### 2.1 基本多继承（Mixin模式）

```cpp
#include <iostream>
#include <string>

class Printable {
public:
    virtual void print() const { std::cout << "Printable" << std::endl; }
    virtual ~Printable() = default;
};

class Serializable {
public:
    virtual std::string serialize() const { return "Serializable{}"; }
    virtual ~Serializable() = default;
};

class Loggable {
public:
    virtual void log(const std::string& msg) const {
        std::cout << "[LOG] " << msg << std::endl;
    }
    virtual ~Loggable() = default;
};

// 组合多个接口/mixin
class Document : public Printable, public Serializable, public Loggable {
    std::string content;
public:
    Document(const std::string& c) : content(c) {}

    void print() const override {
        std::cout << "文档内容: " << content << std::endl;
    }

    std::string serialize() const override {
        return "{\"content\": \"" + content + "\"}";
    }

    void log(const std::string& msg) const override {
        std::cout << "[Document] " << msg << std::endl;
    }
};

int main() {
    Document doc("Hello World");
    doc.print();
    std::cout << doc.serialize() << std::endl;
    doc.log("操作记录");

    // 向上转型到不同的基类
    Printable* p = &doc;
    p->print();

    Serializable* s = &doc;
    std::cout << s->serialize() << std::endl;

    return 0;
}
```

### 2.2 菱形继承问题

```cpp
#include <iostream>

class Animal {
public:
    int legs;
    Animal() : legs(0) { std::cout << "Animal构造" << std::endl; }
    void breathe() { std::cout << "呼吸中..." << std::endl; }
};

class Mammal : public Animal {
public:
    Mammal() { std::cout << "Mammal构造" << std::endl; }
};

class WingedAnimal : public Animal {
public:
    WingedAnimal() { std::cout << "WingedAnimal构造" << std::endl; }
};

// 菱形继承：Bat有两份Animal子对象
class Bat : public Mammal, public WingedAnimal {
public:
    Bat() { std::cout << "Bat构造" << std::endl; }
};

int main() {
    Bat bat;
    // Animal被构造了两次！

    // bat.legs = 2;       // 编译错误！二义性
    // bat.breathe();      // 编译错误！二义性
    bat.Mammal::legs = 2;  // 必须指定路径
    bat.WingedAnimal::legs = 4;  // 另一份

    std::cout << "Mammal::legs = " << bat.Mammal::legs << std::endl;      // 2
    std::cout << "WingedAnimal::legs = " << bat.WingedAnimal::legs << std::endl; // 4

    return 0;
}
```

### 2.3 多继承中的构造顺序

```cpp
#include <iostream>

class A {
public:
    A(const std::string& s) { std::cout << "A(" << s << ")" << std::endl; }
};

class B {
public:
    B(const std::string& s) { std::cout << "B(" << s << ")" << std::endl; }
};

class C {
public:
    C(const std::string& s) { std::cout << "C(" << s << ")" << std::endl; }
};

// 多继承：基类按声明顺序构造
class D : public A, public B, public C {
public:
    D() : C("C"), B("B"), A("A") {
        // 基类按声明顺序(A→B→C)构造，忽略初始化列表顺序
        std::cout << "D构造" << std::endl;
    }
};

int main() {
    D d;
    return 0;
}
```

**输出：**
```
A(A)
B(B)
C(C)
D构造
```

## 三、注意事项与常见陷阱

1. **多继承带来二义性**：同名成员需用`BaseClass::`显式指定
2. **菱形继承导致基类有多个副本**：使用虚继承解决
3. **优先使用单继承+组合替代多继承**：除非设计上确实需要多继承
4. **接口类的多继承是安全的**：纯虚函数无状态，不会产生菱形问题
5. **基类构造顺序按声明顺序**：与初始化列表书写顺序无关
6. **多继承增加理解和维护难度**：应在设计阶段仔细权衡
