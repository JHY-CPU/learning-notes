# 切片问题

## 一、概念说明

**对象切片**（Object Slicing，C++标准 §10.2/§4.10）发生在将派生类对象赋值给基类对象时，派生类独有的成员和虚函数表信息被"切掉"，只保留基类部分。切片后虚函数调用不再多态，派生类特有数据完全丢失。

切片是**编译期行为**，不会有运行时错误或警告，是最隐蔽的继承陷阱之一。

### 1.1 切片发生的场景

| 场景 | 是否切片 | 说明 |
|------|---------|------|
| `Base b = derived;` | 是 | 对象赋值 |
| `void f(Base b)` 按值传参 | 是 | 值传递 |
| `std::vector<Base>` 存储 | 是 | 容器值语义 |
| `Base* p = &derived;` | 否 | 指针不切片 |
| `Base& r = derived;` | 否 | 引用不切片 |

## 二、具体用法

### 2.1 切片现象演示

```cpp
#include <iostream>
#include <string>

class Base {
public:
    int base_val = 1;
    virtual void show() const {
        std::cout << "Base::show() val=" << base_val << std::endl;
    }
    virtual ~Base() = default;
};

class Derived : public Base {
public:
    int derived_val = 2;
    std::string extra = "派生类数据";

    void show() const override {
        std::cout << "Derived::show() base=" << base_val
                  << " derived=" << derived_val
                  << " extra=" << extra << std::endl;
    }
};

int main() {
    Derived d;
    d.base_val = 10;
    d.derived_val = 20;
    d.extra = "重要数据";

    // ===== 切片发生 =====
    Base b = d;  // 拷贝构造：只拷贝Base部分

    std::cout << "--- 切片后 ---" << std::endl;
    b.show();  // Base::show() val=10 — 不再多态！
    // b.derived_val  不存在
    // b.extra        不存在

    // ===== 正确做法：引用 =====
    Base& ref = d;
    ref.show();  // Derived::show() — 多态正常

    // ===== 正确做法：指针 =====
    Base* ptr = &d;
    ptr->show();  // Derived::show() — 多态正常

    return 0;
}
```

### 2.2 函数传参中的切片

```cpp
#include <iostream>
#include <memory>

class Shape {
public:
    virtual double area() const = 0;
    virtual ~Shape() = default;
};

class Circle : public Shape {
    double r;
public:
    Circle(double r) : r(r) {}
    double area() const override { return 3.14159 * r * r; }
};

// 危险：按值传参发生切片
void printAreaBad(Shape s) {
    std::cout << "面积: " << s.area() << std::endl;  // 切片后调用Shape::area()
}

// 正确：按引用传参
void printAreaGood(const Shape& s) {
    std::cout << "面积: " << s.area() << std::endl;  // 多态正常
}

// 正确：按指针传参
void printAreaGood(Shape* s) {
    std::cout << "面积: " << s->area() << std::endl;  // 多态正常
}

int main() {
    Circle c(5);

    // printAreaBad(c);   // 切片！如果Shape不是抽象类会编译通过但结果错误
    printAreaGood(c);     // 正确：78.5398

    return 0;
}
```

### 2.3 容器中的切片

```cpp
#include <iostream>
#include <vector>
#include <memory>

class Base {
public:
    virtual void show() const { std::cout << "Base" << std::endl; }
    virtual ~Base() = default;
};

class Derived : public Base {
    int extra = 42;
public:
    void show() const override { std::cout << "Derived(" << extra << ")" << std::endl; }
};

int main() {
    // 错误：值容器发生切片
    std::cout << "=== 值容器（切片）===" << std::endl;
    std::vector<Base> vec_bad;
    vec_bad.push_back(Derived());  // 切片！
    for (const auto& v : vec_bad)
        v.show();  // 输出 "Base" — 切片后调用Base::show()

    // 正确：智能指针容器
    std::cout << "=== 智能指针容器（无切片）===" << std::endl;
    std::vector<std::unique_ptr<Base>> vec_good;
    vec_good.push_back(std::make_unique<Derived>());
    for (const auto& v : vec_good)
        v->show();  // 输出 "Derived(42)" — 多态正常

    return 0;
}
```

## 三、避免切片的方法

```cpp
// 方法1：使用引用或指针（最常用）
void process(const Base& b) { b.show(); }

// 方法2：禁止拷贝（彻底杜绝切片）
class NoSlice : public Base {
    NoSlice(const NoSlice&) = delete;
    NoSlice& operator=(const NoSlice&) = delete;
};

// 方法3：使用智能指针容器
std::vector<std::unique_ptr<Base>> container;

// 方法4：clone模式（需要时深拷贝）
class Clonable : public Base {
public:
    virtual std::unique_ptr<Clonable> clone() const = 0;
};
```

## 四、注意事项与常见陷阱

1. **按值传参会发生切片**：应使用`const Base&`或`const Base*`
2. **`std::vector<Base>`会切片**：使用`std::vector<std::unique_ptr<Base>>`
3. **切片是编译期行为**：不会有运行时错误，难以发现
4. **使用`= delete`拷贝构造可防止意外切片**：对不应拷贝的多态类
5. **切片丢失虚函数表信息**：对象变为基类类型，不再多态
6. **赋值运算符切片更隐蔽**：`base = derived;`只拷贝基类部分
