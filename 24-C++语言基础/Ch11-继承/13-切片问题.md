# 切片问题

## 一、概念说明

**对象切片**（Object Slicing）发生在将派生类对象赋值给基类对象时，派生类独有的成员被"切掉"，只保留基类部分。切片后虚函数调用也不再是多态的。

## 二、具体用法

### 2.1 切片现象

```cpp
#include <iostream>
#include <string>

class Base {
public:
    int base_val = 1;
    virtual void show() const {
        std::cout << "Base::show() val=" << base_val << std::endl;
    }
    virtual ~Base() = default;
};

class Derived : public Base {
public:
    int derived_val = 2;
    void show() const override {
        std::cout << "Derived::show() base=" << base_val
                  << " derived=" << derived_val << std::endl;
    }
};

int main() {
    Derived d;
    d.base_val = 10;
    d.derived_val = 20;

    Base b = d;  // 切片！derived_val丢失

    std::cout << "--- 切片后 ---" << std::endl;
    b.show();  // Base::show()，不再是多态
    std::cout << "b.derived_val: 不可访问" << std::endl;

    // 正确做法：使用引用或指针
    Base& ref = d;
    ref.show();  // Derived::show()，保持多态
    return 0;
}
```

**输出：**
```
--- 切片后 ---
Base::show() val=10
b.derived_val: 不可访问
Derived::show() base=10 derived=20
```

### 2.2 避免切片的方法

```cpp
// 方法1：使用指针/引用
void process(Base* b) { b->show(); }

// 方法2：禁止拷贝
class NoSlice : public Base {
    NoSlice(const NoSlice&) = delete;
    NoSlice& operator=(const NoSlice&) = delete;
};

// 方法3：使用智能指针
#include <memory>
void process(std::unique_ptr<Base> b) { b->show(); }
```

## 三、注意事项与常见陷阱

- 按值传参会发生切片，应使用引用或指针
- `std::vector<Base>`会切片，应使用`std::vector<Base*>`或`std::vector<std::unique_ptr<Base>>`
- 切片是编译期行为，不会有运行时错误
- 使用`delete`拷贝构造可防止意外切片
