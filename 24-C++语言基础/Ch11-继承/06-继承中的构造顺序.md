# 继承中的构造顺序

## 一、概念说明

在继承体系中，对象的构造遵循严格顺序：**虚基类 → 直接基类 → 成员变量 → 派生类函数体**。这保证了在任何构造阶段，所依赖的基类和成员都已就绪。

## 二、具体用法

### 2.1 多层继承构造

```cpp
#include <iostream>

class A {
public:
    A() { std::cout << "1. A构造" << std::endl; }
};

class B : public A {
public:
    B() { std::cout << "2. B构造" << std::endl; }
};

class C : public B {
    int val;
public:
    C() : val(0) { std::cout << "3. C构造" << std::endl; }
};

int main() {
    C obj;
    return 0;
}
```

**输出：**
```
1. A构造
2. B构造
3. C构造
```

### 2.2 基类和成员的构造顺序

```cpp
#include <iostream>

class Member {
public:
    Member(const std::string& n) { std::cout << "成员 " << n << " 构造" << std::endl; }
};

class Base {
public:
    Base() { std::cout << "基类构造" << std::endl; }
};

class Derived : public Base {
    Member m1;
    Member m2;
public:
    Derived() : m1("M1"), m2("M2") {
        std::cout << "派生类构造体" << std::endl;
    }
};

int main() {
    Derived d;
    return 0;
}
```

**输出：**
```
基类构造
成员 M1 构造
成员 M2 构造
派生类构造体
```

## 三、注意事项与常见陷阱

- 基类**先于**派生类构造
- 成员按**声明顺序**构造，非初始化列表顺序
- 构造顺序与析构顺序完全相反
- 初始化列表书写顺序不影响实际执行顺序
