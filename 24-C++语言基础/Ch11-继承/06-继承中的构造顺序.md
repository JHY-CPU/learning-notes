# 继承中的构造顺序

## 一、概念说明

在继承体系中，对象的构造遵循严格顺序（C++标准 §12.6.2）：**虚基类（声明顺序）→ 直接非虚基类（声明顺序）→ 非静态成员（声明顺序）→ 派生类构造函数体**。这保证了在任何构造阶段，所依赖的基类和成员都已初始化就绪。

### 1.1 完整构造顺序

```
1. 虚基类（按继承声明中的从左到右顺序，深度优先）
2. 直接非虚基类（按类定义中的声明顺序）
3. 非静态成员（按类定义中的声明顺序，非初始化列表顺序）
4. 派生类构造函数体执行
```

## 二、具体用法

### 2.1 多层继承构造顺序

```cpp
#include <iostream>

class A {
public:
    A() { std::cout << "1. A构造" << std::endl; }
    ~A() { std::cout << "A析构" << std::endl; }
};

class B : public A {
public:
    B() { std::cout << "2. B构造" << std::endl; }
    ~B() { std::cout << "B析构" << std::endl; }
};

class C : public B {
    int val;
public:
    C() : val(0) { std::cout << "3. C构造, val=" << val << std::endl; }
    ~C() { std::cout << "C析构" << std::endl; }
};

int main() {
    std::cout << "=== 创建C对象 ===" << std::endl;
    C obj;
    std::cout === "=== 作用域结束 ===" << std::endl;
    return 0;
}
```

**输出：**
```
=== 创建C对象 ===
1. A构造
2. B构造
3. C构造, val=0
=== 作用域结束 ===
C析构
B析构
A析构
```

### 2.2 基类和成员的构造顺序

```cpp
#include <iostream>

class Member {
    std::string name;
public:
    Member(const std::string& n) : name(n) {
        std::cout << "成员 " << name << " 构造" << std::endl;
    }
    ~Member() { std::cout << "成员 " << name << " 析构" << std::endl; }
};

class Base {
public:
    Base() { std::cout << "基类构造" << std::endl; }
    ~Base() { std::cout << "基类析构" << std::endl; }
};

class Derived : public Base {
    Member m1;   // 先声明
    Member m2;   // 后声明
public:
    // 注意：初始化列表的顺序不影响实际构造顺序
    // 实际顺序按声明顺序：m1先于m2
    Derived() : m2("M2"), m1("M1") {
        std::cout << "派生类构造体" << std::endl;
    }
    ~Derived() { std::cout << "派生类析构" << std::endl; }
};

int main() {
    std::cout << "=== 创建Derived ===" << std::endl;
    Derived d;
    std::cout << "=== 结束 ===" << std::endl;
    return 0;
}
```

**输出：**
```
=== 创建Derived ===
基类构造
成员 M1 构造     ← 按声明顺序，非初始化列表顺序
成员 M2 构造
派生类构造体
=== 结束 ===
派生类析构
成员 M2 析构
成员 M1 析构
基类析构
```

### 2.3 多继承的构造顺序

```cpp
#include <iostream>

class A {
public:
    A(const std::string& s) { std::cout << "A(" << s << ")" << std::endl; }
};

class B {
public:
    B(const std::string& s) { std::cout << "B(" << s << ")" << std::endl; }
};

class C : public A, public B {  // A先于B（声明顺序）
    int m1;
    int m2;
public:
    C() : B("B"), A("A"), m2(2), m1(1) {
        // 基类按声明顺序构造：A先，B后（忽略初始化列表顺序）
        // 成员按声明顺序：m1先，m2后
        std::cout << "C构造" << std::endl;
    }
};

int main() {
    C obj;
    return 0;
}
```

**输出：**
```
A(A)     ← 基类按声明顺序，非初始化列表顺序
B(B)
C构造
```

## 三、注意事项与常见陷阱

1. **基类先于派生类构造**：确保派生类构造时基类已就绪
2. **成员按声明顺序构造**：非初始化列表书写顺序，编译器可能警告顺序不一致
3. **构造顺序与析构顺序完全相反**：后构造的先析构
4. **初始化列表书写顺序不影响实际执行顺序**：应保持一致以避免混淆
5. **多继承中基类按声明顺序构造**：`class C : A, B`中A先于B
6. **在构造函数中调用虚函数不会多态**：此时派生类部分尚未构造
7. **构造函数中的异常可能跳过后续初始化**：已构造的部分会被自动析构
