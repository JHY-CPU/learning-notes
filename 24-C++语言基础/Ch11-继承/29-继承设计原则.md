# 继承设计原则

## 一、概念说明

良好的继承设计遵循**Liskov替换原则**（LSP）：派生类对象必须能够替换基类对象而不影响程序的正确性。即，任何使用基类的地方都应该能使用派生类。

## 二、具体用法

### 2.1 遵守LSP

```cpp
#include <iostream>
#include <vector>
#include <memory>

// 好的设计：所有形状都有面积和周长
class Shape {
public:
    virtual double area() const = 0;
    virtual double perimeter() const = 0;
    virtual ~Shape() = default;
};

class Circle : public Shape {
    double r;
public:
    Circle(double r) : r(r) {}
    double area() const override { return 3.14159 * r * r; }
    double perimeter() const override { return 2 * 3.14159 * r; }
};

class Square : public Shape {
    double s;
public:
    Square(double s) : s(s) {}
    double area() const override { return s * s; }
    double perimeter() const override { return 4 * s; }
};

// 函数只依赖Shape接口，不关心具体类型
void printInfo(const Shape& shape) {
    std::cout << "面积=" << shape.area()
              << ", 周长=" << shape.perimeter() << std::endl;
}

int main() {
    Circle c(5);
    Square s(3);
    printInfo(c);  // 可以用Circle替换Shape
    printInfo(s);  // 可以用Square替换Shape
    return 0;
}
```

**输出：**
```
面积=78.5398, 周长=31.4159
面积=9, 周长=12
```

### 2.2 违反LSP的例子

```cpp
// 不好的设计：Square继承Rectangle
// Rectangle有setWidth/setHeight，Square需要保持w==h
// Square不能完全替换Rectangle → 违反LSP
```

## 三、注意事项与常见陷阱

- 派生类不应加强前置条件或减弱后置条件
- 不要用继承仅仅为了代码复用（用组合）
- 继承层次不宜过深（建议不超过3层）
- 基类的`const`约束应被派生类遵守
- 考虑使用`final`类/函数限制过度扩展
