# 构造函数继承

## 一、概念说明

C++11引入`using Base::Base`语法（C++标准 §12.9），允许派生类继承基类的构造函数。编译器为每个基类构造函数生成一个对应的派生类构造函数，这些生成的构造函数简单地转发参数给基类构造函数，**不初始化**派生类新增的成员（除非有类内初始值）。

### 1.1 与默认构造的区别

| 特性 | `using Base::Base` | 手写构造函数 |
|------|-------------------|------------|
| 参数 | 与基类构造函数相同 | 自定义 |
| 基类初始化 | 自动调用基类构造 | 需显式调用 |
| 派生类成员 | 使用类内初始值或默认初始化 | 自定义初始化 |
| 代码量 | 一行 | 多行 |

## 二、具体用法

### 2.1 基本构造函数继承

```cpp
#include <iostream>
#include <string>

class Base {
public:
    Base() { std::cout << "Base()" << std::endl; }
    Base(int x) { std::cout << "Base(int " << x << ")" << std::endl; }
    Base(const std::string& s, int v) : Base() {
        std::cout << "Base(string, int): " << s << ", " << v << std::endl;
    }
    virtual ~Base() = default;
};

class Derived : public Base {
    int extra = 0;  // 类内初始值：继承的构造函数使用此值
public:
    using Base::Base;  // 继承Base的所有构造函数

    // 派生类自己的构造函数
    Derived(int x, int y) : Base(x), extra(y) {
        std::cout << "Derived(int, int): extra=" << extra << std::endl;
    }
};

int main() {
    Derived d1;                    // Base()
    Derived d2(42);                // Base(int 42)
    Derived d3("test", 10);        // Base() → Base(string, int)
    Derived d4(1, 2);              // Base(int 1) → Derived(int, int)

    std::cout << "d2.extra = " << d2.extra << std::endl;  // 0（类内初始值）
    std::cout << "d4.extra = " << d4.extra << std::endl;  // 2

    return 0;
}
```

### 2.2 默认参数与构造函数继承

```cpp
#include <iostream>

class Base {
public:
    Base(int a = 0, int b = 0, int c = 0) {
        std::cout << "Base(" << a << ", " << b << ", " << c << ")" << std::endl;
    }
};

class Derived : public Base {
    int extra = 0;
public:
    using Base::Base;  // 生成多个版本：
    // Derived(int a = 0, int b = 0, int c = 0)
    // 实际生成：Derived(), Derived(int), Derived(int,int), Derived(int,int,int)

    Derived(int a, int b, int c, int d) : Base(a, b, c), extra(d) {
        std::cout << "Derived(..., " << d << ")" << std::endl;
    }
};

int main() {
    Derived d1;           // Base(0, 0, 0)
    Derived d2(1);        // Base(1, 0, 0)
    Derived d3(1, 2);     // Base(1, 2, 0)
    Derived d4(1, 2, 3);  // Base(1, 2, 3)
    Derived d5(1, 2, 3, 4); // Derived自己的构造函数
    return 0;
}
```

### 2.3 构造函数继承与虚继承

```cpp
#include <iostream>

class Base {
public:
    Base(int x) { std::cout << "Base(" << x << ")" << std::endl; }
};

class Middle : virtual public Base {
public:
    using Base::Base;  // 继承Base(int)
    // 注意：虚基类的初始化仍然由最远派生类负责
};

class Derived : public Middle {
public:
    // 必须显式初始化虚基类，不能依赖Middle的继承构造函数
    Derived(int x) : Base(x), Middle() {
        std::cout << "Derived(" << x << ")" << std::endl;
    }
};

int main() {
    Derived d(42);
    // Middle m(10);  // 单独创建Middle时，它负责构造虚基类
    return 0;
}
```

## 三、注意事项与常见陷阱

1. **继承的构造函数不初始化派生类新增成员**：使用类内初始值或默认初始化
2. **基类构造函数有默认参数时会生成多个版本**：每个参数组合一个构造函数
3. **派生类同签名构造函数优先**：如果派生类自定义了同签名的构造函数，它优先于继承的版本
4. **虚继承场景复杂**：最远派生类必须负责虚基类的初始化
5. **构造函数继承不继承访问级别**：生成的派生类构造函数的访问级别与基类原始构造函数相同
6. **不能继承私有/受保护构造函数**：`using Base::Base`只继承可访问的构造函数
