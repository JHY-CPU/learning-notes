# 构造函数继承

## 一、概念说明

C++11引入`using Base::Base`语法，允许派生类继承基类的构造函数。实际上编译器生成的继承构造函数只是简单地调用基类构造函数，不初始化派生类新增的成员。

## 二、具体用法

### 2.1 using继承构造函数

```cpp
#include <iostream>
#include <string>

class Base {
public:
    Base() { std::cout << "Base()" << std::endl; }
    Base(int x) : Base() { std::cout << "Base(int)" << std::endl; }
    Base(const std::string& s) { std::cout << "Base(string)" << std::endl; }
    virtual ~Base() = default;
};

class Derived : public Base {
    int extra = 0;  // 类内初始值初始化
public:
    using Base::Base;  // 继承Base的所有构造函数

    // 额外的构造函数
    Derived(int x, int y) : Base(x), extra(y) {
        std::cout << "Derived(int, int)" << std::endl;
    }
};

int main() {
    Derived d1;              // 继承Base()
    Derived d2(42);          // 继承Base(int)
    Derived d3("test");      // 继承Base(string)
    Derived d4(1, 2);        // Derived自己的构造函数
    return 0;
}
```

**输出：**
```
Base()
Base()
Base(int)
Base(string)
Base()
Base(int)
Derived(int, int)
```

## 三、注意事项与常见陷阱

- 继承的构造函数**不初始化**派生类新增的成员（使用类内初始值或默认初始化）
- 如果基类构造函数有默认参数，继承时会生成多个构造函数版本
- 继承构造函数与派生类定义的同签名构造函数冲突时，派生类版本优先
- `using Base::Base`不能继承私有构造函数
