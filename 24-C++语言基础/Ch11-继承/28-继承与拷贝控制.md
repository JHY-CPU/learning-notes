# 继承与拷贝控制

## 一、概念说明

派生类的拷贝/移动操作必须正确处理基类部分。通常派生类的拷贝构造函数调用基类的拷贝构造函数。如果基类拷贝/移动操作被删除，派生类对应的也会被删除。

## 二、具体用法

### 2.1 派生类拷贝控制

```cpp
#include <iostream>

class Base {
    int baseData;
public:
    Base(int d = 0) : baseData(d) {}
    Base(const Base& o) : baseData(o.baseData) {
        std::cout << "Base拷贝构造" << std::endl;
    }
    Base& operator=(const Base& o) {
        baseData = o.baseData;
        std::cout << "Base拷贝赋值" << std::endl;
        return *this;
    }
    virtual ~Base() = default;
};

class Derived : public Base {
    int derivedData;
public:
    Derived(int b = 0, int d = 0) : Base(b), derivedData(d) {}

    // 派生类拷贝构造：调用基类拷贝构造
    Derived(const Derived& o) : Base(o), derivedData(o.derivedData) {
        std::cout << "Derived拷贝构造" << std::endl;
    }

    // 派生类拷贝赋值
    Derived& operator=(const Derived& o) {
        Base::operator=(o);  // 调用基类赋值
        derivedData = o.derivedData;
        std::cout << "Derived拷贝赋值" << std::endl;
        return *this;
    }
};

int main() {
    Derived d1(10, 20);
    Derived d2 = d1;  // 拷贝构造
    Derived d3;
    d3 = d1;          // 拷贝赋值
    return 0;
}
```

**输出：**
```
Base拷贝构造
Derived拷贝构造
Base拷贝赋值
Derived拷贝赋值
```

## 三、注意事项与常见陷阱

- 派生类拷贝构造**必须显式调用**基类拷贝构造（否则调用基类默认构造）
- 派生类拷贝赋值**必须显式调用**基类拷贝赋值
- 基类析构声明为virtual保证多态删除正确
- 定义了任何拷贝/移动操作会阻止编译器自动生成移动操作
