# 继承概述

## 一、概念说明

**继承**（Inheritance）是面向对象编程的三大特性之一，允许基于已有类创建新类，实现代码复用和层次化设计。子类（派生类）继承父类（基类）的属性和方法，并可添加新的功能。

### 1.1 is-a关系

继承表达"是一种"（is-a）关系：Dog **是** Animal。与之对应，**组合**（composition）表达"有一个"（has-a）关系：Car **有** Engine。

### 1.2 继承 vs 组合

| 关系 | 关键字 | 示例 | 设计原则 |
|------|--------|------|---------|
| is-a | `class Dog : public Animal` | Dog是一种Animal | 用于继承 |
| has-a | `class Car { Engine e; }` | Car有一个Engine | 优先使用 |

## 二、具体用法

### 2.1 基本继承

```cpp
#include <iostream>
#include <string>

class Animal {
protected:
    std::string name;
public:
    Animal(const std::string& n) : name(n) {}
    void eat() { std::cout << name << "在吃东西" << std::endl; }
    void sleep() { std::cout << name << "在睡觉" << std::endl; }
};

class Dog : public Animal {
    std::string breed;
public:
    Dog(const std::string& n, const std::string& b)
        : Animal(n), breed(b) {}
    void bark() { std::cout << name << "在汪汪叫!" << std::endl; }
};

int main() {
    Dog dog("旺财", "柴犬");
    dog.eat();    // 继承自Animal
    dog.sleep();  // 继承自Animal
    dog.bark();   // Dog特有
    return 0;
}
```

**输出：**
```
旺财在吃东西
旺财在睡觉
旺财在汪汪叫!
```

## 三、注意事项与常见陷阱

- "优先使用组合而非继承"是经典设计原则
- 继承层次不宜过深（一般不超过3层）
- 继承关系应符合**Liskov替换原则**（LSP）
- 基类的析构函数应声明为`virtual`
- `protected`成员可被派生类访问，但不能被外部访问
