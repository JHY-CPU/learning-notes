# 继承概述

## 一、概念说明

**继承**（Inheritance，C++标准 §10）是面向对象编程三大特性之一，允许基于已有类创建新类，实现代码复用和层次化设计。派生类（derived class）继承基类（base class）的成员，并可添加新功能或重写已有行为。

### 1.1 is-a关系

继承表达"是一种"（is-a）关系：Dog **是** Animal。与之对应，**组合**（composition）表达"有一个"（has-a）关系：Car **有** Engine。这是选择继承还是组合的核心判断标准。

### 1.2 继承 vs 组合

| 关系 | 语法 | 示例 | 耦合度 | 设计原则 |
|------|------|------|--------|---------|
| is-a | `class Dog : public Animal` | Dog是一种Animal | 紧密 | 用于继承 |
| has-a | `class Car { Engine e; }` | Car有一个Engine | 松散 | **优先使用** |

### 1.3 继承的三种方式

| 方式 | 基类public | 基类protected | 基类private | 用途 |
|------|-----------|--------------|-------------|------|
| public | public | protected | 不可访问 | is-a关系（常用） |
| protected | protected | protected | 不可访问 | 很少使用 |
| private | private | private | 不可访问 | 实现继承（优先用组合） |

## 二、具体用法

### 2.1 基本继承

```cpp
#include <iostream>
#include <string>

class Animal {
protected:
    std::string name;
    int age;
public:
    Animal(const std::string& n, int a) : name(n), age(a) {}
    void eat() { std::cout << name << "在吃东西" << std::endl; }
    void sleep() { std::cout << name << "在睡觉" << std::endl; }
    virtual void speak() const { std::cout << name << "发出声音" << std::endl; }
    virtual ~Animal() = default;
};

class Dog : public Animal {
    std::string breed;
public:
    Dog(const std::string& n, int a, const std::string& b)
        : Animal(n, a), breed(b) {}
    void bark() { std::cout << name << "在汪汪叫!" << std::endl; }
    void speak() const override { std::cout << name << ": 汪汪!" << std::endl; }
};

class Cat : public Animal {
public:
    using Animal::Animal;  // C++11：继承构造函数
    void speak() const override { std::cout << name << ": 喵喵!" << std::endl; }
};

int main() {
    Dog dog("旺财", 3, "柴犬");
    Cat cat("咪咪", 2);

    dog.eat();       // 继承自Animal
    dog.sleep();     // 继承自Animal
    dog.bark();      // Dog特有
    dog.speak();     // 重写版本

    cat.speak();     // 重写版本

    // 多态：基类指针指向派生类
    Animal* animals[] = {&dog, &cat};
    for (auto* a : animals)
        a->speak();  // 多态调用

    return 0;
}
```

### 2.2 继承体系中的对象模型

```cpp
#include <iostream>

class Base {
public:
    int base_val = 42;
    void base_func() { std::cout << "Base::base_func()" << std::endl; }
};

class Derived : public Base {
public:
    int derived_val = 100;
    void derived_func() { std::cout << "Derived::derived_func()" << std::endl; }
};

int main() {
    Derived d;

    // 派生类对象包含完整的基类子对象
    std::cout << "sizeof(Base) = " << sizeof(Base) << std::endl;
    std::cout << "sizeof(Derived) = " << sizeof(Derived) << std::endl;
    // Derived大小 = Base大小 + 派生类新增成员

    // 基类成员在派生类中可用
    d.base_val = 10;     // OK
    d.base_func();       // OK
    d.derived_func();    // OK

    // 地址关系：基类子对象位于派生类对象起始位置（通常）
    std::cout << "&d        = " << &d << std::endl;
    std::cout << "(Base*)&d = " << static_cast<Base*>(&d) << std::endl;

    return 0;
}
```

### 2.3 继承中的构造与析构

```cpp
#include <iostream>

class Base {
public:
    Base() { std::cout << "Base构造" << std::endl; }
    virtual ~Base() { std::cout << "Base析构" << std::endl; }
};

class Derived : public Base {
    int* data;
public:
    Derived() : data(new int[100]) {
        std::cout << "Derived构造" << std::endl;
    }
    ~Derived() override {
        delete[] data;
        std::cout << "Derived析构" << std::endl;
    }
};

int main() {
    Base* p = new Derived();
    // 构造顺序：Base → Derived
    delete p;
    // 析构顺序：Derived → Base（虚析构保证）
    return 0;
}
```

## 三、注意事项与常见陷阱

1. **"优先使用组合而非继承"**：经典设计原则（Effective C++条款32）
2. **继承层次不宜过深**：一般不超过3层，过深增加理解和维护难度
3. **继承关系应符合Liskov替换原则**（LSP）：派生类必须能替换基类使用
4. **基类的析构函数应声明为`virtual`**：否则通过基类指针删除派生类对象是未定义行为（§10.4）
5. **`protected`成员可被派生类访问**：但不能被外部访问，过度使用暴露实现细节
6. **Java/C#继承总是public**：C++支持三种继承方式，public继承表达is-a关系
7. **非public继承很少使用**：private继承可用组合替代，protected继承几乎不用
