# 受保护成员

## 一、概念说明

`protected`访问级别（C++标准 §11.2）允许类自身及其派生类访问成员，但不允许外部访问。它是`private`和`public`之间的中间级别，在继承体系中提供受控的内部接口。

### 1.1 访问级别对比

| 级别 | 类内 | 派生类 | 外部 | 用途 |
|------|------|--------|------|------|
| public | 可以 | 可以 | 可以 | 对外接口 |
| protected | 可以 | 可以 | 不可以 | 继承体系内部接口 |
| private | 可以 | 不可以 | 不可以 | 实现细节 |

## 二、具体用法

### 2.1 protected成员使用

```cpp
#include <iostream>
#include <string>

class Account {
protected:
    double balance;        // 派生类可直接操作
    std::string owner;
    void logTransaction(const std::string& op, double amount) {
        std::cout << "[" << owner << "] " << op << ": " << amount
                  << " 余额: " << balance << std::endl;
    }
public:
    Account(const std::string& o, double b) : owner(o), balance(b) {}
    virtual void display() const {
        std::cout << owner << ": " << balance << "元" << std::endl;
    }
    virtual ~Account() = default;
};

class SavingsAccount : public Account {
    double rate;
public:
    SavingsAccount(const std::string& o, double b, double r)
        : Account(o, b), rate(r) {}

    void addInterest() {
        double interest = balance * rate;  // 直接访问protected成员
        balance += interest;
        logTransaction("利息", interest);  // 调用protected函数
    }

    void display() const override {
        Account::display();
        std::cout << "利率: " << rate * 100 << "%" << std::endl;
    }
};

class CheckingAccount : public Account {
    double overdraftLimit;
public:
    CheckingAccount(const std::string& o, double b, double limit)
        : Account(o, b), overdraftLimit(limit) {}

    bool withdraw(double amount) {
        if (balance + overdraftLimit >= amount) {
            balance -= amount;  // 直接访问protected成员
            logTransaction("取款", amount);
            return true;
        }
        std::cout << "余额不足" << std::endl;
        return false;
    }
};

int main() {
    SavingsAccount sa("张三", 10000, 0.03);
    sa.display();
    sa.addInterest();
    sa.display();

    // sa.balance;  // 编译错误！外部不能访问protected

    CheckingAccount ca("李四", 5000, 2000);
    ca.withdraw(6000);  // OK：在派生类内部访问balance

    return 0;
}
```

### 2.2 NVI模式（Non-Virtual Interface）

```cpp
#include <iostream>

class Game {
protected:
    // protected虚函数：派生类可重写
    virtual bool doStart() = 0;
    virtual void doTurn(int turn) {
        std::cout << "回合" << turn << std::endl;
    }
    virtual bool doIsOver() const = 0;
    virtual void doEnd() {
        std::cout << "游戏结束" << std::endl;
    }

public:
    // public非虚接口：模板方法模式
    void play() {
        if (!doStart()) return;
        for (int turn = 1; !doIsOver(); ++turn)
            doTurn(turn);
        doEnd();
    }
    virtual ~Game() = default;
};

class Chess : public Game {
    int moves = 0;
protected:
    bool doStart() override {
        std::cout << "象棋开始" << std::endl;
        return true;
    }
    void doTurn(int turn) override {
        Game::doTurn(turn);  // 调用基类版本
        std::cout << "  走棋..." << std::endl;
        ++moves;
    }
    bool doIsOver() const override { return moves >= 3; }
    void doEnd() override {
        std::cout << "象棋结束，共" << moves << "步" << std::endl;
    }
};

int main() {
    Chess chess;
    chess.play();  // 外部只能通过public接口调用
    return 0;
}
```

### 2.3 protected的设计考量

```cpp
#include <iostream>

// 方案1：使用protected（暴露实现）
class BadBase {
protected:
    int data_;  // 派生类直接依赖此成员
public:
    BadBase(int d) : data_(d) {}
};

class BadDerived : public BadBase {
public:
    BadDerived() : BadBase(0) {}
    void set(int v) { data_ = v; }  // 直接操作
};

// 方案2：使用private + protected访问器（推荐）
class GoodBase {
    int data_;  // private：派生类不可直接访问
protected:
    // 受保护的访问器：提供受控访问
    int getData() const { return data_; }
    void setData(int v) { data_ = v; }
    virtual void onDataChanged() {}  // 钩子
public:
    GoodBase(int d) : data_(d) {}
};

class GoodDerived : public GoodBase {
public:
    GoodDerived() : GoodBase(0) {}
    void set(int v) {
        setData(v);  // 通过受控接口访问
    }
};
```

## 三、注意事项与常见陷阱

1. **`protected`成员对派生类可见但对外部不可见**：提供继承体系内部的访问
2. **过多`protected`成员暴露实现细节**：优先使用`private`+受保护的访问器方法
3. **虚函数可以是`protected`的**：NVI（Non-Virtual Interface）模式将虚函数设为protected
4. **`protected`继承会将基类的public成员降级为protected**：外部无法访问
5. **protected数据成员使派生类与基类实现耦合**：改变基类实现会影响所有派生类
6. **模板方法模式使用protected钩子函数**：基类定义算法骨架，派生类实现细节
