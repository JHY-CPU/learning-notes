# 受保护成员

## 一、概念说明

`protected`访问级别允许类自身及其派生类访问成员，但不允许外部访问。它是`private`和`public`之间的中间级别。

## 二、具体用法

### 2.1 protected成员使用

```cpp
#include <iostream>

class Account {
protected:
    double balance;  // 派生类可访问
    std::string owner;
public:
    Account(const std::string& o, double b) : owner(o), balance(b) {}
    virtual void display() const {
        std::cout << owner << ": " << balance << "元" << std::endl;
    }
    virtual ~Account() = default;
};

class SavingsAccount : public Account {
    double rate;
public:
    SavingsAccount(const std::string& o, double b, double r)
        : Account(o, b), rate(r) {}

    void addInterest() {
        balance += balance * rate;  // 访问protected成员
        std::cout << "利息已添加, 新余额: " << balance << std::endl;
    }

    void display() const override {
        Account::display();
        std::cout << "利率: " << rate * 100 << "%" << std::endl;
    }
};

int main() {
    SavingsAccount sa("张三", 10000, 0.03);
    sa.display();
    sa.addInterest();
    sa.display();
    // sa.balance;  // 编译错误！外部不能访问protected
    return 0;
}
```

**输出：**
```
张三: 10000元
利率: 3%
利息已添加, 新余额: 10300
张三: 10300元
利率: 3%
```

## 三、注意事项与常见陷阱

- `protected`成员对派生类可见但对外部不可见
- 过多`protected`成员暴露实现细节，优先使用`private`+访问器
- 虚函数可以是`protected`的（NVI模式）
- `protected`继承会将基类的public成员降级为protected
