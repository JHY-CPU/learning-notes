# 继承中的访问控制

## 一、概念说明

继承方式（`public`、`protected`、`private`）决定基类成员在派生类中的可访问性。这是C++特有的机制，Java/C#中继承总是等价于public继承。

### 1.1 访问规则表

| 基类成员 \ 继承方式 | public继承 | protected继承 | private继承 |
|---------------------|-----------|--------------|-------------|
| public成员 | public | protected | private |
| protected成员 | protected | protected | private |
| private成员 | 不可访问 | 不可访问 | 不可访问 |

## 二、具体用法

### 2.1 三种继承方式对比

```cpp
#include <iostream>

class Base {
public:
    int pub = 1;
protected:
    int prot = 2;
private:
    int priv = 3;
};

class PublicDerived : public Base {
    void test() {
        std::cout << pub << std::endl;    // OK: public→public
        std::cout << prot << std::endl;   // OK: protected→protected
        // std::cout << priv << std::endl; // 错误：不可访问
    }
};

class ProtectedDerived : protected Base {
    void test() {
        std::cout << pub << std::endl;    // OK: public→protected
        std::cout << prot << std::endl;   // OK: protected→protected
    }
};

class PrivateDerived : private Base {
    void test() {
        std::cout << pub << std::endl;    // OK: public→private
    }
};

int main() {
    PublicDerived pd;
    std::cout << pd.pub << std::endl;  // OK

    ProtectedDerived ppd;
    // ppd.pub;  // 错误：pub在protected继承后是protected
    return 0;
}
```

## 三、注意事项与常见陷阱

- 绝大多数情况下使用`public`继承
- `private`继承表示"用...实现"（实现继承），优先考虑组合替代
- `protected`继承很少使用
- 继承方式**不改变**基类成员本身的访问级别，只改变在派生类中的可见性
