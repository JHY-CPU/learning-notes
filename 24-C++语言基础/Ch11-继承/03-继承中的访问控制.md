# 继承中的访问控制

## 一、概念说明

继承方式（`public`、`protected`、`private`，C++标准 §10.2）决定基类成员在派生类中的可访问性。这是C++特有的机制，Java/C#中继承总是等价于public继承。

### 1.1 访问规则表

| 基类成员 \ 继承方式 | public继承 | protected继承 | private继承 |
|---------------------|-----------|--------------|-------------|
| public成员 | public | protected | private |
| protected成员 | protected | protected | private |
| private成员 | 不可访问 | 不可访问 | 不可访问 |

### 1.2 继承方式的语义

| 继承方式 | 语义 | 使用频率 | 说明 |
|----------|------|---------|------|
| public | is-a关系 | 非常常用 | 保持基类接口不变 |
| protected | 很少使用 | 极少 | 基类接口变为protected |
| private | is-implemented-in-terms-of | 偶尔 | 用组合替代更佳 |

## 二、具体用法

### 2.1 三种继承方式对比

```cpp
#include <iostream>

class Base {
public:
    int pub = 1;
    void publicFunc() { std::cout << "Base::publicFunc()" << std::endl; }
protected:
    int prot = 2;
    void protectedFunc() { std::cout << "Base::protectedFunc()" << std::endl; }
private:
    int priv = 3;
    void privateFunc() {}  // 任何继承方式都无法访问
};

// public继承：is-a关系
class PublicDerived : public Base {
public:
    void test() {
        std::cout << "pub=" << pub << std::endl;     // OK: public→public
        std::cout << "prot=" << prot << std::endl;   // OK: protected→protected
        // std::cout << priv << std::endl;            // 错误：不可访问
        publicFunc();     // OK
        protectedFunc();  // OK
    }
};

// protected继承
class ProtectedDerived : protected Base {
public:
    void test() {
        std::cout << "pub=" << pub << std::endl;     // OK: public→protected
        std::cout << "prot=" << prot << std::endl;   // OK: protected→protected
        publicFunc();     // OK，但变为protected
    }
};

// private继承：实现继承
class PrivateDerived : private Base {
public:
    void test() {
        std::cout << "pub=" << pub << std::endl;     // OK: public→private
        publicFunc();     // OK，但变为private
    }
    // 使用using选择性暴露
    using Base::pub;           // 重新暴露为public
    using Base::publicFunc;    // 重新暴露为public
};

int main() {
    PublicDerived pd;
    std::cout << pd.pub << std::endl;  // OK: public成员
    pd.publicFunc();                    // OK

    ProtectedDerived ppd;
    // ppd.pub;          // 错误：pub在protected继承后是protected
    // ppd.publicFunc(); // 错误：同上

    PrivateDerived privd;
    std::cout << privd.pub << std::endl;  // OK：using重新暴露
    privd.publicFunc();                    // OK：using重新暴露
    // privd.prot;      // 错误：仍然是private

    return 0;
}
```

### 2.2 继承链中的访问控制累积

```cpp
#include <iostream>

class A {
public:
    int x = 1;
protected:
    int y = 2;
private:
    int z = 3;
};

class B : protected A {
    // x变为protected, y仍为protected, z不可访问
public:
    void show() {
        std::cout << "B: x=" << x << ", y=" << y << std::endl;
    }
};

class C : public B {
    // B中x是protected，C继承后x仍是protected
public:
    void show() {
        std::cout << "C: x=" << x << ", y=" << y << std::endl;
    }
};

int main() {
    C c;
    c.show();
    // c.x;  // 错误：x在B中被降为protected，C中仍为protected
    return 0;
}
```

### 2.3 使用using恢复访问级别

```cpp
#include <iostream>

class Base {
public:
    void func(int) { std::cout << "Base::func(int)" << std::endl; }
    void func(double) { std::cout << "Base::func(double)" << std::endl; }
};

class Derived : private Base {
public:
    // 将Base::func恢复为public
    using Base::func;
    void func(const char*) { std::cout << "Derived::func(const char*)" << std::endl; }
};

int main() {
    Derived d;
    d.func(42);       // OK：using恢复的public
    d.func(3.14);     // OK
    d.func("hello");  // OK：Derived自己的版本
    return 0;
}
```

## 三、注意事项与常见陷阱

1. **绝大多数情况下使用`public`继承**：表达is-a关系，保持接口一致
2. **`private`继承表示"用...实现"**：语义类似组合，但能访问protected成员和重写虚函数
3. **`protected`继承很少使用**：介于public和private之间，实际场景极少需要
4. **继承方式不改变基类成员本身**：只改变在派生类和外部的可见性
5. **使用`using`可恢复被降级的访问级别**：在private/protected继承中选择性暴露基类接口
6. **组合通常优于private继承**：除非需要访问protected成员或重写虚函数
