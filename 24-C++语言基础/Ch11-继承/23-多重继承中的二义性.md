# 多重继承中的二义性

## 一、概念说明

多重继承中，如果多个基类有同名成员，派生类访问时会产生**二义性**（ambiguity）。必须使用作用域解析运算符`BaseClass::`明确指定。

## 二、具体用法

### 2.1 解决二义性

```cpp
#include <iostream>

class A {
public:
    void foo() { std::cout << "A::foo()" << std::endl; }
    void common() { std::cout << "A::common()" << std::endl; }
};

class B {
public:
    void foo() { std::cout << "B::foo()" << std::endl; }
    void common() { std::cout << "B::common()" << std::endl; }
};

class C : public A, public B {
public:
    // 在派生类中提供无歧义的版本
    void foo() {
        A::foo();  // 明确指定
    }
    void common() {
        A::common();
        B::common();
    }
};

int main() {
    C obj;
    obj.A::foo();  // 必须指定
    obj.B::foo();  // 必须指定
    obj.common();  // 派生类自己提供的版本

    // obj.foo();  // 编译错误！二义性（如果C没有定义foo）
    return 0;
}
```

**输出：**
```
A::foo()
B::foo()
A::common()
B::common()
```

## 三、注意事项与常见陷阱

- 同名成员访问必须用作用域解析
- 派生类定义同名成员可遮蔽基类成员
- 使用`using`引入特定基类成员
- 二义性检查在重载决议之前
