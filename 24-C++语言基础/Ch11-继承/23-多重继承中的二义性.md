# 多重继承中的二义性

## 一、概念说明

多重继承中（C++标准 §10.2），如果多个基类有同名成员，派生类访问时会产生**二义性**（ambiguity）。编译器无法确定应该使用哪个基类的成员，必须使用作用域解析运算符`BaseClass::`明确指定。

### 1.1 二义性的来源

| 二义性类型 | 示例 | 解决方案 |
|-----------|------|---------|
| 同名成员变量 | `A::x` vs `B::x` | `obj.A::x` |
| 同名成员函数 | `A::foo()` vs `B::foo()` | `obj.A::foo()` |
| 同名类型 | `A::Type` vs `B::Type` | `A::Type` |
| 菱形继承 | 两份基类子对象 | 虚继承 |

## 二、具体用法

### 2.1 基本二义性解决

```cpp
#include <iostream>

class A {
public:
    int value = 10;
    void foo() { std::cout << "A::foo()" << std::endl; }
    void common() { std::cout << "A::common()" << std::endl; }
    using Type = int;
};

class B {
public:
    int value = 20;
    void foo() { std::cout << "B::foo()" << std::endl; }
    void common() { std::cout << "B::common()" << std::endl; }
    using Type = double;
};

class C : public A, public B {
public:
    // 方案1：在派生类中提供无歧义的版本
    void foo() {
        A::foo();  // 明确选择A的版本
    }

    // 方案2：合并两个基类的行为
    void common() {
        A::common();
        B::common();
    }
};

int main() {
    C obj;

    // obj.value;    // 错误！二义性
    obj.A::value = 100;  // OK：显式指定
    obj.B::value = 200;

    obj.A::foo();  // 必须指定
    obj.B::foo();  // 必须指定
    obj.foo();     // OK：C自己的版本

    obj.common();  // C自己的版本（调用A和B的common）

    // A::Type a = 42;    // 需要消除二义性
    C::A::Type a = 42;   // OK

    return 0;
}
```

### 2.2 函数重载与二义性

```cpp
#include <iostream>

class A {
public:
    void func(int x) { std::cout << "A::func(int " << x << ")" << std::endl; }
};

class B {
public:
    void func(double x) { std::cout << "B::func(double " << x << ")" << std::endl; }
};

class C : public A, public B {
public:
    // using声明可以解决二义性
    using A::func;  // 引入A::func(int)
    using B::func;  // 引入B::func(double)
    // 现在C有两个func重载
};

class D : public A, public B {
    // 不使用using：A::func和B::func都被隐藏
public:
    void test() {
        // func(42);    // 错误！二义性（如果C没有定义同名函数）
        A::func(42);    // OK：显式指定
        B::func(3.14);  // OK：显式指定
    }
};

int main() {
    C c;
    c.func(42);    // A::func(int)
    c.func(3.14);  // B::func(double)

    D d;
    d.test();

    return 0;
}
```

### 2.3 使用虚继承彻底消除二义性

```cpp
#include <iostream>

class Base {
public:
    int value = 0;
    void show() { std::cout << "Base::value=" << value << std::endl; }
};

class A : virtual public Base {
public:
    A() { value = 1; }
};

class B : virtual public Base {
public:
    B() { value = 2; }
};

class C : public A, public B {
public:
    C() { value = 42; }  // 只有一份Base，无二义性
};

int main() {
    C c;
    c.show();    // Base::value=42 — 无二义性
    c.value = 100;  // OK — 只有一份
    std::cout << c.value << std::endl;  // 100

    return 0;
}
```

### 2.4 菱形继承中的二义性

```cpp
#include <iostream>

class Animal {
public:
    int legs = 0;
    void breathe() { std::cout << "呼吸" << std::endl; }
};

class Mammal : public Animal {};
class WingedAnimal : public Animal {};

class Bat : public Mammal, public WingedAnimal {};

int main() {
    Bat bat;
    // bat.legs = 2;       // 编译错误！二义性
    // bat.breathe();      // 编译错误！二义性

    // 必须指定路径
    bat.Mammal::legs = 2;
    bat.WingedAnimal::legs = 4;  // 另一份

    bat.Mammal::breathe();
    bat.WingedAnimal::breathe();

    // 两份legs不同
    std::cout << bat.Mammal::legs << std::endl;        // 2
    std::cout << bat.WingedAnimal::legs << std::endl;  // 4

    return 0;
}
```

## 三、注意事项与常见陷阱

1. **同名成员访问必须用作用域解析**：`obj.BaseA::member`
2. **派生类定义同名成员可遮蔽基类成员**：但不解决根本问题
3. **`using`引入特定基类成员**：将多个基类的重载合并
4. **二义性检查在重载决议之前**：如果存在二义性，编译器不会进入重载决议
5. **虚继承彻底消除菱形二义性**：只保留一份共享基类
6. **二义性是编译时错误**：编译器拒绝含糊的代码
