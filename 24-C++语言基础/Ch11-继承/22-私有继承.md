# 私有继承

## 一、概念说明

**私有继承**（Private Inheritance，C++标准 §10.2）将基类的所有public/protected成员变为派生类的private成员。它表达"用...实现"（is-implemented-in-terms-of）语义，与组合功能相似但能访问基类的protected成员和重写虚函数。

### 1.1 私有继承 vs 组合

| 特性 | 私有继承 | 组合 |
|------|---------|------|
| 语义 | is-implemented-in-terms-of | has-a |
| 访问protected | 可以 | 不可以 |
| 重写虚函数 | 可以 | 不可以 |
| 基类引用转换 | 不可以 | N/A |
| 空基类优化 | 可以（EBO） | 不可以 |
| 推荐程度 | 一般不推荐 | **推荐** |

## 二、具体用法

### 2.1 私有继承实现集合

```cpp
#include <iostream>
#include <vector>

// 私有继承：复用std::vector的实现
class IntSet : private std::vector<int> {
public:
    // 选择性暴露接口
    using std::vector<int>::push_back;
    using std::vector<int>::size;
    using std::vector<int>::begin;
    using std::vector<int>::end;
    using std::vector<int>::empty;

    bool contains(int val) const {
        for (size_t i = 0; i < size(); ++i)
            if ((*this)[i] == val) return true;
        return false;
    }

    void add(int val) {
        if (!contains(val))
            push_back(val);
    }

    // 不暴露insert、erase、clear等不适用于集合的操作
};

int main() {
    IntSet s;
    s.add(3); s.add(1); s.add(3);  // 重复不添加
    s.add(2);

    std::cout << "集合: ";
    for (auto it = s.begin(); it != s.end(); ++it)
        std::cout << *it << " ";
    std::cout << "\n大小: " << s.size() << std::endl;

    // std::vector<int>* p = &s;  // 错误！不能向上转型

    return 0;
}
```

### 2.2 使用私有继承重写虚函数

```cpp
#include <iostream>

class Logger {
public:
    virtual void write(const std::string& msg) {
        std::cout << "[LOG] " << msg << std::endl;
    }
    virtual ~Logger() = default;
};

// 私有继承：自定义日志行为
class FileLogger : private Logger {
    std::string filename;
protected:
    // 通过protected方法暴露虚函数机制
    void write(const std::string& msg) override {
        std::cout << "[" << filename << "] " << msg << std::endl;
    }
public:
    FileLogger(const std::string& f) : filename(f) {}
    using Logger::write;  // 可选：暴露基类write

    void log(const std::string& msg) {
        write(msg);  // 多态调用
    }
};

int main() {
    FileLogger fl("app.log");
    fl.log("系统启动");
    // Logger* p = &fl;  // 错误！不能向上转型
    return 0;
}
```

### 2.3 空基类优化（EBO）

```cpp
#include <iostream>

// 空类
class Empty {};

// 组合：空成员仍然占1字节
struct WithComposition {
    Empty e;   // 至少1字节
    int x;
};

// 私有继承：EBO允许空基类不占空间
struct WithInheritance : private Empty {
    int x;
};

int main() {
    std::cout << "sizeof(Empty) = " << sizeof(Empty) << std::endl;             // 1
    std::cout << "sizeof(WithComposition) = " << sizeof(WithComposition) << std::endl;  // 8 (int + 对齐)
    std::cout << "sizeof(WithInheritance) = " << sizeof(WithInheritance) << std::endl;  // 4 (EBO)

    // 在需要极致内存优化的场景，私有继承的EBO有优势
    return 0;
}
```

### 2.4 using选择性暴露接口

```cpp
#include <iostream>
#include <string>

class Base {
public:
    void publicA() { std::cout << "Base::publicA()" << std::endl; }
    void publicB() { std::cout << "Base::publicB()" << std::endl; }
    void publicC() { std::cout << "Base::publicC()" << std::endl; }
protected:
    void protectedFunc() { std::cout << "Base::protectedFunc()" << std::endl; }
};

class Derived : private Base {
public:
    // 只暴露publicA和publicB，不暴露publicC
    using Base::publicA;
    using Base::publicB;
    // 不using Base::publicC → 外部不可访问

    void useProtected() {
        protectedFunc();  // OK：私有继承可访问protected
    }
};

int main() {
    Derived d;
    d.publicA();   // OK
    d.publicB();   // OK
    // d.publicC();  // 错误：未暴露
    // d.protectedFunc();  // 错误：仍为private

    d.useProtected();

    return 0;
}
```

## 三、注意事项与常见陷阱

1. **私有继承不能向上转型到基类**：`Base* p = &derived;` 编译错误
2. **私有继承可以访问基类的protected成员**：组合做不到这点
3. **优先使用组合**：除非需要访问protected成员或重写虚函数
4. **可用`using`声明选择性地暴露基类接口**：精确控制哪些操作可见
5. **空基类优化（EBO）是私有继承的一个优势**：空类作为基类不占内存
6. **私有继承和组合在语义上的区别**：私有继承是"用...实现"，组合是"有一个"
