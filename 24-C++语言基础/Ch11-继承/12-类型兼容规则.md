# 类型兼容规则

## 一、概念说明

**向上转型**（Upcasting，C++标准 §10.2/§4.10）允许基类指针/引用指向派生类对象。这是多态的基础，由编译器自动完成且总是安全的。**向下转型**（Downcasting）则需要`dynamic_cast`或`static_cast`显式进行。

### 1.1 转型方向

| 转型 | 方向 | 安全性 | 语法 |
|------|------|--------|------|
| 向上转型 | 派生→基类 | 总是安全 | 隐式/自动 |
| 向下转型 | 基类→派生 | 需检查 | `dynamic_cast` |
| 交叉转型 | 同层互转 | 需检查 | `dynamic_cast`（多继承） |

## 二、具体用法

### 2.1 向上转型的各种形式

```cpp
#include <iostream>

class Animal {
public:
    virtual void speak() { std::cout << "Animal sound" << std::endl; }
    virtual ~Animal() = default;
};

class Dog : public Animal {
public:
    void speak() override { std::cout << "汪汪!" << std::endl; }
    void fetch() { std::cout << "捡球!" << std::endl; }
};

void makeSound(Animal& a) {  // 基类引用：多态
    a.speak();
}

void processAnimal(Animal a) {  // 按值传参：切片！
    a.speak();  // 调用Animal::speak()
}

int main() {
    Dog dog;

    // 1. 指针向上转型
    Animal* p = &dog;     // 隐式，安全
    p->speak();           // "汪汪!"（多态）

    // 2. 引用向上转型
    Animal& r = dog;      // 隐式，安全
    r.speak();            // "汪汪!"（多态）

    // 3. 对象赋值（危险！切片）
    Animal a = dog;       // 隐式，但发生切片
    a.speak();            // "Animal sound"（丢失Dog信息）

    // 4. 函数传参
    makeSound(dog);       // 引用传参，多态正常
    processAnimal(dog);   // 值传参，发生切片

    // 5. 通过基类指针无法访问派生类特有方法
    // p->fetch();  // 错误：Animal没有fetch()

    return 0;
}
```

### 2.2 向下转型与dynamic_cast

```cpp
#include <iostream>
#include <vector>
#include <memory>

class Animal {
public:
    virtual void speak() { std::cout << "Animal" << std::endl; }
    virtual ~Animal() = default;
};

class Dog : public Animal {
public:
    void speak() override { std::cout << "汪汪!" << std::endl; }
    void fetch() { std::cout << "捡球" << std::endl; }
};

class Cat : public Animal {
public:
    void speak() override { std::cout << "喵喵!" << std::endl; }
    void purr() { std::cout << "呼噜呼噜" << std::endl; }
};

int main() {
    std::vector<std::unique_ptr<Animal>> animals;
    animals.push_back(std::make_unique<Dog>());
    animals.push_back(std::make_unique<Cat>());

    for (auto& a : animals) {
        a->speak();  // 多态

        // 安全向下转型
        if (auto* dog = dynamic_cast<Dog*>(a.get())) {
            dog->fetch();
        } else if (auto* cat = dynamic_cast<Cat*>(a.get())) {
            cat->purr();
        }
    }

    // 引用版本：失败时抛出std::bad_cast
    try {
        Dog dog;
        Animal& ref = dog;
        Dog& dogRef = dynamic_cast<Dog&>(ref);  // OK

        Cat cat;
        Animal& ref2 = cat;
        Dog& dogRef2 = dynamic_cast<Dog&>(ref2);  // 抛出std::bad_cast
    } catch (const std::bad_cast& e) {
        std::cout << "转型失败: " << e.what() << std::endl;
    }

    return 0;
}
```

### 2.3 类型兼容在容器中的应用

```cpp
#include <iostream>
#include <vector>
#include <memory>

class Base {
public:
    virtual void show() const { std::cout << "Base" << std::endl; }
    virtual ~Base() = default;
};

class Derived : public Base {
public:
    void show() const override { std::cout << "Derived" << std::endl; }
};

int main() {
    // 正确：使用指针/智能指针容器
    std::vector<std::unique_ptr<Base>> vec;
    vec.push_back(std::make_unique<Base>());
    vec.push_back(std::make_unique<Derived>());

    for (const auto& v : vec)
        v->show();  // 多态

    // 错误：值容器会发生切片
    // std::vector<Base> vec2;
    // vec2.push_back(Derived());  // 切片！Derived信息丢失

    return 0;
}
```

## 三、注意事项与常见陷阱

1. **指针/引用向上转型保留多态行为**：虚函数调用正确的派生类版本
2. **对象赋值会发生对象切片**（slicing）：丢失派生类特有信息
3. **向下转型必须使用`dynamic_cast`**：并检查结果（指针版检查nullptr）
4. **`std::vector<Base>`会切片**：使用`std::vector<std::unique_ptr<Base>>`
5. **按值函数参数也会切片**：使用`const Base&`或`const Base*`
6. **`dynamic_cast`需要虚函数**：没有虚函数的类无法使用（RTTI依赖vtable）
