# 函数参数传递

## 一、概念说明

C++中函数参数传递有三种主要方式：**值传递**（pass by value）、**引用传递**（pass by reference）和**const引用传递**（pass by const reference）。选择合适的传递方式直接影响程序的性能和安全性。

值传递会创建参数的副本，函数内修改不影响原变量；引用传递直接操作原变量，无拷贝开销；const引用兼具引用的高效和值传递的安全性。

## 二、具体用法

### 2.1 值传递（Pass by Value）

```cpp
void increment(int x) {
    x++;  // 修改的是副本
    std::cout << "函数内 x = " << x << std::endl;
}

int main() {
    int num = 10;
    increment(num);
    std::cout << "函数外 num = " << num << std::endl;
}
```

```
函数内 x = 11
函数外 num = 10    // 原变量未改变
```

### 2.2 引用传递（Pass by Reference）

```cpp
void increment(int& x) {
    x++;  // 直接修改原变量
}

int main() {
    int num = 10;
    increment(num);
    std::cout << "num = " << num << std::endl;
}
```

```
num = 11    // 原变量被修改
```

### 2.3 const引用传递（推荐）

```cpp
// 大型对象推荐使用const引用，避免拷贝且防止修改
void printInfo(const std::string& name, const int& age) {
    std::cout << name << ", " << age << " years old" << std::endl;
    // age = 20;  // 编译错误：不能修改const引用
}

// 对比不同传递方式的性能
void byValue(std::string s) { }        // 拷贝整个字符串
void byRef(const std::string& s) { }   // 仅传递引用，无拷贝
```

## 三、注意事项与常见陷阱

- **基本类型**（int, double等）值传递即可，拷贝开销极小
- **大型对象**（string, vector等）优先使用`const T&`
- 需要修改原变量时使用`T&`（非const引用）
- 引用必须在声明时绑定，不能为`nullptr`
- 不要返回局部变量的引用，会导致悬垂引用
- `const`引用可以绑定到临时对象，延长其生命周期
