# 尾调用优化

## 一、概念说明

**尾调用**（Tail Call）是指函数的最后一步操作是调用另一个函数。编译器可以将尾调用优化为**跳转**（jump）而非新的函数调用，复用当前栈帧，从而不增加调用栈深度。

**尾递归**（Tail Recursion）是尾调用的特殊情况：函数最后一步递归调用自身。

C++标准**不强制要求**尾调用优化（TCO），但GCC、Clang、MSVC在`-O2`/`-O3`优化级别下通常会执行。

尾调用判定条件（缺一不可）：
1. 函数的返回值就是被调用函数的返回值（不能对返回值做额外计算）
2. 被调用在函数体的最后一条语句
3. 被调用的返回值不需要析构局部对象

## 二、具体用法

### 2.1 非尾调用 vs 尾调用

```cpp
#include <iostream>
using namespace std;

// 非尾调用：递归返回后还需执行乘法
int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);  // 不是尾调用（返回后要乘n）
}

// 尾递归版本：通过累积参数实现
int factorialTail(int n, int acc = 1) {
    if (n <= 1) return acc;
    return factorialTail(n - 1, n * acc);  // 尾调用（返回值直接传递）
}

// 另一个非尾调用的例子
int badSum(int n) {
    if (n <= 0) return 0;
    return n + badSum(n - 1);  // 不是尾调用：返回后要加n
}

// 尾递归版本
int goodSum(int n, int acc = 0) {
    if (n <= 0) return acc;
    return goodSum(n - 1, acc + n);  // 是尾调用
}

int main() {
    cout << factorialTail(10) << endl;  // 3628800
    cout << goodSum(100) << endl;       // 5050

    return 0;
}
```

### 2.2 尾调用与栈溢出

```cpp
#include <iostream>
using namespace std;

// 普通递归：factorial(100000) 大概率栈溢出
// int factorial(int n) {
//     if (n <= 1) return 1;
//     return n * factorial(n - 1);
// }

// 尾递归：factorialTail(100000) 在-O2下可正常运行
// 编译器将其优化为循环，不消耗栈空间
int factorialTail(int n, int acc = 1) {
    if (n <= 1) return acc;
    return factorialTail(n - 1, n * acc);
}

int main() {
    // -O0（无优化）：尾递归仍可能栈溢出
    // -O2/-O3：尾递归被优化为循环，大数安全
    cout << factorialTail(100) << endl;  // 安全

    return 0;
}
```

### 2.3 尾递归转迭代等价

```cpp
#include <iostream>
using namespace std;

// 尾递归
int factorialTail(int n, int acc = 1) {
    if (n <= 1) return acc;
    return factorialTail(n - 1, n * acc);
}

// 等价迭代（编译器优化后的效果）
int factorialIter(int n) {
    int acc = 1;
    while (n > 1) {
        acc *= n;
        n--;
    }
    return acc;
}

int main() {
    // 两者在-O2下生成的汇编几乎相同
    cout << factorialTail(10) << endl;   // 3628800
    cout << factorialIter(10) << endl;   // 3628800
    return 0;
}
```

### 2.5 阻止尾调用优化的情况

```cpp
#include <iostream>
#include <vector>
using namespace std;

struct Resource {
    int data;
    ~Resource() { /* 需要析构 */ }
};

// 情况1：有需要析构的局部变量
int badExample(int n) {
    Resource r{n};  // 有析构函数的局部变量
    if (n <= 0) return 0;
    return badExample(n - 1);  // 不能优化：需要析构r
}

// 修复：避免局部变量
int goodExample(int n) {
    if (n <= 0) return 0;
    return goodExample(n - 1);  // 可以优化
}

// 情况2：返回后做额外操作
int alsoNotTailCall(int n) {
    if (n <= 0) return 0;
    int result = alsoNotTailCall(n - 1);
    return result + 1;  // 不是尾调用：返回后还要加1
}
```

## 三、注意事项与常见陷阱

1. **C++标准不保证尾调用优化**：不要依赖此特性防止栈溢出，关键代码应转为迭代
2. **开启编译器优化（-O2/-O3）才可能触发TCO**：Debug模式下尾递归仍可能栈溢出
3. **尾调用位置不能有需要析构的局部变量**：析构函数必须在尾调用前执行
4. **递归转迭代是最可靠的避免栈溢出方法**：不依赖编译器优化
5. **不同编译器的TCO实现不同**：GCC/Clang较激进，MSVC较保守
6. **虚函数调用通常不被优化为尾调用**：虚表查找阻止优化
7. **尾调用优化影响调试**：栈帧被复用，回溯调用栈信息丢失
