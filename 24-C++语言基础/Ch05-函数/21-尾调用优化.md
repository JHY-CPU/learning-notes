# 尾调用优化

## 一、概念说明

**尾调用**（Tail Call）是指函数的最后一步操作是调用另一个函数。编译器可以将尾调用优化为**跳转**而非新的函数调用，从而不增加调用栈深度。**尾递归**是尾调用的特殊情况（最后调用自身）。

C++标准不强制要求尾调用优化（TCO），但大多数现代编译器在优化模式下会执行。

## 二、具体用法

### 2.1 非尾调用 vs 尾调用

```cpp
// 非尾调用：递归后还需执行乘法
int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);  // 不是尾调用
}

// 尾递归版本：通过累积参数实现
int factorialTail(int n, int acc = 1) {
    if (n <= 1) return acc;
    return factorialTail(n - 1, n * acc);  // 尾调用
}

std::cout << factorialTail(5) << std::endl;  // 输出: 120
```

### 2.2 尾调用与栈溢出

```cpp
// 普通递归：大数会导致栈溢出
// factorial(100000) 可能栈溢出

// 尾递归：编译器可能优化为循环，无栈溢出风险
// factorialTail(100000) 在开启优化时可正常运行
```

### 2.3 迭代等价

```cpp
// 尾递归本质上等价于循环
int factorialIter(int n) {
    int acc = 1;
    while (n > 1) {
        acc *= n;
        n--;
    }
    return acc;
}
```

### 2.4 指示编译器优化

```cpp
// GCC/Clang：使用musttail属性强制尾调用优化
__attribute__((musttail))
return helper(args);

// 一般依赖 -O2/-O3 优化级别
```

## 三、注意事项与常见陷阱

- C++标准**不保证**尾调用优化，不要依赖此特性防止栈溢出
- 开启编译器优化（-O2/-O3）才可能触发TCO
- Debug模式下尾递归仍可能栈溢出
- 尾调用位置不能有局部变量需要析构
- 递归转迭代是最可靠的避免栈溢出方法
