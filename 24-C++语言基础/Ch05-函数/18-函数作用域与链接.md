# 函数作用域与链接

## 一、概念说明

函数的**链接性**（linkage）决定了不同翻译单元中的同名函数是否代表同一实体。C++中函数有三种链接：**外部链接**（默认）、**内部链接**（`static`）和**无链接**（局部函数/lambda）。

`extern`关键字显式声明函数具有外部链接，`static`限制函数仅在当前翻译单元可见。

## 二、具体用法

### 2.1 外部链接（默认）

```cpp
// file1.cpp
// 具有外部链接，其他文件可extern声明后使用
int add(int a, int b) {
    return a + b;
}

// file2.cpp
extern int add(int a, int b);  // 声明，指向file1中的定义

int main() {
    std::cout << add(3, 5) << std::endl;  // 输出: 8
}
```

### 2.2 内部链接（static函数）

```cpp
// utils.cpp
// static函数只在当前文件可见，其他文件无法调用
static int helper(int x) {
    return x * x;
}

int compute(int x) {
    return helper(x) + 1;
}

// 另一个文件无法调用helper()
```

### 2.3 匿名命名空间（推荐替代static）

```cpp
// 现代C++推荐使用匿名命名空间替代static
namespace {
    int internalHelper(int x) {
        return x * 2;
    }
}

void process(int val) {
    std::cout << internalHelper(val) << std::endl;
}
```

### 2.4 inline函数的链接

```cpp
// inline函数在多个翻译单元中可有相同定义
// header.h
inline int square(int x) { return x * x; }
// 每个包含此头文件的翻译单元都有定义，链接器合并为一个
```

## 三、注意事项与常见陷阱

- `static`函数和匿名命名空间函数都是内部链接
- 在头文件中定义非inline函数会导致多重定义错误
- `extern`声明不需要函数体
- `inline`函数允许在多个翻译单元中重复定义（放宽ODR规则）
- C++17的`inline`变量也具有类似特性
