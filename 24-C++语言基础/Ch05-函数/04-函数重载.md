# 函数重载

## 一、概念说明

函数重载（Function Overload，C++11标准 §13）允许在同一作用域内定义多个**同名函数**，只要它们的**参数列表不同**（类型、数量或顺序不同）。编译器根据调用时的实参类型自动选择最佳匹配的版本，称为**重载决议**（overload resolution）。

关键规则：
- 返回类型不同**不能**作为重载的依据
- `const`和非`const`成员函数（基于this指针类型）可构成重载
- 默认参数不参与重载决议

重载决议优先级（从高到低）：
1. 精确匹配
2. 类型提升（如short到int）
3. 标准转换（如int到double）
4. 用户自定义转换
5. 省略号匹配（...）

## 二、具体用法

### 2.1 参数类型不同

```cpp
#include <iostream>
#include <string>
using namespace std;

int add(int a, int b) {
    return a + b;
}

double add(double a, double b) {
    return a + b;
}

string add(const string& a, const string& b) {
    return a + b;
}

int main() {
    cout << add(3, 5) << endl;           // int版本，输出: 8
    cout << add(1.5, 2.3) << endl;       // double版本，输出: 3.8
    cout << add("Hi"s, " there"s) << endl;  // string版本
    return 0;
}
```

### 2.2 参数数量不同

```cpp
int multiply(int a, int b) { return a * b; }
int multiply(int a, int b, int c) { return a * b * c; }

cout << multiply(2, 3) << endl;      // 6
cout << multiply(2, 3, 4) << endl;   // 24
```

### 2.3 const成员函数重载

```cpp
#include <iostream>
#include <string>
using namespace std;

class Text {
    string data;
public:
    Text(const string& s) : data(s) {}

    // 非const版本：可修改
    char& operator[](size_t i) {
        cout << "非const版本\n";
        return data[i];
    }

    // const版本：只读
    const char& operator[](size_t i) const {
        cout << "const版本\n";
        return data[i];
    }
};

int main() {
    Text text("hello");
    const Text ctext("world");

    text[0] = 'H';      // 调用非const版本
    cout << ctext[0] << endl;  // 调用const版本

    return 0;
}
```

### 2.4 重载决议示例

```cpp
#include <iostream>
using namespace std;

void func(int)    { cout << "int\n"; }
void func(double) { cout << "double\n"; }
void func(long)   { cout << "long\n"; }

int main() {
    func(42);      // int字面量 → int（精确匹配）
    func(42.0);    // double字面量 → double（精确匹配）
    func(42.0f);   // float → double（标准转换）

    char c = 'A';
    func(c);       // char → int（整型提升）> char → long

    return 0;
}
```

### 2.5 避免重载歧义

```cpp
// 歧义示例
void process(int) { }
void process(double) { }

// process(42.0f);  // float → int 和 float → double 同样好，编译错误

// 解决方案1：显式类型转换
process(static_cast<int>(42.0f));
process(static_cast<double>(42.0f));

// 解决方案2：添加float重载
void process(float f) { process(static_cast<double>(f)); }
```

## 三、注意事项与常见陷阱

1. **仅返回类型不同不构成重载**：编译报错（无法从调用推导返回类型）
2. **默认参数与重载可能产生歧义**：`void f(int, int=0)` 和 `void f(int)` 在 `f(1)` 时冲突
3. **const引用参数的重载**：`void f(int)` 和 `void f(const int&)` 在 `f(42)` 时歧义
4. **整数字面量默认int，浮点默认double**：`42`是int，`42.0`是double
5. **重载决议优先精确匹配**：其次是类型提升，最后标准转换
6. **函数模板参与重载决议**：模板实例化版本与非模板版本优先级不同
7. **避免过多重载**：超过3-4个重载版本考虑使用命名参数模式或构造函数模板
