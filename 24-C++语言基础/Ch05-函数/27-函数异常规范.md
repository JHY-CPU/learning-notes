# 函数异常规范

## 一、概念说明

C++的异常规范用于声明函数可能抛出的异常类型。C++11废弃了动态异常规范（`throw(...)`），引入了`noexcept`运算符和说明符。`noexcept`表示函数**承诺不抛出异常**，违反此承诺将导致`std::terminate`被调用。

异常安全保证分为三个级别：**基本保证**（不泄漏资源）、**强保证**（操作要么完成要么回滚）、**不抛异常保证**（`noexcept`）。

## 二、具体用法

### 2.1 noexcept说明符

```cpp
void safeFunction() noexcept {
    // 承诺不抛异常
    std::cout << "安全函数\n";
}

void riskyFunction() {
    throw std::runtime_error("出错了");
}

void wrapper() noexcept {
    // 如果内部抛出异常，调用std::terminate
    safeFunction();
}

std::cout << noexcept(safeFunction()) << std::endl;   // 输出: 1
std::cout << noexcept(riskyFunction()) << std::endl;  // 输出: 0
```

### 2.2 条件noexcept

```cpp
// 根据表达式是否可能抛异常来决定
template <typename T>
void swapValues(T& a, T& b) noexcept(std::is_nothrow_move_constructible_v<T>) {
    T temp = std::move(a);
    a = std::move(b);
    b = std::move(temp);
}

// int的swap是noexcept
std::cout << noexcept(swapValues<int>) << std::endl;  // 输出: 1
```

### 2.3 移动操作与noexcept

```cpp
class Resource {
    int* data;
public:
    // 移动构造标记noexcept，vector扩容时才使用移动而非拷贝
    Resource(Resource&& other) noexcept : data(other.data) {
        other.data = nullptr;
    }
    // 移动赋值
    Resource& operator=(Resource&& other) noexcept {
        if (this != &other) {
            delete data;
            data = other.data;
            other.data = nullptr;
        }
        return *this;
    }
};
```

### 2.4 异常安全级别

```cpp
// 基本保证：不泄漏资源
void basicGuarantee() {
    auto ptr = std::make_unique<int>(42);
    riskyOperation();  // 可能抛异常，但unique_ptr保证释放
}

// 强保证：提交或回滚
void strongGuarantee() {
    auto copy = data_;        // 先备份
    modify(data_);            // 可能抛异常
    // 成功则继续，异常则copy销毁恢复原状
}
```

## 三、注意事项与常见陷阱

- `noexcept`是函数类型的一部分，影响重载决议
- 析构函数默认`noexcept`，不应抛异常
- 移动构造/赋值应尽量标记`noexcept`，否则容器可能选择拷贝
- `noexcept`运算符在编译期检查表达式是否可能抛异常
- 过度使用`noexcept`可能导致程序在异常时直接终止
