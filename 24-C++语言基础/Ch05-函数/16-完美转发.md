# 完美转发

## 一、概念说明

完美转发（Perfect Forwarding，C++11标准 §20.2.3）是指在泛型代码中将函数参数**原封不动**地转发给另一个函数——保持其值类别（左值/右值）和const/volatile限定。C++11通过**万能引用**（Universal Reference / Forwarding Reference）和`std::forward`实现。

核心机制：
- **万能引用**：`T&&` 在模板中且T由推导决定时，可根据传入参数类型推导为左值引用或右值引用
- **引用折叠规则**：`T& &` -> `T&`，`T& &&` -> `T&`，`T&& &` -> `T&`，`T&& &&` -> `T&&`
- **std::forward**：条件转换，仅当T推导为非引用类型时才转为右值引用

## 二、具体用法

### 2.1 万能引用

```cpp
#include <iostream>
using namespace std;

// 万能引用：T&& 在模板中且T是推导出来的
template <typename T>
void wrapper(T&& arg) {
    cout << __PRETTY_FUNCTION__ << endl;
}

int main() {
    int x = 42;
    wrapper(x);     // T = int&, arg类型 = int&（左值传入）
    wrapper(42);    // T = int,  arg类型 = int&&（右值传入）
    wrapper(move(x)); // T = int, arg类型 = int&&

    // 注意：非模板中的T&&不是万能引用
    // void func(int&& arg);  // 这是纯右值引用，不是万能引用

    return 0;
}
```

### 2.2 std::forward完美转发

```cpp
#include <iostream>
#include <utility>
using namespace std;

void target(int& x)  { cout << "左值版本: " << x << endl; }
void target(int&& x) { cout << "右值版本: " << x << endl; }

template <typename T>
void forwarder(T&& arg) {
    target(forward<T>(arg));  // 完美转发
}

int main() {
    int x = 10;
    forwarder(x);       // 输出: 左值版本: 10
    forwarder(20);      // 输出: 右值版本: 20
    forwarder(move(x)); // 输出: 右值版本: 10

    return 0;
}
```

### 2.3 不使用forward的问题

```cpp
#include <iostream>
#include <utility>
using namespace std;

void process(string& s)  { cout << "左值: " << s << endl; }
void process(string&& s) { cout << "右值: " << s << endl; }

// 不使用forward：参数永远是左值
template <typename T>
void badForward(T&& arg) {
    process(arg);  // arg有名字，始终是左值！
}

// 正确：使用forward保持值类别
template <typename T>
void goodForward(T&& arg) {
    process(forward<T>(arg));
}

int main() {
    string s = "hello";
    badForward(s);    // 左值
    badForward("hi"); // 左值（即使传入右值）← 丢失了移动语义！

    goodForward(s);    // 左值
    goodForward("hi"); // 右值 ← 保留了移动语义

    return 0;
}
```

### 2.4 工厂函数中的应用

```cpp
#include <iostream>
#include <memory>
#include <utility>
using namespace std;

// 完美转发的工厂函数
template <typename T, typename... Args>
unique_ptr<T> makeUnique(Args&&... args) {
    return unique_ptr<T>(new T(forward<Args>(args)...));
}

class Widget {
    int id;
    string name;
public:
    Widget(int i, const string& n) : id(i), name(n) {
        cout << "构造 Widget(" << id << ", " << name << ")\n";
    }
};

int main() {
    auto w1 = makeUnique<Widget>(1, "Button");
    auto w2 = makeUnique<Widget>(2, "Label");
    // 参数被完美转发到Widget构造函数

    return 0;
}
```

### 2.5 std::apply和完美转发

```cpp
#include <iostream>
#include <tuple>
#include <utility>
using namespace std;

int add(int a, int b) { return a + b; }

int main() {
    auto args = make_tuple(3, 4);
    int result = apply(add, args);  // 内部使用完美转发
    cout << "3 + 4 = " << result << endl;  // 7

    return 0;
}
```

## 三、注意事项与常见陷阱

1. **T&&只在模板参数推导时才是万能引用**：非模板或显式指定类型的T&&是纯右值引用
2. **不使用std::forward会导致参数永远是左值**：有名字的右值引用变量是左值
3. **std::forward仅在模板中、参数为推导出的T时有意义**：其他场景无意义
4. **完美转发是实现make_unique、emplace等函数的关键技术**：库函数的核心机制
5. **某些类型无法完美转发**：花括号初始化列表 `{}` 不能作为模板参数推导
6. **完美转发不处理数组退化**：数组参数会退化为指针，除非用引用捕获
7. **const T&& 不是万能引用**：`const T&&` 始终是右值引用
