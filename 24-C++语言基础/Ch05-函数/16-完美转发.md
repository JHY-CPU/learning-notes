# 完美转发

## 一、概念说明

完美转发（Perfect Forwarding）是指在泛型代码中将函数参数**原封不动**地转发给另一个函数——保持其值类别（左值/右值）和const/volatile限定。C++11通过**万能引用**（Universal Reference）和`std::forward`实现。

引用折叠规则：`T& &` -> `T&`, `T& &&` -> `T&`, `T&& &` -> `T&`, `T&& &&` -> `T&&`。

## 二、具体用法

### 2.1 万能引用

```cpp
// 万能引用：T&& 在模板中且T是推导出来的
template <typename T>
void wrapper(T&& arg) {
    // arg可以是左值引用或右值引用，取决于传入的参数
    std::cout << __PRETTY_FUNCTION__ << std::endl;
}

int x = 42;
wrapper(x);     // T = int&, arg = int&
wrapper(42);    // T = int,  arg = int&&
```

### 2.2 std::forward完美转发

```cpp
void target(int& x)  { std::cout << "左值版本: " << x << std::endl; }
void target(int&& x) { std::cout << "右值版本: " << x << std::endl; }

template <typename T>
void forwarder(T&& arg) {
    target(std::forward<T>(arg));  // 完美转发
}

int x = 10;
forwarder(x);   // 输出: 左值版本: 10
forwarder(20);  // 输出: 右值版本: 20
```

### 2.3 工厂函数中的应用

```cpp
template <typename T, typename... Args>
std::unique_ptr<T> make(Args&&... args) {
    return std::unique_ptr<T>(
        new T(std::forward<Args>(args)...)
    );
}

// 完美转发构造参数
auto str = make<std::string>("hello", 5);  // string("hello", 5)
```

## 三、注意事项与常见陷阱

- `T&&`只在模板参数推导时才是万能引用，`int&&`是纯右值引用
- 不使用`std::forward`会导致参数永远是左值（引用折叠后）
- `std::forward`仅在模板中、参数为推导出的`T`时有意义
- 完美转发是实现`make_unique`、`emplace`等函数的关键技术
- 某些类型（如花括号初始化列表`{}`）无法完美转发
