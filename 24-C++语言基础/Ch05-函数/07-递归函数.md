# 递归函数

## 一、概念说明

递归函数是**直接或间接调用自身**的函数。递归的核心思想是将一个大问题分解为与原问题结构相同但规模更小的子问题。

递归三要素：
1. **基准条件**（base case）：终止递归的条件
2. **递归条件**（recursive case）：函数调用自身
3. **收敛性**：每次递归必须向基准条件靠近

## 二、具体用法

### 2.1 经典递归：阶乘

```cpp
int factorial(int n) {
    if (n <= 1) return 1;        // 基准条件
    return n * factorial(n - 1); // 递归条件
}

std::cout << "5! = " << factorial(5) << std::endl;
// 输出: 5! = 120
// 调用过程: factorial(5) -> 5*factorial(4) -> 5*4*factorial(3) -> ... -> 120
```

### 2.2 递归追踪

```cpp
int fib(int n, int depth = 0) {
    std::string indent(depth * 2, ' ');
    std::cout << indent << "fib(" << n << ")" << std::endl;
    if (n <= 1) return n;
    return fib(n - 1, depth + 1) + fib(n - 2, depth + 1);
}

fib(4);
// 输出:
// fib(4)
//   fib(3)
//     fib(2)
//       fib(1)
//       fib(0)
//     fib(1)
//   fib(2)
//     fib(1)
//     fib(0)
```

### 2.3 尾递归优化

```cpp
// 普通递归：需要保留栈帧
int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);  // 乘法在递归返回后执行
}

// 尾递归：结果通过参数累积
int factorialTail(int n, int acc = 1) {
    if (n <= 1) return acc;
    return factorialTail(n - 1, n * acc);  // 最后一步是递归调用
}

std::cout << factorialTail(5) << std::endl;
// 输出: 120
```

## 三、注意事项与常见陷阱

- 缺少基准条件会导致**无限递归**，最终栈溢出（stack overflow）
- 递归深度过大时优先考虑转为迭代
- 尾递归可被编译器优化为循环（但C++标准不保证）
- 递归代码简洁但可能有额外的函数调用开销
- 使用递归时注意重复计算问题（如斐波那契），可使用记忆化优化
