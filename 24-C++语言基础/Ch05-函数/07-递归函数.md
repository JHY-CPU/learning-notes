# 递归函数

## 一、概念说明

递归函数是**直接或间接调用自身**的函数（C++11标准 §5.2.2）。递归的核心思想是将大问题分解为结构相同但规模更小的子问题，直到达到可直接求解的基准情况。

递归三要素：
1. **基准条件**（base case）：终止递归的条件
2. **递归条件**（recursive case）：函数调用自身
3. **收敛性**：每次递归必须向基准条件靠近

递归与迭代的对比：

| 特性 | 递归 | 迭代 |
|------|------|------|
| 代码可读性 | 树形问题更直观 | 线性问题更直观 |
| 栈空间 | O(n)，可能栈溢出 | O(1) |
| 性能 | 函数调用开销 | 更快 |
| 调试 | 较困难 | 较容易 |

## 二、具体用法

### 2.1 经典递归：阶乘

```cpp
#include <iostream>
using namespace std;

int factorial(int n) {
    if (n <= 1) return 1;        // 基准条件
    return n * factorial(n - 1); // 递归条件
}

int main() {
    cout << "5! = " << factorial(5) << endl;  // 120
    // 调用过程: 5*4*3*2*1 = 120
    return 0;
}
```

### 2.2 斐波那契与记忆化

```cpp
#include <iostream>
#include <vector>
using namespace std;

// 普通递归：O(2^n)，大量重复计算
int fibSlow(int n) {
    if (n <= 1) return n;
    return fibSlow(n - 1) + fibSlow(n - 2);
}

// 记忆化递归：O(n)，避免重复计算
vector<int> memo(100, -1);

int fibMemo(int n) {
    if (n <= 1) return n;
    if (memo[n] != -1) return memo[n];
    memo[n] = fibMemo(n - 1) + fibMemo(n - 2);
    return memo[n];
}

// 尾递归版本
int fibTail(int n, int a = 0, int b = 1) {
    if (n == 0) return a;
    return fibTail(n - 1, b, a + b);  // 尾调用
}

int main() {
    cout << "fib(10) = " << fibMemo(10) << endl;  // 55
    return 0;
}
```

### 2.3 树形递归：目录遍历

```cpp
#include <iostream>
#include <string>
using namespace std;

struct Node {
    string name;
    Node* left = nullptr;
    Node* right = nullptr;
};

// 递归遍历二叉树
void printTree(Node* node, int depth = 0) {
    if (!node) return;  // 基准条件：空节点

    string indent(depth * 2, ' ');
    cout << indent << node->name << endl;

    printTree(node->left, depth + 1);   // 递归左子树
    printTree(node->right, depth + 1);  // 递归右子树
}

int main() {
    Node root{"root"};
    Node l{"left"}, r{"right"};
    Node ll{"left.left"}, lr{"left.right"};
    root.left = &l; root.right = &r;
    l.left = &ll; l.right = &lr;

    printTree(&root);
    // root
    //   left
    //     left.left
    //     left.right
    //   right

    return 0;
}
```

### 2.4 尾递归优化

```cpp
// 普通递归：递归后还需乘法，需要保留栈帧
int factorialNormal(int n) {
    if (n <= 1) return 1;
    return n * factorialNormal(n - 1);  // 不是尾调用
}

// 尾递归：结果通过累积参数传递，最后一步是递归调用
int factorialTail(int n, int acc = 1) {
    if (n <= 1) return acc;
    return factorialTail(n - 1, n * acc);  // 尾调用
}

// 编译器可能将尾递归优化为循环（-O2）
// factorialTail(100000) 在优化模式下不会栈溢出
```

### 2.5 递归转迭代

```cpp
#include <stack>
using namespace std;

// 递归写法
void traverseRecursive(Node* node) {
    if (!node) return;
    cout << node->name << endl;
    traverseRecursive(node->left);
    traverseRecursive(node->right);
}

// 迭代等价（使用显式栈）
void traverseIterative(Node* root) {
    stack<Node*> s;
    if (root) s.push(root);

    while (!s.empty()) {
        Node* node = s.top(); s.pop();
        cout << node->name << endl;

        if (node->right) s.push(node->right);
        if (node->left) s.push(node->left);
    }
}
```

## 三、注意事项与常见陷阱

1. **缺少基准条件导致无限递归**：最终栈溢出（stack overflow）
2. **递归深度过大时优先考虑迭代**：默认栈大小通常1-8MB
3. **尾递归可被优化为循环**：但C++标准不保证，依赖编译器和优化级别
4. **递归代码简洁但有开销**：每次调用需保存返回地址、局部变量
5. **斐波那契等有重复计算的问题**：使用记忆化（memoization）或动态规划
6. **大对象按值传参会增加开销**：递归函数参数尽量用引用
7. **递归深度可通过模板在编译期计算**：constexpr递归在编译期求值，无运行时栈开销
