# 函数返回值

## 一、概念说明

函数的返回值是函数执行后传递给调用者的结果（C++11标准 §6.6.3）。C++提供多种返回方式：值返回、引用返回、右值引用返回，以及编译器优化机制（RVO/NRVO）。

返回值类型影响：
- **值返回**：调用拷贝/移动构造函数（可能被RVO消除）
- **引用返回**：直接返回对已有对象的引用，零开销
- **右值引用返回**：用于移动语义场景

**RVO（Return Value Optimization）** 是C++17起保证的优化（对于prvalue），允许直接在调用者栈空间中构造返回对象，省去拷贝/移动操作。

## 二、具体用法

### 2.1 基本返回值

```cpp
#include <iostream>
#include <string>
using namespace std;

int max(int a, int b) {
    return (a > b) ? a : b;
}

string greet(const string& name) {
    return "Hello, " + name + "!";
}

// 多返回值：使用结构化绑定（C++17）
pair<int, double> stats(const vector<int>& data) {
    int sum = 0;
    for (int v : data) sum += v;
    double avg = static_cast<double>(sum) / data.size();
    return {sum, avg};
}

int main() {
    cout << greet("Alice") << endl;  // Hello, Alice!

    auto [sum, avg] = stats({1, 2, 3, 4, 5});
    cout << "sum=" << sum << " avg=" << avg << endl;  // sum=15 avg=3

    return 0;
}
```

### 2.2 返回引用

```cpp
#include <iostream>
#include <vector>
using namespace std;

// 返回容器元素的引用，允许修改
int& getElement(vector<int>& vec, size_t index) {
    return vec[index];
}

// 返回const引用（只读访问）
const string& findName(const vector<string>& names, size_t idx) {
    return names[idx];
}

int main() {
    vector<int> nums = {10, 20, 30};
    getElement(nums, 1) = 99;  // nums[1]变为99
    cout << nums[1] << endl;   // 99

    return 0;
}
```

**危险**：绝不能返回局部变量的引用！

```cpp
// 错误：返回局部变量引用（悬垂引用）
int& badFunc() {
    int local = 42;
    return local;  // 编译警告，运行时UB
}
```

### 2.3 RVO与NRVO

```cpp
#include <iostream>
#include <vector>
using namespace std;

class BigObject {
    vector<int> data;
public:
    BigObject() { cout << "构造\n"; }
    BigObject(const BigObject&) { cout << "拷贝\n"; }
    BigObject(BigObject&&) { cout << "移动\n"; }
};

// RVO（无名返回值优化）：C++17起保证
BigObject createRVO() {
    return BigObject();  // C++17保证无拷贝/移动
}

// NRVO（命名返回值优化）：非标准强制，但主流编译器支持
BigObject createNRVO() {
    BigObject obj;       // 命名对象
    return obj;          // MSVC/GCC/Clang通常优化掉拷贝
}

// NRVO可能被阻止的情况
BigObject createNoNRVO() {
    BigObject obj;
    if (someCondition) {
        return BigObject();  // 不同返回路径，可能阻止NRVO
    }
    return obj;
}

int main() {
    auto a = createRVO();   // 只输出：构造
    auto b = createNRVO();  // 通常只输出：构造
    return 0;
}
```

### 2.4 隐式移动与返回优化

```cpp
#include <vector>
using namespace std;

vector<int> createVector() {
    vector<int> result = {1, 2, 3, 4, 5};
    return result;  // C++11起自动隐式移动（即使不写std::move）
}

// 返回std::move(local)反而会阻止NRVO，不要这样做！
vector<int> badMove() {
    vector<int> result = {1, 2, 3};
    return std::move(result);  // 反模式：阻止NRVO
}

// auto返回类型推导（C++14）
auto makePair(int x) {
    return make_pair(x, x * x);
}
```

## 三、返回值与异常安全

```cpp
// C++17 std::optional — 可能失败的返回值
#include <optional>
#include <string>
optional<int> parseInt(const string& s) {
    try {
        return stoi(s);
    } catch (...) {
        return nullopt;
    }
}
```

## 四、注意事项与常见陷阱

1. **不要返回局部变量的引用或指针**：会导致悬垂引用（dangling reference）
2. **RVO在C++17起对prvalue保证**：无需手动std::move
3. **返回std::move(local_var)反而阻止NRVO**：编译器无法对被移动的变量应用NRVO
4. **auto返回类型推导（C++14）**：函数体中必须有return语句供推导
5. **引用返回的函数不能返回临时对象**：返回的引用必须指向调用后仍存在的对象
6. **协程（C++20）使用特殊返回类型**：需要co_await/co_yield支持
7. **返回值优化需要对象类型可拷贝/移动**：删除移动构造函数会阻止优化
