# 函数属性

## 一、概念说明

函数属性为编译器提供额外信息，用于优化或生成警告。C++标准定义了标准属性（C++11起），GCC/Clang和MSVC还各自支持编译器特定的扩展属性。

标准属性（可移植）：

| 属性 | 引入版本 | 用途 |
|------|---------|------|
| `[[nodiscard]]` | C++17 | 忽略返回值时产生警告 |
| `[[deprecated]]` | C++14 | 标记为弃用 |
| `[[maybe_unused]]` | C++17 | 抑制未使用警告 |
| `[[fallthrough]]` | C++17 | switch中标明穿透 |
| `[[noreturn]]` | C++11 | 函数永不返回 |
| `[[likely]]` / `[[unlikely]]` | C++20 | 分支概率提示 |

## 二、具体用法

### 2.1 [[nodiscard]] — 不可忽略的返回值

```cpp
#include <iostream>
#include <vector>
using namespace std;

// C++17：编译器会警告忽略返回值
[[nodiscard]] int computeImportantValue() {
    return 42;
}

// C++20：可带提示信息
[[nodiscard("检查返回的错误码")]] int connectDatabase() {
    return 0;  // 0表示成功
}

// C++20：可标注在构造函数上
struct [[nodiscard]] ErrorCode {
    int code;
    ErrorCode(int c) : code(c) {}
};

ErrorCode doWork() { return ErrorCode(0); }

int main() {
    computeImportantValue();  // 警告：忽略了nodiscard函数的返回值
    int result = computeImportantValue();  // 正确
    cout << result << endl;   // 42

    doWork();  // 警告：返回了nodiscard类型但未使用
    (void)doWork();  // 显式忽略，消除警告

    return 0;
}
```

### 2.2 [[deprecated]] — 标记弃用

```cpp
#include <iostream>
using namespace std;

// C++14：标记函数已弃用
[[deprecated("请使用 newFunction 替代")]]
void oldFunction() {
    cout << "旧实现\n";
}

void newFunction() {
    cout << "新实现\n";
}

// 也可标记类、变量等
[[deprecated("使用std::string_view替代")]]
typedef const char* CString;

int main() {
    // oldFunction();  // 警告: 'oldFunction' is deprecated
    newFunction();      // OK

    return 0;
}
```

### 2.3 [[maybe_unused]] — 抑制未使用警告

```cpp
#include <iostream>
using namespace std;

// 抑制函数未使用警告
[[maybe_unused]] void debugHelper() {
    cout << "调试辅助\n";
}

void process([[maybe_unused]] int debugLevel) {
    [[maybe_unused]] int temp = 42;  // 即使未使用也不警告
    cout << "处理中...\n";
}

int main() {
    process(1);
    return 0;
}
```

### 2.4 [[fallthrough]] — switch穿透

```cpp
#include <iostream>
using namespace std;

void processChar(char c) {
    switch (c) {
        case 'a':
        case 'A':
            cout << "字母A" << endl;
            [[fallthrough]];  // 明确表示故意穿透
        case 'e':
        case 'E':
            cout << "元音" << endl;
            break;
        default:
            cout << "其他" << endl;
    }
}
```

### 2.5 [[noreturn]] — 永不返回

```cpp
#include <iostream>
#include <cstdlib>
using namespace std;

// 函数永不返回（如抛异常、终止程序）
[[noreturn]] void fatal(const char* msg) {
    cerr << "致命错误: " << msg << endl;
    abort();  // 终止程序
}

// 编译器知道fatal不返回，不会警告缺少return
int divide(int a, int b) {
    if (b == 0) fatal("除以零");
    return a / b;
}
```

### 2.6 [[likely]] / [[unlikely]]（C++20）

```cpp
#include <iostream>
using namespace std;

int process(int* data, size_t len) {
    int sum = 0;
    for (size_t i = 0; i < len; ++i) {
        if (data[i] > 0) [[likely]] {     // 提示编译器此分支更可能
            sum += data[i];
        } else [[unlikely]] {              // 提示编译器此分支不太可能
            sum -= data[i];
        }
    }
    return sum;
}
```

### 2.7 GCC/Clang编译器扩展属性

```cpp
// 强制内联
__attribute__((always_inline)) inline void fastFunc() { }

// 禁用内联
__attribute__((noinline)) void largeFunc() { }

// 纯函数（无副作用，结果仅依赖参数）
__attribute__((pure)) int square(int x) { return x * x; }

// 常量函数（结果不依赖任何内存状态）
__attribute__((const)) int add(int a, int b) { return a + b; }

// 函数格式化检查（类似printf）
__attribute__((format(printf, 1, 2)))
void myPrintf(const char* fmt, ...);
```

## 三、注意事项与常见陷阱

1. **标准属性具有可移植性**：`[[...]]`语法所有C++17编译器都支持
2. **编译器扩展属性不具备可移植性**：GCC的`__attribute__`在MSVC中不可用
3. **属性不影响程序逻辑**：只影响编译器行为（警告、优化提示）
4. **属性可以组合使用**：`[[nodiscard, deprecated("reason")]]`
5. **[[fallthrough]]必须放在switch case的最后**：作为单独语句，后面跟分号
6. **C++20的[[likely]]/[[unlikely]]可改善分支预测**：帮助编译器优化指令排列
7. **属性标注在声明上即可**：不需要在定义处重复标注
