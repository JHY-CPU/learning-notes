# 函数属性

## 一、概念说明

函数属性为编译器提供额外信息，用于优化或生成警告。C++标准定义了`[[nodiscard]]`、`[[deprecated]]`、`[[maybe_unused]]`等属性，GCC/Clang还支持`__attribute__`扩展。

## 二、具体用法

### 2.1 [[nodiscard]]（C++17）

```cpp
// 编译器会警告：忽略返回值
[[nodiscard]] int computeImportantValue() {
    return 42;
}

void test() {
    computeImportantValue();  // 警告：忽略了nodiscard函数的返回值
    int result = computeImportantValue();  // 正确
    std::cout << result << std::endl;      // 输出: 42
}
```

### 2.2 [[deprecated]]（C++14）

```cpp
// 标记函数已弃用，使用时产生编译警告
[[deprecated("请使用 newFunction 替代")]]
void oldFunction() {
    std::cout << "旧实现\n";
}

void newFunction() {
    std::cout << "新实现\n";
}

// oldFunction();  // 警告: 'oldFunction' is deprecated: 请使用 newFunction 替代
```

### 2.3 [[maybe_unused]]（C++17）

```cpp
// 抑制"未使用变量/函数"的编译警告
[[maybe_unused]] void debugHelper() {
    std::cout << "调试辅助函数\n";
}

void process() {
    [[maybe_unused]] int temp = compute();
    // 即使temp未使用也不会产生警告
}
```

### 2.4 GCC/Clang属性

```cpp
// 强制内联
__attribute__((always_inline)) inline void fastFunc() { }

// 禁用内联
__attribute__((noinline)) void largeFunc() { }

// 函数格式化检查（类似printf）
__attribute__((format(printf, 1, 2)))
void myPrintf(const char* fmt, ...);
```

## 三、注意事项与常见陷阱

- `[[nodiscard]]`在C++20可用于构造函数和枚举值
- 标准属性具有可移植性，编译器扩展属性不具备
- `[[fallthrough]]`用于switch语句中标明故意穿透
- 属性可以组合使用：`[[nodiscard, deprecated]]`
- 属性不影响程序逻辑，只影响编译器行为
