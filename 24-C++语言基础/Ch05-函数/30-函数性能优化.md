# 函数性能优化

## 一、概念说明

函数调用有开销（参数传递、栈帧管理、返回值处理）。C++提供多种机制减少或消除这些开销：内联、传递引用避免拷贝、返回值优化（RVO/NRVO）、移动语义等。

## 二、具体用法

### 2.1 避免不必要的拷贝

```cpp
// 低效：按值传递大对象，产生拷贝
void process(std::string name) {
    std::cout << name << std::endl;
}

// 高效：const引用传递，无拷贝
void processOptimized(const std::string& name) {
    std::cout << name << std::endl;
}
```

### 2.2 移动语义减少拷贝

```cpp
// 使用移动语义转移资源所有权
std::vector<int> createLargeVector() {
    std::vector<int> result(1000000, 42);
    return result;  // 编译器可能优化（RVO），否则移动
}

auto vec = createLargeVector();  // 无拷贝
std::cout << vec.size() << std::endl;
// 输出: 1000000
```

### 2.3 返回值优化（RVO/NRVO）

```cpp
class BigData {
    std::vector<int> data;
public:
    BigData() : data(1000) {}
    BigData(const BigData&) { std::cout << "拷贝（昂贵）\n"; }
};

// NRVO：编译器直接在调用者空间构造，无拷贝
BigData create() {
    BigData obj;
    return obj;  // 无拷贝（NRVO）
}
```

### 2.4 编译期计算

```cpp
// constexpr函数在编译期求值，零运行时开销
constexpr int fibonacci(int n) {
    if (n <= 1) return n;
    return fibonacci(n-1) + fibonacci(n-2);
}

constexpr int fib10 = fibonacci(10);  // 编译期计算为55
std::cout << fib10 << std::endl;       // 输出: 55，无运行时计算
```

### 2.5 避免虚函数开销

```cpp
// CRTP实现编译期多态，避免虚函数表查找
template <typename Derived>
class Base {
public:
    void interface() {
        static_cast<Derived*>(this)->implementation();
    }
};

class Derived : public Base<Derived> {
public:
    void implementation() {
        std::cout << "CRTP实现\n";
    }
};
```

## 三、注意事项与常见陷阱

- 不要过早优化，先测量（profile）再优化
- `inline`是建议，编译器自行决定是否内联
- 大对象（>sizeof指针）优先传const引用
- 小对象（int, double等）值传递更高效
- 编译器优化（-O2/-O3）通常比手动优化更可靠
