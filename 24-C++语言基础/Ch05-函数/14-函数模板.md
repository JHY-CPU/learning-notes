# 函数模板

## 一、概念说明

函数模板允许编写**与类型无关**的通用函数。编译器根据调用时的实参类型**自动推导**模板参数，生成对应类型的函数实例。这实现了泛型编程的核心思想：一次编写，多类型复用。

## 二、具体用法

### 2.1 基本函数模板

```cpp
template <typename T>
T maximum(T a, T b) {
    return (a > b) ? a : b;
}

std::cout << maximum(10, 20) << std::endl;       // 输出: 20 (int)
std::cout << maximum(3.14, 2.72) << std::endl;   // 输出: 3.14 (double)
std::cout << maximum('a', 'z') << std::endl;     // 输出: z (char)
```

### 2.2 多模板参数

```cpp
template <typename T, typename U>
auto add(T a, U b) -> decltype(a + b) {
    return a + b;
}

std::cout << add(3, 4.5) << std::endl;  // 输出: 7.5 (int + double)
```

### 2.3 显式实例化

```cpp
template <typename T>
void print(T value) {
    std::cout << value << std::endl;
}

// 编译器自动推导
print(42);          // T = int

// 显式指定类型
print<double>(42);  // T = double, 输出: 42

// 显式实例化定义（避免多翻译单元中重复实例化）
template void print<int>(int);  // 强制实例化int版本
```

### 2.4 模板特化

```cpp
template <typename T>
bool isEqual(T a, T b) {
    return a == b;
}

// 针对double的特化（处理浮点精度问题）
template <>
bool isEqual<double>(double a, double b) {
    return std::abs(a - b) < 1e-9;
}

std::cout << isEqual(0.1 + 0.2, 0.3) << std::endl;  // 输出: 1
```

## 三、注意事项与常见陷阱

- 模板定义通常放在**头文件**中，因为编译器需要完整定义来实例化
- 无法推导时需显式指定模板参数
- 模板只实例化被使用到的类型，未使用的类型不会报错
- 模板错误信息通常很冗长，C++20的`concept`可以改善
- 不支持的类型操作会在实例化时而非定义时报错
