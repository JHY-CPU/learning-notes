# 函数模板

## 一、概念说明

函数模板（C++11标准 §14.1）允许编写**与类型无关**的通用函数。编译器根据调用时的实参类型**自动推导**模板参数，生成对应类型的函数实例（模板实例化）。这实现了泛型编程的核心思想：一次编写，多类型复用。

C++20引入Concepts，可以对模板参数施加约束，提供更清晰的错误信息。

## 二、具体用法

### 2.1 基本函数模板

```cpp
#include <iostream>
using namespace std;

template <typename T>
T maximum(T a, T b) {
    return (a > b) ? a : b;
}

int main() {
    cout << maximum(10, 20) << endl;       // int版本，20
    cout << maximum(3.14, 2.72) << endl;   // double版本，3.14
    cout << maximum('a', 'z') << endl;     // char版本，z

    // 显式指定模板参数
    cout << maximum<double>(3, 4.5) << endl;  // 4.5（3被转为double）

    return 0;
}
```

### 2.2 多模板参数

```cpp
#include <iostream>
#include <string>
using namespace std;

// 两个不同类型的参数，C++14 auto返回类型
template <typename T, typename U>
auto add(T a, U b) -> decltype(a + b) {
    return a + b;
}

// C++17起可简写为
template <typename T, typename U>
auto multiply(T a, U b) {
    return a * b;
}

int main() {
    cout << add(3, 4.5) << endl;          // 7.5 (int + double)
    cout << multiply(3, 4.5) << endl;     // 13.5
    return 0;
}
```

### 2.3 模板特化

```cpp
#include <iostream>
#include <cmath>
using namespace std;

// 通用模板
template <typename T>
bool isEqual(T a, T b) {
    return a == b;
}

// 针对double的特化（处理浮点精度）
template <>
bool isEqual<double>(double a, double b) {
    return abs(a - b) < 1e-9;
}

// 针对C字符串的特化
template <>
bool isEqual<const char*>(const char* a, const char* b) {
    return strcmp(a, b) == 0;
}

int main() {
    cout << isEqual(0.1 + 0.2, 0.3) << endl;  // 1（特化处理精度）
    cout << isEqual(42, 42) << endl;           // 1

    return 0;
}
```

### 2.4 C++20 Concepts约束

```cpp
#include <iostream>
#include <concepts>
#include <string>
using namespace std;

// C++20：使用concept约束模板参数
template <integral T>
T gcd(T a, T b) {
    while (b) { a %= b; swap(a, b); }
    return a;
}

// 多个concept约束
template <typename T>
    requires integral<T> || floating_point<T>
T square(T x) {
    return x * x;
}

// 自定义concept
template <typename T>
concept Printable = requires(T t, ostream& os) {
    { os << t } -> same_as<ostream&>;
};

template <Printable T>
void printValue(const T& val) {
    cout << val << endl;
}

int main() {
    cout << gcd(12, 8) << endl;    // 4
    cout << square(5.5) << endl;   // 30.25
    printValue(42);                // 42

    // gcd(3.14, 2.72);  // 编译错误：double不满足integral

    return 0;
}
```

### 2.5 非类型模板参数

```cpp
#include <iostream>
#include <array>
using namespace std;

// 非类型模板参数：值而非类型
template <int N>
constexpr int factorial() {
    int result = 1;
    for (int i = 2; i <= N; ++i) result *= i;
    return result;
}

// 数组大小作为模板参数
template <typename T, size_t N>
void printArray(const T (&arr)[N]) {
    for (size_t i = 0; i < N; ++i)
        cout << arr[i] << " ";
    cout << endl;
}

int main() {
    constexpr int f5 = factorial<5>();  // 编译期计算：120
    cout << "5! = " << f5 << endl;

    int nums[] = {1, 2, 3, 4, 5};
    printArray(nums);  // N自动推导为5

    return 0;
}
```

## 三、注意事项与常见陷阱

1. **模板定义通常放在头文件中**：编译器需要完整定义来实例化，放源文件会导致链接错误
2. **无法推导时需显式指定模板参数**：如 `max<int>(3, 4.5)`
3. **模板只实例化被使用到的类型**：未使用类型的代码不会编译，不会报错
4. **模板错误信息通常很冗长**：C++20的concept可以改善错误信息
5. **不支持的类型操作在实例化时报错**：不是定义时报错（SFINAE语义）
6. **隐式实例化 vs 显式实例化**：显式实例化可避免多翻译单元中重复实例化
7. **函数模板不支持偏特化**：但可以通过重载实现类似效果
