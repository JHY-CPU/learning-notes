# 函数重载决议规则

## 一、概念说明

**重载决议**（Overload Resolution）是编译器从一组同名候选函数中选择最佳匹配的过程。编译器按以下步骤进行：确定**候选函数**（同名且可见）、筛选**可行函数**（参数数量匹配且可转换）、评选**最佳匹配**。

匹配优先级：精确匹配 > 提升转换 > 标准转换 > 用户定义转换 > 省略号匹配。

## 二、具体用法

### 2.1 精确匹配优先

```cpp
void func(int)    { std::cout << "int\n"; }
void func(double) { std::cout << "double\n"; }
void func(const char*) { std::cout << "const char*\n"; }

func(42);     // 输出: int （精确匹配）
func(3.14);   // 输出: double （精确匹配）
func("hi");   // 输出: const char* （精确匹配）
```

### 2.2 类型提升 vs 标准转换

```cpp
void func(int)    { std::cout << "int (提升)\n"; }
void func(double) { std::cout << "double (标准转换)\n"; }

char c = 'A';
func(c);  // 输出: int （char提升为int，比转double优先级高）
```

### 2.3 const与非const

```cpp
void process(int& x)       { std::cout << "非const左值引用\n"; }
void process(const int& x) { std::cout << "const左值引用\n"; }
void process(int&& x)      { std::cout << "右值引用\n"; }

int x = 10;
const int y = 20;

process(x);   // 输出: 非const左值引用
process(y);   // 输出: const左值引用
process(42);  // 输出: 右值引用
```

### 2.4 歧义（Ambiguous）调用

```cpp
void func(long)  { }
void func(float) { }

// func(42);  // 编译错误：int可转long和float，同等优先级

// 解决方案：显式转换
func(static_cast<long>(42));   // OK
func(static_cast<float>(42));  // OK
```

## 三、注意事项与常见陷阱

- 整数字面量默认`int`，浮点默认`double`，注意匹配优先级
- 函数模板和非模板函数同时可行时，优先选择非模板（其他条件相同）
- 默认参数不参与重载决议——两个函数仅默认参数不同不构成重载
- `explicit`构造函数阻止隐式转换参与重载决议
- 派生类到基类的转换优先级低于标准转换
