# 默认参数

## 一、概念说明

默认参数允许函数在调用时省略部分实参，编译器自动填入预设的默认值。这在需要向后兼容或提供便捷API时非常有用。

默认参数必须从**右向左**连续指定，中间不能跳过。

## 二、具体用法

### 2.1 基本用法

```cpp
void printMessage(const std::string& msg,
                  const std::string& prefix = "[INFO]",
                  bool newline = true) {
    std::cout << prefix << " " << msg;
    if (newline) std::cout << std::endl;
}

printMessage("Hello");              // [INFO] Hello
printMessage("Error", "[ERROR]");   // [ERROR] Error
printMessage("Debug", "[DBG]", false); // [DBG] Debug (无换行)
```

### 2.2 声明与定义分离

```cpp
// 头文件中指定默认值（只需在声明处指定）
void connect(const std::string& host,
             int port = 8080,
             int timeout = 30);

// 源文件中不再指定默认值
void connect(const std::string& host, int port, int timeout) {
    std::cout << "Connecting to " << host << ":" << port
              << " timeout=" << timeout << "s" << std::endl;
}
```

### 2.3 默认参数与重载

```cpp
// 用默认参数替代简单的重载
void log(const std::string& msg, int level = 1);
// 等价于同时提供 log(string) 和 log(string, int) 两个版本
```

## 三、注意事项与常见陷阱

- 默认值只能在**声明**处指定，不能在定义处重复指定
- 参数必须从右向左连续默认，`void f(int a, int b=1, int c)` 是错误的
- 默认参数不参与重载决议，可能引起歧义
- 默认参数表达式在**调用点**求值，不是定义点
- 虚函数的默认参数根据静态类型决定，可能非预期
