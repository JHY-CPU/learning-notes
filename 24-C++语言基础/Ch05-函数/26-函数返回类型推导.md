# 函数返回类型推导

## 一、概念说明

C++14引入了`auto`返回类型推导，允许编译器自动推断函数的返回类型。此外，C++11的**尾置返回类型**（trailing return type）提供了一种在参数列表后声明返回类型的语法，特别适用于模板和复杂类型表达式。

## 二、具体用法

### 2.1 auto返回类型（C++14）

```cpp
// 编译器自动推导返回类型为int
auto add(int a, int b) {
    return a + b;
}

// 所有return语句必须推导出相同类型
auto getValue(bool flag) {
    if (flag) return 42;      // int
    else return 3.14;         // 编译错误：int vs double
}

std::cout << add(3, 5) << std::endl;  // 输出: 8
```

### 2.2 尾置返回类型（C++11）

```cpp
// 语法：auto 函数名(参数) -> 返回类型
auto multiply(int a, int b) -> int {
    return a * b;
}

// 在模板中特别有用：返回类型依赖于参数
template <typename T, typename U>
auto add(T a, U b) -> decltype(a + b) {
    return a + b;
}

std::cout << add(3, 4.5) << std::endl;  // 输出: 7.5
```

### 2.3 decltype(auto)（C++14）

```cpp
// auto会丢弃引用，decltype(auto)保留精确类型
template <typename Container, typename Index>
decltype(auto) getElement(Container&& c, Index i) {
    return std::forward<Container>(c)[i];
}

std::vector<int> vec = {1, 2, 3};
getElement(vec, 1) = 99;  // 返回int&，可修改
std::cout << vec[1] << std::endl;  // 输出: 99
```

### 2.4 递归中的auto返回

```cpp
// C++14：递归函数的auto需要至少一个return有显式类型
auto factorial(int n) -> int {
    if (n <= 1) return 1;
    return n * factorial(n - 1);
}

std::cout << factorial(5) << std::endl;  // 输出: 120
```

## 三、注意事项与常见陷阱

- `auto`返回类型要求所有return语句推导类型一致
- `auto`会退化（decay）：去掉引用和cv限定符
- `decltype(auto)`保留精确类型（包括引用和cv限定）
- 头文件中声明的函数使用auto时需注意定义可见性
- 返回auto的函数不能用于前向声明（编译器需要看到定义）
