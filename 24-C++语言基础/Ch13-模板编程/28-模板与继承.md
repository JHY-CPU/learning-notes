# 模板与继承

## 一、概念说明

模板与继承结合产生了强大的设计模式，包括CRTP（Curiously Recurring Template Pattern，奇异递归模板模式）和模板基类。CRTP通过将派生类作为模板参数传给基类，实现了编译期多态。

## 二、具体用法

### 2.1 CRTP基本模式

```cpp
// 基类模板：派生类作为模板参数
template <typename Derived>
class Counter {
    static int count;
public:
    Counter() { ++count; }
    Counter(const Counter&) { ++count; }
    ~Counter() { --count; }
    static int getCount() { return count; }
};

template <typename Derived>
int Counter<Derived>::count = 0;

// 派生类将自己传给基类
class Widget : public Counter<Widget> {};
class Gadget : public Counter<Gadget> {};

int main() {
    Widget w1, w2;
    Gadget g1;
    std::cout << "Widget: " << Widget::getCount() << std::endl;  // 2
    std::cout << "Gadget: " << Gadget::getCount() << std::endl;  // 1
}
```

### 2.2 CRTP实现静态多态

```cpp
template <typename Derived>
class Shape {
public:
    void draw() const {
        // 编译期调用派生类的实现
        static_cast<const Derived*>(this)->draw_impl();
    }

    double area() const {
        return static_cast<const Derived*>(this)->area_impl();
    }
};

class Circle : public Shape<Circle> {
    double radius;
public:
    Circle(double r) : radius(r) {}
    void draw_impl() const {
        std::cout << "绘制圆形, 半径=" << radius << std::endl;
    }
    double area_impl() const { return 3.14159 * radius * radius; }
};

class Rectangle : public Shape<Rectangle> {
    double w, h;
public:
    Rectangle(double w, double h) : w(w), h(h) {}
    void draw_impl() const {
        std::cout << "绘制矩形, " << w << "x" << h << std::endl;
    }
    double area_impl() const { return w * h; }
};

int main() {
    Circle c(5.0);
    Rectangle r(3.0, 4.0);
    c.draw();  // 绘制圆形, 半径=5
    r.draw();  // 绘制矩形, 3x4
    std::cout << "圆面积: " << c.area() << std::endl;     // 78.5398
    std::cout << "矩形面积: " << r.area() << std::endl;   // 12
}
```

### 2.3 CRTP实现单例模式

```cpp
template <typename T>
class Singleton {
protected:
    Singleton() = default;
public:
    Singleton(const Singleton&) = delete;
    Singleton& operator=(const Singleton&) = delete;

    static T& instance() {
        static T inst;
        return inst;
    }
};

class Config : public Singleton<Config> {
    friend class Singleton<Config>;
    Config() : value(0) {}
public:
    int value;
};

int main() {
    Config::instance().value = 42;
    std::cout << Config::instance().value << std::endl;  // 42
}
```

### 2.4 模板基类

```cpp
template <typename T>
class ContainerBase {
protected:
    std::vector<T> data;
public:
    void add(const T& val) { data.push_back(val); }
    std::size_t size() const { return data.size(); }
    T& operator[](std::size_t i) { return data[i]; }
};

template <typename T>
class SortedContainer : public ContainerBase<T> {
public:
    void add(const T& val) {
        this->data.push_back(val);  // 注意：需要this->
        std::sort(this->data.begin(), this->data.end());
    }
};

int main() {
    SortedContainer<int> sc;
    sc.add(30); sc.add(10); sc.add(20);
    for (std::size_t i = 0; i < sc.size(); ++i)
        std::cout << sc[i] << " ";  // 10 20 30
}
```

## 三、注意事项与常见陷阱

- CRTP中派生类必须完整定义后才能使用（不能前向声明）
- 模板基类中的名称在派生类中不可见（需要`this->`或`using`）
- CRTP提供编译期多态，没有虚函数开销
- 每个CRTP实例化都是独立的基类，不能多态地持有不同派生类
- CRTP用于：计数器、单例、静态接口、mixin功能
- 虚函数+模板结合有限制（虚函数不能是模板）
