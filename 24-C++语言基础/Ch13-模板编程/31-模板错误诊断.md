# 模板错误诊断

## 一、概念说明

模板错误信息通常非常冗长且难以理解，因为错误可能发生在模板定义、实例化、或深层嵌套的模板调用中。C++20通过Concepts提供了更友好的错误信息，而`static_assert`可以在编译时主动检测和报告错误。

## 二、具体用法

### 2.1 static_assert编译期断言

```cpp
template <typename T>
T safe_sqrt(T val) {
    static_assert(std::is_arithmetic_v<T>, "safe_sqrt需要算术类型");
    static_assert(sizeof(T) >= 4, "safe_sqrt需要至少32位类型");
    return std::sqrt(val);
}

int main() {
    std::cout << safe_sqrt(4.0) << std::endl;   // 2
    std::cout << safe_sqrt(25) << std::endl;    // 5
    // safe_sqrt("hello");  // 编译错误：safe_sqrt需要算术类型
}
```

### 2.2 Concepts改善错误信息

```cpp
// 无concept：错误信息冗长
template <typename T>
auto bad_print(const T& val) -> decltype(std::cout << val) {
    return std::cout << val;
}

// 有concept：错误信息清晰
template <typename T>
    requires requires(std::ostream& os, T v) { os << v; }
void good_print(const T& val) {
    std::cout << val << std::endl;
}

int main() {
    good_print(42);        // OK
    good_print("hello");   // OK
    // good_print(std::vector<int>{});  // 清晰错误：不满足约束
}
```

### 2.3 自定义错误消息

```cpp
template <typename T>
class NumericArray {
    std::vector<T> data;
public:
    NumericArray(std::initializer_list<T> init) : data(init) {
        static_assert(std::is_arithmetic_v<T>,
            "NumericArray只能用于算术类型(int, float, double等)");
    }

    T sum() const {
        static_assert(!std::is_same_v<T, bool>,
            "bool类型不支持求和操作");
        T result = 0;
        for (const auto& x : data) result += x;
        return result;
    }
};

int main() {
    NumericArray<int> arr{1, 2, 3, 4, 5};
    std::cout << arr.sum() << std::endl;  // 15
    // NumericArray<std::string> bad{"a", "b"};  // 编译错误
}
```

### 2.4 检测模板参数的有效性

```cpp
// 编译时检查类型是否可哈希
template <typename T>
concept Hashable = requires(T a) {
    { std::hash<T>{}(a) } -> std::convertible_to<std::size_t>;
};

template <Hashable T>
void process_hash(const T& val) {
    std::cout << "哈希值: " << std::hash<T>{}(val) << std::endl;
}

int main() {
    process_hash(42);              // 哈希值: 42
    process_hash(std::string("hi")); // 哈希值: ...
}
```

## 三、注意事项与常见陷阱

- `static_assert`在编译时求值，条件必须是常量表达式
- C++17起`static_assert`可以不写消息（`static_assert(condition)`）
- Concepts提供的错误信息远优于传统SFINAE
- 使用`requires`表达式可以精确描述约束条件
- `std::is_arithmetic_v`、`std::is_integral_v`等是常用的类型检查
- 对于库开发者，良好的错误消息是API设计的重要部分
- 深层嵌套模板的错误定位仍然困难，需要经验
