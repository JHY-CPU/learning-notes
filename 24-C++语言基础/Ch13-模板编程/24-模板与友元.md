# 模板与友元

## 一、概念说明

模板与友元（Friend）的结合允许一个模板类或函数成为另一个类的友元，从而访问其私有成员。模板友元的声明语法比普通友元更复杂，需要正确处理模板参数关系。

## 二、具体用法

### 2.1 模板类的友元函数

```cpp
template <typename T>
class Box {
    T value;
public:
    Box(T v) : value(v) {}

    // 友元函数模板：所有Box<T>实例都能访问
    template <typename U>
    friend std::ostream& operator<<(std::ostream& os, const Box<U>& box);
};

template <typename T>
std::ostream& operator<<(std::ostream& os, const Box<T>& box) {
    return os << "[" << box.value << "]";
}

int main() {
    Box<int> b1(42);
    Box<std::string> b2("hello");
    std::cout << b1 << std::endl;  // [42]
    std::cout << b2 << std::endl;  // [hello]
}
```

### 2.2 特定模板实例为友元

```cpp
template <typename T>
class BankAccount {
    T balance;
public:
    BankAccount(T b) : balance(b) {}

    // 只让相同类型的Audit成为友元
    template <typename U>
    friend class Audit;
};

template <typename T>
class Audit {
public:
    void check(const BankAccount<T>& acc) {
        std::cout << "余额: " << acc.balance << std::endl;
    }
};

int main() {
    BankAccount<double> acc(1000.50);
    Audit<double> auditor;
    auditor.check(acc);  // 余额: 1000.5
}
```

### 2.3 非模板类中的模板友元

```cpp
class Logger {
    int log_level;
    Logger() : log_level(0) {}

    // 让Config<T>可以访问私有成员
    template <typename T>
    friend class Config;
};

template <typename T>
class Config {
public:
    void configure(Logger& logger, int level) {
        logger.log_level = level;  // 访问私有成员
        std::cout << "日志级别设为: " << level << std::endl;
    }
};

int main() {
    Logger logger;
    Config<int> cfg;
    cfg.configure(logger, 3);  // 日志级别设为: 3
}
```

### 2.4 友元函数的类内定义

```cpp
template <typename T>
class Pair {
    T first, second;
public:
    Pair(T a, T b) : first(a), second(b) {}

    // 类内定义的友元模板函数（通过ADL查找）
    template <typename U>
    friend bool operator==(const Pair<U>& lhs, const Pair<U>& rhs) {
        return lhs.first == rhs.first && lhs.second == rhs.second;
    }
};

int main() {
    Pair<int> p1(1, 2), p2(1, 2);
    std::cout << std::boolalpha << (p1 == p2) << std::endl;  // true
}
```

## 三、注意事项与常见陷阱

- 模板友元声明中`template`关键字不可省略
- 友元关系不能传递：A友元B，B友元C，A和C不一定有访问权限
- 类内定义的友元函数只能通过ADL访问
- 友元模板会增加编译依赖，谨慎使用
- 如果友元函数访问私有成员，其定义必须在友元声明之后
- `friend class`和`friend struct`语义相同
