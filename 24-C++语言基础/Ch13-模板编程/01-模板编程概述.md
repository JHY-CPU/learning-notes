# 模板编程概述

## 一、概念说明

模板（Template）是C++泛型编程的核心机制，允许编写与类型无关的通用代码。模板在**编译期**由编译器根据实际类型生成具体代码，这一过程称为**实例化**（Instantiation）。模板实现了"编译期多态"，与运行时多态（虚函数）互补。

### 1.1 泛型编程思想
泛型编程的核心是将算法与数据类型分离，编写一次即可适用于多种类型。STL就是泛型编程的经典范例——容器、迭代器、算法三者通过模板无缝协作。

### 1.2 编译期多态 vs 运行时多态

| 特性 | 编译期多态（模板） | 运行时多态（虚函数） |
|------|-------------------|---------------------|
| 决定时机 | 编译时 | 运行时 |
| 性能 | 无虚函数开销 | 有虚表查找开销 |
| 代码体积 | 可能膨胀 | 共享同一份代码 |
| 灵活性 | 类型必须在编译期确定 | 运行时动态绑定 |

## 二、模板分类

C++模板主要分为三大类：

```cpp
// 1. 函数模板
template <typename T>
T maximum(T a, T b) {
    return (a > b) ? a : b;
}

// 2. 类模板
template <typename T>
class Stack {
    std::vector<T> data;
public:
    void push(const T& val) { data.push_back(val); }
    T top() const { return data.back(); }
};

// 3. 变量模板（C++14）
template <typename T>
constexpr T pi = T(3.14159265358979323846);
```

```cpp
int main() {
    // 函数模板：类型推导
    std::cout << maximum(3, 7) << std::endl;        // 7
    std::cout << maximum(3.14, 2.72) << std::endl;   // 3.14

    // 类模板：显式指定类型
    Stack<int> intStack;
    intStack.push(42);
    std::cout << intStack.top() << std::endl;         // 42

    // 变量模板
    std::cout << pi<double> << std::endl;             // 3.14159
    std::cout << pi<float> << std::endl;              // 3.14159 (精度不同)
}
```

## 三、模板的核心价值

1. **代码复用**：一套逻辑适用于多种类型
2. **类型安全**：编译期检查，优于宏
3. **零运行时开销**：编译期生成最优代码
4. **STL基础**：标准库大量使用模板

## 四、注意事项与常见陷阱

- 模板的定义通常必须放在头文件中（包含模型）
- 每个不同的模板参数组合都会生成一份独立代码，可能导致代码膨胀
- 模板错误信息通常非常冗长，C++20 Concepts可改善此问题
- `typename`和`class`在模板参数声明中基本等价，但`typename`更推荐
- 模板不支持分离编译（.h和.cpp分开），这是与普通函数的重要区别
