# 显式实例化

## 一、概念说明

显式实例化（Explicit Instantiation）允许程序员手动控制模板何时、何地生成具体代码。通过显式实例化声明（`extern template`）和定义，可以减少编译时间、控制代码膨胀、并改善链接效率。

## 二、具体用法

### 2.1 显式实例化定义

```cpp
// widget.h
template <typename T>
class Widget {
    T value;
public:
    Widget(T v) : value(v) {}
    void show() const { std::cout << "Widget: " << value << std::endl; }
};

// widget.cpp — 显式实例化定义
// 强制编译器在此处生成int和double版本
template class Widget<int>;
template class Widget<double>;
// 此时Widget<int>和Widget<double>的所有成员函数都被实例化
```

### 2.2 extern template（显式实例化声明）

```cpp
// header.h
template <typename T>
void heavy_computation(T val) {
    // 复杂的模板实现...
    std::cout << "计算中: " << val << std::endl;
}

// main.cpp
// extern template 告诉编译器：不要在此文件中实例化，其他地方已有定义
extern template void heavy_computation<int>(int);
extern template void heavy_computation<double>(double);

int main() {
    heavy_computation(42);   // 使用已在别处实例化的版本
    heavy_computation(3.14); // 使用已在别处实例化的版本
    // heavy_computation(1.5f); // 会在此处实例化float版本（没有extern声明）
}
```

### 2.3 减少编译时间

```cpp
// vector_utils.h
template <typename T>
void sort_vector(std::vector<T>& v) {
    std::sort(v.begin(), v.end());
}

template <typename T>
void print_vector(const std::vector<T>& v) {
    for (const auto& x : v) std::cout << x << " ";
    std::cout << std::endl;
}

// vector_utils.cpp
// 在一个编译单元中集中实例化常用类型
template void sort_vector<int>(std::vector<int>&);
template void sort_vector<double>(std::vector<double>&);
template void print_vector<int>(const std::vector<int>&);
template void print_vector<double>(const std::vector<double>&);

// 使用文件中
// extern template void sort_vector<int>(std::vector<int>&);
// extern template void print_vector<int>(const std::vector<int>&);
```

### 2.4 函数模板显式实例化

```cpp
// 只实例化函数模板的特定版本
template <typename T>
T max_of(T a, T b) {
    return (a > b) ? a : b;
}

// 显式实例化定义
template int max_of<int>(int, int);
template double max_of<double>(double, double);

int main() {
    std::cout << max_of(3, 7) << std::endl;    // 7
    std::cout << max_of(2.5, 1.8) << std::endl; // 2.5
    // max_of(3LL, 7LL); // 会隐式实例化long long版本
}
```

## 三、注意事项与常见陷阱

- `extern template`只是声明，仍需在某处有显式实例化定义
- 显式实例化适用于已知会频繁使用的类型组合（如`vector<int>`）
- STL大量使用`extern template`减少包含开销
- 显式实例化类模板会实例化所有成员函数（包括未使用的）
- 函数模板的显式实例化只生成指定的函数
- 在动态库/共享库中，显式实例化可以控制符号的导出
- C++11之前没有`extern template`，需要手动管理
