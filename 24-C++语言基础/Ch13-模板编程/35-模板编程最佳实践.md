# 模板编程最佳实践

## 一、设计原则

模板编程应遵循"何时用模板、如何设计模板、如何避免常见陷阱"的原则。过度使用模板会导致编译时间膨胀、错误信息晦涩和代码维护困难。

## 二、具体用法

### 2.1 何时使用模板

```cpp
// 适合模板的场景：
// 1. 容器类（数据结构与类型解耦）
template <typename T>
class Stack { /* ... */ };

// 2. 通用算法（适用于多种类型）
template <typename Iter>
void my_sort(Iter begin, Iter end) { /* ... */ };

// 3. 编译期计算
template <int N>
struct Factorial { static constexpr int value = N * Factorial<N-1>::value; };

// 不适合模板的场景：
// - 仅处理两三种固定类型（用重载即可）
// - 简单的辅助函数（inline函数更好）
// - 需要运行时多态的情况（用虚函数）
```

### 2.2 模板设计原则

```cpp
// 原则1：最小化模板参数
// 坏：太多参数
template <typename T, typename Alloc, typename Compare, typename Policy>
class BadContainer {};

// 好：合理使用默认参数
template <typename T,
          typename Alloc = std::allocator<T>,
          typename Compare = std::less<T>>
class GoodContainer {};

// 原则2：使用concept约束（C++20）
template <typename T>
    requires std::integral<T> || std::floating_point<T>
T safe_divide(T a, T b) {
    return b ? a / b : 0;
}

// 原则3：提供清晰的错误消息
template <typename T>
void process(const T& val) {
    static_assert(std::is_arithmetic_v<T>, "process()需要算术类型");
    std::cout << val << std::endl;
}
```

### 2.3 代码组织

```cpp
// 好的实践：模板定义放在头文件
// my_template.h
#ifndef MY_TEMPLATE_H
#define MY_TEMPLATE_H

    std::vector<T> data;

// 复杂实现可以分离到.tpp文件
// #include "my_template.tpp"

// 减少编译时间：extern template
// 在常用类型的源文件中显式实例化

// extern template class MyContainer<double>;
```

### 2.4 命名约定

```cpp
// 模板参数命名约定
template <typename T>        // 通用类型
template <typename Container> // 容器类型
template <typename Iter>      // 迭代器类型
template <typename Func>      // 函数/可调用对象
template <int N>              // 整型常量
template <std::size_t Size>   // 大小常量

// 使用描述性名称
template <typename InputIter, typename OutputIter, typename BinaryOp>
OutputIter transform(InputIter first, InputIter last,
                     OutputIter result, BinaryOp op) {
    while (first != last) {
        *result++ = op(*first++);
    }
    return result;
}
```

### 2.5 测试与调试

```cpp
// 针对不同类型的测试
template <typename T>
class StackTest : public ::testing::Test {};

using TestTypes = ::testing::Types<int, double, std::string>;
TYPED_TEST_SUITE(StackTest, TestTypes);

// 使用static_assert确保编译期条件
template <typename T>
class Buffer {
    static_assert(sizeof(T) <= 64, "Buffer类型过大");
    T data[64];
};
```

## 三、注意事项与常见陷阱总结

- 不要过度泛化：如果只用两三种类型，重载可能更好
- 始终考虑编译时间和代码膨胀的代价
- 使用concept（C++20）提供清晰的约束和错误信息
- 模板定义放在头文件，使用`extern template`减少实例化
- 为模板编写针对多种类型的测试
- 提供`static_assert`进行编译期检查
- 模板参数命名应反映其用途
- 记录模板的要求（前置条件、约束）在文档或concept中
