# 模板与性能

## 一、概念说明

模板编程对性能的影响主要体现在三个方面：**代码膨胀**（每个实例化生成独立代码）、**编译时间**（模板实例化增加编译负担）和**运行时优化**（编译期信息带来内联等优化机会）。理解和权衡这三者是高效使用模板的关键。

## 二、具体用法

### 2.1 代码膨胀示例

```cpp
// 每种类型组合都会生成独立代码
template <typename T, typename U>
class Pair {
    T first;
    U second;
public:
    Pair(T a, U b) : first(a), second(b) {}
    void show() const {
        std::cout << first << ", " << second << std::endl;
    }
};

// 以下会生成4个不同的类：
// Pair<int, int>
// Pair<int, double>
// Pair<double, int>
// Pair<double, std::string>
```

### 2.2 减少代码膨胀的策略

```cpp
// 策略1：将公共逻辑提取到非模板基类
class PrintHelper {
protected:
    static void print_line(const std::string& s) {
        std::cout << "[LOG] " << s << std::endl;
    }
};

template <typename T>
class Container : private PrintHelper {
    std::vector<T> data;
public:
    void add(const T& val) {
        data.push_back(val);
        print_line("添加元素");  // 共享同一份代码
    }
};

// 策略2：显式实例化常用类型
// template class Container<int>;
// template class Container<double>;
```

### 2.3 编译时间优化

```cpp
// extern template 减少重复实例化
// 在头文件中声明
// extern template class std::vector<int>;
// extern template class std::vector<double>;

// 在一个cpp文件中集中实例化
// template class std::vector<int>;
// template class std::vector<double>;

// 实际效果：避免每个翻译单元都实例化相同的模板
```

### 2.4 运行时性能优势

```cpp
// 模板允许编译期优化
template <int N>
constexpr int factorial() {
    int result = 1;
    for (int i = 2; i <= N; ++i) result *= i;
    return result;
}

int main() {
    // 编译期计算，运行时零开销
    constexpr int fact10 = factorial<10>();
    std::cout << "10! = " << fact10 << std::endl;  // 3628800

    // 类型特定优化
    auto process_int = [](int* data, int n) {
        // 编译器知道类型，可以内联和优化
        int sum = 0;
        for (int i = 0; i < n; ++i) sum += data[i];
        return sum;
    };
}
```

### 2.5 性能对比表

```cpp
/*

|--------------|---------------|---------------|
| 调用开销      | 零（内联）    | vtable查找    |

```

## 三、注意事项与常见陷阱

- 每个模板实例化都会增加代码体积，需要权衡
- 显式实例化可以控制代码膨胀，但降低了灵活性
- 编译时间随模板使用增加，大型项目尤为明显
- 模板允许编译器进行深层内联优化，这往往是性能优势
- 使用`-ftime-report`（GCC/Clang）分析模板实例化对编译时间的影响
- Profile-guided optimization (PGO) 可以进一步优化模板代码
- Header-only库的编译时间代价需要特别关注
