# 模板与虚函数

## 一、概念说明

C++中虚函数不能是模板函数，因为虚函数表（vtable）需要在编译时确定大小，而模板函数的实例化数量在编译时未知。当需要运行时多态与泛型结合时，需要采用替代方案。

## 二、具体用法

### 2.1 限制：虚函数不能是模板

```cpp
class Base {
public:
    // 错误：虚函数不能是模板
    // template <typename T> virtual void process(T val) = 0;

    // 正确：虚函数使用已知类型
    virtual void process(int val) = 0;
    virtual ~Base() = default;
};
```

### 2.2 替代方案：类型擦除

```cpp
// 使用std::function实现类型擦除
class Processor {
    std::function<void(const std::any&)> handler;
public:
    template <typename T, typename Func>
    Processor(T, Func func) {
        handler = [func](const std::any& val) {
            func(std::any_cast<T>(val));
        };
    }

    void process(const std::any& val) const {
        handler(val);
    }
};

int main() {
    Processor int_proc(int{}, [](int v) {
        std::cout << "int: " << v << std::endl;
    });
    Processor str_proc(std::string{}, [](const std::string& v) {
        std::cout << "string: " << v << std::endl;
    });

    int_proc.process(42);                  // int: 42
    str_proc.process(std::string("hi"));   // string: hi
}
```

### 2.3 替代方案：非模板虚函数+模板构造

```cpp
class Serializer {
public:
    virtual ~Serializer() = default;
    virtual void serialize(std::ostream& os) const = 0;
};

// 模板类实现非模板虚函数
template <typename T>
class TypedSerializer : public Serializer {
    T value;
public:
    TypedSerializer(T v) : value(v) {}

    void serialize(std::ostream& os) const override {
        os << value;
    }
};

int main() {
    std::vector<std::unique_ptr<Serializer>> serializers;
    serializers.push_back(std::make_unique<TypedSerializer<int>>(42));
    serializers.push_back(std::make_unique<TypedSerializer<double>>(3.14));

    for (const auto& s : serializers) {
        s->serialize(std::cout);
        std::cout << " ";
    }
    // 42 3.14
}
```

### 2.4 替代方案：访问者模式

```cpp
class Shape {
public:
    virtual ~Shape() = default;
    virtual void accept(class Visitor& v) = 0;
};

class Visitor {
public:
    virtual ~Visitor() = default;
    virtual void visit(class Circle& c) = 0;
    virtual void visit(class Rectangle& r) = 0;
};

class Circle : public Shape {
public:
    double radius = 5.0;
    void accept(Visitor& v) override { v.visit(*this); }
};

class Rectangle : public Shape {
public:
    double w = 3.0, h = 4.0;
    void accept(Visitor& v) override { v.visit(*this); }
};

class AreaVisitor : public Visitor {
public:
    double area = 0;
    void visit(Circle& c) override {
        area = 3.14159 * c.radius * c.radius;
        std::cout << "圆面积: " << area << std::endl;
    }
    void visit(Rectangle& r) override {
        area = r.w * r.h;
        std::cout << "矩形面积: " << area << std::endl;
    }
};
```

## 三、注意事项与常见陷阱

- 虚函数不能是模板的根本原因：vtable大小必须在编译期确定
- 类型擦除（type erasure）是最常用的替代方案
- 非模板虚函数+模板派生类是最简单直接的替代方案
- `std::any`和`std::function`都使用了类型擦除技术
- 访问者模式适合需要在不修改类的情况下添加操作
- 性能权衡：虚函数有运行时开销，模板有编译时开销
