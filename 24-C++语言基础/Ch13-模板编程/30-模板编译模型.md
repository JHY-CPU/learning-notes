# 模板编译模型

## 一、概念说明

C++模板的编译模型决定了模板代码在何处被解析和实例化。主要有两种模型：**包含模型**（Inclusion Model）和**分离模型**（Separation Model，已基本废弃）。现代C++实践统一使用包含模型。

## 二、具体用法

### 2.1 包含模型（主流做法）

```cpp
// math_utils.h — 模板定义放在头文件中
#ifndef MATH_UTILS_H
#define MATH_UTILS_H

template <typename T>
T maximum(T a, T b) {
    return (a > b) ? a : b;
}

template <typename T>
class Calculator {
    T value;
public:
    Calculator(T v) : value(v) {}
    T square() const { return value * value; }
};

#endif

// main.cpp — 直接包含头文件
// #include "math_utils.h"
// 编译器在此处看到完整定义并实例化
```

### 2.2 为什么不能分离编译

```cpp
// 错误示范：模板声明和定义分离
// widget.h
template <typename T>
class Widget {
    T data;
public:
    Widget(T d);
    void show() const;  // 只有声明
};

// widget.cpp
// #include "widget.h"
template <typename T>
Widget<T>::Widget(T d) : data(d) {}  // 定义在.cpp中

template <typename T>
void Widget<T>::show() const {
    std::cout << data << std::endl;
}

// main.cpp — 链接错误！
// #include "widget.h"
// Widget<int> w(42);  // 链接时找不到Widget<int>::show()的定义
```

### 2.3 包含模型的实现方式

```cpp
// 方式1：全部在头文件（简单模板）
// utils.h
template <typename T>
void print_val(T val) { std::cout << val << std::endl; }

// 方式2：头文件包含实现文件（复杂模板）
// template_impl.h
template <typename T>
class ComplexClass {
    // ...声明
};

// 在头文件末尾包含实现
// #include "template_impl.tpp"

// template_impl.tpp（非标准扩展名，但常见）
// template <typename T>
// ComplexClass<T>::method() { ... }
```

### 2.4 显式实例化减少膨胀

```cpp
// matrix.h
template <typename T>
class Matrix {
    std::vector<T> data;
    int rows, cols;
public:
    Matrix(int r, int c) : data(r * c), rows(r), cols(c) {}
    T& at(int r, int c) { return data[r * cols + c]; }
    // ... 其他方法
};

// matrix.cpp — 显式实例化常用类型
// template class Matrix<int>;
// template class Matrix<double>;
// template class Matrix<float>;

// 使用文件中
// extern template class Matrix<int>;  // 告诉编译器不在此处实例化
```

## 三、注意事项与常见陷阱

- 模板定义必须对使用它的每个翻译单元可见（包含模型）
- 分离模型（`export`关键字）在C++11中被移除
- 头文件包含模板实现可能导致编译时间增加
- 使用`extern template`减少重复实例化
- 模板实现文件常用`.tpp`或`.ipp`扩展名
- Header-only库（如Boost大量组件）就是包含模型的极致应用
- 对于大型模板库，考虑显式实例化常用类型减少编译时间
