# 模板元编程进阶

## 一、概念说明

模板元编程进阶包括类型列表（Type List）、编译期类型映射、以及复杂的类型计算。这些技术是实现类型安全的序列化、反射系统和高级泛型库的基础。

## 二、具体用法

### 2.1 类型列表

```cpp
// 类型列表定义
template <typename... Types>
struct TypeList {};

// 获取类型列表长度
template <typename List>
struct Length;

template <typename... Types>
struct Length<TypeList<Types...>> {
    static constexpr std::size_t value = sizeof...(Types);
};

// 获取第N个类型
template <std::size_t N, typename List>
struct TypeAt;

template <typename Head, typename... Tail>
struct TypeAt<0, TypeList<Head, Tail...>> {
    using type = Head;
};

template <std::size_t N, typename Head, typename... Tail>
struct TypeAt<N, TypeList<Head, Tail...>> {
    using type = typename TypeAt<N-1, TypeList<Tail...>>::type;
};

int main() {
    using MyList = TypeList<int, double, char, float>;
    std::cout << "长度: " << Length<MyList>::value << std::endl;  // 4

    using SecondType = typename TypeAt<1, MyList>::type;
    std::cout << "第2个是double: " << std::is_same_v<SecondType, double> << std::endl; // true
}
```

### 2.2 类型映射

```cpp
// 将列表中的每个类型转为指针
template <typename List>
struct MapPointer;

template <typename... Types>
struct MapPointer<TypeList<Types...>> {
    using type = TypeList<Types*...>;
};

// 将每个类型加const
template <typename List>
struct MapConst;

template <typename... Types>
struct MapConst<TypeList<Types...>> {
    using type = TypeList<const Types...>;
};

int main() {
    using List = TypeList<int, double, char>;
    using PtrList = typename MapPointer<List>::type;
    using ConstList = typename MapConst<List>::type;

    // PtrList 等价于 TypeList<int*, double*, char*>
    std::cout << std::is_same_v<typename TypeAt<0, PtrList>::type, int*> << std::endl; // true
}
```

### 2.3 编译期类型检测

```cpp
// 检查类型是否在列表中
template <typename T, typename List>
struct Contains;

template <typename T>
struct Contains<T, TypeList<>> : std::false_type {};

template <typename T, typename... Tail>
struct Contains<T, TypeList<T, Tail...>> : std::true_type {};

template <typename T, typename Head, typename... Tail>
struct Contains<T, TypeList<Head, Tail...>> : Contains<T, TypeList<Tail...>> {};

int main() {
    using MyList = TypeList<int, double, char>;
    std::cout << Contains<int, MyList>::value << std::endl;     // true
    std::cout << Contains<float, MyList>::value << std::endl;   // false
}
```

### 2.4 编译期条件分支

```cpp
// 编译期switch
template <int N, typename List>
struct Switch;

template <typename First, typename... Rest>
struct Switch<0, TypeList<First, Rest...>> {
    using type = First;
};

template <int N, typename First, typename... Rest>
struct Switch<N, TypeList<First, Rest...>> {
    using type = typename Switch<N-1, TypeList<Rest...>>::type;
};

// 用法：根据索引选择类型
using Types = TypeList<int, double, char, float, long>;
using Selected = typename Switch<2, Types>::type;  // char

int main() {
    std::cout << sizeof(Selected) << std::endl;  // 1 (char的大小)
}
```

## 三、注意事项与常见陷阱

- 类型列表是函数式编程中的List在C++类型系统中的映射
- 模板特化实现模式匹配，类似于Haskell的case/match
- 类型列表技术已部分被`std::tuple`和`std::variant`替代
- 现代C++中这些技术主要用于理解库实现和极端性能场景
- 编译时间会随类型列表长度增加而显著增长
- 考虑使用`constexpr`函数和折叠表达式简化编译期计算
