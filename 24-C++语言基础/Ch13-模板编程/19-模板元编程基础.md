# 模板元编程基础

## 一、概念说明

模板元编程（Template Metaprogramming, TMP）利用模板机制在编译期执行计算。由于模板实例化发生在编译时，所有计算结果在编译期就已确定，运行时零开销。经典应用包括编译期数值计算、类型计算和代码生成。

## 二、具体用法

### 2.1 编译期阶乘

```cpp
// 递归模板实现编译期阶乘
template <int N>
struct Factorial {
    static constexpr int value = N * Factorial<N - 1>::value;
};

// 递归终止条件
template <>
struct Factorial<0> {
    static constexpr int value = 1;
};

int main() {
    // 以下计算在编译期完成
    constexpr int fact5 = Factorial<5>::value;
    constexpr int fact10 = Factorial<10>::value;
    std::cout << "5! = " << fact5 << std::endl;    // 5! = 120
    std::cout << "10! = " << fact10 << std::endl;  // 10! = 3628800

    // 可作为数组大小（需要编译期常量）
    int arr[Factorial<4>::value];  // 大小为24的数组
    std::cout << "数组大小: " << sizeof(arr)/sizeof(int) << std::endl; // 24
}
```

### 2.2 编译期斐波那契

```cpp
template <int N>
struct Fibonacci {
    static constexpr int value = Fibonacci<N-1>::value + Fibonacci<N-2>::value;
};

template <>
struct Fibonacci<0> { static constexpr int value = 0; };

template <>
struct Fibonacci<1> { static constexpr int value = 1; };

int main() {
    std::cout << "fib(0)=" << Fibonacci<0>::value << std::endl;  // 0
    std::cout << "fib(1)=" << Fibonacci<1>::value << std::endl;  // 1
    std::cout << "fib(10)=" << Fibonacci<10>::value << std::endl; // 55
    std::cout << "fib(20)=" << Fibonacci<20>::value << std::endl; // 6765
}
```

### 2.3 编译期求和

```cpp
// 变参模板编译期求和
template <int... Values>
struct Sum;

template <int First, int... Rest>
struct Sum<First, Rest...> {
    static constexpr int value = First + Sum<Rest...>::value;
};

template <>
struct Sum<> {
    static constexpr int value = 0;
};

int main() {
    constexpr int s = Sum<1, 2, 3, 4, 5>::value;
    std::cout << "Sum = " << s << std::endl;  // Sum = 15
}
```

### 2.4 编译期类型选择

```cpp
// 编译期if-else（模板特化版）
template <bool Cond, typename TrueType, typename FalseType>
struct If;

template <typename TrueType, typename FalseType>
struct If<true, TrueType, FalseType> {
    using type = TrueType;
};

template <typename TrueType, typename FalseType>
struct If<false, TrueType, FalseType> {
    using type = FalseType;
};

// 使用
template <bool IsLarge>
using StorageType = typename If<IsLarge, std::string, int>::type;

int main() {
    StorageType<true> big = "large data";
    StorageType<false> small = 42;
    std::cout << big << std::endl;    // large data
    std::cout << small << std::endl;  // 42
}
```

## 三、注意事项与常见陷阱

- 模板元编程本质上是函数式编程：没有变量，只有递归和模式匹配
- `constexpr`函数（C++11/14）可以替代部分模板元编程，更直观
- 递归深度受编译器限制，过深的递归会导致编译失败
- 模板元编程的错误信息极其冗长且难以理解
- C++14的`constexpr`函数和C++17的`if constexpr`使编译期计算更简单
- 经典TMP技术在现代C++中部分被`constexpr`取代，但仍是理解STL和高级库的基础
