# 模板与宏对比

## 一、概念说明

宏（Macro）和模板（Template）都能实现代码泛化，但有本质区别：宏是**文本替换**（预处理阶段），模板是**类型实例化**（编译阶段）。模板提供类型安全、作用域规则和调试支持，而宏虽然更灵活但容易出错。

## 二、具体用法

### 2.1 类型安全对比

```cpp
// 宏：不安全
#define MAX(a, b) ((a) > (b) ? (a) : (b))
// 问题：多次求值、缺少类型检查
int x = 5;
std::cout << MAX(x++, 10) << std::endl;  // x被求值两次！x变成7而非6

// 模板：安全
template <typename T>
T safe_max(T a, T b) {
    return (a > b) ? a : b;  // 每个参数只求值一次
}

int y = 5;
std::cout << safe_max(y++, 10) << std::endl;  // y只求值一次，y=6
```

### 2.2 调试对比

```cpp
// 宏：错误指向宏调用处，不指向宏定义
#define SQUARE(x) ((x) * (x))
// 错误信息可能包含展开后的复杂文本

// 模板：错误指向模板定义处，有完整的类型信息
template <typename T>
T square(T x) {
    return x * x;  // 错误会明确指出T的类型
}
```

### 2.3 适用场景对比

```cpp
// 宏适合的场景：
// 1. 条件编译
#ifdef DEBUG
    #define LOG(msg) std::cout << "[DEBUG] " << msg << std::endl
#else
    #define LOG(msg)
#endif

// 2. 字符串拼接
#define CONCAT(a, b) a##b
int CONCAT(my, Var) = 42;  // int myVar = 42;

// 3. 文件和行号
#define LOCATION __FILE__ ":" + std::to_string(__LINE__)

// 模板适合的场景：
// 1. 类型安全的通用算法
template <typename Iter>
auto distance(Iter first, Iter last) {
    return std::distance(first, last);
}

// 2. 编译期计算
template <int N>
struct Power2 {
    static constexpr int value = 2 * Power2<N-1>::value;
};
```

### 2.4 对比总结表

```cpp
/*
| 特性          | 模板                | 宏                  |
|--------------|---------------------|---------------------|
| 处理阶段      | 编译期              | 预处理期            |
| 类型安全      | 完全安全            | 不安全              |
| 作用域        | 遵循C++作用域规则   | 无作用域，全局替换  |
| 调试支持      | 好                  | 差                  |
| 过度求值      | 不会                | 可能会（多次求值）  |
| 重载          | 支持                | 不支持              |
| 特化          | 支持                | 不支持              |
| 条件编译      | 不支持              | 支持                |
*/
```

## 三、注意事项与常见陷阱

- 宏的常见陷阱：运算符优先级、多次求值、命名污染
- 模板错误信息虽然冗长，但包含类型信息；宏错误信息完全不可读
- 现代C++中应优先使用模板，宏仅用于条件编译等模板无法替代的场景
- `constexpr`函数可以替代大部分宏的性能优化用途
- `inline`函数可以替代简单的函数式宏
- 如果必须用宏，注意用括号包裹每个参数和整个表达式
- 使用`#undef`限制宏的作用范围
