# 内存泄漏检测

## 一、概念说明

内存泄漏指程序分配了内存但未释放，随着时间推移导致内存耗尽。常见原因：忘记`delete`、异常路径未释放、循环引用等。

常用检测工具：
- **Valgrind**（Linux）：最全面的内存检测工具
- **AddressSanitizer (ASan)**：编译器内置（GCC/Clang），开销较小
- **Visual Studio 诊断工具**（Windows）：集成的内存检测

```cpp
// 示例：有内存泄漏的程序
#include <iostream>

void leakyFunction() {
    int* data = new int[100]; // 分配但未释放
    data[0] = 42;
    std::cout << "data[0] = " << data[0] << std::endl;
    // 忘记 delete[] data; !
}

int main() {
    leakyFunction();
    std::cout << "程序结束" << std::endl;
    return 0;
}
```

**输出：**
```
data[0] = 42
程序结束
// Valgrind会报告: "100 bytes in 1 blocks are definitely lost"
```

## 二、具体用法

### 2.1 使用Valgrind检测

```bash
# 编译程序
g++ -g -o leak_test leak_test.cpp

# 运行Valgrind检测
valgrind --leak-check=full --show-leak-kinds=all ./leak_test

# 输出示例：
# ==12345== HEAP SUMMARY:
# ==12345==     in use at exit: 400 bytes in 1 blocks
# ==12345==   total heap usage: 2 allocs, 1 frees, 1,424 bytes allocated
# ==12345==
# ==12345== 400 bytes in 1 blocks are definitely lost in loss record 1 of 1
# ==12345==    at 0x4C2BBAF: operator new[](unsigned long)
# ==12345==    by 0x400678: leakyFunction()
# ==12345==    by 0x400693: main
```

### 2.2 使用AddressSanitizer

```bash
# 编译时启用ASan
g++ -fsanitize=address -g -o asan_test leak_test.cpp

# 运行时会自动报告内存问题
./asan_test

# 输出示例：
# ==12345==ERROR: LeakSanitizer: detected memory leaks
# Direct leak of 400 byte(s) in 1 object(s) allocated from:
#     #0 0x... in operator new[](unsigned long)
#     #1 0x... in leakyFunction()
```

### 2.3 代码级检测（重载new/delete）

```cpp
#include <iostream>
#include <cstdlib>
#include <unordered_map>

static std::unordered_map<void*, size_t> allocMap;
static size_t totalAllocated = 0;

void* operator new(size_t size) {
    void* ptr = std::malloc(size);
    if (!ptr) throw std::bad_alloc();
    allocMap[ptr] = size;
    totalAllocated += size;
    return ptr;
}

void operator delete(void* ptr) noexcept {
    if (allocMap.count(ptr)) {
        totalAllocated -= allocMap[ptr];
        allocMap.erase(ptr);
    }
    std::free(ptr);
}

void leakyFunction() {
    int* p = new int(42);
    std::cout << "*p = " << *p << std::endl;
    // 忘记 delete p;
}

int main() {
    leakyFunction();

    std::cout << "\n=== 内存泄漏报告 ===" << std::endl;
    std::cout << "未释放的分配: " << allocMap.size() << std::endl;
    std::cout << "泄漏字节数: " << totalAllocated << std::endl;

    return 0;
}
```

**输出：**
```
*p = 42

=== 内存泄漏报告 ===
未释放的分配: 1
泄漏字节数: 4
```

## 三、注意事项与常见陷阱

- **Release模式可能隐藏泄漏**：Debug模式检测更可靠。
- **ASan运行时开销约2倍**：不适合生产环境。
- **Valgrind只支持Linux**：Windows可用Dr.Memory替代。
- **重载`new`/`delete`应谨慎**：可能与STL/第三方库冲突。
- **智能指针能消除大部分泄漏**：但仍需注意循环引用。
- **定期在CI中运行泄漏检测**：防止回归。
