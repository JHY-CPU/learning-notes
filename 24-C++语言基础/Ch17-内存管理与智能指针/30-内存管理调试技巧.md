# 内存管理调试技巧

## 一、概念说明

内存问题是最常见的C++ bug来源，包括泄漏、越界、use-after-free、double-free等。有效的调试需要结合工具和编码规范。

主要调试工具：
- **AddressSanitizer (ASan)**：编译器内置，检测越界和use-after-free
- **Valgrind/Memcheck**：运行时检测，最全面但较慢
- **LeakSanitizer (LSan)**：专门检测内存泄漏
- **MemorySanitizer (MSan)**：检测未初始化内存使用
- **Visual Studio 诊断工具**：Windows平台集成

```cpp
#include <iostream>
#include <memory>
#include <vector>

// 调试友好的内存包装器
template <typename T>
class DebugPtr {
    T* ptr;
    static int liveCount;
public:
    DebugPtr(T* p = nullptr) : ptr(p) {
        if (ptr) ++liveCount;
        std::cout << "DebugPtr创建, 活跃数: " << liveCount << std::endl;
    }
    ~DebugPtr() {
        if (ptr) {
            --liveCount;
            std::cout << "DebugPtr销毁, 活跃数: " << liveCount << std::endl;
            delete ptr;
        }
    }
    T* get() const { return ptr; }
    T& operator*() const { return *ptr; }
    static int live() { return liveCount; }
};

template <typename T> int DebugPtr<T>::liveCount = 0;

int main() {
    std::cout << "初始活跃: " << DebugPtr<int>::live() << std::endl;
    {
        DebugPtr<int> p1(new int(42));
        DebugPtr<int> p2(new int(99));
        std::cout << "活跃: " << DebugPtr<int>::live() << std::endl;
    }
    std::cout << "最终活跃: " << DebugPtr<int>::live() << std::endl;
    return 0;
}
```

**输出：**
```
初始活跃: 0
DebugPtr创建, 活跃数: 1
DebugPtr创建, 活跃数: 2
活跃: 2
DebugPtr销毁, 活跃数: 1
DebugPtr销毁, 活跃数: 0
最终活跃: 0
```

## 二、具体用法

### 2.1 常用编译选项

```bash
# Debug构建推荐选项
g++ -g -O0 -Wall -Wextra -Wpedantic \
    -fsanitize=address \        # 内存错误检测
    -fsanitize=undefined \      # 未定义行为检测
    -fno-omit-frame-pointer \   # 更好的栈追踪
    -D_GLIBCXX_DEBUG \          # STL调试模式
    -o debug_build program.cpp

# 运行检测
ASAN_OPTIONS="detect_leaks=1:abort_on_error=1" ./debug_build

# 单独检测泄漏
g++ -g -fsanitize=leak -o leak_check program.cpp
./leak_check
```

### 2.2 内存使用统计

```cpp
#include <iostream>
#include <atomic>
#include <cstdlib>

// 全局内存统计
struct MemoryStats {
    std::atomic<size_t> allocCount{0};
    std::atomic<size_t> deallocCount{0};
    std::atomic<size_t> totalAllocated{0};
    std::atomic<size_t> totalFreed{0};

    void report() const {
        std::cout << "\n=== 内存统计 ===" << std::endl;
        std::cout << "分配次数: " << allocCount << std::endl;
        std::cout << "释放次数: " << deallocCount << std::endl;
        std::cout << "分配总字节: " << totalAllocated << std::endl;
        std::cout << "释放总字节: " << totalFreed << std::endl;
        std::cout << "泄漏字节: " << (totalAllocated - totalFreed) << std::endl;
    }
};

static MemoryStats stats;

void* operator new(size_t size) {
    ++stats.allocCount;
    stats.totalAllocated += size;
    void* p = std::malloc(size);
    if (!p) throw std::bad_alloc();
    return p;
}

void operator delete(void* p) noexcept {
    ++stats.deallocCount;
    // 注意：无法知道原始大小
    std::free(p);
}

int main() {
    int* a = new int(1);
    int* b = new int(2);
    // 忘记 delete a;
    delete b;

    stats.report();
    return 0;
}
```

**输出：**
```
=== 内存统计 ===
分配次数: 2
释放次数: 1
分配总字节: 8
释放总字节: 4
泄漏字节: 4
```

### 2.3 STL调试模式

```cpp
#define _GLIBCXX_DEBUG  // 在所有include之前定义
#include <iostream>
#include <vector>

int main() {
    std::vector<int> vec = {1, 2, 3};

    // STL调试模式会检测越界访问
    try {
        std::cout << vec.at(0) << std::endl;
        // vec.at(10);  // 会抛出异常
    } catch (const std::out_of_range& e) {
        std::cout << "越界: " << e.what() << std::endl;
    }

    return 0;
}
```

**输出：**
```
1
```

## 三、注意事项与常见陷阱

- **Debug模式与Release模式行为可能不同**：某些bug只在Release模式出现。
- **ASan与Valgrind不要同时使用**：会互相干扰。
- **`_GLIBCXX_DEBUG`会改变容器的ABI**：不能与非调试编译的库混用。
- **生产环境不要启用Sanitizer**：性能开销太大。
- **CI/CD中应包含内存检测**：每次提交自动运行ASan测试。
- **Windows可用`Application Verifier`**检测堆问题。
