# 内存碎片问题

## 一、概念说明

内存碎片分为**外部碎片**和**内部碎片**：
- **外部碎片**：已释放的内存块太小、太分散，无法满足大块内存请求
- **内部碎片**：分配的内存块大于实际需要，浪费的部分

长时间运行的程序频繁分配/释放不同大小的内存容易产生碎片，导致"有足够空闲内存但无法分配"的问题。

```cpp
#include <iostream>
#include <vector>

int main() {
    // 模拟外部碎片：分配交替大小的块
    std::vector<int*> ptrs;

    // 分配10个块
    for (int i = 0; i < 10; ++i) {
        ptrs.push_back(new int[i + 1]);
        std::cout << "分配块" << i << ", 大小=" << (i+1)*sizeof(int) << "字节" << std::endl;
    }

    // 释放所有偶数索引的块
    for (int i = 0; i < 10; i += 2) {
        delete[] ptrs[i];
        ptrs[i] = nullptr;
        std::cout << "释放块" << i << std::endl;
    }

    // 现在虽然有空闲内存，但被分割成小块
    // 尝试分配大块可能失败
    std::cout << "\n空闲内存已被分割成不连续的小块" << std::endl;

    // 清理
    for (int i = 1; i < 10; i += 2) {
        delete[] ptrs[i];
    }

    return 0;
}
```

**输出：**
```
分配块0, 大小=4字节
分配块1, 大小=8字节
...
释放块0
释放块2
...

空闲内存已被分割成不连续的小块
```

## 二、具体用法

### 2.1 内存池缓解碎片

```cpp
#include <iostream>
#include <vector>
#include <cstddef>

// 固定大小块分配器 - 无外部碎片
class SlabAllocator {
    struct Block { Block* next; };

    std::vector<char*> slabs;
    Block* freeList = nullptr;
    size_t blockSize;
    size_t slabCapacity;

public:
    SlabAllocator(size_t bSize, size_t capacity = 1024)
        : blockSize(std::max(bSize, sizeof(Block))), slabCapacity(capacity) {
        grow();
    }

    ~SlabAllocator() {
        for (auto* slab : slabs) delete[] slab;
    }

    void* allocate() {
        if (!freeList) grow();
        Block* block = freeList;
        freeList = block->next;
        return block;
    }

    void deallocate(void* ptr) {
        auto* block = static_cast<Block*>(ptr);
        block->next = freeList;
        freeList = block;
    }

private:
    void grow() {
        char* slab = new char[blockSize * slabCapacity];
        slabs.push_back(slab);
        for (size_t i = 0; i < slabCapacity; ++i) {
            auto* block = reinterpret_cast<Block*>(slab + i * blockSize);
            block->next = freeList;
            freeList = block;
        }
    }
};

int main() {
    SlabAllocator pool(sizeof(int), 100);
    std::vector<void*> ptrs;

    // 分配100个块 - 无碎片
    for (int i = 0; i < 100; ++i) {
        ptrs.push_back(pool.allocate());
    }
    std::cout << "分配100个块，无外部碎片" << std::endl;

    // 释放一半
    for (int i = 0; i < 100; i += 2) {
        pool.deallocate(ptrs[i]);
    }
    std::cout << "释放50个块后，仍可分配新块（通过空闲链表）" << std::endl;

    for (int i = 1; i < 100; i += 2) {
        pool.deallocate(ptrs[i]);
    }

    return 0;
}
```

**输出：**
```
分配100个块，无外部碎片
释放50个块后，仍可分配新块（通过空闲链表）
```

## 三、注意事项与常见陷阱

- **固定大小块分配器完全消除外部碎片**：但只适合特定大小对象。
- **频繁分配不同大小的内存最易碎片化**：应尽量统一大小。
- **内存池的容量规划很重要**：过度分配浪费内存，不足则需要扩展。
- **`std::deque`比`std::vector`碎片化更小**：分块分配策略。
- **定期整理（compaction）**：某些运行时（如游戏引擎）定期移动对象消除碎片。
- **操作系统层面的碎片**：虚拟内存映射也会影响碎片。
