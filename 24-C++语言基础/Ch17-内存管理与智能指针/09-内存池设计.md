# 内存池设计

## 一、概念说明

内存池（Memory Pool）预先分配一大块内存，然后以小块方式分配给程序使用。相比频繁调用`malloc`/`new`，内存池减少了系统调用开销，降低了内存碎片，提高了分配速度。

常见类型：
- **固定大小块分配器**：所有块大小相同，用空闲链表管理
- **Slab分配器**：为特定对象类型预分配，Linux内核使用
- **区域分配器（Arena）**：分配后统一释放

```cpp
#include <iostream>
#include <vector>
#include <cstddef>

// 固定大小块内存池
class FixedBlockPool {
    struct Block { Block* next; };

    char* pool;           // 内存池首地址
    Block* freeList;      // 空闲链表
    size_t blockSize;     // 每块大小
    size_t blockCount;    // 块数量

public:
    FixedBlockPool(size_t bSize, size_t count)
        : blockSize(std::max(bSize, sizeof(Block))), blockCount(count) {
        pool = new char[blockSize * blockCount];
        freeList = nullptr;

        // 将所有块串成空闲链表
        for (size_t i = 0; i < blockCount; ++i) {
            auto* block = reinterpret_cast<Block*>(pool + i * blockSize);
            block->next = freeList;
            freeList = block;
        }
        std::cout << "内存池创建: " << blockCount << "块, 每块" << blockSize << "字节" << std::endl;
    }

    ~FixedBlockPool() { delete[] pool; }

    void* allocate() {
        if (!freeList) return nullptr;
        Block* block = freeList;
        freeList = block->next;
        return block;
    }

    void deallocate(void* ptr) {
        auto* block = reinterpret_cast<Block*>(ptr);
        block->next = freeList;
        freeList = block;
    }
};

int main() {
    FixedBlockPool pool(sizeof(int), 5);

    int* a = static_cast<int*>(pool.allocate());
    int* b = static_cast<int*>(pool.allocate());
    *a = 42; *b = 99;
    std::cout << "a=" << *a << " b=" << *b << std::endl;

    pool.deallocate(a);
    pool.deallocate(b);
    std::cout << "内存已归还池" << std::endl;

    return 0;
}
```

**输出：**
```
内存池创建: 5块, 每块8字节
a=42 b=99
内存已归还池
```

## 二、具体用法

### 2.1 Slab分配器概念

```cpp
#include <iostream>
#include <memory>
#include <vector>

// 为特定类型设计的Slab分配器
template <typename T>
class SlabAllocator {
    std::vector<T*> slabs;
    size_t slabSize;

public:
    SlabAllocator(size_t count = 1024) : slabSize(count) {
        T* slab = static_cast<T*>(::operator new(count * sizeof(T)));
        slabs.push_back(slab);
        std::cout << "为 " << typeid(T).name() << " 分配了" << count << "个对象的slab" << std::endl;
    }

    ~SlabAllocator() {
        for (auto* slab : slabs)
            ::operator delete(slab);
    }

    template <typename... Args>
    T* construct(Args&&... args) {
        T* obj = static_cast<T*>(::operator new(sizeof(T)));
        return ::new(obj) T(std::forward<Args>(args)...);
    }

    void destroy(T* obj) {
        obj->~T();
        ::operator delete(obj);
    }
};

struct Node { int data; Node(int d) : data(d) {} };

int main() {
    SlabAllocator<Node> alloc(100);
    Node* n = alloc.construct(42);
    std::cout << "Node data: " << n->data << std::endl;
    alloc.destroy(n);
    return 0;
}
```

**输出：**
```
为 4Node 分配了100个对象的slab
Node data: 42
```

## 三、注意事项与常见陷阱

- **内存池不调用析构函数**：需自行管理对象生命周期。
- **固定大小池只适合特定大小的对象**：不同大小需要不同池。
- **线程安全需要额外同步**：多线程使用时需加锁或使用无锁结构。
- **池内存耗尽时的行为**：应优雅处理分配失败。
- **对齐要求**：确保池中每个块都正确对齐。
