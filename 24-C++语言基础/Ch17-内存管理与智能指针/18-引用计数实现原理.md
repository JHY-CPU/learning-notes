# 引用计数实现原理

## 一、概念说明

`shared_ptr`内部维护一个**控制块（Control Block）**，存储：
- **强引用计数**（strong count）：`shared_ptr`的数量
- **弱引用计数**（weak count）：`weak_ptr`的数量
- **删除器**：释放对象的回调
- **分配器**：可选的内存分配器

引用计数操作使用**原子操作**保证线程安全，但对象本身的访问不是线程安全的。

```cpp
#include <iostream>
#include <memory>
#include <atomic>

// 简化的控制块概念演示
template <typename T>
struct SimplifiedControlBlock {
    std::atomic<long> strong_count{0}; // 强引用计数
    std::atomic<long> weak_count{0};   // 弱引用计数
    T* object;                         // 指向实际对象
    void (*deleter)(T*);               // 删除器函数指针

    SimplifiedControlBlock(T* obj, void (*del)(T*))
        : strong_count(1), weak_count(0), object(obj), deleter(del) {}
};

int main() {
    // 实际的shared_ptr实现
    auto sp = std::make_shared<int>(42);
    std::cout << "强引用计数: " << sp.use_count() << std::endl;

    auto sp2 = sp;
    std::cout << "拷贝后强引用计数: " << sp.use_count() << std::endl;

    std::weak_ptr<int> wp = sp;
    std::cout << "weak_ptr后强引用计数: " << sp.use_count() << std::endl;

    // 注意：weak_count()在C++20起可用
    // std::cout << "弱引用计数: " << sp.weak_count() << std::endl;

    return 0;
}
```

**输出：**
```
强引用计数: 1
拷贝后强引用计数: 2
weak_ptr后强引用计数: 2
```

## 二、具体用法

### 2.1 引用计数的生命周期

```cpp
#include <iostream>
#include <memory>

struct Resource {
    int id;
    Resource(int i) : id(i) { std::cout << "Resource(" << id << ") 创建" << std::endl; }
    ~Resource() { std::cout << "Resource(" << id << ") 析构" << std::endl; }
};

int main() {
    std::weak_ptr<Resource> wp;
    std::cout << "=== 阶段1: 创建shared_ptr ===" << std::endl;
    {
        auto sp = std::make_shared<Resource>(1);
        wp = sp;
        std::cout << "强引用: " << sp.use_count() << std::endl;
    }
    // 对象已释放，但控制块仍在（因为wp存在）
    std::cout << "对象已释放: " << wp.expired() << std::endl;

    std::cout << "\n=== 阶段2: weak_ptr生命周期 ===" << std::endl;
    {
        auto sp1 = std::make_shared<Resource>(2);
        auto sp2 = sp1;
        std::weak_ptr<Resource> wp2 = sp1;

        std::cout << "sp1强引用: " << sp1.use_count() << std::endl;
        sp1.reset();
        std::cout << "sp1.reset后强引用: " << sp2.use_count() << std::endl;
        sp2.reset();
        std::cout << "sp2.reset后 expired: " << wp2.expired() << std::endl;
    }
    // 此时控制块也被释放

    return 0;
}
```

**输出：**
```
=== 阶段1: 创建shared_ptr ===
Resource(1) 创建
强引用: 1
Resource(1) 析构
对象已释放: 1

=== 阶段2: weak_ptr生命周期 ===
Resource(2) 创建
sp1强引用: 2
sp1.reset后强引用: 1
Resource(2) 析构
sp2.reset后 expired: 1
```

### 2.2 原子操作保证线程安全

```cpp
#include <iostream>
#include <memory>
#include <thread>
#include <vector>

int main() {
    auto sp = std::make_shared<int>(0);
    std::vector<std::thread> threads;

    // 多线程同时拷贝shared_ptr是安全的
    // 引用计数的增减是原子操作
    for (int i = 0; i < 10; ++i) {
        threads.emplace_back([sp]() {
            auto local = sp; // 原子增加引用计数
            // 但*sp的修改不是线程安全的!
        });
    }

    for (auto& t : threads) t.join();

    std::cout << "最终引用计数: " << sp.use_count() << std::endl;
    return 0;
}
```

**输出：**
```
最终引用计数: 1
```

## 三、注意事项与常见陷阱

- **`make_shared`合并对象和控制块内存**：`weak_ptr`引用时整个块不会释放。
- **引用计数操作是原子的**：但`shared_ptr`对象本身的线程安全需要自行保证。
- **控制块在最后一个`weak_ptr`销毁时释放**：不是最后一个`shared_ptr`。
- **`use_count()`在多线程中不可靠**：仅适用于调试，不能用于同步逻辑。
- **`unique()`（C++17已弃用）**：不能可靠判断是否独占，用`use_count() == 1`代替。
