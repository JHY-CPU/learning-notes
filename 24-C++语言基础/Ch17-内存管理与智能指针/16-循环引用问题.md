# 循环引用问题

## 一、概念说明

当两个`shared_ptr`互相引用时（A持有B的`shared_ptr`，B持有A的`shared_ptr`），引用计数永远不为零，导致内存泄漏。这就是**循环引用**（circular reference）问题。

解决方案：将其中一方改为`weak_ptr`，`weak_ptr`不增加引用计数。

```cpp
#include <iostream>
#include <memory>

class Node {
    std::string name;
public:
    std::shared_ptr<Node> partner; // 危险：循环引用

    Node(std::string n) : name(std::move(n)) {
        std::cout << "Node '" << name << "' 创建" << std::endl;
    }
    ~Node() {
        std::cout << "Node '" << name << "' 销毁" << std::endl;
    }
};

void circularReference() {
    auto a = std::make_shared<Node>("A");
    auto b = std::make_shared<Node>("B");

    a->partner = b; // b的引用计数变为2
    b->partner = a; // a的引用计数变为2

    std::cout << "a引用计数: " << a.use_count() << std::endl;
    std::cout << "b引用计数: " << b.use_count() << std::endl;

    // 离开作用域时，a和b局部变量销毁
    // 但a->partner指向b，b->partner指向a
    // 引用计数各减1后仍为1，不会释放！
}

int main() {
    circularReference();
    std::cout << "函数结束，但对象没有销毁（内存泄漏）" << std::endl;
    return 0;
}
```

**输出：**
```
Node 'A' 创建
Node 'B' 创建
a引用计数: 2
b引用计数: 2
函数结束，但对象没有销毁（内存泄漏）
```

## 二、具体用法

### 2.1 用weak_ptr打破循环

```cpp
#include <iostream>
#include <memory>

class Person {
    std::string name;
public:
    // 用weak_ptr打破循环引用
    std::weak_ptr<Person> spouse;

    Person(std::string n) : name(std::move(n)) {
        std::cout << name << " 出生" << std::endl;
    }
    ~Person() {
        std::cout << name << " 销毁" << std::endl;
    }

    void setSpouse(std::shared_ptr<Person> p) {
        spouse = p; // weak_ptr不增加引用计数
    }

    void showSpouse() const {
        if (auto s = spouse.lock()) {
            std::cout << name << " 的配偶是 " << s->name << std::endl;
        } else {
            std::cout << name << " 的配偶已不存在" << std::endl;
        }
    }
};

int main() {
    auto husband = std::make_shared<Person>("张三");
    auto wife = std::make_shared<Person>("李四");

    husband->setSpouse(wife);
    wife->setSpouse(husband);

    std::cout << "丈夫引用计数: " << husband.use_count() << std::endl;
    std::cout << "妻子引用计数: " << wife.use_count() << std::endl;

    husband->showSpouse();
    wife->showSpouse();

    return 0;
}
```

**输出：**
```
张三 出生
李四 出生
丈夫引用计数: 1
妻子引用计数: 1
张三 的配偶是 李四
李四 的配偶是 张三
李四 销毁
张三 销毁
```

### 2.2 观察者模式中的循环引用

```cpp
#include <iostream>
#include <memory>
#include <vector>

class Subject;

class Observer : public std::enable_shared_from_this<Observer> {
    std::string name;
public:
    Observer(std::string n) : name(std::move(n)) {
        std::cout << "观察者 " << name << " 创建" << std::endl;
    }
    ~Observer() { std::cout << "观察者 " << name << " 销毁" << std::endl; }
    void update() { std::cout << name << " 收到通知" << std::endl; }
    const std::string& getName() const { return name; }
};

class Subject {
    std::vector<std::weak_ptr<Observer>> observers; // weak_ptr避免循环
public:
    void subscribe(std::shared_ptr<Observer> obs) {
        observers.push_back(obs);
    }
    void notify() {
        for (auto it = observers.begin(); it != observers.end(); ) {
            if (auto obs = it->lock()) {
                obs->update();
                ++it;
            } else {
                std::cout << "移除已销毁的观察者" << std::endl;
                it = observers.erase(it);
            }
        }
    }
};

int main() {
    Subject subject;
    {
        auto obs1 = std::make_shared<Observer>("Obs1");
        auto obs2 = std::make_shared<Observer>("Obs2");
        subject.subscribe(obs1);
        subject.subscribe(obs2);
        subject.notify();
    }
    std::cout << "--- 观察者离开作用域后 ---" << std::endl;
    subject.notify();
    return 0;
}
```

**输出：**
```
观察者 Obs1 创建
观察者 Obs2 创建
Obs1 收到通知
Obs2 收到通知
观察者 Obs2 销毁
观察者 Obs1 销毁
--- 观察者离开作用域后 ---
移除已销毁的观察者
移除已销毁的观察者
```

## 三、注意事项与常见陷阱

- **双向关联关系容易产生循环引用**：父子、观察者等场景。
- **至少一方使用`weak_ptr`**：通常是"非拥有"的一方。
- **`enable_shared_from_this`允许对象安全获取自身的`shared_ptr`**。
- **`weak_ptr::lock()`是线程安全的**：可安全判断对象是否存活。
- **检测循环引用**：检查是否有`shared_ptr`的`use_count`始终不为1。
