# 智能指针与异常安全

## 一、概念说明

异常发生时，函数可能在任何位置提前退出，裸指针容易遗漏`delete`导致内存泄漏。智能指针利用RAII（资源获取即初始化）机制，无论函数如何退出（正常返回或异常），析构函数都会被调用，确保资源释放。

```cpp
#include <iostream>
#include <memory>
#include <stdexcept>

class Connection {
    std::string name;
public:
    Connection(std::string n) : name(std::move(n)) {
        std::cout << "Connection '" << name << "' 建立" << std::endl;
    }
    ~Connection() {
        std::cout << "Connection '" << name << "' 关闭" << std::endl;
    }
    void execute() {
        std::cout << "执行查询..." << std::endl;
        if (name == "db2") throw std::runtime_error("查询失败");
    }
};

void unsafeDemo() {
    // 裸指针 - 异常时不安全
    Connection* conn = new Connection("db1");
    conn->execute();
    // 如果上面抛异常，下面的delete不会执行 -> 内存泄漏!
    delete conn;
}

void safeDemo() {
    // 智能指针 - 异常安全
    auto conn = std::make_shared<Connection>("db2");
    conn->execute(); // 抛异常也不怕，conn自动析构
    // 不需要手动delete
}

int main() {
    unsafeDemo();

    try {
        safeDemo();
    } catch (const std::exception& e) {
        std::cout << "捕获异常: " << e.what() << std::endl;
    }

    return 0;
}
```

**输出：**
```
Connection 'db1' 建立
执行查询...
Connection 'db1' 关闭
Connection 'db2' 建立
执行查询...
Connection 'db2' 关闭
捕获异常: 查询失败
```

## 二、具体用法

### 2.1 多资源异常安全

```cpp
#include <iostream>
#include <memory>
#include <stdexcept>

struct File {
    std::string name;
    File(std::string n) : name(std::move(n)) {
        std::cout << "打开文件: " << name << std::endl;
    }
    ~File() { std::cout << "关闭文件: " << name << std::endl; }
};

void processFiles() {
    auto f1 = std::make_unique<File>("data.csv");
    auto f2 = std::make_unique<File>("output.txt");
    auto f3 = std::make_unique<File>("log.txt");

    // 即使这里抛异常，三个文件都会正确关闭
    if (true) throw std::runtime_error("处理出错");

    std::cout << "处理完成" << std::endl;
}

int main() {
    try {
        processFiles();
    } catch (const std::exception& e) {
        std::cout << "异常: " << e.what() << std::endl;
    }
    return 0;
}
```

**输出：**
```
打开文件: data.csv
打开文件: output.txt
打开文件: log.txt
关闭文件: log.txt
关闭文件: output.txt
关闭文件: data.csv
异常: 处理出错
```

### 2.2 构造函数中的异常安全

```cpp
#include <iostream>
#include <memory>

class Server {
    std::unique_ptr<int> socket;
    std::unique_ptr<int> buffer;
public:
    Server(bool fail) {
        socket = std::make_unique<int>(8080);
        std::cout << "socket创建成功" << std::endl;

        if (fail) {
            throw std::runtime_error("buffer分配失败");
            // socket会被自动释放
        }

        buffer = std::make_unique<int>(4096);
        std::cout << "buffer创建成功" << std::endl;
    }

    ~Server() {
        std::cout << "Server析构" << std::endl;
    }
};

int main() {
    try {
        Server server(true);
    } catch (const std::exception& e) {
        std::cout << "构造失败: " << e.what() << std::endl;
    }
    return 0;
}
```

**输出：**
```
socket创建成功
构造失败: buffer分配失败
```

## 三、注意事项与常见陷阱

- **优先使用智能指针管理动态资源**：确保异常时资源释放。
- **多个`make_unique`/`make_shared`调用的顺序**：同一表达式中求值顺序未定义，应分开写。
- **异常从析构函数中不应逃逸**：析构函数应声明`noexcept`。
- **函数异常规范（C++17起）**：析构函数默认`noexcept`。
- **`shared_ptr`的引用计数是异常安全的**：即使在多线程异常场景下。
