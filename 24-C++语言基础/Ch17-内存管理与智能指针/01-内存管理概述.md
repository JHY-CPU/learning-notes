# 内存管理概述

## 一、概念说明

C++程序运行时，内存主要分为以下几个区域：

- **栈区（Stack）**：由编译器自动管理，存放局部变量、函数参数。函数调用结束自动释放，速度快但空间有限。
- **堆区（Heap）**：程序员手动管理（`new`/`delete`），空间大但需要自行释放，否则内存泄漏。
- **静态/全局区**：存放全局变量、静态变量，程序启动时分配，结束时释放。
- **常量区**：存放字符串字面量等常量，只读。
- **代码区**：存放程序的二进制指令。

智能指针（`unique_ptr`、`shared_ptr`、`weak_ptr`）是C++11引入的RAII工具，自动管理堆内存，避免手动`delete`。

```cpp
#include <iostream>
#include <memory>

void demo() {
    // 栈上对象 - 自动管理
    int stackVar = 42;

    // 堆上对象 - 手动管理（危险）
    int* rawPtr = new int(100);
    std::cout << "堆上值: " << *rawPtr << std::endl;
    delete rawPtr; // 必须手动释放

    // 智能指针 - 自动管理（推荐）
    auto smartPtr = std::make_unique<int>(200);
    std::cout << "智能指针值: " << *smartPtr << std::endl;
    // 离开作用域自动释放，无需delete
}

int main() {
    demo();
    return 0;
}
```

**输出：**
```
堆上值: 100
智能指针值: 200
```

## 二、具体用法

### 2.1 各内存区域对比

| 区域 | 管理方式 | 分配时机 | 生命周期 | 大小限制 |
|------|---------|---------|---------|---------|
| 栈区 | 自动 | 运行时 | 函数作用域 | 较小（几MB） |
| 堆区 | 手动/智能指针 | 运行时 | 程员控制 | 很大（GB级） |
| 静态区 | 自动 | 编译期 | 程序全程 | 中等 |
| 常量区 | 只读 | 编译期 | 程序全程 | 中等 |

### 2.2 内存管理的核心原则

```cpp
#include <iostream>

// RAII原则：资源获取即初始化
class Resource {
    int* data;
public:
    Resource() : data(new int[100]) {
        std::cout << "资源已获取" << std::endl;
    }
    ~Resource() {
        delete[] data;
        std::cout << "资源已释放" << std::endl;
    }
};

void useResource() {
    Resource res; // 构造时获取资源
    // 使用资源...
} // 析构时自动释放

int main() {
    useResource();
    return 0;
}
```

**输出：**
```
资源已获取
资源已释放
```

## 三、注意事项与常见陷阱

- **忘记`delete`**：导致内存泄漏，应优先使用智能指针。
- **重复`delete`**：同一指针释放两次会导致未定义行为。
- **悬挂指针**：`delete`后指针未置空，访问已释放内存。
- **栈溢出**：递归过深或栈上分配过大数组会导致栈溢出。
- **`new[]`与`delete`不匹配**：数组形式必须用`delete[]`释放。
- **跨模块内存管理**：不同DLL中`new`/`delete`可能使用不同堆。
