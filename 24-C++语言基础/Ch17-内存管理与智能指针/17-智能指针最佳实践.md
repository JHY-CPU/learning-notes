# 智能指针最佳实践

## 一、概念说明

使用智能指针时应遵循以下核心原则：
1. **优先使用`make_unique`/`make_shared`**而非直接`new`
2. **单一所有权用`unique_ptr`**，共享所有权才用`shared_ptr`
3. **按值传递`unique_ptr`表示所有权转移**，按const引用传递`shared_ptr`表示共享
4. **避免裸指针和智能指针混用**

```cpp
#include <iostream>
#include <memory>

class Logger {
    std::string name;
public:
    Logger(std::string n) : name(std::move(n)) {
        std::cout << "Logger '" << name << "' 创建" << std::endl;
    }
    ~Logger() { std::cout << "Logger '" << name << "' 销毁" << std::endl; }
    void log(const std::string& msg) const {
        std::cout << "[" << name << "] " << msg << std::endl;
    }
};

// 工厂函数返回unique_ptr（最佳实践）
std::unique_ptr<Logger> createLogger(const std::string& name) {
    return std::make_unique<Logger>(name);
}

// 接收unique_ptr表示获取所有权
void processLogger(std::unique_ptr<Logger> logger) {
    logger->log("处理中...");
    // logger在此处自动释放
}

// 接收const引用表示不获取所有权
void useLogger(const std::shared_ptr<Logger>& logger) {
    logger->log("使用中...");
}

int main() {
    // 工厂模式
    auto logger = createLogger("App");
    logger->log("应用启动");

    // 转移所有权
    processLogger(std::move(logger));
    // logger现在为空

    return 0;
}
```

**输出：**
```
Logger 'App' 创建
[App] 应用启动
[App] 处理中...
Logger 'App' 销毁
```

## 二、具体用法

### 2.1 函数参数传递策略

```cpp
#include <iostream>
#include <memory>

struct Data {
    int value;
    Data(int v) : value(v) {
        std::cout << "Data(" << value << ") 构造" << std::endl;
    }
    ~Data() { std::cout << "Data(" << value << ") 析构" << std::endl; }
};

// 不获取所有权：用裸指针或引用
void readOnly(Data* data) {
    std::cout << "只读访问: " << data->value << std::endl;
}

// 获取所有权：按值传递unique_ptr
void takeOwnership(std::unique_ptr<Data> data) {
    std::cout << "获取所有权: " << data->value << std::endl;
    // data在此释放
}

// 共享所有权：用const引用传递shared_ptr
void shareAccess(const std::shared_ptr<Data>& data) {
    std::cout << "共享访问: " << data->value << std::endl;
}

int main() {
    auto data = std::make_unique<Data>(42);

    // 只读：不转移所有权
    readOnly(data.get());

    // 共享：创建shared_ptr
    auto shared = std::make_shared<Data>(99);
    shareAccess(shared);

    // 转移所有权
    takeOwnership(std::move(data));

    return 0;
}
```

**输出：**
```
Data(42) 构造
只读访问: 42
Data(99) 构造
共享访问: 99
获取所有权: 42
Data(42) 析构
Data(99) 析构
```

### 2.2 容器中使用智能指针

```cpp
#include <iostream>
#include <memory>
#include <vector>

struct Item {
    int id;
    Item(int i) : id(i) { std::cout << "Item(" << id << ")" << std::endl; }
    ~Item() { std::cout << "~Item(" << id << ")" << std::endl; }
};

int main() {
    // unique_ptr容器：独占所有权
    std::vector<std::unique_ptr<Item>> owned;
    owned.push_back(std::make_unique<Item>(1));
    owned.push_back(std::make_unique<Item>(2));

    // shared_ptr容器：共享所有权
    std::vector<std::shared_ptr<Item>> shared;
    auto item3 = std::make_shared<Item>(3);
    shared.push_back(item3); // 引用计数+1
    shared.push_back(item3); // 引用计数+1

    std::cout << "item3引用计数: " << item3.use_count() << std::endl;

    return 0;
}
```

**输出：**
```
Item(1)
Item(2)
Item(3)
item3引用计数: 3
~Item(3)
~Item(2)
~Item(1)
```

## 三、注意事项与常见陷阱

- **`make_shared`无法使用私有构造函数**：需用`shared_ptr(new T())`代替。
- **`make_shared`将对象和控制块连续分配**：对象被`weak_ptr`引用时不会释放内存。
- **不要从`unique_ptr::get()`创建新的智能指针**：会导致双重释放。
- **不要在容器中存储`auto_ptr`**：已被C++11弃用。
- **性能敏感场景避免不必要的`shared_ptr`拷贝**：引用计数操作有开销。
