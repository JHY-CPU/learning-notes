# 内存对齐分配

## 一、概念说明

内存对齐是指数据在内存中的起始地址必须是某个值的整数倍，这对CPU访问效率至关重要。未对齐的内存访问可能导致性能下降甚至硬件错误。

`alignof`查询类型的对齐要求，`alignas`指定变量的对齐方式。C++17引入了`std::aligned_alloc`用于分配对齐内存。

```cpp
#include <iostream>
#include <cstddef>

struct Normal {
    char c;     // 1字节
    int i;      // 4字节
    double d;   // 8字节
};

struct alignas(16) Aligned {
    char c;
    int i;
    double d;
};

int main() {
    std::cout << "char 对齐: " << alignof(char) << std::endl;
    std::cout << "int 对齐: " << alignof(int) << std::endl;
    std::cout << "double 对齐: " << alignof(double) << std::endl;
    std::cout << "Normal 对齐: " << alignof(Normal) << ", 大小: " << sizeof(Normal) << std::endl;
    std::cout << "Aligned 对齐: " << alignof(Aligned) << ", 大小: " << sizeof(Aligned) << std::endl;

    return 0;
}
```

**输出：**
```
char 对齐: 1
int 对齐: 4
double 对齐: 8
Normal 对齐: 8, 大小: 16
Aligned 对齐: 16, 大小: 16
```

## 二、具体用法

### 2.1 C++17 aligned_alloc

```cpp
#include <iostream>
#include <cstdlib>
#include <cstddef>

int main() {
    // 分配对齐到64字节的内存（适合SIMD/缓存行）
    size_t alignment = 64;
    size_t size = 1024;

    // size必须是alignment的整数倍
    void* ptr = std::aligned_alloc(alignment, size);

    if (ptr) {
        std::cout << "分配地址: " << ptr << std::endl;
        std::cout << "地址对齐检查: "
                  << ((reinterpret_cast<uintptr_t>(ptr) % alignment == 0) ? "正确" : "错误")
                  << std::endl;

        int* arr = static_cast<int*>(ptr);
        arr[0] = 42;
        std::cout << "arr[0] = " << arr[0] << std::endl;

        std::free(ptr); // 用free释放
    }

    return 0;
}
```

**输出：**
```
分配地址: 0x...
地址对齐检查: 正确
arr[0] = 42
```

### 2.2 C++17 对齐new

```cpp
#include <iostream>
#include <new>

struct alignas(32) CacheLine {
    int data[8]; // 32字节，对齐到缓存行
};

int main() {
    // C++17: 对齐new自动处理对齐要求
    CacheLine* cl = new CacheLine();
    std::cout << "CacheLine 地址对齐: "
              << ((reinterpret_cast<uintptr_t>(cl) % 32 == 0) ? "32字节对齐" : "未对齐")
              << std::endl;
    std::cout << "CacheLine 大小: " << sizeof(CacheLine) << std::endl;

    delete cl;
    return 0;
}
```

**输出：**
```
CacheLine 地址对齐: 32字节对齐
CacheLine 大小: 32
```

## 三、注意事项与常见陷阱

- **`std::aligned_alloc`的size必须是alignment的整数倍**，否则行为未定义。
- **超大对齐值（超过`__STDCPP_DEFAULT_NEW_ALIGNMENT__`）**可能需要特殊的分配/释放函数。
- **`alignas`可以用于变量、结构体和`new`表达式**。
- **SIMD指令要求特定对齐**：如SSE要求16字节对齐，AVX要求32字节对齐。
- **过度对齐可能浪费内存**：应根据实际硬件需求选择合适的对齐值。
- **`aligned_alloc`用`free`释放**，不能混用`delete`。
