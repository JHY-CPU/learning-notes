# 共享内存

## 一、概念说明

共享内存是最快的进程间通信（IPC）方式，多个进程将同一块物理内存映射到各自的地址空间。适合大数据传输、高性能计算等场景。

- **POSIX**：`shm_open()`/`shm_unlink()` + `mmap()`
- **Windows**：`CreateFileMapping()`/`MapViewOfFile()`
- **Boost.Interprocess**：跨平台的高级封装

```cpp
// POSIX共享内存示例
#include <iostream>
#include <sys/mman.h>
#include <sys/stat.h>
#include <fcntl.h>
#include <unistd.h>
#include <cstring>

int main() {
    const char* name = "/my_shm";
    const size_t SIZE = 1024;

    // 创建共享内存对象
    int shm_fd = shm_open(name, O_CREAT | O_RDWR, 0666);
    ftruncate(shm_fd, SIZE);

    // 映射到地址空间
    void* ptr = mmap(nullptr, SIZE, PROT_READ | PROT_WRITE, MAP_SHARED, shm_fd, 0);

    // 写入数据
    std::strcpy(static_cast<char*>(ptr), "Hello from Process A!");
    std::cout << "写入共享内存: " << static_cast<char*>(ptr) << std::endl;

    // 清理
    munmap(ptr, SIZE);
    close(shm_fd);
    // shm_unlink(name); // 另一个进程读完后再unlink

    return 0;
}
```

**输出：**
```
写入共享内存: Hello from Process A!
```

### 2.2 共享内存中的同步

```cpp
#include <iostream>
// 概念性示例：共享内存 + 互斥锁

/*
// 共享内存中放置数据和同步原语
struct SharedData {
    boost::interprocess::interprocess_mutex mutex;
    char message[256];
    bool ready;
};

// 创建端
shared_memory_object shm(create_only, "SyncMem", read_write);
shm.truncate(sizeof(SharedData));
mapped_region region(shm, read_write);
SharedData* data = static_cast<SharedData*>(region.get_address());

scoped_lock<interprocess_mutex> lock(data->mutex);
std::strcpy(data->message, "安全共享的消息");
data->ready = true;
*/

int main() {
    std::cout << "共享内存同步方案:" << std::endl;
    std::cout << "1. interprocess_mutex 进程间互斥锁" << std::endl;

}
```

## 三、注意事项与常见陷阱

- **共享内存需要进程间同步**：没有同步机制会导致数据竞争。
- **忘记`shm_unlink`/`remove`**：共享内存对象会持久存在。
- **共享内存中的对象不能有虚函数表指针**：不同进程地址不同。
- **使用placement new在共享内存中构造对象**：对象的生命周期跨越进程。
- **共享内存大小在创建时固定**：不能动态扩展。
- **Windows共享内存用`CreateFileMapping(INVALID_HANDLE_VALUE,...)`**。
