# 垃圾回收对比

## 一、概念说明

C++没有内置垃圾回收（GC），而是依赖：
1. **RAII**（资源获取即初始化）：构造时获取，析构时释放
2. **引用计数**：`shared_ptr`自动管理
3. **手动管理**：`new`/`delete`

引用计数（`shared_ptr`）是C++中最接近GC的机制，但它有循环引用问题。追踪式GC（如Java/C#）通过标记-清除算法自动处理循环引用，但有停顿时间。

```cpp
#include <iostream>
#include <memory>

// C++方式：RAII + 智能指针
class FileHandle {
    FILE* fp;
public:
    FileHandle(const char* name) : fp(std::fopen(name, "w")) {
        if (!fp) throw std::runtime_error("打开文件失败");
        std::cout << "文件打开" << std::endl;
    }
    ~FileHandle() {
        if (fp) {
            std::fclose(fp);
            std::cout << "文件关闭（RAII自动管理）" << std::endl;
        }
    }
    void write(const char* text) {
        std::fputs(text, fp);
    }
};

int main() {
    {
        FileHandle file("output.txt");
        file.write("Hello from RAII");
    } // 自动关闭，无需GC

    // 智能指针引用计数方式
    auto sp1 = std::make_shared<int>(42);
    auto sp2 = sp1;
    std::cout << "引用计数: " << sp1.use_count() << std::endl;
    sp1.reset();
    sp2.reset(); // 计数为0，自动释放

    return 0;
}
```

**输出：**
```
文件打开
文件关闭（RAII自动管理）
引用计数: 2
```

## 二、具体用法

### 2.1 引用计数 vs 追踪式GC

| 特性 | 引用计数（C++） | 追踪式GC（Java/C#） |
|------|---------------|-------------------|
| 释放时机 | 立即（引用归零时） | 不确定（GC运行时） |
| 循环引用 | 需手动处理（weak_ptr） | 自动处理 |
| 停顿时间 | 无 | 有（STW暂停） |
| 内存开销 | 每对象一个计数器 | 额外的GC元数据 |
| 确定性释放 | 是（RAII保证） | 否（finalize不确定） |

### 2.2 模拟简单的引用计数GC

```cpp
#include <iostream>
#include <memory>

// 简单的引用计数"GC"概念演示
class RefCounted {
    mutable std::atomic<int> refCount{0};

public:
    virtual ~RefCounted() {
        std::cout << "RefCounted析构" << std::endl;
    }

    void retain() const { ++refCount; }

    void release() const {
        if (--refCount == 0) {
            delete this;
        }
    }

    int count() const { return refCount; }
};

class Node : public RefCounted {
    std::string name;
    Node* child = nullptr;

public:
    Node(std::string n) : name(std::move(n)) {
        std::cout << "Node '" << name << "' 创建" << std::endl;
    }

    ~Node() override {
        std::cout << "Node '" << name << "' 销毁" << std::endl;
        if (child) child->release();
    }

    void setChild(Node* c) {
        if (child) child->release();
        child = c;
        if (child) child->retain();
    }
};

int main() {
    auto parent = new Node("Parent");
    auto child = new Node("Child");

    parent->retain();
    child->retain();

    parent->setChild(child);
    std::cout << "Parent引用: " << parent->count() << std::endl;

    parent->release(); // 释放parent，连带释放child

    return 0;
}
```

**输出：**
```
Node 'Parent' 创建
Node 'Child' 创建
Parent引用: 1
Node 'Child' 销毁
Node 'Parent' 销毁
RefCounted析构
```

## 三、注意事项与常见陷阱

- **C++引用计数不处理循环引用**：需配合`weak_ptr`。
- **引用计数有性能开销**：每次拷贝/销毁都修改计数。
- **RAII是C++的核心优势**：确定性资源释放，比GC更适合系统编程。
- **`shared_ptr`的原子引用计数有缓存竞争**：多线程频繁拷贝影响性能。
- **C++支持精确的资源管理**：文件、锁、网络连接等，不仅限于内存。
