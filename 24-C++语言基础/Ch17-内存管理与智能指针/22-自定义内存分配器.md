# 自定义内存分配器

## 一、概念说明

STL容器通过模板参数接受自定义分配器，可以替换默认的`std::allocator`。自定义分配器适用于：内存池优化、共享内存、调试内存分配等场景。

实现一个STL兼容分配器需提供：`value_type`、`allocate`、`deallocate`、`construct`、`destroy`（或依赖`allocator_traits`默认实现）。

```cpp
#include <iostream>
#include <memory>
#include <vector>

// 统计分配次数的分配器
template <typename T>
class CountingAllocator {
public:
    using value_type = T;
    static int allocCount;
    static int deallocCount;

    CountingAllocator() = default;

    template <typename U>
    CountingAllocator(const CountingAllocator<U>&) {}

    T* allocate(size_t n) {
        ++allocCount;
        std::cout << "分配 " << n << " 个 " << typeid(T).name()
                  << " (总计: " << allocCount << ")" << std::endl;
        return static_cast<T*>(::operator new(n * sizeof(T)));
    }

    void deallocate(T* p, size_t n) {
        ++deallocCount;
        ::operator delete(p);
    }
};

template <typename T> int CountingAllocator<T>::allocCount = 0;
template <typename T> int CountingAllocator<T>::deallocCount = 0;

template <typename T, typename U>
bool operator==(const CountingAllocator<T>&, const CountingAllocator<U>&) { return true; }
template <typename T, typename U>
bool operator!=(const CountingAllocator<T>&, const CountingAllocator<U>&) { return false; }

int main() {
    std::vector<int, CountingAllocator<int>> vec;
    vec.push_back(1);
    vec.push_back(2);
    vec.push_back(3);
    vec.push_back(4);
    vec.push_back(5);

    std::cout << "\n最终分配次数: " << CountingAllocator<int>::allocCount << std::endl;
    return 0;
}
```

**输出：**
```
分配 1 个 i (总计: 1)
分配 2 个 i (总计: 2)
分配 4 个 i (总计: 3)
分配 8 个 i (总计: 4)

最终分配次数: 4
```

## 二、具体用法

### 2.1 内存池分配器

```cpp
#include <iostream>
#include <memory>
#include <vector>
#include <list>

// 基于内存池的分配器
template <typename T, size_t BlockSize = 4096>
class PoolAllocator {
    struct Pool {
        char buffer[BlockSize];
        size_t used = 0;
        Pool* next = nullptr;
    };

    Pool* head = nullptr;

public:
    using value_type = T;

    PoolAllocator() = default;

    template <typename U>
    PoolAllocator(const PoolAllocator<U, BlockSize>&) {}

    T* allocate(size_t n) {
        size_t bytes = n * sizeof(T);
        if (!head || head->used + bytes > BlockSize) {
            Pool* newPool = new Pool();
            newPool->next = head;
            head = newPool;
        }
        T* ptr = reinterpret_cast<T*>(head->buffer + head->used);
        head->used += bytes;
        return ptr;
    }

    void deallocate(T*, size_t) {
        // 池分配器通常在析构时统一释放
    }

    ~PoolAllocator() {
        while (head) {
            Pool* next = head->next;
            delete head;
            head = next;
        }
    }
};

int main() {
    std::list<int, PoolAllocator<int>> lst;
    for (int i = 0; i < 100; ++i)
        lst.push_back(i);

    std::cout << "列表大小: " << lst.size() << std::endl;
    std::cout << "使用池分配器，减少malloc调用次数" << std::endl;

    return 0;
}
```

**输出：**
```
列表大小: 100
使用池分配器，减少malloc调用次数
```

## 三、注意事项与常见陷阱

- **分配器必须是无状态的或可复制的**：容器拷贝时会复制分配器。
- **`allocate`返回的内存必须对齐**：`operator new`保证足够对齐。
- **`deallocate`在池分配器中可能为空操作**：池析构时统一释放。
- **分配器的`==`运算符影响容器比较**：相等的分配器内存可互换。
- **C++17起`construct`/`destroy`从分配器移除**：通过`allocator_traits`使用。
- **`std::scoped_allocator_adaptor`支持嵌套容器的分配器传递**。
