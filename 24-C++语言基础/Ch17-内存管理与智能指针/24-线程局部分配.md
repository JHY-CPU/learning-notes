# 线程局部分配

## 一、概念说明

线程局部存储（TLS）为每个线程提供独立的内存副本，避免锁竞争。使用`thread_local`关键字声明线程局部变量。

线程局部内存分配器结合TLS和内存池，为每个线程维护独立的内存池，实现无锁分配。

```cpp
#include <iostream>
#include <thread>
#include <vector>
#include <mutex>

// 线程局部变量
thread_local int threadCounter = 0;

// 线程局部内存池
class ThreadLocalPool {
    static constexpr size_t POOL_SIZE = 1024;
    char buffer[POOL_SIZE];
    size_t offset = 0;

public:
    void* allocate(size_t size) {
        if (offset + size > POOL_SIZE) throw std::bad_alloc();
        void* ptr = buffer + offset;
        offset += size;
        return ptr;
    }

    void reset() { offset = 0; }
};

thread_local ThreadLocalPool localPool;

std::mutex coutMutex;

void worker(int id) {
    // 每个线程有独立的计数器
    threadCounter = id * 100;

    // 使用线程局部内存池（无需锁）
    int* data = static_cast<int*>(localPool.allocate(sizeof(int) * 10));
    for (int i = 0; i < 10; ++i) {
        data[i] = threadCounter + i;
    }

    {
        std::lock_guard<std::mutex> lock(coutMutex);
        std::cout << "线程" << id << ": threadCounter=" << threadCounter
                  << ", data[0]=" << data[0] << std::endl;
    }
}

int main() {
    std::vector<std::thread> threads;
    for (int i = 0; i < 4; ++i) {
        threads.emplace_back(worker, i);
    }
    for (auto& t : threads) t.join();

    return 0;
}
```

**输出：**
```
线程0: threadCounter=0, data[0]=0
线程1: threadCounter=100, data[0]=100
线程2: threadCounter=200, data[0]=200
线程3: threadCounter=300, data[0]=300
```

## 二、具体用法

### 2.1 无锁分配器

```cpp
#include <iostream>
#include <thread>
#include <vector>
#include <cstddef>

// 无锁线程局部分配器
class LockFreeAllocator {
    struct Block {
        Block* next;
    };

    static constexpr size_t BLOCK_SIZE = 64;
    static constexpr size_t BLOCKS_PER_SLAB = 1024;

    struct Slab {
        char data[BLOCK_SIZE * BLOCKS_PER_SLAB];
        Slab* next;
    };

    thread_local static Block* freeList;
    thread_local static Slab* slabList;

public:
    static void* allocate() {
        if (!freeList) expandSlab();
        Block* block = freeList;
        freeList = block->next;
        return block;
    }

    static void deallocate(void* ptr) {
        auto* block = static_cast<Block*>(ptr);
        block->next = freeList;
        freeList = block;
    }

private:
    static void expandSlab() {
        auto* slab = new Slab();
        slab->next = slabList;
        slabList = slab;

        for (size_t i = 0; i < BLOCKS_PER_SLAB; ++i) {
            auto* block = reinterpret_cast<Block*>(slab->data + i * BLOCK_SIZE);
            block->next = freeList;
            freeList = block;
        }
    }
};

thread_local LockFreeAllocator::Block* LockFreeAllocator::freeList = nullptr;
thread_local LockFreeAllocator::Slab* LockFreeAllocator::slabList = nullptr;

void testAllocator(int id) {
    std::vector<void*> ptrs;
    for (int i = 0; i < 100; ++i) {
        ptrs.push_back(LockFreeAllocator::allocate());
    }
    for (auto* p : ptrs) {
        LockFreeAllocator::deallocate(p);
    }
}

int main() {
    std::vector<std::thread> threads;
    for (int i = 0; i < 4; ++i) {
        threads.emplace_back(testAllocator, i);
    }
    for (auto& t : threads) t.join();

    std::cout << "4线程各分配释放100个块，无锁竞争" << std::endl;
    return 0;
}
```

**输出：**
```
4线程各分配释放100个块，无锁竞争
```

## 三、注意事项与常见陷阱

- **`thread_local`变量在线程结束时销毁**：析构顺序与构造相反。
- **线程局部变量不计入`use_count`等统计**：需要自行管理。
- **线程局部存储有性能开销**：比普通变量访问略慢。
- **`thread_local`不能用于类的非静态成员**：只能用于全局或静态变量。
- **线程数很多时TLS总量可能很大**：注意每个线程的内存占用。
