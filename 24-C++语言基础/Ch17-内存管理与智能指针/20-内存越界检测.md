# 内存越界检测

## 一、概念说明

内存越界（Buffer Overflow）指访问了分配范围之外的内存，可能导致程序崩溃、数据损坏或安全漏洞。常见场景：数组越界、写入超出分配的缓冲区、使用已释放的内存（use-after-free）。

检测工具：
- **AddressSanitizer（ASan）**：检测堆/栈越界、use-after-free
- **Valgrind/Memcheck**：检测越界和未初始化内存
- **Electric Fence**：通过页保护检测堆越界

```cpp
#include <iostream>

int main() {
    int arr[5] = {1, 2, 3, 4, 5};

    // 正常访问
    std::cout << "arr[4] = " << arr[4] << std::endl;

    // 越界访问（未定义行为！）
    // ASan会报告: stack-buffer-overflow
    // arr[5] = 10;  // 注释掉以防崩溃
    // std::cout << "arr[5] = " << arr[5] << std::endl;

    // 堆越界示例
    int* heap = new int[5];
    heap[0] = 42;
    std::cout << "heap[0] = " << heap[0] << std::endl;

    // heap[5] = 100;  // ASan会报告: heap-buffer-overflow
    delete[] heap;

    return 0;
}
```

**输出：**
```
arr[4] = 5
heap[0] = 42
```

## 二、具体用法

### 2.1 ASan检测各种内存错误

```cpp
#include <iostream>
#include <cstdlib>

// Use-after-free示例
void useAfterFree() {
    int* p = new int(42);
    delete p;
    // 使用已释放的内存（ASan报告: heap-use-after-free）
    // std::cout << *p << std::endl;
}

// Double-free示例
void doubleFree() {
    int* p = new int(42);
    delete p;
    // delete p;  // ASan报告: double-free
}

// Stack-use-after-scope示例
void stackScope() {
    int* dangling;
    {
        int local = 42;
        dangling = &local;
    } // local已销毁
    // *dangling = 100;  // ASan报告: stack-use-after-scope
}

int main() {
    std::cout << "内存越界检测示例" << std::endl;
    useAfterFree();
    doubleFree();
    stackScope();
    return 0;
}
```

**输出：**
```
内存越界检测示例
```

### 2.2 编译和运行ASan

```bash
# GCC/Clang编译
g++ -fsanitize=address -fno-omit-frame-pointer -g -o detect detect.cpp

# MSVC编译
cl /fsanitize=address /Zi detect.cpp

# 运行，ASan会在检测到错误时输出详细报告
./detect

# 输出格式示例：
# =================================================================
# ==12345==ERROR: AddressSanitizer: heap-buffer-overflow on address 0x...
# READ of size 4 at 0x... thread T0
#     #0 0x... in main detect.cpp:15
```

### 2.3 边界检查容器

```cpp
#include <iostream>
#include <vector>
#include <stdexcept>

int main() {
    std::vector<int> vec = {1, 2, 3, 4, 5};

    // 安全访问：at()会进行边界检查
    try {
        std::cout << "vec.at(2) = " << vec.at(2) << std::endl;
        std::cout << "vec.at(10) = " << vec.at(10) << std::endl; // 抛异常
    } catch (const std::out_of_range& e) {
        std::cout << "捕获越界异常: " << e.what() << std::endl;
    }

    // 不安全访问：operator[]不检查边界
    // vec[10] = 42;  // 未定义行为！

    return 0;
}
```

**输出：**
```
vec.at(2) = 3
捕获越界异常: vector::_M_range_check: __n (which is 10) >= this->size() (which is 5)
```

## 三、注意事项与常见陷阱

- **ASan运行时开销约2倍，内存开销约3倍**：不适合生产环境。
- **越界访问不一定会立即崩溃**：可能访问到"幸运"的内存位置。
- **Release模式可能隐藏越界**：Debug模式或ASan更可靠。
- **`std::vector::at()`有边界检查**：性能敏感时用`operator[]`但确保安全。
- **ASan与Valgrind可同时使用**：互补覆盖更多场景。
- **Stack Protector编译选项**：`-fstack-protector`检测栈溢出。
