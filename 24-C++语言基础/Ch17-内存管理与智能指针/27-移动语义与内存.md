# 移动语义与内存

## 一、概念说明

C++11引入的移动语义允许"窃取"临时对象的资源，避免昂贵的深拷贝。右值引用（`T&&`）是实现移动语义的基础，`std::move`将左值转换为右值引用。

移动操作通常将源对象的内部指针置为空，然后将资源所有权转移给目标对象。移动后的源对象处于**有效但未指定**的状态。

```cpp
#include <iostream>
#include <vector>
#include <string>

class Buffer {
    int* data;
    size_t size_;
public:
    // 构造
    Buffer(size_t n) : data(new int[n]), size_(n) {
        std::cout << "构造: 分配" << n << "个int" << std::endl;
    }

    // 拷贝构造（深拷贝）
    Buffer(const Buffer& other) : data(new int[other.size_]), size_(other.size_) {
        std::cout << "拷贝: 深拷贝" << size_ << "个int" << std::endl;
        std::copy(other.data, other.data + size_, data);
    }

    // 移动构造（窃取资源）
    Buffer(Buffer&& other) noexcept : data(other.data), size_(other.size_) {
        other.data = nullptr; // 源对象置空
        other.size_ = 0;
        std::cout << "移动: 窃取资源" << std::endl;
    }

    // 析构
    ~Buffer() {
        std::cout << "析构: " << (data ? "释放内存" : "空对象") << std::endl;
        delete[] data;
    }

    size_t size() const { return size_; }
};

Buffer createBuffer() {
    Buffer buf(100); // 局部对象
    return buf;      // 移动构造返回值（编译器可能优化为NRVO）
}

int main() {
    std::cout << "=== 移动语义演示 ===" << std::endl;
    Buffer a(10);
    Buffer b = std::move(a); // 移动构造
    std::cout << "b.size()=" << b.size() << ", a.size()=" << a.size() << std::endl;

    std::cout << "\n=== 返回值优化 ===" << std::endl;
    Buffer c = createBuffer();

    return 0;
}
```

**输出：**
```
=== 移动语义演示 ===
构造: 分配10个int
移动: 窃取资源
b.size()=10, a.size()=0
析构: 空对象

=== 返回值优化 ===
析构: 空对象
析构: 释放内存
析构: 空对象
```

## 二、具体用法

### 2.1 移动避免容器拷贝

```cpp
#include <iostream>
#include <vector>
#include <string>

int main() {
    // 拷贝：复制所有元素
    std::vector<std::string> v1 = {"Hello", "World", "C++"};
    std::vector<std::string> v2 = v1; // O(n)拷贝
    std::cout << "拷贝后v1.size()=" << v1.size() << std::endl;

    // 移动：转移所有权，O(1)
    std::vector<std::string> v3 = std::move(v1); // O(1)移动
    std::cout << "移动后v1.size()=" << v1.size() << ", v3.size()=" << v3.size() << std::endl;

    // emplace_back：原地构造，避免临时对象
    std::vector<std::string> v4;
    v4.emplace_back(10, 'A'); // 直接在vector中构造string
    std::cout << "emplace: " << v4[0] << std::endl;

    return 0;
}
```

**输出：**
```
拷贝后v1.size()=3
移动后v1.size()=0, v3.size()=3
emplace: AAAAAAAAAA
```

### 2.2 std::forward完美转发

```cpp
#include <iostream>
#include <utility>

void process(int& x) { std::cout << "左值: " << x << std::endl; }
void process(int&& x) { std::cout << "右值: " << x << std::endl; }

template <typename T>
void wrapper(T&& arg) {
    // std::forward保持参数的值类别
    process(std::forward<T>(arg));
}

int main() {
    int x = 42;
    wrapper(x);            // 左值
    wrapper(100);          // 右值
    wrapper(std::move(x)); // 右值
    return 0;
}
```

**输出：**
```
左值: 42
右值: 100
右值: 42
```

## 三、注意事项与常见陷阱

- **移动后源对象处于有效但未指定状态**：只能析构或重新赋值。
- **`std::move`不移动任何东西**：只是类型转换，真正的移动由移动构造/赋值完成。
- **`noexcept`对移动很重要**：`std::vector`在扩容时只使用`noexcept`的移动。
- **POD类型移动等价于拷贝**：`int`、`double`等没有移动收益。
- **`std::forward`仅用于模板中的万能引用**：保持参数的左/右值类别。
