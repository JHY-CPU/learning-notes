# 自定义删除器

## 一、概念说明

智能指针默认使用`delete`释放资源，但有些资源需要特殊释放方式（如关闭文件句柄、释放C库资源）。自定义删除器允许指定释放逻辑。

删除器可以是：函数指针、函数对象（仿函数）、lambda表达式。删除器的类型是`unique_ptr`模板参数的一部分，不影响`shared_ptr`的类型。

```cpp
#include <iostream>
#include <memory>
#include <cstdio>

// 函数指针删除器
void fileCloser(FILE* fp) {
    if (fp) {
        std::cout << "关闭文件" << std::endl;
        std::fclose(fp);
    }
}

// 函数对象删除器
struct FreeDeleter {
    void operator()(void* p) const {
        std::cout << "free释放内存" << std::endl;
        std::free(p);
    }
};

int main() {
    // unique_ptr + 函数指针删除器
    std::unique_ptr<FILE, decltype(&fileCloser)> file(
        std::fopen("test.txt", "w"), fileCloser);

    // unique_ptr + lambda删除器（需decltype）
    auto lambdaDeleter = [](int* p) {
        std::cout << "lambda删除" << std::endl;
        delete p;
    };
    std::unique_ptr<int, decltype(lambdaDeleter)> p1(new int(42), lambdaDeleter);

    // shared_ptr + lambda删除器（不影响类型）
    std::shared_ptr<int> p2(new int(99), [](int* p) {
        std::cout << "shared_ptr lambda删除" << std::endl;
        delete p;
    });

    std::cout << "p1值: " << *p1 << std::endl;
    std::cout << "p2值: " << *p2 << std::endl;

    return 0;
}
```

**输出：**
```
p1值: 42
p2值: 99
shared_ptr lambda删除
lambda删除
关闭文件
```

## 二、具体用法

### 2.1 C资源管理

```cpp
#include <iostream>
#include <memory>

// 管理C库的资源
extern "C" {
    // 假设的C库API
    typedef struct { int fd; } connection_t;
    connection_t* open_conn(const char* addr) {
        std::cout << "打开连接: " << addr << std::endl;
        return new connection_t{42};
    }
    void close_conn(connection_t* conn) {
        std::cout << "关闭连接 fd=" << conn->fd << std::endl;
        delete conn;
    }
}

int main() {
    auto connDeleter = [](connection_t* conn) {
        if (conn) close_conn(conn);
    };

    std::unique_ptr<connection_t, decltype(connDeleter)> conn(
        open_conn("localhost:3306"), connDeleter);

    std::cout << "使用连接 fd=" << conn->fd << std::endl;
    // 离开作用域自动调用close_conn

    return 0;
}
```

**输出：**
```
打开连接: localhost:3306
使用连接 fd=42
关闭连接 fd=42
```

### 2.2 状态删除器

```cpp
#include <iostream>
#include <memory>

class TrackingDeleter {
    int& counter;
public:
    TrackingDeleter(int& c) : counter(c) {}
    void operator()(int* p) {
        ++counter;
        std::cout << "第" << counter << "次删除" << std::endl;
        delete p;
    }
};

int main() {
    int deleteCount = 0;
    TrackingDeleter deleter(deleteCount);

    {
        std::unique_ptr<int, TrackingDeleter> p1(new int(1), deleter);
        std::unique_ptr<int, TrackingDeleter> p2(new int(2), deleter);
    }

    std::cout << "总共删除了 " << deleteCount << " 个对象" << std::endl;
    return 0;
}
```

**输出：**
```
第1次删除
第2次删除
总共删除了 2 个对象
```

## 三、注意事项与常见陷阱

- **`unique_ptr`的删除器是类型的一部分**：不同删除器类型不同，不能互相赋值。
- **`shared_ptr`的删除器不是类型的一部分**：可以自由替换删除器。
- **Lambda删除器需要`decltype`**：作为`unique_ptr`模板参数时。
- **空删除器（no-op）也合法**：适用于栈上对象或全局对象。
- **删除器应处理空指针**：避免对`nullptr`执行非法操作。
