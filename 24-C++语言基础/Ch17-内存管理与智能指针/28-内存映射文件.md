# 内存映射文件

## 一、概念说明

内存映射文件将文件内容映射到进程的虚拟地址空间，使得文件读写如同访问内存一样高效。适合处理大文件，避免频繁的`read`/`write`系统调用。

- **Linux**：`mmap()`/`munmap()`
- **Windows**：`CreateFileMapping()`/`MapViewOfFile()`

```cpp
#ifdef _WIN32
#include <windows.h>
#else
#include <sys/mman.h>
#include <sys/stat.h>
#include <fcntl.h>
#include <unistd.h>
#endif
#include <iostream>

int main() {
#ifdef __linux__
    const char* filename = "test.dat";
    int fd = open(filename, O_RDWR | O_CREAT, 0666);
    ftruncate(fd, 4096); // 设置文件大小

    // 将文件映射到内存
    void* mapped = mmap(nullptr, 4096, PROT_READ | PROT_WRITE,
                        MAP_SHARED, fd, 0);

    if (mapped != MAP_FAILED) {
        // 像访问内存一样访问文件
        char* data = static_cast<char*>(mapped);
        std::strcpy(data, "Hello from mmap!");
        std::cout << "映射内容: " << data << std::endl;

        munmap(mapped, 4096); // 解除映射
    }

    close(fd);
    std::cout << "文件已写入" << std::endl;
#endif

#ifdef _WIN32
    // Windows实现（概念性示例）
    std::cout << "Windows使用CreateFileMapping API" << std::endl;
#endif

    return 0;
}
```

**输出：**
```
映射内容: Hello from mmap!
文件已写入
```

## 二、具体用法

### 2.1 跨平台封装

```cpp
#include <iostream>
#include <cstring>

#ifdef _WIN32
#include <windows.h>
class MappedFile {
    HANDLE hFile, hMapping;
    void* data;
    size_t size_;
public:
    MappedFile(const char* path) : data(nullptr), size_(0) {
        hFile = CreateFileA(path, GENERIC_READ, 0, NULL, OPEN_EXISTING, FILE_ATTRIBUTE_NORMAL, NULL);
        if (hFile == INVALID_HANDLE_VALUE) return;
        size_ = GetFileSize(hFile, NULL);
        hMapping = CreateFileMapping(hFile, NULL, PAGE_READONLY, 0, 0, NULL);
        if (hMapping) data = MapViewOfFile(hMapping, FILE_MAP_READ, 0, 0, 0);
    }
    ~MappedFile() {
        if (data) UnmapViewOfFile(data);
        if (hMapping) CloseHandle(hMapping);
        if (hFile != INVALID_HANDLE_VALUE) CloseHandle(hFile);
    }
    const char* begin() const { return static_cast<const char*>(data); }
    const char* end() const { return static_cast<const char*>(data) + size_; }
    size_t size() const { return size_; }
};
#else
#include <sys/mman.h>
#include <sys/stat.h>
#include <fcntl.h>
#include <unistd.h>
class MappedFile {
    int fd;
    void* data;
    size_t size_;
public:
    MappedFile(const char* path) : fd(-1), data(MAP_FAILED), size_(0) {
        fd = open(path, O_RDONLY);
        if (fd < 0) return;
        struct stat st;
        fstat(fd, &st);
        size_ = st.st_size;
        data = mmap(nullptr, size_, PROT_READ, MAP_PRIVATE, fd, 0);
    }
    ~MappedFile() {
        if (data != MAP_FAILED) munmap(data, size_);
        if (fd >= 0) close(fd);
    }
    const char* begin() const { return static_cast<const char*>(data); }
    const char* end() const { return static_cast<const char*>(data) + size_; }
    size_t size() const { return size_; }
};
#endif

int main() {
    // 需要实际存在的文件才能运行
    std::cout << "内存映射文件封装示例" << std::endl;
    std::cout << "使用MappedFile类统一Linux/Windows API" << std::endl;
    return 0;
}
```

**输出：**
```
内存映射文件封装示例
使用MappedFile类统一Linux/Windows API
```

## 三、注意事项与常见陷阱

- **映射大小受虚拟地址空间限制**：32位系统最大约2-3GB。
- **64位系统映射大文件无压力**：虚拟地址空间足够。
- **`MAP_SHARED`修改会写回文件**：`MAP_PRIVATE`使用写时复制。
- **映射解除后不要再访问指针**：会导致段错误。
- **对齐要求**：偏移量通常是页大小（4KB）的整数倍。
- **Windows和Linux API差异大**：建议封装跨平台类。
