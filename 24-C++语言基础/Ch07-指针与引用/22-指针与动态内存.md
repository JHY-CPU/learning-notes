# 指针与动态内存

## 一、概念说明

动态内存通过`new`/`delete`在**堆**上分配和释放内存，与栈上分配不同，动态内存的生命周期由程序员控制。C++推荐使用智能指针自动管理动态内存。

## 二、具体用法

### 2.1 new / delete

```cpp
// 分配单个对象
int* p = new int(42);
std::cout << *p << std::endl;  // 输出: 42
delete p;  // 释放

// 分配数组
int* arr = new int[100];
for (int i = 0; i < 100; ++i) arr[i] = i;
std::cout << arr[50] << std::endl;  // 输出: 50
delete[] arr;  // 用delete[]释放数组
```

### 2.2 内存泄漏

```cpp
// 危险：忘记释放导致内存泄漏
void leak() {
    int* p = new int(42);
    // 忘记 delete p;
    // 函数返回后p丢失，内存永远无法释放
}

// 正确：RAII管理
void safe() {
    auto p = std::make_unique<int>(42);
    // 自动释放
}
```

### 2.3 异常安全

```cpp
// 不安全：异常可能导致泄漏
void unsafe() {
    int* a = new int(1);
    riskyOperation();  // 可能抛异常
    int* b = new int(2);
    delete a;
    delete b;
}

// 安全：智能指针
void safe() {
    auto a = std::make_unique<int>(1);
    riskyOperation();  // 异常时a自动释放
    auto b = std::make_unique<int>(2);
}
```

### 2.4 placement new

```cpp
#include <new>

alignas(int) char buffer[sizeof(int)];  // 预分配缓冲区

// 在已有内存中构造对象
int* p = new (buffer) int(42);
std::cout << *p << std::endl;  // 输出: 42

// 手动析构（不释放内存）
p->~int();
```

### 2.5 内存分配失败

```cpp
// new失败默认抛出 std::bad_alloc
try {
    int* p = new int[1000000000000LL];  // 尝试分配极大内存
} catch (const std::bad_alloc& e) {
    std::cout << "分配失败: " << e.what() << std::endl;
}

// nothrow版本：失败返回nullptr
int* p = new(std::nothrow) int[1000000000000LL];
if (!p) std::cout << "分配失败\n";
```

## 三、注意事项与常见陷阱

- `new[]`必须用`delete[]`释放，`new`用`delete`释放
- 同一指针`delete`两次是未定义行为
- `delete`后应将指针设为`nullptr`
- 优先使用智能指针，避免手动管理内存
- `new`失败抛异常，`new(nothrow)`失败返回`nullptr`
