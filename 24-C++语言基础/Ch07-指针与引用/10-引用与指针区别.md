# 引用与指针区别

## 一、概念说明

引用和指针都提供间接访问，但有本质区别：引用是别名，指针是存储地址的变量。引用必须初始化、不可为空、不可重新绑定；指针可以为空、可以重新指向。

## 二、具体用法

### 2.1 语法对比

```cpp
int x = 42;

// 指针：需要取地址&和解引用*
int* ptr = &x;
*ptr = 50;
std::cout << x << std::endl;  // 输出: 50

// 引用：直接使用，无需特殊运算符
int& ref = x;
ref = 60;
std::cout << x << std::endl;  // 输出: 60
```

### 2.2 可重新绑定 vs 不可

```cpp
int a = 10, b = 20;

// 指针可以重新指向
int* p = &a;
p = &b;  // OK
std::cout << *p << std::endl;  // 输出: 20

// 引用不可重新绑定
int& ref = a;
// ref绑定到a后永远是a的别名
ref = b;  // 这是赋值，a变为20
std::cout << a << std::endl;  // 输出: 20
```

### 2.3 函数参数场景

```cpp
// 使用引用（推荐）：语法简洁，不可为空
void swap(int& a, int& b) {
    int temp = a;
    a = b;
    b = temp;
}

// 使用指针：需要检查空指针
void swapPtr(int* a, int* b) {
    if (!a || !b) return;  // 需要空检查
    int temp = *a;
    *a = *b;
    *b = temp;
}

int x = 1, y = 2;
swap(x, y);
// swapPtr(&x, &y);  // 指针版本
std::cout << x << " " << y << std::endl;  // 输出: 2 1
```

### 2.4 使用场景选择

```cpp
// 优先引用的场景
void process(const std::string& str);  // 大对象参数
int& getAt(size_t i);                   // 返回容器元素

// 必须用指针的场景
int* ptr = nullptr;           // 可能为空
ptr = new int(42);            // 动态分配
delete ptr;

// 多态场景
Shape* shape = new Circle();  // 需要可空
delete shape;
```

## 三、注意事项与常见陷阱

- 引用更安全（不可能为空），指针更灵活（可为空、可重新绑定）
- 引用的底层实现通常是常量指针，但语义不同
- 数组元素不能是引用（引用不能构成数组）
- 引用没有多级引用（`int&&`是右值引用，不是引用的引用）
- 函数参数默认优先使用`const T&`，需要修改时用`T&`
