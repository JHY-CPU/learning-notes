# 指针对齐

## 一、概念说明

内存对齐（Alignment）是指数据在内存中的地址必须是某个值（通常是类型的大小）的整数倍。对齐影响**内存布局**、**访问性能**，以及某些指令（如SIMD）的正确性。

`alignof`查询类型的对齐要求，`alignas`指定变量或类型的对齐。

## 二、具体用法

### 2.1 查看对齐要求

```cpp
std::cout << "char对齐: " << alignof(char) << std::endl;      // 输出: 1
std::cout << "int对齐: " << alignof(int) << std::endl;        // 输出: 4
std::cout << "double对齐: " << alignof(double) << std::endl;  // 输出: 8
std::cout << "指针对齐: " << alignof(int*) << std::endl;      // 输出: 8 (64位)

struct S {
    char c;    // 1字节 + 3字节填充
    int i;     // 4字节
};
std::cout << "S大小: " << sizeof(S) << std::endl;     // 输出: 8
std::cout << "S对齐: " << alignof(S) << std::endl;    // 输出: 4
```

### 2.2 alignas指定对齐

```cpp
// 指定更大对齐
alignas(16) int data[4];  // 16字节对齐

// SIMD友好对齐
alignas(32) float simdData[8];

// 对齐检查
std::cout << "data对齐: " << alignof(decltype(data)) << std::endl;  // 输出: 16
std::cout << "地址: " << (uintptr_t)data % 16 << std::endl;         // 输出: 0
```

### 2.3 aligned_alloc（C++17）

```cpp
#include <cstdlib>

// 分配对齐内存
void* ptr = std::aligned_alloc(64, 256);  // 64字节对齐，256字节
if (ptr) {
    std::cout << "对齐检查: " << ((uintptr_t)ptr % 64) << std::endl;  // 输出: 0
    std::free(ptr);
}
```

### 2.4 结构体对齐控制

```cpp
// 紧凑排列（无填充）
#pragma pack(push, 1)
struct Packed {
    char c;
    int i;
};
#pragma pack(pop)

std::cout << "Packed大小: " << sizeof(Packed) << std::endl;  // 输出: 5

// 对齐到指定值
struct alignas(16) Aligned {
    int x, y;
};
std::cout << "Aligned大小: " << sizeof(Aligned) << std::endl;  // 输出: 16
```

## 三、注意事项与常见陷阱

- 对齐不当可能导致性能下降或总线错误（某些架构上）
- `alignas`只能指定比自然对齐更大的值
- SIMD操作（SSE/AVX）要求特定对齐（16/32/64字节）
- `#pragma pack`是编译器扩展，可移植性差
- 过度对齐浪费内存，需在性能和空间之间权衡
