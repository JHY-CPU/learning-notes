# 悬垂指针

## 一、概念说明

**悬垂指针**（Dangling Pointer）指向已被释放或已销毁的对象。使用悬垂指针是**未定义行为**，可能导致程序崩溃、数据损坏或安全漏洞。悬垂引用是引用的等价问题。

## 二、常见成因与示例

### 2.1 返回局部变量的指针/引用

```cpp
// 危险！
int* bad() {
    int local = 42;
    return &local;  // 返回局部变量地址
}  // local销毁，返回的指针成为悬垂指针

// 正确方式
int good() {
    int local = 42;
    return local;  // 返回值，有拷贝
}
```

### 2.2 使用已释放的内存

```cpp
int* p = new int(42);
delete p;
// p现在是悬垂指针
// std::cout << *p;  // 未定义行为！

// 解决：释放后置空
delete p;
p = nullptr;
if (p) std::cout << *p;  // 安全检查
```

### 2.3 容器扩容失效

```cpp
std::vector<int> v = {1, 2, 3};
int* p = &v[1];  // 获取元素地址

v.push_back(4);  // 可能触发扩容，p失效！
// std::cout << *p;  // 未定义行为

// 解决：使用索引而非指针
size_t index = 1;
v.push_back(4);
std::cout << v[index] << std::endl;  // 安全
```

### 2.4 悬垂引用

```cpp
// 返回局部string的引用
const std::string& badRef() {
    std::string s = "Hello";
    return s;  // 悬垂引用
}

// 使用悬垂引用
// const std::string& ref = badRef();
// std::cout << ref;  // 未定义行为
```

### 2.5 生命周期管理

```cpp
struct Data { int value; };

std::unique_ptr<Data> owner = std::make_unique<Data>(42);
Data* observer = owner.get();  // 获取观察用的裸指针

std::cout << observer->value << std::endl;  // 输出: 42

owner.reset();  // owner释放对象
// observer现在是悬垂指针！
// observer->value;  // 未定义行为
```

## 三、避免方法

- 返回值而非指针/引用
- 使用智能指针自动管理生命周期
- 容器扩容后刷新迭代器/指针
- 释放后立即置`nullptr`
- 启用地址消毒器（ASan）检测悬垂指针

## 四、注意事项

- 悬垂指针的使用**不一定**立即崩溃，可能在后续操作中才表现
- 编译器通常无法检测悬垂指针（运行时检测工具如ASan、Valgrind可检测）
- `std::weak_ptr`是解决`shared_ptr`悬垂问题的方案
