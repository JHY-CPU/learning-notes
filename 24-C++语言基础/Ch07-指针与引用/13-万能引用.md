# 万能引用

## 一、概念说明

万能引用（Universal Reference，也称转发引用 Forwarding Reference）出现在**模板参数推导**上下文中，形式为`T&&`。它可以根据传入参数的值类别，自动推导为左值引用或右值引用。

判断准则：`T&&`中`T`是**被推导的模板参数**时是万能引用，否则是右值引用。

## 二、具体用法

### 2.1 万能引用的推导

```cpp
template <typename T>
void func(T&& arg) {
    // T的推导结果取决于传入参数的值类别
    if constexpr (std::is_lvalue_reference_v<T>) {
        std::cout << "推导为左值引用\n";
    } else {
        std::cout << "推导为非引用（右值）\n";
    }
}

int x = 42;
func(x);     // T = int&,  arg = int&
func(42);    // T = int,   arg = int&&
// 输出:
// 推导为左值引用
// 推导为非引用（右值）
```

### 2.2 引用折叠规则

```cpp
// 四种情况：
// T&  &  → T&
// T&  && → T&
// T&& &  → T&
// T&& && → T&&

// 左值传入：T = int&，T&& = int& && = int&
// 右值传入：T = int， T&& = int&&
```

### 2.3 用于工厂函数

```cpp
template <typename T, typename... Args>
std::unique_ptr<T> create(Args&&... args) {
    // 完美转发：保持参数的值类别
    return std::unique_ptr<T>(new T(std::forward<Args>(args)...));
}

auto str = create<std::string>("Hello", 5);
std::cout << *str << std::endl;  // 输出: Hello
```

### 2.4 非万能引用的T&&

```cpp
// 显式指定类型：不是万能引用，是纯右值引用
template <typename T>
void func(std::vector<T>&& vec) {  // 不是万能引用
    // vec必须是右值
}

std::vector<int> v = {1, 2, 3};
// func(v);       // 编译错误：v是左值
func(std::move(v));  // OK
```

## 三、注意事项与常见陷阱

- 只有`T&&`（T被推导）才是万能引用，`int&&`、`vector<T>&&`都不是
- 万能引用必须配合`std::forward`才能正确转发值类别
- `auto&&`也是万能引用
- 万能引用可能导致意外的重载决议，需谨慎使用
- C++20的`concept`可以约束万能引用的接受类型
