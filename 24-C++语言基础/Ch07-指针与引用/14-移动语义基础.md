# 移动语义基础

## 一、概念说明

移动语义允许**转移资源所有权**而非拷贝数据，是C++11最重要的特性之一。通过右值引用和`std::move`，可以避免不必要的深拷贝，大幅提升涉及动态内存的程序性能。

## 二、具体用法

### 2.1 移动 vs 拷贝

```cpp
std::vector<int> src(1000000, 42);

// 拷贝：复制所有数据
auto copy = src;
std::cout << "src大小: " << src.size() << std::endl;      // 输出: 1000000
std::cout << "copy大小: " << copy.size() << std::endl;    // 输出: 1000000

// 移动：转移数据指针
auto moved = std::move(src);
std::cout << "moved大小: " << moved.size() << std::endl;  // 输出: 1000000
std::cout << "src大小: " << src.size() << std::endl;       // 输出: 0
```

### 2.2 实现移动语义

```cpp
class Buffer {
    int* data;
    size_t size;
public:
    Buffer(size_t n) : data(new int[n]), size(n) {}

    // 移动构造
    Buffer(Buffer&& other) noexcept
        : data(other.data), size(other.size) {
        other.data = nullptr;
        other.size = 0;
    }

    // 移动赋值
    Buffer& operator=(Buffer&& other) noexcept {
        if (this != &other) {
            delete[] data;
            data = other.data;
            size = other.size;
            other.data = nullptr;
            other.size = 0;
        }
        return *this;
    }

    ~Buffer() { delete[] data; }
};
```

### 2.3 在容器中的应用

```cpp
std::vector<std::string> strings;

// push_back使用移动避免拷贝
std::string temp = "Hello";
strings.push_back(std::move(temp));  // 移动temp到vector
// temp现在为空

// emplace_back就地构造，更高效
strings.emplace_back("World");

for (const auto& s : strings) std::cout << s << " ";
// 输出: Hello World
```

### 2.4 函数返回值优化

```cpp
std::vector<int> create() {
    std::vector<int> result = {1, 2, 3, 4, 5};
    return result;  // RVO或隐式移动
}

auto v = create();  // 无拷贝
std::cout << v.size() << std::endl;  // 输出: 5
```

## 三、注意事项与常见陷阱

- `std::move`后源对象处于有效但未指定状态
- 移动操作应标记`noexcept`（vector扩容时需要）
- 移动不适用于POD类型（int, double等）
- 被移动的对象可以安全销毁或被赋新值
- 移动语义是`unique_ptr`等不可拷贝类型的基础
