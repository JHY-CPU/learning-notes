# 指针与多态

## 一、概念说明

多态通过**基类指针/引用**指向**派生类对象**实现。调用虚函数时，根据实际对象类型而非指针类型来决定调用哪个版本，称为**动态分派**（Dynamic Dispatch）。

## 二、具体用法

### 2.1 基类指针指向派生类

```cpp
class Shape {
public:
    virtual ~Shape() = default;
    virtual double area() const = 0;
};

class Circle : public Shape {
    double radius;
public:
    Circle(double r) : radius(r) {}
    double area() const override { return 3.14159 * radius * radius; }
};

class Rectangle : public Shape {
    double w, h;
public:
    Rectangle(double w, double h) : w(w), h(h) {}
    double area() const override { return w * h; }
};
```

### 2.2 多态调用

```cpp
// 基类指针数组
std::vector<std::unique_ptr<Shape>> shapes;
shapes.push_back(std::make_unique<Circle>(5.0));
shapes.push_back(std::make_unique<Rectangle>(3.0, 4.0));

// 多态调用：根据实际类型调用area()
for (const auto& s : shapes) {
    std::cout << "面积: " << s->area() << std::endl;
}
// 输出:
// 面积: 78.5398
// 面积: 12
```

### 2.3 引用实现多态

```cpp
void printArea(const Shape& shape) {
    std::cout << "面积: " << shape.area() << std::endl;
}

Circle c(5);
Rectangle r(3, 4);
printArea(c);  // 输出: 面积: 78.5398
printArea(r);  // 输出: 面积: 12
```

### 2.4 dynamic_cast安全转换

```cpp
void process(Shape* shape) {
    // 安全向下转换
    if (auto* circle = dynamic_cast<Circle*>(shape)) {
        std::cout << "这是圆，半径相关: "
                  << circle->area() << std::endl;
    } else {
        std::cout << "不是圆\n";
    }
}

Circle c(10);
process(&c);  // 输出: 这是圆，半径相关: 314.159
```

### 2.5 虚析构函数的重要性

```cpp
class Base {
public:
    virtual ~Base() { std::cout << "~Base\n"; }
};

class Derived : public Base {
    int* data;
public:
    Derived() : data(new int[100]) {}
    ~Derived() override {
        delete[] data;
        std::cout << "~Derived\n";
    }
};

Base* p = new Derived();
delete p;  // 无虚析构：只调用~Base，data泄漏！
// 有虚析构时输出:
// ~Derived
// ~Base
```

## 三、注意事项与常见陷阱

- 基类析构函数必须是`virtual`（如果通过基类指针删除派生类对象）
- `override`关键字帮助编译器检查虚函数签名
- 多态有虚函数表（vtable）开销（额外指针+间接调用）
- 构造函数和析构函数中调用虚函数不会触发动态分派
- `unique_ptr<Base>`管理派生类对象时自动调用正确析构函数
