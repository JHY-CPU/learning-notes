# 指针最佳实践

## 一、概念说明

现代C++中指针的最佳实践核心是**表达所有权语义**。不同的智能指针和引用类型表达了不同的所有权和使用意图，选择正确的类型能提升代码的安全性和可读性。

## 二、决策指南

### 2.1 选择流程图

```
需要间接访问对象？
├── 否 → 使用值类型或const引用
└── 是 → 谁拥有该对象？
    ├── 唯一拥有者 → std::unique_ptr
    ├── 共享拥有者 → std::shared_ptr
    ├── 不拥有（观察者）→ 能保证非空？
    │   ├── 能 → T& 或 const T&
    │   └── 不能 → T* 或 std::optional
    └── 延迟获取 → std::weak_ptr
```

### 2.2 实践代码

```cpp
// 1. 函数参数：不拥有，非空 → const引用
void process(const std::string& name);

// 2. 函数参数：可修改，非空 → 非const引用
void increment(int& value);

// 3. 函数参数：可为空 → 指针
void setCallback(void (*cb)(int));

// 4. 独占资源 → unique_ptr
auto file = std::make_unique<FileHandle>("data.txt");

// 5. 共享资源 → shared_ptr
auto cache = std::make_shared<Cache>();

// 6. 观察shared_ptr → weak_ptr
std::weak_ptr<Cache> observer = cache;

// 7. 函数返回新对象 → 按值返回
std::string createName() { return "Alice"; }

// 8. 函数返回容器元素 → 引用
int& getAt(std::vector<int>& v, size_t i) { return v[i]; }
```

### 2.3 反模式

```cpp
// 反模式1：用裸指针管理所有权
void bad() {
    int* p = new int(42);
    // ... 忘记delete
}

// 反模式2：用shared_ptr无脑代替所有指针
void alsoBad() {
    auto p = std::make_shared<int>(42);  // 不需要共享却用shared_ptr
}

// 正确：根据所有权语义选择
void good() {
    auto p = std::make_unique<int>(42);  // 独占所有权
}
```

## 三、最佳实践清单

1. **优先值语义**：能用栈对象就不用指针
2. **`const`优先**：默认const引用，需要修改时才用非const
3. **智能指针优先**：`unique_ptr` > `shared_ptr` > 裸指针
4. **`make_unique`/`make_shared`优先**：比`new`更安全
5. **裸指针仅用于观察**：不表达所有权
6. **`nullptr`优先**：用`nullptr`不用`NULL`或`0`
7. **启用检测工具**：ASan、UBSan、Valgrind
8. **遵循项目规范**：团队编码标准优先
