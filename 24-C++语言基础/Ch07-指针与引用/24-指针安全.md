# 指针安全

## 一、概念说明

C++中指针的灵活性也是安全隐患的来源：空指针解引用、野指针、缓冲区溢出、双重释放等。现代C++通过**智能指针**、**引用**、**范围检查**等手段大幅降低指针相关的安全问题。

## 二、安全实践

### 2.1 用智能指针替代裸指针

```cpp
// 不安全：手动管理
void unsafe() {
    Widget* w = new Widget();
    // ... 如果中间抛异常，w泄漏
    delete w;
}

// 安全：unique_ptr自动管理
void safe() {
    auto w = std::make_unique<Widget>();
    // 异常安全，自动释放
}
```

### 2.2 用引用替代指针

```cpp
// 不需要空值时用引用
void process(const std::string& name) {  // 引用，不能为空
    std::cout << name << std::endl;
}

// 需要空值时用optional
std::optional<std::string> findUser(int id) {
    if (id > 0) return "Alice";
    return std::nullopt;
}
```

### 2.3 使用span替代指针+大小

```cpp
// 不安全：指针和大小分离
void process(int* data, size_t size);

// 安全：span包含大小信息
void process(std::span<const int> data) {
    for (int x : data) std::cout << x << " ";
}
```

### 2.4 使用容器替代动态数组

```cpp
// 不安全
int* arr = new int[100];
// arr[200] = 1;  // 越界但不报错
delete[] arr;

// 安全
std::vector<int> vec(100);
vec.at(200);  // 抛出 std::out_of_range
```

### 2.5 启用编译器安全检查

```bash
# GCC/Clang: 地址消毒器
g++ -fsanitize=address -g program.cpp

# 未定义行为消毒器
g++ -fsanitize=undefined -g program.cpp

# 运行时边界检查
g++ -D_GLIBCXX_DEBUG program.cpp
```

### 2.6 最佳实践清单

```cpp
// 1. 优先智能指针
auto p = std::make_unique<int>(42);

// 2. 优先引用
void f(const std::string& s);

// 3. 优先容器
std::vector<int> v;

// 4. 使用at()进行边界检查
v.at(0);

// 5. 使用nullptr而非NULL
int* ptr = nullptr;
```

## 三、注意事项与常见陷阱

- 裸指针不表达所有权语义，调用者不知道是否需要释放
- `delete`后置`nullptr`可以减少悬垂指针风险
- AddressSanitizer可检测大多数内存错误
- Debug模式下的STL边界检查有开销但能捕获越界
- 指针安全是团队规范，需要编码标准和代码审查
