# 完美转发详解

## 一、概念说明

完美转发（Perfect Forwarding）在泛型代码中保持参数的**值类别**（左值/右值）和**cv限定符**原封不动地传递给下一层函数。实现依赖于万能引用（`T&&`）和`std::forward`。

## 二、具体用法

### 2.1 问题：参数类别丢失

```cpp
void target(int& x)  { std::cout << "左值版本\n"; }
void target(int&& x) { std::cout << "右值版本\n"; }

template <typename T>
void badWrapper(T arg) {
    target(arg);  // arg永远是左值！即使传入右值
}

badWrapper(42);  // 输出: 左值版本（错误！）
```

### 2.2 解决：std::forward

```cpp
template <typename T>
void goodWrapper(T&& arg) {
    target(std::forward<T>(arg));  // 保持值类别
}

int x = 42;
goodWrapper(x);   // 输出: 左值版本
goodWrapper(42);  // 输出: 右值版本
```

### 2.3 实际应用：make_unique

```cpp
// std::make_unique的简化实现
template <typename T, typename... Args>
std::unique_ptr<T> makeUnique(Args&&... args) {
    return std::unique_ptr<T>(
        new T(std::forward<Args>(args)...)
    );
}

// 完美转发构造参数
struct Point {
    int x, y;
    Point(int x, int y) : x(x), y(y) {}
};

auto p = makeUnique<Point>(3, 4);  // 参数完美转发到构造函数
```

### 2.4 区分std::move和std::forward

```cpp
// std::move：无条件转为右值
// std::forward<T>：仅当T是右值时转为右值

template <typename T>
void process(T&& arg) {
    // 无条件移动：总是转为右值
    consume(std::move(arg));

    // 条件转发：保持原始值类别
    // consume(std::forward<T>(arg));
}
```

### 2.5 引用折叠在转发中的作用

```cpp
// 左值传入：T = int&, T&& = int& && = int&, forward返回int&
// 右值传入：T = int,  T&& = int&&,       forward返回int&&

template <typename T>
void wrapper(T&& arg) {
    std::cout << typeid(T).name() << std::endl;
}

int x = 10;
wrapper(x);   // T = int&
wrapper(10);  // T = int
```

## 三、注意事项与常见陷阱

- `std::forward`只在模板中、参数为推导出的`T&&`时有意义
- 不使用`std::forward`则所有参数在函数体内都是左值
- `std::forward<T>(arg)`在T为左值引用时返回左值引用
- 完美转发是实现`make_shared`、`emplace`等函数的关键
- 某些类型（如初始化列表`{0,1,2}`）无法完美转发
