# 引用折叠规则

## 一、概念说明

引用折叠（Reference Collapsing）是C++11引入的规则，决定了当"引用的引用"出现时如何合并。规则共四种情况，核心原则：**两个引用中只要有一个是左值引用，结果就是左值引用**。

## 二、四种折叠规则

```cpp
// T&  &   → T&
// T&  &&  → T&
// T&& &   → T&
// T&& &&  → T&&

// 示例
int x = 42;
using LRef = int&;
using RRef = int&&;

LRef&  r1 = x;  // int& & = int&
LRef&& r2 = x;  // int& && = int&
RRef&  r3 = x;  // int&& & = int&
// RRef&& r4 = 42;  // int&& && = int&& (不能绑定到x)
```

## 三、具体用法

### 3.1 模板中的应用

```cpp
template <typename T>
void func(T&& arg) {
    // 左值传入：T = int&, T&& = int& && = int&
    // 右值传入：T = int,  T&& = int&&
}

int x = 42;
func(x);   // T = int&,  arg类型 = int&
func(42);  // T = int,   arg类型 = int&&
```

### 3.2 完美转发中的折叠

```cpp
template <typename T>
void wrapper(T&& arg) {
    // std::forward根据T的类型决定返回左值还是右值引用
    target(std::forward<T>(arg));
}
```

### 3.3 类型萃取中的引用折叠

```cpp
// std::forward的简化实现
template <typename T>
T&& forward(std::remove_reference_t<T>& arg) {
    return static_cast<T&&>(arg);
}

// T = int& 时：int& && = int&  (返回左值引用)
// T = int   时：int&& = int&&  (返回右值引用)
```

### 3.4 using别名中的折叠

```cpp
template <typename T>
using RemoveRef = typename std::remove_reference<T>::type;

// 引用折叠出现在模板实例化时
using T1 = RemoveRef<int&>;   // int
using T2 = RemoveRef<int&&>;  // int
```

## 四、注意事项与常见陷阱

- 引用折叠只在模板推导、`decltype`、`using`/`typedef`等间接场景出现
- 直接写`int&& &`等是语法错误
- 引用折叠是万能引用和完美转发的基础
- 规则可总结为：**左值引用优先**（除两右值引用得右值引用外）
- 理解引用折叠是掌握C++11移动语义的关键
