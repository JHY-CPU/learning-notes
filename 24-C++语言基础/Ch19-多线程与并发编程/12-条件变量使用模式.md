# 条件变量使用模式

## 一、概念说明

条件变量的正确使用需要遵循特定模式：在持有锁的情况下检查条件，使用谓词等待，处理虚假唤醒。常见模式包括：单次通知、广播通知、超时等待。

```cpp
#include <iostream>
#include <thread>
#include <mutex>
#include <condition_variable>

// 模式1: 简单等待-通知
std::mutex mtx1;
std::condition_variable cv1;
int data = 0;

void waitForData() {
    std::unique_lock<std::mutex> lock(mtx1);
    cv1.wait(lock, []{ return data != 0; }); // 谓词等待
    std::cout << "收到数据: " << data << std::endl;
}

void sendData(int val) {
    {
        std::lock_guard<std::mutex> lock(mtx1);
        data = val;
    }
    cv1.notify_one();
}

// 模式2: 超时等待
std::mutex mtx2;
std::condition_variable cv2;
bool event = false;

void waitWithTimeout() {
    std::unique_lock<std::mutex> lock(mtx2);
    if (cv2.wait_for(lock, std::chrono::seconds(1), []{ return event; })) {
        std::cout << "事件已发生" << std::endl;
    } else {
        std::cout << "等待超时" << std::endl;
    }
}

int main() {
    std::thread t1(waitForData);
    std::thread t2(sendData, 42);
    t1.join(); t2.join();

    std::thread t3(waitWithTimeout);
    t3.join();

    return 0;
}
```

**输出：**
```
收到数据: 42
等待超时
```

## 二、具体用法

### 2.1 虚假唤醒处理

```cpp
#include <iostream>
#include <thread>
#include <mutex>
#include <condition_variable>

// 虚假唤醒：OS可能在没有notify的情况下唤醒wait
// 解决方案：始终使用谓词版本的wait

std::mutex mtx;
std::condition_variable cv;
bool ready = false;

// 错误写法（不推荐）
void badWait() {
    std::unique_lock<std::mutex> lock(mtx);
    // 不带谓词的wait：可能被虚假唤醒
    // cv.wait(lock);
    // if (ready) { ... } // 可能条件不满足
}

// 正确写法
void goodWait() {
    std::unique_lock<std::mutex> lock(mtx);
    // 带谓词：虚假唤醒后会重新检查条件
    cv.wait(lock, []{ return ready; });
    std::cout << "安全唤醒，条件满足" << std::endl;
}

int main() {
    std::thread t(goodWait);

    std::this_thread::sleep_for(std::chrono::milliseconds(100));
    {
        std::lock_guard<std::mutex> lock(mtx);
        ready = true;
    }
    cv.notify_one();

    t.join();
    return 0;
}
```

**输出：**
```
安全唤醒，条件满足
```

### 2.2 生产消费完整模式

```cpp
#include <iostream>
#include <thread>
#include <mutex>
#include <condition_variable>
#include <queue>

template <typename T>
class ThreadSafeQueue {
    std::queue<T> queue_;
    mutable std::mutex mtx_;
    std::condition_variable cv_;
    bool done_ = false;

public:
    void push(T item) {
        {
            std::lock_guard<std::mutex> lock(mtx_);
            queue_.push(std::move(item));
        }
        cv_.notify_one();
    }

    bool pop(T& item) {
        std::unique_lock<std::mutex> lock(mtx_);
        cv_.wait(lock, [this]{ return !queue_.empty() || done_; });
        if (queue_.empty()) return false;
        item = std::move(queue_.front());
        queue_.pop();
        return true;
    }

    void finish() {
        {
            std::lock_guard<std::mutex> lock(mtx_);
            done_ = true;
        }
        cv_.notify_all();
    }
};

int main() {
    ThreadSafeQueue<int> q;
    std::thread producer([&q](){
        for (int i = 0; i < 5; ++i) q.push(i);
        q.finish();
    });

    std::thread consumer([&q](){
        int val;
        while (q.pop(val)) std::cout << "消费: " << val << std::endl;
    });

    producer.join();
    consumer.join();
    return 0;
}
```

**输出：**
```
消费: 0
消费: 1
消费: 2
消费: 3
消费: 4
```

## 三、注意事项与常见陷阱

- **虚假唤醒是正常现象**：OS实现允许，必须用谓词保护。
- **`notify`不需要持有锁**：但条件修改需要在锁保护下。
- **`notify_all`适合广播场景**：如配置更新通知所有线程。
- **`wait_for`返回false表示超时**：返回true表示条件满足。
- **`wait_until`使用绝对时间点**：`wait_for`使用相对时间。
