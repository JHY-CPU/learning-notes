# 并发数据结构

## 一、概念说明

并发数据结构是线程安全的容器，内部封装了同步机制。相比每次手动加锁访问普通容器，它提供更细粒度的锁、更好的并发性能。

```cpp
#include <iostream>
#include <thread>
#include <mutex>
#include <queue>
#include <condition_variable>

// 线程安全队列
template <typename T>
class ConcurrentQueue {
    mutable std::mutex mtx_;
    std::queue<T> queue_;
    std::condition_variable cv_;

public:
    void push(T item) {
        std::lock_guard<std::mutex> lock(mtx_);
        queue_.push(std::move(item));
        cv_.notify_one();
    }

    bool tryPop(T& item) {
        std::lock_guard<std::mutex> lock(mtx_);
        if (queue_.empty()) return false;
        item = std::move(queue_.front());
        queue_.pop();
        return true;
    }

    void waitAndPop(T& item) {
        std::unique_lock<std::mutex> lock(mtx_);
        cv_.wait(lock, [this] { return !queue_.empty(); });
        item = std::move(queue_.front());
        queue_.pop();
    }

    bool empty() const {
        std::lock_guard<std::mutex> lock(mtx_);
        return queue_.empty();
    }

    size_t size() const {
        std::lock_guard<std::mutex> lock(mtx_);
        return queue_.size();
    }
};

int main() {
    ConcurrentQueue<int> q;

    std::thread producer([&q] {
        for (int i = 0; i < 5; ++i) {
            q.push(i);
            std::cout << "入队: " << i << std::endl;
        }
    });

    std::thread consumer([&q] {
        for (int i = 0; i < 5; ++i) {
            int val;
            q.waitAndPop(val);
            std::cout << "出队: " << val << std::endl;
        }
    });

    producer.join();
    consumer.join();
    return 0;
}
```

**输出（顺序可能不同）：**
```
入队: 0
出队: 0
入队: 1
出队: 1
...
```

## 二、具体用法

### 2.1 线程安全哈希表

```cpp
#include <iostream>
#include <thread>
#include <mutex>
#include <unordered_map>
#include <shared_mutex>
#include <vector>

template <typename K, typename V>
class ConcurrentHashMap {
    static constexpr size_t BUCKET_COUNT = 16;

    struct Bucket {
        std::unordered_map<K, V> map;
        mutable std::shared_mutex mtx;
    };

    std::vector<Bucket> buckets{BUCKET_COUNT};

    Bucket& getBucket(const K& key) {
        return buckets[std::hash<K>{}(key) % BUCKET_COUNT];
    }

public:
    void insert(K key, V value) {
        auto& bucket = getBucket(key);
        std::unique_lock lock(bucket.mtx);
        bucket.map[std::move(key)] = std::move(value);
    }

    bool find(const K& key, V& value) const {
        const auto& bucket = getBucket(key);
        std::shared_lock lock(bucket.mtx);
        auto it = bucket.map.find(key);
        if (it == bucket.map.end()) return false;
        value = it->second;
        return true;
    }

    bool erase(const K& key) {
        auto& bucket = getBucket(key);
        std::unique_lock lock(bucket.mtx);
        return bucket.map.erase(key) > 0;
    }
};

int main() {
    ConcurrentHashMap<std::string, int> map;

    std::thread t1([&map] { map.insert("key1", 100); });
    std::thread t2([&map] { map.insert("key2", 200); });
    t1.join(); t2.join();

    int val;
    if (map.find("key1", val))
        std::cout << "key1 = " << val << std::endl;
    if (map.find("key2", val))
        std::cout << "key2 = " << val << std::endl;

    return 0;
}
```

**输出：**
```
key1 = 100
key2 = 200
```

## 三、注意事项与常见陷阱

- **分段锁提高并发**：按哈希分桶，不同桶可并行访问。
- **迭代器不是线程安全的**：遍历需加全局锁或快照。
- **`std::vector`不适合并发**：扩容时所有迭代器失效。
- **`tbb::concurrent_hash_map`是工业级实现**：Intel TBB库提供。
- **死锁风险**：跨桶操作需要按序加锁或使用`scoped_lock`。
