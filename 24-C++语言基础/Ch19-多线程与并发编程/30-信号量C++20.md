# 信号量 C++20

## 一、概念说明

C++20引入了`std::counting_semaphore`和`std::binary_semaphore`（`<semaphore>`头文件）。信号量控制对有限资源的并发访问，比mutex更灵活。

- `counting_semaphore<N>`：计数从0到N
- `binary_semaphore`：等价于`counting_semaphore<1>`

```cpp
#include <iostream>
#include <thread>
#include <semaphore>
#include <vector>

std::counting_semaphore<3> sem(3); // 最多3个并发

void worker(int id) {
    sem.acquire(); // 获取许可
    std::cout << "线程" << id << " 进入（并发数<=3）" << std::endl;
    std::this_thread::sleep_for(std::chrono::milliseconds(100));
    std::cout << "线程" << id << " 离开" << std::endl;
    sem.release(); // 释放许可
}

int main() {
    std::vector<std::thread> threads;
    for (int i = 0; i < 6; ++i) {
        threads.emplace_back(worker, i);
    }
    for (auto& t : threads) t.join();
    return 0;
}
```

**输出（顺序可能不同，但同时最多3个）：**
```
线程0 进入（并发数<=3）
线程1 进入（并发数<=3）
线程2 进入（并发数<=3）
线程0 离开
线程3 进入（并发数<=3）
...
```

## 二、具体用法

### 2.1 binary_semaphore作为事件通知

```cpp
#include <iostream>
#include <thread>
#include <semaphore>

std::binary_semaphore eventSem{0}; // 初始为0

void waiter() {
    std::cout << "等待事件..." << std::endl;
    eventSem.acquire(); // 阻塞直到信号
    std::cout << "事件已发生" << std::endl;
}

void signaler() {
    std::this_thread::sleep_for(std::chrono::milliseconds(100));
    std::cout << "发送信号" << std::endl;
    eventSem.release();
}

int main() {
    std::thread t1(waiter);
    std::thread t2(signaler);
    t1.join();
    t2.join();
    return 0;
}
```

**输出：**
```
等待事件...
发送信号
事件已发生
```

### 2.2 信号量 vs 条件变量

| 特性 | 信号量 | 条件变量+mutex |
|------|--------|--------------|
| 用途 | 限流/事件 | 条件等待 |
| 配合mutex | 不需要 | 必须 |
| 虚假唤醒 | 无 | 有 |
| 递减 | 自动 | 手动 |
| API复杂度 | 简单 | 较复杂 |

## 三、注意事项与常见陷阱

- **`release()`增加计数**：可能唤醒等待的`acquire()`。
- **`acquire()`减少计数**：为0时阻塞。
- **`try_acquire()`非阻塞尝试**：成功返回true。
- **`try_acquire_for()`/`try_acquire_until()`支持超时**。
- **信号量不保证公平性**：某些线程可能饥饿。
- **需要C++20支持**：GCC 11+、Clang 14+、MSVC 2019 16.8+。
