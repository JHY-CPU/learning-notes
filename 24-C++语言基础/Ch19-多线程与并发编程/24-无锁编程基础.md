# 无锁编程基础

## 一、概念说明

无锁（Lock-free）编程使用原子操作而非锁来实现线程安全。优点：无死锁、无优先级反转、对信号安全。缺点：实现复杂、难以验证正确性。

核心操作：CAS（Compare-And-Swap），由`compare_exchange`实现。

```cpp
#include <iostream>
#include <atomic>
#include <thread>
#include <vector>

// 无锁计数器
class LockFreeCounter {
    std::atomic<int> count{0};
public:
    void increment() {
        int old = count.load();
        while (!count.compare_exchange_weak(old, old + 1)) {
            // old被自动更新为当前值
        }
    }
    int get() const { return count.load(); }
};

int main() {
    LockFreeCounter counter;

    std::vector<std::thread> threads;
    for (int i = 0; i < 4; ++i) {
        threads.emplace_back([&counter] {
            for (int j = 0; j < 100000; ++j) counter.increment();
        });
    }
    for (auto& t : threads) t.join();

    std::cout << "计数器: " << counter.get() << std::endl;
    return 0;
}
```

**输出：**
```
计数器: 400000
```

## 二、具体用法

### 2.1 ABA问题

```cpp
#include <iostream>
#include <atomic>

// ABA问题：值从A变为B再变回A，CAS认为没变
// 解决方案：使用带版本号的指针

template <typename T>
struct TaggedPointer {
    T* ptr;
    uintptr_t tag; // 版本号

    bool operator==(const TaggedPointer& other) const {
        return ptr == other.ptr && tag == other.tag;
    }
};

// 无锁栈简化示例
struct Node {
    int data;
    Node* next;
};

std::atomic<Node*> head{nullptr};

void push(int val) {
    Node* newNode = new Node{val, nullptr};
    Node* oldHead = head.load();
    do {
        newNode->next = oldHead;
    } while (!head.compare_exchange_weak(oldHead, newNode));
}

int main() {
    std::cout << "ABA问题说明:" << std::endl;
    std::cout << "1. 线程1读取head=A" << std::endl;
    std::cout << "2. 线程2将head改为B" << std::endl;
    std::cout << "3. 线程2将head改回A" << std::endl;
    std::cout << "4. 线程1的CAS成功（但A已不同）" << std::endl;
    std::cout << "解决: 使用带版本号的CAS" << std::endl;

    push(1); push(2); push(3);
    Node* n = head.load();
    while (n) { std::cout << n->data << " "; n = n->next; }
    std::cout << std::endl;

    // 清理
    while (n = head.load()) {
        head.store(n->next);
        delete n;
    }

    return 0;
}
```

**输出：**
```
ABA问题说明:
1. 线程1读取head=A
2. 线程2将head改为B
3. 线程2将head改回A
4. 线程1的CAS成功（但A已不同）
解决: 使用带版本号的CAS
3 2 1
```

## 三、注意事项与常见陷阱

- **ABA问题是无锁编程的经典陷阱**：用版本号/标签指针解决。
- **无锁不等于无等待**：无锁保证系统级进展，无等待保证线程级进展。
- **内存回收困难**：线程可能持有已删除节点的指针。解决方案：hazard pointer、RCU。
- **验证正确性极其困难**：使用CDSChecker等工具。
- **x86上`lock cmpxchg`有完整内存屏障**：其他平台需注意内存序。
- **生产环境优先使用成熟库**：如folly、libcds。
