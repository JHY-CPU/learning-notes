# 原子操作

## 一、概念说明

`std::atomic`提供无锁的原子操作，保证操作不可分割。比mutex更轻量，适合简单计数、标志位等场景。

常用操作：`load`（读取）、`store`（写入）、`exchange`（交换）、`compare_exchange_weak/strong`（CAS）。

```cpp
#include <iostream>
#include <thread>
#include <atomic>
#include <vector>

std::atomic<int> counter{0};

void increment(int times) {
    for (int i = 0; i < times; ++i) {
        counter.fetch_add(1, std::memory_order_relaxed); // 原子加
    }
}

int main() {
    std::vector<std::thread> threads;
    for (int i = 0; i < 4; ++i) {
        threads.emplace_back(increment, 100000);
    }
    for (auto& t : threads) t.join();

    std::cout << "counter = " << counter.load() << std::endl; // 400000
    return 0;
}
```

**输出：**
```
counter = 400000
```

## 二、具体用法

### 2.1 基本操作

```cpp
#include <iostream>
#include <atomic>

int main() {
    std::atomic<int> val{42};

    // load / store
    std::cout << "load: " << val.load() << std::endl;
    val.store(100);
    std::cout << "store后: " << val.load() << std::endl;

    // exchange: 设置新值，返回旧值
    int old = val.exchange(200);
    std::cout << "exchange: 旧值=" << old << ", 新值=" << val.load() << std::endl;

    // fetch_add / fetch_sub
    val.fetch_add(10);
    std::cout << "fetch_add(10): " << val.load() << std::endl;
    val.fetch_sub(5);
    std::cout << "fetch_sub(5): " << val.load() << std::endl;

    // 运算符重载
    val += 25;
    ++val;
    std::cout << "运算符: " << val.load() << std::endl;

    return 0;
}
```

**输出：**
```
load: 42
store后: 100
exchange: 旧值=100, 新值=200
fetch_add(10): 210
fetch_sub(5): 205
运算符: 231
```

### 2.2 原子标志

```cpp
#include <iostream>
#include <thread>
#include <atomic>

std::atomic_flag lockFlag = ATOMIC_FLAG_INIT;

void criticalSection(int id) {
    // 自旋锁
    while (lockFlag.test_and_set(std::memory_order_acquire)) {
        // 自旋等待
    }

    std::cout << "线程" << id << " 进入临界区" << std::endl;

    lockFlag.clear(std::memory_order_release);
}

int main() {
    std::thread t1(criticalSection, 1);
    std::thread t2(criticalSection, 2);
    t1.join();
    t2.join();
    return 0;
}
```

**输出：**
```
线程1 进入临界区
线程2 进入临界区
```

## 三、注意事项与常见陷阱

- **`atomic`只保证单个变量的原子性**：多个原子变量的复合操作不是原子的。
- **`is_lock_free()`检查是否无锁**：某些平台的`atomic<double>`可能用锁实现。
- **`memory_order`影响性能和正确性**：默认`memory_order_seq_cst`最安全。
- **`atomic_flag`是唯一保证无锁的类型**：其他`atomic`特化可能用锁。
- **浮点原子操作C++20起支持**：`atomic<float>`/`atomic<double>`。
