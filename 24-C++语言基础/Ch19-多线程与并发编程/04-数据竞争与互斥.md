# 数据竞争与互斥

## 一、概念说明

**数据竞争（Data Race）**：两个或以上线程同时访问同一内存位置，且至少一个是写操作，且没有同步。数据竞争导致**未定义行为**。

**竞态条件（Race Condition）**：程序结果依赖于线程执行的相对时序。数据竞争是竞态条件的一种。

```cpp
#include <iostream>
#include <thread>
#include <vector>

int counter = 0; // 共享变量

void increment() {
    for (int i = 0; i < 100000; ++i) {
        ++counter; // 数据竞争！三个操作：读、加、写
    }
}

int main() {
    std::thread t1(increment);
    std::thread t2(increment);
    t1.join();
    t2.join();

    // 期望200000，实际可能更少
    std::cout << "counter = " << counter << std::endl;
    return 0;
}
```

**输出（不确定）：**
```
counter = 134567  // 每次运行结果不同！
```

## 二、具体用法

### 2.1 用mutex解决数据竞争

```cpp
#include <iostream>
#include <thread>
#include <mutex>
#include <vector>

int counter = 0;
std::mutex mtx;

void safeIncrement() {
    for (int i = 0; i < 100000; ++i) {
        std::lock_guard<std::mutex> lock(mtx);
        ++counter;
    }
}

int main() {
    std::thread t1(safeIncrement);
    std::thread t2(safeIncrement);
    t1.join();
    t2.join();

    std::cout << "counter = " << counter << std::endl; // 恒为200000
    return 0;
}
```

**输出：**
```
counter = 200000
```

### 2.2 竞态条件示例

```cpp
#include <iostream>
#include <thread>
#include <mutex>

class BankAccount {
    int balance;
    std::mutex mtx;
public:
    BankAccount(int b) : balance(b) {}

    // 竞态条件：检查和操作之间可能被其他线程修改
    void unsafeWithdraw(int amount) {
        if (balance >= amount) { // 检查
            // 其他线程可能在这里修改balance!
            balance -= amount;    // 操作
        }
    }

    // 安全版本：加锁保护整个检查-操作
    void safeWithdraw(int amount) {
        std::lock_guard<std::mutex> lock(mtx);
        if (balance >= amount) {
            balance -= amount;
        }
    }

    int getBalance() const { return balance; }
};

int main() {
    std::cout << "数据竞争和竞态条件演示完成" << std::endl;
    return 0;
}
```

**输出：**
```
数据竞争和竞态条件演示完成
```

## 三、注意事项与常见陷阱

- **`++i`不是原子操作**：涉及读-改-写三个步骤。
- **数据竞争是未定义行为**：可能导致崩溃、数据损坏、看似正确的结果。
- **`ThreadSanitizer`可检测数据竞争**：`-fsanitize=thread`。
- **互斥是最基本的同步手段**：但过度使用影响性能。
- **最小化锁的范围**：只保护真正需要同步的代码。
