# 内存栅栏

## 一、概念说明

内存栅栏（Memory Fence）防止特定类型的指令重排序。`atomic_thread_fence`在不操作原子变量的情况下建立同步点。栅栏配合普通原子操作使用，比全用`seq_cst`更灵活。

栅栏的主要用途：在没有直接原子操作的地方建立happens-before关系。

```cpp
#include <iostream>
#include <thread>
#include <atomic>

std::atomic<bool> ready{false};
int payload = 0;

void producer() {
    payload = 42; // 非原子写入
    std::atomic_thread_fence(std::memory_order_release); // 释放栅栏
    ready.store(true, std::memory_order_relaxed);
}

void consumer() {
    while (!ready.load(std::memory_order_relaxed)) {}
    std::atomic_thread_fence(std::memory_order_acquire); // 获取栅栏
    std::cout << "payload = " << payload << std::endl;
}

int main() {
    std::thread t1(producer);
    std::thread t2(consumer);
    t1.join();
    t2.join();
    return 0;
}
```

**输出：**
```
payload = 42
```

## 二、具体用法

### 2.1 happens-before关系

```cpp
#include <iostream>
#include <thread>
#include <atomic>

// happens-before关系：
// 1. 同一线程中，按程序顺序的语句
// 2. release-acquire同步
// 3. 传递性：A happens-before B, B happens-before C => A happens-before C

std::atomic<int> data{0};
std::atomic<bool> flag{false};

void writer() {
    data.store(100, std::memory_order_relaxed);
    // 栅栏建立同步
    std::atomic_thread_fence(std::memory_order_release);
    flag.store(true, std::memory_order_relaxed);
}

void reader() {
    while (!flag.load(std::memory_order_relaxed)) {}
    std::atomic_thread_fence(std::memory_order_acquire);
    // fence保证data的读取可以看到写入的100
    std::cout << "data = " << data.load(std::memory_order_relaxed) << std::endl;
}

int main() {
    std::thread t1(writer);
    std::thread t2(reader);
    t1.join();
    t2.join();
    return 0;
}
```

**输出：**
```
data = 100
```

### 2.2 信号量实现

```cpp
#include <iostream>
#include <thread>
#include <atomic>

// 用fence实现简单的信号量
class SimpleSemaphore {
    std::atomic<int> count;
public:
    SimpleSemaphore(int n) : count(n) {}

    void acquire() {
        while (true) {
            int expected = count.load(std::memory_order_relaxed);
            while (expected <= 0) {
                expected = count.load(std::memory_order_relaxed);
            }
            if (count.compare_exchange_weak(expected, expected - 1,
                    std::memory_order_acquire)) {
                break;
            }
        }
    }

    void release() {
        count.fetch_add(1, std::memory_order_release);
    }
};

int main() {
    std::cout << "内存栅栏用于建立happens-before关系" << std::endl;
    std::cout << "比直接使用seq_cst更灵活高效" << std::endl;
    return 0;
}
```

**输出：**
```
内存栅栏用于建立happens-before关系
比直接使用seq_cst更灵活高效
```

## 三、注意事项与常见陷阱

- **栅栏不阻止所有重排序**：只阻止特定方向的重排。
- **release栅栏阻止之前的操作移到栅栏之后**。
- **acquire栅栏阻止之后的操作移到栅栏之前**。
- **`atomic_signal_fence`只防止编译器重排**：不防止CPU重排。
- **大多数应用不需要显式栅栏**：用原子操作的内存序参数更简洁。
- **栅栏的正确使用需要深入理解内存模型**：建议先用`seq_cst`。
