# 异步编程模式

## 一、概念说明

异步编程允许程序在等待IO或计算结果时继续执行其他任务。C++支持的异步模式：
- **回调（Callback）**：传递函数指针，操作完成后调用
- **Future/Promise**：返回future对象，需要时get结果
- **协程（C++20）**：用co_await挂起，自动恢复

```cpp
#include <iostream>
#include <future>
#include <functional>
#include <vector>

// Future模式
std::future<int> asyncCompute(int x) {
    return std::async(std::launch::async, [x] {
        std::this_thread::sleep_for(std::chrono::milliseconds(100));
        return x * x;
    });
}

// 回调模式
void asyncWithCallback(int x, std::function<void(int)> callback) {
    std::thread([x, cb = std::move(callback)] {
        std::this_thread::sleep_for(std::chrono::milliseconds(100));
        cb(x * x);
    }).detach();
}

int main() {
    // Future模式
    auto fut1 = asyncCompute(5);
    auto fut2 = asyncCompute(10);
    std::cout << "5*5=" << fut1.get() << ", 10*10=" << fut2.get() << std::endl;

    // 回调模式
    std::mutex mtx;
    bool done = false;
    asyncWithCallback(7, [&](int result) {
        std::lock_guard<std::mutex> lock(mtx);
        std::cout << "回调结果: 7*7=" << result << std::endl;
        done = true;
    });

    // 等待回调完成
    std::this_thread::sleep_for(std::chrono::milliseconds(200));
    return 0;
}
```

**输出：**
```
5*5=25, 10*10=100
回调结果: 7*7=49
```

## 二、具体用法

### 2.1 链式异步操作

```cpp
#include <iostream>
#include <future>
#include <string>

// 模拟链式异步
std::future<std::string> fetchUser(int userId) {
    return std::async(std::launch::async, [userId] {
        std::this_thread::sleep_for(std::chrono::milliseconds(50));
        return std::string("User") + std::to_string(userId);
    });
}

std::future<std::string> fetchPosts(const std::string& user) {
    return std::async(std::launch::async, [user] {
        std::this_thread::sleep_for(std::chrono::milliseconds(50));
        return user + "'s posts: [Post1, Post2]";
    });
}

int main() {
    // 链式调用
    auto userFut = fetchUser(42);
    std::string user = userFut.get();

    auto postsFut = fetchPosts(user);
    std::string posts = postsFut.get();

    std::cout << posts << std::endl;
    return 0;
}
```

**输出：**
```
User42's posts: [Post1, Post2]
```

### 2.2 异步模式对比

| 模式 | 优点 | 缺点 |
|------|------|------|
| 回调 | 简单直接 | 回调地狱、错误处理困难 |
| Future | 链式调用、异常传递 | 不能自动链式 |
| 协程 | 同步写法、最优雅 | 编译器支持有限 |

## 三、注意事项与常见陷阱

- **`future`析构会阻塞**：避免创建不保存的`async`返回值。
- **回调中的异常难以捕获**：应在回调内处理。
- **`std::function`有拷贝开销**：性能敏感时用函数指针。
- **协程需要C++20**：编译器支持尚不完善。
- **`co_await`自动挂起和恢复**：是最现代的异步写法。
