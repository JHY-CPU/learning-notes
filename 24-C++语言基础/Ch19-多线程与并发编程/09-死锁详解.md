# 死锁详解

## 一、概念说明

死锁（Deadlock）指两个或以上线程互相等待对方持有的锁，导致所有线程永久阻塞。死锁的四个必要条件：

1. **互斥**：资源一次只能被一个线程持有
2. **持有并等待**：持有一个锁的同时等待另一个
3. **不可剥夺**：锁只能由持有者释放
4. **循环等待**：A等B，B等A

```cpp
#include <iostream>
#include <thread>
#include <mutex>

std::mutex mtx1, mtx2;

void threadA() {
    std::lock_guard<std::mutex> lock1(mtx1);
    std::cout << "线程A持有mtx1，等待mtx2..." << std::endl;
    std::this_thread::sleep_for(std::chrono::milliseconds(100));

    std::lock_guard<std::mutex> lock2(mtx2); // 死锁！
    std::cout << "线程A获取两个锁" << std::endl;
}

void threadB() {
    std::lock_guard<std::mutex> lock2(mtx2);
    std::cout << "线程B持有mtx2，等待mtx1..." << std::endl;
    std::this_thread::sleep_for(std::chrono::milliseconds(100));

    std::lock_guard<std::mutex> lock1(mtx1); // 死锁！
    std::cout << "线程B获取两个锁" << std::endl;
}

int main() {
    // 死锁示例（会卡住！）
    // std::thread t1(threadA);
    // std::thread t2(threadB);
    // t1.join(); t2.join();

    std::cout << "死锁演示（已注释防止卡住）" << std::endl;
    std::cout << "线程A: lock(mtx1) -> lock(mtx2)" << std::endl;
    std::cout << "线程B: lock(mtx2) -> lock(mtx1)" << std::endl;
    std::cout << "结果: A等B释放mtx2，B等A释放mtx1 -> 死锁" << std::endl;

    return 0;
}
```

**输出：**
```
死锁演示（已注释防止卡住）
线程A: lock(mtx1) -> lock(mtx2)
线程B: lock(mtx2) -> lock(mtx1)
结果: A等B释放mtx2，B等A释放mtx1 -> 死锁
```

## 二、具体用法

### 2.1 死锁检测思路

```cpp
#include <iostream>
#include <thread>
#include <mutex>
#include <chrono>

// 简化的死锁检测：使用超时
std::timed_mutex tmtx1, tmtx2;

void safeThreadA() {
    if (tmtx1.try_lock_for(std::chrono::milliseconds(100))) {
        std::lock_guard<std::timed_mutex> lock1(tmtx1, std::adopt_lock);
        std::this_thread::sleep_for(std::chrono::milliseconds(50));

        if (tmtx2.try_lock_for(std::chrono::milliseconds(100))) {
            std::lock_guard<std::timed_mutex> lock2(tmtx2, std::adopt_lock);
            std::cout << "线程A安全获取两个锁" << std::endl;
        } else {
            std::cout << "线程A获取mtx2超时，放弃" << std::endl;
        }
    }
}

int main() {
    std::thread t1(safeThreadA);
    t1.join();
    return 0;
}
```

**输出：**
```
线程A安全获取两个锁
```

## 三、注意事项与常见陷阱

- **一致的加锁顺序**：所有线程按相同顺序获取锁是最有效的预防手段。
- **使用`std::lock()`同时获取多个锁**：避免死锁。
- **`scoped_lock`（C++17）自动处理多锁**：推荐使用。
- **超时检测**：`try_lock_for`可以避免无限等待。
- **避免在持有锁时调用用户代码**：用户代码可能获取其他锁。
- **`ThreadSanitizer`可检测部分死锁**：但不是所有情况。
