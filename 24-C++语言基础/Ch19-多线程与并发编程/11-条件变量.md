# 条件变量

## 一、概念说明

`std::condition_variable`允许线程等待某个条件成立，避免忙等待（busy-waiting）。与`unique_lock`配合使用，`wait()`自动释放锁并阻塞，`notify_one()`/`notify_all()`唤醒等待线程。

```cpp
#include <iostream>
#include <thread>
#include <mutex>
#include <condition_variable>
#include <queue>

std::mutex mtx;
std::condition_variable cv;
std::queue<int> dataQueue;
bool finished = false;

void producer() {
    for (int i = 0; i < 5; ++i) {
        std::this_thread::sleep_for(std::chrono::milliseconds(100));
        {
            std::lock_guard<std::mutex> lock(mtx);
            dataQueue.push(i);
            std::cout << "生产: " << i << std::endl;
        }
        cv.notify_one(); // 通知一个等待线程
    }
    {
        std::lock_guard<std::mutex> lock(mtx);
        finished = true;
    }
    cv.notify_all();
}

void consumer(int id) {
    while (true) {
        std::unique_lock<std::mutex> lock(mtx);
        cv.wait(lock, []{ return !dataQueue.empty() || finished; });

        while (!dataQueue.empty()) {
            int val = dataQueue.front();
            dataQueue.pop();
            std::cout << "消费者" << id << " 处理: " << val << std::endl;
        }

        if (finished && dataQueue.empty()) break;
    }
}

int main() {
    std::thread p(producer);
    std::thread c1(consumer, 1);
    std::thread c2(consumer, 2);
    p.join(); c1.join(); c2.join();
    return 0;
}
```

**输出（顺序可能不同）：**
```
生产: 0
消费者1 处理: 0
生产: 1
消费者2 处理: 1
...
```

## 二、具体用法

### 2.1 wait的基本模式

```cpp
#include <iostream>
#include <thread>
#include <mutex>
#include <condition_variable>

std::mutex mtx;
std::condition_variable cv;
bool ready = false;

void waiter() {
    std::unique_lock<std::mutex> lock(mtx);
    // 等待直到ready为true（避免虚假唤醒）
    cv.wait(lock, []{ return ready; });
    std::cout << "等待者被唤醒" << std::endl;
}

void notifier() {
    std::this_thread::sleep_for(std::chrono::milliseconds(100));
    {
        std::lock_guard<std::mutex> lock(mtx);
        ready = true;
    }
    cv.notify_one();
    std::cout << "通知者发送信号" << std::endl;
}

int main() {
    std::thread t1(waiter);
    std::thread t2(notifier);
    t1.join();
    t2.join();
    return 0;
}
```

**输出：**
```
通知者发送信号
等待者被唤醒
```

## 三、注意事项与常见陷阱

- **必须在`unique_lock`下使用**：`wait()`需要能释放和重新获取锁。
- **必须使用谓词版本的`wait`**：避免虚假唤醒。
- **`notify`不需要持有锁**：但通常在持有锁时修改条件。
- **`notify_all`唤醒所有等待线程**：它们竞争获取锁。
- **`wait_for`/`wait_until`支持超时**：避免无限等待。
