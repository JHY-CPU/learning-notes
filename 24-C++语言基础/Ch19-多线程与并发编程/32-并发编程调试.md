# 并发编程调试

## 一、概念说明

并发bug（数据竞争、死锁、活锁）难以复现和调试。专用工具：
- **ThreadSanitizer (TSan)**：检测数据竞争
- **Helgrind**（Valgrind工具）：检测锁使用错误
- **Deadlock Detector**：检测死锁
- **Clang Thread Safety Analysis**：编译期注解检查

```cpp
#include <iostream>
#include <thread>
#include <mutex>

int counter = 0;
std::mutex mtx;

// 正确的写法
void safeIncrement() {
    std::lock_guard<std::mutex> lock(mtx);
    ++counter;
}

// 有数据竞争的写法（TSan会检测到）
void unsafeIncrement() {
    ++counter; // 数据竞争！
}

int main() {
    // 使用ThreadSanitizer编译:
    // g++ -fsanitize=thread -g -o test test.cpp

    std::thread t1(safeIncrement);
    std::thread t2(safeIncrement);
    t1.join();
    t2.join();

    std::cout << "counter = " << counter << std::endl;
    return 0;
}
```

**输出：**
```
counter = 2
```

## 二、具体用法

### 2.1 ThreadSanitizer使用

```bash
# 编译
g++ -fsanitize=thread -g -O1 -o race_test race_test.cpp

# 运行
./race_test

# TSan输出示例：
# WARNING: ThreadSanitizer: data race (pid=12345)
#   Write of size 4 at 0x... by thread T2:
#     #0 unsafeIncrement() race_test.cpp:15
#   Previous write of size 4 at 0x... by thread T1:
#     #0 unsafeIncrement() race_test.cpp:15
```

### 2.2 Clang Thread Safety Analysis

```cpp
#include <mutex>

class BankAccount {
    std::mutex mu;
    int balance GUARDED_BY(mu) = 0;

public:
    void deposit(int amount) EXCLUSIVE_LOCKS_REQUIRED(mu) {
        balance += amount;
    }

    void safeDeposit(int amount) {
        std::lock_guard<std::mutex> lock(mu);
        deposit(amount); // 编译器检查mu是否持有
    }

    int getBalance() {
        std::lock_guard<std::mutex> lock(mu);
        return balance;
    }
};

// 注解：
// GUARDED_BY(mu) - 变量受mu保护
// EXCLUSIVE_LOCKS_REQUIRED(mu) - 调用前必须持有mu
// LOCKS_EXCLUDED(mu) - 调用前不能持有mu
// LOCK_RETURNED(mu) - 函数返回mu的引用

int main() {
    std::cout << "Thread Safety Analysis: 编译期检测锁使用" << std::endl;
    return 0;
}
```

**输出：**
```
Thread Safety Analysis: 编译期检测锁使用
```

### 2.3 调试技巧

```cpp
#include <iostream>
#include <thread>
#include <mutex>
#include <sstream>

// 线程安全的日志
class ThreadSafeLogger {
    std::mutex mtx;
public:
    template <typename... Args>
    void log(Args&&... args) {
        std::lock_guard<std::mutex> lock(mtx);
        std::ostringstream oss;
        oss << "[Thread " << std::this_thread::get_id() << "] ";
        (oss << ... << std::forward<Args>(args));
        std::cout << oss.str() << std::endl;
    }
};

ThreadSafeLogger logger;

int main() {
    std::thread t1([&] { logger.log("线程1工作"); });
    std::thread t2([&] { logger.log("线程2工作"); });
    t1.join(); t2.join();
    return 0;
}
```

**输出（示例）：**
```
[Thread 140234567890] 线程1工作
[Thread 140234567891] 线程2工作
```

## 三、注意事项与常见陷阱

- **TSan运行时开销约5-15倍**：仅用于测试，不能用于生产。
- **TSan在Windows支持有限**：Linux上效果最佳。
- **Helgrind比TSan慢但更全面**：Valgrind套件的一部分。
- **`-O1`或以上优化级别**：TSan需要知道函数边界。
- **`TSAN_OPTIONS`环境变量**：配置检测行为（如`halt_on_error=1`）。
- **并发bug是偶发的**：多次运行才能发现，CI中应反复测试。
