# 并行算法 C++17

## 一、概念说明

C++17在`<algorithm>`中增加了执行策略参数，允许标准算法自动并行化。执行策略定义在`<execution>`头文件中：

- `execution::sequenced_policy`：顺序执行（默认）
- `execution::parallel_policy`：并行执行
- `execution::parallel_unsequenced_policy`：并行+向量化

```cpp
#include <iostream>
#include <algorithm>
#include <execution>
#include <vector>
#include <numeric>

int main() {
    std::vector<int> data(1000000);
    std::iota(data.begin(), data.end(), 1); // 1,2,3,...,1000000

    // 并行求和
    long long sum = std::reduce(std::execution::par,
                                 data.begin(), data.end(), 0LL);
    std::cout << "并行求和: " << sum << std::endl;

    // 并行排序
    std::vector<int> toSort(data);
    std::sort(std::execution::par, toSort.begin(), toSort.end());
    std::cout << "并行排序完成, 首元素: " << toSort.front()
              << ", 末元素: " << toSort.back() << std::endl;

    // 并行for_each
    std::for_each(std::execution::par, data.begin(), data.end(),
                  [](int& x) { x *= 2; });
    std::cout << "并行for_each完成, data[0]=" << data[0] << std::endl;

    return 0;
}
```

**输出：**
```
并行求和: 500000500000
并行排序完成, 首元素: 1, 末元素: 1000000
并行for_each完成, data[0]=2
```

## 二、具体用法

### 2.1 支持并行的算法列表

```cpp
#include <iostream>
#include <algorithm>
#include <execution>
#include <vector>
#include <numeric>

int main() {
    std::vector<int> v = {5, 3, 1, 4, 2};

    // 并行排序
    std::sort(std::execution::par, v.begin(), v.end());

    // 并行查找
    auto it = std::find(std::execution::par, v.begin(), v.end(), 3);
    if (it != v.end()) {
        std::cout << "找到: " << *it << std::endl;
    }

    // 并行变换
    std::vector<int> result(v.size());
    std::transform(std::execution::par, v.begin(), v.end(), result.begin(),
                   [](int x) { return x * x; });

    std::cout << "平方: ";
    for (int x : result) std::cout << x << " ";
    std::cout << std::endl;

    // 并行归约
    int product = std::reduce(std::execution::par, v.begin(), v.end(), 1,
                              std::multiplies<int>());
    std::cout << "乘积: " << product << std::endl;

    return 0;
}
```

**输出：**
```
找到: 3
平方: 1 4 9 16 25
乘积: 120
```

### 2.2 执行策略对比

| 策略 | 特点 | 适用场景 |
|------|------|---------|
| `seq` | 顺序执行 | 小数据量 |
| `par` | 多线程并行 | 大数据量 |
| `par_unseq` | 并行+SIMD | 计算密集 |

## 三、注意事项与常见陷阱

- **需要编译器支持并链接TBB等库**：GCC需要`-ltbb`。
- **算法函数必须是线程安全的**：不能有副作用或数据竞争。
- **数据量太小时并行开销反而更大**：通常需要数千元素以上。
- **`par_unseq`允许向量化**：但函数不能有同步操作。
- **执行策略是提示**：编译器/运行时可选择忽略。
