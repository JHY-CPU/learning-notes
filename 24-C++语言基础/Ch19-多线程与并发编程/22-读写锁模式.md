# 读写锁模式

## 一、概念说明

读写锁（Read-Write Lock）允许多个读线程并发访问，写线程独占访问。适用于读多写少的场景，如缓存、配置管理。

```cpp
#include <iostream>
#include <thread>
#include <shared_mutex>
#include <vector>
#include <chrono>

class ConfigManager {
    std::string config_;
    mutable std::shared_mutex mtx_;

public:
    // 多线程可同时读
    std::string getConfig() const {
        std::shared_lock<std::shared_mutex> lock(mtx_);
        return config_;
    }

    // 写入时独占
    void setConfig(std::string cfg) {
        std::unique_lock<std::shared_mutex> lock(mtx_);
        config_ = std::move(cfg);
    }
};

int main() {
    ConfigManager cm;
    cm.setConfig("v1.0");

    // 多个读者
    std::vector<std::thread> readers;
    for (int i = 0; i < 4; ++i) {
        readers.emplace_back([&cm, i] {
            for (int j = 0; j < 3; ++j) {
                std::cout << "读者" << i << ": " << cm.getConfig() << std::endl;
                std::this_thread::sleep_for(std::chrono::milliseconds(10));
            }
        });
    }

    // 一个写者
    std::thread writer([&cm] {
        std::this_thread::sleep_for(std::chrono::milliseconds(50));
        cm.setConfig("v2.0");
        std::cout << "写者: 更新为v2.0" << std::endl;
    });

    for (auto& t : readers) t.join();
    writer.join();
    return 0;
}
```

**输出（顺序可能不同）：**
```
读者0: v1.0
读者1: v1.0
...
写者: 更新为v2.0
读者2: v2.0
```

## 二、具体用法

### 2.1 读写锁性能对比

```cpp
#include <iostream>
#include <thread>
#include <shared_mutex>
#include <mutex>
#include <chrono>

// 读写锁 vs 普通互斥锁

int data = 0;
std::shared_mutex rwMtx;
std::mutex normalMtx;

void testRWLock(int readers) {
    auto start = std::chrono::high_resolution_clock::now();

    std::vector<std::thread> threads;
    for (int i = 0; i < readers; ++i) {
        threads.emplace_back([] {
            for (int j = 0; j < 10000; ++j) {
                std::shared_lock<std::shared_mutex> lock(rwMtx);
                volatile int v = data;
                (void)v;
            }
        });
    }
    for (auto& t : threads) t.join();

    auto end = std::chrono::high_resolution_clock::now();
    auto ms = std::chrono::duration_cast<std::chrono::milliseconds>(end - start).count();
    std::cout << "读写锁(" << readers << "读者): " << ms << "ms" << std::endl;
}

int main() {
    testRWLock(4);
    testRWLock(8);

    std::cout << "读多写少场景，读写锁性能更优" << std::endl;
    return 0;
}
```

**输出（示例）：**
```
读写锁(4读者): 15ms
读写锁(8读者): 25ms
读多写少场景，读写锁性能更优
```

## 三、注意事项与常见陷阱

- **`shared_mutex`开销比普通`mutex`大**：读少写多时不划算。
- **写优先 vs 读优先**：不同实现策略不同，可能写饥饿。
- **读锁不能升级为写锁**：需先释放再获取。
- **`shared_timed_mutex`支持超时**：避免死锁。
- **C++14才有`shared_lock`，C++17才有`shared_mutex`**。
