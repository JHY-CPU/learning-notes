# 并发编程概述

## 一、概念说明

**并发（Concurrency）** 指多个任务在重叠的时间段内执行，**并行（Parallelism）** 指多个任务真正同时执行。并发是逻辑上的同时，并行是物理上的同时。

- **进程**：独立的地址空间，资源开销大，通信通过IPC
- **线程**：共享进程地址空间，创建开销小，通信通过共享内存

C++11起在`<thread>`、`<mutex>`、`<atomic>`、`<future>`等头文件中提供了标准并发支持。

```cpp
#include <iostream>
#include <thread>
#include <chrono>

void task(const std::string& name, int count) {
    for (int i = 0; i < count; ++i) {
        std::cout << name << " 步骤 " << i << std::endl;
        std::this_thread::sleep_for(std::chrono::milliseconds(100));
    }
}

int main() {
    std::cout << "主线程开始" << std::endl;

    // 创建两个并发任务
    std::thread t1(task, "任务A", 3);
    std::thread t2(task, "任务B", 3);

    t1.join(); // 等待线程结束
    t2.join();

    std::cout << "主线程结束" << std::endl;
    return 0;
}
```

**输出（顺序可能不同）：**
```
主线程开始
任务A 步骤 0
任务B 步骤 0
任务A 步骤 1
任务B 步骤 1
任务A 步骤 2
任务B 步骤 2
主线程结束
```

## 二、具体用法

### 2.1 并发模型对比

| 特性 | 多进程 | 多线程 |
|------|--------|--------|
| 地址空间 | 独立 | 共享 |
| 创建开销 | 大 | 小 |
| 通信方式 | IPC（管道/socket） | 共享内存 |
| 健壮性 | 高（隔离） | 低（相互影响） |
| 调试难度 | 中等 | 困难 |
| 适用场景 | 微服务、沙箱 | 计算密集、IO密集 |

### 2.2 获取硬件信息

```cpp
#include <iostream>
#include <thread>

int main() {
    // 获取硬件支持的并发线程数（逻辑核心数）
    unsigned int cores = std::thread::hardware_concurrency();
    std::cout << "硬件并发数: " << cores << std::endl;
    // 如果无法确定，返回0

    // 当前线程ID
    std::cout << "主线程ID: " << std::this_thread::get_id() << std::endl;

    return 0;
}
```

**输出（示例）：**
```
硬件并发数: 8
主线程ID: 12345
```

## 三、注意事项与常见陷阱

- **数据竞争是并发最大风险**：多个线程同时写同一数据导致未定义行为。
- **`thread`对象析构前必须`join`或`detach`**：否则`std::terminate`被调用。
- **线程局部存储`thread_local`**：每个线程有独立副本。
- **不要在热路径中创建/销毁线程**：开销很大，应使用线程池。
- **并发调试困难**：使用ThreadSanitizer等工具检测。
