# 线程局部存储

## 一、概念说明

`thread_local`关键字声明线程局部变量，每个线程有独立的副本。适用于线程私有的缓存、计数器、随机数生成器等。

```cpp
#include <iostream>
#include <thread>
#include <vector>

thread_local int threadCounter = 0;
thread_local std::string threadName;

void worker(int id) {
    threadName = "Worker-" + std::to_string(id);
    threadCounter = id * 100;

    for (int i = 0; i < 3; ++i) {
        ++threadCounter;
        std::cout << threadName << ": counter=" << threadCounter << std::endl;
    }
}

int main() {
    std::vector<std::thread> threads;
    for (int i = 0; i < 3; ++i) {
        threads.emplace_back(worker, i);
    }
    for (auto& t : threads) t.join();

    // 主线程的threadCounter仍是0
    std::cout << "主线程counter: " << threadCounter << std::endl;
    return 0;
}
```

**输出（顺序可能不同）：**
```
Worker-0: counter=101
Worker-1: counter=201
Worker-2: counter=301
Worker-0: counter=102
...
主线程counter: 0
```

## 二、具体用法

### 2.1 线程本地随机数

```cpp
#include <iostream>
#include <thread>
#include <random>
#include <mutex>

std::mutex coutMtx;

void generateRandom(int id) {
    // 每个线程独立的随机数生成器
    thread_local std::mt19937 gen(std::random_device{}());
    thread_local std::uniform_int_distribution<int> dist(1, 100);

    for (int i = 0; i < 3; ++i) {
        int val = dist(gen);
        std::lock_guard<std::mutex> lock(coutMtx);
        std::cout << "线程" << id << ": " << val << std::endl;
    }
}

int main() {
    std::thread t1(generateRandom, 1);
    std::thread t2(generateRandom, 2);
    t1.join();
    t2.join();
    return 0;
}
```

**输出（示例）：**
```
线程1: 42
线程1: 17
线程1: 88
线程2: 53
线程2: 91
线程2: 36
```

### 2.2 线程本地缓存

```cpp
#include <iostream>
#include <thread>
#include <unordered_map>
#include <string>

// 线程本地缓存
thread_local std::unordered_map<int, int> factorialCache;

int factorial(int n) {
    if (n <= 1) return 1;
    auto it = factorialCache.find(n);
    if (it != factorialCache.end()) return it->second;
    int result = n * factorial(n - 1);
    factorialCache[n] = result;
    return result;
}

int main() {
    std::thread t1([] {
        std::cout << "5! = " << factorial(5) << std::endl;
        std::cout << "缓存大小: " << factorialCache.size() << std::endl;
    });
    std::thread t2([] {
        std::cout << "3! = " << factorial(3) << std::endl;
        std::cout << "缓存大小: " << factorialCache.size() << std::endl;
    });
    t1.join();
    t2.join();
    return 0;
}
```

**输出：**
```
5! = 120
缓存大小: 4
3! = 6
缓存大小: 2
```

## 三、注意事项与常见陷阱

- **`thread_local`的初始化在线程首次使用时进行**：有延迟开销。
- **`thread_local`不能用于类的非静态成员**：只能用于全局/静态变量。
- **线程结束时析构`thread_local`对象**：析构顺序与构造相反。
- **大量线程+大`thread_local`对象=大量内存占用**。
- **`thread_local`与`static`不同**：`static`是全局共享的。
