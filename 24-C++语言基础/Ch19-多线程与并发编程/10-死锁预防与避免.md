# 死锁预防与避免

## 一、概念说明

预防死锁的方法：
1. **按序加锁**：所有线程按统一顺序获取锁
2. **`std::lock()`**：同时锁定多个mutex，无死锁
3. **`std::scoped_lock`（C++17）**：RAII版本的`lock()`
4. **避免嵌套锁**：持有一个锁时不要获取另一个
5. **超时机制**：`try_lock_for`避免无限等待

```cpp
#include <iostream>
#include <thread>
#include <mutex>

std::mutex mtx1, mtx2;

// 方法1：scoped_lock同时获取多个锁（推荐）
void safeOperation() {
    std::scoped_lock lock(mtx1, mtx2); // 同时获取，无死锁
    std::cout << "安全获取两个锁" << std::endl;
}

// 方法2：std::lock + adopt_lock
void safeOperation2() {
    std::lock(mtx1, mtx2); // 同时锁定，无死锁
    std::lock_guard<std::mutex> lock1(mtx1, std::adopt_lock);
    std::lock_guard<std::mutex> lock2(mtx2, std::adopt_lock);
    std::cout << "使用std::lock安全获取" << std::endl;
}

// 方法3：按序加锁（所有线程按同一顺序获取）
void orderedLock() {
    std::lock_guard<std::mutex> lock1(mtx1); // 总是先lock mtx1
    std::lock_guard<std::mutex> lock2(mtx2); // 再lock mtx2
    std::cout << "按序加锁" << std::endl;
}

int main() {
    std::thread t1(safeOperation);
    std::thread t2(safeOperation);
    t1.join();
    t2.join();

    std::thread t3(orderedLock);
    std::thread t4(orderedLock);
    t3.join();
    t4.join();

    return 0;
}
```

**输出：**
```
安全获取两个锁
安全获取两个锁
按序加锁
按序加锁
```

## 二、具体用法

### 2.1 层次锁

```cpp
#include <iostream>
#include <thread>
#include <mutex>
#include <stdexcept>

// 层次锁：线程只能按层次递减的顺序获取锁
class HierarchicalMutex {
    std::mutex mtx;
    unsigned long const hierarchyValue;
    unsigned long previousHierarchyValue;
    static thread_local unsigned long thisThreadHierarchy;

    void checkHierarchy() {
        if (thisThreadHierarchy <= hierarchyValue) {
            throw std::logic_error("违反锁层次顺序");
        }
        previousHierarchyValue = thisThreadHierarchy;
        thisThreadHierarchy = hierarchyValue;
    }

    void restoreHierarchy() {
        thisThreadHierarchy = previousHierarchyValue;
    }

public:
    explicit HierarchicalMutex(unsigned long val) : hierarchyValue(val) {}

    void lock() {
        checkHierarchy();
        mtx.lock();
    }

    void unlock() {
        mtx.unlock();
        restoreHierarchy();
    }

    bool try_lock() {
        checkHierarchy();
        if (!mtx.try_lock()) return false;
        return true;
    }
};

thread_local unsigned long HierarchicalMutex::thisThreadHierarchy(ULONG_MAX);

HierarchicalMutex highLevel(10000);
HierarchicalMutex lowLevel(5000);

void correctOrder() {
    std::lock_guard<HierarchicalMutex> lock1(highLevel); // 先获取高层级
    std::lock_guard<HierarchicalMutex> lock2(lowLevel);  // 再获取低层级
    std::cout << "层次锁顺序正确" << std::endl;
}

int main() {
    std::thread t(correctOrder);
    t.join();

    // std::thread t2([](){
    //     std::lock_guard<HierarchicalMutex> lock1(lowLevel);
    //     std::lock_guard<HierarchicalMutex> lock2(highLevel); // 异常！
    // });

    return 0;
}
```

**输出：**
```
层次锁顺序正确
```

## 三、注意事项与常见陷阱

- **`scoped_lock`是C++17最佳实践**：自动处理多个锁的获取顺序。
- **`std::lock()`可能多次尝试**：性能略低于按序加锁。
- **层次锁适合大型项目**：强制规范锁的获取顺序。
- **锁的粒度越细越好**：减少锁持有时间。
- **避免在持有锁时做IO或复杂计算**：会延长锁持有时间。
