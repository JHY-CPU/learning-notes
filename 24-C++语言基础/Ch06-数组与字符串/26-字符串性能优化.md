# 字符串性能优化

## 一、概念说明

`std::string`的性能优化包括：**SSO（Short String Optimization）**短字符串优化、移动语义避免拷贝、`reserve`预分配、`string_view`避免临时对象等。

## 二、具体用法

### 2.1 SSO短字符串优化

```cpp
// 大多数实现：短字符串（约15-22字符）直接存储在对象内部
// 避免堆分配，极大提升短字符串性能

std::string shortStr = "Hello";        // SSO，无堆分配
std::string longStr(100, 'A');         // 超过SSO阈值，堆分配

std::cout << "short capacity: " << shortStr.capacity() << std::endl;
std::cout << "long capacity: " << longStr.capacity() << std::endl;
// 输出可能为:
// short capacity: 15 (或22，取决于实现)
// long capacity: 100
```

### 2.2 移动语义

```cpp
std::string create() {
    std::string result = "This is a very long string...";
    return result;  // 移动（或RVO），无拷贝
}

std::string s = create();  // 高效
std::cout << s << std::endl;

// 显式移动
std::string src = "data";
std::string dst = std::move(src);  // src变为空
std::cout << dst << std::endl;     // 输出: data
```

### 2.3 reserve预分配

```cpp
// 不预留空间：可能多次扩容
std::string s1;
for (int i = 0; i < 10000; ++i) {
    s1 += "a";  // 每次可能触发扩容和拷贝
}

// 预留空间：一次分配
std::string s2;
s2.reserve(10000);
for (int i = 0; i < 10000; ++i) {
    s2 += "a";  // 无需扩容
}

std::cout << "s1 capacity: " << s1.capacity() << std::endl;
std::cout << "s2 capacity: " << s2.capacity() << std::endl;
```

### 2.4 string_view避免拷贝

```cpp
// 函数参数使用string_view避免临时string创建
void process(std::string_view sv) {
    std::cout << sv.substr(0, 5) << std::endl;
}

process("Hello World");  // 无拷贝，直接使用字面量
// 输出: Hello
```

### 2.5 append vs +

```cpp
std::string s;

// 高效：原地追加
s.append("Hello");
s.append(" World");

// 低效：每次创建新临时对象
// s = s + "Hello" + " World";
```

## 三、注意事项与常见陷阱

- SSO阈值因编译器实现而异（通常15或22字符）
- 移动后的string处于有效但未指定状态，只可销毁或赋值
- `reserve`只改变capacity，不改变size
- 字符串拼接在循环中优先用`append`或`+=`
- 对性能关键路径，使用benchmark工具测量
