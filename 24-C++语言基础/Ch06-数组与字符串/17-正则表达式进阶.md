# 正则表达式进阶

## 一、概念说明

本节深入正则表达式的高级功能：分组捕获、回调替换、命名捕获组、非贪婪匹配，以及正则表达式的性能注意事项。

## 二、具体用法

### 2.1 分组捕获

```cpp
std::string text = "2024-01-15";
std::regex datePat(R"((\d{4})-(\d{2})-(\d{2}))");
std::smatch m;

if (std::regex_match(text, m, datePat)) {
    std::cout << "完整: " << m[0] << std::endl;  // 输出: 2024-01-15
    std::cout << "年: " << m[1] << std::endl;     // 输出: 2024
    std::cout << "月: " << m[2] << std::endl;     // 输出: 01
    std::cout << "日: " << m[3] << std::endl;     // 输出: 15
}
```

### 2.2 回调替换

```cpp
std::string text = "price: 100, tax: 20";
std::regex numPat(R"(\d+)");

// 将所有数字乘以2
std::string result = std::regex_replace(text, numPat,
    [](const std::smatch& m) {
        int val = std::stoi(m.str());
        return std::to_string(val * 2);
    });
std::cout << result << std::endl;
// 输出: price: 200, tax: 40
```

### 2.3 非贪婪匹配

```cpp
// 默认贪婪：匹配尽可能多的字符
std::string html = "<b>bold</b> and <i>italic</i>";
std::regex greedy(R"(<.*>)");
std::smatch m;
std::regex_search(html, m, greedy);
std::cout << "贪婪: " << m[0] << std::endl;
// 输出: 贪婪: <b>bold</b> and <i>italic</i>

// 非贪婪：匹配尽可能少的字符（加?）
std::regex lazy(R"(<.*?>)");
std::regex_search(html, m, lazy);
std::cout << "非贪婪: " << m[0] << std::endl;
// 输出: 非贪婪: <b>
```

### 2.4 命名捕获组

```cpp
// ECMAScript语法支持命名组
std::string line = "Name: Alice, Age: 25";
std::regex pat(R"(Name: (?<name>\w+), Age: (?<age>\d+))");
std::smatch m;
if (std::regex_search(line, m, pat)) {
    std::cout << m[1] << " is " << m[2] << " years old" << std::endl;
}
// 输出: Alice is 25 years old
```

### 2.5 迭代器遍历所有匹配

```cpp
std::string text = "IP: 192.168.1.1 and 10.0.0.1";
std::regex ipPat(R"(\d+\.\d+\.\d+\.\d+)");

for (auto it = std::sregex_iterator(text.begin(), text.end(), ipPat);
     it != std::sregex_iterator(); ++it) {
    std::cout << "IP: " << it->str() << std::endl;
}
// 输出:
// IP: 192.168.1.1
// IP: 10.0.0.1
```

## 三、注意事项与常见陷阱

- 贪婪与非贪婪匹配结果差异大，注意选择
- 正则对象编译开销大，应作为静态/全局对象复用
- 回调替换在C++11后支持lambda，更灵活
- 灾难性回溯：嵌套量词可导致指数级时间复杂度
- 对于简单字符串操作，正则可能过度，考虑用`find`/`substr`
