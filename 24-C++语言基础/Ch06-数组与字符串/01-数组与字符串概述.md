# 数组与字符串概述

## 一、概念说明

数组是最基本的数据结构，用于存储相同类型的连续元素。C++提供了两种数组体系：**C风格数组**（原生数组）和**C++标准容器**（`std::array`、`std::vector`）。字符串同样有C风格（`char[]`）和C++风格（`std::string`）两种。

本章全面覆盖数组操作、字符串处理、STL算法等核心内容。

## 二、对比表

| 特性 | C风格数组 | std::array | std::vector | char[] | std::string |
|------|----------|-----------|-------------|--------|-------------|
| 大小 | 编译期固定 | 编译期固定 | 运行时动态 | 编译期固定 | 运行时动态 |
| 越界检查 | 无 | at()有 | at()有 | 无 | at()有 |
| 知道大小 | 不直接知道 | size() | size() | strlen() | size() |
| 可拷贝 | 不可 | 可以 | 可以 | 不可 | 可以 |
| 传递给函数 | 退化为指针 | 按值/引用 | 按值/引用 | 退化为指针 | 按值/引用 |

## 二、选择指南

```cpp
// 固定大小 + 编译期已知 → std::array
std::array<int, 5> fixed = {1, 2, 3, 4, 5};

// 动态大小 → std::vector
std::vector<int> dynamic = {1, 2, 3};

// 字符串操作 → std::string
std::string name = "C++ Programming";

// 与C API交互 → C风格数组或string.c_str()
const char* cstr = name.c_str();
```

## 三、注意事项与常见陷阱

- C风格数组不包含长度信息，容易越界
- 优先使用STL容器而非原生数组
- `std::string`比C字符串安全但有少量开销
- 字符串字面量有多种前缀（`L""`, `u""`, `u8""`, `U""`）
- 多维数组在内存中是线性排列的（行优先）
