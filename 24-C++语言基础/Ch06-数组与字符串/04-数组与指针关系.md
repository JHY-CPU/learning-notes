# 数组与指针关系

## 一、概念说明

C++中数组和指针关系密切：数组名在大多数表达式中**退化为指向首元素的指针**。但数组**不是**指针——`sizeof(arr)`返回整个数组大小，而`sizeof(ptr)`返回指针大小。

`&arr`（整个数组的地址）和`arr`（退化为首元素指针）**值相同但类型不同**。

## 二、具体用法

### 2.1 数组退化为指针

```cpp
int arr[5] = {10, 20, 30, 40, 50};

// arr退化为 int*
int* p = arr;

std::cout << *p << std::endl;       // 输出: 10
std::cout << *(p + 2) << std::endl; // 输出: 30

// sizeof区别
std::cout << sizeof(arr) << std::endl;  // 输出: 20 (5*4字节)
std::cout << sizeof(p) << std::endl;    // 输出: 8 (指针大小)
```

### 2.2 &arr vs arr

```cpp
int arr[5] = {1, 2, 3, 4, 5};

std::cout << "arr  = " << arr << std::endl;    // 首元素地址
std::cout << "&arr = " << &arr << std::endl;   // 数组地址（值相同）

// 类型不同！
int* p1 = arr;         // int*
int (*p2)[5] = &arr;   // int(*)[5] 指向含5个int的数组

// 指针运算不同
std::cout << "arr+1  = " << (void*)(arr+1) << std::endl;  // +4字节
std::cout << "&arr+1 = " << (void*)(&arr+1) << std::endl; // +20字节
```

### 2.3 数组作为函数参数退化

```cpp
// 以下三种声明等价，arr都退化为指针
void func(int arr[]);
void func(int arr[5]);  // 5被忽略！
void func(int* arr);

// 传递数组大小的方法
void process(int* arr, size_t size) {
    for (size_t i = 0; i < size; ++i) {
        std::cout << arr[i] << " ";
    }
}
```

### 2.4 不退化的场景

```cpp
int arr[5] = {1, 2, 3, 4, 5};

// sizeof不退化
std::cout << sizeof(arr) / sizeof(arr[0]) << std::endl;  // 输出: 5

// 引用不退化
int (&ref)[5] = arr;  // 引用整个数组
std::cout << sizeof(ref) << std::endl;  // 输出: 20

// decltype不退化
using ArrType = decltype(arr);  // int[5]，非int*
```

## 三、注意事项与常见陷阱

- 数组传参会丢失大小信息，需额外传递size参数
- `&arr + 1`跳过整个数组，`arr + 1`仅跳过一个元素
- 函数参数中`int arr[10]`实际是`int*`，`sizeof`返回指针大小
- 用引用传参数组可保留大小信息：`void f(int (&arr)[5])`
- C++17的`std::size()`可用于获取C风格数组大小
