# 安全字符串操作

## 一、概念说明

缓冲区溢出是C/C++中最常见的安全漏洞之一。安全的字符串操作需要：**边界检查**、**使用安全函数**、**优先使用STL容器**。C11引入了带`_s`后缀的安全函数，C++则推荐使用`std::string`。

## 二、具体用法

### 2.1 不安全 vs 安全操作

```cpp
// 不安全：可能溢出
char dest[10];
// strcpy(dest, "This is a very long string");  // 溢出！

// 安全：strncpy限制长度
char safe[10];
strncpy(safe, "Hello World", sizeof(safe) - 1);
safe[sizeof(safe) - 1] = '\0';  // 确保终止
std::cout << safe << std::endl;  // 输出: Hello Wor
```

### 2.2 C11安全函数

```cpp
// strcpy_s, strcat_s, sprintf_s 等（Windows/MSCV支持）
// errno_t strcpy_s(char *dest, rsize_t destsz, const char *src);

char buf[20];
// strcpy_s(buf, sizeof(buf), "Hello");  // MSVC专用
```

### 2.3 snprintf替代sprintf

```cpp
char buf[20];

// 不安全：sprintf不检查长度
// sprintf(buf, "%s %s %s", "Very", "Long", "String");  // 溢出！

// 安全：snprintf自动截断
int written = snprintf(buf, sizeof(buf), "%s %s %s",
                       "Very", "Long", "String");
std::cout << buf << std::endl;        // 输出: Very Long String
std::cout << "需要: " << written << " 字节" << std::endl;  // 需要: 17 字节
```

### 2.4 std::string（推荐）

```cpp
// 最安全的方式：使用std::string
std::string s = "Hello";
s += " World";  // 自动管理内存，无溢出风险
std::cout << s << std::endl;  // 输出: Hello World

// 使用std::string_view避免不必要拷贝
void safeFunc(std::string_view sv) {
    std::cout << sv.substr(0, 10) << std::endl;
}
```

### 2.5 输入验证

```cpp
// 安全的用户输入
char name[50];
std::cin.getline(name, sizeof(name));  // 限制读取长度

if (std::cin.fail()) {
    std::cin.clear();
    std::cin.ignore(10000, '\n');
    std::cout << "输入过长\n";
}
```

## 三、注意事项与常见陷阱

- 始终使用`strncpy`/`snprintf`而非`strcpy`/`sprintf`
- `strncpy`可能不添加`\0`，必须手动处理
- 优先使用`std::string`避免所有缓冲区溢出问题
- `gets()`已从C11移除，绝对不要使用
- `scanf("%s", buf)`不安全，用`fgets`或`std::cin.getline`
