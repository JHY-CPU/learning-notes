# 数组与范围检查

## 一、概念说明

原生C风格数组不提供范围检查，越界访问是未定义行为。C++提供了更安全的替代方案：`std::array`的`at()`方法提供运行时边界检查，C++20的`std::span`提供了轻量级的连续序列视图。

## 二、具体用法

### 2.1 std::array与at()

```cpp
#include <array>

std::array<int, 5> arr = {10, 20, 30, 40, 50};

// at()进行边界检查
try {
    std::cout << arr.at(2) << std::endl;  // 输出: 30
    std::cout << arr.at(10) << std::endl; // 抛出异常
} catch (const std::out_of_range& e) {
    std::cout << "越界: " << e.what() << std::endl;
}

// operator[] 不检查
// arr[10];  // 未定义行为，不报错
```

### 2.2 std::span（C++20）

```cpp
#include <span>

void print(std::span<const int> sp) {
    std::cout << "大小: " << sp.size() << std::endl;
    for (int x : sp) std::cout << x << " ";
    std::cout << std::endl;
}

// 从vector
std::vector<int> v = {1, 2, 3, 4, 5};
print(v);
// 输出:
// 大小: 5
// 1 2 3 4 5

// 从C风格数组
int arr[] = {10, 20, 30};
print(arr);
// 输出:
// 大小: 3
// 10 20 30

// 子视图
std::span sub = std::span(v).subspan(1, 3);
print(sub);
// 输出:
// 大小: 3
// 2 3 4
```

### 2.3 bounds-checked访问

```cpp
// 自定义安全访问包装
template <typename Container>
auto safeAt(Container& c, size_t i) -> decltype(auto) {
    if (i >= std::size(c))
        throw std::out_of_range("index out of range");
    return c[i];
}

int arr[] = {1, 2, 3};
try {
    safeAt(arr, 5);
} catch (const std::exception& e) {
    std::cout << e.what() << std::endl;  // 输出: index out of range
}
```

## 三、注意事项与常见陷阱

- `std::array::at()`有运行时开销，性能敏感场景用`operator[]`
- `std::span`是非拥有视图，不延长底层数据的生命周期
- `span`可以是静态大小（`span<int, 5>`）或动态大小（`span<int>`）
- Debug模式下很多STL实现提供边界检查
- 考虑使用地址消毒器（ASan）检测越界访问
