# C++11 线程库

## 一、概念说明

C++11在`<thread>`、`<mutex>`、`<condition_variable>`、`<future>`、`<atomic>`中提供了标准线程支持（C++11 §30），结束了依赖平台API（Windows线程、POSIX线程）的历史。这使得C++成为真正的多线程语言。

### 1.1 核心组件一览

| 头文件 | 组件 | 用途 |
|--------|------|------|
| `<thread>` | `std::thread`, `this_thread` | 创建和管理线程 |
| `<mutex>` | `mutex`, `lock_guard`, `unique_lock`, `scoped_lock` | 互斥锁 |
| `<condition_variable>` | `condition_variable` | 条件变量（线程同步） |
| `<future>` | `future`, `promise`, `async`, `packaged_task` | 异步任务和返回值 |
| `<atomic>` | `atomic`, `atomic_flag` | 无锁原子操作 |

```cpp
#include <iostream>
#include <thread>
#include <atomic>

std::atomic<int> counter{0};

void increment(int times) {
    for (int i = 0; i < times; ++i) {
        counter.fetch_add(1, std::memory_order_relaxed);
    }
}

int main() {
    std::thread t1(increment, 100000);
    std::thread t2(increment, 100000);
    t1.join();
    t2.join();

    std::cout << "counter=" << counter.load() << std::endl;
    return 0;
}
```

**输出：**
```
counter=200000
```

## 二、具体用法

### 2.1 thread基础

```cpp
#include <iostream>
#include <thread>
#include <vector>

void worker(int id) {
    std::cout << "线程 " << id << " 开始, ID=" << std::this_thread::get_id() << std::endl;
    std::this_thread::sleep_for(std::chrono::milliseconds(100));
    std::cout << "线程 " << id << " 结束" << std::endl;
}

int main() {
    // 创建多个线程
    std::vector<std::thread> threads;
    for (int i = 0; i < 3; ++i) {
        threads.emplace_back(worker, i);
    }

    // 等待所有线程
    for (auto& t : threads) {
        t.join();
    }

    std::cout << "所有线程完成" << std::endl;
    return 0;
}
```

**输出：**
```
线程 0 开始, ID=...
线程 1 开始, ID=...
线程 2 开始, ID=...
线程 0 结束
线程 1 结束
线程 2 结束
所有线程完成
```

### 2.2 mutex与锁

```cpp
#include <iostream>
#include <thread>
#include <mutex>
#include <vector>

int shared_counter = 0;
std::mutex mtx;

void safe_increment(int times) {
    for (int i = 0; i < times; ++i) {
        std::lock_guard<std::mutex> lock(mtx); // RAII锁
        ++shared_counter;
    }
}

int main() {
    std::thread t1(safe_increment, 10000);
    std::thread t2(safe_increment, 10000);
    t1.join();
    t2.join();

    std::cout << "shared_counter=" << shared_counter << std::endl;
    return 0;
}
```

**输出：**
```
shared_counter=20000
```

### 2.3 future与async

```cpp
#include <iostream>
#include <future>
#include <vector>

int computeSum(int n) {
    int sum = 0;
    for (int i = 1; i <= n; ++i) sum += i;
    return sum;
}

int main() {
    // 异步执行
    auto future1 = std::async(std::launch::async, computeSum, 1000);
    auto future2 = std::async(std::launch::async, computeSum, 2000);

    // 获取结果（会阻塞直到完成）
    std::cout << "sum(1..1000)=" << future1.get() << std::endl;
    std::cout << "sum(1..2000)=" << future2.get() << std::endl;

    return 0;
}
```

**输出：**
```
sum(1..1000)=500500
sum(1..2000)=2001000
```

### 2.4 promise与future配对

```cpp
#include <iostream>
#include <thread>
#include <future>
#include <string>

void producer(std::promise<std::string> prom) {
    std::this_thread::sleep_for(std::chrono::milliseconds(100));
    prom.set_value("来自生产者的数据");
}

int main() {
    std::promise<std::string> prom;
    auto future = prom.get_future();

    std::thread t(producer, std::move(prom));

    std::cout << "等待数据..." << std::endl;
    std::cout << "收到: " << future.get() << std::endl;

    t.join();
    return 0;
}
```

**输出：**
```
等待数据...
收到: 来自生产者的数据
```

## 三、注意事项与常见陷阱

1. **`thread`析构前必须`join`或`detach`**：否则`std::terminate()`被调用。
2. **`mutex`不能拷贝**：互斥锁只能移动（如果支持）。
3. **`future`析构会阻塞**：如果关联`async`任务（`std::launch::async`），析构时等待完成。
4. **`atomic`只保证单个变量的原子性**：复合操作需要额外同步。
5. **避免死锁**：按固定顺序获取锁，或使用`std::scoped_lock`同时获取多个锁。
6. **详细内容参见Ch19多线程章节**。
