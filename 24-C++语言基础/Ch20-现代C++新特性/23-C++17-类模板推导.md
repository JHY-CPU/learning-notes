# C++17 类模板推导 (CTAD)

## 一、概念说明

C++17引入类模板参数推导（CTAD，Class Template Argument Deduction，C++17 §17.9），允许在构造类模板对象时不显式指定模板参数。编译器从构造函数参数自动推导类型，减少冗长的类型声明。

### 1.1 之前 vs 之后

```cpp
// C++17之前：需要显式指定
std::pair<int, double> p1(42, 3.14);
std::vector<int> v1 = {1, 2, 3};
std::lock_guard<std::mutex> lock1(mtx);

// C++17 CTAD：自动推导
std::pair p2(42, 3.14);               // pair<int, double>
std::vector v2 = {1, 2, 3};           // vector<int>
std::lock_guard lock2(mtx);           // lock_guard<mutex>
```

```cpp
#include <iostream>
#include <vector>
#include <string>
#include <mutex>
#include <utility>
#include <array>

int main() {
    // pair/tuple推导
    std::pair p(42, 3.14);
    std::tuple t(1, 2.5, std::string("hello"));

    // vector推导
    std::vector v = {1, 2, 3};
    std::vector vs = {std::string("a"), std::string("b")};

    // array推导
    std::array a = {1, 2, 3, 4, 5}; // array<int, 5>

    std::cout << "p: " << p.first << ", " << p.second << std::endl;
    std::cout << "v大小: " << v.size() << std::endl;
    std::cout << "a大小: " << a.size() << std::endl;

    return 0;
}
```

**输出：**
```
p: 42, 3.14
v大小: 3
a大小: 5
```

## 二、具体用法

### 2.1 自定义推导指引

当编译器无法正确推导时，可以提供推导指引（Deduction Guides）。

```cpp
#include <iostream>
#include <string>

template <typename T>
class Wrapper {
    T value;
public:
    Wrapper(T v) : value(std::move(v)) {}
    const T& get() const { return value; }
};

// 推导指引：const char* 推导为 string
template <typename T>
Wrapper(const char*) -> Wrapper<std::string>;

// 推导指引：指针推导为对应类型
template <typename T>
Wrapper(T*) -> Wrapper<T>;

int main() {
    Wrapper w1(42);             // Wrapper<int>
    Wrapper w2(3.14);           // Wrapper<double>
    Wrapper w3("hello");        // Wrapper<std::string>（通过指引）

    std::cout << w1.get() << std::endl;
    std::cout << w2.get() << std::endl;
    std::cout << w3.get() << std::endl;

    return 0;
}
```

**输出：**
```
42
3.14
hello
```

### 2.2 实际应用

```cpp
#include <iostream>
#include <vector>
#include <memory>
#include <functional>

// 自定义容器
template <typename T>
class Stack {
    std::vector<T> data;
public:
    // 推导指引从initializer_list
    Stack(std::initializer_list<T> init) : data(init) {}
    void push(T val) { data.push_back(std::move(val)); }
    T pop() { T val = std::move(data.back()); data.pop_back(); return val; }
    bool empty() const { return data.empty(); }
};

int main() {
    // 自动推导为Stack<int>
    Stack s = {1, 2, 3, 4, 5};
    std::cout << "top: " << s.pop() << std::endl;

    // make_unique/make_shared风格的推导
    auto p = std::make_unique<int>(42); // C++14就有

    return 0;
}
```

### 2.3 聚合类型的推导

```cpp
#include <iostream>

struct Point {
    double x, y;
};

int main() {
    // 聚合类型也可以推导（C++17）
    Point p{3.0, 4.0};
    auto [x, y] = p;
    std::cout << "Point(" << x << ", " << y << ")" << std::endl;

    // std::array推导
    std::array a{1, 2, 3, 4, 5}; // std::array<int, 5>
    std::cout << "array大小: " << a.size() << std::endl;

    return 0;
}
```

## 三、注意事项与常见陷阱

1. **推导指引需要与类模板在同一命名空间**：否则编译器找不到。
2. **聚合类型也可以推导**：`std::array a{1, 2, 3}`推导为`array<int, 3>`。
3. **C++20允许从initializer_list推导**：进一步改进。
4. **推导失败时编译器报错**：需要显式指定模板参数。
5. **CTAD让代码更简洁但可能降低可读性**：复杂类型建议显式指定。
6. **`std::vector v{1, 2, 3}` vs `std::vector v(1, 2, 3)`**：前者是`{1,2,3}`，后者可能编译错误（不匹配任何构造函数）。
