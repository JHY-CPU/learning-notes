# C++17 并行算法

## 一、概念说明

C++17为标准库算法添加了执行策略参数（C++17 §29.5），支持顺序、并行和向量化执行。定义在`<execution>`头文件中，允许程序员控制算法的执行方式，充分利用多核CPU和SIMD指令。

### 1.1 执行策略

| 策略 | 含义 | 线程安全要求 |
|------|------|-------------|
| `std::execution::seq` | 顺序执行 | 无 |
| `std::execution::par` | 并行执行 | 回调必须线程安全 |
| `std::execution::par_unseq` | 并行+向量化 | 回调不能有同步 |
| `std::execution::unseq` | 向量化（C++20） | 不能有同步 |

```cpp
#include <iostream>
#include <algorithm>
#include <execution>
#include <vector>
#include <numeric>

int main() {
    std::vector<int> data(100000);
    std::iota(data.begin(), data.end(), 0);

    // 顺序执行
    auto sum1 = std::reduce(std::execution::seq, data.begin(), data.end(), 0LL);

    // 并行执行
    auto sum2 = std::reduce(std::execution::par, data.begin(), data.end(), 0LL);

    // 并行+向量化
    auto sum3 = std::reduce(std::execution::par_unseq, data.begin(), data.end(), 0LL);

    std::cout << "seq: " << sum1 << std::endl;
    std::cout << "par: " << sum2 << std::endl;
    std::cout << "par_unseq: " << sum3 << std::endl;

    return 0;
}
```

**输出：**
```
seq: 4999950000
par: 4999950000
par_unseq: 4999950000
```

## 二、具体用法

### 2.1 支持的算法

```cpp
#include <iostream>
#include <algorithm>
#include <execution>
#include <vector>
#include <numeric>

int main() {
    std::vector<int> v(10000);
    std::iota(v.begin(), v.end(), 1);

    // 并行排序
    std::sort(std::execution::par, v.begin(), v.end(), std::greater<>());

    // 并行for_each
    std::for_each(std::execution::par, v.begin(), v.end(),
                  [](int& x) { x *= 2; });

    // 并行查找
    auto it = std::find(std::execution::par, v.begin(), v.end(), 1000);

    // 并行transform
    std::vector<int> result(v.size());
    std::transform(std::execution::par, v.begin(), v.end(), result.begin(),
                   [](int x) { return x * x; });

    // 并行归约
    auto sum = std::reduce(std::execution::par, v.begin(), v.end(), 0LL);

    // 并行count
    auto cnt = std::count(std::execution::par, v.begin(), v.end(), 42);

    std::cout << "并行算法完成" << std::endl;
    std::cout << "sum=" << sum << ", count=" << cnt << std::endl;
    return 0;
}
```

### 2.2 性能对比

```cpp
#include <iostream>
#include <algorithm>
#include <execution>
#include <vector>
#include <numeric>
#include <chrono>

int main() {
    std::vector<int> data(10000000);
    std::iota(data.begin(), data.end(), 0);
    std::shuffle(data.begin(), data.end(), std::mt19937{42});

    // 测试并行排序
    auto copy1 = data;
    auto start = std::chrono::high_resolution_clock::now();
    std::sort(std::execution::seq, copy1.begin(), copy1.end());
    auto seq_time = std::chrono::duration_cast<std::chrono::milliseconds>(
        std::chrono::high_resolution_clock::now() - start).count();

    auto copy2 = data;
    start = std::chrono::high_resolution_clock::now();
    std::sort(std::execution::par, copy2.begin(), copy2.end());
    auto par_time = std::chrono::duration_cast<std::chrono::milliseconds>(
        std::chrono::high_resolution_clock::now() - start).count();

    std::cout << "顺序排序: " << seq_time << "ms" << std::endl;
    std::cout << "并行排序: " << par_time << "ms" << std::endl;
    if (seq_time > 0)
        std::cout << "加速比: " << (double)seq_time / par_time << "x" << std::endl;

    return 0;
}
```

### 2.3 注意执行安全

```cpp
#include <iostream>
#include <algorithm>
#include <execution>
#include <vector>
#include <mutex>

int main() {
    std::vector<int> v = {1, 2, 3, 4, 5};
    std::mutex mtx;
    std::vector<int> results;

    // 错误：results.push_back不是线程安全的
    // std::for_each(std::execution::par, v.begin(), v.end(),
    //     [&](int x) { results.push_back(x); }); // 数据竞争！

    // 正确：加锁保护
    std::for_each(std::execution::par, v.begin(), v.end(),
        [&](int x) {
            std::lock_guard<std::mutex> lock(mtx);
            results.push_back(x);
        });

    std::cout << "安全并行操作完成" << std::endl;
    return 0;
}
```

## 三、注意事项与常见陷阱

1. **需要链接并行后端**：GCC需要TBB（`-ltbb`）或libpstl；MSVC默认支持。
2. **算法回调必须线程安全**（`par`和`par_unseq`）：lambda不能有数据竞争。
3. **数据量太小并行开销得不偿失**：通常需要数千元素以上才能体现并行优势。
4. **`par_unseq`允许向量化**：回调函数不能有同步操作（锁、原子变量等）。
5. **不是所有算法都支持所有策略**：查cppreference确认。
6. **异常行为**：并行算法中抛出异常可能导致程序终止。
7. **详细内容参见Ch16并行算法章节**。
