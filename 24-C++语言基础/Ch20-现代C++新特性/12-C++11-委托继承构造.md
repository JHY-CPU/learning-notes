# C++11 委托构造与继承构造

## 一、概念说明

**委托构造**（Delegating Constructors，C++11 §12.6.2）：一个构造函数可以调用同类的另一个构造函数，避免代码重复。
**继承构造**（Inheriting Constructors，C++11 §7.3.3）：使用`using Base::Base`让派生类继承基类的构造函数。

### 1.1 为什么需要

在C++11之前，多个构造函数之间常有重复的初始化逻辑，只能提取到`init()`私有方法中。委托构造让代码更直接。

```cpp
#include <iostream>
#include <string>

class Connection {
    std::string host;
    int port;
    bool ssl;
public:
    // 主构造函数（包含所有初始化逻辑）
    Connection(std::string h, int p, bool s)
        : host(std::move(h)), port(p), ssl(s) {
        std::cout << "连接: " << host << ":" << port
                  << (ssl ? " (SSL)" : "") << std::endl;
    }

    // 委托构造：调用主构造函数
    Connection() : Connection("localhost", 8080, false) {}
    Connection(std::string h) : Connection(h, 8080, false) {}
    Connection(std::string h, int p) : Connection(h, p, false) {}
};

int main() {
    Connection c1;
    Connection c2("api.example.com");
    Connection c3("secure.example.com", 443, true);
    return 0;
}
```

**输出：**
```
连接: localhost:8080
连接: api.example.com:8080
连接: secure.example.com:443 (SSL)
```

## 二、具体用法

### 2.1 委托构造的最佳实践

```cpp
#include <iostream>
#include <string>

class Config {
    std::string name_;
    int timeout_;
    bool debug_;
    std::string logFile_;

    // 初始化逻辑集中
    void initDefaults() {
        timeout_ = 30;
        debug_ = false;
        logFile_ = "app.log";
    }

public:
    // 主构造函数
    Config(std::string name, int timeout, bool debug, std::string logFile)
        : name_(std::move(name))
        , timeout_(timeout)
        , debug_(debug)
        , logFile_(std::move(logFile)) {}

    // 委托构造：简化版本
    Config(std::string name)
        : Config(std::move(name), 30, false, "app.log") {}

    Config(std::string name, int timeout)
        : Config(std::move(name), timeout, false, "app.log") {}

    void print() const {
        std::cout << "Config{name=" << name_
                  << ", timeout=" << timeout_
                  << ", debug=" << debug_
                  << ", log=" << logFile_ << "}" << std::endl;
    }
};

int main() {
    Config c1("default");
    Config c2("production", 60);
    Config c3("dev", 10, true, "debug.log");

    c1.print();
    c2.print();
    c3.print();

    return 0;
}
```

**输出：**
```
Config{name=default, timeout=30, debug=0, log=app.log}
Config{name=production, timeout=60, debug=0, log=app.log}
Config{name=dev, timeout=10, debug=1, log=debug.log}
```

### 2.2 继承构造

```cpp
#include <iostream>
#include <string>

class Base {
    int value;
public:
    Base() : value(0) {
        std::cout << "Base()" << std::endl;
    }
    Base(int v) : value(v) {
        std::cout << "Base(" << v << ")" << std::endl;
    }
    Base(int v, int extra) : value(v + extra) {
        std::cout << "Base(" << v << ", " << extra << ")" << std::endl;
    }
    int getValue() const { return value; }
};

class Derived : public Base {
    std::string label;
public:
    using Base::Base; // 继承所有Base的构造函数

    // 派生类自己的构造函数
    Derived(int v, std::string l) : Base(v), label(std::move(l)) {
        std::cout << "Derived(int, string)" << std::endl;
    }

    void show() const {
        std::cout << "Derived value=" << getValue() << std::endl;
    }
};

int main() {
    Derived d1;           // 调用Base()
    Derived d2(42);       // 调用Base(int)
    Derived d3(10, 20);   // 调用Base(int, int)
    Derived d4(5, "test"); // 调用Derived(int, string)

    d2.show();
    return 0;
}
```

**输出：**
```
Base()
Base(42)
Base(10, 20)
Base(5)
Derived(int, string)
Derived value=42
```

### 2.3 委托构造的异常安全

```cpp
#include <iostream>
#include <memory>
#include <stdexcept>

class Resource {
    int* data;
    size_t size;
public:
    // 主构造函数：可能抛异常
    Resource(size_t n) : data(new int[n]), size(n) {
        std::cout << "分配" << n << "个int" << std::endl;
    }

    // 委托构造：如果n为0则抛异常
    Resource(size_t n, bool mustBePositive) : Resource(n) {
        if (!mustBePositive && n == 0) {
            // 注意：此时data已分配，需要清理
            // 更好的做法是在委托前检查
            throw std::invalid_argument("n必须为正数");
        }
    }

    ~Resource() {
        delete[] data;
        std::cout << "释放" << std::endl;
    }
};
```

## 三、注意事项与常见陷阱

1. **委托构造不能有初始化列表的成员**：所有成员初始化在被委托的构造函数中完成。
2. **委托构造中不能使用`this`访问成员**：成员尚未初始化完成。
3. **继承构造不继承默认参数**：需要派生类显式定义需要的版本。
4. **继承构造遇到冲突时**：派生类自己的构造函数优先。
5. **虚继承的类不能用继承构造**：限制较多。
6. **委托构造链中抛异常**：已构造的部分会被正常析构。
