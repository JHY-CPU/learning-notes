# C++11 委托构造与继承构造

## 一、概念说明

**委托构造**：一个构造函数可以调用同类的另一个构造函数，避免代码重复。
**继承构造**：使用`using Base::Base`让派生类继承基类的构造函数。

```cpp
#include <iostream>
#include <string>

class Connection {
    std::string host;
    int port;
    bool ssl;
public:
    // 主构造函数
    Connection(std::string h, int p, bool s)
        : host(std::move(h)), port(p), ssl(s) {
        std::cout << "连接: " << host << ":" << port
                  << (ssl ? " (SSL)" : "") << std::endl;
    }

    // 委托构造
    Connection() : Connection("localhost", 8080, false) {}
    Connection(std::string h) : Connection(h, 8080, false) {}
    Connection(std::string h, int p) : Connection(h, p, false) {}
};

int main() {
    Connection c1;
    Connection c2("api.example.com");
    Connection c3("secure.example.com", 443, true);
    return 0;
}
```

**输出：**
```
连接: localhost:8080
连接: api.example.com:8080
连接: secure.example.com:443 (SSL)
```

## 二、具体用法

### 2.1 继承构造

```cpp
#include <iostream>

class Base {
    int value;
public:
    Base() : value(0) { std::cout << "Base()" << std::endl; }
    Base(int v) : value(v) { std::cout << "Base(" << v << ")" << std::endl; }
    Base(int v, int extra) : value(v + extra) {
        std::cout << "Base(" << v << ", " << extra << ")" << std::endl;
    }
    int getValue() const { return value; }
};

class Derived : public Base {
public:
    using Base::Base; // 继承所有Base的构造函数

    void show() const {
        std::cout << "Derived value=" << getValue() << std::endl;
    }
};

int main() {
    Derived d1;
    Derived d2(42);
    Derived d3(10, 20);
    d2.show();
    return 0;
}
```

**输出：**
```
Base()
Base(42)
Base(10, 20)
Derived value=42
```

## 三、注意事项与常见陷阱

- **委托构造不能有初始化列表的成员**：所有成员初始化在被委托的构造函数中。
- **委托构造中不能使用`this`访问成员**：成员尚未初始化。
- **继承构造不继承默认参数**：需要显式定义。
- **继承构造遇到冲突时**：派生类自己的构造函数优先。
- **虚继承的类不能用继承构造**。
