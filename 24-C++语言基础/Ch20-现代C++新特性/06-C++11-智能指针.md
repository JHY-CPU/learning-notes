# C++11 智能指针

## 一、概念说明

C++11在`<memory>`中引入三种智能指针，实现自动内存管理（C++11 §20.7）。它们遵循RAII原则，离开作用域时自动释放资源。

### 1.1 三种智能指针

| 类型 | 所有权 | 额外开销 | 适用场景 |
|------|--------|---------|---------|
| `unique_ptr` | 独占 | 零（与裸指针同大小） | 默认选择 |
| `shared_ptr` | 共享 | 引用计数（控制块） | 多所有者 |
| `weak_ptr` | 观察 | 弱引用计数 | 打破循环引用 |

```cpp
#include <iostream>
#include <memory>

struct Resource {
    std::string name;
    Resource(std::string n) : name(std::move(n)) {
        std::cout << name << " 创建" << std::endl;
    }
    ~Resource() { std::cout << name << " 销毁" << std::endl; }
};

int main() {
    // unique_ptr：独占所有权
    auto u1 = std::make_unique<Resource>("独占资源");
    // auto u2 = u1; // 错误：不能拷贝
    auto u2 = std::move(u1); // 移动所有权
    std::cout << "u1=" << (u1 ? "有效" : "空") << std::endl;

    // shared_ptr：共享所有权
    auto s1 = std::make_shared<Resource>("共享资源");
    {
        auto s2 = s1; // 拷贝，引用计数+1
        std::cout << "引用计数: " << s1.use_count() << std::endl;
    } // s2销毁，引用计数-1
    std::cout << "离开后引用计数: " << s1.use_count() << std::endl;

    // weak_ptr：弱引用
    std::weak_ptr<Resource> w = s1;
    if (auto locked = w.lock()) {
        std::cout << "lock成功: " << locked->name << std::endl;
    }

    std::cout << "离开作用域..." << std::endl;
    return 0;
}
```

**输出：**
```
独占资源 创建
u1=空
共享资源 创建
引用计数: 2
离开后引用计数: 1
lock成功: 共享资源
离开作用域...
独占资源 销毁
共享资源 销毁
```

## 二、具体用法

### 2.1 unique_ptr进阶

```cpp
#include <iostream>
#include <memory>
#include <vector>

// 自定义删除器
struct FileDeleter {
    void operator()(FILE* fp) const {
        if (fp) { fclose(fp); std::cout << "文件关闭" << std::endl; }
    }
};

int main() {
    // 工厂模式
    auto create = [](int v) { return std::make_unique<int>(v); };
    auto p = create(42);

    // unique_ptr数组
    auto arr = std::make_unique<int[]>(10);
    arr[0] = 1;

    // 自定义删除器
    std::unique_ptr<FILE, FileDeleter> file(fopen("test.txt", "w"));

    // 转移所有权
    std::vector<std::unique_ptr<int>> vec;
    vec.push_back(std::move(p));

    std::cout << "vec[0]=" << *vec[0] << std::endl;
    return 0;
}
```

### 2.2 shared_ptr与循环引用

```cpp
#include <iostream>
#include <memory>

struct Node {
    std::string name;
    std::shared_ptr<Node> next;
    std::weak_ptr<Node> prev; // weak_ptr打破循环

    Node(std::string n) : name(std::move(n)) {
        std::cout << name << " 创建" << std::endl;
    }
    ~Node() { std::cout << name << " 销毁" << std::endl; }
};

int main() {
    {
        auto a = std::make_shared<Node>("A");
        auto b = std::make_shared<Node>("B");
        a->next = b;
        b->prev = a; // weak_ptr不增加引用计数
        std::cout << "a引用计数: " << a.use_count() << std::endl; // 1
        std::cout << "b引用计数: " << b.use_count() << std::endl; // 2
    }
    std::cout << "正常销毁（无泄漏）" << std::endl;
    return 0;
}
```

**输出：**
```
A 创建
B 创建
a引用计数: 1
b引用计数: 2
B 销毁
A 销毁
正常销毁（无泄漏）
```

### 2.3 参数传递策略

```cpp
#include <iostream>
#include <memory>

// 接收所有权
void takeOwnership(std::unique_ptr<int> p) {
    std::cout << "获取所有权: " << *p << std::endl;
}

// 借用（不获取所有权）
void observe(const std::shared_ptr<int>& p) {
    std::cout << "借用: " << *p << " (引用计数: " << p.use_count() << ")" << std::endl;
}

// 共享所有权
void share(std::shared_ptr<int> p) {
    std::cout << "共享: 引用计数=" << p.use_count() << std::endl;
}

int main() {
    auto p = std::make_unique<int>(42);
    takeOwnership(std::move(p));

    auto sp = std::make_shared<int>(99);
    observe(sp);   // 不增加引用计数
    share(sp);     // 增加引用计数

    return 0;
}
```

## 三、注意事项与常见陷阱

1. **优先使用`make_unique`/`make_shared`**：避免直接`new`，更安全且更高效。
2. **循环引用用`weak_ptr`打破**：A持有B的`shared_ptr`，B持有A的`weak_ptr`。
3. **不要用同一裸指针创建多个`shared_ptr`**：会双重释放，导致未定义行为。
4. **`unique_ptr`大小等于裸指针**：零额外开销，是默认推荐的智能指针。
5. **`make_shared`合并分配**：对象和控制块在一起，`weak_ptr`存在时整个块不释放。
6. **`shared_ptr`的引用计数操作是原子的**：有同步开销，不需要共享时用`unique_ptr`。
