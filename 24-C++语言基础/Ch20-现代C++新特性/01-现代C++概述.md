# 现代C++概述

## 一、概念说明

现代C++指C++11及之后的标准（C++11/14/17/20/23）。核心哲学：**零开销抽象**、**RAII**、**类型安全**、**编译期计算**。

C++演进时间线：
- **C++11**：革命性更新，移动语义、lambda、智能指针、线程库
- **C++14**：C++11的完善，泛型lambda、变量模板
- **C++17**：实用主义，结构化绑定、optional、filesystem、并行算法
- **C++20**：四大特性，Concepts、Ranges、Modules、Coroutines
- **C++23**：持续完善，std::print、expected、flat_map

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
#include <numeric>

int main() {
    // C++11: auto, range-for, lambda
    std::vector<int> nums = {1, 2, 3, 4, 5};
    auto doubled = nums; // auto类型推导

    std::transform(nums.begin(), nums.end(), doubled.begin(),
                   [](int x) { return x * 2; }); // lambda

    // C++17: 结构化绑定
    auto [min, max] = std::minmax_element(doubled.begin(), doubled.end());

    // C++20: 范围（概念性）
    // auto result = nums | std::views::transform([](int x){ return x*2; })
    //                     | std::views::filter([](int x){ return x > 4; });

    std::cout << "原始: ";
    for (int n : nums) std::cout << n << " ";
    std::cout << "\n翻倍: ";
    for (int n : doubled) std::cout << n << " ";
    std::cout << "\n最小: " << *min << ", 最大: " << *max << std::endl;

    return 0;
}
```

**输出：**
```
原始: 1 2 3 4 5
翻倍: 2 4 6 8 10
最小: 2, 最大: 10
```

## 二、具体用法

### 2.1 各版本关键特性

| 版本 | 关键特性 |
|------|---------|
| C++11 | `auto`, `nullptr`, `move`, `unique_ptr`, `shared_ptr`, `lambda`, `thread`, `enum class`, `constexpr` |
| C++14 | 泛型lambda, 变量模板, `make_unique`, 返回类型推导 |
| C++17 | 结构化绑定, `if constexpr`, `optional`, `variant`, `string_view`, `filesystem`, `parallel algorithms` |
| C++20 | `Concepts`, `Ranges`, `Modules`, `Coroutines`, `format`, `span`, `consteval`, `constinit` |
| C++23 | `std::print`, `expected`, `flat_map`, 多维下标, `std::generator` |

### 2.2 编译器支持

```bash
# GCC
g++ -std=c++11  # C++11
g++ -std=c++17  # C++17
g++ -std=c++20  # C++20
g++ -std=c++23  # C++23

# Clang
clang++ -std=c++17

# MSVC
# /std:c++17, /std:c++20, /std:c++latest
```

## 三、注意事项与常见陷阱

- **默认编译器可能不启用最新标准**：需显式指定`-std=c++XX`。
- **特性支持因编译器而异**：查[cppreference](https://en.cppreference.com/w/cpp/compiler_support)。
- **向后兼容**：C++20代码用C++17标准编译会失败，但反之通常可以。
- **新特性不等于更好**：有时简单代码比高级特性更易维护。
- **学习建议**：先掌握C++11/14核心特性，再逐步学习17/20/23。
