# 现代C++概述

## 一、概念说明

现代C++指C++11及之后的标准（C++11/14/17/20/23/26）。核心哲学：**零开销抽象**（Zero-overhead Abstraction）、**RAII**、**类型安全**（Type Safety）、**编译期计算**（Compile-time Computation）。这些原则确保高级抽象不增加运行时开销。

### 1.1 标准演进时间线

| 版本 | 年份 | 定位 | 核心特性 |
|------|------|------|---------|
| C++11 | 2011 | 革命性更新 | 移动语义、lambda、智能指针、线程库、`auto`、`nullptr`、`enum class` |
| C++14 | 2014 | 完善C++11 | 泛型lambda、变量模板、`make_unique`、`constexpr`增强 |
| C++17 | 2017 | 实用主义 | 结构化绑定、`if constexpr`、`optional`/`variant`/`any`、`filesystem`、并行算法 |
| C++20 | 2020 | 四大特性 | Concepts、Ranges、Modules、Coroutines、`format`、`span`、三路比较 |
| C++23 | 2023 | 持续完善 | `std::print`、`expected`、`flat_map`、`std::generator` |
| C++26 | 2026 | 未来展望 | 反射（Reflection）、Contracts、模式匹配、SIMD |

### 1.2 设计哲学

```
现代C++三大设计原则：
1. 零开销抽象：高级特性编译后与手写C代码效率相当
2. 资源获取即初始化（RAII）：资源管理自动化
3. 编译期计算：尽可能多的工作在编译期完成
```

## 二、具体用法

### 2.1 各版本关键特性速览

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
#include <numeric>
#include <string>

int main() {
    // C++11: auto, range-for, lambda, 统一初始化
    std::vector<int> nums = {1, 2, 3, 4, 5};
    auto doubled = nums;

    std::transform(nums.begin(), nums.end(), doubled.begin(),
                   [](int x) { return x * 2; });

    // C++14: 泛型lambda
    auto generic_print = [](const auto& val) { std::cout << val << " "; };

    // C++17: 结构化绑定
    auto [min, max] = std::minmax_element(doubled.begin(), doubled.end());
    std::cout << "最小: " << *min << ", 最大: " << *max << std::endl;

    // C++20: 范围for_each
    // std::ranges::for_each(doubled, [](int x){ std::cout << x << " "; });

    return 0;
}
```

**输出：**
```
最小: 2, 最大: 10
```

### 2.2 编译器支持与编译选项

```bash
# GCC
g++ -std=c++11 main.cpp  # C++11
g++ -std=c++14 main.cpp  # C++14
g++ -std=c++17 main.cpp  # C++17
g++ -std=c++20 main.cpp  # C++20
g++ -std=c++23 main.cpp  # C++23（部分支持）

# Clang
clang++ -std=c++17 main.cpp
clang++ -std=c++20 main.cpp

# MSVC
# /std:c++17, /std:c++20, /std:c++latest
```

### 2.3 特性检测宏

```cpp
#include <iostream>

int main() {
    // 标准版本检测
    #if __cplusplus >= 202002L
        std::cout << "C++20或更高" << std::endl;
    #elif __cplusplus >= 201703L
        std::cout << "C++17" << std::endl;
    #elif __cplusplus >= 201402L
        std::cout << "C++14" << std::endl;
    #elif __cplusplus >= 201103L
        std::cout << "C++11" << std::endl;
    #endif

    // 特性检测（C++20）
    #ifdef __cpp_concepts
        std::cout << "支持Concepts" << std::endl;
    #endif
    #ifdef __cpp_lib_ranges
        std::cout << "支持Ranges库" << std::endl;
    #endif

    return 0;
}
```

## 三、注意事项与常见陷阱

1. **默认编译器可能不启用最新标准**：需显式指定`-std=c++XX`。
2. **特性支持因编译器而异**：查[cppreference编译器支持](https://en.cppreference.com/w/cpp/compiler_support)。
3. **向后兼容**：C++20代码用C++17标准编译会失败，但反之通常可以。
4. **新特性不等于更好**：有时简单代码比高级特性更易维护。
5. **学习建议**：先掌握C++11/14核心特性，再逐步学习17/20/23。
6. **`__cplusplus`宏值**：C++11=201103L，C++14=201402L，C++17=201703L，C++20=202002L。
