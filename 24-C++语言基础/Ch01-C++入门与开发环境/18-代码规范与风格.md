# 代码规范与风格

## 一、概念说明

代码规范定义了代码的**命名、格式、结构**等约定。统一的代码风格能提高可读性、减少bug、方便团队协作。虽然C++没有官方规范，但业界有多种广泛采用的风格指南。

## 二、具体用法

### 2.1 常见命名风格

```cpp
#include <iostream>
#include <string>
#include <vector>

// Google风格命名示例
// 变量和函数：snake_case（小写字母+下划线）
int student_count;
double calculate_average(const std::vector<int>& scores);

// 类名：PascalCase（每个单词首字母大写）
class StudentManager {
private:
    // 成员变量：尾部下划线，避免与参数冲突
    std::string name_;
    int age_;
    // 常量：k开头+PascalCase
    static const int kMaxStudents = 100;

public:
    // 构造函数
    StudentManager(const std::string& name, int age)
        : name_(name), age_(age) {}

    // Getter方法
    const std::string& GetName() const { return name_; }
    int GetAge() const { return age_; }
};

// 命名空间：snake_case
namespace data_processing {
    // 枚举值：k开头+PascalCase
    enum class LogLevel {
        kInfo,
        kWarning,
        kError
    };
}

// 宏定义：全大写+下划线
#define MAX_BUFFER_SIZE 1024
#define ENABLE_DEBUG_MODE
```

### 2.2 代码格式

```cpp
// 推荐的缩进和空格风格
class Example {
public:
    // 缩进：2个空格（Google风格）或4个空格（多数项目）
    void Function(int param1, int param2) {
        // 运算符两侧加空格
        int result = param1 + param2;

        // 控制语句关键字后加空格
        if (result > 0) {
            std::cout << "正数" << std::endl;
        } else if (result < 0) {
            std::cout << "负数" << std::endl;
        } else {
            std::cout << "零" << std::endl;
        }

        // 花括号位置：同行（Google/Java风格）或另起一行（Allman风格）
        for (int i = 0; i < 10; i++) {
            Process(i);
        }
    }

private:
    // 成员变量与函数之间空一行
    int member_var_;

    void Process(int value) {
        // 函数体内空行分隔逻辑块
        PreProcess(value);

        // 核心逻辑
        DoWork(value);

        PostProcess(value);
    }

    void PreProcess(int) {}
    void DoWork(int) {}
    void PostProcess(int) {}
};
```

```cpp
// 文件头注释
/**

```

### 2.4 clang-format自动格式化

```yaml
# .clang-format 配置文件
BasedOnStyle: Google
IndentWidth: 2
ColumnLimit: 100
AllowShortFunctionsOnASingleLine: Inline
BreakBeforeBraces: Attach
SortIncludes: true
```

```bash
# 使用clang-format格式化代码
clang-format -i main.cpp              # 原地格式化
clang-format -style=Google main.cpp   # 使用Google风格
clang-format --dry-run main.cpp       # 仅检查不修改
```

## 三、注意事项与常见陷阱

1. **团队优先**：项目应该统一使用一种风格，个人偏好服从团队规范
2. **一致性最重要**：比选择哪种风格更重要的是保持一致
3. **使用自动化工具**：clang-format可以自动格式化代码，避免人工争论格式
4. **不要过度注释**：好的代码应该自解释，注释解释"为什么"而非"做什么"
5. **头文件包含顺序**：不同规范有不同的顺序要求，遵循所选规范
