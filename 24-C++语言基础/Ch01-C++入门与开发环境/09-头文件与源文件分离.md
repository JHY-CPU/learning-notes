# 头文件与源文件分离

## 一、概念说明

C++项目通常将代码分为**头文件（.h/.hpp）**和**源文件（.cpp）**。头文件包含声明（接口），源文件包含定义（实现）。这种分离实现了**接口与实现分离**，提高了代码的可维护性和编译效率。

## 二、具体用法

### 2.1 基本结构

```cpp
// ===== math_utils.h ===== （头文件：声明）
#ifndef MATH_UTILS_H
#define MATH_UTILS_H

// 函数声明（只有签名，没有函数体）
int add(int a, int b);
int multiply(int a, int b);
double circleArea(double radius);

// 类声明
class Rectangle {
private:
    double width;
    double height;
public:
    Rectangle(double w, double h);
    double area() const;
    double perimeter() const;
};

#endif // MATH_UTILS_H
```

```cpp
// ===== math_utils.cpp ===== （源文件：定义）
#include "math_utils.h"
#include <cmath>

// 函数定义（实现声明的函数）
int add(int a, int b) {
    return a + b;
}

int multiply(int a, int b) {
    return a * b;
}

double circleArea(double radius) {
    return M_PI * radius * radius;
}

// 类成员函数定义
Rectangle::Rectangle(double w, double h)
    : width(w), height(h) {}

double Rectangle::area() const {
    return width * height;
}

double Rectangle::perimeter() const {
    return 2 * (width + height);
}
```

```cpp
// ===== main.cpp ===== （主程序文件）
#include <iostream>
#include "math_utils.h"  // 包含自己的头文件

int main() {
    std::cout << "3 + 4 = " << add(3, 4) << std::endl;
    std::cout << "圆面积: " << circleArea(5.0) << std::endl;

    Rectangle rect(3.0, 4.0);
    std::cout << "矩形面积: " << rect.area() << std::endl;
    std::cout << "矩形周长: " << rect.perimeter() << std::endl;
    return 0;
}
```

输出：
```
3 + 4 = 7
圆面积: 78.5398
矩形面积: 12
矩形周长: 14
```

### 2.2 编译多文件项目

```bash
# 方式一：一起编译
g++ -std=c++17 main.cpp math_utils.cpp -o program

# 方式二：分别编译再链接
g++ -std=c++17 -c main.cpp -o main.o
g++ -std=c++17 -c math_utils.cpp -o math_utils.o
g++ main.o math_utils.o -o program
```

### 2.3 声明与定义的区别

```cpp
// 声明：告诉编译器"这个东西存在"，不分配内存
extern int globalCount;          // 变量声明
void func();                     // 函数声明
class MyClass;                   // 类的前向声明

// 定义：告诉编译器"这个东西是什么"，分配内存
int globalCount = 0;             // 变量定义
void func() { /* ... */ }       // 函数定义
class MyClass { /* ... */ };     // 类定义
```

## 三、注意事项与常见陷阱

1. **头文件中不要放定义**（除内联函数、模板、const/constexpr外），否则多重包含会导致重复定义错误
2. **包含自己的头文件作为测试**：每个`.cpp`文件应该先`#include`对应的`.h`文件，确保头文件可以独立编译
3. **前向声明减少依赖**：如果只需要指针或引用，使用前向声明而非`#include`
4. **包含顺序**：先包含自己的头文件，再包含第三方库，最后包含标准库
5. **避免循环包含**：A.h包含B.h，B.h包含A.h，使用前向声明解决
