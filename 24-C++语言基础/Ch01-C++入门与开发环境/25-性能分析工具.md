# 性能分析工具

## 一、概念说明

性能分析（Profiling）是测量程序运行时性能特征的过程，包括**CPU使用率**、**内存分配**、**函数调用次数**和**执行时间**等。找到性能瓶颈是优化程序的第一步。

## 二、具体用法

### 2.1 gprof —— GCC内置分析器

```cpp
// profile_demo.cpp —— 性能分析示例
#include <iostream>
#include <cmath>
#include <vector>

double computeExpensive(double x) {
    double result = 0;
    for (int i = 0; i < 1000000; i++) {
        result += std::sin(x * i) * std::cos(x * i);
    }
    return result;
}

double computeCheap(double x) {
    return x * x + 2 * x + 1;
}

int main() {
    std::vector<double> results;
    for (int i = 0; i < 10; i++) {
        results.push_back(computeExpensive(i * 0.1));
        results.push_back(computeCheap(i * 0.1));
    }
    std::cout << "计算完成，共" << results.size() << "个结果" << std::endl;
    return 0;
}
```

```bash
# 编译时加入-pg选项
g++ -pg -O2 -std=c++17 profile_demo.cpp -o profile_demo

# 运行程序（自动生成gmon.out）
./profile_demo

# 分析结果
gprof profile_demo gmon.out
```

输出（简化）：
```
  %   cumulative   self              self     total
 time   seconds   seconds    calls  ms/call  ms/call  name
 95.23     2.86     2.86       10   286.00   286.00  computeExpensive(double)
  2.38     2.93     0.07       10     7.00     7.00  computeCheap(double)
```

### 2.2 Valgrind —— 内存分析（Linux）

```cpp
// memory_demo.cpp
#include <iostream>
#include <vector>

void memoryIssue() {
    int* arr = new int[1000];    // 分配内存
    for (int i = 0; i < 1000; i++) {
        arr[i] = i * i;
    }
    // 忘记 delete[] arr; 导致内存泄漏

    int* p = new int(42);
    delete p;
    // *p = 10;  // use-after-free（如果取消注释）
}

int main() {
    memoryIssue();
    std::vector<int> v(10000);
    for (size_t i = 0; i < v.size(); i++) {
        v[i] = i;
    }
    return 0;
}
```

```bash
# 编译（必须加-g）
g++ -g -O0 memory_demo.cpp -o memory_demo

# 内存泄漏检测
valgrind --leak-check=full ./memory_demo

# 缓存分析（找到缓存未命中热点）
valgrind --tool=cachegrind ./memory_demo

# 堆分析
valgrind --tool=massif ./memory_demo
```

### 2.3 perf —— Linux系统级分析

```bash
# 记录性能数据
perf record -g ./profile_demo

# 查看报告
perf report

# 统计特定事件
perf stat ./profile_demo

# 查看缓存未命中
perf stat -e cache-misses,cache-references ./profile_demo
```

### 2.4 简单的代码级计时

```cpp
#include <iostream>
#include <chrono>
#include <vector>
#include <algorithm>

class Timer {
    std::chrono::high_resolution_clock::time_point start;
    std::string name;
public:
    Timer(const std::string& n) : name(n) {
        start = std::chrono::high_resolution_clock::now();
    }
    ~Timer() {
        auto end = std::chrono::high_resolution_clock::now();
        auto duration = std::chrono::duration_cast<std::chrono::microseconds>(end - start);
        std::cout << name << " 耗时: " << duration.count() << " 微秒" << std::endl;
    }
};

int main() {
    std::vector<int> data(1000000);
    std::iota(data.begin(), data.end(), 0);

    {
        Timer t("排序");
        std::sort(data.begin(), data.end(), std::greater<int>());
    }

    {
        Timer t("查找中位数");
        auto mid = data.begin() + data.size() / 2;
        std::nth_element(data.begin(), mid, data.end());
    }

    return 0;
}
```

输出（示例）：
```
排序 耗时: 98765 微秒
查找中位数 耗时: 12345 微秒
```

### 2.5 工具选择指南

| 目标 | 工具 | 平台 |
|------|------|------|
| CPU热点分析 | gprof, perf, VTune | 全平台 |
| 内存泄漏检测 | Valgrind, ASan | Linux为主 |
| 缓存性能 | cachegrind, perf | Linux |
| 堆内存分析 | massif, heaptrack | Linux |
| 函数调用图 | gprof, callgrind | 全平台 |
| 简单计时 | chrono库 | 全平台 |

## 三、注意事项与常见陷阱

1. **分析开销**：分析工具会显著降低程序速度，结果是相对的不是绝对的
2. **Debug vs Release**：内存分析用Debug编译，性能分析用Release编译（加-g保留符号）
3. **采样 vs 检测**：gprof/perf是采样方式（开销小但可能遗漏），Valgrind是检测方式（准确但慢）
4. **不要猜测**：性能优化要基于数据，凭直觉判断瓶颈位置经常是错误的
5. **微优化陷阱**：先优化算法（O(n²)→O(n log n)），再考虑代码级微优化
