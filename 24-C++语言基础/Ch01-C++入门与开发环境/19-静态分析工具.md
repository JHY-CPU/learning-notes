# 静态分析工具

## 一、概念说明

静态分析工具在**不运行程序**的情况下分析源代码，发现潜在的bug、内存泄漏、未定义行为和代码风格问题。它们是编译器警告的补充，能发现更深层次的问题。

## 二、具体用法

### 2.1 clang-tidy

clang-tidy是基于Clang的C++ linter工具，能检查代码风格、bug和现代化建议。

```cpp
// 有问题的代码 example.cpp
#include <stdio.h>      // C头文件，应用<cstdio>
#include <cstring>

void badFunction() {
    char buf[100];
    int x = 0;
    int y;              // 未初始化
    printf("hello\n");  // 应使用cout
    strcpy(buf, "test");
    if (x = 1) {        // 应使用==而非=
        y = 2;
    }
}
```

```bash
# 运行clang-tidy
clang-tidy example.cpp -- -std=c++17

# 使用配置文件
clang-tidy example.cpp --config-file=.clang-tidy

# 自动修复问题
clang-tidy example.cpp --fix -- -std=c++17
```

`.clang-tidy`配置文件：
```yaml
Checks: >
  -*,
  bugprone-*,
  modernize-*,
  performance-*,
  readability-*,
  -modernize-use-trailing-return-type

WarningsAsErrors: ''
HeaderFilterRegex: '.*'
FormatStyle: google
```

输出示例：
```
example.cpp:3:1: warning: inclusion of deprecated C header 'stdio.h'
  3 | #include <stdio.h>
    | ^~~~~~~~~~~~~~~~~~
example.cpp:8:5: warning: variable 'y' is not initialized
  8 |     int y;
    |     ^
example.cpp:9:5: warning: use of C-style printf
  9 |     printf("hello\n");
    |     ^
example.cpp:11:9: warning: using the result of an assignment as a condition
 11 |     if (x = 1) {
    |         ^
```

### 2.2 cppcheck

cppcheck专注于检测**未定义行为**和**内存错误**，误报率低。

```cpp
// 有内存问题的代码 memory_bug.cpp
#include <iostream>

void memoryLeak() {
    int* arr = new int[100];  // 内存泄漏：没有delete
    arr[0] = 42;
}

void bufferOverflow() {
    int arr[5];
    for (int i = 0; i <= 5; i++) {  // 越界：i <= 5应为 i < 5
        arr[i] = i;
    }
}

void nullPointer() {
    int* p = nullptr;
    *p = 10;  // 空指针解引用
}

int main() {
    memoryLeak();
    bufferOverflow();
    nullPointer();
    return 0;
}
```

```bash
# 基本用法
cppcheck memory_bug.cpp

# 启用所有检查
cppcheck --enable=all memory_bug.cpp

# 指定C++标准
cppcheck --std=c++17 memory_bug.cpp

# 输出到文件
cppcheck --enable=all --output-file=report.txt .

# XML格式输出（便于CI集成）
cppcheck --xml --enable=all memory_bug.cpp 2> report.xml
```

输出示例：
```
memory_bug.cpp:5:5: error: Memory leak: arr [memleak]
    int* arr = new int[100];
    ^
memory_bug.cpp:10:22: error: Array 'arr[5]' accessed at index 5, which is out of bounds. [arrayIndexOutOfBounds]
    for (int i = 0; i <= 5; i++) {
                     ^
memory_bug.cpp:17:5: error: Null pointer dereference: p [nullPointer]
    *p = 10;
    ^
```

### 2.3 编译器的静态检查

```bash
# GCC/Clang的警告选项（也是一种静态分析）
g++ -Wall -Wextra -Wpedantic -Wshadow -Wconversion main.cpp

# Clang特有的警告
clang++ -Weverything main.cpp  # 开启所有警告（过于严格）

# MSVC
cl /W4 /WX main.cpp  # 等级4警告，警告即错误
```

### 2.4 在VS Code中集成

```json
// .vscode/settings.json
{
    "C_Cpp.codeAnalysis.clangtidy.enabled": true,
    "C_Cpp.codeAnalysis.clangtidy.checks": [
        "bugprone-*",
        "modernize-*",
        "performance-*"
    ],
    "C_Cpp.codeAnalysis.runAutomatically": true
}
```

## 三、注意事项与常见陷阱

1. **不要忽视警告**：警告通常是真实问题的信号，开启`-Werror`将警告视为错误
2. **误报处理**：静态分析工具有误报，但不要轻易禁用检查，先理解为什么报告
3. **定期运行**：将静态分析集成到CI/CD流程中，每次提交自动检查
4. **组合使用**：不同工具有不同优势，建议同时使用编译器警告、clang-tidy和cppcheck
5. **分析速度**：全量静态分析可能很慢，可以在CI中使用，本地开发时仅检查当前文件
