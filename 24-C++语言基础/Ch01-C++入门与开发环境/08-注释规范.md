# 注释规范

## 一、概念说明

注释是源代码中被编译器忽略的文本，用于解释代码意图、记录作者信息和生成文档。良好的注释习惯是专业程序员的基本素养。C++支持两种基本注释风格和一种文档注释风格。

## 二、具体用法

### 2.1 单行注释

```cpp
#include <iostream>
using namespace std;

int main() {
    int radius = 5;           // 圆的半径
    double pi = 3.14159;      // 圆周率近似值

    // 计算圆的面积
    double area = pi * radius * radius;

    cout << "面积: " << area << endl;  // 输出结果
    return 0;  // 程序正常退出
}
```

输出：
```
面积: 78.5397
```

### 2.2 多行注释

```cpp
#include <iostream>
using namespace std;

/*
 * 函数名: calculateAverage
 * 功能:   计算整数数组的平均值
 * 参数:   arr - 整数数组
 *         size - 数组大小
 * 返回值: double类型的平均值
 * 作者:   张三
 * 日期:   2026-05-06
 */
double calculateAverage(const int arr[], int size) {
    if (size <= 0) return 0.0;

    int sum = 0;
    for (int i = 0; i < size; i++) {
        sum += arr[i];
    }
    return static_cast<double>(sum) / size;
}

int main() {
    int scores[] = {85, 92, 78, 95, 88};
    int n = sizeof(scores) / sizeof(scores[0]);

    /* 计算并输出平均分 */
    cout << "平均分: " << calculateAverage(scores, n) << endl;
    return 0;
}
```

输出：
```
平均分: 87.6
```

### 2.3 Doxygen文档注释风格

Doxygen是一个文档生成工具，可以从源代码注释自动生成HTML/PDF文档。

```cpp
/**
 * @brief 计算两个数的最大公约数
 *
 * 使用欧几里得算法（辗转相除法）计算两个非负整数的最大公约数。
 *
 * @param a 第一个非负整数
 * @param b 第二个非负整数
 * @return int 两个数的最大公约数
 *
 * @note 如果任一参数为负数，返回-1
 * @warning a和b不能同时为0
 *
 * @code
 * int result = gcd(12, 8);  // result = 4
 * @endcode
 *
 * @see lcm() 用于计算最小公倍数
 */
int gcd(int a, int b) {
    if (a < 0 || b < 0) return -1;
    while (b != 0) {
        int temp = b;
        b = a % b;
        a = temp;
    }
    return a;
}

/// @brief 简短的单行Doxygen注释
/// 这是详细的描述部分
int square(int x) {
    return x * x;
}
```

### 2.4 注释的最佳实践

```cpp
#include <iostream>
#include <vector>
using namespace std;

// ==================== 学生管理系统模块 ====================
// 版本: 1.0
// 描述: 提供学生信息的基本管理功能

class Student {
public:
    string name;     // 学生姓名
    int id;          // 学号（唯一标识）
    double gpa;      // 绩点（0.0 ~ 4.0）

    // 构造函数：初始化学生信息
    Student(const string& n, int i, double g)
        : name(n), id(i), gpa(g) {}
};

/// @brief 查找绩点最高的学生
/// @param students 学生列表
/// @return 绩点最高的学生索引，列表为空返回-1
int findTopStudent(const vector<Student>& students) {
    if (students.empty()) return -1;

    int topIdx = 0;
    for (size_t i = 1; i < students.size(); i++) {
        if (students[i].gpa > students[topIdx].gpa) {
            topIdx = i;
        }
    }
    return topIdx;
}

int main() {
    vector<Student> students = {
        Student("张三", 1001, 3.8),
        Student("李四", 1002, 3.95),
        Student("王五", 1003, 3.6)
    };

    int best = findTopStudent(students);
    cout << "最优学生: " << students[best].name
         << " (GPA: " << students[best].gpa << ")" << endl;
    return 0;
}
```

输出：
```
最优学生: 李四 (GPA: 3.95)
```

## 三、注意事项与常见陷阱

1. **注释要维护**：过时的注释比没有注释更糟糕，修改代码时同步更新注释
2. **不要注释显而易见的代码**：`int x = 5; // 把5赋给x`这种注释毫无意义
3. **解释"为什么"而非"做什么"**：好的注释解释设计意图和决策原因
4. **TODO和FIXME标记**：使用`// TODO: 需要实现`和`// FIXME: 有bug`标记待办事项
5. **避免注释掉大段代码**：使用版本控制系统（Git）管理历史代码，不要用注释"保存"旧代码
