# 编译选项与优化

## 一、概念说明

编译选项控制编译器的行为，包括标准版本、警告级别、优化程度和调试信息等。合理使用编译选项能提高代码质量、发现潜在bug并优化程序性能。

## 二、具体用法

### 2.1 C++标准版本

```bash
# 指定C++标准版本
g++ -std=c++11 main.cpp -o main   # C++11
g++ -std=c++14 main.cpp -o main   # C++14
g++ -std=c++17 main.cpp -o main   # C++17（推荐）
g++ -std=c++20 main.cpp -o main   # C++20
g++ -std=c++23 main.cpp -o main   # C++23

# 使用最新的实验性支持
g++ -std=c++2b main.cpp -o main
```

### 2.2 警告选项

```cpp
// 这段代码存在潜在问题
#include <iostream>
using namespace std;

int main() {
    int x = 10;
    int y;              // 未初始化变量
    bool flag = x;      // 隐式转换
    int z = x + y;      // 使用未初始化的值
    cout << z << endl;
    return 0;
}
```

```bash
# 开启基本警告
g++ -Wall main.cpp -o main

# 开启额外警告
g++ -Wall -Wextra main.cpp -o main

# 将警告视为错误（零警告编译）
g++ -Wall -Wextra -Werror main.cpp -o main

# 严格模式（GCC）
g++ -Wall -Wextra -Wpedantic main.cpp -o main

# 常用组合（推荐项目使用）
g++ -std=c++17 -Wall -Wextra -Wpedantic -Werror main.cpp -o main
```

输出示例：
```
main.cpp: In function 'int main()':
main.cpp:7:9: warning: 'y' is used uninitialized [-Wuninitialized]
    7 |     int z = x + y;
      |         ^~~~~~~~~
```

### 2.3 优化级别

```cpp
// test.cpp —— 测试优化效果
#include <iostream>
using namespace std;

// 编译器可能将此函数内联
inline int square(int x) {
    return x * x;
}

int main() {
    int sum = 0;
    // 编译器可能将此循环优化为常量
    for (int i = 0; i < 1000; i++) {
        sum += square(i);
    }
    cout << "结果: " << sum << endl;
    return 0;
}
```

```bash
# 无优化 —— 编译最快，生成代码最大，适合调试
g++ -O0 test.cpp -o test

# 基本优化
g++ -O1 test.cpp -o test

# 推荐优化级别 —— 大多数场景的最佳选择
g++ -O2 test.cpp -o test

# 激进优化 —— 可能增加编译时间和代码大小
g++ -O3 test.cpp -o test

# 优化体积 —— 嵌入式系统常用
g++ -Os test.cpp -o test

# 仅对体积小的代码优化
g++ -Og test.cpp -o test  # 适合调试时使用
```

### 2.4 调试选项

```bash
# 生成调试信息（GDB/LLDB可用）
g++ -g main.cpp -o main

# 调试+基本优化（调试体验可能受影响）
g++ -g -Og main.cpp -o main

# 生成更详细的调试信息
g++ -g3 main.cpp -o main
```

### 2.5 其他常用选项

```bash
# 指定头文件搜索路径
g++ -I./include -I/usr/local/include main.cpp -o main

# 指定库文件搜索路径
g++ -L./lib main.cpp -lmylib -o main

# 定义预处理宏
g++ -DDEBUG -DVERSION=2 main.cpp -o main

# 生成位置无关代码（共享库需要）
g++ -fPIC -shared mylib.cpp -o libmylib.so

# 启用地址消毒器（检测内存错误）
g++ -fsanitize=address -g main.cpp -o main

# 启用未定义行为消毒器
g++ -fsanitize=undefined -g main.cpp -o main
```

## 三、注意事项与常见陷阱

1. **调试与优化冲突**：`-O2`及以上优化会使调试信息不准确，调试时用`-Og`或`-O0`
2. **-Wall不是所有警告**：`-Wall`名称有误导性，需要配合`-Wextra`才能覆盖更多警告
3. **不同编译器选项不同**：GCC和Clang基本兼容，MSVC使用不同的选项格式（`/W4`、`/O2`等）
4. **优化可能引入bug**：如果代码存在未定义行为，开启优化后可能表现不同
5. **构建脚本中统一管理**：将编译选项写在CMakeLists.txt或Makefile中，确保团队一致
