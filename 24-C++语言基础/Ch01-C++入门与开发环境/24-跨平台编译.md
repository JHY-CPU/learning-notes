# 跨平台编译

## 一、概念说明

C++程序需要在不同操作系统（Windows、Linux、macOS）上编译和运行。跨平台编译通过**预处理宏**、**条件编译**和**跨平台库**来处理平台差异。

## 二、具体用法

### 2.1 预定义平台宏

```cpp
#include <iostream>

// 检测操作系统
#if defined(_WIN32) || defined(_WIN64)
    #define PLATFORM_WINDOWS
    #include <windows.h>
#elif defined(__linux__)
    #define PLATFORM_LINUX
    #include <unistd.h>
#elif defined(__APPLE__)
    #define PLATFORM_MACOS
    #include <TargetConditionals.h>
#else
    #define PLATFORM_UNKNOWN
#endif

// 检测编译器
#if defined(__GNUC__)
    #define COMPILER_GCC
#elif defined(_MSC_VER)
    #define COMPILER_MSVC
#elif defined(__clang__)
    #define COMPILER_CLANG
#endif

void showPlatform() {
    #ifdef PLATFORM_WINDOWS
        std::cout << "平台: Windows" << std::endl;
    #elif defined(PLATFORM_LINUX)
        std::cout << "平台: Linux" << std::endl;
    #elif defined(PLATFORM_MACOS)
        std::cout << "平台: macOS" << std::endl;
    #else
        std::cout << "平台: 未知" << std::endl;
    #endif

    #ifdef COMPILER_GCC
        std::cout << "编译器: GCC " << __GNUC__ << "." << __GNUC_MINOR__ << std::endl;
    #elif defined(COMPILER_MSVC)
        std::cout << "编译器: MSVC " << _MSC_VER << std::endl;
    #elif defined(COMPILER_CLANG)
        std::cout << "编译器: Clang" << std::endl;
    #endif
}

int main() {
    showPlatform();
    return 0;
}
```

输出（Linux下）：
```
平台: Linux
编译器: GCC 13.2
```

### 2.2 跨平台路径处理

```cpp
#include <iostream>
#include <string>

// 跨平台路径分隔符
#ifdef _WIN32
    constexpr char PATH_SEP = '\\';
    constexpr const char* PATH_SEP_STR = "\\";
#else
    constexpr char PATH_SEP = '/';
    constexpr const char* PATH_SEP_STR = "/";
#endif

std::string buildPath(const std::string& dir, const std::string& file) {
    return dir + PATH_SEP_STR + file;
}

int main() {
    std::string path = buildPath("data", "config.txt");
    std::cout << "路径: " << path << std::endl;
    return 0;
}
```

输出（Linux）：
```
路径: data/config.txt
```

### 2.3 跨平台休眠函数

```cpp
#include <iostream>
#include <chrono>
#include <thread>

// 跨平台毫秒级休眠
void sleepMs(int milliseconds) {
    std::this_thread::sleep_for(std::chrono::milliseconds(milliseconds));
}

// 也可以使用预处理（C++11之前的方式）
/*
#ifdef _WIN32
    #include <windows.h>
    void sleepMs(int ms) { Sleep(ms); }
#else
    #include <unistd.h>
    void sleepMs(int ms) { usleep(ms * 1000); }
#endif
*/

int main() {
    std::cout << "等待1秒..." << std::endl;
    sleepMs(1000);
    std::cout << "完成!" << std::endl;
    return 0;
}
```

输出：
```
等待1秒...
完成!
```

### 2.4 CMake跨平台配置

```cmake
cmake_minimum_required(VERSION 3.20)
project(CrossPlatform)

set(CMAKE_CXX_STANDARD 17)

# 平台检测
if(WIN32)
    message(STATUS "Windows平台")
    add_definitions(-DPLATFORM_WINDOWS)
    set(PLATFORM_LIBS ws2_32)
elseif(UNIX AND NOT APPLE)
    message(STATUS "Linux平台")
    add_definitions(-DPLATFORM_LINUX)
    set(PLATFORM_LIBS pthread dl)
elseif(APPLE)
    message(STATUS "macOS平台")
    add_definitions(-DPLATFORM_MACOS)
endif()

# 编译器选项
if(CMAKE_CXX_COMPILER_ID MATCHES "GNU|Clang")
    add_compile_options(-Wall -Wextra)
elseif(MSVC)
    add_compile_options(/W4)
endif()

add_executable(app main.cpp)
target_link_libraries(app ${PLATFORM_LIBS})
```

### 2.5 跨平台库推荐

```cpp
// 使用跨平台库避免手动处理平台差异
#include <filesystem>   // C++17 文件系统（标准库）
#include <thread>       // C++11 多线程（标准库）
#include <fstream>      // 文件操作（标准库）

// 第三方跨平台库：
// - Boost：功能全面的C++库
// - Qt：GUI和应用框架
// - SDL2：多媒体和游戏
// - GLFW：窗口和OpenGL上下文
// - fmt：格式化库
// - spdlog：日志库
```

## 三、注意事项与常见陷阱

1. **不要硬编码路径**：使用`std::filesystem::path`处理路径，自动适配平台分隔符
2. **字节序问题**：不同平台的字节序可能不同（大端/小端），网络传输时要注意
3. **类型大小差异**：`long`在Windows是4字节，在Linux 64位是8字节，使用`<cstdint>`中的固定大小类型
4. **换行符差异**：Windows用`\r\n`，Unix用`\n`，用`.gitattributes`和`std::endl`处理
5. **避免平台专属API**：优先使用C++标准库功能，减少对操作系统API的直接调用
