# 对象池

## 一、概念说明

**对象池**（Object Pool）预先分配一块连续内存，通过`placement new`在已分配的内存上构造对象，避免频繁的`new/delete`带来的性能开销和内存碎片。适用于需要频繁创建销毁相同大小对象的场景。

## 二、具体用法

### 2.1 简单对象池实现

```cpp
#include <iostream>
#include <vector>
#include <new>

class Connection {
    int id;
public:
    Connection(int id) : id(id) { std::cout << "连接" << id << "创建" << std::endl; }
    ~Connection() { std::cout << "连接" << id << "销毁" << std::endl; }
    int getId() const { return id; }
};

template<typename T>
class ObjectPool {
    std::vector<T*> pool;
public:
    ObjectPool(size_t count) {
        for (size_t i = 0; i < count; ++i) {
            void* mem = ::operator new(sizeof(T));
            pool.push_back(static_cast<T*>(mem));
        }
        std::cout << "池已预分配 " << count << " 个槽位" << std::endl;
    }

    template<typename... Args>
    T* acquire(Args&&... args) {
        if (pool.empty()) throw std::bad_alloc();
        T* obj = pool.back();
        pool.pop_back();
        return new (obj) T(std::forward<Args>(args)...);
    }

    void release(T* obj) {
        obj->~T();
        pool.push_back(obj);
    }

    ~ObjectPool() {
        for (T* ptr : pool) ::operator delete(ptr);
    }
};

int main() {
    ObjectPool<Connection> pool(3);
    Connection* c1 = pool.acquire(1);
    Connection* c2 = pool.acquire(2);
    std::cout << "释放c1" << std::endl;
    pool.release(c1);
    Connection* c3 = pool.acquire(3);
    pool.release(c2);
    pool.release(c3);
    return 0;
}
```

**输出：**
```
池已预分配 3 个槽位
连接1创建
连接2创建
释放c1
连接1销毁
连接3创建
连接2销毁
连接3销毁
```

## 三、注意事项与常见陷阱

- 对象池释放的对象必须调用析构函数
- 对象池适合固定大小对象的高频创建销毁
- 预分配的内存必须保证对齐要求
- 注意对象池大小，过小会导致分配失败
- 可结合空闲链表进一步优化性能
