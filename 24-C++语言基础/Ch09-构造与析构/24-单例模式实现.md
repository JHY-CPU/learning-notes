# 单例模式实现

## 一、概念说明

**单例模式**确保一个类只有一个实例，并提供全局访问点。在C++11起，使用**函数内局部静态变量**是实现线程安全单例的最简洁方式（Meyers' Singleton）。

## 二、具体用法

### 2.1 Meyers' Singleton（推荐）

```cpp
#include <iostream>

class Logger {
private:
    Logger() { std::cout << "Logger构造" << std::endl; }
    ~Logger() { std::cout << "Logger析构" << std::endl; }

public:
    // C++11保证局部静态变量初始化是线程安全的
    static Logger& getInstance() {
        static Logger instance;
        return instance;
    }

    // 禁止拷贝和移动
    Logger(const Logger&) = delete;
    Logger& operator=(const Logger&) = delete;

    void log(const std::string& msg) {
        std::cout << "[LOG] " << msg << std::endl;
    }
};

int main() {
    Logger::getInstance().log("程序启动");
    Logger::getInstance().log("处理请求");
    Logger::getInstance().log("程序结束");
    return 0;
}
```

**输出：**
```
Logger构造
[LOG] 程序启动
[LOG] 处理请求
[LOG] 程序结束
Logger析构
```

### 2.2 带配置的单例

```cpp
#include <iostream>
#include <string>

class AppConfig {
private:
    std::string dbHost;
    int dbPort;

    AppConfig() : dbHost("localhost"), dbPort(3306) {}

public:
    static AppConfig& instance() {
        static AppConfig cfg;
        return cfg;
    }

    AppConfig(const AppConfig&) = delete;
    AppConfig& operator=(const AppConfig&) = delete;

    void setDbHost(const std::string& host) { dbHost = host; }
    void setDbPort(int port) { dbPort = port; }

    const std::string& getDbHost() const { return dbHost; }
    int getDbPort() const { return dbPort; }
};

int main() {
    AppConfig::instance().setDbHost("192.168.1.100");
    AppConfig::instance().setDbPort(5432);

    std::cout << "DB: " << AppConfig::instance().getDbHost()
              << ":" << AppConfig::instance().getDbPort() << std::endl;
    return 0;
}
```

**输出：**
```
DB: 192.168.1.100:5432
```

## 三、注意事项与常见陷阱

- C++11起局部静态变量初始化是线程安全的，无需额外加锁
- 单例模式可能隐藏依赖关系，降低可测试性
- 全局单例的析构顺序在不同翻译单元间是未定义的
- 考虑依赖注入（DI）作为单例的替代方案
- 禁用拷贝和移动操作防止意外复制
