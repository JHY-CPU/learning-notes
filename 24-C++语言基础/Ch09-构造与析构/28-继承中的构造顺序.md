# 继承中的构造顺序

## 一、概念说明

在继承体系中，对象的构造遵循严格的顺序：**基类 → 成员变量 → 派生类**。这保证了在任何构造阶段，所依赖的部分都已被初始化。

### 1.1 构造顺序规则

1. 虚基类（按继承层次，最远派生类负责构造）
2. 直接基类（按声明顺序）
3. 非静态成员变量（按声明顺序，非初始化列表顺序）
4. 派生类构造函数体

## 二、具体用法

### 2.1 多层继承构造顺序

```cpp
#include <iostream>

class A {
public:
    A() { std::cout << "1. A构造" << std::endl; }
};

class B {
public:
    B() { std::cout << "2. B构造" << std::endl; }
};

class C : public A {
    B b;  // 成员变量
public:
    C() : b() {
        std::cout << "4. C构造（成员B已在第2步构造）" << std::endl;
    }
};

int main() {
    std::cout << "创建C对象:" << std::endl;
    C obj;
    return 0;
}
```

**输出：**
```
创建C对象:
1. A构造
2. B构造
4. C构造（成员B已在第2步构造）
```

### 2.2 多层继承链

```cpp
#include <iostream>

class GrandParent {
public:
    GrandParent() { std::cout << "GrandParent构造" << std::endl; }
};

class Parent : public GrandParent {
public:
    Parent() { std::cout << "Parent构造" << std::endl; }
};

class Child : public Parent {
public:
    Child() { std::cout << "Child构造" << std::endl; }
};

int main() {
    Child c;
    return 0;
}
```

**输出：**
```
GrandParent构造
Parent构造
Child构造
```

## 三、注意事项与常见陷阱

- 初始化列表的书写顺序**不影响**实际构造顺序
- 构造顺序由**声明顺序**和**继承层次**决定
- 基类未构造完成前，无法访问基类成员
- 多继承时基类按**声明顺序**（不是初始化列表顺序）构造
- 虚基类由最远派生类（most derived class）负责构造
