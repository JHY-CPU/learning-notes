# 继承中的构造顺序

## 一、概念说明

在继承体系中，对象的构造遵循严格的顺序（C++11标准 §12.6.2）：**基类 → 成员变量 → 派生类**。这保证了在任何构造阶段，所依赖的部分都已被初始化。

构造顺序规则（按优先级从高到低）：
1. **虚基类**（按继承层次，最远派生类负责构造）
2. **直接基类**（按声明顺序，不是初始化列表顺序）
3. **非静态成员变量**（按声明顺序，不是初始化列表顺序）
4. **派生类构造函数体**

关键：构造顺序由**声明顺序和继承层次**决定，与初始化列表的书写顺序无关。

## 二、具体用法

### 2.1 单层继承构造顺序

```cpp
#include <iostream>
using namespace std;

class A {
public:
    A() { cout << "1. A构造" << endl; }
};

class B {
public:
    B() { cout << "2. B构造" << endl; }
};

class C : public A {
    B b;  // 成员变量
public:
    C() : b() {
        cout << "3. C构造体（成员B已在第2步构造）" << endl;
    }
};

int main() {
    cout << "创建C对象:" << endl;
    C obj;
    // 输出:
    // 创建C对象:
    // 1. A构造
    // 2. B构造
    // 3. C构造体（成员B已在第2步构造）

    return 0;
}
```

### 2.2 多层继承链

```cpp
#include <iostream>
using namespace std;

class GrandParent {
public:
    GrandParent() { cout << "GrandParent构造" << endl; }
};

class Parent : public GrandParent {
public:
    Parent() { cout << "Parent构造" << endl; }
};

class Child : public Parent {
public:
    Child() { cout << "Child构造" << endl; }
};

int main() {
    Child c;
    // 输出:
    // GrandParent构造
    // Parent构造
    // Child构造

    return 0;
}
```

### 2.3 多继承构造顺序

```cpp
#include <iostream>
using namespace std;

class A {
public:
    A() { cout << "A构造" << endl; }
};

class B {
public:
    B() { cout << "B构造" << endl; }
};

class C {
public:
    C() { cout << "C构造" << endl; }
};

// 继承顺序决定构造顺序：A → B → C → 成员 → Derived
class Derived : public A, public B {
    C member;
public:
    Derived() : member() {
        cout << "Derived构造体" << endl;
    }
};

int main() {
    Derived d;
    // 输出:
    // A构造
    // B构造
    // C构造
    // Derived构造体

    return 0;
}
```

### 2.4 初始化列表顺序不影响构造顺序

```cpp
#include <iostream>
using namespace std;

class Base {
public:
    Base(int v) { cout << "Base(" << v << ")" << endl; }
};

class Member {
public:
    Member(int v) { cout << "Member(" << v << ")" << endl; }
};

class Derived : public Base {
    Member m;
public:
    // 初始化列表写成 m(2), Base(1)，但实际先构造Base再构造m
    Derived() : m(2), Base(1) {
        cout << "Derived构造体" << endl;
    }
};

int main() {
    Derived d;
    // 输出（注意顺序）:
    // Base(1)      ← 先基类
    // Member(2)    ← 再成员
    // Derived构造体 ← 最后派生类

    return 0;
}
```

### 2.5 传递参数给基类构造函数

```cpp
#include <iostream>
#include <string>
using namespace std;

class Animal {
    string name;
public:
    Animal(const string& n) : name(n) {
        cout << "Animal: " << name << endl;
    }
};

class Dog : public Animal {
    string breed;
public:
    Dog(const string& name, const string& b)
        : Animal(name),  // 传递参数给基类
          breed(b) {
        cout << "Dog breed: " << breed << endl;
    }
};

int main() {
    Dog d("Buddy", "Golden Retriever");
    // Animal: Buddy
    // Dog breed: Golden Retriever

    return 0;
}
```

## 三、注意事项与常见陷阱

1. **初始化列表的书写顺序不影响实际构造顺序**：只与声明顺序和继承层次有关
2. **构造顺序由声明顺序和继承层次决定**：与程序员的意图无关
3. **基类未构造完成前，无法安全访问基类成员**：在基类构造函数中虚函数不表现为多态
4. **多继承时基类按声明顺序构造**：`class D : public A, public B` 先A后B
5. **虚基类由最远派生类负责构造**：即使中间类也声明了虚基类的初始化，也只由最终派生类调用
6. **编译器会警告初始化列表顺序与声明顺序不一致**：注意此类警告
