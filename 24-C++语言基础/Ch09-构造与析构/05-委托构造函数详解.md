# 委托构造函数详解

## 一、概念说明

**委托构造函数**（Delegating Constructor）是C++11引入的特性，允许一个构造函数调用同一个类的另一个构造函数来完成初始化工作。这有助于消除构造函数之间的重复代码，遵循DRY（Don't Repeat Yourself）原则。

### 1.1 使用场景

当多个构造函数有共同的初始化逻辑时，可以将公共部分提取到一个"目标构造函数"中，其他构造函数通过委托调用它。

## 二、具体用法

### 2.1 基本语法

```cpp
#include <iostream>
#include <string>

class Server {
private:
    std::string host;
    int port;
    int timeout;
    bool verbose;

    void init() {
        std::cout << "Server初始化: " << host << ":" << port
                  << ", timeout=" << timeout << std::endl;
    }

public:
    // 目标构造函数：包含所有成员的初始化
    Server(const std::string& h, int p, int t, bool v)
        : host(h), port(p), timeout(t), verbose(v) {
        init();
    }

    // 委托构造函数：调用目标构造函数
    Server(const std::string& h, int p)
        : Server(h, p, 30, false) {}   // 委托到四参数构造

    Server(const std::string& h)
        : Server(h, 8080, 30, false) {} // 委托到四参数构造

    Server()
        : Server("localhost", 8080, 30, false) {} // 委托到四参数构造
};

int main() {
    Server s1;
    Server s2("192.168.1.1");
    Server s3("192.168.1.1", 3306);
    Server s4("192.168.1.1", 3306, 60, true);
    return 0;
}
```

**输出：**
```
Server初始化: localhost:8080, timeout=30
Server初始化: 192.168.1.1:8080, timeout=30
Server初始化: 192.168.1.1:3306, timeout=30
Server初始化: 192.168.1.1:3306, timeout=60
```

### 2.2 避免重复验证逻辑

```cpp
#include <iostream>
#include <stdexcept>
#include <string>

class Config {
private:
    int maxConnections;
    std::string logLevel;

    // 目标构造函数中做所有验证
    Config(int max, const std::string& level)
        : maxConnections(max), logLevel(level) {
        if (max <= 0) throw std::invalid_argument("maxConnections must be positive");
        if (level != "DEBUG" && level != "INFO" && level != "ERROR")
            throw std::invalid_argument("Invalid log level");
    }

public:
    // 各种便捷构造函数，都委托到主构造函数
    Config() : Config(100, "INFO") {}
    Config(int max) : Config(max, "INFO") {}
    Config(const std::string& level) : Config(100, level) {}

    void display() const {
        std::cout << "maxConn=" << maxConnections << ", level=" << logLevel << std::endl;
    }
};

int main() {
    Config c1; c1.display();
    Config c2(200); c2.display();
    Config c3("DEBUG"); c3.display();
    return 0;
}
```

**输出：**
```
maxConn=100, level=INFO
maxConn=200, level=INFO
maxConn=100, level=DEBUG
```

## 三、注意事项与常见陷阱

- 委托构造函数中**不能再使用成员初始化列表**初始化成员
- 不允许**循环委托**（A委托B，B委托A），会导致编译错误或未定义行为
- 目标构造函数先执行完毕后，委托构造函数的函数体才会执行
- 委托构造函数抛出异常时，已构造的成员会被正常析构
- 适合消除构造函数中的重复初始化和验证代码
