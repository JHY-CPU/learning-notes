# 对象生命周期管理

## 一、概念说明

C++中对象的生命周期由其存储方式决定。理解不同存储类型的对象生命周期对于避免悬空引用和内存泄漏至关重要。对象分为**栈对象**、**堆对象**、**静态对象**和**线程局部对象**。

## 二、具体用法

### 2.1 栈对象（自动存储）

```cpp
#include <iostream>

class Widget {
    std::string name;
public:
    Widget(const std::string& n) : name(n) {
        std::cout << "构造: " << name << std::endl;
    }
    ~Widget() { std::cout << "析构: " << name << std::endl; }
};

void demo() {
    Widget w1("栈对象A");
    {
        Widget w2("栈对象B");
    }  // w2在此处析构
    // w1在函数结束时析构
}

int main() {
    std::cout << "进入demo" << std::endl;
    demo();
    std::cout << "离开demo" << std::endl;
    return 0;
}
```

**输出：**
```
进入demo
构造: 栈对象A
构造: 栈对象B
析构: 栈对象B
析构: 栈对象A
离开demo
```

### 2.2 堆对象（动态存储）

```cpp
#include <iostream>
#include <memory>

class Resource {
public:
    Resource() { std::cout << "堆对象构造" << std::endl; }
    ~Resource() { std::cout << "堆对象析构" << std::endl; }
};

int main() {
    // 裸指针方式（不推荐）
    Resource* raw = new Resource();
    delete raw;  // 必须手动释放

    // 智能指针方式（推荐）
    auto ptr = std::make_unique<Resource>();
    // 离开作用域自动释放
    return 0;
}
```

**输出：**
```
堆对象构造
堆对象析构
堆对象构造
堆对象析构
```

### 2.3 静态对象与局部静态

```cpp
#include <iostream>

class Singleton {
public:
    Singleton() { std::cout << "静态对象构造" << std::endl; }
    ~Singleton() { std::cout << "静态对象析构" << std::endl; }
};

static Singleton globalObj;  // 全局静态，程序开始时构造

void func() {
    static Singleton localObj;  // 局部静态，首次调用时构造
    std::cout << "func被调用" << std::endl;
}

int main() {
    std::cout << "main开始" << std::endl;
    func();
    func();  // 第二次调用不再构造
    std::cout << "main结束" << std::endl;
    return 0;
}
```

**输出：**
```
静态对象构造
main开始
静态对象构造
func被调用
func被调用
main结束
静态对象析构
静态对象析构
```

## 三、注意事项与常见陷阱

- 栈对象离开作用域自动析构，不需要手动管理
- 堆对象必须由程序员负责释放（或使用智能指针）
- 局部静态对象线程安全初始化（C++11起）
- 返回局部对象的引用或指针是**未定义行为**
- 静态对象的析构顺序在不同翻译单元间是未定义的（"静态初始化顺序"问题）
