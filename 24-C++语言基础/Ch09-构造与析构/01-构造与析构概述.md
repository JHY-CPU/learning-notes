# 构造与析构概述

## 一、概念说明

在C++中，每个类都可以定义**构造函数**（Constructor）和**析构函数**（Destructor），它们是对象生命周期的两个关键节点。构造函数在对象创建时自动调用，负责初始化对象的成员变量和分配资源；析构函数在对象销毁时自动调用，负责释放资源和清理工作。

### 1.1 对象生命周期

对象从创建到销毁的完整过程：
1. **创建阶段**：分配内存 → 调用构造函数 → 对象可用
2. **使用阶段**：对象处于有效状态，可以调用成员函数
3. **销毁阶段**：调用析构函数 → 释放内存 → 对象消亡

### 1.2 RAII理念

**RAII**（Resource Acquisition Is Initialization，资源获取即初始化）是C++最核心的编程理念之一：
- 在构造函数中获取资源（内存、文件句柄、锁等）
- 在析构函数中释放资源
- 利用对象的生命周期自动管理资源，避免泄漏

```cpp
#include <iostream>
#include <fstream>
#include <string>

class FileHandler {
private:
    std::fstream file;
public:
    // 构造函数：获取资源
    FileHandler(const std::string& filename) {
        file.open(filename, std::ios::out);
        std::cout << "文件已打开: " << filename << std::endl;
    }
    // 析构函数：释放资源
    ~FileHandler() {
        if (file.is_open()) {
            file.close();
            std::cout << "文件已关闭" << std::endl;
        }
    }
    void write(const std::string& data) {
        file << data;
    }
};
```

## 二、具体用法

### 2.1 基本语法

```cpp
class Person {
private:
    std::string name;
    int age;
public:
    // 构造函数：与类同名，无返回值
    Person(const std::string& n, int a) : name(n), age(a) {
        std::cout << "构造函数被调用: " << name << std::endl;
    }
    // 析构函数：类名前加~，无参数，无返回值
    ~Person() {
        std::cout << "析构函数被调用: " << name << std::endl;
    }
    void display() const {
        std::cout << "姓名: " << name << ", 年龄: " << age << std::endl;
    }
};

int main() {
    std::cout << "=== 进入main函数 ===" << std::endl;
    {
        Person p1("张三", 25);
        p1.display();
        std::cout << "=== 即将离开作用域 ===" << std::endl;
    }
    std::cout << "=== 离开作用域后 ===" << std::endl;
    return 0;
}
```

**输出：**
```
=== 进入main函数 ===
构造函数被调用: 张三
姓名: 张三, 年龄: 25
=== 即将离开作用域 ===
析构函数被调用: 张三
=== 离开作用域后 ===
```

### 2.2 构造与析构的调用时机

| 场景 | 构造时机 | 析构时机 |
|------|---------|---------|
| 栈对象 | 定义时 | 离开作用域时 |
| 堆对象 | `new`时 | `delete`时 |
| 静态对象 | 程序启动时 | 程序结束时 |
| 全局对象 | 程序启动时 | 程序结束时 |
| 临时对象 | 表达式求值时 | 表达式结束时 |

## 三、注意事项与常见陷阱

- 构造函数**没有返回值**，也不需要写`void`
- 析构函数**不能有参数**，因此不能重载
- 一个类可以有多个构造函数（重载），但只能有一个析构函数
- 如果用户没有定义任何构造函数，编译器会生成一个默认构造函数
- 如果定义了任何构造函数，编译器将不再自动生成默认构造函数
- 构造函数和析构函数**不能被声明为`const`**
- 建议始终遵循RAII原则管理资源
