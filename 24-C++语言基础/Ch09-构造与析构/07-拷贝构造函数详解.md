# 拷贝构造函数详解

## 一、概念说明

**拷贝构造函数**（Copy Constructor）是一种特殊的构造函数，使用同类型的另一个对象来初始化新对象。其签名通常为`ClassName(const ClassName& other)`。

### 1.1 触发时机

拷贝构造函数在以下情况被调用：
- 使用已有对象初始化新对象：`MyClass a = b;`
- 函数按值传递参数时
- 函数按值返回对象时
- 抛出或捕获异常对象时

### 1.2 深拷贝 vs 浅拷贝

- **浅拷贝**：仅复制指针值，两个对象指向同一块内存（编译器默认行为）
- **深拷贝**：为新对象分配独立的内存，复制原始数据

## 二、具体用法

### 2.1 默认浅拷贝的问题

```cpp
#include <iostream>

class ShallowArray {
public:
    int* data;
    int size;
    ShallowArray(int s) : data(new int[s]), size(s) {
        for (int i = 0; i < s; ++i) data[i] = i;
    }
    ~ShallowArray() {
        delete[] data;  // 危险！两个对象指向同一块内存
        std::cout << "析构完成" << std::endl;
    }
    // 编译器生成的浅拷贝：data指向同一块内存
};

int main() {
    ShallowArray a1(5);
    ShallowArray a2 = a1;  // 浅拷贝！
    // a1和a2.data指向同一块内存
    // 程序结束时双重delete → 未定义行为（崩溃）
    return 0;
}
```

### 2.2 正确的深拷贝实现

```cpp
#include <iostream>

class IntArray {
private:
    int* data;
    int size;
public:
    IntArray(int s) : data(new int[s]()), size(s) {}

    // 深拷贝构造函数
    IntArray(const IntArray& other) : data(new int[other.size]), size(other.size) {
        for (int i = 0; i < size; ++i) {
            data[i] = other.data[i];
        }
        std::cout << "深拷贝构造: size=" << size << std::endl;
    }

    ~IntArray() { delete[] data; }

    void set(int idx, int val) { data[idx] = val; }
    int get(int idx) const { return data[idx]; }
};

int main() {
    IntArray a1(3);
    a1.set(0, 10); a1.set(1, 20); a1.set(2, 30);

    IntArray a2 = a1;  // 深拷贝
    a2.set(0, 999);

    std::cout << "a1[0]=" << a1.get(0) << ", a2[0]=" << a2.get(0) << std::endl;
    return 0;
}
```

**输出：**
```
深拷贝构造: size=3
a1[0]=10, a2[0]=999
```

### 2.3 拷贝构造的触发场景

```cpp
#include <iostream>

class Widget {
public:
    Widget() { std::cout << "默认构造" << std::endl; }
    Widget(const Widget&) { std::cout << "拷贝构造" << std::endl; }
};

Widget createWidget() {
    Widget w;
    return w;  // 可能触发拷贝（编译器通常优化掉）
}

void takeByValue(Widget w) {  // 按值传参
    std::cout << "函数体" << std::endl;
}

int main() {
    std::cout << "--- 场景1: 直接初始化 ---" << std::endl;
    Widget w1;
    Widget w2 = w1;

    std::cout << "--- 场景2: 按值传参 ---" << std::endl;
    takeByValue(w1);

    std::cout << "--- 场景3: 函数返回 ---" << std::endl;
    Widget w3 = createWidget();
    return 0;
}
```

**输出：**
```
--- 场景1: 直接初始化 ---
默认构造
拷贝构造
--- 场景2: 按值传参 ---
拷贝构造
函数体
--- 场景3: 函数返回 ---
默认构造
```

## 三、注意事项与常见陷阱

- 管理动态资源的类**必须**自定义拷贝构造函数（深拷贝）
- 拷贝构造的参数必须是**引用**（`const ClassName&`），否则会无限递归
- C++17的**强制拷贝消除**（guaranteed copy elision）减少了不必要的拷贝
- 编译器可能优化掉拷贝构造（RVO/NRVO），但类仍需要定义拷贝构造
- `=delete`可以禁止拷贝
