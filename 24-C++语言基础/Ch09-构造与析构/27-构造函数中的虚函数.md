# 构造函数中的虚函数

## 一、概念说明

在构造函数中调用虚函数**不会发生动态多态**。这是因为派生类部分尚未构造完成，vptr（虚函数表指针）在构造过程中逐步更新。此时调用虚函数将绑定到**当前类**的版本，而非派生类的重写版本。

### 1.1 原理

当基类构造函数执行时，对象的类型被视为基类而非派生类。vptr首先指向基类的vtable，派生类构造函数执行时才更新为派生类的vtable。

## 二、具体用法

### 2.1 构造期间虚函数行为

```cpp
#include <iostream>

class Base {
public:
    Base() {
        std::cout << "Base构造中...";
        print();  // 调用Base::print，而非Derived::print
        std::cout << std::endl;
    }
    virtual void print() { std::cout << "Base::print()"; }
    virtual ~Base() = default;
};

class Derived : public Base {
    int value;
public:
    Derived(int v) : value(v) {
        std::cout << "Derived构造中...";
        print();  // 调用Derived::print（此时vptr已更新）
        std::cout << std::endl;
    }
    void print() override {
        std::cout << "Derived::print(), value=" << value;
    }
};

int main() {
    Derived d(42);
    // 输出顺序说明构造时虚函数的行为
    return 0;
}
```

**输出：**
```
Base构造中...Base::print()
Derived构造中...Derived::print(), value=42
```

### 2.2 析构中同理

```cpp
#include <iostream>

class Base {
public:
    virtual void log() { std::cout << "Base::log" << std::endl; }
    virtual ~Base() {
        std::cout << "Base析构中...";
        log();  // 调用Base::log
        std::cout << std::endl;
    }
};

class Derived : public Base {
public:
    void log() override { std::cout << "Derived::log" << std::endl; }
    ~Derived() override {
        std::cout << "Derived析构中...";
        log();  // 调用Derived::log
        std::cout << std::endl;
    }
};

int main() {
    Derived d;
    return 0;
}
```

**输出：**
```
Derived析构中...Derived::log
Base析构中...Base::log
```

## 三、注意事项与常见陷阱

- 构造函数中调用虚函数调用的是**当前类**的版本
- 这是C++语言标准规定的行为，不是编译器bug
- 避免在构造/析构中调用虚函数，改用工厂方法或两阶段初始化
- 如果必须在构造中执行多态行为，可使用NVI（Non-Virtual Interface）模式
- 析构函数中同样适用：派生类析构后，虚函数调用绑定到基类版本
