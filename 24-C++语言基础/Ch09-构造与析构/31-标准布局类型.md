# 标准布局类型

## 一、概念说明

**标准布局类型**（Standard Layout Type）的内存布局与C语言兼容，可以安全地在C和C++之间传递。标准布局类型保证了成员在内存中的排列顺序和对齐方式的确定性。

### 1.1 判断标准

类型是标准布局类型，当且仅当：
- 所有非静态成员有相同的访问控制（全public或全private）
- 没有虚函数和虚基类
- 所有基类都是标准布局
- 没有与第一个非静态成员同类型的基类

## 二、具体用法

### 2.1 标准布局检查

```cpp
#include <iostream>
#include <type_traits>

struct Standard {
    int x;
    double y;
    char z;
};

struct NotStandard1 {
    int a;
private:
    double b;  // 混合访问控制，非标准布局
};

struct NotStandard2 {
    virtual void f() {}  // 有虚函数，非标准布局
};

int main() {
    std::cout << "Standard: " << std::is_standard_layout_v<Standard> << std::endl;
    std::cout << "NotStandard1: " << std::is_standard_layout_v<NotStandard1> << std::endl;
    std::cout << "NotStandard2: " << std::is_standard_layout_v<NotStandard2> << std::endl;

    // 标准布局保证：第一个成员偏移量为0
    Standard s;
    std::cout << "offsetof(x): " << offsetof(Standard, x) << std::endl;
    std::cout << "offsetof(y): " << offsetof(Standard, y) << std::endl;
    return 0;
}
```

**输出：**
```
Standard: 1
NotStandard1: 0
NotStandard2: 0
offsetof(x): 0
offsetof(y): 8
```

### 2.2 C/C++互操作

```cpp
#include <iostream>
#include <cstring>

// 标准布局类型可用于C/C++互操作
struct Packet {
    int type;
    int length;
    char data[256];
};

static_assert(std::is_standard_layout_v<Packet>, "必须是标准布局");

extern "C" void process_packet(const Packet* p) {
    std::cout << "type=" << p->type << ", len=" << p->length << std::endl;
}
```

## 三、注意事项与常见陷阱

- 标准布局类型可以安全地与C语言代码互操作
- `offsetof`宏仅对标准布局类型有效
- 标准布局不等于平凡（有标准布局但非平凡的类型）
- `std::is_trivially_copyable_v`比`is_standard_layout_v`更常用于判断是否可`memcpy`
- 标准布局类型的第一个成员地址等于对象地址
