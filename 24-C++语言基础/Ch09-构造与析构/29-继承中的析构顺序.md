# 继承中的析构顺序

## 一、概念说明

析构顺序与构造顺序**完全相反**（C++11标准 §12.4）：**派生类析构函数体 → 成员变量（声明逆序） → 基类**。这保证了对象在析构过程中，所依赖的部分仍然有效。

析构顺序规则（与构造相反）：
1. **派生类析构函数体**
2. **非静态成员变量（声明逆序析构）**
3. **直接基类（声明逆序析构）**
4. **虚基类（最远派生类负责析构）**

### 虚析构的必要性

通过基类指针 `delete` 派生类对象时，如果基类析构函数不是 `virtual` 的，将导致**未定义行为**（通常只调用基类析构函数，派生类资源泄漏）。

## 二、具体用法

### 2.1 基本析构顺序

```cpp
#include <iostream>
using namespace std;

class A {
public:
    ~A() { cout << "~A析构" << endl; }
};

class B {
public:
    ~B() { cout << "~B析构" << endl; }
};

class C : public A {
    B member;
public:
    ~C() { cout << "~C析构函数体" << endl; }
};

int main() {
    {
        C obj;
        cout << "--- 即将析构 ---" << endl;
    }
    // 输出:
    // --- 即将析构 ---
    // ~C析构函数体    ← 先析构派生类函数体
    // ~B析构          ← 再析构成员（声明逆序，B在A之前声明为成员）
    // ~A析构          ← 最后析构基类

    return 0;
}
```

### 2.2 多层继承析构顺序

```cpp
#include <iostream>
using namespace std;

class GrandParent {
public:
    ~GrandParent() { cout << "~GrandParent析构" << endl; }
};

class Parent : public GrandParent {
public:
    ~Parent() { cout << "~Parent析构" << endl; }
};

class Child : public Parent {
public:
    ~Child() { cout << "~Child析构" << endl; }
};

int main() {
    {
        Child c;
        cout << "--- 析构 ---" << endl;
    }
    // 输出:
    // --- 析构 ---
    // ~Child析构
    // ~Parent析构
    // ~GrandParent析构

    return 0;
}
```

### 2.3 虚析构函数

```cpp
#include <iostream>
using namespace std;

class Base {
public:
    virtual ~Base() { cout << "~Base析构" << endl; }
};

class Derived : public Base {
    int* data;
public:
    Derived() : data(new int[100]) {
        cout << "Derived构造，分配100个int" << endl;
    }

    ~Derived() override {
        delete[] data;
        cout << "~Derived析构，释放内存" << endl;
    }
};

int main() {
    Base* ptr = new Derived();
    delete ptr;  // 虚析构保证正确调用~Derived
    // 输出:
    // Derived构造，分配100个int
    // ~Derived析构，释放内存
    // ~Base析构

    // 如果Base的析构不是virtual：
    // 只输出 ~Base析构（内存泄漏！）

    return 0;
}
```

### 2.4 纯虚析构函数

```cpp
#include <iostream>
using namespace std;

// 抽象基类：纯虚析构函数
class Interface {
public:
    virtual ~Interface() = 0;  // 纯虚析构函数
};

// 纯虚析构函数必须提供实现！
Interface::~Interface() {
    cout << "~Interface析构" << endl;
}

class Impl : public Interface {
public:
    ~Impl() override {
        cout << "~Impl析构" << endl;
    }
};

int main() {
    Interface* p = new Impl();
    delete p;
    // 输出:
    // ~Impl析构
    // ~Interface析构

    return 0;
}
```

### 2.5 成员析构顺序（逆序声明）

```cpp
#include <iostream>
using namespace std;

class Resource {
    string name;
public:
    Resource(const string& n) : name(n) {
        cout << "  " << name << " 构造" << endl;
    }
    ~Resource() {
        cout << "  " << name << " 析构" << endl;
    }
};

class Owner {
    Resource r1{"成员1"};    // 先声明
    Resource r2{"成员2"};    // 后声明
    Resource r3{"成员3"};    // 最后声明

public:
    Owner() { cout << "Owner构造体" << endl; }
    ~Owner() { cout << "Owner析构体" << endl; }
};

int main() {
    {
        Owner o;
        cout << "--- 析构 ---" << endl;
    }
    // 输出:
    //   成员1 构造
    //   成员2 构造
    //   成员3 构造
    // Owner构造体
    // --- 析构 ---
    // Owner析构体
    //   成员3 析构    ← 逆序析构
    //   成员2 析构
    //   成员1 析构

    return 0;
}
```

## 三、注意事项与常见陷阱

1. **析构顺序是构造顺序的逆序**：先构造的后析构
2. **基类析构函数应声明为virtual**：如果有派生类且可能通过基类指针删除
3. **纯虚析构函数必须提供实现**：`= 0` 仍需函数体（与普通纯虚函数不同）
4. **final类不需要虚析构**：但保持一致性仍是好习惯
5. **析构中的虚函数调用行为与构造中类似**：绑定到当前类版本，不是最派生类版本
6. **异常从析构函数中抛出可能导致程序终止**：C++11起析构函数默认`noexcept`
7. **智能指针的析构顺序**：成员变量按声明逆序析构，智能指针自动释放所管理的对象
