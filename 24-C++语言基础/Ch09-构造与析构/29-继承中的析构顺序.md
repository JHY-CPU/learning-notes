# 继承中的析构顺序

## 一、概念说明

析构顺序与构造顺序**完全相反**：**派生类析构函数体 → 成员变量（声明逆序） → 基类**。这保证了对象在析构过程中，所依赖的部分仍然有效。

### 1.1 虚析构的必要性

通过基类指针`delete`派生类对象时，如果基类析构函数不是`virtual`的，将导致**未定义行为**（通常只调用基类析构，派生类资源泄漏）。

## 二、具体用法

### 2.1 多层析构顺序

```cpp
#include <iostream>

class A {
public:
    ~A() { std::cout << "~A析构" << std::endl; }
};

class B {
public:
    ~B() { std::cout << "~B析构" << std::endl; }
};

class C : public A {
    B member;
public:
    ~C() { std::cout << "~C析构函数体" << std::endl; }
};

int main() {
    {
        C obj;
        std::cout << "--- 即将析构 ---" << std::endl;
    }
    return 0;
}
```

**输出：**
```
--- 即将析构 ---
~C析构函数体
~B析构（成员）
~A析构
```

### 2.2 基类虚析构

```cpp
#include <iostream>

class Base {
public:
    virtual ~Base() { std::cout << "~Base析构" << std::endl; }
};

class Derived : public Base {
    int* data;
public:
    Derived() : data(new int[100]) {}
    ~Derived() override {
        delete[] data;
        std::cout << "~Derived析构" << std::endl;
    }
};

int main() {
    Base* ptr = new Derived();
    delete ptr;  // 虚析构保证正确调用~Derived
    return 0;
}
```

**输出：**
```
~Derived析构
~Base析构
```

## 三、注意事项与常见陷阱

- 析构顺序是构造顺序的逆序
- 基类析构函数应声明为`virtual`（如果有派生类）
- 纯虚析构函数必须提供实现（`= 0`但仍需函数体）
- `final`类不需要虚析构（但保持一致性仍是好习惯）
- 析构中的虚函数调用行为与构造中类似（绑定到当前类版本）
