# 编译器生成的特殊成员

## 一、概念说明

C++编译器会根据用户的定义情况自动生成特殊成员函数。C++11/14/20对这些规则进行了调整，特别是引入了**移动操作**和`=default`/`=delete`控制。

### 1.1 特殊成员函数一览

| 特殊成员函数 | 自动生成条件 |
|-------------|-------------|
| 默认构造函数 | 用户未定义任何构造函数 |
| 析构函数 | 用户未定义 |
| 拷贝构造函数 | 用户未定义拷贝/移动操作和析构 |
| 拷贝赋值运算符 | 用户未定义拷贝/移动操作和析构 |
| 移动构造函数 | 用户未定义拷贝/移动操作和析构 |
| 移动赋值运算符 | 用户未定义拷贝/移动操作和析构 |

## 二、具体用法

### 2.1 C++11/14/20变化

```cpp
#include <iostream>
#include <type_traits>

// 用户未定义任何特殊成员 → 全部自动生成
struct AllDefault {
    int x;
    double y;
};

// 定义了析构 → 阻止移动操作生成（C++11）
struct HasDestructor {
    int x;
    ~HasDestructor() {}
};

// 定义了拷贝构造 → 阻止移动操作生成
struct HasCopy {
    int x;
    HasCopy(const HasCopy& o) : x(o.x) {}
};

int main() {
    std::cout << "AllDefault trivial: " << std::is_trivial_v<AllDefault> << std::endl;
    std::cout << "HasDestructor movable: "
              << std::is_move_constructible_v<HasDestructor> << std::endl;
    std::cout << "HasCopy movable: "
              << std::is_move_constructible_v<HasCopy> << std::endl;
    return 0;
}
```

**输出：**
```
AllDefault trivial: 1
HasDestructor movable: 0
HasCopy movable: 0
```

### 2.2 使用=default显式要求生成

```cpp
#include <iostream>

struct Base {
    int value;
    // 显式要求编译器生成默认构造
    Base() = default;
    // 显式要求编译器生成拷贝构造
    Base(const Base&) = default;
    // 显式要求编译器生成移动构造
    Base(Base&&) = default;
    Base& operator=(const Base&) = default;
    Base& operator=(Base&&) = default;
    ~Base() = default;
};

int main() {
    Base b1;
    b1.value = 42;
    Base b2 = b1;  // 使用编译器生成的拷贝构造
    std::cout << "b2.value = " << b2.value << std::endl;
    return 0;
}
```

**输出：**
```
b2.value = 42
```

## 三、注意事项与常见陷阱

- 定义析构函数会阻止移动操作的自动生成
- 定义任何拷贝操作会阻止移动操作的自动生成
- 定义任何移动操作会阻止拷贝操作的自动生成
- `=default`在类内声明和类外定义的行为可能不同（triviality）
- C++20引入了更多`=default`的比较操作符生成规则
- 使用`std::is_trivially_copyable_v`检查类型是否可平凡复制
