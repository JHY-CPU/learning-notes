# 不可复制类型

## 一、概念说明

某些资源类型天然不支持拷贝（如文件句柄、互斥锁、`unique_ptr`等）。使用`=delete`可以显式禁止拷贝操作，强制编译器在发现拷贝尝试时报错，而非在运行时出现双重释放等未定义行为。

## 二、具体用法

### 2.1 禁止拷贝

```cpp
#include <iostream>
#include <memory>
#include <fstream>

class Connection {
private:
    int fd;  // 文件描述符
public:
    Connection(int f) : fd(f) {}

    // 禁止拷贝
    Connection(const Connection&) = delete;
    Connection& operator=(const Connection&) = delete;

    // 允许移动
    Connection(Connection&& other) noexcept : fd(other.fd) {
        other.fd = -1;
    }
    Connection& operator=(Connection&& other) noexcept {
        if (this != &other) { fd = other.fd; other.fd = -1; }
        return *this;
    }

    ~Connection() {
        if (fd >= 0) std::cout << "关闭连接 " << fd << std::endl;
    }
};

int main() {
    Connection c1(1);
    // Connection c2 = c1;  // 编译错误！拷贝已删除
    Connection c2 = std::move(c1);  // 移动OK
    return 0;
}
```

**输出：**
```
关闭连接 1
关闭连接 -1
```

### 2.2 unique_ptr的不可复制语义

```cpp
#include <iostream>
#include <memory>

class Resource {
public:
    Resource() { std::cout << "资源获取" << std::endl; }
    ~Resource() { std::cout << "资源释放" << std::endl; }
};

void process(std::unique_ptr<Resource> p) {
    std::cout << "处理中..." << std::endl;
}

int main() {
    auto p = std::make_unique<Resource>();
    // auto p2 = p;           // 编译错误：unique_ptr不可拷贝
    process(std::move(p));   // 移动转移所有权
    std::cout << "p已为空: " << (p == nullptr) << std::endl;
    return 0;
}
```

**输出：**
```
资源获取
处理中...
资源释放
p已为空: 1
```

## 三、注意事项与常见陷阱

- `=delete`可用于任何函数，不仅限于特殊成员函数
- 不可复制类型通常需要定义移动操作
- `std::mutex`、`std::atomic`、`std::unique_ptr`都是不可复制类型
- 不可复制类型可以放入`std::vector`，但只能使用移动操作
- `=delete`最好放在`public`区域，以便产生更清晰的错误信息
