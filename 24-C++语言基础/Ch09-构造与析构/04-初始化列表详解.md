# 初始化列表详解

## 一、概念说明

**成员初始化列表**（Member Initializer List）是构造函数中在函数体执行之前初始化成员变量的语法。它位于构造函数参数列表和函数体之间，以冒号`:`开头，用逗号分隔各成员的初始化。

### 1.1 初始化列表 vs 赋值

在构造函数体内赋值实际上是**先默认构造、再赋值**，而初始化列表是**直接初始化**。对于类类型的成员，初始化列表可以避免一次不必要的默认构造。

## 二、具体用法

### 2.1 基本语法

```cpp
#include <iostream>
#include <string>

class Person {
private:
    std::string name;    // 类类型成员
    const int id;        // const成员必须在初始化列表中初始化
    int& refAge;         // 引用成员必须在初始化列表中初始化
public:
    Person(const std::string& n, int i, int& age)
        : name(n),        // 直接初始化，调用string的拷贝构造
          id(i),          // const成员初始化
          refAge(age)     // 引用成员初始化
    {
        // 如果在体内赋值：name先默认构造为空串，再赋值为n
        // name = n;  // 效率更低
    }

    void display() const {
        std::cout << name << ", ID=" << id << ", age=" << refAge << std::endl;
    }
};

int main() {
    int age = 25;
    Person p("张三", 1001, age);
    p.display();
    return 0;
}
```

**输出：**
```
张三, ID=1001, age=25
```

### 2.2 初始化顺序必须与声明顺序一致

```cpp
#include <iostream>

class MyClass {
private:
    int a;
    int b;
    int sum;  // sum在a、b之后声明
public:
    // 警告：虽然写的是 b(a), a(10)，但实际按声明顺序：先a后b后sum
    MyClass(int val)
        : b(val),     // 这行先写但实际后执行！
          a(10),      // a先执行
          sum(a + b)  // sum最后执行
    {
        std::cout << "a=" << a << " b=" << b << " sum=" << sum << std::endl;
    }
};

int main() {
    MyClass obj(5);
    // a先初始化为10，b初始化为5，sum=10+5=15
    return 0;
}
```

**输出：**
```
a=10 b=5 sum=15
```

### 2.3 委托构造与继承中的初始化列表

```cpp
#include <iostream>

class Base {
protected:
    int x;
public:
    Base(int x) : x(x) { std::cout << "Base(" << x << ")" << std::endl; }
};

class Derived : public Base {
private:
    int y;
public:
    Derived(int a, int b)
        : Base(a),   // 先初始化基类
          y(b)       // 再初始化自身成员
    {
        std::cout << "Derived(" << y << ")" << std::endl;
    }
};

int main() {
    Derived d(10, 20);
    return 0;
}
```

**输出：**
```
Base(10)
Derived(20)
```

## 三、注意事项与常见陷阱

- **以下成员必须在初始化列表中初始化**：
  - `const`成员
  - 引用成员
  - 没有默认构造函数的类类型成员
  - 基类没有默认构造函数时
- **初始化顺序由声明顺序决定**，与初始化列表中的书写顺序无关
- 编译器通常会对简单类型优化，但类类型成员用初始化列表更高效
- 在初始化列表中不要使用依赖尚未初始化成员的表达式
- 建议：**始终优先使用初始化列表**而非构造函数体赋值
