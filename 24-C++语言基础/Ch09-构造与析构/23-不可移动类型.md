# 不可移动类型

## 一、概念说明

某些类型天然不支持移动操作，例如`std::mutex`（包含平台特定的同步原语，无法通过指针转移所有权）或包含不可移动成员的类型。不可移动类型不能放入需要移动操作的容器中。

不可移动的原因：
- 包含不可移动的成员（如`std::mutex`、`std::atomic`）
- 资源与特定内存地址绑定（如文件描述符包装类的特殊设计）
- 显式删除了移动操作

## 二、具体用法

### 2.1 包含不可移动成员的类型

```cpp
#include <iostream>
#include <mutex>
using namespace std;

class ThreadSafeCounter {
    mutable mutex mtx;  // mutex不可移动
    int value = 0;

public:
    ThreadSafeCounter() = default;
    ThreadSafeCounter(int v) : value(v) {}

    // 编译器不会为包含mutex的类生成移动操作
    // 隐式删除：移动构造和移动赋值被删除

    void increment() {
        lock_guard<mutex> lock(mtx);
        ++value;
    }

    int get() const {
        lock_guard<mutex> lock(mtx);
        return value;
    }
};

int main() {
    ThreadSafeCounter c1(10);
    c1.increment();

    // ThreadSafeCounter c2 = move(c1);  // 编译错误！
    // vector<ThreadSafeCounter> v; v.push_back(c1);  // 编译错误！（需要移动）

    cout << "c1 = " << c1.get() << endl;  // 11

    return 0;
}
```

### 2.2 显式删除移动操作

```cpp
#include <iostream>
#include <utility>
using namespace std;

class Pinned {
    int value;

public:
    Pinned(int v) : value(v) {}

    // 显式禁止拷贝和移动（Pinned类型：不可移动也不可复制）
    Pinned(const Pinned&) = delete;
    Pinned& operator=(const Pinned&) = delete;
    Pinned(Pinned&&) = delete;
    Pinned& operator=(Pinned&&) = delete;

    int get() const { return value; }
};

int main() {
    Pinned p(42);
    cout << p.get() << endl;  // 42

    // Pinned p2 = p;            // 编译错误：拷贝已删除
    // Pinned p3 = move(p);      // 编译错误：移动已删除
    // Pinned p4(p);              // 编译错误

    return 0;
}
```

### 2.3 不可移动但可复制的类型（罕见）

```cpp
#include <iostream>
using namespace std;

class CopyOnly {
    int* data;
    size_t size;

public:
    CopyOnly(size_t n) : data(new int[n]), size(n) {
        for (size_t i = 0; i < n; ++i) data[i] = 0;
    }

    // 拷贝：深拷贝
    CopyOnly(const CopyOnly& other)
        : data(new int[other.size]), size(other.size) {
        copy(other.data, other.data + size, data);
    }

    CopyOnly& operator=(const CopyOnly& other) {
        if (this != &other) {
            delete[] data;
            size = other.size;
            data = new int[size];
            copy(other.data, other.data + size, data);
        }
        return *this;
    }

    // 删除移动操作
    CopyOnly(CopyOnly&&) = delete;
    CopyOnly& operator=(CopyOnly&&) = delete;

    ~CopyOnly() { delete[] data; }
};
```

### 2.4 适合存储不可移动类型的容器

```cpp
#include <iostream>
#include <mutex>
#include <list>
#include <deque>
#include <array>
using namespace std;

int main() {
    // std::list：不需要移动元素（节点独立分配）
    list<mutex> mutexList;
    mutexList.emplace_back();  // OK：emplace直接在节点中构造

    // std::deque：通常不需要移动已有元素
    deque<mutex> mutexDeque;
    mutexDeque.emplace_back();  // OK

    // std::array：固定大小，不涉及元素移动
    array<mutex, 3> mutexArray;  // OK

    // std::vector：扩容时需要移动元素 → 编译错误
    // vector<mutex> mutexVec;
    // mutexVec.emplace_back();  // 如果触发扩容则需要移动

    cout << "容器创建成功" << endl;

    return 0;
}
```

### 2.5 unique_lock的移动语义对比

```cpp
#include <iostream>
#include <mutex>
using namespace std;

int main() {
    mutex mtx;

    // unique_lock是可移动但不可复制的
    unique_lock<mutex> lock1(mtx);
    // unique_lock<mutex> lock2 = lock1;  // 编译错误：不可复制

    unique_lock<mutex> lock2 = move(lock1);  // OK：可移动
    // lock1现在不拥有锁

    // 但mutex本身不可移动
    mutex m1;
    // mutex m2 = move(m1);  // 编译错误

    cout << "unique_lock可移动，mutex不可移动" << endl;

    return 0;
}
```

## 三、注意事项与常见陷阱

1. **std::mutex、std::atomic、已join/detach的std::thread是不可移动的**
2. **不可移动类型不能作为std::vector的元素**：vector扩容时需要移动元素
3. **可以用std::list或std::deque存储不可移动类型**：不需要移动已有元素
4. **std::array中不可移动类型是安全的**：固定大小，不涉及元素重分配
5. **不可移动类型通常也是不可复制的**：但技术上可以只删除移动操作
6. **包含不可移动成员的类隐式删除移动操作**：编译器不会生成移动构造/赋值
7. **使用emplace而非push_back**：直接在容器中构造，避免先创建再移动
