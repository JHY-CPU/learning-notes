# 不可移动类型

## 一、概念说明

某些类型天然不支持移动操作，例如`std::mutex`（包含平台特定的同步原语，无法通过指针转移）或包含不可移动成员的类型。不可移动类型不能放入需要移动的容器（如`std::vector`扩容时）。

## 二、具体用法

### 2.1 定义不可移动类型

```cpp
#include <iostream>
#include <mutex>

class NonMovable {
    std::mutex mtx;  // mutex不可移动
    int data;
public:
    NonMovable(int d) : data(d) {}

    // 可以正常析构，但不能移动
    // 编译器不会为包含mutex的类生成移动操作

    void update(int val) {
        std::lock_guard<std::mutex> lock(mtx);
        data = val;
    }

    int get() const { return data; }
};

int main() {
    NonMovable nm1(10);
    // NonMovable nm2 = std::move(nm1);  // 编译错误
    nm1.update(20);
    std::cout << "data = " << nm1.get() << std::endl;
    return 0;
}
```

**输出：**
```
data = 20
```

### 2.2 显式删除移动操作

```cpp
#include <iostream>

class Pinned {
    int value;
public:
    Pinned(int v) : value(v) {}

    // 显式禁止拷贝和移动
    Pinned(const Pinned&) = delete;
    Pinned& operator=(const Pinned&) = delete;
    Pinned(Pinned&&) = delete;
    Pinned& operator=(Pinned&&) = delete;

    int get() const { return value; }
};

int main() {
    Pinned p(42);
    std::cout << p.get() << std::endl;
    // Pinned p2 = p;          // 编译错误
    // Pinned p3 = std::move(p); // 编译错误
    return 0;
}
```

**输出：**
```
42
```

## 三、注意事项与常见陷阱

- `std::mutex`、`std::atomic`、`std::thread`（已join/detach）是不可移动的
- 不可移动类型不能作为`std::vector`的元素（vector扩容需要移动）
- 可以使用`std::list`或`std::deque`存储不可移动类型（不需要移动元素）
- `std::array`中不可移动类型是安全的（不涉及元素移动）
- 不可移动类型通常也是不可复制的
