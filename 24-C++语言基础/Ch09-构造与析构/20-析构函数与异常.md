# 析构函数与异常

## 一、概念说明

析构函数中抛出异常是C++中最危险的操作之一。如果析构过程中抛出异常且在析构函数内未捕获，而此时另一个异常正在传播，程序将调用`std::terminate()`直接终止。C++11起析构函数默认隐式`noexcept(true)`。

## 二、具体用法

### 2.1 析构函数异常导致程序终止

```cpp
#include <iostream>
#include <stdexcept>

class Dangerous {
public:
    ~Dangerous() noexcept(false) {  // 显式允许抛异常（不推荐）
        throw std::runtime_error("析构异常");
    }
};

int main() {
    try {
        Dangerous d;
        throw std::logic_error("业务异常");
        // d析构时又抛异常 → std::terminate()
    } catch (const std::exception& e) {
        std::cout << e.what() << std::endl;  // 永远不会执行
    }
    return 0;
}
```

**输出：**
```
terminate called after throwing an instance of 'std::runtime_error'
  what(): 析构异常
Aborted
```

### 2.2 正确处理析构中的异常

```cpp
#include <iostream>
#include <stdexcept>

class Safe {
public:
    ~Safe() noexcept {
        try {
            // 可能抛异常的操作
            bool error = true;
            if (error) {
                // 不要直接throw，而是记录错误
                std::cerr << "析构时发生错误（已记录）" << std::endl;
            }
        } catch (...) {
            std::cerr << "析构异常被捕获，防止程序终止" << std::endl;
        }
    }
};

int main() {
    Safe s;
    std::cout << "程序正常结束" << std::endl;
    return 0;
}
```

**输出：**
```
析构时发生错误（已记录）
程序正常结束
```

## 三、注意事项与常见陷阱

- **永远不要在析构函数中抛出未捕获的异常**
- C++11起析构函数默认`noexcept(true)`
- 如果析构必须执行可能失败的操作，应在内部用try-catch捕获
- 将清理操作分成"关键"和"非关键"部分，非关键部分失败可记录但不抛出
- 使用`=delete`可标记`noexcept(false)`析构为已删除（C++20起）
