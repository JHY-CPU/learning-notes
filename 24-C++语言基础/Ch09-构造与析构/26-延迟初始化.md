# 延迟初始化

## 一、概念说明

**延迟初始化**（Lazy Initialization）将对象的构造推迟到第一次使用时，避免不必要的资源消耗。常见实现包括局部静态变量和`std::optional`（C++17）。

## 二、具体用法

### 2.1 使用std::optional

```cpp
#include <iostream>
#include <optional>
#include <string>

class Database {
    std::string connection;
public:
    Database(const std::string& conn) : connection(conn) {
        std::cout << "数据库连接建立: " << conn << std::endl;
    }
    ~Database() { std::cout << "数据库连接关闭" << std::endl; }

    std::string query(const std::string& sql) {
        return "结果集[" + sql + "]";
    }
};

class DataService {
    std::optional<Database> db;  // 延迟初始化
    std::string connStr;
public:
    DataService(const std::string& conn) : connStr(conn) {}

    // 按需初始化
    Database& getDb() {
        if (!db.has_value()) {
            db.emplace(connStr);  // 第一次使用时才构造
        }
        return *db;
    }

    std::string fetchData(const std::string& sql) {
        return getDb().query(sql);
    }
};

int main() {
    DataService service("localhost:3306");
    std::cout << "服务已创建（数据库未连接）" << std::endl;
    std::cout << service.fetchData("SELECT * FROM users") << std::endl;
    return 0;
}
```

**输出：**
```
服务已创建（数据库未连接）
数据库连接建立: localhost:3306
结果集[SELECT * FROM users]
数据库连接关闭
```

### 2.2 局部静态延迟初始化

```cpp
#include <iostream>
#include <string>

class Config {
public:
    std::string value;
    Config(const std::string& v) : value(v) {
        std::cout << "Config初始化: " << v << std::endl;
    }
};

const Config& getConfig() {
    static Config cfg("default_config");  // 首次调用时初始化
    return cfg;
}

int main() {
    std::cout << "main开始" << std::endl;
    std::cout << getConfig().value << std::endl;
    std::cout << getConfig().value << std::endl;  // 第二次不再构造
    return 0;
}
```

**输出：**
```
main开始
Config初始化: default_config
default_config
default_config
```

## 三、注意事项与常见陷阱

- `std::optional`在C++17起可用，提供值语义的延迟初始化
- 局部静态变量在C++11起是线程安全初始化的
- 延迟初始化适合创建成本高但不一定被使用的对象
- 注意多线程下的竞态条件（使用`std::call_once`或局部静态）
- `std::optional`的`emplace()`支持原地构造，避免额外拷贝
