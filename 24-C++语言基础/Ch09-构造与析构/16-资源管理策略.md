# 资源管理策略

## 一、概念说明

C++中管理资源有多种策略，选择合适的策略取决于所有权语义、性能需求和安全性要求。主要策略包括：**深拷贝**、**移动语义**、**引用计数**（共享所有权）和**独占所有权**。

## 二、具体用法

### 2.1 策略对比

| 策略 | 类型 | 适用场景 |
|------|------|---------|
| 深拷贝 | 值语义 | 资源轻量、需要独立副本 |
| 移动语义 | 转移所有权 | 临时对象、容器操作 |
| 引用计数 | 共享所有权 | 多个对象共享同一资源 |
| 独占所有权 | 单一所有者 | 明确的单一拥有者 |

### 2.2 深拷贝策略

```cpp
#include <iostream>
#include <cstring>

class MyString {
    char* data;
    size_t len;
public:
    MyString(const char* s) : len(strlen(s)), data(new char[len + 1]) {
        strcpy(data, s);
    }
    // 深拷贝：每个对象拥有独立数据
    MyString(const MyString& o) : len(o.len), data(new char[o.len + 1]) {
        strcpy(data, o.data);
    }
    ~MyString() { delete[] data; }
};
```

### 2.3 引用计数策略（shared_ptr）

```cpp
#include <iostream>
#include <memory>

class Resource {
public:
    std::string name;
    Resource(const std::string& n) : name(n) {
        std::cout << "创建: " << name << std::endl;
    }
    ~Resource() {
        std::cout << "销毁: " << name << std::endl;
    }
};

int main() {
    std::shared_ptr<Resource> p1 = std::make_shared<Resource>("数据库连接");
    {
        std::shared_ptr<Resource> p2 = p1;  // 引用计数+1
        std::cout << "引用计数: " << p1.use_count() << std::endl;
    }  // p2析构，引用计数-1
    std::cout << "引用计数: " << p1.use_count() << std::endl;
    return 0;
}
```

**输出：**
```
创建: 数据库连接
引用计数: 2
引用计数: 1
销毁: 数据库连接
```

### 2.4 独占所有权策略（unique_ptr）

```cpp
#include <iostream>
#include <memory>

int main() {
    auto p1 = std::make_unique<int>(42);
    // auto p2 = p1;  // 编译错误：不可拷贝
    auto p2 = std::move(p1);  // 转移所有权
    std::cout << "*p2 = " << *p2 << std::endl;
    // p1此时为nullptr
    return 0;
}
```

**输出：**
```
*p2 = 42
```

## 三、注意事项与常见陷阱

- 优先使用`std::unique_ptr`（零开销抽象）
- 需要共享所有权时使用`std::shared_ptr`（有引用计数开销）
- 注意`std::shared_ptr`的循环引用问题，配合`std::weak_ptr`使用
- 深拷贝适合资源较轻且需要独立修改的场景
- 移动语义适合临时对象和容器操作，避免不必要的拷贝
