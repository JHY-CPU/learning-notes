# 析构函数详解

## 一、概念说明

**析构函数**（Destructor）在对象生命周期结束时自动调用，负责释放对象持有的资源。析构函数名以波浪号`~`开头，没有参数也没有返回值，每个类最多只能有一个析构函数。

### 1.1 析构顺序

- 成员变量按**声明的逆序**析构
- 派生类析构后，基类析构自动调用（析构顺序与构造顺序相反）
- 数组中每个元素按逆序析构

### 1.2 虚析构函数

通过基类指针删除派生类对象时，如果基类析构不是虚函数，将导致**未定义行为**（通常只调用基类析构，派生类资源泄漏）。

## 二、具体用法

### 2.1 基本析构函数

```cpp
#include <iostream>

class Logger {
private:
    std::string name;
public:
    Logger(const std::string& n) : name(n) {
        std::cout << "[" << name << "] 创建" << std::endl;
    }
    ~Logger() {
        std::cout << "[" << name << "] 销毁" << std::endl;
    }
};

int main() {
    Logger a("A");
    {
        Logger b("B");
        Logger c("C");
    }  // b和c在此处析构（逆序：C先，B后）
    Logger d("D");
    std::cout << "=== main结束 ===" << std::endl;
    return 0;
}
```

**输出：**
```
[A] 创建
[B] 创建
[C] 创建
[C] 销毁
[B] 销毁
[D] 创建
=== main结束 ===
[D] 销毁
[A] 销毁
```

### 2.2 虚析构函数

```cpp
#include <iostream>

class Base {
public:
    Base() { std::cout << "Base构造" << std::endl; }
    virtual ~Base() { std::cout << "Base析构" << std::endl; }
};

class Derived : public Base {
    int* data;
public:
    Derived() : data(new int[100]) {
        std::cout << "Derived构造" << std::endl;
    }
    ~Derived() override {
        delete[] data;
        std::cout << "Derived析构" << std::endl;
    }
};

int main() {
    Base* ptr = new Derived();  // 基类指针指向派生类
    delete ptr;  // 如果Base析构不是virtual，则不会调用Derived析构！
    return 0;
}
```

**输出：**
```
Base构造
Derived构造
Derived析构
Base析构
```

### 2.3 析构函数与异常

```cpp
#include <iostream>
#include <stdexcept>

class SafeResource {
public:
    ~SafeResource() noexcept {  // 析构默认就是noexcept
        try {
            // 如果析构中可能抛异常，应在内部捕获
            std::cout << "安全释放资源" << std::endl;
        } catch (...) {
            std::cerr << "析构异常被捕获" << std::endl;
        }
    }
};
```

## 三、注意事项与常见陷阱

- **基类析构函数应该声明为`virtual`**，否则通过基类指针删除派生类对象会导致资源泄漏
- 析构函数不应抛出异常，如果内部可能抛异常应自行捕获
- 析构函数不能被重载（无参数）
- C++11起析构函数默认隐式`noexcept`
- `=delete`可以禁止析构，但对象将无法被销毁（不能`delete`，不能离开作用域）
- 异常从析构函数抛出且未被捕获时，程序会调用`std::terminate()`
