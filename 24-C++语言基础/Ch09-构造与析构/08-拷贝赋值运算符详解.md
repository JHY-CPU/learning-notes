# 拷贝赋值运算符详解

## 一、概念说明

**拷贝赋值运算符**（Copy Assignment Operator）用于将一个已有对象的值赋给另一个已有对象。它与拷贝构造的区别在于：拷贝构造是**初始化新对象**，拷贝赋值是**修改已存在对象**。

### 1.1 拷贝交换惯用法

Copy-and-Swap Idiom是实现拷贝赋值的经典模式：先用拷贝构造创建临时对象，再用`swap`交换资源，最后临时对象析构释放旧资源。这种方式天然提供**强异常安全保证**。

## 二、具体用法

### 2.1 基本实现

```cpp
#include <iostream>
#include <algorithm>

class IntArray {
private:
    int* data;
    int size;
public:
    IntArray(int s) : data(new int[s]()), size(s) {}

    // 拷贝构造
    IntArray(const IntArray& other) : data(new int[other.size]), size(other.size) {
        std::copy(other.data, other.data + size, data);
    }

    // 拷贝赋值运算符：传统自赋值安全写法
    IntArray& operator=(const IntArray& other) {
        if (this != &other) {         // 自赋值检查
            delete[] data;            // 释放旧资源
            size = other.size;
            data = new int[size];     // 分配新资源
            std::copy(other.data, other.data + size, data);
        }
        return *this;                 // 返回自身引用
    }

    ~IntArray() { delete[] data; }

    void set(int i, int v) { data[i] = v; }
    int get(int i) const { return data[i]; }
};

int main() {
    IntArray a1(3), a2(3);
    a1.set(0, 10); a1.set(1, 20);
    a2 = a1;   // 拷贝赋值
    std::cout << "a2[0]=" << a2.get(0) << ", a2[1]=" << a2.get(1) << std::endl;
    return 0;
}
```

**输出：**
```
a2[0]=10, a2[1]=20
```

### 2.2 拷贝交换惯用法（推荐）

```cpp
#include <iostream>
#include <algorithm>

class Buffer {
private:
    char* data;
    int size;
public:
    Buffer(int s) : data(new char[s]()), size(s) {}

    Buffer(const Buffer& other) : data(new char[other.size]), size(other.size) {
        std::copy(other.data, other.data + size, data);
    }

    // swap成员函数（noexcept）
    friend void swap(Buffer& a, Buffer& b) noexcept {
        using std::swap;
        swap(a.data, b.data);
        swap(a.size, b.size);
    }

    // 拷贝赋值：参数按值传递（触发拷贝构造）
    Buffer& operator=(Buffer other) {  // 按值传参=隐式拷贝
        swap(*this, other);            // 交换资源
        return *this;                  // other析构时释放旧资源
    }

    ~Buffer() { delete[] data; }

    void fill(char c) { std::fill(data, data + size, c); }
};

int main() {
    Buffer b1(10), b2(5);
    b1.fill('A');
    b2 = b1;  // 拷贝赋值
    std::cout << "赋值完成" << std::endl;
    return 0;
}
```

**输出：**
```
赋值完成
```

### 2.3 自赋值检查

```cpp
#include <iostream>

class Resource {
public:
    int* ptr;
    Resource(int v) : ptr(new int(v)) {}

    Resource& operator=(const Resource& other) {
        // 方式一：显式检查自赋值
        if (this == &other) return *this;

        delete ptr;
        ptr = new int(*other.ptr);
        return *this;
    }

    ~Resource() { delete ptr; }
};

int main() {
    Resource r1(42);
    r1 = r1;  // 自赋值，安全处理
    std::cout << "*r1.ptr = " << *r1.ptr << std::endl;
    return 0;
}
```

**输出：**
```
*r1.ptr = 42
```

## 三、注意事项与常见陷阱

- 拷贝赋值必须处理**自赋值**（`a = a;`），否则可能导致释放后使用
- 拷贝交换惯用法天然避免自赋值问题（先拷贝再交换）
- 返回`*this`的引用以支持**链式赋值**（`a = b = c;`）
- 拷贝赋值和拷贝构造通常成对出现（Rule of Three）
- 拷贝交换惯用法天然提供**强异常安全保证**
