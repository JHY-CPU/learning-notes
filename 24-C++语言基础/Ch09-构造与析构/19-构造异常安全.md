# 构造异常安全

## 一、概念说明

构造函数中抛出异常时，对象被视为**从未构造成功**，析构函数**不会被调用**。因此需要特别注意：已经构造的成员会被正常析构，但尚未构造的成员和基类不会析构。如果在构造函数体内分配了资源并在异常前未释放，就会导致**资源泄漏**。

### 1.1 异常构造的规则

- 异常前已构造的成员（按声明顺序）会被自动析构
- 基类若已构造会自动析构
- 对象本身不调用析构函数（因为构造未完成）
- 使用初始化列表可确保成员构造失败时正确回滚

## 二、具体用法

### 2.1 构造函数中的资源泄漏

```cpp
#include <iostream>
#include <stdexcept>

class Leaky {
    int* data;
    int* backup;
public:
    Leaky() : data(new int[100]) {
        backup = new int[50];
        throw std::runtime_error("构造中出错");
        // backup已分配但不会被释放！
        // data和backup都泄漏了
    }
    ~Leaky() {
        delete[] data;    // 永远不会执行！
        delete[] backup;
    }
};

int main() {
    try {
        Leaky l;
    } catch (const std::exception& e) {
        std::cout << "捕获: " << e.what() << std::endl;
    }
    return 0;
}
```

**输出：**
```
捕获: 构造中出错
```

### 2.2 使用RAII保证异常安全

```cpp
#include <iostream>
#include <memory>
#include <stdexcept>

class Safe {
    std::unique_ptr<int[]> data;
    std::unique_ptr<int[]> backup;
public:
    Safe() : data(std::make_unique<int[]>(100)),
             backup(std::make_unique<int[]>(50)) {
        throw std::runtime_error("构造中出错");
        // 异常时已构造的unique_ptr成员会自动释放
    }
    // 无需手动析构，unique_ptr自动管理
};

int main() {
    try {
        Safe s;
    } catch (const std::exception& e) {
        std::cout << "捕获: " << e.what() << std::endl;
        std::cout << "资源已自动释放" << std::endl;
    }
    return 0;
}
```

**输出：**
```
捕获: 构造中出错
资源已自动释放
```

### 2.3 函数try块

```cpp
#include <iostream>
#include <stdexcept>

class Base {
public:
    Base(int v) {
        if (v < 0) throw std::invalid_argument("负数");
    }
};

class Derived : public Base {
    int* ptr;
public:
    // 函数try块：捕获初始化列表和构造函数体的异常
    Derived(int v) try
        : Base(v), ptr(new int[v]) {
    } catch (...) {
        std::cout << "Derived构造异常被捕获" << std::endl;
        throw;  // 重新抛出
    }
};
```

## 三、注意事项与常见陷阱

- 构造函数抛异常时析构函数**不会**被调用
- 使用成员初始化列表和RAII包装类可保证异常安全
- 函数try块（function try block）可用于捕获基类/成员初始化的异常
- 函数try块的catch中如果不重新抛出，编译器会自动重新抛出
- 永远不要在构造函数中让资源处于需要析构函数清理的半初始化状态
