# 序列容器对比

## 一、概念说明

序列容器（vector、deque、list）按元素插入顺序存储，但底层数据结构不同，导致操作复杂度差异巨大。正确选择容器对性能至关重要。

## 二、性能对比

### 2.1 操作复杂度对比

```cpp
/*
| 操作              | vector      | deque       | list        |
|-------------------|-------------|-------------|-------------|
| 随机访问          | O(1)        | O(1)        | O(n)        |
| 尾部插入          | 均摊O(1)    | O(1)        | O(1)        |
| 头部插入          | O(n)        | O(1)        | O(1)        |
| 中间插入          | O(n)        | O(n)        | O(1)*       |
| 尾部删除          | O(1)        | O(1)        | O(1)        |
| 头部删除          | O(n)        | O(1)        | O(1)        |
| 中间删除          | O(n)        | O(n)        | O(1)*       |
| 内存布局          | 连续        | 分段连续    | 非连续      |
| 缓存友好度        | 高          | 中          | 低          |

* 需要已定位到该位置的迭代器
*/
```

### 2.2 代码示例

```cpp
#include <vector>
#include <deque>
#include <list>
#include <iostream>
#include <chrono>

int main() {
    const int N = 100000;

    // vector：随机访问快
    std::vector<int> vec;
    for (int i = 0; i < N; ++i) vec.push_back(i);
    int sum = 0;
    for (int i = 0; i < N; ++i) sum += vec[i];  // O(1)访问
    std::cout << "vector随机访问: " << sum << std::endl;

    // deque：两端操作快
    std::deque<int> deq;
    for (int i = 0; i < N; ++i) {
        deq.push_front(i);  // O(1)头部插入
        deq.push_back(i);   // O(1)尾部插入
    }

    // list：中间插入删除快（如果已有迭代器）
    std::list<int> lst;
    for (int i = 0; i < N; ++i) lst.push_back(i);
    auto it = lst.begin();
    std::advance(it, N / 2);
    lst.insert(it, 999);  // O(1)在已知位置插入
}
```

### 2.3 选择策略

```cpp
// 默认选择vector
std::vector<int> default_choice;

// 需要频繁头部操作
std::deque<int> needs_front_ops;
needs_front_ops.push_front(1);
needs_front_ops.pop_front();

// 需要频繁任意位置插入删除
std::list<int> needs_middle_ops;
auto it = needs_middle_ops.begin();
needs_middle_ops.insert(it, 42);  // O(1)
needs_middle_ops.erase(it);       // O(1)

// 需要稳定引用/指针
std::list<int> needs_stable_refs;
// list的插入删除不影响其他元素的引用
```

## 三、选择决策树

```cpp
/*
1. 需要随机访问？
   - 是 → vector（或deque如果需要头部操作）
   - 否 → 继续

2. 频繁头部操作？
   - 是 → deque
   - 否 → 继续

3. 频繁中间插入删除？
   - 是 → list
   - 否 → vector

4. 需要元素引用稳定？
   - 是 → list
   - 否 → vector
*/
```

## 四、注意事项与常见陷阱

- vector的缓存友好性通常使其即使在理论复杂度不占优时也更快
- list的指针开销（每个元素额外两个指针）在小对象时显著
- deque的分段连续内存不如vector缓存友好
- 不要过早优化：先用vector，分析后再考虑切换
