# 自定义哈希函数

## 一、概念说明

当自定义类型作为`unordered`容器的键时，需要提供哈希函数（`std::hash`特化）和相等比较（`operator==`或自定义谓词）。哈希函数应尽量均匀分布以减少冲突。好的哈希函数是无序容器性能的关键。

### 1.1 要求

| 要求 | 说明 |
|------|------|
| `operator==` | 必须提供，用于解决哈希冲突 |
| `std::hash`特化 | 或自定义哈希函数对象 |
| 一致性 | 相等的对象必须有相同的哈希值 |
| 分布均匀 | 减少冲突，避免性能退化 |

## 二、具体用法

### 2.1 特化std::hash

```cpp
#include <unordered_map>
#include <iostream>
#include <string>

struct Point {
    int x, y;
    bool operator==(const Point& other) const {
        return x == other.x && y == other.y;
    }
};

// 特化std::hash（放在std命名空间）
namespace std {
    template <>
    struct hash<Point> {
        size_t operator()(const Point& p) const {
            // 组合哈希：简单XOR（可能有冲突）
            return std::hash<int>{}(p.x) ^ (std::hash<int>{}(p.y) << 1);
        }
    };
}

int main() {
    std::unordered_map<Point, std::string> point_names;
    point_names[{1, 2}] = "A";
    point_names[{3, 4}] = "B";

    std::cout << point_names[{1, 2}] << std::endl;  // A
}
```

### 2.2 自定义哈希函数对象

```cpp
struct Person {
    std::string name;
    int age;
};

// 不修改std命名空间，使用自定义哈希对象
struct PersonHash {
    size_t operator()(const Person& p) const {
        size_t h1 = std::hash<std::string>{}(p.name);
        size_t h2 = std::hash<int>{}(p.age);
        return h1 ^ (h2 << 1);
    }
};

struct PersonEqual {
    bool operator()(const Person& a, const Person& b) const {
        return a.name == b.name && a.age == b.age;
    }
};

void demo() {
    std::unordered_map<Person, int, PersonHash, PersonEqual> ages;
    ages[{"Alice", 25}] = 100;
    ages[{"Bob", 30}] = 200;

    std::cout << ages[{"Alice", 25}] << std::endl;  // 100
}
```

### 2.3 boost::hash_combine风格（推荐）

```cpp
// 业界标准的组合哈希方法
template <typename T>
void hash_combine(size_t& seed, const T& val) {
    seed ^= std::hash<T>{}(val) + 0x9e3779b9 + (seed << 6) + (seed >> 2);
}

struct Triple {
    int a, b, c;
    bool operator==(const Triple& other) const {
        return a == other.a && b == other.b && c == other.c;
    }
};

namespace std {
    template <>
    struct hash<Triple> {
        size_t operator()(const Triple& t) const {
            size_t seed = 0;
            hash_combine(seed, t.a);
            hash_combine(seed, t.b);
            hash_combine(seed, t.c);
            return seed;
        }
    };
}

// 为什么hash_combine比简单XOR好？
// 1. XOR对顺序不敏感：hash(a,b) == hash(b,a)
// 2. XOR容易产生冲突：a ^ a == 0
// 3. hash_combine使用黄金比例常数，分布更均匀
```

### 2.4 使用std::tuple简化

```cpp
#include <tuple>

struct Person2 {
    std::string name;
    int age;
    bool operator==(const Person2& other) const {
        return std::tie(name, age) == std::tie(other.name, other.age);
    }
};

namespace std {
    template<>
    struct hash<Person2> {
        size_t operator()(const Person2& p) const {
            // 利用tuple的哈希
            return std::hash<std::string>{}(p.name) ^
                   (std::hash<int>{}(p.age) << 1);
        }
    };
}
```

## 三、哈希质量测试

```cpp
#include <unordered_map>
#include <vector>

void test_hash_quality() {
    // 测试哈希分布均匀性
    struct BadHash {
        size_t operator()(const Point& p) const {
            return p.x;  // 只用x，忽略y → 大量冲突
        }
    };

    struct GoodHash {
        size_t operator()(const Point& p) const {
            size_t seed = 0;
            hash_combine(seed, p.x);
            hash_combine(seed, p.y);
            return seed;
        }
    };

    // 插入大量数据测试桶分布
    std::unordered_map<Point, int, GoodHash> good_map;
    for (int i = 0; i < 10000; ++i)
        good_map[{i, i * 2}] = i;

    std::cout << "good bucket_count: " << good_map.bucket_count() << std::endl;
    std::cout << "good load_factor: " << good_map.load_factor() << std::endl;
}
```

## 四、注意事项与常见陷阱

1. **哈希函数必须与`operator==`一致**：相等的对象必须有相同的哈希值
2. **哈希冲突影响性能**：应尽量均匀分布，避免所有元素落入同一桶
3. **避免简单XOR组合**：对顺序敏感的数据会产生大量冲突
4. **`boost::hash_combine`是业界标准**：黄金比例常数保证良好分布
5. **哈希函数对象必须是const的**：`operator()`应标记为const
6. **特化`std::hash`需谨慎**：必须保证满足哈希要求（一致性、分布性）
