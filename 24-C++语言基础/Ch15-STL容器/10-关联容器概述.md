# 关联容器概述

## 一、概念说明

关联容器（C++标准 §23.4）通过键（key）组织数据，提供高效的查找操作。分为**有序关联容器**（基于红黑树，O(log n)）和**无序关联容器**（基于哈希表，C++11，平均O(1)）。有序容器保证元素按序排列，支持范围查询；无序容器提供更快的查找但不保证顺序。

### 1.1 两大类对比

| 特性 | 有序关联容器 | 无序关联容器 |
|------|------------|------------|
| 底层结构 | 红黑树 | 哈希表 |
| 查找复杂度 | O(log n) | 平均O(1)，最坏O(n) |
| 元素顺序 | 按键排序 | 无序 |
| 范围查询 | 支持 | 不支持 |
| 键要求 | 可比较（operator<） | 可哈希（std::hash） |
| 引入版本 | C++98 | C++11 |

## 二、容器分类

### 2.1 有序关联容器

```cpp
#include <map>
#include <set>

// map：键值对，键唯一
std::map<std::string, int> scores = {{"Alice", 90}, {"Bob", 85}};

// multimap：键值对，键可重复
std::multimap<std::string, int> grades;
grades.insert({"Alice", 90});
grades.insert({"Alice", 85});  // Alice有两个成绩

// set：只有键，键唯一
std::set<int> unique_nums = {1, 2, 3, 4, 5};

// multiset：只有键，键可重复
std::multiset<int> bag = {1, 1, 2, 2, 3};
```

### 2.2 无序关联容器（C++11）

```cpp
#include <unordered_map>
#include <unordered_set>

// 基于哈希表
std::unordered_map<std::string, int> hash_map = {{"key", 42}};
std::unordered_set<int> hash_set = {1, 2, 3, 4, 5};

// 允许重复
std::unordered_multimap<std::string, int> hash_mmap;
std::unordered_multiset<int> hash_mset;
```

### 2.3 基本操作对比

```cpp
#include <map>
#include <unordered_map>
#include <iostream>

void operations_demo() {
    std::map<std::string, int> ordered;
    std::unordered_map<std::string, int> unordered;

    // 插入（两种容器接口相同）
    ordered["alice"] = 90;
    ordered.insert({"bob", 85});
    ordered.emplace("charlie", 95);

    unordered["alice"] = 90;
    unordered.insert({"bob", 85});

    // 查找
    auto it = ordered.find("alice");
    if (it != ordered.end())
        std::cout << it->first << ": " << it->second << std::endl;

    // 检查存在
    if (ordered.count("alice") > 0) { /* 存在 */ }
    if (ordered.contains("alice")) { /* C++20 */ }

    // 删除
    ordered.erase("charlie");

    // 有序容器特有：范围查询
    auto lo = ordered.lower_bound("alice");  // >= "alice"
    auto hi = ordered.upper_bound("bob");    // > "bob"

    // 无序容器特有：桶信息
    std::cout << "buckets: " << unordered.bucket_count() << std::endl;
    std::cout << "load_factor: " << unordered.load_factor() << std::endl;
}
```

### 2.4 选择指南

```cpp
/*
| 需求                     | 选择                              |
|------------------------|----------------------------------|
| 需要有序遍历            | map / set                        |
| 需要O(log n)稳定查找    | map / set                        |
| 需要最快查找            | unordered_map / unordered_set    |
| 允许重复键              | multimap / multiset              |
| 需要键值对              | map / unordered_map              |
| 只需要键                | set / unordered_set              |
| 需要范围查询            | map / set（有序容器）              |
| 需要确定性性能          | map / set（O(log n)最坏保证）      |
*/
```

### 2.5 自定义类型作为键

```cpp
// 有序容器：需要operator<或自定义比较器
struct Person {
    std::string name;
    int age;
    bool operator<(const Person& other) const {
        return std::tie(name, age) < std::tie(other.name, other.age);
    }
};
std::map<Person, int> person_map;  // OK

// 无序容器：需要std::hash特化和operator==
struct PersonHash {
    size_t operator()(const Person& p) const {
        return std::hash<std::string>{}(p.name) ^
               (std::hash<int>{}(p.age) << 1);
    }
};
struct PersonEqual {
    bool operator()(const Person& a, const Person& b) const {
        return a.name == b.name && a.age == b.age;
    }
};
std::unordered_map<Person, int, PersonHash, PersonEqual> hash_person_map;
```

## 三、注意事项与常见陷阱

1. **有序容器的键类型必须支持`operator<`**：或提供自定义比较器
2. **无序容器的键类型必须可哈希**：需要`std::hash`特化和`operator==`
3. **`map::operator[]`在键不存在时会插入默认值**：可能意外修改map
4. **无序容器的遍历顺序不确定**：不要依赖插入顺序
5. **有序容器的迭代器在插入删除后一般仍有效**：被删元素除外
6. **无序容器rehash时所有迭代器失效**
7. **自定义类型作为键需要满足相应要求**：比较或哈希+相等
