# 迭代器失效

## 一、概念说明

容器的插入、删除操作可能导致迭代器、指针、引用失效。不同容器的失效规则不同，理解这些规则是编写安全代码的关键。

## 二、各容器失效规则

```cpp
/*
| 容器          | 插入失效                  | 删除失效                    |
|--------------|--------------------------|---------------------------|
| vector       | 扩容时全部失效            | 被删位置及之后失效          |
| deque        | 头尾操作可能全部失效      | 头尾操作全部失效            |
| list         | 不失效                   | 被删元素失效                |
| forward_list | 不失效                   | 被删元素失效                |
| set/map      | 不失效                   | 被删元素失效                |
| unordered_*  | 可能全部失效(rehash)     | 被删元素失效                |
*/
```

### 2.1 vector失效示例

```cpp
#include <vector>
#include <iostream>

int main() {
    std::vector<int> v = {1, 2, 3, 4, 5};
    auto it = v.begin() + 2;

    // push_back可能使it失效（如果扩容）
    v.push_back(6);
    // *it;  // 未定义行为！

    // 安全做法：使用索引或重新获取迭代器
    size_t index = 2;
    v.push_back(6);
    it = v.begin() + index;  // 重新获取
}
```

### 2.2 安全删除

```cpp
void safe_erase() {
    std::vector<int> v = {1, 2, 3, 4, 5};

    // 错误：erase使迭代器失效
    // for (auto it = v.begin(); it != v.end(); ++it) {
    //     if (*it % 2 == 0) v.erase(it);  // it已失效！
    // }

    // 正确：erase返回新迭代器
    for (auto it = v.begin(); it != v.end(); ) {
        if (*it % 2 == 0) it = v.erase(it);
        else ++it;
    }

    // C++20: erase_if
    std::erase_if(v, [](int x) { return x % 2 == 0; });
}
```

## 三、注意事项

- vector扩容时所有迭代器失效
- unordered容器rehash时所有迭代器失效
- list/set/map插入不失效（被删元素除外）
- 使用erase返回值更新迭代器
- 在循环中修改容器时特别小心
