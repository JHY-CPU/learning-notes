# 迭代器失效

## 一、概念说明

容器的插入、删除操作可能导致迭代器、指针、引用失效（C++标准 §23.2.1）。不同容器的失效规则不同，理解这些规则是编写安全代码的关键。使用已失效的迭代器是**未定义行为**，可能导致崩溃或数据损坏。

### 1.1 失效规则总表

```cpp
/*
| 容器          | 插入失效                  | 删除失效                    |
|--------------|--------------------------|---------------------------|
| vector       | 扩容时全部失效            | 被删位置及之后失效          |
| deque        | 头尾操作可能全部失效      | 头尾操作全部失效，中间全部   |
| list         | 不失效                   | 被删元素失效                |
| forward_list | 不失效                   | 被删元素的后继失效          |
| set/map      | 不失效                   | 被删元素失效                |
| unordered_*  | rehash时全部失效         | 被删元素失效                |
*/
```

## 二、具体用法

### 2.1 vector失效示例

```cpp
#include <vector>
#include <iostream>

int main() {
    std::vector<int> v = {1, 2, 3, 4, 5};

    // 场景1：push_back可能导致扩容
    auto it = v.begin() + 2;
    v.push_back(6);  // 如果扩容，it失效
    // *it;  // 未定义行为！

    // 安全做法：使用索引或重新获取迭代器
    size_t index = 2;
    v.push_back(6);
    it = v.begin() + index;  // 重新获取

    // 场景2：insert使被操作位置及之后失效
    it = v.begin() + 1;
    v.insert(v.begin() + 3, 99);  // it仍然有效（在插入点之前）
    // 但it之后的迭代器全部失效
}
```

### 2.2 安全删除

```cpp
void safe_erase() {
    std::vector<int> v = {1, 2, 3, 4, 5};

    // 错误：erase使迭代器失效
    // for (auto it = v.begin(); it != v.end(); ++it) {
    //     if (*it % 2 == 0) v.erase(it);  // it已失效！
    // }

    // 正确1：erase返回新迭代器
    for (auto it = v.begin(); it != v.end(); ) {
        if (*it % 2 == 0) it = v.erase(it);
        else ++it;
    }

    // 正确2：remove-erase惯用法
    v = {1, 2, 3, 4, 5};
    v.erase(std::remove_if(v.begin(), v.end(),
        [](int x) { return x % 2 == 0; }), v.end());

    // 正确3：C++20 erase_if
    std::erase_if(v, [](int x) { return x % 2 == 0; });
}
```

### 2.3 deque失效规则

```cpp
#include <deque>

void deque_invalidation() {
    std::deque<int> d = {1, 2, 3, 4, 5};

    // 注意：deque的失效规则比vector更严格
    // push_front和push_back都可能使所有迭代器失效
    auto it = d.begin() + 2;
    d.push_back(6);   // 可能使所有迭代器失效
    d.push_front(0);  // 可能使所有迭代器失效
    // *it;  // 未定义行为！

    // 中间insert/erase使所有迭代器失效
    it = d.begin() + 2;
    d.insert(d.begin() + 1, 99);  // it失效
    d.erase(d.begin() + 1);       // 所有迭代器失效
}
```

### 2.4 list/set/map的安全性

```cpp
#include <list>
#include <set>

void stable_containers() {
    std::list<int> lst = {1, 2, 3, 4, 5};

    // list的插入不失效
    auto it = lst.begin();
    std::advance(it, 2);
    lst.push_back(6);   // it仍然有效
    lst.insert(it, 99); // it仍然有效

    // list的删除：只有被删元素失效
    auto del = lst.begin();
    lst.erase(del);     // del失效
    // 但其他迭代器仍然有效

    // set/map同理
    std::set<int> s = {1, 2, 3};
    auto sit = s.find(2);
    s.insert(4);        // sit仍然有效
    s.erase(sit);       // sit失效
}
```

### 2.5 unordered容器失效

```cpp
#include <unordered_map>

void unordered_invalidation() {
    std::unordered_map<int, int> um;
    for (int i = 0; i < 100; ++i) um[i] = i;

    // rehash时所有迭代器失效
    um.reserve(1000);  // 可能触发rehash
    // 所有之前的迭代器失效

    // 插入也可能触发rehash
    auto it = um.find(42);
    um[200] = 200;      // 可能触发rehash，it可能失效

    // 安全做法：先reserve足够空间
    um.clear();
    um.reserve(201);
    for (int i = 0; i < 201; ++i) um[i] = i;
    // 现在迭代器安全
}
```

## 三、注意事项与常见陷阱

1. **vector扩容时所有迭代器失效**：使用`reserve`避免
2. **unordered容器rehash时所有迭代器失效**：使用`reserve`避免
3. **list/set/map插入不失效**：被删元素除外
4. **使用`erase`返回值更新迭代器**：`it = vec.erase(it);`
5. **在循环中修改容器时特别小心**：考虑使用索引或反向迭代器
6. **C++20的`erase_if`是最安全的删除方式**
7. **deque的失效规则比vector更严格**：两端操作也使迭代器失效
