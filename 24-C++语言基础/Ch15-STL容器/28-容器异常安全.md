# 容器异常安全

## 一、概念说明

STL容器在不同操作中有不同的异常安全保证（C++标准 §23.2.1）。了解这些保证有助于编写异常安全的代码。异常安全分为三个级别：**基本保证**（不泄漏）、**强保证**（成功或不变）、**不抛异常保证**（noexcept）。

### 1.1 异常安全级别

| 级别 | 说明 | 适用场景 |
|------|------|---------|
| nothrow/noexcept | 保证不抛异常 | 析构、swap、移动 |
| 强保证 | 成功或回到原状态 | push_back、insert |
| 基本保证 | 不泄漏资源，对象有效 | sort、operator[] |

## 二、常见操作的异常保证

```cpp
/*
| 容器操作              | 异常保证         | 说明                   |
|----------------------|-----------------|-----------------------|
| vector::push_back    | 强保证          | 扩容失败则不变         |
| vector::emplace_back | 强保证          | 同上                  |
| vector::insert       | 强保证          | 同上                  |
| vector::erase        | noexcept        | 不抛异常               |
| vector::clear        | noexcept        | 不抛异常               |
| list::push_back      | 强保证          | 链表插入               |
| map::insert          | 强保证          | 树插入                 |
| map::operator[]      | 基本保证        | 可能插入默认值         |
| sort                 | 基本保证        | 比较函数可能抛异常      |
| swap                 | noexcept        | O(1)交换               |
*/
```

## 三、具体用法

### 3.1 emplace的异常安全性

```cpp
#include <vector>
#include <iostream>

class Widget {
    int value;
public:
    Widget(int v) : value(v) {
        if (v < 0) throw std::invalid_argument("负值");
    }
};

int main() {
    std::vector<Widget> vec;
    vec.emplace_back(1);
    vec.emplace_back(2);

    try {
        vec.emplace_back(-3);  // 构造时抛异常
    } catch (...) {
        // 强保证：vec保持{1, 2}不变
        std::cout << "大小: " << vec.size() << std::endl;  // 2
    }
}
```

### 3.2 拷贝 vs 移动异常安全

```cpp
// 移动构造的noexcept影响扩容策略
struct Safe {
    int* data;
    Safe(Safe&& other) noexcept : data(other.data) {
        other.data = nullptr;  // 安全：noexcept
    }
};

struct Unsafe {
    int* data;
    Unsafe(Unsafe&& other) : data(other.data) {
        // 可能抛异常（如日志记录失败）
        other.data = nullptr;
    }
};

void demo() {
    std::vector<Safe> v1;     // 扩容时使用移动（快）
    std::vector<Unsafe> v2;   // 扩容时使用拷贝（慢但安全）

    // 规则：移动构造应标记noexcept
    // 否则扩容时使用拷贝保证强异常安全
}
```

### 3.3 强异常安全的实现

```cpp
#include <vector>
#include <algorithm>

// 自定义操作的强异常安全
template<typename T>
void safe_insert(std::vector<T>& vec, size_t pos, const T& value) {
    // 1. 先拷贝一份
    std::vector<T> backup = vec;

    try {
        // 2. 执行可能失败的操作
        vec.insert(vec.begin() + pos, value);
    } catch (...) {
        // 3. 回滚
        vec = std::move(backup);
        throw;
    }
}

// 更好的方案：利用push_back的强保证
template<typename T>
void safe_push(std::vector<T>& vec, const T& value) {
    // push_back本身提供强保证
    vec.push_back(value);  // 成功或不变
}
```

### 3.4 自定义类型的异常安全

```cpp
class MyContainer {
    std::vector<int> data;
public:
    // 移动构造：noexcept
    MyContainer(MyContainer&& other) noexcept = default;

    // 移动赋值：noexcept
    MyContainer& operator=(MyContainer&& other) noexcept = default;

    // 拷贝构造：强保证
    MyContainer(const MyContainer& other) = default;

    // 自定义插入：利用vector的强保证
    void insert(int value) {
        data.push_back(value);  // 强保证
    }

    // 清空：noexcept
    void clear() noexcept { data.clear(); }

    // 交换：noexcept
    void swap(MyContainer& other) noexcept { data.swap(other.data); }
};
```

## 四、最佳实践

```cpp
/*
1. 移动操作标记noexcept
   - 影响vector扩容策略
   - 是STL算法的要求（如std::move_if_noexcept）

2. 利用容器自身的强保证
   - push_back/emplace_back提供强保证
   - 基于此构建更复杂操作

3. swap是noexcept
   - 用于实现强异常安全（copy-and-swap惯用法）

4. 析构函数不抛异常
   - STL容器析构是noexcept的
   - 自定义类型析构也应noexcept

5. 不要在析构中抛异常
   - 导致std::terminate
*/
```

## 五、注意事项与常见陷阱

1. **`push_back`提供强异常安全**：成功或不变，但扩容时的拷贝/移动可能抛异常
2. **移动操作应标记noexcept**：否则扩容时使用拷贝，性能下降
3. **自定义allocator也需要保证异常安全**
4. **排序算法只提供基本保证**：比较函数抛异常时容器仍有效但顺序不确定
5. **C++11后swap是noexcept**：可用于实现强异常安全
6. **不要在容器析构时抛异常**：导致程序终止
