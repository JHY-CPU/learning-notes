# 容器异常安全

## 一、概念说明

STL容器在不同操作中有不同的异常安全保证。了解这些保证有助于编写异常安全的代码。

## 二、常见操作的异常保证

```cpp
/*
| 容器操作              | 异常保证         |
|----------------------|-----------------|
| vector::push_back    | 强保证          |
| vector::emplace_back | 强保证          |
| vector::insert       | 强保证          |
| vector::erase        | 不抛异常(noexcept) |
| vector::clear        | 不抛异常        |
| list::push_back      | 强保证          |
| map::insert          | 强保证          |
| map::operator[]      | 基本保证        |
| sort                 | 基本保证        |
*/
```

### 2.1 emplace的安全性

```cpp
#include <vector>
#include <iostream>

class Widget {
    int value;
public:
    Widget(int v) : value(v) {
        if (v < 0) throw std::invalid_argument("负值");
    }
};

int main() {
    std::vector<Widget> vec;
    vec.emplace_back(1);
    vec.emplace_back(2);

    try {
        vec.emplace_back(-3);  // 抛异常
    } catch (...) {
        // vec保持{1, 2}不变（强保证）
        std::cout << "大小: " << vec.size() << std::endl;  // 2
    }
}
```

### 2.2 拷贝 vs 移动异常安全

```cpp
// 如果移动构造是noexcept，扩容时使用移动
// 否则使用拷贝（保证强安全）

struct Safe {
    Safe(Safe&&) noexcept { /* 安全 */ }
};

struct Unsafe {
    Unsafe(Unsafe&&) { /* 可能抛异常 */ }
};

void demo() {
    std::vector<Safe> v1;   // 扩容时移动
    std::vector<Unsafe> v2; // 扩容时拷贝（更慢但安全）
}
```

## 三、注意事项

- `push_back`提供强异常安全（成功或不变）
- 移动操作应标记noexcept以获得最佳性能
- 自定义allocator也需要保证异常安全
- 排序算法只提供基本保证
