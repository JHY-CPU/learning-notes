# 容器内存管理

## 一、概念说明

STL容器通过分配器（Allocator，C++标准 §23.10）管理内存。默认使用`std::allocator`，也可以自定义分配器实现内存池、统计等需求。C++17引入的`std::pmr`（多态内存资源）提供了更灵活的运行时内存管理方案。

### 1.1 分配器的作用

```
分配器抽象了内存分配策略：
- 默认allocator：使用::operator new/delete
- 内存池allocator：预分配大块，减少系统调用
- 统计allocator：记录分配次数和大小
- 共享内存allocator：跨进程内存
- 栈allocator：pmr::monotonic_buffer_resource
```

## 二、具体用法

### 2.1 默认分配器

```cpp
#include <vector>
#include <memory>
#include <iostream>

int main() {
    // 默认使用std::allocator
    std::vector<int> v1 = {1, 2, 3};
    std::vector<int, std::allocator<int>> v2 = {1, 2, 3};  // 显式指定

    // 内存管理接口
    v1.reserve(100);       // 预分配
    v1.shrink_to_fit();    // 释放多余
}
```

### 2.2 自定义分配器：计数

```cpp
#include <vector>
#include <cstddef>
#include <iostream>

template <typename T>
struct CountingAllocator {
    using value_type = T;
    static size_t alloc_count;
    static size_t total_bytes;

    T* allocate(size_t n) {
        alloc_count += n;
        total_bytes += n * sizeof(T);
        return static_cast<T*>(::operator new(n * sizeof(T)));
    }

    void deallocate(T* p, size_t n) noexcept {
        alloc_count -= n;
        total_bytes -= n * sizeof(T);
        ::operator delete(p);
    }

    template <typename U>
    CountingAllocator(const CountingAllocator<U>&) noexcept {}
    CountingAllocator() = default;

    // 分配器相等性（同类型视为相等）
    bool operator==(const CountingAllocator&) const { return true; }
    bool operator!=(const CountingAllocator&) const { return false; }
};

template <typename T>
size_t CountingAllocator<T>::alloc_count = 0;
template <typename T>
size_t CountingAllocator<T>::total_bytes = 0;

void counting_demo() {
    std::vector<int, CountingAllocator<int>> v;
    for (int i = 0; i < 1000; ++i) v.push_back(i);

    std::cout << "分配元素数: " << CountingAllocator<int>::alloc_count << std::endl;
    std::cout << "分配字节数: " << CountingAllocator<int>::total_bytes << std::endl;
}
```

### 2.3 pmr分配器（C++17）

```cpp
#include <vector>
#include <memory_resource>
#include <iostream>

void pmr_demo() {
    // 方案1：单调缓冲区（避免系统调用）
    char buffer[1024];
    std::pmr::monotonic_buffer_resource pool{
        std::data(buffer), std::size(buffer)
    };

    std::pmr::vector<int> v{&pool};
    for (int i = 0; i < 100; ++i) v.push_back(i);
    // 内存从buffer分配，超大时回退到new

    // 方案2：同步池（线程安全）
    std::pmr::synchronized_pool_resource sync_pool;
    std::pmr::vector<int> v2{&sync_pool};

    // 方案3：非同步池（更快，不线程安全）
    std::pmr::unsynchronized_pool_resource unsync_pool;
    std::pmr::vector<int> v3{&unsync_pool};

    // 全局pmr
    std::pmr::vector<int> v4;  // 使用默认资源（new/delete）
}
```

### 2.4 内存池分配器

```cpp
#include <vector>
#include <memory>

// 简化的内存池分配器
template <typename T>
struct PoolAllocator {
    using value_type = T;

    struct Pool {
        std::vector<T*> blocks;
        size_t current = 0;
        T* buffer = nullptr;
        static constexpr size_t BLOCK_SIZE = 1024;

        Pool() : buffer(static_cast<T*>(::operator new(BLOCK_SIZE * sizeof(T)))) {}
        ~Pool() { ::operator delete(buffer); for (auto* b : blocks) ::operator delete(b); }

        T* allocate() {
            if (current < BLOCK_SIZE) return &buffer[current++];
            // 扩展
            blocks.push_back(buffer);
            buffer = static_cast<T*>(::operator new(BLOCK_SIZE * sizeof(T)));
            current = 0;
            return &buffer[current++];
        }
    };

    static Pool& get_pool() { static Pool pool; return pool; }

    T* allocate(size_t n) {
        if (n == 1) return get_pool().allocate();
        return static_cast<T*>(::operator new(n * sizeof(T)));
    }
    void deallocate(T*, size_t) noexcept {}  // 池分配不单独释放

    template<typename U> PoolAllocator(const PoolAllocator<U>&) noexcept {}
    PoolAllocator() = default;
    bool operator==(const PoolAllocator&) const { return true; }
    bool operator!=(const PoolAllocator&) const { return false; }
};
```

## 三、注意事项与常见陷阱

1. **自定义分配器需要满足Allocator要求**：`allocate`、`deallocate`、`value_type`等
2. **pmr（C++17）提供了更灵活的运行时内存管理**：不需要模板参数不同
3. **内存池分配器可减少碎片和系统调用**：适用于频繁小对象分配
4. **分配器相等的容器才能互操作**：`operator==`必须正确定义
5. **`rebind`用于分配不同类型**：容器内部可能需要分配节点
6. **pmr的`monotonic_buffer_resource`适用于短期对象**：析构时一次性释放
