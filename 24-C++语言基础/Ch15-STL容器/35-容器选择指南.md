# 容器选择指南

## 一、决策树

```cpp
/*
选择容器的决策流程：

1. 需要键值对？
   ├─ 是 → 需要有序？
   │    ├─ 是 → map
   │    └─ 否 → unordered_map
   └─ 否 → 继续

2. 需要唯一元素？
   ├─ 是 → 需要有序？
   │    ├─ 是 → set
   │    └─ 否 → unordered_set
   └─ 否 → 继续（序列容器）

3. 需要随机访问？
   ├─ 是 → 需要头部操作？
   │    ├─ 是 → deque
   │    └─ 否 → vector（默认）
   └─ 否 → 需要频繁中间插入删除？
        ├─ 是 → list
        └─ 否 → vector

4. 特殊需求：
   - 固定大小 → array
   - LIFO → stack
   - FIFO → queue
   - 优先级 → priority_queue
   - 可选值 → optional
   - 多类型 → variant
   - 任意类型 → any
*/
```

## 二、常见场景推荐

```cpp
/*
| 场景                          | 推荐容器                |
|------------------------------|------------------------|
| 默认选择                      | vector                 |
| 字典/映射                     | unordered_map          |
| 有序字典                      | map                    |
| 去重                          | unordered_set          |
| 有序去重                      | set                    |
| 频繁头部操作                  | deque                  |
| 频繁任意位置插入删除           | list                   |
| 固定大小数组                  | array                  |
| 栈操作                        | stack                  |
| 队列操作                      | queue                  |
| 优先级处理                    | priority_queue         |
| 函数返回多个值                | pair / tuple           |
| 可选值                        | optional               |
| 字符串                        | string                 |
| 非拥有视图                    | string_view / span     |
*/
```

## 三、性能考虑

```cpp
/*
性能优化原则：
1. vector默认：缓存友好，大多数场景最快
2. 避免过早优化：先用vector，分析后再切换
3. reserve预分配：减少扩容次数
4. emplace替代push：避免临时对象
5. 哈希表vs红黑树：查找密集用unordered，遍历用ordered
6. 小数据用array：零开销
*/
```

## 四、注意事项

- 没有万能容器，选择取决于具体场景
- 先用默认容器（vector），性能分析后再优化
- 考虑异常安全、迭代器失效、内存布局
- 自定义类型作为键需要hash或比较器
- 不要过度工程化：简单场景用简单容器
