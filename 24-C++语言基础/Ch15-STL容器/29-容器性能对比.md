# 容器性能对比

## 一、概念说明

不同容器的操作复杂度差异巨大，选择合适的容器对程序性能至关重要。现代CPU的缓存架构使得**缓存友好性**往往比理论复杂度更重要——vector即使在O(n)操作中也可能比list的O(1)操作更快。

### 1.1 复杂度对比总表

```cpp
/*
| 容器              | 随机访问 | 插入(头) | 插入(尾) | 插入(中) | 查找    | 缓存友好 |
|-------------------|---------|---------|---------|---------|--------|---------|
| vector            | O(1)    | O(n)    | O(1)*   | O(n)    | O(n)   | 最佳     |
| deque             | O(1)    | O(1)    | O(1)    | O(n)    | O(n)   | 中等     |
| list              | O(n)    | O(1)    | O(1)    | O(1)**  | O(n)   | 差       |
| forward_list      | O(n)    | O(1)    | N/A     | O(1)**  | O(n)   | 差       |
| set/map           | N/A     | N/A     | N/A     | O(log n)| O(log n)| 差      |
| unordered_set/map | N/A     | N/A     | N/A     | O(1)*** | O(1)***| 中等     |
| array             | O(1)    | N/A     | N/A     | N/A     | O(n)   | 最佳     |

* 均摊  ** 需要已定位迭代器  *** 平均
*/
```

## 二、实际性能测试

### 2.1 顺序插入对比

```cpp
#include <vector>
#include <list>
#include <deque>
#include <chrono>
#include <iostream>

template<typename Container, typename Name>
void bench_insert(Name name, int N) {
    Container c;
    auto start = std::chrono::high_resolution_clock::now();

    for (int i = 0; i < N; ++i) {
        if constexpr (requires { c.push_back(i); })
            c.push_back(i);
        else
            c.push_front(i);
    }

    auto end = std::chrono::high_resolution_clock::now();
    auto ms = std::chrono::duration_cast<std::chrono::milliseconds>(end - start);
    std::cout << name << " 插入" << N << ": " << ms.count() << "ms" << std::endl;
}

int main() {
    const int N = 1000000;
    bench_insert<std::vector<int>>("vector", N);
    bench_insert<std::deque<int>>("deque", N);
    bench_insert<std::list<int>>("list", N);
}
```

### 2.2 随机访问对比

```cpp
void bench_access(int N) {
    std::vector<int> vec(N);
    std::list<int> lst(N);
    for (int i = 0; i < N; ++i) { vec[i] = i; lst.push_back(i); }

    // vector随机访问
    auto start = std::chrono::high_resolution_clock::now();
    volatile int sum = 0;
    for (int i = 0; i < N; ++i) sum += vec[i];
    auto end = std::chrono::high_resolution_clock::now();
    std::cout << "vector随机访问: "
              << std::chrono::duration_cast<std::chrono::microseconds>(end - start).count()
              << "us" << std::endl;

    // list遍历（理论O(n)访问）
    start = std::chrono::high_resolution_clock::now();
    sum = 0;
    for (auto it = lst.begin(); it != lst.end(); ++it) sum += *it;
    end = std::chrono::high_resolution_clock::now();
    std::cout << "list遍历: "
              << std::chrono::duration_cast<std::chrono::microseconds>(end - start).count()
              << "us" << std::endl;
    // vector通常快5-10倍（缓存预取）
}
```

### 2.3 查找对比

```cpp
#include <set>
#include <unordered_set>
#include <algorithm>

void bench_find(int N) {
    // 准备数据
    std::vector<int> vec(N);
    std::set<int> ordered;
    std::unordered_set<int> unordered;
    for (int i = 0; i < N; ++i) {
        vec[i] = i;
        ordered.insert(i);
        unordered.insert(i);
    }

    // vector线性查找
    auto start = std::chrono::high_resolution_clock::now();
    volatile int count = 0;
    for (int i = 0; i < 10000; ++i)
        if (std::find(vec.begin(), vec.end(), N - 1 - i) != vec.end()) ++count;
    auto end = std::chrono::high_resolution_clock::now();
    std::cout << "vector find: "
              << std::chrono::duration_cast<std::chrono::microseconds>(end - start).count()
              << "us" << std::endl;

    // set O(log n)
    start = std::chrono::high_resolution_clock::now();
    for (int i = 0; i < 10000; ++i)
        if (ordered.count(N - 1 - i)) ++count;
    end = std::chrono::high_resolution_clock::now();
    std::cout << "set count: "
              << std::chrono::duration_cast<std::chrono::microseconds>(end - start).count()
              << "us" << std::endl;

    // unordered_set O(1)
    start = std::chrono::high_resolution_clock::now();
    for (int i = 0; i < 10000; ++i)
        if (unordered.count(N - 1 - i)) ++count;
    end = std::chrono::high_resolution_clock::now();
    std::cout << "unordered_set count: "
              << std::chrono::duration_cast<std::chrono::microseconds>(end - start).count()
              << "us" << std::endl;
}
```

## 三、选择建议

```cpp
/*
性能优化原则：
1. vector默认：缓存友好，大多数场景最快
2. 避免过早优化：先用vector，分析后再切换
3. reserve预分配：减少扩容次数
4. emplace替代push：避免临时对象
5. 哈希表vs红黑树：查找密集用unordered，遍历用ordered
6. 小数据用array：零开销
7. 稳定引用用list：引用不随插入删除失效
*/
```

## 四、注意事项与常见陷阱

1. **缓存友好性胜过理论复杂度**：vector的O(n)查找可能比list的O(n)查找快10倍
2. **reserve避免扩容**：已知大小时预分配
3. **emplace替代push**：避免不必要的拷贝/移动
4. **小对象不要用list**：每个元素额外16字节指针开销
5. **unordered容器需要好的哈希函数**：冲突多时性能退化
6. **实际测试比理论分析更重要**：不同编译器、数据量结果不同
