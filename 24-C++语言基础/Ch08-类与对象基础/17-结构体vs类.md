# 结构体 vs 类

## 一、概念说明

`struct`和`class`在C++中几乎完全相同，唯一的区别是**默认访问权限**：`struct`默认`public`继承和访问，`class`默认`private`。

## 二、具体用法

### 2.1 默认访问区别

```cpp
// struct: 默认public
struct S {
    int x;  // public
    void foo() {}  // public
};

// class: 默认private
class C {
    int x;  // private
    void foo() {}  // private
};

S s;
s.x = 10;  // OK

C c;
// c.x = 10;  // 编译错误：private
```

### 2.2 默认继承区别

```cpp
struct BaseS { int x; };
class BaseC  { int x; };

struct DerivedS : BaseS { };  // 默认public继承
class DerivedC  : BaseC  { };  // 默认private继承

DerivedS ds;
ds.x = 10;  // OK: public继承

DerivedC dc;
// dc.x = 10;  // 编译错误: private继承
```

### 2.3 使用约定

```cpp
// struct: 纯数据聚合，无不变式
struct Point {
    double x, y;  // 所有成员public
};

// class: 有封装，维护不变式
class BankAccount {
    double balance;  // private，需要保护
public:
    void deposit(double amount);
    double getBalance() const;
};
```

### 2.4 可互换使用（但不推荐）

```cpp
// 技术上可以用struct定义类
struct MyClass {
private:
    int data;
public:
    MyClass(int d) : data(d) {}
};

// 但违反了使用约定，降低可读性
```

## 三、注意事项与常见陷阱

- 唯一技术区别是默认访问权限
- 使用约定：`struct`用于纯数据聚合，`class`用于有封装的类型
- 代码审查中struct有private成员可能被质疑
- `struct`和`class`在模板模板参数中不可互换（严格来说类型不同）
- 团队应统一约定使用规范
