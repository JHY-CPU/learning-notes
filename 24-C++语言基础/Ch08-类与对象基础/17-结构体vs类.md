# 结构体 vs 类

## 一、概念说明

`struct`和`class`在C++中几乎完全相同（C++11标准 §9），唯一的区别是**默认访问权限**：

| 特性 | struct | class |
|------|--------|-------|
| 默认成员访问 | public | private |
| 默认继承方式 | public | private |
| 支持模板 | 是 | 是 |
| 支持所有OOP特性 | 是 | 是 |

这是C++与C的关键区别：C的struct没有成员函数、访问控制、继承等。

## 二、具体用法

### 2.1 默认访问区别

```cpp
#include <iostream>
using namespace std;

// struct: 默认public
struct S {
    int x;          // public
    void foo() {}   // public
};

// class: 默认private
class C {
    int x;          // private
    void foo() {}   // private
public:
    C(int v) : x(v) {}
    int getX() const { return x; }
};

int main() {
    S s;
    s.x = 10;       // OK：public

    C c(42);
    // c.x = 10;     // 编译错误：private
    cout << c.getX() << endl;  // 42

    return 0;
}
```

### 2.2 默认继承区别

```cpp
#include <iostream>
using namespace std;

struct BaseS { int x = 1; };
class BaseC  { int x = 2; };

struct DerivedS : BaseS { };   // 默认public继承
class DerivedC  : BaseC  { };  // 默认private继承

// 也可以显式指定
class DerivedC2 : public BaseC { };  // 显式public继承

int main() {
    DerivedS ds;
    ds.x = 10;       // OK：public继承，x可访问

    DerivedC dc;
    // dc.x = 10;    // 编译错误：private继承

    return 0;
}
```

### 2.3 使用约定

```cpp
#include <iostream>
#include <string>
using namespace std;

// struct约定：纯数据聚合，无不变式
struct Point {
    double x, y;  // 所有成员public
};

// struct约定：POD类型、简单聚合
struct Color {
    uint8_t r, g, b, a;
};

// class约定：有封装，维护不变式
class BankAccount {
    double balance;        // private，需要保护
    string owner;
public:
    BankAccount(const string& o, double b)
        : owner(o), balance(max(0.0, b)) {}

    bool deposit(double amount) {
        if (amount <= 0) return false;  // 维护不变式
        balance += amount;
        return true;
    }

    double getBalance() const { return balance; }
};

int main() {
    // struct直接使用
    Point p{3.0, 4.0};
    cout << "(" << p.x << "," << p.y << ")" << endl;

    // class通过接口使用
    BankAccount acc("Alice", 1000);
    acc.deposit(500);
    cout << acc.getBalance() << endl;  // 1500

    return 0;
}
```

### 2.4 何时用struct vs class

```cpp
// 使用struct的场景：
// 1. 纯数据聚合
struct Config {
    int width;
    int height;
    bool fullscreen;
};

// 2. 函数对象/仿函数
struct Square {
    int operator()(int x) const { return x * x; }
};

// 3. 模板元编程中的trait
template <typename T>
struct is_pointer { static constexpr bool value = false; };

template <typename T>
struct is_pointer<T*> { static constexpr bool value = true; };

// 使用class的场景：
// 1. 有不变式需要维护
class Temperature {
    double celsius;
public:
    Temperature(double c) {
        if (c < -273.15) throw invalid_argument("低于绝对零度");
        celsius = c;
    }
};

// 2. 需要封装的复杂逻辑
// 3. 作为基类使用
```

## 三、注意事项与常见陷阱

1. **唯一技术区别是默认访问权限**：除此之外完全等价
2. **使用约定：struct用于纯数据聚合，class用于有封装的类型**：违反约定降低可读性
3. **代码审查中struct有private成员可能被质疑**：团队应统一约定
4. **struct和class在模板模板参数中不可互换**：`template<template<class> class T>` 和 `template<template<struct> class T>` 不同（虽然实际很少用）
5. **C++11起struct也可以使用类的所有特性**：构造函数、虚函数、继承等
6. **团队应统一使用规范**：避免混用造成困惑
