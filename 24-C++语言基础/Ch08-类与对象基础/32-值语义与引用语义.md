# 值语义与引用语义

## 一、概念说明

**值语义**（Value Semantics）表示对象的拷贝是独立副本，修改不影响原对象。**引用语义**（Reference Semantics）表示对象通过引用/指针共享，修改影响所有引用。C++默认支持值语义，引用语义通过指针和引用实现。

## 二、具体用法

### 2.1 值语义

```cpp
// 值语义：拷贝是独立的
std::vector<int> v1 = {1, 2, 3};
std::vector<int> v2 = v1;  // 拷贝

v2.push_back(4);
std::cout << v1.size() << std::endl;  // 输出: 3 (v1不受影响)
std::cout << v2.size() << std::endl;  // 输出: 4
```

### 2.2 引用语义

```cpp
// 引用语义：共享同一对象
auto shared = std::make_shared<std::vector<int>>(std::initializer_list<int>{1, 2, 3});
auto ref = shared;  // 共享所有权

ref->push_back(4);
std::cout << shared->size() << std::endl;  // 输出: 4 (共享修改)
std::cout << ref->size() << std::endl;     // 输出: 4
```

### 2.3 选择策略

```cpp
// 值语义适用场景
struct Point { double x, y; };  // 小型、不可变、频繁拷贝
struct Config { std::string name; int value; };  // 独立配置

// 引用语义适用场景
class Database {
    // 大型对象、需要共享、多态
};
std::shared_ptr<Database> db = std::make_shared<Database>();
```

### 2.4 值语义的设计优势

```cpp
// 值语义更可预测：无共享状态问题
class Session {
    std::string userId;
    std::vector<std::string> history;  // 值语义：独立副本
public:
    Session clone() const { return *this; }  // 完整拷贝
};

Session s1;
s1.history.push_back("action1");
Session s2 = s1.clone();
s2.history.push_back("action2");
std::cout << s1.history.size() << std::endl;  // 输出: 1 (不受s2影响)
```

### 2.5 多态中的值语义

```cpp
// std::variant提供值语义的多态
using Value = std::variant<int, double, std::string>;

std::vector<Value> values = {42, 3.14, "hello"};
for (const auto& v : values) {
    std::visit([](const auto& val) {
        std::cout << val << " ";
    }, v);
}
// 输出: 42 3.14 hello
```

## 三、注意事项与常见陷阱

- 值语义更易推理，引用语义更灵活
- 大对象频繁拷贝性能差，考虑移动语义或引用语义
- `std::any`和`std::variant`提供值语义的类型擦除
- 指针和引用本身是值语义，但提供对对象的引用语义访问
- 函数式编程偏好值语义（无副作用、易并行）
