# 聚合与组合

## 一、概念说明

**聚合**（Aggregation）和**组合**（Composition）都是"has-a"关系，表示一个类包含另一个类的对象作为成员。区别在于**生命周期管理**：组合中部分的生命周期与整体绑定，聚合中部分可独立存在。

## 二、具体用法

### 2.1 组合（Composition）

```cpp
class Engine {
    int horsepower;
public:
    Engine(int hp) : horsepower(hp) {
        std::cout << "引擎(" << hp << "HP) 构造\n";
    }
    ~Engine() { std::cout << "引擎析构\n"; }
    void start() { std::cout << "引擎启动\n"; }
};

class Car {
    Engine engine;  // 组合：Engine的生命周期与Car绑定
    std::string brand;
public:
    Car(const std::string& b, int hp) : engine(hp), brand(b) {}
    ~Car() { std::cout << brand << " 析构\n"; }

    void start() {
        std::cout << brand << " ";
        engine.start();
    }
};

Car car("Tesla", 500);
car.start();
// 输出:
// 引擎(500HP) 构造
// Tesla 引擎启动
// 离开作用域:
// Tesla 析构
// 引擎析构
```

### 2.2 聚合（Aggregation）

```cpp
class Student {
    std::string name;
public:
    Student(const std::string& n) : name(n) {}
    const std::string& getName() const { return name; }
};

class Classroom {
    std::vector<Student*> students;  // 聚合：Student独立于Classroom存在
public:
    void addStudent(Student* s) { students.push_back(s); }
    void list() const {
        for (const auto* s : students)
            std::cout << s->getName() << " ";
        std::cout << std::endl;
    }
};

Student alice("Alice"), bob("Bob");
Classroom room;
room.addStudent(&alice);
room.addStudent(&bob);
room.list();
// 输出: Alice Bob
// Student的生命周期独立于Classroom
```

### 2.3 设计选择

```cpp
// 组合：强拥有关系
class Window {
    std::string title;
    // Window拥有其MenuBar
    // MenuBar不可脱离Window存在
};

// 聚合：弱拥有关系
class University {
    std::vector<Professor*> professors;
    // Professor可在多所大学任职
    // Professor独立于University存在
};
```

## 三、注意事项与常见陷阱

- 组合中成员按声明顺序构造、逆序析构
- 聚合使用指针或引用，不拥有对象的生命周期
- 设计时优先考虑组合（更强的封装性）
- 聚合需要注意悬垂引用（部分对象已销毁）
- 继承是"is-a"关系，聚合/组合是"has-a"关系
