# 类大小与内存布局

## 一、概念说明

类的大小受数据成员类型、对齐要求、虚函数表指针（vptr）和填充字节（padding）影响。理解内存布局对性能优化和底层编程至关重要。

## 二、具体用法

### 2.1 基本大小计算

```cpp
class Simple {
    char a;     // 1字节
    int b;      // 4字节
    char c;     // 1字节
};

std::cout << sizeof(Simple) << std::endl;  // 输出: 12
// 布局: a(1) + padding(3) + b(4) + c(1) + padding(3) = 12
```

### 2.2 成员排列优化

```cpp
// 优化排列：减少填充
class Optimized {
    int b;      // 4字节
    char a;     // 1字节
    char c;     // 1字节 + padding(2)
};

std::cout << sizeof(Optimized) << std::endl;  // 输出: 8 (更紧凑)
```

### 2.3 虚函数表指针

```cpp
class NoVirtual {
    int x;
};
class WithVirtual {
    int x;
    virtual void foo() {}
};

std::cout << sizeof(NoVirtual) << std::endl;   // 输出: 4
std::cout << sizeof(WithVirtual) << std::endl;  // 输出: 16 (64位系统)
// vptr占8字节 + int占4字节 + 填充4字节 = 16
```

### 2.4 空类与空基类优化

```cpp
class Empty {};
class NotEmpty {
    Empty e;  // 空类成员占至少1字节
    int x;
};

// 空基类优化（EBO）
class Base {};
class Derived : public Base {
    int x;
};

std::cout << sizeof(Empty) << std::endl;    // 输出: 1
std::cout << sizeof(NotEmpty) << std::endl;  // 输出: 8 (4+1+填充3)
std::cout << sizeof(Derived) << std::endl;   // 输出: 4 (EBO：基类不占空间)
```

### 2.5 alignof查看对齐

```cpp
class Padded {
    char a;      // align 1
    double b;    // align 8
    int c;       // align 4
};

std::cout << sizeof(Padded) << std::endl;   // 输出: 24
std::cout << alignof(Padded) << std::endl;  // 输出: 8
// a(1) + pad(7) + b(8) + c(4) + pad(4) = 24
```

## 三、注意事项与常见陷阱

- 成员按声明顺序在内存中排列
- 填充（padding）是为了满足对齐要求
- 一个虚函数和十个虚函数的vptr大小相同（都指向一个vtable）
- 空基类优化（EBO）可节省空间
- `#pragma pack`可改变对齐，但影响性能和可移植性
