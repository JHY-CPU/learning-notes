# 类的静态多态

## 一、概念说明

**静态多态**（Static Polymorphism / Compile-time Polymorphism）在编译期确定调用哪个函数。最常用的技术是**CRTP**（Curiously Recurring Template Pattern，奇异递归模板模式），将派生类作为模板参数传给基类。

## 二、具体用法

### 2.1 CRTP基本模式

```cpp
template <typename Derived>
class Shape {
public:
    double area() const {
        // 编译期绑定：调用Derived的areaImpl
        return static_cast<const Derived*>(this)->areaImpl();
    }

    void describe() const {
        std::cout << "面积: " << area() << std::endl;
    }
};

class Circle : public Shape<Circle> {
    double radius;
public:
    Circle(double r) : radius(r) {}
    double areaImpl() const { return 3.14159 * radius * radius; }
};

class Rectangle : public Shape<Rectangle> {
    double w, h;
public:
    Rectangle(double w, double h) : w(w), h(h) {}
    double areaImpl() const { return w * h; }
};

Circle c(5);
Rectangle r(3, 4);
c.describe();  // 输出: 面积: 78.5398
r.describe();  // 输出: 面积: 12
```

### 2.2 CRTP实现接口

```cpp
template <typename Derived>
class Printable {
public:
    void print() const {
        std::cout << static_cast<const Derived*>(this)->toString() << std::endl;
    }
};

class Name : public Printable<Name> {
    std::string value;
public:
    Name(const std::string& v) : value(v) {}
    std::string toString() const { return "Name: " + value; }
};

Name n("Alice");
n.print();  // 输出: Name: Alice
```

### 2.3 静态多态 vs 动态多态

```cpp
// 静态多态：编译期绑定，无虚函数开销
// 缺点：不能统一存储（不能有 Shape* 指向不同派生类）
std::vector<Circle> circles;  // 只能存Circle

// 动态多态：运行时绑定，有虚函数开销
// 优点：可以统一存储
std::vector<std::unique_ptr<Shape2>> shapes;  // 可存任意派生类
```

### 2.4 CRTP禁用拷贝

```cpp
class NonCopyable {
public:
    NonCopyable() = default;
    NonCopyable(const NonCopyable&) = delete;
    NonCopyable& operator=(const NonCopyable&) = delete;
};

class MyClass : private NonCopyable {
    // MyClass不可拷贝
};
```

## 三、注意事项与常见陷阱

- CRTP中基类模板的每个实例化是不同类型
- 静态多态不能用于异构集合
- CRTP派生类的方法必须在基类模板实例化之前可见
- 静态多态在嵌入式或高性能场景更常见
- `if constexpr`（C++17）提供了另一种编译期多态方式
