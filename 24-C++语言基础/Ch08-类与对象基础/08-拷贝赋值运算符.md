# 拷贝赋值运算符

## 一、概念说明

拷贝赋值运算符（`operator=`）在**已有对象**被赋值为另一个对象时调用。它与拷贝构造的区别在于：拷贝构造是**创建新对象**，拷贝赋值是**替换已有对象**的内容。

## 二、具体用法

### 2.1 基本实现

```cpp
class MyString {
    char* data;
    size_t len;
public:
    MyString(const char* s = "") : len(strlen(s)) {
        data = new char[len + 1];
        memcpy(data, s, len + 1);
    }

    // 拷贝赋值运算符
    MyString& operator=(const MyString& other) {
        if (this != &other) {  // 自赋值检查
            delete[] data;  // 释放旧资源
            len = other.len;
            data = new char[len + 1];
            memcpy(data, other.data, len + 1);
        }
        return *this;  // 返回自身引用
    }

    ~MyString() { delete[] data; }

    void print() const { std::cout << data << std::endl; }
};

MyString s1("Hello"), s2("World");
s1 = s2;  // 拷贝赋值
s1.print();  // 输出: World
```

### 2.2 Copy-and-Swap惯用法

```cpp
class Buffer {
    int* data;
    size_t size;
public:
    Buffer(size_t n) : data(new int[n]), size(n) {}

    // 友元swap
    friend void swap(Buffer& a, Buffer& b) noexcept {
        using std::swap;
        swap(a.data, b.data);
        swap(a.size, b.size);
    }

    // copy-and-swap：异常安全
    Buffer& operator=(Buffer other) {  // 按值接收（拷贝/移动）
        swap(*this, other);  // 交换内容
        return *this;  // other析构释放旧资源
    }

    ~Buffer() { delete[] data; }
};
```

### 2.3 自赋值检查

```cpp
class Resource {
    int* ptr;
public:
    Resource& operator=(const Resource& other) {
        if (this != &other) {  // 防止自赋值
            delete ptr;
            ptr = new int(*other.ptr);
        }
        return *this;
    }
};

Resource r;
r = r;  // 自赋值，需要检查
```

### 2.4 返回引用的重要性

```cpp
class Counter {
    int count;
public:
    Counter& operator=(const Counter& other) {
        count = other.count;
        return *this;  // 必须返回引用，支持链式赋值
    }
};

Counter a, b, c;
a = b = c;  // 链式赋值：a.operator=(b.operator=(c))
```

## 三、注意事项与常见陷阱

- 拷贝赋值必须处理自赋值（`a = a`）
- 先释放旧资源，再分配新资源
- 返回`*this`的引用以支持链式赋值
- copy-and-swap提供异常安全保证
- Rule of Three/Five/Zero：实现拷贝构造、拷贝赋值、析构（或都不实现）
