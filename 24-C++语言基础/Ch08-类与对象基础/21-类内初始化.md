# 类内初始化

## 一、概念说明

C++11允许在类内声明数据成员时提供**默认初始化值**（Default Member Initializer）。如果构造函数的初始化列表未初始化该成员，则使用类内初始值。

## 二、具体用法

### 2.1 基本类内初始化

```cpp
class Config {
    int port = 8080;
    std::string host = "localhost";
    bool debug = false;
    double timeout = 30.0;

public:
    Config() = default;

    Config(int p) : port(p) {}  // 只覆盖port，其余用类内默认值

    void show() const {
        std::cout << host << ":" << port
                  << " debug=" << debug << std::endl;
    }
};

Config c1;
c1.show();  // 输出: localhost:8080 debug=0

Config c2(3000);
c2.show();  // 输出: localhost:3000 debug=0
```

### 2.2 与初始化列表的关系

```cpp
class Example {
    int a = 1;   // 类内默认值
    int b = 2;
    int c = 3;
public:
    // 初始化列表优先于类内初始化
    Example() : a(10) {}
    // a = 10 (初始化列表), b = 2 (类内), c = 3 (类内)
};

Example e;
```

### 2.3 使用花括号初始化

```cpp
class Container {
    std::vector<int> data = {1, 2, 3};  // 花括号初始化
    int count = 0;
    std::string name{"default"};        // 圆括号也可以
};

Container c;
```

### 2.4 static成员的类内初始化

```cpp
class Counter {
public:
    static inline int count = 0;  // C++17 inline static
    // 之前版本需要类外定义

    Counter() { ++count; }
};

Counter a, b;
std::cout << Counter::count << std::endl;  // 输出: 2
```

## 三、注意事项与常见陷阱

- 类内初始化不能使用圆括号初始化非静态成员（与函数声明歧义）
- `int x(42);` 在类内被解析为函数声明，用`int x{42};`或`int x = 42;`
- 初始化顺序由成员**声明顺序**决定，不是类内初始值的书写顺序
- `const`和引用成员可在类内初始化（C++11）
- 类内初始化简化了多构造函数的代码
