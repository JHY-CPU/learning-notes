# 类与单元测试

## 一、概念说明

可测试的类设计遵循**依赖注入**、**接口抽象**和**单一职责**原则。单元测试验证类的行为是否符合预期，使用Google Test等框架可以方便地编写和运行测试。

## 二、具体用法

### 2.1 可测试的类设计

```cpp
// 依赖接口，而非具体实现
class ILogger {
public:
    virtual void log(const std::string& msg) = 0;
    virtual ~ILogger() = default;
};

class UserService {
    ILogger& logger;
public:
    // 依赖注入：通过构造函数注入
    explicit UserService(ILogger& l) : logger(l) {}

    void createUser(const std::string& name) {
        // 创建用户逻辑
        logger.log("用户创建: " + name);
    }
};
```

### 2.2 Mock对象

```cpp
class MockLogger : public ILogger {
public:
    std::vector<std::string> messages;

    void log(const std::string& msg) override {
        messages.push_back(msg);
    }

    bool containsMessage(const std::string& expected) const {
        return std::any_of(messages.begin(), messages.end(),
            [&](const std::string& m) { return m.find(expected) != std::string::npos; });
    }
};

// 测试
void testUserCreation() {
    MockLogger logger;
    UserService service(logger);

    service.createUser("Alice");

    if (logger.containsMessage("Alice")) {
        std::cout << "测试通过\n";
    } else {
        std::cout << "测试失败\n";
    }
}
// 输出: 测试通过
```

### 2.3 使用Google Test框架

```cpp
#include <gtest/gtest.h>

class Calculator {
public:
    int add(int a, int b) { return a + b; }
    int divide(int a, int b) {
        if (b == 0) throw std::invalid_argument("除数为零");
        return a / b;
    }
};

TEST(CalculatorTest, AddWorks) {
    Calculator calc;
    EXPECT_EQ(calc.add(2, 3), 5);
    EXPECT_EQ(calc.add(-1, 1), 0);
}

TEST(CalculatorTest, DivideByZeroThrows) {
    Calculator calc;
    EXPECT_THROW(calc.divide(10, 0), std::invalid_argument);
}

TEST(CalculatorTest, DivideWorks) {
    Calculator calc;
    EXPECT_EQ(calc.divide(10, 3), 3);
    EXPECT_NEAR(10.0 / 3, 3.333, 0.001);  // 浮点比较
}
```

### 2.4 测试友好的设计

```cpp
class Clock {
public:
    virtual ~Clock() = default;
    virtual std::chrono::system_clock::time_point now() const {
        return std::chrono::system_clock::now();
    }
};

class MockClock : public Clock {
    std::chrono::system_clock::time_point fixedTime;
public:
    MockClock(std::chrono::system_clock::time_point t) : fixedTime(t) {}
    std::chrono::system_clock::time_point now() const override {
        return fixedTime;
    }
};

// 使用MockClock可测试时间相关的逻辑
```

## 三、最佳实践

1. **依赖注入**：通过构造函数注入依赖，便于替换为Mock
2. **接口隔离**：小接口比大接口更易测试
3. **无全局状态**：全局变量使测试隔离困难
4. **const成员函数**：无副作用的函数更易测试
5. **异常安全**：测试异常路径与正常路径
6. **可观察性**：提供`getter`使状态可验证

## 四、注意事项与常见陷阱

- 单元测试应快速、独立、可重复、自验证
- 不要测试实现细节，测试公开行为
- Mock过度会使测试与实现耦合
- 测试覆盖率高不等于测试质量高
- 先写测试（TDD）可改善类的设计
