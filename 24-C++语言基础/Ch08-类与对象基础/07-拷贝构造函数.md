# 拷贝构造函数

## 一、概念说明

拷贝构造函数用已有对象创建新对象。当编译器需要复制对象时会调用拷贝构造。管理资源的类必须实现**深拷贝**，否则默认的浅拷贝会导致多个对象共享同一资源。

## 二、具体用法

### 2.1 拷贝构造触发条件

```cpp
class MyClass {
public:
    MyClass(const MyClass& other) {
        std::cout << "拷贝构造被调用\n";
    }
};

MyClass a;
MyClass b = a;        // 拷贝构造
MyClass c(a);         // 拷贝构造
MyClass d = MyClass(a); // 拷贝构造

void func(MyClass x);  // 按值传递
func(a);               // 拷贝构造

MyClass func2() {
    MyClass local;
    return local;  // 可能被优化（RVO）
}
```

### 2.2 深拷贝 vs 浅拷贝

```cpp
class String {
    char* data;
    size_t len;
public:
    String(const char* s) : len(strlen(s)) {
        data = new char[len + 1];
        memcpy(data, s, len + 1);
    }

    // 深拷贝：分配独立内存
    String(const String& other) : len(other.len) {
        data = new char[len + 1];
        memcpy(data, other.data, len + 1);
        std::cout << "深拷贝\n";
    }

    ~String() { delete[] data; }

    void print() const { std::cout << data << std::endl; }
};

String s1("Hello");
String s2 = s1;  // 深拷贝
s2.print();  // 输出: Hello
// s1和s2各有独立的data，互不影响
```

### 2.3 禁止拷贝

```cpp
class UniqueResource {
    FILE* file;
public:
    UniqueResource(const char* name) : file(fopen(name, "r")) {}

    // 禁止拷贝
    UniqueResource(const UniqueResource&) = delete;
    UniqueResource& operator=(const UniqueResource&) = delete;

    ~UniqueResource() { if (file) fclose(file); }
};
```

### 2.4 浅拷贝的问题

```cpp
class Shallow {
    int* data;
public:
    Shallow(int val) : data(new int(val)) {}
    // 默认拷贝构造是浅拷贝：共享data指针

    // ~Shallow() { delete data; }  // 二次释放！
};

// 正确做法：自定义深拷贝或禁用拷贝
```

## 三、注意事项与常见陷阱

- 默认拷贝构造执行**浅拷贝**（逐成员拷贝）
- 管理动态资源的类必须实现深拷贝或禁用拷贝
- 拷贝构造的参数必须是`const`引用（非引用会导致无限递归）
- 拷贝构造中不要依赖其他未初始化的成员
- C++11起编译器可在某些情况下省略拷贝（RVO/NRVO）
