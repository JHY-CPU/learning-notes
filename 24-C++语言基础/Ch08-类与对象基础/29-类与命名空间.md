# 类与命名空间

## 一、概念说明

命名空间用于组织代码、避免命名冲突。类可以定义在命名空间内，通过`Namespace::ClassName`访问。前向声明允许在类的完整定义之前声明类的存在。

## 二、具体用法

### 2.1 命名空间中的类

```cpp
namespace Graphics {
    class Point {
    public:
        double x, y;
        Point(double x, double y) : x(x), y(y) {}
    };

    class Circle {
        Point center;
        double radius;
    public:
        Circle(Point c, double r) : center(c), radius(r) {}
        double area() const { return 3.14159 * radius * radius; }
    };
}

Graphics::Point p(0, 0);
Graphics::Circle c(p, 5);
std::cout << c.area() << std::endl;  // 输出: 78.5398
```

### 2.2 嵌套命名空间

```cpp
namespace MyLib::Graphics::Shapes {  // C++17嵌套命名空间
    class Rectangle {
        double w, h;
    public:
        Rectangle(double w, double h) : w(w), h(h) {}
        double area() const { return w * h; }
    };
}

MyLib::Graphics::Shapes::Rectangle r(3, 4);
std::cout << r.area() << std::endl;  // 输出: 12
```

### 2.3 前向声明

```cpp
class B;  // 前向声明：B是一个类（此时不完整）

class A {
    B* bPtr;  // 可以使用指针（不需要完整类型）
public:
    void setB(B* b) { bPtr = b; }
};

class B {
    int value;
public:
    B(int v) : value(v) {}
};
```

### 2.4 命名空间别名

```cpp
namespace VeryLongNamespaceName {
    class Widget {};
}

namespace VLN = VeryLongNamespaceName;  // 别名

VLN::Widget w;  // 简洁访问
```

### 2.5 using声明

```cpp
namespace MyLib {
    class String {};
}

// using声明（引入特定名称）
void func() {
    using MyLib::String;  // 仅在此作用域内
    String s;  // OK
}

// using指令（引入整个命名空间，不推荐在头文件中）
// using namespace MyLib;
```

## 三、注意事项与常见陷阱

- 不要在头文件中使用`using namespace`（污染全局命名空间）
- 前向声明只能用于指针和引用，不能用于值类型的声明
- `namespace A::B`是C++17语法，之前需嵌套`namespace A { namespace B { } }`
- 匿名命名空间提供内部链接（类似`static`）
- 命名空间不能定义在函数或类内部
