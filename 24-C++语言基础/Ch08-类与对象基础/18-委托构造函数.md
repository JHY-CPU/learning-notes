# 委托构造函数

## 一、概念说明

C++11允许构造函数**委托**（delegate）同一类的另一个构造函数执行初始化。这避免了多个构造函数之间的代码重复，所有公共初始化逻辑可集中在一个"目标构造函数"中。

## 二、具体用法

### 2.1 基本委托

```cpp
class Connection {
    std::string host;
    int port;
    int timeout;

public:
    // 目标构造函数
    Connection(const std::string& h, int p, int t)
        : host(h), port(p), timeout(t) {}

    // 委托构造函数
    Connection(const std::string& h, int p)
        : Connection(h, p, 30) {}  // 默认timeout=30

    Connection(const std::string& h)
        : Connection(h, 8080, 30) {}  // 默认port和timeout

    void show() const {
        std::cout << host << ":" << port << " timeout=" << timeout << std::endl;
    }
};

Connection c1("localhost", 3000, 60);
Connection c2("localhost", 3000);
Connection c3("localhost");

c1.show();  // 输出: localhost:3000 timeout=60
c2.show();  // 输出: localhost:3000 timeout=30
c3.show();  // 输出: localhost:8080 timeout=30
```

### 2.2 委托与异常

```cpp
class Resource {
    int* data;
public:
    // 目标构造函数
    Resource(int size) : data(new int[size]) {
        std::cout << "分配 " << size << " 个int\n";
    }

    // 委托构造：捕获异常
    Resource() try : Resource(100) {
        std::cout << "默认构造完成\n";
    } catch (...) {
        std::cout << "构造失败\n";
        // 异常自动传播
    }
};
```

### 2.3 委托 vs 工具函数

```cpp
class Config {
    std::string name;
    int value;
public:
    // 委托方式
    Config(const std::string& n, int v) : name(n), value(v) {}
    Config(const std::string& n) : Config(n, 0) {}
    Config() : Config("default", 0) {}

    // 对比：init函数方式（不推荐，因为对象已"构造"）
    // void init(const std::string& n, int v) { name = n; value = v; }
};
```

## 三、注意事项与常见陷阱

- 委托构造中不能同时使用初始化列表初始化成员
- 目标构造函数先执行完，委托构造的函数体后执行
- 委托链可以形成循环（编译器会检测并报错）
- 委托构造比在每个构造函数中调用`init()`函数更好
- 如果目标构造函数抛异常，委托构造也会抛异常
