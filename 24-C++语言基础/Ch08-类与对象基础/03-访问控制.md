# 访问控制

## 一、概念说明

C++提供三种访问控制符：**public**（任何地方可访问）、**private**（仅类内和友元可访问）、**protected**（类内、派生类和友元可访问）。`class`默认`private`，`struct`默认`public`。

## 二、具体用法

### 2.1 三种访问级别

```cpp
class Employee {
public:
    std::string name;
    void display() const {
        std::cout << name << ", 薪资: " << salary << std::endl;
    }

protected:
    std::string department;

private:
    double salary;  // 仅类内和友元可访问
};

Employee emp;
emp.name = "Alice";       // OK: public
// emp.salary = 5000;     // 编译错误: private
emp.display();            // OK: 通过public函数访问private数据
// 输出: Alice, 薪资: 0
```

### 2.2 封装的好处

```cpp
class Temperature {
    double celsius;
public:
    void setCelsius(double c) {
        if (c < -273.15) {
            throw std::invalid_argument("低于绝对零度");
        }
        celsius = c;
    }
    double getFahrenheit() const {
        return celsius * 9.0 / 5.0 + 32;
    }
};

Temperature t;
t.setCelsius(100);
std::cout << t.getFahrenheit() << std::endl;  // 输出: 212
```

### 2.3 访问控制与继承

```cpp
class Base {
public:
    int pub;
protected:
    int prot;
private:
    int priv;
};

class Derived : public Base {
    void foo() {
        pub = 1;    // OK
        prot = 2;   // OK
        // priv = 3;  // 编译错误
    }
};

Base b;
b.pub = 1;      // OK
// b.prot = 2;  // 编译错误
```

### 2.4 struct vs class

```cpp
// struct: 默认public继承和访问
struct Point {
    int x, y;  // public
};

// class: 默认private继承和访问
class Secret {
    int data;  // private
};
```

## 三、注意事项与常见陷阱

- `private`限制的是类（类型），不是对象——同类的不同对象可互相访问private成员
- `protected`成员对派生类可见但不是public
- 不要将所有成员都设为public，这破坏了封装
- 访问控制不影响内存布局（struct和class的内存布局相同）
- friend声明可以绕过访问控制
