# 析构函数

## 一、概念说明

析构函数在对象销毁时自动调用，负责释放资源（关闭文件、释放内存等）。析构函数无参数无返回值，每个类最多一个析构函数。如果类管理资源，必须遵循**RAII**原则。

## 二、具体用法

### 2.1 基本析构函数

```cpp
class Resource {
    int* data;
public:
    Resource(size_t n) : data(new int[n]) {
        std::cout << "分配 " << n << " 个int\n";
    }

    ~Resource() {
        delete[] data;
        std::cout << "释放内存\n";
    }
};

{
    Resource r(100);
    // 使用r...
}  // r离开作用域，析构函数自动调用
// 输出:
// 分配 100 个int
// 释放内存
```

### 2.2 调用顺序

```cpp
class Member {
public:
    Member(const std::string& n) : name(n) {
        std::cout << name << " 构造\n";
    }
    ~Member() { std::cout << name << " 析构\n"; }
private:
    std::string name;
};

class Container {
    Member m1, m2;
public:
    Container() : m1("A"), m2("B") {
        std::cout << "Container 构造\n";
    }
    ~Container() { std::cout << "Container 析构\n"; }
};

{
    Container c;
}
// 输出:
// A 构造
// B 构造
// Container 构造
// Container 析构
// B 析构（逆序）
// A 析构
```

### 2.3 RAII原则

```cpp
class FileHandler {
    FILE* file;
public:
    FileHandler(const char* name) : file(fopen(name, "r")) {
        if (!file) throw std::runtime_error("打开文件失败");
    }

    ~FileHandler() {
        if (file) {
            fclose(file);
            std::cout << "文件已关闭\n";
        }
    }

    // 禁止拷贝（独占资源）
    FileHandler(const FileHandler&) = delete;
    FileHandler& operator=(const FileHandler&) = delete;
};
```

### 2.4 虚析构函数

```cpp
class Base {
public:
    virtual ~Base() { std::cout << "~Base\n"; }
};

class Derived : public Base {
    int* data;
public:
    Derived() : data(new int[100]) {}
    ~Derived() override {
        delete[] data;
        std::cout << "~Derived\n";
    }
};

Base* p = new Derived();
delete p;  // 正确调用~Derived和~Base
// 输出:
// ~Derived
// ~Base
```

## 三、注意事项与常见陷阱

- 通过基类指针删除派生类对象时，基类析构函数必须是`virtual`
- 析构函数不应抛异常（默认`noexcept`）
- 构造失败时析构函数不会被调用，需清理部分初始化的资源
- 析构顺序：成员按声明逆序析构，然后调用基类析构
- `unique_ptr`和`shared_ptr`的自定义删除器可替代手动析构
