# 不可变对象

## 一、概念说明

不可变对象（Immutable Object）在创建后状态不可修改。不可变性带来线程安全、可缓存、易推理等优势。C++通过`const`成员、`const`成员函数和设计模式实现不可变性。

## 二、具体用法

### 2.1 const对象

```cpp
class Point {
    double x_, y_;
public:
    Point(double x, double y) : x_(x), y_(y) {}
    double x() const { return x_; }
    double y() const { return y_; }
    double distance() const { return std::sqrt(x_ * x_ + y_ * y_); }
};

const Point origin(0, 0);  // const对象
std::cout << origin.distance() << std::endl;  // 输出: 0
// origin.x_ = 10;  // 编译错误：不能修改const对象
```

### 2.2 不可变设计模式

```cpp
class ImmutableString {
    std::string data;
public:
    // 仅通过构造函数设置
    explicit ImmutableString(const std::string& s) : data(s) {}

    // 只有const访问
    const std::string& str() const { return data; }
    size_t size() const { return data.size(); }
    char at(size_t i) const { return data.at(i); }

    // 修改返回新对象
    ImmutableString append(const std::string& suffix) const {
        return ImmutableString(data + suffix);
    }

    ImmutableString toUpper() const {
        std::string upper = data;
        std::transform(upper.begin(), upper.end(), upper.begin(), ::toupper);
        return ImmutableString(upper);
    }
};

ImmutableString s("hello");
auto s2 = s.append(" world").toUpper();
std::cout << s.str() << std::endl;   // 输出: hello
std::cout << s2.str() << std::endl;  // 输出: HELLO WORLD
```

### 2.3 Builder模式构建不可变对象

```cpp
class Person {
    std::string name;
    int age;
    // 私有构造函数
    Person(const std::string& n, int a) : name(n), age(a) {}

public:
    class Builder {
        std::string name;
        int age = 0;
    public:
        Builder& setName(const std::string& n) { name = n; return *this; }
        Builder& setAge(int a) { age = a; return *this; }
        Person build() { return Person(name, age); }
    };

    std::string toString() const {
        return name + " (" + std::to_string(age) + ")";
    }
};

auto person = Person::Builder().setName("Alice").setAge(25).build();
std::cout << person.toString() << std::endl;
// 输出: Alice (25)
```

### 2.4 std::optional实现不可变可能值

```cpp
class Config {
    std::optional<std::string> host;
    std::optional<int> port;
public:
    Config(std::optional<std::string> h, std::optional<int> p)
        : host(std::move(h)), port(std::move(p)) {}

    Config withHost(const std::string& h) const {
        return Config(h, port);  // 返回新对象
    }

    const std::optional<std::string>& getHost() const { return host; }
};
```

## 三、注意事项与常见陷阱

- 不可变对象的"修改"通过创建新对象实现
- 频繁修改时不可变可能有性能开销（考虑移动语义或StringBuilder）
- `const`对象只能调用`const`成员函数
- 不可变对象天然线程安全（无数据竞争）
- 不可变性是函数式编程的核心原则之一
