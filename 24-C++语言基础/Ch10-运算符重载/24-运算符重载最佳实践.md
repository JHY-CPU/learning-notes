# 运算符重载最佳实践

## 一、概念说明

运算符重载应遵循**最小惊讶原则**，保持与内建类型一致的语义。以下是最常见的最佳实践规则。

## 二、具体用法

### 2.1 核心规则速查表

| 运算符 | 实现方式 | 返回类型 |
|--------|---------|---------|
| `+ - * /` | 非成员函数 | 新对象（值） |
| `+= -= *= /=` | 成员函数 | `*this`引用 |
| `== != < >` | 非成员或成员 | `bool` |
| `++ --` 前置 | 成员函数 | `*this`引用 |
| `++ --` 后置 | 成员函数 | 旧值（值） |
| `[]` | 成员函数 | 引用 |
| `()` | 成员函数 | 取决于用途 |
| `<< >>` | 非成员友元 | 流引用 |
| `=` | 成员函数 | `*this`引用 |

### 2.2 一致性示例

```cpp
#include <iostream>

class Vec3 {
public:
    double x, y, z;
    Vec3(double x=0, double y=0, double z=0) : x(x), y(y), z(z) {}

    // 复合赋值：成员，返回引用
    Vec3& operator+=(const Vec3& v) { x+=v.x; y+=v.y; z+=v.z; return *this; }

    // 二元算术：非成员，基于+=实现
    friend Vec3 operator+(Vec3 a, const Vec3& b) { return a += b; }

    // 比较：一致
    bool operator==(const Vec3& v) const { return x==v.x && y==v.y && z==v.z; }
    bool operator!=(const Vec3& v) const { return !(*this == v); }

    // 输出
    friend std::ostream& operator<<(std::ostream& os, const Vec3& v) {
        return os << "(" << v.x << "," << v.y << "," << v.z << ")";
    }
};

int main() {
    Vec3 a(1,2,3), b(4,5,6);
    std::cout << "a+b=" << a+b << std::endl;
    std::cout << "a==b: " << (a==b) << std::endl;
    return 0;
}
```

**输出：**
```
a+b=(5,7,9)
a==b: 0
```

## 三、注意事项与常见陷阱

- 不要滥用运算符重载，语义必须直观
- 保持运算符的数学性质（`+`的交换性、结合性等）
- 基于复合赋值实现二元算术避免重复代码
- 始终同时定义`==`和`!=`
- 考虑使用C++20的`<=>`减少比较运算符代码
