# 运算符重载与继承

## 一、概念说明

运算符重载可以在继承体系中使用，但需要特别注意**虚运算符**和**类型兼容**问题。通常将比较等通用运算符声明为虚函数以支持多态比较。

## 二、具体用法

### 2.1 继承中的运算符

```cpp
#include <iostream>
#include <memory>

class Shape {
public:
    virtual ~Shape() = default;
    virtual bool operator==(const Shape& other) const = 0;
    virtual std::unique_ptr<Shape> clone() const = 0;
    virtual double area() const = 0;
};

class Circle : public Shape {
    double radius;
public:
    Circle(double r) : radius(r) {}

    bool operator==(const Shape& other) const override {
        auto* c = dynamic_cast<const Circle*>(&other);
        return c && c->radius == radius;
    }

    std::unique_ptr<Shape> clone() const override {
        return std::make_unique<Circle>(*this);
    }

    double area() const override { return 3.14159 * radius * radius; }
};

int main() {
    Circle c1(5), c2(5), c3(3);
    std::cout << "c1 == c2: " << (c1 == c2) << std::endl;
    std::cout << "c1 == c3: " << (c1 == c3) << std::endl;

    // 基类引用比较
    const Shape& s1 = c1;
    const Shape& s2 = c3;
    std::cout << "s1 == s2: " << (s1 == s2) << std::endl;
    return 0;
}
```

**输出：**
```
c1 == c2: 1
c1 == c3: 0
s1 == s2: 0
```

## 三、注意事项与常见陷阱

- 运算符可以声明为`virtual`以支持多态
- 比较运算符需要`dynamic_cast`检查实际类型
- 拷贝赋值运算符不应声明为虚函数（但clone模式可替代）
- 继承中的运算符签名必须一致才能正确覆盖
