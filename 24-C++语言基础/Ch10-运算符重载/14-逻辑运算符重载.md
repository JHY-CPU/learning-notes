# 逻辑运算符重载

## 一、概念说明

逻辑运算符`&&`和`||`的内建版本具有**短路求值**特性（左侧为真时不再求右侧）。但重载版本是**函数调用**，失去了短路特性——两个操作数都会被求值。这可能导致意外行为。

## 二、具体用法

### 2.1 短路特性丢失

```cpp
#include <iostream>

class SafeBool {
    bool val;
public:
    SafeBool(bool v) : val(v) {}

    // 重载&&（丢失短路！）
    SafeBool operator&&(const SafeBool& rhs) const {
        std::cout << "两个操作数都被求值!" << std::endl;
        return SafeBool(val && rhs.val);
    }

    explicit operator bool() const { return val; }
};

int main() {
    SafeBool a(true), b(false);
    // 内建&&：a为true才会求b
    // 重载&&：a和b都先被求值（函数参数求值规则）
    auto result = a && b;
    std::cout << "结果: " << static_cast<bool>(result) << std::endl;
    return 0;
}
```

**输出：**
```
两个操作数都被求值!
结果: 0
```

## 三、注意事项与常见陷阱

- **不建议**重载`&&`和`||`，因为会丢失短路语义
- 重载版本作为函数调用，参数求值顺序未定义
- 使用`explicit operator bool()`配合内建逻辑运算符更安全
- 如需短路行为，使用命名函数（如`and_then`）
