# 逻辑运算符重载

## 一、概念说明

逻辑运算符`&&`和`||`的内建版本具有**短路求值**特性（C++11标准 §5.14、§5.15）：`&&`左侧为false时不再求右侧，`||`左侧为true时不再求右侧。但重载版本是**函数调用**，失去了短路特性——两个操作数都会被求值。

这使得重载 `&&` 和 `||` 几乎总是错误的设计决策。

## 二、具体用法

### 2.1 短路特性丢失的问题

```cpp
#include <iostream>
using namespace std;

class SafeBool {
    bool val;
public:
    SafeBool(bool v) : val(v) {}

    // 重载&&（丢失短路！）
    SafeBool operator&&(const SafeBool& rhs) const {
        cout << "两个操作数都被求值!" << endl;
        return SafeBool(val && rhs.val);
    }

    explicit operator bool() const { return val; }
};

bool mightFail() {
    cout << "mightFail() 被调用（不应发生!）" << endl;
    return false;
}

int main() {
    SafeBool a(false), b(true);

    // 内建&&：a为false时不会求b
    // if (false && mightFail()) → mightFail不会被调用

    // 重载&&：a和b都先被求值（函数参数求值规则）
    auto result = a && b;
    cout << "结果: " << static_cast<bool>(result) << endl;

    return 0;
}
```

### 2.2 推荐替代方案

```cpp
#include <iostream>
using namespace std;

class CheckedPtr {
    int* ptr;
public:
    explicit CheckedPtr(int* p = nullptr) : ptr(p) {}

    // 使用explicit operator bool，让内建&&||处理短路
    explicit operator bool() const { return ptr != nullptr; }

    int& operator*() { return *ptr; }
};

class Validator {
    int value;
public:
    Validator(int v) : value(v) {}

    explicit operator bool() const { return value > 0; }

    bool isValid() const { return value > 0; }
};

int main() {
    int x = 42;
    CheckedPtr p1(&x);
    CheckedPtr p2(nullptr);

    // 使用内建&&（保留短路语义）
    if (p1 && *p1 > 0) {  // p1为false时不会解引用
        cout << "p1有效且正值" << endl;
    }

    if (p2 && *p2 > 0) {  // p2为false，短路，不执行*p2
        cout << "p2有效" << endl;  // 不会执行
    }

    // Validator示例
    Validator v1(10), v2(-5);
    if (v1 && v2.isValid()) {  // 内建&&，保留短路
        cout << "都有效" << endl;
    }

    return 0;
}
```

## 三、注意事项与常见陷阱

1. **不建议重载&&和||**：会丢失短路语义，这是C++中少数几个不建议重载的运算符
2. **重载版本作为函数调用，参数求值顺序未定义**：两个操作数都先被求值
3. **使用explicit operator bool()配合内建逻辑运算符更安全**：保留短路特性
4. **如需短路行为，使用命名函数**：如 `and_then()`、`or_else()`
5. **实际应用中几乎找不到重载&&和||的代码**：标准库和常见库都没有
6. **C++17的std::optional::and_then提供了函数式的短路链**：推荐替代方案
