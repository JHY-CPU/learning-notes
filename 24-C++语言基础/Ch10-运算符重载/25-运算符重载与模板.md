# 运算符重载与模板

## 一、概念说明

模板与运算符重载结合可以编写**通用代码**。使用SFINAE或C++20 concepts可以在编译期检测类型是否支持特定运算符，提供更好的错误信息和重载决议。

| 技术 | 引入版本 | 特点 |
|------|---------|------|
| SFINAE | C++11 | 基于替换失败的重载决议 |
| void_t | C++17 | 简化SFINAE检测 |
| Concepts | C++20 | 声明式约束，清晰的错误信息 |

## 二、具体用法

### 2.1 SFINAE检测运算符

```cpp
#include <iostream>
#include <type_traits>
using namespace std;

// 检测是否有operator+
template<typename T, typename = void>
struct has_plus : false_type {};

template<typename T>
struct has_plus<T, void_t<decltype(declval<T>() + declval<T>())>>
    : true_type {};

// 检测是否有operator<<
template<typename T, typename = void>
struct is_printable : false_type {};

template<typename T>
struct is_printable<T, void_t<decltype(cout << declval<T>())>>
    : true_type {};

// 检测是否有operator==
template<typename T, typename = void>
struct has_equality : false_type {};

template<typename T>
struct has_equality<T, void_t<decltype(declval<T>() == declval<T>())>>
    : true_type {};

int main() {
    cout << "int有operator+: " << has_plus<int>::value << endl;       // 1
    cout << "int可打印: " << is_printable<int>::value << endl;        // 1
    cout << "int有operator==: " << has_equality<int>::value << endl;  // 1

    // 自定义类型
    struct NoAdd {};
    struct WithAdd {
        WithAdd operator+(const WithAdd&) const { return {}; }
    };

    cout << "NoAdd有+: " << has_plus<NoAdd>::value << endl;    // 0
    cout << "WithAdd有+: " << has_plus<WithAdd>::value << endl; // 1

    return 0;
}
```

### 2.2 C++20 Concepts约束

```cpp
#include <iostream>
#include <concepts>
#include <string>
using namespace std;

// 定义concept约束运算符
template<typename T>
concept Addable = requires(T a, T b) {
    { a + b } -> convertible_to<T>;
};

template<typename T>
concept Printable = requires(T t, ostream& os) {
    { os << t } -> same_as<ostream&>;
};

template<typename T>
concept Comparable = requires(T a, T b) {
    { a == b } -> convertible_to<bool>;
    { a < b } -> convertible_to<bool>;
};

// 使用concept约束的泛型函数
template<Addable T>
T sum(T a, T b) {
    return a + b;
}

template<Printable T>
void printValue(const T& val) {
    cout << val << endl;
}

template<Comparable T>
T minVal(T a, T b) {
    return a < b ? a : b;
}

int main() {
    cout << sum(1, 2) << endl;            // 3
    cout << sum(1.5, 2.5) << endl;        // 4
    cout << sum("hi"s, " there"s) << endl; // hi there

    printValue(42);                       // 42
    printValue(3.14);                     // 3.14

    cout << minVal(10, 20) << endl;       // 10
    cout << minVal(3.14, 2.72) << endl;   // 2.72

    // sum(1, "hello") → 编译错误：int和const char*不满足Addable

    return 0;
}
```

### 2.3 通用打印函数

```cpp
#include <iostream>
#include <type_traits>
using namespace std;

// 使用if constexpr处理不同运算符能力的类型
template<typename T>
void smartPrint(const T& val) {
    if constexpr (is_arithmetic_v<T>) {
        cout << "数值: " << val << endl;
    } else if constexpr (is_pointer_v<T>) {
        cout << "指针: " << val << endl;
    } else {
        cout << "其他类型" << endl;
    }
}

int main() {
    smartPrint(42);       // 数值: 42
    smartPrint(3.14);     // 数值: 3.14
    int x = 10;
    smartPrint(&x);       // 指针: 0x...

    return 0;
}
```

## 三、注意事项与常见陷阱

1. **SFINAE是C++11/14/17检测运算符的标准方法**：基于替换失败的重载决议
2. **C++20 concepts提供更清晰的约束语法**：推荐使用
3. **模板运算符重载注意ADL（参数依赖查找）规则**：非成员运算符的查找范围
4. **友元函数模板在类内定义时需要注意可见性**：每个实例化生成独立的友元函数
5. **SFINAE表达式不要有副作用**：只用于类型推导
6. **Concept可以组合**：`template<typename T> concept X = Addable<T> && Printable<T>;`
