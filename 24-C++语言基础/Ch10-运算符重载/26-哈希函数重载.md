# 哈希函数重载

## 一、概念说明

要让自定义类型作为`std::unordered_map`或`std::unordered_set`的键，需要特化`std::hash`并定义`operator==`。哈希函数应保证相等的对象产生相同的哈希值。

## 二、具体用法

### 2.1 特化std::hash

```cpp
#include <iostream>
#include <unordered_map>
#include <string>

struct Point {
    int x, y;
    bool operator==(const Point& o) const {
        return x == o.x && y == o.y;
    }
};

// 特化std::hash
namespace std {
    template<>
    struct hash<Point> {
        size_t operator()(const Point& p) const {
            // 组合哈希：使用标准技术
            size_t h1 = std::hash<int>{}(p.x);
            size_t h2 = std::hash<int>{}(p.y);
            return h1 ^ (h2 << 1);
        }
    };
}

int main() {
    std::unordered_map<Point, std::string> umap;
    umap[{1, 2}] = "点A";
    umap[{3, 4}] = "点B";

    Point key = {1, 2};
    if (umap.count(key))
        std::cout << "找到: " << umap[key] << std::endl;

    for (auto& [p, name] : umap)
        std::cout << "(" << p.x << "," << p.y << ")=" << name << std::endl;
    return 0;
}
```

**输出：**
```
找到: 点A
(3,4)=点B
(1,2)=点A
```

## 三、注意事项与常见陷阱

- 哈希函数必须与`operator==`一致（相等则哈希相等）
- 组合哈希避免简单异或导致碰撞
- 不要在`std`命名空间中添加新函数，只允许特化模板
- C++20可使用`std::hash`的折叠表达式简化
- 哈希碰撞过多会影响`unordered_map`性能
