# 哈希函数重载

## 一、概念说明

要让自定义类型作为`std::unordered_map`或`std::unordered_set`的键，需要**特化`std::hash`**并定义`operator==`（C++11标准 §20.9.13）。哈希函数应保证相等的对象产生相同的哈希值。

哈希容器要求：
- `operator==`：判断键相等
- `std::hash<T>`：计算哈希值
- 哈希值相等是`==`的必要条件（反之不成立）

## 二、具体用法

### 2.1 特化std::hash

```cpp
#include <iostream>
#include <unordered_map>
#include <string>
#include <functional>
using namespace std;

struct Point {
    int x, y;
    bool operator==(const Point& o) const {
        return x == o.x && y == o.y;
    }
};

// 特化std::hash
namespace std {
    template<>
    struct hash<Point> {
        size_t operator()(const Point& p) const {
            // 组合哈希：使用标准技术避免碰撞
            size_t h1 = hash<int>{}(p.x);
            size_t h2 = hash<int>{}(p.y);
            return h1 ^ (h2 << 1);  // 或使用更复杂的组合方式
        }
    };
}

int main() {
    unordered_map<Point, string> umap;
    umap[{1, 2}] = "点A";
    umap[{3, 4}] = "点B";
    umap[{5, 6}] = "点C";

    Point key = {1, 2};
    if (umap.count(key))
        cout << "找到: " << umap[key] << endl;  // 点A

    for (const auto& [p, name] : umap)
        cout << "(" << p.x << "," << p.y << ")=" << name << " ";
    cout << endl;

    return 0;
}
```

### 2.2 使用自定义哈希函数对象

```cpp
#include <iostream>
#include <unordered_map>
#include <string>
using namespace std;

struct Employee {
    string name;
    int id;

    bool operator==(const Employee& o) const {
        return name == o.name && id == o.id;
    }
};

// 自定义哈希函数对象（不用特化std::hash）
struct EmployeeHash {
    size_t operator()(const Employee& e) const {
        size_t h1 = hash<string>{}(e.name);
        size_t h2 = hash<int>{}(e.id);
        return h1 ^ (h2 << 1);
    }
};

int main() {
    // 使用自定义哈希作为第三个模板参数
    unordered_map<Employee, double, EmployeeHash> salaries;

    salaries[{"Alice", 1001}] = 80000;
    salaries[{"Bob", 1002}] = 90000;

    Employee key{"Alice", 1001};
    if (salaries.count(key))
        cout << key.name << " 薪资: " << salaries[key] << endl;

    return 0;
}
```

### 2.3 哈希组合技术

```cpp
#include <iostream>
#include <functional>
using namespace std;

// 更好的哈希组合（减少碰撞）
struct GoodHash {
    // boost::hash_combine 的简化版本
    static void hashCombine(size_t& seed, size_t value) {
        seed ^= value + 0x9e3779b9 + (seed << 6) + (seed >> 2);
    }

    size_t operator()(int x, int y, int z) const {
        size_t seed = 0;
        hashCombine(seed, hash<int>{}(x));
        hashCombine(seed, hash<int>{}(y));
        hashCombine(seed, hash<int>{}(z));
        return seed;
    }
};

struct Vec3i {
    int x, y, z;
    bool operator==(const Vec3i& o) const {
        return x == o.x && y == o.y && z == o.z;
    }
};

namespace std {
    template<>
    struct hash<Vec3i> {
        size_t operator()(const Vec3i& v) const {
            size_t seed = 0;
            GoodHash::hashCombine(seed, hash<int>{}(v.x));
            GoodHash::hashCombine(seed, hash<int>{}(v.y));
            GoodHash::hashCombine(seed, hash<int>{}(v.z));
            return seed;
        }
    };
}

int main() {
    unordered_map<Vec3i, string> coords;
    coords[{1, 2, 3}] = "原点附近";
    coords[{100, 200, 300}] = "远处";

    cout << coords[{1, 2, 3}] << endl;  // 原点附近

    return 0;
}
```

## 三、注意事项与常见陷阱

1. **哈希函数必须与operator==一致**：相等则哈希相等
2. **组合哈希避免简单异或导致碰撞**：使用hash_combine等标准技术
3. **不要在std命名空间中添加新函数**：只允许特化模板
4. **C++20可使用std::hash的折叠表达式简化**：减少样板代码
5. **哈希碰撞过多会影响unordered_map性能**：好的哈希函数很重要
6. **自定义哈希函数对象可作为模板参数传入**：比特化std::hash更灵活
