# 三元运算符不可重载

## 一、概念说明

**三元条件运算符`?:`不可重载**。C++标准禁止重载此运算符是因为它需要**延迟求值**（短路行为），而重载机制是函数调用，函数参数无法实现延迟求值。

### 1.1 替代方案

使用成员函数`if_else()`或命名函数模拟三元运算符的行为。

## 二、具体用法

### 2.1 使用成员函数替代

```cpp
#include <iostream>

class SmartValue {
    double val;
public:
    SmartValue(double v) : val(v) {}

    // 模拟三元运算的分支选择
    SmartValue if_else(const SmartValue& trueVal, const SmartValue& falseVal) const {
        return val ? trueVal : falseVal;
    }

    double get() const { return val; }

    friend std::ostream& operator<<(std::ostream& os, const SmartValue& v) {
        return os << v.val;
    }
};

int main() {
    SmartValue cond(1), a(100), b(200);
    SmartValue result = cond.if_else(a, b);
    std::cout << "结果: " << result << std::endl;

    SmartValue cond2(0);
    std::cout << "结果: " << cond2.if_else(a, b) << std::endl;
    return 0;
}
```

**输出：**
```
结果: 100
结果: 200
```

## 三、注意事项与常见陷阱

- `?:`不可重载，这是语言规范的硬性限制
- 原因是函数调用会强制求值所有参数，丢失短路特性
- 可以用成员函数或模板函数替代
- C++20的`std::is_constant_evaluated()`提供了编译期分支选择
