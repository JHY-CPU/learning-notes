# 三元运算符不可重载

## 一、概念说明

**三元条件运算符`?:`不可重载**（C++11标准 §13.5）。C++标准禁止重载此运算符是因为它需要**延迟求值**（短路行为：只求值被选中的分支），而重载机制是函数调用，函数参数无法实现延迟求值。

不可重载的运算符一览：
- `.`（点运算符）
- `.*`（成员指针运算符）
- `::`（作用域解析）
- `?:`（三元条件）
- `sizeof`
- `typeid`
- `alignof`

## 二、具体用法

### 2.1 使用成员函数替代

```cpp
#include <iostream>
using namespace std;

class SmartValue {
    double val;
public:
    SmartValue(double v) : val(v) {}

    // 模拟三元运算的分支选择
    SmartValue if_else(const SmartValue& trueVal, const SmartValue& falseVal) const {
        return val ? trueVal : falseVal;
    }

    double get() const { return val; }

    friend ostream& operator<<(ostream& os, const SmartValue& v) {
        return os << v.val;
    }
};

int main() {
    SmartValue cond(1), a(100), b(200);
    SmartValue result = cond.if_else(a, b);
    cout << "结果: " << result << endl;  // 100

    SmartValue cond2(0);
    cout << "结果: " << cond2.if_else(a, b) << endl;  // 200

    return 0;
}
```

### 2.2 使用模板函数替代

```cpp
#include <iostream>
#include <type_traits>
using namespace std;

// 模板化的条件选择（保留延迟求值）
template<typename T, typename F, typename U>
auto ternary(bool cond, T&& trueVal, F&& falseVal) {
    // 注意：这里两个分支都会被求值（不完全等价于?:）
    // 但对于值类型通常是可接受的
    using CommonType = common_type_t<decay_t<T>, decay_t<F>>;
    return cond ? static_cast<CommonType>(forward<T>(trueVal))
                : static_cast<CommonType>(forward<F>(falseVal));
}

// lambda延迟求值版本
template<typename T, typename F>
auto lazyTernary(bool cond, T&& trueFn, F&& falseFn) {
    return cond ? forward<T>(trueFn)() : forward<F>(falseFn)();
}

int main() {
    cout << ternary(true, 10, 20) << endl;   // 10
    cout << ternary(false, 10, 20) << endl;  // 20

    // lambda版本保留延迟求值
    auto result = lazyTernary(true,
        []() { cout << "真分支" << endl; return 100; },
        []() { cout << "假分支" << endl; return 200; });
    cout << result << endl;  // 只输出"真分支"，返回100

    return 0;
}
```

### 2.3 std::optional作为替代

```cpp
#include <iostream>
#include <optional>
using namespace std;

int main() {
    optional<int> val = 42;
    optional<int> empty;

    // 使用value_or模拟条件取值
    cout << val.value_or(0) << endl;    // 42
    cout << empty.value_or(0) << endl;  // 0

    // 使用transform链式操作
    auto result = val.transform([](int v) { return v * 2; });
    cout << result.value_or(-1) << endl;  // 84

    return 0;
}
```

## 三、注意事项与常见陷阱

1. **`?:`不可重载，这是语言规范的硬性限制**：设计原因在于延迟求值
2. **原因：函数调用会强制求值所有参数**：丢失短路特性
3. **可用成员函数或模板函数替代**：但不完全等价（延迟求值丢失）
4. **C++20的 `std::is_constant_evaluated()` 提供编译期分支选择**：用于constexpr编程
5. **C++23的 `if consteval` 替代了部分场景**：更清晰的编译期分支
6. **lambda延迟求值可以保留短路特性**：但语法更复杂
