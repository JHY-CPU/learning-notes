# 解引用与箭头运算符

## 一、概念说明

`operator*`和`operator->`用于使自定义类表现为**智能指针**或**迭代器**。`operator->`的特殊之处在于它被递归调用直到返回真正的指针。

## 二、具体用法

### 2.1 简单智能指针

```cpp
#include <iostream>

template<typename T>
class SmartPtr {
    T* ptr;
public:
    explicit SmartPtr(T* p = nullptr) : ptr(p) {}
    ~SmartPtr() { delete ptr; }

    // 解引用：返回引用
    T& operator*() { return *ptr; }
    const T& operator*() const { return *ptr; }

    // 箭头：返回指针
    T* operator->() { return ptr; }
    const T* operator->() const { return ptr; }

    // 禁止拷贝
    SmartPtr(const SmartPtr&) = delete;
    SmartPtr& operator=(const SmartPtr&) = delete;
};

struct Widget {
    int id;
    void display() { std::cout << "Widget " << id << std::endl; }
};

int main() {
    SmartPtr<Widget> w(new Widget{42});
    (*w).display();     // operator*()
    w->display();       // operator->()
    std::cout << "id=" << w->id << std::endl;
    return 0;
}
```

**输出：**
```
Widget 42
Widget 42
id=42
```

### 2.2 迭代器中的应用

```cpp
#include <iostream>

class SimpleIter {
    int* ptr;
public:
    SimpleIter(int* p) : ptr(p) {}
    int& operator*() const { return *ptr; }
    int* operator->() const { return ptr; }
    SimpleIter& operator++() { ++ptr; return *this; }
    bool operator!=(const SimpleIter& o) const { return ptr != o.ptr; }
};
```

## 三、注意事项与常见陷阱

- `operator->`返回指针后编译器自动继续`->`调用（递归语义）
- 智能指针应禁止或正确实现拷贝/移动操作
- `std::unique_ptr`和`std::shared_ptr`已实现了这些运算符
- 自定义迭代器需要同时实现`*`和`->`
