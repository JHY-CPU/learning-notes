# 类型转换运算符

## 一、概念说明

**类型转换运算符**（Conversion Operator）`operator T()`定义了从类类型到其他类型的转换。C++11起可用`explicit`关键字防止意外的隐式转换。

## 二、具体用法

### 2.1 隐式与显式转换

```cpp
#include <iostream>

class Celsius {
    double temp;
public:
    explicit Celsius(double t) : temp(t) {}

    // 显式转换运算符（推荐）
    explicit operator double() const { return temp; }

    // 隐式转换（不推荐，可能有意外行为）
    operator std::string() const {
        return std::to_string(temp) + "°C";
    }
};

void printTemp(double t) { std::cout << t << " 度" << std::endl; }

int main() {
    Celsius c(36.6);
    double d = static_cast<double>(c);  // 显式转换OK
    std::cout << d << std::endl;

    std::string s = c;  // 隐式转换OK（operator string非explicit）
    std::cout << s << std::endl;

    // printTemp(c);  // 编译错误！explicit double阻止隐式转换
    return 0;
}
```

**输出：**
```
36.6
36.600000°C
```

### 2.2 bool转换

```cpp
#include <iostream>

class File {
    bool isOpen;
public:
    File(bool open) : isOpen(open) {}

    // explicit operator bool：用于条件判断
    explicit operator bool() const { return isOpen; }
};

int main() {
    File f1(true), f2(false);
    if (f1) std::cout << "f1已打开" << std::endl;
    if (!f2) std::cout << "f2未打开" << std::endl;
    // bool b = f1;  // 编译错误！explicit阻止
    return 0;
}
```

**输出：**
```
f1已打开
f2未打开
```

## 三、注意事项与常见陷阱

- 建议转换运算符加`explicit`防止意外隐式转换
- `operator bool()`应加`explicit`防止与整数混用
- 避免定义多个转换运算符，可能导致歧义
- 标准库`std::optional`、`std::ifstream`都使用了`explicit operator bool`
- 转换运算符可以是虚函数
