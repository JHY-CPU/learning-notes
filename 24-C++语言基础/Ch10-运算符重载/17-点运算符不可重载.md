# 点运算符不可重载

## 一、概念说明

**点运算符`.`和指向成员的点运算符`.*`不可重载**。这是为了保证对象成员访问的基本语义不会被改变。如果允许重载`.`，将无法可靠地访问对象的真实成员。

### 1.1 替代方案

使用`operator->`和`operator*`间接实现类似"代理"访问的功能。

## 二、具体用法

### 2.1 使用operator->间接实现

```cpp
#include <iostream>
#include <string>

// 不能重载.但可以重载->实现代理模式
class StringProxy {
    std::string* str;
public:
    StringProxy(std::string* s) : str(s) {}
    std::string* operator->() { return str; }
    const std::string* operator->() const { return str; }
};

int main() {
    std::string s = "Hello";
    StringProxy proxy(&s);
    // proxy. 不能重载，但proxy->可以访问string的方法
    std::cout << proxy->size() << std::endl;
    std::cout << proxy->substr(0, 3) << std::endl;
    return 0;
}
```

**输出：**
```
5
Hel
```

## 三、注意事项与常见陷阱

- `.`和`.*`不可重载是语言硬性规定
- 可通过`operator->`和`operator*`实现智能指针和代理模式
- C++23增加了`operator.`的实验性提案（P0416），但尚未正式
- 标准库`std::reference_wrapper`使用`operator T&`间接实现类似代理
