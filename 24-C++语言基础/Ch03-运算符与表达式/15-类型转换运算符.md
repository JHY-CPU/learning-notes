# 类型转换运算符

## 一、概念说明

C++提供了四种显式类型转换运算符（C++11标准 §5.2.9、§5.2.7、§5.2.11、§5.2.10），替代了C风格的强制转换，提供更安全、意图更明确的类型转换：

| 运算符 | 安全性 | 求值时机 | 典型场景 |
|--------|--------|---------|---------|
| `static_cast` | 编译时检查 | 编译期 | 数值转换、void*转换、类层次转换 |
| `dynamic_cast` | 运行时检查 | 运行期 | 多态类型安全向下转型 |
| `const_cast` | 编译时检查 | 编译期 | 添加/移除const或volatile |
| `reinterpret_cast` | 最低安全性 | 编译期 | 底层二进制重新解释 |

## 二、具体用法

### 2.1 static_cast — 最常用的转换

```cpp
#include <iostream>
#include <cstdint>
using namespace std;

class Base { public: virtual ~Base() = default; };
class Derived : public Base { public: void extra() { cout << "Derived::extra\n"; } };

int main() {
    // 1. 数值类型转换（隐式可转的都能用static_cast）
    double pi = 3.14159;
    int i = static_cast<int>(pi);           // 截断：3
    short s = static_cast<short>(100000);   // 溢出行为由实现定义

    // 2. void* 与具体类型指针互转
    int value = 42;
    void* vp = &value;
    int* ip = static_cast<int*>(vp);        // 安全：原本就是int*

    // 3. 类层次中的上行/下行转换
    Derived d;
    Base* bp = &d;                          // 上行：隐式允许
    Derived* dp = static_cast<Derived*>(bp); // 下行：无运行时检查（危险！）

    // 4. 枚举与整数互转
    enum class Color { Red = 1, Green = 2 };
    int colorVal = static_cast<int>(Color::Red);
    Color c = static_cast<Color>(2);

    // 5. 左值引用转换
    const int cx = 10;
    int& ref = const_cast<int&>(cx);        // 注意：这里用const_cast

    cout << "pi→int: " << i << endl;
    cout << "void*→int*: " << *ip << endl;
    cout << "enum→int: " << colorVal << endl;

    return 0;
}
```

### 2.2 dynamic_cast — 运行时安全转型

```cpp
#include <iostream>
using namespace std;

class Animal { public: virtual ~Animal() = default; };
class Dog : public Animal { public: void bark() { cout << "汪汪!\n"; } };
class Cat : public Animal { public: void meow() { cout << "喵喵!\n"; } };

void interact(Animal* a) {
    // 指针转换：失败返回nullptr
    if (Dog* d = dynamic_cast<Dog*>(a)) {
        d->bark();
    } else if (Cat* c = dynamic_cast<Cat*>(a)) {
        c->meow();
    }

    // 引用转换：失败抛出 std::bad_cast
    try {
        Dog& dogRef = dynamic_cast<Dog&>(*a);
        cout << "引用转换成功\n";
    } catch (const bad_cast& e) {
        cout << "引用转换失败: " << e.what() << endl;
    }
}

int main() {
    Dog dog;
    Cat cat;
    interact(&dog);  // 汪汪!
    interact(&cat);  // 引用转换失败

    return 0;
}
```

**要点**：基类必须至少有一个虚函数（通常虚析构函数即可），否则编译失败。

### 2.3 const_cast — 移除/添加const限定

```cpp
#include <iostream>
using namespace std;

void legacy_print(char* str) {  // 第三方C库，不接受const
    cout << str << endl;
}

int main() {
    // 合法用法：原始对象本身不是const
    char mutableBuf[] = "hello";
    const char* cp = mutableBuf;
    legacy_print(const_cast<char*>(cp));  // 安全：原始对象可修改

    // 危险用法：原始对象是const → 修改它是UB
    const int x = 42;
    int* p = const_cast<int*>(&x);
    *p = 100;            // 未定义行为！x可能在只读段
    cout << *p << endl;  // 可能输出100，但行为不可预测

    return 0;
}
```

### 2.4 reinterpret_cast — 底层二进制重解释

```cpp
#include <iostream>
#include <cstdint>
using namespace std;

int main() {
    // 指针 ↔ 整数
    int x = 42;
    uintptr_t addr = reinterpret_cast<uintptr_t>(&x);
    int* p = reinterpret_cast<int*>(addr);
    cout << "地址: 0x" << hex << addr << dec << endl;
    cout << "取值: " << *p << endl;

    // 类型双关：读取float的二进制表示
    float f = 3.14f;
    uint32_t bits = reinterpret_cast<uint32_t&>(f);  // 严格别名违规！
    // C++20推荐用 std::bit_cast：
    // uint32_t bits = std::bit_cast<uint32_t>(f);

    // 函数指针转换（实现定义的行为）
    using FuncPtr = void(*)();
    int (*intFunc)() = []() { return 42; };
    FuncPtr fp = reinterpret_cast<FuncPtr>(intFunc);  // 调用是UB

    return 0;
}
```

## 三、转换选择指南

| 场景 | 推荐转换 | 原因 |
|------|---------|------|
| 数值类型转换 | `static_cast` | 编译时检查，意图明确 |
| void* → 具体指针 | `static_cast` | 安全且足够 |
| 多态向下转型 | `dynamic_cast` | 运行时类型安全 |
| 非多态向下转型 | `static_cast` | 编译时已知类型 |
| 去除const | `const_cast` | 唯一合法方式 |
| 指针↔整数 | `reinterpret_cast` | 底层操作 |
| 跨类型位模式转换 | `std::bit_cast` | C++20，安全的类型双关 |

## 四、注意事项与常见陷阱

1. **优先使用命名转换而非C风格转换**：`(int)ptr`可以同时执行多种转换，命名转换则限制单一语义，更安全、可搜索
2. **dynamic_cast需要虚函数**：基类没有任何虚函数时编译错误（无法获取RTTI信息）
3. **const_cast修改const对象是UB**：只有原始对象本身就不是const时，去掉const才是安全的
4. **reinterpret_cast可能违反严格别名规则**：编译器可能基于假定做优化，导致意外结果；C++20推荐 `std::bit_cast`
5. **static_cast下行转换不做运行时检查**：如果实际类型不对，后续访问可能UB
6. **避免在模板中使用C风格转换**：它会尝试const_cast、static_cast、reinterpret_cast直到编译通过，行为不可控
7. **std::bit_cast (C++20) 是reinterpret_cast的安全替代**：用于位模式复制，要求类型大小相同且是平凡可复制的
