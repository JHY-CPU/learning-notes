# 类型转换运算符

## 一、概念说明

C++提供了四种显式类型转换运算符：`static_cast`、`dynamic_cast`、`const_cast`和`reinterpret_cast`，替代了C风格的类型转换，提供更安全、意图更明确的转换。

## 二、具体用法

### 2.1 四种转换运算符速览

```cpp
#include <iostream>
using namespace std;

class Base { public: virtual ~Base() = default; };
class Derived : public Base { public: void extra() {} };

int main() {
    // static_cast：编译时类型检查，最常用
    double pi = 3.14;
    int i = static_cast<int>(pi);           // 数值转换
    void* vp = &i;
    int* ip = static_cast<int*>(vp);        // void* → int*

    // dynamic_cast：运行时类型检查，需要虚函数
    Base* bp = new Derived();
    Derived* dp = dynamic_cast<Derived*>(bp);  // 向下转型
    if (dp) dp->extra();
    delete bp;

    // const_cast：添加/移除const
    const int x = 42;
    const int* cp = &x;
    int* p = const_cast<int*>(cp);          // 移除const

    // reinterpret_cast：底层二进制重新解释
    int val = 42;
    uintptr_t addr = reinterpret_cast<uintptr_t>(&val);

    cout << "static_cast: " << i << endl;
    cout << "const_cast: " << *p << endl;
    cout << "reinterpret_cast: 0x" << hex << addr << dec << endl;

    return 0;
}
```

输出：
```
static_cast: 3
const_cast: 42
reinterpret_cast: 0x7ffd...
```

详细内容请参考 Ch02/15~18 的类型转换专题。

## 三、注意事项与常见陷阱

1. **优先使用这四种而非C风格转换**：更安全、更易搜索
2. **static_cast最常用**：数值转换、void*转换、类层次转换
3. **dynamic_cast需要虚函数**：基类没有虚函数则编译错误
4. **const_cast改const对象是UB**：只有原始非const对象才能安全地去const
5. **reinterpret_cast最危险**：仅在底层编程时使用
