# 无符号数陷阱

## 一、概念说明

无符号整数只能表示非负值。当有符号和无符号混合运算时，有符号数会被转换为无符号数，可能导致意外结果。

## 二、具体用法

### 2.1 混合运算陷阱

```cpp
#include <iostream>
using namespace std;

int main() {
    int a = -1;
    unsigned int b = 1;

    // 陷阱：有符号转无符号
    // -1 转为 UINT_MAX（4294967295）
    cout << "-1 < 1u? " << (a < b ? "true" : "false") << endl;  // false!
    cout << "-1 as unsigned: " << (unsigned int)a << endl;

    // 循环陷阱
    vector<int> vec = {1, 2, 3};
    // 危险！当vec为空时size()=0，0-1=UINT_MAX
    for (int i = 0; i < vec.size() - 1; i++) {
        // vec.size()返回size_t（无符号）
        // i < vec.size() - 1 在vec为空时变成 0 < 4294967295 = true
    }

    // 安全写法
    for (size_t i = 0; i + 1 < vec.size(); i++) {
        cout << vec[i] << " ";
    }
    cout << endl;

    return 0;
}
```

输出：
```
-1 < 1u? false
-1 as unsigned: 4294967295
1 2
```

### 2.2 回绕行为

```cpp
#include <iostream>
using namespace std;

int main() {
    unsigned int x = 0;
    x--;  // 回绕到最大值
    cout << "0 - 1 = " << x << endl;  // 4294967295

    // 无符号减法问题
    unsigned int a = 5, b = 10;
    unsigned int diff = a - b;  // 回绕
    cout << "5 - 10 = " << diff << endl;  // 4294967291

    // 解决方案：使用有符号类型进行可能为负的运算
    int sa = 5, sb = 10;
    int sdiff = sa - sb;
    cout << "有符号: 5 - 10 = " << sdiff << endl;  // -5

    return 0;
}
```

输出：
```
0 - 1 = 4294967295
5 - 10 = 4294967291
有符号: 5 - 10 = -5
```

## 三、注意事项与常见陷阱

1. **混合运算总是转无符号**：`int + unsigned`结果是`unsigned`
2. **循环条件**：`i < vec.size() - 1`在空vector时有问题
3. **不要用无符号表示"不应为负"**：用有符号+断言更安全
4. **size_t是无符号的**：与int比较时要注意
5. **回绕行为是定义的**：无符号回绕是合法的（非UB），但可能不是期望的
