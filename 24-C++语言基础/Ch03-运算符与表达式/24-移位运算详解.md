# 移位运算详解

## 一、概念说明

左移`<<`将所有位向左移动，右移`>>`向右移动。移位运算高效地实现乘除2的幂次方。有符号数和无符号数的右移行为不同。

## 二、具体用法

### 2.1 左移与右移

```cpp
#include <iostream>
#include <bitset>
using namespace std;

int main() {
    unsigned int x = 0b00001010;  // 10

    // 左移：低位补0
    cout << "原值:     " << bitset<8>(x) << " = " << x << endl;
    cout << "x << 1:  " << bitset<8>(x << 1) << " = " << (x << 1) << endl;  // 20
    cout << "x << 2:  " << bitset<8>(x << 2) << " = " << (x << 2) << endl;  // 40
    cout << "x << 3:  " << bitset<8>(x << 3) << " = " << (x << 3) << endl;  // 80

    // 右移：高位补0（无符号）
    cout << "x >> 1:  " << bitset<8>(x >> 1) << " = " << (x >> 1) << endl;  // 5
    cout << "x >> 2:  " << bitset<8>(x >> 2) << " = " << (x >> 2) << endl;  // 2

    // 有符号数的右移：算术移位（补符号位）
    int y = -8;  // 11111000（二进制补码）
    cout << "\ny = " << y << endl;
    cout << "y >> 1: " << (y >> 1) << endl;  // -4（补1）
    cout << "y >> 2: " << (y >> 2) << endl;  // -2（补1）

    // 乘除2的幂
    cout << "\n5 * 8 = " << (5 << 3) << endl;    // 40
    cout << "40 / 4 = " << (40 >> 2) << endl;    // 10

    return 0;
}
```

输出：
```
原值:     00001010 = 10
x << 1:  00010100 = 20
x << 2:  00101000 = 40
x << 3:  01010000 = 80
x >> 1:  00000101 = 5
x >> 2:  00000010 = 2

y = -8
y >> 1: -4
y >> 2: -2

5 * 8 = 40
40 / 4 = 10
```

### 2.2 逻辑移位 vs 算术移位

```cpp
#include <iostream>
#include <bitset>
using namespace std;

int main() {
    // 无符号右移：逻辑移位（高位补0）
    unsigned int u = 0xF0000000;
    cout << "unsigned >>:" << endl;
    cout << "  原值: " << bitset<32>(u) << endl;
    cout << "  >> 4: " << bitset<32>(u >> 4) << endl;

    // 有符号右移：算术移位（高位补符号位）
    int s = 0xF0000000;  // 负数
    cout << "\nsigned >>:" << endl;
    cout << "  原值: " << bitset<32>(s) << " (" << s << ")" << endl;
    cout << "  >> 4: " << bitset<32>(s >> 4) << " (" << (s >> 4) << ")" << endl;

    // 左移行为：两种都是逻辑左移（低位补0）
    // 但有符号数左移超出范围是UB

    return 0;
}
```

输出（示例）：
```
unsigned >>:
  原值: 11110000000000000000000000000000
  >> 4: 00001111000000000000000000000000

signed >>:
  原值: 11110000000000000000000000000000 (-268435456)
  >> 4: 11111111000000000000000000000000 (-16777216)
```

## 三、注意事项与常见陷阱

1. **移位范围**：移位量 >= 类型位数是未定义行为
2. **有符号右移**：是实现定义的（算术还是逻辑），不要依赖具体行为
3. **左移负数**：有符号数左移是未定义行为
4. **移位代替乘除**：`x << n`等于`x * 2^n`，编译器通常会自动优化
5. **优先级**：移位优先级低于算术运算符，`1 << 2 + 1`是`1 << 3`
