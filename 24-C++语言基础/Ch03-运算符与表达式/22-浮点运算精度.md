# 浮点运算精度

## 一、概念说明

浮点数使用有限的二进制位表示实数，存在**精度丢失**问题。IEEE 754标准定义了浮点数的特殊值（inf、NaN）和运算规则。

## 二、具体用法

### 2.1 精度丢失

```cpp
#include <iostream>
#include <iomanip>
#include <cmath>
using namespace std;

int main() {
    // 经典的0.1 + 0.2问题
    double a = 0.1;
    double b = 0.2;
    double c = a + b;

    cout << fixed << setprecision(20);
    cout << "0.1 实际存储: " << a << endl;
    cout << "0.2 实际存储: " << b << endl;
    cout << "0.1 + 0.2:    " << c << endl;
    cout << "0.3 实际存储: " << 0.3 << endl;
    cout << "0.1+0.2 == 0.3? " << (c == 0.3 ? "true" : "false") << endl;

    // 累积误差
    float sum = 0.0f;
    for (int i = 0; i < 1000000; i++) {
        sum += 0.000001f;
    }
    cout << "\n累加100万次(float): " << sum << endl;

    double sum2 = 0.0;
    for (int i = 0; i < 1000000; i++) {
        sum2 += 0.000001;
    }
    cout << "累加100万次(double): " << sum2 << endl;

    return 0;
}
```

输出（示例）：
```
0.1 实际存储: 0.10000000000000000555
0.2 实际存储: 0.20000000000000001110
0.1 + 0.2:    0.30000000000000004441
0.3 实际存储: 0.29999999999999998890
0.1+0.2 == 0.3? false

累加100万次(float): 0.95367431640625000000
累加100万次(double): 1.00000000000000066613
```

### 2.2 安全比较方法

```cpp
#include <iostream>
#include <cmath>
#include <cfloat>
using namespace std;

// 方法1：绝对误差比较
bool absEqual(double a, double b, double eps = 1e-9) {
    return abs(a - b) < eps;
}

// 方法2：相对误差比较
bool relEqual(double a, double b, double relTol = 1e-9) {
    double diff = abs(a - b);
    double maxVal = max(abs(a), abs(b));
    return diff <= relTol * maxVal;
}

// 方法3：ULP比较（最精确）
bool ulpEqual(double a, double b, int maxUlps = 4) {
    if (isnan(a) || isnan(b)) return false;
    if (isinf(a) || isinf(b)) return a == b;
    // 简化版：使用DBL_EPSILON
    return abs(a - b) <= DBL_EPSILON * max(abs(a), abs(b)) * maxUlps;
}

int main() {
    double x = 0.1 + 0.2;
    double y = 0.3;

    cout << "直接==: " << (x == y ? "相等" : "不等") << endl;
    cout << "绝对误差: " << (absEqual(x, y) ? "相等" : "不等") << endl;
    cout << "相对误差: " << (relEqual(x, y) ? "相等" : "不等") << endl;
    cout << "ULP比较: " << (ulpEqual(x, y) ? "相等" : "不等") << endl;

    return 0;
}
```

输出：
```
直接==: 不等
绝对误差: 相等
相对误差: 相等
ULP比较: 相等
```

## 三、注意事项与常见陷阱

1. **永远不要用==比较浮点数**：由于精度限制，几乎总是不相等
2. **float精度只有6-7位**：大多数场景应使用double
3. **特殊值**：NaN不等于任何值（包括自身），inf可以比较
4. **累加误差**：大量累加会累积误差，考虑Kahan求和算法
5. **金融计算用定点数**：不要用浮点数做金额计算，使用整数（分）或decimal库
