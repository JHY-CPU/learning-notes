# 表达式求值顺序

## 一、概念说明

C++中表达式的求值顺序**不总是确定的**。编译器可以自由选择子表达式的计算顺序，这可能导致**未定义行为**。

## 二、具体用法

### 2.1 C++17前的求值顺序

```cpp
#include <iostream>
using namespace std;

int counter = 0;

int next() {
    return ++counter;
}

int main() {
    // C++17前：函数参数的求值顺序不确定
    // f(next(), next()) 可能先算第一个参数，也可能先算第二个

    // 这是未定义行为（C++17前）
    // cout << next() << " " << next() << endl;
    // 可能输出 "1 2" 或 "2 1"

    // 安全做法：分开调用
    int a = next();
    int b = next();
    cout << a << " " << b << endl;

    return 0;
}
```

### 2.2 C++17的保证

```cpp
#include <iostream>
using namespace std;

int main() {
    // C++17保证了以下求值顺序：
    // 1. a.b → 先求a再求b
    // 2. a->b → 先求a再求b
    // 3. a(b, c, d) → 先求a，再从左到右求b,c,d
    // 4. a = b → 先求b再求a
    // 5. a[b] → 先求a再求b
    // 6. a << b, a >> b → 先求a再求b
    // 7. a && b, a || b → 先求a再求b（且有短路）
    // 8. a ? b : c → 先求a，再求b或c之一
    // 9. a, b → 先求a再求b

    // 但以下仍不确定：
    // a + b * c → 先算a还是先算b*c？
    // f() + g() → 先调用f还是g？

    int x = 1;
    // x = x++;  // 仍然未定义行为（修改+读取无序列点）

    cout << "C++17: 函数参数从左到右求值" << endl;

    return 0;
}
```

### 2.3 序列点与未定义行为

```cpp
#include <iostream>
using namespace std;

int main() {
    int i = 5;

    // 未定义行为：同一表达式中修改和读取同一变量
    // i = i++;           // UB
    // i = ++i;           // UB
    // a[i] = i++;        // UB
    // cout << i << i++;  // UB

    // 安全写法
    int j = 5;
    int temp = j;
    j++;
    int result = temp;   // 分开操作

    cout << "安全: " << result << ", " << j << endl;

    // C++11引入的序列点替代：序列化前后关系
    // i = ++i + 1;  // UB：++i修改i，然后赋值也修改i，无序列点
    i = 5;
    i = i + 1;  // OK：赋值前完成右边所有计算
    cout << "i = " << i << endl;

    return 0;
}
```

输出：
```
安全: 5, 6
i = 6
```

## 三、注意事项与常见陷阱

1. **同一变量多次修改是UB**：`i = i++`、`arr[i] = i++`等
2. **C++17改善了部分情况**：函数参数从左到右求值
3. **不要依赖求值顺序**：将复杂表达式拆分为多个简单语句
4. **volatile不解决序列点问题**：`volatile`不能代替正确的同步
5. **使用编译器警告**：开启`-Wall -Wsequence-point`可以检测部分问题
