# 变量声明与定义

## 一、概念说明

C++严格区分**声明**（declaration）和**定义**（definition）。声明告诉编译器"这个变量/函数存在"，定义则为其分配存储空间。理解这个区别对多文件项目至关重要。

## 二、具体用法

### 2.1 声明与定义的区别

```cpp
// ===== header.h =====
#ifndef HEADER_H
#define HEADER_H

// 声明：告诉编译器存在这些实体，不分配存储
extern int globalCount;           // 变量声明
void printMessage();              // 函数声明
class Calculator;                 // 类的前向声明

#endif

// ===== source.cpp =====
#include "header.h"
#include <iostream>

// 定义：实际分配存储空间或提供实现
int globalCount = 0;              // 变量定义

void printMessage() {             // 函数定义
    std::cout << "计数: " << globalCount << std::endl;
}

class Calculator {                // 类定义
public:
    int add(int a, int b) { return a + b; }
};

int main() {
    globalCount = 42;
    printMessage();

    Calculator calc;
    std::cout << "3 + 4 = " << calc.add(3, 4) << std::endl;
    return 0;
}
```

输出：
```
计数: 42
3 + 4 = 7
```

### 2.2 extern关键字

```cpp
// ===== config.h =====
extern const int MAX_USERS;      // const的extern声明
extern double pi;

// ===== config.cpp =====
#include "config.h"

const int MAX_USERS = 100;       // 定义
double pi = 3.14159265358979;

// ===== main.cpp =====
#include <iostream>
#include "config.h"

int main() {
    std::cout << "最大用户数: " << MAX_USERS << std::endl;
    std::cout << "PI: " << pi << std::endl;
    return 0;
}
```

输出：
```
最大用户数: 100
PI: 3.14159
```

### 2.3 ODR规则（One Definition Rule）

```cpp
// ODR规则：
// 1. 每个变量/函数在整个程序中只能有一个定义
// 2. 每个类/枚举/模板在每个编译单元中只能有一个定义
// 3. 内联函数/模板/constexpr可以在多个编译单元中定义（相同定义）

// 错误示例：多重定义
// file1.cpp
int x = 10;  // 定义

// file2.cpp
int x = 20;  // 链接错误：多重定义

// 正确做法：
// header.h
extern int x;  // 声明

// file1.cpp
int x = 10;   // 唯一定义

// file2.cpp
#include "header.h"
// 使用 x，不再定义

// ===== 内联函数：例外 =====
// header.h
inline int square(int x) { return x * x; }  // 多个文件包含OK
constexpr int cube(int x) { return x * x * x; }  // 同上
```

### 2.4 static在声明定义中的作用

```cpp
// 文件作用域static：限制链接范围（类似匿名命名空间）
// file1.cpp
static int internalVar = 100;   // 仅在file1.cpp可见
int sharedVar = 200;            // 全局可见

// file2.cpp
// static int internalVar;  // 链接错误：找不到定义
extern int sharedVar;        // OK

#include <iostream>
int main() {
    std::cout << "共享变量: " << sharedVar << std::endl;
    return 0;
}
```

## 三、注意事项与常见陷阱

1. **ODR是最常见的链接错误来源**：在头文件中定义非const变量会导致多重定义
2. **const变量的特殊规则**：const变量默认内部链接，头文件中定义const变量不会导致多重定义
3. **extern const**：如果需要在多个文件中共享const变量，使用`extern const`声明+单独定义
4. **inline替代static**：现代C++推荐用匿名命名空间替代`static`函数
5. **前向声明减少依赖**：类只需要指针/引用时，用前向声明替代`#include`
