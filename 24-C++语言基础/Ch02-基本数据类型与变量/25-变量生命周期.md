# 变量生命周期

## 一、概念说明

变量的生命周期（lifetime）是指变量从**创建到销毁**的时间段。C++有四种存储期：自动存储期、静态存储期、线程存储期和动态存储期，它们决定了变量的创建和销毁时机。

## 二、具体用法

### 2.1 自动存储期（局部变量）

```cpp
#include <iostream>
using namespace std;

class Resource {
    string name;
public:
    Resource(const string& n) : name(n) {
        cout << "创建: " << name << endl;
    }
    ~Resource() {
        cout << "销毁: " << name << endl;
    }
};

void demo() {
    Resource r1("r1");        // 进入函数时创建
    {
        Resource r2("r2");    // 进入块时创建
        cout << "在内层块中" << endl;
    }  // r2在此处销毁（析构函数调用）
    cout << "r2已销毁" << endl;
}  // r1在此处销毁

int main() {
    cout << "进入main" << endl;
    demo();
    cout << "退出main" << endl;
    return 0;
}
```

输出：
```
进入main
创建: r1
创建: r2
在内层块中
销毁: r2
r2已销毁
销毁: r1
退出main
```

### 2.2 静态存储期（全局/静态变量）

```cpp
#include <iostream>
using namespace std;

// 静态存储期：程序启动时创建，程序结束时销毁
int globalVar = 100;                    // 全局变量
static int fileStatic = 200;            // 文件作用域静态变量

void func() {
    static int callCount = 0;           // 静态局部变量
    callCount++;
    cout << "func被调用 " << callCount << " 次" << endl;
}

// 初始化顺序（跨文件不确定，同文件按定义顺序）
int a = 1;
int b = a + 1;  // b = 2（a在此文件中已初始化）

class Singleton {
    static Singleton* instance;
public:
    static Singleton* getInstance() {
        if (!instance) {
            instance = new Singleton();
        }
        return instance;
    }
private:
    Singleton() { cout << "Singleton构造" << endl; }
};
Singleton* Singleton::instance = nullptr;

int main() {
    func();
    func();
    func();

    cout << "globalVar: " << globalVar << endl;

    return 0;
}
```

输出：
```
func被调用 1 次
func被调用 2 次
func被调用 3 次
globalVar: 100
```

### 2.3 线程存储期（C++11）

```cpp
#include <iostream>
#include <thread>
using namespace std;

// 线程局部存储：每个线程有自己的副本
thread_local int threadCounter = 0;

void increment(int threadId) {
    threadCounter += threadId;
    cout << "线程" << threadId << ": counter = " << threadCounter << endl;
}

int main() {
    thread t1(increment, 1);
    thread t2(increment, 2);
    thread t3(increment, 3);

    t1.join();
    t2.join();
    t3.join();

    cout << "主线程: counter = " << threadCounter << endl;

    return 0;
}
```

输出（示例）：
```
线程1: counter = 1
线程2: counter = 2
线程3: counter = 3
主线程: counter = 0
```

### 2.4 动态存储期（堆内存）

```cpp
#include <iostream>
#include <memory>
using namespace std;

class Widget {
    int id;
public:
    Widget(int i) : id(i) {
        cout << "Widget#" << id << " 创建" << endl;
    }
    ~Widget() {
        cout << "Widget#" << id << " 销毁" << endl;
    }
};

int main() {
    // 手动管理生命周期（不推荐）
    Widget* raw = new Widget(1);
    delete raw;  // 手动销毁

    // 智能指针自动管理（推荐）
    {
        auto ptr = make_unique<Widget>(2);
        cout << "在作用域内" << endl;
    }  // ptr离开作用域，自动delete

    cout << "智能指针已自动销毁" << endl;

    // shared_ptr引用计数
    shared_ptr<Widget> sp1 = make_shared<Widget>(3);
    {
        shared_ptr<Widget> sp2 = sp1;  // 引用计数+1
        cout << "引用计数: " << sp1.use_count() << endl;
    }  // sp2销毁，引用计数-1
    cout << "引用计数: " << sp1.use_count() << endl;

    return 0;
}
```

输出：
```
Widget#1 创建
Widget#1 销毁
Widget#2 创建
在作用域内
Widget#2 销毁
智能指针已自动销毁
Widget#3 创建
引用计数: 2
引用计数: 1
Widget#3 销毁
```

## 三、注意事项与常见陷阱

1. **悬垂引用**：引用已销毁的变量会导致未定义行为
2. **静态初始化顺序惨案**：跨文件的静态变量初始化顺序不确定
3. **RAII原则**：利用构造函数/析构函数管理资源，确保资源在生命周期结束时释放
4. **智能指针管理动态内存**：避免手动`new`/`delete`，使用`unique_ptr`和`shared_ptr`
5. **thread_local每个线程独立**：不同线程的`thread_local`变量互不影响
