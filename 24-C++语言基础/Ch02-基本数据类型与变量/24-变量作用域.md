# 变量作用域

## 一、概念说明

作用域（scope）定义了变量名的**可见范围**。C++有多种作用域：块作用域、函数作用域、类作用域、命名空间作用域和文件作用域。理解作用域是避免命名冲突和bug的关键。

## 二、具体用法

### 2.1 作用域类型

```cpp
#include <iostream>
using namespace std;

// 文件作用域/全局作用域
int globalVar = 100;

// 命名空间作用域
namespace MyLib {
    int libVar = 200;
}

// 类作用域
class MyClass {
public:
    static int classVar;  // 类作用域
    void method();        // 类作用域
};
int MyClass::classVar = 300;

void demo() {
    // 块作用域（函数内）
    int localVar = 42;

    {
        // 嵌套块作用域
        int innerVar = 99;
        cout << "innerVar: " << innerVar << endl;
    }
    // innerVar在这里不可见

    // for循环作用域
    for (int i = 0; i < 3; i++) {
        cout << i << " ";
    }
    cout << endl;
    // i在这里不可见（C++11起）

    cout << "localVar: " << localVar << endl;
}

int main() {
    demo();
    cout << "globalVar: " << globalVar << endl;
    cout << "MyLib::libVar: " << MyLib::libVar << endl;
    cout << "MyClass::classVar: " << MyClass::classVar << endl;

    return 0;
}
```

输出：
```
innerVar: 99
0 1 2
localVar: 42
globalVar: 100
MyLib::libVar: 200
MyClass::classVar: 300
```

### 2.2 作用域遮蔽（Shadowing）

```cpp
#include <iostream>
using namespace std;

int value = 100;  // 全局变量

int main() {
    int value = 200;  // 遮蔽全局变量
    cout << "局部value: " << value << endl;
    cout << "全局value: " << ::value << endl;  // 使用作用域解析运算符

    {
        int value = 300;  // 遮蔽外层的局部变量
        cout << "内层value: " << value << endl;
        cout << "外层value: " << ::value << endl;  // 这是全局的！
    }

    cout << "外层value: " << value << endl;  // 恢复到200

    // 参数遮蔽
    auto func = [](int value) {
        cout << "参数value: " << value << endl;
        int value = 999;  // 错误：参数被同名变量遮蔽
    };
    func(42);

    return 0;
}
```

输出：
```
局部value: 200
全局value: 100
内层value: 300
外层value: 100
外层value: 200
参数value: 42
```

### 2.3 作用域与生命周期

```cpp
#include <iostream>
using namespace std;

int globalVar = 1;  // 文件作用域，整个程序生命周期

void func() {
    static int staticLocal = 0;  // 块作用域，整个程序生命周期
    staticLocal++;
    cout << "staticLocal: " << staticLocal << endl;

    int autoLocal = 10;  // 块作用域，进入块时创建，退出时销毁
    autoLocal++;
    cout << "autoLocal: " << autoLocal << endl;
}

int main() {
    func();  // staticLocal: 1, autoLocal: 11
    func();  // staticLocal: 2, autoLocal: 11（重新创建）
    func();  // staticLocal: 3, autoLocal: 11

    return 0;
}
```

输出：
```
staticLocal: 1
autoLocal: 11
staticLocal: 2
autoLocal: 11
staticLocal: 3
autoLocal: 11
```

## 三、注意事项与常见陷阱

1. **作用域遮蔽**：内层变量遮蔽外层同名变量是常见bug来源，开启`-Wshadow`警告
2. **全局变量慎用**：全局变量增加耦合，难以追踪修改来源
3. **最小作用域原则**：变量应在最内层的作用域中声明，尽量推迟声明位置
4. **::运算符**：`::value`访问全局作用域的`value`，`ClassName::member`访问类成员
5. **for循环变量**：C++11起`for`循环的循环变量只在for语句和循环体中可见
