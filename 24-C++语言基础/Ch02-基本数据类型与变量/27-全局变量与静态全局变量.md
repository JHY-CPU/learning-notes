# 全局变量与静态全局变量

## 一、概念说明

全局变量在所有函数外部定义，具有**文件/全局作用域**和**静态存储期**。静态全局变量（`static`修饰）限制了全局变量的**链接性**，使其仅在当前编译单元可见。

## 二、具体用法

### 2.1 全局变量

```cpp
// ===== global_demo.cpp =====
#include <iostream>
using namespace std;

// 全局变量：具有外部链接性，可被其他文件访问
int globalCounter = 0;
const double PI = 3.14159;  // const全局变量默认内部链接

// 全局变量在main之前初始化
int earlyInit = []() {
    cout << "全局变量初始化（main之前）" << endl;
    return 42;
}();

void increment() {
    globalCounter++;
}

int main() {
    cout << "earlyInit: " << earlyInit << endl;
    increment();
    increment();
    cout << "globalCounter: " << globalCounter << endl;
    return 0;
}
```

输出：
```
全局变量初始化（main之前）
earlyInit: 42
globalCounter: 2
```

### 2.2 静态全局变量

```cpp
// ===== file1.cpp =====
#include <iostream>
using namespace std;

// 静态全局变量：内部链接性，仅在本文件可见
static int fileCounter = 0;
static string fileName = "file1.cpp";

// 匿名命名空间方式（推荐替代static）
namespace {
    int anonymousVar = 100;
}

void file1Func() {
    fileCounter++;
    cout << fileName << " 计数: " << fileCounter << endl;
    cout << "匿名变量: " << anonymousVar << endl;
}

// ===== file2.cpp =====
// extern int fileCounter;  // 链接错误：fileCounter是静态的
// extern string fileName;  // 链接错误

// 自己的静态全局变量（不同文件中同名不冲突）
static int fileCounter = 0;  // OK：各文件独立
```

### 2.3 全局变量的初始化顺序

```cpp
#include <iostream>
using namespace std;

// 同一编译单元内：按定义顺序初始化
int a = 10;
int b = a + 5;    // b = 15（a已经初始化）

// 不同编译单元：顺序不确定（静态初始化顺序惨案）
// file1.cpp: extern int x;
// file2.cpp: int x = 42;
// file1.cpp中x可能还未初始化就使用了

// 解决方案1：函数内静态变量（懒初始化）
int& getGlobalX() {
    static int x = 42;  // 首次使用时初始化
    return x;
}

// 解决方案2：construct on first use
class GlobalConfig {
    static GlobalConfig* instance;
public:
    static GlobalConfig& getInstance() {
        static GlobalConfig inst;
        return inst;
    }
    int value = 100;
};
GlobalConfig* GlobalConfig::instance = nullptr;

int main() {
    cout << "a: " << a << ", b: " << b << endl;
    cout << "getGlobalX(): " << getGlobalX() << endl;
    cout << "配置值: " << GlobalConfig::getInstance().value << endl;

    return 0;
}
```

输出：
```
a: 10, b: 15
getGlobalX(): 42
配置值: 100
```

### 2.4 全局常量

```cpp
#include <iostream>
using namespace std;

// 头文件中的全局常量
// 方式1：const（内部链接，每个文件有自己的副本）
const int MAX_BUFFER = 1024;

// 方式2：constexpr（隐式内部链接）
constexpr int PAGE_SIZE = 4096;

// 方式3：inline变量（C++17，共享单一定义）
inline int sharedVersion = 1;

// 方式4：extern const（声明在头文件，定义在一个源文件）
// header: extern const int CONFIG_VALUE;
// source: extern const int CONFIG_VALUE = 42;

int main() {
    cout << "MAX_BUFFER: " << MAX_BUFFER << endl;
    cout << "PAGE_SIZE: " << PAGE_SIZE << endl;
    cout << "sharedVersion: " << sharedVersion << endl;
    return 0;
}
```

输出：
```
MAX_BUFFER: 1024
PAGE_SIZE: 4096
sharedVersion: 1
```

## 三、注意事项与常见陷阱

1. **静态初始化顺序惨案**：不同编译单元的全局变量初始化顺序不确定，使用懒初始化解决
2. **避免全局可变状态**：全局可变变量难以追踪修改来源，优先使用函数返回引用
3. **const全局变量特殊**：`const`全局变量默认内部链接，不需要`static`修饰
4. **匿名命名空间替代static**：C++推荐用匿名命名空间替代`static`全局变量
5. **inline变量（C++17）**：解决了头文件中定义全局变量的多重定义问题
