# 类型推断与类型别名

## 一、概念说明

C++提供了`typedef`和`using`来创建类型别名，以及`auto`和`decltype`来进行类型推断。类型别名使复杂类型更易读，类型推断简化了类型声明。

## 二、具体用法

### 2.1 typedef

```cpp
#include <iostream>
#include <vector>
#include <map>
using namespace std;

// typedef创建类型别名
typedef unsigned int uint;
typedef vector<int> IntVector;
typedef map<string, vector<int>> StringToIntListMap;
typedef int (*CompareFunc)(const void*, const void*);

// 函数指针typedef示例
typedef void (*SignalHandler)(int);

void myHandler(int sig) {
    cout << "收到信号: " << sig << endl;
}

int main() {
    uint age = 25;
    IntVector nums = {1, 2, 3, 4, 5};
    StringToIntListMap data;
    data["scores"] = {90, 85, 92};

    SignalHandler handler = myHandler;
    handler(2);

    cout << "age: " << age << endl;
    cout << "nums大小: " << nums.size() << endl;
    cout << "data[\"scores\"][0]: " << data["scores"][0] << endl;

    return 0;
}
```

输出：
```
收到信号: 2
age: 25
nums大小: 5
data["scores"][0]: 90
```

### 2.2 using（C++11推荐）

```cpp
#include <iostream>
#include <vector>
#include <functional>
using namespace std;

// using创建类型别名（更清晰的语法）
using uint = unsigned int;
using IntVector = vector<int>;
using StringMap = map<string, string>;
using Callback = function<void(int)>;

// using的优势：模板别名
template<typename T>
using Vec = vector<T>;

template<typename K, typename V>
using HashMap = map<K, V>;

// typedef无法直接创建模板别名
// template<typename T>
// typedef vector<T> Vec;  // 编译错误

void onEvent(int id) {
    cout << "事件ID: " << id << endl;
}

int main() {
    // 使用模板别名
    Vec<int> nums = {1, 2, 3};
    Vec<string> names = {"Alice", "Bob"};
    HashMap<string, int> ages = {{"Alice", 25}, {"Bob", 30}};

    Callback cb = onEvent;
    cb(42);

    cout << "nums[0]: " << nums[0] << endl;
    cout << "ages[\"Alice\"]: " << ages["Alice"] << endl;

    return 0;
}
```

输出：
```
事件ID: 42
nums[0]: 1
ages["Alice"]: 25
```

### 2.3 typedef vs using对比

```cpp
#include <iostream>
#include <vector>
using namespace std;

// 语法对比
typedef vector<int> IntVec1;       // typedef方式
using IntVec2 = vector<int>;       // using方式（推荐）

// 函数指针：typedef语法复杂
typedef void (*FuncPtr1)(int, int);
using FuncPtr2 = void(*)(int, int);  // using更清晰

// 数组类型
typedef int Array1[10];
using Array2 = int[10];

// const修饰的区别
typedef const int* CIntPtr1;       // 指向const int的指针
using CIntPtr2 = const int*;       // 同上

void func(int a, int b) {
    cout << a + b << endl;
}

int main() {
    IntVec1 v1 = {1, 2, 3};
    IntVec2 v2 = {4, 5, 6};

    FuncPtr1 fp1 = func;
    FuncPtr2 fp2 = func;

    fp1(1, 2);
    fp2(3, 4);

    Array1 a1 = {};
    Array2 a2 = {};

    cout << "a1大小: " << sizeof(a1) / sizeof(int) << endl;
    cout << "v2[0]: " << v2[0] << endl;

    return 0;
}
```

输出：
```
3
7
a1大小: 10
v2[0]: 4
```

### 2.4 实际应用中的类型别名

```cpp
#include <iostream>
#include <vector>
#include <cstdint>
#include <memory>
using namespace std;

// 跨平台类型别名
using byte = uint8_t;
using int32 = int32_t;
using uint32 = uint32_t;

// 智能指针别名
template<typename T>
using Ptr = shared_ptr<T>;

template<typename T>
using UniquePtr = unique_ptr<T>;

// 容器别名
template<typename T>
using List = vector<T>;

template<typename K, typename V>
using Dict = map<K, V>;

class Config {
public:
    int32 maxConnections = 100;
    byte flags = 0;
};

int main() {
    Ptr<Config> config = make_shared<Config>();
    List<int32> numbers = {1, 2, 3, 4, 5};
    Dict<string, int32> scores = {{"Alice", 95}, {"Bob", 87}};

    cout << "最大连接: " << config->maxConnections << endl;
    cout << "numbers[0]: " << numbers[0] << endl;
    cout << "Bob的分数: " << scores["Bob"] << endl;

    return 0;
}
```

输出：
```
最大连接: 100
numbers[0]: 1
Bob的分数: 87
```

## 三、注意事项与常见陷阱

1. **using更推荐**：语法更清晰，支持模板别名，是C++11以后的推荐方式
2. **typedef不创建新类型**：别名和原类型完全相同，不能重载
3. **模板别名只能用using**：`typedef`不能直接创建模板别名
4. **注意const的绑定**：`typedef const int* P`和`using P = const int*`都是指向const的指针
5. **作用域**：类型别名遵循普通的作用域规则
