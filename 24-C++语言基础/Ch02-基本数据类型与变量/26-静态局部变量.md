# 静态局部变量

## 一、概念说明

静态局部变量使用`static`关键字声明，具有**块作用域**但**静态存储期**。它只初始化一次，函数调用结束后值保持不变，下次调用时仍然存在。

## 二、具体用法

### 2.1 基本用法

```cpp
#include <iostream>
using namespace std;

void counter() {
    static int count = 0;  // 只初始化一次
    count++;
    cout << "函数被调用 " << count << " 次" << endl;
}

int main() {
    counter();  // 1
    counter();  // 2
    counter();  // 3
    counter();  // 4
    return 0;
}
```

输出：
```
函数被调用 1 次
函数被调用 2 次
函数被调用 3 次
函数被调用 4 次
```

### 2.2 懒初始化

```cpp
#include <iostream>
#include <string>
#include <vector>
using namespace std;

// 静态局部变量实现懒初始化（首次使用时才创建）
vector<string>& getConfig() {
    static vector<string> config = {  // 首次调用时初始化
        "server=localhost",
        "port=8080",
        "debug=true"
    };
    return config;
}

string& getDefaultName() {
    // 复杂对象的懒初始化
    static string name = []() {
        // 可以执行复杂的初始化逻辑
        string result = "默认";
        result += "名称";
        return result;
    }();  // 立即调用lambda
    return name;
}

int main() {
    // 首次调用：初始化config
    auto& config = getConfig();
    cout << "配置项: " << config.size() << endl;

    // 后续调用：直接返回已有对象
    auto& sameConfig = getConfig();
    sameConfig.push_back("mode=production");

    cout << "修改后配置项: " << getConfig().size() << endl;
    cout << "默认名称: " << getDefaultName() << endl;

    return 0;
}
```

输出：
```
配置项: 3
修改后配置项: 4
默认名称: 默认名称
```

### 2.3 C++11线程安全初始化

```cpp
#include <iostream>
#include <thread>
#include <vector>
using namespace std;

// C++11保证：静态局部变量的初始化是线程安全的
// 多个线程同时首次进入函数，只有一个线程执行初始化

class Logger {
public:
    Logger() {
        cout << "Logger初始化（只发生一次）" << endl;
    }
    void log(const string& msg) {
        cout << "[LOG] " << msg << endl;
    }
};

Logger& getLogger() {
    static Logger instance;  // C++11保证线程安全
    return instance;
}

void worker(int id) {
    getLogger().log("线程 " + to_string(id) + " 开始工作");
}

int main() {
    vector<thread> threads;
    for (int i = 0; i < 5; i++) {
        threads.emplace_back(worker, i);
    }
    for (auto& t : threads) t.join();

    return 0;
}
```

输出（示例）：
```
Logger初始化（只发生一次）
[LOG] 线程 0 开始工作
[LOG] 线程 1 开始工作
[LOG] 线程 2 开始工作
[LOG] 线程 3 开始工作
[LOG] 线程 4 开始工作
```

## 三、注意事项与常见陷阱

1. **只初始化一次**：`static int x = init();`中的`init()`只在首次进入时调用
2. **C++11前不保证线程安全**：多线程同时首次初始化可能导致竞争条件
3. **懒初始化替代全局变量**：静态局部变量比全局变量更安全（延迟初始化、避免静态初始化顺序问题）
4. **不可复制**：静态局部变量的引用/指针不要返回给可能长期持有者
5. **析构顺序**：静态局部变量按构造的逆序析构（在程序结束时）
