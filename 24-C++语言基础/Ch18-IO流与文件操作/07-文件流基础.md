# 文件流基础

## 一、概念说明

文件流类`<fstream>`提供文件读写能力：
- `ifstream`：输入文件流（读文件）
- `ofstream`：输出文件流（写文件）
- `fstream`：双向文件流（读写）

构造时可指定文件名和打开模式，也可以用`open()`方法后打开。

```cpp
#include <iostream>
#include <fstream>
#include <string>

int main() {
    // 写文件
    {
        std::ofstream ofs("example.txt");
        if (!ofs.is_open()) {
            std::cerr << "打开文件失败" << std::endl;
            return 1;
        }
        ofs << "第一行" << std::endl;
        ofs << "第二行" << std::endl;
        ofs << 42 << " " << 3.14 << std::endl;
    } // ofs析构自动关闭

    // 读文件
    {
        std::ifstream ifs("example.txt");
        std::string line;
        while (std::getline(ifs, line)) {
            std::cout << line << std::endl;
        }
    }

    return 0;
}
```

**输出：**
```
第一行
第二行
42 3.14
```

## 二、具体用法

### 2.1 文件操作完整流程

```cpp
#include <iostream>
#include <fstream>

int main() {
    const char* filename = "test_data.txt";

    // 1. 写入数据
    std::ofstream ofs(filename);
    if (!ofs) {
        std::cerr << "无法创建文件" << std::endl;
        return 1;
    }
    ofs << "Item1 100" << std::endl;
    ofs << "Item2 200" << std::endl;
    ofs << "Item3 300" << std::endl;
    ofs.close();
    std::cout << "写入完成" << std::endl;

    // 2. 读取数据
    std::ifstream ifs(filename);
    if (!ifs) {
        std::cerr << "无法打开文件" << std::endl;
        return 1;
    }

    std::string name;
    int value;
    int total = 0;
    while (ifs >> name >> value) {
        std::cout << name << ": " << value << std::endl;
        total += value;
    }
    std::cout << "总计: " << total << std::endl;

    return 0;
}
```

**输出：**
```
写入完成
Item1: 100
Item2: 200
Item3: 300
总计: 600
```

### 2.2 文件流的RAII特性

```cpp
#include <iostream>
#include <fstream>

void processFile() {
    std::fstream fs("data.txt", std::ios::in | std::ios::out | std::ios::trunc);

    if (!fs) {
        throw std::runtime_error("无法打开文件");
    }

    fs << "RAII保证文件关闭" << std::endl;

    fs.seekg(0); // 回到开头
    std::string line;
    std::getline(fs, line);
    std::cout << "读取: " << line << std::endl;

    // 不需要手动close，析构函数会自动关闭
}

int main() {
    try {
        processFile();
        std::cout << "文件已自动关闭" << std::endl;
    } catch (const std::exception& e) {
        std::cerr << e.what() << std::endl;
    }
    return 0;
}
```

**输出：**
```
读取: RAII保证文件关闭
文件已自动关闭
```

## 三、注意事项与常见陷阱

- **检查`is_open()`或`operator bool`**：确保文件成功打开。
- **文件流析构时自动关闭**：通常不需要手动`close()`。
- **`ofstream`默认截断文件**：使用`ios::app`追加模式。
- **文件名含中文/空格可能需要宽字符路径**：Windows下可用`std::wstring`。
- **文件锁定**：C++标准库不提供文件锁，需用平台API。
