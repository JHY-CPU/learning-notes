# IO流概述

## 一、概念说明

C++的IO流基于`<iostream>`库，提供类型安全、可扩展的输入输出机制。所有流类形成一个层次结构，以`ios_base`为基类，派生出输入流（`istream`）、输出流（`ostream`）和双向流（`iostream`）。

标准流对象：`cin`（标准输入）、`cout`（标准输出）、`cerr`（标准错误，无缓冲）、`clog`（标准日志，有缓冲）。

```cpp
#include <iostream>
#include <sstream>
#include <fstream>

int main() {
    // 标准IO
    std::cout << "标准输出" << std::endl;
    std::cerr << "标准错误（无缓冲）" << std::endl;
    std::clog << "标准日志（有缓冲）" << std::endl;

    // 字符串流
    std::stringstream ss;
    ss << "数值: " << 42 << ", 浮点: " << 3.14;
    std::cout << ss.str() << std::endl;

    // 文件流
    std::ofstream ofs("temp.txt");
    ofs << "文件内容" << std::endl;
    ofs.close();

    std::ifstream ifs("temp.txt");
    std::string line;
    std::getline(ifs, line);
    std::cout << "读取: " << line << std::endl;

    return 0;
}
```

**输出：**
```
标准输出
标准错误（无缓冲）
标准日志（有缓冲）
数值: 42, 浮点: 3.14
读取: 文件内容
```

## 二、具体用法

### 2.1 流类层次结构

```
ios_base
  └── basic_ios<char>
        ├── basic_istream<char> (istream)
        │     └── basic_ifstream<char> (ifstream)
        │     └── basic_istringstream<char> (istringstream)
        ├── basic_ostream<char> (ostream)
        │     └── basic_ofstream<char> (ofstream)
        │     └── basic_ostringstream<char> (ostringstream)
        └── basic_iostream<char> (iostream)
              └── basic_fstream<char> (fstream)
              └── basic_stringstream<char> (stringstream)
```

### 2.2 流的基本操作

```cpp
#include <iostream>

int main() {
    int a;
    double b;
    std::string s;

    std::cout << "输入一个整数、一个浮点数和一个字符串: ";
    std::cin >> a >> b >> s;

    std::cout << "整数: " << a << std::endl;
    std::cout << "浮点: " << b << std::endl;
    std::cout << "字符串: " << s << std::endl;

    // 检查流状态
    if (std::cin.good()) {
        std::cout << "输入成功" << std::endl;
    }

    return 0;
}
```

**输出（示例）：**
```
输入一个整数、一个浮点数和一个字符串: 42 3.14 hello
整数: 42
浮点: 3.14
字符串: hello
输入成功
```

## 三、注意事项与常见陷阱

- **`std::endl`会刷新缓冲区**：频繁使用影响性能，用`'\n'`代替。
- **`cin >>`跳过空白字符**：`getline`可以读取含空格的整行。
- **流失败后不再接受输入**：需调用`clear()`和`ignore()`恢复。
- **`cerr`无缓冲**：适合紧急错误信息，确保立即输出。
- **头文件`<iostream>`自动包含`<istream>`和`<ostream>`**。
