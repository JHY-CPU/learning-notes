# 管道与重定向

## 一、概念说明

标准流（`stdin`/`stdout`/`stderr`）可以被重定向到文件或其他程序（管道）。C++程序无需修改代码即可利用shell的重定向功能，也可以在程序内用`freopen`或`dup2`重定向。

Shell重定向：`>`输出到文件，`<`从文件输入，`|`管道连接。

```cpp
#include <iostream>
#include <string>

int main() {
    // 程序不需要知道自己是被管道调用还是直接运行
    // stdin/stdout自动处理

    std::string line;
    std::cout << "请输入（EOF结束）:" << std::endl;

    while (std::getline(std::cin, line)) {
        std::cout << ">> " << line << std::endl;
    }

    std::cout << "输入结束" << std::endl;
    return 0;
}
```

**Shell用法：**
```bash
# 直接运行
./program

# 从文件输入
./program < input.txt

# 输出到文件
./program > output.txt

# 管道
echo "hello" | ./program
cat file.txt | ./program | sort
```

## 二、具体用法

### 2.1 程序内重定向

```cpp
#include <iostream>
#include <cstdio>

int main() {
    // 保存原始stdout
    auto* original = stdout;

    // 重定向stdout到文件
    std::freopen("output.txt", "w", stdout);
    std::cout << "这写入到文件" << std::endl;
    printf("C风格输出也到文件\n");

    // 恢复stdout（Windows/POSIX方式略有不同）
    // stdout = original; // POSIX
    std::freopen("CON", "w", stdout); // Windows

    std::cout << "这显示在屏幕上" << std::endl;

    return 0;
}
```

### 2.2 管道通信（POSIX）

```cpp
#include <iostream>
#include <cstdio>
#include <array>

int main() {
    // 使用popen执行命令并读取输出
    FILE* pipe = popen("ls -la", "r"); // Windows用"dir"
    if (!pipe) {
        std::cerr << "popen失败" << std::endl;
        return 1;
    }

    std::array<char, 256> buffer;
    std::cout << "命令输出:" << std::endl;
    while (fgets(buffer.data(), buffer.size(), pipe)) {
        std::cout << buffer.data();
    }

    int status = pclose(pipe);
    std::cout << "\n退出状态: " << status << std::endl;

    return 0;
}
```

**输出（示例）：**
```
命令输出:
total 48
drwxr-xr-x  2 user group 4096 May  6 10:30 .
drwxr-xr-x 10 user group 4096 May  6 10:00 ..
-rw-r--r--  1 user group 1234 May  6 10:30 main.cpp

退出状态: 0
```

### 2.3 写入管道

```cpp
#include <iostream>
#include <cstdio>

int main() {
    // 使用popen写入到另一个程序
    FILE* pipe = popen("sort", "w");
    if (!pipe) return 1;

    fprintf(pipe, "banana\n");
    fprintf(pipe, "apple\n");
    fprintf(pipe, "cherry\n");

    pclose(pipe); // sort的输出到stdout
    return 0;
}
```

**输出：**
```
apple
banana
cherry
```

## 三、注意事项与常见陷阱

- **`freopen`后原来的流指针无效**：恢复需要保存文件描述符。
- **`popen`在Windows上有限制**：只能单向通信。
- **管道缓冲区满会阻塞**：生产者和消费者需要协调。
- **`pclose`等待子进程结束**：确保所有输出已读取。
- **标准错误`stderr`不受重定向影响**：需要单独处理。
