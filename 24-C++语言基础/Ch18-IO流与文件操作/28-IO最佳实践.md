# IO最佳实践

## 一、概念说明

C++ IO操作的最佳实践总结：性能优化、错误处理、资源管理、跨平台兼容性。遵循这些原则可以编写高效、健壮的IO代码。

```cpp
#include <iostream>
#include <fstream>
#include <string>

// 1. 使用RAII管理文件资源
class FileGuard {
    std::fstream fs;
public:
    FileGuard(const std::string& path, std::ios::openmode mode)
        : fs(path, mode) {
        if (!fs) throw std::runtime_error("无法打开: " + path);
    }
    std::fstream& stream() { return fs; }
    ~FileGuard() { /* 自动关闭 */ }
};

// 2. 安全的文件读取
std::string safeRead(const std::string& path) {
    std::ifstream ifs(path, std::ios::binary);
    if (!ifs) throw std::runtime_error("读取失败: " + path);

    ifs.seekg(0, std::ios::end);
    auto size = ifs.tellg();
    ifs.seekg(0, std::ios::beg);

    std::string content(size, '\0');
    ifs.read(&content[0], size);
    return content;
}

int main() {
    // 写入测试
    {
        std::ofstream ofs("test.txt");
        ofs << "测试内容\n第二行";
    }

    // 安全读取
    auto content = safeRead("test.txt");
    std::cout << content << std::endl;

    return 0;
}
```

**输出：**
```
测试内容
第二行
```

## 二、具体用法

### 2.1 性能优化清单

```cpp
#include <iostream>
#include <fstream>
#include <vector>

int main() {
    // 好的做法
    std::ios::sync_with_stdio(false); // 禁用同步
    std::cin.tie(nullptr);             // 解除绑定

    // 1. 使用'\n'代替endl
    std::cout << "快速输出\n";

    // 2. 批量读取代替逐字节
    std::ifstream ifs("data.bin", std::ios::binary);
    std::vector<char> buffer(8192);
    while (ifs.read(buffer.data(), buffer.size()) || ifs.gcount()) {
        // 处理 buffer.data(), 大小为 ifs.gcount()
    }

    // 3. 预留容器空间
    std::vector<std::string> lines;
    lines.reserve(1000); // 预留减少重新分配

    // 4. 使用ostringstream构建字符串
    std::ostringstream oss;
    oss.reserve(4096);
    for (int i = 0; i < 100; ++i) {
        oss << "Line " << i << '\n';
    }
    std::string result = oss.str();

    // 5. 文件IO用大缓冲区
    std::ofstream ofs("big.txt");
    char largeBuf[65536];
    ofs.rdbuf()->pubsetbuf(largeBuf, sizeof(largeBuf));

    std::cout << "IO优化配置完成\n";
    return 0;
}
```

**输出：**
```
快速输出
IO优化配置完成
```

### 2.2 错误处理清单

```cpp
#include <iostream>
#include <fstream>

int main() {
    // 1. 总是检查文件是否打开
    std::ifstream ifs("data.txt");
    if (!ifs.is_open()) {
        std::cerr << "文件打开失败\n";
        return 1;
    }

    // 2. 检查读取结果
    int val;
    if (ifs >> val) {
        std::cout << "读取: " << val << '\n';
    } else if (ifs.eof()) {
        std::cout << "文件为空\n";
    } else {
        std::cout << "读取错误\n";
        ifs.clear();
    }

    // 3. 使用异常（可选）
    // ifs.exceptions(std::ios::failbit | std::ios::badbit);

    // 4. 检查写入结果
    std::ofstream ofs("output.txt");
    ofs << "数据\n";
    if (!ofs) {
        std::cerr << "写入失败\n";
    }

    // 5. 跨平台路径
    // 使用 std::filesystem::path 代替硬编码分隔符

    return 0;
}
```

**输出（假设data.txt存在且包含数字）：**
```
读取: <文件中的数字>
```

### 2.3 最佳实践总结

| 方面 | 推荐做法 |
|------|---------|
| 缓冲 | 用`'\n'`代替`endl` |
| 同步 | 竞赛编程中关闭`sync_with_stdio` |
| 文件 | 用RAII管理，检查打开状态 |
| 二进制 | 始终使用`ios::binary` |
| 路径 | 用`std::filesystem::path` |
| 大文件 | 用内存映射或流式处理 |
| 错误 | 检查流状态，区分EOF和错误 |
| 编码 | 用UTF-8，避免`<codecvt>` |

## 三、注意事项与常见陷阱

- **关闭同步后不要混用`printf`/`cout`**：输出顺序不确定。
- **`tellg()`在文本模式下可能不准确**：用二进制模式获取准确位置。
- **大文件不要全部加载到内存**：使用流式或内存映射。
- **文件句柄是有限资源**：RAII确保及时释放。
- **`std::endl`不只是`'\n'`**：还会刷新缓冲区，性能差距可达10倍。
