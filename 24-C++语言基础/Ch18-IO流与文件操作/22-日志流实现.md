# 日志流实现

## 一、概念说明

基于IO流实现的日志系统支持级别过滤、时间戳、文件/控制台输出切换。利用流的链式特性和自定义缓冲区，可以构建灵活的日志框架。

日志级别：TRACE < DEBUG < INFO < WARN < ERROR < FATAL

```cpp
#include <iostream>
#include <fstream>
#include <chrono>
#include <ctime>
#include <iomanip>

enum class LogLevel { DEBUG, INFO, WARN, ERROR };

class Logger {
    std::ostream* out;
    LogLevel minLevel;

    static const char* levelStr(LogLevel l) {
        switch (l) {
            case LogLevel::DEBUG: return "DEBUG";
            case LogLevel::INFO:  return "INFO";
            case LogLevel::WARN:  return "WARN";
            case LogLevel::ERROR: return "ERROR";
        }
        return "UNKNOWN";
    }

    static std::string timestamp() {
        auto now = std::chrono::system_clock::now();
        auto t = std::chrono::system_clock::to_time_t(now);
        std::tm tm = *std::localtime(&t);
        std::ostringstream oss;
        oss << std::put_time(&tm, "%Y-%m-%d %H:%M:%S");
        return oss.str();
    }

public:
    Logger(std::ostream& os, LogLevel level = LogLevel::DEBUG)
        : out(&os), minLevel(level) {}

    class LogEntry {
        std::ostream* os;
    public:
        LogEntry(std::ostream* o) : os(o) {}
        template <typename T>
        LogEntry& operator<<(const T& val) {
            *os << val;
            return *this;
        }
        ~LogEntry() { *os << std::endl; }
    };

    LogEntry log(LogLevel level) {
        if (level >= minLevel) {
            *out << "[" << timestamp() << "] [" << levelStr(level) << "] ";
        }
        return LogEntry(out);
    }
};

int main() {
    Logger logger(std::cout, LogLevel::DEBUG);

    logger.log(LogLevel::DEBUG) << "调试信息: 变量x=" << 42;
    logger.log(LogLevel::INFO)  << "应用启动完成";
    logger.log(LogLevel::WARN)  << "配置文件未找到，使用默认值";
    logger.log(LogLevel::ERROR) << "连接数据库失败: 超时";

    return 0;
}
```

**输出（示例）：**
```
[2026-05-06 10:30:00] [DEBUG] 调试信息: 变量x=42
[2026-05-06 10:30:00] [INFO] 应用启动完成
[2026-05-06 10:30:00] [WARN] 配置文件未找到，使用默认值
[2026-05-06 10:30:00] [ERROR] 连接数据库失败: 超时
```

## 二、具体用法

### 2.1 宏简化日志调用

```cpp
#include <iostream>
#include <fstream>

// 简化的日志宏
#define LOG_DEBUG(logger) logger.log(LogLevel::DEBUG)
#define LOG_INFO(logger)  logger.log(LogLevel::INFO)
#define LOG_ERROR(logger) logger.log(LogLevel::ERROR)

// 使用示例
int main() {
    Logger logger(std::cout);

    LOG_INFO(logger) << "服务器启动在端口 " << 8080;
    LOG_DEBUG(logger) << "内存使用: " << 1024 << " MB";
    LOG_ERROR(logger) << "文件不存在: " << "config.ini";

    return 0;
}
```

**输出（示例）：**
```
[2026-05-06 10:30:00] [INFO] 服务器启动在端口 8080
[2026-05-06 10:30:00] [DEBUG] 内存使用: 1024 MB
[2026-05-06 10:30:00] [ERROR] 文件不存在: config.ini
```

## 三、注意事项与常见陷阱

- **日志输出不是线程安全的**：多线程需加锁或使用无锁队列。
- **`endl`刷新缓冲区**：日志中用`'\n'`或最后统一flush以提高性能。
- **级别过滤应在构造LogEntry之前判断**：避免无用的格式化开销。
- **文件日志应考虑轮转**：按大小或日期分割日志文件。
- **避免在热路径中做复杂日志**：即使是DEBUG级别也有开销。
