# 文件打开模式

## 一、概念说明

`std::ios`定义了文件打开模式标志，可组合使用：

| 模式 | 说明 |
|------|------|
| `ios::in` | 读模式（ifstream默认） |
| `ios::out` | 写模式（ofstream默认，截断文件） |
| `ios::app` | 追加模式（在尾部写入） |
| `ios::ate` | 打开后定位到文件尾 |
| `ios::trunc` | 截断已有文件 |
| `ios::binary` | 二进制模式 |

```cpp
#include <iostream>
#include <fstream>

int main() {
    // 截断写入（默认）
    {
        std::ofstream ofs("test.txt");
        ofs << "第一行" << std::endl;
    }

    // 追加写入
    {
        std::ofstream ofs("test.txt", std::ios::app);
        ofs << "第二行" << std::endl;
    }

    // 读写模式
    {
        std::fstream fs("test.txt", std::ios::in | std::ios::out);
        std::string line;
        while (std::getline(fs, line)) {
            std::cout << line << std::endl;
        }
    }

    // 读写+创建（不存在则创建）
    {
        std::fstream fs("newfile.txt",
            std::ios::in | std::ios::out | std::ios::trunc);
        fs << "新建文件内容" << std::endl;
        fs.seekg(0);
        std::string s;
        std::getline(fs, s);
        std::cout << s << std::endl;
    }

    return 0;
}
```

**输出：**
```
第一行
第二行
新建文件内容
```

## 二、具体用法

### 2.1 模式组合规则

```cpp
#include <iostream>
#include <fstream>

int main() {
    // ios::app 隐含 ios::out
    // ios::trunc 需要 ios::out

    // 1. 只读
    std::ifstream ifs1("test.txt", std::ios::in);

    // 2. 只写（截断）
    std::ofstream ofs1("test.txt", std::ios::out);

    // 3. 追加写入
    std::ofstream ofs2("log.txt", std::ios::app);

    // 4. 读写不截断
    std::fstream fs1("data.txt",
        std::ios::in | std::ios::out);

    // 5. 读写+截断
    std::fstream fs2("data.txt",
        std::ios::in | std::ios::out | std::ios::trunc);

    // 6. 二进制读写
    std::fstream fs3("bin.dat",
        std::ios::in | std::ios::out | std::ios::binary);

    std::cout << "各种模式文件已配置" << std::endl;
    return 0;
}
```

**输出：**
```
各种模式文件已配置
```

### 2.2 ate vs app区别

```cpp
#include <iostream>
#include <fstream>

int main() {
    // app: 每次写入都在尾部（不可定位到其他位置写入）
    {
        std::ofstream ofs("app_test.txt", std::ios::app);
        ofs << "尾部写入";
        ofs.seekp(0); // 定位到开头
        ofs << "仍然写在尾部"; // app模式下seekp无效
    }

    // ate: 打开时在尾部，但可以定位到其他位置
    {
        std::ofstream ofs("ate_test.txt", std::ios::out | std::ios::ate);
        ofs << "初始在尾部";
        ofs.seekp(0); // 可以定位到开头
        ofs << "覆盖开头";
    }

    // 验证
    std::ifstream ifs("ate_test.txt");
    std::string s;
    std::getline(ifs, s);
    std::cout << "ate模式结果: " << s << std::endl;

    return 0;
}
```

**输出：**
```
ate模式结果: 覆盖开头尾部
```

## 三、注意事项与常见陷阱

- **`ofstream`默认`ios::out | ios::trunc`**：会清空已有文件。
- **`app`模式写入不可控位置**：始终在文件尾部。
- **`ate`模式只影响初始位置**：之后可以任意定位。
- **二进制和文本模式不能混用**：同一文件应用一致的模式打开。
- **`ios::in | ios::out`不会创建文件**：文件不存在时打开失败。
