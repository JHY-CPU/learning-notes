# 文件状态检测

## 一、概念说明

流对象维护一组状态位，用于检测IO操作的结果：
- `good()`：一切正常
- `eof()`：到达文件尾
- `fail()`：逻辑错误（类型不匹配、文件打开失败）
- `bad()`：严重错误（硬件故障、流损坏）
- `rdstate()`：返回当前状态位
- `clear()`：重置状态位

```cpp
#include <iostream>
#include <fstream>

int main() {
    std::ifstream ifs("test.txt");

    // 检查打开状态
    if (!ifs) { // 等价于 if (ifs.fail())
        std::cout << "文件打开失败" << std::endl;
        return 1;
    }
    std::cout << "文件打开成功" << std::endl;
    std::cout << "good=" << ifs.good() << ", eof=" << ifs.eof()
              << ", fail=" << ifs.fail() << ", bad=" << ifs.bad() << std::endl;

    // 读取直到EOF
    int val;
    while (ifs >> val) {
        std::cout << "读取: " << val << std::endl;
    }

    std::cout << "读取结束后状态:" << std::endl;
    std::cout << "good=" << ifs.good() << ", eof=" << ifs.eof()
              << ", fail=" << ifs.fail() << ", bad=" << ifs.bad() << std::endl;

    // 区分EOF和错误
    if (ifs.eof()) {
        std::cout << "正常到达文件尾" << std::endl;
    } else if (ifs.fail()) {
        std::cout << "读取错误" << std::endl;
    }

    return 0;
}
```

**输出（假设test.txt包含"10 20 30"）：**
```
文件打开成功
good=1, eof=0, fail=0, bad=0
读取: 10
读取: 20
读取: 30
读取结束后状态:
good=0, eof=1, fail=1, bad=0
正常到达文件尾
```

## 二、具体用法

### 2.1 错误恢复

```cpp
#include <iostream>
#include <sstream>
#include <limits>

int main() {
    std::istringstream iss("42 abc 99");
    int val;

    while (true) {
        if (iss >> val) {
            std::cout << "读取: " << val << std::endl;
        } else if (iss.eof()) {
            std::cout << "到达末尾" << std::endl;
            break;
        } else {
            std::cout << "输入错误，跳过..." << std::endl;
            iss.clear(); // 清除错误状态
            iss.ignore(std::numeric_limits<std::streamsize>::max(), ' ');
        }
    }

    return 0;
}
```

**输出：**
```
读取: 42
输入错误，跳过...
读取: 99
到达末尾
```

### 2.2 状态查询方法总结

| 方法 | 含义 | 何时为true |
|------|------|-----------|
| `good()` | 无错误 | 所有位都为0 |
| `eof()` | 到达文件尾 | 读取超过文件末尾 |
| `fail()` | 逻辑错误 | 类型不匹配、文件打开失败 |
| `bad()` | 严重错误 | 流损坏、IO硬件错误 |
| `operator bool()` | 可用 | 等价于`!fail()` |
| `operator!()` | 不可用 | 等价于`fail()` |

## 三、注意事项与常见陷阱

- **`eof`和`fail`常同时为true**：最后一次读取失败会同时设置两个位。
- **`clear()`只重置状态，不丢弃数据**：需配合`ignore()`跳过坏数据。
- **`bad()`比`fail()`更严重**：`bad`表示流本身不可用，`fail`只是操作失败。
- **`operator bool()`是最简洁的检查方式**：`if (stream)`等价于`if (stream.good())`。
- **错误后不`clear()`则后续操作全部失败**：必须先恢复状态。
