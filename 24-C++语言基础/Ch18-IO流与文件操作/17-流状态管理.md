# 流状态管理

## 一、概念说明

流的状态由`iostate`位掩码控制，包含四个位：`goodbit`(0)、`eofbit`(1)、`failbit`(2)、`badbit`(4)。操作失败时设置相应位，后续IO操作会跳过直到状态被`clear()`清除。

流的异常机制允许将状态位转换为异常，通过`exceptions()`方法设置。

```cpp
#include <iostream>
#include <sstream>

int main() {
    std::istringstream iss("42 abc");
    int val;

    iss >> val;
    std::cout << "第一次读取: " << val << ", good=" << iss.good() << std::endl;

    iss >> val; // 类型不匹配
    std::cout << "第二次读取失败: fail=" << iss.fail() << ", eof=" << iss.eof() << std::endl;

    // 清除错误状态
    iss.clear();
    std::string s;
    iss >> s;
    std::cout << "清除后读取: " << s << std::endl;

    return 0;
}
```

**输出：**
```
第一次读取: 42, good=1
第二次读取失败: fail=1, eof=0
清除后读取: abc
```

## 二、具体用法

### 2.1 流异常机制

```cpp
#include <iostream>
#include <sstream>
#include <stdexcept>

int main() {
    std::istringstream iss("hello");
    int val;

    // 设置异常：fail时抛异常
    iss.exceptions(std::ios::failbit | std::ios::badbit);

    try {
        iss >> val; // "hello"不能解析为int
    } catch (const std::ios_base::failure& e) {
        std::cout << "捕获IO异常: " << e.what() << std::endl;
        iss.clear(); // 必须清除异常状态
    }

    // 恢复后继续使用
    iss.clear();
    iss.seekg(0);
    std::string s;
    iss >> s;
    std::cout << "恢复后: " << s << std::endl;

    return 0;
}
```

**输出：**
```
捕获IO异常: basic_ios::clear
恢复后: hello
```

### 2.2 状态位操作

```cpp
#include <iostream>
#include <sstream>

int main() {
    std::istringstream iss("42");
    int a, b;

    iss >> a;
    std::cout << "读取a=" << a << std::endl;

    iss >> b; // 失败：没有更多数据
    std::cout << "读取b状态: " << iss.rdstate() << std::endl;

    // 检查具体状态
    if (iss.eof()) std::cout << "已到文件尾" << std::endl;
    if (iss.fail()) std::cout << "读取失败" << std::endl;

    // 设置和清除特定状态位
    iss.clear(iss.rdstate() & ~std::ios::failbit); // 只清除fail位
    std::cout << "清除fail后 rdstate=" << iss.rdstate() << std::endl;

    // 重置所有状态
    iss.clear();
    std::cout << "完全重置后 good=" << iss.good() << std::endl;

    return 0;
}
```

**输出：**
```
读取a=42
读取b状态: 3
已到文件尾
读取失败
清除fail后 rdstate=1
完全重置后 good=1
```

## 三、注意事项与常见陷阱

- **`eof`和`fail`常同时设置**：读取到最后一个数据后继续读。
- **`clear()`不丢弃输入缓冲区中的数据**：需配合`ignore()`。
- **异常模式适合快速失败场景**：默认不抛异常。
- **`rdstate()`返回原始位掩码**：可用于精确控制状态。
- **`setstate(s)`相当于`clear(rdstate() | s)`**：设置额外的状态位。
