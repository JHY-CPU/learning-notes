# 内存映射IO

## 一、概念说明

内存映射IO将文件直接映射到进程的虚拟地址空间，避免了`read`/`write`的用户态-内核态数据拷贝。适合大文件处理、数据库实现、高性能日志等场景。

与普通IO相比：减少系统调用、操作系统自动管理换页、支持随机访问。

```cpp
#ifdef _WIN32
#include <windows.h>
#else
#include <sys/mman.h>
#include <sys/stat.h>
#include <fcntl.h>
#include <unistd.h>
#endif
#include <iostream>

int main() {
    // 创建测试文件
    const char* filename = "mapped.dat";
    const size_t SIZE = 4096;

#ifdef __linux__
    // 创建并初始化文件
    int fd = open(filename, O_RDWR | O_CREAT | O_TRUNC, 0666);
    ftruncate(fd, SIZE);

    // 内存映射
    char* mapped = static_cast<char*>(
        mmap(nullptr, SIZE, PROT_READ | PROT_WRITE, MAP_SHARED, fd, 0));

    if (mapped != MAP_FAILED) {
        // 直接通过指针写入文件
        std::strcpy(mapped, "通过内存映射写入的数据");
        std::cout << "写入: " << mapped << std::endl;

        // 同步到磁盘
        msync(mapped, SIZE, MS_SYNC);

        // 读取验证
        std::cout << "读取: " << mapped << std::endl;

        munmap(mapped, SIZE);
    }

    close(fd);
#endif

    std::cout << "内存映射IO完成" << std::endl;
    return 0;
}
```

**输出：**
```
写入: 通过内存映射写入的数据
读取: 通过内存映射写入的数据
内存映射IO完成
```

## 二、具体用法

### 2.1 大文件随机访问

```cpp

#include <cstring>

// 概念性示例：用内存映射实现大文件数据库
// 实际代码需要平台特定API

class MappedDB {
```

## 三、注意事项与常见陷阱

- **映射大小受虚拟地址空间限制**：32位系统最大约2-3GB。
- **文件大小为0时不能映射**：需先`ftruncate`扩展。
- **`MAP_SHARED`修改会写回磁盘**：`MAP_PRIVATE`使用写时复制。
- **映射解除后指针失效**：继续访问会导致段错误。
- **对齐要求**：偏移必须是页大小的整数倍。
- **Windows实现差异大**：`CreateFileMapping` + `MapViewOfFile`。
