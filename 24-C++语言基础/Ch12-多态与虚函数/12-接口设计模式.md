# 接口设计模式

## 一、概念说明

**接口类**（C++标准 §10.4）只定义纯虚函数，无数据成员，类似Java/C#的`interface`。接口定义行为契约，实现类提供具体行为。多接口继承实现"多重继承"的功能而不引入菱形问题。

### 1.1 接口类特征

| 特征 | 要求 |
|------|------|
| 纯虚函数 | 所有函数都是`virtual ... = 0` |
| 数据成员 | 无（约定） |
| 析构函数 | `virtual ~Interface() = default;` |
| 构造函数 | 只有默认构造 |
| 多继承 | 安全（无状态，无菱形问题） |

## 二、具体用法

### 2.1 接口定义与实现

```cpp
#include <iostream>
#include <vector>
#include <memory>
#include <string>

// 接口类
class IReadable {
public:
    virtual std::string read() = 0;
    virtual ~IReadable() = default;
};

class IWritable {
public:
    virtual void write(const std::string& data) = 0;
    virtual ~IWritable() = default;
};

class ICloseable {
public:
    virtual void close() = 0;
    virtual ~ICloseable() = default;
};

// 实现多个接口
class FileIO : public IReadable, public IWritable, public ICloseable {
    std::string filename;
    bool isOpen = false;
public:
    FileIO(const std::string& f) : filename(f), isOpen(true) {
        std::cout << "打开文件: " << filename << std::endl;
    }

    std::string read() override {
        return isOpen ? "文件内容[" + filename + "]" : "文件未打开";
    }

    void write(const std::string& data) override {
        if (isOpen)
            std::cout << "写入" << filename << ": " << data << std::endl;
    }

    void close() override {
        isOpen = false;
        std::cout << "关闭文件: " << filename << std::endl;
    }
};

// 函数只依赖需要的接口
void copyData(IReadable& source, IWritable& dest) {
    auto data = source.read();
    dest.write(data);
}

int main() {
    FileIO src("source.txt");
    FileIO dst("dest.txt");

    copyData(src, dst);  // 只需要IReadable和IWritable

    // 按接口管理
    std::vector<std::unique_ptr<ICloseable>> resources;
    resources.push_back(std::make_unique<FileIO>("log.txt"));
    for (auto& r : resources)
        r->close();

    return 0;
}
```

### 2.2 接口与依赖注入

```cpp
#include <iostream>
#include <memory>

class ILogger {
public:
    virtual void info(const std::string& msg) = 0;
    virtual void error(const std::string& msg) = 0;
    virtual ~ILogger() = default;
};

class IDatabase {
public:
    virtual bool execute(const std::string& sql) = 0;
    virtual ~IDatabase() = default;
};

// 具体实现
class ConsoleLogger : public ILogger {
public:
    void info(const std::string& msg) override {
        std::cout << "[INFO] " << msg << std::endl;
    }
    void error(const std::string& msg) override {
        std::cerr << "[ERROR] " << msg << std::endl;
    }
};

class MockDatabase : public IDatabase {
public:
    bool execute(const std::string& sql) override {
        std::cout << "[MOCK] 执行: " << sql << std::endl;
        return true;
    }
};

// 业务逻辑依赖接口，不依赖具体实现
class UserService {
    ILogger& logger;
    IDatabase& db;
public:
    UserService(ILogger& l, IDatabase& d) : logger(l), db(d) {}

    void createUser(const std::string& name) {
        logger.info("创建用户: " + name);
        if (!db.execute("INSERT INTO users VALUES('" + name + "')"))
            logger.error("创建失败");
    }
};

int main() {
    ConsoleLogger logger;
    MockDatabase db;
    UserService service(logger, db);

    service.createUser("张三");
    return 0;
}
```

### 2.3 接口版本管理

```cpp
#include <iostream>

// 基础接口
class IService {
public:
    virtual void process() = 0;
    virtual ~IService() = default;
};

// 扩展接口
class IServiceV2 : public IService {
public:
    virtual void processAsync() = 0;
    virtual int version() const { return 2; }
};

// 进一步扩展
class IServiceV3 : public IServiceV2 {
public:
    virtual void processBatch(int count) = 0;
    int version() const override { return 3; }
};

class MyService : public IServiceV3 {
public:
    void process() override { std::cout << "同步处理" << std::endl; }
    void processAsync() override { std::cout << "异步处理" << std::endl; }
    void processBatch(int count) override {
        std::cout << "批量处理" << count << "个" << std::endl;
    }
};

int main() {
    MyService svc;

    // 不同版本的接口
    IService* v1 = &svc;
    v1->process();

    IServiceV2* v2 = &svc;
    v2->processAsync();
    std::cout << "版本: " << v2->version() << std::endl;

    IServiceV3* v3 = &svc;
    v3->processBatch(10);

    return 0;
}
```

## 三、注意事项与常见陷阱

1. **接口类不应有数据成员**：Java/C#强制，C++靠约定
2. **虚析构函数必须有实现**：`= default`即可
3. **多接口继承不会产生菱形问题**：没有数据成员
4. **接口用于解耦和依赖注入**：使代码可测试、可替换
5. **接口命名常以`I`前缀**：如`IDatabase`、`ILogger`（约定非强制）
6. **接口接口隔离原则**：接口应小而精，避免"胖接口"
