# 接口设计模式

## 一、概念说明

**接口类**只定义纯虚函数，无数据成员，类似Java/C#的`interface`。接口定义行为契约，实现类提供具体行为。多接口继承实现"多重继承"的功能而不引入菱形问题。

## 二、具体用法

### 2.1 接口与实现

```cpp
#include <iostream>
#include <vector>
#include <memory>

// 接口
class IReadable {
public:
    virtual std::string read() = 0;
    virtual ~IReadable() = default;
};

class IWritable {
public:
    virtual void write(const std::string& data) = 0;
    virtual ~IWritable() = default;
};

// 实现
class FileIO : public IReadable, public IWritable {
    std::string filename;
public:
    FileIO(const std::string& f) : filename(f) {}
    std::string read() override { return "文件内容:" + filename; }
    void write(const std::string& data) override {
        std::cout << "写入" << filename << ": " << data << std::endl;
    }
};

int main() {
    FileIO file("data.txt");
    IReadable* r = &file;
    IWritable* w = &file;
    std::cout << r->read() << std::endl;
    w->write("Hello");
    return 0;
}
```

**输出：**
```
文件内容:data.txt
写入data.txt: Hello
```

## 三、注意事项与常见陷阱

- 接口类不应有数据成员
- 虚析构函数必须有实现
- 接口用于解耦和依赖注入
- 接口继承不会产生菱形问题（无状态）
