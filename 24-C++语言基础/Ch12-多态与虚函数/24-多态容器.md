# 多态容器

## 一、概念说明

存储多态对象需要使用**指针或引用**（C++标准 §10.2），因为容器按值存储会发生对象切片。常用方案是`std::vector<std::unique_ptr<Base>>`，结合智能指针自动管理内存。

### 1.1 容器方案对比

| 方案 | 是否切片 | 内存管理 | 性能 |
|------|---------|---------|------|
| `vector<Base>` | 是（切片） | 自动 | 好 |
| `vector<Base*>` | 否 | 手动 | 好 |
| `vector<unique_ptr<Base>>` | 否 | 自动 | 好 |
| `vector<shared_ptr<Base>>` | 否 | 自动（引用计数） | 略慢 |
| `vector<reference_wrapper<Base>>` | 否 | 不管理 | 好 |

## 二、具体用法

### 2.1 unique_ptr多态容器

```cpp
#include <iostream>
#include <vector>
#include <memory>
#include <algorithm>

class Shape {
public:
    virtual void draw() const = 0;
    virtual double area() const = 0;
    virtual std::unique_ptr<Shape> clone() const = 0;
    virtual ~Shape() = default;
};

class Circle : public Shape {
    double r;
public:
    Circle(double r) : r(r) {}
    void draw() const override { std::cout << "绘制圆(r=" << r << ")" << std::endl; }
    double area() const override { return 3.14159 * r * r; }
    std::unique_ptr<Shape> clone() const override {
        return std::make_unique<Circle>(*this);
    }
};

class Rectangle : public Shape {
    double w, h;
public:
    Rectangle(double w, double h) : w(w), h(h) {}
    void draw() const override { std::cout << "绘制矩形(" << w << "x" << h << ")" << std::endl; }
    double area() const override { return w * h; }
    std::unique_ptr<Shape> clone() const override {
        return std::make_unique<Rectangle>(*this);
    }
};

int main() {
    std::vector<std::unique_ptr<Shape>> shapes;
    shapes.push_back(std::make_unique<Circle>(5));
    shapes.push_back(std::make_unique<Rectangle>(3, 4));
    shapes.push_back(std::make_unique<Circle>(2));

    // 遍历
    double totalArea = 0;
    for (const auto& s : shapes) {
        s->draw();
        totalArea += s->area();
    }
    std::cout << "总面积: " << totalArea << std::endl;

    // 排序（按面积）
    std::sort(shapes.begin(), shapes.end(),
        [](const auto& a, const auto& b) { return a->area() < b->area(); });

    std::cout << "\n排序后:" << std::endl;
    for (const auto& s : shapes)
        s->draw();

    // 深拷贝
    std::vector<std::unique_ptr<Shape>> copies;
    for (const auto& s : shapes)
        copies.push_back(s->clone());

    return 0;
}
```

### 2.2 reference_wrapper方案

```cpp
#include <iostream>
#include <vector>
#include <functional>

class Shape {
public:
    virtual void draw() const = 0;
    virtual ~Shape() = default;
};

class Circle : public Shape {
public:
    void draw() const override { std::cout << "圆" << std::endl; }
};

class Rectangle : public Shape {
public:
    void draw() const override { std::cout << "矩形" << std::endl; }
};

int main() {
    Circle c;
    Rectangle r;

    // reference_wrapper存储引用而非拷贝
    std::vector<std::reference_wrapper<Shape>> shapes;
    shapes.push_back(c);
    shapes.push_back(r);

    for (auto& s : shapes)
        s.get().draw();  // 注意：需要.get()

    return 0;
}
```

### 2.3 多态容器的常见操作

```cpp
#include <iostream>
#include <vector>
#include <memory>
#include <algorithm>

class Animal {
public:
    virtual void speak() const = 0;
    virtual std::string name() const = 0;
    virtual ~Animal() = default;
};

class Dog : public Animal {
public:
    void speak() const override { std::cout << "汪汪!" << std::endl; }
    std::string name() const override { return "狗"; }
};

class Cat : public Animal {
public:
    void speak() const override { std::cout << "喵喵!" << std::endl; }
    std::string name() const override { return "猫"; }
};

int main() {
    std::vector<std::unique_ptr<Animal>> zoo;
    zoo.push_back(std::make_unique<Dog>());
    zoo.push_back(std::make_unique<Cat>());
    zoo.push_back(std::make_unique<Dog>());

    // 查找
    auto it = std::find_if(zoo.begin(), zoo.end(),
        [](const auto& a) { return a->name() == "猫"; });
    if (it != zoo.end())
        (*it)->speak();

    // 移除
    zoo.erase(
        std::remove_if(zoo.begin(), zoo.end(),
            [](const auto& a) { return a->name() == "狗"; }),
        zoo.end());

    std::cout << "剩余: " << zoo.size() << "个动物" << std::endl;

    return 0;
}
```

## 三、注意事项与常见陷阱

1. **不要使用`std::vector<Base>`（会切片）**：这是最常见的错误
2. **使用`std::unique_ptr`自动管理内存**：无需手动delete
3. **多态对象的排序需要自定义比较器**：不能直接用`<`
4. **`std::reference_wrapper`可存储引用而非拷贝**：适用于对象在别处管理
5. **`shared_ptr`适用于共享所有权**：多个容器持有同一对象
6. **容器中的元素不可拷贝时使用move语义**：`push_back(std::move(ptr))`
