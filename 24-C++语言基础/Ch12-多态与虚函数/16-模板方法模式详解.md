# 模板方法模式详解

## 一、概念说明

**模板方法模式**（Template Method Pattern，GoF设计模式）在基类定义算法骨架（非虚的模板方法），将可变步骤声明为虚函数供派生类定制。模板方法固定了算法的整体结构，派生类只能改变特定步骤的实现。

### 1.1 核心结构

```
模板方法（非虚）：定义算法骨架
  ├── 步骤1（非虚/可选虚）：固定逻辑
  ├── 步骤2（纯虚）：必须实现
  ├── 步骤3（虚，有默认）：可选重写
  └── 步骤4（非虚）：固定逻辑
```

## 二、具体用法

### 2.1 游戏框架示例

```cpp
#include <iostream>

class Game {
public:
    // 模板方法：定义游戏流程骨架
    void play() {
        initialize();
        startPlay();
        if (hasExtraMode())
            extraMode();
        endPlay();
        cleanup();
    }
    virtual ~Game() = default;

protected:
    // 必须实现的步骤
    virtual void initialize() = 0;
    virtual void startPlay() = 0;
    virtual void endPlay() = 0;

    // 可选步骤（钩子方法）
    virtual bool hasExtraMode() const { return false; }
    virtual void extraMode() {}

    // 固定步骤
    void cleanup() {
        std::cout << "清理资源..." << std::endl;
    }
};

class Chess : public Game {
protected:
    void initialize() override {
        std::cout << "棋盘已摆好" << std::endl;
    }
    void startPlay() override {
        std::cout << "下棋中..." << std::endl;
    }
    void endPlay() override {
        std::cout << "棋局结束" << std::endl;
    }
};

class Football : public Game {
protected:
    void initialize() override {
        std::cout << "球员已就位" << std::endl;
    }
    void startPlay() override {
        std::cout << "比赛中..." << std::endl;
    }
    void endPlay() override {
        std::cout << "比赛结束" << std::endl;
    }
    bool hasExtraMode() const override { return true; }
    void extraMode() override {
        std::cout << "加时赛开始!" << std::endl;
    }
};

int main() {
    Chess chess;
    std::cout << "=== 象棋 ===" << std::endl;
    chess.play();

    std::cout << "\n=== 足球 ===" << std::endl;
    Football football;
    football.play();

    return 0;
}
```

### 2.2 数据处理框架

```cpp
#include <iostream>
#include <string>
#include <vector>

class DataProcessor {
public:
    // 模板方法
    void process(const std::string& filename) {
        std::cout << "=== 开始处理 ===" << std::endl;
        auto rawData = readData(filename);
        auto parsed = parse(rawData);
        validate(parsed);
        auto result = transform(parsed);
        output(result);
        std::cout << "=== 处理完成 ===" << std::endl;
    }
    virtual ~DataProcessor() = default;

protected:
    virtual std::string readData(const std::string& filename) {
        std::cout << "读取文件: " << filename << std::endl;
        return "raw_data";
    }

    virtual std::vector<std::string> parse(const std::string& raw) = 0;

    virtual void validate(const std::vector<std::string>& data) {
        std::cout << "验证 " << data.size() << " 条数据" << std::endl;
    }

    virtual std::vector<std::string> transform(const std::vector<std::string>& data) = 0;

    void output(const std::vector<std::string>& data) {
        for (const auto& d : data)
            std::cout << "  输出: " << d << std::endl;
    }
};

class CSVProcessor : public DataProcessor {
protected:
    std::vector<std::string> parse(const std::string& raw) override {
        std::cout << "CSV解析" << std::endl;
        return {"行1", "行2", "行3"};
    }
    std::vector<std::string> transform(const std::vector<std::string>& data) override {
        std::vector<std::string> result;
        for (const auto& d : data)
            result.push_back("CSV:" + d);
        return result;
    }
};

int main() {
    CSVProcessor processor;
    processor.process("data.csv");
    return 0;
}
```

## 三、注意事项与常见陷阱

1. **模板方法应声明为非虚函数**：防止派生类重写算法骨架
2. **虚函数步骤可提供默认实现**：派生类可选择重写
3. **钩子方法（hook）是可选的虚函数步骤**：通过返回bool控制流程
4. **NVI模式可以添加前置/后置处理逻辑**：日志、验证、锁等
5. **模板方法调用protected虚函数**：不要在模板方法中暴露虚函数为public
6. **与策略模式的区别**：模板方法通过继承改变算法步骤，策略通过组合改变整个算法
