# 模板方法模式详解

## 一、概念说明

**模板方法模式**在基类定义算法骨架（非虚的模板方法），将可变步骤声明为虚函数供派生类定制。这是NVI（Non-Virtual Interface）模式的核心。

## 二、具体用法

### 2.1 非虚接口模式

```cpp
#include <iostream>

class Game {
public:
    // 模板方法：非虚，定义算法骨架
    void play() {
        initialize();
        startPlay();
        endPlay();
    }
    virtual ~Game() = default;

protected:
    virtual void initialize() = 0;
    virtual void startPlay() = 0;
    virtual void endPlay() { std::cout << "游戏结束!" << std::endl; }
};

class Chess : public Game {
protected:
    void initialize() override { std::cout << "棋盘已摆好" << std::endl; }
    void startPlay() override { std::cout << "下棋中..." << std::endl; }
};

class Football : public Game {
protected:
    void initialize() override { std::cout << "球员已就位" << std::endl; }
    void startPlay() override { std::cout << "踢球中..." << std::endl; }
    void endPlay() override { std::cout << "比赛结束，统计比分" << std::endl; }
};

int main() {
    Chess chess;
    Football football;
    chess.play();
    std::cout << "---" << std::endl;
    football.play();
    return 0;
}
```

**输出：**
```
棋盘已摆好
下棋中...
游戏结束!
---
球员已就位
踢球中...
比赛结束，统计比分
```

## 三、注意事项与常见陷阱

- 模板方法应声明为非虚函数
- 虚函数步骤可提供默认实现
- 钩子方法（hook）是可选的虚函数步骤
- NVI模式可以添加前置/后置处理逻辑
