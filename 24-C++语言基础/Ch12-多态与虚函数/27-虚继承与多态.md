# 虚继承与多态

## 一、概念说明

虚继承中（C++标准 §10.2），虚基类子对象位于对象末尾，通过**虚基类表**（vbtable）的偏移量定位。虚继承+多态的内存布局更复杂，包含vptr（虚函数表指针）和vbptr（虚基类表指针），导致`sizeof`显著增大。

### 1.1 虚继承+多态的内存布局

```
class D : public B, public C  (B和C虚继承A)

D的内存布局：
[B子对象: vptr_B] → B的vtable（含虚基类偏移）
[B子对象: b_val]
[C子对象: vptr_C] → C的vtable（含虚基类偏移）
[C子对象: c_val]
[D的成员]
[A子对象: vptr_A] → A的vtable（虚基类在末尾）
[A子对象: a_val]
```

## 二、具体用法

### 2.1 虚继承的vtable

```cpp
#include <iostream>

class A {
public:
    virtual void foo() { std::cout << "A::foo" << std::endl; }
    int a_val = 1;
};

class B : virtual public A {
public:
    void foo() override { std::cout << "B::foo" << std::endl; }
    int b_val = 2;
};

class C : virtual public A {
public:
    void foo() override { std::cout << "C::foo" << std::endl; }
    int c_val = 3;
};

class D : public B, public C {
public:
    void foo() override { std::cout << "D::foo" << std::endl; }
    int d_val = 4;
};

int main() {
    D d;
    d.A::a_val = 100;

    A* pa = &d;
    pa->foo();  // D::foo（多态）

    std::cout << "sizeof(A) = " << sizeof(A) << std::endl;
    std::cout << "sizeof(B) = " << sizeof(B) << std::endl;
    std::cout << "sizeof(D) = " << sizeof(D) << std::endl;
    // D比预期大：vptr_B + vptr_C + 成员 + vptr_A + a_val（虚基类在末尾）

    return 0;
}
```

### 2.2 虚继承中的指针转换

```cpp
#include <iostream>

class A {
public:
    virtual void f() { std::cout << "A::f" << std::endl; }
    int a_val = 1;
};

class B : virtual public A {
public:
    void f() override { std::cout << "B::f" << std::endl; }
    int b_val = 2;
};

class C : public B {
public:
    void f() override { std::cout << "C::f" << std::endl; }
    int c_val = 3;
};

int main() {
    C c;
    c.a_val = 100;

    C* pc = &c;
    B* pb = &c;
    A* pa = &c;

    std::cout << "C* = " << (void*)pc << std::endl;
    std::cout << "B* = " << (void*)pb << std::endl;
    std::cout << "A* = " << (void*)pa << std::endl;
    // A*可能与C*不同（虚基类在末尾）

    // 所有指针调用都能正确多态
    pc->f();  // C::f
    pb->f();  // C::f
    pa->f();  // C::f

    // 值相同（通过同一虚基类）
    std::cout << "pc->a_val = " << pc->a_val << std::endl;
    std::cout << "pb->a_val = " << pb->a_val << std::endl;
    std::cout << "pa->a_val = " << pa->a_val << std::endl;

    return 0;
}
```

### 2.3 虚继承的性能开销

```cpp
#include <iostream>

class Plain {
public:
    int x;
    virtual void f() {}
};

class VirtBase : virtual public Plain {
public:
    int y;
};

class NormalDerived : public Plain {
public:
    int z;
};

int main() {
    std::cout << "Plain:         " << sizeof(Plain) << std::endl;
    std::cout << "NormalDerived: " << sizeof(NormalDerived) << std::endl;
    std::cout << "VirtBase:      " << sizeof(VirtBase) << std::endl;
    // VirtBase更大：vptr + y + vbptr（虚基类指针）
    // 虚继承的间接访问有性能开销

    VirtBase vb;
    vb.x = 42;  // 通过vbptr间接访问虚基类成员
    // 比普通继承多一次指针解引用

    return 0;
}
```

## 三、注意事项与常见陷阱

1. **虚继承的内存布局更复杂**：vptr + vbptr
2. **虚基类子对象位于派生类对象的末尾**：需要间接访问
3. **虚继承的虚函数调用需要两次间接访问**：vptr → vtable → 虚基类偏移 → 调用
4. **`sizeof`会因虚继承显著增大**：额外的指针和对齐
5. **虚继承只在解决菱形问题时使用**：不要滥用
6. **虚继承+多态是C++最复杂的对象模型**：调试时需要理解内存布局
