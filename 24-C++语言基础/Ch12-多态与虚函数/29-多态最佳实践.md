# 多态最佳实践

## 一、概念说明

正确使用多态的关键在于理解何时使用虚函数、何时使用模板，以及如何设计良好的继承层次。

## 二、具体用法

### 2.1 决策指南

| 需求 | 推荐方案 |
|------|---------|
| 运行时切换行为 | 虚函数（动态多态） |
| 编译期确定行为 | 模板（静态多态） |
| 异构容器 | `vector<unique_ptr<Base>>` |
| 性能关键路径 | 模板 + `final` |
| 接口扩展 | 纯虚函数接口 |
| 算法可替换 | 策略模式 |

### 2.2 示例

```cpp
#include <iostream>
#include <memory>
#include <vector>

// 好的设计：接口分离
class Readable {
public:
    virtual std::string read() = 0;
    virtual ~Readable() = default;
};

class Writable {
public:
    virtual void write(const std::string&) = 0;
    virtual ~Writable() = default;
};

// 实现类只实现需要的接口
class DataSource : public Readable {
public:
    std::string read() override { return "数据"; }
};

class DataSink : public Writable {
public:
    void write(const std::string& data) override {
        std::cout << "写入: " << data << std::endl;
    }
};

int main() {
    std::vector<std::unique_ptr<Readable>> readers;
    readers.push_back(std::make_unique<DataSource>());
    for (auto& r : readers) std::cout << r->read() << std::endl;
    return 0;
}
```

**输出：**
```
数据
```

## 三、注意事项与常见陷阱

- 基类析构函数必须是virtual
- 始终使用`override`标记重写的虚函数
- 避免在构造/析构中调用虚函数
- 不要对不需要多态的类使用虚函数
- 考虑使用`std::function`代替简单的虚函数接口
