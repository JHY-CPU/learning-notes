# 多态最佳实践

## 一、概念说明

正确使用多态的关键在于理解何时使用虚函数、何时使用模板，以及如何设计良好的继承层次。以下是经过实践检验的最佳实践规则。

## 二、决策指南

| 需求 | 推荐方案 | 原因 |
|------|---------|------|
| 运行时切换行为 | 虚函数（动态多态） | 类型在运行时确定 |
| 编译期确定行为 | 模板（静态多态） | 消除虚函数开销 |
| 异构容器 | `vector<unique_ptr<Base>>` | 避免切片 |
| 性能关键路径 | 模板 + `final` | 可内联 |
| 接口扩展 | 纯虚函数接口 | 开闭原则 |
| 算法可替换 | 策略模式 | 运行时灵活 |
| 类型不相关但行为相同 | 类型擦除 | 无需继承关系 |

## 三、核心规则

### 3.1 基类设计

```cpp
#include <iostream>
#include <memory>

// 规则1：基类析构函数必须是virtual
class Base {
public:
    virtual ~Base() = default;  // 必须
    // 规则2：使用纯虚函数定义接口
    virtual void process() = 0;
    // 规则3：提供非虚公共接口（NVI模式）
    void execute() {
        preCheck();
        process();
        postCheck();
    }
protected:
    virtual void preCheck() { std::cout << "前置检查" << std::endl; }
    virtual void postCheck() { std::cout << "后置处理" << std::endl; }
};
```

### 3.2 派生类设计

```cpp
// 规则4：始终使用override
class Derived : public Base {
public:
    void process() override {  // 始终使用override
        std::cout << "Derived处理" << std::endl;
    }
    // 规则5：不要在构造/析构中调用虚函数
    // 规则6：如果不可扩展，使用final
};

class Sealed final : public Base {
public:
    void process() override { std::cout << "最终实现" << std::endl; }
};
```

### 3.3 多态容器

```cpp
#include <vector>
#include <memory>

// 规则7：使用智能指针容器
void containerDemo() {
    std::vector<std::unique_ptr<Base>> items;
    items.push_back(std::make_unique<Derived>());

    // 规则8：不要使用std::vector<Base>（会切片）

    // 规则9：遍历时使用const auto&
    for (const auto& item : items)
        item->process();
}
```

### 3.4 接口设计

```cpp
// 规则10：接口应小而精（接口隔离原则）
class IReadable {
public:
    virtual std::string read() = 0;
    virtual ~IReadable() = default;
};

class IWritable {
public:
    virtual void write(const std::string&) = 0;
    virtual ~IWritable() = default;
};

// 规则11：组合优于继承
class Processor {
    IReadable& reader;  // 组合：依赖接口
    IWritable& writer;
public:
    Processor(IReadable& r, IWritable& w) : reader(r), writer(w) {}
    void process() {
        auto data = reader.read();
        writer.write(data);
    }
};
```

### 3.5 性能考量

```cpp
// 规则12：性能关键场景考虑模板
template<typename T>
void fastProcess(T& obj) {
    obj.process();  // 编译期绑定，可内联
}

// 规则13：使用final允许devirtualization
class Optimized final : public Base {
public:
    void process() final override {
        std::cout << "可优化的实现" << std::endl;
    }
};

// 规则14：考虑std::variant替代虚函数
#include <variant>
using Shape = std::variant<Circle, Rectangle>;
// 编译时确定所有类型，无虚函数开销
```

## 四、常见错误

```cpp
// 错误1：基类析构非virtual
class BadBase {
public:
    ~BadBase() {}  // 应该是virtual
};

// 错误2：按值传递多态对象
void badFunc(Base b) {}  // 应该用const Base&

// 错误3：值容器存储多态对象
std::vector<Base> badVec;  // 应该用vector<unique_ptr<Base>>

// 错误4：构造函数中调用虚函数
class BadInit {
public:
    BadInit() { init(); }  // init不会多态
    virtual void init() {}
};

// 错误5：不用override
class BadDerived : public Base {
    void process() {}  // 应该写 void process() override
};
```

## 五、注意事项

1. **基类析构函数必须是virtual**：否则通过基类指针delete是未定义行为
2. **始终使用`override`标记重写的虚函数**：防止签名不匹配导致隐藏
3. **避免在构造/析构中调用虚函数**：此时虚函数不会多态
4. **不要对不需要多态的类使用虚函数**：vptr增加对象大小
5. **考虑使用`std::function`代替简单的虚函数接口**：更灵活
6. **接口应小而精**：遵循接口隔离原则
7. **优先使用组合而非继承**：除非确实是is-a关系
