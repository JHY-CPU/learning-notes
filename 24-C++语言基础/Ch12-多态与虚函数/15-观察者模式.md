# 观察者模式

## 一、概念说明

**观察者模式**（Observer Pattern，GoF设计模式）定义对象间一对多的依赖关系，当一个对象（Subject）状态改变时，所有依赖它的对象（Observer）都会收到通知并自动更新。这是事件驱动编程的基础。

### 1.1 角色说明

| 角色 | 职责 |
|------|------|
| Subject（主题） | 维护观察者列表，发送通知 |
| Observer（观察者） | 接收通知并处理 |
| ConcreteSubject | 具体主题，状态变化时通知 |
| ConcreteObserver | 具体观察者，实现处理逻辑 |

## 二、具体用法

### 2.1 基于虚函数的观察者

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
#include <string>

class Observer {
public:
    virtual void onNotify(const std::string& event, const std::string& data) = 0;
    virtual ~Observer() = default;
};

class Subject {
    std::vector<Observer*> observers;
public:
    void attach(Observer* o) {
        observers.push_back(o);
    }
    void detach(Observer* o) {
        observers.erase(
            std::remove(observers.begin(), observers.end(), o), observers.end());
    }
    void notify(const std::string& event, const std::string& data) {
        for (auto* o : observers)
            o->onNotify(event, data);
    }
};

class Logger : public Observer {
public:
    void onNotify(const std::string& event, const std::string& data) override {
        std::cout << "[日志] " << event << ": " << data << std::endl;
    }
};

class UI : public Observer {
public:
    void onNotify(const std::string& event, const std::string& data) override {
        std::cout << "[UI] 更新显示: " << event << " - " << data << std::endl;
    }
};

class Auditor : public Observer {
public:
    void onNotify(const std::string& event, const std::string& data) override {
        std::cout << "[审计] 记录: " << event << " - " << data << std::endl;
    }
};

class Button : public Subject {
    std::string label;
public:
    Button(const std::string& l) : label(l) {}
    void click() {
        std::cout << "按钮 [" << label << "] 被点击" << std::endl;
        notify("click", label);
    }
};

int main() {
    Button button("提交");
    Logger logger;
    UI ui;
    Auditor auditor;

    button.attach(&logger);
    button.attach(&ui);
    button.attach(&auditor);

    button.click();

    std::cout << "\n移除审计员后:" << std::endl;
    button.detach(&auditor);
    button.click();

    return 0;
}
```

### 2.2 智能指针版本

```cpp
#include <iostream>
#include <vector>
#include <memory>
#include <algorithm>

class Observer : public std::enable_shared_from_this<Observer> {
public:
    virtual void update(const std::string& msg) = 0;
    virtual ~Observer() = default;
};

class Subject {
    std::vector<std::weak_ptr<Observer>> observers;
public:
    void subscribe(std::shared_ptr<Observer> o) {
        observers.push_back(o);
    }

    void publish(const std::string& msg) {
        // 清理已销毁的观察者
        observers.erase(
            std::remove_if(observers.begin(), observers.end(),
                [](const std::weak_ptr<Observer>& w) { return w.expired(); }),
            observers.end());

        for (auto& wp : observers) {
            if (auto sp = wp.lock())
                sp->update(msg);
        }
    }
};

class ConcreteObserver : public Observer {
    std::string name;
public:
    ConcreteObserver(const std::string& n) : name(n) {}
    void update(const std::string& msg) override {
        std::cout << name << " 收到: " << msg << std::endl;
    }
};

int main() {
    Subject subject;
    auto obs1 = std::make_shared<ConcreteObserver>("观察者A");
    auto obs2 = std::make_shared<ConcreteObserver>("观察者B");

    subject.subscribe(obs1);
    subject.subscribe(obs2);
    subject.publish("第一条消息");

    obs1.reset();  // 观察者A销毁
    subject.publish("第二条消息");  // 只有B收到

    return 0;
}
```

## 三、注意事项与常见陷阱

1. **注意观察者的生命周期管理**：悬空指针是常见bug
2. **考虑使用`std::weak_ptr`防止循环引用**：Subject持有weak_ptr
3. **通知顺序不保证**：不要依赖通知顺序
4. **考虑使用信号槽库**：如Boost.Signals2或Qt信号槽
5. **线程安全问题**：多线程下需要加锁保护观察者列表
6. **观察者模式是发布-订阅模式的基础**：事件系统的核心
