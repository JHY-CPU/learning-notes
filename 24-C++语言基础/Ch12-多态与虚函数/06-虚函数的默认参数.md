# 虚函数的默认参数

## 一、概念说明

虚函数可以有默认参数，但**默认参数的值是静态绑定的**——根据指针/引用的**静态类型**（编译时类型）决定，而非对象的动态类型（运行时类型）。这是C++中一个常见且隐蔽的陷阱（C++标准 §10.4）。

### 1.1 绑定规则

| 组件 | 绑定方式 | 决定因素 |
|------|---------|---------|
| 函数体（调用哪个函数） | 动态绑定 | 对象的动态类型 |
| 默认参数值 | 静态绑定 | 指针/引用的静态类型 |

## 二、具体用法

### 2.1 默认参数的陷阱

```cpp
#include <iostream>

class Base {
public:
    virtual void print(int x = 10) {
        std::cout << "Base::print(" << x << ")" << std::endl;
    }
    virtual ~Base() = default;
};

class Derived : public Base {
public:
    void print(int x = 20) override {  // 不同的默认值
        std::cout << "Derived::print(" << x << ")" << std::endl;
    }
};

int main() {
    Derived d;
    Base* p = &d;

    p->print();     // 调用Derived::print，但参数是Base的默认值10！
    d.print();      // 调用Derived::print，参数是Derived的默认值20
    p->print(30);   // 调用Derived::print，参数是显式传入的30

    return 0;
}
```

**输出：**
```
Derived::print(10)   ← 注意：虽然调用的是Derived，参数却是10（Base的默认值）
Derived::print(20)
Derived::print(30)
```

### 2.2 陷阱产生的原因

```cpp
#include <iostream>

class Shape {
public:
    virtual void draw(std::string color = "红色") {
        std::cout << "绘制" << color << "的Shape" << std::endl;
    }
    virtual ~Shape() = default;
};

class Circle : public Shape {
public:
    void draw(std::string color = "蓝色") override {
        std::cout << "绘制" << color << "的Circle" << std::endl;
    }
};

void render(Shape* shapes[], int n) {
    for (int i = 0; i < n; ++i)
        shapes[i]->draw();  // 永远使用"红色"作为默认值！
}

int main() {
    Circle c;
    Shape* shapes[] = {&c};
    render(shapes, 1);  // 输出：绘制红色的Circle
    // 期望"蓝色"，实际"红色"

    c.draw();  // 输出：绘制蓝色的Circle（直接调用，使用Circle的默认值）
    return 0;
}
```

### 2.3 避免陷阱的方法

```cpp
#include <iostream>

// 方法1：不使用默认参数
class Base1 {
public:
    virtual void process(int x) = 0;  // 无默认参数
    virtual ~Base1() = default;
};

// 方法2：NVI模式（推荐）
class Base2 {
public:
    void process() { doProcess(10); }  // 非虚，固定的默认值
    void process(int x) { doProcess(x); }
    virtual ~Base2() = default;
protected:
    virtual void doProcess(int x) {
        std::cout << "Base2::doProcess(" << x << ")" << std::endl;
    }
};

class Derived2 : public Base2 {
protected:
    void doProcess(int x) override {
        std::cout << "Derived2::doProcess(" << x << ")" << std::endl;
    }
};

// 方法3：派生类不重新定义默认参数
class Base3 {
public:
    virtual void work(int mode = 0) {
        std::cout << "Base3::work(" << mode << ")" << std::endl;
    }
    virtual ~Base3() = default;
};

class Derived3 : public Base3 {
public:
    void work(int mode = 0) override {  // 保持相同的默认值
        std::cout << "Derived3::work(" << mode << ")" << std::endl;
    }
};

int main() {
    Derived2 d2;
    Base2* p2 = &d2;
    p2->process();   // 10（由NVI保证）

    Derived3 d3;
    Base3* p3 = &d3;
    p3->work();      // 0（默认值一致）

    return 0;
}
```

## 三、注意事项与常见陷阱

1. **避免在虚函数中使用默认参数**：或保证派生类使用相同的默认值
2. **默认参数在编译期解析（静态绑定）**：编译器根据指针类型填入默认值
3. **函数调用在运行时解析（动态绑定）**：vtable决定调用哪个版本
4. **两者不一致是bug的常见来源**：函数体是Derived的，参数是Base的
5. **考虑使用NVI模式避免此问题**：将默认参数放在非虚公共接口中
6. **`override`不检查默认参数**：只检查参数类型，不检查默认值
