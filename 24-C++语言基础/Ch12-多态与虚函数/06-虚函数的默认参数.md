# 虚函数的默认参数

## 一、概念说明

虚函数可以有默认参数，但**默认参数的值是静态绑定的**——根据指针/引用的**静态类型**决定，而非对象的动态类型。这是C++中一个常见陷阱。

## 二、具体用法

### 2.1 默认参数的陷阱

```cpp
#include <iostream>

class Base {
public:
    virtual void print(int x = 10) {
        std::cout << "Base::print(" << x << ")" << std::endl;
    }
};

class Derived : public Base {
public:
    void print(int x = 20) override {  // 不同的默认值
        std::cout << "Derived::print(" << x << ")" << std::endl;
    }
};

int main() {
    Derived d;
    Base* p = &d;

    p->print();     // 调用Derived::print，但参数是Base的默认值10！
    d.print();      // 调用Derived::print，参数是Derived的默认值20
    return 0;
}
```

**输出：**
```
Derived::print(10)  ← 注意：虽然调用的是Derived，参数却是10
Derived::print(20)
```

## 三、注意事项与常见陷阱

- **避免**在虚函数中使用默认参数，或保证派生类使用相同的默认值
- 默认参数在编译期解析（静态绑定）
- 函数调用在运行时解析（动态绑定）
- 这两者不一致是bug的常见来源
- 考虑使用非虚接口（NVI）模式避免此问题
