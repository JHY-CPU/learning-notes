# 工厂模式

## 一、概念说明

**工厂模式**通过工厂方法创建对象，客户端不直接使用`new`。工厂模式隐藏对象创建细节，便于扩展和管理对象生命周期。

## 二、具体用法

### 2.1 简单工厂

```cpp
#include <iostream>
#include <memory>
#include <string>

class Document {
public:
    virtual void open() = 0;
    virtual ~Document() = default;
};

class PdfDoc : public Document {
public:
    void open() override { std::cout << "打开PDF文档" << std::endl; }
};

class WordDoc : public Document {
public:
    void open() override { std::cout << "打开Word文档" << std::endl; }
};

class DocumentFactory {
public:
    static std::unique_ptr<Document> create(const std::string& type) {
        if (type == "pdf") return std::make_unique<PdfDoc>();
        if (type == "word") return std::make_unique<WordDoc>();
        return nullptr;
    }
};

int main() {
    auto doc1 = DocumentFactory::create("pdf");
    auto doc2 = DocumentFactory::create("word");
    doc1->open();
    doc2->open();
    return 0;
}
```

**输出：**
```
打开PDF文档
打开Word文档
```

### 2.2 抽象工厂

```cpp
#include <iostream>
#include <memory>

class Button {
public:
    virtual void render() = 0;
    virtual ~Button() = default;
};

class WindowsButton : public Button {
public:
    void render() override { std::cout << "Windows按钮" << std::endl; }
};

class MacButton : public Button {
public:
    void render() override { std::cout << "Mac按钮" << std::endl; }
};

class GUIFactory {
public:
    virtual std::unique_ptr<Button> createButton() = 0;
    virtual ~GUIFactory() = default;
};

class WindowsFactory : public GUIFactory {
public:
    std::unique_ptr<Button> createButton() override {
        return std::make_unique<WindowsButton>();
    }
};
```

## 三、注意事项与常见陷阱

- 工厂返回基类智能指针，隐藏具体类型
- 简单工厂违反开闭原则（添加新产品需修改工厂）
- 抽象工厂允许创建产品族
- 使用注册机制实现开闭原则
