# 工厂模式

## 一、概念说明

**工厂模式**通过工厂方法创建对象，客户端不直接使用`new`。工厂模式隐藏对象创建细节，便于扩展和管理对象生命周期。有三种主要形式：简单工厂、工厂方法、抽象工厂。

### 1.1 工厂模式对比

| 模式 | 特点 | 扩展方式 |
|------|------|---------|
| 简单工厂 | 静态方法，按条件创建 | 修改工厂（违反开闭） |
| 工厂方法 | 虚函数，子类决定创建 | 添加子类 |
| 抽象工厂 | 创建产品族 | 添加工厂类 |

## 二、具体用法

### 2.1 简单工厂

```cpp
#include <iostream>
#include <memory>
#include <string>

class Document {
public:
    virtual void open() = 0;
    virtual std::string type() const = 0;
    virtual ~Document() = default;
};

class PdfDoc : public Document {
public:
    void open() override { std::cout << "打开PDF文档" << std::endl; }
    std::string type() const override { return "PDF"; }
};

class WordDoc : public Document {
public:
    void open() override { std::cout << "打开Word文档" << std::endl; }
    std::string type() const override { return "Word"; }
};

class TextDoc : public Document {
public:
    void open() override { std::cout << "打开文本文档" << std::endl; }
    std::string type() const override { return "Text"; }
};

class DocumentFactory {
public:
    static std::unique_ptr<Document> create(const std::string& type) {
        if (type == "pdf") return std::make_unique<PdfDoc>();
        if (type == "word") return std::make_unique<WordDoc>();
        if (type == "text") return std::make_unique<TextDoc>();
        return nullptr;
    }
};

int main() {
    auto doc1 = DocumentFactory::create("pdf");
    auto doc2 = DocumentFactory::create("word");
    if (doc1) doc1->open();
    if (doc2) doc2->open();
    return 0;
}
```

### 2.2 抽象工厂

```cpp
#include <iostream>
#include <memory>

class Button {
public:
    virtual void render() = 0;
    virtual ~Button() = default;
};

class WindowsButton : public Button {
public:
    void render() override { std::cout << "渲染Windows按钮" << std::endl; }
};

class MacButton : public Button {
public:
    void render() override { std::cout << "渲染Mac按钮" << std::endl; }
};

class Checkbox {
public:
    virtual void render() = 0;
    virtual ~Checkbox() = default;
};

class WindowsCheckbox : public Checkbox {
public:
    void render() override { std::cout << "渲染Windows复选框" << std::endl; }
};

class MacCheckbox : public Checkbox {
public:
    void render() override { std::cout << "渲染Mac复选框" << std::endl; }
};

// 抽象工厂
class GUIFactory {
public:
    virtual std::unique_ptr<Button> createButton() = 0;
    virtual std::unique_ptr<Checkbox> createCheckbox() = 0;
    virtual ~GUIFactory() = default;
};

class WindowsFactory : public GUIFactory {
public:
    std::unique_ptr<Button> createButton() override {
        return std::make_unique<WindowsButton>();
    }
    std::unique_ptr<Checkbox> createCheckbox() override {
        return std::make_unique<WindowsCheckbox>();
    }
};

class MacFactory : public GUIFactory {
public:
    std::unique_ptr<Button> createButton() override {
        return std::make_unique<MacButton>();
    }
    std::unique_ptr<Checkbox> createCheckbox() override {
        return std::make_unique<MacCheckbox>();
    }
};

// 客户端只依赖抽象工厂
void createUI(GUIFactory& factory) {
    auto button = factory.createButton();
    auto checkbox = factory.createCheckbox();
    button->render();
    checkbox->render();
}

int main() {
    WindowsFactory winFactory;
    MacFactory macFactory;

    std::cout << "=== Windows UI ===" << std::endl;
    createUI(winFactory);

    std::cout << "=== Mac UI ===" << std::endl;
    createUI(macFactory);

    return 0;
}
```

### 2.3 注册式工厂

```cpp
#include <iostream>
#include <memory>
#include <string>
#include <unordered_map>
#include <functional>

class Product {
public:
    virtual void use() = 0;
    virtual ~Product() = default;

    using Creator = std::unique_ptr<Product>(*)();
    static std::unordered_map<std::string, Creator>& registry() {
        static std::unordered_map<std::string, Creator> reg;
        return reg;
    }

    static std::unique_ptr<Product> create(const std::string& type) {
        auto it = registry().find(type);
        if (it != registry().end()) return it->second();
        return nullptr;
    }
};

class ProductA : public Product {
public:
    void use() override { std::cout << "ProductA" << std::endl; }
    static std::unique_ptr<Product> create() { return std::make_unique<ProductA>(); }
};

class ProductB : public Product {
public:
    void use() override { std::cout << "ProductB" << std::endl; }
    static std::unique_ptr<Product> create() { return std::make_unique<ProductB>(); }
};

// 自动注册
struct AutoRegister {
    AutoRegister() {
        Product::registry()["A"] = ProductA::create;
        Product::registry()["B"] = ProductB::create;
    }
} autoReg;

int main() {
    auto p1 = Product::create("A");
    auto p2 = Product::create("B");
    if (p1) p1->use();
    if (p2) p2->use();
    return 0;
}
```

## 三、注意事项与常见陷阱

1. **工厂返回基类智能指针，隐藏具体类型**：客户端不依赖具体实现
2. **简单工厂违反开闭原则**：添加新产品需修改工厂
3. **抽象工厂允许创建产品族**：如Windows/Mac的整套UI组件
4. **使用注册机制实现开闭原则**：新产品自行注册，不修改工厂
5. **工厂模式是依赖注入的基础**：反转控制
6. **考虑使用`std::make_unique`而非`new`**：工厂方法应返回智能指针
