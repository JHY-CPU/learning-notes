# 多态概述

## 一、概念说明

**多态**（Polymorphism，希腊语"多种形态"）是面向对象编程三大特性之一，指同一接口在不同上下文中表现出不同行为。C++支持两种多态：

- **静态多态**（编译期）：函数重载、运算符重载、模板、CRTP
- **动态多态**（运行时）：虚函数、继承、vtable

### 1.1 核心区别

| 特性 | 静态多态 | 动态多态 |
|------|---------|---------|
| 绑定时机 | 编译期 | 运行时 |
| 实现方式 | 模板/重载 | 虚函数/vtable |
| 性能 | 更快（可内联） | 有间接调用开销 |
| 灵活性 | 编译时确定类型 | 运行时可扩展 |
| 异构容器 | 不支持 | 支持 |
| 代码膨胀 | 每个类型一份 | 共享一份 |

## 二、具体用法

### 2.1 动态多态（虚函数）

```cpp
#include <iostream>
#include <vector>
#include <memory>

class Animal {
public:
    virtual void speak() const { std::cout << "..." << std::endl; }
    virtual void move() const { std::cout << "移动" << std::endl; }
    virtual ~Animal() = default;
};

class Dog : public Animal {
public:
    void speak() const override { std::cout << "汪汪!" << std::endl; }
    void move() const override { std::cout << "奔跑" << std::endl; }
};

class Cat : public Animal {
public:
    void speak() const override { std::cout << "喵喵!" << std::endl; }
    void move() const override { std::cout << "跳跃" << std::endl; }
};

void letItSpeak(const Animal& a) {
    a.speak();   // 运行时多态
    a.move();
}

int main() {
    Dog dog;
    Cat cat;
    letItSpeak(dog);  // 汪汪! / 奔跑
    letItSpeak(cat);  // 喵喵! / 跳跃

    // 异构容器
    std::vector<std::unique_ptr<Animal>> animals;
    animals.push_back(std::make_unique<Dog>());
    animals.push_back(std::make_unique<Cat>());
    for (const auto& a : animals)
        a->speak();

    return 0;
}
```

### 2.2 静态多态（模板）

```cpp
#include <iostream>

template<typename T>
void process(T& obj) {
    obj.speak();  // 编译期绑定
    obj.move();
}

struct DogT {
    void speak() const { std::cout << "汪汪!" << std::endl; }
    void move() const { std::cout << "奔跑" << std::endl; }
};

struct CatT {
    void speak() const { std::cout << "喵喵!" << std::endl; }
    void move() const { std::cout << "跳跃" << std::endl; }
};

int main() {
    DogT dog;
    CatT cat;
    process(dog);  // 编译期确定调用DogT::speak
    process(cat);  // 编译期确定调用CatT::speak
    return 0;
}
```

### 2.3 多态的实现条件

```cpp
// 动态多态的三个必要条件：
// 1. 继承关系
// 2. 基类有虚函数
// 3. 通过基类指针/引用调用

class Base {
public:
    virtual void f() {}  // 条件2：虚函数
};

class Derived : public Base {  // 条件1：继承
public:
    void f() override {}  // 重写
};

int main() {
    Derived d;
    Base* p = &d;  // 条件3：基类指针
    p->f();        // 多态：调用Derived::f

    Base& r = d;   // 条件3：基类引用
    r.f();         // 多态：调用Derived::f

    Base b = d;    // 切片！不是多态
    b.f();         // 调用Base::f

    return 0;
}
```

## 三、注意事项与常见陷阱

1. **动态多态需要基类引用或指针才能触发**：值传递会发生切片
2. **动态多态必须有虚函数和继承**：缺一不可
3. **静态多态不支持异构容器**：不能放不同类型的对象
4. **性能敏感场景考虑静态多态**：消除间接调用开销
5. **可扩展性需求选择动态多态**：运行时添加新类型
6. **`std::function`和`std::variant`也是多态机制**：类型擦除和代数数据类型
