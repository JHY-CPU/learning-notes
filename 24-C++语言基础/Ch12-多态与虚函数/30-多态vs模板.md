# 多态vs模板

## 一、概念说明

运行时多态（虚函数）和编译期多态（模板，C++标准 §13/§10.3）各有优劣。选择取决于类型是否在编译时已知、是否需要异构容器、性能要求等。现代C++还提供了`std::variant`、`std::function`、CRTP等折中方案。

### 1.1 核心对比

| 特性 | 虚函数（运行时多态） | 模板（编译期多态） |
|------|-------------------|------------------|
| 类型确定 | 运行时 | 编译时 |
| 异构容器 | 支持 | 不支持 |
| 性能 | 间接调用 | 可内联 |
| 代码膨胀 | 无 | 每个类型一份 |
| 可扩展性 | 添加派生类 | 不需改源码 |
| 编译时间 | 较短 | 较长 |
| 调试难度 | 较易 | 较难（模板错误信息长） |

## 二、具体用法

### 2.1 运行时多态（虚函数）

```cpp
#include <iostream>
#include <vector>
#include <memory>

class Shape {
public:
    virtual double area() const = 0;
    virtual void draw() const = 0;
    virtual ~Shape() = default;
};

class Circle : public Shape {
    double r;
public:
    Circle(double r) : r(r) {}
    double area() const override { return 3.14159 * r * r; }
    void draw() const override { std::cout << "绘制圆" << std::endl; }
};

class Rectangle : public Shape {
    double w, h;
public:
    Rectangle(double w, double h) : w(w), h(h) {}
    double area() const override { return w * h; }
    void draw() const override { std::cout << "绘制矩形" << std::endl; }
};

// 异构容器：不同类型的Shape放在一起
double totalArea(const std::vector<std::unique_ptr<Shape>>& shapes) {
    double sum = 0;
    for (const auto& s : shapes) sum += s->area();
    return sum;
}

int main() {
    std::vector<std::unique_ptr<Shape>> shapes;
    shapes.push_back(std::make_unique<Circle>(5));
    shapes.push_back(std::make_unique<Rectangle>(3, 4));
    std::cout << "总面积: " << totalArea(shapes) << std::endl;
    return 0;
}
```

### 2.2 编译期多态（模板）

```cpp
#include <iostream>
#include <vector>

// 不需要继承关系
struct CircleT { double r; double area() const { return 3.14159 * r * r; } };
struct RectT   { double w, h; double area() const { return w * h; } };
struct TriT    { double a, b, c;
    double area() const {
        double s = (a+b+c)/2;
        return std::sqrt(s*(s-a)*(s-b)*(s-c));
    }
};

// 通用函数模板：编译期绑定
template<typename T>
double computeArea(const T& shape) {
    return shape.area();  // 编译期解析
}

// 同构容器模板
template<typename T>
double sumAreas(const std::vector<T>& shapes) {
    double sum = 0;
    for (const auto& s : shapes) sum += s.area();
    return sum;
}

int main() {
    CircleT c{5};
    RectT r{3, 4};

    std::cout << "圆面积: " << computeArea(c) << std::endl;
    std::cout << "矩形面积: " << computeArea(r) << std::endl;

    std::vector<CircleT> circles = {CircleT{5}, CircleT{3}};
    std::cout << "所有圆面积: " << sumAreas(circles) << std::endl;

    return 0;
}
```

### 2.3 折中方案

```cpp
#include <iostream>
#include <variant>
#include <functional>

// 方案1：std::variant（编译时确定所有类型）
struct CircleV { double r; double area() const { return 3.14159*r*r; } };
struct RectV   { double w, h; double area() const { return w*h; } };
using ShapeV = std::variant<CircleV, RectV>;

double variantArea(const std::vector<ShapeV>& shapes) {
    double sum = 0;
    for (const auto& s : shapes)
        sum += std::visit([](const auto& sh) { return sh.area(); }, s);
    return sum;
}

// 方案2：CRTP（编译期多态 + 接口统一）
template<typename Derived>
class ShapeCRTP {
public:
    double area() const {
        return static_cast<const Derived*>(this)->areaImpl();
    }
};

class CircleCRTP : public ShapeCRTP<CircleCRTP> {
    double r;
public:
    CircleCRTP(double r) : r(r) {}
    double areaImpl() const { return 3.14159 * r * r; }
};

// 方案3：std::function（类型擦除）
double funcArea(std::function<double()> areaFunc) {
    return areaFunc();
}

int main() {
    // variant方案
    std::vector<ShapeV> shapes = {CircleV{5}, RectV{3, 4}};
    std::cout << "variant总面积: " << variantArea(shapes) << std::endl;

    // CRTP方案
    CircleCRTP cc(5);
    std::cout << "CRTP面积: " << cc.area() << std::endl;

    // function方案
    double r = 5;
    std::cout << "function面积: " << funcArea([r]() { return 3.14159*r*r; }) << std::endl;

    return 0;
}
```

### 2.4 选择指南

```cpp
/*
场景1：需要异构容器 → 虚函数或std::variant
场景2：性能关键 → 模板或CRTP
场景3：类型在运行时确定 → 虚函数
场景4：类型在编译时确定 → 模板
场景5：需要运行时灵活性 → 虚函数 + 策略模式
场景6：避免代码膨胀 → 虚函数（模板会为每个类型生成代码）
场景7：需要接口统一 → 虚函数或CRTP
场景8：不需要继承关系 → std::function或类型擦除
*/
```

## 三、注意事项与常见陷阱

1. **性能关键场景优先模板**：虚函数的间接调用有开销
2. **需要运行时灵活性时用虚函数**：类型在运行时确定
3. **可以结合使用**：CRTP + 虚函数，根据场景选择
4. **C++20 concepts让模板约束更清晰**：错误信息更友好
5. **`std::variant`是折中方案**：编译时确定所有类型，运行时选择
6. **不要过早优化**：先用虚函数保证正确性，再考虑模板优化
7. **模板的代码膨胀问题**：每个类型实例化一份代码，增加二进制大小
