# 多态vs模板

## 一、概念说明

运行时多态（虚函数）和编译期多态（模板）各有优劣。选择取决于类型是否在编译时已知、是否需要异构容器、性能要求等。

## 二、具体用法

### 2.1 对比示例

```cpp
#include <iostream>
#include <chrono>

// 运行时多态：虚函数
class Shape {
public:
    virtual double area() const = 0;
    virtual ~Shape() = default;
};

class Circle : public Shape {
    double r;
public:
    Circle(double r) : r(r) {}
    double area() const override { return 3.14159 * r * r; }
};

// 编译期多态：模板
template<typename T>
double computeArea(const T& shape) {
    return shape.area();  // 编译期绑定
}

struct CircleT {
    double r;
    double area() const { return 3.14159 * r * r; }
};

int main() {
    // 运行时多态
    Circle c(5.0);
    const Shape* s = &c;
    std::cout << "虚函数面积: " << s->area() << std::endl;

    // 编译期多态
    CircleT ct{5.0};
    std::cout << "模板面积: " << computeArea(ct) << std::endl;

    // 对比
    std::cout << "\n=== 选择指南 ===" << std::endl;
    std::cout << "虚函数: 运行时类型未知、异构容器" << std::endl;
    std::cout << "模板: 编译时类型已知、性能关键" << std::endl;
    return 0;
}
```

**输出：**
```
虚函数面积: 78.5398
模板面积: 78.5398

=== 选择指南 ===
虚函数: 运行时类型未知、异构容器
模板: 编译时类型已知、性能关键
```

### 2.2 混合使用

```cpp
// 使用type erasure桥接两者
// std::function就是运行时多态+模板的结合
#include <functional>
std::function<double()> calc = []() { return 42.0; };
```

## 三、注意事项与常见陷阱

| 特性 | 虚函数（运行时） | 模板（编译期） |
|------|----------------|--------------|
| 类型确定 | 运行时 | 编译时 |
| 异构容器 | 支持 | 不支持 |
| 性能 | 间接调用 | 可内联 |
| 代码膨胀 | 无 | 每个类型一份 |
| 可扩展性 | 添加派生类 | 无需改源码 |

- 性能关键场景优先模板
- 需要运行时灵活性时用虚函数
- 可以结合使用（CRTP + 虚函数）
- C++20 concepts让模板约束更清晰
