# 纯接口与混入

## 一、概念说明

**混入类**（Mixin）是提供特定功能片段的小型类，通过继承组合到目标类中。与纯接口不同，混入类可以有实现。混入实现"能力注入"。

## 二、具体用法

### 2.1 混入类

```cpp
#include <iostream>
#include <string>

// 纯接口
class Printable {
public:
    virtual std::string toString() const = 0;
    virtual ~Printable() = default;
};

// 混入：提供日志能力
class Loggable {
public:
    void log(const std::string& msg) const {
        std::cout << "[LOG] " << msg << std::endl;
    }
};

// 混入：提供可序列化能力
class Serializable {
public:
    std::string serialize() const {
        return "{\"data\": \"serialized\"}";
    }
};

// 组合多个混入
class Document : public Printable, public Loggable, public Serializable {
    std::string content;
public:
    Document(const std::string& c) : content(c) {}
    std::string toString() const override {
        return "Document: " + content;
    }
};

int main() {
    Document doc("报告");
    std::cout << doc.toString() << std::endl;
    doc.log("文档已创建");
    std::cout << doc.serialize() << std::endl;
    return 0;
}
```

**输出：**
```
Document: 报告
[LOG] 文档已创建
{"data": "serialized"}
```

## 三、注意事项与常见陷阱

- 混入提供功能片段而非完整实现
- 注意菱形继承问题（使用虚继承或组合）
- C++中CRTP也可实现编译期混入
- 混入过多会增加类的复杂度
