# 纯接口与混入

## 一、概念说明

**混入类**（Mixin）是提供特定功能片段的小型类，通过继承组合到目标类中。与纯接口（只有纯虚函数）不同，混入类可以有实现。混入实现"能力注入"——为目标类添加特定功能。

### 1.1 纯接口 vs 混入 vs 抽象基类

| 类型 | 虚函数 | 数据成员 | 实现 | 用途 |
|------|--------|---------|------|------|
| 纯接口 | 纯虚 | 无 | 无 | 行为契约 |
| 混入 | 可选 | 可选 | 有 | 功能片段 |
| 抽象基类 | 纯虚+普通虚 | 有 | 部分 | 公共逻辑 |

## 二、具体用法

### 2.1 混入类组合

```cpp
#include <iostream>
#include <string>

// 纯接口
class Printable {
public:
    virtual std::string toString() const = 0;
    virtual ~Printable() = default;
};

// 混入：提供日志能力
class Loggable {
public:
    void log(const std::string& msg) const {
        std::cout << "[LOG] " << msg << std::endl;
    }
};

// 混入：提供可序列化能力
class Serializable {
public:
    std::string serialize() const {
        // 简单序列化实现
        return "{\"type\": \"object\"}";
    }
};

// 混入：提供可比较能力
template<typename Derived>
class Comparable {
public:
    bool operator>(const Derived& rhs) const {
        return static_cast<const Derived*>(this)->compareTo(rhs) > 0;
    }
    bool operator<(const Derived& rhs) const {
        return static_cast<const Derived*>(this)->compareTo(rhs) < 0;
    }
};

// 组合多个混入
class Document : public Printable,
                 public Loggable,
                 public Serializable,
                 public Comparable<Document> {
    std::string content;
    int priority;
public:
    Document(const std::string& c, int p) : content(c), priority(p) {}

    std::string toString() const override {
        return "Document[" + content + ", p=" + std::to_string(priority) + "]";
    }

    int compareTo(const Document& o) const {
        return priority - o.priority;
    }
};

int main() {
    Document doc1("报告A", 3);
    Document doc2("报告B", 5);

    std::cout << doc1.toString() << std::endl;
    doc1.log("文档已创建");
    std::cout << doc1.serialize() << std::endl;
    std::cout << "doc1 > doc2: " << (doc1 > doc2) << std::endl;  // 0

    return 0;
}
```

### 2.2 CRTP实现编译期混入

```cpp
#include <iostream>

// 编译期混入：日志能力
template<typename Derived>
class LogMixin {
public:
    void log(const std::string& msg) const {
        std::cout << "[" << static_cast<const Derived*>(this)->name()
                  << "] " << msg << std::endl;
    }
};

// 编译期混入：计数能力
template<typename Derived>
class CounterMixin {
    static int count;
public:
    CounterMixin() { ++count; }
    ~CounterMixin() { --count; }
    static int getCount() { return count; }
};

template<typename Derived>
int CounterMixin<Derived>::count = 0;

// 组合混入
class Service : public LogMixin<Service>, public CounterMixin<Service> {
public:
    std::string name() const { return "Service"; }
    void doWork() {
        log("开始工作");
        // ...
        log("工作完成");
    }
};

class Worker : public LogMixin<Worker>, public CounterMixin<Worker> {
public:
    std::string name() const { return "Worker"; }
};

int main() {
    Service s1, s2;
    Worker w1;

    s1.doWork();
    std::cout << "Service数量: " << Service::getCount() << std::endl;  // 2
    std::cout << "Worker数量: " << Worker::getCount() << std::endl;   // 1

    return 0;
}
```

### 2.3 混入与依赖注入

```cpp
#include <iostream>

// 混入：提供缓存能力
template<typename Key, typename Value>
class CacheMixin {
    std::unordered_map<Key, Value> cache;
protected:
    Value* getCached(const Key& key) {
        auto it = cache.find(key);
        return it != cache.end() ? &it->second : nullptr;
    }
    void setCache(const Key& key, const Value& value) {
        cache[key] = value;
    }
};

// 混入：提供重试能力
class RetryMixin {
protected:
    template<typename Func>
    auto withRetry(Func&& func, int maxRetries = 3) {
        for (int i = 0; i < maxRetries; ++i) {
            try {
                return func();
            } catch (...) {
                if (i == maxRetries - 1) throw;
                std::cout << "重试 " << (i+1) << std::endl;
            }
        }
    }
};

class DataService : public CacheMixin<std::string, std::string>,
                    public RetryMixin {
public:
    std::string fetch(const std::string& key) {
        if (auto* cached = getCached(key))
            return *cached;

        return withRetry([&]() {
            // 模拟网络请求
            std::string value = "data[" + key + "]";
            setCache(key, value);
            return value;
        });
    }
};
```

## 三、注意事项与常见陷阱

1. **混入提供功能片段而非完整实现**：小而专注
2. **注意菱形继承问题**：使用虚继承或组合
3. **C++中CRTP也可实现编译期混入**：消除虚函数开销
4. **混入过多会增加类的复杂度**：适度使用
5. **混入是AOP（面向切面编程）的基础**：横切关注点
6. **Python等语言有内建的Mixin支持**：C++通过多重继承实现
