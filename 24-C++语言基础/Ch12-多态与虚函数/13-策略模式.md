# 策略模式

## 一、概念说明

**策略模式**（Strategy Pattern，GoF设计模式）定义一系列算法，将每个算法封装起来，并使它们可以互换。通过虚函数实现运行时策略切换，或通过模板实现编译期策略选择。策略模式让算法独立于使用它的客户端。

### 1.1 运行时 vs 编译期策略

| 特性 | 运行时策略（虚函数） | 编译期策略（模板） |
|------|-------------------|------------------|
| 切换时机 | 运行时 | 编译时 |
| 性能 | 间接调用 | 可内联 |
| 灵活性 | 高（运行时可换） | 低（编译时固定） |
| 实现方式 | 继承+虚函数 | 模板参数 |

## 二、具体用法

### 2.1 运行时策略切换

```cpp
#include <iostream>
#include <vector>
#include <memory>
#include <algorithm>

class SortStrategy {
public:
    virtual void sort(std::vector<int>& data) = 0;
    virtual std::string name() const = 0;
    virtual ~SortStrategy() = default;
};

class BubbleSort : public SortStrategy {
public:
    void sort(std::vector<int>& data) override {
        std::cout << "冒泡排序" << std::endl;
        for (size_t i = 0; i < data.size(); ++i)
            for (size_t j = 0; j < data.size() - i - 1; ++j)
                if (data[j] > data[j + 1]) std::swap(data[j], data[j + 1]);
    }
    std::string name() const override { return "冒泡排序"; }
};

class QuickSort : public SortStrategy {
public:
    void sort(std::vector<int>& data) override {
        std::cout << "快速排序" << std::endl;
        std::sort(data.begin(), data.end());
    }
    std::string name() const override { return "快速排序"; }
};

class MergeSort : public SortStrategy {
public:
    void sort(std::vector<int>& data) override {
        std::cout << "归并排序" << std::endl;
        std::stable_sort(data.begin(), data.end());
    }
    std::string name() const override { return "归并排序"; }
};

class Sorter {
    std::unique_ptr<SortStrategy> strategy;
public:
    void setStrategy(std::unique_ptr<SortStrategy> s) {
        strategy = std::move(s);
        std::cout << "切换策略: " << strategy->name() << std::endl;
    }
    void sort(std::vector<int>& data) {
        if (strategy) strategy->sort(data);
    }
};

int main() {
    Sorter sorter;
    std::vector<int> data = {5, 2, 8, 1, 9, 3};

    sorter.setStrategy(std::make_unique<BubbleSort>());
    sorter.sort(data);

    sorter.setStrategy(std::make_unique<QuickSort>());
    sorter.sort(data);

    return 0;
}
```

### 2.2 编译期策略（模板）

```cpp
#include <iostream>
#include <vector>

// 策略作为模板参数
struct AscendingSort {
    static bool compare(int a, int b) { return a < b; }
};

struct DescendingSort {
    static bool compare(int a, int b) { return a > b; }
};

template<typename ComparePolicy>
class Sorter {
public:
    static void sort(std::vector<int>& data) {
        for (size_t i = 0; i < data.size(); ++i)
            for (size_t j = i + 1; j < data.size(); ++j)
                if (ComparePolicy::compare(data[j], data[i]))
                    std::swap(data[i], data[j]);
    }
};

int main() {
    std::vector<int> data = {5, 2, 8, 1, 9};

    Sorter<AscendingSort>::sort(data);
    std::cout << "升序: ";
    for (int x : data) std::cout << x << " ";
    std::cout << std::endl;

    Sorter<DescendingSort>::sort(data);
    std::cout << "降序: ";
    for (int x : data) std::cout << x << " ";

    return 0;
}
```

### 2.3 压缩策略示例

```cpp
#include <iostream>
#include <string>
#include <memory>

class CompressionStrategy {
public:
    virtual std::string compress(const std::string& data) = 0;
    virtual ~CompressionStrategy() = default;
};

class NoCompression : public CompressionStrategy {
public:
    std::string compress(const std::string& data) override {
        return data;
    }
};

class LZCompression : public CompressionStrategy {
public:
    std::string compress(const std::string& data) override {
        return "[LZ压缩]" + data;
    }
};

class FileProcessor {
    std::unique_ptr<CompressionStrategy> compressor;
public:
    FileProcessor(std::unique_ptr<CompressionStrategy> c)
        : compressor(std::move(c)) {}

    void process(const std::string& data) {
        auto compressed = compressor->compress(data);
        std::cout << "处理: " << compressed << std::endl;
    }
};

int main() {
    FileProcessor p1(std::make_unique<NoCompression>());
    p1.process("原始数据");

    FileProcessor p2(std::make_unique<LZCompression>());
    p2.process("原始数据");

    return 0;
}
```

## 三、注意事项与常见陷阱

1. **策略模式适合运行时切换算法**：如排序、压缩、加密算法
2. **如果策略固定，考虑模板（编译期选择）**：消除虚函数开销
3. **策略对象可以是有状态的**：与纯函数不同
4. **使用智能指针管理策略对象的生命周期**：`unique_ptr<Strategy>`
5. **策略模式与状态模式相似**：但意图不同（策略是可替换算法，状态是自动转换）
6. **C++20的`std::function`也可实现简单策略**：用lambda代替继承
