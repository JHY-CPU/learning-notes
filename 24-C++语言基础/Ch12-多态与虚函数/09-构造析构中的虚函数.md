# 构造析构中的虚函数

## 一、概念说明

在构造函数和析构函数中调用虚函数**不会发生多态**（C++标准 §10.4/§12.6.2）。因为此时对象的派生类部分尚未构造（构造时）或已经析构（析构时），vptr指向当前类的vtable。这是语言标准规定的行为，不是编译器bug。

### 1.1 vptr的变化过程

```
创建Derived对象：
1. Base()开始 → vptr指向Base的vtable → 调用Base::init()
2. Base()结束 → vptr改为指向Derived的vtable
3. Derived()开始 → vptr指向Derived的vtable → 调用Derived::init()
4. Derived()结束 → 对象完成

销毁Derived对象：
1. ~Derived()开始 → vptr指向Derived的vtable → 调用Derived::cleanup()
2. ~Derived()结束 → vptr改为指向Base的vtable
3. ~Base()开始 → vptr指向Base的vtable → 调用Base::cleanup()
4. ~Base()结束 → 对象销毁
```

## 二、具体用法

### 2.1 构造期间的行为

```cpp
#include <iostream>

class Base {
public:
    Base() {
        std::cout << "Base构造中: ";
        init();  // 调用Base::init（非多态！vptr指向Base的vtable）
    }
    virtual void init() {
        std::cout << "Base::init" << std::endl;
    }
    virtual ~Base() {
        std::cout << "Base析构中: ";
        cleanup();  // 调用Base::cleanup（非多态！vptr已回退）
    }
    virtual void cleanup() {
        std::cout << "Base::cleanup" << std::endl;
    }
};

class Derived : public Base {
    int* data;
public:
    Derived() : data(new int[10]) {
        std::cout << "Derived构造中: ";
        init();  // 调用Derived::init（此时vptr已指向Derived的vtable）
    }
    void init() override {
        std::cout << "Derived::init" << std::endl;
    }
    ~Derived() override {
        std::cout << "Derived析构中: ";
        cleanup();  // 调用Derived::cleanup
        delete[] data;
    }
    void cleanup() override {
        std::cout << "Derived::cleanup" << std::endl;
    }
};

int main() {
    Derived d;
    return 0;
}
```

**输出：**
```
Base构造中: Base::init       ← Base构造中调用的是Base版本
Derived构造中: Derived::init  ← Derived构造中调用的是Derived版本
Derived析构中: Derived::cleanup
Base析构中: Base::cleanup     ← Base析构中调用的是Base版本
```

### 2.2 两阶段初始化模式

```cpp
#include <iostream>
#include <memory>

// 问题：构造函数中无法多态初始化
class Base {
public:
    Base() {
        // init()如果在这里调用，无法调用Derived::init()
    }
    virtual ~Base() = default;

    // 解决方案：两阶段初始化
    void initialize() {
        // 公共初始化
        std::cout << "Base公共初始化" << std::endl;
        // 调用派生类的初始化（此时对象已完整构造）
        doInitialize();
    }

protected:
    virtual void doInitialize() {
        std::cout << "Base::doInitialize" << std::endl;
    }
};

class Derived : public Base {
public:
    Derived() {}  // 构造函数中不调用初始化

protected:
    void doInitialize() override {
        std::cout << "Derived::doInitialize" << std::endl;
        // 此时可以安全使用虚函数
    }
};

int main() {
    Derived d;
    d.initialize();  // 完整对象构造后调用，多态正常
    return 0;
}
```

### 2.3 工厂方法替代构造中的虚调用

```cpp
#include <iostream>
#include <memory>

class Connection {
protected:
    // 不在构造函数中调用init
    Connection() = default;

public:
    virtual ~Connection() = default;

    // 工厂方法：创建后调用虚函数
    template<typename T>
    static std::unique_ptr<T> create() {
        auto obj = std::make_unique<T>();
        obj->initialize();  // 此时对象已完整构造
        return obj;
    }

protected:
    virtual void initialize() {
        std::cout << "Connection::initialize" << std::endl;
    }
};

class TcpConnection : public Connection {
protected:
    void initialize() override {
        std::cout << "TcpConnection初始化（建立TCP连接）" << std::endl;
    }
};

int main() {
    auto conn = Connection::create<TcpConnection>();
    // TcpConnection初始化（建立TCP连接）
    return 0;
}
```

## 三、注意事项与常见陷阱

1. **构造函数中虚函数调用的是当前类的版本**：vptr尚未指向派生类的vtable
2. **析构函数中虚函数调用的是当前类的版本**：vptr已回退到当前类的vtable
3. **这是语言标准规定的行为**：所有编译器都遵循此规则
4. **避免在构造/析构中调用虚函数**：除非明确知道调用的是哪个版本
5. **替代方案：两阶段初始化或工厂方法**：在对象完整构造后调用虚函数
6. **如果必须在构造中初始化，使用非虚的初始化函数**：由派生类在构造体中调用
