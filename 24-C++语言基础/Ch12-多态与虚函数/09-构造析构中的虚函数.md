# 构造析构中的虚函数

## 一、概念说明

在构造函数和析构函数中调用虚函数**不会发生多态**。因为此时对象的派生类部分尚未构造或已经析构，vptr指向当前类的vtable。

## 二、具体用法

### 2.1 构造期间行为

```cpp
#include <iostream>

class Base {
public:
    Base() {
        std::cout << "Base构造中: ";
        init();  // 调用Base::init（非多态）
    }
    virtual void init() { std::cout << "Base::init" << std::endl; }
    virtual ~Base() {
        std::cout << "Base析构中: ";
        cleanup();  // 调用Base::cleanup
    }
    virtual void cleanup() { std::cout << "Base::cleanup" << std::endl; }
};

class Derived : public Base {
    int* data;
public:
    Derived() : data(new int[10]) {
        std::cout << "Derived构造中: ";
        init();  // 调用Derived::init（此时vptr已更新）
    }
    void init() override { std::cout << "Derived::init" << std::endl; }
    ~Derived() override {
        std::cout << "Derived析构中: ";
        cleanup();
        delete[] data;
    }
    void cleanup() override { std::cout << "Derived::cleanup" << std::endl; }
};

int main() {
    Derived d;
    return 0;
}
```

**输出：**
```
Base构造中: Base::init
Derived构造中: Derived::init
Derived析构中: Derived::cleanup
Base析构中: Base::cleanup
```

## 三、注意事项与常见陷阱

- 构造函数中虚函数调用的是**当前类**的版本
- 这是语言标准规定的行为，不是编译器bug
- 避免在构造/析构中调用虚函数
- 替代方案：两阶段初始化或工厂方法
- 如果必须用，了解当前调用的是哪个版本
