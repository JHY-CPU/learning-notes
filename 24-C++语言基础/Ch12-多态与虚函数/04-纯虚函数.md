# 纯虚函数

## 一、概念说明

**纯虚函数**（C++标准 §10.4）用`= 0`声明，使类成为**抽象类**（abstract class），不能被实例化。纯虚函数强制派生类提供实现，定义了接口契约。它是实现"面向接口编程"的核心机制。

### 1.1 纯虚函数 vs 普通虚函数

| 特性 | 普通虚函数 | 纯虚函数 |
|------|----------|---------|
| 声明 | `virtual void f();` | `virtual void f() = 0;` |
| 默认实现 | 必须有 | 可以有（但不自动继承） |
| 类实例化 | 可以 | 不可以（抽象类） |
| 派生类实现 | 可选 | 必须实现 |

## 二、具体用法

### 2.1 纯虚函数定义接口

```cpp
#include <iostream>
#include <vector>
#include <memory>

class Serializer {
public:
    virtual void serialize(const std::string& data) = 0;
    virtual std::string format() const = 0;
    virtual ~Serializer() = default;
};

class JsonSerializer : public Serializer {
public:
    void serialize(const std::string& data) override {
        std::cout << format() << ": { \"data\": \"" << data << "\" }" << std::endl;
    }
    std::string format() const override { return "JSON"; }
};

class XmlSerializer : public Serializer {
public:
    void serialize(const std::string& data) override {
        std::cout << format() << ": <data>" << data << "</data>" << std::endl;
    }
    std::string format() const override { return "XML"; }
};

class BinarySerializer : public Serializer {
public:
    void serialize(const std::string& data) override {
        std::cout << format() << ": [" << data.size() << " bytes]" << std::endl;
    }
    std::string format() const override { return "Binary"; }
};

void saveAll(const std::vector<std::unique_ptr<Serializer>>& serializers,
             const std::string& data) {
    for (const auto& s : serializers)
        s->serialize(data);
}

int main() {
    // Serializer s;  // 编译错误！抽象类不能实例化

    std::vector<std::unique_ptr<Serializer>> serializers;
    serializers.push_back(std::make_unique<JsonSerializer>());
    serializers.push_back(std::make_unique<XmlSerializer>());
    serializers.push_back(std::make_unique<BinarySerializer>());

    saveAll(serializers, "Hello World");
    return 0;
}
```

### 2.2 抽象类可以有非纯虚函数

```cpp
#include <iostream>
#include <cmath>

class Shape {
protected:
    std::string name;
public:
    Shape(const std::string& n) : name(n) {}

    // 纯虚函数
    virtual double area() const = 0;
    virtual double perimeter() const = 0;

    // 非纯虚函数（有默认实现）
    virtual void describe() const {
        std::cout << name << ": 面积=" << area()
                  << ", 周长=" << perimeter() << std::endl;
    }

    // 非虚函数
    std::string getName() const { return name; }

    virtual ~Shape() = default;
};

class Circle : public Shape {
    double r;
public:
    Circle(double r) : Shape("圆"), r(r) {}
    double area() const override { return M_PI * r * r; }
    double perimeter() const override { return 2 * M_PI * r; }
    // 继承describe()的默认实现
};

class Rectangle : public Shape {
    double w, h;
public:
    Rectangle(double w, double h) : Shape("矩形"), w(w), h(h) {}
    double area() const override { return w * h; }
    double perimeter() const override { return 2 * (w + h); }
    // 重写describe()
    void describe() const override {
        std::cout << name << "(" << w << "x" << h << "): 面积="
                  << area() << std::endl;
    }
};

int main() {
    Circle c(5);
    Rectangle r(3, 4);

    Shape* shapes[] = {&c, &r};
    for (auto* s : shapes)
        s->describe();

    return 0;
}
```

### 2.3 中间抽象类

```cpp
#include <iostream>

class Animal {
public:
    virtual void speak() const = 0;
    virtual void eat() const = 0;
    virtual ~Animal() = default;
};

// 中间类：实现部分纯虚函数，传递其他
class Mammal : public Animal {
public:
    void eat() const override { std::cout << "哺乳进食" << std::endl; }
    // speak()仍然是纯虚函数，传递给下一层
};

class Dog : public Mammal {
public:
    void speak() const override { std::cout << "汪汪" << std::endl; }
    // eat()继承自Mammal
};

// Mammal仍然是抽象类
// Dog实现了所有纯虚函数，可以实例化

int main() {
    // Mammal m;  // 错误：抽象类
    Dog d;
    d.speak();  // 汪汪
    d.eat();    // 哺乳进食
    return 0;
}
```

## 三、注意事项与常见陷阱

1. **包含纯虚函数的类不能实例化**：`Shape s;` 编译错误
2. **派生类必须实现所有纯虚函数才能实例化**：否则仍是抽象类
3. **纯虚函数可以有函数体**：`= 0`只表示"必须重写"，不禁止提供实现
4. **纯虚析构必须提供实现**：`= 0`后加函数体
5. **抽象类可以有数据成员和非纯虚函数**：并非只能有纯虚函数
6. **从抽象类派生的中间类可以继续不实现纯虚函数**：将责任传递给下一层
