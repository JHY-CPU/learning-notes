# 多态与智能指针

## 一、概念说明

`std::unique_ptr<Base>`和`std::shared_ptr<Base>`可以指向`Derived`对象（C++标准 §20.11.1），实现多态管理。智能指针自动管理内存，析构时正确调用派生类析构函数（即使通过基类指针删除）。

### 1.1 智能指针与多态

| 特性 | `unique_ptr<Base>` | `shared_ptr<Base>` |
|------|-------------------|-------------------|
| 所有权 | 独占 | 共享 |
| 多态 | 支持 | 支持 |
| 拷贝 | 不可 | 可以 |
| 内存开销 | 指针大小 | 指针+控制块 |
| 性能 | 更快 | 略慢（原子操作） |

## 二、具体用法

### 2.1 unique_ptr多态

```cpp
#include <iostream>
#include <memory>
#include <vector>

class Resource {
public:
    virtual void use() = 0;
    virtual std::string type() const = 0;
    virtual ~Resource() = default;
};

class Database : public Resource {
public:
    void use() override { std::cout << "使用数据库" << std::endl; }
    std::string type() const override { return "数据库"; }
    ~Database() override { std::cout << "关闭数据库连接" << std::endl; }
};

class Cache : public Resource {
public:
    void use() override { std::cout << "使用缓存" << std::endl; }
    std::string type() const override { return "缓存"; }
    ~Cache() override { std::cout << "清理缓存" << std::endl; }
};

int main() {
    // 多态容器
    std::vector<std::unique_ptr<Resource>> resources;
    resources.push_back(std::make_unique<Database>());
    resources.push_back(std::make_unique<Cache>());

    for (auto& r : resources) {
        r->use();
        std::cout << "类型: " << r->type() << std::endl;
    }

    // 自动析构，正确调用派生类析构
    std::cout << "\n=== 自动析构 ===" << std::endl;
    return 0;
}
```

### 2.2 shared_ptr多态

```cpp
#include <iostream>
#include <memory>
#include <vector>

class Base {
public:
    virtual void show() const { std::cout << "Base" << std::endl; }
    virtual ~Base() { std::cout << "~Base" << std::endl; }
};

class Derived : public Base {
    std::shared_ptr<Base> parent;  // 可选：引用父对象
public:
    Derived(std::shared_ptr<Base> p = nullptr) : parent(p) {}
    void show() const override { std::cout << "Derived" << std::endl; }
    ~Derived() override { std::cout << "~Derived" << std::endl; }
};

int main() {
    std::shared_ptr<Base> p = std::make_shared<Derived>();
    // 引用计数管理生命周期

    std::vector<std::shared_ptr<Base>> shared;
    shared.push_back(p);      // 引用计数+1
    shared.push_back(p);      // 引用计数+1

    std::cout << "引用计数: " << p.use_count() << std::endl;  // 3

    shared.clear();
    std::cout << "引用计数: " << p.use_count() << std::endl;  // 1

    // 最后一个shared_ptr销毁时自动析构
    return 0;
}
```

### 2.3 shared_ptr的自定义deleter

```cpp
#include <iostream>
#include <memory>

class Connection {
public:
    virtual ~Connection() = default;
    virtual void close() = 0;
};

class TcpConnection : public Connection {
public:
    void close() override { std::cout << "TCP关闭" << std::endl; }
    ~TcpConnection() override { std::cout << "~TcpConnection" << std::endl; }
};

int main() {
    // 自定义deleter
    auto deleter = [](Connection* conn) {
        std::cout << "自定义清理: ";
        conn->close();
        delete conn;
    };

    std::shared_ptr<Connection> conn(new TcpConnection(), deleter);
    // 当引用计数归零时，调用deleter而非直接delete
    // 这允许在析构前执行清理逻辑

    return 0;
}
```

### 2.4 weak_ptr打破循环引用

```cpp
#include <iostream>
#include <memory>

class Node {
public:
    std::shared_ptr<Node> strong;   // 强引用
    std::weak_ptr<Node> weak;       // 弱引用（打破循环）
    std::string name;

    Node(const std::string& n) : name(n) {
        std::cout << name << " 构造" << std::endl;
    }
    ~Node() { std::cout << name << " 析构" << std::endl; }
};

int main() {
    // 循环引用问题
    {
        auto a = std::make_shared<Node>("A");
        auto b = std::make_shared<Node>("B");
        a->strong = b;  // A → B（强引用）
        b->strong = a;  // B → A（强引用）→ 循环引用！
        // a和b永远不会析构
    }
    std::cout << "--- 循环引用：A和B不会析构 ---" << std::endl;

    // 使用weak_ptr打破循环
    {
        auto a = std::make_shared<Node>("A2");
        auto b = std::make_shared<Node>("B2");
        a->strong = b;  // A2 → B2（强引用）
        b->weak = a;    // B2 → A2（弱引用）
    }
    std::cout << "--- weak_ptr：正确析构 ---" << std::endl;

    return 0;
}
```

## 三、注意事项与常见陷阱

1. **`shared_ptr<Base>`的析构正确调用派生类析构**：无需虚析构（但应保持）
2. **`unique_ptr<Base>`转移所有权用`std::move`**：独占所有权语义
3. **循环引用用`std::weak_ptr`打破**：`weak_ptr.lock()`获取`shared_ptr`
4. **`make_shared`比`new`更高效**：单次内存分配
5. **自定义deleter适用于需要清理逻辑的场景**：如关闭连接、释放锁
6. **`enable_shared_from_this`允许对象获取自身的`shared_ptr`**：需要对象已被shared_ptr管理
