# 虚函数性能分析

## 一、概念说明

虚函数调用通过vtable间接跳转，比直接调用多一次指针解引用（读vptr → 读vtable条目 → 跳转）。但在现代CPU上，这个开销通常可以忽略（1-3纳秒）。编译器还可以通过**devirtualization**（去虚化）优化消除虚调用开销。

### 1.1 性能对比概览

| 调用方式 | 开销 | 可内联 | 典型延迟 |
|----------|------|--------|---------|
| 直接调用 | 无 | 可以 | ~0 ns |
| 内联函数 | 无 | 已内联 | ~0 ns |
| 虚函数调用 | vptr+vtable间接跳转 | 通常不能 | ~1-3 ns |
| 函数指针调用 | 类似虚函数 | 通常不能 | ~1-3 ns |

## 二、具体用法

### 2.1 性能对比测试

```cpp
#include <iostream>
#include <chrono>

class Base {
public:
    virtual int virtualFunc(int x) { return x * 2; }
    int directFunc(int x) { return x * 2; }
    virtual ~Base() = default;
};

class Derived : public Base {
public:
    int virtualFunc(int x) override { return x * 3; }
};

int main() {
    Derived d;
    Base* p = &d;
    const int N = 100000000;
    volatile int result = 0;

    // 虚函数调用
    auto start = std::chrono::high_resolution_clock::now();
    for (int i = 0; i < N; ++i)
        result = p->virtualFunc(i);
    auto end = std::chrono::high_resolution_clock::now();
    auto vtime = std::chrono::duration_cast<std::chrono::milliseconds>(end - start);

    // 直接调用
    start = std::chrono::high_resolution_clock::now();
    for (int i = 0; i < N; ++i)
        result = d.directFunc(i);
    end = std::chrono::high_resolution_clock::now();
    auto dtime = std::chrono::duration_cast<std::chrono::milliseconds>(end - start);

    std::cout << "虚函数: " << vtime.count() << "ms" << std::endl;
    std::cout << "直接调用: " << dtime.count() << "ms" << std::endl;
    // 典型差异：虚函数比直接调用慢10-50%
    return 0;
}
```

### 2.2 Devirtualization优化

```cpp
#include <iostream>

class Base {
public:
    virtual int compute() const { return 1; }
    virtual ~Base() = default;
};

class Derived final : public Base {
public:
    int compute() const override { return 2; }
};

// 编译器可以通过以下方式devirtualize：
// 1. final类：知道没有派生类
// 2. final函数：知道不能被进一步重写
// 3. 局部对象：编译器看到实际类型
// 4. LTO（链接时优化）：全局分析

int processFinal(const Derived& d) {
    // Derived是final类 → 编译器可以devirtualize
    return d.compute();  // 可能优化为直接调用
}

int processLocal() {
    Derived d;
    Base* p = &d;
    // 局部对象 → 编译器看到实际类型 → devirtualize
    return p->compute();
}

int main() {
    Derived d;
    std::cout << processFinal(d) << std::endl;
    std::cout << processLocal() << std::endl;
    return 0;
}
```

### 2.3 性能优化策略

```cpp
#include <iostream>

// 策略1：使用final
class Widget final {
public:
    virtual void render() final { std::cout << "render" << std::endl; }
    virtual ~Widget() = default;
};

// 策略2：使用模板替代虚函数（CRTP）
template<typename Derived>
class CRTPBase {
public:
    void process() {
        static_cast<Derived*>(this)->doProcess();
    }
};

class CRTPDerived : public CRTPBase<CRTPDerived> {
public:
    void doProcess() { std::cout << "CRTP process" << std::endl; }
};

// 策略3：使用std::variant替代虚函数
#include <variant>
struct Circle { double r; double area() const { return 3.14159*r*r; } };
struct Square { double s; double area() const { return s*s; } };
using Shape = std::variant<Circle, Square>;

double totalArea(const std::vector<Shape>& shapes) {
    double sum = 0;
    for (const auto& s : shapes)
        sum += std::visit([](const auto& shape) { return shape.area(); }, s);
    return sum;
}

int main() {
    Widget w;
    w.render();  // devirtualized

    CRTPDerived cd;
    cd.process();  // 编译期绑定，可内联

    std::vector<Shape> shapes = {Circle{5}, Square{3}};
    std::cout << "总面积: " << totalArea(shapes) << std::endl;
    return 0;
}
```

## 三、注意事项与常见陷阱

1. **虚函数的开销主要是间接跳转**：一次指针解引用（vptr → vtable → 函数地址）
2. **现代CPU分支预测通常能很好处理虚调用**：开销通常可忽略
3. **`final`类/函数允许编译器devirtualize**：消除查表开销
4. **内联在虚函数中通常不可能**：编译器不知道实际类型
5. **性能关键路径考虑模板（静态多态）替代**：CRTP或std::variant
6. **先测量再优化**：虚函数开销通常不是性能瓶颈
