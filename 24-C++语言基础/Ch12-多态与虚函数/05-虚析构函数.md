# 虚析构函数

## 一、概念说明

基类析构函数必须声明为`virtual`（C++标准 §10.4），否则通过基类指针删除派生类对象时，只调用基类析构函数，导致派生类资源泄漏。虚析构是保证多态对象正确销毁的基础。

### 1.1 虚析构的必要性

| 场景 | 基类析构非virtual | 基类析构virtual |
|------|------------------|---------------|
| `Base* p = new Derived(); delete p;` | 只调用~Base()（**泄漏**） | 调用~Derived()再~Base() |
| `unique_ptr<Base> p = make_unique<Derived>();` | 只调用~Base()（**泄漏**） | 正确调用~Derived() |
| `Derived d;`（栈对象） | 正确 | 正确 |

## 二、具体用法

### 2.1 虚析构的必要性

```cpp
#include <iostream>

class Base {
public:
    Base() { std::cout << "Base构造" << std::endl; }
    virtual ~Base() { std::cout << "Base析构" << std::endl; }
};

class Derived : public Base {
    int* data;
public:
    Derived() : data(new int[100]) {
        std::cout << "Derived构造, 分配内存" << std::endl;
    }
    ~Derived() override {
        delete[] data;
        std::cout << "Derived析构, 释放内存" << std::endl;
    }
};

int main() {
    Base* p = new Derived();
    // 构造：Base构造 → Derived构造
    delete p;
    // 析构：Derived析构 → Base析构（虚析构保证）
    // 如果~Base()不是virtual，只调用~Base()，data泄漏！
    return 0;
}
```

### 2.2 没有虚析构的后果

```cpp
#include <iostream>

class BadBase {
public:
    BadBase() = default;
    ~BadBase() { std::cout << "~BadBase()" << std::endl; }  // 非虚！
};

class BadDerived : public BadBase {
    int* resource;
public:
    BadDerived() : resource(new int[1000]) {}
    ~BadDerived() {
        delete[] resource;
        std::cout << "~BadDerived() - 释放资源" << std::endl;
    }
};

void demonstrateBug() {
    BadBase* p = new BadDerived();
    delete p;
    // 输出：~BadBase()
    // ~BadDerived()未被调用！resource内存泄漏！
    // 1000个int永远不会被释放
}
```

### 2.3 纯虚析构函数

```cpp
#include <iostream>

class Interface {
public:
    // 纯虚析构：使类成为抽象类
    virtual ~Interface() = 0;
};

// 纯虚析构必须有实现！
Interface::~Interface() {
    std::cout << "~Interface()" << std::endl;
}

class Impl : public Interface {
    int* data;
public:
    Impl() : data(new int[42]) {}
    ~Impl() override {
        delete data;
        std::cout << "~Impl()" << std::endl;
    }
};

int main() {
    // Interface i;  // 错误：抽象类
    Interface* p = new Impl();
    delete p;
    // ~Impl() → ~Interface()
    return 0;
}
```

### 2.4 final类与虚析构

```cpp
#include <iostream>

// final类：没有派生类
class Widget final {
public:
    virtual void show() { std::cout << "Widget" << std::endl; }
    // 虚析构仍然推荐（保持一致性）
    virtual ~Widget() = default;
    // 由于final，编译器知道没有派生类
    // show()可以被devirtualize
};

// final类不需要虚析构的特殊情况
class Simple final {
    int x;
public:
    Simple(int x) : x(x) {}
    // 没有虚函数 → 没有vtable → sizeof更小
    // 不需要virtual ~Simple() = default;
};
```

## 三、注意事项与常见陷阱

1. **如果类有虚函数，析构函数必须是virtual的**：这是基本规则
2. **虚析构的开销很小**：只增加一个vtable条目（一个指针的大小）
3. **纯虚析构函数必须提供实现**：`= 0`后加函数体
4. **`final`类不需要虚析构但保持一致性是好习惯**：有虚函数就加virtual析构
5. **C++11起析构函数隐式`noexcept`**：析构中不应抛异常
6. **`shared_ptr`的deleter不依赖虚析构**：但仍应保持virtual一致性
