# 纯虚函数实现

## 一、概念说明

纯虚函数虽然标记为`= 0`（C++标准 §10.4），但**可以提供函数体**。派生类仍必须显式重写，但可以通过`Base::func()`调用基类提供的默认实现。这允许基类提供公共逻辑，同时强制派生类做出是否使用默认实现的决定。

### 1.1 纯虚函数实现的方式

| 方式 | 语法 | 说明 |
|------|------|------|
| 类外定义 | `void Base::f() { ... }` | 传统方式 |
| 类内定义 | `virtual void f() = 0 { ... }` | C++11起支持 |
| 受保护的非虚函数 | `void defaultF() { ... }` | NVI风格 |

## 二、具体用法

### 2.1 纯虚函数带默认实现

```cpp
#include <iostream>

class Logger {
public:
    // 纯虚函数声明
    virtual void log(const std::string& msg) = 0;
    virtual ~Logger() = default;

    // 提供默认实现的辅助函数
    void defaultLog(const std::string& msg) {
        std::cout << "[LOG] " << msg << std::endl;
    }
};

class ConsoleLogger : public Logger {
public:
    void log(const std::string& msg) override {
        defaultLog(msg);  // 使用基类提供的默认逻辑
    }
};

class FileLogger : public Logger {
public:
    void log(const std::string& msg) override {
        std::cout << "[FILE] " << msg << std::endl;  // 完全自定义
    }
};

class TimestampLogger : public Logger {
public:
    void log(const std::string& msg) override {
        defaultLog(msg);  // 复用默认逻辑
        std::cout << "  (带时间戳)" << std::endl;  // 添加额外逻辑
    }
};

int main() {
    ConsoleLogger cl;
    FileLogger fl;
    TimestampLogger tl;

    cl.log("控制台消息");
    fl.log("文件消息");
    tl.log("带时间戳消息");

    return 0;
}
```

### 2.2 类内定义纯虚函数体（C++11）

```cpp
#include <iostream>

class Abstract {
public:
    // C++11：类内定义纯虚函数体
    virtual void pure() = 0;

    // 纯虚函数带默认实现
    virtual void commonLogic() = 0 {
        std::cout << "Abstract::commonLogic() - 公共逻辑" << std::endl;
    }

    virtual ~Abstract() = default;
};

class Concrete : public Abstract {
public:
    void pure() override {
        std::cout << "Concrete::pure()" << std::endl;
    }
    void commonLogic() override {
        Abstract::commonLogic();  // 显式调用基类实现
        std::cout << "Concrete的附加逻辑" << std::endl;
    }
};

class Simple : public Abstract {
public:
    void pure() override {
        std::cout << "Simple::pure()" << std::endl;
    }
    void commonLogic() override {
        std::cout << "Simple完全自定义的实现" << std::endl;
    }
};

int main() {
    Concrete c;
    c.commonLogic();
    // Abstract::commonLogic() - 公共逻辑
    // Concrete的附加逻辑

    Simple s;
    s.commonLogic();
    // Simple完全自定义的实现

    return 0;
}
```

### 2.3 模板方法模式结合纯虚函数

```cpp
#include <iostream>

class Algorithm {
public:
    // 模板方法：定义算法骨架
    void execute() {
        step1();
        step2();  // 纯虚函数：必须实现
        step3();  // 纯虚函数但有默认实现
        step4();
    }
    virtual ~Algorithm() = default;

protected:
    void step1() { std::cout << "步骤1: 初始化（固定）" << std::endl; }
    virtual void step2() = 0;  // 必须实现
    virtual void step3() = 0 {
        std::cout << "步骤3: 默认清理" << std::endl;  // 默认实现
    }
    void step4() { std::cout << "步骤4: 完成（固定）" << std::endl; }
};

class MyAlgorithm : public Algorithm {
protected:
    void step2() override {
        std::cout << "步骤2: 自定义处理" << std::endl;
    }
    void step3() override {
        Algorithm::step3();  // 使用默认实现
        std::cout << "步骤3: 额外清理" << std::endl;
    }
};

int main() {
    MyAlgorithm algo;
    algo.execute();
    return 0;
}
```

## 三、注意事项与常见陷阱

1. **纯虚函数的函数体必须在类外定义**：或在类内`= 0 { }`（C++11）
2. **派生类仍然必须重写纯虚函数**：即使基类提供了函数体
3. **纯虚析构函数必须提供实现**：`= 0`后加函数体
4. **提供默认实现可以减少代码重复**：派生类可选择使用
5. **基类的纯虚函数体只能通过`Base::func()`显式调用**：不会自动继承
6. **使用纯虚函数+默认实现比普通虚函数更能强制派生类思考**：设计意图更明确
