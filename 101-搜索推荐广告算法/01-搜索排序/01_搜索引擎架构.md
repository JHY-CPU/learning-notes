# 搜索引擎架构 - 搜索推荐广告算法


## 1. 搜索引擎整体架构


搜索引擎是一个复杂的系统，核心流程包括：网页采集（Crawling）→ 索引构建（Indexing）→ 查询处理（Query Processing）→ 排序（Ranking）。


| 阶段 | 功能 | 关键技术 |
| --- | --- | --- |
| 采集（Crawling） | 从互联网抓取网页内容 | 爬虫框架、URL调度、去重 |
| 索引（Indexing） | 对网页内容建立倒排索引 | 分词、倒排索引、压缩存储 |
| 查询理解 | 理解用户搜索意图 | 意图识别、Query改写、同义词扩展 |
| 召回（Retrieval） | 从索引中快速找到候选文档 | 倒排索引检索、语义检索 |
| 排序（Ranking） | 对候选文档进行精细排序 | BM25、LTR、语义相关性模型 |
| 展示（Rendering） | 结果呈现给用户 | 摘要生成、高亮、相关搜索 |


> **Note:** 现代搜索引擎通常采用多阶段排序架构：第一阶段用倒排索引快速召回Top-K文档（毫秒级），第二阶段用复杂模型精排（百毫秒级），保证效果和效率的平衡。


## 2. 倒排索引原理


倒排索引（Inverted Index）是搜索引擎的核心数据结构，建立从词项（Term）到包含该词项的文档列表的映射。


### 正排索引 vs 倒排索引


| 索引类型 | 结构 | 查询方式 | 效率 |
| --- | --- | --- | --- |
| 正排索引 | 文档ID → 文档内容 | 遍历所有文档 | O(N)，极慢 |
| 倒排索引 | 词项 → 文档ID列表 | 直接查找词项 | O(1)查找，极快 |


> **Example:** **倒排索引示例：**
>
>
> Doc1: "机器学习是人工智能的核心"
>
>
> Doc2: "深度学习是机器学习的子领域"
>
>
>
>
> 倒排表：
>
>
> "机器" → [Doc1, Doc2]
>
>
> "学习" → [Doc1, Doc2]
>
>
> "人工智能" → [Doc1]
>
>
> "深度" → [Doc2]
>
>
> "子领域" → [Doc2]


### 倒排表的附加信息


- **词频（TF）：**
   词项在文档中出现的次数，用于BM25等排序算法
- **位置信息（Position）：**
   词项在文档中的位置，用于短语查询
- **文档频率（DF）：**
   包含该词项的文档数量，用于IDF计算


## 3. 分词与词项处理


### 中文分词


- **基于规则：**
   最大匹配法（正向/逆向/双向最大匹配）
- **基于统计：**
   HMM、CRF序列标注
- **基于深度学习：**
   BERT分词、SentencePiece
- **常用工具：**
   jieba、HanLP、LAC（百度）


### 词项处理


- **大小写归一化：**
   统一转为小写
- **词干提取（Stemming）：**
   将词还原为词干形式（running→run）
- **词形还原（Lemmatization）：**
   基于词典的还原（better→good）
- **停用词过滤：**
   移除"的"、"是"、"在"等高频无意义词
- **同义词扩展：**
   将查询词扩展为同义词集合


## 4. BM25算法


BM25（Best Matching 25）是信息检索中最经典的排序算法，基于TF-IDF思想的改进版本。


$$
BM25公式：
                BM25(D, Q) = Σ IDF(q_i) × (f(q_i, D) × (k1 + 1)) / (f(q_i, D) + k1 × (1 - b + b × |D|/avg_dl))
                其中：
                f(q_i, D) = 词项q_i在文档D中的词频
                |D| = 文档D的长度
                avg_dl = 文档集合的平均长度
                k1 = 词频饱和参数（通常1.2~2.0）
                b = 文档长度归一化参数（通常0.75）
                IDF(q_i) = log((N - df(q_i) + 0.5) / (df(q_i) + 0.5) + 1)
$$


### BM25 vs TF-IDF的改进


| 改进点 | TF-IDF | BM25 |
| --- | --- | --- |
| 词频处理 | 线性增长 | 有饱和效应（k1参数控制） |
| 文档长度 | 无归一化 | 长度归一化（b参数控制） |
| IDF形式 | log(N/df) | 改进的IDF公式，避免负值 |


```
import math
from collections import Counter

class BM25:
    def __init__(self, documents, k1=1.5, b=0.75):
        self.k1 = k1
        self.b = b
        self.documents = documents
        self.N = len(documents)
        self.avg_dl = sum(len(d) for d in documents) / self.N

        # 计算文档频率
        self.df = Counter()
        for doc in documents:
            for term in set(doc):
                self.df[term] += 1

    def idf(self, term):
        """计算IDF"""
        df = self.df.get(term, 0)
        return math.log((self.N - df + 0.5) / (df + 0.5) + 1)

    def score(self, query, doc_idx):
        """计算BM25得分"""
        doc = self.documents[doc_idx]
        doc_len = len(doc)
        tf = Counter(doc)
        score = 0
        for term in query:
            if term in tf:
                numerator = tf[term] * (self.k1 + 1)
                denominator = tf[term] + self.k1 * (
                    1 - self.b + self.b * doc_len / self.avg_dl
                )
                score += self.idf(term) * numerator / denominator
        return score

    def search(self, query, top_k=10):
        """搜索并排序"""
        scores = [(i, self.score(query, i)) for i in range(self.N)]
        scores.sort(key=lambda x: x[1], reverse=True)
        return scores[:top_k]
```


## 5. Query理解


### 意图识别


| 意图类型 | 描述 | 示例 |
| --- | --- | --- |
| 导航型 | 寻找特定网站 | "淘宝官网" |
| 信息型 | 获取知识或答案 | "什么是机器学习" |
| 交易型 | 完成某项操作 | "下载微信" |
| 本地型 | 寻找本地服务 | "附近餐厅" |


### Query改写技术


- **拼写纠错：**
   "机习学习" → "机器学习"
- **同义词扩展：**
   "电脑" → "电脑 OR 计算机 OR PC"
- **查询扩展：**
   添加相关词提升召回
- **查询缩减：**
   移除无关词提升精确度
- **实体识别：**
   识别查询中的命名实体


## 6. 搜索相关性评估指标


| 指标 | 全称 | 计算方式 | 特点 |
| --- | --- | --- | --- |
| NDCG | 归一化折损累积增益 | DCG / IDCG | 考虑排序位置和多级相关性 |
| MAP | 平均准确率均值 | 各查询AP的均值 | 二值相关性下的经典指标 |
| MRR | 平均倒数排名 | 1/首个相关文档排名 | 关注第一个结果的排名 |


$$
NDCG@K：
                DCG@K = Σ(i=1→K) rel_i / log2(i+1)
                NDCG@K = DCG@K / IDCG@K
                其中 rel_i 为第i个位置文档的相关性等级，IDCG为理想排序下的DCG
$$


> **Example:** **NDCG计算示例：**
> 假设相关性等级为[3, 2, 3, 0, 1, 2]
>
>
> DCG@3 = 3/log2(2) + 2/log2(3) + 3/log2(4) = 3 + 1.26 + 1.5 = 5.76
>
>
> 理想排序[3, 3, 2]：IDCG@3 = 3/1 + 3/1.58 + 2/2 = 5.90
>
>
> NDCG@3 = 5.76 / 5.90 = 0.976


## 总结


- 搜索引擎架构包括采集、索引、查询理解、召回、排序、展示六个核心阶段
- 倒排索引是搜索引擎的核心数据结构，建立词项到文档列表的映射
- 中文分词是中文搜索的基础，jieba是最常用的工具
- BM25是经典排序算法，通过词频饱和和文档长度归一化改进TF-IDF
- Query理解包括意图识别、拼写纠错、同义词扩展等技术
- NDCG是搜索排序最常用的评估指标，考虑了排序位置和多级相关性
